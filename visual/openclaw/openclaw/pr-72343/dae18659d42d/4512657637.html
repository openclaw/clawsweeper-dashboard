PR #72343: Fresh CLI Installs Start Onboarding

ELI5: this PR changes what happens when someone runs bare openclaw. If the install looks fresh, it starts classic setup. If the user already has authored config, it keeps opening Crestodian. Invalid configs still stay on the repair path.

Maintainer Summary

Plain-language change: a brand-new user running openclaw gets the setup wizard first. A configured user running openclaw still gets Crestodian.

Fresh Install

Missing config, empty config, or metadata-only config.

Route: classic onboarding.

->

Configured Install

Config has authored settings.

Route: Crestodian.

->

Invalid Config

Existing config needs repair.

Route: stays away from onboarding so repair remains reachable.
