## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Use a stable cursor for pagination:** `src/components/useTimelineRouteData.ts:130-134`
  - body: `loadMore` uses only the last item's `createdAt` as the next `until` cursor, while `/api/query` filters with `t.created_at < ?` and orders only by `t.created_at desc`. If more than one tweet shares the page-boundary timestamp, any same-timestamp rows not included in the first page are permanently skipped. Please paginate with a deterministic cursor such as `created_at` plus tweet id, and cover that boundary in tests.
  - confidence: 0.86
- **[P2] Ignore stale Load more responses after filters change:** `src/components/useTimelineRouteData.ts:136-138`
  - body: A pending `loadMore` request always appends into the current `items` state when it resolves. If the user clicks Load more and then changes search, reply filter, account, or collection before the request finishes, the old page can be appended to the newly loaded feed. Add an abort/request identity guard like the initial load path before committing the appended page.
  - confidence: 0.78
