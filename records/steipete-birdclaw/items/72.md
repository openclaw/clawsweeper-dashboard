The issue requests clearer selected-state styling and semantics for the Today time-range buttons, plus future custom or preset digest ranges.

## Best Possible Solution

Split the work: land a focused active styling and ARIA-state improvement for the existing Today buttons, and scope custom or saved ranges separately against the existing since/until backend.

No full high-confidence visual reproduction yet. Source inspection does confirm the structural gaps: the Today buttons lack pressed/selected ARIA state and the active styling is the shared subtle segment class.

## Solution Assessment

Yes for the narrow active-state half: stronger selected styling plus `aria-pressed` on the existing buttons is the smallest maintainable repair. The custom range and saved preset ideas should be scoped separately by maintainers.
