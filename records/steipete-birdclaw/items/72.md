## Summary

Keep open: current main still lacks a semantic pressed/selected state for the Today period buttons, uses only the shared subtle active segment styling, and has no Today UI for custom ranges even though the digest backend accepts explicit windows.

## What This Changes

Requests a clearer active/selected state for the Today view period selector plus follow-up support for custom time ranges or saved presets.

## Best Possible Solution

Land a focused selected styling and aria-pressed fix for the existing period buttons, then split custom ranges/presets into a maintainer-scoped enhancement that reuses the existing since/until backend support.

## Reproduction Assessment

Source inspection gives a medium-confidence reproduction path for the semantic gap: the Today period buttons only change classes and expose no pressed/selected ARIA state. The visual contrast claim still needs live browser or screenshot proof, and custom ranges are an enhancement rather than a failing behavior reproduction.

## Solution Assessment

Partly: the proposed active-state styling plus aria-pressed direction is a narrow maintainable fix for the current selector. The custom range and saved preset ideas should remain separate until maintainers choose the Today-route UX.
