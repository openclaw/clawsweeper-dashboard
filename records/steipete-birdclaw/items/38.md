## Summary

Keep open. Current main still documents the missing `bird auth import-cookies` setup path, while Birdclaw itself only discovers and runs a `bird` executable via PATH/env/config, so this needs a small docs/support fix rather than cleanup closure.

## What This Changes

The issue asks how to configure Birdclaw authentication when the documented `bird auth import-cookies` command is missing from the installed `bird` CLI.

## Best Possible Solution

Update the setup docs to remove or qualify the stale `bird auth import-cookies` path, explain that Birdclaw uses a `bird` binary via PATH/`BIRDCLAW_BIRD_COMMAND`/`mentions.birdCommand`, and point cookie/env and xurl setup to the supported docs.

## Reproduction Assessment

Yes. Current source docs still contain the exact `bird auth import-cookies` instruction, and the public Homebrew tap inspection did not show a `bird.rb` formula; I did not run the external `bird` 0.8.0 binary itself.

## Solution Assessment

Yes. A docs/support correction is the narrow maintainable fix because current Birdclaw code already supports binary-path configuration, while importing a separate `bird config.json5` would be a new capability.
