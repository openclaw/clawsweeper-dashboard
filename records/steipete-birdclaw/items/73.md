Keep open: current main still has manual-only web sync controls, and the requested default-off scheduler, interval setting, status display, persistence, and failure backoff remain unimplemented. This is a feature and product-direction item rather than a broken existing contract.

## What This Changes

The issue requests an optional web auto-sync mode with per-page intervals, local persistence, status text, running-job de-dupe, and failure backoff for timeline-like web pages.

## Best Possible Solution

If maintainers approve the direction, implement a default-off scheduler that reuses `/api/sync` and `postSync`, preserves server de-dupe, persists per-page intervals locally, shows last/next/error status, backs off after failures, and adds focused hook/component tests.
