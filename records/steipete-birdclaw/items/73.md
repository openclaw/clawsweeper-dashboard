Request adds a default-off web auto-sync control with per-page interval, status/error display, backoff, in-flight de-dupe, and local persistence around the existing sync UI.

## Best Possible Solution

If maintainers approve the product direction, add a default-off page-local auto-sync hook/control that reuses `/api/sync`, preserves server de-dupe, persists interval settings locally, and covers success, in-flight, error, and backoff states with focused component/route tests.
