## Summary

Keep open: current main still has manual web sync controls and separate scheduler-friendly CLI jobs, but no opt-in web auto-sync control with persisted interval, status, and backoff. This is a feature/product decision rather than a broken existing contract.

## What This Changes

Request to add default-off web UI auto-sync controls with per-page interval, persisted local setting, status display, running-job de-dupe, and failure backoff.

## Best Possible Solution

If maintainers approve the direction, add a default-off web scheduler that reuses /api/sync, preserves server de-dupe, persists scoped intervals locally, and covers status/backoff behavior with focused component and route tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms the requested interval scheduler, persisted setting, and backoff/status UI are absent from the current manual sync flow.

## Solution Assessment

Unclear until maintainers choose the product shape. A client-side scheduler over postSync is a bounded first implementation, but existing launchd jobs and docs/spec.md's broader serve-level background-sync note mean the permanent surface needs product judgment.
