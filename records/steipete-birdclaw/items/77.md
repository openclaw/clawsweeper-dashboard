Keep open: the feature is useful, but the PR can export a partially streamed failed digest as a completed PDF, and the external draft does not include real browser or print-output proof.

## What This Changes

The PR adds an Export PDF button to the Today digest route, print-only PDF metadata/styles, and a route test that exercises `window.print`.

## Best Possible Solution

Use the terminal digest result as export readiness, preserve current partial/error feedback, and verify the completed print output in a real browser before merge.

## Reproduction Assessment

Yes, from source inspection: a stream can emit markdown deltas, later fail, and leave `loading` false while `result` is still null. The PR enables export from non-empty markdown alone, so that partial failed digest can be exported.

## Solution Assessment

No; the feature direction is plausible, but readiness should be tied to `result` rather than streamed markdown, with a delta-then-error regression test and real browser proof.
