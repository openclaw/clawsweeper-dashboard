## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve xurl OAuth2 when bearer env vars exist:** `src/lib/xurl.ts:451-452`
  - body: This branch sends every `/2/` JSON call through `runBearerJsonCommandEffect` whenever a bearer env var is set. That includes calls made by `runOAuth2JsonCommandEffect` after it has selected `--auth oauth2` and `--username`, because those args still contain a `/2/` endpoint. Users with a generic bearer token in their shell would silently bypass xurl for DMs, bookmarks, mentions, and other account-scoped reads, so please keep OAuth2-selected paths on xurl unless bearer mode is explicitly requested for that call.
  - confidence: 0.88
- **[P2] Honor command options in bearer fetches:** `src/lib/xurl.ts:398-415`
  - body: The bearer helper only receives `args` and `attempt`, so it drops the existing `timeoutMs`, `signal`, and `onAttempt` behavior that `runJsonCommandEffect` applies to xurl below. Callers such as search and tweet lookups rely on those options for cancellation, deadlines, and retry telemetry; with bearer configured, a hung fetch can ignore the caller's abort/deadline and observers never see attempts. Pass the options into the bearer path and wire them through fetch/retry handling.
  - confidence: 0.8
