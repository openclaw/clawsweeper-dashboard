- **[P1] Keep bearer status from masquerading as xurl:** `src/lib/xurl.ts:229-235`
  - body: This returns `installed: true` and `availableTransport: "xurl"` solely because a bearer env var exists, before checking the xurl binary or xurl auth.   - confidence: 0.88

  - body: Changing the repo check to require `repoPath` to equal `--show-toplevel` makes an existing backup path inside a parent Git worktree look like a non-repo, after which setup can run `git init` in that subdirectory. That changes repo ownership and remote behavior for existing backup users; preserve the old layout or fail with a documented migration before initializing a nested repo.
