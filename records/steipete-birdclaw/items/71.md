## Summary

Keep open: current main and v0.8.5 still omit avatar fields when normalizing bird tweet authors, so profile upserts cannot persist the available GraphQL avatar URL; no duplicate or open fix PR was found.

## What This Changes

Live bird sync should persist avatar URLs from GraphQL user payloads into `profiles.avatar_url` for timeline, likes, bookmarks, and mentions authors.

## Best Possible Solution

Map bird/GraphQL avatar fields into the existing Xurl-compatible `profile_image_url` field before `ingestTweetPayload`, preserve existing avatar URLs when absent, and cover bird timeline, likes/bookmarks, and mentions with regression tests.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: bird tweet users are normalized without an avatar field, and profile persistence only writes avatars from `profile_image_url`. I did not run a live account-backed `bird` reproduction.

## Solution Assessment

Yes; the narrow maintainable fix is to translate the available bird/GraphQL avatar field into the existing `profile_image_url` model rather than adding a parallel persistence path.
