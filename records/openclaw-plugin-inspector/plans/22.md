## Summary

This should stay open: current main still uses a flag-unaware first non-hyphen token search for capture entrypoints, the flag-first case is valid, and I found no duplicate or fixing PR.

## Plan

Fix capture entrypoint positional parsing so values consumed by flags are not mistaken for the plugin entrypoint. Add a dependency-free helper or local function that skips operands for value-taking capture flags such as `--output`, `--plugin-root`, `--root` where supported, and `--sdk`; apply it to `src/capture-cli.js` and `src/cli.js` `runCapture`. Add regression tests for `--output <out> <entrypoint>` ordering and for `--output <path>` with no positional entrypoint failing with `capture requires an entrypoint path` without writing the output file. Preserve offline behavior, report field names, finding codes, package version, and publishing metadata; add an Unreleased Fixed changelog entry if the repo expects one for user-facing CLI fixes.
