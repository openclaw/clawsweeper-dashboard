## Summary

Keep open: current main still selects the first non-flag token as the capture entrypoint, so a value passed to `--output`, `--plugin-root`, or `--sdk` can be treated as plugin code before the intended positional path is considered.

## Plan

Fix capture entrypoint parsing so values consumed by known value-taking flags (`--output`, `--plugin-root`, and `--sdk`) are not selected as the positional entrypoint. Do not add a runtime CLI parsing dependency, do not change stable report field names or finding codes, and preserve existing `--mock-sdk`/`--real-sdk` behavior. Add focused regression coverage for flags before the entrypoint and for `--output <path>` without a positional entrypoint so the CLI errors instead of treating the output path as plugin code. Add a concise `CHANGELOG.md` fix entry for the user-facing CLI behavior; do not bump or publish the npm package.
