## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Force batch scans to stay static by default:** `src/batch.js:111`
  - body: `runPluginCheck` falls back to each plugin's `capture.runtime` config and allows execution when the existing environment opt-in is set. Because this new command is documented for exported ClawHub corpora, batch scans should pass `capture: false` unless there is an explicit batch runtime opt-in, otherwise an untrusted plugin can make the corpus analysis import its code.
  - confidence: 0.86
- **[P2] Reject invalid concurrency values:** `src/cli.js:63`
  - body: `Number(readFlag(...))` can produce `NaN`; that value reaches `runWithConcurrency`, where `Array.from({ length: NaN })` starts no workers. A typo like `--concurrency nope` can therefore report zero inspected plugins and let `--check` pass, so validate a finite positive integer before running.
  - confidence: 0.9
