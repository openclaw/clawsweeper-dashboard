## Summary

Keep open: current main and v0.3.15 still directly rewrite existing package.json files in both reported paths, and the linked candidate fix remains open and unmerged.

## What This Changes

The issue asks for package.json rewrites in init --scripts and prune-workspace-dev-deps to be made atomic so interrupts cannot leave a corrupt manifest.

## Best Possible Solution

Review and either land https://github.com/openclaw/plugin-inspector/pull/41 with adequate proof and tests, or replace it with an equivalent dependency-light atomic package.json writer for both manifest rewrite paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live current-main corruption run was reproduced here, but source inspection confirms both reported paths rewrite an existing package.json with fs.writeFile and a fault-injection test should exercise the interrupt window.

## Solution Assessment

Is this the best way to solve the issue? Yes in direction: a dependency-light sibling temp file plus fsync and rename is the narrow maintainable fix, with the linked PR currently owning that implementation path.
