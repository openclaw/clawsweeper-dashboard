## Summary

Keep open: current main still has the capture argument parsing bug, the source-level reproduction matches the report, and I found no merged fix or canonical duplicate owning the work.

## What This Changes

The issue reports that capture CLI positional parsing treats flag values such as `--output ./out.json` as the entrypoint instead of the intended plugin path.

## Best Possible Solution

Add dependency-light, value-flag-aware positional parsing for capture entrypoints, preserve the documented positional-first flow, and cover flags-before-entrypoint plus missing-entrypoint overwrite cases.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction: the current predicate returns `./out.json` for `--output ./out.json ./my-plugin.js`. I did not run a full capture because that path imports plugin code and may write artifacts.

## Solution Assessment

Yes, the narrowest maintainable fix is to exclude values consumed by known value-taking flags before selecting the positional entrypoint, with regression coverage and no new runtime dependency.
