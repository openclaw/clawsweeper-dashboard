- `impact:data-loss`: The omitted positional argument case can overwrite the file passed to `--output`, including a plugin source file, with capture JSON.

## Summary

Keep open: current main still misidentifies a value-taking flag's argument as the capture entrypoint, and a scratch reproduction confirmed both the flag-before-entrypoint failure and the source-overwrite case.

## What This Changes

Fix capture CLI parsing so positional entrypoint detection ignores values consumed by flags like `--output`, `--plugin-root`, and `--sdk`.

## Best Possible Solution

Add a small shared argument helper or equivalent local parser that skips values consumed by value-taking flags, then cover both capture entrypoints and the missing-positional case without adding runtime dependencies.

## Reproduction Assessment

Yes. A scratch fixture on current main reproduced `--output ./out.json ./src/index.js` importing `out.json` instead of the plugin, and reproduced `--output ./src/index.js` overwriting the scratch source file with capture JSON.

## Solution Assessment

Yes. The best fix is a narrow CLI parsing repair that excludes values consumed by `--output`, `--plugin-root`, and `--sdk`, with regression coverage for both helper and public capture paths.
