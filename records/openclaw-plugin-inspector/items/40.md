## Summary

Keep open: current main and v0.3.15 still directly rewrite existing package.json files in both reported paths, and https://github.com/openclaw/plugin-inspector/pull/41 is the linked candidate fix but remains open and unmerged.

## What This Changes

The issue reports that plugin-inspector rewrites an existing package.json with plain fs.writeFile in init --scripts and prune-workspace-dev-deps, risking manifest corruption if interrupted mid-write.

## Best Possible Solution

Review and either land https://github.com/openclaw/plugin-inspector/pull/41 with adequate proof and tests, or replace it with an equivalent dependency-light atomic package.json writer covering both existing manifest rewrite paths.

## Reproduction Assessment

No live current-main corruption run was reproduced here; source inspection does confirm both reported paths rewrite an existing package.json with fs.writeFile, and a fault-injection test should be able to exercise the interrupt window.

## Solution Assessment

Yes in direction: a dependency-light sibling temp file plus fsync and rename is the narrow maintainable fix, with the linked PR currently owning that implementation path.
