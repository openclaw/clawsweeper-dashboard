## Summary

Keep this PR open for maintainer security/compatibility acceptance: the patch appears correct and well-proven, but it intentionally makes ownerless GitHub Actions session resumes fail closed before token rotation.

## What This Changes

The PR requires GitHub Actions action-session registrations and resumes to prove the stable owner before rotating the session token, with matching docs and regression tests.

## Best Possible Solution

Land the fail-closed owner-proof requirement if maintainers accept the rollout impact, keeping the included regression tests and docs as the new contract.

## Reproduction Assessment

Yes. From source inspection, current main only requires `owner` for new work and only checks owner mismatch when a resolved owner exists, so an existing `workKey` resume without owner can reach the token update path.

## Solution Assessment

Yes, subject to maintainer rollout acceptance. Requiring `resolvedOwner` before any registration update is the narrowest code fix, and the PR preserves matching-owner resumes while rejecting ownerless and wrong-owner resumes.
