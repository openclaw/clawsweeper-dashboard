## Summary

Keep open for maintainer security and compatibility acceptance. Current main still allows an existing GitHub Actions `workKey` resume without owner proof to reach token rotation, and this PR is a narrow, well-proven fix, but it intentionally makes ownerless resume callers fail closed with HTTP 400.

## What This Changes

The PR requires owner proof for GitHub Actions action-session resumes, updates the API docs, and adds regression coverage for ownerless and owner-matching resumes.

## Best Possible Solution

Land the fail-closed owner-proof requirement with the included docs and tests once maintainers explicitly accept the caller-compatibility impact.

## Reproduction Assessment

Yes. Source inspection on current main shows an existing workKey resume with omitted owner can pass the new-work owner check and reach the session update that rotates `agent_token_hash`.

## Solution Assessment

Yes, if maintainers accept the compatibility break. Requiring `resolvedOwner` before any registration insert or update is the narrowest maintainable fix, and the PR preserves matching-owner resume behavior.
