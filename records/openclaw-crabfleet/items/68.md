## Summary

The PR correctly fixes a real security boundary by requiring owner proof before any GitHub Actions workKey resume can rotate the session agent token, and the redacted local Worker proof is sufficient; the remaining blocker is maintainer acceptance of the intentional compatibility break for existing ownerless resume callers.

## What This Changes

The PR changes GitHub Actions action-session registration so all registrations and resumes require a resolved owner, updates API/docs wording, and adds regression tests for ownerless, matching-owner, and wrong-owner resume behavior.

## Best Possible Solution

Land the fail-closed owner-proof requirement if maintainers accept the compatibility impact, with the included regression tests and docs as the upgrade contract.

## Reproduction Assessment

Yes, from source inspection: current main requires owner only for new work and only checks owner mismatch when a resolved owner is present, so an existing workKey resume without owner can reach the token rotation/update path. I did not execute the service locally during this read-only review.

## Solution Assessment

Yes, if maintainers accept the fail-closed compatibility break: requiring resolvedOwner before any registration update is the narrowest fix for the missing owner-proof check, and the PR preserves matching-owner resume behavior.
