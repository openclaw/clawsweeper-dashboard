## Summary

Keep open: the session-scoped native VNC direction looks useful, but the current branch can break existing OAuth gateway setups, still accepts malformed grant broker URLs, and lacks after-fix real behavior proof.

## What This Changes

The branch adds a session-scoped native VNC grant API across the Worker/runtime-adapter contract and macOS client, replacing published provider lease IDs with Fleet session IDs and passing grant tickets to the Crabbox CLI over stdin.

## Best Possible Solution

Preserve existing OAuth session/fleet authorization during upgrades, enforce a canonical grant-broker URL contract, and require live macOS/Crabbox proof before merge.

## Reproduction Assessment

Yes. Source inspection shows main authorizes OAuth with session/fleet resources only, while the PR unconditionally adds the native-VNC POST resource; the broker URL issue is also source-reproducible because the exact URL helper preserves raw noncanonical strings.

## Solution Assessment

No. Session-scoped grants are the right direction, but the current implementation should preserve the existing OAuth read path during upgrade and strictly canonicalize broker URLs before handing grants to the client/CLI.
