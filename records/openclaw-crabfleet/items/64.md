## What This Changes

The branch adds a session-scoped native VNC grant endpoint, changes Fleet/macOS to use session IDs instead of provider lease IDs, and passes the Crabbox grant ticket to the CLI over stdin with tests/docs.

## Best Possible Solution

Keep the grant-based design, add the native-vnc resource to OAuth discovery and tests, remove the changelog edit, and require redacted live macOS/Crabbox proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: use an OAuth-protected deployment where resource/audience enforcement is per MCP route, then request native VNC and the client will call a route it never included in OAuth discovery. I did not run the macOS flow because this review is read-only and the PR lacks live proof.

## Solution Assessment

No, not yet. The session-scoped grant design is the right direction, but the OAuth protected-resource list must include the native-vnc route and the release-owned changelog edit should be removed before merge.
