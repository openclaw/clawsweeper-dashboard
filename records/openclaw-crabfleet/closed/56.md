## Summary

Current main already covers the execution-substrate question with a documented live execution boundary: Container sessions use the built-in Cloudflare Sandbox, Crabbox sessions use the versioned runtime adapter, and GitHub Actions sessions use the relay path; the runtime-adapter work shipped in v0.2.0.

## What This Changes

The issue asks whether crabfleet should standardize agent-run execution on a sandbox substrate such as crabbox or mitos instead of leaving execution isolation per agent.

## Best Possible Solution

Keep the existing Sandbox plus versioned Crabbox runtime-adapter boundary; open narrower follow-ups only for concrete gaps in that adapter contract or for a specific new substrate integration.
