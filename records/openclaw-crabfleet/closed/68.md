## Summary

This PR appears technically sound and well proven, but it should stay open for explicit maintainer acceptance because it intentionally changes the GitHub Actions session resume contract from compatibility-tolerant to fail-closed owner proof before token rotation.

## What This Changes

The PR requires `owner` on every GitHub Actions action-session registration or resume, updates the public docs, and adds regression coverage for ownerless, matching-owner, and wrong-owner resume behavior.

## Best Possible Solution

Land the fail-closed owner-proof requirement if maintainers accept the rollout impact, keeping the included regression tests and docs as the new contract.

## Reproduction Assessment

Yes. Source inspection of current main shows an existing `workKey` resume without `owner` can pass the new-work-only owner guard and reach the token-hash update path.

## Solution Assessment

Yes, subject to maintainer rollout acceptance. Requiring `resolvedOwner` before any registration update is the narrowest code fix, and the PR preserves matching-owner resumes while rejecting ownerless and wrong-owner resumes.
