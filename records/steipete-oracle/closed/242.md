## Summary

Current main does not implement the requested Gemini 3.5 Flash web mapping or tier-based model configuration. This should stay open for maintainer product review because the tier abstraction would change Oracle's documented explicit-model contract, and Gemini web model headers need live validation.

## What This Changes

The issue asks to update Gemini web model references and replace hardcoded Gemini web model IDs with a tier-based configuration layer.

## Best Possible Solution

Decide whether Oracle should keep explicit model IDs with refreshed Gemini mappings or add stable tier aliases, then update the web/API mapping, docs, and regression coverage with live Gemini web proof for the chosen contract.

## Reproduction Assessment

Not applicable; this is a model-support and configuration-design request rather than a reproduced broken behavior report. Source inspection gives a high-confidence check that the requested mapping and tier abstraction are absent on current main.

## Solution Assessment

Unclear; refreshing stale Gemini IDs is likely useful, but the proposed tier contract is not obviously the best fix without maintainer direction. A narrower alternative is to keep explicit model names and centralize only the web fallback/header mapping unless maintainers want tier aliases.
