## What This Changes

The branch extends browser attachment readiness matching to accept ChatGPT duplicate-renamed and ellipsized attachment chip labels, tightens count fallback behavior, adds expression tests, and updates the changelog.

## Best Possible Solution

Keep the scoped active-composer approach, add exact duplicate-rename matching that also covers short basenames safely, and require redacted after-fix browser proof before merge.

## Reproduction Assessment

Yes for the introduced short-basename gap: source inspection shows the duplicate-rename branch is skipped when item.stem.length is below four, so a chip like go(1).md cannot satisfy go.md. I did not run a live ChatGPT browser reproduction.

## Solution Assessment

No: the direction is maintainable, but the implementation should cover exact duplicate-renamed shapes for short basenames and include after-fix live browser proof before merge.

- **[P2] Handle short duplicate-renamed basenames:** `src/browser/actions/promptComposer.ts:383-386`
