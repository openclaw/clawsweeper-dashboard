## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Keep the frame fallback when the target read is incomplete:** `src/browser/actions/deepResearch.ts:218-219`
  - body: `readDeepResearchTargetResult` can return a non-completed status for an in-progress or merely text-bearing iframe. Because this nullish fallback is skipped for any non-null target result, a completed legacy/in-page frame result is never checked, so supported inline Deep Research rendering can still time out. Only let a completed target result short-circuit the frame path, or read both and apply the existing scoped guard.
  - confidence: 0.81
- **[P1] Continue scanning target sessions after incomplete iframe reads:** `src/browser/actions/deepResearch.ts:446-447`
  - body: Auto-attach can yield more than one iframe target, and the current target filter admits every `type: "iframe"` target. Returning on the first non-completed result with text or in-progress state can miss a later completed Deep Research OOPIF and leave the run stuck; keep the best incomplete result but scan every candidate for completion first.
  - confidence: 0.78
