## Summary

Keep this PR open: the target-first OOPIF extraction is the right direction, but the latest scoping repair still has a blocking CDP session-boundary bug for Oracle’s browser-WSEndpoint path and needs fresh real-browser proof for the latest head.

## What This Changes

The PR switches Deep Research completion to target-first OOPIF extraction, removes browser-wide target enumeration in favor of auto-attach, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the target-first OOPIF extraction, but make target discovery use the actual current-page session binding on every connection path and then provide fresh live browser proof for the latest head.

## Reproduction Assessment

Yes: source inspection shows current main chooses the in-page frame reader whenever Page is present, while production and reattach pass both Page and client into `waitForDeepResearchCompletion`; that makes the target reader unreachable in the reported production shape. The remaining PR regression is also source-reproducible on the browser-WSEndpoint wrapper because raw `send` is not session-bound there.

## Solution Assessment

No: target-first OOPIF extraction is the right direction, but this implementation still relies on raw CDP calls that are not page-scoped for all Oracle client wrappers. The safer fix is to bind Target auto-attach to the current page session or prove target ancestry before evaluating and returning report text.
