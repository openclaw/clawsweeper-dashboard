## Summary

Keep open: the reported `select`-strategy GPT-5.5 Instant hang is still actionable on current main, and the adjacent open work covers a different `current`-strategy thinking-time path rather than this picker loop.

## What This Changes

The issue reports that browser mode hangs when `--browser-model-strategy select` tries to switch from a current `Pro Extended` ChatGPT composer pill to `gpt-5.5-instant`.

## Best Possible Solution

Make `ensureModelSelection` either reach the requested Instant variant or return a bounded `option-not-found` when only a non-Instant GPT-5.5 row is available, with a regression test for the reported Pro Extended picker state.

## Reproduction Assessment

Yes, at source level: the reported DOM shape can keep producing a non-target GPT-5.5 match, and current code only checks the fail-fast timeout when no match exists. I did not run the live remote-Chrome smoke.

## Solution Assessment

Yes: the narrow fix is to stop treating a plain GPT-5.5 row as progress for an Instant request unless it opens a path to Instant, and otherwise fail fast with the existing `option-not-found` style error.
