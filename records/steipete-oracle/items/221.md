- `P1`: This is a current-release browser workflow regression that prevents attachment prompts from being submitted even when ChatGPT is ready to send.

## Summary

Keep open: current main still has the reported browser attachment readiness false negative, and the issue is already paired with an open fixing pull request that uses closing syntax.

## What This Changes

Forced ChatGPT browser attachments can throw `attachment-send-not-ready` even when the attachment chip, prompt text, and enabled Send button are present.

## Best Possible Solution

Review and land https://github.com/steipete/oracle/pull/222 or an equivalent narrow form-root fix, then verify forced browser attachments submit successfully before release.

## Reproduction Assessment

Yes. The issue contains concrete synthetic-file browser commands and DOM evidence, a commenter reproduced it against current source `aef77fe5` / Oracle `0.12.1`, and source inspection matches the reported false negative.

## Solution Assessment

Yes. Anchoring attachment readiness to the enabled Send button’s enclosing form is the narrowest maintainable fix for the observed wrong-root selection, and it avoids a broader page-wide attachment scan.
