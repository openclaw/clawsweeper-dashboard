## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not inherit the parent's wait default for follow-up sessions:** `src/cli/browserFollowUp.ts:137`
  - body: The docs and CLI output make follow-up default to return a detached child unless `--wait` is passed, but this line copies `parent.options.waitPreference`. Most browser parents are stored with `waitPreference:true` because browser runs block by default, so `oracle follow-up <parent> --prompt ...` will wait unexpectedly and can reintroduce client timeout behavior unless users know to pass `--no-wait`.
  - confidence: 0.9
- **[P3] Avoid launching browser finalizers for API detach:** `bin/oracle-cli.ts:2525`
  - body: `launchDetachedSession` is used by the top-level detach path, and current detach policy only returns true there for Pro API runs. Starting the browser-only finalizer here gives every detached API run an unnecessary second process and a confusing finalizer log; keep the finalizer on the browser follow-up/MCP detached paths or guard it by engine.
  - confidence: 0.81
