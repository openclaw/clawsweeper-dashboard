## Summary

Keep open: current main still persists browser transcripts, Deep Research reports, and generated images only; generic ChatGPT downloadable files are not modeled or downloaded. Likely related people for routing are Peter Steinberger for the current artifact implementation and @pdurlej for the related browser-artifacts work.

## What This Changes

Add ChatGPT browser-mode detection, cookie-backed downloading, metadata, and session rendering for generic downloadable assistant file artifacts such as ZIPs, wheels, and sdists.

## Best Possible Solution

Generalize the existing browser artifact layer to harvest downloadable file links from the final assistant turn, fetch them through the active browser session, persist safe filenames under the session artifacts directory, and expose file metadata through session render/status output.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection gives a high-confidence path: current main has no generic file artifact kind and only harvests generated image URLs, though I did not run a live Pro browser smoke for this review.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the supported link sources and metadata contract; the narrow technical direction is to reuse the existing cookie-backed artifact pipeline with a generic file artifact type.
