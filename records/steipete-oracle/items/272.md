## Summary

## What This Changes

Raises the default per-file attachment limit from 1 MB to 500 MB, documents the new default, and propagates maxFileSizeBytes through browser and remote execution options with focused tests.

## Best Possible Solution

Land one coherent limit policy: either keep defaults within the lowest supported browser/remote transport cap, or deliberately raise and prove the transfer path before documenting 500 MB as the default.

## Reproduction Assessment

Yes for the review finding by source inspection: after this PR, a file over 20 MB and under 500 MB can pass the new default guard while the browser DataTransfer path still rejects over 20 MB. I did not run a browser smoke in this read-only review.

## Solution Assessment

No. The branch changes the default and propagates the value, but the narrow maintainable solution must also align browser/remote transport limits or document and test the exception before shipping the larger default.
