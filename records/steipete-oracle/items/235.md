- **[P2] Persist the picker diagnostic in non-verbose runs:** `src/browser/actions/thinkingTime.ts:59-60`
  - body: `logPickerDiagnostic` emits `[thinking-time] model-picker diagnostic...`, but the session runner’s non-verbose logger only forwards always-on messages that start with `[browser] `, and its `sessionLog` is a no-op unless verbose. That means the fatal Pro Extended failure path still loses the new diagnostic during normal session runs, so future selector drift remains undiagnosable from saved output. Route this through the accepted browser logging path and add a non-verbose regression test.
  - confidence: 0.94
- **[P2] Keep valid one-word answers out of the placeholder list:** `src/browser/actions/assistantResponse.ts:48-59`
  - body: The new predicate treats exact text like `Thinking`, `Reading`, `Searching`, and `Working` as placeholders everywhere. If ChatGPT’s completed answer is one of those valid short words, the observer/snapshot paths return null and browser mode can wait until timeout or mark capture incomplete. Gate these status labels on active generation/progress UI, or otherwise preserve completed final turns, and add a regression case for an exact status-word answer.
  - confidence: 0.83
