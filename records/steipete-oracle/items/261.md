## Summary

Keep open. Current main covers the older `Called tool` / OOPIF placeholder path from the merged Deep Research extraction work, but this issue reports newer status-panel and instant-fallback captures; the current main-DOM completion fallback can still accept any finished assistant turn over 40 characters that is not an exact tool stub.

## What This Changes

The issue asks Oracle browser mode to return only final ChatGPT Deep Research reports, or fail clearly when ChatGPT exposes interim status, `Stop research`, or non-research fallback content.

## Best Possible Solution

Tighten Deep Research completion so Oracle accepts only a connector-confirmed final report or strong final-report DOM evidence, and otherwise records an incomplete/clear failure for interim status panels or non-research fallback turns.

## Reproduction Assessment

Yes at source level, but not with a live ChatGPT run here. The current completion expression can treat a finished assistant turn with status/fallback text as a Deep Research result as long as it is over 40 characters and not exactly `Called tool` or `Used tool`.

## Solution Assessment

No, current main is not sufficient for the reported state variants. The narrow fix is to make Deep Research finality stricter and fail incomplete for status panels or instant fallback responses while preserving the OOPIF report reader.
