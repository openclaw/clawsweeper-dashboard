## Summary

Keep open. Current main has the OOPIF Deep Research extraction work, but source inspection still shows only narrow tool-stub rejection and no explicit guard for the multi-line planning/status or silent non-research fallback content described here.

## What This Changes

The issue asks Oracle browser mode to return only final ChatGPT Deep Research reports, or fail clearly, instead of treating status panels, stop-state UI, or instant fallback responses as final output.

## Best Possible Solution

Tighten Deep Research completion so final-report evidence is required, interim status/planning panels are rejected, and silent non-research fallbacks produce a clear Deep Research-specific error.

## Reproduction Assessment

Yes at source level: a finished Deep Research assistant turn with multi-line status text and more than 40 characters can pass the current main-DOM completion gate because only exact tool stubs are rejected. I did not run a fresh paid live Deep Research smoke.

## Solution Assessment

Yes: fixing the completion detector is the narrowest maintainable path because docs already require final Deep Research report capture rather than wrapper/status content. The repair should add focused regression cases for the issue examples and avoid changing unrelated browser response handling.
