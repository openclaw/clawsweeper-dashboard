The issue requests that persisted HTTP MCP servers be resolved consistently for targeted `list <name>` and `call server.tool` commands, including selector-plus-`--http-url` calls.

## Best Possible Solution

Review and land https://github.com/openclaw/mcporter/pull/219 if maintainers accept its selector semantics; otherwise keep this issue as the canonical tracker for a narrower compatible parser fix.

## Reproduction Assessment

Yes. The selector-plus-`--http-url` failure is source-reproducible on current main because the qualified selector can remain after HTTP target preparation and then be used as the tool name; I did not run a live HTTP MCP server in this read-only review.

## Solution Assessment

Yes, the linked PR is the right narrow direction because it splits the selector before ad-hoc HTTP registration and adds focused regression coverage. The issue should stay open until that PR merges or maintainers choose a different compatible parser fix.
