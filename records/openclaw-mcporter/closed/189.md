## Summary

Keep open. Current main supports normal response rendering, MCP resource reads, and opt-in saving for image content blocks, but I did not find a per-server or per-tool policy that stores large text/tool responses as files.

## What This Changes

The issue requests a per-server or per-tool option to store large MCP tool responses as files instead of returning all content directly to the model context.

## Best Possible Solution

Design an explicit response-spill policy that preserves existing stdout and CallResult contracts while letting selected tools return file references for large text or resource payloads.
