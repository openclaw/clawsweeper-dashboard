## Summary

Keep open: the branch is a plausible narrow fix for the still-open stdout truncation bug, but it needs real behavior proof and focused regression coverage before merge because it changes the forced-exit timing path.

## What This Changes

The PR changes `src/cli.ts` so the forced-exit timer waits for an empty stdout write callback, with a 2s safety timeout, before calling `process.exit`.

## Best Possible Solution

Land a bounded stdout-flush fix with redacted real terminal proof and regression coverage for the greater-than-64KB delayed pipe while preserving exit codes and the anti-hang cap.

## Reproduction Assessment

Yes. The linked issue gives a concrete greater-than-64KB pipe repro, and current main still emits large JSON through stdout before a timed hard exit with no drain handling; I did not execute the repro in this read-only pass.

## Solution Assessment

Yes, with validation. Waiting for stdout flush behind a bounded timeout is a narrow maintainable direction, but the branch should prove the delayed-pipe behavior and safety cap before merge.
