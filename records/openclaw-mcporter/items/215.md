## Summary

Keep open: current main still has the timed hard-exit path without stdout drain handling, and this PR is a narrow candidate fix for the linked truncation bug, but it needs real behavior proof and validation of the new bounded wait before merge.

## What This Changes

The PR changes `src/cli.ts` so the forced-exit path waits for an empty stdout write callback before calling `process.exit`, with a 2s safety timeout.

## Best Possible Solution

Ship a bounded stream-flush fix in the forced-exit path with real delayed-pipe proof and regression coverage while preserving nonzero exit codes and the anti-hang timeout.

## Reproduction Assessment

Yes. The linked issue gives a concrete greater-than-64KB pipe repro, and current main still emits large JSON through stdout before a timed hard exit with no drain handling; I did not execute the repro in this read-only pass.

## Solution Assessment

Yes, with validation. Waiting for stdout flush behind a bounded timeout is the narrow maintainable direction, but the branch should prove the delayed-pipe behavior and safety cap before merge.
