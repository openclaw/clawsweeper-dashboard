- **[P2] Fail when a recorded request has no response:** `src/runtime/replay-transport.ts:46-49`
  - body: If a recording contains a send with an id but no matching recv, this branch shifts it off the queue and then returns without delivering a response. The MCP client waiting on that id will hang until timeout, and `close()` cannot catch it because the entry was consumed; keep notifications response-less, but make id-bearing requests without a recorded response fail clearly.
  - confidence: 0.88
- **[P2] Make partial replay failures reach the CLI:** `src/runtime/replay-transport.ts:53-54`
  - body: Throwing from `ReplayTransport.close()` is not enough for real `mcporter replay` runs because `runtime.close()` routes through `closeTransportAndWait()`, which logs and suppresses transport close errors. A replay that skips the tail of a recording can still exit successfully; propagate this check through the runtime/CLI path and cover that path in tests.
  - confidence: 0.86
