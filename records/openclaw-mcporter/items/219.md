## Summary

Keep open for maintainer review: current main still sends the full dotted selector as the MCP tool name for selector-plus-HTTP-url calls, and this PR fixes that path with focused tests, but it intentionally changes how dotted ad-hoc HTTP positionals are interpreted.

## What This Changes

The PR updates CLI call parsing to split dotted selectors before ad-hoc HTTP registration and adds regression tests for configured/ad-hoc HTTP call selectors and configured HTTP list JSON schema output.

## Best Possible Solution

Land the narrow parser and regression-test fix once maintainers accept the dotted `--http-url` selector precedence and the normal green gate is clean.

## Reproduction Assessment

Yes from source inspection: current main leaves `xhs.check_login_status` intact when `--http-url` is present, then resolves that full selector as the MCP tool name. I did not execute the CLI because this was a read-only review.

## Solution Assessment

Yes, with a compatibility caveat: splitting before ad-hoc registration is the narrow fix for the reported path, and the PR now covers configured, named ad-hoc, and literal `--tool` cases. Maintainers still need to accept the new positional precedence for dotted ad-hoc calls.
