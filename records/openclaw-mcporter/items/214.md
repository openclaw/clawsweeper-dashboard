## Summary

Keep open: current main still prints large CLI results in one shot and then schedules a timed hard process.exit after runtime cleanup without waiting for stdio drain. The issue is already paired with open candidate fixes, so it should stay open until one linked PR lands or maintainers choose a different repair.

## What This Changes

The issue reports silent truncation of large CLI stdout when output is piped because the forced-exit path can kill the process before buffered stdout drains.

## Best Possible Solution

Review and land a single linked fix that flushes buffered CLI stdout and stderr before the hard-exit fallback while preserving the existing anti-hang protection.

## Reproduction Assessment

Yes: the report provides concrete pipe-versus-redirect commands, and current source still has console.log output followed by a forced process.exit path without drain handling. I did not run the live pipe reproduction during this read-only review.

## Solution Assessment

Yes: a bounded stdout/stderr flush before forced exit is the narrow maintainable fix, as long as the anti-hang fallback and regression coverage for large piped output and stalled readers remain in place.
