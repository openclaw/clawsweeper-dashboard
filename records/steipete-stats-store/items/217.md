## Summary

Keep open: the reported channel-crossing bug is credible, but this patch rewrites all mismatched direct `.xml` base URLs and can break existing custom stable-feed filenames; it also needs after-fix real behavior proof.

## What This Changes

The PR changes appcast URL construction so a stored `.xml` URL is reused only when its filename matches the requested appcast path, and adds a regression test for prerelease-to-stable feed swapping.

## Best Possible Solution

Preserve existing direct-XML compatibility for custom stable feed filenames while adding an explicit, tested path for prerelease/stable channel separation and real proxy proof.

## Reproduction Assessment

Yes. From current main source, a stored `https://example.com/updates/appcast-prerelease.xml` with a request for `/appcast.xml` returns the stored prerelease URL because line 90 short-circuits on the requested default filename.

## Solution Assessment

No. The patch fixes the prerelease example, but it does so by changing the documented direct-XML default behavior; the safer path is to preserve custom exact XML URLs or make the channel mapping explicit.
