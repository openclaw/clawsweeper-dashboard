## Best Possible Solution

Keep the prerelease-to-stable protection, preserve documented direct custom stable XML URLs for default clients, and require redacted real proxy output before merge.

## Reproduction Assessment

Yes from source inspection: current main returns a stored `appcast-prerelease.xml` URL unchanged when the client requests `appcast.xml`. I did not run tests in this read-only review, but the route condition and added regression test make the failure path clear.

## Solution Assessment

No, not as written. The narrow maintainable fix should distinguish prerelease-channel crossing without dropping support for custom direct stable XML filenames documented as usable appcast URLs.
