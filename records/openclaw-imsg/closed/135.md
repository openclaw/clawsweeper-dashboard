## Summary

Keep open: current main still awaits a Contacts permission request in the `.notDetermined` path used by `imsg chats` and `imsg history`, while the docs say Contacts is optional and should only affect resolved name fields.

## What This Changes

The issue reports that `imsg chats` and `imsg history` can hang when Contacts authorization is `.notDetermined` because contact resolution requests access from a CLI path.

## Best Possible Solution

Make contact enrichment fail open for undecided Contacts permission so core chats/history reads continue with raw handles, then cover that behavior with deterministic tests.

## Reproduction Assessment

Yes from source inspection, but not as a live macOS run in this environment: `chats` and `history` await `ContactResolver.create()`, and the `.notDetermined` branch awaits `requestAccess`. The reporter's Swift probe supplies the macOS runtime symptom.

## Solution Assessment

Yes; fail-open contact enrichment is the narrowest fix because the docs already say Contacts is optional and skipped Contacts permission should only omit resolved names.
