## Summary

Keep open: current main still exposes `chat-delete` and `chats.delete`, and the injected helper still returns the reported `deleteChat: not available` error when `IMChatRegistry` lacks that selector.

## What This Changes

The issue reports that the exposed CLI and JSON-RPC chat deletion paths fail on macOS 26.5.1 because the injected bridge only calls `IMChatRegistry.deleteChat:`.

## Best Possible Solution

Add a verified macOS 26-safe chat deletion path or capability-gate `chat-delete`/`chats.delete` when deletion is unsupported, with focused regression coverage and live proof on a disposable chat.

## Reproduction Assessment

## Solution Assessment

Unclear: the current implementation is not the best end state because it advertises deletion while only trying one private selector. The maintainable fix is either a verified macOS 26-compatible deletion path or clear capability gating when deletion is unavailable.
