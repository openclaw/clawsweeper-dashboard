## Summary

Keep open: current main and v0.11.1 still expose `chat-delete`/`chats.delete`, and the injected helper still uses only `IMChatRegistry.deleteChat:` with the exact reported error when that selector is absent; no related fix or canonical replacement was found.

## What This Changes

The issue reports that `imsg chat-delete` and RPC `chats.delete` are exposed but fail at runtime on macOS 26.5.1 because the injected `deleteChat` selector is unavailable.

## Best Possible Solution

Add a macOS 26-safe chat deletion path or capability-gate `chat-delete`/`chats.delete` when deletion is unsupported, with focused regression coverage and live proof on a disposable chat.

## Reproduction Assessment

Yes for source and reporter-provided reproduction: current main still returns the exact error whenever `IMChatRegistry` lacks `deleteChat:`, and the issue body gives v0.11.1/macOS 26.5.1 CLI/RPC commands. I did not run the live macOS runtime in this Linux checkout.
