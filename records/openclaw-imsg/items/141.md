## Summary

Keep open: current main still has watch-side URL balloon duplicate suppression, but it does not coalesce a text row plus URL preview row into one logical message across history, search, and watch.

## What This Changes

Request source-level coalescing of Apple URL-preview split-send rows so consumers receive one logical message instead of separate text and URLBalloonProvider rows.

## Best Possible Solution

Define one shared MessageStore coalescing layer used by history, search, and watch, preserving the originating text row identity while leaving non-URL balloons and genuinely separate sends untouched.

## Reproduction Assessment

Yes, source-level: current history and search paths append one Message per SQL row, and watch only suppresses duplicate URL balloon rows. A deterministic fixture with adjacent same-chat/same-sender text plus com.apple.messages.URLBalloonProvider rows would exercise the current gap.

## Solution Assessment

No, the current watch-only dedupe is not the full solution requested here. The likely maintainable path is a shared coalescer, but maintainers need to decide the stable output shape first.
