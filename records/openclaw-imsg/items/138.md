I found no blocking diff bug; the main remaining uncertainty is that the exact macOS 26.5 crash path has not been proven fixed on a 26.5 host.

## What This Changes

The PR moves injected dylib bootstrap work from the constructor into a main-queue `bridgeBootstrap()` call to avoid macOS 26 dyld initializer ordering crashes.

## Best Possible Solution

## Solution Assessment
