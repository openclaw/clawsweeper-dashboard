## Summary

Current main still drops the plain JSON-RPC `reply_to` value before bridge dispatch, and the open linked pull request is the active candidate fix, so this issue should stay open until that fix lands or is replaced.

## What This Changes

The issue asks for plain JSON-RPC `send` to honor `reply_to` by forwarding the requested parent GUID through the bridge so outgoing replies set the correct reply/thread fields.

## Best Possible Solution

Review, prove, and land or replace https://github.com/openclaw/imsg/pull/145 with a narrow fix that forwards reply aliases into `selectedMessageGuid`, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes, from source inspection: current main's plain RPC send path never reads reply aliases and its bridge helper does not include `selectedMessageGuid`, while `send.rich` and `send-rich` already do. I did not run a live Messages bridge reproduction because this review avoided live `chat.db` or Messages side effects.

## Solution Assessment

Yes, the narrow maintainable solution is to reuse the existing `selectedMessageGuid` bridge contract for plain RPC `send` and add focused regression coverage. The linked pull request is the active candidate, but it should be reviewed and proven before this issue is closed.
