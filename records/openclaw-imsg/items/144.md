## Summary

Keep open. Current main still does not forward `reply_to` from plain JSON-RPC `send` into the bridge threading payload, and the linked closing PR is still open, so the issue should remain the canonical tracker until that repair lands or is replaced.

## What This Changes

The issue asks for JSON-RPC `send` to honor `reply_to` like `send.rich` and `send-rich --reply-to`, threading through the bridge-selected parent message.

## Best Possible Solution

Land or replace the linked repair PR with a narrow fix that forwards reply aliases to `selectedMessageGuid` for plain RPC `send` and adds focused deterministic bridge tests.

## Reproduction Assessment

Yes, source-reproducible: current main never reads `reply_to` on plain RPC `send` and never forwards `selectedMessageGuid` to the bridge helper. I did not run a live Messages bridge reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow existing-contract path: mirror `send.rich` reply alias handling and pass the result to the bridge as `selectedMessageGuid`. The open linked PR is the current candidate, but it should be reviewed and proven before merge.
