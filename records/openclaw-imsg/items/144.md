## Summary

Current main still drops the plain JSON-RPC `reply_to` value before bridge dispatch, and the open linked pull request is the active candidate fix, so this issue should stay open until that fix lands or is replaced.

## What This Changes

The issue asks for JSON-RPC `send` to honor `reply_to` the same way `send-rich --reply-to` does by sending the requested parent GUID through the bridge reply path.

## Best Possible Solution

Review, prove, and land or replace https://github.com/openclaw/imsg/pull/145 with a narrow fix that forwards reply aliases into `selectedMessageGuid` for plain RPC `send`, then close this issue once that fix reaches main.

## Reproduction Assessment

Yes, from source inspection: current main's plain RPC send path never forwards a reply alias into `selectedMessageGuid`, while `send.rich` and `send-rich` already do. I did not run a live Messages bridge reproduction because this review was read-only and avoided live `chat.db` or Messages side effects.

## Solution Assessment

Yes, the narrow maintainable solution is to reuse the existing `selectedMessageGuid` bridge contract for plain RPC `send` and add focused regression coverage. The linked pull request is the active candidate, but it should be reviewed and proven before this issue is closed.
