## Best Possible Solution

Land the narrow bridge-parameter fix after maintainers accept the bridge-only reply semantics and the contributor adds redacted live proof that explicit reply targets thread correctly and fallback is blocked.

## Reproduction Assessment

## Solution Assessment

Yes, with proof still required. Reusing the existing bridge `selectedMessageGuid` contract is the narrow maintainable fix, and blocking unthreaded fallback avoids silently losing the requested reply target.
