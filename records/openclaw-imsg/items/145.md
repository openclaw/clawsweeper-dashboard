## What This Changes

The PR makes plain JSON-RPC `send` parse reply-target aliases, forward them to bridge sends as `selectedMessageGuid`, block AppleScript fallback when a threaded reply cannot be honored, and adds focused RPC bridge tests.

## Best Possible Solution

Land a narrow fix that makes plain RPC `send` match `send.rich` reply threading through the bridge, with redacted real proof for both successful bridge threading and fail-closed no-bridge behavior.

## Reproduction Assessment

## Solution Assessment

Yes, the implementation is the narrow maintainable direction because it reuses the existing bridge `selectedMessageGuid` contract already used by `send.rich`; it still needs positive live bridge proof before merge.
