## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Allow validation probes without a child process:** `src/network-frontage.mjs:341-342`
  - body: `validateFrontage()` calls `waitForTcp(allocation.frontageHost, allocation.frontagePort, 2000)` without passing a child, but this guard treats a missing child as exited because `child?.exitCode` is `undefined` and `undefined !== null`. A real `--network-frontage loopback` run will therefore mark the frontage BLOCKED immediately after the proxy starts; guard this check with `if (child && child.exitCode !== null)`.
  - confidence: 0.94
- **[P2] Gate cron completion and attribution booleans:** `src/evaluator.mjs:395-396`
  - body: The cron surface and release profile declare `cronRunCompleted` and `cronTriggerAttributed`, but this evaluator block only checks `cronRunMs` and helper errors. Because the helper exits successfully when a run completes even if attribution is false, Kova can pass a cron scenario that did not prove cron trigger attribution; add boolean threshold checks and a negative self-check case.
  - confidence: 0.86
- **[P3] Use one MCP attribution metric name:** `src/evaluator.mjs:973`
  - body: The surfaces and known metric list use `mcpToolCallErrorAttributed`, but the evaluator records and reports `mcpInvalidToolErrorAttributed`. That leaves JSON consumers looking for the declared metric unable to find it, so use one stable metric id across the surface contract, evaluator measurement, and violation.
  - confidence: 0.78
