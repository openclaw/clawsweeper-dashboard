## What This Changes

The issue asks ClawSweeper to make queueable/actionable OpenClaw issue labels intentionally stale-safe instead of relying on later stale-workflow churn.

## Best Possible Solution

Make the ClawSweeper advisory-label sync add a durable stale exemption such as `no-stale` and remove `stale` when an OpenClaw issue becomes queueable/actionable, with tests and docs covering the label transition.

## Reproduction Assessment

## Solution Assessment

Yes, the narrowest maintainable fix is to extend the existing advisory-label sync path so actionable queue labels intentionally set or clear stale-protection state, rather than relying on GitHub `updated_at` side effects.
