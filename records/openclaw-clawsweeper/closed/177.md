## Summary

Keep open. Current main still has a source-reproducible path that can create a public router response for a re-review command on a closed item, and https://github.com/openclaw/clawsweeper/pull/188 is the open closing PR that should land or be replaced before this issue closes.

## What This Changes

The issue asks ClawSweeper to stop acknowledging or dispatching old re-review commands that predate closure of a closed pull request.

## Best Possible Solution

Land or replace https://github.com/openclaw/clawsweeper/pull/188 with a narrow router/webhook guard plus regression coverage for pre-close and post-close re-review commands on closed PRs.

## Reproduction Assessment

Yes, from source inspection rather than a live run: current main still routes closed-item `re_review` commands to a public comment action, while the stale-command guard only covers automerge/autofix.

## Solution Assessment

Yes. The narrow maintainable solution is to no-op stale pre-close re-review commands for closed PRs before public ack/comment dispatch while preserving supported fresh commands on open items.
