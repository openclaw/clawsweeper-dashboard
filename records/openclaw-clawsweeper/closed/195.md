## Summary

Keep this PR open for normal maintainer review: current main does not yet expose `mergeable_state` in compact PR context, while the branch narrowly adds that context, prompt guidance, and regression coverage for the linked false-positive merge-risk report at https://github.com/openclaw/clawsweeper/issues/119.

## What This Changes

This PR adds PR `mergeable_state` to compact review context, updates the review prompt to treat clean behind-base drift as rebase/review-staleness rather than deletion risk, and adds focused regression coverage.

## Best Possible Solution

Land the narrow context, prompt, and regression-test change after normal maintainer review so future PR reviews distinguish clean behind-base drift from actual merge-result regressions.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source-level: current main omits `mergeable_state` from compact PR context, and the PR adds a fixture that renders a clean mergeable PR through the review prompt path; I did not run tests because this review is read-only.

## Solution Assessment

Is this the best way to solve the issue? Yes for the current boundary: exposing GitHub's merge state and tightening prompt guidance is a narrow maintainable fix, while a full merge-tree comparison can remain future hardening if false positives persist.
