## Best Possible Solution

Merge the marker-based inert guard after maintainers accept marker-bearing comments as non-command comments and normal checks stay green.

## Reproduction Assessment

## Solution Assessment

Yes. Centralizing marker detection in comment-router-core and reusing it from the webhook is the narrowest maintainable fix, with the target-dispatcher docs updated for the fallback Actions path.
