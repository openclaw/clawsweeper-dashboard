Full review comments:

- **[P1] Resolve the config from the repo root:** `src/repair/target-toolchain-config.ts:182`
  - body: In the built repair worker this module lives at `dist/repair/target-toolchain-config.js`, so `dirname(dirname(import.meta.url))` resolves to `dist`. That makes `defaultConfigPath()` look for `dist/config/target-repositories.json`, so `resolveTargetRepoToolchain("openclaw/clawhub")` falls back to the default pnpm/null toolchain and the central ClawHub bun fix is not used at runtime. Please use the shared repair root helper or another compiled-safe repo-root calculation and add a non-injected resolver/preflight test.
  - confidence: 0.93
- **[P2] Check the OpenClaw hard fallback before owner fallback:** `src/repair/target-toolchain-config.ts:67`
  - body: The PR says OpenClaw keeps a code-level safety net if the config entry is removed, but the generic `openclaw` owner fallback returns first. If `core_target_overrides["openclaw/openclaw"]` is ever dropped while the owner fallback remains, this returns a null changed gate instead of the hardcoded `pnpm check:changed` fallback. Please make the core repo fallback win before the owner fallback or explicitly merge the owner fallback with the hard override.
  - confidence: 0.78
