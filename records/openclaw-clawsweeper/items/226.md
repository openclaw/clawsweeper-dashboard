## Summary

Keep open: the live linked OpenClaw issue still has duplicate marker-backed ack comments for two separate command comments after the current coalescing fix was deployed, so this is not safely implemented on main.

## What This Changes

The issue asks ClawSweeper to make command acknowledgement comments idempotent so one issue command leaves one marker-backed ack/status comment.

## Best Possible Solution

Make ack convergence deterministic across duplicate webhook deliveries, delayed redeliveries, and Worker instances, with router-side or longer-window cleanup so extra marker-backed acks are removed even if the fast webhook cleanup misses them.

## Reproduction Assessment

Yes. The public linked comments still show two ack comments with the same source-comment marker for each of two commands, and the relevant dashboard/webhook code has not changed since the deployed c01469876 fix.

## Solution Assessment

No for the current partial mitigation. The in-memory coalescing and short delayed prune passes are useful, but the live duplicate evidence shows ClawSweeper needs a more deterministic cleanup/idempotency path.
