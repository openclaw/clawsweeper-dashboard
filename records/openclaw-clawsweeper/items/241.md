## Summary

This PR is useful but not merge-ready: the ClawHub Bun path is addressed, but the new per-repo config still leaves pnpm targets with `changed_gate: null` vulnerable to being rewritten back to `pnpm check:changed`.

## What This Changes

The branch adds per-target toolchain config, Bun validation setup/support, validation-command routing changes, ignored `notify` checks, docs/prompts, and regression tests for repair target validation.

## Best Possible Solution

Keep the per-repo toolchain approach, but make every validation fallback honor `changed_gate: null` so no target receives an undeclared `check:changed` command.

## Reproduction Assessment

Yes. Source inspection shows that a pnpm target with `changedGate: null` and a broad command such as `pnpm check` still reaches the pnpm branch and returns `pnpm check:changed`; I did not run tests in this read-only review.

## Solution Assessment

No. The Bun-specific path is on the right track, but the implementation should route pnpm fallback through the configured changed gate and preserve the original command when no gate is configured.
