## Summary

Keep open: the live GitHub Search path is well proven, but the new gitcrawl enrichment queries `threads.body_excerpt` for `cluster_groups` databases even though the default/current gitcrawl schema stores `threads.body`, so that optional source can silently return no siblings; maintainers also need to accept the default-on Search rollout.

## What This Changes

The branch adds related-item enrichment from gitcrawl clusters and live GitHub issue search, wires exact-event workflow configuration, and updates docs, prompt text, and focused query tests.

## Best Possible Solution

Land the advisory context expansion after schema-aware gitcrawl body selection and fixture coverage, with a maintainer-owned choice on whether live Search remains default-on for exact event reviews.

## Reproduction Assessment

Yes for the blocking patch defect: PR-head source selects `t.body_excerpt` for `cluster_groups` gitcrawl databases, while the current full gitcrawl schema defines `threads.body`; the helper then returns an empty list on sqlite failure.

## Solution Assessment

No; the direction is useful, but the implementation should first make gitcrawl body-column selection schema-aware and leave the live Search default as an explicit maintainer rollout choice.
