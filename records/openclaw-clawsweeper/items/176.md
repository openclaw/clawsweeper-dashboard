## Summary

Keep this PR open: the live GitHub Search path has good terminal proof, but the new portable gitcrawl lookup can silently return no siblings because its SQL does not match the current gitcrawl schema.

## What This Changes

The PR adds advisory related-item enrichment for ClawSweeper reviews from explicit links, local reports, optional gitcrawl clusters, and exact-event GitHub Search, plus docs, workflow env wiring, and query tests.

## Best Possible Solution

Land the related-context enrichment after the gitcrawl lookup has schema-backed fixture coverage and maintainers accept the exact-event GitHub Search default plus opt-out.

## Reproduction Assessment

Not applicable as a user bug report; this is a feature PR. The blocking defect is source-reproducible by comparing the PR's portable gitcrawl SQL with the current gitcrawl schema.

## Solution Assessment

No, not yet. The live GitHub Search path is a reasonable narrow implementation, but the gitcrawl path should use a supported schema and have fixture coverage before merge.
