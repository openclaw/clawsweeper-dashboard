## Best Possible Solution

Land the enrichment after gitcrawl SQL handles current and pruned schemas, with a maintainer-owned decision on whether exact-event GitHub Search remains default-on or starts opt-in.

## Reproduction Assessment

Yes for the blocking patch defect: source inspection shows the PR queries `threads.body_excerpt`, while current gitcrawl schema exposes `threads.body`. Because the sqlite helper returns `[]` on failure, this would silently drop gitcrawl related items.

## Solution Assessment
