## Summary

Keep open: the cursor-rotation direction is useful and the PR includes strong CLI proof, but the workflow still publishes cursor directories even when no cursor file was written, so `repair:publish-main` can delete durable cursor state.

## What This Changes

The PR adds cursor rotation and processed-limit wiring for untargeted proof-nudges and bot-proof scans, with workflow, docs, and regression test updates.

## Best Possible Solution

Publish exact cursor files only after the corresponding execute lane wrote them, and add workflow assertions that dry-runs and empty runs do not publish missing cursor paths.

## Reproduction Assessment

Yes. Source inspection shows current main starts untargeted proof-lane scans from the same sorted prefix and stops at `processedLimit`, so skipped early candidates can consume repeated windows.

## Solution Assessment

No, not as currently patched. Durable cursor rotation is the right narrow direction, but workflow cursor publication must be limited to cursor files actually written by execute-mode runs.
