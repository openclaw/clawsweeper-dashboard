## Summary

Keep open: the cursor rotation is useful and has credible runtime proof, but current head still publishes cursor paths for dry-run untargeted workflow runs even though the CLI only writes cursor files in execute mode; that can delete durable cursor state through publish-main.

## What This Changes

The PR adds durable cursor rotation and processed-limit tuning for proof-nudges and bot-proof scans, updates the scheduled workflow, documents the cursor behavior, and adds focused tests.

## Best Possible Solution

Land the cursor rotation only after workflow cursor publication is tied to each lane's execute mode and dry-runs remain non-mutating.

## Reproduction Assessment

Yes. Source inspection shows current main starts untargeted proof-lane scans from the same sorted prefix and stops at processedLimit, so skipped early candidates can consume repeated windows.

## Solution Assessment

No, not as currently patched. Durable cursor rotation is the right narrow direction, but workflow publishing must be gated so dry-run runs cannot delete cursor state.
