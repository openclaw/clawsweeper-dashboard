## Best Possible Solution

Keep the cursor-rotation fix, but only publish proof cursor paths after the corresponding lane actually runs in execute mode and can write cursor state.

## Solution Assessment

No, not as currently patched. Durable cursor rotation is the right narrow direction, but workflow publishing needs to be gated so dry-run runs cannot delete cursor state.
