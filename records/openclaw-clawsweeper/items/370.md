## Summary

Keep open: the PR adds the requested opt-in stale PR cadence and now wires the env into the workflow, but the normal active-floor backfill path can still select unchanged stale PRs after the six-hour backfill age, so the configured cadence is not consistently honored.

## What This Changes

Adds an optional `CLAWSWEEPER_STALE_PULL_REQUEST_REVIEW_DAYS` cadence for inactive stale PR reviews, wires it through scheduled sweep planning and dashboard audit, documents it, and adds scheduler-policy coverage for `shouldReviewItem`.

## Best Possible Solution

Apply the configured stale-PR cadence consistently to due selection, active-floor current-review backfill, dashboard cadence math, docs, and tests, unless maintainers explicitly choose and document a narrower due-only setting.

## Reproduction Assessment

Yes. Source inspection shows that with a weekly stale-PR cadence, `shouldReviewItem` can return false for a quiet PR reviewed about a day ago, while `reviewBackfillCandidate` can still return that same not-due item once it is older than the six-hour floor-backfill age.

## Solution Assessment

No. The helper and workflow wiring are the right narrow shape, but the implementation should either apply the same cadence to active-floor backfill or explicitly narrow the documented behavior with tests.
