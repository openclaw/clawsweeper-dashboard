## Summary

The PR is a technically coherent, well-proven fix for cross-workflow provider stampedes, but it deliberately makes the shared state repository a fail-closed dependency for exact reviews and therefore needs explicit maintainer approval rather than automated closure or repair.

## What This Changes

The PR adds a state-repository-backed weighted provider lease for exact reviews, renewal and timeout handling, terminal command statuses, real Actions proof jobs, and focused unit/workflow coverage.

## Best Possible Solution

Keep the lease implementation and its TTL recovery, then merge only after maintainers explicitly approve the fail-closed state dependency and production lease settings, with a follow-up observability/tuning path if real traffic shows excessive waits or state-induced failures.

## Reproduction Assessment

Yes at source level: current main has no cross-run provider reservation, so multiple independent exact-review workflow instances can invoke Codex concurrently beyond a shared provider limit. The PR's live Actions run also reproduces the relevant contention model with independent runners.

## Solution Assessment

Yes for the overload mechanism: a state-backed compare/retry lease is narrower and more controllable than starting Codex and relying on provider failures. The fail-closed outage policy and concrete production limits remain maintainer decisions rather than correctness defects.
