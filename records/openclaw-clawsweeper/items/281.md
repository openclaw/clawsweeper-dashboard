## Summary

The shared-state lease direction is sound, but the fixed lease can expire while its exact review is still running, defeating the global capacity gate. This member-authored automation PR also needs the pending live GitHub Actions proof and explicit maintainer acceptance before merge.

## What This Changes

The PR adds a state-repository-backed weighted provider lease that gates exact-review Codex invocation, reports capacity wait outcomes, and releases the lease before publishing review state.

## Best Possible Solution

Renew the lease while its review is active, or enforce a tested TTL comfortably longer than every possible post-acquisition path, then provide redacted live Actions evidence for contention, timeout, normal release, and cancellation recovery.

## Reproduction Assessment

Yes at the source and simulation level: independent runs currently lack shared provider-capacity coordination, and the Git-backed tests reproduce competing acquisitions. The production provider overload and long-running expiration behavior have not yet been demonstrated in live Actions.

## Solution Assessment

No, not yet: the shared-state lease is a reasonable coordination mechanism, but a non-renewed fixed TTL cannot safely represent ownership for a potentially longer-running review. Active renewal or a validated runtime bound is needed before this becomes the narrowest reliable solution.
