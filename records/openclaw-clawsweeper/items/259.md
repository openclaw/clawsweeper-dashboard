## Summary

## What This Changes

This PR adds a scheduled/manual conflict self-heal workflow, scanner, repair job intent, branch repair status handling, and tests for ClawSweeper-owned conflicting PR branches.

## Best Possible Solution

Persist the generated self-heal job to the state repo before dispatch, then prove both the exact-head execution path and stale-head skip path with dry-run and execute-mode evidence.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: run execute-mode self-heal for a candidate and the worker will hydrate state without the newly generated job, restore an expected_head_sha "unknown" placeholder, and skip at exact-head verification. I did not run the workflow because this review is read-only.

## Solution Assessment

No, the current PR shape is not the best mergeable solution because the generated job is not durably available to the worker before dispatch. The maintainable path is to publish state first, then dispatch, and keep the unknown-head restore path as a non-mutating fallback.
