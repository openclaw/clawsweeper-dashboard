## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection is sufficient: current main always passes --execute for targeted synced-verdict routing, while the PR head omits it unless CLAWSWEEPER_FEATURE_REPAIR_ENABLED is exactly 1 and defaults that variable to 0.

## Solution Assessment

No, not as a purely code-level fix. The implementation is internally consistent, but default-off repair dispatch is an upgrade and operations policy change that needs either a compatibility-preserving default or explicit maintainer rollout approval.
