## Summary

Keep open: this is a current ClawSweeper automation bug with source support and concrete run examples; the event re-review status can report success based on step outcomes without proving the durable verdict marker was refreshed before routing labels.

## What This Changes

The issue reports that targeted `@clawsweeper re-review` can mark Complete while the durable PR review marker and derived labels remain pinned to an older head SHA.

## Best Possible Solution

Gate event re-review completion and verdict routing on a verified durable review marker for the reviewed PR head, with a clear non-complete status when sync is skipped, stale, or fails.

## Reproduction Assessment

Yes. The issue provides two concrete GitHub run examples, and current source shows the Complete status is based on step success rather than verified durable marker freshness; I did not run a mutating live workflow repro.

## Solution Assessment

No, the current path is not the best fix because it routes success optimistically. A narrow repair should add verification or outputs proving the durable marker was updated to the reviewed head before marking Complete and routing labels.
