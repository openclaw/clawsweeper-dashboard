## Repair Work Prompt

Implement https://github.com/openclaw/clawsweeper/issues/185 as a conservative prompt-only change. Update prompts/review-item.md and prompts/review-commit.md so reviewers must inspect the full target AGENTS.md when present and treat it as optional repository policy/review guidance below system/developer instructions, the ClawSweeper review prompt, the output schema, close gates, and the configured repository profile/fallback. In the item prompt, say PRs, issues, proposed fix paths, tests, docs changes, validation, and automation recommendations should be judged against applicable AGENTS.md policy; concrete actionable PR conflicts should use existing reviewFindings, while issue or non-patch policy conflicts should route through risks, bestSolution, solutionAssessment, workReason, or evidence. In the commit prompt, AGENTS.md conflicts should become findings only when they create concrete bug, regression, security, compatibility, validation, supply-chain, or similar maintainer-relevant risk. Preserve fallback-first behavior; do not add an AGENTS parser, new config file, schema fields, labels, close reasons, dispatcher changes, webhook changes, or profile migration. Bump REVIEW_POLICY_VERSION in src/clawsweeper.ts so existing reviews refresh, and update the prompt-preservation tests in test/clawsweeper.test.ts. No changelog entry is expected unless maintainers request one.
