## Summary

The request is still useful but not implemented on current main. ClawSweeper already gathers related-item evidence from links, local reports, optional gitcrawl data, and opt-in GitHub search, but current docs and rendering treat that as advisory input for duplicate/superseded review rather than a public root-cause cluster section with relationship classes and a consolidation recommendation.

## What This Changes

The issue requests an advisory issue-review lane that groups likely shared-root-cause issues and PRs and recommends one consolidation path without taking automatic action.

## Best Possible Solution

Add a narrow advisory cluster-summary layer on top of the existing relatedItems context, with explicit high-confidence relationship classes, clear consolidation guidance, durable lower-confidence data, and tests proving normal one-issue reviews stay unchanged when no meaningful cluster exists.

## Reproduction Assessment

Not applicable: this is a new advisory workflow capability request, not a broken existing contract. Source inspection confirms related-item evidence exists, but the requested cluster classification and consolidation output is not present.

## Solution Assessment

Unclear until maintainers choose the comment shape and confidence policy. The narrowest maintainable path is to reuse current relatedItems/gitcrawl inputs and add conservative classification/rendering rather than building a separate clustering system first.
