## Summary

Keep this open: current main has related-item discovery, gitcrawl cluster import, autonomous repair-cluster jobs, and some duplicate-PR suppression, but it still does not implement the requested per-reviewed-item durable root-cause coordinator with canonical lane selection, sibling state/comment updates, duplicate repair suppression for every cluster member, and post-merge verified cleanup.

## What This Changes

The issue requests durable root-cause cluster coordination across related issues and PRs so ClawSweeper can choose one canonical fix lane and suppress duplicate repair work.

## Best Possible Solution

Extend current related discovery and repair-cluster lanes with a maintainer-approved durable cluster schema, canonical-lane policy, sibling state/comment updates, duplicate repair suppression, and post-merge verification while preserving existing apply and merge gates.

Not applicable: this is a feature and coordination request, not a report of broken existing behavior. The relevant verification is source/docs inspection, which shows partial building blocks but no complete durable coordinator.

## Solution Assessment
