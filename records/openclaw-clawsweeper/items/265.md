## What This Changes

The issue requests automated root-cause cluster coordination across related issues and PRs, including durable cluster state, canonical fix-lane selection, duplicate repair suppression, sibling updates, and verified close/supersede actions.

## Best Possible Solution

Extend the existing typed root-cause assessment and repair-cluster lanes into a maintainer-approved coordinator that promotes one canonical lane per cluster while preserving deterministic live-state and close/merge gates.
