## Summary

The patch is a focused, source-correct fix for an automation misclassification: it keeps Vercel authorization failures visible as terminal blockers while preventing them from launching an inappropriate Codex write repair. No actionable correctness or security defect was found, but the external-PR proof gate remains unmet because the PR supplies tests rather than after-fix evidence from a real ClawSweeper run.

## What This Changes

The PR classifies Vercel Git authorization statuses as external action-required blockers, excludes those blockers from failed-check repair reasons, and distinguishes Codex execution-mode preflight failures.

## Best Possible Solution

Retain the narrow URL-based external-action classification and distinct Codex execution-mode telemetry, with a redacted live run demonstrating the intended terminal-blocker and no-repair behavior before merge.

## Reproduction Assessment

Yes, at source level with high confidence: a Vercel authorization StatusContext reported as FAILURE currently enters the generic terminal-failure path and is accepted as a repairable failed check. The PR body also identifies a concrete occurrence at https://github.com/openclaw/clawhub/pull/2571.

## Solution Assessment

Yes. Keeping the authorization status as a terminal blocker while excluding only the externally actionable URL from write repair is narrower and safer than ignoring Vercel checks or broadly suppressing ACTION_REQUIRED conclusions.
