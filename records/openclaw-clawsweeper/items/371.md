## Summary

Keep open: the PR fixes a real close-marker/router mismatch, but the trusted close route can reach autoclose after only reviewed-head gating and does not show the same protected-label, source-drift, and close-policy gates used by the durable apply close path.

## What This Changes

The branch emits explicit close and close-required markers for PR close reviews, maps trusted close markers to autoclose, adds PR head-SHA gating, and covers the rendering/router behavior with focused tests.

## Best Possible Solution

Keep the explicit close markers, but make trusted close execution reuse or faithfully mirror the durable apply close gates before any autoclose mutation runs.

## Reproduction Assessment

Yes from source inspection: current main renders PR close decisions as needs-human and the trusted parser has no close-marker branch, so a close-intent review cannot reach autoclose on current main.

## Solution Assessment

No, not yet. Emitting explicit close markers is the right narrow direction, but the mutating route should preserve the existing close-apply safety gates rather than relying only on reviewed-head freshness.
