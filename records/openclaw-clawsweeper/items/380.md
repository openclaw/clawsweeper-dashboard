## Summary

Keep this PR open: the patch is narrow and source-consistent, but it changes the comment-router crash semantics and needs maintainer acceptance plus real behavior proof before merge.

## What This Changes

The PR adds a `claimed` command-ledger state for dispatch-bearing comment-router commands, writes that claim before acknowledgements and dispatch, and adds focused tests for claim suppression and final ledger upgrades.

## Best Possible Solution

Land this only with maintainer-approved at-most-once semantics, or add stale-claim recovery so duplicate dispatches are prevented without silently dropping commands after a pre-dispatch failure.

## Reproduction Assessment

Yes, source inspection gives a high-confidence current-main path: dispatch-bearing commands execute before the ledger is appended and written, so a failure after dispatch but before the final write can replay the same command version. I did not run a destructive live dispatch.

## Solution Assessment

Unclear. The implementation is narrow and tested, but treating `claimed` as terminal introduces an acknowledged pre-dispatch command-loss window that needs maintainer acceptance or a stale-claim recovery design before merge.
