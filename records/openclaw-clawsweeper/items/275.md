## Best Possible Solution

- **[P1] Preserve the exact-event timeout fallback:** `.github/workflows/sweep.yml:370`
  - body: This step now ignores the `CODEX_TIMEOUT_MS` env fallback that is still defined above it and instead always uses `steps.target.outputs.codex_timeout_ms`. Please keep the existing fallback for missing payloads and only use the adaptive value when one is supplied.
