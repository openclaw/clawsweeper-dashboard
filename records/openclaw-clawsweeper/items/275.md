## What This Changes

The branch computes a bounded 10–30 minute Codex timeout for pull-request webhooks from changed files, changed lines, and video-proof URLs, dispatches it in `codex_timeout_ms`, and applies it to the exact-event workflow command and outer timeout.

## Best Possible Solution

Use the proposed bounded timeout contract and, after merge, confirm the first large or media-bearing webhook review logs the computed budget and completes without `ETIMEDOUT`.

## Reproduction Assessment

Yes at source level: current main fixes webhook exact-event reviews at 600000ms inside a 12-minute wrapper, while the linked 71-file PR with two video proofs reportedly reached `ETIMEDOUT` repeatedly after media processing consumed the same wall-clock budget. This review did not rerun that external PR, so the status is source-reproducible rather than directly reproduced.

## Solution Assessment

Yes. Passing one bounded value through the existing repository-dispatch contract and reusing the established outer-budget formula is the narrowest maintainable fix; moving media preprocessing into another job would be a larger architectural optimization.
