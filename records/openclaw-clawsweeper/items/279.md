Full review comments:

- **[P1] Require actual upgrade proof before passing:** `src/clawsweeper.ts:9451-9455`
  - body: `dataModelUpgradeProofFromReport()` builds the positive proof text from `bestSolution`, `reviewFindings`, and `risks`, which are often future-looking instructions. A report that says “merge after migration or upgrade proof” can satisfy the positive regex and emit a pass marker even though the proof is still missing, defeating the guard requested by https://github.com/openclaw/clawsweeper/issues/225.
  - confidence: 0.9

  - body: The source issue explicitly calls out SQL `ON CONFLICT` behavior changes, but this detector only matches DDL/schema-version tokens here. A migration or database helper patch that changes `ON CONFLICT ... DO UPDATE` can leave `data_model_change` false and skip the warning/pass gate, so add that pattern and a regression test.
  - confidence: 0.86
