## Summary

Keep open for maintainer triage: current main has the advisory label projection and the read-only triage dashboard that this proposal builds on, but it does not yet implement label-based area routing groups, dry-run routing reports, or opt-in Discord digests. The remaining work is a new product/configuration decision, not a narrow bug fix or cleanup close.

## What This Changes

The issue proposes a label-based routing layer that groups existing issue labels into maintainer-owned area queues, starting with dashboard views and dry-run reports before optional Discord digests.

## Best Possible Solution

Make phase 1 a maintainer-owned, read-only dashboard grouping layer backed by the existing triage snapshot or gitcrawl read model, then add dry-run reports before any opt-in Discord delivery.

## Reproduction Assessment

Not applicable; this is a feature proposal rather than a bug report with a failing behavior to reproduce. The source check confirms current main lacks the proposed routing-group layer, but that is missing functionality rather than a regression.

## Solution Assessment

Yes, the dashboard-first, no-pings, no-assignment rollout is the safest direction if maintainers want this feature. The exact routing groups, delay rules, backing read model, and Discord policy need maintainer decisions before this should become an implementation task.
