## Summary

Keep open; current main has advisory issue labels and fixed read-only triage dashboard views, but it does not yet have the proposed label-to-area routing map, dashboard area queues, dry-run routing reports, or optional Discord digests, and the ClawSweeper repository profile keeps issues open for maintainer triage.

## What This Changes

Request to group existing issue labels into maintainer-owned area queues, starting with dashboard grouping and later dry-run reporting plus optional Discord digest routing.

## Best Possible Solution

If maintainers want this, land a small dashboard-only area grouping config on top of existing advisory labels first, preferably using gitcrawl/portable-store data where practical, and decide dry-run artifacts plus Discord digests separately.

## Reproduction Assessment

Not applicable. This is a feature proposal for a new routing/grouping layer, and source review shows existing advisory-label views rather than a failing behavior to reproduce.

## Solution Assessment

Unclear. The dashboard-only first phase fits the existing read-only triage boundary, but the group taxonomy, config ownership, gitcrawl backing choice, and Discord rollout need maintainer direction before implementation.
