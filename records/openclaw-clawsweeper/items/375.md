## Summary

Keep open: the apply-queue rotation is useful, but the patch currently has a workflow shell-scope bug that can stop apply runs under `set -u` before publishing results or queueing continuation.

## What This Changes

The PR adds cursor-backed batching for automatic apply candidates, persists apply cursor files, clears auto-selected item numbers for continuations, and stamps processed records with `apply_checked_at`.

## Best Possible Solution

Initialize the new apply cursor state inside the `Apply unchanged proposed decisions with checkpoints` step before any use, and keep the continuation behavior covered for both explicit and auto-selected runs.

## Reproduction Assessment

Yes from source: the PR head workflow runs `set -euo pipefail`, initializes the new variables in `apply-preselect`, and then uses them in the later apply step where they are not defined.

## Solution Assessment

No; the cursor direction is reasonable, but the workflow state has to be initialized in the same shell step that uses it before this can safely merge.
