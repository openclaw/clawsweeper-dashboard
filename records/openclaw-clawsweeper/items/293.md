## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scrub provider keys before exposing tools:** `src/repair/openai-compatible-tools-runner.ts:42-44`
  - body: After capturing the configured provider API key, the runner only deletes GitHub token variables. In workspace-write mode the model can call `run_command`, and that shell inherits `process.env`, so the selected `apiKeyEnv` can be printed into tool results or transcripts. Delete the captured provider key from the tool environment and pass an explicit scrubbed env to command execution.
  - confidence: 0.87
- **[P1] Preserve the Codex CLI login config:** `src/repair/run-worker.ts:367-370`
  - body: Current main passes `codexLoginConfig()` through `codexConfigArgs()` so operators can force the local ChatGPT/OAuth login method. This PR removes that config entry for the default `codex-cli` backend, so existing local-OAuth repair runs can fall back to the wrong auth path even when they did not opt into the new backend.
  - confidence: 0.86
- **[P1] Keep the existing Codex spawn wrapper:** `src/repair/run-worker.ts:298-318`
  - body: The default Codex backend now goes through raw `spawn(modelBackendCommand(), ...)` instead of `spawnCodex`, which bypasses `CODEX_BIN`, Windows `.cmd` resolution/escaping, detached process groups, and the existing process-tree timeout cleanup. Keep the existing wrapper for `codex-cli` and route only the OpenAI-compatible adapter through the alternate command path.
  - confidence: 0.86
