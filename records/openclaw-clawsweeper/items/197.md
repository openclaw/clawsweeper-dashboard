## Label Justifications

- `P1`: Incorrect close/apply decisions can remove valid contributor PRs from the active review queue while the source issue remains unresolved.
- `impact:other`: The impact is maintainer-visible automation state: PRs can be closed as superseded by unsafe or unmerged replacement work.

## Summary

Keep open: current source and live GitHub evidence show the superseded-close path can treat an open but not merge-ready replacement PR as canonical, and no current-main guard validates that replacement before applying the close.

## What This Changes

The issue asks ClawSweeper to avoid closing a PR as duplicate or superseded by another PR unless the replacement is merged, still viable, or backed by an open canonical issue that preserves the work.

## Best Possible Solution

Add a conservative canonical-target validator for duplicate/superseded PR closes, then keep or route the source PR to the open canonical issue whenever the referenced replacement PR is closed unmerged, F/no-proof, already proposed for close, conflicted, or otherwise not a safe landing path.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current code promotes linked open PRs without checking replacement viability, and the live OpenClaw chain shows the reported close sequence. I did not run an apply-path repro because this review is read-only.

## Solution Assessment

Yes, a narrow guard in the duplicate/superseded promotion/apply path is the maintainable fix. It should preserve merged or genuinely viable canonical targets while blocking unsafe replacement PRs and falling back to the open canonical issue when appropriate.
