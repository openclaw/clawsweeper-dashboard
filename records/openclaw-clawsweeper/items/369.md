## What This Changes

## Best Possible Solution

Make local-range default to a no-GitHub and no-host-download review envelope, or explicitly document and test an opt-in media-download boundary, then land only after final-head diagnostic proof is posted.

## Solution Assessment

No, not yet: the implementation mostly fits the requested capability, but the local-range network boundary should be made internally consistent before merge. The narrowest repair is to skip media-proof preprocessing for local-range by default or make it opt-in with explicit tests and docs.
