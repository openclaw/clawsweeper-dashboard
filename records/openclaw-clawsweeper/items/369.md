- **[P1] Isolate gh config and prompt for local-range:** `src/clawsweeper.ts:16163-16165`
  - body: `--local-range` deletes token env vars, but unlike existing offline `local-review` it does not point `GH_CONFIG_DIR` at an empty directory or add the no-network local-review prompt.   - confidence: 0.9
- **[P1] Treat local-range as local for Codex auth:** `src/clawsweeper.ts:16297-16298`
  - body: A standalone `--local-range` run still passes `preserveCodexAuth: localOnly`, so unless the user also adds `--local-only`, `runCodex` strips local Codex credentials and uses hosted defaults. That breaks the advertised pre-submission local workflow for normal local review setups.
  - confidence: 0.87
- **[P2] Parse rename and copy rows before diffing files:** `src/clawsweeper.ts:16057-16060`
  - body: `git diff --name-status` emits rename/copy rows with both old and new paths, but this parser keeps everything after the first tab as one filename. The follow-up `git diff -- <old\tnew>` can produce an empty patch, so local-range reviews can miss renamed or copied files.
  - confidence: 0.92
- **[P2] Avoid OWNER metadata for synthetic local PRs:** `src/clawsweeper.ts:16074-16076`
  - body: The synthetic local item is a pull request with `authorAssociation: "OWNER"`, while the proof gate treats maintainer-authored PRs differently from external PRs. That can undercut this feature's stated goal of exercising real-behavior-proof decisions for pre-submission work.
