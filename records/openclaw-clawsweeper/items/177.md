## Summary

Keep open. Current main still has command-router and hosted-webhook paths that can create or update public re-review status on closed items, and I did not find a merged fix or canonical duplicate for this exact stale closed-PR command replay.

## What This Changes

The issue asks ClawSweeper to ignore or silently skip stale re-review commands from closed, unmerged PRs instead of posting fresh acknowledgements or queuing work.

## Best Possible Solution

Add a stale-command guard that skips old re-review/comment-router commands when the item is closed, preserve explicitly supported post-close commands, and cover both hosted fast-ack and fallback router paths with regression tests.

## Reproduction Assessment

Yes at source level: live GitHub comments show the stale ack/status churn, and current main still accepts or responds to closed-item re-review commands without a closed-state stale guard. I did not run a mutating webhook/router repro.

## Solution Assessment

Yes, a narrow router/webhook guard is the best fix: skip stale closed-item re-review commands before ack/dispatch/comment creation, then add focused regression coverage for the hosted and fallback paths.
