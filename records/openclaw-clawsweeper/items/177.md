## Summary

Keep open. Current main still has a path that can acknowledge a stale `re_review` command on a closed item, and the existing stale-command guard only covers automerge/autofix activations, so this is a narrow command-router idempotency bug rather than cleanup-close material.

## What This Changes

The issue reports stale `@clawsweeper re-review` comments being acknowledged again on a closed OpenClaw PR days after closure.

## Best Possible Solution

Add a shared stale closed-item command guard so pre-close `re_review` comments on closed PRs are skipped without new public ack/status comments or dispatch, while preserving current re-review behavior for open issues and PRs.

## Reproduction Assessment

Yes, source-reproducible: route a `re_review` command for a closed PR without an existing command response marker and current main schedules a public comment instead of skipping it. I did not run the live router because this review is read-only.

## Solution Assessment

Yes; the narrowest maintainable fix is to reuse the closed-at/comment-created-at stale-command logic for `re_review` and apply it before webhook fast ack and router dispatch. A broader workflow rewrite is not needed.
