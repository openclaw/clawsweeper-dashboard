## Summary

This PR is still necessary: current main does not have the lower-effort transport fallback, while the branch adds a bounded recovery path with degraded-review disclosure, confidence capping, and focused regression coverage. Keep it open for maintainer review because accepting lower-effort verdict routing after Codex transport exhaustion is an automation-policy decision, not a closeable or already-implemented cleanup item.

## What This Changes

Adds a configurable `runCodex` lower-reasoning-effort fallback after retryable Codex transport exhaustion, degraded-review annotation/confidence capping, and regression tests.

## Best Possible Solution

Land this shape only if maintainers accept in-process degraded review routing with the confidence cap and disclosure; otherwise move the downgrade decision into an explicit recovery-lane or dispatch-policy gate.
