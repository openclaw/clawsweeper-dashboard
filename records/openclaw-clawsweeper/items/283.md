## Summary

This PR remains useful: current `main` still stops with a low-confidence placeholder after the ordinary Codex transport retries, while the branch adds a narrowly bounded recovery pass with strong runtime proof and focused coverage. No blocking correctness or security defect was found; maintainers should review and land the automation-policy change rather than close the PR.

## What This Changes

The PR adds one lower-reasoning-effort Codex attempt after retryable transport exhaustion, annotates successful fallback reviews as degraded, caps their confidence, and adds regression tests.

## Best Possible Solution

Land the single-attempt fallback with its original-deadline guard, degraded-review disclosure, retained transport diagnostics, and confidence cap, while keeping the ordinary high-effort path unchanged.

## Reproduction Assessment

Yes. The supplied built-runtime harness deterministically shows current main exhausting its same-effort retries and returning the failure path, then shows the branch recovering with exactly one low-effort invocation.

## Solution Assessment

Yes. A single effort step-down within the existing deadline is narrower and safer than globally reducing reasoning effort or extending every review timeout, and the confidence cap prevents degraded output from authorizing closure.
