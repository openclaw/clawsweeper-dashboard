## Repair Work Prompt

Implement https://github.com/openclaw/clawsweeper/issues/225 as a narrow review-automation change. Update prompts/review-item.md so stored data model changes are explicitly treated as upgrade/compatibility-sensitive and ask for migration or upgrade proof. In src/clawsweeper.ts, add a deterministic data-model-change detector parallel to configSurfaceChangeFromContext that inspects PR file paths and patches for likely persistent schema changes, including SQL DDL/index/conflict changes, migrations/schema installers, stored JSON state shape changes, vector/embedding identity or compatibility metadata, persistent cache schemas, and doctor/migration/repair code that backfills stored state. Persist durable report frontmatter such as data_model_change and data_model_surfaces, render a concise public warning when relevant, and prevent a silent automerge/repair-loop pass for material data model changes without adequate migration or upgrade proof. Keep the warning informational when proof is adequate, avoid flagging pure query-only refactors or non-semantic docs wording, preserve the flat items/closed report layout, and do not edit CHANGELOG.md.
