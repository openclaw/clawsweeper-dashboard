## Summary

This PR is still useful and not implemented on current main: the current review and assist paths still hardcode `codex` plus API login, while the PR narrowly addresses Windows `.cmd` spawning and local OAuth review runs. Keep it open for maintainer review because it changes core automation launch behavior and would benefit from focused regression coverage before merge.

## What This Changes

The PR updates `src/clawsweeper.ts` so review and assist Codex subprocesses can use `CODEX_BIN`, enable `shell: true` on Windows, and override Codex `forced_login_method` through `CLAWSWEEPER_CODEX_LOGIN_METHOD`.

## Best Possible Solution

Land the Windows subprocess compatibility fix with focused tests that pin the Codex binary, Windows shell behavior, stdin prompt delivery, and login-method default/override for review and assist runs.

## Reproduction Assessment

Yes, by running a local ClawSweeper review on Windows with npm's `codex.cmd` wrapper and a ChatGPT OAuth login. I did not execute that Windows path here, but current source plus the PR's before/after terminal output provide a clear reproduction path.

## Solution Assessment

Yes, the proposed direction is narrow: it keeps the CI default API login, limits shell spawning to Windows, and adds operator overrides for local runs. The safer merge shape is to add regression tests for the new subprocess contract before landing.
