## Summary

Keep open: the Windows spawn problem is plausible and the PR has useful terminal evidence, but the patch drops the generated Codex prompt by passing the prompt file path as the positional prompt text.

## What This Changes

The PR changes the Codex review and assist subprocess calls in `src/clawsweeper.ts` to use `CODEX_BIN`, Windows shell execution, and prompt file paths instead of stdin.

## Best Possible Solution

Keep the Windows `.cmd` wrapper fix, but preserve a supported prompt transport such as stdin or shell redirection and add a regression test proving the fake Codex process receives the full generated prompt body.

## Reproduction Assessment

Source-reproducible: current main uses `-` plus `input: prompt`, while the PR replaces that with a bare `promptPath`. Local `codex exec --help` confirms that positional `[PROMPT]` is prompt text, not a documented prompt-file input.

## Solution Assessment

No; `CODEX_BIN` and Windows shell execution address the wrapper part, but replacing stdin with a file path is not the narrow supported fix. A safer implementation should keep the prompt body on a supported input channel and test that contract.
