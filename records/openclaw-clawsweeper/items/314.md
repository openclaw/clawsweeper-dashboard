## Summary

Keep this PR open: the author-wide mode is useful and has dry-run proof, but the patch currently writes default jobs into a path shape that the existing repair worker owner gate will misinterpret, so the generated jobs are not dispatch-compatible.

## What This Changes

The PR adds a `--all-open` author-wide mode to `repair:pr-intake` and covers it with a new fake-`gh` unit test.

## Best Possible Solution

Keep the author-wide mode only if its generated durable jobs preserve the existing `jobs/<target-owner>/...` owner contract or the workflow owner resolution is deliberately updated and tested.

## Reproduction Assessment

Yes for the review finding: source inspection shows author-wide default output under `jobs/author/...`, while the worker derives the allowed owner from the first segment after `jobs/`. I did not run the PR head because this review is read-only and the command writes job files.

## Solution Assessment

No: adding an author-wide intake mode is plausible, but the current implementation is not the narrowest maintainable solution until generated jobs remain compatible with the existing owner-gated dispatch path. A safer fix is to preserve `jobs/<target-owner>/...` or deliberately update the workflow contract with tests.
