## Summary

Keep open for maintainer triage: the activity workflow cancellation is real, but current code and docs define that lane as an intentionally lossy observer that does not run review/apply, so the claimed no-comment/no-label impact is not yet proven for this path and overlaps with the confirmed exact-review capacity issue.

## What This Changes

Requests making GitHub issue/PR activity intake durable enough that cancelled `github activity to openclaw` runs cannot lose review-critical events before ClawSweeper processing.

## Best Possible Solution

Keep review-critical dispatch and recovery in the exact-review scheduler/backfill path, and only change `github-activity` durability if maintainers decide the observer should become part of reliable intake.

## Reproduction Assessment

Partially. The cancellable activity workflow behavior is source-visible and live Actions show cancelled observer runs, but there is not yet a high-confidence reproduction that this path drops exact-review dispatch or synced labels.

## Solution Assessment

Unclear. Changing `github-activity` concurrency alone is not evidently the best fix because current docs make that lane observation-only; the confirmed no-review behavior belongs to exact-review capacity/recovery unless maintainers intentionally broaden the observer contract.
