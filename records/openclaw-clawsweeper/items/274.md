## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Scrub secrets before launching Claude:** `src/clawsweeper.ts:6415`
  - body: This passes the entire parent environment to the configurable Claude binary, unlike the existing Codex path that uses `codexEnv()` to remove GitHub tokens, app private keys, and OpenAI/Codex API keys. A local `CLAUDE_BIN` wrapper or real CLI process can now receive repository credentials from the invoking shell, so the Claude path should use a scrubbed env and only add the auth/proof variables it actually needs.
  - confidence: 0.94
- **[P1] Keep Claude out of bypass mode for read-only review:** `src/clawsweeper.ts:6402-6404`
  - body: The new engine asks Claude to run with `--permission-mode bypassPermissions` while only disallowing edit tools. That still leaves non-edit tools such as shell execution outside ClawSweeper's read-only sandbox, and the dirty checkout guard will not catch untracked writes or data exfiltration. Use a read-only/restricted tool policy instead of permission bypass before enabling this engine.
  - confidence: 0.89
