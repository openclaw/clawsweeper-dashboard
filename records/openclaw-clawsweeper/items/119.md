## Summary

Keep open: the report describes a valid ClawSweeper review bug, and current main still lacks an explicit merge-result or three-way-merge guard for PR merge-risk findings.

## What This Changes

The issue asks ClawSweeper to distinguish true merge-result regressions from normal behind-main drift before applying scary merge-risk labels or blocking findings.

## Best Possible Solution

Add review-context and prompt/test coverage so deletion/regression findings use an actual merge result when available, while clean behind-main drift is reported as stale/rebase-needed instead of compatibility or session-state risk.
