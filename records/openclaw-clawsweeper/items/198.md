## Summary

Keep open: the PR is aimed at a real unsafe-close bug and adds useful apply-time guard coverage, but the current patch still treats an open clean replacement PR with unknown/no proof as a safe canonical target and the contributor supplied only build/lint/unit-test validation, not real apply behavior proof.

## What This Changes

The branch adds apply-time validation for PR-to-PR duplicate/superseded close targets, updates review guidance and README policy text, and expands apply-decision tests.

## Best Possible Solution

Require a positive proof signal for open canonical replacement PRs, keep merged replacements allowed, block unknown/no-proof replacements, and add both regression coverage and real apply dry-run proof.

## Reproduction Assessment

Yes, source-reproducible: the PR branch returns a safe result when an open clean canonical PR has no proof report or proof labels, and the added happy-path test covers that state.

## Solution Assessment

No. The direction is right, but the current implementation is not the best fix because it still allows unknown/no-proof open replacement PRs to justify duplicate/superseded closure.
