Full review comments:

- **[P1] Define the artifact contract before reading it:** `src/repair/execute-fix-artifact.ts:3352-3353`
  - body: `must_touch` and `must_touch_files` are not declared in the canonical `fix_artifact` schema, which uses `additionalProperties: false`. Normal planner output cannot contain these fields, so `repairCheckpointMustTouchFiles()` returns an empty contract and the new gate silently bypasses enforcement in production; please define one canonical field through the schema, prompts, validators, and producers before relying on it here.
  - confidence: 0.95
- **[P2] Parse status paths structurally:** `src/repair/execute-fix-artifact.ts:3318`
  - body: The helper runs plain `git status --porcelain` and later slices/trims text lines, so Git-quoted paths such as files with spaces will not match the unquoted contract path and untracked directories can collapse to a directory entry. Use a structural `--porcelain=v1 -z --untracked-files=all` parser before enforcing a fail-closed path contract.
  - confidence: 0.9
