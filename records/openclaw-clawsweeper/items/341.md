## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not harden likely_files into a contract:** `src/repair/deterministic-automerge-result.ts:61-63`
  - body: This derives `repair_contract.must_touch` directly from `likelyFiles`, so deterministic automerge repairs with small PR diffs will require every checkpoint to touch one of the original changed files. A valid fix that only adds a sibling test/helper or updates a different file to satisfy review can now be rejected; the same pattern in `src/repair/commit-finding-intake.ts` needs the same treatment, so only emit this contract from an explicit non-advisory source.
  - confidence: 0.91
- **[P2] Require the full contract shape at runtime:** `src/repair/repair-checkpoint-contract.ts:82-86`
  - body: The schema requires `must_touch`, `match`, and `scope`, but this validator only rejects bad `match`/`scope` values when those keys are present, and the executor defaults missing `match` to `any`. Handwritten or deterministic artifacts can bypass the schema and activate a fail-closed checkpoint gate with implicit semantics, so the runtime validators should enforce the same required shape and reject schema-extra keys.
  - confidence: 0.87
