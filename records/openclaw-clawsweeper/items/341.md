## Summary

Keep this PR open, but it is not merge-ready because the patch still turns advisory `likely_files` into a fail-closed checkpoint contract in deterministic producer paths and leaves runtime contract validation looser than the schema.

## What This Changes

Adds a `fix_artifact.repair_contract` schema/prompt/validator field, enforces it around repair checkpoint commits, emits it from two deterministic repair producers, and adds focused repair tests.

## Best Possible Solution

Keep `repair_contract` explicit and schema-aligned: enforce it only when an independent contract is declared, leave `likely_files` advisory, and add producer/validator tests for those boundaries.

## Solution Assessment

No as submitted. The central checkpoint helper and `-z` parser direction look maintainable, but producer activation must not convert advisory `likely_files` into a hard contract without explicit policy and tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not harden likely_files into a contract:** `src/repair/deterministic-automerge-result.ts:61-62`
  - body: This derives `repair_contract.must_touch` directly from `likelyFiles`, so deterministic automerge repairs with small PR diffs will require every checkpoint to touch one of the original changed files. A valid fix that only adds a sibling test/helper or updates a different file to satisfy review can now be rejected; the same pattern in `src/repair/commit-finding-intake.ts` needs the same treatment, so only emit this contract from an explicit source that is not advisory `likely_files`.
  - confidence: 0.91
- **[P2] Require the full contract shape at runtime:** `src/repair/repair-checkpoint-contract.ts:87-90`
  - body: The schema requires `must_touch`, `match`, and `scope`, but this validator only rejects bad `match`/`scope` values when those keys are present, and the executor then defaults missing `match` to `any`. Handwritten or deterministic artifacts can bypass the schema and activate a fail-closed checkpoint gate with implicit semantics, so the runtime validators should enforce the same required shape.
  - confidence: 0.86
