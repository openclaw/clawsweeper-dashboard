## Summary

Keep open: current main still makes `--root` additive and has no `--exclude-codex` or exclusive-root option. This is a useful but new CLI behavior request, so it needs maintainer choice on the public flag semantics before implementation.

## What This Changes

The issue requests a skill-cleaner option to audit only user-specified roots without including default Codex skill directories.

## Best Possible Solution

Add a backward-compatible explicit mode such as `--root-only` that limits filesystem discovery to supplied roots while preserving existing additive `--root` behavior by default.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current `--root` handling is additive because `discoverRoots()` includes default roots before appending `extraRoots`.

## Solution Assessment

Unclear until maintainers pick the CLI contract. The narrowest compatibility-safe path appears to be a new opt-in exclusive-root flag rather than changing existing `--root` semantics.
