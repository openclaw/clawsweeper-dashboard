## Summary

Keep open: current `main` still makes `skill-cleaner --root` additive and has no `--exclude-codex`, `--root-only`, or exclusive-root mode. This is a valid narrow CLI feature request, but it needs maintainer choice on the public flag semantics before implementation.

## What This Changes

The issue requests a way for `skill-cleaner --root` to audit only supplied roots without always including default Codex skill paths.

## Best Possible Solution

A maintainer should choose the public CLI contract, preferably preserving additive `--root` behavior and adding an explicit opt-in such as `--exclude-codex` or `--root-only` if isolated scans are desired.

## Reproduction Assessment

Yes from source inspection: `--root` values are parsed into `extraRoots` and appended after the default Codex/plugin/repo roots in `discoverRoots()`. I did not run the CLI because source inspection is enough to verify the reported semantics without scanning user-local skill/log state.

## Solution Assessment

No final solution is confirmed yet; adding an explicit opt-in flag is safer for compatibility than changing `--root` to replacement mode by default, but the maintainer should choose the public CLI shape.
