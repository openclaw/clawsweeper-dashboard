## Summary

Do not run cleanup close for this PR: live GitHub state shows it was already merged, and the merged diff has a concrete automation regression where streamed validation output is no longer captured for existing diagnostic fallback and repair logic.

## What This Changes

The PR makes fix-executor target validation stream stdout/stderr in GitHub Actions, bounds non-streamed validation buffers, and writes a no-comment failed report on SIGINT/SIGTERM.

## Best Possible Solution

Keep validation output visible in Actions while retaining a bounded, redacted captured copy for debug records, failure reports, changed-gate fallback, and validation-repair prompts.

## Reproduction Assessment

Yes from source inspection: when streaming is enabled by default in GitHub Actions and `allowFailure` is false, validation runs with inherited stdout/stderr, so `child.stdout` and `child.stderr` are unavailable for downstream diagnostic parsing.

## Solution Assessment

No. Streaming validation output is the right direction, but the implementation should stream and capture a bounded copy instead of replacing captured pipes with inherited stdio only.
