## Summary

Keep open: current main still restores an unset prior execute/fix gate value to `1`, which contradicts the documented default-closed gate behavior and can leave write-mode automation enabled after an `--open-execute-window` run.

## What This Changes

The issue asks for `--open-execute-window` gate restoration to preserve the prior closed/unset state instead of permanently setting execute/fix gates to `1`.

## Best Possible Solution

Restore unset prior gate values to a closed state, preferably across both self-heal and requeue open-window helpers, with focused regression coverage for empty prior values.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path without mutating GitHub: `readGate()` returns `""`, `openGate()` stores it, and the restore maps it to `"1"`. A live reproduction should be done only against a scratch repository because it changes Actions variables.

## Solution Assessment

Yes, restoring unset prior values to a closed state or deleting variables when they were absent is the narrow maintainable fix. The same pattern in `scripts/requeue-job.mjs` should be handled with the self-heal path to avoid a partial repair.
