## Summary

Keep open: current main still restores an unset temporary execute gate to `1`, which contradicts the documented closed-by-default gate behavior and can leave write-mode automation open after the window should close.

## What This Changes

Fix `--open-execute-window` restoration so unset prior `CLOWNFISH_ALLOW_EXECUTE` and `CLOWNFISH_ALLOW_FIX_PR` variables do not remain set to `1` after self-heal runs.

## Best Possible Solution

Normalize an unset prior gate to a closed state during temporary-window cleanup, preserve real prior values, and apply the same invariant to the requeue helper with mocked regression coverage.

## Reproduction Assessment

Yes at source level: current main maps a missing variable to `""`, stores it in `gateRestores`, and restores with `gate.previous || "1"`. I did not run the live `gh variable` mutation path because this review is read-only.

## Solution Assessment

Yes: the narrow maintainable fix is to restore an unset prior gate to closed, preferably `0` to match existing docs, while preserving explicit previous values and covering the same pattern in requeue.
