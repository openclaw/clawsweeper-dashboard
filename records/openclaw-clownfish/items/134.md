## Summary

Keep open: current main has the basic exact-head equality gate, but it still lacks the requested full 40-character SHA contract and has tests that accept a shortened marker SHA, so the central hardening work is not implemented.

## What This Changes

The issue asks Clownfish to require ClawSweeper pass markers to carry the exact full 40-character reviewed PR head SHA before automerge can proceed.

## Best Possible Solution

Land a narrow maintainer-reviewed router guardrail that centralizes full-40-hex SHA validation for pass markers and merge args, with focused tests and docs.

## Reproduction Assessment

Yes. Source inspection shows current main still accepts sha=abc123 in the trusted-pass parser test and passes abc123 into --match-head-commit, while the router lacks a full-40-character SHA validator.

## Solution Assessment

Yes, the narrow shared-validator shape is the best maintainable direction. Because this is a merge-automation trust boundary, the prepared fork commit should be reviewed or cherry-picked by a maintainer rather than auto-closed as already done.
