## Summary

Keep open: current main still has exact-head equality checks, but it does not enforce the requested full 40-character ClawSweeper pass-marker SHA contract in the router, merge-argument builder, docs, or regression tests.

## What This Changes

Request to harden Clownfish so ClawSweeper automerge pass markers are mergeable only when they carry the exact full 40-character reviewed PR head SHA.

## Best Possible Solution

Land a narrow maintainer-reviewed router hardening PR that centralizes full-40-hex SHA validation for ClawSweeper pass markers and automerge merge args, with focused tests and docs.

## Reproduction Assessment

Yes. Source inspection shows current main still accepts sha=abc123 in trusted-pass parser tests and passes shortened values into merge-arg construction while lacking a full-40-character SHA validator.

## Solution Assessment

Yes. A shared validator used by classification, final readiness, merge-argument construction, tests, and docs is the narrow maintainable fix; because this is a merge-automation trust boundary, maintainer/security review should own landing it.
