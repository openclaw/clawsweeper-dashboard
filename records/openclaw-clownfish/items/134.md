## Summary

Keep open: current main still lacks the requested full 40-character SHA validation in the trusted ClawSweeper automerge pass-marker path, even though it does compare marker SHA to GitHub's headRefOid when that live view is available.

## What This Changes

Harden Clownfish's ClawSweeper automerge pass-marker handling so mergeable pass markers and merge arguments require the exact full reviewed PR head SHA.

## Best Possible Solution

Land a narrow maintainer/security-reviewed router guardrail that centralizes full-40-hex SHA validation for ClawSweeper pass markers and automerge merge args, with focused tests and docs.

## Reproduction Assessment

Yes. Source inspection against current main shows the parser/tests accept sha=abc123 and merge-arg construction passes abc123 to --match-head-commit, while no full-40-hex validator is present.

## Solution Assessment

Yes. A shared validator across pass classification, final readiness, merge args, tests, and docs is the narrow maintainable fix; the security-sensitive landing path should stay with maintainers.
