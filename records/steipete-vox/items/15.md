## Summary

Keep this PR open for normal maintainer merge review: current main and the latest release still drop path prefixes in the TwiML and CLI URL builders, while the PR is a focused fix with regression coverage and sufficient deterministic proof.

## What This Changes

The PR changes TwiML stream URL generation and the `vox dial` TwiML fallback so route paths are appended under any `VOX_PUBLIC_BASE_URL` pathname prefix.

## Best Possible Solution

Land the focused URL-prefix fix after maintainer review, preserving existing root-base behavior while supporting path-prefixed reverse-proxy deployments.

## Reproduction Assessment

Yes. Current main uses absolute-path URL construction, and a direct URL API check reproduces the `/vox` prefix being dropped for both `/twilio` and `/twiml`.

## Solution Assessment

Yes. Updating both the TwiML stream URL helper and the `vox dial` fallback is the narrow maintainable fix for the two affected URL builders.
