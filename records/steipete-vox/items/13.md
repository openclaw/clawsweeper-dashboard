## Summary

Keep open: current main and v0.1.0 still have unbounded `query_agent` calls, and the PR is a useful repair, but the new default 10 second timeout changes upgrade behavior and needs maintainer sign-off or an upgrade-compatible default.

## What This Changes

The PR adds `VOX_AGENT_TIMEOUT_MS`, per-query HTTP/subprocess agent timeouts, `query_agent` error recovery, sanitized HTTP agent failures, and regression tests.

## Best Possible Solution

Keep the per-query timeout, recovery, and sanitization path, but land it only after maintainers explicitly accept the 10 second default or choose an upgrade-compatible default or opt-out.

## Reproduction Assessment

Yes. On current main, a `VOX_AGENT_CMD` process that never writes a JSONL response, or an HTTP agent that never responds, leaves `handleResponseDone` awaiting `opts.agent.query` without a timeout or recovery path.

## Solution Assessment

Mostly yes. Per-query timeouts with error `function_call_output` recovery and sanitized HTTP response bodies are the right repair shape, but the default timeout remains an upgrade-compatibility decision.
