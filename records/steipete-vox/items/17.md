## Summary

Keep this PR open: the workflow is useful in shape, but current main does not honor the `VITE_BASE=/vox/` value it relies on, and the new workflow grants deploy/OIDC permissions to the build job unnecessarily.

## What This Changes

Adds `.github/workflows/deploy-website.yml` to build `website/` with pnpm and deploy `website/dist` to GitHub Pages on pushes to `main`.

## Best Possible Solution

Make the website config honor `VITE_BASE` and scope Pages deployment permissions to the deploy job before merging the workflow.

## Reproduction Assessment

## Solution Assessment

No; the workflow is the right general mechanism, but it should not merge until the base-path config and least-privilege permissions are included or already present on main.
