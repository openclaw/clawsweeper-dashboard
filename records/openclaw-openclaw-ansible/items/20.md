## Summary

Keep open: current main still has the reported minimal-Debian sudo prerequisite gap. The playbook writes a scoped sudoers file and validates it with `visudo`, but the Linux prerequisite package list still omits `sudo`; the related fresh-Debian issue was closed while explicitly leaving this problem open.

## What This Changes

The issue reports that a fresh Debian 13 install without the sudo package fails when the playbook writes the scoped sudoers file for the OpenClaw user.

## Best Possible Solution

Add `sudo` to the Debian/Ubuntu prerequisite path before `user.yml` writes `/etc/sudoers.d/{{ openclaw_user }}`, keep scoped sudo and `visudo` validation intact, and update the harness so a no-sudo base image is covered.

## Reproduction Assessment

Yes at source level: current main installs system tools before `user.yml`, omits `sudo` from that package list, then writes `/etc/sudoers.d/{{ openclaw_user }}` and validates with `/usr/sbin/visudo`. I did not establish the failure by running a fresh Debian 13 VM.

## Solution Assessment

Yes: the narrow maintainable fix is to install `sudo` before the scoped sudoers task runs, without broadening OpenClaw's sudo permissions or weakening `visudo` validation.
