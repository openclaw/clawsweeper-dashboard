## Summary

Keep open: current main removed the original crashing Homebrew task, but the follow-up discussion and current source leave Linux skill-dependency/Homebrew support unresolved, so closing this as implemented would drop a real product and security decision.

## What This Changes

The issue reports that the playbook failed while installing Homebrew as root, with later comments clarifying that Linuxbrew support is now missing for skill dependencies.

## Best Possible Solution

Maintainers should choose one supported Linux skill-dependency path: either a secure non-root Linuxbrew bootstrap with PATH/idempotency validation, or explicit documentation that Homebrew is an external prerequisite and stale Ansible Homebrew variables should be removed.

## Reproduction Assessment

No high-confidence current-main crash reproduction exists because the old crashing Homebrew task is gone. The remaining Linuxbrew support gap is source-confirmed by the lack of brew tasks and issue comments showing skill dependency failures without brew, but I did not run a VM reproduction.

## Solution Assessment

Unclear: re-adding the old Homebrew task is not automatically the best solution because the installer must preserve non-root execution, idempotency, and security review. A maintainer should first decide whether this repository owns Linuxbrew installation or only documents it as an external prerequisite.
