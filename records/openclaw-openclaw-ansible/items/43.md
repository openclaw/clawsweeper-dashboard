## Summary

Keep this PR open, but it should not merge as-is: the Netbird/provider work is still useful, while the patch drops the existing `tailscale_enabled` configuration path, lacks inspectable real behavior proof, and currently reports merge conflicts.

## What This Changes

This PR replaces the Tailscale-only enablement flow with a `vpn_provider` option, adds Netbird install and auto-connect tasks, adds Tailscale auto-connect, and updates sudoers, firewall, welcome text, inventory, and docs.

## Best Possible Solution

Keep the provider abstraction, but preserve `tailscale_enabled` as a backward-compatible alias or migration path, add focused upgrade/fresh-install coverage for both VPN providers, and make the Netbird sudo/firewall boundary explicit before merge.

## Reproduction Assessment

Yes for the compatibility finding by source inspection: current `main` gates Tailscale install and firewall behavior on `tailscale_enabled`, while the PR switches those gates to `vpn_provider` without an alias. I did not run a full Ansible install in this read-only review.

## Solution Assessment

No, not as-is. The maintainable path is to keep the new provider option while preserving the existing Tailscale variable or providing an explicit tested migration.
