## Summary

## Best Possible Solution

Preserve the old `tailscale_enabled` behavior as a backward-compatible alias or add a tested migration/deprecation path, then land Netbird behind the new selector with redacted fresh-install and upgrade proof for firewall, connection, and sudo boundaries.

## Reproduction Assessment

## Solution Assessment

No, not as currently shaped; adding a provider selector is plausible, but the narrow maintainable solution must preserve the existing Tailscale setting or make the migration explicit and proven.
