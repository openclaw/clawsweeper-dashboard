## Summary

This PR still contains useful unique work, including a source-confirmed fix for the unused Tailscale auth key, but it is not merge-ready because the provider migration breaks existing Tailscale inventories, removes pre-install sudoers validation, conflicts with current main, and lacks inspectable real-host proof.

## What This Changes

The PR adds selectable Tailscale or NetBird provisioning and automatic connection, then makes firewall rules, sudo permissions, installation output, inventory examples, and documentation provider-aware.

## Best Possible Solution

Preserve `tailscale_enabled` as a compatibility alias during a documented deprecation period, keep explicit `vpn_provider` selection authoritative, restore atomic `visudo` validation, and prove minimal firewall and sudo behavior for fresh NetBird installs and upgraded Tailscale systems.

## Reproduction Assessment

Yes for the Tailscale subproblem: current-main source exposes and documents an auth key but never consumes it in the task path. NetBird support is a new capability rather than a current-main failure.

## Solution Assessment

No, not as currently shaped; the provider abstraction is reasonable, but the maintainable solution must preserve existing inventories, retain sudoers validation, and prove that each new privilege and network change is necessary.
