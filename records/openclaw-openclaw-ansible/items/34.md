## Summary

## What This Changes

Ensure Tailscale-enabled installs let the OpenClaw service user use Tailscale serve mode and honor `tailscale_authkey` for unattended setup.

## Best Possible Solution

Keep this issue open until the installer has a security-reviewed Tailscale path that wires `tailscale_authkey` with `no_log`, grants or documents the service-user operator requirement for serve mode, and validates both unattended and manual setup.

## Reproduction Assessment

Yes, by source inspection rather than by running a VM: current main declares and documents `tailscale_authkey` but never uses it, and it does not grant the service user the Tailscale operator capability the reported `tailscale serve` error requires.

## Solution Assessment

Unclear until maintainers choose the permission boundary. A narrow fix should avoid full sudo, keep auth-key handling redacted, and either grant the Tailscale operator deliberately after the service user exists or document that serve mode requires a manual operator command.
