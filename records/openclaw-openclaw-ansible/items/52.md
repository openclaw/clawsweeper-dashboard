## Summary

Keep this PR open because it is the right narrow candidate for the linked Tailscale serve/auth-key bug, but the latest head still has a blocking source-level defect: fresh auth-key installs can skip the operator grant after auto-connect because the saved backend state is not refreshed.

## What This Changes

The PR wires `tailscale_authkey` into Linux Tailscale setup, adds a post-user `tailscale set --operator` task, and adds narrowed Tailscale operator/serve sudoers entries for the OpenClaw service user.

## Best Possible Solution

Refresh or recompute Tailscale state after auth-key connection before granting the operator, make the operator task idempotent, keep auth-key handling redacted, and keep the narrowed sudoers boundary.

No. The direction is narrow and appropriate, but the latest head checks a pre-connect `BackendState`, so a fresh auth-key install can still miss the operator grant it needs.
