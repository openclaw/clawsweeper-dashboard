## Summary

Keep open. This is a useful focused fix for the Tailscale issue, but the patch can fail fresh auth-key installs because it sets the operator before the `openclaw` user is created, and it broadens Tailscale sudo authority without live proof or maintainer acceptance.

## What This Changes

The PR adds Tailscale auth-key auto-connect, sets the Tailscale operator for `openclaw`, updates manual setup guidance, and adds `tailscale set --operator`/`serve` sudoers entries.

## Best Possible Solution

Set the Tailscale operator only after the service user exists, keep privileged Tailscale setup in the root-owned installer unless maintainers explicitly accept broader sudo authority, and require redacted live proof for fresh and existing Tailscale setups.

## Reproduction Assessment

Yes, by source inspection. Current main declares and documents `tailscale_authkey` without using it, and the PR diff shows the new operator grant runs in the Tailscale task before user creation.

## Solution Assessment

No, not as currently implemented. The direction is narrow, but the operator grant needs to run after the service user exists and the sudoers expansion needs maintainer-approved security boundaries plus real-host proof.
