## Summary

Keep open: current main does not implement version pinning, and this PR is useful, but the new validation still permits range-shaped package specs that contradict the documented exact-pin contract.

## What This Changes

The PR adds an `openclaw_version` release-mode variable, uses it in the global pnpm install command, skips reinstalling matching exact pins, and updates defaults plus README/configuration docs.

## Best Possible Solution

Tighten the accepted grammar so exact pins and supported dist-tags cannot be confused with npm ranges, then keep the default `latest` behavior unchanged.

## Reproduction Assessment
