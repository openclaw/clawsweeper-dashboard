## What This Changes

## Best Possible Solution

Merge only after validation rejects semver-range aliases before `pnpm install` and maintainers accept the new release-version selection policy; keep the unsafe pnpm plugin path work tracked separately.

## Reproduction Assessment

## Solution Assessment

No, not quite: the approach is otherwise narrow, but it should exclude any value semver treats as a range before pnpm sees the package spec. That keeps the docs and reproducible-pinning guarantee true.
