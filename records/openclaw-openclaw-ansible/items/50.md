## Summary

Keep open: the force-push addressed the earlier proof, shell quoting, full-version comparison, and changelog concerns, but the new accepted version grammar still has a narrow idempotency bug for valid prerelease-style pins with additional hyphens.

## What This Changes

The PR adds an `openclaw_version` release-mode variable, validates it, uses it in `pnpm install`, skips reinstalling matching exact pins, and updates README/configuration docs.

## Best Possible Solution

Make validation, docs, and skip comparison share one explicit supported exact-version grammar while preserving `latest` as the default moving tag.

## Reproduction Assessment

Yes, from source inspection: a value such as `1.2.3-alpha-1` passes the new validation but fails the exact-version skip regex, so the install task keeps running on reruns instead of being skipped.

## Solution Assessment

Mostly yes: a default-`latest` version variable is a narrow way to add pinning, but the implementation should align validation, documentation, and skip matching before it is merge-ready.
