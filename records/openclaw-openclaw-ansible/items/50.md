## Summary

Keep open: current main does not provide `openclaw_version`, and the PR is useful, but the exact-pin skip can leave a development-mode symlink in place when switching back to release mode if the version text matches.

## What This Changes

The PR adds an `openclaw_version` release-mode variable with validation, docs, and exact-version skip logic for the global pnpm install.

## Best Possible Solution

Keep the `latest` default and exact-pin capability, but only skip pnpm install after proving the current binary is the pnpm-installed package for that pin, or force reinstall on source/mode changes.

## Reproduction Assessment

Yes, by source inspection: run development mode so `~/.local/bin/openclaw` points at the source checkout, then rerun release mode with an exact pin matching that build's reported version. The PR skip condition can then skip pnpm install based only on the version string.

## Solution Assessment

No, not yet: exact pinning with validation is a reasonable shape, but the skip must verify package source or force reinstall on mode/source changes before this is the narrow maintainable solution.
