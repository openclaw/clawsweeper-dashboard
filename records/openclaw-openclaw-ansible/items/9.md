## Summary

Current main now has a supported inventory-driven remote deployment path, while the localhost playbook remains the local one-command installer path. This is implemented on main through the collection deploy workflow and has not been proven shipped in a release tag.

## What This Changes

The issue asks for OpenClaw Ansible installation to support remote hosts instead of requiring the local localhost playbook.

## Best Possible Solution

Keep the dedicated inventory-based remote deployment workflow and continue preserving the separate localhost installer for one-command local installs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the original local installer still uses localhost/local, and current main now provides a separate remote deployment playbook and inventory workflow.

## Solution Assessment

Yes. A separate deploy playbook is the narrow maintainable solution because it keeps curl-to-local behavior intact while letting normal Ansible inventory target remote hosts.
