## Summary

Keep open: current main still accepts arbitrary nonempty active-memory recall text after only no-recall and timeout checks, and the reporter added fresh live examples of the same contamination family after the initial review.

## What This Changes

The issue asks active-memory to reject assistant chitchat from recall output so only no context or a compact memory summary is injected.

## Best Possible Solution

Harden `normalizeActiveSummary` now with focused fail-closed validation and regression tests, while leaving a stricter structured recall protocol as a separate follow-up.

## Reproduction Assessment

Yes, from source inspection and reporter live examples: nonempty assistant-style recall output can pass normalization and be wrapped into hidden active-memory context. I did not run a live GLM reproduction in this read-only review.

## Solution Assessment

Yes for the immediate fix: parser-side hardening is the narrowest maintainable repair because normal and timeout-partial paths both funnel through `normalizeActiveSummary`. A JSON-only protocol would be stronger but should not block the small regression fix.
