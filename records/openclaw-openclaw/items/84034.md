## Summary

Keep open: current main and the latest release still accept arbitrary nonempty active-memory recall text after only no-recall and timeout-boilerplate filtering, while the related fix PR was closed unmerged and is not on main.

## What This Changes

The issue asks active-memory recall to inject only no context or a compact factual memory summary, never assistant-style chitchat inside the hidden active_memory_plugin block.

## Best Possible Solution

Add focused parser-side rejection in normalizeActiveSummary with regressions for normal recall and timeout-partial recovery, then track strict structured recall output separately as a larger hardening pass.

## Reproduction Assessment

Yes, source-reproducible: current main accepts nonempty recall text after only sentinel and timeout filtering, and both normal recall and timeout recovery can wrap that text into hidden active-memory context. I did not run a credentialed GLM session, so this is not marked fully reproduced.

## Solution Assessment

Yes for the immediate fix: normalizeActiveSummary is the shared choke point for normal and timeout-partial recall output. A strict JSON recall protocol would be stronger long-term hardening, but it is larger than the narrow bug fix.
