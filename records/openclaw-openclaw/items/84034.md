## Summary

Keep open: current main and the latest release still accept arbitrary single-line active-memory recall text after only no-recall and timeout filtering, and https://github.com/openclaw/openclaw/pull/89443 is the open closing PR that should land before this issue closes.

## What This Changes

The issue asks for active-memory recall to reject assistant chitchat and inject only no recall or compact factual memory summaries.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89443 or an equivalent `normalizeActiveSummary` guard, then track strict structured recall output as separate hardening.

## Reproduction Assessment

Yes, source-reproducible: current main normalizes raw subagent output by rejecting only no-recall and timeout values, then can wrap any other nonempty single-line text into hidden active-memory context. I did not run a live GLM reproduction.

## Solution Assessment

Yes for the immediate fix: `normalizeActiveSummary` is the narrow shared choke point for normal recall and timeout-partial recovery. A strict JSON recall protocol would be stronger long-term but is broader than this issue’s hotfix path.
