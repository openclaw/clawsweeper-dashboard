## Summary

Keep open: current main still accepts arbitrary nonempty active-memory recall text after only no-recall and timeout-boilerplate filtering, so the reporter's assistant-style GLM output can become hidden prompt context.

## What This Changes

The issue asks active-memory recall to inject only no context for `NONE`/no recall or a compact factual memory summary, never assistant chitchat.

## Best Possible Solution

Harden `normalizeActiveSummary` now with focused parser-side rejection and regression coverage for normal and timeout-partial paths; consider strict structured recall output as a later follow-up.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a reported chitchat string passed to `normalizeActiveSummary` is nonempty, not a no-recall value, not timeout boilerplate, and therefore becomes a summary that the prompt-prefix path can inject.

## Solution Assessment

Yes. Parser-side hardening in `normalizeActiveSummary` is the narrowest maintainable first fix because both normal recall and timeout-partial recovery already flow through that function; a JSON protocol is stronger but larger follow-up work.
