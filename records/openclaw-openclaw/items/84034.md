## Summary

Keep open: current main and the latest release still accept arbitrary nonempty active-memory recall text unless it matches no-recall or timeout boilerplate values, so the reported assistant chatter can still enter hidden prompt context.

## What This Changes

The issue reports active-memory accepting assistant-style non-memory recall output and injecting it into `<active_memory_plugin>` instead of suppressing it as no recall.

## Best Possible Solution

Harden `normalizeActiveSummary` now so normal and timeout-partial recall reject assistant chatter before prompt injection, then consider a structured recall protocol as a separate follow-up.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a nonempty chitchat `rawReply` flows through `normalizeActiveSummary` and into the hidden prompt prefix. I did not run a live GLM session, so this is source-reproducible rather than fully reproduced locally.

## Solution Assessment

Yes, parser-side hardening in `normalizeActiveSummary` is the narrowest maintainable first fix because both normal recall and timeout-partial recovery already funnel through that function. A strict structured protocol would be stronger but is larger than the immediate bug fix.
