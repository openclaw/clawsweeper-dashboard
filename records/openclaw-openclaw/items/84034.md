The issue remains a valid canonical bug: current main still accepts nonempty active-memory subagent text without chitchat validation, and the only current fix is the still-open https://github.com/openclaw/openclaw/pull/95391. Per the issue/PR pairing rule, this should stay open until that PR lands or maintainers choose a different fix path.

## What This Changes

The issue reports that active-memory recall can accept assistant-style chitchat as a memory summary and inject it into runtime context instead of treating it as no recall.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95391 if maintainers accept the bounded normalizer-guard tradeoff, or close/replace that PR with an equivalent active-memory parser-side fix before closing this issue.

## Reproduction Assessment

Yes, source-reproducible: current main's normal and timeout recall paths both pass subagent output through normalizeActiveSummary before hidden context injection, and that normalizer has no chitchat-specific validation. I did not run a live provider reproduction.

## Solution Assessment

Yes for the immediate repair path: a shared parser-side guard in active-memory summary normalization is the narrowest hotfix because it covers normal recall and timeout-partial recovery. A structured recall protocol can remain a follow-up hardening task.
