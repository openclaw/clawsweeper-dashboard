## Summary

Keep open: current main and the latest release still accept arbitrary nonempty active-memory recall text after only no-recall and timeout filtering, while the only focused fix PR is closed unmerged.

## What This Changes

The issue reports that active-memory sometimes accepts assistant-style recall chatter as a memory summary and injects it into hidden prompt context.

## Best Possible Solution

Land a focused parser-side guard in normalizeActiveSummary with regressions for normal recall and timeout-partial recovery, then track strict structured recall output as separate hardening.

## Reproduction Assessment

Yes, source-reproducible: feed the reported chatter as raw recall output and current normalizeActiveSummary returns a nonempty single-line summary, which before_prompt_build then wraps into active_memory_plugin context. I did not run the live GLM setup in this read-only review.

## Solution Assessment

Yes, the narrow immediate fix is to harden normalizeActiveSummary because normal recall and timeout recovery already converge there. A strict JSON recall protocol is stronger long-term hardening but larger than the immediate bug fix.
