## Summary

Keep open: current main and v2026.6.8 still lack assistant-chitchat validation in active-memory summary normalization, and both linked fix PRs are closed unmerged, so no canonical landed fix owns the remaining bug.

## What This Changes

The issue requests that active-memory drop assistant-style recall boilerplate and inject only `NONE` or compact factual memory summaries into hidden active-memory context.

## Best Possible Solution

Add a focused active-memory parser-side guard that drops assistant-style non-memory replies before `buildPromptPrefix`, covers timeout-partial and grounded normal recall paths, and leaves strict structured recall output as separate hardening.

## Reproduction Assessment

Yes, source-reproducible: current main still accepts nonempty normalized assistant text in the timeout-partial path and has no factual-memory validation in `normalizeActiveSummary`. I did not run a live GLM/provider reproduction.

## Solution Assessment

Yes for the immediate fix: the narrowest maintainable layer is the shared active-memory summary normalization and acceptance boundary. A JSON-only recall protocol would be stronger long-term but is broader than the hotfix this issue needs.
