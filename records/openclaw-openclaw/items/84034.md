## Summary

Keep open: current main still has the weak active-memory parser boundary, and the related fix PR is closed unmerged rather than present on main.

## What This Changes

The issue reports active-memory recall accepting assistant-style chatter as a memory summary and injecting it into `<active_memory_plugin>` hidden context.

## Best Possible Solution

Add parser-side rejection in `normalizeActiveSummary` for the reported assistant chatter family now, with focused regressions for normal recall and timeout-partial recovery, then track strict structured recall output separately.

## Reproduction Assessment

Yes, source-reproducible: current main accepts any nonempty recall text that is not a no-recall value or timeout boilerplate, then normal and timeout-partial paths can inject it as active-memory context. I did not run a live GLM reproduction in this read-only review.

## Solution Assessment

Yes, parser-side hardening at `normalizeActiveSummary` is the narrowest first fix because both implicated paths already use that function. A structured JSON recall protocol is better long-term hardening but is larger than the immediate bug fix.
