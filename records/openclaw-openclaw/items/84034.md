## Summary

Keep open: current main and latest release still accept arbitrary nonempty active-memory recall text after only NONE/timeout filtering, so the reported assistant chitchat can still be injected into the hidden prompt prefix.

## What This Changes

The issue asks active-memory recall to inject only NONE/no context or a compact factual memory summary, never assistant-style chatter, clarification boilerplate, time/model announcements, or help-offer text.

## Best Possible Solution

Harden `normalizeActiveSummary` with focused parser-side rejection for assistant chatter now, covering normal and timeout-partial paths, and leave a stricter structured recall protocol as a separate follow-up if needed.

## Reproduction Assessment

Yes, from source inspection: current main accepts any nonempty, non-NONE, non-timeout recall text and injects it as active-memory context. I did not run a live GLM reproduction in this checkout, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes, the narrow first fix is parser-side validation in `normalizeActiveSummary` because both normal recall and timeout-partial recovery already funnel through that function. A JSON-only protocol would be stronger but is larger than the immediate bug fix.
