## Summary

Keep open: current main and the latest release still accept arbitrary nonempty active-memory recall text outside a small no-recall/timeout list, so the reported assistant chatter can still reach the hidden prompt prefix. The closed, unmerged fix PR is useful evidence, but it is not on main.

## What This Changes

The issue asks active-memory recall to inject only no context or compact factual memory summaries, never assistant-style chatter or clarification text.

## Best Possible Solution

Harden `normalizeActiveSummary` so both normal recall and timeout-partial recovery reject the reported assistant chatter while preserving valid compact memory summaries; consider strict structured recall output as a later protocol hardening step.

## Reproduction Assessment

Yes. Source inspection shows current main will normalize and inject nonempty assistant chatter unless it matches the small no-recall or timeout-boilerplate lists; I did not run a live GLM reproduction.

## Solution Assessment

Yes. Parser-side validation in `normalizeActiveSummary` is the narrowest maintainable first fix because both normal and timeout recovery paths already pass through it; a JSON protocol is stronger but larger follow-up work.
