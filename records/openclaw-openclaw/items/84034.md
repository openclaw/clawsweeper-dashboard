## Summary

Keep open: current main and the latest shipped release still accept arbitrary nonempty active-memory recall text after only no-recall and timeout filtering, and the related fix PR is closed unmerged rather than implemented on main.

## What This Changes

The issue requests parser-side hardening so active-memory recall does not inject assistant chatter into `<active_memory_plugin>` when no valid memory summary exists.

## Best Possible Solution

Apply a focused normalizer hotfix in active-memory with regression coverage for the reported chatter variants and valid-summary controls, then track strict structured recall output as follow-up hardening.

## Reproduction Assessment

Yes, at source level: current main normalizes raw active-memory subagent output by rejecting only no-recall and timeout values, then accepts remaining nonempty text for prompt injection. I did not run the reporter's live GLM setup, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes for the immediate fix: normalizeActiveSummary is the shared choke point for normal recall and timeout-partial recovery. A strict JSON recall protocol would be stronger long-term hardening, but it is larger than the narrow bug fix.
