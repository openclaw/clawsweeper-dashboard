## Summary

Keep open: current main and the latest stable tag still accept arbitrary nonempty active-memory recall text unless it is a known no-recall value or timeout boilerplate, so the reported assistant-style chatter can become hidden prompt context.

## What This Changes

The issue reports active-memory recall accepting assistant-style fallback text as memory context instead of suppressing it as no recall.

## Best Possible Solution

Add narrow fail-closed validation in active-memory recall parsing with regression tests for normal and timeout-partial paths, while leaving structured JSON recall and cross-plugin tag registry work to separate follow-ups.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: feed assistant-style nonempty text through the active-memory recall result and it is normalized as a summary, then wrapped into the hidden prompt prefix. I did not run a live GLM model reproduction.

## Solution Assessment

Yes. Parser-side hardening in normalizeActiveSummary is the narrowest maintainable first fix because normal recall and timeout_partial recovery already funnel through that function; a strict structured response protocol is a larger follow-up.
