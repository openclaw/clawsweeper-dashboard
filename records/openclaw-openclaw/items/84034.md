## Summary

Keep open: current main and the latest shipped release still normalize active-memory recall by accepting arbitrary nonempty text after only no-recall and timeout checks, so the reported assistant chatter can still become hidden active-memory context. The closed related fix PR is useful source work, but it was not merged and no other canonical open fix was found.

## What This Changes

The issue asks active-memory to reject assistant-style recall chatter and inject only no context or a compact factual memory summary.

## Best Possible Solution

Add a focused parser-side guard in normalizeActiveSummary with regressions for normal recall and timeout-partial recovery, then track strict structured recall output separately if maintainers want deeper hardening.

## Reproduction Assessment

Yes, at source level: feed a chatter string through the current rawReply or timeout-partial path and normalizeActiveSummary returns it as a valid nonempty summary unless it matches only no-recall or timeout boilerplate. I did not run a live credentialed provider reproduction in this read-only review.

## Solution Assessment

Yes for the immediate fix: normalizeActiveSummary is the shared choke point for normal and timeout-partial recall output, so parser-side rejection plus focused regression tests is the narrowest maintainable repair. A strict JSON response protocol is a larger follow-up, not required to fix the reported leakage class.
