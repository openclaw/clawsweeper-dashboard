## Summary

Current main still lacks the requested left-side conversation management workflow. The existing Control UI supports session listing, session selection, deletion, sorting/filtering in the Sessions admin table, and a per-session label field, but not persistent manual ordering, folder/tag/custom grouping, moving conversations between groups, or double-click/right-click rename in a left conversation sidebar.

## What This Changes

Add ChatGPT/Claude-style management to the left conversation list, including drag ordering, folders/tags/custom categories, rename, and moving conversations between groups.

## Best Possible Solution

Design an additive session organization contract in the Gateway/session store for user-defined folders or tags/categories, manual ordering, and move/update operations, then implement the Control UI conversation list interactions against that contract with focused protocol, store, UI, and docs coverage.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection gives a high-confidence absence check: the Control UI and session protocol expose selection/listing/labels but not the requested sidebar organization primitives.

## Solution Assessment

No current patch is proposed, and an automated UI-only label tweak would not be the best fix. The maintainable solution is a small product design pass followed by an additive persistent session organization API and matching Control UI behavior.
