## Summary

Current main still lacks the requested left-side conversation management workflow. The existing Control UI has navigation groups, a chat session select, and a Sessions admin table with label editing, sorting/filtering, and deletion, but no persistent manual ordering, folder/tag/custom grouping, move operation, or double-click/right-click rename flow.

## What This Changes

Request to add left-side conversation list management with drag ordering, folders/tags/custom categories, rename, and move-between-group workflows.

## Best Possible Solution

Define an additive session organization contract in the Gateway/session store, then implement the Control UI conversation-list interactions and docs against that stable protocol.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives a high-confidence absence check: the UI and protocol expose session listing, selection, labels, and overrides, but not the requested organization primitives.

## Solution Assessment

No current patch is proposed, and a UI-only change over the existing label field would not solve the request. The maintainable path is a small product/API design pass followed by additive protocol, store, UI, and docs work.
