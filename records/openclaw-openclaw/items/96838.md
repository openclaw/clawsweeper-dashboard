## Summary

Keep open: the PR addresses a real WebChat/channel Stop gap, but the branch still retargets broad `agent:main` Stop requests to hidden channel sessions and overlaps the broader open WebChat Stop recovery work.

## What This Changes

The PR changes WebChat `/stop` handling and Gateway `chat.abort`/`chat.send` abort resolution so selected or inferred hidden channel sessions, including WeChat sessions, can be aborted.

## Best Possible Solution

Keep explicit selected channel-session abort support, remove or explicitly approve the broad hidden-channel guessing paths, and coordinate the remaining Stop recovery behavior with https://github.com/openclaw/openclaw/pull/92003 before merge.

## Reproduction Assessment

## Solution Assessment

No, not as submitted. The explicit channel-key exception is the narrow fix boundary, but broad UI/Gateway guessing needs removal or explicit maintainer acceptance before merge.
