Keep open. Current main still leaves ordinary non-forum Telegram group messages on the no-last-route path, and the core recorder can return from that path before the inbound metadata write is awaited; the linked repair PR is closed unmerged, so there is still a narrow actionable persistence bug.

Telegram group chat history should be durably preserved across multi-day group conversations and gateway restarts.

## Best Possible Solution

Add focused regression coverage and make regular Telegram group inbound session metadata durable without changing the one-session-per-regular-group contract or forum-topic isolation.
