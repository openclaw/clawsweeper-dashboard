## Summary

## What This Changes

Adds an iOS native ElevenLabs realtime STT WebSocket client, batch/WAV helpers, Settings language/model controls, and TalkModeManager routing that auto-selects ElevenLabs STT when a Talk API key is available.

## Best Possible Solution

Route iOS realtime transcription through the Gateway-owned Talk transcription session API, keep Apple Speech as the compatible default unless maintainers approve an opt-in provider mode, and prove fresh-install, upgrade, PTT, continuous, interrupt, and fallback behavior on a real device.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route iOS STT through the Gateway Talk contract:** `apps/ios/Sources/Voice/TalkModeManager.swift:596-600`
  - body: This creates a native ElevenLabs websocket client from the Talk API key even though current main already exposes Gateway-owned transcription sessions and a registered ElevenLabs realtime provider. Merging this would give iOS a separate provider/auth/readiness/fallback implementation that can drift from `talk.session.*` behavior and silently change existing users with an ElevenLabs key from Apple Speech to this new path.
  - confidence: 0.9
- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: A fixed 300 ms sleep after `commit()` does not guarantee ElevenLabs has returned `committed_transcript`. If the final arrives after this window, the method sets `isListening = false` and disconnects the client, so PTT can return an empty or stale partial transcript.
  - confidence: 0.88
- **[P1] Recover active captures after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:615-620`
  - body: After websocket/auth failures, the handler only calls `restartRecognitionAfterError()`, which immediately returns outside continuous mode and does not request Apple Speech permission for a fallback path that was skipped when ElevenLabs was selected. Push-to-talk attempts can therefore end with no transcript, and continuous fallback may fail for users who never granted speech recognition permission.
  - confidence: 0.82
- **[P2] Gate audio sends on a real websocket ready state:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `isConnected` is set immediately after `resume()`, before the websocket handshake or ElevenLabs `session_started` message. Callers can send audio and commits into a socket that later fails authentication or readiness, making the guards look successful while speech input is dropped.
  - confidence: 0.8
