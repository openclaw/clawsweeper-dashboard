## Summary

Keep open for maintainer review: this is a useful iOS Talk feature direction, but the branch is not safe to merge because current main has since added a Gateway-owned ElevenLabs realtime transcription provider/session path, while this PR still direct-connects from iOS, lacks real behavior proof, and has source-reproducible transcript-loss/fallback bugs.

## What This Changes

The PR adds direct iOS ElevenLabs realtime WebSocket STT, batch STT/WAV helpers, Talk Mode streaming integration, and Settings STT model/language pickers.

## Best Possible Solution

Rebase around the current Gateway Talk transcription/session APIs, use the bundled ElevenLabs provider through Gateway-managed credentials, preserve Apple Speech permission and locale fallback behavior, and provide redacted real-device proof.

## Reproduction Assessment

No high-confidence end-to-end real-device reproduction is available. The blocking issues are source-reproducible from the PR head plus the current Talk/ElevenLabs provider contract: commit/final transcript handling, fixed PTT teardown, and fallback recovery can lose speech input.

## Solution Assessment

No. The feature may be useful, but the best merge path is to rework it around the Gateway-owned Talk transcription provider/session model instead of adding a separate native vendor socket and duplicated STT settings.

Full review comments:

- **[P1] Route iOS STT through the Gateway Talk contract:** `apps/ios/Sources/Voice/TalkModeManager.swift:593-600`
  - body: Current main already has a bundled ElevenLabs realtime transcription provider plus talk.session.create/appendAudio for transcription sessions. Creating a separate native ElevenLabs socket and STT settings path would duplicate provider config, credential routing, and transcript behavior instead of using the supported source of truth.
  - confidence: 0.86
- **[P1] Send the documented manual commit frame:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:71`
  - body: The current provider and ElevenLabs realtime contract commit manual audio by sending an input_audio_chunk frame with commit=true. This standalone message_type=commit frame can fail to produce a committed_transcript, so PTT release can submit empty or stale partial text.
  - confidence: 0.88
- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-402`
  - body: After commit(), the code sleeps for a fixed 300 ms, then clears isListening and stops recognition. If the committed transcript arrives later, handleTranscript ignores it or the receive task is already canceled, so push-to-talk can lose the final utterance.
  - confidence: 0.9
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: An ElevenLabs-first start skips SFSpeechRecognizer authorization. When retries flip useElevenLabsSTT to false and restart recognition, first-time users can fall back into Apple Speech without permission and still lose speech input instead of recovering.
  - confidence: 0.84
- **[P2] Preserve the configured Apple Speech locale:** `apps/ios/Sources/Voice/TalkModeManager.swift:639`
  - body: Current main resolves Apple Speech through TalkSpeechLocale using local and gateway speech-locale settings. This fallback constructs a generic SFSpeechRecognizer, so users with a configured speech language can silently lose that setting after fallback.
  - confidence: 0.78
