## Summary

Keep open: this is a live feature PR whose central iOS ElevenLabs STT capability is not implemented on current main, and the branch still has source-visible reliability defects plus unresolved privacy/architecture and proof gates.

## What This Changes

Adds native iOS ElevenLabs realtime and batch STT clients, STT model/language settings, TalkModeManager integration, and Swift source-list entries.

## Best Possible Solution

Approve the Talk architecture and privacy boundary first, then land either a gateway-owned realtime Talk integration or an explicit-opt-in native iOS STT path with robust readiness, fallback, docs, tests, changelog, and redacted device proof.

## Reproduction Assessment

Yes for the PR review findings from source inspection: the branch shows the fixed 300 ms PTT wait, optimistic WebSocket readiness, fallback path, PTT recovery gap, and automatic STT enablement. No live iOS behavior was reproduced because the PR does not attach inspectable runtime proof.

## Solution Assessment

No. The direct automatic native vendor WebSocket path is not the best merge shape until maintainers approve the privacy and architecture boundary; a gateway-owned realtime/relay design or explicit-opt-in native mode is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wait for committed PTT transcripts before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The ElevenLabs PTT stop path sends commit and then waits a fixed 300 ms before clearing listening state and stopping recognition. If committed_transcript arrives later, handleTranscript ignores it and PTT can submit an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Request speech permission before Apple fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: When realtime STT reaches the retry limit, this path flips useElevenLabsSTT to false and restarts recognition directly. Users who started on ElevenLabs may never have granted Apple Speech permission, so fallback can fail instead of recovering input.
  - confidence: 0.86
- **[P1] Recover active push-to-talk after STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime error handler always calls restartRecognitionAfterError, but that helper returns unless captureMode is continuous. In push-to-talk mode, socket or auth failures can leave the active capture without recovered transcript or immediate Apple fallback.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready after the server handshake:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: isConnected becomes true immediately after task.resume(), before the WebSocket upgrade or first ElevenLabs server message succeeds. Callers can send audio and commits to a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84
