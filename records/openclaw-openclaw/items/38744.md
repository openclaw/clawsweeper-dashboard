## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route iOS STT through the Gateway Talk contract:** `apps/ios/Sources/Voice/TalkModeManager.swift:588-600`
  - body: Current main already exposes Gateway-owned transcription sessions and a bundled ElevenLabs realtime provider. Opening a separate native ElevenLabs socket from iOS duplicates credential routing, provider config, model/language selection, event semantics, and future fallback behavior instead of using talk.session.create plus talk.session.appendAudio.
  - confidence: 0.88
- **[P1] Send manual commits as input audio chunks:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:71`
  - body: The PR sends a standalone {"message_type":"commit"} frame, but current main and the official ElevenLabs realtime STT reference use an input_audio_chunk frame with commit=true for manual commit. This can prevent PTT release from receiving the committed transcript. ([elevenlabs.io](https://elevenlabs.io/docs/api-reference/speech-to-text/v-1-speech-to-text-realtime?utm_source=openai))
  - confidence: 0.9
- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-402`
  - body: After commit(), the code sleeps for a fixed 300 ms, then clears isListening and stops recognition. If the committed_transcript arrives later, handleTranscript returns early or the receive loop has already been canceled, so PTT can submit an empty or stale partial transcript.
  - confidence: 0.91
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: The ElevenLabs-first path skips SFSpeechRecognizer authorization, but the error fallback flips useElevenLabsSTT to false and calls restartRecognitionAfterError(), which starts Apple Speech without asking for permission. First-time users can therefore fail both the realtime path and the fallback path.
  - confidence: 0.84
- **[P2] Preserve the configured Apple Speech locale:** `apps/ios/Sources/Voice/TalkModeManager.swift:639`
  - body: Current main resolves Apple Speech through TalkSpeechLocale using local and gateway speech-locale settings. This PR's Apple Speech path constructs SFSpeechRecognizer() directly, so users with a configured speech language can silently lose that preference.
  - confidence: 0.8
- **[P2] Mark the websocket ready only after session_started:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:48`
  - body: connect() sets isConnected=true immediately after task.resume(), before ElevenLabs sends session_started. With an invalid key, unsupported account, or failed upgrade, audio and commit frames can be accepted by local guards while the socket is still failing, delaying fallback and losing speech input.
  - confidence: 0.78
