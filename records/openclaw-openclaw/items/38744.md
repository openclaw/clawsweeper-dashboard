## Summary

Keep this PR open, but it should not merge as written: source review confirms multiple blocking realtime STT bugs, missing real-device proof, and a product/privacy decision around native iOS microphone streaming.

## What This Changes

The PR adds native iOS ElevenLabs realtime and batch STT clients, TalkModeManager wiring, STT model/language Settings controls, and Swift source-list entries.

## Best Possible Solution

Land a maintainer-approved iOS Talk design that uses the Gateway transcription/realtime relay where possible, or an explicit tokenized native STT path with robust commit handling, fallback behavior, tests, docs, and real-device proof.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head, current main's provider/session code, and the official ElevenLabs realtime contract reproduces the commit framing, fixed PTT teardown, fallback permission, PTT recovery, and readiness blockers. No live iOS device run was performed.

## Solution Assessment

No. The branch bypasses the gateway-owned provider/session boundary and automatically streams microphone audio with client-side auth, so a maintainer-approved relay or tokenized native design is safer before code-level repair.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Send commits as input audio chunks:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:71`
  - body: The realtime API and current main provider commit manual transcription by sending an `input_audio_chunk` with `commit: true`, but this method sends a standalone `message_type: "commit"` frame. PTT release can therefore wait for a committed transcript that the provider never emits.
  - confidence: 0.9
- **[P1] Wait for the committed PTT transcript:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: After `commit()`, this path sleeps for a fixed 300 ms and then clears listening state and stops recognition. If the committed transcript arrives after that window, `handleTranscript` drops it and PTT can submit an empty or truncated partial.
  - confidence: 0.9
- **[P1] Request Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: When realtime STT reaches the retry limit, this path flips to Apple Speech and restarts recognition without first requesting Speech permission. Users who started on ElevenLabs may never have granted it, so fallback can fail instead of recovering input.
  - confidence: 0.86
- **[P1] Recover push-to-talk after realtime STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime error handler always calls `restartRecognitionAfterError()`, but that helper returns unless capture mode is continuous. In push-to-talk mode, socket or auth failures can leave the active capture without a recovered transcript or immediate fallback.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready after server confirmation:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:48`
  - body: `isConnected` becomes true immediately after `task.resume()`, before the WebSocket upgrade or `session_started` event succeeds. Callers can send audio and commits to a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84
