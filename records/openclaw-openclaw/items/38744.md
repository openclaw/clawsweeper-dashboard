## Summary

Keep open: the ElevenLabs iOS STT feature is not implemented on current main, but this PR is not mergeable as-is because it has blocking realtime/PTT fallback bugs, is conflicting with newer iOS Talk Mode behavior, and still lacks real behavior proof.

## What This Changes

The PR adds an iOS ElevenLabs realtime WebSocket speech-to-text path, batch STT fallback client, PCM/WAV utilities, and Settings pickers for STT model and language.

## Best Possible Solution

Rework the feature on current main with an explicit iOS STT provider design, robust realtime session/commit handling, preserved Apple Speech locale and permission fallback, and real iPhone or simulator-plus-service proof before merge.

## Reproduction Assessment

No high-confidence end-to-end runtime reproduction was established in this read-only review. The blocking failure paths are source-reproducible from the PR branch: PTT uses a fixed post-commit sleep, realtime fallback restarts Apple Speech without ensuring permission, and current-main speech locale support is not preserved.

## Solution Assessment

No. The feature direction may be useful, but this branch is not the best merge path until it is rebased onto current iOS Talk Mode, the realtime reliability bugs are fixed, and maintainers confirm the native client/provider ownership model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The realtime PTT path sends `commit()` and then sleeps for a fixed 300 ms before clearing `isListening` and disconnecting. A slower `committed_transcript` will arrive after `handleTranscript` starts returning early or after the receive loop is canceled, so PTT can submit an empty or stale partial transcript instead of the final result.
  - confidence: 0.9
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: When realtime STT reaches the retry limit, this flips `useElevenLabsSTT` to false and immediately restarts recognition. Startup skipped `requestSpeechPermission()` while ElevenLabs was active, so first-time users can fall back into Apple Speech without authorization and lose speech input instead of recovering.
  - confidence: 0.86
- **[P2] Defer connected state until the realtime session is accepted:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `isConnected` becomes true immediately after `task.resume()`, before the WebSocket handshake or a server session message succeeds. With an invalid key or rejected realtime session, audio and commit frames are accepted by the guards and queued onto a failing socket, delaying fallback and making the UI think STT is live.
  - confidence: 0.82
- **[P2] Preserve the configured Apple Speech locale:** `apps/ios/Sources/Voice/TalkModeManager.swift:638-640`
  - body: Current main resolves Apple Speech through `TalkSpeechLocale` using the local and gateway speech-locale settings. The branch's Apple fallback constructs a generic `SFSpeechRecognizer()`, so after rebase the fallback path would ignore the configured speech language and regress non-default locale users.
  - confidence: 0.78
