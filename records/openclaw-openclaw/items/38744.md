The PR adds native iOS ElevenLabs realtime and batch STT clients, PCM/WAV conversion, TalkModeManager integration, STT model/language settings, and Swift source-list entries.

## Best Possible Solution

Land a maintainer-approved Talk realtime path that uses the gateway relay/provider contract when preferred, or an explicitly opt-in native iOS path with robust WebSocket readiness, fallback, permission handling, docs, changelog, and real-device validation.

## Solution Assessment

No. The proposed direct native vendor STT path is a useful prototype, but it is not the narrowest maintainable solution until maintainers choose it over the existing gateway realtime relay/provider path and harden the privacy and failure behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Wait for committed PTT transcripts before stopping:** `apps/ios/Sources/Voice/TalkModeManager.swift:392-399`
  - body: The ElevenLabs PTT stop path sends commit() and then waits a fixed 300 ms before setting isListening = false and tearing recognition down. If committed_transcript arrives after that window, handleTranscript ignores it or the receive task is already gone, so PTT can return an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Request speech permission before Apple fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:620-627`
  - body: After realtime STT reaches the retry limit, the handler flips useElevenLabsSTT to false and restarts through the Apple path without requesting SFSpeechRecognizer authorization for users who started on ElevenLabs. Invalid keys or realtime access failures can leave speech input broken instead of degrading cleanly.
  - confidence: 0.86
- **[P1] Recover push-to-talk after realtime STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620-627`
  - body: The realtime STT error handler calls restartRecognitionAfterError(), but that helper returns unless captureMode is continuous. In push-to-talk, a WebSocket/auth failure can end the active capture with an empty transcript and no in-place fallback.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready only after the server handshake:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: isConnected is set immediately after task.resume(), before the WebSocket upgrade or ElevenLabs session_started message succeeds. With an invalid key or unavailable service, callers can enqueue audio and commits onto a socket that has not become usable, delaying failure and losing speech input.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `apps/ios/Sources/Settings/SettingsTab.swift:339-358`
  - body: This adds a user-facing iOS feature and settings surface, but the PR file list does not include CHANGELOG.md. Repo policy requires user-facing feat changes to be represented in release notes.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention because the PR auto-enables direct live microphone streaming to ElevenLabs from an existing Talk API key.

Concerns:

- **[medium] Require opt-in for direct ElevenLabs microphone streaming:** `apps/ios/Sources/Voice/TalkModeManager.swift:2128`

That sensitive data-flow change needs explicit user-facing opt-in or a maintainer-approved gateway relay design before merge.
