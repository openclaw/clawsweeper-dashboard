## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The realtime PTT path sends `commit()`, sleeps for a fixed 300 ms, then clears `isListening` and calls `stopRecognition()`. If ElevenLabs delivers `committed_transcript` after that window, `handleTranscript` will ignore it or the receive loop will already be canceled, so PTT can submit an empty or stale partial transcript.
  - confidence: 0.9
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: After repeated realtime STT failures, this path flips `useElevenLabsSTT` to false and restarts recognition, but startup skipped `requestSpeechPermission()` while ElevenLabs was active. First-time users can therefore fall back into Apple Speech without authorization and lose speech input instead of recovering.
  - confidence: 0.86
- **[P1] Recover active PTT captures after realtime STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime STT error handler always calls `restartRecognitionAfterError()`, which returns unless `captureMode == .continuous`. In push-to-talk, a websocket/auth failure leaves the active capture with no fallback path, so audio stops producing transcripts until the user starts over.
  - confidence: 0.82
- **[P2] Defer connected state until the realtime session is accepted:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:48`
  - body: `isConnected` becomes true immediately after `task.resume()`, before the WebSocket handshake or first server session message succeeds. With an invalid key or rejected realtime session, audio and commit frames pass the guard and queue onto a failing socket while the UI treats STT as live.
  - confidence: 0.82
- **[P2] Preserve the configured Apple Speech locale:** `apps/ios/Sources/Voice/TalkModeManager.swift:638-640`
  - body: Current main resolves Apple Speech through `TalkSpeechLocale` using local and gateway speech-locale settings. The branch's Apple fallback constructs a generic `SFSpeechRecognizer()`, so after rebase the fallback path would ignore configured speech language for non-default locale users.
  - confidence: 0.78
