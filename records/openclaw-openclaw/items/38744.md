## Summary

Keep open: current main does not include the native iOS ElevenLabs STT path, and the PR still has reliability plus privacy/product blockers before it is mergeable.

## What This Changes

The PR adds native iOS ElevenLabs realtime and batch STT clients, PCM/WAV conversion, TalkModeManager integration, STT settings, and Swift source-list entries.

## Best Possible Solution

Choose the Talk architecture first, then either route iOS through the gateway-owned realtime relay/provider surface or approve native iOS STT with explicit opt-in, robust readiness/fallback handling, docs, changelog, and device validation.

## Reproduction Assessment

Yes for the PR review findings from source inspection: the diff shows the fixed PTT commit window, premature WebSocket readiness state, and fallback paths. Live iOS microphone behavior still needs real-device validation.

## Solution Assessment

No. The native vendor STT path is a useful prototype, but current main already has gateway-owned realtime provider and relay surfaces, so direct iOS streaming should not merge without an explicit architecture and privacy decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Wait for committed PTT transcripts before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:392-395`
  - body: The ElevenLabs PTT stop path sends `commit()` and then waits a fixed 300 ms before setting listening false and stopping recognition. A delayed `committed_transcript` can arrive after teardown and be ignored, leaving PTT with an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Recover Apple fallback after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:623-627`
  - body: When the realtime socket fails, the handler can re-enter the same ElevenLabs path or restart without preparing the Apple Speech path. Invalid keys or missing realtime access can leave speech input broken instead of degrading to the existing recognizer.
  - confidence: 0.86
- **[P1] Recover push-to-talk after realtime STT socket errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:623-627`
  - body: The realtime error restart path is continuous-mode oriented, so active push-to-talk captures can end with no recovered transcript after websocket or auth failures. Add a PTT-specific fallback or completion path so the current capture does not silently fail.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready only after the server handshake:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `isConnected` is set immediately after `task.resume()`, before the WebSocket upgrade or ElevenLabs `session_started` event succeeds. Callers can enqueue audio and commits on a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `apps/ios/Sources/Settings/SettingsTab.swift:339`
  - body: This is a user-facing iOS feature and settings change, but the PR file list does not include `CHANGELOG.md`. Repo policy requires release-note coverage for user-facing `feat` changes.
  - confidence: 0.82
