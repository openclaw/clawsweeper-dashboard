## Summary

Keep this PR open for maintainer review, but do not merge it as-is. Current main still lacks iOS ElevenLabs STT, so the feature is not obsolete; however, the proposed native direct-to-ElevenLabs path has blocking reliability issues, an automatic live-microphone third-party data-flow concern, and overlaps with newer gateway/provider realtime transcription architecture that should be chosen deliberately.

## What This Changes

This PR adds native iOS ElevenLabs realtime and batch STT clients, PCM conversion, TalkModeManager integration, STT model/language settings, and Swift source-list entries.

## Best Possible Solution

Decide the Talk architecture first. If iOS should use provider-generic realtime Talk, wire the iOS client to the gateway realtime session/relay so provider credentials stay gateway-side; if a native iOS ElevenLabs STT path is preferred, add explicit opt-in, robust WebSocket readiness, committed-transcript waiting, Apple Speech fallback permission/recovery, docs, and focused iOS tests before merge.

## Reproduction Assessment

Yes for the current-main absence and the PR-introduced failure modes: current main has no iOS ElevenLabs STT path, and the PR diff statically shows the 300 ms committed-transcript wait, early connected flag, and fallback restart paths. Live end-to-end audio behavior still needs device validation.

## Solution Assessment

No. The proposed implementation is a useful prototype, but auto-enabling a native vendor WebSocket client is not the safest maintainable solution until maintainers choose it over the existing gateway/provider realtime architecture and harden the failure and privacy paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Wait for committed PTT transcripts before stopping:** `apps/ios/Sources/Voice/TalkModeManager.swift:392-395`
  - body: The ElevenLabs PTT stop path sends `commit()` and then sleeps for a fixed 300 ms before setting `isListening = false` and disconnecting recognition. If the `committed_transcript` arrives later, `handleTranscript` is ignored or the receive task is gone, so PTT can return an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Recover Apple fallback after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:623-627`
  - body: After realtime STT reaches the retry limit, this handler flips `useElevenLabsSTT` off and calls `restartRecognitionAfterError()`. That path only restarts continuous capture and does not request SFSpeech permission first, so users who started on ElevenLabs can fail to recover to Apple Speech, and PTT captures do not recover in-place.
  - confidence: 0.86
- **[P2] Mark the WebSocket ready only after the server handshake:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47`
  - body: `isConnected` is set immediately after `task.resume()`, before the WebSocket upgrade or the ElevenLabs `session_started` message succeeds. With an invalid key or unavailable service, callers can enqueue audio and commits onto a socket that has not become usable, delaying failure and losing speech input.
  - confidence: 0.84
