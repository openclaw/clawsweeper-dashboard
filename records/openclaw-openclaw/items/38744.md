## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route iOS STT through the Gateway Talk contract:** `apps/ios/Sources/Voice/TalkModeManager.swift:588-600`
  - body: This adds a second provider stack inside the iOS app and automatically selects it when an ElevenLabs key exists, bypassing the Gateway-owned Talk provider/session surface. That creates conflicting credential, model, fallback, and session-event behavior for the same Talk feature.
  - confidence: 0.88
- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-402`
  - body: After `commit()`, the code sleeps for a fixed 300 ms and then clears `isListening` and stops recognition. If the committed transcript arrives later, `handleTranscript` ignores it or the receive loop has already been canceled, so PTT can submit an empty or stale partial transcript.
  - confidence: 0.91
- **[P1] Recover active captures after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:613-621`
  - body: Realtime websocket or auth failures do not reliably fall back inside the current active capture, especially in push-to-talk mode where the continuous restart path is not sufficient. Users with an invalid or unsupported realtime key can lose speech input instead of degrading to Apple Speech.
  - confidence: 0.84
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: The ElevenLabs-first start path skips `SFSpeechRecognizer` authorization, but the retry-limit fallback flips back to Apple Speech and restarts recognition without requesting speech permission. First-time users can therefore fail both the realtime path and the fallback path.
  - confidence: 0.82
