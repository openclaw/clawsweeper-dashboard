## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Send the documented manual commit frame:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:71`
  - body: The realtime API publishes manual commits as an input_audio_chunk frame with commit=true, but this sends a separate message_type=commit frame. PTT release can therefore fail to produce a committed_transcript and fall back to an empty or stale partial.
  - confidence: 0.86
- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The realtime PTT path sends commit(), sleeps for a fixed 300 ms, then clears isListening and stops recognition. If the committed transcript arrives later, handleTranscript ignores it or the receive loop has been canceled, so PTT can submit empty or stale text.
  - confidence: 0.9
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: After repeated realtime STT failures, this flips useElevenLabsSTT to false and restarts recognition, but startup skipped requestSpeechPermission while ElevenLabs was active. First-time users can fall back into Apple Speech without authorization and lose speech input instead of recovering.
  - confidence: 0.86
- **[P1] Recover active PTT captures after realtime STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime error handler always calls restartRecognitionAfterError(), which returns unless captureMode is continuous. In push-to-talk, a websocket or auth failure leaves the active capture without fallback, so the attempt can end with no transcript.
  - confidence: 0.82
- **[P2] Defer connected state until the realtime session is accepted:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:48`
  - body: isConnected becomes true immediately after task.resume(), before the WebSocket handshake or first server session message succeeds. With an invalid key or rejected session, audio and commit frames pass the guard while the UI treats STT as live.
  - confidence: 0.82
- **[P2] Preserve the configured Apple Speech locale:** `apps/ios/Sources/Voice/TalkModeManager.swift:639-640`
  - body: Current main resolves Apple Speech through TalkSpeechLocale using local and gateway speech-locale settings. This fallback constructs a generic SFSpeechRecognizer, so after rebase non-default locale users would lose their configured speech language.
  - confidence: 0.78
