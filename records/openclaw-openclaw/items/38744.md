## Summary

Keep open. Current main does not implement the proposed iOS ElevenLabs STT path, and this PR still needs contributor proof plus source-confirmed fixes for PTT finalization and fallback recovery before it is mergeable.

## What This Changes

The PR modifies iOS Talk Mode to use ElevenLabs realtime WebSocket STT when an ElevenLabs API key is available, adds STT model/language settings, and adds Swift STT client/PCM encoder files.

## Best Possible Solution

Keep the PR open for maintainer direction, but require a proof-backed revision that follows the ElevenLabs realtime event contract, waits for committed transcripts deterministically, handles Apple Speech fallback permissions, and either aligns with the gateway realtime transcription provider surface or gets explicit iOS owner approval for direct client STT.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Send the documented manual commit frame:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:65-66`
  - body: PTT relies on this method to finalize the segment, but it sends a standalone `{"message_type":"commit"}` frame. The current OpenClaw provider and ElevenLabs realtime event contract use an `input_audio_chunk` frame with `commit: true` (typically empty audio plus `sample_rate`) for manual commit, so this can fail to produce `committed_transcript` and leave PTT using a stale partial.
  - confidence: 0.9
- **[P1] Wait for the committed transcript before stopping PTT:** `apps/ios/Sources/Voice/TalkModeManager.swift:394-395`
  - body: The fixed 300 ms sleep after `client.commit()` is still a race. If ElevenLabs returns `committed_transcript` after that window, `endPushToTalk()` clears `isListening` and disconnects the websocket, so `handleTranscript` ignores the final text and the request can be sent with an incomplete partial transcript.
  - confidence: 0.87
- **[P2] Gate audio streaming on session_started:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `connect()` marks the client connected immediately after `task.resume()`, before ElevenLabs has accepted the websocket and sent `session_started`. The audio tap can start sending chunks during a failed or not-yet-ready session, which makes auth/network failures look connected and can drop the first utterance instead of queuing until the server confirms readiness.
  - confidence: 0.84
- **[P2] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:637-639`
  - body: After realtime STT failures trip the retry limit, this flips to Apple Speech and immediately calls `restartRecognitionAfterError()`. Because start/PTT skipped `requestSpeechPermission()` while ElevenLabs was active, users with an invalid or unsupported ElevenLabs key can fail to recover unless they had already granted Apple Speech permission.
  - confidence: 0.82
