## Summary

Keep open: this is a plausible feature PR, but it is not cleanup-closeable because current main does not implement native iOS ElevenLabs STT and the branch still has source-visible reliability bugs, an unresolved privacy/architecture decision, conflicts, and no real-device proof.

## What This Changes

This branch adds native iOS ElevenLabs realtime and batch STT clients, Settings model/language selectors, and TalkModeManager wiring that switches from Apple Speech when an ElevenLabs API key is configured.

## Best Possible Solution

Land a maintainer-approved iOS Talk design that either reuses the Gateway-owned transcription/realtime relay or adds an explicit-opt-in native STT path with client-safe auth, robust PTT commit handling, fallback recovery, docs, tests, and real-device proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff plus the official ElevenLabs contract is enough to reproduce the review blockers: wrong commit frame shape, fixed 300 ms PTT teardown, Apple fallback without Speech permission, and PTT errors routed through a continuous-only restart helper.

## Solution Assessment

No. The branch bypasses the established Gateway-owned provider/relay boundary and should be redesigned or explicitly approved before the code-level defects are worth repairing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Send ElevenLabs commits as input audio chunks:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:65-66`
  - body: The realtime API commits manual transcription by sending an `input_audio_chunk` with `commit: true`; this method sends a separate `message_type: "commit"` frame, so PTT release can wait for a committed transcript the provider never emits.
  - confidence: 0.9
- **[P1] Wait for committed PTT transcripts before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The ElevenLabs PTT stop path sends commit, sleeps a fixed 300 ms, then clears listening state and stops recognition. If the committed transcript arrives later, `handleTranscript` ignores it and PTT can submit an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Request Speech permission before Apple fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: When realtime STT reaches the retry limit, this path flips to Apple Speech and restarts recognition without requesting Speech permission. Users who started on ElevenLabs may never have granted that permission, so fallback can fail instead of recovering input.
  - confidence: 0.86
- **[P1] Recover push-to-talk after realtime STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime error handler always calls `restartRecognitionAfterError()`, but that helper returns unless capture mode is continuous. In push-to-talk mode, socket or auth failures can leave the active capture without a recovered transcript or immediate Apple fallback.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready after server confirmation:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:48`
  - body: `isConnected` becomes true immediately after `task.resume()`, before the WebSocket upgrade or first ElevenLabs server message succeeds. Callers can send audio and commits to a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84
