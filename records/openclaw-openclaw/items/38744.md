  - body: Setting `useElevenLabsSTT` from API-key presence silently moves existing iOS Talk users from Apple Speech to a new native provider path.   - body: A fixed 300 ms sleep after `commit()` does not guarantee ElevenLabs has returned `committed_transcript`. If the final arrives later, this sets `isListening = false` and stops recognition, so PTT can return an empty or stale partial transcript.
  - confidence: 0.88
- **[P1] Recover active captures after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:615-620`
  - body: After websocket/auth failures, this only flips the future path and calls `restartRecognitionAfterError()`, which returns outside continuous mode and does not recover the current PTT capture. Users can lose the active utterance instead of falling back cleanly to Apple Speech with the right permission flow.
  - confidence: 0.84
- **[P2] Gate audio sends on a real websocket ready state:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `isConnected` becomes true immediately after `resume()`, before the websocket handshake or ElevenLabs `session_started` message. Audio and commits can then be sent into a connection that later fails auth or readiness, making speech input look accepted while it is dropped.
  - confidence: 0.82
