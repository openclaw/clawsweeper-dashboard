## Summary

Keep this PR open: current main does not provide the requested native iOS ElevenLabs STT path, but the branch still has correctness, privacy/product, and proof blockers that need maintainer review before merge.

## What This Changes

This PR adds native iOS ElevenLabs realtime and batch STT clients, streams mic PCM to ElevenLabs, wires TalkModeManager and Settings STT selectors, and adds Swift source-list entries.

## Best Possible Solution

Land a maintainer-approved Talk architecture: either a gateway-owned realtime/transcription relay for iOS or an explicit-opt-in native STT path with robust readiness, PTT commit waiting, fallback recovery, docs, tests, and redacted real-device proof.

## Reproduction Assessment

Yes for the PR review blockers from source inspection: PR head shows the fixed 300 ms PTT wait, optimistic ready flag, fallback without Speech permission, and PTT error recovery routed through a continuous-only restart helper. No live iOS runtime proof is attached.

## Solution Assessment

No. The automatic direct native vendor WebSocket path is not the best merge shape until maintainers approve the privacy and architecture boundary; current Gateway relay/provider contracts point to a server-owned or explicit-opt-in design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wait for committed PTT transcripts before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The ElevenLabs PTT stop path sends commit and then waits a fixed 300 ms before clearing listening state and stopping recognition. If committed_transcript arrives later, handleTranscript ignores it and PTT can submit an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Request Speech permission before Apple fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: When realtime STT reaches the retry limit, this path flips useElevenLabsSTT to false and restarts recognition directly. Users who started on ElevenLabs may never have granted Apple Speech permission, so fallback can fail instead of recovering input.
  - confidence: 0.86
- **[P1] Recover push-to-talk after realtime STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime error handler always calls restartRecognitionAfterError, but that helper returns unless captureMode is continuous. In push-to-talk mode, socket or auth failures can leave the active capture without recovered transcript or immediate Apple fallback.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready after server confirmation:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:48`
  - body: isConnected becomes true immediately after task.resume(), before the WebSocket upgrade or first ElevenLabs server message succeeds. Callers can send audio and commits to a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84
