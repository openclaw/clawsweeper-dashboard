## Summary

Keep open. Current main still uses Apple Speech for iOS Talk and does not include the PR's native ElevenLabs STT files or settings, so the branch is not obsolete; however, the proposed native direct-to-ElevenLabs path is not mergeable as-is because prior review comments and static diff inspection show committed-transcript loss, weak fallback recovery, premature WebSocket readiness, and a product/security decision around live microphone data leaving the device.

## What This Changes

This PR adds native iOS ElevenLabs realtime and batch STT clients, PCM/WAV conversion, TalkModeManager integration, STT model/language settings, and Swift source-list entries.

## Best Possible Solution

Decide the Talk architecture first. If iOS should use provider-generic realtime Talk, route iOS through the Gateway realtime session/relay so provider credentials and vendor sockets stay gateway-side; if a native iOS ElevenLabs STT path is approved, add explicit opt-in, robust WebSocket readiness, committed-transcript waiting, Apple Speech fallback recovery, docs, changelog, and focused iOS tests before merge.

## Reproduction Assessment

Yes for the static and current-main checks: current main lacks native iOS ElevenLabs STT, and the PR diff shows the fixed PTT commit wait, premature connected flag, and fallback recovery paths. Live audio behavior still needs real-device validation before merge.

## Solution Assessment

No. The proposed implementation is a useful prototype, but direct native vendor STT auto-enabled from a Talk API key is not the narrowest maintainable solution until maintainers choose it over the existing gateway/provider realtime relay architecture and harden the failure and privacy paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Wait for committed PTT transcripts before stopping:** `apps/ios/Sources/Voice/TalkModeManager.swift:392-395`
  - body: The ElevenLabs PTT stop path sends `commit()` and then waits a fixed 300 ms before setting `isListening = false` and disconnecting. If `committed_transcript` arrives after that window, the receive task is gone or `handleTranscript` ignores it, so PTT can return an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Recover Apple fallback after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:623-627`
  - body: After realtime STT failures, the handler flips or retries recognition through paths that do not reliably request Apple Speech permission or recover an active push-to-talk capture. Users who started on ElevenLabs can remain without speech input instead of degrading to the existing Apple path.
  - confidence: 0.86
- **[P2] Mark the WebSocket ready only after the server handshake:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `isConnected` is set immediately after `task.resume()`, before the WebSocket upgrade or ElevenLabs `session_started` message succeeds. With an invalid key or unavailable service, callers can enqueue audio and commits onto a socket that has not become usable, delaying failure and losing speech input.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `apps/ios/Sources/Settings/SettingsTab.swift:339`
  - body: This adds a user-facing iOS feature and settings surface, but the PR file list does not include `CHANGELOG.md`. Repo policy requires user-facing `feat` changes to have a changelog entry so the release notes cover the new behavior.
  - confidence: 0.82
