## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wait for committed PTT transcripts before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:396-398`
  - body: The realtime PTT stop path sends commit and then waits a fixed 300 ms before setting listening false and stopping recognition. A delayed committed_transcript can arrive after teardown and be ignored, leaving PTT with an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:615-620`
  - body: After repeated realtime STT failures this path flips to Apple Speech and calls restartRecognitionAfterError, but users who started on ElevenLabs never granted SFSpeechRecognizer permission because start skipped that request. The fallback can therefore fail instead of restoring speech input.
  - confidence: 0.86
- **[P1] Recover push-to-talk after realtime STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:609-620`
  - body: The realtime error handler always calls restartRecognitionAfterError, whose guard only supports continuous capture. In push-to-talk, a websocket or auth failure can end the active capture without a recovered transcript or fallback result.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready only after session start:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: isConnected is set immediately after task.resume, before the websocket upgrade or ElevenLabs session_started message succeeds. Callers can send audio and commits into a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84
