## Summary

Keep open for maintainer review: the native iOS ElevenLabs STT feature is not implemented on current main and is not superseded, but the proposed patch has source-visible reliability blockers, a sensitive microphone-streaming data-flow decision, and no inspectable real-device proof.

## What This Changes

The PR adds native iOS ElevenLabs realtime and batch STT clients, TalkModeManager integration, STT settings controls, and Swift source-list entries.

## Best Possible Solution

Approve the Talk architecture first, then land either a gateway-owned realtime Talk integration or an explicit-opt-in native STT path with robust readiness, fallback, docs, changelog, tests, and real-device proof.

## Reproduction Assessment

Yes for the PR review findings from source inspection: the PR commit shows the fixed 300 ms PTT wait, optimistic WebSocket readiness, fallback error path, and automatic STT enablement. Live iOS device behavior is not reproduced because the PR provides no inspectable runtime proof.

## Solution Assessment

No. The direct native vendor WebSocket prototype is not the best merge shape without a maintainer decision, because current main already has gateway-owned realtime provider and relay boundaries that preserve provider-secret ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wait for committed PTT transcripts before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The ElevenLabs PTT stop path sends commit and then waits a fixed 300 ms before clearing listening state and stopping recognition. If the committed transcript arrives after that window, handleTranscript ignores it and PTT can submit an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Request speech permission before Apple fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: When realtime STT reaches the retry limit, this path flips useElevenLabsSTT to false and restarts recognition directly. Users who started on ElevenLabs may never have granted Apple Speech permission, so fallback can fail instead of recovering speech input.
  - confidence: 0.86
- **[P1] Recover active push-to-talk after STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime error handler always calls restartRecognitionAfterError, but that helper only restarts continuous captures. In push-to-talk mode, socket or auth failures can leave the active capture without a recovered transcript or immediate Apple fallback.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready after the server handshake:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: isConnected becomes true immediately after task.resume(), before the WebSocket upgrade or ElevenLabs session_started event succeeds. Callers can send audio and commits to a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The patch adds no dependency or CI supply-chain surface, but it introduces automatic native iOS microphone streaming to ElevenLabs from an existing Talk API key.

Concerns:

- **[medium] Require opt-in before streaming mic audio to ElevenLabs:** `apps/ios/Sources/Voice/TalkModeManager.swift:2130`
