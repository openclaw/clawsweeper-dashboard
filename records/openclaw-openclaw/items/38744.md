## Summary

Keep open: current main does not include the native iOS ElevenLabs STT path or settings picker, and the PR still has blocking reliability and privacy findings around PTT commit timing, WebSocket readiness, fallback recovery, and automatic vendor microphone streaming.

## What This Changes

This PR adds native iOS ElevenLabs realtime and batch STT clients, PCM/WAV conversion, TalkModeManager routing, STT model/language settings, and Swift source-list entries.

## Best Possible Solution

Choose the iOS Talk architecture first: prefer the Gateway/provider realtime relay, or explicitly approve native iOS ElevenLabs STT with user opt-in, robust readiness/fallback handling, docs, changelog, and focused iOS coverage.

## Reproduction Assessment

Yes for the PR findings from source inspection: the diff shows the 300 ms PTT commit window, premature `isConnected` flip, fallback paths, and automatic STT enablement. Live iOS audio behavior still needs real-device validation.

## Solution Assessment

No. The proposed direct native vendor STT path is useful prototype work, but it is not the narrowest maintainable solution until maintainers choose it over the existing Gateway realtime relay and harden the privacy and failure paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wait for committed PTT transcripts before stopping:** `apps/ios/Sources/Voice/TalkModeManager.swift:392-395`
  - body: The ElevenLabs PTT stop path sends `commit()` and then waits a fixed 300 ms before setting `isListening = false` and tearing recognition down. If `committed_transcript` arrives after that window, the receive path can be gone or `handleTranscript` will ignore it, so PTT can return an empty or truncated partial.
  - confidence: 0.9
- **[P1] Recover Apple Speech fallback after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:623-627`
  - body: After realtime STT errors, the handler flips toward Apple Speech only after retries but restarts through the continuous-only error path and does not request Apple Speech permission for users who started on ElevenLabs. Invalid keys or realtime access failures can leave speech input broken instead of degrading to the existing Apple path.
  - confidence: 0.86
- **[P2] Mark the WebSocket ready only after the server handshake:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `isConnected` is set immediately after `task.resume()`, before the WebSocket upgrade or server message succeeds. With an invalid key or unavailable service, callers can enqueue audio and commits onto a socket that has not become usable, delaying failure and losing speech input.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `apps/ios/Sources/Settings/SettingsTab.swift:339`
  - body: This is a user-facing iOS feature with a new settings surface, but the PR file list does not include `CHANGELOG.md`. Repo policy requires user-facing feature changes to be represented in release notes.
  - confidence: 0.82
