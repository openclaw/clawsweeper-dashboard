Overall confidence: 0.91

Full review comments:

- **[P1] Route iOS STT through the Gateway Talk contract:** `apps/ios/Sources/Voice/TalkModeManager.swift:588-600`
  - body: This adds a second ElevenLabs transcription stack inside the iOS app and auto-selects it when an ElevenLabs key exists, bypassing the current `talk.session.create`/`talk.session.appendAudio` provider contract. That creates conflicting credential, model, readiness, fallback, and session-event behavior for the same Talk feature.
  - confidence: 0.91
- **[P1] Preserve the configured Apple Speech locale:** `apps/ios/Sources/Voice/TalkModeManager.swift:639`
  - body: The Apple Speech fallback now creates `SFSpeechRecognizer()` directly, dropping the current `TalkSpeechLocale` local/Gateway selection. Users who configured a non-default speech locale would silently fall back to the device default after this merge.
  - confidence: 0.88
- **[P1] Wait for the committed PTT transcript before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:396-402`
  - body: After `commit()`, the code sleeps for a fixed 300 ms and then clears `isListening` and stops recognition. If `committed_transcript` arrives later, `handleTranscript` ignores it or the receive loop is already canceled, so PTT can submit an empty or stale partial transcript.
  - confidence: 0.92
- **[P1] Recover active captures after realtime STT failures:** `apps/ios/Sources/Voice/TalkModeManager.swift:613-620`
  - body: The realtime error handler only calls the continuous restart path; in push-to-talk, `restartRecognitionAfterError()` returns immediately, so an auth or WebSocket failure can leave the active capture with no fallback transcript.
  - confidence: 0.86
- **[P1] Request Apple Speech permission before fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: The ElevenLabs-first paths skip `SFSpeechRecognizer` authorization, but the retry-limit fallback flips back to Apple Speech and restarts recognition without requesting that permission. First-time users can therefore fail both realtime STT and the fallback path.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready only after provider readiness:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: `isConnected` is set immediately after `task.resume()`, before the WebSocket handshake or ElevenLabs `session_started` message. Audio and commit sends can run against a socket that is still connecting or already rejected, dropping speech frames until the receive loop reports an error.
  - confidence: 0.82
