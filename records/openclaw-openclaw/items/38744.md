## Summary

Keep open: current main has related gateway/provider realtime infrastructure, but it does not implement this native iOS ElevenLabs STT path, and the PR still has correctness, proof, and privacy architecture blockers.

## What This Changes

The PR adds native iOS ElevenLabs realtime WebSocket STT, batch STT/WAV helpers, TalkModeManager integration, STT settings controls, and Swift source-list entries.

## Best Possible Solution

Approve the iOS Talk realtime architecture first, then land either a gateway-owned realtime session path or an explicit-opt-in native STT path with robust readiness, fallback, docs, tests, and real-device proof.

## Reproduction Assessment

Yes for the PR review findings: the c80087ffaf1cd7e2d6cc9f551fdca8077613f077 patch shows the fixed 300 ms PTT wait, optimistic WebSocket readiness, fallback error path, and automatic STT enablement. Live iOS behavior is not reproduced because the PR provides no inspectable runtime proof.

## Solution Assessment

No. The direct native vendor WebSocket path is not the best merge shape until maintainers decide the privacy and credential boundary, especially because current main already has gateway-owned realtime provider and relay contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wait for committed PTT transcripts before teardown:** `apps/ios/Sources/Voice/TalkModeManager.swift:397-398`
  - body: The ElevenLabs PTT stop path sends commit and waits a fixed 300 ms before clearing listening state and stopping recognition. If committed_transcript arrives after that window, handleTranscript ignores it and PTT can submit an empty or truncated partial transcript.
  - confidence: 0.9
- **[P1] Request speech permission before Apple fallback:** `apps/ios/Sources/Voice/TalkModeManager.swift:618-620`
  - body: When realtime STT reaches the retry limit, this path flips useElevenLabsSTT to false and restarts recognition directly. Users who started on ElevenLabs may never have granted Apple Speech permission, so fallback can fail instead of recovering speech input.
  - confidence: 0.86
- **[P1] Recover active push-to-talk after STT errors:** `apps/ios/Sources/Voice/TalkModeManager.swift:620`
  - body: The realtime error handler always calls restartRecognitionAfterError, but that helper returns unless captureMode is continuous. In push-to-talk mode, socket or auth failures can leave the active capture without a recovered transcript or immediate Apple fallback.
  - confidence: 0.84
- **[P2] Mark the WebSocket ready after session start:** `apps/ios/Sources/Voice/ElevenLabsRealtimeSTTClient.swift:47-48`
  - body: isConnected becomes true immediately after task.resume(), before the WebSocket upgrade or ElevenLabs session_started event succeeds. Callers can send audio and commits to a socket that never became usable, delaying failure and losing speech input.
  - confidence: 0.84
