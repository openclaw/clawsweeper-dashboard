## Summary

Keep open; current main still lacks the requested Signal quote behavior, and this PR still drops quote metadata for authorNumber-only visibility and Signal group reply paths while missing required release/API bookkeeping.

## What This Changes

The branch adds Signal inbound ReplyToId context, reaction heartbeat wakeups, outbound quote RPC parameters, first-chunk quote handling, and a heartbeat SDK export.

## Best Possible Solution

Finish the Signal plugin implementation by preserving raw quote author metadata through inbound context and outbound reply delivery for group sends, include authorNumber in quote visibility, and add focused tests plus changelog/API baseline updates.

## Reproduction Assessment

Yes. A static reproduction is a Signal group quote payload with quote.id, quote.text, and authorNumber but no author/authorUuid under allowlist visibility; the resolver treats the sender as missing and drops ReplyToId. A second reproduction is a Signal group reply with replyToId only, where the helper cannot infer quote-author and send.ts omits quote metadata.

## Solution Assessment

No. The feature belongs in the Signal plugin, but this patch is not mergeable until it carries quote-author metadata for groups, honors authorNumber in visibility decisions, and completes the required changelog/API bookkeeping.
