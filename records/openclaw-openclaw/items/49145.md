## Summary

Keep this PR open. Current main still lacks the requested Signal reply/quote behavior, but the current PR head still has merge-blocking quote metadata gaps for authorNumber-only inbound quotes and Signal group outbound quoted replies, plus a missing user-facing changelog entry.

## What This Changes

The PR adds Signal inbound ReplyToId quote extraction, reaction heartbeat wakeups, outbound quote RPC parameters, first-chunk-only quoted sends, and one status JSON test expectation update.

## Best Possible Solution

Finish the Signal plugin by preserving verified quote timestamp and author metadata across inbound context, reply payload routing, queued retry, and all chunked Signal send paths, with generic core changes limited to typed payload plumbing.

## Reproduction Assessment

Yes. A static reproduction is a Signal group quote payload with quote.id, quote.text, and authorNumber but no author/authorUuid under allowlist visibility; the resolver ignores authorNumber, so ReplyToId can be dropped. A second path is a Signal group reply with replyToId only, where quoteAuthor cannot be inferred from group:* and no Signal quote metadata is sent.

## Solution Assessment

No. The feature direction belongs in the Signal plugin, but this PR is not mergeable until quote author metadata is carried through the group paths, authorNumber participates in visibility decisions, focused tests are added, and the user-facing change is documented.
