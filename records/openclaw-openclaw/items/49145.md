## Summary

This PR should stay open for maintainer follow-up rather than cleanup-close: current main still lacks several requested Signal reply behaviors, but the branch is not mergeable as-is because group quoted replies can drop quote-author metadata and no live Signal proof is present.

## What This Changes

The PR adds Signal inbound quote IDs, reaction-triggered heartbeat wakeups, outbound quote parameters, first-chunk reply handling, and abort-signal threading across Signal send paths.

## Best Possible Solution

Adopt the useful Signal reply/reaction pieces through a maintained branch that propagates gated quote-author metadata through all reply/send paths, adds focused regression coverage, and posts redacted live Signal proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: Signal group replies can reach sendMessageSignal with replyToId but no quoteAuthor, while the quote helper refuses to infer authors for group targets. I did not run a live Signal setup.

## Solution Assessment

No. The feature direction is useful, but the current patch should carry visibility-gated raw quote-author metadata through the Signal reply payload and send paths, or explicitly scope outbound quoted replies to direct messages before merge.
