## Summary

Keep this PR open. Current main does not implement the requested Signal reply context or quoted-reply behavior, but the latest PR head still has two P2 correctness gaps around Signal group quote author handling, so it is not ready to merge as-is.

## What This Changes

The PR adds Signal inbound ReplyToId quote context, reaction-triggered heartbeat wakeups, outbound quote-timestamp/quote-author support, first-chunk-only quoting across Signal send paths, a focused heartbeat SDK export, and one status JSON test expectation update.

## Best Possible Solution

Update the branch so Signal quote sender resolution accepts authorNumber, then carry verified quote author metadata through inbound context, reply payload routing, queued/retried deliveries, sendPayload, formatted text/media, monitor delivery, and sendMessageSignal while preserving quote visibility policy and first-chunk-only behavior.

## Reproduction Assessment

Yes. A static reproduction is a Signal group quote payload with id, text, and authorNumber but no author/authorUuid in an allowlist visibility mode; the PR accepts authorNumber in the type but the resolver ignores it, so ReplyToId and quoted context can be dropped. A second path is a Signal group reply carrying only replyToId, where the PR cannot infer quote-author and therefore sends no Signal quote metadata.

## Solution Assessment

No. The feature direction is useful and belongs in the Signal plugin, but this PR is not the best mergeable fix until it preserves quote author metadata for group replies and includes authorNumber in quote visibility decisions.
