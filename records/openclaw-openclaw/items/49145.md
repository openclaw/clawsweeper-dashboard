## Summary

Keep this PR open for maintainer follow-up: the Signal reply/reaction direction is useful and not implemented on current main, but the current patch still has source-level quoted-reply blockers and lacks real Signal behavior proof before merge.

## What This Changes

The PR adds Signal quote IDs to inbound context, wakes sessions on reactions, adds Signal quote send parameters, and applies first-chunk-only reply handling across Signal outbound paths.

## Best Possible Solution

Adopt the useful Signal reply/reaction pieces through a maintained branch that propagates visibility-gated quote-author metadata through every reply/send path, adds focused regression coverage, and posts redacted live Signal proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: Signal group replies can reach sendMessageSignal with replyToId but no quoteAuthor, and authorNumber-only quote senders are not used by the visibility resolver. I did not run a live Signal setup.

## Solution Assessment

No. The feature direction is reasonable, but this patch should either thread quote-author metadata through the Signal reply payload and all send paths, or explicitly scope native quoted replies to direct messages before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Pass quote authors with Signal reply ids:** `extensions/signal/src/monitor.ts:359`
  - body: `resolveSignalQuoteParams` cannot infer an author for `group:*`, and signal-cli requires quote authors when a quote timestamp is sent. This monitor path forwards only `replyToId`, so Signal group replies lose native quote metadata even when inbound quote author data was available; thread the gated quote author through the reply payload and send options, or scope native quoted replies to DMs.
  - confidence: 0.87

  - body: The PR adds `quote.authorNumber` to the Signal quote shape, but `resolveSignalQuoteContext` still resolves quote senders only from `author` and `authorUuid`. In group allowlist modes, authorNumber-only quotes can be treated as sender-missing, which drops `ReplyToId` and quote context even though Signal supplied an author number.
  - confidence: 0.78
