## Summary

Keep open: the Signal feature is not already on main and is plausible, but the patch still drops quote-author context for group quoted replies and lacks required real Signal proof, so it should not be closed or merged yet.

## What This Changes

The branch adds Signal quote/reply metadata propagation, reaction heartbeat wakeups, outbound quote parameters, first-chunk-only quote handling, a heartbeat SDK export, and abort-signal threading.

## Best Possible Solution

Carry gated raw quote-author metadata through the Signal reply payload and every Signal send path that forwards replyToId, add focused regression coverage, then provide redacted live Signal proof before merge.

## Reproduction Assessment

Yes for the PR defect by source inspection: a Signal group reply with payload.replyToId reaches sendMessageSignal without quoteAuthor, while the helper refuses to infer authors for group targets, so no quote params are emitted. I did not run a live Signal setup.

## Solution Assessment

No. The direction is maintainable, but the current patch should either propagate a visibility-gated quote author for group replies or deliberately limit/document outbound quote support to direct messages.

## Review Findings
