## Summary

Keep open. Current main still lacks native Signal outbound quote parameters and immediate reaction wake, while this PR has source-level quote blockers and no real Signal proof. The quote-reply portion overlaps active work in https://github.com/openclaw/openclaw/pull/36630 and https://github.com/openclaw/openclaw/pull/56791, but the reaction-wake behavior is not clearly superseded.

## What This Changes

The PR adds Signal inbound ReplyToId quote context, reaction-triggered heartbeat wake, outbound signal-cli quote parameters, first-chunk quote behavior, and abortSignal passthrough.

## Best Possible Solution

Continue with a maintained Signal quote-reply implementation in the canonical active PRs, carry over reaction wake only if the Signal owner wants that behavior, and require focused tests plus redacted live Signal proof before merge.

## Reproduction Assessment

Yes for the PR defects by source inspection: group replies cannot infer quoteAuthor from replyToId alone, and authorNumber-only quotes are not considered by resolveSignalQuoteContext. I did not run a live Signal account flow in this read-only review.

## Solution Assessment

No. The feature direction is useful, but this branch should not merge as-is because it drops group quote authors, ignores authorNumber in visibility checks, and lacks real Signal proof; the cleaner path is to land the maintained quote-reply PRs and decide reaction wake separately.
