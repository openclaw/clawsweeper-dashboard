## Summary

Keep open: this is a plausible small fix, but it is not merge-ready because it exposes `sessions_spawn` in local embedded sessions without proving an executable spawn path and it checks only defaults instead of the effective per-agent/default allowlist.

## What This Changes

The branch changes `createOpenClawTools` so embedded tool construction includes `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty, without changing spawn execution or adding tests.

## Best Possible Solution

Gate embedded `sessions_spawn` on the same effective allowlist used by `sessions_spawn` and `agents_list`, then either keep local embedded mode hidden or add a proven local execution bridge.

## Reproduction Assessment

Yes, source-reproducible: local TUI enters embedded mode, current main hides `sessions_spawn` unless `allowGatewaySubagentBinding` is true, and the PR changes that gate based on defaults. I did not establish a live TUI spawn run.

## Solution Assessment

No. `createOpenClawTools` is the right materialization layer, but the patch is too narrow and should share effective allowlist policy while resolving or explicitly excluding local embedded execution.
