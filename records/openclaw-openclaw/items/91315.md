## Summary

Keep this PR open for review, but it is not merge-ready: the patch exposes a model-visible `sessions_spawn` tool in local embedded TUI without proving that path can execute, and it only checks the defaults allowlist while runtime policy also supports per-agent allowlists.

## What This Changes

The PR changes `src/agents/openclaw-tools.ts` so embedded mode includes `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Make tool exposure derive from the same effective per-agent/default allowlist used by spawn execution, and expose it only in TUI paths that can execute it, with focused regression coverage and redacted live tool-inventory or spawn proof.

## Reproduction Assessment

Yes, from source: embedded mode currently hides `sessions_spawn` unless `allowGatewaySubagentBinding` is true, while this PR would expose it for defaults-only configs and still miss per-agent allowlists. I did not run a live TUI because this review was read-only.

## Solution Assessment

No. `createOpenClawTools` is the right layer for inventory, but the condition must mirror the effective allowlist and should not expose a local embedded tool unless the execution path is supported and proven.
