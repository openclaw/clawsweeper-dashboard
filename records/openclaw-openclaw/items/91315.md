## Summary

Keep open: current main still hides `sessions_spawn` on the reported embedded path, so the PR is not obsolete, but this branch is not merge-ready because it exposes an unproven local embedded execution path and only checks the defaults allowlist instead of the effective requester allowlist. The parallel open PR at https://github.com/openclaw/openclaw/pull/91446 has the same defaults-only issue and weaker proof, so it is not a safe superseding target.

## What This Changes

The PR changes `createOpenClawTools` so embedded mode includes `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Derive embedded `sessions_spawn` exposure from the same effective requester allowlist used by execution, then either keep local embedded spawn hidden or implement and prove local `agent` dispatch before merge.

## Reproduction Assessment

Yes, source-reproducible: current main sets local TUI into embedded mode and hides `sessions_spawn` unless `allowGatewaySubagentBinding` is true. I did not run a live Windows/npm or TUI spawn reproduction in this read-only review.

## Solution Assessment

No. `createOpenClawTools` is the right materialization layer, but the patch should share the effective per-agent/default target policy and resolve local embedded execution before making the tool visible there.
