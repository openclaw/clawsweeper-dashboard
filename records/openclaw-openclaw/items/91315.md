## Summary

Keep open: the PR targets a real current-main bug, but it is not merge-ready because the embedded visibility gate does not mirror effective allowlist policy and can advertise `sessions_spawn` in local TUI without a proven execution path.

## What This Changes

The PR changes `createOpenClawTools` to include `sessions_spawn` in embedded mode when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Materialize `sessions_spawn` only when the effective requester allowlist permits it and the embedded runtime can actually execute it, with tests and redacted TUI proof.

## Reproduction Assessment

Yes by source inspection for the missing-tool bug: local TUI sets embedded mode and current main excludes `sessions_spawn` unless `allowGatewaySubagentBinding` is true. I did not run a live Windows/npm TUI session.

## Solution Assessment

No. The materialization layer is plausible, but this patch is too broad for local embedded execution and too narrow for per-agent allowlists.
