## Summary

Keep open: current main still has the reported embedded-mode missing-tool gate, but this PR is not merge-ready because it exposes a Gateway-dependent spawn tool in local embedded mode and only checks the defaults allowlist path.

## What This Changes

The PR changes `createOpenClawTools` so embedded tool assembly includes `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Materialize `sessions_spawn` only when the effective requester allowlist permits it and the current embedded runtime can execute it, with local TUI proof or explicit Gateway-only gating.

## Reproduction Assessment

Yes for the local embedded path by source inspection: local TUI sets embedded mode and current main hides `sessions_spawn` unless `allowGatewaySubagentBinding` is true. I did not establish a live TUI spawn run or fully prove the non-local gateway TUI variant.

## Solution Assessment

No. `createOpenClawTools` is a plausible materialization layer, but the patch should mirror the effective per-agent/default allowlist and avoid exposing a Gateway-dependent tool where local embedded execution is unsupported or unproven.
