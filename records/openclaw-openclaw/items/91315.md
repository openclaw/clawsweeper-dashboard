## Summary

Keep open: the PR targets a real `sessions_spawn` inventory gap, but the patch is not merge-ready because it exposes the tool in local embedded mode without proving the execution path works, and it only checks defaults rather than the effective per-agent/default allowlist. The similar open PR at https://github.com/openclaw/openclaw/pull/91446 is not a safe superseding target because it has the same defaults-only shape and lacks sufficient proof.

## What This Changes

The branch changes `createOpenClawTools` so embedded sessions include `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Make `sessions_spawn` exposure share the effective requester allowlist and only expose it in embedded contexts where the gateway/subagent execution path is supported, with focused coverage and redacted live TUI or tool-inventory proof.

## Reproduction Assessment

Yes, from source: current `main` hides `sessions_spawn` in embedded mode unless `allowGatewaySubagentBinding` is true, and the PR would expose it for defaults-only config. The local embedded execution failure is also source-reproducible because native spawn dispatches Gateway `method: "agent"` while the embedded stub does not support it.

## Solution Assessment

No. `createOpenClawTools` is the right inventory layer, but this patch is not the best fix because it does not mirror the executor's effective allowlist or settle whether local embedded spawning is supported.
