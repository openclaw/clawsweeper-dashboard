## Summary

Keep open: the PR targets a real `sessions_spawn` exposure gap, but the patch is not merge-ready because it exposes a gateway-dependent spawn tool in local embedded TUI sessions and only checks default `allowAgents`, not the effective per-agent allowlist.

## What This Changes

The branch changes `src/agents/openclaw-tools.ts` so embedded tool construction includes `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Derive embedded exposure from the same effective requester allowlist used by execution, and only expose local embedded `sessions_spawn` if that runtime is implemented and proven; otherwise keep local mode gated and document the gateway requirement.

## Reproduction Assessment

Yes, from source inspection: current main gates `sessions_spawn` out of embedded mode unless `allowGatewaySubagentBinding` is true, and local TUI sets embedded mode. The PR defects are also source-reproducible because the new condition checks defaults only and local native spawn still goes through gateway dispatch.

## Solution Assessment

No. `createOpenClawTools` is the right layer for tool materialization, but the condition needs to mirror the effective allowlist and the local embedded execution contract must be decided and proven before merge.
