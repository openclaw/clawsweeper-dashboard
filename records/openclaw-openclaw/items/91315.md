## Summary

Keep open: the branch targets a real agents/TUI bug, but the current patch is not merge-ready because it exposes `sessions_spawn` for one defaults-only embedded case without proving local execution and without mirroring the documented per-agent allowlist path.

## What This Changes

The PR changes `createOpenClawTools` so embedded sessions include `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Make embedded `sessions_spawn` exposure derive from the same effective requester allowlist used by `sessions_spawn` and `agents_list`, and either keep local embedded mode gated off or implement and prove a working local spawn path.

## Reproduction Assessment

Yes, from source: local TUI enters embedded mode, the PR exposes `sessions_spawn` based on defaults, and the execution path still needs Gateway-backed `agent` dispatch proof. A per-agent-only allowlist also remains excluded by the new condition.

## Solution Assessment

No. `createOpenClawTools` is the right layer for tool materialization, but the patch is too narrow and should share the effective allowlist policy while resolving local embedded execution before merge.
