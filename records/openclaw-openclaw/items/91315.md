## Summary

Keep open; this is a useful narrow attempt, but it is not merge-ready because it exposes `sessions_spawn` from a defaults-only check while leaving the local embedded execution path and per-agent allowlist contract unresolved.

## What This Changes

The PR changes the agents tool factory so embedded mode includes `sessions_spawn` when `agents.defaults.subagents.allowAgents` is non-empty.

## Best Possible Solution

Make `sessions_spawn` exposure derive from the same effective per-agent/default allowlist used by spawn execution, and either implement/prove local embedded spawning or keep local embedded mode gated off.

## Reproduction Assessment

Yes from source: a per-agent `subagents.allowAgents` config still leaves the PR's new defaults-only gate false, while local embedded mode would see the tool without a supported gateway dispatch path.

## Solution Assessment

No; the current patch fixes one inventory case but is not the best fix because it does not share the executor's effective allowlist precedence or settle whether local embedded spawning is supported.
