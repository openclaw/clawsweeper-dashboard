## Summary

Keep this PR open: it is a focused fix for a real logging bug, but the proposed helper can still permanently cache the fallback value and the external PR lacks after-fix real macOS log proof.

## What This Changes

This PR replaces the module-level structured file-log hostname constant with a lazy cached hostname helper.

## Best Possible Solution

Update the helper so empty hostname lookups do not poison the cache, add focused regression coverage for a transient empty hostname, and provide redacted after-fix macOS log proof before merge.

## Reproduction Assessment

No high-confidence local macOS reproduction was established in this read-only review. The source path is clear, though: if `os.hostname()` returns an empty string before the cached value is set, current main writes `"unknown"` permanently.

## Solution Assessment

No. Resolving on first log write is close, but caching the fallback value preserves the same failure mode if the first lazy lookup is empty; cache only non-empty hostnames or resolve per record until a valid value appears.
