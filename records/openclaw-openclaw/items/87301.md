## Summary

The PR is a focused fix for a real logging bug, and the contributor added useful before/after gateway log proof, but the current patch can still poison hostname state when the same transient empty lookup occurs during the initializer or first file log.

## What This Changes

The PR adds a logging hostname initializer, imports it before logger entry points, and changes the JSONL hostname field from a module constant to a cached lazy helper.

## Best Possible Solution

Repair the PR so hostname recovery uses a non-poisoning resolver for both tslog `_meta.hostname` and OpenClaw JSONL `hostname`, with a regression test for empty-then-valid `os.hostname()`.

## Reproduction Assessment

No high-confidence local macOS reproduction was run in this read-only review. The current source clearly caches hostname at import time, and the linked report plus PR proof show real gateway logs with `_meta.hostname` stuck at `unknown`.

## Solution Assessment

No, not yet. The direction is right, but the patch still relies on a single early lookup for tslog and caches the `unknown` fallback for top-level JSONL fields, so it needs a non-poisoning retry/lazy path before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retry hostname before seeding tslog:** `src/logging/logger-hostname-init.ts:7`
  - body: The reported failure is that `os.hostname()` can be empty during ESM/module load, but this initializer also calls it once during the same static import path and leaves `HOSTNAME` unset when that transient empty value occurs. `tslog` builds static runtime meta at import time from the hostname env vars and then reuses it for every `_meta`, so this can still leave `_meta.hostname` as `unknown` for the process.
  - confidence: 0.86
- **[P2] Do not cache the unknown hostname fallback:** `src/logging/logger.ts:109`
  - body: This stores `unknown` in `cachedHostname` when the first file log hits the same transient empty hostname state. Because the cache is never retried, the top-level JSONL `hostname` field can remain `unknown` for the gateway lifetime; cache only non-empty hostnames and use `unknown` as a per-record fallback until one is observed.
  - confidence: 0.9
