## Summary

Keep open: this is a real logging bug with usable gateway log proof, but the patch still caches the fallback `unknown` value if the first file-log write sees an empty hostname, so the process-lifetime failure mode can remain.

## What This Changes

The PR adds early hostname environment initialization for tslog and replaces the logger’s module-level `HOSTNAME` constant with a cached `getHostname()` helper for structured file-log records.

## Best Possible Solution

Only cache non-empty hostnames, return `unknown` without caching while the OS value is empty, keep the tslog env seeding, and add focused coverage for empty-then-valid hostname recovery.

## Reproduction Assessment

Yes for source-level reproduction: current `main` caches a single fallback hostname and the PR still caches `unknown` if the first log write sees an empty value. I did not live-reproduce the intermittent macOS timing in this read-only review.

## Solution Assessment

No: the direction is right, but caching the fallback value is not the best fix. The helper should retry until it observes a non-empty hostname and should include regression coverage for the reported empty-then-valid sequence.
