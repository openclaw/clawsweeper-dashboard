## Summary

Keep open: current main still lacks a Feishu outbound pacing/minimum-interval configuration, and the Feishu reply, comment, outbound, media, and streaming-card send paths do not share a proactive per-account pacing helper. The related #13615 and #70879 reports remain useful context, but they track broader cross-provider limiting and retry/backoff rather than this Feishu-specific outbound pacing request.

## What This Changes

Add configurable Feishu channel message pacing, including a minimum interval between outbound messages and how chunked LLM responses are sent.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu message delivery paths while leaving cross-provider limits to #13615 and retry/backoff to #70879.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes. The narrow maintainable solution is a Feishu plugin pacing helper with top-level and per-account config, not a generic core rate limiter or a retry/backoff change.
