## Summary

Keep open: current main still lacks Feishu outbound pacing/minimum-interval configuration, and the reply, comment, message-tool, media, and streaming-card delivery paths do not share a proactive per-account pacing helper. Related #13615 and #70879 remain useful context, but they track broader rate limiting and retry/backoff rather than this Feishu-specific pacing request.

## What This Changes

Request to add configurable message pacing for Feishu outbound sends and optionally use that setting when segmenting LLM responses.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broader provider limits to #13615 and retry/backoff to #70879.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes. The narrow maintainable solution is a Feishu plugin pacing helper with top-level and per-account config, not a generic core limiter or retry/backoff-only change.
