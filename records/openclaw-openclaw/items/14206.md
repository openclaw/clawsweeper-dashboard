## Summary

Keep open: current main and the latest shipped tag still lack a Feishu outbound pacing config or shared minimum-interval helper. The broader rate-limit tracker and Feishu retry/backoff tracker are related, but they do not supersede this proactive Feishu-specific pacing request.

## What This Changes

The issue requests configurable Feishu message pacing, including a minimum interval between outbound messages and possible coordination with response chunking.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broad cross-provider limits and reactive retry/backoff to their separate trackers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement/config request. Source inspection gives high confidence that the requested Feishu pacing config and shared outbound pacing helper are absent on current main.

## Solution Assessment

Yes directionally: a Feishu-owned per-account pacing helper is narrower and more maintainable than a broad core limiter for this request. The exact config name, defaults, inheritance, and endpoint coverage need maintainer scoping first.
