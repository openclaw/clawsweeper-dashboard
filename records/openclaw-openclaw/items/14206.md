## Summary

Keep open: current main still lacks a Feishu outbound pacing or minimum-interval setting, and the broad rate-limit tracker plus Feishu retry/backoff tracker do not supersede this proactive Feishu-specific request.

## What This Changes

Request a Feishu channel setting that enforces a minimum interval between outbound messages and may influence response chunking.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broad provider limits and reactive retry/backoff to their own trackers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and still sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes, directionally. The maintainable solution is a Feishu plugin pacing helper with top-level and per-account config, but the exact public config name, default, and coverage need maintainer scoping first.
