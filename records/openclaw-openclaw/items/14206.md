## What This Changes

The issue requests a configurable Feishu channel minimum interval/rate limit for outbound messages, including better handling of chunked LLM responses.

## Best Possible Solution

Add a Feishu-owned outbound scheduler around message create/reply calls, configurable at top level and per account, with docs/schema/manifest coverage and tests proving chunked text, comments, media, cards, and streaming-card initial sends all share the intended pacing boundary.

## Solution Assessment

Yes, the best fix is a Feishu-plugin outbound scheduler rather than a broad core rate-limit layer for this issue. Keep LLM chunking and reactive retry as separate concerns, with pacing applied at the message API boundary.
