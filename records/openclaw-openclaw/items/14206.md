## Summary

Keep open. Current main still lacks a Feishu outbound minimum-interval or pacing setting, and chunked replies, comment replies, message-tool sends, media sends, and streaming-card creates can still make consecutive Feishu API calls without a shared per-account gate.

## What This Changes

Request adding configurable minimum-interval outbound pacing for Feishu messages, including bursty chunked LLM replies.

## Best Possible Solution

Add a Feishu-owned, default-off per-account outbound pacing layer resolved from top-level and account config, then wire it through every Feishu delivery path that can burst API calls.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: set a small Feishu textChunkLimit or produce a long/message-tool/media response that splits into multiple sends, then inspect the current loops to see consecutive Feishu API calls with no configured minimum interval.

## Solution Assessment

Yes. A Feishu plugin-owned pacing gate is the narrowest maintainable solution because the reported failure is Feishu outbound delivery frequency; a core-wide limiter is broader than this issue, while retry-only handling would not prevent the initial burst.
