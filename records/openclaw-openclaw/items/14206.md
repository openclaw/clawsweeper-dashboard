## Summary

Keep open: current main and the latest release still lack a Feishu outbound pacing or minimum-interval setting, and the related broad rate-limit, retry/backoff, and streaming-throttle items do not cover this proactive per-account outbound message pacing request.

## What This Changes

Request to add configurable Feishu message rate limiting, including a minimum interval between sent messages and possibly chunking behavior that avoids bursty Feishu API sends.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone delivery paths while leaving broad cross-provider limits and reactive retry/backoff in their separate trackers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement/config request. Source inspection of current main and v2026.5.12 gives high confidence that the requested Feishu pacing config and shared outbound pacing helper are absent.

## Solution Assessment

Yes directionally: Feishu-owned per-account pacing is narrower than a broad core limiter for this request. The exact public config contract and path coverage should be decided by maintainers before implementation.
