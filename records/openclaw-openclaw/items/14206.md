## Summary

Keep open: current main still lacks a Feishu outbound minimum-interval or pacing configuration, and the main reply, comment, outbound, media, and streaming-card send paths still call Feishu APIs without a shared pacing helper. The related #13615 and #70879 items are broader or retry-focused and do not supersede this narrow Feishu plugin enhancement.

## What This Changes

Add configurable message pacing for Feishu/Lark outbound messages, ideally reducing bursty chunked replies that can trip Feishu API limits.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, surface it in schema/manifest/docs, and apply it consistently to burst-prone Feishu message delivery paths while leaving broad cross-provider limits and retry/backoff to #13615 and #70879.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes. The narrow maintainable direction is a Feishu plugin pacing helper with top-level and per-account config, not a generic core rate limiter or retry/backoff change.
