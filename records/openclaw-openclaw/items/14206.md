## Summary

Keep open. Current main and v2026.6.8 still have reactive Feishu send retry handling, but they do not implement the requested proactive outbound pacing/minimum-interval configuration before bursty chunks, media, comment replies, outbound messages, or streaming-card sends reach Feishu.

## What This Changes

Request to add configurable Feishu outbound message pacing/minimum-interval behavior, ideally also influencing how responses are segmented into messages.

## Best Possible Solution

Add a Feishu-owned outbound scheduler around message create/reply style sends, configurable at top level and per account with safe defaults, and leave broader cross-provider rate limiting to the separate design tracker.

## Reproduction Assessment

Yes, at source level: current main has Feishu chunked/comment/outbound send paths and reactive retry handling, but no pre-send minimum interval or scheduler. I did not run a live Feishu burst test in this read-only review.

## Solution Assessment

Yes. A Feishu-plugin-local pacing layer is the best shape for this issue; the merged retry helper is useful adjacent work but only recovers after Feishu rejects a burst.
