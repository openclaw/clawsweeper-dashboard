## Summary

Keep open. Current main still lacks a Feishu outbound minimum-interval setting or shared per-account pacing gate, and the relevant reply, comment, message-tool, media, and streaming-card create paths can still issue consecutive Feishu API sends without configurable spacing.

## What This Changes

Request to add configurable message rate limiting for Feishu outbound delivery and, if possible, coordinate it with response chunking.

## Best Possible Solution

Add a Feishu-owned, default-off per-account outbound pacing layer resolved from top-level and account config, then wire it through every Feishu delivery path that can burst message API calls.

## Reproduction Assessment

Yes. A high-confidence static path exists: configure a small Feishu textChunkLimit or produce a long/message-tool/media response, then the current loops call Feishu send APIs consecutively without a configured minimum interval.

## Solution Assessment

Yes. A Feishu plugin-owned pacing gate is the narrowest maintainable direction because the reported failure is Feishu outbound delivery frequency; #13615 is broader cross-provider policy work, and #70879 is retry/backoff after rate-limit errors rather than proactive spacing.
