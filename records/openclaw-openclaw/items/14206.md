## Summary

Keep open. Current main and v2026.6.8 have reactive Feishu send retry handling, but they still do not implement the requested proactive outbound minimum-interval or pacing configuration across chunked replies, comment replies, media/outbound sends, or streaming-card initial sends.

## What This Changes

Add Feishu channel message rate limiting so users can configure a minimum interval between outbound messages, with pacing that also accounts for response chunking.

## Best Possible Solution

Add a Feishu-owned, per-account outbound pacing layer around message create/reply style sends, configurable at top level and per account with safe default-off behavior, while leaving broad cross-provider rate limiting to the separate design tracker.

## Reproduction Assessment

Yes, at source level: current main chunks Feishu replies and comment replies, then calls Feishu send/create/reply paths without a pre-send scheduler or minimum-interval config. I did not run a live Feishu burst test in this read-only review.

## Solution Assessment

Yes. A Feishu-plugin-local pacing layer is the narrowest maintainable solution for this issue; the merged retry helper is useful adjacent work but only recovers after Feishu rejects a burst.
