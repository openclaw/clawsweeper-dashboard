## Summary

Keep open: current main and the latest shipped release still lack a Feishu outbound minimum-interval setting or shared pacing gate, and the open broad rate-limit and Feishu retry issues track adjacent but different work.

## What This Changes

Add configurable Feishu outbound message pacing so bursty chunked LLM replies can wait between Feishu API sends.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broad core limits and reactive retries to their separate trackers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and shared outbound pacing helper.

## Solution Assessment

Yes directionally: a Feishu-owned per-account pacing helper is narrower than a core-wide limiter for this request. The exact public config name, default, inheritance semantics, and delivery-path coverage should be decided before implementation.
