## Summary

Keep open: current main still lacks a Feishu outbound minimum-interval or pacing configuration, and the main reply, comment, outbound, media, and streaming-card paths still send through Feishu APIs without a shared per-account pacing helper. The broader #13615 and retry-focused #70879 issues are related but do not supersede this narrower Feishu plugin request.

## What This Changes

Add configurable message pacing for the Feishu channel so bursty chunked responses do not trigger Feishu API flow-control bans.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu message delivery paths while leaving cross-provider limits to #13615 and retry/backoff to #70879.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes. The narrow maintainable path is a Feishu plugin pacing helper with top-level and per-account config, not a generic core rate limiter or retry/backoff change.
