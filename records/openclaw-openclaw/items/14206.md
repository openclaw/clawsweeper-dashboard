## Summary

Keep open: current main still lacks a Feishu outbound pacing or minimum-interval setting, and the related broad rate-limit and Feishu retry/backoff trackers do not cover this proactive Feishu-specific config request.

## What This Changes

Request to add configurable Feishu channel message rate limiting so outbound messages are paced and long responses can be segmented with that pacing in mind.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broad provider limits and reactive retry/backoff to their separate trackers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and still sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes, directionally. A Feishu plugin pacing helper is the maintainable direction, but the public config name, default, inheritance behavior, and send-surface coverage should be scoped by maintainers first.
