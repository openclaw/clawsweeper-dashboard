## Summary

Keep open. Current main still lacks a Feishu-owned outbound pacing configuration and shared send gate, while the related open trackers cover broader global rate limiting and reactive retry/backoff rather than this proactive channel-specific request.

## What This Changes

The issue requests configurable minimum-interval/rate-limiting behavior for Feishu message sends, including reducing bursts from chunked LLM responses.

## Best Possible Solution

Add a Feishu-plugin-owned per-account outbound pacing helper, expose it in schema/manifest/docs at top level and per account, and route burst-capable Feishu delivery paths through it while keeping broader global rate limiting and reactive retry/backoff in their separate trackers.
