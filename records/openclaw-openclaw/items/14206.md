## Summary

Keep open: current main still lacks a Feishu outbound pacing/minimum-interval setting, and the related broad rate-limit and Feishu retry/backoff trackers do not supersede this proactive Feishu-specific request.

## What This Changes

Add configurable Feishu message pacing so bursty chunked or outbound sends wait a minimum interval between Feishu API calls.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broad provider limits and reactive retry/backoff to their own trackers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and still sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes, directionally. The maintainable fix is a Feishu plugin pacing helper with top-level and per-account config, not a generic core limiter or retry/backoff-only change.
