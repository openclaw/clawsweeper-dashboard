## Summary

Keep open: current main and the latest release include reactive Feishu send retry/backoff, but they still do not provide the requested proactive outbound minimum-interval or pacing configuration for Feishu message bursts.

## What This Changes

Request to add configurable Feishu message rate limiting/minimum send interval, and possibly adjust response chunking, to avoid Feishu temporary API bans from bursty outbound sends.

## Best Possible Solution

Add a Feishu-plugin-owned per-account outbound pacing layer that composes with the existing retry helper, is exposed in schema/manifest/docs, and covers burstable Feishu delivery surfaces without becoming broad core rate limiting.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has reactive retry but no Feishu outbound minimum-interval configuration, and chunked reply/comment paths can still issue consecutive sends without a pacing gate.

## Solution Assessment

Yes. A Feishu-owned per-account pacing helper is the narrow maintainable direction; the broad cross-provider rate-limit tracker and the merged reactive retry fix should remain separate scopes unless maintainers intentionally merge them.
