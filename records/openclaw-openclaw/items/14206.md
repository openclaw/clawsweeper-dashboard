## Summary

Keep open. Current main and the latest v2026.6.8 release have reactive Feishu send retries, but still do not provide the requested proactive outbound pacing/minimum-interval configuration before bursty chunks, media, comment replies, outbound messages, or streaming-card sends reach Feishu.

## What This Changes

Request to add configurable proactive outbound message pacing/minimum interval for the Feishu channel, including behavior that can reduce bursty chunked responses.

## Best Possible Solution

Add a Feishu-owned outbound scheduler around message create/reply-style sends, configurable at top level and per account with safe defaults, while leaving broader cross-provider rate limiting to https://github.com/openclaw/openclaw/issues/13615 and retry-after-failure behavior to the merged work behind https://github.com/openclaw/openclaw/issues/70879.

## Reproduction Assessment

Yes, at source level: current main has Feishu chunked/comment/outbound send paths and reactive retry handling, but no pre-send minimum interval or scheduler. I did not run a live Feishu burst test in this read-only review.

## Solution Assessment

Yes. A Feishu-plugin-local pacing layer is the best shape for this issue; the merged retry helper is useful adjacent work but only recovers after Feishu rejects a burst.
