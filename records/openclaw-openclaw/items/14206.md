## Summary

Keep open. Current main still has no Feishu-owned outbound pacing or minimum-interval configuration, and the related broad rate-limit and Feishu retry/backoff trackers cover adjacent but not superseding work.

## What This Changes

The issue requests a configurable minimum interval for Feishu outbound messages, with possible alignment to how LLM responses are segmented into Feishu messages.

## Best Possible Solution

Add a Feishu-plugin-owned per-account outbound pacing helper, expose it in config/schema/manifest/docs, and apply it consistently to burstable Feishu delivery surfaces while leaving reactive retry/backoff and broad cross-provider limiting on their existing trackers.

## Reproduction Assessment

## Solution Assessment

Yes. A Feishu-plugin-owned outbound pacing helper is the narrowest maintainable direction for this issue, while Feishu retry/backoff after rate-limit errors should remain tracked separately at https://github.com/openclaw/openclaw/issues/70879.
