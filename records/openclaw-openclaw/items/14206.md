## Summary

Keep open: current main still lacks a Feishu outbound pacing/minimum-interval setting or shared delivery gate, and the related broad rate-limit and Feishu retry/backoff issues do not supersede this proactive Feishu-specific request.

## What This Changes

Request a configurable Feishu channel minimum interval or rate limiter so chunked LLM replies and other outbound sends do not burst Feishu APIs.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broad core limits and reactive retries to their separate trackers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection against current main gives high confidence that the requested Feishu pacing config and shared outbound pacing helper are absent.

## Solution Assessment

Yes directionally: a Feishu-owned per-account pacing helper is narrower and cleaner than a broad core limiter for this request. The exact public config name, default, inheritance behavior, and delivery-path coverage should be decided by maintainers before implementation.
