## What This Changes

Request to add Feishu channel message rate limiting, including a configurable minimum interval and possibly adjusted response chunking.

## Best Possible Solution

Add a Feishu-owned outbound scheduler around message create/reply calls, configurable at top level and per account, keyed narrowly enough to protect rate-limited targets without unnecessarily serializing independent chats, and document it alongside existing Feishu message limits.

## Reproduction Assessment

Yes, at source level. Current main has no Feishu outbound pacing config or scheduler, and existing tests/code permit concurrent and chunked message sends without proactive spacing.

## Solution Assessment

Yes, with a narrower boundary than changing LLM segmentation. A Feishu message-API scheduler covers text, cards, comments, media, and streaming-card initial sends while keeping chunking and retry behavior as separate concerns.
