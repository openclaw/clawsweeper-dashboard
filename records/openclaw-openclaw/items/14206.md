## Summary

Keep open: current main still lacks Feishu outbound pacing/minimum-interval configuration, and the Feishu reply, comment, message-tool, media, and streaming-card paths do not share a proactive per-account pacing helper. The related #13615 and #70879 reports are useful context, but they track broader rate limiting and retry/backoff rather than this Feishu-specific pacing request.

## What This Changes

Add configurable minimum-interval pacing for Feishu outbound messages, including bursty chunked responses.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, expose it in schema/manifest/docs, and apply it consistently to burst-prone Feishu delivery paths while leaving broader provider limits to #13615 and retry/backoff to #70879.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested Feishu pacing config and sends chunked/outbound messages without a shared minimum-interval helper.

## Solution Assessment

Yes. The narrow maintainable solution is a Feishu plugin pacing helper with top-level and per-account config, not a generic core limiter or a retry/backoff-only change.
