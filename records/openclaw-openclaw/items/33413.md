## Summary

Keep open: current main and v2026.6.10 still use a generic Slack assistant thread status and do not update it from tool progress events; the direct implementation PRs are closed unmerged, and the broader Slack-safe progress issue is adjacent rather than a replacement.

## What This Changes

Request Slack assistant thread status text to reflect the currently running tool instead of staying on a generic typing message.

## Best Possible Solution

Keep this as the direct tracker for Slack assistant-thread tool status, with any fix reusing shared progress labels, deduping unchanged status writes, clearing reliably, and coordinating with the Slack-safe visibility policy.

## Reproduction Assessment

Yes, from source inspection: current Slack dispatch only sets and clears `is typing...`, while the agent runner already emits `onToolStart` events that Slack does not use for assistant-thread status text. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Yes for keeping this issue open, but not for the earlier Slack-only map implementation attempts. The best fix should derive labels from the shared progress display model, dedupe status writes, include focused tests and live Slack proof, and account for the open Slack-safe visibility policy.
