## Summary

Keep open: current main and v2026.6.8 still hardcode Slack assistant thread status to `is typing...` and do not update it from Slack `onToolStart`; https://github.com/openclaw/openclaw/pull/94246 is open as the active fix candidate but has not merged.

## What This Changes

Request Slack assistant thread status to update with per-tool progress labels during agent runs instead of only showing static `is typing...`.

## Best Possible Solution

Land a narrow Slack-owned fix through the active PR or a replacement that derives status text from the shared progress display model, dedupes unchanged writes, clears reliably, and includes focused tests plus live Slack proof.

## Reproduction Assessment

Yes for the feature gap from source inspection: current Slack dispatch only sets and clears `is typing...`, while the agent runner already emits `onToolStart` events that Slack does not use for assistant status text. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

Yes, keeping this issue open while the active PR is reviewed is the right path. The best implementation should reuse the existing Slack status helper and shared progress labels rather than adding a new config option or a separate Slack-only tool registry by default.
