## Summary

Keep open: current main and v2026.6.6 still only start and clear Slack assistant thread status with generic text; the direct implementation PRs are closed unmerged, so the requested tool-level progress remains valid and narrow.

## What This Changes

Request Slack assistant thread status to change from a static typing indicator to tool-specific progress labels during agent tool execution.

## Best Possible Solution

Implement this in Slack dispatch using the existing `onToolStart` and `setSlackThreadStatus` paths, derive text from the shared progress display model, dedupe unchanged writes, preserve final clearing, and prove it with focused tests plus live Slack evidence.

## Reproduction Assessment

Yes for the feature gap from source inspection, not as a failing bug reproduction. Current Slack dispatch sets only `is typing...` and clears it, while agent execution already emits `onToolStart` events that Slack does not use for assistant status text.

## Solution Assessment

Yes, keeping this open and queueing a narrow Slack-owned fix is the best path. The maintainable implementation should reuse the existing status helper and shared progress labels rather than a new Slack-only tool map or a new config option by default.
