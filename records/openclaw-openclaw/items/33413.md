## Summary

Keep open: current main still exposes the tool-start signal and Slack native status wrapper, but Slack dispatch only uses native `assistant.threads.setStatus` for static reply start/clear and routes tool progress to reactions or draft preview instead.

## What This Changes

Request Slack to update native assistant thread status with human-readable tool labels during agent tool execution.

## Best Possible Solution

Add Slack-local native status wiring that maps known tool names to concise labels, updates `ctx.setSlackThreadStatus` on tool-start events when a status thread exists, preserves start/clear cleanup, and covers the behavior with focused Slack tests plus docs/changelog updates.

## Reproduction Assessment

Yes. A source-level reproduction is clear: run Slack dispatch through reply start and invoke `onToolStart({ name: "exec" })`; current main only updates native Slack status for static start/clear while tool starts update reactions and draft preview progress.

## Solution Assessment

Yes. Slack-local wiring is the narrowest maintainable solution because core already emits `onToolStart` and Slack already owns `setSlackThreadStatus`; a broader plugin hook or new core status API is unnecessary for this request.
