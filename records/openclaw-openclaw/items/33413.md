## Summary

Keep open: current main still exposes the core tool-start signal and Slack native status wrapper, but Slack dispatch only writes native assistant-thread status at reply start/clear and sends tool progress to reactions or draft previews instead.

## What This Changes

Request Slack native assistant-thread status to show human-readable labels for the active tool during agent runs instead of only the static typing status.

## Best Possible Solution

Add Slack-local native status updates on tool-start events with concise labels, dedupe repeated labels, preserve start/clear cleanup, and cover the behavior with focused Slack tests plus docs/changelog updates.

## Reproduction Assessment

Yes for source-level verification. Run Slack dispatch with a reply thread and invoke `onToolStart({ name: "exec" })`; current main updates native status only for start/clear while tool starts update reactions and draft preview progress.

## Solution Assessment

Yes. Slack-local callback wiring is the narrowest maintainable direction because core already emits `onToolStart` and Slack already owns `setSlackThreadStatus`; broader custom status config should stay with the separate customization request.
