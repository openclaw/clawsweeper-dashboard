Keep open: current main and v2026.6.10 still only set and clear Slack assistant thread status as `is typing...`; the Slack tool-start callback does not update assistant status text, and the direct candidate PRs are closed unmerged.

## What This Changes

Request to update Slack assistant thread status during agent tool execution with human-readable per-tool progress labels instead of a static typing indicator.

## Best Possible Solution

Implement Slack-owned progress-status wiring using shared progress display labels, dedupe unchanged status writes, clear reliably, avoid new config unless maintainers accept it, and validate with focused tests plus redacted live Slack proof.

## Reproduction Assessment

Yes, from source inspection: current Slack dispatch only sets and clears `is typing...`, while agent execution already emits `onToolStart` events that Slack does not use for assistant status text. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Unclear until maintainer direction on Slack-visible progress detail is settled. The implementation shape should reuse shared progress labels and dedupe status writes rather than adding a Slack-only registry or default config surface.
