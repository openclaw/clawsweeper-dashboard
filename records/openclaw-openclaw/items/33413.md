## Summary

Keep open: current main and v2026.6.10 still set Slack assistant thread status only to `is typing...` and clear it at the end; tool-start callbacks update reactions and preview progress, but not `assistant.threads.setStatus`. The latest direct fix attempts are closed unmerged, and the remaining implementation should be reconciled with the open Slack-safe progress visibility work before another PR is queued.

## What This Changes

The issue requests Slack assistant thread status text that changes per running tool instead of staying on a static typing message.

## Best Possible Solution

Keep this as the canonical direct request, then implement it through a Slack-owned safe progress/status policy that derives labels from the shared progress display model, deduplicates unchanged status writes, clears reliably, and includes focused tests plus redacted live Slack proof.

## Reproduction Assessment

Yes, from source inspection: current main starts Slack assistant status with `is typing...`, clears it afterward, and routes tool starts only to reactions/preview progress. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Unclear as a product-ready implementation: the `onToolStart` seam is appropriate, but the best fix should reuse shared progress labels, avoid repeated Slack status writes, and account for Slack-safe visibility rather than adding another standalone Slack tool map.
