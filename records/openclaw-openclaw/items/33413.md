Keep open: current `main` and latest release still set Slack assistant thread status to `is typing...` on start and clear it on stop, while tool-start events only drive reactions/preview progress. The direct implementation PRs are closed unmerged, so the requested Slack tool-level status behavior remains actionable.

## What This Changes

The issue requests Slack assistant thread status updates that show the currently running tool instead of a static typing message.

## Best Possible Solution

Implement the status update in Slack dispatch using the existing `onToolStart` seam, derive text from the shared progress/tool display model, dedupe unchanged status writes, preserve final clearing, and add focused Slack dispatch tests plus live Slack proof before landing.

## Reproduction Assessment

Yes, from source inspection. Current `main` and `v2026.6.6` set Slack assistant status only to `is typing...` and clear it, while `onToolStart` receives tool events without updating assistant thread status.

## Solution Assessment

Yes, the requested `onToolStart` wiring is the right seam, but the best implementation should reuse the shared progress display model and dedupe status writes rather than add a Slack-only tool-label registry.
