## Summary

Keep open: current main and the latest release still enforce append-only target-path protection but allow the permitted memory-flush write to append arbitrary model content without size bounds, validation, dedupe, rollback, or a validator hook.

## What This Changes

The issue requests bounded and validated append semantics for pre-compaction memory-flush writes to `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Define a minimal stable memory-flush append contract at the runtime/plugin boundary: conservative pre-mutation size and line bounds, model-facing rejection, redacted telemetry, focused tests, and only a maintainer-approved validator/config surface if needed.

## Reproduction Assessment

Yes for the source-level path: current main accepts and appends arbitrary allowed-path memory-flush content without size or validation checks. No live end-to-end long-session reproduction was run here, but the reporter supplied repeated natural recurrence evidence.

## Solution Assessment

Unclear. The broad config example may be too much surface for OpenClaw's config bar, while a narrower built-in bound plus a maintainer-approved plugin/runtime validator seam looks like the more maintainable direction.
