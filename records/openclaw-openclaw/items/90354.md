## Summary

Keep open. Current main and the latest release still only provide the adjacent append-only target/path guard; the requested bounded size/line checks, validation or rollback hook, duplicate protection, and structured accepted/rejected telemetry remain product/API work.

## What This Changes

Request bounded and validated pre-compaction memory-flush append semantics for daily memory writes, including size/line guardrails, validation or rollback behavior, telemetry, and stronger silent-mode handling.

## Best Possible Solution

Add a narrow memory-flush append contract at the runtime/plugin write boundary with conservative built-in preflight limits and telemetry first, then expose configurable validator or rollback hooks only after maintainers decide the plugin/config API shape.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: source inspection gives a clear narrow path where an oversized allowed-path memory-flush write is appended without size or line validation, but I did not live-reproduce the reported Feishu long-session recurrence.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the API shape; a small built-in preflight guard at the existing wrapper is the safest first step, while broad user-configurable validator and rollback hooks need product/API review.
