## Summary

Keep open. Current main and the latest release still provide append-only target-path protection, but they do not implement pre-mutation size/line bounds, post-write validation, rollback/reject policy, duplicate protection, or a stable validator hook for pre-compaction memory-flush appends.

## What This Changes

Request bounded and validated append semantics for pre-compaction memory flush writes to `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Define a minimal stable memory-flush append contract at the runtime/plugin boundary: pre-mutation size and line bounds, safe structured failure telemetry, focused tests, and a validator hook only after maintainers settle the API shape.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection shows the allowed memory-flush write path appends arbitrary model content without size or validation checks, and the reporter provided repeated natural recurrence evidence.

## Solution Assessment

Unclear. The broad config surface proposed in the issue may be too large for OpenClaw’s config bar, but a narrower built-in bound plus a maintainer-approved plugin/runtime validator seam looks like the maintainable direction.
