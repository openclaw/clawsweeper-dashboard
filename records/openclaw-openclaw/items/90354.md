## Summary

Keep open. Current main and latest release still restrict memory-flush writes to the target daily-memory path and append-only mode, but they do not implement pre-mutation size/line bounds, daily-memory validation, duplicate/scaffold rejection, rollback/post-check policy, or a stable validator hook.

## What This Changes

The issue requests runtime-level bounded, validated, and observable append behavior for pre-compaction memory-flush writes to daily memory files.

## Best Possible Solution

Define a minimal stable memory-flush append contract at the runtime/plugin boundary: pre-mutation size and line bounds, safe structured rejection/telemetry, focused tests, and a validator hook only after maintainers settle the API shape.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection shows current main and v2026.6.8 still append arbitrary allowed-path content, and the reporter supplied repeated natural recurrence evidence including the June 18/19 recurrence.

## Solution Assessment
