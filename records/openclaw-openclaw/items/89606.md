## Summary

Keep open: current main still lets a `policy-changed` refresh reuse a persisted `plugins[]` view without checking whether durable path/npm install records are missing from that view, and the linked downstream NemoClaw report/workaround supports real user impact. The open closing PR is the active fix path, but it has not merged yet.

## What This Changes

The issue asks for `policy-changed` plugin registry refreshes to keep path/npm install-record-backed plugins present in `plugins[]` so inspection, allowlist validation, slash commands, and plugin CLI commands keep working.

## Best Possible Solution

Land one canonical registry refresh fix, likely the open closing PR, that makes `policy-changed` refresh rebuild when durable path/npm install records are recoverable but missing from `plugins[]`.

## Reproduction Assessment

Yes. A persisted index with path/npm `installRecords` missing from `plugins[]` can still pass the current `policy-changed` fast path from source inspection, though I did not rerun the downstream GPU sandbox.

## Solution Assessment

Yes. Fixing the persisted registry refresh guard/rebuild path is the best solution because it restores the canonical runtime registry view instead of adding a validator-only fallback.
