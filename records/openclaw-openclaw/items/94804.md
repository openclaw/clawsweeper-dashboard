## Summary

Keep open: current main has targeted wake, delivery-context carry, hook mappings, next-turn injection, and dedupe primitives, but it still lacks the requested first-class async route registry with lifecycle, matching, ambiguity, privacy, and safe-action policy.

## What This Changes

Request a framework-level async result routing layer that maps later external events back to the originating topic session, posts a visible summary, and wakes that session safely.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract first, then decide which minimal Gateway storage or SDK seam is needed instead of adding a one-off core ledger.
