## Summary

Keep open: current main and v2026.6.8 provide targeted wake, hook, delivery-context, and local dedupe building blocks, but not the requested first-class async route registry with lifecycle, matching, ambiguity handling, privacy policy, and approval-safe action policy.

## What This Changes

The issue requests a first-class async route registry that maps later external events back to originating topic sessions, delivers safe visible summaries, dedupes, handles ambiguity, and wakes the saved session.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, dedupe, and approval policy before deciding whether any Gateway-owned registry or storage belongs in core.

Not applicable: this is a feature request, not a reported broken current contract. Source inspection confirms the lower-level wake, hook, delivery-context, and dedupe pieces exist while the requested async route registry and policy layer are not present.
