## Summary

Keep open: current main and v2026.6.9 provide lower-level targeted wake, hook/cron ingress, delivery context, and dedupe pieces, but not the requested first-class async route registry with lifecycle, event matching, ambiguity handling, privacy policy, and safety policy.

The issue requests a framework-level async routing layer that maps later external events back to originating topic sessions for visible summaries, targeted wakes, dedupe, ambiguity handling, and approval-safe follow-up.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, dedupe, commitments, and approval policy before deciding whether any core-owned registry is warranted.
