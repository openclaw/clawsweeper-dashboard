## Summary

Keep open: current main has the lower-level targeted wake, delivery-context, hook mapping, and dedupe pieces, but it still lacks the requested first-class async route registry with route lifecycle, matching, ambiguity handling, privacy policy, and safe action policy.

## What This Changes

The issue requests a framework-level async result routing layer that maps later external events back to the originating topic session and can notify, summarize, wake, dedupe, and enforce safety policy.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, and dedupe primitives without adding a heavyweight core event bus by default.

## Reproduction Assessment

Not applicable: this is a feature request, not a reported broken current contract. Source inspection confirms the lower-level targeted wake pieces exist while the requested route registry and policy layer are not present.

## Solution Assessment

Unclear as an implementation direction. A core route registry may be useful, but VISION.md pushes optional capability toward plugins and warns against heavy orchestration layers, so maintainers should choose the API/product boundary before implementation.
