## Summary

Keep open: current main has targeted wake, hook mappings, delivery-context carry, plugin event hooks, and dedupe building blocks, but it still lacks the requested first-class async route registry with lifecycle, matching, ambiguity handling, privacy policy, and safe action policy.

## What This Changes

The issue requests a first-class async route registry that matches later external events back to originating topic sessions, posts visible summaries, wakes the saved session, dedupes events, and enforces privacy/safety policy.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, dedupe, and approval policy before deciding whether any core storage or Gateway-owned registry is warranted.

Not applicable: this is a feature request, not a reported broken current behavior. Source inspection confirms the lower-level wake, hook, delivery-context, plugin-event, and dedupe pieces exist while the requested route registry and policy layer are not present.
