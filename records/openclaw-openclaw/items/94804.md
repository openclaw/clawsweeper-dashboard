## Summary

Keep open: current main and the latest release provide lower-level targeted wake, hook mapping, delivery-context, and dedupe primitives, but not the requested first-class async route registry with lifecycle, matching, ambiguity handling, privacy policy, and safe action policy.

## What This Changes

Requested a framework-level async callback-routing layer that matches later external events back to the originating topic session, posts a safe visible summary, and wakes that session.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, and dedupe primitives without adding a heavyweight core event bus by default.

## Reproduction Assessment

Not applicable: this is a feature request, not a reported broken current contract. Source inspection confirms the lower-level targeted wake pieces exist while the requested route registry and policy layer are not present.

## Solution Assessment

Unclear as an implementation direction. A core route registry may be useful, but the best owner boundary needs maintainer product judgment because VISION.md pushes optional capability toward plugins and warns against heavy orchestration layers.
