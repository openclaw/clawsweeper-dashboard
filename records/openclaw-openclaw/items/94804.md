## Summary

Keep open: current main and the latest release have targeted wake, hook mapping, delivery-context, and dedupe primitives, but not the requested first-class async route registry with lifecycle, matcher, ambiguity handling, privacy policy, and safe action routing.

## What This Changes

Request a first-class async result routing layer that matches later external events to originating topic sessions, posts summaries, wakes the saved session, dedupes events, and enforces safe action policy.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, and dedupe primitives without adding a heavyweight core event bus by default.

## Reproduction Assessment

Not applicable: this is a feature request, not a reported broken current contract. Source inspection confirms lower-level targeted wake pieces exist while the requested route registry and policy layer are not present.

## Solution Assessment

Unclear as an implementation direction. A first-class route registry may be useful, but VISION.md pushes optional capability toward plugins and warns against heavy orchestration layers, so maintainers should choose the core/plugin API boundary first.
