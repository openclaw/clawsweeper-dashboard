## Summary

Keep open: current main and v2026.6.10 have targeted wake, hook, delivery-context, dedupe, commitments, and Task Flow building blocks, but not the requested first-class async route registry with external-event matching, lifecycle, ambiguity handling, privacy policy, and safe action routing.

## What This Changes

The issue requests a first-class async route registry that maps later external events to the originating topic session, posts safe visible summaries, dedupes events, and wakes the saved session.

## Best Possible Solution

Keep this open for maintainer design: define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, dedupe, commitments, and approval policy before deciding whether any core-owned registry is warranted.

## Reproduction Assessment

Not applicable: this is a feature request, not a reported broken current contract. Source and docs inspection show the lower-level wake, hook, delivery-context, dedupe, commitment, and Task Flow pieces exist while the requested async route registry and policy layer are not present.

## Solution Assessment

Unclear as an implementation direction. A first-class route contract may be useful, but VISION.md and AGENTS.md push toward a lean core and plugin/API seams, so maintainers should choose the product, storage, privacy, and safety boundary before implementation.
