## Summary

Keep open: current main has targeted wake, hooks, delivery-context, dedupe, and automation building blocks, but not the requested async route registry with lifecycle, external-event matching, ambiguity handling, dedupe ledger, privacy policy, and safe action policy.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, dedupe, commitments, Task Flow, and approval policy before deciding whether core-owned storage is warranted.
