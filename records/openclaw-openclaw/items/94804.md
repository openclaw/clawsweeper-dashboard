## Summary

Keep open: current main and the latest release have targeted wake, delivery-context, hook, dedupe, and plugin-event building blocks, but they do not implement the requested first-class async route registry with lifecycle, matching, ambiguity handling, privacy policy, and approval-safe action policy.

## What This Changes

Request a framework-level async route contract that matches later external events to an originating topic session, posts a visible summary, and wakes that session safely.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, event subscriptions, and dedupe primitives, with maintainers deciding which minimal core seam is missing before implementation.
