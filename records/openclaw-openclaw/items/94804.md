## Summary

Keep open: current main and v2026.6.8 have targeted wake, hooks, delivery-context, and dedupe building blocks, but not the requested first-class async route registry with matching, lifecycle, ambiguity, privacy, and action-policy semantics.

## What This Changes

The issue requests a first-class async route layer that matches later external events back to originating topic sessions, posts safe visible summaries, dedupes events, and wakes the saved session.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, and dedupe primitives without adding a heavyweight core event bus by default.

## Reproduction Assessment

Not applicable: this is a feature request, not a reported broken current contract. Source inspection confirms the lower-level wake and hook pieces exist while the requested route registry and policy layer are not present.

## Solution Assessment

Unclear as an implementation direction. A framework route layer may be useful, but maintainers need to choose the core/plugin/API, storage, privacy, and safety boundary before implementation.
