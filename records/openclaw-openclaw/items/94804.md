## Summary

Keep open: current main has targeted wake, delivery-context, hooks, and dedupe building blocks, but it does not provide the requested first-class async route registry with lifecycle, matching, ambiguity handling, visible-topic delivery, and privacy/safety policy.

## What This Changes

The issue requests a framework-level async result routing layer that maps later external events back to the originating topic session and safely wakes or updates that session.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, and dedupe primitives without adding a heavyweight core event bus by default.

Not applicable: this is a feature request, not a reported broken current contract; source inspection shows targeted wake pieces exist while the requested registry and policy layer do not.
