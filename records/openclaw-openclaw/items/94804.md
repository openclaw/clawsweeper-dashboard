## Summary

Keep open: current main and the latest release provide targeted wake, hooks, delivery-context, cron/task, and dedupe primitives, but not the requested first-class async route registry with lifecycle, matching, ambiguity handling, privacy policy, and safe action policy.

## What This Changes

The issue requests a first-class async result routing layer that maps later external events back to originating topic sessions with registry state, matching, dedupe, visible delivery, and safe targeted wake behavior.

## Best Possible Solution

Define a narrow plugin/API-friendly async route contract that composes existing hooks, targeted wake, delivery context, dedupe, and approval policy before deciding whether any core storage or Gateway-owned registry is warranted.
