## Summary

Keep open: current main and the latest release still contain the reported `includeDetails` guard, and an open linked PR is the active fix candidate, but the exact always-suppress policy still needs maintainer review because current tests intentionally surface some exec warnings.

## What This Changes

The issue asks OpenClaw to suppress exec-like tool error warnings regardless of `includeDetails` in `resolveToolErrorWarningPolicy`.

## Best Possible Solution

Make the exec-like warning policy explicit, then land or revise the linked PR with focused regression coverage for includeDetails=true, full-verbose, heartbeat, and cron cases before closing this issue.

## Reproduction Assessment

Yes, source-level: current main still has the `isExecLikeToolName(...) && !includeDetails` guard, and current tests show exec warnings are intentionally emitted in several detail-mode paths. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear. Removing the guard is a plausible narrow direction, but the best fix needs maintainer confirmation because existing mutating-tool and detail-mode behavior may be intentional in some contexts.
