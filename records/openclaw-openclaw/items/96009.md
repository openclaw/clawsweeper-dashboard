## What This Changes

The issue asks OpenClaw to suppress exec-like embedded-agent tool error warnings even when detailed error payloads are included.

## Best Possible Solution

Define the exec-like warning contract for includeDetails/full/cron/heartbeat cases, then land or replace the linked fix with focused regression coverage for the chosen policy.

## Reproduction Assessment

Yes, source-level: current main still has the `isExecLikeToolName(...) && !includeDetails` guard, and includeDetails=true exec cases are covered by current tests that expect warning payloads. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear until maintainer policy review: removing the guard is a plausible narrow direction, but the fix must reconcile the mutating branch and current tests that intentionally keep some exec detail warnings visible.
