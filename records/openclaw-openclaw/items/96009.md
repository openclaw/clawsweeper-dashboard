## Summary

Keep open: current main and the latest release still have the reported `includeDetails` guard, but existing tests deliberately expect exec warnings when details are included, so the linked fix PR needs maintainer review rather than automatic cleanup or closure.

## What This Changes

The issue asks OpenClaw to suppress exec-like tool failure warnings even when redacted details are included.

## Best Possible Solution

Codify the intended includeDetails policy for exec-like failures, then land or revise the linked PR with focused coverage for verbose, cron, heartbeat, and Codex command paths.

## Reproduction Assessment

Yes for a source-level reproduction: current main falls through to a warning when `isExecLikeToolName(...)` is true and `includeDetails` is true. I did not run a live channel turn, and current tests indicate this may be intentional in some contexts.

## Solution Assessment

Unclear as written: removing the guard matches the issue’s requested behavior, but it conflicts with current tests that preserve verbose/cron/heartbeat exec diagnostics. The safer path is a maintainer decision on the warning policy before merging the linked PR.
