## Repair Work Prompt

Fix the macOS companion Voice Wake built-in tester so it accepts the trigger-only flow promised by the UI and documented/runtime behavior. In current main, `VoiceWakeTestCard` tells users to say a trigger word, but `VoiceWakeTester` only reports `.detected` when `match.command` is non-empty, and the fallback path uses `minCommandLength: 1`, so bare trigger speech can fail as `No trigger heard`. Align `VoiceWakeTester` and the timeout/silence fallback with `VoiceWakeRuntime` trigger-only pause semantics, preserving false-positive protection for ordinary speech that merely mentions a trigger. Add focused Swift tests proving trigger-only tester success for configured triggers and rejection for trailing ordinary mentions. Do not change provider/TTS behavior, Talk Mode playback, global routing semantics, or bundled trigger defaults. Update docs only if behavior text changes, and add a user-facing changelog fix entry crediting `Thanks @zoiks65`; also mention #59767 as duplicate context if useful. Validate with Swift formatting/lint and targeted macOS Swift tests where available, then run the changed gate in Testbox if the repair lane has maintainer access.
