Keep this open: current main and the latest release still route the reported Windows npm certificate failure through generic npm install failure handling, while the only direct Windows fallback PR was closed unmerged after TLS-bypass concerns.

## What This Changes

The issue reports that the Windows PowerShell installer fails during `npm install -g openclaw@latest` with `UNABLE_TO_VERIFY_LEAF_SIGNATURE`.

## Best Possible Solution

Add Windows installer handling that detects npm certificate failures and prefers trusted CA or `cafile` guidance; any insecure retry should be explicit, warned, and maintainer-approved.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The source path is clear: if npm emits the reported certificate error, the Windows installer still falls through to generic npm failure handling.

## Solution Assessment
