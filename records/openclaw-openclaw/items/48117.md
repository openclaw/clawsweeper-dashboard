## What This Changes

The issue requests that the Windows PowerShell installer recover from npm TLS certificate verification failures during `npm install -g openclaw@latest`.

## Best Possible Solution

Choose a secure Windows installer TLS recovery path that preserves verification where possible, such as explicit CA/cafile/system-CA handling or an explicit opt-in fallback, then prove it on a real Windows install path.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the Windows installer still delegates to npm and only special-cases missing Git, so a returned TLS verification error would follow the reported failure path.

## Solution Assessment

No/unclear. The closed `--strict-ssl=false` retry is a plausible workaround, but the best supported fix needs maintainer/security approval and real Windows proof before becoming installer behavior.
