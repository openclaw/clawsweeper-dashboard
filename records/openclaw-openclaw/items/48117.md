## Summary

This should stay open: current main still treats the reported Windows npm certificate failure as a generic npm install failure, and the only Windows-specific fallback PR was closed unmerged after TLS-bypass concerns.

## What This Changes

The issue reports that the documented Windows PowerShell installer fails during `npm install -g openclaw@latest` with `UNABLE_TO_VERIFY_LEAF_SIGNATURE`.

## Best Possible Solution

Add Windows installer handling for npm certificate failures that prefers trusted CA/system-store or `cafile` guidance, with any insecure retry limited to explicit, clearly warned user opt-in and focused installer tests.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The source path is clear: if npm emits the reported TLS error during the documented Windows installer command, the installer falls through to generic npm failure handling with no certificate-specific recovery.

## Solution Assessment

Unclear. Detecting npm certificate failures in the Windows installer is the right area, but the final recovery design should avoid silent TLS verification downgrade and needs maintainer security direction.
