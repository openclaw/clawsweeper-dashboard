## Summary

Keep this open: current main and v2026.5.20 still route the reported Windows npm leaf-certificate failure through generic npm failure handling. The direct Windows retry PR was closed unmerged, and the merged Linux nvm CA fix is adjacent but does not cover the PowerShell installer path.

## What This Changes

The issue reports that the documented Windows PowerShell installer fails when npm exits with `UNABLE_TO_VERIFY_LEAF_SIGNATURE` while installing `openclaw@latest`.

## Best Possible Solution

Add Windows installer handling that detects npm certificate verification failures and prefers trusted CA, `cafile`, or `NODE_EXTRA_CA_CERTS` guidance; any insecure retry should be explicit, warned, and security-approved.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The reporter provided a concrete Windows PowerShell failure, and source inspection shows that the same npm certificate error would currently fall through to generic npm failure handling.

## Solution Assessment

Unclear. The Windows installer is the right surface, but the previous `strict-ssl=false` retry approach is not the best fix without a safer trusted-CA path or explicit security-approved opt-in.
