## Summary

Keep open: current main and the latest release still treat the Windows PowerShell npm TLS failure as a generic npm install failure, while the merged related work only fixes Linux nvm CA handling and the exact Windows retry PR was closed unmerged after security concerns.

## What This Changes

The issue reports that installing OpenClaw through the Windows PowerShell installer fails when npm returns `UNABLE_TO_VERIFY_LEAF_SIGNATURE`.

## Best Possible Solution

Keep this issue open for a security-reviewed Windows installer fix that detects npm certificate failures and guides or opts users into a trusted CA path without silently disabling TLS verification.

## Reproduction Assessment

No high-confidence live reproduction was run against a Windows host with the certificate-store mismatch, but the source path is clear: `install.ps1` calls npm once and treats any nonzero exit as a generic install failure.

## Solution Assessment

No. The previously proposed `--strict-ssl=false` retry would solve some installs but needs a maintainer-approved security design; a safer path should prefer trusted CA guidance/export or an explicit opt-in fallback.
