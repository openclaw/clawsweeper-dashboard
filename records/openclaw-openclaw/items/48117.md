## Summary

Keep this open: current main and v2026.5.22 still route the reported Windows npm leaf-certificate failure through generic npm failure handling. The direct Windows retry PR is closed unmerged, and the merged Linux CA work is adjacent but does not cover the PowerShell installer path.

## What This Changes

The issue reports that the Windows PowerShell installer fails during global npm install with `UNABLE_TO_VERIFY_LEAF_SIGNATURE`.

## Best Possible Solution

Add Windows installer handling that detects npm certificate verification failures and prefers trusted CA, `cafile`, or `NODE_EXTRA_CA_CERTS` guidance; any insecure retry should be explicit, warned, and maintainer-approved.

## Reproduction Assessment

No high-confidence live Windows reproduction was run against current main. The reporter provided a concrete PowerShell failure, and source inspection shows the same npm certificate error would currently fall through to generic npm failure handling.

## Solution Assessment

Unclear. The Windows installer is the correct surface, but the closed `strict-ssl=false` retry approach is not the best fix without a safer trusted-CA path or explicit security-approved opt-in.
