## Summary

This should stay open: current main and the latest shipped release still run the Windows installer npm install once and fall back only to generic failure guidance, while the only linked Windows TLS fallback PR is closed unmerged after security concerns around TLS bypass.

## What This Changes

The issue reports the Windows PowerShell installer failing during global npm install with `UNABLE_TO_VERIFY_LEAF_SIGNATURE` instead of completing installation.

## Best Possible Solution

Add Windows installer handling for npm certificate failures that prefers trusted CA/system-store or `cafile` guidance, with any insecure retry limited to explicit, clearly warned user opt-in and covered by focused `install.ps1` tests.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The source path is clear: if npm emits the reported TLS error, `Install-OpenClaw` falls through to generic npm failure handling with no certificate-specific recovery.

## Solution Assessment

Unclear. Detecting npm certificate failures in the Windows installer is the right area, but the final recovery design should avoid silent TLS verification downgrade and needs maintainer security direction.
