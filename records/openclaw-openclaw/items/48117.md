## Summary

Keep this open: current main and the latest release still route the reported Windows npm leaf-certificate failure through generic npm failure handling. The direct Windows fallback PR was closed unmerged, and the merged Linux nvm CA fix is adjacent but does not cover the PowerShell installer path.

## What This Changes

Fix Windows PowerShell installer failures when npm reports `UNABLE_TO_VERIFY_LEAF_SIGNATURE` during the global OpenClaw install.

## Best Possible Solution

Add Windows installer handling that detects npm certificate verification failures and prefers trusted CA, `cafile`, or `NODE_EXTRA_CA_CERTS` guidance; any insecure retry should be explicit, warned, and security-approved.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The reporter provided a concrete Windows PowerShell failure, and source inspection shows that the same npm certificate error would currently fall through to generic npm failure handling.

## Solution Assessment

Unclear. The Windows installer is the right surface, but the previous `strict-ssl=false` retry approach needs a safer trusted-CA or explicit opt-in design before it is the best fix.
