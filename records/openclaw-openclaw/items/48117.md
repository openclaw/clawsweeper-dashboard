## What This Changes

The issue reports that the Windows PowerShell installer fails during `npm install -g openclaw@latest` with `UNABLE_TO_VERIFY_LEAF_SIGNATURE`.

## Best Possible Solution

Land a security-reviewed Windows installer fix that keeps strict TLS verification as the default, detects npm certificate-verification failures, and offers explicit operator-controlled CA remediation or recovery without silently disabling verification.

## Reproduction Assessment

Yes at source level, but not as a live Windows repro. Current `main` and `v2026.6.8` still send the reported npm TLS failure through the generic installer failure path, and a live Windows environment with the mismatched trust chain is still needed for full proof.

## Solution Assessment

Unclear until maintainers choose the trust boundary. The safest direction is not the closed automatic `--strict-ssl=false` retry, but an explicit and well-tested CA trust remediation path that preserves secure defaults.
