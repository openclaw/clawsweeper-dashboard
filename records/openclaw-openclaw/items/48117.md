## What This Changes

The issue asks OpenClaw to handle a Windows PowerShell installer failure where `npm install -g openclaw` exits with `UNABLE_TO_VERIFY_LEAF_SIGNATURE`.

## Best Possible Solution

Ship a maintainer-approved Windows installer response that preserves strict TLS by default, detects npm certificate failures, prefers CA/cafile or `NODE_EXTRA_CA_CERTS` guidance, and avoids any insecure retry unless it is explicit and security-reviewed.

## Reproduction Assessment

Yes from source for the handling gap, but not as a live Windows repro. The current PowerShell installer has no TLS-error-specific branch around its npm install call, so a reported certificate verification failure exits through the generic npm failure path.

## Solution Assessment

Unclear until maintainers choose the security posture. The closed candidate PR's retry-with-`--strict-ssl=false` shape is a plausible unblocker, but the safer endpoint should preserve strict TLS by default and prefer explicit CA configuration or operator consent.
