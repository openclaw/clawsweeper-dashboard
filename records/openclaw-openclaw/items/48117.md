## Summary

Keep open. Current main still has the Windows installer call `npm install -g` and report generic npm failure output, with no TLS-certificate detection, system-CA retry, or targeted `cafile`/`NODE_EXTRA_CA_CERTS` guidance; the earlier linked fix PR was closed unmerged after review raised a security concern about retrying with `--strict-ssl=false`.

## What This Changes

The issue reports that the Windows PowerShell installer fails during global npm installation with `UNABLE_TO_VERIFY_LEAF_SIGNATURE`.

## Best Possible Solution

Add Windows installer handling that preserves TLS verification, such as detecting npm certificate failures and retrying with trusted system/extra CA configuration when supported, then print clear `cafile`/CA guidance if it still fails.

## Reproduction Assessment

No. There is a credible user transcript and current-source proof that the Windows installer does not handle npm certificate failures, but no live Windows/corporate-CA reproduction was established against current main.

## Solution Assessment

Unclear. The earlier `--strict-ssl=false` retry is not the best default because it weakens installer TLS; a maintainer should choose a secure system-CA or explicit-CA remediation path.
