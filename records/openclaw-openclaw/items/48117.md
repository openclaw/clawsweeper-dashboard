## What This Changes

The issue reports that the Windows PowerShell installer fails during the npm install step with `UNABLE_TO_VERIFY_LEAF_SIGNATURE` instead of installing successfully or giving certificate-specific recovery guidance.

## Best Possible Solution

Add Windows installer handling that detects npm certificate verification failures and prefers trusted CA, `cafile`, or `NODE_EXTRA_CA_CERTS` guidance, with any insecure retry limited to an explicit security-reviewed opt-in.

## Solution Assessment
