## Best Possible Solution

Add Windows installer handling for npm certificate verification failures that preserves strict TLS by default, gives trusted `cafile` or `NODE_EXTRA_CA_CERTS` guidance, and permits insecure retry only through explicit maintainer-approved user consent.

## Reproduction Assessment

## Solution Assessment

Unclear. The Windows installer is the right surface, but the closed automatic `strict-ssl=false` retry approach is not the best fix unless the recovery flow preserves secure defaults or requires explicit user consent.
