## Summary

Keep open: current main has not solved the central issue. The root models.json path still persists env-backed auth-profile keyRef as a bare env-var marker, the audit still flags arbitrary all-caps apiKey strings, and plugin catalog sidecars still serialize apiKey; the related models PR was closed unmerged and the catalog-only PR remains open.

## What This Changes

Issue reports that generated per-agent models.json, plus follow-up plugin catalog sidecars, persist provider auth as bare apiKey strings and trigger plaintext secret audit findings or credential exposure.

## Best Possible Solution

Keep this issue as the canonical security/auth tracking item until one reviewed fix covers both generated root models.json auth markers and plugin catalog sidecars, with focused audit and regeneration proof.

## Reproduction Assessment

Yes at source level, but not by a live command in this read-only review. The generator projects env keyRefs to bare strings, the audit flags arbitrary all-caps models.json apiKey strings, and the plugin catalog writer still serializes apiKey into generated catalog.json.

## Solution Assessment

No, current main is not the best solved state yet. A durable fix should use the existing non-secret marker contract or omit catalog auth after maintainer/security approval rather than writing resolved secrets or leaving custom env marker strings to trip the audit.
