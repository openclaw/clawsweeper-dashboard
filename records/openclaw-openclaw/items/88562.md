## What This Changes

Prevent generated `models.json` and plugin-owned catalog snapshots from persisting provider authentication as bare strings or resolved plaintext credentials.

## Best Possible Solution

Enforce marker-or-omission persistence from the unresolved source-config snapshot before every root and plugin catalog write, resolve credentials only at request time, add plugin-catalog audit coverage, and define safe regeneration or cleanup for stale leaked snapshots.

## Reproduction Assessment

Yes. The shared serialization path is clear from current source, and the discussion includes concrete `models.json` output plus a fresh v2026.6.5 production survey of plugin catalog snapshots; this read-only review did not execute a local credential-bearing reproduction.

## Solution Assessment

No, a `models.json`-only marker or audit exception is insufficient. The narrow maintainable fix must enforce the invariant before both root and plugin serialization and cover stale artifacts and audit detection.
