## Summary

Keep open: this PR targets a source-reproducible Claude CLI thinking bug, uses a generic plugin-owned CLI argv seam instead of hard-coding Anthropic behavior in the runner, and I found no blocking review findings in the diff.

## What This Changes

The PR adds a CLI backend `resolveExecutionArgs` hook, wires the Anthropic Claude CLI backend to map non-off `/think` levels to Claude Code `--effort`, and updates docs, changelog, tests, and the Plugin SDK API baseline.

## Best Possible Solution

Land this PR after normal review and CI so Claude CLI thinking remains plugin-owned, works for fresh/resumed/live runs, and stays documented as `/think` budget control separate from reasoning visibility.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: select `claude-cli`, set a non-off `/think` level, and the CLI runner still builds argv from backend args that contain no `--effort`, even though `thinkLevel` exists on the run params.

## Solution Assessment

Yes. The PR's plugin-owned `resolveExecutionArgs` seam is the narrow maintainable solution because it keeps Claude CLI dialect policy in the Anthropic plugin while letting the generic runner apply backend-owned per-run argv rewrites before spawn/live-session startup.
