The PR adds a plugin-owned CLI backend arg-resolution hook and wires the Anthropic `claude-cli` backend to map non-off OpenClaw `/think` levels to Claude Code `--effort`, with docs, changelog, tests, and SDK baseline hash updates.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main shows `claude-cli` fresh/resume args omit `--effort` and the runner has no request-scoped arg hook, so a non-off `thinkLevel` cannot reach the spawned Claude CLI.

## Solution Assessment

Yes. The PR uses a narrow optional backend seam and keeps Claude-specific CLI dialect mapping in `extensions/anthropic`, which is more maintainable than hard-coding `claude-cli` behavior in the core runner.
