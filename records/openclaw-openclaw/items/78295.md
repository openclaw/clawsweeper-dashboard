## Summary

Keep open. Current main still does not support the exact claim that a normal gateway restart runs doctor, but the issue remains a valid config-loss bug report because update-driven doctor repair and provider preset/default-model paths can still rewrite model configuration, and the related open provider-default PR does not fully cover the reported restart/doctor trigger.

## What This Changes

The issue reports that gateway restart or adjacent `openclaw doctor --non-interactive` operations can revert user-authored model assignments and shrink the configured model catalog.

## Best Possible Solution

Reproduce the current-main macOS restart/update/doctor path with a custom multi-provider model catalog, then fix the specific writer or preset path that drops user-authored model entries while landing the narrower provider-default preservation work separately if it remains correct.

## Reproduction Assessment

No. The issue has detailed steps and related before/after symptoms, but current main's normal gateway restart path does not invoke doctor; update-run doctor repair and provider preset writes remain plausible triggers that need live isolation.

## Solution Assessment

Unclear. A new `--preserve-config` flag is not the best first fix; the maintainable path is to identify the exact mutating path and make that existing config write additive or preservation-aware.
