## Summary

Keep open: the reported CLI-runtime maintenance bug is real and current main still gates preflight/memory flush on the raw provider, but this PR still has a blocker because the new gate path can diverge from the actual dispatch resolver for persisted runtime overrides.

## What This Changes

The PR changes the preflight-compaction and memory-flush gates to resolve an effective CLI runtime before deciding whether to skip embedded maintenance, with regression tests for configured runtimes, auth-profile pins, and persisted overrides.

## Best Possible Solution

Use one canonical execution-provider resolver for runReplyAgent, followup-runner, preflight compaction, and memory flush, then keep regressions for configured CLI runtimes, direct-API auth pins, compatible persisted overrides, and incompatible/stale overrides.
