## What This Changes

The PR changes `openclaw status` to reconcile inspectable tasks once, pass that snapshot to registry and audit summaries, and adds focused regression coverage plus a changelog entry.

## Best Possible Solution

Land the focused shared-snapshot status fix after normal maintainer review and merge checks; keep https://github.com/openclaw/openclaw/issues/73531 open until this PR merges.

Yes. Source inspection on current main shows one `getStatusSummary()` invocation reaches two helpers that each call `reconcileInspectableTasks()`, which in turn reaches the clone/sort-heavy `listTaskRecords()` path.

## Solution Assessment

Yes. Sharing one per-status-call snapshot is narrower than adding a module-level cache and invalidation policy, and it preserves the zero-argument helper behavior for other callers.
