## Summary

Keep open for review/merge rather than cleanup: current main still has the duplicate task reconciliation path, the branch fixes that path with a focused shape and sufficient terminal proof, but the exact-head dependency audit is failing and needs maintainer handling before merge.

## What This Changes

The branch computes one reconciled task snapshot in `getStatusSummary()`, passes it to registry and audit summary helpers via backward-compatible optional parameters, and adds focused regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent per-status shared-snapshot fix after the security audit gate is resolved, preserving zero-argument helper fallback behavior for other callers.

## Reproduction Assessment

Yes by source inspection: current main calls both task summary helpers from `getStatusSummary()`, and each helper calls `reconcileInspectableTasks()` before the clone-and-sort `listTaskRecords()` path. I did not run a live 10K-task benchmark in this read-only review.

## Solution Assessment

Yes. Passing one per-status-call snapshot is narrower than the issue's module-level cache suggestion and avoids the extra-work helper wrapper that led https://github.com/openclaw/openclaw/pull/78028 to be closed.
