## What This Changes

The PR adds optional pre-reconciled task snapshot parameters to task registry/audit summary helpers, updates `getStatusSummary()` to reconcile once and share that snapshot, and adds regression tests plus a changelog entry for the status performance fix.

## Best Possible Solution

Land this PR or an equivalent narrow fix once exact-head validation is green: `openclaw status` should reconcile inspectable tasks once per summary build, derive registry and audit summaries from that snapshot, and keep zero-argument helper behavior for tasks and gateway call sites.
