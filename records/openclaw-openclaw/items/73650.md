## Summary

Keep open. Current main still reconciles inspectable tasks twice in the status path, and this PR is the right narrow direction, but the latest patch is not merge-ready because the new maintenance test runtime omits the current required `listAcpSessionEntries` seam.

## What This Changes

The branch reconciles inspectable tasks once in `getStatusSummary()`, passes that shared snapshot into registry and audit summary helpers, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent main-branch fix after updating the new test harness for the current task-maintenance runtime contract; the status path should reconcile once per invocation, and the summary helpers should keep their zero-argument fallback for other call sites.

## Reproduction Assessment

Yes. The current-main reproduction is source-level: `getStatusSummary()` calls two summary helpers, and each helper independently calls `reconcileInspectableTasks()`, which reaches the `listTaskRecords()` clone/sort path.

## Solution Assessment

No as-is. The shared-snapshot approach is the narrow maintainable fix, but the PR must add the missing current runtime stub before it is mergeable.
