## Summary

Keep open. Current main still duplicates task reconciliation in the status path, while this PR is a narrow backward-compatible fix with targeted regression coverage, visible exact-head checks green, and no actionable review findings.

## What This Changes

The PR adds optional reconciled task snapshot parameters to the task registry/audit summary helpers, updates `getStatusSummary()` to compute one snapshot, and adds tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow per-status-call snapshot fix while preserving zero-argument helper behavior for non-status consumers.

## Reproduction Assessment

Yes. The source-level reproduction on current main is one `getStatusSummary()` call reaching two helpers, each of which independently calls `reconcileInspectableTasks()` and therefore repeats the `listTaskRecords()` clone/sort path.

## Solution Assessment

Yes. Passing a per-call reconciled snapshot is narrower than a module-level cache and avoids invalidation or staleness risk while still removing the redundant status work.
