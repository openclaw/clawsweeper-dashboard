## Summary

Keep open: current main still performs duplicate task reconciliation in the status summary path, and the PR is a narrow additive fix with no blocking review findings found in the diff.

## What This Changes

The PR adds an optional reconciled task snapshot parameter to the task registry and audit summary helpers, shares one snapshot inside `getStatusSummary()`, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent status-only shared-snapshot fix after exact-head CI, preserving the zero-argument helper fallback for other consumers.

## Reproduction Assessment

Yes. Source inspection on current main shows one `getStatusSummary()` call reaches two helpers that each call `reconcileInspectableTasks()`, which walks the clone/sort-heavy `listTaskRecords()` path.

## Solution Assessment

Yes. Passing a per-status-call snapshot is narrower than a module-level cache and avoids invalidation risk while keeping existing zero-argument helper behavior intact.
