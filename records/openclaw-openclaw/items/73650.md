## What This Changes

The PR makes `getStatusSummary()` reconcile inspectable tasks once, passes that snapshot to the registry and audit summaries, and adds focused regression coverage plus a changelog entry.

## Best Possible Solution

## Solution Assessment

Yes. Passing one per-status-call snapshot is the narrowest maintainable fix for this hot path and avoids adding a module-level cache with invalidation policy risk.
