## What This Changes

The branch adds optional reconciled-task snapshot parameters to the task registry and audit summary helpers, updates `getStatusSummary()` to share one snapshot, and adds regression tests plus a changelog entry for the status performance fix.

## Best Possible Solution

Land this PR or an equivalent narrow main-branch fix after exact-head validation is green: status should reconcile inspectable tasks once per summary build, derive registry and audit summaries from that snapshot, and keep zero-argument helper behavior for other command and gateway call sites.

## Reproduction Assessment

Yes. The current-main reproduction is source-level: one `getStatusSummary()` call reaches both task summary helpers, and each helper independently calls `reconcileInspectableTasks()` and then `listTaskRecords()` clone/sort work.

## Solution Assessment

Yes. Sharing a per-status-call snapshot is the narrowest maintainable fix; it removes the duplicated work without introducing module-level cache invalidation or stale-reference behavior.
