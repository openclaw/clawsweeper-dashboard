## What This Changes

The PR adds an optional reconciled task snapshot parameter to the registry and audit summary helpers, shares one snapshot in `getStatusSummary()`, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent status-only shared-snapshot fix after exact-head CI completes, while preserving the zero-argument helper fallback for other consumers.

## Reproduction Assessment

Yes. Source inspection on current main shows one `getStatusSummary()` call reaches two helpers that each call `reconcileInspectableTasks()`, which in turn walks the clone/sort-heavy `listTaskRecords()` path.

## Solution Assessment

Yes. Passing a per-status-call snapshot is narrower than adding a module-level cache and avoids invalidation risk while preserving existing zero-argument helper behavior.
