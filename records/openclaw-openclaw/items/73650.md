## Summary

Keep open for maintainer merge review: current main still duplicates task reconciliation in the status path, and this PR fixes the linked performance bug with a narrow shared-snapshot change plus focused regression coverage.

## What This Changes

The PR adds an optional pre-reconciled task snapshot to the task registry and audit summary helpers, uses it in `getStatusSummary()`, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent status-only shared-snapshot fix after maintainer review, while preserving the zero-argument helper fallback for other consumers.

## Reproduction Assessment

Yes. Source inspection on current main shows one `getStatusSummary()` call reaches two helpers that each invoke `reconcileInspectableTasks()` over the `listTaskRecords()` clone/sort path.

## Solution Assessment

Yes. Passing a per-status-call snapshot is narrower than a module-level cache and avoids invalidation risk while preserving current helper behavior for other call sites.
