## Summary

Keep open: current main has Codex-owned Computer Use MCP setup and existing provider plugin hooks, but it does not implement the requested native provider Computer Use tool-type negotiation, tight screenshot/action loop, or host callback abstraction; this is a large product/API/security design item rather than conservative cleanup-close material.

## What This Changes

The issue requests a cross-provider Computer Use mode that injects native provider CU tools, normalizes action/result loops, and delegates desktop actions through a host callback protocol.

## Best Possible Solution

Design a narrow plugin/provider-runtime seam for native Computer Use tool types and host execution callbacks, then implement provider-specific adapters behind explicit opt-in config only if maintainers approve the security and upgrade model.
