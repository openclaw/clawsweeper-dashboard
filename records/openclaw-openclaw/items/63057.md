## Summary

Keep open: the PR targets a real Anthropic transport bug, but the branch is not ready because the new `> 0` guard still permits provider-invalid positive thinking budgets and the external contributor has not supplied real behavior proof.

## What This Changes

The PR changes Anthropic legacy thinking payload construction to use a nullish fallback for `thinkingBudgetTokens` and omit the manual `thinking` block when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport guard/helper that emits legacy manual thinking only when the resolved budget satisfies the provider contract, preserves adaptive thinking, and covers zero, sub-minimum, default, and valid-budget paths in focused transport tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` model with `model.maxTokens` around 1500 and `reasoning: "high"` resolves `thinkingBudgetTokens` to 476, and this PR would still send that invalid legacy thinking budget.

## Solution Assessment

No. Preserving an explicit zero is useful, but `budgetTokens > 0` is too weak; the narrow maintainable fix is to skip or reject legacy manual thinking budgets that do not satisfy Anthropic's documented constraints.
