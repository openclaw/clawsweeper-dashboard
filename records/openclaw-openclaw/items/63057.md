## Summary

Keep open. The PR targets a real Anthropic transport edge that is still present on current main, but the branch is not merge-ready because its `> 0` guard still allows provider-invalid positive budgets and the PR has no after-fix real behavior proof.

## What This Changes

The PR changes Anthropic transport request construction so legacy manual thinking uses a nullish budget fallback and omits the thinking block when the resolved budget is zero.

## Best Possible Solution

Emit legacy manual Anthropic thinking only when the resolved budget satisfies the provider contract, preserve adaptive thinking paths, and cover zero, undefined, sub-minimum, and valid-budget cases with focused transport tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` reasoning model capped around 1500 tokens resolves `thinkingBudgetTokens` to 476, and this PR would still serialize that provider-invalid manual thinking block.

## Solution Assessment

No. Nullish coalescing and omitting zero are the right direction, but the guard must also enforce Anthropic's manual-budget range and the branch needs regression tests before merge.
