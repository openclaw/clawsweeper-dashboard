## Summary

Keep open: the branch targets a real Anthropic transport bug, but it is not merge-ready because the guard still allows positive manual thinking budgets below Anthropic's 1,024-token minimum and the PR has no after-fix real behavior proof.

## What This Changes

The PR changes Anthropic legacy thinking payload construction to use a nullish fallback for `thinkingBudgetTokens` and omit the `thinking` block when the resolved budget is zero.

## Best Possible Solution

Add a small Anthropic transport validity guard/helper that emits legacy manual thinking only for provider-valid budgets, preserves adaptive thinking behavior, and covers zero, sub-minimum, default, and valid-budget cases in colocated tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` model with `model.maxTokens` around 1500 and `reasoning: "high"` resolves `thinkingBudgetTokens` to 476, and this PR would still send that invalid budget.

## Solution Assessment

No. Preserving explicit zero is useful, but `budgetTokens > 0` is too weak; the narrow maintainable fix is to skip or reject legacy manual thinking budgets that do not satisfy Anthropic's documented token-budget constraints.
