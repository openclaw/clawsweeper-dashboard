## Summary

Keep this PR open: it targets a real Anthropic transport bug, but the proposed guard is incomplete because it still permits positive manual thinking budgets below Anthropic's valid range, and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes Anthropic legacy thinking request construction to use a nullish fallback for `thinkingBudgetTokens` and omit the manual `thinking` block when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that emits legacy manual thinking only when the resolved budget satisfies Anthropic's constraints, preserves adaptive thinking behavior, and adds targeted zero, sub-minimum, default, and valid-budget tests.

## Reproduction Assessment

Yes, by source inspection. A non-adaptive `anthropic-messages` model with `model.maxTokens` around 1500 and `reasoning: "high"` resolves `thinkingBudgetTokens` to 476, and this PR would still send that provider-invalid manual thinking budget.

## Solution Assessment

No. Preserving explicit zero is useful, but `budgetTokens > 0` is too weak; the fix should gate manual Anthropic thinking on the documented minimum and max-token relationship, or omit thinking when a valid manual budget cannot be formed.
