## Summary

Keep open: the zero-budget fix is pointed at a real Anthropic transport bug, but the patch still permits provider-invalid positive manual-thinking budgets and lacks contributor-supplied real behavior proof before merge.

## What This Changes

The branch changes `src/agents/anthropic-transport-stream.ts` to use a nullish fallback for legacy Anthropic thinking budgets and omit the manual `thinking` block when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that emits legacy manual thinking only for provider-valid budgets, preserves adaptive thinking, and adds focused tests for zero, undefined fallback, sub-minimum, and valid-budget cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` model capped around 1500 tokens with `reasoning: "high"` resolves `thinkingBudgetTokens` to 476, and this PR would still send that invalid positive budget.

## Solution Assessment

No. Preserving explicit zero is useful, but `budgetTokens > 0` is too weak; the maintainable fix should gate legacy manual thinking on Anthropic-valid budget constraints and cover those paths with tests.
