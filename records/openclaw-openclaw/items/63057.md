## Summary

Keep open. The PR addresses a real Anthropic transport payload bug still present on current main, but the branch is not mergeable because its `> 0` guard still allows provider-invalid positive budgets and it lacks focused tests plus after-fix real behavior proof.

## What This Changes

The PR changes `src/agents/anthropic-transport-stream.ts` to use a nullish fallback for legacy Anthropic thinking budgets and omit the manual `thinking` block when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that emits legacy manual thinking only when the resolved budget satisfies the provider contract, preserves adaptive thinking, and adds focused tests for zero, undefined fallback, sub-minimum, and valid-budget cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` model capped around 1500 tokens with `reasoning: "high"` resolves `thinkingBudgetTokens` to 476, and this PR would still send that provider-invalid positive budget.

## Solution Assessment

No. Preserving explicit zero is the right direction, but `budgetTokens > 0` is too weak; the maintainable fix should gate legacy manual thinking on Anthropic-valid budget constraints and cover the edge cases with tests.
