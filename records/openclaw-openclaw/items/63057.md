## Summary

Keep this PR open: it targets a real Anthropic transport bug, but the proposed guard is incomplete because it still allows positive provider-invalid budgets, and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes `src/agents/anthropic-transport-stream.ts` so legacy Anthropic manual-thinking payloads use `?? 1024` and omit `thinking` when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that emits legacy manual thinking only when the resolved budget satisfies the provider contract, preserves adaptive thinking, and adds focused tests for zero, undefined fallback, sub-minimum, and valid-budget cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` model capped around 1500 tokens with `reasoning: "high"` resolves a 476-token budget, and this PR would still send that provider-invalid manual `thinking` block.

## Solution Assessment

No. Preserving explicit zero is the right direction, but `budgetTokens > 0` is too weak; the fix should gate on Anthropic-valid budget constraints and add targeted regression coverage.
