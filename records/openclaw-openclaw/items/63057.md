## Summary

Keep open: the PR addresses a real Anthropic zero-budget fallback bug, but the branch is still incomplete because its `budgetTokens > 0` guard would still send provider-invalid positive budgets below Anthropic's minimum, it adds no focused regression coverage, and the external contributor has not posted after-fix real behavior proof.

## What This Changes

The PR changes Anthropic transport request construction so legacy manual thinking uses a nullish budget fallback and omits `thinking` when the resolved budget is zero.

## Best Possible Solution

Update the Anthropic transport to emit legacy manual thinking only when the resolved budget satisfies the provider contract, preserve adaptive thinking and Kimi-compatible valid manual thinking, and add focused transport tests for zero, undefined fallback, sub-minimum, and valid budgets.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` model capped around 1500 tokens with `reasoning: "high"` resolves `thinkingBudgetTokens` to 476, and this PR would still serialize that provider-invalid manual `thinking` block.

## Solution Assessment

No. Nullish coalescing and omitting zero are the right direction, but `budgetTokens > 0` is weaker than Anthropic's manual thinking contract; the safer fix is to skip manual thinking unless the budget is provider-valid and to cover the edge cases in tests.
