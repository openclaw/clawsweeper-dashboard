## Summary

Keep open: this is a plausible Anthropic transport bug fix, but the current PR still permits provider-invalid positive manual thinking budgets below 1,024 tokens and lacks contributor-supplied real behavior proof.

## What This Changes

The PR changes legacy Anthropic thinking payload construction to use a nullish fallback for `thinkingBudgetTokens` and omit the manual `thinking` block when the resolved budget is zero.

## Best Possible Solution

Emit legacy manual thinking only when the resolved budget is provider-valid, preserve adaptive thinking behavior, and add focused tests for zero, undefined fallback, sub-minimum, and valid-budget cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-adaptive `anthropic-messages` model with `model.maxTokens` around 1500 and `reasoning: "high"` resolves `thinkingBudgetTokens` to 476, and this PR would still send that invalid positive budget.

## Solution Assessment

No. Preserving explicit zero is useful, but `budgetTokens > 0` is too weak; the narrow maintainable fix is to skip or reject legacy manual thinking budgets that do not satisfy Anthropic's documented token-budget constraints.
