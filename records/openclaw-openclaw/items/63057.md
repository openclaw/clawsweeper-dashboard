## What This Changes

The branch updates Anthropic legacy thinking payload construction to use a nullish 1024 fallback and omit the manual thinking block when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that only emits legacy enabled thinking when the resolved budget satisfies provider constraints, preserves adaptive and valid Kimi/manual thinking paths, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: a non-adaptive reasoning model with `maxTokens` around 1500 resolves `thinkingBudgetTokens` to 476, and this PR's `> 0` guard would still emit that invalid budget.

## Solution Assessment

No. The nullish fallback and zero omission are useful, but the guard should require a provider-valid budget and tests for zero, undefined fallback, positive sub-minimum, and valid-budget cases.
