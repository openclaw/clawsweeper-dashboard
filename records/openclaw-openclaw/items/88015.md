## Summary

Keep open: current main and latest release still have the session-delivery replay gap, and the open linked PR is the focused fix candidate rather than a reason to close early.

## What This Changes

The issue reports that session-delivery crash recovery can blindly replay an unacked restart-continuation `agentTurn`, re-running the turn and duplicating the reply.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88885, or an equivalent focused recovery-state fix, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enqueue an `agentTurn`, let `deliver` succeed without completing `ackSessionDelivery`, then run recovery again and the same pending entry is delivered again.

## Solution Assessment

Yes. The marker-and-refuse approach in the linked PR is the narrow maintainable path because it reuses the shared SQLite `recovery_state` column and matches the outbound queue's no-blind-replay contract, subject to maintainer acceptance of the fail-safe tradeoff.
