## Summary

Keep open: current main and v2026.6.9 still have the deliver-then-ack session recovery path with no session-level recovery marker, and the open linked pull request is the focused fix candidate rather than a reason to close before merge.

## What This Changes

The issue reports that crash recovery for unacked session-delivery agent turns can replay a turn after it already ran but before the queue entry was acknowledged, causing duplicate execution and duplicate replies.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88885, or an equivalent focused recovery-state fix, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enqueue an agentTurn, let deliver succeed without completing ackSessionDelivery, then run recovery again and the same pending entry is delivered again.

## Solution Assessment

Yes for the direction, but not complete yet. Reusing the shared SQLite recovery_state column is the narrow maintainable fix shape; the exact fail-safe semantics need maintainer acceptance through the linked PR before the issue can close.
