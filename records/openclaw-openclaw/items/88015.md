## Summary

Keep open: current main and the latest release still replay session-delivery entries without a durable send marker, and the open closing PR already owns the focused fix path.

## What This Changes

The issue reports that session-delivery crash recovery can replay an unacked `agentTurn` after delivery succeeds but ack is skipped, causing duplicate turn execution and duplicate replies.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88885, or an equivalent focused recovery-state fix, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enqueue an `agentTurn`, let `deliver` succeed without completing `ackSessionDelivery`, then run recovery again and the same pending entry is delivered again.

## Solution Assessment

Yes. The marker-and-refuse approach is the narrow maintainable fix because it reuses the existing shared SQLite `recovery_state` column and matches the outbound queue's no-blind-replay contract.
