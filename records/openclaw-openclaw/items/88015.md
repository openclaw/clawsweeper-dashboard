## Summary

Keep open: current main and the latest release still have the session-delivery blind-replay gap, and the open linked PR is the focused fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests a recovery fix so an unacked restart-continuation agent turn is not blindly replayed after a crash once delivery may already have run.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88885, or an equivalent focused recovery-state fix, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enqueue an `agentTurn`, let `deliver` succeed without completing `ackSessionDelivery`, then run recovery again and the same pending entry is delivered again.

## Solution Assessment

Yes, the recovery-state direction is the narrow maintainable fix because it reuses the shared SQLite `recovery_state` column and matches the outbound queue's no-blind-replay contract; the exact fail-safe semantics still need maintainer acceptance.
