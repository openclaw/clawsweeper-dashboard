## Summary

Keep open: current `main` and the latest release still have the session-delivery replay gap, and an open linked PR already owns the proposed fix path. The issue should stay open until that PR lands or maintainers choose a different repair.

## What This Changes

Prevent session-delivery recovery from replaying an unacked `agentTurn` after a crash, which can re-run the turn and send a duplicate reply.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88016, or an equivalent focused fix that adds durable send-attempt state and regression coverage for the session-delivery queue.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: enqueue an `agentTurn`, let `deliver` succeed without completing `ackSessionDelivery`, then run recovery again and the same pending entry is delivered again. I did not run tests because this review is read-only.

## Solution Assessment

Yes, the requested marker-and-refuse path is the narrow maintainable fix because it matches the existing outbound queue's `unknown_after_send` recovery contract. The linked PR should still be reviewed for implementation quality before merge.
