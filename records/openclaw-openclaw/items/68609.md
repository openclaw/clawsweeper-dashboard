## Summary

Keep open. Current main still leaves the pre-prompt compaction precheck silent when the computed route is `fits`; the source proves the observability gap, while the larger cache-ttl/large-context overflow theory still needs instrumentation before any heuristic change.

## What This Changes

Add diagnostics for pre-prompt auto-compaction decisions, especially `route: "fits"`, and investigate whether cache-ttl pruning masks large-context estimates.

## Best Possible Solution

Add low-noise debug logging for every pre-prompt precheck result, then use those diagnostics to decide whether cache-ttl pruning or threshold inputs need a separate behavioral fix.

## Reproduction Assessment

Yes for the observability gap: current main calls the precheck and the `fits` route falls through without a route/estimate log. Unclear for the full long-running Opus/cache-ttl overflow path without a live trace or targeted simulation.

## Solution Assessment

Yes for the immediate fix: all-routes debug logging at the pre-prompt call site is the narrowest maintainable first step. Broader compaction heuristic changes should wait until the new diagnostics prove the cache-ttl interaction.
