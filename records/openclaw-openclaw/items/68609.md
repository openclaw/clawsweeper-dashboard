## Summary

Keep open. Current main still does not log the pre-prompt compaction precheck decision when the computed route is `fits`, so the concrete observability request is unresolved; the larger cache-ttl/large-context reliability theory remains plausible but should be validated after instrumentation rather than closed or changed speculatively.

## What This Changes

Request diagnostic coverage for pre-prompt auto-compaction decisions, especially the silent `fits` route, and investigation of whether cache-ttl pruning can mask token pressure for large-context Opus sessions.

## Best Possible Solution

Add a low-noise debug-level log for every pre-prompt compaction precheck result, including `fits`, with numeric pressure fields and session/model context, then use those diagnostics to decide whether cache-ttl pruning or threshold inputs need a separate behavioral change.

## Reproduction Assessment

Yes for the observability gap: source inspection shows the `fits` route is returned and then falls through without the requested pre-prompt route/estimate log. Unclear for the full long-running Opus/cache-ttl overflow path without a live session trace or targeted simulation harness.

## Solution Assessment

Yes for the immediate fix: all-routes debug logging at the pre-prompt call site is the narrowest maintainable first step. Broader compaction heuristic changes are not yet proven to be the best solution.
