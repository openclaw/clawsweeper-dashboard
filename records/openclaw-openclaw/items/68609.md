Keep open. Current main still has a silent pre-prompt `route: "fits"` path, so the requested observability fix is not implemented; the cache-ttl/large-context reliability hypothesis remains plausible but needs instrumentation or a focused reproduction before changing compaction behavior.

## What This Changes

The issue requests debug visibility for every preemptive compaction precheck decision, especially `route: "fits"`, and raises a possible cache-ttl pruning interaction for large-context Opus sessions.

## Best Possible Solution

Add low-noise debug logging for every pre-prompt compaction precheck result, including `route: "fits"`, then use those diagnostics or a focused harness to decide whether cache-ttl pruning or threshold inputs need a separate behavioral change.

## Solution Assessment
