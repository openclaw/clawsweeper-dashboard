## Summary

Keep open. Current main still has the deterministic pre-prompt observability gap: `route: "fits"` is computed by the precheck but falls through without logging the route or token fields, while the full Opus/cache-ttl overflow hypothesis remains unproven without live logs or a targeted simulation.

## What This Changes

Issue requests reliable preemptive compaction diagnostics, especially debug logging for the `fits` precheck route, and raises a possible cache-ttl masking interaction for large-context models.

## Best Possible Solution

Add low-noise debug logging for every pre-prompt precheck route first, then use those diagnostics to decide whether cache-ttl pruning or threshold inputs need a separate behavioral fix.

## Reproduction Assessment

Yes for the observability gap: source inspection on current main shows the pre-prompt precheck runs and the `fits` route falls through without a route/estimate log. The full long-running Opus/cache-ttl overflow path is still unclear without live logs or a focused simulation.

## Solution Assessment

Yes for the immediate repair: all-route debug logging at the pre-prompt call site is the narrowest maintainable first step. Broader heuristic changes should wait until the diagnostics prove the cache-ttl interaction.
