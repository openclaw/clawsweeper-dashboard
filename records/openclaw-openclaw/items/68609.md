## Summary

Keep open. Current main and the latest release still compute the pre-prompt `route: "fits"` result without logging that decision path, and the cache-ttl masking hypothesis remains plausible but not yet proven by live reproduction.

## What This Changes

The issue requests reliable preemptive compaction behavior and debug observability for fitted precheck routes under large-context cache-ttl sessions.

## Best Possible Solution

Add low-noise debug logging for every pre-prompt precheck route first, then use those diagnostics to decide whether cache-ttl pruning or threshold inputs need a separate behavioral fix.

## Reproduction Assessment

Yes for the observability gap: source inspection on current main and v2026.5.7 shows the pre-prompt precheck runs and the fitted route falls through without route/estimate logging. The full long-running Opus/cache-ttl overflow path remains unclear without live logs or a focused simulation.

## Solution Assessment

Yes for the immediate repair: all-route debug logging at the pre-prompt call site is the narrowest maintainable first step and matches the existing mid-turn pattern. Broader heuristic changes should wait until diagnostics prove the cache-ttl interaction.
