## Summary

Keep this open: current main has first-class command cron payloads and conservative doctor conversion/warnings, but the requested manifest-grade audit/governance metadata is still absent from the cron protocol and list JSON. The open candidate pull request at https://github.com/openclaw/openclaw/pull/92502 uses closing syntax for this issue, but it remains unmerged and has review/proof blockers, so this issue should stay open until that path is reviewed, narrowed, or replaced.

## What This Changes

The issue asks OpenClaw to replace hidden deterministic scheduled agentTurn scripts with explicit command-job support, audit warnings, and manifest-ready cron metadata.

## Best Possible Solution

Define the smallest generic cron audit/governance metadata contract that belongs in OpenClaw core, reuse the existing doctor-owned command-conversion policy for classification, and land it through a focused PR or a narrowed version of https://github.com/openclaw/openclaw/pull/92502.

## Reproduction Assessment

Yes for a source-level feature gap: current main clearly supports command payloads and doctor warnings, while the cron schema/list surfaces lack the requested governance and audit metadata. This is not a failing current-main bug reproduction.

## Solution Assessment

Unclear until maintainers decide the product boundary. The shipped command payload and conservative doctor path are good foundations, but the requested manifest metadata needs a generic core contract rather than support-control-specific fields or a duplicate heuristic.
