## Summary

The shell check shows command cron jobs are already supported, documented, serialized, and tested on current main, but the issue remains necessary because explicit governance metadata and audit warnings for deterministic `agentTurn` jobs are still absent. This is a product-contract extension rather than a bug or a safe autonomous repair.

## What This Changes

Add explicit governance metadata to deterministic cron command jobs, warn when narrow deterministic work is scheduled as an isolated agent turn, and expose the metadata in JSON listings.

## Best Possible Solution

Extend the existing SQLite-backed `command` payload with a minimal maintainer-approved governance contract, expose it through the existing cron list/show JSON schema, and implement audit warnings from explicit structured facts while preserving existing agent-turn jobs unchanged.

## Reproduction Assessment

Yes for source-level gap verification: current main clearly supports command jobs and job status, while exhaustive targeted searches find no requested governance fields or deterministic-agent-turn audit warning. This is not a failing existing-behavior bug.

## Solution Assessment

The existing first-class command payload is the best implementation boundary, but the issue's exact metadata vocabulary and warning classification need maintainer product judgment before implementation.
