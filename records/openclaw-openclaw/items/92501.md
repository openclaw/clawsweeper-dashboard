## Summary

Current main and v2026.6.6 already provide first-class command cron jobs plus conservative doctor conversion and warnings for recognizable shell-style agent turns. However, this issue has a meaningful unique remainder: manifest-grade ownership and governance metadata is not represented in the cron protocol or list JSON, so maintainers should decide that product boundary before implementation.

## What This Changes

Add manifest-grade governance metadata to deterministic cron command jobs and expose audit guidance for remaining deterministic isolated agent turns while preserving existing scheduled agent turns.

## Best Possible Solution

Build on the shipped command payload and doctor warning path, then define the smallest generic automation metadata object needed by multiple core consumers; prefer explicit conversion annotations over broader prompt heuristics and leave support-control-specific policy in its owning repository.

## Reproduction Assessment

Yes for source-level assessment: current schemas, doctor logic, docs, and release history clearly show which requested behavior is shipped and which governance fields remain absent. This is a feature-gap analysis rather than a failing current-main bug reproduction.

## Solution Assessment

Partly yes: the shipped command payload and conservative doctor conversion are the narrowest safe execution and migration design. The proposed governance fields need a maintainer-approved generic contract before they can be considered the best core solution.
