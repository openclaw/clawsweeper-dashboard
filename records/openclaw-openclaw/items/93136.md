## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require referer for per-deployment attribution:** `src/agents/sessions/sdk.ts:189-195`
  - body: OpenRouter uses `HTTP-Referer` as the required app identifier, while `X-OpenRouter-Title` only changes display metadata when paired with that referer. This path lets a title override fall back to the shared `https://openclaw.ai` referer, so the advertised per-agent spend grouping collapses under the same app identity; require a unique referer for this mode or document title-only as cosmetic. ([openrouter.ai](https://openrouter.ai/docs/app-attribution))
  - confidence: 0.9
- **[P2] Validate the referer as a URL before returning it:** `src/agents/sessions/sdk.ts:237-243`
  - body: `sanitizeHttpReferer()` accepts any value with an `http://` or `https://` prefix, including `https://` or strings with embedded control characters. That contradicts the fallback-on-invalid contract and can send invalid header values to provider requests; parse with `new URL`, require an http(s) protocol and host, and reject header control characters before returning the override.
  - confidence: 0.86
