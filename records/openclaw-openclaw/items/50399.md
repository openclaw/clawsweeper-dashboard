## Summary

Keep open: the Feishu duration bug is valid on current main, but this PR is not landable because it is stale against current Feishu media behavior and introduces import-boundary, metadata, routing, duration-coverage, and ffprobe execution regressions.

## What This Changes

The PR branch adds local ffprobe duration probing for Feishu audio/video sends and passes `duration` into Feishu file uploads while rewriting Feishu download metadata and outbound media routing.

## Best Possible Solution

Replace or repair this as a small Feishu-only patch on current main that keeps existing metadata, MIME routing, and `audioAsVoice`, computes duration with the shared async `runFfprobe` helper for prepared Opus/MP4 uploads, and adds focused regression tests plus changelog.

## Reproduction Assessment

Yes by source inspection: current main accepts and forwards `duration` in `uploadFileFeishu`, but `sendMediaFeishu` never supplies it for Opus/MP4 uploads. Feishu’s API docs say omitted audio/video duration prevents the concrete duration from displaying.

## Solution Assessment

No. The PR targets the right Feishu API field, but it rewrites stable metadata/routing code, misses remote/buffer/transcoded media paths, and runs ffprobe synchronously from PATH instead of using the existing trusted async helper.
