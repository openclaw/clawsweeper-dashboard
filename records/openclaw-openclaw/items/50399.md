## Summary

Keep open. This PR targets a real Feishu audio-duration bug, but the branch is not landable against current main because it regresses current media routing/download metadata and only computes duration for one local-path case.

## What This Changes

This PR changes `extensions/feishu/src/media.ts` to probe local audio/video duration with `ffprobe`, pass it to Feishu file upload, and refactor Feishu media routing and download metadata handling.

## Best Possible Solution

Revise or replace this PR with a small Feishu-only patch that preserves current routing/download metadata, computes a finite millisecond duration for native and transcoded Feishu audio/video uploads, and asserts the duration reaches `im.file.create`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Feishu audio/video routing reaches `uploadFileFeishu`, the helper can send `duration`, but current `sendMediaFeishu` does not compute or pass it; I did not run a live Feishu UI send.

## Solution Assessment

No. The PR addresses the right bug, but the implementation replaces current routing and metadata helpers and only handles raw local paths; the safer solution is a narrow additive duration helper around the existing media path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Restore the download metadata helper:** `extensions/feishu/src/media.ts:148`
  - body: `downloadImageFeishu` now returns only `{ buffer }`, and resource downloads use a narrow `response.headers` parser. That drops current metadata sources such as SDK `header`, `data.mime_type`, `filename*`, and recovered UTF-8 filenames, so inbound media can lose content type or filename before `bot-content.ts` saves it.
  - confidence: 0.93
- **[P2] Keep supported MIME routing intact:** `extensions/feishu/src/media.ts:483-487`
  - body: This inline routing maps any `video/*` media to Feishu `mp4`, while MIME-identified Ogg/Opus audio with a generic filename still falls back to `stream`. That regresses current main's centralized routing for supported MIME types and can send unsupported videos as playable Feishu media or playable audio as plain files.
  - confidence: 0.9
- **[P2] Compute duration from the uploaded media bytes:** `extensions/feishu/src/media.ts:494`
  - body: Duration is only probed when the original `mediaUrl` is a non-HTTP local path. `sendMediaFeishu` also uploads `mediaBuffer`, remote URLs, and prepared/transcoded buffers, so those valid Feishu audio/video sends still omit `duration` and keep the reported UI-duration bug.
  - confidence: 0.92
- **[P2] Avoid blocking the gateway while probing duration:** `extensions/feishu/src/media.ts:501-510`
  - body: `spawnSync` runs in the outbound send path and can block the Node event loop for up to the 10s timeout per malformed or slow file. Use an async helper or existing media-runtime process wrapper so one Feishu media send does not stall unrelated gateway work.
  - confidence: 0.85
