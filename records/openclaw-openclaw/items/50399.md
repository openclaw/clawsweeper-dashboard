## Summary

Keep this PR open: the Feishu duration bug is valid and still present on current main, but this branch is not landable because it changes stable Feishu media routing/download behavior and uses a synchronous PATH-resolved ffprobe path.

## What This Changes

The PR changes `extensions/feishu/src/media.ts` to probe Feishu audio/video duration with ffprobe, pass it to uploads/sends, and rewrites media download metadata and outbound routing logic.

## Best Possible Solution

Apply a small Feishu-only fix on current main that preserves metadata extraction, MIME-aware routing, and `audioAsVoice`, then computes duration for proven Opus/MP4 upload buffers through the shared async `runFfprobe` helper with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes, by source inspection: current main routes Opus/MP4 media and `uploadFileFeishu` can forward `duration`, but `sendMediaFeishu` never supplies it. The related user report #53798 describes the visible Feishu voice-duration symptom for the same path.

## Solution Assessment

No. The PR targets the right Feishu field, but the maintainable fix is narrower: keep the existing metadata/routing helpers and add duration probing after media preparation using the shared async ffprobe wrapper.
