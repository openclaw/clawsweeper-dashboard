## Summary

Keep this PR open as useful source work, but it is not landable as-is. Current main still has the Feishu `duration` upload sink without passing duration from `sendMediaFeishu`, while this PR fixes only part of the bug and introduces routing, metadata, import, and ffprobe execution regressions.

## What This Changes

The PR changes Feishu media sending to probe local audio/video duration with ffprobe, pass `duration` to file uploads, and rewrite parts of Feishu download metadata extraction and outbound media routing in `extensions/feishu/src/media.ts`.

## Best Possible Solution

Replace the branch with a small Feishu-only fix on current main that keeps existing metadata extraction, MIME-aware routing, `audioAsVoice`, and local/runtime SDK barrels, then computes duration through the shared async ffprobe helper and passes it only for proven Feishu Opus/MP4 uploads.

## Reproduction Assessment

Yes by source inspection: current main accepts `duration` in `uploadFileFeishu` but `sendMediaFeishu` never supplies it for Opus/MP4 sends. I did not live-run Feishu, so this is source-reproducible rather than a live reproduced result.

## Solution Assessment

No. The PR targets the right Feishu API field, but it is not the best solution because it rewrites stable routing/metadata code, misses URL/buffer/transcoded media paths, and runs ffprobe synchronously through PATH instead of the existing trusted async helper.
