## Summary

Keep open: the Feishu duration bug is valid, but this PR is not landable because it regresses existing Feishu metadata/routing behavior and bypasses the shared ffprobe execution path.

## What This Changes

The PR modifies Feishu media sending to probe local audio/video duration with ffprobe, pass duration to file uploads, and rewrites Feishu download metadata extraction and outbound media routing in `extensions/feishu/src/media.ts`.

## Best Possible Solution

Apply a small Feishu-only fix on current main that preserves metadata extraction, MIME-aware routing, and `audioAsVoice`, then computes and passes duration for proven Opus/MP4 uploads through the shared async ffprobe helper.

## Reproduction Assessment

Yes by source inspection: current main forwards `duration` if supplied, but `sendMediaFeishu` never supplies it for Opus/MP4 uploads. Feishu's own docs state omitted audio/video duration prevents displaying the concrete file duration.

## Solution Assessment

No. The PR targets the correct Feishu API field, but it is not the best fix because it rewrites stable routing/metadata paths, misses buffer/remote/transcoded media duration paths, and uses synchronous PATH-based ffprobe instead of the shared helper.

  - body: This imports `openclaw/plugin-sdk/feishu`, but the root package exports do not include that subpath and the extension boundary expects plugin-owned runtime helpers to go through the local barrel.   - confidence: 0.91

  - body: The replacement only reads lowercase `response.headers` and a plain `filename=` form, while current main also handles `header`, SDK/top-level/data MIME fields, RFC 5987 filenames, and CJK filename recovery. Inbound saving and existing tests rely on those `contentType`/`fileName` values.
  - confidence: 0.92
- **[P2] Preserve MIME-specific Feishu media routing:** `extensions/feishu/src/media.ts:483-487`
  - body: Mapping any `video/*` payload to `mp4` and requiring audio extensions for Opus drops the current centralized MIME-aware routing. CDN-style `audio/ogg` without a useful filename can become a generic file, while non-MP4 videos can be sent through Feishu's MP4 media path.
  - confidence: 0.89
- **[P2] Probe duration after media preparation:** `extensions/feishu/src/media.ts:494-522`
  - body: The duration probe only runs for non-HTTP `mediaUrl` strings, so `mediaBuffer`, remote downloads, and `audioAsVoice` transcode output can still upload Opus/MP4 without `duration`. That leaves supported send paths with the reported missing-duration behavior.
  - confidence: 0.88
- **[P2] Use the shared async ffprobe helper:** `extensions/feishu/src/media.ts:501-509`
  - body: Calling `spawnSync("ffprobe", ...)` in outbound delivery blocks the gateway event loop and resolves through PATH. The repo already exposes `runFfprobe`, which uses trusted system-binary resolution and async `execFile`.
  - confidence: 0.85
