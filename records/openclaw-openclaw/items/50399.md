## Summary

Close as superseded: the remaining Feishu duration bug is already tracked by the open canonical issue, while this PR branch is not landable because it rewrites current metadata/routing behavior and adds blocking ffprobe/runtime-import problems.

## What This Changes

The PR modifies `extensions/feishu/src/media.ts` to probe local Feishu audio/video duration with `ffprobe`, pass that duration to file uploads, and rewrite Feishu download metadata and outbound media routing.

## Best Possible Solution

Keep the canonical duration bug open and land a narrow Feishu patch that preserves current metadata and MIME-aware routing, computes duration from the actual uploaded audio/video bytes, and covers the duration field with focused tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main routes Feishu Ogg/Opus media to `uploadFileFeishu` without computing or passing `duration`. I did not run a live Feishu UI send in this read-only review.

## Solution Assessment

No. The PR targets the right bug, but the implementation is broader than necessary, regresses current metadata/routing behavior, misses remote and buffer-backed media duration paths, and blocks the send path with synchronous probing.

- **[P2] Restore the local Feishu runtime import:** `extensions/feishu/src/media.ts:5`
  - body: `openclaw/plugin-sdk/feishu` is not a current root package export, and extension production files are guarded against importing their own plugin-specific SDK facade. Keep the current local runtime barrel import so the branch can pass contracts and runtime resolution after rebase.
  - confidence: 0.9
- **[P2] Keep image download MIME metadata:** `extensions/feishu/src/media.ts:148`
  - body: `downloadImageFeishu` now returns only `{ buffer }`, dropping the current `contentType` extracted from Feishu response metadata. Callers that save or route downloaded images can silently lose the MIME hint and fall back to weaker detection.
  - confidence: 0.88
- **[P2] Restore the full resource metadata parser:** `extensions/feishu/src/media.ts:180-191`
  - body: The replacement parser only reads lowercase `response.headers` and plain `filename=`. Current main also handles header aliases, SDK/data fallbacks, `filename*`, and UTF-8 recovery, so inbound resources can lose MIME types and non-ASCII filenames.
  - confidence: 0.9
- **[P2] Keep outbound routing on the MIME-aware helper:** `extensions/feishu/src/media.ts:481-487`
  - body: The inline routing maps every `video/*` response to Feishu `mp4` and maps MIME-identified Ogg/Opus audio with a generic filename to `stream`. That regresses current main's centralized MIME/extension contract and can send supported media with the wrong Feishu message type.
  - confidence: 0.88
- **[P2] Compute duration for every uploaded media byte source:** `extensions/feishu/src/media.ts:494`
  - body: Duration probing only runs for a non-HTTP `mediaUrl`, but `sendMediaFeishu` also supports remote URLs, buffers, and voice-intent transcoded buffers. Those supported audio/video paths would still upload without `duration`, leaving the reported symptom unfixed there.
  - confidence: 0.86
- **[P2] Avoid blocking ffprobe on the gateway send path:** `extensions/feishu/src/media.ts:501`
  - body: `spawnSync("ffprobe", ...)` blocks the Node event loop until the process exits or times out. A slow or malformed caller-selected file can stall unrelated gateway message handling during outbound Feishu sends.
  - confidence: 0.84
