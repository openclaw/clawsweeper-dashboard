## Summary

Keep open. The PR targets a real Feishu duration bug that is still present on current main and is also tracked by https://github.com/openclaw/openclaw/issues/53798, but this branch is not landable because it is stale against the current Feishu media path and would regress routing, download metadata, voice-intent transcoding, and local file safety.

## What This Changes

The PR changes `extensions/feishu/src/media.ts` to add ffprobe-based duration detection for local audio/video uploads while refactoring Feishu media routing and download metadata handling.

## Best Possible Solution

Replace this branch with a narrow Feishu-only patch that keeps current `resolveFeishuOutboundMediaKind`, `extractFeishuDownloadMetadata`, safe file-read, and `audioAsVoice` paths, then adds safe finite duration extraction and focused tests proving `client.im.file.create` receives duration for native audio/video uploads.

## Reproduction Assessment

Yes. Source inspection shows current main routes native Feishu `.opus`/`.ogg` audio through `sendMediaFeishu`, but the final `uploadFileFeishu` call omits `duration`; I did not perform a live Feishu UI send.

## Solution Assessment

No. The branch addresses the right bug but is not the best merge shape because it is local-path-only and regresses current media routing, metadata extraction, safe file reads, and voice-intent transcoding; a smaller patch on current main is safer.
