## Summary

Keep open: the Feishu duration bug is real and source-reproducible, but this PR is not landable because it conflicts with current main, weakens existing Feishu media metadata/routing behavior, and lacks sufficient OpenClaw after-fix real behavior proof.

## What This Changes

The PR changes Feishu media sending to probe local audio/video duration with ffprobe and pass that value to Feishu file uploads.

## Best Possible Solution

Land a narrow current-main Feishu media fix, or a replacement PR, that computes finite duration from the actual uploaded native or transcoded audio/video bytes while preserving current metadata extraction and MIME-aware routing.

## Reproduction Assessment

Yes. Source inspection follows message-tool media/asVoice through Feishu action handling into `sendMediaFeishu`, where Opus/Ogg audio reaches `uploadFileFeishu` without `duration`; the Feishu API contract explains why that omits the visible duration.

## Solution Assessment

No. The requested bug fix belongs in the Feishu plugin, but this PR refactors established routing/metadata code, only handles local-path duration probing, and blocks the event loop with synchronous ffprobe.

  - confidence: 0.9
- **[P2] Keep routing on the MIME-aware helper:** `extensions/feishu/src/media.ts:483-487`
  - body: Replacing `resolveFeishuOutboundMediaKind` with inline checks changes the MIME contract: generic `audio/opus` URLs without an `.opus`/`.ogg` filename fall back to `stream`, while every `video/*` becomes Feishu `mp4`. That can send playable voice notes as files or unsupported videos as MP4 media.
  - confidence: 0.88
- **[P2] Compute duration for the uploaded media bytes:** `extensions/feishu/src/media.ts:494`
  - body: The duration code only runs for a non-HTTP `mediaUrl`, but `sendMediaFeishu` also supports remote URLs and `mediaBuffer`. Those supported audio/video paths still upload without duration, so the reported missing-duration behavior remains outside local-path sends.
  - confidence: 0.87
- **[P2] Move ffprobe off the synchronous send path:** `extensions/feishu/src/media.ts:501-509`
  - body: Calling `spawnSync("ffprobe", ...)` inside `sendMediaFeishu` blocks the Node event loop until the process exits or times out. A slow or malformed local file can stall unrelated gateway message handling, so duration probing should use an async bounded helper or existing media runtime path.
  - confidence: 0.84
