## Summary

Current main and the latest release already contain the guarded Control UI session-picker blur behavior and regression coverage for the reported no-switch/dropdown-flash path, so this duplicate can close as implemented.

## What This Changes

The issue reports that the Control UI chat session picker stopped switching to non-default sessions in v2026.5.26-v2026.5.27 until the picker search re-rendered.

## Best Possible Solution

Keep the shipped guarded blur implementation and treat the broader isolated-cron portion of https://github.com/openclaw/openclaw/issues/88208 separately from this now-fixed picker regression.

## Reproduction Assessment

Yes. Source comparison shows the old unconditional blur handler could clear the picker before click activation, and current source plus tests cover the empty-blur then click-another-session path.

## Solution Assessment

Yes. The implemented fix is the narrowest maintainable solution for this symptom because it skips only destructive empty-query blur work while preserving explicit search and non-empty blur behavior.
