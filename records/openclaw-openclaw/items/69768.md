## Summary

Keep open. Current main still preserves JSON Schema conditional keywords on the Google/Gemini tool-schema path, and open PR #70497 is already the focused source/test fix path.

## What This Changes

The issue reports Gemini Flash HTTP 400 failures for tool-heavy cron jobs and asks whether OpenClaw should fix tool-schema serialization or document safer cron guidance.

## Best Possible Solution

Land #70497 or an equivalent focused patch that strips `if`, `then`, and `else` from Gemini tool schemas with regression coverage; handle any broader cron guidance separately.

## Reproduction Assessment

Yes. Pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })` and current main preserves those keys.

## Solution Assessment

Yes. A narrow provider-boundary schema sanitization fix is the best way to address the reported 400s; docs-only guidance or failover-only changes would still send invalid schemas to Gemini.
