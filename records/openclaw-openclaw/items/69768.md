## Summary

Keep open. Current main still preserves JSON Schema conditional keywords on the Google/Gemini tool-schema path, and #70497 is already open as the focused source/test fix rather than closing before that work lands.

## What This Changes

The issue reports Gemini Flash cron tool-call failures from HTTP 400 responses on complex tool schemas and asks whether OpenClaw should fix schema serialization or document operational guidance.

## Best Possible Solution

Land #70497 or an equivalent focused patch that strips `if`, `then`, and `else` from Gemini tool schemas with regression coverage, leaving broader cron docs or failover policy changes to separate maintainer follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })` and observe that current main preserves those keys.

## Solution Assessment

Yes. The narrow provider-boundary sanitization fix is the best way to solve the reported schema rejection; docs-only guidance or failover-only changes would still send invalid tool schemas to Gemini.
