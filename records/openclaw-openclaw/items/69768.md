## Summary

Keep open. Current main and the latest release still preserve `if`, `then`, and `else` on the Google/Gemini tool-schema path, while the focused fix is still an open PR rather than merged code.

## What This Changes

The issue reports Gemini Flash returning HTTP 400/no-body for tool-heavy cron payloads because conditional JSON Schema keywords can pass through Gemini tool-schema cleanup.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/70497 with a focused Gemini cleaner patch and regression coverage; treat cron docs or failure-notification guidance as separate follow-up if maintainers still want it.

## Reproduction Assessment

Yes for a source-level reproduction: pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })` on current main and those keys remain. I did not live-replay the provider 400.

## Solution Assessment

Yes. The narrowest maintainable fix is to strip the three conditional JSON Schema keywords at the Gemini provider-boundary sanitizer; docs-only guidance or failover-only changes would still send invalid tool schemas.
