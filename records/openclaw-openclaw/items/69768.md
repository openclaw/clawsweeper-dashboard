## What This Changes

The issue asks OpenClaw to address Gemini Flash cron failures on tool-heavy payloads by stripping unsupported conditional JSON Schema keywords and clarifying cron failure visibility guidance.

## Best Possible Solution

Land a refreshed version of https://github.com/openclaw/openclaw/pull/70497 containing only the Gemini sanitizer change and regression coverage, then handle any broader cron production guidance separately.

## Reproduction Assessment

Yes for a source-level path: pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })` and those keys remain on current main. I did not live-replay the provider HTTP 400.

## Solution Assessment

Yes. The narrow maintainable fix is to strip the three conditional JSON Schema keywords at the Gemini provider-boundary sanitizer; docs-only or failover-only changes would still allow invalid tool schemas through.
