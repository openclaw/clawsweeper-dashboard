## Summary

Keep open. Current main still lets JSON Schema conditional keywords reach Gemini tool schemas, and the already-open focused fix PR remains the right maintainer path rather than closing this issue now.

## What This Changes

The issue reports Gemini Flash failures for tool-heavy cron jobs and asks whether OpenClaw should fix schema serialization, document avoidance guidance, or improve cron failure-visibility guidance.

## Best Possible Solution

Land #70497 or an equivalent focused patch that adds `if`, `then`, and `else` to the Gemini unsupported-keyword cleaner with regression coverage, while leaving broader cron failover policy or docs changes to separately reproduced follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })`; current main preserves those keys because they are not in the unsupported-keyword set.

## Solution Assessment

Yes. The narrow provider-boundary sanitization fix is the best way to solve the reported schema rejection; docs-only guidance or failover-only changes would still send invalid tool schemas to Gemini.
