## Summary

Keep open. Current main and the latest release still omit `if`, `then`, and `else` from the Gemini schema cleaner, while the focused fix remains in an open dirty PR that still needs proof/cleanup.

## What This Changes

The issue reports Gemini Flash cron jobs failing on tool-heavy payloads because unsupported conditional JSON Schema keywords can reach the Gemini tool-schema path, with follow-up questions about docs and failure notifications.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/70497 with only the Gemini sanitizer change and regression coverage; handle cron failure-notification guidance separately if maintainers want a docs follow-up.

## Reproduction Assessment

Yes for a source-level path: pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })` and those keys remain on current main. I did not live-replay the provider HTTP 400.

## Solution Assessment

Yes. The narrowest maintainable fix is to strip the three conditional JSON Schema keywords at the Gemini provider-boundary sanitizer; docs-only or failover-only changes would still allow invalid tool schemas through.
