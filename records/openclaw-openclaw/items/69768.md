## Summary

Keep open. Current main and the latest release still omit `if`, `then`, and `else` from the Gemini schema cleaner, and the focused fix remains in the open, unmerged https://github.com/openclaw/openclaw/pull/70497.

## What This Changes

The issue reports Gemini Flash cron runs failing on tool-heavy payloads and asks whether OpenClaw should sanitize the schema path or document safer cron model guidance.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/70497 with a focused Gemini sanitizer patch and regression test, leaving any cron failure-notification guidance as a separate follow-up.

## Reproduction Assessment

Yes, at source level: pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })` on current main and those keys are not stripped. I did not live-replay the provider HTTP 400.

## Solution Assessment

Yes. The narrow maintainable fix is to strip the three conditional JSON Schema keywords at the Gemini provider-boundary sanitizer and cover it with a regression test; docs-only or failover-only changes would still send invalid schemas.
