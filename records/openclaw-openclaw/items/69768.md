## Summary

Keep open. Current main still lets `if`, `then`, and `else` pass through the Gemini tool-schema cleaner, and the focused fix remains in an open conflicting PR rather than landed on main.

## What This Changes

The issue reports Gemini Flash cron jobs with tool-heavy schemas failing because conditional JSON Schema keywords can pass through the Gemini sanitizer and trigger provider HTTP 400 errors.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/70497 with only the Gemini sanitizer change and regression coverage, leaving broader cron guidance as a separate docs decision.

## Reproduction Assessment

Yes for a high-confidence source path: pass a schema containing `if`, `then`, or `else` through `cleanSchemaForGemini` or `normalizeToolParameterSchema({ modelProvider: "google" })` and those keys remain on current main. I did not live-replay the provider HTTP 400.

## Solution Assessment

Yes. The narrow maintainable fix is to strip the three conditional JSON Schema keywords at the Gemini provider-boundary sanitizer; docs-only or failover-only changes would still allow invalid tool schemas through.
