## Summary

Keep open. Current main now fixes the narrow New Chat create-and-switch path, but it still does not provide the requested native Control UI conversation-history sidebar with chronological rows, search in the Chat flow, inline rename/context actions, and defined archive behavior.

## What This Changes

Add a ChatGPT-style Control UI session sidebar for persistent chat history creation, switching, search, rename, delete/archive, and cross-refresh persistence.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Maintainers should choose the canonical tracker, review #61176 as the current sidebar candidate, and land a scoped sidebar/search/rename/delete-or-archive design against the existing session APIs with focused UI/controller tests and docs once behavior ships.

Not applicable as a runtime bug reproduction; this is a feature request. The current-main gap is high-confidence from static inspection: Chat still uses a header session select, Sessions remains a separate table, and the native conversation-history sidebar is absent.

## Solution Assessment
