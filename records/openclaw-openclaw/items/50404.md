## Summary

Keep open. Current main now fixes the narrow New Chat create-and-switch path, but it still lacks the requested native Control UI conversation-history sidebar with chronological rows, in-chat search, inline rename/context actions, and defined archive behavior.

## What This Changes

The issue requests a ChatGPT-style Control UI session sidebar for saving, listing, searching, renaming, deleting or archiving, and switching between persistent chat sessions.

## Best Possible Solution

Keep a native Control UI conversation-management track open, choose the canonical tracker/PR, and ship a scoped sidebar/search/rename/delete-or-archive design on top of the existing session APIs with focused UI/controller tests and docs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. The current-main gap is high-confidence from source inspection: Chat still uses a header session select, the left sidebar only contains the Chat tab, and Sessions management remains a separate table.

## Solution Assessment

Unclear as one bundled change. The merged New Chat create/switch fix is a good narrow slice, but the full sidebar, search, rename, delete/archive, and mobile UX need maintainer product scoping before an automated repair should attempt it.
