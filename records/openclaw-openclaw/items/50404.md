## Summary

Keep open. Current main has Gateway session primitives, a Chat header session dropdown, and a separate Sessions table, but it still lacks the requested native Control UI conversation-history sidebar with chronological rows, search, inline rename/context actions, archive semantics, and ChatGPT-style switching. The related workaround comments are useful context, and open PR #73725 addresses only the narrower New Chat create-and-switch slice.

## What This Changes

The issue requests a ChatGPT-style Control UI session sidebar for listing saved conversations, creating new sessions, switching between sessions, renaming/deleting/archiving sessions, and persisting history across refreshes and remote use.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and land or close #73725 for the immediate create-and-switch behavior, then designate a canonical sidebar tracker and implement the remaining sidebar/search/rename/delete/archive UX against `sessions.list`, `sessions.create`, `sessions.patch`, and `sessions.delete`, with docs updated once the behavior ships.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current gap is high-confidence from static inspection: Chat is a single sidebar tab, session switching is a header `<select>`, and the New Session button still sends `/new`.

## Solution Assessment

Unclear as one bundled change. The maintainable path is to split the immediate create/switch fix from the larger sidebar/search/rename/delete/archive workflow, because backend primitives mostly exist but the UX and archive semantics still need maintainer scoping.
