## Summary

Keep open for revision: the PR targets a real Slack feature gap, but the patch duplicates the shared tool display registry and can leave stale tool-specific statuses for unmapped tools; live Slack proof is also still missing.

## What This Changes

The PR adds a Slack-local tool-name-to-status map and calls `ctx.setSlackThreadStatus` from Slack `onToolStart` when a thread status is active.

## Best Possible Solution

Wire Slack status updates through the existing shared progress/tool display model with fallback text, add focused dispatch coverage for mapped and unknown tools, then provide redacted live Slack proof of the status transition and clear.

## Reproduction Assessment

Yes, from source inspection: current main and v2026.6.8 set Slack assistant thread status to `is typing...` and the Slack `onToolStart` handler does not update that status text. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

No. Wiring `onToolStart` is the right area, but a Slack-only map is not the best fix when the shared tool display/progress model already provides labels and fallbacks.
