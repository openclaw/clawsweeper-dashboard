## Summary

Keep this PR open: it fixes a real Codex media-provider omission on current main, but the patch should normalize empty `agentDir` values before forwarding them into the app-server auth bridge so optional callers keep the previous default-agent fallback.

## What This Changes

The PR forwards `req.agentDir` into Codex media app-server client creation and adds a regression test plus an active changelog entry.

## Best Possible Solution

Forward only non-empty request agent directories through the existing Codex app-server APIs, preserve default fallback behavior for omitted values, and keep the focused regression test and changelog entry.

## Reproduction Assessment

Yes. The original bug is reproducible by observing current main create a Codex media app-server client with `profile` but without the non-empty request `agentDir`; the added empty-agentDir review finding is reproducible by calling the optional runtime path without `agentDir`.

## Solution Assessment

No as written. Forwarding `agentDir` through the existing factory and isolated-client APIs is the right shape, but the implementation should normalize empty strings to `undefined` before crossing the auth/client boundary.
