## Summary

Keep open. Current main still builds Telegram's model-visible recent context from the Telegram account/chat message cache, while Gateway chat turns persist through the shared session transcript path; the linked fix PR is still open and not merged.

## What This Changes

The issue asks for cross-transport replies in one shared session to use session-scoped recent context instead of a per-transport Telegram cache.

## Best Possible Solution
