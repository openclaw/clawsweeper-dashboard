## Summary

Keep open. Current main still builds Telegram's model-visible recent conversation context from a Telegram account/chat cache, while Gateway/iOS turns are written through the shared session path and do not enter that cache; no merged related item currently owns this exact cross-ingress stale-context bug.

## What This Changes

Request that recent model context for one shared agent session be assembled across ingresses so a Telegram reply can see the newest iOS/Gateway turn.

## Best Possible Solution

Add a session-scoped recent-context source for shared direct sessions, backed by canonical session transcript/state, while preserving group/topic isolation and Telegram reply-target semantics.

## Reproduction Assessment

Yes at source level: Telegram chat_window context is filtered by Telegram account/chat/message cache, and Gateway chat.send turns do not write that cache. The exact stale tool execution was not replayed live in this read-only review.

## Solution Assessment

Yes in direction: the durable fix should read bounded recent context from the canonical shared session transcript/state, not merge ad hoc per-ingress caches or add a config knob.
