## Summary

Keep open. The requested Gateway capability is plausible and the PR includes live proof, but the current merge result still drops required abort state, narrows `/stop` session matching, removes image-model override handling, loses TTS supplement markers, and leaves the expanded public protocol undocumented.

## What This Changes

The PR extends Gateway `chat.inject` with optional `role` and `idempotencyKey`, adds user-role transcript append support, updates generated Swift protocol models, and adds idempotency-focused transcript tests.

## Best Possible Solution

Rebase or repair the branch so the additive `chat.inject` role/idempotency contract lands without regressing current Gateway abort, image attachment, and TTS display behavior, then document the public protocol and coordinate it with the related metadata work.

## Reproduction Assessment

Yes for the blocking review findings: the merge result source shows removed current-main paths, and current-main types/tests establish the expected abort, image-model, and TTS contracts. The feature request itself is source-reproducible because current main's closed `chat.inject` schema rejects the new fields.

## Solution Assessment

No. The locked append/idempotency approach is directionally reasonable, but this branch is not yet the best landing path because it regresses unrelated Gateway behavior and needs public protocol documentation plus coordination with the overlapping metadata PR.

  - confidence: 0.95
