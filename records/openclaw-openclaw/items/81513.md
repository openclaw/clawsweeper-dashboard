## Summary

Keep open. The branch implements the requested Gateway `chat.inject` role/idempotency feature, but the latest head still drops several current-main Gateway behaviors around abort cleanup, provider-auth aborts, image-model routing, TTS supplement projection, and public protocol docs.

## What This Changes

The PR expands Gateway `chat.inject` with optional `role` and `idempotencyKey`, moves idempotency dedupe into the locked transcript append path, updates generated Swift models, and adds transcript replay/idempotency tests.

## Best Possible Solution

Land an additive Gateway protocol update that preserves current abort, provider-auth, image attachment, and TTS projection behavior, documents the expanded `chat.inject` contract, and coordinates with the related metadata PR before merge.

## Reproduction Assessment

Yes. The requested feature is source-reproducible because current main's closed `chat.inject` schema rejects `role` and `idempotencyKey`, and the blocking PR regressions are visible by comparing the PR head against current main source and tests.

## Solution Assessment

No. The locked idempotency direction is reasonable, but this branch is not the best landing path until it restores unrelated current-main Gateway behavior and documents the public protocol change.
