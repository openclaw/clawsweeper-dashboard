## Summary

Keep this PR open. It is a plausible implementation for the related `chat.inject` feature request, and the supplied live proof covers sequential replay, but the latest head still advertises idempotent replay while checking the key outside the transcript append critical section.

## What This Changes

The PR adds optional `role` and `idempotencyKey` to Gateway `chat.inject`, routes `role: "user"` through a user transcript append helper, regenerates Swift protocol models, and adds focused transcript coverage.

## Best Possible Solution

Land an additive, documented `chat.inject` contract after making idempotency durable under the transcript append lock and keeping generated clients aligned.

## Reproduction Assessment

Yes for the review finding by source inspection: the PR checks `idempotencyKey` before the append helper enters the per-transcript queue/write lock. The broader item is a feature request rather than a current-main bug reproduction.

## Solution Assessment

No. The direction matches the related request, but the patch should make idempotency atomic and update the public protocol/docs surface before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Make chat.inject idempotency atomic:** `src/gateway/server-methods/chat.ts:1471-1475`
