## Summary

Keep open: the requested Gateway capability is not on current main, and the draft branch still has merge-blocking regressions from reverting unrelated current Gateway chat.send, abort, media, and TTS behavior, plus a public protocol docs gap.

## What This Changes

The PR extends Gateway `chat.inject` with optional `role` and `idempotencyKey`, adds user-role transcript append support, and dedupes idempotent replays.

## Best Possible Solution

Rebase on current main, keep existing Gateway chat.send, abort, image-model, and TTS behavior intact, then land the additive `chat.inject` role/idempotency contract with public docs and focused tests.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head shows the removed current-main paths, and current tests/docs establish the expected contracts. The feature request itself is source-reproducible because current main's closed `chat.inject` schema rejects the new fields.

## Solution Assessment

No. The locked append/idempotency approach is directionally reasonable, but this branch must first stop reverting unrelated current-main Gateway behavior and document the expanded public protocol.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve canonical session matching for /stop:** `src/gateway/server-methods/chat.ts:1613`
  - body: The PR narrows abort matching to `active.sessionKey === params.sessionKey`, and the call site still passes the raw request key. Current main also matches the canonical key and backing sessionId before persisting partials, so `/stop` sent through an alias such as `main` can miss an active run registered under the canonical key and fail to abort it. Restore the alias/sessionId handling while adding `chat.inject` support.
  - confidence: 0.91
- **[P1] Keep the image-model override path:** `src/gateway/server-methods/chat.ts:2146-2149`
  - body: This branch removes the `resolveImageModelOverridePlan` path and only keeps image attachments inline when the current session model itself supports images. Existing Gateway behavior uses the configured image model and forwards `modelOverride`/fallbacks so image attachments work even when the session model is text-only; current tests assert that contract. With this diff those sends are routed as text-only media paths instead of using the configured vision model.
  - confidence: 0.95
- **[P2] Keep TTS supplement markers on injected media replies:** `src/gateway/server-methods/chat.ts:2702-2712`
  - body: The PR drops the `openclawTtsSupplement` marker when appending delayed media-only TTS replies. `projectRecentChatDisplayMessages` depends on that marker to merge the audio supplement into the original assistant message; without it, WebChat history can show a separate audio assistant row instead of the merged reply. Preserve the marker while refactoring the append helper.
  - confidence: 0.88
- **[P2] Document the new chat.inject contract:** `src/gateway/protocol/schema/logs-chat.ts:69-70`
  - body: Adding `role` and `idempotencyKey` expands the public Gateway protocol, but the docs still describe `chat.inject` only generically and omit user-role appends, idempotent replay responses, canonical message ids, parent-chain behavior, and duplicate-broadcast semantics. Update the public Gateway docs so callers can use the new contract correctly.
  - confidence: 0.9
