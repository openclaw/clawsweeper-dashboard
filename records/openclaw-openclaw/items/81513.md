## Summary

Keep this PR open: it addresses a real Gateway API gap with supplied live proof, but the current head still regresses existing Gateway stop, auth-provider abort, image-model routing, TTS projection, and protocol documentation behavior. The final `chat.inject` shape also overlaps with https://github.com/openclaw/openclaw/pull/82471 and needs coordinated human review before repair or landing.

## What This Changes

This PR extends Gateway `chat.inject` with `role` and `idempotencyKey`, adds locked transcript idempotency for injected messages, updates Swift protocol models, and expands transcript append tests.

## Best Possible Solution

Land one coordinated additive `chat.inject` update that supports user-role/idempotent replay while preserving current stop matching, auth-provider aborts, image-model routing, TTS projection, generated clients, and public docs.

## Reproduction Assessment

Yes. Current main’s closed schema rejects the new `role` and `idempotencyKey` fields, and the blocking regressions are source-reproducible by comparing PR head `09ec74efdd6dce5f11d2f01c8e064f7666ff1153` with current main.

## Solution Assessment

No. The locked idempotency direction is reasonable, but this branch is not ready until it restores unrelated current-main Gateway behavior and coordinates the public protocol shape with the overlapping metadata PR.

  - confidence: 0.92

  - confidence: 0.94
