## Summary

Keep open: the PR is a plausible additive Gateway feature and current main still lacks user-role `chat.inject` plus idempotent replay support, but the patch is not merge-ready because it expands a public protocol contract without updating the public docs. The related issue at https://github.com/openclaw/openclaw/issues/79820 remains the user-facing need, and https://github.com/openclaw/openclaw/pull/82471 is adjacent metadata/idempotency work rather than a full superseding replacement for user-role transcript replay.

## What This Changes

The PR extends Gateway `chat.inject` with optional `role` and `idempotencyKey`, routes user-role injections through transcript append, dedupes idempotent replays, updates Swift protocol models, and adds focused transcript tests.

## Best Possible Solution

Land a documented additive Gateway contract that covers assistant defaults, user-role injects, idempotency keys, deduped replay responses, canonical message ids, parent-chain behavior, and duplicate-broadcast suppression.

## Reproduction Assessment

Yes for the review finding: PR-head source expands `chat.inject`, while current main docs still describe assistant-only injection. The broader request is a new Gateway protocol capability rather than a current-main bug reproduction.

## Solution Assessment

Not yet. The locked append/idempotency implementation is coherent, but the public docs and API contract need a narrow update before this is the maintainable merge path.
