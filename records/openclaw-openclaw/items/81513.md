## Summary

Keep open because the latest head still drops current-main Gateway behavior around stop-command matching, provider-auth aborts, image-model routing, and TTS history projection while adding a public protocol change.

## What This Changes

The PR extends Gateway `chat.inject` with `role` and `idempotencyKey`, adds user-role transcript append and idempotent replay handling, refreshes generated Swift models, and expands focused transcript tests.

## Best Possible Solution

Land one coordinated additive `chat.inject` update that supports user-role/idempotent replay while preserving stop alias matching, auth-provider abort metadata, image-model override routing, TTS projection, docs, and generated clients.

## Reproduction Assessment

Yes. Current main’s closed schema rejects the new `role` and `idempotencyKey` fields, and the blocking regressions are source-reproducible by comparing PR head `cd2b5dae51399432c926341e93016e69b7beef95` with current main.

## Solution Assessment

No. The locked idempotency direction is reasonable, but this branch is not the best landing path until it restores unrelated current-main Gateway behavior and coordinates the public protocol shape with the overlapping metadata PR.
