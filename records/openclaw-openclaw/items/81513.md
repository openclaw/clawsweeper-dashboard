## Summary

Keep this PR open: the implementation is a plausible active Gateway protocol feature with supplied live proof, but it still has a blocking docs mismatch because the public WebChat and Control UI docs describe `chat.inject` as assistant-only while the PR adds user-role injection and idempotent replay semantics.

The PR extends Gateway `chat.inject` with optional `role` and `idempotencyKey`, routes user-role transcript appends through the shared transcript writer, returns deduped replay success, regenerates Swift models, and adds focused tests.
