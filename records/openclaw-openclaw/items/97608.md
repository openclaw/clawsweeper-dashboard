## Summary

## Best Possible Solution

- **[P2] Preserve the local-service lease across retry:** `src/agents/provider-transport-fetch.ts:905`
  - body: This branch releases the local-service lease before the retry and never reacquires it, so a successful retry can stream with the sidecar marked idle. Providers with `idleStopMs` can then be stopped while the retried response is still running; keep the lease through the response lifecycle or acquire a fresh lease for the retry.
  - confidence: 0.88

  - body: The retry reuses the same `guardedFetchOptions`, including `baseInit.body`. When the caller passes a `Request` with a body, the first fetch attempt can disturb or lock that stream, so the retry can fail locally instead of making a fresh provider request; restrict retries to replayable bodies or rebuild a buffered request body.
