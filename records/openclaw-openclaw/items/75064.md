## What This Changes

The PR moves memory plugin state storage into `createPluginRuntimeStore` and adds mirrored-runtime regression tests for memory-state plus Memory Wiki bridge/status artifact visibility.

## Best Possible Solution

Land a narrow compatibility fix that makes the active memory capability registry visible to canonical and mirrored runtime module instances while preserving the existing memory plugin SDK API. A host-owned capability service can remain a later durability improvement if maintainers want to remove global-store compatibility scaffolding.

## Solution Assessment

Yes for the immediate regression. Reusing `createPluginRuntimeStore` is the narrowest existing in-process duplicate-module sharing mechanism, while the broader host-owned capability service in the PR body is better treated as follow-up architecture work.
