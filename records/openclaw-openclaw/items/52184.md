## Label Justifications

- `P2`: The issue affects a bounded macOS gateway LaunchAgent path-selection workflow that can break after Volta Node upgrades, but it is not an emergency.
- `impact:crash-loop`: A LaunchAgent pinned to a removed Volta tools/image Node binary can fail to start or restart after Node upgrades.

## Summary

Keep open: current main and the latest release still leave Volta `tools/image` Node paths unchanged in the no-system-Node fallback path, and the remaining question is whether maintainers want Volta shim fallback to be a supported LaunchAgent policy.

## What This Changes

The issue requests macOS gateway LaunchAgent generation to prefer the stable Volta shim path over a version-pinned Volta Node image path.

## Best Possible Solution

Keep this issue as the canonical maintainer decision point, then land or reject a narrow resolver and regression-test update after the Volta shim fallback policy is confirmed.

## Reproduction Assessment

Yes. Source inspection is enough: with a supported Volta tools/image execPath and no supported system Node, current main returns the unchanged current executable because only Homebrew Cellar paths are stabilized.

## Solution Assessment

Unclear. Resolver-level canonicalization is the narrow plausible code boundary, but maintainers still need to decide whether the Volta shim is a supported service fallback or whether system/Homebrew Node remains the intended policy.
