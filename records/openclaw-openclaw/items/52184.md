## Summary

Keep open: current main and the latest release still leave Volta `tools/image` Node paths unchanged in the no-system-Node fallback path, while the exact target-repo PR attempt was closed unmerged and the remaining question needs maintainer policy review.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer a stable Volta shim node path over a version-pinned Volta tools/image node path.

## Best Possible Solution

Keep this issue as the canonical maintainer decision point, then land a narrow resolver/test update only if maintainers confirm Volta shim fallback is the supported policy.

## Reproduction Assessment

Yes. Source inspection is enough: with a supported Volta tools/image execPath and no supported system Node, `resolvePreferredNodePath` returns `resolveStableNodePath(execPath)`, and current main only normalizes Homebrew Cellar paths.

## Solution Assessment

Unclear. Resolver-level canonicalization is the narrow plausible boundary, but maintainers still need to decide whether the Volta shim is the right service fallback instead of continuing to recommend system/Homebrew Node.
