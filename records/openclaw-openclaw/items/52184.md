## Label Justifications

- `P2`: This is a normal-priority macOS gateway setup reliability problem affecting a limited Volta-managed Node path.
- `impact:crash-loop`: A LaunchAgent pinned to a removed Volta image Node executable can leave the managed gateway service unable to start after Node upgrades.

## Summary

Keep open: current main and v2026.6.10 still only normalize Homebrew Cellar Node paths, while the requested Volta image-path to shim behavior remains a maintainer policy decision with an open fork PR and a closed same-repo attempt.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer a stable Volta shim path over a version-pinned Volta image Node path when no better system Node is available.

## Best Possible Solution

Maintainers should decide the daemon runtime policy, then either land a narrow resolver-and-test change with Volta launchd proof or close this in favor of the broader runtime-pin policy path.

## Reproduction Assessment

Yes, source-level: current `resolveStableNodePath` misses the reported Volta image path and returns it unchanged. The remaining behavior is clear from the helper and `resolvePreferredNodePath` caller without mutating a macOS LaunchAgent.

## Solution Assessment

Unclear until maintainers settle the runtime policy. Resolver-level canonicalization is the narrow implementation seam if Volta shim fallback is accepted; otherwise explicit runtime pinning or documentation should own the workflow.
