## Label Justifications

- `P2`: The issue affects macOS gateway LaunchAgent stability for Volta-managed Node installs but has a bounded service-install surface and an existing candidate fix path.
- `impact:crash-loop`: A version-pinned Volta Node executable in the LaunchAgent can leave the managed gateway unable to start after Node upgrades.

## Summary

Keep open: current main and the latest release still leave Volta tools/image Node paths unchanged, while the issue has a same-topic closed PR and an open cross-repository candidate that should not be treated as shipped implementation.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer a stable Volta shim Node path over a version-pinned Volta image path.

## Best Possible Solution

Review or replace the narrow Volta normalization candidate against the current daemon runtime policy, then land a tested helper-level fix only if maintainers accept Volta shim fallback as supported behavior.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection gives a medium-confidence reproduction path: a supported Volta tools/image `process.execPath` falls through `resolveStableNodePath` unchanged when no supported system Node is selected first.

## Solution Assessment

Unclear. Extending `resolveStableNodePath` is the narrowest implementation boundary, but maintainers still need to accept or reject Volta shim fallback versus the current documented preference for system Node in managed services.
