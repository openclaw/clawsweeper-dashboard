## Summary

Keep this open: current main still returns Volta tools/image Node paths unchanged when no supported system Node is available, and the explicit fix PR for this issue is open and unmerged.

## What This Changes

Request that macOS gateway LaunchAgent generation prefer a stable system Node, then a stable Volta shim path, instead of a version-pinned Volta tools/image Node path.

## Best Possible Solution

Review, rebase, and land or supersede https://github.com/openclaw/openclaw/pull/52252 with complete Volta image-path canonicalization, focused tests, and redacted real Volta gateway/install proof.

## Reproduction Assessment

Yes by source inspection. Current main only normalizes Homebrew Cellar paths, and the no-supported-system-Node fallback can still carry an unchanged Volta tools/image Node path into generated gateway service arguments.

## Solution Assessment

Yes directionally. Extending the stable-node resolver is the narrowest place to solve this, but the current open PR needs its coverage and proof gaps resolved before the issue should close.
