## Summary

Keep open: current main and the latest shipped release still leave Volta `tools/image/node/<version>/bin/node` paths unchanged when no supported system Node is available, and the open linked implementation PR remains the active fix path rather than a reason to close this issue early.

## What This Changes

The issue asks macOS gateway service generation to prefer Volta's stable shim over a version-pinned Volta tools/image Node executable when no supported system Node is available.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/52252 with resolver coverage for Volta image paths, preserving supported-system-Node preference and fallback behavior.

## Reproduction Assessment

Yes, source-level. A supported Volta `tools/image/node/<version>/bin/node` execPath with no supported system Node still flows through `resolveStableNodePath` unchanged and can be returned into gateway service generation.

## Solution Assessment

Yes in direction: resolver-level canonicalization is the narrow maintainable boundary. The linked PR still needs review/proof before this issue should close.
