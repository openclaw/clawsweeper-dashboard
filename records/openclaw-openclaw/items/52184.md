## Summary

Keep open: current main and the latest release still lack Volta tools/image-to-shim normalization, while open PR #52252 already targets the exact remaining resolver gap.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer the stable Volta shim path over a version-pinned Volta Node image path.

## Best Possible Solution

Review and land PR #52252, or an equivalent focused resolver change, with coverage for accessible and missing Volta shim paths after supported system Node candidates are considered.

## Reproduction Assessment

Yes. A source reproduction is to use a supported `/Users/example/.volta/tools/image/node/24.14.0/bin/node` execPath with no supported system Node; current main routes it through `resolveStableNodePath` and returns the version-pinned path unchanged.

## Solution Assessment

Yes. The narrow resolver-level approach is the best current fix because gateway install already consumes the selected Node executable, while broader LaunchAgent PATH canonicalization was handled separately by #75233/#75246.
