## Summary

Keep open: current main still does not canonicalize Volta `tools/image/node/<version>/bin/node` to the stable Volta shim, and the open linked implementation PR remains the active fix path rather than a reason to close the issue before merge.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer `~/.volta/bin/node` over version-pinned Volta `tools/image` Node paths when Volta is the selected Node manager.

## Best Possible Solution

Land or supersede the linked resolver fix so Volta image paths canonicalize to accessible stable shims, with regression coverage for default/custom Volta homes while preserving supported-system-Node preference.

## Reproduction Assessment

Yes, source-level. A supported Volta `tools/image/node/<version>/bin/node` execPath with no supported system Node still flows through `resolveStableNodePath` unchanged and can be returned by `resolvePreferredNodePath`.

## Solution Assessment

Yes in direction: resolver-level canonicalization is the narrow maintainable boundary. The active PR needs review/proof completion before this issue should close.
