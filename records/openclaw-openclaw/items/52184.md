## Summary

Keep open: current main still does not normalize Volta `tools/image/node/<version>/bin/node` paths to the stable Volta shim, and the issue is paired with an open unmerged candidate PR that should be reviewed rather than superseded by a new cleanup action.

## What This Changes

The issue requests choosing a stable Volta node shim for generated macOS gateway LaunchAgents instead of pinning `ProgramArguments[0]` to a Volta `tools/image/node/<version>/bin/node` path.

## Best Possible Solution

Review the open linked PR against current daemon path policy, then either land a narrow stable-node helper fix or document that OpenClaw intentionally requires system Node for managed gateway services.

## Reproduction Assessment

Yes from source, with medium confidence: current `resolveStableNodePath` returns non-Homebrew paths unchanged, so the reported Volta `tools/image` executable remains pinned unless a system Node candidate is available. I did not run a live macOS LaunchAgent install because this review is read-only.

## Solution Assessment

Unclear: extending `resolveStableNodePath` is the narrowest implementation shape, but the docs and labels show an unresolved maintainer decision about whether Volta shim fallback should be supported for managed services instead of requiring system Node.
