## Summary

Keep open: current main and the latest release still canonicalize only Homebrew Cellar Node paths, so Volta image paths can still reach generated gateway service arguments; the open closing PR remains the active implementation path.

## What This Changes

Request macOS gateway service generation to prefer the stable Volta shim when the selected Node path is a Volta tools/image version directory.

## Best Possible Solution

Review, fix, and land or explicitly supersede https://github.com/openclaw/openclaw/pull/52252 with Volta resolver coverage for the intended layouts and redacted real Volta gateway/install proof.

## Reproduction Assessment

Yes, source-level. A supported Volta image-path `execPath` with no supported system Node still flows through `resolveStableNodePath` unchanged and can become the gateway service node program.

## Solution Assessment

Yes in direction. Resolver-level canonicalization is the right boundary, but this issue should stay open until the linked implementation lands or is explicitly superseded.
