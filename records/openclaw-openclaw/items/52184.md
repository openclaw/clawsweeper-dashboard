## Summary

Keep this issue open. Current main still only normalizes Homebrew Cellar Node paths; a Volta tools/image executable is returned unchanged and can still be written into the macOS LaunchAgent. The issue also has open closing PRs, so the maintainer path is to review/land or close those candidates rather than closing the issue early.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer the stable Volta shim path over a version-pinned Volta tools/image Node executable.

## Best Possible Solution

Resolve this through PR #52252 or an equivalent focused change: teach `resolveStableNodePath` to detect Volta tools/image Node executables, prefer an executable stable Volta shim, fall back to the original path when unavailable, and add targeted resolver plus runtime-path coverage. Keep the broader LaunchAgent PATH canonicalization tracked separately by #75233/#75246.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: pass `/Users/example/.volta/tools/image/node/24.14.0/bin/node` to `resolveStableNodePath`; current main does not match the Homebrew Cellar regex and returns that version-pinned path unchanged.

## Solution Assessment

Yes. The proposed resolver-level fix is the narrowest maintainable solution for this issue because the install plan already routes selected Node executables through `resolveStableNodePath`; changing that seam avoids unrelated launchd, PATH, or service-audit behavior.
