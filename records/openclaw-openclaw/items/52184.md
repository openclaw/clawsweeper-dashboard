## Summary

Keep open: current main and v2026.5.18 still leave Volta `tools/image/node/<version>/bin/node` paths unchanged, and the open linked implementation PR remains the active fix path rather than a reason to close early.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer a stable Volta shim path over a version-pinned Volta Node image path when a supported system Node is not available.

## Best Possible Solution

Review, fix, and land or supersede https://github.com/openclaw/openclaw/pull/52252 with resolver coverage for Volta image paths while preserving supported-system-Node preference and fallback behavior.

## Reproduction Assessment

Yes, source-level. A supported Volta `tools/image/node/<version>/bin/node` execPath with no supported system Node still flows through `resolveStableNodePath` unchanged and can be returned into gateway service generation.

## Solution Assessment

Yes in direction: resolver-level canonicalization is the narrow maintainable boundary. The linked PR still needs review, any required fixes, and real Volta proof before this issue should close.
