## Summary

Keep open: current main and the latest release still leave Volta `tools/image` Node paths unchanged when no supported system Node is available, and the open implementation PR still references this issue with closing syntax rather than being merged.

## What This Changes

The issue asks macOS gateway LaunchAgent generation to prefer a stable Volta shim Node path over a version-pinned Volta `tools/image` Node path.

## Best Possible Solution

Land the linked resolver fix or a replacement that canonicalizes Volta `tools/image` Node paths to accessible stable Volta shims, includes custom Volta-home coverage, and preserves supported-system-Node preference.

## Reproduction Assessment

Yes, source-level. A supported `/Users/example/.volta/tools/image/node/24.14.0/bin/node` execPath with no supported system Node flows through `resolveStableNodePath` unchanged and is returned by `resolvePreferredNodePath`.

## Solution Assessment

Yes in direction: resolver-level normalization is the narrow maintainable boundary. The active PR is the right path to review, but it should be updated and proven before this issue closes.
