## Summary

Keep open: current main and the latest release still only normalize Homebrew Cellar node paths, while the Volta image-path request remains owned by an open linked PR and needs the maintainer policy decision on whether Volta shim fallback is supported for LaunchAgent services.

## What This Changes

Request OpenClaw to prefer a stable Volta shim node path over version-pinned Volta image node paths when generating macOS gateway LaunchAgents.

## Best Possible Solution

Review the open linked Volta shim PR against the daemon runtime policy, then land a narrow helper-and-test change if maintainers accept Volta shim fallback as supported.

## Reproduction Assessment

Yes, source-level. Current `resolveStableNodePath` returns non-Homebrew paths unchanged, so the reported Volta image node path is still not redirected to the Volta shim on current main.

## Solution Assessment

Unclear until maintainers settle the runtime policy. The linked PR's narrow `resolveStableNodePath` extension is the right implementation seam if Volta shim fallback is accepted.
