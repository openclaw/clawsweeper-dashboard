## Summary

Keep open: current main still canonicalizes Homebrew Cellar Node paths only, so Volta tools/image Node paths can still flow into the generated gateway service command, and the open linked implementation PR remains the active fix path before this issue should close.

## What This Changes

Request that macOS gateway LaunchAgent Node path selection prefer a stable Volta shim over version-pinned Volta tools/image Node paths when no supported system Node is available.

## Best Possible Solution

Land or supersede the linked resolver fix so Volta image paths canonicalize to accessible stable shims with regression coverage for default/custom Volta homes, while preserving supported-system-Node preference and fallback behavior.

## Reproduction Assessment

Yes, source-level. A supported Volta tools/image node execPath with no supported system Node still passes through resolveStableNodePath unchanged and can be returned by resolvePreferredNodePath; I did not run a live macOS LaunchAgent.

## Solution Assessment

Yes in direction: resolver-level canonicalization is the narrow maintainable boundary because service generation consumes the selected node path. The linked PR still needs review and proof completion before this issue should close.
