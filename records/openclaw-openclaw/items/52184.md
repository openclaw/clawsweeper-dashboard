## Summary

Keep open: current main and latest release still leave Volta `tools/image` Node paths unchanged when no supported system Node is available, and the open implementation PR remains the active fix path with unresolved review/proof work.

## What This Changes

Prefer an accessible Volta shim over a version-pinned Volta `tools/image` Node path when generating the macOS gateway LaunchAgent.

## Best Possible Solution

Land or supersede the linked resolver fix so Volta image paths canonicalize to accessible stable shims, with coverage for default/custom Volta homes while preserving supported-system-Node preference.

Yes, source-level. A supported Volta `tools/image/node/<version>/bin/node` execPath with no supported system Node flows through `resolveStableNodePath` unchanged and is returned by `resolvePreferredNodePath`.
