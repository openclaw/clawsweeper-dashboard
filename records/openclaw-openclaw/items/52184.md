## Summary

Keep open: current main still has no Volta tools/image-to-shim normalization, so users without a supported system Node can still get a version-pinned Volta Node path in generated service arguments. Open PR #52252 already targets the exact resolver gap, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Prefer the stable Volta shim Node executable when generating macOS gateway LaunchAgent arguments instead of persisting a version-specific Volta tools/image path.

## Best Possible Solution

Review and land PR #52252, or an equivalent focused change, so the resolver prefers an executable `~/.volta/bin/node` for Volta tools/image paths after supported system Node candidates are considered, with targeted resolver and runtime-path tests.

## Reproduction Assessment

Yes. A source reproduction is to pass `/Users/example/.volta/tools/image/node/24.14.0/bin/node` as a supported current execPath with no supported system Node; current main routes it through `resolveStableNodePath` and returns the version-pinned path unchanged.

## Solution Assessment

Yes. The narrow resolver-level change in the open PRs is the most maintainable fix because the gateway install path already consumes the selected Node executable; broader LaunchAgent PATH canonicalization is separately handled by #75233/#75246.
