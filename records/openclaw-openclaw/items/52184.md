## Summary

Keep open: current main and v2026.5.7 still leave Volta tools/image Node paths unchanged when no supported system Node is available, and the linked implementation PR remains open with review blockers rather than merged.

## What This Changes

Prefer a stable Volta shim for the macOS gateway LaunchAgent Node executable when the current Node path is a Volta tools/image path and no supported system Node is available.

## Best Possible Solution

Land the linked resolver fix or a replacement that canonicalizes Volta tools/image Node paths to accessible stable Volta shims, includes custom Volta-home coverage, and preserves supported-system-Node preference.

## Reproduction Assessment

Yes, source-level. A supported /Users/example/.volta/tools/image/node/24.14.0/bin/node execPath with no supported system Node flows through resolveStableNodePath unchanged and is returned by resolvePreferredNodePath.

## Solution Assessment

Yes in direction: resolver-level normalization is the narrow maintainable fix. The existing linked PR is the active path, but it should be updated and proven before this issue closes.
