## Summary

Keep this open: current main and the latest release still return Volta tools/image Node paths unchanged when no supported system Node is available, and an open linked PR already targets the requested resolver change.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer a stable Volta shim executable over a version-pinned Volta tools/image Node executable.

## Best Possible Solution

Review the linked Volta resolver PR against non-interactive launchd behavior, then either land a tested resolver update or document why OpenClaw intentionally keeps the version-pinned Volta path fallback.

## Reproduction Assessment

Yes, source inspection provides a high-confidence reproduction path: a Volta tools/image `process.execPath` flows through `resolveStableNodePath`, which returns non-Cellar paths unchanged on current main. I did not run a live macOS launchd install in this read-only review.

## Solution Assessment

Unclear. The proposed resolver change is narrow and plausible, but the Volta shim behavior in non-interactive service contexts needs maintainer review before calling it the best permanent fallback.
