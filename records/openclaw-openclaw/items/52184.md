## Label Justifications

- `P2`: The issue affects a bounded macOS gateway LaunchAgent path-selection workflow that can break after Volta Node upgrades, but it is not an emergency.
- `impact:crash-loop`: A LaunchAgent pinned to a removed Volta tools/image Node binary can fail to start or restart after a Volta Node upgrade.

## Summary

Keep open: current main and latest release v2026.6.10 still leave Volta tools/image Node paths unchanged when no supported system Node is available, and the open fork PR is a candidate implementation rather than a reason to close this issue before maintainer policy review.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer the stable Volta shim path over a version-pinned Volta tools/image Node path.

## Best Possible Solution

Keep this issue as the canonical policy decision, then either land a narrow resolver/test update for Volta shim fallback or explicitly keep system/Homebrew Node as the only supported stable daemon runtime target.

## Reproduction Assessment

Yes. Source inspection is enough: with a supported Volta tools/image execPath and no supported system Node, current main returns the unchanged executable because only Homebrew Cellar paths are stabilized.

## Solution Assessment

Unclear. Resolver-level canonicalization is the narrow plausible code boundary, but maintainers still need to decide whether a Volta shim is an acceptable supported LaunchAgent fallback.
