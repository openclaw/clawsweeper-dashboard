## Summary

Keep open: current main and the latest release still return Volta `tools/image` Node paths unchanged when no supported system Node is available, and the open implementation PR remains the active closing path with unresolved proof/review work.

## What This Changes

The issue asks for macOS gateway LaunchAgent generation to prefer a stable Volta shim path over a version-pinned Volta image Node path.

## Best Possible Solution

Land or supersede the linked resolver fix so Volta image paths canonicalize to accessible stable shims, with coverage for default/custom Volta homes while preserving supported-system-Node preference.

## Reproduction Assessment

Yes, source-level. Current main returns non-Cellar paths unchanged in `resolveStableNodePath`, and `resolvePreferredNodePath` returns that unchanged value for a supported version-manager execPath when no supported system Node is available.

## Solution Assessment

Yes in direction. Resolver-level canonicalization is the maintainable boundary because gateway install already consumes `resolvePreferredNodePath`; the active PR still needs its review blockers and real behavior proof resolved before this issue closes.
