## Summary

Keep open: current main and v2026.5.22 still normalize Homebrew Cellar Node paths only, so a Volta `tools/image/node/<version>/bin/node` path remains version-pinned when no supported system Node is available. The target-repo PR for this was closed as not merge-ready, and the remaining work needs maintainer confirmation that direct Volta shim fallback is the desired LaunchAgent policy.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer the stable Volta shim path over a version-pinned Volta Node image path.

## Best Possible Solution

Land or adopt one narrow target-repo patch in `resolveStableNodePath` that maps accessible Volta image Node paths to the stable shim only after maintainer confirmation of that fallback policy, with focused resolver/runtime tests and real macOS launchd/Volta proof.

## Reproduction Assessment

Yes, source-level: current main returns non-Cellar paths unchanged in `resolveStableNodePath`, and `resolvePreferredNodePath` returns that unchanged value for a version-managed execPath when no supported system Node is found. No live macOS Volta LaunchAgent was run in this read-only review.

## Solution Assessment

Unclear: resolver-level canonicalization is the right seam, but choosing the Volta shim as a launchd fallback needs maintainer approval because current docs warn version-manager service Node paths can break and related Volta shim behavior has been problematic elsewhere.
