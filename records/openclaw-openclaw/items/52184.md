## Label Justifications

- `P2`: This is a bounded macOS gateway service stability improvement for Volta-managed installs with limited but real availability impact.
- `impact:crash-loop`: A LaunchAgent pinned to an obsolete version-managed Node executable can leave the supervised gateway unable to start after Node upgrades.

## Summary

Keep open: current main and the latest release still return Volta `tools/image` Node paths unchanged in the no-system-Node fallback path, and the only linked implementation is an open cross-repo PR rather than a merged target-repo fix.

## What This Changes

The issue requests that macOS gateway LaunchAgent generation prefer a stable Volta shim path over a version-pinned Volta Node image path.

## Best Possible Solution

Review the linked Volta resolver candidate against launchd and non-interactive Volta behavior, then either land a target-repo resolver update with focused tests and real service proof or document why the version-pinned fallback remains intentional.

## Reproduction Assessment

Yes. Source inspection is a high-confidence reproduction path: a Volta `tools/image` `process.execPath` reaches `resolveStableNodePath`, which returns non-Cellar paths unchanged on current main.

## Solution Assessment

Unclear. Resolver-level canonicalization is the narrow likely boundary, but maintainers still need to decide whether the Volta shim is the safe fallback for macOS launchd when no supported system Node exists.
