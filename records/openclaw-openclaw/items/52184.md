## Summary

Keep open: current main still does not normalize Volta `tools/image/node/<version>/bin/node` paths to the stable Volta shim when no supported system Node is available. The related macOS PATH and system-Node-preference work reduced adjacent LaunchAgent fragility, but it did not implement the requested `ProgramArguments[0]` fallback, and the Volta-specific implementation is still only represented by an open fork PR.

## What This Changes

Request macOS gateway LaunchAgent generation to prefer the stable Volta Node shim over a version-pinned Volta image path.

## Best Possible Solution

Land one narrow target-repo change, or adopt the linked fork work, that maps accessible Volta image Node paths to the stable Volta shim only after maintainer confirmation of that fallback policy.

## Reproduction Assessment

Yes from source inspection: the current path resolver normalizes Homebrew Cellar paths but returns non-Cellar version-manager paths unchanged, and the preferred-node fallback returns that unchanged value when no supported system Node exists.

## Solution Assessment

Unclear: the Volta shim preference is a small and plausible implementation path, but maintainers should first decide whether direct Volta shim use is the desired fallback and whether to adopt the existing fork PR.
