## Summary

Keep open: the PR addresses a real Discord multi-account routing gap that is not implemented on current main, but it is not mergeable as-is because it has concrete routing regressions and lacks inspectable real Discord proof.

## What This Changes

The PR adds Discord route-owner resolution, explicit cross-bot mention bypass, bot-name mention matching, bot identity fallback, and focused Discord monitor tests.

## Best Possible Solution

Land one canonical Discord-owned routing repair that selects targeted bot accounts from Discord mention/reference metadata before default routing, suppresses non-target default owners, uses plugin SDK seams, and proves the behavior with focused tests and live evidence.

## Reproduction Assessment

Yes, at source level. Current main still routes before cross-bot target metadata and compares mention/reply state only to the running bot ID, while the PR diff exposes the DM and stale-account blockers directly.

## Solution Assessment

No, not as-is. The direction is useful, but the patch should exclude DMs from owner blocking, ignore inactive accounts, use plugin SDK/public seams, and be reconciled with the canonical Discord routing policy before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:442`
  - body: The owner check now runs when `!threadBinding && !configuredBinding`, including DMs, but `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding, this handler drops the DM even though the owning account will not receive this DM event.
  - confidence: 0.91
- **[P2] Ignore inactive accounts when choosing the owner:** `extensions/discord/src/monitor/route-owner.ts:40`
  - body: `resolveDiscordRouteOwner` includes account IDs found only in bindings, even when that Discord account is no longer configured or running. A stale binding can win ownership and cause all active accounts to drop the channel message for a nonexistent handler.
  - confidence: 0.87
- **[P2] Use plugin SDK seams from the Discord plugin:** `extensions/discord/src/monitor/route-owner.ts:1-7`
  - body: This new production module imports `../../../../src/**` internals from inside `extensions/discord`. Bundled plugin production code should use `openclaw/plugin-sdk/*` or a documented public seam, otherwise the Discord plugin remains coupled to core source layout and current plugin-boundary checks.
  - confidence: 0.82
