## Summary

Keep open: current main still does not implement the Discord cross-bot owner routing behavior, but this PR is not mergeable as-is because it has blocking routing bugs, a plugin-boundary violation, missing real behavior proof, and overlap with open canonical routing work.

## What This Changes

The PR adds Discord route-owner resolution, explicit cross-bot mention bypass handling, tests, and a bot user ID fallback when Discord identity lookup fails.

## Best Possible Solution

Land one Discord plugin routing change that computes target bot/account from Discord mention and reply metadata before default-owner routing, uses active account metadata and SDK seams, and proves suppression with focused tests.

## Reproduction Assessment

Yes, at source level. Current main routes before cross-bot target selection and compares mention/reply metadata only to the running bot ID, while the PR body and related issues describe a concrete two-bot Discord channel setup.

## Solution Assessment

No. The direction is useful, but this patch needs active-account ownership, Discord metadata-only bypass, SDK imports, wildcard route priority handling, and real behavior proof before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Exclude DMs from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:437-452`
  - body: The new owner check runs for direct messages, but the mention bypass is disabled for DMs. If another account has a more specific direct-peer binding for the sender, this account can silently drop a DM that the other bot will not receive in this handler.
  - confidence: 0.92
- **[P2] Restrict route owners to active accounts:** `extensions/discord/src/monitor/route-owner.ts:35-40`
  - body: `resolveDiscordRouteOwner` includes account IDs discovered only from bindings. A stale or removed account binding can win ownership and make active monitors drop messages for a bot that is not running.
  - confidence: 0.88
- **[P2] Use plugin SDK routing imports:** `extensions/discord/src/monitor/route-owner.ts:1-8`
  - body: This new Discord plugin production module imports config and routing internals through `../../../../src/**`. Bundled plugin code should use `openclaw/plugin-sdk/*` or local public exports so it does not bypass the supported plugin boundary.
  - confidence: 0.9
- **[P2] Handle wildcard peer priority:** `extensions/discord/src/monitor/route-owner.ts:10-18`
  - body: Current main's route resolver can return `matchedBy: "binding.peer.wildcard"`, but this priority table has no entry for it. After rebasing, wildcard owner routes will either fail the type contract or compare with an undefined priority.
  - confidence: 0.84
- **[P2] Require Discord mention metadata for owner bypass:** `extensions/discord/src/monitor/message-handler.preflight.ts:96-100`
  - body: `buildBotNameMentionRegexes` makes `@` optional and merges the regex into owner-bypass matching. Ordinary text containing the bot username can therefore bypass channel ownership even though the canonical routing policy requires Discord mention metadata.
  - confidence: 0.78
