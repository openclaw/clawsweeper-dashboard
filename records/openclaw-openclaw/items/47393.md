## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:442`
  - body: The new owner check runs for DMs because it only gates on thread/configured binding state, while `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding for the same sender, this bot drops a legitimate DM even though the other account will not receive this DM event.
  - confidence: 0.91
- **[P2] Filter owner candidates to active accounts:** `extensions/discord/src/monitor/route-owner.ts:35-40`
  - body: `resolveDiscordRouteOwner` includes account IDs found only in bindings. A stale binding for a removed or disabled Discord account can win ownership and make active accounts drop the channel message for a handler that is not running.
  - confidence: 0.88
- **[P2] Import routing through plugin SDK seams:** `extensions/discord/src/monitor/route-owner.ts:1-8`
  - body: This new Discord production module imports config, routing, and session-key internals with `../../../../src/**` paths. Extension production code should use `openclaw/plugin-sdk/*` or a documented local/public seam, otherwise the bundled plugin depends on private core layout and violates the extension boundary.
  - confidence: 0.86
