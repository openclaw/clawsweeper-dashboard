## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Exclude DMs from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:442-452`
  - body: The owner check runs for direct messages while bypassOwnerBecauseMentioned is always false for DMs. If another account has a more specific direct-peer binding for the same sender, this handler drops a legitimate DM even though that other account will not receive this DM event.
  - confidence: 0.91
- **[P2] Limit owner candidates to active accounts:** `extensions/discord/src/monitor/route-owner.ts:35-40`
  - body: resolveDiscordRouteOwner includes account IDs found only in bindings. A stale binding for a removed or disabled Discord account can win ownership and make active accounts drop channel messages for a handler that is not running.
  - confidence: 0.88
- **[P2] Route owner logic through SDK seams:** `extensions/discord/src/monitor/route-owner.ts:1-8`
  - body: This new Discord production module imports config, routing, and session-key internals via ../../../../src/** paths. Extension production code should use openclaw/plugin-sdk/* or a documented local/public seam so bundled plugins do not depend on private core layout.
  - confidence: 0.86
- **[P2] Require Discord metadata for owner bypass:** `extensions/discord/src/monitor/message-handler.preflight.ts:85-100`
  - body: The bot-name regex treats plain text like @Bot Name as a mention even when Discord did not resolve a user mention. That can let a non-owner account bypass the owner guard from incidental text, while the canonical Discord routing issue calls for native mention and reply metadata to drive cross-bot targeting.
  - confidence: 0.78
