## Summary

Keep open: the PR addresses a real Discord multi-account routing problem, but the branch is not safe to merge as-is because it can drop DMs, can select stale/non-running owner accounts, crosses the plugin/core boundary, and still lacks inspectable real Discord proof.

## What This Changes

The PR adds Discord route-owner resolution, explicit cross-bot mention bypass, bot-name mention matching, bot identity fallback, and focused Discord monitor tests.

## Best Possible Solution

Land one canonical Discord-owned routing repair that derives targeted bot accounts from Discord mention/reference metadata before default routing, suppresses non-target default owners, uses plugin SDK seams, and proves the behavior with focused tests plus real Discord evidence.

## Reproduction Assessment

Yes, at source level. Current main still routes before cross-bot target metadata and compares mention/reply state only to the running bot ID, while the PR diff exposes concrete DM and stale-account regressions.

## Solution Assessment

No, not as-is. The direction is useful, but the patch should exclude DMs from owner blocking, ignore inactive accounts, use plugin SDK/public seams, and be reconciled with the canonical Discord routing policy before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:442`
  - body: The owner check now runs when `!threadBinding && !configuredBinding`, including DMs, but `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding, this handler drops the DM even though the owning account will not receive this DM event.
  - confidence: 0.91
- **[P2] Filter owner candidates to active accounts:** `extensions/discord/src/monitor/route-owner.ts:40-47`
  - body: `resolveDiscordRouteOwner` includes account IDs found only in bindings, even when that Discord account is no longer configured or running. A stale binding can win ownership and cause all active accounts to drop the channel message for a nonexistent handler.
  - confidence: 0.87
- **[P2] Import routing through plugin SDK seams:** `extensions/discord/src/monitor/route-owner.ts:1-7`
  - body: This new production module imports `../../../../src/**` internals from inside the Discord plugin. Bundled plugin production code should use `openclaw/plugin-sdk/*` or a documented public seam, otherwise the Discord plugin stays coupled to core source layout and current plugin-boundary checks.
  - confidence: 0.84
