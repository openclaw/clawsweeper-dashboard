## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Exclude DMs from owner suppression:** `extensions/discord/src/monitor/message-handler.preflight.ts:442`
  - body: The owner check runs for direct messages, but `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding for the same sender, this handler drops a legitimate DM even though the other bot does not receive this DM event, causing silent message loss in multi-account setups.
  - confidence: 0.91
- **[P1] Remove bot-name text from explicit target routing:** `extensions/discord/src/monitor/message-handler.preflight.ts:427-430`
  - body: Merging `botNameMentionRegexes` into the owner bypass and final mention regexes makes plain text like `@Bot Name` count as an explicit Discord target even when Discord mention metadata is absent. The canonical routing direction is native mention/reply metadata, so this can misroute prose or copied text to the alternate bot.
  - confidence: 0.86
- **[P2] Filter owner candidates to running Discord accounts:** `extensions/discord/src/monitor/route-owner.ts:40`
  - body: `listDiscordCandidateAccountIds` includes account IDs found only in bindings, so stale bindings for removed accounts can win ownership and make active accounts drop messages as owned by a bot that is not running. Owner resolution should consider only configured/active Discord accounts, or otherwise prove the owner can process the event.
  - confidence: 0.88
- **[P2] Use plugin SDK routing seams:** `extensions/discord/src/monitor/route-owner.ts:1-8`
  - body: This extension production file imports config and routing internals through `../../../../src/**`, which violates the extension boundary and conflicts with the current plugin-SDK-based Discord monitor shape on main. Rebase this helper onto `openclaw/plugin-sdk/*` exports or add a proper SDK seam first.
  - confidence: 0.86
