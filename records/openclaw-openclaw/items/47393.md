## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:442-452`
  - body: The owner check runs for DMs while the explicit-mention bypass is always false for DMs. A more specific direct-peer binding on another account can make this handler drop a legitimate DM even though that other bot will not receive this DM event.
  - confidence: 0.91
- **[P2] Ignore non-running accounts when resolving owners:** `extensions/discord/src/monitor/route-owner.ts:35-40`
  - body: resolveDiscordRouteOwner includes account IDs found only in bindings. A stale or disabled Discord account binding can win ownership and cause active accounts to drop messages for a handler that is not running.
  - confidence: 0.88
- **[P2] Route through plugin SDK seams:** `extensions/discord/src/monitor/route-owner.ts:1-8`
  - body: This new Discord production module imports config, routing, and session-key internals via relative ../../../../src paths. Extension production code should use openclaw/plugin-sdk subpaths or a documented local/public seam so the plugin does not depend on private core layout.
  - confidence: 0.86
- **[P2] Require Discord metadata for owner bypass:** `extensions/discord/src/monitor/message-handler.preflight.ts:85-100`
  - body: The bot-name regex treats plain text such as @Bot Name as a mention even when Discord did not resolve a user mention. That can let a non-owner account bypass the owner guard from incidental text, contrary to the metadata-first Discord routing contract.
  - confidence: 0.78
