## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Exclude DMs from owner suppression:** `extensions/discord/src/monitor/message-handler.preflight.ts:438-452`
  - body: The owner check can run for direct messages, but `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding for the same sender, this account will drop the DM as owned by the other account even though this handler is the one receiving the DM event.
  - confidence: 0.9
- **[P1] Filter owner candidates to active accounts:** `extensions/discord/src/monitor/route-owner.ts:35-39`
  - body: `listDiscordCandidateAccountIds` includes account IDs from bindings even if those accounts are no longer configured or running. A stale binding can therefore win ownership and make active accounts drop channel traffic with no responder left.
  - confidence: 0.88
- **[P1] Use native Discord targets for owner bypass:** `extensions/discord/src/monitor/message-handler.preflight.ts:85-100`
  - body: The bypass treats a bot-name text regex as an explicit target, but the canonical Discord routing issue says plain text names are not mentions and routing should use Discord mention/reply metadata. This can let ordinary text bypass the owner guard and route a message to the wrong bot.
  - confidence: 0.86
- **[P2] Keep Discord plugin code on SDK seams:** `extensions/discord/src/monitor/route-owner.ts:1-7`
  - body: This new production module imports config and routing internals from `../../../../src/**`, which violates the current extension boundary and will drift from the plugin SDK/local Discord seams used on main.
  - confidence: 0.9
