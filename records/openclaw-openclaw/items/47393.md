## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Exclude DMs from owner suppression:** `extensions/discord/src/monitor/message-handler.preflight.ts:442-452`
  - body: The owner check runs whenever there is no thread/configured binding, including DMs, but `bypassOwnerBecauseMentioned` is forced false for DMs. If another account has a more specific direct-peer binding, this handler drops the DM as owned by that account even though the user has no mention bypass path and the other account may never receive this DM event.
  - confidence: 0.9
- **[P1] Filter owner candidates to active configured accounts:** `extensions/discord/src/monitor/route-owner.ts:35-41`
  - body: `listDiscordCandidateAccountIds` includes `listBoundAccountIds`, so a stale binding for an account that is no longer configured or running can win ownership. Active accounts would then drop the message as owned by a nonexistent account, leaving the channel with no responder.
  - confidence: 0.86
- **[P1] Use native Discord targets for owner bypass:** `extensions/discord/src/monitor/message-handler.preflight.ts:85-100`
  - body: Adding `botUserName` regexes makes plain text count as an explicit owner bypass and normal mention. The canonical routing issue requires native `mentions[]` and reply metadata because text-name matching can target the wrong bot and conflict with configured mention patterns.
  - confidence: 0.82
- **[P2] Move route-owner imports onto plugin SDK seams:** `extensions/discord/src/monitor/route-owner.ts:1-8`
  - body: The new production module imports config and routing internals through `../../../../src/**`, but `extensions/AGENTS.md` requires bundled plugin production code to use `openclaw/plugin-sdk/*` or local barrels. Reintroducing core-relative imports conflicts with the current extension boundary and import guards.
  - confidence: 0.84
