## Summary

Keep open: the underlying Discord multi-bot routing problem still exists on current main, but this PR is not mergeable as-is because it introduces direct-message, stale-account, and plugin-boundary regressions and still lacks inspectable real behavior proof.

## What This Changes

The PR adds Discord route-owner resolution across accounts, explicit and bot-name mention bypasses, bot identity fallback behavior, and focused Discord monitor tests.

## Best Possible Solution

Land one canonical Discord-owned routing repair that derives targeted bot accounts from Discord mention/reference metadata before default routing, suppresses non-target default owners, uses plugin SDK seams, and proves behavior with focused tests plus redacted real Discord evidence.

## Reproduction Assessment

Yes, at source level. Current main routes before cross-bot target metadata and compares mention/reply state only to the running bot ID, while the PR diff exposes concrete direct-message and stale-account regressions.

## Solution Assessment

No, not as-is. The direction is useful, but the patch should exclude DMs from owner blocking, ignore inactive accounts, use plugin SDK/public seams, and reconcile text-name versus Discord metadata semantics with the canonical routing issue.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:442`
  - body: The new owner check runs for DMs because it only gates on thread/configured binding state, while `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding for the same sender, this bot drops a legitimate DM even though the other account will not receive this DM event.
  - confidence: 0.91
- **[P2] Filter owner candidates to active accounts:** `extensions/discord/src/monitor/route-owner.ts:35-40`
  - body: `resolveDiscordRouteOwner` includes account IDs found only in bindings. A stale binding for a removed or disabled Discord account can win ownership and make active accounts drop the channel message for a handler that is not running.
  - confidence: 0.88
- **[P2] Import routing through plugin SDK seams:** `extensions/discord/src/monitor/route-owner.ts:1-8`
  - body: This new Discord production module imports config, routing, and session-key internals with `../../../../src/**` paths. Extension production code should use `openclaw/plugin-sdk/*` or a documented local/public seam, otherwise the bundled plugin depends on private core layout and violates the extension boundary.
  - confidence: 0.86
