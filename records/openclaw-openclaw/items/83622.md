## Summary

Keep open for maintainer review: the patch is coherent, default-off, covered by focused tests, and now has visible Telegram proof, but it intentionally changes Telegram DM progress delivery when enabled and needs Telegram owner acceptance before merge.

## What This Changes

This PR adds a default-off Telegram DM `sendMessageDraft` path for transient tool-progress drafts, with Telegram-scoped config/schema/docs and regression coverage.

## Best Possible Solution

Land only after a Telegram owner accepts the default-off native-draft delivery policy; keep final answers on the persistent message path and preserve the fallback/guardrail tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new optional Telegram transport feature. Current main has edited-preview tool progress only, while the PR and recording demonstrate a new native-draft path behind config.

## Solution Assessment

Yes, if maintainers want this feature: the PR keeps native drafts default-off, DM-only, allowlist-capable, and out of answer/final delivery. The safer alternative is to keep all Telegram progress on edited previews if maintainers do not want any native draft transport yet.
