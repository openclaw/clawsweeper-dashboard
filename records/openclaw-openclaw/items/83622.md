## Summary

## What This Changes

This PR adds an opt-in Telegram native `sendMessageDraft` path for transient private-DM tool-progress updates, new `streaming.preview` config fields, Telegram docs/maintainer-note updates, and regression tests.

## Best Possible Solution

Keep the native draft lane Telegram-scoped, default-off, and canary-gated, then land only after redacted live Telegram proof shows ephemeral progress before a normal persistent final answer.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new optional Telegram transport feature. Source inspection confirms current main only uses the edited preview stream for tool progress, while live native-draft behavior still needs inspectable proof.

## Solution Assessment

No, not yet. The runtime gating mostly matches the maintainer boundary, but the config/schema surface should be Telegram-specific instead of shared across channels before this is the cleanest solution.
