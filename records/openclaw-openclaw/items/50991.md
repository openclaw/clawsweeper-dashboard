## Summary

Keep open: current main still leaves bundled session-memory scoped to `/new` and `/reset`, while `/compact` emits a separate compaction lifecycle event and the related continuity-helper PR does not wire compact-time archive capture.

## What This Changes

Request to trigger the bundled session-memory archive when a session is compacted so valuable pre-compaction context is preserved.

## Best Possible Solution

Define one compact-time memory-continuity contract, likely on `session:compact:before`, then wire session-memory through it with the needed workspace/session context, duplicate-write guards, tests, and docs.

## Reproduction Assessment

Yes, source-level. Current main shows session-memory only accepts `command:new` and `command:reset`, while `/compact` goes through the compaction runtime and `session:compact:before` lifecycle without writing the requested archive.

## Solution Assessment

No for the literal `command:compact`-only proposal. If maintainers want this behavior, `session:compact:before` or a dedicated memory lifecycle contract is the maintainable hook point so manual, automatic, and engine-owned compactions share one path.
