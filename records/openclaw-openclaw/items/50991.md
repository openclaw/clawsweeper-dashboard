## Summary

Keep open: current main still implements the bundled session-memory archive only for `/new` and `/reset`; compaction emits lifecycle hooks and has separate memory-flush machinery, but it does not write the requested session-memory archive before `/compact`.

## What This Changes

The issue requests adding compact-time session-memory capture so valuable conversation context is archived before session compaction.

## Best Possible Solution

Define one compact-time memory-continuity contract, likely on `session:compact:before` or a dedicated memory lifecycle hook, then wire session-memory through it with tests, docs, duplicate guards, and data-retention boundaries.

## Reproduction Assessment

Yes, source-level. Current main shows session-memory only accepts `command:new` and `command:reset`, while `/compact` goes through the compaction runtime and `session:compact:before` without writing the requested archive.

## Solution Assessment

No for the literal `command:compact`-only proposal. If maintainers want this behavior, a compaction lifecycle or memory-continuity contract is the maintainable hook point so manual, automatic, and engine-owned compactions share one path.
