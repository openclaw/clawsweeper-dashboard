## Summary

Keep open. Current main still keeps the bundled session-memory hook scoped to `/new` and `/reset`; the existing compaction lifecycle is available, but it is not wired to the session-memory archive path, and the related backup/open-memory discussions do not provide a merged native fix.

## What This Changes

The issue requests that session-memory archival run when sessions are compacted so pre-compaction context is saved before transcript compression.

## Best Possible Solution

Wire session-memory to the existing session:compact:before lifecycle, pass the minimal transcript/workspace/config context needed to write one pre-compaction archive, and update metadata, docs, and regression tests without adding a new config option.

## Reproduction Assessment

Yes. A high-confidence source path exists: inspect the bundled session-memory hook, follow `/compact` through handleCompactCommand, and compare it with the separate session:compact:before lifecycle emitted by the compaction runtime.

## Solution Assessment

No for the literal command:compact-only proposal. The narrower maintainable solution is to use session:compact:before so manual, gateway, and automatic compaction share one pre-compaction archival path.
