## Summary

Keep open: current main and v2026.6.10 still limit the bundled session-memory archive hook to `/new` and `/reset`. Compaction has `session:compact:before/after`, memoryFlush, and post-index sync surfaces, but none currently implement the requested session-memory archive capture; the KeepMyClaw comment is useful complementary backup context, not an OpenClaw core implementation.

## What This Changes

Request to make the bundled session-memory hook save conversation context when compaction runs, instead of only on `/new` or `/reset`.

## Best Possible Solution

Define the compact-time archival contract, then wire session-memory through `session:compact:before` with metadata, docs, regression tests, and explicit retention/dedupe behavior if maintainers approve it.

## Reproduction Assessment

Yes, at source level: session-memory metadata and handler logic only register and accept `/new` and `/reset`, while compaction currently emits `session:compact:before/after`. I did not run a live compaction because this review is read-only.

## Solution Assessment

Unclear as proposed: the preservation gap is real, but `command:compact` is not the current lifecycle surface. The safer path is a maintainer-approved `session:compact:before` session-memory contract rather than a new command event alias.
