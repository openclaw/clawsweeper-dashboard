## Summary

Keep open. Current main and v2026.5.20 still scope the bundled session-memory archive to `/new` and `/reset`; `/compact` runs through the compaction runtime and emits separate compaction lifecycle hooks that do not currently write the requested session-memory archive. The discussion's KeepMyClaw link is useful as an external backup/snapshot comparison, but it does not satisfy the requested OpenClaw session-memory behavior.

## What This Changes

The issue requests that session-memory preservation also run at compact time so valuable conversation context is captured before compaction rewrites or summarizes the transcript.

## Best Possible Solution

Define one compact-time memory-continuity contract, likely on `session:compact:before` or a dedicated memory lifecycle surface, then wire session-memory through it with required context, duplicate-write guards, focused tests, and docs.

## Reproduction Assessment

Yes, source-level. Current main shows session-memory only accepts `command:new` and `command:reset`, while `/compact` goes through compaction runtime and `session:compact:before` without writing the requested archive.

## Solution Assessment

No for the literal `command:compact` event-list-only proposal. If maintainers want this behavior, a compaction lifecycle or memory-continuity contract is the maintainable hook point so manual, automatic, and engine-owned compactions share one path.
