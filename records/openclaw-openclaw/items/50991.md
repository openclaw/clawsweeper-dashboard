## Summary

Keep open. Current main has generic compaction lifecycle hooks and pre-compaction memory-flush support, but the bundled session-memory hook still only registers and handles `/new` and `/reset`; `/compact` does not currently produce a session-memory archive through that hook. The KeepMyClaw comment is useful as complementary backup context, but it does not implement the requested native OpenClaw preservation path.

## What This Changes

The issue requests that session-memory archival run when `/compact` is invoked so the pre-compaction conversation is saved without requiring `/new` or `/reset`.

## Best Possible Solution

Keep this as the compact-specific session-memory tracker. The desired end state is a narrow hook-compatible path that saves the active transcript before compaction, preferably through the existing `session:compact:before` lifecycle so text commands and Gateway compaction use the same behavior.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: enable or inspect the bundled session-memory hook, follow `/compact` through `handleCompactCommand`, and observe that it calls compaction directly while the hook handler only writes for `command` actions `new` and `reset`.

## Solution Assessment

No for the literal `command:compact` proposal as the only fix. The better maintainable direction is to adapt session-memory to the existing pre-compaction lifecycle event and pass the missing context needed to reuse the current writer.
