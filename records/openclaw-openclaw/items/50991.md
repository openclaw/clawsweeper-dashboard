## Summary

Keep open. Current main still leaves the bundled session-memory hook limited to `/new` and `/reset`; compaction lifecycle hooks exist, but session-memory is not registered for or able to consume them. The KeepMyClaw discussion is useful backup context, but it does not satisfy the requested native OpenClaw hook path.

## What This Changes

Request to make session-memory archive the active conversation when compaction runs, instead of only when new/reset session commands run.

## Best Possible Solution

Wire session-memory to the existing `session:compact:before` lifecycle with the transcript/workspace/config context it needs, then update bundled hook metadata, docs, and regression tests.

## Reproduction Assessment

Yes. Static source reproduction is high-confidence: enable or inspect the bundled session-memory hook, follow `/compact` into `handleCompactCommand`, and observe that compaction emits lifecycle events while the session-memory handler only accepts `command:new` and `command:reset`.

## Solution Assessment

No for the literal `command:compact`-only proposal. The more maintainable fix is to use the existing `session:compact:before` lifecycle so manual, gateway, and automatic compaction share one pre-compaction archival path.
