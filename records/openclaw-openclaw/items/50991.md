## Summary

Keep open. Current main and the latest release still keep the bundled session-memory archive on `/new` and `/reset` only; compaction has lifecycle hooks and memoryFlush support, but neither is the requested raw session-memory archive on compact.

## What This Changes

The issue requests firing the session-memory hook when compaction runs so valuable session context is archived before transcript compression.

## Best Possible Solution

Decide the memory-continuity contract first; if approved, wire session-memory through session:compact:before with the needed workspace/session context, focused tests, and docs while avoiding duplicate writes with memoryFlush/recent continuity.

## Reproduction Assessment

Yes for a source-level behavior gap. Current main shows session-memory only accepts command:new and command:reset, while `/compact` goes through the compaction runtime and session:compact:before lifecycle without writing the requested archive.

## Solution Assessment

No for the literal command:compact-only proposal. If maintainers want this behavior, session:compact:before is the maintainable hook point so manual, automatic, and engine-owned compaction share one pre-compaction archival path.
