## Summary

Keep open: current main still does not trigger the bundled session-memory archive on compact, and the adjacent automatic memory-flush and compaction lifecycle paths do not provide the same `/new`/`/reset` archive behavior. The remaining work needs a maintainer decision on the compact-time memory lifecycle contract before automation changes transcript persistence.

## What This Changes

Add compact-time triggering for the bundled session-memory archive so pre-compaction context is saved without requiring `/new` or `/reset`.

## Best Possible Solution

Define a compact-time memory preservation contract, likely on `session:compact:before` with the session context session-memory needs, then wire archive behavior through it with duplicate guards, docs, and retention bounds.

## Reproduction Assessment

Yes, source-level: current main registers and filters session-memory to `/new` and `/reset`, while `/compact` uses the compaction lifecycle path and does not write the requested archive.

## Solution Assessment

No for the literal one-line `command:compact` proposal; a lifecycle-based solution is likely safer so manual, automatic, and native/Codex compactions share one preservation contract.
