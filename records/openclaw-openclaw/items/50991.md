## Summary

Keep open. Current main still does not make the bundled session-memory archive run on compaction; it has a separate pre-compaction memory-flush mechanism, but that is not the same raw archive path requested here.

## What This Changes

Request to make the bundled session-memory archive save conversation context at compaction time before history is summarized.

## Best Possible Solution

Decide whether session-memory should complement the existing memory-flush system; if yes, wire it to session:compact:before with minimal transcript/workspace/config context and focused docs/tests.

## Reproduction Assessment

Yes for the behavior gap: source inspection shows session-memory handles only command:new and command:reset, while /compact goes through the separate compaction runtime and session:compact:before lifecycle. I did not run a live compaction because this read-only review can prove the missing wiring from current main.

## Solution Assessment

No for the literal command:compact-only proposal. If maintainers want this beyond the existing memory flush, the better solution is to use session:compact:before so manual, automatic, and engine-owned compaction share one pre-compaction archival path.
