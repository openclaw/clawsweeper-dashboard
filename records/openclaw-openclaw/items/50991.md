## Summary

Keep open: current main and the latest release still limit the bundled session-memory archive to `/new` and `/reset`; `/compact` uses the compaction lifecycle path and does not write the requested session-memory archive. The related continuity PR is a helper-layer slice only, so this still needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue requests adding a compact-time session-memory trigger so conversation context is recorded before compaction changes the transcript.

## Best Possible Solution

Define a compact-time memory preservation contract, likely on `session:compact:before` or a dedicated memory lifecycle hook, then wire session-memory through it with the needed context, duplicate guards, docs, tests, and transcript-retention boundaries.

## Reproduction Assessment

Yes, source-level. Current main shows session-memory only accepts command `new` and `reset`, while `/compact` goes through the compaction runtime and emits `session:compact:before` without writing the requested archive.

## Solution Assessment

No for the literal one-line `command:compact` proposal. If maintainers want this behavior, a compaction lifecycle or memory-continuity contract is the maintainable hook point so manual, automatic, and engine-owned compactions share one path.
