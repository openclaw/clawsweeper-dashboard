## Summary

Keep open: current main and the latest release still register the bundled session-memory hook only for `/new` and `/reset`, while compaction emits separate `session:compact:*` lifecycle events. Related continuity work is useful context, but no merged or open canonical PR currently implements this exact compact-time session-memory archive behavior.

## What This Changes

The issue requests firing the session-memory hook when `/compact` runs so transcript-derived memory is captured before compaction changes session context.

## Best Possible Solution

Define the compact-time memory contract first, then wire session-memory or a narrower continuity hook through the existing `session:compact:before` lifecycle without changing the current `/new` and `/reset` defaults.

## Reproduction Assessment

Yes for source behavior: current main registers and handles session-memory only for `/new` and `/reset`, while compaction emits `session:compact:before/after` lifecycle events. This is a feature gap rather than a broken documented contract.

## Solution Assessment

Unclear pending maintainer direction: firing session-memory on compact is plausible, but the best shape depends on whether compact should write full archive notes, recent continuity snapshots, or rely on the existing memoryFlush path.
