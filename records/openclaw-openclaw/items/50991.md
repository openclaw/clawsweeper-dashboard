## Summary

Keep open: current main and the latest release still limit the bundled session-memory archive hook to `/new` and `/reset`, while manual and Gateway compaction use the `session:compact:before/after` lifecycle instead of a `command:compact` event, and the only related continuity PR was closed unmerged.

## What This Changes

Request to trigger bundled session-memory archive capture before `/compact` compacts the active conversation.

## Best Possible Solution

Decide the compact-time memory archival contract, then wire session-memory through `session:compact:before` with focused handler, metadata, docs, and regression coverage if maintainers want this automatic persistence.

## Reproduction Assessment

Yes, at source level: session-memory metadata and handler logic only register and accept `/new` and `/reset`, while compaction currently emits `session:compact:before/after` instead. I did not run a live compaction because this review is read-only.

## Solution Assessment

Unclear as proposed: the preservation gap is real, but `command:compact` is not the current lifecycle surface. The maintainable fix is likely a maintainer-approved `session:compact:before` session-memory contract rather than a new command event alias.
