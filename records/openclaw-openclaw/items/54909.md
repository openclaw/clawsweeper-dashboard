## Summary

Keep open. Current main still forwards generic Telegram callback_query fallback data to the agent as the raw callback payload, while the Telegram docs promise the `callback_data: <value>` text form. The linked PR #54962 remains open and unmerged with closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

Route Telegram inline button callback_query fallback payloads to the agent as documented `callback_data: <value>` text so configured instructions can trigger real tool calls instead of acting on raw callback text.

## Best Possible Solution

Review and land PR #54962, or apply the same narrow change in `extensions/telegram/src/bot-handlers.runtime.ts` so only generic Telegram callback fallback data is forwarded as `callback_data: ${data}`. Add a regression test for generic callback prefixing and close this issue after the fix merges.
