## Summary

Keep open. Current main still routes generic Telegram callback_query fallback text to the agent as the raw callback payload, while the public Telegram docs promise the `callback_data: <value>` text form; the linked fix PR is still open and unmerged, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue reports that Telegram inline button callbacks are forwarded without the documented `callback_data:` prefix, preventing agent instructions from reliably triggering the configured HTTP tool.

## Best Possible Solution

Review and land PR #54962, or apply the same targeted fix in `extensions/telegram/src/bot-handlers.runtime.ts`, with regression coverage that prefixes only generic callback fallback text while preserving internal callback handlers.

## Reproduction Assessment

Yes. Source inspection on current main shows generic callbacks are routed as raw `data`, and the existing unit test asserts raw payload delivery instead of the documented `callback_data:` form.

## Solution Assessment

Yes. Prefixing only the generic callback fallback is the narrowest maintainable fix because the callback subscription and special internal callback handlers already exist.
