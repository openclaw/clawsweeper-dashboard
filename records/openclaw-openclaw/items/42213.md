## Summary

Keep open. Current main still has a source-reproducible stale `chatRunId` path that can keep queued Control UI messages blocked, and the same-author implementation candidate at https://github.com/openclaw/openclaw/pull/42258 is still open rather than merged.

## What This Changes

The issue reports that Control UI chat can remain busy after a missed terminal chat event, leaving queued messages unflushed until another prompt nudges the session.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/42258, or replace it with an equivalent Control UI watchdog that uses authoritative `agent.wait` terminal results before clearing busy state and flushing the queue.

## Reproduction Assessment

Yes, at source level. Leave `chatRunId` non-null after a missed terminal chat event, submit a normal message, and the message queues while `flushChatQueue` returns because `isChatBusy` remains true.

## Solution Assessment

Yes, directionally. A focused Control UI `agent.wait` recovery path is the narrow maintainable fix, but the open branch needs rebase, review fixes, and real browser/gateway proof before it should close this issue.
