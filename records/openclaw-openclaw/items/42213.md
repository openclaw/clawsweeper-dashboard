## Summary

Keep open. Current main still relies on matching chat terminal events to clear `chatRunId` and flush the queue, and the same-author implementation candidate #42258 remains open and unmerged.

## What This Changes

The issue requests automatic recovery for stale Control UI chat runs so queued messages flush after a long `chat.send` finishes without a terminal UI event.

## Best Possible Solution

Review and land #42258, or an equivalent narrow Control UI recovery path that probes `agent.wait` after inactivity and only clears busy state from authoritative terminal results.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: leave `chatRunId` non-null, submit a normal message, and the UI enqueues it while `flushChatQueue` returns because the stale busy latch never clears without a matching terminal event.

## Solution Assessment

Yes. The linked `agent.wait` watchdog direction is the narrowest maintainable fix because it uses an existing backend contract and does not require changing chat protocol or queue semantics.
