## Summary

Keep open. Current main still has the reported stale-busy Control UI shape, and the same-author implementation candidate #42258 is open and unmerged, so this issue should remain open until that PR or an equivalent fix is resolved.

## What This Changes

The issue asks the Control UI chat to recover from stale busy runs and flush queued messages after gateway-side `chat.send` completion.

## Best Possible Solution

Review and land #42258, or an equivalent narrow Control UI watchdog that probes `agent.wait` after inactivity, clears busy state only on authoritative terminal results, and adds regression coverage for queued-message flushing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: leave `chatRunId` non-null, submit another normal message, and the UI enqueues it while `flushChatQueue` returns because the stale busy latch never clears without a matching terminal event.

## Solution Assessment

Yes. The linked `agent.wait` watchdog direction is the narrowest maintainable fix because it uses an existing backend contract and does not require changing chat protocol or queue semantics.
