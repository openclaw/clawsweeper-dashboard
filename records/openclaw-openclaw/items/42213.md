## Summary

Keep open. Current main still has a source-reproducible stale `chatRunId` path that can keep Control UI chat busy and block queued-message flushing, and the same-author closing PR remains open/unmerged.

## What This Changes

Report asks Control UI chat to recover automatically from a stale `chatRunId` after missed terminal events so queued messages flush when the gateway run has finished.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/42258, or replace it with an equivalent Control UI recovery path that uses authoritative `agent.wait` terminal results before clearing busy state and flushing the queue.

## Reproduction Assessment

Yes, at source level. Leave `chatRunId` non-null after a missed terminal chat event, submit a normal message, and the message queues while `flushChatQueue` returns because `isChatBusy` remains true.

## Solution Assessment

Yes, directionally. A focused Control UI `agent.wait` recovery path is the narrow maintainable fix, but the current linked branch still needs rebase, review, and real browser/gateway proof before it should close this issue.
