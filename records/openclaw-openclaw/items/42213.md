## Summary

Keep open. Current main still has the stale Control UI busy-state path, and open PR #42258 is already the same-author implementation candidate with closing intent.

## What This Changes

Recover Control UI chat from stale `chatRunId` after long `chat.send` runs so queued user messages flush automatically.

## Best Possible Solution

Review and land #42258, or an equivalent narrow Control UI watchdog that probes `agent.wait` after inactivity and clears busy state only from authoritative terminal backend status.

## Reproduction Assessment

Yes at source level. Leave `chatRunId` non-null, submit a normal message, and the UI enqueues it while `flushChatQueue` returns until a matching terminal chat event clears the run id.

## Solution Assessment

Yes. The `agent.wait` watchdog direction is the narrowest maintainable fix because the backend already supports waiting on `chat.send` runs that finish without lifecycle events, without changing chat protocol or queue semantics.
