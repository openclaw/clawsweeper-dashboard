## Summary

Keep open. Current main still has a source-reproducible stale `chatRunId` path that can keep Control UI chat busy and block queue draining, and the same-author closing PR at https://github.com/openclaw/openclaw/pull/42258 is still open and unmerged.

## What This Changes

The issue reports that Control UI chat can remain busy after a missed terminal event, leaving queued user messages unflushed until another prompt or run unsticks the session.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/42258, or replace it with an equivalent Control UI recovery path that uses authoritative `agent.wait` terminal results before clearing busy state and flushing the queue.

## Reproduction Assessment

Yes, at source level. Leave `chatRunId` non-null after a missed terminal chat event, submit a normal message, and the message queues while `flushChatQueue` returns because `isChatBusy` remains true.

## Solution Assessment

Yes, directionally. A focused Control UI `agent.wait` recovery path is the narrow maintainable fix, but the linked branch still needs rebase, review, and real browser/gateway proof before this issue should close.
