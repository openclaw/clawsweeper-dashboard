## Summary

Keep open. Current main still has a source-reproducible stale busy latch path, and same-author PR #42258 remains open with closing intent for this issue.

## What This Changes

The issue requests automatic Control UI chat recovery when a long-running `chat.send` has already finished on the gateway but the UI missed the terminal event and left queued messages blocked.

## Best Possible Solution

Review and land #42258, or an equivalent narrow Control UI recovery path that probes `agent.wait` after inactivity and only clears busy state from authoritative terminal results.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: leave `chatRunId` non-null, submit a normal message, and `flushChatQueue` returns because the stale busy latch never clears without a matching terminal event.

## Solution Assessment

Yes. The linked `agent.wait` watchdog direction is the narrowest maintainable fix because it uses an existing backend contract without changing chat protocol or queue semantics.
