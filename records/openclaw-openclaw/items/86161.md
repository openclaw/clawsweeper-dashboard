## Summary

Keep open: current main and v2026.5.22 still route Telegram message-tool edits through text editing and require content, so the reported media caption and reply-markup edit gap remains.

## What This Changes

The issue requests Telegram message edit support for media captions and inline keyboard reply markup instead of always using text editing.

## Best Possible Solution

Fix the Telegram plugin edit path so presentation-only edits use reply-markup editing, media caption edits use caption editing, and text messages keep the existing text edit behavior.

## Reproduction Assessment

Yes from source inspection, though I did not run a live Telegram repro. Current main requires content/message for edit and then calls editMessageText, while Telegram and grammy expose separate caption and reply-markup edit methods.

## Solution Assessment

Yes, the requested direction is the right owner-boundary fix if kept inside the Telegram plugin and covered by focused tests. The safer implementation should preserve text-edit behavior and route only unambiguous presentation-only or media-caption cases to the other Telegram APIs.
