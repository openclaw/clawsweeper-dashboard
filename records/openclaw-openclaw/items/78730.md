## Summary

Keep open. Current main and the latest release still have the oversized-line text-loss path, but this PR head has a correctness issue and lacks required real behavior proof, so it should not merge as-is.

## What This Changes

The PR changes Gateway transcript tail parsing to recover text blocks from oversized multimodal JSONL lines, append an omitted-image placeholder, and add a regression test.

## Best Possible Solution

Preserve recoverable text from oversized content arrays, append media-specific placeholders only when omitted media is detected, keep a neutral oversized marker otherwise, and include real redacted chat.history proof.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.7 shows a JSONL transcript line over 256 KiB with text plus inline image data hits the oversized-line fallback and emits only the generic placeholder before chat.history projection.

## Solution Assessment

No. The PR direction is narrow, but appending the image placeholder for any recovered text block is not the best fix; it should first detect omitted image/media content or use a neutral oversized marker.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
