## Summary

This PR is a plausible fix for a real gateway history bug, but it should stay open because the patch can fabricate an image placeholder for oversized non-image content and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes Gateway recent transcript tail parsing to recover text blocks from oversized multimodal JSONL entries, append an omitted-image placeholder, and add a regression test for `readRecentSessionMessagesAsync`.

## Best Possible Solution

Keep the PR open, make the omitted-media placeholder conditional on an actual omitted image/media payload or use a neutral oversized marker, add the changelog entry, and verify with real `chat.history` output.

## Reproduction Assessment

Yes. A transcript JSONL line with `content` containing a normal text block plus a 300 KiB inline image crosses `MAX_TRANSCRIPT_PARSE_LINE_BYTES`, and current `main` routes it to the whole-message oversized placeholder by source inspection.

## Solution Assessment

No. The direction is narrow, but the current patch should only emit the image placeholder when an image/media block was actually omitted, and it still needs changelog plus real behavior proof before merge.
