## Summary

Keep open: this focused PR addresses a real Gateway `chat.history` bug, but the patch can still fabricate an image placeholder whenever any text is recovered and it lacks contributor real behavior proof.

## What This Changes

The branch adds an oversized-transcript fallback that extracts text blocks from oversized multimodal JSONL lines, adds an image placeholder, and covers the text-plus-image case in a Gateway session-utils test.

## Best Possible Solution

Preserve recoverable text from oversized content arrays while appending a media placeholder only when omitted media is actually detected, keep a neutral placeholder otherwise, and verify with real `chat.history` output.

## Reproduction Assessment

Yes. Source inspection shows a transcript JSONL line over 256 KiB with text plus inline image data enters the oversized-line fallback before `chat.history` can preserve adjacent text; the PR-introduced placeholder bug is also source-reproducible with an oversized text-only content array.

## Solution Assessment

No. The direction is the right narrow Gateway boundary, but the current diff should distinguish omitted media from other oversized content and needs real after-fix behavior proof before merge.
