## Summary

Keep open. Current main still has the reported Gateway history bug, but this PR needs a correctness fix because it appends an image placeholder whenever any text is recovered, and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR recovers text blocks from oversized multimodal transcript lines, appends an omitted-image placeholder, and adds a regression test for recent session history replay.

## Best Possible Solution

Preserve recoverable text from oversized content arrays while replacing only confirmed oversized media blocks, keep a neutral placeholder for other oversized records, and verify with real `chat.history` output.

## Reproduction Assessment

Yes. Source inspection shows a transcript JSONL line over 256 KiB with text plus a large inline image enters the oversized-line fallback and becomes a whole-message placeholder before `chat.history` can preserve the text; I did not execute a live repro because this review is read-only.

## Solution Assessment

No. The direction is narrow, but the patch should only emit the image/media placeholder when it actually detects omitted media; otherwise it can fabricate an image in `chat.history`.
