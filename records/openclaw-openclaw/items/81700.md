## Summary

Keep open. This MEMBER draft PR is protected by the `maintainer` label and has useful Codex auth classifier/stale-profile work, but it also removes current-main macOS Keychain sidecar compatibility that was added for legacy Codex `oauthRef` recovery, so it needs explicit maintainer review before merge.

The PR expands Codex OAuth refresh-failure matching and user guidance, rejects stale auto-selected Codex auth profiles, changes legacy OAuth sidecar Keychain handling, and adds auth regression tests plus changelog coverage.

## Best Possible Solution

Land the classifier, copy, and stale-profile fixes after maintainers either preserve the current read-only Keychain migration path or explicitly approve the re-auth-only upgrade behavior; keep the broader Codex probe work tracked in https://github.com/openclaw/openclaw/issues/82710.

## Reproduction Assessment

Yes, at source level: current main lacks several current Codex terminal refresh-token patterns, and the PR adds focused coverage for those messages and stale Codex auth-profile selection. A live revoked-token session was not reproduced.

## Solution Assessment

No, not as a complete merge path: centralizing classifier and stale-profile handling is the right shape, but unconditional Keychain sidecar removal is an upgrade-sensitive policy change. The safer path is to keep the read-only compatibility/migration behavior by default unless maintainers explicitly approve the breaking re-auth behavior with upgrade proof.
