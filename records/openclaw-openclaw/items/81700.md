## Summary

Keep open. This is a draft, protected MEMBER PR with useful Codex OAuth classifier and stale-routing work, but the current branch still removes the read-only macOS Keychain recovery path that current main uses to keep legacy Codex sidecars migratable, so it needs an explicit maintainer upgrade decision before merge.

## What This Changes

The PR expands Codex OAuth refresh-failure classification and guidance, removes legacy OAuth sidecar Keychain recovery, and updates OpenAI/Codex auth-profile routing and bootstrap logic with regression coverage.

## Best Possible Solution

Preserve the read-only Keychain migration path by default, or explicitly approve and document re-auth-only upgrade behavior with focused macOS upgrade proof before landing the classifier and stale-routing fixes.

## Reproduction Assessment

Yes, at source level: current main lacks several current Codex terminal refresh-token patterns and can choose openai-codex from stale auth.order.openai state, while the PR adds focused regression coverage. A live revoked-token session was not reproduced.

## Solution Assessment

No, not as a complete merge path: centralizing the classifier and stale-profile handling is the right shape, but unconditional Keychain sidecar removal is an upgrade-sensitive policy change. The safer path is to keep read-only compatibility migration unless maintainers explicitly approve the breaking re-auth behavior with upgrade proof.
