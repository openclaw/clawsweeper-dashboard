## Best Possible Solution

Land the narrow crypto-random seed change after normal CI and maintainer review, preserving the existing run ID format and import/rollback behavior.

## Reproduction Assessment

## Solution Assessment

Yes. Replacing only the seed entropy with randomBytes(16).toString('hex') is the narrowest maintainable fix because it keeps the existing chatgpt-<12 hex> output and rollback/import record contract intact.
