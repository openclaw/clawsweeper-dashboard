## Summary

Keep open. Current main still honors env and managed proxy routing without a reachability fallback, and the requested silent direct fallback needs maintainer security/product direction because the documented managed-proxy contract is fail-closed.

## What This Changes

Add smart proxy reachability detection so OpenClaw falls back to direct network access when a configured local proxy port is unavailable.

## Best Possible Solution

Define proxy-unavailable behavior in the shared proxy layer: keep managed and explicit proxy modes fail-closed, and consider only a documented opt-in fallback for ambient local proxy-manager env vars with focused tests and docs.

## Reproduction Assessment

Yes, source-reproducible. Set HTTPS_PROXY or HTTP_PROXY to a closed loopback port and exercise resolveProxyFetchFromEnv or fetchWithSsrFGuard; current main selects EnvHttpProxyAgent and rethrows request failures instead of falling back direct.

## Solution Assessment

No as proposed. A gaxios-only probe is too narrow, and silent direct fallback conflicts with the documented fail-closed proxy guardrail; the maintainable path is shared, explicit, and policy-gated.
