## Summary

Keep open. Current main still routes configured or env proxy traffic through proxy dispatchers without a reachability probe or request-time direct retry, while the requested silent direct fallback conflicts with the documented fail-closed proxy contract and needs maintainer policy/security direction.

## What This Changes

Request automatic proxy reachability detection so OpenClaw falls back to direct network access when a configured local proxy port is closed.

## Best Possible Solution

Define proxy-unavailable behavior in the shared proxy layer: keep managed, explicit, and mandatory proxies fail-closed, and consider only an opt-in bounded convenience fallback for ambient local proxy-manager env vars with tests and docs.

## Reproduction Assessment

Yes. Set HTTPS_PROXY or HTTP_PROXY to a closed loopback port and exercise resolveProxyFetchFromEnv or fetchWithSsrFGuard in trusted env proxy mode; source inspection shows the proxy dispatcher is selected and request errors are rethrown rather than retried direct.

## Solution Assessment

No as proposed. A gaxios-only reachability probe is too narrow, and silent direct fallback should not become the default for a documented fail-closed proxy guardrail.
