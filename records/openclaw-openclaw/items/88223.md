## Summary

Keep open. Current main still ties WhatsApp composing presence to the normal reply lifecycle and still sends a composing update immediately before outbound send; the requested immediate-on-receipt behavior and WhatsApp-specific cadence/config decision are not implemented or owned by a canonical related issue.

## What This Changes

The issue requests faster WhatsApp typing feedback and displayed replies by sending composing presence earlier, avoiding redundant pre-send composing, and possibly adding/tuning WhatsApp typing configuration.

## Best Possible Solution

Have maintainers define the WhatsApp-owned responsiveness behavior first, then implement a focused plugin/core change that sends the first composing signal at the earliest safe accepted-inbound point and keeps cadence/defaults aligned with the existing typing configuration model.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection supports the reported shape: current main starts composing through the reply lifecycle, uses a 6-second default refresh cadence, and still sends composing immediately before outbound WhatsApp messages.

## Solution Assessment

Unclear. Moving the first composing update earlier is plausible, but lowering defaults and adding a WhatsApp-specific config option should be decided against the existing global/session typing configuration before implementation.
