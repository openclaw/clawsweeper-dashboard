## Summary

Keep open: current main still has a source-reproducible CLI-backend preview gap, and the issue is already paired with open closing PR #76914. The later discussion usefully links broader streaming-related PRs, but the direct fix should be reviewed and landed before closing this issue.

## What This Changes

The issue requests forwarding CLI-backend assistant deltas into the existing channel preview path so Telegram shows live draft updates before final delivery.

## Best Possible Solution

Review and land or reject #76914; the end state should bridge same-run CLI assistant events into the shared onPartialReply and typing-signal path without adding Telegram-specific CLI logic or new configuration.

## Reproduction Assessment

Yes, source-reproducible: current main parses Claude CLI deltas and emits assistant events, while Telegram preview consumes onPartialReply and the CLI reply-runner branch does not connect those paths. A commenter also reported live reproduction, but I did not run the external Telegram/Claude CLI setup.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix: bridge assistant events in the shared reply-runner path rather than hard-code a Telegram-only workaround. Because #76914 is already open with closing syntax, this issue should remain open until that PR is resolved.
