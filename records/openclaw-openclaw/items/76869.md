## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 is already the direct closing implementation candidate.

## What This Changes

Request to stream Claude CLI backend assistant text deltas into Telegram's live preview path instead of only delivering the final text.

## Best Possible Solution

Land a narrow shared runner bridge that forwards active-run CLI assistant events through the same typing and `onPartialReply` path used by native providers, with regression coverage and no new config.

## Reproduction Assessment

Yes, source-reproducible: a JSONL Claude CLI backend emits assistant agent events, while Telegram draft previews update only from `replyOptions.onPartialReply`; current main does not bridge those during the CLI run. I did not execute the external Telegram/Claude CLI setup.

## Solution Assessment

Yes, the narrow maintainable fix is in the shared reply runner rather than a Telegram-only subscription. PR #76914 is the current implementation candidate, so this issue should stay open until that PR merges or is closed.
