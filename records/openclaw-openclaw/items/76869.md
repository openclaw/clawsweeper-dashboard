## Summary

Keep open: current main still routes CLI assistant deltas only through agent events while Telegram previews consume `onPartialReply`, and open PR #76914 already uses closing syntax for the narrow fix.

## What This Changes

The issue asks for CLI backend assistant deltas to feed the existing Telegram partial-reply preview stream instead of waiting for final delivery.

## Best Possible Solution

Bridge active-run CLI assistant events into the shared partial-reply and typing-signal path with regression coverage, then land #76914 or a narrow replacement.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL CLI backend with Telegram preview streaming; CLI deltas emit as assistant agent events while Telegram drafts only receive `onPartialReply`. I did not run the external Telegram/Claude CLI setup live.

## Solution Assessment

Yes, the narrow maintainable fix is to bridge CLI assistant events in the shared reply runner rather than adding Telegram-specific polling of the event bus. PR #76914 is the active candidate, so this issue should stay open until it merges or is closed.
