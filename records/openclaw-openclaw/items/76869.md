## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 already targets the narrow bridge.

## What This Changes

Forward CLI backend assistant text deltas into Telegram's live preview path instead of showing only the final assembled response.

## Best Possible Solution

Bridge active-run CLI assistant events into the shared partial-preview callback and typing-signal path, with regression coverage proving CLI deltas update Telegram drafts without adding configuration.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL Claude CLI backend and Telegram preview streaming, then CLI deltas emit as assistant agent events while Telegram listens to `onPartialReply`. The discussion also includes an independent reproduction, but I did not run the external Telegram/Claude CLI setup.

## Solution Assessment

Yes, the narrow maintainable fix is to bridge CLI assistant events in the shared reply runner into `onPartialReply` and `signalTextDelta`, rather than adding a Telegram-only listener. Open PR #76914 is the paired implementation candidate, so this issue should stay open until that PR merges or is closed.
