## Summary

Keep open: current main still has the reported CLI preview gap, and the linked open PR is the right implementation path to review before this issue closes.

## What This Changes

The issue requests that Claude CLI backend assistant text deltas feed Telegram's live preview stream instead of only delivering the final message.

## Best Possible Solution

Review and land PR #76914, or an equivalent shared reply-runner bridge that forwards current-run CLI assistant deltas into the existing partial-preview callback path with regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main parses Claude CLI stream-json deltas and emits assistant events, while Telegram preview only receives opts.onPartialReply from the embedded/native callback path. A commenter also reported a live reproduction, though I did not run the external Telegram/Claude CLI setup in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable fix is to bridge current-run assistant events in the shared reply runner to onPartialReply and signalTextDelta. The linked open PR targets that boundary and should be reviewed or landed before this issue closes.
