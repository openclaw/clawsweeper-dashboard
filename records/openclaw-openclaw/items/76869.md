## Summary

Keep open: current main still routes CLI assistant deltas through the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 already uses closing syntax for the narrow fix.

## What This Changes

The issue requests that JSONL CLI backend assistant deltas, especially from `claude-cli`, update Telegram live preview streaming the same way native API partial replies do.

## Best Possible Solution

Bridge active-run CLI assistant events into the shared partial-preview callback and typing-signal path, with regression coverage proving CLI deltas update Telegram drafts without adding configuration.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL CLI backend and Telegram preview streaming, then CLI deltas emit as assistant agent events while Telegram listens to `onPartialReply`. The issue discussion also includes an independent reproduction, but I did not run the external Telegram/Claude CLI setup.

## Solution Assessment

Yes, the narrow maintainable fix is to bridge CLI assistant events in the shared reply runner rather than adding a Telegram-only subscription. PR #76914 is the current implementation candidate, so the issue should stay open until that PR merges or is closed.
