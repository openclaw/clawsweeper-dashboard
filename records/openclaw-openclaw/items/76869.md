## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview consumes `onPartialReply`, and open PR #76914 already targets this issue with closing syntax, so the issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue asks OpenClaw to stream Claude CLI backend assistant deltas into Telegram live preview instead of only delivering the final assembled reply.

## Best Possible Solution

Land PR #76914 or an equivalent narrow bridge from same-run CLI assistant events into the existing `onPartialReply` and typing-signal path, with regression coverage in the reply runner and Telegram preview surface.

## Reproduction Assessment

Yes, source-reproducible: configure a Claude CLI JSONL backend with Telegram preview streaming and current main emits assistant deltas only as agent events while Telegram listens for `onPartialReply`. I did not run the external Telegram/Claude CLI live setup in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is the suggested shared reply-runner bridge, not a Telegram-specific CLI special case. The already open PR #76914 appears to target that path and should be reviewed or replaced rather than queueing a duplicate repair.
