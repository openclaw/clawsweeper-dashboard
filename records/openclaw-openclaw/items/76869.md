## Summary

Keep open: current main still emits CLI assistant deltas only on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 is already the narrow closing implementation candidate. The related PRs linked in discussion cover adjacent Telegram/reasoning/event-sink work, but they do not supersede the current bug before the closing PR lands.

## What This Changes

The issue asks for CLI backend assistant text deltas to reach Telegram live preview streaming the same way native API partial replies do.

## Best Possible Solution

Land or close PR #76914 after review, with the bridge in the shared reply runner and regression coverage proving CLI assistant deltas update channel preview callbacks without new configuration.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a JSONL CLI backend and Telegram preview streaming, then CLI deltas emit as assistant agent events while Telegram only receives draft text through `onPartialReply`; the issue discussion also includes an independent reproduction.

## Solution Assessment

Yes. The narrow maintainable solution is to bridge active-run CLI assistant events into the shared partial-reply and typing-signal path rather than adding a Telegram-only subscription; PR #76914 is the current candidate for that approach.
