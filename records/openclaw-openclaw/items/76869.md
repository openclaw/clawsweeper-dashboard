## Summary

Keep open: current main parses and emits Claude CLI stream-json assistant deltas, but the Telegram draft preview path only consumes the native onPartialReply callback, so the reported CLI preview gap remains source-reproducible.

## What This Changes

Wire Claude CLI JSONL assistant deltas into Telegram live preview streaming so CLI-backed turns show incremental text like native Anthropic API turns.

## Best Possible Solution

Add a generic reply-runner bridge from current-run assistant agent events to the existing partial-preview callback path, with Telegram coverage proving CLI deltas update the draft preview without adding configuration.

## Reproduction Assessment

Yes, source-reproducible: configure a Claude CLI JSONL backend with Telegram preview streaming and the CLI path emits assistant deltas only on the agent-event bus while Telegram listens for onPartialReply. I did not run the live external Telegram/Claude CLI setup during this read-only review.

## Solution Assessment

Yes, the suggested direction is essentially right, but the narrowest maintainable fix should bridge assistant events in the shared reply runner rather than hard-code a Telegram-specific CLI path.
