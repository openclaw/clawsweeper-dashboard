## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 already uses closing syntax for the narrow fix.

## What This Changes

The issue requests that CLI backend assistant deltas, especially `claude-cli` stream-json output, update Telegram live previews instead of appearing only in final delivery.

## Best Possible Solution

Review and land or close PR #76914; the desired end state is a shared runner bridge from active CLI assistant events into `handlePartialForTyping` and `opts.onPartialReply` with regression coverage.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL CLI backend and Telegram preview streaming, then CLI deltas emit as `stream: "assistant"` events while Telegram listens to `onPartialReply`. I did not run the external Telegram and Claude CLI setup, but the source path and issue discussion are consistent.

## Solution Assessment

Yes, the narrowest maintainable fix is to bridge active CLI assistant events in the shared reply runner rather than adding a Telegram-only event subscription. PR #76914 is the current implementation candidate, so this issue should stay open until that PR merges or is closed.
