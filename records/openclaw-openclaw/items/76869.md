## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 is the linked closing implementation candidate.

## What This Changes

Telegram preview streaming should show live assistant text deltas when using `claude-cli` CLI backends instead of waiting for final-only delivery.

## Best Possible Solution

Bridge active-run CLI assistant events in the shared reply runner into `onPartialReply` and typing signals, with regression coverage proving CLI deltas update Telegram drafts without new configuration.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a JSONL CLI backend and Telegram preview streaming, then CLI deltas emit as assistant agent events while Telegram listens to `onPartialReply`.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge CLI assistant events in the shared reply runner; PR #76914 is the active implementation candidate, so this issue should stay open until that PR merges or is closed.
