## Summary

Keep open: current main still has the source-level gap, and the open closing PR #76914 is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

Telegram preview streaming does not receive live text deltas from CLI backends such as `claude-cli`, so users see only the final assembled response.

## Best Possible Solution

Review and land #76914, or an equivalent shared reply-runner bridge, so CLI assistant deltas feed the existing preview callback path with regression coverage.

## Reproduction Assessment

Yes, source-reproducible: a Claude CLI JSONL backend emits assistant deltas on the agent-event bus, while Telegram preview streaming consumes `onPartialReply`; the issue discussion also includes an independent reproduction confirmation.

## Solution Assessment

Yes, the narrow maintainable fix is to bridge current-run CLI assistant deltas in the shared reply runner into the existing `onPartialReply` and typing-signal path, then keep Telegram as a consumer of that generic callback.
