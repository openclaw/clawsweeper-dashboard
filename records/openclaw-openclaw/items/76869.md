## Summary

Keep open: current main parses Claude CLI stream-json assistant deltas and emits them on the agent-event bus, but the Telegram draft preview path is only wired through `onPartialReply`, so the reported CLI preview gap remains source-reproducible.

## What This Changes

Wire Claude CLI backend assistant text deltas into the same live Telegram preview path used by native model streaming.

## Best Possible Solution

Add a generic reply-runner bridge from current-run CLI assistant deltas to the existing partial-preview callback path, with regression coverage proving Telegram draft previews update without new configuration.

## Reproduction Assessment

Yes, source-reproducible: configure a Claude CLI JSONL backend with Telegram preview streaming and the CLI path emits assistant deltas only as agent events while Telegram listens to `onPartialReply`. I did not run the live external Telegram/Claude CLI setup during this read-only review.

## Solution Assessment

Yes, the suggested direction is essentially right, but the narrowest maintainable fix should bridge assistant events in the shared reply runner rather than add a Telegram-specific CLI path.
