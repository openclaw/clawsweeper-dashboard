## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and PR #76914 is an open closing PR for the narrow bridge.

## What This Changes

Request that CLI backend assistant deltas, especially `claude-cli` stream-json output, feed Telegram live preview instead of appearing only in the final reply.

## Best Possible Solution

Bridge active-run CLI assistant events into the shared `onPartialReply` and typing-signal path in `agent-runner-execution.ts`, with regression coverage and no new config; review and land #76914 if it preserves that boundary.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL CLI backend with Telegram preview streaming and CLI deltas emit as assistant agent events while Telegram listens to `onPartialReply`. The issue also has an independent reproduction comment, but I did not run the external Telegram/Claude CLI setup.

## Solution Assessment

Yes, the best fix is the narrow shared-runner bridge described by PR #76914, not a Telegram-only event-bus subscription. That keeps channel preview behavior behind the existing callback contract.
