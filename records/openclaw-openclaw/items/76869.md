## Summary

Keep open: current main still emits CLI assistant deltas only on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and #76914 is an open closing PR for the narrow bridge.

## What This Changes

Telegram users using a JSONL CLI backend such as `claude-cli` want assistant text deltas to update the same live preview stream as native API responses.

## Best Possible Solution

Land a narrow shared reply-runner bridge from active-run CLI assistant events into `onPartialReply` and typing text signals, with regression coverage for CLI deltas reaching Telegram previews.

## Reproduction Assessment

Yes, source-reproducible: a JSONL CLI backend emits assistant deltas as agent events, while Telegram preview updates only from `onPartialReply`. The issue discussion also includes an independent reproduction, but I did not run the external Telegram/Claude CLI setup.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge CLI assistant events in the shared reply runner rather than adding a Telegram-only event-bus subscription; #76914 is the current implementation candidate.
