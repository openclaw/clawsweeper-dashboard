## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`; the already-linked #76914 is the active fix candidate and should be reviewed or closed before this issue is resolved.

## What This Changes

The request is to make Claude CLI backend assistant deltas update Telegram live preview the same way native API partial replies do.

## Best Possible Solution

Review and land #76914, or an equivalent shared reply-runner bridge that forwards same-run CLI assistant events into `onPartialReply` and typing signals without adding Telegram-specific CLI handling.

## Reproduction Assessment

Yes, source-reproducible: configure a Claude CLI JSONL backend with Telegram preview streaming and the CLI path emits assistant deltas only on `AgentEventStream` while Telegram listens to `onPartialReply`. I did not run the live Telegram/Claude CLI integration in this read-only sweep.

## Solution Assessment

Yes, the narrowest maintainable direction is to bridge current-run CLI assistant events in the shared reply runner into the existing partial-preview callback path. A Telegram-only subscriber would solve the symptom but would duplicate channel-specific streaming policy.
