## Summary

Keep open: current `main` still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 is already the closing implementation candidate.

## What This Changes

The issue requests streaming CLI backend assistant text deltas through Telegram preview like native API replies.

## Best Possible Solution

Bridge active-run CLI assistant deltas into the shared partial-preview callback and typing-signal path, with regression coverage proving CLI deltas update Telegram drafts without adding configuration.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL CLI backend and Telegram preview streaming, then CLI deltas emit only as assistant agent events while Telegram listens to `onPartialReply`. The discussion also includes an independent reproduction, but I did not run the external Telegram/Claude CLI setup.

## Solution Assessment

Yes, the narrow maintainable fix is to bridge CLI assistant events in the shared reply runner rather than adding a Telegram-only subscription. PR #76914 is the current implementation candidate, so the issue should stay open until that PR merges or is closed.
