## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and #76914 is an open unmerged closing PR for the narrow bridge.

## What This Changes

The issue reports that CLI backend assistant deltas are not forwarded into Telegram live preview streaming.

## Best Possible Solution

Review and land or close #76914; the desired end state is a shared runner bridge from active CLI assistant events into partial preview callbacks, with regression coverage and no new configuration.

## Reproduction Assessment

Yes. Source inspection shows CLI JSONL deltas emit as assistant agent events, but current main does not bridge those active-run events into `opts.onPartialReply`, while Telegram preview updates only from that callback.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge CLI assistant events in the shared reply runner instead of adding a Telegram-only event-bus subscription; #76914 is the current implementation candidate.
