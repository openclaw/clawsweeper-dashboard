## Summary

Keep open: current main still routes CLI assistant deltas onto the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 is the active closing fix candidate. The related discussion around #74508, #75587, and #71817 is useful context, but #76914 is the narrow PR that directly targets this bug.

## What This Changes

The issue asks for CLI backend assistant text deltas to drive Telegram preview streaming like the native API path.

## Best Possible Solution

Land PR #76914 or an equivalent shared reply-runner bridge that forwards active-run CLI assistant deltas into `onPartialReply` and typing signals with regression coverage.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL `claude-cli` backend with Telegram preview streaming, and CLI deltas emit only as assistant agent events while Telegram listens to `onPartialReply`. The issue discussion also has an independent live reproduction confirmation.

## Solution Assessment

Yes. The best fix is the narrow shared bridge from active-run CLI assistant events into the existing partial-preview callback path; PR #76914 is the current implementation candidate, so this issue should stay open until that PR merges or is closed.
