## Summary

Keep open: current main still leaves CLI assistant deltas on the agent-event bus while Telegram previews consume `onPartialReply`, and PR #76914 is already the open closing implementation candidate.

## What This Changes

The issue requests live Telegram preview updates for CLI-backed Claude responses instead of final-only delivery.

## Best Possible Solution

Bridge active-run CLI assistant events into the shared partial-preview callback and typing-signal path, with regression coverage proving CLI deltas update Telegram drafts without adding configuration.

## Reproduction Assessment

Yes. Source inspection shows a JSONL CLI backend emits assistant deltas as agent events while Telegram preview streaming listens to `onPartialReply`; the issue discussion also has an independent reproduction, though I did not run the external Telegram/Claude CLI setup.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge CLI assistant events in the shared reply runner rather than adding Telegram-specific event-bus subscription logic; PR #76914 is the current implementation candidate.
