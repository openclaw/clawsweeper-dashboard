## Summary

Keep open. Current main still drops the Codex app-server idle-timeout flags before the embedded retry gate sees them; the proposed code path looks functionally aligned with the retry contract, but the PR is missing the required user-facing changelog entry before landing.

## What This Changes

This PR persists timeout and idle-timeout state in `CodexAppServerEventProjector` and marks the run-level wallclock timeout as non-idle in `runCodexAppServerAttempt`.

## Best Possible Solution

Land the timeout flag propagation with a user-facing changelog entry and focused Codex app-server assertions for idle versus wallclock timeout semantics.

## Reproduction Assessment

Yes. Existing `runCodexAppServerAttempt()` tests can simulate a silent app-server turn with very short idle timers; current main returns `timedOut: true` while the projector still reports `idleTimedOut: false`, which prevents the retry gate from engaging.

## Solution Assessment

Yes. Setting the flags at the projector timeout source and marking the outer wallclock timeout as non-idle is the narrowest maintainable fix for the embedded runner contract.
