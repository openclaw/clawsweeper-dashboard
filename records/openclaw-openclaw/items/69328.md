## Summary

Keep this PR open for maintainer review. Current main still lacks both the ACP zero-diff verifier and the direct active-run session.message append path, but the proposed ACP verifier would misclassify real oneshot work that creates only untracked files or runs before the worktree has a HEAD commit.

## What This Changes

The PR adds an ACP oneshot worktree-diff verification gate, persistent-session exemptions, regression tests, and Control UI logic/tests to append active-run session.message payloads directly into chat history.

## Best Possible Solution

Rebase and finish this as a narrow ACP/UI reliability change: make the oneshot verifier account for staged, unstaged, untracked, and unborn-HEAD worktrees before any fail-close terminal status, keep persistent sessions exempt, and reconcile any direct session.message append with the existing final-reload/dedupe design.

## Reproduction Assessment

Yes. A high-confidence static reproduction is an ACP oneshot task whose worktree contains only untracked new files, or a newly initialized repo with no HEAD; the PR verifier returns hasChanges=false and the terminal gate records blocked or failed despite real work.

## Solution Assessment

No. The direction may be useful, but this is not the best mergeable solution until the verifier handles untracked files and unborn HEAD explicitly, and the UI append path is reconciled with current final reconciliation behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Include untracked files in the ACP diff gate:** `src/agents/acp-verification-gate.ts:19-22`
  - body: `git diff --stat HEAD` only reports tracked staged/unstaged changes and errors when `HEAD` is unborn. Because this helper returns `hasChanges: false` on any error, a oneshot ACP run that creates only new files, or runs in a newly initialized repo, is misclassified as zero-diff and then blocked or failed by the new terminal gate.
  - confidence: 0.92
