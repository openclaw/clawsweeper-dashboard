## Summary

Keep this PR open. Current main still lacks the proposed ACP zero-diff verifier and active-run session.message append path, but the PR is not mergeable as-is because the new verifier misses untracked-only worktrees and unborn HEAD repositories, which would create false blocked/failed ACP oneshot results.

## What This Changes

The PR adds an ACP oneshot worktree-diff verification gate with persistent-session exemptions, regression coverage, and Control UI logic/tests to append active-run session.message payloads into chat history.

## Best Possible Solution

Finish this PR as a narrow ACP/UI reliability fix: make the verifier detect staged, unstaged, untracked, and unborn-HEAD worktrees without shelling through user input, keep persistent sessions exempt, and rebase the UI append path onto current final-reload and message-filtering behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is an ACP oneshot task whose worktree contains only untracked new files, or a newly initialized repo with no HEAD; the PR verifier returns `hasChanges=false` and the terminal gate can mark real work blocked or failed.

## Solution Assessment

No. The direction is useful, but this is not the best mergeable solution until the verifier handles untracked files and unborn HEAD explicitly and the UI append path is reconciled with current main’s final reload behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Include untracked and unborn HEAD changes in the diff gate:** `src/agents/acp-verification-gate.ts:19-22`
  - body: `git diff --stat HEAD` only reports tracked changes and fails when `HEAD` does not exist. Because this helper catches that as `hasChanges: false`, a oneshot ACP task that only creates new files, or runs in a fresh repo, is misclassified as zero-diff and then blocked or failed despite real work.
