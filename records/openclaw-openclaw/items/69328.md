## Summary

Keep open: the PR still carries useful ACP and Control UI work that is not fully on current main, but the head should not merge because the new verifier misclassifies valid worktrees, the branch is conflicting, and external real behavior proof is missing.

## What This Changes

The PR adds an ACP worktree diff verifier, changes ACP terminal task outcomes for zero-diff runs, and appends live `session.message` payloads in Control UI during active chat runs.

## Best Possible Solution

Fix the verifier to count tracked, staged, unstaged, untracked, and unborn-HEAD worktrees, then rebase onto current Control UI session reconciliation and add redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. The PR helper calls `git diff --stat HEAD`, treats empty stdout as no changes, and catches git failures as `hasChanges: false`, which misclassifies untracked-only and unborn-HEAD worktrees before ACP terminal status is set.

## Solution Assessment

No. The direction is reasonable, but this implementation should not merge until the local git verifier covers the full worktree state and the branch is rebased against current main.
