## Summary

Keep this PR open: current main does not implement the central ACP/UI behavior, but the proposed verifier would misclassify real worktree changes in common cases and the external PR still lacks real behavior proof.

## What This Changes

The branch adds an ACP worktree-diff verifier for terminal task handling and changes Control UI `session.message` handling to append active-run transcript messages with tests.

## Best Possible Solution

Fix the verifier to count tracked, staged, unstaged, untracked, and unborn-HEAD worktrees, rebase onto current chat/session reconciliation, and add redacted runtime proof before merge.

## Reproduction Assessment

Yes. The PR head's helper calls `git diff --stat HEAD` and catches failures as `hasChanges: false`; a scratch git probe showed untracked-only worktrees produce empty diff output and unborn HEAD exits 128.

## Solution Assessment

No. The direction is reasonable, but this implementation should not merge until the verifier handles untracked and unborn-HEAD cases and the branch is rebased onto current main.
