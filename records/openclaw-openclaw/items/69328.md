## What This Changes

The PR adds ACP background-task worktree diff verification and active-run Control UI `session.message` appending with transcript dedupe tests.

## Best Possible Solution

Fix the verifier to count staged, unstaged, untracked, and unborn-HEAD worktrees, then rebase the UI append path onto current chat history reconciliation and provide real runtime proof before merge.

## Reproduction Assessment

Yes. The blocking defect is source-reproducible because the helper uses `git diff --stat HEAD`, which does not report untracked files and errors before a first commit, then maps both cases to `hasChanges: false`.

## Solution Assessment

No. The direction is maintainable, but this implementation should not merge until the verifier handles untracked and unborn-HEAD worktrees and the UI changes are rebased against current main.
