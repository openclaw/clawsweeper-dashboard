## Summary

Keep open: the PR addresses a real source-reproducible duplicate delivery gap, but the new dedupe cache needs bounded TTL eviction before merge because unique delivered announce keys are retained indefinitely.

## What This Changes

Adds a process-local dedupe/coalesce cache around `deliverSubagentAnnouncement` plus tests for repeated and concurrent subagent completion announces.

## Best Possible Solution

Keep the delivery-boundary dedupe, but make the delivered-result cache self-pruning or bounded while preserving in-flight coalescing and failed-dispatch retry behavior.

## Reproduction Assessment

Yes at source level. Current main builds a stable announce key from child session and run id, then dispatches every `deliverSubagentAnnouncement` invocation; the PR body also supplies a before/after terminal counter proof, though I did not run it locally in this read-only review.

## Solution Assessment

No, not yet. Wrapping `deliverSubagentAnnouncement` is the right narrow boundary, but the delivered-result cache must evict expired unique keys instead of retaining them for the life of the process.
