## Summary

Keep open: this PR targets a real duplicate subagent-completion delivery bug and includes sufficient terminal proof, but the proposed cache is not mergeable as-is because delivered entries are unbounded and the 5-minute TTL still allows the gateway-TTL-rollover duplicate described in the PR body.

## What This Changes

The PR adds a process-local dedupe/coalescing wrapper around `deliverSubagentAnnouncement`, keyed by `directIdempotencyKey`, plus unit and delivery tests for repeat, failed, and concurrent announce dispatches.

## Best Possible Solution

Keep the delivery-boundary dedupe, but make delivered-result suppression bounded and long enough for the full completion retry window while preserving retries for failed dispatches.

## Reproduction Assessment

Yes. Source inspection shows current main builds a stable announce idempotency key and calls the dispatch boundary directly each time; the PR body also supplies before/after terminal output showing two dispatches before the wrapper and one after.

## Solution Assessment

No, not in the current patch form. The delivery boundary is the right place to coalesce same-key races, but the delivered cache needs bounded eviction and suppression that survives the completion retry window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Bound the delivered announce cache:** `src/agents/subagent-announce-dedupe.ts:88`
  - body: Delivered entries are only checked for expiry when the same key is looked up again. Since normal announce keys are unique per child run, most successful entries can remain in `state.delivered` for the process lifetime; add a sweep or size bound when inserting so this process-local cache cannot grow without bound.
  - confidence: 0.91
- **[P2] Cover retries after the gateway TTL rolls over:** `src/agents/subagent-announce-dedupe.ts:16`
  - body: The new delivered cache expires after the same 5 minutes as the gateway dedupe cache. A retry for the same `directIdempotencyKey` after that point can miss both guards and re-enter dispatch during the 30-minute completion retry window, so the gateway-TTL-rollover duplicate case remains possible; extend the suppression duration or use a durable delivered marker and add post-TTL coverage.
  - confidence: 0.86
