## Summary

Keep this PR open, but not ready as written: the duplicate-announcement fix targets a real source-level race and the supplied terminal proof is adequate, but the new delivered-result cache is unbounded and expired unique announce keys are not swept.

## What This Changes

Adds process-local in-flight coalescing and delivered-result caching around `deliverSubagentAnnouncement` keyed by `directIdempotencyKey`, with unit and delivery-boundary tests.

## Best Possible Solution

Keep the boundary-level dedupe, but make the delivered cache bounded or actively swept to match the gateway dedupe TTL/max behavior while preserving retry on false/error outcomes.

## Reproduction Assessment

Yes, source-level. Current main dispatches directly at the announce boundary, and the PR's added delivery-boundary test exercises two same-key Slack-thread announces where the expected fixed behavior is one gateway call.

## Solution Assessment

No as written. The boundary is the right place for this defense-in-depth dedupe, but the implementation needs eviction or a max size before it is safe for a long-running process.
