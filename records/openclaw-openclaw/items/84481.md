## What This Changes

The PR changes `DeviceInfoHelper.platformStringForDisplay()` so the iOS Settings Platform row returns `iPadOS X.Y.Z` on iPad instead of always returning `iOS X.Y.Z`.

## Best Possible Solution

Land this narrow helper change if maintainers want Settings to use the accurate `iPadOS` platform label, with iOS build or simulator validation as the merge gate.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the UI copy change. Reusing the existing idiom switch in `platformString()` is the narrowest maintainable fix for this helper, and `@MainActor` matches the adjacent device-family/platform helpers.
