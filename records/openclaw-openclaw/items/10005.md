## Summary

Keep open. Current main and the latest shipped tag still render chat `/status` and `session_status` through the shared status-card text path, and there is no documented config/schema key for hiding or replacing the card.

## What This Changes

Requests a new configuration option to hide the rendered session status card for chat `/status` and `session_status`, possibly returning JSON-only output.

## Best Possible Solution

Add a backward-compatible, documented status-card visibility contract for chat `/status` and `session_status`, with schema/types/help/docs and focused coverage for the chosen global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, schema, docs, tests, and the latest release all show the requested status-card visibility control is still absent.

## Solution Assessment

Unclear until maintainers choose the public API and output shape. The maintainable direction is to preserve current rendering by default and thread a shared setting through the existing status builders, rather than treating CLI JSON status as a substitute for chat/tool behavior.
