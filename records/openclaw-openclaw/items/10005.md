## Summary

Keep open. Current main still has no documented status-card visibility setting for chat `/status` or the `session_status` tool, and the related privacy/section-suppression issue was already closed into this canonical tracker.

## What This Changes

The issue requests a configuration option to hide or replace the rendered session status card returned by chat `/status` and the `session_status` tool.

## Best Possible Solution

Add a backward-compatible, documented visibility contract for chat `/status` and `session_status`, with schema/types/help/docs and focused tests for the chosen global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence the requested control is absent because both entry points still render status text and config/docs have no hiding setting.

## Solution Assessment

Unclear until maintainers choose the public API. The maintainable direction is a shared visibility setting threaded through the existing status builders while preserving current rendering by default, not treating CLI `openclaw status --json` as a substitute.
