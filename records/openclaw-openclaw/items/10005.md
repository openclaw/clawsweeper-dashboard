## Summary

Keep open. Current main still unconditionally renders the chat `/status` and `session_status` status card, has no status-card visibility config/schema/docs, and the related privacy request was closed into this tracker.

## What This Changes

Request adding a documented configuration option to suppress the rendered session status card for chat `/status` and `session_status`.

## Best Possible Solution

Add a backwards-compatible, documented visibility contract for chat `/status` and `session_status`, with schema/types/help/docs and focused coverage for the chosen global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence the requested control is absent because both entrypoints still render status text and config/docs have no hiding setting.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable direction is a shared visibility setting threaded through the existing status builders while preserving current rendering by default, not treating CLI `openclaw status --json` as a substitute.
