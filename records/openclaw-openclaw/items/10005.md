## Summary

Keep open: current main and the latest release still render chat `/status` and `session_status` through shared status-card text with no documented hide, JSON-only, or section-visibility setting. The closed per-agent privacy report at https://github.com/openclaw/openclaw/issues/69445 was explicitly superseded by this issue, so this remains the canonical tracker for a maintainer product/security decision.

## What This Changes

Request adds a config-controlled way to hide or replace the rendered chat `/status` and `session_status` status card with JSON-only output.

## Best Possible Solution

Define one shared, documented status visibility contract for chat `/status` and `session_status`, preserve the rendered card by default, and cover the chosen schema/help/docs and tests after maintainers settle the privacy/output shape.

## Reproduction Assessment

Yes, as a source-level feature-gap check rather than a broken existing behavior reproduction. Current main and v2026.6.5 both route `/status` and `session_status` through rendered status-card text with no suppression config or documented equivalent.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
