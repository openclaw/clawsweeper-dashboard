## Summary

Keep open: current main still always renders the chat `/status` and `session_status` status-card text, and there is no supported config key or documented JSON-only/hide mode for those surfaces.

## What This Changes

Request adds a configuration option to hide the rendered chat `/status` and `session_status` status card, with current rendering preserved by default.

## Best Possible Solution

Add a documented, backward-compatible status-card visibility setting for chat `/status` and `session_status`, with schema/types/help/docs and focused tests for the selected global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence that the requested feature is absent because both entrypoints still render status text and the config schemas have no hiding setting.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is a shared visibility setting threaded through the existing status builders while preserving current output by default, not treating CLI `openclaw status --json` as a substitute.
