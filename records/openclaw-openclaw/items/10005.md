## Summary

Current main and v2026.6.5 still render the shared status card for both `/status` and `session_status`, with no supported visibility control; this remains the canonical tracker, but the implementation needs a maintainer product/privacy decision.

## What This Changes

Add a configurable way to hide or reduce the rendered session status card returned by chat `/status` and the `session_status` tool.

## Best Possible Solution

Define one documented status-disclosure policy that preserves current behavior by default, supports appropriate field-level privacy, and specifies separate presentation and structured-data semantics for `/status` and `session_status`.

## Reproduction Assessment

Yes at source level: both supported status entry points return the shared rendered status text, and no current visibility setting or alternate output contract exists.

## Solution Assessment

Unclear; the proposed boolean is plausible but is not yet the best proven solution because whole-card suppression, selective privacy, and structured tool output have different UX and security effects.
