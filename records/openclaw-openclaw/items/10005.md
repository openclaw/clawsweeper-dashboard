## Summary

Keep open. Current main still renders chat `/status` and `session_status` through the shared status-card text path, there is no supported config/docs/schema visibility setting, and the related privacy request is already closed as superseded by this issue.

## What This Changes

Request adds a configuration option to hide or JSON-only render chat `/status` and `session_status` status cards.

## Best Possible Solution

Add a documented, backwards-compatible visibility contract for chat `/status` and `session_status`, with schema/types/help/docs and focused coverage for the chosen global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence that the requested feature is absent because both entrypoints still render status text and the config schemas/docs have no hiding setting.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is a shared visibility setting threaded through the existing status builders while preserving current output by default, not treating CLI `openclaw status --json` as a substitute.
