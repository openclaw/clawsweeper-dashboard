## Summary

Keep open: current main still always renders the shared chat `/status` and `session_status` status-card text with no documented hide, JSON-only, or section-visibility control. The related privacy-focused request at https://github.com/openclaw/openclaw/issues/69445 and CLI JSON request at https://github.com/openclaw/openclaw/issues/13450 were both closed in favor of this tracker, so this remains the canonical product/security decision.

## What This Changes

Request a configuration option to suppress the rendered `/status` and `session_status` status card while preserving current rendering by default.

## Best Possible Solution

Add one shared, documented status visibility contract for both chat `/status` and `session_status`, preserving the rendered card by default and covering privacy-sensitive fields with schema, docs, and focused tests.

## Reproduction Assessment

Yes for source-level verification of the current absence, not as a bug reproduction. Current main routes both `/status` and `session_status` through rendered status-card text with no suppression config or documented equivalent.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
