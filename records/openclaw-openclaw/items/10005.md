## Best Possible Solution

Add one documented shared status visibility contract that flows through `/status` and `session_status`, with schema/types/help/docs and focused tests for default-visible and hidden or reduced-output modes.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared status visibility option, not per-channel special casing, with today’s card output preserved by default.
