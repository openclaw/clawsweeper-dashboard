## Summary

Keep open. Current main still always builds and returns the rendered `/status`/`session_status` card, while the requested config key and hidden-output semantics remain a maintainer product/API choice.

## What This Changes

The issue requests a configuration option to hide the rendered session status card for chat `/status` and the `session_status` tool, defaulting to current rendering.

## Best Possible Solution

Add a documented, default-on status-card visibility control shared by chat `/status` and `session_status`, with schema/types/help/docs and focused tests, while keeping CLI `openclaw status --json` separate.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that current main lacks the requested setting and always returns rendered status text for the chat and tool paths.

## Solution Assessment

Unclear until maintainers choose the canonical config key and hidden-output semantics. The narrow maintainable direction is a shared default-on visibility control, not reusing the separate CLI `status --json` surface.
