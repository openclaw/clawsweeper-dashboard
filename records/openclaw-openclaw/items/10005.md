## Summary

Keep open: current main and v2026.6.8 still render chat `/status` and `session_status` through the shared status-card path with no supported hide, JSON-only, or section-visibility config, and the privacy-focused related report was already closed into this tracker.

## What This Changes

Requests a configuration-controlled way to hide or return JSON instead of the rendered session status card for `/status` and `session_status`.

## Best Possible Solution

Define a maintainer-approved status-card visibility contract that preserves current rendering by default, covers the per-agent usage/billing privacy case, and updates schema, docs, and focused chat/tool tests together.

## Reproduction Assessment

Not applicable as a bug repro, but current behavior is source-reproducible: both `/status` and `session_status` still build and return rendered status-card text with no hide setting.

## Solution Assessment

Unclear as proposed; `session.statusCard` or `agents.defaults.statusCard` is plausible, but the best fix needs maintainer choice on privacy scope, output shape, and config ownership before implementation.
