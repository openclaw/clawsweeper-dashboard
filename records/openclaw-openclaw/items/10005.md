## Summary

Keep open. Current main still has only the rendered chat/tool status-card path, no documented status-card visibility setting, and the related privacy request is closed as superseded by this tracker.

## What This Changes

Request a configuration option to hide or suppress the rendered `/status` and `session_status` status card while preserving current behavior by default.

## Best Possible Solution

Add a documented, backwards-compatible visibility contract for chat `/status` and `session_status`, with schema/types/help/docs and focused coverage for the chosen global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence the requested control is absent because both entrypoints still render status text and the config/docs have no hiding setting.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is a shared visibility setting threaded through the existing status builders while preserving current output by default, not treating CLI `openclaw status --json` as a substitute.
