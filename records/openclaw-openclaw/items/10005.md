## Summary

Keep open: current main still has no status-card visibility control, and the discussion already treats this as the remaining canonical tracker after the narrower privacy request was superseded.

## What This Changes

Add a config setting to hide or suppress the rendered `/status` and `session_status` card while preserving current visible behavior by default.

## Best Possible Solution

Add one documented visibility control in the shared status builder path, keep rendered cards enabled by default, and cover both chat-command and agent-tool callers once maintainers choose the public semantics.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature/config request. Source inspection gives a high-confidence verification path because the current chat and tool paths always produce rendered status text without a suppression setting.

## Solution Assessment

Unclear until a maintainer chooses the public config key and hidden-output semantics. The narrow maintainable direction is a single shared status-builder control, not treating the existing CLI JSON status mode as sufficient.
