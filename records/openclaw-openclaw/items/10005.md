## Summary

Keep open. Current main still has no chat/tool status-card visibility control, and the related privacy request #69445 was closed as superseded by this issue while #13450 covers a separate CLI `status --json` surface.

## What This Changes

Issue requests a config option to hide the rendered session status card for chat `/status` and `session_status`, preserving current rendering by default.

## Best Possible Solution

Add a documented status-card visibility setting chosen by maintainers, default it to current rendering, thread it through the chat and tool status builders, and cover both `/status` and `session_status` with focused tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection shows the current chat and tool paths always build and return the rendered status card, and no proposed config key exists.

## Solution Assessment

Unclear until maintainers choose the canonical config key and hidden-output semantics. The narrow maintainable direction is a documented, default-on visibility control shared by `/status` and `session_status`, not reusing the separate CLI JSON surface.
