## Summary

Keep this open. Current main still renders `/status` and `session_status` through the shared status text/card path, and the config schemas/docs do not expose a status-card visibility setting; the related per-agent privacy request remains superseded by this tracker.

## What This Changes

Request a configuration option to hide rendered chat/tool session status cards, preserving current behavior by default and returning a less cluttered output when disabled.

## Best Possible Solution

Add a documented, backwards-compatible status-card visibility contract for chat `/status` and `session_status`, with schema/types/help/docs and focused coverage for both whole-card hiding and the per-agent usage privacy case if maintainers choose section controls.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence that the requested feature is absent because both entrypoints still render status text and the config schemas have no hiding setting.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is a shared visibility setting threaded through the existing status builders while preserving rendered output by default, not treating CLI `openclaw status --json` as a substitute.
