## Summary

Keep open: current main and the latest release still render `/status` and `session_status` through the shared status-card text path with no documented hide, JSON-only, or section-visibility option. This remains the canonical tracker for the status-card suppression request because the per-agent usage/privacy issue was closed as superseded by this issue and the CLI JSON request is a separate surface.

## What This Changes

The issue requests a configuration option to hide the rendered session status card for chat `/status` and the `session_status` tool while preserving current behavior by default.

## Best Possible Solution

Add one shared, documented status visibility contract for both chat `/status` and `session_status`, preserving the rendered card by default and covering config schema, help/docs, and focused tests.

## Reproduction Assessment

Yes for source-level verification of the missing capability, not as a broken-existing-behavior bug. Current main and `v2026.5.22` both route `/status` and `session_status` through rendered status-card text with no suppression config or documented equivalent.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
