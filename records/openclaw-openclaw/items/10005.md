## Summary

Keep open: current main still has no documented hide, JSON-only, or section-visibility control for the shared chat `/status` and `session_status` status-card output. The remaining work is a product/security contract decision, and the related privacy-focused request was already closed in favor of this tracker.

## What This Changes

Request a config-controlled way to suppress or return structured-only output for rendered `/status` and `session_status` status cards.

## Best Possible Solution

Add one shared, documented status visibility contract for both chat `/status` and `session_status`, preserving the rendered card by default and covering privacy-sensitive fields with schema, docs, and focused tests.

## Reproduction Assessment

Yes for source-level verification of the feature gap: current `main` routes both `/status` and `session_status` through rendered status-card text with no suppression config. This is not a bug reproduction because the request asks for a new visibility option.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
