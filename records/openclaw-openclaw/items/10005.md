## Summary

Keep this open. Current main still lacks a documented status-card visibility control for chat `/status` or `session_status`; both paths continue to build rendered status text, and the related per-agent privacy request was closed as superseded by this tracker.

## What This Changes

The issue requests a new configuration option to hide the rendered session status card for `/status` and `session_status`, preserving current behavior by default.

## Best Possible Solution

Add a documented, backwards-compatible status-card visibility control for chat `/status` and `session_status`, with schema/types/help/docs and focused tests covering both entrypoints plus the per-agent usage privacy case.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence that the requested feature is absent because both entrypoints still render status text and the config schema has no hiding setting.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is a shared visibility setting threaded through the existing status builders while preserving current rendered output by default.
