Keep open: current main and the latest release still render shared `/status` and `session_status` status-card text with no documented hide, JSON-only, or section-visibility control, and related requests were closed in favor of this canonical tracker.

## What This Changes

The issue requests a configuration option to hide or suppress rendered session status cards for chat `/status` and the `session_status` tool.

## Best Possible Solution

Add one shared, documented status visibility contract for both chat `/status` and `session_status`, preserving the rendered card by default and covering privacy-sensitive fields with schema, docs, and focused tests.

## Reproduction Assessment

Yes for source-level verification of the current absence, not as a bug reproduction. Current main routes both `/status` and `session_status` through rendered status-card text with no suppression config or documented equivalent.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
