## Summary

Keep open. Current main still has no documented status-card visibility setting, and both chat `/status` and `session_status` continue to render the shared status text card unconditionally.

## What This Changes

Request a configuration option to hide the rendered `/status` and `session_status` status card, potentially returning JSON-only output instead.

## Best Possible Solution

Add a backward-compatible, documented visibility contract for chat `/status` and `session_status`, preserving current rendering by default and covering the chosen config scope with schema/types/docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source and docs inspection on current main gives high confidence the requested control is absent because both status entry points still render shared status text and config schemas have no hiding key.

## Solution Assessment

Unclear until maintainers choose the public API and output shape. The maintainable direction is a shared setting threaded through the existing status builders while preserving current rendering by default, not treating CLI JSON status output as a substitute for chat/tool behavior.
