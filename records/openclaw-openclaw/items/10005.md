## Summary

Keep this open. Current main still renders the chat `/status` and `session_status` reply through the shared status-card builders with no `session.statusCard`, `agents.defaults.statusCard`, JSON-only mode, or documented hiding equivalent; the related CLI JSON issue is a separate surface, and #69445 was closed as superseded by this tracker.

## What This Changes

The issue requests a configuration option to hide the session status card for chat `/status` and `session_status` while preserving current behavior by default.

## Best Possible Solution

Add a documented, backwards-compatible status-card visibility control for chat `/status` and `session_status`, with schema/types/help/docs plus focused coverage for both entrypoints once maintainers choose the config shape.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main confirms the requested feature is absent: both entrypoints still build rendered status text and the config schema has no hiding option.

## Solution Assessment

Unclear until maintainers choose the product contract. The likely maintainable direction is one shared status visibility setting threaded through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, preserving current rendering by default.
