## Summary

Current main still has no supported way to hide or reduce the chat/tool status card, and this issue remains the canonical tracker after the narrower privacy request was closed in its favor. The request is valid, but the global-versus-per-agent scope and hidden-output behavior require maintainer product and security judgment.

## What This Changes

Add a configurable visibility or reduced-output policy for the rendered session status card shared by chat `/status` and the `session_status` tool.

## Best Possible Solution

Define one canonical, privacy-aware status presentation contract shared by chat and tool callers, preserving the full card by default while supporting an explicitly chosen reduced, structured, or section-filtered mode.

## Reproduction Assessment

Yes at source level: both relevant current-main entry points use the shared rendered-text path, whose contract has no visibility, section-filtering, or alternate-output control.

## Solution Assessment

No, the proposed boolean is not yet a complete solution because maintainers must choose global versus per-agent scope and define whether hidden output is empty, reduced text, section-filtered, or structured data.
