## Summary

Current main and v2026.6.5 still expose the shared rendered status card with no visibility or redaction control, and live GitHub confirms this issue remains the canonical tracker for both clutter and per-agent usage-privacy needs. The item is still necessary, but implementation should wait for a maintainer product decision because whole-card hiding, selective redaction, and tool return semantics are materially different contracts.

## What This Changes

Requests a configuration option to suppress or alter the session status card returned by chat `/status` and the `session_status` tool.

## Best Possible Solution

Define one canonical per-agent status presentation policy that preserves current output by default, supports selective sensitive-field visibility when required, and specifies consistent chat text and structured tool-detail behavior.

## Reproduction Assessment

Yes, at source level with high confidence: chat `/status` and `session_status` both route through the shared rendered status builder, whose contract has no visibility control. No failing runtime test was run because this is a missing capability rather than broken documented behavior.

## Solution Assessment

No, not as currently specified: a single hide-card boolean does not resolve selective privacy redaction or define the tool's replacement output. The best design is one canonical per-agent presentation policy with explicit text and structured-detail semantics.
