## Summary

Keep open: current main and the latest release still render `/status` and `session_status` through the shared status-card path with no documented hide, JSON-only, or section-visibility control; related requests were closed in favor of this canonical tracker.

## What This Changes

The issue requests a configurable way to hide the rendered `/status` and `session_status` status card, preserving current rendering by default and allowing quieter or structured output when disabled.

## Best Possible Solution

Add one shared, documented status visibility contract for both chat `/status` and `session_status`, preserving the rendered card by default and covering schema, help/docs, and regression tests for the selected privacy/output behavior.

## Reproduction Assessment

Yes for source-level verification of the current absence, not as a bug reproduction. Current main routes both `/status` and `session_status` through rendered status-card text with no suppression config or documented equivalent.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
