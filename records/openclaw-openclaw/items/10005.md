## What This Changes

The issue requests a new configuration option to hide or suppress the rendered session status card for `/status` and `session_status`, with current rendering kept as the default.

## Best Possible Solution

Define one shared, documented status visibility contract for chat `/status` and `session_status`, preserve the rendered card by default, and cover the chosen schema/help/docs plus focused tests after maintainers settle the privacy/output shape.

## Reproduction Assessment

Yes, as a source-level feature-gap check rather than a broken existing behavior reproduction. Current main and `v2026.6.5` both route `/status` and `session_status` through rendered status-card text with no suppression config or documented equivalent.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
