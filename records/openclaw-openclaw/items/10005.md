## Summary

Keep this open. Current main still renders chat `/status` and `session_status` through the shared status-card text path, and the config schemas/docs do not expose a visibility setting for hiding the card or returning JSON-only tool output.

## What This Changes

The issue requests a configuration option to hide the rendered session status card for chat `/status` and the `session_status` tool while preserving current behavior by default.

## Best Possible Solution

Add a documented, backwards-compatible status-card visibility contract for chat `/status` and `session_status`, preserving rendered output by default and covering schema/types/help/docs plus focused tests for the chosen global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence that the requested feature is absent because both entrypoints still render status text and the config schemas have no hiding setting.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is a shared visibility setting threaded through the existing status builders while preserving current output by default, not treating CLI `openclaw status --json` as a substitute.
