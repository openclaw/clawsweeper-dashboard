## Summary

Keep this issue open as the canonical tracker: current main and v2026.6.5 still render the shared status card for chat `/status` and `session_status`, while the narrower CLI JSON request and per-agent usage-redaction request were both closed in favor of this issue. The remaining question is a maintainer product decision about whether hiding means suppressing the whole card, redacting selected fields, or returning structured tool details.

## What This Changes

Add a configuration control to hide or replace the rendered session status card for chat `/status` and the `session_status` tool.

## Best Possible Solution

Approve one default-preserving visibility or redaction contract, implement it once in the shared status pipeline for both chat and tool callers, and align schema, help, public docs, and focused tests while keeping CLI `openclaw status --json` separate.

## Reproduction Assessment

Yes at source level: current main’s chat and tool paths still converge on the shared rendered status card, and the builder contract has no visibility mode. This demonstrates a missing capability rather than a failure of an existing documented contract.

## Solution Assessment

Unclear. The shared-builder boundary is the right implementation seam, but maintainers must decide whether whole-card hiding, selective privacy redaction, or structured tool output is the best permanent user contract before choosing a config shape.
