## Summary

Keep open. Current main and the latest shipped tag still render chat `/status` and `session_status` through the shared status-card text path, with no documented visibility setting or schema key for hiding the card.

## What This Changes

Request to add a config option that hides the rendered `/status` and `session_status` status card while preserving current behavior by default.

## Best Possible Solution

Add a backward-compatible, documented status-card visibility contract for chat `/status` and `session_status`, with schema/types/help/docs and focused coverage for the chosen global or per-agent behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source and docs inspection on current main gives high confidence the requested control is absent because both status entry points still render shared status text and config schemas have no hiding key.

## Solution Assessment

Unclear until maintainers choose the public API and output shape. The maintainable direction is a shared setting threaded through the existing status builders while preserving current rendering by default, not treating CLI JSON status output as a substitute for chat/tool behavior.
