## Summary

Keep open: current main and the latest stable release still route chat `/status` and `session_status` through rendered status-card text with no documented hide, JSON-only, or section-visibility contract. The privacy-focused related issue https://github.com/openclaw/openclaw/issues/69445 was closed as superseded by this issue, while the CLI JSON request https://github.com/openclaw/openclaw/issues/13450 is a separate `openclaw status --json` surface.

## What This Changes

Requested a configuration option to suppress the rendered session status card for chat `/status` and the `session_status` tool.

## Best Possible Solution

Add one shared, documented status visibility contract for both chat `/status` and `session_status`, preserving the rendered card by default and covering schema, help/docs, and focused regression tests for the chosen privacy/output behavior.

## Reproduction Assessment

Yes for source-level verification of the current absence, not as a bug reproduction. Current main and `v2026.5.22` both route `/status` and `session_status` through rendered status-card text with no suppression config or documented equivalent.

## Solution Assessment

Unclear until maintainers choose the output contract. The maintainable direction is a central shared visibility option covering both status paths, not per-channel special cases, with the current rendered card kept as the default.
