## Summary

Keep open. Current main still has no Active Memory single-shot mode, and eligible preflight turns still build a query and await the embedded recall sub-agent path; the existing hard-timeout work does not implement the requested one-search plus one-completion architecture.

## What This Changes

Add an opt-in Active Memory `config.mode: "single-shot"` that performs one QMD memory search and one LLM completion instead of the current embedded recall agent loop.

## Best Possible Solution

Implement a backward-compatible embedded default plus opt-in `single-shot` mode in the Active Memory plugin, using direct memory search and one model completion while preserving the existing multi-turn mode for recall-heavy/contextual use cases.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: the requested `config.mode` is absent from the strict manifest and normalized config, and eligible prompt-build turns still await `maybeResolveActiveRecall` through `runEmbeddedPiAgent`.

## Solution Assessment

Yes. An opt-in single-shot mode is the narrowest maintainable direction because it addresses preference-style preflight latency without replacing the existing embedded recall behavior for users who need deeper recall.

## Repair Work Prompt

Implement the Active Memory single-shot mode requested by https://github.com/openclaw/openclaw/issues/72359, preserving the current embedded recall behavior as the default. Add a plugin-owned `plugins.entries.active-memory.config.mode` with default `embedded` and opt-in `single-shot`. In `single-shot`, reuse the existing query construction, perform one memory search through the supported memory-host search surface with the configured QMD search mode/session context, then make one model completion that returns either `NONE` or a summary bounded by `maxSummaryChars`. Do not call `runEmbeddedPiAgent` in the single-shot branch, do not change the default embedded branch, and do not deep-import core internals from the plugin. If a generic one-shot text-completion runtime seam is required, add the smallest backward-compatible SDK/runtime surface with docs/API follow-through. Update the manifest schema/ui hints, runtime config normalization, Active Memory docs, focused tests, and CHANGELOG.md. Cover config schema acceptance/rejection, single-shot not invoking the embedded agent, one memory search, timeout/fail-open handling, `NONE` parsing, summary truncation, cache/status/log behavior, and preservation of embedded mode.
