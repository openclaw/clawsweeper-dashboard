## Summary

Keep open. Current main and v2026.6.10 still implement Active Memory as a blocking embedded-agent recall path with no single-shot mode selector, so the central request remains real but needs maintainer product/API direction before implementation.

## What This Changes

The issue requests an opt-in Active Memory `single-shot` config mode that produces preflight memory context with one memory search and one LLM call instead of embedded-agent recall.

## Best Possible Solution

Keep this as the narrow tracker for an opt-in Active Memory single-shot recall mode, implemented inside the plugin only after maintainers choose the config semantics and validation bar.

## Reproduction Assessment

Yes, at source level: current main has no Active Memory mode selector and still awaits embedded-agent recall from `before_prompt_build`. I did not rerun the reporter's live Telegram/provider latency benchmark.

## Solution Assessment

Unclear until maintainers choose the product semantics. The single-shot direction is plausible, but it adds config surface and changes recall quality, direct model-call policy, provider routing, and the plugin runtime path.
