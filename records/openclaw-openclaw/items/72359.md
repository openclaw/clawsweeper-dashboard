## Summary

Keep open. Current main still lacks an opt-in Active Memory single-shot path; the strict plugin schema, normalized runtime config, docs, and tests continue to describe the blocking embedded recall sub-agent path that calls `runEmbeddedPiAgent`.

## What This Changes

Request an opt-in Active Memory `mode: "single-shot"` that performs one memory search and one completion instead of the embedded recall sub-agent loop.

## Best Possible Solution

Add a backward-compatible Active Memory mode config with embedded recall as the default and an opt-in single-shot branch using existing memory search and simple-completion surfaces.

## Reproduction Assessment

Yes for the missing capability. Current main has no accepted or normalized `config.mode`, and eligible prompt-build hooks still await `maybeResolveActiveRecall`, which calls `runEmbeddedPiAgent`.

## Solution Assessment

Yes. An opt-in plugin-owned single-shot mode is the narrowest maintainable direction because it leaves recall-heavy embedded behavior unchanged while giving preference-style preflight a one-search, one-completion path.
