## Summary

Keep open. Current main still lacks an opt-in Active Memory single-shot path; recent hard-timeout, circuit-breaker, and fast-fail changes bound the existing embedded recall sub-agent but do not remove the multi-turn agent loop for low-latency preflight recall.

## What This Changes

Request an opt-in Active Memory mode that runs one direct memory search and one LLM completion instead of the embedded recall sub-agent loop.

## Best Possible Solution

Add a backward-compatible plugin-owned config.mode with embedded recall as the default and an opt-in single-shot branch that uses existing memory search and simple-completion seams, with docs, manifest schema, tests, and changelog aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: config.mode is absent from the strict manifest and normalized runtime config, and eligible prompt-build turns still await the embedded recall path.

## Solution Assessment

Yes. An opt-in single-shot mode is the narrowest maintainable direction for preference-style preflight latency; replacing the embedded default would be riskier for recall-heavy and contextual use cases.
