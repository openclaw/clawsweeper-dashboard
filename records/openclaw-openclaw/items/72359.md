## Summary

Keep open. Current main still lacks the requested Active Memory single-shot mode: config validation has no `mode` field, runtime config has no discriminator, and eligible prompt-build turns still await the embedded recall agent path.

## What This Changes

Request an opt-in Active Memory `single-shot` mode that performs one bounded memory search plus one direct model completion instead of running the embedded multi-turn recall agent.

## Best Possible Solution

Add a backward-compatible `config.mode` for Active Memory with the current embedded recall agent as the default and a `single-shot` branch that uses the existing memory search manager facade plus simple-completion runtime for one bounded search and one bounded completion, with docs, manifest schema, tests, and changelog updated together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `mode: "single-shot"` is not accepted by the strict manifest schema, is not normalized into runtime config, and eligible prompt-build turns still await the embedded recall path.

## Solution Assessment

Yes. An opt-in single-shot mode is the narrowest maintainable fix for the preference-only/preflight latency case; replacing the default embedded sub-agent would be riskier because recall-heavy and contextual styles still use the existing tool-driven behavior.
