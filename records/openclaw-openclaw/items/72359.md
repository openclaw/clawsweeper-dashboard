## Summary

Keep open. Current main still has only the bounded embedded Active Memory recall sub-agent; the requested opt-in single-shot path is absent from the manifest schema, runtime config, prompt hook, docs, and tests, while #72347 only covered hard-timeout enforceability and #65309 remains a broader latency tracker.

## What This Changes

Request an opt-in Active Memory `single-shot` mode that does one QMD search plus one LLM completion instead of the embedded multi-turn recall agent loop.

## Best Possible Solution

Add a backward-compatible Active Memory `config.mode` with the current embedded-agent behavior as the default and an opt-in `single-shot` branch that uses direct memory search plus one bounded simple completion, with docs, manifest schema, config tests, runtime tests, and changelog coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `mode: "single-shot"` is not accepted by the strict manifest schema, is not normalized into runtime config, and eligible prompt-build turns still await the embedded recall implementation.

## Solution Assessment

Yes. An opt-in mode is the narrowest maintainable solution for low-latency preference/preflight recall because it avoids changing recall-heavy/contextual behavior that still benefits from the tool-driven sub-agent path.

## Repair Work Prompt

Implement issue https://github.com/openclaw/openclaw/issues/72359 as a narrow Active Memory plugin feature. Add a backward-compatible `plugins.entries.active-memory.config.mode` enum with the current embedded behavior as default and an opt-in `single-shot` mode. In `single-shot`, derive the existing query, run one direct memory search through the public memory-search manager/QMD seam, build a compact prompt from the latest message/query plus hits, call the public simple-completion runtime once using the same model-resolution/auth semantics, and return `NONE` or a summary capped by `maxSummaryChars`. Preserve existing eligibility, timeout budget, cache, circuit breaker, logging/status/debug, fail-open behavior, prompt prefix, and embedded-agent mode behavior. Do not change the default mode or remove `runEmbeddedPiAgent` for recall-heavy/contextual use cases. Update `extensions/active-memory/openclaw.plugin.json`, `extensions/active-memory/index.ts`, colocated tests including manifest config tests, `docs/concepts/active-memory.md`, and `CHANGELOG.md` because this is user-facing. Validate with the targeted Active Memory tests, targeted oxfmt, and the changed gate per repo policy.
