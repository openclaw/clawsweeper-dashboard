## Summary

Keep open. Current main and v2026.6.6 still expose Active Memory as a blocking embedded recall sub-agent with no opt-in single-shot/no-agent-loop mode, and the related reliability issues or open prompt-trimming PR do not supersede this requested plugin API decision.

## What This Changes

Request to add an opt-in Active Memory single-shot recall mode that avoids the embedded agent loop for lower-latency preflight memory injection.

## Best Possible Solution

Add a plugin-owned opt-in mode selector, keep the existing embedded sub-agent behavior as the compatible default, and implement the single-shot branch through existing memory-search and runtime LLM seams with docs and focused tests.

## Reproduction Assessment

Yes, at source level: current main has no accepted mode/single-shot config field and eligible Active Memory recall still awaits runEmbeddedAgent from before_prompt_build.

## Solution Assessment

Unclear until maintainer product/API review; the requested single-shot path is plausible and not replaced by current timeout/prompt-trimming work, but defaults and recall-quality tradeoffs still need an owner decision.
