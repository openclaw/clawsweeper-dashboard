## Summary

Keep open. Current main still lacks an opt-in Active Memory single-shot path; the existing hard-timeout, circuit-breaker, and fast-fail code bounds the embedded recall sub-agent but does not provide the requested one-search, one-completion preflight mode.

## What This Changes

Request an opt-in Active Memory `single-shot` mode that bypasses the embedded agent loop and summarizes memory with one QMD search plus one LLM completion.

## Best Possible Solution

Add a backward-compatible plugin-owned mode with the embedded sub-agent as the default and an opt-in single-shot branch using existing memory search and simple-completion seams, with manifest schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main has no accepted `config.mode`, and eligible prompt-build turns still flow through `maybeResolveActiveRecall` into `runEmbeddedPiAgent`.

## Solution Assessment

Yes. An opt-in single-shot branch is the narrowest maintainable direction because it preserves the existing embedded recall behavior for recall-heavy/contextual use while giving latency-sensitive preflight setups a lighter path.

## Repair Work Prompt

Implement the Active Memory single-shot mode requested in https://github.com/openclaw/openclaw/issues/72359. Add a backward-compatible `plugins.entries.active-memory.config.mode` (defaulting to the current embedded sub-agent behavior) with an opt-in single-shot value that derives the existing query, performs one memory/QMD search through plugin-owned/public seams, builds a compact prompt from the message plus hits, calls the existing simple-completion runtime once with the resolved Active Memory model and the same timeout/fail-open discipline, parses `NONE` or a `maxSummaryChars`-bounded summary, and injects the existing hidden prompt prefix. Do not change the default embedded mode, do not move Active Memory policy into core, and do not add bundled-plugin ids outside the plugin boundary. Update `openclaw.plugin.json` schema/UI hints, runtime config normalization, focused tests, docs, and a user-facing changelog entry crediting the reporter if appropriate. Related context: #72347 covered timeout enforceability; #65309 tracks broader foreground timeout symptoms.
