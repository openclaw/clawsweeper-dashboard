## Summary

Keep open. Current main still implements Active Memory as a bounded blocking embedded recall sub-agent with timeout/cache/circuit-breaker mitigations, but it does not provide the requested opt-in single-shot mode for one memory search plus one direct completion.

## What This Changes

The issue requests an opt-in Active Memory single-shot mode that avoids the embedded sub-agent loop for low-latency preflight recall.

## Best Possible Solution

Add a documented, backward-compatible Active Memory mode with the current embedded sub-agent behavior as the default and a single-shot branch that performs one bounded memory search, one bounded direct completion, returns NONE or a capped summary, and keeps recall-heavy/contextual behavior unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `mode: "single-shot"` is not accepted by the strict manifest schema, is not normalized into runtime config, and eligible prompt-build turns still await the embedded recall path.

## Solution Assessment

Yes. An opt-in mode inside the Active Memory plugin is the narrowest maintainable solution because the request is plugin-owned and current main already exposes public helper seams for memory search and simple completion.
