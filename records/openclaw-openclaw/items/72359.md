## Summary

Keep open. Current main still lacks an opt-in Active Memory single-shot path; the plugin schema, normalized runtime config, prompt-build hook, docs, and tests all continue to describe and exercise the blocking embedded recall sub-agent path.

## What This Changes

Request an opt-in Active Memory single-shot mode that performs one memory search and one LLM completion instead of the embedded multi-turn recall agent.

## Best Possible Solution

Add a backward-compatible Active Memory mode config with existing embedded recall as the default and an opt-in single-shot branch using direct memory search plus simple completion, with docs, manifest schema, tests, and changelog aligned.

## Reproduction Assessment

Yes for the missing capability. Source inspection on current main shows config.mode is not accepted or normalized, and eligible before_prompt_build hooks still await maybeResolveActiveRecall, which calls runEmbeddedPiAgent.

## Solution Assessment

Yes. An opt-in plugin-owned single-shot mode is the narrowest maintainable direction because it leaves recall-heavy/contextual embedded behavior unchanged while giving preference-style preflight a one-search, one-completion path.

## Repair Work Prompt

Implement the Active Memory single-shot mode requested in https://github.com/openclaw/openclaw/issues/72359. Add a backward-compatible `plugins.entries.active-memory.config.mode` or equivalent with the current embedded recall behavior as the default and an opt-in `single-shot` path. Keep the change inside the Active Memory plugin unless a truly generic existing SDK seam is needed. For single-shot, derive the same query, run one memory search through the existing memory runtime/manager with the current session visibility and qmd searchMode semantics, call one simple completion through the existing simple-completion SDK helpers, and parse `NONE` or a summary bounded by `maxSummaryChars`. Preserve fail-open behavior, timeout/abort handling, cache/circuit-breaker/status/debug behavior where applicable, and do not make single-shot the default for recall-heavy/contextual users. Update `extensions/active-memory/openclaw.plugin.json`, config normalization/types, tests, docs, and `CHANGELOG.md`. Stop and report if the implementation requires a new core SDK/API seam beyond current simple-completion and memory runtime surfaces.
