## Summary

Keep open. Current main still lacks an opt-in Active Memory single-shot mode: the strict schema has no `config.mode`, runtime config normalization has no discriminator, and eligible turns still await the embedded recall sub-agent path.

## What This Changes

Request an opt-in Active Memory single-shot config mode that runs one QMD search and one LLM call instead of the embedded recall agent loop.

## Best Possible Solution

Add a backward-compatible Active Memory mode selector only after maintainers accept the config/API contract, keeping embedded recall as the default and implementing single-shot through supported memory-search and model-completion seams.

## Reproduction Assessment

Yes for the missing capability: current main has no accepted or normalized `config.mode`, and eligible prompt-build hooks still await `maybeResolveActiveRecall`, which calls `runEmbeddedPiAgent`. I did not run a live latency benchmark because source inspection is enough to prove the requested mode is absent.

## Solution Assessment

Unclear pending maintainer approval. An opt-in single-shot mode is a plausible narrow direction for preference-style preflight, but adding a user-facing config mode and possibly a direct model-completion plugin seam needs product/API judgment first.
