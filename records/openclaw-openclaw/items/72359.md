## Summary

Keep open. Current main still only exposes the embedded Active Memory recall path; the schema, normalized config, docs, tests, and prompt hook do not provide an opt-in single-shot mode.

## What This Changes

Request an opt-in Active Memory config mode that performs one qmd search and one LLM completion instead of the embedded recall agent loop.

## Best Possible Solution

Add a backward-compatible Active Memory mode config that defaults to the current embedded recall behavior and adds an opt-in single-shot branch with docs, schema, and regression coverage.

## Reproduction Assessment

Yes for the missing capability. Current main has no accepted or normalized Active Memory mode field, and eligible prompt-build hooks still route through maybeResolveActiveRecall into runEmbeddedPiAgent.

## Solution Assessment

Yes. An opt-in plugin-owned single-shot mode is the narrowest maintainable direction because it leaves recall-heavy embedded behavior unchanged while giving preference-style preflight a one-search, one-completion path; any needed memory-search reuse should go through a public seam.

Implement opt-in Active Memory single-shot mode for https://github.com/openclaw/openclaw/issues/72359. Add a backward-compatible config discriminator such as plugins.entries.active-memory.config.mode with the current embedded recall behavior as the default, and an opt-in single-shot path that derives the existing query, performs one memory/qmd search, then performs one simple LLM completion that returns NONE or a summary capped by maxSummaryChars. Keep the existing runEmbeddedPiAgent path unchanged for the default/embedded mode.
