## Summary

Keep open. Current main still only supports the blocking embedded Active Memory recall path, with no config discriminator or single-shot branch for one search plus one completion.

## What This Changes

Add an opt-in Active Memory single-shot mode that bypasses the embedded agent loop for low-latency preflight memory injection.

## Best Possible Solution

Add a backward-compatible Active Memory mode config that defaults to embedded recall and offers an opt-in single-shot branch using public memory search and simple-completion helpers, with docs, schema, tests, and changelog updated together.

## Reproduction Assessment

Yes for the missing capability: current main has no accepted or normalized mode field, and the eligible prompt-build path still awaits maybeResolveActiveRecall into runEmbeddedPiAgent. I did not run a live latency benchmark.

## Solution Assessment

Yes. An opt-in plugin-owned mode is the narrowest maintainable path because it preserves the existing recall-heavy embedded behavior while giving preference-style preflight a one-search, one-completion option.
