## Summary

Keep open. Current main still lacks an opt-in Active Memory single-shot mode: the strict manifest and runtime config have no `mode`, docs describe only the blocking memory sub-agent, and eligible prompt-build hooks still await the embedded recall path.

## What This Changes

Request an opt-in Active Memory `mode: "single-shot"` that uses one memory search and one simple completion instead of the embedded recall agent loop.

## Best Possible Solution

Add a backward-compatible Active Memory mode with the current embedded recall as default and an opt-in single-shot branch that uses public simple-completion and memory-runtime surfaces, with schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes for the missing capability. Current-main source has no accepted or normalized mode field, and eligible prompt-build hooks still await `maybeResolveActiveRecall`, whose recall path calls `runEmbeddedPiAgent`.

## Solution Assessment

Yes. An opt-in plugin-owned single-shot mode is the narrowest maintainable direction because it preserves the existing recall-heavy embedded behavior while giving preference-style preflight a one-search, one-completion path.

## Repair Work Prompt

Implement issue #72359 as an opt-in Active Memory single-shot mode. Add a backward-compatible `plugins.entries.active-memory.config.mode` enum, defaulting to the existing embedded/sub-agent behavior, and add a single-shot branch in `extensions/active-memory/index.ts` that reuses existing query derivation, model resolution, timeout/abort handling, cache/circuit-breaker/logging conventions, and summary parsing. The single-shot branch should perform one memory search through public plugin SDK or memory-capability/runtime surfaces, then one simple completion through public simple-completion helpers such as `prepareSimpleCompletionModelForAgent` and `completeWithPreparedSimpleCompletionModel`; do not import from `memory-core/src`, core internals, or other plugin private files. Update `extensions/active-memory/openclaw.plugin.json`, `extensions/active-memory/index.test.ts`, `extensions/active-memory/config.test.ts`, `docs/concepts/active-memory.md`, and `CHANGELOG.md`. Preserve current embedded recall as the default and do not alter the existing hard-timeout fixes or recall-heavy/contextual behavior.
