## Summary

Keep open. Current main still has Active Memory implemented as a blocking embedded recall sub-agent, with no accepted `single-shot` config value, no runtime discriminator, and no documented or tested one-search/one-completion branch. The request is recent, source-confirmed, and narrow enough for a focused repair PR.

## What This Changes

Request an opt-in Active Memory single-shot mode that performs one memory search and one direct model completion instead of the existing embedded recall-agent loop.

## Best Possible Solution

Add a backward-compatible Active Memory mode with the current embedded-agent behavior as the default and an opt-in single-shot branch for preflight use. The new branch should use the existing query builder, plugin-safe memory search manager, and simple completion helper, then return `NONE` or a capped summary without tools or sub-agent transcripts unless explicitly needed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `mode: "single-shot"` is not accepted by the strict manifest or normalized by runtime config, and eligible prompt builds still flow through `maybeResolveActiveRecall` into `runEmbeddedPiAgent`.

## Solution Assessment

Likely yes. An opt-in single-shot mode is the narrowest maintainable fix for the low-latency preference/preflight case, while leaving the existing recall-heavy embedded mode as the default for current users.

## Repair Work Prompt

Implement the Active Memory single-shot mode requested in openclaw/openclaw#72359. Add an opt-in config value such as `plugins.entries.active-memory.config.mode: "single-shot"` while keeping the current embedded-agent behavior as the default. In single-shot mode, reuse the existing query construction and eligibility/cache/fail-open/status plumbing, perform one plugin-safe memory search via the Active Memory search manager, then call the existing simple completion helper once to produce either `NONE` or a summary capped by `maxSummaryChars`. Do not call `runEmbeddedPiAgent` in single-shot mode, do not bypass memory visibility/session filters, and do not change recall-heavy/contextual default behavior. Update `extensions/active-memory/openclaw.plugin.json`, runtime config normalization/types, focused tests, docs, and `CHANGELOG.md` with appropriate non-bot reporter credit. Preserve provider auth, abort/timeout behavior, cache behavior, debug/status lines, and graceful unavailable/empty handling.
