## Summary

Keep open. Current main still has Active Memory implemented as a bounded blocking recall sub-agent with timeout/cache/circuit-breaker mitigations, but there is no opt-in single-shot mode that performs one memory search and one direct model completion without `runEmbeddedPiAgent`.

## What This Changes

The issue requests an opt-in Active Memory single-shot mode for low-latency preflight recall using one memory query and one model completion instead of an embedded agent loop.

## Best Possible Solution

Add a documented, backward-compatible Active Memory `mode` with the current embedded sub-agent behavior as the default and a `single-shot` branch for low-latency recall. The new branch should use the public memory search and simple-completion SDK helpers, perform one bounded completion, return `NONE` or a capped summary, and leave recall-heavy/contextual embedded behavior unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `mode: "single-shot"` is not accepted by the strict manifest schema, is not normalized into runtime config, and eligible prompt-build turns still await the embedded recall path.

## Solution Assessment

Yes. An opt-in single-shot mode is the narrowest maintainable path for the preference-only/preflight latency case, and current main appears to have the public SDK seams needed to implement it without a new broad core API.

## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/72359 as a narrow Active Memory plugin change. Add a backward-compatible `plugins.entries.active-memory.config.mode` with the current embedded-agent behavior as the default and an opt-in `"single-shot"` value. In single-shot mode, keep all existing eligibility, queryMode, timeout, cache/status/logging, circuit-breaker, and fail-open behavior, but do not call `runEmbeddedPiAgent` or expose tools. Use the public `openclaw/plugin-sdk/memory-host-search` facade to perform one memory search for the derived query, honoring the configured qmd search-mode behavior and session/agent context. Use `openclaw/plugin-sdk/simple-completion-runtime` helpers for direct model selection/auth/completion, with the same model resolution intent as the existing Active Memory config. Build a concise prompt from the latest message/query plus bounded memory hits, ask for `NONE` or a summary capped by `maxSummaryChars`, parse/cap the result, and inject only through the existing hidden Active Memory prefix path. Update `extensions/active-memory/openclaw.plugin.json`, runtime types/normalization, focused tests, `docs/concepts/active-memory.md`, and `CHANGELOG.md`. Do not change default behavior, do not move Active Memory ownership into core, and do not alter recall-heavy/contextual embedded behavior beyond routing it through the default mode.
