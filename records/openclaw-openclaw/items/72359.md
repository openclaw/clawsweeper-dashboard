## Summary

Keep open. Current main still lacks an Active Memory single-shot mode: the plugin schema/config do not accept a mode discriminator, and eligible prompt preflight still awaits the embedded recall sub-agent. Related timeout work covers hard deadlines and broader latency symptoms, but no merged main implementation or exact open PR closing this issue was found.

## What This Changes

Add an opt-in Active Memory config mode that performs one memory search and one LLM completion instead of using the embedded recall agent loop.

## Best Possible Solution

Keep the issue open and, if maintainers still want the feature after the related latency PR is reconciled, implement a backward-compatible plugin-owned mode with the embedded loop as the default and docs/tests/changelog for the opt-in path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main does not accept config.mode="single-shot", and eligible preflight turns still route through maybeResolveActiveRecall and runEmbeddedPiAgent.

## Solution Assessment

Yes, with coordination. An opt-in plugin-owned single-shot mode is the narrowest maintainable way to reduce preference-style preflight latency without changing the default recall-heavy/contextual behavior, but it should be reconciled with the open #75200 latency work first.
