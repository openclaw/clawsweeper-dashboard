## Summary

Keep open. Current main still implements Active Memory as a bounded blocking embedded recall sub-agent, and the requested opt-in single-shot path is still absent from the strict manifest schema, runtime config, prompt-build hook, docs, and regression tests. The prior ClawSweeper comment's keep-open assessment still matches current main; #72347 covered timeout enforceability, while this issue remains the narrower architecture request alongside broader latency tracker #65309.

## What This Changes

Request an opt-in Active Memory `single-shot` mode that performs one memory search plus one direct model completion instead of running the embedded agent loop.

## Best Possible Solution

Add a backward-compatible Active Memory `config.mode` with the current embedded sub-agent behavior as the default and an opt-in `single-shot` branch for low-latency preference/preflight recall. The new branch should use public SDK memory-search and simple-completion seams, run one bounded search plus one bounded completion, return `NONE` or a `maxSummaryChars`-capped summary, and leave recall-heavy/contextual embedded behavior unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `mode: "single-shot"` is rejected by the strict manifest schema, is not normalized into runtime config, and eligible prompt-build turns still await the embedded recall path.

## Solution Assessment

Yes. An opt-in single-shot mode is the narrowest maintainable solution for the preference-only/preflight latency case; replacing the default embedded sub-agent path would be riskier because recall-heavy and contextual styles still depend on tool-driven behavior.

## Repair Work Prompt

Implement issue https://github.com/openclaw/openclaw/issues/72359 as a narrow Active Memory plugin feature. Add a backward-compatible `plugins.entries.active-memory.config.mode` option, probably `"embedded" | "single-shot"`, with the current embedded `runEmbeddedPiAgent` behavior as the default. For `single-shot`, keep the existing eligibility, query derivation, timeout/cache/circuit-breaker/status behavior where appropriate, then perform one direct memory search through the public memory search SDK seam and one simple model completion through the public agent-runtime simple-completion seam. The completion prompt must ask for exactly `NONE` or one compact summary, enforce `maxSummaryChars`, honor the configured/resolved model and abort budget, and fail open like the existing path. Do not change default multi-turn behavior, do not move Active Memory owner-specific policy into core, do not import core internals from the plugin, and do not change unrelated memory/provider defaults. Update `extensions/active-memory/openclaw.plugin.json`, `extensions/active-memory/index.ts`, focused tests in `extensions/active-memory/index.test.ts` and `extensions/active-memory/config.test.ts`, docs at `docs/concepts/active-memory.md`, and `CHANGELOG.md` because this is user-facing.
