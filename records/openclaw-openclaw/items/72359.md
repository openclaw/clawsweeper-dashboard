## Summary

Keep open. Current main still lacks an Active Memory single-shot mode, and eligible preflight turns still run through the embedded recall sub-agent path; the hard-timeout work from #72347 does not implement the requested lower-latency architecture.

## What This Changes

Request an opt-in Active Memory `single-shot` mode that performs one memory query plus one no-tools LLM call instead of the current embedded recall sub-agent loop.

## Best Possible Solution

Add a backward-compatible plugin-owned `config.mode` defaulting to the current embedded behavior, with an opt-in single-shot branch that uses the existing memory search manager plus one no-tools model call and preserves existing recall-heavy behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main has no single-shot config, the strict manifest would reject it, and eligible prompt-build turns still await the embedded recall sub-agent path.

## Solution Assessment

Yes. An opt-in mode is the narrowest maintainable direction because it improves latency-sensitive preference-style preflight use without changing the existing multi-turn recall default.

## Repair Work Prompt

Implement the Active Memory single-shot mode requested by https://github.com/openclaw/openclaw/issues/72359. Add a backward-compatible `plugins.entries.active-memory.config.mode` (for example `"embedded" | "single-shot"`) defaulting to the current embedded behavior. For `single-shot`, reuse the existing query derivation, perform one memory search via the existing memory search manager/qmd configuration, build a concise prompt from the user message plus hits, run exactly one no-tools model call (use the existing raw one-shot runtime path such as `modelRun: true`/`disableTools: true` if it is the safe current seam; add only a narrow generic runtime helper if that is insufficient), parse `NONE` or a summary capped by `maxSummaryChars`, preserve fail-open timeout/cache/status/debug behavior, and do not change the current multi-turn default. Update `extensions/active-memory/openclaw.plugin.json`, Active Memory tests, docs, and `CHANGELOG.md`; do not add core owner-specific policy or bundled IDs outside the plugin boundary.
