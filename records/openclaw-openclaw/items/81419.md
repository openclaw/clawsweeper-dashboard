## Summary

Keep open: current main still has a source-backed Xiaomi MiMo OpenAI-compatible replay bug. Xiaomi's own contract requires `reasoning_content` passback for multi-turn tool-call history, while OpenClaw's current Xiaomi path has no Xiaomi-specific wrapper and the existing replay backfill is only installed for DeepSeek V4 model ids.

## What This Changes

The issue asks OpenClaw to include `reasoning_content` on Xiaomi MiMo assistant history with tool calls during multi-turn thinking-mode requests.

## Best Possible Solution

Add a Xiaomi/OpenAI-compatible replay compatibility path, preferably reusing the existing `compat.thinkingFormat: "deepseek"` contract or a Xiaomi-owned wrapper, and add focused regression coverage for MiMo thinking-mode tool-call replay.

## Reproduction Assessment

Yes at source level, but not from a fresh live Xiaomi run. Xiaomi's docs define the 400 condition, and current main's Xiaomi OpenAI-compatible path has no Xiaomi-specific hook or generic accepted compat path that installs the `reasoning_content` replay backfill for MiMo.

## Solution Assessment

Yes: a Xiaomi/plugin-owned wrapper or an existing model-level compat switch wired into the generic wrapper is the narrow maintainable fix. The open Xiaomi output-normalization PR is useful adjacent work, but it does not solve request-history passback.
