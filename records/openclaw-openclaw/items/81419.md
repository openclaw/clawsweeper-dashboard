## What This Changes

The issue requests Xiaomi MiMo OpenAI-compatible multi-turn tool-call requests to preserve or backfill `reasoning_content` on replayed assistant messages so MiMo does not reject later turns with `400 Param Incorrect`.

## Best Possible Solution

Yes, at source level: Xiaomi documents the 400 condition, current main exposes Xiaomi MiMo through OpenAI completions, and current replay wrappers only backfill `reasoning_content` for DeepSeek V4. I did not perform a fresh live Xiaomi API run.
