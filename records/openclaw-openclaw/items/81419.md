## Summary

Keep open: current main still treats Xiaomi MiMo as ordinary OpenAI-compatible replay, has no Xiaomi request-history wrapper, and Xiaomi’s own API guidance requires `reasoning_content` passback for thinking-mode tool-call conversations. The open Xiaomi PR at https://github.com/openclaw/openclaw/pull/60304 addresses a different response-rendering problem, not this request replay path.

## What This Changes

The report requests Xiaomi MiMo thinking/tool-call replay support so assistant tool-call history includes `reasoning_content` on later requests.

## Best Possible Solution

Add a narrow Xiaomi/MiMo replay compatibility path that detects Xiaomi provider IDs or `xiaomimimo.com` endpoints, preserves or backfills `reasoning_content` for thinking-mode assistant tool-call history, and leaves non-Xiaomi providers unchanged.

## Reproduction Assessment

Yes, source and provider-contract evidence give a high-confidence path: MiMo thinking mode plus multi-turn tool calls requires `reasoning_content` replay, while current main has no Xiaomi detection or wrapper for that replay shape. I did not run a live Xiaomi credentialed call in this read-only review.

## Solution Assessment

Yes, a narrow compatibility fix in the Xiaomi/OpenAI-completions request path is the maintainable direction. The existing Xiaomi reasoning-only PR is not sufficient because it normalizes returned text, while this bug is about outgoing history replay.

## Repair Work Prompt

Fix Xiaomi MiMo thinking-mode multi-turn tool-call replay for https://github.com/openclaw/openclaw/issues/81419, and check whether the same fix covers https://github.com/openclaw/openclaw/issues/81318. Current main treats Xiaomi and custom `xiaomi-coding`/`xiaomimimo.com` OpenAI-completions endpoints as ordinary OpenAI-compatible providers, while Xiaomi’s docs require `reasoning_content` to be passed back on assistant tool-call history when thinking is enabled. Implement a narrow compatibility path that covers bundled `xiaomi` models and custom Xiaomi MiMo endpoints by provider id or baseUrl host, preserves/backfills `reasoning_content` for assistant tool-call replay when thinking is enabled, and strips or avoids it when thinking is off. Add regression coverage for built-in Xiaomi MiMo and a custom `xiaomi-coding` provider/baseUrl case, plus a non-Xiaomi control. Add a `CHANGELOG.md` fix entry for the user-visible provider bug.
