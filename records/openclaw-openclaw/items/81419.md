## Repair Work Prompt

Fix the Xiaomi MiMo multi-turn tool-call replay bug for reasoning models. Current main only backfills `reasoning_content` for DeepSeek V4 replay; MiMo OpenAI-compatible providers such as `xiaomi`, `xiaomi-coding`, and `xiaomi-orbit` also require DeepSeek-style `reasoning_content` on replayed assistant messages containing tool calls. Add a narrow Xiaomi/MiMo compatibility path without adding a user config knob, broad transcript migration, or unrelated provider behavior. Prefer reusing or generalizing the existing replay helper in `src/plugin-sdk/provider-stream-shared.ts` and wiring it from the Xiaomi plugin and/or the embedded runner custom OpenAI-compatible fallback so custom `xiaomimimo.com` base URLs are covered. Add focused regression coverage for bundled Xiaomi and custom Xiaomi-compatible provider replay behavior. Coordinate with, but do not fold in, the adjacent response-normalization PR at https://github.com/openclaw/openclaw/pull/60304. Add a single `CHANGELOG.md` fix entry if the repair changes user-visible provider behavior.
