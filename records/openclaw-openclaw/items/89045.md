## Summary

Keep this PR open: it targets a real P1 session/message-loss bug, but the current patch can still clear a live recovery operation during a second visible inbound turn, and the external PR still lacks real after-fix behavior proof.

## What This Changes

The PR clears terminal session lifecycle fields on visible turns, retries reply admission after clearing stale active operations, mirrors recovery in the agent API path, and adds a zero-count dispatch warning.

## Best Possible Solution

Clear terminal session state atomically before admitting the replacement operation, or guard force-clearing so it can only remove the stale operation that belongs to the terminal state, then prove the real channel recovery path.

## Reproduction Assessment

Yes. Source inspection of the PR diff and current dispatch flow shows a high-confidence race where a second visible turn can clear a newly admitted recovery operation before session state is persisted as non-terminal.

## Solution Assessment

No. The PR is a plausible repair for the stuck-session bug, but stale-operation clearing is not yet safely ordered with terminal-state persistence, so it is not the best mergeable fix as written.
