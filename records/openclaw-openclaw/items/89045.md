## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Recover terminal sessions before waiting on active runs:** `src/auto-reply/reply/dispatch-from-config.ts:1417-1439`
  - body: The new force-clear branch only runs after `admitReplyTurn` has already returned `active-run`. For visible turns, that admission waits on the active operation first; when diagnostics-backed stuck-session recovery is disabled it has no timeout, and with default diagnostics it waits the stuck-session abort window before this branch can run. A failed terminal session with a stale active operation can therefore still hang or silently miss the expected immediate recovery. Move the same-session terminal check before the blocking wait, or give terminal recovery its own bounded admission path independent of diagnostics.
  - confidence: 0.84
