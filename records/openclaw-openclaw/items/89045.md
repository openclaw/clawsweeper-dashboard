## Best Possible Solution

## Reproduction Assessment

Yes for the review finding: source inspection shows heartbeat/control active-run admissions are skipped by contract, while the PR force-clears active operations for terminal snapshots without checking `replyTurnKind`. ## Solution Assessment

No: the session recovery shape is plausible, but the dispatch retry is too broad until it is explicitly visible-only. Clearing terminal state in `initSessionState` alone is not enough because the stale reply registry can still block admission, so the best fix is a narrowly guarded registry clear plus proof.
