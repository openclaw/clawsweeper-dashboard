Full review comments:

- **[P1] Recover terminal runs before waiting:** `src/auto-reply/reply/dispatch-from-config.ts:1418-1427`
  - body: The new force-clear branch only runs after the first `admitReplyTurn()` call. For visible turns, that admission waits on the existing active run with `visibleReplyRecoveryWaitMs` before it can return `skipped`; with the default diagnostics values this can be minutes, and with no finite wait it can be unbounded. A failed terminal session with a stale same-session operation can therefore still hang or silently drop the next group message instead of recovering immediately. Move the terminal same-session/non-`terminalRecovery` recovery check before the wait path, or admit this specific terminal snapshot with no active-run wait.
