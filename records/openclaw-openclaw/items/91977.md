## Summary

Keep this PR open for revision: the identity concern is credible, but the patch changes Apple-defined Liquid Glass rendering semantics by removing foreground processing from shared card and control content, including the interactive workspace-link surface.

## What This Changes

The PR moves iOS 26 light-mode Liquid Glass application from the entire surfaced content to only the background layer for shared pro panels and controls.

## Best Possible Solution

Keep `glassEffect` on the composed surface content and preserve structural identity by selecting a stable iOS 26 glass value—such as `.identity` outside light mode—then prove appearance switching and workspace-link interaction in a real simulator run.

## Reproduction Assessment

Yes for the patch regression: the changed receiver and Apple's documented modifier contract make the loss of foreground processing source-reproducible. The original structural-state churn itself has not been demonstrated in a live interaction.

## Solution Assessment

No. The goal is valid, but relocating the effect solves identity by changing rendering semantics; retaining the existing receiver and switching the `Glass` value is narrower and safer.
