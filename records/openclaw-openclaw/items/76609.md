## What This Changes

The PR adds a cold-registry fallback in `resolvePluginTools`, a regression test for tool resolution without pre-warming, and a changelog fix entry.

## Best Possible Solution

Land a narrow fix that cold-loads only the manifest-selected plugin ids before resolving tool factories, with regression coverage for the `plugins.load.paths` cold-registry path.

## Reproduction Assessment

Yes, source-reproducible: current main resolves manifest-owned plugin ids and then returns early when no loaded channel or active registry exists. ## Solution Assessment

Yes; using `ensureStandaloneRuntimePluginRegistryLoaded` inside `resolvePluginTools` is the narrowest maintainable fix because it reuses the existing scoped loader and preserves the fast loaded-registry path. A more exact origin `config` test would improve coverage but does not make the patch incorrect.
