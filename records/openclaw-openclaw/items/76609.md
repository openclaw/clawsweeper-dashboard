## Summary

Keep open: current main still has the cold-registry early return, and this PR is the right narrow repair direction, but the submitted patch should preserve diagnostics when the retry still cannot produce a usable registry before it lands.

## What This Changes

The PR adds a cold-registry fallback in `resolvePluginTools`, a regression test, and a changelog entry for path-based plugin tools loaded from `plugins.load.paths`.

## Best Possible Solution

Land the scoped tool-registry fallback after preserving loader diagnostics on failed cold loads and aligning the regression fixture with `origin: "config"`.

## Reproduction Assessment

Yes. Source inspection shows current main returns cached tools when no loaded plugin tool registry exists, and the linked report's `plugins.load.paths` path is covered by the documented plugin metadata scope.

## Solution Assessment

No, not as submitted. The fallback is the right maintainable approach, but the second-null branch should surface the loader diagnostics it just created and the regression test should mirror the reported config-origin plugin path.
