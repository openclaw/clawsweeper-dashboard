## Summary

Keep open. Current main still defaults/falls back to WhatsApp `dmPolicy: "pairing"`, and a live unknown direct message can still create and send a pairing reply; the personal-number setup prompt only covers guided new setup, not existing or unguided/defaulted configs.

## What This Changes

The issue requests safer personal WhatsApp behavior so unknown contacts do not receive pairing-code replies from the owner’s own number by default.

## Best Possible Solution

Define an explicit WhatsApp account-intent contract, with personal-number accounts landing on owner-only allowlist/self-chat behavior and dedicated bot accounts using pairing only when that intent is explicit and documented.

## Reproduction Assessment

Yes. Source inspection and existing regression tests give a high-confidence path: a WhatsApp DM policy that resolves to `pairing` sends a pairing reply for a recent unknown direct message.

## Solution Assessment

No. The current guided setup prompt is a useful mitigation, but the narrow maintainable fix still needs explicit account intent plus migration or doctor behavior for existing personal-number configs.
