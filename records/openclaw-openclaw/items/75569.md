## Summary

Keep open. Current main still defaults and falls back to WhatsApp `dmPolicy: "pairing"`, and a live non-allowlisted DM can still create and send a pairing reply; the personal-number setup prompt only mitigates guided new setups, not existing or defaulted configs.

## What This Changes

The issue requests safer personal WhatsApp behavior so unknown contacts do not receive pairing-code replies from the owner’s own number by default.

## Best Possible Solution

Define an explicit WhatsApp account-intent contract: personal-number accounts should land on owner-only allowlist/self-chat behavior, while dedicated bot accounts keep pairing only when that intent is explicit and documented.

## Reproduction Assessment

Yes. Source inspection and existing regression tests give a high-confidence reproduction path: a WhatsApp DM policy that resolves to `pairing` sends a pairing reply for a recent unknown direct message.

## Solution Assessment

No. The current guided setup prompt is only a partial mitigation; the maintainable fix needs explicit account intent plus migration or doctor behavior for existing personal-number configs.
