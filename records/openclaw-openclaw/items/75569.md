## Summary

Keep open. Current main still defaults WhatsApp DM access to `pairing` and the live unknown-DM path can send the reported pairing reply; the personal-phone setup flow is a partial mitigation for guided new configs, not a fix for existing or unguided personal-number setups.

## What This Changes

Request changing personal WhatsApp account defaults so unknown contacts are blocked by allowlist/self-chat configuration instead of receiving pairing-code replies.

## Best Possible Solution

Define explicit WhatsApp account intent: personal-number accounts should use `allowlist` with the owner number in `allowFrom` and `selfChatMode`, while dedicated bot accounts keep pairing only after that intent is explicit, with doctor/docs guidance for existing configs.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence reproduction path: no explicit WhatsApp `dmPolicy` resolves to `pairing`, a live unknown DM enters the pairing branch, and regression tests assert a pairing reply is sent.

## Solution Assessment

No for the current behavior. The guided personal-phone setup prompt is useful, but the maintainable fix needs explicit intent plus migration/repair handling so existing and manual personal-number configs stop sending unsolicited pairing replies.
