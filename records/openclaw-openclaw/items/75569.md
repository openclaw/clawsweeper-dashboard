## Summary

Keep open. Current main and the latest shipped schema still preserve WhatsApp `pairing` as the default/fallback, and the live unknown-DM path still sends a pairing-code reply when that policy is active; guided personal-phone setup is only a mitigation for users who choose that path.

## What This Changes

The issue requests changing personal WhatsApp defaults or setup so unknown contacts are not auto-sent pairing codes from a personal account.

## Best Possible Solution

Define explicit WhatsApp account intent and preserve it in setup/config behavior: personal-number accounts should default to owner-only allowlist/self-chat behavior, while dedicated/shared bot accounts should use pairing only when that intent is explicit and documented.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path: current defaults/fallbacks can resolve to `pairing`, and the live unknown-DM branch sends a pairing reply when that policy is active.

## Solution Assessment

No. The existing guided setup prompt is a partial mitigation, but the remaining default/fallback behavior needs an explicit product/security decision for personal versus dedicated account intent.
