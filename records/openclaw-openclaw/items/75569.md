## Summary

Keep open. Current main still defaults WhatsApp DM access to `pairing` when no explicit policy is configured, and the live unknown-DM path still creates a pairing challenge and sends the reply back over WhatsApp. The guided setup personal-phone path is a useful mitigation, but it does not remove the reported default/runtime behavior for existing or unguided personal-number setups.

## What This Changes

The issue requests changing personal WhatsApp account defaults so unknown contacts are not sent pairing codes from the user's personal number, either by detecting personal-account intent or asking during setup and writing an allowlist owner baseline.

## Best Possible Solution

Define an explicit WhatsApp account intent in setup/config so personal-number accounts use `dmPolicy: "allowlist"` with the owner number in `allowFrom`, while dedicated/shared bot numbers use pairing only after that intent is explicit. Pair the behavior change with doctor/docs guidance for existing personal-number configs.

## Reproduction Assessment

Yes. Current source and tests give a high-confidence reproduction path: no WhatsApp config resolves to `pairing`, an unknown live DM takes the pairing branch, and the no-config inbox test asserts that a pairing prompt is sent to the unknown sender.

## Solution Assessment

Unclear. The existing setup prompt is a partial mitigation, but the requested automatic personal-account detection is not clearly the safest durable fix without an explicit account-intent or migration decision.
