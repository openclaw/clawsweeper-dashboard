## Summary

Keep open. Current main still leaves WhatsApp configs without an explicit personal-number choice on the `pairing` schema/runtime fallback, and the live unknown-DM path still sends a WhatsApp pairing reply; the guided setup prompt only mitigates new interactive personal-phone setup.

## What This Changes

The issue requests changing or prompting for the WhatsApp `dmPolicy` default so personal-number accounts do not auto-send pairing-code replies to unknown contacts.

## Best Possible Solution

Define explicit WhatsApp account intent: personal-number accounts should resolve to owner-only allowlist/self-chat behavior, while dedicated/shared bot accounts keep pairing only when that intent is explicit and documented for existing configs.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence reproduction path: absent or fallback WhatsApp `dmPolicy` resolves to `pairing`, and a live unknown DM in that mode sends a pairing reply.

## Solution Assessment

No. The current setup prompt is a useful mitigation, but it is not the best complete fix because existing and unguided personal-number configurations still inherit the pairing fallback.
