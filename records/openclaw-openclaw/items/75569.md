## Summary

Keep open. Current main still has the reported WhatsApp behavior: no explicit WhatsApp DM policy resolves to `pairing`, and the live unknown-DM path creates and sends a pairing reply. The guided personal-phone setup and docs are useful mitigations, but they do not cover existing or unguided personal-number configurations.

## What This Changes

Request that personal WhatsApp accounts default to an allowlist/self-only policy instead of sending pairing-code replies to unknown contacts.

## Best Possible Solution

Add an explicit WhatsApp account intent in setup/config. Personal-number intent should persist `dmPolicy: "allowlist"`, `selfChatMode: true`, and an owner `allowFrom`; dedicated/shared bot intent can keep pairing, with doctor/docs guidance for existing personal-number configs.

## Reproduction Assessment

Yes. Source and regression tests give a high-confidence path: empty WhatsApp config resolves to `pairing`, an unknown live DM reaches the pairing branch, and the no-config inbox test asserts the pairing prompt is sent to the unknown sender.

## Solution Assessment

Unclear. The existing guided setup prompt is a partial mitigation, but the durable fix likely needs explicit account intent plus migration/doctor guidance rather than only a runtime default flip or implicit personal-number detection.
