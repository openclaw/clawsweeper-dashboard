## Summary

Keep open. Current main still resolves an omitted WhatsApp `dmPolicy` to `pairing` and the live unknown-DM branch still creates and sends a pairing reply; the guided personal-phone setup path is a mitigation, not a complete policy fix for existing or unguided personal-number configs.

## What This Changes

The issue asks OpenClaw to avoid defaulting personal WhatsApp accounts to pairing because unknown contacts can receive pairing-code replies from the user's own number.

## Best Possible Solution

Define explicit WhatsApp account intent so personal-number accounts get allowlist/self-chat behavior and dedicated/shared bot accounts retain pairing only when that intent is explicit and documented for existing configs.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted WhatsApp `dmPolicy` resolves to pairing, and current tests assert that live unknown DMs in pairing mode create and send a pairing reply.

## Solution Assessment

No. The current setup prompt is a partial mitigation, but the best fix needs a maintainer policy for existing configs, unguided setup, and dedicated-number compatibility.
