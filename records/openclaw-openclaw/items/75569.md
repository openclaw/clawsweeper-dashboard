## Summary

Keep open. Current main still defaults WhatsApp DM access to `pairing` and can send a pairing-code reply to an unknown live DM; the personal-phone setup prompt is a mitigation for guided setup, not a fix for existing or unguided personal-number configurations.

## What This Changes

The issue requests a safer WhatsApp default or setup flow so personal-number accounts do not send pairing-code replies to arbitrary unknown contacts.

## Best Possible Solution

Define explicit WhatsApp account intent: personal-number accounts should use `allowlist`, owner `allowFrom`, and self-chat safeguards, while pairing remains available only for explicit dedicated/shared bot setups with doctor/docs guidance for existing configs.

## Reproduction Assessment

Yes. Current source and tests give a high-confidence path: absent/default WhatsApp `dmPolicy` resolves to `pairing`, a live non-self DM takes the pairing branch, and tests assert a pairing reply is sent.

## Solution Assessment

No for current behavior. The guided setup prompt is useful, but the maintainable fix needs intent/default/migration semantics so existing and unguided personal-number configs are not left on the pairing default.
