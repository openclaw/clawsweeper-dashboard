## Summary

Keep open. Current main still defaults WhatsApp DM access to `pairing` and the live unknown-DM path can send the reported pairing reply; the personal-phone setup flow is a mitigation for guided/new configuration, not a fix for existing or unguided personal-number setups.

## What This Changes

The issue requests safer WhatsApp personal-account defaults so unknown contacts do not receive pairing-code replies from the user's own number.

## Best Possible Solution

Define explicit WhatsApp account intent in setup/config: personal-number accounts should use `allowlist` with the owner number in `allowFrom`, while shared/dedicated bot numbers should use pairing only after that intent is explicit, with doctor/docs guidance for existing personal configs.

## Reproduction Assessment

Yes. Current source and regression tests provide a high-confidence reproduction path: no WhatsApp config resolves to `pairing`, an unknown live DM takes the pairing branch, and the no-config inbox test asserts a pairing prompt is sent.

## Solution Assessment

No for the current behavior. The guided setup prompt is a useful partial mitigation, but it does not protect existing or unguided personal-number configs; the safer direction is an explicit intent/migration model rather than relying only on the current default.
