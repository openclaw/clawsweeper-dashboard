## Summary

Keep open. Current main still allows existing or unguided WhatsApp configs to resolve to the `pairing` default/fallback, and live unknown direct messages can still receive a pairing-code reply; the personal-number setup prompt is a useful mitigation but not a complete fix.

## What This Changes

Personal WhatsApp accounts should avoid unsolicited pairing-code replies to unknown direct-message senders by default.

## Best Possible Solution

Define and preserve explicit WhatsApp account intent: personal-number accounts should use owner-only allowlist/self-chat behavior, while dedicated/shared bot accounts keep pairing only when that intent is explicit and documented for existing configs.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path: default/fallback `dmPolicy` remains `pairing`, and the live unknown-DM branch creates and sends a WhatsApp pairing reply.

## Solution Assessment

No. The current setup prompt is a partial mitigation, but it is not the complete fix because existing or unguided personal-number configurations can still resolve to pairing.
