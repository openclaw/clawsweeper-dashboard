## Summary

Keep open. Current main still has a default/no-config WhatsApp path where unset `dmPolicy` resolves to `pairing` and a live unknown non-self DM sends a pairing reply; guided setup now mitigates personal-phone mode, but it does not eliminate the reported runtime/default behavior.

## What This Changes

The issue requests that personal WhatsApp account setups avoid default pairing-mode replies to unknown contacts by using allowlist/self-chat owner defaults or an explicit setup intent prompt while retaining pairing for dedicated bot accounts.

## Best Possible Solution

Define and persist WhatsApp account intent so personal-number setups never send pairing replies to arbitrary contacts, while dedicated/shared bot-number mode can still opt into pairing. Existing configs should get a clear setup, doctor, and docs path rather than relying on an ambiguous runtime guess.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence reproduction path: no WhatsApp config or unset `dmPolicy` resolves to pairing, an unknown live non-self DM takes the pairing branch, and the no-config inbox test asserts that a pairing prompt is sent.

## Solution Assessment

Unclear. The current guided setup prompt is a good partial mitigation, but the issue is not fully solved until personal account intent is explicit and enforced for existing/default runtime paths; an automatic identity heuristic alone may be less maintainable.
