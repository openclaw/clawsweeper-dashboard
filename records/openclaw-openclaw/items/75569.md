## Summary

Keep open. Current main still leaves WhatsApp schema/runtime fallback at `pairing`, and the live unknown-DM branch still creates and sends pairing replies, so the personal-number privacy issue is not fully solved by setup mitigation.

## What This Changes

Request changing WhatsApp personal-account defaults so unknown contacts do not receive pairing-code replies from the user's own number.

## Best Possible Solution

Define explicit WhatsApp account intent: personal-number accounts should default to owner-only allowlist/self-chat behavior, while dedicated bot accounts keep pairing only when that intent is explicit and documented.

## Reproduction Assessment

Yes. Source inspection is high-confidence: unset WhatsApp DM policy still resolves to `pairing`, and current regression coverage asserts that live unknown DMs in pairing mode send a pairing reply.

## Solution Assessment

No. The personal-phone setup prompt is a useful mitigation, but it is not the complete solution because existing or unguided personal-number configurations can still resolve to pairing.
