## Summary

Keep open. Current main still leaves unguided or existing WhatsApp configs on the `pairing` default/fallback, and unknown live DMs can still get a WhatsApp pairing-code reply; the personal-number setup prompt mitigates only the guided new-setup path.

## What This Changes

The issue asks OpenClaw to avoid defaulting personal WhatsApp accounts to unsolicited DM pairing replies and instead use an owner-only allowlist or an explicit setup intent choice.

## Best Possible Solution

Define explicit WhatsApp account intent: personal-number accounts should use owner-only allowlist/self-chat behavior, while dedicated/shared bot accounts keep pairing only when that intent is explicit and documented for existing configs.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence reproduction path: default/fallback policy is `pairing`, and the live unknown-DM path sends a pairing reply when access resolves to that mode.

## Solution Assessment

No. The current guided setup prompt is a useful mitigation, but it is not the best complete fix because it does not protect existing or unguided personal-number configurations from the pairing default.
