## Summary

Keep open: current main still has the reported source-level bug, because daily ingestion strips managed dreaming blocks with a heading boundary that can fire on REM’s own internal headings before the explicit end marker.

## What This Changes

The issue reports that managed daily dreaming block stripping stops at internal Markdown headings instead of the REM end marker.

## Best Possible Solution

Update the memory-core strip loop to strip well-formed managed blocks through their explicit end marker, keep a guarded fallback only for malformed blocks, and add REM regression coverage for internal headings.

## Reproduction Assessment

Yes, source-reproducible: a daily REM block containing `### Reflections` will hit the current heading-boundary branch before the `<!-- openclaw:dreaming:rem:end -->` marker, leaving the rest of the REM block visible to daily ingestion. I did not execute a test because this review must not write artifacts.

## Solution Assessment

Yes: the issue’s option (c) is the narrowest maintainable fix for well-formed managed blocks, with the existing malformed-block boundary behavior preserved as a fallback rather than used inside valid blocks.
