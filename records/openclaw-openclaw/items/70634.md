## Summary

Keep open. Current `main` still has single-tier FIFO command-lane ordering, while human/channel turns and `sessions_send` inter-session turns are not separated at enqueue time; the KinthAI production comment also corroborates the starvation pattern as a real runtime issue.

## What This Changes

The issue requests priority routing so human-originated turns can overtake agent-originated inter-session turns in per-agent command lanes.

## Best Possible Solution

Add a generic command-lane priority/tier option, keep same-tier FIFO stable, mark trusted inter-session agent work as background, and cover human-over-agent ordering in focused queue and session-send tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to block one per-session lane entry, enqueue one or more `sessions_send`/`inter_session` turns, then enqueue a later human/channel turn on the same lane and observe FIFO ordering run the human turn after earlier agent work.

## Solution Assessment

Yes for the first repair. A two-tier command-lane priority keyed from trusted provenance is the narrowest maintainable fix; token budgets and broader loop throttling can remain follow-up policy work unless maintainers expand scope.
