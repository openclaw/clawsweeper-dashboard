## Summary

Keep open. Current main still uses a single FIFO command-lane queue without priority or source fields, so a later human/channel turn cannot overtake queued agent-originated `sessions_send` work; the related priority-queue PR at https://github.com/openclaw/openclaw/pull/80199 is still open and protected by maintainer handling.

## What This Changes

The issue requests priority routing so human-originated messages outrank agent-to-agent work in per-agent lane queues.

## Best Possible Solution

Extend the command queue priority contract so human/channel ingress outranks trusted `sessions_send` and `inter_session` work on the same lane while preserving same-tier FIFO and coordinating with the open priority-queue PR.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: block a per-session lane, enqueue `inter_session` `sessions_send` work, then enqueue a later human/channel turn on that same lane; FIFO `push`/`shift` runs the older agent work first.

## Solution Assessment

Yes for the priority-ordering direction, but the maintainable fix should extend the generic queue-priority contract and classify inter-session work below human/channel ingress rather than adding an ad hoc front-of-queue special case.
