## Summary

Keep open. Current main and the latest release have generic foreground/background run priority, but `sessions_send` inter-session traffic still flows into embedded execution as `trigger: "user"`, so the per-session queue cannot prefer true human channel ingress over agent-originated loop traffic.

## What This Changes

The issue requests source-aware queue priority so human-originated messages are not indefinitely delayed behind agent-to-agent loop traffic.

## Best Possible Solution

Define and implement a source-aware scheduler policy where true human channel ingress wins over agent-originated inter-session loop work, then prove it across per-session and global lanes with regression coverage.

## Reproduction Assessment

Yes, from source inspection: `sessions_send` traffic is marked as inter-session provenance but still reaches embedded execution with `trigger: "user"`, which maps to the same foreground queue priority as human turns. I did not run the reporter's live six-agent Discord loop in this read-only review.

## Solution Assessment

No, the current generic foreground/background priority is not a complete fix for this issue because it does not distinguish human ingress from agent-originated loop work. The proposed two-tier queue is plausible, but the best OpenClaw fix should be chosen across provenance, session lanes, queue steering, and loop back-pressure.
