## Summary

Keep open. Current main still has strict FIFO command-lane behavior for the relevant per-session lanes, and `inter_session` provenance from `sessions_send` is not used to place queued agent-originated work behind later human/channel turns. The KinthAI production comment supports the same starvation pattern and points toward the same narrow two-tier queue fix.

## What This Changes

Prevent human/channel messages from being indefinitely delayed behind agent-originated `sessions_send` loops in per-session command lanes.

## Best Possible Solution

Add a generic command-lane tier option, keep existing callers foreground by default, mark trusted `sessions_send` / `inter_session` A2A turns as background, and cover foreground-over-background ordering plus stable same-tier FIFO in focused queue and gateway/tool tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to block one per-session lane entry, enqueue one or more `inter_session` A2A turns, then enqueue a later human/channel turn on the same session lane; current `push`/`shift` FIFO ordering runs the human turn after the earlier agent-originated work.

## Solution Assessment

Yes. A two-tier command-lane priority keyed from trusted provenance is the narrowest maintainable first fix; token-budget throttling and loop back-pressure should remain follow-up policy work unless maintainers explicitly broaden the scope.

## Repair Work Prompt

Fix the per-session command-lane starvation reported at https://github.com/openclaw/openclaw/issues/70634. Implement a conservative two-tier priority in the command queue: preserve FIFO for equal-priority work, do not preempt already-active tasks, and allow later foreground human/channel/control turns to run before queued background agent-originated `sessions_send` / `inter_session` work for the same target session. Prefer a generic `CommandQueueEnqueueOptions` priority/tier seam in `src/process/command-queue.ts` and pass background priority from trusted `sessions_send` / A2A gateway paths rather than brittle payload inspection. Add focused tests proving foreground-over-background ordering and stable background FIFO, plus gateway/tool tests proving `sessions_send` and A2A turns are marked background. Do not bundle budget throttling, token back-pressure, provider policy, or unrelated channel/plugin changes. Add a user-facing `CHANGELOG.md` fix entry crediting @ewrurwteurwU and @kinthaiofficial unless maintainers decide this should remain internal-only.
