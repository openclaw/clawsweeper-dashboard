## Summary

Keep open: current main still drains each command lane from a single FIFO array, and `sessions_send` / `inter_session` provenance reaches execution context but not enqueue ordering; the production corroboration comment supports a narrow two-tier queue repair.

## What This Changes

Prioritize human-originated channel messages ahead of agent-to-agent `sessions_send` work in per-agent command lanes so A2A loops cannot starve operator turns.

## Best Possible Solution

Add a generic foreground/background command-lane tier, default existing human/channel ingress to foreground, mark trusted inter-session A2A work as background, and preserve FIFO within each tier.

## Reproduction Assessment

Yes. A high-confidence code-level repro is to block one per-session lane entry, enqueue one or more `inter_session` / `sessions_send` turns on that same lane, then enqueue a later human/channel turn; current `push`/`shift` FIFO runs the human turn after the earlier agent work.

## Solution Assessment

Yes for the first repair. A provenance-driven two-tier command-lane queue is the narrowest maintainable fix; throttling agent loops or token budgets can be handled separately if maintainers broaden the scope.
