## Summary

Keep open. Current main and the latest release still lack queue ordering that distinguishes human/channel turns from agent-originated `sessions_send` work, and the related open priority-queue PR does not yet solve the specific A2A-versus-human starvation path.

## What This Changes

The issue asks for human-originated messages to outrank agent-originated lane work when agent-to-agent loops keep filling the same session queue.

## Best Possible Solution

Extend the accepted priority-queue design so trusted `sessions_send` / `inter_session` work is lower priority than human/channel ingress on the same lane, with FIFO preserved within each tier and a starvation guard for background work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: block a per-session lane, enqueue one or more `inter_session` turns, then enqueue a later human/channel turn on the same lane; current FIFO `push`/`shift` ordering runs the human turn after the earlier agent-originated work.

## Solution Assessment

Yes for the narrow ordering direction, but the active generic priority PR is not sufficient by itself. The maintainable fix should use the priority queue contract and classify trusted inter-session work below human/channel ingress rather than adding a separate ad hoc queue.
