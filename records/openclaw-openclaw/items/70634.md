## Summary

Keep open. Current main and the latest release still use a FIFO command lane with no priority/source contract, and the open priority PRs track adjacent generic scheduling without yet settling human-vs-`inter_session` loop ordering.

## What This Changes

Request priority scheduling so human/channel messages can overtake agent-originated `sessions_send` work in the same command lane during agent loops.

## Best Possible Solution

Define a queue-priority/source contract that ranks human/channel ingress above `inter_session`/`sessions_send` work on the same lane, preserves same-class FIFO, and coordinates with the active command-priority PRs.

## Reproduction Assessment

Yes. Source inspection on current main shows same-lane entries are appended with `push` and drained with `shift`, while `sessions_send` A2A work uses nested per-session lanes and no priority/source option, so a later human turn cannot overtake older agent work.

## Solution Assessment

Unclear on the exact implementation. A priority contract is the right direction, but maintainers need to decide how human/channel ingress, `inter_session` work, active-run interruption, and loop back-pressure should interact.
