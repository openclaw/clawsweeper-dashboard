## Summary

Keep open. Current main still uses a single FIFO command-lane queue with no source/priority field, so later human/channel turns cannot overtake earlier agent-originated `sessions_send` work; the overlapping priority-queue work in https://github.com/openclaw/openclaw/pull/80199 is still open and does not close this issue.

## What This Changes

Ensure human-originated channel messages cannot be starved behind agent-originated `sessions_send` / `inter_session` work in per-agent lane queues.

## Best Possible Solution

Extend the queue priority contract so human/channel ingress outranks trusted `sessions_send` / `inter_session` work on the same lane, while preserving same-tier FIFO and coordinating with the open priority-queue PR.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: block a per-session lane, enqueue one or more `inter_session` `sessions_send` turns, then enqueue a later human/channel turn on that same lane; current `push`/`shift` FIFO runs the older agent work first.

## Solution Assessment

Yes for the priority-ordering direction, but the maintainable fix should extend the generic queue-priority contract and classify trusted inter-session work below human/channel ingress rather than adding an ad hoc front-of-queue special case.
