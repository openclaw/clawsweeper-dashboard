## What This Changes

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, the parser-boundary fix is the narrowest maintainable solution because both native routing and extraction fallback consume the same parsed pageNumbers contract. A native-only guard would leave the fallback extractor with the same empty-selection ambiguity.
