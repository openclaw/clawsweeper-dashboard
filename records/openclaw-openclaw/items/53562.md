## Summary

Keep open. Current main has a newer `voice.agentSession.target` route option, but the requested per-`autoJoin` text-channel transcript/reply delivery is still not present, and the target-repo implementation PR was closed for channel-owner direction rather than merged.

## What This Changes

The issue requests an optional Discord voice `autoJoin.sessionChannelId` that routes voice transcripts and replies through a companion text channel while preserving voice playback.

## Best Possible Solution

Resolve the product shape first: either fold this into the broader Discord voice-as-I/O design or land a revised, owner-approved narrow config that validates a same-guild text target and delivers text without weakening Discord delivery safety.

## Reproduction Assessment

Yes. Source inspection is enough: current main has no `autoJoin.sessionChannelId` type/schema/docs support, and the batch voice ingress still runs with `deliver: false`. I did not run a live Discord session because the missing config and delivery paths are explicit in source.

## Solution Assessment

Unclear. The proposed field is plausible, but current main now has `voice.agentSession.target`, and the matching PR was closed for channel-owner direction, so the best implementation depends on maintainer product judgment.
