## Best Possible Solution

Choose one canonical Discord voice bridge design, then land a revised implementation with typed config, explicit delivery target semantics, same-guild text-channel validation, mention handling, docs, and focused Discord voice/delivery tests.

## Solution Assessment

Unclear. A per-autoJoin field is plausible, but current main already has global `voice.agentSession` target routing, so maintainers should first choose the canonical config and text-delivery semantics.
