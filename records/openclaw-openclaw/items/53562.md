## Summary

Keep this open. Current main still lacks the requested Discord voice `autoJoin[].sessionChannelId` config surface and still routes voice transcript turns through the joined voice channel with text delivery disabled; an open implementation PR already references this issue, so maintainers should review, land, or supersede that path rather than closing the issue now.

## What This Changes

Request an optional Discord voice `autoJoin[].sessionChannelId` field so auto-joined voice sessions can route transcripts and replies through a companion text channel while preserving voice-channel audio playback.

## Best Possible Solution

Implement this inside the Discord plugin/config surface: add a typed and schema-validated `autoJoin[].sessionChannelId`, propagate it through auto-join/session creation, route the agent session to that text channel when configured, enable delivery only for the paired text route, keep voice playback unchanged, update Discord docs/config metadata, and add focused config plus voice manager/segment tests.

## Reproduction Assessment

Yes. The feature gap is reproducible by static inspection on current main: `autoJoin[].sessionChannelId` is absent from the type and strict schema, `autoJoin()` cannot pass it to `join()`, route resolution still uses the voice channel, and voice segment delivery remains `false`.

## Solution Assessment

Unclear. The requested field is a plausible narrow solution, but the implementation should be reconciled with #73699 and include target-channel validation rather than only changing the route and flipping `deliver` on globally.
