## Summary

Keep open. Current main still lacks the requested `autoJoin.sessionChannelId` config and automatic Discord text transcript/reply delivery; `voice.agentSession.target` partially addresses shared routing, but the remaining bridge behavior is still a product and safety decision with an open implementation PR at https://github.com/BingqingLyu/openclaw/pull/1341 and broader tracking at https://github.com/openclaw/openclaw/issues/73699.

## What This Changes

Requests an optional Discord voice `autoJoin.sessionChannelId` setting that routes voice transcripts and replies into a paired text channel while audio continues in the voice channel.

## Best Possible Solution

Decide the Discord voice bridge contract, likely building on `voice.agentSession.target`, then add opt-in text mirroring with validated targets, speaker-attributed transcript posting, reply delivery plus TTS playback, docs, changelog, and focused tests.
