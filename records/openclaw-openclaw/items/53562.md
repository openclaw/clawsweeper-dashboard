## Summary

Keep open. Current main partially overlaps the request through `voice.agentSession.target`, but it still does not expose an `autoJoin.sessionChannelId` setting or post Discord voice transcripts/replies into a text channel, and an open implementation PR plus the broader bridge issue need maintainer review.

## What This Changes

The issue requests an optional Discord voice `autoJoin` text-channel routing field so voice transcripts and replies can be mirrored to a paired Discord text channel.

## Best Possible Solution

Build on the current `voice.agentSession.target` routing if maintainers keep that boundary, then add an opt-in bridge contract with validated text targets, speaker-attributed transcript posting, reply delivery, voice playback, docs, changelog, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested autoJoin field and text mirroring behavior.

## Solution Assessment

Unclear. A per-autoJoin `sessionChannelId` is a concrete proposal, but current main now has `voice.agentSession.target`, so maintainers should decide whether the bridge belongs there before adding a second routing shape.
