## Summary

Keep this open: current main still lacks the requested per-auto-join `sessionChannelId` field and automatic visible Discord text transcript/reply delivery. The newer `voice.agentSession` surface partially covers session routing, but the same-repository implementation PR was closed for channel-owner direction and the remaining fork PR is conflicting, so this still needs maintainer product/security review.

## What This Changes

The issue requests an optional Discord voice `autoJoin` text-channel target so voice transcripts and replies can route to a companion Discord text channel while audio stays in voice.

## Best Possible Solution

Choose one canonical Discord voice bridge design, preferably reusing `voice.agentSession` where it fits, then land a revised implementation with explicit text delivery semantics, same-guild text target validation, mention handling, docs, and focused voice/delivery tests.

## Reproduction Assessment

Yes. Source inspection shows `autoJoin` accepts only `guildId` and `channelId`, the strict schema rejects `sessionChannelId`, and Discord voice agent turns still use `deliver: false`; no live Discord voice run was attempted in this read-only review.

## Solution Assessment

Unclear. A per-auto-join field is plausible, but current main already has `voice.agentSession` for route selection, so maintainers should first choose the single canonical config and delivery model.
