## Label Justifications

- `P3`: This is a useful opt-in Discord voice enhancement with product-design work remaining, not an urgent regression.
- `impact:message-loss`: The request is about routing voice transcripts and replies into Discord text instead of keeping them audio-only.
- `impact:session-state`: The requested routing changes which Discord agent session receives voice turns and whether voice and text context are unified.

## Summary

Keep open. Current main only offers the global `voice.agentSession.target` route; `voice.autoJoin` has no per-entry `sessionChannelId`, voice ingress still uses `deliver: false`, and the open linked PR plus broader bridge issue mean the remaining work needs maintainer product direction.

## What This Changes

Add an optional `sessionChannelId` to Discord voice `autoJoin` entries so voice transcripts and agent replies can be mirrored to a paired text channel while audio playback continues.

## Best Possible Solution

Decide the canonical Discord voice bridge contract, likely building on `voice.agentSession.target` or a bridge-specific config, then implement opt-in validated text mirroring with speaker attribution, reply delivery plus TTS playback, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an opt-in Discord voice feature request. Source inspection on current main shows the requested autoJoin field and text-channel mirroring behavior are absent.

## Solution Assessment

Unclear. A per-autoJoin `sessionChannelId` is concrete, but current main already has `voice.agentSession.target` and https://github.com/openclaw/openclaw/issues/73699 tracks the broader bridge, so maintainers should choose the contract before implementation.
