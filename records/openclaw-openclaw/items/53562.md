## Summary

Keep this open: current main has `voice.agentSession` for target-session routing, but the requested automatic text-channel transcript/reply mirror is still not implemented, and the linked implementation PR is open but conflicting.

## What This Changes

The issue requests an optional Discord voice autoJoin text-channel session target so STT transcripts and agent replies can route to a companion text channel while audio remains in the voice channel.

## Best Possible Solution

Choose one canonical Discord voice text-bridge design that reuses `voice.agentSession` where possible, adds explicit transcript/reply delivery semantics if maintainers want visible text mirroring, and covers speaker attribution, tool access, docs, schema, and e2e tests.

## Reproduction Assessment

Yes. Source inspection shows the exact requested behavior is still missing: `autoJoin` entries have no text-channel field and voice agent turns still use `deliver: false`.

## Solution Assessment

Unclear. A direct `autoJoin.sessionChannelId` field is plausible, but current main already has `voice.agentSession`, so the best fix should avoid adding a competing config source without maintainer product approval.
