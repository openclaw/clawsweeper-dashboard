## Summary

Keep this PR open for maintainer review: the code change is focused and matches the signed-thinking replay failure, but it intentionally addresses replay recovery only while the linked report also describes write-side prevention.

## What This Changes

The PR changes session replay sanitization so Anthropic, Amazon Bedrock, and Anthropic Vertex strip invalid thinking blocks even when signature preservation is disabled, with regression tests for signed and unsigned-provider cases.

## Best Possible Solution

Land the replay-side recovery if maintainers accept that scope, and keep or create a separate write-side persistence-prevention follow-up if that guarantee is still required.

## Reproduction Assessment

Yes, at source level: current main gates `stripInvalidThinkingSignatures` on `policy.preserveSignatures`, so an Anthropic-style replay history with an unsigned thinking block can survive sanitization when that policy is false. I did not execute a current-main failing run in this read-only review.

## Solution Assessment

Yes for replay recovery: applying the existing sanitizer only to signed-thinking providers is a narrow maintainable fix with negative coverage for unsigned Anthropic-message transports. It is not the full write-side prevention requested in the linked report.
