## Summary

Keep open: the Feishu patch has focused tests and supplied live CardKit proof, but it expands ordinary final-text delivery into CardKit cards for existing streaming-enabled users while a known open Feishu issue tracks that CardKit reply content is not searchable; the branch is also currently dirty against main because of an obsolete gateway hunk.

## What This Changes

The PR changes Feishu reply dispatch so streaming-enabled final and partial plain assistant replies use CardKit cards, discards stale streaming previews before final voice media, adds dispatcher tests, and includes an unrelated gateway cache typing hunk.

## Best Possible Solution

Land a rebased Feishu-only patch after maintainers either gate the plain-final card behavior, compose it with an opt-in searchable fallback, or explicitly accept the Feishu search tradeoff.

## Reproduction Assessment

Yes at source/proof level: current main keeps plain final text on the normal message path unless card rendering is otherwise selected, while the PR body shows a real Feishu p2p CardKit reply after the patch. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment
