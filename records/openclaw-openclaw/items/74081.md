## Summary

Keep open. The PR is a plausible config feature, but the current diff should not merge because it makes generic string substitution value-dependent, can break existing string-typed config fields, and can persist env-backed arrays during write-back.

## What This Changes

The PR adds generic JSON string-array coercion for whole-value `${VAR}` config substitutions and unit tests for the coercion boundaries.

## Best Possible Solution

Define an explicit or schema/path-aware env-backed list contract, preserve authored env references across read/write, then document and changelog the behavior.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR's generic string branch runs before validation, while existing write-back restoration cannot restore a parsed string env reference after the value becomes an array.

## Solution Assessment

No. Whole-value `${VAR}` is narrower than parsing every string, but a schema/path-aware list coercion or explicit list syntax is the safer maintainable contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate array coercion on list-typed config fields:** `src/config/env-substitution.ts:201-204`
  - body: `SINGLE_VAR_REFERENCE.test(value)` runs for every config string, so a string-only field such as `agents.defaults.systemPromptOverride: "${PROMPT}"` becomes an array when `PROMPT` contains JSON array text and then fails validation. Gate this on list-typed paths/schema metadata or use an explicit list syntax.
  - confidence: 0.86
- **[P2] Preserve env refs after array coercion:** `src/config/env-substitution.ts:204`
  - body: Returning `asArray` changes the value type from the parsed `${ALLOW_FROM}` string before write-back. `restoreEnvVarRefs` only restores same-type string/string or array/array leaves, so a read/write round trip can persist the resolved array and lose the authored env reference.
  - confidence: 0.91
- **[P3] Add the user-facing changelog entry:** `src/config/env-substitution.ts:194`
  - body: This is a user-facing config feature, and repo policy requires user-visible `feat` changes to add a `CHANGELOG.md` entry. The branch only changes config substitution code and tests, so release notes would miss the new config contract.
  - confidence: 0.88
