## Summary

Keep open: this PR still needs maintainer product judgment on the config contract, and the current patch introduces source-provable correctness and env-preservation regressions without real behavior proof.

## What This Changes

The PR changes config env substitution to coerce whole-value `${VAR}` references into `string[]` when the environment value is a JSON string array, with unit tests for the coercion boundaries.

## Best Possible Solution

Adopt an explicit or schema-aware list-env contract that only applies to list-typed paths, preserves authored env references during config writes, and documents the accepted behavior.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR's generic string branch would run before validation and can affect string-only fields, while the write-back path cannot restore a parsed string env reference after it becomes an array.

## Solution Assessment

No. Whole-value `${VAR}` is narrower than parsing every string, but without schema/path gating or an explicit list syntax plus write-back preservation, it is not the safest maintainable config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate array coercion to list-typed config paths:** `src/config/env-substitution.ts:200-203`
  - body: `SINGLE_VAR_REFERENCE.test(value)` runs for every config string, so a string-only field such as `agents.defaults.systemPromptOverride: "${PROMPT}"` becomes an array when `PROMPT` contains JSON array text and then fails validation. Gate this on list-typed schema/path metadata or use an explicit list syntax.
  - confidence: 0.86
- **[P2] Preserve env refs after array coercion:** `src/config/env-substitution.ts:203`
  - body: Returning `asArray` changes the value type from the parsed `${VAR}` string before write-back. `restoreEnvVarRefs` only restores same-type string/string or array/array leaves, so a config write can persist the resolved array and lose the authored env reference.
  - confidence: 0.91

## Security Review
