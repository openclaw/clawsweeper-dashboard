## Summary

Keep open. Current main does not implement the PR's requested JSON-array env coercion, and the branch is useful source work, but it should not merge as-is because the generic whole-string coercion changes the contract for every env-backed config string and can break env-reference preservation on write-back.

## What This Changes

The PR changes config env substitution so an exact `${VAR}` value can resolve to a `string[]` when the env value is a JSON string array, and adds boundary tests for that behavior.

## Best Possible Solution

If OpenClaw wants single-env-var list injection, implement it through an explicit and documented list contract or limit it to known list-typed config paths, then preserve authored env references during read/write round trips and add focused tests plus a changelog entry.

## Reproduction Assessment

Yes. The high-confidence static path is `resolveConfigForRead` substituting before validation, followed by `writeConfigFile` calling `restoreEnvVarRefs`; with the PR, a parsed string `${ALLOW_FROM}` can become an incoming array and miss the same-type restoration cases.

## Solution Assessment

No. The branch solves the container/Kubernetes list-injection use case, but a syntax-only global coercion is not the narrowest maintainable fix because exact `${VAR}` is already documented as ordinary substitution for any string-valued config field.

## Review Findings

Full review comments:

- **[P2] Preserve env refs when coercing arrays:** `src/config/env-substitution.ts:203`
  - body: Returning a `string[]` here means an authored `${ALLOW_FROM}` string no longer has the same type as the resolved value that `writeConfigFile` gives to `restoreEnvVarRefs`. That helper restores string/string and array/array leaves only, so a read/write round trip can persist the resolved array and lose the env reference. Add round-trip preservation for whole-ref arrays or keep the read-time value compatible with the parsed source.
  - confidence: 0.91
- **[P2] Gate array coercion on an explicit list contract:** `src/config/env-substitution.ts:200-203`
  - body: The `SINGLE_VAR_REFERENCE` check is global, so any existing string field written as `${VAR}` becomes an array when the env value happens to be JSON array text. String-valued fields such as channel `systemPrompt` can then fail validation or change behavior unexpectedly. Gate this on known list-typed paths/schema metadata, or use an explicit list-ref syntax.
  - confidence: 0.84

  - body: This is a user-facing config feature, and OpenClaw policy requires user-facing `feat` changes to add a `CHANGELOG.md` entry. The branch only touches config substitution code and tests, so release notes would miss the new config contract.
