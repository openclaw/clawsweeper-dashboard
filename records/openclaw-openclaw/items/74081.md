## Summary

Keep open. The feature is plausible, but the patch changes generic string substitution before validation, can break string-only config fields, can lose env-reference preservation on write-back, and is missing the required user-facing changelog entry.

## What This Changes

The PR adds JSON string-array parsing for whole-value `${VAR}` config substitutions and tests the coercion boundaries.

## Best Possible Solution

Define a schema/path-aware or explicit list-valued env substitution contract, preserve authored env refs across read/write, and document and changelog the behavior while keeping string-only fields string-only.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR's generic string branch runs before validation, and current write-back restoration cannot restore an authored string `${VAR}` once the incoming value has become an array.

## Solution Assessment

No. Whole-value `${VAR}` is narrower than parsing every string, but the maintainable fix should be schema/path-aware or use an explicit list syntax and must preserve env refs on write-back.
