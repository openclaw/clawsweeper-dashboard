## Summary

Keep open: current main does not implement this feature, and the PR is not merge-ready because the generic coercion branch can break string-typed config fields, can weaken env-reference preservation on writes, and still lacks real behavior proof.

## What This Changes

The PR adds JSON string-array coercion for whole-value `${VAR}` config strings and unit tests for the new env-substitution behavior.

## Best Possible Solution

Adopt an explicit or schema-aware list-env contract that only applies to list-typed paths, preserves authored env references during writes, and documents the accepted behavior.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR's generic string branch runs before validation and can affect string-only fields, while the write-back restoration path cannot recover a parsed string env reference after it becomes an array.

## Solution Assessment

No. Whole-value `${VAR}` is a plausible direction, but without schema/path gating or an explicit list syntax plus write-back preservation, this is not the narrowest maintainable config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate array coercion to list-typed config paths:** `src/config/env-substitution.ts:200-203`
  - body: `SINGLE_VAR_REFERENCE.test(value)` runs for every config string, so a string-only field such as `agents.defaults.systemPromptOverride: "${PROMPT}"` becomes an array when `PROMPT` contains JSON array text and then fails validation. Gate this on list-typed schema/path metadata or use an explicit list syntax.
  - confidence: 0.86
- **[P2] Preserve env refs after array coercion:** `src/config/env-substitution.ts:203`
  - body: Returning `asArray` changes the value type from the parsed `${VAR}` string before write-back. `restoreEnvVarRefs` only restores same-type string/string or array/array leaves, so a config write can persist the resolved array and lose the authored env reference.
  - confidence: 0.91
