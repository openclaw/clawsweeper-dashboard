## Summary

Keep open: the event-loop problem is source-plausible, but this draft PR lacks real behavior proof, mixes in unrelated agent prompt and plugin-test changes, and currently breaks synchronous credential consumers by returning promises from exported cached readers.

## What This Changes

The PR converts macOS keychain credential reads in `src/agents/cli-credentials.ts` to async `exec` calls, while also adding an unrelated last-message system-prompt anchor and loosening plugin registry contract tests.

## Best Possible Solution

Keep the fix focused on the proven gateway blocking path: profile the macOS workload, preserve existing credential/auth contracts or update every consumer with tests, and remove the unrelated prompt-anchor and contract-test loosening from this PR.

## Reproduction Assessment

No high-confidence live reproduction was established for this review. Current main contains synchronous macOS keychain reads and the linked issue has an event-loop warning, but no macOS Telegram trace ties those reads to the reported stall.

## Solution Assessment

No. Async keychain access may be part of the fix, but this PR changes exported synchronous credential APIs without updating consumers and includes unrelated security-sensitive prompt behavior plus weakened plugin contract tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Update all credential consumers for async reads:** `src/agents/cli-credentials.ts:453-459`
  - body: These exported cached readers are used synchronously through provider-auth and auth/profile code. Returning a Promise here makes callers treat a pending Promise as credentials or read fields from it, so auth discovery and labels break unless every consumer and contract is converted or a sync-compatible wrapper is preserved.
  - confidence: 0.93
- **[P1] Keep user text out of extraSystemPrompt:** `src/agents/pi-embedded-runner/run.ts:357-374`
  - body: This copies `params.prompt` into `extraSystemPrompt`, which the runner appends to the actual system prompt. Labeling it “untrusted” does not preserve role isolation, so channel/user text can survive compaction as system-role content and steer tool-enabled agents; keep latest-message anchoring in user-role/history context.
  - confidence: 0.9
- **[P2] Preserve exact plugin contract coverage:** `src/plugins/contracts/registry.contract.test.ts:21-23`
  - body: Changing the registry helper to `arrayContaining` allows the contract registry to contain plugin ids that are no longer backed by bundled manifests. That masks the drift these contract tests are supposed to catch; keep exact equality or add an explicit allowlist for intentional drift.
  - confidence: 0.86
