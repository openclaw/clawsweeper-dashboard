## Summary

Keep open: the macOS keychain stall is plausible and the PR now supplies copied live-output proof, but the patch still breaks existing synchronous credential-reader contracts, adds an unrelated security-sensitive prompt change, and weakens plugin contract tests.

## What This Changes

The PR converts Claude/Codex CLI keychain reads in `src/agents/cli-credentials.ts` from `execSync` to promisified `exec`, while also adding a last-message `extraSystemPrompt` anchor and loosening two plugin registry contract tests.

## Best Possible Solution

Keep the fix focused on the gateway credential hot path: remove the unrelated prompt and contract-test changes, then either preserve the exported synchronous credential API or migrate every sync consumer with matching tests.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection confirms synchronous macOS `security` keychain reads on current main, and the linked issue/PR body supply liveness and health output, but I did not run the macOS Telegram workload independently.

## Solution Assessment

No. Async keychain access may be part of the right fix, but this PR changes exported credential readers without migrating synchronous consumers and bundles unrelated prompt/security and test-contract changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve sync contract for Claude cached credential reads:** `src/agents/cli-credentials.ts:455-461`
  - body: Changing `readClaudeCliCredentialsCached` to return a `Promise` breaks synchronous callers such as external CLI auth sync and doctor output; they will inspect a promise instead of a credential. Preserve a sync-compatible API or migrate all consumers and tests together.
  - confidence: 0.94
- **[P1] Preserve sync contract for Codex cached credential reads:** `src/agents/cli-credentials.ts:660-666`
  - body: Changing `readCodexCliCredentialsCached` to return a `Promise` makes current auth mode, auth label, and auth-epoch paths treat a pending promise as credentials, misclassifying Codex auth and hashing the wrong identity.
  - confidence: 0.94
- **[P1] Keep user text out of extraSystemPrompt:** `src/agents/pi-embedded-runner/run.ts:357-374`
  - body: This copies `params.prompt` into `extraSystemPrompt`, which is appended to the system prompt. Marking the block as untrusted does not preserve role isolation, so channel/user text can gain stronger placement than normal user-message context.
  - confidence: 0.9
- **[P2] Preserve exact plugin contract coverage:** `src/plugins/contracts/registry.contract.test.ts:21-23`
  - body: Relaxing the registry comparison to `arrayContaining` allows stale registry IDs to remain even when no bundled manifest backs them, masking the drift this contract test is meant to catch.
  - confidence: 0.86
