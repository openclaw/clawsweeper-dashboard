## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Route public keychain reads through async calls:** `src/agents/cli-credentials.ts:669-674`
  - body: The current gateway callers still use `readCodexCliCredentials*`, but this now calls `readCodexKeychainCredentialsSync`, whose shim invokes `execSync`. On macOS this leaves the event-loop block from the linked bug in the hot path, so the observable fix is not applied to current callers.
  - confidence: 0.94
- **[P1] Avoid require inside the ESM credential module:** `src/agents/cli-credentials.ts:293`
  - body: This package runs as ESM, so `require` is undefined at runtime. The catch then returns null, causing macOS Keychain credentials to be ignored whenever the sync shim is used.
  - confidence: 0.9
- **[P1] Keep user prompts out of extraSystemPrompt:** `src/agents/pi-embedded-runner/run.ts:365-368`
  - body: This copies raw `params.prompt` into `extraSystemPrompt`, which is later appended to the model's system prompt. Marking the text as untrusted does not preserve user-role isolation.
  - confidence: 0.92
- **[P2] Do not vary extraSystemPrompt per user turn:** `src/agents/pi-embedded-runner/run.ts:371-373`
  - body: Appending the current prompt to `extraSystemPrompt` changes the session binding hash every turn, so normal chat flows can invalidate with `system-prompt` churn instead of reusing the session.
  - confidence: 0.88
- **[P2] Restore exact registry contract assertion:** `src/plugins/contracts/registry.contract.test.ts:21-23`
  - body: The changed assertion allows missing runtime registrations to pass as long as the manifest list still contains the IDs, weakening the contract test that should catch registry drift.
  - confidence: 0.86
