## What This Changes

The issue asks OpenClaw to suppress the misleading initial restart warning when `lsof` is not installed while preserving gateway startup and fallback behavior.

## Best Possible Solution

Refresh, prove, and land the focused linked PR or a narrow replacement that classifies permanent unavailable-lsof spawn errors in the initial scan while preserving warnings for unexpected failures.

## Reproduction Assessment

Yes, at source level: current main can fall back to bare `lsof`, Node reports ENOENT for a missing command, and the initial scan warns for any spawn error. I did not run a full minimal-container gateway startup in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing permanent unavailable-lsof classification for the initial scan, while keeping warnings for unexpected spawn errors and nonzero statuses.
