## Summary

Keep open: current main still has the reported warning path for an initial lsof spawn error, and linked PR #76364 is the active unmerged fix candidate.

## What This Changes

Request suppressing the gateway restart stale-PID initial lsof warning when lsof is not installed while preserving fallback behavior.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats initial-scan ENOENT/EACCES/EPERM as expected lsof-unavailable states while keeping warnings for real lsof failures.

## Reproduction Assessment

Yes. Source inspection shows that a non-Windows initial stale-PID scan with spawnSync returning an ENOENT-style error reaches the warning path, and the current test suite explicitly expects that warning.

## Solution Assessment

Yes. Mirroring the existing pollPortOnce unavailable-lsof classification in the initial scan is the narrowest maintainable fix; the linked PR is the right implementation candidate to review before closing this issue.
