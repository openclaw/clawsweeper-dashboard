## Summary

Keep open: current main still warns on the initial Unix stale-PID lsof spawn error, and the linked open PR is the active narrow fix candidate for this issue.

## What This Changes

The issue asks gateway restart stale-PID cleanup to skip or downgrade the initial lsof scan warning when lsof is unavailable on minimal Linux installs.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats ENOENT/EACCES/EPERM from the initial lsof scan as expected lsof-unavailable states while preserving warnings for real lsof failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: when no executable lsof candidate exists, resolveLsofCommandSync returns `lsof`, spawnSync can return ENOENT, and findGatewayPidsOnPortSync logs the reported warning.

## Solution Assessment

Yes. Mirroring the existing pollPortOnce unavailable-lsof classification in the initial scan is the narrowest maintainable fix, and the linked PR is the right implementation candidate to review before closing this issue.
