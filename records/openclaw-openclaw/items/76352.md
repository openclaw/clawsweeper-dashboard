## Summary

Keep open: current main still warns for the initial Unix stale-PID lsof spawn error path, and open PR #76364 is the active linked fix candidate.

## What This Changes

The issue requests that gateway restart cleanup silently skip the initial stale-PID `lsof` scan when `lsof` is absent instead of logging a warning.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats ENOENT/EACCES/EPERM from the initial lsof scan as unavailable-lsof while preserving warnings for unexpected failures.

## Reproduction Assessment

Yes. Source inspection shows a non-Windows initial scan with a spawn error reaches the warning path, and the current test suite explicitly expects that warning.

## Solution Assessment

Yes. Mirroring the existing `pollPortOnce` unavailable-lsof classification in the initial scan is the narrowest maintainable fix, and linked PR #76364 is the right implementation candidate to review before closure.
