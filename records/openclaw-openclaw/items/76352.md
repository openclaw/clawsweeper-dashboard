## Summary

Keep open: current main still emits the reported warning when the initial Unix stale-PID lsof scan gets an ENOENT-style spawn error, and open PR #76364 already tracks the narrow fix with closing syntax.

## What This Changes

Silence the initial gateway restart stale-PID lsof warning when lsof is missing or unavailable while preserving warnings for real lsof failures.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats ENOENT/EACCES/EPERM from the initial lsof scan as expected unavailable-lsof states while preserving warnings for unexpected lsof failures.

## Reproduction Assessment

Yes, source-reproducible. A non-Windows findGatewayPidsOnPortSync call where spawnSync returns an ENOENT-style error reaches the warning path, and the current test suite explicitly expects that warning.

## Solution Assessment

Yes. Mirroring the existing pollPortOnce unavailable-lsof classification in the initial scan is the narrowest maintainable fix, and PR #76364 is the active implementation candidate to review before closing this issue.
