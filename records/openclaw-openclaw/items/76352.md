## Summary

Keep open: current main still logs the reported warning when the initial Unix stale-PID lsof spawn returns ENOENT, and linked PR #76364 is open but not merged.

## What This Changes

The issue requests that gateway restart startup skip or downgrade the initial stale-PID lsof warning when lsof is unavailable on minimal systems.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats ENOENT/EACCES/EPERM from the initial lsof scan as expected lsof-unavailable states while preserving warnings for real lsof failures.

## Reproduction Assessment

Yes. Source inspection of current main shows resolveLsofCommandSync can return bare `lsof`, then findGatewayPidsOnPortSync logs a warning for the resulting spawn error; the current test suite also expects that warning.

## Solution Assessment

Yes. Mirroring the existing pollPortOnce unavailable-lsof classification in the initial scan is the narrowest maintainable fix, and linked PR #76364 is the active implementation candidate.
