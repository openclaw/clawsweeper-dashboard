## What This Changes

Request silencing the initial gateway stale-PID `lsof` warning when `lsof` is missing while preserving fallback startup behavior.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats unavailable `lsof` spawn errors as expected while preserving warnings for real lsof failures.

## Reproduction Assessment

Yes. Source inspection shows a missing `lsof` binary can reach the spawn-error warning path, and the current test suite explicitly expects that warning.

## Solution Assessment

Yes. Mirroring the existing `pollPortOnce` unavailable-lsof classification in the initial scan is the narrowest maintainable fix; the linked PR is the right candidate to review before closing this issue.
