## Summary

Keep open: current main still emits the reported warning when the initial Unix stale-PID lsof spawn fails, and open PR #76364 already tracks the narrow repair with closing syntax.

## What This Changes

Suppress the misleading startup warning when the initial stale-PID lsof scan hits an unavailable lsof binary while preserving gateway startup fallback behavior.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats ENOENT/EACCES/EPERM from the initial lsof scan as expected unavailable-lsof states while preserving warnings for real lsof failures.

## Reproduction Assessment

Yes. Source inspection shows resolveLsofCommandSync can return bare `lsof` when common paths are missing, and a spawnSync ENOENT error then reaches the reported warning path in findGatewayPidsOnPortSync.

## Solution Assessment

Yes. Mirroring pollPortOnce's existing unavailable-lsof classification in the initial stale-PID scan is the narrowest maintainable fix; the linked PR #76364 is the active implementation candidate.
