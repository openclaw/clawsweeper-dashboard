## Summary

Keep open: current main still warns when the initial Unix stale-PID scan hits a missing `lsof`, and open PR #76364 is already the linked implementation path for this issue.

## What This Changes

Suppress the gateway restart/startup stale-PID scan warning when `lsof` is not installed, while preserving warnings for real `lsof` failures.

## Best Possible Solution

Review and land #76364, or apply the equivalent narrow main-branch change, then let GitHub close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows the non-Windows `findGatewayPidsOnPortSync` path logs the reported warning when `spawnSync` returns an ENOENT-style error for `lsof`.

## Solution Assessment

Yes. Treating ENOENT/EACCES/EPERM as expected lsof-unavailable states in the initial scan mirrors the existing polling helper and is the narrowest maintainable fix.
