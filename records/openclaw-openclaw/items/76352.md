## Summary

Keep open: current main still emits the reported warning when the initial Unix stale-PID lsof spawn returns ENOENT, and an open linked fix PR already tracks the narrow repair.

## What This Changes

Request to suppress the noisy unavailable-lsof warning during the gateway stale-PID startup scan on systems without lsof installed.

## Best Possible Solution

Review and land PR #76364, or an equivalent narrow patch that treats ENOENT/EACCES/EPERM from the initial lsof scan as expected unavailable-lsof states while preserving warnings for real lsof failures.

## Reproduction Assessment

Yes. From source inspection, a non-Windows findGatewayPidsOnPortSync call with spawnSync returning an ENOENT error reaches the warning path, and the current test suite explicitly expects that warning.

## Solution Assessment

Yes. Mirroring the existing pollPortOnce unavailable-lsof classification in the initial scan is the narrowest maintainable fix; the linked PR #76364 is the right implementation candidate to review before closing this issue.
