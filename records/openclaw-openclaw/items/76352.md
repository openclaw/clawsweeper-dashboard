## Summary

Keep open: current main still emits the reported initial stale-PID lsof warning for an unavailable lsof binary, and open PR #76364 is already the active narrow fix candidate.

## What This Changes

The issue requests that gateway restart startup skip or downgrade the initial stale-PID lsof warning when lsof is not installed.

## Best Possible Solution

Review and land PR #76364, or apply an equivalent narrow patch that treats initial-scan ENOENT/EACCES/EPERM as expected unavailable-lsof states while preserving warnings for real lsof failures.

## Reproduction Assessment

Yes. Source inspection shows a non-Windows findGatewayPidsOnPortSync call with a spawnSync ENOENT error reaches the warning path, and the current unit test explicitly expects that warning.

## Solution Assessment

Yes. Mirroring pollPortOnce's unavailable-lsof classification in the initial scan is the narrowest maintainable repair, and PR #76364 is already aimed at exactly that boundary.
