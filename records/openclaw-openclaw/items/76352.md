## Solution Assessment

Yes, the narrow fix shape is to classify missing or inaccessible `lsof` as an expected unavailable state in the initial scan while preserving warnings for unexpected failures. The existing linked PR targets that boundary, but this issue should remain open until that implementation lands or is replaced.
