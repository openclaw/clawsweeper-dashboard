## Summary

Keep open: current main still emits the reported initial stale-PID lsof warning on spawn-level ENOENT, and an open linked fix PR already tracks the narrow repair.

## What This Changes

The issue requests suppressing the gateway restart stale-PID lsof warning when lsof is unavailable on minimal Unix systems.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76364, or an equivalent narrow patch that treats ENOENT/EACCES/EPERM from the initial lsof scan as expected unavailable-lsof states while preserving warnings for real lsof failures.

## Reproduction Assessment

Yes. From source inspection, a non-Windows findGatewayPidsOnPortSync call whose lsof spawn returns ENOENT reaches the warning path, and the current test suite explicitly expects that warning.

## Solution Assessment

Yes. Mirroring the existing pollPortOnce unavailable-lsof classification in the initial scan is the narrowest maintainable fix; the linked PR is the active implementation candidate to review before closing this issue.
