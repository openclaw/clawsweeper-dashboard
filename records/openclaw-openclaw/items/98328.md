- **[P1] Persist sender metadata before rendering it:** `src/agents/cli-runner/session-history.ts:165-169`
  - body: This fallback only helps messages that already carry sender fields. Normal persisted user turns built through `buildPersistedUserTurnMessage` and `UserTurnInput` do not include `senderLabel`, `senderName`, `senderUsername`, or `senderId`, while `loadCliSessionReseedMessages` reloads the stored message as-is. Real group-chat CLI reseed history can therefore still render every user as `User:` after this PR.
  - confidence: 0.9
- **[P2] Constrain sender labels to one prompt line:** `src/agents/cli-runner/session-history.ts:151-153`
  - body: `coerceSenderLabel` only checks for a non-empty trimmed string before placing the value before `:`, so a channel/display label containing a newline or speaker-like prefix can inject extra model-visible transcript lines. Normalize or escape sender labels to a single safe prompt label before using them as history prefixes.
  - confidence: 0.86
