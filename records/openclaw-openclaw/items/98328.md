## Summary

Keep open: the branch targets the right bug, but it is not merge-ready because the normal CLI group-chat user-turn persistence path does not write the sender fields this renderer now reads, and the new label path trusts unvalidated `unknown` values in model-visible history.

## What This Changes

The PR updates CLI session-history rendering to label user turns from sender metadata fallback fields and adds focused tests for that fallback chain.

## Best Possible Solution

Persist trusted, sanitized sender identity with user-turn transcript messages, then have the renderer use only non-empty sanitized strings with a regression test that loads a real transcript shape through `loadCliSessionReseedMessages`.
