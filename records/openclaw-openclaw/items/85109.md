## Summary

Keep open: current main and the latest release still mirror Codex app-server `webSearch` items from the top-level `query` string and persist only a generic completed-status result, so the reported empty-query transcript behavior remains source-reproducible.

## What This Changes

The issue asks OpenClaw to record truthful Codex-native `web_search` transcript metadata instead of `{"query":""}` plus a status-only result.

## Best Possible Solution

Normalize Codex app-server `webSearch` query/action metadata into truthful transcript entries, use an explicit unavailable-query sentinel instead of an empty string, and add focused regression coverage without changing native search execution.

## Reproduction Assessment

Yes. By source inspection, a Codex app-server `webSearch` item with `query: ""` flows into transcript args as `{query:""}`, and the mirrored result falls back to `{"status":"completed"}`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/codex/src/app-server/event-projector.ts`, deriving args/result text from available `webSearch.action` metadata and avoiding invented empty-query content.
