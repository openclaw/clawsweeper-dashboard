## Summary

Current `main` still has the reported projection shape: Codex `webSearch` items are mirrored into transcript/tool state from `item.query` without rejecting an empty string, and the projected result remains a generic completed status. The issue should stay open as a narrow Codex app-server transcript observability bug.

## What This Changes

The issue requests that Codex-native `web_search` transcript/tool logs avoid empty-query mirroring and preserve available search metadata instead of a status-only stub.

## Best Possible Solution

Normalize Codex app-server `webSearch` projection in `extensions/codex/src/app-server/event-projector.ts` so empty or unavailable queries are truthful, available action metadata is retained, and focused regression coverage proves the mirrored transcript/tool result.

## Reproduction Assessment

Yes, by source inspection: a current `webSearch` item with `query: ""` will be mirrored as `{ query: "" }`, and all `webSearch` results are serialized as `{ status: "completed" }`. I did not run a live Codex session in this read-only review.

## Solution Assessment

Yes, the narrow fix belongs in the Codex app-server event projector, where native tool calls are converted into transcript/tool entries. It should not change native web search execution or provider routing.
