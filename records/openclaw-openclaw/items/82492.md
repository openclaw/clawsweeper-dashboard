## Summary

Keep open: this PR targets a real WhatsApp profile credential-isolation bug, but the latest diff still has the two blocking issues already called out in the prior ClawSweeper review: it turns a public string auth-dir export into a string-like object and includes an unrelated cron validation regression.

## What This Changes

This PR defers WhatsApp default web auth-dir resolution, adds profile auth-dir regression coverage, exposes shared test-env helpers to plugin tests, and loosens one persisted cron payload validation.

## Best Possible Solution

Land a WhatsApp-owned fix that resolves default auth dirs lazily through a primitive string/resolver boundary, drops unrelated cron changes, keeps focused regression tests, and adds redacted real profile-gateway or CLI proof.

## Reproduction Assessment

Yes. Current main is source-reproducible because WA_WEB_AUTH_DIR is computed at module evaluation while resolveOAuthDir is profile-env aware when called after OPENCLAW_STATE_DIR is active; the linked issue also gives concrete profile-gateway commands and logs.

## Solution Assessment

No. Deferring the directory is the right direction, but a string-like object is not the narrowest maintainable fix for a public string contract, and the unrelated cron validation change should be removed or split.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep WA_WEB_AUTH_DIR a primitive string:** `extensions/whatsapp/src/auth-store.ts:62`
  - body: WA_WEB_AUTH_DIR is re-exported through the WhatsApp runtime APIs and typed as a string by the generic web-channel loader. This change makes it a class instance cast to string, but Node path/fs APIs reject string-like objects before coercion, so consumers can throw TypeError when passing this export as a path. Keep this surface primitive, or make the runtime loader call a resolver that returns a primitive string.
  - confidence: 0.93
- **[P2] Keep blank agent-turn payloads invalid:** `src/cron/persisted-shape.ts:67`
  - body: This unrelated cron change makes whitespace-only agentTurn.message values pass persisted-shape validation. The store uses this guard before queueing isolated runs, and the timer then passes job.payload.message to runIsolatedAgentJob, so malformed persisted jobs can run with an empty prompt instead of being skipped for repair. Leave the trim check intact or split a targeted cron change with its own tests and rationale.
  - confidence: 0.82
