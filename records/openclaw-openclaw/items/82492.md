## Summary

Keep open. The linked WhatsApp profile-isolation bug is valid, but this PR head still replaces a public string auth-dir export with a string-like object, weakens unrelated cron payload validation, and lacks after-fix real profile-gateway or CLI behavior proof.

## What This Changes

The PR defers WhatsApp auth-dir resolution for active profiles, adds auth-dir regression tests, exposes plugin test-env helpers, changes channel-add catalog lookup, and loosens persisted cron agent-turn validation.

## Best Possible Solution

Land a focused WhatsApp-owned fix that keeps exported auth-dir values primitive, resolves paths through a runtime resolver or explicit profile env/state, drops unrelated cron changes, and includes focused tests plus redacted real profile-gateway or CLI proof.

## Reproduction Assessment

Yes. Current main is source-reproducible from the eager WA_WEB_AUTH_DIR export plus the env-aware resolveOAuthDir contract, and the linked issue provides concrete profile-gateway commands and logs.

## Solution Assessment

No. Lazy resolution is the right direction, but a string-like object is not a maintainable public string contract, and the unrelated cron validation change should be removed or split.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep WA_WEB_AUTH_DIR a primitive string:** `extensions/whatsapp/src/auth-store.ts:62`
  - body: WA_WEB_AUTH_DIR is exported through the WhatsApp runtime APIs and typed as a string by the generic web-channel loader. This change makes it a class instance cast to string, but Node path/fs APIs reject string-like objects before coercion, so consumers can throw TypeError when passing this export as a path.
  - confidence: 0.93
- **[P2] Keep blank agent-turn payloads invalid:** `src/cron/persisted-shape.ts:67`
  - body: This unrelated cron change makes whitespace-only agentTurn.message values pass persisted-shape validation. The store uses this guard before queueing isolated runs, and the timer passes job.payload.message to runIsolatedAgentJob, so malformed persisted jobs can run with an empty prompt instead of being skipped for repair.
  - confidence: 0.82
