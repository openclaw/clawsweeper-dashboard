## Summary

Keep open: the linked WhatsApp profile-isolation bug is real and the direction is useful, but this PR still breaks the exported auth-dir string contract and only supplies test/lint proof instead of real profile or gateway proof. The branch also carries unrelated cron/pairing churn that should not ride along with a credential-path fix.

## What This Changes

The PR lazily resolves WhatsApp auth directories from the active process environment, adds a WhatsApp profile-dir regression test, and includes unrelated cron/pairing test/runtime edits.

## Best Possible Solution

Land a WhatsApp-owned fix that defers default auth-dir resolution while preserving a primitive string/runtime-loader contract, removes unrelated cron/pairing edits, and includes redacted real profile-gateway or CLI proof for the auth-dir path.

## Reproduction Assessment

Yes. Current main is source-reproducible because `WA_WEB_AUTH_DIR` is still computed at import time, and the linked issue includes concrete profile-gateway commands and logs showing credentials under the default state dir.

## Solution Assessment

No. Deferring the directory is the right direction, but replacing a primitive exported path with a string-like object is not the safest maintainable solution; keep the public/runtime value primitive or move laziness behind a resolver boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep WA_WEB_AUTH_DIR as a primitive string:** `extensions/whatsapp/src/auth-store.ts:62`
  - body: `WA_WEB_AUTH_DIR` is re-exported through the WhatsApp runtime APIs and typed as a string by the generic web-channel loader. This change makes it a class instance cast to `string`, but Node path/fs APIs reject string-like objects before coercion, so callers can fail with `TypeError` when passing this export to `path.join` or related helpers. Keep this surface primitive, or make the runtime loader call a resolver that returns a primitive string.
  - confidence: 0.92
- **[P2] Keep blank cron payloads invalid:** `src/cron/persisted-shape.ts:55-61`
  - body: This unrelated change lets persisted jobs with whitespace-only `systemEvent.text` or `agentTurn.message` pass shape validation. Main jobs then load only to be skipped later, and isolated agent-turn jobs can reach `runIsolatedAgentJob` with an empty prompt, which weakens the persisted cron contract outside the WhatsApp fix. Leave this validation intact or split a targeted cron change with its own tests and rationale.
  - confidence: 0.78
