## Summary

Keep open. Current main still adds Claude's stdio permission prompt hook for live sessions but the JSONL handler returns on every non-`result` record, so a `control_request` cannot be answered with a `control_response`; the related broader allowlist issue does not supersede this narrower live-session stall.

## What This Changes

The issue requests a Claude CLI live-session bridge that handles `control_request` / `can_use_tool` messages and replies with policy-derived `control_response` records instead of timing out.

## Best Possible Solution

Add a tested Claude live-session control-request bridge that replies to native tool permission prompts using OpenClaw's effective exec policy, with fail-closed messaging for unsupported or non-MCP-native paths.

## Reproduction Assessment

Yes. Source inspection on current main shows the live session enables Claude stdio permission prompts but never handles or answers non-`result` control records, matching the reported no-output timeout path.

## Solution Assessment

Yes, with a security review of the exact response schema. The narrow maintainable fix belongs in `src/agents/cli-runner/claude-live-session.ts`, reusing the already-established OpenClaw exec policy mapping instead of adding new config.
