## Summary

Keep open: current main and the latest shipped tag still lack a Claude live-session `control_request` response path, and this issue is already paired with open fixing PRs that should be reviewed or landed before closure.

## What This Changes

The issue reports that Claude CLI live sessions ignore Claude stream-json `control_request` permission prompts and stall until the no-output watchdog timeout instead of writing a matching `control_response`.

## Best Possible Solution

Land one canonical, policy-correct Claude live-session handler that responds to `can_use_tool` frames, denies native tools by default with MCP guidance, and preserves the documented YOLO/bypass and explicit override semantics.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main starts Claude in stdio permission-prompt mode, then returns for every non-`result` JSONL record, so a `control_request` cannot receive a `control_response`.

## Solution Assessment

Yes. The narrow maintainable solution belongs in `src/agents/cli-runner/claude-live-session.ts`, but this issue should stay open until an existing linked fix PR, preferably the policy-aware path in https://github.com/openclaw/openclaw/pull/81971 after review fixes, is merged.
