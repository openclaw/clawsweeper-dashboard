## Summary

Keep open: current main and v2026.5.18 still enable Claude stdio permission prompts but drop non-result stream-json records, and an open linked PR is already the correct fix path before this issue closes.

## What This Changes

The issue asks OpenClaw to answer Claude CLI live-session `control_request` / `can_use_tool` frames with exec-policy-aware `control_response` replies.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81971; the core fix should answer `can_use_tool`, deny native non-Bash tools with MCP guidance, and align Bash only with the documented permissive exec policy.

Yes.
