## Summary

Keep open: current main and the latest shipped tag still force Claude live sessions into stdio permission-prompt mode while dropping non-result JSONL records, and the linked fix PR is open with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to answer Claude CLI live-session `control_request` / `can_use_tool` frames with `control_response` messages so native tool permission prompts do not stall until timeout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80973, or replace it with an equivalent maintainer-approved patch that writes protocol-correct `control_response` frames and settles the native-tool policy boundary.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main starts Claude live sessions with stdio permission prompts, then returns on every non-`result` JSONL record, so `control_request` frames have no response path.

## Solution Assessment

Mostly yes. Handling the frame in `src/agents/cli-runner/claude-live-session.ts` with focused spawn/stdin tests is the narrow maintainable fix, but maintainers should confirm whether native Bash is always denied or allowed only under fully permissive exec policy.
