## What This Changes

The issue asks OpenClaw to stop Feishu `message.edit` from sending invalid interactive-card JSON when a plain-text edit is accompanied by an empty presentation or card payload.

## Best Possible Solution

Keep this open for a narrow Feishu plugin fix that treats empty rich edit payloads as absent and preserves a valid text-edit path, with live/API proof for edits of existing interactive cards.

## Reproduction Assessment

Unclear as an exact current-main repro: the reported raw params use `presentation`, and current Feishu edit ignores `presentation`, but the edit helper still has no empty-card guard and no test covers text plus empty rich payloads.

## Solution Assessment
