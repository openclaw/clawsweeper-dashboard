## Summary

Keep this PR open: the Browserbase Sessions API direction matches the upstream contract, but the patch currently leaves the new profile shape rejected by runtime config validation and leaves most browser action routes using an empty CDP URL.

## What This Changes

The PR adds a Browserbase browser profile driver, per-attach Browserbase Sessions API connectUrl resolution, related docs/tests, and higher remote-CDP timeout defaults.

## Best Possible Solution

Align the public type, runtime schema, help/labels/docs, and central browser route CDP resolution so every Browserbase Playwright action gets a fresh connectUrl, then add focused regression coverage plus redacted real Browserbase proof.

## Reproduction Assessment

Yes for the review blockers by source inspection: the runtime zod schema still rejects the new profile shape, and the shared route helper still passes the unresolved empty cdpUrl into most Playwright routes. I did not run the code because this review is read-only.

## Solution Assessment

No: per-attach connectUrl resolution is the right dependency-backed direction, but the current implementation is not central enough and misses the runtime config schema. The safer implementation is to resolve the Browserbase CDP URL at the common route/context boundary or another single Playwright dispatch seam, and update the canonical config contract in the same patch.
