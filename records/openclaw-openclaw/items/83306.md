## Summary

Keep open: this PR is useful and targets a real Browserbase session-state gap, but it is not merge-ready because the documented profile is still rejected by runtime schema, several shared browser routes still receive an empty CDP URL, signed connectUrl fragments can leak through errors, and inspectable real Browserbase proof is missing.

## What This Changes

The PR adds a Browserbase browser profile driver that resolves a fresh Browserbase CDP connectUrl per attach, updates profile types/capabilities/docs/tests, and raises remote CDP timeout defaults.

## Best Possible Solution

Land a Browserbase driver only after schema/docs/tests, centralized CDP URL resolution, redacted error handling, credential-surface approval, and redacted live Browserbase proof are all aligned; otherwise keep the broader work tracked in https://github.com/openclaw/openclaw/issues/43455.

## Reproduction Assessment

Yes for the PR review blockers by source inspection: the PR head still rejects the documented profile in runtime schema and still passes an empty cdpUrl through shared browser routes. No live Browserbase flow was reproduced because the PR has no inspectable real-environment artifact.

## Solution Assessment

No, not as implemented: per-attach Browserbase connectUrl resolution is a plausible direction, but this patch misses the canonical runtime schema and shared route boundary and leaks signed URL fragments. The safer fix is centralized resolution plus schema/docs/test/redaction alignment and redacted live proof.
