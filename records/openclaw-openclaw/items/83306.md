## Summary

Keep open: the direction is useful, but the PR is not mergeable because the documented profile is still rejected by the runtime schema, many browser routes would still attach with the empty Browserbase placeholder URL, a signed connectUrl can leak through an error path, timeout docs drift from the changed defaults, and the claimed live Browserbase smoke test is not inspectable.

## What This Changes

The PR adds a Browserbase browser profile driver that resolves a fresh Browserbase connectUrl per attach, updates profile types/capabilities/docs/tests, and raises remote CDP timeout defaults.

## Best Possible Solution

Centralize Browserbase CDP URL resolution at the shared route/profile boundary, align runtime schema/help/docs with the new config contract, redact signed URLs in errors, and include redacted real Browserbase proof.

## Reproduction Assessment

Yes for the review blockers by source inspection: the runtime schema still rejects the documented Browserbase profile, and shared browser routes still pass the empty profile cdpUrl. I did not run code or Browserbase live because this review is read-only.

## Solution Assessment

No: per-attach Browserbase connectUrl resolution is the right direction, but this implementation is not central enough and misses the canonical runtime config contract. The safer path is a shared CDP-resolution boundary plus schema/help/docs/tests and redacted proof.
