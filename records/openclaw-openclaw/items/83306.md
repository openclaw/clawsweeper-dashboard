## Summary

Keep open: the PR contains meaningful Browserbase session-driver work that current main and v2026.6.10 do not provide, but it is not merge-ready because the lazy `cdpUrl` resolution is only wired into some attach paths, signed Browserbase URL material can appear in a malformed-response error, contributor-owned OpenClaw end-to-end proof is missing, and the new credential/config surface needs maintainer approval.

## What This Changes

The PR adds a Browserbase browser profile driver that resolves a fresh Browserbase session connect URL per attach and updates browser config, schema/help, docs, tests, and SDK profile types.

## Best Possible Solution

Land a revised Browserbase driver only after live URL resolution is centralized or consistently applied across every browser Playwright/CDP route, signed URL diagnostics are redacted, and the new credential/config surface is maintainer-approved with real Browserbase tool proof.

## Reproduction Assessment

Yes. Source inspection at PR head shows Browserbase profiles resolve to an empty stored `cdpUrl` while shared browser route helpers still pass that unresolved value to Playwright/CDP consumers.

## Solution Assessment

No. Per-attach Browserbase URL resolution is a plausible direction, but the current implementation is too narrow because it patches only some attach call sites instead of the shared browser route boundary.
