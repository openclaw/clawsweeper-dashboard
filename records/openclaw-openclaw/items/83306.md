## Summary

Keep open: the patch is not mergeable yet because the new profile shape is still rejected by the runtime config schema, most Playwright-backed routes still receive the unresolved empty CDP URL, and the PR lacks inspectable real Browserbase proof. This confirms the earlier ClawSweeper review blockers rather than closing the PR.

## What This Changes

The PR adds a Browserbase browser profile driver with per-attach Sessions API connectUrl resolution, related browser profile types/capabilities/docs/tests, and higher remote-CDP timeout defaults.

## Best Possible Solution

Align the public type, runtime schema, help/docs, and a single shared CDP-resolution boundary so every Browserbase Playwright action gets a fresh connectUrl, then add focused regression coverage and redacted real Browserbase proof.

## Reproduction Assessment

Yes for the review blockers by source inspection: the PR head still rejects the documented profile in the runtime schema, and common Playwright route paths still receive profileCtx.profile.cdpUrl even though browserbase profiles resolve it to an empty string. I did not run code or Browserbase live because this review is read-only.

## Solution Assessment

No: per-attach Browserbase connectUrl resolution is the right direction, but this implementation is not central enough and does not update the canonical runtime config contract. The safer fix is to resolve the CDP URL at a common profile/route dispatch boundary and update schema/help/docs/tests together.
