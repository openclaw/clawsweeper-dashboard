## Summary

Keep open: current main and v2026.6.9 do not include the Browserbase driver, so the PR is still meaningful, but it is not merge-ready because shared browser routes still receive an empty CDP URL, malformed signed URLs can leak in errors, full OpenClaw behavior proof is insufficient, and the new credential/config surface needs maintainer approval.

## What This Changes

The PR adds a Browserbase browser profile driver that resolves a fresh Browserbase session connectUrl per attach, updates browser config/schema/docs/tests and Plugin SDK profile types, and raises remote CDP default timeouts.

## Best Possible Solution

Land a revised Browserbase driver after live URL resolution is centralized for all CDP/Playwright browser routes, signed URLs are redacted, the credential/config/default changes are maintainer-approved, and redacted OpenClaw browser-tool proof is attached.

## Solution Assessment

No. Per-attach Browserbase connectUrl resolution is a plausible direction, but the best fix must centralize URL resolution across all browser routes, redact signed URL material, and settle the credential/config surface before merge.
