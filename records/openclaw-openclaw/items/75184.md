## Summary

Keep this open: current main still does not make `agents.defaults.skipBootstrap` suppress normal runtime workspace bootstrap injection, and the open linked pull request is the active fix path rather than a reason to close the issue early.

## What This Changes

The issue asks OpenClaw to either make `agents.defaults.skipBootstrap: true` suppress workspace bootstrap files during normal agent prompt construction or clearly document/remove that runtime implication.

## Best Possible Solution

Resolve this through https://github.com/openclaw/openclaw/pull/75217: either land a reviewed resolver/docs/schema update that preserves hook-injected context and proves the prompt-report path, or explicitly keep creation-only semantics and update generated config help to remove the ambiguity.

## Reproduction Assessment

Yes from source inspection: current main passes runtime config into the bootstrap resolver, but the resolver loads and returns workspace bootstrap files without checking `agents.defaults.skipBootstrap`. I did not run the packaged `openclaw agent --json` repro because this review is read-only and the source path is clear.

## Solution Assessment

Unclear as a product decision, but the linked PR's shared-resolver approach is the likely right implementation boundary if maintainers want this existing flag to suppress runtime injection. The safer alternative is to keep `skipBootstrap` creation-only and document/generated-schema-help that scope while pointing runtime suppression to `contextInjection: "never"`.
