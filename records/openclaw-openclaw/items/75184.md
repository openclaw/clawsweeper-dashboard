## Summary

Keep open. Current main still treats `agents.defaults.skipBootstrap` as a bootstrap-file creation flag, not as a runtime injection skip, and this issue is explicitly paired with open closing PR #75217, so it should stay open until that PR is reviewed, merged, or rejected.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` either to suppress workspace bootstrap injection during agent runs or to be renamed/documented as a creation-only setting with schema help.

## Best Possible Solution

End with one tested public contract: either land or revise PR #75217 so `skipBootstrap` empties runtime bootstrap/context injection and updates coverage, or explicitly preserve creation-only semantics with generated schema/help that directs prompt-size users to `contextInjection: "never"` and bootstrap budgets.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection reproduces the root cause statically: existing workspace bootstrap files remain eligible for injection because the runtime resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended contract. PR #75217 is a plausible narrow implementation of the reporter's requested behavior, while schema/help clarification is the safer direction if `skipBootstrap` should remain creation-only.
