## Summary

Keep this open: current main and the latest release still load existing workspace bootstrap files without checking `agents.defaults.skipBootstrap`, and the open linked pull request is the active fix path rather than a reason to close this issue before merge.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` to either suppress runtime workspace bootstrap injection or be documented/renamed as creation-only instead of appearing as a no-op.

## Best Possible Solution

Resolve this through https://github.com/openclaw/openclaw/pull/75217 or an equivalent maintainer-approved contract update, then close the issue after the runtime/docs/schema behavior is aligned and merged.

## Reproduction Assessment

Yes, at source level: current main passes runtime config into the bootstrap resolver, but the resolver loads existing workspace bootstrap files without checking `agents.defaults.skipBootstrap`. I did not run the packaged CLI repro because this review is read-only and the code path is clear.

## Solution Assessment

Unclear pending maintainer product decision: the linked PR's shared-resolver approach is the narrow implementation boundary if `skipBootstrap` should suppress runtime injection. The safer alternative is to keep creation-only semantics and update generated schema/help/docs so users choose `contextInjection: "never"` or bootstrap limits for runtime prompt control.
