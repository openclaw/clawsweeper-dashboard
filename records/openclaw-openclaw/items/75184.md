## Summary

Keep open. Current main still treats `agents.defaults.skipBootstrap` as a workspace-file creation switch, not a runtime prompt-injection switch, and an open closing PR already targets the reported behavior.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` to either suppress runtime workspace bootstrap-file injection or be clearly documented/renamed as creation-only.

## Best Possible Solution

The config surface should end with one tested contract: either `skipBootstrap` empties runtime bootstrap/context-file injection and the system prompt report reflects that, or the field stays creation-only with schema/help text that directs prompt-size control to `contextInjection: "never"` and bootstrap budgets.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection provides a high-confidence static reproduction because the runtime bootstrap resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a narrow implementation if the flag should mean runtime skip; a schema/help clarification is safer if `skipBootstrap` must remain creation-only.
