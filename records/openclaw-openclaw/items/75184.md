## Summary

Keep open. Current main still treats `agents.defaults.skipBootstrap` as workspace-file creation wiring, not as a runtime injection switch, and open PR #75217 already targets this issue with closing syntax.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` to either skip runtime workspace bootstrap injection or be clearly documented/renamed as creation-only.

## Best Possible Solution

End with one tested public contract: either wire `skipBootstrap` into runtime bootstrap/context-file injection, or keep it creation-only and make schema/help/docs direct prompt-size control to `contextInjection: "never"` and bootstrap budgets.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because existing workspace bootstrap files remain eligible for runtime injection when `skipBootstrap` is true.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a narrow fix if `skipBootstrap` should mean runtime skip; schema/help/docs clarification is safer if the flag must remain creation-only.
