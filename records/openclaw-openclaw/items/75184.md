## Summary

Keep open. Current main still does not make `agents.defaults.skipBootstrap` affect runtime workspace-file injection, and open PR #75217 is the explicit implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` either to skip runtime workspace bootstrap-file injection or to be clearly scoped/renamed as a creation-only setting.

## Best Possible Solution

Review and land PR #75217 if `skipBootstrap` should suppress injection; otherwise close that PR and update schema/help/docs to make the creation-only contract unambiguous.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because the runtime resolver still does not read `skipBootstrap`.

## Solution Assessment

Unclear. PR #75217 is a plausible narrow runtime fix, but preserving `skipBootstrap` as creation-only and documenting `contextInjection: "never"` would be safer if maintainers need compatibility with pre-seeded workspace files.
