## Summary

Keep this open: current main and the latest release still load workspace bootstrap files in the normal runtime resolver without honoring `agents.defaults.skipBootstrap`, and https://github.com/openclaw/openclaw/pull/75217 is the open candidate fix rather than merged implementation.

## What This Changes

The issue reports that `agents.defaults.skipBootstrap: true` does not suppress workspace bootstrap files during normal agent prompt construction.

## Best Possible Solution

Resolve this through https://github.com/openclaw/openclaw/pull/75217 or an equivalent maintainer-approved patch: either honor `skipBootstrap` in the shared resolver while preserving hook-injected context, or explicitly keep the flag creation-only and align generated schema/help, docs, and runtime guidance around `contextInjection: "never"`.

## Reproduction Assessment

Yes, at source level: current main passes config into the bootstrap resolver, but the resolver loads workspace files without checking `agents.defaults.skipBootstrap`. I did not run the packaged `openclaw agent --json` repro because this review is read-only and the source path is clear.

## Solution Assessment

Unclear until maintainers choose the config contract. The shared resolver is the right boundary if the existing flag should suppress runtime injection; otherwise the safer solution is generated schema/help plus docs that keep `skipBootstrap` creation-only and direct runtime suppression to `contextInjection: "never"`.
