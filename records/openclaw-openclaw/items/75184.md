## Summary

Keep open. Current main still uses `agents.defaults.skipBootstrap` for workspace bootstrap file creation, while runtime workspace-file injection is controlled by `contextInjection`, and open PR #75217 is already the closing implementation candidate.

## What This Changes

The issue requests that `agents.defaults.skipBootstrap` either disable runtime workspace bootstrap injection or be clearly documented/renamed as creation-only.

## Best Possible Solution

End with one tested public config contract: either `skipBootstrap` empties runtime workspace-file injection and reports no injected files, or it stays creation-only with schema/help/docs pointing prompt-size control to `contextInjection: "never"` and bootstrap budgets.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because the runtime resolver still does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow runtime fix; a schema/help clarification is safer if the field must remain creation-only.
