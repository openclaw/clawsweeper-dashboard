## Summary

Keep open. Current main still has the reported contract mismatch: `skipBootstrap` is wired to workspace bootstrap-file creation, while the runtime injection paths still resolve bootstrap context without checking that flag; the open closing PR should be reviewed or superseded before this issue is closed.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` to either stop runtime workspace bootstrap-file injection or be clearly documented/renamed as creation-only.

## Best Possible Solution

Settle one public config contract, then either land the linked runtime fix with regression coverage or keep `skipBootstrap` creation-only and make schema/help/docs point users to the supported runtime injection controls.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence static reproduction: runtime prompt construction resolves bootstrap context without reading `skipBootstrap`, while creation paths use the flag only for `ensureBootstrapFiles`.

## Solution Assessment

Unclear. PR #75217 is a plausible narrow runtime fix, but current docs define `skipBootstrap` as creation-only, so maintainers need to choose the intended compatibility contract before closing this issue.
