## Summary

Current main still does not honor `agents.defaults.skipBootstrap` in the shared runtime bootstrap resolver, and the open linked PR is the active implementation path, so this issue should stay open until that PR is landed or explicitly rejected.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` to stop runtime workspace bootstrap file injection, or for the config surface to be clarified if that is not intended.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75217 so the central resolver suppresses workspace bootstrap files when `skipBootstrap` is true while preserving hook-injected context, or explicitly retire/clarify the key if maintainers reject that behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the runtime resolver receives config but never checks `agents.defaults.skipBootstrap` before loading and returning workspace bootstrap files; the issue also provides a concrete `openclaw agent --json` `systemPromptReport` repro.

## Solution Assessment

Yes, if maintainers accept the config contract change. The central resolver is the narrowest shared fix point because embedded, command, realtime, Codex, Copilot, doctor, and report callers all consume `resolveBootstrapFilesForRun` or `resolveBootstrapContextForRun`.
