## Summary

Keep open. Current main still treats `agents.defaults.skipBootstrap` as workspace-file creation wiring, not runtime prompt-injection control, and open PR #75217 already targets this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to either honor `agents.defaults.skipBootstrap: true` for runtime workspace bootstrap injection or clarify/remove the misleading config surface.

## Best Possible Solution

End with one tested public contract: either PR #75217 lands so `skipBootstrap` empties runtime bootstrap/context injection, or the flag remains creation-only with generated schema/help text directing users to `contextInjection: "never"` for prompt-size control.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction: existing workspace bootstrap files remain eligible for runtime injection because the resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if `skipBootstrap` should mean runtime skip; schema/help/docs clarification is safer if the flag must remain creation-only.
