## Summary

Keep open. Current main and v2026.6.10 still load workspace bootstrap files without checking `agents.defaults.skipBootstrap`, and the direct fix is already represented by the open implementation PR at https://github.com/openclaw/openclaw/pull/75217.

## What This Changes

This issue asks OpenClaw to either make `agents.defaults.skipBootstrap` suppress existing workspace bootstrap file injection at runtime or clarify the config contract so users do not expect that behavior.

## Best Possible Solution

Review and either land or deliberately replace https://github.com/openclaw/openclaw/pull/75217 after aligning runtime behavior, generated schema/help metadata, docs, tests, and maintainer acceptance of the compatibility-sensitive config semantics.

## Reproduction Assessment

Yes, at source level. Current main passes runtime config into the bootstrap resolver, but the resolver loads workspace files without checking `agents.defaults.skipBootstrap`, and the prompt report is built from that resolver output.

## Solution Assessment

Unclear until maintainers make the config-contract call. The shared resolver is the right implementation boundary if `skipBootstrap` is meant to suppress runtime injection, but `contextInjection: "never"` is already the explicit runtime-only alternative.
