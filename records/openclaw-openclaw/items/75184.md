## Summary

Keep open: current main and the latest release still load workspace bootstrap files in `resolveBootstrapFilesForRun` without checking `agents.defaults.skipBootstrap`, and the direct fixing pull request remains open. Because that pull request uses closing syntax and changes compatibility-sensitive config semantics, this issue should stay open for maintainer review rather than being closed or queued for a duplicate fix.

## What This Changes

The issue asks OpenClaw to either make `agents.defaults.skipBootstrap` suppress runtime workspace bootstrap injection or make the config/schema clearly state that the flag only affects bootstrap file creation.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/75217 with an explicit maintainer decision: either land the runtime suppression semantics with schema/docs alignment and proof, or close that path and make `contextInjection: "never"` the documented runtime injection control.

## Reproduction Assessment

Yes, at source level: current main passes config into the bootstrap resolver, but the resolver loads existing workspace files without checking `agents.defaults.skipBootstrap`. I did not run the packaged CLI repro because this review is read-only.

## Solution Assessment

Unclear until maintainers decide the config contract. The shared resolver is the right implementation boundary if `skipBootstrap` should suppress runtime injection, but `contextInjection: "never"` now exists as a separate supported runtime injection control.
