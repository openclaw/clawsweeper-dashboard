## Summary

Keep open. Current main still does not make `agents.defaults.skipBootstrap` suppress runtime workspace bootstrap injection, and this issue is paired with open closing PR #75217, so the right cleanup path is to review or supersede that PR rather than close the issue early.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` either to disable workspace bootstrap file injection at runtime or to be documented and named clearly as a creation-only setting.

## Best Possible Solution

End with one tested public contract: either `skipBootstrap` empties runtime workspace bootstrap/context-file injection and `systemPromptReport` reflects that, or the field is clearly labeled as creation-only and users are directed to `contextInjection: "never"` plus bootstrap budgets for prompt-size control.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction: existing workspace bootstrap files remain eligible for runtime injection because the resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if `skipBootstrap` should mean runtime skip; a schema/help/docs clarification is safer if the flag must remain creation-only.
