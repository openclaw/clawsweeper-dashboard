## Summary

Keep open. Current main still treats `skipBootstrap` as a workspace bootstrap-file creation flag rather than a runtime injection switch, and the issue has an open closing PR that should be reviewed or superseded by a deliberate docs/schema contract decision.

## What This Changes

The issue requests that `agents.defaults.skipBootstrap` either suppress runtime workspace bootstrap file injection or be clearly documented/renamed as creation-only.

## Best Possible Solution

End with one tested public contract: either `skipBootstrap` empties runtime bootstrap/context-file injection and the system prompt report reflects that, or the field is clearly labeled as creation-only and users are directed to `contextInjection: "never"` plus bootstrap budgets for prompt-size control.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction: existing workspace bootstrap files remain eligible for runtime injection because the resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if `skipBootstrap` should mean runtime skip; a schema/help clarification is safer if the flag must remain creation-only.
