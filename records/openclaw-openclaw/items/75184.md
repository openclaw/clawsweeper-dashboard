## Summary

Keep open. Current main still leaves `skipBootstrap` as a workspace-file creation flag and does not feed it into runtime bootstrap resolution; PR #75217 is the open closing implementation candidate, so closing before merge would be premature.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` to either suppress runtime workspace bootstrap injection or be clearly documented/renamed as a different-scope setting.

## Best Possible Solution

Settle one tested config contract: either `skipBootstrap` makes runtime bootstrap/context-file injection empty and the prompt report reflects that, or the schema/docs explicitly label it as creation-only and direct prompt-size users to `contextInjection: "never"` plus bootstrap budgets.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because existing workspace bootstrap files remain eligible for runtime injection when `skipBootstrap` is true.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if the flag should mean runtime skip; a schema/help clarification is safer if the flag must remain creation-only.
