Keep open. Current main still does not make `agents.defaults.skipBootstrap` suppress runtime workspace bootstrap injection, and the issue has an open closing PR, so the safe maintainer outcome is to review that PR or choose a docs/schema clarification instead.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap: true` to either skip runtime workspace bootstrap file injection or be clearly documented/renamed as creation-only.

## Best Possible Solution

The config surface should end with one tested contract: either `skipBootstrap` empties runtime bootstrap/context-file injection and the system prompt report reflects that, or the field is clearly labeled as creation-only and users are directed to `contextInjection: "never"` plus bootstrap budgets for prompt-size control.

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction: existing workspace bootstrap files remain eligible for runtime injection because the resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if `skipBootstrap` should mean runtime skip; a schema/help clarification is safer if the flag must remain creation-only.
