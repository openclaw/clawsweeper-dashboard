## Summary

Keep this open: current main and the latest release still load normal workspace bootstrap files in the runtime resolver without honoring `agents.defaults.skipBootstrap`, and https://github.com/openclaw/openclaw/pull/75217 is still the open candidate fix rather than merged implementation.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` either to suppress runtime workspace bootstrap injection or to have its schema/docs clarified as creation-only.

## Best Possible Solution

Resolve this through https://github.com/openclaw/openclaw/pull/75217: either land a reviewed resolver/docs/schema update that preserves hook-injected context, or explicitly keep creation-only semantics and update generated config help to remove the ambiguity.

## Reproduction Assessment

Yes, at source level: current main passes config into the bootstrap resolver, but the resolver loads workspace files without checking `agents.defaults.skipBootstrap`. I did not run the packaged `openclaw agent --json` repro because this review is read-only and the source path is clear.

## Solution Assessment

Unclear until maintainers choose the config contract. The shared resolver is the right boundary if the existing flag should suppress runtime injection; otherwise the safer solution is generated schema/help plus docs that keep `skipBootstrap` creation-only and direct runtime suppression to `contextInjection: "never"`.
