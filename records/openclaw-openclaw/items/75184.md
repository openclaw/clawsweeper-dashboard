## Summary

Keep this open: current main and the latest release still do not make `agents.defaults.skipBootstrap` suppress normal runtime workspace bootstrap injection, and the linked open pull request is the active fix path rather than a reason to close the issue early.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` either to suppress normal workspace bootstrap-file injection or to be renamed, removed, or documented if that is not its supported scope.

## Best Possible Solution

Review and either land or reject the linked runtime resolver fix, with maintainer agreement on whether `skipBootstrap` should skip only workspace bootstrap files while preserving hook-injected context.

## Reproduction Assessment

Yes, by source inspection: normal CLI preparation passes config into `resolveBootstrapContextForRun`, but the resolver loads and returns workspace bootstrap files without checking `agents.defaults.skipBootstrap`. The packaged CLI repro was not run because this review is read-only and the source path is clear.

## Solution Assessment

Unclear as a product decision: honoring the existing flag in the shared resolver is the narrow implementation path if maintainers want runtime suppression, but keeping creation-only semantics and documenting `contextInjection: "never"` as the runtime control is the safer alternative.
