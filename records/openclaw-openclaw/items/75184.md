## Summary

Keep open. Current main still does not wire `agents.defaults.skipBootstrap` into the runtime bootstrap/context-file injection path, and an open closing PR already targets this issue, so the issue should remain open until that PR lands or maintainers choose a different config contract.

## What This Changes

The issue reports that `agents.defaults.skipBootstrap: true` still permits existing workspace bootstrap files to be injected into normal agent system prompts and asks for the flag to either control runtime injection or be documented/renamed as creation-only.

## Best Possible Solution

End with one tested public contract: either `skipBootstrap` empties runtime bootstrap/context-file injection and `systemPromptReport.injectedWorkspaceFiles` reflects that, or the field is clearly creation-only and users are directed to `contextInjection: "never"` plus bootstrap budgets for prompt-size control.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction: existing workspace bootstrap files remain eligible for runtime injection because the resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if `skipBootstrap` should mean runtime skip; schema/help/docs clarification is safer if the flag must remain creation-only.
