## Summary

Keep open. Current main still treats `skipBootstrap` as workspace creation wiring, while runtime prompt injection reads existing workspace bootstrap files; open PR #75217 already targets the issue, so cleanup should wait for that PR to merge or for maintainers to choose a docs/schema-only contract.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` to either stop runtime workspace bootstrap file injection or be documented/renamed as a creation-only option.

## Best Possible Solution

End with one tested public contract: either `skipBootstrap` empties runtime bootstrap/context-file injection and `systemPromptReport.injectedWorkspaceFiles` reflects that, or the field is clearly creation-only and users are directed to `contextInjection: "never"` plus bootstrap budgets for prompt-size control.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because the runtime resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if `skipBootstrap` should mean runtime skip; schema/help/docs clarification is safer if the flag must remain creation-only.
