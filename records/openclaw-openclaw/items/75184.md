## Summary

Keep open. Current main still treats `agents.defaults.skipBootstrap` as a bootstrap-file creation flag, while runtime workspace-file injection is controlled by `contextInjection`, and open PR #75217 is already the closing implementation candidate.

## What This Changes

The issue requests making `agents.defaults.skipBootstrap: true` disable runtime workspace bootstrap-file injection, or clarifying/removing the misleading config surface.

## Best Possible Solution

Review and either land PR #75217 with focused tests/docs/schema/changelog, or explicitly keep `skipBootstrap` creation-only and ship schema/help text that points prompt-size control to `contextInjection: "never"` and bootstrap budgets.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because the runtime resolver still does not read `skipBootstrap`.

## Solution Assessment

Unclear. PR #75217 is a plausible narrow runtime fix, but schema/help clarification is safer if maintainers want to preserve `skipBootstrap` as creation-only.
