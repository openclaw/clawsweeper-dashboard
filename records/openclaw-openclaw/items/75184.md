## Summary

Keep open. Current main still does not read `agents.defaults.skipBootstrap` on the runtime workspace-bootstrap injection path, and the linked closing PR #75217 remains open, so this issue should stay open until that PR is reviewed, landed, or explicitly superseded.

## What This Changes

Request that `agents.defaults.skipBootstrap: true` either suppress runtime workspace-bootstrap prompt injection or be clearly documented/renamed as a creation-only setting.

## Best Possible Solution

Review PR #75217 against the existing config contract; either land a runtime skip with regression coverage and schema/help updates, or keep `skipBootstrap` creation-only and clarify the schema/docs around `contextInjection: "never"`.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence static reproduction: existing workspace files still flow through `resolveBootstrapFilesForRun` without reading `skipBootstrap`, while `skipBootstrap` is only used when deciding whether to create missing files.

## Solution Assessment

Unclear. PR #75217 is a plausible narrow fix, but the best solution depends on whether maintainers want `skipBootstrap` to change semantics or remain creation-only with clearer schema/help text.
