## Summary

Keep open. Current main still treats `skipBootstrap` as workspace-file creation wiring, not a runtime injection switch, and the issue is already paired with open closing PR #75217, so it should remain open until that PR lands or maintainers choose the creation-only/docs contract.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` to either disable runtime workspace bootstrap injection for existing files or be renamed/documented as creation-only.

## Best Possible Solution

End with one tested public contract: either `skipBootstrap` empties runtime bootstrap/context-file injection and `systemPromptReport.injectedWorkspaceFiles` reflects that, or the field is clearly creation-only and users are directed to `contextInjection: "never"` plus bootstrap budgets for prompt-size control.

## Reproduction Assessment

Yes. The issue provides deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because existing workspace bootstrap files remain eligible for runtime injection while the resolver does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended compatibility contract. PR #75217 is a plausible narrow fix if `skipBootstrap` should mean runtime skip; schema/help/docs clarification is safer if the flag must remain creation-only.
