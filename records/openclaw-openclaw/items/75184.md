## Summary

Keep open because current main still does not apply `agents.defaults.skipBootstrap` to runtime bootstrap injection, and open PR #75217 already references this as its closing implementation candidate.

## What This Changes

The report asks for `agents.defaults.skipBootstrap: true` to either stop workspace bootstrap files from being injected into prompts or be clearly scoped away from runtime injection.

## Best Possible Solution

End with one tested public config contract: either `skipBootstrap` empties runtime bootstrap injection and reports no injected workspace files, or it remains creation-only with schema/help/docs steering prompt-size control to `contextInjection: "never"` and bootstrap budgets.

## Reproduction Assessment

Yes. The report gives deterministic CLI steps, and current-main source inspection gives a high-confidence static reproduction because runtime bootstrap resolution still does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended config contract. PR #75217 is a plausible narrow runtime fix; schema/help clarification is safer if the field must remain creation-only.
