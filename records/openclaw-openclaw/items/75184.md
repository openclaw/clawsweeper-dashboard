## Summary

Keep this open: current `main` and the latest release still load workspace bootstrap files without checking `agents.defaults.skipBootstrap`, and the issue is already paired with an open fixing PR that closes it.

## What This Changes

The issue reports that `agents.defaults.skipBootstrap: true` does not suppress workspace bootstrap file injection during normal agent prompt construction.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75217 so `skipBootstrap` suppresses workspace bootstrap files at runtime while preserving hook-provided bootstrap context.

## Reproduction Assessment

Yes. Source inspection shows the normal runtime prompt path passes config into `resolveBootstrapContextForRun`, but the resolver loads workspace bootstrap files without checking `agents.defaults.skipBootstrap`; I did not run a live CLI repro because this review is read-only.

## Solution Assessment

Yes. The best fix is at the runtime resolver boundary used by prompt assembly, with hook overrides still executed; relying only on `contextInjection: "never"` would leave the misleading existing key unresolved.
