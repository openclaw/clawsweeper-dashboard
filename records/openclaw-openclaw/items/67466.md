## Summary

Keep open: the PR targets a real Feishu startup/status latency problem that is still present on current main and v2026.6.6, but the current branch is merge-conflicting, lacks after-fix real Feishu proof, and introduces a gateway lifecycle regression by detaching the Feishu monitor from the `startAccount` promise.

## What This Changes

The PR changes Feishu monitor startup to run startup probes concurrently with 3000ms caps, starts account monitors in the background, caps setup/status probing at 3000ms, and rewrites related Feishu tests.

## Best Possible Solution

Repair and rebase the PR so gateway `startAccount` remains tied to monitor lifetime, account-specific diagnostics are preserved, timeout policy is aligned with current defaults/env behavior, and redacted real Feishu startup/status proof is supplied before merge.

## Reproduction Assessment

Yes for the PR-introduced lifecycle bug: source inspection shows the Feishu gateway calls `monitorFeishuProvider` with `accountId`, while the PR makes that branch return immediately and the gateway treats returned-promise resolution as channel exit/restart. No high-confidence after-fix latency reproduction is present because the PR lacks real Feishu proof.

## Solution Assessment

No. Bounding slow Feishu probes is a plausible mitigation, but this implementation is not the best fix until it preserves gateway monitor lifetime, accounts for the current timeout/env contract, and proves the behavior in a real Feishu setup.

- **[P1] Keep gateway account start tied to monitor lifetime:** `extensions/feishu/src/monitor.ts:87-93`
  - body: `channel.ts` calls this function with `accountId` from the gateway `startAccount` hook, and `server-channels.ts` treats the returned promise settling as channel exit before scheduling restart attempts.   - confidence: 0.96

  - body: If `fetchBotIdentityForMonitor` rejects, the `Promise.allSettled` rejection only contains the error, so this branch logs `feishu[unknown]` and drops the account context needed to diagnose which Feishu account failed. Catch inside the `accounts.map` callback and return a degraded result with the original account instead.
  - confidence: 0.9
