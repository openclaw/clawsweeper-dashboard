Full review comments:

- **[P1] Preserve inherited memory slots during per-agent cleanup:** `src/plugins/slots.ts:201-205`
  - body: This helper is used by uninstall and failed-update cleanup for agent-specific slots. Because it reuses `resetPluginSlotReferences`, removing an agent override writes `memory-core` or `none` into the agent config instead of deleting the override, so an agent that should fall back to a non-default global `memory.recall` remains pinned to the default after cleanup. Please use a per-agent cleanup path that removes stale role keys and add a non-default global inheritance regression.
  - confidence: 0.88
- **[P2] Preserve inherited memory slots in doctor repair:** `src/commands/doctor/shared/stale-plugin-config.ts:449-455`
  - body: Doctor repair has the same inheritance problem for stale agent-scoped role slots: it replaces the per-agent value with the role default instead of removing the override. With a global `memory.recall` set to another provider, `openclaw doctor --fix` would make that agent use `memory-core` rather than inheriting the global provider, which silently changes the user's memory routing.
  - confidence: 0.86
