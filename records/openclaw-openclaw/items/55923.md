## Summary

Keep open. Current main still lacks the agent payload detection, inline WebChat error rendering, and legacy WebSocket query-auth fast rejection this PR is trying to add, so the PR is not obsolete or closeable as implemented-on-main.

## What This Changes

The PR adds explicit failures for downgraded text-form tool calls and placeholder-only replies, renders WebChat run errors inline while preserving partial streams, fast-rejects legacy `?agent=...&token=` WebSocket auth attempts, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land a rebased, focused version of this repair or a canonical replacement that keeps the agent payload surfacing and legacy websocket query-auth fast rejection, while coordinating the WebChat overlap with #40277.

Yes. ## Solution Assessment

Yes, subject to final maintainer review and rebase validation. The proposed shape is narrower than #40150 and targets the remaining symptoms directly; the safer alternative is a replacement branch only if this PR cannot be cleanly rebased or coordinated with #40277.
