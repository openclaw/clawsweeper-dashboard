## Summary

Keep open: the current patch still has a correctness blocker because it reports Copilot state-directory status for every ACP harness, and the external PR still needs observed after-fix real behavior proof.

## What This Changes

The PR adds an `openclaw sessions link` CLI helper with tests, ACPx docs, and a changelog entry to print ACP session keys, acpx IDs, Copilot state paths, and TSV/JSON status output.

## Best Possible Solution

Clarify the command scope, then either filter it to Copilot ACP sessions or make state-path/status resolution backend-aware, with regression coverage and redacted after-fix runtime proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR scans every `entry.acp` session while hardcoding `.copilot/session-state`, and current docs/tests show non-Copilot ACP sessions. The original feature need has before-only operator evidence, not after-fix proof.

## Solution Assessment

No. A generic `sessions link` command should use backend/harness-aware state resolution, or the command should be explicitly Copilot-scoped before it reports Copilot state-directory status.
