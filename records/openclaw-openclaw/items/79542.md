## Summary

Keep open: this is a plausible operator diagnostic feature, but the current PR is not merge-ready because it treats every ACP session as Copilot-backed while hardcoding Copilot state paths, and it still lacks after-fix real behavior proof.

## What This Changes

The PR adds an `openclaw sessions link` CLI command, tests, ACPx docs, and a changelog entry to correlate OpenClaw ACP session keys with backend session IDs and Copilot state paths.

## Best Possible Solution

Clarify the command scope, then either filter it to Copilot ACP sessions or make path/status resolution backend-aware, with a non-Copilot regression test and redacted after-fix runtime proof.

## Reproduction Assessment

Yes for the review finding: the PR diff shows all `entry.acp` rows get a Copilot state-path check, while current docs and tests show ACP sessions can be non-Copilot. The original request is a new diagnostic capability rather than a broken existing contract.

## Solution Assessment

No. The proposed implementation is not the best maintainable solution until it is explicitly Copilot-scoped or resolves state diagnostics through backend/harness-aware contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
