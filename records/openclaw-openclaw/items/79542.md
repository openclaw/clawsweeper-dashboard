## Summary

Keep open: the command needs changes because it lists every ACP session while resolving all state paths under Copilot's session-state directory, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds an `openclaw sessions link` CLI command, ACPx docs, tests, and a changelog entry for printing ACP session keys, acpx session IDs, Copilot state paths, and status output.

## Best Possible Solution

Clarify the scope, then either filter to Copilot ACP sessions or make state-path/status resolution backend-aware for all ACP harnesses, with non-Copilot regression coverage and redacted live command proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR scans every `entry.acp` session while hardcoding `.copilot/session-state`, and current docs/tests show non-Copilot ACP sessions. The original feature need has before-only real-environment evidence, not after-fix proof.

## Solution Assessment

No. The proposed implementation is not the best maintainable shape until it either becomes explicitly Copilot-scoped or resolves state diagnostics through ACP backend/harness-aware contracts.
