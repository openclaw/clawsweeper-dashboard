## Summary

Keep open: the report matches the current status pipeline’s stale-token behavior, and the likely gap is xAI/Grok usage capture rather than context-window lookup.

## What This Changes

The issue requests that `/status` and `session_status` show live context-used tokens for xAI/Grok sessions instead of `?/1.0m`.

## Best Possible Solution

Capture and normalize xAI/Grok prompt usage into `lastCallUsage` or `promptTokens` so session rows become fresh, while preserving the stale-total guard for other providers.

## Reproduction Assessment

Yes from source inspection, but not from an end-to-end live run here. The current status renderer will show `?/1.0m` whenever xAI sessions lack fresh `totalTokens`, and the report supplies the missing live-provider symptom.

## Solution Assessment

Unclear on the exact implementation point, but the best direction is not to loosen the status renderer. The narrow fix should make xAI usage metadata populate the existing fresh-context accounting path.
