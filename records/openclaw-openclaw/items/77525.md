## Summary

Keep open: the report matches current `main` behavior when an xAI/Grok run has a known context window but no fresh prompt or last-call usage snapshot, and the fix should be in provider usage capture rather than the status renderer.

## What This Changes

Make `/status` and `session_status` show actual used context tokens for xAI/Grok sessions instead of `?/<context window>`.

## Best Possible Solution

Normalize xAI/Grok prompt or last-call usage into the existing fresh session accounting path so context usage displays accurately while stale totals remain unknown.

## Reproduction Assessment

Yes from source inspection, but not from an end-to-end live run in this review. Current `main` will render `?/1.0m` whenever a Grok session has `contextTokens` but no fresh `totalTokens`, and the issue supplies the provider-specific live symptom.

## Solution Assessment

Unclear on the exact code insertion point until the xAI usage payload shape is verified. The maintainable direction is to populate `lastCallUsage` or `promptTokens` for xAI rather than relaxing the status renderer’s stale-total protection.
