## Summary

Keep open: the report still matches a real xAI/Grok usage-accounting gap on current main, and the existing status renderer is correctly refusing to invent context usage when the provider path leaves the latest usage snapshot empty.

## What This Changes

Report requests that xAI/Grok sessions populate `/status` context-used tokens instead of rendering `?/1.0m`.

## Best Possible Solution

Normalize xAI Responses usage at the parser or provider stream layer that sees the raw `response.completed` event, then preserve the existing status stale-total guard.

## Reproduction Assessment

Yes. Source inspection plus the reporter's captured Grok usage shows that an all-zero or missed xAI assistant usage snapshot flows through persistence as `totalTokensFresh=false`, which makes `/status` render `?/1.0m`.

## Solution Assessment

Yes on direction, unclear on the exact patch point. The narrow maintainable fix is to populate `lastCallUsage` or `promptTokens` from xAI Responses usage, not to loosen the status renderer's stale-context guard.
