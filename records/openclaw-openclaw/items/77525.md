## What This Changes

The issue asks for xAI/Grok sessions to populate actual context-token usage in `/status` instead of rendering `?/1.0m`.

## Best Possible Solution

Normalize xAI Responses usage before assistant messages are stored so the existing fresh-context accounting path receives a positive prompt snapshot, while preserving the stale-total `?` behavior for genuinely unknown usage.

## Reproduction Assessment

Yes from source inspection and the reporter's captured Grok 4.3 payloads, though not from a live Grok run in this review. A zero assistant usage snapshot on current main flows to `totalTokensFresh=false`, which renders as `?/1.0m`.

## Solution Assessment

Yes on direction: the narrow fix should make xAI/Grok usage populate the existing normalized `lastCallUsage`/prompt-token path. Loosening the status renderer would regress the stale-total guard from related status fixes.
