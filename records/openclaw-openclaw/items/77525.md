## Summary

Keep open: current main still has the reported failure mode when xAI/Grok usage reaches OpenClaw as a zero-only assistant usage snapshot, and the bundled xAI wrapper does not recover provider usage before the status path renders unknown context.

## What This Changes

Show actual used/max context for xAI/Grok `/status` sessions instead of `?/1.0m`.

## Best Possible Solution

Capture and normalize xAI Responses usage into the existing `lastCallUsage` or `promptTokens` path, while preserving the stale-total guard in the status renderer.

## Reproduction Assessment

Yes for the OpenClaw code path: a zero-only Grok assistant usage snapshot leads to no positive prompt total, `totalTokensFresh=false`, and `/status` renders `?/1.0m`. I did not establish a live Grok run locally because no xAI key was available.

## Solution Assessment

Unclear on the exact implementation point until the raw xAI Responses usage event is confirmed. The narrow maintainable fix is provider/dependency usage capture, not a renderer fallback that would reintroduce stale or cumulative totals.
