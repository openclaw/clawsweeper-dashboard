## Summary

Keep open: current main does not implement the requested overview wording and usage-cost hints, and the PR still has source-backed correctness issues plus missing real UI proof.

## What This Changes

The PR updates Control UI usage-cost wording, threads usage cost summary data into overview cards, and adds 7-day and 30-day usage-cost hints to the overview card.

## Best Possible Solution

Keep the clearer wording only if maintainers want it, but compute overview hints from complete calendar windows over one consistent cost scope and fall back when cost coverage is incomplete.

## Reproduction Assessment

Yes for the PR branch via source inspection: PR head renders fixed `7d`/`30d` labels from selected-range daily rows, while current main defaults the usage range to today. This is a PR-introduced behavior, not a current-main bug.

## Solution Assessment

No. The wording direction is plausible, but the implementation should calculate real calendar windows from one cost source and suppress cost-window hints when cost data is incomplete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Compute real windows before labeling cost hints:** `ui/src/ui/views/overview-cards.ts:136-139`
  - body: The new hint labels whatever `usage.cost` daily rows were loaded as `7d` and `30d`. `loadUsage` requests the currently selected start/end range, and the default range is today only, so a fresh dashboard can show a one-day total as both 7-day and 30-day spend.
  - confidence: 0.91
- **[P2] Sum calendar days instead of populated entries:** `ui/src/ui/views/overview-cards.ts:80-84`
  - body: `usageCostSummary.daily` only contains days with usage, but `.slice(-days)` takes the last N populated rows. Intermittent activity can make the advertised 7-day or 30-day total include spend from much older calendar days.
  - confidence: 0.88
- **[P2] Use one cost scope for value and hint:** `ui/src/ui/views/overview-cards.ts:97-103`
  - body: The card value prefers `usageResult.totals`, while the hint comes from `usageCostSummary.daily`. `sessions.usage` is capped at 1000 sessions but `usage.cost` is not, so high-volume profiles can display a value and hint from different datasets.
  - confidence: 0.86
- **[P2] Hide window hints when costs are incomplete:** `ui/src/ui/views/overview-cards.ts:136-139`
  - body: This branch shows dollar-window hints whenever daily rows exist, but the cost summary separately tracks rows with missing costs. Providers or transcripts without cost reporting can therefore surface misleading `$0.00` windows instead of falling back to token/message context.
  - confidence: 0.84
