## Summary

Keep open. Current main has not implemented the wording/card change, but the PR still has source-reproducible Control UI cost-card defects that can mislabel selected-range data as fixed 7-day/30-day windows, overcount sparse activity, mix capped and uncapped data scopes, and show false-zero cost hints when cost coverage is incomplete.

## What This Changes

The PR renames Control UI cost copy to “Usage Cost”, passes `usageCostSummary` into overview cards, and adds 7-day and 30-day usage-cost hints to the overview cost card.

## Best Possible Solution

Keep the clearer Usage Cost wording, but make the overview card's cost hints range-aware, calendar-day based, cost-coverage aware, and data-scope consistent, with a changelog entry and focused UI regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: default overview loading requests today-to-today usage data, while the PR labels any returned daily cost entries as `7d` and `30d`; sparse daily fixtures and `missingCostEntries` cover the other defects.

## Solution Assessment

No. The direction is useful, but this implementation should derive the card value and hints from one consistent cost scope, label only actual calendar coverage, and fall back when cost data is incomplete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Compute actual windows before labeling cost hints:** `ui/src/ui/views/overview-cards.ts:136-139`
  - body: The hint always prints `7d` and `30d`, but overview loading passes the current usage date filters to `usage.cost`; those filters default to today-to-today. A fresh dashboard can therefore present one-day data as 7-day and 30-day spend.
  - confidence: 0.9
- **[P2] Sum calendar days instead of populated entries:** `ui/src/ui/views/overview-cards.ts:80-84`
  - body: `usageCostSummary.daily` only contains days with usage, so `.slice(-days)` takes the last N active days. For intermittent usage, the displayed 7-day or 30-day totals can include older spend outside the advertised calendar window.
  - confidence: 0.88
- **[P2] Use one cost scope for card value and hint:** `ui/src/ui/views/overview-cards.ts:97-103`
  - body: The main value prefers `usageResult.totals`, while the hint uses `usageCostSummary.daily`. `sessions.usage` is capped at 1000 sessions but `usage.cost` is not, so high-volume workspaces can see a card value that disagrees with the hint's data scope.
  - confidence: 0.84
- **[P2] Suppress cost-window hints when coverage is incomplete:** `ui/src/ui/views/overview-cards.ts:136-139`
  - body: Daily entries can exist while cost data is missing, tracked separately by `missingCostEntries`. In that case this branch can show `$0.00` windows instead of the prior token/message context, which is misleading for providers or transcripts without cost reporting.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `ui/src/i18n/locales/en.ts:273`
  - body: This changes user-facing Control UI copy and overview-card behavior, but the diff does not add an `Unreleased` changelog entry. Please add a single-line entry under the appropriate section without bot attribution.
  - confidence: 0.78
