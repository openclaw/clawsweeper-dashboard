The PR renames Control UI cost labels to Usage Cost, threads usageCostSummary into overview cards, and updates i18n metadata for the changed wording.

## Best Possible Solution

Keep the clearer wording if maintainers want it, but rebase onto current main, compute calendar windows from one complete cost source, suppress dollar windows when cost coverage is incomplete, and attach redacted UI proof.

## Reproduction Assessment

Yes. Source inspection is enough: PR head labels whatever usageCostSummary.daily rows were loaded as 7d and 30d, while current main defaults the selected usage range to today and emits sparse daily rows.

## Solution Assessment

No. The wording change is useful, but the implementation should compute real calendar windows from one complete cost source, handle missing cost coverage, and preserve current main's provider-quota overview card after rebase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Compute real windows before labeling cost hints:** `ui/src/ui/views/overview-cards.ts:137-138`
  - body: The hint labels the currently loaded `usageCostSummary.daily` rows as 7d and 30d. `loadUsage` requests the selected start/end range and current main defaults both dates to today, so a normal overview load can show a one-day total as both fixed windows.
  - confidence: 0.91
- **[P2] Sum calendar days instead of populated entries:** `ui/src/ui/views/overview-cards.ts:80-84`
  - body: `usageCostSummary.daily` is sparse on current main, but `slice(-days)` takes the last N populated rows. Intermittent activity can make the advertised 7-day or 30-day total include spend from much older calendar days.
  - confidence: 0.88
- **[P2] Use one cost scope for value and hint:** `ui/src/ui/views/overview-cards.ts:97-103`
  - body: The card value can come from `usageResult.totals` while the hint comes from `usageCostSummary.daily`. `sessions.usage` is capped at 1000 sessions but `usage.cost` is not, so high-volume workspaces can see inconsistent cost totals.
  - confidence: 0.86
- **[P2] Hide window hints when costs are incomplete:** `ui/src/ui/views/overview-cards.ts:136-139`
  - body: This branch shows dollar-window hints whenever daily rows exist, but the summary separately tracks `missingCostEntries`. Providers or transcripts without cost reporting can therefore surface misleading `$0.00` windows instead of falling back to token/message context.
  - confidence: 0.84
