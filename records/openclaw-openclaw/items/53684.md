## Summary

Keep open. Current main still has adjacent status/probe, managed-service lifecycle, LaunchAgent KeepAlive, stability bundles, channel health, cron failure alerts, and rescue-bot docs, but it does not provide the requested gateway-down watchdog, visible notification/escalation path, human-readable startup-error artifact, dead-man channel monitoring, or first-class sibling health monitoring. The issue also has an open implementation candidate in PR #53716, so it should remain open until that PR or a successor is reviewed and landed or closed.

## What This Changes

Request gateway failure recovery with watchdog restart attempts, visible alerts, startup failure diagnostics, channel dead-man monitoring, and multi-instance health notifications.

## Best Possible Solution

Land a focused, tested gateway recovery path that covers health polling, bounded restart or repair attempts, visible local and optional channel alerts, persisted startup failure diagnostics, and documented multi-instance monitoring; if the broader rescue-watchdog PR becomes canonical, keep this issue open until the user-visible watchdog and alert pieces are actually included.
