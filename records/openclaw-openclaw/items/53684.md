## Summary

Keep open. Current main and v2026.5.22 still provide adjacent status/probe, stability-bundle diagnostics, LaunchAgent KeepAlive, channel-health restarts, and rescue-profile docs, but not the requested proactive gateway-down watchdog, visible escalation, startup-error artifact, dead-man alert, or first-class sibling monitoring; open watchdog PRs still need maintainer product/security review.

## What This Changes

The issue requests proactive gateway failure recovery, visible alerts, startup diagnostics, dead-man messaging alerts, and optional multi-instance health monitoring when the OpenClaw gateway crashes or becomes unreachable.

## Best Possible Solution

Keep this as the canonical reliability feature while maintainers choose one recovery design, then land or replace the open watchdog candidates with a focused implementation that reuses current service, diagnostic, health, and notification contracts.

## Reproduction Assessment

## Solution Assessment
