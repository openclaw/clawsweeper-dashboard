## Summary

Keep open. Current main and v2026.4.29 contain source-side installer hardening, but the public hosted installer still serves the stale Node 22/raw apt script that matches the report, and open PR #73855 is an unmerged closing candidate.

## What This Changes

The issue reports that the hosted Linux installer falsely reports Node.js installed after apt/ioctl failures, then exits because Node 22+ is absent.

## Best Possible Solution

Deploy or sync the hosted installer from current scripts/install.sh, then land or replace a hosted-drift guard so future source/hosted mismatches fail visibly.

## Solution Assessment
