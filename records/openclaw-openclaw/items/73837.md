Keep open. Current main has source-side installer hardening, but the live hosted installer still serves the stale Node 22/raw apt path that matches the report, and open PR #73855 still references this issue with closing syntax.

## What This Changes

The issue reports that the public install.sh Node auto-install path can print success after a failed NodeSource/apt install and then crash because node is not on PATH.

## Best Possible Solution

Sync and deploy the hosted openclaw.ai installer from canonical scripts/install.sh, then land drift detection so future installer-source changes cannot leave stale public scripts behind.
