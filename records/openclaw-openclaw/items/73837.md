## Summary

Keep open. Current main has installer-source hardening, but the live hosted installer still serves the stale Node 22/raw apt path that matches the report, and open PR #73855 still references this issue with closing syntax.

## What This Changes

The issue reports that the hosted Linux installer claims Node.js v22 was installed after apt/ioctl failures, then fails its own Node.js PATH/version check.

## Best Possible Solution

Sync and deploy the hosted openclaw.ai installer from canonical scripts/install.sh, then land drift detection so future installer-source changes cannot leave stale public scripts behind.

## Reproduction Assessment

Yes for the user-visible hosted installer path: fetching https://openclaw.ai/install.sh still shows setup_22.x, raw apt-get install nodejs, the Node.js v22 installed success line, and the Node.js v22+ PATH error marker. I did not run a full WSL2 apt install in this read-only review.

## Solution Assessment

No, not as a cleanup close yet. The source hardening is the right code direction, but the issue should stay open until the hosted installer sync and drift guard are reviewed, merged, and deployed.
