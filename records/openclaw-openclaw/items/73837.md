## Summary

Keep open. Current main and the latest release contain installer hardening, but the public hosted installer still serves the stale Node 22/raw apt script that matches the report, and open PR #73855 still uses closing syntax for this issue.

## What This Changes

The issue reports that the public Linux installer on WSL2 fails during NodeSource/apt Node installation, prints a false Node.js success message, then aborts because Node is absent.

## Best Possible Solution

Deploy or sync the hosted installer from current scripts/install.sh, then land a hosted-drift guard so future source/hosted mismatches fail visibly.

## Reproduction Assessment

Yes. Fetching the live hosted installer is a high-confidence artifact-level reproduction because it still contains the stale Node 22 setup path, raw apt install calls, false success text, and final Node.js v22+ PATH error described in the report.

## Solution Assessment

No. The source-side installer hardening is the right implementation direction, but the user-visible issue is not solved until https://openclaw.ai/install.sh serves that hardened script; PR #73855 is useful guard work but not a replacement for the hosted sync.
