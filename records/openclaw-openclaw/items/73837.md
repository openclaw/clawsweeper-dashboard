## Summary

Keep open. Current source and the latest release tag contain installer-side hardening for noninteractive apt and gum ioctl fallback, but the live hosted installer still serves the stale Node 22/raw apt script that matches the report, docs still point users to that hosted script, and open PR #73855 uses closing syntax for this issue.

## What This Changes

The issue asks OpenClaw's Linux installer to detect failed Node.js auto-installation on WSL2/Ubuntu instead of printing a false success message and then crashing on the Node version check.

## Best Possible Solution

Deploy or sync the hosted installer from current `scripts/install.sh`, keep the source hardening, and land or replace the hosted-drift guard so future installer-source mismatches fail loudly before users hit the stale artifact.

## Reproduction Assessment

Yes. The live hosted installer still contains the stale Node 22 NodeSource path, raw apt install calls, narrow gum ioctl detection, false `Node.js v22 installed` message, and final Node v22+ PATH error from the report; an actual WSL2 smoke remains the final runtime confirmation after deployment.

## Solution Assessment

No. The current source hardening is necessary but incomplete while the public hosted installer remains stale; the maintainable fix needs hosted installer sync plus an automated drift check like the linked PR.
