## Summary

Keep open. Current source and the latest release tag contain installer hardening for noninteractive apt and gum ioctl fallback, but the public hosted installer still differs from `scripts/install.sh` and still contains the stale Node 22/raw apt/error strings that match the report. PR #73855 is also still open with closing syntax for this issue, so the issue should remain open until the hosted installer is synced and the linked PR is reviewed or replaced.

## What This Changes

Fix the WSL2/Linux installer Node auto-install path so apt/gum failures are detected accurately and the installer does not report Node installed before a usable `node` is on PATH.

## Best Possible Solution

Sync or redeploy `https://openclaw.ai/install.sh` from the hardened `scripts/install.sh`, then land or replace PR #73855 with a drift check that catches future hosted/source mismatch without blocking normal installer changes. Final proof should compare checksums and run a WSL/Linux install path with no preinstalled Node to ensure success is printed only after `node` is usable.

## Reproduction Assessment

Yes. Fetching the live hosted installer currently reproduces the stale surface by showing the Node 22 setup script, raw apt install, misleading success text, and final PATH error string from the report; a final WSL2 smoke is still needed after deployment.

## Solution Assessment

No. Source-side hardening is necessary but not sufficient while the public hosted installer remains stale; the maintainable fix needs hosted deployment sync plus a reviewed guard against future drift.
