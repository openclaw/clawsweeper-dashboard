## Summary

Keep open: current main still pins the official WeCom install catalog to 2026.4.23 while npm latest is 2026.5.7, and the wizard path calls the npm installer without update/idempotent mode, so an existing managed npm package directory can still fail with the reported already-exists error.

## What This Changes

The issue asks for the configuration wizard to recognize an already-installed newer WeCom plugin instead of attempting to install an older pinned package version.

## Best Possible Solution

Make the WeCom auto-install flow idempotent for an already-installed compatible managed npm package, and refresh the official WeCom catalog metadata to the current trusted release and integrity.

## Reproduction Assessment

Yes from source inspection, though not from a live Docker wizard run: current main still pins WeCom to 2026.4.23 and the wizard calls the npm installer in default install mode, which rejects an existing managed package directory.

## Solution Assessment

No: current main is not the best behavior because it can fail instead of recognizing or updating a compatible installed WeCom package. A narrow owner-boundary fix should refresh the catalog pin and add regression coverage for the existing-install wizard path.
