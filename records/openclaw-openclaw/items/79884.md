## Summary

Keep open: current main still pins the official WeCom install catalog to `@wecom/wecom-openclaw-plugin@2026.4.23` while npm latest is `2026.5.7`, and the managed npm installer rejects an existing target before checking whether the installed version is compatible. The linked open fix PR uses closing syntax, so this issue should stay open until that PR is corrected and merged. https://github.com/openclaw/openclaw/issues/79884

## What This Changes

Make the wizard/plugin install path recognize an already-installed compatible or newer WeCom managed npm package instead of trying to reinstall an older pinned version.

## Best Possible Solution

Land a generic installer-level fix that treats an existing compatible managed npm package as satisfying the requested install only after preserving package manifest, plugin id, integrity, and safety validation; update the WeCom catalog pin if maintainers want the wizard to request `2026.5.7` specifically.

## Reproduction Assessment

Yes from source inspection, though not from a live Docker wizard run. Current main still resolves the old WeCom catalog spec and rejects an existing managed npm target before any compatible installed-version check.

## Solution Assessment

Yes on direction: a generic installer-level idempotency fix is the narrow maintainable solution, not a WeCom-only workaround. The current linked PR needs its validation/comparator/proof blockers resolved before it should close this issue.
