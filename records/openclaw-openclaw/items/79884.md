## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79884. The WeCom wizard/channel setup path currently uses the official external catalog spec `@wecom/wecom-openclaw-plugin@2026.4.23` even though npm latest is 2026.5.7, and then `installPluginFromNpmSpec` fails in install mode when the managed npm package directory already exists. Keep the official external source policy explicit: either update the WeCom catalog to the current exact version plus npm dist integrity and add tests, or add an idempotent compatible-installed check/update path so onboarding/channel setup treats an already-installed same-or-newer managed npm package as satisfied instead of asking users to delete it. Do not switch official external installs to an unpinned floating spec, do not weaken expected-integrity checks, and keep core generic rather than adding WeCom-only runtime branches outside the catalog metadata. Add/update a user-facing changelog entry.
