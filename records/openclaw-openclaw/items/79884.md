## Summary

Keep open: current main still has the reported WeCom wizard install failure path, with the official catalog pinned to an older exact npm version while onboarding uses install mode that rejects an existing managed npm package directory.

## What This Changes

The issue asks the WeCom setup wizard to recognize an already-installed newer managed npm plugin instead of attempting an older install and failing on an existing directory.

## Best Possible Solution

Update the official WeCom catalog spec and integrity to the current trusted release, then make the wizard treat an already-installed exact matching managed npm package as installed without trusting arbitrary newer artifacts against an old integrity pin.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the WeCom catalog provides 2026.4.23, onboarding passes that spec into the npm installer, and default install mode rejects the existing managed npm package directory.

## Solution Assessment

No. Current main is not the best behavior; a narrow fix should refresh the trusted WeCom metadata and add idempotent handling for an already-installed matching managed npm package rather than broadening exact-integrity pins to accept any newer package.
