## What This Changes

## Best Possible Solution

Make the WeCom auto-install path idempotent for an already-installed compatible managed npm package while keeping official external source integrity explicit.

Fix the WeCom wizard/plugin install bug reported at https://github.com/openclaw/openclaw/issues/79884. Current main still has `scripts/lib/official-external-channel-catalog.json` pinned to `@wecom/wecom-openclaw-plugin@2026.4.23`, while npm latest is `2026.5.7`; `ensureOnboardingPluginInstalled` calls `installPluginFromNpmSpec` without update/idempotent handling, so an existing managed npm package directory can fail with `plugin already exists`.
