## Summary

Keep open: the reconnect-herd problem is source-reproducible on current main, but this PR still targets stale wrapper-level `baseDelay` behavior that current main's active Discord scheduler does not read, and it lacks after-fix real Discord proof.

## What This Changes

The branch threads Discord `accountId` into gateway plugin creation and adds deterministic per-account `reconnect.baseDelay` jitter plus wrapper-level tests.

## Best Possible Solution

Thread account identity into the internal Discord gateway options and apply deterministic additive jitter inside `scheduleReconnect` while preserving the reconnect floor, invalid-session floor, 30-second cap, and `maxAttempts` behavior.

## Reproduction Assessment

Yes, by source inspection. Current main schedules Discord reconnects from `minDelayMs` and reconnect attempt count only, so same-attempt multi-account reconnects remain synchronized.

## Solution Assessment

No. Setting `reconnect.baseDelay` in the monitor wrapper is not the best fix because current main's active reconnect scheduler lives in `extensions/discord/src/internal/gateway.ts` and does not read that option.
