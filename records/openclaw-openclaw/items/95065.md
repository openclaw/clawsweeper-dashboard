## Summary

Keep open: the stored requester-origin backfill is useful, but the PR also broadens direct DM media delivery after generic requester-agent handoff errors, which can duplicate media after unknown side effects. External real behavior proof is also missing.

## What This Changes

The PR changes subagent completion delivery so generated DM media can use the requester session's stored delivery context and direct-send when requester handoff fails.

## Best Possible Solution

Keep the stored-origin backfill, but preserve the no-unknown-side-effect guard by limiting direct media fallback to failures that prove no requester-agent delivery occurred or to the existing write-lock recovery path.

## Reproduction Assessment

## Solution Assessment
