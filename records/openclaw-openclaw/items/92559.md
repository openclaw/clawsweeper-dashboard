## Summary

Keep open. Current main and v2026.6.6 still only support static `extra_body` pass-through, and session-aware templating would need a maintainer product/security decision before implementation.

## What This Changes

Request session/channel-aware template variables in `extra_body` so provider request fields can vary per session without duplicating agents.

## Best Possible Solution

Design an explicit opt-in session-attribution/template seam with constrained, sanitized variables, provider/request-shaping tests, and docs, rather than making every existing `extra_body` string dynamic by default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection gives high confidence that current main lacks session-aware `extra_body` templating.

## Solution Assessment

Unclear as requested. Template variables are plausible, but maintainers should first choose the opt-in syntax, provider SDK boundary, cache behavior, and identifier-sanitization policy.
