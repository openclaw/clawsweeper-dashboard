## Summary

Keep open. Current main and v2026.6.8 still only support static `extra_body` pass-through; they do not resolve session/channel template variables, and the related OpenRouter PR is a narrower provider-specific `session_id` feature rather than a full solution for this issue.

## What This Changes

The issue requests session/channel-aware template interpolation for provider `extra_body` values so one agent can send per-session identifiers such as channel names to upstream providers.

## Best Possible Solution

Design an explicit opt-in session-attribution or template seam that exposes a constrained, sanitized session context at request time, with provider-boundary docs and focused tests, instead of making all existing `extra_body` strings dynamic by default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection gives high confidence that current main and v2026.6.8 only handle static `extra_body` values in this path.

## Solution Assessment

Unclear as requested. Template variables are plausible, but maintainers should first choose the opt-in syntax, provider SDK boundary, and raw-versus-derived identifier policy.
