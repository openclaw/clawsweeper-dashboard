## Summary

Keep open. Current main and the latest release support static `extra_body` pass-through only; the requested request-time `{{session.*}}` expansion remains missing and needs a privacy/API-shape decision before implementation. The open related work at https://github.com/openclaw/openclaw/pull/54725 and https://github.com/openclaw/openclaw/issues/88616 is adjacent but does not make this issue obsolete.

## What This Changes

Requests per-session template variables in agent `extra_body` values so a single agent can send channel/session-specific provider request fields.

## Best Possible Solution

Keep this open for a maintainer-owned provider/privacy design: either add an explicit opt-in session metadata templating path for `extra_body`, or land a narrower provider hook seam and let provider plugins own identifier forwarding.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection verifies the current path is static config pass-through and does not resolve session/channel template variables.

## Solution Assessment

Unclear as proposed. `extra_body` templating could solve the user problem, but the best fix likely needs a maintainer decision on the privacy-safe identifier surface before code is queued.
