## Summary

Keep open. Current main and v2026.6.8 still only support static `extra_body` pass-through, while session/channel-aware request metadata remains a product and privacy-sensitive design decision.

## What This Changes

Request session/channel-aware template variables in provider `extra_body` so one agent can send per-session values such as DeepSeek `user_id`.

## Best Possible Solution

Design an explicit opt-in session-attribution seam with constrained, sanitized fields and provider/docs/tests, rather than making all existing `extra_body` strings dynamic by default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection gives high confidence that current main only passes static `extra_body` values and does not expose channel/session context to that path.

## Solution Assessment

Unclear as requested. Template variables are plausible, but maintainers should first choose the opt-in syntax, allowed identifiers, sanitization policy, and owner boundary.
