## Summary

Keep open. Current main still surfaces ACP parent relay progress, stall, and resumed events through `enqueueSystemEvent`, and #55896 is an open, unmerged closing PR for the narrow Discord-thread suppression. Repository policy says this issue should stay open while that implementation PR is pending.

## What This Changes

Suppress ACP parent relay progress, stall, and resumed system events from rendering as ordinary messages in visible Discord ACP worker threads while preserving canonical lifecycle notices.

## Best Possible Solution

Land or replace the #55896-style fix: filter only intermediate ACP parent relay `progress`, `stall`, and `resumed` events for visible Discord thread parent sessions, keep relay logging and terminal lifecycle notices, and cover both the relay and spawn wiring with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: `streamTo: "parent"` starts the parent relay, assistant/stall/resumed events flow through `emit()`, and `emit()` enqueues them into the parent session without Discord-thread filtering.

## Solution Assessment
