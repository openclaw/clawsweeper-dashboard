## Summary

Keep open. Current main still lacks the requested schema-backed `agents.defaults.memorySearch.query.timeoutMs`, `cacheTtlMs`, and `retry` controls; existing Active Memory and QMD tuning knobs are adjacent but do not satisfy this core memory-search config request.

## What This Changes

Request to add durable per-agent memory-search query timeout, result-cache TTL, and retry-policy config and wire it into runtime behavior and docs.

## Best Possible Solution

Add the controls at the shared memory-search query config boundary, resolve them per agent, apply them to the memory-search execution path with graceful fallback, and document how they differ from QMD and Active Memory plugin-specific knobs.

## Solution Assessment

Yes. The requested direction is the narrow maintainable path because it extends the existing `memorySearch.query` config boundary instead of treating plugin-local Active Memory or QMD-only settings as substitutes.
