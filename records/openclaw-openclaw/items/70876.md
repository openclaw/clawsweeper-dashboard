## Summary

Keep this PR open: the underlying `/tools` MCP inventory gap is still present on current main, but this patch is not ready to merge because it misses the cold-session reproduction path, leaks the materialized MCP lease, and mixes in unrelated plugin installer dependency changes.

## What This Changes

The PR adds an `mcp` effective-tool source, appends materialized session MCP tools to `/tools`, and also changes plugin install peer-dependency handling and its test expectation.

## Best Possible Solution

Land a focused `/tools` MCP inventory fix that mirrors the run path for cold and warm sessions, policy filtering, collision naming, and lifecycle disposal, while moving plugin installer dependency changes to a separate reviewed PR.

## Reproduction Assessment

Yes. Configure `mcp.servers`, start a new session, run `/tools verbose` before an agent turn, and compare it with the run path that later materializes bundle MCP tools.

## Solution Assessment

No. The proposed direction is close, but this patch only works after a runtime already exists, does not dispose the materialized lease, and couples the fix to unrelated installer behavior.
