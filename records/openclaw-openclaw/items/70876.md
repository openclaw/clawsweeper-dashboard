- **[P2] Keep /tools from creating MCP runtimes:** `src/auto-reply/reply/commands-info.ts:175-180`
  - body: Resolving a session id does not make this read-only: `getOrCreateSessionMcpRuntime` can replace a stale runtime or create a missing one, and materialization can issue MCP catalog work.   - confidence: 0.9
- **[P2] Dispose the MCP materialization lease:** `src/auto-reply/reply/commands-info.ts:185-188`
  - body: `materializeBundleMcpToolsForRun` returns a `dispose()` handle that releases the runtime lease, but this handler reads `mcpTools.tools` and drops the handle. Each `/tools` call can leave an active lease and prevent the session MCP runtime from being swept at idle.
  - confidence: 0.88
- **[P2] Align the installer test with the production command:** `src/infra/install-package-dir.test.ts:323`
  - body: The test now expects npm to run with `--omit=peer`, but this PR's production command still omits only dev dependencies. The focused installer test will fail as written, so either the test expectation or the production command needs to be corrected before this branch can merge.
  - confidence: 0.95
- **[P2] Run openclaw peer stripping outside the devDependency path:** `src/infra/install-package-dir.ts:64-70`
  - body: The new `peerDependencies.openclaw` removal sits after early returns that require workspace dev dependencies. Plugins that only declare `peerDependencies.openclaw`, or have ordinary dev dependencies, will return before this block and still keep the peer that the change is meant to remove.
  - confidence: 0.9
