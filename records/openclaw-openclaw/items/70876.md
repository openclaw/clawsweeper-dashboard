## Summary

Keep open: current main still omits configured MCP server tools from `/tools`, but this PR is not merge-ready because it only lists MCP tools after an existing runtime has been bound, does not dispose the materialized MCP lease, is dirty against current main, and mixes unrelated plugin installer dependency handling into the inventory fix.

## What This Changes

The branch adds an `mcp` effective-tool source, appends materialized session MCP tools to `/tools`, and changes plugin install peer-dependency handling plus its test expectation.

## Best Possible Solution

Land a focused `/tools` MCP inventory fix that mirrors the run path for cold and warm sessions, applies the same naming and policy behavior, disposes materialized leases after building display metadata, and leaves installer dependency changes to a dedicated review.

## Reproduction Assessment

Yes. Configure `mcp.servers`, start a new session, and run `/tools verbose` before an agent turn; current main builds the command output from core/plugin/channel inventory while the actual run path materializes MCP tools later.

## Solution Assessment

No. The proposed direction is close, but this patch misses cold sessions, leaks the materialization lease, and includes unrelated installer behavior that should not be coupled to a read-only inventory command.
