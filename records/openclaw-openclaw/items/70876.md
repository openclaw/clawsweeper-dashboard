  - body: Resolving a session id does not make this path read-only: `getOrCreateSessionMcpRuntime` can create or replace runtime state, and `materializeBundleMcpToolsForRun` can trigger catalog discovery.   - confidence: 0.9
- **[P2] Dispose the MCP materialization lease:** `src/auto-reply/reply/commands-info.ts:185-188`
  - body: `materializeBundleMcpToolsForRun` returns a `dispose()` handle that releases the runtime lease, but this handler only reads `mcpTools.tools` and drops the handle. Each `/tools` call can leave an active lease and prevent the session MCP runtime from being swept at idle.
  - confidence: 0.88
- **[P2] Align the installer test with the production command:** `src/infra/install-package-dir.test.ts:323`
  - body: The test expects npm to run with `--omit=peer`, but this PR's production command omits only dev dependencies. The focused installer test will fail as written, so either the test expectation or the production command needs to be corrected before this branch can merge.
  - confidence: 0.95
- **[P2] Run OpenClaw peer stripping outside dev dependency filtering:** `src/infra/install-package-dir.ts:64-70`
  - body: The new `peerDependencies.openclaw` removal sits after early returns that require workspace dev dependencies. Plugins that only declare `peerDependencies.openclaw`, or have ordinary dev dependencies, return before this block and still keep the peer that the change is meant to remove.
  - confidence: 0.9
