Full review comments:

- **[P1] Preserve Request policy fields when flattening:** `src/infra/net/proxy-fetch.ts:122-145`
  - body: This helper turns a `Request` into URL plus init but only carries body, signal, headers, and method.   - confidence: 0.9
- **[P2] Require an own URL for request-like inputs:** `src/infra/net/proxy-fetch.ts:90-96`
  - body: The request-like guard uses `"url" in input`, so an inherited prototype url is enough to redirect the proxy fetch and have inherited method/header/body fields considered. In this plugin-facing network egress helper, require an own `url` property or remove the plain-object escape hatch.
  - confidence: 0.78
