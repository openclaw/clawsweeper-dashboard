## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve Request policy fields when flattening:** `src/infra/net/proxy-fetch.ts:122-145`
  - body: `toRequestLikeInit` copies only body, signal, headers, and method from a `Request`. Request inputs also carry policy fields such as redirect, referrer/referrerPolicy, mode, credentials, cache, integrity, and keepalive, so proxied `fetch(request)` can silently run with undici defaults instead of the caller's original request policy.
  - confidence: 0.88
- **[P2] Require an own URL for request-like inputs:** `src/infra/net/proxy-fetch.ts:90-96`
  - body: The request-like guard uses `"url" in input`, so an inherited prototype `url` is enough to choose the proxy target and read related fields. For a plugin-facing network egress helper, require an own `url` property or remove the plain-object path unless a current caller needs it.
  - confidence: 0.78
