## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve Request policy fields when flattening:** `src/infra/net/proxy-fetch.ts:122-145`
  - body: This helper turns a `Request` into URL plus init, but it only carries body, signal, headers, and method. Request inputs also carry `redirect`, `referrer`/`referrerPolicy`, `mode`, `credentials`, `cache`, `integrity`, and `keepalive`; after this patch a proxied `fetch(request)` can fall back to Undici defaults instead of the caller's policy. Preserve the full Request state or use an Undici-backed construction path that keeps it while adding the dispatcher.
  - confidence: 0.9
- **[P2] Require an own URL for request-like inputs:** `src/infra/net/proxy-fetch.ts:90-96`
  - body: The new duck-typed request-like path uses `"url" in input`, so an inherited prototype `url` is enough to redirect the proxy fetch and have inherited method/header/body fields considered. Since this is a plugin-facing network egress helper, require an own `url` property or remove the plain-object escape hatch unless a concrete caller needs it.
  - confidence: 0.78
