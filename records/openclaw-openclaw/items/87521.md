## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve Request metadata when flattening:** `src/infra/net/proxy-fetch.ts:122-145`
  - body: This helper turns a `Request` into `url + init`, but it only carries over body, signal, headers, and method. `Request` also carries policy fields like `redirect`, `referrer`/`referrerPolicy`, `mode`, `credentials`, `cache`, `integrity`, and `keepalive`; after this patch a proxied `fetch(request)` that set `redirect: "manual"` or `referrerPolicy: "no-referrer"` falls back to undici defaults. Please preserve the full Request state, or construct through an undici Request path that keeps those fields while adding the dispatcher.
  - confidence: 0.88
