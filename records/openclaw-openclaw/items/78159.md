## Summary

Keep open: current `main` and the latest release still pass the login-shell env probe options without `windowsHide`, so the reported Windows console-flash path is a real, narrow bug and not covered by the related Windows spawn fixes.

## What This Changes

Add `windowsHide: true` to the login-shell environment probe so Windows gateway shell-env detection does not flash a console window.

## Best Possible Solution

Patch the shell-env exec options to hide Windows child windows, add focused regression coverage, and verify the changed behavior on Windows without changing shell-env import semantics.
