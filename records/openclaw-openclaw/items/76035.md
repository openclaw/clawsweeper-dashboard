## Summary

Keep open: current main still selects blank `BodyForCommands` for hook content, and #76036 is an open closing PR that directly targets this bug, so this issue should remain open until that PR is reviewed and merged.

## What This Changes

Make Slack plugin and internal `message_received` hook content fall back from blank `BodyForCommands` to `RawBody` or `Body` so bot attachment alerts reach hooks.

## Best Possible Solution

Land a narrowly reviewed fix that makes the shared inbound hook mapper skip blank command content while preserving Slack prepare `RawBody`/`BodyForAgent` extraction and nonblank command precedence.

## Reproduction Assessment

Yes. The current code path can be reproduced with a Slack bot message whose top-level `text` is empty and whose attachment has text: Slack prepare leaves `BodyForCommands` blank while preserving `RawBody`, and the mapper returns the blank command body before fallback.

## Solution Assessment

Yes. The paired PR direction is the narrowest maintainable fix: change the shared mapper to prefer only nonblank command content, then cover the Slack input shape and plugin/internal hook outputs with focused tests.
