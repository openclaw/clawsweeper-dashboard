## Summary

Keep open: this PR targets a real P1 preflight compaction bug, but the current patch can still miss later user-stop or restart cancellation after a filtered lifecycle timeout, and its proof is unit-test output rather than real slow-preflight runtime evidence.

## What This Changes

The PR replaces the raw reply-operation abort signal passed to required preflight compaction with a derived signal that filters lifecycle `TimeoutError` and adds unit coverage.

## Best Possible Solution

Use a preflight cancellation source that ignores normal reply lifecycle timeout while still propagating explicit stop, restart, shutdown, or drain cancellation, with regression coverage and real slow-preflight proof.

## Reproduction Assessment

Yes, at source level. In the normal flow the helper is created before lifecycle timeout, so the timeout event can be filtered without starting result polling; later stop or restart then cannot emit another abort from the one-shot source signal.

## Solution Assessment

No. The call-site boundary is plausible, but filtering the already-conflated reply abort signal is not the best fix unless later explicit cancellation still has a separate path to reach compaction.
