## Summary

Keep open: the PR targets a real preflight compaction timeout bug, but the current helper still misses later explicit cancellation after filtering a normal lifecycle timeout and the proof remains test-only.

## What This Changes

The PR replaces the raw reply-operation abort signal passed to budget/preflight compaction with a derived cancellation signal and adds focused unit coverage.

## Best Possible Solution

Use a preflight cancellation source that ignores normal reply lifecycle timeout while still propagating explicit stop, restart, shutdown, or drain cancellation, with focused regression coverage and real slow-preflight proof.

## Reproduction Assessment

Yes, at source level. In the normal flow the helper is created before lifecycle timeout, so the TimeoutError event can be filtered without starting result polling; later stop or restart then cannot emit another abort from the one-shot source signal.

## Solution Assessment

No. The call-site boundary is plausible, but filtering the already-conflated reply abort signal is not the best fix unless later explicit cancellation still has a separate path to reach compaction.
