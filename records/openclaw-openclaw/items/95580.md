## Label Justifications

- `P1`: The PR targets a real large-session compaction regression and the current patch can still break stop/restart cancellation for long preflight compactions.
- `merge-risk: 🚨 session-state`: The diff changes required preflight compaction while session context is near overflow and can keep session state locked or stale longer than intended.

## Summary

Keep open: the patch targets the right preflight compaction boundary, but filtering a one-shot `replyOperation.abortSignal` means a lifecycle `TimeoutError` consumes the only abort event and later user/restart cancellation cannot reach compaction. The PR also still has mock-only proof rather than a real slow-preflight run.

## What This Changes

The PR adds a preflight compaction cancellation-signal helper that suppresses lifecycle `TimeoutError` aborts, uses it at the budget compaction call site, and adds focused unit tests.

## Best Possible Solution

Use a non-conflated cancellation source for preflight compaction: ignore normal reply lifecycle timeout, but still propagate later user stop, restart, shutdown, or drain cancellation, with regression coverage and real slow-preflight proof.

## Reproduction Assessment

Yes, source-reproducible: current main passes `replyOperation.abortSignal` into required budget compaction and the safety wrapper races caller aborts against the configured timeout. The PR-introduced failure is also source-reproducible because an `AbortSignal` cannot emit a second abort after the filtered lifecycle timeout.

## Solution Assessment

No. The call-site boundary is right, but filtering the already-conflated one-shot reply abort signal is not the best fix; a distinct cancellation channel is needed so later explicit aborts still reach compaction.
