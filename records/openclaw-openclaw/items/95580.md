## Summary

Keep open: current main still has the preflight compaction abort-scope bug, but this PR is not merge-ready because the normal timeout-after-helper path still drops later user stop or restart cancellation and the proof remains unit-test only.

## What This Changes

The PR adds a derived preflight compaction cancel signal in `src/auto-reply/reply/agent-runner-memory.ts` and tests lifecycle-timeout filtering around budget compaction.

## Best Possible Solution

Use a non-conflated preflight cancellation source that ignores normal reply lifecycle timeout while still propagating explicit stop or restart cancellation, with regression coverage and real slow-preflight proof.

## Reproduction Assessment

Yes, at source level. In the normal flow the helper is created before lifecycle timeout, so the timeout event can be filtered without starting result polling; later stop or restart then cannot emit another abort from the one-shot source signal.

## Solution Assessment

No. The call-site boundary is right, but filtering the already-conflated reply abort signal is not the best fix unless later explicit cancellation still has a separate path to reach compaction.
