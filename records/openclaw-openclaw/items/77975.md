## Summary

Keep open: current main still has the reported unbounded cache behavior, and the same-author tracking PR #77952 is open, so cleanup-closing this issue would be premature.

## What This Changes

The issue reports that DiscordEntityCache retains unique REST-fetched users, channels, guilds, and members in a Map past their TTL without sweeping or a hard size cap.

## Best Possible Solution

Review and land #77952, or replace it with an equivalent narrow Discord-owned fix that adds expiry sweeping, a hard entry cap, and focused regression coverage.

## Reproduction Assessment

Yes from source inspection: repeated fetches for unique `user:`, `channel:`, `guild:`, or `member:` keys all reach `entries.set`, while unrelated expired keys are never swept and the Map has no maximum size. I did not run a runtime repro because this review was constrained to read-only commands.

## Solution Assessment

Yes: bounding the extension-owned cache in `extensions/discord/src/internal/entity-cache.ts` with eviction and expiry coverage is the narrowest maintainable fix, without changing core channel contracts.
