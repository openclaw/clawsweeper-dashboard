## Summary

Keep open: current main still has the source-reproducible unbounded Discord REST entity-cache behavior, and the same-author open PR at https://github.com/openclaw/openclaw/pull/77952 is the active closing fix path but has not merged.

## What This Changes

The issue asks OpenClaw to bound DiscordEntityCache so distinct expired user, channel, guild, and member REST fetches do not remain retained for the bot lifetime.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77952, or replace it with an equivalent Discord-plugin-local cap/sweep fix plus focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows unique user, channel, guild, and member keys all reach entries.set while only same-key TTL reads are checked, and DiscordClient keeps one cache for its lifetime.

## Solution Assessment

Yes. A Discord plugin-local write-time expiry sweep plus a hard cap is the narrow maintainable fix; the linked PR implements that direction but must merge before this issue can close.
