## Summary

Keep open: current main still has the reported unbounded Discord REST entity cache behavior, and a same-author open PR with closing syntax is already the intended fix path.

## What This Changes

The issue requests bounding DiscordEntityCache REST fetch storage so expired or high-cardinality entities cannot remain in the process Map for the bot lifetime.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77952, or replace it with an equivalent Discord-owned cap/sweep fix and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows repeated unique user, channel, guild, or member fetch keys all reach entries.set while unrelated expired keys are never swept and the Map has no max size.

## Solution Assessment

Yes. Bounding the Discord plugin's existing cache with expiry sweeping and a hard cap is the narrowest maintainable fix, and the open linked PR already takes that direction.
