## Summary

Keep open. Current main supports manual and plugin-assisted device pairing, but it does not provide a WhatsApp or messaging-delivered OTP handshake for Control UI browser authentication; this is a new security-sensitive authentication feature that needs maintainer design rather than cleanup closure.

## What This Changes

Request a WhatsApp or messaging-based one-time-code flow that lets an unrecognized browser authenticate and pair with the Control UI without SSH tunneling.

## Best Possible Solution

Design an opt-in, channel-agnostic device-pair approval flow with explicit security properties, then let WhatsApp or other messaging plugins deliver approval material through that seam.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs show the current pairing behavior and confirm there is no existing WhatsApp or messaging OTP path to reproduce as broken behavior.

## Solution Assessment

Unclear. The request is plausible, but the safer maintainable approach is likely a generic device-pair approval/notification design rather than hardcoding WhatsApp into gateway authentication.
