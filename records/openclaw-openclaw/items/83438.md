## Summary

This is a focused bug-fix PR, but it should stay open because the patch makes registry errors nonfatal based on raw config references rather than effective plugin activation, which can hide errors for non-bundled plugins that are active by default.

## What This Changes

The branch changes config plugin validation so registry errors for discovered plugins only fail when the plugin id is referenced in plugin entries, allow, deny, or slots, and adds regression tests for unrelated broken plugins.

## Best Possible Solution

Keep the compatibility fix, but downgrade registry errors only when the plugin is proven inactive by effective policy, keep active/default-selected plugin errors fatal, add no-allowlist coverage, and include patched CLI proof.

## Reproduction Assessment

Yes, source-level: the PR body gives a beta `openclaw config validate --json` repro with a broken local plugin plus unrelated `plugins.allow`, and current main pushes registry diagnostics before plugin config checks. I did not run the repro because this review is read-only.

## Solution Assessment

No, not as written. The intended allowlist-excluded plugin behavior is sound, but the implementation should use effective activation or exclusion rather than raw explicit references.
