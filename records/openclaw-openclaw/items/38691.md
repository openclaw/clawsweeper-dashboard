## Summary

Keep open. Current main still hard-stops the Android Voice-tab mic when `SpeechRecognizer.isRecognitionAvailable(context)` is false, so the reported OEM-device problem is not implemented on main, but this PR is not merge-ready because it removes the no-backend stop path and lacks real after-fix device proof.

## What This Changes

The PR changes Android Voice-tab mic startup to continue past false recognition-availability reports, read the configured OEM speech recognizer service, and target that recognizer before falling back to the default recognizer.

## Best Possible Solution

Land a current-path Android Voice-tab fallback that targets the configured OEM recognizer when present, preserves a clean disable path when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No high-confidence device reproduction is available from this review. Source inspection shows current main will disable the mic when `isRecognitionAvailable=false`, but the OEM-specific premise and after-fix result still need real affected-device proof.
