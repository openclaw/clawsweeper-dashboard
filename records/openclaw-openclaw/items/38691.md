## Summary

This PR addresses a real Android Voice-tab mic gap on current main, but it should not merge as-is because it removes the unavailable-recognizer stop path, the branch is dirty against the renamed package path, and the real-device proof is still only a text claim.

## What This Changes

The PR changes Android Voice-tab mic startup to continue when `SpeechRecognizer.isRecognitionAvailable(context)` is false, read the configured `voice_recognition_service`, and target that OEM recognizer before falling back to the default recognizer.

## Best Possible Solution

Land an updated current-path Android Voice-tab mic fallback that targets the configured OEM recognizer when present, preserves a clean disable path when no usable backend exists, and includes redacted affected-device proof.

No high-confidence runtime reproduction is available from this review. Source inspection shows current main disables the Voice-tab mic when `isRecognitionAvailable=false`, but the affected OEM-device behavior and after-fix result still need real-device proof.

## Solution Assessment
