## Summary

This PR should stay open, but it is not ready to merge: current main still hard-stops the reported Voice-tab path, while this branch can introduce a no-backend retry loop and still lacks inspectable real-device proof.

## What This Changes

The PR changes Android MicCaptureManager to continue after `isRecognitionAvailable()` is false, read `Settings.Secure.voice_recognition_service`, and create a targeted OEM `SpeechRecognizer` before default fallback.

## Best Possible Solution

Land an updated Android Voice-tab Mic fallback in the current package path that targets the configured OEM recognizer when available, preserves a clean disable path when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main disables Voice-tab Mic whenever `isRecognitionAvailable(context)` is false, but this review did not run on an affected Chinese OEM Android device.

## Solution Assessment

No, not as currently written. The targeted OEM recognizer lookup is a reasonable direction, but the patch needs a no-backend stop path, a current-main rebase, and real-device proof before it is the narrowest maintainable fix.
