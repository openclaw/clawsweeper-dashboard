## Summary

Keep open: the Android Voice-tab OEM recognizer bug is not implemented on current main or the latest release, but this PR still needs real-device proof, a no-backend stop path, and a current-path rebase before it is mergeable.

## What This Changes

The PR changes Android mic startup to continue after `isRecognitionAvailable=false`, target the configured secure-settings speech recognizer component, and fall back to the default recognizer.

## Best Possible Solution

Land a current-path Android Voice-tab fallback that targets a verified configured OEM recognizer when present, preserves a clean disable path when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No high-confidence affected-device reproduction is available from this review. Source inspection does show the current-main failure path when `isRecognitionAvailable(context)` returns false, but the Xiaomi/OEM premise and after-fix behavior still need real proof.

## Solution Assessment

No, not as written. The OEM recognizer lookup is a reasonable narrow direction, but the patch must preserve the no-backend stop path, verify the component path more defensively, move onto the current package path, and include real-device proof before merge.
