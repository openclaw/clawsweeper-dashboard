## Summary

Keep open. Current main and the latest release still disable Voice-tab Mic when recognition is unavailable, but this PR is not merge-ready because it lacks external real-device proof and removes the unsupported-device stop path, which can turn no-backend devices into a 600 ms retry loop.

## What This Changes

The branch updates Android Voice-tab Mic recognizer creation to continue after `isRecognitionAvailable=false`, look up the secure-setting voice recognition service, and target that component before falling back to the default recognizer.

## Best Possible Solution

Land an updated Android Voice-tab Mic fallback in the current package path that targets a configured OEM recognizer when available, disables cleanly when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No. Source inspection gives a clear current-main gate where `isRecognitionAvailable=false` disables Voice-tab Mic, but I did not run an affected Xiaomi/Huawei/OPPO/vivo device or inspect after-fix artifacts.

## Solution Assessment

No, not as currently written. The targeted OEM recognizer lookup is a reasonable direction, but the patch needs a no-backend stop path, a current-main update, and real-device proof before merge.
