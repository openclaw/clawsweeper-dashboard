## Summary

Keep open: the PR addresses a plausible Android Voice Tab bug, but the submitted patch still introduces an unsupported-device retry regression and lacks artifact-backed real behavior proof for an external contribution.

## What This Changes

The PR changes Android manual mic capture to continue after `isRecognitionAvailable=false`, select a configured secure-setting speech recognizer component, and fall back to the default recognizer.

## Best Possible Solution

Port the generic OEM recognizer fallback into the current manual mic path, only bypass the availability stop when a configured recognizer component is usable, preserve a one-shot unsupported-device disable path, and require redacted real-device proof before merge.

## Reproduction Assessment

Yes, source-reproducible. If Android reports recognition unavailable, current main deterministically disables the Voice Tab mic before trying any configured recognizer, and the PR body gives a concrete Xiaomi device where that platform signal is reportedly false despite an OEM recognizer.

## Solution Assessment

No, not as submitted. The targeted OEM recognizer direction is maintainable, but the patch should avoid falling through to repeated default-recognizer retries when no valid backend exists and should be ported to the current package path.
