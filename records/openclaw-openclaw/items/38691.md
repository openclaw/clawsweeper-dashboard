## Summary

The PR is a plausible Android compatibility fix, but it should stay open because the submitted patch still regresses unsupported devices and lacks inspectable real-device proof. Current main still has the reported hard stop, so this is not implemented or superseded.

## What This Changes

The PR changes Android manual mic capture to continue after `SpeechRecognizer.isRecognitionAvailable(context)` is false and try a secure-setting OEM speech recognizer component before falling back to the default recognizer.

## Best Possible Solution

Port the generic OEM recognizer fallback into the current Android manual mic path, bypass the availability stop only when a configured recognizer component is usable, preserve a one-shot unsupported-device disable path, and require redacted real-device proof before merge.

## Reproduction Assessment

Yes for the current-main hard-stop path: if Android reports speech recognition unavailable, `MicCaptureManager.start()` disables the Voice Tab mic before trying any configured recognizer. The specific Xiaomi/OEM behavior is not live-reproduced here and still needs inspectable device proof from the contributor.

## Solution Assessment

No, not as submitted. The component-targeted OEM recognizer direction is maintainable, but the patch should avoid falling through to repeated default-recognizer retries when no valid backend exists and must be ported to the current package path.
