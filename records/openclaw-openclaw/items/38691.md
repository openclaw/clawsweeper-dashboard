## Summary

Keep open: current main still has the reported Android Mic hard stop, and the PR's OEM recognizer direction is plausible, but the patch removes the clean no-backend disable path and still lacks real affected-device proof.

## What This Changes

The PR changes Android MicCaptureManager to continue after `isRecognitionAvailable()` is false, target the configured secure-setting speech recognizer service, and fall back to the default recognizer.

## Best Possible Solution

Land an updated Android Voice-tab Mic fallback in the current package path that targets a configured OEM recognizer when available, disables cleanly when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No. Source inspection gives a clear current-main gate where `isRecognitionAvailable=false` disables Voice-tab Mic, but I did not run an affected Xiaomi/Huawei/OPPO/vivo device or inspect after-fix artifacts.

## Solution Assessment

No, not as currently written. The targeted OEM recognizer lookup is a reasonable direction, but the patch needs a no-backend stop path, a current-main rebase, and real-device proof before merge.
