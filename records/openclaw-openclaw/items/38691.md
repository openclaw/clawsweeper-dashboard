## Summary

Keep open: the Android OEM speech-recognizer bug is still present on current main, but this PR needs changes before merge because it removes the unsupported-device stop without validating a usable recognizer backend.

## What This Changes

The PR changes Android manual mic capture to continue after `isRecognitionAvailable=false`, read the configured secure voice-recognition service, target that component, and fall back to the default recognizer.

## Best Possible Solution

Port the OEM recognizer fallback to the current manual mic path, verify the configured recognition service before use, add the Android package-visibility query, and preserve a one-shot unsupported-device disable path.

## Reproduction Assessment

Yes. Current-main source shows the Voice Tab mic button reaches `MicCaptureManager.start()`, which returns before recognizer creation when `isRecognitionAvailable(context)` is false; the PR body adds a concrete Xiaomi 14 Pro smoke case for the intended OEM path.

## Solution Assessment

No, not as submitted. The targeted OEM recognizer fallback is the right narrow direction, but it must validate that a usable backend exists and keep the no-backend stop instead of falling into the existing retry loop.
