## Summary

Keep open: current main still disables Voice-tab Mic when Android reports no recognizer, and the PR targets a plausible bug, but the branch lacks real affected-device proof and currently regresses the no-backend failure path.

## What This Changes

The branch changes Android Voice-tab Mic recognizer creation to continue past `isRecognitionAvailable=false`, read `voice_recognition_service`, target that OEM component, and fall back to the default recognizer.

## Best Possible Solution

Land an updated Android Voice-tab Mic fallback in the current package path that targets a configured OEM recognizer when available, disables cleanly when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No high-confidence runtime reproduction is available from this review. Source inspection clearly shows current main disables Voice-tab Mic when `isRecognitionAvailable=false`, but I did not inspect a live Xiaomi/Huawei/OPPO/vivo device or after-fix artifact.

## Solution Assessment

No, not as currently written. The OEM recognizer lookup is a reasonable direction, but the patch needs a no-backend stop path, a current-main path update, and real affected-device proof before merge.
