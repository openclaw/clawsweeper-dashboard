## Summary

Keep open: the OEM-recognizer fallback is a plausible fix for an existing Android Voice Tab compatibility bug, but this PR is not merge-ready because it removes the unsupported-device stop unconditionally and still lacks inspectable real-device proof.

## What This Changes

The PR changes Android manual mic capture to continue after speech recognition is reported unavailable, query the configured secure-setting recognizer component, and fall back to the default recognizer.

## Best Possible Solution

Port the generic OEM recognizer fallback into the current manual mic path, bypass the availability stop only when a configured component is usable, preserve a one-shot unsupported-device disable path, and require redacted real-device proof before merge.

## Reproduction Assessment

Yes for the source path, but not for full real-device behavior. Current main deterministically disables the manual mic when Android reports recognition unavailable, while the PR body provides the concrete Xiaomi device claim that still needs inspectable proof.

## Solution Assessment

No, not as submitted. The targeted OEM recognizer approach is maintainable, but the patch must avoid falling through to repeated default-recognizer retries when no backend exists and must be ported to the current package path.
