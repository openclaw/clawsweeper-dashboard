## Summary

Keep open. Current main still disables the Android Voice-tab mic when `SpeechRecognizer.isRecognitionAvailable(context)` is false, so this PR addresses an unfixed compatibility bug, but the patch should not merge as-is because it can retry indefinitely on devices with no usable recognizer backend and lacks required real-device proof.

## What This Changes

The PR changes Android mic capture to continue after `isRecognitionAvailable` is false, read `voice_recognition_service`, and create a targeted OEM `SpeechRecognizer` before default fallback.

## Best Possible Solution

Land an updated Android Voice-tab Mic fallback in the current package path that targets the configured OEM recognizer when available, preserves a clean disable path when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No. Source inspection shows current main disables Voice-tab Mic whenever `isRecognitionAvailable(context)` is false, but this review did not run on an affected Chinese OEM Android device, so there is no high-confidence live reproduction.

## Solution Assessment

No, not as currently written. The targeted OEM recognizer lookup is a reasonable direction, but the patch needs a no-backend stop path and real-device proof before it is the narrowest maintainable fix.
