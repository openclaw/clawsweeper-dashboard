## Summary

This PR still targets a real Android Voice Tab gap on current main, but it is not ready to merge: it removes the unsupported-device stop unconditionally, can fall through into the existing retry path when no usable recognizer exists, and still lacks inspectable real-device proof.

## What This Changes

The PR changes Android manual mic capture to continue after `isRecognitionAvailable=false`, query the configured secure-setting speech recognizer component, and fall back to the default recognizer.

## Best Possible Solution

Port the generic OEM recognizer fallback into the current manual mic path, only bypass the availability stop when a configured recognizer component is usable, preserve a one-shot unsupported-device disable path, and require redacted real-device proof before merge.

## Reproduction Assessment

Yes, source-reproducible for the code path: if Android reports recognition unavailable, current main disables the Voice Tab mic before trying any configured recognizer. The missing piece is inspectable real-device proof for the Xiaomi/OEM platform condition described in the PR body.

## Solution Assessment

No, not as submitted. The targeted OEM recognizer direction is maintainable, but the patch should avoid falling through to repeated default-recognizer retries when no valid backend exists and should be ported to the current package path.
