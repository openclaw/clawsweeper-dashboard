## Summary

Keep open: the PR addresses a plausible Android Voice Tab compatibility bug, but current main has not implemented the OEM recognizer fallback, and the submitted patch still regresses unsupported devices into the retry path while lacking inspectable real-device proof.

## What This Changes

The PR changes Android Voice Tab mic capture to bypass the `isRecognitionAvailable` hard stop, query the configured secure-setting speech recognizer component, and fall back to the default recognizer.

## Best Possible Solution

Port the generic OEM recognizer fallback into the current Voice Tab mic path, bypass the availability stop only when a configured component is usable, preserve a one-shot unsupported-device disable path, and require redacted real-device proof before merge.

## Reproduction Assessment

Yes for the source path, but not for full real-device behavior. Current main deterministically disables the manual mic when Android reports recognition unavailable, while the PR body provides only a textual Xiaomi device claim for the OEM-service part.

## Solution Assessment

No, not as submitted. The targeted OEM recognizer approach is maintainable, but the patch must avoid falling through to repeated default-recognizer retries when no backend exists and must be ported to the current package path.
