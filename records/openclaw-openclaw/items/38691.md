## Summary

Keep open: current main still hard-stops the Voice Tab mic when Android reports no speech recognizer, and this PR is directionally useful but unsafe as submitted because it removes that guard without a verified no-backend disable path.

## What This Changes

The PR changes Android manual mic recognizer creation to continue after `isRecognitionAvailable=false`, read `Settings.Secure` `voice_recognition_service`, target that component, and fall back to the default recognizer.

## Best Possible Solution

Port the OEM recognizer fallback to the current manual mic path, verify any configured `RecognitionService`, add the Android package-visibility query, and keep a one-shot unsupported-device disable path.

## Reproduction Assessment

Yes. The current-main source path shows the Voice Tab manual mic returns before recognizer creation when `isRecognitionAvailable(context)` is false, and the PR body adds a concrete Xiaomi 14 Pro smoke case for the intended OEM path.

## Solution Assessment

No, not as submitted. The targeted OEM recognizer direction is narrow and maintainable, but the patch must validate the configured service and preserve a clean no-backend stop before the early guard can be removed.
