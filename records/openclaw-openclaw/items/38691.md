## Summary

The PR is a plausible fix for a real Android Voice-tab gap, but it should stay open because the patch removes the existing unavailable-recognizer stop path and can retry indefinitely when no usable speech backend exists; it also still lacks contributor-supplied after-fix real-device proof.

## What This Changes

The branch changes Android mic capture to continue after `SpeechRecognizer.isRecognitionAvailable(context)` is false, read `voice_recognition_service`, and create a targeted OEM `SpeechRecognizer` before falling back to the default recognizer.

## Best Possible Solution

Land an updated Android voice fallback in the current `ai.openclaw.app` package that targets the configured OEM recognizer when available, preserves a clean disable path when no usable backend exists, and includes redacted real-device proof from an affected OEM device.

## Reproduction Assessment

No. Source inspection confirms current main disables the Voice-tab mic whenever `isRecognitionAvailable(context)` is false, but this review did not exercise an affected Chinese OEM Android device, so there is no high-confidence live reproduction.

## Solution Assessment

No, not as currently written. The OEM service lookup and targeted `createSpeechRecognizer` direction is reasonable, but the patch must keep a no-backend disable path and be rebased onto the current Android package path before merge.
