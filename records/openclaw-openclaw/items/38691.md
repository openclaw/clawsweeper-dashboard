## Summary

Keep open: current main and the latest release still have the Android Voice-tab hard stop this PR targets, and no merged OpenClaw item supersedes it, but the branch is not merge-ready because it removes the unsupported-device stop path, does not validate the targeted OEM service, and still lacks inspectable real-device proof.

## What This Changes

The PR changes Android mic startup to continue after `SpeechRecognizer.isRecognitionAvailable(context)` returns false and try a configured OEM speech recognizer before falling back to the default recognizer.

## Best Possible Solution

Land a current-path Android Voice-tab fallback that verifies a configured OEM `RecognitionService`, preserves clean disable behavior when no usable recognizer exists, and includes redacted affected-device proof.

## Reproduction Assessment

No high-confidence affected-device reproduction is available. Source inspection proves current main disables the Voice-tab mic when `SpeechRecognizer.isRecognitionAvailable(context)` returns false, but the OEM-device premise still needs inspectable Xiaomi/OEM proof.

## Solution Assessment

No, not as written. The fallback direction is narrow and plausible, but it should validate the OEM component through Android's recognition-service contract and keep a no-backend disable path instead of falling through to repeated retries.
