## Summary

Keep open: current main still has the Android Voice-tab Mic hard stop, so the PR is not obsolete, but the branch should not merge as-is because it removes the no-recognizer stop path and still lacks after-fix real-device proof.

## What This Changes

The PR changes Android MicCaptureManager to continue after `isRecognitionAvailable()` is false and target the configured secure-setting speech recognition service before falling back to the default recognizer.

## Best Possible Solution

Land an updated Voice-tab Mic fallback in the current Android package path that targets the configured OEM recognizer when available, preserves a clean disable path when no usable backend exists, and includes redacted affected-device proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main disables Voice-tab Mic whenever `isRecognitionAvailable(context)` is false, but this review did not run on an affected Chinese OEM Android device or inspect contributor-provided artifacts.

## Solution Assessment

No, not as currently written. The targeted OEM recognizer lookup is a reasonable fix direction, but the patch needs a no-backend stop path, a current-main rebase, and real-device proof before it is the narrowest maintainable solution.
