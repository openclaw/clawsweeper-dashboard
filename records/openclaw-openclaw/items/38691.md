## Summary

Keep this PR open: the reported Android Voice-tab failure is plausible and not implemented on current main, but the branch is conflicting, lacks inspectable real-device proof, and the proposed fallback has correctness gaps around unsupported devices and service validation.

## What This Changes

The PR changes Android Voice-tab speech recognizer startup to continue after `isRecognitionAvailable=false`, query `Settings.Secure.voice_recognition_service`, and target that OEM recognizer before falling back to the default recognizer.

## Best Possible Solution

Land a refreshed Android Voice-tab fallback in the current `ai.openclaw.app` path that verifies the configured `RecognitionService`, preserves a clean disable path when no usable recognizer exists, and includes redacted affected-device proof.

## Reproduction Assessment

No high-confidence affected-device reproduction is available. Source inspection shows current main disables the Voice-tab mic when Android reports no recognizer, but the Xiaomi/OEM false-negative and after-fix transcription result still need inspectable device proof.

## Solution Assessment

No, not as written. The fallback direction is narrow and plausible, but the patch should be rebased onto the current Android package path, validate the OEM component through the Android recognition-service contract, and keep a no-backend stop path instead of retrying indefinitely.

Full review comments:

- **[P2] Preserve the no-backend stop path:** `apps/android/app/src/main/java/ai/openclaw/android/voice/MicCaptureManager.kt:186-190`
  - body: After this block stops returning on `isRecognitionAvailable=false`, devices without a valid OEM recognizer fall through to the default recognizer even though Android documents that default creation fails after that check returns false.   - confidence: 0.88

Android's contract requires checking with `PackageManager.queryIntentServices` that the component exists and provides `RecognitionService.SERVICE_INTERFACE`; without that, stale or non-recognition settings can hit the same failure/retry path.
  - confidence: 0.84
