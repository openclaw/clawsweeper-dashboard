## Summary

Keep open: the reported Android Voice Tab bug is plausible and current main still hard-stops when `isRecognitionAvailable` is false, but this PR is not merge-ready because it can regress unsupported devices into a retry loop and lacks observable after-fix real behavior proof.

## What This Changes

The PR removes the Android Voice Tab hard stop when `SpeechRecognizer.isRecognitionAvailable` is false and adds a `Settings.Secure` `voice_recognition_service` fallback for creating an OEM recognizer.

## Best Possible Solution

Rework the OEM recognizer fallback onto the current Android voice code, preserve a clean disabled state when no valid recognizer backend exists, and require redacted real-device proof from an affected OEM device before merge.

## Reproduction Assessment

No high-confidence live reproduction is available in this review. Source inspection shows current main hard-stops on `isRecognitionAvailable=false`, and the author reports a Xiaomi device fix, but there is no attached device log, screenshot, recording, or artifact.

## Solution Assessment

No, not as submitted. The fallback direction is reasonable, but the patch should target current main and add a no-backend safety valve before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect
