## Summary

Keep open: current main still hard-stops the Android Voice Tab mic when `isRecognitionAvailable(context)` is false, but the submitted patch is not merge-safe because it removes the unsupported-device stop without preserving a no-backend disable path.

## What This Changes

The PR changes Android manual mic recognizer startup to continue after `isRecognitionAvailable=false`, read the secure voice recognition service setting, target that component, and fall back to the default `SpeechRecognizer`.

## Best Possible Solution

Port the OEM recognizer fallback to the current manual mic path, validate the configured `RecognitionService` with Android package visibility in place, and preserve a one-shot disable path when no backend exists.

## Reproduction Assessment

Yes. Source inspection shows the Voice Tab mic path returns before recognizer creation when `isRecognitionAvailable(context)` is false, and the PR body provides a concrete Xiaomi smoke case for the intended OEM path.

## Solution Assessment

No, not as submitted. The targeted OEM recognizer direction is maintainable, but the patch must validate that a backend exists and keep unsupported devices from entering the existing 600 ms retry loop.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
