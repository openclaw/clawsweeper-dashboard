## Summary

## What This Changes

Request to prevent or clearly repair direct package-manager self-updates that bypass OpenClaw update finalization and leave managed Codex/plugin state version-skewed.

## Best Possible Solution

Keep the coordinated updater/finalize path and add a maintainer-approved direct package-manager upgrade guard, explicit finalization prompt, or startup/readiness diagnostic that prevents or clearly repairs core/plugin/Codex skew without surprising package lifecycle side effects.

No high-confidence live reproduction was run. Source inspection shows direct package-manager installs do not invoke the OpenClaw update orchestrator, while `openclaw update` and hidden `openclaw update finalize` own doctor repair and plugin convergence.

## Solution Assessment

Unclear as an implementation path: current main uses the right repair layer for first-class updates, but closing the direct package-manager gap requires maintainers to choose between a guard, finalization hook, or readiness diagnostic.
