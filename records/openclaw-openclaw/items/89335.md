## Summary

Keep open: current main has the supported updater/finalize and doctor repair path, but direct global package-manager self-updates can still bypass that orchestration and need a maintainer product decision before changing package lifecycle, agent prompt, or startup diagnostics.

## What This Changes

The issue asks OpenClaw to prevent, warn on, finalize, or better diagnose direct package-manager self-updates that can leave core, managed plugins, Codex routing, and session metadata version-skewed.

## Best Possible Solution

Keep the coordinated openclaw update/update.run path as the source of truth, then add a maintainer-approved direct-update guard, explicit finalization prompt, or readiness diagnostic with live upgrade proof for both supported and manual paths.

## Reproduction Assessment

No. There is strong source and external incident evidence for the direct package-manager bypass, but this read-only review did not run a live current-main upgrade reproduction.

## Solution Assessment

Unclear as an implementation path. The supported updater/finalize layer is the right owner, but maintainers still need to choose between warning, blocking, surfacing finalize, or startup/readiness diagnosis for direct package-manager updates.
