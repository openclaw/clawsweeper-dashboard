## Summary

Keep open. Current main still lacks the Feishu Bitable toggle, and the PR follows the right Feishu-owned pattern, but it is not merge-ready because the new required config key leaves an existing typed mock incomplete and the user-facing config surface needs docs/changelog follow-through.

## What This Changes

The PR adds a Feishu `tools.bitable` config key to type/schema/default/merge logic and skips Bitable tool registration when merged enabled-account config disables it.

## Best Possible Solution

A Feishu-plugin-owned opt-out should default Bitable on, skip registration only when merged enabled accounts disable it, and ship with typed mocks/tests plus docs/changelog.

## Solution Assessment

Mostly yes, but incomplete. Mirroring the existing Feishu tool-category toggle pattern inside the plugin is maintainable, but the PR should update typed mocks, focused tests, public docs, and the changelog before merge.
