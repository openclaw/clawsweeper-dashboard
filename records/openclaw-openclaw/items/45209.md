## Summary

Keep open. Current main and the latest release still lack a Feishu Bitable tool-category toggle, so the PR is addressing a real gap, but the proposed patch is not merge-ready because adding `bitable` to `FeishuToolsConfig` makes an existing typed test mock incomplete.

## What This Changes

The PR adds a Feishu `tools.bitable` boolean to the plugin config type, schema, defaults, merged account tools config, and Bitable registration guard so `feishu_bitable_*` tools remain enabled by default but can be disabled.

## Best Possible Solution

Land or replace this with a Feishu-plugin-owned opt-out toggle that defaults Bitable tools on, skips Bitable registration only when all enabled Feishu accounts disable it, and includes focused schema/default/registration tests, updated typed mocks, public Feishu docs, and a credited changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `channels.feishu.tools.bitable: false` on current main and Bitable tools will still register because `registerFeishuBitableTools` has no merged tools-config guard; on the PR branch, typechecking should also catch the incomplete `docx.test.ts` mock after `Required<FeishuToolsConfig>` gains `bitable`.

## Solution Assessment

Mostly yes, but not complete. Mirroring the existing Feishu tool-category toggle pattern inside the Feishu plugin is the narrowest maintainable direction; the safer merge path is to keep that scope and repair the typed mock plus tests/docs/changelog before landing.
