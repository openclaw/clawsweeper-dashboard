## Summary

Keep this PR open: current main still lacks the Feishu Bitable toggle and the patch follows the existing tool-toggle pattern, but it needs a small repair before merge because the new `Required<FeishuToolsConfig>` member will break an existing typed test mock and the user-facing config option lacks docs/changelog coverage.

## What This Changes

The PR adds an opt-out `tools.bitable` flag for Feishu/Lark accounts and skips registering `feishu_bitable_*` tools when every enabled account disables it.

## Best Possible Solution

Land the Feishu-owned config toggle after adding regression coverage, updating the typed mocks, and documenting/releasing the new user-facing option.
