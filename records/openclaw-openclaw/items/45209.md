## Summary

Keep open: current main still lacks the Feishu Bitable tools toggle, and this PR is the canonical focused implementation, but it needs a small type/test fix plus real behavior proof before merge.

## What This Changes

The PR adds a `tools.bitable` Feishu plugin config flag, defaults it on, and skips registering `feishu_bitable_*` tools when all enabled accounts disable it.

## Best Possible Solution

Land the existing config-toggle design after updating typed fixtures/coverage and adding proof that `tools.bitable: false` removes Bitable tools while default configs preserve current behavior.

## Reproduction Assessment

Yes for the source-level gap: current main has no `bitable` schema/type/default field and Bitable tool registration has no tools-config guard. I did not run a live Feishu setup in this read-only review.

## Solution Assessment

Mostly yes: extending the existing Feishu `tools` category config and guarding `registerFeishuBitableTools` is the narrowest maintainable direction. The branch should also update affected typed test fixtures and add regression coverage for disabled Bitable registration.
