## Summary

Keep open. Current main still lacks a Feishu `tools.bitable` toggle and still registers Bitable tools whenever enabled Feishu accounts exist; the PR is directionally right but not merge-ready because it omits the typed mock/test/docs/changelog follow-through already called out in the prior ClawSweeper review.

## What This Changes

The PR adds a Feishu plugin `tools.bitable` boolean to config types/schema/default merging and skips Bitable tool registration when merged enabled-account config disables it.

## Best Possible Solution

Land or replace this with a Feishu-plugin-owned opt-out toggle that defaults Bitable tools on, skips Bitable registration only when all enabled Feishu accounts disable it, and includes focused schema/default/registration tests, updated typed mocks, public Feishu docs, and a credited changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, configuring `channels.feishu.tools.bitable: false` cannot affect Bitable registration because the config key and registration guard do not exist.

## Solution Assessment

Mostly yes, but this PR is incomplete. Mirroring the existing Feishu tool-category toggle pattern inside the Feishu plugin is the narrowest maintainable fix, but the merge path should include the missing typed mock/test/docs/changelog follow-through.
