## What This Changes

The PR removes `--skip-trust` from the Google Gemini CLI backend's fresh and resume argument defaults.

## Best Possible Solution

Land a repaired version that drops the incompatible argv for affected Gemini CLI versions, preserves headless workspace trust through `GEMINI_CLI_TRUST_WORKSPACE=true`, and aligns tests, docs, and changelog.

Yes. Source inspection shows current main forwards `--skip-trust` through the Gemini backend argv, and the linked bug report provides the 0.38.1/0.38.2 unknown-argument failure; I did not run a live Gemini CLI binary here.

## Solution Assessment

No, not as submitted. Removing the incompatible argv is the right compatibility direction, but the narrow maintainable fix also needs the documented Gemini trust env default, updated backend contract tests, and a changelog entry.
