## Summary

Keep open. The PR targets a real Gemini CLI compatibility bug, but the submitted patch is incomplete: it removes the rejected flag without preserving the current trust-bypass behavior for newer Gemini CLI versions, and it leaves the backend defaults test and changelog stale.

## What This Changes

The PR removes `--skip-trust` from the bundled Google Gemini CLI backend's fresh and resume argument arrays.

## Best Possible Solution

Land a scoped Google plugin fix that removes the unsupported CLI argument, preserves the existing trust-bypass semantics through the version-compatible environment variable, updates the backend contract tests, and adds the required changelog credit so the linked bug can close on merge.

## Reproduction Assessment

Yes. Current main passes `--skip-trust` through to `gemini`, and upstream Gemini CLI v0.38.1/v0.38.2 has strict yargs parsing with no `skip-trust` option; the submitted patch also has a predictable local test-contract failure because the existing assertions still expect the removed flag.

## Solution Assessment

No, not as submitted. Removing the argv flag is the right compatibility direction for 0.38.x, but the narrow maintainable solution should also set `GEMINI_CLI_TRUST_WORKSPACE=true` for newer Gemini CLI trust behavior, update tests, and add the changelog entry.
