## Summary

The PR addresses a real version-specific Gemini CLI argv failure, but the branch is not merge-ready because it removes only the production flag, leaves backend-default tests stale, lacks after-fix real behavior proof, and may drop the current Gemini CLI workspace-trust mechanism without a compatible replacement.

## What This Changes

The PR removes `--skip-trust` from both fresh and resume `google-gemini-cli` backend argument defaults in the bundled Google plugin.

## Best Possible Solution

Yes. Current main builds and forwards `--skip-trust`, and Gemini CLI v0.38.2 source has strict options without that flag, so the reported failure is source-reproducible for that dependency version; I did not run a live OpenClaw turn.

## Solution Assessment

No, not as submitted. Removing the unsupported argv is directionally useful for v0.38.2, but the branch should preserve current workspace-trust semantics, update the checked defaults, and provide real after-fix proof.
