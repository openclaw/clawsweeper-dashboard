## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Block plaintext rewrites of sealed stores:** `src/agents/auth-profiles/store.ts:435`
  - body: When a sealed auth store is loaded without OPENCLAW_PASSPHRASE, this path can still reach a migration or external-CLI sync write and save the replacement store with no passphrase. That silently converts encrypted credentials back to plaintext and can lose the original sealed data, contradicting the fail-closed claim.
  - confidence: 0.95
- **[P1] Drop stale gateway timeout semantics before rebasing:** `src/gateway/client.ts:517-523`
  - body: Current main already implements requestTimeoutMs with AbortSignal support and leaves expectFinal requests unbounded unless a caller passes timeoutMs. This PR defaults every request, including long-running expectFinal flows, to a 30s timeout, so carrying this slice forward would create a competing source of truth and can break long-running gateway calls.
  - confidence: 0.9
- **[P2] Use the sealed reader for inherited main stores:** `src/agents/auth-profiles/store.ts:413`
  - body: The subagent inheritance path still reads the main auth store as plain JSON before writing it locally. Once the main store is sealed, that read fails and subagents silently stop inheriting credentials, so this path needs to use the same sealed-aware reader as the other auth-store loads.
  - confidence: 0.9
- **[P2] Persist KDF metadata with sealed files:** `src/infra/sealed-json-file.ts:38`
  - body: The envelope records the cipher fields but not the scrypt parameters used to derive the key. That makes the security posture hard to audit and creates an upgrade hazard if the cost parameters need to change later.
  - confidence: 0.84
- **[P2] Write sealed stores atomically with restrictive mode:** `src/infra/sealed-json-file.ts:113-114`
  - body: This writes the sensitive payload and only then chmods it, instead of creating an atomic replacement with mode 0600. Use the existing atomic file-writing pattern or an equivalent restrictive write so new credential files never have a wider-permission window.
  - confidence: 0.82
