Full review comments:

- **[P1] Block plaintext rewrites of sealed auth stores:** `src/agents/auth-profiles/store.ts:435`
  - body: When a sealed `auth-profiles.json` cannot be read because `OPENCLAW_PASSPHRASE` is missing, this path can still decide to persist after legacy/OAuth/CLI sync work. `saveSealedJsonFile` then writes plaintext, so a protected store can be downgraded or partially replaced instead of failing closed.
  - confidence: 0.94
- **[P2] Use the sealed reader for inherited main stores:** `src/agents/auth-profiles/store.ts:412`
  - body: The branch makes auth stores sealable, but the subagent inheritance read still uses the raw JSON reader for the main store. A sealed main store will not parse as JSON there, so subagents can silently miss inherited credentials whenever `OPENCLAW_PASSPHRASE` is enabled.
  - confidence: 0.89
- **[P2] Persist explicit KDF metadata in sealed files:** `src/infra/sealed-json-file.ts:39`
  - body: The new envelope derives keys with `scryptSync(passphrase, salt, 32)` but does not store the scrypt parameters. That makes the security posture hard to audit and makes future tuning or compatibility changes risky for already-written credential files.
  - confidence: 0.84
- **[P1] Drop the stale gateway timeout implementation:** `src/gateway/client.ts:517`
  - body: Current main already implements gateway request timeouts with abort-signal support and leaves `expectFinal` requests unbounded by default. Keeping this older timeout slice would duplicate the supported path and risks regressing long-running accepted/final flows during conflict resolution.
  - confidence: 0.86

## Security Review
