## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Target the current SQLite auth store:** `src/agents/auth-profiles/paths.ts:32`
  - body: Current main stores auth profiles in `openclaw-agent.sqlite`, and docs now treat `auth-profiles.json` as legacy doctor-import input. Sealing the old JSON path would not protect the credential store current OpenClaw actually reads and writes, and it would leave users with a competing secret-storage model.
  - confidence: 0.93
- **[P1] Prevent sealed-store plaintext downgrades:** `src/agents/auth-profiles/store.ts:438`
  - body: When a sealed store exists but `OPENCLAW_PASSPHRASE` is missing, `loadProtectedAuthJson` returns `undefined`; this path can then decide `shouldWrite` and call `saveSealedJsonFile` without a passphrase, replacing encrypted credentials with plaintext. That breaks the PR's fail-closed security claim.
  - confidence: 0.95
- **[P1] Read sealed main auth during inheritance:** `src/agents/auth-profiles/store.ts:412`
  - body: This fallback still reads the main auth store with `loadJsonFile`, which cannot parse the sealed prefix written by this PR. With `OPENCLAW_PASSPHRASE` enabled, subagents will silently skip inherited main credentials.
  - confidence: 0.9
- **[P2] Preserve expectFinal timeout semantics:** `src/gateway/client.ts:520-524`
  - body: The PR computes a default timeout even for `expectFinal` requests, so long-running final-response calls would start failing after 30s. Current main's merged timeout implementation intentionally leaves `expectFinal` unbounded unless a caller opts into `timeoutMs`.
  - confidence: 0.88
- **[P2] Write sealed credentials atomically and privately:** `src/infra/sealed-json-file.ts:113-114`
  - body: `saveSealedJsonFile` writes directly to the destination and only applies `0600` afterward, which creates a weaker permission window and no durable atomic replace for credential material. Reuse or extend the existing file-safe atomic write pattern before this becomes a security boundary.
  - confidence: 0.86
- **[P2] Store explicit KDF parameters in the envelope:** `src/infra/sealed-json-file.ts:39`
  - body: The sealed format derives the key with implicit `scryptSync` defaults and records no KDF parameters, so the offline attack cost and future decryptability are not part of the file-format contract.
  - confidence: 0.82
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:750`
  - body: OpenClaw release generation owns `CHANGELOG.md`; this normal PR should keep release-note context in the PR body or squash message instead of editing the changelog directly.
  - confidence: 0.8
