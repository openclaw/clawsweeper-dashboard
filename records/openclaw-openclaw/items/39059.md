## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Target the current auth-profile store:** `src/agents/auth-profiles/paths.ts:32`
  - body: Current main persists auth-profile secrets through the SQLite `auth_profile_store` row, but this PR seals the old `auth-profiles.json` file path. That can leave active provider credentials unsealed while introducing a competing stale storage path, so the credential hardening needs to operate at the current persistence boundary.
  - confidence: 0.9
- **[P1] Prevent sealed stores from downgrading to plaintext:** `src/agents/auth-profiles/store.ts:438`
  - body: When a sealed store is loaded without `OPENCLAW_PASSPHRASE`, this path treats it as missing and later `saveSealedJsonFile` can write the replacement store without a passphrase. That silently converts encrypted credentials back to plaintext instead of failing closed.
  - confidence: 0.95
- **[P1] Read inherited main stores through the sealed reader:** `src/agents/auth-profiles/store.ts:412`
  - body: The subagent inheritance path still reads the main auth store through the raw JSON reader, so a main store written in the new sealed format is parsed as invalid JSON and subagents stop inheriting credentials when passphrase sealing is enabled.
  - confidence: 0.9
- **[P2] Write sealed credential files atomically with private mode:** `src/infra/sealed-json-file.ts:113-114`
  - body: `saveSealedJsonFile` writes or truncates the credential file directly and only then applies `chmod(0o600)`. Credential material should use the existing atomic/private replace pattern so there is no permissions window, partial-write loss, or symlink-following clobber path.
  - confidence: 0.86
- **[P2] Record explicit KDF parameters in the envelope:** `src/infra/sealed-json-file.ts:38-39`
  - body: The sealed file format derives keys with implicit `scryptSync` defaults and does not store KDF metadata. That makes the encrypted credential format harder to audit and upgrade safely across releases.
  - confidence: 0.82
