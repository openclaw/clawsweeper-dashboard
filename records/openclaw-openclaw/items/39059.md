## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve unbounded expectFinal requests by default:** `src/gateway/client.ts:520-524`
  - body: This timeout calculation applies the 30s default to every request, including `expectFinal` flows. Current main deliberately treats `expectFinal` as unbounded unless a caller passes `timeoutMs`, because agent/session RPCs can legitimately run longer than 30 seconds.
  - confidence: 0.9
- **[P1] Prevent plaintext overwrite when the sealed store is locked:** `src/agents/auth-profiles/store.ts:436-438`
  - body: When a sealed store is loaded without `OPENCLAW_PASSPHRASE`, `loadCoercedStore` returns no store and this migration/import path can still call `saveSealedJsonFile`; without a passphrase that helper writes plaintext over the sealed auth store.
  - confidence: 0.94
- **[P2] Read sealed main stores for subagent inheritance:** `src/agents/auth-profiles/store.ts:412`
  - body: Subagent inheritance still reads the main store with the plaintext JSON reader, so a sealed main `auth-profiles.json` is treated as invalid and credentials are not inherited when passphrase sealing is enabled.
  - confidence: 0.87
- **[P2] Write sealed auth stores with safe replace semantics:** `src/infra/sealed-json-file.ts:113-114`
  - body: This writes credential-bearing content directly to the destination and only chmods afterward. Secret stores should be created with restrictive mode up front and replaced atomically so partial writes, permission windows, and symlink-style clobbering are not introduced with the encryption layer.
  - confidence: 0.82
