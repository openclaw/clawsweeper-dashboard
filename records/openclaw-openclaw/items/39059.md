Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Target credential sealing at the active SQLite store:** `src/agents/auth-profiles/paths.ts:29-32`
  - body: Current main reads and writes auth profiles from `openclaw-agent.sqlite`, but this PR seals `auth-profiles.json`/`oauth.json` instead. That would leave the active credential store unencrypted while giving operators the impression credential-at-rest sealing is complete.
  - confidence: 0.95
- **[P1] Block plaintext downgrades when the passphrase is absent:** `src/agents/auth-profiles/store.ts:435-439`
  - body: If a sealed store cannot be read because `OPENCLAW_PASSPHRASE` is missing, this write path can still persist migration or external-CLI data with no passphrase, replacing encrypted credentials with plaintext. The save path needs to fail closed or become read-only when an existing sealed file cannot be decrypted.
  - confidence: 0.94
- **[P1] Use the sealed reader for inherited main auth:** `src/agents/auth-profiles/store.ts:412-416`
  - body: The inherited main-store path still reads the main auth store through the plain JSON reader, so a sealed main store cannot be inherited by subagents. Use the same protected/sealed reader here or the new passphrase mode breaks credential inheritance.
  - confidence: 0.88
- **[P2] Preserve expectFinal request timeout semantics:** `src/gateway/client.ts:517-524`
  - body: This branch applies a default timeout to every request unless a numeric override is supplied, but current main deliberately leaves `expectFinal` requests without an automatic timeout by default and supports `timeoutMs: null`. Re-porting the stale implementation would break long-running accepted/final gateway flows.
  - confidence: 0.86
- **[P2] Write sealed credentials atomically and privately:** `src/infra/sealed-json-file.ts:113-114`
  - body: `saveSealedJsonFile` writes credential material with `writeFileSync` and only then chmods the destination. Credential writes should use the existing atomic/private-at-create-time replacement path so crashes, permission windows, symlink edges, or chmod failures cannot expose or corrupt auth material.
  - confidence: 0.89
- **[P2] Record explicit KDF parameters in sealed files:** `src/infra/sealed-json-file.ts:39`
  - body: The sealed envelope derives keys with `scryptSync` but does not store explicit KDF parameters. Persisted credential formats need stable, auditable work factors so future tuning does not create ambiguous or incompatible encrypted files.
  - confidence: 0.82
