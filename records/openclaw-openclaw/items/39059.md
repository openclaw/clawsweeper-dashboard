## Summary

Keep open. This maintainer-labeled PR has useful security intent, and current main already ships the gateway timeout portion, but the auth-store sealing branch is not merge-ready because it can downgrade sealed credentials to plaintext and bypasses the safer current auth persistence boundary.

Overall confidence: 0.9

Full review comments:

- **[P1] Prevent sealed stores from downgrading to plaintext:** `src/agents/auth-profiles/store.ts:438`
  - body: When a sealed `auth-profiles.json` cannot be read because `OPENCLAW_PASSPHRASE` is missing, this path can still set `shouldWrite` from legacy/OAuth/CLI sync and persist through `saveSealedJsonFile` without a passphrase. That replaces sealed credentials with plaintext or a partial store instead of failing closed.
  - confidence: 0.93
- **[P2] Read sealed main auth during subagent inheritance:** `src/agents/auth-profiles/store.ts:412`
  - body: The subagent inheritance fallback still uses `loadJsonFile(mainAuthPath)`, but this PR can write the main store in sealed form. With `OPENCLAW_PASSPHRASE` set, subagents skip inherited credentials because the sealed prefix is not JSON; use the sealed-aware reader on this path too.
  - confidence: 0.88
- **[P1] Use a safe atomic writer for sealed auth files:** `src/infra/sealed-json-file.ts:113-114`
  - body: `saveSealedJsonFile` writes credential material with `writeFileSync` and only then applies `chmod(0o600)`, bypassing the existing safer JSON/fs-safe write path. For auth stores, create the file with restrictive mode and atomic replace semantics so new sealing does not weaken file-permission or clobber behavior.
  - confidence: 0.86
- **[P2] Store explicit scrypt parameters in the envelope:** `src/infra/sealed-json-file.ts:39`
  - body: The sealed format derives keys with default `scryptSync` parameters but does not record the KDF settings in the file. That makes future tuning or compatibility ambiguous for a new persisted auth format; write and validate explicit KDF metadata with the envelope before shipping it.
  - confidence: 0.78

## Security Review
