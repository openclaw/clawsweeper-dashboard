  - body: Current main persists auth profiles in `openclaw-agent.sqlite` and tests that `auth-profiles.json` is not created, but this PR seals `auth-profiles.json`; that would add a competing stale credential path while leaving the active store unhardened.

  - confidence: 0.95

  - body: Subagent inheritance still reads the main store with `loadJsonFile`, which cannot parse the sealed format this PR writes; subagents will silently miss inherited credentials whenever passphrase sealing is enabled.
  - confidence: 0.88
- **[P1] Preserve expectFinal timeout semantics:** `src/gateway/client.ts:520-524`
  - body: This default timeout calculation applies to `expectFinal` requests, while current main keeps those long-running flows unbounded unless a caller passes `timeoutMs`; merging this shape would break final-response agent flows.
  - confidence: 0.9
- **[P2] Write sealed credentials atomically and privately:** `src/infra/sealed-json-file.ts:113-114`
  - body: `writeFileSync` followed by `chmodSync` creates a permissions and durability window for credential material; the sealed store writer needs a private-at-create-time temp/rename path before it is safe for auth storage.
  - confidence: 0.86
- **[P2] Store KDF parameters in the sealed envelope:** `src/infra/sealed-json-file.ts:39`
  - body: The sealed format derives keys with implicit `scryptSync` parameters and stores no KDF metadata, so auditability and future decryptability are not part of the file-format contract.
  - confidence: 0.82
