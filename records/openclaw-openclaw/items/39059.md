Full review comments:

- **[P1] Prevent sealed-store plaintext downgrade:** `src/agents/auth-profiles/store.ts:437-438`
  - body: When a sealed `auth-profiles.json` cannot be read because the passphrase is missing, this path can still reach `saveSealedJsonFile` through legacy/OAuth/CLI sync and overwrite the sealed file as plaintext. Treat the unreadable sealed store as a write blocker instead of as an empty store.
  - confidence: 0.91
- **[P1] Read inherited stores through the sealed reader:** `src/agents/auth-profiles/store.ts:416`
  - body: This branch can write the main auth store in sealed form, but the subagent inheritance path still reads the main store with `loadJsonFile`; sealed content will not parse, so subagents silently lose inherited credentials when the passphrase mode is enabled. Use the same protected reader for this path.
  - confidence: 0.88
- **[P2] Write sealed stores atomically with secure mode:** `src/infra/sealed-json-file.ts:113-114`
  - body: `saveSealedJsonFile` writes secret-bearing auth data directly to the destination and only then applies `0600`, unlike the existing atomic credential-write pattern. Use an atomic temp-and-rename helper that creates the temp file with restrictive permissions before writing credentials.
  - confidence: 0.82
