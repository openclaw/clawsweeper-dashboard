  - body: This PR seals `auth-profiles.json`, but current main persists runtime auth profiles in `openclaw-agent.sqlite`; merging this would harden an obsolete JSON path while leaving the actual credential store unchanged and creating a competing persistence model.
  - confidence: 0.91

  - body: When a sealed auth store cannot be read because `OPENCLAW_PASSPHRASE` is missing, this path can still decide to write and `saveSealedJsonFile` falls back to plaintext, replacing encrypted credentials with unsealed JSON instead of failing closed.
  - confidence: 0.95
- **[P1] Preserve current expectFinal timeout semantics:** `src/gateway/client.ts:517-525`
  - body: The PR applies the default request timeout even when `expectFinal` is true, but current main keeps those long-running flows unbounded unless a caller passes `timeoutMs`; merging this stale shape can break agent calls that intentionally wait for a final response.
  - confidence: 0.88
- **[P2] Write sealed credential files atomically and privately:** `src/infra/sealed-json-file.ts:113-114`
  - body: `writeFileSync` followed by `chmodSync` creates a permissions and durability window for credential material; this needs a temp-file/rename or equivalent private-at-create-time writer before it is safe for auth stores.
  - confidence: 0.86
- **[P2] Record explicit KDF parameters in the envelope:** `src/infra/sealed-json-file.ts:39`
  - body: The sealed format derives keys with implicit `scryptSync` defaults and stores no KDF metadata, making the security posture hard to audit and future parameter changes incompatible with existing sealed files.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a credential-sealing primitive with concrete downgrade, KDF, and filesystem-write concerns that need security design work before merge.

Concerns:

- **[high] Sealed credentials can downgrade to plaintext:** `src/agents/auth-profiles/store.ts:435`
  - body: A sealed auth store read without `OPENCLAW_PASSPHRASE` can be treated as missing and later overwritten through `saveSealedJsonFile` as plaintext, undermining the fail-closed security claim.
  - confidence: 0.95
- **[medium] KDF parameters are implicit and not stored:** `src/infra/sealed-json-file.ts:39`
  - body: The envelope omits explicit `scrypt` work factors, so offline attack cost and future decryptability are not a stable file-format contract.
  - confidence: 0.82
- **[medium] Credential writes are non-atomic and chmod after creation:** `src/infra/sealed-json-file.ts:113`
  - body: Writing the target file directly and applying `0600` after the write can expose plaintext or sealed credential bytes during creation and leaves weaker crash/TOCTOU behavior than the existing atomic file helpers.
  - confidence: 0.86
