## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Prevent sealed-store plaintext downgrade:** `src/infra/sealed-json-file.ts:113`
  - body: When a sealed `auth-profiles.json` exists and `OPENCLAW_PASSPHRASE` is missing, the load path can return an empty store and later call this writer without a passphrase. This line then overwrites the sealed file as plaintext, losing the encrypted data and exposing any newly merged credentials; refuse writes over an existing sealed store or force read-only until the passphrase is restored.
  - confidence: 0.94
- **[P1] Read sealed main auth during inheritance:** `src/agents/auth-profiles/store.ts:412`
  - body: This PR can write the main auth-profile store in sealed form, but subagent inheritance still reads it with `loadJsonFile`, which only parses raw JSON. With sealing enabled, subagents silently fail to inherit main credentials; use the sealed-aware loader for this path too.
  - confidence: 0.9
- **[P1] Do not default-timeout expectFinal requests:** `src/gateway/client.ts:520-524`
  - body: The new default timeout applies even when `expectFinal` is true. Long-running agent/chat RPCs can send an accepted response and finish after 30 seconds, so the client would delete the pending request and reject while the server keeps running; current main avoids this by requiring an explicit timeout for `expectFinal` requests.
  - confidence: 0.86
