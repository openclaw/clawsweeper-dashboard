## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prevent sealed-store plaintext downgrade:** `src/infra/sealed-json-file.ts:113`
  - body: When no passphrase is present, this writer falls back to raw JSON even if the destination was previously sealed. After the sealed load path returns an empty store, a legacy/OAuth/CLI sync write can overwrite encrypted credentials as plaintext, so the write path must refuse or stay read-only until the passphrase is restored.
  - confidence: 0.94
- **[P1] Read sealed main auth during inheritance:** `src/agents/auth-profiles/store.ts:412`
  - body: This PR can write the main auth-profile store in sealed form, but subagent inheritance still reads it with `loadJsonFile`, which cannot parse the sealed prefix. With sealing enabled, subagents silently lose inherited credentials; use the sealed-aware loader or redesign this on the current SQLite store.
  - confidence: 0.9
- **[P1] Avoid default timeouts for expectFinal requests:** `src/gateway/client.ts:520-524`
  - body: The new timeout calculation applies the 30s default even for `expectFinal` calls. Those requests can receive an accepted response and legitimately finish later, so the client would delete the pending entry and reject while the server continues work; current main only times these out when a caller supplies `timeoutMs`.
  - confidence: 0.86
