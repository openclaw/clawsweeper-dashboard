## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not time out expectFinal by default:** `src/gateway/client.ts:517-522`
  - body: This default applies even when `opts.expectFinal` is true, so agent/session RPCs that intentionally wait for a final result can now fail after 30 seconds. Current main explicitly tests that wrapper defaults are not injected into `expectFinal` calls, so this needs to preserve unbounded/default-null behavior unless the caller opts into a timeout.
  - confidence: 0.91
- **[P1] Prevent plaintext overwrite when a sealed store is locked:** `src/agents/auth-profiles/store.ts:435`
  - body: When a sealed `auth-profiles.json` exists and `OPENCLAW_PASSPHRASE` is missing, `loadProtectedAuthJson` returns `undefined`; later migration/OAuth/external CLI sync can still set `shouldWrite` and `saveSealedJsonFile` will write plaintext. That breaks the PR's fail-closed claim and can replace sealed credentials with partial unsealed data.
  - confidence: 0.94
- **[P1] Target the current SQLite auth store:** `src/agents/auth-profiles/paths.ts:32`
  - body: This changes runtime creation/writes for `auth-profiles.json`, but current main treats that file and `oauth.json` as legacy doctor-import inputs while the canonical store is `openclaw-agent.sqlite`. Rebase the hardening onto the current SQLite/SecretRef/doctor design instead of adding another runtime JSON credential path.
  - confidence: 0.89
- **[P2] Use atomic sealed writes with explicit KDF metadata:** `src/infra/sealed-json-file.ts:38-110`
  - body: The new helper writes secret-bearing content with `writeFileSync` then `chmodSync`, and the envelope omits explicit scrypt parameters. For a credential hardening path, this should use the repository's atomic replace helper and persist auditable KDF metadata before it becomes a user-facing security format.
  - confidence: 0.82

## Security Review

  - body: A missing passphrase is caught as a load miss, and a later save without a passphrase writes plaintext over the target, which can expose or lose previously sealed credentials.

  - confidence: 0.86
