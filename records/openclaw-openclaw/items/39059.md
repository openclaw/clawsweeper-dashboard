## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Target the current SQLite auth profile store:** `src/agents/auth-profiles/store.ts:526`
  - body: Current main persists auth profiles through `openclaw-agent.sqlite`, and the SQLite regression test asserts `auth-profiles.json` is not read at runtime. This PR's save path only seals the old JSON store, so after a refresh it would either conflict or protect a non-runtime file while real credentials remain in SQLite.
  - confidence: 0.92
- **[P1] Refuse plaintext overwrites of sealed stores:** `src/infra/sealed-json-file.ts:113`
  - body: When `OPENCLAW_PASSPHRASE` is missing, this writer falls back to plaintext even if the destination already contains a sealed store. Combined with the load path that returns an empty/undefined store when the passphrase is absent, a later migration or CLI sync can replace encrypted credentials with plaintext and lose the sealed data.
  - confidence: 0.94
- **[P1] Preserve unbounded defaults for expectFinal calls:** `src/gateway/client.ts:520-524`
  - body: This timeout calculation applies the 30s default to `expectFinal` requests. Current main deliberately leaves `expectFinal` calls unbounded unless the caller supplies `timeoutMs`, with tests for long-running final responses; reintroducing the default timeout would break agent/gateway calls that wait for a final result after an accepted response.
  - confidence: 0.88
- **[P2] Record KDF parameters in the sealed envelope:** `src/infra/sealed-json-file.ts:38-39`
  - body: The new format derives keys with implicit Node `scryptSync` defaults and stores no KDF metadata in the envelope. Credential-at-rest encryption needs explicit parameters so strength and future decryptability are auditable instead of depending on runtime defaults.
  - confidence: 0.78
