## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve unbounded expectFinal requests by default:** `src/gateway/client.ts:520-524`
  - body: This fallback schedules a 30s timeout even for `expectFinal` requests when the caller does not pass `timeoutMs`. Current main and the merged gateway timeout PR keep these long-running agent/session RPCs unbounded by default, so merging this branch would regress legitimate flows unless every caller opts out explicitly.
  - confidence: 0.9
- **[P1] Prevent plaintext overwrite when the sealed store is locked:** `src/agents/auth-profiles/store.ts:436-438`
  - body: When a sealed `auth-profiles.json` is loaded without `OPENCLAW_PASSPHRASE`, the load path returns no store and this write path can still persist migration/OAuth/external-CLI data. Because `saveSealedJsonFile` writes plaintext without a passphrase, a previously sealed store can be silently downgraded or partially replaced.
  - confidence: 0.94
- **[P2] Read sealed main stores for subagent inheritance:** `src/agents/auth-profiles/store.ts:412`
  - body: This inheritance path still uses the plaintext JSON reader for the main store, but the PR can now write that main store in sealed format. With `OPENCLAW_PASSPHRASE` enabled, subagents treat the sealed main store as invalid and fail to inherit credentials.
  - confidence: 0.87
- **[P2] Write sealed auth stores with safe replace semantics:** `src/infra/sealed-json-file.ts:113-114`
  - body: This writes credential-bearing content directly to the destination and chmods afterward. Secret-store writes should be created with restrictive mode up front and replaced atomically so partial writes, permission windows, and symlink-style clobbering are not introduced with the encryption layer.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff changes credential storage and introduces concrete downgrade, write-safety, and key-management concerns.

Concerns:

- **[medium] Sealed auth stores can be downgraded to plaintext:** `src/agents/auth-profiles/store.ts:438`
  - body: A missing passphrase is converted into an absent store, after which migration or sync paths can persist plaintext over a sealed auth store.
  - confidence: 0.94
- **[medium] Secret file writes are not atomic or mode-safe at creation:** `src/infra/sealed-json-file.ts:113`
  - body: `saveSealedJsonFile` writes directly to the destination and chmods afterward, which is weaker than the repository's existing safe secret-file and atomic replace patterns.
  - confidence: 0.82
- **[medium] Passphrase sealing lacks settled recovery policy:**
  - body: The new env-var based encryption surface does not settle lost-passphrase recovery, key rotation, startup/service behavior, or migration from plaintext/current SQLite auth storage.
  - confidence: 0.86
- **[low] KDF parameters are not versioned in the envelope:** `src/infra/sealed-json-file.ts:38`
  - body: The sealed-file format stores algorithm, salt, IV, tag, and ciphertext, but not explicit KDF parameters, making future tuning or compatibility harder to audit.
  - confidence: 0.74
