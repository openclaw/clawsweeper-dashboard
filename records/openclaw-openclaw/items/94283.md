## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Prevent repo-root fallback for missing bundle artifacts:** `extensions/qa-lab/src/evidence-gallery.ts:193`
  - body: When a summary declares a bundle-local artifact such as `runner/result.json`, this fallback means a missing artifact can resolve to a same-named file under the repo root. That makes the gallery mark the artifact as present and serve/preview unrelated repo data, so only use the repo-root candidate for paths that are explicitly repo-root-relative, not after a bundle-local miss.
  - confidence: 0.93
- **[P2] Skip non-object matrix cells before reading fields:** `extensions/qa-lab/src/evidence-gallery.ts:501-505`
  - body: `matrix.cells` is optional producer context, but this cast lets values like `null` reach `cell.surface` and throw, causing the whole `/api/evidence` request to fail for an otherwise valid evidence summary. Filter each cell through `readRecord` before reading fields so malformed optional matrix rows are ignored.
  - confidence: 0.88
- **[P2] Harden artifact responses before streaming local files:** `extensions/qa-lab/src/lab-server.ts:454-457`
  - body: The new artifact route serves local QA artifacts, including logs and arbitrary declared files, with only content type and length. Add at least `cache-control: no-store` and `x-content-type-options: nosniff` so browsers and intermediaries do not cache sensitive proof logs or sniff file content into a more active type.
  - confidence: 0.86
