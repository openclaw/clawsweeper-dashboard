## Summary

Keep open. The reported display bug is source-reproducible, but this PR should not merge as-is because it changes a fresh-only helper globally while preflight compaction still depends on that helper being fresh-only, and the PR body lacks after-fix real behavior proof from a patched runtime.

## What This Changes

The PR changes session token helpers and related status/context/session-row tests so preserved stale `totalTokens` values are returned for display while adding explicit stale checks to some runtime callers.

## Best Possible Solution

Expose preserved stale totals only through display/session projection paths, while keeping compaction, memory, fork, and compact callers on explicit fresh-only semantics.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main preserves some stale `totalTokens` values but hides them from `/context` and gateway rows through the fresh-only helper. This was not live-reproduced in a patched runtime during this read-only review.

## Solution Assessment

No. Fixing the display behavior is appropriate, but changing `resolveFreshSessionTotalTokens` globally is not the narrow maintainable fix while compaction still relies on that helper being fresh-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep stale totals from bypassing transcript re-estimation:** `src/config/sessions/types.ts:553-558`
  - body: Returning stale totals from `resolveFreshSessionTotalTokens` changes every caller of this helper. `runPreflightCompactionIfNeeded` still requests transcript fallback for stale entries, but then skips `estimatePromptTokensFromSessionTranscript` whenever `freshPersistedTokens` is a number; after this helper change, stale preserved totals take that path and can avoid the transcript re-estimation needed for compaction decisions. Keep this helper fresh-only or add a display-only preserved-total helper and update only display/session-row callers.
  - confidence: 0.91
