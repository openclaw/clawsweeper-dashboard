## Summary

Keep open: the display bug is source-reproducible and the direction is useful, but the patch changes a fresh-only helper globally and leaves preflight compaction using that helper to decide whether to re-read stale transcripts. The PR also still lacks patched after-fix real behavior proof.

## What This Changes

The PR changes session token helpers and related status/context/session-row tests so preserved stale `totalTokens` values are displayed, with added direct stale checks in memory flush, parent-fork, and gateway session rows.

## Best Possible Solution

Expose preserved stale totals through display/session projection paths, while keeping compaction, memory, fork, and compact callers on explicit fresh-only semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main hides preserved stale totals in `/context` and status display, and the PR head makes an unchanged preflight compaction caller treat those stale totals as fresh enough to skip transcript re-estimation.

## Solution Assessment

No. The display behavior should be fixed, but changing `resolveFreshSessionTotalTokens` globally is not the narrow maintainable solution while runtime compaction callers still depend on that helper being fresh-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep stale totals from bypassing transcript re-estimation:** `src/config/sessions/types.ts:553-558`
  - body: Returning stale totals from `resolveFreshSessionTotalTokens` changes every caller of this helper. `runPreflightCompactionIfNeeded` still sets `shouldUseTranscriptFallback` for stale entries, but then skips `estimatePromptTokensFromSessionTranscript` whenever `freshPersistedTokens` is a number; after this helper change, stale preserved totals take that path and can avoid the transcript re-estimation needed for compaction decisions. Keep this helper fresh-only or add a display-only preserved-total helper and update only display/session-row callers.
  - confidence: 0.91
