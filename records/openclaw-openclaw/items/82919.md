## Summary

Keep open: the display bug is source-reproducible, but this PR still changes a fresh-only helper globally and leaves preflight compaction able to skip transcript re-estimation for stale totals. The PR also still lacks after-fix real behavior proof from the patched code.

## What This Changes

The PR changes session token helpers and status/context/session row tests so preserved stale `totalTokens` values are displayed, with added direct stale checks in memory-flush, parent-fork, and gateway session rows.

## Best Possible Solution

Expose preserved stale totals only through display/session projection paths, while keeping compaction, memory, fork, and compact callers on explicit fresh-only semantics.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main hides stale preserved totals in `/context` and status display, and the PR diff shows the proposed global helper change. The remaining PR bug is also source-reproducible from the preflight compaction caller that still depends on this helper being fresh-only.

## Solution Assessment

No. The display direction is valid, but changing `resolveFreshSessionTotalTokens` globally is not the narrow maintainable fix while runtime compaction still uses that helper to decide whether transcript re-estimation is needed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep stale totals from bypassing transcript re-estimation:** `src/config/sessions/types.ts:558`

Keep this helper fresh-only or split a display-only preserved-total helper and update only display/session-row callers.
