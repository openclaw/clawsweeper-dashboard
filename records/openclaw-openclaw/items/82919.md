## Summary

Keep open: the display/session-list regression is real, but this PR changes the fresh-only token helper globally and would break compaction and memory paths that still rely on stale totals being ignored. It also lacks after-fix real behavior proof from a patched run.

## What This Changes

The PR makes preserved `totalTokens` visible when `totalTokensFresh` is false, updates related status/context/session-list tests, adds a parent-fork freshness guard, and adds a changelog entry.

## Best Possible Solution

Expose preserved stale totals only through display/session projection paths, while keeping a clearly fresh-only helper or explicit freshness guard for compaction, memory, fork, and compact callers.

## Reproduction Assessment

Yes. Source inspection shows current main hides finite stale totals in `/context` and session rows, and existing memory/preflight tests show why the PR's global helper change would regress fresh-only runtime gating.

## Solution Assessment

No. The requested display direction is valid, but globally changing `resolveFreshSessionTotalTokens` is not the narrow maintainable fix because several runtime paths still depend on stale entries resolving to `undefined`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
