## Summary

Keep open: current main and the latest release still lack typed agent-facing Discord component callback metadata. Richer data exists for plugin interactive handlers, but generic agent callbacks remain text-first, so this is still canonical product/API work rather than conservative cleanup.

## What This Changes

The issue requests structured Discord button interaction metadata, including source message ID, button custom or callback data, label, and user identity, in agent callback context.

## Best Possible Solution

Define a portable typed inbound interaction fact, populate it from Discord component fallback handling with explicit trust classification, and cover the contract with Discord docs plus regression tests.

## Reproduction Assessment

Yes, at source level. Current main builds generic Discord component callbacks as text in `eventText` and uses that for `BodyForAgent`, while richer interaction details remain limited to plugin interactive handlers; I did not run a live Discord click in this read-only review.

## Solution Assessment

No, the Discord-specific JSON sketch captures the need but is not the best final contract. The maintainable fix is a portable typed inbound interaction fact with explicit trust classification and Discord as the first populated channel.
