## Summary

Keep open: current main still sends unmatched Discord button/select callbacks to the agent as plain inbound text, while the Discord interaction object and plugin-interactive path already carry the requested metadata.

## What This Changes

Add structured Discord interaction metadata to agent-visible button/select callback context so agents can identify the source message, clicked control, and user.

## Best Possible Solution

Add a Discord-owned untrusted structured interaction block to fallback component agent turns, with tests, docs, and changelog coverage, while preserving existing callbackData text behavior.

## Reproduction Assessment

Yes, source-reproducible: register a Discord component with callbackData and no matching plugin handler, then inspect the dispatch context in the existing monitor harness. Current source builds only eventText for the fallback path and finalizes no UntrustedStructuredContext.

## Solution Assessment

Yes. The narrow maintainable fix is to add an additive Discord fallback metadata entry through the existing UntrustedStructuredContext path rather than changing core protocol or plugin interactive dispatch.
