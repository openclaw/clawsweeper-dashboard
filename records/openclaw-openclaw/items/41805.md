## Summary

Keep open: current main still sends ordinary Discord component callbacks to the agent as text-only fallback turns, while richer interaction fields are only available to registered plugin interactive handlers. The remaining work is a valid Discord channel/API feature that needs maintainer review for the model-facing metadata and trust boundary.

## What This Changes

Request structured Discord button/select interaction metadata, including source message, clicked component, and sender fields, in normal inbound agent context.

## Best Possible Solution

Define a typed Discord interaction metadata envelope for normal component fallback turns, document trust semantics for each field, and add focused button/select/modal fallback tests without changing registered plugin interactive handlers.

## Reproduction Assessment

Yes, source-based: current main shows ordinary Discord component callbacks becoming BodyForAgent text or callbackData while structured interaction fields remain confined to plugin interactive handlers. I did not run a live Discord click in this read-only review.

## Solution Assessment

Unclear until maintainer API review. The maintainable direction is a typed inbound metadata contract with explicit trust semantics, not label-encoded IDs or ad hoc model-context injection.
