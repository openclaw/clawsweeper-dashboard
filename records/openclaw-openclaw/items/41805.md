## Summary

Keep open: current main still routes ordinary Discord component fallback clicks as plain agent text, while the raw Discord interaction and plugin-handler paths already carry the requested message, button, selected-value, and user metadata.

## What This Changes

Request to include structured Discord interaction metadata in agent-visible button and select callbacks instead of forcing developers to encode IDs in labels.

## Best Possible Solution

Add an additive Discord-owned untrusted structured interaction block to fallback component agent turns, with regression coverage plus Discord docs and changelog updates.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the existing Discord monitor harness can drive a button/select click and inspect lastDispatchCtx, and current main only provides fallback BodyForAgent text.

## Solution Assessment

Yes. Reusing the existing UntrustedStructuredContext rendering path inside the Discord fallback dispatch is the narrowest maintainable solution; no new core channel contract is needed unless maintainers want cross-channel interaction metadata later.
