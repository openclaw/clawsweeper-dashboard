## Summary

Keep open: current main still routes unmatched Discord button/select callback fallbacks to the agent as text-only turns. The related closed issues cover routing and ordinary sender-id metadata, not the missing structured component-interaction payload.

## What This Changes

Request to include Discord component interaction metadata in callback turns so agents do not have to encode IDs in button labels.

## Best Possible Solution

Add a Discord-owned UntrustedStructuredContext entry for fallback component turns while preserving existing callbackData text, routing, and plugin interactive dispatch behavior.

## Reproduction Assessment

Yes, high-confidence source reproduction exists: register a Discord component with callbackData and no matching plugin handler, then inspect the fallback dispatch context. Current main only populates text fields for that turn and leaves structured interaction metadata unset.

## Solution Assessment

Yes, the narrow maintainable solution is an additive Discord fallback metadata block. Plugin interactive handlers already receive equivalent context, so this does not need a core protocol rewrite or a routing change.
