## Summary

Keep open: current main still sends ordinary Discord component fallback clicks to the agent as text-only turns, while richer interaction fields are only available to registered plugin interactive handlers. This is a valid Discord channel/API feature request that needs maintainer product and trust-boundary review before implementation.

## What This Changes

Request structured Discord button/select interaction metadata in normal agent callback context, including source message, clicked component, and sender fields.

## Best Possible Solution

Define a typed, bounded Discord interaction metadata envelope for normal component fallback turns, document trust semantics, and add focused button/select/modal regression coverage without regressing registered plugin interactive handlers.

## Reproduction Assessment

Yes, source-based: a Discord component click without a matching plugin handler becomes BodyForAgent text from callbackData, label, or selected values, and current tests assert that text-only fallback. I did not run a live Discord click in this read-only review.

## Solution Assessment

Unclear until maintainer API review. The maintainable direction is a typed inbound metadata contract with explicit trust semantics, not embedding identifiers in labels or pushing raw Discord payloads into model context ad hoc.
