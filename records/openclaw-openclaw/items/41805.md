## Summary

Keep open: current main still sends normal Discord component callbacks through the text fallback agent path, while richer interaction fields are only available to registered plugin interactive handlers. This remains a valid feature/API request that needs maintainer product and trust-boundary review.

## What This Changes

The issue requests structured Discord interaction metadata for normal agent callbacks when buttons or other components are clicked.

## Best Possible Solution

Define a typed Discord interaction metadata envelope for normal component callbacks, document trusted versus untrusted fields, and add focused button/select/modal fallback tests without regressing plugin interactive handlers.

## Reproduction Assessment

Yes, source-based: current main shows button/select callbacks becoming text fallback agent turns, while structured interaction fields remain confined to plugin interactive handlers. I did not run a live Discord click in this read-only review.

## Solution Assessment

Unclear until maintainer API review. The best direction is a typed inbound metadata contract with explicit trust semantics, not another ad hoc text workaround.
