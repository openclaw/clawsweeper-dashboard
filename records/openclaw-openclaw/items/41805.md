## Summary

Keep open: current main and v2026.6.10 still flatten generic Discord component callbacks into text for the agent, while richer interaction facts only reach registered plugin interactive handlers. The remaining work is a channel/plugin API and trust-boundary decision, not conservative cleanup.

## What This Changes

The issue requests structured Discord button interaction metadata, including source message, component/custom data, and sender identifiers, in agent callbacks.

## Best Possible Solution

Add a portable typed inbound interaction fact for generic agent callbacks, populate it from Discord component fallback handling, and keep sensitive Discord interaction tokens redacted while documenting trust semantics.

## Reproduction Assessment

Yes, at source level: generic Discord component callbacks route through `dispatchDiscordComponentEvent`, and adjacent tests assert only text reaches `BodyForAgent`. I did not run a live Discord click in this read-only review.

## Solution Assessment

Partially: exposing metadata is the right direction, but the proposed Discord-specific JSON sketch should be shaped as a portable typed inbound interaction fact with explicit trust classification.
