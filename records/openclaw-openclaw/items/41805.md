## Summary

Keep open: current main still converts ordinary Discord button/select callbacks into a text fallback for the agent, while richer interaction fields are only exposed to registered plugin interactive handlers. This is a valid channel/API feature request and needs maintainer product/trust-boundary review before implementation.

## What This Changes

The issue requests structured Discord interaction metadata, including source message, clicked component, and sender details, for Discord button callbacks delivered to agents.

## Best Possible Solution

Define a typed Discord interaction metadata envelope for normal component callbacks, document trusted versus untrusted fields, and add focused button/select/modal fallback tests without regressing registered plugin interactive handlers.

## Reproduction Assessment

Yes, source-based: current main shows button/select callbacks becoming text fallback agent turns while structured interaction fields remain confined to registered plugin interactive handlers. I did not run a live Discord click in this read-only review.

## Solution Assessment

Unclear until maintainer API review. The maintainable direction is a typed inbound metadata contract with explicit trust semantics, not another ad hoc text workaround.
