## Summary

Keep open: current main still routes normal Discord component callbacks into the agent as text fallback turns, while the richer structured interaction fields are only exposed to registered plugin interactive handlers. This remains a valid feature/API request that needs maintainer product and trust-model review.

## What This Changes

Request to expose structured Discord button interaction metadata such as message id, button identity/data, label, and clicker identity to normal agent callbacks.

## Best Possible Solution

Define a typed Discord interaction metadata envelope for normal component callbacks, document the trust semantics, and add focused button/select fallback tests without regressing plugin interactive handlers.

## Reproduction Assessment

Yes, source-based. Current main shows button/select callbacks becoming text fallback agent turns, while the structured fields remain confined to plugin interactive handlers.

## Solution Assessment

Unclear until maintainer API review. The plugin interactive handler context is useful partial coverage, but normal agent callbacks need a typed inbound metadata contract rather than another ad hoc text workaround.
