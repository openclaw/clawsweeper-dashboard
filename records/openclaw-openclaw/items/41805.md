## Summary

Keep open: current main still handles normal Discord component callbacks as text fallback turns, while the richer structured interaction fields are only exposed to registered plugin interactive handlers. This is a valid feature/API request, not an already-implemented cleanup candidate.

## What This Changes

Request to expose structured Discord button interaction metadata, including message id, component id or callback id, button label, and clicking user, to normal agent callbacks.

## Best Possible Solution

Define a typed Discord interaction metadata contract for normal inbound component callbacks, document its trust semantics, and add focused Discord component tests for button/select fallback events.

## Reproduction Assessment

Yes, source-based. Current main’s handler, dispatch path, docs, and tests show normal Discord component callbacks reaching the agent as text fallback events rather than a structured interaction metadata envelope.

## Solution Assessment

No current implementation is complete. The plugin interactive handler context is a useful partial surface, but the requested normal agent callback metadata needs a maintainer-approved typed API and trust model before implementation.
