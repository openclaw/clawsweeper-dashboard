## Summary

Keep open: current main still lacks an agent-visible structured metadata contract for general Discord component fallback clicks. The request is valid, but it needs maintainer API-shape review because it changes what inbound turns expose to agents.

## What This Changes

Add structured Discord button interaction metadata, including source message id, component/custom id or callback data, button label, and clicking user identity, to button callback turns.

## Best Possible Solution

Define an additive, documented inbound interaction metadata contract for Discord component fallback events, reuse the existing internal Discord payload fields, and add regression coverage for both matched plugin handlers and general agent fallback turns.

## Reproduction Assessment

Yes, source-level: current main turns general Discord component fallback clicks into `BodyForAgent` text and does not serialize source message, component/custom id, label, or structured interaction metadata. I did not run a live Discord interaction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the API shape. The requested direction is useful, but the maintainable fix should be an additive metadata contract rather than Discord-only prompt-field drift.
