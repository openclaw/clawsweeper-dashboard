## Summary

Keep open: current main has conversation-binding and inbound-claim seams, but no public pre-routing redirect/suppress hook, and the related open `pre_route` PR still needs maintainer API/security review and proof before it can be the fix.

## What This Changes

Request a new plugin hook that runs before inbound session routing so plugins can redirect or suppress channel messages.

## Best Possible Solution

Use the open `pre_route` work as source material only after maintainers settle a narrow documented SDK routing contract with explicit redirect/suppress, trust, timeout, dedupe, audit, tests, docs, and redacted live plugin proof.

## Reproduction Assessment

Not applicable: this is a new plugin API/routing capability request, not broken documented behavior. Source inspection of current main verifies the requested hook and result shape are absent.

## Solution Assessment

Unclear as proposed: a generic first-writer pre-routing hook may solve the use case, but maintainers should decide whether to extend conversation bindings, `inbound_claim`, or the open `pre_route` PR rather than adding a broad interception point by default.
