## Summary

Keep open: this is a new public plugin/API capability request, not an obsolete or already-shipped bug fix. Current main has static conversation binding and `inbound_claim`/message hooks, but it does not expose a generic pre-session hook that can redirect or suppress arbitrary inbound messages before routing.

## What This Changes

Add a plugin hook that runs before inbound messages are assigned to a canonical session so plugins can redirect or suppress delivery for bridging/proxying.

## Best Possible Solution

Design a narrow, documented plugin API extension or binding extension that safely supports pre-session redirect/suppress behavior with explicit precedence, auth, timeout, and regression tests across representative channel paths.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new plugin hook/API request, not broken behavior under an existing contract. Source inspection confirms current main lacks the requested hook while existing conversation binding and `inbound_claim` cover only narrower cases.

## Solution Assessment

Unclear pending maintainer design. A pre-route hook is plausible, but maintainers should first decide whether to extend conversation binding, add a new hook, or use both, and define precedence plus security semantics before implementation.
