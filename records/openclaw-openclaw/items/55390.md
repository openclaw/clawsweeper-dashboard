## Summary

Keep open: this is the active implementation candidate for https://github.com/openclaw/openclaw/issues/55235, but the current patch still has a source-level config-write blocker and lacks real behavior proof. The prior bot review caught useful issues; the snake_case key was fixed, but the include-owned config path and docs heading regression remain actionable.

## What This Changes

The PR adds default-on CLI startup maintenance that writes `openclaw_schema.json` beside `openclaw.json`, backfills `$schema`, exposes `update.maintainConfigJsonSchema`, updates config docs, and adds config/CLI tests.

## Best Possible Solution

Repair this PR to be include-aware: keep schema generation best-effort, backfill only the authored root safely or skip root mutation for include-owned configs, preserve custom `$schema` values, keep docs hierarchy intact, and add redacted real behavior proof.

## Reproduction Assessment

Yes, at source level: use a root `openclaw.json` authored with `$include` and no `$schema`, then run a normal CLI command that triggers PR startup maintenance. The PR calls `writeConfigFile` with `snapshot.config`, and current main rejects root include-owned patches instead of safely editing the authored root.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but this implementation should patch the authored root safely or skip root mutation for include-owned configs instead of writing the resolved runtime snapshot during startup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:96-98`
  - body: `maintainLocalConfigJsonSchemaArtifacts` derives `nextConfig` from the resolved runtime snapshot and passes it to `writeConfigFile` when `$schema` is missing. For a root config authored with `$include`, the current-main write guard rejects that root patch, so normal CLI runs warn and never maintain the schema artifacts; on older bases this same shape risked flattening the authored root. Patch only the authored root safely or skip root mutation for include-owned configs while still allowing best-effort schema-file generation.
  - confidence: 0.9
- **[P3] Keep Paths as a peer config section:** `docs/cli/config.md:47`
  - body: Changing `### Paths` to `#### Paths` nests the general `config get/set/patch` path syntax under the `config schema` section. Leave `Paths` at the existing peer heading level or move the schema note under its own subsection.
  - confidence: 0.74
