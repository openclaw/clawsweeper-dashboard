## Summary

Keep open: the PR is a plausible implementation of the linked schema-maintenance feature, but the current diff still has an include-owned config write bug, a docs heading regression, and no after-fix real behavior proof from the contributor.

## What This Changes

The PR adds default-on CLI maintenance that writes `openclaw_schema.json`, backfills `$schema`, exposes `update.maintainConfigJsonSchema`, updates docs, and adds config/CLI tests.

## Best Possible Solution

Repair or replace the branch with an include-aware, idempotent local schema maintenance path that patches only the authored root when safe, preserves custom `$schema`, keeps docs structure intact, and includes redacted after-fix proof.

## Reproduction Assessment

Yes, at source level: use a root `openclaw.json` authored with `$include` and no `$schema`, then run a normal CLI command that triggers the PR startup maintenance. The PR calls `writeConfigFile` with `snapshot.config`, while current-main write preparation rejects patches touching a root include-owned config.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but this branch should patch the authored root safely or skip root mutation for include-owned configs instead of writing the resolved runtime snapshot during startup.
