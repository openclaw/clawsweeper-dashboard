## What This Changes

The PR adds CLI/config support to generate and maintain a local `openclaw_schema.json`, backfill `$schema`, expose an update opt-out setting, and document/test that behavior.

## Best Possible Solution

Land or replace this branch with an include-aware, idempotent local schema-maintenance path that preserves custom `$schema` values, avoids unsafe startup config mutation, keeps docs hierarchy intact, and includes redacted real behavior proof.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows root `$include` configs without `$schema` go through `writeConfigFile(nextConfig, writeOptions)`, while current main rejects writes that touch include-owned root paths. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. Local schema maintenance is a reasonable feature direction, but the startup path should avoid mutating include-owned root configs and should still write the schema artifact where safe.
