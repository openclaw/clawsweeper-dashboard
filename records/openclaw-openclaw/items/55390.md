## Summary

Keep this PR open. It targets a still-open local schema maintenance feature, but the current branch has a source-level config write bug, a docs hierarchy regression, missing real behavior proof, merge conflicts, and failing checks, so it needs contributor or maintainer repair rather than cleanup close.

## What This Changes

The branch adds default-on CLI maintenance that writes `openclaw_schema.json`, backfills `$schema`, exposes `update.maintainConfigJsonSchema`, updates schema metadata/docs, and adds config/CLI tests.

## Best Possible Solution

Rebase and repair this PR, or replace it with include-aware, idempotent local schema maintenance that preserves custom `$schema` values, current schema metadata, and docs structure.

## Reproduction Assessment

Yes, at source level for the review finding: use a root config authored with `$include` and no `$schema`, then run a normal CLI command that triggers startup maintenance. The PR write path and current-main include-owned write guard show the failure without needing to mutate this checkout.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but this implementation should patch only the authored root safely or skip root mutation for include-owned configs instead of writing the resolved runtime snapshot during startup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:96-98`
  - body: `maintainLocalConfigJsonSchemaArtifacts` builds `nextConfig` from the resolved runtime snapshot and passes it to `writeConfigFile` when `$schema` is missing. For root configs that use `$include`, that startup write can flatten authored include structure on this branch's old base, and after rebasing onto current main it will instead hit the include-owned write guard and warn repeatedly. Patch the authored root directly or skip root mutation for include-owned configs.
  - confidence: 0.88
- **[P3] Keep Paths as a peer config section:** `docs/cli/config.md:47`
  - body: Changing `### Paths` to `#### Paths` nests the general `config get/set/patch` path syntax under the `config schema` section. Leave `Paths` at the existing peer heading level or move the schema note under its own subsection.
  - confidence: 0.74
