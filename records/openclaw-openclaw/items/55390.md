## Summary

Keep open. Current main still lacks the requested local schema maintenance, and this PR is a plausible implementation candidate, but the branch needs changes because startup backfill writes the resolved config snapshot through `writeConfigFile`, which is unsafe for `$include`-owned roots on current main; the docs also accidentally nest the general Paths section under `config schema`.

## What This Changes

The PR adds default-on CLI/config behavior to write `openclaw_schema.json` beside `openclaw.json`, backfill `$schema` when missing, expose `update.maintainConfigJsonSchema`, update docs, and add tests.

## Best Possible Solution

Rebase and land an include-aware implementation that keeps local schema generation best-effort, avoids startup mutation of include-owned root configs, preserves custom `$schema` values and current schema metadata, keeps docs hierarchy intact, and includes real behavior proof.

## Reproduction Assessment

Yes for the blocking PR behavior at source level: use a root config authored with `$include` and no `$schema`, then run a normal CLI command that triggers startup maintenance. I did not execute it because this review is read-only, but the PR write path and current-main include-owned write guard make the failure path clear.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but the startup backfill should patch the authored root safely or skip root mutation for include-owned configs instead of writing the resolved snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:92-93`
  - body: `maintainLocalConfigJsonSchemaArtifacts` creates `nextConfig` from `snapshot.config` and calls `writeConfigFile` when `$schema` is missing. For a root config authored with `$include`, that writes the resolved snapshot through current main's include-owned write guard and either flattens the root on the old base or fails with repeated warnings after rebase. Patch the authored root directly or skip root mutation when the root is include-owned.
  - confidence: 0.88
- **[P3] Keep Paths as a peer config section:** `docs/cli/config.md:44`
  - body: The docs change turns the existing `### Paths` section into `#### Paths` immediately after the schema text, so the general `config get/set/patch` path syntax appears nested under `config schema`. Leave Paths at the existing `###` level or move the local-schema note under its own subsection.
  - confidence: 0.74
