## Summary

Keep open: the feature is useful, but this PR is not merge-ready because the startup backfill path is compatibility-sensitive for include-owned and immutable configs, real behavior proof is missing, and the branch is currently conflicting with stale failed checks.

## What This Changes

The branch adds CLI startup/config-write hooks, local schema generation helpers, schema metadata, docs, and tests to keep `openclaw_schema.json` beside `openclaw.json` and optionally backfill `$schema`.

## Best Possible Solution

Finish or replace this with an idempotent, include-aware, immutable-config-safe maintenance path that patches only safe authored roots or skips root mutation while still generating the schema file, restores the docs hierarchy, and includes redacted real CLI proof.

## Reproduction Assessment

Yes for the PR blocker by source inspection: the branch writes `snapshot.config` through `writeConfigFile` when `$schema` is missing, while current main rejects include-owned writes and Nix mode refuses config writers. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. Local schema maintenance is a reasonable direction, but the implementation needs an include-aware and immutable-config-safe write policy plus a small docs hierarchy fix before merge.
