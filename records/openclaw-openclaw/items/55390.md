## Summary

Keep open: current main still lacks automatic local schema artifact maintenance, but this PR needs changes before merge because the startup backfill is not include-safe and the docs/changelog cleanup remains incomplete.

## What This Changes

The PR adds default-on local `openclaw_schema.json` generation and `$schema` backfilling on CLI startup/config writes, exposes `update.maintainConfigJsonSchema`, and updates schema metadata, docs, and tests.

## Best Possible Solution

Land an include-aware local schema maintenance implementation that stays best-effort and non-destructive, preserves custom `$schema` and opt-out behavior, keeps schema metadata intact, and documents/changelogs the new default-on behavior.

## Reproduction Assessment

Yes. A high-confidence source path is a root `openclaw.json` using `$include` and no `$schema`, followed by a normal CLI command that runs startup maintenance and reaches the PR's `writeConfigFile(nextConfig, ...)` path.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but startup backfill should patch the authored root safely or skip root mutation for include-owned configs instead of writing the resolved/runtime snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:98-99`
  - body: `maintainLocalConfigJsonSchemaArtifacts` backfills `$schema` by passing `nextConfig` derived from `snapshot.config` into `writeConfigFile` during startup. For root `$include` configs, this can flatten the authored root on the PR base and will trip current main's include-owned write guard after rebase, causing warnings on ordinary CLI runs. Patch the authored root safely or skip root mutation for include-owned roots, with regression coverage.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `docs/cli/config.md:69`
  - body: This default-on feature changes CLI startup/config-write behavior and adds a public config key, but the PR has no `CHANGELOG.md` diff. Add a single-line active-version changelog entry with appropriate contributor credit before merge.
  - confidence: 0.84
- **[P3] Keep Paths as a peer config section:** `docs/cli/config.md:71`
  - body: The docs change demotes the general `Paths` guidance under `config schema`, so `config get/set` path syntax appears nested under the schema command. Keep `Paths` at the existing `###` level or move the local-schema note under its own subsection.
  - confidence: 0.74
