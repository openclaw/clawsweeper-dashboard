- **[P1] Preserve explicit schema unsets:** `src/config/io.ts:1775`
  - body: `writeConfigFile` applies `unsetPaths` before this defaulting call, so `openclaw config unset '$schema'` can remove the field and then immediately get `openclaw_schema.json` stamped back into the same persisted write. Please skip the default when the root `$schema` path is explicitly unset, or move defaulting before unset handling so user removal wins.
  - confidence: 0.91
- **[P1] Avoid flattening include-backed config during startup sync:** `src/config/local-json-schema.ts:98`
  - body: This startup path builds `nextConfig` from `snapshot.config`, which is the resolved/runtime shape, and then calls `writeConfigFile` during normal CLI preaction when `$schema` is missing. For configs that use `$include`, that can turn a read-style command into a root config rewrite that inlines included content and drops the authored include structure/comments; schema maintenance should refresh the sidecar without rewriting the root unless a root-owned config write is already happening.
  - confidence: 0.86
