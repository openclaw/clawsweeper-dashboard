## Summary

Keep open: current main still does not maintain local config schema artifacts, and the PR has concrete merge blockers around include-owned configs and release notes.

## What This Changes

The PR adds default-on local `openclaw_schema.json` generation, `$schema` backfilling on config writes/startup, an `update.maintainConfigJsonSchema` opt-out, docs, schema metadata, and tests.

## Best Possible Solution

Rebase and land an include-aware implementation that keeps local schema generation best-effort, avoids startup mutation of include-owned root configs, preserves custom `$schema` values and current schema metadata, and adds the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level path is a root config authored with `$include` and no `$schema`, followed by a normal CLI command that runs startup maintenance; the PR writes from `snapshot.config`, while current main rejects writes that would flatten root includes.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but the proposed startup backfill should patch the authored root safely or skip root mutation for include-owned configs instead of writing the resolved snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:98-99`
  - body: `maintainLocalConfigJsonSchemaArtifacts` backfills `$schema` by passing `nextConfig` from `snapshot.config` into `writeConfigFile` during startup. For root `$include` configs, this can flatten the authored root on the PR base and will trip current main's include-owned write guard after rebase, causing repeated warnings on ordinary CLI runs. Skip root mutation for include-owned roots or use an authored-root-aware patch path, with regression coverage.
  - confidence: 0.88
