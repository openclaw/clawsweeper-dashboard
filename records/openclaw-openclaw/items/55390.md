## Summary

Keep open: current main still lacks local schema maintenance, but this PR remains unsafe to merge because startup backfill writes a resolved config snapshot through include-preservation paths, the docs diff demotes a peer section, and contributor real behavior proof is missing.

## What This Changes

The branch adds default-on CLI/config behavior to write `openclaw_schema.json` beside `openclaw.json`, backfill `$schema`, expose `update.maintainConfigJsonSchema`, update docs, and add tests.

## Best Possible Solution

Rebase and repair this PR or replace it with an include-aware implementation that keeps schema generation best-effort, avoids unsafe root-config mutation, preserves custom `$schema` values and current schema metadata, keeps docs hierarchy intact, and includes real behavior proof.

## Reproduction Assessment

Yes at source level: use a root config authored with `$include` and no `$schema`, then run a normal CLI command that triggers PR startup maintenance. I did not execute it because this review is read-only, but the PR write path and current-main include-owned write guard make the failure path clear.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but this implementation should patch only the authored root safely or skip root mutation for include-owned configs instead of writing the resolved snapshot during startup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:96-98`

  - confidence: 0.88
