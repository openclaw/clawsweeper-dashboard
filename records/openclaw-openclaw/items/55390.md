## Summary

Keep open: current main does not yet provide local schema-artifact maintenance, but this PR should not merge as-is because startup schema backfill is not include-aware and the external PR still lacks real behavior proof.

## What This Changes

The PR adds default-on CLI startup maintenance for a local `openclaw_schema.json`, default `$schema` backfill, schema metadata, docs, and config/CLI tests.

## Best Possible Solution

Repair this PR to keep schema generation best-effort, patch only the authored root or skip root mutation for include-owned configs, preserve custom `$schema` values, keep docs hierarchy intact, and add redacted real behavior proof.

## Reproduction Assessment

Yes, at source level: use a root `openclaw.json` authored with `$include` and no `$schema`, then run a normal CLI command that triggers PR startup maintenance. The PR writes from runtime-shaped `snapshot.config`, and current main rejects root include-owned patches instead of safely editing the authored root.

## Solution Assessment

No. Local schema artifact maintenance is a reasonable direction, but this implementation should patch the authored root safely or skip root mutation for include-owned configs instead of writing the resolved runtime snapshot during startup.
