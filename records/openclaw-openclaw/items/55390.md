## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Avoid rewriting config from startup schema maintenance:** `src/config/local-json-schema.ts:98`
  - body: `maintainLocalConfigJsonSchemaArtifacts()` is now reached from normal command startup, and this branch calls `writeConfigFile` whenever `$schema` is missing. For configs that use `$include`, the writer persists from the resolved snapshot, so a read-style command such as `status` can flatten included config into the root file just to add the schema reference. Limit schema stamping to root config writes that are already happening, or use an include-preserving mutation path.
  - confidence: 0.9
- **[P2] Respect explicit root schema unsets:** `src/config/io.ts:1775`
  - body: `writeConfigFile` applies `unsetPaths` and then immediately calls `withDefaultLocalConfigSchemaRef`, so a user running `openclaw config unset '$schema'` will have the field re-added in the same write while maintenance is enabled. Skip the default when the write explicitly unsets the root `$schema`, or require the user-facing opt-out to be set first and document that contract.
  - confidence: 0.78
