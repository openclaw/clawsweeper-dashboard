## Summary

Keep open. Current main still lacks local schema artifact maintenance; the PR is useful but merge-blocked by include-owned config mutation risk and the missing changelog entry.

## What This Changes

The PR adds best-effort local `openclaw_schema.json` generation, startup/config-write `$schema` backfilling, an `update.maintainConfigJsonSchema` opt-out, docs, schema metadata, and tests.

## Best Possible Solution

Land a rebased include-aware implementation that keeps schema-file generation best-effort, preserves custom `$schema` values, avoids root mutation for `$include`-owned configs, and documents the user-facing change in the active changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a root `$include` config with no `$schema`, followed by a read-style CLI command that runs startup maintenance; the PR writes from the resolved snapshot while current main rejects root-include flattening.

## Solution Assessment

No. Local schema generation is a reasonable feature direction, but writing a resolved config snapshot during startup is not the narrowest safe implementation; an authored-root-aware patch path or skipping root `$schema` backfill for include-owned roots is safer.
