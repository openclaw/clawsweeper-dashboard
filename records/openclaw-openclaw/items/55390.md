## Summary

Keep open: this PR targets a still-open feature request, and current main does not yet maintain a local config schema artifact. The branch remains useful but has a blocking include-owned config write issue and lacks after-fix real behavior proof.

## What This Changes

The branch adds default-on CLI maintenance for a local `openclaw_schema.json`, `$schema` backfill, an `update.maintainConfigJsonSchema` opt-out, docs, and config/CLI tests.

## Best Possible Solution

Repair or replace the branch with an include-aware, idempotent schema maintenance path that preserves custom `$schema`, avoids unsafe startup config mutation, keeps docs hierarchy intact, and includes redacted after-fix proof.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows root `$include` configs with no `$schema` go through `writeConfigFile(nextConfig, writeOptions)`, while current main rejects patches touching include-owned root paths. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. Local schema maintenance is a reasonable direction, but the write path should patch only the authored root when safe or skip root mutation for include-owned configs while still doing best-effort schema-file generation.
