## Summary

Keep open: this is the active implementation PR for the local schema-maintenance feature, but it still has a concrete config-write blocker and no posted real behavior proof.

## What This Changes

The PR adds default-on CLI maintenance for a local `openclaw_schema.json`, `$schema` backfill, an opt-out config key, docs, and config/CLI tests.

## Best Possible Solution

Repair or replace the branch with an include-aware, idempotent schema maintenance path that preserves custom `$schema`, avoids unsafe startup config mutation, keeps the docs hierarchy intact, and includes redacted after-fix proof.

## Reproduction Assessment

Yes, for the PR blocker: a root `openclaw.json` with `$include` and no `$schema` follows the PR path into `writeConfigFile(snapshot.config, writeOptions)`, while current-main write preparation rejects root include-owned patches.

## Solution Assessment

No. Local schema maintenance is a reasonable direction, but this branch should patch only the authored root when safe or skip root mutation for include-owned configs while still doing best-effort schema-file generation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Patch the authored root safely or skip root mutation for include-owned configs while still allowing best-effort schema-file generation.
