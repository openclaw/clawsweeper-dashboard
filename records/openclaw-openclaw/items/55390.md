## What This Changes

The branch adds CLI/config-write hooks, local schema generation helpers, schema metadata, docs, and tests to keep `openclaw_schema.json` beside `openclaw.json` and optionally backfill `$schema` via `update.maintainConfigJsonSchema`.

## Best Possible Solution

Repair or replace the branch with an idempotent local schema maintenance path that preserves custom `$schema`, avoids unsafe startup mutation, respects authored/include-managed and immutable configs, keeps docs hierarchy intact, and includes redacted real CLI proof.

## Reproduction Assessment

Yes for the PR blocker by source inspection: the branch writes `snapshot.config` through `writeConfigFile` during startup when `$schema` is missing, while current main rejects writes touching root `$include` ownership. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. Local schema maintenance is a reasonable direction, but the implementation needs an include-aware and immutable-config-safe write policy plus a small docs hierarchy fix before merge.
