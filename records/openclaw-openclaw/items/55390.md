- **[P1] Respect explicit `$schema` removal:** `src/config/io.ts:1775`
  - body: Applying `withDefaultLocalConfigSchemaRef` during every config write means `openclaw config unset '$schema'` cannot actually persist the removal: `runConfigUnset` passes `unsetPaths`, the writer removes `$schema`, then this defaulting path adds it back before serialization. The schema breadcrumb needs to skip stamping when `$schema` was explicitly unset.
  - confidence: 0.91
- **[P1] Avoid startup config rewrites for schema maintenance:** `src/config/local-json-schema.ts:96`
  - body: This helper is called from CLI startup paths, so a read-style command can rewrite `openclaw.json` just because `$schema` is missing. That is risky for include-backed configs and conflicts with the current writer contract that root/included files are only mutated during explicit OpenClaw-owned config writes; refresh the sidecar without rewriting root config on startup, or move stamping to explicit write paths.
  - confidence: 0.86
