## Summary

Keep this PR open as the active implementation for the linked local schema feature, but it is not merge-ready: current main still lacks automatic `openclaw_schema.json` maintenance, while this branch is conflicting, lacks real behavior proof, and its default-on startup write path can trip guarded config-write behavior.

## What This Changes

The branch adds CLI startup/config-write hooks, schema generation helpers, schema metadata, docs, and tests to keep `openclaw_schema.json` beside `openclaw.json` and default `$schema` via `update.maintainConfigJsonSchema`.

## Best Possible Solution

Finish or replace the branch with an idempotent schema maintenance path that writes the local schema artifact safely, preserves custom `$schema`, skips unsafe `$schema` backfill for include-owned or immutable configs, keeps docs hierarchy intact, and includes redacted real CLI proof.

## Reproduction Assessment

Yes for the merge blocker by source inspection: the branch calls `writeConfigFile(nextConfig, writeOptions)` during startup schema backfill when `$schema` is missing, while current main rejects include-owned root writes and Nix-mode config writes. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. Local schema maintenance is a reasonable direction, but the implementation needs include-aware and immutable-config-safe write policy, refreshed validation, and real behavior proof before merge.
