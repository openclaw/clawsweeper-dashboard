## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve explicit `$schema` removals:** `src/config/io.ts:1775`
  - body: `outputConfig` has already had `options.unsetPaths` applied above, but this call re-adds `$schema` whenever maintenance is enabled. That means `openclaw config unset '$schema'` persists the key the user explicitly removed, so the new default silently overrides existing config-write semantics; skip defaulting when `$schema` is in `unsetPaths` and cover that path.
  - confidence: 0.91
- **[P1] Avoid rewriting include-backed roots during startup sync:** `src/config/local-json-schema.ts:98`
  - body: `maintainLocalConfigJsonSchemaArtifacts` builds from `snapshot.config`, which is the resolved/runtime snapshot after `$include` expansion, then calls `writeConfigFile` during startup. For configs whose root delegates sections through `$include`, adding only `$schema` can rewrite the root from the resolved shape and flatten include directives/comments, contrary to the current include-preservation contract; refresh only the sidecar on startup or use a root-preserving writer with regression coverage.
  - confidence: 0.88
