## Summary

Keep this PR open: the code is focused and the contributor supplied convincing terminal proof, but it adds a new public `openclaw.json` env-substitution modifier that is not documented and still needs explicit maintainer acceptance before merge.

## What This Changes

Adds opt-in `${VAR:json}` handling to config env substitution so whole-value JSON arrays/objects can be parsed and preserved across config write-back, with focused tests.

## Best Possible Solution

Land only if maintainers accept `${VAR:json}` as public config syntax and the docs cover array/object coercion, malformed/scalar values, escaping, embedded references, missing vars, and write-back preservation.

## Reproduction Assessment

Yes by source inspection for the current limitation: current main returns `substituteString(...)` for string config values, so a `${VAR}` reference cannot become an array/object before schema validation. The docs gap is also directly reproducible by the current configuration docs.

## Solution Assessment

No as submitted. The explicit opt-in parser plus write-back preservation is the safer shape compared with prior global coercion, but the PR is not merge-ready until maintainers accept and document the new public config syntax.
