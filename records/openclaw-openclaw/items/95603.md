## Review Findings

Full review comments:

- **[P1] Gate JSON coercion by expected config type:** `src/config/env-substitution.ts:202-205`
  - body: This runs JSON array/object coercion in the generic string branch before config validation, so any string-typed field whose env value happens to be JSON-looking can become an array/object and fail or change behavior. For example, `env.vars.*`, headers, or other documented `${VAR}` string fields should stay strings unless the schema/path explicitly expects structured config.
  - confidence: 0.91
- **[P1] Preserve coerced env refs during config writes:** `src/config/env-substitution.ts:205`
  - body: Once `${ALLOW_FROM}` resolves to an array/object, later config writes pass an array/object as the incoming value while the parsed source is still the string `${ALLOW_FROM}`; current `restoreEnvVarRefs` only restores string-to-string matches and keeps mismatched types. That means an unrelated config write can serialize the resolved env value into `openclaw.json` and drop the operator's env reference.
  - confidence: 0.89
