## Summary

Keep open. The underlying Docker sandbox browser problem is still real on current main, but this draft is not mergeable because it patches removed `src/browser` paths while the active browser bridge now crosses the SDK facade and bundled browser plugin runtime, and it still lacks inspectable Docker/DooD proof plus security review for the network exposure change.

## What This Changes

The PR attempts to fix Dockerized sandbox browser bridge reachability and tab creation by separating host-local and container-advertised browser URLs, adding Docker/CDP host handling, config fields, and regression tests.

## Best Possible Solution

Rework the fix through `src/plugin-sdk/browser-bridge.ts` and `extensions/browser/src/browser/**`, preserve distinct host-local and container-advertised URLs, keep CDP handling scoped, document any approved config knob, and attach redacted Docker/DooD proof.

Yes, at source level: current main still exposes only a local `baseUrl` bridge contract and passes that value into embedded sandbox info. I did not run a Docker/DooD E2E reproduction in this read-only review.

## Solution Assessment

No. The intent is correct, but this branch patches legacy browser files and adds public config/network changes without updating the active SDK/runtime path or providing inspectable real behavior proof.
