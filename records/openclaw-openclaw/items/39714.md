## Summary

The PR should stay open because it targets a real Docker sandbox browser bug, but the branch is not mergeable on current main: it edits old core browser paths that no longer exist while the active bridge and tab-operation code now lives behind the plugin SDK and bundled browser plugin runtime, and it lacks inspectable Docker/DooD proof.

## What This Changes

The PR splits sandbox browser bridge/CDP host addressing, adds sandbox browser bridgeHost/cdpHost config plumbing, advertises a container-reachable bridge URL, and routes attach-only remote tab creation through raw CDP.

## Best Possible Solution

Rework the fix on the active plugin SDK and bundled browser runtime paths, preserve separate host-local and container-advertised URLs, handle attach-only tab creation in the active tab ops, document any accepted config knob, and attach redacted Docker/DooD proof.

## Reproduction Assessment

Yes, at source level. Current main still passes the sandbox bridge URL from the local bridge baseUrl into embedded sandbox info, which is the broken loopback shape for Dockerized agent containers.

## Solution Assessment

No. The intent is right, but this branch is not the best current solution because it edits obsolete core browser modules rather than the active plugin SDK/runtime contract and still needs proof plus network-scope review.
