## Summary

Keep open. The reported Docker sandbox browser problem is still source-reproducible on current main, but this draft is not mergeable as-is because key changes target removed legacy `src/browser/**` paths instead of the active browser plugin SDK/runtime path, and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds separate host-local and advertised sandbox browser bridge/CDP URLs, Docker host-gateway plumbing, attach-only tab-opening changes, config fields, and tests.

## Best Possible Solution

Rework the fix on the active browser plugin SDK/runtime path, keep host-local and container-advertised URLs separate, preserve auth/SSRF/CDP ingress controls, update docs for any config knob, and prove it with focused tests plus a Docker/DooD smoke.

## Reproduction Assessment

Yes, at source level: current main still builds sandbox CDP readiness and embedded `browserBridgeUrl` from loopback/local paths. I did not run a live Docker/DooD smoke in this read-only review.

## Solution Assessment

No. The intent is right, but this PR is not the best fix because it edits removed legacy browser files instead of the active plugin SDK/runtime contract and expands browser-control reachability without current-path security review or inspectable runtime proof.
