## Summary

Keep open: the Docker sandbox browser problem is still real, but this draft is not mergeable against current main because it targets obsolete browser paths, lacks inspectable Docker proof, and changes browser-control reachability without the current SDK/runtime security review.

## What This Changes

The branch attempts to split host-local and container-advertised sandbox browser bridge/CDP addresses, add Docker host-gateway/config handling, and route attach-only remote tab creation through raw CDP.

## Best Possible Solution

Rework the fix on the active plugin SDK and browser runtime paths, keep host-local and container-advertised URLs separate, preserve CDP ingress controls, document any accepted config knob, and attach redacted Docker/DooD proof.

## Reproduction Assessment

## Solution Assessment

No. The intent is correct, but this branch is not the best current solution because it edits obsolete src/browser modules instead of the active plugin SDK/runtime contract and still needs proof plus security review.
