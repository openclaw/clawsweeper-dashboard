## Summary

Keep open: the PR targets a real Docker sandbox browser bug, but the draft is not mergeable as-is because current main moved the active browser bridge behind the plugin SDK/browser plugin runtime while this branch still edits removed `src/browser/**` files and lacks inspectable Docker behavior proof.

## What This Changes

The PR adds separate local and advertised sandbox browser bridge/CDP host plumbing, Docker host-gateway reachability, embedded sandbox URL selection, attach-only tab-opening changes, config fields, and tests.

## Best Possible Solution

Rework or replace the draft on the active plugin SDK/browser runtime path, keep host-local and container-advertised URLs separate, resolve sandbox CDP reachability deliberately, document any config knob, and validate with focused tests plus a Docker/DooD smoke.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main still passes loopback/local browser bridge and CDP values into sandbox paths. I did not run a live Docker/DooD smoke because this was a read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. The intent is correct, but the PR is not the best fix because it edits removed legacy browser files instead of the active SDK/browser plugin runtime contract and still needs real behavior proof.
