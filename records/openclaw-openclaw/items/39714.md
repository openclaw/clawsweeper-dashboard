## Summary

Keep open. The PR targets a real sandbox-browser Docker reachability bug, but the branch is draft/dirty and stale against current main: it changes old `src/browser` files while the active bridge and tab paths now live under the plugin SDK and bundled browser plugin runtime.

## What This Changes

The PR separates host-local and container-advertised sandbox browser bridge/CDP addresses, adds Docker host-gateway/config handling, and changes attach-only remote tab creation to avoid Playwright `newPage()`.

## Best Possible Solution

Rework the fix on the active SDK and browser plugin runtime paths, keep host-local and container-advertised URLs separate, preserve CDP ingress controls, document any accepted config knob, and attach redacted Docker/DooD proof.

## Reproduction Assessment

Yes, at source level. Current main still derives sandbox CDP readiness and bridge URLs from `127.0.0.1`/`baseUrl`, which does not work for a Dockerized gateway managing sibling browser containers; no live Docker/DooD smoke was run in this read-only review.

## Solution Assessment

No. The intent is correct, but this branch is not the best current solution because it edits old `src/browser` modules instead of the active plugin SDK/runtime contract and still needs proof plus security review.
