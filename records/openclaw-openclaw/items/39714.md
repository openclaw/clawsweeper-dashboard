## Summary

Keep open: the PR addresses a real Docker sandbox browser problem, but the branch is not mergeable against current main because it patches obsolete `src/browser` paths while the active bridge and tab logic now live behind the plugin SDK/browser runtime, and it lacks inspectable Docker proof plus security review.

## What This Changes

The branch splits sandbox browser bridge/CDP host addresses, adds Docker host-gateway and bridge/CDP host config handling, and routes attach-only remote tab opening through raw CDP.

## Best Possible Solution

Rework the fix on the active plugin SDK and browser runtime paths, keep host-local and container-advertised URLs separate, preserve CDP ingress controls, document any accepted config knob, and attach redacted Docker/DooD proof.

## Reproduction Assessment

Yes, at source level. Current main still derives sandbox CDP readiness and embedded browser bridge URLs from loopback/local `baseUrl`, which is the broken shape for a Dockerized gateway managing sibling browser containers; no live Docker/DooD smoke was run here.

## Solution Assessment

No. The intent is the right problem, but this branch is not the best current solution because it edits obsolete `src/browser` modules instead of the active plugin SDK/runtime contract and still needs proof plus security review.
