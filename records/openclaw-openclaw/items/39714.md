## Summary

Keep this draft PR open: it targets a real Docker sandbox browser bug, but the branch is stale against current main because the bridge and tab-operation changes are still applied to removed legacy `src/browser/**` paths instead of the active browser plugin SDK/runtime path.

## What This Changes

The PR adds host-local versus container-advertised sandbox browser bridge/CDP plumbing, Docker host-gateway handling, attach-only tab-opening changes, config fields, and regression tests.

## Best Possible Solution

Rework or replace the branch on the active browser plugin SDK/runtime path, keeping host-local and container-advertised URLs separate, resolving Dockerized CDP reachability deliberately, updating docs for any config knob, and proving it with focused tests plus a Docker/DooD smoke.

## Reproduction Assessment

Yes, at source level: current main still builds sandbox CDP readiness and embedded `browserBridgeUrl` from loopback/local paths. I did not run a live Docker/DooD smoke during this read-only review.

## Solution Assessment

No. The intent is correct, but this PR is not the best current fix because it edits removed legacy browser files instead of the active SDK facade and bundled browser plugin runtime.
