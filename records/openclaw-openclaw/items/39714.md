## Summary

Keep open: the Docker sandbox browser problem is still real, but this draft is not mergeable on current main because it edits stale legacy browser paths while the active bridge and tab operations now live behind the browser plugin SDK/runtime facade; it also lacks inspectable after-fix proof and needs security review for browser-control reachability changes.

## What This Changes

The PR adds advertised sandbox browser bridge/CDP host plumbing, Docker host-gateway reachability, embedded sandbox URL selection, attach-only tab-opening changes, config/schema fields, and regression tests.

## Best Possible Solution

Rework the fix on the active browser plugin SDK/runtime path, keep host-local and container-advertised URLs separate, preserve host-side loopback control, handle Dockerized CDP reachability deliberately, document any accepted config knob, and validate with focused tests plus redacted Docker/DooD proof.

No. The intent is correct, but this branch is not the best current solution because it edits removed legacy browser files instead of the active plugin SDK/runtime contract and still needs proof plus security review.
