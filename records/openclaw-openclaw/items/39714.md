## Summary

Keep open. The draft is aimed at a real Docker sandbox browser problem, but it still needs a current-path rewrite because current main routes the bridge through the plugin SDK and bundled browser runtime while this PR edits obsolete browser paths.

## What This Changes

The PR splits sandbox browser bridge/CDP host addresses, adds bridge/CDP host config handling, advertises a container-reachable bridge URL, and routes attach-only tab creation through raw CDP.

## Best Possible Solution

Rework the fix on the active plugin SDK and bundled browser runtime paths, preserve separate host-local and container-advertised URLs, document any accepted config knob, and attach redacted Docker/DooD proof.

## Reproduction Assessment

Yes, at source level. Current main still derives sandbox CDP readiness and embedded browser bridge URLs from loopback/local baseUrl, which is the broken shape for a Dockerized gateway managing sibling browser containers.

## Solution Assessment

No. The intent is right, but this branch is not the best current solution because it edits obsolete src/browser modules instead of the active plugin SDK/runtime contract and still needs proof plus security review.
