## Summary

This PR should stay open as a draft/replacement candidate: the Docker sandbox browser problem is still visible in current `main`, but this branch targets obsolete `src/browser/**` paths and does not port the contract to the active browser plugin SDK/runtime path.

## What This Changes

The PR adds separate local and advertised sandbox browser bridge/CDP hosts, Docker host-gateway reachability, embedded sandbox URL selection, attach-only tab-opening changes, config fields, and related tests.

## Best Possible Solution

Rework the fix on the active browser plugin SDK/runtime path, keep host-local and container-advertised URLs separate, resolve Dockerized CDP reachability deliberately, document any new config knob, and validate with focused tests plus a Docker/DooD smoke.

## Reproduction Assessment

Yes, at source level: current `main` still builds embedded sandbox browser URLs and sandbox CDP readiness from local loopback paths. I did not run a live Docker/DooD smoke because this review is read-only.

## Solution Assessment

No. The intent is correct, but this branch is not the best solution because the active browser bridge now crosses `src/plugin-sdk/browser-bridge.ts` and `extensions/browser/src/browser/**`, while the PR edits obsolete `src/browser/**` paths and needs a deliberate SDK/runtime security review.
