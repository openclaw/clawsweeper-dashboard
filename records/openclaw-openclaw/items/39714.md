## Summary

Keep open: the Dockerized sandbox-browser problem is still valid, but this draft is not mergeable because it targets legacy browser paths while current main uses the plugin SDK/browser runtime contract, and it still needs inspectable Docker proof plus scoped security review.

## What This Changes

This PR adds separate local and advertised sandbox browser bridge/CDP hosts, Docker host-gateway handling, sandbox browser config fields, embedded sandbox URL selection, attach-only tab-opening changes, and tests.

## Best Possible Solution

Rework the fix on the active SDK and browser plugin runtime paths, keep host-local and container-advertised URLs separate, preserve CDP ingress controls, document any accepted config knob, and attach redacted Docker/DooD proof.

## Reproduction Assessment

Yes, at source level: current main's sandbox CDP readiness and bridge URLs are still derived from 127.0.0.1/baseUrl, which is the broken shape for a Dockerized gateway managing sibling containers. No live Docker/DooD smoke was run in this read-only review.

## Solution Assessment

No. The intent is correct, but this branch is not the best current solution because it edits legacy browser modules instead of the active plugin SDK/runtime contract and still needs proof plus security review.
