The PR adds separate local and advertised sandbox browser bridge URLs, Dockerized CDP host resolution, attach-only tab-opening behavior, sandbox browser config fields, tests, Docker Compose host-gateway mapping, and secrets-baseline churn.

## Best Possible Solution

Rework the fix through `src/plugin-sdk/browser-bridge.ts` and `extensions/browser/src/browser/**`, preserve distinct host-local and container-advertised URLs, keep CDP handling scoped, document any approved config knob, and attach redacted Docker/DooD proof.
