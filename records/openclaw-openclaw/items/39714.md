## Summary

Keep open: the underlying Docker sandbox browser reachability problem is still visible from current main, but this draft is not mergeable because it targets pre-refactor core browser paths while main routes browser bridge/control through the plugin SDK and browser plugin runtime.

## What This Changes

The PR branch adds advertised sandbox browser bridge/CDP host plumbing, embedded sandbox URL selection, attach-only remote tab handling, config/schema/help updates, tests, Docker Compose changes, and secrets-baseline churn.

## Best Possible Solution

Rework the draft on the active plugin path: add a deliberate advertised/container-reachable bridge URL contract through src/plugin-sdk/browser-bridge.ts and the browser runtime API, keep host-local bridge use separate, resolve sandbox CDP host/port separately, drop incidental baseline churn, and add focused Docker/non-Docker regression coverage plus docs for any new config knob.

## Reproduction Assessment

Yes at source level: current main still builds sandbox CDP and embedded browserBridgeUrl from loopback/local bridge paths. A live Docker reproduction was not attempted because this was a read-only review.

## Solution Assessment

No. The intent is valid, but the patch is not the best current fix because it targets old core browser files instead of the active plugin SDK/runtime contract and mixes in security-sensitive networking changes.
