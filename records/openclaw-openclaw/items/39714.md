## Summary

Keep open for maintainer handling: the Docker sandbox browser problem is still source-reproducible, but this draft is not mergeable because it targets removed core browser paths instead of the active plugin SDK/browser runtime, lacks inspectable real Docker proof, and changes browser-control network reachability.

## What This Changes

The PR adds separate local and advertised sandbox browser bridge/CDP host handling, Docker host-gateway reachability, embedded sandbox URL selection, attach-only tab opening changes, config fields, and tests.

## Best Possible Solution

Rework the fix through src/plugin-sdk/browser-bridge.ts and the bundled browser runtime, keeping host-local and container-advertised bridge/CDP targets separate, with focused tests, docs for any config knob, and Docker/DooD proof.

## Solution Assessment

No. The intent is right, but this branch is not the best fix because it modifies removed core browser modules; the safer path is a current plugin SDK/runtime rewrite with explicit reachability and security review.
