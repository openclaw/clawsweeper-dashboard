## Summary

Keep open. The PR addresses a real Docker/DooD sandbox browser reachability gap, but current main still lacks the advertised/container-reachable bridge contract and this draft targets obsolete browser paths that are no longer the active runtime surface.

## What This Changes

The PR branch adds advertised sandbox browser bridge/CDP host plumbing, embedded sandbox URL selection, attach-only tab handling, config/schema/help updates, tests, and Docker/secrets-baseline changes.

## Best Possible Solution

Replace or substantially rework the draft on the active plugin path: add a deliberate advertised/container-reachable bridge URL contract through src/plugin-sdk/browser-bridge.ts and the browser plugin runtime API, keep the gateway-local bridge URL separate, use the advertised URL only for sandboxed consumers, resolve sandbox CDP host/port separately, drop incidental baseline churn, and add focused Docker/non-Docker regression coverage plus docs for any new config knob.

## Reproduction Assessment

Yes for a code-level reproduction: current main still constructs sandbox CDP and embedded bridge URLs from loopback/baseUrl paths. Live Docker reproduction was not attempted because this review was read-only.

## Solution Assessment

No. The intent is right, but the proposed patch is not the best current fix because it targets the old src/browser surface and mixes in security-sensitive Docker/bridge exposure; the safer solution is a current-path SDK/runtime rewrite.
