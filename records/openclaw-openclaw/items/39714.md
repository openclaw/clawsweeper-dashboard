## Summary

Keep open. The Docker sandbox browser reachability problem is still visible in current main, but this draft is not mergeable because it edits removed legacy `src/browser/**` paths instead of the active plugin SDK and bundled browser runtime path.

## What This Changes

The branch adds Docker sandbox browser listen/advertise URL and CDP host plumbing, new sandbox browser bridge/CDP host config fields, attach-only tab-opening changes, tests, Docker Compose host-gateway mapping, and secrets-baseline churn.

## Best Possible Solution

Rework the fix through `src/plugin-sdk/browser-bridge.ts` and `extensions/browser/src/browser/**`, preserve separate host-local and container-advertised URLs, keep CDP handling scoped, document any approved config knob, and attach redacted Docker/DooD proof.

## Reproduction Assessment

Yes, at source level. Current main still starts the sandbox browser bridge as loopback-only and embeds that same local URL into sandbox info for the agent container.

## Solution Assessment

No. The intent is right, but this branch is not the best current solution because it patches removed legacy browser modules instead of the active plugin SDK/runtime contract and adds network/config changes without accepted owner review or real Docker proof.
