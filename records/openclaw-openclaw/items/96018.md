Keep open: this is a focused bug-fix PR, but the prompt-only exemption does not unblock the actual `video_generate` call because the execute path still returns active task status through the duplicate guard; after-fix real behavior proof is also missing.

## What This Changes

The PR extracts the media-delivery progress text into a shared constant, adds an `excludeDeliveringCompletion` option to active media task lookup, uses it in prompt-context generation, and adds mocked unit tests.

## Best Possible Solution

Make the generated-but-delivering phase non-blocking consistently for both prompt guidance and the actual `video_generate` duplicate/tool guard, while preserving same-request duplicate protection and delivery-confirmation terminal handling.

## Reproduction Assessment

Yes. Source inspection shows current main keeps the task active through `wakeTaskCompletion`, and the unmodified `video_generate` execute-path duplicate guard can still return active task status during that window.

## Solution Assessment

No. The patch is a plausible partial mitigation, but the best fix must carry the delivering-phase exception into the actual tool guard or introduce a typed non-blocking phase used by both prompt and tool paths.
