## Summary

Keep open. Current main still defaults the agent-visible bootstrap truncation warning to `once` and suppresses repeated truncation signatures; `openclaw doctor` now reports bootstrap file size problems, but the requested default-always behavior or dedicated gateway-startup diagnostic is not implemented.

## What This Changes

The issue requests louder bootstrap-injected file truncation warnings, either by defaulting `bootstrapPromptTruncationWarning` to `always` or by surfacing truncation through doctor and gateway-startup diagnostics.

## Best Possible Solution

Pick a warning policy explicitly, then either change the default with docs/tests/changelog or keep prompt dedupe and add a persistent gateway-level diagnostic that makes truncation visible without prompt noise.

## Reproduction Assessment

Yes for the reported current behavior: source and tests show the default mode is `once`, repeated signatures are suppressed, and per-run log warnings are also deduped by session label. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Unclear. Defaulting to `always` would solve visibility but changes a documented warning/noise tradeoff; a gateway-startup or status diagnostic may be safer if maintainers want operational visibility without repeated prompt injection.
