## Summary

Keep open. Current main and the latest release still default the agent-visible bootstrap truncation warning to `once`; repeated signatures are suppressed, `openclaw doctor` has partial bootstrap-size coverage, and no dedicated gateway-startup diagnostic was found. Related open PRs cover adjacent status/logging/state issues but do not close or fully supersede this warning-policy request.

## What This Changes

The issue requests louder default handling for bootstrap-injected file truncation, either by changing the warning mode default to `always` or by surfacing truncation through doctor/gateway startup diagnostics.

## Best Possible Solution

Pick the warning policy explicitly; if maintainers want operational visibility without prompt noise, add a persistent gateway/status diagnostic using the shared bootstrap budget analysis and align docs/tests with that choice.

## Reproduction Assessment

Yes, by source inspection: current main defaults to `once`, once-mode suppresses previously seen truncation signatures, and tests assert that behavior. I did not run a live session because this review had to keep the checkout read-only.

## Solution Assessment

Unclear. Defaulting to `always` would address visibility directly, but it changes a documented noise/token tradeoff; an operational gateway/status diagnostic may be the more maintainable surface if repeated prompt warnings are not desired.
