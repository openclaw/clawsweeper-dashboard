## Summary

Keep open. Current main still uses and documents `bootstrapPromptTruncationWarning: "once"`; `openclaw doctor` has partial bootstrap-size coverage, but the requested default-`always` policy or dedicated gateway-startup diagnostic is not implemented.

## What This Changes

Requests making bootstrap truncation warnings visible on every affected run by default, or surfacing truncation through doctor/gateway-startup diagnostics.

## Best Possible Solution

Define the intended operational surface, then either change the documented default with tests/changelog or add a persistent gateway/status diagnostic while keeping prompt-level dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: use default config with a bootstrap file over the per-file cap, observe the first warning for a signature, then reuse the seen signature in once mode and the warning is suppressed.

## Solution Assessment

Unclear. Defaulting to `always` directly addresses visibility but changes a documented warning/noise tradeoff; a gateway/status diagnostic may be the narrower operational fix if maintainers want prompt-cache and token stability.
