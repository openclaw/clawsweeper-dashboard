## What This Changes

## Best Possible Solution

Add the setting consistently across config schema/normalization/docs, Talk runtime prompts, and the public `talk.config` protocol/result contract, then merge only after redacted real Talk runtime proof.

## Reproduction Assessment

Yes, from source inspection: current main's strict Talk realtime schema omits `instructions`, and the realtime prompt builders call `buildRealtimeInstructions()` without configured input. I did not run runtime tests in this read-only review.

## Solution Assessment

No as proposed: appending configured text to the built-in prompt is a reasonable direction, but the patch must also update the public `talk.config` result schema and contract coverage, and the new config surface needs maintainer acceptance plus real runtime proof.
