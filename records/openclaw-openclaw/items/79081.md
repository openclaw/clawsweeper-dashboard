## Summary

Keep open: the change is directionally plausible, but the patch currently misses the public Talk config protocol contract and the external PR only lists validation commands, not after-fix real behavior proof.

## What This Changes

The PR adds `talk.realtime.instructions` to Talk config schema, normalization, docs, generated config hashes, and realtime browser/gateway-relay prompt construction.

## Best Possible Solution

Add the setting consistently across runtime config, schema/help/docs, the public gateway protocol result schema, and contract tests, then require redacted real Talk runtime proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main's strict `OpenClawSchema` Talk realtime schema omits `instructions`, and the realtime prompt builder has no configured-instructions input. I did not run tests because this review is read-only.

## Solution Assessment

No as proposed: appending configured text to the built-in prompt is a reasonable implementation direction, but the patch must also update the public `talk.config` protocol schema/contract and include real runtime proof.
