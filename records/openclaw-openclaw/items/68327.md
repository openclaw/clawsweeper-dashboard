## Summary

Keep open. Current main has absorbed several adjacent pieces, but the PR's central auth/secrets latency shortcut and async-lock timeout are not safely implemented on main, and the branch still has merge-blocking functional regressions.

## What This Changes

The PR branch changes auth-profile secrets-runtime loading, async-lock behavior, gateway discovery, startup bootstrap ordering, Windows UI command spawning, and profiling logs to reduce observed gateway startup latency.

## Best Possible Solution

Land a narrower current-main change that creates a secrets-preflight-specific no-external-auth loader, keeps shared OAuth/recovery loaders overlay-capable, adds an async-lock timeout with focused tests, removes or gates temporary profiling through the existing startup trace, and leaves unrelated pricing/channel-latency work to the newer dedicated reports.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the PR changes a shared loader used by OAuth recovery paths, and the PR patch visibly adds unconditional stderr profiling logs. The performance claim itself needs a fresh `pnpm test:startup:gateway -- --runs 5 --warmup 1` comparison on current main.

## Solution Assessment

No. The PR identifies a plausible bottleneck, but changing `loadAuthProfileStoreForSecretsRuntime` globally and adding ungated production diagnostics is not the narrowest maintainable fix; a dedicated secrets-preflight loader split with tests is safer.
