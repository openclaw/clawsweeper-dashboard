## Summary

Keep open: current main still drops the silent-empty opt-in before the embedded cron runner, and https://github.com/openclaw/openclaw/pull/81374 is an open implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

Cron jobs with `delivery.mode: "none"` should pass `allowEmptyAssistantReplyAsSilent` to `runEmbeddedPiAgent` so empty assistant replies are treated as silent success.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81374 or an equivalent narrow fix that forwards the resolved `deliveryPlan.mode === "none"` state into `runEmbeddedPiAgent` with regression coverage.

## Reproduction Assessment

Yes, source-level. Current main resolves `delivery.mode: "none"`, but the embedded cron call does not pass the silent-empty opt-in and the embedded classifier treats an omitted value as false.

## Solution Assessment

Yes for the implementation direction. The narrow maintainable fix is to carry the already-resolved cron no-delivery state to the embedded runner instead of relaxing incomplete-turn classification globally.
