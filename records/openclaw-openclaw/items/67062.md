Request to add `channels.qqbot.mediaAllowHosts` so QQ Bot media downloads can merge operator host patterns with the default SSRF hostname allowlist.

## Best Possible Solution

Add a QQ Bot-owned `mediaAllowHosts` config surface with schema, types, manifest metadata, docs, and focused tests, then normalize and merge configured hosts into the existing guarded SSRF policy while preserving HTTPS-only and private-network protections.

## Solution Assessment

Unclear until maintainers approve the SSRF semantics. The maintainable direction is a QQ Bot plugin-owned config merge through the existing SSRF policy contract, not a broad core media-fetch rewrite.
