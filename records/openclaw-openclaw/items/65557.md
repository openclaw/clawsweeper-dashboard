## Summary

Keep open. Current main has a session model picker and global configured model visibility, including `provider/*`, but it still lacks account-scoped provider/model allowlists, per-account defaults, admin UI policy, and trusted server-side account enforcement; the related per-agent PR is narrower and still open.

## What This Changes

The issue requests per-session or per-account provider/model selection plus admin-controlled provider/model allowlists, defaults, and server-side enforcement.

## Best Possible Solution

Define one trusted server-side effective provider/model policy that composes global, provider-wildcard, per-agent, and per-account restrictions, then enforce it consistently in catalog listing, session patching, commands, Control UI, admin configuration, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives high confidence that current main lacks account-scoped policy inputs in model catalog listing and session model validation.

## Solution Assessment

Unclear until maintainers choose account identity, inheritance, wildcard semantics, and enforcement boundaries. The maintainable direction is a shared server-side policy resolver first, with UI filtering only as presentation.
