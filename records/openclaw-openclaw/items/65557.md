## Summary

Keep this open. Current main has a session chat model picker and a global `agents.defaults.models` allowlist, but it still does not implement account-scoped provider/model allowlists, per-account defaults, wildcard policy, or account-aware server-side rejection. The related #7424 is open but narrower per-agent allowlist work, so it does not supersede this broader admin/account policy request.

## What This Changes

Add provider/model selection with persisted session or account preference plus admin-controlled per-account provider/model allowlists and defaults enforced by the API.

## Best Possible Solution

Maintain one explicit provider/model policy design that composes global defaults, per-agent configuration, and account or user restrictions, then enforce it in catalog listing and every model override path before exposing admin UI controls and docs.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. The verification path is high-confidence source/docs/test inspection: current main exposes only the session picker plus global configured-model allowlisting, not account-scoped admin policy.

## Solution Assessment

Unclear until maintainers define the account identity and precedence model. The narrow maintainable direction is shared server-side policy enforcement reused by `models.list`, `sessions.patch`, chat/run overrides, and OpenAI-compatible model override paths, with UI filtering as a presentation layer only.
