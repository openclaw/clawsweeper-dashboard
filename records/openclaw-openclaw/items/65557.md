## Summary

Keep open. Current main now covers session model selection and global model/provider visibility via `agents.defaults.models` including `provider/*`, but it still lacks the requested account-scoped provider/model allowlists, per-account defaults, and trusted account-aware server-side enforcement.

## What This Changes

Request account/session provider and model selection with admin-controlled allowlists, defaults, UI filtering, and server-side rejection for disallowed providers or models.

## Best Possible Solution

Define a single server-side effective provider/model policy that composes global defaults, per-agent configuration, and account restrictions, then reuse it for catalog listing, session patching, slash commands, tools, Control UI, admin config, and docs.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. Source inspection gives a high-confidence feature-gap check: current model visibility is global and lacks account-scoped policy inputs.

## Solution Assessment

Unclear until maintainers choose account identity, inheritance, wildcard semantics, and enforcement boundaries. The maintainable direction is a shared server-side policy resolver first, with UI filtering only as presentation.
