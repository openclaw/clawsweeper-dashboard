## Summary

Keep open. Current main covers the session-level picker and global `agents.defaults.models` visibility, including `provider/*`, but it still does not define account-scoped provider/model allowlists, per-account defaults, or trusted server-side account enforcement; related per-agent and provider-wildcard work does not supersede this broader admin policy request.

## What This Changes

Request account-aware provider/model selection with admin-controlled per-account allowlists, defaults, and server-side rejection of disallowed model choices.

## Best Possible Solution

Define one trusted server-side effective provider/model policy that composes global, provider-wildcard, per-agent, and per-account restrictions, then enforce it consistently in catalog listing, session patching, commands, Control UI, admin configuration, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and authorization-policy request. Source inspection gives high confidence that current main lacks account-scoped policy inputs in model catalog listing and session model validation.

## Solution Assessment

Unclear until maintainers choose account identity, inheritance, wildcard semantics, and enforcement boundaries. The maintainable direction is a shared server-side policy resolver first, with UI filtering only as presentation.
