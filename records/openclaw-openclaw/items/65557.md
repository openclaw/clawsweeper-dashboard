## Summary

Keep open. Current main supports session model selection and global model/provider visibility through `agents.defaults.models` including `provider/*`, but it still lacks account-scoped provider/model allowlists, per-account defaults, and account-aware server-side enforcement; the related provider wildcard fix and the narrower per-agent allowlist issue do not supersede this request.

## What This Changes

Request provider/model selection per session or account with admin-controlled provider/model allowlists, defaults, and server-side enforcement.

## Best Possible Solution

Define one server-side effective provider/model policy that composes global, agent, and account restrictions, then use it for model listing, session patching, commands, Control UI, admin config, and docs.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. Source inspection gives a high-confidence feature-gap check: current model visibility and validation are global and lack account-scoped policy inputs.

## Solution Assessment

Unclear until maintainers choose account identity, inheritance, wildcard semantics, and enforcement boundaries. The maintainable direction is a shared server-side policy resolver first, with UI filtering only as presentation.
