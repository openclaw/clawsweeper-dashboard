## Summary

Keep open. Current main has a session model picker and global `agents.defaults.models` visibility, including `provider/*`, but it still lacks per-account provider/model allowlists, per-account defaults, and account-aware server-side enforcement; the related per-agent allowlist and provider-wildcard reports do not cover this account/admin policy request.

## What This Changes

Request provider/model selection with per-session or per-account persistence plus admin-controlled per-account provider/model allowlists and defaults enforced server-side.

## Best Possible Solution

Define one server-side effective provider/model policy that composes global, agent, and account restrictions, then apply it consistently to catalog listing, session patching, commands, Control UI, admin config, protocol docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and authorization-policy request. Source inspection gives a high-confidence feature-gap check because current catalog visibility and model override validation lack account-scoped policy inputs.

## Solution Assessment

Unclear until maintainers choose account identity, inheritance, wildcard semantics, and enforcement boundaries. The maintainable direction is a shared server-side policy resolver first, with UI filtering only as presentation.
