## Summary

Keep open: the underlying CLI context-engine bug is real and this PR is directionally useful, but the patch still has blocking lifecycle issues and the external real-behavior proof is only mocked test/build evidence.

## What This Changes

The PR resolves a non-legacy context engine during CLI preparation, finalizes successful CLI attempts through the shared context-engine lifecycle, disposes the engine during cleanup, and adds focused CLI context-engine regression tests.

## Best Possible Solution

Land a CLI lifecycle fix that finalizes from canonical transcript messages, keeps context-engine lifetime safe through maintenance and prepare failures, then attach redacted live CLI/LCM proof before merge.

## Reproduction Assessment

Yes. Source inspection and the linked A/B report give a high-confidence path: configure a non-legacy context engine, route a successful turn through claude-cli/runCliAgent, and current main has no CLI finalizer or turn-maintenance hook.

## Solution Assessment

No. Reusing the shared lifecycle is the right direction, but this implementation needs to preserve canonical transcript text and fix context-engine lifetime/cleanup before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the canonical transcript prompt:** `src/agents/cli-runner.ts:120`
  - body: This fallback ingests `runParams.prompt` whenever callers do not set `transcriptPrompt`. The main agent-command CLI path passes the runtime-shaped `effectivePrompt` to `runCliAgent` and later persists the clean `body`/`transcriptBody`, so normal CLI runs can put a different user message into the context engine than the JSONL transcript. Pass the canonical transcript text through this path or finalize from the persisted transcript snapshot.
  - confidence: 0.88
- **[P2] Keep engines alive for deferred maintenance:** `src/agents/cli-runner.ts:614`
  - body: `finalizeHarnessContextEngineTurn` can schedule turn maintenance in the background when the engine requests it, but this cleanup immediately disposes the same engine instance. Engines that close database handles or other resources in `dispose()` can then fail the queued maintenance, leaving the reported lifecycle gap only partially fixed. Use a lifetime-safe engine for deferred work or run CLI turn maintenance before disposal.
  - confidence: 0.78
- **[P2] Dispose the engine when CLI preparation fails:** `src/agents/cli-runner/prepare.ts:137-142`
  - body: The branch resolves the context engine early in `prepareCliRunContext`, but the rest of preparation still performs fallible auth, bootstrap, MCP, backend preparation, and transcript checks before returning a context. If any of those throw, `runPreparedCliAgent` never runs and the non-legacy engine is not disposed. Wrap the remaining preparation in cleanup or move resolution under an existing runner-level `finally`.
  - confidence: 0.82
