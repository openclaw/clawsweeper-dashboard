## Summary

Keep open: the PR targets a real current-main CLI context-engine lifecycle gap, but it still has lifecycle cleanup bugs and only mock/test proof, not after-fix proof from a real claude-cli plus non-legacy context-engine setup.

## What This Changes

The PR resolves a non-legacy context engine for CLI runs, finalizes successful CLI turns through the shared harness lifecycle, disposes the engine, and adds CLI-runner regression tests.

## Best Possible Solution

Land a focused CLI lifecycle fix that reuses the shared finalizer while tying context-engine lifetime to the full prepare/run/deferred-maintenance path, then attach redacted real runtime proof.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence path: a successful claude-cli run has no context-engine finalizer or turn-maintenance call, and the linked issue provides live A/B evidence; I did not run a live claude-cli setup in this read-only review.

## Solution Assessment

No. Reusing the shared finalizer is the right direction, but this PR still needs safer context-engine cleanup across prepare failure and deferred background maintenance before it is the best maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Dispose engines if CLI preparation fails:** `src/agents/cli-runner/prepare.ts:135-141`
  - body: Resolving the context engine here can allocate plugin-owned resources before the rest of prepare finishes. If a later auth/backend/transcript/prompt-build step throws, prepareCliRunContext never returns and runPreparedCliAgent's finally block never runs, so the engine is left undisposed.
  - confidence: 0.87
- **[P2] Keep engines alive for background maintenance:** `src/agents/cli-runner.ts:612`
  - body: For engines with background turn maintenance, finalizeHarnessContextEngineTurn can queue work that uses this same engine after it returns. The new finally block disposes the engine immediately after the CLI attempt, so queued maintenance can run against closed plugin resources.
  - confidence: 0.82
