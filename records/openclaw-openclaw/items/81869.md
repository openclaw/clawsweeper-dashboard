## Summary

Keep this PR open: it addresses a real CLI context-engine lifecycle gap, but current head still has a blocking lifecycle cleanup problem around resolved context-engine instances.

## What This Changes

The PR resolves a non-legacy context engine during CLI preparation and routes successful CLI turns through shared bootstrap, history snapshot, afterTurn/ingest fallback, and maintenance paths with regression tests.

## Best Possible Solution

Land this PR or a maintainer-adjusted equivalent after adding deterministic cleanup for the resolved CLI context engine without regressing deferred maintenance ownership.

## Reproduction Assessment

Yes, source-level: current main's successful CLI runner returns without invoking the shared context-engine finalizer, and the linked issue includes a concrete CLI-vs-embedded live A/B. I did not rerun a credentialed claude-cli plus LCM setup in this read-only pass.

## Solution Assessment

No, not yet: the shared finalizer direction is the right fix, but current head should also release the resolved context-engine instance after any deferred maintenance it owns. The safer path is to mirror the embedded runner's deterministic cleanup boundary while preserving the new CLI lifecycle wiring.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Dispose the resolved context engine after CLI runs:** `src/agents/cli-runner/prepare.ts:520`
  - body: This stores a freshly resolved non-legacy `contextEngine` on each prepared CLI run, but `runPreparedCliAgent` only runs `preparedBackend.cleanup` in its finalizer. `resolveContextEngine` invokes the registered factory for the selected engine, and the embedded/compaction paths dispose their resolved engines; without a matching CLI cleanup path, engines that own DB handles, clients, timers, or file descriptors can leak resources across repeated CLI turns. Please dispose the resolved engine after any deferred turn maintenance owned by this run settles.
  - confidence: 0.82
