## Summary

Keep open: the underlying CLI context-engine bug is source-backed, but this PR still has a prepare-failure cleanup regression and only test/build proof rather than real claude-cli plus non-legacy context-engine proof.

## What This Changes

The PR resolves a non-legacy context engine during CLI preparation, finalizes successful CLI attempts through the shared harness lifecycle, disposes the engine, and adds CLI context-engine regression tests.

## Best Possible Solution

Keep the focused CLI lifecycle fix, but bind context-engine lifetime to the full prepare/run path and land only with redacted live claude-cli plus non-legacy context-engine proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI runner has no context-engine finalization path, and the linked issue provides a concrete live A/B with claude-cli versus embedded Codex on the same session.

## Solution Assessment

No. Reusing the shared finalizer is the right direction, but resolving a disposable context engine inside prepare without a prepare-failure cleanup path is not the narrowest maintainable fix.
