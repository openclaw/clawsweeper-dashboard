## Summary

Keep open: this is a valid source-backed CLI/context-engine bug fix, but the current PR branch still has a blocking lifecycle cleanup bug and lacks required real behavior proof for the external runtime path.

## What This Changes

The PR adds CLI-runner context-engine resolution, successful-turn finalization through the shared harness lifecycle, disposal, and focused regression tests for afterTurn, ingest fallback, maintenance, failed attempts, and cleanup.

## Best Possible Solution

Keep the focused CLI lifecycle direction, but bind context-engine lifetime to every prepare/run exit path and require redacted live runtime proof before landing.

## Reproduction Assessment

Yes. Current main's CLI runner has no context-engine finalizer or turn-maintenance path, and the linked issue gives a concrete live A/B showing LCM frozen on claude-cli while embedded Codex advanced the same session.

## Solution Assessment

No. Reusing the shared finalizer is the right direction, but resolving a disposable context engine inside prepare without cleanup on later prepare failures is not yet the narrowest maintainable fix.
