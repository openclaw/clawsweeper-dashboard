## Summary

Keep open: the PR targets a valid CLI context-engine lifecycle gap, but it is not merge-ready because real behavior proof is missing and the patch leaves a context-engine cleanup hole during prepare failures.

## What This Changes

This PR adds CLI-runner context-engine resolution, successful-turn finalization, disposal, and regression coverage for afterTurn, ingest fallback, maintenance, failure skip, and cleanup.

## Best Possible Solution

Keep the focused CLI lifecycle fix, but bind context-engine lifetime to the whole prepare/run path and require a live claude-cli plus non-legacy context-engine proof before landing.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI runner has no context-engine finalization path, and the linked issue provides a concrete live A/B with claude-cli versus embedded Codex on the same session.

## Solution Assessment

No. Reusing the shared finalizer is the right direction, but resolving a disposable context engine inside prepare without a prepare-failure cleanup path is not the narrowest maintainable fix.
