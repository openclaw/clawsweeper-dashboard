## Summary

Keep open: the linked bug is real and the shared-finalizer direction is appropriate, but this PR still has a P2 lifecycle cleanup bug and the external real-behavior proof is only tests/builds, not a live claude-cli plus non-legacy context-engine run.

## What This Changes

The branch adds context-engine resolution to CLI preparation, finalizes successful CLI attempts through the shared harness lifecycle, disposes engines after runs, and adds CLI regression tests.

## Best Possible Solution

Reuse the shared finalizer for successful CLI turns while binding resolved engine cleanup to the full prepare/run lifetime, then land with focused regression coverage and redacted live CLI plus LCM proof.

## Reproduction Assessment

Yes. Source inspection on current main shows runPreparedCliAgent has no context-engine finalizer or maintenance path, and the linked issue provides a live A/B with claude-cli frozen while embedded Codex advances LCM.

## Solution Assessment

No. The shared finalizer is the right direction, but resolving a disposable engine inside prepare without a prepare-failure cleanup path is not the narrowest maintainable implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Dispose the engine when CLI preparation fails:** `src/agents/cli-runner/prepare.ts:134-140`
  - body: Resolving the context engine here can allocate plugin-owned resources before later auth, backend, history, or prompt preparation steps finish. The new cleanup only runs after prepareCliRunContext returns and runPreparedCliAgent enters its finally block, so any later prepare failure leaks the resolved engine; wrap the remaining prepare work in cleanup or move resolution into a lifetime that always finalizes.
