The issue requests a plugin SDK extension point for `CliBackendPlugin` JSONL backends to map provider-native line events into OpenClaw assistant, thinking, session, result, and display-only tool-card events.

## Best Possible Solution

Keep this issue open as the API-design venue, and close it only after the linked candidate PR lands or maintainers choose a different explicit plugin SDK contract.

## Reproduction Assessment

Yes for a source-level capability gap: current `main` has no plugin-owned JSONL parser hook and only built-in Claude/Gemini JSONL event dispatch. This is a feature/API request, not a broken existing documented behavior.

## Solution Assessment

Yes on direction, pending API approval: a generic backend-owned parser hook is a better boundary than requiring third-party CLIs to impersonate Claude or Gemini. The exact normalized event contract should be settled through the linked PR review.
