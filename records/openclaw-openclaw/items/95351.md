## Summary

Keep open: current main and the latest release still lack a plugin-owned JSONL parser hook, and the open linked pull request is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests a public `CliBackendPlugin` hook so JSONL CLI backends can map provider-native line events into OpenClaw text, thinking, session, result, and native display tool-card events.

## Best Possible Solution

Keep this issue open as the canonical API-design item, then close it only after the linked implementation lands on main or maintainers explicitly choose a different direction.

## Reproduction Assessment

Yes. Source inspection shows current main has no `parseJsonlEvent`-style hook on `CliBackendPlugin`, and JSONL tool-event parsing is limited to built-in Claude/Gemini dialect handling.

## Solution Assessment

Yes at the owner boundary, but not yet as a settled API. A backend-owned parser hook fits the plugin API direction; the exact event contract should be accepted, narrowed, or rejected through the linked pull request review.
