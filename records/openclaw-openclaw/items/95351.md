## Summary

Keep open: current main still has only built-in Claude/Gemini JSONL parser paths, so the requested generic plugin-owned JSONL parser or display-only tool-event hook remains missing.

## What This Changes

The issue requests a generic `CliBackendPlugin` JSONL parser or display-only tool-event hook so third-party CLI backends can emit native tool-card events without mimicking Claude or Gemini dialects.

## Best Possible Solution

Add a narrow, documented SDK/runtime hook only after maintainers choose the normalized event shape, with parser/runner tests and sanitization for display-only tool events.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no plugin-owned JSONL parser hook and explicitly treats generic JSONL as not supporting tool events. No live runtime repro was run because this is a feature/API gap, not a broken documented behavior.

## Solution Assessment

Unclear: a generic display-only parser hook fits the plugin boundary if maintainers want third-party CLI parity, but the exact SDK shape, normalized event contract, and safety boundary need maintainer design approval.
