Yes. Source inspection shows current main has no plugin-owned JSONL parser hook on CliBackendPlugin, and the JSONL parser is built around current backend config/provider dialect checks rather than plugin-provided line parsing.

## Solution Assessment

Unclear until maintainer API review. A backend-owned parser hook is a plausible clean boundary, but the public normalized event contract and display-only tool-card behavior need maintainer sponsorship before this is the best accepted solution.
