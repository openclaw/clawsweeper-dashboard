## Summary

Keep open: current main and the latest release still lack the plugin-owned JSONL parser hook, and the linked implementation PR is open rather than merged.

## What This Changes

Request a public `CliBackendPlugin` JSONL line parser/display-event hook so third-party CLI backends can emit native tool-card events without impersonating built-in Claude/Gemini dialects.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/95386 as the API-design path, then close this issue only after the accepted hook reaches main or maintainers explicitly reject the direction.

## Reproduction Assessment

Yes, at source level. Current main exposes only built-in Claude/Gemini JSONL parsing and no `CliBackendPlugin` parser hook for plugin-owned line parsing or display-only tool-card events.

## Solution Assessment

Unclear until maintainers approve the public SDK contract. A backend-owned parser hook fits the plugin boundary better than adding provider-specific core dialects, but the normalized event API should be settled through the linked PR review.
