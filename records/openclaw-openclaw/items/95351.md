## Summary

Keep open: current main and the latest release still lack a plugin-owned JSONL parser hook for CLI backends, and the linked implementation PR is open rather than merged.

## What This Changes

Add a public `CliBackendPlugin` hook for JSONL backends to map provider-native line events into OpenClaw text, thinking, session, and native tool-card display events.

## Best Possible Solution

Settle the public SDK contract in https://github.com/openclaw/openclaw/pull/95386 or a maintainer-sponsored replacement, then close this issue after the accepted hook lands on main.

## Reproduction Assessment

Yes. Source inspection shows current main only supports built-in Claude/Gemini JSONL dialect parsing and has no `CliBackendPlugin` hook for plugin-owned line parsing or display-only tool-card events.

## Solution Assessment

Unclear until maintainer API review. A backend-owned parser hook fits the plugin-boundary direction, but the normalized event contract and display-only delivery semantics need explicit approval before it is the best solution.
