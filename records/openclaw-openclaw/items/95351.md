## Summary

Keep open: current main still lacks a plugin-owned JSONL parser hook, and the open linked PR is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests a public `CliBackendPlugin` JSONL parsing hook so third-party CLI backends can map provider-specific JSONL streams into native OpenClaw text, thinking, session, result, and display-only tool-card events.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/95386 as the public plugin SDK API path; close this issue only after the accepted hook ships on main or after maintainers explicitly reject the direction.

## Reproduction Assessment

Yes. Source inspection shows current main only supports built-in Claude/Gemini JSONL dialect parsing and has no `CliBackendPlugin` hook for plugin-owned line parsing or display-only tool-card events.

## Solution Assessment

Yes at the boundary, but not yet merged. A narrow plugin-owned parser hook fits the documented plugin API direction; the exact normalized event contract should be settled in the linked PR review.
