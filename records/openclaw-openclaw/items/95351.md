## Summary

Keep open: current main still exposes only built-in Claude/Gemini JSONL parsing and has no plugin-owned parser or display-only tool-event hook for third-party `CliBackendPlugin` streams.

## What This Changes

The issue requests a generic JSONL parsing or display-only tool-event hook so third-party CLI backends can surface native tool cards without impersonating Claude's stream format.

## Best Possible Solution

Design a narrow plugin SDK/runtime hook that lets CLI backends normalize JSONL records into existing assistant, commentary, session, result, and sanitized display-only tool events, with docs and focused parser/runner tests.

## Reproduction Assessment

Yes for a source-level capability gap: current main has finite built-in JSONL dialect dispatch and no plugin-owned parser hook. This is a feature/API request, not a broken existing documented behavior.

## Solution Assessment

Unclear: a generic display-only parser hook fits the plugin boundary if maintainers want third-party CLI parity, but the exact SDK shape and event contract need maintainer design approval.
