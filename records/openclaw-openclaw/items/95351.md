## Summary

Keep open: current main and the latest release still lack a plugin-owned JSONL parser hook, and the open linked PR is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests a public `CliBackendPlugin` JSONL line-parsing hook so third-party CLI backends can map provider-native JSONL tool events into native display tool cards.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/95386 as the public plugin SDK API path; close this issue only after the accepted hook reaches main or maintainers reject the direction.

## Reproduction Assessment

Yes. Source inspection of current main shows only built-in Claude/Gemini JSONL dialect handling and no `CliBackendPlugin` hook for plugin-owned line parsing or display-only tool-card events.

## Solution Assessment

Yes at the ownership boundary, but the exact public API still needs maintainer review. A backend-owned parser hook is the narrow plugin seam; downstream text heuristics or a provider-specific built-in dialect would be less maintainable.
