## Summary

Keep open. Current main and the latest release still write `.skill` archive entries from unsorted `Path.rglob("*")` traversal, the focused tests do not assert archive entry order, and related implementation PRs are closed unmerged.

## What This Changes

Make skill-creator `.skill` packaging write archive entries in a stable deterministic order.

## Best Possible Solution

Sort accepted files by final POSIX archive path before writing, preserve the existing guards, and add regression coverage for exact `ZipFile.namelist()` order.

## Reproduction Assessment

Yes. A scratch run importing the current packager produced a `.skill` namelist that was not equal to `sorted(names)`, and source inspection shows no ordering step before `zipf.write(...)`.

## Solution Assessment

Yes. Sorting the filtered archive entries by final POSIX arcname is the narrowest maintainable fix and avoids changing validation, exclusions, symlink handling, root-escape checks, or self-output policy.
