## Summary

Keep open. Current main still writes `.skill` entries from unsorted `Path.rglob("*")` traversal, the focused tests do not assert archive order, and the related implementation PRs are closed unmerged even though they confirm the requested fix is narrow.

## What This Changes

Request deterministic ordering for entries in `.skill` archives produced by the skill-creator packager.

## Best Possible Solution

Sort accepted files by final POSIX archive path before writing, and add a regression test that asserts exact `ZipFile.namelist()` order while preserving current filters and guards.

## Reproduction Assessment

Yes. A scratch run against current main produced a `.skill` namelist in traversal order rather than sorted archive order, and source/dependency inspection shows the current loop has no sorting step.

## Solution Assessment

Yes. Sorting accepted entries by archive-relative POSIX path is the narrowest maintainable repair and avoids changing validation, exclusion, symlink, root-escape, or self-output policy.
