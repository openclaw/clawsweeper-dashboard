## Summary

Keep open. Current main and the latest release still write `.skill` archive entries directly from unsorted `Path.rglob("*")` traversal, the focused tests only check membership, and all related implementation PRs are closed unmerged.

## What This Changes

Request to make skill-creator `.skill` archives write entries in deterministic sorted path order.

## Best Possible Solution

Sort accepted files by final POSIX archive path before writing, and add a regression test asserting exact `ZipFile.namelist()` order while preserving current filters and guards.

## Reproduction Assessment

Yes. A scratch run against current main showed the packaged `.skill` namelist is not sorted, and source inspection shows there is no ordering step before `zipf.write(...)`.

## Solution Assessment

Yes. Sorting accepted archive entries by archive-relative POSIX path is the narrowest maintainable fix and avoids changing validation, exclusions, symlink handling, root-escape checks, or self-output policy.
