## Summary

Keep open. Current main and the latest release still write `.skill` archive entries from unsorted `Path.rglob("*")` traversal, the focused tests discard archive entry order, and the directly related implementation PRs are closed unmerged.

## What This Changes

Make the skill-creator `.skill` packager write archive entries in deterministic file order.

## Best Possible Solution

Sort accepted archive entries by final POSIX archive path before writing, and add a regression test that asserts exact `ZipFile.namelist()` order while preserving existing filters and guards.

## Reproduction Assessment

Yes. Source inspection on current main shows direct unsorted `Path.rglob("*")` traversal feeding `zipf.write(...)`, and local Python source shows that traversal is not sorted; I did not generate archive artifacts during this read-only review.

## Solution Assessment

Yes. Sorting accepted entries by final archive-relative POSIX path is the narrowest maintainable repair and avoids changing validation, exclusion, symlink, root-escape, or self-output policy.
