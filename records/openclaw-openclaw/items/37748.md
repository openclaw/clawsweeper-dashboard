## Summary

Keep open: current main and the latest release still write `.skill` archive entries directly from `Path.rglob("*")`, the focused tests do not assert `ZipFile.namelist()` order, and all linked implementation PRs are closed unmerged.

## What This Changes

Sort `.skill` package archive entries deterministically by relative path instead of relying on `Path.rglob` traversal order.

## Best Possible Solution

Sort accepted package entries by their final POSIX archive path before writing, then assert exact `ZipFile.namelist()` order in the focused packager tests.

## Reproduction Assessment

Yes. The prior ClawSweeper scratch run recorded a non-sorted zip namelist, and current main still has the direct unsorted `Path.rglob("*")` write path.

## Solution Assessment

Yes. Sorting already-accepted archive entries by final archive path is the narrowest maintainable fix and avoids changing packaging policy or security checks.
