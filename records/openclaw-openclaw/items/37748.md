## Summary

Keep open. Current main and the latest shipped release still write `.skill` archive entries directly from unsorted `Path.rglob("*")` traversal, the focused package tests discard archive order, and the known implementation attempts are all closed unmerged.

## What This Changes

Make the skill-creator `.skill` packager write archive entries in deterministic path order.

## Best Possible Solution

Sort the filtered package entries by final POSIX archive path before writing, preserve the existing validation/exclusion/security guards, and add regression coverage for exact `ZipFile.namelist()` order.

## Reproduction Assessment

Yes. Current main has an unsorted `rglob` write path, and a scratch package run produced a `.skill` namelist that was not equal to `sorted(names)`.

## Solution Assessment

Yes. Sorting the accepted archive entries by final archive path is the narrowest maintainable fix and avoids changing validation, exclusions, symlink handling, root-escape checks, or self-output behavior.
