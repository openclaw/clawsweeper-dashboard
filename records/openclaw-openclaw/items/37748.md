## Summary

Keep open. Current main and v2026.5.7 still stream `Path.rglob("*")` order directly into the zip, the focused tests convert `namelist()` to sets so they do not assert archive order, and the linked implementation PRs are closed unmerged.

## What This Changes

Make the skill-creator `.skill` packager write archive entries in deterministic sorted order.

## Best Possible Solution

Collect eligible files, compute final archive-relative POSIX names, sort by that name, write in that order, and add a regression test asserting exact `namelist()` order while preserving symlink/exclusion handling.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.7 shows direct unsorted `Path.rglob("*")` traversal feeding `zipf.write(...)`; I did not create package artifacts during this read-only review.

## Solution Assessment

Yes. Sorting accepted entries by final archive-relative POSIX path before writing is the narrowest maintainable repair and avoids changing validation, exclusion, symlink, or path-escape policy.
