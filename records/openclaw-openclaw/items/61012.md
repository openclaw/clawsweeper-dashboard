## Summary

Keep open: current main and the latest release still bypass Telegram's default-account resolver on token-gated message actions, while the linked repair PRs are closed unmerged.

## What This Changes

The issue asks for omitted-account Telegram sends to honor the configured default account in multi-bot setups.

## Best Possible Solution

Route every token-gated Telegram action through resolveTelegramAccount and add regression coverage for omitted accountId plus defaultAccount, while preserving explicit unknown-account fail-closed behavior.

## Reproduction Assessment

Yes, as a source-level reproduction: configure channels.telegram.defaultAccount = "work" with channels.telegram.accounts.work.botToken and no top-level/default/env token, then call handleTelegramAction or openclaw message send without accountId. Current source resolves the token as implicit default before the defaultAccount resolver can run.

## Solution Assessment

Yes. The narrow maintainable fix is to share resolveTelegramAccount in extensions/telegram/src/action-runtime.ts rather than documenting --account as required or adding another local fallback.
