## Summary

Keep open: current main has basic launchd/systemd restart supervision, channel-level health restarts, doctor-owned config repair, and targeted crash-loop hardening, but no gateway-level graduated recovery ladder. The request would add a new configurable core recovery policy with destructive repair, rollback, persistence, and alert behavior that needs maintainer product/ops design.

## What This Changes

Request native gateway crash-loop recovery with configurable thresholds, maintenance mode, config/state repair, reinstall or rollback escalation, and alerting beyond launchd/systemd restarts.

## Best Possible Solution

Design an opt-in gateway recovery controller that persists crash-loop state outside the crashing process, reuses doctor-safe repair primitives, gates destructive state/package rollback behind explicit policy, and documents alerting plus maintenance-mode behavior.
