## Summary

Keep open: current main still has supervisor respawn, restart cooldown/deferral, channel-level health restarts, and doctor-owned repair, but no gateway-level graduated crash recovery ladder. The related open PR https://github.com/openclaw/openclaw/pull/79181 describes only a first-rung rapid-start throttle and does not cover config/state recovery, maintenance mode, or operator alert policy.

## What This Changes

Request adds a native gateway crash-loop recovery ladder with configurable thresholds, maintenance mode, config/state repair tiers, and escalation alerts.

## Best Possible Solution

Design an opt-in recovery controller that tracks crash-loop state outside the crashing process, reuses doctor-safe repair primitives, gates destructive tiers behind explicit policy, and documents maintenance-mode plus alert behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new recovery capability request, not broken documented behavior. Source inspection gives high-confidence evidence that current main lacks the requested gateway.crashRecovery ladder.

## Solution Assessment

Unclear until maintainers choose the product boundary. Native recovery likely belongs around service, gateway, and doctor primitives, but the exact tiers, persistence model, rollback scope, and alert policy need maintainer design review.
