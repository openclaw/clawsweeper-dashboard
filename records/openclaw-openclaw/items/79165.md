## Summary

Keep open. Current main and v2026.6.8 still lack a native graduated gateway crash-recovery ladder; the existing supervisor settings, restart cooldown, channel-health monitor, and doctor/manual recovery paths are adjacent but do not count repeated gateway process failures and escalate through repair or notification tiers.

## What This Changes

Request a native gateway crash-recovery ladder with configurable thresholds, maintenance windows, config rollback or state repair tiers, and user alerting after repeated gateway failures.

## Best Possible Solution

Keep this as the canonical narrow gateway crash-recovery ladder design item, coordinate it with config rollback and the broader watchdog umbrella, and split any implementation into maintainer-approved core recovery slices.
