Keep open: current main still lacks a native graduated gateway crash-recovery ladder, while the existing restart, supervisor, channel-health, and doctor paths only cover adjacent pieces. The reporter's latest comment also explicitly defers implementation until maintainers choose the recovery policy.

## What This Changes

Requests a configurable gateway crash-recovery ladder that counts repeated failures and escalates from restart to config/state repair and operator alerting.

## Best Possible Solution

Design an opt-in recovery controller that persists crash-loop state outside the crashing process, reuses doctor-safe repair primitives, gates destructive actions behind explicit policy, and documents maintenance-mode plus alert behavior.
