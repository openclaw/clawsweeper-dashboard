## Summary

Keep this PR open for maintainer review and CI. The patch is a narrow fix for a source-reproducible approval wait gap, the supplied real behavior proof is sufficient, and I found no blocking correctness or security findings.

## What This Changes

Adds approval runtime token forwarding from agent gateway tool calls through `callGateway` into `GatewayClient` for default plugin approval methods, with tests for default and explicit gateway override behavior.

## Best Possible Solution

Land the narrow helper-layer token forwarding after maintainer review and CI, preserving the existing approval-runtime trust boundary and explicit gateway override isolation.

## Reproduction Assessment

Yes. Source inspection shows current main creates requester-bound plugin approvals, but later local wait calls are not marked approval-runtime because the helper never forwards the existing token; I did not run a live repro in this read-only review.

## Solution Assessment

Yes. Passing the existing in-process approval runtime token through the missing helper layer is narrower than changing approval visibility rules, and the explicit `gatewayUrl` guard preserves the manual override boundary.
