## Summary

Keep this PR open. Current main lacks the token forwarding needed for requester-bound plugin approval waits, while the PR is a narrow fix with focused tests, sufficient live-output proof, and no blocking review findings.

## What This Changes

The branch forwards the existing operator approval runtime token through local scoped `plugin.approval.*` agent gateway calls and tests token use plus explicit gateway override isolation.

## Best Possible Solution

Land the narrow helper-layer fix after maintainer review, keeping approval-runtime trust tied to scoped local calls and preserving explicit gateway override isolation.

## Reproduction Assessment

Yes. Source inspection shows `plugin.approval.request` records requester identity, while current-main wait calls from a later backend connection are not marked approval-runtime because `callGatewayTool` and `callGateway` never forward the existing token.

## Solution Assessment

Yes. Passing the existing in-process approval runtime token through the missing helper layer is narrower than loosening approval visibility rules, and the explicit `gatewayUrl` guard matches the existing operator approvals client pattern.
