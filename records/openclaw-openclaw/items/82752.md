## Summary

Keep open for maintainer review and merge consideration. Current main does not forward the existing approval-runtime token through agent gateway tool calls, while this PR threads it through the missing helper layer with a scoped guard and no blocking review findings.

## What This Changes

The PR forwards the operator approval runtime token from default `plugin.approval.*` agent gateway tool calls through `callGateway` into `GatewayClient`, and adds tests that explicit `gatewayUrl` overrides omit it.

## Best Possible Solution

Land the narrow helper-layer token forwarding after maintainer review, preserving the existing approval-runtime trust boundary and explicit `gatewayUrl` override isolation.

## Reproduction Assessment

Yes. Source inspection shows current main creates requester-bound plugin approvals over one connection, then waits over another without marking the wait connection as approval-runtime, which matches the documented `approval expired or not found` failure path.

## Solution Assessment

Yes. Passing the existing in-process approval runtime token through the missing `callGateway` layer is narrower than loosening approval visibility rules, and the explicit `gatewayUrl` guard preserves the remote/manual override boundary.
