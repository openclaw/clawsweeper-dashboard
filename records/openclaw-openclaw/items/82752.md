## Summary

## What This Changes

The PR passes an operator approval runtime token through agent gateway calls and `callGateway` for default `plugin.approval.*` calls, with regression tests for token inclusion and explicit gateway override exclusion.

## Best Possible Solution

Keep the token-threading fix, but make token inclusion depend on the resolved local/loopback gateway target and add regression coverage for configured remote and env URL defaults.

## Reproduction Assessment

No, not as-is: forwarding the existing token is the right narrow direction, but the gate must distinguish default local targets from configured/env remote targets before sending the auth field.
