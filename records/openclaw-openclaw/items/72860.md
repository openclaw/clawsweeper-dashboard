## Summary

Keep open. Current main has fixes for explicit media payload delivery and block-stream final media preservation, but the ordinary Telegram reply path still has source-level gaps for Markdown image promotion, especially local/generated targets.

## What This Changes

Ensure Telegram ordinary assistant replies preserve image attachments from Markdown or media-bearing payloads while still delivering the reply text.

## Best Possible Solution

Thread Telegram Markdown-image opt-in through ordinary final and block reply normalization, then add a constrained local/generated image allowlist that reuses the existing media access checks.

## Reproduction Assessment

Yes, source-level. Local Markdown image targets still remain text by parser design, and the ordinary agent/block reply call sites do not carry Telegram's Markdown-image extraction opt-in into normalization.

## Solution Assessment

Unclear. The direction is right for generated media, but the best fix should not promote arbitrary local Markdown paths; it should be channel-scoped and pass only approved local media through the existing normalizer.
