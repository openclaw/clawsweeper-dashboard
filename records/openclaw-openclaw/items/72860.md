## Summary

Keep open. Current main still has source-level paths matching the report: local/generated Markdown image targets stay text-only, and mixed text+MEDIA block replies can still have media marked as sent before final dedupe; #75180 tracks the latter but is still open.

## What This Changes

The issue asks for Telegram ordinary assistant replies with text plus image media, including local Markdown-image output and MEDIA-derived payloads, to deliver both text and attachments.

## Best Possible Solution

Review and land #75180 or an equivalent block-stream MEDIA accounting fix, then separately add only a maintainer-approved managed/sandbox/workspace local Markdown-image promotion path if that behavior is intended.

## Reproduction Assessment

Yes, source-level. Current main still keeps local Markdown image paths as text and still records mixed text+media block payload media as sent before final media dedupe.

## Solution Assessment

Unclear as one combined fix. #75180 is the narrow path for text+MEDIA block-stream accounting, but local Markdown-image promotion should not be broadened without an approved allowlist and media access boundary.
