## Summary

Keep open. Current main covers adjacent text-plus-media delivery loss paths, but local/generated Markdown image targets still do not become media URLs in Telegram's opted-in reply parser path.

## What This Changes

Request to preserve Telegram ordinary assistant reply image attachments, including local/generated Markdown image targets, through reply parsing and delivery.

## Best Possible Solution

Define an approved local Markdown-image promotion policy for Telegram, then route accepted candidates through the existing reply media normalizer and deny everything else.

## Reproduction Assessment

Yes. A source-level reproduction is a Telegram-opted parse of `![x](./generated.png)`: current code leaves it as text with no `mediaUrls`, and tests explicitly assert local Markdown targets are not lifted.

## Solution Assessment

Unclear. The requested direction is right, but the best implementation should not broaden `src/media/parse.ts` to arbitrary local paths; it should only pass approved local candidates into existing access-controlled normalization.
