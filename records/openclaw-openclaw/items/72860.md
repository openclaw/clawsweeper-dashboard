## Summary

Keep open. Current main covers remote Markdown images and adjacent Telegram media-loss paths, but local/generated Markdown image targets are still left as text before reply media normalization can attach them.

## What This Changes

Request to make ordinary Telegram assistant replies preserve image attachments, including local/generated Markdown image targets, through the reply parsing and delivery pipeline.

## Best Possible Solution

Add a narrowly scoped local Markdown-image promotion path for Telegram that only accepts approved generated/managed/workspace/sandbox targets and then reuses the existing reply media normalizer.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Telegram opt-in reply such as `![x](./generated.png)`: current parsing leaves it as text with `mediaUrls` undefined, and tests explicitly assert local Markdown targets are not lifted.

## Solution Assessment

Unclear. The report’s direction is right, but the maintainable fix should not broaden the parser to arbitrary local-looking paths; it should pass only approved local candidates into the existing access-controlled media normalization path.
