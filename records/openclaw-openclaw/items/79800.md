## Summary

The report matches OpenClaw's documented group/channel visible-reply policy rather than a core WeCom streaming-delivery failure: normal final model text stays private in groups unless the message tool is used or automatic group replies are explicitly enabled.

## What This Changes

The issue asks OpenClaw to deliver normal model final replies back to a WeCom group chat after the model completes.

## Best Possible Solution

Keep the shared group/channel tool-only default and use the existing `messages.groupChat.visibleReplies: "automatic"` opt-in or the plugin's message tool path for visible WeCom group replies.

## Reproduction Assessment

No high-confidence bug reproduction is present. The supplied logs match the documented default where group final text is private unless sent through the message tool or `messages.groupChat.visibleReplies` is set to `automatic`.

## Solution Assessment

Yes. The existing config flag is the narrowest maintainable way to opt into automatic group replies; core should not special-case WeCom or bypass the shared group visible-reply policy.
