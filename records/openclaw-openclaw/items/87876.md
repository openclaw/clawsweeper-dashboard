## Summary

Keep open: current main and v2026.6.9 still have the Bedrock Converse abort shape where a provider-originated stream abort can become promptError plus aborted, which bypasses both prompt-side failover and result-level fallback.

## What This Changes

The issue requests retry/fallback and transcript preservation for long Bedrock Converse streaming sessions that abort after several minutes.

## Best Possible Solution

Add a narrow Bedrock/provider recovery path that classifies provider-originated stream aborts for existing retry/fallback while preserving intentional cancellation and partial-transcript safety.

## Solution Assessment
