## Repair Work Prompt

Fix Bedrock Converse streaming abort handling for long provider-started streams. Add focused regression coverage that simulates the amazon-bedrock stream throwing or aborting with an AWS modelStreamErrorException/internalServerException or an abort-shaped 'This operation was aborted' after partial stream activity, and verify the run does not silently finish as an aborted prompt-error without fallback. Keep the fix within the Bedrock plugin and existing model-fallback/embedded-runner seams; do not add a new config option or change the params.modelId/profile behavior tracked separately in #87318. Preserve user-initiated abort semantics, attempt configured fallback models for provider/transport-originated aborts, and ensure terminal failure retains useful error and transcript state.
