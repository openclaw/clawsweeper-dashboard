## Summary

Keep open: current main and v2026.6.9 have partial Codex app-server shutdown and one-shot teardown work, but they still do not prove the reported embedded-run client-close/timeout paths abort, wait for, and reap all Codex/native descendant process groups before releasing lane or session state.

## What This Changes

Ensure subagent and cron embedded runs terminate their associated worker, Codex app-server, and native tool process groups when runs complete, fail, time out, or the app-server closes mid-turn.

## Best Possible Solution

Keep this as the canonical lifecycle bug until a maintainer-owned Docker/Codex repro and design prove app-server sidecars and native descendant process groups are interrupted, reaped, and reconciled before lane/session state is released.

## Reproduction Assessment

No high-confidence current-main live reproduction is available: a maintainer tried current main and 2026.5.20/2026.5.22 Docker cron/subagent runs without orphaned processes, but later field reports match current source-plausible client-close and process-group gaps.

## Solution Assessment

No. Current main has partial transport and one-shot cleanup, but the best fix needs one lifecycle contract across OpenClaw abort handling, Codex interrupt/unsubscribe, app-server transport shutdown, native descendant cleanup, and transcript/session reconciliation.
