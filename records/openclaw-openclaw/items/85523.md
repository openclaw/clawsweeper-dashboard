## Summary

Keep open: current main and v2026.5.20 still pass unchecked bootstrap file names into lowercasing in both context building and truncation-warning code, while a related open PR only covers the generic /context display/stat path.

## What This Changes

The issue requests a crash fix for undefined bootstrap file names causing `.toLowerCase()` failures during message bootstrap processing in v2026.5.20.

## Best Possible Solution

Normalize bootstrap file names at the hook/sanitization boundary, deriving a safe name from path or skipping malformed entries, then add regressions for undefined/non-string names in both context construction and truncation-warning/reporting paths.

## Reproduction Assessment

Yes, at source level. A bootstrap entry with a valid path, oversized content, and undefined `name` reaches `trimBootstrapContent(..., file.name, ...)` and `formatBootstrapTruncationWarningLines`, both of which can call `.toLowerCase()` on the missing value.

## Solution Assessment

Yes. The best fix is TypeScript-source boundary hardening with focused regression tests, not patching generated `dist/` or only optional-chaining one stack-frame symptom.
