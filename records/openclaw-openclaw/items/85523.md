## What This Changes

The issue reports a 2026.5.20 crash where an undefined workspace bootstrap file name reaches `.toLowerCase()` during bootstrap context trimming and truncation warnings, blocking agent replies.

## Best Possible Solution

Normalize or reject unnamed bootstrap file entries at the hook/runtime boundary, derive a safe display name from the path only when valid, and add defensive guards plus regression coverage for both trimming and warning formatting.

## Reproduction Assessment

Yes. Source inspection shows a hook-mutated `WorkspaceBootstrapFile` with a valid `path` but missing or non-string `name` can pass current sanitization and then hit `.toLowerCase()` in the reported helpers; I did not run a live gateway repro in this read-only review.

## Solution Assessment

No patch is proposed here, but the best repair is clear: enforce a valid bootstrap-file name before context building and keep the name-comparison helpers defensive so one malformed hook entry cannot crash all turns.
