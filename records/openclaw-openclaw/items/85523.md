## What This Changes

The issue requests fixing a gateway/agent bootstrap crash when a bootstrap file entry has an undefined name and truncation handling calls `.toLowerCase()`.

## Best Possible Solution

Normalize or reject malformed bootstrap entries before context and budget processing, then add regression coverage for undefined/non-string names in the context builder, warning budget path, and hook-updated file flow.

Yes, at source level. A bootstrap entry with a valid path, oversized content, and undefined name reaches `trimBootstrapContent(..., file.name, ...)`, which calls `undefined.toLowerCase()`; the same relevant code is in v2026.5.20 and current main.

## Solution Assessment

Yes, the best fix is a TypeScript-source boundary hardening with regression tests, not patching generated `dist/` or only optional-chaining one stack-frame symptom.
