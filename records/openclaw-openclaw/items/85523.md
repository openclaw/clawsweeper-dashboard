## Summary

Keep open: current main still contains the null-unsafe bootstrap filename paths reported from v2026.5.20, and the related open display-label PR does not cover the message-handling crash path.

## What This Changes

The issue asks OpenClaw to stop crashing when a bootstrap file entry has an undefined name during agent bootstrap context and truncation-warning processing.

## Best Possible Solution

Normalize or reject malformed bootstrap entries before context and budget processing, derive a stable display name from path when possible, and add regression coverage for undefined or non-string names in the context builder, warning budget path, and hook-updated file flow.

## Reproduction Assessment

Yes, at source level. A bootstrap entry with a valid path, oversized content, and undefined name reaches `trimBootstrapContent(..., file.name, ...)`, and the warning path can later call `isAgentsBootstrapName(file.name)` with the same undefined value.

## Solution Assessment

Yes. The best fix is TypeScript-source boundary hardening with regression tests, not patching generated dist output or only optional-chaining one stack-frame symptom.
