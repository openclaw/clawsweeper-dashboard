## Summary

Keep open for maintainer/contributor action: this targets a real gateway run-id tension, but the proposed one-line UUID swap conflicts with the current external chat streaming contract and lacks real behavior proof.

## What This Changes

The PR changes gateway node `agent.request` dispatch to pass a fresh UUID run id to `agentCommandFromIngress` and updates the focused test to assert it differs from the session id.

## Best Possible Solution

Keep the session-scoped external event contract unless maintainers choose a broader gateway/client contract that maps unique backend run ids to session observers and updates gateway plus OpenClawKit tests together.

## Reproduction Assessment

Yes, for the PR regression at source level: current OpenClawKit code drops external agent events whose run id differs from the session id, while this PR changes node `agent.request` to a fresh UUID. The original run-collapse claim still lacks real setup proof.

## Solution Assessment

No. The one-line UUID swap reopens a behavior explicitly reverted in merged https://github.com/openclaw/openclaw/pull/43751 and breaks the current session-observer contract; a maintainable fix needs an API/client design rather than only changing the gateway caller.
