## What This Changes

The PR changes gateway node `agent.request` dispatch to use `randomUUID()` for `runId` and updates the focused test to expect a non-session run id.

## Best Possible Solution

Keep the session-scoped external event contract unless maintainers choose a coordinated gateway and OpenClawKit design that maps unique backend run ids to session observers.

## Reproduction Assessment

Yes for the PR-introduced regression at source level: current OpenClawKit drops external `agent` events whose run id differs from the session id, while this branch changes node `agent.request` to a fresh UUID. The original collapse bug does not have high-confidence real setup reproduction in the PR body or comments.

## Solution Assessment

No. The one-line UUID swap conflicts with current session-observer behavior and repeats a change that merged history restored to `runId: sessionId`; a maintainable fix needs maintainer/API judgment across gateway and client contracts.
