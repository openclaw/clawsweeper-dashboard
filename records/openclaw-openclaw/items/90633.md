  - confidence: 0.93

  - body: This disables the native Code Mode surface when `beforeToolCallPolicyActive` is true, but `bundleMcpThreadConfig` was already built with the original allowlist and is still merged into the Codex thread.   - confidence: 0.9
- **[P1] Preserve policy-protected Codex thread continuity:** `extensions/codex/src/app-server/dynamic-tool-build.ts:401-402`
  - body: Making policy-active turns report `nativeToolSurfaceEnabled=false` sends existing bindings through the lifecycle branch that preserves the old binding and starts an unwritten transient thread. After one protected turn, later protected turns keep forking fresh Codex threads instead of resuming the policy-protected conversation, so users who enable tool policy hooks lose app-server continuity.
  - confidence: 0.86
- **[P2] Keep the pre-generation binding grace path:** `extensions/codex/src/app-server/thread-lifecycle.ts:527-535`
  - body: For bindings written before `nativeHookRelayGeneration`, this new branch clears the valid thread and starts fresh whenever native hook relay is enabled. Current main has a bounded first-resume grace path for exactly this upgrade case, so this change drops existing users' exact Codex conversation continuity instead of using the compatibility path.
  - confidence: 0.88
