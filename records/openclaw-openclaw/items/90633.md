## Summary

Keep open: the PR targets a real Codex/OpenClaw policy-enforcement gap, but it is not merge-ready because the new Codex `bypass_hook_trust` override is global and can run untrusted user or plugin hooks without review.

## What This Changes

This PR disables Codex native tool surfaces while OpenClaw tool policy is active, exposes OpenClaw-owned dynamic shell tools, changes native hook relay trust and unavailable behavior, rotates older hookless bindings, and rejects `/btw` side threads under policy.

## Best Possible Solution

Keep the fail-closed OpenClaw-owned tool routing for policy-active turns, but trust only the generated relay hooks through a narrow trusted-hash or managed-hook mechanism and prove ordinary untrusted Codex hooks remain gated.

## Solution Assessment

No. The OpenClaw-owned dynamic-tool fallback is the right direction, but using Codex's global `bypass_hook_trust` is not the safest implementation because upstream applies it to all hook sources.
