## Summary

This PR is still necessary for the Codex policy-enforcement gap, but it is not merge-ready because the current head still breaks policy-active thread continuity and explicit bundle MCP allowlists in edge cases that existing users can hit.

## What This Changes

The PR changes Codex app-server policy-active runs to disable Codex-owned native shell/file/MCP surfaces, route execution through OpenClaw dynamic tools, trust native hook relay entries via hook state, preserve selected thread bindings, and reject `/btw` side threads under active policy.

## Best Possible Solution

Keep the fail-closed OpenClaw-owned enforcement boundary, but preserve existing Codex thread state and explicit bundle MCP allowlists unless active OpenClaw policy or node-exec policy specifically requires disabling Codex-owned tool surfaces.
