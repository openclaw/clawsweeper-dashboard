## Summary

Keep open. Current main implements the narrow per-agent `thinkingDefault` and flat `agents.list[].params` pieces, but the remaining requested public API is still absent: no per-agent model map or generation block, and no `thinkingStrategy` contract for rules, failure escalation, or AGENTS.md-sourced policy. Related items overlap, but #64792 was closed in favor of this issue and the other open items do not supersede the full design request.

## What This Changes

Request per-agent thinking defaults, per-agent model parameter overrides, and adaptive thinking strategies including rule matching, failure escalation, and AGENTS.md-driven policy.

## Best Possible Solution

Keep this as the canonical maintainer design item, then split implementation once direction is chosen: preserve the shipped per-agent `thinkingDefault` and flat `params` behavior, coordinate generic generation-parameter forwarding with #47852/#32496, and separately define a versioned adaptive thinking policy for rules, tool/test failure escalation, and any AGENTS.md-sourced behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is high-confidence by static inspection of current schema, exported types, runtime merge code, docs, tests, and targeted search for the missing fields.

## Solution Assessment

Unclear. The current flat `agents.list[].params` surface is the narrowest shipped solution for selected-model temperature-style overrides, but the requested per-agent model map and adaptive strategy policy need maintainer API/product direction before a best implementation can be chosen.
