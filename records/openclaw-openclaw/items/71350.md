## Summary

Keep open. Current main implements the narrow per-agent `thinkingDefault` schema and flat `agents.list[].params` override layer, but it still lacks the requested per-agent model/generation map and any `thinkingStrategy` contract for rules, escalation, or AGENTS.md-sourced policy. The related open items overlap with pieces of the request but do not supersede the whole design question.

## What This Changes

The issue requests per-agent thinking defaults, per-agent generation/model parameters, and adaptive thinking policies for rule-based, failure-escalating, and AGENTS.md-driven behavior.

## Best Possible Solution

Keep this as a maintainer design tracker, preserve the shipped `thinkingDefault` and flat `params` behavior, and split follow-up implementation only after maintainers choose the public config shape for per-agent generation parameters and adaptive thinking policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is reproducible by static inspection of the current strict schema, exported types, runtime merge path, docs, tests, and targeted searches for the missing fields.

## Solution Assessment
