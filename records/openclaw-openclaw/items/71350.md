## Summary

Keep open. Current main implements the narrow per-agent `thinkingDefault` and flat `agents.list[].params` override surface, but the broader requested API is still not present: there is no per-agent `models` map or `generation` block, and no `thinkingStrategy` contract for rule matching, failure escalation, or AGENTS.md-sourced policy. The related open items cover important pieces but do not supersede the full request.

## What This Changes

The issue requests per-agent thinking defaults, per-agent model/generation parameter overrides, and adaptive thinking strategies for rules, failure escalation, and AGENTS.md-sourced policy.

## Best Possible Solution

Keep this as the maintainer design umbrella, preserve the shipped `thinkingDefault` and flat `params` behavior, coordinate the per-agent generation-parameter surface with #47852 and #32496, and define a versioned adaptive thinking policy before implementing rules, failure escalation, or AGENTS.md-sourced behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is reproducible by static inspection of current schema, exported types, runtime merge code, docs, tests, and targeted searches for the missing strategy fields.

## Solution Assessment

Unclear. The current flat `agents.list[].params` surface is a narrow shipped solution for selected-model temperature-style overrides, but the requested per-agent model map and adaptive strategy policy need maintainer API/product direction before a best implementation can be chosen.
