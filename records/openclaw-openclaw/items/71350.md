## Summary

Keep open. Current main covers part of the request with per-agent `thinkingDefault`, flat per-agent `params`, and schema-level `agents.list[].models`, but it still lacks runtime per-agent model-param inheritance from `agents.list[].models`, adaptive `thinkingStrategy` rules/escalation, and an AGENTS.md-driven policy contract; the latest comment also adds a related defaults-layer API question that should be handled by maintainers.

## What This Changes

Request per-agent thinking defaults, per-agent model/generation parameters, and adaptive thinking strategies for differentiated agent profiles.

## Best Possible Solution

Keep this as the broad design tracker, preserve the shipped `thinkingDefault` and flat `params` behavior, and have maintainers choose a documented versioned contract before splitting the remaining model-parameter and adaptive-strategy work into focused follow-ups.
