## Summary

Keep open. Current main implements the narrow per-agent `thinkingDefault` and flat `agents.list[].params` behavior, but the broader requested public API is still missing: no per-agent model map or `generation` block, and no `thinkingStrategy` contract for rule matching, failure escalation, or AGENTS.md-sourced policy. Related #47852, #32496, and open PR #44126 overlap with slices of the request but do not supersede the whole design item.

## What This Changes

Request to add per-agent thinking defaults, per-agent model/generation parameter overrides, and rules/escalation/AGENTS.md-based adaptive thinking strategy configuration.

## Best Possible Solution

Keep this as the design-level tracker, preserve the shipped `thinkingDefault` and flat `params` behavior, coordinate generation-parameter work with #47852/#32496, and split adaptive thinking into a versioned config/runtime policy after maintainers choose the public contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is reproducible by static inspection because current strict schema/types/runtime/docs accept `thinkingDefault` and flat `params` but have no `models`, `generation`, or `thinkingStrategy` surface under `agents.list[]`.

## Solution Assessment

Unclear. The current flat `agents.list[].params` implementation is a narrow maintainable answer for selected-model overrides, but the requested model map and adaptive strategy policy require maintainer API/product direction before there is a best implementation.
