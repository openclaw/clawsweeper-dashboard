## Summary

Keep open. Current main implements the narrow per-agent `thinkingDefault` and flat `agents.list[].params` override path, but the broader requested public API still lacks per-agent `models` or `generation` blocks and any adaptive `thinkingStrategy` contract for rules, failure escalation, or AGENTS.md-sourced policy. Related #47852, #32496, and open PR #44126 cover important slices but do not fully supersede this design-level request.

## What This Changes

The issue requests per-agent thinking defaults, per-agent model generation parameters, and adaptive thinking strategies including rule-based, escalation-based, and AGENTS.md-driven policies.

## Best Possible Solution

Keep this as the umbrella design tracker, preserve the shipped `thinkingDefault` and flat `params` behavior, coordinate generation-parameter naming with #47852 and #32496, and split adaptive thinking into a versioned config/runtime policy after maintainers choose the public contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is reproducible by static inspection because current strict schema, types, docs, and runtime support `thinkingDefault` plus flat `params` but not `models`, `generation`, or `thinkingStrategy` under `agents.list[]`.

## Solution Assessment

Unclear. The current flat `agents.list[].params` implementation is a narrow maintainable answer for selected-model overrides, but the requested model-map/generation API and adaptive strategy policy need maintainer product/API review before there is a best implementation.
