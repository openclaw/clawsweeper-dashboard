## Summary

Current main implements the relevant node-exec behavior: explicit `host=node` workdirs are preserved as node-local paths, and full-trust node exec skips the approval-plan path that called `resolveCanonicalApprovalCwdSync`. That fixes the reported 2026.3.13 Linux-gateway to macOS-node failure class and shipped in `v2026.4.29`.

## What This Changes

The issue asks for macOS node-host exec to accept valid `/Users/<user>` workdirs instead of failing canonical cwd validation in a Linux-gateway to macOS-node setup.

## Best Possible Solution

Keep the shipped boundary: the gateway preserves explicit node-local cwd values, and full-trust node exec bypasses approval-only preparation. If a current release still fails under an approval-backed policy, track that separately with policy-specific reproduction data.

## Reproduction Assessment

Yes. The report gives a concrete 2026.3.13 Linux-gateway to macOS-node setup, and current tests cover the key `/Users/...` workdir forwarding path plus the full/off prepare bypass that avoids the failing canonical cwd planner.

## Solution Assessment

Yes for the reported full-trust node-exec failure class. Preserving node-local cwd at the gateway boundary and skipping approval preparation when no approval is required is narrower and safer than weakening node-side canonical cwd checks globally.
