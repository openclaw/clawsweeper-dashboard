- `P2`: This is a normal-priority gateway session continuity bug with limited blast radius to integrations using stable raw external keys.
- `impact:session-state`: The failure can split one logical conversation across different session IDs and transcript/session records.

## Summary

Keep open: current main and the latest release still canonicalize raw external session keys into agent-scoped keys and prune noncanonical candidates, while the open fix PRs for this report are unmerged and currently conflicting.

## What This Changes

The issue asks OpenClaw to preserve stable raw external session keys as aliases of their canonical agent-scoped keys so one logical conversation does not split after internal agent activity.

## Best Possible Solution

Land one shared raw/canonical alias contract that resolves, writes, patches, deletes, resets, compacts, lists, and plugin-mutates each logical session as one alias group.

## Reproduction Assessment

Yes, as a source-level reproduction: current main canonicalizes raw external keys, includes the raw key in the migration candidate set, then prunes noncanonical candidates without a shared raw-alias sync helper. I did not run a live external-channel replay in this read-only review.

## Solution Assessment

Yes, the shared raw/canonical alias contract is the narrow maintainable fix; this issue should stay open until one of the linked PRs lands or maintainers choose a replacement path.
