## Summary

Keep open: current main still canonicalizes non-agent raw session keys into agent-scoped store keys and prunes the raw candidate during migration, while the linked raw-alias fix remains open and unmerged.

## What This Changes

The issue reports that stable raw external session keys can split one logical conversation after gateway sessions_send or nested agent paths canonicalize and prune the raw key.

## Best Possible Solution

Land or replace the linked raw-alias fix with a shared gateway alias lookup and synchronization contract that keeps raw and canonical store keys pointed at the same session entry across all mutation paths.

## Reproduction Assessment

Yes, from source inspection: seed a store row under a raw external key, let gateway migration resolve it to the agent-scoped canonical key and prune the raw candidate, then send another inbound turn using the same raw key. I did not execute a live external-channel replay in this read-only review.

## Solution Assessment

Yes, the best direction is a shared raw-alias lookup/sync contract in the gateway session-store paths, not another one-off special case. The open PR appears to be that implementation path, but it has not landed yet.
