## Summary

Keep open: current main and the latest release still canonicalize raw non-agent session keys into agent-scoped keys and prune noncanonical candidates, while the two raw-alias fix paths are still open and unmerged.

## What This Changes

The issue asks OpenClaw to preserve raw external session-key continuity so internal `sessions.send` or nested agent activity cannot split one logical conversation into multiple session records.

## Best Possible Solution

Keep this issue open until a repaired raw/canonical session alias contract lands, preferably by finishing https://github.com/openclaw/openclaw/pull/83368 or the narrower closing PR at https://github.com/openclaw/openclaw/pull/86826.

## Reproduction Assessment

Yes, source-reproducible: seed a store row under a structured raw key, let a gateway path canonicalize and prune it, then a later inbound lookup by the raw key can miss the pruned row. I did not run a live external-channel replay in this read-only review.

## Solution Assessment

Yes on direction, not yet on landed state: the maintainable fix is a shared raw/canonical alias contract that covers lookup and every persisted mutation path. The issue should stay open while the linked PRs are reviewed, repaired, or superseded by an equivalent merged fix.
