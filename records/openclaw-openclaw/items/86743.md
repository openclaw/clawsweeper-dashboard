## What This Changes

The issue reports that stable raw external session keys can split one logical conversation into separate session records and transcript files after internal `sessions.send` or nested agent activity canonicalizes the same conversation.

## Best Possible Solution

Land one shared raw/canonical session-alias contract at the session storage boundary that covers lookup plus patch, delete, reset, compaction, resolver, node-event, and plugin host-hook mutations.

## Reproduction Assessment

Yes. Source inspection shows raw non-agent keys canonicalize to agent-scoped keys and current alias migration/accessor paths prune noncanonical keys, which matches the reported split path; I did not run a live external-channel replay.

## Solution Assessment

Yes on direction, not on landed state. The best fix is a shared alias-preservation contract across all persisted session mutations, and the open PRs should be reviewed or replaced before the issue closes.
