## Summary

Keep open: current main and the latest release still have the raw-key canonicalization plus migration/prune shape without a shared raw-alias synchronization helper, and the issue is already paired with open candidate fix PRs.

## What This Changes

The issue asks OpenClaw to preserve stable raw external session keys across gateway canonicalization and nested agent/session activity so one logical conversation does not split into multiple session records.

## Best Possible Solution

Land one shared gateway raw-alias preservation contract that keeps raw and canonical session-store entries synchronized across lookup and every mutation path that can touch the same logical session.

## Reproduction Assessment

Yes, as a source-level reproduction: current main canonicalizes raw external keys, includes the raw key in migration candidates, and can prune noncanonical candidates without a shared raw-alias synchronization helper. I did not run a live external-channel replay in this read-only review.

## Solution Assessment

Yes, the best fix shape is a shared alias lookup and synchronization contract across gateway session-store lookup, write, patch, delete, reset, compaction, and plugin hook mutation paths. One-sided preservation in only the initial migration path would leave follow-up mutations inconsistent.
