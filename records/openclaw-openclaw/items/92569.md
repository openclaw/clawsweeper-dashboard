## What This Changes

The issue asks OpenClaw to stop leaking session-scoped MCP server processes across session transitions so stale children do not hold file locks on reconnect.

## Best Possible Solution

Land a lifecycle-safe MCP runtime cleanup that covers reset/delete timeouts and sessionKey rebinds, then close this issue after the fixing PR merges.

Unclear for the open PR path: it targets a real gateway timeout gap, but prior review shows the best fix needs lifecycle-safe drain, force-clear, or deferred lease-release behavior rather than unconditional immediate disposal.
