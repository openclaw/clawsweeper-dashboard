## Summary

## What This Changes

Ensure stale MCP server processes are reused or reaped across session boundaries so file-locking MCP servers do not block the next session.

Land a lifecycle-safe MCP runtime cleanup fix that handles reset timeout and any confirmed sessionKey-rebind leak without closing active MCP leases unexpectedly.
