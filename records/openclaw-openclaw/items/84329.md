## Summary

Keep open: current main still builds normal iMessage JSON-RPC `send` params without any transport selector, and the referenced transport patch exists only as an unmerged commit that is not contained in HEAD or a release. The source and upstream imsg contract support the request, but the default/config policy still needs maintainer judgment and live macOS proof.

## What This Changes

Request normal iMessage outbound sends to use or expose a configurable `imsg` RPC transport so IMCore can be preferred over AppleScript when available.

## Best Possible Solution

Add an iMessage-plugin-owned transport enum with account inheritance, pass it into RPC `send`, document the default/upgrade behavior, and cover older `imsg` compatibility plus live macOS proof.

## Reproduction Assessment

No, there is not a high-confidence live reproduction path from this review. Source inspection does reproduce the missing normal-send transport selector on current main, and upstream imsg source confirms the transport parameter exists.

## Solution Assessment

Unclear until maintainers choose the compatibility default. The maintainable path is a plugin-owned config enum with tested account inheritance and explicit default behavior, rather than an unreviewed unconditional delivery-path change.
