## Summary

Keep open: current main still sends normal iMessage replies through JSON-RPC `send` without a transport selector, and the referenced transport patch commit is not contained in current `main` or any local release tag.

## What This Changes

The issue requests an iMessage normal-send transport selector so OpenClaw can prefer or configure `imsg` IMCore/bridge delivery instead of relying on the AppleScript path.

## Best Possible Solution

Add an iMessage plugin-owned account-level transport enum, pass it into RPC `send`, document the default and upgrade behavior, cover older `imsg` compatibility, and prove the selected policy on a live macOS setup.

## Reproduction Assessment

No high-confidence live reproduction of the AppleScript timeout was established in this read-only review. Source inspection does reproducibly show that current main never sends an `imsg` transport selector on the normal outbound path.

## Solution Assessment

Unclear until maintainers choose the default policy. The maintainable direction is a plugin-owned transport enum with account inheritance, compatibility handling, docs, tests, and live upgrade proof.
