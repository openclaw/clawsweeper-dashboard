## Best Possible Solution

Add a plugin-owned `sendTransport` enum only after maintainers choose the compatibility default, then pass it through normal RPC sends with older-`imsg` handling, docs, focused tests, and live macOS proof.

No. I did not establish a live macOS reproduction; source inspection does show current main has no OpenClaw transport/config selector and leaves normal-send transport behavior to `imsg` defaults.

## Solution Assessment

Unclear. Passing a plugin-owned enum through RPC `send` is plausible, but maintainers should first choose whether default behavior remains omitted/compat, explicit `auto`, or opt-in `bridge`.
