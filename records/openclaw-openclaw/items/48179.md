Keep open. ## Best Possible Solution

Keep this as the canonical maintainer/product tracker and design a generic media/TTS pipeline state contract before adding a Telegram-owned implementation that changes ordering, caching, or delivery behavior.

## Solution Assessment

No single narrow patch is the best solution. The maintainable path is a designed media/TTS pipeline state contract with Telegram integration, not a Telegram-only bypass of ordering, authorization, or transcript-trust boundaries.
