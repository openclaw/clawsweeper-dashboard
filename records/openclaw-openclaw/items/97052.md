The PR widens the session model-registry TypeBox `input` schema to accept `audio` and `video`, and adds a generated plugin catalog regression test.

## Best Possible Solution

Use one shared model input contract for the registry schema, exported `Model.input`, and registered-provider model inputs, keep `document` excluded unless runtime support is intentionally added, and add redacted real gateway/catalog proof before merge.

## Solution Assessment

No, not quite. The schema change is the right owner-boundary fix for the dropped generated catalogs, but the best fix also widens the exported model input contracts so the registry does not return values outside its public type.
