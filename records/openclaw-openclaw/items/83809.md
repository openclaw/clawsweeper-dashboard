## Summary

Keep open: current main has the private Claude live-session registry and stream-json stdin serializer, but it does not expose a supported API or local IPC surface for external user-message injection. This is a new security-sensitive orchestration feature that needs maintainer product and security direction before implementation.

## What This Changes

Expose a supported way for external same-machine orchestrators to inject a user message into an active Claude stream-json live session, optionally through an explicitly enabled Unix-domain socket.

## Best Possible Solution

Design a narrow, authenticated, opt-in core or Gateway runtime contract for live-session injection, or explicitly keep system events and heartbeat delivery as the supported orchestrator wake path.

## Reproduction Assessment

No; this is primarily a feature/API request rather than a reproducible bug. Source inspection confirms the internal live-session pieces exist, but the reported openclaw agent --message hang does not include enough steps or logs to reproduce as a current-main defect.

## Solution Assessment

Unclear; reusing the existing serializer is plausible, but exposing it through an API or UDS listener is a new security-sensitive public surface. Maintainers should first settle auth, opt-in configuration, replay handling, busy-turn behavior, and Gateway versus Claude-specific ownership.

## Risks / Open Questions

- A local socket or external API that writes into an active live session would let same-host processes add user-side input, so opt-in, authorization, socket permissions, replay handling, and audit semantics need explicit design.
- Direct stdin injection could race an active streaming turn or tool call; current main rejects overlapping live turns instead of defining concurrent user-message delivery.
- Adjacent work such as https://github.com/openclaw/openclaw/issues/50739 and https://github.com/openclaw/openclaw/pull/62528 should be considered so OpenClaw does not grow competing wake, hook, and live-session message contracts.
