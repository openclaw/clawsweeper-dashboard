## Summary

Keep open: current main and the latest release still have only a private Claude live-session registry and internal stream-json stdin writer, not a supported external-process user-message injection API. This remains a security-sensitive product/API decision rather than conservative cleanup.

## What This Changes

Expose a supported live-session user-message injection API and optional same-machine UDS listener so external orchestrators can wake an active TUI session.

## Best Possible Solution

Design a narrow, authenticated, opt-in Gateway or live-runtime contract for external wake delivery, then align wake-inbox durability and wake telemetry with that approved contract.

## Solution Assessment
