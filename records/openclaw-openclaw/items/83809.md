## Summary

Keep open: current main and the latest shipped release still have only a private Claude live-session registry and internal stream-json stdin writer. Existing Gateway `sessions.send`, `sessions.steer`, hooks, and heartbeat wake paths are related, but they do not provide the requested external-process same-child injection surface, which needs product and security review first.

## What This Changes

The issue requests a supported external-process API, optionally backed by an opt-in Unix-domain socket, to inject user messages into an active Claude stream-json live session for orchestrator wake delivery.

## Best Possible Solution

Design a narrow, authenticated, opt-in Gateway or live-runtime contract for orchestrator wake delivery, or explicitly keep system events and heartbeat as the supported path.

## Reproduction Assessment

No for the reported `openclaw agent --message` hang: the issue lacks runnable steps, logs, or a live current-main reproduction. Yes for the feature gap: source inspection confirms there is no supported external live-session injection API.

## Solution Assessment

Unclear: reusing the existing serializer is plausible, but exposing it through an API or UDS listener needs a product and security decision first. The safer direction is a documented, authenticated, opt-in runtime contract rather than exposing a private Claude-only helper ad hoc.
