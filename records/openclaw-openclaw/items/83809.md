## Summary

Keep open: current main has a private Claude live-session registry and internal stream-json stdin serializer, but no supported external-process injection API. Existing Gateway send/steer and heartbeat wake paths are useful context, but they do not satisfy this security-sensitive live-child injection request.

## What This Changes

Expose a supported live-session user-message injection API, optionally with an explicitly enabled local UDS listener, so orchestrators can wake an active Claude stream-json session.

## Best Possible Solution

Design a narrow, authenticated, opt-in Gateway or live-runtime contract for orchestrator wake delivery, or explicitly keep system events and heartbeat as the supported path.

## Reproduction Assessment

No for the reported hang: the missing injection API is source-confirmed, but the `openclaw agent --message` hang lacks runnable steps, logs, or a current-main reproduction. This is primarily a feature/API request rather than a strict bug reproduction.

## Solution Assessment

Unclear: reusing the existing serializer is plausible, but exposing it through an API or UDS listener needs a product and security decision first. The safer endpoint is a documented, authenticated, opt-in runtime contract rather than a private Claude-only helper exposed ad hoc.
