## Summary

Keep open: current main still has only a private Claude live-session registry and internal stream-json stdin writer, while the existing Gateway send/steer and heartbeat wake paths do not provide the requested supported external-process injection surface.

## What This Changes

Request a supported API and optional local Unix-domain socket listener for external orchestrators to inject user messages into the active Claude stream-json live session.

## Best Possible Solution

Design a narrow, authenticated, opt-in Gateway or live-runtime contract for orchestrator wake delivery, or explicitly keep system events and heartbeat as the supported path.

## Reproduction Assessment

No for the reported hang: the issue lacks runnable steps, logs, or environment details. Source inspection does confirm that the requested live-session injection API is absent on current main.

## Solution Assessment

Unclear: reusing the internal serializer is plausible, but exposing it through an API or UDS listener needs product and security review first. The safer shape is a documented, authenticated, opt-in runtime contract rather than exposing private Claude-only helpers ad hoc.
