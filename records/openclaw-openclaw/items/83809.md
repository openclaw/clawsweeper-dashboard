## Summary

Keep open: current main and v2026.6.10 still do not expose a supported external-process live-session injection API; the existing hooks and session RPCs are related but do not reuse the active Claude stream-json child as requested, and the proposed UDS/API path needs maintainer security/product approval.

## What This Changes

The issue requests a supported API, and optionally an opt-in Unix-domain socket listener, for external processes to inject user-side messages into an active Claude live session.

## Best Possible Solution

Design a narrow authenticated, opt-in Gateway or live-runtime wake injection contract with audit and durability semantics before exposing any private Claude stream-json helper.

## Reproduction Assessment

Not applicable as a strict bug reproduction: source inspection confirms the requested supported injection API is missing, but the reported `openclaw agent --message` hang does not include runnable current-main steps or logs.

## Solution Assessment

Unclear: reusing the existing stream-json serializer is plausible, but the API and optional UDS listener need an approved trust boundary and delivery contract before implementation is the best fix.
