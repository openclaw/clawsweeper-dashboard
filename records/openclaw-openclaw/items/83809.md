## Summary

Keep open: current main still has the Claude live-session registry and stream-json stdin serializer only as private runtime internals, with no supported external-process injection API or opt-in UDS/Gateway contract. Related open work covers soft steering, system-event priority, or transcript-only injection, but none satisfies this same-child Claude live-session wake request.

## What This Changes

The issue requests a supported API, and optionally an opt-in local UDS listener, for external orchestrators to inject user-side stream-json messages into an active Claude live session.

## Best Possible Solution

Design a narrow, authenticated, opt-in Gateway or live-runtime contract for orchestrator wake delivery, or explicitly keep heartbeat/system events as the supported path.

## Reproduction Assessment

No for the reported `openclaw agent --message` hang: the issue lacks runnable steps, logs, and environment details. Yes as a source-level feature gap: current main has the private live-session map and serializer but no supported external injection API.

## Solution Assessment

Unclear: reusing the existing serializer is plausible, but exposing it directly or via UDS needs product and security design first. A documented, authenticated, opt-in runtime contract is safer than exporting a private Claude-only helper ad hoc.
