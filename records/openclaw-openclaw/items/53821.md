## Summary

This should stay open: current main accepts the sandbox visibility key but only applies the default sandbox clamp, while this PR makes the existing per-agent sandbox key runtime-active with focused tests and terminal proof; the remaining blocker is maintainer acceptance of the compatibility and sandbox-boundary semantics.

## What This Changes

The PR threads `requesterAgentIdOverride` through session list/history/send/status tools and resolves sandbox session visibility from `agents.list[].sandbox.sessionToolsVisibility` before falling back to defaults, with docs and tests.

## Best Possible Solution
