## What This Changes

The PR makes `agents.list[].sandbox.sessionToolsVisibility` override the default sandbox session-tools clamp for the requesting agent and threads that requester identity through list, history, send, and status tools plus docs/tests.

## Best Possible Solution

Maintain the default `spawned` clamp, keep cross-agent policy checks in force, and have a maintainer choose whether to land this narrow existing-schema-backed sandbox override now or fold it into a later broader per-agent session visibility design.
