## Summary

Keep open: the patch appears technically coherent and has supplied real behavior proof, but it intentionally changes sandboxed session-tool visibility semantics and still needs maintainer acceptance as a compatibility, session-state, and security-boundary decision.

## What This Changes

Threads requester agent identity through sandboxed session list/history/send/status resolution so per-agent sandbox `sessionToolsVisibility` can override the default clamp, with docs and focused tests.

## Best Possible Solution
