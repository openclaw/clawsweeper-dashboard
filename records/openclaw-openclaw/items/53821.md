## Summary

Keep open. Current main still ignores the per-agent sandbox session visibility clamp, and the PR head appears to implement the narrow requester-aware fix cleanly with green checks; the remaining step is maintainer approval for the access-control behavior change.

## What This Changes

This PR makes sandboxed session list/history/send/status tools honor `agents.list[].sandbox.sessionToolsVisibility` before falling back to the default sandbox clamp, with docs, changelog, and regression tests.

## Best Possible Solution

Land the narrow per-agent sandbox clamp support after maintainer approval, while keeping broader per-agent session and A2A policy work in #59149/#59170.

## Reproduction Assessment

Yes. Source reproduction on current main is clear: with `tools.sessions.visibility="all"`, default sandbox clamp `"spawned"`, and a named agent override of `"all"`, the resolver still reads only `agents.defaults.sandbox.sessionToolsVisibility` and the affected tools do not pass requester identity.

## Solution Assessment

Yes. The PR uses the existing session visibility resolver and guard path, adding only requester-aware sandbox clamp precedence; that is narrower than bundling the broader per-agent session/A2A policy tracked separately.
