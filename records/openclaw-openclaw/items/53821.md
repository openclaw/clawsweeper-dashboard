## Summary

Keep open. The PR addresses the current main runtime gap and most earlier override-threading review comments, but the latest head still leaves `session_status` using the default sandbox clamp for ordinary per-agent sessions unless `requesterAgentIdOverride` is explicitly present.

## What This Changes

The PR adds runtime wiring, docs, changelog, API hash, and tests for honoring `agents.list[].sandbox.sessionToolsVisibility` in session list/history/send/status tools.

## Best Possible Solution

Patch `session_status` to use the resolved requester agent id for effective visibility, add a focused no-override per-agent status test, and keep broader per-agent tool/A2A policy design in #59149/#59170.

## Reproduction Assessment

Yes. A static reproduction is clear: run a sandboxed `session_status` tool for `agent:tony:main` with per-agent sandbox visibility `all`, default sandbox visibility `spawned`, and no override; the PR passes `undefined` to effective visibility and still clamps to `tree`.

## Solution Assessment

No. The PR's narrow sandbox-clamp direction is appropriate, but the current patch is incomplete until `session_status` passes the computed requester agent id rather than only the raw override.
