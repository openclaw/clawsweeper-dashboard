## Summary

Keep open. The direction is plausible and current main still resolves sandbox session-tool visibility from the default clamp only, but PR head has a TypeScript-blocking row-checker call, lacks required real behavior proof, and changes session access policy that needs maintainer acceptance.

## What This Changes

The PR threads requester agent identity through sandbox session visibility resolution for sessions list/history/send/status, documents per-agent `sandbox.sessionToolsVisibility` precedence, and updates tests, changelog, and the plugin SDK API checksum.

## Best Possible Solution

Land a typecheck-clean narrow sandbox-clamp override with maintainer policy approval and redacted runtime proof, or intentionally fold it into the broader per-agent access-control surface.

## Reproduction Assessment

Yes for the source-level gap: current main reads only the default sandbox clamp and does not pass requester identity into the affected session tools. I did not establish a live runtime reproduction in this read-only review.

## Solution Assessment

No as submitted. Threading requester identity through the existing visibility helpers is the narrow maintainable direction, but the row-checker call must be made typecheck-clean and the policy change still needs real-behavior proof and maintainer approval.
