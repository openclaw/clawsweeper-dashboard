## Summary

Keep open. Current main does not honor the per-agent sandbox session visibility override, but this PR is not merge-ready because the head has a typecheck blocker in `sessions_list`, lacks external real-behavior proof, and changes a session data-access policy that needs maintainer acceptance.

## What This Changes

The PR adds per-agent `agents.list[].sandbox.sessionToolsVisibility` handling, wires requester identity into session list/history/send/status tools, and updates docs, changelog, tests, and the plugin SDK API checksum.

## Best Possible Solution

Land a typecheck-clean, rebased narrow sandbox-clamp override after maintainer policy approval, or intentionally fold it into https://github.com/openclaw/openclaw/pull/59170 if maintainers want one broader per-agent access-control surface.

## Reproduction Assessment

Yes for the source-level gap: current main reads only the default sandbox clamp and does not pass requester identity into the affected session tools. I did not establish live runtime reproduction in this read-only sweep.

## Solution Assessment

No as submitted. Threading requester identity through the existing visibility helpers is the narrow maintainable direction, but the row-checker call signature must be fixed and real-behavior proof plus policy review are still needed before merge.
