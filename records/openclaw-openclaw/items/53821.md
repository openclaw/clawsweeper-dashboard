## Summary

Keep open. Current main still ignores per-agent sandbox session-tools visibility, and the latest PR mostly implements the requested clamp, but `session_status` still misses the agentSessionKey-derived requester path and can over-restrict configured per-agent visibility.

## What This Changes

The PR adds `agents.list[].sandbox.sessionToolsVisibility` precedence for sandboxed session tools, wires requester identity through session list/history/send/status paths, updates docs/changelog, and adds targeted tests.

## Best Possible Solution

Land the narrow sandbox-clamp override after `session_status` uses one resolved requester identity for visibility and session-id fallback, while leaving broader per-agent sessions/A2A policy to #59149/#59170.

## Reproduction Assessment

Yes. Static reproduction is clear: use a sandboxed `agent:tony:main` session with global `tools.sessions.visibility: "all"`, default sandbox clamp `"spawned"`, and `agents.list[{id:"tony"}].sandbox.sessionToolsVisibility: "all"`; without an explicit override, `session_status` still resolves effective visibility from the default clamp.

## Solution Assessment

No. The PR direction is the narrow maintainable fix, but the status path should pass the computed requester id and should not force spawned fallback solely because `requesterAgentIdOverride` is absent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Pass derived requester into status visibility:** `src/agents/tools/session-status-tool.ts:349-354`
  - body: `session_status` now computes `requesterAgentId`, but this call still passes only `opts?.requesterAgentIdOverride` into effective visibility and then forces session-id fallback to spawned when no explicit override is supplied. A normal sandboxed `agent:tony:main` run with a per-agent `sessionToolsVisibility: "all"` override is therefore still clamped by the default `spawned` setting. Pass the resolved requester id through this path and base fallback restriction on the effective clamp.
