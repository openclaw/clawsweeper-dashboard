## Summary

Keep open: current main still ignores per-agent `sandbox.sessionToolsVisibility` at runtime, and this PR is a coherent implementation with sufficient proof, but it needs maintainer acceptance because it activates an existing config key and broadens a named sandboxed agent's session-tool access.

## What This Changes

Adds runtime, docs, and tests so `agents.list[].sandbox.sessionToolsVisibility` can override the default sandbox session-tool clamp for named agents.

## Best Possible Solution

## Solution Assessment
