## Label Justifications

- `P2`: This is a normal-priority session-tool feature with bounded but real compatibility and sandbox-boundary impact.
- `merge-risk: 🚨 compatibility`: Existing named-agent configs with the per-agent sandbox key can behave differently after upgrade because the key becomes runtime-active.
- `merge-risk: 🚨 session-state`: The PR changes which session rows and targets sandboxed agents can see or use across list, history, send, and status tools.
- `merge-risk: 🚨 security-boundary`: The PR intentionally lets a named sandboxed agent opt out of the default spawned-session clamp and use the broader configured session scope.

## Summary

Keep open: current main and the latest release still accept `agents.list[].sandbox.sessionToolsVisibility` in config but only apply the default sandbox clamp at runtime, while this PR is a focused implementation with sufficient real behavior proof. The remaining blocker is maintainer acceptance of the compatibility/security contract and coordination with https://github.com/openclaw/openclaw/pull/59170.

## What This Changes

The PR threads requester agent identity through the session tools and updates docs/tests so `agents.list[].sandbox.sessionToolsVisibility` can override the default sandbox session-tool clamp for named agents.

## Best Possible Solution

Maintainers should explicitly choose the per-agent sandbox visibility contract, either landing this narrow override with upgrade/security acceptance or folding it into the broader per-agent tools policy work.

## Reproduction Assessment

Yes, source-reproducible: current main accepts `agents.list[].sandbox.sessionToolsVisibility`, but the active session visibility resolver only reads the default sandbox clamp. I did not run a live multi-agent setup in this read-only review.

## Solution Assessment

Unclear whether this is the best final product shape: the implementation is a coherent narrow fix, but maintainers still need to decide whether it lands independently or as part of the broader per-agent session/A2A policy work.
