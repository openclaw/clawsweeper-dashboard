## What This Changes

The PR threads `requesterAgentIdOverride` through session list/history/send/status tooling and updates SDK session-visibility helpers/tests so `agents.list[].sandbox.sessionToolsVisibility` can override the default sandbox session-tools clamp per agent.

## Best Possible Solution

Land the narrow sandbox-clamp override only after the runtime wiring, SDK helper compatibility, tests, docs, and changelog all describe the same precedence: per-agent sandbox clamp first, then defaults, with existing behavior unchanged when no per-agent value is set.
