## Summary

Keep open. Current main still has semantic-only memory-lancedb recall, but this PR cannot merge as-is because it targets the obsolete recall hook, adds a capture regression for explicit remember questions, is conflicting, and still lacks after-fix real behavior proof.

## What This Changes

The PR modifies memory-lancedb source and tests to detect freshness-sensitive queries, overfetch and recency-rerank recalled memories, surface timestamp hints, adjust capture heuristics, and cover stale-vs-newer ordering.

## Best Possible Solution

Land or replace this with a refreshed memory-lancedb fix on before_prompt_build that preserves explicit remember-question capture, adds recency-aware recall and provenance context, and includes real setup proof.

## Reproduction Assessment

Yes, source-reproducible. Seed two matching LanceDB memories where the older one has stronger vector similarity, ask a latest/current prompt, and current main's before_prompt_build path injects db.search(vector, 3, 0.3) results without recency reranking or timestamp/provenance hints.

## Solution Assessment

No. Recency-aware ranking and timestamp context are the right direction, but this branch must be ported to the current before_prompt_build path and must not drop explicit remember requests phrased as questions.
