## Summary

Keep this PR open: the underlying memory-lancedb stale-recall bug still exists on current main, but this branch is not merge-ready because it ports the fix onto the legacy recall hook, blocks explicit remember-requests phrased as questions, and lacks real behavior proof.

## What This Changes

The PR updates memory-lancedb source and tests to detect freshness-sensitive prompts, overfetch and rerank results by recency, add timestamp hints, adjust capture heuristics, strip some metadata envelopes, and cover stale-versus-newer recall ordering.

## Best Possible Solution

Land a refreshed memory-lancedb fix that applies freshness-aware recall in before_prompt_build, preserves latest-user-message normalization and timeout behavior, keeps explicit remember requests capturable, and includes focused regression coverage plus real setup proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed two matching memories where the older item has stronger vector similarity, ask a latest/current prompt, and observe current main injecting top semantic db.search(vector, 3, 0.3) results without recency reranking or timestamp hints.

## Solution Assessment

No, not as currently proposed. Recency-aware ranking and timestamp hints are the right narrow direction, but the patch must move to before_prompt_build and avoid dropping explicit remember requests phrased as questions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Port recall changes to the prompt-build hook:** `extensions/memory-lancedb/index.ts:733-741`
  - body: The PR wires freshness-sensitive auto-recall to before_agent_start and embeds event.prompt directly. Current main registers auto-recall on before_prompt_build, normalizes the latest user message, and runs embedding/search through the timeout wrapper, so this patch would miss the active runtime boundary.
  - confidence: 0.88
- **[P2] Allow explicit remember questions through capture:** `extensions/memory-lancedb/index.ts:429-431`
  - body: The new looksLikeQuestion guard runs before memory-trigger matching, so requests like "Can you remember that my backup day is Sunday?" are rejected even though they contain explicit durable-memory intent. Check durable-memory triggers first or exempt remember/store/archive requests before the generic question skip.
  - confidence: 0.9
