## Summary

Keep open: the branch is a plausible fix for the linked stale-recall problem, but current main has moved auto-recall to `before_prompt_build` and this PR still targets the old hook; it also rejects explicit remember questions and lacks real behavior proof.

## What This Changes

The PR changes `memory-lancedb` source and tests to detect freshness-sensitive prompts, overfetch and rerank LanceDB recall by `createdAt`, expose timestamp hints, adjust capture heuristics, and add regression coverage.

## Best Possible Solution

Land a refreshed memory-lancedb change that performs freshness-aware recall in `before_prompt_build`, preserves current prompt normalization and timeout behavior, protects explicit durable-memory capture, and includes focused regression coverage plus real setup proof.

## Reproduction Assessment

Yes. A source-level reproduction is clear: seed two matching memories where the older item has stronger vector similarity, ask a freshness-sensitive prompt, and current main will inject top semantic matches without recency reranking or timestamp/provenance hints.

## Solution Assessment

No, not as currently proposed. Recency-aware reranking and timestamp hints are the right narrow direction, but the patch needs to move onto `before_prompt_build` and avoid rejecting explicit remember requests phrased as questions.
