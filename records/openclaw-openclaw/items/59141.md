## Summary

Keep open: current main still lacks freshness-aware memory-lancedb recall/provenance, and this PR remains the linked implementation candidate, but the branch is not merge-ready because it wires the fix to the old recall hook and regresses explicit remember-style capture.

## What This Changes

The PR adds freshness intent detection, wider LanceDB candidate recall, recency-weighted reranking, timestamp/provenance hints, and regression tests for stale-vs-newer memory ordering.

## Best Possible Solution

Land a refreshed memory-lancedb fix that applies freshness intent detection, recency-aware reranking, and timestamp/provenance hints in the current before_prompt_build path while preserving explicit durable-memory capture semantics and adding focused regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: seed two matching memories where the older item has stronger vector similarity, ask a freshness-sensitive prompt, and current main's before_prompt_build path will inject db.search(vector, 3, 0.3) results without recency reranking or timestamp/provenance hints.

## Solution Assessment

No, not as currently proposed. Recency-aware reranking plus recordedAt/provenance hints is the right narrow direction, but the patch needs to move onto the current prompt-build hook and avoid dropping explicit remember requests phrased as questions.
