## Summary

Keep this PR open, but it is not mergeable as-is: the underlying memory-lancedb freshness bug still exists on current main, while the branch is wired to the old recall hook, drops some explicit remember requests, and lacks required real behavior proof.

## What This Changes

The PR adds freshness intent detection, candidate overfetching, recency-weighted LanceDB memory reranking, timestamp hints, Spanish capture heuristics, and regression tests in `extensions/memory-lancedb`.

## Best Possible Solution

Land a refreshed memory-lancedb fix that applies freshness intent detection, recency-aware reranking, and timestamp/provenance hints in the current `before_prompt_build` path while preserving explicit durable-memory capture semantics.

## Reproduction Assessment

Yes. A source-level reproduction is clear: seed two matching memories where the older item has stronger vector similarity, ask a freshness-sensitive prompt, and current main will inject the fixed top semantic results without recency reranking or timestamp/provenance hints.

## Solution Assessment

No, not as currently proposed. Recency-aware reranking plus recordedAt/provenance hints is the right narrow direction, but the patch needs to move onto the current prompt-build hook and avoid dropping explicit remember requests phrased as questions.

## Review Findings

  - body: The freshness path is wired to `before_agent_start` and embeds `event.prompt` directly, but current main migrated memory-lancedb auto-recall to `before_prompt_build` with latest-user-message normalization and timeout handling.
