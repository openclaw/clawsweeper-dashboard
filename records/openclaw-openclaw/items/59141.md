Full review comments:

- **[P1] Allow explicit remember questions to capture:** `extensions/memory-lancedb/index.ts:430-431`
  - body: `shouldCapture` now rejects anything that looks like a question before checking memory triggers, so `Can you remember that my backup day is Sunday?` or a Spanish `Recuerda esto: ... ?` request is silently skipped even though it contains a durable fact the user explicitly asked to store. This regresses memory capture and can lose user state; check explicit remember/recuerda triggers before applying a broad question filter, or narrow the filter to information-seeking questions only.
  - confidence: 0.9
- **[P1] Keep cursored capture of all new user facts:** `extensions/memory-lancedb/index.ts:774-779`
  - body: Replacing the agent_end loop with `extractLatestUserTexts(event.messages)` means only the last user message in a completed turn can be captured; if the same turn contains two new durable facts, the earlier one is dropped. Current main avoids replay with `autoCaptureCursors` while still iterating new user messages, so the freshness fix should preserve that cursored capture path.
  - confidence: 0.86
- **[P2] Port freshness recall to before_prompt_build:** `extensions/memory-lancedb/index.ts:733-741`
  - body: This branch registers auto-recall on `before_agent_start`, but current main has memory-lancedb on `before_prompt_build` and a boundary invariant that bundled extensions do not use the legacy hook. Move the freshness-sensitive query and rerank logic into the current before_prompt_build handler so it keeps live config, timeout, overfetch, and sludge-filter behavior.
  - confidence: 0.88
