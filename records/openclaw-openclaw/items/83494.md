## Summary

Keep open: current main still turns live `message` tool sends into tool stream blocks and does not project successful sends into readable assistant bubbles. The problem is source-reproducible and fits a focused Control UI transcript fix.

## What This Changes

Control UI should render successful visible replies sent via `message(action="send")` as normal assistant messages while retaining tool details for debugging.

## Best Possible Solution

Add a narrow Control UI projection for successful `message` `send` tool calls that creates a readable assistant bubble from the outbound text, dedupes against existing assistant text or delivery mirrors, and leaves tool details accessible.

## Reproduction Assessment

Yes, source-reproducible: a live tool message with `toolCallId`, a `message` tool call, and a successful result is normalized as `toolResult` and only appended as a tool message. I did not run UI tests because this review was explicitly read-only.

## Solution Assessment

Yes: the narrowest maintainable fix is in the Control UI chat projection/rendering path, with tests for success, failure, and duplicate suppression. Delivery/session persistence should only change if the UI projection cannot reliably identify successful sends.
