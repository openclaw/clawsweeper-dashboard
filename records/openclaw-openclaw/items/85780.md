## Summary

This PR is still necessary: current main and the latest `v2026.5.22` release still route the missing-unit and stopped-runtime `openclaw node status` recovery hints to stderr, while the not-loaded hint path uses stdout. The patch looks narrow and correct, but the protected `maintainer` label plus mock-only proof mean it needs contributor real CLI proof or explicit maintainer override before merge.

## What This Changes

The PR changes `openclaw node status` missing-unit and stopped-runtime hint output from `defaultRuntime.error` to `defaultRuntime.log`, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Land the narrow stream-routing fix with regression tests once real CLI output proof is added or a maintainer explicitly accepts the proof and compatibility tradeoff.
