## Summary

Keep this open: current main still has the reported lane omission, the source path supports the self-deadlock mechanism, and an open fixing PR references this issue with closing syntax, so the issue should wait for that PR or an equivalent replacement to land.

## What This Changes

Active Memory recall should run its embedded subagent on a non-main lane so the before_prompt_build hook cannot block behind the visible main turn it is preparing.

## Best Possible Solution

Review, update, and land https://github.com/openclaw/openclaw/pull/80255 or an equivalent replacement that adds the dedicated active-memory recall lane with regression coverage, while tracking broader reliability defaults separately.

## Reproduction Assessment

Yes. Current main is source-reproducible: before_prompt_build awaits Active Memory recall, recall omits lane, the embedded runner defaults missing lanes to main, and lanes are serial; the issue also has matching reporter and independent runtime logs.

## Solution Assessment

Yes. Passing a dedicated lane at the Active Memory recall call site is the narrowest maintainable fix; changing the embedded-runner default globally would affect unrelated callers.
