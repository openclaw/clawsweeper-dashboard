## Summary

Keep open: current main still has the active-memory recall path awaiting an embedded run without a dedicated lane, and an open closing PR already tracks the narrow fix. This issue should stay open until https://github.com/openclaw/openclaw/pull/80255 merges or is closed.

## What This Changes

Request Active Memory recall subagents to use a dedicated non-main lane during `before_prompt_build` so they cannot self-block behind the user-visible turn.

## Best Possible Solution

Land the plugin-owned dedicated recall lane assignment with focused regression coverage, while keeping broader Active Memory latency and fail-open policy in the separate reliability track.

## Reproduction Assessment

Yes. Source inspection on current main shows an eligible `before_prompt_build` hook awaits active-memory recall, that recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes serialize by default; I did not run the live canary.

## Solution Assessment

Yes. Passing a stable `active-memory` lane at the Active Memory recall call site is the narrowest maintainable fix because it avoids changing embedded-runner defaults for unrelated callers.
