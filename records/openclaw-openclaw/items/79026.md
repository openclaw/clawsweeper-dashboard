## Summary

Keep open: current main still has the source-level lane self-block path, and the open closing PR that targets this issue has not merged yet.

## What This Changes

The issue reports that Active Memory recall starts an embedded subagent from `before_prompt_build` without an explicit lane, letting it default to the main lane held by the parent turn.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80255 so Active Memory recall uses a dedicated lane with regression coverage, while keeping broader Active Memory/QMD latency work tracked separately at https://github.com/openclaw/openclaw/issues/72015.

## Reproduction Assessment

Yes, by source inspection: an eligible `before_prompt_build` hook awaits Active Memory recall, recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes default to one active task.

## Solution Assessment

Yes, the narrow fix is to pass a stable non-main lane at the Active Memory recall call site; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
