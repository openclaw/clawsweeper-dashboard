## Summary

Keep open: current main still has the source-level Active Memory self-deadlock path, and the open closing PR is the right implementation path but has not merged.

## What This Changes

The issue requests that Active Memory embedded recall avoid re-entering the main command lane from `before_prompt_build`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80255, while keeping broader Active Memory fail-open and latency policy work in https://github.com/openclaw/openclaw/issues/72015.

## Reproduction Assessment

Yes, at source level: an eligible `before_prompt_build` hook awaits Active Memory recall, that recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes run one active task by default. I did not live-run the Telegram canary in this read-only review.

## Solution Assessment

Yes. Passing a plugin-owned `active-memory` lane at the recall call site is the narrow maintainable fix; changing embedded-runner defaults globally would affect unrelated callers.
