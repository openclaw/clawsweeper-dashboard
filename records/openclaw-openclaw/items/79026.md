## Summary

Keep this open: current main still has the active-memory self-deadlock path, and an open linked fix PR with closing syntax is the right place to finish the repair before the issue closes.

## What This Changes

Active-memory recall should avoid the default main embedded-agent lane when invoked from `before_prompt_build` so it cannot block the visible turn.

## Best Possible Solution

Land the dedicated active-memory recall lane fix with regression coverage, then close this issue after the linked PR merges; keep broader nonblocking/fail-open active-memory reliability work separate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `before_prompt_build` awaits active-memory recall, the recall call omits `lane`, omitted embedded lanes resolve to `main`, and command lanes are single-concurrency; the issue comments provide matching timeout logs.

## Solution Assessment

Yes. Passing a dedicated lane at the active-memory recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
