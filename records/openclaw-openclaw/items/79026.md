## Summary

Keep open: current main still contains the source-level Active Memory self-deadlock path, and live GitHub shows an open closing PR targeting the narrow dedicated-lane fix. The issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for Active Memory embedded recall launched from before_prompt_build to avoid re-entering the global main lane and timing out the parent turn.

## Best Possible Solution

Land or replace the open dedicated-lane Active Memory fix with regression coverage, and keep broader fail-open or latency policy work separate.

## Reproduction Assessment

Yes, at source level. The hook awaits Active Memory recall, the recall call omits lane, omitted embedded lanes resolve to main, and command lanes run one active task by default; reporter logs and an independent comment match that path.

## Solution Assessment

Yes. Passing a dedicated active-memory lane at the plugin-owned recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers.
