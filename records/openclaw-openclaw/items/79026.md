## Summary

Keep open: current main and the latest release still have the source-level self-deadlock path, and an open closing PR already targets the narrow lane fix. This issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that active-memory recall should avoid queuing its embedded subagent on the main lane from before_prompt_build, where it can self-block and time out.

## Best Possible Solution

Land or replace the open dedicated-lane fix with regression coverage, while keeping broader active-memory fail-open or latency policy work separate.

## Reproduction Assessment

Yes, at source level. An eligible active-memory before_prompt_build hook awaits recall, the recall call omits lane, omitted embedded lanes resolve to main, and lanes default to one active task; reporter logs provide live corroboration.

## Solution Assessment

Yes. Passing a dedicated lane at the active-memory recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
