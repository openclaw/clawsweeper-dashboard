## Summary

Keep open: current main still has the source-level active-memory self-deadlock path, and an open closing PR already targets the narrow lane fix. The issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks active-memory recall to avoid queuing its embedded subagent on the main lane while running from before_prompt_build.

## Best Possible Solution

Land or replace the open dedicated-lane fix with regression coverage, and keep broader active-memory fail-open or latency policy work separate.

## Reproduction Assessment

Yes, at source level. An eligible active-memory before_prompt_build hook awaits recall, the recall call omits lane, omitted embedded lanes resolve to main, and lanes default to one active task; I did not live-run the Telegram canary in this read-only review.

## Solution Assessment

Yes. Passing a dedicated lane at the active-memory recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
