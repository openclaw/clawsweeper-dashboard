## Summary

Keep open: current main still lets active-memory recall inherit the embedded runner's default main lane from inside the awaited prompt-build hook. The live issue now has an open closing PR for the narrow lane fix, so this issue should remain open until that PR lands or is closed.

## What This Changes

The issue asks for active-memory recall subagents launched during `before_prompt_build` to avoid the global main lane so they cannot self-block behind the user-visible turn.

## Best Possible Solution

Land the plugin-owned dedicated lane assignment with focused regression coverage, and leave broader active-memory fail-open or QMD startup reliability work to the separate reliability track.

## Reproduction Assessment

Yes. Source inspection shows an eligible `before_prompt_build` hook awaits active-memory recall, that recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes are single-concurrency; the issue and comments include matching live timeout logs.

## Solution Assessment

Yes. Passing a stable non-main lane at the active-memory recall call site is the narrowest maintainable fix because it avoids changing embedded-runner defaults or adding a new operator config surface.
