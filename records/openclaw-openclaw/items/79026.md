## Summary

Keep open: current main still lets active-memory recall call the embedded runner without a lane, and the open fixing PR is unmerged, so this should remain open until that fix lands or is replaced.

## What This Changes

Ensure active-memory recall from `before_prompt_build` runs on a non-main lane instead of the embedded runner default.

## Best Possible Solution

Land or replace the narrow dedicated-lane fix with regression coverage, then close this as implemented; keep broader active-memory fail-open or latency policy in the separate reliability track.

## Reproduction Assessment

Yes, by source inspection: an eligible `before_prompt_build` hook awaits active-memory recall, that recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes are single-concurrency. I did not run the live Telegram/macOS canary in this read-only review.

## Solution Assessment

Yes. Passing a dedicated lane at the active-memory recall call site is the narrowest maintainable fix; changing `runEmbeddedPiAgent` defaults globally would affect unrelated callers and plugin contracts.
