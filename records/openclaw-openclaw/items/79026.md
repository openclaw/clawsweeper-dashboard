## Summary

Keep open: current main still matches the reported self-deadlock path, and https://github.com/openclaw/openclaw/pull/80255 is an open closing PR for the focused fix. The source path is strong enough to treat this as a valid bug, but the issue should stay open until that PR lands, is repaired, or is replaced.

## What This Changes

Active-memory recall subagents should avoid defaulting to the main command lane when invoked from `before_prompt_build` so direct chat turns do not self-deadlock.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80255 with the dedicated active-memory lane assignment and regression coverage, while leaving broader fail-open or nonblocking active-memory policy to separate reliability work.

## Reproduction Assessment

Yes. Current main has a source-reproducible path: `before_prompt_build` awaits active-memory recall, recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes are single-concurrency; the issue also includes matching real logs.

## Solution Assessment

Yes for the narrow bug. Passing a dedicated lane at the active-memory recall call site is the smallest maintainable fix because changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
