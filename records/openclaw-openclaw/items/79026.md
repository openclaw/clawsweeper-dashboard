## Summary

Keep open: current main still omits a dedicated lane for the active-memory recall subagent, and the issue is already paired with an open closing PR that should be reviewed or landed before the issue closes.

## What This Changes

The issue requests isolating active-memory embedded recall from the global main lane during `before_prompt_build` to avoid a self-deadlock.

## Best Possible Solution

Review and land a narrow fix that passes a stable non-main lane at the active-memory recall call site with a regression assertion, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows the prompt-build hook awaits active-memory recall, recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes are single-concurrency; the issue also includes matching live timeout logs from multiple users.

## Solution Assessment

Yes. The narrowest maintainable fix is to set a dedicated active-memory lane at the plugin-owned embedded recall call site; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
