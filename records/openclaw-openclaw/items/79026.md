## Summary

Keep open: current main still source-reproduces the active-memory main-lane self-block, and an open PR with closing syntax is already the implementation path, so this issue should not be closed before that PR merges.

## What This Changes

Active-memory recall from `before_prompt_build` can enqueue its embedded run on the global `main` lane, causing a nested lane self-deadlock and hook timeout.

## Best Possible Solution

Merge a focused active-memory fix that passes `lane: "active-memory"` with regression coverage, then close this issue through the linked PR; broader fail-open or latency policy can remain separate.

## Reproduction Assessment

Yes. Source inspection shows an eligible `before_prompt_build` hook awaits active-memory recall, the recall omits `lane`, omitted embedded lanes resolve to `main`, and command lanes are serial by default.

## Solution Assessment

Yes. Passing a stable non-main lane at the active-memory plugin call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
