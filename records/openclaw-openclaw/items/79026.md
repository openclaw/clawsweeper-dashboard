## Summary

Keep open: current main still has the active-memory recall path enqueueing its embedded run without a lane, and the embedded runner still defaults omitted lanes to the global main lane, so the reported self-block is source-reproducible.

## What This Changes

Request that Active Memory's before_prompt_build recall subagent use a non-main embedded-run lane to avoid self-deadlocking the user-visible turn.

## Best Possible Solution

Add a stable dedicated active-memory lane to the embedded recall call, cover it with a hook-level regression test that asserts the lane, and keep broader nonblocking/QMD startup policy in the existing reliability track.

## Reproduction Assessment

Yes, by source inspection: an eligible before_prompt_build hook awaits active-memory recall, that recall calls runEmbeddedPiAgent without lane, and the embedded runner maps omitted lanes to main. I did not run the live Telegram/macOS canary in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to set a stable non-main lane at the active-memory recall call site. Changing runEmbeddedPiAgent defaults globally would affect other callers and plugin contracts, while the open broader latency PR would only reduce the timeout cost, not remove this lane self-block.

## Repair Work Prompt

Fix the active-memory before_prompt_build self-deadlock reported at https://github.com/openclaw/openclaw/issues/79026. In `extensions/active-memory/index.ts`, ensure the hidden recall call to `api.runtime.agent.runEmbeddedPiAgent` uses a stable non-main lane such as `lane: "active-memory"` so it cannot enqueue behind the user-visible main-lane turn that is awaiting the hook. Keep the fix plugin-owned; do not change global `runEmbeddedPiAgent` lane defaults, command queue concurrency defaults, or broader active-memory nonblocking/QMD startup policy. Add or update `extensions/active-memory/index.test.ts` coverage to assert the hook-level embedded recall params include the dedicated lane. Add a user-facing `CHANGELOG.md` fix entry. Coordinate with nearby active-memory reliability work at https://github.com/openclaw/openclaw/issues/72015 and https://github.com/openclaw/openclaw/pull/73667, but do not fold their broader latency/defaults changes into this fix.
