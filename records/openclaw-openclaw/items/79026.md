## Summary

Keep open: current main still routes active-memory recall through `runEmbeddedPiAgent` without an explicit lane while `before_prompt_build` awaits it, and live GitHub shows an unmerged PR with closing syntax for this issue. Closing before that PR lands would be premature.

## What This Changes

The issue requests isolating active-memory recall from the main lane when it is invoked from `before_prompt_build`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80255, or an equivalent plugin-owned fix, so active-memory recall uses a stable non-main lane with regression coverage; keep broader active-memory latency work separate.

## Reproduction Assessment

Yes, by source inspection: an eligible `before_prompt_build` hook awaits active-memory recall, that recall omits `lane`, omitted embedded lanes resolve to `main`, and main-lane work is serialized. I did not run the live gateway canary in this read-only review.

## Solution Assessment

Yes. Passing a dedicated lane at the active-memory recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
