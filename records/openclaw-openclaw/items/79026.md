## Summary

Keep open. Current main still matches the reported source-level deadlock path, and the linked open PR owns the narrow fix but has not merged yet, so this issue should close only after that implementation lands.

## What This Changes

The issue reports that active-memory recall omits an embedded-agent lane inside before_prompt_build, causing the recall subagent to default to the main lane and queue behind the parent turn.

## Best Possible Solution

Land the dedicated active-memory recall lane fix in https://github.com/openclaw/openclaw/pull/80255 with regression coverage, while leaving broader active-memory latency and fail-open behavior to separate reliability work.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main awaits active-memory recall inside before_prompt_build, omits lane on the embedded call, defaults omitted lanes to main, and uses serial command lanes; the issue discussion adds matching live timeout logs.

## Solution Assessment

Yes. Passing a dedicated lane at the active-memory recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
