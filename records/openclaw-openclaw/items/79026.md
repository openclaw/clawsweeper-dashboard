## Summary

Keep open: current main still has the source-reproducible main-lane self-block, and https://github.com/openclaw/openclaw/pull/80255 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that Active Memory recall can enqueue its embedded subagent on the global main lane during before_prompt_build, causing a self-blocking timeout.

## Best Possible Solution

Land a plugin-owned dedicated recall lane with regression coverage through the linked PR, while leaving broader Active Memory fail-open or latency policy to the separate reliability track.

## Reproduction Assessment

Yes. Current main shows before_prompt_build awaiting Active Memory recall, the recall call omitting lane, omitted embedded lanes resolving to main, and command lanes serializing by default.

## Solution Assessment

Yes. Passing a stable non-main lane at the active-memory recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers.
