## Summary

Current main and shipped release v2026.4.26 fix the reported npm EEXIST update path by staging npm global installs in a temporary prefix, verifying them, then swapping the package tree and bin shims instead of letting npm write over the live `/usr/local/bin/openclaw`.

## What This Changes

The issue requests that Docker/container package-manager updates replace an existing global OpenClaw binary instead of failing with npm EEXIST.

## Best Possible Solution

Keep the staged npm global update and verified swap behavior; it solves the container EEXIST path without broad npm `--force`.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-reproducible historically: the report shows npm failing on the live `/usr/local/bin/openclaw` path, while current main avoids that path by installing into a staged prefix before swapping.

## Solution Assessment

Is this the best way to solve the issue? Yes; staging and swapping a verified package tree is safer and more maintainable than adding global npm `--force` to overwrite live binaries.
