## Summary

Current main and shipped releases now address the reported npm EEXIST update path by staging npm global installs in a temporary prefix, verifying them, and swapping the package tree plus bin shim instead of asking npm to overwrite the live global binary. The older force-flag PRs are superseded by that shipped implementation.

## What This Changes

The issue asks for container package-manager updates to replace an existing `/usr/local/bin/openclaw` instead of failing with npm EEXIST.

## Best Possible Solution

Keep the shipped staged npm update flow; any new container update failure should be reported against `v2026.4.26` or newer with fresh update logs.

## Reproduction Assessment

No current-main failure reproduction is available because the reported live-prefix npm overwrite path has been replaced. Source, tests, docs, and release provenance show the current path stages and swaps the package and shim instead.

## Solution Assessment

Yes. The staged verified-prefix swap is a narrower and safer fix than broad npm `--force` because it avoids npm overlaying the live install while still replacing the stale package tree and bin shim.
