## Summary

Close as implemented: current main avoids the reported npm `EEXIST` path by staging npm global updates into a temporary prefix, verifying them, and swapping package files/bin shims; that behavior shipped in `v2026.4.26`.

## What This Changes

The issue requests that package-manager updates in containers replace the existing global `openclaw` binary instead of failing with npm `EEXIST` at `/usr/local/bin/openclaw`.

## Best Possible Solution

Keep the shipped staged-install/swap implementation rather than adding broad npm `--force`, because it addresses the bin collision while avoiding reckless global overwrites.

## Reproduction Assessment

Yes. The report maps to the npm global update path, and current main has focused unit coverage for the staged-prefix update and swap path that replaces the live package/bin shim after verification.

## Solution Assessment

Yes. The current implementation is narrower and safer than the proposed `--force` workaround because npm installs into an empty temporary prefix and OpenClaw owns the verified swap into the live prefix.
