The PR changes `handleGatewayRequest` to always rebuild the gateway method registry from the live active plugin registry and adds a regression test for plugin RPC dispatch with a stale startup snapshot.

## Best Possible Solution

Revise the gateway method registry resolution so live plugin registrations are visible without discarding the supplied attached registry, then cover both stale-startup and attached-newer-than-global cases plus redacted live CLI proof.
