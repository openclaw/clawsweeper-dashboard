## What This Changes

The PR changes `handleGatewayRequest` to use a caller-attached gateway method registry only when it owns the requested method, otherwise falling back to a live registry built from active plugin state, and adds regression tests for both paths.

## Best Possible Solution

Land this branch or an equivalent attached-registry-first live-fallback fix after maintainers either accept the chokepoint proof or receive redacted end-user-path terminal/log proof, then close https://github.com/openclaw/openclaw/issues/94127.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: production WebSocket requests can carry an attached method-registry snapshot, and current main chooses that snapshot before live plugin state that `registerGatewayMethod` updates.

## Solution Assessment

Yes for the code shape, with a proof caveat. The dispatcher is the right layer, and attached-registry-first fallback avoids the older live-only sibling regression while making stale misses reach the live plugin registry.
