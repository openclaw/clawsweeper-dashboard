## Summary

Keep open: current main still calls Feishu `spaceNode.list` only once for the `feishu_wiki` `nodes` action, while the upstream API and SDK expose pagination. PR #37641 attempted a fix but is closed unmerged, so the bug remains actionable.

## What This Changes

The issue asks the Feishu plugin's `feishu_wiki` `nodes` action to paginate `spaceNode.list` so knowledge-base parents with more than one page return all child nodes.

## Best Possible Solution

Implement pagination inside the Feishu plugin's `nodes` action using `page_size`, `page_token`, and `has_more`, with regression coverage and a backward-compatible `nodes` result.

## Reproduction Assessment

Yes, source-level. A mocked `spaceNode.list` response with `has_more: true` and `page_token` would currently return only the first response's `items` because `listNodes` never loops.

## Solution Assessment

Yes, the narrowest maintainable fix is local to `extensions/feishu/src/wiki.ts`: keep the existing tool input surface and paginate internally, using PR #37641 only as context rather than copying it verbatim.
