## Summary

Current main and the latest release still return only one Feishu wiki node page; the linked fix PR was closed unmerged, so this valid, bounded bug remains necessary.

## What This Changes

Make the Feishu wiki `nodes` action return child nodes beyond the first paginated API response.

## Best Possible Solution

Add a focused plugin-local pagination loop using the official SDK fields, preserve the existing `{ nodes }` result contract, fail visibly on page errors, guard repeated or missing continuation tokens, and cover the behavior with concise multi-page tests.

## Reproduction Assessment

Yes at source level: current main performs exactly one paginated `spaceNode.list` request and returns only its `items`, while the official SDK contract exposes continuation fields for later pages. A live Feishu space was not exercised during this read-only review.

## Solution Assessment

Yes, with a narrower shape than the closed PR: pagination belongs inside the existing Feishu-owned `nodes` action, but it should use concise tests and preserve later-page errors rather than exporting internals or relying on the SDK iterator that swallows request failures.
