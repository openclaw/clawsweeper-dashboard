## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/37626 in the Feishu plugin. The `feishu_wiki` `nodes` action currently calls `client.wiki.spaceNode.list` once and returns only `nodes`, so parents with more than one Feishu page are truncated. Keep the fix inside `extensions/feishu`: update the `nodes` schema and implementation to support safe pagination using the pinned SDK contract (`page_size`, `page_token`, `has_more`, response `page_token`). Prefer a bounded automatic fetch-all default or a clear continuable response with metadata, but do not add core config/env knobs. Add focused regression tests for multi-page results, continuation metadata, parent token forwarding, missing token/limit handling, and update the existing routing expectation. Do not edit `CHANGELOG.md`. Use https://github.com/openclaw/openclaw/pull/37641 only as historical context because it was closed unmerged.
