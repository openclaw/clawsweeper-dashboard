## Summary

Keep open: current `main` and the latest release still route the Feishu wiki `nodes` action through a single `spaceNode.list` call without pagination, while the issue and the closed unmerged fix PR document a clear >20-node truncation bug.

## What This Changes

The issue requests automatic pagination for the Feishu `feishu_wiki` tool's `nodes` action so wiki child-node listings return all pages instead of only the first 20 records.

## Best Possible Solution

Land a focused Feishu plugin fix in `extensions/feishu/src/wiki.ts`: request a bounded page size, follow `has_more` and response `page_token`, accumulate all node items, cap the loop, preserve the existing `nodes` result, and add focused tests plus docs/schema updates only for any new public contract.

## Reproduction Assessment

Yes. The issue provides a concrete live `feishu_wiki({ action: "nodes", ... })` call against a parent with 50+ children, and current `main` statically confirms that path makes only one API request and returns only that response page.

## Solution Assessment

Yes. The narrowest maintainable fix is inside the Feishu plugin's wiki tool; no core/plugin SDK change is needed because the existing client call already owns the Feishu API request parameters for this action.
