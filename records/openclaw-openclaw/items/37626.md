## Summary

Current main still has the Feishu wiki pagination bug: `feishu_wiki` `nodes` makes a single `spaceNode.list` call and drops the upstream pagination fields, while the Feishu API and SDK both expose `page_size`, `page_token`, and `has_more`. The prior linked PR was closed unmerged, so the issue remains actionable.

## What This Changes

Request automatic pagination for the Feishu `feishu_wiki` `nodes` action so wiki parents with more than one page of children return the full child-node list.

## Best Possible Solution

Implement pagination inside the Feishu plugin-owned wiki tool, using the existing SDK fields and a bounded loop that preserves the simple `nodes` action while adding focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible. A wiki parent with more than one `spaceNode.list` page will hit the current single-call `listNodes` path, which has no loop or caller-accessible `page_token`.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/feishu/src/wiki.ts`: follow `has_more` and `page_token` with `page_size` capped at 50, add a safety bound, and cover multi-page and empty-page-with-has-more cases in tests.
