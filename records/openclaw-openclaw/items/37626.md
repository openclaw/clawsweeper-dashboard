## Summary

Keep open: current main and the latest release still fetch only one Feishu wiki node page, while the Feishu endpoint and locked SDK expose pagination fields. The earlier fix PR was closed unmerged, the later duplicate was closed in favor of this canonical tracker, and no open landing PR currently owns the fix.

## What This Changes

The issue reports that the Feishu wiki `nodes` action returns only the first `spaceNode.list` page instead of fetching or exposing pagination for all child nodes.

## Best Possible Solution

Implement bounded Feishu wiki node pagination inside the Feishu plugin, keep core plugin-agnostic, add focused regression coverage, and document any additive response fields or tool parameters.

## Reproduction Assessment

Yes, source-reproducible. A Feishu wiki parent with more than one `spaceNode.list` page reaches the current single-call `listNodes` path, which neither follows nor exposes the upstream pagination token.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/feishu/src/wiki.ts` and `extensions/feishu/src/wiki-schema.ts`, using the Feishu SDK pagination contract with a bounded request budget rather than changing core plugin behavior.
