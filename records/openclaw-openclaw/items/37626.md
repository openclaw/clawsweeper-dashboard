## Summary

Keep open: current main and the latest release still make only one Feishu wiki node-list request and return only that page, while the linked fix PR was closed unmerged and the later duplicate was already consolidated into this issue.

## What This Changes

The issue asks the Feishu `feishu_wiki` `nodes` action to handle `spaceNode.list` pagination so large wiki folders return more than the first page of child nodes.

## Best Possible Solution

Implement bounded pagination in the Feishu plugin, preserve the existing `nodes` action, expose official continuation metadata where useful, add focused regression coverage, and keep core plugin-agnostic.

## Reproduction Assessment

Yes, source-reproducible. A Feishu wiki parent with more than one `spaceNode.list` page reaches the current single-call `listNodes` path, which neither follows nor exposes the upstream pagination token.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/feishu/src/wiki.ts`, `extensions/feishu/src/wiki-schema.ts`, and the Feishu wiki skill docs/tests, using the Feishu pagination contract with a bounded request budget rather than changing core plugin behavior.
