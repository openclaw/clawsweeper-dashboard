## Summary

Keep open: current main and the latest release still fetch only one Feishu wiki node page, while the Feishu API and pinned SDK expose pagination fields. The prior fix pull request is closed unmerged, and the later duplicate was already consolidated into this issue.

## What This Changes

The issue requests Feishu `feishu_wiki` `nodes` pagination so wiki parents with more than one `spaceNode.list` page return complete node data or continuation metadata.

## Best Possible Solution

Implement bounded pagination inside the Feishu plugin, expose or return the official continuation fields as needed, add focused regression coverage and skill docs, and keep core plugin-agnostic.

## Reproduction Assessment

Yes, source-reproducible. A Feishu wiki parent with more than one `spaceNode.list` page reaches the current single-call `listNodes` path, which neither follows nor exposes the upstream pagination token.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/feishu/src/wiki.ts` and `extensions/feishu/src/wiki-schema.ts`, using the Feishu SDK pagination contract with a bounded request budget rather than changing core plugin behavior.
