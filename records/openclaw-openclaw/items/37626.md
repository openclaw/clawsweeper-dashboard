## Summary

Keep open: current main and the latest release still call Feishu `wiki.spaceNode.list` once and return only `nodes`, while the official API and pinned SDK expose pagination; the linked fix PR was useful context but closed unmerged.

## What This Changes

The issue requests proper pagination for the Feishu `feishu_wiki` `nodes` action so wiki parents with more than one page of children do not return only the first page.

## Best Possible Solution

Implement the fix in the Feishu plugin by making `nodes` pagination explicit and safe, returning complete or continuable results with pagination metadata and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. A parent with more than one Feishu page goes through a current `listNodes` path that makes one API call and discards the SDK/API pagination fields; I did not run a live credential test.

## Solution Assessment

Yes, the best fix is a narrow Feishu plugin change, not core work: pass/consume Feishu pagination fields, bound automatic fetching if used, expose metadata, and add focused tests.
