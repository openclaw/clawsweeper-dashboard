## Summary

Keep open: current `main` and the latest release still fetch only one Feishu wiki child-node page, while the Feishu SDK contract exposes pagination and the prior fixing PR was closed unmerged after useful reporter verification.

## What This Changes

The issue requests Feishu `feishu_wiki` `nodes` to handle `spaceNode.list` pagination so child nodes beyond the first API page are returned.

## Best Possible Solution

Implement Feishu-owned bounded pagination for wiki `nodes`, preserving the existing action while returning enough metadata for callers to know whether results are complete.

## Reproduction Assessment

Yes, source-reproducible: a Feishu wiki parent with more than one `spaceNode.list` page reaches `listNodes`, which currently makes one SDK call and drops `has_more`/`page_token`. I did not run a live Feishu account reproduction.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/feishu/src/wiki.ts`, `extensions/feishu/src/wiki-schema.ts`, and focused Feishu plugin tests, using the SDK pagination contract with a conservative budget.
