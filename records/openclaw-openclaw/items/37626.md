## Summary

Keep open: current main and the latest release still fetch only one Feishu wiki node page, while the pinned Lark SDK exposes `page_size`, `page_token`, `has_more`, and continuation tokens. The earlier pagination PR was closed unmerged, and the later duplicate was closed back to this issue as the canonical tracker.

## What This Changes

The issue asks for the Feishu `feishu_wiki` `nodes` action to handle paginated `spaceNode.list` results so child nodes beyond the first page are not omitted.

## Best Possible Solution

Implement Feishu-owned bounded pagination for the wiki `nodes` action, preserving the existing action while exposing enough metadata for callers to know whether results are complete.

## Reproduction Assessment

Yes, source-reproducible: a Feishu wiki parent with more than one `spaceNode.list` page reaches `listNodes`, which currently makes one SDK call and drops `has_more`/`page_token`. I did not run a live Feishu account reproduction.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin wiki implementation, schema, skill docs, and focused plugin tests, using the documented SDK pagination contract with a conservative page or node budget.
