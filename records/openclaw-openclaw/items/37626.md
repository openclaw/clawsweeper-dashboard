## Summary

Keep open: current main and latest shipped release still use a single `spaceNode.list` call for Feishu wiki nodes, while the pinned SDK supports pagination and the prior fix PR was closed unmerged.

## What This Changes

The issue requests complete Feishu wiki `nodes` results instead of silently returning only the first `spaceNode.list` page.

## Best Possible Solution

Implement Feishu-owned bounded pagination for wiki `nodes`, preserving the existing action while making incomplete results visible through continuation or truncation metadata and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: any Feishu wiki parent whose `spaceNode.list` response has `has_more=true` reaches a current `listNodes` path that makes one SDK call and drops the continuation token. I did not run a live Feishu account reproduction.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/feishu/src/wiki.ts`, `extensions/feishu/src/wiki-schema.ts`, and focused Feishu plugin tests, using the SDK pagination contract with bounded request limits.
