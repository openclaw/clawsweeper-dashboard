## Repair Work Prompt

Fix the Feishu wiki `nodes` pagination bug tracked by https://github.com/openclaw/openclaw/issues/37626. Current `extensions/feishu/src/wiki.ts` calls `client.wiki.spaceNode.list` once and current `extensions/feishu/src/wiki-schema.ts` exposes no pagination fields, so callers can receive silently truncated child node lists. Implement a bounded Feishu-owned pagination path for `nodes`: forward documented pagination parameters or auto-fetch with a conservative max page/node budget, return `has_more`/`page_token` or equivalent completion/truncation metadata so callers know whether results are complete, and do not change unrelated Feishu actions. Use the pinned `@larksuiteoapi/node-sdk@1.64.0` contract and Feishu docs for `page_size`, `page_token`, response `has_more`, and response `page_token`. Add focused regression coverage for multi-page results, token chaining, single-page results, empty results, API errors, missing next token, and budget truncation; update Feishu wiki skill/docs if the tool response or inputs change; add a user-facing changelog entry because this fixes visible plugin behavior. Preserve account routing and string-only wiki space ID validation.
