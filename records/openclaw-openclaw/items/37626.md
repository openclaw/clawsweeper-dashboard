## Summary

Keep open: current main and the latest stable tag still fetch only one Feishu wiki node page, while the upstream Feishu endpoint and SDK expose pagination fields; the prior pagination PR was closed unmerged and the later duplicate was closed in favor of this canonical tracker.

## What This Changes

The issue requests Feishu wiki `nodes` pagination so `feishu_wiki` can return or traverse child nodes beyond the first `spaceNode.list` page.

## Best Possible Solution

Implement bounded Feishu wiki node pagination inside the Feishu plugin, keep core plugin-agnostic, add focused regression coverage, and document any new response fields or tool parameters.

## Reproduction Assessment

Yes, source-reproducible. A Feishu wiki parent with more than one `spaceNode.list` page reaches the current single-call `listNodes` path, which neither follows nor exposes the upstream pagination token.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `extensions/feishu/src/wiki.ts` and `extensions/feishu/src/wiki-schema.ts`, using the existing Feishu SDK pagination contract with a bounded request budget.
