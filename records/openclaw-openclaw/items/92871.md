## Best Possible Solution

Derive assistant status from the shared progress display model, dedupe unchanged status writes, add focused Slack dispatch coverage, and attach redacted live Slack proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main and the latest release set Slack assistant status to `is typing...` and clear it without tool-specific updates, while `onToolStart` already receives tool events.

## Solution Assessment

No. The `onToolStart` seam is appropriate, but this implementation duplicates the shared progress display registry, lacks dedupe for repeated update events, and does not prove the live Slack behavior.
