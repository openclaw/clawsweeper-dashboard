## Summary

Keep this open: current main and the latest release still allow an isolated cron `sessions_send` fire-and-forget announce to enter the A2A ping-pong path, and the linked open fix PR has not merged yet.

## What This Changes

The issue asks for isolated cron `sessions_send` with announce delivery and `timeoutSeconds: 0` to deliver one-way without injecting the target agent's reply context back into the cron run.

## Best Possible Solution

Review and land the narrow linked core fix, or an equivalent change, that detects isolated cron requesters and skips requester-side A2A ping-pong while preserving target announce delivery and normal requester behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: isolated cron produces a `:cron:` run session key, current `sessions_send` starts A2A for `timeoutSeconds: 0`, and A2A first steps the requester when turns are enabled. I did not run a live cron/Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the best current path is the narrow linked fix that gates requester-side ping-pong for isolated cron requesters rather than changing all fire-and-forget A2A behavior.
