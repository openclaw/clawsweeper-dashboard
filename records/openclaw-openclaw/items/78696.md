## Summary

This PR should stay open, but it is not merge-ready: the exec preflight guard is aimed at the right bug, yet the current branch still lets one valid npm global-update form bypass the guard and the external contributor has supplied only unit-test proof.

## What This Changes

The PR adds an exec preflight parser that rejects raw global `openclaw` package-manager installs or updates, regression tests for direct and wrapped commands, and a changelog entry for the linked self-update bug.

## Best Possible Solution

Keep the exec-side guard, extend it to all supported npm global-location forms, and require redacted real behavior proof before normal maintainer merge review.

## Reproduction Assessment

Yes. The linked maintainer issue has concrete Ubuntu field logs for raw `sudo npm install -g openclaw@latest` followed by stale chunk import failures, and current main source has no exec-side guard for that command class.

## Solution Assessment

No, not as currently implemented. The exec preflight boundary is the right place for the fix, but the parser needs to cover npm's separate-value `--location global` form and the PR needs real behavior proof before it is merge-ready.
