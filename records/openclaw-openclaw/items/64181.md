## Summary

Keep open: the bug is real and the patch improves the dash-truncation path, but it still ignores the embedded runner's `isError` payload flag and therefore can slugify short provider failures; contributor-supplied after-fix real behavior proof is also still missing.

## What This Changes

The PR extracts slug response normalization into a helper, adds text-shape rejection and post-truncation dash stripping, and expands colocated slug-generator unit tests.

## Best Possible Solution

Reject `payload.isError === true` before slugification, keep the post-truncation dash fix and focused tests, then land after redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. A mocked `runEmbeddedPiAgent` returning `[{ text: "quota exceeded", isError: true }]` follows the current code path into a non-null slug, and session memory uses non-null slugs in filenames.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but rejecting the embedded runner's `isError` payload contract before text slugification is the narrower and more reliable fix.

## Review Findings

Overall correctness: patch is incorrect
