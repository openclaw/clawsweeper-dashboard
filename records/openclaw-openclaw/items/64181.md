## Summary

Keep open: the PR is focused and useful, but it still misses the embedded-runner error-payload contract, lacks after-fix real behavior proof, and overlaps an open slug-generation rewrite that touches the same helper.

## What This Changes

The PR extracts LLM slug response normalization into `slugifyLLMResponse`, rejects long or multiline slug candidates, strips dashes after truncation, and adds colocated slug-generator tests.

## Best Possible Solution

Reject `payload.isError === true` before slugifying, keep the post-truncation dash fix and focused tests, add generator-level regression coverage, coordinate with the open lightweight slug-generation PR, then land after redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. A mocked `runEmbeddedPiAgent` result with `[{ text: "quota exceeded", isError: true }]` still follows the PR head into non-null slug generation, and session memory uses non-null slugs in filenames.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the reliable fix must honor the embedded runner's `isError` payload contract before calling the text slugifier.
