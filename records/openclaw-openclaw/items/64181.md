## Summary

Keep this PR open: it is a focused bug fix, but the current head still misses the embedded-runner error-payload contract and lacks required after-fix real behavior proof.

## What This Changes

The branch extracts slug response normalization into a helper, rejects several non-slug response shapes, strips dashes after truncation, and adds colocated slug-generator unit tests.

## Best Possible Solution

Amend this PR to reject `payload.isError === true` before text slugification, keep the post-truncation dash fix and focused tests, then land after redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. A mocked `runEmbeddedPiAgent` result with `[{ text: "quota exceeded", isError: true }]` follows current main and the PR head into non-null slug generation, and session memory uses non-null slugs in filenames.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the narrow reliable fix must reject the embedded runner's `isError` payload contract before calling the text slugifier.
