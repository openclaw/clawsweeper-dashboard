## Summary

Keep open: the PR is a useful focused fix for session-memory slug filenames, but PR head still ignores the embedded runner's `isError` payload contract and lacks redacted after-fix real behavior proof from an OpenClaw memory flush.

## What This Changes

The PR extracts slug response normalization into `slugifyLLMResponse`, rejects non-slug-shaped text, strips dashes after truncation, and adds focused slug-generator tests.

## Best Possible Solution

Reject `payload.isError === true` before slugifying, keep the post-truncation dash fix and focused tests, add generator-level regression coverage, then land after redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. A mocked `runEmbeddedPiAgent` result with `[{ text: "quota exceeded", isError: true }]` would still follow PR head into non-null slug generation, and session memory uses non-null slugs in filenames.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the narrow reliable fix must honor the embedded runner's `isError` payload contract before calling the text slugifier.
