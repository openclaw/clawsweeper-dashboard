## Summary

Keep open: this is a focused, plausible bug fix, but the current branch still slugifies `isError` payload text and lacks after-fix real OpenClaw behavior proof.

## What This Changes

The PR extracts slug response normalization into `slugifyLLMResponse`, rejects malformed slug-like text, strips dashes after truncation, and adds focused slug-generator unit tests.

## Best Possible Solution

Land a narrow repair that rejects `payload.isError === true` before slugification, keeps the post-truncation dash strip, adds `generateSlugViaLLM` boundary coverage, and includes redacted after-fix real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main can be reproduced with a mocked `runEmbeddedPiAgent` returning `{ text: "quota exceeded", isError: true }`, because `generateSlugViaLLM` reads only `text` and session memory persists any non-null slug.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the patch should reject `payload.isError === true` before calling `slugifyLLMResponse` instead of relying only on text-shape heuristics.

## Review Findings

Overall correctness: patch is incorrect
