## Summary

This PR should stay open: it targets a real session-memory filename bug, but the patch still ignores the embedded runner's `isError` payload contract, so short provider failures can still become slugs, and contributor after-fix proof is still missing.

## What This Changes

The PR extracts slug response normalization into an exported helper, rejects non-slug-shaped responses, strips dashes after truncation, and adds slug-generator tests.

## Best Possible Solution

Reject `payload.isError === true` before slugifying, keep the post-truncation dash fix and focused tests, add generator-level regression coverage, then land after redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. A mocked `runEmbeddedPiAgent` result with `[{ text: "quota exceeded", isError: true }]` still follows the PR head into non-null slug generation, and session memory uses non-null slugs in filenames.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the narrow reliable fix must honor the embedded runner's `isError` payload contract before calling the text slugifier.
