## Summary

The branch addresses a real current-main bug, but it should stay open because it still accepts short error payloads when `isError` is set and lacks after-fix real behavior proof.

## What This Changes

The PR extracts slug response normalization into `slugifyLLMResponse`, adds shape checks, strips dashes after truncation, and adds focused slug-generator unit tests.

## Best Possible Solution

Reject `isError` payloads before slugification, keep the post-truncation dash strip and focused tests, then land after redacted after-fix OpenClaw proof.

## Reproduction Assessment

Yes, at source level. Current main can be exercised with a mocked `runEmbeddedPiAgent` returning `{ text: "quota exceeded", isError: true }`; `generateSlugViaLLM` reads only `text`, and session memory uses any non-null slug in the filename.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the patch should reject `payload.isError === true` before calling `slugifyLLMResponse` instead of relying only on text-shape heuristics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject error payloads before slugifying:** `src/hooks/llm-slug-generator.ts:134`
