## Summary

Keep open: this is a real session-memory filename bug, but the PR head still ignores `payload.isError`, so short provider error payloads can still become memory slugs; it also still lacks redacted after-fix real behavior proof.

## What This Changes

The PR extracts slug response normalization into a tested helper, rejects long or multiline slug candidates, and strips leading/trailing dashes after truncation in `generateSlugViaLLM`.

## Best Possible Solution

Reject `payload.isError === true` before slugifying, keep the post-truncation dash fix and focused tests, add generator-level regression coverage, coordinate with the lightweight slug PR only if it lands first, then require redacted real memory-file proof.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, a mocked `runEmbeddedPiAgent` result with `[{ text: "quota exceeded", isError: true }]` still flows into non-null slug generation, and the session-memory hook uses non-null slugs in filenames.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the mergeable fix must honor the embedded runner's `isError` payload contract before slugifying and prove the fallback path in a real memory flush.
