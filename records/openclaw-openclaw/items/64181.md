## Summary

Keep open: this is a plausible focused bug fix, but the PR head still accepts embedded-runner payloads marked `isError: true` when the text is short enough, so the core error-payload filename bug is not fully fixed. The branch also still lacks contributor-supplied after-fix real behavior proof.

## What This Changes

The PR extracts slug response normalization into a helper, adds slug-format regression tests, rejects several long/error-looking text responses, and strips dashes after truncation in the session-memory LLM slug generator.

## Best Possible Solution

Amend this PR to reject `payload.isError === true` before slugifying text, keep the post-truncation dash fix and focused tests, then land after redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. Current main and PR head read `payloads[0].text` without checking `isError`, while the embedded runner emits error payloads with `isError: true` and session memory uses any non-null slug in the filename.

## Solution Assessment

No. The helper extraction and dash-order fix are useful, but the reliable fix needs to inspect the payload object and return `null` for `isError: true` before text slugification.
