## Summary

Keep open. The PR is useful source work, but it is not merge-ready against current main because the restore/drain path conflicts with the shared session-switch helper, can replay restored messages before an existing server-side run finishes, silently loses queued attachments after refresh, and adds localStorage retention that needs maintainer review.

## What This Changes

The PR adds a localStorage-backed chat queue helper, persists chatQueue mutations, restores queues during lifecycle/session changes, and flushes restored queues after gateway hello.

## Best Possible Solution

Implement durable WebChat queue persistence inside the shared chat/session lifecycle with active-run completion coordination, explicit attachment replay or visible rejection behavior, focused tests, and a user-facing changelog entry.

## Reproduction Assessment

Yes. A static reproduction is clear: refresh during an active response with queued follow-ups, restore the queue before gateway hello, then the PR drains after chatRunId is cleared while the prior server-side run can still be active; queued attachments also replay as metadata without payload bytes after refresh.

## Solution Assessment

No. localStorage may be part of the eventual fix, but this patch is not the narrowest maintainable solution until it uses the shared session helper, coordinates with active-run state, and defines attachment and retention semantics.

  - body: On refresh during an active response, restored queued messages are flushed immediately after hello clears chatRunId. The old server-side run can still be active, so the queued follow-up can start as a separate run instead of waiting behind the response it was queued after.
  - confidence: 0.85

  - body: chatQueue attachments only carry metadata; the payload bytes live in the in-memory attachment payload store.   - body: Current main routes session switching through switchChatSession and app-render.helpers.ts owns queue save/restore. Adding localStorage restore in the old app-render callback misses the shared helper path and conflicts with the owner boundary added by #73679.
  - confidence: 0.84
