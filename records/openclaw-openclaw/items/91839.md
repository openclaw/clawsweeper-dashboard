## Summary

Keep open. Current main has partial announce-cleanup hardening, but the reported central failure is not fully solved: plain provider `Model not available` wording is still not clearly terminal for subagent announce delivery, failed announce cleanup still probes `chat.history`, and no merged PR or canonical single issue fully owns this provider-error-plus-history-pressure case.

## What This Changes

The issue reports a gateway starvation path where a provider rejects a bad configured model and the subagent completion announce path retries while repeatedly touching chat.history.

## Best Possible Solution

Keep this issue open for maintainer review of a bounded runtime fix that treats terminal provider model errors as non-retryable in subagent completion handoff, preserves the newer blocked-delivery cleanup state, and caps or avoids chat.history probing when delivery is already known to have failed terminally.

## Reproduction Assessment

Yes for source-level reproduction of the involved retry/history paths, but not for a live systemd/provider reproduction in this read-only review. Current main still has non-terminal announce delivery failures and a chat.history mirror probe on failed announce cleanup, while the exact starvation rate remains unverified here.

## Solution Assessment

Unclear as a final fix. Current main partially hardens announce give-up, so maintainers should choose a narrow remaining fix that classifies or bounds terminal provider model errors and history probing without undoing the newer delivery-state behavior.
