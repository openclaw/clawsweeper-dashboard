## Summary

Keep open. Current main still has the reported source surface: model-unavailable wording is not classified as terminal/model-not-found, subagent completion announce failures still flow through retry/give-up accounting, and the registry can still probe requester `chat.history` while resolving failed delivery mirrors.

## What This Changes

The issue asks OpenClaw to bound a terminal provider `model_not_available` failure so the subagent announce path cannot retry into `chat.history` storms or gateway event-loop starvation.

## Best Possible Solution

Keep this open as a distinct mixed provider/announce/history failure until maintainers choose a bounded core-runtime fix that stops terminal model availability errors from entering the announce retry/history path and returns a clear parent failure.

## Reproduction Assessment

No live systemd/provider reproduction was run in this read-only review. Source inspection on current main confirms the implicated provider-error classification gap, subagent announce retry/give-up path, and bounded `chat.history` mirror check still exist.

## Solution Assessment

The requested direction is valid, but not yet the full best fix by itself. The best first fix should narrow the runtime boundary so terminal model-unavailable errors stop announce retries and wake or fail the parent cleanly, with broader config validation and watchdog recovery coordinated separately.
