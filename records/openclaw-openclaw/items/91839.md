## Summary

Keep open. ## What This Changes

The issue reports that a terminal provider `Model not available` error during subagent completion announce can escalate into repeated `chat.history` control-plane calls and gateway starvation instead of a bounded failure.

## Best Possible Solution

Land a maintainer-approved bounded failure path that classifies terminal model-unavailable provider errors correctly, prevents announce cleanup from amplifying control-plane polling, wakes or blocks the parent completion deterministically, and tracks config hot-reload validation separately if needed.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection does show that the exact reported provider wording is not handled by the current model-not-found patterns and that announce cleanup can still call `chat.history` during the retry/give-up path.

## Solution Assessment

Unclear as a single automatic fix. The narrowest maintainable direction likely combines provider error classification with bounded announce delivery semantics, while config hot-reload rejection and watchdog recovery need maintainer product choices.
