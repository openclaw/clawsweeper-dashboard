## Summary

Keep open. Current main and v2026.6.8 still do not show a complete fix for the reported cross-surface path: bare provider `Model not available` text is not covered by the model-not-found classifier, subagent announce delivery still has retry/give-up behavior, and failed announce cleanup can still call requester `chat.history`.

## What This Changes

The issue asks OpenClaw to bound a bad provider model configuration so a terminal model-unavailable error cannot trigger subagent announce retries, chat.history pressure, parent-session stalls, or gateway unresponsiveness.

## Best Possible Solution

Keep this as the combined core-runtime tracker and land a bounded fix that classifies terminal model-unavailable errors, fails or wakes the parent session once, and caps history/mirror work; handle stricter config validation and watchdog recovery through maintainer-approved related work.

## Reproduction Assessment

No live systemd/provider reproduction was run here. The issue is source-reproducible at medium confidence because current main still lacks the exact terminal `Model not available` classification and still has the announce retry/give-up plus chat.history mirror paths.

## Solution Assessment

Yes, the issue remains valid, but the best fix should be staged: first a narrow runtime/provider classification and subagent delivery bound, then maintainer-approved config validation or watchdog changes if needed.
