## Summary

Keep open. Current main and v2026.6.9 still leave the central path unresolved: the reported bare `Model not available: fable` text is not classified as a model-not-found provider error, and the subagent announce retry plus requester `chat.history` mirror-probe path still exists.

## What This Changes

The issue asks OpenClaw to bound a bad provider-model configuration so terminal model-unavailable errors do not cascade into subagent announce retries, `chat.history` polling, and gateway starvation.

## Best Possible Solution

Keep this as the combined core-runtime tracker and land a staged maintainer-approved fix: classify terminal model-unavailable errors, fail or wake the parent once, and bound announce/history work, while using related issues for stricter config validation and watchdog recovery policy.
