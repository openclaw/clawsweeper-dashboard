## Summary

Keep open. Current main still has the reported cross-surface failure shape: the exact model-availability wording is not classified as a terminal provider/model error, subagent completion announce retry/give-up paths remain active, and failed delivery cleanup can still query requester `chat.history`; no merged PR or canonical open item fully owns this combined provider-error, announce, history-pressure, and gateway-starvation report.

## What This Changes

A bad provider model reference can produce a terminal model-unavailable error during subagent completion announce, leading to retries, history-refresh pressure, and gateway starvation instead of a bounded failure.

## Best Possible Solution

Add a bounded core-runtime fix that classifies terminal provider model-availability errors as non-retryable for subagent completion handoff, wakes or blocks the parent with a clear failure, and caps history/mirror work; coordinate stricter config validation and watchdog recovery through the related issues.

## Reproduction Assessment

No live systemd/provider reproduction was run; source inspection shows the provider-error classifier, announce retry/give-up, and `chat.history` mirror paths still exist and lack the exact terminal `Model not available` handling.

## Solution Assessment

Yes, the reported failure should be fixed, but the best fix is a narrow runtime/provider classification and subagent delivery bound first; hard config rejection and watchdog recovery should remain separate maintainer decisions.
