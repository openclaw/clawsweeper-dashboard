## Summary

Keep open. Current main still has the reported failure surface: model availability wording is not clearly classified as terminal, subagent completion announce still has bounded retry/give-up paths, and the history-refresh paths involved in the reported gateway pressure remain present.

## What This Changes

A bad provider model reference can be accepted and later rejected by the provider, after which subagent completion announce retry and history paths can degrade the gateway instead of failing the child run once.

## Best Possible Solution

Add a bounded core-runtime fix that classifies terminal provider model availability errors as non-retryable for subagent completion handoff, wakes or blocks the parent with a clear failure, and adds focused regression coverage for retry and history bounds.

## Reproduction Assessment

No live systemd/provider reproduction was established here; source inspection confirms the involved subagent announce retry, provider-error classification, and history-refresh paths exist, and the exact `Model not available` wording is not clearly terminal-classified.

## Solution Assessment

The requested direction is valid, but the best fix should narrow the runtime boundary first: terminal provider/model errors should stop announce retries and publish a bounded parent failure, with broader config validation and watchdog recovery coordinated with related issues.
