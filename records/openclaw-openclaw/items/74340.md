## Summary

Keep open. The patch is useful for direct successful completion captures, but it leaves an outcome-less frozen-result refresh path that can still capture raw tool text, and there are overlapping open fixes for the same #74073 bug family that need a canonical maintainer choice.

## What This Changes

The PR adds an `ok`-status guard before subagent output selection falls back to `latestRawText` and adds a regression test for successful tool-only histories.

## Best Possible Solution

Land one canonical subagent announce fix that prevents raw tool/toolResult text from becoming a successful completion result across direct completion capture and frozen-result refresh, while preserving timeout/error partial progress and updating docs and changelog.

## Reproduction Assessment

Yes. A source-level reproduction is a successful tool-only history whose latest message is a tool result: current selection reaches `latestRawText`, and with this PR any refresh capture that omits `outcome` still reaches that fallback.

## Solution Assessment

No. The outcome-gated selector guard is the right direction for direct completion freeze, but the safer fix is to carry a successful-capture mode or outcome through refresh capture so raw fallback is suppressed consistently.
