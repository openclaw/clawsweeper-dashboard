## Summary

Keep open: the PR improves the direct successful-completion path, but it still leaves an outcome-less refresh path able to freeze raw tool output, and the user-facing result contract docs/changelog are not updated.

## What This Changes

The PR adds an `ok`-outcome guard so `selectSubagentOutputText` does not return `latestRawText` for successful subagent runs, plus a regression test for tool-only histories.

## Best Possible Solution

Land one canonical subagent announce fix that prevents raw tool/toolResult text from becoming a successful completion result across direct capture and lifecycle refresh, while preserving timeout/error partial-progress behavior and updating docs plus changelog.

## Reproduction Assessment

Yes. A source-level reproduction is a successful tool-only history whose latest message is a tool result: current main returns `latestRawText`, and with this PR an outcome-less refresh capture can still take that fallback.

## Solution Assessment

No. The outcome-gated guard is the right direction for direct successful completion, but the safer fix is to carry a success/no-raw capture mode through refresh or pass the stored run outcome into refresh capture as well.
