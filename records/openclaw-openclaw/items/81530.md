Keep open: current main covers several Telegram forum-topic routing paths, but the latest discussion points to a still source-reproducible sequential-lane gap when Telegram provides `message_thread_id` without the optional forum flags.

## What This Changes

The issue reports Telegram supergroup forum-topic inbound messages showing typing but not producing a topic session or reply.

## Best Possible Solution

Teach Telegram sequential lane selection to preserve topic lanes when reliable topic metadata is present even if optional forum flags are omitted, then prove it with focused regression tests and a live redacted Telegram topic probe.
