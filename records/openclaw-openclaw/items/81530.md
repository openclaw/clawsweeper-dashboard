## What This Changes

The issue reports that Telegram supergroup forum-topic inbound messages show bot typing but do not create or update the expected topic-scoped session or generate a reply.

## Best Possible Solution

Align Telegram sequential/spool lane classification with the handler’s resolved forum metadata, add regression coverage for metadata-omitted forum-topic updates, and require redacted live Telegram topic proof before landing.

## Reproduction Assessment

No for the full live symptom: I did not run a real Telegram supergroup-topic probe on current main. Yes at source level: current code and tests show a metadata-omitted topic update can be sequenced as the chat-wide lane instead of the topic lane.

## Solution Assessment

Yes, the best fix is a narrow Telegram sequencing/spool alignment, not the broader `allowed_updates` proposal. The implementation should avoid misclassifying regular supergroup reply threads while matching the handler’s resolved forum-topic session behavior.
