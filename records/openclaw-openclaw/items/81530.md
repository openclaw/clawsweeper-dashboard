## Summary

Keep open: this is a recent, documented Telegram forum-topic behavior bug with a plausible current-main source gap in the sequential lane key path, and no merged/open canonical fix PR currently owns the remaining case.

## What This Changes

The issue reports that Telegram supergroup forum-topic inbound messages show typing but do not create or update the expected topic-scoped session.

## Best Possible Solution

Add a narrow Telegram sequential-key/forum metadata fix so delivered supergroup topic updates keep topic-specific lanes even when optional raw forum flags are absent, while preserving regular non-forum group behavior and adding regression plus live/diagnostic proof.

## Reproduction Assessment

No for a full live reproduction: I did not run a current-main Telegram supergroup topic probe. Yes for a source-level reproduction: the current key resolver can be followed to a chat-wide lane when a supergroup update has `message_thread_id` but lacks both `chat.is_forum` and `is_topic_message`.

## Solution Assessment

Unclear for the earlier allowed-updates and General-topic suggestions, because current main already handles message updates and General fallback. The narrower maintainable direction is to stop re-deriving incomplete forum state in `getTelegramSequentialKey`, or add a carefully bounded fallback with regression coverage.
