## Summary

Keep open: current `main` still has the shorter `skills update` validation errors and this PR carries the useful CLI diagnostic change, but the added tests are environment-sensitive when `OPENCLAW_PROFILE` is set, so the patch is not merge-ready as written.

## What This Changes

The branch imports `formatCliCommand`, adds usage examples to the two invalid `openclaw skills update` argument errors, and adds command tests for those guards plus the empty `--all` path.

## Best Possible Solution

Keep the runtime diagnostic change, but make the tests profile-aware by stubbing or clearing the command-format environment, or by asserting the relevant formatted command pieces before normal CI.

## Reproduction Assessment

Yes. Source inspection on current main shows the two invalid `skills update` paths still emit the shorter messages, and the PR body includes copied after-fix terminal output for the missing-argument path.

## Solution Assessment

No, not merge-ready as written. The runtime approach is narrow and consistent with existing CLI formatting, but the new tests should not require a contiguous `openclaw skills update` substring when profile-aware formatting is active.
