## Summary

Keep open: the CLI diagnostic change is useful and still missing on current main, but the PR's new assertions are not profile-aware and can fail when `OPENCLAW_PROFILE` causes the formatted example command to include `--profile`.

## What This Changes

The PR adds example-bearing validation errors for invalid `openclaw skills update` argument combinations and adds command tests for those guards.

## Best Possible Solution

Land the narrow CLI diagnostic improvement once the new assertions are deterministic in profiled environments and the focused command tests pass.

## Reproduction Assessment

Yes. Source inspection on current main shows the shorter validation messages, and source inspection of the PR plus existing formatter tests gives a high-confidence reproduction of the profile-sensitive assertion failure.

## Solution Assessment

No, not merge-ready as written. The message change is the right narrow direction, but the tests should clear or stub `OPENCLAW_PROFILE`, or assert command pieces without requiring `openclaw skills update` to be contiguous.
