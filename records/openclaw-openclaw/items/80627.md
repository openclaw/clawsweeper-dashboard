## Summary

Keep open: the CLI diagnostic change is useful, but the new assertions are not profile-aware and can fail when `OPENCLAW_PROFILE` is set, so the PR needs a small test repair before merge.

## What This Changes

The PR adds example-bearing validation errors for invalid `openclaw skills update` argument combinations and adds command tests for those paths.

## Best Possible Solution

Keep the CLI copy improvement, but make the added assertions deterministic with profile-aware formatting before normal CI and maintainer review.

## Reproduction Assessment

Yes. Source inspection shows current main emits the shorter validation messages, and the PR-head test fragility follows directly from `formatCliCommand` inserting `--profile` when `OPENCLAW_PROFILE` is set.

## Solution Assessment

No, not as submitted. The CLI diagnostic change is narrow and maintainable, but the new tests should stub or clear `OPENCLAW_PROFILE`, or assert profile-aware command pieces, before merge.
