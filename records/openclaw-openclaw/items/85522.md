## Summary

Keep open: the patch targets a real current-main gap in async gateway exec visibility, but it is not merge-ready because proof is mock-only and failed approved execs would be emitted through a normalizer path that reports `phase: "end"` command output as completed before checking `status: "failed"`.

## What This Changes

The PR emits a synthetic `command_output` agent event for approved async gateway-hosted exec completions, adds optional command/approval identifiers to that event data, and adds mocked regression coverage for visible and hidden routing.

## Best Possible Solution

Emit the event through the existing agent-event routing contract, keep external-channel output hidden and redacted, fix failed `command_output` SDK normalization, and add redacted real gateway approval proof before merge.
