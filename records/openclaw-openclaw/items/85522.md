## Summary

Keep this PR open, but it is not merge-ready: the command-output event is useful, yet the current patch can bypass current-main stale-followup drop protection after rebase and can misclassify failed approved execs as completed for SDK consumers.

## What This Changes

The PR adds redacted end-phase `command_output` agent events for approved async gateway exec completions, extends command-output event data with `command` and `approvalId`, and adds gateway exec regression tests.

## Best Possible Solution

Keep the gateway-owned event emission, but gate it with the same stale-followup rebind decision, preserve failed/blocked status in SDK normalization, and rebase over the current exec-approval path before merge.
