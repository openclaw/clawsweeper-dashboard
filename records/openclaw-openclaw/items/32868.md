## What This Changes

The issue reports that same-channel block-streamed text replies are queued but not delivered to the channel before subsequent tool execution begins.

## Best Possible Solution

Land a narrow same-channel block delivery barrier so accepted text_end block replies drain through dispatcher delivery before tool execution proceeds, with focused ordering tests and live transport proof when feasible.

## Reproduction Assessment

Yes. Current main source shows the pre-tool flush waits only for the block callback chain while same-channel block delivery is queued through a non-awaitable dispatcher path; multiple channel reports support the same timing failure.

## Solution Assessment

Yes. The narrowest maintainable fix is the linked delivery barrier or equivalent awaitable accepted-block delivery, and this issue should remain open until that PR lands or is rejected.
