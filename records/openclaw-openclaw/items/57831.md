## Summary

Keep open. This PR targets a real cron/direct-message session isolation bug, but the latest head still has build and classifier defects and lacks required after-fix real behavior proof.

## What This Changes

The PR adds a cron session-target guard during normalization/update/runtime, threads session context through cron tool and gateway update paths, and adds regression tests for direct-session downgrade plus safe group reuse.

## Best Possible Solution

Use a shared, well-tested cron session-key classifier at persistence and runtime that blocks real direct-message sessions, preserves safe group/channel and custom named sessions, and includes targeted tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main persists caller session keys for current targets and runtime reuses explicit session:<id> targets. The related issue at https://github.com/openclaw/openclaw/issues/69467 also provides concrete live-DM logs.

## Solution Assessment

No, not as written. The direction is right, but the patch must compile and classify only real direct chat session keys across canonical, account-qualified, and legacy forms without breaking documented custom named sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Import the chat-type helper from its exporting module:** `src/cron/session-target-guard.ts:1`
  - body: `deriveSessionChatType` is not exported by `../sessions/session-key-utils.js` on current main; it is exported from `../sessions/session-chat-type.js`, so the new file will fail build/typecheck as soon as it is compiled.
  - confidence: 0.93
- **[P1] Handle legacy direct keys instead of marking them safe:** `src/cron/session-target-guard.ts:14-16`
  - body: When `parseAgentSessionKey` fails, the guard returns safe before checking legacy or unscoped direct keys such as `telegram:dm:123`; those keys can still be persisted or reused by cron, leaving part of the DM isolation bypass in place.
  - confidence: 0.87
- **[P1] Classify direct markers before broad group tokens:** `src/cron/session-target-guard.ts:17`
  - body: The guard delegates to a helper that returns `group` or `channel` before checking `direct`/`dm`, so account-qualified direct keys containing those broad tokens can still be treated as reusable and bind cron runs to a live DM session.
  - confidence: 0.82
- **[P2] Preserve arbitrary custom session IDs:** `src/cron/session-target-guard.ts:8-9`
  - body: Treating every parseable `agent:<id>:...` value as an agent-scoped chat key downgrades custom targets such as `session:agent:billing:direct:daily`, even though `session:<custom-id>` is documented as a persistent named session.
  - confidence: 0.84
