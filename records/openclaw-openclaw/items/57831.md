  - body: `deriveSessionChatType` returns `group` or `channel` before `direct`, so a direct key with an account segment such as `group` can be treated as safe and still reused. The unsafe predicate needs a canonical direct-session parser rather than the broad display classifier.
  - confidence: 0.83
- **[P1] Cover legacy direct session keys before returning safe:** `src/cron/session-target-guard.ts:14-16`
  - body: The guard returns safe as soon as `parseAgentSessionKey` fails, but legacy keys such as `telegram:dm:123456` are still classified as direct elsewhere. Those persisted or current targets would bypass the isolation boundary this PR is trying to add.
  - confidence: 0.8
