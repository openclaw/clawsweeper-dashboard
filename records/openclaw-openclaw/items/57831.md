## Summary

Keep this PR open: it targets a real cron/session isolation bug still present on current main, but the branch has concrete build and session-key classifier blockers plus missing after-fix real behavior proof.

## What This Changes

This PR adds a cron session-target guard at normalization/update/runtime, threads session context through cron tool and gateway update paths, and adds cron/tool/gateway regression tests.

## Best Possible Solution

Use a shared, well-tested cron session-key classifier at persistence and runtime that blocks canonical and legacy direct DM sessions, preserves safe group/channel and custom named sessions, and includes targeted tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows current targets still persist the caller session and runtime still trusts session:<id>; the related open issue provides concrete live-DM logs for the user-visible context loss.

## Solution Assessment

No, not as written. The direction is right, but the implementation should compile and classify only real direct chat session keys across canonical, account-qualified, and legacy shapes without breaking arbitrary custom session IDs.

  - body: `deriveSessionChatType` is not exported by `../sessions/session-key-utils.js` on current main; it is exported from `../sessions/session-chat-type.js`. This new file will fail build/typecheck as soon as it is compiled.

  - body: Returning safe whenever `parseAgentSessionKey` fails lets unscoped direct keys like `telegram:dm:123` or `telegram:direct:123` continue through as reusable cron sessions, leaving the DM isolation bypass in place for supported legacy formats.
  - confidence: 0.87
- **[P1] Classify direct markers before broad group tokens:** `src/cron/session-target-guard.ts:17`
  - body: The guard delegates to a helper that returns `group` or `channel` before checking `direct`/`dm`, so account-qualified direct keys containing those tokens can still be treated as reusable and bind cron runs to a live DM session.
  - confidence: 0.82
- **[P2] Preserve arbitrary custom session IDs:** `src/cron/session-target-guard.ts:8-9`
  - body: Treating every parseable `agent:<id>:...` value as an agent-scoped chat key downgrades custom targets such as `session:agent:billing:direct:daily`, even though `session:<custom-id>` is documented as a persistent named session.
  - confidence: 0.84
