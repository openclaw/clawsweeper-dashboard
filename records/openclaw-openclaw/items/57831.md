## Summary

Keep open. Current main still has the unsafe cron/session path, but this PR is not mergeable as written because the new guard imports a non-exported helper, misses some direct-session key shapes, can downgrade documented custom sessions, and lacks contributor real behavior proof.

## What This Changes

The PR adds cron session-target guarding during normalization/runtime, passes session context through cron update paths, and adds regression tests for direct-session isolation.

## Best Possible Solution

Use a shared, well-tested cron session-key classifier at persistence and runtime that blocks real direct-message sessions, preserves group/channel and documented custom named sessions, and pairs with a non-cron deleteAfterRun ownership guard.

## Reproduction Assessment

Yes. Current main source plus the related live-DM report give a high-confidence path: current can persist a live direct session key, runtime reuses session:* targets, and deleteAfterRun cleanup can delete that same key.

## Solution Assessment

No, not as written. The layered guard is the right direction, but the patch must compile, classify only real direct chat session keys, preserve documented custom sessions, and include after-fix real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Import the chat-type helper from its exporter:** `src/cron/session-target-guard.ts:1`
  - body: `deriveSessionChatType` is not exported by `../sessions/session-key-utils.js` on current main; the added file imports it there, so the patch will fail build/typecheck when compiled.
  - confidence: 0.93
- **[P1] Check legacy direct keys before returning safe:** `src/cron/session-target-guard.ts:14-16`
  - body: When `parseAgentSessionKey` fails, the guard returns safe before consulting the chat-type classifier. Legacy or unscoped direct keys that the rest of the session code can classify as direct would still be persisted or reused by cron.
  - confidence: 0.86
- **[P1] Prefer direct markers before broad group tokens:** `src/cron/session-target-guard.ts:17`
  - body: The guard delegates to a helper that returns `group` or `channel` before checking `direct`/`dm`, so account-qualified direct keys containing a broad token before the direct marker can still be treated as reusable.
  - confidence: 0.82
- **[P2] Preserve arbitrary custom session IDs:** `src/cron/session-target-guard.ts:8-9`
  - body: `parseAgentSessionKey` accepts any `agent:<id>:...` value, so custom targets such as `session:agent:billing:direct:daily` can be downgraded even though `session:<custom-id>` is documented as a persistent named session.
  - confidence: 0.84

## Security Review
