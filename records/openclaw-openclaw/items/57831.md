- **[P1] Import the chat-type helper from its exporter:** `src/cron/session-target-guard.ts:1`
  - body: `session-key-utils.js` does not export `deriveSessionChatType` on current main; that helper is exported from the session chat-type module, so this new file will fail typecheck until the import is fixed during rebase.
  - confidence: 0.96
- **[P1] Classify legacy direct keys before returning safe:** `src/cron/session-target-guard.ts:14-16`
  - body: The guard returns `false` whenever `parseAgentSessionKey()` fails, but unscoped legacy keys such as `telegram:dm:123456` are still recognized as direct by the shared chat-type classifier. Those persisted targets would keep reusing the live DM session instead of falling back to isolated.
  - confidence: 0.88
- **[P1] Do not let group tokens mask direct sessions:** `src/cron/session-target-guard.ts:17`
  - body: After the import is fixed, this uses a classifier that returns `group` or `channel` before checking `direct`/`dm`. A direct key with an account or peer segment containing one of those tokens can be treated as safe and bypass the intended isolation guard.
  - confidence: 0.86
- **[P1] Keep custom session IDs reusable unless they are real DMs:** `src/cron/session-target-guard.ts:9-17`
  - body: `parseAgentSessionKey()` accepts any `agent:<id>:...` shape, so a documented custom session like `session:agent:billing:direct:daily` can be downgraded to `isolated` even though it is not a channel DM. That silently breaks persistent named-session cron jobs.
  - confidence: 0.88
