## Summary

Keep open: the linked TUI/session-state bug is still present on current main and v2026.5.20, and this PR is the right implementation direction, but it is not merge-ready while the branch is conflicting, lacks real multi-client proof, and does not yet account for current-main pending-run state.

## What This Changes

This PR adds a TUI Gateway `sessions.subscribe` call, dispatches `sessions.changed` reset/new events to a reset handler that reloads history and clears run/stream state, and adds focused handler tests plus a changelog entry.

## Best Possible Solution

Land or replace this with a current-main TUI consumer for `sessions.subscribe` and `sessions.changed` reset/new events that refreshes the active session and clears active, pending, local, watchdog, and suppressed run state.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, by source inspection: current main and v2026.5.20 can emit subscription-gated `sessions.changed` reset/new events, while the TUI neither subscribes to nor handles that event family.

## Solution Assessment

Is this the best way to solve the issue? Yes for the direction: reusing `sessions.subscribe` and `sessions.changed` is the narrow maintainable path, but the branch needs rebase, pending-run handling, and real proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clear pending chat runs on external reset:** `src/tui/tui-event-handlers.ts:566-568`
  - body: Current main tracks `state.pendingChatRunId` after `chat.send` returns and before the first chat event arrives. This reset handler suppresses active/session/finalized runs, then clears `activeChatRunId`, but it never suppresses or nulls the pending run id, so a reset in that gap can still let the old run become active when its first event arrives and repaint the reset session. Add `pendingChatRunId` to the suppressed set, clear it on reset, and cover that pre-event gap in the handler tests.
  - confidence: 0.87
