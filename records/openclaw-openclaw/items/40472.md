## Best Possible Solution

## Reproduction Assessment

Yes by source inspection. Current main emits `sessions.changed` from `sessions.reset` to subscribed clients, while the TUI neither subscribes to nor dispatches that event family, so resetting the active TUI session from another client can leave the TUI stale.

## Solution Assessment

No, not yet. Reusing the existing `sessions.subscribe` / `sessions.changed` contract is the narrow maintainable direction, but this branch needs current-main pending-run cleanup, conflict resolution, and real behavior proof before merge.
