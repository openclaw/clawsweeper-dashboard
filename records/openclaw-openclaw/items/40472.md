## What This Changes

The branch adds TUI `sessions.subscribe` support, dispatches `sessions.changed` reset/new events, clears stale run/stream state on active-session resets, and adds focused handler tests plus a changelog entry.

## Best Possible Solution

Land a rebased TUI consumer for the existing `sessions.subscribe` and `sessions.changed` reset/new contract that clears active, pending, local, watchdog, session, and finalized run state, backed by redacted multi-client proof.

## Reproduction Assessment

Yes, by source inspection. Current main emits subscription-gated `sessions.changed` reset events, while the TUI neither subscribes to nor dispatches that event family, so an external reset can leave stale TUI state.

## Solution Assessment

Yes for the direction, but not yet for this exact patch. Reusing `sessions.subscribe` and `sessions.changed` is the narrow maintainable fix, but the patch must also handle `pendingChatRunId`, rebase cleanly, and include real behavior proof.
