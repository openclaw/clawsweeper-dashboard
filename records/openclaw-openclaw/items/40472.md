## Summary

Keep open: the bug is valid and this PR uses the right session lifecycle contract, but the branch still misses current-main pending-run reset suppression and lacks real multi-client behavior proof before merge.

## What This Changes

The PR adds a TUI backend subscription to `sessions.changed`, dispatches reset/new session lifecycle events to a new TUI handler that clears run state and reloads history, and adds focused handler tests plus a changelog entry.

## Best Possible Solution

Rebase onto current main, keep the `sessions.subscribe`/`sessions.changed` TUI consumer, suppress and clear `pendingChatRunId` during external reset, narrow the changelog overclaim, and attach redacted real TUI plus another-client reset proof.

## Reproduction Assessment

Yes by source inspection. Current main emits `sessions.changed` on `sessions.reset` only to subscribed clients, while the TUI neither subscribes to nor handles that event family.

## Solution Assessment

No, not yet. The `sessions.changed` consumer is the narrow maintainable fix, but the branch needs current-main pending-run cleanup and real behavior proof before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Suppress pending chat runs on reset:** `src/tui/tui-event-handlers.ts:201-213`
  - body: Current main records the run id returned by `chat.send` in `state.pendingChatRunId` before the first chat event arrives. If an external reset lands in that gap, this helper does not add that pending id to `resetSuppressedRuns` or clear it, so a late first delta/final for the old run can still repaint the reset session after rebase.
  - confidence: 0.86
- **[P3] Narrow the changelog reset guarantee:** `CHANGELOG.md:16`
  - body: The changelog says late old-run events cannot repaint reset history, but the patch only suppresses run ids it already knows at reset time and currently misses the pending chat id from main. Please narrow this to the active-session refresh and tracked-run cleanup the patch actually guarantees, or update the code first.
  - confidence: 0.82
