## What This Changes

Report requests that ACP `sessions_spawn({ runtime: "acp", agentId: "claude" })` continue to work after `openclaw gateway restart` instead of failing with `ACP_TURN_FAILED`.

## Best Possible Solution

Keep the current ACP manager/acpx recovery behavior and let the next release carry it to affected packaged installs.

## Reproduction Assessment

Yes. The report has concrete steps, and current-main tests model the relevant restart/stale ACP session cases, including manager restart rehydration and stale persistent `claude` session retry; I did not run a live smoke in this read-only review.

## Solution Assessment

Yes. The current implementation is a narrow runtime recovery fix: rehydrate handles after restart, clear stale resume state, prepare a fresh acpx session, and retry once without adding a new API or config knob.
