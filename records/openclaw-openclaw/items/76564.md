Current main still shows the same option-replay path, so this report is useful confirming WSL2/restart evidence but should not be tracked separately.

## What This Changes

The issue reports that ACP `sessions_spawn({ runtime: "acp", agentId: "claude" })` fails with `ACP_TURN_FAILED` after gateway restart while direct `acpx claude exec` still works.

## Best Possible Solution

Keep #73875 as the canonical bug and review or replace #74048 with a narrow fix that skips unadvertised ACP config keys during generic runtime-control replay.

## Reproduction Assessment

Yes at source level, but not by live execution. Current main deterministically replays persisted generic ACP config options after re-ensuring a runtime handle, and missing `configOptionKeys` does not stop forwarding those options.

## Solution Assessment

Yes. The best path is the canonical #73875/#74048 fix direction: filter or skip `session/set_config_option` calls based on backend-advertised keys, with ACP manager and ACPX regression coverage.
