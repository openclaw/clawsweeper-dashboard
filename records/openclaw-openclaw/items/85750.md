Keep open: the report describes a concrete authenticated Control UI avatar failure, and current main plus an open related PR show this protected avatar route still needs careful review rather than cleanup closure.

## What This Changes

The issue reports that authenticated Control UI webchat sessions over Tailscale can still fetch `/avatar/<agentId>` without the gateway bearer token, causing a 401 and a missing agent avatar.

## Best Possible Solution

Unify all Control UI avatar rendering around authenticated blob URLs or safe fallbacks, then verify the Tailscale webchat path and decide whether the related open PR should be broadened or landed as the canonical fix.

## Reproduction Assessment

Partially: source and tests show `/avatar/main` is auth-gated and that the intended webchat refresh path sends bearer auth, while current main still has a related direct protected local avatar route path. I did not establish a live Tailscale webchat reproduction.

## Solution Assessment

Unclear: the requested direction is correct, but the source evidence suggests the remaining fix may be to eliminate direct protected avatar rendering on every Control UI surface rather than only changing the webchat fetch helper.
