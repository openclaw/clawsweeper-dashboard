## What This Changes

The issue reports that authenticated Control UI webchat sessions over Tailscale request `/avatar/main` without bearer auth and get 401 instead of rendering the configured avatar.

## Best Possible Solution

Keep the avatar route bearer-only unless maintainers explicitly choose a short-lived scoped ticket design, unify UI avatar rendering around authenticated blob URLs or safe fallbacks, and verify the Tailscale path with redacted DevTools or equivalent live proof.

## Reproduction Assessment

## Solution Assessment

Unclear for the linked query-token PR: the maintainable direction is the documented bearer-authenticated fetch/blob path, while reusable token-in-URL auth would need explicit security approval or replacement with a scoped ticket.
