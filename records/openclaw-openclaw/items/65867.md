## Summary

Keep open: current main still contains the duplicated narrow final-tag matchers that miss self-closing and attributed `<final>` variants, and open PRs are already attached as implementation candidates rather than proof of a merged fix.

## What This Changes

The issue reports Gemini/Gemma-generated `<final>` and related reasoning tags leaking into WebUI/Control UI chat responses after upgrading to OpenClaw 2026.4.11/2026.4.12.

## Best Possible Solution

Land or replace the open fix work with one constrained shared final/reasoning-tag parser across streaming, user-facing sanitizer, persisted/replay, and Control UI render paths, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main's sanitizer and streaming regexes leave `<final/>Hello` and attributed final tags visible, matching the reported Gemini/Gemma leak class; no live WebUI run was performed.

## Solution Assessment

Yes for the triage outcome: keeping this issue open is the narrowest maintainable path while the linked fix PRs are reviewed, reconciled, landed, or replaced. Closing now would be premature because current main still has the source-level gap.
