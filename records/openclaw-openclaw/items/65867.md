## Summary

Keep open: origin/main still has the narrow final-tag matchers that miss self-closing and attributed Gemini/Gemma variants, and an open closing PR plus an overlapping open PR are still awaiting reconciliation.

## What This Changes

Users report Gemini/Gemma `<final>` and `<final/>` wrapper tags leaking into Control UI/webchat and delivered messages after upgrading.

## Best Possible Solution

Land one reconciled fix that aligns the duplicated sanitizer and streaming matchers with the shared broad reasoning-tag matcher, with regression coverage for bare, self-closing, attributed, split-stream, and replay/persistence cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by applying the current sanitizer/subscriber regexes to `<final/>Hello` and `<final id="1">Hello</final>`; user comments add real Control UI evidence for the same leak class.

## Solution Assessment

Yes. The narrow maintainable solution is to share or align the existing broad reasoning-tag matcher across the sanitizer and streaming paths; prompt-only mitigation is insufficient because OpenClaw itself can instruct models to emit these tags.
