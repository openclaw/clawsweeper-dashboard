## Summary

Keep open: current main still has the narrow final-tag matchers that miss self-closing and attributed Gemini/Gemma variants, and the provided context shows an open closing PR plus an overlapping PR still awaiting maintainer reconciliation.

## What This Changes

The issue requests stripping Gemini/Gemma `<final>`, `<final/>`, and related reasoning wrapper tags from WebUI/Control UI delivered messages so users only see the final answer text.

## Best Possible Solution

Land one reconciled fix that reuses or aligns the shared broad final-tag matcher across sanitizer, streaming, message-end/replay paths, with regression coverage for bare, self-closing, attributed, split-stream, and code-span cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by applying the current sanitizer/subscriber regexes to `<final/>Hello` and `<final id="1">Hello</final>`; the reported Control UI comments add live-user evidence for the same class of leak.

## Solution Assessment

Yes. The narrow maintainable direction is to share or align the existing broad reasoning-tag matcher instead of adding another ad hoc sanitizer; prompt-only mitigation is not sufficient because OpenClaw itself can instruct models to emit these tags.
