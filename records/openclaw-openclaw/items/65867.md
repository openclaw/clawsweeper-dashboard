## Summary

Keep open: current main and the latest shipped release still have narrower final-tag stripping in the sanitizer and embedded streaming subscriber, while linked open PRs target this exact gap. Because #66225 uses closing syntax for this issue and is still open in the provided context, closure before merge would be premature.

## What This Changes

The issue reports Gemini/Gemma raw `<final>`, `<final/>`, and related reasoning tags appearing in Control UI or delivered assistant messages after upgrading to OpenClaw 2026.4.11.

## Best Possible Solution

Keep this issue open until maintainers review and reconcile #66225 and #48593, then land one fix that reuses or aligns the shared broad final-tag matcher across sanitizer, streaming, message-end/history, and regression tests before closing after merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by applying the current sanitizer/subscriber regexes to `<final/>Hello` or `<final id="1">Hello</final>`; those variants remain visible while the shared reasoning-tag matcher strips them.

## Solution Assessment

Yes. The narrow maintainable fix is to share or align the final-tag matching behavior instead of adding another ad hoc cleanup path, with tests for bare, self-closing, attributed, split-stream, and code-span cases.
