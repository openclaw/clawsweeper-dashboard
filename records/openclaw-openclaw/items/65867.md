## Summary

Keep open: current main still has source-level gaps for the reported tag leak class, and several open PRs now cover overlapping agent and Control UI paths that should be reconciled instead of closing this issue early.

## What This Changes

The issue asks OpenClaw to prevent Gemini/Gemma `<final>` and related reasoning wrapper tags from appearing in Web UI or delivered assistant messages.

## Best Possible Solution

Land one reconciled fix that shares or aligns final/reasoning tag stripping across agent streaming, user-facing sanitizer, persisted/replay, and Control UI grouped-render paths, then close or supersede the partial overlapping PRs.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: current main's sanitizer and streaming regexes leave `<final/>Hello` and `<final id="1">Hello</final>` behind, and the Control UI grouped-render path can render normalized assistant text without the existing internal-scaffolding sanitizer.

## Solution Assessment

Yes, but it should be one reconciled fix rather than separate partial patches. The narrow maintainable solution is to reuse the shared broad reasoning-tag sanitizer/matcher in the duplicated agent paths and ensure final/history Control UI rendering also passes through the same user-visible sanitizer.
