## Summary

Current main still only represents a quoted WhatsApp image as reply-context text (`<media:image>`) and does not download or attach the quoted media for the agent. Prior WhatsApp reply-target media PRs described the intended fix, but they are closed and unmerged, so this remains actionable.

## What This Changes

Allow OpenClaw to see an image that a WhatsApp user references by replying to the image and mentioning OpenClaw, without requiring the user to resend the image.

## Best Possible Solution

Implement this inside the WhatsApp plugin inbound pipeline: resolve downloadable media from `contextInfo.quotedMessage` for image/sticker reply targets, save it through the existing inbound media store, and surface it as normal inbound `MediaPath`/`MediaType` so media understanding and prompt media notes can see it. Keep the existing quoted-body metadata and mention-gating behavior intact.

## Reproduction Assessment

Yes. A source-level reproduction is clear: emit a WhatsApp group `extendedTextMessage` with a direct bot mention and `contextInfo.quotedMessage.imageMessage`; current main will set `replyToBody` to `<media:image>` but leave `mediaPath` undefined because only the top-level message is downloaded.

## Solution Assessment

No current solution is present on main. The narrow maintainable fix is WhatsApp-owned quoted-media extraction and saving, not a core auto-reply change and not asking users to resend images.
