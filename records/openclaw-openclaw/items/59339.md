## Summary

Keep open. Current main and v2026.5.12 still hardcode the fire-and-forget `sessions_send` announce budget to 30s and retain 60s caps in delayed A2A and nested agent-step waits; the merged delayed-reply fix in https://github.com/openclaw/openclaw/pull/76484 only addressed soft wait-window preservation.

## What This Changes

The issue requests a configurable or consistently bounded `sessions_send` A2A announce timeout because the hardcoded fire-and-forget window can drop delayed replies.

## Best Possible Solution

Define one bounded `sessions_send` A2A timeout policy that covers fire-and-forget, initial waits, delayed run waits, reply-back turns, and final announce without unbounded model-controlled waits.

## Reproduction Assessment

Yes, source-reproducible. `sessions_send(timeoutSeconds=0)` still starts background A2A delivery with a 30,000 ms announce budget, and delayed/nested waits still cap at 60,000 ms, so replies completed after those windows can be dropped.

## Solution Assessment

No current implementation is only a partial fix. The remaining work needs a bounded policy rather than raw cap removal or another isolated hardcoded default change.
