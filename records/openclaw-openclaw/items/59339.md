## Summary

Keep open: current main and v2026.6.10 still use a 30s `sessions_send(timeoutSeconds=0)` A2A announce budget, and the remaining decision is the public timeout/default/config contract rather than conservative cleanup.

## What This Changes

Issue requests configurable or longer A2A announce timeout behavior for `sessions_send(timeoutSeconds=0)` to avoid dropped async replies.

## Best Possible Solution

Define one canonical `sessions_send` A2A timeout contract, then update the tool behavior, bounded wait policy, focused tests, and public docs/schema together.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection on current main shows `timeoutSeconds=0` still creates a 30s A2A announce budget and can return before a delayed target reply is available.

## Solution Assessment

Unclear. A raw timeout knob may be useful, but maintainers should first choose whether the right contract is a longer default, reuse of the subagent announce timeout, an A2A-specific option, or a push-result pattern for fire-and-forget sends.
