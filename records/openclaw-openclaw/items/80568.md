## Summary

Keep open: the patch appears to fix the LM Studio SecretInput/env-template auth bug and the earlier header-auth regression concern, but this external PR still needs real after-fix LM Studio behavior proof before merge.

## What This Changes

The PR routes LM Studio string apiKey values through SecretInput resolution, preserves Authorization-header-only auth when an API-key template is unset, and adds runtime/setup regressions plus a changelog entry.

## Best Possible Solution

Land the narrow auth-resolution fix after redacted real LM Studio runtime proof is added, while leaving native endpoint fallback/API-mode work to https://github.com/openclaw/openclaw/issues/80495.

## Reproduction Assessment

Yes, from source: current main returns arbitrary LM Studio `${ENV}` string apiKeys as literals unless they match a known env marker, while the shared SecretInput contract treats `${ENV}` as an env SecretRef. I did not establish a live LM Studio server reproduction in this read-only review.

## Solution Assessment

Yes for the auth-resolution scope: using the existing SecretInput resolver is the narrow maintainable fix, and the follow-up `allowUnresolved` path preserves established Authorization-header auth behavior. The PR should still wait for real runtime proof before merge.
