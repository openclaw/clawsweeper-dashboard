## Summary

Keep open: the patch is a narrow, code-backed fix for the LM Studio env-template auth bug, but it is an external contributor PR and still lacks after-fix real LM Studio runtime proof before merge.

## What This Changes

The PR updates LM Studio runtime and setup auth resolution so configured env-template API keys use SecretInput resolution and unresolved optional templates do not override configured Authorization header auth.

## Best Possible Solution

Merge the auth-resolution fix after the contributor adds redacted real LM Studio runtime proof, while leaving endpoint fallback/native API behavior tracked on https://github.com/openclaw/openclaw/issues/80495.

## Reproduction Assessment

Yes, by source inspection: current main treats an unknown ${LMSTUDIO_API_KEY} template as a literal key because only registered env markers are resolved in the direct-string branch. I did not run a live LM Studio server or tests in this read-only review.

## Solution Assessment

Yes for the auth portion: reusing the existing SecretInput resolver before legacy marker fallback is the narrow maintainable fix, and the Authorization-header guard preserves existing header-only auth behavior. The native endpoint fallback request should remain separate.
