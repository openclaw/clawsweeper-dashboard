## Summary

Keep this PR open: the code change appears to be the right narrow fix for the LM Studio env-template API-key bug, and the earlier header-auth review concern is addressed, but the external PR still lacks real after-fix LM Studio runtime proof.

## What This Changes

The PR routes LM Studio configured `apiKey` strings through SecretInput resolution, preserves Authorization-header-only auth when an env template is unset, and adds runtime/setup regression tests plus a changelog entry.

## Best Possible Solution

Land the narrow auth-resolution patch after contributor-provided redacted real LM Studio runtime proof is added, while leaving native endpoint fallback/API-mode work to https://github.com/openclaw/openclaw/issues/80495.

## Reproduction Assessment

Yes, from source: current main returns arbitrary LM Studio `${ENV}` string `apiKey` values literally unless they match a known marker, while the SecretInput contract treats `${ENV}` as an env SecretRef. I did not run a live LM Studio server in this read-only review.

## Solution Assessment

Yes for the auth-resolution scope: using the existing SecretInput resolver with an `allowUnresolved` escape for Authorization-header auth is the narrow maintainable fix. The separate native endpoint fallback request remains tracked in https://github.com/openclaw/openclaw/issues/80495.
