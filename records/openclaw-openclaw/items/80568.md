## Summary

Keep open: the patch appears to address the PR-scoped LM Studio env-template auth bug and the earlier header-auth regression, but the external real-behavior proof gate is still unmet and the linked native endpoint fallback work remains separate.

## What This Changes

The PR updates LM Studio runtime/setup auth resolution and tests so configured `${ENV}` API-key strings use SecretInput while Authorization-header auth can satisfy auth when an optional API-key template is unset.

## Best Possible Solution

Merge the auth-resolution fix after the contributor adds redacted real LM Studio runtime proof, while keeping native endpoint fallback work tracked separately.

## Reproduction Assessment

Yes for the PR-scoped auth bug by source inspection: current main normalizes `${LMSTUDIO_API_KEY}` to `LMSTUDIO_API_KEY` and returns it as a bearer token because only `LM_API_TOKEN` is a known marker. I did not establish a live LM Studio 401 in this read-only pass.

## Solution Assessment

Yes for the auth-resolution scope: routing configured strings through the existing SecretInput helper before legacy marker fallback is the narrow maintainable fix, and `allowUnresolved` preserves existing Authorization-header auth. The native endpoint fallback request is separate and should not be bundled into this PR.
