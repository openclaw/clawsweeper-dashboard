## What This Changes

The PR resolves LM Studio configured env-template API keys through the shared SecretInput path and preserves header-only Authorization auth when the optional apiKey template is unset.

## Best Possible Solution

Land the narrow SecretInput/env-template auth fix after real LM Studio proof and reconciled CI, while keeping native endpoint fallback as separate provider/API scope.

## Solution Assessment

Yes for the narrow auth bug: reusing the shared SecretInput resolver in the LM Studio provider is the maintainable fix, and the branch now preserves header-only auth. It is not a complete solution for bare `$VAR` compatibility or native `/api/v1/chat` fallback.
