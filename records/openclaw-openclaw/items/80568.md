## What This Changes

The PR resolves LM Studio configured string apiKey values through SecretInput, preserves Authorization-header-only auth when an apiKey template is unset, adds runtime/setup regressions, and updates the changelog.

## Best Possible Solution

Land the narrow auth-resolution fix after the contributor adds redacted real LM Studio runtime proof, while keeping native endpoint fallback/API-mode work with https://github.com/openclaw/openclaw/issues/80495.

## Reproduction Assessment

## Solution Assessment

Yes for the auth-resolution scope: using the existing SecretInput resolver is the narrow maintainable fix, and `allowUnresolved` preserves established Authorization-header auth behavior. The PR should still wait for real runtime proof before merge.
