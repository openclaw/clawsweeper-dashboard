## Summary

Keep open: current main has primary/fallback model selection plus several capability-specific model configs, but no generic per-tool routing surface for normal model-emitted tools. This is a new config and runtime design decision, not an already-implemented cleanup item.

## What This Changes

Add config-driven per-tool model routing so calls such as `edit`, `write`, and `exec` can use a specialized model while the session continues on the primary model.

## Best Possible Solution

Design the core model/tool boundary first, then add a narrow routing API only if it can preserve transcript continuity, fallback behavior, and existing model/session selection rules.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report with a failing current-main path. Source and docs inspection show the requested config and runtime behavior are not present today.

## Solution Assessment

Unclear: the requested `toolRouting` shape is plausible, but current OpenClaw selects a model for the model turn and then executes model-emitted tools, so maintainers should settle the routing boundary before code work.
