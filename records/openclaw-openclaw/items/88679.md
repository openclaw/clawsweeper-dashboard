## Summary

Keep open: current main still has primary/fallback model selection plus capability-specific model configs, but it does not have a generic per-tool model-routing surface for ordinary model-emitted edit/write/exec calls. The existing plugin hooks provide adjacent per-turn model override and before-tool parameter rewrite seams, so this needs a maintainer product/API boundary decision rather than conservative cleanup closure.

## What This Changes

Request a new config/runtime feature that routes specified agent tool calls such as edit, write, and exec to designated provider/model fallback chains while preserving the primary session model for reasoning.

## Best Possible Solution

Decide the plugin-first API boundary first; if current hooks are insufficient, add the smallest typed plugin/runtime extension before considering a broad public `toolRouting` config.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report with a current-main failing path. Source and docs inspection show the requested config/runtime behavior is absent today.

## Solution Assessment

Unclear: the requested `toolRouting` shape is plausible, but current OpenClaw selects the model before tool calls are emitted and tool hooks only rewrite or block calls after that point. Maintainers should settle the routing layer before code work.
