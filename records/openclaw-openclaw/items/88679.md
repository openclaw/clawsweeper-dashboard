## Summary

Keep open: current main and the latest release still lack generic tool-name-based model routing for ordinary edit/write/exec tool calls, and the related ClawHub plugin path covers per-turn/task routing rather than proving this exact tool-call boundary obsolete.

## What This Changes

Add per-tool model routing so selected tool calls can use a different provider/model than the session primary while preserving normal tool results and fallback behavior.

## Best Possible Solution

Keep this open for maintainer API/product review, favoring a plugin-owned design or the smallest generic hook/SDK extension before adding broad core config.

## Reproduction Assessment

Not applicable: this is a feature request rather than a bug. Source and docs checks on current main and `v2026.6.6` show the requested tool-name routing config/runtime path is absent.

## Solution Assessment

Unclear: the requested core `toolRouting` shape is plausible, but it may be the wrong layer. Existing hooks support per-turn model override and tool-call policy separately, so maintainers should decide whether a narrow plugin API extension is needed.
