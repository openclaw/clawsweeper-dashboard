## Summary

Keep open. The PR targets the right workspace-seeding bug and has useful real filesystem proof, but the new ancestor-marker guard is too broad and would also bypass existing setup repair for legitimate nested secondary-agent workspaces that already have their own workspace state.

## What This Changes

The PR changes agent workspace seeding so non-empty directories nested under an ancestor workspace marker do not receive bootstrap files, while empty nested secondary-agent workspaces still do.

## Best Possible Solution

Scope the skip to inherited project directories that lack their own OpenClaw workspace marker, so nested worktrees avoid bootstrap litter while real nested agent workspaces keep normal setup repair.

## Reproduction Assessment

Yes at source level: current main writes bootstrap files for the selected directory and does not inspect an ancestor workspace marker. The introduced PR regression is also source-reproducible from the new guard plus the existing `resolveAgentWorkspaceDir` nesting behavior.

## Solution Assessment

No as written: the fix is in the right function, but the predicate is too broad for legitimate nested secondary-agent workspaces. The safer solution is to skip only inherited nested content that lacks its own workspace marker.
