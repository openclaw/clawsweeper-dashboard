- `P2`: The PR addresses a user-visible plugin binding control bug that can trap conversation state, but the blast radius is limited to plugin-owned binding command routing.

## Summary

## What This Changes

The branch adds a command-escape predicate in `dispatchReplyFromConfig` and regression tests for `/codex detach` bypassing plugin-owned binding dispatch while unauthorized slash text stays bound.

## Best Possible Solution

Land a narrow dispatch escape after lint is fixed and a real bound Telegram or Discord plugin conversation proves control commands escape while normal and unauthorized text remains bound.

## Reproduction Assessment

Yes, source-level: current main lets a handled plugin-owned binding return before downstream command handling, and the linked report gives a concrete Telegram `/codex detach` failure on v2026.5.19. I did not execute a live repro in this read-only review.

## Solution Assessment

Yes, the centralized dispatch escape is the right boundary for this bug, but this implementation still needs the lint repair and real transport proof. Reviewers should also confirm the text-slash escape predicate is not broader than intended.
