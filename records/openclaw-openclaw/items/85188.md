## Summary

Keep open: this PR targets a real source-repro routing bug, but the new command-escape predicate is too broad and can misroute plugin-bound slash text away from the owning plugin.

## What This Changes

The branch adds a dispatch helper that lets authorized native and text slash command turns bypass plugin-owned conversation binding delivery, plus regression tests for authorized and unauthorized `/codex detach` routing.

## Best Possible Solution

Land a narrower escape that preserves plugin-owned binding delivery for unrecognized slash text while allowing recognized authorized native/control/plugin commands to detach or inspect the binding.

## Reproduction Assessment

Yes. From source inspection on current main, a plugin-owned binding can run `runInboundClaimForPluginOutcome` and return `handled` before the normal reply resolver can process the control command.

## Solution Assessment

No. The boundary is right, but the implementation should distinguish recognized authorized control/plugin commands from arbitrary slash-prefixed text before bypassing plugin-owned binding delivery.
