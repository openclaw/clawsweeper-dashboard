## What This Changes

The PR injects a short build-time git commit through tsdown and teaches the CLI commit resolver to prefer that injected value after env overrides but before live git and metadata fallbacks.

## Best Possible Solution

Refresh the branch against current main, keep the build-time commit source ahead of live git, add the scoped lint suppression and focused regression coverage, then provide real HEAD-shift terminal proof before merge.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes on fix direction, not yet on merge readiness. A tsdown build-time define is the right layer because it records the bundled code commit while preserving env overrides and dev-source live git fallback, but the branch needs lint, proof, and rebase/conflict cleanup.
