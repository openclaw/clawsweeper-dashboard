## Summary

Keep open: current main still resolves the visible version commit from live git before baked build metadata and has no injected build-time commit define, while this PR is a narrow fix with focused priority/cache coverage and no blocking code-review findings.

## What This Changes

This PR injects a build-time git SHA through tsdown, reads it in `resolveCommitHash` before live git, and adds resolver priority/cache tests.

## Best Possible Solution

Land this PR or an equivalent narrow fix after refreshed validation, preserving env override, injected build commit, live git, build-info, and package fallback order.

## Reproduction Assessment

Yes, from source inspection. Build `dist`, move HEAD without rebuilding, then run `node dist/index.js --version`; current main routes that output through `resolveCommitHash`, which still reads live git before build-info.

## Solution Assessment

Yes. A tsdown compile-time define read after explicit env overrides and before live git is the narrow maintainable fix, and it preserves dev-from-source plus packaged fallback behavior.
