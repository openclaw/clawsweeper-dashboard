## What This Changes

The PR adds a tsdown build-time commit define and resolves it before live git so CLI version output reports the commit the binary was built from.

## Best Possible Solution

Rebase the branch, add visible after-fix real behavior proof for the CLI HEAD-drift scenario, then land this canonical build-commit fix if refreshed checks pass.

## Reproduction Assessment

## Solution Assessment

Yes. A build-time define is the best narrow fix shape because it records the bundled code's commit while preserving explicit env overrides, dev/source live-git behavior, and build-info/package fallbacks.
