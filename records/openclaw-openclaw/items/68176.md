## What This Changes

This PR adds a tsdown build-time git commit define, reads it in `resolveCommitHash` before live git metadata, and adds resolver priority/cache tests.

## Best Possible Solution

Rebase against current main, keep the env > injected define > live git > build-info > package metadata order, and add redacted terminal or live-output proof for build, HEAD shift, stable `--version`, and env override behavior.

## Reproduction Assessment

Yes, by source inspection and the linked report: current main reads live `.git/HEAD` before `dist/build-info.json`, so a built checkout whose HEAD moves can report the wrong commit. I did not execute the full build/HEAD-shift flow in this read-only review.

## Solution Assessment

Yes, the build-time define is the narrowest maintainable fix because it records the build commit in the bundled code while preserving env overrides and live git for source/dev runs. Reordering build-info ahead of live git would break the existing dev-checkout invariant, and a sidecar-only marker would be a weaker runtime identity source.
