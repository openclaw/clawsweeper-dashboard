The PR adds a tsdown build-time `__OPENCLAW_GIT_COMMIT__` define, teaches `resolveCommitHash` to prefer it after explicit env overrides, and adds regression tests for injected-commit priority and caching.

## Best Possible Solution

Rebase the branch onto current main, add the targeted oxlint suppression for the new compile-time define, and require redacted after-fix terminal output showing a built binary keeps its original commit after HEAD moves.

## Solution Assessment
