## What This Changes

The branch adds a tsdown build-time git SHA define, reads it in `resolveCommitHash` before live git metadata, and adds unit tests for priority order and caching.

## Best Possible Solution

Rebase onto current main, keep the env override > injected define > live git > build-info > package metadata order, and attach redacted terminal proof of the built-binary HEAD-shift scenario before merge.

## Reproduction Assessment

Yes. Source inspection and the linked report show current main reads live `.git/HEAD` before `dist/build-info.json`, so a built checkout whose HEAD moves can report the wrong commit; I did not run the E2E build in this read-only review.

## Solution Assessment

Yes, after rebase and proof. A build-time define is the narrowest maintainable fix because it records identity in the bundled code while preserving env overrides and live git for dev runs; moving build-info ahead of live git would break the existing dev-checkout invariant.
