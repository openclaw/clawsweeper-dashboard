## Summary

## What This Changes

The PR adds a tsdown build-time commit define, reads it before live git in the CLI commit resolver, and covers the new priority/cache behavior with resolver tests.

## Best Possible Solution

Land this PR or an equivalent narrow fix after refreshed validation, preserving env override, injected build commit, live git fallback for source runs, and build-info/package fallbacks.

Yes. Build `dist` on current main, move HEAD without rebuilding, then run `node dist/index.js --version`; source inspection shows the resolver reads live git before build-info, so the commit suffix can drift.

## Solution Assessment

Yes. A tsdown compile-time define read after explicit env overrides and before live git is the narrow maintainable fix, and it keeps dev-from-source and packaged fallback behavior intact.
