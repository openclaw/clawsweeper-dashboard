Keep open. ## What This Changes

The PR adds a tsdown build-time git commit define, reads it in `resolveCommitHash` before live git, and adds resolver priority/cache tests for stable built-binary `--version` output.

## Best Possible Solution

Rebase this PR or a narrow replacement onto current main, keep the env override > injected define > live git > build-info > package gitHead priority, and add real terminal/log proof plus the focused resolver tests before merge.

## Reproduction Assessment

Yes. Source inspection shows current main and the latest release read live `.git/HEAD` before `dist/build-info.json`, so building `dist/`, moving HEAD without rebuilding, then running `node dist/index.js --version` is a high-confidence reproduction path; I did not run it because this review must keep the checkout read-only.

## Solution Assessment

Yes, this is the best fix shape: a compile-time build define records the bundled code's commit while preserving explicit env overrides, source/dev live-git behavior, and build-info/package fallbacks. The branch still needs proof and conflict refresh before it is merge-ready.
