## Summary

Keep open: current main still has the reported live-HEAD-before-build-info resolver behavior and no baked commit define, while this PR is a focused fix with matching tests and no blocking diff defect found.

## What This Changes

The PR injects a build-time git SHA through tsdown, makes `resolveCommitHash` prefer it after env overrides and before live git, and adds resolver priority/cache tests.

## Best Possible Solution

Land this PR or an equivalent narrow fix after fresh validation, preserving the env override, injected build commit, live git, build-info, then package fallback order.

## Reproduction Assessment

Yes. Build `dist` from current main, move HEAD without rebuilding, then rerun `node dist/index.js --version`; source inspection shows current main still calls live git before build-info and has no injected build commit define.

## Solution Assessment

Yes. A tsdown compile-time define read after explicit env overrides and before live git is the narrow maintainable fix because it stabilizes packaged binaries without removing live git metadata for dev-from-source runs.
