## Summary

Keep open: current main and v2026.6.8 still prefer live git metadata before build-info, and the linked issue was closed to this PR as the canonical fix path. The fix shape looks right, but the branch is not merge-ready because it lacks real behavior proof and introduces a lint blocker around the new underscored build define.

## What This Changes

The branch adds a tsdown build-time define for the short commit SHA, reads it before live git in `resolveCommitHash`, and adds regression tests for the new priority order.

## Best Possible Solution

Land a refreshed branch that preserves env override > injected build commit > live git > build-info/package fallback, fixes the lint suppression/allowlist issue, and includes real HEAD-shift proof.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.8 shows live `.git/HEAD` is read before `dist/build-info.json`, so building `dist/`, moving HEAD without rebuilding, then running `node dist/index.js --version` is a high-confidence reproduction path; I did not execute it because this review is read-only.

## Solution Assessment

Yes on fix shape, not yet on merge readiness. A build-time define is the right layer because it records the bundled code's commit while preserving explicit env overrides and source/dev live-git fallback, but the branch needs the lint fix and real behavior proof first.
