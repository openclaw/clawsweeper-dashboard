## What This Changes

This PR adds a tsdown build-time git SHA define, teaches `resolveCommitHash` to prefer it after env overrides, and adds unit coverage for injected commit priority and caching.

## Best Possible Solution

Rebase the branch onto current main, preserve the env > injected define > live git > build-info > package metadata order, and attach redacted terminal proof of the built-binary HEAD-shift scenario before merge.

## Reproduction Assessment

Yes, by source inspection and the linked report: current main reads live `.git/HEAD` before `dist/build-info.json`, so a built checkout whose HEAD moves can report the wrong commit. I did not run the E2E build in this read-only review.

## Solution Assessment

Yes, the build-time define is the narrowest maintainable fix because it records the commit in the bundled code while preserving env overrides and live git for source/dev runs. Reordering build-info ahead of live git would break the existing dev-checkout invariant, and a separate marker file would be a weaker runtime identity source.
