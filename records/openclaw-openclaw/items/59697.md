## Summary

Keep this PR open because current main still has the memory-prefix leak in the Gateway and Control UI user-message display paths, but the branch is not merge-ready due to a concrete sanitizer correctness issue and missing real behavior proof.

## What This Changes

The PR adds a shared sanitizer for memory-plugin `<relevant-memories>` prefixes in Gateway chat history and Control UI message extraction/normalization, plus regression tests and a changelog entry.

## Best Possible Solution

Land a prefix-specific shared sanitizer that removes only the injected memory block and generated separator, preserves user-authored tags and indentation, covers Gateway history plus Control UI rendering/normalization, and includes real after-fix proof.

## Reproduction Assessment

Yes, at source level: current main emits the exact memory prefix, prepends it to the user prompt, and the Gateway/Control UI user display paths do not strip it. I did not establish a live WebUI/Gateway reproduction in this read-only review.

## Solution Assessment

No, not as-is. The PR targets the right code paths, but the helper should strip only the injected block plus generated separator instead of consuming all following whitespace, and the external PR still needs real after-fix proof.
