## Summary

Keep this PR open: current main still leaks the memory-plugin `<relevant-memories>` prefix on user-role Gateway and Control UI display paths, but the PR is not merge-ready because it can remove user-authored indentation and lacks real after-fix proof.

## What This Changes

The PR adds a shared memory-prefix stripper to Gateway chat sanitization and Control UI extraction/normalization, plus regression tests and a changelog entry.

## Best Possible Solution

Land a prefix-specific sanitizer that strips only the injected memory block plus generated separator, preserves user-authored tags and indentation, covers Gateway history and Control UI rendering/normalization, and includes real after-fix proof.

## Reproduction Assessment

Yes, at source level: current main emits the exact memory prefix, prepends it to the user prompt, and the Gateway/Control UI user display paths do not strip it. I did not establish a live WebUI/Gateway reproduction in this read-only review.

## Solution Assessment

No, not as-is. The PR targets the right paths, but the helper should strip only the injected block plus generated separator instead of consuming all following whitespace, and the external PR still needs real after-fix proof.
