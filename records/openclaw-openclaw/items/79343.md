## Summary

Keep open: this is a current regression report against the latest release, and source inspection shows a plausible failure path where Discord can fall through to the catalog-only `installed, not configured, disabled` row even though Discord’s own account inspectors still recognize `channels.discord.token`.

## What This Changes

The issue asks for upgraded Discord channel config under `channels.discord` to be recognized as configured and enabled instead of being reported as `installed, not configured, disabled`.

## Best Possible Solution

Preserve the official external-plugin boundary, but make upgraded `channels.discord` configs either load the installed Discord setup/runtime plugin correctly or produce direct doctor/install guidance instead of a misleading unconfigured catalog row.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review because `pnpm` is unavailable, but source inspection gives a clear plausible path: Discord token inspectors return configured while `channels list --all` falls back to a catalog-only row when the read-only plugin loader does not surface Discord.

## Solution Assessment

Unclear as an automated fix: marking catalog-only rows configured would only mask the symptom, while auto-trusting or auto-installing external Discord plugins after upgrade needs maintainer policy. The narrow maintainable direction is to fix the activation/diagnostic boundary with regression coverage for an upgraded `channels.discord` config.
