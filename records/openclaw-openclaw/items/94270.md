## Summary

Keep open: current main and v2026.6.8 still default omitted agent/Gateway cron run mode to `force`, and the linked open PR https://github.com/openclaw/openclaw/pull/94453 is the active implementation path rather than merged proof.

## What This Changes

Change the agent/MCP cron run default so omitted `runMode` respects due schedule guards instead of force-running, with optional follow-up observability for manual triggers.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/94453 for the default-mode decision, then track trigger/dedup observability separately if maintainers want it.

## Reproduction Assessment

Yes, source-level: current main maps omitted cron agent `runMode` to `force`, Gateway defaults missing `mode` to `force`, and force-mode preflight bypasses enabled and next-run checks. I did not run a live Gateway reproduction to keep this read-only review clean.

## Solution Assessment

Unclear until maintainers decide the compatibility tradeoff. The linked PR is the narrowest default flip, but the documented force default and the requested trigger/dedup observability should not be bundled without product approval.
