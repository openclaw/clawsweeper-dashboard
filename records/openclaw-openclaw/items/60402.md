## Summary

Keep open. Current main still has the heartbeat-disabled targeted wake gap that this PR is trying to address, but the patch is not mergeable as-is because the rewritten targeted wake branch drops the existing per-wake heartbeat override merge from #69021 and the branch currently lacks the required user-facing changelog entry.

## What This Changes

This PR changes heartbeat scheduling so targeted session-scoped event wake requests can bypass global or interval heartbeat-disabled guards, re-arms scheduling when heartbeats are re-enabled, and adds heartbeat runner/wake regression tests.

## Best Possible Solution

Keep the PR’s narrow targeted event-wake direction, but restore heartbeat override merging in the targeted branch, add the missing changelog entry, rebase onto current main, and leave the broader native yielded-parent completion design tracked under #52249.

## Reproduction Assessment

Yes. The PR regression has a high-confidence reproduction path through the existing scheduler test that sends a targeted wake with `heartbeat: { target: "last" }`; after this diff, `runOnce` receives `targetHeartbeat` without that override.

## Solution Assessment

No, not as written. The approach is a reasonable small fix for the ACP heartbeat-disabled seam, but it must preserve targeted heartbeat overrides and it may still be only a Phase A fix for the broader yielded-parent completion issue.
