## Summary

Keep open: the heartbeat-disabled ACP parent wake gap is still source-reproducible on current main, but this PR should not merge as written because it drops current targeted wake override semantics and still lacks contributor real behavior proof.

## What This Changes

The PR moves targeted session wake handling ahead of heartbeat-disabled guards, adds an event-wake bypass for heartbeat-disabled scheduling, resumes interval scheduling after re-enable, and adds heartbeat runner regression tests.

## Best Possible Solution

Rebase or replace this narrow fix on current main so scoped ACP event wakes can run when heartbeat scheduling is disabled while preserving override merging, source/intent forwarding, cooldown semantics, and real behavior proof.

## Reproduction Assessment

Yes, source-level: ACP parent streaming enqueues a parent-session system event and heartbeat wake, while current main can return `disabled` before targeted wake delivery when heartbeat scheduling is off. I did not run a live UI, Matrix, or ACP end-to-end flow in this read-only review.

## Solution Assessment

No, not as written. The direction is narrow and plausible, but the branch must preserve current targeted heartbeat override merging plus newer wake source/intent and cooldown semantics before it is the best fix.
