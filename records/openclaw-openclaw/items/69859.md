## Summary

Keep open: the requested hook-threading feature is still missing on current main, but this PR is not mergeable as submitted because it misses the current delivery contract, omits replay scoping, includes a backup artifact, and has no real behavior proof.

## What This Changes

The branch adds optional `threadId` plumbing for `/hooks/agent`, carries it through cron delivery fields, includes an unrelated cron model-selection change, and adds a backup copy of `src/media/web-media.ts`.

## Best Possible Solution

Rework this as a narrow hook-threading change that parses `threadId`, sets `job.delivery.threadId`, scopes hook replay by `threadId`, adds gateway/cron regression coverage and docs, removes the backup file, and leaves model-selection behavior for a separate PR.

## Reproduction Assessment

Yes, at source level. Current main ignores `payload.threadId` in `normalizeAgentPayload` and never sets `job.delivery.threadId`, while downstream cron delivery already honors `delivery.threadId`.

## Solution Assessment

No. The PR targets the right user problem, but the maintainable fix is to feed the existing `job.delivery.threadId` contract and replay scope, not add stale cron payload fields or unrelated files.
