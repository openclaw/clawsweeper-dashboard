## Summary

Keep open. Current main has OpenRouter response-cache and provider-routing support, but it does not provide an opt-in path that forwards the active OpenClaw session id to OpenRouter for first-request sticky routing; the requested behavior is a valid provider feature that needs maintainer product/privacy review before implementation.

## What This Changes

Request to add an option that forwards OpenClaw's active session id to OpenRouter as `session_id` for sticky routing and prompt-cache affinity.

## Best Possible Solution

Add an explicit OpenRouter-owned opt-in that injects the active run/session affinity value only on verified OpenRouter routes, with focused tests, docs, and privacy wording, or decline the identifier-egress behavior explicitly.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does verify that current main lacks a dynamic OpenRouter session-id forwarding path.

## Solution Assessment

No, not as proposed. A small `resolveExtraParams` merge is not enough because the value is request/session-scoped and third-party identifier egress should be explicit, provider-owned, documented, and tested.
