## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Return a drain result before rechecking health:** `src/gateway/channel-health-monitor.ts:263-272`
  - body: After the 30s drain deadline, `drainActiveRuns` returns the same `void` value as a successful drain. If active work appears after the initial unhealthy snapshot and remains recent, the post-drain health recheck can classify the account as busy/healthy and skip the timeout fallback restart that was supposed to abort and recover the channel. Have the helper report drained, timed out, or stopped, and only suppress restart after a real drain or recovery.
  - confidence: 0.82
- **[P2] Preserve recovery stop semantics:** `src/gateway/channel-health-monitor.ts:291`
  - body: This health-monitor path replaces current main's `stopChannel(..., { manual: false })` with a plain `stopChannel(...)`. Since `stopChannel` defaults `manual` to true, a stop timeout records manual-stop state instead of recovery state and can prevent the automatic restart/backoff path from recovering the channel. Keep the existing non-manual option when stopping for health-monitor recovery.
  - confidence: 0.95
