## Summary

Keep open. Current main still lacks an active-run drain before health-monitor restarts, but this PR remains unsafe as-is because timeout handling can skip the intended restart, the stop call drops current non-manual restart semantics, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a 30s active-run drain before health-monitor channel restarts, a 60s restart cooldown floor, fresh timing resolution, and expanded health-monitor tests.

## Best Possible Solution

Land a rebased gateway fix that drains active runs without losing restart semantics, distinguishes drained/recovered/timeout outcomes, preserves { manual: false }, and includes redacted real Discord or gateway proof.

## Reproduction Assessment

Yes, source-reproducible: current main can restart an unhealthy running account immediately with stopChannel({ manual: false }) and no active-run drain before aborting the channel task. I did not live-reproduce the Discord dropped-text symptom.

## Solution Assessment

No, not as-is. The affected module is the right place, but the branch needs explicit drain outcome handling, timeout restart behavior, preserved non-manual stop semantics, and real-environment proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Treat drain timeouts as restartable:** `src/gateway/channel-health-monitor.ts:268-272`
  - body: If recent activeRuns remain when the 30s drain deadline expires, drainActiveRuns returns without indicating timeout. The post-drain health recheck can then see the account as healthy/busy and continue here, skipping the restart the timeout path is supposed to allow.
  - confidence: 0.91
- **[P2] Preserve non-manual stop semantics:** `src/gateway/channel-health-monitor.ts:291`
  - body: Current main passes { manual: false } for health-monitor stops so stop timeouts remain recovery-restart work. This new stopChannel call omits that option, so stopChannel defaults to manual=true and can mark the account manually stopped on timeout.
  - confidence: 0.88
