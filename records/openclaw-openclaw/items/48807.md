## What This Changes

The PR changes the gateway channel health monitor to wait up to 30 seconds for active runs before stopping a channel, enforce a 60 second restart cooldown floor, add a fresh timing callback, and expand health-monitor tests.

## Best Possible Solution

Land a rebased gateway fix that drains real active runs with explicit outcomes, preserves health-monitor recovery stop semantics, and includes redacted real Discord or gateway proof.

## Reproduction Assessment

Source-reproducible: current main can restart an unhealthy running account immediately with no active-run drain before stopChannel aborts the channel task. The exact production Discord dropped-text symptom was not live-reproduced.

## Solution Assessment

No, not as-is. The affected module is the right place, but the branch needs explicit drain outcome handling, preserved non-manual stop semantics, a rebase, and real-environment proof.

  - confidence: 0.93
