## Summary

Keep open: the drain-before-abort direction is plausible, but PR head still has two source-level regressions, lacks after-fix real behavior proof, and is currently conflicting with main.

## What This Changes

The branch adds a health-monitor drain window, a 60-second restart cooldown floor, fresh timing resolution, and related tests for `src/gateway/channel-health-monitor*`.

## Best Possible Solution

Keep the health-monitor drain direction, but return an explicit drain outcome, continue the restart after timeout, preserve `stopChannel(..., { manual: false })`, rebase onto current main, and require redacted real gateway/Discord proof.

## Reproduction Assessment

Yes, via source inspection: a running disconnected account with recent `activeRuns` and `lastRunActivityAt` can stay busy through `DRAIN_WINDOW_MS`, then PR head treats the post-drain `busy` health result as recovery. The original dropped-reply production symptom still needs after-fix real deployment proof.

## Solution Assessment

No, not as-is. The affected module is the right place, but the patch must distinguish drain timeout from recovery and preserve health-monitor recovery stop semantics before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Treat drain timeouts as restartable:** `src/gateway/channel-health-monitor.ts:268-272`
  - body: When the 30s drain expires while a live run still publishes recent `activeRuns` or `busy` state, this post-drain check sees `reason: "busy"` and skips `stopChannel`/`startChannel`. Return an explicit drain result and only skip for true recovery or operator/config changes.
  - confidence: 0.91
- **[P2] Preserve recovery stop semantics after draining:** `src/gateway/channel-health-monitor.ts:291`
  - body: Current main passes `{ manual: false }` for health-monitor restarts. This moved call omits the option, so `stopChannel` defaults to a manual stop and can suppress the recovery restart path after a timed-out stop.
  - confidence: 0.88
