## Summary

Keep open: current main still restarts unhealthy running channels without a pre-abort drain, but the PR head is not mergeable as-is because a live run that times out the drain window can be mistaken for recovery and skip the promised fallback restart.

## What This Changes

The PR adds a health-monitor drain window before channel restarts, a 60-second restart cooldown floor, fresh timing resolution, and gateway monitor tests.

## Best Possible Solution

Keep the drain-before-abort direction, but make drainActiveRuns return an explicit outcome and only skip restart for stopped, removed, disabled, or genuinely recovered channels; timeout with still-live runs should continue to the fallback restart path.

## Reproduction Assessment

Yes. Source inspection gives a focused reproduction: at PR head, a running disconnected account with activeRuns > 0, busy true, and recent lastRunActivityAt that remains active past DRAIN_WINDOW_MS will make the post-drain health check return healthy/busy and skip restart.

## Solution Assessment

No, not as-is. The branch is in the right gateway module and the drain/cooldown idea is narrow, but timeout must be represented separately from recovery before this is the maintainable fix.
