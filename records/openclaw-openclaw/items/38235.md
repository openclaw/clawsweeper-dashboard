## Summary

Keep open. Current main and the latest release still omit the resolved Slack agent identity from the draft-preview initial send, and open PR #38237 already targets the same fix with closing syntax, so cleanup closure would be premature.

## What This Changes

The issue requests that Slack streaming and draft-preview messages preserve the configured Slack agent username/icon identity instead of showing the default Slack app identity.

## Best Possible Solution

Review and land or replace PR #38237 with a narrow Slack plugin fix that threads identity into draft sends, adds draft-stream regression coverage, and scopes native streaming to verified Slack API support.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists for draft preview streaming: configure a Slack agent identity, enter a draft-preview path, and follow dispatch where `slackIdentity` is resolved but never reaches `createSlackDraftStream` or its initial `sendMessageSlack` call.

## Solution Assessment

Yes for the reproducible draft-preview path: threading the existing `SlackSendIdentity` through the draft stream is the narrow maintainable fix. Native streaming should not be changed without verified Slack SDK/API support for identity overrides.
