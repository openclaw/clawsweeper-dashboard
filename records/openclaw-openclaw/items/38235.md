## Summary

Keep open. Current main still drops Slack agent identity at streaming construction while the normal Slack send/reply path already passes identity, and this issue is paired with open PR #38237 so it should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue requests that Slack streamed messages use the configured agent username and icon, matching non-streaming Slack messages.

## Best Possible Solution

Review and land or close PR #38237, or an equivalent Slack plugin fix, with draft-stream identity threading, focused regression coverage, and a deliberate native-streaming decision.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a Slack agent identity and draft/native streaming, then follow the current dispatch path where `slackIdentity` is resolved but not passed into `createSlackDraftStream` or `startSlackStream`.

## Solution Assessment

Unclear as a complete solution. Forwarding identity through the draft stream is the narrow maintainable fix for preview streaming, but native streaming should be verified against the Slack SDK/API or documented as unsupported.
