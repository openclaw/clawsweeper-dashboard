## Summary

Keep this open: current `main` still drops Slack agent identity on the draft streaming path, and open PR #38237 already uses closing syntax for the narrow repair. The non-streaming reply path has identity support, but draft stream initialization and its first send do not thread that value through.

## What This Changes

Request that Slack streamed messages preserve the configured agent username/icon/emoji behavior already used by non-streaming replies.

## Best Possible Solution

Land or replace #38237 with a narrow Slack plugin patch that threads `SlackSendIdentity` through draft preview creation and regression-tests the first send; verify native streaming limits separately.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the draft stream constructor and initial send lack an identity field even though dispatch has `slackIdentity` in scope and the normal send path supports it.

## Solution Assessment

Yes for the draft path. Threading the existing `SlackSendIdentity` through `createSlackDraftStream` is the narrow maintainable fix; native streaming should not be claimed fixed without Slack SDK/API proof.
