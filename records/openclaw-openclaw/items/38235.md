## Summary

Keep open. Current main still has the Slack draft-preview identity gap: normal Slack reply delivery resolves and forwards `slackIdentity`, but `createSlackDraftStream` has no identity parameter and its first `sendMessageSlack` call omits identity. The issue is also paired with open PR #38237 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Slack streamed replies use the configured agent username and icon, matching non-streamed Slack replies.

## Best Possible Solution

Ship a narrow Slack plugin fix that threads `SlackSendIdentity` into `createSlackDraftStream`, passes the resolved `slackIdentity` from dispatch, forwards it into the initial preview `sendMessageSlack` call, and adds a regression test for that first `chat.postMessage` options object. Native Slack streaming should be verified against the Slack API/SDK contract and documented as a limitation if identity overrides are unavailable there.
