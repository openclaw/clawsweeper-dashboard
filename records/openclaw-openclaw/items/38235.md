## Summary

Keep open. Current main still omits the resolved Slack agent identity from the draft-preview send path, and this issue is paired with open PR #38237 using closing syntax, so cleanup closure would be premature.

## What This Changes

The issue asks Slack streaming delivery to preserve the active agent's custom username and icon, matching non-streaming Slack sends.

## Best Possible Solution

Land or replace #38237 with a complete Slack plugin fix that threads identity into draft sends and either supports native startStream identity through a verified Slack SDK path or documents a confirmed Slack limitation.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure a Slack agent identity and draft preview streaming, then follow the dispatch path where `slackIdentity` is resolved but never reaches `createSlackDraftStream` or its initial `sendMessageSlack` call.

## Solution Assessment

Unclear as a complete path. Threading identity through draft preview is the narrow maintainable fix for partial preview mode, but native streaming needs verified handling because Slack's method docs and Node SDK helper docs do not fully agree.
