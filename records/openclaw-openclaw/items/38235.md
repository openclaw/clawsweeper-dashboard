## Summary

Keep open: current main still drops Slack agent identity on the draft streaming path, while the normal Slack send path supports it, and an open closing PR already targets the narrow repair.

## What This Changes

The issue reports that Slack streaming and draft preview messages do not carry the resolved agent username/icon identity even though normal Slack sends do.

## Best Possible Solution

Thread the existing `SlackSendIdentity` from dispatch into `createSlackDraftStream` and its initial send, add focused regression coverage, and handle native streaming identity as a separate Slack API limitation or follow-up.

## Reproduction Assessment

Yes. Source inspection on current main shows `slackIdentity` is resolved and used for normal delivery but is not passed to `createSlackDraftStream` or the draft stream's first send.

## Solution Assessment

Yes for the draft preview path. Passing the existing Slack identity object through the owner Slack plugin path is the narrow maintainable fix; native streaming should not be claimed fixed without Slack SDK/API proof.
