## Summary

Keep open. Current main still drops Slack agent identity on draft-preview and native streaming construction, while the normal Slack send path already supports custom identity; this issue is also paired with open PR #38237, so it should remain open until that PR or an equivalent complete fix lands.

## What This Changes

The issue requests that Slack custom agent username/icon identity be preserved for streamed messages, matching non-streaming delivery.

## Best Possible Solution

Land a Slack plugin fix that threads the resolved `slackIdentity` into `createSlackDraftStream` and the initial draft `sendMessageSlack` call, adds regression coverage for that first draft `chat.postMessage`, and separately updates `startSlackStream`/dispatch/tests to pass supported `username`/`icon_url`/`icon_emoji` fields for native Slack streaming or documents a verified SDK limitation if the helper rejects them.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure an agent identity and Slack streaming, then follow the current dispatch path where `slackIdentity` is resolved but not passed into `createSlackDraftStream` or `startSlackStream`; no live Slack workspace was needed to identify that deterministic omission.

## Solution Assessment

Unclear for the currently linked PR as a complete solution. Forwarding identity to the draft stream is the right narrow fix for partial preview mode, but the issue should also cover native `chat.startStream` identity now that Slack's current docs list those fields.
