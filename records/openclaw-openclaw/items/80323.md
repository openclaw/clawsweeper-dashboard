## Summary

Keep open. The latest PR head still has blocking correctness issues: the advertised JSONL replay command fabricates Pi/Codex cells instead of running the runtimes, and the live token-efficiency workflow can stop before writing the report it is meant to preserve.

## What This Changes

The PR expands qa-lab with Codex-vs-Pi runtime parity phases 2-5: per-tool fixtures, Codex plugin lifecycle/auth-profile fixtures, token-efficiency reporting, JSONL replay, and QA workflow lanes.

## Best Possible Solution

Land this after JSONL replay invokes actual Pi/Codex runtime cells, the live token lane preserves token reports despite expected drift, and the stack is cleanly based on the Phase 1 runtime-axis work.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows the JSONL CLI wiring a synthetic mock cell runner, and the scheduled live workflow running a drift-sensitive suite before report generation under `set -e` without `--allow-failures`.

## Solution Assessment

No. The harness direction is useful, but this patch does not yet make JSONL replay exercise the runtime-pair path and can lose the token-efficiency report when live parity finds drift.
