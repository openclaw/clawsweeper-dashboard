## Summary

Keep open: the PR adds a useful QA-lab runtime parity harness, but latest head still has merge-blocking correctness issues in JSONL replay and live token reporting, and the supplied proof does not cover those paths.

## What This Changes

This PR adds Codex-vs-Pi runtime parity QA-lab suites, tool/default fixtures, Codex plugin lifecycle fixtures, JSONL replay scaffolding, token/tool reports, private runtime override wiring, and release/live workflow lanes.

## Best Possible Solution

Land a QA harness that exercises real Pi/Codex cells for every advertised lane, preserves live token artifacts on drift, marks missing live usage as unavailable or failing, and documents any remaining report-only rows clearly.

## Reproduction Assessment

Yes. Source inspection at latest head shows the JSONL command injecting a mock cell runner and the live token workflow using set -e before report generation; https://github.com/openclaw/openclaw/issues/80411 provides a concrete live zero-usage token-report repro.

## Solution Assessment

No. The harness direction is useful, but this patch is not the best mergeable solution until replay uses real runtime cells, token reports are generated and invalidated correctly on drift or missing usage, and after-fix proof covers those paths.

Full review comments:

- **[P2] Run JSONL replay through runtime cells:** `extensions/qa-lab/src/cli.runtime.ts:1064`
  - body: `qa jsonl-replay --runtime-pair pi,codex` still injects `createMockJsonlReplayCellRunner()`. That runner fabricates matching Pi/Codex cells from transcript prefixes, so the command cannot catch runtime drift in the replay path it advertises.
  - confidence: 0.93
- **[P2] Generate token reports after drift exits:** `.github/workflows/qa-live-transports-convex.yml:273-288`
  - body: This scheduled lane runs `qa suite` before `qa parity-report --token-efficiency` in one `set -e` shell and does not pass `--allow-failures`. Runtime parity lanes are expected to exit nonzero when they find drift, so the token job can stop before writing the report artifact.
  - confidence: 0.9
- **[P2] Fail live token reports without usage:** `extensions/qa-lab/src/token-efficiency-report.ts:274-284`
  - body: In live mode the report only fails on large token deltas, so a failed run with both runtime cells reporting 0 tokens produces no failures and renders `pass`. The linked live probe observed exactly that, which makes the token-efficiency gate look healthy when no assistant usage was captured.
  - confidence: 0.86
