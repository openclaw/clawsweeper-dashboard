## Summary

Keep open: the harness direction is useful and current-head proof is now supplied, but the PR still has a secret-bearing workflow trust issue and a runtime-report skip semantics bug that should block merge.

## What This Changes

This PR adds a QA-lab Codex-vs-Pi runtime parity harness with runtime-pair suite execution, tool fixtures, JSONL replay, confidence/token reports, and CI/release proof workflows.

## Best Possible Solution

Keep the harness direction, but require trusted-ref/live-secret gating, skip-preserving report semantics, conflict resolution, and a rerun of strict current-head proof before merge.

Yes for the review findings: current PR-head source shows job-scoped live secrets before selected-ref code, and downloaded current-head artifacts show report-only skips rendered as parity-report failures. I did not run repo tests because this review is read-only.
