## Summary

Keep this PR open: the branch adds useful QA Lab runtime-parity infrastructure, but the current patch has blocking correctness issues in the report-only runtime report path and the JSONL replay CLI, and the supplied real behavior proof is not tied to the current head.

## What This Changes

The PR adds QA Lab Codex-vs-Pi runtime parity suites, per-tool fixtures, reports, JSONL replay scaffolding, workflow lanes, and a Codex dynamic-tool accepted-status fix.

## Best Possible Solution

Fix the runtime report semantics for report-only skips, wire or clearly defer JSONL replay runtime execution, then refresh current-head QA proof before maintainer review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: feed buildQaRuntimeParityReport a runtime summary containing a report-only skip, or invoke the new JSONL command path and it will use the synthetic mock cell runner rather than real runtime cells.

## Solution Assessment

No. The direction is useful, but the current implementation is not the best mergeable shape until report-only skips remain non-fatal in the report and JSONL replay either uses the runtime harness or is kept as explicit non-runtime scaffolding.
