## Summary

Keep open: latest head still has blocking QA-harness correctness issues. JSONL replay uses synthetic cells, the live token workflow can skip report generation when drift makes the suite exit nonzero, and live token reporting can mark failed zero-usage runs as a pass; the supplied proof does not cover those paths.

## What This Changes

The branch adds QA-lab runtime-pair execution, tool/default fixtures, Codex plugin lifecycle fixtures, JSONL replay scaffolding, token/tool reports, and release/live workflow lanes for Codex-vs-Pi parity.

## Best Possible Solution

Land the runtime parity harness only after JSONL replay uses real runtime cells, live token reports are preserved and invalidated correctly on drift or missing usage, and remaining broad first-hour rows are either green or explicitly scoped as report-only with proof.

## Reproduction Assessment

Yes. The latest PR head is source-reproducible: the JSONL command injects a mock cell runner, the live workflow uses fail-fast ordering before report generation, and the token report pass path ignores live runtime failure or missing usage; https://github.com/openclaw/openclaw/issues/80411 also documents the live zero-usage symptom.

## Solution Assessment

No. The harness direction is useful, but this is not the best mergeable form until replay exercises the advertised runtime axis and live token artifacts cannot report a healthy pass from failed or missing-usage cells.
