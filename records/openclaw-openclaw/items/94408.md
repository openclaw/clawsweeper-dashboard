## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main requires `key` in `SessionsSendParamsSchema`, and `assertValidParams` runs before `handleSessionSend` reads any fallback key. I did not run a live gateway or subagent reproduction in this read-only review.

## Solution Assessment

No. This patch leaves `key` required, so `sessionKey`-only calls still fail schema validation before the handler fallback can run; the safer path is a maintainer-approved alias contract or caller-side normalization.
