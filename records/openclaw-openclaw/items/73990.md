## Summary

Keep open. Current main still skips contentful transcript entries when there is no meaningful usage object or model identity, and the existing no-telemetry regression fixture still expects a null transcript-usage snapshot. The related open #75357 is adjacent and narrower around requesting streaming usage; it does not supersede this broader fallback-estimation request.

## What This Changes

The issue requests a conservative transcript-derived context-usage estimate for local/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a generic transcript fallback in the Gateway/status usage extractor: keep valid provider-reported usage authoritative, but when a transcript has visible content and no positive usage-derived context total, estimate prompt/context tokens conservatively from transcript text and mark that snapshot usable for status/session displays.

## Reproduction Assessment

Yes. Current main has a deterministic fixture in `src/gateway/session-utils.fs.test.ts` that writes a non-empty user/assistant transcript with no usage telemetry and expects `readLatestSessionUsageFromTranscript` to return `null`.

## Solution Assessment

Yes. A narrow extractor/status fallback is the best fix because it avoids changing provider transport behavior and only applies when there is no positive provider-derived context total to trust.
