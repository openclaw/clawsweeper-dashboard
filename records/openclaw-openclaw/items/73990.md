## Summary

Keep open. Current main still returns no usable transcript usage snapshot for a contentful session when provider usage telemetry is absent, and both status/session-list paths require a positive transcript usage total before they can rescue the context meter. The linked #75357 is adjacent transport telemetry work, but it does not replace this no-telemetry fallback request.

## What This Changes

The issue requests a conservative transcript-derived context usage fallback for local Llama/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a generic transcript-content fallback in the Gateway/status usage path: preserve valid provider usage first, then estimate a conservative prompt/context token count from visible transcript text only when no positive usage-derived context total exists, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. The current `usage-empty` fixture in `src/gateway/session-utils.fs.test.ts` creates a contentful user/assistant transcript without usage telemetry and asserts that transcript usage is null; the status path then drops that missing snapshot before formatting context usage.

## Solution Assessment

Yes. The requested narrow extractor/status fallback is the best maintainable direction because provider transports and usage normalization remain authoritative whenever they supply usable telemetry.
