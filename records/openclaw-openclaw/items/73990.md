## Summary

Keep open. Current main still drops or ignores contentful local/OpenAI-compatible transcripts when no positive provider usage telemetry is present, so the status/session context meter can still stay at unknown or 0% despite real transcript content.

## What This Changes

The issue requests a conservative transcript-derived context usage estimate for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a generic transcript-content estimate only when no positive provider usage snapshot exists, then wire that fallback through status, session list, and session_status displays with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible on current main: the existing contentful no-usage transcript fixture still asserts null, and both status and session-list consumers drop null or zero snapshots. I did not run Vitest because this read-only checkout does not have `pnpm` available.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrow maintainable fix because provider telemetry remains authoritative and the existing streaming usage request path stays independent.
