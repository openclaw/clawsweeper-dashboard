## Summary

Keep open. Current main still drops contentful local/OpenAI-compatible transcripts when assistant turns have no meaningful usage telemetry, so status and session-list context usage can remain empty or zero despite real transcript content.

## What This Changes

Request a conservative transcript-derived context estimate for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a generic bounded fallback in the gateway/status usage path: provider telemetry remains authoritative when present, but contentful transcripts with no usable usage snapshot produce a conservative prompt/context estimate that status and session-list displays treat as usable.

## Reproduction Assessment

Yes. The current `usage-empty` fixture creates a contentful transcript with no usage telemetry and expects the reader to return null, matching the reported empty or zero context display path.

## Solution Assessment

Yes. A narrow extractor/status fallback is the maintainable fix because it preserves provider telemetry while giving local no-telemetry sessions a conservative context meter.
