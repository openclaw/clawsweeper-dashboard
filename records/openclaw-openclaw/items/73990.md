## Summary

Keep open. Current main still has the no-telemetry gap: contentful assistant transcript entries without usage or model identity are skipped, so status and session-list fallbacks can return no usable context snapshot; the related #75357 streaming-usage request is complementary, not a replacement for this fallback.

## What This Changes

Provide a conservative transcript-derived context usage estimate for local/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a shared transcript-content fallback in the gateway usage extractor so status and session-list displays get a positive conservative estimate only when provider telemetry is unavailable.

## Reproduction Assessment

Yes. The existing `usage-empty` transcript fixture is a high-confidence reproduction shape because current main still asserts that contentful transcript lines with no assistant usage snapshot return null.

## Solution Assessment

Yes. A narrow extractor/status fallback is the best fit because it fixes no-telemetry local sessions without changing provider transport behavior or masking real usage when providers report it.
