## Summary

Keep open. Current main still drops contentful local/OpenAI-compatible transcripts when provider usage telemetry is absent or zero, so the closed streaming-usage fixes do not cover the transcript-derived fallback requested here.

## What This Changes

The issue asks OpenClaw to estimate session context usage from transcript content when local/OpenAI-compatible providers do not return usable usage telemetry.

## Best Possible Solution

Add a shared conservative transcript-content estimate used only when no positive provider usage snapshot exists, then cover both status and session-list displays with focused regression tests.

## Reproduction Assessment

Yes. Current main has a contentful no-usage transcript fixture that asserts null, and the status/session-list paths discard null or zero snapshots before rendering context usage.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because provider telemetry remains authoritative while the separate streaming-usage capture path stays independent.
