## Summary

Keep open. Current main still lacks the requested no-telemetry transcript-content estimate: a contentful transcript with no assistant usage snapshot returns null, and downstream status/session paths drop null or non-positive usage, while the closed OpenAI-compatible streaming fixes only help endpoints that return usage telemetry.

## What This Changes

The issue requests a conservative transcript-derived context estimate for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a bounded transcript-content estimate fallback that is used only when no positive provider usage snapshot exists, while keeping provider-reported usage authoritative across status and session-list displays.

## Reproduction Assessment

Yes, source-reproducible: current coverage writes user and assistant content with no usage telemetry and asserts the transcript usage reader returns null, and the status/session display paths then discard null or non-positive snapshots. I did not run a live local LLM endpoint because this was a read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is in the transcript usage extraction/status fallback path: estimate from bounded transcript content only when telemetry is absent or zero, and preserve current positive provider telemetry behavior.
