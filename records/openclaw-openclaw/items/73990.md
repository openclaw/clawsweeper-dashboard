## Summary

Keep open. Current main still ignores contentful transcript entries when provider usage telemetry is absent, so local/OpenAI-compatible sessions can fall back to null or zero-ish context display despite the documented transcript fallback behavior.

## What This Changes

The issue asks OpenClaw to estimate local/OpenAI-compatible context usage from transcript content when provider usage telemetry is absent or zero.

## Best Possible Solution

Keep provider-reported usage authoritative and synthesize a conservative prompt/context estimate from transcript content only when no positive provider usage snapshot exists, with regression coverage for the affected display surfaces.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a transcript with user/assistant content and no usage telemetry reaches the existing fixture in src/gateway/session-utils.fs.test.ts:1458 and currently returns null instead of a context estimate.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because native provider telemetry and the separate streaming include_usage path remain authoritative when they work.
