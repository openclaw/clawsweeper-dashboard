## Summary

Keep open. Current main still lacks a transcript-content estimate when local/OpenAI-compatible transcripts have real messages but no positive provider usage telemetry; the related streaming `include_usage` fix only helps endpoints that return usage after being asked.

## What This Changes

Request a conservative transcript-derived context estimate for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a shared bounded transcript-content estimate used only when no positive provider usage snapshot exists, then cover extractor, status, session_status, and relevant session-list displays with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a current-main fixture with user and assistant content but no usage asserts null, and the downstream status/session paths discard null or zero snapshots before rendering context usage.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because provider telemetry remains authoritative and the separate OpenAI-compatible streaming telemetry path stays unchanged.
