## Summary

Keep open. Current main still returns no usable transcript snapshot for contentful local/OpenAI-compatible sessions when assistant turns lack meaningful usage telemetry; the related #75357 covers requesting streaming usage, but it does not replace this no-telemetry fallback.

## What This Changes

The issue asks OpenClaw to estimate context usage from transcript content for local/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Provider telemetry should remain authoritative, with a bounded transcript-text estimate only when no usable usage snapshot exists, shared by status and session-list displays.

## Reproduction Assessment

Yes. A focused reproduction can use the existing contentful usage-empty transcript shape, which current main asserts returns null instead of a conservative estimate.

## Solution Assessment

Yes. A narrow extractor/status fallback is the best fit because it fixes no-telemetry sessions without changing provider transport behavior or masking #75357’s streaming-usage capture work.
