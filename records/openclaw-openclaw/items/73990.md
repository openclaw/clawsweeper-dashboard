## Summary

Keep open. Current main still lacks a transcript-derived usage estimate when local/OpenAI-compatible assistant turns have content but no positive usage telemetry, and the related #75357 streaming-usage request does not cover this no-telemetry fallback.

## What This Changes

Request a conservative transcript-derived context estimate for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a shared transcript-text fallback that estimates prompt tokens only when no positive provider snapshot exists, with regression coverage for status and session-list displays.

## Reproduction Assessment

Yes. The current contentful no-usage transcript fixture asserts a null usage snapshot, and the status/session-list code then has no positive context value to display.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because provider telemetry remains authoritative while #75357 can handle the separate streaming-usage capture gap.
