## Summary

Keep open. Current main still drops contentful local/OpenAI-compatible transcripts when no positive provider usage telemetry or model identity is present, and the separate streaming `include_usage` work only helps endpoints that return telemetry after being asked.

## What This Changes

Local Llama/OpenAI-compatible sessions should fall back to a conservative transcript-derived context estimate when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a shared bounded transcript-content estimate only when no positive provider usage snapshot exists, then cover extractor, status, session_status, and session-list displays with focused regressions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a current-main fixture with user and assistant content but no usage telemetry still asserts null, and downstream status/session paths discard null or zero snapshots before rendering context usage.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because provider telemetry remains authoritative and the OpenAI-compatible streaming telemetry path stays unchanged.
