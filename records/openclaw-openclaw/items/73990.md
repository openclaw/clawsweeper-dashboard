## Summary

Keep open. Current main still returns no transcript usage snapshot for contentful sessions when provider telemetry is absent, and the status/session display paths then drop that null or non-positive usage despite docs promising transcript fallback for custom-provider zero displays.

## What This Changes

The issue requests a conservative transcript-derived context estimate for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a bounded transcript-content estimate only when no positive provider usage snapshot exists, while keeping provider-reported usage and preserved stale totals authoritative.

## Reproduction Assessment

Yes, source-reproducible: current tests create a contentful transcript with no usage telemetry and assert the usage reader returns null, and current status/session paths discard that null or non-positive usage. I did not run a live local LLM endpoint because this was a read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is in transcript usage extraction/status fallback: estimate from bounded transcript content only when telemetry is absent or zero, and preserve positive provider telemetry behavior.
