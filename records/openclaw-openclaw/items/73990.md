## Summary

Keep open. Current main still has the no-telemetry gap: transcript content without positive provider usage is not converted into a conservative context estimate, so status surfaces can still collapse to null or zero despite the separate `stream_options.include_usage` fix for endpoints that do return telemetry.

## What This Changes

The issue requests a transcript-derived context usage fallback for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a shared conservative transcript-content estimate used only when no positive provider usage snapshot exists, then cover `/status`, `session_status`, and `sessions.list` with focused regressions.

## Reproduction Assessment

Yes, by source inspection. Current main has a contentful no-usage transcript fixture that asserts null, and the status/session-list consumers discard null or zero snapshots before rendering context usage.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because provider telemetry stays authoritative while the existing streaming-usage request path remains separate.
