## Summary

Keep open. Current main still returns no usable context snapshot for contentful local/OpenAI-compatible transcripts when usage telemetry is absent or zero, so the user-visible 0%/missing-context display remains possible; the related #75357 covers streaming usage capture, not this no-telemetry fallback.

## What This Changes

Add a conservative transcript-derived context usage estimate for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Land a narrow extractor/status fix that estimates transcript context only when no positive provider usage snapshot exists, while keeping #75357’s streaming-usage capture as separate work.

## Reproduction Assessment

Yes. Current main has a contentful no-usage transcript fixture that expects a null snapshot, and the status/session-list paths require a non-null positive snapshot before displaying meaningful context usage.

## Solution Assessment

Yes. A bounded transcript fallback is the narrowest maintainable fix because provider telemetry stays authoritative and local endpoints that omit telemetry still get a conservative context meter.
