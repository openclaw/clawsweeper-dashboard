## Summary

Keep open. Current main still lacks a transcript-content estimate when a local/OpenAI-compatible transcript has real messages but no positive provider usage telemetry; the existing streaming-usage fix only helps endpoints that return usage after `stream_options.include_usage` is requested.

## What This Changes

Request a conservative transcript-derived context estimate for local/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a shared conservative transcript-content estimate used only when no positive provider usage snapshot exists, then cover status, session-list, and session_status displays with focused regression tests.

## Reproduction Assessment

Yes. Current main has a contentful no-usage transcript fixture that asserts null, and the status/session-list paths discard null or zero snapshots before rendering context usage.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because provider telemetry remains authoritative while the separate streaming-usage capture path stays independent.
