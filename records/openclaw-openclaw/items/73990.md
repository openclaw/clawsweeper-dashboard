## Summary

Keep open. Current main still lacks a transcript-content fallback when provider usage telemetry is absent, and the existing regression fixture still asserts that a contentful no-usage transcript returns null. The related #75357 report is adjacent transport work for endpoints that can return streaming usage, but it does not replace this no-telemetry fallback request.

## What This Changes

The issue asks OpenClaw to estimate local/OpenAI-compatible session context usage from transcript content when provider usage telemetry is missing or zero.

## Best Possible Solution

Add a generic, bounded transcript-content estimate in the gateway/status usage path: preserve provider telemetry when present, otherwise estimate prompt/context tokens from renderable transcript text and mark that fallback as usable for status and session-list displays.

## Reproduction Assessment

Yes. The current `usage-empty` fixture creates a contentful transcript with no usage telemetry and expects the reader to return null, which matches the reported permanent zero/empty context display path.

## Solution Assessment

Yes. A narrow extractor/status fallback is the maintainable solution because provider transports remain authoritative when they report usage, while local no-telemetry sessions still get a conservative context meter.

## Repair Work Prompt

Fix the local/OpenAI-compatible no-usage telemetry context display bug tracked by https://github.com/openclaw/openclaw/issues/73990. Current main returns null for contentful transcripts when assistant records have no provider usage, causing `/status`, `session_status`, and session-list context usage to stay empty/zero. Implement a generic bounded transcript-content fallback in the gateway usage reader: keep valid provider usage authoritative, but when usage is absent/zero and transcript content exists, estimate prompt/context tokens conservatively from renderable transcript text and mark the snapshot usable. Preserve bounded reads for session listing and do not conflate this with #75357's transport-side streaming-usage request unless the same patch explicitly covers both. Update or replace the `usage-empty` regression test, add status/session-list coverage as needed, and add a user-facing `CHANGELOG.md` fix entry with appropriate reporter credit. Do not change provider defaults, transport routing, or compaction policy beyond what is required for this fallback.
