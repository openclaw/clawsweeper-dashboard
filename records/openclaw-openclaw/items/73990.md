## Summary

Keep open. Current main still ignores contentful local/OpenAI-compatible transcripts when no positive provider usage snapshot exists, so the related #75357 streaming `include_usage` fix does not cover endpoints that omit telemetry entirely.

## What This Changes

Local/OpenAI-compatible sessions should estimate context usage from transcript content when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a shared bounded transcript-content estimate only when no positive provider usage snapshot exists, then thread it through status, session_status, and session-list displays without overriding real telemetry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a current-main fixture with user and assistant content but no usage telemetry asserts null, and downstream status/session paths discard null or zero snapshots before rendering context usage.

## Solution Assessment

Yes. A bounded extractor/status fallback is the narrowest maintainable fix because provider telemetry remains authoritative and the separate OpenAI-compatible streaming telemetry path stays unchanged.
