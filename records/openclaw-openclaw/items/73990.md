## Summary

Keep open. Current main still has the no-telemetry gap: the transcript usage extractor requires provider usage, cost, or model identity before returning a snapshot, and the existing fixture for a contentful transcript without usage still expects null. The linked #75357 is adjacent OpenAI-compatible transport telemetry work, but it does not cover the transcript-derived fallback requested here.

## What This Changes

The issue requests a conservative transcript-derived context usage estimate for local Llama/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a generic, bounded transcript-content fallback in the Gateway/status usage extraction path: keep provider usage authoritative, estimate prompt/context tokens from visible transcript text only when no positive usage-derived context total exists, and cover status plus session-list behavior with focused regressions and a changelog entry.

## Reproduction Assessment

Yes. The current `usage-empty` fixture in `src/gateway/session-utils.fs.test.ts` is a high-confidence reproduction because it creates a non-empty user/assistant transcript with no usage telemetry and verifies that current main returns no usable usage snapshot.

## Solution Assessment

Yes. The requested extractor/status fallback is the narrowest maintainable fix because it leaves provider telemetry authoritative and only fills the display gap for local/OpenAI-compatible providers that omit usage.
