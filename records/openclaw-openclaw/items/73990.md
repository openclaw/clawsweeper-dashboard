## Summary

Keep open. Current main still has the same no-telemetry gap: transcript usage extraction only returns a usable snapshot when it sees nonzero usage, derived prompt totals, cost, or model identity, so contentful local/OpenAI-compatible transcripts without provider usage still fall through to null and status/listing displays cannot backfill a nonzero context estimate.

## What This Changes

The issue asks OpenClaw to estimate context usage from transcript content when local/OpenAI-compatible providers do not report usable token telemetry.

## Best Possible Solution

Add a shared, bounded transcript-content estimate only when no usable usage snapshot exists, then feed that snapshot through both status and session-list displays while keeping provider telemetry authoritative.

## Reproduction Assessment

Yes. A focused reproduction can write a transcript with real user/assistant content but no usable assistant usage; current main already asserts that shape returns null instead of an estimate.

## Solution Assessment

Yes. A narrow extractor/status fallback is the maintainable fix because it handles no-telemetry local sessions without conflating this with #75357’s provider transport usage-capture work.
