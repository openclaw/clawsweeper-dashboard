## Summary

Keep open. Current main still drops contentful transcript chunks that have no provider usage telemetry, and the status/session-list paths cannot recover a useful context meter when the extractor returns null or no positive total.

## What This Changes

The issue asks OpenClaw to estimate status/session context usage from transcript content for local Llama/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a generic transcript-content fallback in the Gateway/status usage path: preserve valid provider usage first, then estimate conservative prompt/context tokens from visible transcript text only when no positive usage-derived context total exists, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. The current `usage-empty` fixture creates a contentful user/assistant transcript without usage telemetry and asserts that the transcript usage reader returns null, which is the reported zero-context path.

## Solution Assessment

Yes. A narrow extractor/status fallback is the best maintainable direction because provider telemetry remains authoritative when present, while local/OpenAI-compatible no-telemetry sessions still get a conservative context estimate.
