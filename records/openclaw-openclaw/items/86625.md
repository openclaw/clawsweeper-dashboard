## Summary

This PR is still useful and should stay open for maintainer review: it removes the docs redirect that currently collides with the `openclaw docs` MCP server URL, adds a focused regression guard, and supplies deploy-candidate plus local Worker proof; the remaining blocker is the intentional compatibility tradeoff for the public `/mcp` docs redirect and the coordinated external docs-router deployment.

## What This Changes

The branch removes the `/mcp` redirect from `docs/docs.json` and adds a `docsSearchCommand` regression assertion that the CLI MCP server path is not also a docs redirect source.

## Best Possible Solution

Land the redirect removal with the regression guard only after maintainers accept the short-URL compatibility tradeoff and coordinate the companion docs-router deployment so `/mcp` serves the MCP endpoint while `/cli/mcp` remains the docs page.

## Reproduction Assessment

Yes, source and issue evidence give a high-confidence reproduction path: `openclaw docs heartbeat` calls mcporter with `https://docs.openclaw.ai/mcp.search_open_claw`, which resolves the server path to `/mcp`; the linked report shows the shipped service returned HTML there instead of SSE. I did not rerun production from this read-only review.

## Solution Assessment

Yes, this is a narrow and maintainable repo-side fix if maintainers accept reclaiming `/mcp` for the MCP endpoint: remove the docs redirect and keep a regression guard aligned with the CLI selector. The only safer alternative is a product decision to move the MCP endpoint to a different non-colliding path and ship the corresponding CLI/docs-router change.
