Full review comments:

- **[P1] Keep dry-run out of repair mode:** `src/cli/program/register.maintenance.ts:37`
  - body: `doctor --fix --dry-run` still passes `repair: true` and only adds `dryRun`. The Doctor prompter derives `shouldRepair` from repair mode, so downstream health contributions treat this as a writeful fix run and structured repairs get `dryRun: false`; a preview flag needs to override repair posture before those paths run.
  - confidence: 0.92
- **[P1] Disable writeful preflight during dry-run:** `src/commands/doctor-config-flow.ts:72`
  - body: `--dry-run` still calls `runDoctorConfigPreflight` with the default state and legacy-config migrations enabled, and with repair-prefixed config recovery enabled. That means a command advertised as preview-only can copy legacy config, migrate state, or recover `openclaw.json` before the final dry-run branch returns `shouldWriteConfig: false`.
  - confidence: 0.93
- **[P1] Redact config values before printing diffs:** `src/commands/doctor/finalize-config-flow.ts:25-29`
  - body: The new flat diff serializes raw old and new values for every changed config path. OpenClaw config can contain tokens, API keys, SecretRefs, local service env vars, and credential-bearing URLs, so preview output can leak secrets into terminal logs or PR proof unless it uses the existing sensitive-path/redaction machinery.
  - confidence: 0.9
