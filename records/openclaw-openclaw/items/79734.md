## Summary

Keep open: source review found the new flag is not yet a reliable read-only preview, and the submitted proof does not exercise the patched dry-run command.

## What This Changes

Adds a `--dry-run` option to `openclaw doctor` with config diff output, docs, and unit tests.

## Best Possible Solution

Implement doctor dry-run as a true read-only planning mode across config repair and downstream doctor contributions, previewing the same config mutations as `--fix` without writing files, state, services, or secrets.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `openclaw doctor --fix --dry-run` still enters repair-mode doctor contributions, while plain `openclaw doctor --dry-run` skips `runDoctorRepairSequence`. A config or state fixture that triggers those repairs would exercise both failures.

## Solution Assessment

No. Adding the diff only at `finalizeDoctorConfigFlow` is too narrow; dry-run needs to control repair planning and write suppression for the whole doctor flow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep dry-run out of repair mode:** `src/flows/doctor-health.ts:13`
  - body: With `--fix --dry-run`, this guard only skips the Nix write assertion; the original `repair` option still reaches `createDoctorPrompter`, so downstream doctor contributions see `prompter.shouldRepair === true` and can migrate sandbox registries, repair sessions, reinstall services, or mutate config before the final writer runs. Dry-run must disable or separately model repair side effects across the whole doctor flow, not just suppress the config-flow write.
  - confidence: 0.91
- **[P2] Preview the same repairs as --fix:** `src/commands/doctor-config-flow.ts:71-72`
  - body: For plain `openclaw doctor --dry-run`, `shouldRepair` remains false, so the existing `runDoctorRepairSequence` branch is skipped. That means the diff omits many mutations that `doctor --fix` would apply, such as stale plugin config, bundled plugin load path, allowlist, and invalid plugin config repairs. Dry-run should plan those repairs without committing them.
  - confidence: 0.88
- **[P3] Report clean dry-runs explicitly:** `src/commands/doctor/finalize-config-flow.ts:12`
  - body: This branch only runs when `pendingChanges` is true, so a clean `openclaw doctor --dry-run` never emits the promised `No config changes detected.` note. It falls through to the normal non-repair path instead, which can leave users without confirmation that dry-run actually evaluated the config.
  - confidence: 0.77
