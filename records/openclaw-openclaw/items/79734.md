## Summary

Keep this PR open but do not merge yet: it targets a real doctor preview feature, but the current patch still exposes raw config values, leaves repair-mode health side effects active under `--fix --dry-run`, and lacks convincing patched dry-run/no-write proof.

## What This Changes

The PR adds a `doctor --dry-run` CLI option, dry-run config diff output, write guards, docs, and finalize-flow tests.

## Best Possible Solution

Implement dry-run as an end-to-end read-only planning mode that redacts diff values, compares against an immutable original config, disables all repair side effects, and is proven with real no-write output.

## Reproduction Assessment

Yes. By source inspection of the PR head, `--fix --dry-run` still leaves repair mode active, compares against mutable config state, and emits raw serialized config values.

## Solution Assessment

No. Guarding the final config write and one repair sequence is too narrow; the safer solution is a read-only dry-run contract across config flow, prompter behavior, and doctor health contributions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Redact dry-run diff values before printing:** `src/commands/doctor/finalize-config-flow.ts:25-29`
  - body: The diff prints serialized old/new values for every config path, so repairs involving `gateway.auth.token`, provider `apiKey`, channel tokens, or other sensitive fields can leak secrets to terminal output. Build the diff from redacted config objects or mask sensitive paths before calling `note()`.
  - confidence: 0.91
- **[P1] Disable repair mode during dry runs:** `src/flows/doctor-health.ts:13`
  - body: This dry-run exception only bypasses the write-mode assertion; the command still builds the prompter from `--fix` options, so downstream health contributions can auto-approve repairs that copy auth stores, migrate legacy state, rebuild plugin/sandbox registries, or run repair scripts before the final config writer is skipped. Force a read-only prompter/repair mode when `dryRun` is true.
  - confidence: 0.9
- **[P2] Compare dry runs against the original config:** `src/commands/doctor-config-flow.ts:72`
  - body: With `--fix --dry-run`, `shouldRepair` stays true and existing config mutation helpers replace `cfg` with the proposed config before finalization. The finalizer can then compare `cfg` to `candidate` and report no diff for the exact changes `--fix` would apply; keep an immutable baseline or run preview mutations without advancing `cfg`.
  - confidence: 0.88
- **[P3] Report clean dry runs:** `src/commands/doctor/finalize-config-flow.ts:13`
  - body: The dry-run branch is gated by `pendingChanges`, so a clean `openclaw doctor --dry-run` falls through without the documented `No config changes detected.` note. The added test misses this because it sets `pendingChanges: true` despite the test title saying there are no pending changes.
  - confidence: 0.8
