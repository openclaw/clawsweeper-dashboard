## Summary

Keep open: this PR adds the requested CLI surface, but the patch still leaves `--fix --dry-run` in repair mode for later doctor contributions and skips repair planning that the dry-run output claims to preview. The supplied proof also does not show patched `openclaw doctor --dry-run` behavior or a verified no-write result.

## What This Changes

Adds a `--dry-run` flag to `openclaw doctor`, threads it through doctor config finalization, renders a flat config diff, updates docs, and adds finalize-flow tests.

## Best Possible Solution

Implement doctor dry-run as a whole-command read-only planning mode that renders the same config/state/service repair plan as `--fix` without writing files, installing packages, running service repairs, or prompting to mutate.

## Reproduction Assessment

Yes for the review blockers: source inspection shows `openclaw doctor --fix --dry-run` still constructs a repair-mode prompter and runs side-effecting health contributions, while skipping repair-sequence planning. A fixture with legacy auth/state/sandbox data or a config needing repair-sequence mutations would exercise the failures.

## Solution Assessment

No. Suppressing the final config writer and skipping the repair sequence is not the narrow maintainable solution; dry-run needs a read-only repair planner and a prompter/flow mode that prevents all doctor side effects while still reporting them.
