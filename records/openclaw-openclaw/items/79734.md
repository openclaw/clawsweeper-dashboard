## Summary

Keep open: the doctor dry-run feature is still useful and not implemented on current main or v2026.6.8, but this patch is not merge-ready because the dry-run flag is applied too late in the doctor flow and does not yet prove a no-write/no-side-effect run.

## What This Changes

Adds a `--dry-run` option to `openclaw doctor --fix`, threads it through doctor config finalization and write guards, prints a flat proposed config diff, and updates doctor docs/tests.

## Best Possible Solution

Implement doctor dry-run as a command-wide preview mode that disables preflight/state/service/plugin writes before any repair-mode prompter or health contribution runs, redacts diff values, and uses the current structured repair preview contract where possible.

The contributor proof does not show a real patched dry-run command with unchanged config/state/plugin/service outputs.

## Solution Assessment

No. The patch is a plausible config-flow preview, but the best current fix should be command-wide and side-effect-aware, especially now that main has a structured repair preview contract for converted doctor checks.
