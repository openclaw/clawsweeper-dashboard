Full review comments:

- **[P1] Disable writeful preflight during dry-run:** `src/commands/doctor-config-flow.ts:72`
  - body: This makes `--dry-run` enter repair-oriented preflight before the preview decision runs. `runDoctorConfigPreflight` can migrate legacy state/config and recover invalid config files, so dry-run needs to be normalized before this call or preflight must be forced into a read-only mode.
  - confidence: 0.92
- **[P1] Keep dry-run out of repair mode before health checks:** `src/flows/doctor-health.ts:13`
  - body: The new guard only skips the Nix write assertion; it leaves `options.repair` true. `createDoctorPrompter` therefore keeps `shouldRepair` enabled for downstream health contributions, which can still run repair, plugin, state, package, and service side effects under a dry-run invocation.
  - confidence: 0.9
- **[P1] Redact config values before printing diffs:** `src/commands/doctor/finalize-config-flow.ts:25-29`
  - body: The new diff prints raw old and new values for every changed config path. OpenClaw config includes sensitive paths such as `gateway.auth.token` and SecretRef objects, so copied terminal output can expose credentials or secret identifiers unless the preview uses path-aware redaction.
  - confidence: 0.89
