## Summary

This is an actionable unresolved bug on current main: same-model signed thinking blocks are preserved and serialized into later Anthropic requests without accounting for the request’s effective thinking mode. A viable open PR explicitly closes this issue, so the issue should remain open until that implementation is corrected as needed and merged.

## What This Changes

Prevent reused sessions from repeatedly failing Anthropic requests when persisted thinking blocks conflict with the current thinking mode.

## Best Possible Solution

Derive one effective thinking mode at the shared Anthropic request boundary, omit incompatible thinking from completed prior turns when disabled, preserve active tool-loop blocks, and prove recovery in a real reused session.

## Reproduction Assessment

Yes at the source level. The reported configuration-change sequence maps directly to current main preserving and serializing signed thinking while a later request can omit a compatible thinking configuration; a live API reproduction was not independently executed.

## Solution Assessment

Yes in direction: normalization belongs in shared Anthropic request conversion rather than cron handling or persistent storage. The implementation must cover every effectively disabled path and distinguish completed prior turns from an active tool-use continuation.
