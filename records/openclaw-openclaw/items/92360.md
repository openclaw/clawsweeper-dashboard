## Summary

Keep this open: current `main` still has a source-level path that can replay stored Anthropic thinking blocks without an enabled current thinking configuration, and the linked closing PR is open, unmerged, and still under review.

## What This Changes

The issue reports that long-lived persistent Anthropic sessions can be poisoned by stored thinking blocks after thinking is disabled or changed, causing all later turns to be rejected until the session is reset.

## Best Possible Solution

Review, fix, and land https://github.com/openclaw/openclaw/pull/92373 or a replacement that derives the effective Anthropic thinking mode before payload conversion and strips or downgrades incompatible prior thinking blocks while preserving required active tool-use thinking blocks.

## Reproduction Assessment

Yes at source level: current `main` can replay stored same-model Anthropic thinking blocks while the current payload lacks an enabled thinking configuration. I did not run a live Anthropic persistent-session repro in this read-only review.

## Solution Assessment

Yes on direction, but not yet complete on the linked PR. The right fix belongs at the Anthropic request boundary rather than cron-specific handling, and it should use the effective request mode instead of only an explicit `thinkingEnabled === false` value.
