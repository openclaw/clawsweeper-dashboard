## Summary

Keep this issue open: current main still builds Anthropic payloads that can replay signed thinking blocks while omitting an enabled thinking configuration, and the issue-specific closing PR is open with a review blocker instead of merged.

## What This Changes

The issue reports that long-lived Anthropic sessions can retain prior Anthropic thinking blocks and then fail future turns when the current thinking setting is disabled, omitted, or changed.

## Best Possible Solution

Derive the effective Anthropic thinking mode before payload conversion and normalize prior thinking blocks for every disabled or omitted-thinking request, while preserving mandatory/adaptive thinking and active tool-turn constraints.

## Reproduction Assessment

Yes at source level: current main can replay signed Anthropic thinking blocks from persisted same-model history while the current request sends no enabled thinking configuration. I did not run a live Anthropic persistent-session repro in this read-only review.

## Solution Assessment

Yes, the best direction is request-boundary normalization in the Anthropic provider path, not a cron-only workaround. The open linked PR is the right work lane but still needs the effective-disabled case and real behavior proof before merge.
