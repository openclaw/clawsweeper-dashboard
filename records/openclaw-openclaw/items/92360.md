## Summary

Current `main` still permits native Anthropic thinking blocks to be replayed without a matching current thinking configuration. Keep this issue open while the linked implementation PR is reviewed and completed; it currently misses an effective-disabled path and lacks live persistent-session proof.

## What This Changes

Prevent persisted Anthropic thinking blocks from making long-lived sessions reject every later turn after thinking becomes disabled or incompatible.

## Best Possible Solution

Compute one effective Anthropic thinking mode before message conversion, downgrade or omit incompatible historical thinking whenever the request is effectively disabled, preserve blocks required by active thinking and tool-use contracts, and prove recovery with a real persistent session.
