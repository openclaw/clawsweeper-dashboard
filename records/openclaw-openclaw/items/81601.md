## Summary

Keep open for maintainer review and landing. The PR is a focused fix for a real ClawHub archive body stall, with reader-owned cancellation and regression coverage; I found no blocking code or security findings, but two plugin-contract CI jobs are currently failing outside the touched files.

## What This Changes

The branch adds ClawHub response-body read helpers, stalled-body tests, and a changelog entry for skill, package, and ClawPack downloads.

## Best Possible Solution

Land this reader-owned ClawHub archive body timeout fix after CI is green or confirmed unrelated, then supersede https://github.com/openclaw/openclaw/pull/80006 and let the linked issue close on merge.

## Reproduction Assessment

Yes, at source level: current main clears the ClawHub request timeout when fetch returns, then archive paths await direct response.arrayBuffer() calls. A headers-then-stalled body can therefore leave the install path pending, and the PR proof exercises that stalled-stream condition after the patch.

## Solution Assessment

Yes. The reader-owned getReader() loop is the narrow maintainable fix because it can cancel the active body read while preserving existing checksum and temp-file behavior; it is safer than the older body.cancel-after-arrayBuffer approach.
