## Summary

Keep open for normal maintainer review: the PR addresses a source-reproducible ClawHub stalled-body hang with a narrow reader-owned timeout and I found no blocking correctness or security findings.

## What This Changes

The PR adds a timed reader-owned ClawHub archive body read for skill, package, and ClawPack downloads, passes idle-timeout errors to stream cancellation, adds focused stalled-body tests, aligns one Mantis evidence test, and updates the changelog.

## Best Possible Solution

Land this PR after maintainer review and CI, then close https://github.com/openclaw/openclaw/issues/52073 and supersede https://github.com/openclaw/openclaw/pull/80006.

## Reproduction Assessment

Yes, at source level: current main clears the fetch timeout once headers arrive and then awaits direct arrayBuffer reads for the affected archive paths. A Response with headers plus a body stream that never closes follows that exact hang path.

## Solution Assessment

Yes. Owning the body reader through the shared readResponseWithLimit helper is the narrow maintainable fix because it can time out and cancel the active read; the older https://github.com/openclaw/openclaw/pull/80006 approach is superseded by this reader-owned cancellation path.
