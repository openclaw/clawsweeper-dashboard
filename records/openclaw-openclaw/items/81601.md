## Summary

Keep open for normal maintainer review: this is a focused bug-fix PR for a source-reproducible ClawHub archive body-stall gap, with sufficient terminal proof, green checks, and no blocking correctness or security findings found.

## What This Changes

The PR routes ClawHub skill, package, and ClawPack archive downloads through a reader-owned body-read timeout/cancel helper, adds stalled-body regression tests, and updates the changelog.

## Best Possible Solution

Land this reader-owned ClawHub archive body timeout fix after maintainer review, then resolve https://github.com/openclaw/openclaw/issues/52073 and supersede https://github.com/openclaw/openclaw/pull/80006 in favor of this implementation.

## Reproduction Assessment

Yes, at source level: current main clears the fetch timeout when fetch returns a Response, then directly awaits response.arrayBuffer() for ClawPack, package, and skill archive bodies. A headers-then-stalled body can therefore remain pending; the PR body supplies after-fix terminal proof using real ReadableStream bodies in the download functions.

## Solution Assessment

Yes, this is the narrow maintainable fix: owning body.getReader() lets the code time out and cancel the active read while preserving digest and temp-file behavior. It is safer than the older approach in https://github.com/openclaw/openclaw/pull/80006, which attempted to cancel the response body while arrayBuffer() owned the stream.
