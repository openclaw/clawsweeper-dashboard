## Summary

Keep open for normal PR review and landing: current main still has the stalled-body gap, this patch fixes the implicated ClawHub archive paths with focused coverage and sufficient terminal proof, and I found no blocking functional or security findings.

## What This Changes

The PR replaces direct ClawHub archive `arrayBuffer()` reads with an owned `ReadableStream` reader timeout/cancel helper, adds stalled-body regression tests, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Land this reader-owned ClawHub archive body timeout fix after maintainer review, then resolve https://github.com/openclaw/openclaw/issues/52073 and supersede https://github.com/openclaw/openclaw/pull/80006 in favor of this implementation.

## Reproduction Assessment

Yes, at source level: current main clears the fetch timeout before direct archive `arrayBuffer()` calls, so a headers-then-stalled body can remain pending. The PR body also supplies after-fix terminal output showing timeout and cancellation for all three affected download paths.

## Solution Assessment

Yes, the reader-owned `getReader()` loop is the narrow maintainable fix because it cancels the active body read and preserves existing digest and temp-file behavior. It is safer than the older overlapping approach in https://github.com/openclaw/openclaw/pull/80006, which tried to cancel the response body while `arrayBuffer()` owned the stream.
