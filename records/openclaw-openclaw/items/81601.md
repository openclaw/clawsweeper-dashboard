## What This Changes

The PR routes ClawHub skill, package, and ClawPack archive downloads through the shared response-body idle-timeout helper, passes timeout errors to reader cancellation, adds stalled-body tests, and updates the changelog.

## Best Possible Solution

Land this focused reader-owned ClawHub archive body timeout fix after required CI and maintainer review, then resolve https://github.com/openclaw/openclaw/issues/52073 and supersede https://github.com/openclaw/openclaw/pull/80006.

## Reproduction Assessment

Yes. At source level, current main clears the fetch timeout as soon as headers arrive and then awaits direct `arrayBuffer()` reads for ClawPack, package archive, and skill archive bodies, so a headers-then-stalled body can remain pending.

## Solution Assessment

Yes. The latest PR head uses the existing reader-owned `readResponseWithLimit` helper and passes the timeout error to active reader cancellation, which is narrower and safer than cancelling a locked `Response.body` after `arrayBuffer()` starts.
