- **[P2] Regenerate channel config metadata:** `src/config/zod-schema.providers-core.ts:344`
  - body: Adding `rateLimit` to the Telegram Zod schema is not enough because OpenClaw also feeds generated bundled channel metadata into config schema and validation. - **[P2] Bound stale sender buckets:** `extensions/telegram/src/rate-limiter.ts:128`
  - body: `TelegramRateLimiter` keeps a process-lifetime `buckets` Map and only prunes the event list for the sender being checked. A rotating-sender flood can therefore grow this Map without returning to old keys, recreating a memory-pressure shape in the new DoS hardening path; add a max-tracked-key or global prune strategy before merge.
  - confidence: 0.8
- **[P2] Implement or remove non-silent drop policies:** `extensions/telegram/src/bot-handlers.runtime.ts:2690-2694`
  - body: The schema exposes `errorReply` and `summary`, but denied events only log and return here. Operators choosing either mode would still get silent drops, so implement those behaviors with tests or keep this PR's public config to silent-only.
  - confidence: 0.9
