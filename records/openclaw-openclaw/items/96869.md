## Summary

Keep open: current main still has the visible/native-reasoning stream split described in the report, and https://github.com/openclaw/openclaw/pull/96870 is an open candidate fix rather than a merged implementation.

## What This Changes

The issue reports silent truncation of visible assistant text when an OpenAI-compatible reasoning stream splits an inline code span between `content` and `reasoning_content` deltas.

## Best Possible Solution

Land a proof-backed fix that routes visible inline-code continuations through one canonical partitioner state in both OpenAI-compatible streaming paths while preserving private reasoning boundaries.

## Reproduction Assessment

Yes. The issue includes a deterministic source-level repro, and current main still shows native reasoning deltas bypassing the visible inline-code partitioner in the implicated stream paths; I did not run a live provider repro in this read-only review.

## Solution Assessment

Unclear as a final implementation. Recovering the visible continuation is the right end state, but maintainers still need to confirm the privacy-preserving routing rule and review the linked PR's same-chunk coverage before merge.
