## Summary

Keep open: current main and v2026.6.10 still separate native reasoning deltas from the inline-code-aware visible partitioner, and the open linked PR is a candidate fix rather than a merged resolution.

## What This Changes

The issue reports silent visible message loss when OpenAI-compatible reasoning streams split an inline code span across visible `content` and native `reasoning_content` deltas.

## Best Possible Solution

Review and refine https://github.com/openclaw/openclaw/pull/96870 into one shared streaming rule that covers same-chunk and split-chunk inline-code boundaries in both OpenAI-compatible paths, then land it with focused regression and real provider proof.

## Reproduction Assessment

Yes, at source level: current main still sends native reasoning deltas through thinking appenders while the visible inline-code state only sees `content`, and the issue includes a deterministic TS repro. I did not run a local or live provider repro because this cleanup review was read-only.

## Solution Assessment

Unclear as a final implementation: preserving the visible continuation is the right end state, but maintainers should confirm the privacy-preserving rule and require same-chunk plus sibling-path coverage before merge.
