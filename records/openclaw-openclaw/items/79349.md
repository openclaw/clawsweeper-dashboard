## Summary

Keep open: current main still has a source-reproducible Feishu plugin delivery gap where withdrawn/not-found thread replies throw instead of falling back, while the docs advertise Feishu inline and thread reply support.

## What This Changes

Feishu group-chat quoted replies can generate an assistant response but lose delivery when the Feishu thread reply API returns a withdrawn/not-found target error.

## Best Possible Solution

Keep the Feishu plugin behavior thread-aware, but make withdrawn/not-found reply-target failures deliver visibly for normal group quoted replies while preserving safety for native topic-only contexts.

## Reproduction Assessment

Yes, from source: the current unit test at `extensions/feishu/src/send.reply-fallback.test.ts:214` exercises Feishu code `230011` with `replyInThread: true` and proves no top-level send fallback occurs. I did not establish a live Feishu failure path in this read-only review.

## Solution Assessment

The unconditional fallback proposed in the issue is close but should be narrowed. The safest maintainable fix is a Feishu-local fallback policy that allows top-level group delivery only when the failed thread reply came from a normal group quote/reply path, while keeping native topic safety intact.
