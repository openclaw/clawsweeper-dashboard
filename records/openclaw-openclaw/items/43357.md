## Summary

Keep open. Current main still has the ASCII-only Discord outbound mention parser, so the Unicode/CJK rewrite work is not obsolete, but this PR is not ready to merge because the CJK URL guard still misses Markdown link destinations and the branch also carries Yuanbao catalog edits that current main has already moved past.

## What This Changes

The PR updates Discord mention rewriting and tests to support Unicode/CJK handles, while also carrying Yuanbao external channel catalog pin and contract expectation changes.

## Best Possible Solution

Land a Discord-only parser and regression-test update after Markdown link URL destinations are guarded, then coordinate explicitly with #59818 for the broader cross-account directory-cache behavior. The Yuanbao catalog pin should be removed from this PR or rebased away because current main already owns that state.

## Reproduction Assessment

Yes. Current main can be checked by seeding the Discord directory cache with `张三` and calling `rewriteDiscordKnownMentions('你好@张三，收到请回复。', { accountId: 'default' })`; the source regex shows it will not match. The remaining PR bug is reproducible by applying the PR-head URL-token logic to `[资料](https://x.com/你好@张三)`, where the URL guard fails before the CJK rewrite pass.

## Solution Assessment

No, not as-is. The narrow Discord parser direction is maintainable, but the PR should skip Markdown link URL destinations and rebase away the stale Yuanbao catalog overlap before it is a safe merge candidate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Skip Markdown link URLs in the CJK pass:** `extensions/discord/src/mentions.ts:104-108`
  - body: The URL guard only checks the whitespace-delimited token after stripping leading wrapper characters. For `[资料](https://x.com/你好@张三)`, that leaves `资料](https://...)`, so the URL-scheme check fails and the CJK pass rewrites the link destination into `https://x.com/你好<@...>`. Detect Markdown link URL destinations, or skip URL-looking substrings within the token, before applying the CJK rewrite.
  - confidence: 0.9
