## Summary

Keep open: current main still has the ASCII-only Discord mention parser, so the PR is addressing a real gap, but the PR head is dirty against main and still has merge-blocking issues around mentionAliases, Markdown-link URL destinations, and changelog coverage.

## What This Changes

The PR expands Discord outbound mention rewriting to Unicode/CJK handles, adds email/URL/reserved-mention regression tests, and carries Yuanbao catalog pin changes that current main already has.

## Best Possible Solution

Land one canonical Discord fix that supports Unicode/CJK outbound mention rewriting while preserving mentionAliases, avoiding URL/Markdown-link/code-span/reserved-mention regressions, adding regression tests, and carrying the required changelog entry.

## Reproduction Assessment

Yes, from source and a focused regex check. Current main's ASCII-only candidate pattern returns no match for `你好@张三`, so cached Unicode handles cannot reach the directory lookup path.

## Solution Assessment

No. The narrow parser direction is maintainable, but this PR head needs a rebase, alias-contract preservation, Markdown-link destination guarding, changelog coverage, and coordination with #59818 before it is the best merge path.
