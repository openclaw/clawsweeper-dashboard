## Summary

Keep open. The PR addresses a real Discord outbound mention gap because current main still only rewrites ASCII handles, but the proposed URL guard still lets the new CJK pass rewrite inside Markdown link destinations. The branch also still carries stale Yuanbao catalog changes that current main already has, and #59818 is an active broader PR for the same CJK symptom plus cross-account cache behavior.

## What This Changes

The PR updates Discord mention rewriting and tests to support Unicode/CJK handles with email and URL safeguards, and also includes Yuanbao external channel catalog pin and contract expectation changes.

## Best Possible Solution

Land a Discord-only parser and regression-test update after Markdown link URL destinations are guarded, then coordinate with #59818 for the broader cross-account directory-cache behavior. Rebase away the current-main Yuanbao catalog overlap and add the required user-facing changelog entry before merge.

## Reproduction Assessment

Yes. Current main can be checked by seeding the Discord directory cache with `张三` and calling `rewriteDiscordKnownMentions('你好@张三，收到请回复。', { accountId: 'default' })`; the source regex shows it will not match. The remaining PR bug is source-reproducible with `[资料](https://x.com/你好@张三)`, where the token guard does not recognize the Markdown link destination as a URL.

## Solution Assessment

No, not as-is. The narrow Discord parser direction is maintainable, but the PR should skip Markdown link URL destinations, rebase away stale catalog overlap, and coordinate with #59818 before it is the best merge path.
