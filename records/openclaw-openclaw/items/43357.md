## Summary

Keep open. The underlying Discord Unicode/CJK mention bug is valid on current main, but this PR head is dirty against main, would lose the current mentionAliases contract unless rebased carefully, still misses Markdown-link URL destinations in the CJK pass, and overlaps open #59818.

## What This Changes

The PR expands Discord outbound mention parsing to Unicode/CJK handles, adds email/URL/reserved/code-span regressions, and carries Yuanbao catalog pin updates that current main already contains.

## Best Possible Solution

Land one canonical Discord fix, coordinated with #59818, that supports Unicode/CJK outbound mention rewriting while preserving mentionAliases, avoiding URL/Markdown-link/code-span/reserved-mention regressions, adding focused tests, and including the required changelog entry.

## Reproduction Assessment

Yes. Current main's ASCII-only mention regex does not match `你好@张三`, so a cached Unicode handle cannot reach the directory lookup path; a focused regex check confirmed zero matches.

## Solution Assessment

No. The parser direction is reasonable, but this PR is not the best merge path until it is rebased, preserves mentionAliases, guards Markdown link destinations, adds changelog coverage, and is coordinated with #59818.
