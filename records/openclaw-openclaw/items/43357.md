## Summary

Keep open: current main still has the ASCII-only Discord mention candidate path, so the Unicode/CJK rewrite bug is real, but this PR needs a current-main rebase, correctness fixes, and real Discord behavior proof before merge.

## What This Changes

The PR changes Discord mention rewriting to match Unicode/CJK handles and adds mention parsing regressions, while also modifying official Yuanbao external channel catalog pins.

## Best Possible Solution

Rebase a narrow Discord-only patch that preserves current alias support, applies URL-aware Unicode mention parsing consistently, removes Yuanbao catalog changes, and adds focused regressions plus redacted live/logged Discord proof.

## Reproduction Assessment

Yes. A focused source reproduction is to cache a CJK handle and call `rewriteDiscordKnownMentions("@架构师", { accountId: "default" })`; current main's ASCII regex never matches that handle, though I did not run tests in this read-only review.

## Solution Assessment

No. The Unicode parser direction is appropriate, but this PR is not the best current-main solution because it drops alias support, guards only one rewrite pass against URL corruption, and carries unrelated stale Yuanbao catalog pins.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve configured mention aliases:** `extensions/discord/src/mentions.ts:65-68`
  - body: Current main resolves configured `mentionAliases` before directory-cache lookup, and Discord send/webhook paths pass those aliases into this function. The PR's new `rewriteCandidate` only calls `resolveDiscordDirectoryUserId`, so alias-only mentions such as configured `@OpsLead` would stop rewriting after a rebase that keeps this helper. Preserve the alias lookup and its regression coverage.
  - confidence: 0.88
- **[P2] Apply URL checks before delimiter rewrites:** `extensions/discord/src/mentions.ts:74-82`
  - body: The URL guard only runs during the second CJK-adjacent pass. The first delimiter pass still treats punctuation inside URL tokens as a valid mention boundary, so a cached handle in `https://x.com/path(@张三)` is rewritten and the link is corrupted. Skip URL tokens before both passes, or route both passes through the same URL-context check.
  - confidence: 0.86
- **[P2] Remove the stale Yuanbao catalog pin:** `scripts/lib/official-external-channel-catalog.json:45-47`
  - body: This Discord PR also changes the official Yuanbao install source to `openclaw-plugin-yuanbao@2.11.0`. Current main already pins `2.13.0` with a different integrity hash, so carrying this through conflict resolution would downgrade an unrelated external channel install source. Drop the catalog/test changes from this PR.
  - confidence: 0.9
