## Summary

Keep open. Current main still has the ASCII-only Discord outbound mention matcher, but this PR head is dirty against main, overlaps the broader open #59818, and still needs mechanical fixes before it is mergeable.

## What This Changes

The PR broadens Discord outbound mention rewriting and tests for Unicode/CJK handles, email safety, and URL safety, while also carrying Yuanbao catalog pin and contract updates.

## Best Possible Solution

Land one canonical Discord fix that supports Unicode/CJK outbound mention rewriting while preserving `mentionAliases`, avoiding URL/Markdown-link/code-span/reserved-mention regressions, adding regression tests, and carrying the required changelog entry.

## Reproduction Assessment

Yes. Current main source shows the outbound mention regex is ASCII-only, so a cached `张三` handle in `你好@张三` will not match; the remaining PR blockers are source-visible on the PR head.

## Solution Assessment

No. The narrow parser direction is maintainable, but this head must preserve the current alias contract, skip Markdown link destinations, add the changelog entry, and be reconciled with #59818 before it is the best merge path.
