## Summary

Keep open: current main still has ASCII-only Discord mention candidate parsing, so the Unicode/CJK mention bug is real, but this PR is not ready to merge because it mixes an unrelated Yuanbao catalog downgrade, would lose current alias behavior if applied as-is, and lacks real Discord proof. The overlapping work in https://github.com/openclaw/openclaw/pull/59818 is useful context but is still open, so it is not a shipped superseding fix.

## What This Changes

The branch expands Discord outbound mention parsing and tests for Unicode/CJK handles, while also changing Yuanbao external catalog pins and matching catalog tests.

## Best Possible Solution

Land a Discord-only Unicode/CJK mention rewrite that preserves alias, email, URL, code-span, reserved-mention, and account-scope safeguards while leaving Yuanbao catalog pins unchanged.

## Reproduction Assessment

Yes, at source level: current main's Discord mention regex only matches ASCII handles, so cached CJK handles such as `@张三` are not selected for rewriting. I did not run live Discord proof in this read-only review.

## Solution Assessment

No. Unicode-aware parsing is the right direction, but this branch needs to preserve current alias behavior, remove unrelated catalog metadata changes, and provide real Discord proof before it is a maintainable fix.
