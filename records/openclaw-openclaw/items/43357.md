## Summary

Keep open. Current main still has the ASCII-only Discord outbound mention parser, so the reported Unicode/CJK gap is real, but this PR head is not merge-ready because it is behind current main's `mentionAliases` contract, still rewrites CJK handles inside Markdown link destinations, lacks the required user-facing changelog entry, and overlaps the broader open #59818 path.

## What This Changes

The PR updates Discord outbound mention parsing and tests to support Unicode/CJK handles, while also carrying a Yuanbao catalog npm pin/integrity contract change that current main already has.

## Best Possible Solution

Land one canonical Discord fix that supports Unicode/CJK outbound mention rewriting while preserving configured aliases, reserved/code-span/email/URL and Markdown-link safeguards, current tests, and the required changelog entry.

## Reproduction Assessment

Yes. Current main can be reproduced from source by caching `张三` and calling `rewriteDiscordKnownMentions('你好@张三，收到请回复。', { accountId: 'default' })`; the ASCII-only regex on current main cannot match that handle.

## Solution Assessment

No. The narrow Discord parser direction is maintainable, but this head must carry current-main `mentionAliases`, skip Markdown link destinations, add a changelog entry, and be coordinated with #59818 before it is the best merge path.
