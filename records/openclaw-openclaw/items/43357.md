## Summary

Keep open: the Discord Unicode mention gap is still source-reproducible on current main, but this PR is not mergeable as-is because its head drops current `mentionAliases` behavior and includes unrelated Yuanbao catalog metadata changes. It also still lacks after-fix real Discord behavior proof.

## What This Changes

The PR updates Discord mention parsing and tests for Unicode/CJK handles, email safety, and URL guards, while also changing Yuanbao external channel catalog pins and related tests.

## Best Possible Solution

Land a Discord-only Unicode mention parser that preserves `mentionAliases`, email, URL, code-span, reserved-mention, and account-scope safeguards while leaving official external catalog pins unchanged.

## Reproduction Assessment

Yes, at source level: current main's Discord mention regex only matches ASCII handles, so cached CJK handles such as `@张三` are not selected for rewriting. I did not run live Discord proof in this read-only review.

## Solution Assessment

No. Unicode-aware parsing is the right direction, but this branch needs to preserve current alias behavior, remove unrelated catalog metadata changes, and provide real Discord proof before it is a maintainable fix.
