## Summary

Keep open: current main still has a source-visible Matrix configure dependency-repair path that resolves a root from bundled runtime location and runs a local package-manager install there. The related Matrix runtime-dependency report and closed unmerged PR address missing declarations, not this unsafe configure-time cwd/installer behavior.

## What This Changes

The issue asks OpenClaw to stop `openclaw configure` for Matrix from running npm in the user's global prefix/node_modules root and deleting unrelated global packages.

## Best Possible Solution

Make Matrix configure/install repair refuse any package-manager run outside a verified OpenClaw or Matrix package root, then cover the packaged npm user-prefix layout with regression proof.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: `openclaw configure` reaches `ensureMatrixSdkInstalled`, which can run local npm with a root derived from bundled `import.meta.url`. I did not execute the full repro because it is explicitly destructive to a package tree.

## Solution Assessment

No, the current implementation is not the best way to solve it because package repair must not run from a global prefix/node_modules root. The narrow fix is to resolve or require a safe package/plugin root and add packaged-prefix regression coverage, keeping the separate Matrix root-dependency question isolated.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80401. Current Matrix configure calls `ensureMatrixSdkInstalled`; in packaged npm installs the root derived from bundled `import.meta.url` can be the global prefix/node_modules root, then local `npm install --omit=dev --silent` runs there. Make this path non-destructive: never run npm/pnpm when the resolved cwd lacks a verified OpenClaw or Matrix package root or is a global node_modules/prefix root; either resolve a safe package/plugin root or fail with actionable `openclaw doctor --fix` / plugin install guidance. Add regression coverage that simulates the packaged dist/global npm prefix layout and verifies command argv/cwd cannot target the prefix root. Keep https://github.com/openclaw/openclaw/issues/77896 as separate root-dependency packaging work unless a maintainer explicitly chooses to combine it. Add a changelog fix entry and avoid changing unrelated Matrix runtime behavior.
