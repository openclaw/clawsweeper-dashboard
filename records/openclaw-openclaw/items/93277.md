## What This Changes

The PR marks Discord cron notification payloads with OpenClaw reply-context metadata, records sent Discord message IDs, and preserves marked bot-authored quote bodies while keeping ordinary bot self-replies suppressed.

## Best Possible Solution

Persist marked notification receipt policy in a bounded Discord/plugin-owned durable state path, then require real Discord or gateway proof that marked notification replies preserve context while ordinary bot self-replies stay suppressed.

## Solution Assessment

No. The structured marker is a reasonable direction, but module-local memory is not the right lifetime for a user reply window; the maintainable fix needs bounded durable state plus live Discord or gateway proof.
