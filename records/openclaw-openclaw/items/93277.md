## Summary

Keep open. The PR is a focused Discord reply-context bugfix, but the current branch stores marked Discord message IDs only in process memory and still lacks real Discord or gateway proof.

## What This Changes

The branch marks Discord cron notification payloads, records delivered Discord message IDs for marked payloads, and preserves only those bot-authored quote bodies when users reply.

## Best Possible Solution

Persist marked Discord platform message IDs in a bounded Discord/plugin-owned durable store with cleanup, keep ordinary self-reply suppression, and add redacted real Discord or gateway proof.

## Reproduction Assessment

Yes, source-level. Current main suppresses self-authored Discord quote bodies, and the PR's remaining restart/process-boundary failure follows directly from the module-local Set allowlist.

## Solution Assessment

No. The structured marker is a reasonable direction, but module-local memory is not the right lifetime for a user reply window; the maintainable fix needs bounded durable state plus real Discord or gateway proof.
