## Summary

## Best Possible Solution

## Reproduction Assessment

Yes, source-level: current main marks Discord bot-authored reply targets as `isSelf`, the shared finalizer defaults `suppressSelfQuoteBody` to true, and the Discord regression test expects `ReplyToBody` to stay undefined. I did not run a live Discord gateway replay.

## Solution Assessment

No. The one-line opt-out is broader than the requested safe context case because it reverses current suppression for all Discord bot-authored reply targets; the maintainable path is a narrow, approved opt-in or preserving the existing default.
