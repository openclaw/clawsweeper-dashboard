## Summary

Keep open. The PR targets a real Discord reply-context gap, but the current patch is not merge-ready because the marker that makes the context exception work is process-local and the supplied proof is only a local source harness with fake IDs.

## What This Changes

The branch marks Discord cron notification payloads, records marked Discord send result IDs in process memory, and preserves only those marked self-authored quote bodies when users reply.

## Best Possible Solution

Persist marked Discord platform message IDs in bounded Discord/plugin-owned durable state with cleanup, keep ordinary self-reply suppression, and add redacted real Discord or gateway proof before merge.

## Reproduction Assessment

Yes, source-level. Current main marks referenced Discord bot messages as self-authored and the shared inbound finalizer strips their quote body unless a caller opts out.

## Solution Assessment

No. The structured marker is a reasonable direction, but a module-local Set is not the right lifetime for a user reply window; the maintainable fix needs bounded durable state or explicit maintainer acceptance plus real Discord proof.
