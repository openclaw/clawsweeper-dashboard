## Summary

Keep open: the branch addresses a real Discord reply-context gap, but the marker is stored in process-local memory, so a restart or different handler process loses the exception before the user replies; the supplied proof is also local harness output rather than real Discord or gateway evidence.

## What This Changes

The PR marks Discord cron notification payloads with reply-context metadata, records delivered Discord message IDs in Discord outbound delivery, and preserves only marked bot-authored Discord quote bodies when users reply.

## Best Possible Solution

Persist marked Discord platform message IDs in a bounded Discord-owned durable state path with cleanup, keep ordinary self-reply suppression, and add redacted real Discord or gateway proof.

## Reproduction Assessment

Yes, source-level: current main marks referenced Discord bot messages as self-authored and the shared inbound finalizer suppresses self-authored quote bodies unless a caller opts out. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

No. The structured marker is a reasonable direction, but module-local memory is the wrong lifetime for a user reply window; the maintainable fix needs bounded durable state plus real Discord or gateway proof.
