## Summary

Keep open. The PR addresses a real Discord reply-context gap, but it is not merge-ready because the preservation marker is process-local and the supplied proof is only a local source-harness run with fake Discord IDs.

## What This Changes

The PR marks Discord cron notification payloads, records delivered Discord message IDs for marked payloads, and preserves only those bot-authored quote bodies when users reply.

## Best Possible Solution

Persist marked Discord platform message IDs in a bounded Discord/plugin-owned durable state path with cleanup, keep ordinary self-reply suppression, and add redacted real Discord or gateway proof.

## Reproduction Assessment

Yes, source-level. Current main marks referenced Discord bot messages as self-authored and the shared inbound finalizer strips their quote body unless a caller opts out.

## Solution Assessment

No. The structured marker is a reasonable direction, but module-local memory is not the right lifetime for a user reply window; the maintainable fix needs bounded durable state plus real Discord or gateway proof.
