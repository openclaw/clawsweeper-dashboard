## Summary

Keep open: current main and the latest release still use the broad first-assistant transcript check, and there is still no supported retry-prompt hook or override. The linked open PR covers only the guard half and is not a complete replacement for this issue.

## What This Changes

Fix attempt-execution so session history means assistant-after-latest-user, and add a supported fallback retry continuation-prompt hook or override.

## Best Possible Solution

Land or replace the bounded assistant-after-latest-user guard fix, then make an explicit maintainer decision on whether retry-prompt tuning belongs in a plugin hook, lifecycle event, config override, or no new core surface.
