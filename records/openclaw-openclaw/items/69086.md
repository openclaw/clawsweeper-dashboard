## Summary

Keep open: current main and the latest release still use the broad assistant-anywhere session guard, and there is still no supported retry-prompt override; the open linked PR covers only the guard fix while the hook remains a product/API decision.

## What This Changes

The issue requests narrowing attempt-execution session-history detection to assistant-after-latest-user and adding a supported fallback retry prompt override hook.

## Best Possible Solution

Land or replace the focused guard fix, then decide the retry-prompt override as an explicit fallback prompt extension point with narrow tests and documented ownership.

## Solution Assessment

Unclear for the full issue. The guard fix shape is narrow and well supported, but the retry-prompt hook needs a maintainer decision on whether the permanent surface should be config, plugin hook, lifecycle event, or a smaller built-in prompt change.
