## Summary

Keep this issue open. Current main still has the source-provable session-history guard bug, the retry-prompt override request is not implemented, and an open linked PR is the active implementation path for the guard fix, so closing this before that PR is resolved would be premature.

## What This Changes

The issue requests that fallback retry session detection only count assistant messages after the latest user turn and that OpenClaw expose a supported retry continuation prompt override.

## Best Possible Solution

Review and resolve the linked guard-fix PR, then either split or explicitly decide the retry-prompt override as a small default-preserving config surface.

## Reproduction Assessment

Yes. Source inspection shows a user -> assistant -> fresh user JSONL transcript returns `true` at the earlier assistant record before the latest user turn can reset the state; I did not execute tests because this review is read-only.

## Solution Assessment

Unclear for the full issue. The linked PR is the right implementation path for the guard bug after review, but the retry-prompt hook should be a narrow maintainer-approved config surface or split follow-up rather than bundled into an automatic cleanup close.
