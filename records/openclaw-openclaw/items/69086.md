## What This Changes

The issue asks to scope attempt-execution fallback retry history detection to assistant-after-latest-user and add a supported override for the retry continuation prompt.

## Best Possible Solution

Land a focused core fix that tracks assistant-after-latest-user in the bounded JSONL scan, adds the missing regression tests, and exposes a documented default-preserving retry marker override.

Yes. A JSONL transcript shaped user -> assistant -> fresh user reproduces the guard bug by returning `true` at the earlier assistant record before the later user turn is considered.

## Solution Assessment

Yes. The narrowest maintainable repair is to keep the existing bounded parser, reset state on newer user records, and add a config-backed retry marker override rather than a broader hook unless maintainers request one.

## Repair Work Prompt

Fix openclaw/openclaw#69086 with a focused replacement PR. In `src/agents/command/attempt-execution.helpers.ts`, change the session transcript scan so `sessionFileHasContent()` returns true only when an assistant message appears after the latest user message in the inspected JSONL records; preserve the existing max-record bound, symlink rejection, invalid JSON handling, and assistant-after-user true case. Add regression tests in `src/agents/command/attempt-execution.test.ts` for user -> assistant -> fresh user returning false and latest user -> assistant returning true. Add a small default-preserving operator override for the fallback retry marker/prompt, preferably under agent defaults config, thread it through `runAgentAttempt()`/`resolveFallbackRetryPrompt()`, document it in the relevant config help and `docs/concepts/model-failover.md`, and add a user-facing changelog entry. Do not change broader model fallback policy, provider cooldown classification, plugin APIs, or runtime selection semantics unless required by the narrow override. Use #69109 only as source context; it was closed unmerged. Note #65760 already fixed body preservation, so keep prepend semantics by default.
