Full review comments:

- **[P2] Propagate fallback reasons through debounce batches:** `extensions/telegram/src/bot-handlers.runtime.ts:239`
  - body: The new `mediaUnavailableText` is only forwarded in the single-entry debounce path. A caption/text document whose download failed has `allMedia.length === 0`, so if it coalesces with another debounced entry, the synthetic multi-entry message is sent without the unavailable reason and the user still gets an ambiguous fallback.
  - confidence: 0.9
- **[P2] Carry document fallback reasons through media groups:** `extensions/telegram/src/bot-handlers.runtime.ts:406`
  - body: `resolveMedia` can now return a document placeholder plus `unavailableText` without a path, but media-group processing discards non-path results and calls `processMessage` with no options. Failed grouped documents therefore lose the new reason instead of preserving it in the final inbound body.
  - confidence: 0.86
- **[P2] Rebase optional media results through reply-chain callers:** `extensions/telegram/src/bot/delivery.resolve-media.ts:300`
  - body: This PR makes `resolveMedia` return values where `path` is optional. Current main has a reply-chain media caller that still treats any non-null result as path-bearing, so the rebased branch needs to guard on `media?.path` there too or it can pass undefined paths and fail type checking.
  - confidence: 0.82
