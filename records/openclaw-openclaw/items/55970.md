## Summary

Keep open: the PR targets a real Telegram fallback gap that is still present on current main and the latest release, but the current patch still drops the new reason through multi-entry debounce and has only test/mock verification.

## What This Changes

The PR changes Telegram inbound media resolution and message context rendering so failed document downloads can append a file-too-large or download-failed reason, with focused resolver/body tests.

## Best Possible Solution

Land a revised Telegram plugin fix that carries the document-unavailable reason through resolver, body rendering, single-entry and multi-entry debounce paths, with focused tests and real diagnostic proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: make Telegram `getFile` fail for a document with `file is too big` or exhausted retries; current main returns no media and can render only the document placeholder without a reason.

## Solution Assessment

No, not as currently proposed. The resolver/body strategy is the right narrow fix, but the patch must preserve the reason through multi-entry debounce and provide real behavior proof before merge.
