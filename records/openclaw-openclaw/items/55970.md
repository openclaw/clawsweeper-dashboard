## Summary

Keep open: this is a plausible Telegram bug fix, but the patch still loses the new fallback reason in debounce bursts and media-group handling, and the external PR only has mock/local-test proof rather than real behavior proof.

## What This Changes

The PR changes Telegram inbound document media resolution and body rendering so failed document downloads can append an unavailable reason, with focused resolver and body-rendering tests.

## Best Possible Solution

Land a rebased Telegram plugin fix that carries document-unavailable reasons through resolver, body rendering, debounce, and media groups, backed by focused tests and redacted real Telegram proof.

## Reproduction Assessment

Yes, source-reproducible: on current main, a Telegram document whose getFile reports file-too-large or exhausts retries makes resolveMedia return null, and the body path has no unavailable-reason option. I did not run a live Telegram transfer in this read-only review.

## Solution Assessment

No, not as currently proposed. The resolver/body approach is the right narrow direction, but the patch must also preserve the reason through multi-entry debounce and media-group handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Propagate fallback reasons through debounce bursts:** `extensions/telegram/src/bot-handlers.runtime.ts:1092`
  - body: The new `mediaUnavailableText` is saved on queued debounce entries, but multi-entry flushes build synthetic message options without merging `entry.options`. A caption/text plus failed document has no media path and is debounce-eligible, so when it batches with another text update the final body loses the unavailable reason again.
  - confidence: 0.91
- **[P2] Forward media-group fallback reasons:** `extensions/telegram/src/bot-handlers.runtime.ts:406`
  - body: This guard now skips failed document results that only contain `unavailableText`, but `processMediaGroup` never forwards that reason into message-context options. A document in a Telegram media group can still reach body rendering without the explicit fallback reason this PR is meant to add.
  - confidence: 0.82
