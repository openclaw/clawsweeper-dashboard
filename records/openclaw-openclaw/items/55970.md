## Summary

Keep open. Current main still lacks document-unavailable reason plumbing, and this PR is useful but incomplete because multi-entry Telegram debounce can still drop the new reason and the required changelog entry is missing.

## What This Changes

The PR updates Telegram inbound document media resolution and body rendering to return explicit unavailable text for failed document downloads, threads that option through message context, and adds focused resolver/body tests.

## Best Possible Solution

Land a revised Telegram plugin fix that carries the document-unavailable reason through resolver, body rendering, single-entry and multi-entry debounce paths, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence seam reproduction: make Telegram `getFile` fail for a document with either file-too-large or exhausted retries, then build the inbound body; current main returns only the placeholder path without a reason.

## Solution Assessment

No, not as currently proposed. The resolver/body strategy is the right narrow direction, but the PR must preserve the unavailable reason through multi-entry debounce flushes and satisfy the changelog gate before it fully solves the issue.
