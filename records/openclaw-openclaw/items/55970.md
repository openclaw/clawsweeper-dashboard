## Summary

Keep open. The PR addresses a valid Telegram document fallback gap, but the current patch still drops the new unavailable reason through debounce bursts and media-group handling, and the external contributor has not provided after-fix real Telegram proof.

## What This Changes

The PR adds Telegram document download fallback text plumbing and focused tests so failed inbound document downloads keep `<media:document>` and append a file-too-large or download-failed reason.

## Best Possible Solution

Land a rebased Telegram plugin fix that carries document-unavailable reasons through resolver, body rendering, debounce, and media groups, backed by focused tests and redacted real Telegram proof.

## Reproduction Assessment

Yes, source-reproducible: on current main, a Telegram document whose getFile reports file-too-large or exhausts retries returns null from resolveMedia, leaving body rendering without an unavailable-reason option. I did not run a live Telegram transfer in this read-only review.

## Solution Assessment

No, not as currently proposed. The resolver/body approach is the right narrow direction, but the patch must also preserve the reason through multi-entry debounce and media-group handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Propagate fallback reasons through debounce bursts:** `extensions/telegram/src/bot-handlers.runtime.ts:1092`
  - body: The new `mediaUnavailableText` is stored on queued entries, but multi-entry flushes build synthetic message options without merging `entry.options`. A caption/text plus failed document has no media path and is debounce-eligible, so batching it with another update drops the unavailable reason this PR is meant to surface.
  - confidence: 0.91
- **[P2] Forward media-group fallback reasons:** `extensions/telegram/src/bot-handlers.runtime.ts:406`
  - body: After this guard skips failed document results that only carry `unavailableText`, `processMediaGroup` still calls `processMessage` with undefined options. A document inside a Telegram media group can therefore reach body rendering without the explicit fallback reason.
  - confidence: 0.82
