## Summary

Keep open. This PR targets a valid Telegram document fallback bug, but the current patch is incomplete because the new unavailable reason can still be dropped in debounce and media-group paths, and the PR still lacks after-fix real Telegram proof.

## What This Changes

This PR changes Telegram inbound media resolution and body rendering so failed document downloads keep `<media:document>` and append either a file-too-large or download-failed reason, with focused tests.

## Best Possible Solution

Land a rebased Telegram plugin fix that carries document-unavailable reasons from resolver through body rendering, debounce, and media-group handling, backed by focused tests and redacted real Telegram proof.

## Reproduction Assessment

Yes, source-reproducible: mock a Telegram document whose `ctx.getFile()` rejects with a file-too-big error or exhausts retries; current main returns null and no unavailable reason reaches body rendering. I did not run a live Telegram transfer in this read-only review.

## Solution Assessment

No, not as currently proposed. The resolver/body approach is the right narrow direction, but the patch must preserve the reason through debounce and media-group paths and include real Telegram proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Propagate fallback reasons through debounce bursts:** `extensions/telegram/src/bot-handlers.runtime.ts:1092`
  - body: The new `mediaUnavailableText` is stored on queued entries, but multi-entry flushes build synthetic options without merging `entry.options`. A caption/text plus failed document has no saved media and is debounce-eligible, so batching it with another update drops the unavailable reason this PR is meant to surface.
  - confidence: 0.91
- **[P2] Forward media-group fallback reasons:** `extensions/telegram/src/bot-handlers.runtime.ts:406`
  - body: This branch now ignores failed document results that only carry `unavailableText`, and the later media-group processing call still passes no options. A failed document in that path can therefore reach body rendering without the explicit fallback reason.
  - confidence: 0.82
