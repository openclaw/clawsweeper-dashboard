## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Propagate fallback reasons through debounce batches:** `extensions/telegram/src/bot-handlers.runtime.ts:269-273`
  - body: A failed document with text is debounce-eligible because the pathless fallback leaves `allMedia` empty. When another entry shares the debounce window, this multi-entry flush builds options without merging entry options, so `mediaUnavailableText` is dropped and the agent can still receive ambiguous placeholder/text without the unavailable reason.
  - confidence: 0.89
- **[P2] Carry unavailable reasons through media groups:** `extensions/telegram/src/bot-handlers.runtime.ts:406-417`
  - body: `resolveMedia` can now return a document fallback object with no path, but media-group processing only pushes `media?.path` results and then calls `processMessage` with no options. A failed document inside a grouped update therefore loses the new fallback reason instead of surfacing it in the final body.
  - confidence: 0.82
- **[P2] Rebase the optional media return across reply-chain callers:** `extensions/telegram/src/bot/delivery.resolve-media.ts:300-305`
  - body: The PR widens `resolveMedia` so a truthy result may not have `path`. Current main has a newer reply-chain caller that still builds `TelegramMediaRef` from any truthy media, so the rebase must guard on `media?.path` there as well or failed reply documents can create pathless media refs.
  - confidence: 0.78
