## Summary

Keep open. Current `main` already carries most of the PR's hardening around exact-name media trust, client-tool conflict detection, and plugin metadata preservation, but it does not implement the PR's stated streaming preflight contract of failing conflicting OpenResponses tool configs before any SSE body is emitted.

## Risks / Open Questions

- Closing this PR as implemented would discard a still-unlanded behavioral delta around streaming OpenResponses preflight semantics.
- Any eventual merge/rework will need careful rebase because `main` already contains overlapping hardening in `src/agents/*`, `src/gateway/*`, and `src/plugins/tools.ts`.
- The remaining gap is narrow but user-visible: stream clients may still receive initial SSE events before a tool-name conflict is surfaced.
