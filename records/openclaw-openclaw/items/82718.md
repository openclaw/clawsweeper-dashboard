## Summary

Keep this PR open: the useful Telegram docs and doctor diagnostic are still needed, but the current warning/docs do not spell out that account-local `groups` replace the root group map for that account once present, which can make a partial account-local map silently drop root-allowed groups for that account.

## What This Changes

The PR updates Telegram group-configuration docs and adds a non-mutating `openclaw doctor` warning with focused tests for multi-account configs that have root groups but missing account-local groups.

## Best Possible Solution

Keep the docs and doctor warning, but update the wording and tests to say that account-local Telegram groups replace the root group map for that account, so users must copy every allowed group or use `"*"` when adding account-scoped group options.

## Reproduction Assessment

## Solution Assessment

No, not as written. Docs and doctor are the right layer, but the current wording should include the account-local replacement/copy semantics before the diagnostic is merge-ready.
