## Summary

This PR should stay open rather than be cleaned up: it contains a focused Telegram docs and doctor diagnostic change with useful proof, but the new warning still needs maintainer resolution because current runtime treats root groups differently across generic group allowlisting and account-merged group settings.

## What This Changes

The PR updates Telegram group-configuration docs, removes retired Telegram thread-reply fields from a docs field list, and adds a non-mutating Telegram doctor warning plus tests for multi-account configs with root groups but missing account-local groups.

## Best Possible Solution

Make the warning precise about which account-local group settings do not inherit from root groups, or intentionally change runtime and tests so account-local groups are the canonical multi-account contract.

## Reproduction Assessment

Yes for the review concern: source inspection shows the warning can fire for omitted account-local groups while current runtime still falls back to root groups for generic group allowlisting. The contributor also supplied redacted real Telegram gateway logs for the reported operator failure.

## Solution Assessment

No, not as written; docs and doctor are a good diagnostic layer, but the warning should match the current split runtime contract or be paired with an intentional runtime change.
