## Summary

Keep open: this is a focused fix for the linked Windows Scheduled Task visible-console bug, but it is still draft, still changes a compatibility-sensitive Windows launch default to VBS/Windows Script Host, and the supplied proof does not demonstrate the native Windows no-console behavior.

## What This Changes

The PR changes Windows daemon Scheduled Task and Startup-folder fallback defaults from direct `gateway.cmd` launchers to generated `gateway.vbs` launchers, keeps explicit falsey opt-out behavior, and updates focused daemon tests around default VBS actions and Startup fallback cleanup.

## Best Possible Solution

Land the VBS default only after native Windows proof is added, task-launcher cleanup handles both variants, docs match the new launcher shape, and maintainers accept the WSH/VBS default tradeoff.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.6` leave the normal Gateway Scheduled Task on `gateway.cmd` when the hidden-launcher env is unset, and the linked issue gives Windows 11 reproduction steps. I did not run a live Windows desktop reproduction in this read-only review.

## Solution Assessment

Yes for the narrow code location: reusing the existing generated VBS launcher is the maintainable path for the visible console bug. It is not merge-ready until native Windows proof, task-launcher cleanup, docs alignment, and maintainer acceptance of the default change are settled.
