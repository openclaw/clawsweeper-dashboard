## Summary

Keep open: current main and the latest release still use the direct `gateway.cmd` path by default, so this PR remains the active fix for the linked Windows Scheduled Task bug. It is not merge-ready yet because the default switch still needs native Windows no-console proof, compatibility acceptance, and small opt-out/docs repairs.

## What This Changes

The PR changes Windows Scheduled Task and Startup-folder fallback launcher selection so unset `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER` uses the generated VBS launcher, with daemon tests updated for default and legacy cleanup paths.

## Best Possible Solution

Land the VBS-default scheduled-task fix only after preserving accepted falsey opt-outs, updating Windows launcher docs, and adding redacted native Windows proof or maintainer-recorded equivalent.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 leave the Gateway Scheduled Task on `gateway.cmd` when the hidden-launcher env is unset, and the linked issue provides Windows 11 reproduction steps. I did not reproduce the visible desktop window live.

## Solution Assessment

Yes on the fix shape: reusing the existing VBS hidden launcher is the narrowest maintainable repair for the reported scheduled-task console window. The branch still needs opt-out parsing, docs alignment, native Windows proof, and maintainer compatibility acceptance before merge.
