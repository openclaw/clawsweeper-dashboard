## What This Changes

The PR changes Windows daemon Scheduled Task and Startup-folder fallback launcher selection so unset `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER` uses generated VBS launchers by default, with install and fallback tests updated.

## Best Possible Solution

Land the VBS default only after the Windows docs match the new launcher behavior, native Windows no-console proof or maintainer acceptance is recorded, and the falsey opt-out path remains covered.

## Reproduction Assessment

## Solution Assessment

Yes for the core code shape: reusing the existing VBS launcher is the narrowest fix for the reported Scheduled Task console window. The PR is not fully merge-ready until docs, native proof, and compatibility acceptance are handled.
