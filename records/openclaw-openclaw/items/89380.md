## Summary

Keep open: this is a focused fix for the linked Windows Scheduled Task console-window bug, and current main plus v2026.6.6 still default the gateway task to `gateway.cmd`; the PR is not merge-ready until the native Windows proof gap and small docs/cleanup issues are handled.

## What This Changes

The PR changes Windows gateway Scheduled Task and Startup-folder fallback defaults to use the generated `gateway.vbs` hidden launcher unless `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER` is explicitly falsey, and updates daemon tests around those defaults and fallback cleanup.

## Best Possible Solution

Land the VBS default after native Windows proof or maintainer acceptance is recorded, Windows launcher docs are aligned, and cleanup covers both task and Startup-folder launcher variants.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 leave the gateway Scheduled Task on `gateway.cmd` when the hidden-launcher env is unset, and the linked issue gives Windows 11 reproduction steps. I did not reproduce the desktop window live.

## Solution Assessment

Yes, reusing the existing VBS launcher is the narrow fix for the reported launcher bug. It is not merge-ready until native proof, docs alignment, and task-launcher opt-out cleanup are handled or explicitly accepted by maintainers.
