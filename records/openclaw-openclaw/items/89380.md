## Summary

Keep open: this PR is a plausible fix for the Windows Scheduled Task console-window bug, but it still needs native Windows/no-console proof, compatibility acceptance for the VBS default, and small docs/cleanup follow-through before merge.

## What This Changes

The PR makes unset `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER` default Windows Scheduled Task and Startup-folder fallback installs to generated VBS launchers, with daemon tests updated for fresh installs, task updates, and legacy startup cleanup.

## Best Possible Solution

Land the VBS default after native Windows proof or maintainer acceptance is recorded, Windows launcher docs are aligned, and the falsey opt-out cleanup path is covered.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 leave the Gateway Scheduled Task on `gateway.cmd` when the hidden-launcher env is unset, and the linked issue gives Windows 11 reproduction steps. I did not reproduce the desktop window live.

## Solution Assessment

Yes, reusing the existing VBS launcher is the narrow code path for the reported bug. It is not merge-ready until native proof, docs alignment, and opt-out cleanup are handled or explicitly accepted by maintainers.
