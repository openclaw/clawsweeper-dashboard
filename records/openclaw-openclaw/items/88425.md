## Summary

Keep open: the PR addresses a real ACP/context-engine gap, but the patch is not merge-ready because it runs context-engine assembly through ACP’s text-only turn path without honoring host requirements or non-text assemble outputs, and real behavior proof is still missing.

## What This Changes

The PR routes ACP dispatch prompt text through the active context engine before invoking ACP turn execution while preserving transcript persistence on the original prompt text and adding regression tests.

## Best Possible Solution

ACP should either advertise and enforce a limited context-engine host contract that only supports text-projected context, or grow a richer ACP turn contract before enabling engines that require system prompt or projection semantics.

## Reproduction Assessment

Yes, source-level: current main sends ACP `turnPromptText` directly to `acpManager.runTurn`, while the context-engine contract exists for pre-prompt assembly. This read-only review did not run a live ACP/context-engine setup.

## Solution Assessment

No. The PR targets the right dispatch seam, but it is too narrow because it consumes only the last assembled user message and does not enforce context-engine host capability or system-prompt/projection semantics.
