## Summary

Keep this issue open because current main still has the reported fail-fast supplement aggregation, and an open linked PR already uses closing syntax for this exact bug. The right cleanup point is after that PR is reviewed and merged or otherwise superseded.

## What This Changes

Fix `memory_search` supplement aggregation so one failing corpus supplement does not discard results from other registered supplements.

## Best Possible Solution

Land or close the linked PR after review, with a regression test that registers one successful and one rejecting corpus supplement and verifies surviving results are still returned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, and call `memory_search` with `corpus="all"` or `corpus="wiki"`; current main will reject through `Promise.all` before returning the successful supplement result.

## Solution Assessment

Yes. The narrow maintainable fix is to isolate supplement search failures, e.g. `Promise.allSettled`, while preserving the existing flatten, sort, slice, and merge behavior; the open linked PR appears to target that exact boundary.
