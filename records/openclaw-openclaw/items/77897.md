## Summary

Keep this issue open because current main and the latest release still use fail-fast supplement aggregation, while two open closing PRs already target the same bug. The right next step is to review, land, or supersede one of those PRs rather than close the issue early.

## What This Changes

The issue reports that memory corpus supplement search uses fail-fast `Promise.all`, so one rejecting supplement can discard healthy supplement results.

## Best Possible Solution

Land or supersede one linked fix PR with per-supplement failure isolation, preserved diagnostics for failed supplements, and focused regression coverage for partial success.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="all"` or `corpus="wiki"`; current main awaits fail-fast `Promise.all` before returning healthy supplement results.

## Solution Assessment

Yes. Per-supplement isolation with `Promise.allSettled` or equivalent is the narrow maintainable fix, as long as the final patch preserves useful diagnostics for rejected or all-failed supplement searches.
