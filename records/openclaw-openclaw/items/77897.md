## What This Changes

The issue asks memory-core search to preserve results from healthy corpus supplements when another registered supplement rejects.

## Best Possible Solution

Review and land one narrow linked PR that preserves fulfilled sibling supplement results, keeps true supplement unavailability visible, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus=wiki` or `corpus=all` and current `Promise.all` rejects the whole supplement fan-out.

## Solution Assessment

Yes. Isolating per-supplement search failures in `searchMemoryCorpusSupplements` is the right owner boundary; the remaining choice is which linked implementation to land.
