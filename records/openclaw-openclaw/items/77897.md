## Summary

Keep this issue open because current main and the latest release still use fail-fast supplement aggregation, while two open closing PRs already target the same bug. The issue should wait for one of those PRs to be reviewed, landed, or superseded.

## What This Changes

The issue asks for `memory_search` corpus supplement failures to be isolated so one rejecting supplement does not discard results from healthy supplements.

## Best Possible Solution

Review and land or supersede one linked fix PR with regression coverage for partial supplement failure while preserving the intended diagnostic behavior when all supplements fail.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="all"` or `corpus="wiki"`; current main rejects through `Promise.all` before returning healthy supplement results.

## Solution Assessment

Yes. Per-supplement isolation with `Promise.allSettled` or equivalent is the narrow maintainable fix, with regression coverage for partial success and clear all-failed diagnostics.
