## Summary

Keep this issue open because current main still uses fail-fast supplement aggregation, and two open closing PRs already target the same bug. The issue should wait for one of those PRs to be reviewed, landed, or superseded rather than creating a duplicate repair lane.

## What This Changes

The issue requests per-supplement failure isolation so a rejected memory corpus supplement does not discard healthy `memory_search` results.

## Best Possible Solution

Review and land or supersede one linked fix PR with regression coverage for partial supplement failure while preserving clear behavior when all supplements fail.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="all"` or `corpus="wiki"`; current main rejects through `Promise.all` before returning healthy supplement results.

## Solution Assessment

Yes. Per-supplement isolation with `Promise.allSettled` or equivalent is the narrow maintainable fix, with regression coverage to preserve sorting, slicing, merging, and all-failed diagnostics.
