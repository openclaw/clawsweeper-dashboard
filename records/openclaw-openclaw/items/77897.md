## What This Changes

The issue requests changing memory corpus supplement aggregation so one rejecting supplement does not discard healthy supplement search results.

## Best Possible Solution

Review or consolidate the linked PRs and land one narrow fix that preserves fulfilled sibling supplement results while keeping true supplement unavailability visible and covered by regression tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="wiki"` or `corpus="all"`; current `Promise.all` rejects the whole supplement fan-out.
