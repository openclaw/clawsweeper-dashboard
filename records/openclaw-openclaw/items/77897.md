Keep this issue open because current main still uses fail-fast supplement aggregation, and live open fix PRs with closing references already target the same bug: https://github.com/openclaw/openclaw/pull/77899 and https://github.com/openclaw/openclaw/pull/78035.

## What This Changes

The issue requests isolating memory corpus supplement search failures so one rejecting supplement does not discard successful sibling results from `memory_search`.

## Best Possible Solution

Review and land or supersede one linked fix PR with regression coverage for partial supplement failure while preserving explicit behavior for all-failed supplements.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="all"` or `corpus="wiki"`; current main rejects through `Promise.all` before returning the healthy supplement result.

## Solution Assessment

Yes. The narrow maintainable fix is per-supplement isolation, such as `Promise.allSettled` or equivalent, while keeping the existing sort, slice, merge, and diagnostic behavior intact.
