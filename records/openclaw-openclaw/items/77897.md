## Summary

Keep this issue open because current main still uses fail-fast supplement aggregation, and two live open PRs already reference and target the same bug.

## What This Changes

Make memory corpus supplement search tolerate one supplement rejecting without discarding results from healthy supplements.

## Best Possible Solution

Review and land or supersede one linked fix PR with regression coverage for partial supplement failure while preserving existing sorting, merging, and outage diagnostics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="all"` or `corpus="wiki"`; current main rejects through `Promise.all` before returning healthy supplement results.

## Solution Assessment

Yes. The narrow maintainable fix is per-supplement isolation, such as `Promise.allSettled` or equivalent, while preserving the existing sort, slice, merge, and explicit all-failed diagnostics.
