## What This Changes

The issue requests that memory corpus supplement search preserve results from healthy supplements when another supplement's search rejects.

## Best Possible Solution

Review or consolidate the linked PRs and land one narrow fix that preserves fulfilled sibling supplement results while keeping true supplement unavailability visible and covered by regression tests.

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="wiki"` or `corpus="all"`; current `Promise.all` rejects the whole supplement fan-out.
