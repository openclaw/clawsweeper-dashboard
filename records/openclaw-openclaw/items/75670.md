## Summary

Keep open. Current main still has the reported split: Matrix routing and send helpers preserve mixed-case Matrix thread roots, but the shared inbound session recorder lowercases the full opaque session key before persistence and last-route updates.

## What This Changes

The issue asks OpenClaw to preserve Matrix event ID casing for thread session keys and Matrix thread reply relations.

## Best Possible Solution

Matrix thread-root event IDs should stay byte-for-byte case-preserved across the shared session recording boundary while intentional normalization for stable base session-key components remains intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to route a mixed-case Matrix thread root through resolveMatrixInboundRoute and then recordInboundSession; the Matrix route keeps the suffix, and src/channels/session.ts lowercases it before persistence.

## Solution Assessment

Yes. The best fix is a narrow case-preserving canonicalization seam for thread suffixes or Matrix-owned opaque IDs, not a broad removal of session-key normalization.
