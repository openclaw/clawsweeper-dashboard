## Summary

Keep open. Current main still lowercases full Matrix-derived session keys in shared record/store/routing paths while Matrix thread routing intentionally preserves opaque event IDs, so the reported duplicate-session and invalid relation target bug remains source-reproducible.

## What This Changes

The issue requests preserving case-sensitive Matrix room and event IDs throughout thread session keys and Matrix reply delivery.

## Best Possible Solution

Add a single case-preserving session-key seam for Matrix opaque room and thread identifiers across route construction, inbound recording, store lookup, delivery metadata, outbound recovery, and Matrix send relation tests.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix room/thread ID is preserved by Matrix routing, then lowercased by shared session-key construction, inbound recording, and store lookup before outbound delivery can reuse it as the Matrix relation target.

## Solution Assessment

Yes. The best fix is a narrow typed canonicalization seam for Matrix opaque identifiers, not stuck-session recovery alone and not a broad removal of generic lowercase normalization.
