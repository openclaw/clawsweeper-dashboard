## Summary

Keep open. Current main has Matrix-local route and send paths that preserve mixed-case Matrix thread event IDs, but the shared inbound session recorder still lowercases the full session key before persistence and last-route updates, so the reported split between mixed-case runtime routing and lowercased stored session keys remains plausible and actionable.

## What This Changes

The issue requests that Matrix thread event IDs remain case-sensitive end-to-end so each Matrix thread uses one session and replies use the original event ID in Matrix relations.

## Best Possible Solution

Preserve Matrix event ID casing across the shared session-recording and session-key parsing boundary while keeping intentional lowercase normalization for agent/channel/base key components, then add focused Matrix regression coverage for mixed-case thread roots through inbound recording and reply delivery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a mixed-case Matrix thread root through the Matrix inbound handler with threadReplies enabled: Matrix routing keeps the mixed-case suffix, then src/channels/session.ts lowercases it before persistence; the reporter's Synapse logs provide matching production evidence.

## Solution Assessment

Yes for the requested direction, but the implementation should be narrow. The maintainable fix is a shared canonicalization helper or channel-provided session-key preservation seam, not a blanket removal of all session-key normalization.
