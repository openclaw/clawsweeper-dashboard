## Summary

Keep open. Current main still preserves Matrix thread IDs in the Matrix route, then lowercases the full session key in shared session recording and store lookup, matching the duplicate-session and invalid relation failure described by two deployments.

## What This Changes

Fix Matrix session-key handling so case-sensitive Matrix room IDs and thread event IDs are preserved through inbound recording, store lookup, and outbound reply delivery.

## Best Possible Solution

Preserve opaque Matrix room and event identifiers byte-for-byte in session recording, store resolution, delivery metadata, and outbound recovery while keeping lowercase canonicalization for non-opaque session-key components.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to route a mixed-case Matrix thread through resolveMatrixInboundRoute, then record it through recordInboundSession and resolveSessionStoreEntry; the route keeps the suffix mixed-case while persistence lowercases it.

## Solution Assessment

Yes. The narrow maintainable fix is a case-preserving canonicalization seam for opaque Matrix identifiers, not disabling Matrix threading or treating the broader stuck-session recovery issue as the primary repair.
