## Summary

Keep open. Current main still has a source-reproducible Matrix/session-key case collapse: Matrix preserves exact room and thread event IDs at the plugin boundary, but shared routing, inbound recording, and session-store lookup still lowercase opaque key components that Matrix treats as case-sensitive.

## What This Changes

The issue requests that Matrix room and thread event IDs remain case-preserved across session key construction, persistence, lookup, and outbound thread delivery.

## Best Possible Solution

Add a case-preserving provider-opaque identifier contract for Matrix room and event IDs across route construction, inbound recording, store lookup, delivery metadata, transcript lookup, and outbound thread recovery while retaining generic canonicalization for non-opaque keys.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix room/thread ID is preserved in Matrix helpers, then shared route construction, inbound recording, and store lookup lowercase the opaque identifier before persistence or recovery.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve Matrix opaque identifier components or carry provider-exact delivery facts forward instead of reconstructing Matrix targets and thread IDs from lowercased session keys.
