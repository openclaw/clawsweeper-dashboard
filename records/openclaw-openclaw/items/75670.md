## Summary

Keep open. Current main and the latest release still lowercase full session keys in shared recorder/store paths while Matrix thread routing preserves case-sensitive event IDs and Matrix send uses the recovered thread ID as the relation target, so the reported boundary mismatch remains valid.

## What This Changes

The issue requests preserving Matrix room and thread event ID casing across session keys, session-store lookup, and thread reply delivery.

## Best Possible Solution

Add a coordinated Matrix-safe session-key canonicalization path that preserves opaque Matrix room and event IDs through construction, recording, store lookup, delivery metadata, and outbound recovery while retaining existing normalization for non-opaque keys.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread ID is preserved by Matrix routing, then shared recorder/store code lowercases the full key before persistence and lookup, and Matrix send writes the recovered thread ID directly into `m.relates_to`.

## Solution Assessment

Yes. The maintainable fix is a narrow case-preserving seam for Matrix opaque identifiers across the session-key boundary; stuck-session recovery or a builder-only patch would not fix invalid Matrix relation targets.
