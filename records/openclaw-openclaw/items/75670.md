## Summary

Keep open. Current main still has a source-reproducible Matrix case-collapse path: Matrix thread roots are preserved in the Matrix route helper, but shared route construction, inbound recording, store lookup, and delivery lookup still lowercase Matrix room/thread session identifiers.

## What This Changes

The issue asks OpenClaw to preserve case-sensitive Matrix room IDs and thread event IDs across session-key construction, persistence, lookup, and threaded reply delivery.

## Best Possible Solution

Add a case-preserving opaque-identifier path for Matrix session components across construction, recording, store lookup, delivery metadata, transcript lookup, and outbound thread recovery, with mixed-case room and thread regression tests.

## Reproduction Assessment

Yes, source-level. Matrix route helpers preserve mixed-case thread roots, then shared route construction, inbound recording, store lookup, and delivery lookup paths lowercase peer IDs or full session keys before persistence and recovery.

## Solution Assessment

Yes. The best fix is a narrow case-preserving seam for Matrix opaque identifiers; stuck-session recovery in https://github.com/openclaw/openclaw/issues/71127 may help after a wedge but would not prevent invalid Matrix relation targets.
