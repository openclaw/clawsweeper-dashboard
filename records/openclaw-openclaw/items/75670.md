## Summary

Keep open. Current `main` still has a source-reproducible Matrix/session boundary mismatch: Matrix routing preserves mixed-case room and thread event identifiers, but generic session-key construction, inbound recording, and store lookup still lowercase opaque identifiers, matching the reporter’s logs and follow-up confirmations.

## What This Changes

The issue reports that Matrix room and thread event identifiers are lowercased when used in session keys, causing duplicate stuck sessions and invalid thread reply relations.

## Best Possible Solution

Add a case-preserving provider-opaque identifier seam for Matrix room and event IDs across route construction, inbound recording, store lookup, delivery metadata, transcript lookup, and outbound thread recovery, with regression tests that keep generic session-key canonicalization intact.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread root is preserved by the Matrix route helpers, then generic session construction, recording, and store lookup lowercases the route/session key before persistence or lookup.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve Matrix opaque identifier components or carry them as prepared delivery facts instead of reconstructing them from lowercased session keys; https://github.com/openclaw/openclaw/issues/71127 may improve stuck-session recovery but would not fix invalid Matrix relation targets.
