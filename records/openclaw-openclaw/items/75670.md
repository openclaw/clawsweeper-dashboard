## Summary

Keep open. Current main still has the reported Matrix/session boundary mismatch: Matrix preserves mixed-case thread IDs during inbound routing, but shared session construction, recording, and store lookup still lowercase opaque room/event IDs before persistence and lookup.

## What This Changes

The issue asks OpenClaw to preserve case-sensitive Matrix room and event IDs across thread session keys, session persistence, and thread reply delivery.

## Best Possible Solution

Introduce a generic case-preserving path for provider-opaque session-key components, have Matrix use it for room and event IDs, and add regression coverage from inbound route through store lookup and outbound thread relation delivery.

## Reproduction Assessment

Yes, source-level with high confidence. A mixed-case Matrix thread ID is preserved by `resolveMatrixInboundRoute`, then the shared route/session/store layers lowercase the peer ID or full session key before persistence and lookup.

## Solution Assessment

Yes, the best fix is a coordinated case-preserving session-key component strategy for opaque provider IDs. Stuck-session recovery in https://github.com/openclaw/openclaw/issues/71127 would help recovery symptoms but would not fix invalid Matrix relation targets.
