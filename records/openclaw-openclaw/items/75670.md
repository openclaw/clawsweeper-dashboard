## Summary

Keep open. Current main still has the reported Matrix/core boundary mismatch: Matrix routing preserves mixed-case room and thread event IDs, but the shared inbound recorder and session store lowercase the entire session key before persistence and lookup, so the second deployment confirmation is consistent with the source path rather than a separate stale report.

## What This Changes

Preserve Matrix room and event ID casing across thread session keys and delivery recovery so Matrix threads do not create duplicate stuck sessions or invalid relations.

## Best Possible Solution

Add a narrow case-preserving canonicalization seam for Matrix opaque room and thread event IDs across record, store lookup, delivery metadata, and outbound recovery, with regression tests that keep generic session-key normalization intact.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread root passes through `resolveMatrixInboundRoute` unchanged, then `recordInboundSession` and `resolveSessionStoreEntry` lowercase the full key before persistence and lookup.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve opaque Matrix room/event ID bytes at the session-key boundary; stuck-session recovery in #71127 would be useful separately but would not fix invalid Matrix relation targets.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75670. Current main preserves Matrix thread IDs in `resolveMatrixInboundRoute` / `resolveMatrixThreadSessionKeys`, but `recordInboundSession`, `normalizeStoreSessionKey`, and `resolveSessionStoreEntry` lowercase whole session keys, corrupting case-sensitive Matrix room IDs and `:thread:$event` suffixes before delivery recovery and outbound `m.relates_to` construction. Implement a narrow case-preserving canonicalization path for Matrix-owned opaque suffixes across record, update, read, and delivery recovery. Keep existing lowercase canonicalization for generic/non-opaque session keys, do not add a new config option, and do not treat stuck-session abort policy as the primary fix. Add regression tests for mixed-case Matrix room IDs and mixed-case thread roots through route -> record/store/updateLastRoute -> extractDeliveryInfo/send relation. Add a user-facing `CHANGELOG.md` fix entry. Stop and ask if the fix would require a broad session-key grammar redesign.
