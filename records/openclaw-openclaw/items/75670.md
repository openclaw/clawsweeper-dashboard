## Summary

Keep open. Current main still lowercases full session keys in shared routing/record/store paths while Matrix treats room IDs and thread event IDs as opaque case-sensitive identifiers, matching the reporter's failure mode and the deployment confirmations in the discussion.

## What This Changes

The issue reports that Matrix session-key normalization lowercases case-sensitive room and thread event IDs, creating duplicate sessions and invalid thread reply relations.

## Best Possible Solution

Preserve Matrix opaque room/event ID casing across routing, session recording/store lookup, delivery metadata, and outbound thread relation recovery while keeping generic lowercase normalization for non-Matrix session keys.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread root is preserved by Matrix routing, then shared recorder/store/route construction paths lowercase full session keys before persistence or lookup, which is enough to reproduce the case-collapse boundary on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a Matrix-aware case-preserving session-key/delivery seam; generic stuck-session recovery in https://github.com/openclaw/openclaw/issues/71127 and outbound hook propagation in https://github.com/openclaw/openclaw/pull/73706 are adjacent but do not fix invalid Matrix relation targets.
