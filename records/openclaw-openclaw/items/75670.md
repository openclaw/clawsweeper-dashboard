## Summary

Keep open. Current main still preserves Matrix thread IDs inside the Matrix route helper, but shared route construction, inbound recording, session-store lookup, and delivery lookup paths still lowercase Matrix room/thread session identifiers, so the reported root cause remains source-reproducible and is not superseded by the related stuck-session or outbound-hook items.

## What This Changes

The issue requests preserving case-sensitive Matrix room and thread event IDs in session keys and outbound thread relations to avoid duplicate stuck sessions and failed Matrix thread delivery.

## Best Possible Solution

Add a case-preserving opaque-identifier path for Matrix session components across construction, route resolution, metadata recording, store lookup, delivery context, transcript lookup, and outbound thread recovery, with regression tests for mixed-case room IDs and thread event IDs.

## Reproduction Assessment

Yes, source-level. Matrix route helpers preserve mixed-case thread roots, then shared route/session recording and store lookup paths lowercase peer IDs or full session keys before persistence and delivery lookup.

## Solution Assessment

Yes. The best fix is a narrow case-preserving seam for Matrix opaque identifiers; stuck-session recovery in https://github.com/openclaw/openclaw/issues/71127 would help after a wedge but would not prevent invalid Matrix relation targets.
