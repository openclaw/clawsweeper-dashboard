## Summary

Keep open. Current main still lowercases opaque Matrix room/thread identifiers at the generic routing, recording, and session-store boundaries, while the Matrix channel code and docs preserve those identifiers as case-sensitive; the linked stuck-session issue and outbound-hook PR do not cover this root cause.

## What This Changes

The issue reports that Matrix room and thread event identifiers are lowercased in session keys, causing duplicate sessions and invalid Matrix thread reply relations.

## Best Possible Solution

Preserve Matrix opaque room and event IDs across routing, session metadata, store lookup, delivery context, transcript lookup, and outbound thread recovery while keeping generic canonicalization for channels that can safely use it.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread root is preserved by Matrix route helpers, then generic route/session recording and store lookup lowercases the room or whole session key before persistence and later delivery recovery.

## Solution Assessment

Yes. The narrow maintainable fix is a case-preserving opaque-identifier seam for Matrix route components; https://github.com/openclaw/openclaw/issues/71127 can improve stuck-session recovery, but it would not prevent invalid Matrix relation targets.
