## Summary

Keep open. Current main still preserves mixed-case Matrix thread roots in Matrix routing, then lowercases the full opaque session key in the shared inbound session recorder, so the reported split between Matrix thread identity and persisted session identity remains source-reproducible.

## What This Changes

The issue requests preserving Matrix thread event ID casing across thread session keys and Matrix reply relations so each Matrix thread maps to one working session.

## Best Possible Solution

Preserve Matrix thread-root event IDs byte-for-byte across the shared session recording boundary, while keeping stable normalization for non-opaque base session-key components.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to route a mixed-case Matrix thread root through resolveMatrixInboundRoute and then record it through recordInboundSession; Matrix keeps the suffix mixed-case, while src/channels/session.ts lowercases it before persistence.

## Solution Assessment

Yes. The narrow maintainable fix is a case-preserving canonicalization seam for opaque thread suffixes or Matrix-owned IDs, not disabling Matrix threading or adding stuck-session recovery as the primary repair.
