## Summary

Keep open. Current main still preserves mixed-case Matrix thread roots in Matrix routing but lowercases the full session key in shared session recording and store lookup, so the reported duplicate-session and outbound relation failure path remains valid.

## What This Changes

The issue requests preserving Matrix event ID case across thread session keys, persisted session metadata, and Matrix thread reply delivery.

## Best Possible Solution

Add a narrow case-preserving session-key/delivery metadata path for opaque Matrix thread roots, with regression tests proving mixed-case event IDs survive recording, store lookup, reply context recovery, and outbound relation construction.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a mixed-case Matrix thread root through resolveMatrixInboundRoute and then record it through recordInboundSession; routing keeps :thread:$AbC while the shared recorder/store lowercases the key before persistence.

## Solution Assessment

Yes. The maintainable direction is a targeted canonicalization seam for opaque thread suffixes or Matrix delivery metadata, not disabling Matrix threading or relying on the broader stuck-session recovery work in #71127.
