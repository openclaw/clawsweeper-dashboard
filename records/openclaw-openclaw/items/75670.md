## Summary

Keep open. Current main still lowercases Matrix room and thread identifiers in shared session-key paths even though Matrix code and docs treat those identifiers as case-sensitive; the issue discussion includes real deployment evidence, and the latest clarification does not invalidate the original thread/session bug.

## What This Changes

The issue asks OpenClaw to preserve case-sensitive Matrix room and event IDs in thread session keys and outbound thread relations so one session is created and replies target the real Matrix event.

## Best Possible Solution

Add a case-preserving canonicalization seam for Matrix opaque room and event identifiers across routing, recording, store lookup, delivery metadata, and recovery, while keeping existing lowercase behavior for channels whose identifiers are case-insensitive.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread root is preserved by the Matrix route helper, then shared recorder/store paths lowercase the full session key before persistence and lookup.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve Matrix opaque identifiers at the session-key boundary; stuck-session recovery in https://github.com/openclaw/openclaw/issues/71127 is separate and would not fix invalid Matrix relation targets.
