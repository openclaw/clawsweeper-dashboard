## Summary

Keep open. Current main still has the source-level mismatch described in the report: Matrix routing preserves mixed-case thread roots, but the shared inbound session recorder and store canonicalization lowercase the whole session key before persistence.

## What This Changes

The issue requests preserving Matrix thread event ID casing through session-key construction and reply delivery so mixed-case Matrix thread IDs do not create duplicate stuck sessions or invalid Matrix relations.

## Best Possible Solution

Preserve Matrix thread-root event IDs byte-for-byte across session recording, store lookup, delivery metadata, and outbound relation recovery while keeping lowercase normalization for non-opaque session-key components.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a mixed-case Matrix thread root through resolveMatrixInboundRoute and then record it through recordInboundSession; the Matrix route keeps the suffix mixed-case, while src/channels/session.ts lowercases it before persistence.

## Solution Assessment

Yes. The narrow maintainable fix is a case-preserving canonicalization seam for opaque Matrix thread suffixes or persisted delivery thread IDs, not disabling Matrix threading or treating stuck-session recovery in #71127 as the primary repair.
