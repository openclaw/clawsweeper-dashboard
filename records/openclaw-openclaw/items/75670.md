## Summary

Keep open. Current main still has the reported Matrix/core session-key casing mismatch: Matrix thread routing preserves mixed-case event IDs, but shared routing, recording, and store lookup paths still lowercase full session keys, which can collapse Matrix room IDs and thread event IDs before delivery.

## What This Changes

Preserve Matrix case-sensitive room and thread event IDs across session keys and delivery so Matrix threads create one session and replies target the original event ID.

## Best Possible Solution

Add a narrow case-preserving session-key/canonicalization seam for Matrix opaque identifiers across route construction, inbound recording, store lookup, delivery metadata, and outbound recovery, with regression coverage for mixed-case room IDs and thread event IDs.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix room/thread can enter Matrix routing as case-sensitive data, but current main lowercases full session keys in generic route construction, inbound recording, and store lookup before outbound relation recovery.

## Solution Assessment

Yes. The maintainable direction is not stuck-session recovery or a new config knob; it is a focused case-preserving seam for Matrix opaque identifiers while keeping generic session-key normalization intact.
