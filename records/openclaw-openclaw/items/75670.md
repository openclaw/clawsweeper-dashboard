## Summary

Keep open. Current main still has the Matrix/core session-key mismatch described in the report: Matrix routing preserves mixed-case thread roots, but shared session recording and store canonicalization lowercase the full key before persistence.

## What This Changes

Matrix thread session keys should preserve case-sensitive event IDs so inbound routing, persisted session metadata, and outbound Matrix relations use one exact thread root.

## Best Possible Solution

Add a case-preserving canonicalization path for opaque Matrix thread suffixes across session recording, store lookup, delivery extraction, and outbound relation construction, while retaining lowercase alias behavior for generic session keys.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to route a Matrix threadId such as `$AbC123:example.org` through resolveMatrixInboundRoute and then record it through recordInboundSession; current main preserves the suffix in Matrix routing and lowercases it at the shared recorder/store boundary.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve opaque Matrix thread event IDs at the Matrix/core session-key boundary, not to disable Matrix threading or treat the broader stuck-session recovery tracked by #71127 as the primary repair.
