## Summary

Keep open. Current main and the latest release still have the Matrix/core boundary mismatch: Matrix preserves mixed-case opaque thread IDs, but shared session recording and store lookup lowercase the full session key before persistence and lookup.

## What This Changes

The issue asks OpenClaw to preserve Matrix room and thread event ID casing through session routing, storage, and outbound thread replies to avoid duplicate stuck sessions and Matrix unknown-event delivery failures.

## Best Possible Solution

Keep the issue open and fix the shared session-store canonicalization seam so Matrix opaque room IDs and event IDs are preserved end to end while existing generic canonical keys continue to dedupe.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread root is preserved by Matrix routing, then the shared recorder and store lookup lowercase the full session key before persistence and recovery.

## Solution Assessment

Yes. The narrow maintainable fix is a case-preserving canonicalization seam for Matrix opaque IDs at the shared session/store/delivery boundaries; stuck-session recovery in #71127 would not fix invalid Matrix relation targets.
