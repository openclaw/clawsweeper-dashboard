## Summary

Keep open. Current main still preserves Matrix thread-root case in Matrix routing, but the shared inbound recorder and session store lower-case the whole session key before persistence, matching the duplicate-session and invalid relation failure described in the report.

## What This Changes

Matrix thread session keys should preserve case-sensitive event IDs so one thread session is recorded and outbound replies use the original event ID in `m.relates_to`.

## Best Possible Solution

Add a narrow case-preserving canonicalization path for opaque Matrix thread ids across session recording, store lookup, delivery metadata, and outbound relation recovery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to route a mixed-case Matrix thread root through `resolveMatrixInboundRoute` and then record it through `recordInboundSession`; the route keeps the suffix mixed-case while the recorder/store lowercases it.

## Solution Assessment

Yes. The narrow maintainable direction is preserving opaque Matrix thread suffixes at the Matrix/core session-key boundary, not disabling Matrix threading or treating the stuck-session recovery work in #71127 as the primary repair.
