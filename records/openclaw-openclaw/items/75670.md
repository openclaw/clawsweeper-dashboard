## Summary

Keep open. Current main still has the source-level Matrix/core mismatch reported here: Matrix routing preserves mixed-case thread roots, but shared session recording and store lookup still lowercase full session keys, which can corrupt opaque Matrix room and event identifiers.

## What This Changes

The issue asks OpenClaw to preserve case-sensitive Matrix room and thread event IDs when constructing, recording, looking up, and using Matrix session keys.

## Best Possible Solution

Preserve Matrix room IDs and thread event IDs byte-for-byte across session-key construction, recording, store lookup, delivery metadata recovery, and outbound relations while keeping lowercase normalization for structural tokens and legacy non-opaque keys.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a mixed-case Matrix room/thread route through resolveMatrixInboundRoute and then record it through recordInboundSession; current main keeps the Matrix thread suffix mixed-case in routing but lowercases the full persisted key.

## Solution Assessment

Yes. The narrow maintainable fix is a case-preserving canonicalization seam for Matrix opaque identifiers, not disabling Matrix threading or treating #71127's generic stuck-session recovery as the primary repair.

## Repair Work Prompt

Fix the Matrix session-key case-folding bug described in https://github.com/openclaw/openclaw/issues/75670. Current main preserves Matrix thread IDs in Matrix routing, but recordInboundSession and session-store lookup lowercase full session keys, corrupting opaque Matrix room IDs and thread event IDs. Implement a narrow case-preserving normalization seam for Matrix opaque identifiers across inbound recording, resolveSessionStoreEntry/updateLastRoute, delivery-info recovery, and outbound thread relation recovery. Preserve lowercase canonicalization for structural tokens and existing non-opaque/legacy channel keys; do not disable Matrix threadReplies and do not make #71127 stuck-session abort behavior the primary fix. Add regression tests for mixed-case Matrix room IDs and mixed-case Matrix thread event IDs through inbound route -> recordInboundSession/store -> delivery info/outbound send relation. Add a CHANGELOG.md fix entry because this is user-facing.
