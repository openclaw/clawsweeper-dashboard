## Summary

Keep open. Current main still has a source-reproducible Matrix/core boundary mismatch: Matrix preserves mixed-case room and thread event IDs in routing, but shared session-key construction, recording, and store lookup still lowercase opaque Matrix identifiers, matching the original report and follow-up deployment evidence.

## What This Changes

The issue asks OpenClaw to treat Matrix room and thread event IDs as case-sensitive across session routing, persistence, lookup, and thread reply delivery.

## Best Possible Solution

Add a narrow case-preserving canonicalization seam for Matrix opaque room and event IDs across route construction, inbound recording, store lookup, delivery metadata, transcript lookup, and outbound thread recovery, with regression coverage that keeps generic session-key normalization intact.

## Reproduction Assessment

Yes, source-level. A mixed-case Matrix thread root is preserved by `resolveMatrixInboundRoute`, then `recordInboundSession` and `resolveSessionStoreEntry` lowercase the full key before persistence and lookup.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve Matrix opaque identifier components at the session boundary; https://github.com/openclaw/openclaw/issues/71127 may improve stuck-session recovery but would not fix invalid Matrix relation targets.
