## Summary

Current main and the latest release already parse MEDIA directive lines and carry the parsed media URLs into normalized outbound payloads before plugin/channel dispatch, so the reported missing-field behavior does not match the shipped code.

## What This Changes

The issue asks for MEDIA directive parsing to include parsed attachment URLs in the payload passed to plugin send interfaces.

## Best Possible Solution

Keep the shipped media directive parser and outbound payload normalization; if a specific plugin still drops mediaUrls, track that as a plugin-specific delivery bug with real payload logs.

## Reproduction Assessment

No. Source inspection and existing regression tests show current main parses MEDIA lines and propagates mediaUrls into outbound payloads, so I do not have a current-main failing path for the reported missing field.

## Solution Assessment

Yes. The existing TypeScript outbound payload plan is the maintainable fix point because it normalizes parsed directives before channel/plugin delivery; the suggested Go core/session.go patch does not apply to this repository.
