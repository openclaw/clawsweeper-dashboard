## Summary

## What This Changes

The PR adds an opt-in `agents.defaults.runtimeContextPlacement` setting that can keep extracted runtime context in the submitted user prompt instead of appending it to the system prompt.

## Best Possible Solution

Merge a documented, release-noted opt-in only after maintainers accept the new config contract; otherwise settle the runtime-context cache policy in core without adding this setting.

## Reproduction Assessment

Yes for a source-level reproduction path: current main separates runtime-only context from the visible prompt and appends it to the system prompt, so volatile metadata can mutate the outbound system prompt across turns. I did not run a live Ollama cache log reproduction.

## Solution Assessment
