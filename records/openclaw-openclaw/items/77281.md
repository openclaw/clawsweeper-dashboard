## Summary

Current main already fixes the cache-locality problem through #77521 by keeping ordinary per-turn runtime context out of chat system prompts while preserving runtime-only events on their existing system-context path. The PR is now redundant, and its branch still has a runtime-only early-return regression.

## What This Changes

The PR adds optional `agents.defaults.runtimeContextPlacement`, wires `prompt` mode into embedded-runner runtime-context prompt splitting, and updates schema, docs, changelog, and focused tests.

## Best Possible Solution

Keep the current main implementation from #77521, which fixes cache locality without adding a new public config key and retains the dedicated runtime-only event path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: the PR head's `prompt` mode returns before the empty-transcript runtime-only branch, while current main tests show that runtime-only events must keep the marker prompt and system-context handling. I did not live-run Ollama cache logs in this read-only review.

## Solution Assessment

Yes for cleanup: closing this PR is the best outcome because current main already solves the cache issue through a narrower runtime-context delivery change. The PR branch is no longer the best implementation because it adds a config knob and changes runtime-only event behavior under that knob.
