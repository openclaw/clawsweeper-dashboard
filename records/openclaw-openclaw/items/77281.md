## Summary

Keep this PR open. The earlier changelog and public-docs review feedback appears addressed in the latest PR context, but the current diff is not merge-ready because the new prompt-placement early return would also catch runtime-only events with an empty `transcriptPrompt`, bypassing the existing runtime-only marker/system-context path and submitting synthetic runtime text as the user prompt.

## What This Changes

The PR adds an optional `agents.defaults.runtimeContextPlacement` config setting, threads it into embedded-runner runtime-context prompt splitting, and updates schema/help metadata, generated schema, docs, changelog, and focused tests.

## Best Possible Solution

Preserve the opt-in cache-friendly layout only for real user-turn prompts, while keeping runtime-only events on the existing runtime-only system-context path and documenting the accepted config contract if maintainers want this knob.

## Reproduction Assessment

Yes, for a source-level reproduction path: current main splits runtime context from `transcriptPrompt` and appends it to the system prompt before model submission, so changing per-turn metadata mutates that prompt. I did not live-run Ollama cache logs in this read-only review.

## Solution Assessment

No, not as written. The PR is a narrow plausible direction for user-turn cache locality, but the early return must not bypass existing runtime-only event handling, and maintainers still need to accept the new config option.
