The PR adds an optional `agents.defaults.runtimeContextPlacement` config mode, threads it into embedded-runner runtime-context prompt placement, and updates schema/help docs, changelog, and focused tests.

## Best Possible Solution

Keep the #77521 main implementation; any future user-facing placement knob should be proposed separately only if maintainers want an opt-out from the default hidden-context behavior.

## Reproduction Assessment

Yes, source-level: current main has checked-in tests proving runtime context no longer reaches the submitted prompt or system prompt on normal transcript turns, and the PR branch regression is visible from its early return before the runtime-only branch.
