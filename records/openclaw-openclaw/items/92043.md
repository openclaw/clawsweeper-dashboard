## Label Justifications

- `P1`: The shipped default can leave affected long sessions unable to complete compaction or continue ordinary agent turns without operator intervention.
- `impact:session-state`: Interrupted compaction commits no summary progress, leaving the existing session above its usable context budget.
- `impact:crash-loop`: Later turns can restart the same operation under the same timeout without an established convergence mechanism.

## Summary

Keep open: the defect remains in the latest release and current main. Compaction still receives one operation-wide 180-second budget, timeout aborts discard the rolling chunk summary, and the default user-facing failure omits both the timeout cause and its configuration control; the permanent recovery contract needs maintainer judgment.

## What This Changes

Ensure legitimately long compactions make reusable progress or fail with actionable timeout guidance instead of restarting the same full pipeline on later turns.

## Best Possible Solution

Adopt one maintainer-approved recovery contract that keeps session locks bounded, guarantees convergence or reusable progress after interruption, and always exposes the timeout cause and supported configuration path.

## Reproduction Assessment

Yes, at source level with high confidence. The latest release and current main show one outer timeout, local-only chunk progress, immediate timeout propagation, and no transcript adoption until successful completion.

## Solution Assessment

Unclear. The diagnosis is correct and better failure guidance is straightforward, but maintainers must choose whether the durable recovery boundary is resumable canonical state, per-stage budgeting, or earlier deterministic reduction while preserving bounded locks.
