## Summary

Keep open: current main and the latest release still have the reported compaction timeout shape, and https://github.com/openclaw/openclaw/pull/92899 is the open implementation candidate with closing syntax.

## What This Changes

The issue reports that default compaction uses one 180s wall-clock budget across a multi-chunk summarization pipeline, discards completed timeout progress, and hides timeout/config guidance from default failure text.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92899 if maintainers accept its bounded timeout-progress semantics; otherwise define the canonical compaction recovery state machine before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main still has the one-wall-clock compaction wrapper, timeout-shaped chunk failures still rethrow before partial-summary attachment, and default preflight copy still hides timeout guidance; I did not run the synthetic Vitest or a live slow-provider compaction in this read-only review.

## Solution Assessment

Likely, but maintainer judgment is still needed. The linked PR targets the inner chunk timeout, outer safety timeout race, native AgentSession abort reason, and failure-copy paths without changing public config shape or defaults; the open question is whether its bounded session-state commit window is the desired runtime contract.
