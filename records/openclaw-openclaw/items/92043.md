## Summary

Keep open: current main and the latest release still show the reported compaction timeout shape, and https://github.com/openclaw/openclaw/pull/92899 is an open clean fix candidate with closing syntax, so this issue should stay open until that PR is landed or explicitly rejected.

## What This Changes

The issue reports that aggregate compaction timeouts can discard completed chunk progress and hide timeout/config guidance, causing long compactions to fail repeatedly instead of making forward progress.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92899 if maintainers accept its bounded timeout-progress semantics; otherwise define the canonical compaction recovery state machine before closing this issue.

## Reproduction Assessment

Yes, at source level: current main still has the one-wall-clock compaction wrapper, timeout-shaped chunk failures still rethrow before partial-summary attachment, and default preflight copy still hides timeout guidance. I did not run the synthetic Vitest or a live slow-provider compaction in this read-only review.

## Solution Assessment

Likely yes for the current candidate: the open PR targets the inner chunk timeout, outer safety-timeout, AgentSession abort reason, and failure-copy paths without changing the public config shape or default. The remaining question is maintainer acceptance of the bounded session-state commit window.
