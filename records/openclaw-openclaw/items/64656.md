- **[P2] Thread memory keys through attempt lifecycle calls:** `src/agents/pi-embedded-runner/run.ts:635`
  - body: Passing `memoryConversationKey` into the attempt is not enough because `attempt.ts` still calls bootstrap, assemble, and finalization with only `params.sessionKey`.   - confidence: 0.88
- **[P2] Preserve legacy memory identity on first reset:** `src/gateway/session-reset-service.ts:603-606`
  - body: For existing sessions without `memoryConversationKey`, `/reset` falls back to `randomUUID()`, which rotates the LCM identity instead of preserving the legacy session identity promised by the comment. Seed the preserved key from the existing session identity on first reset.
  - confidence: 0.9
- **[P2] Sanitize legacy auto overrides before model selection:** `src/auto-reply/reply/agent-runner-execution.ts:619-620`
  - body: `sanitizeFailoverPollutedEntry` runs inside fallback persistence after model selection has already consumed a legacy `modelOverrideSource: "auto"` entry. Polluted sessions can still run the fallback first; the migration needs to happen before selection resolves the run model.
  - confidence: 0.82
- **[P2] Clear failover status when the primary succeeds again:** `src/auto-reply/status.ts:814-815`
  - body: `/status` treats `entry.failoverState.active` as current, but the PR only clears that state on reset or explicit user model changes. A later successful primary run can still display an old temporary failover as active.
  - confidence: 0.8
