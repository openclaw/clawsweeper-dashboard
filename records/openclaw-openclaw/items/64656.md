## Summary

Keep open: current main only implements the reset cleanup portion of this area, while this PR still proposes broader failover-state, LCM identity, and overflow-recovery changes with correctness blockers and a stale context-engine surface.

## What This Changes

The PR moves auto-failover persistence into `SessionEntry.failoverState`, adds `memoryConversationKey` for LCM/context-engine identity, and adds overflow diagnostics plus a compaction/truncation circuit breaker.

## Best Possible Solution

Rebase and narrow the branch: decide whether to keep the documented `modelOverrideSource: "auto"` policy or introduce a versioned `failoverState`, wire any context-engine identity split through the current harness lifecycle, and keep only overflow recovery changes backed by focused tests.

## Reproduction Assessment

Yes for the PR review findings: they are reproducible by tracing the PR diff against current model-selection, reset, status, and context-engine call paths. No live provider failure is required to see the stale migration timing, legacy reset identity, or stale status state problems.

## Solution Assessment

No. The proposed branch is not the best solution as-is because it mixes a broad session contract change, LCM/plugin identity semantics, and overflow recovery while leaving concrete lifecycle and state-consistency issues.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Move auto-override sanitization before selection:** `src/auto-reply/reply/agent-runner-execution.ts:594-595`
  - body: `sanitizeFailoverPollutedEntry` runs only inside fallback persistence, after the stored `providerOverride`/`modelOverride` have already been used to build the current run. Upgraded sessions polluted by the old auto override still take one more fallback turn instead of retrying the configured primary; migrate or clear the auto override before `createModelSelectionState` resolves the run model.
  - confidence: 0.86
- **[P2] Seed reset memory keys from the legacy session:** `src/gateway/session-reset-service.ts:604-607`
  - body: For existing sessions without `memoryConversationKey`, `/reset` falls back to `randomUUID()`. That rotates legacy sessions away from their existing context-engine identity even though reset is intended to preserve continuity; seed the preserved key from the existing session identity instead.
  - confidence: 0.88
- **[P2] Clear failover state after primary recovery:** `src/auto-reply/status.ts:810-812`
  - body: The new status line treats `entry.failoverState.active` as current, but normal model selection no longer reads that state and the PR only clears it on reset or explicit user model changes. After a later primary run succeeds, `/status` can still report an old temporary failover as active.
  - confidence: 0.8
