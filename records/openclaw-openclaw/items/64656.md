- **[P2] Move auto-override sanitization before selection:** `src/auto-reply/reply/agent-runner-execution.ts:596-597`
  - body: `sanitizeFailoverPollutedEntry` runs only when persisting a fallback, after model selection has already used the legacy `providerOverride`/`modelOverride`.   - confidence: 0.86
- **[P2] Seed reset memory keys from the legacy session:** `src/gateway/session-reset-service.ts:603-606`
  - body: For existing sessions without `memoryConversationKey`, `/reset` falls back to `randomUUID()`, which rotates legacy sessions away from their existing context-engine identity even though the new behavior says reset preserves continuity. Seed the preserved key from the existing session identity instead of generating a new one for first reset.
  - confidence: 0.88
- **[P2] Route memory identity through the harness lifecycle:** `src/agents/pi-embedded-runner/run/attempt.context-engine-helpers.ts:144-156`
  - body: Current main routes these attempt helpers through the harness lifecycle, but the PR restores local helper logic and adds the memory key there. After rebase, bootstrap, assemble, afterTurn, ingest, and maintenance still need the same identity contract in `src/agents/harness/context-engine-lifecycle.ts`; otherwise `/new` and compaction can use inconsistent conversation keys.
  - confidence: 0.82
- **[P2] Clear failover state after primary recovery:** `src/auto-reply/status.ts:813-815`
  - body: `/status` treats `entry.failoverState.active` as current, but the PR only clears it on reset or explicit user model changes. Once the primary model later succeeds, status can still report an old temporary failover as active, so clear the state when normal selection/runs return to the primary.
  - confidence: 0.8
