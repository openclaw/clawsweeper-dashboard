## Summary

Keep open: the reported NVIDIA NIM token-field bug is source-provable, but this PR still hard-codes a provider host in core and lacks contributor after-fix real behavior proof.

## What This Changes

The branch adds NVIDIA NIM baseUrl detection to OpenAI-completions compatibility defaults and tests that the endpoint uses `max_tokens`.

## Best Possible Solution

Model NIM's legacy token field through NVIDIA-owned compat metadata or a generic manifest request seam, keep core provider-agnostic, and require focused regression coverage plus redacted live NIM proof.

## Reproduction Assessment

Yes for the source-level path: current main routes NVIDIA through OpenAI-completions, lacks a NIM `maxTokensField` override, and the transport emits `max_completion_tokens` by default. A live NIM request still needs contributor proof with credentials redacted.

## Solution Assessment

No: the PR fixes the outbound field selection, but hard-coding `integrate.api.nvidia.com` in core conflicts with the provider/plugin ownership boundary. A NVIDIA-owned compat entry or explicit manifest/request seam is the safer fix.
