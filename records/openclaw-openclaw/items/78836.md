## Summary

Keep open: the reported NVIDIA NIM token-field mismatch is credible from current source and NVIDIA docs, but this PR needs real behavior proof and should avoid putting NVIDIA-host ownership directly in core agents code.

## What This Changes

The PR adds NVIDIA NIM baseUrl detection to OpenAI-completions compatibility defaults and tests that NIM uses `max_tokens` while unrelated custom endpoints keep `max_completion_tokens`.

## Best Possible Solution

Carry the NIM `max_tokens` default through provider-owned compatibility metadata or a generic manifest/request seam, add focused regression coverage and a changelog entry, then merge after redacted real behavior proof is attached.

## Reproduction Assessment

Yes for the current-main source path: NVIDIA models use the OpenAI-completions route, the payload builder follows `maxTokensField`, and the default currently resolves to `max_completion_tokens`. No live NIM request was run in this read-only review.

## Solution Assessment

No: the PR addresses the symptom, but putting `integrate.api.nvidia.com` directly into core agents code conflicts with the provider/plugin ownership model; a provider-owned or generic manifest compatibility path is safer.
