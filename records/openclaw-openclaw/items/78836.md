## Summary

Keep this PR open because the NVIDIA NIM token-field bug is still present on current main and the branch contains a meaningful fix attempt, but it is not merge-ready due to missing inspectable live proof, a duplicated NVIDIA hostname classifier, and an unchanged sibling completions request builder.

## What This Changes

The PR adds NVIDIA NIM baseUrl detection to the agent OpenAI-completions compatibility defaults and tests so NIM-hosted models choose `max_tokens` instead of `max_completion_tokens`.

## Best Possible Solution

Centralize the NIM token-field default on the existing `nvidia-native` endpoint/capability signal, cover both completions request builders or retire one path, and require redacted live NIM proof before merge.

## Solution Assessment

No as written. The target behavior is needed, but the maintainable fix should reuse central `nvidia-native` endpoint attribution and cover the registered built-in completions path instead of adding a parallel hostname detector.
