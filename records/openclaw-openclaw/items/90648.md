## Summary

Keep open: current main and the latest release still use the regex-based Discord mention code-region skipper, and this PR is the active proof-backed fix for the linked Discord mention-alias bug. ## What This Changes

The PR replaces Discord mention alias code-region skipping with a backtick-aware scanner and adds regression tests for fenced and inline code cases.

## Best Possible Solution

Land the scoped backtick-only scanner and regression coverage after ordinary maintainer review, then let the linked Discord bug close when this PR merges.

## Reproduction Assessment

## Solution Assessment

Yes. The Discord-local backtick scanner is the narrowest maintainable fix here because the shared code-region helpers include tilde fences, while this PR preserves the existing Discord tilde-run alias behavior and covers that explicitly.
