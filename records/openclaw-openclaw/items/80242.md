## Summary

Keep open: the patch is a focused fix for a source-reproducible subagent completion routing bug and I found no blocking code or security finding, but it still needs redacted real Telegram or equivalent runtime proof before merge.

## What This Changes

The PR makes `sessions_spawn` requester ownership prefer `runSessionKey`, resolves subagent completion origins requester-first with child fallback, and adds focused regression tests.

## Best Possible Solution

Land this focused routing fix after proof shows the originating parent session receives the completion announce, while leaving mediated-delivery policy to https://github.com/openclaw/openclaw/pull/80223.

## Reproduction Assessment

Yes, source-reproducible: current main passes the sandbox session key into `sessions_spawn` while only `session_status` receives `runSessionKey`, and completion-origin resolution checks the child binding before the requester binding. I did not run live Telegram proof in this read-only review.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix because it reuses the existing live run-session key and generic bound-delivery router instead of adding Telegram-specific routing. The remaining blocker is real behavior proof, not a code-path objection.
