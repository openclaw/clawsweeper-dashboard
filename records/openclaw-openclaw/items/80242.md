## Summary

## What This Changes

The branch makes `sessions_spawn` requester ownership prefer `runSessionKey` over the sandbox peer key, checks requester bindings before child bindings for subagent completion origins, and adds focused agent tests.

## Best Possible Solution

Land the focused routing fix after redacted Telegram or equivalent runtime proof confirms the originating parent session receives the completion announce; keep mediated-delivery policy separate in https://github.com/openclaw/openclaw/pull/80223.

## Reproduction Assessment

Yes, source-reproducible: current main passes the sandbox session key into `sessions_spawn`, stores that as the requester, then resolves completion origin through the child binding before requester binding. I did not run live Telegram proof in this read-only review.

## Solution Assessment

Yes, the proposed path is the narrow maintainable fix because it reuses the already-threaded `runSessionKey` and makes bound delivery requester-first without adding Telegram-specific policy. The remaining blocker is proof, not a code-path objection.
