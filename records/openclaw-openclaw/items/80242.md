## Summary

Keep open: the PR is a focused fix for a source-reproducible subagent completion routing bug, but it is an external Telegram-affecting behavior change without after-fix real runtime proof, so it should not merge yet.

## What This Changes

The PR routes `sessions_spawn` requester ownership through the live run session key and makes subagent completion origin resolution prefer the requester binding before falling back to the child binding, with focused regression tests.

## Best Possible Solution

Land the focused requester-session routing fix after redacted Telegram or equivalent runtime proof and targeted tests, while keeping mediated-delivery policy work in https://github.com/openclaw/openclaw/pull/80223 separate.

## Reproduction Assessment

Yes, source-reproducible: current main passes a sandbox/channel session key into `sessions_spawn` while separately carrying the live run session key, and completion-origin resolution checks child binding before requester binding. I did not live-run the Telegram scenario during this read-only review.

## Solution Assessment

Yes as a code direction: using the already-threaded `runSessionKey` for spawn ownership and preferring requester binding before child fallback is the narrow maintainable fix. The remaining blocker is real behavior proof, not an identified code defect.
