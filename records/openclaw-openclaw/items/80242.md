## Summary

Keep open: the patch is a plausible narrow fix for the linked subagent completion misroute, but this external PR still lacks after-fix real behavior proof from a real OpenClaw setup, so it should not merge yet.

## What This Changes

The PR routes `sessions_spawn` requester ownership through `runSessionKey` when available and makes subagent completion origin resolution prefer the requester binding before falling back to the child binding, with focused regression tests.

## Best Possible Solution

Keep this scoped routing fix paired with the linked bug, add redacted real-session proof that the parent session receives the completion after the patch, and leave the separate mediated-delivery policy to https://github.com/openclaw/openclaw/pull/80223 or a narrower follow-up.

## Reproduction Assessment

Yes, source-level: current main passes the sandbox/channel session key into `sessions_spawn` while carrying `runSessionKey` separately, then stores that key as the requester and checks the child binding first during completion-origin resolution. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the proposed code path is the narrow maintainable fix direction: use the live run session for requester ownership and prefer requester bindings for completion delivery. Merge should still wait for real after-fix proof and green checks.
