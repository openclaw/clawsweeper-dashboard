## Summary

Keep open: this is a plausible focused bug fix, but the external PR cannot merge as-is because required live Telegram proof is missing and one added delivery-policy hunk is incomplete for group/channel completions.

## What This Changes

The PR prefers `runSessionKey` for `sessions_spawn` requester ownership, resolves subagent completion origin through requester bindings before child bindings, adds `subagent_announce` to mediated completion handling, and adds routing regression tests.

## Best Possible Solution

Land the requester-session routing fix after redacted live Telegram proof, and either remove the mediated-delivery hunk or complete it with message-tool-only guidance/runtime wiring before merging.

## Reproduction Assessment

No live current-main reproduction was run in this read-only review. Source inspection gives a high-confidence path: embedded runs can build tools with the Telegram sandbox peer key, `sessions_spawn` records that key as the requester, and completion origin currently checks child bindings before requester bindings.

## Solution Assessment

Partly: using the already-threaded `runSessionKey` for spawn ownership and checking requester bindings first is the narrow fix for the central bug. The added `subagent_announce` mediated-delivery change is not complete without message-tool-only guidance or equivalent runtime wiring.

## Review Findings
