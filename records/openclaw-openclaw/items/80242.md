## Summary

Keep open: the patch is a focused fix for a source-reproducible P1 subagent completion routing bug, but this external PR still lacks the required after-fix real Telegram proof before merge.

## What This Changes

The PR separates sandbox/policy session ownership from completion routing ownership for `sessions_spawn`, prefers requester-bound completion routes before child-bound routes, and adds focused regression tests.

## Best Possible Solution

Land this routing-owner fix after real Telegram proof shows a subagent spawned from the originating parent session reports completion back to that parent while sandbox policy remains scoped to the channel peer session.

## Reproduction Assessment

Yes from source inspection, though not from a live Telegram run in this review. Current main passes the sandbox/channel peer key into `sessions_spawn` registration and resolves child bindings before requester bindings, matching the linked wrong-session delivery path.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: keep `agentSessionKey` for sandbox and policy, add `completionOwnerKey` for completion ownership, and prefer requester binding before child fallback. The remaining blocker is real behavior proof, not a different implementation shape.
