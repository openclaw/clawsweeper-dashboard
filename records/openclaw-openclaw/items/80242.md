## What This Changes

The PR changes subagent spawn ownership to prefer `runSessionKey`, makes completion-origin routing prefer requester bindings before child bindings, and adds focused regression tests for requester-key and binding precedence.

## Best Possible Solution

Land the narrow routing fix after redacted live Telegram or runtime proof shows completion announces returning to the originating parent session, while keeping mediated-delivery policy work in https://github.com/openclaw/openclaw/pull/80223 separate.

## Reproduction Assessment

Yes, source-reproducible: current main passes the sandbox/channel peer key into `sessions_spawn` ownership and checks child bindings before requester bindings. I did not run a live Telegram setup.

## Solution Assessment

Yes, the proposed code direction is the narrow maintainable fix: preserve the live run session as requester ownership and make requester binding authoritative before child fallback. The remaining blocker is proof, not a concrete code defect in the patch.
