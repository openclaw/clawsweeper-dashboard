## Summary

Keep open: this is a focused bug-fix PR for a real source-visible subagent completion routing bug, and current main has not implemented the completion-owner-key part of the fix. The patch direction looks correct, but external contributor real Telegram behavior proof is still missing before merge.

## What This Changes

The PR separates subagent completion ownership from the channel sandbox session and changes completion routing to prefer requester-session bindings before child-session fallback.

## Best Possible Solution

Land this routing fix after real Telegram proof shows a subagent spawned from the originating parent session reports completion back to that parent while the channel peer session remains sandbox/policy-scoped.

## Reproduction Assessment

Yes from source inspection, though not from a live Telegram run in this review. Current main passes the sandbox/channel peer key into sessions_spawn registration and resolves child bindings before requester bindings, matching the linked wrong-session delivery report.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix: keep agentSessionKey for sandbox and policy, add completionOwnerKey for completion ownership, and prefer requester binding before child fallback; the remaining blocker is proof, not a different implementation shape.
