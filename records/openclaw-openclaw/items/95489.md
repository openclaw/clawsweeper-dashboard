## Summary

Keep open: current main and the latest release still have the source-level path where `claude-cli` generic failure text can be wrapped as normal visible output, causing the fallback classifier to treat the attempt as complete instead of advancing the configured fallback chain.

## What This Changes

The issue reports that a `claude-cli` out-of-credits failure is surfaced as final assistant text instead of triggering configured model fallbacks.

## Best Possible Solution

Add a narrow fallback classification for exact generic external runner failure text from CLI results before committed delivery, while preserving normal visible-output and side-effect guards.

Yes at source level: current main wraps CLI `output.text` as a normal payload and the fallback classifier returns `null` before error classification when that payload is visible. I did not run a live exhausted Claude Code account.

## Solution Assessment

Yes, a narrow classifier or CLI result-shaping fix is the best path; changing fallback ordering, config, or provider defaults would be broader than needed.
