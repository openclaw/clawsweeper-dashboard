## Summary

Keep open: current main and v2026.6.9 still short-circuit a normal visible claude-cli generic failure payload before fallback classification, and several open PRs target the issue but none has merged.

## What This Changes

The issue reports that claude-cli out-of-credits failure text is delivered as a final assistant response instead of advancing the configured model fallback chain.

## Best Possible Solution

Land one narrow classifier or CLI result-shaping fix that treats only the exact generic external runner failure payload as fallback-worthy while preserving normal visible-output, delivery, hook-block, abort, and side-effect guards.

## Reproduction Assessment

Yes, at source level: current main can wrap CLI `output.text` as a normal visible payload, and the fallback classifier returns null on visible non-error payloads before error classification. I did not run a live exhausted Claude Code account.

## Solution Assessment

Yes: a narrow classifier or CLI result-shaping fix is the best path; changing fallback ordering, config, or provider defaults would be broader than needed.
