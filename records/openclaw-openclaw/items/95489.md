## Summary

Keep open: current main and the latest release still have the source-level bypass, and open closing PRs now own the fix path rather than making the issue closeable before merge.

## What This Changes

The issue reports that Claude CLI out-of-credits generic failure output is treated as a final visible response instead of advancing configured model fallbacks.

## Best Possible Solution

Land one narrow canonical classifier fix that treats the exact generic external runner failure text as fallback-worthy only while preserving no-retry guards, then close this issue and supersede duplicate PRs.

## Reproduction Assessment

Yes at source level: current main's CLI runner turns output text into a normal payload, and the fallback classifier returns null for visible non-error payloads before error classification. I did not run a live exhausted Claude account in this read-only review.

## Solution Assessment

Yes: the best fix is a narrow classifier or CLI result-shaping change that uses the existing fallback contract instead of changing fallback ordering or configuration. The immediate maintainer path is to review and land one of the open closing PRs, then close this issue.
