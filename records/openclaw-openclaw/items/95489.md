## Summary

Keep open: current main still treats a `claude-cli` generic runner failure emitted as normal visible text as a completed embedded result, so the configured fallback chain can be skipped before any fallback candidate runs.

## What This Changes

The issue requests that `claude-cli` out-of-credits failures advance configured model fallbacks instead of returning the generic runner failure text as the final assistant reply.

## Best Possible Solution

Add a narrow fallback classification for exact generic external runner failure text from CLI results before committed delivery, while preserving normal visible-output and side-effect guards.

## Reproduction Assessment

Yes at source level: current main builds a normal visible payload from CLI `output.text`, and the fallback classifier returns `null` for that payload before provider-error classification. I did not run a live exhausted Claude Code account.

## Solution Assessment

Yes, the maintainable fix is narrow: classify the exact generic external runner failure as fallback-worthy at the CLI/result-classifier boundary, without changing fallback ordering, config, or side-effect replay policy.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/95489. Current main can treat the exact `GENERIC_EXTERNAL_RUN_FAILURE_TEXT` emitted by a `claude-cli` failure as a normal visible assistant payload, so `classifyEmbeddedAgentRunResultForModelFallback` returns `null` and `runWithModelFallback` completes instead of trying configured fallbacks. Implement a narrow repair at the CLI result/classifier boundary: the exact generic external runner failure should become fallback-worthy only when there is no committed outbound/message-tool delivery and no hook block, while ordinary visible assistant replies, deliberate silent replies, and side-effecting results must remain non-replayable. Prefer existing helpers such as `isGenericExternalRunFailureText`; do not add config/env options, change fallback ordering, weaken side-effect guards, or edit CHANGELOG.md. Add regression coverage for the visible non-error generic failure text and for a normal visible reply that must not fallback. Be aware of adjacent open PR https://github.com/openclaw/openclaw/pull/95400, which touches the same classifier for `isError` usage-limit payloads.
