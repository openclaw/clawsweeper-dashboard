## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not pool clients with one-shot request callbacks:** `src/gateway/call.ts:682`
  - body: `GatewayClient` stores the `onHelloOk` callback from construction and invokes it when a connection handshake completes. Reusing an existing client here means a later call with the same method/auth keeps the first call's closure; `client.start()` can run the old request path while the new promise never issues its request and eventually times out. Keep per-call clients or redesign pooling around a reusable connection with per-request dispatch.
  - confidence: 0.93
- **[P1] Preserve gateway request cancellation hooks:** `src/gateway/call.ts:711-714`
  - body: Current main's gateway call path accepts `signal`, `onAccepted`, and `onSignalAbort` and forwards the relevant pieces to `client.request`. This PR drops those options from the request call, breaking existing cancellation/progress behavior for callers that rely on aborting or observing accepted gateway requests.
  - confidence: 0.91
- **[P1] Keep repair-capable doctor checks on the default path:** `src/flows/doctor-health-contributions.ts:892`
  - body: `doctor:skills` currently runs `maybeRepairSkillReadiness`, and the docs advertise that `doctor --fix` can repair unavailable skills. Marking it `deepOnly` means the default `openclaw doctor --fix` no longer performs that existing repair unless users know to add `--deep`, which is a behavior break rather than a transparent performance optimization.
  - confidence: 0.88
- **[P2] Keep OAuth shadow cleanup after sidecar migration:** `src/commands/doctor/repair-sequencing.ts:138-150`
  - body: The sidecar migration removes legacy `oauthRef` fields and writes inline credentials; stale shadow cleanup intentionally skips profiles while those refs are still present. Running both in the same `Promise.all` lets cleanup scan before migration finishes, leaving a stale OAuth shadow that the current ordered sequence removes in one doctor pass.
  - confidence: 0.86
