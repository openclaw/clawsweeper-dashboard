## Summary

Keep open: current main improves `sessions_send` timeout handling and has adjacent durable queues, but it does not provide the requested persistent agent-to-agent inbox with at-least-once retry and acknowledgement semantics.

## What This Changes

Request adds a durable agent-to-agent message queue/inbox with retry, acknowledgement, and idempotency so `sessions_send` failures or timeouts do not lose messages.

## Best Possible Solution

Define the first-party A2A delivery contract, likely aligned with #63789, before wiring durable queue/retry behavior into `sessions_send`.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main lacks a durable `sessions_send` inbox, while existing tests cover delayed reply and terminal-timeout behavior rather than at-least-once delivery.

## Solution Assessment

Unclear as an implementation path: the requested direction is plausible, but a full persistent queue is not the narrowest safe change until maintainers settle the A2A handoff and acknowledgement contract.
