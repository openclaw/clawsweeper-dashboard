## Summary

Keep open. Current main still lacks a general durable A2A inbox for `sessions_send`; the existing durable session-delivery queue is wired to restart-continuation recovery, while ordinary sends still depend on direct Gateway agent dispatch and timeout/error reporting.

## What This Changes

Add a persistent, acknowledged, retried message queue for `sessions_send` so target agents can recover missed A2A messages after timeouts or restarts.

## Best Possible Solution

Define a core reliable A2A delivery contract that persists accepted `sessions_send` work before dispatch and drains it with explicit acknowledgement, retry, dedupe, permission, and provenance behavior.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence reproduction: ordinary `sessions_send` dispatches directly to Gateway `agent`, and timeout/error paths do not expose a durable queue entry or acknowledgement state.

## Solution Assessment

Unclear until maintainer design. A persistent queue is the right class of solution, but the proposed per-agent file plus BOOT polling model should likely become a core Gateway/session contract instead of an agent-prompt workaround.
