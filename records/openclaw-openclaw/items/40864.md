## Summary

Keep open. Current main still lacks a durable A2A inbox or persistent `sessions_send` acceptance queue; the existing durable session-delivery queue is scoped to restart-continuation recovery, so the earlier ClawSweeper keep-open assessment still holds on `9891f304222bf77bf439c269b56e123bf01aaee5`.

## What This Changes

Add persistent, acknowledged, retryable delivery for `sessions_send`/agent-to-agent messages so accepted sends are not lost across timeouts or restarts.

## Best Possible Solution

Define a core durable A2A delivery contract that persists accepted `sessions_send` work before dispatch and drains it with explicit acknowledgement, retry, dedupe, policy, and provenance behavior.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Ordinary `sessions_send` dispatches through Gateway `agent`, and timeout/error paths expose no durable queue entry or acknowledgement state.

## Solution Assessment

Unclear until maintainer design. A persistent queue is the right class of solution, but the proposed per-agent file plus BOOT polling model should likely become a core Gateway/session delivery contract instead of an agent-prompt workaround.
