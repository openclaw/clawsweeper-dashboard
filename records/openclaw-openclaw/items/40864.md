## Summary

Keep open. Current main still lacks a durable inbox or acknowledged retry contract for ordinary `sessions_send`; the durable session-delivery queue is scoped to restart-continuation recovery, and A2A announce failures are only logged after a best-effort send.

## What This Changes

Request a persistent message queue/inbox for `sessions_send` so timed-out or failed A2A sends can be retried and acknowledged.

## Best Possible Solution

Define a Gateway-owned reliable A2A delivery contract that persists accepted `sessions_send` work before dispatch and drains it with explicit ack, retry, dedupe, permission, and provenance behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows ordinary `sessions_send` dispatches directly through Gateway `agent`, and timeout/error results do not expose durable queue or acknowledgement state.

## Solution Assessment

Unclear until maintainer design. A persistent queue is the right class of solution, but the proposed per-agent inbox plus BOOT polling should likely become a core Gateway/session contract rather than an agent-prompt workaround.
