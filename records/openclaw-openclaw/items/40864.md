## Summary

Keep open: current main still lacks the requested durable `sessions_send` inbox/queue with at-least-once retry and acknowledgement semantics. Existing queue code is scoped to restart-continuation recovery, while the active A2A handoff work is still a proposal and does not implement this request yet.

## What This Changes

Requests a durable `sessions_send` inbox/queue with retry, acknowledgements, and polling so A2A messages survive timeouts and restarts.

## Best Possible Solution

Define the first-party A2A handoff/ack contract first, then add durable `sessions_send` handoff storage, retry, acknowledgement, docs, and focused tests without folding in the higher-level meeting lifecycle.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Current source shows direct `sessions_send` dispatch plus timeout/A2A follow-up, not a persistent inbox/retry contract.

## Solution Assessment

Unclear as a direct implementation request. A durable queue is plausible, but the maintainable path is to sequence it after the A2A handoff contract in https://github.com/openclaw/openclaw/pull/63789 and keep plugin-style message buses on the ClawHub/community path unless core API gaps remain.
