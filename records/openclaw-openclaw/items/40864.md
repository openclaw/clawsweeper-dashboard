## Summary

Keep open: current main still lacks the requested durable `sessions_send` inbox with at-least-once retry and acknowledgement semantics. The existing durable session delivery queue is scoped to restart continuation recovery, and the related #63789 item is an open draft protocol proposal rather than an implementation.

## What This Changes

The issue requests a persistent per-agent message queue for `sessions_send` so failed or timed-out agent-to-agent messages can be retried, acknowledged, and recovered.

## Best Possible Solution

Define the first-party A2A handoff contract, likely informed by #63789, then add durable retry/acknowledgement behavior in the generic session delivery path with focused tests and docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source and docs inspection show current `sessions_send` has timeout-oriented direct delivery and A2A announce handling, not the requested persistent inbox contract.

## Solution Assessment

Unclear as a direct implementation request. A durable queue is plausible, but the maintainable path is to settle the A2A handoff and ack semantics before wiring persistent retry behavior into `sessions_send`.
