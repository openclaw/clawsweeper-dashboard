## Summary

Keep open: the explicit Feishu filename override bug still appears valid on current main, but this PR is not mergeable as-is because it lacks real behavior proof and introduces security-sensitive unrelated changes.

## What This Changes

The PR threads filename metadata through outbound and Feishu send paths, adds Feishu local upload tests, and also changes workflows, CODEOWNERS, local docs/scripts, skills, an OpenAI WebSocket helper, and uv.lock.

## Best Possible Solution

Replace this branch with a focused Feishu/outbound patch that carries explicit filename overrides through the guarded media loader, preserves upload sanitization and mediaLocalRoots, and leaves CI/release/local tooling/skills out.

## Reproduction Assessment

Yes. Current main exposes filename on the message tool, but source inspection shows send(action: "send") builds payloads without filename and Feishu media sends therefore cannot receive the override; I did not run a credentialed live Feishu send.

## Solution Assessment

No. Filename threading is the right direction, but this PR bypasses existing media guards, removes upload hardening, and bundles unrelated security-sensitive repository changes; the safer solution is a narrow patch through the existing loader/upload contracts.
