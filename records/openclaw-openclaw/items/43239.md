## Summary

Keep open. The Feishu filename goal is plausible, but this branch is not mergeable: it introduces Feishu media regressions, bypasses local media safeguards, deletes CODEOWNERS, commits a gateway token, includes unrelated local scripts/skills/workflow churn, and provides no real behavior proof.

## What This Changes

The PR threads filename values through outbound Feishu sends while also changing CI workflows, adding local docs/scripts/skills, adding OpenAI WebSocket header plumbing, and adding a uv lockfile.

## Best Possible Solution

Narrow the change to filename plumbing only, keep media reads behind loadWebMedia/mediaLocalRoots and sanitizeFileNameForUpload, remove unrelated workflows/docs/scripts/skills/secrets, and prove the Feishu upload in a real setup.

## Reproduction Assessment

Yes for the PR blockers by source inspection: on the PR head, non-image mediaUrl reaches uploadFileFeishu as a raw string and absolute local paths bypass mediaLocalRoots. I did not run live Feishu proof because this is a read-only review.

## Solution Assessment

No. The maintainable fix is a smaller Feishu/outbound contract change that preserves caller filenames without weakening media loading, filename sanitization, CODEOWNERS, or repository hygiene.
