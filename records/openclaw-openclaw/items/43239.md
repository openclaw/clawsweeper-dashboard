## What This Changes

## Best Possible Solution

Land a focused Feishu/outbound fix that carries filename overrides into sendMediaFeishu through the existing guarded media path, retains sanitizeFileNameForUpload and mediaLocalRoots enforcement, and drops unrelated repo/workflow/skill/script changes.

## Reproduction Assessment

Yes, source-reproducible. Current main exposes a filename parameter, but action send is not hydrated as an attachment action, the outbound context has no filename field, and the Feishu action send path does not pass fileName to sendMediaFeishu; I did not run a live Feishu send.

## Solution Assessment

No. Filename threading is the right direction, but this branch bypasses existing media guards, removes upload hardening, and bundles unrelated security-sensitive changes; the maintainable solution is a narrow patch through the current loader/upload contracts.
