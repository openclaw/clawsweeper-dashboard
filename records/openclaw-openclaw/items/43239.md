## Summary

Keep open, but do not merge as-is. The Feishu filename problem is plausible and worth fixing, yet this branch mixes that fix with unrelated security-sensitive churn, has no real behavior proof, and introduces clear Feishu media and repository-safety regressions.

## What This Changes

The PR threads a `filename` field through message sending and Feishu outbound media handling while also changing Feishu upload behavior, adding tests/skills/local docs/scripts, and modifying CI/CODEOWNERS files.

## Best Possible Solution

Land a narrow Feishu/message-send patch that propagates `filename` through the existing payload and adapter contracts while preserving guarded media loading and UTF-8-safe upload sanitization.

## Reproduction Assessment

Yes for the PR-introduced blockers: source inspection shows remote non-image URLs become filesystem streams and local non-image paths bypass `mediaLocalRoots`. The original filename preservation bug is plausible from the current send contract, but this PR does not provide live Feishu reproduction or after-fix proof.

## Solution Assessment

No. The filename plumbing is directionally right, but this branch is not the best fix because it bypasses media validation, drops upload sanitization, and carries unrelated security-sensitive files; a narrow patch on the existing loader/sanitizer path is safer.
