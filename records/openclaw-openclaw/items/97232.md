## Summary

Keep open: the stranded pending-final bug is real, but this PR is not merge-ready because the new terminal replay path can fall back to stale session route metadata and there is no real after-fix behavior proof.

## What This Changes

This PR adds restart recovery that directly sends captured pending final text for terminal main sessions via `message.action` and clears or records pending-final state after the attempt.

## Best Possible Solution

Replay terminal pending finals only through a valid saved pending or restart recovery delivery context, leave contextless entries pending or diagnostic-failed without sending, and add focused regression coverage plus redacted real behavior proof before merge.

Yes, at source level. Current main skips terminal non-running pending-final sessions, and on the PR branch a terminal pending final without saved delivery context can send via legacy session fallback; I did not run tests because this review is read-only.
