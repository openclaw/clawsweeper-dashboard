## Summary

Keep open: the feature may be useful, but the current PR is not mergeable because source inspection still shows an invalid duplicate gateway helper export, credential/auth-mode regressions, and no real behavior proof.

## What This Changes

Adds `gateway.auth.tokenExpiryHours`, in-process expiry warnings, an `openclaw auth rotate` CLI, config metadata/tests, and a small Matrix import-barrel cleanup.

## Best Possible Solution

Rework rotation to operate only on active plaintext token-mode configs, reject env/SecretRef-managed tokens, make restart/activation explicit, emit expiry warnings from the shared HTTP auth path, and prove the flow in a redacted real gateway run.

## Reproduction Assessment

Yes. Source inspection of PR head `c848796663343a57e135221be79537eb750c9a7b` reproduces the blocking defects, and current main confirms the env-substituted `sourceConfig`, config-first credential precedence, shared HTTP auth path, and restart-backed gateway config semantics they depend on.

## Solution Assessment

No. The requested feature direction may be useful, but this patch is not the best implementation because it changes auth mode/secret-source semantics, misses common HTTP paths, and does not prove live gateway behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Delete the duplicate browser origin helper:** `src/gateway/http-utils.ts:46-57`
  - body: This PR head declares `resolveHttpBrowserOriginPolicy` twice in the same module. The duplicate top-level export makes `http-utils` invalid before gateway HTTP auth can load, so keep one helper and rebase onto the current auth-helper split.
  - confidence: 0.96
- **[P1] Preserve the configured auth mode:** `src/commands/auth-rotate.ts:43`
  - body: Running `auth rotate` on a config using `password`, `none`, or `trusted-proxy` reaches this write and persists `mode: "token"`, silently changing the gateway auth strategy. Reject non-token modes or require an explicit mode change before writing a token.
  - confidence: 0.95
- **[P1] Reject env-managed tokens before writing config:** `src/commands/auth-rotate.ts:23-27`
  - body: This check only rejects refs visible in `snapshot.sourceConfig`; current main exposes `sourceConfig` after env substitution, and deployments that rely on `OPENCLAW_GATEWAY_TOKEN` with no config token also pass this check. Rotation can therefore persist a plaintext config token that takes precedence over the env-managed source.
  - confidence: 0.91
- **[P2] Emit expiry warnings from shared HTTP auth:** `src/gateway/http-auth-helpers.ts:35-40`
  - body: The warning runs only in `authorizeGatewayBearerRequestOrReply`, while normal HTTP routes use the shared HTTP auth utility path. Those common routes will not evaluate `tokenExpiryHours`, making the new config ineffective for ordinary gateway usage.
  - confidence: 0.87
- **[P2] Make rotated token activation explicit:** `src/commands/auth-rotate.ts:53-55`
  - body: This updates only CLI process memory and prints the new token immediately, but gateway auth config is restart-backed. A running gateway can still require the old token until restart completes, so users can switch too early and see unauthorized failures.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff changes gateway credential persistence and auth mode handling and has concrete mode-switch and secret-source risks.

Concerns:

- **[high] Token rotation can silently change auth mode:** `src/commands/auth-rotate.ts:43`
  - body: Persisting `mode: "token"` during rotation can replace password, none, or trusted-proxy auth without explicit operator consent, changing the gateway security posture and potentially breaking access.
  - confidence: 0.95
- **[medium] Env-managed tokens can become plaintext config secrets:** `src/commands/auth-rotate.ts:23`
  - body: When `OPENCLAW_GATEWAY_TOKEN` or an env-template token is the source of truth, the command can persist a new plaintext token into config, changing credential precedence and leaving a secret on disk unexpectedly.
