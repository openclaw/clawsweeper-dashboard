## Summary

Keep open. Current main does not implement the requested auth rotation/token-expiry feature, but this PR should not merge as-is because the head still has blocking build, auth-mode, secret-source, and proof gaps.

## What This Changes

The PR adds a gateway token-expiry warning config/state, an `openclaw auth rotate` CLI with tests/generated config metadata, and a small Matrix import-barrel cleanup.

## Best Possible Solution

Rework against current main's `http-auth-utils`/auth-resolve layout so rotation only updates active plaintext token-mode configs, refuses external secret sources, makes activation/restart semantics explicit, and proves the CLI plus warning path in a real gateway.

## Reproduction Assessment

Yes for the blocking PR findings by source inspection: the PR head shows the duplicate top-level helper and unconditional auth-mode write, and current main shows config-token precedence plus the shared HTTP auth path. I did not execute the branch because this is a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this implementation changes auth mode/source semantics, duplicates a helper, and does not match the current shared gateway HTTP auth layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the duplicate browser origin policy helper:** `src/gateway/http-utils.ts:46-57`
  - body: This file now declares `resolveHttpBrowserOriginPolicy` twice at top level. TypeScript/ESM will reject the duplicate implementation, so importing the HTTP auth utility module can fail before the new auth behavior runs.
  - confidence: 0.95
- **[P1] Preserve non-token auth modes during rotation:** `src/commands/auth-rotate.ts:43-44`
  - body: Running the new rotate command on a config using `password`, `none`, or `trusted-proxy` reaches this write and persists `mode: "token"`, silently changing the gateway auth strategy. Reject non-token modes or require an explicit mode change before writing a token.
  - confidence: 0.94
- **[P1] Reject env-managed tokens before writing config:** `src/commands/auth-rotate.ts:24-27`
  - body: This check only rejects SecretRef values already in config. If the deployment uses `OPENCLAW_GATEWAY_TOKEN` with no config token, the command writes a new plaintext config token; current auth resolution prefers config credentials, so future startups use the new file secret instead of the env-managed token.
  - confidence: 0.9
- **[P2] Run expiry warnings from the shared HTTP auth flow:** `src/gateway/http-auth-helpers.ts:35-40`
  - body: The warning is emitted only from `authorizeGatewayBearerRequestOrReply`, but normal current HTTP routes authenticate through the shared request auth helper. Token-authenticated requests on those paths would not evaluate `tokenExpiryHours`, making the config appear ineffective for common gateway usage.
  - confidence: 0.87
- **[P2] Clarify restart-gated token activation:** `src/commands/auth-rotate.ts:53-55`
  - body: The command prints the new token immediately after writing config and calls process-local expiry state in the short-lived CLI process. A running gateway can still require the old in-memory token until restart/reload completes, so scripts may switch too early and receive unauthorized responses.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff changes gateway credential persistence and auth mode behavior and still has concrete mode-switch and secret-source risks.

Concerns:

- **[high] Token rotation can silently change auth mode:** `src/commands/auth-rotate.ts:43`
  - body: Persisting `mode: "token"` during rotation can replace password, none, or trusted-proxy auth without explicit operator consent, changing the gateway security posture and potentially breaking access.
  - confidence: 0.94
- **[medium] Env-managed tokens can become plaintext config secrets:** `src/commands/auth-rotate.ts:24`
  - body: When `OPENCLAW_GATEWAY_TOKEN` is the source of truth and config has no token, the command can persist a new plaintext token into config, changing credential precedence and leaving a secret on disk unexpectedly.
  - confidence: 0.9
