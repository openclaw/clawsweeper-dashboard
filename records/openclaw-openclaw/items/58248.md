## Summary

Keep open, but this PR is not merge-ready: current main still lacks the requested auth-rotation/expiry feature, and the PR head has blocking correctness and credential-source issues.

## What This Changes

The PR adds a gateway token expiry warning config/state path and an `openclaw auth rotate` CLI that persists a generated gateway auth token.

## Best Possible Solution

Rework against current main so rotation only updates an already-active plaintext config token, rejects external secret sources and non-token modes, emits expiry warnings from the shared HTTP auth path, and documents the new user-facing workflow.

## Reproduction Assessment

Yes. The PR head visibly declares a duplicate top-level `resolveHttpBrowserOriginPolicy`, and the rotate command visibly writes `mode: "token"` without checking the prior auth mode or env-managed token source.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the narrowest maintainable solution because it breaks module loading and changes credential/auth-mode semantics implicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Remove the duplicate browser-origin policy declaration:** `src/gateway/http-utils.ts:46`
  - body: The PR head declares `resolveHttpBrowserOriginPolicy` twice in this ESM module. That prevents the HTTP auth module from loading/building, so keep one declaration before merging.
  - confidence: 0.96
- **[P1] Preserve non-token auth modes when rotating:** `src/commands/auth-rotate.ts:43`
  - body: Running `auth rotate` on a config using `password`, `none`, or `trusted-proxy` reaches this write and persists `mode: "token"`, silently changing the gateway auth strategy. Reject non-token modes or require an explicit mode change.
  - confidence: 0.94
- **[P1] Reject env-managed tokens before writing config:** `src/commands/auth-rotate.ts:23-34`
  - body: The command only checks the config value. If the deployment relies on `OPENCLAW_GATEWAY_TOKEN` with no config token, this path writes a plaintext config token, and current auth resolution gives config credentials precedence on startup/reload.
  - confidence: 0.9
- **[P2] Move expiry warnings into the shared HTTP auth path:** `src/gateway/http-auth-helpers.ts:35-40`
  - body: Current main authenticates common HTTP routes through the shared HTTP auth utility, but this warning only runs in the bearer helper. Put the expiry check in the shared path so `tokenExpiryHours` applies to normal gateway HTTP usage.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/cli/auth-cli.ts:18-20`
  - body: This adds a user-facing CLI command and config field without a `CHANGELOG.md` entry. Add a single-line entry under the active release section so the auth workflow is discoverable.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The patch changes gateway credential persistence and auth mode behavior, with concrete mode-switch and secret-source risks.

Concerns:

- **[high] Token rotation can silently change auth mode:** `src/commands/auth-rotate.ts:43`
  - body: Persisting `mode: "token"` during rotation can replace password, none, or trusted-proxy auth without explicit operator consent, changing the gateway security posture and potentially breaking access.
  - confidence: 0.94
- **[medium] Env-managed tokens can become plaintext config secrets:** `src/commands/auth-rotate.ts:23`
  - body: When `OPENCLAW_GATEWAY_TOKEN` is the source of truth and config has no token, the PR writes a new plaintext token into config, changing credential precedence and persisting a secret unexpectedly.
  - confidence: 0.9
