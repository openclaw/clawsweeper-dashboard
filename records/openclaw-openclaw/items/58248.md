Full review comments:

- **[P1] Delete the duplicate browser-origin helper:** `src/gateway/http-utils.ts:46`
  - body: The branch declares `resolveHttpBrowserOriginPolicy` twice in this module, so importing the gateway HTTP auth helper can fail before the gateway starts. Keep the current-main shared helper shape and remove the stale duplicate copy.
  - confidence: 0.96
- **[P1] Preserve the configured auth mode:** `src/commands/auth-rotate.ts:43`
  - body: `auth rotate` always writes `mode: "token"`, which silently changes password, none, or trusted-proxy deployments into token auth. Reject rotation unless the existing mode is token, or update only the token without changing the selected auth strategy.
  - confidence: 0.95
- **[P1] Reject env-managed gateway tokens:** `src/commands/auth-rotate.ts:24-27`
  - body: The guard only checks the snapshot value, but config snapshots have already resolved `${ENV}` templates and a deployment may use `OPENCLAW_GATEWAY_TOKEN` with no config token. In those cases this command can persist a new plaintext token and make config-first auth ignore the env-managed source on future starts.
  - confidence: 0.9
- **[P2] Move the expiry warning into shared HTTP auth:** `src/gateway/http-auth-helpers.ts:35-39`
  - body: The warning is emitted only from `authorizeGatewayBearerRequestOrReply`, while current main routes most HTTP API auth through the shared request auth utility. Put the consume/log check on the shared successful token-auth path so `tokenExpiryHours` works for normal gateway traffic.
  - confidence: 0.86
