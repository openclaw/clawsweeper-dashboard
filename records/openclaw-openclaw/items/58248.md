## Summary

Keep open. Current main still lacks the proposed `auth rotate` command and `tokenExpiryHours` config, but this PR is not merge-ready because the patch changes gateway auth mode and credential-source semantics and wires the warning into a stale/non-shared HTTP auth path; prior bot review comments raised the same core blockers.

## What This Changes

The PR adds a `gateway.auth.tokenExpiryHours` warning path and an `openclaw auth rotate` CLI that persists a new gateway token, plus generated config metadata and tests.

## Best Possible Solution

Rework the feature so rotation only updates an already-active plaintext config token, rejects or explicitly handles non-token modes and env/SecretRef-managed secrets, emits expiry warnings from the current shared HTTP auth path, documents restart activation, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. From the PR diff, configure `gateway.auth.mode` as `password`, `none`, or `trusted-proxy` and run the proposed rotate command; it persists `mode: "token"`. A second high-confidence path is a deployment using `OPENCLAW_GATEWAY_TOKEN` or an env-template token, where the command can persist a new plaintext config token.

## Solution Assessment

No. The feature direction is useful, but this patch is not the best solution because it bypasses current credential-source semantics, misses the current shared HTTP auth path, and reports a newly printed token before restart-class config changes are active.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-token auth modes when rotating:** `src/commands/auth-rotate.ts:44`
  - body: This write forces `gateway.auth.mode` to `"token"`, so running `auth rotate` on `password`, `none`, or `trusted-proxy` configs silently changes the gateway auth strategy. Reject non-token modes or require an explicit mode change instead.
  - confidence: 0.94
- **[P1] Reject env-managed tokens before writing config:** `src/commands/auth-rotate.ts:21-31`
  - body: The command only checks the config token field. With `OPENCLAW_GATEWAY_TOKEN` and no config token, or an authored `${OPENCLAW_GATEWAY_TOKEN}` that has already been substituted in the snapshot, this writes a plaintext config token that can override the external secret source on future startup.
  - confidence: 0.9
- **[P2] Move expiry warnings into the shared HTTP auth path:** `src/gateway/http-auth-helpers.ts:30-40`
  - body: The warning is emitted only from this bearer helper, but current main routes common endpoints through `checkGatewayHttpRequestAuth` and `authorizeGatewayHttpRequestOrReply` in `http-auth-utils.ts`. Put the warning in the shared auth path so `tokenExpiryHours` applies to normal gateway requests.
  - confidence: 0.86
- **[P2] Account for restart before printing the rotated token:** `src/commands/auth-rotate.ts:53-55`
  - body: `gateway.auth.token` is a restart-class config change, but the command prints success and the new token immediately after writing config. A running gateway may still accept only the old token, so this should trigger/wait for activation or clearly warn that restart is required.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/cli/auth-cli.ts:18`
  - body: This adds a user-facing CLI command and config field without a `CHANGELOG.md` entry. Add a single-line entry under the active release section so the new auth workflow is discoverable.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The patch changes gateway credential persistence and auth-mode behavior, with concrete mode-switch and secret-source risks.

Concerns:

- **[high] Token rotation can silently change auth mode:** `src/commands/auth-rotate.ts:44`
  - body: Persisting `mode: "token"` during rotation can replace password, none, or trusted-proxy auth without explicit operator consent, changing the gateway security posture and potentially breaking access.
  - confidence: 0.94
- **[medium] Env-managed tokens can become plaintext config secrets:** `src/commands/auth-rotate.ts:21`
  - body: When an env var or env-template is the token source of truth, the proposed command can write a new plaintext token into config, changing credential precedence and persisting a secret unexpectedly.
  - confidence: 0.9
