Full review comments:

- **[P1] Delete the duplicate origin-policy helper:** `src/gateway/http-utils.ts:46-57`
  - body: `resolveHttpBrowserOriginPolicy` is declared twice in this module on PR head, which makes the ESM module invalid and can break gateway HTTP imports before the feature runs. Keep one implementation only after rebasing onto current main.
  - confidence: 0.96
- **[P1] Reject non-token auth modes before rotating:** `src/commands/auth-rotate.ts:43`
  - body: This assignment silently converts password, none, or trusted-proxy configs to token auth. `auth rotate` should fail unless the current effective mode is already token, or it needs an explicit maintainer-approved migration path.
  - confidence: 0.95
- **[P1] Reject env-managed tokens before persisting:** `src/commands/auth-rotate.ts:24-27`
  - body: The SecretRef check is not enough for env-managed deployments: current config snapshots may already contain resolved `${VAR}` values, and an unset config token can still be supplied by `OPENCLAW_GATEWAY_TOKEN`. Persisting a new token here changes the source of truth and can put a plaintext credential on disk.
  - confidence: 0.92
- **[P2] Move the expiry warning into shared HTTP auth:** `src/gateway/http-auth-helpers.ts:35-39`
  - body: This warning only runs through `authorizeGatewayBearerRequestOrReply`, while common HTTP routes use the shared gateway HTTP auth wrapper. As written, `gateway.auth.tokenExpiryHours` can do nothing for normal authenticated gateway traffic.
  - confidence: 0.86
- **[P2] Warn that the new token is restart-backed:** `src/commands/auth-rotate.ts:54-55`
  - body: The command prints the rotated token immediately after writing config, but current main treats `gateway.auth.token` and `gateway.auth.mode` changes as restart-backed. Scripts that switch to the printed token before the gateway restarts can hit unauthorized failures.
  - confidence: 0.83
