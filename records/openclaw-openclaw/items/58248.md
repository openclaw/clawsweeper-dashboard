## Summary

Keep open. Current main does not implement the new gateway token-expiry warning or `openclaw auth rotate`, so the PR is not obsolete; however the proposed patch is not safe to merge as-is because it changes auth/secret behavior and has concrete review blockers in the token-rotation flow.

## What This Changes

The PR adds a `gateway.auth.tokenExpiryHours` warning path, in-process token issue state, generated config metadata, tests, and a new `openclaw auth rotate` CLI that writes a fresh gateway token to config.

## Best Possible Solution

Rework the feature against current main’s auth architecture: rotate only an already-active, plaintext-config token unless the user explicitly opts into a mode/source change; keep env and SecretRef-managed tokens external; emit expiry warnings from the shared HTTP auth path; and either trigger/wait for a gateway restart or clearly report that the printed token is inactive until restart completes.

## Reproduction Assessment

Yes. The main blocker can be reproduced from the PR diff by configuring `gateway.auth.mode` as `password`, `none`, or `trusted-proxy` and running the proposed rotate command; the command writes `mode: "token"` into config. A second high-confidence path is setting `OPENCLAW_GATEWAY_TOKEN` with no config token, where the PR still persists a new plaintext token.

## Solution Assessment

No. The direction is useful, but this patch is not the best way to solve it because it bypasses current credential-source semantics, only partially covers the current HTTP auth flow, and does not account for restart-only gateway auth changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-token auth modes when rotating:** `src/commands/auth-rotate.ts:44`
  - body: `auth rotate` writes `mode: "token"` unconditionally. On a gateway configured for `password`, `none`, or `trusted-proxy`, running the helper silently changes the auth strategy on disk; reject non-token modes or preserve the existing mode instead.
  - confidence: 0.94
- **[P1] Reject env-managed tokens before writing config:** `src/commands/auth-rotate.ts:21-31`
  - body: The command only checks `gateway.auth.token` in the config. If the deployment uses `OPENCLAW_GATEWAY_TOKEN` with no config token, rotation persists a new plaintext config token and changes the source of truth away from env-managed credentials.
  - confidence: 0.9
- **[P2] Move expiry warnings into the shared HTTP auth path:** `src/gateway/http-auth-helpers.ts:32-42`
  - body: The warning is wired only through the PR's bearer helper, but current main routes normal HTTP endpoints through `authorizeGatewayHttpRequestOrReply` / `checkGatewayHttpRequestAuth`. As written, many authenticated gateway requests would never evaluate `tokenExpiryHours`.
  - confidence: 0.86
- **[P2] Account for restart before printing the rotated token:** `src/commands/auth-rotate.ts:53-55`
  - body: The command writes `gateway.auth.token`, calls an in-process state reset in the short-lived CLI process, and immediately prints success. Gateway auth config changes are restart-class changes on current main, so users can switch clients to a token the running gateway has not loaded yet.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR changes gateway auth and token storage behavior and introduces concrete auth-mode and secret-source risks.

Concerns:

- **[high] Token rotation can silently change auth mode:** `src/commands/auth-rotate.ts:44`
  - body: Writing `mode: "token"` during rotation can replace password, none, or trusted-proxy auth with token auth without explicit operator consent, changing the gateway security posture and potentially breaking access.
  - confidence: 0.94
- **[medium] Env-managed token deployments can be converted to plaintext config:** `src/commands/auth-rotate.ts:21`
  - body: When `OPENCLAW_GATEWAY_TOKEN` is the source of truth and config has no token, the PR still writes a new plaintext token into the config file, changing credential precedence and persisting a secret unexpectedly.
  - confidence: 0.9
