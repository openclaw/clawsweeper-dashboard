Full review comments:

- **[P1] Remove the duplicate browser-origin helper:** `src/gateway/http-utils.ts:65`
  - body: This file already declares `resolveHttpBrowserOriginPolicy` above, and this second exported declaration makes the module invalid (`Identifier ... has already been declared`). Any HTTP auth path importing this module can fail to load.
  - confidence: 0.96
- **[P1] Preserve the configured auth mode:** `src/commands/auth-rotate.ts:43`
  - body: `auth rotate` always writes `mode: "token"`, so running it against password, none, or trusted-proxy configs silently changes the gateway auth strategy. Reject non-token modes or preserve the existing mode instead of rewriting it.
  - confidence: 0.97
- **[P1] Reject env-managed gateway tokens:** `src/commands/auth-rotate.ts:23-27`
  - body: The command only checks `gateway.auth.token` in config, so a deployment using `OPENCLAW_GATEWAY_TOKEN` with no config token still gets a new plaintext token written to config. Because config tokens win over env tokens, future startups can ignore the intended env-managed credential.
  - confidence: 0.94
- **[P2] Move the expiry warning to the shared HTTP auth path:** `src/gateway/http-auth-helpers.ts:35-40`
  - body: This warning runs only in the older bearer helper, while current main's normal HTTP auth goes through `checkGatewayHttpRequestAuth` in `src/gateway/http-auth-utils.ts`. As written, `gateway.auth.tokenExpiryHours` can miss normal authenticated gateway traffic.
  - confidence: 0.88
- **[P2] Do not reset gateway expiry state in the CLI process:** `src/commands/auth-rotate.ts:53`
  - body: `setGatewayTokenIssuedAtNow()` updates process-local module state, but `openclaw auth rotate` exits after writing config. The running gateway's warning clock is unchanged unless the gateway process itself resets or restarts.
  - confidence: 0.86
