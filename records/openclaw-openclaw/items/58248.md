- **[P1] Remove the duplicate browser-origin helper:** `src/gateway/http-utils.ts:65`
  - body: This branch declares `resolveHttpBrowserOriginPolicy` twice in the same module, so importing `src/gateway/http-utils.ts` fails with a duplicate identifier. Rebase onto current main's `src/gateway/http-auth-utils.ts` shape and keep one implementation.
  - confidence: 0.98
- **[P1] Preserve the configured auth mode:** `src/commands/auth-rotate.ts:43`
  - body: `auth rotate` always writes `mode: "token"`, so running it on a gateway configured for password, none, or trusted-proxy auth silently changes the auth strategy. Reject non-token modes or preserve the existing mode instead of rewriting it.
  - confidence: 0.97
- **[P1] Reject env-managed gateway tokens:** `src/commands/auth-rotate.ts:23-27`
  - body: The command only inspects `gateway.auth.token`, so a deployment using `OPENCLAW_GATEWAY_TOKEN` with no config token will get a new plaintext config token written and future auth resolution will prefer that over the env-managed secret. Detect env-managed credentials and fail unless maintainers approve an explicit migration path.
  - confidence: 0.94
- **[P2] Move the expiry warning to the shared HTTP auth path:** `src/gateway/http-auth-helpers.ts:35-40`
  - body: The warning runs only in the old bearer helper, while current main routes normal HTTP auth through the shared `checkGatewayHttpRequestAuth` path. Put the consume/log gate on the shared successful token-auth path so the new option works for normal gateway traffic.
  - confidence: 0.9
- **[P2] Do not reset gateway expiry state in the CLI process:** `src/commands/auth-rotate.ts:53`
  - body: `setGatewayTokenIssuedAtNow()` updates module-local memory in the short-lived CLI process, not the running gateway process that emits expiry warnings. Either reset state in the gateway process through a real reload/restart flow or remove this misleading no-op.
  - confidence: 0.9
