## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove the duplicate browser origin helper:** `src/gateway/http-utils.ts:46-57`
  - body: This branch declares `resolveHttpBrowserOriginPolicy` twice in the same module. TypeScript/ESM will reject the duplicate top-level export, so importing the HTTP auth utility module can fail before the new auth behavior runs.
  - confidence: 0.95
- **[P1] Preserve existing auth mode during rotation:** `src/commands/auth-rotate.ts:43-44`
  - body: Running `auth rotate` on a config using `password`, `none`, or `trusted-proxy` reaches this write and persists `mode: "token"`, silently changing the gateway auth strategy. Reject non-token modes or require an explicit mode change before writing a token.
  - confidence: 0.94
- **[P1] Reject env-managed tokens before writing config:** `src/commands/auth-rotate.ts:24-27`
  - body: This check only rejects SecretRef values already in config. If the deployment uses `OPENCLAW_GATEWAY_TOKEN` with no config token, the command writes a plaintext config token; current auth resolution prefers config credentials, so future startups use the file secret instead of the env-managed token.
  - confidence: 0.9
- **[P2] Emit expiry warnings from the shared HTTP auth path:** `src/gateway/http-auth-helpers.ts:35-40`
  - body: The warning is emitted only from `authorizeGatewayBearerRequestOrReply`, but current normal HTTP routes authenticate through the shared HTTP auth utility path. Those routes will not evaluate `tokenExpiryHours`, making the config ineffective for common gateway usage.
  - confidence: 0.87
- **[P2] Clarify restart-gated token activation:** `src/commands/auth-rotate.ts:53-55`
  - body: The command prints the new token immediately after writing config and updates only process-local CLI memory. A running gateway can still require the old in-memory token until restart or reload completes, so users or scripts may switch too early and get unauthorized responses.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff changes gateway credential persistence and auth mode handling, and still has concrete mode-switch and secret-source risks.

Concerns:

- **[high] Token rotation can silently change auth mode:** `src/commands/auth-rotate.ts:43`
  - body: Persisting `mode: "token"` during rotation can replace password, none, or trusted-proxy auth without explicit operator consent, changing the gateway security posture and potentially breaking access.
  - confidence: 0.94

  - body: When `OPENCLAW_GATEWAY_TOKEN` is the source of truth and config has no token, the command can persist a new plaintext token into config, changing credential precedence and leaving a secret on disk unexpectedly.
