Overall confidence: 0.94

Full review comments:

- **[P1] Remove the duplicate browser-origin helper:** `src/gateway/http-utils.ts:65`
  - body: `resolveHttpBrowserOriginPolicy` is exported twice in this file, so the module is invalid and HTTP auth imports can fail at startup. Keep the current-main shared helper instead of adding a second declaration.
  - confidence: 0.97
- **[P1] Preserve the configured auth mode:** `src/commands/auth-rotate.ts:43`
  - body: `auth rotate` always writes `mode: "token"`, so running it against password, none, or trusted-proxy configs silently changes the gateway auth strategy. Reject non-token modes or preserve the existing mode instead of rewriting it.
  - confidence: 0.97
- **[P1] Reject env-managed gateway tokens:** `src/commands/auth-rotate.ts:23-27`
  - body: The command checks only the config token, so a deployment using `OPENCLAW_GATEWAY_TOKEN` with no config token can get a new plaintext token written to config. Since config tokens win over env tokens, future startups can ignore the intended env-managed credential.
  - confidence: 0.94
- **[P2] Move expiry warnings into shared HTTP auth:** `src/gateway/http-auth-helpers.ts:35-40`
  - body: The warning is emitted only from the stale bearer helper, while current main's normal HTTP routes use `checkGatewayHttpRequestAuth` in `src/gateway/http-auth-utils.ts`. Wire the warning into the shared auth path so `gateway.auth.tokenExpiryHours` covers normal gateway traffic.
  - confidence: 0.9
- **[P2] Remove the CLI-process expiry reset:** `src/commands/auth-rotate.ts:53`
  - body: `setGatewayTokenIssuedAtNow()` updates process-local module state, but `openclaw auth rotate` exits after writing config. The running gateway's warning clock is unchanged unless the gateway process resets or restarts, so this call is misleading dead code.
  - confidence: 0.88
- **[P2] Make rotation activation explicit:** `src/commands/auth-rotate.ts:54-55`
  - body: Gateway auth token and mode changes are restart-planned on current main, but the command logs success and prints the new token immediately. Warn, wait, or trigger the required gateway activation path so scripts do not switch to a token the running gateway does not accept yet.
  - confidence: 0.84
