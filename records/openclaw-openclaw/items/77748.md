  - body: `createActionGate` defaults missing action keys to enabled, and this PR adds `history` as an optional config key.   - body: Baileys `fetchMessageHistory` returns a request id and `messaging-history.set` includes `peerDataRequestSessionId`, but this code stores one pending request per chat JID.   - confidence: 0.9
- **[P2] Use the canonical OpenAI provider in the Docker helper:** `scripts/docker/openai-codex-auth.sh:8`
  - body: Current main and docs treat `openai-codex` as legacy migration input, while bare `--provider openai` starts ChatGPT/Codex OAuth. A new Docker helper should not create fresh legacy-profile/auth-order state; route through `--provider openai` and any current method flag needed for the intended OAuth flow.
  - confidence: 0.91
- **[P2] Make the Docker OAuth callback actually reachable:** `docker-compose.yml:32`
  - body: The PR publishes host port 1455, but Codex binds the OAuth callback listener to `127.0.0.1` and uses a localhost redirect with a 1457 fallback. A host browser hitting `localhost:1455` is not proven to reach that container listener; document the device/manual flow or add a tested host-reachable path without widening the listener unsafely.
