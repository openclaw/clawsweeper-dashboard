Full review comments:

- **[P1] Move the startup fix onto the current planner:** `src/plugins/channel-plugin-ids.ts:1-2`
  - body: On current main this file is a compatibility barrel that re-exports `gateway-startup-plugin-ids.ts`; the branch replaces it with the older implementation. The actual startup planner now lives in `src/plugins/gateway-startup-plugin-ids.ts`, so this change cannot merge cleanly and would drop current startup exports unless rebased and moved.
  - confidence: 0.95
- **[P1] Guard history reads to the current WhatsApp context:** `extensions/whatsapp/src/channel.ts:140-142`
  - body: Registering `history` as a gateway action exposes message content, but it is not added to the existing cross-context guard or a trusted-requester check. A tool-driven WhatsApp session could request history for an arbitrary JID via `to`/`chatJid`; restrict it to the current chat or an explicit trusted owner path before advertising it.
  - confidence: 0.88
- **[P2] Bind the OAuth callback port to loopback:** `docker-compose.yml:30`
  - body: Publishing `1455:1455` binds the host side on all interfaces by default, which is broader than a localhost OAuth callback should be. Bind the host port to `127.0.0.1` or keep the documented manual redirect paste path for Docker/headless flows.
  - confidence: 0.84
