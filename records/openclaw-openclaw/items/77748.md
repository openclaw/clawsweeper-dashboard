  - body: `createActionGate` defaults missing action keys to enabled, and this PR adds `history` as an optional config key.   - body: Baileys `fetchMessageHistory` returns a request id and `messaging-history.set` includes `peerDataRequestSessionId`, but this code stores one pending request per chat JID and resolves the first matching chat batch. Concurrent requests or unrelated sync chunks for the same chat can satisfy the wrong caller; key pending history by the returned request id and match the event id before resolving.
  - confidence: 0.89
- **[P2] Use the canonical OpenAI provider in the Docker helper:** `scripts/docker/openai-codex-auth.sh:8`
  - body: Current main and docs treat `openai-codex` as legacy migration input, while bare `--provider openai` starts ChatGPT/Codex OAuth.   - confidence: 0.91
