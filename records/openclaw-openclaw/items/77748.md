  - body: createActionGate treats missing action keys as enabled, so adding history here advertises a new chat-history read action to every existing WhatsApp config.   - confidence: 0.89
- **[P1] Use the canonical OpenAI provider for Docker auth:** `scripts/docker/openai-codex-auth.sh:8`
  - body: This helper signs in with --provider openai-codex, but current OpenClaw treats openai-codex as legacy migration input and exposes ChatGPT/Codex OAuth through the openai provider. As written, the Docker helper can fail with an unknown provider or create stale auth state instead of exercising the supported route.
  - confidence: 0.95
- **[P2] Correlate the on-demand history response:** `extensions/whatsapp/src/inbound/monitor.ts:619-640`
  - body: The pending request is keyed only by chat JID, and the handler resolves it from any messaging-history.set containing that JID. Baileys returns a request id from fetchMessageHistory and emits syncType plus peerDataRequestSessionId, so this should wait for the matching ON_DEMAND response instead of returning unrelated recent/full sync messages.
  - confidence: 0.86
