## Summary

Keep open: this is a focused Gateway bug-fix PR, but the patch is not safe as written because it broadens silent local approval to paired-device scope upgrades and reverses the current read-to-admin protection added by merged security hardening.

## What This Changes

The PR changes WebSocket device-pairing so local `scope-upgrade` requests can be silently approved through a new helper, with helper-level tests.

## Best Possible Solution

## Solution Assessment

No: applying local silent approval to every `scope-upgrade` is too broad for the current paired-device baseline invariant. The safer solution is the narrower local CLI shared-auth fix tracked in https://github.com/openclaw/openclaw/issues/95997 and https://github.com/openclaw/openclaw/pull/96002.
