## Summary

Keep this PR open, but it is not merge-ready: the linked session-memory duplication bug is real from source inspection, while the patch would also drop standalone delivery-mirror replies that current history contracts preserve as visible assistant output.

## What This Changes

The PR updates the bundled session-memory transcript reader to skip delivery-mirror assistant entries and collapse consecutive duplicate assistant text, with regression tests for the linked duplication report.

## Best Possible Solution

Keep the transcript-layer mitigation only if it dedupes duplicate raw/cleaned assistant text while preserving standalone delivery mirrors that represent visible replies, with a regression for both cases and real after-fix proof.

## Reproduction Assessment

Yes, at source level: current main extracts text from consecutive assistant rows, so raw-thinking plus cleaned-text rows with the same visible text produce duplicate `assistant:` lines. I did not run a live thinking-enabled model flow in this read-only review.

## Solution Assessment

No, not as written: deduping identical assistant text is a plausible mitigation, but unconditionally dropping delivery mirrors is too broad for the existing history/storage contract. A better fix preserves standalone mirrors and only removes rows proven to duplicate an adjacent or equivalent assistant output.
