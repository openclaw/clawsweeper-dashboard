## Summary

Keep this PR open: the updated head fixes the earlier broad delivery-mirror skip, but the current dedupe state can still suppress a real later assistant reply when the same text appears after a user turn, and the external contributor proof gate is still missing.

## What This Changes

The PR changes the bundled session-memory transcript reader to collapse repeated assistant text and adds regression tests for thinking/delivery-mirror duplicate rows.

## Best Possible Solution

Keep the transcript-layer mitigation narrow by deduping only adjacent included assistant rows, add a same-text-after-user regression, and require redacted real setup proof before merge.

## Reproduction Assessment

Yes at source level: current main extracts text from consecutive assistant rows, so raw-thinking plus cleaned-text rows with the same visible text produce duplicate `assistant:` lines. I did not run a live thinking-enabled model flow in this read-only review.

## Solution Assessment

No, not yet: the transcript-layer mitigation is acceptable, but the current state machine is too broad because it is not limited to adjacent included assistant rows. Resetting the dedupe state on visible user rows is the safer narrow fix.
