## Summary

Keep open. Current main mitigates one Bonjour failure mode and bounds discovery advertisement, but source, tests, docs, and maintainer discussion still leave the broader default bundled-plugin/minimal-profile upgrade policy unresolved.

## What This Changes

The issue requests that upgrades and default bundled plugin loading not allow optional plugins, especially Bonjour, to block gateway startup or recovery on constrained hosts.

## Best Possible Solution

Define a maintainer-owned startup and upgrade policy that preserves explicit disables and configured plugin sets, keeps optional discovery/provider/channel plugins non-blocking, and documents or implements a minimal-profile recovery path for constrained hosts.

## Reproduction Assessment

Unclear for the complete live upgrade failure because the Debian/npm-global upgrade path was not replayed. The remaining policy surface is reproducible by inspection: current main still includes Bonjour in the empty-config Gateway startup set and documents it as default-on.

## Solution Assessment

No, the current main mitigations are not the complete solution. The Bonjour restart cap, lazy loading, and discovery timeout are useful narrow fixes, but the durable fix needs a product decision on default bundled-plugin activation and upgrade recovery behavior.
