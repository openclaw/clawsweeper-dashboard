## Summary

Keep open. Current main has useful Bonjour and plugin-loading mitigations, but the maintainer discussion explicitly left the broader default bundled-plugin/minimal-profile startup policy unresolved, and current source still includes Bonjour in the default empty-config startup surface.

## What This Changes

The issue reports a 2026.4.2 to 2026.4.26 upgrade path where default bundled plugins, especially Bonjour mDNS probing churn, made gateway startup unreliable and asks for safer minimal/default plugin startup behavior.

## Best Possible Solution

Define a maintainer-owned startup and upgrade policy that preserves small configured plugin sets, keeps optional discovery/provider/channel plugins non-blocking, and documents or implements a minimal recovery profile for constrained hosts.

## Reproduction Assessment

Unclear for the complete live upgrade failure because this read-only review did not replay the Debian/npm-global upgrade path. The remaining policy surface is reproducible by inspection: current main still includes Bonjour in empty-config startup and documents it as default-on.

## Solution Assessment

No, the current mitigations are not the complete solution. The Bonjour timeout/restart caps and narrower disabled-channel handling reduce failure modes, but the durable fix needs a product decision on default bundled-plugin activation and minimal-profile upgrade recovery.
