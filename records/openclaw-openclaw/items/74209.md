## Summary

Keep open. Current main contains partial mitigations for Bonjour churn and plugin runtime-deps startup cost, but the source, tests, docs, and maintainer discussion still show the broader default bundled-plugin and minimal-profile upgrade policy is unresolved.

## What This Changes

The issue asks OpenClaw to avoid upgrades enabling or initializing unused bundled plugins, especially Bonjour, in a way that can block gateway startup and to provide a minimal or safe recovery profile.

## Best Possible Solution

Define and implement a maintainer-owned startup and upgrade policy that preserves explicit disables/configured plugin sets, keeps optional discovery/provider/channel plugins non-blocking, and documents or provides a minimal recovery profile for constrained hosts.

## Reproduction Assessment

Unclear for the full live upgrade failure because the Debian/npm-global 2026.4.2 to 2026.4.26 path was not replayed. The remaining policy surface is reproducible by source inspection: Bonjour is still default-on and expected in the empty-config Gateway startup set.

## Solution Assessment

No, the current main mitigations are useful but incomplete. The restart cap, discovery timeout, and runtime-deps reductions address narrow failure modes; the durable fix needs a product decision on default bundled-plugin activation and safe-mode recovery.
