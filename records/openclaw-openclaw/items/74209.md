## Summary

Keep open. Current main has mitigations for Bonjour churn and bounded discovery startup, but the broader reported problem remains: default-enabled bundled startup still includes Bonjour and several optional plugins, and maintainer discussion explicitly kept this open for the unresolved minimal-profile/default-plugin policy.

## What This Changes

The issue asks OpenClaw to prevent default bundled plugins, especially Bonjour, from being enabled or loaded after upgrade in ways that can block Gateway startup or make unused optional plugins affect normal operation.

## Best Possible Solution

Define a maintainer-owned startup and upgrade policy that keeps explicitly configured or core-required plugins active, keeps optional discovery/provider/channel plugins non-blocking, preserves explicit disables and allowlists, and provides a documented minimal-profile or safe-mode recovery path for constrained hosts.

## Reproduction Assessment

Unclear for the full live failure: the specific Debian/npm-global upgrade path was not replayed. The remaining policy surface is reproducible by inspection because current main still default-enables Bonjour and keeps multiple default bundled plugins in the empty-config Gateway startup set.

## Solution Assessment

No, current main is not the complete solution for the reported class. The Bonjour sliding-window cap and discovery startup timeout are good mitigations, but they do not settle the broader default bundled-plugin/minimal-profile upgrade behavior.
