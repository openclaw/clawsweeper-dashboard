## Summary

Keep open. Current main has useful narrow mitigations for Bonjour churn and discovery startup blocking, but the broader default bundled-plugin/minimal-profile upgrade policy described by the reporter and maintainer comment remains unresolved.

## What This Changes

The issue asks OpenClaw to avoid enabling or initializing a broad unused bundled plugin set after upgrade, especially when Bonjour/mDNS can stall gateway startup on constrained hosts.

## Best Possible Solution

Define and implement a maintainer-owned startup and upgrade policy that preserves explicit disables and configured plugin sets, keeps optional discovery/provider/channel plugins non-blocking, and provides a documented minimal-profile recovery path.

## Reproduction Assessment

Unclear for the complete live upgrade failure because this review did not replay the Debian/npm-global upgrade. The remaining policy concern is reproducible by inspection: current main still includes Bonjour in the empty-config startup set and documents it as default-on.

## Solution Assessment

No, current main is not yet the complete solution. The Bonjour restart cap and discovery timeout reduce specific failure modes, but the durable fix needs a product decision on default bundled-plugin activation and upgrade recovery behavior.
