## Summary

Keep open. Current main contains useful Bonjour and runtime-deps mitigations, but it still default-starts Bonjour and other bundled sidecars for empty config, and maintainer discussion explicitly kept the broader default-plugin/minimal-profile upgrade policy unresolved.

## What This Changes

The issue asks OpenClaw to prevent upgrades through the 2026.4.26 bundled plugin set from enabling or initializing unused optional plugins in a way that can block gateway startup or normal operation.

## Best Possible Solution

Define and document a maintainer-owned startup and upgrade policy that preserves small explicit plugin sets, keeps optional discovery/channel/provider sidecars non-blocking, and gives operators a minimal recovery path.

## Reproduction Assessment

Unclear for the full Debian/npm-global upgrade failure because I did not replay that host path. The remaining policy surface is source-reproducible: current main still default-starts Bonjour and other bundled sidecars for empty config, with tests documenting that behavior.

## Solution Assessment

No, current main is not the complete solution. The landed Bonjour timeout/restart caps and disabled-channel/runtime-deps mitigations are good narrow fixes, but changing default bundled-plugin activation or adding a minimal recovery mode needs maintainer product policy.
