## Summary

Keep open. Current main contains useful Bonjour and startup mitigations, but the maintainer discussion and current source still leave the broader default bundled-plugin/minimal-profile upgrade policy unresolved.

## What This Changes

The issue requests that upgrades avoid loading broad unused bundled plugins by default, keep optional Bonjour discovery non-blocking, and provide a minimal or safe recovery path for constrained hosts.

## Best Possible Solution

Define and document a maintainer-owned startup and upgrade policy that preserves explicit disables and configured plugin sets, loads only needed plugins during recovery, and keeps optional discovery/provider/channel plugins non-blocking.

## Reproduction Assessment

Unclear for the full live upgrade failure because the Debian/npm-global path on a constrained host was not replayed. The remaining policy surface is reproducible by inspection: Bonjour is still default-on and still part of the empty-config Gateway startup set.

## Solution Assessment

No, current main is not the complete solution. The Bonjour restart cap, discovery timeout, and global plugin-disable fast path are useful mitigations, but the durable fix needs a product decision on configured-only/minimal startup behavior.
