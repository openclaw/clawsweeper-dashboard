## Summary

Keep open. Current main has narrowed the worst Bonjour watchdog and runtime-deps failure modes, but the broader default bundled-plugin/minimal-profile upgrade policy remains unresolved and a maintainer explicitly kept this issue open after the narrow Bonjour fix.

## What This Changes

The issue asks OpenClaw to prevent default bundled plugins, especially Bonjour, from making upgraded gateways unusable and to provide a safer minimal/configured plugin startup path.

## Best Possible Solution

Adopt and document a maintainer-owned startup and upgrade policy that preserves small explicit plugin sets, keeps optional discovery/channel/provider plugins non-blocking, and gives operators a minimal-profile recovery path.

## Reproduction Assessment

Unclear for the full Debian/npm-global upgrade failure because I did not replay that host path. The remaining policy surface is source-reproducible: current main still default-starts Bonjour and other bundled sidecars for empty config, with tests documenting that behavior.

## Solution Assessment

No, current main is not the complete solution. The Bonjour timeout/restart caps and disabled-channel filtering are useful mitigations, but changing default bundled-plugin activation or adding a minimal recovery profile needs maintainer policy work.
