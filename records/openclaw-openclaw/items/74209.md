## Summary

Keep open. Current main has targeted mitigations for Bonjour churn and bounded discovery startup, but the maintainer discussion and current source both show the broader default bundled-plugin/minimal-profile upgrade policy is still unresolved.

## What This Changes

The issue asks OpenClaw to avoid enabling or initializing optional bundled plugins, especially Bonjour, in a way that can block or destabilize gateway startup after upgrades.

## Best Possible Solution

Define a maintainer-owned startup and upgrade policy that preserves explicit disables and configured plugin sets, keeps optional discovery/provider/channel plugins non-blocking, and provides a documented minimal-profile or safe-mode recovery path for constrained hosts.

## Reproduction Assessment

Unclear for the full live upgrade failure because the Debian/npm-global path was not replayed. The remaining policy surface is reproducible by inspection: current main still includes Bonjour in the empty-config Gateway startup set and documents it as default-on.

## Solution Assessment

No, the current mitigations are not the complete solution. The Bonjour restart cap and discovery startup timeout are useful narrow fixes, but the durable fix is a maintainer decision on default bundled-plugin activation and upgrade recovery behavior.
