## Summary

Keep open. Current main has useful Bonjour and plugin-startup mitigations, but the maintainer discussion and current source still leave the broader default bundled-plugin and minimal-profile upgrade policy unresolved.

## What This Changes

The issue asks OpenClaw to prevent default bundled plugins, especially Bonjour, from making upgraded gateways unusable when those optional plugins are not configured or needed.

## Best Possible Solution

Define a maintainer-owned startup and upgrade policy that preserves explicit disables and small configured plugin sets, keeps optional discovery/provider/channel plugins non-blocking, and documents a minimal-profile recovery path.

## Reproduction Assessment

Unclear for the complete live upgrade failure because the affected Debian/npm-global path was not replayed. The remaining policy surface is reproducible by inspection: current main still includes Bonjour in the empty-config startup set and documents it as default-on.

## Solution Assessment

No, the current mitigations are not the complete solution. The Bonjour restart caps, discovery timeout, and disabled-channel filtering are useful narrow fixes, but the durable answer needs a product decision on default bundled-plugin activation and upgrade recovery behavior.
