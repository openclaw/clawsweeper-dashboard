## Summary

Keep open. Current main includes a narrow Bonjour watchdog mitigation, but the maintainer discussion explicitly says the broader default bundled-plugin and minimal-profile upgrade policy remains unresolved, and the current source still default-enables Bonjour plus default-selected bundled startup/runtime-dependency behavior.

## What This Changes

The issue asks OpenClaw to prevent default bundled plugins, especially Bonjour and optional unused plugins, from blocking gateway startup after upgrades and to provide a minimal configured recovery/startup path.

## Best Possible Solution

Define a durable upgrade/startup policy that loads only configured, explicitly enabled, or core-required plugins by default; keeps explicit disables, denylists, and allowlists authoritative; makes LAN discovery and optional provider/voice surfaces non-blocking or opt-in where appropriate; and documents a minimal-profile or safe-mode recovery path for operators.

## Reproduction Assessment

Unclear for the full live gateway stall because the exact Debian/npm-global upgrade path was not replayed. Yes for the remaining policy surface: current main statically shows Bonjour/default startup inclusion and default-enabled runtime-dependency behavior in source, docs, and tests.

## Solution Assessment

No, current main is not the complete solution. The Bonjour sliding-window cap is a good narrow mitigation, but the best fix is a broader startup/upgrade policy for default bundled plugins and recovery behavior.
