## Summary

Keep open. Current main has useful Bonjour and runtime-deps mitigations, but the broader default-plugin/minimal-profile upgrade policy is still unresolved and current source still default-starts Bonjour plus other bundled sidecars.

## What This Changes

Prevent upgrades from enabling or loading unused bundled plugins, especially Bonjour, in a way that can block gateway startup, and provide a minimal recovery/startup path.

## Best Possible Solution

Adopt and document a maintainer-owned startup and upgrade policy that preserves small explicit plugin sets, keeps optional discovery/channel/provider plugins non-blocking, and gives operators a minimal-profile recovery path.

## Reproduction Assessment

Unclear for the full Debian/npm-global upgrade failure because I did not replay that host path. The remaining policy surface is source-reproducible: current main still default-starts Bonjour and other bundled sidecars for empty config, with tests documenting that behavior.

## Solution Assessment

No, current main is not the complete solution. The Bonjour timeout/restart caps and disabled-channel filtering are good narrow mitigations, but changing default bundled-plugin activation or adding a minimal recovery profile needs maintainer policy work.
