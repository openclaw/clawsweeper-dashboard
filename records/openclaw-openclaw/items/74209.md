## Summary

Keep open. Current main has narrow mitigations for Bonjour churn and bounded discovery startup, but the maintainer discussion and current source still leave the broader default bundled-plugin and minimal-profile upgrade policy unresolved.

## What This Changes

The issue asks OpenClaw upgrades and Gateway startup to avoid enabling or initializing broad optional bundled plugins, especially Bonjour, in a way that blocks readiness, and to provide a minimal or safe recovery path.

## Best Possible Solution

Define a maintainer-owned startup and upgrade policy that preserves explicit disables and configured plugin sets, keeps optional discovery/provider/channel plugins non-blocking, and documents a user-facing minimal-profile or safe-mode recovery path.

## Reproduction Assessment

Unclear for the full live upgrade failure because the Debian/npm-global path was not replayed. The remaining policy surface is reproducible by inspection: current main still includes Bonjour in the empty-config Gateway startup set and documents it as default-on.

## Solution Assessment

No, the current mitigations are not the complete solution. The Bonjour restart cap and discovery startup timeout are useful narrow fixes, but the durable fix needs a maintainer decision on default bundled-plugin activation and upgrade recovery behavior.
