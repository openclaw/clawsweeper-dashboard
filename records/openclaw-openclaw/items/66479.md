## Summary

Keep open: current main still rejects BOM-prefixed markdown frontmatter before skill metadata is parsed, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/81748 remains open with closing syntax for this issue.

## What This Changes

The issue requests that BOM-prefixed `SKILL.md` frontmatter be parsed so Windows-edited skills remain visible in `openclaw skills list`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81748, or an equivalent shared parser fix with parser and workspace skill discovery regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. Node preserves the leading U+FEFF, current main checks for `---` before removing it, and the skill loader drops entries whose description is not parsed.

## Solution Assessment

Yes. Fixing `src/markdown/frontmatter.ts` is the narrowest maintainable solution because skills delegate to the shared parser; fixing only the skill loader would leave other frontmatter consumers inconsistent.
