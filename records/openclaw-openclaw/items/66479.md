## Summary

Keep open: current main still rejects BOM-prefixed markdown frontmatter before skill metadata is parsed, and the open linked fix at https://github.com/openclaw/openclaw/pull/81748 should remain the implementation path until it merges.

## What This Changes

Allow workspace skills whose `SKILL.md` starts with a UTF-8 BOM before frontmatter to remain discoverable in `openclaw skills list`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81748, or an equivalent shared parser fix with parser and skill discovery regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. The source-level path is high confidence: Node preserves the leading U+FEFF, current main checks for `---` before removing it, and the skill loader drops entries whose description is not parsed.

## Solution Assessment

Yes. Fixing `src/markdown/frontmatter.ts` is the narrowest maintainable solution because skills delegate to the shared parser; fixing only the skill loader would leave other frontmatter consumers inconsistent.
