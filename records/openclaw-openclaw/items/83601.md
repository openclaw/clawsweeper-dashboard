## Summary

Keep this open. Current main still preserves the literal string `"null"` as a cron `sessionKey`, and an open linked fix PR references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

The issue reports that cron job updates using `patch.sessionKey: null` through the cron tool can persist the literal string `"null"` instead of clearing the session binding.

## Best Possible Solution

Review and land a narrow fix that clears model-produced `sessionKey: "null"` values without changing unrelated valid identifiers, then close this issue from the merged PR.

## Reproduction Assessment

Yes by source inspection: current main preserves non-empty string `sessionKey` values through cron normalization and patch storage, while the tool schema can steer callers away from real JSON null. I did not run a live cron tool/Gateway update in this read-only review.

## Solution Assessment

Yes for the issue boundary: the maintainable fix belongs in cron tool normalization/schema handling because the Gateway contract already accepts real null. The open linked PR is the right implementation candidate, but it needs its review feedback resolved before merge.
