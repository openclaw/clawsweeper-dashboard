## What This Changes

The issue reports that updating a cron job through the agent-facing cron tool with `patch.sessionKey: null` stores the literal string "null" instead of clearing the session binding.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/83618 or an equivalent fix lands, preserving the OpenAPI-compatible cron tool schema while normalizing the nullable `agentId` and `sessionKey` clear sentinels correctly.

## Reproduction Assessment

## Solution Assessment
