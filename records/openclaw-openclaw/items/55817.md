## Summary

Keep open: current main still has the session-store-to-missing-transcript gap and this PR is the right narrow direction, but the branch needs an exclusive, non-clobbering header create plus the required changelog entry before merge.

## What This Changes

The PR adds an optional transcript-header creation path during session-file resolution, enables it from `initSessionState()`, and adds reset/rollover regression tests.

## Best Possible Solution

Land a small branch revision that writes the rollover header before the store update using `flag: "wx"`, treats `EEXIST` as success, preserves non-ENOENT failures, and records the fix in the active changelog.

## Reproduction Assessment

Yes. Current main shows the store can be updated to a rotated `sessionFile` without creating the transcript, and the PR's remaining race follows from a creator landing between `stat()` and default overwrite `writeFile()`.

## Solution Assessment

No. Creating the header before the store update is the right direction, but the proposed helper must be non-clobbering and tolerate `EEXIST` as success.
