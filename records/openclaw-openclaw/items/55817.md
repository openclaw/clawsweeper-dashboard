## Summary

Keep open: this PR targets a real session rollover consistency bug, but the proposed header bootstrap can still clobber a transcript created between its stat and write steps, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR adds optional header-only transcript creation during session-file resolution, enables it from `initSessionState()`, and adds reset/rollover regression tests.

## Best Possible Solution

Land a focused version that creates the new transcript before updating `sessions.json` with non-clobbering exclusive create, preserves existing transcripts on races, and includes the regression tests plus changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows rollover can mint a new `sessionId` and persist its `sessionFile` before any `.jsonl` is created; the PR race is also visible from the stat-then-default-write helper.

## Solution Assessment

No. Creating the header before the store update is the right direction, but the header write must be exclusive and treat `EEXIST` as success so a concurrent append cannot be overwritten.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Create transcript headers with exclusive create:** `src/config/sessions/session-file.ts:32-35`
  - body: The helper stats the path and then writes with default overwrite semantics. If another rollover or append creates the transcript between those steps, this write can truncate real content back to a header-only file; use exclusive create and treat `EEXIST` as success.
