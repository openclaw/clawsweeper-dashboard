## Summary

Keep open: the PR targets a real session/transcript consistency bug, but the current patch can still clobber a transcript created by a competing append path and is missing the required changelog entry.

## What This Changes

The PR adds optional transcript-header creation during session-file resolution, enables it from session initialization, and adds reset/rollover regression coverage.

## Best Possible Solution

Revise the PR to prewarm the transcript before updating the session store with exclusive create semantics, treat `EEXIST` as success, add the changelog entry, rebase onto current main, and keep the focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main can rotate a session key to a new `sessionFile` in `sessions.json` without creating the referenced `.jsonl`; I did not run the test path because `pnpm` is unavailable in this read-only environment.

## Solution Assessment

No. Creating the header before the store update is the right direction, but the PR must use non-clobbering exclusive creation so a concurrent transcript creator cannot be overwritten.
