## Summary

Keep open. The PR addresses a real source-reproducible session durability bug, but the current head still has blocking issues: header creation can clobber concurrent transcript content, the branch imports the old pi-coding-agent package name, and external real behavior proof is missing.

## What This Changes

The PR adds an optional transcript-header bootstrap to session-file resolution, enables it from initSessionState, and adds reset/rollover regression tests.

## Best Possible Solution

Create the successor transcript before publishing the store entry using exclusive `wx` creation with `EEXIST` treated as success, update the package imports, keep focused regression coverage, and coordinate broader reset durability through https://github.com/openclaw/openclaw/issues/75151.

## Reproduction Assessment

Yes, source-level. Current main can mint a new session id in initSessionState and persist the resolved sessionFile before any code guarantees that JSONL exists.

## Solution Assessment

No, not as-is. The PR targets the right invariant, but the helper needs non-clobbering exclusive creation, current package imports, and contributor after-fix runtime proof before merge.
