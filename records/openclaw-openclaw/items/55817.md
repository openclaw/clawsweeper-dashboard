## Summary

Keep open. The PR targets a real session persistence bug, but the current branch can still clobber a concurrently-created transcript, imports the old pi-coding-agent package name, and has only unit-test proof from an external contributor.

## What This Changes

The PR adds immediate header creation for rollover transcripts in `resolveAndPersistSessionFile()`, enables it from `initSessionState()`, and adds reset/rollover regression tests.

## Best Possible Solution

Create the successor transcript with a non-clobbering exclusive header write before publishing the store entry, keep the focused regression tests, and coordinate broader reset/replay durability with https://github.com/openclaw/openclaw/issues/75151.

## Reproduction Assessment

Yes. From source inspection, current main can persist a new `sessionFile` during `initSessionState()` rollover before any code guarantees the successor JSONL exists.

## Solution Assessment

No. The PR points at the right invariant, but the helper should use exclusive creation with `EEXIST` treated as success, the package imports need rebasing to current main, and real reset/rollover proof is still required before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Create transcript headers exclusively:** `src/config/sessions/session-file.ts:30-33`
  - body: The helper stats the path and then writes with default overwrite behavior. If another request creates or appends the same missing transcript between those steps, this write can truncate real content back to a header-only file; create with `flag: "wx"` and treat `EEXIST` as success.
  - confidence: 0.88
- **[P2] Use the current pi-coding-agent package:** `src/config/sessions/session-file.ts:3`
  - body: Current main depends on `@earendil-works/pi-coding-agent`, but this PR imports `CURRENT_SESSION_VERSION` from the old `@mariozechner/pi-coding-agent` package in production and tests, so the branch will not resolve until those imports are rebased.
  - confidence: 0.92
