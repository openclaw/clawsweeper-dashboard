## Summary

Keep open: the PR targets a real source-reproducible session persistence gap, but the latest branch is not merge-ready because header creation can overwrite concurrent transcript content, it still imports the old pi-coding-agent package, it conflicts with current main, and external real behavior proof is missing.

## What This Changes

The PR adds a transcript-header bootstrap option to session-file resolution, enables it from session initialization, and adds reset/rollover regression tests.

## Best Possible Solution

Create the successor transcript before publishing the store entry using exclusive creation with EEXIST treated as success, update the package imports and current session-file contract, keep focused regression coverage, and leave broader reset recovery to the linked follow-up issue.

## Reproduction Assessment

Yes, source-level. Current main can mint a new session id in initSessionState and persist the resolved sessionFile before any code guarantees that JSONL exists.

## Solution Assessment

No, not as-is. The PR targets the right invariant, but it needs non-clobbering exclusive file creation, a rebase onto current imports/contracts, and contributor after-fix runtime proof before merge.
