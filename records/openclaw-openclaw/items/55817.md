## Summary

Keep open. Current main still has the session rollover consistency gap, and this PR is a plausible bug fix direction, but the proposed patch is not merge-ready because it can clobber a transcript created by a concurrent writer, imports the old pi-coding-agent package name, and still lacks contributor-owned real reset/rollover proof.

## What This Changes

The PR adds a transcript-header creation option to session-file resolution, enables it from session initialization, and adds reset/rollover regression tests.

## Best Possible Solution

Create the successor transcript with a non-clobbering exclusive header write before publishing the store entry, keep the focused regression tests, and coordinate with the broader reset/replay durability work.

## Reproduction Assessment

Yes, from source inspection. Current main can persist a new `sessionFile` during `initSessionState()` rollover before any code guarantees the successor JSONL exists.

## Solution Assessment

No. The PR points at the right invariant, but the helper should use exclusive creation with `EEXIST` treated as success, the package import needs rebasing to current main, and real reset/rollover proof is still required before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Create transcript headers exclusively:** `src/config/sessions/session-file.ts:30-33`
  - body: The helper stats the path and then writes with default overwrite behavior. If another rollover or append creates the transcript between those steps, this write can truncate real content back to a header-only file; create with `flag: "wx"` and treat `EEXIST` as success.
  - confidence: 0.88
- **[P2] Use the current pi-coding-agent package:** `src/config/sessions/session-file.ts:3`
  - body: Current main depends on `@earendil-works/pi-coding-agent`, but this PR imports `CURRENT_SESSION_VERSION` from the old `@mariozechner/pi-coding-agent` package in production and tests. Rebase/update the imports or the changed files will not resolve.
  - confidence: 0.92
