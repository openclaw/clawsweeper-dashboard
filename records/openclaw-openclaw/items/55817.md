## Summary

Keep open. Current main still has the reported missing-transcript window, and this PR is directionally useful, but the proposed header bootstrap can overwrite a concurrently-created transcript because it checks existence separately from a default write.

## What This Changes

The PR adds optional transcript-header creation during session-file resolution, enables it from `initSessionState()`, and adds reset/rollover regression tests.

## Best Possible Solution

Keep the PR open, change the bootstrap write to an exclusive create that treats `EEXIST` as success, add focused non-clobbering coverage, and then validate the session tests before maintainer review.

## Reproduction Assessment

Yes. Current main shows the original window by persisting a fresh `sessionFile` during `initSessionState()` without creating it, and the PR regression can be reproduced by racing two header-ensure callers between the stat and write steps.

## Solution Assessment

No. Creating the header before the store update is the right direction, but the implementation should use an atomic create (`flag: "wx"` or equivalent) and treat `EEXIST` as success so transcript contents are never truncated.
