## Summary

Keep open. The PR targets a real source-reproducible session durability gap, but the branch still has a clobbering race in the new transcript bootstrap helper, imports the old pi-coding-agent package name, and lacks external-contributor real behavior proof.

## What This Changes

The PR adds immediate header-only transcript creation during `initSessionState()` rollover and adds reset/rollover regression tests.

## Best Possible Solution

Create the successor transcript with a non-clobbering exclusive header write before publishing the store entry, keep the focused regression tests, and coordinate broader reset/replay durability through https://github.com/openclaw/openclaw/issues/75151.

## Reproduction Assessment

Yes, source-level. Current main can mint a new session id and persist the resolved `sessionFile` during rollover before any code guarantees the successor JSONL exists.

## Solution Assessment

No. The PR points at the right invariant, but the helper should use exclusive creation with `EEXIST` treated as success, the imports need rebasing to the current package name, and real behavior proof is still required.

## Review Findings

Overall correctness: patch is incorrect
