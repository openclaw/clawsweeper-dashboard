## Summary

Keep open. The PR targets a real source-level session durability gap, but the branch still has a non-atomic transcript creation bug, stale package imports, and no after-fix real behavior proof.

## What This Changes

The PR adds a transcript-header bootstrap option to session-file resolution, enables it from `initSessionState`, and adds reset/rollover regression tests.

## Best Possible Solution

Create the successor transcript before publishing the store entry using exclusive `wx` creation with `EEXIST` treated as success, update the package imports, keep focused regression coverage, and coordinate broader reset durability through https://github.com/openclaw/openclaw/issues/75151.

## Reproduction Assessment

Yes, source-level. Current main can mint a new session id in `initSessionState` and persist the resolved `sessionFile` before any code guarantees that JSONL exists.

## Solution Assessment

No, not as-is. Prewarming before the store update is the right invariant, but the helper must be non-clobbering, use current package imports, and be backed by real reset or rollover proof before merge.
