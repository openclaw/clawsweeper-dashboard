## Summary

Keep open. Current main still has the missing-transcript rollover gap and this PR is directionally useful, but the proposed helper can clobber a transcript created between its stat and write, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds optional transcript-header creation to `resolveAndPersistSessionFile()`, enables it from `initSessionState()`, and adds reset/rollover regression coverage.

## Best Possible Solution

Create the header before the store update using exclusive create, treat `EEXIST` as success, preserve non-ENOENT errors, keep the focused regression coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, from source inspection rather than execution. Current main resolves and persists a new `sessionFile` mapping during `initSessionState()` rollover before any code guarantees that the new `.jsonl` exists.

## Solution Assessment

No. Creating the header before updating `sessions.json` is the right direction, but the PR should use non-clobbering exclusive creation so a competing append or rollover cannot be truncated back to a header-only file.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P1] Create transcript headers exclusively:** `src/config/sessions/session-file.ts:30-33`
