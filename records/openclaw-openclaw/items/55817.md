## Summary

Keep open: current main still has the missing-transcript rollover gap, but this PR needs fixes before merge because its header creation can clobber a concurrently-created transcript and the branch is stale against the current package import surface.

## What This Changes

The PR adds transcript-header creation to session-file resolution, enables it from session initialization, and adds reset/rollover regression coverage.

## Best Possible Solution

Create the successor transcript with an exclusive write before publishing the store entry, treat `EEXIST` as success, rebase to current imports, keep focused regression coverage, and add redacted real reset/rollover proof.

## Reproduction Assessment

Yes, from source inspection. Current main can persist a new `sessionFile` mapping during `initSessionState()` rollover before any code guarantees the new transcript exists.

## Solution Assessment

No. The direction is right, but the PR should use non-clobbering exclusive creation, update the stale package imports, and provide real behavior proof before merge.
