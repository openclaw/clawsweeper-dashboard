## Summary

Keep open: current main still has the missing-transcript window, and the PR is directionally right but needs an atomic header create plus a changelog entry before merge.

## What This Changes

The PR adds optional header-only transcript creation during session-file resolution, enables it from session initialization, and adds reset/rollover regression tests.

## Best Possible Solution

Land a small revision that creates the rollover header before the store update using exclusive create, treats `EEXIST` as success, preserves non-ENOENT errors, and records the fix in the active changelog.

## Reproduction Assessment

Yes. Current main shows the original bug path by updating the session mapping without creating the new transcript, and the PR regression follows from two callers racing between `stat` and default `writeFile()`.

## Solution Assessment

No. Creating the header before the store update is the right direction, but the write must be non-clobbering (`flag: "wx"` or equivalent) with `EEXIST` treated as success.
