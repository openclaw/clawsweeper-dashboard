## Summary

Keep open: this PR is aimed at a real session persistence bug, but the proposed header bootstrap can still overwrite a transcript created by a competing rollover or append path.

## What This Changes

The PR adds optional transcript-header creation during session-file resolution, enables it from session initialization, and adds reset/rollover regression tests.

## Best Possible Solution

Revise the PR to create the header before the store update with non-clobbering exclusive create, treat `EEXIST` as success, preserve non-ENOENT errors, add the changelog entry, and land the focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main can persist a new `sessionFile` mapping without creating the `.jsonl`, and the PR regression is visible from the stat-then-default-write helper.

## Solution Assessment

No. Creating the header before updating `sessions.json` is the right direction, but the write must be exclusive so a competing creator cannot be clobbered.

## Review Findings
