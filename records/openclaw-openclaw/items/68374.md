## Summary

Keep open: current main still drops Claude CLI thinking before the Gateway HTTP APIs can expose it, the public Responses docs still say `reasoning` is ignored, and the direct implementation candidate remains open rather than merged.

## What This Changes

The issue requests exposing Claude CLI `thinking_delta` content as gated structured reasoning on `/v1/responses` and `/v1/chat/completions`.

## Best Possible Solution

Keep this issue open while maintainers review or replace https://github.com/openclaw/openclaw/pull/56674 with a gated HTTP reasoning contract that covers Claude `thinking_delta`, streaming and non-streaming Responses output, Chat Completions compatibility, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main still drops Claude `thinking_delta` in the CLI parser and both HTTP serializers only emit assistant text/lifecycle events, while the issue includes concrete CLI and Gateway curl evidence.

## Solution Assessment

Unclear. Exposing structured HTTP reasoning is a plausible direction, but maintainers still need to settle the explicit opt-in/security boundary and the Chat Completions compatibility shape before this is the narrowest maintainable solution.
