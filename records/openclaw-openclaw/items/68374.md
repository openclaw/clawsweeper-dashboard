## Summary

Keep open. Current main still drops Claude CLI thinking before the HTTP API can expose it, the Responses and Chat Completions handlers remain assistant-text-only, and related PR #56674 is open implementation work rather than a merged fix.

## What This Changes

Expose Claude CLI thinking blocks as structured reasoning on Gateway HTTP `/v1/responses` and optionally `/v1/chat/completions`, gated by reasoning visibility controls.

## Best Possible Solution

Keep this issue open while maintainers review #56674 or a refreshed replacement that preserves Claude `thinking_delta`, emits explicitly gated structured reasoning for streaming and non-streaming `/v1/responses`, defines any Chat Completions compatibility shape, and updates docs plus focused tests.

## Reproduction Assessment

Yes. The issue includes a concrete Claude CLI and curl repro, and source inspection on current main confirms the parser only accepts text deltas while the HTTP handlers emit assistant text only.

## Solution Assessment

Unclear. Structured Responses reasoning output is a plausible direction, but maintainers still need to settle privacy defaults, request opt-in, event naming, and Chat Completions compatibility.
