## Summary

Keep open. Current main still does not expose Claude CLI thinking on the HTTP APIs: the Claude CLI JSON parser accepts only text deltas, `/v1/responses` and `/v1/chat/completions` serialize assistant text only, and the docs still mark Responses `reasoning` as ignored. The related implementation PR remains open and conflicting, with API/security review still needed.

## What This Changes

Expose Claude CLI thinking blocks as structured reasoning output on `/v1/responses` and `/v1/chat/completions`, gated by existing reasoning visibility controls.

## Best Possible Solution

Keep this issue open until a reviewed design captures Claude `thinking_delta`, exposes HTTP reasoning only through explicit visibility controls, defines both Responses and Chat Completions wire shapes, and updates focused gateway/parser docs and tests.

## Reproduction Assessment

Yes. The reporter provided concrete Claude CLI and Gateway curl evidence, and current-main source inspection confirms the Claude CLI parser and both HTTP handlers still carry assistant text but not reasoning.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers still need to settle privacy gating, event naming, and the Chat Completions compatibility shape before this is the narrowest maintainable fix.
