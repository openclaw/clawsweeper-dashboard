## Summary

Keep open. Current main still drops Claude CLI thinking before the HTTP surfaces can expose it: the Claude stream-json parser only accepts text deltas, `/v1/responses` and `/v1/chat/completions` only consume assistant text, and the docs still mark Responses `reasoning` as ignored. Related PR #56674 is an open implementation candidate to review or replace, not proof that the issue is resolved.

## What This Changes

Expose Claude CLI thinking/reasoning blocks through the Gateway HTTP APIs, especially `/v1/responses` and `/v1/chat/completions`, gated by existing reasoning visibility controls or an explicit HTTP opt-in.

## Best Possible Solution

Keep this issue open while maintainers review PR #56674 or replace it with a refreshed implementation that preserves Claude `thinking_delta` as agent reasoning, gates HTTP exposure through `reasoningDefault` or a deliberate request opt-in, emits structured streaming and non-streaming `/v1/responses` reasoning, defines the chat-completions compatibility shape, and updates docs plus focused parser/gateway tests.

## Reproduction Assessment

Yes. The issue includes a concrete runtime repro, and current-main static checks reproduce the drop path with high confidence: Claude CLI `thinking_delta` is ignored and both HTTP handlers only emit assistant text.

## Solution Assessment

Unclear. Structured reasoning output items are directionally consistent with the request, but the best maintainable solution still needs maintainer judgment on opt-in behavior, event names, and whether `/v1/chat/completions` should expose a compatibility field or stay Responses-only.
