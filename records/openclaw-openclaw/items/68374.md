## What This Changes

Request to expose Claude CLI thinking/reasoning content through `/v1/responses` and `/v1/chat/completions`, gated by existing reasoning controls or an explicit HTTP opt-in.

## Best Possible Solution

Review or replace https://github.com/openclaw/openclaw/pull/56674 with a gated HTTP reasoning contract that preserves Claude `thinking_delta`, covers streaming and non-streaming Responses, defines the Chat Completions compatibility shape, updates docs, and adds focused tests.

Yes, at source level. Current main still drops Claude CLI `thinking_delta` before agent events and both Gateway HTTP serializers only emit assistant text/lifecycle events, while the issue includes concrete CLI and Gateway curl evidence.

## Solution Assessment

Unclear. Exposing structured HTTP reasoning is plausible, but maintainers still need to settle the opt-in/security boundary and `/v1/chat/completions` compatibility shape before the exact implementation path is the best maintainable fix.
