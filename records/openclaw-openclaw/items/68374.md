## Summary

Keep open. Current main still drops Claude CLI thinking before the HTTP APIs can expose it, the docs still mark Responses `reasoning` as ignored, and the related implementation remains open with API/security review outstanding.

## What This Changes

Expose Claude CLI thinking blocks as reasoning on the Gateway `/v1/responses` and `/v1/chat/completions` HTTP APIs, gated by reasoning visibility settings.

## Best Possible Solution

Keep this issue open until a reviewed implementation preserves Claude `thinking_delta`, exposes HTTP reasoning only through explicit visibility controls, defines Responses and Chat Completions wire shapes, and updates focused tests/docs.

Yes. The issue includes concrete Claude CLI and Gateway curl evidence, and current-main source inspection confirms the parser and HTTP handlers still carry assistant text but not reasoning.

## Solution Assessment

Unclear. Exposing structured HTTP reasoning is plausible, but maintainers still need to settle privacy gating, event naming, and the Chat Completions compatibility shape.
