Keep open. Request to expose Claude CLI thinking blocks as reasoning on the Gateway `/v1/responses` and `/v1/chat/completions` HTTP surfaces, gated by existing reasoning settings.

## Best Possible Solution

Keep this issue as the canonical HTTP reasoning API decision, then land or replace the closed same-surface attempt with a default-silent, explicitly gated implementation covering Claude CLI thinking capture, streaming and non-streaming `/v1/responses`, a deliberate `/v1/chat/completions` contract, docs, and focused tests.
