## Summary

Keep open: current main still filters Claude CLI thinking before the HTTP APIs can expose it, docs still mark Responses `reasoning` as ignored, and the only implementation candidate remains open with API/security review and proof gaps.

## What This Changes

Expose Claude CLI thinking blocks as structured reasoning through `/v1/responses` and `/v1/chat/completions`, gated by reasoning visibility controls.

## Best Possible Solution

Keep this issue open while maintainers review or replace the linked implementation with a gated HTTP reasoning contract covering Claude `thinking_delta`, streaming and non-streaming Responses output, Chat Completions compatibility, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. The issue includes concrete Claude CLI and Gateway curl evidence, and current main still drops `thinking_delta` while both HTTP handlers serialize assistant text only.

## Solution Assessment

Unclear. Exposing structured HTTP reasoning is a plausible direction, but maintainers still need to settle privacy gating, event names, and the `/v1/chat/completions` compatibility shape before this is the narrowest maintainable solution.
