## Summary

Keep open. Current main still does not expose Claude CLI thinking as HTTP reasoning: the parser only emits text deltas, the Responses and Chat Completions handlers remain assistant-text-oriented, and the docs still mark Responses `reasoning` as ignored; #56674 is an open, unmerged implementation candidate rather than a shipped fix.

## What This Changes

The issue requests exposing Claude CLI thinking blocks as structured reasoning output on `/v1/responses` and `/v1/chat/completions`, gated by existing reasoning controls.

## Best Possible Solution

Land or replace #56674 with a maintainer-approved contract that preserves Claude `thinking_delta` as agent reasoning, emits structured Responses reasoning for stream and non-stream HTTP clients, deliberately defines any Chat Completions compatibility shape, and updates docs plus parser/gateway tests.

## Reproduction Assessment

Yes. The issue includes a concrete live repro, and current-main static inspection gives a high-confidence reproduction path: Claude `thinking_delta` is ignored and the HTTP handlers only translate assistant text to client-visible output.

## Solution Assessment

Unclear. Structured Responses reasoning items are directionally maintainable, but the final fix still needs maintainer decisions on privacy gating, event naming, and whether Chat Completions should expose reasoning or remain Responses-first.
