## Summary

Keep open. Current main still drops Claude CLI thinking before the HTTP APIs can expose it, docs still say Responses `reasoning` is ignored, and the related implementation PR remains open with API/security review still needed.

## What This Changes

Expose Claude CLI thinking blocks as structured reasoning output on `/v1/responses` and `/v1/chat/completions`, gated by existing reasoning visibility controls.

## Best Possible Solution

Keep this issue open until a reviewed implementation preserves Claude `thinking_delta`, exposes HTTP reasoning only through explicit visibility controls, defines both HTTP wire shapes, and updates focused tests/docs.

## Reproduction Assessment

Yes, at source level. The issue includes concrete Claude CLI and Gateway curl evidence, and current-main source inspection confirms Claude CLI thinking is filtered while both HTTP handlers serialize assistant text only.

## Solution Assessment

Unclear. Exposing structured HTTP reasoning is a plausible direction, but maintainers still need to settle privacy gating, event naming, and the `/v1/chat/completions` compatibility shape before this is the narrowest maintainable solution.
