## Summary

Keep open. Current main still does not expose Claude CLI thinking on the HTTP surfaces: the CLI parser ignores `thinking_delta`, `/v1/responses` and `/v1/chat/completions` serialize assistant text only, and the docs still mark Responses `reasoning` as ignored; https://github.com/openclaw/openclaw/pull/56674 is open implementation work, not a merged fix.

## What This Changes

The issue requests exposing Claude CLI thinking blocks as opt-in structured reasoning on `/v1/responses` and `/v1/chat/completions`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/56674 or a refreshed replacement that preserves Claude `thinking_delta`, gates HTTP reasoning exposure explicitly, defines the Responses and Chat Completions wire shapes, and adds focused parser/gateway docs and tests.

## Reproduction Assessment

Yes. The issue includes a concrete Claude CLI and Gateway curl repro, and current-main source inspection confirms the parser and HTTP handlers only carry assistant text; I did not run a live Claude/Gateway repro in this read-only pass.

## Solution Assessment

Unclear. Structured Responses reasoning output is a plausible direction, but the best fix needs maintainer decisions on privacy gating, event naming, and whether `/v1/chat/completions` should expose reasoning or remain Responses-first.
