## Summary

Keep open. Current main still lacks the requested HTTP reasoning surface: Claude CLI thinking deltas are not preserved as reasoning events, the Responses and Chat Completions HTTP handlers remain assistant-text-oriented, the Responses schema has no reasoning SSE events, and the docs still describe `reasoning` input as ignored. Related PR #56674 is an open implementation candidate rather than a merged fix.

## What This Changes

Expose Claude CLI thinking blocks as structured reasoning output on `/v1/responses` and `/v1/chat/completions`, gated by existing reasoning visibility settings or an explicit HTTP opt-in.

## Best Possible Solution

Keep this issue open while maintainers review PR #56674 or replace it with a refreshed implementation that captures Claude `thinking_delta`, represents reasoning in the agent event/result contract, emits structured `/v1/responses` reasoning for streaming and non-streaming requests, deliberately defines any `/v1/chat/completions` compatibility field, gates exposure through `reasoningDefault` or an explicit per-request opt-in, and updates docs plus focused parser/gateway tests.

## Reproduction Assessment

Yes. The issue includes a concrete live repro, and current-main static inspection gives a high-confidence reproduction path: Claude `thinking_delta` is ignored and the HTTP handlers only translate assistant text to client-visible output.

## Solution Assessment

Unclear. Structured reasoning output items are directionally aligned with the request, but the best maintainable fix still needs maintainer decisions on privacy gating, event names, and whether Chat Completions should expose reasoning or remain Responses-first.
