## Summary

Keep open. Current main still drops Claude CLI thinking before it can reach the HTTP API, and the Responses/Chat Completions handlers remain assistant-text-only; related PR #56674 is open implementation work, not a merged fix.

## What This Changes

Expose Claude CLI thinking/reasoning on the Gateway HTTP `/v1/responses` surface and optionally `/v1/chat/completions`, gated by existing reasoning visibility controls.

## Best Possible Solution

Review and land #56674 or a refreshed replacement that preserves Claude `thinking_delta`, emits opt-in structured reasoning for streaming and non-streaming `/v1/responses`, defines any Chat Completions compatibility shape, and updates docs plus parser/gateway tests.

## Reproduction Assessment

Yes. The issue includes a concrete Claude CLI and curl repro, and current-main static inspection confirms the parser and HTTP handlers only carry assistant text to the client.

## Solution Assessment

Unclear. A structured Responses reasoning item is a plausible direction, but the maintainable fix needs explicit maintainer choices around privacy defaults, request opt-in, event naming, and Chat Completions compatibility.
