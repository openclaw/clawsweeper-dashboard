## What This Changes

## Best Possible Solution

Keep the shared Slack command normalizer approach, but source the bot's actual profile display name with a safe `auth.user` fallback, add focused coverage for that source, and require redacted live Slack proof before merge.

## Reproduction Assessment

No live end-to-end reproduction was established in this read-only review. The narrow source path is reproducible: current main strips Slack mentions only, so `ada stop` remains `ada stop` and does not match the abort trigger set.

## Solution Assessment

No: the shared normalizer is the right layer, but using `auth.test().user` as a display name is not the best fix for a display-name-prefixed command. Resolve the bot user's profile display name, fall back to the auth username, and prove the real Slack DM path.
