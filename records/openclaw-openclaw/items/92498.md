## Summary

This PR addresses the real Slack transcript-loss path, but it should not merge as written because ordinary final replies cannot reach the new post-hook capture branch, leaving persisted transcript content out of sync with the reply actually delivered. The repair is narrow, but after-fix real Slack proof is also still required.

The PR changes same-channel Slack final-reply delivery to mirror delivered assistant output into the owning session transcript and adds a regression test.

## Best Possible Solution
