## Summary

Keep open: the PR targets a real P1 Slack transcript-loss bug and includes useful live Slack proof, but the current head introduces a blocking transcript-ownership regression where CLI-owned finals can bypass post-delivery hook mirroring and persist raw or cancelled content.

## What This Changes

The PR adds same-channel Slack final-reply transcript mirroring, CLI assistant transcript persistence, session-binding refresh plumbing, and regression coverage for dispatcher hook rewrites/removals.

## Best Possible Solution

Make the successful post-hook delivered payload the transcript source of truth for channel-visible CLI finals, with regression tests for CLI-owned Slack/routed replies where hooks rewrite or remove content.

## Reproduction Assessment

Yes, source-level. A CLI-owned payload is pre-persisted and marked `assistantTranscriptOwned`, then dispatch skips the post-hook Slack mirror even though the dispatcher still runs hook rewrites/cancellations before visible delivery.

## Solution Assessment

No. Mirroring same-channel Slack finals is the right layer for the reported bug, but the current ownership shortcut is not the best fix because it bypasses the delivery-hook result for CLI-owned finals.
