## Summary

Keep open: the web_search owner fix is pointed at a real source-level bug and the PR includes real behavior proof, but the patch also broadens sandboxed web_fetch provider loading beyond bundled providers and lacks the required changelog entry.

## What This Changes

The PR removes bundled-origin filtering from web provider owner/provider resolution and adds tests for global web_search provider scoping.

## Best Possible Solution

Land a narrow fix that removes bundled-origin filtering only where explicit global provider ownership needs it, preserves sandboxed web_fetch bundled-only behavior, and records the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Yes. The current main source path filters explicit web_search owner lookup by origin: "bundled", and the manifest resolver excludes global owners when that filter is present; the PR body also supplies before/after live output for a Brave global-plugin setup.

## Solution Assessment

No, not as written. The web_search owner-resolution change is the right direction, but the web_fetch sandboxed branch should keep its bundled-only boundary rather than loading all plugin origins.
