## Summary

Current main already fixes the reported Azure Responses failure: unset Azure OpenAI API versions now resolve to `preview`, the focused test expectation was updated, and the fix is included in `v2026.5.14-beta.2`.

## What This Changes

The issue requests changing the `azure-openai-responses` default API version so `gpt-5.5` calls no longer fail with Azure 400 `API version not supported` when `AZURE_OPENAI_API_VERSION` is unset.

## Best Possible Solution

Keep the merged `preview` default with the existing environment override; users on `v2026.5.12` can use the env workaround or a newer beta until the next stable includes the fix.

## Reproduction Assessment

No on current main: the source path now resolves an unset Azure API-version environment to `preview`, and the focused test asserts that behavior. The pre-fix failure was source-proven and live-proven in the linked source PR.

## Solution Assessment

Yes. The merged change is the narrow maintainable fix because it changes the actual SDK `api-version` query default while preserving `AZURE_OPENAI_API_VERSION` for pinned Azure deployments.
