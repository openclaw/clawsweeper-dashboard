## Summary

Current main already fixes the reported Azure Responses failure: unset Azure OpenAI Responses calls now resolve to `api-version=preview`, the Azure client still honors explicit env overrides, and the merged replacement PR includes live Azure proof from the reporter's setup.

## What This Changes

The issue reports that `azure-openai-responses/gpt-5.5` fails with Azure 400 errors when `AZURE_OPENAI_API_VERSION` is unset because OpenClaw used an obsolete default API version.

## Best Possible Solution

Keep the merged `preview` default for Azure Responses while preserving `AZURE_OPENAI_API_VERSION` for pinned Azure deployments; handle image-generation API-version defaults separately if they become a concrete report.

## Reproduction Assessment

No on current main. The old source-level reproduction no longer exists because an unset Azure OpenAI API version now resolves to `preview`, and the linked replacement PR preserved live Azure proof for the original failure.

## Solution Assessment

Yes. The landed fix is the narrow maintainable change for this bug: it updates the Azure Responses transport default and focused test without changing unrelated image-generation behavior or removing explicit env overrides.
