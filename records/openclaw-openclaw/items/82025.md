## Summary

Keep open: current main still defaults the Azure Responses transport to `2024-12-01-preview`, while Microsoft’s current Azure OpenAI docs show Responses was added after that version and the v1 GA route no longer requires `api-version`. ([learn.microsoft.com](https://learn.microsoft.com/en-us/azure/ai-services/openai/api-version-lifecycle))

## What This Changes

The issue asks OpenClaw to stop sending an Azure OpenAI Responses API version that is too old for `azure-openai-responses/gpt-5.5` calls.

## Best Possible Solution

Update the Azure Responses transport so legacy Azure Responses requests use a supported version and `/openai/v1` GA routes are not forced through an obsolete `api-version` contract.

## Reproduction Assessment

Yes at source level, but not with a live Azure credentialed call: unset `AZURE_OPENAI_API_VERSION` flows into `AzureOpenAI` as `2024-12-01-preview` for `azure-openai-responses` requests.

## Solution Assessment

The proposed default bump is directionally correct, but the best fix should verify the Azure v1 GA route and avoid forcing `api-version` where Microsoft now recommends the OpenAI client shape.
