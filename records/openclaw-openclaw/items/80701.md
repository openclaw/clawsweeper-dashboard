## Summary

Close as configuration/support rather than an OpenClaw source bug: Telegram privacy mode only controls what Telegram delivers to the bot, while OpenClaw still requires its documented group and sender authorization config before group messages or mentions are admitted.

## What This Changes

The issue asks for Telegram group messages, including bot mentions, to respond instead of being skipped by the reported unmentioned-group filtering.

## Best Possible Solution

Keep the documented Telegram group access contract, and ask for a new actionable bug only if the documented group/sender config still fails with sanitized config and exact skip logs.

## Reproduction Assessment

Yes for the support symptom, source-level only: current main can skip Telegram group messages under the documented default allowlist policy when the group or sender is not configured. That is expected behavior, not a reproduced source defect.

## Solution Assessment

Yes. The narrow maintainable answer is configuration guidance through `channels.telegram.groups`, `groupPolicy`, `groupAllowFrom`/`allowFrom`, and `requireMention`, not a core bypass of Telegram group admission.
