## Summary

This should close as a Telegram setup/configuration report rather than a core bug: current source, schema, tests, and docs all show that Telegram group access is intentionally fail-closed until group and sender authorization are configured, and @mentions are evaluated only after that access gate passes.

## What This Changes

The issue reports that Telegram bots skip all group messages, including mentions, because group-message handling is blocked by an internal setting that is not user configurable.

## Best Possible Solution

Keep the existing Telegram group authorization model and route this case to configuration: add the group under channels.telegram.groups and authorize senders through groupAllowFrom or allowFrom, or intentionally set groupPolicy to open for broad group access.

## Reproduction Assessment

Yes for the observed skip path, but not for a core bug: source inspection shows Telegram group messages are skipped when the group or sender is not authorized by OpenClaw config. The issue lacks a redacted effective config showing that the documented group and sender authorization were already present.

## Solution Assessment

Yes. The current design keeps group authorization separate from mention detection, so @mentions should not bypass groupPolicy/groups/groupAllowFrom; the maintainable path is configuration or a new report with proof that configured authorization is still ignored.
