The issue requests bounded timeout and cleanup behavior for `scpFile()` during remote sandbox media staging.

## Best Possible Solution

## Reproduction Assessment

Yes, source inspection of current main shows `scpFile()` spawns `scp` and only settles on child `error` or `exit`, so a child that never exits leaves the Promise pending. I did not run a live hung-SCP transfer in this read-only review.

## Solution Assessment

The best fix is at the `scpFile()` boundary so all remote media staging callers get bounded cleanup. The timeout duration and slow-transfer failure policy still need maintainer acceptance, so the open linked PR remains the review path.
