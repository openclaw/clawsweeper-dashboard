## Summary

Keep open: current main and the latest release still lack an SCP timeout in sandbox media staging, and the same-author open PR is the active fix path but has not merged.

## What This Changes

The issue requests adding a timeout and cleanup guard to sandbox media SCP staging so hung remote transfers do not leave child processes and promises alive indefinitely.

## Best Possible Solution

Land or replace the linked PR with a bounded scpFile timeout that kills and settles hung children, preserves existing SCP sanitization, and includes maintainer-approved timeout policy plus focused cleanup coverage.

## Reproduction Assessment

Yes, source-reproducible: current main's scpFile Promise only settles on child error or exit, so a spawned scp that never emits either leaves staging pending. I did not run a live hung-network SCP reproduction in this read-only review.

## Solution Assessment

Unclear. A timeout in scpFile is the right boundary, but the exact deadline and slow-transfer behavior need maintainer acceptance; the linked PR is the active candidate.
