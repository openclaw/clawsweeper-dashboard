## Summary

Keep open: current main and the latest release still have an unbounded `scp` subprocess path, and the linked contributor PR is open rather than merged.

## What This Changes

The issue reports that sandbox remote media staging spawns `scp` without a timeout, so hung transfers can leave a child process and staging Promise alive indefinitely.

## Best Possible Solution

Keep this issue open until a bounded SCP timeout with maintainer-approved deadline behavior and focused regression coverage lands, likely via https://github.com/openclaw/openclaw/pull/98141 or a replacement.

## Reproduction Assessment

Yes, source-reproducible: current main `scpFile` awaits only child `error` or `exit`, so a child that remains running keeps the Promise pending. I did not run a live hung SCP transfer in this read-only review.

## Solution Assessment

Unclear. Adding a bound in `scpFile` is the right layer for the hang, but the specific timeout duration and failure behavior need maintainer acceptance because they can change slow-transfer behavior.
