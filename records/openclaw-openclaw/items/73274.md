## Summary

Keep open: current main and v2026.6.6 still lack the requested public plugin runtime API for persistent cross-session transcript appends. The reporter’s workaround plugin is useful context, but the remaining work is a plugin API and security-boundary decision, with a related implementation PR still open and conflicting.

## What This Changes

Request to expose a persistent plugin-facing API for appending assistant messages to arbitrary session transcripts so cron/channel plugins can preserve cross-session context beyond in-memory system events.

## Best Possible Solution

Define and land a narrow, documented, trusted transcript-write API with explicit authorization and idempotency semantics, either by reworking the related PR or choosing a different runtime-session method.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source review on current main and v2026.6.6 shows the public append API is absent while the internal helper and ephemeral system-event workaround exist.

## Solution Assessment

Unclear: a public transcript-write seam may be the right direction, but exposing the raw internal helper is not automatically the best boundary until maintainers settle trust, audit, idempotency, and message-shape semantics.
