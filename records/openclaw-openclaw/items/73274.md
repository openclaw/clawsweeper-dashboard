## Summary

Keep open. Current main still does not expose a durable transcript append helper on `api.runtime.agent.session`; the existing in-memory system-event and durable next-turn injection seams do not create permanent session transcript history, and this API needs Plugin SDK/security design before implementation.

## What This Changes

Expose a plugin runtime API for durably appending assistant-style messages to another session transcript, mainly for cron-to-channel context mirroring.

## Best Possible Solution

Define a constrained, documented Plugin SDK transcript-append contract with explicit capability or operator opt-in, assistant-only default semantics, idempotency, redaction/audit expectations, transcript update behavior, and contract tests.

## Reproduction Assessment

Not applicable as a feature/API request. The high-confidence static check is that current main's runtime types, runtime wiring, docs, and source search omit a plugin transcript append helper while the internal helper and tests exist.

## Solution Assessment

Unclear. Exposing the raw internal helper, especially with a `role` option, is not proven to be the safest design; a constrained authorized assistant-only API or cron/delivery-owned mirror path would be safer.

## Security Review

Status: needs_attention

Summary: The requested API is security-sensitive because it would grant plugins durable cross-session transcript write authority.

Concerns:

- **[medium] Require authority for cross-session transcript writes:** `src/plugins/runtime/types-core.ts:146`
  - body: Adding an append helper under `api.runtime.agent.session` would let plugin code persist content into arbitrary session transcripts, so it should require an explicit trusted capability or operator opt-in rather than ordinary hook access alone.
  - confidence: 0.9
- **[medium] Keep public writes assistant-only unless redesigned:** `src/config/sessions/transcript.ts:225`
  - body: The existing exact helper rejects non-assistant messages, while the proposal includes a `role` option. User-role transcript writes need clear attribution, replay, and audit semantics before becoming public API.
  - confidence: 0.88
- **[medium] Preserve transcript update and idempotency semantics:** `src/config/sessions/transcript.ts:283`
  - body: The safe internal path writes through `appendSessionTranscriptMessage`, dedupes idempotency keys, and emits transcript update events; a public API should preserve those behaviors instead of encouraging manual JSONL appends.
  - confidence: 0.86
