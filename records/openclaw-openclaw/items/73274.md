## Summary

Keep open. Current main still lacks a durable transcript append helper on the plugin runtime surface and the related workaround plugin only confirms the in-memory system-event seam; the related open PR at https://github.com/openclaw/openclaw/pull/80208 is not merged and leaves Plugin SDK/security design unresolved.

## What This Changes

Request exposing a durable Plugin Runtime API so plugins can append assistant transcript messages to other sessions.

## Best Possible Solution

Define a constrained, documented Plugin SDK/runtime transcript-append contract with explicit authority, assistant-attributed writes, idempotency/update-event semantics, docs, API/export checks, and focused contract tests.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than broken documented behavior. The high-confidence static check is that current main omits the durable append API from runtime types, runtime wiring, SDK exports, docs, and plugin runtime mocks while the internal helper exists.

## Solution Assessment

Unclear. Exposing the raw internal helper, especially with a role option or exact-message writes, is not the safest proven contract; a constrained authority-gated assistant-write API is the more maintainable direction.

## Security Review
