## Summary

Keep open: current main and v2026.6.8 still lack the requested public plugin transcript-append API. The internal append helper and built-in cron delivery mirroring exist, and the linked workaround plugin shows real demand, but the remaining work is a public plugin API plus session-state security decision rather than completed core behavior.

## What This Changes

Request to expose a durable plugin-facing API for appending assistant messages to another session transcript instead of relying on in-memory system-event workarounds.

## Best Possible Solution

Keep this as the canonical product/security discussion until maintainers choose and document a permission-aware transcript-write seam, then land it with SDK docs, export/baseline updates, and focused session-state tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source and release inspection confirm the requested public append API is absent while the internal helper and in-memory system-event workaround exist.

## Solution Assessment

Unclear: a durable plugin transcript-write seam is aligned with the plugin roadmap, but maintainers still need to choose between an api.runtime.agent.session method, a narrow SDK subpath, or a more constrained permission-aware writer.
