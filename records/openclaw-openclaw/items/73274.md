## Summary

Keep open. Current main still lacks the durable plugin runtime transcript append API, and the existing workaround plugin confirms a real gap that is blocked on public Plugin SDK and security design rather than cleanup closure.

## What This Changes

The issue requests a durable `api.runtime.agent.session` helper for plugins to append assistant messages into another session's transcript.

## Best Possible Solution

Define a constrained, documented Plugin SDK transcript-append contract with explicit capability or operator opt-in, assistant-only default semantics, idempotency, transcript update behavior, and contract tests.

## Reproduction Assessment

Not applicable as a feature/API request. The high-confidence static check is that current main has the internal durable helper and ephemeral system events, but no public plugin runtime transcript append API.

## Solution Assessment

Unclear. Exposing the raw internal helper, especially with a `role` option, is not proven to be the safest shape; a narrower authorized assistant-message API or cron/delivery-owned mirror path would be safer.

## Security Review

Status: needs_attention

Summary: The requested public API is security-sensitive because it would let plugin code persist content into arbitrary session transcripts.

Concerns:

- **[medium] Require authority for cross-session transcript writes:** `src/plugins/runtime/types-core.ts:156`
  - body: A helper under `api.runtime.agent.session` would grant durable cross-session transcript mutation, so it should require an explicit capability or operator opt-in instead of ordinary hook access alone.
  - confidence: 0.9
- **[medium] Keep public transcript writes assistant-only by default:** `src/config/sessions/transcript.ts:216`
  - body: The existing exact append helper rejects non-assistant messages; public user-role writes need clear attribution, replay, and audit semantics before being exposed.
  - confidence: 0.88
- **[medium] Preserve idempotency and transcript update semantics:** `src/config/sessions/transcript.ts:262`
  - body: Any public API should preserve the internal path's idempotency checks, transcript-manager append, and update-event behavior so plugins are not pushed toward manual JSONL writes.
  - confidence: 0.86
