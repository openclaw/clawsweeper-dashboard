## Summary

Keep open. Current main now fixes the cron-specific destination transcript mirror through https://github.com/openclaw/openclaw/pull/80786, but it still does not expose a supported plugin runtime or SDK transcript-append contract; the remaining generic API request overlaps the still-open https://github.com/openclaw/openclaw/pull/80208 and needs Plugin SDK/security authority design.

## What This Changes

Expose a durable transcript-append API for plugins so they can write assistant messages into a selected session transcript.

## Best Possible Solution

Define a constrained, documented plugin runtime/SDK transcript-append contract with explicit authority, assistant-attributed writes, idempotency/update-event semantics, docs, API/export checks, and focused contract tests, or explicitly decide that cron mirroring is core-owned and the generic plugin API is out of scope.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. The high-confidence static check is that current main still lacks an append method on `api.runtime.agent.session` and lacks a `transcript.runtime` package export while the internal durable helper exists.

## Solution Assessment

Unclear. The cron-specific problem now has a narrow core fix, but exposing raw transcript helpers is not the safest plugin contract; a constrained capability-gated runtime API is the maintainable direction if plugin writes remain desired.
