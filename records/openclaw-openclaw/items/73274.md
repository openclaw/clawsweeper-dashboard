## Summary

Keep open. Current main still lacks a durable transcript append helper on `api.runtime.agent.session`; the linked `openclaw-plugin-cron-context` workaround confirms the need but relies on nonpersistent system events, and the requested cross-session transcript writer needs Plugin SDK/security design before implementation.

## What This Changes

Expose a durable plugin runtime API for appending assistant messages to a target session transcript, mainly so cron output can be mirrored into channel session history.

## Best Possible Solution

Ship a constrained, documented Plugin SDK transcript append contract with explicit authority or operator opt-in, assistant-only default semantics, idempotency/update-event preservation, and focused contract tests.

## Reproduction Assessment

Not applicable as a feature/API request. The high-confidence static check is that current main omits a plugin runtime transcript append helper while the internal helper and tests already exist.

## Solution Assessment

Unclear. The use case is valid, but exposing the raw internal helper or a role-selecting writer is not proven safest; a constrained assistant-only API or cron/delivery-owned mirror path would be safer.

## Security Review

Status: needs_attention

Summary: The requested API is security-sensitive because it would grant plugins durable cross-session transcript write authority.

Concerns:

- **[medium] Require authority for transcript writes:** `src/plugins/runtime/types-core.ts:156`
  - body: Adding an append helper under `api.runtime.agent.session` would let plugin code persist content into arbitrary session transcripts, so it should require an explicit capability or operator opt-in rather than ordinary hook access alone.
  - confidence: 0.9
- **[medium] Keep public writes assistant-only unless redesigned:** `src/config/sessions/transcript.ts:229`
  - body: The internal exact helper rejects non-assistant messages, while the proposal includes a `role` option; user-role writes need clear attribution, replay, and audit semantics before becoming public API.
  - confidence: 0.88
- **[medium] Preserve append safety semantics:** `src/config/sessions/transcript.ts:262`
  - body: The safe internal path deduplicates idempotency keys, appends through the transcript writer, and emits transcript update events; a public API should preserve those behaviors instead of encouraging manual JSONL appends.
  - confidence: 0.86
