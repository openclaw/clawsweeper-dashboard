## Label Justifications

- `P2`: This is a normal-priority plugin API gap with a real third-party workaround and limited blast radius, not an urgent runtime outage.
- `impact:session-state`: The requested capability is about preserving cross-session transcript context so future sessions read the correct history.

## Summary

Keep open: current main already ships core cron delivery mirroring into destination session transcripts, but the requested public plugin runtime transcript append API is still absent and the related implementation PR has not landed.

## What This Changes

Request to expose a persistent transcript append helper through the plugin runtime so plugins can write assistant output into another session's JSONL transcript.

## Best Possible Solution

Keep this open until maintainers decide whether to land a narrow documented transcript append runtime API, accept the existing shipped cron mirror as sufficient, or redirect the broader API shape to the related PR.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high confidence that the requested public runtime helper is absent while the internal helper exists.

## Solution Assessment

Unclear until maintainers decide the API boundary. A direct helper may be the right narrow seam, but the shipped cron mirror reduces the immediate use-case pressure and cross-session transcript writes need explicit trust semantics.
