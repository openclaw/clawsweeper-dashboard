## Summary

Keep open. Current main still does not expose a durable transcript-append helper on the plugin runtime surface, and the requested capability crosses a Plugin SDK security boundary rather than being a cleanup-close candidate.

## What This Changes

Expose an appendAssistantMessageToSessionTranscript-like API under api.runtime.agent.session so plugins can persist messages into other sessions' transcripts.

## Best Possible Solution

Define a constrained Plugin SDK transcript-append capability with explicit operator opt-in, assistant-only mirror semantics unless deliberately expanded, idempotency, transcript update/audit/redaction behavior, docs, and contract tests.

## Reproduction Assessment

Not applicable as a feature/API request. The high-confidence static check is that current main lacks the requested plugin runtime helper while the internal durable helper and existing prompt-context alternatives are present.

## Solution Assessment

Unclear. Exposing the internal helper directly, especially with a role option, is not yet proven to be the safest path; a narrower authorized assistant-only API or cron-owned mirroring path should be designed first.

## Security Review

Status: needs_attention

Summary: The requested API is security-sensitive because it would create a durable cross-session transcript write path for plugin code.

Concerns:

- **[medium] Gate cross-session transcript write authority:** `src/plugins/runtime/types-core.ts:146`
  - body: Adding this under api.runtime.agent.session would let plugin code persist content into arbitrary stored sessions, so it should require an explicit trusted capability or operator opt-in rather than ordinary runtime access.
  - confidence: 0.9
- **[medium] Avoid public user-role writes without attribution design:** `src/config/sessions/transcript.ts:225`
  - body: The internal exact helper rejects non-assistant messages, while the proposed API includes a role option; user-role writes need caller identity, audit, and replay rules before public exposure.
  - confidence: 0.88
- **[medium] Preserve transcript update and audit semantics:** `src/config/sessions/transcript.ts:283`
  - body: The safe internal path appends through the session transcript helper and emits transcript update events, so a public API should define idempotency, update, redaction, and audit behavior instead of encouraging manual JSONL appends.
  - confidence: 0.84
