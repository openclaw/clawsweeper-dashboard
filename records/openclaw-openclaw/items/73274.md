## Summary

Keep open: current main and v2026.6.6 still do not expose a public plugin transcript-append API, and the remaining work is a plugin API plus security-boundary decision rather than stale cleanup. The linked workaround plugin and open related PR are useful context, but the PR is still open and not a merged or safe superseding resolution.

## What This Changes

Request to expose a plugin-accessible API for appending assistant messages to session transcripts so plugins can persist cross-session cron or context output.

## Best Possible Solution

Decide and document a narrow trusted plugin transcript-write contract, then either rework and land the related PR or add an api.runtime.agent.session method with security and compatibility coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source review shows current main and v2026.6.6 lack the requested public append API while the internal helper exists.

## Solution Assessment

Unclear: a transcript-write seam may be the right plugin API, but exposing the raw internal helper is not automatically the best boundary until maintainers settle trust, audit, idempotency, and message-shape semantics.
