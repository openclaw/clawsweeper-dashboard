## What This Changes

The PR adds start-argument path fallback for embedded read-image tool results and tests extractor and handler media delivery.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level: current main's extractor still returns no media for image content with no `MEDIA:` token or `details.path`, while `handleToolExecutionEnd` has start args but does not pass them into extraction. I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as currently shaped. The read start-arg fallback is the right direction, but this branch needs a current-main port and a read-specific trust gate before it is the maintainable fix.
