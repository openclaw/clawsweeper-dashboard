## Summary

Keep this PR open. Current main still lacks the start-args fallback for embedded read-image tool results, while the linked bug remains open and the overlapping PR #46268 was closed unmerged rather than shipped.

## What This Changes

The PR adds a start-args fallback for embedded read-image media extraction, threads tool start args into non-verbose tool-result media emission, and adds extractor plus handler regression tests.

## Best Possible Solution

Rebase or replace this PR with a narrow current-main patch that makes tool-result media extraction start-args-aware, or has the read tool attach structured details.media/details.path for successful image reads. Preserve existing raw-tool/builtin-tool trust checks, add coverage for path, file_path, filePath, and queued final outbound media delivery, then resolve the linked Feishu tracker.

Yes. The high-confidence code-level reproduction is an embedded read result with image content but no MEDIA: directive or details.path while the original tool start args contain file_path/path; current tests and source show that this leaves no media path to queue.

## Solution Assessment
