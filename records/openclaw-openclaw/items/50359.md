## Summary

Keep open. Current main still lacks the read-image start-args media fallback, while this PR is the canonical focused implementation path but now needs adaptation to the current media-artifact queueing code before merge.

## What This Changes

This PR falls back to the original read tool start args for image tool results without `details.path`, threads those args into embedded media emission, and adds extractor plus handler regression tests.

## Best Possible Solution

Land a current-main implementation that adds the start-args fallback to `extractToolResultMediaArtifact`, passes `startData.args` into `emitToolResultOutput`, preserves trusted local media filtering and artifact metadata, covers `path`/`file_path`/`filePath` plus handler queueing, then resolve https://github.com/openclaw/openclaw/issues/41744.
