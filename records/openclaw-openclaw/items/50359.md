## Summary

## What This Changes

The PR threads embedded tool start arguments into tool-result media extraction so pathless read-image results can fall back to `path`, `file_path`, or `filePath`, with extractor and handler regression tests.

## Best Possible Solution

Port the fallback onto current main's media artifact path, gate start-arg recovery to exact trusted `read` results, preserve local-media filtering, and prove real channel delivery with redacted evidence.

## Reproduction Assessment

Yes, by source inspection. Current main still returns no media for pathless read image content while original read args are available in the handler but not passed into media extraction.

## Solution Assessment

No, not as currently shaped. Falling back to read start args is the right narrow direction, but this PR needs a current-main port and a read-specific trust gate before it is the best mergeable solution.
