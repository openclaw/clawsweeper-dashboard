## Summary

## What This Changes

The PR passes `syncOfficialPluginInstalls` and normalized `update.channel` to the npm plugin updater only for `openclaw plugins update --all`, with CLI coverage that targeted updates remain unset.

## Best Possible Solution

Update the plugin update docs for the trusted-official `--all` exception, then land this narrow CLI routing if maintainers accept the compatibility-sensitive exact-pin behavior; leave broader manual exact-spec downgrade policy to https://github.com/openclaw/openclaw/issues/84256.

## Solution Assessment

Mostly yes. Using the existing official-sync and update-channel updater contract at the bulk CLI call site is the narrow maintainable fix, but docs need to describe the new `--all` exception before merge.
