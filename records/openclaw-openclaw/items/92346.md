## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Finalize stale runs before retry accounting:** `src/agents/subagent-orphan-recovery.ts:259-266`
  - body: This new stale branch increments `failed` and queues a `failedRuns` entry, but it never calls `finalizeInterruptedSubagentRun` itself; the scheduler only performs fire-and-forget finalization after retries are exhausted. Because staleness is not transient, a stale active run can remain active for the retry window and a zero-update or thrown finalization still has no observed result. Await retirement here and report `failed` only when the stale run cannot be finalized.
  - confidence: 0.86
- **[P2] Update stale-aborted recovery docs:** `src/agents/subagent-registry-helpers.ts:193`
  - body: Removing the `abortedLastRun` exemption makes stale restart-aborted restored runs prune instead of remain recoverable, but `docs/tools/subagents.md` still describes the old exception. Update the liveness-and-recovery docs before shipping this behavior change.
  - confidence: 0.91
