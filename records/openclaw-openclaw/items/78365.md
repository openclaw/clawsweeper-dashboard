## Summary

Keep open. Current main still has the cited `verbose: false` Discord outbound calls, but source inspection shows that flag feeds Discord retry logging rather than tool-progress rendering; the more plausible blocker is Discord channel/guild visibility policy suppressing draft/progress delivery, which needs maintainer judgment before changing behavior.

## What This Changes

The issue asks to restore Discord verbose/tool-progress updates in v2026.5.4 and remove the suspected hardcoded suppression in the Discord outbound adapter.

## Best Possible Solution

Clarify and align Discord progress delivery so `verboseDefault`, `toolProgressDetail`, and `channels.discord.streaming.mode="progress"` either produce documented updates in channels or diagnostics/docs explain the required visible-reply configuration.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection gives a plausible current-main path: Discord guild/channel delivery can suppress progress callbacks and drafts, while the cited `verbose` flag is only retry verbosity.

## Solution Assessment

No. Passing through the adapter's `verbose` option is probably not the best fix because that option does not render tool progress; the maintainable fix is in Discord source-reply/progress policy or documentation.
