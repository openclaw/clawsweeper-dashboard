Keep this open: current main still accepts `gateway.nodes.denyCommands` as a plain string array and only the security audit path detects ineffective unknown entries. A focused open PR already references this issue with closing syntax, so the issue should stay open until that PR or a replacement lands.

## What This Changes

The issue requests write-time validation or warning for invalid `gateway.nodes.denyCommands` entries so users do not rely on ineffective denylist values.

## Best Possible Solution

Keep the issue open and land a focused, rebased validator that warns during config mutation using the shared audit/runtime command catalog while preserving exact custom/plugin command IDs.

## Reproduction Assessment

Yes. Source inspection shows current main accepts any string array for `gateway.nodes.denyCommands`, while runtime removes deny entries only by exact trimmed string match and audit detection runs later.

## Solution Assessment

Yes in direction. The narrow maintainable fix is a config mutation warning that shares audit/runtime command vocabulary; `security audit --fix` alone would leave the misleading write-time success behavior in place.
