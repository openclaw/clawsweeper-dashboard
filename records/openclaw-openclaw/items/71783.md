## Summary

Keep open. Current main still leaves the reported metadata-free stale-name QMD conflict unresolved: verified path/pattern conflicts can be rebound, but a name-only stale `life-main` conflict can still skip creation of the configured `life` collection, and #72297 is the active narrow follow-up.

## What This Changes

The issue requests reconciling or clearly surfacing QMD path/pattern conflicts when an existing collection uses the same path and glob under a different name.

## Best Possible Solution

Land #72297 or an equivalent memory-core change that keeps automatic rebind limited to verified path/pattern metadata and adds a clear manual repair hint for unverified stale-name conflicts.

## Reproduction Assessment

Yes. The issue gives exact QMD stderr for the stale `life-main` versus configured `life` conflict, and current unit coverage models a name-only stale collection plus add conflict that still produces a skipped-add warning.

## Solution Assessment

Yes for the active PR direction, but no for current main. The narrow maintainable fix is metadata-gated automatic rebind plus diagnostic guidance for unverified conflicts, rather than deleting collections based only on stderr text.
