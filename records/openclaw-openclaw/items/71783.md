## Summary

Keep open. Current main still leaves the reported metadata-free stale-name QMD conflict on the skipped-add path, and PR #72297 is the active implementation candidate rather than a merged fix.

## What This Changes

OpenClaw should reconcile or clearly diagnose QMD collections registered under a stale name for the same path/pattern so configured memory searches do not query a missing collection.

## Best Possible Solution

Review and land PR #72297 or an equivalent memory-core change that keeps automatic removal limited to verified path/pattern matches while giving users a clear repair hint for unverified stale-name conflicts.

## Reproduction Assessment

Yes. The issue gives exact QMD stderr for the stale `life-main` versus configured `life` conflict, and current unit coverage models the metadata-free conflict path that still warns and skips.

## Solution Assessment

No for current main. The narrow maintainable fix is a verified rebind when QMD exposes path/pattern metadata plus an actionable diagnostic for name-only conflicts, rather than deleting collections based only on stderr text.
