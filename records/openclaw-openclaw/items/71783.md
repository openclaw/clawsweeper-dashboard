## Summary

Keep open. Current main still leaves the reported metadata-free stale-name QMD conflict on the skipped-add path, while PR #72297 is the active implementation candidate for surfacing a safer repair hint.

## What This Changes

The issue reports that a QMD collection already registered under a stale name can block OpenClaw from adding the configured collection name and cause later memory searches to fail.

## Best Possible Solution

Keep this open until PR #72297 or an equivalent memory-core change lands with metadata-gated automatic rebind plus a clear manual repair hint for unverified stale-name conflicts.

## Reproduction Assessment

Yes. The issue gives exact QMD stderr for the stale `life-main` versus configured `life` conflict, and current unit coverage models the metadata-free add-conflict path that still produces a skipped-add warning.

## Solution Assessment

Yes for the active PR direction, but no for current main. The narrow maintainable fix is verified path/pattern rebind plus a diagnostic repair hint when QMD exposes only the conflicting stale name, without deleting collections based only on stderr text.
