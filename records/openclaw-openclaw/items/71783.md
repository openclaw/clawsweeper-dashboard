## Summary

Keep open. Current main still handles only verified path/pattern QMD collection conflicts; the reported stale-name case can still fall through to a skipped add and leave searches using the configured collection name, while open PR #72297 remains the active implementation candidate.

## What This Changes

The issue asks OpenClaw to recover or clearly diagnose QMD collections registered under a stale name such as `life-main` when config expects `life`.

## Best Possible Solution

Keep this open until PR #72297 or an equivalent memory-core change lands with metadata-gated automatic rebind plus a clear manual repair hint for unverified stale-name conflicts.

## Reproduction Assessment

Yes. The issue includes exact QMD stderr for the `life` versus `life-main` conflict, and current unit coverage models the metadata-free add-conflict path that still produces a skipped-add warning.

## Solution Assessment

Yes for the active PR direction, but no for current main. The narrow maintainable fix is verified path/pattern rebind plus a diagnostic repair hint when QMD only exposes the conflicting stale name, without deleting collections based only on stderr text.
