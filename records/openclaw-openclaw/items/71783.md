## Summary

Keep open. Current main still leaves the reported QMD stale-name conflict unresolved when QMD only exposes a conflicting collection name without path/pattern metadata, and open PR #72297 remains the active implementation candidate for the diagnostic/repair-hint path.

## What This Changes

The issue requests QMD boot-time collection registration to reconcile or clearly report an existing collection at the same path/pattern when the configured collection name differs.

## Best Possible Solution

Keep this issue open until PR #72297 or an equivalent memory-core change lands. The desired end state is metadata-gated automatic rebind for proven path/pattern matches plus an explicit manual repair hint when only stderr exposes the stale conflicting collection name.

## Reproduction Assessment

Yes. The issue includes exact QMD stderr for the stale `life-main` versus configured `life` conflict, and current unit coverage models the metadata-free list plus add-conflict path that still logs a skipped-add warning instead of recovery.

## Solution Assessment

No for current main. The narrow maintainable solution is to keep destructive rebinds gated on structured metadata and add a clear stale-conflict repair hint for unverified stderr-only conflicts, which is the direction of PR #72297.
