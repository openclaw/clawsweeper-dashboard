## Summary

Keep open. Current main has restart deferral and shutdown drain mitigations, but it still lacks a durable accepted-inbound Slack/gateway store and replay path for events already acknowledged before SIGTERM; the related pending-inbound PR remains open and unmerged.

## What This Changes

The issue requests wiring a durable message store into the active gateway/Slack path so accepted in-flight messages survive SIGTERM and restart.

## Best Possible Solution

Land, split, or supersede the pending-inbound recovery work with a durable accepted-inbound handoff, startup replay, shutdown handling, idempotent delivery semantics, and regression proof for an accepted Slack turn interrupted before reply delivery.

## Reproduction Assessment

Yes, source-reproducible. Current Slack ingress accepts events into in-memory dedupe/debounce and live dispatch without a durable accepted-inbound store, so a SIGTERM after Socket Mode ACK can still lose the event.

## Solution Assessment

Unclear for the exact SQLite API shape, but durable accepted-inbound handoff is the right direction. The maintainable path is to review, land, split, or replace https://github.com/openclaw/openclaw/pull/49431 rather than create duplicate repair work.
