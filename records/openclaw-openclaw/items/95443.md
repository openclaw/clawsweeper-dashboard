## Best Possible Solution

Land one canonical lifecycle-generation guard for restart-recovery marking, keep stale-generation and running-session recovery covered by tests, then close this issue after the merged fix is on main.

A broader reset-policy or config change is not needed for this restart-recovery false positive, but maintainers still need to choose between the overlapping PRs.
