## Summary

Keep open: current main and v2026.5.2 still emit the reported generated env-wrapper-first LaunchAgent shape, but the exact live macOS failure mode and safe fix shape need maintainer confirmation because the existing docs/tests intentionally protect managed secrets outside plist EnvironmentVariables.

## What This Changes

The issue asks OpenClaw to stop doctor/update from rewriting macOS LaunchAgent plists into a wrapper-first shape that reportedly makes Node parse the generated env-wrapper shell script and crash the gateway.

## Best Possible Solution

Confirm the crash on macOS, then preserve owner-only env-file handling while making the generated wrapper invocation robust and covered by doctor/update regression tests.

## Reproduction Assessment

No high-confidence live reproduction in this review. Source inspection reproduces the wrapper-first plist writer on current main and v2026.5.2 when managed service env exists, but it does not prove the launchd-to-Node crash path.

## Solution Assessment

Unclear: the inline `EnvironmentVariables` workaround would address the visible plist shape but conflicts with the current secret-handling contract. If the crash reproduces, the safer fix is to keep owner-only env storage and make the wrapper invocation launchd-proof.
