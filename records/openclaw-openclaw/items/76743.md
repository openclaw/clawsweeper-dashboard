## Summary

Keep open: current main and v2026.5.2 still write the reported wrapper-first LaunchAgent shape when managed service env exists, while docs/tests show that shape is intentional for owner-only secret handling and the live launchd crash still needs macOS confirmation.

## What This Changes

Prevent doctor/update from rewriting macOS LaunchAgent plists into a wrapper-first `ProgramArguments` shape that reportedly makes the Gateway crash on launch.

## Best Possible Solution

Confirm the live launchd failure on macOS, then preserve owner-only env-file handling while making the LaunchAgent wrapper invocation launchd-proof and covered by doctor/update regression tests.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection reproduces wrapper-first `ProgramArguments` on current main and v2026.5.2 when managed env exists, but it does not prove that launchd makes Node parse the wrapper script.

## Solution Assessment

Unclear: the report's direct `EnvironmentVariables` workaround would address the visible plist shape but conflicts with the current secret-handling contract. A safer fix, if the crash reproduces, is to keep owner-only env storage and make the wrapper invocation robust under launchd.
