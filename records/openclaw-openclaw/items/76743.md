## Summary

Keep open: current main and v2026.5.2 still generate the reported wrapper-first LaunchAgent shape when managed service environment exists, while current docs/tests intentionally assert that owner-only wrapper design. The remaining blocker is live macOS launchd confirmation plus a security-preserving fix shape.

## What This Changes

Fix `openclaw doctor --fix` and update-triggered LaunchAgent rewrites so macOS managed gateway services do not crash when env-wrapper `ProgramArguments` are emitted.

## Best Possible Solution

Confirm the live launchd failure on macOS, then preserve owner-only env-file handling while making the LaunchAgent wrapper invocation launchd-proof and covered by doctor/update regression tests.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection reproduces wrapper-first `ProgramArguments` on current main and v2026.5.2 when managed env exists, but it does not prove the reported Node-parses-wrapper crash under launchd.

## Solution Assessment

Unclear: the direct inline-env workaround would address the visible plist shape but conflicts with the current secret-handling contract. If the crash reproduces, the safer direction is a launchd-proof wrapper invocation that keeps managed secrets out of the plist.
