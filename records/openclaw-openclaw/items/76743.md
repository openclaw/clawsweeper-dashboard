## Summary

Keep open: the wrapper-first LaunchAgent plist shape remains current-main behavior and is intentionally covered by docs/tests for owner-only environment handling, while the live launchd crash and safe fix shape still need macOS maintainer confirmation.

## What This Changes

The issue reports that doctor/update rewrites the macOS LaunchAgent to wrapper-first ProgramArguments, causing Node to parse the generated env wrapper and crash the Gateway.

## Best Possible Solution

Preserve owner-only managed service env handling unless maintainers explicitly approve a storage-model change, and add a macOS launchd-proof invocation path with regression coverage for install, doctor repair, and update rewrites.

## Reproduction Assessment

Unclear: source inspection gives a high-confidence path to the reported wrapper-first plist shape when managed service env exists, but not to the live macOS launchd crash where Node parses the wrapper. A macOS LaunchAgent smoke is still needed.

## Solution Assessment

Unclear: the direct EnvironmentVariables workaround would avoid the wrapper but bypasses the documented owner-only env-file design. A safer repair should first confirm launchd behavior, then keep file-backed env while making the wrapper invocation robust.
