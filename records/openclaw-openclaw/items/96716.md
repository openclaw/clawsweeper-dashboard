## What This Changes

The issue reports MCP tool calls failing when an optional MCP tool argument is supplied as JSON null through OpenClaw while the same call succeeds from other MCP clients.

## Best Possible Solution

Land one corrected, proof-backed PR that fixes null handling before MCP dispatch, preserves intentional nullable arguments, and adds regression coverage plus redacted real MCP proof.

## Reproduction Assessment

Yes, with medium confidence from source inspection and reporter evidence: current main and v2026.6.10 pass MCP tool input through, and validation can coerce null to an empty string for string schemas before execution. A live EKS MCP reproduction and raw wire capture were not run here.

## Solution Assessment
