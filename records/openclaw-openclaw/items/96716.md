## Summary

Keep open: current main and v2026.6.10 still allow nullable JSON-schema union arguments to be coerced through the string branch, and open linked pull requests now own the candidate fix path instead of this being implemented on main.

## What This Changes

The issue asks OpenClaw to preserve optional nullable MCP tool arguments instead of turning them into empty strings when a schema allows both string and null.

## Best Possible Solution

Land a focused validation-layer fix that preserves schema-declared null values for nullable unions, with regression coverage, then close this issue after the chosen PR merges.

## Reproduction Assessment

Yes, source-reproducible with medium confidence: current validation can convert null to an empty string when an anyOf union tries string before null. The reporter supplied live comparison evidence, but I did not run the awslabs EKS MCP server or capture the raw wire payload.

## Solution Assessment

Yes: the best fix is in the shared JSON-schema validation layer, where the null-to-empty-string conversion happens. Stripping nulls later at the MCP boundary would be broader and can erase meaningful schema-declared nullable values.
