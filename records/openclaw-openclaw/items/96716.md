## What This Changes

The issue reports that MCP tool calls with optional nullable arguments can behave as if null was sent as an empty string in OpenClaw.

## Best Possible Solution

Keep this issue open while maintainers review the candidate PRs and land one schema-safe fix with regression coverage and redacted real MCP proof.

## Reproduction Assessment

Yes from source, but not from a live MCP server in this review. Current validation can coerce null to an empty string for an anyOf string/null schema, and the MCP call path forwards its validated input to the runtime.

## Solution Assessment

Unclear until the linked PRs are resolved. The safest fix appears to preserve null through validation and only omit nulls when the MCP input schema proves absence is the intended optional-field representation.
