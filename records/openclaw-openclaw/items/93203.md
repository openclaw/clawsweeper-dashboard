## Summary

Keep this open: current main and v2026.6.8 still intentionally keep Mattermost DMs non-threaded, while this request asks for a new default-off opt-in DM threading contract that needs maintainer product/config direction.

## What This Changes

Request to add a Mattermost config option that lets direct-message conversations start thread-scoped sessions instead of always staying flat.

## Best Possible Solution

Have maintainers choose the Mattermost DM threading config shape, then implement it in the plugin with default off, schema/docs/tests, session/history handling, and live Mattermost proof.

## Reproduction Assessment

Yes. Source inspection shows direct Mattermost chats still force replyToMode off, suppress reply ids, and skip direct-message history keys; I did not run a live Mattermost workspace.

## Solution Assessment

Unclear until maintainers choose the config contract. A default-off DM-specific mode or Slack-style per-chat-type override is plausible, but current docs and tests intentionally preserve flat DMs.
