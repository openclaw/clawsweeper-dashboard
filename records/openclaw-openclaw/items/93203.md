## Summary

Keep this open: current main and the latest release still keep Mattermost direct messages flat and linear, while the requested opt-in DM threading mode is a new config/product contract that needs maintainer direction.

## What This Changes

The issue asks for a default-off Mattermost DM threading config, such as `channels.mattermost.dmReplyToMode`, so direct-message mentions can opt into thread-scoped sessions.

## Best Possible Solution

Maintainers should choose the DM threading contract, then land a default-off plugin-owned schema/docs/tests change with live proof that preserves existing flat DMs.

## Reproduction Assessment

Yes. Source inspection shows direct Mattermost chats still force `replyToMode` off, suppress DM reply roots, and skip DM history keys; I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. A default-off `dmReplyToMode` or Slack-like per-chat-type shape is plausible, but current docs and tests define flat DMs, so this should not be treated as a strict bug fix.
