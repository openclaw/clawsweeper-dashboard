## Summary

Keep open: current main and the latest release still make Mattermost direct messages non-threaded, and this issue is the focused remaining product/config decision split from the closed umbrella. The source gap is real, but adding opt-in DM threading changes a documented Mattermost contract and should wait for maintainer direction on the config shape.

## What This Changes

Request adding an opt-in Mattermost direct-message threading config so DM mentions can create independent thread-scoped sessions while defaulting to flat DMs.

## Best Possible Solution

If maintainers accept the direction, add a default-off Mattermost-owned DM threading contract with schema/types/docs/tests and live Mattermost proof, while preserving flat DMs for existing users.

## Reproduction Assessment

Yes. Source inspection shows direct chats still force replyToMode off, suppress reply ids, and skip direct-message history keys; I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. A default-off DM-specific mode or per-chat-type reply mode is plausible, but current docs intentionally define Mattermost DMs as flat.
