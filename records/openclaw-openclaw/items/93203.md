## Summary

Keep open: current main and v2026.6.8 still hard-code Mattermost direct messages as non-threaded, while this asks for a new opt-in DM threading/config contract that needs maintainer product review.

## What This Changes

Request to add a default-off Mattermost DM threading configuration so direct-message @mentions can create independent thread-scoped sessions.

## Best Possible Solution

Maintainers should choose the DM threading knob shape, then implement it inside the Mattermost plugin with default-off behavior, schema/docs/tests, session/history coverage, and live Mattermost proof.

## Reproduction Assessment

Yes. Source inspection on current main shows direct chats force replyToMode off, suppress reply/root ids, and skip direct-message history keys; I did not run a live Mattermost workspace.

## Solution Assessment

Unclear until maintainers choose the config contract. A default-off DM-specific mode or a per-chat-type mode are plausible, but this should not be auto-implemented as a strict bug fix because current docs define flat DMs.
