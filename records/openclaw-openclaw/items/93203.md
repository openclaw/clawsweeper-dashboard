## Summary

Keep open: current main and v2026.6.6 still intentionally keep Mattermost DMs non-threaded, while this issue asks for a new opt-in DM threading contract that needs maintainer product/config approval.

## What This Changes

Request to add a Mattermost DM-specific threading option so direct-message mentions can start independent thread-scoped sessions without changing the current default.

## Best Possible Solution

Approve a single Mattermost plugin contract for opt-in DM threading, keep the default flat-DM behavior, then update schema/types/docs and focused tests around direct-chat reply mode, root-id resolution, and session/history isolation.

## Reproduction Assessment

Yes, source inspection shows the requested behavior is absent: direct Mattermost chats force replyToMode off, suppress reply roots, and have tests/docs preserving flat DMs. I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

Unclear until maintainers decide the contract. The proposed default-off knob is plausible and backward-compatible, but the repository's config bar means the exact shape should be approved before a PR.
