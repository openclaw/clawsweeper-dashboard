## Summary

Keep this open: current main and the latest release still preserve Mattermost DMs as non-threaded, while this request asks for a new opt-in DM threading contract that needs maintainer product/config review.

## What This Changes

The issue requests a Mattermost config option so direct-message mentions can opt into independent threaded sessions while defaulting existing DMs to non-threaded behavior.

## Best Possible Solution

If maintainers accept the product direction, add an explicit opt-in Mattermost DM threading contract in the plugin with default off, schema/docs/tests, session/history coverage, and live Mattermost proof.

## Reproduction Assessment

Yes. Source inspection shows direct chats still force replyToMode off, suppress reply ids, and skip direct-message history keys; I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. A default-off DM-specific mode is a plausible safe shape, but the existing documented flat-DM behavior means this should not be auto-implemented as a strict bug fix.
