## Summary

Keep this open: current main and the latest release still define Mattermost DMs as flat/non-threaded, while this request asks for a new default-off DM threading config that needs maintainer product approval.

## What This Changes

Mattermost direct-message threading is requested to become configurable so DMs can opt into thread-scoped sessions while preserving the current flat default.

## Best Possible Solution

Approve or reject a default-off Mattermost DM threading contract; if approved, implement it in the plugin schema/runtime/docs with focused tests and real Mattermost proof for both new opt-in behavior and existing defaults.

## Reproduction Assessment

Yes. Source inspection shows direct chats still force `replyToMode` off, suppress direct-message reply roots, and skip direct-message history keys; I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. A default-off DM-specific mode is plausible, but current docs and tests define flat DMs, so this should not be treated as a strict bug fix.
