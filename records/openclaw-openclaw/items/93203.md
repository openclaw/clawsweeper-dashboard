## Summary

Keep this open: current main and the latest release still hardcode Mattermost DMs as non-threaded and have no accepted DM-specific threading config, while this request is a valid plugin-owned feature/config decision rather than an already-fixed bug.

## What This Changes

Request to add a backward-compatible Mattermost config knob so direct messages can opt into thread-scoped sessions instead of always using one flat DM session.

## Best Possible Solution

Keep this issue open for a maintainer decision on a Mattermost-owned, default-off DM threading option, then implement it across schema/types/docs and the inbound, outbound, thread-session, and history paths with focused tests.

## Reproduction Assessment

Yes, for current behavior: source inspection shows direct Mattermost chats force replyToMode off, suppress effective thread ids, and skip history keys. I did not run a live Mattermost repro because this read-only review could not exercise a real server path safely.

## Solution Assessment

Unclear until maintainers choose the config shape. The proposed default-off opt-in is the safest direction if accepted, but maintainers still need to decide whether that should be a new dmReplyToMode knob or a change to replyToMode semantics.
