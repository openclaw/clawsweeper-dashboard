## Summary

Keep this open: current main and v2026.6.8 still enforce flat Mattermost DMs, while this asks for a new default-off DM threading/config contract that needs maintainer product approval.

## What This Changes

Request to add a default-off Mattermost direct-message threading config so DMs can opt into thread-scoped sessions instead of always staying flat.

## Best Possible Solution

Maintainers should either approve a default-off Mattermost DM threading contract with schema, docs, tests, and live proof, or explicitly keep the documented flat-DM behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows direct chats force replyToMode off, suppress DM reply roots, and skip DM history keys; I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. A default-off DM-specific mode is plausible, but current docs and tests define flat DMs, so this should not be treated as a strict bug fix.
