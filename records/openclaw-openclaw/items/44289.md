## Summary

Keep open. Current main still only generates the JSON SecretRef credential matrix; the Markdown credential surface remains a hand-maintained marked list that is checked for drift after the fact, so the reported branch-churn problem remains actionable.

## What This Changes

Generate the SecretRef reference docs from secret target registry metadata so contributors do not manually sync multiple reference artifacts.

## Best Possible Solution

Add a single SecretRef docs generation/check path that derives both the JSON matrix and the Markdown supported/unsupported blocks from the canonical registry-backed matrix, with stable ordering and the existing docs test retained as a regression guard.

## Reproduction Assessment

Yes. The maintenance gap is reproducible by static inspection on current main: the generator writes only the JSON matrix while the Markdown surface has separate marked lists that the test merely compares after drift exists.

## Solution Assessment

Yes. Extending the existing registry-backed generator to update/check both artifacts is the narrowest maintainable fix; it avoids changing SecretRef runtime behavior and keeps the docs prose outside the generated block intact.
