## Best Possible Solution

Land only after a Gateway/WebChat owner approves the opt-in config and SSE event contract, or revise the event name/scoping before it becomes documented behavior.

## Reproduction Assessment

## Solution Assessment

Mostly yes: an opt-in non-transcript progress event avoids fake assistant deltas and keeps the default off. The remaining question is whether maintainers approve this exact public config/event contract and default WebChat scoping.
