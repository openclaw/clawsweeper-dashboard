## Summary

## What This Changes

The branch extends `openclaw secrets audit` to scan `.env`, `auth-profiles.json`, and generated `models.json` backup siblings for plaintext residue, adds tests, wraps `tryReadSecretFileSync`, and carries an OpenRouter routing cleanup already present on current main.

## Best Possible Solution

Land the sidecar backup residue scan after preserving fs-safe's fail-closed helper contract, scanning safe default sidecar backups even when active config is invalid, and keeping config artifact scanning aligned with https://github.com/openclaw/openclaw/pull/83275.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the review findings: the PR wrapper catches fs-safe policy errors, current main expects those errors to throw, and sidecar backup scanning is skipped when `snapshot.valid` is false.

## Solution Assessment

No. The better solution keeps the backup-audit addition but preserves fs-safe helper semantics, runs sidecar backup discovery independently of active-config validity, and refreshes real CLI proof at the current head.
