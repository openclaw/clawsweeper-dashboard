## Summary

Keep open. The security-audit direction is useful and has real CLI proof, but the patch still lets plaintext auth-profile backups feed active `REF_SHADOWED` state, so it can produce misleading security diagnostics before merge.

## What This Changes

The PR extends `openclaw secrets audit` to scan sibling `.env`, `auth-profiles.json`, and generated `models.json` backup files for plaintext residue while suppressing backup-only SecretRef resolution and adding tests.

## Best Possible Solution

Keep the env/auth/models backup scan, but make backup collectors residue-only for both ref resolution and active provider shadowing while leaving config backup artifacts to https://github.com/openclaw/openclaw/pull/83275.

## Reproduction Assessment

Yes, source-reproducible. The PR body provides real CLI output for the backup scan, and the diff shows backup auth-store scans still use the active auth collector with only ref collection disabled.

## Solution Assessment

No, not as written. The scoped backup enumeration is maintainable, but backup scans need a residue-only mode that does not affect active precedence or shadowing diagnostics.
