## Summary

Keep this PR open, but it is not merge-ready: the backup scanner misses OpenClaw’s numbered config backup ring and also treats backup-only SecretRefs like active refs, which can create false security assurance or new `secrets audit --check` failures.

## What This Changes

The PR extends `secrets audit` to scan sibling backup files for `openclaw.json`, `.env`, `auth-profiles.json`, and generated `models.json`, with focused test coverage.

## Best Possible Solution

Reuse or align with the existing config-backup artifact approach from https://github.com/openclaw/openclaw/pull/83275, then add any unique env/auth/models backup scanning without resolving backup-only SecretRefs and prove it with real `openclaw secrets audit --json` output.

## Reproduction Assessment

Yes, source-reproducible. Current main has no backup enumeration, while the PR matcher skips `.bak.1` despite OpenClaw’s backup ring creating it, and the PR routes backup configs through active SecretRef resolution.

## Solution Assessment

No, not as written. The maintainable path is to scan actual backup artifact patterns for plaintext residue while keeping backup-only refs out of active unresolved/shadowing checks.
