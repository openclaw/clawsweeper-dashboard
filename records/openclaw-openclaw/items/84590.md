## Summary

Keep open: this is a useful security-audit expansion with sufficient real CLI proof, but the current patch still lets auth-profile backup plaintext feed active provider-shadowing state and can emit misleading `REF_SHADOWED` diagnostics.

## What This Changes

The branch extends `openclaw secrets audit` to scan `.env`, `auth-profiles.json`, and generated `models.json` sibling backups for plaintext residue while suppressing backup-only SecretRef resolution and adding focused tests.

## Best Possible Solution

Keep the env/auth/models backup scan, but make backup collectors residue-only for both ref resolution and active provider shadowing, while leaving `openclaw.json` backup artifacts to the existing config-backup PR.

## Reproduction Assessment

Yes. The source path is clear: the new backup scan calls the active auth collector with only ref collection disabled, and the PR body's own CLI output shows a backup-only run with `shadowedRefCount: 1`.

## Solution Assessment

No, not as written. The backup enumeration is a good boundary, but the auth backup path needs a residue-only mode that cannot mutate active shadowing state.
