## Summary

Keep open: the backup-scan direction and CLI proof are useful, but the current head still lets auth-profile backup plaintext update active provider-shadowing state, so stale backup credentials can produce misleading `REF_SHADOWED` diagnostics.

## What This Changes

The PR extends `openclaw secrets audit` to scan `.env`, `auth-profiles.json`, and generated `models.json` backup siblings for plaintext secret residue, plus a small OpenRouter provider-routing spread cleanup.

## Best Possible Solution

Keep the env/auth/models backup scan, but make auth-profile backup collection residue-only for both SecretRef resolution and active provider shadowing while leaving `openclaw.json` backup artifacts to the existing config-backup PR.

## Reproduction Assessment

Yes. Source inspection is enough: the new backup auth scan calls the shared auth collector with only ref collection disabled, and plaintext credentials still update the same provider state used by `collectShadowingFindings`.

## Solution Assessment

No, not as written. The backup enumeration is a good boundary, but the auth-profile backup path needs a residue-only mode that cannot mutate active precedence diagnostics.
