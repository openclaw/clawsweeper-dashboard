## Summary

Keep open: the latest head still lets auth-profile backup plaintext update active shadowing state, so historical backups can produce misleading live `REF_SHADOWED` diagnostics.

## What This Changes

The PR extends `openclaw secrets audit` to scan `.env`, `auth-profiles.json`, and generated `models.json` backup siblings for plaintext residue while suppressing backup-only SecretRef resolution.

## Best Possible Solution

Keep the `.env`, auth-profile, and generated models backup scan, but make backup collectors residue-only so historical files cannot affect active SecretRef resolution or precedence diagnostics.

## Reproduction Assessment

Yes. Source inspection of the PR head shows backup auth scans set only `includeRefs: false`, while the plaintext and OAuth branches still call `trackAuthProviderState`; the PR body's redacted CLI proof also reports `shadowedRefCount: 1` in the backup-only scenario.

## Solution Assessment

No, not as written. The backup enumeration is a good boundary, but backup files need a residue-only collection mode that cannot mutate active precedence or SecretRef resolution state.
