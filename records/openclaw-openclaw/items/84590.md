## Summary

## What This Changes

The PR extends `openclaw secrets audit` to scan `.env`, `auth-profiles.json`, and generated `models.json` backup siblings for plaintext residue, suppresses backup-only SecretRef resolution/shadowing, and adjusts a secret-file helper plus OpenRouter routing spread cleanup.

## Best Possible Solution

Land a residue-only sidecar backup scan that runs independently of active-config validity, leaves `openclaw.json` backup/rejected artifacts to https://github.com/openclaw/openclaw/pull/83275, and records the new `--check` failure mode as an intentional security-audit compatibility change.

## Reproduction Assessment

Yes. Source inspection of PR head is enough: backup discovery and collection are inside `if (snapshot.valid)`, while the invalid-config branch bypasses them and only later scans active `.env` plus legacy auth files.

## Solution Assessment

No. The maintainable shape is to keep the residue-only collectors but run sidecar backup discovery from state-dir/default paths even when active config parsing fails, with regression coverage and refreshed exact-head proof.
