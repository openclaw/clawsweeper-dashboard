## What This Changes

The PR adds a cron model preflight API-key resolver and sends an Authorization bearer header on local provider GET /models probes.

## Best Possible Solution

Resolve the probe credential through the established provider auth/discovery credential contract, filter non-runnable markers, preserve authless local providers, and prove the change with focused tests plus redacted after-fix provider logs.

## Reproduction Assessment

Yes. Source inspection of current main shows local cron preflight calls GET /models or /api/tags with no headers, and the adjacent mocked test confirms 401 is a reachable response path rather than a failure path.

## Solution Assessment

No. Adding auth to the probe is the right direction, but this implementation should use the established discovery credential contract and marker filtering instead of raw config/env probing.
