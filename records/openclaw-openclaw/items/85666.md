## What This Changes

## Best Possible Solution

Keep the Anthropic API-key skip behavior, but express no-fallback through an explicit typed usage-auth result or a maintainer-approved documented contract, with SDK/contract tests and real status-path proof before merge.

## Solution Assessment

No, not as-is. Skipping standard Anthropic API keys for usage status is the right bug boundary, but an empty credential sentinel is not the safest plugin-contract shape.
