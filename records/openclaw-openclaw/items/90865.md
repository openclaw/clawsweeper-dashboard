## Summary

Keep open: current main has the intended Discord config-hot-reload-to-channel-restart path and a newer watcher polling fallback, but there is still no proof that the reported Docker/v2026.6.1 path now picks up newly added Discord guild channels without restart. Discord reconnect alone still uses the monitor's startup allowlist snapshot, so this needs focused live reload proof before cleanup closure.

## What This Changes

The issue requests that newly added `channels.discord.guilds.*.channels.*` entries become live for Discord inbound messages without a full gateway restart.

## Best Possible Solution

Prove the `openclaw.json` edit path end to end for a Discord guild channel addition, then repair the existing gateway reload-to-Discord-channel-restart path if it fails.

## Reproduction Assessment

No high-confidence current-main live reproduction was established. Source shows a Discord monitor that is not restarted keeps its startup `guildEntries` snapshot and drops newly added allowlisted channels, but source also shows config saves should restart Discord through hot reload.

## Solution Assessment

Unclear. The maintainable first fix is to prove and repair the existing config reload/channel restart path, not to add reconnect-time disk reads or a separate Discord reload command before the failure mode is known.
