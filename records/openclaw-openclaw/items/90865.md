Keep open: current main and the latest release both have the generic `channels.discord` hot-reload restart path, but source still shows Discord inbound handling depends on a provider-startup guild/channel snapshot, and no focused proof shows that adding `channels.discord.guilds.*.channels.*` reaches and rebuilds that snapshot in the reported scenario.

The issue asks OpenClaw to make newly added configured Discord channels receive inbound events without requiring a full gateway restart.
