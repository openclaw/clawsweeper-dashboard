Keep open: the report remains plausible and in-scope, but current main does not provide close-quality proof that the Discord post-startup channel subscription problem is fixed. ## What This Changes

The issue reports that adding a Discord guild channel entry to `openclaw.json` after gateway startup does not make inbound Discord events from that channel live until a full gateway restart.

## Best Possible Solution

Keep this issue open and first capture a focused repro showing whether a `channels.discord.guilds.*.channels.*` edit emits a config reload, plans `restart-channel:discord`, stops/starts Discord, and refreshes the inbound guild/channel snapshot; then repair the narrow failing layer without adding a separate reload command unless maintainers choose that product surface.

## Reproduction Assessment

No high-confidence current-main reproduction was established. Source inspection shows the stale-snapshot failure mode is plausible, but current main also has a generic Discord config hot-reload restart path that should refresh that snapshot if it fires correctly.

## Solution Assessment

Unclear. The maintainable fix is probably in the existing gateway reload or Discord channel restart path; adding a reconnect-time disk reread or `discord reload` admin command would be a broader alternative only after the intended reload path is proven insufficient.
