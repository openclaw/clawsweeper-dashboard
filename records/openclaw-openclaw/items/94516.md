## What This Changes

The issue reports that the Copilot plugin fails registration on OpenClaw 2026.6.8 because `api.runtime.state` is undefined when the plugin calls `openSyncKeyedStore`.

## Best Possible Solution

Land one canonical fix that skips runtime-only Copilot registration outside `full` mode, preserves durable session storage in full runtime, and includes reporter-equivalent `openclaw plugins list` or `openclaw status` proof.

## Reproduction Assessment

Yes from source inspection: current main still calls `api.runtime.state.openSyncKeyedStore` directly while the CLI metadata loader passes `runtime: {}`, matching the reporter's failure when `api.runtime.state` is undefined. I did not run the installed reporter-equivalent CLI path in this read-only review.

## Solution Assessment

No, the issue is not solved on main yet. The best repair should keep metadata loading non-runtime and preserve durable Copilot session storage when full runtime state is available.
