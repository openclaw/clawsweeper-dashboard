## What This Changes

The issue reports `@openclaw/copilot` 2026.6.8 failing registration because `api.runtime.state` is undefined while the plugin calls `openSyncKeyedStore` during startup.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the canonical path: a plugin-local fallback is plausible because the harness treats persistence as optional, but fixing host state injection may better preserve the current plugin runtime contract.
