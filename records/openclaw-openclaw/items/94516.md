## Summary

Keep open: the reported Copilot registration failure is source-plausible, and an open draft PR already references this issue with closing syntax, so the issue should wait for that PR or a replacement fix to land.

## What This Changes

The issue reports that the Copilot plugin fails to register on OpenClaw 2026.6.8 because `api.runtime.state` is undefined when the plugin calls `openSyncKeyedStore`.

## Best Possible Solution

Keep this issue open until the linked PR is reviewed and landed, or replaced by a narrower fix that ensures Copilot registration never sees an incompatible runtime API shape.

## Reproduction Assessment

Yes from source, but not by executing the command: Copilot registration dereferences `api.runtime.state.openSyncKeyedStore`, and the loader has registration surfaces that can pass an empty runtime object. The exact installed 2026.6.8 environment still needs live confirmation.

## Solution Assessment

Unclear: preserving durable plugin state through the normal runtime proxy is the right default, but maintainers should decide whether the missing-state compatibility belongs in the Copilot entrypoint or in the loader/runtime API contract.
