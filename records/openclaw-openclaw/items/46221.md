## Summary

Keep open: current main still has the per-channel-peer system-event delivery gap, and this PR is useful but not merge-ready because the fallback drains the configured default main key, applies to every non-main session, and lacks inspectable real behavior proof.

## What This Changes

The PR branch changes formatted system-event draining to also merge the canonical main-session queue into non-main session drains and adds a per-channel-peer regression test.

## Best Possible Solution

Port the receiver-side fallback onto current main using the active session's agent id and selected generic-event consumption, then add focused tests and redacted Telegram proof for the approved DM/non-main scope.

## Reproduction Assessment

Yes, at source level: main enqueue paths still queue system events under a main key, per-channel-peer DMs use agent:<id>:<channel>:direct:<peer>, and current main drains formatted events only from params.sessionKey. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No. The receiver-side fallback is plausible, but this patch should derive the fallback key from the active session's agent, preserve selected queue consumption, and either restrict or explicitly document/test the broader non-main-session behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope fallback drain to the active agent:** `src/auto-reply/reply/session-system-events.ts:82`
  - body: resolveMainSessionKey(params.cfg) returns the configured default agent's main key, not the active session's agent. A session such as agent:ops:telegram:direct:... can drain agent:main:main while still missing agent:ops:main, so derive the fallback from params.sessionKey instead.
  - confidence: 0.91
- **[P2] Preserve selective system-event consumption:** `src/auto-reply/reply/session-system-events.ts:87`
  - body: Current main keeps exec-completion entries queued for the heartbeat path by consuming only selected generic events. The fallback drains the entire main queue, so a normal reply turn can steal heartbeat-owned entries; use the same selected-consumption contract for the fallback key.
  - confidence: 0.84
- **[P2] Constrain the main-key merge scope:** `src/auto-reply/reply/session-system-events.ts:83-87`
  - body: The mainSessionKey !== params.sessionKey check also matches group, channel, thread, and custom sessions. That can consume and clear a main-session wake event from an unrelated conversation; restrict this fallback to approved scopes or add maintainer-approved coverage for the broader behavior.
  - confidence: 0.78
