## Summary

Keep open. Current main still has the source-level delivery gap this PR targets, but the patch is stale/conflicting and has session-boundary defects that need contributor proof plus maintainer scope judgment before merge.

## What This Changes

The PR changes formatted system-event draining to merge the canonical main-session queue into non-main session drains and adds a per-channel-peer DM regression test.

## Best Possible Solution

Port a receiver-side fallback onto current main that derives the fallback main key from the active session's agent id, preserves selected-event consumption, and either restricts scope to approved DM isolation or documents and tests broader non-main delivery explicitly.

## Reproduction Assessment

Yes at source level: current main queues main-session wake/system events while per-channel-peer DMs resolve to a different key, and formatted draining consumes only the active key. I did not run live Telegram or Discord in this read-only review.

## Solution Assessment

No, not as currently proposed. Receiver-side fallback is plausible, but it should use the active agent's main key, preserve selected-event consumption, and get maintainer approval for any non-DM scope expansion.

- **[P1] Scope fallback drain to the active agent:** `src/auto-reply/reply/session-system-events.ts:82`

  - confidence: 0.84
