## Summary

Keep open. Current main still matches the reported browser tab-selection race: the cold-start `openTab("about:blank")` result is discarded, local-managed profiles still require `wsUrl` candidates, and a second `/json/list` response without `webSocketDebuggerUrl` can still lead to `BrowserTabNotFoundError`. The linked fix PR is closed unmerged and no open replacement PR was found.

## What This Changes

Request to harden browser tab selection so `ensureTabAvailable()` does not throw when local-managed CDP discovers a newly opened tab before its per-tab `webSocketDebuggerUrl` is populated.

## Best Possible Solution

Land a narrow browser plugin fix that captures the bootstrap `openTab()` result, polls briefly when `supportsPerTabWs` filtering removes otherwise visible tabs, tolerates transient `listTabs()` failures during that window, and falls back to the latest usable target-id tab when `wsUrl` never appears. Keep the separate Playwright AbortSignal orphan-operation note out of this repair.

## Reproduction Assessment

Yes. A deterministic reproduction is to have the first `listTabs()` return `[]`, have `openTab("about:blank")` return a created target, then have the next `listTabs()` return that target without `wsUrl`; current main filters the tab out and throws `BrowserTabNotFoundError`.

## Solution Assessment

Yes. The best fix is a focused change in `extensions/browser/src/browser/server-context.selection.ts` with colocated regression tests; moving this into core or broadening it to Playwright AbortSignal cleanup would make the repair larger than the observed tab-selection race.

Fix openclaw/openclaw#63343 with a narrow browser plugin PR. Current `ensureTabAvailable()` in `extensions/browser/src/browser/server-context.selection.ts` still discards the cold-start `openTab("about:blank")` result, immediately relists tabs, and filters local-managed candidates to entries with `wsUrl`, so a newly created CDP target can be filtered out before `webSocketDebuggerUrl` appears. Add focused regression tests in `extensions/browser/src/browser/server-context.tab-selection-state.test.ts` for missing wsUrl recovery, fallback after polling exhausts, openTab result use when relisting stays empty, and transient polling failure tolerance.
