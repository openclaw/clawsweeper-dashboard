## Summary

Keep open: current main still lacks the VS Code debugging surface, and the proposed one-click clean debug flow can remove generated runtime assets that `run-node` does not recreate.

## What This Changes

The PR adds VS Code launch/tasks configs, an opt-in `OUTPUT_SOURCE_MAPS` sourcemap toggle, a `clean:dist` script, and VS Code debugging docs.

## Best Possible Solution

Land the DX improvement only after the checked-in launch path preserves or recreates generated runtime assets, with docs that distinguish one-click debugging from manual source-map builds.

## Reproduction Assessment

Yes for the introduced regression by source inspection: start from a clean `dist`, use the proposed `Rebuild and Debug Gateway` path, then exercise `/export-session`; the task removes `dist` and the follow-up runner does not copy the export templates.

## Solution Assessment

No. The sourcemap toggle and docs are reasonable, but deleting all of `dist/` before a narrower `run-node` rebuild is not the safest maintainable one-click debug path.
