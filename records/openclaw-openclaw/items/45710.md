## Summary

Keep this PR open. Current main still does not provide the proposed VS Code debugging path, but the PR's one-click clean launch flow can delete generated runtime assets that the narrower `run-node` rebuild does not recreate.

## What This Changes

The PR adds VS Code launch/tasks configs, an opt-in tsdown source-map env toggle, a `clean:dist` package script, and VS Code debugging docs.

## Best Possible Solution

Land the DX improvement only after the VS Code launch flow enables source-map debugging without deleting generated runtime assets, with docs separating one-click and manual workflows.

## Reproduction Assessment

Yes for the introduced regression by source inspection: the PR's clean prelaunch task removes `dist/`, then `run-node` rebuilds only tsdown plus runtime-postbuild artifacts, leaving export HTML and A2UI assets unrecreated.

## Solution Assessment

No. The source-map toggle is a reasonable building block, but the current clean prelaunch workflow is not the safest maintainable fix because it removes assets outside the runner's rebuild contract.
