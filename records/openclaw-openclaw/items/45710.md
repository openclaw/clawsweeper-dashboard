## Summary

Keep this PR open. The DX goal is still unimplemented on main, but the proposed clean debug launch can remove generated runtime assets that the narrower `run-node` rebuild does not recreate, so the one-click path needs one blocking repair before merge.

## What This Changes

The PR adds VS Code launch/tasks configs, an `OUTPUT_SOURCE_MAPS`-gated tsdown sourcemap toggle, a `clean:dist` script, and VS Code gateway debugging docs.

## Best Possible Solution

Land the DX improvement after the checked-in debug launch uses a supported source-map build path that leaves generated runtime assets available, with docs matching the chosen one-click and manual workflows.

## Reproduction Assessment

Yes, by source inspection for the PR-introduced failure: start from a clean `dist`, launch `Rebuild and Debug Gateway`, then use `/export-session`. The launch deletes `dist`, while the follow-up `run-node` rebuild does not copy `dist/export-html`.

## Solution Assessment

No. The sourcemap toggle and docs are reasonable, but the current clean-plus-`run-node` launch is not the safest one-click rebuild path because it drops generated runtime assets outside `runtime-postbuild`.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Preserve generated assets in clean debug launches:** `.vscode/launch.json:8`
