## Summary

Keep this PR open. Current main still lacks the proposed VS Code launch/tasks/source-map/docs path, but the patch is not ready because the one-click rebuild flow deletes `dist/` and then uses a narrower `run-node` rebuild that does not recreate all runtime assets.

## What This Changes

The PR adds VS Code Gateway debug configurations, a `clean:dist` task/script, an opt-in `OUTPUT_SOURCE_MAPS` tsdown toggle, and VS Code debugging documentation.

## Best Possible Solution

Land the DX improvement after the debug launch path preserves or recreates full-build runtime assets and the docs accurately describe that supported path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the introduced regression: start from a clean `dist`, use the PR's `Rebuild and Debug Gateway` flow, then run `/export-session`; source inspection shows the required `dist/export-html` templates are not recreated.

## Solution Assessment

Is this the best way to solve the issue? No. The opt-in source-map toggle is reasonable, but deleting all of `dist/` before a narrower `run-node` rebuild is not the safest maintainable one-click debug path.
