## Summary

Keep this PR open. Current main still lacks the proposed VS Code debugging support, but the PR's one-click clean rebuild path can remove generated runtime assets that the narrower `run-node` rebuild does not recreate, so the patch should not merge as-is.

## What This Changes

The branch adds VS Code launch/tasks configs, a `clean:dist` script, an opt-in `OUTPUT_SOURCE_MAPS` tsdown sourcemap toggle, and a VS Code section in the debugging guide.

## Best Possible Solution

Land the DX improvement after the one-click launch path uses a supported source-map build flow that preserves or recreates required generated assets, with docs that clearly distinguish manual `OUTPUT_SOURCE_MAPS=1 pnpm build` usage from the automated launch path.

## Reproduction Assessment

Yes for the PR-introduced regression: start the proposed `Rebuild and Debug Gateway` flow after `clean:dist`, then exercise `/export-session` or `/__openclaw__/a2ui`. Source inspection shows the follow-up `run-node` path rebuilds TypeScript/runtime-postbuild artifacts but not the export HTML or A2UI copy steps.

## Solution Assessment

No. The opt-in sourcemap toggle is a reasonable direction, but deleting all of `dist/` before a narrower `run-node` rebuild is not the safest one-click debug solution; a full source-map debug build, asset-preserving clean, or dedicated debug-build task would be more maintainable.
