## Summary

Keep this PR open. Current main still lacks the proposed VS Code debugging path, but the PR should not merge as-is because the clean one-click launch removes `dist/export-html` and then relies on a narrower `run-node` rebuild path that does not recreate those runtime templates.

## What This Changes

The PR adds VS Code launch/tasks files, an opt-in tsdown source-map toggle, a `clean:dist` script, and VS Code debugging documentation.

## Best Possible Solution

Land the DX improvement only after the automated launch path preserves or recreates required full-build runtime assets, while keeping source maps opt-in and dependency-free.

## Reproduction Assessment

Yes for the blocking review finding: start from a clean `dist`, use the PR's `Rebuild and Debug Gateway` flow, then use `/export-session`; source inspection shows the templates were deleted and are not recreated by `run-node`. I did not run a live VS Code session in this read-only review.

## Solution Assessment

No. The source-map toggle and docs are reasonable, but deleting all of `dist/` before a `run-node` rebuild is not the narrowest maintainable debug path because that runner does not perform the full asset-copy pipeline.
