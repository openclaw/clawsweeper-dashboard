## Summary

Keep open. Current main still lacks OpenAI `/models` catalog discovery, and this PR is a plausible implementation slice tied to unresolved provider-catalog design in #74481, with small cleanup findings before maintainer review.

## What This Changes

The PR adds an OpenAI discovery helper and tests that fetch an OpenAI-compatible `/models` endpoint, map results into `ModelDefinitionConfig[]`, cache them, and export the helper from the OpenAI plugin API barrel.

## Best Possible Solution

Settle #74481's provider-catalog contract first, then either merge this helper after cache/changelog cleanup or fold it into the eventual OpenAI hook with timeout, network-safety, docs, and UI/model-list follow-through.

## Reproduction Assessment

Not applicable for the feature request itself; this is an implementation PR, not a bug report. The cache finding is reproducible by calling the helper twice within the TTL using equivalent base URLs that differ only by trailing slash and observing two fetches.

## Solution Assessment

Unclear as the final product solution. The helper is a reasonable primitive, but the narrowest maintainable fix depends on the provider-catalog API decisions still open in #74481.
