Full review comments:

- **[P1] Reuse the guarded live catalog runtime:** `extensions/openai/discovery.ts:181-187`
  - body: This new helper builds a URL from caller-controlled `baseUrl` and sends the provider bearer token with raw `fetchFn`. Current main already has `provider-catalog-live-runtime` for live model catalogs, including SSRF guarding, timeout, auth-scoped cache keys, and fallback behavior; landing a second unguarded path would bypass that security boundary for custom OpenAI-compatible endpoints.
  - confidence: 0.9
- **[P2] Scope the cache by the full auth identity:** `extensions/openai/discovery.ts:76-80`
  - body: `cacheKey()` keeps only the last eight characters of the API key, so two keys for the same base URL with the same suffix share a discovered catalog. For tenant-scoped gateways that can silently show the wrong model list; use the shared hashed cache key parts or hash the full auth identity instead of truncating it.
  - confidence: 0.86
- **[P3] Keep test resets out of the public barrel:** `extensions/openai/api.ts:12`
  - body: `resetOpenAIDiscoveryCacheForTest` is marked test-only but is exported from the OpenAI plugin public `api.ts` barrel. If this helper remains, keep the reset local to tests or a dedicated test-only path so it does not become a public plugin contract by accident.
  - confidence: 0.78
