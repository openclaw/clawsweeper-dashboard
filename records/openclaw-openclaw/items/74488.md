## Summary

Keep open. Current main still lacks OpenAI upstream `/models` catalog discovery, and this PR is a plausible primitive, but it needs the #74481 provider-catalog contract decision plus small cache/changelog cleanup before merge.

## What This Changes

Adds an exported OpenAI discovery helper and tests that fetch an OpenAI-compatible `/models` endpoint, map metadata into `ModelDefinitionConfig[]`, cache results, and fail soft.

## Best Possible Solution

Settle the provider-catalog seam in #74481, then either merge this helper after cleanup or fold it into the final OpenAI catalog hook with timeout, network-safety, docs, changelog, and UI/listing follow-through.

## Reproduction Assessment

Not applicable for the feature request itself; this is an implementation PR, not a bug report. The cache finding is reproducible by calling the helper twice within the TTL with equivalent base URLs that differ only by trailing slash and observing two fetches.

## Solution Assessment

Unclear as the final product solution. The helper is a reasonable first slice, but the narrowest maintainable path depends on the still-open #74481 provider-catalog API and should add network-safety before runtime use.
