## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use the guarded live catalog fetch path:** `extensions/openai/discovery.ts:181-187`
  - body: `baseUrl` is intended to come from a configured OpenAI-compatible upstream, but this helper defaults to raw `fetch` and sends `Authorization: Bearer` to `${baseUrl}/models` without the existing SSRF guard, timeout, or guarded-fetch release lifecycle. If this is wired into catalog resolution, a malicious or mistyped provider URL can make OpenClaw send credentials to an unsafe network target; build this on `provider-catalog-live-runtime` instead.
  - confidence: 0.9
- **[P2] Keep the cache reset out of the public API:** `extensions/openai/api.ts:12`
  - body: `api.ts` is the OpenAI plugin public barrel, so exporting `resetOpenAIDiscoveryCacheForTest` turns a test-only mutable cache hook into a supported plugin API. Keep tests importing that helper from the private module or a dedicated test-support path so downstream plugins do not depend on it.
  - confidence: 0.86
