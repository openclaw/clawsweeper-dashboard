## What This Changes

The branch exports a new OpenAI `discoverOpenAIModels()` helper via the OpenAI plugin API, adds `/models` response mapping and cache logic, and adds unit tests for that helper.

## Best Possible Solution

Settle the provider-owned dynamic catalog seam, then land guarded integrated OpenAI discovery with deterministic merge behavior, cache invalidation, UI refresh wiring, tests, docs, and real endpoint proof.

## Reproduction Assessment

Not applicable for the feature request itself; this is not broken documented behavior. Source inspection gives high confidence that current main does not fetch configured OpenAI-compatible `/models` endpoints for the picker.

## Solution Assessment

Unclear as the final product solution. A discovery primitive is useful, but the maintainable fix should be tied to the chosen provider-catalog seam and use the existing guarded fetch/timeout patterns before it is exported or wired into UI refresh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard the default discovery fetch:** `extensions/openai/discovery.ts:181-187`
  - body: This exported helper defaults to raw `fetch` against caller-supplied `baseUrl` while sending `Authorization: Bearer ...`. Before it becomes a catalog/UI discovery primitive, use the existing SSRF/timeout guard or require callers to inject a guarded fetch so private/internal hosts and hung endpoints are not reached from the gateway process.
  - confidence: 0.82
- **[P3] Normalize the base URL before cache lookup:** `extensions/openai/discovery.ts:170-177`
  - body: The cache key is built from raw `baseUrl` before trailing slashes are stripped for the endpoint, so `https://x/v1` and `https://x/v1/` hit the same `/models` URL but bypass each other's TTL cache and last-known-good fallback. Normalize once and use that value for both the key and endpoint.
  - confidence: 0.9
- **[P3] Fix the extension lint failures:** `extensions/openai/discovery.ts:95-104`
  - body: The live PR checks show lint-related failures on this branch, and the added discovery code/test style is the changed surface. Update the reported formatting/type-assertion issues so the bundled extension lint lane can pass.
  - confidence: 0.78
