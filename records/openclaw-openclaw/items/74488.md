Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route discovery through the guarded live-catalog runtime:** `extensions/openai/discovery.ts:168-185`
  - body: This helper defaults to raw `fetch` and sends `Authorization: Bearer ${apiKey}` to an endpoint derived from `baseUrl`. Current main has `provider-catalog-live-runtime` for guarded `/models` discovery with SSRF policy, auth handling, TTL cache, and fallback, so this separate credentialed fetch path bypasses the maintained security boundary.
  - confidence: 0.91
- **[P1] Key the cache by the full auth scope:** `extensions/openai/discovery.ts:80`
  - body: Caching by only `apiKey.slice(-8)` lets different credentials for the same base URL share a cached model list when their suffix matches. Multi-tenant gateways can return credential-specific catalogs, so this needs the shared auth-scoped key or another collision-resistant full auth-scope key.
  - confidence: 0.88
