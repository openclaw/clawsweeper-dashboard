## What This Changes

This PR adds an OpenAI plugin discovery helper and tests to fetch an OpenAI-compatible `/models` endpoint, map entries to `ModelDefinitionConfig[]`, cache results, and export the helper from the OpenAI API barrel.

## Best Possible Solution

Settle the provider-owned catalog seam, then land guarded integrated OpenAI discovery with normalized cache keys, deterministic merge semantics, Gateway/Control UI refresh wiring, tests, docs, and real endpoint proof.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard credentialed discovery fetch:** `extensions/openai/discovery.ts:181-187`
  - body: This exported helper defaults to raw `fetch` against caller-supplied `baseUrl` while sending `Authorization: Bearer ...`. Before it becomes the catalog/UI discovery primitive, use the existing SSRF/timeout guard or require callers to inject a guarded fetch so internal hosts and hung endpoints are not reached from the gateway process.
  - confidence: 0.82
- **[P3] Normalize the base URL before cache lookup:** `extensions/openai/discovery.ts:171-178`
  - body: The cache key is built from raw `baseUrl` before trailing slashes are stripped for the endpoint, so `https://x/v1` and `https://x/v1/` hit the same `/models` URL but bypass each other's TTL cache and last-known-good fallback. Normalize once and use that value for both the key and endpoint.
  - confidence: 0.9
- **[P3] Fix the extension lint violations:** `extensions/openai/discovery.ts:95-104`
  - body: The PR's extension lint jobs fail on the new discovery files, including one-line `if` bodies here and unnecessary `RequestInit` assertions in the test. Update the new files so the bundled extension lint lane can pass.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces an exported credentialed network helper whose default path uses raw `fetch` for caller-supplied URLs.

Concerns:

- **[medium] Unguarded credentialed model discovery fetch:** `extensions/openai/discovery.ts:181`
