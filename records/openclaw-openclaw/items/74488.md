## Summary

Keep open. The branch is a plausible first slice, but it is not mergeable yet: current main still lacks integrated dynamic OpenAI catalog discovery, while this PR has missing real behavior proof, an unguarded credentialed fetch path, failing checks, and unresolved product/API questions in https://github.com/openclaw/openclaw/issues/74481.

## What This Changes

The PR exports a new OpenAI discovery helper and tests that fetch an OpenAI-compatible `/models` endpoint, map responses to `ModelDefinitionConfig[]`, and cache results.

## Best Possible Solution

Settle the provider-owned dynamic catalog seam, then land guarded integrated OpenAI discovery with normalized cache keys, deterministic merge semantics, UI refresh wiring, tests, docs, and real endpoint proof.

## Reproduction Assessment

Not applicable for the feature request itself; this is not broken documented behavior. Source inspection gives high confidence that current main does not fetch configured OpenAI-compatible `/models` endpoints for the picker, and the review findings are visible from the PR diff and check state.

## Solution Assessment

Unclear as the final product solution. A discovery primitive is useful, but the maintainable fix should be tied to the chosen provider-catalog seam and use the existing guarded fetch/timeout patterns before it is exported or wired into UI refresh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard the default discovery fetch:** `extensions/openai/discovery.ts:181-187`
  - body: This exported helper defaults to raw `fetch` against caller-supplied `baseUrl` while sending `Authorization: Bearer ...`. Before this becomes the catalog/UI discovery primitive, make the default path use the existing SSRF/timeout guard or require callers to inject a guarded fetch so private/internal hosts and hung endpoints are not reached from the gateway process.
  - confidence: 0.82
- **[P3] Normalize the base URL before cache lookup:** `extensions/openai/discovery.ts:170-177`
  - body: The cache key is built from raw `baseUrl` before trailing slashes are stripped for the endpoint, so `https://x/v1` and `https://x/v1/` hit the same `/models` URL but bypass each other's TTL cache and last-known-good fallback. Normalize once and use that value for both the key and endpoint.
  - confidence: 0.9
- **[P3] Fix the extension lint failures:** `extensions/openai/discovery.ts:95-104`
  - body: The PR's extension lint job is failing on the new discovery files, including one-line `if` bodies and unnecessary `RequestInit` assertions in the added tests. Update the reported locations so the bundled extension lint lane can pass.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces an exported credentialed network helper that defaults to raw `fetch` for caller-supplied URLs.

Concerns:

- **[medium] Unguarded credentialed model discovery fetch:** `extensions/openai/discovery.ts:181`
