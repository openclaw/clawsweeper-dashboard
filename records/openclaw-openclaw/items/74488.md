## Summary

Keep open. Current main still lacks OpenAI `/models` catalog discovery, and this PR is a plausible primitive, but it needs maintainer agreement on #74481's provider-catalog contract plus cache/API/changelog cleanup before merge.

## What This Changes

Adds an OpenAI discovery helper and tests that fetch an OpenAI-compatible `/models` endpoint, map it into `ModelDefinitionConfig[]`, cache per base URL/key suffix, and export the helper from the OpenAI plugin API barrel.

## Best Possible Solution

Settle #74481's catalog contract, then merge a cleaned-up OpenAI discovery implementation through that chosen path with cache, timeout/network-safety, docs, UI/model-list, and changelog follow-through.

## Reproduction Assessment

Not applicable for the feature itself; this is an implementation PR, not a bug report. The cache finding is reproducible by calling the helper twice within the TTL with equivalent base URLs that differ only by trailing slash and observing two fetches.

## Solution Assessment

Unclear as the final product solution. The helper is a reasonable primitive, but the maintainable fix depends on the provider-catalog API and refresh semantics being decided in #74481.
