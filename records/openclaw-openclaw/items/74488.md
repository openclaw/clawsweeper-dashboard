## Summary

Keep open. Current main still lacks OpenAI-compatible dynamic catalog discovery, and this PR is a first-slice helper with unresolved provider-catalog/API decisions, mock-only proof, and concrete review issues around guarded credentialed fetches, cache normalization, and failing extension lint.

## What This Changes

The PR adds an exported OpenAI discovery helper and tests that fetch an OpenAI-compatible `/models` endpoint, map responses to `ModelDefinitionConfig[]`, and cache results.

## Best Possible Solution

Settle the provider-catalog contract in the linked design issue, then land an integrated OpenAI discovery path with guarded fetches, normalized cache keys, deterministic merge semantics, UI refresh follow-through, tests, docs, and real endpoint proof.

## Reproduction Assessment

Not applicable for the feature request itself; this is not a broken documented behavior. The review findings are source-reproducible from the PR diff and CI logs: cache lookup uses unnormalized input, default fetch is unguarded, and extension lint is failing.

## Solution Assessment

Unclear as the final product solution. A discovery primitive is useful, but the maintainable fix should be tied to the chosen provider-catalog seam and use the existing guarded fetch/timeout patterns before it is exported or wired into UI refresh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard the default discovery fetch:** `extensions/openai/discovery.ts:181-187`
  - body: This exported helper defaults to raw `fetch` against caller-supplied `baseUrl` while sending `Authorization: Bearer ...`. Before this becomes the catalog/UI discovery primitive, make the default path bounded with the existing SSRF/timeout guard or require callers to inject a guarded fetch so private/internal hosts and hung endpoints are not reached from the gateway process.
  - confidence: 0.82
- **[P3] Normalize the base URL before cache lookup:** `extensions/openai/discovery.ts:171-178`
  - body: The cache key is built from raw `baseUrl` before trailing slashes are stripped for the endpoint, so `https://x/v1` and `https://x/v1/` hit the same `/models` URL but bypass each other's TTL cache and last-known-good fallback. Normalize once and use that value for both the key and endpoint.
  - confidence: 0.9
- **[P3] Fix the extension lint failures:** `extensions/openai/discovery.ts:95-104`
  - body: The PR's extension lint job is failing on the new discovery files, including one-line `if` bodies and unnecessary `RequestInit` assertions. Please update all reported locations so `pnpm lint:extensions:bundled` can pass.
  - confidence: 0.83
