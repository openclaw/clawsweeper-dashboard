## Summary

Keep open. Current main still lacks OpenAI `/models` catalog discovery, and this PR is a plausible first slice, but it remains tied to the open #74481 provider-catalog design decisions and has two P3 cleanup items before merge.

## What This Changes

The PR adds a standalone OpenAI discovery helper and tests that fetch an OpenAI-compatible `/models` endpoint, map results to `ModelDefinitionConfig[]`, cache responses, and export the helper from the OpenAI plugin API barrel.

## Best Possible Solution

Settle #74481's provider-catalog contract, then either merge this helper after cache/changelog cleanup or fold it into the eventual OpenAI catalog hook with network bounds and UI/list follow-through.

## Reproduction Assessment

Not applicable for the feature request itself; this is an implementation PR, not a bug report. The cache finding is source-reproducible by calling the helper twice within the TTL using equivalent base URLs that differ only by trailing slash and observing two fetches.

## Solution Assessment

Unclear as the final product solution. The helper is a reasonable primitive, but the narrowest maintainable path depends on the provider-catalog API decisions still open in #74481.
