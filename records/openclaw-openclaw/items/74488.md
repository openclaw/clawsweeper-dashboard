## Summary

Keep open. Current main still does not discover configured OpenAI-compatible model catalogs, and this partial helper PR needs real behavior proof, guarded fetch/cache fixes, and maintainer provider-catalog review before merge.

## What This Changes

Adds an exported OpenAI discovery helper and tests that fetch an OpenAI-compatible `/models` endpoint, map responses to `ModelDefinitionConfig[]`, and cache results.

## Best Possible Solution

Choose the provider-owned catalog refresh seam, then land integrated OpenAI-compatible discovery with guarded network access, deterministic merge behavior, UI refresh wiring, tests, docs, and real endpoint proof.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than broken documented behavior. Source inspection confirms current main only uses static or explicit configured catalog rows for the OpenAI picker and does not fetch configured `/models` catalogs.

## Solution Assessment

No, not as a merge-ready final solution. The discovery primitive is useful, but it should be tied to the chosen provider-catalog seam and use guarded fetch/timeout/cache semantics before being exported or wired into picker refresh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard the default discovery fetch:** `extensions/openai/discovery.ts:181-187`
  - body: This helper defaults to raw `fetch` against caller-supplied `baseUrl` while sending `Authorization: Bearer ...`. Before it becomes a catalog/UI discovery primitive, use the existing SSRF/timeout guard or require callers to inject a guarded fetch so the gateway does not reach private/internal hosts or hang on discovery.
  - confidence: 0.84
- **[P3] Normalize the base URL before cache lookup:** `extensions/openai/discovery.ts:171-178`
  - body: The cache key is built from raw `baseUrl` before trailing slashes are stripped for the endpoint, so `https://x/v1` and `https://x/v1/` hit the same `/models` URL but bypass each other's TTL cache and last-known-good fallback. Normalize once and use that value for both the key and endpoint.
  - confidence: 0.9

  - body: The PR's lint/check jobs are failing on style in the added discovery files, including one-line conditionals in this helper and type assertions in the new tests. Clean those up so the bundled extension gates can pass.
