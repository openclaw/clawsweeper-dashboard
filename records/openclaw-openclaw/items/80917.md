## Summary

Keep open. The PR now includes a runtime fix, but the current guard is too broad: it turns any fallback-produced silent final reply into a visible error, which can break intentional group silence, and the external PR still lacks real after-fix behavior proof.

## What This Changes

The PR adds a `runReplyAgent` runtime fallback error payload plus a mocked e2e regression for a model fallback that returns `NO_REPLY`.

## Best Possible Solution

Carry a typed operational-failure signal from the fallback/run-result boundary into final delivery, while preserving intentional `NO_REPLY` suppression and adding real channel proof.

## Reproduction Assessment

Yes for a source-level path: current main can normalize a fallback `NO_REPLY` final payload to an empty reply and return `undefined`. I did not run the test because this review is read-only and local tests can write artifacts.

## Solution Assessment

No. The PR targets the right area, but the current condition is not the narrowest maintainable fix because it cannot distinguish operational no-visible-reply failure from an intentional silent fallback response.
