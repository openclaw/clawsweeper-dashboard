## Summary

## What This Changes

The PR rejects Z.AI onboarding command text during API-key setup, surfaces polluted Z.AI static profile hints through doctor, adds `issueKind` to the provider auth doctor context, and adds Z.AI/Anthropic/OpenAI regression tests.

## Best Possible Solution

Keep the provider-owned Z.AI input rejection, but make static API-key doctor hints opt in through a static-specific plugin contract or declared capability before merge.
