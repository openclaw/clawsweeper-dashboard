## What This Changes

The PR scopes all-scope `cron.runs` job-name enrichment through paginated agent-scoped cron job listing, adds an optional `agentId` protocol field, updates generated Swift models/docs/changelog, and adds regression tests.

## Best Possible Solution

Land this or an equivalent narrow fix after redacted real-behavior proof and maintainer security confirmation, while leaving broader per-agent run-log isolation to https://github.com/openclaw/openclaw/issues/26370.

## Reproduction Assessment

Yes, source-reproducible: current main builds all-scope `cron.runs` job names from the unfiltered cron list, and the run-log reader exposes that map in search text and returned `jobName` fields. I did not run a live gateway repro because this review is read-only and the repository checkout must remain clean.

## Solution Assessment

Mostly yes: reusing the existing `listPage()` agent filter is the narrow maintainable fix for the named job-name leak, and the latest branch also documents the new protocol field. The remaining merge gate is real behavior proof and maintainer acceptance of the narrower compatibility/security boundary.
