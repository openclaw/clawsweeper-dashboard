## Best Possible Solution

## Solution Assessment

Mostly yes: reusing `listPage()` is the narrow maintainable fix for the job-name leak because that path already owns agent filtering. The remaining merge blockers are real behavior proof and maintainer acceptance of the narrower compatibility/security boundary.
