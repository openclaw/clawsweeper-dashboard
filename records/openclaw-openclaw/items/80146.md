## Summary

Keep open: the code change appears to address the cron run-name leak, but the external PR still has only mocked/source-handler proof and needs maintainer acceptance of the narrower all-scope run-history boundary before merge.

## What This Changes

The PR scopes all-scope `cron.runs` job-name enrichment through paginated agent-scoped cron listing, adds an optional `agentId` protocol field, updates generated Swift/docs/changelog, and adds regression tests.

## Best Possible Solution

Land this or an equivalent narrow fix after redacted real-behavior proof and maintainer security/compatibility confirmation, while leaving broader cron ownership isolation to https://github.com/openclaw/openclaw/issues/26370.

## Reproduction Assessment

Yes. Source inspection shows current main builds `cron.runs(scope: "all")` job names from an unfiltered cron list, and the all-run reader uses that map for search text and returned `jobName` fields.

## Solution Assessment

Mostly yes. Reusing the existing scoped `listPage()` contract is the narrow maintainable fix for the reported name leak, but maintainers still need to accept the partial all-scope boundary and compatibility behavior.
