## Summary

Keep open: current main still has the unscoped all-scope cron run-name enrichment, and this focused PR targets the right leak but still needs real behavior proof and maintainer/security review before merge.

## What This Changes

This PR changes `cron.runs(scope: "all")` job-name enrichment to page through agent-scoped cron list results, adds `agentId` to the cron runs protocol, refreshes the Swift model, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land this or an equivalent narrow fix after redacted real behavior proof, a small protocol-doc update, and maintainer/security confirmation of the remaining all-scope run-log boundary.

## Reproduction Assessment

Yes, as source-reproducible rather than live-reproduced: current main builds all-scope `cron.runs` job names from the unfiltered cron list and the run-log reader exposes that map in search text and returned `jobName` fields.

## Solution Assessment

Mostly yes. Reusing `listPage()` for enrichment is the narrow maintainable fix for the named leak, but the PR should document the new protocol field and wait for real behavior proof plus maintainer/security review.
