## Summary

## What This Changes

The PR changes `cron.runs(scope: "all")` job-name enrichment to use agent-scoped cron list pages, adds an optional `agentId` runs parameter, refreshes generated Swift protocol models, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this or an equivalent narrow fix after redacted real behavior proof, a protocol-doc update, and maintainer/security confirmation of the remaining all-scope run-log boundary.

## Reproduction Assessment

Yes, source-reproducible: current main builds `cron.runs(scope: "all")` job names from the unfiltered cron list and the run-log reader exposes that map in search text and returned `jobName` fields.

## Solution Assessment

Mostly yes. Reusing `listPage()` for enrichment is the narrow maintainable fix for the named leak, but the public protocol docs and contributor real behavior proof still need follow-through.
