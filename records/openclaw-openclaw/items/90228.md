## Summary

Keep open: current main still has the unguarded Anthropic direct/custom tool schema read, and this draft maintainer PR is a focused bug fix, but the proposed snapshot is shallow and can still let deeper schema getters fail during request JSON serialization.

## What This Changes

The PR adds a guarded Anthropic tool snapshot helper and a regression test so direct/custom tools with unreadable schema descriptors are skipped while valid sibling tools stay in the payload.

## Best Possible Solution

Use the existing runtime schema-projection or an equivalent JSON-safe clone for Anthropic direct/custom tool snapshots, add a regression for a getter below parameters.properties, and land one canonical Anthropic schema-guard PR.

## Reproduction Assessment

Yes. Source inspection shows current main reads direct/custom Anthropic tool schema fields without a guard, and a throwing getter on those fields would abort request construction; I did not run tests because this review is read-only.

## Solution Assessment

No. The patch is in the right owner boundary, but the snapshot should serialize or project the schema before storing it so deeper getter failures are caught before JSON.stringify builds the request body.
