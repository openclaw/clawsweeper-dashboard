- **[P1] Protect in-process restarts before sweeping claims:** `src/gateway/server-startup-ingress-sweep.ts:35`
  - body: The new guard skips only when a persisted supervisor handoff exists. SIGUSR1 restarts use the run-loop's in-process restart path and do not write that handoff before the next `startGatewayServer`, while channel shutdown and Telegram handler drain are bounded and can continue after timeout. The next bootstrap can therefore sweep an old but still-owned claim and requeue or duplicate the inbound event; carry an in-process restart/lifecycle owner into this guard before calling the sweep.
