## Summary

## What This Changes

The PR adds a gateway startup sweep that releases stale `channel_ingress_events` claims back to `pending` before channel workers start.

## Best Possible Solution

Land the guarded startup sweep after the normal exact-head CI and maintainer gates, keeping the fix bounded to shared SQLite ingress recovery with no new config or protocol surface.

## Solution Assessment

Yes. A startup sweep before channel workers start is the narrow maintainable fix because the existing per-queue drain path may never run while the shared queue is stuck behind claimed rows.
