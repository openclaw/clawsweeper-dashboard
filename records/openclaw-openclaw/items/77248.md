## Summary

Current main already covers the reported Telegram forum-topic drop: topic replies are routed back to the thread, and Telegram now treats a queued final with no observed delivery as undelivered instead of silently completing.

## What This Changes

The issue reports that automatic Telegram forum-topic replies generate a transcript response but no Telegram `sendMessage` attempt, while DMs and explicit message-tool sends work.

## Best Possible Solution

Keep the current observed-delivery guard, forum-topic fallback, and topic routing regression coverage, then let the next release carry the fix if it has not already shipped.

## Reproduction Assessment

No, not against current main: source inspection shows normal Telegram topic turns default to dispatch, and current regression tests cover both topic `sendMessage` routing and the queued-final-without-delivery fallback.

## Solution Assessment

Yes. The current fix is narrow and maintainable because it verifies actual Telegram delivery before considering the turn complete, without changing Telegram forum-topic configuration or adding new policy knobs.
