## Summary

Close as implemented on current main and shipped in v2026.5.3: the 2026.5.2 forum-topic symptom matches #76554, fixed by #76764, and current Telegram dispatch no longer treats a queued final as delivered without an observed send, edit, fallback, or intentional suppression.

## What This Changes

The issue reports that automatic Telegram replies generated for forum topic sessions are marked done but never delivered, while direct messages and explicit message-tool sends with threadId work.

## Best Possible Solution

Keep the shipped v2026.5.3 Telegram dispatch behavior; any new report after upgrading should include fresh gateway logs showing a remaining no-send path.

## Reproduction Assessment

Yes for the pre-fix 2026.5.2 bug: this issue and #76554 describe the same forum-topic path where final transcript text exists but no sendMessage is logged, and current main has regression coverage for a queued final with no observed delivery. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes: the shipped fix is the narrow owner-module change, because it removes queuedFinal as proof of delivery and requires an observed Telegram send/edit/fallback before marking the turn answered.
