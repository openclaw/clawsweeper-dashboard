## Summary

Current main treats Telegram forum topics as group/channel rooms: normal final answers are message-tool-only by default, while explicit message-tool sends and automatic mode already preserve the topic thread. The reported symptoms match documented configuration behavior rather than a Telegram delivery bug in the source tree.

## What This Changes

The issue asks OpenClaw to automatically deliver generated Telegram forum-topic replies back to their original topic thread instead of ending with no sendMessage call.

## Best Possible Solution

Keep the current group/topic visible-reply policy and direct users who want automatic room posts to set messages.groupChat.visibleReplies to automatic, while treating failures under that explicit setting as a new Telegram delivery bug.

## Reproduction Assessment

Yes for the observed no-sendMessage path: current source and tests show default group/channel turns suppress automatic final delivery. No high-confidence Telegram transport bug is reproduced because explicit message-tool and automatic-mode topic delivery paths are implemented.

## Solution Assessment

Yes. The narrow maintainable answer is configuration/docs guidance, not a Telegram-specific auto-send exception, because forum topics share the group/channel visible-reply policy and the existing automatic opt-in already covers legacy behavior.
