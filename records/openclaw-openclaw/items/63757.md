## Summary

Keep open. Current main still receives Telegram reaction updates but only converts added reactions into human-readable system events; neither the internal hook surface nor the typed plugin hook catalog exposes a stable structured reaction event for downstream feedback automation.

## What This Changes

The issue requests a stable structured Telegram reaction hook/event that downstream hooks or plugins can consume without patching core.

## Best Possible Solution

Add an additive typed reaction event seam, preferably cross-channel, after existing Telegram authorization and notification filtering; keep feedback storage and reward logic in hooks or plugins built on that seam.

## Reproduction Assessment

Yes, source-level. Driving the Telegram `message_reaction` path with `reactionNotifications: "all"` reaches a handler that enqueues text system events, while the internal and plugin hook catalogs have no reaction event to subscribe to.

## Solution Assessment

Yes directionally, but the maintainable solution is not a Telegram-only local patch. The best path is an additive stable hook contract whose payload and cross-channel scope are decided before implementation.
