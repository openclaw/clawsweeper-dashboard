## Summary

Keep open. Current main and v2026.5.19 still expose accepted Telegram reactions as session-scoped text system events, while the internal and public plugin hook catalogs have no stable structured reaction payload for downstream feedback integrations.

## What This Changes

Expose Telegram reaction updates as a stable structured internal/plugin hook for downstream feedback integrations.

## Best Possible Solution

Define an additive observation-only structured reaction hook/API, preferably generic across channels, then emit it from Telegram only after existing authorization and reaction notification filters pass, with SDK docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, source-level. An authorized Telegram message_reaction reaches the current handler and can enqueue a text system event, while current internal and plugin hook catalogs expose no structured reaction event to subscribe to.

## Solution Assessment

Unclear until maintainers choose the API shape. A typed additive reaction hook is the maintainable direction; an ad hoc Telegram-only local patch would risk drifting from the broader reaction/wake contract.
