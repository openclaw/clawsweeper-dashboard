## Summary

Keep open. Current main still misparses pnpm progress lines as unused-file paths, and this PR targets the right script with a focused parser guard plus a regression test. I found no correctness or security blocker in the proposed diff, so this should proceed through normal maintainer review and exact-head CI rather than cleanup closure.

## What This Changes

The PR adds repo-path validation to Knip compact unused-file parsing and a regression test for pnpm dlx progress lines before the unused-files section.

## Best Possible Solution

Merge the narrow parser and regression-test change after the required exact-head checks pass, keeping this behavior in the deadcode script rather than adding broader CI or Knip-wrapper changes.

## Reproduction Assessment

Yes. Feeding current main's parser pnpm `Progress:` lines before `Unused files (1)` reproduces the bad output deterministically without running Knip itself.

## Solution Assessment

Yes. Filtering compact rows by repo-path shape at the parser boundary is a narrow maintainable fix, and the added test preserves file-only compact output while covering the captured pnpm noise case.
