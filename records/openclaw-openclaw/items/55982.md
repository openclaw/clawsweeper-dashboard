- **[P1] Persist skipWhenIdle in cron jobs:** `src/cli/cron-cli/register.cron-add.ts:255`
  - body: The CLI now sends `skipWhenIdle`, but `createJob` constructs the persisted job field-by-field and `applyJobPatch` only handles known top-level fields.   - confidence: 0.94
- **[P1] Wire activity data before treating sessions as idle:** `src/cron/service/timer.ts:1054-1072`
  - body: This branch treats missing `getLastInboundAtMs` data as idle and returns a skipped run, but production `buildGatewayCronService` does not provide that callback. Any persisted non-`at` main-session job with `skipWhenIdle` would therefore skip every run in the normal gateway service until a real activity timestamp is supplied or the missing-data policy changes.
  - confidence: 0.92
- **[P2] Regenerate protocol clients for skipWhenIdle:** `src/gateway/protocol/schema/cron.ts:268`
  - body: Adding `skipWhenIdle` to the gateway cron schema changes the protocol contract, but the generated Swift protocol models still expose cron jobs and add params only through `failureAlert`. The generated artifacts need to be refreshed so protocol checks and clients match the schema.
  - confidence: 0.86
