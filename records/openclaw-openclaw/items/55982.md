## Summary

Keep open. The latest PR commit appears to address the earlier Greptile findings for one-shot jobs and `idleMs` minimums, but the branch still drops `skipWhenIdle` during normal add/update and would skip configured jobs in production because no activity provider is wired while missing activity data is treated as idle.

## What This Changes

The PR adds a `skipWhenIdle` cron job option, CLI add/edit flags, gateway protocol schema support, timer skip logic, and skipWhenIdle tests.

## Best Possible Solution

Persist the new option through cron add/update, wire idle checks to the target main session's `lastInteractionAt`, and add regression coverage plus cron docs/changelog before merge.

## Reproduction Assessment

Yes, source-level. The PR sends and validates `skipWhenIdle`, but current service creation/patch paths omit it, and production cron construction lacks the activity callback that the PR's skip logic requires.

## Solution Assessment

No. The one-shot and schema-minimum fixes are the right direction, but the implementation is incomplete until normal add/edit persist the option and production uses a real target-session activity timestamp.
