## Summary

Keep this PR open. The cache-miss bug and the helper direction are well supported, but the current patch changes the documented plugin hook system-context contract from cacheable static guidance to dynamic-suffix text, so it needs a maintainer compatibility decision or a narrower contract-preserving fix before merge.

## What This Changes

The PR adds cache-boundary append/ensure helpers and routes hook system context plus override runtime context through them, with regression tests for stable prompt-cache prefixes.

## Best Possible Solution

Preserve the existing cacheable static hook contract and add a clear dynamic system-context path for volatile runtime text, or explicitly approve and document the breaking hook contract change before merge.

## Reproduction Assessment

Yes. Source inspection shows current main joins hook system context around the marker-bearing base prompt while Anthropic only cache-controls the pre-marker block; the PR body also supplies live provider before/after cache counter output.

## Solution Assessment

Unclear. Boundary-aware composition is the right shape for volatile runtime context, but moving the public hook system-context fields below the marker conflicts with their documented cacheable-static contract.
