- **[P1] Wire dynamic hook fields through all prompt builders:** `src/plugins/hook-before-agent-start.types.ts:67-79`
  - body: The PR makes `prependDynamicSystemContext` and `appendDynamicSystemContext` public on `PluginHookBeforePromptBuildResult`, but only the embedded runner applies them. The CLI path and the generic agent harness used by Codex still pass only the static system fields, so plugins using the new documented API will silently lose dynamic system context depending on backend. Either wire these fields through every prompt-build consumer or keep the API internal/narrowly documented.
  - confidence: 0.9
- **[P1] Keep appendSystemContext above the cache marker:** `src/agents/pi-embedded-runner/run/attempt.thread-helpers.ts:58-63`
  - body: This implementation says static hook context stays in the cacheable prefix, but the flat `prepend/base/append` join appends `appendSystemContext` after the entire marker-bearing base prompt. For normal prompts where the boundary is inside `baseSystemPrompt`, that places the supposedly static append below the marker, so bundled/static callers still do not get the documented cacheable-prefix behavior. Insert static append before the boundary or change the contract/tests to match the actual placement.
  - confidence: 0.88
