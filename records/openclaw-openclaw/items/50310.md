## Summary

Keep open. Current main still resets archived Discord thread sessions in place and lacks a Discord liveness-based or agent-safe stale-session cleanup path; the related Dashboard pruning issue is relevant but does not fully cover the Discord archived-thread and agent-maintenance surface here.

## What This Changes

Request automatic or agent-accessible cleanup for Discord thread session rows that remain in `sessions.json` after the backing Discord thread is archived or deleted.

## Best Possible Solution

Add a Discord-aware stale-orphan preview/apply cleanup path that uses plugin-owned reachability checks, revalidates before deletion, and deletes through existing Gateway session writers under explicit admin or narrowly scoped agent permissions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: archived Discord thread handling sets matching session rows to `updatedAt = 0`, and `sessions.list` still reads those persisted rows from the local store.

## Solution Assessment

Unclear. The proposed startup GC, agent delete tool, and TTL expiry each cover part of the problem, but a preview/apply cleanup contract with Discord liveness checks is safer than unrestricted automatic deletion.
