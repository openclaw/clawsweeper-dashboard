## Summary

Keep open. Current main still resets archived Discord thread sessions in place, `sessions.list` still reports persisted rows without Discord reachability checks, and the newer admin cleanup/delete surfaces do not provide Discord-aware stale-orphan pruning or an agent-safe GC policy.

## What This Changes

Request automatic cleanup or deletion of stale Discord thread session rows after backing threads are archived or deleted.

## Best Possible Solution

Add a Discord-owned stale-orphan liveness check plus a Gateway-owned preview/apply cleanup path that revalidates candidates before deletion and uses the existing session writer, transcript archival, and explicit admin or narrowly scoped agent permission boundary.

## Reproduction Assessment

Yes, at source level. An archived Discord thread calls `closeDiscordThreadSessions`, which leaves the row present with `updatedAt = 0`, and `sessions.list` still lists persisted rows without Discord reachability checks.

## Solution Assessment

Unclear. Startup GC, a broad agent delete tool, and TTL expiry each solve part of the report, but the safer maintainable direction is plugin-owned liveness plus Gateway-owned preview/apply deletion semantics.
