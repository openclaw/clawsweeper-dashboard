## Summary

Keep open. Current main still resets archived Discord thread sessions in place and lists them from the local session store, while generic cleanup intentionally preserves channel/thread rows and the agent tool surface still has no delete or GC tool.

## What This Changes

Request auto-cleanup or agent-safe deletion of stale Discord thread session entries from `sessions.json` after backing Discord threads are archived, deleted, or inaccessible.

## Best Possible Solution

Add a Discord-aware stale-orphan preview/apply cleanup path that delegates liveness checks to the Discord plugin, revalidates candidates before deletion, and deletes through the Gateway session writer under explicit admin or narrowly scoped agent permission.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an archived Discord thread triggers `closeDiscordThreadSessions`, which leaves the session-store row with `updatedAt = 0`, and `sessions.list` still lists persisted rows without a Discord reachability check.

## Solution Assessment

Unclear. Startup GC, a broad agent delete tool, and TTL expiry each solve part of the problem, but the safer maintainable direction is an explicit preview/apply cleanup contract with plugin-owned liveness checks and Gateway-owned deletion semantics.
