## Summary

Keep open. Current main and latest `v2026.5.22` still reset archived Discord thread sessions in place and list stored session rows without Discord reachability or agent-safe stale-thread cleanup, while the overlapping Dashboard prune issue does not fully cover this Discord/agent cleanup policy.

## What This Changes

The issue requests automatic cleanup or deletion support for stale Discord thread session entries that remain in `sessions.json` after the backing Discord thread is archived or deleted.

## Best Possible Solution

Add Discord-owned stale-thread liveness and candidate discovery plus a Gateway-owned preview/apply cleanup path that revalidates before deletion, preserves transcript history intentionally, and exposes explicit admin or narrowly scoped agent permissions.

## Reproduction Assessment

Yes, at source level. An archived Discord thread calls `closeDiscordThreadSessions`, which leaves the row present with `updatedAt = 0`, and `sessions.list` still lists persisted rows without Discord reachability checks.

## Solution Assessment

Unclear. Startup GC, broad agent deletion, and TTL expiry each solve part of the report, but the maintainable direction needs explicit plugin-owned liveness plus Gateway-owned preview/apply deletion semantics.
