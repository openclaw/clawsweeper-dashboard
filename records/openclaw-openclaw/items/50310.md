## Summary

Keep open. Current main still marks archived Discord thread sessions stale in place, lists persisted session-store rows without Discord reachability checks, and lacks a Discord-aware prune/delete agent tool or cleanup policy; the broader dashboard prune issue and deleted-channel PR overlap but do not supersede the Discord archive/delete semantics here.

## What This Changes

The issue asks OpenClaw to remove or expose cleanup for stale Discord thread session entries that remain in `sessions.json` after the backing thread is archived, deleted, or inaccessible.

## Best Possible Solution

Add a Discord-aware stale-orphan preview/apply cleanup path that delegates liveness checks to the Discord plugin, revalidates candidates before deletion, and deletes through the Gateway session writer under explicit admin or narrowly scoped agent permission.

## Reproduction Assessment

Yes, at source level. An archived Discord thread triggers `closeDiscordThreadSessions`, which leaves the session-store row present with `updatedAt = 0`, and `sessions.list` still lists persisted rows without a Discord reachability check.

## Solution Assessment

Unclear. Startup GC, a broad agent delete tool, and TTL expiry each solve part of the report, but the safer maintainable direction is an explicit plugin-owned liveness check plus Gateway-owned preview/apply deletion semantics.
