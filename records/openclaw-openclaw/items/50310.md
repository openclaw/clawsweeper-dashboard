## Summary

Keep open. Current main has generic session cleanup, operator-level `sessions.delete`, selected-row Dashboard deletion, and Discord archive handling, but it still does not implement Discord reachability-based stale-thread GC or an agent-safe `sessions_delete`/`sessions_gc` tool for orphaned Discord thread sessions.

## What This Changes

The issue asks for automatic or agent-invoked cleanup of stale Discord thread session entries that remain in `sessions.json` after the backing Discord thread is archived, deleted, or inaccessible.

## Best Possible Solution

Add an additive, plugin-owned Discord liveness/cleanup path that previews candidates, revalidates before deletion, preserves transcript policy, and exposes only narrowly scoped operator or agent controls.

## Reproduction Assessment

No high-confidence runtime reproduction was run because this is a feature request. Source inspection does confirm the missing pieces: no Discord delete/liveness GC path and no agent-facing `sessions_delete` or `sessions_gc` tool.

## Solution Assessment

Unclear until maintainers choose the cleanup policy. The current generic cleanup and explicit delete paths are useful partial building blocks, but they are not the best complete solution for Discord orphan detection by themselves.
