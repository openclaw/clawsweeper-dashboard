## Summary

Keep open. Current main and v2026.6.10 still mark archived Discord thread sessions stale instead of deleting them, preserve durable Discord channel/thread rows during generic cleanup, and expose only admin explicit-key deletion rather than a Discord orphan preview or GC path.

## What This Changes

Request automatic or agent/operator-triggered cleanup for Discord session rows whose backing thread or channel is archived, deleted, or inaccessible.

## Best Possible Solution

Define a maintainer-approved Discord/plugin-owned orphan cleanup flow that previews and revalidates unreachable candidates, then deletes through the existing session lifecycle with explicit permissions and transcript-retention semantics.

## Reproduction Assessment

Yes, at source level: current main resets archived Discord thread rows, preserves external Discord-like conversation rows during generic pruning, and lacks a Discord reachability cleanup or agent-facing delete/GC tool. I did not run a live Discord 403/404 deletion scenario in this read-only review.

## Solution Assessment

No, the current implementation is only a partial manual/operator path. The best fix is a plugin-owned reachability preview/apply flow that reuses existing delete hooks, not unconditional startup polling or unrestricted agent deletion.
