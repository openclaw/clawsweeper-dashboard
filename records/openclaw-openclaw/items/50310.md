## Summary

Keep open. Current main still marks archived Discord thread sessions stale rather than pruning their session-store rows, and the existing delete/cleanup paths are admin or CLI surfaces rather than a Discord liveness-based or agent-safe cleanup feature; related #49259 overlaps but is broader Dashboard pruning work and remains open.

## What This Changes

The issue requests automatic or agent-accessible cleanup of stale Discord thread session entries that remain in the session store after the backing Discord thread is archived or deleted.

## Best Possible Solution

Add a Discord-owned orphan reconciliation flow that previews and then deletes only confirmed unreachable Discord channel/thread session rows through existing Gateway session writers, with explicit agent/admin/UI permission semantics.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: seed `agent:<agentId>:discord:channel:<threadId>`, invoke the current archived-thread close path, and the key remains in the store with `updatedAt = 0` while `sessions.list` reads from that store.

## Solution Assessment

Unclear. The proposed options each solve part of the problem, but a scoped preview/apply prune using Discord liveness checks and existing Gateway deletion is safer than a broad unrestricted agent delete tool.
