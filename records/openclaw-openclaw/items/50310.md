## Summary

Keep open. Current main still marks archived Discord thread sessions stale and preserves durable channel/thread rows during cleanup; the remaining work is product-sensitive orphan cleanup rather than already-implemented behavior.

## What This Changes

Request automatic or explicit pruning of Discord thread session entries that point to archived, deleted, or inaccessible threads so they stop polluting `sessions_list`.

## Best Possible Solution

Design a product-approved Discord orphan lifecycle with previewable candidates, plugin-owned reachability checks, explicit retention rules, and clear admin or agent permissions before any destructive cleanup runs.

## Reproduction Assessment

Yes, source inspection provides a high-confidence reproduction path: archived Discord threads are reset to `updatedAt: 0`, while `sessions.list` reads persisted rows and cleanup preserves durable Discord channel/thread entries. I did not run a live Discord deletion or archive scenario in this read-only review.

## Solution Assessment

Unclear as proposed: the need is valid, but unconditional startup GC, a new agent delete tool, and TTL auto-expiry each make different product and safety tradeoffs. A preview-first prune flow with explicit reachability and retention semantics looks safer than deleting rows automatically by default.
