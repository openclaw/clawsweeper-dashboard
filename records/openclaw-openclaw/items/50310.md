## Summary

Keep open. Current main has partial Discord thread-binding reachability cleanup, generic session maintenance, and operator-level session deletion, but it still does not remove orphaned Discord channel/thread rows from `sessions.json` or expose an agent-safe `sessions_delete`/`sessions_gc` path.

## What This Changes

Request automatic or agent-invoked cleanup of `sessions.json` entries for Discord thread sessions whose backing threads are archived, deleted, or inaccessible.

## Best Possible Solution

Add a plugin-owned Discord stale-session cleanup flow that previews candidates, revalidates reachability before deleting rows, preserves transcript policy, and exposes narrowly scoped operator or agent controls only after maintainer policy approval.

## Reproduction Assessment

No runtime reproduction was run because this is a feature request, but source inspection gives a high-confidence source reproduction. Discord archived-thread handling marks matching rows stale, generic maintenance protects channel/thread rows, and no agent-facing delete/GC tool exists.

## Solution Assessment

Unclear until maintainers choose the destructive cleanup policy. The narrow maintainable path is plugin-owned reachability preview/apply cleanup, not unconditional generic core startup polling.
