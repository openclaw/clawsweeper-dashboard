## Summary

Keep open. Current main and the latest release still only mark archived Discord thread sessions stale, preserve external channel/thread rows during generic maintenance, and lack an agent-safe Discord session delete or GC path; the broader Dashboard prune request overlaps but does not fully supersede this Discord-specific plugin and permission policy work.

## What This Changes

Request automatic or agent-triggered cleanup for stale Discord thread session entries whose backing thread is archived, deleted, or inaccessible.

## Best Possible Solution

Add a Discord plugin-owned preview/apply stale-session cleanup flow that verifies reachability, revalidates candidates before deletion, preserves transcript policy, and exposes operator or agent controls only after maintainer-approved permission semantics.

## Reproduction Assessment

Yes, at source level; no live Discord repro was run. Source and tests show archived-thread handling leaves rows in sessions.json, generic maintenance preserves channel/thread rows, and the agent tool catalog lacks delete or GC tools.

## Solution Assessment

Unclear until maintainers choose the destructive cleanup and permission policy. The maintainable direction is plugin-owned reachability preview/apply cleanup rather than unconditional generic startup polling or a broad delete tool by default.
