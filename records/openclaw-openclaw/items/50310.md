## Summary

Keep open. Current main still resets Discord archived-thread session rows instead of pruning them, and the existing admin delete/cleanup paths do not provide Discord liveness-based or agent-safe stale session GC; related #49259 overlaps but remains broader Dashboard pruning work.

## What This Changes

Add automatic or agent/admin-triggered cleanup for stale Discord session entries whose backing threads or channels are archived, deleted, or inaccessible.

## Best Possible Solution

Add a Discord-owned orphan reconciliation flow that previews and then deletes only confirmed unreachable Discord channel/thread session rows through existing Gateway session mutation paths, coordinated with the broader Dashboard pruning request.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: seed a `agent:<agentId>:discord:channel:<threadId>` row, invoke the archived-thread close path, and the key remains in the store with `updatedAt = 0` while `sessions.list` reads from that store.

## Solution Assessment

Unclear. The requested options each solve part of the problem, but a scoped preview/apply prune using Discord liveness checks and existing Gateway deletion is safer than exposing a broad unrestricted agent delete tool.
