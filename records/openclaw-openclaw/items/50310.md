## Summary

Keep open. Current main still only marks archived Discord thread sessions stale and offers generic/manual session cleanup; it does not prune confirmed orphaned Discord channel/thread entries from sessions.json, hide them from sessions_list, or expose a scoped agent-safe sessions_delete/sessions_gc tool.

## What This Changes

Request automatic cleanup or an agent-accessible cleanup path for stale Discord thread session entries that remain in sessions.json after the backing Discord thread is archived, deleted, or inaccessible.

## Best Possible Solution

Add an additive, Discord-owned orphan reconciliation path, preferably through a generic session prune preview/apply contract that calls plugin-owned reachability checks, revalidates before deletion, and removes or hides only confirmed orphaned Discord session entries from sessions.json and sessions_list.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: seed a matching agent:<agentId>:discord:channel:<threadId> store entry, run the current Discord thread close path, and the tested behavior leaves the key present with updatedAt reset to 0; sessions.list then enumerates local store rows without Discord liveness validation.

## Solution Assessment

Unclear. The current implementation is not sufficient, and the issue’s options need maintainer choice; the safest direction is a scoped prune preview/apply workflow with Discord reachability checks, not a broad unrestricted agent deletion tool.
