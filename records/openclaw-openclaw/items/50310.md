## Summary

Keep open. Current main still resets archived Discord thread sessions by setting `updatedAt = 0` while leaving the `sessions.json` row in place, and existing delete/cleanup surfaces are admin/UI/plugin-owned paths rather than a Discord liveness-based or agent-safe cleanup feature.

## What This Changes

Request automatic or agent-accessible cleanup for stale Discord thread session rows that remain after Discord threads are archived, deleted, or become inaccessible.

## Best Possible Solution

Add a Discord-owned orphan reconciliation flow that can preview confirmed unreachable Discord channel/thread session rows and delete them through existing Gateway session writers with explicit admin or narrowly scoped agent permissions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `agent:<agentId>:discord:channel:<threadId>`, invoke the archived-thread close path, and the key remains in the store with `updatedAt = 0` while `sessions.list` still reads from the local store.

## Solution Assessment

Unclear. The proposed options each cover part of the problem, but a preview/apply prune using Discord liveness checks and existing Gateway deletion semantics is safer than an unrestricted agent `sessions_delete` tool.
