## Summary

Keep open: the PR targets a real source-visible reminder-guard gap, but current main does not implement it and the branch still has blocking regex and target-filter defects plus missing real behavior proof.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to the reminder guard and a regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Land one consolidated reminder-guard update that preserves same-session suppression and adds a narrow, documented enabled isolated `agentTurn` sweep contract with correct word-boundary matching, target filtering, positive and negative tests, and redacted real channel or gateway proof.

## Reproduction Assessment

Yes, at source level. Current main only recognizes same-turn cron adds or same-session cron rows, and the PR head's new sweep path has source-visible defects in the regex bytes and target filtering.

## Solution Assessment

No. The direction is plausible, but this branch should fix the regex bytes, restrict the heuristic to enabled isolated `agentTurn` jobs, add negative coverage, and be reconciled with the overlapping reminder-guard PRs before merge.
