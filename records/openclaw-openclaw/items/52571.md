## Summary

Keep open. The PR targets a real current-main reminder-guard gap, but the branch is not merge-ready because the new sweep detection is inert with raw backspace regex bytes, the heuristic is broader than its isolated-cron contract, and changelog coverage is missing.

## What This Changes

The PR adds a reminder-sweep-like cron fallback to suppress the unscheduled-reminder note and extends the runReplyAgent reminder-guard tests.

## Best Possible Solution

Preserve same-session cron suppression and add a narrow enabled isolated agentTurn reminder-sweep suppression path with correct regex bytes, positive and negative tests, and changelog coverage.

## Reproduction Assessment

Yes. Current-main behavior is visible in the mocked runReplyAgent tests and the guard call path: same-session active crons suppress the note, unrelated crons do not, and zero successfulCronAdds triggers the warning path.

## Solution Assessment

No, not as-is. The direction is maintainable once the regex is fixed, the heuristic is limited to isolated cron jobs, negative coverage is added, and the user-facing changelog entry is present.
