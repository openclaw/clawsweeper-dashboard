## Summary

## What This Changes

The PR adds a sweep-like cron-job heuristic to suppress the unscheduled-reminder note and adds a regression test for agent-persisted reminders handled by a separate sweep job.

## Best Possible Solution

Define one narrow reminder-guard contract for sweep-backed reminders: preserve same-session suppression, add a scoped enabled isolated agentTurn sweep path with positive and negative tests, and consolidate it with the overlapping reminder-guard fixes before merge.

## Reproduction Assessment

Yes, at source level. Current main still only recognizes same-turn cron adds or same-session cron rows, and PR head visibly fails its intended sweep match because the regex contains raw U+0008 bytes.

## Solution Assessment

No. The direction is plausible, but this branch needs the matcher fixed, the isolated-job contract narrowed or explicitly approved, conflicts resolved, and real behavior proof added before it is the best fix.
