## Summary

Keep open. Current main still only suppresses the unscheduled-reminder note for successful `cron.add` calls or enabled cron jobs with the same session key, while this PR has useful source work for the sweep-backed reminder case but introduces blocking review defects.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to `hasSessionRelatedCronJobs` and a `runReplyAgent` regression test so an existing recurring sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression, add a narrow path for enabled isolated `agentTurn` reminder-sweep jobs, use real JavaScript word-boundary escapes, cover both positive and negative cases, and add the required changelog entry for the user-visible fix.

## Reproduction Assessment

Yes. The current-main false-positive path is covered by the existing mocked `runReplyAgent` tests, and the PR-specific regression is reproducible from the diff bytes plus a direct Node probe showing the new regex fails on the PR's own representative sweep text.

## Solution Assessment

No, not as-is. The direction is plausible, but the implementation needs the isolated-job filter, a corrected regex, negative coverage, and a changelog entry before it is the narrow maintainable fix.
