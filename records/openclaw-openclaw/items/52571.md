## Summary

Keep open: the underlying false reminder-disclaimer behavior is still present on current main and the latest release, but this PR is not merge-ready because the new sweep matcher is broken, the heuristic is broader than the stated isolated-cron contract, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to suppress the unscheduled-reminder note and adds one runReplyAgent regression test for a separate sweep cron.

## Best Possible Solution

Land one consolidated reminder-guard update that preserves same-session suppression and adds a narrow enabled isolated agentTurn sweep contract with correct word-boundary matching, negative coverage, and redacted real channel proof.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.18 append the note when reminder text has successfulCronAdds set to zero and no same-session enabled cron is found, while the PR-head regex and missing isolated filter are directly visible from the source bytes.

## Solution Assessment

No. The direction is plausible, but this branch should fix the regex bytes, restrict the heuristic to enabled isolated agentTurn sweep jobs, add negative coverage, provide real behavior proof, and coordinate with the overlapping reminder-guard PRs before merge.
