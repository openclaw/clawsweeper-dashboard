## Summary

Keep open: current main still lacks sweep-backed reminder suppression, but this branch is not mergeable because the new regex contains raw backspace bytes, the helper is broader than the isolated-cron contract, and external real behavior proof is absent.

## What This Changes

The PR adds a sweep-like cron heuristic to the reminder guard and a regression test so a pre-existing reminder sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Define and land one narrow reminder-guard contract for enabled isolated agentTurn sweep jobs, with correct matching, target filtering, positive and negative tests, and redacted real channel or gateway proof.

## Reproduction Assessment

Yes, at source level. Current main only recognizes same-turn cron adds or same-session cron rows, and the PR head has source-visible defects in the regex bytes and target filtering.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the branch should fix the regex bytes, restrict the heuristic to enabled isolated agentTurn jobs, add negative coverage, and be reconciled with overlapping reminder-guard work before merge.
