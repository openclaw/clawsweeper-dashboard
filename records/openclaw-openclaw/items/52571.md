## Summary

Keep open. Current main still only suppresses the reminder guard for successful cron adds or same-session enabled cron jobs, and this PR has concrete blockers before it can safely cover sweep-backed reminders.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to the reminder guard and a runReplyAgent regression test for sweep-backed reminders.

## Best Possible Solution

Preserve same-session cron suppression and add a narrow isolated agentTurn reminder-sweep suppression path with correct regex bytes, positive and negative coverage, and a user-facing changelog entry.

## Reproduction Assessment

Yes. Current main reproduces the underlying gap through the existing runReplyAgent guard path, and the PR-specific failure is visible by inspecting the raw U+0008 regex bytes against the PR's own sweep test text.

## Solution Assessment

No, not as-is. The direction is maintainable, but the patch needs real word-boundary escapes, an isolated-session filter, focused negative coverage, and changelog coverage before merge.
