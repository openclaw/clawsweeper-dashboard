## Summary

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to the reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Land one consolidated reminder-guard update that preserves same-session suppression and adds a narrow enabled isolated agentTurn sweep contract with correct matching, positive and negative coverage, conflict resolution, and redacted real channel proof.

## Reproduction Assessment

Yes, at source level. Current main appends the note when reminder text has successfulCronAdds set to zero and no same-session enabled cron is found, and the PR-head regex/session-target defects are visible from source plus a focused Node probe.

## Solution Assessment

No, not as written. The direction is plausible, but this branch needs correct word-boundary matching, an isolated-agentTurn-only heuristic, negative coverage, conflict resolution, and real behavior proof before merge.
