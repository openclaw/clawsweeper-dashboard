## Summary

Keep open. Current main still has the sweep-backed reminder gap, and this PR is useful source work, but the proposed patch is not correct yet because the sweep regex contains control bytes and the heuristic is broader than the isolated-cron contract.

## What This Changes

The branch adds a reminder-sweep-like cron heuristic to `hasSessionRelatedCronJobs` and a `runReplyAgent` regression test for sweep-backed reminders.

## Best Possible Solution

Preserve same-session cron suppression and add a narrow isolated `agentTurn` reminder-sweep suppression path with correct regex bytes, positive and negative coverage, and a user-facing changelog entry.

## Reproduction Assessment

Yes. The current-main behavior is visible in the existing mocked `runReplyAgent` tests, and the PR-specific failure is reproducible by inspecting the raw regex bytes plus a Node probe against the PR's own sweep text.

## Solution Assessment

No, not as-is. The direction is maintainable, but the branch needs the regex fixed, the heuristic limited to isolated cron jobs, focused negative coverage, and changelog coverage before merge.
