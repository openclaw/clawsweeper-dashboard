## Summary

Keep this PR open: it targets a real reminder-guard false positive, but the current branch still has blocking correctness defects and lacks after-fix real behavior proof from a real setup.

## What This Changes

The branch adds a reminder-sweep-like cron-job heuristic to suppress the unscheduled-reminder note and adds a runReplyAgent regression test.

## Best Possible Solution

Preserve same-session suppression and add one narrow, documented contract for enabled isolated `agentTurn` reminder-sweep coverage, with correct word-boundary matching, isolated-target filtering, positive and negative tests, and redacted real channel or gateway proof.

## Reproduction Assessment

Yes, at source level. Current main only recognizes same-turn cron adds or same-session cron rows, and the PR head also has source-visible defects in the regex and target filtering.

## Solution Assessment

No, not as proposed. The branch should use real word-boundary escapes, restrict the heuristic to enabled isolated `agentTurn` jobs, add negative coverage, and be reconciled with the overlapping reminder-guard PRs.
