## Summary

Keep open: current main still lacks sweep-backed reminder suppression, but this PR has blocking correctness defects and no required real behavior proof. The overlapping reminder-guard PRs also need maintainer consolidation before this path is ready to land.

## What This Changes

The PR adds a reminder-sweep-like cron-job heuristic to the auto-reply reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve same-session suppression and add a narrow isolated agentTurn reminder-sweep contract with positive and negative coverage, or fold that contract into the selected broader reminder-guard fix.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main only recognizes same-turn cron adds or same-session cron rows, and focused tests still expect unrelated cron rows to append the note. I did not run runtime tests because this review is read-only.

## Solution Assessment

No, not as proposed. The branch should use real word-boundary escapes, restrict the heuristic to enabled isolated agentTurn cron jobs, add negative coverage, and be reconciled with the overlapping reminder-guard PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use real word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:35`
  - body: The added regex contains literal U+0008 control bytes around the alternation instead of JavaScript `\b` word-boundary escapes. Normal cron names/messages such as `family-reminder sweep` will not match, so the new suppression path stays inactive for the intended sweep-backed reminder case.
  - confidence: 0.97
- **[P2] Limit sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:25`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job, not only isolated jobs. A current/custom/session cron from another context whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply, which is broader than the PR's stated contract.
  - confidence: 0.88
