## Summary

Keep open: current main still lacks a sweep-backed suppression path, but this PR is not mergeable as-is because the helper’s regex uses U+0008 control bytes, the heuristic is broader than its isolated-cron contract, and no after-fix real behavior proof is attached.

## What This Changes

The PR adds a reminder-sweep-like cron-job heuristic to the reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve same-session suppression and add a narrow, documented path for enabled isolated agentTurn reminder-sweep jobs with correct word-boundary matching, an isolated-target check, positive and negative coverage, and coordination with overlapping reminder-guard fixes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main only recognizes same-turn cron adds or same-session cron rows, so an agent-persisted reminder backed by a separate isolated sweep job still gets the note.

## Solution Assessment

No, not as proposed. The branch should use real word-boundary escapes, restrict the heuristic to enabled isolated agentTurn cron jobs, add negative coverage, and be reconciled with the overlapping reminder-guard PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use real word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:36`
  - body: The added regex contains literal U+0008 control bytes around the alternation instead of `\b` word-boundary escapes, so names/messages like `family-reminder sweep` will not match. The new suppression path therefore stays inactive for the regression case and the guard note can still be appended.
  - confidence: 0.97
- **[P2] Restrict sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:26`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job, not only isolated jobs. A main, custom, or session cron from another context whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply, which is broader than the PR's stated contract.
  - confidence: 0.88
