## Summary

Keep open: current main still has the same-session-only reminder guard, but this external PR cannot merge as-is because its sweep regex contains raw backspace bytes, the heuristic is broader than the stated isolated-cron contract, and no real behavior proof is attached.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to suppress the unscheduled reminder note and adds one runReplyAgent regression test for a separate sweep cron.

## Best Possible Solution

Land one consolidated reminder-guard update that preserves same-session suppression and adds a narrow enabled isolated agentTurn sweep contract with correct word-boundary matching, negative coverage, and redacted real channel proof.

## Reproduction Assessment

Yes, at source level. Current main appends the note when reminder text has successfulCronAdds set to zero and only an unrelated or sweep cron exists, and the PR head defects are visible from the decoded source bytes and missing target check.

## Solution Assessment

No. The direction is plausible, but this branch should fix the regex bytes, restrict the heuristic to enabled isolated agentTurn jobs, add negative coverage, and coordinate with the overlapping reminder-guard PRs before merge.
