## Summary

Keep open: current main and the latest release still compute auth-profile cooldowns from local stepped backoff without a provider reset timestamp, and the linked fix PR is open/unmerged rather than an implemented fix.

## What This Changes

The issue asks OpenClaw to honor provider reset timestamps for rate-limit cooldowns and make multi-key profile rotation less aggressive for transient provider errors.

## Best Possible Solution

Review and either land a corrected version of https://github.com/openclaw/openclaw/pull/97770 or keep this issue as the canonical place to settle a typed reset-hint and error-class-aware rotation policy.

## Reproduction Assessment

Yes, at source level: the reported reset-text quota error is classified as `rate_limit`, and the auth-profile failure path computes cooldowns from local counters without reset metadata. I did not run a live exhausted ZAI account in this read-only review.

## Solution Assessment

No, the issue is not solved on current main; the open linked PR is a candidate but still needs review and merge before this can close. The maintainable solution should carry validated reset hints through all relevant profile-failure paths and decide the default rotation policy explicitly.
