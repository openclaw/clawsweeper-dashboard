## Summary

## What This Changes

The PR widens four compaction threshold config fields to accept percentage strings, adds a token-threshold resolver, applies it in memory preflight/flush paths, and updates config help text.

## Best Possible Solution

Preserve numeric configs and unset defaults, resolve percentage strings consistently at the config/memory ownership boundary used by embedded compaction and memory-core, and add focused tests plus redacted real behavior proof for active-model context-window scaling.

## Reproduction Assessment

Partially. Source inspection gives a high-confidence reproduction of the PR defects: newly valid percentage strings are still rejected by numeric consumers, and the default resolver parameters change existing fallback behavior; no live model-switch compaction proof is supplied.

## Solution Assessment

No. Percentage strings are a plausible direction, but this implementation resolves too late for key consumers and changes defaults; the safer solution is consistent resolution at the config/memory boundary with numeric backward compatibility.
