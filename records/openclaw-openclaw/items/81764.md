## What This Changes

The PR adds Telegram HTML-to-plain fallback helpers and a regression test so HTML parse-mode retries derive readable plain text from rendered anchors when no caller `plainText` fallback is provided.

## Best Possible Solution

Land the narrow Telegram sender fallback hardening after required CI, preserving explicit caller `plainText` and deriving readable fallback text only for HTML-mode sends without one.
