## Summary

Keep open: the performance target is valid, but this patch makes discovery existence probes a process-wide cache with no owner-scoped lifetime, so stale misses can affect later plugin discovery or install work in the same process. It also needs the required changelog entry for a user-facing perf change.

## What This Changes

The PR adds a shared memoized `fs.existsSync` helper, routes plugin scan probes in bundle manifest, bundled directory, and SDK alias resolution through it, and adds unit tests for cache invalidation.

## Best Possible Solution

Keep the startup optimization, but make existence memoization scoped to a single discovery/startup pass or an explicit plugin-loader owner cache, add stale false-to-true regression coverage, and add the changelog entry.

## Reproduction Assessment

Yes from source inspection: a first `existsSyncCached` miss for a plugin marker path is stored indefinitely, and the plugin discovery/install callers shown on current main do not invalidate that cache before later probes in the same process.

## Solution Assessment

No. The narrow optimization is worthwhile, but a shared process-global cache is broader than the plugin discovery boundary allows; a scan-scoped or explicitly owned cache is the safer maintainable shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope existence caching to discovery:** `src/shared/cached-fs.ts:18`
  - body: `existsSyncCached` stores every result in a process-wide map and never expires it. A false miss while scanning a plugin root can therefore be reused by later discovery/install work after `.codex-plugin/plugin.json`, `skills`, SDK artifacts, or other marker files are created in the same Node process, which violates the plugin boundary rule that discovery metadata stays fresh unless the caller owns an explicit cache. Please make this cache scoped to the scan/loader operation or invalidate it at the owning boundary instead of sharing it globally.
  - confidence: 0.9
