## Summary

Keep open. Current main still loads bootstrap files from the workspace only, and this PR is the active implementation for the linked warning-only fix, but the patch still has merge-blocking behavior and test issues that match prior review discussion.

## What This Changes

The PR adds an agentDir bootstrap-file warning helper, threads agentDir into embedded attempt bootstrap resolution, and adds tests for the diagnostic behavior.

## Best Possible Solution

Keep workspace-only bootstrap loading and add a warning-only diagnostic for recognized agentDir bootstrap filenames only when the corresponding effective workspace file is absent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: put SOUL.md in both workspaceDir and agentDir and the PR helper still warns; with SOUL.md only in agentDir, current main does not load that content and returns the normal workspace missing placeholder.

## Solution Assessment

No, not as currently patched. The warning-only direction is the narrow backward-compatible fix, but this implementation needs the workspace guard, corrected placeholder expectations, caller plumbing, and changelog entry before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Skip warnings when the workspace file is loaded:** `src/agents/bootstrap-files.ts:273-277`
  - body: The helper warns as soon as `agentDir/<name>` exists. When `workspaceDir/<name>` also exists, that workspace file is the effective loaded copy, so this creates a false warning and contradicts the PR's both-directories case; check the workspace path before calling `params.warn`.
  - confidence: 0.94
- **[P2] Assert missing placeholders instead of absence:** `src/agents/bootstrap-files.agentdir-warning.test.ts:39`
  - body: `resolveBootstrapFilesForRun` keeps standard absent workspace bootstrap files as `missing: true` placeholders. These assertions expect SOUL.md/AGENTS.md entries to be absent, so the tests fail even though agentDir content was not loaded; assert missing/non-content state instead.
  - confidence: 0.91
- **[P2] Pass agentDir through CLI prompt preparation:** `src/agents/bootstrap-files.ts:293`
  - body: The new diagnostic only runs when callers pass `agentDir`, but CLI prompt preparation already resolves `agentDir` for auth and still calls `resolveBootstrapContextForRun` without it. CLI backend runs with ignored agentDir bootstrap files would remain silent; thread the known agentDir through this caller and cover it.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/agents/bootstrap-files.ts:273`
  - body: This adds a user-facing runtime warning for a bug fix but leaves `CHANGELOG.md` unchanged. Repo policy requires a single-line entry under the active Fixes section with suitable contributor credit.
  - confidence: 0.86
