## Summary

Keep open. Current main still has the reported workspace-only bootstrap behavior and this PR is a plausible narrow fix path, but the proposed patch is not merge-ready because it can emit false warnings when the effective workspace already contains the bootstrap file, and it lacks the required changelog entry for a user-facing fix.

## What This Changes

The PR threads an optional `agentDir` into bootstrap resolution, adds an `agentDir` bootstrap-file warning helper and tests, and passes `agentDir` from the embedded runner call site.

## Best Possible Solution

Ship the warning-only diagnostic as the backward-compatible fix: warn only for recognized bootstrap filenames present in `agentDir` and absent from the effective workspace, keep workspace-only loading semantics unless maintainers separately approve per-agent merge/override behavior, and cover the behavior with focused tests plus a changelog entry.

## Reproduction Assessment

Yes. The PR's own both-directories test is a high-confidence reproduction path: create the same bootstrap filename in both `workspaceDir` and `agentDir`; the current PR helper checks only `agentDir` before warning.

## Solution Assessment

No, not as currently implemented. A warning-only diagnostic is the narrow maintainable approach for this bug, but it must suppress warnings when the workspace copy exists and add the required changelog entry before merge.
