## Summary

Keep open: current main still has the silent `agentDir` bootstrap-file gap, and this PR is the active closing candidate for the linked bug, but the proposed patch still has correctness issues and lacks after-fix real behavior proof.

## What This Changes

The branch adds an `agentDir` bootstrap-file warning helper, threads `agentDir` into one embedded-runner bootstrap call, and adds warning-focused tests.

## Best Possible Solution

Land a rebased warning-only diagnostic that checks the effective workspace file first, covers every bootstrap resolver path, preserves current placeholder semantics, and leaves `agentDir` merge/override behavior to a separate decision.

## Reproduction Assessment

Yes, by source inspection. Current main loads bootstrap context from `workspaceDir` only, and the PR’s false-warning path is visible when the same filename also exists in the workspace.

## Solution Assessment

No, not as currently patched. Warning-only is a narrow maintainable fix, but this branch false-warns, has assertions that conflict with current missing-placeholder behavior, and covers only one resolver path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip warnings when the workspace file is loaded:** `src/agents/bootstrap-files.ts:271-275`
  - body: The helper warns as soon as `agentDir/<name>` exists, but when `workspaceDir/<name>` also exists that workspace file is the effective bootstrap input. This creates a false warning for correctly configured workspaces; check the workspace path or effective file set before calling `params.warn`.
  - confidence: 0.94
- **[P2] Assert missing placeholders instead of absent entries:** `src/agents/bootstrap-files.agentdir-warning.test.ts:38`
  - body: `loadWorkspaceBootstrapFiles` returns missing placeholders for standard bootstrap filenames when those workspace files are absent. These assertions expect `SOUL.md` and `AGENTS.md` entries to be absent, so the tests fail even though `agentDir` content was not loaded; assert `missing` or content state instead.
  - confidence: 0.91
- **[P2] Pass agentDir through every bootstrap resolver path:** `src/agents/pi-embedded-runner/run/attempt.ts:773`
  - body: This only forwards `agentDir` from one embedded-runner call. The embedded preloaded/fallback paths, compaction, and CLI preparation still resolve bootstrap context without an agent directory, so they can continue silently ignoring `agentDir` bootstrap files.
  - confidence: 0.84
