## Summary

Keep open: current main and v2026.5.7 still resolve bootstrap context from workspace files only, and this PR is the active closing candidate for https://github.com/openclaw/openclaw/issues/29387. The patch is not mergeable as-is because it can emit false warnings, has test expectations that contradict current placeholder behavior, misses runtime caller coverage, and lacks contributor real-behavior proof.

## What This Changes

The PR adds a bootstrap warning for recognized files under agentDir, threads agentDir into one embedded-runner bootstrap call, and adds tests for warning behavior.

## Best Possible Solution

Keep the workspace-only bootstrap contract for now, add a warning-only diagnostic across runtime paths that know the effective agentDir, and leave agentDir merge/override semantics to a separate product decision.

## Reproduction Assessment

Yes by source inspection. Current main loads bootstrap files from workspaceDir only, and the PR helper's own code path warns on agentDir files before checking whether the corresponding workspace file is already loaded.

## Solution Assessment

No, not as currently patched. A warning-only diagnostic is the narrowest maintainable fix, but this implementation needs a workspace/effective-load guard, corrected placeholder assertions, and complete caller coverage before merge.

  - body: The helper warns as soon as agentDir/<name> exists, but when workspaceDir/<name> also exists that workspace file is the effective bootstrap input. This creates a false warning for correctly configured workspaces; check the workspace path/effective file before calling params.warn.

  - confidence: 0.91

CLI preparation and compaction also resolve bootstrap context and will still silently ignore agentDir bootstrap files, so the warning will not cover the reported runtime surface unless every effective caller passes or resolves agentDir.
  - confidence: 0.82
