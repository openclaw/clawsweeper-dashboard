## Summary

Keep this PR open. Current main still does not warn about bootstrap-named files in `agentDir`, so the contribution targets a real gap, but the proposed patch has merge-blocking correctness issues: it emits a false warning when the workspace copy is present and adds assertions that contradict the existing missing-placeholder bootstrap contract.

## What This Changes

The PR adds an agentDir bootstrap-file warning helper, passes `agentDir` into embedded bootstrap context resolution, and adds tests for the warning behavior.

## Best Possible Solution

Land this as a warning-only diagnostic that preserves workspace-only bootstrap loading: warn only for recognized bootstrap filenames present in `agentDir` when the effective workspace file is absent, keep missing placeholders intact, and document the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: create `SOUL.md` in both `workspaceDir` and `agentDir`, and the PR helper still warns; create `SOUL.md` only in `agentDir`, and the added test expects no `SOUL.md` result even though current main returns a missing placeholder for that standard filename.

## Solution Assessment

No, not as currently implemented. A warning-only diagnostic is the narrowest maintainable direction, but this patch needs the workspace-existence guard, placeholder-aware assertions, and changelog before it is the right fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip warnings when the workspace copy exists:** `src/agents/bootstrap-files.ts:273-277`
  - body: The helper warns as soon as `agentDir/<name>` exists, but when `workspaceDir/<name>` also exists that workspace file is the one being loaded. This produces a false warning for correctly configured workspaces and contradicts the PR's own both-directories test; check the workspace path before calling `params.warn`.
  - confidence: 0.94
