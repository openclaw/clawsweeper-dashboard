## Summary

Keep open. The PR targets a valid remaining gap from #29387, and current main still only loads bootstrap files from the workspace, but the patch has merge-blocking issues: it warns even when the workspace copy is the effective file, two new tests contradict current missing-placeholder behavior, and the user-facing fix lacks the required changelog entry.

Repair this as a warning-only diagnostic: keep the workspace-only bootstrap loading contract, warn only when a recognized bootstrap filename exists in agentDir and the corresponding effective workspace file is absent, preserve warning dedupe behavior through the existing runtime warning wrapper, fix tests around missing placeholders, and add the required changelog entry.
