Keep open: this PR targets a real linked bug and is the active implementation candidate, but it is not merge-ready because the patch has correctness issues and no contributor real-behavior proof.

## What This Changes

The PR adds an `agentDir` warning path to bootstrap resolution, passes `agentDir` into one embedded-runner call, and adds tests for ignored bootstrap files under `agentDir`.

## Best Possible Solution

Land a rebased warning-only diagnostic that checks the effective workspace file first, preserves existing warning dedupe, threads the effective `agentDir` through all resolver callers, and updates tests to match missing-placeholder behavior.

## Reproduction Assessment

Yes, by source inspection. Current main resolves bootstrap files from `workspaceDir` only, and the PR helper warns on `agentDir` files before checking whether the workspace copy is the effective loaded file.

## Solution Assessment

No, not as currently patched. Warning-only is the narrow maintainable direction, but this implementation false-warns for valid workspace files, has incompatible tests, and covers only one resolver caller.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip warnings when the workspace file is loaded:** `src/agents/bootstrap-files.ts:275-284`
  - body: The helper warns as soon as `agentDir/<name>` exists, but when `workspaceDir/<name>` also exists that workspace file is the effective bootstrap input. This creates a false warning for correctly configured workspaces; check the workspace path or effective file set before calling `params.warn`.
  - confidence: 0.94
- **[P2] Assert missing placeholders instead of absent bootstrap entries:** `src/agents/bootstrap-files.agentdir-warning.test.ts:38`
  - body: `loadWorkspaceBootstrapFiles` returns missing placeholders for standard bootstrap filenames when those workspace files are absent. These assertions expect `SOUL.md` and `AGENTS.md` entries to be absent, so the tests fail even though `agentDir` content was not loaded; assert `missing`/content state instead.
  - confidence: 0.91
- **[P2] Pass agentDir through every bootstrap resolver path:** `src/agents/pi-embedded-runner/run/attempt.ts:776`
  - body: This only forwards `agentDir` from one embedded-runner call. CLI preparation, compaction, and the embedded preloaded/fallback paths also resolve bootstrap context and will still silently ignore `agentDir` bootstrap files unless every effective caller supplies the agent directory.
  - confidence: 0.84
