Full review comments:

- **[P2] Skip warnings when the workspace file is loaded:** `src/agents/bootstrap-files.ts:275-284`
  - body: The helper warns as soon as `agentDir/<name>` exists, but when `workspaceDir/<name>` also exists that workspace file is the effective bootstrap input. This creates a misleading warning for correctly configured workspaces; check the workspace path or effective file set before calling `params.warn`.
  - confidence: 0.94
- **[P2] Assert missing placeholders instead of absent entries:** `src/agents/bootstrap-files.agentdir-warning.test.ts:38`
  - body: `loadWorkspaceBootstrapFiles` returns missing placeholders for standard bootstrap filenames when those workspace files are absent. These assertions expect `SOUL.md` and `AGENTS.md` entries to be absent, so the tests fail even though `agentDir` content was not loaded; assert the `missing` and content state instead.
  - confidence: 0.91
- **[P2] Thread agentDir through every resolver path:** `src/agents/pi-embedded-runner/run/attempt.ts:773`
  - body: This only forwards `agentDir` from one embedded-runner call. CLI preparation, compaction, embedded preloaded/fallback paths, and command prompt assembly also resolve bootstrap context and will still silently ignore `agentDir` bootstrap files unless every effective caller supplies the agent directory.
  - confidence: 0.84
