## Summary

Keep open. The requested warning addresses a real workspace/agentDir confusion, but the PR currently emits false warnings when the workspace file is actually loaded, adds tests that contradict current missing-placeholder behavior, and needs the required changelog entry before merge.

## What This Changes

The PR adds agentDir bootstrap-file warning checks to the bootstrap resolver, forwards agentDir from embedded attempts, and adds a new test file for the warning behavior.

## Best Possible Solution

Keep the workspace-only bootstrap contract and add a warning-only diagnostic that fires only for recognized bootstrap filenames in agentDir when the corresponding effective workspace file is absent, with corrected tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: put SOUL.md in both workspaceDir and agentDir and the PR helper still warns; put SOUL.md only in agentDir and current main does not load that content, instead returning the normal workspace missing placeholder.

## Solution Assessment

No, not as currently patched. A warning-only diagnostic is the narrowest backward-compatible direction for the silent-ignore behavior, but this implementation needs the workspace guard, test expectation fixes, and changelog before it is mergeable.
