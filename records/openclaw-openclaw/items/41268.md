## Summary

Keep open: current main still has no warning for agent-level model overrides, so the central UX improvement remains useful, but this branch is stale against the current config-set pipeline and has merge-blocking review issues.

## What This Changes

The PR adds CLI warning output and tests for `openclaw config set` when updating `agents.defaults.model` while configured agents have explicit model overrides.

## Best Possible Solution

Add one sanitized advisory in the shared non-dry-run `runConfigOperations` set path after a successful write, keyed off parsed operation paths and covered by focused CLI tests.

## Reproduction Assessment

Yes, from source inspection rather than a live run: current main resolves an explicit agent model before the default and `config set` emits only the generic restart text after writing.

## Solution Assessment

No. The warning is the right narrow UX direction, but this patch needs to target the current shared config operation pipeline and sanitize printed config values before it is merge-ready.
