Keep open. The warning is still a useful CLI diagnostic, but the PR branch is stale/conflicting, adds the warning to an obsolete inline config-set path, still prints config-controlled values unsanitized, and lacks contributor real behavior proof.

## What This Changes

The PR adds a post-write `openclaw config set` warning for explicit `agents.list[].model` overrides when changing `agents.defaults.model`, plus config CLI tests.

## Best Possible Solution

Add a sanitized advisory from the shared successful non-dry-run `runConfigOperations` path for relevant `config set` operations while preserving documented per-agent model precedence.

## Reproduction Assessment

Yes, by source inspection. Current `main` resolves `agents.list[].model` before `agents.defaults.model`, and `config set` currently reports only the shared generic restart message after writing.

## Solution Assessment

No, not as currently patched. A warning is the right narrow direction, but it needs to run from `runConfigOperations` and sanitize all config-derived terminal output.
