## Best Possible Solution

Add a sanitized advisory from the shared successful non-dry-run `runConfigOperations` set path for default-model writes, preserving documented per-agent model precedence and leaving the broader `models set` behavior to its canonical issue.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: `config set agents.defaults.model.primary` currently emits only the generic restart message while effective model resolution still prefers an explicit `agents.list[].model`. I did not run the CLI because this review is read-only.

## Solution Assessment

No, not as currently patched. A warning is the right narrow direction, but it should be rebased into the current shared config mutation path and sanitize all config-derived terminal output.
