## Summary

Keep open. The underlying diagnostic gap is still present on current main, but this PR patches an older config writer path, lacks contributor after-fix real CLI proof, and introduces unsanitized warning output, so it needs branch updates before merge rather than cleanup closure.

## What This Changes

The PR adds a post-write `openclaw config set` warning for default-model changes when agents in `agents.list[]` have explicit model overrides, plus focused CLI tests.

## Best Possible Solution

Rebase the warning into the shared non-dry-run config operation path, sanitize displayed values, keep per-agent precedence unchanged, coordinate with the active model-shadowing diagnostics, and add focused tests plus redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection: current main resolves `agents.list[].model` before `agents.defaults.model`, while `config set` currently prints only the generic restart output from `runConfigOperations`. I did not run a live CLI command in this read-only review.

## Solution Assessment

No. The warning direction is reasonable, but this branch should move the check into `runConfigOperations`, sanitize terminal output, and coordinate with adjacent model-warning diagnostics before merge.
