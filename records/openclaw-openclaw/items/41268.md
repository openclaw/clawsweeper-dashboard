## Summary

Keep open: the diagnostic is useful and current main still lacks it, but this branch is stale/conflicting, wires the warning into an obsolete config-set path, lacks real CLI proof, and prints config-controlled values without the existing terminal sanitizer.

## What This Changes

The PR adds a post-write `openclaw config set` warning for `agents.defaults.model` changes when `agents.list[]` entries have explicit model overrides, plus focused CLI tests.

## Best Possible Solution

Rebase the diagnostic into `runConfigOperations` after successful non-dry-run writes, sanitize displayed config values, preserve documented per-agent precedence, and add focused tests plus redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main resolves an explicit agent model before `agents.defaults.model`, while `config set` only emits the generic restart message, so a per-agent override can shadow a changed default without a CLI warning.

## Solution Assessment

No. The warning direction is reasonable, but this branch should be rebased into the shared config operation path and sanitize warning values before merge.
