## Summary

Keep open. The config-set diagnostic is still useful, but this PR branch is stale relative to current main's shared config mutation path, lacks after-fix real CLI proof, and prints config-controlled warning values without the existing terminal sanitizer.

## What This Changes

The PR adds a post-write `openclaw config set` warning when `agents.defaults.model` is changed while `agents.list[]` entries have explicit model overrides.

## Best Possible Solution

Implement a sanitized advisory from the shared successful non-dry-run config mutation path, preserve documented per-agent precedence, and keep broader `models set` reconciliation under the linked follow-up issue.

## Reproduction Assessment

Yes, by source inspection. Current main resolves `agents.list[].model` before `agents.defaults.model`, and `config set` currently reports only the shared generic success message after writing.

## Solution Assessment

No, not as currently patched. A warning is the right narrow direction, but it needs to run from `runConfigOperations` and sanitize all config-derived terminal output.
