## Summary

Keep this PR open: the requested warning is still useful and not implemented on current main, but the branch is stale against the current config mutation pipeline, lacks required real CLI proof, and introduces unsanitized terminal output for config-controlled values.

## What This Changes

The PR adds a post-write CLI warning and tests for `openclaw config set agents.defaults.model*` when configured agents have explicit `agents.list[].model` overrides.

## Best Possible Solution

Add one sanitized post-write advisory in the shared non-dry-run `runConfigOperations` set path, keyed off parsed default-model operation paths and covered by focused CLI tests.

## Reproduction Assessment

Yes, by source inspection: current main resolves `agents.list[].model` before `agents.defaults.model`, and the active `config set` path logs only the generic restart message after writing. I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

No. The warning is the right narrow UX direction, but this patch should move into the current shared operation pipeline and sanitize displayed config values before merge.
