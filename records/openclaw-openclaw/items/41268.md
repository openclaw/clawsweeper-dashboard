## Summary

This PR should stay open: it targets a real CLI confusion, but the branch is stale against the active config mutation path and still needs real CLI proof before merge.

## What This Changes

The PR adds a post-write `openclaw config set` warning for `agents.defaults.model` changes when `agents.list[]` entries have explicit model overrides, plus focused CLI tests.

## Best Possible Solution

Keep the warning direction, but integrate it into the current shared config operation path, sanitize displayed values, and leave broader model-shadowing policy work to the related issue cluster.

## Reproduction Assessment

Yes, by source inspection. Current main resolves `agents.list[].model` before `agents.defaults.model`, while `config set` only prints the generic restart message after writing, so a changed default can be shadowed without a warning.

## Solution Assessment

No, not as currently patched. The warning is a reasonable narrow direction, but this branch should be rebased into `runConfigOperations` and sanitize displayed config values before merge.
