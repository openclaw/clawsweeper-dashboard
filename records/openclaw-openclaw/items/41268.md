## Summary

Keep open. The requested CLI warning is useful and still missing from current main, but this branch is not merge-ready because it targets the old inline `config set` writer, lacks after-fix real CLI proof, and prints config-controlled values without sanitization.

## What This Changes

The PR adds a CLI warning and tests for `openclaw config set agents.defaults.model*` when configured agents have explicit model overrides.

## Best Possible Solution

Add one sanitized post-write advisory in the shared non-dry-run `runConfigOperations` set path, keyed off parsed default-model operation paths and covered by focused CLI tests.

## Reproduction Assessment

Yes, by source inspection: current main resolves `agents.list[].model` before `agents.defaults.model`, and the active `config set` path logs only the generic restart message after writing. I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

No. The warning is the right narrow UX direction, but this patch should be moved into the current shared operation pipeline and sanitize displayed config values before merge.
