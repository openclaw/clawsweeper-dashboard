## What This Changes

The PR adds a CLI warning after default-model `config set` changes when configured agents have explicit model overrides, plus focused config CLI tests.

## Best Possible Solution

Keep the warning direction, but implement it on the shared current-main config operation path, sanitize displayed values, and coordinate wording with the broader model-shadowing issue.

## Reproduction Assessment

Yes, by source inspection. Current main resolves explicit `agents.list[].model` before `agents.defaults.model`, while `config set` currently emits only the generic restart message after writing.

## Solution Assessment

No, not as currently patched. A warning is a narrow maintainable direction, but this branch should be rebased into `runConfigOperations` and sanitize config-controlled output before merge.
