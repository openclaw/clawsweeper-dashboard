## What This Changes

The PR adds a post-write `openclaw config set` warning and tests for agents with explicit `agents.list[].model` overrides when changing `agents.defaults.model`.

## Best Possible Solution

Add a sanitized advisory from the shared successful non-dry-run `runConfigOperations` path for relevant `config set` operations while preserving documented per-agent model precedence.

## Reproduction Assessment

Yes. Source inspection shows current main resolves explicit `agents.list[].model` before `agents.defaults.model`, while `config set` currently emits only the generic restart message after a successful write.

## Solution Assessment

No, not as currently patched. A warning is the right narrow direction, but it should be rebased into the shared config mutation path and sanitize all config-derived terminal output.
