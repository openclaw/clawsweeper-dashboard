## Summary

Keep open. The warning direction is useful and current main still has the shadowing behavior, but this branch is stale against the shared config mutation path, prints config-derived values without the terminal sanitizer, and still lacks real behavior proof.

## What This Changes

The PR adds a CLI warning and focused tests for agent-level model overrides after setting `agents.defaults.model` paths.

## Best Possible Solution

Implement a sanitized advisory from the shared successful `runConfigOperations` path, preserve documented per-agent precedence, and leave broader `models set` reconciliation to the linked model-shadowing issue.

## Reproduction Assessment

Yes, by source inspection. Current main still lets explicit `agents.list[].model` take precedence over `agents.defaults.model`, and `config set` currently reports only the generic restart message after writing.

## Solution Assessment

No, not as currently patched. A warning is a reasonable narrow UX fix, but it needs to run from the shared config mutation path and sanitize all config-derived terminal output.
