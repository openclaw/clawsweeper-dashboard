## Summary

Keep open: the requested warning is still useful and current main still lacks it, but this branch is stale against the active config-set pipeline and needs real behavior proof before merge.

## What This Changes

The PR adds CLI warning output and tests when `openclaw config set` updates `agents.defaults.model` while configured agents have explicit model overrides.

## Best Possible Solution

Implement the override warning once in the shared non-dry-run `runConfigOperations` set path after a successful write, using parsed operation paths, the final normalized config, sanitized output, and focused tests for single, batch, and bracket paths.

## Reproduction Assessment

Yes, from source inspection. Configure an agent with `agents.list[].model`, run `openclaw config set agents.defaults.model.primary <model>`, and current main will update the default while per-agent model resolution still takes priority and only the generic restart message is emitted.

## Solution Assessment

No. The warning is the right narrow UX direction, but this patch needs to target the current shared config-set operation pipeline and sanitize printed agent/model fields before it is merge-ready.
