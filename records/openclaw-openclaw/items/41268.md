## Summary

Keep open: the warning is useful, but this PR is not safely mergeable against current main because it patches an older config-set writer, lacks after-fix real CLI proof, and prints config-controlled values without sanitization.

## What This Changes

The PR adds a CLI warning and tests for `openclaw config set agents.defaults.model*` when `agents.list[]` entries have explicit model overrides.

## Best Possible Solution

Add one sanitized post-write advisory in the shared non-dry-run `runConfigOperations` set path, keyed off parsed default-model operation paths and covered by focused CLI tests plus real CLI proof.

## Reproduction Assessment

Yes, by source inspection: current main resolves agent-specific models before defaults and `config set` logs only the generic restart message. I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

No. The warning direction is narrow and useful, but this patch should move into the current shared operation pipeline and sanitize config-controlled values before merge.

Overall correctness: patch is incorrect
