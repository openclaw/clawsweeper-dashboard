## Summary

Keep open. The PR addresses a real config/model shadowing diagnostic gap, but the branch is stale against current main’s shared config mutation path, lacks after-fix real CLI proof, and prints config-derived values without terminal sanitization.

## What This Changes

The branch adds a post-write `openclaw config set` warning for default model paths when `agents.list[]` entries define explicit model overrides, plus focused warning/no-warning tests.

## Best Possible Solution

Rebase the warning into the shared non-dry-run config operation path, sanitize displayed values, preserve documented per-agent precedence, coordinate with the related model-shadowing diagnostics, and add focused tests plus redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.model.primary` to a new model while an `agents.list[]` entry has its own model; current main still resolves the explicit agent model first and `config set` only prints the generic restart message.

## Solution Assessment

No. The warning direction is reasonable, but this branch should move the check into `runConfigOperations`, sanitize terminal output, and coordinate with the adjacent model-warning work before merge.
