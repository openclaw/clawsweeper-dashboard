## Summary

Keep open. The warning direction is useful, but the branch is not ready against current main because it patches an obsolete inline config-set write path, prints config-controlled values without the terminal sanitizer, and still lacks after-fix real CLI proof.

## What This Changes

This PR adds a CLI warning after `openclaw config set agents.defaults.model...` when configured agents have explicit model overrides, plus focused config CLI tests.

## Best Possible Solution

Add a sanitized post-write advisory in `runConfigOperations` for successful default-model `config set` operations, while coordinating the broader `models set` shadowing UX with https://github.com/openclaw/openclaw/issues/76128.

## Reproduction Assessment

Yes, by source inspection. Current main keeps per-agent model precedence and `config set` emits only the generic restart message after writing, so a default-model update can be shadowed without a warning.

## Solution Assessment

No, not as currently patched. The warning is a reasonable narrow direction, but it needs to be rebased into the shared config operation path and sanitize displayed config values before merge.
