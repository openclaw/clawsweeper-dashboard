## Best Possible Solution

Keep the warning behavior, but implement it once in the shared non-dry-run set operation path after a successful `replaceConfigFile`, using parsed operation paths and the final config snapshot so value, bracket, builder, and batch flows stay consistent without changing model resolution, schema, or gateway restart semantics.
