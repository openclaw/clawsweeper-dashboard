## Summary

Keep this PR open: it addresses a real model-shadowing confusion case, but the branch is stale against the current shared `config set` implementation and still has a concrete terminal-output safety issue.

## What This Changes

The branch adds a post-write `openclaw config set` warning when changing `agents.defaults.model` while agents have explicit model overrides, plus focused config CLI tests.

## Best Possible Solution

Add a sanitized advisory from the shared `runConfigOperations` path for successful default-model config changes, while leaving broader `models set` reconciliation to the linked model-shadowing issue.

## Reproduction Assessment

Yes, by source inspection. Current main still gives explicit per-agent models precedence over `agents.defaults.model`, and the shared `config set` path only prints the generic restart message after writing.

## Solution Assessment

No, not as currently patched. The warning is a reasonable narrow UX direction, but it needs to run from the shared config mutation path and sanitize all config-derived terminal output.
