## Summary

Keep this PR open: it is a focused and useful WebChat exec-approval fix with supplied live-output proof, but the diff currently catches WebChat diagnostics/direct-followup exec approvals and would bypass their existing direct async report path.

## What This Changes

The PR changes WebChat gateway exec approvals to wait inline and return approved command output or denial from the original exec tool call while preserving async behavior for other channels.

## Best Possible Solution

Limit inline waiting to normal WebChat agent exec tool calls, and keep diagnostics/direct approval followups on the existing async direct-report path.

## Reproduction Assessment

Yes for the review finding by source inspection: `/diagnostics` passes `messageProvider: webchat` plus `approvalFollowupMode: direct`, and the PR's new WebChat-only inline branch runs before direct followup target creation. I did not run a live WebChat/Control UI repro in this read-only review.

## Solution Assessment

No, not quite: the inline approach is right for normal WebChat exec turns, but it needs a guard so direct diagnostics followups keep their established approval/report behavior.
