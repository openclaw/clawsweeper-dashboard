## Reproduction Assessment

Yes for part of the issue from source: current iOS code can override parsed non-realtime Talk routing back to realtime relay when the active provider is OpenAI. The full early-session-close symptom still needs a live iOS/provider repro.

## Solution Assessment

No current fix is proven on main. The best path is a bounded iOS/Gateway Talk routing repair with regression tests, plus live provider proof for the immediate close behavior before changing relay lifecycle policy.
