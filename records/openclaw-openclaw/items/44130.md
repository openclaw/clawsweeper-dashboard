## Summary

Keep open. Current main still renders the TUI chat log as an append/prune container with no transcript viewport state, scroll anchor, auto-follow policy, paging, wheel handling, jump-to-latest action, or TUI auto-scroll setting. The related implementation PR #51128 is still open and unmerged, so it is not proof that the reported behavior is fixed on main.

## What This Changes

The issue asks for stable TUI transcript scrolling during sends and streaming, including preserving reading position, reducing disruptive auto-scroll jumps, and possibly adding auto-scroll controls or jump-to-latest behavior.

## Best Possible Solution

Add a TUI transcript viewport layer owned by the TUI path: preserve user scroll anchors across streaming and history rebuilds, auto-follow only when already at the latest output, provide explicit paging or wheel navigation and a jump-to-latest affordance, and keep the separate Control UI/WebChat auto-scroll request tracked by #7648.
