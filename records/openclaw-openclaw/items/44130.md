## Summary

Keep open: current main still lacks TUI transcript viewport or anchor-aware follow behavior, and a member explicitly confirmed the shipped WebChat auto-scroll selector does not cover terminal TUI scrolling.

## What This Changes

Report asks for `openclaw tui` to stop disruptive transcript scroll jumps and preserve a stable reading position while messages are sent or streamed.

## Best Possible Solution

Implement a coherent TUI transcript viewport and anchor-aware follow policy that preserves reading position across sends, streaming growth, history reloads, and long transcript redraws.

## Reproduction Assessment

Yes for source-level reproduction: current main mutates and rebuilds `ChatLog` on send, streaming, final response, and history reload without viewport or anchor state. I did not run a live terminal recording, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes, keeping this open is the right outcome. The maintainable path is a reviewed TUI viewport/follow-policy fix or corrected version of https://github.com/openclaw/openclaw/pull/51128, not treating the shipped WebChat selector as covering this terminal surface.
