## Summary

Keep open: current main still lacks a TUI transcript viewport, scroll anchor, at-bottom policy, or scroll input path, and the related TUI scrolling PR remains open rather than shipped.

## What This Changes

Requested TUI behavior is to keep the transcript viewport stable during sends and streaming, with controlled auto-follow or scroll affordances.

## Best Possible Solution

Add a TUI-owned transcript viewport around `ChatLog` that tracks user position, preserves anchors across renders and history reloads, auto-follows only under the chosen policy, and provides explicit scroll or jump-to-latest controls while keeping the separate WebChat request scoped to #7648.

## Reproduction Assessment

Source-reproducible, but not live-reproduced: current main has no TUI viewport state, scroll input path, or anchor preservation around the ChatLog update and history reload paths. A live terminal run would still be useful to characterize the visible jump precisely.

## Solution Assessment

Yes for keeping this open: the narrow maintainable fix belongs in the TUI ChatLog/root/event/session path, not in the web chat scroll code. Maintainers should review #51128 first and either land it as part of the fix or supersede it with a fuller transcript viewport policy.
