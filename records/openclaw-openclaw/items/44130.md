## Summary

Keep open: current main still lacks TUI transcript viewport or anchor-aware follow behavior, a member explicitly confirmed the shipped WebChat fix does not cover terminal TUI scrolling, and the related TUI scrolling PR remains open rather than merged and proven.

## What This Changes

The issue asks for `openclaw tui` to preserve a stable chat viewport instead of jumpy auto-scroll behavior while sending, streaming, and reading messages.

## Best Possible Solution

Land a coherent TUI transcript viewport and anchor-aware follow policy that preserves reading position across sends, streaming growth, history reloads, and long redraws, with real terminal proof.

## Reproduction Assessment

Yes for source-level reproduction: current main mutates and redraws `ChatLog` on send, streaming, final response, and history reload without carrying viewport or anchor state. I did not run a live terminal recording, so this is source-reproducible rather than fully reproduced.

## Solution Assessment
