## Summary

Keep open: current main still lacks a TUI transcript viewport or anchor-aware follow policy, and a member explicitly confirmed the shipped Control UI/WebChat selector does not cover terminal TUI behavior. The open TUI scroll PR at https://github.com/openclaw/openclaw/pull/51128 is a plausible path, but it still needs maintainer UX review, rebase work, and real terminal proof rather than cleanup closure.

## What This Changes

The issue reports disruptive viewport jumps in `openclaw tui` during send, streaming, and render updates, and asks for stable TUI transcript anchoring or controlled auto-follow behavior.

## Best Possible Solution

Implement a coherent TUI transcript viewport and anchor-aware follow policy that preserves reading position across sends, streaming growth, history reloads, and long transcript redraws, coordinated with the open TUI scroll PR and the narrower scrollback-clearing issue.

## Reproduction Assessment

Yes for source-level reproduction: current main mutates `ChatLog` on send, streaming, final response, and history reload without viewport or anchor state. I did not run a live terminal recording, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes, keeping this open is the right outcome. The maintainable path is a reviewed TUI viewport/follow-policy fix or a corrected version of https://github.com/openclaw/openclaw/pull/51128, not treating the shipped WebChat selector as covering this terminal surface.
