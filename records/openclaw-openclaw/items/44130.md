## Summary

Keep open: current main and the latest shipped release still lack TUI transcript viewport or anchor-aware follow behavior, and a member explicitly confirmed the merged WebChat auto-scroll selector does not cover terminal TUI scrolling.

## What This Changes

The issue asks for stable, predictable TUI chat scrolling so sends, streaming output, and redraws do not disrupt the user’s reading position.

## Best Possible Solution

Land a coherent TUI transcript viewport and anchor-aware follow policy that preserves reading position across sends, streaming growth, history reloads, and long redraws, with real terminal proof and coordination with the open TUI scroll PR and scrollback issue.

## Reproduction Assessment

Yes for source-level reproduction: current main mutates and redraws ChatLog on send, streaming, final response, and history reload without carrying viewport or anchor state. I did not run a live terminal recording, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes, keeping this open is the right outcome. The maintainable path is a reviewed TUI viewport/follow-policy fix, likely by correcting or replacing https://github.com/openclaw/openclaw/pull/51128, not treating the shipped WebChat selector as covering this terminal surface.
