## Summary

Keep open: current main still lacks a TUI transcript viewport, scroll-anchor state, and at-bottom follow policy, while the related TUI scrolling PR remains open rather than shipped.

## What This Changes

The issue reports disruptive TUI transcript scroll jumping during normal `openclaw tui` chat use and asks for stable viewport or controlled auto-follow behavior.

## Best Possible Solution

Add a TUI-owned transcript viewport around `ChatLog` that preserves anchors across streaming renders and history reloads, follows only under an explicit at-bottom policy, and provides clear scroll or jump-to-latest controls.

## Reproduction Assessment

Yes from source inspection: the send, streaming, final, and history reload paths mutate `ChatLog` without viewport or anchor state. I did not live-run the terminal UI, so the exact visible jump pattern is not characterized here.

## Solution Assessment

Yes for keeping this open: the narrow maintainable fix belongs in the TUI ChatLog/root/event/session path, not in the separate WebChat request. Maintainers should review #51128 first and either land it or supersede it with the fuller transcript viewport behavior.
