## Summary

Keep this PR open: the security direction is right, but the patch only updates the ACP-core fallback redaction list while OpenClaw’s configured ACP runtime path uses the app logging redactor and bypasses that fallback.

## What This Changes

The PR adds an ACP-core URL-userinfo redaction regex and a regression test for password-bearing and token-only URL credentials.

## Best Possible Solution

Apply URL-userinfo redaction at the canonical OpenClaw logging/ACP redaction boundary, or ensure the ACP fallback still runs after configured host redaction, with coverage for both direct ACP-core and configured OpenClaw imports.

## Reproduction Assessment

Yes, this is source-reproducible: the configured ACP wrapper installs the app redactor, and ACP-core returns that redactor before running the newly patched fallback patterns. I did not run a live ACP backend session in this read-only review.

## Solution Assessment

No. The regex is a plausible fallback improvement, but the best fix must cover the configured OpenClaw redaction stack and prove the actual formatter path that operator-visible surfaces use.
