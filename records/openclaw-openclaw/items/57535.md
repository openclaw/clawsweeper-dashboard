## Summary

Keep open. Current `main` supports loading skills from multiple local roots and extra directories, and it documents third-party skills as untrusted, but it does not implement the requested mandatory confirmation/provenance check for non-ClawHub/local skill sources. The only concrete warning I found is an install-time warning for some non-bundled installer metadata paths, and that code still treats `openclaw-extra` and `openclaw-managed` as trusted for that warning path.

## Evidence

- **Local and extra-dir skills are first-class load sources:** Workspace skill loading explicitly scans configured `skills.load.extraDirs`, managed skills, personal/project agent skills, and workspace skills. This is the current load path for local sources, and there is no confirmation gate in this code path.
