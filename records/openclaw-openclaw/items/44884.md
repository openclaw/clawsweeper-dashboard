  - body: `createRequestRateLimiter()` is now always constructed even when `gateway.requestRateLimit` is omitted, so existing non-loopback HTTP and WebSocket clients can start receiving 429s after upgrade.   - confidence: 0.88

  - body: Replacing the old loopback fallback with a thrown startup error makes a gateway that previously recovered by binding broadly fail to start instead.   - confidence: 0.82
- **[P2] Thread the audit logger through real auth callers:** `src/gateway/server-http.ts:764`
