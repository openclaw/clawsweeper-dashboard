## Summary

Keep open: this Gateway security PR is not implemented on current main and still has blocking config-schema, auth-audit, and real-runtime proof gaps; the related limiter PR covers only one subset and is also still open.

## What This Changes

The PR adds Gateway public-network hardening across IP allow/block lists, per-IP request limiting, TLS and credential warnings, auth audit logging, HSTS behavior, and bind fallback handling.

## Best Possible Solution

Rebase or replace this with one coherent Gateway hardening design that validates the public config surface, uses proxy-aware client-IP contracts, completes audit logging, documents the operator workflow, and includes redacted real-runtime proof.

## Solution Assessment
