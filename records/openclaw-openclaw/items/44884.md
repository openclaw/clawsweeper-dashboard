- **[P1] Add `terminatedUpstream` to the strict TLS schema:** `src/config/types.gateway.ts:16-17`
  - body: The PR tells operators to set `gateway.tls.terminatedUpstream`, but the current strict TLS schema only accepts `enabled`, `autoGenerate`, `certPath`, `keyPath`, and `caPath`. Users following the new guidance can hit config validation failures or an undiscoverable setting until schema/help/docs are aligned.
  - confidence: 0.91
- **[P1] Add the ACL and rate-limit keys to the config contract:** `src/config/types.gateway.ts:443-450`
  - body: These new Gateway keys are advertised as operator config, but the strict Gateway zod schema still does not include `ipAllowlist`, `ipBlocklist`, or `requestRateLimit`. Config files using the new fields can be rejected or remain undiscoverable unless schema/help/docs are updated with the runtime change.
  - confidence: 0.92
- **[P1] Do not turn request throttling on for existing gateways:** `src/gateway/server.impl.ts:530-531`
  - body: `startGatewayServer` now creates a limiter unconditionally, so deployments with no explicit setting get the new 120 req/min/IP behavior after upgrade. That can produce 429s for existing HTTP or WebSocket clients and needs an explicit compatibility policy, opt-in default, or tested migration/upgrade proof.
  - confidence: 0.9
- **[P1] Keep strict bind failures out of the shared resolver:** `src/gateway/net.ts:253-285`
  - body: `resolveGatewayBindHost` is also used by doctor, CLI preflight, and display paths, not only final Gateway startup. Throwing from the shared resolver can turn diagnostic/status flows into hard errors; keep the resolver tolerant or move fail-closed behavior to the startup boundary and update every caller deliberately.
  - confidence: 0.86
- **[P2] Thread the audit logger through auth callers:** `src/gateway/auth.ts:85-94`
  - body: The new `auth_success` and `auth_failure` events are only emitted when callers pass `auditLogger`, but the normal HTTP/WS auth call sites still omit it. As written, the audit log records IP/rate-limit events but misses the core credential outcomes the PR advertises.
  - confidence: 0.83
