Full review comments:

- **[P1] Add the Gateway hardening keys to the config contract:** `src/config/types.gateway.ts:443-450`
  - body: This PR reads `gateway.ipAllowlist`, `gateway.ipBlocklist`, `gateway.requestRateLimit`, and `gateway.tls.terminatedUpstream`, but the branch only adds TypeScript types. The strict Zod schema, config help/labels, and public configuration docs still do not know these keys, so users following the PR body can end up with settings that fail validation or are undiscoverable via the normal config surfaces.
  - confidence: 0.95
- **[P1] Do not enable request throttling for every upgraded gateway:** `src/gateway/server.impl.ts:531`
  - body: `createRequestRateLimiter` is called even when `gateway.requestRateLimit` is unset, and the limiter supplies a 120 requests/min/IP default. That silently changes existing public Gateway behavior after upgrade and can return 429s to current HTTP or WebSocket clients without an explicit operator opt-in, migration, or maintainer-approved default policy.
  - confidence: 0.93
- **[P1] Keep fail-closed bind errors out of the shared resolver:** `src/gateway/net.ts:253-255`
  - body: Throwing from `resolveGatewayBindHost` makes every caller strict, including doctor/security diagnostics, CLI status/run health-host resolution, and network-discovery display. Current main already rejects unsafe loopback/custom fallback in `resolveGatewayRuntimeConfig`, so the safer shape is to keep the shared resolver diagnostic-friendly or split a strict startup helper instead of turning all callers into runtime error paths.
  - confidence: 0.88
- **[P2] Thread the audit logger into auth verification call sites:** `src/gateway/auth.ts:86-94`
  - body: Adding `auditLogger` to the auth parameter type is not enough: the HTTP readiness/auth stages, OpenAI/OpenResponses/tools/canvas/plugin auth calls, and WebSocket auth context do not pass the new logger into `authorizeGatewayConnect`. As written, the advertised `auth_success` and `auth_failure` JSONL trail remains incomplete for normal credential verification paths.
  - confidence: 0.86
