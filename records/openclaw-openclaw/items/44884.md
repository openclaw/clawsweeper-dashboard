  - confidence: 0.9

  - body: `createRequestRateLimiter(cfgAtStart.gateway?.requestRateLimit)` is always constructed, so existing non-loopback deployments that omit this new config still get the default 120 req/min/IP limiter after upgrade.   - confidence: 0.86

  - body: Changing loopback, tailnet, and custom bind fallback from returning `0.0.0.0` to throwing startup errors can break existing deployments that currently recover from bind failures. Please keep the shipped fallback behavior as the default or gate the fail-closed behavior behind an explicit, documented, tested upgrade path that maintainers approve.
  - confidence: 0.82
