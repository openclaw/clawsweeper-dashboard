Adds Gateway public-network hardening with IP allow/block lists, per-IP request limits, credential/TLS warnings, auth audit JSONL logging, auto-HSTS, and stricter bind fallback handling.

## Best Possible Solution

Keep the work open for a security-focused rebase that aligns the public config schema/help/docs, threads auth audit logging through normal HTTP and WebSocket auth paths, and reconciles the overlapping request-limiter design before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough to reproduce the review findings: the strict config schema lacks the new keys, and normal HTTP/WS auth callers do not pass an audit logger into the new auth audit hooks.

## Solution Assessment

No. The security direction is useful, but this branch is not the best merge shape until the config contract is complete, auth audit events are emitted through normal auth paths, and maintainer review settles the overlap with related request-limiter work.
