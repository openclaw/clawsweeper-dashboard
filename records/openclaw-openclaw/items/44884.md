  - body: The PR adds operator-facing gateway settings here, but the strict gateway Zod schema does not include `ipAllowlist`, `ipBlocklist`, `requestRateLimit`, or `tls.terminatedUpstream`.   - confidence: 0.93

  - body: Adding `auditLogger` to the authorization API is not enough while the real HTTP/WebSocket auth callers do not pass it through.   - confidence: 0.86
- **[P1] Do not enable request throttling by default without upgrade proof:** `src/gateway/server.impl.ts:530-531`
  - body: `createRequestRateLimiter` is constructed unconditionally, so existing non-loopback gateway traffic starts receiving 429s at the new default limit even when the operator did not opt in. For a public gateway hardening change, keep the existing behavior by default or provide a validated config/migration/docs path plus real upgrade proof for the new default.
  - confidence: 0.82
