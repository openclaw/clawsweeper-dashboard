- **[P1] Add the Gateway hardening keys to the config schema:** `src/config/types.gateway.ts:443-450`
  - body: This PR reads new `gateway.ipAllowlist`, `gateway.ipBlocklist`, and `gateway.requestRateLimit` settings from runtime config, but only adds TypeScript types. The Gateway zod config object is strict and the schema/help/docs do not define these keys, so users following the PR body can hit unknown-key validation or undiscoverable config.
  - confidence: 0.95
- **[P1] Keep request throttling opt-in until defaults are approved:** `src/gateway/server.impl.ts:531`
  - body: `createRequestRateLimiter` is called even when `gateway.requestRateLimit` is unset, and the limiter supplies a 120 requests/min/IP default. That silently changes existing public Gateway behavior after upgrade and can return 429s to current HTTP or WebSocket clients without explicit operator opt-in, migration, or maintainer-approved default policy.
  - confidence: 0.93
- **[P1] Keep strict bind failures out of the shared resolver:** `src/gateway/net.ts:253-255`
  - body: Throwing from `resolveGatewayBindHost` makes every caller strict, including diagnostics/status/network-display style callers. Current main already rejects unsafe loopback/custom fallback in `resolveGatewayRuntimeConfig`, so the safer shape is to keep the shared resolver diagnostic-friendly or split a strict startup helper.
  - confidence: 0.88
- **[P2] Pass authAuditLogger through credential auth callers:** `src/gateway/auth.ts:86-94`
  - body: Adding `auditLogger` to the auth parameter type is not enough: the HTTP/plugin/canvas auth call sites still pass only the existing rate limiter. As written, the advertised `auth_success` and `auth_failure` JSONL trail remains incomplete for normal credential outcomes.
  - confidence: 0.9
- **[P1] Import the current net-policy IP helper:** `src/gateway/ip-access-control.ts:9`
  - body: The new IP ACL module imports `../shared/net/ip.js`, but current main moved these helpers behind `@openclaw/net-policy/ip`. After rebasing this stale branch, this import needs to use the current package path or the new module will not build against main.
  - confidence: 0.86
