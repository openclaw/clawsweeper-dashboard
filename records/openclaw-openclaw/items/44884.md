## Summary

Keep this PR open, but do not merge it as-is: current main does not already contain the proposed gateway hardening, and the branch still has concrete config-contract and auth-audit wiring blockers.

## What This Changes

The PR adds gateway public-network hardening across IP allow/block lists, request rate limiting, TLS and credential warnings, audit logging, HSTS behavior, bind fallback handling, and related tests.

## Best Possible Solution

Revise this PR or a replacement branch so the new gateway controls are fully registered in the config contract, documented, covered by regression tests, and aligned with #48336 before maintainer security review.

## Reproduction Assessment

Yes for the review blockers: the PR patch itself shows the new config keys only in TypeScript types and shows no auditLogger parameter being passed into auth verification call sites. Current main also lacks the proposed hardening, so this is not already implemented.

## Solution Assessment

No, not as-is. The direction is useful, but the branch needs config schema/help/docs/generated updates and complete audit-event wiring before it is the narrowest maintainable security hardening path.
