## Summary

Keep open. Current main does not implement the full gateway hardening proposal, and the PR still has merge-blocking gaps in config schema registration and auth audit coverage.

## What This Changes

The PR adds gateway public-network hardening: IP allow/block lists, request rate limiting, credential/TLS warnings, auth audit logging, auto-HSTS behavior, and stricter bind fallback handling.

## Best Possible Solution

Keep this PR open for maintainer security review, then either fix the schema/docs/audit blockers on the branch or split the hardening into smaller gateway PRs with changed-gate validation.

## Reproduction Assessment

Not applicable. This PR is a broad security hardening proposal rather than a single reported runtime bug; the blocking issues are reproducible by source inspection of the diff and current strict config schema.

## Solution Assessment

No. The direction is useful, but this is not yet the best mergeable implementation because new public config keys are not registered in the config schema/help/docs and advertised auth outcome audit events are not wired through normal auth paths.
