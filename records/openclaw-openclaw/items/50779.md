Keep open. Current main and latest release still drop direct tool-result image blocks in Control UI chat, and the open linked PR is the right active fix path but has not merged yet.

## What This Changes

The issue asks Control UI chat to render or clearly preview images returned by tools instead of showing only text such as `Read image file [image/png]`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/86852 or an equivalent narrow Control UI grouped-render fix with real behavior proof, then close this issue as implemented with the merge commit and release provenance.

## Reproduction Assessment

Yes, as a high-confidence source-level reproduction: current main emits and validates direct `{ type: "image", data, mimeType }` tool-result blocks, while the Control UI grouped renderer ignores that shape. I did not run a live Control UI session in this read-only review.

## Solution Assessment

Yes for the narrow bug boundary. The best fix is to preserve the existing image-rendering path and add direct `data` plus `mimeType` handling in `extractImages()`, while keeping broader inline media product decisions in https://github.com/openclaw/openclaw/issues/64607.
