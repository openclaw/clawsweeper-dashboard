## Summary

Keep open. Current main and the latest release still drop the direct tool-result image block shape in Control UI chat, and the open linked PR is the active fix candidate rather than merged implementation.

## What This Changes

The issue reports that Control UI chat does not visibly render image content returned by tools such as `read`, leaving only text like `Read image file [image/png]`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/86852 or an equivalent narrow grouped-render fix, then close this issue with merge and release provenance while keeping broader media work in https://github.com/openclaw/openclaw/issues/64607.

## Reproduction Assessment

Yes. Source inspection shows current main emits and validates direct `{ type: "image", data, mimeType }` tool-result blocks while the grouped renderer ignores that shape; I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes for the narrow bug boundary. The maintainable fix is to add direct `data` plus `mimeType` handling in `extractImages()` while leaving broader inline image/audio/video/web preview policy to the umbrella media issue.
