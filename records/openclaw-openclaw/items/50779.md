## Summary

Keep open. Current main still does not surface the direct read-tool image result shape in Control UI chat: tools can return `{ type: "image", data, mimeType }`, but the renderer only extracts image blocks with `source` or `url`, and live tool events flatten results to text. Open PR #52025 appears to target this bug, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests visible inline or preview rendering for image data returned by tools in Control UI chat.

## Best Possible Solution

Control UI should preserve structured image data from stored history and live tool-result updates, render direct `{ type: "image", data, mimeType }` blocks as safe inline previews or obvious thumbnails, and cover the managed-media/local-media paths with focused regression tests.

## Reproduction Assessment

Yes, by source-level reproduction. A current read-tool image result can contain `{ type: "image", data, mimeType }`, while the current Control UI extraction and live stream paths do not turn that shape into a visible preview.

## Solution Assessment

Yes for the issue direction: the narrow maintainable fix belongs in Control UI/tool-stream image handling with regression coverage. The open PR should be reviewed or replaced rather than closing this issue before a fix is merged.
