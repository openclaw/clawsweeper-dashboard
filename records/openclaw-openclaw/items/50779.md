Keep open. Current main still does not render direct read-tool image result blocks inline in Control UI chat, and open PR #52025 targets the same bug but is not merged.

## What This Changes

The issue requests inline or clearly clickable Control UI previews for images returned by tool output, especially read-tool image results.

## Best Possible Solution

Land or replace #52025 with a focused Control UI fix that preserves structured image data for stored and live tool results and adds regression coverage.

Yes, by source-level reproduction. Current read-tool image results can contain direct `{ type: "image", data, mimeType }` blocks, while the current Control UI extraction and live tool-stream paths do not convert that shape into a visible preview.

## Solution Assessment

Yes for the issue direction: the narrow maintainable fix belongs in Control UI/tool-result image handling with focused tests. Because #52025 is already open for this path, the issue should stay open until that PR lands or is replaced.
