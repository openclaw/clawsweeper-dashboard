## Summary

Keep open. Current main still has the reported Control UI gap: tool results can contain direct image blocks with `data` and `mimeType`, but the chat renderer only extracts `image` blocks with `source` or `url`, and live tool events rebuild results as text-only tool output. Open PR #52025 targets this exact path, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests inline rendering or an obvious preview for image results returned by tools in Control UI chat.

## Best Possible Solution

Review and land PR #52025, or replace it with a narrow Control UI fix that preserves structured image results for stored history and live tool-result updates with regression coverage.

## Reproduction Assessment

Yes, by source-level reproduction. Current main produces and normalizes direct tool image blocks, while the Control UI renderer/live stream paths do not turn that shape into a visible preview.

## Solution Assessment

Yes for the issue direction: the narrow maintainable fix belongs in Control UI tool-result image handling, with tests for both stored and live result paths. The active PR should be reviewed, landed, or replaced rather than closing this issue first.
