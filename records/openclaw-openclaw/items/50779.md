## Summary

Keep open. Current main and the latest release still do not extract direct `{type:"image", data, mimeType}` image blocks in Control UI chat, and the open closing PR remains the active fix candidate rather than merged implementation.

## What This Changes

The issue reports that Control UI chat records tool-returned image output but renders only text instead of an inline image or obvious preview.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/86852 or an equivalent narrow grouped-render fix, then close this issue with merge and release provenance while keeping broader inline media work in https://github.com/openclaw/openclaw/issues/64607.

## Reproduction Assessment

Yes, by source inspection. Current main emits and validates direct `{ type: "image", data, mimeType }` tool-result blocks, while the grouped renderer ignores that shape; I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes for the narrow bug boundary. The best fix is to add direct `data` plus `mimeType` handling in `extractImages()` while leaving broader image/audio/video/web preview work to the separate inline-media issue.
