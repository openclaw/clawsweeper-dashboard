## Summary

Keep this open. Current main still does not render the read-tool image result shape inline: the tool layer emits direct image blocks with `data` and `mimeType`, while the Control UI renderer only extracts `source.type === "base64"` or `url` image blocks, and the live tool stream flattens structured results into text-only tool-result blocks. The related open PR #52025 is an implementation candidate, not proof that the bug is fixed.

## What This Changes

The issue asks for Control UI Chat to show images returned by tools inline, or at least provide an obvious clickable preview/thumbnail in the tool output panel.

## Best Possible Solution

Control UI should preserve structured image blocks from stored history and live tool-result updates, convert direct `{ type: "image", data, mimeType }` blocks into safe inline previews or obvious thumbnails, and keep existing managed-media, local-media, and CSP protections intact with focused renderer and tool-stream regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to render or stream a tool-result message whose content contains `{ type: "image", data: "...", mimeType: "image/png" }`; current main either fails to extract that direct image block or flattens it to text before rendering.

## Solution Assessment

Unclear whether the open PR is the best implementation until maintainers review it. The best direction is a narrow Control UI/tool-stream fix with regression tests, not a broader remote-media or all-media feature change.
