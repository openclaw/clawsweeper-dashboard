## Summary

Keep open: the cleanup is useful, but the patch is not land-ready because the new build-only dialect uses Kysely's DummyDriver while still exposing a full Kysely execution surface, which can silently no-op if direct execution leaks through. The protected maintainer label and mock-only proof also require explicit maintainer handling before merge.

## What This Changes

The PR removes the custom async node:sqlite Kysely driver and retargets the dialect/tests around compile-only Kysely usage plus synchronous SQLite execution helpers.

## Best Possible Solution

Keep the build-only cleanup only if unsupported Kysely execution fails fast or is otherwise guarded, then have a maintainer decide whether the remaining proof is sufficient for this internal SQLite refactor.

## Reproduction Assessment

Yes for the review finding from source inspection: the PR-backed Kysely object can still call .execute(), and Kysely 0.29.2 DummyConnection.executeQuery returns an empty rows result. I did not run a live repro because this review is read-only.

## Solution Assessment

No: making the dialect build-only is a reasonable cleanup, but using DummyDriver without a fail-fast execution guard is not the safest implementation because direct execution remains type-accessible.
