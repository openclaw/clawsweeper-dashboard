## Summary

Keep this PR open, but it is not merge-ready: the patch is a coherent cross-path implementation, but it changes shipped compaction config semantics and the supplied terminal proof only greps patched output instead of executing a runtime path that prints the effective reserve.

## What This Changes

This PR changes compaction reserve resolution so explicit `agents.defaults.compaction.reserveTokens` is not raised by the implicit 20,000-token floor when `reserveTokensFloor` is omitted, across agent settings, memory flush, Codex projection/native startup, tests, and docs.

## Best Possible Solution

Preserve the shipped implicit-floor behavior by default unless maintainers explicitly approve the semantic change; if approved, add runtime proof that prints the effective reserve and reconcile the overlapping compaction PRs before merge.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.10` raise explicit `reserveTokens` by the implicit floor when `reserveTokensFloor` is omitted. I did not execute tests because this review is read-only.

## Solution Assessment

No, not yet: the implementation is plausible and covers sibling paths, but the existing documented solution is `reserveTokensFloor: 0`, so changing omitted-floor semantics needs maintainer approval and stronger runtime proof.
