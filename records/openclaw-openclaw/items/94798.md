- **[P1] Keep capping explicit reserve floors:** `src/agents/agent-settings.ts:80`
  - body: This branch only uses the capped `reserveTokensFloor` when `reserveTokens` is absent. If a small-context model sets both `reserveTokens` and an explicit higher `reserveTokensFloor`, the new `Math.max(..., configuredReserveTokensFloor)` can raise the reserve above the safe max again, undoing the guard at lines 64-76 that prevents every prompt from being treated as overflow. Use the capped floor when a floor is explicitly configured, and use zero only when no floor is configured.
  - confidence: 0.9
- **[P2] Keep Codex reserve readers consistent:** `extensions/codex/src/app-server/context-engine-projection.ts:109-112`
  - body: This changes the Codex context projection reader so an omitted `reserveTokensFloor` no longer applies the 20k default, but `startup-binding.ts` still applies that default for native-thread reserve handling. A Codex run with `reserveTokens` set and no floor can therefore use different reserve semantics depending on the path, while the docs say the explicit reserve is used as-is. Update the sibling reader or narrow the docs/tests to exclude it.
  - confidence: 0.82
