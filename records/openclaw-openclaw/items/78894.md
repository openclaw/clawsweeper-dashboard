## Summary

Current main and the shipped release line already implement plugin-owned dreaming cleanup so this missing-scope warning should not occur from a clean `v2026.5.6` install; the fix is covered by gateway plugin runtime tests and release provenance.

## What This Changes

The issue reports that the bundled memory-core dream promotion cron logs repeated narrative cleanup failures because isolated cron agent turns lack `operator.admin`.

## Best Possible Solution

Keep the shipped plugin-owned cleanup contract; if this warning still appears from a clean current install, handle it as a new report with package/dist proof rather than reopening the already-fixed scope path.

## Reproduction Assessment

No for current main: the old failure path is clear, but current source and regression tests route plugin-owned `deleteSession` through an admin-scoped synthetic cleanup client while preserving the plugin-owner guard.

## Solution Assessment

Yes; the shipped solution is narrow because it keeps `sessions.delete` admin-gated generally, adds plugin ownership metadata, and grants admin only for the plugin-owned cleanup dispatch.
