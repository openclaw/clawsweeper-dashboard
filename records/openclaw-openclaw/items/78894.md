## What This Changes

The issue requests that memory-core's managed dreaming cron stop failing narrative session cleanup with `missing scope: operator.admin` during isolated cron runs.

## Best Possible Solution

Keep the owner-scoped synthetic admin cleanup model and treat any post-upgrade recurrence as a fresh packaging/runtime report with exact installed dist evidence.

## Reproduction Assessment

No. Current main and tagged v2026.5.6 source route memory-core-owned cleanup through an owner-scoped admin client, and the regression tests cover the former missing-scope path.

## Solution Assessment

Yes. The shipped fix is narrower than granting broad admin to all plugin fallback calls because it preserves `operator.admin` for arbitrary deletes while allowing a plugin to clean up sessions it created.
