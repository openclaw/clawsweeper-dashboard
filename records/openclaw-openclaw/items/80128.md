Keep open: current main implements the `doctor` route-bound warning through https://github.com/openclaw/openclaw/pull/80250, but that merged work was intentionally scoped to diagnostics and does not resolve the broader runtime self-tool-awareness request.

## What This Changes

The issue requests clearer runtime tool-policy self-awareness for agents missing the `message` tool plus a `doctor` warning for channel-routed agents whose policy excludes `message`.

## Best Possible Solution

Keep the merged doctor diagnostic, then make an explicit maintainer decision on whether runtime prompts or a self-introspection tool should tell agents why `message` is unavailable.

## Reproduction Assessment

Yes for the channel/tool-policy mismatch: source inspection shows a curated allowlist without `message` filters that tool out, and current main now warns via `doctor`. No for the exact model-confabulation wording, which I did not reproduce live against Discord/Ollama.

## Solution Assessment

No, closing this issue now would skip the runtime self-awareness half. The doctor-only implementation is the right narrow diagnostic fix, but the remaining prompt/API direction needs maintainer judgment or a deliberately split follow-up.
