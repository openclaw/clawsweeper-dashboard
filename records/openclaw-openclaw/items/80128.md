## Summary

Keep open: current main still source-verifiably filters `message` out when an explicit allowlist omits it, and `openclaw doctor` only warns about visible-reply policy, not channel-bound explicit message actions. The linked fix PR at https://github.com/openclaw/openclaw/pull/80157 is open, unmerged, and reviewed as needing changes, so this issue should stay open until that path lands or is replaced.

## What This Changes

Request that agents understand when the `message` tool is excluded by their tool policy and that `openclaw doctor` warn for channel-bound agents missing that tool.

## Best Possible Solution

Land a corrected doctor warning that uses resolved tool-policy semantics, then either add compact policy context to the existing tool prompt or split that runtime self-awareness work into a narrower follow-up before closing.

## Reproduction Assessment

Yes. A non-empty agent allowlist without `message` source-reproducibly filters that tool out, while current main lacks a channel-binding doctor warning for the same mismatch; I did not live-reproduce the model wording.

## Solution Assessment
