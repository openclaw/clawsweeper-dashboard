## Summary

Keep open: current main still has the source-verifiable limitation where an explicit allowlist can remove `message`, while `doctor` only warns about visible-reply policy and does not warn for routed/channel-bound agents missing explicit message actions. The canonical open PR covers the doctor-warning half, but the runtime self-tool-awareness part still needs maintainer direction or a deliberate split.

## What This Changes

The issue requests runtime self-awareness for missing tools and a `doctor` warning when a channel-routed agent lacks the `message` tool.

## Best Possible Solution

Land a corrected doctor diagnostic for routed agents missing `message`, then either add compact resolved-tool-policy context for the runtime or split that product/API decision into a narrower follow-up before closing this issue.

## Reproduction Assessment

Yes, for the current limitation: source inspection shows a non-empty `tools.allow` without `message` filters that tool out, and current `doctor` lacks the routed-agent explicit-action warning. I did not live-run Discord/Ollama to reproduce the exact model wording.

## Solution Assessment

Yes for the first half: a deterministic `doctor` diagnostic is the narrow maintainable fix for the configuration mismatch. The runtime self-tool-awareness half should be approved as prompt/context work or split into a follow-up rather than closed implicitly by the doctor-only PR.
