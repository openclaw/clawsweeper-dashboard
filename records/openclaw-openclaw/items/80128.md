## Summary

Keep open: the routed-agent `doctor` warning has landed and is documented, but the issue explicitly asked for a second runtime self-tool-awareness/introspection improvement that current main still does not provide as an approved contract.

## What This Changes

The issue asks OpenClaw to warn when channel-routed agents lack the `message` tool and to give agents reliable self-awareness of missing tool permissions.

## Best Possible Solution

Keep the shipped doctor warning, then have maintainers either split the runtime self-tool-awareness request into the broader effective-tool diagnostics design or approve a narrow in-agent introspection surface.

## Reproduction Assessment

Yes for the remaining source-level limitation: current main filters `message` out under an explicit allowlist and exposes only available tools, not removed-tool policy provenance. I did not live-run Discord/Ollama to reproduce the exact fabricated wording.

## Solution Assessment

Unclear as an implementation choice. The doctor-only fix was the right narrow diagnostic, but the remaining runtime solution needs a maintainer decision between prompt context, a self-introspection tool, or the broader effective-tool diagnostics contract.
