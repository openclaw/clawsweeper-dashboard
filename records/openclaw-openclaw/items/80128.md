## Summary

Keep open: current main still has the source-verifiable policy mismatch and doctor coverage gap for channel-bound agents missing `message`; the linked hclsys commits are useful reference work, but they are not on current `main` and cover only the doctor-warning half.

## What This Changes

The issue asks for agents to report when `message` is unavailable under their tool policy and for `openclaw doctor` to warn when channel-bound agents lack that tool.

## Best Possible Solution

Reuse the existing policy-filtered tool prompt surface for a compact missing-tool explanation, and add a lightweight doctor warning that intersects route-bound agents with resolved `message` availability.
