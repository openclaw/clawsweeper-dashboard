## Summary

Keep open: current main still has the source-verifiable mismatch where an explicit tool allowlist can remove `message`, while doctor only warns about visible-reply policy and does not warn for channel-bound agents missing explicit message actions. Open PRs already reference this issue, so it should stay open until a fix is merged or the remaining runtime-context half is deliberately split.

## What This Changes

Request that agents truthfully surface missing `message` tool access and that `openclaw doctor` warn when a channel-bound agent's resolved tool policy omits `message`.

## Best Possible Solution

Review and land a corrected doctor warning, then add compact resolved-tool-policy context or split that runtime-introspection part into a follow-up before closing this issue.

## Reproduction Assessment

Yes, source-reproducible: a non-empty `tools.allow` without `message` filters that tool out, and current doctor has no route-bound agent warning for explicit message actions. I did not live-reproduce the exact Discord/Ollama wording.

## Solution Assessment

Yes, the narrow maintainable direction is to extend the existing policy-filtered prompt and doctor diagnostic surfaces. A new always-available introspection tool would be broader unless maintainers choose that API direction.
