## Summary

Keep open: current main and the latest release still leave the non-main empty-provider skip path without a models.json bootstrap, and the open closing PR should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue requests that newly created non-main agents avoid reaching runtime with a missing models.json when provider planning skips due to an empty provider set.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/58373 with a narrow skip-branch fix that preserves existing per-agent models.json and resolves the credential-copy boundary explicitly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: agents.create can leave a non-main agent without models.json, an empty provider plan returns skip, ensureOpenClawModelsJson writes nothing, and the embedded runner can then throw Unknown model.

## Solution Assessment

Yes for triage: keeping this issue open while the linked closing PR is unmerged is the right workflow. The implementation should stay narrow and either sanitize copied catalog data or receive explicit maintainer/security approval for raw same-user models.json propagation.
