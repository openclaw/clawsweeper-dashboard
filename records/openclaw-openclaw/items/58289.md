## What This Changes

The issue reports that a newly created non-main agent can miss `models.json` when provider planning returns `skip`, then fail later with `Unknown model`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/58373 with a narrow skip-branch fix that preserves existing per-agent registries and explicitly resolves the credential-copy boundary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.create` can leave a non-main agent without `models.json`, an empty provider plan returns `skip`, `ensureOpenClawModelsJson` writes nothing, and the embedded runner can then throw `Unknown model`.

## Solution Assessment

Yes for triage: keeping this issue open while the linked closing PR is unmerged is the right workflow. The implementation should stay narrow and either sanitize copied catalog data, fail early with actionable guidance, or receive explicit maintainer/security approval for raw same-user `models.json` propagation.
