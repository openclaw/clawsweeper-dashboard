The branch mirrors successful sessions_yield tool results into assistant-visible WebChat history, preserves raw result access, updates chat.message.get/SSE/subagent announce handling, and adds gateway plus agent regression coverage.

## Best Possible Solution

Land one canonical gateway projection that mirrors only successful sessions_yield waits, preserves raw toolResult/source ids, and keeps chat.history, chat.message.get, incremental replay, and subagent announce coverage aligned.

## Reproduction Assessment

Yes. Source inspection shows current main stores the sessions_yield wait text as a toolResult payload and default WebChat hides toolResult rows, so the replay-only status can disappear without a mirror projection.

## Solution Assessment

Yes. The display projection is the right inspected layer because WebChat docs say chat.history reads the transcript and applies display projection, while writing extra assistant transcript rows would broaden durable storage semantics.
