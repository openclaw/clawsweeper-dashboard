## Summary

Keep open: current main only partially addresses the temporal context problem with message-level Slack envelopes, but it does not implement the requested per-image timestamp plus thread-position annotation, no `annotateImages` config exists, and the only linked implementation PR was closed unmerged with review concerns.

Request image timestamp, sender, and thread-position metadata in assembled channel thread history so models can reason about stale screenshots.

## Best Possible Solution

Define a channel/SDK-owned supplemental media metadata contract, then implement channel support with tests for file-only messages, multi-image messages, capped history windows, sender/timestamp formatting, and prompt placement.
