## Summary

Keep open. Current main still lacks prompt-visible per-image timestamp, sender, and thread-position annotations in the concrete Slack thread-history path, and the only linked implementation PR is closed unmerged with review notes that still need a fresh, narrower fix.

## What This Changes

The issue requests timestamp, sender, and thread-position metadata for images included in channel thread history so vision-capable models can reason about image freshness.

## Best Possible Solution

Add a Slack-owned annotation in thread-history formatting using sanitized image metadata and window-qualified position wording, then consider cross-channel generalization only after another channel shows the same need.

## Reproduction Assessment

Yes at source level: Slack retains file-bearing thread messages, and current main formats them without image-specific timestamp, sender, or position text. I did not establish a live Slack workspace reproduction.

## Solution Assessment

Yes for the Slack slice: adding the annotation where Slack thread history is formatted is the narrowest maintainable path. A fresh implementation should avoid adding a new config knob unless maintainers explicitly want one.
