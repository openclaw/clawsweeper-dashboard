## Summary

Keep open. Current main and v2026.5.22 still only give recent history images a sender/message-id note, while Slack thread history has no prompt-visible per-image sent-time or position annotation; the only implementation attempt, https://github.com/openclaw/openclaw/pull/48725, was closed unmerged.

## What This Changes

Requests prompt-visible sent-time, sender, and thread-position metadata for images included from prior thread history so agents can reason about image freshness.

## Best Possible Solution

Add compact timestamp, position, and sender annotations in the existing history-image and Slack thread-history paths while preserving current image retention behavior.

## Reproduction Assessment

Yes, at source level. Current main can attach recent history images and format Slack thread history, but the prompt-visible image note lacks sent-time and position metadata; I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Yes for a bounded implementation. Extending the existing InboundHistory image note and Slack thread formatter is narrower than stripping old images or adding a broad new channel API, while the old PR's total-count and multi-image details need refinement.
