## Summary

Keep open. Current main still assembles Slack thread-history image/file messages without per-image sent-time, sender, or position annotation; the only directly related PR is closed unmerged, and no open superseding item owns the work.

## What This Changes

Request annotating prior thread-history images with timestamp, sender, and message position metadata so models can reason about image freshness.

## Best Possible Solution

Add a Slack-owned annotation path at thread-history formatting with sanitized metadata and focused regression tests; promote a shared SDK/channel seam only after another channel needs the same contract.

## Reproduction Assessment

Yes, at source level: Slack history messages with files flow through resolveSlackThreadHistory, but resolveSlackThreadContextData formats only text/message-id context and never emits per-image sent-time, sender, or position metadata. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Yes, the best path is a narrow Slack-first implementation at the existing thread-history formatting point. Do not copy the closed PR's core helper/config shape unless maintainers explicitly choose a shared plugin SDK seam or opt-out setting.
