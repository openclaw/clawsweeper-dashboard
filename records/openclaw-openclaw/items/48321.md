The issue requests prompt-visible timestamp, sender, and thread-position metadata for images retained in channel thread history.

## Best Possible Solution

Add a narrow Slack-owned thread-history annotation using existing Slack message/file metadata, with regression coverage and clear window-qualified position wording before considering a shared cross-channel contract.

## Solution Assessment
