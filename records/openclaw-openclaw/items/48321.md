## Summary

Keep open. Current main and latest release still only provide partial image context: recent history images get a sender/message-id note, while Slack thread history has timestamped message envelopes but no prompt-visible per-image timestamp plus position annotation; the only linked implementation attempt was closed unmerged.

## What This Changes

Request adds prompt-visible timestamp, sender, and thread-position metadata for images carried from prior threaded channel messages.

## Best Possible Solution

Add compact default-on per-image annotations carrying sender, sent time, and accurate position/window context in the shared InboundHistory image path and Slack thread formatter, without changing image retention or adding config by default.

## Reproduction Assessment

Yes, source-level. Current main can attach recent history images, but appendRecentHistoryImageContext only emits sender/message-id text, and Slack thread history formats timestamped messages without image-specific timestamp plus position annotations.

## Solution Assessment

Yes, a bounded implementation in the shared history-image helper plus Slack thread formatter is the best current path. It is narrower than stripping old images, and safer than the closed Slack-only PR's config-heavy shape with inaccurate total counts.
