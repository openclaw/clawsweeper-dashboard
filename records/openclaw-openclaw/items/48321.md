## Summary

Keep open. Current main still lacks prompt-visible per-image timestamp, sender, and thread-position annotations for the concrete Slack thread-history path, and the directly related implementation PR was closed unmerged.

## What This Changes

The issue requests adding sent-time, sender, and thread-position metadata before image payloads in threaded channel history so models can reason about image freshness.

## Best Possible Solution

Add a Slack-owned annotation at the existing thread-context assembly point using sanitized image metadata and clearly window-qualified position wording; consider a shared channel contract later only with multi-channel evidence.

Yes at source level: Slack thread history preserves file-bearing messages, but the formatter emits message envelopes without per-image timestamp, sender, or position annotations, and thread-starter media is passed separately as MediaPaths. I did not run a live Slack workspace reproduction.

## Solution Assessment

Yes for the Slack slice: the narrow maintainable fix belongs in Slack thread-history formatting with focused tests. The closed PR is useful prior art, but a fresh fix should avoid misleading total counts and duplicated annotations for multi-image messages.
