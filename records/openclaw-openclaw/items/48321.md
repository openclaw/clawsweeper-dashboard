Keep open: current main and the latest release only partially address related history-image attachment handling, and attached prior images still do not get explicit sent-time and thread-position annotations. The closed Slack-only candidate and the merged recent-history-image PR do not supersede the requested shared metadata feature.

## What This Changes

The issue requests timestamp, position, and sender annotations for image attachments in channel thread history so models can reason about image freshness.

## Best Possible Solution

Add the annotation in the shared recent-history image context path, deriving sender, message id, sent timestamp or relative age, and bounded history position from existing `InboundHistory` data without adding a new config option by default.

## Reproduction Assessment

Yes for the source-level gap: current main can attach recent history images and only labels them with sender/message id, not sent time or thread position. I did not reproduce the live model failure in Slack or another channel.

## Solution Assessment

Yes, if implemented in the shared history-image helper. That is narrower and more maintainable than a Slack-only formatter or a new disable-config surface, while preserving the existing current-media precedence rules.
