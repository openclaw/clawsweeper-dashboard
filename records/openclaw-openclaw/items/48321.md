## Summary

Keep open. Current main and the latest release still do not annotate Slack thread-history images with sent time, sender, or message position, and the directly related implementation PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to annotate image-bearing thread-history entries with timestamp, sender, and position metadata before they are sent to the model.

## Best Possible Solution

Implement a Slack-owned annotation path first, with sanitized per-image context and focused Slack tests; promote a shared SDK seam only if another channel later needs the same contract.

## Reproduction Assessment

Yes, at source level: Slack history messages with files flow through resolveSlackThreadHistory, but resolveSlackThreadContextData formats only text/message id metadata and never emits per-image timestamp, sender, or position data. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Yes, the maintainable direction is a narrow Slack plugin fix at the existing thread-history formatting point. Avoid copying the closed PR's core helper shape into extension production code unless a documented plugin SDK seam is deliberately added.
