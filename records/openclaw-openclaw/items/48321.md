## Summary

Keep open. Current main still lacks per-image sent-time, sender, and thread-position metadata for Slack thread-context media, and the directly related implementation PR #48725 was closed unmerged rather than shipped.

## What This Changes

Request to annotate images included from channel thread history with sent time, sender, and thread position metadata so models can reason about image freshness.

## Best Possible Solution

Implement a Slack-owned annotation path for image-bearing thread context first, with sanitized sent time, sender, and message position/total; generalize only if another channel needs the same SDK contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Slack thread history with file-bearing messages reaches `resolveSlackThreadContextData`, but current formatting emits text/envelope data and generic media notes without per-image timestamp, sender, or thread-position metadata.

## Solution Assessment

Yes, the requested direction is reasonable, but the narrow maintainable fix is Slack-owned annotation rather than a broad all-channel/core helper change. Add a config knob only if maintainers explicitly want an opt-out.
