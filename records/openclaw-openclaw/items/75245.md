## Summary

Keep open. Current main still wraps plugin hook handlers with a shallow-cloned context, while the `agent:bootstrap` path reads mutations back from the original event context; the open linked PR #75249 explicitly targets this issue and should be reviewed or landed before this issue closes.

## What This Changes

The issue requests fixing plugin `registerHook()` so mutable hook context changes, especially `agent:bootstrap` updates to `context.bootstrapFiles`, survive the plugin wrapper.

## Best Possible Solution

Preserve the original hook context object in the plugin wrapper while temporarily exposing and restoring `pluginConfig`, then add regression coverage for plugin-registered `agent:bootstrap` mutation and `pluginConfig` cleanup.

## Reproduction Assessment

Yes. Register an `api.registerHook('agent:bootstrap')` handler that assigns a sentinel `event.context.bootstrapFiles` array, then resolve bootstrap context; current main still gives the handler a cloned context while core reads the original context afterward.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `src/plugins/registry.ts`; a bootstrap-only workaround would leave the generic mutable plugin hook contract broken.
