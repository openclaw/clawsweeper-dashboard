## Summary

Keep this issue open: current main and the latest release still shallow-copy plugin hook context in `registerHook`, so plugin `agent:bootstrap` context replacements are still dropped, and the prior focused repair PRs are closed unmerged.

## What This Changes

The issue requests that `api.registerHook()` preserve mutable `event.context` changes so `agent:bootstrap` handlers can replace `bootstrapFiles` before prompt/context injection.

## Best Possible Solution

Fix the plugin registry wrapper to pass the original context with temporary `pluginConfig` restoration, then add focused regression coverage for plugin-level `agent:bootstrap` mutation propagation and config cleanup.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the plugin wrapper clones `evt.context`, while bootstrap resolution reads `bootstrapFiles` from the original event after hook dispatch.

## Solution Assessment

Yes. The best fix is at the generic `registerHook` wrapper, not a bootstrap-only workaround, because the documented mutable hook context contract is broader than one consumer.
