## Summary

Keep open. Current main still has the shallow-copying plugin hook wrapper described in the report, and the linked open PR #75249 references this issue with closing syntax, so the issue should remain open until that fix lands or is replaced.

## What This Changes

Ensure plugin-registered hooks preserve mutable `event.context` changes, especially `agent:bootstrap` updates to `bootstrapFiles`, while still exposing per-plugin `pluginConfig`.

## Best Possible Solution

Review and land PR #75249, or an equivalent narrow change in the plugin hook wrapper that passes the original context object while temporarily adding and restoring `pluginConfig`, with regression coverage for `api.registerHook('agent:bootstrap')` mutation and pluginConfig cleanup.

## Reproduction Assessment

Yes. A high-confidence source reproduction is clear: register a plugin hook via `api.registerHook('agent:bootstrap')`, assign a sentinel array to `event.context.bootstrapFiles`, then resolve bootstrap context and observe current main still reads the original event context.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the plugin hook wrapper so all mutable hook contracts work consistently; special-casing `agent:bootstrap` would leave the generic SDK behavior broken.
