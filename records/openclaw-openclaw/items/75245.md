## Summary

Keep open. Current main still wraps plugin hook handlers with a shallow-cloned context while the bootstrap hook consumer reads `bootstrapFiles` back from the original event context; PR #75249 was closed unmerged, so the narrow fix remains actionable.

## What This Changes

The issue requests preserving mutable `api.registerHook()` context changes, especially `agent:bootstrap` `bootstrapFiles` mutations, through the plugin hook wrapper.

## Best Possible Solution

Land or replace the PR #75249-style fix: pass the original context object to plugin handlers, expose `pluginConfig` only during that handler call with `finally` cleanup, and add regression coverage for `agent:bootstrap` mutation plus `pluginConfig` cleanup.

## Reproduction Assessment

Yes. Register a plugin with `api.registerHook('agent:bootstrap')` that assigns `event.context.bootstrapFiles`, then call the bootstrap override path; current main sends that assignment to the wrapper clone while `applyBootstrapHookOverrides()` reads the unchanged original event context.

## Solution Assessment

Yes. The narrowest maintainable fix is to preserve the original context object and temporarily restore/delete `pluginConfig`; a bootstrap-only special case would miss the generic wrapper bug, and an immutable return-value API would be a larger SDK change.

## Repair Work Prompt

Fix openclaw/openclaw#75245. Current main still shallow-clones `event.context` in `src/plugins/registry.ts` when wrapping `api.registerHook()` handlers to inject `pluginConfig`, which drops mutable context assignments such as `agent:bootstrap` `event.context.bootstrapFiles = updated`. Implement the narrow wrapper repair: pass the original context object to the handler, temporarily attach `pluginConfig`, and restore the previous value or delete it in a `finally` block so pluginConfig remains visible during the handler but does not leak afterward. Do not special-case `agent:bootstrap`, do not change the broader hook API, and preserve sequential handler isolation for `pluginConfig`. Add regression coverage showing a plugin-registered `agent:bootstrap` hook can replace bootstrap files and that `pluginConfig` is visible during the handler but absent/restored after dispatch. Add a user-facing changelog fix entry unless maintainers decide this is internal-only.
