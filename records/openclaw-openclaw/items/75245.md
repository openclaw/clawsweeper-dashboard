## Summary

Keep open. Current main still has the reported wrapper behavior: plugin hook handlers receive a shallow-cloned `event.context`, while the bootstrap hook consumer reads mutations from the original event context after hook execution. PR #75249 is already open as the focused fix and references this issue for closure, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that `api.registerHook()` drops plugin mutations to `event.context`, especially `agent:bootstrap` changes to `bootstrapFiles`, because the plugin registry wrapper passes a shallow-cloned context object.

## Best Possible Solution

Land PR #75249 or an equivalent narrow fix that preserves the original hook context object while temporarily exposing and restoring `pluginConfig`, with regression coverage for plugin-registered `agent:bootstrap` mutation propagation and `pluginConfig` cleanup.

## Reproduction Assessment

Yes. Registering an `api.registerHook('agent:bootstrap')` handler that assigns a sentinel `bootstrapFiles` array should reproduce the issue because current main mutates the wrapper clone while `applyBootstrapHookOverrides()` reads the original event context.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `src/plugins/registry.ts`; a bootstrap-only special case would leave the generic mutable hook contract broken.
