- `impact:session-state`: The bug causes child-agent bootstrap context and prompt-report state to diverge from plugin-mutated session context.

## Summary

Keep this issue open: current main and the latest release still use the shallow-copied plugin hook context while the agent bootstrap path and docs expect mutable bootstrap context, and the open linked PR owns the exact fix before this issue should close.

## What This Changes

The issue reports that `api.registerHook()` drops plugin mutations to `event.context.bootstrapFiles` before `agent:bootstrap` output is converted into injected context files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89002, preserving mutable hook context while keeping per-handler `pluginConfig` cleanup, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `registerHook` clones `evt.context`, while `applyBootstrapHookOverrides` later reads `bootstrapFiles` from the original event context after hook dispatch.

## Solution Assessment

Yes. The best fix is at the generic plugin registry wrapper rather than a bootstrap-only workaround, because the bug is context identity loss for all registered hook handlers.
