## Summary

Keep open. Current main still wraps plugin hook handlers with a shallow-cloned `event.context`, while the `agent:bootstrap` consumer reads the original event context after hooks run; PR #75249 is open with closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that `api.registerHook()` preserve mutable `event.context` changes, especially `agent:bootstrap` updates to `context.bootstrapFiles`, instead of dropping assignments made on the wrapper clone.

## Best Possible Solution

Preserve the original hook context object in `src/plugins/registry.ts` while temporarily exposing and restoring `pluginConfig`, then add regression coverage for plugin-registered `agent:bootstrap` mutation propagation and `pluginConfig` cleanup.

## Reproduction Assessment

Yes. Register a plugin with `api.registerHook('agent:bootstrap')` that replaces `event.context.bootstrapFiles`, then resolve bootstrap context; current main will still build injected context from the original files because the assignment happens on the wrapper clone.

## Solution Assessment

Yes. The narrow maintainable fix is in the plugin hook wrapper; a bootstrap-only special case would leave the generic mutable `registerHook()` contract broken.
