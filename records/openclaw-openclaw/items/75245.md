## Summary

Keep open. Current main and latest release still shallow-clone plugin hook context, while the documented `agent:bootstrap` contract and bootstrap consumer expect `context.bootstrapFiles` mutations to survive; the open linked fix PR should be reviewed and landed before this issue closes.

## What This Changes

The issue asks for plugin `registerHook()` to preserve mutable hook context changes, especially `agent:bootstrap` updates to `context.bootstrapFiles`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77349 or an equivalent narrow wrapper fix, then let the issue close through the merged PR.

## Reproduction Assessment

Yes. A plugin loaded through `api.registerHook('agent:bootstrap')` that assigns `event.context.bootstrapFiles` follows a source-proven failing path: the wrapper mutates a cloned context, then bootstrap resolution reads the unchanged original context.

## Solution Assessment

Yes. Preserving the original context in the generic plugin wrapper with scoped `pluginConfig` cleanup is the narrow compatible fix; the merged bootstrap-routing PR does not address the wrapper clone.
