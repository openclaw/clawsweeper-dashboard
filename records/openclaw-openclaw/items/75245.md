## Summary

Keep open. Current main still shallow-clones `event.context` inside the plugin `registerHook` wrapper, while the documented `agent:bootstrap` contract and bootstrap consumer expect `context.bootstrapFiles` mutations to survive the hook call.

## What This Changes

The issue requests that plugin hooks registered with `api.registerHook()` preserve mutable `event.context` changes, especially `agent:bootstrap` replacements of `bootstrapFiles`.

## Best Possible Solution

Preserve the original hook context object in `registerHook`, expose `pluginConfig` only for the active handler with cleanup/restoration, and add regression coverage for plugin-loader bootstrap mutation propagation plus config non-leakage.

## Reproduction Assessment

Yes. A plugin loaded through `api.registerHook('agent:bootstrap')` that assigns `event.context.bootstrapFiles` follows a source-proven failing path: the wrapper mutates a cloned context, then bootstrap resolution reads the unchanged original context.

## Solution Assessment

Yes. The narrowest maintainable fix is in the plugin registry wrapper; special-casing bootstrap would leave the generic mutable hook contract broken, and a return-value hook API would be a larger compatibility change.

## Repair Work Prompt

Fix the plugin hook mutable context regression reported in https://github.com/openclaw/openclaw/issues/75245. Current `src/plugins/registry.ts` shallow-clones `event.context` while injecting `pluginConfig`, which drops assignment-style mutations such as `event.context.bootstrapFiles = updated` for `agent:bootstrap`. Preserve the original context object for the handler, temporarily expose `pluginConfig`, and restore/delete `pluginConfig` in `finally` so config remains handler-local and does not leak to the shared event context. Add regression coverage proving a plugin loaded through `api.registerHook('agent:bootstrap')` can replace bootstrap files and that `pluginConfig` remains visible during the handler but absent/restored afterward. Keep the fix in the plugin registry/loader boundary; do not special-case bootstrap in `src/agents` and do not introduce a new hook return-value API or config option. Add a user-facing changelog fix entry if repo policy requires it for this runtime behavior change.
