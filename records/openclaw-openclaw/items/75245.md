## Summary

Keep open. Current main still has the reported plugin hook wrapper behavior, and the open linked PR explicitly targets this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that plugin-registered hooks receive a shallow-cloned context, causing `agent:bootstrap` mutations such as replacing `bootstrapFiles` to be dropped before prompt injection.

## Best Possible Solution

Preserve the original hook context object in the plugin wrapper while temporarily exposing and restoring `pluginConfig`, then add regression coverage for plugin-registered `agent:bootstrap` mutation and `pluginConfig` cleanup.

## Reproduction Assessment

Yes. Register an `api.registerHook('agent:bootstrap')` handler that assigns a sentinel array to `event.context.bootstrapFiles`, then resolve bootstrap context and observe current main still builds injected context from the original bootstrap files because the assignment happens on the wrapper clone.

## Solution Assessment

Yes. The narrow maintainable fix is in `src/plugins/registry.ts`; a bootstrap-only special case would leave the generic mutable hook contract broken.
