## Summary

Keep open. Current main still shallow-clones `event.context` in the plugin `registerHook` wrapper, while `agent:bootstrap` reads the original event context after hooks run and the docs define `context.bootstrapFiles` as mutable.

## What This Changes

Preserve mutation-style `registerHook` context changes so `agent:bootstrap` plugins can replace or remove bootstrap files before prompt injection.

## Best Possible Solution

Preserve the original hook context object in `registerHook`, temporarily expose `pluginConfig` with restoration in `finally`, and add regression coverage for plugin-loaded `agent:bootstrap` mutation propagation plus config non-leakage.

## Reproduction Assessment

Yes. A plugin loaded through `api.registerHook('agent:bootstrap')` that assigns `event.context.bootstrapFiles` follows a source-proven failing path: the wrapper mutates a cloned context, then bootstrap resolution reads the unchanged original context.

## Solution Assessment

Yes. The narrowest maintainable fix is in the plugin registry wrapper; special-casing bootstrap would leave the generic mutable hook contract broken, and a return-value API would be a larger compatibility change.
