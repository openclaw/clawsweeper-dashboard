## Summary

Keep this issue open: current main and the latest release still clone `event.context` in the plugin hook wrapper, while the bootstrap path still expects mutable `bootstrapFiles`; the linked fixing PR is open, draft, mergeable, and owns the exact remaining work.

## What This Changes

Ensure `api.registerHook()` handlers can mutate the shared hook context so `agent:bootstrap` `bootstrapFiles` changes propagate into injected agent context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89002, preserving mutable hook context while keeping per-handler `pluginConfig` cleanup, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `registerHook` clones `evt.context`, while `applyBootstrapHookOverrides` later reads `bootstrapFiles` from the original event context after hook dispatch.

## Solution Assessment

Yes. The best fix is at the generic plugin registry wrapper, not a bootstrap-only workaround, because the bug is caused by context identity loss for all registered hook handlers.
