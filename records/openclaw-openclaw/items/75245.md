## Summary

Keep open. Current main still shallow-clones plugin hook context in `registerHook`, while the documented `agent:bootstrap` path expects `context.bootstrapFiles` mutations to survive; an open closing PR already tracks the narrow fix.

## What This Changes

The issue asks for plugin `registerHook()` context mutations, especially `agent:bootstrap` `bootstrapFiles` changes, to propagate to the original hook event context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77349 or an equivalent narrow wrapper fix, then let this issue close through the merged PR.

## Reproduction Assessment

Yes. Source inspection shows a plugin loaded through `api.registerHook('agent:bootstrap')` that assigns `event.context.bootstrapFiles` mutates a wrapper clone, while bootstrap resolution later reads the original `event.context.bootstrapFiles`.

## Solution Assessment

Yes. Preserving the original context in the generic plugin wrapper with scoped `pluginConfig` cleanup is the narrow compatible fix; adding a return-value API or bootstrap-only special case would be broader than needed.
