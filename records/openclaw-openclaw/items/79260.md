## Best Possible Solution

Land the post-upgrade mode after absent installed-index state returns an empty report, while leaving the broader upgrade scanner design tracked by https://github.com/openclaw/openclaw/issues/17215.

## Reproduction Assessment

Yes for the blocking finding: source inspection of the PR head shows `runPostUpgradeProbes` reads the installed-index path directly before building a report. I did not execute it because this review is read-only, but the missing-file failure follows from that code path.

## Solution Assessment

No, not quite yet. The narrow maintainable fix is to reuse or mirror the existing tolerant installed-index reader semantics so absent state produces an empty report instead of a runtime failure.
