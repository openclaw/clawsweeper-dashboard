## Summary

Keep open: the PR adds a useful post-upgrade doctor mode, but the entry probe still re-implements installed plugin entry resolution with weaker path checks, so it can miss source-only TypeScript package failures and report invalid escaping entries as healthy.

## What This Changes

Adds `openclaw doctor --post-upgrade [--json]` with plugin entry and manifest-drift probes, CLI/docs/changelog wiring, and focused tests.

## Best Possible Solution

Keep the new mode scoped, but reuse or exactly mirror the existing boundary-safe package entry resolver and add regression fixtures for TypeScript source-only, inferred built peer, escaping path, blank runtimeExtensions, and runtimeExtensions mismatch cases before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a package with `openclaw.extensions: ["./src/index.ts"]`, source present, and no built `dist` peer is rejected by current resolver/tests but produces no PR finding; an absolute or `../` runtime entry can also pass the PR check if the target exists.

## Solution Assessment

No. The requested CLI direction is reasonable, but the implementation should share the current package-entry resolver instead of maintaining weaker direct filesystem checks.
