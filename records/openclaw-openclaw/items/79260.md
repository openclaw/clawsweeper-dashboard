## Summary

Keep this PR open: the new post-upgrade doctor mode is useful and has real terminal proof, but the current branch still crashes before producing findings when the installed plugin index file is absent.

## What This Changes

Adds `openclaw doctor --post-upgrade [--json]`, plugin compatibility probes for unresolved entries and manifest drift, CLI/docs/changelog wiring, and focused tests.

## Best Possible Solution

Make post-upgrade mode treat an absent persisted installed-plugin index as an empty install set with no findings and exit 0, while preserving error findings for broken enabled plugins.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows `runPostUpgradeProbes` directly reads `params.installsPath`, so a fresh state dir without `plugins/installs.json` rejects before any no-finding output can be emitted.

## Solution Assessment

No; the PR is close, but the best fix is to handle missing persisted plugin-index state through the existing nullable-read pattern or an explicit empty report before exposing the new CLI contract.
