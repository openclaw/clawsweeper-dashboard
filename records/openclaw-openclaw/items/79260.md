## Summary

Keep this PR open. The feature is focused and mostly lines up with the plugin install/discovery contract, but the new probe runner currently fails before producing a report when the installed plugin index file is absent, which is a realistic clean-state and CI smoke-check case for a new doctor mode.

## What This Changes

The PR adds `openclaw doctor --post-upgrade [--json]`, plugin entry-resolution and manifest-drift probes, CLI/docs/changelog wiring, and focused regression tests.

## Best Possible Solution

Land the scoped plugin-compat doctor probe after it handles absent installed-index state gracefully and maintainers approve the new CLI/report contract.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. The review path is source inspection of current main versus PR head plus the supplied terminal proof and fixture tests.

## Solution Assessment

No, not quite yet. Delegating entry validation to the shared resolver is the right maintainable direction, but the runner should first handle a missing installed plugin index as an empty/no-findings state or through the existing installed-index reader instead of crashing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle missing installed-index state:** `src/commands/doctor-post-upgrade.ts:41`
  - body: `runPostUpgradeProbes` reads the resolved `plugins/installs.json` directly, so a fresh state dir or clean CI smoke run without that file throws `ENOENT` before returning the documented JSON/text report. Existing installed-index readers treat missing persisted state as `null`; this mode should mirror that and emit the normal no-findings report instead of failing before any probe runs.
