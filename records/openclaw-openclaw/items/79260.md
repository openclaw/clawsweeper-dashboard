## Summary

Keep open: this is useful doctor/plugin compatibility work with sufficient terminal proof, but the PR still breaks its documented output contract for fresh or no-plugin state because it raw-reads `plugins/installs.json` before it can emit an empty report.

## What This Changes

The PR adds `openclaw doctor --post-upgrade [--json]` with plugin entry and manifest-drift probes, CLI/docs/changelog wiring, and focused tests.

## Best Possible Solution

Land the post-upgrade plugin probe after absent or invalid installed-index state returns an empty structured report, while leaving the broader risk-scored upgrade scanner design tracked by https://github.com/openclaw/openclaw/issues/17215.

## Reproduction Assessment

Yes for the blocking finding by source inspection: a fresh `OPENCLAW_STATE_DIR` without `plugins/installs.json` reaches the raw `fs.readFile` before any report is emitted. I did not execute the command because this review is read-only, but the failure follows from the PR head and the existing persisted-index contract.

## Solution Assessment

No, not yet. The maintainable fix is to reuse or mirror the existing tolerant installed-index reader semantics so absent or invalid state produces an empty report instead of a runtime failure.
