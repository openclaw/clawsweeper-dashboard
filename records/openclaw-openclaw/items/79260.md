## Summary

Keep open: the doctor mode is useful and the PR supplies real terminal proof, but current head still disagrees with the existing plugin entry resolver on runtime entry inference, plugin-root boundaries, and malformed runtimeExtensions.

## What This Changes

Adds `openclaw doctor --post-upgrade [--json]` with plugin entry and manifest-drift probes, CLI/docs/changelog wiring, and focused tests.

## Best Possible Solution

Keep the scoped doctor mode, but route entry probing through the existing package-entry resolver or a shared helper so install, discovery, and post-upgrade diagnostics agree.

## Reproduction Assessment

Yes. Source inspection against current main gives concrete fixtures for escaping entries, inferred built `dist` output, source-only TypeScript entries, and malformed runtimeExtensions that the PR probe does not mirror.

## Solution Assessment

No. The CLI direction is useful, but duplicating partial `fs.access` checks is not the narrowest maintainable solution; the probe should reuse or exactly mirror the current resolver contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Mirror inferred runtime entry resolution:** `src/commands/doctor-post-upgrade.ts:85-86`
  - body: When no explicit runtime entry is present, current install/discovery can accept missing TypeScript source entries if the built `dist/*.js` peer exists, and can reject source-only TypeScript packages without compiled output. This probe only checks the source path, so it can reject valid installed packages and miss invalid source-only ones.
  - confidence: 0.95
- **[P2] Keep entry checks inside plugin roots:** `src/commands/doctor-post-upgrade.ts:68-69`
  - body: Resolving plugin-authored entries with `path.resolve(record.rootDir, entry)` plus `fs.access` accepts absolute paths or `..` escapes when the target exists. Current entry resolution rejects paths that leave the plugin package directory, so this can mark an invalid plugin healthy.
  - confidence: 0.92
- **[P2] Validate runtimeExtensions before indexing:** `src/commands/doctor-post-upgrade.ts:62-67`
  - body: Current resolver rejects blank runtimeExtensions values and length mismatches before resolving entries. This code treats the field as an unchecked positional list, so malformed packages can fall through to source-entry checks and avoid the compatibility error install/discovery would report.
  - confidence: 0.89
