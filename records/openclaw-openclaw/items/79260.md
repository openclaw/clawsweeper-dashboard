## Summary

Keep open. This is a focused CLI feature with useful proof and docs, but the latest patch still duplicates weaker plugin entry resolution logic, so it can miss broken installed plugins and mark invalid paths healthy.

## What This Changes

The PR adds `openclaw doctor --post-upgrade [--json]` with plugin entry and manifest-drift probes, CLI option wiring, docs, changelog, and focused tests.

## Best Possible Solution

Keep the scoped post-upgrade mode, but route `plugin.entry_unresolved` through the existing package-entry validation/resolution contract and add fixtures for source-only TypeScript, inferred built runtime, escaping paths, and runtimeExtensions shape cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a package with `openclaw.extensions: ["./src/index.ts"]` can be misclassified depending on compiled runtime output, and absolute or `../` entries can be accepted by the PR even though current main rejects them.

## Solution Assessment

No. The CLI direction is reasonable, but the probe should reuse or exactly mirror the existing package-entry resolver instead of maintaining weaker direct filesystem checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Mirror packaged runtime entry resolution:** `src/commands/doctor-post-upgrade.ts:85-86`
  - body: When no explicit `runtimeExtensions` entry is present, this checks only the source path. Current package install/discovery also tries inferred built runtime entries and rejects source-only TypeScript entries without compiled output, so this probe can both false-error valid packaged plugins and miss broken source-only ones.
  - confidence: 0.94
- **[P2] Keep probed entries inside plugin roots:** `src/commands/doctor-post-upgrade.ts:68-69`
  - body: Resolving plugin-declared entries with `path.resolve(record.rootDir, runtimeEntry)` and then `fs.access` accepts absolute or parent-directory paths when that target exists. Current plugin validation rejects escaping entries, so this mode can report an invalid plugin as healthy and probe paths outside the plugin package.
  - confidence: 0.91
