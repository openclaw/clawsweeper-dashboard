## Summary

Keep open: the PR still duplicates only part of the package-entry resolver, so it can misreport installed plugin health in cases current main accepts or rejects differently.

## What This Changes

Adds `openclaw doctor --post-upgrade [--json]` with plugin entry and manifest-drift probes, CLI/docs/changelog wiring, and focused tests.

## Best Possible Solution

Keep the scoped doctor mode, but have entry probing share or exactly mirror the existing package-entry resolver for runtimeExtensions shape, inferred built output, source-only TypeScript, and root-boundary checks.

## Reproduction Assessment

Yes. Source inspection gives concrete fixtures from current main: source-only TypeScript without compiled output, inferred built `dist` entries, escaping paths, and malformed runtimeExtensions are all handled by existing resolver tests but not by the PR probe.

## Solution Assessment

No. The CLI direction is useful, but duplicating partial `fs.access` checks is not the narrowest maintainable fix; the probe should reuse or mirror the current resolver contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Mirror built runtime entry resolution:** `src/commands/doctor-post-upgrade.ts:85-86`
  - body: When no explicit `runtimeExtensions` entry is present, current OpenClaw looks for inferred built output and rejects source-only TypeScript packages without it. This check only tests the source path, so it can mark valid packages with missing source but built `dist` output as errors and mark source-only TypeScript packages as healthy.
  - confidence: 0.95
- **[P2] Keep entry checks inside plugin roots:** `src/commands/doctor-post-upgrade.ts:68-69`
  - body: Resolving plugin-declared entries with `path.resolve(record.rootDir, entry)` plus `fs.access` accepts absolute or parent-directory paths when the target exists. Current plugin validation rejects entries that escape the package directory, so this probe should use the same boundary-safe resolver before considering an entry healthy.
  - confidence: 0.92
- **[P2] Validate runtimeExtensions before indexing:** `src/commands/doctor-post-upgrade.ts:63-66`
  - body: Current resolver rejects blank `openclaw.runtimeExtensions` values and length mismatches before resolving entries. This code treats the field as a plain array and falls back per index, letting packages that current OpenClaw rejects pass the post-upgrade probe.
  - confidence: 0.88
