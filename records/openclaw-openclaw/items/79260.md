## Summary

Keep open: the feature is useful, but the patch is not merge-ready because the new entry probe still duplicates weaker filesystem checks instead of the existing plugin entry resolver, so it can miss invalid installed-plugin cases.

## What This Changes

Adds `openclaw doctor --post-upgrade [--json]` with plugin entry and manifest-drift probes, CLI/docs/changelog wiring, and focused tests.

## Best Possible Solution

Keep the scoped doctor mode, but have `plugin.entry_unresolved` reuse or exactly mirror the existing boundary-safe package-entry validation before merge.

## Reproduction Assessment

Yes. Source inspection gives concrete fixtures: source-only TypeScript without `dist`, escaping absolute or `../` entries, and malformed `runtimeExtensions` are rejected by current OpenClaw but can produce no PR finding.

## Solution Assessment

No. The CLI direction is reasonable, but duplicating partial `fs.access` checks is not the safest implementation; the entry probe should share the current package-entry resolver contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Report source-only TypeScript entries:** `src/commands/doctor-post-upgrade.ts:86`
  - body: This only reports `plugin.entry_unresolved` when the source entry is missing. Current install/discovery rejects an installed package with `openclaw.extensions: ["./src/index.ts"]` when no explicit runtime entry or inferred `dist/*.js` peer exists, so this mode can report a skipped plugin as healthy.
  - confidence: 0.95
- **[P2] Keep entry probes inside plugin roots:** `src/commands/doctor-post-upgrade.ts:68-69`
  - body: Resolving plugin-declared runtime and source entries with `path.resolve(record.rootDir, entry)` plus `fs.access` accepts absolute or parent-directory paths when the target exists. Current plugin validation rejects escaping entries, so the probe should use the same boundary-safe resolver before marking an entry healthy.
  - confidence: 0.92
- **[P2] Validate runtimeExtensions shape:** `src/commands/doctor-post-upgrade.ts:63-66`
  - body: The current resolver rejects blank `openclaw.runtimeExtensions` entries and length mismatches before resolving entries. This code treats blank values as absent and ignores extra or missing runtime entries, letting packages that current OpenClaw rejects pass the post-upgrade probe.
  - confidence: 0.88
