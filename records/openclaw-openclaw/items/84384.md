## Summary

Keep open: current main still has the native Google thinking keepalive pattern, but the shared OpenAI-compatible completions stream path is not proven to emit any assistant-stream event for Vertex/Gemini reasoning- or usage-only chunks, so the reported idle-timeout failure remains plausible and actionable.

## What This Changes

The issue requests that Vertex/OpenAI-compatible Gemini streams treat reasoning-only activity as stream activity so the LLM idle watchdog does not abort before a Telegram-visible response is delivered.

## Best Possible Solution

Add a focused OpenAI-compatible completions streaming fix that treats Gemini/Vertex reasoning-only or usage-only activity as a keepalive without leaking hidden reasoning as visible text, with regression coverage next to the existing reasoning stream tests.
