## Summary

Keep open. Current main and the latest release still do not expose editable named OpenClaw-owned system prompt sections: Safety and Self-Update remain literal core prompt text, provider section overrides are limited to three provider-owned ids, and the related implementation attempts are either closed-unmerged or still incomplete/open.

## What This Changes

Request to make fixed OpenClaw system prompt sections editable, for example through SAFETY.md-style section overrides or a template mechanism.

## Best Possible Solution

Decide the approved prompt customization boundary; if accepted, add a narrow non-executable section override contract with schema/docs/tests, and if rejected, document the existing whole-prompt, hook, bootstrap/context, and provider-overlay paths as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives high-confidence verification that SAFETY.md-style files are not recognized and OpenClaw-owned sections remain fixed prompt text.

## Solution Assessment

Unclear. Section-level editability may be useful, but broad MDX/template editing would create a larger trust surface; a narrow documented non-executable override contract would be safer if maintainers approve the capability.
