## Summary

Keep open. Current main still lacks editable named OpenClaw-owned system prompt sections; Safety, CLI guidance, and Self-Update remain literal core prompt text, while existing customization is limited to whole-prompt replacement, hooks/context injection, and provider-owned overrides for three narrow section ids.

## What This Changes

Requests a user-editable or templateable way to override fixed OpenClaw system prompt sections such as Safety and system tips, for example with a SAFETY.md-style file.

## Best Possible Solution

Decide the product/security boundary for editable core prompt sections; if accepted, add a narrow non-executable override contract with schema or workspace docs and regression tests, and if rejected, document the existing whole-prompt, hook, bootstrap/context, and provider-overlay paths as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives high-confidence verification that SAFETY.md-style files are not recognized and the requested core sections remain literal prompt text.

## Solution Assessment

Unclear. Section-level editability is plausible, but broad MDX/template editing would create a larger trust surface; a narrow documented non-executable section override contract would be safer if maintainers approve the capability.
