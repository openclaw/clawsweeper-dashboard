## Summary

Keep open. Current main still lacks editable named OpenClaw-owned system prompt sections; Safety, OpenClaw Self-Update, and related core guidance remain literal prompt text, while related PRs are either closed-unmerged or still open with unresolved API/security gaps.

## What This Changes

Request to let users edit fixed OpenClaw system prompt sections, for example through SAFETY.md-style section overrides or a prompt template mechanism.

## Best Possible Solution

Decide the approved prompt customization boundary; if accepted, add a narrow non-executable section override contract with docs and regression tests, and if rejected, document the existing whole-prompt, hook, bootstrap/context, and provider-overlay paths as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives high-confidence verification that SAFETY.md-style section files are not recognized and the requested core sections remain literal prompt text.

## Solution Assessment

Unclear. Section-level editability may be useful, but broad MDX/template editing would create a larger trust surface; a narrow documented non-executable override contract would be safer if maintainers approve the capability.
