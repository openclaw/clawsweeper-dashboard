## What This Changes

Request to make hardcoded OpenClaw system prompt sections editable, for example through SAFETY.md-style files or a prompt template surface.

## Best Possible Solution

Settle the supported prompt customization boundary; if accepted, implement a narrow non-executable section override contract with docs, trust-boundary guidance, and regression tests, otherwise document the existing escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, latest-release inspection, and related GitHub state give high-confidence verification that SAFETY.md-style or section-level editing is not supported on current main.

## Solution Assessment

Unclear. Broad template or MDX editability would expand the prompt trust surface, so a maintainer-approved non-executable section override contract is the safer direction if the capability is accepted.
