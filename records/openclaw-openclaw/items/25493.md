## Summary

Keep open: current main and the latest release still do not provide section-level editing for OpenClaw-owned system prompt sections. Existing supported paths are whole-prompt replacement, plugin hook mutation, and narrow provider-owned section overrides, while the directly related implementation attempts remain closed and unmerged.

## What This Changes

Request to make OpenClaw-owned system prompt sections such as Safety and system tips editable, potentially through workspace markdown files or a templating surface.

## Best Possible Solution

Maintainers should decide whether core sections become schema-backed or workspace-file-backed overrides; if approved, implement that with safety-boundary docs and prompt regression tests, otherwise document the existing escape hatches as the supported customization model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, latest-release inspection, and related GitHub state give high confidence that section-level prompt editing is still missing.

## Solution Assessment

Unclear. Section-level editability is plausible, but changing Safety and OpenClaw-owned prompt sections needs maintainer product and security-boundary judgment before choosing schema-backed config, workspace files, or documentation-only closure.
