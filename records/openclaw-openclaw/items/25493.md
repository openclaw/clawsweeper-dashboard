## Summary

Keep open. Current main still does not expose editable named core prompt sections: Safety, OpenClaw CLI Quick Reference, and Self-Update remain OpenClaw-owned prompt text, while existing customization surfaces are whole-prompt replacement, hook/context additions, channel snippets, or provider overrides for only three narrow section ids.

## What This Changes

The issue asks for OpenClaw-owned system prompt sections such as Safety to be editable, for example through workspace files like `SAFETY.md` or a template mechanism.

## Best Possible Solution

Decide the product/security boundary for editable core prompt sections; if accepted, add a narrow documented override mechanism with regression tests and trust-boundary docs, and if rejected, document the existing whole-prompt, hook, bootstrap, context-engine, channel, and provider-overlay paths as the intentional customization model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high-confidence verification that `SAFETY.md`-style files are not recognized and the requested core sections remain literal prompt text.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer product/security judgment; a narrow non-executable section override contract is safer than broad MDX-style templating unless maintainers intentionally choose that larger surface.
