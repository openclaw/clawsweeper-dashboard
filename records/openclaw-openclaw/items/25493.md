## Summary

Keep open. Current main still does not provide SAFETY.md-style or section-level user editing for OpenClaw-owned system prompt sections; the available paths are whole-prompt replacement, prompt-mutating hooks, provider-owned overlays for a small section set, and workspace bootstrap files that do not include the requested section names.

## What This Changes

The issue requests editable OpenClaw system prompt templates, especially fixed OpenClaw-owned sections such as Safety.

## Best Possible Solution

Settle the approved prompt customization boundary; if accepted, implement a narrow non-executable section override contract with docs and regression tests, otherwise document the existing escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and live GitHub state give high-confidence verification that SAFETY.md-style or section-level editing is not supported on current main.

## Solution Assessment

Unclear. Broad template or MDX editability would expand the prompt trust surface, so a maintainer-approved non-executable section override contract is the safer direction if the capability is accepted.
