## Summary

Keep open: current main and v2026.5.20 still do not provide section-level editing for OpenClaw-owned system prompt sections. The supported paths remain whole-prompt replacement, hook mutation, and narrow provider-owned overrides, while related implementation attempts are closed unmerged or only cover narrower slices.

## What This Changes

Request to add editable system prompt template sections, such as Safety/system tips, possibly through SAFETY.md or a template language.

## Best Possible Solution

Maintainers should decide whether core sections become schema-backed or workspace-file-backed overrides; if approved, implement that with explicit safety-boundary docs and prompt regression tests, otherwise document the existing escape hatches as the supported customization model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, release inspection, and related GitHub state give high confidence that section-level prompt editing is still missing.

## Solution Assessment

Unclear. Section-level editability is plausible, but changing Safety and OpenClaw-owned prompt sections needs maintainer product and security-boundary judgment before choosing schema-backed config, workspace files, or documentation-only closure.
