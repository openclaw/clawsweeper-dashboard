## Summary

Keep open. Current main and v2026.6.6 still hardcode the requested OpenClaw-owned prompt sections, while the directly related implementation attempts are closed unmerged and the remaining question is a product/security boundary decision.

## What This Changes

The issue asks OpenClaw to make fixed system prompt sections or templates user-editable, for example through SAFETY.md or a template override mechanism.

## Best Possible Solution

Define one canonical section-override contract with an explicit allowlist, deterministic precedence, opt-in trust boundary, docs, and prompt regression tests, or document the existing hook/provider-overlay surfaces as the intended boundary.

## Reproduction Assessment

Yes at source level, though this is a feature request rather than a bug. Current main still renders core prompt sections from literals and exposes only a narrow provider-owned section override surface.

## Solution Assessment

Unclear. A SAFETY.md-style override is plausible, but the best fix needs maintainer choice on schema-backed versus workspace-file-backed overrides, precedence, and the trust boundary for safety text.
