## Summary

Keep open. Current main still does not expose editable named OpenClaw-owned system prompt sections such as Safety or Self-Update, the direct workspace-file implementation PR is closed unmerged, and the newer identity-line PR covers only a narrower adjacent prompt-control slice.

## What This Changes

Request to make fixed OpenClaw system prompt template sections editable, for example through SAFETY.md or a template mechanism.

## Best Possible Solution

Decide the approved prompt customization boundary; if accepted, add a narrow non-executable section override contract with schema or workspace-file docs and regression tests, and if rejected, document the existing whole-prompt, hook, bootstrap, and provider-overlay paths as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives high-confidence verification that SAFETY.md-style files are not recognized and OpenClaw-owned sections remain fixed prompt text.

## Solution Assessment

Unclear. Section-level editability may be useful, but broad MDX/template editing would create a larger prompt trust surface; a narrow documented non-executable override contract would be safer if maintainers approve the capability.
