## Summary

Keep open. Current main still lacks editable named OpenClaw-owned system prompt sections; Safety, CLI guidance, and Self-Update remain literal core prompt text, while the available customization surfaces are whole-prompt replacement, prompt hooks, bootstrap/context injection, and provider-owned overrides for only three narrow section ids.

## What This Changes

The issue requests editable/customizable OpenClaw-owned system prompt sections, for example allowing a `SAFETY.md` or template-backed override for fixed prompt text.

## Best Possible Solution

Decide the product/security boundary for editable core prompt sections; if accepted, add a narrow non-executable override contract with schema/docs/tests, and if rejected, document the existing whole-prompt, hook, bootstrap/context, and provider-overlay paths as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives high-confidence verification that `SAFETY.md`-style files are not recognized and the requested core sections remain literal prompt text.

## Solution Assessment

Unclear. Section-level editability is plausible, but broad MDX/template editing would create a larger trust surface; a narrow documented section override contract would be safer if maintainers approve the capability.
