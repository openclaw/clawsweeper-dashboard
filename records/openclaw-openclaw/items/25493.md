## Summary

Keep open. Current main and v2026.5.12 still hardcode the requested OpenClaw-owned prompt sections, while existing customization paths cover whole-prompt replacement, hooks, bootstrap context, and provider-owned overrides for only three named sections. The directly related implementation PR is closed unmerged, and the adjacent open config-injection issue does not replace the section-level Safety/template decision here.

## What This Changes

The issue requests user-editable templates or files for OpenClaw-owned system prompt sections such as Safety instead of fixed prompt text.

## Best Possible Solution

Settle the approved prompt customization boundary; if accepted, add a narrow non-executable section override contract with schema or workspace-file semantics, cache handling, public docs, and regression tests, otherwise document the existing whole-prompt, hook, bootstrap, and provider-overlay escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, release inspection, and related-item review give high-confidence verification that SAFETY.md-style files and user-editable OpenClaw-owned sections are not currently supported.

## Solution Assessment

Unclear. Broad template or MDX editability would expand the prompt trust surface; a maintainer-approved non-executable section override model would be safer if the capability is accepted.
