## Summary

Keep open. Current main still lacks editable named OpenClaw-owned system prompt sections: Safety, CLI guidance, and Self-Update are literal prompt text, while existing customization is limited to whole-prompt replacement, prompt hooks, context/bootstrap injection, and provider-owned overrides for three narrow section ids.

## What This Changes

Make fixed OpenClaw-owned system prompt sections, especially Safety/system tips, editable through section-level templates or files such as `SAFETY.md`.

## Best Possible Solution

Decide the product/security boundary for editable core prompt sections; if accepted, add a narrow non-executable override contract with schema/docs/tests, and if rejected, document the supported whole-prompt, hook, bootstrap/context, and provider-overlay paths as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source/docs inspection on current main gives high-confidence verification that `SAFETY.md`-style files are not recognized and the requested core sections remain literal prompt text.

## Solution Assessment

Unclear. Section-level editability is a plausible direction, but broad MDX/template editing would be a larger trust surface; a narrow documented section override contract would be safer if maintainers approve the capability.
