## Summary

Keep open. Current main and v2026.6.10 still hardcode OpenClaw-owned prompt sections, while the available mechanisms are provider-owned section overlays, plugin/hooks, workspace bootstrap files, or narrower config proposals rather than the requested editable Safety/system-template contract.

## What This Changes

The issue requests user-editable OpenClaw system prompt sections, such as allowing SAFETY.md or a template language to override fixed prompt text.

## Best Possible Solution

Define one canonical, maintainer-approved section override contract with explicit trust boundaries, precedence, docs, schemas or workspace-file rules, and regression coverage; otherwise document the existing hook/provider/bootstrap paths as the supported alternatives.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives a high-confidence feature-gap reproduction: current main and v2026.6.10 have literal core sections and no SAFETY.md-style or named user section override contract.

## Solution Assessment
