## Summary

Keep open. Current main and v2026.6.10 still hardcode OpenClaw-owned Safety, Tool Call Style, and Self-Update prompt sections, while the available hook/provider/workspace mechanisms remain narrower than the requested editable system-prompt-template contract.

## What This Changes

Request to make fixed OpenClaw system prompt sections editable, such as by allowing workspace files like SAFETY.md or a template mechanism to override built-in prompt text.

## Best Possible Solution

Define one canonical, maintainer-approved section override contract with explicit trust boundaries, precedence, docs, schema or workspace-file rules, and prompt regression coverage; if rejected, document the existing hook/provider/bootstrap paths as the supported alternatives.

## Reproduction Assessment

Yes for the source-level feature gap, not as a bug reproduction: current main and v2026.6.10 still have literal core prompt sections and no SAFETY.md-style or named user section override contract.

## Solution Assessment

Unclear until maintainers decide the product/security boundary. The narrow maintainable path is one canonical override model, not broad MDX execution, ad hoc file names, or another partial prompt knob.
