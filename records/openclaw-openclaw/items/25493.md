## Summary

Keep open. Current main and the latest release still keep the requested OpenClaw-owned prompt sections as fixed core prompt text; existing customization is whole-prompt replacement, plugin/hook mutation, or provider-owned overrides for three narrow sections, while the direct workspace-file implementation was closed unmerged and adjacent work does not cover Safety/template editability.

## What This Changes

The issue requests editable OpenClaw system prompt template sections, especially fixed Safety and system guidance, potentially through files such as `SAFETY.md` or a template language.

## Best Possible Solution

Decide the approved prompt customization boundary; if accepted, add a narrow non-executable section override contract with schema/docs, prompt-cache handling, and regression tests, otherwise document the existing escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, release-tag inspection, and related-item review give high-confidence verification that `SAFETY.md`-style files are not recognized and OpenClaw-owned sections remain fixed prompt text.

## Solution Assessment

Unclear. Section-level editability may be useful, but broad template or MDX editing would create a larger trust surface; a narrow documented non-executable override contract would be safer if maintainers approve the capability.
