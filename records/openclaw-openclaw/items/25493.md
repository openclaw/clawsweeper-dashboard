## Summary

Keep open. Current main still hardcodes the requested OpenClaw-owned prompt sections and only offers whole-prompt replacement, prompt hooks, and provider-owned overrides for three named sections; related PR #25938 is closed unmerged and #53942 was closed as a duplicate back here.

## What This Changes

Request section-level customization of OpenClaw-owned system prompt content such as Safety, tool-call style, CLI quick reference, and self-update guidance.

## Best Possible Solution

Decide the product/security boundary for editable core prompt sections; if accepted, add a documented narrow section override mechanism with regression tests, and if rejected, document the existing whole-prompt, hook, bootstrap, and provider-overlay paths as the supported customization model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path: requested SAFETY.md-style files are not recognized, and the typed provider section override contract excludes Safety, CLI Quick Reference, and Self-Update.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer product/security judgment; a narrow explicit section override model would be safer than broad MDX-style templating unless maintainers intentionally want a larger template surface.
