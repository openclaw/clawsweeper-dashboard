## Summary

Keep open. Current main still has fixed OpenClaw-owned prompt sections and only supports whole-prompt replacement, prompt-mutating hooks, and narrow provider-owned overrides, so the requested section/template editability remains an unresolved product and security decision.

## What This Changes

Request to make fixed OpenClaw system prompt sections editable, such as via SAFETY.md or a template layer, instead of requiring source patches or whole-prompt replacement.

## Best Possible Solution

Settle the approved prompt customization boundary; if accepted, implement a narrow non-executable section override contract with docs and regression tests, otherwise document the current escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and related GitHub state give high-confidence verification that SAFETY.md-style or section-level editing is not currently supported on main.

## Solution Assessment

Unclear. Broad template or MDX editability would expand the prompt trust surface; a maintainer-approved non-executable section override contract would be safer if the capability is accepted.
