## Summary

Keep open. Current main and v2026.6.8 still do not provide user-editable OpenClaw-owned system prompt section templates; the remaining blocker is a product/security decision about whether Safety-style sections should become overrideable and under what trust boundary.

## What This Changes

The issue requests editable OpenClaw-owned system prompt template sections, such as replacing hardcoded Safety content with a workspace file or template override.

## Best Possible Solution

Define one canonical, maintainer-approved section override contract with explicit allowlist, deterministic precedence, trust-boundary docs, and regression coverage, or document the existing whole-prompt, hook, and provider-overlay surfaces as the intended boundary.

## Reproduction Assessment

Yes at source level, though this is a feature request rather than a bug. Current main and v2026.6.8 still render core prompt sections from literals and expose only a narrow provider-owned section override surface.

## Solution Assessment

Unclear. A `SAFETY.md`-style override is plausible, but the best fix needs maintainer choice on schema-backed versus workspace-file-backed overrides, precedence, runtime coverage, prompt-cache stability, and safety trust boundaries.
