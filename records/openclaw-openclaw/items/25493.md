## Summary

Keep open. Current main and v2026.6.8 still hardcode OpenClaw-owned system prompt sections, while the available prompt mutation paths are whole-prompt, hook/plugin, provider-owned, channel-specific, or narrower than the requested editable section-template capability.

## What This Changes

The issue requests a user-editable template or override surface for fixed OpenClaw system prompt sections, such as a SAFETY.md-style override.

## Best Possible Solution

Define one canonical, maintainer-approved section override contract with explicit trust boundaries, precedence, docs, schemas or workspace file rules, and regression coverage; otherwise document existing hook/provider/channel paths as the supported alternatives.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives a high-confidence feature-gap reproduction: current main and v2026.6.8 have literal core sections and no SAFETY.md-style or named user section override contract.

## Solution Assessment

Unclear until maintainers decide the product/security boundary. The narrowest maintainable solution is a single canonical override model rather than ad hoc file names, broad MDX execution, or another partial prompt knob.
