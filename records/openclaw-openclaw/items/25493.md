## Summary

Keep open. Current main still does not provide editable named OpenClaw-owned prompt sections; Safety, CLI guidance, and Self-Update remain literal core prompt text, and the existing customization paths are either whole-prompt replacement, hooks, bootstrap/context injection, or provider-owned overrides for only three narrow section ids.

## What This Changes

Request to make OpenClaw-owned system prompt template sections editable, for example through SAFETY.md or a template mechanism.

## Best Possible Solution

Decide the product/security boundary for editable core prompt sections; if accepted, add a narrow non-executable override mechanism with docs and regression tests, and if rejected, document existing whole-prompt, hook, bootstrap, context-engine, channel, and provider-overlay customization paths as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source/docs inspection on current main gives high-confidence verification that `SAFETY.md`-style files are not recognized and the requested core sections remain literal prompt text.

## Solution Assessment

Unclear. The requested direction is plausible, but broad template editing such as MDX would be a larger trust surface; a narrow documented section override contract would be safer if maintainers approve the capability.
