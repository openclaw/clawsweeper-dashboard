## Summary

Keep open. Current main and v2026.5.12 still lack section-level or SAFETY.md-style editing for OpenClaw-owned prompt sections; this remains the canonical tracker after the direct PR closed unmerged, the duplicate was folded here, and newer related work covers only narrower identity/config-prompt surfaces.

## What This Changes

Request to make fixed OpenClaw-owned system prompt sections editable, for example through SAFETY.md or template-style overrides.

## Best Possible Solution

Decide the approved prompt customization boundary; if accepted, add a narrow non-executable section override contract with schema/docs/cache handling and regression tests, otherwise document existing promptOverride, hooks, and provider overlays as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, release inspection, and related-item review give high-confidence verification that SAFETY.md-style files and user-editable OpenClaw-owned sections are not currently supported.

## Solution Assessment

Unclear. Broad template or MDX editability would expand the trust surface; a narrower documented, non-executable section override model would be safer if maintainers approve the capability.
