## Summary

Keep open. Current main at 542821cd still does not provide section-level editing for the OpenClaw-owned prompt sections named by the report; Safety, CLI Quick Reference, and Self-Update remain core literals, while existing customization is limited to full prompt replacement, prompt hooks, provider overlays, and three provider-owned section overrides. The related implementation PR #25938 is closed and unmerged, #53942 is an open narrower follow-up, and the recent maintainer-side comment explicitly kept this for product/security judgment.

## What This Changes

The issue asks for OpenClaw-owned system prompt sections, especially Safety/system tips, to be editable through a mechanism such as workspace files or a template language.

## Best Possible Solution

Decide the product/security boundary for editable OpenClaw-owned prompt sections. If accepted, add a documented, schema-backed or workspace-file-backed section override mechanism with tests for Safety, Tool Call Style, CLI Quick Reference, and Self-Update; if rejected, document the supported alternatives and explicitly coordinate that answer with #53942.
