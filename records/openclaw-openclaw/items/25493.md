## Summary

Keep open. Current main still lacks SAFETY.md-style or section-level editing for OpenClaw-owned system prompt sections; the supported surfaces are whole-prompt replacement, prompt-mutating hooks, and provider-owned overrides for only three named sections. The direct implementation attempts are closed unmerged, while related open prompt-control work is adjacent rather than a shipped replacement.

## What This Changes

Request support for editing OpenClaw-owned system prompt template sections, such as Safety, with a workspace file or template-style override surface.

## Best Possible Solution

Settle the approved prompt customization boundary; if accepted, implement a narrow non-executable section override surface with schema or workspace-file semantics, cache handling, public docs, and regression tests, otherwise document the current escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, related PR state, and targeted searches give high-confidence verification that SAFETY.md-style files and user-editable OpenClaw-owned sections are not currently supported.

## Solution Assessment

Unclear. Broad MDX or template editability would expand the prompt trust surface; a maintainer-approved, non-executable section override contract would be safer if the capability is accepted.
