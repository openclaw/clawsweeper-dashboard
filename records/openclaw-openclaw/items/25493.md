## Summary

Keep open. Current main and v2026.5.12 still lack SAFETY.md-style or section-level editing for OpenClaw-owned system prompt sections; the available surfaces are whole-prompt replacement, hook/runtime context mutation, bootstrap context, and provider-owned overrides for only three named sections. The directly related implementation at https://github.com/openclaw/openclaw/pull/25938 and the broader disabled-section attempt at https://github.com/openclaw/openclaw/pull/48442 are closed unmerged, while https://github.com/openclaw/openclaw/issues/50103 and https://github.com/openclaw/openclaw/pull/82165 are adjacent prompt-control work rather than a shipped replacement for this decision.

## What This Changes

Request to make fixed OpenClaw system prompt sections editable, for example through SAFETY.md or templated section overrides.

## Best Possible Solution

Settle the approved prompt customization boundary; if accepted, implement a narrow non-executable section override surface with schema or workspace-file semantics, cache handling, public docs, and regression tests, otherwise document current escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, related PR state, and release inspection give high-confidence verification that SAFETY.md-style files and user-editable OpenClaw-owned sections are not currently supported.

## Solution Assessment

Unclear. Broad MDX or template editability would expand the prompt trust surface; a maintainer-approved, non-executable section override contract would be safer if the capability is accepted.
