## Summary

Keep open. Current main and the latest release still do not provide user-editable OpenClaw-owned prompt sections; the directly related implementation attempts are closed unmerged, and the narrower related issue was closed in favor of this issue as the canonical product/security decision.

## What This Changes

The issue requests editable or overrideable OpenClaw-owned system prompt template sections, especially fixed safety guidance in `src/agents/system-prompt.ts`.

## Best Possible Solution

Define one canonical, documented section-override contract with an explicit allowlist, deterministic precedence, opt-in trust boundary, docs, and prompt regression tests; otherwise document the existing hook and provider-overlay surfaces as the intended customization boundary.

## Reproduction Assessment

Yes at source level, though this is a feature request rather than a bug: current main still renders core prompt sections from literals and only exposes a narrow provider-owned section override surface.

## Solution Assessment

Unclear. A `SAFETY.md`-style workspace override is plausible, but the best solution needs maintainer choice on whether to add an allowlisted section-override contract or keep prompt customization limited to hooks/provider overlays.
