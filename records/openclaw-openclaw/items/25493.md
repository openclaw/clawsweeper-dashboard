- `impact:security`: The request explicitly includes replacing the core Safety section, which changes the prompt trust boundary and secure defaults.

## Summary

Current main and v2026.6.5 still keep Safety and other OpenClaw-owned prompt content in the core builder, while the supported provider override contract remains limited to three named behavioral sections. The related implementation PRs were closed unmerged, so the capability remains a live product/security decision rather than conservative cleanup.

## What This Changes

Allow users to edit or override OpenClaw-owned system prompt sections, potentially through workspace files or templates.

## Best Possible Solution

Define a narrow, typed, text-only customization contract with explicit precedence and immutable security invariants; either expose approved sections through a reviewed plugin/config boundary or document that core Safety remains non-overridable.

## Reproduction Assessment

Not applicable: this issue requests a new customization capability, and current source plus v2026.6.5 consistently show that arbitrary core-section editing is not an existing contract.

## Solution Assessment

No, not as proposed: making the prompt editable “as much as possible” or adding MDX is too broad for a security-sensitive core surface. A typed, text-only, explicitly scoped override or contribution contract is the safer maintainable direction if maintainers approve it.
