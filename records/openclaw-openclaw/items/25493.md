## Summary

## What This Changes

Request to let users edit fixed OpenClaw system-prompt sections such as Safety, potentially through workspace files or templates.

## Best Possible Solution

Define a narrow typed text-only section customization contract with explicit precedence, allowed sections, security invariants, docs, and regression tests, or document that core Safety remains non-overridable.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source, docs, tests, related PR state, and v2026.6.5 inspection show the requested section-level editing is not currently supported.

## Solution Assessment

No, not as broadly proposed: making the prompt editable as much as possible or adding MDX is too open-ended for a security-sensitive prompt surface. A typed, text-only, explicitly scoped override or contribution contract is the safer path if maintainers approve it.
