## Summary

Keep open. Current main still does not provide section-level editing for the OpenClaw-owned prompt sections named by the report: Safety, CLI Quick Reference, and Self-Update remain core literals, while the existing customization surfaces are full-prompt replacement, prompt hooks, provider overlays, and three provider-owned section overrides. The direct implementation PR #25938 is closed and unmerged, and #53942 was closed as a duplicate back to this issue, so this remains the canonical product/security decision.

## What This Changes

The issue requests making hardcoded OpenClaw system prompt template sections, especially Safety, editable through a configurable or file-backed override mechanism.

## Best Possible Solution

Decide the product/security boundary for editable OpenClaw-owned prompt sections. If accepted, add a documented, schema-backed or workspace-file-backed section override mechanism with regression tests for Safety, Tool Call Style, CLI Quick Reference, and Self-Update; if rejected, document whole-prompt replacement, before_prompt_build, agent:bootstrap, and provider overlays as the supported customization paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path: SAFETY.md-style files are not recognized by workspace loading, and the provider section id type does not include Safety, CLI Quick Reference, or Self-Update.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer product/security judgment; a narrow explicit section override model would be more maintainable than broad MDX-style templating unless maintainers intentionally want a larger template surface.
