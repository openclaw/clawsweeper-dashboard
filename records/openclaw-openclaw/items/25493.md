## Summary

Keep open: current main and the latest release still do not support SAFETY.md-style or section-level editing for OpenClaw-owned system prompt sections, and the remaining choice is a product/security boundary rather than conservative cleanup.

## What This Changes

The issue requests editable OpenClaw system prompt templates or sections, such as overriding Safety with workspace files or a template mechanism.

## Best Possible Solution

Settle the approved prompt customization boundary; if accepted, implement a narrow non-executable section override contract with docs and regression tests, otherwise document the existing escape hatches as intentional.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, latest-release inspection, and related GitHub state give high-confidence verification that SAFETY.md-style or section-level editing is not supported on current main.

## Solution Assessment

Unclear. Broad template or MDX editability would expand the prompt trust surface, so a maintainer-approved non-executable section override contract is the safer direction if the capability is accepted.
