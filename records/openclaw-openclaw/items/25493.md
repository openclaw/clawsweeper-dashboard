## Summary

Keep open. Current main still does not expose editable Safety, CLI Quick Reference, or Self-Update prompt sections, and the remaining work is a feature with product/security implications rather than cleanup.

## What This Changes

Request to make OpenClaw's fixed system prompt sections, especially Safety, editable through a supported template or override mechanism such as `SAFETY.md`.

## Best Possible Solution

Decide the product/security boundary for editable core prompt sections; if accepted, add a documented narrow override mechanism with regression tests, and if rejected, document the existing whole-prompt, hook, bootstrap, and provider-overlay paths as the supported customization model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high-confidence verification that `SAFETY.md`-style files are not recognized and Safety, CLI Quick Reference, and Self-Update remain literal core prompt sections.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer product/security judgment; a narrow section override contract would be safer than broad MDX-style templating unless maintainers intentionally want that larger surface.
