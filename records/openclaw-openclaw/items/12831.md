## Summary

Keep this open: current main has useful backend tool-event plumbing, but it still lacks the requested first-class Activity panel / observer mode and the remaining work needs product and privacy scoping.

## What This Changes

Add a separate activity/observer surface for tool calls so users can monitor sensitive agent work without putting verbose output in the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over existing `tool-events` and `session.tool` plumbing, with TUI, per-connection verbose state, and external streams handled as deliberate follow-ups.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence check that the independent observer surface is absent while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a staged observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for TUI, per-connection verbose state, and external streams.
