## Summary

Keep open. Current main has backend `tool-events` and `session.tool` plumbing plus live tool cards in chat, but the requested separate Activity panel/observer mode is still not implemented; the overlapping implementation at https://github.com/openclaw/openclaw/pull/60814 remains an open draft with review blockers.

## What This Changes

Request a separated activity/tool-call observer surface so users can monitor agent activity without putting verbose output in the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker and ship a scoped Control UI Activity panel over existing `tool-events` and `session.tool` plumbing first, then split per-connection verbose, external stream, and TUI work into follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/state, tool events still become chat tool messages, and TUI verbose-off behavior still suppresses tool cards.

## Solution Assessment

No complete solution is on main yet. The narrow maintainable first slice is likely a dedicated Control UI Activity panel backed by existing event plumbing, coordinated with https://github.com/openclaw/openclaw/pull/60814 before broader TUI, external-stream, or per-connection verbose work proceeds.
