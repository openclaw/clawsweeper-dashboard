## Summary

Keep open. Current main has partial backend observer plumbing, but it still lacks the requested first-class Activity panel or separated observer mode, and the overlapping activity-dashboard work remains an open draft at https://github.com/openclaw/openclaw/pull/60814.

## What This Changes

Requests a separated Activity/observer surface for tool calls, with possible Control UI panel, per-connection verbose state, subscriber stream, and TUI split pane.

## Best Possible Solution

Keep this as the broad observer-mode tracker and first ship a scoped, privacy-reviewed Control UI Activity panel over existing `tool-events` and `session.tool` plumbing, with TUI, external stream, Telegram parity, and per-connection verbose behavior split into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/state, tool events still become chat tool messages, and TUI tool display remains verbose-gated.

## Solution Assessment

No complete solution is on main yet. The safest direction is not a broad automated dashboard drop, but a maintainer-scoped Activity panel that reuses current event plumbing and coordinates with https://github.com/openclaw/openclaw/pull/60814.
