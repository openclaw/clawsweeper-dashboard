## Summary

Keep open. Current main has backend tool-event plumbing and chat-rendered live tool cards, but it still lacks a first-class Activity panel or observer mode separate from the chat stream, and the related draft PR remains unmerged.

## What This Changes

Request a separate tool-call Activity panel or observer mode, with possible per-connection verbose settings, external activity stream, and TUI split-pane support.

## Best Possible Solution

Keep this as the canonical observer-mode tracker; ship a dedicated Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing, then split per-connection verbose, TUI, and external-stream work into focused follow-ups if still desired.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source-inspection path for the missing behavior. Current main routes live tool activity into chat state, lacks a Control UI `session.tool` activity handler, and keeps TUI tool rendering tied to verbose state.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice appears to be a Control UI Activity panel using existing backend event plumbing, with maintainers reviewing or splitting draft PR #60814 before implementation proceeds.
