## Summary

Keep open. Current main has useful backend `tool-events` and `session.tool` plumbing plus live tool cards in chat, but it still lacks the requested first-class Activity panel, observer mode, per-connection monitoring model, external activity stream, and TUI split pane; the closest implementation, https://github.com/openclaw/openclaw/pull/60814, is still an open draft.

## What This Changes

Request to separate verbose tool-call activity from the main chat stream through a Control UI activity panel, observer stream, per-connection monitoring, and possible TUI split pane.

## Best Possible Solution

Keep this as the broad observer-mode tracker and ship a scoped Control UI Activity panel over existing `tool-events` and `session.tool` plumbing first, then split per-connection verbose, external stream, TUI, and channel-specific visibility into follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: there is no Activity route/state, tool events still become `chatToolMessages`, and TUI verbose-off behavior still suppresses tool cards.

## Solution Assessment

No complete solution is on main yet. The narrow maintainable first slice is likely a dedicated Control UI Activity panel backed by existing event plumbing, coordinated with https://github.com/openclaw/openclaw/pull/60814 before broader TUI, external stream, or per-connection verbose work proceeds.
