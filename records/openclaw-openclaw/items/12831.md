## Summary

Keep open. Current main has useful backend event plumbing, but it still lacks the requested separate Activity panel/observer mode; tool events are still modeled into the chat stream, and the related dashboard implementation remains an open draft with prior review blockers.

## What This Changes

The issue requests a separate observer/activity surface for tool calls so verbose agent activity can be monitored without cluttering the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker and ship a scoped, privacy-reviewed Control UI Activity panel over existing tool-events/session.tool plumbing, with narrower follow-ups for TUI, external observer streams, Telegram parity, and per-connection verbose state.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/state, tool events still become chat tool messages, and TUI tool display remains verbose-gated.

## Solution Assessment

No complete solution is on main yet. The best path is a smaller maintainer-approved Activity panel that reuses existing event plumbing and coordinates with https://github.com/openclaw/openclaw/pull/60814 before expanding to other surfaces.
