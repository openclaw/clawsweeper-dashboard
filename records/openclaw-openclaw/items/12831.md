## Summary

Keep this open: current main has useful backend tool-event plumbing and chat-rendered tool cards, but the requested first-class Activity panel / observer mode is still missing and overlaps an open draft implementation PR that maintainers need to scope.

## What This Changes

The issue requests a separate activity/observer surface for tool calls so users can monitor sensitive agent work without mixing verbose output into the main chat.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over existing `tool-events` / `session.tool` plumbing, with TUI split view, per-connection verbose behavior, and external observer streams handled as deliberate follow-ups.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection of current main gives a high-confidence check that the requested independent observer surface is absent while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a staged, maintainer-scoped observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for TUI, per-connection verbose state, and external streams.
