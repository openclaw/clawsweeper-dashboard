## Summary

Keep this open as the broad observer-mode tracker: current main has backend tool-event plumbing and chat-rendered tool cards, but not the requested independent Activity panel, observer state, TUI split, or external observer stream.

## What This Changes

The issue requests separating verbose tool activity from the main chat stream through an Activity panel, connection-scoped observer state, external tool-activity stream, or TUI split pane.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over the existing `tool-events` / `session.tool` plumbing, with TUI split view, per-connection verbose behavior, and external observer streams handled as deliberate follow-ups.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a staged, maintainer-scoped observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for TUI, per-connection verbose state, and external streams.
