## Summary

Keep open. Current main still has only partial backend observer plumbing: tool events can reach registered WebSocket/session subscribers, but Control UI still renders live tool activity inside Chat, TUI tool cards remain verbose-gated, and the related dashboard implementation remains an open draft rather than shipped behavior.

## What This Changes

Request a separated activity/tool-call observer surface so users can monitor verbose tool output without cluttering the main chat stream.

## Best Possible Solution

Ship a dedicated Control UI Activity panel backed by the existing `tool-events` and `session.tool` plumbing, then split per-connection verbose settings, TUI split-pane behavior, and external subscriber streams into focused follow-ups if still desired.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: tool events become `chatToolMessages`, Chat renders them inline, and the TUI suppresses tool cards unless verbose is enabled.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice is a Control UI Activity panel over existing backend event plumbing, with maintainers reviewing or splitting https://github.com/openclaw/openclaw/pull/60814 before broader TUI, external stream, or per-connection setting work proceeds.
