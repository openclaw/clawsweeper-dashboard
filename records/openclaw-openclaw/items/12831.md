## Summary

Keep open. Current main has useful `tool-events` and `session.tool` backend plumbing, but it still lacks the requested separate Activity panel, observer mode, per-connection monitoring, external activity stream, and TUI split-pane; the nearest implementation remains an open draft PR.

## What This Changes

The issue requests a separate activity/observer surface for tool calls, with Control UI, per-connection verbose, external stream, and TUI split-pane options.

## Best Possible Solution

Keep this as the broad observer-mode tracker and ship a scoped Control UI Activity panel over existing `tool-events`/`session.tool` plumbing before splitting TUI, external stream, and per-connection setting follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: tool events still become `chatToolMessages`, Chat renders them inline, and the TUI suppresses tool cards when verbose is off.

## Solution Assessment

No complete solution is on main yet. The maintainable first slice is likely a dedicated Control UI Activity panel backed by existing event plumbing, coordinated with https://github.com/openclaw/openclaw/pull/60814 before broader TUI, external stream, or per-connection verbose work proceeds.
