## Summary

Keep open. Current main has useful backend observer plumbing and live chat tool cards, but it still lacks the requested first-class Activity panel or observer mode separate from the chat stream; maintainer discussion already treats this as the broad tracker with an open draft implementation PR.

## What This Changes

The issue requests a separate Activity/observer surface for tool calls and related agent activity so verbose output does not clutter the main chat stream.

## Best Possible Solution

Keep this as the canonical observer-mode tracker and ship a dedicated Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing, then split TUI, per-connection verbose, and external-stream work into focused follow-ups if they remain in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat state/cards, has no `session.tool` Activity handler, and keeps TUI tool rendering tied to verbose state.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice appears to be a Control UI Activity panel using existing backend event plumbing, with maintainers reviewing or splitting draft PR #60814 before implementation proceeds.
