## Summary

Keep open. Current main has useful backend event plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the user-facing Activity panel / observer mode is still missing; the linked draft PR #60814 is open and unmerged, and maintainer context treats this as broad product follow-up.

## What This Changes

Request a separate activity/observer surface for tool calls so users can monitor agent work without injecting verbose output into the main conversation.

## Best Possible Solution

Keep this as the canonical observer-mode tracker. Ship a dedicated Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, expand/collapse, and truncation behavior, then split per-connection verbose settings, TUI split-pane, and external observer stream work into deliberate follow-ups if maintainers keep them in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat state/cards, lacks an Activity/observer route, and has TUI behavior/tests that suppress tool events when verbose is off.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first step appears to be a Control UI Activity panel over the existing backend event plumbing, with broader TUI and external-stream work scoped after maintainer review of draft PR #60814.
