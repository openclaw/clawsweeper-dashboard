## Summary

Keep open. Current main has useful backend tool-event plumbing and live chat tool cards, but it still does not provide a first-class Activity panel / observer mode; the related draft implementation PR remains open and maintainer context already treats this as the broad tracker.

## What This Changes

Requested a separate Activity panel / observer mode so users can monitor tool calls and agent activity without cluttering the main conversation.

## Best Possible Solution

Keep this as the canonical observer-mode tracker, ship a dedicated Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing, then split TUI, per-connection verbose, and external-stream work into focused follow-ups if still in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat state/cards, lacks a `session.tool` Activity-panel handler, and keeps TUI tool rendering tied to verbose state.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice appears to be a Control UI Activity panel using the existing backend event plumbing, with draft PR #60814 reviewed or split before implementation proceeds.
