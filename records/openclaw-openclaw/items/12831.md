## Summary

Keep open. Current main has useful backend `tool-events` and `session.tool` plumbing, but it still lacks the requested first-class Activity panel, observer mode, per-connection monitoring model, external activity stream, and TUI split pane; tool events still route into chat rendering today.

## What This Changes

The issue requests a separated activity/tool-call observer surface so users can monitor agent work without putting verbose tool output in the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker and ship a scoped Control UI Activity panel over existing `tool-events`/`session.tool` plumbing before splitting TUI, external stream, and per-connection setting follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: there is no Activity route/state, tool events still become `chatToolMessages`, and TUI verbose-off behavior still suppresses tool cards.

## Solution Assessment

No complete solution is on main yet. The maintainable first slice is likely a dedicated Control UI Activity panel backed by existing event plumbing, coordinated with https://github.com/openclaw/openclaw/pull/60814 before broader TUI, external stream, or per-connection verbose work proceeds.
