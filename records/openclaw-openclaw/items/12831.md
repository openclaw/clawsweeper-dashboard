## Summary

Keep open. Current main has useful backend `tool-events` and `session.tool` plumbing, but the requested first-class Activity panel / observer mode is still missing from the Control UI and TUI, and the related draft implementation PR remains open and unmerged.

## What This Changes

Add an observer/activity mode that separates live tool-call activity from the main chat stream, with possible Control UI, per-connection verbose, external stream, and TUI split-pane surfaces.

## Best Possible Solution

Ship a dedicated Control UI Activity panel backed by the existing `tool-events` and `session.tool` plumbing, then split per-connection verbose, TUI split-pane, and external subscriber work into focused follow-ups if still needed.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives high confidence current main lacks the requested separated observer surface. Tool activity still flows into chat state, the Control UI lacks a `session.tool` activity handler, and TUI tool rendering remains verbose-gated.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice is a Control UI Activity panel over existing backend event plumbing, with maintainers reviewing or splitting draft PR #60814 before implementation proceeds.
