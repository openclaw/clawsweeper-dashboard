## Summary

Keep open. Current main has backend tool-event plumbing and live chat tool cards, but it still does not provide the requested separate Activity panel or observer mode outside the conversation stream.

## What This Changes

Request a separate activity/observer surface for tool calls, including a Control UI panel, per-connection verbose behavior, an external activity stream, and a TUI split pane.

## Best Possible Solution

Keep this as the canonical observer-mode tracker; ship a dedicated Control UI Activity panel on top of existing tool-events/session.tool plumbing, then split TUI, per-connection verbose, and external-stream work into focused follow-ups if they remain in scope.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence missing-behavior check. Current main still routes live tool events into chat state/cards and keeps TUI tool display tied to verbose state.

## Solution Assessment

No, current main is not yet a complete solution. The narrowest maintainable first slice appears to be a Control UI Activity panel using existing backend event plumbing, with maintainers reviewing or splitting draft PR #60814 before broader surfaces proceed.
