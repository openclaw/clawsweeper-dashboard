## Summary

Keep open. Current main has backend tool-event plumbing and chat-rendered live tool cards, but it still lacks the requested first-class observer mode separated from the chat stream, and the related draft implementation PR remains unmerged.

## What This Changes

Request to separate live tool-call activity from the main chat stream through a Control UI Activity panel, per-connection verbose behavior, an observer stream, or a TUI split pane.

## Best Possible Solution

Ship a dedicated Control UI Activity panel backed by existing `tool-events` and `session.tool` plumbing, then split per-connection verbose, TUI, and external subscriber work into focused follow-ups if still needed.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence path for the missing behavior. Current main routes live tool activity into chat state, lacks a Control UI `session.tool` activity handler, and keeps TUI tool rendering tied to verbose state.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice is a Control UI Activity panel over the existing backend event plumbing, with maintainers reviewing or splitting draft PR #60814 before implementation proceeds.
