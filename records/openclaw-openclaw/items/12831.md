## Summary

Keep open. Current main and the latest release still provide partial backend `tool-events`/`session.tool` plumbing and chat-rendered tool cards, but not the requested first-class Activity panel, independent observer model, per-connection verbose state, external activity stream, or TUI split pane.

## What This Changes

Request to separate verbose tool-call activity from the main chat stream through an Activity panel, observer stream, per-connection verbosity, and/or TUI split view.

## Best Possible Solution

Scope and ship a privacy-reviewed Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing first, then split per-connection verbose state, external observer streams, TUI panes, and Telegram parity into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/model, no Control UI `session.tool` consumer, tool events still become chat messages, and TUI display remains verbose-gated.

## Solution Assessment

Unclear until maintainers scope the UX/API/privacy contract. The maintainable path is to reuse existing `tool-events` and `session.tool` plumbing rather than introduce a competing event pipeline or treat per-connection verbose as the primary fix.
