## Summary

Keep open. Current main has useful backend `tool-events` and `session.tool` plumbing, but the requested separated observer surface is still missing from Control UI, TUI, per-connection verbose state, and external activity streaming, and the live issue is already routed for product and security review.

## What This Changes

Requests a separate activity/observer surface for tool calls so verbose details can be monitored outside the main chat stream.

## Best Possible Solution

Scope a privacy-reviewed Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing first, then split per-connection verbose, external stream, and TUI work into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/model, tool events still become chat tool messages, and TUI display remains verbose-gated.

## Solution Assessment

Unclear until maintainers scope the UX/API/privacy contract. The maintainable path is to reuse the existing `tool-events` and `session.tool` plumbing rather than introduce a competing event pipeline.
