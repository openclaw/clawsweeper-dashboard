## Summary

Keep open. Current main has useful backend `tool-events` and `session.tool` plumbing, but the requested separated observer surface is still missing from Control UI, TUI, per-connection verbose state, and external activity streaming; the issue also already carries product/security-review labels and has an open draft implementation thread.

## What This Changes

Request a separated Activity panel or observer mode for tool calls so users can monitor agent activity without mixing verbose output into the main chat.

## Best Possible Solution

Scope and ship a privacy-reviewed Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing first, then split per-connection verbose state, external observer streams, TUI panes, and Telegram parity into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/model, no Control UI `session.tool` consumer, tool events still become chat tool messages, and TUI display remains verbose-gated.

## Solution Assessment

Unclear until maintainers scope the UX/API/privacy contract. The maintainable path is to reuse the existing `tool-events` and `session.tool` plumbing rather than introduce a competing event pipeline or treat per-connection verbose as the primary fix.
