## Summary

Keep open. Current main still only has backend observer plumbing and inline chat/TUI verbose behavior; the requested separate Activity panel, observer mode, and independently scoped monitoring surfaces are not shipped, and the related dashboard work is still an open draft.

## What This Changes

The issue requests a separate activity/observer surface for tool calls so users can monitor verbose agent activity without cluttering the main chat stream.

## Best Possible Solution

Ship a dedicated Control UI Activity panel backed by existing `tool-events` and `session.tool` plumbing, then split per-connection verbose settings, TUI split-pane behavior, and external observer streams into reviewed follow-ups if still desired.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: tool events still become `chatToolMessages`, Chat renders them inline, and the TUI suppresses tool cards unless verbose is enabled.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice is likely a Control UI Activity panel over the existing backend event plumbing, coordinated with https://github.com/openclaw/openclaw/pull/60814 before broader TUI, external stream, or per-connection setting work proceeds.
