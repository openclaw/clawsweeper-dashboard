## Summary

Keep this open. Current main has useful backend observer plumbing, but the user-facing Activity panel/observer mode is still missing: Control UI still renders live tool activity as chat tool cards, the TUI still gates tool cards on verbose state, and the related draft implementation at https://github.com/openclaw/openclaw/pull/60814 remains open and unmerged.

## What This Changes

The issue requests a separate activity/observer surface for tool calls so users can monitor agent work without putting verbose output in the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker and build a maintainer-approved Activity panel on top of existing tool-events/session.tool plumbing, with explicit privacy defaults, independent UI state, filtering, truncation, and separate follow-ups for TUI and external streams.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check: the requested Activity panel is absent while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The best solution is a smaller product-approved observer slice with privacy-safe event handling, then separate decisions for per-connection verbose behavior, TUI split view, and external observer streams.
