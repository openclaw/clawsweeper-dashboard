## Summary

Keep open. Current main has useful backend `tool-events` and `session.tool` plumbing, but it still lacks the requested first-class Activity panel or observer mode separated from the chat stream, and the overlapping dashboard PR is still an open draft with prior review blockers.

## What This Changes

Request a separated Activity/tool-call observer surface for live agent activity, with Control UI, TUI, external stream, and per-connection verbose options.

## Best Possible Solution

Keep this as the broad observer-mode tracker and first ship a scoped Control UI Activity panel over existing `tool-events` and `session.tool` plumbing, with privacy-safe defaults and narrower follow-ups for TUI, external streams, Telegram parity, and per-connection verbose state.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/state, tool events still become chat tool messages, and the TUI remains verbose-gated.

## Solution Assessment

No complete solution is on main yet. The maintainable path is a smaller maintainer-approved Activity panel that reuses current event plumbing and coordinates with https://github.com/openclaw/openclaw/pull/60814 before expanding to other surfaces.
