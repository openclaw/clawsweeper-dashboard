## Summary

Keep open. Current main still has backend `tool-events`/`session.tool` plumbing and chat-embedded tool cards, but it does not implement the requested separate Activity panel, observer mode, per-connection verbose behavior, external activity stream, or TUI split pane; the live issue is already labeled for maintainer product and security review.

## What This Changes

Request to separate verbose tool-call activity from the main chat through an Activity/observer surface, with possible Control UI, per-connection, external stream, and TUI variants.

## Best Possible Solution

Scope a privacy-reviewed Control UI Activity panel over the existing `tool-events`/`session.tool` plumbing, then split TUI, external stream, and per-connection behavior into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: there is no Activity route/model, tool events still become chat tool messages, and TUI display remains verbose-gated.

## Solution Assessment

Unclear until maintainers scope the UX/API/privacy contract. The maintainable path is to reuse the existing `tool-events` and `session.tool` plumbing rather than introduce a second competing event pipeline.
