## Summary

Keep open. Current main has useful backend `tool-events` and `session.tool` plumbing, but the requested separate Activity panel/observer mode is still missing, and the overlapping dashboard implementation remains an open draft with review blockers.

## What This Changes

The issue requests a separate activity/observer surface for tool calls so verbose tool output can be monitored without cluttering the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker and ship a scoped Control UI Activity panel over existing `tool-events` and `session.tool` plumbing first, then split per-connection verbose, external stream, and TUI work into follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/state, tool events still become chat messages, and TUI verbose-off behavior still suppresses tool cards.

## Solution Assessment

No complete solution is on main yet. The narrow maintainable first slice is likely a dedicated Control UI Activity panel backed by existing event plumbing, coordinated with https://github.com/openclaw/openclaw/pull/60814 before broader TUI, external-stream, or per-connection verbose work proceeds.
