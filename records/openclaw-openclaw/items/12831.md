## Summary

Keep open. Current main still has backend observer plumbing and live tool cards, but the requested first-class Activity panel / observer mode separate from the chat stream is not implemented, and the related draft PR remains unmerged.

## What This Changes

Request a separate activity observer for tool calls and output, with possible Control UI panel, per-connection verbose state, external stream, and TUI split-pane variants.

## Best Possible Solution

Keep this as the canonical observer-mode tracker; ship a dedicated Control UI Activity panel over the existing `tool-events` and `session.tool` plumbing, then split per-connection verbose, TUI, and external-stream work into focused follow-ups if still desired.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool activity into chat cards, lacks a `session.tool` Activity handler in Control UI, and keeps TUI tool rendering tied to verbose state.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first slice appears to be a Control UI Activity panel using existing backend event plumbing, with maintainers reviewing or splitting draft PR #60814 before implementation proceeds.
