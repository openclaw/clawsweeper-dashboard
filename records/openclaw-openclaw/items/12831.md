## Summary

Keep open. Current main has backend tool-event plumbing and live chat tool cards, but it still does not provide the requested separated Activity panel, observer mode, per-connection verbose behavior, external activity stream, or TUI split-pane surface; the closest implementation remains an open draft that needs maintainer scoping.

## What This Changes

The issue requests separating verbose/tool activity from the main chat stream through an Activity panel or observer surfaces such as per-connection verbose state, a subscriber endpoint, and a TUI split pane.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over existing tool-events/session.tool plumbing, with TUI, external stream, and per-connection behavior split into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: there is no Activity route/files, tool events still become chat tool messages, and TUI display remains verbose-gated.

## Solution Assessment

Unclear until maintainers scope the UX/API/privacy contract. The maintainable path is a scoped Activity panel that reuses existing event plumbing rather than a broad automated dashboard drop.
