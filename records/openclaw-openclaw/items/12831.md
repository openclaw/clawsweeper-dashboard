## Summary

Keep this open. Current main has useful backend observer plumbing, but the requested first-class Activity panel / observer mode is still missing: Control UI still routes live tool activity through chat state, the TUI still hides tool cards when verbose is off, and the overlapping dashboard PR remains an open draft with unresolved privacy and product-shape concerns.

## What This Changes

Request a dedicated activity/tool-call observer mode that separates verbose tool activity from the main chat stream, with possible Control UI, per-connection, stream endpoint, and TUI variants.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Activity panel on top of existing tool-events/session.tool plumbing, then split TUI, per-connection verbose state, and external observer streams into deliberate follow-ups if needed.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check: the requested Activity panel/observer mode is absent while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a smaller maintainer-approved observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for per-connection verbose behavior, TUI split view, and external streams.
