## Summary

Keep this open. Current main has useful gateway observer plumbing and live tool cards, but it still lacks the requested independent Activity panel / observer mode: Control UI routes tool activity through chat state, TUI tool cards remain verbose-gated, and the broader endpoint/per-connection setting scope still needs product and security review.

## What This Changes

The issue requests a first-class activity observer surface that separates verbose tool-call activity from normal chat, with Control UI, per-connection verbose, external stream, and TUI variants.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Activity panel over the existing tool-events/session.tool plumbing, then split TUI, per-connection verbose state, and external observer streams into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: the requested independent Activity panel/observer mode is absent while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a maintainer-scoped observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for per-connection verbose behavior, TUI split view, and external streams.
