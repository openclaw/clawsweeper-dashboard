## Summary

Keep this open: current main has backend `tool-events` and `session.tool` plumbing plus chat-rendered tool cards, but it still lacks the requested first-class Activity panel / observer mode and the remaining scope needs maintainer product and privacy decisions.

## What This Changes

The issue requests a separate Activity panel or observer mode for tool calls so users can monitor sensitive agent activity without putting verbose output in the main chat.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over existing `tool-events` and `session.tool` plumbing, with TUI split view, per-connection verbose behavior, and external observer streams handled as deliberate follow-ups.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence check that the independent observer surface is absent while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a staged, maintainer-scoped observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for TUI, per-connection verbose state, and external streams.
