## Summary

Keep this open as the broad observer-mode tracker: current main has useful backend tool-event plumbing, but the requested independent Activity panel / observer mode is still absent and needs product and security scoping before implementation.

## What This Changes

The issue requests a separate observer/activity surface for tool calls so users can monitor agent work without mixing verbose tool output into the main chat.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over the existing tool-events/session.tool plumbing, then split TUI, per-connection verbose state, and external observer streams into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks an independent Activity panel/observer mode while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a maintainer-scoped observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for per-connection verbose behavior, TUI split view, and external streams.
