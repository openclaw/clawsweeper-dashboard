## Summary

Keep open. Current main has useful backend tool-event plumbing and live chat tool cards, but it still lacks the requested separated Activity panel or observer mode, and the overlapping dashboard implementation remains an open draft that needs maintainer scoping.

## What This Changes

Request a separated activity/observer surface so users can monitor tool calls without mixing verbose output into the main chat, with possible Control UI, per-connection, external stream, and TUI variants.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over existing tool-events/session.tool plumbing, with TUI, external stream, and per-connection behavior split into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/files, tool events still become chat tool messages, and TUI display remains verbose-gated.

## Solution Assessment

Unclear until maintainers scope the UX/API/privacy contract. The maintainable path is a scoped Activity panel that reuses existing event plumbing rather than a broad automated dashboard drop.
