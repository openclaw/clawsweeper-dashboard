## Summary

Keep open. Current main still has backend tool-event plumbing and an open draft dashboard PR, but it does not yet provide the requested first-class Activity panel or observer mode separated from the chat stream.

## What This Changes

The issue requests a separate activity/tool-call observer surface so users can monitor tool calls without pushing verbose output into the main conversation.

## Best Possible Solution

Keep this as the broad observer-mode tracker and first ship a scoped Control UI Activity panel over the existing `tool-events`/`session.tool` plumbing; split per-connection verbose, external observer stream, Telegram parity, and TUI work into narrower follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the separated observer surface: no Activity route/state, tool events still become chat messages, and TUI tool cards remain verbose-gated.

## Solution Assessment

No complete solution is on main yet. The maintainable first step is likely a dedicated Control UI Activity panel that reuses existing event plumbing and is coordinated with https://github.com/openclaw/openclaw/pull/60814 before broadening to other surfaces.
