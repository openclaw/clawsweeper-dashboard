## Summary

Keep this open as the broad observer-mode tracker: current main has useful gateway tool-event plumbing, but the requested independent Activity panel / observer mode is still not implemented and needs product and security scoping.

## What This Changes

The issue requests a user-facing observer mode that separates tool calls and agent activity from the main chat stream across Control UI, per-connection verbosity, external streams, and TUI layouts.

## Best Possible Solution

Keep this as the broad observer-mode tracker and scope a privacy-reviewed Control UI Activity panel over the existing tool-events/session.tool plumbing, then split TUI, per-connection verbose state, and external observer streams into deliberate follow-ups.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: the requested independent Activity panel/observer mode is absent while partial backend event plumbing exists.

## Solution Assessment

No, not as a one-shot automated fix. The maintainable path is a maintainer-scoped observer slice with explicit privacy defaults, independent UI state, filtering/truncation behavior, and separate decisions for per-connection verbose behavior, TUI split view, and external streams.
