## Summary

Keep open: current main and v2026.6.8 still use a shared inbound media store, reject scoped managed inbound refs, and lack durable channel/chat/sender provenance; the related per-agent tracker and closed PR do not fully cover this channel-level request.

## What This Changes

Request to namespace or durably index inbound media by channel/account/chat so passively received DM and group media can be separated and attributed.

## Best Possible Solution

Define a maintainer-approved scoped inbound-media/provenance contract that preserves existing media:// refs while adding durable channel/account/chat attribution across storage, resolution, staging, cleanup, docs, and tests.

## Reproduction Assessment

Yes, at source level: current main saves representative inbound channel media into the shared inbound bucket, rejects scoped media:// refs, stages by basename, and has no durable media provenance columns for channel/chat/sender. I did not run a live WhatsApp multi-group deployment in this read-only review.

## Solution Assessment

Unclear as proposed: channel-namespaced paths or an index are plausible, but an opt-in toggle and save-path-only change would leave resolver, staging, cleanup, SDK, and privacy semantics incomplete. The best fix is a maintainer-approved scoped media/provenance contract across the pipeline.
