## Summary

Keep open: current main and v2026.6.9 still use a shared inbound media bucket with resolver/staging paths that reject or collapse channel-scoped refs, and the requested durable provenance/storage change needs maintainer product and security direction.

## What This Changes

Request channel/account/chat namespacing or persisted provenance for inbound media so DM and group attachments remain separable and auditable.

## Best Possible Solution

Define one canonical media provenance/index contract, then update channel save callers, managed-media resolvers, sandbox staging, docs, migrations or doctor behavior, and regression coverage together.

## Reproduction Assessment

Yes for source-level behavior: current main still saves representative channel media under the shared inbound subdir, rejects nested inbound media refs, and normalizes managed paths to basename-only refs. I did not run a live multi-channel media ingest scenario in this read-only review.

## Solution Assessment

Unclear as requested: channel-namespaced paths and a persisted index are both plausible, but the best fix depends on the durable provenance schema, privacy/retention policy, migration story, and whether a new config toggle should exist.
