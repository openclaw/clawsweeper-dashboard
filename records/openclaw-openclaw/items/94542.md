- `impact:data-loss`: Per-file channel/chat provenance for passively received media is not durably retained, so source attribution cannot be reconstructed later.

## Summary

Keep open: current main and v2026.6.8 still lack channel/account/chat provenance for inbound media, and the broader media-isolation tracker does not fully cover this same-agent DM/group attribution request.

## What This Changes

The issue requests channel-namespaced or indexed inbound media storage so passively received files can be separated and attributed by channel, account, chat, sender, and receive time.

## Best Possible Solution

Define a maintainer-approved scoped inbound-media/provenance contract, likely SQLite-backed, that preserves existing media:// refs while adding channel/account/chat attribution through save, resolve, stage, cleanup, and docs paths.

## Reproduction Assessment

Yes at source level: current main saves representative inbound channel media into the shared inbound bucket, rejects scoped media:// refs, and lacks durable media provenance columns for channel/chat/sender. I did not run a live multi-channel WhatsApp setup in this read-only review.

## Solution Assessment

Unclear as proposed: channel-namespaced paths are a plausible direction, but an opt-in toggle and save-path-only change would leave resolver, staging, cleanup, SDK, and privacy semantics incomplete. The best fix is a maintainer-approved scoped media/provenance contract across the pipeline.
