## Summary

Keep open: current main still has the stale Twilio no-start stream window, and there is no linked fix PR or duplicate owner for the remaining work.

## What This Changes

The issue reports that Twilio inbound voice-call stream TwiML can mark a call active before any media stream connects, causing later inbound calls to be queued into hold music.

## Best Possible Solution

Add a Twilio-provider pending-stream TTL for call SIDs activated by inbound stream TwiML, cancel it on successful stream registration and existing terminal cleanup paths, and keep queueing for genuinely active streams.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: one inbound webhook returns stream TwiML and adds the call SID, then a later inbound webhook queues if no WebSocket start, unregister, or terminal status callback clears it. I did not run a live Twilio call in this read-only review.

## Solution Assessment

Yes, a provider-owned pending-stream timeout canceled by `registerCallStream()` and existing cleanup paths is the narrowest maintainable fix. Making queueing configurable would be a separate product choice and should not be required for this stale-state bug.

## Repair Work Prompt

Fix the Twilio voice-call stale active-stream bug reported in https://github.com/openclaw/openclaw/issues/81122. In `extensions/voice-call/src/providers/twilio.ts`, add provider-owned pending-stream timeout cleanup for call SIDs added from inbound stream TwiML: when `activateStreamCallSid` is added to `activeStreamCalls`, schedule a short unref'ed timer that removes the SID if `registerCallStream()` has not recorded a stream by then. Cancel and delete that timer on `registerCallStream()`, `unregisterCallStream()`, terminal status cleanup, and `hangupCall()`. Preserve existing queue behavior while a real media stream is active; do not add a config option or disable queueing globally. Add focused fake-timer coverage in `extensions/voice-call/src/providers/twilio.test.ts` showing that a second inbound call queues before the pending-start TTL expires but streams after a no-start timeout clears the stale SID, plus coverage that successful registration cancels the timeout.
