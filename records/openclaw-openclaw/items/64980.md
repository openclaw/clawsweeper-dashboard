## Summary

Keep open. Current main still treats Matrix live/finality metadata as outbound draft-preview behavior, while inbound Matrix replacements are suppressed before agent routing and the shared agent/plugin contexts do not expose streaming, finality, or replacement relation state.

## What This Changes

The issue requests agent-visible Matrix streaming, finality, or replacement metadata so multi-agent rooms can avoid acting on partial streamed mentions.

## Best Possible Solution

Add an additive typed inbound Matrix/core contract for live draft, finality, and replacement relation state, or intentionally suppress Matrix live draft events until finalization, then propagate the chosen state through Matrix inbound routing, `MsgContext`, and hook mappers with focused contract tests.

## Reproduction Assessment

No high-confidence live reproduction was established. The issue gives a concrete Matrix multi-agent scenario and current source shows the requested metadata is absent or suppressed, but this review did not run a live Matrix room.

## Solution Assessment

Unclear as proposed. A finality signal would address the user problem, but the maintainable fix should be a typed public inbound state or routing suppression policy rather than ad hoc raw `is_streaming`, `is_final`, or Matrix relation fields.
