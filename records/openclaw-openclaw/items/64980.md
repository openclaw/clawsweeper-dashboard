## Summary

Keep open. Current main still only marks OpenClaw's own outbound Matrix draft previews as live/finalized and still drops inbound Matrix edit/replacement events before agent routing, so agents do not get a streaming/finality/relation signal for the reported multi-agent coordination case.

## What This Changes

Request to expose Matrix inbound streaming, finality, or edit-relation metadata to agents so mention-gated multi-agent rooms can ignore partial streamed message content.

## Best Possible Solution

Add an additive typed inbound Matrix/core contract for live draft, finality, and replacement relation state, or intentionally suppress Matrix live draft events until finalization, then propagate the chosen state through Matrix inbound routing, `MsgContext`, and inbound hook mappers with focused Matrix and hook contract tests.

## Reproduction Assessment

Unclear. The issue gives a concrete Matrix `blockStreaming: true` multi-agent scenario and current source shows the requested metadata is absent, but this review did not run a live Matrix room to prove the truncated mention response end to end.

## Solution Assessment

Unclear as proposed. Exposing `is_streaming` or `is_final` would address the user-visible gap, but the maintainable fix should be a typed public inbound state or routing suppression policy rather than raw Matrix content keys selected without API review.
