## Summary

Keep open. Current main still handles Matrix streaming metadata on OpenClaw's outbound draft-preview path only, while inbound Matrix replacement events are dropped before agent routing and the shared agent/plugin contexts have no streaming, finality, or relation fields.

## What This Changes

Request to expose Matrix inbound streaming, finality, or edit-relation metadata to agents so multi-agent rooms can avoid acting on partial streamed message content.

## Best Possible Solution

Add an additive typed inbound Matrix/core contract for live draft, finality, and replacement relation state, or intentionally suppress Matrix live draft events until finalization, then propagate that state through Matrix inbound routing, `MsgContext`, and inbound hook mappers with focused contract tests.

## Reproduction Assessment

Unclear. The issue gives a concrete Matrix `blockStreaming: true` multi-agent scenario and current source shows the requested metadata is absent, but this review did not run a live Matrix room to prove the truncated mention response end to end.

## Solution Assessment

Unclear as proposed. Exposing a finality signal would address the user-visible gap, but the maintainable fix should be a typed public inbound state or routing suppression policy rather than raw Matrix content keys chosen without API review.
