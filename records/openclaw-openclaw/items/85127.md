## Summary

Keep this PR open: it fixes the explicit-runtime external harness path and now includes acceptable terminal proof, but the added bypass still misses documented `agentRuntime.id: "auto"` plugin harness selection, so the same stale-auth gate can block auto-selected external harnesses.

## What This Changes

The PR changes `runWithModelFallback` to skip provider auth cooldown checks for registered non-Codex plugin harness runtimes and adds a regression test for a stale Anthropic auth profile blocking `claude-tmux`.

## Best Possible Solution

Mirror the same forced-vs-auto harness selection semantics used by the embedded runner before applying the outer cooldown gate, then cover both explicit and auto-selected non-Codex plugin harnesses while preserving PI fallback and Codex bootstrap behavior.

## Reproduction Assessment

Yes. Source inspection shows current main checks provider auth cooldowns before the plugin harness run path, and the PR proof/test demonstrates the explicit-runtime stale Anthropic auth case; I did not execute the scenario in this read-only review.

## Solution Assessment

No, not yet. The explicit-runtime bypass is the right shape, but it should also cover documented auto-selected plugin harnesses by reusing or matching the real harness-selection decision before the cooldown gate.

- **[P2] Handle auto-selected plugin harnesses before cooldown:** `src/agents/model-fallback.ts:384`
