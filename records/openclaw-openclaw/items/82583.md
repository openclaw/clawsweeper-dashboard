## Summary

Keep open: current main still contains the cleanup path that can clear recoverable subagent final-delivery payloads and then skip later resume once cleanup is marked complete, and https://github.com/openclaw/openclaw/pull/82584 is an open linked fix candidate for this issue.

## What This Changes

The issue requests preserving or suspending recoverable subagent final-delivery payloads when announce retries are exhausted instead of burying them behind completed cleanup.

## Best Possible Solution

Land a reviewed fix that keeps recoverable keep-mode final-delivery payloads durable through retry exhaustion, avoids hot loops, and prunes expired or over-pressure suspended payloads with audit metadata.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: retry exhaustion reaches the give-up path, clears pending final delivery, completes cleanup, and future resume skips cleanup-complete runs; the issue also includes live 2026.5.12 audit evidence.

## Solution Assessment

Yes at the issue level. The narrow maintainable direction is to change the subagent registry lifecycle state machine so recoverable final deliveries are suspended and bounded rather than marking cleanup complete; the linked PR still needs normal review before merge.
