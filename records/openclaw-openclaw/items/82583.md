## Summary

Keep open: current main and the latest shipped tag still contain the retry-exhaustion path that can clear a recoverable pending final-delivery payload and then mark cleanup complete, and the open linked PR is the active fix candidate.

## What This Changes

Prevent recoverable subagent final-delivery payloads from being buried when announce cleanup retries are exhausted.

## Best Possible Solution

Land a reviewed lifecycle fix that keeps recoverable keep-mode final-delivery payloads durable through retry exhaustion, avoids hot loops, and bounds suspended payloads with expiry or pressure pruning plus regression coverage.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: retry exhaustion enters the give-up path, clears pendingFinalDeliveryPayload, completes cleanup, and later resume skips cleanup-complete runs; the issue also includes real 2026.5.12 audit evidence.

## Solution Assessment

Yes at the issue level. The narrow maintainable direction is to change the subagent registry lifecycle state machine so recoverable final deliveries are suspended and bounded rather than treated as terminal cleanup.
