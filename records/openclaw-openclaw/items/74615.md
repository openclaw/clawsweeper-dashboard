## Summary

Keep open. Current main still applies static per-message mention gating, so an unmentioned follow-up after an initial mention is skipped or stored instead of being processed under a temporary attention window.

## What This Changes

Request an opt-in group-chat attention window that processes follow-up messages for a configurable duration after the agent is mentioned.

## Best Possible Solution

Add an opt-in shared mention-policy or activation-window contract with per-conversation expiry state, config schema/help, docs, and focused channel wiring that preserves allowlists, command authorization, and existing mention-gating defaults.

## Reproduction Assessment

Yes. Configure a supported group/channel with `requireMention` enabled, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The requested UX is valid, but a single `attentionWindow` field is not enough by itself; the maintainable fix likely needs shared policy semantics plus channel-specific delivery constraints and tests.
