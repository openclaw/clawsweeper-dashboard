## Summary

Keep open. Current `main` still has static per-message mention gating: a mentioned group message does not start any remembered attention/auto-listen timer, and unmentioned follow-ups are still skipped or stored for context only. The related items cover adjacent mention detection, command bypass, or model-routing behavior, not this cross-channel post-mention attention-window feature.

## What This Changes

Request an opt-in configurable attention window that temporarily processes unmentioned group/channel follow-up messages after the agent is mentioned.

## Best Possible Solution

Define an additive shared attention-window contract for channel mention policy and config, with per-conversation expiry semantics, privacy defaults, transport capability notes, SDK/schema/docs coverage, and channel adapter tests before wiring individual plugins through it.

## Reproduction Assessment

Yes. Configure a group/channel with `requireMention` enabled, send one mentioned message, then send an unmentioned follow-up in the same conversation; current `main` evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The requested behavior is valid, but the proposed single `attentionWindow` field is not enough by itself; the maintainable fix needs a shared policy/state contract plus per-channel capability handling.
