## Summary

Keep open. Current main still implements static per-message mention gating with no attention-window config, runtime state, docs, or tests, and the request needs maintainer product/API and privacy review before implementation.

## What This Changes

Add a configurable group-chat attention window that temporarily processes unmentioned follow-up messages after an agent mention across messaging platforms.

## Best Possible Solution

Define an opt-in shared mention-activation contract with per-conversation expiry state, channel-specific policy hooks, schemas, docs, and regression coverage while preserving current mention-only defaults.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main behavior trace: with requireMention enabled, each group message is evaluated independently and unmentioned follow-ups take the skip/store path.

## Solution Assessment

Unclear. The requested UX is coherent, but the proposed single attentionWindow key is not enough by itself; maintainers need to define activation scope, expiry, reset rules, privacy boundaries, and platform-specific constraints.
