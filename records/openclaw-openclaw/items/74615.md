## Summary

Keep open. Current main still has stateless per-message mention gating with no attention-window config, runtime state, docs, or tests, and the request needs maintainer product/API design before implementation.

## What This Changes

The issue requests an opt-in group-chat attention window so one mention temporarily lets follow-up unmentioned messages reach the agent.

## Best Possible Solution

Define an opt-in shared mention-activation contract with per-conversation expiry state, schemas, docs, focused channel wiring, and defaults that preserve current mention-only behavior.

## Reproduction Assessment

Yes, as a current-main behavior trace rather than a bug repro: with requireMention enabled, a mentioned group message does not store timed activation state, so a later unmentioned follow-up is evaluated independently and takes the skip/store path.

## Solution Assessment

Unclear. The requested UX is coherent, but a single attentionWindow key is not enough by itself; the maintainable solution needs shared semantics for activation scope, expiry, delivery limits, allowlists, and channel-specific constraints.
