## Summary

Keep open. Current main still lacks any timed attention-window config or runtime state; group mention gating is still evaluated per inbound message, so an unmentioned follow-up after a mention is skipped or stored for context instead of triggering a reply.

## What This Changes

Request an opt-in configurable attention window that processes unmentioned group-chat follow-ups for a fixed duration after the agent is mentioned.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, config schema/help/docs, and focused channel wiring that preserves allowlists, command authorization, pending-history behavior, and current defaults.

## Reproduction Assessment

Yes. Configure a supported group/channel with requireMention enabled, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The requested UX is coherent, but a single attentionWindow key is not enough by itself; the maintainable path needs shared semantics for activation, expiry scope, delivery limits, and channel-specific behavior.
