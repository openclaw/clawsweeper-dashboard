## Summary

Keep open. Current main still uses static per-message mention gating plus limited implicit reply/thread cases; there is no configurable timed attention window, so this remains a cross-channel product/API feature request.

## What This Changes

Request an opt-in group-chat attention window that temporarily processes unmentioned follow-up messages after the agent is mentioned.

## Best Possible Solution

Define an opt-in shared attention-window contract with per-conversation expiry state, clear scope/reset semantics, schemas, docs, and channel-by-channel wiring that preserves current defaults.

## Reproduction Assessment

Yes for the current limitation. With requireMention enabled, source inspection shows a mentioned message does not store timed activation state, so a later unmentioned follow-up is evaluated independently and skipped unless it qualifies as an existing implicit reply/thread case.

## Solution Assessment

Unclear. The requested UX is coherent, but a single duration key is not enough without a shared policy for expiry scope, reset behavior, allowlists, delivery limits, and channel-specific constraints.
