## Summary

Keep open. Current main still has static per-message mention gating: after an initial mention, an unmentioned group follow-up is skipped or only stored as pending context by representative channel handlers, so the requested configurable attention window remains unimplemented.

## What This Changes

Request a configurable group-chat attention window that temporarily processes unmentioned follow-up messages after the agent is mentioned.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, schema/help/docs, and focused channel wiring that preserves allowlists, command authorization, pending-history behavior, and current defaults.

## Reproduction Assessment

Yes. Configure a supported group/channel with requireMention enabled, send one mentioned message, then an unmentioned follow-up; current source shows the follow-up is evaluated only on its own mention facts and then skipped or pending-stored.

## Solution Assessment

Unclear. The UX direction is valid, but the proposed single attentionWindow key is not enough by itself; the maintainable path needs shared policy semantics, expiry scope, docs, and channel-specific delivery constraints.
