## Summary

Keep open. Current main still has static per-message mention gating plus limited permanent activation toggles; it does not implement a configurable mention-started timer that lets unmentioned follow-up messages trigger for a bounded window across group chat surfaces.

## What This Changes

Request a configurable group-chat attention window so an initial mention temporarily lets unmentioned follow-up messages trigger the agent before returning to mention-only mode.

## Best Possible Solution

Define an opt-in shared attention-window contract with per-conversation expiry state, explicit defaults, privacy semantics, platform limitations, tests, and docs, then wire channel plugins through that contract without weakening existing authorization or mention-gating boundaries.

## Reproduction Assessment

Yes. Configure a group/channel with `requireMention: true`, send one message that mentions the agent, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The requested behavior is valid, but the proposed single per-channel `attentionWindow` seconds field needs maintainer design because the maintainable fix likely spans shared state, plugin contracts, privacy defaults, and per-platform delivery limits.
