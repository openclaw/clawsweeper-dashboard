## Summary

Keep open. Current main still uses static, per-message mention gating for groups/channels and has no configurable attention-window state that would process unmentioned follow-ups after an initial mention; the related items cover adjacent mention detection, command bypass, Feishu routing, or model/context routing rather than this cross-channel timer behavior.

## What This Changes

Add an opt-in, configurable group-chat attention window that temporarily treats follow-up messages after a mention as triggerable without requiring every message to mention the agent.

## Best Possible Solution

Define an additive shared attention-window contract with explicit opt-in defaults, per-conversation expiry semantics, platform limitation handling, tests, and docs, then wire channel plugins through that contract while preserving existing auth and mention-gating boundaries.

## Reproduction Assessment

Yes. Configure a group/channel with requireMention enabled, send a mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The user problem is valid, but the proposed per-channel attentionWindow field needs maintainer design because the durable fix likely spans shared policy, config schema, timer state, privacy semantics, and channel-specific delivery limits.

## Risks / Open Questions

- A cross-channel attention window changes privacy and token-exposure semantics by processing unmentioned group messages after a mention, so default, scope, expiry, and audit behavior need explicit design.
- Some transports cannot deliver unmentioned group messages unless platform settings permit it; Telegram Bot API privacy mode is already called out by status/docs for requireMention=false-style behavior.
- Timer state must be scoped per account/channel/thread/session and either persist or intentionally reset across restarts without bypassing group allowlists or sender authorization.
