## Summary

Keep open. Current main still implements static per-message mention gating for group/channel messages and does not have configurable attention-window state after a mention; the existing related items cover adjacent mention detection or routing behavior, not this cross-channel timer feature.

## What This Changes

Add a configurable group-chat attention window so a mention temporarily lets the agent process unmentioned follow-up messages before reverting to mention-only behavior.

## Best Possible Solution

Define an additive, opt-in shared attention-window contract in the channel mention policy/config surface, including per-conversation expiry semantics, platform capability notes, tests, and docs, then wire channel plugins through that contract without weakening existing allowlist or command-gating boundaries.

## Reproduction Assessment

Yes. Configure a group/channel with `requireMention` enabled, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The user problem is valid, but the proposed single `attentionWindow` field needs maintainer design because the maintainable fix spans shared policy, config schema, timer state, privacy semantics, and channel-specific delivery limits.

## Risks / Open Questions

- Processing unmentioned group messages after a mention changes privacy and token-exposure semantics, so default, scope, expiry, and audit behavior need explicit maintainer design.
- Some transports may not deliver unmentioned group messages unless platform-level settings permit it; Telegram privacy mode is already documented as a requirement for `requireMention=false`-style behavior.
- Timer state needs clear scoping across account, channel, thread/topic, session, and restart behavior before a cross-channel implementation is safe.
