## Summary

Keep open. Current main still has static, per-message mention gating with implicit mention exceptions for replies/threads, but no configurable conversation-level attention window that starts after a mention and temporarily processes arbitrary unmentioned follow-ups. The existing related reports are adjacent mention-routing or mention-detection work, not a duplicate tracker for this cross-channel timer feature.

## What This Changes

Add an opt-in configurable attention window so mentioning the agent in a group chat temporarily lets it process unmentioned follow-up messages before reverting to mention-only behavior.

## Best Possible Solution

Define an additive shared attention-window contract for group/channel mention policy, with opt-in config, per-conversation expiry semantics, platform capability notes, tests, and docs, then wire bundled channel plugins through that contract without weakening existing allowlist or command-gating boundaries.

## Reproduction Assessment

Yes. Configure a group/channel with requireMention enabled, send one mentioned message, then send an unmentioned follow-up that is not a reply-to-bot or authorized command; current main evaluates only the follow-up message facts and takes the skip/store path.

## Solution Assessment

Unclear. The proposed duration field addresses the user problem, but the maintainable solution needs product/API decisions around privacy defaults, timer scoping, platform delivery limits, and whether this belongs in shared channel policy or narrower channel-specific capabilities.

## Risks / Open Questions

- Processing unmentioned group messages after a mention changes privacy and token-exposure semantics, so default, scope, auditability, and opt-in behavior need explicit maintainer design.

- Timer state needs a clear scope across account, channel, thread/topic, agent, restart, and multi-agent setups before a cross-channel implementation is safe.
