## Label Justifications

- `P3`: This is a useful group-chat ergonomics feature, but it is not a regression in established behavior and needs product/security direction first.
- `impact:message-loss`: With mention-only mode enabled, follow-up group messages after the initial mention are intentionally skipped unless users keep mentioning the agent.
- `impact:security`: The proposal would process unmentioned group traffic during the attention window, changing the privacy boundary for group content.

## Summary

Keep open: current main and v2026.6.8 still have static `mention`/`always` group activation plus always-on ambient room events, but no timed post-mention attention window. The open contextual activation thread overlaps, but it is broader and not a safe canonical replacement for this narrower product/security decision.

## What This Changes

Request a configurable group-chat attention window that temporarily processes follow-up messages after a mention without requiring repeated @mentions.

## Best Possible Solution

Choose a consent-aware, default-off group activation design that reuses the shared mention-gating and ambient room-event seams, then document and test the behavior across supported group channels.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and release inspection show current main lacks timed attention-window behavior while static mention gating and always-on ambient room events exist.

## Solution Assessment

Unclear: the proposed `attentionWindow` config is plausible, but maintainers should first decide the consent-aware cross-channel boundary and whether this should be a core mode, plugin/API seam, or ambient room-event enhancement.
