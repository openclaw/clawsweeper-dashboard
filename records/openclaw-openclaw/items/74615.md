## Summary

Keep open. Current main still has static per-message mention gating with no timed attention-window config or per-conversation expiry state, so the requested group-chat follow-up behavior remains unimplemented and needs maintainer design across shared channel policy, privacy, and channel-specific delivery behavior.

## What This Changes

The issue requests an opt-in configurable attention window that lets a group/chat process unmentioned follow-up messages for a limited time after the agent is mentioned.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, schema/help/docs, and focused channel wiring that preserves current defaults, allowlists, command authorization, and pending-history behavior.

## Reproduction Assessment

Yes. Configure a supported group/channel with requireMention enabled, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The UX request is coherent, but a single attentionWindow key is not enough by itself; the maintainable fix needs shared semantics for activation, expiry scope, delivery limits, reset behavior, and channel-specific exceptions.
