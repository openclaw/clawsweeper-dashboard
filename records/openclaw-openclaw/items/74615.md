## Summary

Keep open. Current main still has per-message mention gating, so an unmentioned follow-up after a mention is skipped or stored when `requireMention` is enabled; no `attentionWindow` config, schema, docs, or shared expiry state exists. WhatsApp has manual `/activation always`, but it is not a timed auto-listen window and docs say other surfaces ignore it.

## What This Changes

Request an opt-in attention window that temporarily processes unmentioned group follow-ups after a mention before reverting to mention-only mode.

## Best Possible Solution

Add an opt-in shared mention-policy or activation contract with per-conversation expiry state, config schema/help, docs, and focused channel wiring that preserves allowlists, command authorization, and existing mention-gating defaults.

## Reproduction Assessment

Yes. Configure a supported group/channel with `requireMention` enabled, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the existing skip/store path.

## Solution Assessment

Unclear. The UX request is valid, but the proposed single `attentionWindow` field is not enough by itself; the maintainable fix likely needs shared policy semantics plus channel-specific delivery and privacy constraints.
