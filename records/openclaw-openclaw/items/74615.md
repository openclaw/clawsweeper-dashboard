## Summary

Keep open: current main and v2026.6.10 still provide static mention/always group activation plus always-on ambient room events, not a timed post-mention attention window. Related group-context issues overlap, but none is an exact canonical replacement for this narrower timer-based product and security decision.

## What This Changes

Request a configurable timed attention window that lets group chats process follow-up messages after one mention before returning to mention-only mode.

## Best Possible Solution

Keep this open until maintainers choose a default-off, consent-aware group activation design that either implements a timed post-mention window or deliberately folds that behavior into a broader activation model.
