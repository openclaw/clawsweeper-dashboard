## Summary

Keep open. Current main has static per-message mention gating plus manual session-scoped `/activation` modes, but it does not implement a configurable timed attention window after a mention.

## What This Changes

Request an opt-in configurable attention window that lets group chats process unmentioned follow-up messages for a limited time after the agent is mentioned.

## Best Possible Solution

Design an opt-in shared mention-policy activation contract with per-conversation expiry state, schemas, docs, and focused channel wiring that preserves current defaults and allowlist semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs give a high-confidence current-state check: group mention handling is static and has no timed activation surface.

## Solution Assessment

Unclear. The requested UX is coherent, but a single `attentionWindow` key is not enough by itself; the maintainable solution needs shared semantics for activation scope, expiry, reset, allowlists, and channel-specific delivery constraints.
