## Summary

Keep open. Current main still uses per-message mention gating, with only existing Slack thread and WhatsApp activation exceptions, and has no cross-channel timed attention-window config or expiry state for later unmentioned follow-ups.

## What This Changes

Requests an opt-in timed attention window that processes unmentioned group-chat follow-ups for a configurable duration after an initial mention, then returns to mention-only behavior.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, schema/help/docs, and focused channel wiring that preserves allowlists, command authorization, pending-history behavior, and platform delivery constraints.

## Reproduction Assessment

Yes. Configure a supported group/channel with requireMention enabled, send one mentioned message, then send an unmentioned non-thread follow-up; current code evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The requested UX is coherent, but a single attentionWindow key is not enough without shared semantics for activation scope, expiry, privacy limits, delivery constraints, and channel-specific exceptions.
