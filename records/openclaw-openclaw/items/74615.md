## Summary

Keep open. Current main still has static per-message mention gating, so a mentioned group-chat message does not start a timed window that processes later unmentioned follow-ups.

## What This Changes

Add a configurable group-chat attention window that temporarily treats unmentioned follow-up messages as in-scope after the agent is mentioned.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, schema/help/docs, and focused channel wiring that preserves allowlists, command authorization, pending-history behavior, and current defaults.

## Reproduction Assessment

Yes. Configure a supported group/channel with requireMention enabled, send one mentioned message, then an unmentioned follow-up; current code evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The requested UX is coherent, but a single attentionWindow key is not enough by itself; the maintainable solution needs shared semantics for activation, expiry scope, delivery limits, and channel-specific behavior.
