## Summary

Keep open. Current main still implements static per-message mention gating with reply/thread and command bypasses, but no configurable attention-window state that lets unmentioned follow-up messages trigger after an initial mention. The related reports cover narrower mention detection, command bypass, or model/context routing cases and do not supersede this cross-channel product/API request.

## What This Changes

The issue requests a configurable group-chat attention window that processes follow-up messages for a limited duration after an initial mention, then returns to mention-only mode.

## Best Possible Solution

Define an opt-in shared attention-window contract with per-conversation expiry semantics, explicit defaults, platform limitations, tests, docs, and a plugin-facing API shape, then wire bundled channel plugins through that contract while preserving existing allowlist and command-gating boundaries.

## Reproduction Assessment

Yes. Configure a group/channel with `requireMention: true`, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The user problem is valid, but the proposed per-channel `attentionWindow` field needs maintainer design because the maintainable fix likely spans shared state, config/schema/docs, plugin contracts, privacy semantics, and channel-specific delivery constraints.
