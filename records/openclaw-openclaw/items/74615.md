## Summary

Keep open: current main and v2026.6.10 still provide static mention-gated or always-on group handling, plus adjacent ambient-room and Slack thread behavior, but no configurable timed post-mention attention window. The remaining work is a product/security decision because it would intentionally process unmentioned group messages for a period after a mention.

## What This Changes

Request a configurable post-mention attention window that temporarily lets group follow-up messages bypass mention gating across Slack and other channels.

## Best Possible Solution

Define a maintainer-sponsored, default-off group activation design that reuses shared mention gating and ambient-room semantics, with explicit privacy/config behavior before cross-channel implementation.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and docs show current main supports static mention-gated, always-on, ambient-room, and Slack thread-adjacent behavior, but not a timed post-mention mode.

## Solution Assessment

Unclear: a timer-based attention window is plausible, but maintainers need to choose the privacy boundary and whether this belongs in shared group activation, per-channel config, or a broader contextual-activation design.
