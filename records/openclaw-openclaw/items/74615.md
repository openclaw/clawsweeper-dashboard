## Summary

Keep open. Current main still lacks any attention-window config or remembered conversation state; shared mention gating is evaluated from the current inbound message facts, and representative bundled channel paths still skip or store unmentioned group follow-ups under `requireMention`. The feature request is valid, but it needs maintainer product/API decisions around opt-in privacy scope, expiry semantics, config schema/docs, and platform visibility limits before implementation.

## What This Changes

The issue requests an opt-in, configurable window after a group-chat mention during which OpenClaw processes unmentioned follow-up messages before reverting to mention-only mode.

## Best Possible Solution

Add an opt-in shared channel mention-policy contract for attention windows with explicit per-conversation expiry semantics, config schema/help, docs, and focused tests, then wire supported channel plugins through that contract while preserving existing allowlist, command-auth, and mention-gating boundaries.

## Reproduction Assessment

Yes. Configure a group/channel with `requireMention` enabled, send a mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up message's own mention facts and reaches the skip/store path.

## Solution Assessment

Unclear. The user problem is valid, but the proposed single `attentionWindow` field is not enough as a final design because the maintainable fix spans shared policy, config schema, privacy semantics, timer state, docs, and platform delivery limits.
