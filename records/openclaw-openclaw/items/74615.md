## Summary

Keep open. Current main still has static per-message mention gating: after a mention, unmentioned follow-ups are skipped or stored for pending context by the channel handlers instead of being processed, so the requested configurable attention window remains a shared product/API gap.

## What This Changes

Request an opt-in configurable group-chat attention window that processes unmentioned follow-up messages for a duration after a mention, then returns to mention-only mode.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, schema/help/docs, and focused channel wiring that preserves allowlists, command authorization, pending-history behavior, and current defaults.

## Reproduction Assessment

Yes. Configure a supported group/channel with `requireMention` enabled, send one mentioned message, then an unmentioned follow-up; the current helper evaluates only the follow-up's own facts and channel code takes the skip/store path.

## Solution Assessment

Unclear. The UX request is valid, but a single `attentionWindow` key is not enough by itself; the maintainable path needs shared policy semantics, expiry scope, docs, and channel-specific delivery constraints.
