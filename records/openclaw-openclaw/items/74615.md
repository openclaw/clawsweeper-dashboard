## Summary

Keep open: current main and v2026.6.6 support static mention/always activation plus always-on ambient room events, but not a timed post-mention auto-listen window. The related contextual activation issue is broader and still open, so it is useful context rather than a safe duplicate close.

## What This Changes

Request an opt-in configurable timer that temporarily processes follow-up group messages after the agent is mentioned, then returns to requireMention behavior.

## Best Possible Solution

Design one consent-aware group activation mechanism that can express a timed post-mention window centrally, reuses ambient room-event behavior where appropriate, and avoids duplicating per-channel gating logic.
