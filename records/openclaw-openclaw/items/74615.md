## Summary

Keep open. Current main still has only static per-message mention gating, so an unmentioned follow-up after a mention is still skipped or stored rather than processed; the requested attention window remains a real cross-channel product/API gap.

## What This Changes

The issue requests an opt-in attention window that lets group-chat follow-up messages through for a limited time after the agent is mentioned.

## Best Possible Solution

Add an opt-in shared mention-policy or activation contract with per-conversation expiry state, config schema/help, docs, and focused channel wiring that preserves allowlists, command authorization, and existing mention-gating defaults.

## Reproduction Assessment

Yes. Configure a supported group/channel with `requireMention` enabled, send one mentioned message, then send an unmentioned follow-up; current main evaluates the follow-up's own mention facts and takes the existing skip/store path.

## Solution Assessment

Unclear. The UX request is valid, but the proposed single `attentionWindow` field is not enough by itself; the maintainable fix likely needs shared policy semantics plus channel-specific delivery constraints and tests.
