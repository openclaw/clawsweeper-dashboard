## Summary

Keep open. Current main still implements static per-message mention gating, so a mention does not start a timed window that allows later unmentioned group messages to trigger the agent.

## What This Changes

Request an opt-in configurable attention window that temporarily treats follow-up group messages as mentioned after the agent is mentioned once.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, schemas, docs, and focused channel wiring that preserves current defaults and allowlist semantics.

## Reproduction Assessment

Yes. With requireMention enabled, a mentioned group message does not record any timed activation state, so a later unmentioned follow-up is evaluated on its own mention facts and takes the skip/store path.

## Solution Assessment

Unclear. The requested UX is coherent, but a single attentionWindow key is not enough by itself; the maintainable solution needs shared semantics for activation, expiry scope, delivery limits, and channel-specific behavior.
