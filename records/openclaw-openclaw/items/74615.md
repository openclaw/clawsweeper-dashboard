## Summary

Keep open: current main and v2026.6.10 still lack a timed post-mention attention window. Existing ambient room events are always-on, and the related contextual/model-routing issues overlap but do not supersede this narrower timer-based product and security decision.

## What This Changes

The issue requests a configurable post-mention attention window so group chats temporarily process follow-up messages without repeated mentions.

## Best Possible Solution

Choose a default-off, consent-aware group activation design that reuses shared mention-gating and ambient room-event boundaries before implementing timed listening across channels.
