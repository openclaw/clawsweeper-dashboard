## Summary

Keep open: current main and v2026.6.8 still only support static `mention`/`always` group activation plus always-on ambient room events, not a timed post-mention attention window. The related contextual-activation thread overlaps, but this narrower timer-based product/security decision is not superseded by it.

## What This Changes

The issue requests a configurable group-chat attention window that temporarily processes unmentioned follow-up messages after an explicit mention, then returns to mention-only mode.

## Best Possible Solution

Choose a default-off, consent-aware group activation design that reuses shared mention-gating and ambient room-event boundaries, then document and test the behavior across supported group channels.
