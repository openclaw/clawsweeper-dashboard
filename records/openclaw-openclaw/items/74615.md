## Summary

Keep open. Current main still has static per-message mention gating and no configurable attention-window state, so unmentioned follow-ups after a mention are skipped or stored as pending context rather than processed.

## What This Changes

Request an opt-in configurable group-chat attention window that processes unmentioned follow-ups for a short duration after the agent is mentioned.

## Best Possible Solution

Add an opt-in shared mention-policy activation contract with per-conversation expiry state, schema/help/docs, and channel wiring that preserves allowlists, command authorization, pending-history behavior, and platform delivery constraints.

## Reproduction Assessment

Yes. Configure a supported group with requireMention enabled, send one mentioned message, then send an unmentioned follow-up; the current helper evaluates only the follow-up's own facts and the channel handler takes the no-mention skip/store path.

## Solution Assessment

Unclear. The requested UX is valid, but a single attentionWindow key is not enough without shared expiry semantics, privacy defaults, docs, and channel-specific delivery handling.
