## Summary

Keep open: current main still exposes static group activation behavior and room-level ambient events, but it does not implement a timed post-mention attention window that temporarily treats follow-up group messages as in-scope. The request remains a product/security decision because it would expand what group messages are processed after a mention and likely add channel config or state.

## What This Changes

Request a configurable group-chat attention window that starts after an agent mention, processes follow-up messages for a limited duration, and then returns to mention-only behavior.

## Best Possible Solution

Keep this as an explicit product/security design item until maintainers decide whether the timer belongs in channel config, shared group activation policy, or a broader contextual-activation model.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken existing behavior. Source inspection and the issue body show the current behavior is static mention-only versus always-listen group handling, with no timed post-mention mode to reproduce as a bug.

## Solution Assessment

Unclear: the timer-based proposal is plausible, but it is not yet clearly the best product shape versus the broader contextual activation work in https://github.com/openclaw/openclaw/issues/38349 or context-window routing in https://github.com/openclaw/openclaw/issues/13962. Maintainers should settle the privacy boundary and channel/config ownership before code changes.
