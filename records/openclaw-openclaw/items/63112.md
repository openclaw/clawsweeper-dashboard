## Summary

Keep open: current main still lacks the warning for shell-looking main-session cron systemEvent payloads, and this active contributor PR supplies a focused CLI fix with tests and real terminal proof.

## What This Changes

The PR adds a shared shell-command heuristic, pre-RPC cron add/edit stderr warnings for main-session systemEvent payloads, focused tests, and a changelog entry.

## Best Possible Solution

Land this narrow CLI warning, or an equivalent add/edit warning, while preserving current main-session systemEvent runtime semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows cron add/edit can send shell-looking systemEvent text to a main-session job without a warning, while the runtime only enqueues that text as a system event.

## Solution Assessment

Yes. A non-blocking pre-RPC warning is the narrow maintainable fix because executing systemEvent text as shell would change cron semantics and safety assumptions.
