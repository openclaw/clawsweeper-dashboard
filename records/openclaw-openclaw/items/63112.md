## What This Changes

The PR adds a shared shell-command heuristic, cron add/edit stderr warnings for main-session systemEvent payloads, focused tests, and a changelog entry.

## Best Possible Solution

Land this narrow CLI warning, or an equivalent add/edit warning, while preserving current main-session systemEvent runtime semantics.

## Reproduction Assessment

Yes. Source inspection shows current main can send shell-looking systemEvent text to a main-session cron job without warning, while the timer enqueues that text as a system event instead of executing it.

## Solution Assessment

Yes. A non-blocking pre-RPC warning is the narrow maintainable fix because executing shell-looking systemEvent text would change cron semantics and safety assumptions.
