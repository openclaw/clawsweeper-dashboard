Keep this PR open: current main still lacks the requested warning for shell-looking main-session cron system events, and this branch remains the linked implementation candidate, but it needs a merge refresh and one small policy cleanup before merge.

## What This Changes

The PR adds a shared shell-command heuristic and pre-RPC stderr warnings for `cron add` and `cron edit` when the effective cron payload is a main-session `systemEvent`.

## Best Possible Solution

Continue normal maintainer review; ClawSweeper found no patch-correctness issue.

## Reproduction Assessment

Yes. Source inspection shows current `cron add` and `cron edit` can send shell-looking main-session `systemEvent` text to the gateway without warning, while the runtime treats main-session system events as notifications rather than shell execution.

## Solution Assessment

Yes, with cleanup. A non-blocking CLI warning is the narrowest maintainable mitigation now that command cron is the supported execution path, but the branch should drop `CHANGELOG.md` and be refreshed against current main.
