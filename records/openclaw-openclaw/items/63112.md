## What This Changes

This PR adds a shared shell-command heuristic, cron add/edit stderr warnings for main-session systemEvent payloads that look executable, focused CLI/shared tests, and a changelog entry.

## Best Possible Solution

Land this narrow CLI warning, or an equivalent warning with the same add/edit coverage, while preserving main-session systemEvent runtime semantics.

## Reproduction Assessment

Yes. Source inspection of current main shows cron add/edit can send shell-looking systemEvent text to a main-session job without warning, while the runtime only enqueues that text as a system event.

## Solution Assessment

Yes. A non-blocking pre-RPC warning is the narrow maintainable fix because executing shell-looking systemEvent text would change cron runtime semantics and safety assumptions.
