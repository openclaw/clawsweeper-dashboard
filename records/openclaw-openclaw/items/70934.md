## Summary

Keep open. Current main and the latest stable tag still put setup/auth/gateway/workspace/bootstrap work before an ordinary value-producing task, and the first-run bootstrap path still asks identity/persona questions before normal replies, so this remains a real onboarding product request rather than cleanup closure.

## What This Changes

The issue requests a value-first first-run onboarding flow that defers optional agent identity, reply style, symbols, tone, and persona customization until after a quick useful result.

## Best Possible Solution

Define a maintainer-approved first-run contract that keeps required auth/security prerequisites early while moving optional identity, persona, reply-style, and symbol customization behind a starter task or template-based value moment.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection show current main and v2026.5.18 still sequence setup/customization/bootstrap before a normal value-producing task.

## Solution Assessment

Unclear. The value-first direction is plausible, but the best implementation needs maintainer decisions about mandatory trust/auth prerequisites versus optional personalization that can safely move later.
