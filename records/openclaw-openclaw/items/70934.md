## Summary

Keep open. Current main still sequences setup, auth/model, gateway/channel/search/skills/hooks, and bootstrap/persona work before the first ordinary useful task, so the requested value-first onboarding contract is not implemented.

## What This Changes

Request to make first-run onboarding show useful starter tasks or an immediate quick result before optional persona, naming, reply-style, channel, and other customization steps.

## Best Possible Solution

Define a maintainer-approved first-run contract that keeps only mandatory security/auth/runtime prerequisites before the first result, offers a small set of starter tasks, and moves identity, persona, reply style, channel, search, skills, and other customization into optional follow-up steps across CLI, macOS, Control UI, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current CLI and macOS flows still put setup/bootstrap/persona work before the first normal useful task.

## Solution Assessment
