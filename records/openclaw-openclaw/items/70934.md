Keep open. The issue requests a value-first first-run onboarding flow that offers starter tasks before optional personalization and setup customization.

## Best Possible Solution

Define a maintainer-approved first-run contract that shows a low-risk starter task or template early while keeping mandatory auth, permission, Gateway exposure, and security decisions explicit before privileged behavior runs.
