## Summary

Keep open. Current main and the latest release have QuickStart, a post-setup TUI/Web hatch, and an opt-in modern setup assistant, but the default onboarding path still asks for setup/auth/Gateway/channel choices before a first useful task; this remains a product and security-scoping decision rather than a cleanup close.

## What This Changes

The issue requests a first-run onboarding flow that presents starter tasks or a quick useful result before optional personalization and nonessential setup choices.

## Best Possible Solution

Define a maintainer-approved first-run contract that offers a low-risk starter task or template early while keeping required auth, permission, Gateway exposure, and security choices explicit before privileged behavior runs.
