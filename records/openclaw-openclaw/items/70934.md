## Summary

Keep open. Current main has not adopted the requested value-first onboarding flow: CLI, macOS, and docs still put setup/auth/gateway/channel/search/skills/bootstrap/persona work before a normal useful task, and the remaining change needs maintainer UX and trust-boundary scoping.

## What This Changes

The issue requests first-run onboarding that shows useful starter tasks before optional personalization such as agent name, reply style, symbols, tone, or persona choices.

## Best Possible Solution

Define a maintainer-approved first-run contract that preserves mandatory auth/security prerequisites, offers starter tasks as the first value moment, and moves identity/persona/reply-style/channel customization into optional follow-up steps across CLI, macOS, Control UI, docs, and tests.
