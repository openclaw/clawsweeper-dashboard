## Summary

Keep open. Current main still does not provide the requested value-first first-run flow: the recommended CLI path performs setup/auth/model/gateway/channel/search/skills/hooks work before the first normal task, and the macOS local path puts onboarding chat after connection, wizard, and permissions while its kickoff starts the BOOTSTRAP/SOUL persona ritual rather than a quick useful starter task. The issue is recent, non-maintainer-authored, only labeled enhancement, and the supplied context does not show a linked closing PR or canonical duplicate.

## What This Changes

The issue asks OpenClaw to reduce first-run onboarding friction by showing useful starter tasks and an initial result before optional personalization such as agent naming, reply style, symbols, tone, or persona preferences.

## Best Possible Solution

Define a maintainer-approved first-run contract that keeps only mandatory model/session/security prerequisites before first value, presents a small set of useful starter tasks or templates, runs one immediately, and moves identity, persona, channel, style, search, skill, and other customization into optional follow-up steps across CLI, macOS, Control UI/TUI, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current main still sequences setup/bootstrap/persona work ahead of the first ordinary useful task on the main CLI and macOS onboarding paths.

## Solution Assessment

Unclear. The value-first direction is plausible, but the best implementation depends on maintainer UX and security decisions about the minimum safe prerequisites before an agent can run a starter task.
