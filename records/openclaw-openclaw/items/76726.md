## Summary

Keep open: current main still has a plausible Feishu group path where an @mention can show typing and dispatch to the agent while generic group visible-reply policy suppresses final delivery, and the Feishu-specific product contract is not clear enough to close as expected behavior.

## What This Changes

The issue reports that Feishu group @mentions in OpenClaw 2026.5.2 trigger typing and agent generation but send no final reply, with dispatch counts ending at `queuedFinal=false, replies=0`.

## Best Possible Solution

Maintainers should decide the Feishu group @mention contract, then either add a targeted Feishu automatic-reply path with regression coverage or clarify Feishu troubleshooting/setup docs around `messages.groupChat.visibleReplies: "automatic"`.

## Reproduction Assessment

No, not at high confidence for the reporter's exact setup. Source inspection gives a clear default-config path to typing plus `queuedFinal=false, replies=0`, but the issue lacks the effective Feishu and visible-reply configuration needed for a live current-main reproduction.

## Solution Assessment

Unclear. A narrow Feishu-specific automatic-reply change may be correct if @mentions are meant to imply visible replies, but current generic docs make message-tool-only the default for group rooms, so maintainers need to choose the intended policy first.
