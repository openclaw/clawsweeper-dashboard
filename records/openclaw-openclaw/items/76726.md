Keep open: current main strongly suggests the symptom can happen under the documented generic group visible-reply policy, but the Feishu-specific expected behavior is ambiguous enough that this should get maintainer policy/docs review rather than cleanup closure.

## What This Changes

The issue reports that Feishu group @mentions start typing and run the agent, but no automatic final reply is delivered to the group.

## Best Possible Solution

Maintainers should decide whether Feishu group @mentions should override the generic group visible-reply default; then either add a Feishu-specific automatic-reply path with regression coverage or clarify the Feishu troubleshooting docs around `messages.groupChat.visibleReplies: "automatic"`.

## Solution Assessment

No clear code fix is proven yet. Forcing automatic Feishu group replies would conflict with the documented generic group/channel default, so the safer path is a maintainer policy decision followed by either docs or a targeted Feishu behavior change.
