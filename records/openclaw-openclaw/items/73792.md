## Summary

Keep open. Current main still lacks unexpected MCP transport-close eviction, so the PR is not obsolete, but the proposed patch is not merge-ready because it treats intentional startup cleanup as an unexpected close and is stale against current main's test harness cleanup.

## What This Changes

The PR wires MCP transport close/error callbacks into session MCP runtimes, disposes cached runtimes when a transport closes, logs the eviction, and adds a stdio child SIGKILL regression test.

## Best Possible Solution

Land a small MCP runtime fix that wires transport-close eviction before SDK connect, retires the cached runtime only for unexpected closes, preserves healthy multi-server sessions during intentional startup cleanup, and updates current-main tests plus changelog coverage.

## Reproduction Assessment

Yes. The original bug can be reproduced by materializing a session-scoped stdio MCP probe, killing the child process, and observing that current main keeps returning the cached runtime; the PR regression is reproducible with a multi-server config where one server fails during startup while another should stay usable.

## Solution Assessment

No, not yet. The callback-before-connect approach matches the SDK chaining contract, but the implementation must guard intentional closes and the test imports must be reconciled with current main before this is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard intentional closes before retiring the runtime:** `src/agents/pi-bundle-mcp-runtime.ts:277-280`
  - body: This handler also runs when OpenClaw intentionally closes a transport in the startup/listing failure path, because the MCP transport contract says `onclose` fires from `close()` and OpenClaw's stdio `close()` waits for the child `close` event. In a multi-server config, one failed server cleanup can therefore retire the whole in-flight runtime and dispose healthy server sessions. Mark intentional cleanup before closing and make the handler no-op when the session is already absent.
  - confidence: 0.9
- **[P2] Update the test helper imports for current main:** `src/agents/pi-bundle-mcp-runtime.test.ts:5-9`
  - body: Current main's `pi-bundle-mcp-test-harness.ts` no longer exports `makeTempDir`, `waitForFileText`, or `writeBundleProbeMcpServer`, but the new test imports all three from that module. Merging the stale branch as-is will fail test typecheck/module resolution; reintroduce/re-export those helpers or adapt the test to the current helper layout.
  - confidence: 0.93
