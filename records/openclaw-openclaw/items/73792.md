## Summary

Keep open for the existing review path: current main still has the stale cached stdio MCP runtime bug, but the submitted patch has a concrete lifecycle regression on intentional startup cleanup and still needs a changelog/test-helper rebase before merge.

## What This Changes

The PR wires session MCP transport close/error callbacks into runtime-cache eviction, refactors manager disposal into a shared helper, logs transport-close retirement, and adds a stdio child SIGKILL regression test.

## Best Possible Solution

Land a rebased narrow fix that evicts only unexpected transport closes, preserves partial multi-server startup behavior, uses current exported test helpers or adds them intentionally, and records the MCP recovery fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes, from source inspection. Current main can cache a stdio MCP session, the transport close path lets the SDK clear the client transport, and later `getOrCreate` returns the cached runtime without checking liveness.

## Solution Assessment

No, not as submitted. Wiring close before connect follows the SDK callback contract, but the patch must distinguish unexpected transport death from intentional cleanup closes and rebase the test/changelog pieces before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard intentional closes before retiring the runtime:** `src/agents/pi-bundle-mcp-runtime.ts:277-279`
  - body: This handler also runs when OpenClaw intentionally closes a transport during startup failure cleanup, because `close()` fires the same transport `onclose` callback. In a multi-server config, one failed server can trigger `onTransportClosed` while creation is still in flight, then dispose the whole runtime after healthy servers connect, so the caller receives a runtime that immediately throws `bundle-mcp runtime disposed`. Mark intentional cleanup before closing and make this handler no-op when the session was already removed.
  - confidence: 0.92
- **[P2] Import helpers from a current exported module:** `src/agents/pi-bundle-mcp-runtime.test.ts:5-10`
  - body: Current main's `pi-bundle-mcp-test-harness` only exports `cleanupBundleMcpHarness`, so the added imports for `makeTempDir`, `waitForFileText`, and `writeBundleProbeMcpServer` will not resolve as written. Rebase the test onto the current helper layout or intentionally export these helpers from the harness with cleanup coverage.
  - confidence: 0.94
- **[P2] Add the required changelog entry:** `src/agents/pi-bundle-mcp-runtime.ts:277-279`
  - body: This is a user-facing MCP recovery fix, but the PR only changes runtime code and tests. OpenClaw policy requires a single-line `CHANGELOG.md` Fixes entry for user-facing fixes before merge.
  - confidence: 0.87
