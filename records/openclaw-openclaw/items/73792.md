## Summary

Keep open: current main and the latest release still have the stale cached bundle-MCP runtime path, and this PR targets the right lifecycle seam, but the submitted patch can retire healthy multi-server runtimes on intentional cleanup closes and its new test imports do not match current exports.

## What This Changes

The PR adds MCP transport close/error hooks in session MCP runtime, evicts cached runtimes through the manager on transport close, logs retirement, and adds a SIGKILL-based regression test.

## Best Possible Solution

Land a rebased narrow fix that evicts only unexpected transport closes, preserves multi-server startup cleanup behavior, uses current test helper boundaries, and lets maintainers add the release changelog entry during landing.

## Reproduction Assessment

Yes, by source inspection. Current main can return a cached session runtime whose MCP client transport was cleared after child close, so a later tool call reaches the SDK request path and rejects with Not connected.

## Solution Assessment

No, not as submitted. Installing the handler before connect matches the SDK chaining contract, but the patch must guard intentional cleanup closes and fix stale test imports before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Guard intentional closes before retiring the runtime:** `src/agents/pi-bundle-mcp-runtime.ts:276-279`
  - body: This callback also runs for intentional transport.close() because the MCP Transport contract says onclose fires when close() is called, and OpenClaw's stdio close path waits on child close. In the per-server startup-failure catch path, disposeSession(session) runs before sessions.delete(serverName), so one failed server can trigger disposeSessionInternal for the whole session and dispose healthy MCP connections; mark intentional cleanup before closing and no-op when the server was already removed.
  - confidence: 0.92
- **[P2] Import probe helpers from exported modules:** `src/agents/pi-bundle-mcp-runtime.test.ts:5-10`
  - body: Current main's pi-bundle-mcp-test-harness only exports cleanupBundleMcpHarness, so the added imports for makeTempDir, waitForFileText, and writeBundleProbeMcpServer will not resolve. Rebase the test onto the current helper layout, for example by importing the probe server helper from bundle-mcp-shared.test-harness and temp helpers from the existing test helper module or by intentionally exporting them.
  - confidence: 0.95
