## Summary

Keep this PR open: the reported MCP stale-runtime bug is real by source inspection, but the submitted patch still has a correctness regression in intentional close handling and the added test imports helpers that current main does not export.

## What This Changes

The PR adds MCP transport close/error hooks to session MCP runtime, routes close callbacks into manager-side runtime eviction, logs retirement, and adds a SIGKILL regression test.

## Best Possible Solution

Land a rebased narrow fix that evicts only unexpected transport closes, preserves multi-server startup cleanup, uses current test helper boundaries, and leaves changelog handling to maintainers during landing.

## Reproduction Assessment

Yes by source inspection. Current main can return a cached session runtime whose MCP client transport was cleared after child close, so a later tool call reaches the SDK request path and rejects with Not connected.

## Solution Assessment

No, not as submitted. Installing the handler before connect is the right direction, but the patch must guard intentional cleanup closes and rebase the test helper imports before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Guard intentional closes before retiring the runtime:** `src/agents/pi-bundle-mcp-runtime.ts:276-279`
  - body: This handler also runs for intentional transport.close(), and the startup-failure catch path still calls disposeSession(session) before sessions.delete(serverName). In a multi-server config, one failed server can therefore trigger disposeSessionInternal for the whole session and dispose healthy MCP connections; mark intentional cleanup before closing and no-op the handler when the server was already removed.
  - confidence: 0.92
- **[P2] Import probe helpers from exported modules:** `src/agents/pi-bundle-mcp-runtime.test.ts:5-10`
  - body: Current main's pi-bundle-mcp-test-harness only exports cleanupBundleMcpHarness, so the added imports for makeTempDir, waitForFileText, and writeBundleProbeMcpServer will not resolve. Rebase the test onto the current helper layout, such as importing the probe server helper from bundle-mcp-shared.test-harness and temp helpers from test/helpers/temp-dir.ts, or intentionally export the helpers here.
  - confidence: 0.95
