## Summary

Keep open. Current main still lacks unexpected MCP transport-close eviction, and the PR addresses a valid stale-session bug, but the proposed patch is not merge-ready because its close handler also fires on intentional startup cleanup and the new test imports helpers no longer exported on current main.

## What This Changes

The PR wires MCP transport close/error callbacks into session MCP runtimes, refactors manager disposal for close-triggered eviction, logs transport-close retirement, and adds a stdio child SIGKILL regression test.

## Best Possible Solution

Land a small MCP runtime fix that wires transport-close eviction before SDK connect, retires cached runtimes only for unexpected closes, preserves healthy multi-server sessions during intentional startup cleanup, and adds current-main regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. The original bug can be reproduced by materializing a session-scoped stdio MCP probe, killing its child process, and observing current main keep returning the cached runtime; the PR regression is reproducible with a multi-server config where one server fails during startup while another should remain usable.

## Solution Assessment

No, not as proposed. Callback-before-connect matches the SDK chaining contract, but the implementation must mark intentional cleanup before closing and guard the onclose handler before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard intentional closes before retiring the runtime:** `src/agents/pi-bundle-mcp-runtime.ts:277-280`
  - body: This handler also runs when OpenClaw intentionally closes a transport in the startup/listing failure path, because the MCP transport contract says `onclose` fires from `close()` and OpenClaw's stdio `close()` waits for the child `close` event. In a multi-server config, one failed server cleanup can retire the whole in-flight runtime and dispose healthy server sessions, so mark intentional cleanup before closing and make the handler no-op when the session is already absent.
  - confidence: 0.9

Merging the branch as-is will fail test typecheck/module resolution; reintroduce/re-export those helpers intentionally or adapt the test to the current helper layout.
  - confidence: 0.93
