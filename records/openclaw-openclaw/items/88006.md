## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Make window ordering deterministic across recorders:** `src/trajectory/runtime.ts:322`
  - body: `compareTrajectoryWindowLines` only compares millisecond `ts` and the per-recorder `sourceSeq`/`seq`, but every `createTrajectoryRuntimeRecorder` starts `seq` at 1. If an older recorder and a newer recorder emit events in the same millisecond, a stale flush that runs after the newer flush reads the newer line first, appends the older line, and the stable sort keeps that wrong order because both keys tie. Add a cross-recorder tie-breaker or preserve a monotonic write order, plus a same-timestamp regression test, so exported trajectories do not misorder overlapping recorder events.
  - confidence: 0.86
