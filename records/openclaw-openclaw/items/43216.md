## Summary

Keep open. Current main has restored the duplicate heartbeat suppression half, but the plugin-channel heartbeat fallback remains source-reproducible because `target: "last"` can recover a stored plugin route and then drop it when strict outbound resolution cannot see an unbootstrapped channel registry.

## What This Changes

The issue asks for heartbeat `target: "last"` to preserve stored plugin-channel routes before plugin registry bootstrap and to suppress identical heartbeat text within 24 hours.

## Best Possible Solution

Keep the restored duplicate suppression and add a scoped heartbeat fallback that preserves a trusted stored plugin route for `target: "last"` when strict plugin lookup is unavailable, with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: store an external plugin channel route in session delivery context, leave the active plugin channel registry empty, and resolve heartbeat delivery with `target: "last"`; session recovery can keep the route but strict outbound resolution can convert it to `no-target`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in heartbeat/outbound target resolution and should preserve current account checks, direct-message policy, explicit target validation, allowFrom behavior, and plugin parser/thread handling rather than copying the closed PR’s broader branch churn.
