## Summary

Keep open: current main has restored duplicate heartbeat suppression, but the plugin-channel heartbeat fallback is still incomplete because a recovered session route is revalidated through outbound plugin resolution and can still degrade to `no-target` when the plugin registry is unavailable.

## What This Changes

The issue asks OpenClaw to complete heartbeat plugin-channel fallback for stored session routes and ensure duplicate heartbeat payloads are suppressed within 24 hours.

## Best Possible Solution

Add a narrow heartbeat target fallback that preserves the recovered stored plugin route only when plugin resolution is unavailable, keep the restored dedupe behavior, and cover the fallback with focused outbound target tests.

## Reproduction Assessment

Yes at source level: current main preserves raw plugin-owned session routes in `resolveSessionDeliveryTarget()`, then `resolveHeartbeatDeliveryTarget()` can still discard that route when `resolveOutboundTarget()` cannot resolve an unregistered plugin channel.
