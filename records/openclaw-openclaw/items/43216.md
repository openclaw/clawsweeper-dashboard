## Summary

Keep open. Current main has restored duplicate heartbeat suppression, but the plugin-channel fallback gap remains source-reproducible because heartbeat target="last" can recover a stored plugin route and then drop it when strict outbound resolution cannot see an unbootstrapped channel registry.

## What This Changes

The issue asks to preserve stored plugin-channel heartbeat routes when the plugin registry is unbootstrapped and to suppress identical heartbeat payloads within 24 hours.

## Best Possible Solution

Keep the restored duplicate suppression and add a scoped heartbeat fallback that preserves a trusted stored plugin route for target="last" when strict plugin lookup is unavailable, with real registry-backed regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: store an external plugin channel route in session delivery context, leave the active channel registry empty, and resolve heartbeat delivery with target="last"; session recovery keeps the route but strict outbound resolution can convert it to no-target.

## Solution Assessment

Yes. The narrow maintainable fix belongs in heartbeat/outbound target resolution and should preserve current account checks, direct-message policy, explicit target validation, and plugin parser behavior rather than copying the closed PR’s broader branch churn.
