## Summary

Keep open. Current main restores duplicate heartbeat suppression, but the plugin-channel fallback gap remains source-reproducible: heartbeat target="last" can recover a stored external plugin route and then drop it when strict outbound plugin resolution cannot see the unbootstrapped channel registry; related #46203 is closed and unmerged.

## What This Changes

The issue asks to make heartbeat delivery preserve stored plugin-channel routes before registry bootstrap and suppress duplicate heartbeat text within 24 hours.

## Best Possible Solution

Keep the restored duplicate suppression and add a scoped heartbeat fallback that preserves a trusted stored plugin route for target="last" when strict plugin lookup is unavailable, with real registry-backed regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: store an external plugin channel route in session delivery context, leave the active channel registry empty, and resolve heartbeat delivery with target="last"; session recovery keeps the route, but strict outbound resolution can convert it to no-target.

## Solution Assessment

Yes. The narrow maintainable fix belongs in heartbeat/outbound target resolution, preserving account checks, direct-message policy, explicit target validation, and plugin parser behavior; #46203 is useful context but should not be copied with unrelated generated/config changes.
