## Summary

Keep open: current main and v2026.6.6 restore duplicate heartbeat suppression, but the plugin-channel heartbeat fallback can still turn a recovered stored plugin route into `no-target` when outbound plugin resolution is unavailable.

## What This Changes

The issue asks OpenClaw to make heartbeat delivery preserve stored plugin-channel routes before plugin registry bootstrap and suppress identical heartbeat payloads within 24 hours.

## Best Possible Solution

Keep the restored dedupe path and land a narrow resolver fix that preserves recovered stored plugin-channel routes only in the plugin-unavailable heartbeat fallback case.
