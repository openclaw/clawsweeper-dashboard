## Summary

Keep open. Duplicate heartbeat suppression is restored on current main and in v2026.5.12, but the plugin-channel heartbeat `target: "last"` fallback is still incomplete: the recovered stored route is passed through strict outbound resolution without bootstrap or a trusted stored-route fallback, so an unbootstrapped plugin registry can still degrade to `no-target`.

## What This Changes

The issue asks OpenClaw to make heartbeat `target: "last"` delivery work for stored plugin-channel routes before plugin bootstrap and to preserve 24-hour duplicate heartbeat suppression.

## Best Possible Solution

Add a narrow heartbeat `target: "last"` fallback or bootstrap opt-in at the outbound-resolution boundary that preserves stored plugin route account/thread metadata after strict resolution fails, while leaving duplicate suppression unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: store a plugin-channel route in session metadata, keep the plugin registry empty or unbootstrapped, resolve heartbeat `target: "last"`, and the strict outbound lookup can still return `no-target`.

## Solution Assessment

Yes for the remaining fallback. The maintainable fix is bounded to heartbeat target resolution and regression tests, not a global plugin bootstrap policy or duplicate-suppression rewrite.
