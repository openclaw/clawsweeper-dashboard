## Summary

Keep open. Current main has restored duplicate heartbeat suppression, but the heartbeat plugin-channel `target: "last"` fallback still re-enters outbound target resolution without bootstrap/fallback handling, and the only linked implementation PR is closed unmerged.

## What This Changes

The issue asks for heartbeat `target: "last"` delivery to reuse stored plugin-channel routes before plugin bootstrap and for duplicate heartbeat text to be suppressed within the 24-hour dedupe window.

## Best Possible Solution

Add a narrow heartbeat `target: "last"` fallback or bootstrap opt-in at the outbound-resolution boundary that preserves stored account and thread metadata, while leaving the restored duplicate-suppression path unchanged.

## Reproduction Assessment

Yes, at source level. With a stored plugin-channel route and an unloaded external channel plugin, heartbeat `target: "last"` can recover the session route and then fail in `resolveOutboundTarget()` because the call does not opt into bootstrap.

## Solution Assessment

Yes for the remaining fallback. The maintainable fix is bounded to heartbeat target resolution and regression coverage, not a global plugin bootstrap policy or another duplicate-suppression rewrite.
