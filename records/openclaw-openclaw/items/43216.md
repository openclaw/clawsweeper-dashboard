## Repair Work Prompt

Fix the remaining heartbeat plugin-channel fallback gap from https://github.com/openclaw/openclaw/issues/43216. In current main, `resolveSessionDeliveryTarget()` can recover stored plugin-channel route metadata, but `resolveHeartbeatDeliveryTarget()` still calls `resolveOutboundTarget()` and converts an unavailable plugin/failed normalization into `channel: "none", reason: "no-target"`. Add a narrow fallback in `src/infra/outbound/targets.ts` that uses the already-resolved stored last route only in the heartbeat outbound-resolution failure path, preserves `lastAccountId`/thread semantics, and does not bypass plugin-owned normalization when a plugin is available. Add or update tests in `src/infra/outbound/targets.test.ts` for an empty/unbootstrapped plugin registry and in `src/infra/heartbeat-runner.returns-default-unset.test.ts` only if the duplicate suppression coverage needs adjustment; do not revive the closed PR's unrelated generated config/context changes and do not edit `CHANGELOG.md`.
