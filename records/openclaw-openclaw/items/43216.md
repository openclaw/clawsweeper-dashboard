## Summary

Keep open. Duplicate heartbeat suppression is restored, but current main and the latest release still route heartbeat `target: "last"` through `resolveOutboundTarget()` without `allowBootstrap`, so stored plugin-channel routes can degrade to `no-target` before plugin bootstrap.

## What This Changes

The issue asks to restore end-to-end plugin-channel heartbeat fallback and 24-hour duplicate heartbeat suppression.

## Best Possible Solution

Add a narrow heartbeat `target: "last"` fallback or bootstrap opt-in at the outbound-resolution boundary, preserving stored account/thread metadata and leaving restored duplicate suppression unchanged.

## Reproduction Assessment

Yes, at source level. `resolveHeartbeatDeliveryTarget()` can recover stored plugin route metadata and then call `resolveOutboundTarget()` without `allowBootstrap`, while unloaded external plugins require explicit bootstrap opt-in.

## Solution Assessment

Yes for the remaining fallback. The maintainable fix is bounded to heartbeat target resolution and regression coverage, not a global plugin bootstrap policy or another duplicate-suppression rewrite.
