## Summary

Keep open. Current main and v2026.5.12 restore duplicate heartbeat suppression, but the heartbeat `target: "last"` path still sends the recovered plugin route through strict outbound resolution and can degrade to `no-target` when the plugin registry is not bootstrapped.

## What This Changes

The issue asks to finish heartbeat plugin-channel fallback for stored session routes and restore duplicate heartbeat suppression within the 24-hour dedupe window.

## Best Possible Solution

Add a narrow heartbeat `target: "last"` fallback that uses trusted stored plugin route metadata only after strict resolution fails, while leaving duplicate suppression unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: store a plugin-channel route in session metadata, keep the plugin registry empty or unbootstrapped, resolve heartbeat `target: "last"`, and strict outbound lookup can still return `no-target`.

## Solution Assessment

Yes for the remaining fallback. The maintainable fix is bounded to heartbeat target resolution and regression tests, not a global plugin bootstrap policy or startup migration.
