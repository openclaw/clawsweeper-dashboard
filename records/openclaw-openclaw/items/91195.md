## Summary

Keep open: current main and the latest release still close Feishu streaming cards through dispatcher idle/error cleanup, while post-review Feishu changes did not add recovery for a card that starts and then never reaches settlement. Related Feishu streaming trackers overlap, but none fully supersedes this delayed-turn stuck-open report.

## What This Changes

The issue reports Feishu streaming cards that can remain open forever with an empty body when an agent turn is delayed after upgrading to v2026.6.1.

## Best Possible Solution

Add a Feishu-owned recovery path that safely closes, discards, or visibly falls back when an already-started streaming card loses dispatch settlement, with live proof that legitimate long-running turns still finish normally.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. The reporter supplied concrete v2026.6.1 logs, and source inspection still shows started streaming cards are closed by later idle/error settlement rather than an independent stuck-card recovery owner.

## Solution Assessment

Unclear. The Feishu dispatcher/session lifecycle is the likely owner boundary, but the exact timeout, close, discard, or fallback policy needs live Feishu validation so valid long-running turns are not closed prematurely.
