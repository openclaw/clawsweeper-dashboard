## Summary

Keep open: the report describes a fresh Feishu streaming-card regression in the latest release, and current main still relies on dispatcher idle/error settlement to close an active card rather than providing an independent stuck-card recovery path.

## What This Changes

The issue asks OpenClaw to prevent Feishu streaming cards from remaining permanently open with an empty body when an agent turn is delayed or fails to complete normally.

## Best Possible Solution

Add a bounded Feishu streaming-card recovery path that closes or visibly falls back when dispatch abandons a started card, while preserving normal long-running turns and the existing streaming=false workaround.

## Reproduction Assessment

No, not as a high-confidence current-main runtime repro. The reporter supplied concrete v2026.6.1 logs, and source inspection shows a started card only closes when dispatcher idle/error settlement runs, but this review did not recreate the delayed Feishu turn live.

## Solution Assessment

Unclear. A lifecycle recovery in the Feishu streaming-card owner path looks like the right area, but the timeout/fallback semantics need maintainer and live Feishu validation so normal long turns are not closed prematurely.
