## Summary

Keep open: current main still starts Feishu CardKit streaming before the turn is necessarily settled, and the cleanup/fallback paths remain tied to idle, final, error, or dispatch-completion handling rather than an independent recovery for a started card that never reaches settlement.

## What This Changes

The issue reports a v2026.6.1 Feishu regression where streaming cards can remain permanently open with an empty body when an accepted agent turn is delayed or never completes normal settlement.

## Best Possible Solution

Add a Feishu-owned recovery path that safely closes, discards, or visibly falls back when an already-started streaming card loses dispatch settlement, with live proof that legitimate long-running turns still finish normally.

## Reproduction Assessment

No high-confidence live current-main reproduction was run; reporter logs show the stuck lifecycle, and current source still shows no independent recovery owner after a streaming card starts and misses settlement.

## Solution Assessment

Unclear; the Feishu dispatcher/session lifecycle is the likely owner boundary, but maintainers still need to choose and live-validate the timeout, close, discard, or fallback policy.
