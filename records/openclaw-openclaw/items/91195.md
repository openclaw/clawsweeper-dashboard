## Summary

Keep open: current main and v2026.6.6 still start Feishu streaming cards from partial/reasoning/status output before final settlement, while close/reset is still driven by dispatcher idle or delivery error, so the delayed-turn stuck-open lifecycle is not proven fixed or superseded.

## What This Changes

The issue reports Feishu streaming cards remaining permanently open with empty content when an agent turn is delayed after upgrading to v2026.6.1.

## Best Possible Solution

Add a live-proven Feishu streaming-card recovery path that closes, discards, or visibly falls back when a started card loses dispatch settlement, without prematurely closing valid long-running turns.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. Source inspection shows the relevant lifecycle path: a streaming card can start before final settlement, while close/reset still depends on later idle/error settlement.

## Solution Assessment

Unclear. A Feishu-owned lifecycle recovery is the likely direction, but the exact close/fallback timeout needs maintainer and live Feishu validation to avoid breaking legitimate long-running turns.
