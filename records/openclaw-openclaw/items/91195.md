## Summary

Keep open: current main and v2026.6.6 still have Feishu streaming cards start before final settlement while close/reset is driven by dispatcher idle or delivery error, so this delayed-turn stuck-open lifecycle is not proven fixed or safely superseded.

## What This Changes

Investigate a Feishu streaming card regression where delayed agent turns can leave an empty streaming card open permanently.

## Best Possible Solution

Add a live-proven Feishu streaming-card recovery path that closes, discards, or visibly falls back when a started card loses dispatch settlement, without prematurely closing valid long-running turns.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the relevant current-main lifecycle path: a streaming card can start from reply start, partial text, or reasoning preview, while close/reset still depends on dispatcher idle or error settlement.

## Solution Assessment

Unclear. A Feishu-owned lifecycle recovery is the likely direction, but the exact timeout or fallback boundary needs maintainer and live Feishu validation to avoid breaking legitimate long-running turns.
