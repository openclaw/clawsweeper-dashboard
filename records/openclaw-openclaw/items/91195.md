## Summary

Keep open: current main and v2026.6.9 still start Feishu CardKit streaming before final settlement and only close or fall back through dispatch idle/completion paths, so the delayed-turn stuck-card failure is not proven fixed.

## What This Changes

Reporter asks OpenClaw to fix Feishu streaming cards that can start during a delayed agent turn and remain permanently open/blank instead of closing or falling back.

## Best Possible Solution

Add a Feishu-owned recovery path that safely closes, discards, or visibly falls back for started CardKit streams that miss normal settlement, with live Feishu proof that legitimate long-running turns are not prematurely closed.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. Source inspection plus reporter logs show the vulnerable lifecycle: a card can start before final/idle settlement, while close/fallback run only from later dispatch completion, idle, or error paths.

## Solution Assessment

Unclear until maintainers choose the recovery policy; the Feishu dispatcher/streaming session is the right owner boundary, but an unconditional timeout could close valid long turns.
