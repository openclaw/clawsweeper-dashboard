## Summary

Keep this PR open. It addresses a real Android chat parity gap that current main still has, and prior regex/admin-scope review items were followed up, but the patch is not merge-ready because the model picker is wired through a best-effort `sessions.patch` call that requires `operator.admin` while normal Android operator connections intentionally do not request admin; since `chat.send` has no per-send model field, selecting a model can silently fail or race and the next run can still use the previous/default model.

## What This Changes

This PR expands Android chat with richer session and agent controls, create/delete chat flows, timeline/status rendering, cleaner Telegram/thread labels, gateway session-title sanitization, explicit session mutation support, and related Android/gateway tests.

## Best Possible Solution

Land a narrower, validated Android chat parity slice only after model selection is made an authorized and ordered operation: either await an explicit admin-scoped `sessions.patch` before sends while keeping default operator connects non-admin, or add a reviewed gateway-supported per-send model contract. Then rebase, add the required changelog entry, and let Android/gateway maintainers decide whether to merge the full slice or split follow-up UI work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: connect Android with the default operator scopes, select a different model in the new composer, and send; source inspection shows the model update goes through admin-scoped `sessions.patch` while normal Android connects omit admin, and `chat.send` carries no model parameter.

## Solution Assessment

Is this the best way to solve the issue? No, not as-is. The PR targets the right missing surfaces, but model selection needs an authorized, awaited session mutation path before it can be a reliable Android chat control.
