## Summary

Keep open: this is an active broad Android feature PR, and current main does not already contain the central Android chat parity work. ## What This Changes

The branch expands Android chat with agent/session switching, model and thinking controls, session create/delete, runtime status notices, richer timeline/session labels, tests, related gateway/session-title cleanup, and oc-path formatting churn.

## Best Possible Solution

Land a narrowed, CI-green Android chat update after clearing the sticky model-patch error and adding real Android proof for the model-select-and-send path.

## Reproduction Assessment

Yes, source-level for the review finding. In PR head, a failed sessions.patch leaves modelPatchError set, and awaitPendingModelPatchBeforeSend rejects any later send while that stale field remains non-null.

## Solution Assessment

No. The overall Android chat direction is plausible, but the model-selection guard should scope or clear a failed patch error after rejecting the raced send so normal later sends are not permanently blocked.
