## Summary

Keep open: current main still lacks the Android chat parity surfaces, and the PR remains useful but not merge-ready because the new model picker can race with send and the user-facing change lacks the required changelog entry.

## What This Changes

The PR expands Android chat with richer timeline/status rendering, agent/session controls, create/delete flows, model/thinking controls, session-title cleanup, gateway session mutations, and related Android/gateway tests.

## Best Possible Solution

Repair the model-selection/send ordering and add the required changelog entry, then have Android/gateway maintainers decide whether to land this as one parity slice or split it into smaller upstreamable pieces.

## Reproduction Assessment

Yes for the patch bug: on the PR head, choose a model and immediately send before sessions.patch resolves; the send path sends chat.send without a model field, so the gateway uses the previous session model. Feature parity itself is not a bug report, but current-main source comparison shows the requested Android chat behavior is still missing.

## Solution Assessment

No, not as-is. The PR targets real missing Android chat surfaces and earlier regex/admin-scope review items are fixed, but the model picker must block or await the session patch before send, or the gateway must gain a validated per-send model path.
