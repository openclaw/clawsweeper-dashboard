## Summary

Keep open. Current main still treats `agentRuntime.id: "claude-cli"` as the supported Claude CLI execution selector; `cliBackends` alone configures a CLI backend/fallback and does not switch PI, native `sessions_spawn`, or raw model runs away from the Anthropic provider path.

## What This Changes

The issue requests that PI/native subagent/model-run dispatch prefer a configured Claude CLI backend for Claude OAuth credentials instead of continuing through the direct Anthropic provider path.

## Best Possible Solution

Define one Claude CLI execution contract across chat, PI/native subagents, raw model runs, and fallback paths: either keep and diagnose the explicit `agentRuntime.id` requirement or add a documented opt-in for eligible OAuth paths to route through the configured CLI backend.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; the resolver has no CLI runtime to return, native spawn does not add one, and the attempt reaches `runEmbeddedPiAgent`.

## Solution Assessment

Unclear. The requested inference is plausible for user cost consistency, but it conflicts with the documented provider/model/runtime split, so a maintainer should choose between clearer diagnostics and a deliberate opt-in before implementation.
