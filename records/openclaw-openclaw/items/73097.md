## Summary

Keep open. Current main still makes `agentRuntime.id: "claude-cli"` the supported Claude CLI execution selector; `cliBackends` alone remains backend/fallback configuration and does not make PI, native subagent spawn, or raw model runs switch away from the Anthropic provider path.

## What This Changes

The issue requests that PI/subagent execution honor configured Claude CLI backends for Claude OAuth credentials so subagents and chat use the same transport.

## Best Possible Solution

Define one Claude CLI execution contract across chat, PI/native subagents, raw model runs, and fallback paths: either keep and diagnose the explicit `agentRuntime.id` requirement, or add a documented opt-in that routes eligible Anthropic PI paths through the configured CLI backend.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. With an Anthropic model and `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`, the resolver has no CLI runtime to return and the attempt reaches `runEmbeddedPiAgent`; live billing is not required to prove that transport split.

## Solution Assessment

Unclear. The requested cliBackends inference is plausible, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so a maintainer should choose between explicit diagnostics and a deliberate opt-in before implementation.
