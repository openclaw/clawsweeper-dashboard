## Summary

Keep open. Current main still makes `agentRuntime.id: "claude-cli"` the supported Claude CLI execution selector; `cliBackends` alone defines backend config/fallback behavior and does not switch PI or native subagent dispatch away from the Anthropic provider path.

## What This Changes

The issue requests that PI/native subagent model dispatch honor configured Claude CLI backends so OAuth-backed Claude traffic uses the same CLI transport as chat instead of falling through to direct Anthropic API.

## Best Possible Solution

Define one Claude CLI execution contract across chat, PI, native subagents, raw model runs, and fallback paths: either diagnose missing `agentRuntime.id: "claude-cli"`, or add an explicit opt-in that routes eligible Anthropic PI paths through the configured Claude CLI backend with docs and regression coverage.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; current source leaves CLI resolution undefined and reaches the PI/provider path, so live billing is not needed to prove the transport split.

## Solution Assessment

Unclear. The requested cliBackends inference is plausible, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so a maintainer should choose between explicit diagnostics and a deliberate opt-in before implementation.
