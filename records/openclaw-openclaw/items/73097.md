## Summary

Keep open. Current main still makes `agentRuntime.id` the Claude CLI execution selector while `cliBackends` only configures backend details/fallback entries, so a cliBackends-only Anthropic OAuth setup can still route PI/subagent paths through the direct provider transport.

## What This Changes

Issue requests that PI/subagent and related model-run paths honor configured Claude CLI backends for Anthropic OAuth credentials so they use the same transport as chat.

## Best Possible Solution

Define one documented Claude CLI execution contract across chat, native subagents, raw model runs, and fallback paths: either diagnose the required `agentRuntime.id` clearly or add an explicit maintainer-approved opt-in for eligible Anthropic PI paths to use Claude CLI.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; the resolver has no CLI runtime to return, native subagent spawn forwards no runtime override, and the attempt path can reach `runEmbeddedPiAgent`.

## Solution Assessment

Unclear. Inferring execution transport from `cliBackends` would address the reported billing asymmetry, but current docs, tests, and migration code make `agentRuntime` the selector, so the safer path is a maintainer-approved opt-in or clearer diagnostic.
