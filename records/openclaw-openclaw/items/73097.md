## Summary

Keep open. Current main supports Claude CLI execution when `agentRuntime.id: "claude-cli"` is configured or migrated, but `cliBackends` alone still does not select that transport, and raw model-run style paths are explicitly kept on PI/direct provider execution.

## What This Changes

The issue requests that PI, subagent, and related model-run paths consult configured Claude CLI `cliBackends` or an explicit opt-in so Claude OAuth-backed runs use the same CLI transport as chat.

## Best Possible Solution

Define one documented Claude CLI execution contract across chat, native subagents, raw model runs, and fallback paths: either require/diagnose `agentRuntime.id` clearly or add a deliberate opt-in for eligible Anthropic OAuth paths to use Claude CLI transport.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; the resolver has no CLI runtime to return, native subagent spawn forwards no runtime override, and the attempt path reaches PI/direct provider execution.

## Solution Assessment

Unclear. Inferring from `cliBackends` would address the reported asymmetry, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so maintainers should choose between clearer diagnostics and a new opt-in before implementation.
