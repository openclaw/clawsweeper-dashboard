## Summary

Keep open. Current main still routes Claude CLI execution through the documented `agentRuntime.id` selector, while `cliBackends` only configures a backend once that runtime/provider has already been selected, so the reported asymmetry remains a maintainer runtime-policy question rather than a closeable implemented bug.

## What This Changes

Requests that PI/native subagent and raw model execution derive Claude CLI transport from `cliBackends`/Claude CLI OAuth configuration so they do not fall through to direct Anthropic API billing.

## Best Possible Solution

Define one documented Claude CLI transport contract across chat, PI/native subagents, raw model runs, and fallback paths: either diagnose the required `agentRuntime.id` clearly or add a deliberate opt-in for eligible Anthropic paths to delegate through Claude CLI.

## Reproduction Assessment

Yes for the OpenClaw routing behavior by source inspection. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; the resolver has no CLI runtime to return, the attempt path can reach `runEmbeddedPiAgent`, and native `sessions_spawn` forwards no runtime override.

## Solution Assessment

Unclear. Inferring execution transport from `cliBackends` would address the reported asymmetry, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so maintainers should choose between stronger diagnostics and an explicit opt-in behavior.
