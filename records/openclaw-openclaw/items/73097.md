## Summary

Keep open. Current main still does not derive Claude CLI execution from a `cliBackends`-only Anthropic configuration, while docs and tests make `agentRuntime.id` the execution selector, so this remains a runtime-policy feature question rather than a closeable implemented bug.

## What This Changes

Request the PI, subagent, and model-run execution paths to route eligible Claude OAuth Anthropic calls through configured Claude CLI backends so their transport and billing behavior matches chat.

## Best Possible Solution

Define one documented Claude CLI transport contract across chat, PI/native subagents, raw model runs, and fallback paths: either diagnose the required `agentRuntime.id` explicitly or add a deliberate opt-in for eligible Anthropic paths to delegate through Claude CLI.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; the resolver has no CLI runtime to return, the attempt path can reach `runEmbeddedPiAgent`, and native `sessions_spawn` forwards no runtime override.

## Solution Assessment

Unclear. Inferring execution transport from `cliBackends` would address the reported asymmetry, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so maintainers should choose between stronger diagnostics and a new opt-in behavior.
