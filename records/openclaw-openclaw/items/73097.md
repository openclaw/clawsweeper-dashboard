## Summary

Keep open. Current main supports Claude CLI execution through the explicit `agentRuntime.id: "claude-cli"` contract, and migration/docs point users there, but the reported `cliBackends`-only PI/subagent routing asymmetry is still present. The related policy threads in the issue body are useful context, but they do not supersede this narrower remaining runtime-selection question.

## What This Changes

The issue requests that PI, native subagent, and related execution paths consult configured Claude CLI backends so OAuth-backed Anthropic runs use the same CLI transport as normal chat.

## Best Possible Solution

Define one supported runtime/billing contract and make it consistent across chat, native subagents, raw model runs, channel boot, and fallback paths. The maintainable end state is either explicit `agentRuntime.id: "claude-cli"` plus diagnostics for legacy `cliBackends`-only configs, or a deliberate opt-in policy that routes eligible Anthropic OAuth runs through the configured CLI backend with regression coverage.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. With canonical `anthropic/*`, configured `agents.defaults.cliBackends.claude-cli`, and no `agentRuntime.id`, current source leaves execution on the canonical Anthropic provider and native subagent runs reach the embedded PI path.

## Solution Assessment

Unclear. Inferring CLI transport from `cliBackends` alone may not be the best fix because current docs use `agentRuntime` as the executor-selection contract; an explicit opt-in or doctor repair may be safer than silent inference.
