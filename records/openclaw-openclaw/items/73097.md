## Summary

Keep open. Current main has partial progress around provider/model `agentRuntime` and normal agent attempts can route canonical Anthropic models through Claude CLI, but `cliBackends` alone is still not an execution selector and direct embedded surfaces still have unresolved transport semantics.

## What This Changes

Request Claude CLI `cliBackends` or an explicit policy to make PI/native subagent, raw model, and plugin model calls use the Claude CLI transport for OAuth-backed Anthropic credentials.

## Best Possible Solution

Define one Claude CLI runtime contract across chat, `sessions_spawn`, raw model runs, and plugin model-call helpers, then either migrate users to provider/model `agentRuntime` with clear diagnostics or add a deliberate opt-in CLI delegation policy.

## Reproduction Assessment

Yes for the OpenClaw routing behavior from source inspection: `cliBackends` is not consulted by runtime-policy resolution, raw model runs are forced to PI, and `llm-task` calls the embedded runner directly. The Anthropic billing effect itself remains reporter-provided rather than live-replayed here.

## Solution Assessment

Unclear. Provider/model `agentRuntime` is the cleaner current selector for normal agent attempts, but making `cliBackends` drive PI/raw/plugin transports or adding `preferCliBackend` changes runtime policy and needs maintainer choice.
