## Summary

Keep open. Current main has partially improved the supported model-scoped `agentRuntime` path, but `cliBackends` alone is still not an execution selector and direct embedded/raw paths such as `llm-task` and one-shot model runs remain separate from Claude CLI execution.

## What This Changes

Request CLI-backed Claude OAuth setups to use the same CLI transport for PI/subagent/helper execution instead of falling through to direct Anthropic API calls.

## Best Possible Solution

Define one documented Claude CLI transport contract across normal turns, subagents, helper plugins, raw model runs, and diagnostics, then either route eligible paths through CLI dispatch or fail closed with clear unsupported-config errors.

## Reproduction Assessment

Yes, by source inspection. Current main routes supported model-scoped Claude CLI turns through `runCliAgent`, while `llm-task` still calls `runEmbeddedPiAgent` directly and raw model runs force `agentHarnessId: "pi"`.

## Solution Assessment

Unclear. The current provider/model-scoped `agentRuntime` path is the safer partial solution, but inferring transport from `cliBackends` alone would change billing, auth, fallback, and tool-support semantics and needs maintainer policy first.
