## Summary

Keep open. Current main partially addresses the release-era schema and normal chat/subagent routing gap through provider/model-scoped `agentRuntime`, but it still does not implement the requested `cliBackends`-only inference and direct embedded/raw PI paths remain a separate transport-policy gap.

## What This Changes

Requests that PI/native embedded execution honor configured Claude CLI backends so OAuth-backed Anthropic subagents and related model calls use the same CLI transport as chat instead of direct API.

## Best Possible Solution

Define one documented Claude CLI transport contract across chat, native subagents, direct embedded plugin calls, raw model runs, and fallback paths: either diagnose unsupported `cliBackends`-only configs clearly or add an explicit opt-in for eligible Anthropic text runs to delegate through `claude-cli`.

## Reproduction Assessment

Yes, by source inspection. A `cliBackends`-only Anthropic config is not an execution selector on current main, direct `runEmbeddedPiAgent` callers select agent harnesses instead of CLI backends, and raw model runs are explicitly kept on PI.

## Solution Assessment

No, not fully. Provider/model `agentRuntime.id: "claude-cli"` is the right partial direction for normal agent attempts, but it does not answer whether `cliBackends` should imply transport or how direct embedded/raw model paths should behave.
