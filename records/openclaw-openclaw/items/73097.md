## Summary

Keep open. Current main supports Claude CLI agent-turn routing through `agentRuntime.id: "claude-cli"`, but `cliBackends` alone remains a backend/fallback configuration surface and raw model runs still bypass CLI routing, so the requested policy is not fully implemented.

## What This Changes

The issue asks PI/subagent and related model execution paths to honor configured Claude CLI backends so the same OAuth credential does not split across CLI and direct Anthropic API transports.

## Best Possible Solution

Define one Claude CLI execution contract across chat, subagents, fallback, and raw model-run surfaces: either require and diagnose `agentRuntime.id: "claude-cli"`, or add an explicit opt-in that routes eligible Anthropic OAuth paths through the configured CLI backend with docs and regression coverage.

## Reproduction Assessment

Yes for routing behavior. With an Anthropic model and `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`, current source leaves CLI resolution undefined and reaches the PI/provider path; live Anthropic billing is not required to verify that split.

## Solution Assessment

Unclear. The requested `cliBackends` inference is plausible, but current docs, migration, and tests make `agentRuntime` the execution selector, so maintainers should choose between stronger diagnostics and an intentional CLI-routing opt-in.
