## Summary

Keep open. Current main partially addresses normal agent-turn routing through provider/model-scoped `agentRuntime`, but the requested `cliBackends`-only transport inference is still not implemented and direct embedded/raw PI paths remain separate from Claude CLI execution.

## What This Changes

The issue requests that Claude CLI backend configuration prevent subagents, raw model runs, and embedded plugin calls from falling back to direct Anthropic API transport for the same OAuth credential.

## Best Possible Solution

Define one documented Claude CLI transport contract across normal turns, subagents, direct embedded plugin calls, raw model runs, and fallback paths, either by explicitly supporting eligible CLI delegation there or by diagnosing unsupported configurations clearly.

## Reproduction Assessment

Yes, by source inspection. Current main still keeps raw model runs on PI, `llm-task` still calls `runEmbeddedPiAgent` directly, and `cliBackends` alone is not an execution selector.

## Solution Assessment

No, the issue's proposed `cliBackends` inference is not yet clearly the safest fix. Provider/model `agentRuntime.id: "claude-cli"` is the current partial path, but direct embedded/raw behavior needs maintainer runtime and billing-policy direction before implementation.
