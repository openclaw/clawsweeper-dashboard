## Summary

Keep open. Current main still makes `agentRuntime.id: "claude-cli"` the supported Claude CLI execution selector, while `cliBackends` alone only configures available CLI backends and does not make PI, raw model runs, or native subagent dispatch leave the Anthropic provider path.

## What This Changes

The issue requests that PI/native subagent model dispatch honor configured Claude CLI backends so Claude OAuth traffic uses the same CLI transport as chat.

## Best Possible Solution

Define one explicit Claude CLI execution contract across chat, PI, native subagents, raw model runs, and fallback paths: either diagnose missing `agentRuntime.id: "claude-cli"` clearly, or add a documented opt-in that routes eligible Anthropic PI paths through the configured Claude CLI backend with regression coverage.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. With an Anthropic model and `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`, current source leaves CLI runtime resolution undefined and reaches the PI/provider path; live billing is not needed to verify that split.

## Solution Assessment

Unclear. The requested `cliBackends` inference is plausible, but current docs, migration code, and tests make `agentRuntime.id` the executor-selection contract, so a maintainer should choose between stricter diagnostics and an explicit opt-in before implementation.
