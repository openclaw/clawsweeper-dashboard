## Summary

Keep open. Current main still uses `agentRuntime` or a session runtime override to select Claude CLI execution, while `cliBackends` remains backend/fallback configuration, so a cliBackends-only Anthropic setup can still reach the PI/direct provider path for subagent execution.

## What This Changes

Request PI/native subagent execution to honor Claude CLI backend configuration so OAuth-backed Anthropic runs use the same CLI transport as chat instead of direct Anthropic API.

## Best Possible Solution

Define one documented Claude CLI execution contract across chat, PI/native subagents, raw model runs, and fallback paths: either diagnose the required `agentRuntime.id` explicitly or add a deliberate opt-in for eligible Anthropic PI paths to delegate through Claude CLI.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; the resolver has no CLI runtime to return, the attempt path reaches `runEmbeddedPiAgent`, and native `sessions_spawn` forwards no runtime override.

## Solution Assessment

Unclear. Inferring from `cliBackends` would address the reported billing asymmetry, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so maintainers should choose between clearer diagnostics and a new opt-in before implementation.
