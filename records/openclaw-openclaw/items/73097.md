## Summary

Keep open. Current main still routes Claude CLI execution through an explicit `agentRuntime.id` or CLI provider selector, while `cliBackends` alone does not switch Anthropic PI/subagent execution away from the provider path.

## What This Changes

Request PI and subagent routing to honor configured Claude CLI backends for OAuth-backed Anthropic runs so chat and spawned-agent paths use the same transport.

## Best Possible Solution

Define one Claude CLI execution contract across chat, PI, subagents, raw model runs, and fallback paths: either require and diagnose `agentRuntime.id: "claude-cli"`, or add an explicit opt-in that delegates eligible Anthropic PI paths through the configured CLI backend.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; current source leaves CLI resolution undefined and reaches the PI/provider path, so live Anthropic billing is not required to verify the split.

## Solution Assessment

Unclear. The requested `cliBackends` inference is plausible, but current docs, migration, and tests make `agentRuntime` the executor-selection contract, so a maintainer should choose between better diagnostics and a deliberate opt-in before code changes.
