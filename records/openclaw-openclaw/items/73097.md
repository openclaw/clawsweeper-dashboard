## Summary

Keep open. Current main still treats `agentRuntime.id` as the Claude CLI execution selector and `cliBackends` as fallback/backend configuration, so a cliBackends-only setup can still route PI/subagent execution through the Anthropic provider path; changing that inference needs maintainer runtime-policy input.

## What This Changes

Request PI/native subagent and related model-run paths route eligible Anthropic OAuth calls through configured Claude CLI backends instead of direct Anthropic API transport.

## Best Possible Solution

Define one documented Claude CLI execution contract across chat, PI/native subagents, raw model runs, and fallback paths: either diagnose the required `agentRuntime.id` explicitly or add a deliberate opt-in for eligible Anthropic PI paths to use Claude CLI.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. Configure an Anthropic model with `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id`; the resolver has no CLI runtime to return and the attempt path reaches `runEmbeddedPiAgent`, while native subagent spawn forwards no runtime override.

## Solution Assessment

Unclear. Inferring from `cliBackends` would address the reported billing asymmetry, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so maintainers should choose between clearer diagnostics and a new opt-in before implementation.
