## Summary

Keep open. Current main still treats `agentRuntime.id` as the executor selector and does not derive Claude CLI execution from a `cliBackends`-only Anthropic/OAuth setup, so the reported routing asymmetry remains a real product-policy question rather than an implemented or duplicate cleanup item.

## What This Changes

The issue requests that PI/native subagent execution honor Claude CLI-backed `cliBackends` configuration, or an equivalent opt-in, so subagents and chat use the same transport for Claude subscription credentials.

## Best Possible Solution

Define one documented Claude CLI transport contract across chat, PI/native subagents, raw model runs, and fallback paths: either diagnose the required `agentRuntime.id` explicitly or add a deliberate opt-in for eligible Anthropic paths to delegate through Claude CLI.

## Reproduction Assessment

Yes for the OpenClaw routing behavior: a config with Anthropic models and `agents.defaults.cliBackends.claude-cli` but no `agentRuntime.id` has no CLI runtime for the resolver to return, so the attempt path can reach PI/native provider execution. Live billing reproduction would require Anthropic credentials and was not run.

## Solution Assessment

Unclear. Inferring transport from `cliBackends` would address the reported asymmetry, but current docs, tests, and migration code make `agentRuntime` the executor-selection contract, so maintainers should choose between stronger diagnostics/migration and a new opt-in behavior.
