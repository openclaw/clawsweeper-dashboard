## Summary

Keep open. Current main and the latest release still validate raw stripped `--resources` tokens against lowercase allowed names, so mixed-case valid values fail; the related implementation PRs are closed and unmerged, and no canonical open item supersedes this narrow skill-script improvement.

## What This Changes

Accept mixed-case resource names passed to the skill-creator `init_skill.py --resources` option while preserving validation and order-preserving deduplication.

## Best Possible Solution

Normalize each non-empty resource token by stripping and lowercasing before validation and first-seen deduplication, return canonical lowercase resource names, and add focused `parse_resources()` regression coverage.
