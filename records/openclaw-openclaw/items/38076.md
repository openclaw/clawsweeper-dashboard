In `skills/skill-creator/scripts/init_skill.py`, make `parse_resources()` treat valid resource names case-insensitively by stripping and lowercasing non-empty comma-separated tokens before validation and first-seen deduplication. Preserve canonical lowercase returned resource names and existing allowed values (`scripts`, `references`, `assets`).
