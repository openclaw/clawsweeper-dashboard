The issue asks for `init_skill.py --resources` parsing to accept mixed-case `scripts`, `references`, and `assets` values while preserving validation and order-preserving deduplication.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82693 if it remains correct; otherwise replace it with the same narrow parser normalization and focused regression tests, then close this issue from the merge.

## Reproduction Assessment

Yes. A current-main `python3 -B` parser probe shows lowercase values return the expected list, while mixed-case valid values and case-differing duplicates exit as unknown resources.

## Solution Assessment

Yes. The narrow maintainable fix is to lowercase non-empty `--resources` tokens before validation and deduplication while returning canonical lowercase names; the linked PR targets that path with focused tests.
