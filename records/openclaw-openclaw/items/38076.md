## Summary

Keep open. Current main and the latest release still reject mixed-case `--resources` values, and the linked fix attempts are closed and unmerged, so the requested parser improvement remains actionable.

## What This Changes

Request that skill-creator `init_skill.py --resources` accept mixed-case resource tokens case-insensitively while preserving validation and order-preserving deduplication.

## Best Possible Solution

Normalize each non-empty resource token by stripping and lowercasing before validation and first-seen deduplication, return canonical lowercase resource names, and add focused parser regression coverage.

## Reproduction Assessment

Yes for the observable behavior: a read-only current-main probe shows lowercase values succeed while mixed-case valid values and case-only duplicates exit with unknown-resource errors. As a classification matter, this is a small DX expansion of accepted CLI input rather than a documented contract bug.

## Solution Assessment

Yes. A narrow token normalization in `parse_resources()` plus focused unittest coverage is the smallest maintainable path; no core architecture, plugin API, security, or config change is needed.
