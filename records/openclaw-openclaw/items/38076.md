## What This Changes

The issue requests case-insensitive parsing for `skill-creator` `init_skill.py --resources` values while preserving validation and order-preserving deduplication.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82693 if it remains correct, then close this issue from the merge.

## Reproduction Assessment

Yes. A current-main `python3 -B` parser probe reproduces the issue: lowercase resource tokens succeed, while semantically valid mixed-case tokens exit with an unknown-resource error.

## Solution Assessment

Yes. The narrow maintainable fix is to lowercase non-empty resource tokens inside `parse_resources()` before validation and deduplication, with focused parser regression tests; the active linked PR already targets that path.
