## Summary

Keep open: current main still trusts a raw `agent.wait` timeout in the subagent announce path, and this PR is a targeted fix with regression coverage, but it is not complete because the required user-facing changelog entry is missing.

## What This Changes

The PR passes `childSessionKey` into subagent announce waits, reconciles timeout wait results against persisted child session state, and adds regression coverage for the completed-child timeout race.

## Best Possible Solution

Land the timeout-only reconciliation approach after adding the release-note bullet, then let the linked bug close through the PR merge; broader helper deduplication can be handled separately if maintainers want it.

## Solution Assessment

Yes for the runtime change, but the submission is incomplete. Rechecking persisted child-session state only after a timeout is the narrowest maintainable guardrail for this race; the safer alternative before merge is only to add the missing changelog entry, not to expand the fix.
