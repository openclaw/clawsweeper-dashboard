## Summary

Keep open: current main and v2026.5.19 still expose only ok/error/skipped cron run statuses, delivery-target and required direct-delivery failures still return error, and the linked implementation at https://github.com/openclaw/openclaw/pull/80139 is open and unmerged.

## What This Changes

Cron should distinguish successful task execution from failed result delivery so delivery-only failures surface as a warning rather than a failed cron run.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/80139, then coordinate any broader normalized delivery-reason work through https://github.com/openclaw/openclaw/issues/64413.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main has no warning status, delivery-target/direct-delivery failures still return status "error", and error status still drives cron error counters; I did not run a mutating cron job in this read-only review.

## Solution Assessment

Yes as a product direction, but it is not implemented on main yet. The narrow maintainable path is an additive warning-state contract that preserves fatal execution failures as errors while keeping delivery diagnostics separate.
