## Summary

Keep open: current main still records delivery-target and direct delivery failures as `error`, the latest release predates any warning-status implementation, and the open linked PR is the active but unmerged implementation path.

## What This Changes

The issue requests a cron run status model that distinguishes successful task execution with failed delivery from true execution failure, surfacing delivery-only failures as a warning in Control UI.

## Best Possible Solution

Review, rebase or replace, and land https://github.com/openclaw/openclaw/pull/80139 or an equivalent maintainer-approved warning-status implementation, while keeping https://github.com/openclaw/openclaw/issues/92460 separate for delivery-channel propagation.

## Reproduction Assessment

Yes, source-level. Current main still models cron run status as `ok | error | skipped`, and the delivery-target/direct-send failure paths still return `error`; I did not run a live cron job in this read-only review.

## Solution Assessment

No close yet. The warning-status direction is the right shape for the requested behavior, but the open implementation must be reviewed for the public status-contract and compatibility implications before this issue is resolved.
