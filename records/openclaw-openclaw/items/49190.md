Keep open: current main still lacks the requested cron `warning` status, delivery-target failures still return `error`, and the linked implementation PR remains open rather than merged.

## What This Changes

Request a cron run status model that reports execution success with delivery failure as `warning` instead of `error`, including Control UI rendering and delivery metadata preservation.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80139, with broader delivery-reason taxonomy coordinated through https://github.com/openclaw/openclaw/issues/64413.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main has no warning status, delivery-target and direct-delivery failures still return `error`, and error status still drives cron error counters; I did not run a mutating cron job in this read-only review.

## Solution Assessment

Yes as a product direction, but it is not implemented on main yet. The narrow maintainable path is an additive warning-state contract that preserves fatal execution failures as errors while keeping delivery diagnostics separate.
