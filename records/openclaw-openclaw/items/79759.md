The PR adds Google/Gemini-specific default image input inference to `resolveProviderModelInput` and unit tests for Google fallback model input defaults.

## Best Possible Solution

Fix the normalized Gemini Flash 3 miss in the Google provider dynamic model resolver so the resolved model preserves transport, catalog, and capability metadata, then add regression coverage and redacted runtime proof for the media-understanding image path.

## Reproduction Assessment

No live reproduction was established in this read-only review. Source inspection plus the linked logs provide a credible source-reproducible path: current main can synthesize a text-only configured fallback and the media-understanding image runtime rejects models without `image` input.

## Solution Assessment

No. The intent is right, but the narrow maintainable fix should likely live in the Google provider forward-compatible model resolver rather than adding Google provider IDs to the shared input default helper.
