## Summary

Keep open. Current main has adjacent pruning, staged summarization, compaction-provider hooks, and post-compaction memory/index side effects, but it still does not implement the requested multi-layer architecture; the same author also has an open implementation PR for a subset, so the issue should remain open until that PR is reviewed or the work is split.

## What This Changes

The issue requests adopting several Claude Code-inspired compaction layers: old tool-output pruning before summarization, analysis scratchpad summaries, post-compact restoration, session memory persistence, partial compaction, anti-tool-call guardrails, and transcript recovery references.

## Best Possible Solution

Keep this as an umbrella until maintainers either land or reject the narrowed PR, then split the remaining restoration, memory, partial-compaction, and transcript-recovery work into scoped follow-ups with explicit security boundaries.
