## Summary

Keep open. Current main already has related pieces such as cache-TTL tool-result pruning, staged summarization, memory flushes, post-compaction AGENTS refresh, and context-engine seams, but it does not implement the requested Claude Code-style architecture as a whole; the same author also has open PR #58455 covering only a subset of this issue.

## What This Changes

Request to adopt several Claude Code compaction patterns in OpenClaw, including pre-summarization tool-output pruning, analysis scratchpad summaries, post-compact restoration, stronger user-message preservation, session memory, partial compaction, anti-tool-call guardrails, and transcript references.

## Best Possible Solution

Review and either land, revise, or close PR #58455 for the microCompact, scratchpad, and transcript-reference subset, then split any remaining accepted work into explicit core/context-engine tasks for file restoration, stronger user-message preservation, session memory behavior, and true partial compaction semantics.

## Reproduction Assessment

Not applicable; this is an architecture enhancement rather than a bug report. Current-main verification comes from source/docs comparison and the linked open PR rather than a failing runtime reproduction.

## Solution Assessment

Unclear. A wholesale Claude Code architecture copy is not proven to be the best OpenClaw design; the safer path is narrow, tested changes that fit the existing safeguard, context-pruning, memory, and context-engine boundaries.
