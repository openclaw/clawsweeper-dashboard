## Summary

Keep open: current main and v2026.6.8 already cover meaningful parts of the compaction-quality request, including old tool-result pruning, structured safeguard validation, recent-turn preservation, staged partial-summary recovery, provider seams, transcript rotation, and post-compaction memory sync; the remaining analysis-scratchpad, transcript-reference, restoration, partial-compaction semantics, and security boundaries still need maintainer design.

## What This Changes

The issue requests adopting several Claude Code-inspired compaction layers, including old tool-output pruning, think-then-summarize prompting, post-compaction restoration, user-message preservation, session memory, partial compaction, anti-tool-call guardrails, and transcript recovery references.

## Best Possible Solution

Split this into maintainer-owned design slices: keep the existing pruning, safeguard, provider, transcript, and memory-sync pieces; decide whether to sponsor a narrow prompt/pruning experiment; and design transcript references plus restoration with explicit privacy/session-state policy.
