## Summary

Keep open. Current main and the latest release block the normal busy-submit path, but Ctrl+O still replaces a busy TUI status with a non-busy tools-expanded/collapsed status, and `renderStatus` stops the Loader for non-busy statuses.

## What This Changes

The issue reports that the OpenClaw TUI activity spinner stops animating when input or tool-output toggling happens during an active run.

## Best Possible Solution

Preserve busy Loader ownership while showing transient tool-output notices, and add focused regression coverage for Ctrl+O during streaming/running without changing queue policy or adding configuration.

## Reproduction Assessment

Yes, at source level. Put the TUI in a busy `activityStatus` such as `streaming` or `running`, press Ctrl+O, and the handler stores a non-busy tools status that makes `renderStatus` stop the Loader.

## Solution Assessment

Yes. The narrow maintainable fix is to separate transient tool-output notices from busy run status/Loader ownership, not to change message queue semantics or add a setting.
