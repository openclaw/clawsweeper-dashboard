## Summary

Keep open. Current main still has a source-backed TUI path where Ctrl+O during a busy run replaces the busy status with a non-busy tool-toggle notice, which makes the status renderer stop the Loader animation.

## What This Changes

The issue requests that the TUI loading spinner keep animating when a user queues another message or toggles tool output while an agent run is active.

## Best Possible Solution

Keep the issue open until a focused TUI fix preserves the active busy Loader through transient UI notices, with regression coverage for Ctrl+O and active-run send attempts.

## Reproduction Assessment

Yes, via high-confidence source inspection: put the TUI in a busy activityStatus such as streaming, press Ctrl+O, and the handler stores a non-busy tools expanded/collapsed status that renderStatus uses to stop the Loader. I did not run a live Windows terminal reproduction in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to keep busy run state authoritative while showing transient UI notices, rather than changing queue policy or adding a new setting.
