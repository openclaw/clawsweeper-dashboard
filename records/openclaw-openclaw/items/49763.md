## Summary

Keep open. Current main blocks normal busy-message submits, but Ctrl+O still replaces a busy TUI activity status with non-busy tool-toggle text, and the status renderer stops the loader for non-busy statuses.

## What This Changes

The issue asks for the TUI activity spinner to keep animating when a user submits or toggles tool output while a run is still active.

## Best Possible Solution

Keep the busy run loader state separate from transient tool-output notices, and add a focused regression test for Ctrl+O during streaming/running without changing queue policy or adding config.

## Reproduction Assessment

Yes, at source level. Put the TUI in a busy `activityStatus` such as `streaming` or `running`, trigger Ctrl+O, and the handler stores a non-busy tools status that makes `renderStatus` stop the loader.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve busy loader ownership while showing transient tool-toggle feedback, rather than changing message queue semantics or adding a setting.
