## Summary

## What This Changes

The issue requests that the TUI busy spinner keep animating when a user queues or toggles output while an agent run is still active.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92909 or an equivalent narrow TUI status-preservation fix, then close this issue through the merged closing reference.

## Reproduction Assessment

Yes, source-level: Ctrl+O during an active run calls `setActivityStatus("tools expanded" | "tools collapsed")`, and those values are not busy statuses, so `renderStatus()` stops the loader. I did not run a live Windows terminal reproduction in this read-only review.

## Solution Assessment

Yes: the best fix is a narrow TUI status-preservation change with regression coverage, matching the open linked PR. Queue delivery policy should stay separate unless maintainers intentionally combine it with the gateway queue work.
