## Summary

Current main fixes the central cross-plugin mutation risk by isolating each npm plugin install, but this issue should remain open because Claude ACP failures still lack the wrapper-stderr diagnostics added specifically for Codex, and the supported upgrade behavior for legacy Claude adapters remains a maintainer product decision.

## What This Changes

Prevent unrelated plugin installation from changing a working Claude ACP adapter, preserve compatible ACP setup across upgrades, and expose actionable spawn-failure diagnostics.

## Best Possible Solution

Keep per-plugin npm projects and exact ACPX-owned adapter pins, add redacted Claude wrapper stderr/cause propagation using the established Codex pattern, and document an explicit upgrade policy that preserves configured adapter overrides while clearly stating supported platform behavior.

## Reproduction Assessment

No, not at high confidence on current main: source and history establish the former shared-install risk and adapter migration, but there is no live Android 10/Termux reproduction; the remaining Claude diagnostic gap is directly source-reproducible.

## Solution Assessment

No, the posted config override is a useful supported workaround but not the best complete resolution; installer isolation is now shipped, while Claude-specific error propagation and an explicit adapter compatibility policy remain necessary.
