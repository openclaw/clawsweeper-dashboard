## Label Justifications

- `impact:session-state`: The observed failure is sessions_spawn(runtime="acp") returning only a generic internal error for the configured Claude ACP agent.

## Summary

Keep open: current main has moved ACPX to an exact plugin-local Claude ACP package and documents an explicit command override, but it still does not settle the compatibility policy for existing working Claude ACP adapter choices or clearly solve the reported generic Claude spawn diagnostics.

## What This Changes

The issue reports that installing an unrelated plugin can rewrite the ACPX Claude adapter from a previously working legacy package to the newer package and break ACP session spawn on Termux.

## Best Possible Solution

Define the ACPX upgrade policy for existing Claude adapter setups, then either document the override-only stance as intentional or add a narrow doctor/update path plus Claude stderr diagnostics that preserves working adapters until an explicit upgrade.

## Reproduction Assessment

No high-confidence live reproduction was established because the report depends on Android Termux plus an older working Claude ACP adapter. Source inspection supports the core path: plugin installs run managed npm convergence, ACPX startup rewrites generated wrappers, and Claude errors do not get the Codex stderr-tail diagnostics path.

## Solution Assessment

Unclear: the documented command override may be an acceptable operator-owned solution for legacy adapters, but the repository still needs a maintainer decision on whether ordinary plugin installs must preserve or migrate previous working ACP adapter choices automatically.
