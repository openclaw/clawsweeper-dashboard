## Label Justifications

- `P1`: The report describes a fresh user-facing regression where installing an unrelated plugin can break ACP session spawning for an existing setup.
- `impact:session-state`: The observed failure is through `sessions_spawn(runtime="acp")`, leaving ACP session creation unusable for the configured Claude agent.

## Summary

Keep open: the report is fresh, concrete, and still plausible on current main because plugin install runs managed npm convergence while ACPX startup rewrites the Claude wrapper from the currently resolved installed adapter. Current main documents a manual command override and has moved to a newer patched Claude ACP package, but it does not establish an upgrade-safe policy for preserving an older working adapter or surfacing Claude-specific spawn diagnostics.

## What This Changes

Prevent `openclaw plugins install` from silently changing an existing ACP Claude adapter and improve diagnostics when ACP spawn fails.

## Best Possible Solution

Define an upgrade-safe ACPX policy where ordinary plugin installs preserve existing ACP adapter choices unless an explicit update or doctor flow changes them, and extend redacted diagnostics to Claude ACP failures.

## Reproduction Assessment

No high-confidence live reproduction was established because the report depends on Android Termux plus a prior legacy Claude ACP adapter. Source inspection does support the core mechanism: plugin install can converge the shared npm root, and ACPX startup rewrites the generated Claude wrapper from the installed adapter.

## Solution Assessment

Unclear: current main pins a patched newer Claude ACP package and documents command overrides, but that does not settle whether ordinary plugin installs should preserve older working adapters. The safer end state is a maintainer-approved ACPX/package policy plus a narrow Claude diagnostics improvement.
