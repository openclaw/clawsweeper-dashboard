## Label Justifications

- `P1`: The report describes a user-facing regression where installing an unrelated plugin can leave ACP `sessions_spawn` unusable for an existing Claude setup.
- `impact:session-state`: The observed failure is in ACP session creation and can prevent the configured Claude ACP session from starting.
- `impact:auth-provider`: The affected path changes ACP runtime adapter resolution and the generated Claude ACP wrapper package target.

## Summary

Keep open: current main has moved to a pinned bundled Claude ACP adapter and documents override paths, but it does not settle whether unrelated plugin installs must preserve an existing working adapter or provide Claude-side diagnostics. The recent workaround comment is useful context, but it is not enough evidence for a cleanup close while the compatibility policy remains unresolved.

## What This Changes

The issue reports that installing an unrelated plugin can replace the working Claude ACP adapter wrapper with a newer package and leave `sessions_spawn(runtime="acp")` failing with only `Internal error`.

## Best Possible Solution

Define an upgrade-safe ACPX policy where unrelated plugin installs do not silently replace an existing working Claude ACP adapter unless an explicit update or doctor path owns that change, and add redacted Claude ACP diagnostics for generic spawn failures.

## Reproduction Assessment

No high-confidence live reproduction was established because the report depends on Android Termux plus an older working adapter setup. Source inspection does show the relevant mechanisms: plugin install performs managed npm convergence, ACPX startup rewrites the Claude wrapper from the resolved adapter, and Claude failures lack the Codex-only stderr augmentation path.

## Solution Assessment

Unclear: relying on documented agent-command overrides may be acceptable, but it does not answer whether unrelated plugin installs should preserve an existing working adapter by default. The best fix needs a maintainer policy decision plus a narrow diagnostics improvement if the compatibility behavior is kept.
