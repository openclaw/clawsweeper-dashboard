## Summary

Keep open: the latest head appears to be a valid performance fix with no blocking code findings, and the contributor has now posted before/after timing proof. It still needs normal maintainer handling because the PR intentionally removes shipped runtime fallback behavior and live CI was still in progress during review.

## What This Changes

The PR removes per-access case-insensitive session-store key scans from gateway/session/plugin host lookups, adds startup migration coverage for mixed-case keys, and gates embedded TUI session reads on that migration.

## Best Possible Solution

Land the exact-only runtime cleanup after green CI if maintainers accept startup/local migration as the compatibility boundary for legacy mixed-case session keys.

## Reproduction Assessment

Yes. Current main scans store keys case-insensitively inside the per-access resolver, and the PR proof gives a large-store before/after timing path over the exported resolver; I did not run the benchmark locally in this read-only review.

## Solution Assessment

Yes, with the compatibility caveat. Moving legacy mixed-case convergence to startup/local migration is the cleaner owner boundary for the hot path, but maintainers need to accept that migration-only upgrade boundary before merge.
