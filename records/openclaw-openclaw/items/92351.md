## Label Justifications

- `P1`: The reported regression leaves a connected gateway unable to answer inbound messages after an in-place update or rollback.
- `merge-risk: 🚨 message-delivery`: The patch changes failure and retry semantics on central inbound reply loaders while leaving earlier reply-dispatch imports unguarded.

## Summary

The stale-dist failure is credible, but this branch is not merge-ready: it guards only inner `get-reply` loaders while earlier reply-dispatch lazy imports remain exposed, clears an unrecoverable loader so later messages retry the same missing chunk, and lacks after-fix proof from a live package rotation through the actual inbound path.

## What This Changes

The PR adds a stale OpenClaw dist-module classifier, wraps seven auto-reply lazy loaders with cache clearing and restart-required logging, and adds unit tests plus a standalone synthetic proof script.

## Best Possible Solution

Centralize rotated-dist classification and recovery at the gateway process lifecycle boundary, make the response portable across supervisors, prevent repeated retries against permanently missing chunks, and cover both outer reply dispatch and inner runtime imports with a live package-swap inbound-message proof.

## Reproduction Assessment

Yes at the source level: the linked production steps and current lazy-import architecture provide a high-confidence path for a running gateway to reference removed hashed chunks after package replacement. The branch’s claimed recovery outcome is not reproduced end to end.

## Solution Assessment

No. The narrow classifier improvement is sound, but guarding only seven inner loaders and clearing them without process recovery is not the best fix; the repository’s existing lifecycle-owned recovery pattern should cover the complete reply path.

Full review comments:

- **[P1] Cover the outer reply runtime imports:** `src/auto-reply/reply/get-reply.ts:137-153`
  - body: These guards only run after `dispatch-from-config.ts` has successfully imported `get-reply-from-config.runtime.js`. Put rotation handling at a boundary covering the complete reply dispatch path rather than only these inner loaders.
  - confidence: 0.95
- **[P1] Do not clear a loader until restart can repair it:** `src/auto-reply/reply/get-reply.ts:119-126`
  - body: Clearing the promise cannot restore a chunk deleted by the package swap. Each later inbound turn can create another import attempt, fail again, and emit another error/warning pair until the process restarts. Keep this terminal state cached or trigger a single process-owned restart/fail-fast path instead of enabling repeated retries.
  - confidence: 0.94
- **[P2] Use supervisor-neutral restart guidance:** `src/auto-reply/reply/get-reply.ts:125-127`
  - body: The warning prescribes a systemd user service, but OpenClaw also supports launchd, containers, foreground runs, and custom supervisors. In those installations the command is unusable and may misdirect recovery. Route through existing restart ownership or emit portable guidance derived from the active service environment.
  - confidence: 0.92
