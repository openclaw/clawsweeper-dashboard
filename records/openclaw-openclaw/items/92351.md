## Summary

The PR addresses a credible P1 outage, but its guard sits inside `get-reply.ts` while the inbound pipeline first lazy-imports `get-reply.runtime.ts`; dist rotation can therefore fail before any guarded loader runs. The branch also lacks real update/rollback proof, so it is not merge-ready.

## What This Changes

The PR adds a stale-dist module-not-found classifier, wraps seven inner reply-module lazy loads with restart logging and cache clearing, adds classifier unit tests, and adds a synthetic reproduction script.

## Best Possible Solution

Use one gateway-wide stale-dist recovery boundary that detects rotation at top-level dispatch failures and requests a safe supervisor restart, or make update/rollback tooling guarantee restart; add focused outer-loader coverage and real global-install rotation proof.

## Reproduction Assessment

Yes. Install or roll back a global OpenClaw version without restarting the running gateway, then send an inbound Signal message; the reported old hashed chunk import fails while the process remains up. The PR has not demonstrated the after-fix version of this scenario.

## Solution Assessment

No. Guarding selected inner loaders cannot cover a failure while lazy-importing the outer reply runtime, and a single lifecycle-owned restart path is narrower and more maintainable than duplicated request-path policy.
