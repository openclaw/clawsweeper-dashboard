## Summary

Keep open. Current main still lacks SIGPIPE suppression on the macOS exec approval IPC sockets, and the PR targets the right client and accepted-server descriptors, but it still needs the required changelog entry and normal macOS validation before merge.

## What This Changes

The PR adds `SO_NOSIGPIPE` hardening to macOS exec approval IPC socket descriptors and adds a Swift regression test for the descriptor option.

## Best Possible Solution

Land a rebased, validated version that guards the client fd and each accepted server-side fd, keeps the socket-option regression test, and adds the required single-line changelog entry.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection does show current main writes to the client and accepted server sockets without a SIGPIPE guard, which validates the targeted hardening path.

## Solution Assessment

Yes, with merge-readiness caveats. `SO_NOSIGPIPE` on the client and accepted server descriptors is a narrow maintainable fix for this failure mode; the listener-fd guard is harmless but nonessential, and the missing changelog entry still needs to be added.
