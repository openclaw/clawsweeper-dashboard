## Summary

Keep open. Current main still lacks descriptor-level SIGPIPE hardening on the macOS exec approval IPC sockets, and the PR is a focused fix with a scoped regression test, but it still needs the required changelog entry and macOS IPC validation before merge.

## What This Changes

The PR adds a macOS exec approval socket descriptor guard that sets `SO_NOSIGPIPE` on client, accepted server, and listener socket descriptors, plus a Swift regression test for the socket option.

## Best Possible Solution

Land a rebased, validated version of the focused descriptor hardening that guards the client fd and each accepted server-side fd, keeps the socket-option regression test, adds the required changelog entry, and either removes or explicitly accepts the nonessential listener-fd guard.

## Reproduction Assessment

Unclear. The PR provides concrete real-app repro steps and current main has unguarded `FileHandle` socket writes, but the available evidence does not prove this exact SIGPIPE path is the observed app-exit root cause.

## Solution Assessment

Yes, with small merge-readiness caveats. Descriptor-level `SO_NOSIGPIPE` on the client and accepted server fds is the narrow maintainable fix for this failure mode; the listener-fd guard is unnecessary but not a correctness blocker, and the changelog entry is still required.
