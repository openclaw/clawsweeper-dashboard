## Summary

## What This Changes

The PR hardens macOS exec approval socket descriptors with `SO_NOSIGPIPE` and adds a Swift regression test for the descriptor option.

## Best Possible Solution

Land a rebased, macOS-validated version that keeps the client and accepted-fd `SO_NOSIGPIPE` guard plus the focused regression test, while keeping broader exec approval UX changes out of scope.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection does show current main writes to client and accepted Unix-domain sockets without `SO_NOSIGPIPE`, matching the documented no-reader SIGPIPE failure mode. ([developer.apple.com](https://developer.apple.com/library/archive/documentation/System/Conceptual/ManPages_iPhoneOS/man2/setsockopt.2.html?utm_source=openai))

## Solution Assessment

Yes, as a patch direction. Guarding the client fd and each accepted server-side client fd is the narrow maintainable fix for this IPC failure mode; the extra listening-fd guard is harmless but not essential.
