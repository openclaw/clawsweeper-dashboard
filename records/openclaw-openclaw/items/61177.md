## Summary

Keep open: current main still lacks this descriptor guard and the patch direction is narrow, but this external PR is not merge-ready because it still lacks after-fix macOS IPC proof and live metadata reports conflicts.

## What This Changes

The branch adds an ExecApprovalsSocketDescriptorGuard that sets SO_NOSIGPIPE on macOS exec approval sockets and adds a Swift socketpair/getsockopt regression test.

## Best Possible Solution

Land a rebased, macOS-validated version that applies SO_NOSIGPIPE to the client and accepted-fd write paths and keeps the focused descriptor test without expanding approval behavior.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection of current main does show the client and accepted server descriptors write through FileHandle without SO_NOSIGPIPE, which supports the no-reader SIGPIPE failure mode.

## Solution Assessment

Yes, as a patch direction: applying SO_NOSIGPIPE to the client fd and each accepted server-side fd is the narrow maintainable fix for this IPC failure mode. The listening-fd guard is nonessential but harmless, while merge still needs proof and rebase work.
