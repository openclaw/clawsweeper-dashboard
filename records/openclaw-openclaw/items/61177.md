## What This Changes

The PR adds a macOS exec approval socket descriptor guard that enables `SO_NOSIGPIPE` on the client socket, accepted server sockets, and listener, plus a Swift regression test using `socketpair` and `getsockopt`.

## Best Possible Solution

Land a rebased, validated version of the descriptor hardening that guards the client fd and each accepted server-side fd, keeps the socket-option regression test, and adds the required single-line changelog entry.

## Reproduction Assessment

No, not as a full app crash reproduction. Source inspection of current main shows the client request and server responses write to AF_UNIX `FileHandle` sockets without `SO_NOSIGPIPE`, which is enough to validate the targeted failure mode.

## Solution Assessment

Yes, with one merge-readiness caveat. `SO_NOSIGPIPE` on the client and accepted server descriptors is a narrow maintainable fix for this failure mode; the listener-fd guard is harmless, and the missing changelog entry still needs repair. citeturn0search1turn0search4
