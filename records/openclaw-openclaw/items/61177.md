## Summary

Keep open: current main still has unguarded macOS exec-approval socket writes, and this PR is a focused plausible hardening fix, but it still lacks after-fix real macOS approval-flow proof before merge.

## What This Changes

The PR adds a Darwin `SO_NOSIGPIPE` descriptor guard, applies it to the exec approval client and accepted server socket fds, and adds a Swift socketpair regression test.

## Best Possible Solution

Land a refreshed version of this descriptor hardening after redacted real macOS approval-flow proof shows Allow Once returns `exec-res` without app exit or client timeout.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection shows unguarded socket write paths, and Apple's socket contract matches the reported no-reader `SIGPIPE` failure mode.

## Solution Assessment

Yes, as a patch direction: applying `SO_NOSIGPIPE` to the client fd and each accepted server-side fd is the narrow maintainable fix for this IPC failure mode. The listening-fd guard is unnecessary for the central behavior but not a blocking correctness issue.
