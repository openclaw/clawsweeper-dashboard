## Summary

Keep this PR open. Current main still lacks the proposed `SO_NOSIGPIPE` hardening and regression test, while the PR is a focused defensive change for the macOS exec approval IPC path; it needs rebase/merge-conflict follow-up and targeted macOS validation before maintainer landing.

## What This Changes

The PR adds an `ExecApprovalsSocketDescriptorGuard`, applies `SO_NOSIGPIPE` to macOS exec approval client, accepted server, and listener socket descriptors, and adds a Swift regression test that verifies the option is enabled.

## Best Possible Solution

Land a rebased version of this focused hardening that applies SIGPIPE suppression to the client fd and each accepted server-side fd, keeps the descriptor regression test, and either removes or explicitly accepts the listener-fd guard before macOS IPC validation.

## Reproduction Assessment

Unclear. The PR gives a concrete real-app repro path, and current main has unguarded `FileHandle.write` calls on the relevant sockets, but I did not have live macOS app proof that this exact SIGPIPE path is the crash root cause.

## Solution Assessment

Yes, with a small cleanup caveat. Descriptor-level `SO_NOSIGPIPE` on the client and accepted server fds is the narrow maintainable fix for this failure mode; the listener-fd guard should be removed or called out because it does not protect accepted sockets.
