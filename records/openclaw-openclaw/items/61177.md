The PR adds a macOS `SO_NOSIGPIPE` descriptor guard to exec approval client and accepted server sockets, plus a Swift socketpair regression test.

## Best Possible Solution

Land a refreshed version that hardens the client and accepted-fd exec approval write paths with `SO_NOSIGPIPE` and includes redacted real macOS IPC proof that Allow Once returns `exec-res` without app exit.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection does show the implicated current-main write paths, and Apple’s socket contract matches the reported no-reader `SIGPIPE` failure mode.

## Solution Assessment

Yes, as a patch direction: applying `SO_NOSIGPIPE` to the client fd and each accepted server-side fd is the narrow maintainable fix for this IPC failure mode. The listening-fd guard is unnecessary but not a blocking correctness issue.
