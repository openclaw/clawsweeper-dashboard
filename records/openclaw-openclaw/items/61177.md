## Summary

Keep open: the PR is a plausible narrow fix for unguarded macOS exec approval socket writes on current main, and I found no blocking code-review finding. It still lacks after-fix real macOS app IPC proof, so this external PR should not merge until the contributor shows the real Allow Once/exec-res path working after the patch.

## What This Changes

The PR adds a macOS exec approval socket descriptor guard using SO_NOSIGPIPE on created and accepted Unix socket fds, plus a Swift socketpair regression test.

## Best Possible Solution

Land a macOS-validated version that hardens the client and accepted-fd write paths with SO_NOSIGPIPE and keeps the focused descriptor regression test without expanding approval behavior.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection shows current main writes on unguarded client and accepted server sockets, and Apple's SO_NOSIGPIPE contract matches the reported no-reader failure mode.

## Solution Assessment

Yes, as a patch direction: applying SO_NOSIGPIPE to the client fd and each accepted server-side fd is the narrow maintainable fix for this IPC failure mode. The listening-fd guard is extra but harmless, while merge still needs after-fix real macOS proof.
