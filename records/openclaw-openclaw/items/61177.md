## What This Changes

The PR adds a Darwin `SO_NOSIGPIPE` descriptor guard around macOS exec approval Unix socket client/server fds and adds a Swift socketpair regression test.

## Best Possible Solution

Land a refreshed version that hardens the client and accepted-fd exec approval write paths with `SO_NOSIGPIPE` and includes redacted real macOS IPC proof for the Allow Once flow.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection shows unguarded current-main socket write paths, and Apple's socket contract matches the reported no-reader `SIGPIPE` failure mode.

## Solution Assessment

Yes, as a patch direction: applying `SO_NOSIGPIPE` to the client fd and each accepted server-side fd is the narrow maintainable fix for this IPC failure mode. The listening-fd guard is unnecessary for the central behavior but not a blocking correctness issue.
