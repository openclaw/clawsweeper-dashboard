## Summary

## What This Changes

The PR adds a macOS exec approval socket descriptor guard using `SO_NOSIGPIPE` on the client and accepted server fds, plus a Swift socketpair/getsockopt regression test.

## Best Possible Solution

Land a macOS-validated version that hardens the client and accepted-fd write paths with `SO_NOSIGPIPE` and keeps the focused descriptor regression test without expanding approval behavior.

## Reproduction Assessment

No, not as a high-confidence full app crash reproduction. Source inspection does show current main writes on unguarded client and accepted server sockets, which matches the reported no-reader `SIGPIPE` failure mode.

## Solution Assessment

Yes, as a patch direction: applying `SO_NOSIGPIPE` to the client fd and each accepted server-side fd is the narrow maintainable fix for this IPC failure mode. The listening-fd guard is extra but harmless, while merge still needs after-fix real macOS proof.
