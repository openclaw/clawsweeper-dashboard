## Summary

Keep open. The author addressed the earlier ClawSweeper findings, but the latest diff still misses the disabled-by-allowlist installed-index path from #76095: it filters persisted diagnostics while allowing a fresh `channelConfigs` compatibility warning to be generated for disabled installed records.

## What This Changes

The PR suppresses and dedupes non-bundled `channelConfigs` compatibility warnings, filters installed-index diagnostics, and adds regression tests plus a changelog entry.

## Best Possible Solution

Update this PR so disabled installed records cannot emit either persisted or freshly generated `channelConfigs` compatibility warnings, then rebase and land it after targeted registry/config validation tests pass.

## Reproduction Assessment

Yes. The linked issue gives exact startup output, and the remaining gap is reproducible with an installed-index record whose `enabled` is false because the plugin is omitted from a non-empty allowlist while its manifest declares a channel without `channelConfigs`.

## Solution Assessment

No. The PR is the right narrow area, but filtering only persisted installed-index diagnostics does not cover fresh diagnostics emitted while rebuilding the manifest registry for disabled installed records.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Filter fresh disabled-index channelConfig warnings:** `src/plugins/manifest-registry-installed.ts:183-185`

Carry the index enabled state into compatibility suppression or filter generated diagnostics as well.
