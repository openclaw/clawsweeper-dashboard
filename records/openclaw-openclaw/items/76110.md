## Summary

Keep open. The PR targets a valid cosmetic plugin warning regression, but the current diff only handles freshly generated registry diagnostics, leaves persisted installed-index warnings that startup can replay, suppresses replacement diagnostics too broadly, and is missing the required changelog entry.

## What This Changes

The PR threads normalized plugin config into manifest compatibility diagnostics, suppresses the missing `channelConfigs` warning for explicitly disabled plugins, removes same-path replacement diagnostic emission, and adds two manifest-registry tests.

## Best Possible Solution

Deduplicate and suppress only the `channelConfigs` diagnostic for inactive plugins across fresh and persisted installed-index diagnostics, while preserving one warning for active non-bundled plugins and adding focused installed-index/config-validation tests plus a changelog entry.

## Reproduction Assessment

Yes. The linked issue includes exact startup output, and source inspection shows a high-confidence reproduction path through `loadPluginManifestRegistryForInstalledIndex({ includeDisabled: true })` with seeded `index.diagnostics` for a disabled external channel plugin.

## Solution Assessment

No. The proposed helper-level suppression is only a partial fix; the safer solution is to filter/dedupe diagnostics at the installed-index replay boundary and preserve compatibility diagnostics for active replacement records.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Filter disabled installed-index diagnostics:** `src/plugins/manifest-registry.ts:447-449`
  - body: The new early return only suppresses diagnostics generated during this registry pass. Startup/config validation can enter through `loadPluginManifestRegistryForInstalledIndex`, which seeds `params.index.diagnostics` before calling the registry, so a persisted `without channelConfigs metadata` warning will still be surfaced for a disabled plugin and the linked restart warning can remain. Please filter or dedupe seeded diagnostics for disabled plugins and cover that installed-index path.
  - confidence: 0.92
- **[P2] Preserve diagnostics for replacement records:** `src/plugins/manifest-registry.ts:722`
  - body: Removing the same-physical-plugin diagnostic call assumes the first occurrence already emitted all compatibility warnings. That is false when the first same-root candidate is bundled or otherwise warning-exempt and a higher-precedence non-bundled/config record replaces it: the active record can now miss the `channelConfigs` and provider-auth compatibility warnings entirely. Deduplicate by stable diagnostic key instead of skipping replacement diagnostics wholesale.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `CHANGELOG.md:11-15`
  - body: This changes user-visible startup/config warning behavior, but the PR only updates source and tests. Repo policy requires user-facing fixes to add an Unreleased `CHANGELOG.md` entry; add a single-line Fixes bullet without forcing bot or maintainer attribution.
  - confidence: 0.94
