## Summary

Keep open. The linked bug is valid on current main and this PR targets the right plugin diagnostic paths, but the patch introduces a blocking diagnostic-attribution regression when a distinct higher-precedence plugin replaces an earlier non-bundled duplicate.

## What This Changes

The PR suppresses disabled-plugin and stale installed-index `channelConfigs` warnings, deduplicates repeated compatibility diagnostics, adds regression tests, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Merge a narrow fix that preserves disabled-plugin and same-root duplicate suppression while keeping compatibility diagnostics tied to the active winning plugin record for true distinct duplicates.

## Reproduction Assessment

Yes. The linked issue provides concrete startup warning output, and current main source shows the warning is emitted before enabled-state suppression, replayed from installed-index diagnostics, and surfaced by config validation.

## Solution Assessment

No, not as written. Filtering disabled and stale diagnostics belongs in these paths, but deduplication should be scoped to same-root repeats or include enough record identity to avoid retaining diagnostics from an overridden plugin.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Deduplicate only same-root compatibility diagnostics:** `src/plugins/manifest-registry.ts:419-420`
  - body: The new helper dedupes by only `pluginId + message` across the whole registry. If a lower-precedence non-bundled plugin is seen first and a distinct higher-precedence candidate with the same id wins later, the earlier compatibility warning is kept and the winner's warning is dropped, so gateway diagnostics can report the `channelConfigs` or provider-auth warning with the overridden plugin's source. Scope this dedupe to the same-physical-plugin path or include source/record identity so true duplicate replacements keep diagnostics attached to the loaded record.
  - confidence: 0.82
