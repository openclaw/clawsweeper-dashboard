## Summary

Keep open: current main still lacks cron fan-out, and this PR has merge-blocking delivery-mode and session-routing issues before it can be a safe canonical implementation.

## What This Changes

The PR adds cron `delivery.additionalTargets` fan-out across cron types/schema/normalization/runtime dispatch, CLI edit support, tests, and lazy command-session runtime access.

## Best Possible Solution

Land one canonical cron fan-out implementation that keeps schema, normalization, delivery planning, runtime dispatch, trace/status reporting, CLI/UI/docs, changelog, and regression tests aligned with explicit semantics for announce, webhook, no-delivery, primary-success gating, `channel: "last"`, thread targets, and per-target failure isolation.

## Reproduction Assessment

Yes for the review findings. Static diff inspection against current main shows webhook configs can accept `additionalTargets` without a webhook fan-out path, and additional `channel: "last"` targets resolve with the cron run session key instead of the job delivery session key.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best mergeable solution until delivery-mode semantics, job-session routing, current-main rebase drift, docs, and changelog are reconciled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/cron/isolated-agent/run.ts:772-775`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode cron jobs can accept and persist extra targets. The fan-out path only runs when `deliveryResult.sendOccurred === true`, while webhook delivery is posted later by gateway notifications, so webhook jobs silently skip all additional targets. Please reject the field for unsupported modes or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve additional last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:762-767`
  - body: Additional targets can use `channel: "last"`, but this call passes `params.agentSessionKey` into `resolveDeliveryTarget`. Primary cron delivery resolves against the job's configured session key, so session-bound jobs can fan out to the wrong conversation or fail unexpectedly. Pass the job delivery session key through and cover it with a regression test.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/gateway/protocol/schema/cron.ts:191`
  - body: This introduces a user-facing cron delivery feature plus CLI/API surface, but the PR does not update `CHANGELOG.md`. Repo policy requires user-facing feature and fix changes to be reflected in the active changelog before merge.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The patch has no supply-chain changes, but the added outbound fan-out has a concrete wrong-session delivery risk.

Concerns:

- **[medium] Fan-out may resolve the wrong last recipient:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
