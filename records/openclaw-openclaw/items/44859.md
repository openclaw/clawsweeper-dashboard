- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode cron jobs can now accept and persist extra targets. The runtime fan-out is gated on `deliveryResult.sendOccurred === true`, but webhook delivery is emitted later by gateway cron notifications rather than by the direct-send `dispatchCronDelivery` path, so those extra targets are silently skipped. Please reject `additionalTargets` for webhook mode or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: Additional targets can use `channel: "last"`, but the helper resolves them with the cron run session key instead of the configured job delivery session used for primary delivery. Session-scoped jobs can therefore route fan-out to the wrong channel/thread or fail unexpectedly. Pass the job delivery session context into this helper and cover `channel: "last"` with a regression test.
  - confidence: 0.84
