## Summary

Keep this PR open for maintainer review: current main still lacks cron multi-target delivery, but the branch has blocking delivery-mode/session-routing bugs, failed checks, and no inspectable real behavior proof.

## What This Changes

The PR adds an optional cron delivery.additionalTargets field with schema, normalization, runtime fan-out, CLI edit flags, and tests, plus lazy channel-runtime access in commands-session.

## Best Possible Solution

Land one canonical, documented cron fan-out implementation that keeps schema, CLI/UI/docs, delivery planning, runtime dispatch, trace/status reporting, and tests aligned for announce, webhook, none, channel:"last", and per-target failure isolation.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the webhook schema/runtime mismatch and the session-key mismatch for channel:"last" additional targets without needing a live setup.

## Solution Assessment

No. The feature belongs in core if maintainers want cron fan-out, but this patch is not the best merge shape until unsupported modes are rejected or implemented and additional targets resolve against the same job session contract as primary delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject or implement webhook additional targets:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode jobs can save it, but fan-out only runs when `deliveryResult.sendOccurred` is true after direct announce delivery. Webhook jobs never set that flag, so accepted additional targets are silently skipped; either reject this field for webhook/none modes or dispatch fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve extra targets with the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: Additional target resolution passes `params.agentSessionKey`, while the primary delivery path resolves from the job session key. For `channel: "last"` or session-scoped jobs, this can route fan-out from the fresh cron run session instead of the configured chat session; pass the job delivery session key through and cover this with a regression test.
  - confidence: 0.86
