## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/cron/isolated-agent/run.ts:772`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode cron jobs can now accept and persist extra targets. The runtime fan-out is gated on `deliveryResult.sendOccurred === true`, but webhook delivery is completed by gateway notifications rather than this direct-send path, so webhook jobs silently skip all additional targets. Please either reject `additionalTargets` for webhook mode or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:762-767`
  - body: Additional targets can use `channel: "last"`, but this call passes the cron run's `agentSessionKey` into `resolveDeliveryTarget`. Primary delivery resolves against the job's configured session key, so session-bound jobs can resolve additional `last` targets from the wrong fresh cron session or fail closed. Pass the job delivery session key into this helper and cover `channel: "last"` fan-out with a regression test.
  - confidence: 0.86
