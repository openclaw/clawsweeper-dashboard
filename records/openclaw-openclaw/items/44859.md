## Summary

Keep open: current main still has only single-target cron delivery, and this PR is useful but not merge-ready. The latest head still has a webhook-mode contract mismatch, a wrong-session routing concern for `channel: "last"`, missing inspectable real behavior proof, and a security-relevant chance of routing output to the wrong recipient.

## What This Changes

The PR adds cron `delivery.additionalTargets` across type/schema/normalization/delivery planning, runtime fan-out, cron edit CLI support, tests, and a lazy channel-runtime access cleanup.

## Best Possible Solution

Land one canonical, documented cron fan-out contract that aligns schema, normalization, runtime dispatch, status/trace behavior, CLI/UI surfaces, docs, and regression tests, with explicit behavior for announce, webhook, no-delivery, `channel: "last"`, primary-success gating, and per-target failure isolation.

## Reproduction Assessment

Yes for the review findings from source inspection. The PR accepts `additionalTargets` in the shared delivery schema while fan-out only runs from the direct-send path, and additional `channel: "last"` targets are resolved with the cron run session key rather than the job session key.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable solution until supported modes and session-routing semantics are made explicit and fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to shared delivery properties, so webhook-mode cron jobs can accept and persist extra targets. Runtime fan-out only runs after `deliveryResult.sendOccurred`, while webhook delivery is posted later by gateway notifications, so webhook jobs silently skip the additional targets; either reject the field for webhook/noop modes or dispatch fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: Additional targets can use `channel: "last"`, but this call passes the cron run's `agentSessionKey` to `resolveDeliveryTarget`. Primary delivery resolves against the job session key, so session-bound jobs can route fan-out to a fresh or wrong conversation; pass the job delivery session key through and cover it with a regression test.
  - confidence: 0.86
