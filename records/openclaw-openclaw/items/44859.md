## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode cron jobs can accept and persist extra targets. The new fan-out path only runs when direct delivery reports `sendOccurred`, while webhook delivery is completed later by gateway notifications, so webhook jobs silently skip every additional target. Please either reject additional targets for webhook/no-direct-send modes or wire fan-out into the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:762-767`
  - body: Additional targets can use `channel: "last"`, but `deliverToAdditionalTargets` resolves them with the cron run `agentSessionKey` instead of the job delivery session context used by the primary target. Session-bound jobs can therefore send fan-out to the wrong chat/thread or fail unexpectedly; pass the job delivery session key and add a regression test for `last` fan-out.
  - confidence: 0.84
