## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Restrict additional targets to supported delivery modes:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to shared delivery properties, so webhook and no-op delivery configs can accept and persist extra targets. The fan-out path only runs after `deliveryResult.sendOccurred === true`, which webhook delivery never sets because gateway webhook posting happens outside `dispatchCronDelivery`; those extra targets are silently skipped. Either reject the field outside supported modes or add a webhook completion fan-out path too.
  - confidence: 0.9
- **[P2] Resolve additional last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: Primary delivery resolves `channel: "last"` using `params.job.sessionKey`, but the additional-target helper passes `sessionKey: params.agentSessionKey`. For session-scoped cron jobs this can resolve the extra target from the fresh cron run session instead of the configured delivery session, causing fan-out to route to the wrong conversation or fail unexpectedly.
  - confidence: 0.86
