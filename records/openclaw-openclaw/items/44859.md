## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook and no-op delivery configs can now accept and persist extra targets. The runtime fan-out only runs when the primary direct send reports `sendOccurred`, which webhook delivery never does, so those accepted additional targets are silently skipped. Please either reject additional targets outside supported modes or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: Additional targets are resolved with `sessionKey: params.agentSessionKey`, while primary cron delivery resolves from the job's configured delivery session. For `channel: "last"` or session-derived account/thread metadata, this can target the wrong chat/thread or fail unexpectedly for session-scoped jobs. Pass the job delivery session key through this helper and cover `channel: "last"` fan-out with a regression test.
  - confidence: 0.86
