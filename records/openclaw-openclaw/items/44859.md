## Summary

Keep open: this is still a user-facing cron fan-out feature PR, not something already implemented on main, and the current branch has unresolved delivery-mode/session-routing defects plus missing real behavior proof.

## What This Changes

The PR adds cron `delivery.additionalTargets` fan-out across cron types, gateway schema, normalization, delivery dispatch, CLI edit handling, and tests, plus lazy channel-runtime access in session command handling.

## Best Possible Solution

Land one canonical, documented cron output/fan-out contract that keeps schema, normalization, announce/webhook runtime dispatch, session-key routing, traces/status, CLI/UI/docs, tests, and live proof aligned.

## Reproduction Assessment

Yes for the PR findings: source inspection of PR head `556d481b344faf49a2f1c08d789c6a34a663d259` shows webhook `additionalTargets` are accepted without a webhook fan-out path, and `channel: "last"` fan-out resolves with the run session key. The requested multi-target delivery capability itself is a feature request, not a current-main bug reproduction.

## Solution Assessment

No. The branch is a plausible implementation direction, but it is not the best mergeable solution until webhook semantics, job-session routing, status/docs coverage, real behavior proof, and the relationship to the broader cron output-routing contract are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to the shared delivery properties, so webhook-mode cron jobs can validate and persist extra targets. The only fan-out call is gated on direct announce delivery's `sendOccurred`, while webhook delivery is posted later by gateway notifications, so webhook jobs silently skip all additional targets. Restrict this field to supported modes or dispatch fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:754-766`
  - body: Additional targets can use `channel: "last"`, but this helper resolves them and builds the outbound session with the cron run `agentSessionKey`. Primary delivery resolves from the job's configured delivery session, so custom/current-session jobs can send fan-out to the wrong chat/thread or fail unexpectedly. Pass the job delivery session key into this helper and cover it with a regression test.
  - confidence: 0.84

## Security Review

Status: needs_attention
