## Summary

Keep open: current main still lacks cron multi-target delivery, but this PR is not mergeable as-is because webhook-mode configs silently skip additional targets, `channel: "last"` fan-out resolves from the wrong session context, and external real behavior proof is still missing.

## What This Changes

The PR adds a cron `delivery.additionalTargets` fan-out field across the cron contract, normalization, delivery dispatch, CLI edit options, and tests, plus a lazy channel-runtime lookup change.

## Best Possible Solution

Land one canonical cron fan-out implementation that aligns schema, normalization, delivery planning, dispatch, trace/status reporting, CLI/docs/tests, and explicit semantics for announce, webhook, no-delivery, primary-success gating, `channel: "last"`, and per-target failure isolation.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. A webhook-mode cron job can accept `delivery.additionalTargets` through the PR schema, but webhook primary delivery runs through gateway notifications while fan-out only runs from the direct-send `sendOccurred` path; `channel: "last"` fan-out likewise resolves from `params.agentSessionKey` instead of the job delivery session.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the current patch needs a consistent supported-mode decision and the same delivery-session contract for primary and additional targets before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/cron/isolated-agent/run.ts:770-772`
  - body: `additionalTargets` is accepted through the shared delivery schema, including webhook-mode cron jobs, but the runtime only fans out when `deliveryResult.sendOccurred` is true. Webhook primary delivery is posted later by gateway notifications rather than `dispatchCronDelivery`, so webhook jobs silently skip all additional targets; reject the field for webhook mode or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve additional last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:762-767`
  - body: Additional targets can use `channel: "last"`, but this call resolves them with the cron run's `agentSessionKey`. Primary delivery resolves from the job's delivery session, so session-scoped jobs can send extra output to the wrong last channel/thread or fail closed; pass the job delivery session key into this helper and cover it with a regression test.
  - confidence: 0.86
