Full review comments:

- **[P1] Require a real content-block shape before parsing strings:** `src/shared/chat-content.ts:22-23`
  - body: The new guard accepts any JSON array of objects with a `type` key. A valid plain string like `[{"type":"event","payload":"x"}]` now recurses into the array path and returns `null` because there are no `type === "text"` blocks, so callers that treat null as empty content can silently drop message text. Keep the original string unless the parsed array is clearly chat content, such as recognized content-block types with extractable text.
