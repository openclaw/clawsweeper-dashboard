## Summary

Keep open: current main still has the source-level vLLM stringified-content bug, and this PR is useful source work but needs a narrower parser guard, a rebase onto current main's helper shape, and the required changelog entry before merge.

## What This Changes

The PR teaches `extractTextFromChatContent` to parse JSON-stringified content-block arrays and adds colocated unit tests for plain strings, arrays, stringified arrays, and edge cases.

## Best Possible Solution

Land a rebased helper change that recognizes only convincing stringified chat content block arrays, preserves arbitrary JSON strings, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes, source-level. On current main, `extractTextFromChatContent(JSON.stringify([{ type: "text", text: "hello" }]))` follows the plain string path and preserves the JSON text instead of using the existing array extraction path.

## Solution Assessment

No. The direction is right, but the current PR guard is too broad and the branch is stale against current main; the safer solution is a stricter chat-block shape check integrated with the current helper and tests.
