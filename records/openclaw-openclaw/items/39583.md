## Summary

Keep open: the PR targets a real helper-level bug, but the patch is not merge-ready because the current guard can drop unrelated JSON-array strings and the external PR still lacks real vLLM behavior proof.

## What This Changes

The PR parses JSON-stringified chat content block arrays in `extractTextFromChatContent` and adds unit coverage for strings, arrays, stringified arrays, mixed blocks, and option handling.

## Best Possible Solution

Rework the shared helper on current main to parse only high-confidence OpenAI-style content-block arrays, preserve unrelated JSON strings, keep focused regression tests, and add redacted vLLM output proof.

## Reproduction Assessment

Yes at the helper level: current main returns string content unchanged at `src/shared/chat-content.ts:44`, so a JSON-stringified `[{"type":"text","text":"hello"}]` array remains raw JSON by source inspection. The live vLLM user flow has not been proven in this PR.

## Solution Assessment

No. The shared helper is the right surface, but the proposed type-only detection is too broad and should be narrowed before it changes plain-string handling.

## Review Findings
