## Summary

Keep open: current main still has the source-level stringified content-block asymmetry, and this PR targets the right shared helper, but the patch is not ready because its JSON-array guard can drop unrelated string content and it still lacks after-fix vLLM/OpenAI-compatible runtime proof.

## What This Changes

The branch adds JSON parsing for stringified chat content-block arrays in `extractTextFromChatContent` and adds focused helper tests for plain strings, arrays, stringified arrays, mixed content, and edge cases.

## Best Possible Solution

Ship a rebased shared-helper fix that parses only recognized JSON-stringified chat content arrays, preserves plain strings and unrelated JSON arrays, keeps current coercion hardening, adds regression coverage, and includes redacted vLLM runtime proof.

## Reproduction Assessment

Yes at source level: current main returns a raw JSON string for `JSON.stringify([{type:"text",text:"hello"}])` because the string branch never parses JSON, while the array branch extracts `hello`. I did not run a live vLLM setup in this read-only review.

## Solution Assessment

No as currently written; parsing stringified chat blocks is the right narrow direction, but the PR must use a stronger chat-content shape check and preserve current main's coercion hardening before merge.
