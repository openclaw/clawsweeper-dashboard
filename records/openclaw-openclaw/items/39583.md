## Summary

Keep open: current main still treats JSON-stringified vLLM-style content blocks as ordinary strings, but this PR is not merge-ready because its type-only parser can drop arbitrary JSON string content and the contributor has not provided real vLLM behavior proof.

## What This Changes

The PR modifies `extractTextFromChatContent` to parse JSON-stringified arrays of content blocks and adds colocated unit tests for plain strings, arrays, stringified arrays, mixed content, and edge cases.

## Best Possible Solution

Land a rebased or replacement helper change that only recognizes convincing stringified chat content-block arrays, preserves arbitrary JSON strings, adds focused regression coverage, and includes redacted real vLLM proof.

## Reproduction Assessment

Yes, source-level. On current main, a JSON string such as `JSON.stringify([{ type: "text", text: "hello" }])` follows the string branch in `extractTextFromChatContent` and never reaches the existing array text-block extractor.

## Solution Assessment

No. The direction is right, but the PR's `type`-only guard is too broad; the safer fix is a stricter chat-block parser that requires recognized content-block shape and extractable text before leaving the string path.
