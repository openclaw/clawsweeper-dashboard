## Summary

Keep open. The PR still targets a real current-main asymmetry in `extractTextFromChatContent`, but the current head is not merge-ready because its stringified-array heuristic now treats any parsed object array with a `type` key as chat content and can drop legitimate JSON string content with no text blocks.

## What This Changes

The PR adds JSON parsing for stringified chat content-block arrays in `extractTextFromChatContent` and adds colocated unit tests for plain strings, arrays, stringified arrays, mixed blocks, and fallback cases.

## Best Possible Solution

Keep the shared helper as the normalization boundary, but parse string inputs only when the parsed value has a recognizable chat content-block shape, preserve arbitrary JSON strings such as event arrays, mirror the real-array extraction behavior for mixed media/text blocks, add focused regression tests, and include the required user-facing changelog entry before merge.

## Reproduction Assessment

Yes. On current main, `extractTextFromChatContent(JSON.stringify([{ type: "text", text: "hello" }]))` follows the plain-string path and returns raw JSON, while passing the equivalent array extracts `hello`.

## Solution Assessment

No, not as the PR currently stands. Parsing stringified content-block arrays in `extractTextFromChatContent` is the right narrow fix, but the PR head needs a stricter chat-shape guard so non-chat JSON strings are preserved.
