## Summary

Keep open: current main still preserves JSON-stringified content blocks as plain string text, so the reported vLLM bug is real, but this PR should not merge yet because its `type`-only parser can drop arbitrary JSON strings and the contributor has not provided real vLLM proof.

## What This Changes

The PR adds JSON-stringified content-block parsing to `extractTextFromChatContent` and adds shared helper tests for plain strings, arrays, stringified arrays, mixed content, and edge cases.

## Best Possible Solution

Land a rebased parser that only treats convincing stringified chat content-block arrays as blocks, preserves arbitrary JSON strings, adds regression coverage for mixed and non-chat arrays, and includes redacted real vLLM proof.

## Reproduction Assessment

Yes, source-level. On current main, `extractTextFromChatContent(JSON.stringify([{ type: "text", text: "hello" }]))` follows the plain string branch, while array content already extracts text blocks.

## Solution Assessment

No. The direction is right, but the current `type`-only guard is too broad because it can turn unrelated JSON strings into `null`; a stricter chat-block recognizer is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require real chat blocks before parsing strings:** `src/shared/chat-content.ts:21-22`
