## Summary

Keep open: current main still has the stringified content-block bug, and the PR is useful but currently introduces an over-broad JSON-array parse path that can drop unrelated string content.

## What This Changes

The PR adds JSON parsing in `extractTextFromChatContent` for stringified content-block arrays and adds colocated tests for plain strings, arrays, stringified arrays, mixed blocks, and fallback cases.

## Best Possible Solution

Keep the shared helper as the normalization boundary, parse only recognizable chat content-block JSON strings, preserve unrelated JSON strings, mirror mixed media/text array extraction, add regression tests, and include the required changelog entry before merge.

## Reproduction Assessment

Yes, from source inspection. On current main, `extractTextFromChatContent(JSON.stringify([{ type: "text", text: "hello" }]))` takes the string branch and returns raw JSON text, while the equivalent array takes the array branch and extracts `hello`.

## Solution Assessment

No, not as the PR currently stands. Parsing stringified content-block arrays in `extractTextFromChatContent` is the right narrow boundary, but the guard must require recognizable chat-block shape so arbitrary JSON strings are preserved.

## Review Findings
