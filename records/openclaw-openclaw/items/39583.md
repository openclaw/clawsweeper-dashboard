## Summary

Keep open: the linked vLLM double-serialization bug is still source-reproducible on current main, and this PR is directionally useful, but the proposed guard can drop unrelated JSON strings and the external PR still lacks required after-fix real behavior proof.

## What This Changes

This PR adds JSON parsing for stringified chat content-block arrays in `extractTextFromChatContent` and adds focused helper tests.

## Best Possible Solution

Add a conservative stringified content-block parser that only recurses for recognized chat block shapes, preserves unrelated JSON strings, covers mixed and false-positive cases, and attach redacted vLLM/OpenAI-compatible proof.

## Reproduction Assessment

Yes at source level: current main returns JSON-stringified content-block arrays as plain text while real arrays extract text blocks, matching the linked vLLM nested-content symptom. I did not run a live vLLM setup in this read-only review.

## Solution Assessment

No as written; parsing stringified chat blocks is the right direction, but the guard should require recognized chat-content block shapes before leaving the plain-string path and the PR needs real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Narrow JSON-block detection before recursing:** `src/shared/chat-content.ts:21-23`
