## Best Possible Solution

Rebase onto current main, parse only recognized chat content-block arrays while preserving unrelated JSON text, add focused regression coverage, and attach redacted vLLM runtime proof.

## Reproduction Assessment

## Solution Assessment

No as currently written. Parsing stringified supported chat blocks is the right direction, but the guard must validate recognized chat content shapes before switching away from plain-string handling.
