## Summary

Keep open: current main still passes JSON-stringified content blocks through as plain strings, but this PR is not ready to merge because its parser can drop non-chat JSON strings, misses nested serialized block text from the reported shape, and lacks real vLLM runtime proof.

## What This Changes

The PR adds JSON-stringified content-block parsing to `extractTextFromChatContent` and adds unit tests for plain strings, arrays, stringified arrays, mixed blocks, sanitizing, and join behavior.

## Best Possible Solution

Update the shared helper on current main to parse only recognized content-block arrays, recurse through nested serialized text payloads, preserve current text-coercion hardening, add focused regression coverage, and include redacted vLLM runtime proof.

## Reproduction Assessment

Yes, at source level. Current main returns any non-empty string after sanitize/normalize, so JSON-stringified content-block arrays remain raw text, and the PR head source shows the false-positive and nested-serialization paths without needing a live vLLM server.

## Solution Assessment

No. The fix belongs in the shared helper, but this patch should narrow detection to real chat text blocks, unwrap nested serialized text, and preserve current coercion behavior before it is the best merge path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve typed JSON strings that are not chat blocks:** `src/shared/chat-content.ts:21-23`
  - body: The new guard treats any JSON array of objects with a `type` key as chat content. A plain string such as `[{"type":"event","payload":"x"}]` now recurses into the array path and returns `null` because there are no text blocks, where the helper previously preserved the non-empty string; require a recognized chat text shape or at least one extractable text block before switching away from plain-string handling.
  - confidence: 0.9
- **[P2] Unwrap nested serialized block text:** `src/shared/chat-content.ts:25`
  - body: The reported nested output includes a text block whose `text` is itself a serialized content-block array. After this top-level parse, the array path takes that inner JSON string as ordinary text and returns it, so nested or double-serialized content can still leak; recurse through text block payloads or add a focused regression test for that shape.
  - confidence: 0.84
