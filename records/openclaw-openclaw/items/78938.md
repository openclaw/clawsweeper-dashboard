## Summary

Keep open: current main still stacks Codex's native pragmatic model instructions with OpenClaw developer instructions and the friendly GPT-5 overlay, and there is no supported Codex plugin config for choosing the native base-instructions/personality mode.

## What This Changes

Expose Codex harness configuration for selecting the base instruction source and persona mode so OpenClaw avoids conflicting or duplicated prompt layers.

## Best Possible Solution

Define a small, documented Codex prompt policy for the bundled plugin, or explicitly document that Codex native base instructions remain runtime-owned while OpenClaw supports only SOUL.md and GPT-5 overlay controls.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection and prompt snapshots do confirm the current prompt-layer stack that motivated the request.

## Solution Assessment

Unclear until maintainer product direction is set. Exposing native Codex base/personality controls may be the right path, but it changes prompt ownership and public config, so it should not be treated as a narrow bug fix.
