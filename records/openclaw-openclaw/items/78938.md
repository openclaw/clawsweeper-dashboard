## Summary

This should stay open: current main has only adjacent GPT-5 overlay controls, not the requested Codex base-instruction/persona selection, and the prompt policy needs maintainer design rather than conservative cleanup.

## What This Changes

The issue requests Codex plugin settings to choose a single base prompt and persona instead of combining Codex and OpenClaw prompt/persona overlays.

## Best Possible Solution

Design a maintainer-approved Codex harness prompt policy that either reuses the existing GPT-5 overlay knob or adds explicit plugin/agent config for base-instruction and persona selection, with docs and prompt snapshot coverage.

## Reproduction Assessment

Not applicable; this is a configuration/product feature request, not a broken existing contract. Source inspection confirms the relevant current behavior: Codex receives Codex-owned base instructions plus OpenClaw developer instructions and the GPT-5 overlay.

## Solution Assessment

Unclear: exposing both base-instruction and persona settings may be reasonable, but the best shape depends on Codex app-server support and OpenClaw’s prompt-policy defaults. The maintainable boundary is the bundled Codex harness/plugin config, not an automatic cleanup close.
