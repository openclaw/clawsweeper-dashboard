## Summary

This PR should stay open: the two-phase timeout feature is still useful and not implemented on current main, but the branch is not merge-ready against current main's provider timeout, abort, doctor-migration, and QA contracts, and it still lacks real after-fix behavior proof.

## What This Changes

The PR adds first-token versus idle LLM stream timeout handling, a new `agents.defaults.llm.firstTokenTimeoutSeconds` config/schema field, timeout-phase metadata, and focused tests.

## Best Possible Solution

Define first-token timeout semantics on a supported provider/model timeout surface, preserve provider timeout and abort contracts, update schema/docs/doctor and QA detection together, and prove the behavior in a real slow or cold model setup.

## Solution Assessment
