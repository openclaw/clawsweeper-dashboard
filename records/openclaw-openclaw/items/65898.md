## Summary

Keep open, but do not merge this branch as-is. The first-token/idle split is useful, but current main has moved the relevant timeout ownership toward provider-scoped request timeouts; this PR drops current provider/local timeout parameters, adds a new key under a doctor-removed legacy block, leaves QA timeout leak detection incomplete, and lacks after-fix real behavior proof.

## What This Changes

The PR splits embedded-agent LLM stream watchdogs into first-token and mid-stream idle phases, adds `agents.defaults.llm.firstTokenTimeoutSeconds`, propagates timeout phase metadata, and updates schema/test fixtures.

## Best Possible Solution

Define the first-token timeout contract on the supported timeout surface, preserve current provider request and local-provider semantics, update doctor/schema/docs/QA together, and prove the behavior in a real slow or cold model setup.

## Reproduction Assessment

Yes for review purposes: source inspection of current main and the PR diff shows the dropped provider timeout parameters, legacy config conflict, and QA matcher gap. I did not run a live slow-model reproduction in this read-only pass.

## Solution Assessment

No. Splitting first-token and idle windows is a reasonable direction, but this branch is not the best implementation until timeout config ownership is resolved and current provider-scoped timeout behavior is preserved.
