## Summary

Keep open: the two-phase timeout idea is useful, but this branch is not mergeable against current main because it drops current provider/model timeout inputs, loses stream abort plumbing, adds a new knob under a legacy config block, and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds a first-token phase to the LLM stream watchdog, a new `agents.defaults.llm.firstTokenTimeoutSeconds` config field, phase-aware timeout metadata/errors, schema updates, and focused tests.

## Best Possible Solution

Define two-phase timeout semantics on a supported provider/model-scoped surface, preserve provider timeout/local-provider/abort contracts, update doctor/schema/docs/QA together, and prove it in a real slow or cold model setup.

## Reproduction Assessment

Yes for the review defects: current-main source and the PR diff show the omitted provider/model inputs, lost abort plumbing, legacy config surface, and QA matcher gap. The user-facing slow/cold model behavior still lacks real runtime proof.

## Solution Assessment

No. The two-phase watchdog idea is plausible, but this branch needs to target the current supported timeout surface and preserve provider timeout, local-provider, abort, doctor, and QA contracts before it is the right implementation.
