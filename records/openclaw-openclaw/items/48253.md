## Summary

Keep open. Current main has memory-host file helper exports and an active memory plugin slot, but the host-managed scan/watch/change-event lifecycle is still private to memory-core, and the linked backend-abstraction thread does not supersede this narrower ingestion API request.

## What This Changes

Expose a public memory plugin API for host-owned memory file scanning, watching, incremental change detection, and stale cleanup.

## Best Possible Solution

Design an additive, versioned memory file-ingestion contract for the active memory plugin, backed by host-owned scan/watch lifecycle code, contract tests, docs, and a memory-core dogfood implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API gap. Static inspection on current main gives high-confidence evidence that no public host-managed memory file scan/change event contract exists.

## Solution Assessment

Unclear at the exact interface level. The direction fits OpenClaw's plugin-boundary policy, but maintainers should choose the lifecycle, event payload, chunking ownership, and failure semantics before implementation.
