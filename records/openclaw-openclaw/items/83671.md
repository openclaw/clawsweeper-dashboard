## Summary

Keep this open: current main no longer forces code-mode-only by default, but it still documents and supports `appServer.codeModeOnly`, forwards it into Codex thread config, and has no evidence that native command execution now completes when Codex emits the raw `custom_tool_call` path described by the reporter.

## What This Changes

The issue reports that native Codex app-server command execution closes before turn completion when OpenClaw starts Codex threads with `features.code_mode_only=true`.

## Best Possible Solution

Preserve the current default `code_mode=true` command path, then land a narrow Codex app-server fix that either bridges raw code-mode-only tool calls correctly or clearly gates `codeModeOnly` with docs and live proof.

## Reproduction Assessment

Yes. ## Solution Assessment

No patch is proposed here. The maintainable fix should first verify the Codex app-server `code_mode_only` contract, then either bridge the raw tool-call path or disable/gate the setting with tests and live proof.
