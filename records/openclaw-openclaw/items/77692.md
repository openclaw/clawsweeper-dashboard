## Summary

Keep open: current main still leaves the Xiaomi Token Plan TTS failure unresolved, and the linked open PR is an empty branch rather than a landed fix. The latest live comment is useful because it narrows the issue toward Token Plan endpoint/baseUrl resolution, with Bearer auth still a compatible direction but not sufficient by itself.

## What This Changes

Token Plan users report Xiaomi MiMo TTS failing with 401 because the bundled Xiaomi speech provider uses the pay-as-you-go request shape for Token Plan credentials and endpoints.

## Best Possible Solution

Fix the Xiaomi plugin so Token Plan TTS can use the correct regional endpoint from the existing config surfaces, use an auth shape compatible with Token Plan and pay-as-you-go keys, update focused tests, and land or replace the linked PR accordingly.

## Reproduction Assessment

Yes, with a Xiaomi Token Plan key and regional Token Plan base URL the reported Gateway TTS path is concrete, and current source still uses the old request/config resolution. I did not run the live endpoint locally because no redacted Xiaomi key was available in this review.

## Solution Assessment

No, the current linked PR is not the best solution because it has no diff and the latest live data shows endpoint selection/baseUrl inheritance must be covered too. The narrow maintainable fix should stay inside the Xiaomi plugin and its TTS regression tests.
