Keep open: current main and v2026.5.7 still send the Xiaomi TTS request with `api-key`, the TTS resolver does not inherit `models.providers.xiaomi.baseUrl`, and the linked fixing PR is open but empty while later live data points to endpoint handling as the main blocker.

## What This Changes

Xiaomi MiMo TTS should work for Token Plan credentials and regional Token Plan endpoints instead of returning 401s for valid `tp-` users.

## Best Possible Solution

Keep this open until the Xiaomi plugin handles Token Plan-compatible endpoint selection/auth with focused regression coverage and docs for the supported Token Plan setup path.

Yes: the live v2026.5.7 comment gives concrete Token Plan CLI/curl steps, and current main source still sends the same `api-key` and default-endpoint request shape. I did not run a live endpoint check locally because no Xiaomi Token Plan credential was used.

## Solution Assessment

No: the linked PR is not a working fix because it has no diff, and a header-only change may miss the endpoint inheritance problem surfaced by later live data. The narrow maintainable solution is a plugin-local fix with tests for request headers and Token Plan base URL resolution.
