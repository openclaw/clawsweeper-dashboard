## Summary

Keep open: current main and v2026.5.7 still send `api-key` for Xiaomi TTS, tests assert that request shape, and the linked closing PR is open but empty. ## What This Changes

The issue requests fixing Xiaomi MiMo TTS for Token Plan users by changing auth toward Bearer and, after follow-up, ensuring Token Plan base URLs are honored.

## Best Possible Solution

Update the Xiaomi speech provider to resolve Token Plan endpoints correctly, settle Bearer versus `api-key` compatibility from live evidence, cover the request shape and baseUrl fallback in tests, then land or replace the linked PR with docs/changelog coverage.
