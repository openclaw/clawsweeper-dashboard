## Summary

Keep open: current main still does not solve Xiaomi Token Plan TTS, and the latest live issue data points to Token Plan endpoint propagation as the primary failure path. The paired closing PR at https://github.com/openclaw/openclaw/pull/79185 is still open with zero changed files, so this issue should remain open until that PR is fixed, merged, or superseded.

## What This Changes

The issue asks for Xiaomi MiMo TTS to work with Token Plan credentials/endpoints, initially by changing auth to Bearer and later with evidence that Token Plan base URL propagation is the primary blocker.

## Best Possible Solution

Fix Xiaomi TTS endpoint selection in the Xiaomi plugin, likely by honoring explicit TTS baseUrl/env and falling back to the configured Xiaomi model-provider baseUrl, then update tests/docs while keeping auth aligned with current vendor and live proof.

## Reproduction Assessment

Yes. The latest issue comment gives a concrete OpenClaw 2026.5.7 Token Plan gateway command that fails until `XIAOMI_BASE_URL` reaches the Gateway, and current main source still shows Xiaomi TTS does not inherit `models.providers.xiaomi.baseUrl`; I did not rerun it live without a Token Plan key.

## Solution Assessment

No. A header-only Bearer change is no longer the best fix because Xiaomi's current Token Plan docs and the latest live data show `api-key` can work on Token Plan endpoints; the safer fix is endpoint/baseUrl propagation plus regression coverage, with auth changed only if live/vendor proof requires it.
