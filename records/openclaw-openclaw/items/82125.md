## Summary

Keep open. Current main and the latest shipped tag still route native Windows gateway supervision through Scheduled Tasks or Startup-folder launchers, without durable crash restart or a Windows Service adapter; the related watchdog/service reports were closed into this thread, so this remains the canonical maintainer decision point.

## What This Changes

Add Windows gateway crash auto-restart parity for native Windows managed startup, either through a watchdog task or a Windows Service-style supervisor path.

## Best Possible Solution

Define a first-class Windows crash-recovery contract, then implement it consistently across install/update/uninstall/status/doctor with focused tests, docs, and real Windows proof.

## Reproduction Assessment

Yes for the source-level gap: current main and v2026.5.18 create/update only the logon Scheduled Task path, with no watchdog or Windows Service adapter. No live Windows crash-kill reproduction was run here, but the issue includes a concrete Windows 11 field report.

## Solution Assessment

Unclear. The dedicated watchdog task is plausible and the Windows Service field report is useful, but maintainers should choose one Windows supervisor contract before implementation.
