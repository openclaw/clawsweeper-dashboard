## Summary

Keep open. Current main still installs and updates the native Windows gateway as a single logon Scheduled Task, while Linux and macOS service definitions have explicit crash-restart semantics. The requested watchdog is a real feature gap, but the exact Windows lifecycle design needs maintainer review before implementation.

## What This Changes

Request Windows `gateway install` crash auto-restart parity, likely via a dedicated watchdog Scheduled Task that checks the gateway port and launches `gateway.cmd` when needed.

## Best Possible Solution

Define a first-class Windows crash-recovery contract, then implement it in the Scheduled Task install/update/uninstall/status path with focused tests, docs, and real Windows proof.

## Reproduction Assessment

Yes, for the current source behavior: current main creates and updates only the logon Scheduled Task path, with no watchdog or repetition repair. No high-confidence real Windows runtime reproduction was performed in this read-only review.

## Solution Assessment

Unclear. The dedicated watchdog design is plausible, but the best fix should be chosen as part of the Windows service lifecycle contract rather than added only as an install-time side task.
