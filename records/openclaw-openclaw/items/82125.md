## Summary

Keep open. Current main still routes native Windows gateway install/update through a single logon Scheduled Task with no durable crash-restart watchdog, while Linux and macOS service definitions have explicit restart semantics; the requested parity is valid but needs a Windows supervisor lifecycle decision before implementation.

## What This Changes

Add Windows gateway crash auto-restart parity by installing or repairing a durable watchdog Scheduled Task for the gateway.

## Best Possible Solution

Define a first-class Windows crash-recovery contract, then implement it across Scheduled Task install/update/uninstall/status/doctor behavior with focused tests, docs, and real Windows proof.

## Reproduction Assessment

Yes, for the source-level gap: current main creates and updates only the logon Scheduled Task path, with no watchdog or repetition repair. A real Windows runtime reproduction was not performed in this read-only review.

## Solution Assessment

Unclear. The dedicated watchdog design is plausible, but the best fix should be chosen as a Windows service lifecycle contract rather than added only as an install-time side task.
