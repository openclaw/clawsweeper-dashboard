## Summary

Keep open. Current main still routes native Windows gateway management through a single logon Scheduled Task with no crash-restart watchdog, and the related field reports point to the same lifecycle gap; the fix needs a maintainer design decision rather than cleanup closure.

## What This Changes

Add Windows crash auto-restart parity for native gateway installs, likely via a watchdog Scheduled Task or Windows Service path.

## Best Possible Solution

Define a first-class Windows crash-recovery contract and implement it consistently across install, update, stop, uninstall, status, and doctor, with focused tests and real Windows proof.

## Reproduction Assessment

Yes, for source-level reproduction: current main routes win32 gateway install/update to Scheduled Tasks and creates or updates only the logon task path. The live Windows kill-PID scenario is reported in the issue comments but was not rerun in this read-only review.

## Solution Assessment

Unclear. The dedicated watchdog task is plausible, but the Windows Service comments and existing Scheduled Task upgrade paths show this should be decided as a Windows supervisor lifecycle contract, not as an automated one-off install side task.
