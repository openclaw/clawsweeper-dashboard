## Summary

Keep open. Current main still routes native Windows gateway lifecycle through a logon Scheduled Task, while the issue has fresh field evidence and design input showing the crash-recovery gap is real; the remaining question is the Windows supervisor contract, not whether the source gap exists.

## What This Changes

Request Windows gateway crash auto-restart parity with Linux systemd and macOS launchd, likely via a watchdog task or Windows Service adapter.

## Best Possible Solution

Define the Windows gateway crash-recovery lifecycle contract first, then implement the chosen supervisor path across install, update, stop, uninstall, status, doctor repair, tests, docs, and real Windows proof.

## Reproduction Assessment

Yes for the source-level behavior: current main creates and updates only the logon Scheduled Task path, with no watchdog, repetition repair, or Windows Service adapter. No live Windows crash-restart run was performed in this read-only review.

## Solution Assessment

Unclear. The dedicated watchdog design is plausible, but the later Windows Service field report shows this should be decided as a first-class Windows supervisor lifecycle design rather than an automated one-off Scheduled Task patch.
