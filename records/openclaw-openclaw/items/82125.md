## Summary

Keep open. Current main and the latest shipped release still install or update native Windows gateway supervision as a single logon Scheduled Task, while Linux and macOS have explicit restart/keep-alive service semantics. The gap is source-reproducible, but adding a watchdog is a new Windows service lifecycle feature that needs maintainer design review.

## What This Changes

The issue requests Windows gateway crash auto-restart support, likely through a dedicated watchdog Scheduled Task that starts the gateway when port 18789 is not listening.

## Best Possible Solution

Define a first-class Windows crash-recovery contract, then implement it consistently across Scheduled Task install, update, stop, uninstall, status, docs, and focused Windows validation.

## Reproduction Assessment

Yes, for the current source behavior: current main creates and updates only a logon Scheduled Task on Windows, with no watchdog, repeated port check, or crash-restart contract. A real Windows Task Scheduler reproduction was not run in this read-only review.

## Solution Assessment

Unclear. The proposed dedicated watchdog is plausible and is backed by the reporter's production workaround, but the best fix should be chosen as part of the Windows gateway lifecycle contract rather than added only as an install-time side task.
