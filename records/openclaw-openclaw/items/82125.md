## Summary

Keep open. Current main and the latest release still install/update the native Windows gateway as a single logon Scheduled Task with no durable crash-restart watchdog, while Linux and macOS have explicit supervisor restart semantics; the requested Windows lifecycle design is useful but needs maintainer review before implementation.

## What This Changes

Requests Windows gateway crash auto-restart support, likely via a dedicated watchdog Scheduled Task, to match Linux systemd and macOS launchd recovery behavior.

## Best Possible Solution

Define a first-class Windows crash-recovery contract, then implement it across Scheduled Task install, update, stop, uninstall, status, doctor/docs, and real Windows validation.

## Reproduction Assessment

Yes, source-reproducible: current main routes Windows installs to Scheduled Tasks, creates only `/SC ONLOGON`, and updates existing tasks only with `/Change /TR`. No real Windows Task Scheduler run was performed in this read-only review.

## Solution Assessment

Unclear. A dedicated watchdog task is plausible, but the best fix should be selected as part of the Windows supervisor lifecycle contract and coordinated with broader watchdog/recovery work.
