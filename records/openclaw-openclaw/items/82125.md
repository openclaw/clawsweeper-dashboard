## Summary

Keep open. Current main still gives native Windows only a logon Scheduled Task while Linux and macOS have explicit supervisor restart semantics, and the newer field comment adds real Windows evidence that this is broader than a one-line task trigger tweak.

## What This Changes

The issue asks for Windows gateway crash auto-restart parity with Linux systemd Restart=always and macOS launchd KeepAlive.

## Best Possible Solution

Define a first-class Windows gateway supervisor contract, then implement it across install, update, stop, restart, uninstall, status, audit, docs, and upgrade behavior with real Windows proof.

## Reproduction Assessment

Yes for the source-level gap: current main creates and updates only a logon Scheduled Task path with no crash-restart watchdog or Windows Service adapter. I did not run a live Windows crash-restart reproduction in this read-only review.

## Solution Assessment

Unclear. The dedicated watchdog is plausible, but the newer Windows Service field evidence means the maintainable fix should come from an explicit Windows lifecycle decision rather than an install-only side task.
