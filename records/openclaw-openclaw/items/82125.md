## Summary

Keep open. Current main still treats native Windows gateway management as a Scheduled Task, creates an `ONLOGON` task without crash-restart/watchdog semantics, and the closed related reports have already been routed into this issue as the canonical Windows supervision decision point.

## What This Changes

The issue requests Windows gateway crash auto-restart parity with Linux systemd and macOS launchd supervision.

## Best Possible Solution

Define the Windows gateway supervision contract first, then implement a compatible Scheduled Task watchdog or Windows Service path with install/update/status/uninstall coverage and real Windows proof.

## Reproduction Assessment

Yes for the source-level gap: current main maps `win32` to Scheduled Tasks, creates only an `ONLOGON` task, and has no watchdog or Windows Service adapter. No live Windows kill-and-restart reproduction was performed in this read-only review.

## Solution Assessment

Unclear. The dedicated watchdog design is plausible, and the field comment makes a credible Windows Service case, but maintainers should choose the supervision contract before implementation.
