## Summary

Keep open. Current main still manages the native Windows gateway through Scheduled Task / Startup-folder paths without crash auto-restart, while Linux and macOS have explicit supervisor restart behavior; the issue also has fresh Windows 11 field evidence and related reports were intentionally folded into this thread.

## What This Changes

The issue requests Windows gateway crash-restart parity by adding a watchdog or Windows service-style lifecycle instead of relying only on an ONLOGON Scheduled Task.

## Best Possible Solution

Define a first-class Windows gateway supervisor contract, then implement the chosen Scheduled Task watchdog or Windows Service adapter across install, update, stop, restart, status, uninstall, docs, and Windows host proof.

## Reproduction Assessment

Yes for the source-level gap: current main creates and updates only the Windows Scheduled Task path without a watchdog/repetition repair, while Linux and macOS render explicit restart semantics. I did not run a live Windows crash-restart reproduction in this read-only review.

## Solution Assessment

Unclear. A dedicated watchdog task is plausible and has field evidence, but the newer Windows Service findings mean maintainers should decide the Windows supervisor contract before implementation.
