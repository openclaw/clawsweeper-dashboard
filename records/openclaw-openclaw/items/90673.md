## Summary

Keep open: current main has important Codex turn-watch, client-close, and stuck-session recovery primitives, but the exact inter-session `sessions_send` timeout policy and multi-session stall isolation reported here are still not proven solved.

## What This Changes

The issue requests hard Codex `model_call` recovery plus safer `sessions_send` timeout handling so a timed-out delegation cannot leave requester, target, or unrelated sessions stuck.

## Best Possible Solution

Keep this as a core runtime stability item and define one coordinated timeout/recovery policy across Codex turn watches, `sessions_send` wait semantics, target-run cleanup, requester release, and diagnostic recovery.

No high-confidence live current-main reproduction was established. ## Solution Assessment

No. Current main has useful partial recovery, but the best fix needs a maintainer-approved policy for target cancellation or detachment, requester release, and unrelated-session isolation instead of changing one timeout branch in isolation.
