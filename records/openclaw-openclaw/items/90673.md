## Summary

Keep open: current main has useful Codex timeout cleanup, shared-client retirement, and `sessions_send` wait handling, but source and related open reports still do not prove the requested inter-session timeout chain, requester release, target cancellation/detach policy, and unrelated-session isolation are fully fixed.

## What This Changes

The issue asks OpenClaw to recover cleanly when a Codex-backed inter-session `sessions_send` delegation times out and leaves requester, target, or unrelated sessions stalled.

## Best Possible Solution

Keep this as a core runtime stability bug and design one coordinated recovery policy across Codex turn watchdogs, `sessions_send` timeout semantics, requester release, and stalled-run abort-drain, with live or targeted regression proof.

## Reproduction Assessment

No high-confidence live reproduction against current main was established. Source inspection confirms the relevant wait-only `agent.wait`, partial Codex watchdog/cleanup paths, and narrow retry gates, but not the full Docker/NAS multi-session stall.

## Solution Assessment

No, current main is not enough to call the issue solved. The best fix needs maintainer-approved timeout semantics plus coverage for target recovery, requester release, and shared Codex app-server isolation.
