## Summary

Keep open: current main has useful Codex turn watchdogs, timed-out client retirement, and diagnostic abort-drain recovery, but I could not prove the complete inter-session `sessions_send` timeout stall is fixed; the wait-timeout path still soft-detaches work, and related Codex/session stall reports remain open.

## What This Changes

The issue asks OpenClaw to recover cleanly when a Codex-backed `sessions_send` delegation times out, without leaving the requester, target, or unrelated sessions stalled.

## Best Possible Solution

Define one recovery policy for `sessions_send` wait timeouts, add a focused Codex dynamic-tool timeout repro covering requester release, target abort/detach, and unrelated-session isolation, then wire diagnostics and turn interruption to that policy.
