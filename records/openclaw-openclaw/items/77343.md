## Summary

Keep open: current main still has a source-reproducible Barnacle label-refresh bug, and #77361 is already an open closing PR for the same issue.

## What This Changes

Barnacle should recompute and update PR candidate labels when a draft PR is edited or marked ready for review.

## Best Possible Solution

Review and land #77361 or an equivalent narrow fix that adds ready_for_review coverage and reconciles stale Barnacle-owned candidate labels.

## Reproduction Assessment

Yes. From current-main source inspection, a PR with an existing triage: blank-template label can later have a meaningful body but still only pass through addMissingLabels, and ready_for_review is not wired into either the workflow or script allowlist.

## Solution Assessment

Yes. The narrow maintainable fix is to add ready_for_review to both event surfaces and reconcile candidate labels by removing stale Barnacle-owned labels while preserving existing suppression and privileged-author guards.
