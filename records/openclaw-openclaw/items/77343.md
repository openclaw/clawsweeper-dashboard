## Summary

Keep open: current `main` still has the Barnacle label refresh bug, and the linked open PR #77361 is the correct implementation path rather than closing this issue before merge.

## What This Changes

The issue reports that Barnacle leaves stale PR candidate labels after a draft PR is edited and marked ready for review.

## Best Possible Solution

Land or close PR #77361 after review; the desired end state is Barnacle reconciling its own candidate labels on PR edits and `ready_for_review` without broadening automation risk.

## Reproduction Assessment

Yes, from source inspection: a PR with stale `triage: blank-template` can enter the current candidate-label path after its body becomes meaningful, but current `main` only adds missing labels and ignores `ready_for_review`. I did not run a live workflow reproduction.

## Solution Assessment

Yes, the narrow maintainable fix is to add `ready_for_review` to both the workflow and script allowlist, then reconcile only Barnacle-owned candidate labels with regression coverage.
