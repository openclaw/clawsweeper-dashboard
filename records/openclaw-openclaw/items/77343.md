## Summary

Keep open: current main has a source-reproducible Barnacle label refresh bug, because PR candidate labels are add-only and the auto-response workflow does not run on `ready_for_review`.

## What This Changes

The issue asks Barnacle to refresh stale PR triage labels after a draft PR is edited and marked ready for review.

## Best Possible Solution

Update Barnacle to resync its candidate labels on PR edits and `ready_for_review`, removing stale Barnacle-owned candidate labels while preserving suppression/protected labels.

## Reproduction Assessment

Yes, from source inspection: a PR that already has `triage: blank-template` and later no longer classifies as blank will pass through `addMissingLabels` with no removals, and `ready_for_review` is not wired as an Auto response event.

## Solution Assessment

Yes, the narrow maintainable fix is to include `ready_for_review` in both the workflow and script event allowlist, then make candidate-label reconciliation remove stale Barnacle-owned candidate labels with regression coverage.

## Repair Work Prompt

Fix issue 77343: Barnacle should refresh stale PR candidate labels after a draft PR is edited and marked ready for review. In `.github/workflows/auto-response.yml`, add `ready_for_review` to the Auto response `pull_request_target` trigger. In `scripts/github/barnacle-auto-response.mjs`, include `ready_for_review` in the PR candidate-event allowlist and change candidate-label handling so Barnacle-owned candidate labels (`Object.values(candidateLabels)`) are reconciled: add newly matching labels and remove stale candidate labels that no longer match. Preserve existing privileged-author skips, `bad-barnacle` suppression, active PR limit cleanup, close-label behavior, and do not execute PR branch code. Add or update tests in `test/scripts/barnacle-auto-response.test.ts` for a PR that starts with `triage: blank-template`, then has meaningful body/content on `edited` or `ready_for_review`, and verify the stale candidate label is removed. This is workflow/script/test-only behavior; no changelog is required unless maintainers consider Barnacle behavior user-facing.
