## What This Changes

The issue reports Barnacle leaving stale candidate labels on a draft PR after the PR body is updated and the PR is marked ready for review.

## Best Possible Solution

Review and land PR #77361, or an equivalent narrow fix, so Barnacle reruns on ready_for_review and reconciles only its own candidate labels.

## Solution Assessment

Yes. The narrow maintainable solution is to add ready_for_review to the workflow and script allowlist, then reconcile Barnacle-owned candidate labels with regression coverage.
