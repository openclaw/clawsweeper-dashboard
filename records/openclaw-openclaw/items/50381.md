## Summary

## What This Changes

Adds a `--source gateway|llm|all` option to `openclaw logs`, extends `logs.tail` source metadata and hints, and formats cache-trace JSONL as CLI trace output.

## Best Possible Solution

Expose cache-trace logs through the existing redacted, config-safe log-tail contract, preserve current fallback and follow behavior, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes for the review defects by source inspection: the PR branch returns raw `readLogSlice` output, calls `loadConfig()` in the LLM path, passes `opts.rich` into trace JSON rendering under `--plain`, and maps missing source metadata to `llm`. I did not run tests because this was a read-only review.

## Solution Assessment

No. The user-facing feature is useful, but the implementation should extend the shared redacted, config-safe log-tail path instead of adding a parallel raw reader in the gateway handler.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve log-tail redaction before returning lines:** `src/gateway/server-methods/logs.ts:208`
  - body: The handler now responds with raw `readLogSlice` results. Current main routes log tails through `readConfiguredLogTail`, which redacts sensitive values before returning lines, so this can expose tokens or credentials through the RPC.
  - confidence: 0.94
- **[P2] Keep LLM log lookup out of invalid-config paths:** `src/gateway/server-methods/logs.ts:59-76`
  - body: `resolveLlmLogFile()` and `isLlmSourceEnabled()` call `loadConfig()`. `openclaw logs` is a troubleshooting path, so `--source llm` and especially `--source all` should not fail because the config file is malformed or being repaired.
  - confidence: 0.86
- **[P2] Honor --plain when rendering trace JSON:** `src/cli/logs-cli.ts:347`
  - body: When stdout is a TTY and the user passes `--plain`, `pretty` becomes false but `rich` can remain true. This trace rendering path still passes `opts.rich` into `formatPrettyJson`, so JSON blocks can contain ANSI styling despite the plain-output contract.
  - confidence: 0.9
- **[P2] Default missing source metadata to gateway:** `src/cli/logs-cli.ts:88`
  - body: `LogsTailPayload.source` is optional, but `toPublicSource(undefined)` returns `llm`. Older gateways that omit the new field would have normal gateway lines and notices mislabeled as LLM output.
  - confidence: 0.88
