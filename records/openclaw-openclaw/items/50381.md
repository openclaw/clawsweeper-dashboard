## Summary

Keep open. This is a useful CLI diagnostics feature, but the PR is not merge-ready: the live branch is conflicting, lacks real behavior proof, and source inspection confirms a sensitive log-tail redaction bypass plus config and plain-output compatibility defects.

## What This Changes

The PR adds `--source gateway|llm|all` to `openclaw logs`, extends `logs.tail` with source/hint metadata, and pretty-prints cache-trace JSONL output.

## Best Possible Solution

Rebase onto current main and implement source-aware log tailing through shared helpers that preserve redaction, invalid-config diagnostics, local fallback/follow behavior, docs, and real CLI proof.

## Reproduction Assessment

Yes for the review defects by source inspection. The PR branch returns raw `readLogSlice` output, calls `loadConfig()` in the LLM path, and passes `opts.rich` into trace JSON rendering even when `--plain` disables pretty output.

## Solution Assessment

No. Exposing existing cache-trace data through `openclaw logs` is a reasonable direction, but this patch duplicates log reading instead of extending the redacted, config-safe log-tail contract on current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve log-tail redaction before returning lines:** `src/gateway/server-methods/logs.ts:208`
  - body: The handler now responds with raw `readLogSlice` results. Current main routes log tails through `readConfiguredLogTail`, which masks CLI tokens and other sensitive values before `logs.tail` returns them, so this would expose secrets through the RPC.
  - confidence: 0.94
- **[P2] Keep LLM log lookup out of invalid-config paths:** `src/gateway/server-methods/logs.ts:59`
  - body: `resolveLlmLogFile()` calls `loadConfig()`. `openclaw logs` is allowlisted for invalid config troubleshooting, so `--source llm` and especially `--source all` should not fail because the config file is malformed.
  - confidence: 0.86
- **[P2] Honor --plain when rendering trace JSON:** `src/cli/logs-cli.ts:347`
  - body: When stdout is a TTY and the user passes `--plain`, `pretty` becomes false but `rich` remains true. Trace rendering still passes `opts.rich` into `formatPrettyJson`, so JSON blocks can contain ANSI styling despite the plain-output contract.
  - confidence: 0.9
- **[P2] Default missing source metadata to gateway:** `src/cli/logs-cli.ts:87-88`
  - body: `LogsTailPayload.source` is optional, but `toPublicSource(undefined)` currently returns `llm`. Older gateways that omit the new field would have normal gateway lines and notices mislabeled as LLM output.
  - confidence: 0.88
