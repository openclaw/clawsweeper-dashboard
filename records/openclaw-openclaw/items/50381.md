## Summary

Keep this PR open. Current main still does not provide `openclaw logs --source llm|all`, so the work is not obsolete, but the branch is not merge-ready because it broadens a read-scoped RPC to cache-trace contents, regresses config-broken troubleshooting paths, and still has smaller CLI correctness issues from prior review.

## What This Changes

The PR adds `--source gateway|llm|all` to `openclaw logs`, extends `logs.tail` with source and hint metadata, reads `diagnostics.cacheTrace` JSONL for LLM output, pretty-prints trace JSON, changes the default log tail byte window, and adds CLI/gateway tests.

## Best Possible Solution

Keep the feature direction, but land it only after the source-aware CLI/RPC path preserves current local fallback behavior, uses a config-tolerant cache-trace path resolver, defines and documents the cache-trace redaction or permission contract, updates CLI/protocol docs, and passes the targeted CLI/gateway tests plus the appropriate changed gate.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting the CLI option registration, protocol schema, handler, and docs; the main PR defects are reproducible from the PR head by following the `loadConfig()` calls, `rich` calculation, and `toPublicSource(undefined)` branch.

## Solution Assessment

No, not as currently proposed. Reusing cache trace is a maintainable direction, but the implementation should first settle the sensitive-data access contract, keep troubleshooting logs available when config is broken, honor `--plain`, and rebase onto current logs fallback behavior.

  - body: `logs.tail` currently requires only `operator.read`, but cache trace events can include prompts, system content, and messages by default.   - confidence: 0.9

  - confidence: 0.88
- **[P2] Honor plain mode for trace JSON:** `src/cli/logs-cli.ts:470`
  - body: `rich` remains true when stdout is a TTY and the user passes `--plain`, and the LLM trace renderer passes that value into JSON formatting. This still emits ANSI styling in a mode documented as no-styling output.
  - confidence: 0.86
- **[P2] Default missing sources to gateway:** `src/cli/logs-cli.ts:88`
  - body: `source` is optional for compatibility with older gateway responses, but `toPublicSource(undefined)` currently returns `llm`. Older gateway log payloads would be mislabeled in output and notices.
  - confidence: 0.92

  - body: The PR branch predates current main's implicit-loopback fallback and `fetchLogs` now calls `logs.tail` directly. When rebasing this feature, keep the fallback for gateway logs so `openclaw logs` still works when local RPC closes, times out, or requires pairing.
  - confidence: 0.78
