  - confidence: 0.9

  - body: `resolveLlmLogFile()` and `isLlmSourceEnabled()` call `loadConfig()`. An invalid config can make `--source llm` fail, and `--source all` rejects the whole fetch before showing gateway logs, which regresses a troubleshooting path.
  - confidence: 0.88

  - body: This fetch path removed current main's implicit local file-log fallback and transient `--follow` reconnect loop. `openclaw logs --source gateway` should keep working when local RPC is unavailable or a follow poll disconnects.
  - confidence: 0.88

  - body: When stdout is a rich TTY and the user passes `--plain`, `pretty` is false but `rich` remains true, so `formatPrettyJson(trace.parsed, opts.rich)` still emits ANSI escapes. Pass a non-rich flag for plain output.
  - confidence: 0.86

  - body: `source` is optional for compatibility with older gateway responses, but `toPublicSource(undefined)` returns `llm`. Default anything except an explicit `llm` to `gateway` so older gateway log lines and notices are not mislabeled.
  - confidence: 0.92
