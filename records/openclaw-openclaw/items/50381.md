## Best Possible Solution

Rebase onto current main and implement source-aware tailing through shared, redacted, config-safe log-tail helpers, with protocol/docs follow-through and redacted terminal proof for `llm` and `all`.

## Reproduction Assessment

Yes for the review defects by source inspection: the PR branch returns raw `readLogSlice` output, calls `loadConfig()` in the LLM path, and passes `opts.rich` into trace JSON rendering even when `--plain` disables pretty output.

## Solution Assessment

No. Exposing existing cache-trace data through `openclaw logs` is reasonable, but the implementation should extend the shared redacted, config-safe log-tail contract instead of duplicating log reading in the gateway handler.

Full review comments:

- **[P1] Preserve log-tail redaction before returning lines:** `src/gateway/server-methods/logs.ts:208`
  - body: The handler now responds with raw `readLogSlice` results. Current main routes log tails through `readConfiguredLogTail`, which applies sensitive-value redaction before `logs.tail` returns lines, so this would expose tokens or credentials through the RPC.
  - confidence: 0.94

  - body: `resolveLlmLogFile()` calls `loadConfig()`. `openclaw logs` is documented as an invalid-config troubleshooting command, so `--source llm` and especially `--source all` should not fail because the config file is malformed.
  - confidence: 0.86
- **[P2] Honor --plain when rendering trace JSON:** `src/cli/logs-cli.ts:347`
  - body: When stdout is a TTY and the user passes `--plain`, `pretty` becomes false but `rich` can remain true. This trace rendering path still passes `opts.rich` into `formatPrettyJson`, so JSON blocks can contain ANSI styling despite the plain-output contract.
  - confidence: 0.9
- **[P2] Default missing source metadata to gateway:** `src/cli/logs-cli.ts:88`
  - body: `LogsTailPayload.source` is optional, but `toPublicSource(undefined)` returns `llm`. Older gateways that omit the new field would have normal gateway lines and notices mislabeled as LLM output.
  - confidence: 0.88
