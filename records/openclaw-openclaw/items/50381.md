## Summary

Keep open: the feature is useful, but the PR is not mergeable as-is because the proposed log reader bypasses current log-tail redaction, has source/TTY/config correctness defects, and lacks real behavior proof from a real CLI/Gateway run.

## What This Changes

The PR adds `openclaw logs --source gateway|llm|all`, extends `logs.tail` metadata for sources and hints, and pretty-prints `diagnostics.cacheTrace` JSONL output.

## Best Possible Solution

Rebase onto current main, add source-aware log-tail helpers that preserve redaction and invalid-config diagnostics, keep `--plain` truly unstyled, document the new source option, and provide redacted terminal or recording proof of `openclaw logs --source llm` and `--source all`.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR branch returns raw `readLogSlice` results instead of the current redacted helper, and the `--plain`/missing-source behavior is directly visible in the PR head.

## Solution Assessment

No. Exposing existing cache-trace diagnostics through `openclaw logs` is a reasonable direction, but the maintainable fix should extend the existing log-tail/redaction/config contracts instead of duplicating them in the Gateway method.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve log-tail redaction before returning lines:** `src/gateway/server-methods/logs.ts:208`
  - body: The new handler responds with `readLogSlice` results directly, bypassing the current `readConfiguredLogTail` redaction path. Current main masks CLI tokens before `logs.tail` returns them, so this would expose secrets in gateway logs and trace output through the RPC.
  - confidence: 0.94
- **[P2] Keep LLM log lookup out of invalid-config paths:** `src/gateway/server-methods/logs.ts:59`
  - body: `resolveLlmLogFile()` calls `loadConfig()` before reading the trace file. `openclaw logs` is explicitly allowed to keep working with invalid config, so `--source llm` and `--source all` should not fail just because the config file is malformed.
  - confidence: 0.82
- **[P2] Respect --plain when rendering trace JSON:** `src/cli/logs-cli.ts:470`
  - body: When stdout is a TTY and the user passes `--plain`, `pretty` is disabled but `rich` remains true. Trace rendering then passes `opts.rich` into `formatPrettyJson`, so the JSON block can still contain ANSI escapes despite the no-styling contract.
  - confidence: 0.9
- **[P2] Default missing source metadata to gateway:** `src/cli/logs-cli.ts:87-88`
  - body: `LogsTailPayload.source` is optional, but `toPublicSource(undefined)` currently returns `llm`. Older gateways that omit `source` would have ordinary gateway lines and notices mislabeled as LLM output.
  - confidence: 0.88
