## What This Changes

The PR adds `openclaw logs --source gateway|llm|all`, extends `logs.tail` source metadata, reads cache-trace JSONL as LLM logs, and pretty-prints trace JSON in the CLI.

## Best Possible Solution

Keep the feature open, but require a maintainer-approved LLM trace access/redaction contract, a rebase that preserves current logs resilience, docs updates, and real behavior proof before merge.

## Reproduction Assessment

Yes, source-level. Current main lacks `--source` and an LLM `logs.tail` branch, while the PR head adds them; the blocking defects are visible in the PR source and review comments.

## Solution Assessment

No. Reusing cache trace may be a reasonable backend, but this is not the best mergeable form until raw trace access is gated or redacted and the branch is rebased over current logs fallback/retry behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate or redact cache-trace log results:** `src/gateway/server-methods/logs.ts:208`
  - body: `logs.tail` is read-scoped today, but this response now returns cache-trace lines that can include prompt text, system content, and message payloads by default. Add a stricter scope, redaction mode, or explicit access contract before serving these lines through the existing read RPC.
  - confidence: 0.92
- **[P1] Avoid full config loads while serving logs:** `src/gateway/server-methods/logs.ts:58-59`
  - body: `resolveLlmLogFile()` and `isLlmSourceEnabled()` call `loadConfig()`, so an invalid config makes `openclaw logs --source llm` and `--source all` fail through `UNAVAILABLE`. Logs are a diagnostic path during config breakage, so the LLM path should avoid throwing full config loads or degrade without hiding gateway logs.
  - confidence: 0.88
- **[P2] Rebase over current logs fallback and retry behavior:** `src/cli/logs-cli.ts:61-64`
  - body: This branch calls `logs.tail` directly and falls into the generic error path, while current main now falls back to configured file logs for implicit local RPC failures and retries transient `--follow` disconnects. Preserve those behaviors when adding multiple sources.
  - confidence: 0.86
- **[P2] Honor --plain when rendering trace JSON:** `src/cli/logs-cli.ts:347`
  - body: `buildParsedOutputLine()` renders trace JSON with `opts.rich`, so a rich TTY plus `--plain` can still color the JSON block even though plain mode promises no ANSI styling. Gate trace JSON color on the plain/pretty decision too.
  - confidence: 0.86
- **[P2] Default missing log sources to gateway:** `src/cli/logs-cli.ts:87-88`
  - body: `source` is optional for older gateway responses, but `toPublicSource(undefined)` currently returns `llm`. Older gateway payloads would be mislabeled in output and notices; the safe fallback should be `gateway`.
  - confidence: 0.92
