  - body: `logs.tail` is read-scoped, but this new LLM path returns cache-trace lines that can include prompt, system, and message payloads. Add a stricter permission, redaction mode, or explicit documented access contract before serving this source.
  - confidence: 0.93
- **[P1] Keep gateway log tails on the redacted helper:** `src/gateway/server-methods/logs.ts:198-208`
  - body: The shared `readLogSlice` result is also used for `source: "gateway"`, bypassing current main's `readConfiguredLogTail` redaction before responding. Keep the existing helper for gateway logs or apply the same redaction before returning lines.
  - confidence: 0.9
- **[P1] Avoid full config reads in logs.tail:** `src/gateway/server-methods/logs.ts:59`
  - body: `resolveLlmLogFile()` and `isLlmSourceEnabled()` call `loadConfig()`. Invalid config can make `--source llm` fail, and `--source all` can reject the whole request before showing gateway logs, which regresses a troubleshooting path.
  - confidence: 0.88
- **[P2] Preserve logs fallback and follow reconnects:** `src/cli/logs-cli.ts:633-635`
  - body: The new catch path exits on any fetch error, so `openclaw logs --source gateway` drops current main's implicit local file-log fallback and transient `--follow` reconnect loop. Preserve those behaviors for gateway logs.
  - confidence: 0.88
- **[P2] Honor plain mode for trace JSON:** `src/cli/logs-cli.ts:347`
  - body: When stdout is a rich TTY and the user passes `--plain`, `pretty` is false but `rich` remains true, so `formatPrettyJson(trace.parsed, opts.rich)` can still emit ANSI escapes. Pass a non-rich flag for plain output.
  - confidence: 0.86
- **[P2] Default missing log sources to gateway:** `src/cli/logs-cli.ts:87-89`
  - body: `source` is optional for older gateway responses, but `toPublicSource(undefined)` returns `llm`. Default anything except an explicit `llm` to `gateway` so older gateway log lines and notices are not mislabeled.
  - confidence: 0.92
- **[P2] Regenerate protocol and document source fields:** `src/gateway/protocol/schema/logs-chat.ts:9-23`
  - body: The schema adds `source` and `hint`, and the CLI adds `--source`, but generated protocol clients and CLI/logging docs are not updated. Sync the public contract so SDK/UI clients and users see the new fields and behavior.
  - confidence: 0.8
