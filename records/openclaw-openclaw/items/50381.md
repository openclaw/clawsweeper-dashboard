## Summary

Keep this PR open. The requested LLM log source is not on main, but this branch is not mergeable because it exposes cache-trace content through read-scoped logs, drops current logs resilience, and lacks real behavior proof.

## What This Changes

The branch adds `openclaw logs --source gateway|llm|all`, extends `logs.tail` source/hint metadata, tails `diagnostics.cacheTrace`, and pretty-prints LLM trace JSON with tests.

## Best Possible Solution

Define a safe LLM trace access contract, rebase on current logs fallback/retry behavior, document the protocol/CLI surface, and land with real CLI proof.

## Reproduction Assessment

Yes, source-level. Current main has no `--source` option or LLM `logs.tail` branch, while the PR head adds them and the blocking defects are visible in the diff.

## Solution Assessment

No. Reusing cache trace may be a reasonable backend, but this is not the best mergeable form until access/redaction is approved and current logs resilience is preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate or redact cache-trace log results:** `src/gateway/server-methods/logs.ts:208`
  - body: `logs.tail` is reachable with `operator.read`, but this branch returns cache-trace lines that can include prompt, system, and message payloads. Add a stricter scope, redaction mode, or documented access contract before serving the LLM source.
  - confidence: 0.92
- **[P1] Avoid full config loads while serving logs:** `src/gateway/server-methods/logs.ts:59-76`
  - body: `resolveLlmLogFile()` and `isLlmSourceEnabled()` call `loadConfig()`. An invalid config can make `--source llm` or `--source all` fail through `UNAVAILABLE`, and `all` can hide gateway logs in the diagnostic path.
  - confidence: 0.88
- **[P2] Preserve local fallback and follow retry behavior:** `src/cli/logs-cli.ts:61-64`
  - body: This fetch path removes current main's local file fallback and transient `--follow` reconnect loop. That regresses `openclaw logs` when implicit local RPC is unavailable or disconnects during follow.
  - confidence: 0.88
- **[P2] Honor plain mode for trace JSON:** `src/cli/logs-cli.ts:347`
  - body: `formatPrettyJson(trace.parsed, opts.rich)` still colorizes JSON when stdout is a rich TTY and `--plain` disables pretty output. Pass a non-rich flag for plain output so no ANSI escapes are emitted.
  - confidence: 0.86
- **[P2] Default missing log sources to gateway:** `src/cli/logs-cli.ts:87-88`
  - body: `source` is optional for older gateway responses, but `toPublicSource(undefined)` returns `llm`, so older gateway log lines and notices would be mislabeled. Default anything except explicit `llm` to `gateway`.
  - confidence: 0.92
