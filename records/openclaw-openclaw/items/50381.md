## Summary

Keep open: the feature is useful, but the current PR has blocking security and logs-resilience issues, plus no real behavior proof for the new CLI flow.

## What This Changes

The PR adds a `--source gateway|llm|all` option to `openclaw logs`, extends `logs.tail` source metadata, and renders cache-trace JSONL as pretty LLM trace output.

## Best Possible Solution

Land a rebased source-aware logs implementation only after a maintainer-approved cache-trace access/redaction contract, preservation of current logs resilience, focused tests/docs, and real behavior proof.

## Reproduction Assessment

Yes, source-level. Current main lacks `--source` and an LLM `logs.tail` branch, while the PR head adds them; the blocking defects are visible by source inspection, but no live after-fix proof was supplied.

## Solution Assessment

No. Reusing cache trace may be a reasonable backend, but this is not the best mergeable form until raw trace access is gated or redacted and the branch preserves current logs fallback/retry behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate or redact cache-trace log results:** `src/gateway/server-methods/logs.ts:208`
  - body: `logs.tail` is authorized as `operator.read`, but this response now returns cache-trace JSONL lines, and cache trace can include prompt, system, and message content by default. Add a stricter scope, explicit redaction, or a documented access contract before exposing these lines through the existing read RPC.
  - confidence: 0.92
- **[P1] Avoid full config loads while serving logs:** `src/gateway/server-methods/logs.ts:58-59`
  - body: `resolveLlmLogFile()` and `isLlmSourceEnabled()` call `loadConfig()`. If config parsing is broken, `openclaw logs --source llm` fails and `--source all` rejects before showing gateway logs, regressing a diagnostic path users need during config breakage.
  - confidence: 0.88
- **[P2] Preserve current logs fallback and follow retry:** `src/cli/logs-cli.ts:61-65`
  - body: This branch calls `logs.tail` directly from `fetchLogs()`, while current main has implicit local file-log fallback and transient `--follow` reconnect handling. Rebase the multi-source implementation so those resilience paths remain.
  - confidence: 0.86
- **[P2] Honor --plain for trace JSON colors:** `src/cli/logs-cli.ts:347`
  - body: When stdout is rich but `--plain` is passed, `rich` can still be true and this trace path colorizes the pretty JSON block. Plain output should pass a non-rich flag so it does not emit ANSI escapes.
  - confidence: 0.86
- **[P2] Default missing log sources to gateway:** `src/cli/logs-cli.ts:87-88`
  - body: The response `source` field is optional for older gateways, but `toPublicSource(undefined)` returns `llm`. Treat only explicit `llm` as LLM so older gateway payloads are not mislabeled.
  - confidence: 0.9
