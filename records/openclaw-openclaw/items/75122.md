## Summary

Keep open. Current main has useful mitigations for Baileys timing, transcript scan responsiveness, reconnect queue draining, and legacy runtime-deps cleanup, but the central reliability asks remain unresolved: liveness warnings still lack async-resource/blocking-call attribution, and direct WhatsApp auto-replies still bypass durable reconnect-safe replay.

The issue asks OpenClaw to fix event-loop blockers causing WhatsApp 408 disconnects, add blocker attribution, replay queued replies after reconnect, expose Baileys timing knobs, and clean stale plugin runtime-deps trees.
