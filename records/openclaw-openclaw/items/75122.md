## What This Changes

The issue reports WhatsApp 408 disconnects attributed to long Gateway event-loop stalls and asks for blocker-attributed diagnostics, reconnect-safe reply replay, Baileys timing knobs, and stale runtime-deps cleanup.

Keep this as an active WhatsApp/Gateway reliability investigation. The desired end state is blocker-attributed liveness diagnostics, removal or offloading of the confirmed synchronous path, reconnect-safe WhatsApp reply delivery with duplicate-send protection, and a documented runtime-deps retention model that matches packaged upgrade behavior.
