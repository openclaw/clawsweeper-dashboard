## Summary

Keep open. Current main has shipped Android notification-forwarding controls, but accepted third-party app notifications can still be forwarded into node-session system events, and the issue remains security/product-sensitive with live-repro labels.

## What This Changes

The issue reports Android notification forwarding duplicating WhatsApp messages into a node session, causing duplicate or wrong-group replies across session boundaries.

## Best Possible Solution

Define one canonical ownership rule: native-provider notifications are suppressed before node-session prompting, and node-origin replies cannot send to unrelated channel targets without source binding.

## Reproduction Assessment

No live Android plus WhatsApp run was performed. Source inspection shows accepted third-party Android notifications still enter node-session system events, but the wrong-group outbound send needs live proof.

## Solution Assessment

No. Android forwarding controls are a useful mitigation, but relying on operator package choices is not the best fix for a native-provider ownership conflict.
