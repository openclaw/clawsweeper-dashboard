## Summary

Keep this PR open. Current main still has the pre-PR TUI formatter behavior, the linked bug remains open, and the proposed diff is a focused implementation candidate with the earlier review guardrails addressed.

## What This Changes

This PR updates TUI copy-sensitive token detection and regression tests so punctuation-wrapped dotted IDs, bare dotted IDs, underscore-bound identifiers, and UUID-like tokens remain unchunked while punctuation-only and prose-like long tokens still wrap.

## Best Possible Solution

Land a rebased version of this focused TUI formatter and regression-test change, or deliberately supersede it with the broader #69344 approach if maintainers want a wider long-token policy change. The unrelated formatting-only daemon test hunk can be dropped before landing.

## Reproduction Assessment

Yes. On current main, a long backtick- or asterisk-wrapped dotted ID, or a long dot-only identifier without underscores, reaches `sanitizeRenderableText`, fails the raw-token copy-sensitive checks, and is chunked with inserted spaces every 32 characters.

## Solution Assessment

Yes. The PR keeps the fix inside the existing TUI formatter seam, classifies the punctuation-stripped candidate, narrows dot/hyphen preservation with alphanumeric and digit-bearing guardrails, and adds focused regression tests for the bug and prior review concerns.
