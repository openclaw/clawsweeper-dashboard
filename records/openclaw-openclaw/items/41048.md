## Summary

This PR should stay open: the null-media warning intent is useful for #40991, but the branch is stale against current Telegram media behavior and would regress shipped fallback metadata while adding an unthrottled auto-reply path.

## What This Changes

The PR branch adds a Telegram warning reply when resolveMedia() returns null and changes inbound body construction to omit media placeholders when allMedia is empty.

## Best Possible Solution

Adapt the Telegram fix on current main so null media-resolution failures produce an accurate, rate-limited user-visible signal while preserving placeholder/file_id fallback metadata and covering all-failed media groups.

## Reproduction Assessment

Yes from source inspection. resolveTelegramFileWithRetry() can return null for file-too-big or exhausted retry failures, processInboundMessage then builds allMedia = [], and current body-helper tests lock in the fallback metadata behavior that this PR would change.

## Solution Assessment

No. The warning direction is useful, but the proposed placeholder suppression is no longer the narrowest maintainable fix because current main intentionally preserves Telegram media metadata, and the new warning path needs throttling plus album handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve fallback media metadata:** `src/telegram/bot-message-context.body.ts:141`
  - body: Conditioning the placeholder on allMedia.length drops current-main '<media:video> [file_id:...]' metadata for captioned failed downloads. Rebase this so the agent still sees Telegram file_id context when the attachment cannot be downloaded.
  - confidence: 0.9
- **[P2] Handle all-failed media groups before dropping markers:** `src/telegram/bot-message-context.body.ts:141`
  - body: Media-group updates return before the new single-message warning branch, so an album where every getFile() returns null can be silently dropped if placeholders are suppressed. Add album-level fallback or warning handling first.
  - confidence: 0.86
- **[P2] Throttle repeated media-failure replies:** `src/telegram/bot-handlers.ts:1031-1036`
  - body: This branch sends one Telegram warning for every null media resolution in an allowed chat. Add per-chat, per-sender, or media-group cooldown/dedupe so a sender cannot spam the chat or Telegram send API.
  - confidence: 0.8
- **[P3] Use reason-neutral warning text:** `src/telegram/bot-handlers.ts:1031`
  - body: resolveTelegramFileWithRetry() returns null for both the Bot API size limit and exhausted transient retries, so the fallback warning should mention temporary download failures unless the reason is carried through explicitly.
  - confidence: 0.78
