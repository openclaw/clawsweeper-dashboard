## Summary

Keep open: the reported Telegram null-media bug is source-reproducible on current main, but this PR is not merge-ready because it drops current fallback metadata, misses media-group null failures, adds an unthrottled auto-reply path, and lacks real Telegram proof.

## What This Changes

The PR changes legacy Telegram handler/body files to send a warning when media resolution returns null and to omit generated media placeholders when no attachment resolves.

## Best Possible Solution

Adapt the fix on current `extensions/telegram` code so null getFile failures produce an accurate, rate-limited user-visible warning while preserving fallback media/file_id context, covering media groups, and adding real Telegram proof.

## Reproduction Assessment

Yes, from source inspection: getFile failures can return null, `resolveMedia()` propagates null, and the current single-message handler warns only on thrown errors. Current tests also prove the empty-media fallback metadata behavior that this PR would remove.

## Solution Assessment

No. The warning direction is useful, but suppressing placeholders is not the best current-main fix because it drops fallback file_id context, misses all-failed media groups, lacks reply throttling, and has no real Telegram proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve fallback media metadata:** `src/telegram/bot-message-context.body.ts:141`
  - body: Current main intentionally prepends `<media:video> [file_id:...]` for captioned failed downloads. Making `placeholder` empty when `allMedia` is empty drops that metadata and leaves the agent with only caption text.
  - confidence: 0.9
- **[P2] Handle all-failed media groups before dropping markers:** `src/telegram/bot-message-context.body.ts:141`
  - body: Media-group updates return before the new single-message warning branch, and `processMediaGroup` only forwards successfully resolved media. If every item resolves null, this change can drop the album path with no user-visible warning.
  - confidence: 0.86
- **[P2] Throttle repeated media-failure replies:** `src/telegram/bot-handlers.ts:1031-1036`
  - body: This branch sends a Telegram warning for every null media resolution in an allowed chat. Add per-chat, per-sender, or media-group cooldown/dedupe so repeated failing media cannot spam chats or send capacity.
  - confidence: 0.82
- **[P3] Use reason-neutral warning text:** `src/telegram/bot-handlers.ts:1031`
  - body: `resolveTelegramFileWithRetry()` returns null for both Bot API size-limit failures and exhausted transient retries, but this message mostly points at file size. Make it generic unless the null reason is carried through explicitly.
  - confidence: 0.78
