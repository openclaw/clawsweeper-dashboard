## Summary

Current main still has the resolved-model header gap for GitHub Copilot Pi compaction, and this PR is a focused implementation with adequate after-fix runtime proof and no concrete correctness finding from the diff review.

## What This Changes

The PR adds GitHub Copilot default IDE-style request headers through provider request config, threads resolved request headers into Pi resolved-model paths, and adds regression tests for dynamic and configured Copilot models.

## Best Possible Solution

Land this narrow fix through the existing provider request config seam so Pi compaction receives Copilot defaults while preserving explicit header override precedence.

## Reproduction Assessment

Yes from source and the provided logs. Current main can resolve dynamic or pass-through github-copilot models without synthesized IDE headers, and the PR body shows the matching Pi compaction 400 missing Editor-Version failure.

## Solution Assessment

Yes. Reusing resolveProviderRequestConfig() is the narrowest maintainable fix because Pi compaction consumes resolved model/auth headers and bypasses the normal Copilot stream wrapper path.
