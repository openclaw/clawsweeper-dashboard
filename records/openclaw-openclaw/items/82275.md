The PR adds GitHub Copilot IDE-style default headers to provider request resolution and threads resolved headers into Pi resolved-model paths, with regression tests for dynamic and configured Copilot models.

## Best Possible Solution

Land a narrow fix that derives Copilot default headers through the existing provider request config seam so Pi compaction receives them without changing explicit header override precedence.
