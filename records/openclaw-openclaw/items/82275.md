## Summary

Keep open: this is a valid, focused Copilot compaction fix with strong live proof, but the current head is not merge-ready because the new provider-request-config import pulls the SDK auth facade into a core request-config path and fails CI in secrets import tests.

## What This Changes

The PR adds default GitHub Copilot IDE headers to provider request resolution, threads those resolved headers through Pi resolved-model paths, and adds regression coverage for dynamic and configured Copilot models.

## Best Possible Solution

Keep the resolved-model header fix, but move Copilot header constants/building onto a dependency-light core/provider helper so provider-request-config does not import the SDK auth facade, then rerun focused agent and secrets coverage plus changed gates.

## Reproduction Assessment

Yes. Source inspection plus the supplied logs show current main can route GitHub Copilot Pi compaction through activeSession.compact() with resolved models that lack synthesized IDE headers, matching the reported missing Editor-Version failure.

## Solution Assessment

No, not as currently implemented. Reusing resolveProviderRequestConfig is the right seam, but the Copilot header helper must be dependency-light instead of importing plugin-sdk/provider-auth into provider-request-config.
