## Summary

Close as implemented on current main: the reported Telegram generated-image delivery failure matches the now-fixed shared media-loader path for in-limit images when optional image optimization is unavailable, while current main also has the generated-media queue, local media roots, and Telegram `sendPhoto` path in place.

## What This Changes

The issue reports that `image_generate` returns valid `MEDIA:` image paths but Telegram does not deliver those generated images to the chat.

## Best Possible Solution

Keep the shared media-loader fallback on main and let the next release carry it; reopen only with logs from a post-3dcff build showing a different failing stage.

## Reproduction Assessment

Yes for the fixed failure mode: mocking missing optional image optimization in the shared media loader reproduces the pre-fix Telegram MEDIA failure class, and current main has coverage proving in-limit PNGs now load instead of throwing. A failing current-main Telegram run was not established.

## Solution Assessment

Yes. Fixing the shared outbound media loader is the narrowest maintainable solution because generated-media queuing and Telegram `sendPhoto` already exist, and the patch preserves fail-closed behavior for HEIC and oversize media.
