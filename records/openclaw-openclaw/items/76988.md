## Summary

Close as implemented on current main: the post-release media queue/flush path now carries `image_generate` media into outbound replies, and Telegram loads those local media roots and sends images with `sendPhoto`.

## What This Changes

Generated `image_generate` media should be delivered to Telegram chats instead of only surfacing `MEDIA:` paths.

## Best Possible Solution

Keep the current main implementation, ship it in the next release, and reopen only with redacted current-main Telegram delivery logs if generated images still fail after updating.

## Reproduction Assessment

Yes at source/test level, not as a live current-main failure. The fix commit adds the missing generated-media queue/flush path and regression tests that exercise `image_generate` media becoming outbound reply media; current main no longer presents a clear failing source path.

## Solution Assessment

Yes. The current solution is the narrow maintainable path: keep generated tool media as structured pending reply media and let the existing Telegram media loader/sendPhoto path deliver it, rather than adding Telegram-specific special cases to `image_generate`.
