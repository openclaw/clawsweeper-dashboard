## What This Changes

Request to replace the Mission Control allowlisted agent-file panel with workspace and system file-browsing scope levels.

## Best Possible Solution

Converge the workspace-scoped browser/API with https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, and decide any system filesystem scope separately with explicit security approval.

## Reproduction Assessment

Yes, source-level: current main rejects non-allowlisted agent file names and the UI consumes only `agents.files.*`. This is feature absence rather than a broken existing contract, so no live failure repro is required to understand it.

## Solution Assessment

No, directly removing the whitelist and adding system browsing is not the best settled fix. The safer path is an additive, root-pinned, permission-scoped workspace API/UI plus a separate maintainer decision on any host filesystem scope.
