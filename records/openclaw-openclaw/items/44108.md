## Summary

Keep open. Current main and the latest release still expose only the allowlisted bootstrap-file API, while the overlapping workspace-file-management PR is open and does not settle the requested system-wide scope or security boundary.

## What This Changes

The issue requests replacing the Control UI agent file allowlist with scope levels for core MC files, all agent workspace files, and system-wide filesystem browsing.

## Best Possible Solution

Keep this issue open for maintainer product/security review, converge the workspace-scoped part with https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, and decide the system filesystem scope separately.

## Reproduction Assessment

Yes, source-level: current main still rejects non-allowlisted agent file names and has no registered `agents.workspace.*` file-management API. This is a feature request rather than a failing existing contract, so a live failure repro is not applicable.

## Solution Assessment

No, directly removing the whitelist and adding system browsing is not the best settled fix yet. The safer direction is an additive, root-pinned, permission-scoped workspace API/UI plus a separate maintainer decision on any system filesystem scope.
