## Summary

Keep open. Current main still exposes only the fixed bootstrap-file browser; workspace-recursive browsing is tracked by the open canonical workspace API issue and PR, while host-system browsing remains a separate security and product decision.

## What This Changes

Request to replace the fixed Control UI agent-file allowlist with scope levels for core files, all workspace files, and host-system browsing.

## Best Possible Solution

Keep the allowlisted `agents.files` surface, consolidate workspace-recursive browsing under the canonical `agents.workspace.*` API/UI work, and split host-system browsing into an explicitly admin-gated security design.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence absence proof: arbitrary file names still hit the allowlist, no `agents.workspace.*` namespace is on main, and the UI has no scope selector or tree browser.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing should use an additive, permission-scoped API/UI path, while host filesystem browsing should be a separate security-reviewed admin boundary rather than removing the existing bootstrap-file allowlist.
