## Summary

Keep open. Current main still exposes the Control UI agent file browser as a fixed bootstrap-file surface; the requested workspace-recursive and system browser scopes remain new capabilities, and the system scope needs explicit security/product design.

## What This Changes

Requests replacing the fixed agent file whitelist with selectable file-browser scopes for core files, all workspace files, and system files.

## Best Possible Solution

Add an additive workspace-scoped browser first while preserving workspace boundary checks, and treat full system browsing as a separate admin-gated security design.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: arbitrary names still hit the whitelist, listing is fixed, and the protocol/UI have no scope, tree, or search surface.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing is a plausible additive Gateway/UI feature, but full system browsing should be split behind explicit security and product approval rather than folded into the existing bootstrap-file methods.
