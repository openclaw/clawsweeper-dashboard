## Summary

Keep open. Current main and the latest release still expose only the Core Files allowlist; workspace-confined management is already tracked separately, but the requested System scope remains a product and security decision rather than conservative cleanup.

## What This Changes

Request to replace the Control UI Core Files allowlist with a scope selector for core files, all agent workspace files, and host-system filesystem browsing.

## Best Possible Solution

Keep the existing Core Files API stable, evaluate workspace-confined management through https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, and require separate security-reviewed product design before any host filesystem browser ships.

## Reproduction Assessment

Yes, at source level for the reported limitation: current main deterministically lists and accepts only the Core Files allowlist. This is intentional existing scope rather than a broken current contract, so it is a feature request, not a strict bug reproduction.

## Solution Assessment

No as one unified scope-selector change. The safer path is a separate workspace-confined API/UI for custom agent files and an independently approved authorization, sandbox, and audit model before host filesystem access.
