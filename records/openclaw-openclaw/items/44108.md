## Summary

Keep open. Current main still has the reported limitation: the Gateway agent-file surface is restricted to bootstrap/core workspace files plus MEMORY.md, and neither the protocol nor Control UI exposes scope levels, recursive workspace browsing, search/tree navigation, or system-file browsing. The workspace-recursive part is concrete, but the requested system scope needs maintainer product and security scoping before implementation.

## What This Changes

Request to broaden the Control UI agent file browser from fixed core workspace files to selectable scopes for core files, all agent workspace files, and optionally system files with safety controls.

## Best Possible Solution

Define an additive Gateway protocol for scoped file browsing: first support workspace-recursive list/read/write inside existing fs-safe workspace boundaries, then treat any system scope as a separately approved admin-gated design. The Control UI should add a scope selector, directory tree/search, and aligned docs/tests once that contract is clear.

## Reproduction Assessment

Yes. A current-main agents.files.get or agents.files.set call for a custom name such as elevenlabs-lexicon.pls will hit the ALLOWED_FILE_NAMES check and return unsupported file, while agents.files.list cannot surface it because it enumerates only bootstrap files and MEMORY.md.

## Solution Assessment

Unclear as proposed. Removing the whitelist for all three requested scopes in one change is not the safest path; a workspace-recursive scope is a narrow maintainable first step, while system browsing needs a separate security model and maintainer approval.
