## Summary

Keep open. Current main and the latest release still expose only the allowlisted Core Files surface; the workspace-recursive portion is covered by an open canonical issue and PR, while the host-system scope still needs explicit product and security direction.

## What This Changes

Requests replacing the Core Files allowlist with scoped browser levels for core files, all workspace files, and host system files.

## Best Possible Solution

Review the workspace-scoped API/UI work under https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, and treat host-system browsing as a separate security-approved design with explicit roots and write policy.

## Solution Assessment

Unclear as proposed. The workspace portion should be an additive, security-reviewed workspace API/UI, while full-system browsing needs a separate maintainer decision rather than removing the allowlist in place.
