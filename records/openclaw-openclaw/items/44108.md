## Summary

Keep open. Current main still exposes only the fixed bootstrap-file browser; the workspace-recursive part is already tracked by the open canonical issue and implementation PR, while the requested host-system scope remains a distinct security/product decision.

## What This Changes

Request to replace allowlisted Control UI agent file access with scoped core-file, recursive workspace, and host-system file browsing.

## Best Possible Solution

Consolidate recursive workspace browsing under https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, then split the host-system scope into an explicitly admin-gated security design before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence absence proof: arbitrary file names still hit the allowlist, no `agents.workspace.*` methods exist, and the UI has no scope selector or tree browser.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing should use the canonical additive API/UI path, while host filesystem browsing should be a separate security-reviewed admin boundary rather than extending the bootstrap `agents.files` helpers directly.
