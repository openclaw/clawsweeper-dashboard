## Summary

Keep open. Current main still exposes the agent file browser as a fixed bootstrap/MEMORY file surface, and the requested workspace-recursive plus system-file browser is a user-facing feature expansion that needs product and security design first.

## What This Changes

Request to replace the Control UI agent-file whitelist with selectable MC/workspace/system browsing scopes.

## Best Possible Solution

Add an additive Gateway protocol and UI for workspace-scoped browsing first, preserving workspace boundary and file-safety checks, and treat system browsing as a separately approved admin-gated design with tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection still gives a high-confidence current-main check: arbitrary names fail the whitelist and list returns bootstrap/MEMORY files only.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing inside the agent workspace is plausible, but full system browsing should be split behind explicit security and product decisions.
