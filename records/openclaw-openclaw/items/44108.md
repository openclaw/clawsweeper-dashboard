## Summary

Current main still has the reported limitation: the agent file surface is restricted to bootstrap and memory files, and neither the Gateway protocol nor the Control UI has scope levels, recursive workspace browsing, or system-file browsing. The issue should stay open because the workspace part is concrete, while the requested system scope needs maintainer product/security review before implementation.

## What This Changes

Request to broaden the Control UI agent file browser from fixed core workspace files to selectable scopes covering core files, all agent workspace files, and optionally system files with safety controls.

## Best Possible Solution

Define an additive Gateway protocol for scoped file browsing: first a workspace-recursive list/read/write path that stays inside fs-safe workspace boundaries, and only add a system scope if maintainers approve a separate admin-gated security model. The Control UI should then get a scope selector, directory tree/search, and tests/docs aligned with that contract.
