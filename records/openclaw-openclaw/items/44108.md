## Summary

Keep open. Current main still exposes a bootstrap-file-only `agents.files.*` surface, while workspace-wide browsing is being handled in related #61368/#62417 and the requested system-wide scope needs explicit product and security review.

## What This Changes

The issue requests replacing the managed file whitelist with file-browser scope levels for core files, all workspace files, and the host filesystem.

## Best Possible Solution

Review, repair, and either land or replace #62417 for the workspace-scoped `agents.workspace.*` API, then track any host-filesystem browser as a separate explicit security/product decision.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source and docs give high-confidence absence proof because current main only supports managed bootstrap workspace files.

## Solution Assessment

No, directly removing `ALLOWED_FILE_NAMES` and adding a system scope is not the safest maintainable path. A separate path-based workspace API with existing `fs-safe` boundaries, plus a distinct decision for host-filesystem access, is the safer direction.
