## Summary

Keep open. Current main and v2026.5.12 still expose the narrow allowlisted bootstrap-file browser, the workspace-recursive portion is already represented by an open canonical issue and PR, and the requested host-system scope still needs product and security review.

## What This Changes

Request to replace the Control UI agent file allowlist with scope levels for core files, all workspace files, and host-system browsing/editing.

## Best Possible Solution

Keep the existing allowlisted agents.files surface, consolidate workspace-recursive browsing under a security-reviewed agents.workspace.* API/UI, and split host-system browsing into a separate admin-gated filesystem access design.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence absence proof: arbitrary names still hit ALLOWED_FILE_NAMES, no agents.workspace.* methods exist on current main, and the UI has only Core Files tabs.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing should use an additive permission-scoped API/UI path, while host filesystem browsing needs a separate security-reviewed admin boundary rather than simply removing the bootstrap-file allowlist.
