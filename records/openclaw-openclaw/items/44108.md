## Summary

Keep open. Current main still restricts the Control UI agent file surface to bootstrap/MEMORY files, and the requested workspace-recursive plus system browser is a new user-facing feature with security/product design still unresolved.

## What This Changes

Request to replace the Control UI agent-file whitelist with selectable MC, all-workspace, and system browsing scopes.

## Best Possible Solution

Add an additive Gateway protocol and Control UI path for workspace-scoped browsing first, preserving workspace boundary checks, and split full-system browsing into a separately approved admin-gated design.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: arbitrary non-allowlisted names still fail and list returns only bootstrap/MEMORY files.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing inside the agent workspace is plausible, but full system browsing should be separated behind explicit security and product decisions.
