## Summary

Keep open. Current main and the latest release still expose the agent file browser as a fixed bootstrap/MEMORY file surface, while the requested workspace-recursive and system browser scopes are a user-facing feature expansion that needs product and security design.

## What This Changes

Request to replace the Control UI agent file whitelist with selectable MC/workspace/system file-browser scopes.

## Best Possible Solution

Add an additive workspace-scoped browser first, preserving workspace boundary and file-safety checks, and handle full system browsing as a separate admin-gated design with tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: arbitrary names still hit ALLOWED_FILE_NAMES and list returns only bootstrap/MEMORY files.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing inside the agent workspace is plausible as an additive Gateway/UI feature, but full system browsing should be split behind explicit security and product approval.
