## Summary

Keep open. Current main and the latest release still expose only the allowlisted agent bootstrap-file API, and the related workspace-file PR is open and only partially covers this issue because the requested system-wide scope still needs maintainer product and security direction.

## What This Changes

The issue requests replacing the agent file whitelist with scope-selectable browsing and editing for core files, all agent workspace files, and potentially the host filesystem.

## Best Possible Solution

Keep this open for maintainer product/security review, converge the workspace-scoped portion with https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, and decide the system-scope request separately before implementation.

## Reproduction Assessment

Yes, source-level: current `agents.files.*` rejects unsupported names through `ALLOWED_FILE_NAMES`, and current main has no registered `agents.workspace.*` agent file-management API. I did not run a live repro because this is a feature/security design request rather than a broken existing contract.

## Solution Assessment

No, directly removing the whitelist and adding a system browser is not the best settled fix yet. The safer shape is an additive, permission-scoped, root-pinned workspace API/UI plus a separate maintainer decision on whether any system filesystem scope belongs in core.
