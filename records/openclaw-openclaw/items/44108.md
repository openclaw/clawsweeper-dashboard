## Summary

Current main still intentionally limits the Control UI to Core Files, while the active related issue and PR cover only workspace-confined management. This issue remains necessary for the distinct host-system scope, which needs explicit product and security direction rather than duplicate implementation.

## What This Changes

Add file-browser scopes for Core Files, all files within an agent workspace, and the host filesystem.

## Best Possible Solution

Keep the existing Core Files API stable, evaluate workspace-confined management through https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, and require a separate security-reviewed product design before exposing any host filesystem browser.

## Reproduction Assessment

Yes for the reported limitation: current source deterministically lists and accepts only the Core Files allowlist. The limitation is intentional existing scope, so this is a feature request rather than a reproduced bug.

## Solution Assessment

No as a unified scope selector. A separate workspace-confined API is the narrow maintainable solution for custom agent files, while host filesystem access needs an independently approved authorization and sandbox model.
