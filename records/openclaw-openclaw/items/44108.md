## Summary

Keep open. Current main still exposes only the bootstrap/MEMORY agent-file surface, and the requested workspace-recursive plus system-file browser is a user-facing feature expansion with security/product scope to settle first.

## What This Changes

Request to replace the agent file-browser whitelist with scope levels for core files, recursive workspace files, and optionally system files.

## Best Possible Solution

Define an additive Gateway protocol for scoped file browsing: support recursive workspace list/read/write within existing fs-safe workspace boundaries first, then treat system browsing as a separately approved admin-gated design with matching UI and docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection still gives a high-confidence check: a custom name such as elevenlabs-lexicon.pls hits the ALLOWED_FILE_NAMES rejection, and list only returns bootstrap files plus MEMORY.md.

## Solution Assessment

Unclear as proposed. A workspace-recursive scope inside the existing safe workspace boundary is maintainable, but removing the whitelist while also adding system browsing needs a separate security and product decision.
