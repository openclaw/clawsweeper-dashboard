## Summary

Keep open: current main and v2026.6.10 still expose only the allowlisted `agents.files.*` bootstrap-file API, the workspace-scoped part overlaps the open workspace-management issue and PR, and the requested system filesystem scope still needs product/security approval.

## What This Changes

The issue asks to replace the Mission Control agent-files allowlist with selectable scopes for bootstrap files, all workspace files, and eventually system filesystem browsing.

## Best Possible Solution

Converge the workspace-scoped browser/API with https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, then decide any system filesystem scope separately with explicit security approval.

## Reproduction Assessment

Not applicable as a bug repro; this is feature absence, not broken existing behavior. Source inspection gives high-confidence absence proof because current main rejects non-allowlisted agent file names and the UI consumes only `agents.files.*`.

## Solution Assessment

No, directly removing the whitelist and adding system browsing is not the best settled fix. The safer path is an additive, root-pinned, permission-scoped workspace API/UI plus a separate maintainer decision on any host filesystem scope.
