## Summary

Keep open. Current main still has the reported limitation: the Gateway agent-file API is restricted to bootstrap/core workspace files plus MEMORY.md, and neither the protocol nor Control UI exposes scoped browsing, recursive workspace listing, tree/search navigation, or system-file access. The workspace-recursive part is concrete, but full system browsing needs maintainer product and security design before implementation.

## What This Changes

The issue requests replacing the fixed agent-file whitelist with selectable file-browser scopes for core files, all agent workspace files, and optionally system files.

## Best Possible Solution

Define an additive Gateway protocol for scoped file browsing: support workspace-recursive list/read/write inside existing fs-safe workspace boundaries first, then handle any system scope as a separately approved admin-gated design with aligned Control UI, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: agents.files.get or agents.files.set for a custom name such as elevenlabs-lexicon.pls hits the ALLOWED_FILE_NAMES check and returns unsupported file, while agents.files.list cannot surface it because it enumerates only bootstrap files and MEMORY.md.

## Solution Assessment

Unclear as proposed. Removing the whitelist for all requested scopes in one change is not the safest path; workspace-recursive browsing is a maintainable first step, while system browsing needs separate maintainer security approval.
