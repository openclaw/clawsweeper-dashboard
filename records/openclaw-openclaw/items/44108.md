## Summary

Keep open. Current main still exposes the Control UI agent file browser as a fixed bootstrap-file surface; the workspace-recursive portion is already tracked by https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, but this issue also asks for a system filesystem scope that still needs explicit maintainer security and product direction.

## What This Changes

The issue requests replacing the Control UI agent-file allowlist with selectable core-file, all-workspace, and system file-browser scopes.

## Best Possible Solution

Consolidate workspace-recursive browsing under the canonical workspace-file-management issue and PR, and treat full system browsing as a separate admin-gated security/product design rather than extending the bootstrap `agents.files` methods directly.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: arbitrary names still hit the allowlist, listing is fixed, and the protocol/UI have no scope, tree, or search surface.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing is already being designed in the canonical workspace API/UI thread, while full system browsing should be split into an explicit admin/security design before implementation.
