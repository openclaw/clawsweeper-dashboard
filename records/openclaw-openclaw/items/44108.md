## Summary

Keep open. Current main still exposes only the fixed bootstrap/MEMORY agent-file surface, and the requested workspace-recursive plus system-file browser is a user-facing feature expansion with security and product scope to settle first.

## What This Changes

The issue requests replacing the agent file whitelist with scoped workspace and system browsing in the Control UI file browser.

## Best Possible Solution

Define an additive Gateway protocol for scoped file browsing: support recursive workspace list/read/write inside existing fs-safe workspace boundaries first, then treat system browsing as a separately approved admin-gated design with matching UI, docs, and tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: custom names still hit the ALLOWED_FILE_NAMES rejection, and the list method only returns bootstrap files plus MEMORY.md.

## Solution Assessment

Unclear as proposed. A workspace-recursive scope within the existing safe workspace boundary is maintainable, but removing the whitelist while also adding system browsing needs a separate security and product decision.
