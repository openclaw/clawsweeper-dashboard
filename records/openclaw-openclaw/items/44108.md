## Summary

Keep open. Current main still implements the Control UI agent file editor as a bootstrap/core-file editor, with a hardcoded file-name allowlist and no protocol/UI scope selector. The request is concrete and not old enough for stale cleanup, but the full filesystem scope needs maintainer product and security review before an automated fix PR.

## What This Changes

Request to replace the Control UI agent file whitelist with selectable Core Files, All Workspace Files, and System filesystem browsing/editing scopes.

## Best Possible Solution

Split the work into an additive scoped file API: keep the current core-file behavior as the default, add a workspace-bounded recursive browser using the existing safe file helpers and explicit size/text limits, update the Control UI with a scope selector/tree/search, and handle full-system browsing only after separate maintainer security approval.

## Reproduction Assessment

Yes. For the missing capability, current source inspection is enough: a custom workspace file such as `elevenlabs-lexicon.pls` is not in `ALLOWED_FILE_NAMES`, and the protocol has no scope field, so `agents.files.get/set` cannot address it through the Control UI path.

## Solution Assessment

No proposed patch is under review. The workspace-scope direction is reasonable, but the full System scope should not be bundled into a mechanical fix without a separate access-control and safety design.
