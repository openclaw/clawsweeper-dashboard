## Summary

Keep open. Current main still exposes the agent file browser as a fixed core-file surface, while recursive workspace browsing and full system browsing are new user-facing capabilities that need product and security design before implementation.

## What This Changes

The issue requests replacing the agent file browser whitelist with workspace-recursive and system-level browsing scopes.

## Best Possible Solution

Add an additive workspace-scoped browser first, preserving workspace boundary and file-safety checks, and handle full system browsing as a separate admin-gated design with protocol, UI, tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main check: arbitrary file names still hit `ALLOWED_FILE_NAMES`, list remains fixed, and the protocol has no scope/tree/search parameters.

## Solution Assessment

Unclear as proposed. Workspace-recursive browsing inside the agent workspace is a plausible additive Gateway/UI feature, but full system browsing should be split behind explicit security and product approval.
