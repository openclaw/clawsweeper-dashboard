## Summary

Keep open. Current main still exposes only the allowlisted Core Files browser; the workspace-recursive part is already covered by an open canonical issue and PR, and the host-system scope needs explicit product and security direction.

## What This Changes

The issue requests replacing the Core Files whitelist with selectable MC, workspace-recursive, and host-system file browsing/editing scopes.

## Best Possible Solution

Review the workspace API/UI under https://github.com/openclaw/openclaw/issues/61368 and https://github.com/openclaw/openclaw/pull/62417, keep `agents.files.*` as the compatibility surface, and handle host-system browsing as a separate approved security design with explicit roots and write policy.

## Solution Assessment

Unclear as proposed. An additive workspace API/UI plus a separate security-approved host-system design is safer than removing the allowlist and adding broad filesystem scopes in one step.
