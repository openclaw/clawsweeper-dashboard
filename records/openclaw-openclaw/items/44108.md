## Summary

Keep open. Current main and v2026.6.1 still expose only the allowlisted Core Files API; the workspace-bound portion is already represented by related work, and the requested host-system scope still needs maintainer product and security direction.

## What This Changes

Request to replace the hardcoded agent file whitelist with scope levels for core files, all agent workspace files, and optional full-system browsing.

## Best Possible Solution

Keep the current core-files API until maintainers choose a bounded workspace API path and separately decide whether any operator-controlled host-system browsing should exist.

## Reproduction Assessment

Yes for the current limitation: source inspection shows `agents.files.get/set` reject non-allowlisted names and `agents.files.list` only returns bootstrap plus memory files. This is a feature request rather than a broken existing contract.

## Solution Assessment

No as a direct whitelist removal. The safer path is an additive, permissioned workspace API for agent workspace files plus a separate maintainer security decision for any host-system browsing.
