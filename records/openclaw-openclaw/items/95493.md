## Summary

Keep open: this is an active member-authored candidate fix for the linked GitHub Copilot encrypted reasoning replay bug, and current main plus the latest release still preserve the failing `encrypted_content` shape. The patch is focused at the Copilot provider payload boundary and I found no blocking correctness issue.

## What This Changes

The PR changes the GitHub Copilot connection-bound ID sanitizer so kept Responses reasoning replay items retain their IDs and summaries but drop `encrypted_content`, with focused tests updated for valid, idless, and unsafe reasoning items.

## Best Possible Solution

Copilot Responses payloads should never send token-bound stale `encrypted_content`; keep reasoning IDs/summaries for continuity and strip only the opaque encrypted field at the provider boundary.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.9` preserve valid/idless Copilot reasoning replay items with `encrypted_content`, and the linked issue reports the resulting provider rejection. I did not run a live rotated-token Copilot session.

## Solution Assessment

Yes, with a live-proof caveat: the final Copilot `onPayload` wrapper is the narrow provider-owned place to remove token-bound encrypted replay while preserving IDs/summaries and leaving core and other providers unchanged.
