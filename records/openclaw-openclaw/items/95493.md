The PR changes the GitHub Copilot Responses payload sanitizer to remove `encrypted_content` from kept reasoning replay items and updates focused sanitizer tests.

## Best Possible Solution

Use the provider-owned Copilot Responses send boundary to prevent token-bound stale encrypted replay from leaving OpenClaw, while preserving reasoning IDs and summaries unless live proof shows a broader replay policy is needed.

## Reproduction Assessment

Yes, source-level: current main can replay kept Copilot reasoning items with `encrypted_content`, and the linked issue reports provider rejection on later turns. I did not establish a live rotated-token Copilot session reproduction in this read-only review.

## Solution Assessment

Yes, with a proof caveat: the final Copilot `onPayload` wrapper is the narrow provider-owned place to remove token-bound encrypted replay while leaving core and other providers unchanged. The broader history-drop or drain-retry paths are useful alternatives but touch larger or different failure surfaces.
