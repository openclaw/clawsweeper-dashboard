## What This Changes

The PR updates the GitHub Copilot Responses replay sanitizer to delete `encrypted_content` from retained reasoning input items and adjusts focused sanitizer tests to expect IDs and summaries to remain while ciphertext is removed.

## Best Possible Solution

Land the provider-owned Copilot Responses boundary sanitizer if maintainers accept summary-only encrypted-replay recovery, with CI green and live aged-session proof added only if credentials and history are available.

## Reproduction Assessment

Yes, source-level: current main can replay `thinkingSignature` reasoning items with `encrypted_content` through both Responses paths before the Copilot wrapper sends the payload. I did not run a live Copilot token-rotation repro because that requires private credentials and aged session history.

## Solution Assessment

Yes, this is the narrow provider-owned boundary fix: Copilot-specific token-bound ciphertext is stripped in the Copilot `onPayload` wrapper without changing generic OpenAI/Anthropic replay logic. The safer alternative would be broader session repair, but that is not needed to make outgoing Copilot requests stop carrying stale ciphertext.
