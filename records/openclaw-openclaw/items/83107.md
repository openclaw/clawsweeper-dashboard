## Summary

Keep open. Current main and v2026.5.12 already document that exact user session model selections can supersede configured defaults and fail instead of falling back, but the requested diagnostic surface still appears missing: no INFO-level model resolution chain and no startup warning for stale user-selected session/picker overrides.

## What This Changes

Requests model-resolution INFO traces, stale picker/session override warnings, docs for picker-over-config behavior, and an optional strict registry-miss mode.

## Best Possible Solution

Keep exact model-ref and session override semantics, then add targeted resolver diagnostics and stale user-override surfacing once maintainers choose log verbosity and strict-mode scope.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the relevant current-main path: persisted user session model overrides can supersede defaults, user overrides are strict, and registry misses collapse to Unknown model without the requested trace.

## Solution Assessment

Unclear as written. Narrow diagnostics and stale-override warnings look appropriate, but fuzzy matching should be avoided and the optional strict-mode config needs maintainer product judgment.
