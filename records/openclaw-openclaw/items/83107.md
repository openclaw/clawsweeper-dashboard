## Summary

Keep open. Current main and v2026.5.12 already make user-selected session models exact and strict, but the requested diagnostic surface is still real work: there is no model-resolution trace, no startup warning for stale user-selected session model overrides, and the proposed strict mode needs a product decision.

## What This Changes

The issue requests INFO-level model-resolution diagnostics, stale picker/session override warnings, docs clarity, and an optional strict model-resolution mode.

## Best Possible Solution

Keep exact model refs and intentional session override semantics, but add targeted diagnostics for selected-model source, registry miss causes, and stale user session overrides; only add strict-mode config if maintainers explicitly want that policy surface.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does confirm the key current-main behavior: persisted user session model overrides can win over defaults, user overrides are strict, and registry misses collapse to `Unknown model` without the requested resolution trace.

## Solution Assessment

Unclear as written. Diagnostics and stale-override warnings look like the narrow maintainable direction, while fuzzy matching or a new strict-mode config needs explicit maintainer product judgment.
