## Summary

Keep open. Current main and v2026.5.12 already document that picker/session model selections are exact and can supersede the configured default for that session, but the requested diagnostic surface is still real work: there is no clear INFO-level model resolution chain or startup warning for stale user-selected session model overrides.

## What This Changes

Add model-resolution observability, stale picker/session-selection warnings, clearer override documentation, and an optional strict resolution mode.

## Best Possible Solution

Keep exact model refs and intentional session override semantics, but add targeted diagnostics for selected-model source, registry miss causes, and stale user session overrides, with any strict-mode config decided explicitly by maintainers.

## Reproduction Assessment

No high-confidence live reproduction was run; this read-only review did not replay the reporter's OpenRouter/Telegram session. Source inspection does show the key path: persisted session model overrides can win over defaults, user overrides are strict, and registry misses collapse to Unknown model without the requested trace.

## Solution Assessment

Unclear as written. Better than fuzzy matching is a narrow diagnostics and stale-override warning path, while the optional strict-mode config needs maintainer product judgment.
