## What This Changes

The PR changes the shared doctor empty-allowlist scanner to skip the top-level channel record when active account entries exist and updates scanner unit-test expectations.

## Best Possible Solution

Preserve root/default account diagnostics and only suppress the parent group-allowlist false positive when every active account has an effective group sender allowlist, with redacted real `openclaw doctor` proof.

## Reproduction Assessment

Yes, source-level. Current main scans the top-level channel record before accounts, so the reported false warning follows from the described config; no live doctor command was run because this review kept the checkout read-only.

## Solution Assessment

No. The fix direction is right, but gating the entire top-level scan on active accounts is too broad; the safer fix is a narrower group-warning suppression that preserves root/default-account diagnostics.
