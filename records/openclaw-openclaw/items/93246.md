## Best Possible Solution

Full review comments:

- **[P2] Preserve checks for hybrid default accounts:** `src/commands/doctor/shared/empty-allowlist-scan.ts:116-117`
  - body: This skips the root `channels.<id>` check whenever any active `accounts` entry exists. That is not true for hybrid channels such as Feishu: top-level credentials still create an active `default` account while named accounts also exist, and Feishu wires that model through `createHybridChannelConfigAdapter`. With `channels.feishu.dmPolicy="allowlist"`, empty root `allowFrom`, and an active named account, doctor would stop warning about the still-active default account blocking DMs. Please only skip the parent check when the root is not an active account scope, and add coverage for the hybrid/default-account shape.
