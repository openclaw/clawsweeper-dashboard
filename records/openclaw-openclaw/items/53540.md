## Summary

Keep open. Current main and the latest tagged release still leave the default OpenRouter-to-Anthropic large tool-parameter path exposed: OpenRouter text models resolve through the OpenAI-compatible transport, native Anthropic fine-grained tool streaming is on a separate transport, and the embedded runner still times silent stream gaps with the idle watchdog.

## What This Changes

The issue requests that embedded runner LLM calls survive or clearly handle large tool-call parameter generation instead of failing with `Network connection lost.`

## Best Possible Solution

Keep this as the canonical tracker and land a tested provider/runtime policy that either streams OpenRouter Anthropic tool arguments where supported or safely handles silent large tool-argument generation without abandoning the run.

## Reproduction Assessment

Yes. The reporter provided precise OpenRouter/Anthropic large-Write steps, and current source confirms the default OpenRouter route plus idle watchdog shape that can time out when no stream event arrives; I did not run a live provider replay.

## Solution Assessment

No complete fix is on current main. The best fix needs maintainer/provider-policy choice plus regression and real OpenRouter or controlled-proxy proof, not only a one-line timeout bump.
