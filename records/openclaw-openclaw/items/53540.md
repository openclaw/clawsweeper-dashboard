- `impact:message-loss`: The failing model call produces no final assistant output or file, so the user-visible result of the subagent turn is lost.

## Summary

Keep this open: current main and v2026.6.8 still keep the default cloud-provider LLM idle watchdog at about 120s unless an explicit provider/run timeout is configured, and OpenRouter Anthropic models still run through the OpenAI-compatible path without an OpenRouter-specific tool-argument streaming fix.

## What This Changes

The issue requests fixing embedded agent failures when slow providers generate large tool-call parameters long enough to exceed the model stream idle/request timeout.

## Best Possible Solution

Maintainers should decide the provider/runtime policy, then prove the chosen path with a live OpenRouter large tool-parameter run and keep the provider timeout mitigation documented as the fallback.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The reporter gives a concrete OpenRouter Opus large-Write repro, and source inspection shows the current 120s idle watchdog can still abort silent provider generation, but live OpenRouter proof is still missing.

## Solution Assessment

Unclear. The current provider timeoutSeconds path is a useful mitigation, but the best permanent fix needs maintainer choice between default timeout policy, provider-specific behavior, and tool-argument streaming support.
