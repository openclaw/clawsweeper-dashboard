## Summary

Keep this open: current main still has a chunk-idle LLM watchdog that can abort a silent OpenAI-compatible provider stream, and OpenRouter Anthropic routes still lack source or live proof that large tool-call argument generation now streams often enough to avoid that path.

## What This Changes

The issue reports embedded-runner failures when slow OpenRouter Anthropic tool-call parameter generation produces no streamed chunks for long enough that the model request is aborted as a network failure.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose the provider/runtime policy and prove it with a live OpenRouter large tool-parameter run plus focused regression coverage.

No high-confidence live reproduction was established. Source inspection still shows a current path where a silent OpenAI-compatible stream can hit the 120s idle watchdog, but the issue's exact old `sessions_spawn` timeout syntax is no longer the current tool contract and needs an updated live OpenRouter run.

## Solution Assessment

Unclear. The existing provider `timeoutSeconds` path is a mitigation, but the durable fix needs maintainer choice between timeout defaults, OpenRouter-specific Anthropic streaming behavior, and avoiding one huge tool payload.
