## Summary

Keep open. Current main still leaves the default OpenRouter to Anthropic large tool-parameter path exposed: OpenRouter is routed through the OpenAI-compatible transport, native Anthropic fine-grained tool streaming is on a separate transport, and the embedded runner still aborts silent stream gaps through the idle watchdog.

## What This Changes

The issue asks OpenClaw to avoid losing embedded/main agent runs when an OpenRouter Anthropic model generates large tool-call parameters slowly enough to hit request or idle timeout behavior.

## Best Possible Solution

Keep this as the canonical tracker and land a provider/runtime policy that handles silent large tool-argument generation, with a synthetic OpenAI-compatible regression and real OpenRouter Anthropic or controlled-proxy proof.

## Reproduction Assessment

Yes. The issue gives a precise OpenRouter Anthropic large-Write path, and current main still routes that model through OpenAI-compatible streaming with a chunk-idle watchdog; I did not run a live provider replay in this read-only review.

## Solution Assessment

No complete fix is on current main. Existing provider timeout configuration can mitigate opted-in setups, but the default OpenRouter Anthropic path still lacks streaming/tool-argument or timeout-policy proof, and the related PR is not merged.
