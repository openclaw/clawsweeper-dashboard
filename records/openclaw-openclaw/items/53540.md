## Summary

Keep open. Current main added a useful configured-timeout mitigation, but the default OpenRouter to Anthropic path still routes through OpenAI-compatible streaming with the 120s idle watchdog, while the broader split-timeout work remains an open PR rather than a landed fix.

## What This Changes

The issue reports that OpenClaw's embedded runner loses OpenRouter Anthropic calls when a slow model generates large tool-call parameters, especially large Write arguments.

## Best Possible Solution

Keep this as the canonical tracker until OpenRouter Anthropic large silent tool-argument generation is either streamed where supported or handled by a default runtime policy without losing the run, with a focused regression and live or controlled-proxy proof.

## Reproduction Assessment

Yes. The issue gives a concrete OpenRouter Anthropic large-Write reproducer, and current source still shows the default OpenRouter route using OpenAI-compatible streaming plus the 120s chunk-idle watchdog; I did not run a live provider replay.

## Solution Assessment

No complete fix is on current main. The existing provider timeout knob is now a useful mitigation, but the default route still needs a maintainer policy choice between larger/default timeout behavior, OpenRouter-specific tool-argument streaming, or another runtime strategy.
