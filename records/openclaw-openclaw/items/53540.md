## Summary

Keep open. Current main mitigates only explicitly configured provider timeouts; the default OpenRouter Anthropic large tool-parameter path still uses OpenAI-compatible streaming and can hit the embedded runner's chunk-idle watchdog, with https://github.com/openclaw/openclaw/pull/65898 open but not merged.

## What This Changes

The issue asks OpenClaw's embedded runner to survive or clearly handle long silent LLM tool-parameter generation, especially OpenRouter-routed Anthropic large Write calls.

## Best Possible Solution

Keep this as the canonical tracker and land one chosen provider/runtime policy with a synthetic silent-stream regression plus real OpenRouter Anthropic or controlled-proxy proof.

## Reproduction Assessment

Yes. The issue gives a precise OpenRouter Anthropic Opus large-Write setup, and current main still routes OpenRouter through OpenAI-compatible streaming plus the per-chunk idle watchdog; I did not run a live provider replay.

## Solution Assessment

No complete fix is on current main. Explicit provider timeout configuration can mitigate opted-in setups, but the default OpenRouter Anthropic path still needs a chosen timeout, streaming, or tool-payload policy with regression proof.
