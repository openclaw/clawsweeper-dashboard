The issue reports embedded runner LLM calls failing with "Network connection lost" when a slow provider emits a large tool-call parameter payload.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose the provider/runtime policy and prove the chosen behavior with a live OpenRouter large tool-parameter run plus focused regression coverage.

Unclear. Increasing provider timeouts is an available mitigation, but the durable fix needs maintainer choice between timeout defaults, OpenRouter-specific Anthropic streaming support, or changing the large tool-payload workflow.
