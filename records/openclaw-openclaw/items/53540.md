## Label Justifications

- `P1`: The issue describes a reproducible embedded runner failure that can abort real agent/subagent work and leave no output when large tool parameters are generated.
- `impact:session-state`: The failed model call can discard accumulated subagent/session work before a final result is produced.
- `impact:message-loss`: The user-visible result or requested file/message is never delivered after the LLM call fails.

## Summary

Keep this open: current main has related mitigations for explicit provider timeouts and native Anthropic eager tool input streaming, but the reported OpenRouter-to-Anthropic path still resolves through the OpenAI-compatible OpenRouter route and can still be governed by the chunk-level idle watchdog when no tool-argument chunks arrive.

## What This Changes

The issue asks OpenClaw to stop embedded runner failures when a model takes too long to generate large tool-call parameters, especially OpenRouter-routed Anthropic `Write` calls.

## Best Possible Solution

Keep this as the canonical tracker and make a maintainer provider/runtime decision: either support tool-argument streaming for OpenRouter-routed Anthropic models where possible, adopt a safe timeout policy for silent tool-argument generation, or change large file/tool payload behavior with focused regressions.

## Reproduction Assessment

Yes at source level, but not as a live current-main run. Current main still routes OpenRouter Anthropic models through `openai-completions`, lacks an OpenRouter-specific tool-argument streaming hook, and the idle watchdog still times silent iterator progress.

## Solution Assessment

No single current mitigation is the best complete solution yet. Explicit provider timeouts help configured users, but the default OpenRouter path still needs a maintainer-approved provider/runtime policy and regression coverage for silent large tool-argument generation.
