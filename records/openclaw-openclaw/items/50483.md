Overall confidence: 0.86

Full review comments:

- **[P2] Preserve optimistic assistant finals through stale refreshes:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1046-1049`
  - body: This appends the final assistant message, then immediately refreshes history. If that first history response is stale and contains the prompt or prior messages but not the new assistant final, `reconcileRunRefreshMessages` only preserves trailing user messages, so the just-shown assistant response is dropped and can flicker or disappear until another refresh catches up.
  - confidence: 0.86
- **[P2] Mark successful recovered bootstraps as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:110-113`
  - body: `load()` sets `hasLoadedInitialState` before bootstrap and the catch path resets it, but a later successful manual `refresh()` never sets the flag back to true. After recovering from a startup error, the next view `onAppear` can run another bootstrap and clear pending runs or streaming text during an active reply.
  - confidence: 0.82
