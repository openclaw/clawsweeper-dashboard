## Summary

Keep this PR open. Current main still lacks the full shared chat streaming/layout fix, and the PR contains useful source work, but the latest head is not merge-ready because it makes chat view loading one-shot after the first bootstrap and mixes an unrelated Windows installer execution-path change into a chat-focused patch.

## What This Changes

This PR updates shared OpenClawKit chat parsing, agent-event payload decoding, chat view scrolling and typing state, ChatViewModel session/stream handling, Swift regression tests, and Windows installer failure handling.

## Best Possible Solution

Revise this into a focused shared chat fix: keep the useful session-key decoding, parser stability, stream scoping, scroll/typing stability, and regression tests; preserve a later refresh path for chat appearances; and remove or split the Windows installer behavior into a dedicated installer-reviewed PR.

## Reproduction Assessment

Yes. The main blocking regression is statically reproducible from OpenClawChatView.onAppear calling load() and the PR head making load() return after the first successful bootstrap; the installer concern is directly visible in the diff.

## Solution Assessment

No. The chat direction is useful, but the current PR is not the narrowest maintainable solution until it preserves the later refresh path and separates the installer behavior for focused validation.
