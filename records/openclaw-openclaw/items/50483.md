Overall confidence: 0.87

Full review comments:

- **[P2] Mark successful bootstrap refreshes as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:254`
  - body: If the first `load()` bootstrap fails, this resets `hasLoadedInitialState` to false, but a later successful manual `refresh()` calls `bootstrap()` without setting it back to true. The next view `load()` can start a full bootstrap and clear `pendingRuns`/`streamingAssistantText`, so navigating back during a recovered in-flight reply can drop live UI state.
  - confidence: 0.88
- **[P2] Preserve optimistic final replies across refresh:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1131`
  - body: `appendFinalAssistantMessage` appends a final assistant message before `refreshHistoryAfterRun()` replaces `messages`, but the refresh reconciliation only preserves trailing user messages. If the first post-final history response is stale and lacks that assistant final, the reply can appear and then vanish until a later refresh.
  - confidence: 0.86
