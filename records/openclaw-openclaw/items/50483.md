## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve optimistic assistant finals across refresh:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1131`
  - body: `appendFinalAssistantMessage` adds the assistant reply locally, and the caller immediately refreshes history. If the first history response is stale, `reconcileRunRefreshMessages` rebuilds from `incoming` and only carries forward trailing user messages, so the just-shown assistant final can disappear until another refresh. Track these optimistic assistant finals until history confirms them, or avoid appending them before a reconciliation that can preserve them.
  - confidence: 0.86
- **[P2] Mark recovered bootstraps as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:254`
  - body: After an initial `load()` failure this line clears `hasLoadedInitialState`, but a later successful manual `refresh()` calls `bootstrap()` without setting the flag back to true. The next `onAppear` can therefore run another full bootstrap and clear pending run/tool/stream state during a new reply. Set the flag on any successful bootstrap, not only before `load()` starts one.
  - confidence: 0.82
