Full review comments:

- **[P2] Mark successful refresh recovery as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:112`
  - body: `load()` sets `hasLoadedInitialState` before bootstrap, but a later successful `refresh()` calls `bootstrap()` without ever setting the flag. After recovering from an initial bootstrap failure, the next view `onAppear` can call `load()` again and run a full bootstrap, clearing pending runs and `streamingAssistantText` during a new in-flight reply.
  - confidence: 0.88
- **[P2] Preserve optimistic final replies across stale refreshes:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1131`
  - body: This appends a final assistant message immediately, then schedules a history refresh whose reconciliation only carries forward trailing user messages. If the first post-final history response is stale and does not include that assistant final yet, the just-shown reply can disappear until another refresh, so the optimistic final needs to be preserved until canonical history catches up.
  - confidence: 0.86

  - body: `CHANGELOG.md` is release-generated for this repo, so normal PRs should keep release-note context in the PR body or commit message instead of editing the changelog directly. Please remove this entry from the branch.
