## Summary

Keep open: the branch is a plausible shared ChatUI fix, but latest head still introduces a load/onAppear refresh regression and lacks inspectable real behavior proof, so it should not merge as-is.

## What This Changes

The branch updates shared OpenClawKit chat streaming/session routing, assistant-text segment IDs, scroll and typing visibility behavior, Swift tests, and the changelog for iOS chat streaming stability.

## Best Possible Solution

Keep the session-scoped streaming/parser direction, but narrow the bootstrap-churn guard so active streams are protected while later chat appearances can still refresh, then add focused Swift regression coverage and redacted real device/simulator or terminal proof.

## Reproduction Assessment

Yes, source-reproducible for the PR-head regression: current main calls load() from ChatView.onAppear and load() bootstraps each time, while PR head returns after the first successful bootstrap. I did not run the Swift suite in this read-only pass.

## Solution Assessment

No. The session-scoped streaming fix is useful, but the permanent load latch is broader than the maintainable fix; it should guard bootstrap churn or in-flight refreshes without removing later appearance refreshes.
