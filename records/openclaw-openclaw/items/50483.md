## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P1] Preserve optimistic assistant finals during refresh:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1046-1049`
  - body: This appends the terminal assistant message and then immediately schedules `refreshHistoryAfterRun()`, but `reconcileRunRefreshMessages` only carries forward trailing user messages. If the first post-final history response is stale and omits the assistant final, the just-displayed reply is replaced out of `messages` until a later refresh, causing visible message loss/flicker. Track appended assistant finals or merge them until history confirms them.
  - confidence: 0.86
- **[P2] Mark successful refresh recovery as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:254-255`
  - body: After a failed initial load, this reset allows retry, but `refresh()` calls `bootstrap()` directly and a successful bootstrap never sets `hasLoadedInitialState` back to true. The next `onAppear` can call `load()`, run another bootstrap, and clear `pendingRuns`/`streamingAssistantText` during an active reply after recovery. Set the flag on any successful bootstrap path.
  - confidence: 0.82
