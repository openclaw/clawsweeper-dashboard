## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve optimistic finals across refresh:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1046`
  - body: This appends the final assistant reply immediately before `refreshHistoryAfterRun()` replaces `messages` from the history response. If that first post-final history response is stale, `reconcileRunRefreshMessages` only carries forward trailing user messages, so the assistant final can appear and then disappear; track the optimistic assistant final until history confirms it, or avoid appending before reconciliation.
  - confidence: 0.88
- **[P2] Mark successful recovered bootstraps as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:254`
  - body: After an initial `load()` bootstrap fails, `refresh()` can later succeed while `hasLoadedInitialState` remains false because only the failure path changes the flag. A later `onAppear` `load()` will bootstrap again and clear pending runs/tool calls/streaming text, which can drop an in-flight reply after recovery; set the flag on any successful bootstrap as well.
  - confidence: 0.86
