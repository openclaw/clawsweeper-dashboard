## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve appended finals through stale refreshes:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1046`
  - body: This appends the assistant final and immediately schedules `refreshHistoryAfterRun()`, but `reconcileRunRefreshMessages` only carries forward trailing user messages when incoming history is stale. If the first post-final history response does not yet include the assistant reply, the just-shown final can disappear until a later refresh, so track/merge optimistic assistant finals until history confirms them.
  - confidence: 0.89
- **[P2] Mark successful refresh recovery as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:112`
  - body: After an initial `load()` bootstrap fails, a later successful manual `refresh()` calls `bootstrap()` without setting `hasLoadedInitialState` back to true. The next `onAppear` can therefore run another bootstrap and clear pending runs, tool calls, and `streamingAssistantText` during a new reply; set the loaded flag after any successful bootstrap.
  - confidence: 0.86
