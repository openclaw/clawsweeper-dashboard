## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Mark successful refresh recovery as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:254`
  - body: If the first `load()` bootstrap fails, `hasLoadedInitialState` is reset to false, but a later successful `refresh()` calls `bootstrap()` without ever setting it true. The next view `load()` can start another bootstrap and clear `pendingRuns`/`streamingAssistantText`, so navigating back during a recovered in-flight reply can drop live UI state.
  - confidence: 0.88
- **[P2] Preserve optimistic assistant finals through refresh:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1131`
  - body: `appendFinalAssistantMessage` appends a final assistant message before `refreshHistoryAfterRun()` replaces `messages`, but the branch's reconciliation only preserves local user messages. If the first post-final history response is stale and lacks the assistant final, the reply can appear and then vanish until another refresh; preserve optimistic assistant finals until history confirms them.
  - confidence: 0.86
