## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Mark successful recovery bootstraps as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:112-113`
  - body: `refresh()` can recover after an initial `load()` bootstrap failure, but that successful manual bootstrap never flips `hasLoadedInitialState` back to true. The next view `onAppear` will call `load()` again and run a full bootstrap, clearing pending runs, tool calls, and `streamingAssistantText` while a reply may be in flight. Set the loaded flag only after any successful bootstrap, or otherwise mark successful refresh recovery as loaded.
  - confidence: 0.87
- **[P2] Preserve optimistic assistant finals through refresh:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1131-1136`
  - body: `appendFinalAssistantMessage` appends an assistant final immediately, then `refreshHistoryAfterRun()` rebuilds `messages` through `reconcileRunRefreshMessages`, which only preserves trailing user messages. If the first non-empty history response is stale and lacks that assistant final, the just-shown assistant reply can disappear until another refresh. Track optimistic assistant final IDs or merge them until history confirms them.
  - confidence: 0.86
