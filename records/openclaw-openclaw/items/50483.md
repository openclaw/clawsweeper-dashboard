## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Mark recovered refreshes as loaded:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:254`
  - body: If the first `load()` bootstrap fails, `hasLoadedInitialState` is reset to false, but a later successful `refresh()` runs `bootstrap()` without ever setting it true. The next view `load()` can start another bootstrap and clear `pendingRuns`/`streamingAssistantText`, so navigating back during a recovered in-flight reply can drop live UI state.
  - confidence: 0.88
- **[P2] Preserve appended assistant finals through refresh:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1131`
  - body: The helper appends a final assistant message before `refreshHistoryAfterRun()` replaces `messages` through `reconcileRunRefreshMessages`, which only carries trailing user messages from `previous`. If the first post-final history response is stale and lacks the assistant final, the reply can appear and then vanish until another refresh; track or preserve optimistic assistant finals until history confirms them.
  - confidence: 0.86
