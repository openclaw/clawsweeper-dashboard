## Summary

Close as ClawHub/out-of-core: OMLX/VMLX-style local provider support is useful, but current main already exposes provider-plugin auth, catalog, live discovery, dynamic-model, and capability seams, while the remaining generic modality/onboarding friction is tracked separately.

Request adding first-class support for more local model providers, especially OMLX and VMLX, with OpenAI-compatible autodiscovery and API-key onboarding.

## Best Possible Solution

Keep OpenClaw core focused on generic provider/plugin seams, publish OMLX/VMLX support as ClawHub provider plugins, and handle broader modality/provider-management UX in the existing canonical issues.
