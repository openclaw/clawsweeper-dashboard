## What This Changes

The issue reports a regression where direct OpenAI API-key requests for `openai/gpt-5.4-mini` and `openai/gpt-5.4-nano` time out after upgrading from 2026.4.23 to 2026.4.29, apparently after routing through `openai-responses` instead of Chat Completions.

## Best Possible Solution

Keep the issue open and live-test direct OpenAI API-key runs for both models on current main, then adjust the OpenAI provider transport policy, docs, and regression coverage to match the verified API behavior.

## Reproduction Assessment

## Solution Assessment
