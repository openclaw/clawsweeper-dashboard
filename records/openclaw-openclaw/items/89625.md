## Summary

Keep open: current main and the latest release still lack Slack `ignoreOtherMentions`, while open Slack PRs already own the implementation path and need maintainer review before this issue closes.

## What This Changes

Add a Slack channel-level `ignoreOtherMentions` option matching Discord so ambient Slack rooms can ignore messages that mention someone else without requiring every message to mention the bot.

## Best Possible Solution

Land one focused Slack PR that adds the opt-in config/schema/resolver/runtime/docs/tests and explicitly settles user-group and implicit-thread semantics, then close this issue from the merged PR.

## Reproduction Assessment

Yes, source-level. Current main and `v2026.6.6` have no Slack `ignoreOtherMentions` schema/type/resolver/docs/runtime path, while Discord has the analogous behavior; no live Slack workspace was exercised in this read-only review.

## Solution Assessment

Unclear. An opt-in Slack config option is plausible, but because it changes message suppression and the open PRs differ on semantics, maintainers should choose or review one implementation rather than queue another fix.
