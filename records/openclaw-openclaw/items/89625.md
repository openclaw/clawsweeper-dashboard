## Summary

Keep open: current main still lacks Slack `ignoreOtherMentions` in the Slack config, resolver, runtime gating, and docs, while Discord has the analogous option and an open unmerged PR tracks the same Slack feature. Because this is an opt-in Slack message-delivery config surface, it should be handled through that PR or a narrowed replacement rather than closed now.

## What This Changes

Request to add a Slack `ignoreOtherMentions` channel config option so Slack can skip messages that mention other users or user groups without mentioning the bot.

## Best Possible Solution

Land a focused Slack implementation that keeps the default unchanged, adds the config/schema/resolver/docs/tests together, and explicitly defines user, user-group, and implicit-thread semantics before suppressing messages.

## Reproduction Assessment

Yes, source-level: current main exposes Slack mention metadata and `requireMention` gating, but no Slack `ignoreOtherMentions` config, resolver field, docs entry, or drop branch. I did not run a live Slack workspace because this review is read-only.

## Solution Assessment

Unclear: an opt-in Slack config option is a plausible and likely maintainable shape, but the exact Slack semantics for user-group mentions and implicit bot-thread participation need maintainer review before landing.
