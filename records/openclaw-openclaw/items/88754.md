## What This Changes

The PR normalizes U+300C-U+300F fullwidth quote characters to ASCII angle brackets before the shared reasoning-tag stripper runs, and adds fullwidth delimiter tests.

## Best Possible Solution

Support fullwidth delimiters only around recognized reasoning/final tag candidates, preserve ordinary visible CJK punctuation, and keep the already-landed stateful WebChat parser as the linked issue’s fix.

## Solution Assessment

No for the current PR shape. The shared helper is a plausible boundary, but the implementation should normalize only recognized tag candidates so visible CJK punctuation remains unchanged.

- **[P1] Preserve visible CJK quotes outside stripped tags:** `src/shared/text/reasoning-tags.ts:47-51`

  - confidence: 0.94
