## Summary

Keep open. The sanitizer idea is useful and still not implemented on current main, but this patch is not merge-ready because it globally rewrites visible CJK quote punctuation whenever a real reasoning tag is present, and contributor after-fix proof is still missing.

## What This Changes

The branch normalizes U+300C-U+300F fullwidth quote characters to ASCII angle brackets in the shared reasoning-tag stripper and adds colocated fullwidth delimiter tests.

## Best Possible Solution

Support fullwidth delimiters only while parsing recognized reasoning/final tag candidates, preserve ordinary visible CJK punctuation, and require after-fix real behavior proof before merge.

## Reproduction Assessment

Yes, at source level for the PR-introduced regression: the branch globally rewrites fullwidth quote characters before returning visible slices from the shared sanitizer. I did not run tests because this was a read-only checkout review.

## Solution Assessment

No for the current PR shape. The shared helper is a plausible boundary, but delimiter normalization should be constrained to recognized tag candidates and the fixtures should use supported tag names.
