## What This Changes

The PR changes `stripReasoningTagsFromText` to replace U+300C-U+300F with ASCII angle brackets before tag matching and adds two fullwidth-delimiter test cases.

## Best Possible Solution

Support fullwidth delimiters only around recognized reasoning/final tag candidates while preserving ordinary visible CJK punctuation, and treat https://github.com/openclaw/openclaw/pull/87671 as the canonical WebChat streaming leak fix.

## Reproduction Assessment

Yes. Source inspection shows current main misses fullwidth-delimited tag markers, and the PR-introduced regression is also source-reproducible because `cleaned` is built from globally normalized text.

## Solution Assessment

No. The shared sanitizer is a reasonable boundary, but this implementation should normalize only recognized tag candidates and keep ordinary visible text slices in their original punctuation.
