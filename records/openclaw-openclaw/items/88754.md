## What This Changes

The PR updates the shared reasoning-tag stripper to normalize CJK/fullwidth quote delimiters before stripping reasoning tags and adds focused tests for those delimiter variants.

## Best Possible Solution

Implement fullwidth delimiter support only around recognized reasoning/final tag candidates, preserve ordinary visible CJK punctuation, and keep the already-landed stateful WebChat parser as the linked issue’s fix.

## Reproduction Assessment

Yes for the PR-introduced failures by source inspection: the diff globally normalizes CJK quotes before returning visible slices, and the added /log fixture does not match the supported reasoning close-tag regex.

## Solution Assessment

No; the fix is too broad because it changes unrelated visible punctuation. The better solution is tag-candidate parsing or slice-local normalization that keeps non-tag text unchanged.

- **[P1] Preserve visible CJK quotes outside stripped tags:** `src/shared/text/reasoning-tags.ts:47-51`
  - body: The new `normalizedText` replaces every U+300C-U+300F character in the whole message before `cleaned` is returned. Normalize only recognized tag delimiters, or keep original slices for visible text.
  - confidence: 0.94

  - body: This fixture normalizes `『/log』` to `</log>`, but the stripper only recognizes think/thinking/thought/antthinking close tags. As written, the case describes behavior the regex does not implement; use a supported reasoning close tag or change the expected output.
