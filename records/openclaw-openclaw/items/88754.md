## What This Changes

The PR normalizes U+300C-U+300F quote characters to ASCII angle brackets in the shared reasoning tag stripper and adds two fullwidth-delimiter fixtures.

## Best Possible Solution

Support fullwidth delimiters only around recognized reasoning/final tag candidates while preserving ordinary visible CJK punctuation, and keep https://github.com/openclaw/openclaw/pull/87671 as the WebChat streaming fix.

## Reproduction Assessment

Yes. The PR-introduced failure is source-reproducible because `cleaned` is built from globally normalized text, and the new fixtures are source-reproducible as unsupported close-tag cases.

## Solution Assessment

No. The shared sanitizer is a reasonable boundary, but the implementation should normalize only recognized tag candidates and keep visible text slices in the original punctuation.
