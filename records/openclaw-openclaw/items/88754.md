## Summary

Keep open: the PR targets a real sanitizer gap, but the current patch is not merge-ready because it normalizes CJK quote punctuation across the whole visible message and the new fixtures do not describe valid closing reasoning tags. It also lacks real behavior proof and should not close the broader WebChat streaming leak without proving that path.

## What This Changes

The PR changes the shared reasoning-tag stripper to normalize U+300C-U+300F punctuation before stripping tags and adds focused unit cases for fullwidth delimiter variants.

## Best Possible Solution

Keep the fix narrowly in the shared text parser, but normalize only recognized reasoning/final tag delimiters, preserve ordinary visible CJK punctuation, correct the fixtures, and leave the broader WebChat streaming issue open unless that path is separately proven fixed.

## Reproduction Assessment

Yes at source level: current main's reasoning-tag regexes only match ASCII angle brackets, so fullwidth-delimited tag text would bypass stripReasoningTagsFromText. I did not establish a live WebChat/Gemini stream reproduction in this read-only review.

## Solution Assessment

No: the current approach is too broad because it rewrites all matching CJK quote characters before parsing and does not prove the linked WebChat streaming path. The safer fix is candidate-level delimiter handling that preserves visible punctuation, plus focused sanitizer and real behavior proof.
