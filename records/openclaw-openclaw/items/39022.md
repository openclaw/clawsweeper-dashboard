## Summary

Keep open: current main still treats `/model` as a persistent session selection and has no parser, session-state, notification, or config contract for turn-limited auto-revert. The related one-shot model override issue and image/PDF auto-revert PR overlap on cost control but do not cover multi-turn budgets or per-agent limit policy.

## What This Changes

The issue requests `/model <name> --turns N` support that temporarily switches models for a fixed number of turns, warns near expiry, and reverts to the session default.

## Best Possible Solution

Define the model-governance contract first, then implement parser support, durable countdown state, warning/revert notifications, docs, and regression tests around the existing session model override path.

## Reproduction Assessment

Yes for the missing feature by source inspection. `/model gpt-5 --turns 10` has no parsed turn-budget field, and current session/config types have nowhere to store, warn, decrement, or revert a model override.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, and public config shape before a narrow code change is safe.
