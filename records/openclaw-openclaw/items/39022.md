## Summary

Keep open: current main and v2026.6.10 still have persistent user `/model` and `sessions.patch` model overrides, but no `/model --turns`, turn lease counter, warning/revert state, or per-model turn-limit config. Related open work covers one-shot overrides and image-analysis auto-revert only partially, so this remains a product/session-state design item.

## What This Changes

The issue requests `/model <name> --turns N` plus per-model config that temporarily switches models for a bounded number of turns and then auto-reverts to the session default.

## Best Possible Solution

Define one opt-in session-owned model lease contract across `/model`, Gateway session APIs, and model-selection state, with explicit turn counting, warning/revert semantics, docs, and tests for auth/profile pins, queued work, cron, resets, and live switches.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main and v2026.6.10 have no `/model --turns` parser path, lease counter, warning state, or per-model turn-limit config to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested turn lease is plausible, but the maintainable path should be one session-owned lease mechanism rather than prompt enforcement, ad hoc config, or a second model-switch path.
