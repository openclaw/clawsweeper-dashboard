## Summary

Keep open: current main still supports sticky per-session `/model` selection plus narrower one-turn/fallback discussions, but it does not implement `/model --turns N`, countdown warnings, per-agent/per-model turn-limit config, or automatic default reversion.

## What This Changes

Add `/model <name> --turns N` with warning notifications, automatic reversion to the session default, and per-agent/per-model turn-limit configuration.

## Best Possible Solution

Define a canonical model-lease contract first; if accepted, implement additive parser, config/schema, durable session lease state, warning/revert notifications, docs, and focused tests around override expiration.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a new model-governance feature request. Source inspection on current main shows no parser, directive, config, session-state, or docs contract for `/model --turns`.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable path is an additive model-lease design that composes with existing session model pins, strict user-selected fallbacks, cron primaries, one-turn proposals, and live-switch behavior.
