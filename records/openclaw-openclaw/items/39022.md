## Summary

Keep open: current main and the latest release still support persistent `/model` session selections and auto-fallback recovery, but not `/model --turns`, turn-lease warning/revert state, or per-model turn-limit config. This remains a compatibility-sensitive product/session-routing decision, not conservative cleanup.

## What This Changes

The issue requests `/model <name> --turns N` plus per-model config so temporary premium model selections warn near expiry and auto-revert to the session default.

## Best Possible Solution

Maintain one explicit, default-preserving temporary model-routing contract before implementation, then add focused parser/config/session-state/docs coverage for turn leases if maintainers choose this path.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested `/model --turns` and per-model turn-limit config.

## Solution Assessment

Unclear until maintainers choose the public contract. A turn lease may be useful, but it should be reconciled with the open one-shot override and fallback-persistence designs rather than added as another competing model-routing path.
