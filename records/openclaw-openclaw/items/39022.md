## Summary

Keep open: current `main` still supports sticky session model selection, but not `/model --turns`, turn-budget warning state, automatic revert, or the proposed public `models.limits` contract. This is a useful model-governance feature, but it needs maintainer product/API decisions before implementation.

## What This Changes

Add `/model <name> --turns N` plus per-agent configuration so temporary premium-model selections warn near expiry and revert to the default model after N turns.

## Best Possible Solution

Define the model-governance contract first, then add parser support, durable countdown state, warning/revert notifications, docs, and focused regression coverage around session model overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new model-governance behavior. Source inspection confirms current `/model` parsing, session state, config schema, and docs have nowhere to store, decrement, warn, or revert a turn budget.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, default-revert semantics, and public config shape.
