## Summary

Keep open: current main and v2026.6.10 still support persistent user model selections and fallback recovery, but not `/model --turns`, turn-lease warning/revert state, or per-model turn-limit config. This remains a maintainer product/session-state decision rather than conservative cleanup.

## What This Changes

The issue requests `/model <name> --turns N` plus per-agent/per-model limits that warn before expiry and automatically revert to the default model after the turn budget.

## Best Possible Solution

Define one opt-in session-owned model lease contract on the existing model-selection path, then implement parser, config/schema, session persistence, Gateway/API behavior, docs, and focused upgrade tests around that single contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection is high-confidence that current main lacks `/model --turns`, lease counters, warning state, and per-model turn-limit config.

## Solution Assessment

Unclear until maintainers choose the product contract. The proposed turn lease is plausible, but the maintainable path should be one session-owned model-routing mechanism rather than prompt-based enforcement or a second ad hoc model-switch path.
