## Summary

Keep this open: current main still only supports persistent `/model` session overrides, while the requested turn-limited temporary override would be a new core/config feature with unresolved turn-count semantics.

## What This Changes

Add `/model <name> --turns N` plus per-agent config limits so premium model overrides warn near expiry and automatically revert after N turns.

## Best Possible Solution

Add this as a core model-session feature with parser support, persisted temporary override metadata, completed-turn accounting, warning/revert delivery, config schema/docs, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection is still high confidence: the `/model` parser, command metadata, session state, and config schema have no turn-budget field or expiry path on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but the narrowest maintainable implementation needs maintainer choices for turn counting, warning timing, and the public config contract.
