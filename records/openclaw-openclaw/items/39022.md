## Summary

Keep open: current main and v2026.6.10 still only support persistent `/model` and `sessions.patch` model overrides, while the requested N-turn lease, warning, auto-revert, and per-model config contract remain unimplemented and require maintainer product/API review.

## What This Changes

The issue requests `/model <name> --turns N`, warning before expiry, automatic return to the session default model, and per-agent model turn-limit configuration.

## Best Possible Solution

Define one additive, session-owned model lease contract across `/model`, Gateway session APIs, and model-selection state, with explicit turn counting, warning/revert semantics, docs, and tests for queues, cron, resets, fallbacks, auth profiles, and live switches.

## Reproduction Assessment

Not applicable as a feature request. Source and release inspection show current main and v2026.6.10 have persistent `/model` and `sessions.patch` overrides, but no `/model --turns`, lease counter, warning state, or per-model turn-limit config.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested turn lease is plausible, but the maintainable path should be one session-owned lease mechanism rather than prompt enforcement, ad hoc config, or a second model-switch path.
