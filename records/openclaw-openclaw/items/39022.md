## Summary

Current main and v2026.6.5 still use sticky session-scoped model overrides; the requested turn-counted warning and auto-revert behavior remains absent, distinct from the narrower image-analysis issue, and requires a maintainer product decision.

## What This Changes

Add `/model <name> --turns N` and optional per-agent limits so temporary model choices warn before expiry and revert to the session default after a bounded number of turns.

## Best Possible Solution

Define a closed temporary model-override state at the canonical session/model-routing boundary, with explicit counting, warning, expiry, persistence, fallback, and revert-target semantics, then add only the smallest command or configuration surface that the approved governance policy requires.

## Reproduction Assessment

Not applicable: current main and v2026.6.5 consistently implement sticky session model overrides, while this issue requests a new temporary-selection mode rather than reporting broken existing behavior.

## Solution Assessment

No, not as currently specified: a native temporary override is plausible and belongs near the existing session model-selection path, but the proposed flag and per-model config lack the lifecycle semantics needed for one maintainable source of truth.
