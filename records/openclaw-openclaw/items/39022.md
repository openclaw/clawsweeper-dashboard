## Summary

Current main still lacks turn-counted temporary `/model` selections, warning notifications, and automatic reversion; this remains a concrete feature request, but it needs maintainer policy direction before code should be queued.

## What This Changes

Add `/model <name> --turns N` and per-agent model-limit config so temporary premium model selections warn near expiry and revert to the session default after the configured number of turns.

## Best Possible Solution

Define one model-expiry contract first, then implement it as explicit core session/model policy with schema/docs and regression coverage for `/model`, live switching, reset behavior, queued work, and provider/auth cleanup.
