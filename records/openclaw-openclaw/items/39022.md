## Summary

Keep open: current main has persistent `/model` session overrides and auto-fallback recovery state, but no turn-counted temporary model lease or per-model max-turn policy; adding this needs maintainer product/API judgment before implementation.

## What This Changes

Request support for `/model <name> --turns N` and per-agent model turn limits that warn before automatically reverting to the default model.

## Best Possible Solution

Define one explicit temporary model-selection policy in core session/model routing, with turn-count state, warning/revert behavior, complete model/auth/fallback cleanup, docs, and focused regression coverage before exposing `/model --turns` or config limits.

Not applicable. This is a feature request for new `/model --turns` and config-limit behavior, while current source and docs define persistent user model selections instead of turn-expiring ones.
