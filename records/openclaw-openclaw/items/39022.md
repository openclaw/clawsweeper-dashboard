- `impact:session-state`: The requested lease counter, warning state, and restoration target would alter persisted session model-selection state.

## Summary

Keep open. Current main and v2026.6.6 intentionally support sticky user-selected session model overrides but do not implement a turn-counted lease; this is a distinct cost-governance feature requiring maintainer decisions about model lifetime, turn accounting, persistence, and configuration.

## What This Changes

Add a `/model <name> --turns N` override with advance warning, automatic restoration, and optional per-agent model limits.

## Best Possible Solution

Define one canonical session-owned temporary model lease with explicit counting, warning, cancellation, concurrency, and restoration semantics, while preserving ordinary sticky `/model` selection unless maintainers deliberately approve enforced operator limits.

## Reproduction Assessment

Not applicable. This requests a new model-override lifetime policy, and source plus release inspection shows current sticky user selection behaves according to the existing contract.

## Solution Assessment

No, the exact command flag and wildcard configuration are not yet the best proven solution. A canonical opt-in lease contract should be designed first, with any enforced cost policy treated as a separate explicit product decision.
