## Summary

Keep open: current main still implements `/model` as a sticky session selection and has no `--turns` parser metadata, countdown state, warning/revert path, or `models.limits` config contract. This remains a feature request that needs model-governance product/API decisions before implementation.

## What This Changes

Add `/model <name> --turns N` plus per-agent model limit config so temporary model switches warn near expiry and auto-revert to the default model.

## Best Possible Solution

Define the model-governance contract first, then implement parser support, durable countdown state, warning/revert notifications, docs, and focused regression coverage around the existing session model override path.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new model-governance behavior. Source inspection does confirm the missing capability: current `/model` parsing and session/config types have nowhere to store, decrement, warn, or revert a turn budget.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, and public config shape before a narrow code change is safe.
