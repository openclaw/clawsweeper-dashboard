## Summary

Keep this issue open. Current main still supports persistent session model overrides, but it has no parser output, session state, config contract, warning path, or post-turn decrement/revert logic for turn-limited model switches.

## What This Changes

The issue requests `/model <name> --turns N` with near-expiry warnings, automatic reversion to the default model, and configurable per-agent/per-model turn limits.

## Best Possible Solution

Add a core model-session feature, not a plugin workaround: parse and validate `/model <provider/model> --turns N`, persist temporary override metadata on the target session, decrement after the chosen completed-turn boundary, warn near expiry, clear the override through the existing helper, notify the user, and document/test the config contract.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: `/model gpt-5 --turns 10` is parsed as a normal model directive with no turn-budget field, and there is no session/config state where such a budget could be stored or decremented.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer design for the public config shape and exact turn-counting semantics before a safe automated fix PR should be attempted.
