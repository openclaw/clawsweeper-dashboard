## Summary

Keep this open: current main still treats `/model` as a persistent session model selection, and the requested turn-limited auto-revert would add new command parsing, session countdown state, notification behavior, and public config policy. The related one-shot override issue and image-analysis auto-revert PR overlap on cost control, but neither covers multi-turn model budgets or per-agent model-limit policy.

## What This Changes

Add `/model <name> --turns N` with warning and automatic reversion to the session default model, plus per-agent model limit configuration.

## Best Possible Solution

Define the model-governance contract first, then implement parser support, durable countdown state, warning/revert notifications, docs, and regression tests around the existing session model override path.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence evidence that the requested `/model ... --turns` and `models.limits` behavior is not present on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, and public config shape before a narrow code change is safe.
