## Summary

Current main still does not implement per-model turn budgets for `/model`. The model directive/parser, native command metadata, session override state, persistence helper, and agent defaults schema all support persistent model selection, profile/runtime overrides, and fallback-source tracking, but not a turn countdown, warning threshold, expiry notification, or the requested `models.limits`/`maxTurns` chat-model config contract.

## What This Changes

The issue requests `/model <name> --turns N` with warning near expiry and automatic reversion to the session default model after the specified number of turns.

## Best Possible Solution

Keep the issue open and design a first-class temporary model override contract in the core model/session flow: parse `/model <provider/model> --turns N`, validate any documented per-agent/per-model limit config, persist countdown metadata on the session, decrement it after the chosen completed-turn boundary, send warning/revert notifications, and clear the override through the existing model override helper when it expires.
