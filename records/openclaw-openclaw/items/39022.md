## Summary

Keep open: current main and the latest release still expose sticky `/model` session overrides only; the requested turn-counted temporary model selection, warning, and auto-revert behavior is missing and overlaps adjacent model/cost-governance work that needs maintainer policy direction.

## Best Possible Solution

Define one model-expiry contract first, then implement it as core session model policy with explicit state, schema/docs, notifications, and regression coverage across `/model`, live switching, reset, auth, and queued work.
