## Summary

Keep this open: current main still only supports persistent session model overrides, and turn-limited auto-revert would require new parser, session-state, notification, and config-contract behavior.

## What This Changes

The issue requests `/model <name> --turns N` plus per-agent model turn limits that warn before expiry and revert to the default model after the budget is used.

## Best Possible Solution

Define the turn-counting and config contract first, then implement parser support, session countdown state, warning/revert notifications, docs, and regression tests around the existing model override path.

## Reproduction Assessment

Yes for the missing feature by source inspection: `/model gpt-5 --turns 10` has no parsed turn-budget field, and current session/config types have nowhere to store, warn, decrement, or revert it. This is a feature gap rather than a broken existing contract.

## Solution Assessment

Unclear. The direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, and public config shape before a narrow code change is safe.
