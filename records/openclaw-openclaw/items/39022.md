## Summary

Keep open: current main still only supports sticky session model selection, while the requested turn budget, warning, auto-revert, and public `models.limits` contract remain absent. This is a useful model-governance feature, but it needs maintainer product/API decisions before implementation.

## What This Changes

Add `/model <name> --turns N` with warning and automatic reversion to the session default model after the turn budget expires.

## Best Possible Solution

Define the model-governance contract first, then add parser support, durable countdown state, warning/revert notifications, docs, and focused regression coverage around the existing session model override path.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new model-governance behavior. Source inspection does confirm the missing capability: current `/model` parsing, session state, config schema, and docs have nowhere to store, decrement, warn, or revert a turn budget.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, default-revert semantics, and public config shape before a narrow code change is safe.
