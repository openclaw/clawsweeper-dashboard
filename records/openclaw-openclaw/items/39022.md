## Summary

Keep open: current main and v2026.6.6 still only support persistent `/model` and session model overrides; turn-scoped model leases plus per-model turn-limit config remain an unresolved product/session-state decision.

## What This Changes

Request support for `/model <name> --turns N` and optional per-agent/per-model limits that warn before automatically reverting to the default model.

## Best Possible Solution

Define one opt-in session-owned model lease contract in the existing `/model` and session override path, then land schema, docs, and focused tests that preserve explicit user selections and provider/auth pins.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable direction is a single opt-in model-lease mechanism, not prompt-based enforcement or a second model-switch path.
