## Summary

Keep open: current main and v2026.6.8 still support persistent `/model` selections and auto fallback recovery, but not `/model --turns`, turn-lease warning/revert state, or per-model turn-limit config; this remains a product/session-state decision rather than conservative cleanup.

## What This Changes

Add `/model <name> --turns N` plus optional per-model configuration so temporary model switches automatically warn and revert to the session default after a fixed number of turns.

## Best Possible Solution

Define one opt-in session-owned model lease contract on the existing `/model` and session override path, with schema/docs/tests that preserve explicit user selections and auth-profile pins.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has no `/model --turns` parser path, lease counter, warning state, or per-model turn-limit config to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the product contract. The proposed turn lease is plausible, but the maintainable path should be a single session-owned lease mechanism rather than prompt-based enforcement or a second model-switch path.
