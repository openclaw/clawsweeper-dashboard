## Summary

Keep this open: current main still only supports persistent session model overrides, and the requested turn-limited temporary override would require new parser output, session state, config schema, expiry logic, warning delivery, and maintainer decisions about turn-count semantics.

## What This Changes

The issue requests `/model <name> --turns N` with warning and automatic reversion to the default model, plus per-agent `openclaw.json` model turn limits.

## Best Possible Solution

Add this as a core model-session feature: parse and validate `--turns`, persist temporary override metadata on the target session, decrement at the chosen completed-turn boundary, warn before expiry, clear through the existing model override path, notify the user, and document/test the config contract.

## Reproduction Assessment

Yes. Source inspection is high confidence: `/model gpt-5 --turns 10` has no parsed turn-budget field, and current session/config types have nowhere to store, warn, decrement, or revert it.

## Solution Assessment

Unclear. The direction is plausible, but the best fix needs maintainer choices for turn counting, warning timing, and config shape before a narrow implementation should be attempted.
