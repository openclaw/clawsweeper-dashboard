## Summary

Keep open: current main still only supports persistent model switching, not a turn-limited model override with countdown warning, automatic default revert, or a public model-limit policy. This is a plausible cost-governance feature, but it crosses command semantics, durable session state, notifications, and config API design.

## What This Changes

Request adds `/model <name> --turns N` plus per-agent model-limit policy so temporary premium model switches warn and automatically revert after N turns.

## Best Possible Solution

Define the model-governance contract first; if accepted, implement parser support, durable countdown state, warning/revert notifications, docs, and focused regression coverage around session model overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new model-governance behavior. Source inspection at current main shows no parser, directive contract, session-state, config, or docs path for `/model --turns`.

## Solution Assessment

Unclear. The requested behavior is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, default-revert semantics, and public config shape.
