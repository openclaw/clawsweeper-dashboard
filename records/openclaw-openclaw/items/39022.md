## Summary

Keep open: current main and the latest release still treat `/model` as a sticky session selection, with no parsed turn budget, countdown state, warning/revert path, or `models.limits` config contract. Related open work covers one-shot overrides and broader budget controls, but not this multi-turn per-model auto-revert policy.

## What This Changes

Add `/model <name> --turns N` plus optional per-agent model turn limits that warn and revert to the default model after the budget is consumed.

## Best Possible Solution

Define the model-governance contract first, then implement parser support, durable countdown state, warning/revert notifications, docs, and regression tests around the existing session model override path.

## Reproduction Assessment

Yes for the missing feature by source inspection: `/model gpt-5 --turns 10` has no parsed turn-budget field, and session/config types have nowhere to store, warn, decrement, or revert a model override. This is not a bug reproduction because the request is for new behavior.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, and public config shape before a narrow code change is safe.
