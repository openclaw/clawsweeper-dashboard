## Summary

Keep this open: current `main` still supports persistent `/model` overrides only, while turn-limited temporary model selection would require new command parsing, session state, config schema, expiry/revert behavior, warnings, and product decisions about turn-count semantics.

## What This Changes

Request adding `/model <name> --turns N` plus per-agent config limits so temporary premium model overrides warn before expiry and automatically revert to the default model.

## Best Possible Solution

Add this as a core model-session feature after design approval: parse and validate `--turns`, persist temporary override metadata on the target session, decrement at the chosen completed-turn boundary, warn before expiry, clear through the existing model override path, notify the user, and document/test the config contract.

## Reproduction Assessment

Yes. Source inspection is high confidence: `/model gpt-5 --turns 10` has no parsed turn-budget field, and current session/config types have nowhere to store, warn, decrement, or revert it.

## Solution Assessment

Unclear. The direction is plausible, but the best fix needs maintainer choices for turn counting, warning timing, notification behavior, and config shape before a narrow implementation should be attempted.
