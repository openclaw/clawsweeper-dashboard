## Summary

Keep open: current main still has sticky `/model` session selection and no `--turns` parser field, countdown state, warning/revert path, or `models.limits` config contract. This is a clear feature request that needs model-governance product/API decisions before implementation.

## What This Changes

Add `/model <name> --turns N` and config-backed per-model turn caps that warn and revert to the default model after the turn limit.

## Best Possible Solution

Define the model-governance contract first, then implement parser support, durable countdown state, warning/revert notifications, docs, and regression coverage around the existing session model override path.

## Reproduction Assessment

Yes, for the missing capability by source inspection: `/model gpt-5 --turns 10` has no parsed turn-budget field, and session/config types have nowhere to store, warn, decrement, or revert a model override. This is not a bug reproduction because the request is for new behavior.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, and public config shape before a narrow code change is safe.
