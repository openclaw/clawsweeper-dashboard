## Summary

Keep open: current main still implements persistent `/model` session selection, but not a turn-limited override, warning countdown, automatic default revert, or the proposed public `models.limits` contract. This is a plausible cost-governance feature, but it adds new command semantics, session state, and config policy, so it needs maintainer product/API review before implementation.

## What This Changes

Requests `/model <name> --turns N` plus per-agent config to warn and revert to the default model after a fixed number of turns.

## Best Possible Solution

Define the model-governance contract first; if accepted, add parser support, durable countdown state, warning/revert notifications, docs, and focused regression coverage around session model overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new model-governance behavior. Source inspection gives high confidence that current main has no parser, config, session-state, or docs path for `/model --turns`.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, default-revert semantics, and public config shape.
