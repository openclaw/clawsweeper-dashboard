## Summary

Keep open: current main still supports persistent session model switching, not a turn-limited model lease with countdown warning, automatic default revert, or public model-limit policy. The request is plausible cost-governance work, but it crosses command syntax, session state, provider/auth routing, notifications, and config design.

## What This Changes

The issue requests `/model <name> --turns N` plus per-agent model-limit config so temporary premium-model switches warn near expiry and revert to the session default after a fixed number of turns.

## Best Possible Solution

Define the model-governance contract first; if accepted, implement parser support, durable countdown state, warning/revert notifications, docs, and focused regression coverage around session model overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a new model-governance feature request. Source inspection on current main shows no parser, directive contract, session-state, config, or docs path for `/model --turns`.

## Solution Assessment

Unclear. The requested behavior is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, default-revert semantics, and public config shape.
