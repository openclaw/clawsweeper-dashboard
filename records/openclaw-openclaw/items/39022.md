## Summary

Keep open: current main still has persistent `/model` and `sessions.patch` model selections, but no `/model --turns`, turn lease counter, warning/revert state, or per-model turn-limit config. The related open items cover one-turn overrides, fallback persistence, and image-specific reverts, but none supersede this broader N-turn model lease request.

## What This Changes

Request support for per-model turn-limited model switches such as `/model <name> --turns N`, including warning, auto-revert, and per-agent config limits.

## Best Possible Solution

Define one opt-in session-owned model lease contract across `/model` and the Gateway/session model override path, preserving explicit user selections and auth pins while documenting turn counting, warnings, and revert behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has no `/model --turns` parser path, turn counter, warning state, or per-model turn-limit config to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the product contract. The proposed turn lease is plausible, but the maintainable path should be one session-owned lease mechanism rather than prompt enforcement, ad hoc cron cleanup, or a second competing model-switch path.
