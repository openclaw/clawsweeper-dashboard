## Summary

Keep open: current main still supports sticky session model switching, not a turn-limited model lease with countdown warnings and automatic default reversion. The related one-turn and image-revert requests are open but narrower, so they do not supersede this broader model-governance feature.

## What This Changes

The issue requests `/model <name> --turns N` plus per-agent model limit configuration, expiry warnings, and automatic reversion to the default model after a bounded number of turns.

## Best Possible Solution

Define the model-governance contract first; if accepted, implement additive parser/config/session lease state, warning and revert notifications, docs, and focused regression coverage around model override expiration.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a new model-governance feature request. Source inspection on current main shows no parser, directive contract, session-state, config, or docs path for `/model --turns`.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable path is an additive model-lease design that works with existing session override, auth, fallback, and live-switch semantics rather than treating it as a narrow bug fix.
