## Summary

Keep this open: current `main` still implements `/model` as a persistent session selection, and per-model turn limits with warning and auto-revert would add new model-governance behavior across parsing, session state, notifications, config, docs, and tests. The related one-shot override and image-analysis auto-revert items overlap on cost control, but they do not cover multi-turn budgets or per-agent maximum-turn policy.

## What This Changes

Add `/model <name> --turns N` and configurable per-model turn limits that warn near expiry and revert the session to its default model after the budget is used.

## Best Possible Solution

Define the model-governance contract first, then implement parser support, durable countdown state, warning and revert notifications, public config schema/docs, and focused regression tests around the existing session model override path.

## Reproduction Assessment

Yes for the missing feature by source inspection. `/model gpt-5 --turns 10` has no parsed turn-budget field, and current session/config types have nowhere to store, warn, decrement, or revert a model override.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer choices for turn boundaries, warning timing, notification behavior, persistence semantics, and public config shape before a narrow code change is safe.
