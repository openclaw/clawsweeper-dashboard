## What This Changes

The issue requests `web_fetch` to tolerate minor LLM URL formatting mistakes, especially whitespace immediately after `http://` or `https://`.

## Best Possible Solution

Add a small `web_fetch` URL canonicalization step before cache-keying, URL parsing, and guarded fetch, then keep the existing strict URL and SSRF validation on the normalized target.

## Reproduction Assessment

Yes, from source inspection: `web_fetch` trims only the outer string and then calls `new URL(params.url)`, while Node rejects `https:// docs.openclaw.ai`. I did not run a full gateway/tool invocation in this read-only review.

## Solution Assessment

Yes, a narrow tool-boundary canonicalizer is the best maintainable shape if it only fixes the observed protocol whitespace and leaves strict URL/SSRF checks intact. A config option or broad whitespace sanitizer would add unnecessary surface area.
