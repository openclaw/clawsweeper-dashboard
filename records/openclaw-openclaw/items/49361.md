## Summary

Keep open. Current main still omits `agents.list[].identity.name` from the Google Chat message-mode typing indicator fallback, and the PR's one-line runtime change matches the documented identity config path without introducing a concrete correctness or security issue. The linked broader Google Chat issue should not be fully auto-closed by this PR because it also tracks markdown/plain-text rendering.

## What This Changes

This PR updates the Google Chat typing-indicator display-name fallback in `extensions/googlechat/src/monitor.ts` to try `agent.identity.name` after top-level `agent.name` and before the hardcoded `OpenClaw` fallback.

## Best Possible Solution

Land the narrow Google Chat fallback change for the typing-indicator behavior, add a small regression test around message-mode typing with `identity.name` and no top-level `name`, and keep the markdown rendering portion of #49350 tracked separately.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure a routed Google Chat agent with `identity.name` but no top-level `name`, leave `channels.googlechat.typingIndicator` in message mode, and current main will send `_OpenClaw is typing..._` because the fallback returns the hardcoded string.

## Solution Assessment

Yes for the typing-indicator half. Adding `agent.identity.name` before the hardcoded fallback is the narrowest maintainable fix within the Google Chat plugin, but it is not a complete solution for the linked markdown-rendering issue.
