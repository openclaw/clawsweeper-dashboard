## Summary

Keep open: the patch appears to be the right narrow code shape with no blocking code findings, and current main plus the latest release still have the warning-backoff/recovery coupling. It still needs real behavior proof before merge because the PR body only shows regression tests and local check output, not an after-fix gateway or diagnostic-path run.

## What This Changes

The PR changes diagnostic session attention so warning/event backoff can stay throttled while recovery-eligible stuck sessions still return a classification, and updates diagnostic tests for throttled recovery ticks.

## Best Possible Solution

Land this PR after redacted real diagnostic-path proof is added, then let it close https://github.com/openclaw/openclaw/issues/92742 while keeping QQ fast-ack and hook watchdog work separate.

## Reproduction Assessment

Yes. Source inspection shows current main can return `undefined` from `logSessionAttention` during warning backoff before the heartbeat reaches `requestStuckSessionRecovery`; I did not run a live QQ gateway reproduction in this read-only review.

## Solution Assessment

Yes for the code shape. Separating warning/event suppression from the returned recovery classification is the narrowest maintainable fix, and it avoids the event-cadence regression present in the related open branch.
