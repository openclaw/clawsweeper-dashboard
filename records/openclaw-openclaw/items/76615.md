## Summary

Keep this PR open. The credential-masking direction is valid and source inspection confirms current main echoes credential text prompts, but the patch is not ready because it drops existing/default secret handling on masked prompts, leaves the bundled session CLI echo path unmasked, and needs the required changelog entry.

## What This Changes

The PR adds a `sensitive` flag to wizard text prompts, routes sensitive clack prompts through `@clack/prompts.password()`, marks onboarding credential prompts sensitive, and adds session/search coverage.

## Best Possible Solution

Land a complete masking fix that preserves existing secret resolution without displaying secrets, masks every bundled wizard client path that renders sensitive text, and records the user-facing hardening in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main routes credential entry through `prompter.text()`, and the clack text implementation has no sensitive/password branch.

## Solution Assessment

No. The approach is directionally right, but the current patch is not the best mergeable solution until it preserves default/existing secret behavior and masks the session CLI path too.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve masked prompt defaults:** `src/wizard/clack-prompter.ts:123-126`
  - body: The sensitive branch drops `initialValue` before validation. Gateway prompts in this PR still pass existing/env values through `initialValue`, so pressing Enter no longer keeps the current token/password and required prompts reject blank input; preserve the default outside the rendered password value or update those callers to keep existing secrets without echoing them.
  - confidence: 0.88
- **[P2] Mask session wizard text in the macOS CLI:** `src/wizard/session.ts:126-127`
  - body: Forwarding `sensitive` through `WizardSession` is not enough while the bundled `openclaw-mac wizard` CLI still reads every text step with the normal echoing line reader. Credential prompts rendered through that session client can still land in terminal history, so the client needs a no-echo path for sensitive text steps.
  - confidence: 0.78
