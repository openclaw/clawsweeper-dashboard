## Summary

Keep this PR open. Current main still routes onboarding credential prompts through cleartext wizard text input, but this patch should not merge as-is because the masked path changes existing `initialValue` behavior, leaves a session wizard CLI client echoing sensitive text, and lacks the required changelog entry.

## What This Changes

The PR adds a `sensitive` option to wizard text prompts, routes clack sensitive prompts through `@clack/prompts.password()`, marks onboarding credential prompts sensitive, and adds session/search assertions.

## Best Possible Solution

Land a version that masks credential prompts across clack and session clients, preserves existing/default secret resolution without rendering secret values, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main credential setup call sites use `prompter.text()`, and the clack implementation always dispatches to `text()` rather than `password()`. I did not run an interactive TTY transcript because this read-only review avoided tests and generated artifacts.

## Solution Assessment

No. The direction is right, but the current patch is not the best mergeable form because it silently drops masked prompt defaults, does not update every bundled session client that can render sensitive text steps, and misses required release-note coverage.
