## Summary

## What This Changes

The PR adds a sensitive wizard text flag, routes sensitive Clack prompts through password input, marks onboarding credential prompts as sensitive, adds a macOS CLI no-echo reader, and updates tests plus changelog.

## Best Possible Solution

Merge the sensitive prompt plumbing after macOS CI is green or proven unrelated, keeping existing-secret confirm prompts and focused regression coverage.

## Solution Assessment

Yes, with CI still to resolve. The PR uses the narrow existing wizard prompter/session boundary, avoids prefilled sensitive values by adding masked-preview confirm prompts, and replaces the macOS getpass path with a termios no-echo reader.
