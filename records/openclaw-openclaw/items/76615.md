## Summary

Keep this PR open: the main-branch leak is source-reproducible and the fix direction is right, but the branch still has merge blockers around default credential handling, the session CLI renderer, and changelog coverage.

## What This Changes

The PR adds a sensitive flag to wizard text prompts, routes sensitive clack prompts through password input, marks onboarding credential prompts sensitive, and updates focused assertions.

## Best Possible Solution

Land a version that masks credential entry in every bundled wizard renderer, preserves existing/default secret resolution without rendering secrets, and records the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main routes credential setup through WizardPrompter.text(), and the clack implementation always calls @clack/prompts.text() with no sensitive flag.

## Solution Assessment

No. The proposed direction is right, but the current patch is not the best mergeable form until it preserves default credential behavior, masks the macOS CLI session path, and adds the required changelog entry.
