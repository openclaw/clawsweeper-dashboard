## Summary

Keep open. Current main and latest release still lack the requested public config/env opt-out for external CLI auth-profile discovery, and the live discussion explicitly keeps this issue as the canonical tracker after the internal discovery refactor landed.

## What This Changes

Request a first-class config/env or logging policy to disable external CLI auth-profile sync when users do not want local CLI credentials discovered by OpenClaw.

## Best Possible Solution

Keep this as the canonical tracker and add an owner-approved auth-level external CLI discovery policy with aligned config types, strict schema/help/docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows auth.externalCliSync is absent from the public config type/schema, and the sync gate only honors internal options plus OPENCLAW_AUTH_STORE_READONLY.

## Solution Assessment

Yes at the direction level. A central auth-level policy is the clean boundary, but maintainers still need to approve the exact config/env shape and precedence before implementation.
