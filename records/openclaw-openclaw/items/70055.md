## Summary

Keep open. Current main and latest release v2026.6.8 still lack the requested public config or env opt-out for external CLI auth-profile sync, and the live maintainer discussion explicitly keeps this issue as the canonical tracker after the internal discovery refactor landed.

## What This Changes

Requests a public config, environment, or log-level opt-out for external CLI auth-profile credential sync so installed CLIs do not cause unwanted auth discovery and noisy CLI output.

## Best Possible Solution

Keep this as the canonical tracker and add an owner-approved central auth-level external CLI discovery policy with strict schema/help/docs coverage, default-on compatibility, and focused auth-store tests.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing user-facing surface: the auth config type/schema has no opt-out field and the sync gate only honors internal options plus `OPENCLAW_AUTH_STORE_READONLY`.

## Solution Assessment

Yes at the direction level: a central auth-level policy is the right boundary for this behavior. The exact config/env shape and precedence remain a maintainer product/API decision before implementation should proceed.
