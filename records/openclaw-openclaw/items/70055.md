Keep open. Current main and v2026.6.10 still do not expose a public config or environment opt-out for external CLI auth discovery, and maintainer discussion explicitly kept this issue as the canonical follow-up after the internal discovery refactor landed.

## What This Changes

Request a public config or environment opt-out for external CLI auth-profile discovery so local CLI credentials are not read or overlaid unless wanted.

## Best Possible Solution

Keep this as the canonical tracker and add an owner-approved auth-level external CLI discovery policy with aligned config types, strict schema/help/docs, and focused tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainer product direction is set. A central auth-level opt-out is the clean boundary, but the exact config/env shape and precedence still need approval before implementation.
