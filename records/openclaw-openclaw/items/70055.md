## Summary

Keep open. Current main has the internal scoped/explicit external CLI discovery work, but it still lacks the requested user-facing config or environment opt-out, and a maintainer explicitly kept this as the canonical tracker after #75209 merged and #61694 was closed as the duplicate.

## What This Changes

Add a supported, default-on config or environment control to disable or scope external CLI credential discovery for auth profiles.

## Best Possible Solution

Add a documented, default-on auth-level external CLI discovery policy with an env override, gate the existing discovery-intent path centrally, and cover config schema/help, auth docs, changelog, and focused auth/gateway/model-list tests.

## Reproduction Assessment

Yes for the remaining gap: current-main source, schema, docs, and tests show no user-facing config or environment opt-out. The original double info-log reproduction is no longer the best proof because `models list` now avoids external overlays and the sync log path is debug-level/scoped.

## Solution Assessment

Yes: a narrow, additive config/env gate at the existing `ExternalCliAuthDiscovery` boundary is the maintainable direction. It should not redesign provider auth, plugin APIs, or external CLI bootstrap semantics.
