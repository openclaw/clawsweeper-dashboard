## What This Changes

The issue requests a first-class config or environment opt-out for external CLI auth-profile credential discovery/sync while preserving the default-enabled behavior.

## Best Possible Solution

Keep this as the canonical tracker and add an owner-approved auth-level external CLI discovery policy with aligned config types, strict schema, help/docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.18 shows no public auth.externalCliSync/config or dedicated env override, while the auth-store sync gate only reads internal options and OPENCLAW_AUTH_STORE_READONLY.

## Solution Assessment

Yes at the direction level. A central auth config/env gate is the narrow maintainable path, but the exact schema and precedence should be approved because it controls credential discovery behavior.
