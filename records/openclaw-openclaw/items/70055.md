## Summary

Keep open. Current main has explicit internal external-CLI discovery modes and scoped probing, but it still has no user-facing config or environment opt-out; a maintainer also confirmed this issue is the canonical tracker for that remaining policy surface.

## What This Changes

Request a supported `openclaw.json` and/or environment opt-out for external CLI auth-profile sync/discovery.

## Best Possible Solution

Keep this as the canonical tracker and add an owner-approved auth-level external CLI discovery policy, plus docs/schema/help and focused auth-profile/config coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gap: config types/schema/help have no `auth.externalCliSync` field, and the auth-store gate still reads only internal options plus `OPENCLAW_AUTH_STORE_READONLY`.

## Solution Assessment

Yes at the direction level. A central additive auth config/env gate is the maintainable path, but the exact public schema and precedence should be approved first because this is an auth credential policy surface.
