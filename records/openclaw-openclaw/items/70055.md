## Summary

Keep open. Current main has narrowed external CLI credential discovery and removed part of the original noisy behavior, but it still has no user-facing config or environment policy to disable external CLI auth-profile discovery; maintainer discussion also explicitly keeps this as the canonical tracker for that remaining opt-out.

## What This Changes

The issue requests a default-on config/env opt-out for external CLI auth-profile discovery so local Codex, Claude, or MiniMax CLI credentials are not imported into OpenClaw runtime auth unless wanted.

## Best Possible Solution

Add a documented, default-on auth-level external CLI discovery policy that can disable runtime external CLI overlays from config and an environment override, while preserving the current scoped discovery behavior, no-prompt status paths, and runtime-only persistence model.

## Reproduction Assessment

Yes. Current-main source and docs provide a high-confidence reproduction for the remaining feature gap: there is no config or environment field that disables external CLI auth discovery. The exact old `models list` double info-log reproduction is partly obsolete because current main now avoids external overlays there and uses scoped discovery elsewhere.

## Solution Assessment

Yes. A central config/env gate in the auth-profile overlay/discovery path is the narrowest maintainable solution; provider-specific flags or only downgrading logs would leave the generic runtime auth coupling in place.
