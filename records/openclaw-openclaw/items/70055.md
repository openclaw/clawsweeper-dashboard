## Summary

Keep open. Current main has the internal scoped external CLI discovery cleanup, but it still lacks a user-facing config or environment opt-out, and the maintainer discussion explicitly kept this issue as the canonical tracker after #75209 merged and #61694 was closed as the duplicate.

## What This Changes

Request a default-on config or env opt-out for external CLI credential discovery into auth profiles so local CLI credentials are not imported or logged when users do not want OpenClaw to use them.

## Best Possible Solution

Add a documented auth-level external CLI discovery policy with an env override that feeds the existing `ExternalCliAuthDiscovery` gate centrally while preserving default-on and scoped behavior.

## Reproduction Assessment

Yes. Current-main source, strict schema, help metadata, docs, and exact searches show users cannot set `auth.externalCliSync` or an env override; the issue’s log-noise symptom is partly mitigated by scoped discovery, but the requested opt-out remains absent.

## Solution Assessment

Yes. A narrow additive config/env gate at the existing external CLI discovery boundary is the maintainable path; provider-specific flags, startup migrations, or a provider-auth redesign would be broader than this request.
