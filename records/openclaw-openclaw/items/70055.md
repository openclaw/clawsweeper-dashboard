## What This Changes

Add a default-on config or environment policy that lets users disable external CLI credential discovery/sync for auth profiles.

## Best Possible Solution

Add a documented auth-level external CLI discovery policy with an env override that feeds the existing discovery gate centrally while preserving scoped discovery behavior.

## Reproduction Assessment

Yes. Source inspection of current main shows the remaining gap: users cannot set `auth.externalCliSync` or an env override, while external CLI discovery is still controlled by internal call-site options.

## Solution Assessment

Yes. A narrow additive config/env gate at the existing `ExternalCliAuthDiscovery` and auth-store load boundary is the maintainable path; it should not redesign provider auth, plugin APIs, or bootstrap semantics.
