## Summary

Keep open. Current main has narrowed and quieted parts of external CLI credential discovery, but it still lacks the requested user-facing config or environment opt-out, and the live maintainer discussion keeps this issue as the canonical tracker after the duplicate was closed.

## What This Changes

The issue requests a user-facing config or environment control to disable external CLI credential discovery/sync into auth profiles.

## Best Possible Solution

Keep this as the canonical tracker and add a documented auth-level external CLI discovery policy, with an env override, that feeds the existing central discovery gate without widening credential probing.

## Reproduction Assessment

Yes. Source inspection of current main shows the remaining gap: the config type/schema/help have no `auth.externalCliSync` field and the auth-store gate still reads only internal call-site options plus `OPENCLAW_AUTH_STORE_READONLY`.

## Solution Assessment

Yes at the direction level. A central additive auth config/env gate is the maintainable path, but the exact public schema and env precedence should be approved before implementation because this is an auth/credential policy surface.
