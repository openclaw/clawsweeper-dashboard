## Summary

Keep open. Current main still classifies the read-only `config.schema.lookup` gateway action as warning-eligible, and open PRs #42898 and #42962 already target the same bug.

## What This Changes

The issue asks for missing-path `config.schema.lookup` failures to return a structured tool error without sending a user-facing channel notification.

## Best Possible Solution

Land one canonical classifier-level fix that respects compound gateway method read/write semantics, with regression coverage proving lookup misses stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `gateway` with `action: "config.schema.lookup"` and a missing `path` returns `config schema path not found` while the action is still classified as mutating for warning rendering.

## Solution Assessment

Yes. The narrow maintainable fix is to classify compound gateway actions by their read-only leaf or gateway method scope before payload rendering; a one-off payload suppression is less general.
