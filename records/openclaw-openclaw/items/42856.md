## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating even though the gateway scope contract marks that method read-only, and two open closing PRs already cover the issue.

## What This Changes

The issue asks for missing-path `config.schema.lookup` errors to return to the agent without emitting user-facing Telegram/channel warning notifications.

## Best Possible Solution

Land one coordinated classifier-level fix so compound gateway read-only actions are non-mutating for warning policy while real gateway writes, restarts, and failures still warn, with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: a `gateway` tool call with `action: "config.schema.lookup"` and a missing path returns `config schema path not found`, but current main classifies that dotted action as mutating and renders mutating tool errors as warnings.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the mutation classifier respect compound gateway read-only semantics; a payload-only exception is incomplete unless the runtime also marks the lookup miss non-mutating.
