## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating, so missing-path lookup errors can still flow into user-facing warning payloads; the linked open fix PR should be reviewed and landed or replaced before this issue closes.

## What This Changes

The issue asks OpenClaw to stop sending user-facing warnings for benign failed `config.schema.lookup` gateway calls.

## Best Possible Solution

Apply a classifier-level fix that marks declared/read-scoped compound gateway actions as non-mutating while preserving fail-closed behavior for unknown or write-like gateway actions.

## Reproduction Assessment

Yes. Source inspection shows a missing `config.schema.lookup` path returns `config schema path not found`, while the current mutation classifier still treats the dotted gateway action as mutating and passes that state into warning rendering.

## Solution Assessment

Yes. The narrow maintainable fix is to classify read-scoped compound gateway actions correctly; suppressing one payload error string would leave the underlying mutation state wrong.
