## Summary

Keep open. Current `main` still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating, so the reported missing-path error can still flow into the visible failed-tool warning path; the open closing PR should be reviewed, fixed, proven, landed, or replaced.

## What This Changes

The issue reports that read-only `config.schema.lookup` misses return a structured error but also produce visible channel warning notifications.

## Best Possible Solution

Classify declared read-scoped gateway actions such as `config.get` and `config.schema.lookup` as non-mutating while keeping unknown or write-like gateway actions fail-closed, then land or replace the linked PR with focused coverage and real behavior proof.

## Reproduction Assessment

Yes, at source level. Current `main` returns the reported `config schema path not found` error for lookup misses while still classifying the dotted gateway action as mutating, which the payload path renders as a warning candidate.

## Solution Assessment

Yes for the issue direction, but the linked PR should avoid broad leaf-verb matching. The narrow maintainable fix is to classify only declared/read-scoped gateway actions as non-mutating and keep unsupported gateway actions fail-closed.
