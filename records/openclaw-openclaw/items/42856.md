## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as warning-eligible even though the gateway scope table treats that method as read-only, and the linked fix PRs remain open and unmerged.

## What This Changes

The issue requests that read-only `config.schema.lookup` misses return a structured error without producing user-facing channel warnings.

## Best Possible Solution

Land one classifier-level fix that respects compound gateway read/write semantics, with regression coverage proving lookup misses stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.schema.lookup` misses return `config schema path not found`, the current classifier still marks the compound action as mutating, and payload code renders mutating tool errors as warnings.

## Solution Assessment

Yes. A classifier-level fix that treats compound read-only gateway methods as non-mutating is the narrowest durable solution; a payload-only exception for one error string would leave other compound read-only actions misclassified.
