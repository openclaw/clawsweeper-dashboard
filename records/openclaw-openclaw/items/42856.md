## Summary

Keep open. Current main still treats the compound read-only gateway lookup as a mutating tool call, so the reported path can still produce a user-facing warning; open PRs #42898 and #42962 already target the issue and should be reviewed rather than queuing another repair.

## What This Changes

The issue asks OpenClaw to keep nonexistent-path `config.schema.lookup` errors as structured tool errors without broadcasting a visible channel notification.

## Best Possible Solution

Land one canonical fix that makes warning policy respect compound gateway method semantics, preferably by classifying `gateway` actions from their leaf read/write behavior before payload rendering. Add regression coverage proving missing `config.schema.lookup` paths stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: call the `gateway` tool with `action: "config.schema.lookup"` and a missing path, which returns the structured `config schema path not found` error while current main still marks the tool call as mutating and warning-eligible.

## Solution Assessment

No fix is on main yet. The classifier-level direction in #42898 appears narrower and more maintainable than a payload-only special case; any suppression should still ensure read-only compound gateway actions are classified correctly.
