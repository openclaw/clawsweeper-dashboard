## Summary

Keep open. Current main at 66c64a29ee607888fb5597a7794b0ad77124920e still marks `gateway` calls with `action: "config.schema.lookup"` as mutating, so missing-path lookup errors can still feed user-facing warning payloads; https://github.com/openclaw/openclaw/pull/42898 remains the open closing PR and should be reviewed, proven, landed, or replaced before this issue closes.

## What This Changes

A Telegram-visible bug where missing-path `config.schema.lookup` returns the expected structured error but also produces a user-facing gateway warning.

## Best Possible Solution

Land or replace the classifier-level fix so declared/read-scoped compound gateway actions are non-mutating while unknown or write-like gateway actions remain fail-closed.

## Reproduction Assessment

Yes, at source level. A missing `config.schema.lookup` path returns `config schema path not found`, while the current classifier treats the dotted gateway action as mutating and the payload path renders mutating tool failures as warnings.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the mutation classifier for declared/read-scoped compound gateway actions; suppressing one error string in payload rendering would leave the underlying action classification wrong.
