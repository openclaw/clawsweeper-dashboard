## Summary

Keep open because current main and the latest release still route the reported read-only lookup miss through the mutating-tool warning path, and an open linked PR still references this issue as its closing target.

## What This Changes

Read-only `config.schema.lookup` misses should return the structured gateway error to the agent without posting a visible warning into the active channel.

## Best Possible Solution

Classify declared read-scoped compound gateway actions as non-mutating while keeping unknown or write-like gateway actions fail-closed, with regression coverage and redacted gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level: current main returns the reported missing-path error while classifying `gateway` plus `config.schema.lookup` as mutating, which feeds the visible warning policy. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes for the issue direction, but the narrowest maintainable fix is the classifier boundary rather than broad payload suppression. The fix should use the gateway method scope contract or an exact read-only action list so unknown actions remain fail-closed.
