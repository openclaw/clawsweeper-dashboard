## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as warning-eligible, and open PRs #42898 and #42962 already target this bug, so this issue should stay open until one canonical fix lands.

## What This Changes

The issue asks for missing-path `config.schema.lookup` errors to remain a silent read-only tool result instead of producing channel notifications.

## Best Possible Solution

Land one classifier-level fix that respects compound gateway read/write semantics, with regression coverage proving `config.schema.lookup` misses stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `gateway` tool call with `action: "config.schema.lookup"` and a missing `path` returns `config schema path not found` while the current classifier still marks that call as mutating for warning rendering.

## Solution Assessment

Yes. The narrow maintainable fix is to classify compound gateway actions by their read-only method or leaf semantics before payload rendering; a one-off payload suppression would be less general.
