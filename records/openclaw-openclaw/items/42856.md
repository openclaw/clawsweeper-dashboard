## Summary

Keep open. Current main still classifies the compound read-only gateway lookup as warning-eligible, and open PRs #42898 and #42962 already target the issue, so this should stay open until one canonical fix lands or the competing approaches are closed.

## What This Changes

Suppress user-facing channel notifications when the agent gateway tool receives expected read-only `config.schema.lookup` path-miss errors.

## Best Possible Solution

Land one canonical classifier-level fix that respects compound gateway method read/write semantics, with regression coverage proving missing `config.schema.lookup` paths stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: call the `gateway` tool with `action: "config.schema.lookup"` and a missing `path`, which current main forwards to the gateway, returns `config schema path not found`, and still carries a mutating-action warning path.

## Solution Assessment

Yes, the narrow maintainable direction is to classify compound gateway actions by their read-only leaf or method scope before payload rendering; a payload-only suppression is a less general fallback.
