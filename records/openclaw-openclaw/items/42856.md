## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating, so the reported missing-path error can still flow into the visible failed-tool warning path; the open linked PR should be reviewed, proven, landed, or replaced.

## What This Changes

The issue requests that read-only `config.schema.lookup` missing-path errors return to the agent without producing user-facing channel notifications.

## Best Possible Solution

Classify declared read-scoped gateway actions such as `config.schema.lookup` as non-mutating while keeping unknown or write-like gateway actions fail-closed.

## Reproduction Assessment

Yes, at source level. Current main returns the reported missing-path error while still classifying the dotted `config.schema.lookup` gateway action as mutating, which feeds the visible warning payload path.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix is to classify read-scoped gateway actions correctly, not to broadly suppress gateway errors at the payload layer.
