## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating, so the reported missing-path lookup error can still flow into the user-visible warning path; the linked open PR should be reviewed, proven, landed, or replaced.

## What This Changes

The issue asks for read-only `config.schema.lookup` path misses to return structured tool errors without posting Telegram or other channel-facing warning notifications.

## Best Possible Solution

Classify declared read-scoped compound gateway actions such as `config.schema.lookup` as non-mutating while leaving unknown or write-like gateway actions fail-closed.

## Reproduction Assessment

Yes, at source level. Current main returns the reported missing-path error while still classifying dotted `config.schema.lookup` gateway actions as mutating, which feeds the visible warning payload path.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix is to correct gateway action mutation classification rather than broadly suppressing gateway errors at payload rendering time.
