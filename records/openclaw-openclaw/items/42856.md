## Summary

Keep open. Current main and the latest shipped release still classify `gateway` calls with `action: "config.schema.lookup"` as mutating, so the reported read-only lookup miss can still flow into a user-facing failed-tool warning; the linked fix PR at https://github.com/openclaw/openclaw/pull/42898 should be repaired/proven and landed or replaced.

## What This Changes

The issue requests that read-only `config.schema.lookup` missing-path errors return to the agent without emitting visible Telegram or gateway warning messages.

## Best Possible Solution

Classify declared read-scoped compound gateway actions such as `config.schema.lookup` as non-mutating while keeping unknown or write-like gateway actions fail-closed, then land with regression coverage and redacted gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main returns the reported missing-path error while still classifying dotted `config.schema.lookup` gateway actions as mutating, which feeds the visible warning payload path.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix is to correct read-scoped gateway action classification, not to broadly suppress all gateway errors at the payload layer.
