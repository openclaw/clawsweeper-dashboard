## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating, so the reported missing-path error can still become a user-facing channel warning; the open linked pull request should be reviewed, proven, landed, or replaced.

## What This Changes

A non-existent `config.schema.lookup` path should return the structured gateway error silently instead of producing a visible Telegram warning.

## Best Possible Solution

Classify declared read-scoped compound gateway actions such as `config.schema.lookup` as non-mutating while keeping unknown or write-like gateway actions fail-closed.

## Reproduction Assessment

Yes, at source level. Current main returns the reported missing-path error while still classifying the dotted `config.schema.lookup` gateway action as mutating, which feeds the visible warning payload path.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix is to classify read-scoped gateway actions correctly, not to broadly suppress gateway errors at the payload layer.
