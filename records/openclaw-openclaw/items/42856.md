## Summary

Keep open: current main and the latest release still classify `gateway` `config.schema.lookup` misses as mutating, so the reported user-facing warning path remains. The target-repo fix attempts were closed unmerged; the remaining work should stay on this issue or a successor target-repo PR.

## What This Changes

The issue asks for missing-path `config.schema.lookup` errors to return to the agent without emitting visible channel notifications.

## Best Possible Solution

Fix the mutation classifier so declared read-scoped gateway actions like `config.schema.lookup` are non-mutating while unknown or write-like gateway actions remain fail-closed, then add focused regression coverage and redacted gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level: current main returns the reported missing-path error while classifying `gateway` plus `config.schema.lookup` as mutating, which feeds the visible warning policy. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes for the issue direction, but the maintainable fix belongs at the classifier boundary rather than a payload-only special case. It should use the gateway read-scope contract or a narrow exact allowlist so unknown actions remain fail-closed.
