## Summary

Keep open. Current `origin/main` and the latest release still classify `gateway` calls with `action: "config.schema.lookup"` as mutating even though the gateway scope table marks the method read-only, and the linked closing PRs remain open.

## What This Changes

The issue asks OpenClaw to return `config.schema.lookup` path-miss errors to the agent without sending a visible warning to active channels such as Telegram.

## Best Possible Solution

Respect compound gateway read-only semantics in the mutation classifier so lookup misses stay silent while real gateway writes, restarts, and failures still warn.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a `gateway` call with `action: "config.schema.lookup"` and a missing path returns `config schema path not found`, but the current mutation classifier treats that dotted action as mutating and the payload builder renders mutating tool errors as user-facing warnings.

## Solution Assessment

Yes. The narrowest maintainable fix is at the classifier/compound-action level, because special-casing one payload error string would leave other read-only compound gateway methods misclassified.
