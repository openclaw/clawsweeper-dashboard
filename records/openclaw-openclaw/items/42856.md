## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating, so missing-path lookup errors can still reach user-facing warning rendering; the paired open fix at https://github.com/openclaw/openclaw/pull/42898 should be reviewed and landed or replaced before this issue closes.

## What This Changes

The issue asks for missing-path `config.schema.lookup` errors to return silently to the agent without creating user-facing channel notifications.

## Best Possible Solution

Land an adjusted classifier-level fix that marks declared/read-scoped gateway lookup actions as non-mutating while keeping unsupported or write-like gateway actions fail-closed, with focused regression coverage and redacted real Telegram or gateway proof.

## Reproduction Assessment

Yes, at source level. A `gateway` tool call with `action: "config.schema.lookup"` and a missing path returns `config schema path not found`, but current main classifies that dotted action as mutating and the payload path renders mutating tool errors as warnings.

## Solution Assessment

Yes, the best fix boundary is the mutation classifier respecting declared or read-scoped compound gateway actions. A payload-only exception for one error string is less complete because the runtime state would still misclassify the lookup miss.
