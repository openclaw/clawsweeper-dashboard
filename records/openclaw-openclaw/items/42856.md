## Summary

Keep open. Current main still treats the compound read-only gateway action as warning-eligible, and open PRs #42898 and #42962 already target the same bug, so the issue should stay open until one canonical fix lands or the competing approaches are closed.

## What This Changes

Suppress user-facing channel warnings when read-only gateway `config.schema.lookup` calls miss a config path while still returning the structured tool error to the agent.

## Best Possible Solution

Land one canonical classifier-level fix that respects compound gateway method read/write semantics, with regression coverage proving missing `config.schema.lookup` paths stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call the `gateway` tool with `action: "config.schema.lookup"` and a missing `path`; current main returns `config schema path not found` while still classifying the tool error as mutating for warning rendering.

## Solution Assessment

Yes. The narrow maintainable fix is to classify compound gateway actions by their read-only leaf or gateway method scope before payload rendering; a one-off payload suppression would be less general.
