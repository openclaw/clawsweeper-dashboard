## Summary

Keep open. Current main still treats `gateway` calls with `action: config.schema.lookup` as mutating even though the gateway contract marks that RPC read-only, and the open linked PR remains the implementation path before this issue should close.

## What This Changes

The issue reports that read-only `config.schema.lookup` misses return a structured gateway error but also trigger a visible channel warning.

## Best Possible Solution

Land or replace the linked classifier fix so declared read-scoped compound gateway actions are non-mutating while unknown or write-like gateway actions remain fail-closed, with regression coverage and redacted gateway or Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main returns the reported missing-path error while still classifying dotted `config.schema.lookup` gateway actions as mutating, which feeds the visible warning policy.

## Solution Assessment

Yes for the issue direction. The best narrow fix is to classify read-scoped compound gateway actions correctly at the mutation classifier, preferably from the gateway method scope or exact read-only contract rather than broad payload suppression.
