## Summary

Keep open. Current main still classifies `gateway` `config.schema.lookup` as mutating for warning purposes even though the gateway method is read-scoped and returns the reported `config schema path not found` error; linked PRs #42898 and #42962 remain open implementation candidates.

## What This Changes

Read-only `config.schema.lookup` misses should return the structured error without broadcasting a warning to active chat channels.

## Best Possible Solution

Land one canonical classifier-level fix that respects compound gateway method read/write semantics, with regression coverage proving missing `config.schema.lookup` paths stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: a `gateway` tool call with `action: "config.schema.lookup"` and a missing `path` returns the expected lookup error, but current main still marks that compound action as mutating and routes it through warning payload rendering.

## Solution Assessment

Yes. The narrow maintainable solution is to classify compound gateway actions by read-only leaf or gateway method scope before payload rendering; a payload-only suppression is a less general fallback.
