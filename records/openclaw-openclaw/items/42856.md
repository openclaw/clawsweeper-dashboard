## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as warning-eligible even though the gateway scope table marks the method read-only, and open PRs #42898 and #42962 already target this bug.

## What This Changes

The issue requests suppressing user-facing notifications when read-only `config.schema.lookup` calls miss a config schema path.

## Best Possible Solution

Land one classifier-level fix that respects compound gateway read/write semantics, with regression coverage proving lookup misses stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.schema.lookup` misses return `config schema path not found`, the current classifier still marks the compound action as mutating, and the payload code renders mutating tool errors as warnings.

## Solution Assessment

Yes. A classifier-level fix that treats `config.schema.lookup` as read-only is the narrowest durable solution; a payload-only exception for one error string would leave other compound read-only gateway actions misclassified.
