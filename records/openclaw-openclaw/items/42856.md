## Summary

Keep open. Current main still treats `gateway` calls with `action: "config.schema.lookup"` as warning-eligible even though the gateway contract marks that method read-only, and the two source-repo fix PRs that close this issue are still open and unmerged.

## What This Changes

Make non-existent `config.schema.lookup` read-only misses return only a structured gateway error without a user-facing Telegram/tool-warning notification.

## Best Possible Solution

Land one classifier-level fix that respects compound gateway read/write semantics, with regression coverage proving lookup misses stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the lookup miss returns `config schema path not found`, the agent classifier marks the compound gateway action as mutating, and the payload layer renders mutating tool errors as user-facing warnings.

## Solution Assessment

Yes. A classifier-level fix that uses gateway read/write semantics or the compound leaf verb is the narrowest durable solution; a payload-only exception for one error string would leave other compound read-only gateway actions misclassified.
