## Summary

Keep open: current main still routes a missing `config.schema.lookup` through the mutating-tool warning path, and open closing PRs already target the same bug.

## What This Changes

Suppress user-facing channel notifications for failed read-only `config.schema.lookup` misses while still returning the structured error to the agent.

## Best Possible Solution

Land one canonical classifier-level fix that respects compound gateway read/write semantics, with regression coverage for silent `config.schema.lookup` misses and preserved warnings for real mutating failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `gateway` tool call with `action: "config.schema.lookup"` and a missing path returns `config schema path not found`, while the current mutation classifier still marks that compound action as mutating.

## Solution Assessment

Yes. A classifier-level fix that recognizes compound read-only gateway methods is the narrowest maintainable solution; a payload-only suppression would be less general and should not replace read/write classification.
