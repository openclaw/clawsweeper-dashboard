## Summary

Keep open. Current main still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating, so the reported missing-path error can still reach the user-facing warning path; https://github.com/openclaw/openclaw/pull/42898 remains the open closing PR and should be reviewed, proven, landed, or replaced.

## What This Changes

The issue asks OpenClaw to stop sending user-facing channel notifications for read-only `config.schema.lookup` missing-path errors.

## Best Possible Solution

Classify declared read-scoped gateway actions such as `config.schema.lookup` as non-mutating while keeping unknown or write-like gateway actions fail-closed.

## Reproduction Assessment

Yes, at source level. Current main returns the reported `config schema path not found` error for lookup misses while still classifying the dotted gateway action as mutating, which the payload path renders as a warning candidate.

## Solution Assessment

Yes, the narrow maintainable fix is in the mutation classifier using the gateway read-scope contract. Suppressing only one payload error string would leave the underlying action classification wrong.
