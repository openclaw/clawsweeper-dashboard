## What This Changes

## Best Possible Solution

Revise the local TUI command boundary so help/autocomplete and dispatch agree: every advertised local slash command is handled, safely routed, or explicitly rejected before `sendMessage(raw)`, with local PTY proof.

## Reproduction Assessment

## Solution Assessment

No. The two-command patch is a useful mitigation, but the best fix is a local TUI command boundary where advertised shared commands are handled, routed, or rejected before the default send path.
