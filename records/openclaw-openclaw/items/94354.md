## Summary

## What This Changes

The branch adds local-mode `/status` and `/compact` switch cases that show unsupported-command system messages plus two unit tests asserting they do not call `sendChat`.

## Best Possible Solution

Make local TUI help/autocomplete and dispatch agree so every advertised local slash command is handled, safely routed, or explicitly rejected before it can reach the embedded model path, then prove it with a local PTY or live TUI run.

## Reproduction Assessment

Yes, from source: slash input enters `handleCommand`, local discovery/help expose shared commands, and unhandled names fall through to `sendMessage(raw)` and then `agentCommandFromIngress`. I did not run the local PTY repro because this review is read-only.

## Solution Assessment

No. The two-command branch is a plausible mitigation, but the best fix is the full local TUI command boundary plus local PTY or live proof rather than adding only two special cases.
