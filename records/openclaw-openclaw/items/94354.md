## What This Changes

The PR adds local-mode `/status` and `/compact` cases to the TUI command handler and unit tests that assert those two commands show an unsupported-mode system message instead of calling `sendChat`.

## Best Possible Solution

Make local TUI discovery, help, and dispatch agree so every advertised local slash command is handled, safely routed, or explicitly rejected before embedded send, then prove representative commands in a local TUI path.

## Reproduction Assessment

Yes, from source: slash input enters `handleCommand`, local discovery/help advertise shared commands, unhandled command names call `sendMessage(raw)`, and the embedded backend passes the message into `agentCommandFromIngress`. I did not run the local PTY repro in this read-only review.

## Solution Assessment

No. The two-command switch case is a plausible mitigation, but the best fix is a local command boundary that covers all advertised unsupported shared commands before embedded send.
