## Best Possible Solution

Make local TUI discovery/help and dispatch agree so every advertised local slash command is handled, safely routed, or explicitly rejected before embedded send, then prove it with local PTY or live TUI output.

## Reproduction Assessment

## Solution Assessment

No. The two-command switch case is a useful mitigation, but the best fix is a shared local command boundary that covers all advertised local commands with PTY or live TUI proof.
