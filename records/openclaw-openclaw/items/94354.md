## Best Possible Solution

Revise the local TUI command boundary so help/autocomplete and dispatch agree: every advertised local slash command is handled, safely routed, or explicitly rejected before the default send path, with local PTY proof.

## Reproduction Assessment

Yes from source: slash input enters `handleCommand`, local help/autocomplete expose shared commands, and unhandled command names reach `sendMessage(raw)` and then `agentCommandFromIngress`. I did not run the local PTY repro because this review is read-only.

## Solution Assessment

No. The two-command branch is a useful mitigation, but the best fix is to make local TUI discovery/help and dispatch share the same unsupported-command boundary so every advertised shared command is handled, routed, or rejected before `sendMessage(raw)`.
