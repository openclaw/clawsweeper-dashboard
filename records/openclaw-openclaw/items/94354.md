## Summary

Keep open: this is a focused and useful bug-fix PR, but it is not merge-ready because it only handles two advertised local slash commands while the same local fallback path still sends other advertised shared commands to the embedded model, and the contributor proof is unit/static output rather than real local TUI behavior.

## What This Changes

Adds local-mode `/status` and `/compact` cases to the TUI command handler plus two tests that expect unsupported-command system messages instead of `sendChat` calls.

## Best Possible Solution

Make local TUI help/autocomplete and dispatch agree so every advertised local slash command is handled, safely routed, or explicitly rejected before it can reach the embedded model path, then prove it with local PTY or live TUI output.

## Reproduction Assessment

Yes, from source: slash input enters `handleCommand`, local discovery/help expose shared commands, and unhandled names fall through to `sendMessage(raw)` and then `agentCommandFromIngress`. I did not run the local PTY repro because this review is read-only.

## Solution Assessment

No. The two-command branch is a plausible mitigation, but the best fix is the full local TUI command boundary plus local PTY or live proof rather than adding only two special cases.
