## Summary

Keep open: this PR is an active fix candidate for the canonical local TUI slash-command fallthrough, but it is not merge-ready because it only special-cases `/status` and `/compact` while other advertised shared commands can still fall through to the local embedded model path, and the contributor proof is only unit-test/static output rather than real TUI behavior.

## What This Changes

The PR adds local-mode `/status` and `/compact` cases to the TUI command handler and two focused unit tests that assert those commands show unsupported system messages instead of calling `sendChat`.

## Best Possible Solution

Make local TUI help/autocomplete and dispatch agree so every advertised local slash command is handled, safely routed, or explicitly rejected before the embedded model path, then prove it with local PTY or live TUI output.

## Reproduction Assessment

Yes. Source shows slash input enters `handleCommand`, local autocomplete/help expose shared commands, unhandled names reach `sendMessage(raw)`, and the embedded backend passes that text to `agentCommandFromIngress`; I did not run the PTY repro because this review is read-only.

## Solution Assessment

No. The two-command special case is a plausible mitigation, but the best fix is the full local TUI command boundary plus local PTY or live proof rather than only adding two switch cases.
