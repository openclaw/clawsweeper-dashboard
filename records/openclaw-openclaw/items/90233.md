## Label Justifications

- `P1`: The report describes a real channel workflow where a final assistant reply can be generated locally but never become visible to the user after gateway restart recovery.
- `impact:message-loss`: The affected behavior is a missing or stranded user-visible Signal/channel final reply after an interrupted send.
- `impact:session-state`: The durable session/transcript can appear complete while delivery state remains ambiguous or failed outside the session-visible flow.

## Summary

Current main still has the central failure class: a `send_attempt_started` outbound delivery with no adapter reconciliation is not replayed, but it is only logged and moved/marked in queue state, so the Signal-visible final reply can remain missing without a session-level recovery signal. A broader design issue and a partial draft PR exist, but neither is a safe merged replacement for this Signal/restart report.

## What This Changes

The issue asks OpenClaw to prevent or surface stranded Signal final replies when a gateway restart interrupts delivery after `send_attempt_started`.

## Best Possible Solution

Pick one owner-approved recovery contract for ambiguous outbound sends, then implement it generically through the existing delivery queue and channel adapter lifecycle, with Signal-specific reconciliation or a durable session/operator-visible failed-delivery signal as needed.

## Reproduction Assessment

Yes at source level, but not from a live Signal restart in this read-only review. A queued Signal outbound entry with `recoveryState: "send_attempt_started"` reaches recovery, Signal lacks `reconcileUnknownSend`, and current recovery refuses replay while only logging or queue-marking the failure.

## Solution Assessment

Unclear as a final product choice. The current fail-closed recovery avoids duplicate sends but does not satisfy the issue's user/session-visible recovery requirement; the better fix needs maintainer choice between adapter reconciliation, pending/backoff quarantine, explicit failed-delivery surfacing, or planned-restart draining.
