## Summary

Keep open. Current main still has the media auth gate that blocks plugin-only local/no-auth media providers, and this PR is a plausible active fix, but the external contributor has not provided after-fix real behavior proof from a real setup.

## What This Changes

The PR adds a `resolveSyntheticAuth` hook for media understanding providers, uses it as an audio/video missing-auth fallback after normal auth resolution, documents it, and adds local no-auth regression coverage.

## Best Possible Solution

Land this PR or an equivalent provider-owned media synthetic-auth hook after contributor real behavior proof is added and normal merge validation is green.

## Reproduction Assessment

Yes. By source inspection on current main, an explicit plugin-only media audio provider without credentials reaches `resolveApiKeyForProvider` and `requireApiKey` before `transcribeAudio` can run.

## Solution Assessment

Yes. The PR’s resolver-first fallback is the narrowest maintainable direction because literal, env, profile, and configured auth still take precedence while unmarked providers remain fail-closed.
