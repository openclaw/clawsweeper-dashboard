## Summary

## What This Changes

The PR changes the CLI plugin install planner and tests so scoped npm package specs can resolve matching bundled plugins before npm fallback.

## Best Possible Solution

Merge a refreshed patch that keeps exact bundled npmSpec lookup, points npm-bypass guidance at npm:<package>, leaves runtime duplicate precedence unchanged, and includes real after-fix CLI proof.

Yes, at source level: current main returns null for non-bare @openclaw/feishu before bundled lookup, while the bundled lookup already matches npmSpec and Feishu advertises @openclaw/feishu. I did not run the CLI flow in this read-only review.
