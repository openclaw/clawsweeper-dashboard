## Summary

Keep open: current main still bypasses bundled lookup for scoped specs, and the PR is a focused fix; the remaining blockers are proof quality and one minor UX copy issue.

## What This Changes

The PR changes the CLI install planner and tests so non-bare specs exact-match bundled plugin `npmSpec` metadata before falling through to npm.

## Best Possible Solution

Land the exact bundled `npmSpec` pre-npm lookup with the warning pointing to `npm:<package>` once real CLI proof and fresh targeted validation are available.

## Reproduction Assessment

Yes, at source level. Current main returns `null` for non-bare specs before any bundled lookup, while bundled source metadata can exact-match `@openclaw/feishu`.

## Solution Assessment

Yes, with one copy adjustment. Reusing the existing exact bundled `npmSpec` lookup is the narrow maintainable fix, but the bypass warning should direct npm-only installs to `npm:<package>`.
