## Summary

## What This Changes

The PR changes the CLI plugin install planner and tests so scoped npm specs can resolve exact matching bundled plugins before npm fallback.

## Best Possible Solution

Merge a refreshed patch that keeps exact bundled npmSpec lookup, points npm bypass guidance to npm:<package>, leaves runtime duplicate precedence unchanged, and includes redacted real CLI proof.

## Solution Assessment

Yes, mostly: reusing the existing exact npmSpec bundled lookup is the narrow maintainable fix without changing runtime precedence. The merge shape should also update the bypass warning to npm:<package> and include real CLI proof.
