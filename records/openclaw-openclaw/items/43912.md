## What This Changes

The PR updates CLI plugin install planning and helper tests so exact scoped npm specs can resolve to matching bundled plugin sources before npm fallback.

## Best Possible Solution

Resolve exact bundled npmSpec values before npm fallback, keep runtime duplicate precedence unchanged, update the bypass guidance to npm:<package>, and require redacted live CLI proof before landing.

## Reproduction Assessment

Yes, at source level. Current main returns null for scoped specs before bundled lookup, while the install command later falls through toward npm and bundled sources already expose exact npmSpec metadata such as @openclaw/feishu.

## Solution Assessment

Yes, with one copy fix. Reusing the existing exact npmSpec lookup is the narrowest maintainable fix, but the warning should direct users to npm:<package> for an explicit npm-only bypass.
