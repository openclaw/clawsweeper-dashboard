## Summary

Keep open: current main still skips bundled pre-planning for exact scoped npm specs, and the PR is a narrow source-level fix, but the external PR still needs real CLI behavior proof before merge and has one small warning-copy issue.

## What This Changes

The PR changes CLI plugin install planning so exact scoped npm specs can resolve to bundled plugin sources before npm fallback, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Resolve exact bundled npmSpec values before npm fallback, preserve runtime duplicate precedence, and point explicit registry bypass guidance at npm:<package>.

## Reproduction Assessment

Yes, at source level. Current main returns null for scoped specs before bundled lookup, while bundled sources already expose exact npmSpec metadata such as @openclaw/feishu.

## Solution Assessment

Yes, with one copy fix. Reusing the existing exact npmSpec lookup is the narrow maintainable fix, but the bypass warning should direct users to npm:<package>.
