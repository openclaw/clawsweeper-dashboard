## Summary

Keep open: current main still bypasses bundled pre-install planning for scoped specs, and this PR is the right narrow fix direction, but it still needs small warning-copy and changelog cleanup before merge.

## What This Changes

The PR updates CLI plugin install planning so scoped npm specs can match bundled plugins by exact npmSpec before npm fallback, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land or adapt the resolver change so exact bundled npmSpec matches win before registry/npm installation, unmatched scoped or versioned specs still fall through, runtime duplicate precedence remains unchanged, bypass guidance points users to npm:, and the user-facing fix is documented in the changelog.

## Reproduction Assessment

Yes, at source level. On current main, resolveBundledInstallPlanBeforeNpm returns null for @openclaw/voice-call before any bundled lookup, and the existing helper test asserts that scoped specs skip bundled pre-planning.

## Solution Assessment

Yes, with small cleanup. Reusing the existing exact npmSpec bundled lookup is the narrowest maintainable fix; the remaining changes are to make npm: the documented bypass and add the required changelog entry.
