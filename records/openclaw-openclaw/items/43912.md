## Summary

Keep open: current main still has the scoped-spec bundled lookup gap and this PR is a narrow useful fix, but the external-contributor real behavior proof gate is still missing and the npm-bypass warning needs a small correction.

## What This Changes

The PR changes CLI bundled-plugin pre-npm planning so scoped npm specs are looked up by exact bundled `npmSpec`, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Merge a refreshed patch that keeps the exact bundled `npmSpec` lookup, points npm-bypass guidance at `npm:<package>`, and leaves runtime duplicate precedence unchanged.

## Reproduction Assessment

Yes, at source level: current main returns `null` for scoped specs before bundled lookup, while bundled Feishu exposes `npmSpec: "@openclaw/feishu"`. I did not run the CLI flow in this read-only review.

## Solution Assessment

Yes, mostly: reusing the existing exact `npmSpec` bundled lookup is the narrow maintainable fix without changing runtime precedence. The merge shape should also update the bypass warning to `npm:<package>`.
