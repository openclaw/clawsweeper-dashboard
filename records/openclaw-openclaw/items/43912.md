## Summary

Keep this PR open: current main still bypasses bundled lookup for scoped npm specs, and the branch is a plausible narrow fix, but the external PR still needs real CLI behavior proof before merge and has one small warning-copy issue.

## What This Changes

The PR changes the CLI bundled plugin install planner and its tests so non-bare specs can exact-match bundled `npmSpec` metadata before generic npm fallback.

## Best Possible Solution

Use the existing exact bundled `npmSpec` lookup before npm fallback, update the warning to point registry-only bypasses to `npm:<package>`, then land after real CLI proof and fresh validation.

## Reproduction Assessment

Yes, at source level. Current main returns `null` for non-bare specs before bundled lookup, while bundled source metadata can exact-match `npmSpec` values such as `@openclaw/feishu`.

## Solution Assessment

Yes, with one copy adjustment. Reusing the existing exact `npmSpec` lookup is the narrow maintainable fix, but the bypass warning should point npm-only installs to `npm:<package>`.
