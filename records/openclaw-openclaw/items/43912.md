## Summary

Keep open: current main still bypasses bundled pre-install planning for scoped npm specs, and this PR is a narrow fix for that gap. It still needs contributor-supplied real behavior proof before merge, plus a minor warning-copy cleanup.

## What This Changes

The PR changes CLI bundled-plugin pre-install planning so non-bare specs are looked up by exact `npmSpec`, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land a refreshed version that keeps the exact bundled `npmSpec` lookup, points bypass guidance at `npm:<package>`, and includes redacted after-fix CLI proof plus focused validation.

## Reproduction Assessment

Yes, at source level: current main returns `null` for scoped specs before bundled lookup, while bundled Feishu exposes `npmSpec: "@openclaw/feishu"` and the existing test locks the bypass. I did not run the CLI flow in this read-only review.

## Solution Assessment

Yes, mostly: reusing the existing exact `npmSpec` bundled lookup is the narrow maintainable fix without changing runtime precedence. The merge shape should also update the bypass warning and include real after-fix CLI proof.
