## Summary

Keep open. Current main still bypasses bundled pre-install planning for scoped npm specs, and this PR is a focused fix for that behavior; the only concrete review issue I found is low-severity guidance text that should be updated before merge.

## What This Changes

The PR changes the CLI bundled install planner and tests so non-bare specs are matched by exact bundled npmSpec while bare specs continue to use pluginId lookup.

## Best Possible Solution

Land or adapt the resolver change so exact bundled npmSpec matches win before registry/npm installation, unmatched scoped or versioned specs still fall through, runtime duplicate precedence remains unchanged, and bundled-bypass guidance points users to the explicit npm: prefix.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the helper returns null for @openclaw/voice-call and the test suite asserts that scoped specs skip bundled pre-planning.

## Solution Assessment

Yes, with a small wording fix. Reusing the existing exact npmSpec bundled lookup is the narrowest maintainable implementation; the final patch should also update bypass guidance to npm:<package>.
