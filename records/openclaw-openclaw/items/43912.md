## Summary

This PR still carries a useful narrow fix for a source-reproducible CLI bug, but it should stay open for contributor-supplied real CLI proof before merge and one small warning-copy cleanup.

## What This Changes

The PR changes bundled plugin install planning and tests so exact scoped npm specs can resolve through bundled `npmSpec` metadata before generic npm fallback.

## Best Possible Solution

Use the existing exact bundled `npmSpec` lookup before npm fallback while keeping explicit `npm:` as the registry-only escape hatch.

## Reproduction Assessment

Yes, at source level. Current main returns `null` for non-bare specs before bundled lookup, while bundled source metadata can exact-match `npmSpec` values such as `@openclaw/feishu`.

## Solution Assessment

Yes, with one copy adjustment. Reusing the existing exact `npmSpec` lookup is the narrow maintainable fix, but the bypass warning should point npm-only installs to `npm:<package>`.
