## Summary

Keep open: current main still has the scoped-spec bundled install gap, and the PR is a narrow valid fix, but the external contributor still needs real CLI behavior proof before merge and the warning copy should point explicit npm bypasses at `npm:<package>`.

## What This Changes

The PR changes CLI bundled install planning so scoped npm specs can match bundled plugin `npmSpec` values before npm fallback and updates helper tests.

## Best Possible Solution

Land a refreshed version that resolves exact bundled `npmSpec` values before npm fallback, preserves runtime duplicate precedence, points explicit npm bypass guidance at `npm:<package>`, and includes redacted live CLI proof.

## Reproduction Assessment

Yes, at source level: current main returns `null` for non-bare specs in `resolveBundledInstallPlanBeforeNpm` before any bundled lookup, while the install command calls that helper before npm fallback. I did not run the CLI because this review was read-only.

## Solution Assessment

Yes, with a small warning-copy correction: reusing the existing exact `npmSpec` lookup is the narrow maintainable fix and keeps runtime duplicate precedence unchanged. The reliable bypass guidance should be `npm:<package>`, not a scoped package name.
