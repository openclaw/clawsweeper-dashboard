## Summary

Keep open: current main still skips scoped specs before bundled lookup, and the PR is a narrow fit with no blocking code findings, but this external PR still needs real CLI behavior proof before merge.

## What This Changes

The PR changes `resolveBundledInstallPlanBeforeNpm` so scoped install specs that exactly match a bundled plugin npm spec use the bundled install path, with tests for matching and non-matching scoped specs.

## Best Possible Solution

Merge the narrow helper and regression-test change after the contributor adds real CLI proof that a scoped bundled install resolves to the bundled plugin and the PR has fresh green checks.

## Reproduction Assessment

Yes from source, but not from an executed run: current main returns null before bundled lookup for non-bare specs, while the install command only takes the bundled path when that pre-plan returns a value. I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. Reusing the existing exact `npmSpec` bundled lookup in the pre-npm plan is narrower than changing runtime duplicate precedence or generic npm parsing, and the PR keeps unmatched scoped specs on npm.
