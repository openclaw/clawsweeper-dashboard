The PR changes CLI bundled plugin install planning so scoped npm specs can resolve matching bundled plugin metadata before npm install.

## Best Possible Solution

Land the exact bundled `npmSpec` pre-npm lookup, update the bypass warning to point to `npm:<package>`, then require redacted real CLI proof and fresh targeted validation before merge.

## Solution Assessment

Yes, with one copy adjustment. Reusing the existing exact bundled `npmSpec` lookup is the narrow maintainable fix, but npm-only bypass guidance should point to `npm:<package>`.
