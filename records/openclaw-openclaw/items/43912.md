## What This Changes

The branch changes the CLI bundled pre-NPM install planner so non-bare scoped specs use exact bundled `npmSpec` lookup and updates the focused unit tests.

## Best Possible Solution

Keep the exact `npmSpec` lookup approach, add warning text that points registry-intent users to `npm:<package>`, and require a redacted real CLI install transcript before merge.

## Reproduction Assessment

Yes at source level: current main returns `null` for non-bare `@openclaw/...` specs before bundled lookup, while bundled sources can already match exact `npmSpec`. I did not run the CLI because this review is read-only.

## Solution Assessment

Mostly yes: exact `npmSpec` lookup is the narrow maintainable fix and avoids changing runtime duplicate precedence. The merge-ready solution should also document the `npm:<package>` override in the warning and include real CLI proof.
