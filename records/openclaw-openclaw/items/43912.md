## Summary

Keep open. Current main still bypasses bundled pre-install planning for scoped specs, and this PR is a narrow source-level fix, but it still needs contributor real behavior proof and one warning-copy adjustment before merge.

## What This Changes

The PR changes the plugin install planner and tests so scoped npm specs exact-match bundled plugin `npmSpec` metadata before falling through to npm.

## Best Possible Solution

Use the existing exact bundled `npmSpec` lookup before npm fallback, revise the registry-bypass warning to point at `npm:<package>`, then land after real CLI proof and fresh validation.

## Reproduction Assessment

Yes, at source level. Current main returns `null` for non-bare specs before bundled lookup, while bundled source metadata can exact-match `npmSpec` values such as `@openclaw/feishu`.

## Solution Assessment

Yes, with one copy adjustment. Reusing the existing exact `npmSpec` lookup is the narrow maintainable fix, but the bypass warning should point npm-only installs to `npm:<package>`.
