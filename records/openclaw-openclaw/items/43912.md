## Summary

Keep open. Current main still bypasses scoped npm specs before bundled lookup, while this PR uses an existing exact npmSpec lookup and has a narrow plausible fix; the remaining merge blockers are a misleading bypass hint and a missing changelog entry.

## What This Changes

The PR changes CLI bundled install planning so non-bare specs can match bundled plugins by exact npmSpec before npm installation, and updates helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land or adapt the resolver change so exact bundled npmSpec matches win before registry/npm installation, unmatched scoped or versioned specs still fall through, runtime duplicate precedence remains unchanged, bypass guidance points users to npm:, and the user-facing fix is documented in the changelog.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: resolveBundledInstallPlanBeforeNpm returns null for @openclaw/voice-call, and the current helper test asserts that scoped specs skip bundled pre-planning.

## Solution Assessment

Yes, with small cleanup. Reusing the existing exact npmSpec bundled lookup is the narrowest maintainable fix; the branch should also update bypass guidance to npm: and add the required changelog entry.
