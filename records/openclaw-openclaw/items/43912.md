## Summary

Keep open: current main and v2026.5.20 still bypass bundled lookup for scoped `@openclaw/*` install specs, and this PR is a focused likely-correct fix. It is not merge-ready until the branch has real CLI behavior proof and the new source-selection warning points users to the `npm:` escape hatch.

## What This Changes

The branch changes the plugin install planner so scoped install specs can match bundled plugins by exact `npmSpec` before falling through to npm, with targeted unit tests.

## Best Possible Solution

Land a version that uses exact bundled `npmSpec` lookup for scoped specs, tells users to use `npm:<package>` for registry installs, and includes real CLI before/after proof.

## Reproduction Assessment

Yes, at source level: current main returns `null` for non-bare `@openclaw/feishu` before bundled lookup, while the Feishu package advertises that exact `npmSpec`. I did not run the CLI because this review is read-only.

## Solution Assessment

Mostly yes: reusing the existing exact bundled `npmSpec` lookup is the narrow maintainable fix without changing runtime duplicate precedence. The merge-ready version should add `npm:<package>` escape-hatch wording and real CLI proof.
