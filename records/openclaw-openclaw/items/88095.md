## Summary

Keep open: current main and the latest release still implement Voice Wake, push-to-talk, and Talk as transcript/wake-word pipelines with no enrolled-speaker profile or speaker-match gate. ## What This Changes

The issue requests speaker-recognition enrollment for Voice Wake, push-to-talk, and Talk mode so non-enrolled voices are ignored.

## Best Possible Solution

Define one speaker-gating contract for native voice input that covers enrollment UX, local state ownership, provider egress policy, fallback semantics, and plugin/API boundaries before implementation.
