## Summary

No blocking line-level defect found in the patch. Keep this PR open for maintainer review because it intentionally changes state-dir dotenv compatibility and auth-provider failure behavior from passing literal references through to omitting those keys.

## What This Changes

The PR adds a state-dir dotenv filter that skips whole-value unresolved shell references and adds focused Vitest coverage for skipped references, dollar-bearing credentials, and missing `.env` files.

## Best Possible Solution

Land the parser change only after maintainers accept the compatibility/auth tradeoff, keeping the narrow whole-value filter and focused regression tests.

## Reproduction Assessment

Yes, from source inspection and the linked report: current main reads `dotenv.parse()` output and stores every non-empty normalized value, while dotenv 17.4.2 parsing removes quotes but does not expand `$VAR` or `${VAR}`. I did not run a live launchd/systemd reproduction in this read-only review.

## Solution Assessment

Unclear as a product decision: the patch is a narrow maintainable way to stop persisting literal shell references, but maintainers must decide whether skipping rather than resolving is the desired upgrade behavior.
