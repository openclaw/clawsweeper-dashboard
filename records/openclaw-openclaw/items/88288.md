## Summary

Keep open for normal PR review: current main still accepts unresolved whole-value shell references from the state-dir `.env`, while this PR adds a focused filter plus after-fix install-plan proof. The code looks correct, but merge remains maintainer-sensitive because it changes credential behavior from persisted literal values to omitted service-env values.

## What This Changes

The PR adds state-dir `.env` parsing logic that skips whole-value shell references and adds focused Vitest coverage for skipped refs, preserved dollar-bearing credentials, and missing `.env` files.

## Best Possible Solution

Land the focused filter and regression coverage after maintainers accept omission as the intended upgrade behavior; if env references should be supported, that should be an explicit resolution or SecretRef design instead of preserving literals.

Yes, source inspection gives a high-confidence path: current main keeps every non-empty parsed dotenv value and the gateway install path propagates those values into generated service environments. I did not run a live service-manager repro in this read-only review.

## Solution Assessment

Yes, the PR is a narrow parser-level fix for the literal-reference bug and the new tests cover the key false-positive case. The remaining question is not code shape but whether maintainers want omission rather than reference resolution as the permanent behavior.
