The PR filters whole-value shell reference strings out of state-dir `.env` service environment parsing and adds Vitest coverage for unresolved references, preserved dollar-bearing credentials, normal values, and missing `.env` files.

## Best Possible Solution

Land the narrow parser/service-env filter with regression coverage after contributor-supplied patched-path proof and maintainer acceptance of the omit-rather-than-resolve compatibility behavior.
