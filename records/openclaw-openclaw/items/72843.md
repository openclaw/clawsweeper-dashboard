## Summary

Keep open. The reported bug is source-reproducible on current main, but this PR is not ready because the head still misses documented relative/container gateway launches and lacks observable after-fix real behavior proof.

## What This Changes

The PR changes agent context-window cache startup gating and tests so gateway daemon invocations can eager-warm model context metadata.

## Best Possible Solution

Recognize supported absolute, relative, and container `dist/index.js gateway` OpenClaw daemon entries while still rejecting arbitrary plugin-sdk `index.js` consumers, with regression coverage for both sides of that boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported `node .../dist/index.js gateway` path cannot eager-warm because the entry is unrecognized and `gateway` is skipped, while session/status paths synchronously fall back to 200k; a user comment also confirms the same symptom on OpenClaw 2026.5.3.

## Solution Assessment

No, not as submitted. Removing `gateway` from the skip list and guarding generic `index.js` consumers is the right direction, but the ancestor-only check still misses documented Compose, Podman, and Fly daemon entries.
