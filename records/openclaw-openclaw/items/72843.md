## Summary

Keep open: the underlying cold-cache gateway bug is valid, and the PR correctly moves toward warming `gateway`, but the latest branch still misses documented relative/container `dist/index.js` gateway launches and lacks the required changelog entry.

## What This Changes

The PR changes agent context-window cache startup gating to recognize selected `index.js`/`index.mjs` OpenClaw entry paths, stop skipping `gateway`, and add argv-focused unit coverage.

## Best Possible Solution

Update `shouldEagerWarmContextWindowCache` so supported OpenClaw gateway daemon entries, including relative `dist/index.js` and container paths, warm the cache while arbitrary plugin-sdk `index.js` scripts still do not, then record the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported daemon argv shape is rejected and `gateway` is skipped, while session/status code uses synchronous lookups that fall back to 200k when the cache is cold.

## Solution Assessment

No, not as-is. Removing `gateway` from the skip list is the right direction, but the PR's `openclaw`-ancestor guard misses supported container and relative daemon entries; a more precise entry predicate plus changelog coverage is needed.
