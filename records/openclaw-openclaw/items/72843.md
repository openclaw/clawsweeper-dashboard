## Summary

Keep this PR open: it addresses a real shipped/current bug, but the latest patch still misses supported relative `dist/index.js` gateway launches and lacks observable after-fix real behavior proof.

## What This Changes

The branch changes agent context-window cache eager-warmup gating and tests so gateway invocations using `dist/index.js` can load model context metadata at startup.

## Best Possible Solution

Recognize supported absolute, relative, and container `dist/index.js gateway` entries while preserving the plugin-sdk false-positive guard, then land with regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.7 shows the daemon warmup path is skipped and the session/status paths synchronously fall back to 200k; a user also confirmed the symptom on OpenClaw 2026.5.3.

## Solution Assessment

No, not as submitted. Removing `gateway` from the skip list and guarding generic `index.js` consumers is the right direction, but the ancestor-only check misses documented relative/container gateway launches.
