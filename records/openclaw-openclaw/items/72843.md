## Summary

Keep this PR open, but do not merge it as-is. It targets a real current-main gateway context-window warmup bug, but the revised `index.js` guard still misses supported container/cloud daemon launches and the user-facing fix needs a changelog entry.

## What This Changes

The PR changes agent context-cache startup gating so gateway invocations can eager-warm context-window metadata, with focused tests for the new argv recognition behavior.

## Best Possible Solution

Revise the PR so `shouldEagerWarmContextWindowCache` recognizes documented npm/global, dev-checkout, Docker/Compose, Podman, and Fly gateway daemon entries while preserving the generic plugin-sdk `index.js` false-positive guard, then add regression tests and a changelog entry.

## Reproduction Assessment

Yes. On current main, `node .../dist/index.js gateway` is rejected by the CLI-entry check and `gateway` is skipped; on the PR head, `node dist/index.js gateway` still returns false because it has no `openclaw` ancestor, while docs/templates show that exact launch shape.

## Solution Assessment

No, not as-is. Removing `gateway` from the skip list is appropriate and the generic `index.js` guard matters, but the ancestor-only gate is too narrow for supported packaged/container daemon entries.
