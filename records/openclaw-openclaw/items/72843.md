## Summary

Keep open. Current main still has the reported cold-cache gateway behavior, and this PR targets the right area, but the current branch is not merge-ready because the tightened `index.js` guard still misses supported container/Fly/Podman gateway launches that run from `/app/dist/index.js` or `dist/index.js`, and the user-facing fix lacks a changelog entry.

## What This Changes

The PR changes agent context-cache startup gating and adds tests so gateway invocations can eager-warm context-window metadata.

## Best Possible Solution

Revise the PR so `shouldEagerWarmContextWindowCache` recognizes real OpenClaw packaged `dist/index.js` daemon entries from npm/global installs, dev checkouts, Docker/Compose, Podman, and Fly while preserving the generic plugin-sdk `index.js` false-positive guard, then cover both accepted and rejected argv shapes in tests and add the release-note entry.

## Reproduction Assessment

Yes. A focused call to `shouldEagerWarmContextWindowCache` with supported gateway argv shapes reproduces both current main's miss and the PR head's remaining `/app/dist/index.js`/`dist/index.js` miss; the synchronous fallback path is visible in `getSessionDefaults` and `status-message.ts`.

## Solution Assessment

No, not as-is. Dropping `gateway` from the skip list is the right direction and the plugin-sdk guard is necessary, but requiring an exact `openclaw` ancestor is narrower than the documented packaged daemon launch contract.
