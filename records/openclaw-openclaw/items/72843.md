## Summary

Keep open: current main still has the reported cold-cache gateway bug, and the PR is aimed at the right code path, but the latest head is not merge-ready because its ancestor-only `index.js` check still rejects documented relative/container gateway launches and the user-facing fix lacks changelog coverage.

## What This Changes

The PR changes agent context-window eager-warmup gating to recognize OpenClaw `dist/index.js` gateway launches, remove `gateway` from the skip list, and add argv regression tests.

## Best Possible Solution

Revise `shouldEagerWarmContextWindowCache` so supported npm/global, dev-checkout, Docker/Compose, Podman, Fly, and packaged gateway daemon entries warm the cache while arbitrary plugin-sdk `index.js` scripts remain cold, then add regression coverage and a single-line changelog fix.

## Reproduction Assessment

Yes, from source inspection. Current main rejects `node .../dist/index.js gateway` before eager warmup and the session/status paths use synchronous context lookups that fall back to 200k when the cache is cold.

## Solution Assessment

No, not as-is. Removing `gateway` from the skip list and guarding generic `index.js` consumers is the right direction, but the ancestor-only match misses documented relative/container daemon entries and the PR needs a changelog entry.
