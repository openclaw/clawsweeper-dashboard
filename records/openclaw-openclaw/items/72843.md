## Summary

Keep open: the reported gateway context-window warmup bug is real on current main, but the PR head still misses supported container/Fly/Podman `node dist/index.js gateway` launch shapes and lacks the required changelog entry.

## What This Changes

The PR changes agent context-cache startup gating and tests so gateway invocations can eager-warm context-window metadata while guarding generic `index.js` plugin-sdk imports.

## Best Possible Solution

Revise `shouldEagerWarmContextWindowCache` to recognize documented npm/global, dev-checkout, Docker/Compose, Podman, and Fly gateway daemon entries while preserving the generic plugin-sdk false-positive guard, then add regression tests and a changelog entry.

## Reproduction Assessment

Yes. Current main rejects `node .../dist/index.js gateway` through both the CLI-entry check and the `gateway` skip, and PR head still rejects relative/container `dist/index.js` launch paths documented in the repo.

## Solution Assessment

No, not as-is. Removing `gateway` from the skip list and guarding `index.js` are the right direction, but the ancestor-only check is narrower than supported launch shapes; a command/path-aware allowlist with negative generic-index tests is safer.
