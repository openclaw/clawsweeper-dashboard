## Summary

Keep this PR open. The reported main-branch bug is real, but the PR head still misses documented container/cloud gateway daemon launches because its new `index.js` gate requires an exact `openclaw` ancestor, so `node dist/index.js gateway` and `/app/dist/index.js` continue to skip eager warmup.

## What This Changes

The PR changes agent context-cache startup gating so gateway invocations can eager-warm context-window metadata and adds focused tests for the new gating behavior.

## Best Possible Solution

Revise the PR so `shouldEagerWarmContextWindowCache` recognizes documented npm/global, dev-checkout, Docker/Compose, Podman, and Fly gateway daemon entries while preserving the generic plugin-sdk `index.js` false-positive guard, then add regression tests for both boundaries and a user-facing changelog entry.

## Reproduction Assessment

Yes. A focused check of `shouldEagerWarmContextWindowCache`/the PR-head ancestor predicate with `dist/index.js` or `/app/dist/index.js` gateway argv reproduces the remaining miss, and current docs/templates show those are supported daemon launch shapes.

## Solution Assessment

No, not as-is. Removing `gateway` from the skip list is appropriate and the generic `index.js` guard matters, but the ancestor-only fix is too narrow; a safer solution should recognize packaged OpenClaw `dist/index.js` from container/relative entries without matching arbitrary downstream `index.js` scripts.
