## Summary

Keep this PR open but do not merge it as-is. Current main still has the reported daemon warmup bug, and the PR fixes the right area, but the current head still misses supported container/cloud `node dist/index.js gateway` launch shapes and lacks the required changelog entry.

## What This Changes

The PR changes agent context-window eager-warmup gating and adds tests so gateway invocations can warm model context metadata at startup.

## Best Possible Solution

Revise the PR so `shouldEagerWarmContextWindowCache` recognizes documented npm/global, dev-checkout, Docker/Compose, Podman, Fly, GCP, and Hetzner gateway daemon entries while preserving the generic plugin-sdk `index.js` guard, then add regression tests and a changelog entry.

## Reproduction Assessment

Yes. On current main, `node .../dist/index.js gateway` is rejected by the CLI-entry check and `gateway` is skipped; on the PR head, relative `node dist/index.js gateway` and container `/app/dist/index.js gateway` launches still fail the new ancestor check.

## Solution Assessment

No, not as-is. Removing `gateway` from the skip list is appropriate, but the `index.js` recognition should key off supported OpenClaw daemon entry shapes without allowing arbitrary plugin-sdk consumer `index.js` scripts.
