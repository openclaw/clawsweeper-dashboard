## Summary

Keep open. Current main still lacks the requested native iOS Ask OpenClaw AppIntent/Shortcuts action with a typed message parameter and spoken dialog reply; existing deep-link and `/hooks/agent` paths are useful building blocks but do not expose this Siri flow.

## What This Changes

The issue requests native iOS Siri/AppIntent support so users can ask OpenClaw a question from Siri or Shortcuts and receive a spoken gateway reply.

## Best Possible Solution

Add a focused iOS AskOpenClaw AppIntent/AppShortcutsProvider that accepts a message, uses the approved paired-gateway trust path, returns a dialog/spoken result, and documents the Siri/Shortcuts behavior while coordinating with #40874.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence reproduction of the feature gap: there is no native Ask OpenClaw AppIntent/dialog implementation, and the existing deep-link and webhook paths do not return a Siri-spoken reply.

## Solution Assessment

No current implementation solves this yet. The direct `/hooks/agent` sketch is not enough by itself because that handler returns a run id, so the maintainable path should settle paired-gateway trust and spoken-response handling before wiring the AppIntent.
