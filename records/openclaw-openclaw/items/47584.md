## Summary

Keep open. Current main still lacks a native iOS Ask OpenClaw AppIntent/AppShortcutsProvider that accepts a Siri or Shortcuts question and returns a spoken dialog reply; the existing iOS deep-link and gateway hook paths dispatch nonblocking run work instead, and the related open PR only covers launching Talk Mode.

## What This Changes

Request native iOS Siri/Shortcuts AppIntent support for asking OpenClaw a dictated question and receiving a spoken reply.

## Best Possible Solution

Define a focused iOS Ask OpenClaw AppIntent/AppShortcutsProvider backed by an approved paired-gateway reply-capture contract, then document Siri/Shortcuts privacy behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested native AppIntent/ProvidesDialog flow.

## Solution Assessment

No current implementation solves this yet. The requested direction is plausible, but a direct /hooks/agent call is not sufficient without an approved wait/reply contract and Siri trust model.
