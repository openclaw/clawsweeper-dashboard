## Summary

Keep open. Current main still lacks the requested native iOS Ask OpenClaw AppIntent/Shortcuts action with a message parameter and spoken dialog reply; the existing deep-link and gateway event paths are useful building blocks, and open PR #40874 overlaps Action Button/AppIntent entry but does not close this typed Siri question/response flow.

## What This Changes

The issue requests a native iOS Siri/App Intents action that accepts a question, sends it to OpenClaw, and returns the reply through Siri/Shortcuts dialog.

## Best Possible Solution

Add a focused iOS AskOpenClaw AppIntent/AppShortcutsProvider that accepts a message parameter, uses the approved paired-gateway trust path, returns a dialog/spoken result, and documents the Siri/Shortcuts behavior.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction of the feature gap: no native Ask OpenClaw AppIntent/dialog implementation exists, App Intents metadata remains disabled, and only the deep-link gateway event path is wired.

## Solution Assessment
