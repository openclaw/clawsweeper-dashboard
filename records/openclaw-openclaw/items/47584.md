## Summary

Keep this open: current main and the latest release do not expose a native Siri/App Intent ask flow, and the related open iOS PR only opens Talk Mode rather than accepting a dictated question and returning a spoken answer. The request is aligned with companion-app direction, but it needs maintainer product and security design before implementation.

## What This Changes

The issue requests native iOS Siri/App Intent support so users can ask OpenClaw a dictated question and receive a spoken reply through the paired Gateway.

## Best Possible Solution

Define a narrow iOS-owned App Intent design that uses the paired Gateway chat/Talk response path with explicit consent, session targeting, and real-device Siri proof, then land it through the existing open iOS PR if it expands safely or through a fresh focused PR.

## Reproduction Assessment

Not applicable: this is a feature request rather than a bug report. Source inspection confirms the requested native App Intent surface is absent from current main and the latest release.

## Solution Assessment

No as written: posting to /hooks/agent is only a plausible trigger, not the best spoken-reply contract, because the current hook endpoint returns a runId asynchronously. The safer solution should reuse the paired Gateway chat/Talk flow or add an explicit response contract after product and security review.
