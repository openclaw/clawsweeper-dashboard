## Summary

Keep open. Current main has gateway hook, deep-link, Talk Mode, and AppIntents framework building blocks, but it still does not provide a native iOS Ask OpenClaw AppIntent/AppShortcutsProvider path with a message parameter and Siri/Shortcuts dialog result. Related PR #40874 overlaps Action Button/AppIntent work but remains open and is scoped to launching Talk Mode rather than a typed ask-and-spoken-reply intent.

## What This Changes

Add native iOS Siri/Shortcuts AppIntent support so users can ask OpenClaw a dictated or typed question, dispatch it to the paired Gateway, and receive a spoken/dialog reply.

## Best Possible Solution

Add a focused iOS AppIntent/AppShortcutsProvider surface in apps/ios that accepts a message parameter, uses the paired-gateway trust model or a maintainer-approved successor, dispatches through a narrow agent-request helper, returns an AppIntents dialog/value result for Siri and Shortcuts, and updates iOS docs/tests after simulator or device verification.

## Reproduction Assessment

Yes. Source search and project metadata on current main reproduce the feature gap with high confidence: no native Ask OpenClaw AppIntent exists, and App Intents metadata generation remains disabled. Runtime Siri testing is not needed to prove absence.

## Solution Assessment

No current implementation solves the request. The proposed AppIntent direction is plausible, but the best fix needs maintainer-approved trust, session, reply, and overlap handling rather than directly posting to /hooks/agent from a new Siri surface without that design.
