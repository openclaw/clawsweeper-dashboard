## Summary

Keep open. Current main still lacks the requested native iOS Ask OpenClaw Siri/AppIntent action with a typed message parameter and spoken dialog reply; existing AppIntents linkage, deep-link dispatch, voice wake, and Live Activity code are only building blocks, and the related open PR at https://github.com/openclaw/openclaw/pull/40874 overlaps Action Button/AppIntent entry but does not close this typed Siri question/response flow.

## What This Changes

Request native iOS Siri/AppIntent and Shortcuts support so users can ask OpenClaw a question and receive a spoken reply without opening the app.

## Best Possible Solution

Add a focused iOS Ask OpenClaw AppIntent/AppShortcutsProvider that accepts a message, uses the approved paired-gateway trust path, captures a reply suitable for spoken dialog, and documents Siri/Shortcuts privacy behavior.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: no native Ask OpenClaw AppIntent/dialog implementation exists, App Intents metadata remains disabled, and only deep-link/voice/gateway-event building blocks are present.

## Solution Assessment
