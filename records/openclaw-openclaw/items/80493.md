## Summary

Keep open: this PR has the protected `maintainer` label, is missing required after-fix real behavior proof, and the diff currently broadens `/btw` from a documented tool-less side query into a tool-capable Codex side thread without an explicit policy decision.

## What This Changes

The PR adds an optional agent-harness side-question hook, implements it for the Codex app-server harness by forking an ephemeral Codex thread for `/btw` or `/side`, updates Codex app-server protocol types, adds tests, and revises the BTW docs.

## Best Possible Solution

Keep the native Codex auth/thread transport direction, but preserve the documented `/btw` side-result isolation and no-tools policy unless maintainers explicitly approve a new side-thread tool model; non-Codex harnesses should retain a safe fallback or get a documented hook contract.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection on current main does show `/btw` resolving Codex OAuth and then using the generic provider-stream path, matching the linked 401 transport mismatch report.

## Solution Assessment

No, not as written. Native Codex side threads are a plausible transport fix, but this diff should not broaden `/btw` into a tool-capable, mutation-capable side run or break non-Codex harness fallback without an explicit maintainer policy decision.
