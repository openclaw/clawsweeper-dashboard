## Summary

Keep open. Current main still has a source-level path where a configured Google model id like `google/gemini-3.1-flash-lite-preview` can be preserved into native Gemini URL construction as `models/google/...`, and the two related fix PRs are closed unmerged.

## What This Changes

The issue asks OpenClaw to stop sending provider-prefixed Google Gemini model ids to the Google Generative AI API during sub-agent spawn or session reset.

## Best Possible Solution

Land one narrow Google-owned normalization fix that strips redundant `google/` before native Google request construction while preserving wrapper-provider nested ids and covering sub-agent/reset configured-model paths.

## Solution Assessment
