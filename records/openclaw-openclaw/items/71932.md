Keep open. Current main still has a credible Google native-transport path where a provider-prefixed inline Google model id is preserved and then sent as `models/google/<model>`, and the discussion now includes a corroborating Google provider config while overlapping fixes remain open.

## What This Changes

Fix Google sub-agent and session-reset model resolution so configured Gemini model ids like `google/gemini-3.1-flash-lite-preview` reach the Google Generative AI API as provider-local ids.

## Best Possible Solution

Land one scoped fix that strips only redundant active Google provider prefixes before Google alias lookup and native Google request URL construction, while preserving valid nested model ids for wrapper providers and adding regression coverage for sub-agent spawn and session reset.
