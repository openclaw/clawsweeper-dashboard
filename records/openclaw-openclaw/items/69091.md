## Summary

Keep this PR open. Current main still contains the three generic transport-stream throws that the PR targets, and the proposed diff is a narrow, behavior-focused fix with no definite correctness or supply-chain regression found in the reviewed patch.

## What This Changes

The PR changes three transport finalization throws to use `output.errorMessage || "An unknown error occurred"` and adds shared-helper unit tests for propagation and fallback behavior.

## Best Possible Solution

Land this PR or an equivalent focused patch for the shared/OpenAI Responses paths, preserve the generic fallback when no provider message exists, and either add a small Google transport companion fix or record that as a separate provider-owned follow-up if the Gemini repro remains in scope.

## Reproduction Assessment

Yes. A deterministic source-level repro exists on current main: finalize an output with `stopReason: "error"` and a concrete `errorMessage`; the current shared helper throws the generic string and the catch/failure path stores that generic message back into the stream error.

## Solution Assessment

Yes for the already-populated `output.errorMessage` paths: using the existing provider message with the current fallback is the narrowest maintainable fix. It is unclear whether it fully covers the Gemini production repro without also setting `output.errorMessage` in the Google provider-owned stream.
