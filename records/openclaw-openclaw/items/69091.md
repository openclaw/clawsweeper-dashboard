## Summary

Keep open. Current main still drops already-populated provider error messages in the shared and OpenAI Responses transport finalization paths, and this PR is a focused propagation/test patch with no discrete correctness or security finding found.

## What This Changes

The PR changes three agent transport finalization throws to prefer `output.errorMessage` over the generic fallback and adds shared-helper unit tests for propagation and fallback behavior.

## Best Possible Solution

Land this PR or an equivalent focused patch for already-populated `output.errorMessage` paths, and handle Google finishReason message population as an explicit follow-up if maintainers want the Gemini report fully covered.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared helper and both OpenAI Responses branches throw a generic message on `stopReason: "error"`, and their failure paths then store that generic string back into `output.errorMessage`.

## Solution Assessment

Yes for the shared/OpenAI paths that already have `output.errorMessage`; using that field with the existing fallback is the narrowest maintainable fix. It is not a complete Google finishReason fix unless that transport also starts populating `output.errorMessage`.
