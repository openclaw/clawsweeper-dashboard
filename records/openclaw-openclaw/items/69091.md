## Summary

Keep this PR open. Current main still has the three generic transport-stream throws targeted by the patch, and the proposed diff is narrow with no definite correctness or supply-chain regression found; the only substantive maintainer question is whether the Gemini/Google repro also needs a provider-owned companion change.

## What This Changes

The PR changes three agent transport finalization throws to prefer `output.errorMessage` over the generic fallback and adds shared-helper unit tests for propagation and fallback behavior.

## Best Possible Solution

Land this PR or an equivalent focused patch for the already-populated `output.errorMessage` paths, keep the generic fallback for missing messages, and make an explicit maintainer call on whether Google finish-reason message population belongs in this PR or a separate provider follow-up.

## Reproduction Assessment

Yes. A deterministic source-level repro exists on current main: finalize an output with `stopReason: "error"` and a concrete `errorMessage`; the current shared helper throws the generic string and the failure path stores that generic message back into the final error event.

## Solution Assessment

Yes for shared/OpenAI paths that already populate `output.errorMessage`; using that field in the throw is the narrowest maintainable fix. It is unclear whether this alone is the best complete solution for the Gemini repro because the current Google transport does not populate `output.errorMessage` before finalization.
