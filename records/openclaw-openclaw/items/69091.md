## Summary

Keep open. Current main still loses already-populated provider error text in the shared and OpenAI Responses transport finalization paths, and the PR is a narrow source-backed fix; the remaining merge blocker is the required user-facing changelog entry, with a follow-up question around Google finishReason message population.

## What This Changes

The PR changes three transport finalization throws to prefer `output.errorMessage` over the generic fallback and adds shared-helper tests for propagation and fallback behavior.

## Best Possible Solution

Land this focused propagation fix with the required changelog entry, then track Google finishReason message population separately if maintainers want the Gemini report fully covered.

## Reproduction Assessment

Yes. Source inspection on current main shows `finalizeTransportStream` and both OpenAI Responses branches throw a generic message on `stopReason: "error"`, and their failure paths then store that generic string back into `output.errorMessage`.

## Solution Assessment

Yes for paths that already have `output.errorMessage`: reusing that field with the existing generic fallback is the narrowest maintainable fix. The PR should add the required changelog entry and should not be treated as a complete Google finishReason fix unless that transport also starts populating the field.
