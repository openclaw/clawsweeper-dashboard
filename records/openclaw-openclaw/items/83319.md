This PR replaces the per-agent `agents.list[].subagents.model` inline schema with `AgentModelSchema` and adds a schema test accepting `{ primary, timeoutMs }`.

## Best Possible Solution

Yes, source-reproducible. Current main's per-agent subagent model schema rejects `timeoutMs`, while the default subagent path uses `AgentModelSchema` with `timeoutMs`; I did not run a parser in this read-only pass.

No. Reusing `AgentModelSchema` is mechanically narrow, but it bakes in `timeoutMs` support after a maintainer said that subagent timeout config should not exist and opened https://github.com/openclaw/openclaw/pull/83339 to reject it consistently.
