## Summary

Keep open for explicit maintainer handling: the patch is focused, but it now conflicts with the linked maintainer follow-up and open replacement PR that reject `timeoutMs` for subagent model config instead of allowing it per agent.

## Best Possible Solution

No. Reusing `AgentModelSchema` is mechanically narrow, but it is no longer the best path because the maintainer follow-up resolves the drift by keeping subagent model config to `primary` and `fallbacks` only.
