## Summary

Keep open. Current main does not implement this volume config surface, and the active PR has addressed the earlier reserved-target feedback, but it still has a source-reproducible sandbox bind validation defect, missing real Docker proof, and a live merge conflict.

## What This Changes

The PR adds optional `sandbox.docker.volumes` config for ephemeral, named, and bind mounts, wires those mounts into sandbox and browser Docker create args, rejects them on SSH/OpenShell backends, and adds docs, tests, and a changelog entry.

## Best Possible Solution

Land only a rebased implementation that validates structured bind volume settings directly, or rejects colon-bearing bind sources before both policy checks and Docker args, with redacted real Docker proof for all three strategies.

## Reproduction Assessment

Yes for the PR-head defect by source inspection: a bind source such as `/tmp/ws:/outside` is accepted by the new schema, converted to a legacy colon-delimited bind string for validation, and then emitted as the original Docker `--mount` source. Current main cannot reproduce the new feature behavior because `sandbox.docker.volumes` is not present there.

## Solution Assessment

No, not as currently implemented. The maintainable path is to validate the structured bind fields directly, or reject colon-bearing bind sources before both policy validation and Docker arg construction.
