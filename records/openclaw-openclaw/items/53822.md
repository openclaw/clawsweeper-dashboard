## Summary

Keep open. The PR is not implemented on main, and the latest head still has a concrete P2 regression: exact-head CI shows the new config-validation import path breaks doctor tests that mock `config/paths`.

## What This Changes

The PR adds optional `sandbox.docker.volumes` entries with `ephemeral`, `named`, and `bind` strategies, Docker/browser mount wiring, validation, docs, changelog, and SSH/OpenShell rejection tests.

## Best Possible Solution

Keep the feature on the PR branch, but move reserved-volume-target detection into a pure low-dependency helper shared by config validation and runtime sandbox validation, then rerun the sandbox/config/doctor checks.

## Reproduction Assessment

Yes. The exact-head CI annotations for `checks-node-agentic-commands-doctor` reproduce the failure without local mutation: config validation loads sandbox runtime security, which loads `config/paths` exports missing from existing doctor-test mocks.

## Solution Assessment

No, not as-is. The volume feature direction is reasonable and the earlier reserved-target/browser/non-Docker gaps are addressed, but config validation should not import the sandbox runtime module that depends on state-dir path constants.

## Review Findings
