## Summary

Keep open: the feature is not on current main, and the latest PR head still has a source-reproducible bind-volume validation bypass plus missing real Docker behavior proof. The earlier reserved-target and browser/SSH/OpenShell concerns appear addressed, but this head is not merge-ready.

## What This Changes

The PR adds optional `sandbox.docker.volumes` config for ephemeral, named, and bind-style Docker mounts, including schema/config merging, Docker and browser create-arg wiring, SSH/OpenShell rejection, docs, tests, and a changelog entry.

## Best Possible Solution

Keep the PR open, validate bind-strategy volumes without a lossy `source:target:mode` adapter or reject colon-bearing bind sources, add regression coverage, and require redacted real Docker output showing the new volume strategies after the fix.

## Reproduction Assessment

Yes. From source inspection, a bind volume source such as `/tmp/ws:/outside` passes the PR schema checks, is validated as `/tmp/ws` after colon-splitting, and is emitted to Docker as `source=/tmp/ws:/outside`.

## Solution Assessment

No. The feature direction is reasonable, but the current implementation should validate the structured bind source directly or reject colon-bearing bind sources before building Docker `--mount` arguments.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject colon-bearing bind volume sources:** `src/agents/sandbox/docker.ts:386`
  - body: Bind-strategy volumes are serialized as `source:target:mode` for validation, but Docker receives the original `--mount source`. A source like `/tmp/ws:/outside` is validated as `/tmp/ws` while Docker mounts `/tmp/ws:/outside`, so reject `:` in bind volume sources or validate the structured source directly.
  - confidence: 0.88
