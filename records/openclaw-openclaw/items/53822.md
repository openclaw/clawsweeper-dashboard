## Summary

Keep this PR open. Current main still has only bind-string Docker sandbox mounts, so the requested `sandbox.docker.volumes` surface is not implemented on main; the open branch is still useful but not merge-ready because it exposes the new config without matching the existing SSH/OpenShell unsupported-setting rejection path.

## What This Changes

The PR adds an additive `sandbox.docker.volumes` config with `ephemeral`, `named`, and `bind` strategies, schema/validation and merge support, Docker `--mount` wiring, browser sandbox mounting, generated config schema updates, and sandbox/config tests.

## Best Possible Solution

Keep the PR open as the implementation candidate, fix unsupported backend handling for `sandbox.docker.volumes`, add the missing user-facing docs/changelog if maintainers accept the feature, then rebase and validate the sandbox/config surface before merge.

## Reproduction Assessment

Yes. For the branch defect, configure the PR head with `backend: "ssh"` or OpenShell plus `sandbox.docker.volumes`; schema/type support now accepts the field, while `createSshSandboxBackend` only rejects `sandbox.docker.binds` and has no volume-mount implementation.

## Solution Assessment

No, not as-is. The additive Docker volume strategy is a reasonable direction for Docker sandboxes, but the patch should first fail closed for SSH/OpenShell and document the new config surface before it is the narrowest maintainable solution.
