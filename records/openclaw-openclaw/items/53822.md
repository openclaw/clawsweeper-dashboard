## Summary

Keep open. Current main does not already have this config surface, and the PR head still has a security-sensitive bind-source validation mismatch plus missing real Docker proof.

## What This Changes

The PR adds optional `sandbox.docker.volumes` config for ephemeral, named, and bind-style Docker mounts, with validation, Docker `--mount` wiring, backend rejections, docs, tests, and changelog.

## Best Possible Solution

Keep the PR open, validate structured bind volume sources directly or reject colon-bearing bind sources in schema and runtime, add a regression test, and require redacted real Docker proof before merge.

## Reproduction Assessment

Yes. Source inspection against PR head is enough: `/tmp/ws:/outside` becomes `/tmp/ws:/outside:/cache:rw` for validation, whose current parser reads host `/tmp/ws`, while Docker receives `source=/tmp/ws:/outside`.

## Solution Assessment

No. The feature direction is plausible, but the current implementation should not validate structured `--mount` bind sources through a colon-delimited legacy bind-string adapter.

## Review Findings
