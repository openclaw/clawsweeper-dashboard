## Summary

Keep open: the latest head adds the requested volume surface and addresses the earlier comma-injection/reserved-target review thread, but bind-strategy volume sources can still bypass allowed-root validation through colon-splitting, and the external PR still lacks real Docker behavior proof.

## What This Changes

The PR adds `sandbox.docker.volumes` for ephemeral, named, and bind-style Docker mounts, with config/schema support, Docker and browser create-arg wiring, SSH/OpenShell rejection, docs, tests, and a changelog entry.

## Best Possible Solution

Keep the PR open, validate bind-strategy volumes without a lossy `source:target:mode` adapter or reject colon-bearing bind sources, add regression coverage, and require redacted real Docker output showing the new volume strategies after the fix.

## Reproduction Assessment

Yes. From source inspection, a bind volume source such as `/tmp/ws:/outside` passes the PR's schema checks, is validated as `/tmp/ws` after colon-splitting, but is emitted to Docker as `/tmp/ws:/outside`.

## Solution Assessment

No. The feature direction is reasonable, but the current implementation should validate the structured bind source directly or reject colon-bearing bind sources before building Docker `--mount` arguments.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject colon-bearing bind volume sources:** `src/agents/sandbox/docker.ts:386`
  - body: `volumeSettingToBindSpec` reserializes declarative bind volumes as `source:target:mode` before calling `validateSandboxSecurity`. Because `splitSandboxBindSpec` treats the first colon as the host/target separator, a source like `/tmp/ws:/outside` is checked as `/tmp/ws` while Docker `--mount` uses `/tmp/ws:/outside`, bypassing the allowed source roots when that sibling path exists. Reject `:` for bind volume sources or validate the structured source directly before emitting `--mount`.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff adds a sandbox mount surface and still has a concrete bind-source validation bypass for colon-bearing paths.
