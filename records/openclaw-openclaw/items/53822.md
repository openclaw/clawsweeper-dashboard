## Summary

Keep this PR open: current main still lacks `sandbox.docker.volumes`, and the latest PR head has one blocking sandbox mount validation bug before it is safe to merge.

## What This Changes

This PR adds typed `sandbox.docker.volumes` config for ephemeral, named, and bind Docker mounts, wires it into sandbox/browser Docker create args, rejects volumes on SSH/OpenShell, and updates docs, changelog, generated schema, and tests.

## Best Possible Solution

Keep the additive volumes feature, but validate bind-strategy volume sources as structured paths, with regression coverage for delimiter-containing POSIX sources, before merge.

## Reproduction Assessment

Yes. A unit-level reproduction is to configure a bind volume whose absolute POSIX `source` contains `:`, then observe that validation colon-splits a synthetic bind string while `appendVolumeMounts` emits the full source to Docker `--mount`.

## Solution Assessment

No. The feature direction is maintainable, but the current validation adapter should be replaced with structured source/target validation or a deliberate delimiter rejection before emitting `--mount`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Validate bind-volume sources without colon-joining:** `src/agents/sandbox/docker.ts:386`
  - body: For `strategy: "bind"`, this formats the structured source/target back into a legacy `source:target:mode` string for validation, but the actual Docker call later uses `--mount` with the original `source`. A POSIX source containing `:` is accepted by the schema and mounted as written, while the validator only checks the prefix before the first colon, so source-root, symlink, and blocked-path checks can be applied to the wrong host path. Validate the structured source directly, or reject such delimiter cases before appending the mount.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete sandbox mount validation gap for bind-strategy Docker volumes.

Concerns:

- **[medium] Bind-volume source validation can check the wrong path:** `src/agents/sandbox/docker.ts:386`
  - body: Bind-strategy volume sources are validated after conversion to a colon-delimited `-v`-style string, but Docker receives the original structured `--mount` source. Colon-containing POSIX paths can therefore make validation inspect a different path than the one mounted.
