## Summary

Keep this PR open. Current main still has no `sandbox.docker.volumes` support, and the PR is a useful implementation candidate, but the patch remains incorrect because it makes volumes valid public config while SSH and OpenShell backends neither apply nor reject them.

## What This Changes

The PR adds `sandbox.docker.volumes` with `ephemeral`, `named`, and `bind` strategies, validation/merge support, Docker and browser `--mount` wiring, generated schema updates, and sandbox/config tests.

## Best Possible Solution

Keep the contributor PR open, fix it to fail closed for SSH and OpenShell volume configs unless those backends gain real support, add user-facing sandbox docs and a changelog entry if maintainers accept the feature, then run the targeted sandbox/config validation before secops review.

## Reproduction Assessment

Yes. On the PR head, configure `backend: "ssh"` or OpenShell with `sandbox.docker.volumes`; the new schema/type surface accepts the field, while the current SSH and OpenShell backend code only rejects `sandbox.docker.binds` and has no volume mount implementation.

## Solution Assessment

No, not as-is. Declarative Docker volumes are a reasonable Docker-backend direction, and the PR now addresses the prior reserved-target/browser gaps, but the narrow maintainable solution must reject unsupported non-Docker backends and document the public config surface before merge.
