## Summary

Keep open: current main does not have the requested Docker volume config surface, and this PR is still useful, but it is not merge-ready because real Docker behavior proof is missing and the PR head still has a bind-volume validation bypass in a sandbox security path.

## What This Changes

The PR adds optional `sandbox.docker.volumes` config for ephemeral, named, and bind Docker mounts, wires it into sandbox/browser Docker create args, rejects it on SSH/OpenShell backends, and adds docs, tests, and changelog coverage.

## Best Possible Solution

Land only a rebased implementation that validates structured bind volume settings directly, or forbids colon-bearing bind sources before both policy checks and Docker args, with redacted Docker runtime proof for all three strategies.

## Reproduction Assessment

Yes for the PR-head defect by source inspection: a bind source such as `/tmp/ws:/outside` is converted to legacy `source:target:mode` validation while Docker receives the original `--mount source`. Current main cannot reproduce the new feature behavior because `sandbox.docker.volumes` is not present there.

## Solution Assessment

No, not as currently implemented. The maintainable path is to validate structured bind volume settings directly, or reject colon-bearing bind sources before both policy validation and Docker `--mount` construction.
