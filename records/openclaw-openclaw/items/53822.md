## Summary

Keep open: current main does not implement `sandbox.docker.volumes`, and the PR remains a useful feature branch, but the head is not merge-safe because the new Docker `--mount` strings interpolate unescaped `source` and `target` values that can smuggle additional mount keys past sandbox validation.

## What This Changes

The PR adds optional `sandbox.docker.volumes` config for ephemeral, named, and bind-style Docker mounts with validation, config merging, Docker/browser create-arg wiring, SSH/OpenShell rejection, docs, tests, generated schema, and a changelog entry.

## Best Possible Solution

Keep the PR open, make `--mount` construction CSV-safe or reject mount-control characters before Docker args are emitted, and add regression tests for injected `type`, `source`, and `target` fields.

## Reproduction Assessment

Yes for the review finding from source inspection: a volume `source` like `cache,type=bind,source=/etc` or a `target` like `/cache,target=/workspace` is accepted by the new config validation but becomes extra Docker `--mount` fields. I did not run Docker locally in this read-only review.

## Solution Assessment

No, not as currently patched. The feature direction is reasonable, but the mount argument builder needs CSV-safe encoding or stricter validation before this is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Build --mount args without CSV key injection:** `src/agents/sandbox/docker.ts:406-416`
  - body: `appendVolumeMounts` interpolates `source` and `target` directly into Docker's comma-separated `--mount` spec. Docker parses that spec as CSV key/value fields and later repeated keys overwrite earlier ones, so a named volume source like `cache,type=bind,source=/etc` or a target like `/cache,target=/workspace` can bypass the new bind-source and reserved-target checks. Build these fields with CSV quoting/escaping or reject comma-bearing values before emitting `--mount`.
  - confidence: 0.87
