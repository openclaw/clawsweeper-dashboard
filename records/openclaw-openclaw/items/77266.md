## Summary

Keep open: current main has adjacent managed-npm recovery for explicit update/install paths, but the cold persisted registry can still hide recoverable external plugin packages after direct package-manager upgrades.

## What This Changes

The issue reports that externally installed WhatsApp and BlueBubbles plugins disappear from list/status surfaces after upgrading OpenClaw from 2026.5.2 to 2026.5.3-1 with `pnpm add -g openclaw@latest`.

## Best Possible Solution

Teach snapshot validation/rebuild paths to account for recovered managed-npm install records before trusting persisted registry state, with warnings/coverage rather than startup-time npm repair.

## Reproduction Assessment

Yes, source-reproducible: create a state dir with managed-npm plugin packages present but a persisted plugin index or install ledger that omits them; current snapshot code can return or derive from that stale persisted state and omit the plugins from list/status.

## Solution Assessment

No, current main is not the complete solution; the narrow maintainable fix is to merge or compare recovered managed-npm records during registry snapshot trust/rebuild, not to ask users to reinstall plugins or run npm during Gateway startup.

## Repair Work Prompt

Fix the external-plugin preservation regression reported at https://github.com/openclaw/openclaw/issues/77266. On current main, direct package-manager upgrades can leave a persisted plugin registry or install ledger that omits npm-managed external plugins still present under `~/.openclaw/npm`; `openclaw plugins list`, status, and startup metadata can then trust that stale state and hide those plugins. Update `src/plugins/plugin-registry-snapshot.ts` and related installed-index recovery code so recovered managed-npm install records are merged or compared before returning a persisted snapshot, and rebuild/diagnose when persisted registry state omits recoverable packages. Keep package-manager execution limited to explicit install/update/doctor flows; do not run npm during Gateway startup or read-only status/list commands. Add focused regression tests for empty and partial persisted ledgers plus a persisted snapshot that omits recoverable npm packages, and cover `plugins list`/status snapshot output. Add a user-facing `CHANGELOG.md` fix entry in the active fixes section, without guessing attribution.
