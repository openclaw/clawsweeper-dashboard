## What This Changes

The issue requests that externally installed npm plugins survive a direct OpenClaw package-manager upgrade from 2026.5.2 to 2026.5.3-1 instead of disappearing from plugin and channel status until reinstalled.

## Best Possible Solution

Make registry snapshot trust account for recoverable managed-npm install records after package-manager upgrades, and surface a warning when previously registered external plugins are missing or recovered.

## Reproduction Assessment

Yes, source-reproducible: a managed npm root can contain plugin package records while a current-looking persisted registry omits them, allowing the snapshot loader to return stale metadata before managed-npm recovery proves drift.

## Solution Assessment

No, the current adjacent fixes are not sufficient for this report; the safer narrow fix is registry/ledger recovery and diagnostics for managed-npm plugins, not running npm during startup or relying on users to reinstall plugins.
