## Summary

Keep open: current main has adjacent managed-npm recovery for update/runtime paths, but the persisted plugin registry snapshot can still hide recoverable external plugins from list/status-style snapshot consumers after a direct package-manager upgrade.

## What This Changes

Upgrading OpenClaw via a package manager should preserve externally installed WhatsApp and BlueBubbles plugins without requiring manual reinstall.

## Best Possible Solution

Merge recovered managed-npm install records into the plugin registry snapshot path before trusting persisted state, surface a repair warning when persisted state omits recoverable packages, and add focused regression coverage without running npm during Gateway startup.

## Reproduction Assessment

Yes, source-reproducible: a managed npm root can contain plugin packages while the persisted registry omits them, and current snapshot code can return or derive from that stale persisted state. I did not execute a live upgrade because this review is read-only.

## Solution Assessment

No, current main is only a partial solution for this report. The narrow maintainable fix is registry/ledger recovery and warning coverage for snapshot consumers, not package-manager execution during Gateway startup.
