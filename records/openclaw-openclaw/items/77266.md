## Summary

Keep open: current main still has a source-level path where a stale persisted plugin registry can hide recoverable managed-npm plugins, and the linked closing PR #77464 is open rather than merged.

## What This Changes

External plugins installed under OpenClaw's managed npm root should remain visible after direct package-manager upgrades instead of being hidden by stale persisted registry state.

## Best Possible Solution

Land a focused fix that makes registry snapshot, list, status, and startup recover managed-npm install records before trusting persisted state, then close this via the linked PR.

## Reproduction Assessment

Yes, source-reproducible: current main can return a persisted registry that omits a managed-npm plugin before loading recovered install records from the npm root. The documented manual package-manager upgrade path does not require reinstalling those plugins, and list/status consume that snapshot.

## Solution Assessment

Yes, the narrow repair direction in #77464 is the best maintainable path: invalidate or bypass stale persisted registry data and let existing managed-npm recovery records participate, without running npm during Gateway startup. The issue should stay open until that PR or an equivalent fix lands.
