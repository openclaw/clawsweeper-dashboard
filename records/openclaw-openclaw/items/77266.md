## What This Changes

The issue reports that externally installed WhatsApp and BlueBubbles plugins disappear from OpenClaw list/status surfaces after upgrading via a package manager from 2026.5.2 to 2026.5.3-1.

## Best Possible Solution

Make registry snapshot, plugin list/status, and startup discovery merge or invalidate against recovered managed-npm install records before trusting persisted state, while keeping package-manager execution confined to explicit install/update/doctor flows.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: packages can be recoverable under the managed npm root while `loadPluginRegistrySnapshotWithMetadata` returns or derives from a persisted registry that omits those records.

## Solution Assessment
