## What This Changes

Preserve externally installed npm plugins across direct OpenClaw package-manager upgrades so channels such as WhatsApp and BlueBubbles remain visible and loaded without manual reinstall.

## Best Possible Solution

Teach the plugin registry snapshot and inspection paths to account for recovered managed-npm install records before trusting persisted state, while keeping package-manager execution limited to explicit install/update/doctor flows.

## Reproduction Assessment

Yes, source-reproducible: a managed npm root can contain plugin packages while the persisted registry/install ledger omits them, and the snapshot path can return or derive from that stale persisted state without loading recovered records.

## Solution Assessment

No, current main is not the complete solution for this report; the safer fix is registry/ledger recovery plus warning/coverage, not running npm during Gateway startup or relying on users to reinstall plugins after package-manager upgrades.
