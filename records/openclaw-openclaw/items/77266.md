## Summary

Keep open: current main still has a source-reproducible stale plugin-registry snapshot path, and open PR #77464 is the active closing implementation that should be reviewed or landed before this issue closes.

## What This Changes

Externally installed managed-npm plugins disappear from list/status surfaces after a direct package-manager upgrade until each plugin is manually reinstalled.

## Best Possible Solution

Land a narrow registry snapshot recovery fix that invalidates persisted snapshots missing recoverable managed-npm packages, while keeping package-manager execution limited to explicit install/update/doctor flows.

## Reproduction Assessment

Yes, source-reproducible: a stale persisted plugin registry can omit a plugin that still exists in the managed npm root, and the current snapshot path can return or derive from that stale persisted state without loading recovered records.

## Solution Assessment

Yes for the linked PR direction, not for current main: registry/ledger recovery before trusting persisted state is the narrow maintainable fix, and running npm during Gateway startup would be a riskier alternative.
