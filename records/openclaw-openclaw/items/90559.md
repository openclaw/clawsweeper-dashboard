## Summary

The central failure is not fully resolved: current main adds an unreleased five-minute Feishu queue eviction, but it neither cancels the original run nor filters self-authored receive events, while the linked self-echo and session-lock repairs are still open.

## What This Changes

## Best Possible Solution

Keep this issue as the canonical compound incident, land the narrow Feishu self-authored-event filter and validated lock repair, then define and prove a generic cancellation/backpressure policy that prevents starvation without overlapping mutable session work.

## Solution Assessment

No, a single configurable queue timeout is not the best complete solution because eviction without cancellation can overlap same-session work; the connector filter, lock repair, and generic cancellation/backpressure semantics should be handled as ownership-correct changes with end-to-end proof.
