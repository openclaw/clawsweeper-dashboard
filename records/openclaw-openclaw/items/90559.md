## Summary

Keep open: current main and v2026.6.5 still lack the Feishu receive-path self-message filter, and the shipped per-chat queue timeout still evicts without aborting the original run. A narrow open PR covers the self-echo piece, but the queue/session-lock/backpressure scope still needs maintainer decision.

## What This Changes

The issue asks OpenClaw to stop Feishu slow-LLM incidents that combine same-chat queue starvation, non-aborting timeout/session locks, and bot self-echo feedback loops.

## Best Possible Solution

Land or reject the narrow Feishu self-open-id receive filter after maintainer review, then keep the broader queue abort, backpressure, collect/steer, and session-lock behavior tracked through the linked canonical items.

## Reproduction Assessment

Partly. Current source gives a high-confidence path for the missing self-message filter and non-aborting queue timeout, and a live v2026.6.5 comment reproduces queue/session-lock symptoms; the full self-talk loop has not been independently reproduced on current main.

## Solution Assessment

Unclear. The narrow self-open-id filter is the right first connector fix, but aborting timed-out runs, adding backpressure, or changing per-chat concurrency require maintainer decisions about Feishu delivery and session-state semantics.
