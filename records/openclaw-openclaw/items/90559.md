## What This Changes

Fix the Feishu slow-LLM incident by preventing same-chat queue starvation, self-authored message feedback loops, and session-lock/observability loss under long-running turns.

## Best Possible Solution

Keep this as the canonical Feishu slow-LLM incident until maintainers land or reject the narrow self-echo fix, choose the queue abort/backpressure boundary, and route the session-lock work through the linked session-state fixes.

## Reproduction Assessment

Yes for the source-level prerequisites, but not for the full live chain on current main. Current code still lacks a self-open-id receive filter and still evicts slow Feishu queue tasks without aborting them; the live follow-up reproduced the queue/session-lock symptoms on v2026.6.5 but not the self-talk echo.

## Solution Assessment

No single narrow implementation is proven best yet. The self-open-id PR is a good narrow mitigation, but the complete fix needs a maintainer decision across Feishu queue boundaries, abort/backpressure behavior, and session-lock policy.
