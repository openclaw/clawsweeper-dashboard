## Summary

Keep open: current main and v2026.6.5 still lack the Feishu receive-boundary self-message filter, and the new live comment strengthens the queue/session-lock part of the report rather than making it stale or implemented.

## What This Changes

The issue reports a Feishu DM failure mode where slow agent runs, per-chat queueing, self-echoed bot messages, and session write locks can combine into repeated replies and unusable sessions.

## Best Possible Solution

Land or reject the narrow self-open-id filter in https://github.com/openclaw/openclaw/pull/90572, then keep the queue/session-state pieces tied to https://github.com/openclaw/openclaw/issues/85692, https://github.com/openclaw/openclaw/issues/54409, and https://github.com/openclaw/openclaw/pull/88919 until maintainers choose the timeout, abort, and lock policy.

## Reproduction Assessment

Yes for part of the issue: current source and a live v2026.6.5 comment support the Feishu queue timeout and session-lock symptoms. The self-talk leg remains source-supported through the missing receive-boundary filter and open PR proof, but a natural self-echo was not captured in this review.

## Solution Assessment

No single proposed fix is sufficient. The open receive-filter PR is the best narrow solution for bot-open-id self echoes, while timeout abortion, queue-depth backpressure, collect/steer behavior, and session-lock reentry require maintainer scope decisions.
