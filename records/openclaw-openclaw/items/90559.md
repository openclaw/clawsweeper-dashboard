## Summary

Keep open: current main still lacks the Feishu receive-path self-message filter, while the shipped queue cap only evicts stuck tasks from the queue chain and does not abort the underlying run. The open self-filter PR is a useful partial fix, but the remaining queue/session-lock behavior is still split across open follow-ups and needs maintainer scoping.

## What This Changes

The issue asks OpenClaw to prevent Feishu slow-run queue starvation, bot self-message loops, and session lock observability failures under slow LLM responses.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90572 or an equivalent early Feishu self-filter, then resolve the remaining abort/backpressure/session-lock policy through the open queue and session follow-ups instead of queuing another broad repair PR from this umbrella issue.

## Reproduction Assessment

Partly: source inspection gives a clear path for the missing Feishu self-authored receive filter, and the newest comment gives live v2026.6.5 evidence for queue/session-lock symptoms. I did not establish a complete current-main live Feishu self-talk reproduction in this read-only review.

## Solution Assessment

No for the issue as one bundled repair: the self-filter is a narrow connector fix, but queue abort, backpressure, timeout configurability, and session-lock recovery are separate policy surfaces. The maintainable path is to land or replace the focused self-filter PR and keep the broader queue/session decisions on their canonical follow-ups.
