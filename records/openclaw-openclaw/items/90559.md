## Summary

Keep open. The shipped/current Feishu receive path now has a self-open-id echo filter, but the broader slow-run incident remains only partially addressed because same-key Feishu work still waits behind the full handler path and the timeout eviction explicitly does not abort the original agent task.

## What This Changes

The issue reports Feishu slow-run same-chat queue blocking, missing abort/backpressure, and bot self-echo feedback that can flood a DM and damage session observability.

## Best Possible Solution

Keep the shipped self-open-id filter, then land or replace a maintainer-approved Feishu queue/session repair that preserves ordering/control lanes, aborts or fences stale runs, and proves live Feishu slow-run behavior.

## Reproduction Assessment

## Solution Assessment
