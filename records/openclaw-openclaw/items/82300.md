## Summary

Keep open: current main receives edited Telegram updates, but implementation and tests only use them to refresh reply-chain cache, not to dispatch edited user instructions. This is a fresh UX feature that needs a maintainer decision on active-turn edit semantics, not a closeable duplicate or implemented item.

## What This Changes

The issue requests that Telegram `edited_message` updates replace pending user input or enqueue the edited content for the next agent turn.

## Best Possible Solution

Define explicit Telegram edit semantics, then implement them in the Telegram plugin and debounce layer with focused tests plus real Telegram proof for the visible chat behavior.

## Reproduction Assessment

Yes, by source inspection: send a normal Telegram message so it enters the inbound dispatch path, then edit it; the edited update reaches a handler that only refreshes reply-chain cache. I did not run the live Telegram reproduction in this read-only pass.

## Solution Assessment

No automated fix direction is settled yet. Replacing pending debounce entries is plausible, but handling edits after an agent turn has started is product behavior and should be decided before implementation.
