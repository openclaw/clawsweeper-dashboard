## Summary

Keep open. Current main still pads code-mode Markdown tables with code-unit length, while this draft PR targets the shared renderer used by Telegram and Discord table fallback; I found no blocking correctness issue in the patch.

## What This Changes

The PR adds display-width-aware padding for code-mode Markdown tables plus CJK, emoji, and symbol regression tests in the shared Markdown IR renderer.

## Best Possible Solution

Land this PR or a narrow replacement once it is out of draft, with the shared code-table display-width fix, regression tests, CI, and any maintainer-requested Discord proof; keep https://github.com/openclaw/openclaw/issues/55512 open until a fix merges.
