## Reproduction Assessment

Yes, source-level. A Markdown table rendered with `tableMode: "code"` reaches `renderTableAsCode` on current main, where column width and row padding still use `String.length`, which undercounts CJK display columns.

## Solution Assessment

Yes. Fixing display-width padding inside the shared `renderTableAsCode` path is the narrowest maintainable solution because Telegram code-table rendering and Discord table fallback both route through shared Markdown table utilities.
