## What This Changes

The branch adds display-width-aware padding for code-mode Markdown tables in `src/markdown/ir.ts` and adds CJK, emoji, and symbol regression tests.

## Best Possible Solution

Move the PR out of draft when ready, run focused Markdown and channel validation, then land this shared renderer fix or a small equivalent after normal review.

## Reproduction Assessment

Yes. On current `main`, a CJK Markdown table rendered with `markdownToIR(..., { tableMode: "code" })` reaches `renderTableAsCode`, where both width calculation and padding still use `.length`; Mantis also captured before/after Telegram Desktop proof for this PR.

## Solution Assessment

Yes. The patch changes the shared `renderTableAsCode` path used by Telegram and Discord table fallback; a future dedupe with terminal width helpers could be considered, but it is not a blocking correctness issue for this focused fix.
