## Summary

Keep open: current main and the latest release support extra Debian packages only as a build-time Docker/Podman image customization, while this issue asks for a remote-image bootstrap or derived-image mode. The follow-up comment correctly narrows the remaining decision to production-safe prebuild behavior versus risky entrypoint-time package installation.

## What This Changes

Add an opt-in Docker image mode or startup/prebuild path that installs deployer-selected APT packages while keeping the default image slim.

## Best Possible Solution

Define a maintainer-approved Docker extension-image contract: keep build-time package baking as the production path, add a clear derived-image/prebuild workflow if desired, and only allow entrypoint installation as an explicit convenience fallback with security docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection do show the current behavior with high confidence: extra packages are supported at build time only, while remote pulled images are just pulled and started.

## Solution Assessment

Unclear until maintainers choose the product/security boundary. The latest commenter’s prebuild/derived-image direction looks safer for production than entrypoint-time apt install because it avoids long gateway downtime and reduces startup privilege surprises.

## Risks / Open Questions

- A runtime APT bootstrap would need root privileges, network access, writable package-manager state, and explicit failure/caching behavior inside official images.
- Reusing OPENCLAW_IMAGE_APT_PACKAGES for startup-time work would change the semantics of a documented build-time variable and could surprise existing Docker/Podman users.
- The related Synology/macvlan report notes some hosts block apt-get inside containers, so entrypoint-time installation may fail exactly where the feature is meant to help.
- Installing packages during gateway startup can lengthen downtime on cold starts or image refreshes unless the production path builds a derived image before restart.
