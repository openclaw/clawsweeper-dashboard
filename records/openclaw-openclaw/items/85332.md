## Summary

Keep open: current main and the latest release support extra APT packages only as a local Docker/Podman image-build option, while this request is for a pulled-image bootstrap or derived-image workflow. The remaining work is a new Docker packaging feature with downtime and supply-chain tradeoffs, not an implemented behavior on main.

## What This Changes

Add an opt-in Docker image/bootstrap path that installs deployer-selected Debian packages for pulled or slim OpenClaw images without bloating the default image.

## Best Possible Solution

If maintainers want this, prefer an explicit prebuild or derived-image workflow from the current official image, with runtime entrypoint installation only as a documented dev fallback and with security, caching, failure, and downtime behavior spelled out.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source and docs inspection give high confidence that current main only supports build-time APT package customization, not pulled-image startup or prebuild bootstrap behavior.

## Solution Assessment

Unclear; the user problem is real, but entrypoint-time apt install is not clearly the best production shape because it can add downtime and security exposure. A derived-image prebuild path looks safer but needs maintainer product and security approval.

## Risks / Open Questions

- A startup-time installer could keep the gateway offline while apt resolves mirrors and installs packages, which the latest reporter comment explicitly calls out as a production boundary.
- Running deployer-selected APT package installation inside the official container startup path would need root or image-build authority, network access, writable package-manager state, and clear failure/caching semantics.
- Some target environments, including the related Synology/macvlan report at https://github.com/openclaw/openclaw/issues/58078, may block apt-get inside containers, so an entrypoint-only implementation may not solve the deployment gap.
