Issue asks for a slim official Docker image workflow that lets deployments configure extra APT packages without maintaining a custom local image.

## Best Possible Solution

Maintainers should choose an explicit derived-image or prebuild/restart contract, with docs and tests, while preserving the slim non-root default image and avoiding package installation during gateway downtime.
