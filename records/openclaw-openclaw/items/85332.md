## Summary

Keep open: current main and v2026.6.8 still only support deployer-selected APT packages during local image builds, while this issue asks for a remote-image startup or prebuild path that still needs Docker packaging, downtime, and security direction.

## What This Changes

The issue requests a slim Docker image or bootstrap mode that can install deployer-specified APT packages without baking them into the default image.

## Best Possible Solution

Maintainers should choose an explicit Docker packaging contract, likely a production-safe derived-image/prebuild path with clear docs, tests, and security review, while keeping the default image slim and non-root.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main and v2026.6.8 only support APT packages during local image build, not pulled-image startup or prebuild automation.

## Solution Assessment

Unclear. The requested capability addresses a real deployment ergonomics gap, but entrypoint-time APT installation is not the safest production default; maintainers should choose a derived-image prebuild command, documented Dockerfile pattern, or guarded dev-only bootstrap.
