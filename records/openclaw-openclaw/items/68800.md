## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible. `openclaw mcp serve` is documented as stdio, current startup/preflight runs before the command action, the `mcp` policy does not suppress stdout, and config preflight can emit warning notes to stdout.

## Solution Assessment

No, not as currently proposed. `routeLogsToStderr()` is the right mechanism as a defensive layer, but protocol stdout protection needs to happen before CLI startup/config preflight and be proven over raw stdio.
