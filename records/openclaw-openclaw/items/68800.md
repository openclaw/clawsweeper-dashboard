## Summary

Keep open: current main still has the reported stdout-isolation bug, but this PR is not merge-ready because its routing happens after a CLI config-warning path that can already write to stdout, and the branch carries broad changelog drift.

## What This Changes

The PR routes channel MCP server logs to stderr inside `serveOpenClawChannelMcp`, adds an in-memory channel MCP tools/list capability test, and adds a changelog entry.

## Best Possible Solution

Route MCP serve stdout isolation before any CLI startup/config preflight can emit notes, keep a focused stdio regression for `tools/list`, and reduce the changelog change to the single relevant fix entry.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: `mcp serve` is a stdio server, current main still reaches config/bootstrap before channel-server stderr routing, and config-warning notes flow through `process.stdout`. I did not run a live stdio client in this read-only review.

## Solution Assessment

No, not as currently placed. Calling `routeLogsToStderr()` is the right ingredient, but it needs to happen before CLI preAction/config preflight output, and the branch needs changelog cleanup plus inspectable real behavior proof.
