## Summary

Keep open: this is a focused bug-fix PR, but the proposed routing happens too late to protect `openclaw mcp serve` from CLI preAction/config-warning stdout output, and the PR still lacks inspectable real stdio proof.

## What This Changes

The PR calls `routeLogsToStderr()` inside `serveOpenClawChannelMcp`, adds an in-memory channel MCP `tools/list` capability test, and updates `CHANGELOG.md`.

## Best Possible Solution

Route MCP serve stdout isolation in the CLI startup path before config/bootstrap notes can write, then verify clean stdio and normal `tools/list` behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw mcp serve` is a stdio MCP server, CLI preAction/config preflight can run before the command action, and current main lacks mcp-specific stderr routing before that work.

## Solution Assessment

No, not as currently proposed. `routeLogsToStderr()` is the right ingredient, but calling it inside `serveOpenClawChannelMcp` misses earlier preAction/config output; the routing should happen before startup bootstrap and be backed by stdio proof.
