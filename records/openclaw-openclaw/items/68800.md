## Summary

Keep open: the linked bug remains valid and this PR is in the right general area, but the routing call is placed after CLI preAction/config preflight can already write to stdout, so it does not fully protect MCP stdio.

## What This Changes

The PR calls `routeLogsToStderr()` inside `serveOpenClawChannelMcp`, adds an in-memory `tools/list` capability test, and adds a changelog entry.

## Best Possible Solution

Route MCP serve stdout isolation in the CLI startup path before config/bootstrap preflight, keep the normal channel MCP `tools/list` behavior, and add real stdio proof.

## Reproduction Assessment

Yes, at source level: `openclaw mcp serve` is documented as a stdio MCP server, current main runs preAction/config bootstrap before the command action, and config preflight can emit `Config warnings` to stdout. I did not run a live stdio client in this read-only review.

## Solution Assessment

No, not as currently placed. `routeLogsToStderr()` is the right ingredient, but it needs to run before CLI preAction/config preflight output, and the added in-memory test does not prove stdio isolation.
