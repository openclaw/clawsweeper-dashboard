## Summary

Keep open: this PR targets a real MCP stdio bug, but the patch routes logs too late to protect CLI preAction/config-preflight output and still lacks inspectable after-fix raw stdio proof.

## What This Changes

The PR routes channel MCP server logs to stderr, adds an in-memory `tools/list` capability test, and adds a changelog entry for the MCP stdout fix.

## Best Possible Solution

Reserve stdout for `openclaw mcp serve` before CLI startup/config preflight, keep the server-level stderr routing as a defense if useful, and prove strict raw stdio `tools/list` behavior.

## Reproduction Assessment

Yes, at source level: `openclaw mcp serve` uses an SDK stdio transport that writes JSON-RPC to stdout, while current CLI preAction/config preflight can emit config warnings before the command action without MCP-specific stdout suppression. I did not run a live raw stdio session in this read-only review.

## Solution Assessment

No, not as currently proposed. `routeLogsToStderr()` is the right mechanism, but it must run before startup/config preflight output can reach stdout, not only inside `serveOpenClawChannelMcp`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
