## Summary

Keep open: the fix direction is right, but the added routing is still too late to protect CLI preAction/config-preflight output, and this external PR still lacks inspectable real stdio proof.

## What This Changes

The PR adds stderr routing to the channel MCP server startup, adds an in-memory channel MCP tools/list capability test, and adds a changelog entry.

## Best Possible Solution

Route MCP serve stdout isolation before any CLI bootstrap/config notes can write to stdout, preserve channel tools/list request behavior, and provide real stdio proof.

## Reproduction Assessment

Yes, at source level: openclaw mcp serve is a stdio command, Commander preAction runs config/bootstrap work before the command action, the mcp policy does not suppress that path, and config preflight can emit Config warnings to stdout. I did not run a live Hermes or raw stdio client in this read-only review.

## Solution Assessment

No, not as currently placed. routeLogsToStderr is the right ingredient, but putting it inside serveOpenClawChannelMcp only protects output after the action starts; the stdout-safe policy needs to run before CLI preAction/config preflight output.
