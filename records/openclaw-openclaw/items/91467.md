- `merge-risk: 🚨 availability`: Merging this can leave runaway cron jobs executing after the task CLI reports cancellation succeeded.

## Summary

Keep this PR open for repair: it fixes the current unsupported-runtime error only by clearing the task ledger, but it does not stop the active cron run and can make a runaway job appear cancelled while it continues executing.

## What This Changes

The PR changes task cancellation so cron runtime tasks bypass child-session termination like CLI tasks and can be marked cancelled in the task ledger.

## Best Possible Solution

Cron cancellation should either abort the active cron run through a cron-owned lifecycle hook before marking the task cancelled, or continue to report a clear unsupported/operator escape-hatch response without clearing the ledger row.

## Reproduction Assessment

Yes from source inspection: cron service creates task records with runtime `cron`, and current `cancelTaskById` returns the unsupported-runtime reason for non-CLI, non-ACP, non-subagent tasks with a child session.

## Solution Assessment

No: the proposed ledger-only bypass is not the best fix because it does not abort the cron run or update cron active/run state before marking the task terminal.
