## Summary

Keep open: the outbound scaffolding leak is fixed on current main and in the latest release, but the remaining group-source routing problem is still present because explicit model-supplied sender targets are preserved and same-provider target drift is allowed by default.

## What This Changes

This issue requests runtime hardening for weak-model message.send calls: sanitize outbound visible text and prevent group replies from being routed to the sender DM instead of the source chat.

## Best Possible Solution

Keep the shipped FM-3 sanitizer, then resolve FM-2 with a maintainer-approved group-source target policy backed by focused normalization, policy, and delivery tests.

## Reproduction Assessment

Yes, at source level for the residual FM-2 path: current normalization preserves explicit to/channelId values and default policy allows same-provider target drift. I did not run a live weak-model WhatsApp or Signal group reproduction in this read-only review.

## Solution Assessment

No, the current implementation is only a partial solution. The FM-3 sanitizer is the right shared-boundary fix, but FM-2 still needs maintainer policy on whether to reject, warn, auto-correct, or preserve explicit sender targets from group turns.
