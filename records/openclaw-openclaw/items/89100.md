## Summary

Keep open: the FM-3 outbound scaffolding leak is fixed on current main and in `v2026.6.8`, but the FM-2 sender-target routing path remains because explicit model-supplied `to`/`target` values are still honored for same-provider sends.

## What This Changes

The issue requests runtime hardening for `message.send` so outbound tool text cannot leak inbound scaffolding and group replies cannot drift from the inbound chat to the sender DM.

## Best Possible Solution

Keep the shipped FM-3 sanitizer, then resolve FM-2 with a maintainer-approved source-visible group-target policy backed by focused normalization, policy, and delivery tests.

## Reproduction Assessment

Yes, at source level for the residual FM-2 path: current normalization preserves explicit `to`/`channelId` values and same-provider policy allows target drift by default. I did not run a live weak-model WhatsApp/Signal group reproduction in this read-only review.

## Solution Assessment

No, the issue is not fully solved yet. The FM-3 sanitizer is the right shared-boundary fix and has shipped, but the remaining explicit-target guard needs maintainer policy on reject, warn, correct, or preserve behavior.
