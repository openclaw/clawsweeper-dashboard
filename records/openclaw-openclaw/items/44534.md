## Summary

Keep open: current `main` and the latest release still emit `target-none` for the default `target: "none"` heartbeat path. The related fix at https://github.com/openclaw/openclaw/pull/46870 was closed unmerged, and targeted search did not find an open PR that now owns this cleanup.

## What This Changes

Request clearer heartbeat skipped wording when the default delivery target is `none`, so disabled delivery is not confused with routing failure.

## Best Possible Solution

Make the target-none skip reason clearly configuration-driven, preferably by renaming it to `delivery-disabled` and updating focused expectations without changing heartbeat delivery behavior.

## Reproduction Assessment

Yes. Source inspection shows the default target-none path returns `reason: "target-none"`, and the existing tests assert that value; I did not run a live gateway heartbeat in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is a diagnostic rename or equivalent CLI rendering hint, with no changes to heartbeat scheduling, target resolution, or delivery defaults.
