## Summary

Keep open: current main still emits the misleading `target-none` heartbeat skip reason for the intentional default no-delivery path, and the only linked fix PR is closed unmerged.

## What This Changes

The issue requests clearer heartbeat skip diagnostics when default heartbeat delivery is intentionally disabled with `target: "none"`.

## Best Possible Solution

Update heartbeat delivery diagnostics so the intentional default `target: "none"` path reads as configuration-disabled delivery while preserving the existing no-delivery behavior and routing semantics.

## Reproduction Assessment

Yes. Source inspection against current main shows unset or explicit `target: "none"` resolves to `reason: "target-none"`, tests assert that value, and the runner forwards it into skipped heartbeat events.

## Solution Assessment

Yes. A focused diagnostic-only change in the target resolver, last-heartbeat presentation, and matching tests is the narrow maintainable path; additive metadata or a CLI hint is safer if reason-string compatibility matters.
