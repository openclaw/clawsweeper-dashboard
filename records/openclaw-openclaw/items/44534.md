## Summary

Current main still emits `target-none` for the default or explicit heartbeat `target: "none"` path and exposes that raw reason through the last-heartbeat surface. The only linked fix PR was closed unmerged, so this remains a narrow, actionable diagnostic cleanup rather than a close candidate.

## What This Changes

Request clearer heartbeat skipped output when delivery is disabled by default instead of reporting `target-none`.

## Best Possible Solution

Rename only the disabled-by-configuration skip reason to a clearer value such as `delivery-disabled`, update focused tests and changelog, and leave heartbeat scheduling and routing unchanged.
