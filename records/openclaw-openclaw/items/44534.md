## Repair Work Prompt

Fix the heartbeat skipped-delivery diagnostic for the default no-delivery configuration. Preserve the default behavior that `agents.defaults.heartbeat.target` unset or `none` does not deliver heartbeats, but stop surfacing the ambiguous `target-none` wording to `last-heartbeat` consumers and `openclaw system heartbeat last`; use either a clearer reason such as `delivery-disabled` or additive target metadata plus a CLI hint if compatibility is preferred. Update focused regression coverage for the resolver, runner, and CLI display path. Do not change heartbeat routing semantics, delivery target selection, channel behavior, or config defaults. Related refs: https://github.com/openclaw/openclaw/issues/44534 and closed unmerged https://github.com/openclaw/openclaw/pull/46870. Changelog is likely needed if the user-visible CLI/JSON diagnostic changes.
