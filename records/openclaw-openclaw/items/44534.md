## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/44534. Current main reports skipped heartbeat delivery for the default `agents.defaults.heartbeat.target: "none"` path as `reason: "target-none"`, which users read as a missing recipient instead of intentional config-disabled delivery. Keep the change narrow: update `resolveHeartbeatDeliveryTarget` and the heartbeat event/CLI surface so intentional target-none no-delivery is reported as configuration-disabled behavior, preferably `delivery-disabled` or additive `rawTarget`/`effectiveTarget` metadata plus a human-readable `openclaw system heartbeat last` hint if preserving reason-string compatibility matters. Update focused regression tests in the resolver/runner and CLI surfaces. Add a user-facing changelog entry. Do not change heartbeat scheduling, prompt execution, routing, directPolicy behavior, channel/plugin resolution, or the default `target: "none"` behavior.
