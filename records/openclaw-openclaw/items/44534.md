## Summary

Current main and the latest release still emit the ambiguous `target-none` reason without explanatory metadata. Both linked fixes closed unmerged for contributor queue or inactivity reasons, not because the problem was rejected; the issue should remain open for an owner decision on the compatibility-safe diagnostic contract.

## What This Changes

Clarify heartbeat skip diagnostics when the configured delivery target is `none`.

## Best Possible Solution

Preserve `target-none` for machine consumers, add an optional configuration-disabled explanation at the resolver/event boundary, and render it in human-facing heartbeat status without a misleading missing-channel placeholder.

## Reproduction Assessment

Yes. Current main deterministically routes unset or explicit `target: "none"` through the `target-none` skip branch, and the last-event payload exposes that raw reason without explanatory metadata.

## Solution Assessment

Yes for the compatibility-safe direction developed in https://github.com/openclaw/openclaw/pull/83279: preserve the stable reason and add a human-readable explanation; a direct reason rename is not the best solution.
