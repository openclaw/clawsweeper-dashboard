## Summary

Keep open. Current main still documents per-call `host=node` from `auto` as unconditionally allowed, while the runtime and tests reject node/gateway overrides when a sandbox runtime is active; this PR is the narrow docs-only correction for that mismatch.

## What This Changes

The PR updates one sentence each in `docs/tools/exec.md`, `docs/tools/exec-approvals.md`, and `docs/nodes/index.md` to state that `host=node` and `host=gateway` overrides from `auto` are only allowed when no sandbox runtime is active.

## Best Possible Solution

Land this docs-only PR, or equivalent wording on the same three pages, so the public docs match the shipped exec routing contract and the linked bug can close after merge.

## Reproduction Assessment

Yes. The mismatch is reproducible by comparing the current docs lines with `resolveExecTarget({ configuredTarget: "auto", requestedTarget: "node", sandboxAvailable: true })`, which current tests expect to throw.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable solution because current runtime behavior and regression tests already enforce the sandbox-active rejection rule.
