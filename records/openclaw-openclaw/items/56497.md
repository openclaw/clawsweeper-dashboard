## Summary

Keep open. Current main does not implement Volcengine Responses session caching, and this is a plausible provider-local feature, but the branch is not mergeable as written because it is behind the current pi package namespace and has cache invalidation, retry isolation, and documented-default mismatches; the opt-in vendor retention behavior also needs human product/privacy review.

## What This Changes

The PR adds an opt-in Volcengine Responses session-cache wrapper configured through `params.cache`, wires it via `wrapStreamFn`, and adds tests plus provider docs.

## Best Possible Solution

Land a provider-local Volcengine cache wrapper only after rebasing onto the current provider surface, switching to the current pi namespace, fixing cache invalidation/retry/default semantics, and documenting opt-in vendor retention clearly.

## Reproduction Assessment

Not applicable. This is a new opt-in provider feature PR rather than a current-main bug report, so review is based on the PR diff, current-main Volcengine behavior, upstream pi-ai source, and inspected screenshots.

## Solution Assessment

No, not as written. A provider-local wrapper is a plausible direction, but the branch needs namespace, prompt-invalidation, retry-isolation, documented-default, and retention-policy fixes before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new file imports `@mariozechner/pi-*`, but current main only declares and uses the `@earendil-works/pi-*` packages. After rebasing, the added module will not resolve until the imports are updated.
  - confidence: 0.97
- **[P1] Include prompt input in cache invalidation:** `extensions/volcengine/cache.ts:277`
  - body: The cache hash ignores the system/developer prompt because pi-ai puts it in `input`, not `instructions`. After a prompt or profile change, the wrapper can reuse the old `previous_response_id` and trim the updated prompt from the continuation request.
  - confidence: 0.91
- **[P2] Retry stale response IDs with an isolated stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry recurses with the same output stream. If the failed attempt emitted start, text, or tool events before the error, those events are delivered before the retry output.
  - confidence: 0.84
- **[P2] Make the thinking default match the docs:** `extensions/volcengine/cache.ts:111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans. Users who omit it do not get the documented payload behavior.
  - confidence: 0.86
