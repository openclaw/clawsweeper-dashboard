## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current Pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main declares only `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, while this new file imports the old `@mariozechner/*` packages. The PR will fail to resolve/typecheck on current main until the imports are updated and any plugin-local runtime dependency declaration is added.
  - confidence: 0.95
- **[P1] Invalidate continuations when prompt input changes:** `extensions/volcengine/cache.ts:276-278`
  - body: The cache hash only covers `target.tools` and `target.instructions`, but current Responses payloads carry system/developer prompts as `input` items. A prompt/profile change in the same session can reuse the old `previous_response_id`, trim the request to continuation input, and run under stale instructions.
  - confidence: 0.9
- **[P2] Retry stale response IDs without reusing output:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response branch recursively calls `pipeStream(output, false)` with the same output stream. If the failed attempt emitted any events before the error, consumers will see those events followed by retry events, so the retry should buffer until safe or use a fresh output path.
  - confidence: 0.86
- **[P3] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:110-111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper then omits `thinking`. Either send the documented default or document the actual request behavior.
  - confidence: 0.84
