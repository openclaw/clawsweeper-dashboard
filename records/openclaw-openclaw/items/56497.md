- **[P1] Rebase onto current stream contracts:** `extensions/volcengine/cache.ts:1-3`
  - body: The new wrapper imports `StreamFn`, model types, and the event-stream helper from `@mariozechner/*`, but current main exposes these contracts through OpenClaw packages and the Volcengine plugin only declares `@openclaw/plugin-sdk`.   - confidence: 0.94

  - body: `computeContentHash` ignores `target.input`, but current Responses payload construction puts the system/developer prompt in `input`. A prompt/profile change can therefore reuse an old `previous_response_id`, trim the request to trailing items, and continue under stale provider-side instructions.
  - confidence: 0.91
- **[P2] Disclose provider-side retention for cache mode:** `extensions/volcengine/cache.ts:319-330`
  - body: Cache mode unconditionally sets `store=true` and `expire_at`, which asks Volcengine to retain conversation context for reuse. The user-facing docs and security notes need to disclose that TTL/privacy boundary before users opt in.
  - confidence: 0.88
- **[P3] Align the documented thinking default:** `docs/providers/volcengine.md:72-73`
  - body: The docs say `thinking` defaults to true, but `resolveCacheConfig` leaves it undefined when omitted and the wrapper only sends `thinking` for explicit booleans. Either set that default in code or document that thinking is unchanged unless configured.
  - confidence: 0.83
