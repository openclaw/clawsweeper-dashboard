## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: After rebasing onto current main, this new file will not resolve because OpenClaw now depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, not the old `@mariozechner/*` package names. Update these imports before merge.
  - confidence: 0.97
- **[P1] Invalidate cached sessions when the system prompt changes:** `extensions/volcengine/cache.ts:277`
  - body: The cache hash only includes `target.tools` and `target.instructions`, but current pi-ai Responses payloads carry the system/developer prompt in `input`. With a stable session id, prompt/profile changes can reuse the old `previous_response_id` and then trim the updated prompt away on continuation turns.
  - confidence: 0.9
- **[P2] Avoid replaying partial events on retry:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the failed attempt emitted any start or content events before the invalid `previous_response_id` error, consumers will see those partial events followed by the retry output.
  - confidence: 0.78
