  - body: The cache hash only includes `target.tools` and `target.instructions`, while current Responses payloads carry system/developer prompts in `input`.   - confidence: 0.91

  - body: Enabling cache forces `store=true` and `expire_at`, which means the provider retains response state for continuation. The PR security section says data scope is unchanged and the docs omit this retention behavior, so users and maintainers cannot evaluate the privacy tradeoff.
  - confidence: 0.88
- **[P3] Align the documented thinking default:** `docs/providers/volcengine.md:72-73`
  - body: The docs say omitted `cache.thinking` falls back to `true`, but the implementation returns `thinking: undefined` when the field is not configured and only sends it when explicitly boolean. Users following the docs may not get the documented payload.
  - confidence: 0.78
