## Summary

Keep open: current main does not implement this, and the branch is useful but still has blocking provider-local correctness gaps that would make cache-enabled runs inconsistent before merge.

## What This Changes

The PR adds an opt-in Volcengine Ark Responses session-cache stream wrapper, provider wiring, unit tests, and docs for `params.cache` configuration.

## Best Possible Solution

Keep the feature provider-local and opt-in, cover both Volcengine provider ids, invalidate cached continuation state on all stable request inputs, isolate stale-response retries, document the `openai-responses` requirement, and add the required changelog entry before merge.

## Reproduction Assessment

Yes. Source inspection plus focused tests can show `volcengine-plan` misses the wrapper, prompt changes can reuse stale state, and a fake stream can emit partial events before a stale-response error to show mixed retry output.

## Solution Assessment

No. The provider-local wrapper is the right boundary, but this patch needs hook coverage, stricter invalidation, isolated retry behavior, and changelog coverage to be the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Register the cache wrapper for volcengine-plan:** `extensions/volcengine/index.ts:81`
  - body: `wrapStreamFn` is added only to the provider entry whose id is `volcengine`. Runtime hook lookup resolves by requested provider id plus aliases/hookAliases, and current Volcengine exposes `volcengine-plan` as a separate model provider, so `volcengine-plan/*` cache params will not run this wrapper. Add hook coverage and tests for both ids.
  - confidence: 0.88
- **[P1] Invalidate the cache when system input changes:** `extensions/volcengine/cache.ts:277-278`
  - body: The continuation hash misses system/developer input carried in the Responses payload. A same-session prompt or profile change can reuse the old response id, trim the request to continuation input, and drop the updated instruction. Include the effective system/developer input in invalidation.
  - confidence: 0.9
- **[P2] Retry stale response ids without reusing emitted output:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry writes the retry attempt into the same output stream used by the failed attempt. If any start or partial event was already emitted, consumers see failed-attempt events followed by retry events. Buffer until retry is ruled out or retry through a fresh output path.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `docs/providers/volcengine.md:38`
  - body: This is a user-facing provider feature and the PR does not update `CHANGELOG.md`. Add one single-line entry under the active Changes section, crediting `@stayrascal`.
  - confidence: 0.93
