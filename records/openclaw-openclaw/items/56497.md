## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main depends on `@earendil-works/pi-*` at `0.74.0`, but this new module imports `@mariozechner/pi-*`. Rebasing or merging the branch as-is leaves the cache wrapper with unresolved imports.
  - confidence: 0.97
- **[P1] Invalidate cache entries when Responses input changes:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `tools` and `instructions`, but pi builds the system/developer prompt into `input`. A prompt/profile change can therefore reuse an old `previous_response_id` and then trim away the updated prompt.
  - confidence: 0.92
- **[P2] Retry stale response IDs through a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the failed attempt already emitted `start` or partial events, consumers can see those events concatenated with the retry result.
  - confidence: 0.88
- **[P2] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:111-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans.
  - confidence: 0.86
