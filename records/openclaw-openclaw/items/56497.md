## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner/*` packages. The branch will not resolve after merge until these imports match the current dependency namespace.
  - confidence: 0.98
- **[P1] Hash the prompt before reusing response IDs:** `extensions/volcengine/cache.ts:277`
  - body: `computeContentHash` only sees `target.tools` and `target.instructions`, but the Responses payload carries the system/developer prompt in `target.input`. A prompt/profile change can therefore reuse an old `previous_response_id`, trim the updated prompt out of the continuation request, and run under stale instructions.
  - confidence: 0.91
- **[P2] Isolate retries from the failed stream output:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the first attempt emitted any start or content events before the invalid `previous_response_id` error, those events are delivered before the retry output.
  - confidence: 0.84
- **[P3] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:111`
  - body: The new docs say `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` when omitted, so the wrapper never sends `target.thinking` unless users configure it explicitly. Either set the documented default in code or change the docs to match the actual behavior.
  - confidence: 0.78
