  - confidence: 0.97
- **[P1] Invalidate cached sessions when the system prompt changes:** `extensions/volcengine/cache.ts:277`
  - body: The cache hash only includes `target.tools` and `target.instructions`, but current pi-ai places the system/developer prompt in the Responses `input` array. With a stable session id, a prompt/profile change can reuse the old `previous_response_id` and then trim the new prompt away on continuation turns.
  - confidence: 0.9
- **[P2] Avoid replaying partial events on retry:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry recurses into `pipeStream(output, false)` using the same output stream. If the failed attempt emitted any start/content events before the invalid `previous_response_id` error, consumers will see those partial events followed by the retry output.
  - confidence: 0.78
