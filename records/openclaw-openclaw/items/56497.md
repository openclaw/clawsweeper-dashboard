## Best Possible Solution

Land provider-local Volcengine cache support through the existing `wrapStreamFn` seam after owner approval, using the current dependency namespace, prompt-aware invalidation, fresh-stream retry behavior, and aligned docs/tests.

## Solution Assessment

No, not as written. A provider-local wrapper is the right boundary, but the current patch needs namespace/dependency cleanup, prompt-aware invalidation, retry isolation, documented default alignment, and owner acceptance of retention semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi dependency namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new module imports `@mariozechner/pi-*`, but current main and dependency metadata use `@earendil-works/pi-*`, and the Volcengine plugin package does not declare the old namespace. After merge, this file will not resolve in the current tree.
  - confidence: 0.97
- **[P1] Include the Responses input in cache invalidation:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `tools` and `instructions`. Current Responses payloads carry the normal system/developer prompt in `input`, so a session-keyed cache can reuse an old `previous_response_id` after prompt/profile changes and then trim away the updated prompt on continuation turns.
  - confidence: 0.9
- **[P2] Retry stale response IDs on a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry pipes the second attempt into the same output stream. Responses transports can emit `start` or partial events before the error, so the consumer can receive failed-attempt events before the retry result.
  - confidence: 0.88
- **[P2] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but the resolver returns `undefined` and the wrapper sends no `thinking` field unless the user sets it explicitly. Users following the docs will not get the documented payload behavior.
  - confidence: 0.86
