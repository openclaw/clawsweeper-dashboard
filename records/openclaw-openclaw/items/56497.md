## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Import the current Pi packages:** `extensions/volcengine/cache.ts:1-3`
  - body: This new runtime file imports `@mariozechner/pi-agent-core` and `@mariozechner/pi-ai`, but current OpenClaw pins the renamed `@earendil-works/pi-*` packages and the Volcengine package declares no `@mariozechner` runtime dependency. The extension build/tests will fail module resolution; switch to the current packages and declare the needed runtime deps in `extensions/volcengine/package.json`.
  - confidence: 0.96
- **[P2] Include the system prompt in cache invalidation:** `extensions/volcengine/cache.ts:276-278`
  - body: `buildOpenAIResponsesParams` carries the non-Codex system prompt in `input`, not `instructions`, so this hash stays unchanged when a session's prompt/profile changes. A cached continuation can then reuse `previous_response_id` and trim the new prompt away, leaving Volcengine on stale instructions; include the system/developer input item or equivalent cache-boundary content before reusing the response id.
  - confidence: 0.88
