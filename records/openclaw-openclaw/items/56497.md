## Summary

Keep open: current main still lacks Volcengine session-cache support, and this PR is a useful provider-local direction, but the current branch has blocking correctness issues plus a provider retention decision around opt-in `store: true`.

## What This Changes

The branch adds an opt-in Volcengine `openai-responses` session-cache wrapper, cache unit tests, and provider documentation for `params.cache`.

## Best Possible Solution

Land a provider-local Volcengine Responses cache wrapper only after rebase, current pi imports, full input-based invalidation, isolated stale-id retry, accurate defaults/docs/tests, and provider-owner approval of retention semantics.

## Reproduction Assessment

Not applicable for the requested feature itself: this adds opt-in Volcengine behavior rather than fixing a current-main bug. The review blockers are source-reproducible from the PR head, current main package declarations, and pi-ai `v0.74.0` Responses source.

## Solution Assessment

No, not as written. The provider-local wrapper is the right boundary, but the patch must be rebased, use current package contracts, invalidate against full Responses input, isolate retry output, align documented defaults, and get retention-semantics approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new module imports `@mariozechner/pi-*`, but current main declares and imports `@earendil-works/pi-*`; after rebase these imports will not resolve.
  - confidence: 0.97
- **[P1] Include Responses input in cache invalidation:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `target.tools` and `target.instructions`, while pi-ai carries the Responses system/developer prompt in `input`. A prompt or profile change can therefore reuse an old `previous_response_id` and trim away the updated prompt.
  - confidence: 0.92
- **[P2] Retry stale response IDs with a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream, so any `start` or partial events emitted before the error can be concatenated with the retry result.
  - confidence: 0.88
- **[P2] Align the documented thinking default:** `extensions/volcengine/cache.ts:111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans.
  - confidence: 0.86
