## Summary

Keep open: the Volcengine session-cache feature is not on current main and the branch still has blocking correctness problems around current pi package imports, prompt-sensitive cache invalidation, and stale-response retry streaming.

## What This Changes

Adds an opt-in Volcengine `openai-responses` session cache wrapper, provider wiring, unit tests, and docs for `params.cache`.

## Best Possible Solution

Keep the feature provider-local, but land it only after maintainer acceptance of the cache retention contract plus fixes for current imports, prompt-aware invalidation, retry stream isolation, and focused regression coverage.

## Reproduction Assessment

Not applicable. This is a provider feature PR rather than a current-main bug report, so I reviewed the PR diff, current main source, upstream pi source, review comments, and screenshot proof.

## Solution Assessment

No, not as currently written. The provider-local hook is plausible, but the patch needs current package imports, prompt-aware cache invalidation, retry isolation, and maintainer acceptance for `params.cache`/`store` semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, not the old `@mariozechner/*` packages, so this new file will not resolve after rebase/merge. Update these imports to the current namespace.
  - confidence: 0.97
- **[P1] Invalidate cached sessions when the system prompt changes:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash ignores the system/developer prompt because the Responses path carries it in `target.input`, not `target.instructions`. With a stable session id, a prompt/profile change can reuse the old `previous_response_id` and then trim the updated prompt out of the continuation request.
  - confidence: 0.9
- **[P2] Avoid replaying partial events on retry:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the failed attempt emitted start or content events before the invalid `previous_response_id` error, consumers will see those partial events followed by the retry output.
  - confidence: 0.78
