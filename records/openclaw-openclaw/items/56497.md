## Summary

Keep open. Current main still lacks Volcengine session-cache support, and this PR carries unique provider-local feature work, but the branch is not mergeable as written because it still has package-contract, cache-invalidation, retry-stream, and vendor-retention issues.

## What This Changes

The PR adds an opt-in Volcengine `openai-responses` session-cache wrapper, cache tests, and `params.cache` provider documentation.

## Best Possible Solution

Land a provider-local Volcengine Responses cache feature only after it uses current pi imports, invalidates on full Responses input changes, isolates retry output, aligns docs with defaults, and has explicit retention-semantics approval.

## Reproduction Assessment

Not applicable as a current-main bug; current main does not implement this opt-in feature. The merge blockers are source-reproducible from the PR head, current package declarations, and the pi v0.74.0 Responses source.

## Solution Assessment

No, not as written. The provider-local wrapper is the right boundary, but the implementation needs current dependency contracts, full input-based invalidation, isolated retry behavior, and explicit retention-semantics review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main depends on `@earendil-works/pi-*` 0.74.0, but this new module imports `@mariozechner/pi-*`. Those pi packages are not declared in the root package, so the added cache wrapper will fail to resolve after merge or rebase.
  - confidence: 0.97
- **[P1] Invalidate cached response IDs when input changes:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `tools` and `instructions`, but the Responses request carries the system/developer prompt in `input`. With a stable `sessionId`, changing the prompt can reuse the old `previous_response_id` and then trim away the updated prompt before sending.
  - confidence: 0.92
- **[P2] Retry stale response IDs through a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The retry path calls `pipeStream(output, false)` with the same output stream. If the failed attempt already emitted `start` or partial events before the stale-response error, those events remain and are concatenated with the retry output.
  - confidence: 0.88
- **[P2] Match the documented thinking default:** `extensions/volcengine/cache.ts:111-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` returns `thinking: undefined` and the wrapper only sends `thinking` for explicit booleans. Users following the docs do not get the documented default behavior.
  - confidence: 0.86
