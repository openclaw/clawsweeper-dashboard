## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the current Pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main declares the Pi packages under `@earendil-works/*`, but this new runtime file imports the old `@mariozechner/*` names. Those imports are not workspace dependencies, so the cache wrapper will fail to build or load after merging.
  - confidence: 0.97
- **[P1] Hash system input before reusing sessions:** `extensions/volcengine/cache.ts:276-278`
  - body: The content hash only covers `target.tools` and `target.instructions`, but Pi's Responses path places the system/developer prompt in `target.input`. A prompt/profile change in the same session can reuse the prior `previous_response_id`, trim the request to continuation input, and run under stale instructions.
  - confidence: 0.92
- **[P2] Retry stale response IDs through isolated output:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry recursively writes the second attempt into the same event stream. If the failed attempt emitted a start or partial event before the stale-ID error, consumers can receive failed-attempt events followed by retry events in one assistant response.
  - confidence: 0.86
- **[P2] Align the documented thinking default:** `extensions/volcengine/cache.ts:110-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` when explicitly configured. Users following the docs will not get the documented request shape.
  - confidence: 0.84
