## Summary

Keep open. Current main does not implement Volcengine session caching, and this PR is useful but still has blocking provider-local correctness issues before it can merge.

## What This Changes

The PR adds an opt-in Volcengine Ark Responses session-cache stream wrapper, tests, and provider/model docs for `params.cache`.

## Best Possible Solution

Land the provider-local cache only after both Volcengine provider ids are covered, continuation state invalidates on system/developer input changes, stale-response retry cannot leak failed-attempt events, docs disclose the opt-in cache/store behavior, and the changelog is updated.

## Reproduction Assessment

Yes. The remaining blockers are source-reproducible from the PR patch and current hook/runtime code: `volcengine-plan` lacks hook coverage, system prompt changes are not part of the cache hash, and retry reuses the same output stream.

## Solution Assessment

No. The provider-local wrapper is the right boundary, but the current patch needs hook coverage, stricter invalidation, isolated retry behavior, and changelog/docs coverage before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Register the cache wrapper for volcengine-plan:** `extensions/volcengine/index.ts:78`
  - body: `wrapStreamFn` is attached only through the `volcengine` provider entry. Hook lookup matches the requested provider id plus aliases/hookAliases, so `volcengine-plan/*` requests will not use this wrapper even when cache params are enabled.
  - confidence: 0.86
- **[P1] Invalidate cache when system input changes:** `extensions/volcengine/cache.ts:244-249`
  - body: The cache hash only covers `target.tools` and `target.instructions`, but the Responses transport carries `context.systemPrompt` in `target.input` as a system/developer item. A same-session prompt change can reuse `previous_response_id` and then continuation trimming drops the updated instruction.
  - confidence: 0.9
- **[P2] Retry through a fresh output stream:** `extensions/volcengine/cache.ts:313`
  - body: The stale-response retry recurses with the same output stream. If the failed attempt has already pushed a start or partial event, consumers receive failed-attempt events followed by retry events.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `docs/providers/volcengine.md:38`
  - body: This is a user-facing provider feature and the diff does not update `CHANGELOG.md`. Add a single-line entry in the active Changes section with contributor credit for `@stayrascal`.
  - confidence: 0.92
