## Summary

Keep open: this is a useful Volcengine provider feature that is not implemented on current main, but the branch is not correct against current main because it still imports the old pi package namespace and can reuse a stale session cache after system prompt changes.

## What This Changes

Adds a Volcengine session-cache stream wrapper for Ark Responses, with `previous_response_id` tracking, cache request fields, tests, and docs for `params.cache`.

## Best Possible Solution

Keep the feature provider-local, rebase it onto current main, use the current pi package namespace, and make cache invalidation account for system/developer prompt changes before merge.

## Reproduction Assessment

Not applicable. This is a feature PR rather than a bug report, and the contributor supplied after-fix screenshots/log output showing cache-read behavior instead of a current-main failure reproduction.

## Solution Assessment

No, not as currently written. The provider-local approach is plausible, but the branch needs current-main package imports and safer session invalidation before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main declares `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new module imports the old `@mariozechner/*` packages. After rebasing onto current main, this file will not resolve unless these imports are updated to the current package names.
  - confidence: 0.97
- **[P1] Invalidate the cache when the system prompt changes:** `extensions/volcengine/cache.ts:277`
  - body: With a real `sessionId`, the cache key ignores `context.systemPrompt`, and current pi-ai puts the system/developer prompt in the Responses `input` array rather than `target.instructions`. Hashing only `target.tools` and `target.instructions` lets a later prompt/profile/context-engine change reuse the old `previous_response_id` and then trim away the updated prompt on continuation turns.
  - confidence: 0.9
- **[P2] Avoid replaying partial events on retry:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry recurses into `pipeStream(output, false)` using the same output stream. If the first attempt emitted any start/content events before the invalid `previous_response_id` error, consumers will see those partial events followed by the retry's events, corrupting the visible stream.
  - confidence: 0.78
