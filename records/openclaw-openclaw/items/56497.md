## Best Possible Solution

Land a provider-local Volcengine Responses cache wrapper only after rebase, current pi imports, input-based invalidation, isolated retry output, accurate defaults/docs/tests, and explicit approval of retention semantics.

## Reproduction Assessment

Not applicable for the requested feature itself: this adds opt-in Volcengine behavior rather than fixing a current-main bug. The review blockers are source-reproducible from the PR head and current-main package/transport contracts.

## Solution Assessment

No, not as written. The provider-local wrapper is the right boundary, but it needs a rebase, current package contracts, full Responses-input invalidation, isolated stale-id retry, aligned defaults/docs, and retention-semantics approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main declares `@earendil-works/pi-*` packages only; this new module imports `@mariozechner/pi-*`, so rebasing the PR as-is leaves unresolved imports.
  - confidence: 0.97
- **[P1] Include Responses input in cache invalidation:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `tools` and `instructions`, but Responses carries the system/developer prompt in `input`; prompt/profile changes can reuse an old `previous_response_id` and trim away the updated prompt.
  - confidence: 0.92
- **[P2] Retry stale response IDs with a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream, so any `start` or partial events emitted before the error can be concatenated with the retry result.
  - confidence: 0.88
- **[P2] Align the documented thinking default:** `extensions/volcengine/cache.ts:111-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans.
  - confidence: 0.86
