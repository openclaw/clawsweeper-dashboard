## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new module imports `@mariozechner/pi-*`, but current main only declares and imports `@earendil-works/pi-*`. After rebasing, the added file will not resolve until these imports use the current package names.
  - confidence: 0.97
- **[P1] Include prompt input in cache invalidation:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `tools` and `instructions`, but the current Responses transport puts the system/developer prompt into `input`. A prompt/profile change in the same session can reuse the old `previous_response_id` and then trim the updated prompt from the continuation payload.
  - confidence: 0.91
- **[P2] Retry stale response IDs with an isolated stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the failed attempt emitted any start, text, or tool events before the error, those events are still delivered before the retry result.
  - confidence: 0.84
- **[P2] Make the thinking default match the docs:** `extensions/volcengine/cache.ts:111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans. Users who omit it do not get the documented payload behavior.
  - confidence: 0.86
