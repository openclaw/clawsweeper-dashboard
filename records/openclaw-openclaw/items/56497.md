## Summary

Keep open: this is a plausible provider-local Volcengine feature, but the branch is stale against current main and still has correctness blockers around dependency namespace, provider hook preservation, and session invalidation.

## What This Changes

The PR adds an opt-in Volcengine Responses session-cache wrapper, cache utility tests, provider wiring, and docs for `params.cache`.

## Best Possible Solution

Keep the feature provider-local and opt-in, rebase it onto the current Volcengine plugin surface, and land only after cache invalidation, retry isolation, docs/code defaults, focused tests, and live proof are fixed.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: the PR head imports undeclared package names, is based on an older Volcengine provider entry, and can miss system-prompt changes because the Responses transport carries the system prompt in `input`.

## Solution Assessment

No. A provider-local opt-in wrapper is the right boundary, but this patch needs to be rebased to current main and tightened before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the current Pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main declares the Pi runtime packages as `@earendil-works/*`, and the package graph guardrail treats old `@mariozechner/pi-*` names as unsupported. These new runtime imports are undeclared, so the Volcengine plugin will not build or load against current main until they use the current namespace.
  - confidence: 0.97
- **[P1] Rebase without dropping current Volcengine hooks:** `extensions/volcengine/index.ts:81`
  - body: The PR head adds `wrapStreamFn` to an older provider entry that lacks current main's `hookAliases: ["volcengine-plan"]`, `normalizeResolvedModel`, and speech-provider registration. Provider hook lookup uses those aliases, so `volcengine-plan/*` cache settings and existing compatibility behavior can be lost unless the rebase preserves the current fields.
  - confidence: 0.93
- **[P1] Invalidate sessions when system input changes:** `extensions/volcengine/cache.ts:276-278`
  - body: The invalidation hash only covers `target.tools` and `target.instructions`, but Pi's OpenAI Responses payload puts `context.systemPrompt` into `target.input` as a `developer` or `system` item. A prompt/profile change can therefore reuse an old `previous_response_id`, trim the request to continuation input, and run under stale instructions.
  - confidence: 0.91
- **[P2] Retry stale response IDs through isolated output:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same event stream. If the failed attempt emitted a start or partial event before the retry decision, consumers can receive failed-attempt events followed by retry events in one response.
  - confidence: 0.86
- **[P2] Align the documented thinking default:** `extensions/volcengine/cache.ts:110-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` when configured. Either make the code default match the docs or document the actual request shape.
  - confidence: 0.84
