## Summary

Keep open. Current main still lacks Volcengine Responses session-cache support, and this PR is useful but not mergeable as written because it is behind the current pi package namespace and still has cache invalidation, retry-stream, docs/default, and retention-policy issues.

## What This Changes

The PR adds an opt-in Volcengine Responses session-cache wrapper configured through `params.cache`, wires it into the provider, and adds tests plus provider docs.

## Best Possible Solution

Land a provider-local Volcengine cache wrapper only after owner review accepts the retention/config semantics and the branch is rebased with the namespace, invalidation, retry isolation, and docs/default issues fixed.

## Reproduction Assessment

Not applicable. This is a new opt-in provider feature PR rather than a current-main bug report; review is based on the PR diff, current-main provider surface, upstream pi-ai source, live PR state, and inspected screenshots.

## Solution Assessment

No, not as written. A provider-local wrapper is plausible, but the current diff must be rebased and corrected before it is the narrow maintainable solution, and vendor response retention needs owner/product acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new file imports `@mariozechner/pi-*`, but current main only declares `@earendil-works/pi-*`. After rebasing, the added module will not resolve until these imports use the current package names.
  - confidence: 0.97
- **[P1] Include prompt input in cache invalidation:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `tools` and `instructions`, but the Responses transport carries the system/developer prompt in `input`. A prompt/profile change in the same session can reuse an old `previous_response_id` and then trim the updated prompt from the continuation payload.
  - confidence: 0.91
- **[P2] Retry stale response IDs with an isolated stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the failed attempt emitted any start, text, or tool events before the error, those events are still delivered before the retry result.
  - confidence: 0.84
- **[P2] Make the thinking default match the docs:** `extensions/volcengine/cache.ts:111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans. Users who omit it do not get the documented payload behavior.
  - confidence: 0.86
