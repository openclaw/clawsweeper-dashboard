- **[P1] Port the wrapper to current SDK stream contracts:** `extensions/volcengine/cache.ts:1-3`
  - body: The new runtime file imports `@mariozechner/pi-agent-core` and `@mariozechner/pi-ai`, but current main no longer declares those packages and exposes stream contracts through OpenClaw's SDK/llm-core surfaces.   - confidence: 0.95
- **[P1] Invalidate cache when Responses input changes:** `extensions/volcengine/cache.ts:276-278`
  - body: The invalidation hash only covers `tools` and `instructions`, but current Responses payloads carry the normal system/developer prompt in `input`. A prompt or profile change can reuse the old `previous_response_id`, trim away the new input, and continue under stale instructions.
  - confidence: 0.9
- **[P2] Disclose retained Volcengine response state:** `extensions/volcengine/cache.ts:319-330`
  - body: When cache is enabled, the wrapper forces `store=true` and adds `expire_at`, retaining response state provider-side for the configured TTL. The PR security section and docs currently say data scope is unchanged, so users do not get the needed retention disclosure.
  - confidence: 0.88
- **[P3] Align the documented thinking default:** `docs/providers/volcengine.md:72-73`
  - body: The docs say omitted `thinking` falls back to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends a `thinking` object for explicit true or false. Either make the implementation default true or document that omission preserves provider behavior.
  - confidence: 0.82
