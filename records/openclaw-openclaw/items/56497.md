## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new module imports `@mariozechner/pi-*`, but current main only declares and uses `@earendil-works/pi-*`. The added file will not resolve in this workspace after merge.
  - confidence: 0.97
- **[P1] Invalidate cache when Responses input prompts change:** `extensions/volcengine/cache.ts:277`
  - body: The hash only covers `tools` and `instructions`, while the Responses path carries the system/developer prompt in `input`. A prompt or profile change can reuse the old `previous_response_id` and then trim the updated prompt from the continuation request.
  - confidence: 0.91
- **[P2] Retry stale response IDs with a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry recurses with the same `output` stream. If the failed attempt emitted any start/text/tool events before the error, those partial events are delivered before the retry result.
  - confidence: 0.84
- **[P2] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:110-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` stores `undefined` and the wrapper only sends `target.thinking` when the value is explicitly true or false.
  - confidence: 0.86
