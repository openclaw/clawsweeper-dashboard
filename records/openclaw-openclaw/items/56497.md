  - body: When cache is enabled, the wrapper forces `store=true` and adds `expire_at`, which tells Volcengine to retain response state provider-side for the configured TTL. The PR security section and docs currently say data scope is unchanged; update the contract and docs, or make retention an explicit accepted behavior before merge.
  - confidence: 0.88
- **[P3] Align the documented thinking default:** `docs/providers/volcengine.md:72-73`
  - body: The docs say `thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` when omitted and the wrapper only writes `target.thinking` for explicit true/false values. Either make the implementation default true or document that omitted `thinking` leaves the provider default untouched.
