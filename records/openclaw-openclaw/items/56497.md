## Summary

Keep open: the feature is useful and still absent from current main, but this PR is not merge-ready because it imports retired pi package names, can reuse a stale Responses session after system prompt changes, and can replay partial events during stale-response retry.

## What This Changes

The PR adds a Volcengine provider session-cache wrapper for Ark Responses, `params.cache` documentation, and focused cache utility tests.

## Best Possible Solution

Keep the feature provider-local, rebase it onto current main, update the pi imports, include system/developer input in cache invalidation, retry without replaying stale stream output, and land only after maintainer acceptance of the cache config contract.

## Reproduction Assessment

Not applicable. This is a feature PR rather than a bug report, and the available evidence is after-fix screenshot proof rather than a current-main failure reproduction.

## Solution Assessment

No, not as currently written. The provider-local direction is plausible, but the branch needs current-main imports, safer prompt-aware invalidation, and retry-stream isolation before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new module imports the old `@mariozechner/*` packages. After rebasing onto current main, these imports will not resolve.
  - confidence: 0.97
- **[P1] Invalidate the cache when system input changes:** `extensions/volcengine/cache.ts:277-278`
  - body: The invalidation hash only covers `target.tools` and `target.instructions`, while current Responses payloads carry the system/developer prompt in `input`. A prompt/profile change can reuse the old `previous_response_id` and then trim away the updated prompt on continuation turns.
  - confidence: 0.9
- **[P2] Retry stale responses without replaying partial events:** `extensions/volcengine/cache.ts:354`
  - body: The retry path calls `pipeStream(output, false)` with the same output stream. If the first attempt emitted any start/content events before the stale `previous_response_id` error, consumers will see those partial events followed by the retry output.
  - confidence: 0.78
