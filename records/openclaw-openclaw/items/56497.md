## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This file imports `@mariozechner/pi-*`, but current main only declares and imports `@earendil-works/pi-*`. The new cache module will fail to resolve in this tree until these imports are rebased to the current package names.
  - confidence: 0.97
- **[P1] Hash the full Responses input before continuation:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `target.tools` and `target.instructions`, while pi-ai Responses puts the system/developer prompt in `input`. A prompt/profile change can reuse the old `previous_response_id`, then continuation mode trims away the updated prompt and runs under stale instructions.
  - confidence: 0.91
- **[P2] Retry stale response IDs with a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the first attempt has already pushed `start` or partial events before the error, consumers can see failed-attempt events concatenated with the retry result.
  - confidence: 0.87
- **[P2] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans. Users following the documented default will not get the documented payload behavior.
  - confidence: 0.86
