Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai` 0.74.0, not `@mariozechner/*`. This new file will not resolve after merge until the imports are updated.
  - confidence: 0.97
- **[P1] Include the system prompt in the cache invalidation hash:** `extensions/volcengine/cache.ts:277`
  - body: The Responses transport carries the system/developer prompt in `target.input`, but this hash only checks `target.tools` and `target.instructions`. A prompt/profile change can reuse the old `previous_response_id` and then trim the updated prompt out of the continuation request.
  - confidence: 0.9
- **[P2] Return a fresh stream when retrying stale response IDs:** `extensions/volcengine/cache.ts:354`
  - body: The retry path recurses with the same output stream. If the failed attempt emitted start or content events before the invalid `previous_response_id` error, those partial events are delivered before the retry output.
  - confidence: 0.8
