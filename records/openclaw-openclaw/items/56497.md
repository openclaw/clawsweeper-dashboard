## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new module imports `@mariozechner/pi-*`, but current main only declares `@earendil-works/pi-*`. The extension will not resolve in this tree until these imports use the current package names.
  - confidence: 0.97
- **[P1] Hash the Responses input when invalidating cache:** `extensions/volcengine/cache.ts:277`
  - body: The invalidation hash only covers `tools` and `instructions`, while current non-Codex Responses payloads carry the system/developer prompt in `input`. A prompt/profile change can reuse the old `previous_response_id` and then continuation mode trims away the updated prompt.
  - confidence: 0.91
- **[P2] Retry stale response IDs with a fresh output stream:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. Since the base Responses transport can emit `start` or partial events before an error, failed-attempt events can be delivered before the retry result.
  - confidence: 0.87
- **[P2] Make the thinking default match the docs:** `extensions/volcengine/cache.ts:111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans. Users following the documented default will not get the documented payload behavior.
  - confidence: 0.86
