Full review comments:

- **[P1] Rebase onto current stream packages:** `extensions/volcengine/cache.ts:1-3`
  - body: The new wrapper imports `StreamFn`, `Model`, and `createAssistantMessageEventStream` from the old `@mariozechner/*` packages, but current main has no matching dependencies and exposes these contracts through OpenClaw packages. As-is, the added file will not resolve against current main after merge.
  - confidence: 0.92
- **[P1] Invalidate cache when the input prompt changes:** `extensions/volcengine/cache.ts:276-282`
  - body: `computeContentHash` ignores `target.input`, but the current Responses path puts the system/developer prompt into that input array. If a session prompt changes without shrinking the input length, the wrapper reuses `previous_response_id`, filters the request to trailing user/tool items, and Volcengine continues under stale provider-side instructions.
  - confidence: 0.9
- **[P2] Disclose provider-side retention for cache mode:** `extensions/volcengine/cache.ts:319-330`
  - body: Cache mode sets `store = true` and `expire_at`, so enabling this feature asks Volcengine to retain conversation state server-side. The new docs and PR security section currently frame the feature as no data-scope change, which leaves users without the privacy/retention warning they need before opting in.
  - confidence: 0.86
- **[P3] Align the documented thinking default with the wrapper:** `docs/providers/volcengine.md:72-73`
  - body: The docs say `thinking` defaults to `true`, but `resolveCacheConfig` leaves it undefined when omitted and the wrapper only sends a thinking body for explicit boolean values. Either set the default in code or document that thinking is unchanged unless configured.
  - confidence: 0.82
