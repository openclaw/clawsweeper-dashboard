## Summary

Keep open: current main does not implement Volcengine Responses session caching, and this useful provider feature PR still has blocking build/correctness issues plus a maintainer product/privacy decision around opt-in vendor retention.

## What This Changes

Adds an opt-in Volcengine Responses session-cache wrapper configured through `params.cache`, wires it through provider `wrapStreamFn`, and adds tests and docs for the cache behavior.

## Best Possible Solution

Land a provider-local Volcengine cache wrapper only after rebasing onto the current provider surface, switching to the current pi package namespace, fixing cache invalidation/retry/default semantics, and documenting opt-in vendor retention clearly.

## Reproduction Assessment

Not applicable. This is a new provider feature PR rather than a current-main bug report, so review is based on the PR diff, current-main Volcengine behavior, dependency source, and supplied screenshots.

## Solution Assessment

No, not as currently written. A provider-local wrapper is plausible, but the branch needs namespace, prompt-invalidation, retry-isolation, docs/default, and retention-policy fixes before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This new file imports `@mariozechner/pi-agent-core` and `@mariozechner/pi-ai`, but current main only declares the `@earendil-works/*` pi packages. As written, the added module will not resolve in this workspace after merge.
  - confidence: 0.97
- **[P1] Invalidate cached responses when input prompts change:** `extensions/volcengine/cache.ts:277`
  - body: Responses carries the system/developer prompt in `input`, but this hash only covers `tools` and `instructions`. After a prompt or profile change, the wrapper can reuse the old `previous_response_id` and trim the updated prompt from the continuation request, leaving the next turn under stale instructions.
  - confidence: 0.91
- **[P2] Retry stale response IDs with a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The retry path calls `pipeStream(output, false)` with the same output stream. If the failed attempt emitted start, text, or tool events before the invalid-response error, those partial events are delivered before the retry output.
  - confidence: 0.84
- **[P2] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:110-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `target.thinking` when the value is explicitly true or false. Users following the documented default will not get the documented payload behavior.
  - confidence: 0.86
