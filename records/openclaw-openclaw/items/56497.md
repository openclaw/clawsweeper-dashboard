Full review comments:

- **[P1] Use the current Pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner/*` packages. Since those packages are not declared in the workspace, the PR will fail to build/typecheck on current main until the imports are updated.
  - confidence: 0.95
- **[P1] Preserve hook coverage for volcengine-plan:** `extensions/volcengine/index.ts:81`
  - body: The PR head adds `wrapStreamFn` to the `volcengine` provider entry but lacks current main's `hookAliases: ["volcengine-plan"]`. Hook lookup resolves by provider id/aliases, so `volcengine-plan/*` cache settings will miss the wrapper unless the PR is rebased while preserving the alias or otherwise registers hook coverage for both provider ids.
  - confidence: 0.9
- **[P1] Invalidate cache when system input changes:** `extensions/volcengine/cache.ts:277-278`
  - body: The invalidation hash only covers `target.tools` and `target.instructions`, but the Responses transport carries the system/developer prompt as an `input` item. A prompt or profile change in the same session can therefore reuse the old response id, trim the payload to continuation input, and run under stale instructions.
  - confidence: 0.9
- **[P2] Retry stale response ids without reusing emitted output:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response branch retries by calling `pipeStream(output, false)` with the same output stream. If the failed attempt emitted any start or partial events before the error, consumers will see those events followed by retry events; buffer until retry is ruled out or retry through a fresh output path.
  - confidence: 0.86
- **[P2] Align the thinking default with the documented config:** `extensions/volcengine/cache.ts:110-111`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` when it is explicitly `true` or `false`. Either make the code default match the docs or document the real default so users do not get a different request shape than promised.
  - confidence: 0.84
