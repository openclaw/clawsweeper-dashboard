## Summary

Keep open. Current main still has no Volcengine session-cache wrapper, and the PR is useful but not merge-ready because the proposed wrapper misses the `volcengine-plan` hook path and has continuation/retry correctness blockers.

## What This Changes

The PR adds an opt-in Volcengine Ark Responses session-cache stream wrapper, unit tests, and provider/model docs for `params.cache` configuration.

## Best Possible Solution

Keep the implementation provider-local, cover both Volcengine provider ids, invalidate continuation state when system/developer input changes, retry stale response ids without exposing failed-attempt events, document the opt-in provider-side cache/store behavior, and add the required changelog entry before merge.

## Reproduction Assessment

Yes. A focused test can resolve/wrap `volcengine-plan` and observe the wrapper is absent; another can reuse a session id after changing `context.systemPrompt`; a fake stream can emit a partial event before a stale-response error to show mixed retry output.

## Solution Assessment

No. The provider-local wrapper is the right boundary, but the current patch needs hook coverage, stricter invalidation, isolated retry behavior, and changelog coverage to be the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Register the cache wrapper for volcengine-plan:** `extensions/volcengine/index.ts:81`
  - body: `wrapStreamFn` is attached only to the `volcengine` provider entry. Runtime hook lookup matches the requested provider id plus `aliases`/`hookAliases`, so `volcengine-plan/*` runs will not reach this wrapper even when cache params are enabled. Add hook coverage for the coding provider and test both ids.
  - confidence: 0.86
- **[P1] Invalidate cache when system input changes:** `extensions/volcengine/cache.ts:277-278`
  - body: The cache hash only covers `target.tools` and `target.instructions`, but the Responses transport puts `context.systemPrompt` into `target.input` as a leading system/developer item. A same-session prompt/profile change can reuse `state.responseId`, after which continuation mode trims input and drops the updated instruction.
  - confidence: 0.9
- **[P2] Retry without reusing emitted stream output:** `extensions/volcengine/cache.ts:354`
  - body: The stale-response retry calls `pipeStream(output, false)` with the same output stream. If the failed attempt already emitted a start or partial event before the error, consumers receive failed-attempt events followed by retry events; buffer until retry is decided or retry through a fresh output path.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `docs/providers/volcengine.md:38`
  - body: This is a user-facing provider feature and the diff does not update `CHANGELOG.md`. Add a single-line entry in the active Changes section with contributor credit for `@stayrascal` before merge.
  - confidence: 0.92
