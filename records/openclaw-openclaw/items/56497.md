## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current pi package namespace:** `extensions/volcengine/cache.ts:1-3`
  - body: This module imports `@mariozechner/pi-*`, but current main depends on `@earendil-works/pi-*` 0.75.1 and the Volcengine package does not declare the old namespace. As merged onto main, the new module will fail module resolution or violate the plugin runtime dependency contract.
  - confidence: 0.97
- **[P1] Hash the full Responses input before reusing cache:** `extensions/volcengine/cache.ts:277`
  - body: The cache hash only covers `tools` and `instructions`, while the Responses request carries converted prompt/profile content in `input`. A prompt or profile change in the same session can reuse the old `previous_response_id`, trim the request, and continue under stale instructions.
  - confidence: 0.93
- **[P2] Retry stale response IDs through a fresh stream:** `extensions/volcengine/cache.ts:354`
  - body: The retry path calls `pipeStream(output, false)` with the same output stream. If the failed attempt already emitted `start` or partial events before the stale-response error, those events are concatenated with the retry output.
  - confidence: 0.88
- **[P2] Align the thinking default with the docs:** `extensions/volcengine/cache.ts:111-113`
  - body: The docs say omitted `cache.thinking` defaults to `true`, but `resolveCacheConfig` leaves it `undefined` and the wrapper only sends `thinking` for explicit booleans. Users following the docs will not get the documented default behavior.
  - confidence: 0.86
