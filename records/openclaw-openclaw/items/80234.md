## Summary

Keep open: current main still lets a Discord native reply to the bot become a `reply_to_bot` implicit mention, and https://github.com/openclaw/openclaw/pull/80235 is an open closing PR rather than a merged fix.

## What This Changes

The issue requests a Discord policy that lets multi-bot deployments disable bot-authored reply-only implicit mentions while preserving human reply UX and explicit textual mentions.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80235 with a Discord-owned config policy that defaults compatibly and preserves explicit textual mentions.

## Reproduction Assessment

Yes, at source level. A guild bot-authored message whose native reply references the receiving bot reaches `reply_to_bot`, and `allowBots="mentions"` then accepts that implicit mention path.

## Solution Assessment

Yes in direction, pending maintainer API approval. A Discord-owned config policy is the narrow maintainable solution because it preserves current human reply behavior while giving multi-bot deployments an explicit opt-out for bot-authored reply-only wakes.
