## Summary

Keep this issue open: current main still lets Discord native replies to the bot become `reply_to_bot` implicit mentions without checking whether the sender is another bot, and the open linked PR is the active implementation candidate.

## What This Changes

Request a Discord policy that preserves human reply UX while allowing multi-bot deployments to require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with a maintainer-approved Discord config shape plus redacted real multi-bot Discord proof.

## Reproduction Assessment

Yes, at source level: a guild bot-authored reply with `allowBots` enabled and `referencedMessage.author.id` matching the receiving bot id reaches `reply_to_bot` and can satisfy mention gating. I did not run a live Discord gateway reproduction.

## Solution Assessment

Unclear as a final implementation until maintainers approve the config API, but the direction is narrow and Discord-owned. Keeping this issue open while the linked PR is reviewed is the right path.
