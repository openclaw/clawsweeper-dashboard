## Summary

Keep this issue open: current main and the latest release still allow a Discord native reply to the bot to become a `reply_to_bot` implicit mention, and the linked open PR is the active implementation candidate.

## What This Changes

Request a Discord policy/config path that preserves human native-reply activation while allowing deployments to require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with a maintainer-approved Discord policy that preserves human reply UX while allowing bot-authored reply-only activation to be disabled.

## Reproduction Assessment

Yes, at source level. A guild bot-authored message with `allowBots` enabled and `referencedMessage.author.id` equal to the receiving bot id reaches `reply_to_bot` implicit mention handling on current main.

## Solution Assessment

Yes in direction, but it needs maintainer API approval. A Discord-owned config policy is the narrow maintainable place to distinguish human-authored and bot-authored native reply activation.
