## Summary

Keep this issue open: current main still allows Discord `reply_to_bot` implicit mention activation without distinguishing bot-authored native replies, and https://github.com/openclaw/openclaw/pull/80235 is the active linked implementation candidate.

## What This Changes

Allow Discord deployments to keep human reply activation while requiring explicit mentions for bot-authored native replies.

## Best Possible Solution

Review, adjust if needed, and land or replace https://github.com/openclaw/openclaw/pull/80235 so Discord has a maintainer-approved sender-class policy for implicit reply mentions.

## Reproduction Assessment

Yes, at source level. A guild bot-authored message whose `referencedMessage.author.id` equals the receiving bot id becomes `reply_to_bot`, which current mention gating treats as an effective mention.

## Solution Assessment

Yes in direction. A Discord-owned config policy for implicit reply mentions is the narrow maintainable solution, but the exact API/defaults and the linked PR's merge state still need maintainer handling.
