## Summary

Keep this issue open: current main still treats a Discord native reply to the bot as an implicit `reply_to_bot` mention without distinguishing bot-authored replies, and the open linked PR at https://github.com/openclaw/openclaw/pull/80235 is the active implementation candidate.

## What This Changes

The issue requests a Discord policy that preserves human reply-to-bot activation while letting multi-bot deployments require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with a Discord-owned policy that preserves human native-reply activation while allowing bot-authored reply-only activation to be disabled.

## Reproduction Assessment

Yes, at source level: a guild bot-authored message with `allowBots=true` or `allowBots="mentions"` can carry `referencedMessage.author.id` equal to the receiving bot id, which current main maps to `reply_to_bot`. I did not run a live Discord gateway repro.

## Solution Assessment

Yes in direction: a Discord-owned policy on implicit reply mentions is the narrow maintainable place to solve this. The exact config API and defaults still need maintainer review through the linked PR.
