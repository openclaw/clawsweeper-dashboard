## Summary

Keep this issue open because current main still allows the reported implicit `reply_to_bot` activation path, and the linked open PR is the active implementation candidate rather than a merged fix.

## What This Changes

Add Discord policy so bot-authored native replies to the bot can require explicit mentions while preserving human reply UX.

## Best Possible Solution

Land a maintainer-approved Discord policy that excludes bot-authored native reply-only activation when configured, while preserving human replies and explicit textual mentions.

## Reproduction Assessment

Yes, at source level: a guild bot-authored message with `allowBots=true` or `allowBots="mentions"` and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path. I did not run a live Discord gateway repro in this read-only review.

## Solution Assessment

Unclear on the final API shape: a Discord-owned policy/config is the right boundary, but maintainers still need to accept the exact `implicitReplyMentions` design. The issue should remain open while the linked PR is reviewed or replaced.
