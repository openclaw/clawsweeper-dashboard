## Summary

Keep this issue open: current main still has the source-reproducible Discord reply-to-bot implicit mention behavior, and the open linked PR is the active implementation path with supplied real Discord proof.

## What This Changes

The issue asks for Discord native reply mention gating to distinguish bot-authored replies from human-authored replies so multi-bot deployments can require explicit mentions from bots.

## Best Possible Solution

Review, repair if needed, and land or replace https://github.com/openclaw/openclaw/pull/80235 with a maintainer-approved Discord policy that preserves human reply UX while allowing bot-authored reply-only activation to be disabled.

## Reproduction Assessment

Yes, at source level: a guild bot-authored message with `allowBots=true` or `allowBots="mentions"` and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path on current main. I did not run a live Discord gateway repro in this read-only review.

## Solution Assessment

Yes in direction: a Discord-owned implicit reply mention policy is the narrow maintainable place to solve this. The issue should remain open until the linked PR lands or is replaced because current main still lacks the sender-class policy knob.
