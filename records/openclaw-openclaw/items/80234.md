## Summary

Keep this open: current main still has the reported Discord implicit `reply_to_bot` path, and the open linked PR is the active implementation candidate with closing syntax. The issue should stay open until that PR lands or is replaced.

## What This Changes

Allow multi-bot Discord deployments to require explicit mentions for bot-authored native replies while preserving human reply UX.

## Best Possible Solution

Review and either land or replace the linked Discord PR with a maintainer-approved config shape that preserves human reply UX while allowing bot-authored reply-only activation to be disabled.

Yes, at source level: a guild bot-authored message with `allowBots` enabled and `referencedMessage.author.id` equal to the receiving bot id reaches the implicit `reply_to_bot` mention path. I did not run a live Discord gateway reproduction.

## Solution Assessment

Yes in direction: a Discord-owned preflight/config policy is the narrow maintainable place to solve this. The issue should remain open while the linked PR's API shape, tests, docs, and real behavior proof are reviewed.
