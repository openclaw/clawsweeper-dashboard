## Summary

Keep open: current main still lets a Discord native reply to the bot become a `reply_to_bot` implicit mention, and the open closing PR remains the active implementation path rather than a reason to close this issue now.

## What This Changes

The issue requests a Discord policy/config path so bot-authored native replies do not count as reply-to-bot implicit mentions unless explicitly allowed.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80235 with an approved Discord-owned policy that preserves human native reply activation and explicit mentions while allowing bot-authored reply-only activation to be disabled.

## Reproduction Assessment

Yes, at source level. A guild bot-authored Discord message with bot handling enabled and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path on current main.

## Solution Assessment

Yes in direction, pending maintainer API approval. A Discord-owned config policy is the narrow maintainable boundary for preserving human reply UX while letting multi-bot deployments disable bot-authored reply-only activation.
