## Summary

Keep this issue open: current main still lets a Discord native reply whose referenced author is the receiving bot become an implicit `reply_to_bot` mention, and https://github.com/openclaw/openclaw/pull/80235 is an open closing PR for the requested policy.

## What This Changes

The request is to preserve Discord human reply activation while letting multi-bot deployments require explicit mentions for bot-authored native replies.

## Best Possible Solution

Land or replace the open Discord PR with a config, schema, docs, and tests implementation that preserves current defaults while letting bot-authored reply-only activation be disabled without breaking explicit mentions or human replies.

## Reproduction Assessment

Yes, at source level: a guild bot-authored message with `allowBots=true` and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path on current main. I did not run a live Discord gateway reproduction in this read-only review.

## Solution Assessment

Yes in direction: a Discord-owned policy knob is the narrow maintainable solution for multi-bot deployments. This issue should remain open until the linked PR lands or is replaced by an accepted implementation.
