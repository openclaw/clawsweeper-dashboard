## Summary

Keep this issue open: current `main` still lets a Discord native reply to the bot become an implicit `reply_to_bot` mention, and the linked open PR is the active closing implementation path rather than a reason to close the issue early.

## What This Changes

Request a Discord policy that preserves human native-reply activation while letting multi-bot deployments require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and land or replace the linked Discord PR with a maintainer-approved config shape/defaults so it closes this issue after merge.

## Reproduction Assessment

Yes, at source level: a guild bot-authored native reply whose `referencedMessage.author.id` equals the receiving bot id reaches the `reply_to_bot` implicit mention path. I did not run a live Discord gateway repro in this read-only pass.

## Solution Assessment

Yes in direction, but not complete on `main`: a Discord-owned sender-class policy for implicit reply mentions is the narrow maintainable solution. The existing linked PR is the right review path, with maintainer judgment needed on the config API/defaults.
