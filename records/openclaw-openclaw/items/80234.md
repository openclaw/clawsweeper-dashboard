## Summary

Keep open: current main and the latest release still let a Discord native reply to the bot become a `reply_to_bot` implicit mention, and the linked open closing PR is the active implementation path rather than a reason to close this issue now.

## What This Changes

The issue requests a Discord policy that lets multi-bot deployments require explicit mentions for bot-authored native replies while preserving human reply-to-bot UX.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with an approved Discord-owned policy that distinguishes bot-authored from human-authored native reply activation while preserving explicit mentions.

## Reproduction Assessment

Yes, at source level: a Discord guild bot-authored native reply with `referencedMessage.author.id` equal to this bot id becomes a `reply_to_bot` implicit mention and can pass the mention gate. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

Likely yes in direction, pending maintainer API approval. A Discord-owned config policy can preserve human reply UX while letting multi-bot deployments disable bot-authored reply-only activation, but that new config surface needs maintainer review.
