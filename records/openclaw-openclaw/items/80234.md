## Summary

Keep open: current main and the latest release still allow a Discord native reply to the bot to become a `reply_to_bot` implicit mention, and the linked closing PR remains open/conflicting rather than merged.

## What This Changes

The issue requests a Discord policy that preserves human native reply activation while allowing multi-bot deployments to require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with a Discord-owned policy that distinguishes bot-authored from human-authored native reply activation while preserving explicit mentions.

## Reproduction Assessment

Yes, at source level: a Discord guild bot-authored native reply with `referencedMessage.author.id` equal to the receiving bot id becomes a `reply_to_bot` implicit mention and can pass the mention gate. I did not run a fresh live Discord replay in this read-only review; the linked PR supplies redacted live proof for the proposed behavior.

## Solution Assessment

Likely yes in direction, pending maintainer API approval. Existing `allowBots` and bot-loop protection do not separately preserve human reply UX while requiring explicit mentions for bot-authored native replies, so a Discord-owned policy knob is the narrow maintainable path if maintainers accept the config surface.
