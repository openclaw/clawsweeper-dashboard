## Summary

Keep open: current main still treats a Discord native reply to the bot as a `reply_to_bot` implicit mention, and the linked open PR with closing syntax is the active implementation path rather than a reason to close this issue now.

## What This Changes

The issue requests a Discord policy/config path that preserves human native reply activation while letting multi-bot deployments require explicit mentions for bot-authored replies.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with an approved Discord config policy that distinguishes bot-authored from human-authored native reply activation while preserving explicit mentions.

## Reproduction Assessment

Yes, at source level. A guild bot-authored Discord message with bot handling enabled and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path on current main.

## Solution Assessment

Yes in direction, pending maintainer API approval. A Discord-owned config policy is the narrow maintainable boundary for preserving human reply UX while allowing multi-bot deployments to disable bot-authored reply-only activation.
