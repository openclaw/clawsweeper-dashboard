## Summary

Keep open: current main still allows a Discord native reply to the bot to become a `reply_to_bot` implicit mention, and https://github.com/openclaw/openclaw/pull/80235 is the open closing implementation path rather than a reason to close this issue early.

## What This Changes

The issue asks for Discord multi-bot deployments to preserve human native-reply activation while optionally requiring explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with an approved Discord-owned config policy that distinguishes bot-authored from human-authored native reply activation while preserving explicit mentions.

## Reproduction Assessment

Yes, at source level. A guild bot-authored Discord message with bot handling enabled and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path on current main.

## Solution Assessment

Yes in direction, pending maintainer API approval. A Discord-owned config policy is the narrow maintainable boundary for preserving human reply UX while allowing multi-bot deployments to disable bot-authored reply-only activation.
