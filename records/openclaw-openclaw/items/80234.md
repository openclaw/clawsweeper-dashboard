## Summary

Keep this issue open: current main still lets a Discord native reply to the bot become an implicit `reply_to_bot` mention, and the linked open PR is the active implementation path rather than a reason to close early.

## What This Changes

The issue asks for Discord bot-authored native replies to be distinguishable from human replies so multi-bot deployments can require explicit mentions.

## Best Possible Solution

Review and land or replace the linked Discord PR with a maintainer-approved sender-class policy that preserves human reply UX while letting multi-bot deployments disable bot-authored reply-only activation.

## Reproduction Assessment

Yes, at source level: a guild bot-authored message with `allowBots=true` or `allowBots="mentions"` and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path. I did not run a live Discord gateway repro in this read-only review.

## Solution Assessment

Yes in direction: a Discord-owned reply-mention policy is the narrow maintainable place to solve this. The issue should remain open while https://github.com/openclaw/openclaw/pull/80235 is reviewed, repaired if needed, and landed or replaced.
