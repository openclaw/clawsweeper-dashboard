## Summary

Keep this issue open: current main and the latest release still allow Discord native replies to the bot to satisfy mention gating through the implicit `reply_to_bot` path, and https://github.com/openclaw/openclaw/pull/80235 is the active open implementation candidate with closing syntax.

## What This Changes

The issue requests a Discord policy that preserves human native-reply activation while letting multi-bot deployments require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80235 with a maintainer-approved Discord policy that suppresses bot-authored reply-only activation without breaking explicit mentions or human reply UX.

## Reproduction Assessment

Yes, at source level: a guild bot-authored Discord message with `allowBots=true` and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path on current main. I did not run a live Discord gateway repro in this read-only review, but the linked PR includes redacted live Discord proof for the reported behavior.

## Solution Assessment

Yes in direction: a Discord-owned policy knob is the narrow maintainable place to distinguish bot-authored native replies from human reply UX. The issue should stay open while the linked PR's implementation, tests, docs, and config API are reviewed.
