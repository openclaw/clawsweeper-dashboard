## Summary

Keep open: current main still treats Discord native replies to the bot as `reply_to_bot` implicit mentions without separating human-authored and bot-authored replies, and the linked open PR is the active closing implementation path rather than a reason to close this issue early.

## What This Changes

The issue asks for a Discord policy that preserves human native-reply activation while allowing multi-bot deployments to require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with an approved Discord config policy that distinguishes bot-authored from human-authored native reply activation while preserving explicit mentions.

## Reproduction Assessment

Yes, source-level: a guild bot-authored Discord message with bot handling enabled and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path on current main.

## Solution Assessment

Yes in direction, pending maintainer API approval. A Discord-owned config policy is the narrow maintainable boundary for preserving human reply UX while allowing multi-bot deployments to disable bot-authored reply-only activation.
