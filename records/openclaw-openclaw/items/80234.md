## Summary

Keep this open: current main still allows Discord native replies to the bot to satisfy mention gating through the implicit `reply_to_bot` path, and the open closing PR is the active implementation path.

## What This Changes

Allow Discord multi-bot deployments to disable reply-only implicit mentions from bot-authored native replies while preserving human reply activation.

## Best Possible Solution

Review, adjust if needed, and land or replace the linked Discord PR so the policy is implemented through a maintainer-approved config shape and focused preflight tests.

## Reproduction Assessment

Yes, at source level: a guild bot-authored Discord message with `allowBots=true` or `allowBots="mentions"` and `referencedMessage.author.id` matching the receiving bot reaches `reply_to_bot` implicit mention handling. I did not run a live Discord gateway repro in this read-only review.

## Solution Assessment

Yes in direction: a Discord-owned config/preflight policy is the narrow maintainable place to solve this. The exact public key, defaults, and migration/docs shape still need maintainer acceptance through the linked PR.
