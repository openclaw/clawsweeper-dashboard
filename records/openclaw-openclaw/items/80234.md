## Summary

Keep this issue open because current main still has the reported Discord implicit `reply_to_bot` path and the linked open PR is the active implementation candidate with closing syntax.

## What This Changes

Request a Discord policy that lets multi-bot deployments require explicit mentions for bot-authored native replies while preserving human reply UX.

## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/80235 with a maintainer-approved Discord config policy that preserves compatibility defaults and explicit mentions.

## Reproduction Assessment

Yes, at source level: a guild bot-authored message with `allowBots` enabled and `referencedMessage.author.id` equal to the receiving bot id reaches the `reply_to_bot` implicit mention path. I did not run a live Discord gateway repro in this read-only review.

## Solution Assessment

Yes in direction, pending maintainer API approval. A Discord-owned policy on implicit reply mentions is the narrow maintainable place to solve this, and the linked PR is already the active candidate.
