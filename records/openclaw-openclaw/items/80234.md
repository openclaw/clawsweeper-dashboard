## Summary

Keep this issue open: current main still lets a Discord native reply to the bot become a `reply_to_bot` implicit mention without checking whether the sender is a bot, and the linked open PR is the active implementation path.

## What This Changes

Request a Discord policy that preserves human native-reply UX while letting multi-bot deployments require explicit mentions for bot-authored native replies.

## Best Possible Solution

Review and either land or replace the linked Discord PR with a maintainer-approved config shape that preserves human reply UX while disabling bot-authored reply-only activation when configured.

## Reproduction Assessment

Yes, at source level: a guild bot-authored native reply with `referencedMessage.author.id` equal to the receiving bot id reaches `reply_to_bot` and can satisfy mention gating. I did not run a live Discord gateway repro in this read-only review.

## Solution Assessment

Unclear for the exact API shape: a Discord-owned policy for sender-class implicit reply mentions is the right boundary, but the config defaults and linked PR implementation still need maintainer review and proof.
