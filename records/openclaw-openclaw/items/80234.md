## Summary

Keep this issue open: current main still allows a Discord native reply to the bot to satisfy mention gating through the implicit `reply_to_bot` path, and the linked open PR is the active implementation candidate. It should stay open until that PR lands or is replaced.

## What This Changes

Request a Discord policy/config option so bot-authored native replies do not count as implicit reply-to-bot mentions unless explicitly allowed.

## Best Possible Solution

Review and either land or replace the linked Discord PR with a maintainer-approved config shape that preserves human reply UX while letting multi-bot deployments disable bot-authored reply-only activation.

## Reproduction Assessment

Yes, at source level: a guild bot-authored message with `allowBots=true` and `referencedMessage.author.id` equal to the receiving bot id reaches `reply_to_bot` implicit mention handling. I did not run a live Discord gateway repro in this read-only review.

## Solution Assessment

Yes in direction: a Discord-owned policy on implicit reply mentions is the narrow maintainable place to solve this. The issue should remain open while the linked PR's API shape, tests, docs, and real behavior proof are reviewed.
