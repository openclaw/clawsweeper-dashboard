## Summary

Keep open: current main and v2026.5.18 still have the reported node approval authorization shape, and an open linked PR is already the implementation candidate rather than a reason to close this issue.

## What This Changes

The issue reports that `openclaw nodes approve` blocks exec-capable node automation because approval requires `operator.admin` while automation only has `operator.pairing`.

## Best Possible Solution

Preserve command-sensitive approval scopes by default, then land or replace the linked PR with a documented automation-safe admin approval path unless maintainers explicitly reclassify node approval policy.

## Reproduction Assessment

Yes. Source inspection shows the CLI/session can request only pairing scope while the approval handler requires derived admin scope for `system.run`-class node requests; the supplied Multipass script is plausible but was not run here.

## Solution Assessment

Unclear. The issue is valid, but the current linked PR’s always-pairing fix is not clearly the best solution because it weakens the existing authorization boundary without a maintainer policy decision.
