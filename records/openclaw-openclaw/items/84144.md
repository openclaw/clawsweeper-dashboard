## Summary

Keep open: current main and v2026.5.18 still have the reported source-level authorization shape, and the linked open PR that would close this issue is unmerged and needs maintainer security/product review before this issue can be resolved.

## What This Changes

The issue asks for an automation-safe way to approve exec-capable node command surfaces after node command exposure became gated behind `openclaw nodes approve`.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/84251 or replace it with a policy-approved fix that preserves command-sensitive approval by default while giving automation a documented trusted approval path.

## Reproduction Assessment

Yes, source-backed. Current main derives `operator.admin` for `system.run` node approvals and the CLI approval path does not request that scope; the issue body supplies a plausible Multipass live reproduction that I did not run here.

## Solution Assessment

Unclear. Lowering every node approval to pairing-only is not the best unreviewed fix because it weakens the documented command-sensitive gate; maintainers should choose the intended trusted automation path first.
