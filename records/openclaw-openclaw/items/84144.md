## What This Changes

The issue reports that `openclaw nodes approve` now requires `operator.admin` for exec-capable node surfaces after v2026.5.18, blocking automation that only has `operator.pairing`.

## Best Possible Solution

Keep the command-surface approval fail-closed by default, then add or document an automation-safe approval path that can use trusted admin credentials for exec-capable node requests, with coverage for pairing-only, write, and admin cases.

## Reproduction Assessment

Yes. Source inspection shows `openclaw nodes approve` requests only `operator.pairing`, while pending `system.run`-class node approvals derive `operator.admin` and reject sessions that lack it.

## Solution Assessment

Unclear. The open PR's all-pairing approach would solve automation by weakening the documented scope gate, so maintainers should first decide whether to preserve the layered admin check or intentionally reclassify node approval policy.
