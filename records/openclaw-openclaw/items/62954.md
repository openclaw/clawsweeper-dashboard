## Summary

Keep open: current main still persists the heartbeat run's runtime model/provider into the shared session entry, and display surfaces still resolve those persisted fields. The direct fix PR #62987 is closed unmerged, so this remains a narrow valid bug.

## What This Changes

The issue asks OpenClaw to prevent heartbeat turns from overwriting the main session entry's persisted `model` and `modelProvider` fields used by chat history and Control UI model display.

## Best Possible Solution

Add heartbeat-aware preservation at the session usage persistence boundary so heartbeat turns keep the existing persisted session model identity while normal turns continue updating runtime model metadata and all usage/context fields.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: persist a heartbeat run with different `modelUsed`/`providerUsed` into an existing session, then resolve the session model through `resolveSessionModelRef`; current main will surface the heartbeat model from persisted runtime fields.

## Solution Assessment

Yes. The narrowest maintainable fix is to pass `isHeartbeat` or a clearly named preservation flag into `persistSessionUsageUpdate` and preserve only `model` and `modelProvider` for heartbeat turns.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/62954. Heartbeat turns currently persist their runtime `modelUsed`/`providerUsed` into the main session entry, causing `chat.history` and Control UI display to show the heartbeat model until a normal turn runs. Implement the narrow fix at the session usage persistence boundary: thread `isHeartbeat` or a clearly named preservation flag through `persistRunSessionUsage`/`persistSessionUsageUpdate` from both `runReplyAgent` and `createFollowupRunner`, and for heartbeat turns preserve the existing `entry.model` and `entry.modelProvider` in both the usage/fresh-context branch and the model/context-only branch. Normal non-heartbeat turns must keep current behavior, and heartbeat turns should still persist usage, token, cost, context, system prompt report, timestamps, and non-display metadata as they do today. Use closed PR #62987 only as a reference; do not broaden this into CLI session binding policy or heartbeat cost accounting. Add focused regression coverage in `src/auto-reply/reply/session.test.ts`, update `CHANGELOG.md` under Unreleased Fixes with appropriate human credit, run targeted tests/formatting, and use the changed gate in Testbox before handoff per repo policy.
