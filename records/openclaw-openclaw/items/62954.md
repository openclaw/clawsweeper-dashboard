## Summary

Keep open. Current main still persists the just-used runtime model/provider into the session entry during heartbeat runs, and display resolution still reads those persisted fields; the direct fix PR #62987 is closed unmerged, so the stale UI model bug remains a narrow valid issue.

## What This Changes

The issue asks OpenClaw to stop heartbeat turns from overwriting the main session entry's persisted `model` and `modelProvider` fields used by `chat.history` and Control UI model display.

## Best Possible Solution

Add a heartbeat-aware preservation mode at the session usage persistence boundary so heartbeat turns keep the existing persisted session model identity while normal turns continue updating runtime model metadata and all usage, token, cost, context, and system-prompt fields keep their current semantics.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: persist a heartbeat run with a different `modelUsed`/`providerUsed` into an existing session, then resolve the session display model through `resolveSessionModelRef`; current main will surface the heartbeat model from persisted runtime fields.

## Solution Assessment

Yes. The narrowest maintainable solution is to thread `isHeartbeat` or a clearly named preservation flag into `persistSessionUsageUpdate`/`persistRunSessionUsage` and preserve only `model` and `modelProvider` for heartbeat turns; broader cost or CLI binding policy should remain separate.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/62954. Heartbeat turns currently persist their runtime `modelUsed`/`providerUsed` into the main session entry, causing `chat.history`/Control UI display to show the heartbeat model until a normal turn runs. Implement the narrow fix at the session usage persistence boundary: thread `isHeartbeat` or a clearly named preservation flag through `persistRunSessionUsage`/`persistSessionUsageUpdate` from both `runReplyAgent` and `createFollowupRunner`, and for heartbeat turns preserve the existing `entry.model` and `entry.modelProvider` in both the usage/fresh-context branch and the model/context-only branch. Normal non-heartbeat turns must keep current behavior, and heartbeat turns should still persist usage, token, cost, context, system prompt report, and timestamps as they do today. Use closed PR #62987 only as reference; do not broaden this PR into CLI session binding policy or heartbeat cost accounting. Add focused regression coverage in `src/auto-reply/reply/session.test.ts`, and update `CHANGELOG.md` under the active Fixes section with credit to the reporter and useful prior PR author if applicable.
