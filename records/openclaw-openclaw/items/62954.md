## Summary

Keep open: current main still has the heartbeat stale-display path because usage persistence cannot distinguish heartbeat turns and still stores the just-used heartbeat runtime model/provider into the shared session entry.

## What This Changes

Prevent heartbeat turns from overwriting the persisted session model/modelProvider fields that UI and chat-history surfaces use for current-model display.

## Best Possible Solution

Add heartbeat-aware preservation at the session usage persistence boundary so heartbeat turns keep the existing persisted session runtime identity while normal turns continue updating model and usage metadata.

## Reproduction Assessment

Yes, by source inspection. A heartbeat turn with a different model/provider reaches `persistRunSessionUsage` without heartbeat state, `persistSessionUsageUpdate` stores that pair on the shared entry, and `chat.history` resolves that entry for display.

## Solution Assessment

Yes. Passing `isHeartbeat` or a narrowly named preservation flag into `persistSessionUsageUpdate`/`persistRunSessionUsage` is the smallest maintainable fix; cost exclusion and CLI binding behavior should stay separate unless maintainers expand scope.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/62954. Heartbeat auto-reply turns using `agents.defaults.heartbeat.model` currently pass the heartbeat `modelUsed`/`providerUsed` into `persistRunSessionUsage`, and `persistSessionUsageUpdate` writes those values into `SessionEntry.model`/`modelProvider`, which makes UI/chat-history display the heartbeat model as the current session model. Add a boolean `isHeartbeat` or narrowly named preservation flag to `persistSessionUsageUpdate`/`persistRunSessionUsage`; pass `opts?.isHeartbeat === true` from `runReplyAgent` and `createFollowupRunner`; when true, preserve `entry.model` and `entry.modelProvider` in both the usage/fresh-context branch and the model/context-only branch while continuing to update usage, context, cost, totalTokens freshness, systemPromptReport, CLI metadata, and all non-heartbeat model updates. Add focused regression coverage in `src/auto-reply/reply/session.test.ts` that seeds a session with model/provider A, persists a heartbeat run with model/provider B, and asserts the stored display model/provider remain A; cover both persistence branches if practical. Do not change model selection semantics, heartbeat cost accounting, or CLI session binding behavior unless maintainers explicitly expand the scope. Add a single-line user-facing `CHANGELOG.md` fix entry with appropriate human attribution.
