## What This Changes

The PR updates Telegram dispatch and docs so normal group tool/progress summaries are delivered when session verbose state or the verbose default is enabled, with focused dispatch tests for enabled, default-enabled, and disabled cases.

## Best Possible Solution

Evaluate Telegram normal-group verbose eligibility through the live `shouldEmitVerboseProgress()` predicate at each progress delivery point, then land only after maintainers accept `/verbose` and verbose defaults as the intended group disclosure boundary or choose a narrower explicit opt-in policy.
