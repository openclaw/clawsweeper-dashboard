## Summary

Keep this PR open: the shared-dispatch code direction is focused and the Telegram live proof is sufficient, but the added group docs overstate the behavior because Slack non-direct surfaces remain explicitly suppressed.

## What This Changes

## Best Possible Solution

Keep the explicit-session gate, caveat the Slack non-direct exception in the new group docs, and let maintainers decide whether the broader private-room config request in https://github.com/openclaw/openclaw/issues/76135 still needs a separate follow-up.

## Solution Assessment

Mostly yes. The explicit-session gate is a maintainable fix for the reported `/verbose` bug and avoids inherited defaults, but the docs need to reflect the still-existing Slack non-direct exception before merge.
