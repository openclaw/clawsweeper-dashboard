The PR changes dispatch gating, focused tests, and Telegram docs so normal Telegram groups can emit tool/progress summaries when `/verbose` or verbose defaults are on.

## Best Possible Solution

Keep the safe default and implement a shared group/channel verbose-progress policy with live `/verbose` evaluation, explicit opt-in semantics, privacy docs, and focused tests.

## Reproduction Assessment

Yes from source and supplied live transcript: current main's group gate suppresses ordinary group progress, and the PR body shows before/after Telegram QA output. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The current patch proves the need, but the maintainable fix should use a generic group/channel policy and preserve live `/verbose` revocation instead of hard-coding a Telegram-only snapshot.

## Review Findings

- **[P1] Keep the group verbose gate live:** `src/auto-reply/reply/dispatch-from-config.ts:1490`
