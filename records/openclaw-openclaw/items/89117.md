## Summary

Keep this PR open: the GA model support is useful, but the PR currently sends a task-type field with `gemini-embedding-2` even though Google's current Gemini Embedding 2 docs say that field is unsupported, and the only verification in the PR body is mocked/unit-test proof.

## What This Changes

The PR adds `gemini-embedding-2` to the Google memory embedding provider, normalizes the preview id to the GA id, widens Embedding 2 dimension validation, and reuses the provider normalizer for multimodal capability checks.

## Best Possible Solution

Update the provider, tests, and docs so GA requests follow Google's current contract, explicitly validate the preview-to-GA upgrade behavior, and require redacted real Gemini proof before merge.

## Reproduction Assessment

Yes for the PR defect: the PR head source and tests show `taskType` being sent with `gemini-embedding-2`, while the official Gemini docs say that field is unsupported for this model. I did not run a live Gemini credentialed request.

## Solution Assessment

No. Adding the GA id is the right owner boundary, but the current patch is not the best solution until it handles the GA task-type contract, updates user-facing docs/help, and proves a real request against Gemini.
