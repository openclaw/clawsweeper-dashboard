## Summary

Keep open: the GA model support is useful and current main does not already provide it, but the PR is not merge-ready because it still sends unsupported `taskType` for `gemini-embedding-2`, silently remaps the shipped preview id into a different cache/index identity, leaves config guidance stale, and lacks real Gemini API proof.

## What This Changes

The PR updates the Google memory embedding provider to accept `gemini-embedding-2`, normalize `gemini-embedding-2-preview` to the GA id, widen Embedding 2 dimensions to 128-3072, and reuse the normalized model check for multimodal embedding support.

## Best Possible Solution

Update the Google provider, tests, docs/help, and cache/index identity handling so GA requests match Google's contract, preview upgrades are explicit, and redacted live Gemini proof covers the real provider path.

## Reproduction Assessment

Yes for the PR defects at source level: PR head routes `gemini-embedding-2` through request builders that still emit `taskType`, and current memory identity code shows the preview remap changes cache/index identity. I did not run a credentialed live Gemini request.

## Solution Assessment

No. The Google provider is the right boundary, but this implementation is not the best fix until it handles the GA task-type contract, preview upgrade identity, docs/help alignment, and live provider proof.
