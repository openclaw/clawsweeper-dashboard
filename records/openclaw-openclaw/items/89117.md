## Summary

Keep this PR open, but it is not merge-ready: current main still lacks GA `gemini-embedding-2` support, while the PR head still sends an unsupported GA `taskType`, silently remaps shipped preview configs into a new cache/index identity, leaves public config guidance stale, and has no real Gemini API proof from the contributor.

## What This Changes

The PR updates the Google memory embedding provider to accept `gemini-embedding-2`, normalize `gemini-embedding-2-preview` to the GA id, widen Embedding 2 dimensions to 128-3072, and reuse the same model check for multimodal embedding support.

## Best Possible Solution

Update the Google provider, tests, docs/help, and cache/index identity handling so GA requests match Google's contract, preview upgrades are explicit, and a redacted live Gemini run proves the real provider path.

## Reproduction Assessment

Yes for the PR defects at source level: PR head routes `gemini-embedding-2` through request builders that still emit `taskType`, while Google's current docs say `task_type` is unsupported for that model. I did not run a credentialed live Gemini request.

## Solution Assessment

No. The Google provider is the right boundary, but this implementation is not the best fix until it handles the GA task-type contract, preview upgrade identity, docs/help alignment, and live provider proof.
