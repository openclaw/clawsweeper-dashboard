## Summary

Keep open: current main does not yet support the GA Gemini Embedding 2 id, but this PR is not merge-ready because it still sends unsupported GA `taskType`, silently remaps preview configs to a new model/cache identity, leaves user-facing guidance stale, and has no real Gemini proof.

## What This Changes

The PR updates the Google memory embedding provider and tests to add the `gemini-embedding-2` GA model id, widen output-dimensionality validation, normalize the preview id to GA, and share model normalization with multimodal capability checks.

## Best Possible Solution

Update the Google provider, tests, docs, and schema help so GA requests match Google's current contract, preview-to-GA index identity is explicit, and merge happens only after redacted live Gemini proof.

## Reproduction Assessment

Yes for the PR defects at source level: PR head routes `gemini-embedding-2` through request builders that still emit `taskType`, while Google's docs say `task_type` is unsupported for that model. I did not run a credentialed live Gemini request.

## Solution Assessment

No. The Google provider is the right boundary, but this implementation is not the best fix until it handles the GA task-type contract, preview upgrade behavior, docs/help alignment, and real provider proof.
