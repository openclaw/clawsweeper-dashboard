## Summary

The happy path is well proved and the earlier watcher lifecycle races were repaired, but the PR is not merge-ready because it creates an agent-reachable persistent shell-execution path that bypasses OpenClaw’s established exec approval, sandbox, environment-sanitization, command-guard, and timeout pipeline; downgrade handling also remains unsafe for the new persisted schedule kind.

## What This Changes

Adds an `on-exit` cron schedule, CLI and tool inputs, SQLite persistence, and gateway-owned command watchers that wake the originating session when the watched shell command exits.

## Best Possible Solution

Keep origin-aware wake delivery, but execute watched work through one explicit policy owner: preferably reuse the existing exec approval/sandbox/environment/timeout contract; if durable gateway execution is intentionally more privileged, remove the agent-facing command surface and expose it only through a trusted operator workflow with documented downgrade and resource-limit behavior.

## Reproduction Assessment

Yes for the blocking defect: source inspection deterministically shows the agent cron schema accepts persisted command text and the gateway launches it directly with `bash -lc`, outside the existing exec policy pipeline. The PR’s logs separately establish the intended happy-path wake behavior.

## Solution Assessment

No. The origin-aware wake reuse is sound, but raw ProcessSupervisor spawning is not the best execution boundary because it creates a second, less-governed shell authority instead of reusing OpenClaw’s established exec contract.

  - confidence: 0.98
- **[P1] Preserve on-exit jobs across release rollback:** `src/cron/store/row-codec.ts:49-62`
  - body: The new schedule kind is stored in the existing cron table, but older code decodes only `at`, `every`, and `cron`, filters unknown rows from the loaded store, and later replaces all rows from that reduced snapshot. A rollback followed by any cron write can therefore silently delete persisted `on-exit` jobs. Add an explicit migration/versioned preservation strategy and exercise downgrade plus subsequent store mutation before making this a public persisted contract.
  - confidence: 0.93
