## Summary

The feature is implemented coherently and backed by convincing live logs, but it intentionally creates durable agent-controlled shell execution under the long-lived gateway and introduces a persisted schedule kind that is not downgrade-preserved; those are explicit product/security and upgrade decisions that require owner acceptance before merge.

## What This Changes

The PR adds an `on-exit` cron schedule, CLI and tool inputs, SQLite persistence, and gateway-owned command watchers that wake the originating session when the watched shell command exits.

## Best Possible Solution

Either explicitly approve and document durable gateway-level command execution plus non-preserved downgrade as the initial product contract, or pause this PR and first unify cron command payloads and `on-exit` watchers behind an owner-approved execution and versioned persistence policy.
