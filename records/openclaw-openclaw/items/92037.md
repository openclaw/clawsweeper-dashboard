- **[P2] Preserve on-exit schedules in Control UI:** `ui/src/ui/controllers/cron.ts:584-585`

  - confidence: 0.92

  - body: Hydrating `.onExit` as `.cron` lets the editor open, but the normal save path builds and upserts a cron schedule. Until the editor is read-only for on-exit or preserves the original schedule, saving can erase the watched command and cwd.
  - confidence: 0.9
- **[P2] Update cron-tool recovery for on-exit:** `src/agents/tools/cron-tool.ts:151-159`
  - body: The tool schema exposes `command` and `cwd`, but the cron tool canonicalizer still recognizes only at/every/cron schedule kinds and does not recover flat command/cwd fields. Model-friendly calls such as `scheduleKind:"on-exit"` with a top-level command can be dropped or rejected despite the advertised tool surface.
  - confidence: 0.84
