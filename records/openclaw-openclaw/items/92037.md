  - body: The PR marks non-cron schedule kinds as read-only, but `startCronEdit`/`startCronClone` still hydrate an on-exit job into the normal cron form and `addCronJob` always rebuilds `schedule` through `buildCronSchedule`.   - confidence: 0.92
- **[P2] Gate macOS on-exit saves:** `apps/macos/Sources/OpenClaw/CronJobEditor+Helpers.swift:47`
  - body: Hydrating `.onExit` as `.cron` lets the editor open, but the normal save path builds and upserts a cron schedule. Until the editor is read-only for on-exit or preserves the original schedule, a user can erase the watched command/cwd simply by saving the job.
  - confidence: 0.9
- **[P2] Update cron-tool recovery for on-exit:** `src/agents/tools/cron-tool.ts:151-159`
  - body: The tool schema exposes `command` and `cwd`, but the cron tool's existing flat-argument recovery still recognizes only at/every/cron schedule kinds and does not recover flat command/cwd fields.
