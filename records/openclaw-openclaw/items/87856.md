## Summary

## What This Changes

The PR changes model-call diagnostic byte accounting to use incremental stream deltas and snapshotless chunks, updates logging docs, and adds focused regression coverage.

## Best Possible Solution

Land only after maintainers explicitly accept the compact `responseStreamBytes` semantics, or revise to preserve the old meaning under a separate compatibility path if downstream consumers need it.

## Reproduction Assessment

## Solution Assessment

Yes, with a maintainer compatibility decision. The wrapper owns model-call diagnostic accounting and the stream event contract makes delta strings available, but changing the exposed metric semantics is the part that needs explicit acceptance.
