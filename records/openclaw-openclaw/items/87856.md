## Summary

Keep open for maintainer review: the core accounting change is well-targeted and proof-backed, but it intentionally rebaselines exposed `responseStreamBytes` semantics and leaves OTEL metric metadata on the old wording.

## What This Changes

The PR changes embedded-agent model-call diagnostics to count high-frequency stream deltas from incremental `delta` bytes, strips growing `partial` snapshots from fallback stream byte accounting, adds focused regression tests, updates logging docs, and includes two small type-only cleanup commits.

## Best Possible Solution

Land after aligning every exposed metric/docs description with the bounded `responseStreamBytes` semantics and after a maintainer explicitly accepts the metric rebaseline, while keeping https://github.com/openclaw/openclaw/issues/86599 open for the broader local-model starvation cluster.

I did not run the repro in this read-only review, but the PR adds a focused regression test that would exercise the path.

## Solution Assessment

Mostly yes: counting high-frequency deltas from incremental bytes and omitting `partial` snapshots is a narrow fix for the diagnostic overhead. The remaining issue is surface alignment: OTEL metric metadata and maintainer compatibility acceptance should accompany the semantic change.
