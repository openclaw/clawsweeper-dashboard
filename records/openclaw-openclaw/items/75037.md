## Summary

Keep open. Current main at c38d94677c09b1b698dc3a674cc60895f5b54d2f still has the reported timestamp-refresh path, and the linked fix PR #75066 is open and unmerged, so this issue should remain the tracker until that PR or an equivalent patch lands.

## What This Changes

Fix compaction and memory-flush metadata writes so they do not refresh session activity timestamps and prevent legacy daily or idle reset.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that applies the existing preserve-activity merge policy to compaction and memory-flush bookkeeping, with regression coverage for legacy rows missing sessionStartedAt and lastInteractionAt.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a legacy session-store row missing lifecycle timestamps, followed by current compaction or memory-flush metadata persistence and then initSessionState/evaluateSessionFreshness on the next real user message.

## Solution Assessment

Yes. Extending the existing preserve-activity policy to metadata-only compaction and flush writes is the narrowest maintainable fix; changing the global legacy fallback would have a wider behavior surface.
