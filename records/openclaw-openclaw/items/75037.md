## Summary

Keep open. Current `main` still has the reported timestamp refresh path for compaction and memory-flush bookkeeping, and the provided GitHub context shows open PR #75066 specifically closes this issue but has not merged.

## What This Changes

The issue reports that compaction and memory-flush metadata writes refresh `updatedAt`, causing legacy daily and idle session reset checks to treat stale sessions as fresh.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that keeps compaction and memory-flush bookkeeping from refreshing activity timestamps while preserving normal activity refreshes for real session turns, with focused legacy daily and idle reset regression coverage.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a legacy session-store row missing `sessionStartedAt`, and for idle also missing `lastInteractionAt`, followed by current compaction or memory-flush metadata persistence before the next real user message reaches freshness evaluation.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for metadata-only compaction and flush writes is the narrowest maintainable direction; changing the global legacy fallback would affect a wider reset surface.
