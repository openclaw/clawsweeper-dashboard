## Summary

Keep open. Current main still refreshes `updatedAt` in compaction and memory-flush bookkeeping, and the closing PR #75066 remains open/unmerged, so this issue should track the remaining fix until that lands.

## What This Changes

The issue asks for compaction and memory-flush metadata writes to preserve session activity timestamps so legacy daily/idle reset freshness checks are not skipped.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping, with regression coverage for legacy daily and idle rows.

## Reproduction Assessment

Yes. A legacy session row missing lifecycle timestamps, followed by a compaction or memory-flush metadata write and the next `initSessionState` freshness check, is a high-confidence source-level reproduction path; this read-only review did not run the pasted test.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for selected metadata-only callers is the narrowest maintainable fix; changing the global legacy freshness fallback would affect a broader reset surface.
