## Summary

Keep open. Current main still refreshes `updatedAt` during compaction and memory-flush bookkeeping, and the linked implementation PR remains open rather than merged.

## What This Changes

The issue requests that compaction and memory-flush metadata writes stop refreshing session activity timestamps so legacy daily and idle reset rows still expire correctly.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping, with focused regression coverage for legacy daily and idle rows.

## Reproduction Assessment

Yes. The high-confidence path is a legacy session row missing lifecycle timestamps, a compaction or flush metadata write on current main, and the next `initSessionState` freshness check; I verified the source path rather than running tests because this review is read-only.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for selected metadata-only callers is the narrowest maintainable fix; changing the global legacy fallback would affect a broader reset surface.
