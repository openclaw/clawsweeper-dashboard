## Summary

Keep open. Current main still lets compaction and memory-flush metadata writes advance `updatedAt`, and the linked open fix PR at https://github.com/openclaw/openclaw/pull/75066 uses closing syntax for this issue, so this should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue reports that compaction and memory-flush metadata writes refresh `updatedAt`, causing legacy daily/idle reset fallback logic to treat stale sessions as fresh.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75066, or an equivalent narrow preserve-activity patch with regression coverage for legacy daily and idle rows.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a legacy row missing lifecycle timestamps can have `updatedAt` refreshed by compaction or flush metadata before `initSessionState` evaluates daily or idle freshness.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for selected metadata-only writes is the narrowest maintainable fix; changing the reset fallback globally would affect a broader session lifecycle surface.
