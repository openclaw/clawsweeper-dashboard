## Summary

Keep open. Current main still has both reported metadata write paths, and open PR #75066 already references this issue with a narrow closing implementation path.

## What This Changes

The issue asks for compaction and memory-flush bookkeeping to preserve session activity timestamps so legacy sessions still reset across daily and idle boundaries.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping.

## Reproduction Assessment

Yes. A legacy session row missing lifecycle timestamps, and with no recoverable transcript header, can have `updatedAt` advanced by compaction or flush metadata before the next freshness check.

## Solution Assessment

Yes. Reusing the existing preserve-activity policy for metadata-only writes is narrower than changing the global reset fallback or store merge defaults.
