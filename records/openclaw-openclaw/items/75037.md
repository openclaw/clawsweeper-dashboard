## Summary

Keep open. Current main still has both reported metadata-write paths that can refresh `updatedAt`, and the provided GitHub context shows PR #75066 is open with closing syntax for this issue, so the issue should stay open until that fix or an equivalent patch lands.

## What This Changes

Prevent compaction and memory-flush bookkeeping from advancing session activity timestamps used by daily and idle reset fallbacks for legacy entries.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping while preserving the existing lifecycle timestamp recovery behavior.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. A legacy entry without explicit lifecycle timestamps and without a recoverable transcript header can still have `updatedAt` advanced by compaction or flush metadata before the next freshness check.

## Solution Assessment

Yes. Reusing the existing preserve-activity policy for metadata-only compaction and flush writes is the narrowest maintainable fix; changing the global reset fallback would affect a broader session-reset surface.
