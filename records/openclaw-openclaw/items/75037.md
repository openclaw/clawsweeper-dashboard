## Summary

Keep open. Current main still has the reported session activity timestamp behavior, and the linked closing PR remains open/unmerged, so this issue should continue tracking the fix until that implementation lands.

## What This Changes

The issue reports that compaction and memory-flush metadata writes advance session `updatedAt`, causing legacy daily/idle reset checks to treat stale sessions as fresh.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping with regression coverage for legacy daily and idle rows.

## Reproduction Assessment

Yes. A legacy session row missing lifecycle timestamps, followed by a compaction or memory-flush metadata write and the next `initSessionState` freshness check, is a high-confidence source-level reproduction path; this read-only review did not run the pasted test.

## Solution Assessment

Yes. Reusing the existing preserve-activity policy for these metadata-only callers is the narrowest maintainable fix; rewriting the global legacy freshness fallback would affect a broader reset surface.
