## Summary

Keep open. Current main still refreshes activity timestamps in the compaction and memory-flush metadata paths, and PR #75066 is the open closing implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Prevent compaction and memory-flush bookkeeping writes from advancing `updatedAt` so legacy daily and idle session reset stays accurate.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping, with regression coverage for legacy daily and idle rows.

## Reproduction Assessment

Yes. A legacy session entry missing lifecycle timestamps and lacking a resolvable session header, followed by compaction or flush metadata persistence and the next `initSessionState` freshness check, is a high-confidence source-level reproduction path; this read-only review did not run the pasted test.

## Solution Assessment

Yes. Reusing the existing preserve-activity policy for metadata-only callers is the narrowest maintainable fix; rewriting the global legacy freshness fallback would affect a broader reset surface already partly covered by JSONL header recovery.
