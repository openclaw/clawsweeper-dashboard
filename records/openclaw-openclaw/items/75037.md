## Summary

Keep open. Current main still lets compaction and memory-flush bookkeeping refresh `updatedAt`, and legacy daily/idle freshness still falls back to activity timestamps that can resolve from `updatedAt`; the linked fix PR #75066 is open and unmerged, so this issue should remain the tracker until that patch or an equivalent lands.

## What This Changes

Compaction and memory-flush metadata writes should preserve session activity timestamps so legacy session rows still expire under daily and idle reset policies.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that makes compaction and memory-flush bookkeeping preserve activity timestamps, with regression coverage for legacy rows missing `sessionStartedAt` and `lastInteractionAt`.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a legacy session-store row missing lifecycle timestamps, followed by current compaction or memory-flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real user message.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for metadata-only compaction and flush writes is the narrowest maintainable solution; rewriting the global legacy fallback would have broader behavior impact.
