## Summary

Keep open. Current main still has the reported timestamp-refresh behavior, and open PR #75066 specifically closes this issue but is not merged, so this issue should remain the tracker until that PR or an equivalent fix lands.

## What This Changes

The issue reports that compaction and memory-flush metadata writes refresh session activity timestamps, causing legacy daily and idle reset checks to treat stale sessions as fresh.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch, so compaction and memory-flush bookkeeping preserve activity timestamps while real agent turns continue to advance activity normally.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a legacy session-store row missing lifecycle timestamps, followed by current compaction or memory-flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real user message; this review verified the path in source rather than running tests because the task is read-only.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for metadata-only compaction and flush writes is the narrowest maintainable fix; changing the global legacy fallback would have a wider behavior surface.
