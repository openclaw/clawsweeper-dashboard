## Summary

Keep open. Current main still lets compaction and memory-flush metadata writes refresh `updatedAt`, and the open linked fix at https://github.com/openclaw/openclaw/pull/75066 is still the active closing implementation path.

## What This Changes

The issue reports that compaction and memory-flush metadata writes advance `updatedAt`, causing legacy sessions without lifecycle timestamps to skip daily or idle reset.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75066, or an equivalent narrow preserve-activity patch with regression coverage for legacy daily and idle rows.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a legacy row missing lifecycle timestamps can have `updatedAt` refreshed by compaction or flush metadata before `initSessionState` evaluates daily or idle freshness.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for selected metadata-only writers is the narrowest maintainable fix; changing the reset fallback globally would affect a broader session lifecycle surface.
