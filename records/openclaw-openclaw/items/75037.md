## Summary

Keep open. Current main still lets compaction and memory-flush metadata writes advance `updatedAt`; the newer transcript-header lifecycle fallback mitigates legacy rows with readable headers, but the source-level no-header legacy path remains, and PR #75066 is already open with closing syntax for this issue.

## What This Changes

Request that compaction and memory-flush metadata writes stop refreshing session activity timestamps so legacy daily/idle reset freshness remains stale when appropriate.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping while keeping the transcript-header lifecycle fallback.

## Reproduction Assessment

Yes. A legacy session row missing `sessionStartedAt`, and for idle also `lastInteractionAt`, with no recoverable transcript header can have `updatedAt` advanced by compaction or flush metadata before the next `initSessionState` freshness check.

## Solution Assessment

Yes. Reusing the existing preserve-activity policy for metadata-only compaction and flush writes is the narrowest maintainable fix; changing the global reset fallback would affect a broader session-reset surface.
