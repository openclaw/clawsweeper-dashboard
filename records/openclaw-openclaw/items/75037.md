## Summary

Keep open. Current main still has the reported timestamp-refresh behavior, and the linked closing PR is open and unmerged, so this issue should remain the tracker until that PR or an equivalent fix lands.

## What This Changes

Compaction and memory-flush metadata writes should preserve activity timestamps so legacy sessions missing lifecycle fields still reset on daily or idle boundaries.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch, so compaction and memory-flush bookkeeping preserve activity timestamps while real agent turns continue to advance activity normally.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a legacy session-store row missing lifecycle timestamps, followed by compaction or memory-flush metadata persistence and then initSessionState/evaluateSessionFreshness on the next real user message; this read-only review verified the source path rather than running the pasted test.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for metadata-only compaction and flush writes is the narrowest maintainable fix; changing the global legacy fallback would affect a wider behavior surface.
