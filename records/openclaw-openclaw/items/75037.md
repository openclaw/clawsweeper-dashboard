## Summary

Keep open. Current main still lets compaction and memory-flush bookkeeping advance `updatedAt`, and the reset path can still fall back to that value for legacy rows without lifecycle timestamps or a readable transcript header; the linked fix PR remains open/unmerged.

## What This Changes

Legacy compaction and flush metadata writes should preserve activity timestamps so daily and idle resets still expire stale legacy sessions.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping with focused regression coverage.

## Reproduction Assessment

Yes. A legacy row missing `sessionStartedAt` and, for idle reset, `lastInteractionAt`, followed by a compaction or flush metadata write, follows current main into the `updatedAt` fallback on the next `initSessionState` freshness check.

## Solution Assessment

Yes. Reusing the existing preserve-activity policy for metadata-only compaction and flush callers is the narrow maintainable fix; a broader reset-policy rewrite would touch more session behavior than needed.
