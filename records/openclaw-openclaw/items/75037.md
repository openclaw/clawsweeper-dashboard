## Summary

Keep open. Current main still has both reported timestamp-refresh paths, and the linked closing PR is still open and unmerged in the provided context, so this issue should remain the tracker until that fix or an equivalent patch lands.

## What This Changes

Compaction and memory-flush metadata writes should preserve activity timestamps so legacy daily and idle reset checks do not treat stale sessions as fresh.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that makes metadata-only compaction and memory-flush writes preserve activity timestamps, with regression coverage for legacy rows missing sessionStartedAt and lastInteractionAt.

## Reproduction Assessment

Yes. A high-confidence reproduction is to create a legacy session-store row missing lifecycle timestamps, persist compaction or memory-flush metadata after a daily boundary or within an idle window, then run the next real user message through initSessionState/evaluateSessionFreshness.

## Solution Assessment

Yes. Extending the existing preserve-activity merge policy to the two metadata-only write paths is the narrowest maintainable fix; changing the global legacy fallback would affect more session behavior.
