## Summary

Keep open. Current main still has the reported activity-timestamp refresh paths, and the linked implementation PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to preserve activity timestamps when compaction and memory-flush bookkeeping update session metadata, so legacy daily/idle resets still fire.

## Best Possible Solution

Review and land PR #75066 or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping, with focused legacy daily/idle regression coverage.

## Reproduction Assessment

Yes. A legacy session row without recoverable lifecycle timestamps can be refreshed by compaction or flush metadata writes, then evaluated through initSessionState; I verified the source path in read-only mode rather than running the reproduction test.

## Solution Assessment

Yes. Reusing the existing preserve-activity policy for selected metadata-only writes is the narrowest maintainable fix; rewriting the global reset fallback would affect a broader behavior surface.
