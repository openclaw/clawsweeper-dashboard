## Summary

Keep open. Current main still refreshes `updatedAt` in both compaction-count and memory-flush bookkeeping paths, and the linked fix PR #75066 is still an open implementation candidate rather than merged code.

## What This Changes

Prevent compaction and memory-flush metadata writes from refreshing session activity timestamps so legacy daily and idle resets still fire.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that applies preserve-activity semantics only to compaction and memory-flush bookkeeping, with regression coverage for legacy daily and idle rows.

## Reproduction Assessment

Yes. A legacy session row missing lifecycle timestamps, followed by a compaction or memory-flush metadata write and then the next `initSessionState` freshness check, is a high-confidence source-level reproduction path; this read-only review did not run tests.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge policy for metadata-only callers is the narrowest maintainable fix; changing the global legacy freshness fallback would affect a broader reset surface.
