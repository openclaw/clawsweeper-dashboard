## Summary

Keep open for maintainer product/security review: current main has provider-level request proxies and managed process-level proxy routing, but auth profiles remain credential-only and still do not support profile-scoped transport selection.

## What This Changes

The issue requests an optional auth-profile `proxyUrl` field so provider requests can use a proxy selected by the active credential profile.

## Best Possible Solution

Define an approved credential-safe profile-scoped transport contract, or a config-level alternative, with explicit precedence, SecretRef/redaction handling, docs, and regression coverage.
