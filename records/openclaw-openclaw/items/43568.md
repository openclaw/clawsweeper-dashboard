## Summary

Current main already covers the operational need through provider-level request proxy controls, while keeping auth-profiles credential-only by design.

## What This Changes

Request to add an optional `proxyUrl` field to auth-profiles so each provider account can route provider HTTP calls through its own proxy.

## Best Possible Solution

Keep the shipped `models.providers.*.request.proxy` design and document provider-id/profile patterns rather than adding transport metadata to auth-profiles.

## Reproduction Assessment

Not applicable. This is a feature/config request, and source plus docs inspection confirms current main already provides the proxy routing surface outside auth-profiles.

## Solution Assessment

Yes. Provider request transport overrides are the narrower maintainable solution because they keep credentials in auth-profiles and transport policy in `models.providers.*.request`.
