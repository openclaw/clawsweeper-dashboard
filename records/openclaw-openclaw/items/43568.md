## Summary

Keep open for maintainer product/security review: current main and v2026.5.12 have provider-level request proxy config and global proxy routing, but auth profiles remain credential-only and do not define profile-scoped transport selection.

## What This Changes

Request adds an optional `proxyUrl` field to auth-profiles so different provider credentials can route model API traffic through different proxies.

## Best Possible Solution

Define an approved credential-safe profile-scoped transport contract, or choose a config-level alternative, with explicit precedence, SecretRef/redaction handling, docs, and regression coverage before implementation.

## Reproduction Assessment

Not applicable; this is a feature request, not broken documented behavior. Source and release inspection show auth-profile schema/storage lacks `proxyUrl` while model-provider `request.proxy` exists on a separate config surface.

## Solution Assessment

No; a small `proxyUrl` passthrough is not the best safe fix until maintainers decide credential storage, redaction, precedence, and egress/SSRF semantics.
