## Summary

Keep open for maintainer product/security review: current main and the latest release have provider-level request proxies and managed process proxy routing, but auth profiles remain credential-only and still lack profile-scoped proxy selection.

## What This Changes

Request adds an optional auth-profile `proxyUrl` field so each provider profile can route model-provider HTTP requests through a dedicated proxy.

## Best Possible Solution

Define an approved credential-safe profile-scoped transport contract, or a config-level alternative, with explicit precedence, SecretRef/redaction handling, docs, and regression coverage.
