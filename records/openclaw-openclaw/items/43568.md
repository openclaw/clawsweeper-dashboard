## Summary

Keep open for maintainer product/security review: current main has provider-level request proxies and managed/global proxy routing, but auth profiles remain credential-only and still do not define profile-scoped transport selection. The latest user comment also confirms the current workaround is still separate OpenClaw instances with separate `HTTPS_PROXY` values.

Request adds an optional `proxyUrl` field to auth-profiles so each model provider credential profile can route requests through a dedicated proxy.

## Best Possible Solution

Define an approved credential-safe profile-scoped transport contract, or a config-level alternative, with explicit precedence, SecretRef/redaction handling, docs, and regression coverage.
