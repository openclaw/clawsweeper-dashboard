Keep open. Current main still lacks auth-profile-scoped proxy routing; existing proxy support is route/global scoped, and adding a public auth-profile proxy field needs maintainer and security design.

## What This Changes

Request adds an optional auth-profile `proxyUrl` so requests using a selected provider credential can route through that profile's proxy.

## Best Possible Solution

Retry the Codex review after fixing the execution failure.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken documented behavior. Source inspection shows auth-profile resolution returns credential identity only while proxy routing is currently under model-provider request and global proxy surfaces.

## Solution Assessment

Unclear. The requested `proxyUrl` field is plausible, but maintainers need to decide whether transport belongs in credential storage or should remain under provider route request policy.
