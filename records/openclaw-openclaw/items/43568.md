## Summary

Request to add an optional per-auth-profile proxy URL so different provider credentials can route API requests through different proxies.

## Best Possible Solution

Define an approved credential-safe profile-scoped transport contract, or choose a config-level alternative, with explicit precedence, SecretRef/redaction handling, docs, and regression coverage before code lands.

No; a small proxyUrl passthrough is not the best safe fix until maintainers decide storage, redaction, precedence, and egress semantics for credential-adjacent proxy routing.
