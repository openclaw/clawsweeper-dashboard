## Summary

Keep open for maintainer product/security review: current main has provider-level and managed proxy routing, but it still has no credential-profile-scoped transport contract and the requested auth-profile `proxyUrl` would change credential storage and provider egress behavior.

## What This Changes

Add per-auth-profile proxy routing so different provider credentials can send model requests through different proxy endpoints.

## Best Possible Solution

Define an approved credential-safe profile-scoped transport contract, or a config-level equivalent, with explicit precedence, SecretRef/redaction handling, docs, and focused regression coverage.
