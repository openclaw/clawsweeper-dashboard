## Summary

Keep open for maintainer product/security review: current main has provider-level request proxies and global managed proxy routing, but auth profiles remain credential-only and still lack profile-scoped transport selection.

## What This Changes

Add an optional `proxyUrl` field to auth profiles so requests made with a selected provider credential can use that profile-specific proxy.

## Best Possible Solution

Define an approved credential-safe profile-scoped transport contract, or a config-level alternative, with explicit precedence, SecretRef/redaction handling, docs, and regression coverage.
