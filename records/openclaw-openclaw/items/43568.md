## Summary

Keep open for maintainer product/security review: current main has global and provider-request proxy plumbing, but auth profiles are still credential-only and do not carry profile-level proxy routing.

## What This Changes

The issue requests a `proxyUrl` field in auth-profiles so each provider profile can route model HTTP requests through its own proxy.

## Best Possible Solution

Make an explicit maintainer product/security decision first; if accepted, define a credential-safe transport contract with precedence, redaction, docs, and tests across auth-profile selection and provider request dispatch.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main lacks a profile-level proxy field and keeps auth-profiles scoped to credential material.

## Solution Assessment

No. The proposed small `proxyUrl` passthrough is not yet the best safe fix because current contracts separate credentials from request transport and proxy policy is security-sensitive.
