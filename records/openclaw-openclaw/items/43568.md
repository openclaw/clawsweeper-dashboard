## Summary

Keep open for maintainer product/security review: current main has global managed proxy routing and model-provider `request.proxy`, but auth profiles remain credential-only and do not define profile-scoped transport selection.

## What This Changes

Request to add an optional profile-scoped proxy setting to auth profiles so each provider credential profile can route model-provider requests through its own proxy.

## Best Possible Solution

Define a credential-safe profile-scoped transport contract, or choose a config-level alternative, with explicit precedence, SecretRef/redaction handling, docs, and regression coverage before adding auth-profile proxy routing.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken documented behavior. Source inspection gives high confidence that auth profiles have no profile-scoped proxy field while provider request config has a separate proxy override surface.

## Solution Assessment

No; a small `proxyUrl` passthrough in `auth-profiles.json` is not the best safe fix until maintainers decide credential storage, redaction, precedence, and network egress semantics.
