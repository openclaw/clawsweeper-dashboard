## Summary

The official `openclaw@latest` npm tarball does not match the reported corruption: the reported paths are intentionally excluded from the package and are not referenced by the shipped `dist/postinstall-inventory.json`.

## What This Changes

The issue reports that `openclaw update`/`update.run` fails because `openclaw@latest` is missing generated `dist` files.

## Best Possible Solution

Keep the current package contract and release checks, and use the documented installer-based npm recovery for local bad installs unless a new report proves the official tarball inventory is wrong.

## Reproduction Assessment

No. The official `openclaw@latest` tarball and its shipped inventory do not reference any of the eight reported missing paths, and current source intentionally excludes them.

## Solution Assessment

Not applicable as a source fix. The current manifest, inventory generator, verifier, and release checks already treat these paths as omitted/private rather than required package files.
