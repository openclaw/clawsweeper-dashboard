## Summary

Keep open: the underlying CJK context-pruning bug is still partly present on current main, but this PR targets the removed `pi-extensions` path and needs a focused port to the active `pi-hooks` pruner before merge.

## What This Changes

The PR adds a CJK/non-Latin weighted string estimator to the legacy Pi context pruner, applies it to token-estimation paths, and adds estimator tests.

## Best Possible Solution

Port the useful behavior to `src/agents/pi-hooks/context-pruning`, reuse `src/utils/cjk-chars.ts`, and add focused regressions for CJK-heavy tool results and serialized tool-call arguments.

## Reproduction Assessment

Yes, at source level. Current main’s active pruner still uses raw UTF-16 length for soft-trim eligibility and raw serialized argument length for assistant tool calls, while this PR changes only the obsolete path.

## Solution Assessment

No. The requested direction is right, but this PR is not the narrowest maintainable solution because it patches removed files and does not align all active soft-trim budget units.
