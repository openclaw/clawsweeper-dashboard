## Summary

Keep open: the reported bug is valid and the branch contains useful source ideas, but this PR is not merge-ready because it edits the retired `pi-extensions` pruner while current main runs `pi-hooks`; the active path is only partially CJK-aware and the external PR lacks real behavior proof.

## What This Changes

The PR adds a local CJK-aware string-length estimator to the legacy `pi-extensions` context pruner, applies it to several token-estimation paths including soft-trim length checks, and adds estimator tests.

## Best Possible Solution

Port the CJK-aware sizing fix to the active `src/agents/pi-hooks/context-pruning` pruner using `src/utils/cjk-chars.ts`, keep soft-trim threshold/extraction/accounting units consistent, cover tool-call arguments if still in scope, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, at source level. Current main's active pruner uses CJK-aware sizing for text blocks but still uses raw `p.length` for soft-trim eligibility and raw `JSON.stringify(...).length` for assistant tool-call arguments; no live runtime run was performed.

## Solution Assessment

No. The direction is useful, but changing the legacy `pi-extensions` pruner is not the maintainable fix; the work should be ported or replaced against `pi-hooks` with consistent units and proof.
