## Summary

Keep open: the PR is useful source work, but it is not mergeable as-is because it patches the removed `pi-extensions` pruner while the active `pi-hooks` pruner still has CJK raw-length gaps in soft-trim and tool-call argument sizing.

## What This Changes

The PR adds a local non-Latin weighted character estimator to the old `pi-extensions` context pruner, applies it to size estimation paths, and adds estimator tests.

## Best Possible Solution

Use the shared CJK estimator in the active `src/agents/pi-hooks/context-pruning` pruner for message sizing, serialized tool-call arguments, and soft-trim budget/extraction, then add focused regressions and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main still uses raw UTF-16 length for soft-trim eligibility and serialized tool-call arguments, which reproduces the underestimation path for CJK-heavy content without needing a new product decision.

## Solution Assessment

No. The requested behavior is the right direction, but this PR patches an obsolete path and changes soft-trim measurement without making extraction use the same unit.
