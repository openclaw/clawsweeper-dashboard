## Summary

Keep this PR open for maintainer handling: the bug is still source-reproducible in current main, but this branch targets the removed `pi-extensions` pruner path and lacks required real behavior proof, so it should not merge as-is.

## What This Changes

The branch adds a local CJK/non-Latin weighted string estimator to the old `pi-extensions` context pruner, applies it to message-size and soft-trim estimates, and adds estimator tests.

## Best Possible Solution

Port the CJK-aware sizing fix to `src/agents/pi-hooks/context-pruning`, reuse `src/utils/cjk-chars.ts`, cover tool-call arguments and soft-trim with consistent units, add focused regressions, and require real behavior proof or maintainer override before merge.

## Reproduction Assessment

Yes, at source level. Current main still uses raw UTF-16 length for active-pruner soft-trim eligibility and serialized tool-call arguments; no runtime test was run because this review was read-only.

## Solution Assessment

No. The requested behavior is the right direction, but this patch targets a removed path and its weighted soft-trim length change is unsafe without matching extraction and budget semantics.
