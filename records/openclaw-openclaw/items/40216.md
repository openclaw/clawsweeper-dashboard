## Summary

Keep open: the bug is real and the PR has useful intent, but the branch updates the removed `pi-extensions` pruner path while current main runs the `pi-hooks` pruner and still has CJK underestimation gaps in soft-trim eligibility and tool-call argument sizing.

## What This Changes

The PR adds a local CJK/non-Latin weighted string estimator to the old context-pruning pruner, applies it to message-size estimation, and adds estimator tests.

## Best Possible Solution

Port the CJK-aware sizing fix to `src/agents/pi-hooks/context-pruning`, reuse `src/utils/cjk-chars.ts`, make soft-trim budget/extraction units consistent, and add focused regressions.

## Reproduction Assessment

Yes, at source level. Current main still uses raw UTF-16 length for active-pruner soft-trim eligibility and serialized tool-call arguments; no runtime test was run because this was a read-only review.

## Solution Assessment

No. The requested behavior is the right direction, but this patch targets a removed path and its weighted soft-trim length change is incomplete without matching extraction semantics.
