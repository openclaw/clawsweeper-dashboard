## Summary

Keep open: current main still has a source-level Feishu dispatcher path that can drop tool-result text while a streaming card is active, and https://github.com/openclaw/openclaw/pull/79815 is an open closing PR for the same bug.

## What This Changes

Feishu group-chat browser/tool-result turns stop returning visible message bodies and can leave later group replies appearing blocked while private chat continues working.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79815; if it lands, let the merge close this issue, otherwise keep this issue as the canonical Feishu tool-result delivery bug.

## Reproduction Assessment

Yes at source level, not live: current main returns from the active Feishu streaming-card branch before delivering text for tool results, so text-only browser output can be dropped. I did not run a live Feishu/macOS/Chrome scenario in this read-only review.

## Solution Assessment

Yes, the linked PR direction appears to be the narrowest maintainable fix: keep streaming-card handling for non-tool block/final messages and let tool results use the existing text/media delivery path. Merge still needs real behavior proof and maintainer review on that PR.
