## Summary

Keep open: current `main` still has a source-visible Feishu streaming-card path that can bypass tool-result text delivery, and an open linked PR is already the active fix path for this issue.

## What This Changes

The issue reports that Feishu group-chat browser/tool operations can stop returning message bodies and leave later group mentions unanswered while private chat continues to work.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79815 after real Feishu group-chat proof is supplied, or replace it with an equivalent narrow dispatcher fix that preserves documented group reply policy.

## Reproduction Assessment

Yes, at source level: a browser/tool result can enter the Feishu dispatcher while a streaming card is active, and current `main` can return from that branch before normal text delivery. I did not run a live Feishu group-chat setup in this read-only review.

## Solution Assessment

Yes: the best fix direction is a narrow Feishu dispatcher change that lets tool-result messages fall through to the existing delivery path while preserving the current streaming-card behavior for non-tool replies.
