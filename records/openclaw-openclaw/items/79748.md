## Summary

Keep open: current main and the latest release still have the Feishu active streaming-card branch that can drop text-only tool results, and the linked open fix PR is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue reports Feishu group chats becoming unresponsive after browser/tool operations because replies that work in private chats are not delivered in the group.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79815 after redacted real Feishu group-chat proof and green checks, or apply an equivalent narrow dispatcher fix with regression tests.

## Reproduction Assessment

Yes at source level: current main routes browser/tool output through `sendToolResult`, and the Feishu active streaming-card branch returns before normal delivery for tool payloads. I did not run a live Feishu group-chat reproduction.

## Solution Assessment

Yes for the active fix direction: exempting tool payloads from the active streaming-card early return is the narrowest maintainable fix because it reuses existing card/media delivery. The issue should stay open until that PR or an equivalent fix merges.
