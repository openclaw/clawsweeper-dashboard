## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not persist ambient NODE_OPTIONS into service env:** `src/daemon/service-env.ts:340`
  - body: `resolveNodeOptions` reads `env.NODE_OPTIONS` and line 340 writes the result into generated service metadata. Current daemon install policy intentionally filters `NODE_OPTIONS` because flags like `--require` and `--import` alter Node startup, so this can make a shell or stale service-control value durable. Generate only the managed heap flag from config/defaults, or parse a heap flag without carrying arbitrary `NODE_OPTIONS` through.
  - confidence: 0.94
- **[P2] Detect underscore heap flags before appending:** `src/daemon/service-env.ts:269-271`
  - body: Node accepts `--max_old_space_size`, but this check only looks for `--max-old-space-size`. A user with `NODE_OPTIONS=--max_old_space_size=2048` would get a second `--max-old-space-size` appended, so the later generated value wins and silently changes their heap cap.
  - confidence: 0.88
