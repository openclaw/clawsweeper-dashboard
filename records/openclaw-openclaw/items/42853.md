## Summary

Keep open. The requested heap-limit feature is still absent from current main and related Linux/macOS OOM reports make the problem plausible, but this PR is not mergeable as-is because it persists ambient `NODE_OPTIONS`, misses the active setup path, regresses node-service PATH handling, and lacks real install/regenerate proof.

## What This Changes

The PR adds `gateway.nodeMaxOldSpaceMb`, schema/help labels, managed heap `NODE_OPTIONS` generation for gateway/node services, a wizard prompt, and service-env tests.

## Best Possible Solution

Ship a refreshed implementation that derives a heap-only service startup value from typed config or an approved default, rebases onto `src/wizard/setup.gateway-config.ts` and current node-service PATH handling, and validates regenerated service metadata across supported daemon targets.

## Reproduction Assessment

Yes, at source level: current main has no `gateway.nodeMaxOldSpaceMb` type/schema field and generated gateway/node service environments do not synthesize managed heap `NODE_OPTIONS`. I did not live-reproduce the long-running Linux or macOS OOM workloads.

## Solution Assessment

No. A typed heap config is a plausible direction, but this branch should not persist ambient `NODE_OPTIONS`, should rebase onto the active setup and node-service install paths, and needs real install/update proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: `resolveNodeOptions` preserves or appends the caller's ambient `NODE_OPTIONS`, and `buildCommonServiceEnvironment` writes that value into generated service metadata. Installing with `NODE_OPTIONS="--require /tmp/evil.js"` could persist an arbitrary Node startup hook despite the current host-env blocker; derive the service value only from approved heap config/defaults.
  - confidence: 0.92
- **[P2] Detect underscore heap flags before appending:** `src/daemon/service-env.ts:269`
  - body: This helper only checks for `--max-old-space-size`, but Node also accepts `--max_old_space_size` in `NODE_OPTIONS`. A user with the underscore form would get a second heap flag appended, and the later value can unexpectedly override their existing limit.
  - confidence: 0.86
- **[P2] Move the heap prompt to the active setup wizard:** `src/wizard/onboarding.gateway-config.ts:310-318`
  - body: Current main uses `src/wizard/setup.gateway-config.ts`, while this branch adds the prompt to the removed `onboarding.gateway-config.ts` path. After rebase, users would not see or clear the new setting through the active setup flow unless the prompt is moved.
  - confidence: 0.9
- **[P2] Preserve node service PATH additions:** `src/commands/node-daemon-install-helpers.ts:60-63`
  - body: Current main passes `extraPathDirs` from the selected `nodePath` into `buildNodeServiceEnvironment`. This replacement call adds the heap config but omits that argument, so supervised node services can lose the selected Node toolchain on PATH.
  - confidence: 0.82
