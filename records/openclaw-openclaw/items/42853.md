## Summary

Keep open: the OOM problem is real and current main still lacks this config surface, but this PR is not merge-ready because it persists ambient NODE_OPTIONS into durable service metadata and is stale against current setup and node-service install paths.

## What This Changes

This PR adds gateway.nodeMaxOldSpaceMb config/schema/help, injects auto-detected or configured V8 heap limits into gateway and node service NODE_OPTIONS, threads install helpers, adds a wizard prompt, and expands service-env tests.

## Best Possible Solution

Ship a refreshed implementation for #52092 and #76658 that adds the documented config contract while generating a deterministic heap-only service startup value, ports setup UI to the active wizard, preserves node-service PATH handling, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. #52092 and #76658 give concrete OOM scenarios and manual heap-flag workarounds, while the PR regressions are source-reproducible from resolveNodeOptions, the active setup wizard, current node install helper behavior, and local Node startup-flag probes.

## Solution Assessment

No. Raising the supervised Node heap via durable config is a plausible direction, but this implementation should not preserve ambient NODE_OPTIONS and must be rebased onto the current setup and node-service install paths before it is maintainable.

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: resolveNodeOptions preserves or appends the caller's existing NODE_OPTIONS, and buildCommonServiceEnvironment writes that into durable service metadata. An install run with NODE_OPTIONS="--require /tmp/evil.js" would persist that startup hook despite the current host-env blocker, so generate only the heap flag from config/default or use an equivalent non-ambient startup mechanism.

  - body: Node accepts --max_old_space_size as well as --max-old-space-size, but this helper only checks for the hyphen spelling. Users who already pinned heap with the underscore form would get a second heap flag appended, and the generated value can override their limit.

  - confidence: 0.88
