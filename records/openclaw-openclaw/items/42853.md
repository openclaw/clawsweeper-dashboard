## Summary

Keep open: current main still lacks the durable gateway heap-limit config, and the linked Linux/macOS OOM reports show this is useful work. The PR is not merge-ready because it persists ambient `NODE_OPTIONS`, targets a stale wizard file, needs rebase care around node-service PATH handling, and still lacks after-fix real service proof.

## What This Changes

The PR adds `gateway.nodeMaxOldSpaceMb`, generated daemon `NODE_OPTIONS` heap injection for gateway/node services, config schema/help metadata, a wizard prompt, and service-env tests.

## Best Possible Solution

Ship a refreshed implementation that derives a heap-only startup value from typed config or an approved default, updates the active setup/config/docs surface, preserves service PATH hardening, and proves regenerated service metadata across supported daemon targets.

## Reproduction Assessment

Yes at source level: current main has no `gateway.nodeMaxOldSpaceMb` type/schema field and generated gateway/node service environments do not synthesize managed heap `NODE_OPTIONS`. I did not live-reproduce the long-running Linux/macOS OOM workloads from the linked reports.

## Solution Assessment

No. A typed heap-limit config is the right direction, but this branch should generate only an approved heap flag, move the prompt to the active setup wizard, preserve current node-service PATH handling, and include real install/update proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Generate only an approved heap NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: `resolveNodeOptions` preserves or appends the caller's ambient `NODE_OPTIONS`, and `buildCommonServiceEnvironment` writes that value into generated service metadata. Current install hardening intentionally blocks arbitrary `NODE_OPTIONS`; persisting values such as `--require /tmp/hook.js` would give supervised services durable Node startup hooks. Build the service value from the typed heap config/default only.
  - confidence: 0.93
- **[P2] Detect underscore heap flags before appending:** `src/daemon/service-env.ts:269`
  - body: Node accepts `--max_old_space_size` in `NODE_OPTIONS`, but this check only detects the hyphenated spelling. Users who already set the underscore form would get a second heap flag appended, and the later flag can unexpectedly override their pinned limit.
  - confidence: 0.88
- **[P2] Move the heap prompt to the active setup wizard:** `src/wizard/onboarding.gateway-config.ts:310-318`
  - body: Current main uses `src/wizard/setup.gateway-config.ts`; this branch adds the prompt to `src/wizard/onboarding.gateway-config.ts`, which is no longer the active setup path. After rebase, users would not see or clear the new setting through setup unless the prompt moves to the current wizard file.
  - confidence: 0.9
- **[P2] Preserve node-service PATH additions when rebasing:** `src/commands/node-daemon-install-helpers.ts:60-63`
  - body: Current main passes `extraPathDirs` from the selected `nodePath` into `buildNodeServiceEnvironment` so supervised node services keep the chosen Node toolchain on PATH. This branch's updated call only passes `env` and `nodeMaxOldSpaceMb`, so resolving it against main can drop those PATH additions.
  - confidence: 0.82
