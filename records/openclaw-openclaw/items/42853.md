## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not persist ambient NODE_OPTIONS:** `src/daemon/service-env.ts:340`
  - body: This copies env.NODE_OPTIONS into the managed service environment whenever it already contains a heap flag, and appends the heap flag to any other existing NODE_OPTIONS. Current install policy intentionally strips NODE_OPTIONS because flags like --require or --import change Node startup, so the managed service should generate a heap-only value instead of preserving arbitrary ambient startup flags.
  - confidence: 0.94
- **[P1] Detect underscore heap aliases before appending:** `src/daemon/service-env.ts:269`
  - body: Node accepts --max_old_space_size as the same heap limit flag, and a later --max-old-space-size value wins. With the current substring check, a user-pinned NODE_OPTIONS="--max_old_space_size=2048" would get a second heap flag appended and be silently overridden by the auto/configured value.
  - confidence: 0.9
- **[P2] Pass the loaded config into node install:** `src/commands/node-daemon-install-helpers.ts:62`
  - body: This helper reads params.config?.gateway?.nodeMaxOldSpaceMb, but the node CLI call to buildNodeInstallPlan does not pass its loaded config. A normal openclaw node install would therefore ignore an explicit gateway.nodeMaxOldSpaceMb setting and fall back to auto-detect for the separately installed node service.
  - confidence: 0.86
