## Summary

Keep open. The heap-limit feature is still useful and current main does not implement it, but this PR is not ready because it persists ambient `NODE_OPTIONS`, is stale against the active setup/node-service paths, and lacks after-fix service install/update proof.

## What This Changes

The PR adds `gateway.nodeMaxOldSpaceMb`, config schema/help metadata, daemon `NODE_OPTIONS` heap injection for gateway and node services, a wizard prompt, and service-env tests.

## Best Possible Solution

Ship a refreshed implementation that derives a heap-only service startup value from typed config or an approved default, updates the active setup/config/docs surface, preserves service PATH hardening, and proves regenerated service metadata across supported daemon targets.

## Reproduction Assessment

Yes at source level: current main has no `gateway.nodeMaxOldSpaceMb` type/schema field and generated gateway/node service environments do not synthesize a managed heap `NODE_OPTIONS`. I did not live-reproduce the long-running OOM workloads from the linked Linux and macOS reports.

## Solution Assessment

No. A durable typed heap config is plausible, but this branch should not persist ambient `NODE_OPTIONS`, must rebase onto the active setup and node-service install paths, and needs real install/update proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: `resolveNodeOptions` preserves or appends the caller's ambient `NODE_OPTIONS`, and that value is written into generated service metadata. Installing with `NODE_OPTIONS="--require /tmp/evil.js"` could persist an arbitrary Node startup hook despite the current host-env blocker; derive the service value only from approved heap config/defaults.
  - confidence: 0.92
- **[P2] Detect underscore heap flags before appending:** `src/daemon/service-env.ts:269`
  - body: Node accepts `--max_old_space_size` in `NODE_OPTIONS`, but this helper only checks for the hyphenated spelling. A user with `NODE_OPTIONS=--max_old_space_size=2048` would get a second heap flag appended, and the later flag can unexpectedly override the existing limit.
  - confidence: 0.86
- **[P2] Move the heap prompt to the active setup wizard:** `src/wizard/onboarding.gateway-config.ts:310-318`
  - body: Current main uses `src/wizard/setup.gateway-config.ts`, while this branch adds the prompt to the old `onboarding.gateway-config.ts` path. After rebase, users would not see or clear the new setting through the active setup flow unless this is moved.
  - confidence: 0.9
- **[P2] Preserve node service PATH additions:** `src/commands/node-daemon-install-helpers.ts:57-60`
  - body: Current main passes `extraPathDirs` from the selected `nodePath` into `buildNodeServiceEnvironment`. This replacement call adds the heap config but omits that argument, so supervised node services can lose the selected Node toolchain on PATH after rebase.
  - confidence: 0.82
