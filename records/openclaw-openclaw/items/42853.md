## Summary

Keep open: current main still lacks the requested Gateway heap-limit config, and related reports show real user impact, but this branch is not mergeable as-is because it persists ambient `NODE_OPTIONS`, misses current setup/node-service paths, and lacks required real behavior proof.

## What This Changes

The PR adds a `gateway.nodeMaxOldSpaceMb` config/schema/help field, injects a resolved V8 heap flag into gateway and node service environments, threads install helpers, adds a wizard prompt, and expands service-env tests.

## Best Possible Solution

Ship a refreshed implementation that derives a heap-only service startup value from typed config or an approved default, rebases onto `src/wizard/setup.gateway-config.ts` and current node-service PATH handling, and validates regenerated launchd/systemd/Windows/node-host metadata.

## Reproduction Assessment

Yes, at source level: current main has no `gateway.nodeMaxOldSpaceMb` schema/type field and generated gateway/node service environments do not synthesize a heap-only `NODE_OPTIONS`. I did not live-reproduce the long-running OOM workloads from the related reports.

## Solution Assessment

No. A typed durable heap config is a plausible direction, but this diff should not persist ambient `NODE_OPTIONS`, should handle both Node heap flag spellings if preserving an existing heap flag, and must be rebased onto the active setup and node-service install paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: `resolveNodeOptions` preserves or appends the caller's existing `NODE_OPTIONS`, and this value is written into generated service metadata. Running install with `NODE_OPTIONS="--require /tmp/evil.js"` could persist an arbitrary Node startup hook, bypassing current install hardening; derive the service value only from the approved heap config/default.
  - confidence: 0.92
- **[P2] Detect underscore heap flags before appending:** `src/daemon/service-env.ts:269`
  - body: Node accepts `--max_old_space_size` in `NODE_OPTIONS`, but this helper only checks for the hyphenated spelling. A user with `NODE_OPTIONS=--max_old_space_size=2048` would get a second heap flag appended, so the generated value can unexpectedly override the existing limit.
  - confidence: 0.86
- **[P2] Move the heap prompt to the active setup wizard:** `src/wizard/onboarding.gateway-config.ts:310-318`
  - body: Current main uses `src/wizard/setup.gateway-config.ts`, while `src/wizard/onboarding.gateway-config.ts` is no longer tracked. Adding the prompt here means users will not see or clear the new setting through the current setup flow after rebase.
  - confidence: 0.9
- **[P2] Preserve node service PATH additions:** `src/commands/node-daemon-install-helpers.ts:60-63`
  - body: Current main passes `extraPathDirs` from the selected `nodePath` into `buildNodeServiceEnvironment`. This replacement call adds `nodeMaxOldSpaceMb` but omits that argument, so supervised node services can lose the selected Node toolchain on PATH.
  - confidence: 0.82
