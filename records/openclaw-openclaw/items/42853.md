## Summary

Keep open: current main still lacks durable gateway heap configuration, and the linked Linux/macOS OOM reports support the need, but this PR is not merge-ready because it persists ambient NODE_OPTIONS, targets stale setup paths, and needs maintainer security/product review for the startup policy and default sizing.

## What This Changes

The PR adds gateway.nodeMaxOldSpaceMb config/schema/help, injects a resolved --max-old-space-size value into gateway/node service NODE_OPTIONS, threads install helpers, adds an onboarding prompt, and expands service-env tests.

## Best Possible Solution

Ship a refreshed implementation tied to the OOM reports that derives a deterministic heap-only service startup value, ports setup UI to the active wizard, preserves node-service PATH handling, and includes schema/help/docs/tests/changelog with explicit maintainer approval for the default sizing policy.

## Reproduction Assessment

Yes, at source level. Current main has no gateway.nodeMaxOldSpaceMb config and does not emit NODE_OPTIONS in generated gateway/node service environments, while the linked Linux and macOS reports give concrete OOM scenarios and manual heap workarounds.

## Solution Assessment

No. The durable config direction is plausible, but this patch should not persist ambient NODE_OPTIONS and must be rebased onto the current setup and node-service install paths before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: resolveNodeOptions keeps or appends the caller's existing NODE_OPTIONS, and buildCommonServiceEnvironment writes that into durable service metadata. That regresses current install hardening: an install run with NODE_OPTIONS="--require /tmp/evil.js" can persist that startup hook. Derive the service value only from the approved heap config/default, or use an equivalent non-ambient startup mechanism.
  - confidence: 0.92
- **[P2] Detect underscore heap flags before appending:** `src/daemon/service-env.ts:265`
  - body: Node accepts --max_old_space_size as well as --max-old-space-size, but the helper only checks the hyphenated spelling. A user who already pinned heap with the underscore form would get a second heap flag appended, so the generated value can unexpectedly override their limit.
  - confidence: 0.86
- **[P2] Port the heap prompt to the active setup wizard:** `src/wizard/onboarding.gateway-config.ts:307`
  - body: Current main uses src/wizard/setup.gateway-config.ts and the onboarding gateway-config file is gone. Adding the prompt to src/wizard/onboarding.gateway-config.ts means users will not see or clear the new config through the current setup flow after rebase.
  - confidence: 0.9
- **[P2] Preserve node service PATH additions:** `src/commands/node-daemon-install-helpers.ts:57-60`
  - body: Current main passes extraPathDirs from the selected nodePath into buildNodeServiceEnvironment. This PR's updated call adds nodeMaxOldSpaceMb but omits that argument, so supervised node services can lose the selected node toolchain on PATH.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/config/types.gateway.ts:413`
  - body: This adds a user-facing gateway config field and changes installed service behavior, but the PR does not update CHANGELOG.md. Add one single-line entry under the active version with appropriate non-bot attribution.
  - confidence: 0.78
