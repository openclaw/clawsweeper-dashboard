## Summary

Keep this PR open for maintainer review, but it should not merge as-is. Current main still lacks the requested heap-limit config and service-env behavior, so the PR is relevant to the open gateway OOM report, but the proposed implementation regresses startup-env hardening by persisting ambient NODE_OPTIONS, misses Node's underscore heap flag spelling, and adds the setup prompt to a wizard file that no longer exists on current main.

## What This Changes

The PR adds a gateway.nodeMaxOldSpaceMb config/schema/help field, injects a computed or configured V8 heap limit into gateway and node service NODE_OPTIONS, threads install/setup paths, and adds service-env tests.

## Best Possible Solution

Ship a refreshed implementation that adds a documented gateway.nodeMaxOldSpaceMb contract, generates a deterministic heap-only startup setting for supervised gateway and node services, ports the prompt into the current setup flow with blank-to-auto behavior, recognizes both Node heap flag spellings when parsing existing input, and covers the OOM path with focused config/service-env/setup tests.

## Reproduction Assessment

Yes. The related OOM has concrete reproduction steps in #52092, and the PR regressions are statically reproducible from the diff: an install-time NODE_OPTIONS such as --require /tmp/evil.js would be preserved in durable service startup, and an existing --max_old_space_size value would not be detected.

## Solution Assessment

No. Raising the supervised Node heap limit is a plausible narrow fix for this OOM class, but this implementation should not preserve ambient NODE_OPTIONS and must be rebased onto the current setup wizard path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: resolveNodeOptions preserves the caller's existing NODE_OPTIONS or appends to it, and buildCommonServiceEnvironment writes that into durable service startup. That bypasses the current install hardening that strips NODE_OPTIONS; an install run with NODE_OPTIONS="--require /tmp/evil.js" would persist that startup hook. Generate only the heap flag, or use an equivalent non-ambient startup mechanism.
  - confidence: 0.91
- **[P2] Respect underscore heap flags before appending:** `src/daemon/service-env.ts:274`
  - body: Node accepts --max_old_space_size as well as --max-old-space-size, but the helper only checks for the hyphen spelling. A user who already pinned heap with the underscore flag would get a second heap flag appended, and the later generated value can silently override their limit.
  - confidence: 0.86
- **[P2] Port the prompt to the current setup wizard:** `src/wizard/onboarding.gateway-config.ts:307`
  - body: The PR adds the heap prompt to src/wizard/onboarding.gateway-config.ts, but current main uses src/wizard/setup.gateway-config.ts and no longer has that onboarding file. Without rebasing this prompt into the active setup code, users will not see or clear the new config through setup.
  - confidence: 0.87
