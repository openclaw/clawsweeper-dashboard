## Summary

Keep open: current main still lacks a durable gateway heap setting and the linked Linux/macOS OOM reports make the user problem real, but this PR is not mergeable as-is because it reintroduces ambient `NODE_OPTIONS` startup control and targets stale setup/install surfaces.

## What This Changes

The PR adds `gateway.nodeMaxOldSpaceMb`, config schema/help labels, service-environment heap flag generation, install-helper threading, an onboarding prompt, and service-env tests.

## Best Possible Solution

Ship a refreshed implementation tied to #52092 and #76658 that adds the config contract, generates a deterministic heap-only startup value, updates the active setup wizard, preserves node-service PATH handling, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. The linked Linux and macOS reports provide concrete OOM paths and manual heap workarounds, and current-main source inspection confirms there is no durable gateway heap config or service-generated `NODE_OPTIONS` today.

## Solution Assessment

No. The durable config direction is plausible, but this patch should not persist ambient `NODE_OPTIONS` and must be rebased onto the current setup and node-service install paths before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: `resolveNodeOptions` preserves or appends the caller's existing `NODE_OPTIONS`, and `buildCommonServiceEnvironment` writes that into durable service metadata. An install run with `NODE_OPTIONS="--require /tmp/evil.js"` would persist that startup hook despite current install hardening. Generate only the heap flag from config/default, or use an equivalent non-ambient startup mechanism.
  - confidence: 0.92
- **[P2] Respect underscore heap flags before appending:** `src/daemon/service-env.ts:265`
  - body: Node accepts `--max_old_space_size` as well as `--max-old-space-size`, but the helper only checks the hyphen spelling. A user who already pinned heap with the underscore form would get a second heap flag appended, and the generated value can override their limit.
  - confidence: 0.86
- **[P2] Port the prompt to the current setup wizard:** `src/wizard/onboarding.gateway-config.ts:307`
  - body: The heap prompt is added to `src/wizard/onboarding.gateway-config.ts`, but current main uses `src/wizard/setup.gateway-config.ts` and `setup.ts` imports that module. Without moving this into the active setup flow, users will not see or clear the new config through setup.
  - confidence: 0.88
- **[P2] Preserve node service PATH additions:** `src/commands/node-daemon-install-helpers.ts:57-60`
  - body: The node install change is based on the old `buildNodeServiceEnvironment({ env })` call. Current main also passes `extraPathDirs` from the selected `nodePath`; the rebase needs to keep that argument or supervised node services can lose the selected node toolchain on PATH.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `src/config/types.gateway.ts:413`
  - body: This is a user-facing feature that adds a gateway config field and changes installed service behavior, but the PR does not touch `CHANGELOG.md`. Add a single-line entry under the active version with appropriate non-bot attribution.
  - confidence: 0.78
