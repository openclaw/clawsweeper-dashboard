## What This Changes

The PR updates the opencode-go stalled-stream wrapper to re-arm its idle timer on post-delta block-boundary events and adds a regression test for boundary-only gaps between deltas.

## Best Possible Solution

Land this provider-owned wrapper fix after redacted real opencode-go stream proof confirms post-delta boundary events keep the stream alive while true stalls still abort.

## Reproduction Assessment

Yes. The linked issue and current main source show a clear source-level reproduction: boundary events are forwarded while only delta events re-arm the idle timer; I did not run tests in this read-only review.

## Solution Assessment

Yes, as a code fix. The provider-owned opencode-go wrapper is the right layer, and gating boundary liveness behind the first real delta preserves the first-event timer contract.
