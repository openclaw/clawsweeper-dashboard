## Summary

Keep open: current main still lacks a same-profile retry when overloaded errors cannot rotate to another auth profile, but this PR still has a cancellation bug in the new retry sleep and is missing the required changelog entry.

## What This Changes

The PR adds a bounded same-auth-profile retry path for overloaded LLM errors, extends failover decision logging, and adds runner harness coverage.

## Best Possible Solution

Keep the same-profile retry, but implement it as a small extension of the existing overload failover path while preserving AbortError cancellation semantics, current overload config behavior, changelog coverage, and regression tests.

## Reproduction Assessment

Yes, from source inspection: with one auth profile, `advanceAuthProfile()` returns false, so overloaded prompt/assistant failures do not retry the same profile before fallback or surfacing. A live Anthropic 529 is not needed to verify that control-flow gap.

## Solution Assessment

No, not yet. The retry direction is narrow and useful, but the patch should first normalize retry-sleep aborts like the existing overload backoff path and add the required changelog entry.
