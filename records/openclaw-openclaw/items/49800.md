## Summary

Current main still does not implement same-profile overloaded retry, but this PR is not ready to merge because the new retry sleep path preserves neither the existing abort contract nor contributor-supplied real behavior proof; the retry scope also overlaps broader configurable provider-retry work.

## What This Changes

The PR adds bounded same-profile retries for overloaded LLM errors in the embedded runner, extends failover decision logging, and adds a mocked overloaded-retry runner test.

## Best Possible Solution

Land a rebased retry implementation only if it preserves abort normalization, fits the chosen provider/agent retry contract, and includes focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main lacks same-profile overloaded retry, but the PR only supplies mocked retry coverage and says a live Anthropic 529 could not be reproduced on demand.

## Solution Assessment

No. The targeted retry is plausible, but this patch should not merge until retry sleep aborts are normalized and maintainers settle its relationship to configurable provider retry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Normalize retry sleep aborts:** `src/agents/pi-embedded-runner/run.ts:935`
