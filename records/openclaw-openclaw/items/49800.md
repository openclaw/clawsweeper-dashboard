## Summary

Keep this PR open for normal maintainer review: current main still lacks the same-auth-profile overload retry path, but the branch should not merge as-is because cancellation during the new retry sleep is misclassified, real behavior proof is missing, and the retry scope overlaps the broader configurable LLM retry work.

## What This Changes

The branch adds a bounded same-auth-profile retry loop for overloaded LLM errors in the embedded runner, extends failover logging with `retry_same_profile`, and adds mocked retry-loop coverage.

## Best Possible Solution

Choose one canonical LLM retry design, preserve abort/timeout/profile-rotation/model-fallback semantics, add focused cancellation coverage, then require redacted real behavior proof before merge.

## Reproduction Assessment

No high-confidence live reproduction is available: the PR body says a live Anthropic 529 could not be reproduced on demand. Source inspection does confirm current main lacks same-profile retry and the PR's cancellation defect is source-reproducible from the direct `sleepWithAbort` call.

## Solution Assessment

No. The retry direction is plausible, but this patch should not merge until retry-sleep aborts are normalized, the branch is rebased, real behavior proof is added, and maintainers choose its scope relative to broader configurable retry work.
