## Summary

Keep open. The same-profile overloaded retry is still useful and not implemented on current main, but the PR has a concrete abort-handling bug and lacks external after-fix real behavior proof.

## What This Changes

The PR adds bounded same-profile retry handling for transient overloaded LLM errors before profile rotation or model fallback.

## Best Possible Solution

Rework the same-profile overload retry against the current failover helpers, normalize retry-sleep aborts as `AbortError`, keep focused regression coverage, and attach redacted real runtime proof before merge.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main lacks the same-profile retry path, and the PR's test mocks two overloaded assistant results before success, but the contributor did not verify a real Anthropic 529 after the fix.

## Solution Assessment

No. The retry direction is plausible, but this patch should not merge as-is because the new sleep path does not preserve abort semantics and the branch needs to be adapted to current failover code.

Full review comments:

- **[P1] Normalize retry sleep aborts:** `src/agents/pi-embedded-runner/run.ts:935`
  - body: This new retry path awaits `sleepWithAbort` directly. Wrap this sleep the same way `maybeBackoffBeforeOverloadFailover` does.
