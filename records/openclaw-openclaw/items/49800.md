## Summary

Keep open: current main still does not implement same-auth-profile overloaded retry, so the PR is not obsolete, but the patch should not merge as-is because the new retry sleep drops AbortError normalization and external real behavior proof is missing.

## What This Changes

The PR adds a bounded same-auth-profile retry loop for overloaded LLM failover decisions, logs a `retry_same_profile` decision, and adds a mocked run-loop regression test.

## Best Possible Solution

Port the retry into current main’s failover helper structure, preserve AbortError cancellation semantics, keep focused regression coverage, and align the retry policy with the broader configurable LLM retry direction.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main has rotation/fallback/surface handling for overloaded failures but no same-profile retry after rotation fails, while the PR’s proof is mocked rather than a real overloaded provider run.

## Solution Assessment

No, not as proposed. The behavior direction is plausible, but the patch needs abort normalization, a rebase onto the current failover helper structure, and real behavior proof before it is the narrowest maintainable fix.
