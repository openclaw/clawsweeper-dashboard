## Summary

Keep open. Current main still does not implement bounded same-auth-profile retry for transient overloaded LLM errors, so this PR is not obsolete, but the proposed patch has a blocking cancellation regression and is currently dirty against main.

## What This Changes

This PR adds a bounded same-auth-profile retry loop for overloaded embedded-runner LLM failures, extends failover decision logging with `retry_same_profile`, and adds run-loop retry test coverage plus harness mocks.

## Best Possible Solution

Land a rebased or replacement implementation that adds bounded same-profile overload retry to the current prompt and assistant failover structure, normalizes cancellation exactly like the existing overload backoff helper, covers overloaded-overloaded-success and abort-during-backoff behavior, and leaves broader configurable retry policy to #49376.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing behavior by showing overloaded errors route through rotation/fallback/surface paths without `retry_same_profile`; the PR's cancellation bug is also directly reproducible from `sleepWithAbort` rejecting with a plain error during abort.

## Solution Assessment

No, not as currently proposed. The narrow same-profile retry is a reasonable maintainable direction, but it needs AbortError wrapping and a rebase onto the current split assistant-failover module before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve cancellation during overload retry sleeps:** `src/agents/pi-embedded-runner/run.ts:935`
  - body: The new retry helper awaits `sleepWithAbort` directly. That helper rejects with a plain `Error("aborted")`, while the outer fallback runner only treats `name === "AbortError"` as user cancellation, so stopping a run during this delay can be handled as a normal provider failure and continue into extra fallback attempts/API calls. Wrap the rejected sleep as an `AbortError`, matching `maybeBackoffBeforeOverloadFailover`.
  - confidence: 0.91
