## Summary

## What This Changes

The PR adds bounded same-profile retry for overloaded LLM errors in the embedded runner, a `retry_same_profile` failover decision, harness mocks, and an overloaded-retry runner test.

## Best Possible Solution

Rebase this work onto current failover helpers, preserve abort and overloaded cooldown/probe semantics, align it with the chosen provider retry contract, add focused abort/retry coverage, and attach redacted real behavior proof.

## Reproduction Assessment

No high-confidence real reproduction is available. Source inspection shows current main lacks a same-profile overload retry path, and the PR adds a mocked overloaded assistant retry test, but no live or redacted runtime proof shows the after-fix behavior.

## Solution Assessment

No. The targeted retry is plausible, but this patch should not merge until retry sleep aborts are normalized, the branch is rebased onto current failover helpers, and maintainers settle its relationship to configurable provider retry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Normalize retry sleep aborts:** `src/agents/pi-embedded-runner/run.ts:935`
  - body: This new retry helper awaits `sleepWithAbort` directly. That helper rejects cancellation as a plain `Error("aborted")`, while the fallback runner only treats `AbortError` as user cancellation, so stopping a run during this backoff can be handled as an ordinary candidate failure and continue into extra fallback/API attempts. Wrap this sleep the same way the existing overload-failover backoff does.
  - confidence: 0.9
