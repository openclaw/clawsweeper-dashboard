## Summary

Keep open. Current main has adjacent overloaded profile-rotation and fallback handling, but it still does not implement this PR's core same-auth-profile retry for single-profile overloaded failures; the proposed patch remains useful but has a blocking cancellation bug and needs rebasing onto the current split runner/failover structure.

## What This Changes

This PR adds a bounded same-auth-profile retry loop for overloaded embedded-runner LLM failures, extends failover decision logging, and adds retry test coverage plus harness mocks.

## Best Possible Solution

Land a rebased or replacement implementation on current main that adds bounded same-auth-profile retry for overloaded prompt and assistant failures, preserves cancellation as `AbortError`, fits the existing `auth.cooldowns` failover model, and leaves broader configurable transient retry policy to #49376 unless maintainers explicitly combine them.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with a single auth profile, overloaded prompt or assistant failures try `advanceAuthProfile()`, cannot rotate, and then fall through to fallback or surface-error behavior without a same-profile retry.

## Solution Assessment

No, not as currently proposed. The narrow retry direction is maintainable, but the patch must wrap aborts like the existing overload backoff helper and be rebased onto the current `run/assistant-failover.ts` split before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve cancellation during overload retry sleeps:** `src/agents/pi-embedded-runner/run.ts:935`
  - body: The new retry helper awaits `sleepWithAbort` directly. That helper rejects with a plain `Error("aborted")`, while the outer fallback runner only treats `name === "AbortError"` as user cancellation, so stopping a run during this delay can be handled as a normal provider failure and continue into extra fallback attempts/API calls. Wrap the rejected sleep as an `AbortError`, matching `maybeBackoffBeforeOverloadFailover`.
  - confidence: 0.91
