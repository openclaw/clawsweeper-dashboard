## Summary

Keep open. Current main still routes non-context-engine Codex app-server compaction through native Codex, has no first-class force-local switch, and the related open PRs address adjacent fail-closed/intercept behavior rather than this clean ChatGPT-OAuth local-compaction policy.

## What This Changes

The issue asks for a first-class way to force local Codex compaction for ChatGPT-OAuth sessions whose remote compact request times out behind Cloudflare on very large contexts.

## Best Possible Solution

Define a maintainer-approved Codex compaction policy: add a narrow documented force-local or force-context-engine route with tests and live proof, or document an upstream/context-engine escape hatch that avoids provider-name spoofing.
