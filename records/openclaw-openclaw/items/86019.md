## Summary

Keep open. Current main has native Codex compaction retry handling and an owning-context-engine escape path, but it still lacks a first-class OpenClaw knob to force local Codex compaction for the ChatGPT-OAuth timeout case.

## What This Changes

The issue requests a clean OpenClaw configuration or environment knob to force local compaction for ChatGPT-OAuth sessions whose Codex remote compaction stalls behind Cloudflare.

## Best Possible Solution

Define a maintainer-approved Codex compaction policy: either add a narrow documented force-local/force-context-engine option with tests, or document the context-engine-owned compaction path as the supported escape hatch and avoid provider-name spoofing.

## Reproduction Assessment

No high-confidence live repro was run. Source and upstream Codex inspection support the reported routing shape, but the large ChatGPT-OAuth Cloudflare timeout itself remains unverified in this review.

## Solution Assessment

Unclear. A force-local knob is plausible, but maintainers should first choose whether this belongs as a Codex-specific config/env override, a documented context-engine route, or an upstream Codex setting.
