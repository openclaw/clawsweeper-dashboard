## Summary

Keep open: current main still routes non-context-engine Codex compaction through native app-server compaction and has no supported force-local switch; related open issues and PRs cover timeout/recovery pieces but do not settle this requested config contract.

## What This Changes

The issue asks for a supported way to force local compaction for ChatGPT-OAuth Codex sessions when native remote compaction stalls on large contexts.

## Best Possible Solution

Decide the Codex compaction ownership contract, then either add and document an explicit force-local or disable-remote compaction setting with upgrade-safe tests, or document the supported context-engine workaround if maintainers reject a new knob.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection does reproduce the missing supported knob and the native Codex compaction route, but the large-context Cloudflare timeout itself remains reporter-provided evidence.

## Solution Assessment

Unclear until maintainers choose the Codex compaction contract. A force-local knob is plausible, but safer alternatives include an upstream Codex disable-remote option, explicit context-engine guidance, or first reducing runaway compact payload growth in the related open issue.
