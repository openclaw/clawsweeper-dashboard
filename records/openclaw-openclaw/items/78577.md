Keep open: current main still denies all Control UI framing and has no `gateway.controlUi` config/schema/docs escape hatch, but the requested opt-in relaxes a security boundary and needs maintainer policy review rather than cleanup closure.

## What This Changes

Request an opt-in `gateway.controlUi` frame-ancestor policy that can allow same-origin framing while keeping deny-by-default behavior.

## Best Possible Solution

If maintainers accept the use case, add a validated deny-by-default `gateway.controlUi` frame policy with documented CSP/X-Frame-Options semantics and regression coverage for deny, same-origin, and multi-origin CSP-only cases.
