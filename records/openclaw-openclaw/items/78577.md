## Summary

Close as duplicate/superseded: current main and the latest release still lack the requested Control UI framing opt-in, but the remaining configurable Control UI security-header policy is already tracked by the open canonical issue at https://github.com/openclaw/openclaw/issues/47565.

## What This Changes

The issue requests an opt-in gateway Control UI framing configuration so operators can relax CSP frame-ancestors and X-Frame-Options for same-origin iframe embeds.

## Best Possible Solution

Keep the remaining work consolidated in https://github.com/openclaw/openclaw/issues/47565, then implement one narrow typed Control UI security-header policy only after maintainers approve the frame/CSP/XFO contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing capability: Control UI responses set `X-Frame-Options: DENY` and CSP `frame-ancestors 'none'`, and the strict `gateway.controlUi` schema has no frame policy field.

## Solution Assessment

No, not as a separate standalone issue. The better solution is to handle the valid same-origin framing case through the canonical configurable Control UI header policy thread at https://github.com/openclaw/openclaw/issues/47565.
