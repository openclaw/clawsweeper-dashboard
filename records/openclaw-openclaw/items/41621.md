## Summary

Keep open for maintainer review: current main still wires both the Agents Skills panel and the top-level Skills tab to the same `skillsFilter` state, while this PR makes the requested state split. The linked bug was closed for the click/autofill symptom, but the independent filter-memory behavior proposed here is not implemented on main and needs a rebase/product decision before merge.

## What This Changes

The PR adds an `agentSkillsFilter` app/view-state field and rewires only the Agents Skills panel to read and write that field while leaving the top-level Skills tab on `skillsFilter`.

## Best Possible Solution

Land a rebased equivalent of the state split if independent Agents Skills and top-level Skills filters are the intended UI behavior, with a small regression test covering the isolation. If maintainers only wanted the closed click/autofill mitigation, close this PR as product-superseded rather than implemented.

## Solution Assessment
