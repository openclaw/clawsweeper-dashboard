Keep open: current main still routes web_fetch through the shared SSRF guard while gateway host exec proceeds from approval/allowlist handling to shell spawn without an equivalent destination guard. The issue asks OpenClaw to resolve or clarify that gateway host exec can use tools like curl to reach localhost/private URLs after web_fetch blocks the same destinations.

## Best Possible Solution

Maintainer/security owners should define the host-exec egress contract, then implement robust isolation/proxy enforcement or a documented partial mitigation with regression coverage for the chosen boundary.
