## Summary

Keep open. Current main still has the reported boundary mismatch, and the linked exec-SSRF PR was closed unmerged after review found bypasses and exec-semantics regressions; this needs maintainer/security policy direction, not cleanup closure.

## What This Changes

The issue reports that allowed host exec can curl private/internal URLs that web_fetch blocks under SSRF policy.

## Best Possible Solution

Settle and document the host-exec egress contract, then either implement robust isolation/proxy enforcement with tests or explicitly keep host exec outside web_fetch SSRF guarantees.

## Reproduction Assessment

Yes, at source level. The issue provides a localhost transcript, and current main shows web_fetch using the SSRF guard while host exec reaches the shell/process path without a destination classifier.

## Solution Assessment

Unclear. The proposed argv-parser mitigation is brittle and the prior PR showed bypass/overblocking risks; a robust egress boundary or explicit host-exec policy needs maintainer security judgment.
