## Summary

Keep open. Current main still routes WhatsApp inbound direct chats through the global DM scope default, and this PR’s code direction is narrow, but the latest diff is incomplete because it changes user-visible session-key behavior without docs, changelog, or focused WhatsApp inbound regression coverage.

## What This Changes

The PR adds a `dmScopeOverride` option to `resolveAgentRoute` and passes `per-channel-peer` from WhatsApp inbound routing so direct chats use peer-scoped session keys.

## Best Possible Solution

Land the routing override with a WhatsApp inbound regression test, updated WhatsApp/session docs, and an Unreleased changelog note for the session-key compatibility change.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the bug: WhatsApp direct messages pass a direct peer into `resolveAgentRoute`, and unset `session.dmScope` defaults to `main`, so distinct contacts share `agent:<agentId>:main`.

## Solution Assessment

No as submitted. The per-call override is a maintainable direction for the bug, but the PR should also document the new inbound behavior and add focused regression coverage before merge.
