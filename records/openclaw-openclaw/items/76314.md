## Summary

Keep this PR open: current main still has the reported WhatsApp inbound DM collapse path, but the branch needs a small follow-up before merge because it changes WhatsApp session semantics without updating the public docs/changelog.

## What This Changes

Adds `dmScopeOverride` to `resolveAgentRoute`, uses it for WhatsApp inbound routing, and forwards configured agent IDs through subagent target policy.

## Best Possible Solution

Land a focused fix that isolates WhatsApp inbound DMs per peer, documents and changelogs the compatibility change, and adds regression coverage for distinct WhatsApp direct peers.

## Reproduction Assessment

Yes. On current main, source inspection reproduces the path: WhatsApp direct messages call `resolveAgentRoute`, which defaults unset `session.dmScope` to `main`, while existing route tests show per-peer isolation only when an isolated scope is configured.

## Solution Assessment

No as submitted. The per-call override is a narrow code direction for the routing bug, but the PR should update docs/changelog and add a WhatsApp inbound regression test; the unrelated subagent change should be split or explicitly reviewed.
