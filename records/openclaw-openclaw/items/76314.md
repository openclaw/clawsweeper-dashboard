## Summary

Keep open. The PR addresses a source-reproducible WhatsApp DM session collision, but its last-route change still suppresses per-contact deliveryContext updates when a main DM owner pin is present, so it should not merge until that guard is narrowed.

## What This Changes

The branch adds a per-call dmScope override, applies per-channel-peer routing to inbound WhatsApp direct messages, and updates WhatsApp last-route handling, tests, docs, and changelog.

## Best Possible Solution

Preserve the per-contact WhatsApp DM routing change and make last-route metadata follow the scoped session, with the main-session owner pin applied only to shared main-session writes.

## Reproduction Assessment

Yes. Source inspection of current main shows WhatsApp direct messages call resolveAgentRoute with a direct peer but no override, and resolveAgentRoute defaults dmScope to cfg.session?.dmScope ?? "main".

## Solution Assessment

No as submitted. The per-call override is a narrow fit, but the scoped last-route branch should not reuse the main-session pin guard; Telegram's update target check is the safer local pattern.
