## Summary

Keep open. Current main still collapses default WhatsApp direct messages into the agent main session, and this PR addresses the central route-scoping bug, but the latest diff still applies the main-session pin guard to scoped per-contact last-route writes, leaving an allowed non-owner contact path without deliveryContext metadata.

## What This Changes

The PR adds a per-call dmScope override to shared route resolution, applies it to inbound WhatsApp DMs, records last-route metadata for scoped direct sessions, and updates WhatsApp docs, changelog, and tests.

## Best Possible Solution

Update WhatsApp last-route handling so main DM pinning only suppresses writes to the shared main session, while per-contact scoped sessions always record their own deliveryContext, with a pinned scoped-DM regression test alongside the existing docs and changelog updates.

## Reproduction Assessment

Yes. Source inspection of current main shows default WhatsApp DMs reach resolveAgentRoute with a direct peer but no override, and resolveAgentRoute defaults dmScope to "main", producing the shared agent:<agentId>:main session key.

## Solution Assessment

No as submitted. The route override is a narrow fit and the PR addresses unpinned scoped last-route writes, but the new branch still reuses a main-session pin guard that should not suppress per-contact session deliveryContext updates.
