## Summary

Keep open. The latest head is narrowed to the WhatsApp route fix, docs, changelog, and coverage, but it switches inbound DMs to session-scoped routes without persisting last-route delivery metadata for those new per-contact sessions.

## What This Changes

The PR adds a per-call routing dmScope override, applies it to inbound WhatsApp direct messages, and updates WhatsApp docs, changelog, and a regression test.

## Best Possible Solution

Land per-contact WhatsApp DM session keys together with last-route/deliveryContext persistence for the routed direct session and focused regression coverage.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: two inbound WhatsApp DMs with different peer IDs and default session config both reach resolveAgentRoute with dmScope "main" and therefore share agent:<agentId>:main.

## Solution Assessment

No as submitted. The dmScopeOverride route seam is a narrow fit, but WhatsApp also needs to persist last-route metadata for the per-contact direct session instead of only handling the main-session path.
