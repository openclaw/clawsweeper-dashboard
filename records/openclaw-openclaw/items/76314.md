## Summary

Keep open and request changes. The PR fixes the normal WhatsApp inbound DM route with the existing per-channel-peer session-key shape, but it misses the documented WhatsApp broadcast-DM fan-out path, which rebuilds per-agent direct routes from the global dmScope and would still collapse different contacts to agent main sessions under the default config.

## What This Changes

The PR adds a per-call dmScope override to shared route resolution, applies it to WhatsApp inbound routing, records scoped WhatsApp DM last-route metadata, and updates WhatsApp docs, changelog, and regression tests.

## Best Possible Solution

Extend the WhatsApp direct-DM scoping fix through broadcast fan-out so normal and broadcast inbound DMs both use per-contact session keys, then land with focused WhatsApp regression coverage and the existing docs/changelog updates.

## Reproduction Assessment

Yes. Source inspection on current main shows normal WhatsApp DMs pass a peer into resolveAgentRoute while the resolver defaults dmScope to main, and the broadcast-DM path still constructs direct session keys from cfg.session.dmScope.

## Solution Assessment

No as-is. The base-route override is the right mechanism for the normal inbound path, but the patch should also cover documented WhatsApp broadcast DMs or the direct-chat isolation claim remains incomplete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Scope WhatsApp broadcast DMs too:** `extensions/whatsapp/src/auto-reply/monitor/on-message.ts:97`
  - body: `dmScopeOverride` only scopes the base route. Direct DMs that match `cfg.broadcast[peerId]` return through `maybeBroadcastMessage` before that route is processed, and broadcast fan-out rebuilds each agent route with `dmScope: params.cfg.session?.dmScope`; default configs therefore still send different broadcast DM contacts into `agent:<agent>:main` despite the new direct-chat isolation behavior/docs. Please pass the WhatsApp per-channel scope into broadcast route construction and add a DM broadcast regression.
  - confidence: 0.84
