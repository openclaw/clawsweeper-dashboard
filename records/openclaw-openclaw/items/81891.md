## What This Changes

## Best Possible Solution

Add the missing Google Meet realtime SecretRef contract and ensure Google Meet command/tool realtime execution uses the resolved runtime snapshot before provider config normalization, with focused regression coverage for join/create and provider apiKey paths.

Yes. The current source path is sufficient to reproduce at code level: Google Meet realtime provider configs are not declared as SecretRef targets, then unresolved provider apiKey objects are forwarded into strict provider normalization.

Unclear that the broad loader-order change suggested in the issue is the best first fix.
