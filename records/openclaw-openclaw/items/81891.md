## Summary

Keep open: current main still has a source-traceable SecretRef lifecycle gap for Google Meet realtime paths, and the related fixes only covered narrower status, Talk, Voice Call, and OpenAI keychain cases.

## What This Changes

The issue reports that Google Meet realtime join/create and gateway tool paths fail when realtime provider API keys are configured as exec-backed 1Password SecretRefs.

## Best Possible Solution

Resolve Google Meet realtime provider SecretRefs through the existing runtime snapshot or plugin secret-input contract before provider normalization, then cover join/create and gateway-tool paths with focused regression tests.

## Reproduction Assessment

Yes. The current source path is high confidence: Google Meet realtime provider config can bypass a SecretRef contract and reach OpenAI realtime normalization, which rejects unresolved SecretRef objects with the reported error shape.

## Solution Assessment

Yes in direction, but the exact boundary needs maintainer choice. A focused fix should use the existing SecretRef/runtime snapshot seams rather than broad ad hoc plugin-loading changes unless tests prove the broader order change is required.
