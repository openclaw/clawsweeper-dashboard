- **[P2] Use an exported or local host type:** `ui/src/ui/gateway-sso-postmessage.ts:1`
  - body: `SettingsHost` is local to `app-settings.ts` on current main, but this new module imports it. Typecheck already fails on this line, so either export that type intentionally or define the narrow SSO host shape locally.
  - confidence: 0.97

  - body: `createSettings()` is typed as `UiSettings` but omits required fields such as chat visibility, split, nav width, and nav collapse state. `check-test-types` fails, so the helper needs the full current settings shape or a narrower test host type.
  - confidence: 0.94
- **[P2] Reconnect when the token changes during startup:** `ui/src/ui/gateway-sso-postmessage.ts:95-97`
  - body: The listener reconnects only when `host.connected` is already true. If the SSO token arrives after the Gateway client is created but before hello sets `connected`, the ack is sent while the active client still uses the stale token captured at connect start.
  - confidence: 0.9
- **[P2] Require HTTPS for production SSO origins:** `ui/src/ui/gateway-sso-postmessage.ts:26-33`
  - body: The allowlist accepts both HTTP and HTTPS before allowing `clusterclaw.ai` and subdomains, so cleartext production origins can inject a gateway token. Keep HTTP limited to loopback development origins and require HTTPS for ClusterClaw hosts.
  - confidence: 0.88
