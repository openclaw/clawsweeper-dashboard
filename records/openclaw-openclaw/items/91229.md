## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use an exported or local host type:** `ui/src/ui/gateway-sso-postmessage.ts:1`
  - body: `SettingsHost` is private to `app-settings.ts` on current main, but this new module imports it. Any real TypeScript check of this module will fail; either export that type deliberately or define the narrow SSO host shape locally.
  - confidence: 0.96
- **[P2] Reconnect when tokens change during startup:** `ui/src/ui/gateway-sso-postmessage.ts:90-92`
  - body: The listener only calls `reconnect` when `host.connected` is already true. If the parent sends the SSO token after `connectGateway` creates the client but before hello sets `connected`, the ack is sent while the active client still uses the stale token captured at connect start.
  - confidence: 0.9
- **[P2] Require HTTPS for production SSO origins:** `ui/src/ui/gateway-sso-postmessage.ts:25-32`
  - body: The allowlist accepts both HTTP and HTTPS before checking for `clusterclaw.ai`, so cleartext ClusterClaw origins can inject a gateway token. Keep HTTP for loopback development only and require HTTPS for production origins.
  - confidence: 0.86
