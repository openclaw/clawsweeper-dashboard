## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use a valid host type for the SSO listener:** `ui/src/ui/gateway-sso-postmessage.ts:1`
  - body: `SettingsHost` is private to `app-settings.ts` on current main, but this new module imports it. A real TypeScript check of this module will fail; define the narrow SSO host shape locally or deliberately export a public host type.
  - confidence: 0.96
- **[P2] Reconnect when the token changes during startup:** `ui/src/ui/gateway-sso-postmessage.ts:95-96`
  - body: The listener only reconnects when `host.connected` is already true. If the parent sends the SSO token after `connectGateway` creates the client but before hello marks it connected, the ack is sent while the active client still uses the stale token captured at connect start.
  - confidence: 0.9
- **[P2] Require HTTPS for production SSO origins:** `ui/src/ui/gateway-sso-postmessage.ts:26-33`
  - body: The allowlist accepts both HTTP and HTTPS before allowing `clusterclaw.ai` and subdomains, so a cleartext production origin can inject a gateway token. Keep HTTP for localhost/127.0.0.1 development origins only.
  - confidence: 0.86
