- **[P2] Use an exported or local host type:** `ui/src/ui/gateway-sso-postmessage.ts:1`
  - body: `SettingsHost` is private to `app-settings.ts` on current main, but this new module imports it. The PR head already fails prod type checking with TS2459; either export that type deliberately or define the narrow SSO host shape locally.
  - confidence: 0.96
- **[P2] Complete the settings fixture:** `ui/src/ui/gateway-sso-postmessage.test.ts:20-28`
  - body: `createSettings()` is typed as `UiSettings` but omits required fields such as chat display and navigation settings, so test type checking fails on this PR head. Build the fixture from the existing settings defaults or include the full shape.
  - confidence: 0.92
- **[P2] Reconnect when tokens change during startup:** `ui/src/ui/gateway-sso-postmessage.ts:95-96`
  - body: The listener only calls `reconnect` when `host.connected` is already true. If the parent sends the SSO token after `connectGateway` creates the client but before hello sets `connected`, the ack is sent while the active client still uses the stale token captured at connect start.
  - confidence: 0.9
- **[P2] Require HTTPS for production SSO origins:** `ui/src/ui/gateway-sso-postmessage.ts:26-33`
  - body: The allowlist accepts both HTTP and HTTPS before checking for `clusterclaw.ai`, so a cleartext ClusterClaw origin can inject a gateway token. Keep HTTP for loopback development only and require HTTPS for production origins.
  - confidence: 0.87
