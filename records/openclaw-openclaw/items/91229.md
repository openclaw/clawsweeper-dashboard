## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use an exported or local host type:** `ui/src/ui/gateway-sso-postmessage.ts:1`
  - body: `SettingsHost` is private to `app-settings.ts` on current main, but this new module imports it. The PR's `check-prod-types` job already fails with TS2459, so either export that type deliberately or define the narrow SSO host shape locally.
  - confidence: 0.96
- **[P2] Reconnect when the token changes during startup:** `ui/src/ui/gateway-sso-postmessage.ts:95-96`
  - body: The listener only calls `reconnect` once `host.connected` is true. If the parent sends the SSO token after `connectGateway` creates the client but before hello marks it connected, the ack is sent while the active client still uses the stale token captured at connect start.
  - confidence: 0.9
- **[P2] Require HTTPS for production SSO origins:** `ui/src/ui/gateway-sso-postmessage.ts:26-33`
  - body: The allowlist accepts both HTTP and HTTPS before checking for `clusterclaw.ai`, so `http://clusterclaw.ai` and cleartext subdomains can inject a gateway token. Keep HTTP for loopback development only and require HTTPS for production ClusterClaw origins.
  - confidence: 0.86
