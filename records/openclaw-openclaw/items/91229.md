Overall confidence: 0.92

Full review comments:

- **[P1] Require HTTPS for production SSO origins:** `ui/src/ui/gateway-sso-postmessage.ts:26-33`
  - body: The allowlist accepts `http:` before trusting `clusterclaw.ai` hosts, so a cleartext ClusterClaw origin can inject a gateway token. Keep HTTP limited to loopback development origins and require HTTPS for production SSO origins.
  - confidence: 0.88

  - body: `SettingsHost` is private to `app-settings.ts` on current main, but this new module imports it, which fails prod type checking with TS2459. Export that type deliberately or define the narrow SSO host shape locally.
  - confidence: 0.97
- **[P2] Complete the settings fixture:** `ui/src/ui/gateway-sso-postmessage.test.ts:19-28`
  - body: `createSettings()` is typed as `UiSettings` but omits required chat display and navigation fields, so test type checking fails on this PR head. Build the fixture from current defaults or include the full shape.
  - confidence: 0.94
- **[P2] Reconnect when tokens change during startup:** `ui/src/ui/gateway-sso-postmessage.ts:95-96`
  - body: The listener only calls `reconnect` when `host.connected` is already true. If the parent sends the SSO token after `connectGateway` creates the client but before hello sets `connected`, the ack is sent while the active client still uses the stale token captured at connect start.
  - confidence: 0.9
