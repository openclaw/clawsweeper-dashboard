## Summary

Keep open. Current main has useful plan-only and reminder/cron-specific mitigations, but it still does not enforce the general invariant for final replies that promise to come back later without starting a real follow-up mechanism.

## What This Changes

The issue asks OpenClaw to prevent or annotate final agent replies that promise a later follow-up when the same turn did not start cron, session spawn, queued work, or another real follow-up path.

## Best Possible Solution

Add a conservative final-reply consistency guard in the reply pipeline that annotates or rewrites unbacked future-follow-up promises only when no cron, spawned session, queued task, messaging delivery, or other real follow-up mechanism was created in the same turn.

## Reproduction Assessment

No high-confidence live reproduction was established on current main because the behavior is model-dependent and the reporter saw improvement on 2026.4.23. Source inspection does show a concrete remaining gap for broader promise-style final text such as coming back with a short message after non-plan tool activity.

## Solution Assessment

No, current main is only a partial solution. The maintainable fix is to extend the existing reply-pipeline guard using runner evidence for real follow-up mechanisms, rather than relying on prompts, post-reply commitments, or provider-specific plan-only retries.
