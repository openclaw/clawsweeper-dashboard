## Summary

Keep open. Current main has adjacent strict-agentic, reminder-note, accepted-spawn, and opt-in commitment mechanisms, but it still does not clearly enforce the broader invariant that a visible later-follow-up promise must be backed by same-turn follow-up evidence.

## What This Changes

The issue asks OpenClaw to prevent or annotate agent replies that promise a later follow-up without creating a real background, scheduled, spawned, or queued follow-up action.

## Best Possible Solution

Define the reply-pipeline contract for future-follow-up language, then add focused enforcement and regression coverage that checks same-turn evidence such as cron creation, accepted `sessions_spawn`, or other durable background action before delivering a promise.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live current-main run; source inspection shows credible uncovered cases for broad `come back later` promises and non-safe single-tool paths, but a fresh live Telegram/TUI repro would raise confidence.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the enforcement policy; extending the existing reminder/strict-agentic paths is plausible, but the product decision is whether to retry, annotate, rewrite, or gate broad future commitments.
