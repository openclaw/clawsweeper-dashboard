## Summary

Keep open. Current main has adjacent fixes for reminder disclaimers, strict-agentic plan-only recovery, commitments, and durable final-delivery retries, but it still does not enforce the broader reply-pipeline invariant that a visible promise to follow up later must be backed by same-turn follow-up evidence.

## What This Changes

The issue asks OpenClaw to prevent or annotate final agent replies that promise a later follow-up when no background action, scheduled job, spawned session, or other follow-up mechanism was started.

## Best Possible Solution

Define a conservative reply-pipeline consistency guard that checks same-turn evidence for cron creation, spawned sessions, durable background tasks, queued follow-up work, or explicit message sends before annotating or rewriting concrete future-follow-up promises.

## Reproduction Assessment

No, we do not have a high-confidence live current-main reproduction. Source inspection does show that final replies with “come back later” style text can bypass the existing reminder-only guard and are not checked against broader same-turn follow-up evidence.

## Solution Assessment

Mostly yes, a runtime consistency guard is the right layer, but the exact rewrite versus note behavior and false-positive tolerance need maintainer product judgment before implementation.
