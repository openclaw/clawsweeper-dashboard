## Summary

Keep open. Current main has useful reminder-note and incomplete-turn mitigations, but it still does not enforce the broader invariant that a final reply promising to come back later must be backed by a real follow-up mechanism in the same turn.

## What This Changes

The issue requests a runtime reply-pipeline guard for unbacked future follow-up promises, such as saying the agent will come back later without starting a cron, subagent, queued task, or other background action.

## Best Possible Solution

Extend the existing reply-pipeline guard conservatively so unbacked future-follow-up promises are annotated or rewritten only when the same turn did not create cron, spawned-session, queued follow-up, messaging delivery, or another explicit background mechanism.

## Reproduction Assessment

Unclear for a high-confidence live reproduction on current main because model output is stochastic and the reporter saw improvement after 2026.4.23. Source inspection does show the guard gap for broader final text such as promising to come back with a short message.

## Solution Assessment

No, current main is not the complete best solution because reminder notes and plan-only retries cover only narrower cases. The best fix is a focused extension of the existing guard using same-turn evidence rather than a broad new commitment-gating feature.
