## Summary

Keep open because current main only has adjacent partial behavior: a cron-specific reminder note and opt-in inferred commitments, neither of which enforces that a visible generic follow-up promise is backed by same-turn work.

## What This Changes

Request runtime enforcement so final replies that promise a later follow-up are backed by a real same-turn mechanism or corrected before delivery.

## Best Possible Solution

Define a core reply-pipeline contract for unbacked future commitments, then test it across reminder cron, commitments, subagent/background work, and common false positives.

## Reproduction Assessment

No live repro was run; source inspection shows current main only covers narrow cron-backed reminder phrases and leaves the reported generic “come back later” promise class unenforced by default.

## Solution Assessment

No, current main is not the complete solution because reminder notes and opt-in commitments are useful adjacent mechanisms but do not establish the requested same-turn backing invariant.
