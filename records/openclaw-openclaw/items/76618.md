## What This Changes

The PR changes Comfy Cloud polling to accept `success` as completion and `error` as a terminal failure in `extensions/comfy/workflow-runtime.ts`.

## Best Possible Solution

Keep the polling fix in the Comfy plugin, but preserve Cloud failure details, add focused `success`/`error`/`error_message` coverage, refresh the branch, and require redacted real Comfy Cloud proof before merge.

## Reproduction Assessment

Partly: source inspection shows current main would keep polling for `success` or `error` status responses until timeout. The `error` value is documented upstream, while `success` still needs live/redacted proof because the official status schema inspected did not list it.

## Solution Assessment

No: `waitForCloudCompletion` is the right boundary, but this is not the best merge shape until it preserves `error_message`, adds focused coverage, refreshes the branch, and includes real Cloud proof.
