## What This Changes

The PR changes Comfy Cloud polling in `extensions/comfy/workflow-runtime.ts` to treat `success` as completion and `error` as terminal failure.

## Best Possible Solution

Keep the fix at `waitForCloudCompletion`, parse `error_message`, add focused mocked coverage for `success`, `error`, and diagnostics, refresh the branch, and require redacted real Comfy Cloud proof before merge.

## Reproduction Assessment

No live reproduction is present, but source inspection makes the failure path clear: current main keeps polling on `success` or `error` because neither status is terminal. Official Comfy documentation supports the `error`/`error_message` path and documents `success` as an observed actual value.

## Solution Assessment
