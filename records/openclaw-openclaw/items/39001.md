## Summary

Keep open: the Feishu merge_forward gap is real and the patch is directionally correct after the earlier mention-replacement feedback, but the branch currently fails bundled-extension lint and still lacks after-fix real behavior proof from a Feishu/OpenClaw setup.

## What This Changes

This PR updates the Feishu merge_forward formatter to prefix each forwarded sub-message with its sender ID, resolve `@_user_N` placeholders from per-item `mentions`, and adjust the merge_forward regression test.

## Best Possible Solution

Keep the Feishu plugin change, switch mention ordering to lint-compliant `toSorted`, add focused sender/mention regression coverage, and require redacted real Feishu output before merge.

## Reproduction Assessment

Yes, at source level: current main fetches merge_forward items and its parser ignores item `mentions` and sub-message sender data when building lines. I did not verify this with a live Feishu tenant during the read-only review.

## Solution Assessment

Mostly, but not merge-ready as written. Using the existing Feishu formatter plus the per-item sender and mentions map is the narrow owner-local fix, but the patch needs the `toSorted` lint repair and real behavior proof before merge.
