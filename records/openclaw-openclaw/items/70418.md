## Summary

Keep open. Current main still lacks the proposed opt-in, silent Anthropic Cache TTL Warmer; the closed heartbeat-cache work addresses a related path, but this issue remains an explicit product and architecture decision about paid background model calls that bypass conversation semantics.

## What This Changes

Add an opt-in per-agent Cache TTL Warmer that refreshes Anthropic prompt-cache TTL across idle periods without using heartbeat delivery, transcript, or channel semantics.

## Best Possible Solution

Keep the design issue open until maintainers choose the ownership model; an approved implementation should be explicit opt-in, reuse stable conversation-prefix construction, discard responses, avoid tool execution and transcript/channel/memory writes, and document cost.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/design proposal. The absence check is high-confidence from current source, generated schema, docs, runtime, and synonym searches on main.

## Solution Assessment

Unclear because there is no proposed patch and the product boundary is unsettled. A dedicated warmer may be the right end state, but the safest path depends on maintainer policy for paid background calls and whether core should expose a new plugin/runtime seam.
