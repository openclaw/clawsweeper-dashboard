## Summary

Keep open. Current main still suppresses ordinary final replies in message-tool-only group/topic turns, Telegram suppresses its fallback in that mode, and the requested guard or diagnostic is not merged on main.

## What This Changes

The issue requests runtime enforcement or diagnostics when Telegram forum-topic group sessions configured for message-tool-visible replies finish with substantive final text but no successful message-tool send.

## Best Possible Solution

Land or supersede the shared suppressed-delivery diagnostic work, then separately decide whether Telegram should add an opt-in auto-repair while preserving private-by-default group behavior.

## Reproduction Assessment

Yes, at source level. Current main resolves the reported group/topic mode to message_tool_only, suppresses ordinary final delivery, and Telegram disables its fallback for that mode; the live comments add current-release Telegram evidence, but I did not run a live topic repro.

## Solution Assessment

No for automatic cleanup. A shared diagnostic is the narrowest maintainable first step, while blocking or auto-sending final text would change the documented privacy/tool-only contract and needs maintainer direction.
