## Summary

Keep open. Current main still permits this behavior: group/channel turns in `message_tool_only` mode suppress normal final delivery, and Telegram suppresses its fallback in that mode, so the requested guard, warning, or auto-repair is not implemented.

## What This Changes

Request a runtime guard or diagnostic for Telegram forum-topic turns where `visibleReplies="message_tool"` and the assistant emits final text without using `message(action=send)`.

## Best Possible Solution

Add a cross-channel delivery-policy diagnostic at the shared reply-dispatch or agent-run boundary, then consider Telegram auto-promotion only after maintainers choose how it should interact with the tool-only privacy contract.

## Reproduction Assessment

Yes at source level: current main resolves group/channel replies to `message_tool_only`, skips ordinary final payload delivery, and Telegram suppresses the silent fallback. I did not run a live Telegram forum-topic reproduction.

## Solution Assessment

No for automatic cleanup: the requested guard is valid but the best fix needs a maintainer choice between warning-only diagnostics and behavior-changing auto-repair. A narrow diagnostic first is the safest likely direction.
