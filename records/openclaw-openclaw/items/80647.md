## Summary

Keep open. Current main still permits the Telegram/forum-topic symptom: normal final text is private in `message_tool_only` mode, Telegram suppresses its fallback there, and the added 2026.5.7 live reproduction shows this is still an operator-visible gap rather than a closed transport issue.

## What This Changes

The issue requests a runtime guard, diagnostic, or auto-repair for Telegram group/forum-topic turns where `visibleReplies="message_tool"` and the assistant emits final text without calling the message tool.

## Best Possible Solution

Decide the shared visible-reply policy with the https://github.com/openclaw/openclaw/issues/77320 cluster, then add a scoped diagnostic for suppressed non-empty finals with no successful message-tool delivery before considering any auto-repair.

## Reproduction Assessment

Yes at source level: current main routes Telegram forum-topic turns through group chat context, resolves them to `message_tool_only`, skips ordinary final payload delivery, and suppresses Telegram's fallback. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

Unclear for implementation: a diagnostic guard is the narrowest likely improvement, while auto-repair or fallback delivery needs explicit maintainer policy because it changes tool-only privacy semantics.
