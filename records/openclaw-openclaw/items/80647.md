## What This Changes

The issue requests a runtime guard, repair, or diagnostic when Telegram group/forum-topic sessions in `message_tool` visible-reply mode finish with substantive private final text and no successful message-tool send.

## Best Possible Solution

Preserve the `message_tool` privacy contract, but land a shared, default-visible delivery-policy diagnostic for suppressed substantive finals; treat auto-repair or fallback delivery as a separate product decision.
