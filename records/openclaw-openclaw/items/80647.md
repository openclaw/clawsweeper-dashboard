## Summary

Keep open: current main still suppresses ordinary final text in Telegram group/topic `message_tool_only` turns, and the open linked PR with closing syntax is the active implementation path for diagnostics.

## What This Changes

Add a runtime guard, diagnostic, or repair path so Telegram forum-topic turns in `message_tool` visible-reply mode do not complete silently when final text is private.

## Best Possible Solution

Review and land the linked diagnostic PR if it matches the desired policy, then track any auto-repair behavior as a separate product decision.

Yes, at source level: group/topic turns resolve to `message_tool_only`, generic dispatch skips ordinary final payloads, and Telegram suppresses its fallback in that mode. I did not run a live Telegram bot/topic fixture in this review.

## Solution Assessment

Yes for keeping this issue open and paired with the open diagnostic PR: a diagnostic preserves the existing privacy contract while making the failure visible. Auto-promoting final text would be a broader policy change.
