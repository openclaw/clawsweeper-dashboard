## Summary

No discrete correctness defect is evident, but merge still needs explicit owner acceptance of the compatibility change and after-fix proof from a real gateway/runtime setup rather than an in-process synthetic harness.

## What This Changes

The PR restricts run-scoped `sessions_send` durable fallback to structurally valid cron runs with `not_streaming` or `no_active_run` outcomes, while returning explicit errors for unsafe queue failures and preserving ordinary session gateway routing.

## Best Possible Solution
