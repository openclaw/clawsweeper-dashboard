## Summary

## What This Changes

The branch narrows `sessions_send` active-run retry routing, preserves only the cron durable-parent fallback for stranded cron run keys, and adds focused regression coverage for unsafe rejection, source-reply mismatch, ordinary active sessions, and non-cron run-looking keys.

## Best Possible Solution

Merge the narrowed `sessions_send` boundary fix only after maintainers accept that rejected non-cron and unsafe run-scoped active delivery should fail explicitly while preserving the shipped stranded-cron parent fallback.

## Reproduction Assessment

## Solution Assessment

Yes, with a maintainer tradeoff: this is a narrower owner-boundary fix than deleting all fallback behavior because it preserves the shipped cron parent fallback and restores ordinary active-session gateway routing. The remaining question is whether maintainers accept explicit errors for the blocked fallback cases.
