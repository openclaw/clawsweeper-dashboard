## Summary

The PR is still necessary: current `main` retains the broad durable-parent reroute introduced for stranded cron pingbacks, while this focused patch prevents unsafe run-scoped queue failures from crossing into durable sessions and preserves the established cron and ordinary-session paths. No discrete correctness defect remains, but merging intentionally changes fallback compatibility and therefore needs owner acceptance.

## What This Changes

The branch limits fire-and-forget `sessions_send` active-queue delivery to run-scoped targets, permits durable-parent fallback only for stranded cron runs with `not_streaming` or `no_active_run`, and expands regression coverage for rejected, mismatched, ordinary, cron, and non-cron routing cases.

## Best Possible Solution

Land the narrow owner-boundary fix after an experienced sessions-runtime owner accepts the explicit-failure contract, preserving only the structurally valid stranded-cron fallback and ordinary gateway routing.

## Reproduction Assessment

Yes for the implicated routing defect: current source and the supplied production-tool harness establish that unsuccessful run-scoped active delivery can enter the durable gateway path. The reporter's complete intermittent fleet corruption sequence was not reproduced.

## Solution Assessment

Yes, conditionally: narrowing the existing retry at the `sessions_send` owner boundary is the cleanest bounded fix and preserves the known cron exception, but the intentionally stricter fallback contract requires maintainer ownership.
