## Summary

Keep open: current main and v2026.6.6 still lack a supported durable guard for Dreaming session transcript ingestion. QMD memory search has opt-in session indexing and policy diagnostics, but Dreaming still calls session transcript ingestion during light dreaming and the plugin schema does not expose the existing source-list field as supported config.

## What This Changes

The issue requests a supported setting or durable sentinel to disable session transcript ingestion into Dreaming/QMD during staged QMD migrations.

## Best Possible Solution

Memory-core should expose and honor a supported ingestion policy, likely by documenting/testing `dreaming.phases.light.sources` or an equivalent disable switch, while coordinating narrower cron/archive fixes with https://github.com/openclaw/openclaw/issues/72611 and https://github.com/openclaw/openclaw/issues/90313.

## Reproduction Assessment

Yes, at source level. Current main resolves a light-dreaming `sources` list but still calls session transcript ingestion unconditionally, and the supported plugin schema does not expose a session-ingestion disable field.

## Solution Assessment

Unclear until maintainers choose the contract. A local sentinel is not the best upstream shape by itself; the maintainable path is a memory-core-owned supported config or policy surface with tests and docs.
