## Summary

Keep open: current main and v2026.6.9 still have Dreaming session transcript ingestion without a supported migration-wide disable guard, and the live related PRs cover archive/cron pollution rather than this operator off switch.

## What This Changes

Request a supported memory-core Dreaming guard or config/sentinel to disable session transcript ingestion during QMD migrations so session-corpus and short-term recall state stay clean.

## Best Possible Solution

Expose one canonical operator-controlled Dreaming session-ingestion disable path, preserving current defaults and adding schema/docs/tests that prove it prevents session-corpus writes and recall promotion during migrations.

## Reproduction Assessment

Yes, at source level. Current main still calls Dreaming transcript ingestion from light and REM phases, writes session-corpus records, and has a test proving memorySearch being disabled does not stop that path; I did not run live validation in this read-only review.

## Solution Assessment

Unclear as a final product shape. The operational need is valid, but maintainers should choose a canonical config or policy guard rather than preserving an ad hoc installed-runtime sentinel.
