## Summary

Keep open: current main and the latest release still ingest session transcripts into `memory/.dreams/session-corpus` when dreaming runs, and the supported schema/docs do not expose the requested durable per-ingestion guard.

## What This Changes

The issue requests a supported config switch or durable sentinel to disable session transcript ingestion into dreaming/QMD during staged QMD migrations.

## Best Possible Solution

Add a canonical memory-core dreaming session-ingestion disable control with schema/docs/status or doctor coverage, preserving current defaults and proving package updates keep the guard effective.

## Reproduction Assessment

Yes, source-level: enable memory-core dreaming with a session transcript and trigger light dreaming; existing tests assert `session-ingestion.json`, `session-corpus/2026-04-05.txt`, and the transcript snippets appear.

## Solution Assessment

Unclear as a final implementation: the local sentinel patch proves the operational need, but upstream should choose one canonical config or sentinel contract rather than preserving an installed-runtime workaround.
