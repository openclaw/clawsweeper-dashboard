## Summary

Keep open: current main still has Dreaming session transcript ingestion enabled by default, the latest release still lacks the requested durable disable guard, and the related open PRs address archive/cron filtering rather than this migration-wide operator switch.

## What This Changes

Request a first-class memory-core setting or durable sentinel to disable Dreaming/QMD session transcript ingestion during staged QMD migrations.

## Best Possible Solution

Choose one canonical memory-core guard, preferably by extending an approved data-handling or Dreaming config contract, then add schema/docs/tests proving it prevents session-corpus writes and short-term recall ingestion while preserving current default behavior.

## Reproduction Assessment

Yes, at source level. Current main still calls Dreaming session transcript ingestion from light and REM phases, and current tests show this ingestion still happens when memorySearch is disabled; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as a final product shape. The operational need is valid, but maintainers should choose a canonical config or policy contract rather than preserving an ad hoc installed-runtime sentinel.
