## Summary

Keep open: current main and v2026.6.9 still run Dreaming session transcript ingestion without a supported migration-wide disable guard, and the closest related PRs cover archive/cron/chatter filtering rather than this operator off switch.

## What This Changes

Issue requests a supported config switch or durable sentinel to disable Dreaming/QMD session transcript ingestion during staged memory migrations.

## Best Possible Solution

Choose one canonical memory-core guard, preferably by extending an approved data-handling or Dreaming config contract, then add schema/docs/tests proving it prevents session-corpus writes and short-term recall ingestion while preserving current default behavior.
