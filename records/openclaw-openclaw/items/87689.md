## Summary

Keep open: current main and the latest release still run Dreaming session transcript ingestion from the light and REM phases without a supported migration-wide disable guard, while the closest related work only addresses archive/cron filtering rather than this durable operator switch.

## What This Changes

Request a durable memory-core Dreaming control to disable session transcript ingestion during QMD migrations so session-corpus and recall/promotion state stay clean across package updates.

## Best Possible Solution

Add one canonical memory-core Dreaming session-ingestion disable control, or fold the same outcome into an approved sessionFilter policy, with schema, docs, tests, and defaults that preserve current behavior.

## Reproduction Assessment

Yes, at source level: current main still calls Dreaming transcript ingestion from light and REM phases, writes session-corpus files, and has no supported disable key in schema or docs. I did not run a live QMD migration workspace because this review is read-only.

## Solution Assessment

Unclear as a final product shape: the operational need is valid, but maintainers should choose a canonical config, sentinel, or broader sessionFilter contract instead of preserving a local installed-runtime workaround.
