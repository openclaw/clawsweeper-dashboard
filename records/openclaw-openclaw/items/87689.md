## Summary

Keep open: current main and v2026.6.8 still ingest session transcripts into Dreaming without a supported per-ingestion disable guard, and the closest related work addresses archive/cron filtering rather than the requested migration-wide off switch.

## What This Changes

The issue requests a supported Dreaming setting or durable sentinel to disable session transcript ingestion into the QMD/dreaming corpus during staged migrations.

## Best Possible Solution

Add one canonical memory-core Dreaming session-ingestion disable control, or fold the same outcome into an approved sessionFilter policy, with schema, docs, tests, and defaults that preserve current behavior.
