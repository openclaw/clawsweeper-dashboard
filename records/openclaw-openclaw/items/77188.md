## Summary

Keep open: the feature addresses the linked multi-agent cron-list noise, but the patch still has a source-reproducible filtering bug and is missing the required user-facing docs/changelog/test coverage.

## What This Changes

The PR adds an optional `agentId` filter for `cron.list` across the protocol schema, cron service, gateway handler, agent cron tool, CLI, and generated Swift protocol models.

## Best Possible Solution

Land the focused cron-list feature after the service filter uses the same normalized/default-agent semantics as cron creation/runtime and after docs, changelog, and regression tests cover the new behavior.

## Reproduction Assessment

Yes. Source inspection shows current main has no `agentId` field in `cron.list`, and the PR defect is reproducible from the diff because normalized/default cron job agent IDs are compared against the raw requested filter.

## Solution Assessment

No, not yet. The API shape is reasonable, but the maintainable fix should normalize/coalesce the filter value consistently and ship with service, gateway/protocol, tool, CLI, docs, and changelog coverage.

Overall correctness: patch is incorrect
