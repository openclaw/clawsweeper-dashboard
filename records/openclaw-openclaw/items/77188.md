## Summary

Keep open: the feature gap is real and this PR is the canonical implementation path for #77118, but current main still lacks the filter and the PR branch needs a narrow repair before automerge.

## What This Changes

The PR adds optional `agentId` filtering to `cron.list` across the gateway protocol, cron service paging, gateway handler, agent cron tool, `openclaw cron list --agent`, and generated Swift protocol models.

## Best Possible Solution

Land a focused cron-list agent filter after normalizing requested agent IDs at the service boundary and updating protocol, tool, CLI, docs, changelog, generated models, and regression coverage together.

## Reproduction Assessment

Yes. Source inspection is enough for the PR defect: stored cron job agent IDs are normalized on create/update, while the PR filter compares `opts.agentId` directly to `job.agentId`; current main also has no list-side agent filter.

## Solution Assessment

No, not yet. The API shape is reasonable, but the maintainable fix should normalize the requested filter value before comparison and ship with focused tests, docs, changelog, and protocol generated output.
