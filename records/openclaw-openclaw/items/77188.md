## Summary

Keep open: the PR addresses the real cron filtering gap from #77118, but it is not merge-ready because the new service filter compares raw agent IDs against normalized stored IDs and the user-facing feature lacks required docs/changelog coverage.

## What This Changes

The PR adds an optional `agentId` filter for `cron.list` through the protocol, service, gateway handler, agent cron tool, CLI, and generated Swift protocol models.

## Best Possible Solution

Land a focused version that normalizes the requested list filter at the service boundary, preserves backward-compatible unfiltered CLI behavior, and ships with regression tests plus cron docs/changelog coverage.

## Reproduction Assessment

Yes. Source inspection shows current main has no `agentId` parameter for `cron.list`, and the PR defect is source-reproducible because stored cron agent IDs are sanitized while the new filter compares the raw requested value.

## Solution Assessment

No, not yet. The API direction is reasonable, but the maintainable fix should normalize the requested agent ID before comparing and include focused tests, docs, and changelog coverage.
