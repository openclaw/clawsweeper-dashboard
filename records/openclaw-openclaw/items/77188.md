## Summary

Keep open; this PR is not ready to merge. It implements the requested cron list surface, but the exact head still compares the requested `agentId` raw against normalized stored IDs and lacks the required docs, changelog, and focused regression coverage.

## What This Changes

The PR adds an optional `agentId` filter to cron list across the protocol schema, cron service, gateway handler, agent cron tool, CLI, and generated Swift protocol models.

## Best Possible Solution

Land this focused cron list feature after the filter shares the existing agent ID normalization contract and the user-facing behavior is covered in tests, docs, and the changelog; then the linked issue can close through the merge.

## Reproduction Assessment

Yes. Source inspection of current main shows `cron.list` has no `agentId` parameter, and source inspection of the PR shows a mixed-case or whitespace-padded filter would miss a job stored with the normalized agent ID.

## Solution Assessment

No, not yet. The API shape is reasonable, but the maintainable fix needs to normalize the requested filter value consistently with add/edit and ship with regression coverage plus docs/changelog updates.
