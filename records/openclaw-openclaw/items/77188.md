## Summary

Keep open: the branch addresses a real cron multi-agent list-filter gap, but it is not ready to merge because the proposed filter compares raw agent IDs while existing create/edit paths canonicalize them, and the user-facing docs/changelog/tests are missing.

## What This Changes

The PR adds an optional agentId filter to cron.list across the protocol, service, gateway handler, agent tool, and CLI --agent option.

## Best Possible Solution

Land the feature after normalizing the requested list agentId consistently and adding focused regression coverage, docs, and a changelog entry.

## Reproduction Assessment

Yes for the PR defect by source inspection: current create/update paths normalize agent IDs, while the proposed list filter compares the requested value verbatim. Current main also source-reproduces the linked feature gap because cron.list has no agentId parameter.

## Solution Assessment

No, not yet: the API shape is reasonable, but the maintainable fix should canonicalize the filter value through the same agent-id normalization path and ship tests/docs/changelog with the user-facing feature.
