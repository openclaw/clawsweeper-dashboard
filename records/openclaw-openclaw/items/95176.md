## Summary

Keep open: the patch is aimed at the right guarded model transport layer and I found no discrete line-level correctness defect, but it still needs real affected-path proof and focused retry-path coverage before merge because it changes shared provider fallback behavior.

## What This Changes

The branch adds an exported `UND_ERR_SOCKET` detector, retries `buildGuardedModelFetch` once on that error with local-service lease cleanup, and adds detector unit tests.

## Best Possible Solution

Land the guarded-transport retry only after focused retry-path coverage and redacted affected-environment proof confirm it prevents same-provider socket failures without breaking fallback behavior.

## Reproduction Assessment

Yes for the source-level path: current main routes Anthropic Messages through guarded fetch, rethrows pre-response `UND_ERR_SOCKET`/`fetch failed`, and existing failover classification treats that as timeout fallback. I did not live-reproduce the multi-hour stale-socket trigger in this read-only review.

## Solution Assessment

Yes, this is likely the best layer: a bounded retry in guarded model transport preserves SSRF, proxy, local-service, and logging policy better than the older raw SDK dispatcher proposal. The remaining gaps are proof and focused retry-path coverage, not a different implementation layer.
