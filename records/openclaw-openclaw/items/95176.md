## Summary

Keep open: this is a plausible guarded-transport fix for the linked Anthropic socket fallback bug, but the branch still lacks affected-path real behavior proof and appears too narrow for the linked issue's code-less `fetch failed` repro.

## What This Changes

The PR adds an `UND_ERR_SOCKET` detector, retries `buildGuardedModelFetch` once on that error, releases local-service leases around the retry, and adds detector unit tests.

## Best Possible Solution

Land a guarded-transport same-provider retry only after it covers the intended transient pre-response failure scope and has focused retry-path coverage plus redacted affected-environment proof.

## Reproduction Assessment

Yes for the source-level path: current main routes Anthropic Messages through guarded fetch, rethrows pre-response fetch failures, and classifies fetch/network failures as timeout fallback attempts. I did not live-reproduce the multi-hour stale-socket trigger in this read-only review.

## Solution Assessment

No as currently scoped: guarded transport is the right layer, but the `UND_ERR_SOCKET`-only condition can leave the linked bare `fetch failed` repro unresolved. The safer solution is a bounded retry for the agreed transient pre-response failure shape with focused coverage and real gateway proof.
