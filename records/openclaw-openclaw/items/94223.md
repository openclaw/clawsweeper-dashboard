## Summary

Keep open; the reported Codex reply bug is real, but this PR is not merge-ready because failed turns with only partial streamed deltas would now resolve as successful replies, and the supplied proof does not exercise the recovered failed-turn path.

## What This Changes

The PR changes Codex conversation reply collection to prefer collected assistant text over a failed-turn error, adds collector regressions, and adds `ps` to the read-only shell-command classifier.

## Best Possible Solution

Keep the PR open, revise the collector so failed-turn recovery requires completed assistant-message evidence rather than any streamed delta, and then add redacted real behavior proof for both recovered and unrecovered failed turns.

## Reproduction Assessment

Yes, source-level: current main rejects failed Codex conversation turns before returning collected assistant text, and upstream Codex can mark a turn failed from sticky `last_error`. I did not run a live failed-turn reproduction because this read-only review cannot execute artifact-producing tests or inject failures into the reporter's Slack runtime.

## Solution Assessment

No. The collector is the right owner boundary, but the implementation is too broad because streamed deltas are progress, not final assistant-message completion; the safer fix should track completed assistant text separately from in-progress deltas.
