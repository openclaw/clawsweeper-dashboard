The PR changes the Codex conversation turn collector to prefer recovered assistant text over failed-turn errors, adds collector and agent warning tests, marks `ps` shell probes as read-only, and adds an internal handoff note.

## Best Possible Solution

Revise the collector so failed-turn recovery requires completed assistant-message evidence, keep streamed-delta-only failed turns as errors, and add redacted real behavior proof for recovered and unrecovered failed turns.

Yes. Source-level reproduction is high-confidence: a current-main collector given `status: failed` plus final `agentMessage` text rejects before returning the text, while the PR's added test shows the new unsafe partial-delta-only failed-turn path.

## Solution Assessment

No. The collector is the right layer for this Slack/Codex reply bug, but the current implementation is too broad because streamed deltas are progress, not completed assistant-message proof.
