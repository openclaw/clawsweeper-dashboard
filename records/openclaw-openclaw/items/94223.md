## Summary

Keep open: the collector is the right boundary for the Slack-visible Codex reply, but this branch currently treats any collected text, including partial streamed deltas, as recovered output and can hide a true failed turn. The supplied proof shows a patched bundle loaded and gateway restart, not the recovered failed-turn behavior itself.

## What This Changes

The branch changes the Codex conversation turn collector to return collected assistant text for failed turns with text, adds collector regressions, classifies `ps` shell probes as read-only, and adds an ops handoff note.

## Best Possible Solution

Revise the collector to separate completed assistant-message text from streamed deltas, suppress failed-turn errors only when completed assistant evidence exists, preserve unrecovered failures, and add redacted real behavior proof for both recovered and unrecovered failed turns.

## Reproduction Assessment

Yes, source-level. Current main rejects failed Codex conversation turns before returning collected assistant text, and sibling Codex source shows failed status can coexist with completed assistant-message data; I did not run a live recovered failed-turn reproduction.

## Solution Assessment

No. The collector is the right layer, but the implementation is too broad because streamed deltas are progress rather than completed assistant-message proof; the safer fix should track completed assistant text separately from in-progress deltas.
