## Summary

Keep open: the collector is the right boundary, but the patch is unsafe as submitted because it lets any collected streamed delta suppress a failed-turn error. The supplied live output shows the hotfixed bundle loaded and restarted, but it does not exercise the recovered or unrecovered failed-turn behavior.

## What This Changes

This PR changes the Codex conversation turn collector to resolve failed turns with collected assistant text, adds collector regressions and a handoff note, and classifies `ps` shell probes as read-only.

## Best Possible Solution

Track completed assistant-message text separately from streamed deltas, only forgive failed turns when a completed agentMessage or turn snapshot supplies final text, and add redacted real behavior proof for recovered and unrecovered failed turns.

## Reproduction Assessment

Yes, source-level: current main rejects failed Codex conversation turns before returning collected assistant text, and upstream Codex can mark a turn failed from sticky `last_error`. I did not run a live failed-turn reproduction because this read-only review cannot inject failures into the reporter's Slack runtime.

## Solution Assessment

No. The collector is the right owner boundary, but the implementation is too broad because streamed deltas are progress rather than completed assistant-message evidence.
