## Summary

Keep open: the collector is the right owner boundary, but this patch is not merge-ready because it suppresses failed-turn errors whenever any streamed assistant delta exists, and the supplied proof does not exercise the recovered failed-turn behavior.

## What This Changes

The PR changes Codex conversation turn collection to prefer collected assistant replies over failed-turn errors, adds focused tests, classifies `ps` as a read-only shell command, and adds an ops handoff note.

## Best Possible Solution

Keep the PR open, revise the collector so failed-turn recovery requires completed assistant-message evidence rather than any streamed delta, and then add redacted real behavior proof for both recovered and unrecovered failed turns.

## Reproduction Assessment

Yes. Source-level inspection shows current main rejects failed Codex conversation turns before returning collected assistant text, and sibling Codex source can mark a turn failed from sticky `last_error`; I did not run a live failed-turn reproduction in this read-only review.

## Solution Assessment

No. The collector is the right owner boundary, but the implementation is too broad because streamed deltas are progress, not final assistant-message completion; the safer fix should track completed assistant text separately from in-progress deltas.
