## Summary

Keep open: source confirms the stale-banner class of bug is real and the conversation collector is a plausible boundary, but this patch broadens failed-turn recovery to any collected assistant text and the supplied proof does not exercise the changed failed-turn behavior.

## What This Changes

The PR changes Codex conversation turn collection to prefer collected assistant text over failed-turn errors, adds collector and agent-summary regressions, adds `ps` to the read-only shell-command classifier, and adds an implementation handoff note.

## Best Possible Solution

Keep the fix at the conversation-reply collector boundary, but track completed assistant-message evidence separately from streamed deltas and only replace a failed-turn error when completed assistant text exists, unless maintainers explicitly accept delta-only recovery.

## Reproduction Assessment

Yes, source-level: current main rejects failed Codex conversation turns before returning collected assistant text, and sibling Codex source can mark a turn failed from sticky `last_error`. I did not run live Slack/Codex failure injection in this read-only review.

## Solution Assessment

No. The collector is the right owner boundary, but the current implementation is too broad because it treats streamed deltas as recovered final output; the safer fix should require completed assistant-message evidence.
