## Summary

Keep open: current main still has the recovered failed-turn bug, but this PR is not merge-ready because it treats any streamed assistant delta as recovered output and the supplied proof does not exercise the changed behavior.

## What This Changes

The branch changes the Codex conversation turn collector to prefer collected assistant text on failed turns, adds regression tests and a handoff note, and classifies `ps` shell probes as read-only.

## Best Possible Solution

Track completed assistant-message text separately from deltas; only suppress failed-turn errors when `item/completed` or the turn snapshot provides final `agentMessage` text, then add redacted real Slack/Codex proof for recovered and unrecovered failures.

## Reproduction Assessment

Yes, source-level. Current main rejects failed Codex conversation turns before returning collected assistant text, and sibling Codex source can mark a turn failed from sticky `last_error` while still emitting assistant item notifications.

## Solution Assessment

No. The collector is the right owner boundary, but the implementation is too broad because streamed deltas are progress rather than authoritative item completion; completed assistant-message evidence is the safer fix.
