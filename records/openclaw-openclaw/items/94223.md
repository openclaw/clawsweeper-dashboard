## Summary

Keep open: the PR targets a real Codex conversation reply bug, but the patch currently treats partial streamed deltas as recovered final answers and can hide true failed-turn errors; the supplied proof also shows a hotfixed bundle loading, not the recovered failed-turn behavior.

## What This Changes

The PR changes the Codex conversation turn collector to return collected assistant text on failed turns, adds collector regressions, marks `ps` shell probes as read-only, and adds an internal handoff note.

## Best Possible Solution

Revise the collector to require completed assistant-message evidence before overriding failed status, keep partial-delta-only failures as errors, and add redacted real behavior proof for recovered and unrecovered failed turns.

## Reproduction Assessment

Yes, source-level. Current main rejects failed Codex conversation turns before returning collected assistant text, and sibling Codex source shows failed status can coexist with assistant-message data; I did not run a live recovered failed-turn reproduction.

## Solution Assessment

No. The collector is the right layer, but the current implementation is too broad because streamed deltas are progress rather than completed assistant-message proof.
