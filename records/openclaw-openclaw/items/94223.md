## Summary

Keep open: the PR targets a real Codex reply-delivery bug, but the current patch is still too broad because it treats any collected assistant text, including partial streamed deltas, as enough to suppress a failed-turn error.

## What This Changes

The PR changes the Codex conversation turn collector to resolve failed turns with collected reply text, adds recovered-failure regression tests, adds a handoff note, and classifies `ps` shell probes as read-only.

## Best Possible Solution

Revise the collector so failed-turn recovery requires completed assistant-message evidence, preserve unrecovered failure rejection, then add redacted real behavior proof for both recovered and unrecovered failed turns.

## Reproduction Assessment

Yes, source-level: current main rejects failed Codex conversation turns before returning collected assistant text, and upstream Codex can mark turns failed from sticky `last_error`. I did not run a live failed-turn reproduction because this read-only review did not inject failures into a real Slack runtime.

## Solution Assessment

No. The collector is the right owner boundary, but this implementation is too broad because streamed deltas are progress rather than final assistant-message completion; the safer fix should track completed assistant text separately.
