## Summary

Keep open: the collector is the right boundary, but the PR currently treats any collected streamed delta as recovered output, so a true failed Codex turn with partial assistant text could suppress the failure banner and deliver incomplete text. The supplied proof also shows a patched bundle and gateway restart, not the recovered failed-turn behavior itself.

## What This Changes

The branch changes Codex conversation turn collection to resolve failed turns with collected assistant text, adds focused tests and an internal handoff note, and classifies `ps` shell probes as read-only for agent warning payloads.

## Best Possible Solution

Track completed assistant-message evidence separately from streamed deltas, resolve failed turns only when a completed agent message exists, keep no-completed-text failures rejecting, and add redacted real behavior proof for both recovered and unrecovered failed turns.

## Reproduction Assessment

Yes, at source level. Current main rejects failed Codex conversation turns before returning collected assistant text, and sibling Codex source can mark a turn failed from sticky `last_error`; I did not run a live failed-turn reproduction in this read-only review.

## Solution Assessment

No, not as submitted. The collector is the right owner boundary, but streamed deltas are progress rather than completed assistant-message evidence, so the safer fix must distinguish completed text from partial text before suppressing failures.
