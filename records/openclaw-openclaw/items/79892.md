## Summary

Keep open: the patch targets the right daemon/launchd boundary and includes useful live-output evidence, but the restart hook misses the current-process launchd handoff route and the repository real-behavior proof gate is still failing required field content.

## What This Changes

The PR adds a daemon log-retention helper, wires it into macOS launchd install/plist-rewrite/restart paths, adds focused tests, and updates the changelog for bounded gateway stdout/stderr logs.

## Best Possible Solution

Move retention so both direct and detached launchd restart routes rotate before kickstart, keep focused regression coverage for both routes, and reformat the live macOS evidence to satisfy the proof gate.

## Reproduction Assessment

Yes, for the log-retention defect at source level: current main maps launchd stdout/stderr to `logs/gateway.log` and `logs/gateway.err.log` and the plist emits `StandardOutPath`/`StandardErrorPath` with no cap. I did not reproduce the separate CPU loop.

## Solution Assessment

No, not yet: the owner boundary is right, but the proposed restart integration is incomplete because current-process launchd restarts return through the detached handoff before the new retention call. The narrow fix is to apply the same retention before both restart routes or inside the handoff helper.
