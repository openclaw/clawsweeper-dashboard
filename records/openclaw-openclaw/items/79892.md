## Summary

Keep open. The PR is useful and has real macOS proof, but current main now writes macOS LaunchAgent stdout through the supervisor log path under `~/Library/Logs/openclaw` and suppresses stderr to `/dev/null`; the branch still rotates state-dir stdout/stderr, so it would miss the active sink after rebase.

## What This Changes

The PR adds env-configurable rotation for macOS launchd gateway stdout/stderr logs, hooks it into install/plist rewrite/restart paths, adds daemon tests, and updates the changelog.

## Best Possible Solution

Rebase or replace the PR with daemon-owned retention for the current macOS LaunchAgent supervisor stdout sink, and make any legacy stderr cleanup explicit while leaving the linked CPU investigation open.

## Reproduction Assessment

Yes, for the PR blocker by source inspection: current main writes macOS launchd stdout through `resolveGatewaySupervisorLogPaths` and sends stderr to `/dev/null`, while the branch rotates `resolveGatewayLogPaths` stdout/stderr. I did not live-run a macOS gateway in this read-only review.

## Solution Assessment

No as-is. The narrow maintainable fix is to rebase onto current main, rotate the active supervisor stdout path, and either omit stale stderr handling or make it an explicit, separately tested legacy cleanup policy.
