## What This Changes

The PR adds macOS launchd gateway log retention with env-configurable size/archive limits, hooks it into install and restart paths, adds daemon tests, and updates the changelog.

## Best Possible Solution

Land a rebased version that preserves current-main stderr redirection, applies bounded retention where launchd still owns file sinks, and leaves the CPU-loop investigation tracked in https://github.com/openclaw/openclaw/issues/79422.

## Reproduction Assessment

Yes for the log-retention half by source inspection: current main still writes launchd stdout to a fixed `gateway.log` path with no size cap. The original stderr growth path is now partly changed on current main because LaunchAgent stderr is redirected to `/dev/null`.

## Solution Assessment

Yes, with rebase care: rotating launchd-owned file sinks at install/rewrite/restart is the narrow daemon-owner fix. The safer merge path is to keep current-main stderr redirection and treat stderr rotation as stale-log cleanup rather than an active LaunchAgent stderr sink.
