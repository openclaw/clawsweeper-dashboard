## Summary

Keep open: the PR is a focused response to a real daemon log-growth problem and includes real macOS proof, but it is not correct as-is against current main because main now suppresses launchd stderr and the PR also rotates during stage-only plist rewrites that do not reopen launchd file descriptors.

## What This Changes

The PR adds a launchd gateway log-retention helper, hooks it into install/plist-rewrite/restart paths, adds daemon tests, and updates the changelog.

## Best Possible Solution

Rebase or replace the PR so retention only runs at launchd transitions that actually reopen files, targets current launchd-owned stdout plus any explicitly chosen legacy stderr cleanup, and leaves CPU-loop or heartbeat dedupe work to the linked issues.

## Reproduction Assessment

Yes for the remaining launchd stdout retention gap by source inspection: current main writes stdout to a fixed `gateway.log` path without daemon retention. No live current-main reproduction was established for the old active `gateway.err.log` stderr sink because Darwin stderr is now `/dev/null`.

## Solution Assessment

No as-is: the old-base approach needs to be realigned with current launchd stderr handling and should not rotate on stage-only plist rewrites. The narrower path is retention around install/restart transitions that reopen launchd-owned files, with legacy stderr cleanup handled explicitly if maintainers want it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Do not rotate during stage-only rewrites:** `src/daemon/launchd.ts:671`
  - body: `writeLaunchAgentPlist` is also used by `stageLaunchAgent`, including doctor update repair mode. Rotating a live launchd stdout file from that stage-only path does not restart or kickstart the service, so the running process can keep appending to the renamed `.1` archive while the expected active `gateway.log` is missing until a later restart. Move retention out of the stage-only path or gate it to transitions that actually reopen launchd's file descriptors.
  - confidence: 0.82
- **[P2] Align retention with current stderr suppression:** `src/daemon/log-retention.ts:115-118`
  - body: Current main points Darwin LaunchAgent stderr at `/dev/null` and treats `gateway.err.log` as stale, but this helper still rotates `resolveGatewayLogPaths(env).stderrPath` and the tests/changelog describe it as an active launchd-owned stderr sink. Please rebase and either restrict active retention to stdout or add separately named, tested legacy cleanup so the PR does not imply launchd still writes stderr there.
  - confidence: 0.86
