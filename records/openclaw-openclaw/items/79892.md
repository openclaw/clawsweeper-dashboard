## Summary

Keep open: the PR is useful but not merge-ready because current main now sends macOS launchd stderr to `/dev/null` while the branch still rotates and describes `gateway.err.log` as an active launchd-owned stderr sink.

## What This Changes

Adds daemon log-retention helpers, hooks them into macOS launchd install/rewrite/restart paths, adds daemon tests, and updates the changelog for gateway stdout/stderr log rotation.

## Best Possible Solution

Rebase on current main, retain only daemon log retention that matches actual launchd-owned files, explicitly scope any legacy `gateway.err.log` cleanup, and leave CPU-loop or heartbeat dedupe work to the related issues.

## Reproduction Assessment

Yes for the remaining daemon stdout retention gap by source inspection: current main writes launchd stdout to a fixed `gateway.log` path with no retention hook. No for the original active Darwin stderr sink, because current main sets launchd stderr to `/dev/null`.

## Solution Assessment

No as-is: the old-base retention approach was reasonable when launchd wrote stderr to `gateway.err.log`, but current main needs the branch rebased and narrowed to the log files launchd actually owns. A safer path is stdout retention plus explicit, separately tested legacy stderr cleanup if maintainers want it.
