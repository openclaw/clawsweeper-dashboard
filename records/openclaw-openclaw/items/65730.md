## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate punctuation shaping on short follow-ups:** `src/auto-reply/reply/chat-shaping.ts:25`
  - body: `shouldApplyChatShaping` returns true for any message ending in `?` or `!` after only broad deny-lists and a 1200-character cap. Detailed non-follow-up questions can therefore receive persona chat-shaping guidance, which is broader than this PR's stated scope.
  - confidence: 0.86
- **[P2] Treat channel sessions as shared chats:** `src/auto-reply/reply/get-reply-run.ts:314`
  - body: The PR head derives `isGroupChat` from only `ChatType === "group"`, so channel sessions bypass the no-nearby-context guard used for shared chats. That increases false-positive shaping in multi-user channels and also conflicts with current main's broader group/channel handling.
  - confidence: 0.8
- **[P2] Count thread starters as nearby group context:** `src/auto-reply/reply/chat-shaping.ts:66`
  - body: For group chats, this early return ignores `threadStarterBody` even though the later nearby-context check includes it. First replies in a thread with only starter context will skip shaping despite having nearby context.
  - confidence: 0.84
- **[P2] Select semantic content before metadata:** `src/auto-reply/reply/deictic-context.ts:109`
  - body: `pushCandidate` keeps the last non-empty line, but Slack thread history appends a `[slack message id: ...]` metadata line after each message. The resolver can present that ID line as the referent instead of the message content.
  - confidence: 0.9
- **[P2] Require nearby context for bare English pronouns:** `src/auto-reply/reply/deictic-context.ts:26`
  - body: The direct-reference regex matches ordinary short sentences starting with `it`, `this`, or `that` even when no reply or thread context exists. That can inject a resolver note for plain updates and bias replies toward ambiguity handling.
  - confidence: 0.84
- **[P3] Remove the unbounded json matcher:** `src/auto-reply/reply/chat-shaping.ts:13`
  - body: The extra `/json/u` matcher duplicates the word-boundary JSON check but also matches substrings like `jsonl` or `jsonify`, suppressing chat shaping for messages that are not structured JSON-output requests.
  - confidence: 0.8
