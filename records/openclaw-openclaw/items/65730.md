## Summary

Keep this PR open. Current main still does not implement the intended thread-aware follow-up behavior, but the proposed patch has blocking correctness and prompt-safety issues that need maintainer review before merge.

## What This Changes

The PR adds auto-reply deictic-context and chat-shaping prompt helpers, wires them into reply prompt construction, updates memory-core recall guidance, adds focused tests, and adjusts a Discord component test import.

## Best Possible Solution

Land a revised thread-aware follow-up implementation that keeps nearby reply/thread context primary, narrows deictic and chat-shaping triggers to true short follow-ups, labels candidate context as untrusted, preserves shared-channel behavior, covers Slack metadata and thread-starter edge cases, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. The remaining PR issues are reproducible by static inspection of pure predicates and prompt assembly on the proposed diff: detailed punctuation-ended prompts shape, group thread-starter-only context is skipped, bare English pronoun statements trigger without nearby context, and Slack metadata can become the selected candidate line.

## Solution Assessment

No. The direction is plausible, but this patch is not yet the best maintainable fix because it over-applies shaping, misses a nearby-context case, can select transport metadata as the referent, and needs explicit untrusted-boundary wording for prompt safety.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate punctuation-only shaping on short follow-ups:** `src/auto-reply/reply/chat-shaping.ts:25`
  - body: `shouldApplyChatShaping` treats any message ending in `?` or `!` as conversational after only the 1200-character cap and deny-lists. Detailed questions that are not short follow-ups can therefore get persona/style guidance, which is broader than the PR's stated scope.
  - confidence: 0.86
- **[P2] Honor thread starters in the group shaping guard:** `src/auto-reply/reply/chat-shaping.ts:66`
  - body: For shared chats, this early return ignores `threadStarterBody` even though the later nearby-context check includes it. First replies in a thread that only have starter context will skip chat shaping despite having the nearby context this helper is meant to use.
  - confidence: 0.84
- **[P2] Select semantic thread text before metadata:** `src/auto-reply/reply/deictic-context.ts:109`
  - body: `pushCandidate` keeps the last non-empty line, but Slack thread history appends a `[slack message id: ...]` metadata line after each message. The resolver can present that ID line as the referent instead of the message content, degrading the new deictic behavior.
  - confidence: 0.9
- **[P2] Require nearby context for bare pronoun deictics:** `src/auto-reply/reply/deictic-context.ts:26`
  - body: The direct-reference pattern matches ordinary short sentences starting with `it`, `this`, or `that` even when no reply or thread context exists. Messages like `It still fails after reboot.` will get a resolver note with no nearby candidates, biasing the reply toward unnecessary ambiguity handling.
  - confidence: 0.84
- **[P3] Remove the unbounded json matcher:** `src/auto-reply/reply/chat-shaping.ts:13`
  - body: The extra `/json/u` matcher duplicates the word-boundary JSON check but also matches substrings like `jsonl` and `jsonify`. That can suppress chat shaping for messages that are not actually structured JSON-output requests.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `extensions/memory-core/src/prompt-section.ts:17`
  - body: This is a user-facing auto-reply and memory prompt fix, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active Fixes section and credit the contributor so release notes cover the behavior change.
  - confidence: 0.78
