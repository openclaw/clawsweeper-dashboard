## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate punctuation shaping on short follow-ups:** `src/auto-reply/reply/chat-shaping.ts:25`
  - body: `shouldApplyChatShaping` returns true for any message ending in `?` or `!` after only broad deny-lists and a 1200-character cap. Detailed non-follow-up questions can therefore receive persona chat-shaping guidance, which is broader than this PR's stated scope.
  - confidence: 0.86
- **[P2] Treat channel chats as shared context:** `src/auto-reply/reply/get-reply-run.ts:547`
  - body: The new call passes `isGroupChat`, but the PR head derives that flag from only `ChatType === "group"` and excludes `channel` sessions. Shared channel messages therefore bypass the no-nearby-context guard that group chats use, increasing false-positive shaping in multi-user channels.
  - confidence: 0.81
- **[P2] Count thread starters in the shared-chat guard:** `src/auto-reply/reply/chat-shaping.ts:66`
  - body: For group chats, this early return ignores `threadStarterBody` even though the later nearby-context check includes it. First replies in a thread with only starter context will skip shaping despite having nearby context.
  - confidence: 0.84
- **[P2] Select semantic thread text before metadata:** `src/auto-reply/reply/deictic-context.ts:109`
  - body: `pushCandidate` keeps the last non-empty line, but Slack thread history appends a `[slack message id: ...]` metadata line after each message. Short follow-ups can therefore resolve against transport metadata instead of the actual referent text.
  - confidence: 0.82
- **[P2] Require nearby context for bare English pronouns:** `src/auto-reply/reply/deictic-context.ts:26`
  - body: The first direct-reference pattern matches any short sentence starting with `this`, `that`, or `it`, so messages like `It still fails after reboot.` are treated as deictic without reply or thread context. That injects ambiguity-handling guidance into ordinary status updates.
  - confidence: 0.83
- **[P2] Mark resolver candidates as untrusted:** `src/auto-reply/reply/deictic-context.ts:119`
  - body: The resolver note promotes reply and thread text as primary referents without explicitly labeling those candidates as untrusted user/channel content. Since this prefix is prepended ahead of the message body, prompt-injection resistance depends too much on model behavior.
  - confidence: 0.8
- **[P3] Remove the duplicate loose json matcher:** `src/auto-reply/reply/chat-shaping.ts:13`
  - body: The loose `/json/u` matcher is redundant with the bounded JSON/YAML/XML/CSV/SQL pattern but also matches substrings like `jsonl` and `jsonify`, which can suppress chat shaping for unrelated words.
  - confidence: 0.78
