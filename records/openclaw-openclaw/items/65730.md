## Summary

Keep this PR open. Current main still does not implement the proposed thread-aware deictic resolver, chat-shaping prompt helper, or memory-core thread-first recall guidance, but the PR head is not ready to merge because it introduces prompt-behavior false positives, misses shared-channel/thread-starter cases, can choose transport metadata as a referent, and needs prompt-safety tightening.

## What This Changes

The PR adds deictic-context and chat-shaping prompt helpers for auto-reply follow-ups, updates memory-core recall guidance to prefer nearby thread/session context, adds focused tests, and updates a Discord component test import.

## Best Possible Solution

Land a revised thread-aware follow-up implementation that keeps nearby context primary, narrows resolver and chat-shaping triggers to true short follow-ups, treats all reply/thread candidate text as explicitly untrusted, preserves current shared-channel behavior, and covers Slack/Discord/thread-starter edge cases plus memory-core prompt guidance with focused tests and a changelog entry.

## Reproduction Assessment

Yes. The problematic PR behaviors are reproducible by static inspection of pure predicates and prompt assembly: a detailed message ending in `?`, a first thread reply with only `threadStarterBody`, a channel conversation, and Slack thread history with a trailing metadata line all follow directly from the PR head source.

## Solution Assessment

No. The overall direction is plausible, but this patch is not the narrowest maintainable solution yet because it over-applies shaping, misses current shared-channel and thread-starter context, can extract transport metadata as the referent, and needs explicit untrusted-boundary wording for prompt safety.

Full review comments:

- **[P2] Gate punctuation-only shaping on short follow-ups:** `src/auto-reply/reply/chat-shaping.ts:25`
  - body: The punctuation signal makes any message ending in `?` or `!` apply the chat-shaping note after only the 1200-character cap. Detailed questions that are not short conversational follow-ups can therefore get persona/style guidance, which is broader than the PR's stated scope.
  - confidence: 0.86
- **[P2] Treat channel chats as shared context for shaping:** `src/auto-reply/reply/get-reply-run.ts:547`
  - body: The PR head still computes `isGroupChat` only for `ChatType === "group"`, but the new shaping guard relies on that flag. Shared channel sessions can therefore bypass the no-nearby-context guard and get chat-shaping guidance even when they should be treated like group conversations.
  - confidence: 0.82
- **[P2] Honor thread starters in the group shaping guard:** `src/auto-reply/reply/chat-shaping.ts:66`
  - body: For shared chats, this early return ignores `threadStarterBody`, even though the later nearby-context check includes it. First replies in a thread that only have starter context will skip chat shaping despite having the nearby context the helper is meant to use.
  - confidence: 0.84
- **[P2] Select semantic thread text before metadata:** `src/auto-reply/reply/deictic-context.ts:109`
  - body: `pushCandidate` always keeps the last non-empty line, but Slack thread history appends a `[slack message id: ...]` metadata line after each message. The resolver can present that ID line as the referent instead of the message content, degrading the new deictic behavior.
  - confidence: 0.9
- **[P2] Require nearby context for bare pronoun deictics:** `src/auto-reply/reply/deictic-context.ts:26`
  - body: The first English direct-reference pattern matches ordinary short sentences starting with `it`, `this`, or `that` even when no reply or thread context exists. Messages like `It still fails after reboot.` will get a resolver note with no nearby candidates, biasing the reply toward unnecessary ambiguity handling.
  - confidence: 0.84
- **[P3] Remove the unbounded json matcher:** `src/auto-reply/reply/chat-shaping.ts:13`
  - body: The extra `/json/u` matcher duplicates the word-boundary JSON check but also matches substrings like `jsonl` and `jsonify`. That can suppress chat shaping for messages that are not actually structured JSON-output requests.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `extensions/memory-core/src/prompt-section.ts:17`
