## Summary

Keep open: the PR implements most of the requested exec env injection and includes real gateway-mode proof, but the latest head still exports the TypeScript node-host fallback sentinel as `OPENCLAW_SESSION_KEY`.

## What This Changes

This PR adds docs, changelog, tests, and exec child-process env injection for `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` across local/gateway, node-host, and macOS exec paths.

## Best Possible Solution

Keep the env-var contract, but preserve node routing fallbacks separately from exported child-process metadata and add regression coverage for omitted node-host session metadata.

## Reproduction Assessment

Yes. Source inspection of latest PR head shows `parseSystemRunPhase` defaults an omitted session key to `"node"`, and the new `execEnv` exports that value directly.

## Solution Assessment

No. The feature direction is maintainable, but the latest diff should export an empty string when node-host session metadata is unavailable while keeping the internal routing fallback for events.

## Review Findings
