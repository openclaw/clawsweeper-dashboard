## Summary

Keep this PR open. The requested exec environment contract is not on current main, and the PR is useful, but the node-host path still routes the trusted values through the untrusted env override payload that shell-wrapper filtering drops before the child process starts.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` injection for exec child processes, plus docs and regression tests for local/gateway and node-host paths.

## Best Possible Solution

Route session and agent identifiers as trusted exec runtime metadata, merge them into the final child environment after user env sanitization in both TypeScript and macOS node-host runtimes, keep local injection ordered after user env, and add final-child coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: `host=node` exec uses shell-wrapper argv, and the receiving node-host sanitizers drop non-allowlisted `OPENCLAW_*` env overrides before spawning the child.

## Solution Assessment

No. Adding the values to `requestedEnv` is not the best node-host fix because that field is intentionally treated as untrusted user override data; a separate trusted metadata path is safer and preserves override-blocking semantics.
