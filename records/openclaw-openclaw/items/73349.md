## Summary

Keep open: this is a coherent feature PR and current main lacks the requested env injection, but the latest diff still exports the internal `node` session sentinel as `OPENCLAW_SESSION_KEY` on the direct node-host path when no session metadata is available.

## What This Changes

The PR injects `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` into exec child-process environments across gateway, sandbox/PTY, node-host, and macOS paths, with docs, changelog, and regression tests.

## Best Possible Solution

Normalize the direct node-host env export to the same empty-string contract, add no-session regression coverage for host=node, then maintainer-review and land the public env contract with the existing docs and changelog.

## Reproduction Assessment

Yes for source reproduction: current main only injects `OPENCLAW_SHELL=exec`, and PR head's direct node-host path defaults omitted `sessionKey` to `node` before exporting it. I did not run live exec because this review was read-only.

## Solution Assessment

No, not yet: injecting trusted values at child-env composition points is the right direction, but the direct node-host path needs the same sentinel normalization as the macOS companion path before merge.

## Review Findings
