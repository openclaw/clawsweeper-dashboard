## Summary

Keep this PR open, but it is not merge-ready: current main does not implement the requested exec env contract, and the PR’s node-host path still routes trusted identifiers through user env overrides that shell-wrapper sanitization drops before the final child process sees them.

## What This Changes

The PR injects `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` into exec child environments across gateway, sandbox, PTY, and node-host paths, with docs and regression tests.

## Best Possible Solution

Carry session and agent identifiers as trusted exec runtime metadata, merge them after user env sanitization in gateway, sandbox, node-host, and macOS companion paths, document the always-set empty-string semantics, and include changelog plus final-child coverage.

## Reproduction Assessment

Yes for the review finding. Source inspection shows `host=node` uses shell-wrapper argv, then node-host and macOS sanitizers filter shell-wrapper env overrides through an allowlist that excludes the proposed `OPENCLAW_*` keys.

## Solution Assessment

No. Runtime injection is the right general direction, but the node-host implementation should not send trusted identifiers through `requestedEnv`; they need a trusted metadata path merged after request env filtering.
