## Summary

Keep this PR open: current main still lacks the requested exec environment contract, and the proposed node-host path does not actually deliver the new variables because shell-wrapper env filtering drops them before child execution.

## What This Changes

The PR injects `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` into exec child environments, updates exec/environment docs, and adds regression tests for gateway and node-host paths.

## Best Possible Solution

Carry session and agent identifiers as trusted exec runtime metadata for every host, merge them after user env sanitization in gateway, sandbox, node, and macOS execution, document the always-set empty-string semantics, and add changelog plus final-child coverage.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: `host=node` builds shell-wrapper argv, then node-host `sanitizeSystemRunEnvOverrides(..., shellWrapper: true)` filters `OPENCLAW_*` out of `params.env` before spawning the child.

## Solution Assessment

No. The gateway/sandbox runtime injection is the right shape, but the node-host implementation should not put trusted identifiers into `requestedEnv`, because that field is sanitized as user override data.
