## Summary

Keep open: the PR addresses a real feature request, but it still has a host=node correctness blocker, missing changelog coverage, and insufficient real behavior proof for an external contribution.

## What This Changes

The PR injects `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` into exec child environments, updates exec/environment docs, and adds gateway/node regression tests.

## Best Possible Solution

Carry session and agent identifiers as trusted exec runtime metadata merged after env sanitization across gateway, sandbox, PTY, node-host, and macOS companion paths, then document the always-set semantics and ship a changelog entry with real behavior proof.

## Reproduction Assessment

Yes for the review finding. Source inspection shows node exec is shell-wrapped and downstream node/macOS env sanitizers drop non-allowlisted `OPENCLAW_*` requested-env entries before child process spawn.

## Solution Assessment

No. Runtime metadata injection is the right direction, but `host=node` should not carry trusted identifiers through request env overrides that are intentionally filtered; it needs a trusted metadata path plus changelog and real proof.
