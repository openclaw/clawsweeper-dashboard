## Summary

Keep open: latest head still has merge-blocking issues, including a Swift compile error and inconsistent empty-string clearing for the new exec metadata contract.

## What This Changes

The PR adds docs, changelog, tests, and exec/node-host env merges to expose `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` to exec child processes.

## Best Possible Solution

Keep the env-var contract, but normalize trusted agent/session values per exec host and set both keys unconditionally after sanitized user env, with regression tests for missing values.

## Reproduction Assessment

Yes. Source inspection of latest head gives a high-confidence reproduction path for the blockers: the Swift nil-coalescing expression cannot compile, and the node-host env merge deterministically exports the `node` fallback when the caller omits a session key.

## Solution Assessment

No. The direction is maintainable, but the implementation should first use a consistent trusted metadata composition model across gateway/sandbox/PTY, TS node fallback, Swift companion, and direct macOS node runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Initialize Swift env from the non-optional context:** `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift:456`
  - body: `ExecApprovalEvaluation.env` is a non-optional `[String: String]`, so `context.env ?? ProcessInfo.processInfo.environment` cannot type-check. Initialize from `context.env` directly, then apply the trusted metadata values.
  - confidence: 0.96
- **[P2] Clear missing metadata in the Swift companion:** `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift:457-462`
  - body: This path only assigns `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` when request fields are non-empty. Because the sanitized env inherits from the app process, missing metadata can leave stale inherited values instead of the documented empty string.
  - confidence: 0.9
- **[P2] Clear missing metadata in MacNodeRuntime:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeRuntime.swift:916-922`
  - body: `MacNodeRuntime` has the same conditional merge: a missing `agentId` leaves any inherited `OPENCLAW_AGENT_ID` visible to the child process. Set the trusted keys unconditionally after `evaluation.env` so the documented empty-string clearing holds here too.
  - confidence: 0.87
- **[P2] Do not export the node fallback as a session key:** `src/node-host/invoke-system-run.ts:669`
  - body: When `params.sessionKey` is omitted, `phase.sessionKey` has already defaulted to the internal `"node"` sentinel. Exporting that as `OPENCLAW_SESSION_KEY` violates the empty-string contract and can make callback scripts treat `node` as a routable session.
  - confidence: 0.91
