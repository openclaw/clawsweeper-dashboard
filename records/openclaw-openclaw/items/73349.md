## Summary

Keep this PR open because the requested env injection is useful and mostly implemented, but the latest head still exports the internal node-host `node` sentinel as `OPENCLAW_SESSION_KEY` on the direct node-host path, contradicting the new docs contract.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` injection for exec child processes across gateway, node-host, macOS node mode, docs, tests, and changelog.

## Best Possible Solution

Keep the env-injection approach, normalize the direct node-host sentinel to an empty string before export, add omitted-session node-host coverage, then let maintainers decide the public env contract.

## Reproduction Assessment

Yes for source reproduction: current main only injects `OPENCLAW_SHELL=exec`, and the PR head still defaults omitted node-host session metadata to the internal `node` sentinel before exporting it. I did not run live exec or tests because this review is read-only.

## Solution Assessment

No, not yet: injecting trusted values at child-env composition points is the right direction, but the direct node-host path needs the same sentinel normalization as the macOS companion path before merge.

## Review Findings

Overall confidence: 0.88
