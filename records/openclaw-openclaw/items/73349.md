## Summary

Keep open: the feature direction is sound and current main does not already implement it, but the latest PR head still has correctness blockers in node-host and macOS env composition.

## What This Changes

The branch adds docs, changelog, tests, and exec env injection for `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` across gateway/sandbox/PTY, node-host, and macOS node-runtime paths.

## Best Possible Solution

Keep the env-var contract, but compose trusted agent/session metadata per exec host and write both keys unconditionally after sanitized user env, while keeping internal routing fallbacks separate from exported env values.

## Reproduction Assessment

Yes for the review blockers. Source inspection of latest PR head shows deterministic paths where missing metadata is not cleared and where the node fallback sentinel is exported as `OPENCLAW_SESSION_KEY`.

## Solution Assessment

No. The requested direction is maintainable, but the current diff is not the best implementation until trusted metadata clearing is consistent across Swift and TypeScript node-host paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Clear missing metadata in the Swift companion:** `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift:457-462`
  - body: This path only overwrites `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` when request values are non-empty. Since `context.env` comes from sanitized process env, missing metadata can leave inherited `OPENCLAW_*` values visible instead of the documented empty strings; set both trusted keys unconditionally after the sanitized env merge.
  - confidence: 0.91
- **[P2] Clear missing metadata in MacNodeRuntime:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeRuntime.swift:917-922`
  - body: `execEnv` inherits the sanitized app env, but the new trusted values are only assigned when non-empty. A run without `agentId` or session metadata can therefore leak stale inherited `OPENCLAW_*` values to the child process; assign both keys unconditionally using empty strings when unavailable.
  - confidence: 0.9
- **[P2] Do not export the node fallback as a session key:** `src/node-host/invoke-system-run.ts:669`
  - body: When `params.sessionKey` is omitted, `phase.sessionKey` has already defaulted to the internal `"node"` sentinel. Exporting that value as `OPENCLAW_SESSION_KEY` breaks the empty-string contract and can make callback scripts treat `node` as a routable session; preserve routing fallback separately from the env value.
  - confidence: 0.92
