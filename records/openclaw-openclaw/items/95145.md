## Summary

Keep open: current main and the latest release still render Darwin kernel releases into agent runtime OS metadata, and several open PRs reference this issue with closing syntax, so the issue should close only after one canonical fix lands.

## What This Changes

The issue requests that OpenClaw report the macOS product version on macOS 26/Tahoe instead of deriving the visible OS version from Darwin 25.x kernel releases.

## Best Possible Solution

Land one canonical runtime-OS helper fix that uses `sw_vers -productVersion` for Darwin agent runtime metadata, preserves non-Darwin prompt output, avoids double-appending architecture, and keeps status/trajectory summary shape stable.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on Darwin, three runtime prompt builders still pass `os.release()` into `runtimeInfo.os`, and `buildRuntimeLine()` renders that value.

## Solution Assessment

Yes. Using the macOS product version for Darwin runtime metadata is the narrow maintainable direction; the remaining maintainer choice is which open candidate PR has the best helper, cache, fallback, and proof shape.
