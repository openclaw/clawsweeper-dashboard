The issue requests configurable TUI footer fields for model, tokens, estimated cost, provider balance, and rate-limit/cooldown status.

## Best Possible Solution

Keep this as the broader product/API tracker until maintainers choose a generic, typed, cached, secret-safe footer/status contribution model that coordinates with existing usage surfaces and the narrower plugin status-provider track.

## Solution Assessment
