## Summary

Keep open: current main and v2026.6.8 still have a fixed TUI footer and only expose the remote-host footer toggle, while existing `/status`, `/usage full`, `/usage cost`, and `messages.usageTemplate` cover related usage reporting outside the persistent TUI footer. The narrower plugin-status-provider work is useful context but does not fully cover configurable live cost, balance, and cooldown fields.

## What This Changes

The issue requests configurable TUI footer fields for model, tokens, estimated cost, provider balance, and rate-limit/cooldown status.

## Best Possible Solution

Keep this open as the broader product/API tracker, use existing `/status` and `/usage` surfaces meanwhile, and let maintainers decide whether the durable fix is a TUI footer API, a plugin status-provider seam, or no live footer expansion.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source and release inspection confirm the requested live configurable TUI footer is absent on current main and v2026.6.8.

## Solution Assessment

No high-confidence best fix is established yet. A direct core footer template or provider-balance poller is not clearly best without maintainer direction because existing `/usage` and provider usage surfaces cover part of the need and repository policy favors plugin/API seams for optional capability.
