## Summary

Keep open. Current main still renders the TUI footer from a fixed built-in field list and has no footer template/config option, CLI flag, or plugin status-provider API that can persistently show custom cost, balance, or cooldown/rate-limit fields. Existing `/status`, `/usage`, provider usage hooks, and session plugin debug/status lines are useful adjacent surfaces, and UrRhb's burn0 comment is useful design input, but they do not satisfy the requested configurable TUI footer.

## What This Changes

Request configurable TUI footer fields/template for model, token count, estimated cost, provider balance, and rate-limit/cooldown status.

## Best Possible Solution

Keep this as the broader product/API tracker. A complete solution should combine a reviewed TUI footer customization/rendering path with a generic plugin/provider status contribution seam, while keeping provider-specific cost, balance, and quota calls cached, secret-safe, failure-tolerant, and documented.
