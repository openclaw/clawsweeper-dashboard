## Summary

Keep open: current main and v2026.6.6 still have a fixed TUI footer with no cost/balance/rate-limit footer fields, no footer template config, and no plugin status-provider API. The related work at https://github.com/openclaw/openclaw/issues/64086 and https://github.com/openclaw/openclaw/pull/72557 is the narrower plugin-API path, while this issue still needs a maintainer product/security decision for the broader footer UX.

## What This Changes

Request configurable TUI footer fields or providers that can show model, token, estimated cost, remaining balance, and rate-limit status.

## Best Possible Solution

Keep this as the broad footer-status tracker until maintainers choose a plugin/provider/status-line extension model and then land one coherent TUI rendering path with documented credential and security boundaries.

## Reproduction Assessment

Not applicable as a feature request rather than a broken existing behavior. Source and release checks confirm the current footer is fixed and the requested customization/status-provider surfaces are absent on current main and v2026.6.6.

## Solution Assessment

Unclear. A direct core footer template is not obviously the best fit under the repo's lean-core/plugin guidance; the safer path is likely a maintainer-approved plugin/status-provider or explicitly trusted status-line design before provider-specific cost and balance display is added.
