## Risks / Open Questions

- The final UX/API shape is unsettled: footer templates, plugin status providers, and command-rendered status lines have different security and maintenance tradeoffs.
- Provider balance and billing data can be delayed or provider-specific, so any native footer field needs provider-owned contracts, caching, and clear accuracy semantics.
- A shell-command status-line approach is security-sensitive because it introduces auto-running local commands, so it should not be treated as a routine automated fix.
