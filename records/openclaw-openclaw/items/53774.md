## Summary

Keep open: current main and the latest release still render a fixed TUI footer with only the remote-host footer toggle configurable. Main now has a custom `/usage full` response footer template, but that is a chat reply/footer surface, not the TUI footer template, balance display, cooldown display, or plugin status-provider API requested here; the related open PR covers only a narrower API seam and explicitly excludes TUI rendering and cost-display work.

## What This Changes

The issue requests configurable TUI footer fields for estimated session cost, remaining provider balance, and rate-limit or cooldown status.

## Best Possible Solution

Keep this as the canonical product/API tracker until maintainers choose the supported extension path, likely a plugin-facing status provider plus provider-owned cost, balance, and rate-limit data sources rather than a one-off core footer template.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection confirms the requested live TUI footer template/status-provider fields are still absent on current main.

## Solution Assessment

No for the direct config-template proposal as written; a maintainer-approved plugin/API seam with provider-owned usage, balance, and rate-limit data appears better aligned with the repository direction. Existing `messages.usageTemplate` is useful but solves the per-response usage footer, not this TUI footer request.
