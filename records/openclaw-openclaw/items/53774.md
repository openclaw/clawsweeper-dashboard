## Summary

Keep open. Current main still has a fixed TUI footer and adjacent cost/quota reporting surfaces, but no configurable footer template or plugin/provider status contribution path for persistent cost, balance, or cooldown fields. Related open work covers pieces of the design, but none supersedes the broader request.

## What This Changes

Request configurable TUI footer fields/templates for estimated session cost, provider balance, and rate-limit or cooldown status.

## Best Possible Solution

Keep this as the broader product/API tracker. A complete solution should add a reviewed generic footer customization path plus a plugin/provider status contribution seam, while leaving provider-specific cost, balance, quota, and cooldown data ownership in provider plugins with documented, cached, secret-safe behavior.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs provide high-confidence verification that the requested configurable footer fields are absent, while adjacent `/status`, `/usage`, and provider usage windows exist outside the persistent TUI footer.

## Solution Assessment

Unclear for implementation until maintainers choose the API and UX shape. The narrowest maintainable direction appears to be a generic footer renderer/template plus a typed plugin/provider status contribution API, not hard-coding cost, balance, and rate-limit logic directly into the TUI renderer.
