## Label Justifications

- `P1`: The PR targets a user-facing reply death loop where repeated memoryFlush failures can prevent real replies from being delivered.
- `merge-risk: 🚨 compatibility`: The patch changes fail-closed versus fail-open recovery behavior and broadens the existing `notifyUser` option without a new config key.

- `merge-risk: 🚨 message-delivery`: The patch stops delivering memoryFlush error payloads as terminal replies and instead lets the normal user reply proceed, with an optional degraded notice.

## Summary

Keep open for maintainer review: current main still has the terminal memoryFlush error path, and this PR is a coherent candidate fix with sufficient proof, but it intentionally changes reply delivery, session rotation, and existing notice semantics.

## What This Changes

The PR changes auto-reply memoryFlush failure handling so failed maintenance no longer aborts the user reply, rotates exhausted sessions while preserving transcripts, clears stale flush metadata on reset, and documents the broadened compaction notice behavior.

## Best Possible Solution

Merge only after maintainers accept the fail-open recovery semantics and reconcile the open pre-run ordering PR, keeping the regression tests and docs that pin non-terminal delivery, non-destructive rotation, and broadened `notifyUser` behavior.

## Reproduction Assessment

Yes for the code path: current main still calls `replyOperation.fail("run_failed")` and returns before the main reply when memoryFlush produces visible error payloads. I did not run a live provider-crash reproduction in this read-only pass, but the PR body includes redacted Gateway logs showing the after-fix path delivering all 12 replies under repeated mock flush 500s.

## Solution Assessment

Unclear as a product decision, but the implementation is a coherent candidate fix: it logs memoryFlush maintenance errors, rotates only after tracked exhaustion, preserves transcripts, and continues the main reply. The unresolved question is whether maintainers want those permanent fail-open/session-rotation/notice semantics and how to reconcile them with https://github.com/openclaw/openclaw/pull/84792.
