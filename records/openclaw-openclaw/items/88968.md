## Summary

Keep open: this is a credible fix for the memoryFlush reply-abort loop with sufficient real behavior proof and no concrete line-level defect found, but it intentionally changes reply delivery, session recovery, and existing `notifyUser` semantics, and it overlaps the open memory-flush ordering PR.

## What This Changes

The PR changes `runReplyAgent` so memoryFlush visible errors are logged instead of terminally failing the reply, rotates exhausted sessions while preserving old transcripts, clears memoryFlush reset state, and updates docs/help for the expanded notice behavior.

## Best Possible Solution

Land this only after maintainers explicitly accept the fail-open recovery policy and reconcile the final memoryFlush/preflight ordering with the open same-pipeline PR.
