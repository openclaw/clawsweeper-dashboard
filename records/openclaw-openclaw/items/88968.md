## What This Changes

The PR changes the auto-reply memory-flush pipeline to log flush maintenance errors, continue the main reply, rotate and clear exhausted session state while preserving the old transcript, and document the expanded `notifyUser` notice semantics.

## Best Possible Solution

Land a bounded recovery that lets normal replies continue after flush maintenance fails, preserves old transcripts during exhaustion rotation, clears stale flush metadata on resets, and documents the expanded notice behavior once maintainers accept the upgrade tradeoff.
