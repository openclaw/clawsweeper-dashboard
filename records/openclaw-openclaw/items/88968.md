## Summary

Keep this PR open for maintainer review: the patch looks coherent and well-proven, but it deliberately changes reply-delivery and session-recovery semantics in a compatibility-sensitive path.

## What This Changes

The branch changes memory-flush failures to log and continue, rotates exhausted reply sessions non-destructively, clears stale flush metadata on reset, and updates notifyUser documentation/help text.

## Best Possible Solution

Land this only after maintainers accept fail-open reply behavior, non-destructive session rotation, and the expanded notifyUser semantics as the desired recovery policy.

## Solution Assessment

Yes as a bounded code fix if maintainers accept the policy tradeoff: it removes the terminal abort, rotates the exhausted session non-destructively, and keeps the old transcript. The remaining question is product ownership of those compatibility-sensitive semantics, not a discrete code defect.
