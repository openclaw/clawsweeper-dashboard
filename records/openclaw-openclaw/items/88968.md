## Summary

Keep open for maintainer review. Current main still has a source-level path where visible memory-flush errors abort the normal reply, and this PR supplies a coherent fix with tests and real Gateway proof, but fail-open delivery, automatic session rotation, and expanded `notifyUser` semantics are compatibility/session/message-delivery decisions.

## What This Changes

The branch changes auto-reply memory-flush failures to log and continue, rotates exhausted sessions while preserving old transcripts, clears flush state on reset, and updates `notifyUser` docs and tests.

## Best Possible Solution

Land only after exact-head checks finish and a maintainer accepts the fail-open reply, session-rotation, and expanded-notice semantics; keep broader configurable reset or compaction policy with https://github.com/openclaw/openclaw/issues/30452 and https://github.com/openclaw/openclaw/issues/85645.

## Solution Assessment

Is this the best way to solve the issue? Mostly yes as a bounded, non-destructive recovery path that builds on the accepted predecessor policy; the remaining decision is maintainer acceptance of fail-open delivery, automatic rotation, and notice semantics.
