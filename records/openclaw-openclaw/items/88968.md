## Summary

## What This Changes

The branch changes memory-flush visible errors in `runReplyAgent` into logged maintenance failures, rotates exhausted sessions while preserving transcripts, clears stale flush state on reset, and documents the existing `notifyUser` setting as covering degraded memory-flush notices.

## Best Possible Solution

Land this PR or an equivalent recovery only after maintainers explicitly accept the fail-open rotation/notice semantics and reconcile the pipeline overlap with https://github.com/openclaw/openclaw/pull/84792.

## Solution Assessment

Mostly yes. The patch is a bounded fix for the observed abort loop and follows existing session-reset/rebind patterns, but fail-open delivery, automatic rotation, and expanded notice semantics are compatibility-sensitive product choices.
