## Summary

Keep open for maintainer review: the patch appears technically sound and the live/log proof plus CI are strong, but it intentionally changes compatibility-sensitive reply-delivery and session-recovery behavior and overlaps the open memory-flush ordering PR.

## What This Changes

This PR changes auto-reply memory-flush failures to log and continue, rotate exhausted sessions while preserving transcripts, clear stale flush state on resets, and update existing `notifyUser` docs/help.

## Best Possible Solution

Land this or an equivalent maintainer-approved fail-open recovery after explicitly accepting the rotation/notice semantics and resolving the overlap with https://github.com/openclaw/openclaw/pull/84792.
