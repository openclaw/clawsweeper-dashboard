## Summary

Keep open for maintainer review: current main and the latest release still have the terminal memoryFlush visible-error path, and this PR appears to fix that path, but it intentionally changes reply delivery, session-state recovery, and existing `notifyUser` semantics.

## What This Changes

The PR changes memoryFlush failure handling so maintenance errors no longer abort the main reply, exhausted sessions rotate with preserved transcripts, reset paths clear memoryFlush state, and docs/tests cover the revised behavior.

## Best Possible Solution

Land this only after maintainers explicitly accept fail-open memory maintenance, non-destructive exhausted-session rotation, expanded `notifyUser` semantics, and the interaction with https://github.com/openclaw/openclaw/pull/84792.
