## Summary

Keep open for maintainer review: the patch appears correct and current main/latest release still abort the reply on visible memory-flush errors, but the recovery policy intentionally changes message delivery, session rotation, and existing notifyUser behavior.

## What This Changes

The PR changes auto-reply memory-flush failure handling to log and continue, rotate exhausted sessions non-destructively, clear flush metadata on reset, and update compaction notice docs/help.

## Best Possible Solution

Land this only after maintainers accept the fail-open reply behavior, non-destructive session rotation, and expanded notifyUser semantics as the desired recovery policy.

## Solution Assessment

Yes as a bounded code fix if maintainers accept the policy tradeoff: it removes the terminal abort, rotates the exhausted session non-destructively, and keeps the old transcript. The remaining question is product ownership of those compatibility-sensitive semantics, not a discrete code defect.
