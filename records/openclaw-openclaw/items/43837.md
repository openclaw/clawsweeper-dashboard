## Summary

The PR adds one-shot `/think <level> <body>` parsing and resolution, Telegram multiline command-body normalization, xhigh unsupported handling, regression tests, and generated Plugin SDK baseline updates.

## Best Possible Solution

Land a rebased, scoped implementation that keeps one-shot thinking separate from persisted session state, preserves the final multiline Telegram prompt, excludes local-only generated baselines, and includes redacted live-channel proof.

Yes, source-reproducible. Current main has no `oneShotThinkLevel` state and resolves thinking from override, persistent directive, or session state only, so `/think <level> <body>` cannot apply a non-persistent level today.

## Solution Assessment

No, not yet. The separate one-shot directive state is the right implementation direction, but this branch still needs generated-baseline cleanup, final Telegram multiline-body preservation, a clean rebase, and real behavior proof before merge.
