## Summary

Keep open: current main does not implement one-shot thinking, and the PR still has a concrete cross-channel routing blocker plus a missing required changelog entry before merge.

## What This Changes

The PR adds one-shot `/think`, `/thinking`, and `/t` parsing with a non-persistent `oneShotThinkLevel`, Telegram multiline command-body normalization, xhigh validation coverage, focused tests, and generated Plugin SDK API baseline updates.

## Best Possible Solution

Keep the non-persistent one-shot directive design, but separate command authorization/debounce from slash-command interception or add a validated one-shot exception so reply handling is reached on every supported channel.

## Reproduction Assessment

Yes. Source inspection is enough: PR head asserts one-shot `/think <level> <body>` is still `hasControlCommand`, while current Discord guild preflight drops `hasControlCommand` text before directive resolution runs.

## Solution Assessment

No. The one-shot directive slot is the right narrow direction, but the current PR keeps the new message syntax on a command-interception path for Discord guild text messages and still needs release-note coverage.
