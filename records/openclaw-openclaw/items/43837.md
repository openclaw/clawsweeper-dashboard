## Summary

Keep this PR open, but it is not merge-ready: the one-shot `/think` implementation is useful and not present on current main, yet the current head still leaves shared command gates classifying `/think <level> <body>` as a control command, is merge-conflicting, and has only automated proof for Telegram-visible behavior.

## What This Changes

The PR adds one-shot `/think`, `/thinking`, and `/t` parsing, transient reply directive state, Telegram multiline command-body preservation, unsupported-`xhigh` handling, generated API baselines, and focused tests.

## Best Possible Solution

Rebase onto current main, make one-shot think a first-class shared command-gate case without weakening authorization, preserve transient-versus-persisted state, and add live Telegram proof plus focused cross-channel tests.

## Reproduction Assessment

Yes for the blocking patch defect: source inspection shows PR-head `hasControlCommand()` still returns true for `/think <level> <body>`, and current channel gates consume that boolean before reply directives run. I did not execute a live channel reproduction in this read-only review.

## Solution Assessment

No: the feature direction is useful, but this branch is only a plausible partial fix until shared command-gate behavior, current-main conflicts, and live Telegram proof are handled. The better solution keeps the parser and directive state but fixes the owner boundary at the shared command gate with channel coverage.
