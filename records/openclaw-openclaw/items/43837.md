## Summary

Keep this PR open: the feature direction is useful, but the current branch still classifies one-shot `/think <level> <body>` messages as control commands, so Discord guild preflight can drop them before reply directive handling runs; the user-facing feature also needs a changelog entry.

## What This Changes

The branch adds one-shot `/think`, `/thinking`, and `/t` message parsing, a non-persistent `oneShotThinkLevel` directive path, Telegram multiline command-body normalization, xhigh validation coverage, tests, and generated Plugin SDK API baseline updates.

## Best Possible Solution

Revise the PR so validated one-shot `/think <level> <body>` messages reach normal reply handling on every channel while directive-only tails remain protected as commands, then refresh focused tests, generated baselines if needed, and the changelog.

## Reproduction Assessment

Yes. The high-confidence source path is PR head expecting `hasControlCommand('/think high write me a poem')` to be true while Discord guild preflight returns `null` for non-DM text commands before `resolveReplyDirectives` can apply `oneShotThinkLevel`.

## Solution Assessment

No. The non-persistent directive slot is the right narrow direction, but command authorization/debounce detection needs to be separated from slash-command interception, or a validated one-shot exception needs to be added with Discord preflight coverage.
