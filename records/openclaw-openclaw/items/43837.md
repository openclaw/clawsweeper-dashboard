## Summary

Keep this PR open: the feature shape is useful, but the latest branch still keeps validated one-shot `/think <level> <body>` messages classified as control commands, so channel ingress paths such as Discord guild preflight can drop them before the new one-shot directive handling runs.

## What This Changes

The PR adds one-shot `/think`, `/thinking`, and `/t` message parsing, a non-persistent `oneShotThinkLevel` directive path, Telegram multiline command-body normalization, xhigh validation coverage, tests, and generated Plugin SDK API baseline updates.

## Best Possible Solution

Revise the PR so validated one-shot `/think <level> <body>` messages reach the normal reply path on every channel while directive-only tails such as `/think high /status` remain protected as commands. The maintainable shape is a split between authorization/debounce detection and command interception, with focused Discord preflight coverage, the existing non-persistent directive slot preserved, Telegram multiline behavior retained, and generated baseline/changelog files refreshed before merge.

Yes. The high-confidence reproduction is static: PR head expects `hasControlCommand('/think high write me a poem')` to be true, while Discord guild preflight drops non-DM text commands before reply directive resolution can apply `oneShotThinkLevel`.

## Solution Assessment

No. The requested product behavior and non-persistent directive slot are the right narrow direction, but the current implementation is not the best fix because it preserves the command classification that prevents the new path from running on some ingress surfaces.
