## Summary

Keep open: current main still lacks one-shot `/think <level> <body>` support, and the PR head still has a blocking channel interception gap because it keeps one-shot text classified as `hasControlCommand` while Discord non-DM preflight drops those messages before directive resolution.

## What This Changes

The PR adds non-persistent one-shot `/think`, `/thinking`, and `/t` handling, Telegram multiline command-body normalization, xhigh validation coverage, focused tests, and generated Plugin SDK API baseline refreshes.

## Best Possible Solution

Keep the non-persistent one-shot directive design, but separate command authorization/debounce from text slash-command interception or add a validated one-shot exception so the new syntax reaches reply handling on supported channels, then add the active-version changelog bullet.

## Reproduction Assessment

Yes. Source inspection is enough: PR head keeps `/think <level> <body>` classified as `hasControlCommand`, and Discord non-DM preflight drops `hasControlCommand` text before directive resolution can apply `oneShotThinkLevel`.

## Solution Assessment

No. The one-shot directive slot is a maintainable direction, but keeping the new syntax on the slash-command interception path leaves a supported channel path broken and the feature still lacks release-note coverage.
