## Summary

Keep open. Current main still has cooperative abort, queue cleanup, foreground reply freshness fencing, and tool abort-signal plumbing, but it does not yet provide the requested session-wide run-currentness contract, broad stale-output fence, pre-tool skip gate, subprocess/session-scope invalidation, or default new-message takeover semantics; the linked draft PR remains an open partial implementation path.

## What This Changes

The issue requests a unified session run-invalidation mechanism so `/stop` or a newer message cancels stale runs, fences late output, skips pending tools, cancels owned subprocesses, and lets the newer instruction take over.

## Best Possible Solution

Keep this as the canonical core interruptibility tracker and coordinate staged PRs that define a session currentness contract, fence output sites, add pre-tool/process cancellation gates, and prove the behavior with regression and real-runtime evidence.
