## Summary

Keep open for https://github.com/openclaw/openclaw/issues/70319. Current main still has abort, queue, steering, and process-cancel primitives, but it does not provide the unified per-session generation invalidation, broad stale-output fencing, pre-tool currentness gate, or default new-message takeover requested here; active PRs #70363 and #75226 are the right review path rather than a close.

## What This Changes

The issue requests stronger core interruption semantics so `/stop` or a new message invalidates an active run and prevents stale tools, progress, typing, and replies from continuing.

## Best Possible Solution

Keep this as the canonical core interruptibility tracker, review or split draft PR #70363, coordinate #75226, and land focused fences, tool gates, process-cancellation wiring, takeover semantics, and regression tests.
