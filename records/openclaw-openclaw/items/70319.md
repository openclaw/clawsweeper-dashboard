## Summary

Keep open. Current main still has cooperative abort, queue cleanup, and a narrow foreground freshness fence, but not the requested unified run-currentness contract, broad stale-output fencing, pre-tool gate, subprocess/session scope invalidation, or default new-message takeover; the linked draft PR is still the active partial implementation path.

## What This Changes

Request stronger run interruption semantics so `/stop` or a newer message invalidates the active run, blocks stale side effects, cancels owned work, skips pending tools, and makes the newest input dominant.

## Best Possible Solution

Keep this as the canonical core interruptibility tracker and coordinate staged PRs that define a session currentness contract, fence emission sites, add pre-tool/process cancellation gates, and prove the behavior with regression plus real-runtime evidence.
