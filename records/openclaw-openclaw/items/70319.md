## Summary

Keep open. Current main still has useful abort, queue cleanup, steering, and process-cancel primitives, but it does not implement unified per-session generation invalidation, broad stale-output fencing, a pre-tool currentness gate, or default new-message takeover; the same-author draft PR #70363 and related PR #75226 remain active partial work.

## What This Changes

Request stronger run interruptibility so `/stop` or a new message invalidates the active run, cancels owned work, skips stale tools, and fences stale visible output.

## Best Possible Solution

Keep this as the canonical core interruptibility tracker, review or split #70363, coordinate #75226, then land focused fences, tool-gates, process-cancellation wiring, new-message takeover, and regression tests.
