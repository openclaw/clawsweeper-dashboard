## Summary

Keep open. Current main fixes the silent no-selector `openclaw agent` path, but current source/docs still leave material parts unresolved: stale no-selector examples remain, Lobster and llm-task are still optional agent tools rather than root CLI commands, and the headless gateway delay has not been reproduced or ruled out.

## What This Changes

The issue requests regression cleanup for silent `openclaw agent` no-output behavior, missing `openclaw lobster` and `openclaw llm-task` root command handling, and slow gateway discovery on headless Ubuntu.

## Best Possible Solution

Split the item along the actual product boundaries: keep the explicit session-selector guard, update stale no-selector agent docs, make a maintainer decision on whether Lobster and llm-task should expose plugin-owned root CLI commands or remain documented as optional tools, and request a focused headless Linux trace for the gateway delay if it still reproduces.

## Reproduction Assessment

No, not end-to-end. Static inspection reproduces the docs and CLI-registration mismatch, but the original no-selector silent exit is fixed in current source and the 30s headless delay still needs a live timing/config/gateway URL trace.

## Solution Assessment

Unclear for the bundled issue. The selector guard is the right narrow runtime fix for the no-output subcase, but root commands for Lobster/llm-task are a product/API decision and the discovery-delay symptom needs reproduction before choosing a code path.
