## Summary

Keep open. Current main has the missing-selector guard for both gateway and local agent paths, but selector-free docs examples still exist, Lobster and llm-task remain optional tools rather than root CLI commands, and the reported headless gateway delay still lacks a current-main trace.

## What This Changes

This issue reports a regression where `openclaw agent` could exit without output, Lobster and llm-task were expected as root CLI commands, and headless Linux gateway discovery appeared to add a 30s delay.

## Best Possible Solution

Split the item by boundary: keep the selector guard, land or make equivalent docs fixes for selector-free examples, decide whether Lobster and llm-task should expose plugin-owned CLI groups, and ask for a focused current-main headless Linux trace if the gateway delay still reproduces.

## Reproduction Assessment

No, not end-to-end. Source inspection reproduces the current docs and optional-tool registration mismatches, but the original no-selector silent exit is fixed and the 30s headless gateway delay still needs a live current-main trace.

## Solution Assessment

Unclear for the bundled issue. The selector guard is the right narrow runtime fix, the related docs PR is narrow for one stale snippet, and the Lobster/llm-task root-command expectation needs maintainer product direction before code changes.
