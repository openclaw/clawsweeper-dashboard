## Summary

Keep this PR open: it addresses a real source-reproducible content-preservation bug and includes useful proof, but the current branch makes nested `__openclaw.inboundDecoration.bareBody` authoritative without an independent host-owned decorated marker, which is not merge-ready for a durable session/security boundary.

## What This Changes

Adds a nested `__openclaw.inboundDecoration.bareBody` field plus resolver helpers, then updates transcript persistence, Control UI, CLI import dedupe, memory session export, historical replay, and regression tests to prefer that field over sentinel stripping.

## Best Possible Solution

Land one marker-gated trusted inbound-decoration contract, preserve legacy sentinel stripping only for unmarked historical rows, and fold this branch's useful consumer coverage into the chosen contract.

## Reproduction Assessment

Yes. Source inspection shows current main still builds inbound metadata as literal sentinel text and user-role consumers strip or compare that text heuristically, so a user-authored matching block can be treated like host metadata.

## Solution Assessment

No, not as written. The out-of-band bare-body contract is the right layer, but this PR needs a host-owned marker before trusting the durable field and maintainers need to choose the final contract shape.
