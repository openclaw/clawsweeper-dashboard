## Summary

Keep open. The PR addresses a real source-reproducible content-loss bug, but the current branch still makes a new serialized bare-body field authoritative without an independent host-owned decorated marker, so it needs maintainer contract/security review before merge.

## What This Changes

This PR adds `__openclaw.inboundDecoration.bareBody` plus resolver helpers, persists it on user turns, and updates UI display/extraction, CLI import dedupe, memory session export, historical replay, and tests to prefer that trusted bare body over sentinel stripping.

## Best Possible Solution

Land one marker-gated trusted inbound-decoration contract, keep sentinel stripping only as legacy fallback for unmarked historical rows, and fold the useful consumer coverage from this branch into that chosen shape.

## Reproduction Assessment

Yes, source-reproducible. Current main still builds inbound metadata as literal sentinel text and user-role consumers strip or compare that text heuristically, so a user-authored matching block can be treated like host metadata.

## Solution Assessment

No, not as written. The out-of-band bare-body contract is the right layer, but this PR needs a host-owned marker before trusting the durable field and maintainers need to choose the final contract shape.
