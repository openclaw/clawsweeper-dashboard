## Summary

Keep this PR open, but do not land it as-is. The root cause is credible and the patch touches the right service-env path, but the new parser-level `$VAR` filter can silently drop valid existing credentials from `~/.openclaw/.env`, and the supplied proof is only summarized with private local artifact paths.

## What This Changes

The PR filters shell-style `$VAR` and `${VAR}` values out of state-dir `.env` parsing and adds config plus gateway install plan regression tests.

## Best Possible Solution

## Solution Assessment

No. The PR targets the right path, but the proposed filter is broader than the reported unresolved-reference shape and should preserve literal dollar-containing credentials.
