## Summary

Keep open. Current main still adds a hidden 30s Active Memory setup grace to the configured 15s recall timeout, uses that inflated value for both the plugin watchdog and the `before_prompt_build` hook registration, and current tests still assert the resulting 45s default. This matches the prior ClawSweeper review and remains a valid, narrow plugin-owned timeout bug.

## What This Changes

Make Active Memory's `before_prompt_build` blocking budget respect `config.timeoutMs` by default instead of adding an undocumented 30s setup grace that can hold the main lane for 45s.

## Best Possible Solution

Fix this at the Active Memory plugin boundary so the user-visible `config.timeoutMs` is the actual blocking hook budget by default. If setup grace is still needed for cold starts, expose it as an explicit advanced plugin config field with manifest/docs/test coverage rather than silently extending the main-lane budget.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: with default `config.timeoutMs` of 15_000 and `DEFAULT_SETUP_GRACE_TIMEOUT_MS` of 30_000, current main registers and watchdogs Active Memory `before_prompt_build` work at 45_000 ms, and tests assert that behavior.

## Solution Assessment

Unclear. The proposed one-line default-grace change is plausible and narrow, but the final patch should update the intentional setup-grace tests and decide whether any grace remains as an explicit documented advanced option.
