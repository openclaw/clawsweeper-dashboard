## Summary

Keep open. Current main still gives Active Memory a hidden 30s setup grace on top of `config.timeoutMs`, and that inflated value is used both for the plugin watchdog and the `before_prompt_build` handler registration, so the reported 15s-to-45s behavior is present in source and locked in by tests.

## What This Changes

The issue asks Active Memory to stop extending its configured recall timeout with a hidden 30s setup grace that makes the default `before_prompt_build` budget 45s instead of 15s.

## Best Possible Solution

Fix this in the Active Memory plugin boundary: make the user-visible `config.timeoutMs` the actual blocking hook budget by default, or expose any setup grace as an explicit documented advanced option, then update focused tests, docs, and the changelog without changing generic hook semantics unless a shared API gap appears.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: with the default `config.timeoutMs` of 15_000 and `DEFAULT_SETUP_GRACE_TIMEOUT_MS` of 30_000, current main registers and watchdogs Active Memory's `before_prompt_build` work at 45_000 ms, and tests assert that value.

## Solution Assessment

Unclear. The proposed one-line default-grace change is plausible and narrow, but the final patch should also update the intentional setup-grace tests and decide whether an explicit documented grace knob is needed for slow first-run setup.
