Keep open: current main still documents Claude ACP as a supported `sessions_spawn` target, routes the reported call through the same ACP spawn and turn-consumption path, and I found no merged fix or duplicate. ## What This Changes

Claude ACP sessions spawned through `sessions_spawn({ runtime: "acp" })` should complete like direct ACPX CLI/API instead of failing with `ACP_TURN_FAILED`.

## Best Possible Solution

Fix the managed ACP Claude path so the Claude-assigned session id from `session/new` is preserved for `session/prompt`, surface upstream JSON-RPC `error.data` in diagnostics, and add focused regression plus live Claude ACP smoke coverage.
