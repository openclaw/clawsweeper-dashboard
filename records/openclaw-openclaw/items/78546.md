Keep open: current main still supports Claude as an ACP `sessions_spawn` target, but the ACP runtime-control path still has a concrete unsupported-config failure mode for Claude/non-Codex ACP sessions, and the related fix PRs remain open or only partially superseded.

## What This Changes

The issue reports that Claude ACP sessions launched through OpenClaw `sessions_spawn(runtime: "acp")` fail with ACP turn errors even though direct ACPX and Claude ACP calls succeed.

## Best Possible Solution

Land or replace the open ACP control fix with a core capability-aware runtime-control path that skips unsupported auto-injected options for Claude ACP while preserving explicit user config validation and adding live/targeted regression proof.
