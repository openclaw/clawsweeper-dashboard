## What This Changes

The PR adds Windows detached update restart health recovery tests/wiring and also changes gateway remote schema validation, Matrix contract tests, install-smoke dependency assertions, Tlon dependency resolution, the lockfile, and the secrets baseline.

## Best Possible Solution

Current main should receive a focused update-command fix that preserves the first unhealthy health snapshot, adds Windows detached restart regression coverage, and leaves dependency/config/contract surfaces untouched.

## Reproduction Assessment

Yes, at source level. Current main's detached update restart path does not pass includeUnknownListenersAsStale, while restart-health only treats Windows unknown listeners as stale when that flag is set.

## Solution Assessment

No. The narrow restart-health direction is valid, but this PR is not the best solution because its fallback probe can discard first-snapshot evidence and the branch carries unrelated dependency, config, workflow, lockfile, and contract changes.
