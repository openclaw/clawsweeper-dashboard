## Summary

Keep open: the Windows detached update restart recovery gap is still present on current main, but this PR should not merge as-is because it is stale/unmergeable and mixes unrelated dependency, config, contract-test, workflow, lockfile, and baseline changes into the restart fix.

## What This Changes

This PR hardens `openclaw update` detached restart health recovery and tests while also changing Tlon dependency resolution, Matrix contract coverage, gateway remote schema validation, install-smoke, the lockfile, and the secrets baseline.

## Best Possible Solution

Land a narrow replacement that only fixes Windows detached/package update restart health recovery, preserves stale-PID and expected-version evidence, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main can be forced through the update detached/package restart verification path on Windows with an unknown-listener stale PID, and that path does not pass the helper opt-in flag required to classify the listener as stale.

## Solution Assessment

No. The restart-health direction is valid, but this PR is not the best fix because it can overwrite the first unhealthy snapshot and it includes unrelated dependency, schema, contract, workflow, lockfile, and baseline changes.

- **[P1] Restore an installable Tlon dependency source:** `extensions/tlon/package.json:7`
  - body: This changes `@tloncorp/api` so the lockfile resolves through a GitHub codeload archive in a narrow update-restart PR.   - body: `waitForDetachedRestartHealth` overwrites the first unhealthy result with a zero-attempt Windows fallback probe.   - confidence: 0.86
- **[P2] Honor remote disabled state before accepting it:** `src/config/zod-schema.ts:733`
  - body: Adding `gateway.remote.enabled` makes `enabled: false` validate, but credential planning still activates remote token/password surfaces from mode, URL, Tailscale, or fallback state. Either wire the flag through runtime planning or keep the key rejected.
  - confidence: 0.84
- **[P2] Use Matrix's public contract surface:** `src/channels/plugins/contracts/registry.ts:10`
  - body: The contract registry imports Matrix private `src` internals directly even though core channel contract helpers should use public plugin/API surfaces. This couples core tests to Matrix implementation files and violates the boundary this suite is meant to enforce.
  - confidence: 0.78
