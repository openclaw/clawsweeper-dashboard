## What This Changes

The issue asks for gateway/model-stream processing to yield or offload work so very long streamed responses do not starve Node's event loop.

## Best Possible Solution

Investigate the full long-output delivery path and add cooperative yielding or backpressure at the remaining starvation boundary, with regression coverage that proves timers/WebSocket work can run while a large assistant stream drains.

## Reproduction Assessment

No high-confidence local reproduction is available from this review. The issue has strong production diagnostics and a plausible source path, but a synthetic queued-stream harness or live long-output DeepSeek run is still needed.
