## Summary

Keep open. Current main still matches the reported Telegram gateway-delegation bug, and #75478 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

`openclaw message send --channel telegram` should delegate Telegram sends to the live gateway message-action path instead of loading Telegram runtime in the local CLI process.

## Best Possible Solution

Review and land PR #75478, or an equivalent narrow fix that preserves Telegram gateway execution mode, avoids local Telegram runtime loading for targeted CLI sends, and keeps the regression tests.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: the CLI eagerly loads the channel plugin registry and the visible Telegram wrapper hides the gateway-owned execution-mode hook required for delegation.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because Telegram-specific ownership stays in the Telegram plugin while CLI dispatch uses the existing gateway message-action contract.
