## Summary

Keep open. Current main still matches the reported Telegram gateway-delegation failure, and #75478 is an open unmerged implementation candidate explicitly paired with this issue.

## What This Changes

The issue requests that Telegram CLI sends delegate to the live gateway and return JSON without initializing the local Telegram runtime in the CLI process.

## Best Possible Solution

Land #75478 or an equivalent narrow fix that preserves Telegram gateway execution mode through the wrapper, avoids local Telegram runtime loading for targeted CLI sends, and keeps regression coverage in the Telegram plugin and CLI message helper tests.

## Reproduction Assessment

Yes. Current main gives a high-confidence source-level reproduction path: Telegram's wrapper hides the gateway-owned execution-mode hook, and the CLI loads configured channel plugins before dispatch.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable solution because Telegram-specific ownership stays in the Telegram plugin while CLI sends use the existing gateway message-action contract.
