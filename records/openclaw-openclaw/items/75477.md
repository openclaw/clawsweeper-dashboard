## Summary

Keep open. Current main still matches the reported Telegram gateway-delegation bug, and PR #75478 is an open closing-linked implementation candidate, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Requests that Telegram CLI message sends delegate to the live gateway `message.action` path without loading the local Telegram runtime.

## Best Possible Solution

Review and land PR #75478 or an equivalent narrow replacement that preserves Telegram gateway execution mode through the wrapper, dispatches targeted Telegram CLI sends through the gateway, and keeps focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level path exists on current main: the Telegram wrapper hides the gateway execution-mode hook, and the CLI loads channel plugins before message dispatch.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it keeps Telegram-specific behavior in the Telegram plugin while using the existing generic execution-mode and gateway call contracts.
