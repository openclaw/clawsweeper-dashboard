Keep open. Current main still matches the Telegram gateway-delegation bug, and the explicitly linked PR remains open/unmerged, so this issue should stay open until that fix or an equivalent lands.

## What This Changes

The issue requests that Telegram CLI sends delegate to the live gateway and return JSON without initializing the local Telegram runtime in the CLI process.

## Best Possible Solution

Review, repair if needed, and land PR #75478 or an equivalent narrow fix before closing this issue.

## Reproduction Assessment

Yes. Current main gives a high-confidence source-level reproduction path: Telegram's public wrapper hides the gateway execution-mode hook, and the CLI loads configured channel plugins before dispatch.

## Solution Assessment

Yes in direction. The maintainable path is to keep Telegram ownership in the plugin, preserve its gateway execution mode through the wrapper, and route targeted CLI sends through the existing gateway message-action contract after PR review.
