## Summary

Keep open. Current main still matches the reported Telegram gateway-delegation bug, and PR #75478 is open with closing linkage, so this issue should remain open until that implementation lands or is replaced.

## What This Changes

Request that Telegram CLI message sends delegate through the live gateway instead of initializing Telegram runtime locally and hanging.

## Best Possible Solution

Land PR #75478 or an equivalent narrow replacement that preserves Telegram gateway execution mode through the wrapper, dispatches targeted Telegram CLI sends through message.action before local runtime loading, and updates focused regression coverage.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: Telegram sends through the CLI load channel plugins before dispatch, while the visible Telegram wrapper hides the gateway-owned execution-mode hook required for delegation.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it keeps Telegram-specific behavior in the Telegram plugin, uses the existing generic execution-mode contract, and adjusts only the CLI dispatch/gateway helper behavior needed for this case.
