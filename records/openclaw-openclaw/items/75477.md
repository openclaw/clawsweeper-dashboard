## Summary

Keep open. Current main still matches the reported Telegram gateway-delegation bug, and open PR #75478 is explicitly linked as the implementation candidate, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Route Telegram CLI message sends through the live gateway message.action path without initializing Telegram runtime locally, while preserving gateway execution mode and normal CLI gateway scopes.

## Best Possible Solution

Land PR #75478 or an equivalent narrow fix that forwards Telegram gateway execution mode, avoids local Telegram runtime loading for targeted CLI sends, and keeps regression coverage in the Telegram plugin and CLI message helper tests.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: the CLI loads channel plugins before dispatch, and the visible Telegram wrapper hides the gateway-owned execution-mode hook required for delegation.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because Telegram-specific ownership stays in the Telegram plugin while the CLI dispatch path uses existing gateway message-action contracts.
