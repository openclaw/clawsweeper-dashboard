## Summary

Keep open: current main still parses the reported OpenRouter model with a 1M context window and a 262144-token output cap, so the existing proxy-like clamp does not address the endpoint-specific overflow shown in the logs.

## What This Changes

Reported regression where OpenRouter dynamic model metadata can make OpenClaw request the endpoint context length as the default output length, causing immediate context-overflow failures after upgrade.

## Best Possible Solution

Use OpenRouter's endpoint-specific context limit when available for runtime context budgeting, while preserving explicit overrides and leaving the generic transport clamp as a backstop.

## Reproduction Assessment

Yes. Source plus live catalog inspection show that the reported OpenRouter model exposes a 1M top-level context, a 262144-token endpoint context, and a 262144 endpoint completion cap that current main parses into incompatible runtime budgets.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize the OpenRouter endpoint context budget before it reaches transport clamping, not to add broader compaction or session-reset behavior.
