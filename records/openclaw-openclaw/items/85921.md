## Label Justifications

- `impact:auth-provider`: The failure is tied to provider/model routing metadata for OpenRouter and the selected model's token budget.

## Summary

Keep open: current main still trusts OpenRouter catalog `max_completion_tokens` as the model output cap, and the reported model currently advertises that value as the full endpoint context length, matching the overflow log.

## What This Changes

The issue reports that OpenRouter model calls after 2026.5.12 request the provider's full context length as output tokens, causing immediate context overflow for some models.

## Best Possible Solution

Normalize OpenRouter-discovered default output caps so catalog values equal to the provider context length do not become the default completion budget, while preserving explicit `maxTokens` overrides.

## Reproduction Assessment

Yes, source and catalog inspection give a high-confidence reproduction path: configure `openrouter/nvidia/nemotron-3-super-120b-a12b:free` and send a normal agent turn. I did not run an authenticated live inference request, but the reporter's log and current OpenRouter catalog match the current code path.

## Solution Assessment

No, asking users to hand-enter `maxTokens` for every OpenRouter model is not the best fix. The maintainable path is to normalize unsafe catalog-derived defaults while keeping explicit overrides intact.

## Repair Work Prompt

Fix the OpenRouter context-overflow regression reported in https://github.com/openclaw/openclaw/issues/85921. Current main maps OpenRouter catalog `top_provider.max_completion_tokens` directly to model `maxTokens`; for `nvidia/nemotron-3-super-120b-a12b:free`, OpenRouter currently reports both top-provider context length and max completion tokens as 262144, so OpenClaw can request a completion equal to the full endpoint context and overflow once prompt/tool tokens are added. Keep the fix bounded to OpenRouter/provider model metadata normalization: do not require users to enumerate every model, do not add a new config option, and preserve explicit user/provider/model `maxTokens` overrides. Add regression coverage for an OpenRouter catalog row whose `top_provider.max_completion_tokens` equals the provider context length, plus a case where a smaller legitimate max-output value is preserved. Add a changelog fix entry under the active version.
