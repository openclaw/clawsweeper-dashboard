## Summary

Keep open: current main and the latest release still accept and document `agents.defaults.timeFormat`, but the `sessions.list` defaults payload and Control UI timestamp renderers still do not carry or apply that value.

## What This Changes

Control UI/WebChat should respect the existing `agents.defaults.timeFormat` setting when displaying session-related timestamps.

## Best Possible Solution

Thread the existing `timeFormat` value through the `sessions.list` defaults contract and apply it to absolute Control UI/WebChat timestamp renderers, while keeping elapsed duration formatting unchanged unless maintainers explicitly define duration-specific semantics.

## Reproduction Assessment

Yes, at source level: configure `agents.defaults.timeFormat` to `"24"`; current `sessions.list` defaults and UI types cannot carry that value, and the timestamp renderers have no input to apply it. I did not run a live UI repro because the checkout review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing config value through the gateway/UI contract and preserve locale-default behavior for `auto`, rather than adding a second UI-only preference.
