## Summary

Keep open: current main and the latest release still define and document `agents.defaults.timeFormat`, but the `sessions.list` defaults contract and Control UI/WebChat timestamp renderers still do not carry or apply it.

## What This Changes

Control UI/WebChat should respect the existing `agents.defaults.timeFormat` setting when rendering session duration and timestamp display.

## Best Possible Solution

Thread `timeFormat` through the `sessions.list` defaults contract and UI types, apply it to absolute time-of-day renderers, and keep pure elapsed durations on duration-specific formatting unless a separate product decision defines otherwise.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.timeFormat` to `"24"`; current `sessions.list` defaults and UI types cannot carry that value, and the WebChat timestamp formatter has no input to apply it.

## Solution Assessment

Yes. The narrow maintainable repair is to reuse the existing config value through the gateway/UI contract and preserve locale-default behavior for `auto`, rather than adding a second UI preference or reviving the closed utility-only PR.
