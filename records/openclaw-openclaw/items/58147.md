## Best Possible Solution

Thread the existing `timeFormat` value through the sessions/defaults UI contract, apply explicit `auto`/`12`/`24` clock formatting in affected Control UI/WebChat timestamp renderers, and leave elapsed-duration formatting separate unless maintainers explicitly define that behavior.
