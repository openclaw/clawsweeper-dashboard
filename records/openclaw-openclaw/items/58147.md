## Summary

Keep open: current main still defines and documents `agents.defaults.timeFormat`, but the sessions defaults contract and affected Control UI/WebChat timestamp renderers still have no way to receive or apply that value.

## What This Changes

The issue requests that Control UI/WebChat session time displays honor the existing `agents.defaults.timeFormat` setting.

## Best Possible Solution

Thread the existing `timeFormat` value through the sessions/defaults UI contract and use it in the affected timestamp renderers, while leaving stopwatch-style durations separate unless maintainers define duration-specific semantics.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.timeFormat` to `"24"` and inspect `sessions.list` defaults plus the Control UI timestamp renderers; the value currently cannot reach or alter browser timestamp formatting.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing config value through the gateway/UI contract and preserve locale-default behavior for `auto`, rather than adding a separate UI preference.
