## Summary

Keep open: current main and latest release still render Control UI/WebChat timestamps through browser locale formatting without any `agents.defaults.timeFormat` input, while repository docs now describe that setting as applying to downstream surfaces.

## What This Changes

The issue asks Control UI/WebChat session and timestamp displays to respect the existing `agents.defaults.timeFormat` setting.

## Best Possible Solution

Reuse the existing `agents.defaults.timeFormat` contract, expose the resolved/preferred format to Control UI, and apply it in shared timestamp helpers without adding `gateway.controlUi.timezone`.

## Reproduction Assessment

Yes, at source level: current main has no `timeFormat` consumer under `ui/src`, while WebChat timestamp rendering hardcodes locale hour/minute formatting. I did not run a browser scenario in this read-only review, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse `agents.defaults.timeFormat` and centralize UI hour-format options. The separate `gateway.controlUi.timezone` feature request should not be folded into this bug fix.
