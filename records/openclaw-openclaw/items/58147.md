## Summary

Keep open: current main still accepts and documents `agents.defaults.timeFormat`, but the `sessions.list` defaults contract and Control UI timestamp renderers still have no path to receive or apply that value.

## What This Changes

The issue asks Control UI/WebChat session duration and timestamp display to honor the existing `agents.defaults.timeFormat` setting.

## Best Possible Solution

Thread the existing `timeFormat` value through the session defaults payload and UI types, then consume it in the affected timestamp renderers while keeping elapsed-duration formatting separate unless maintainers define duration-specific semantics.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.timeFormat` to `"24"`; current `sessions.list` defaults and UI types cannot carry that value, and the timestamp renderers have no input to apply it.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing config value through the gateway/UI contract and preserve locale-default behavior for `auto`, rather than adding another UI preference.
