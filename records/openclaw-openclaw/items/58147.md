## Summary

Keep open: current main still defines and documents `agents.defaults.timeFormat`, but the `sessions.list` defaults contract and Control UI timestamp renderers still have no path to receive or apply that value.

## What This Changes

The issue asks Control UI/WebChat session duration and timestamp displays to respect the existing `agents.defaults.timeFormat` setting.

## Best Possible Solution

Thread the existing `timeFormat` value through `sessions.list` defaults and Control UI types, apply it to absolute timestamp renderers, and leave elapsed durations unchanged unless maintainers explicitly define duration-specific semantics.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.timeFormat` to `"24"`; current `sessions.list` defaults and UI types cannot carry that value, and the timestamp renderers have no input to apply it.

## Solution Assessment

Yes. The maintainable fix is to reuse the existing config value through the gateway/UI contract and preserve locale-default behavior for `auto`, rather than adding another UI preference.
