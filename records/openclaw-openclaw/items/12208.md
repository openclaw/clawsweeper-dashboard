## Summary

Keep open. Current main and the latest release still lack bounded per-bubble body scrolling for native SwiftUI and Android chat bubbles, and the related WebChat reader issue is a different UI surface.

## What This Changes

The issue requests always wrapping large native chat bubble bodies in a bounded inner scroll view instead of only enabling scrolling when truncation is detected.

## Best Possible Solution

Define native tall-bubble criteria and nested-scroll gesture policy, then add bounded body scroll containers for regular and streaming SwiftUI and Android bubbles while preserving transcript scrolling and auto-scroll behavior.

## Reproduction Assessment

Yes, as a source-level check: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list. I did not run mobile device or simulator proof for the gesture interaction.

## Solution Assessment

Unclear as written: always scrolling `.large` bubbles is plausible, but current native code has no `.large` or truncation-threshold contract. Maintainers should define the tall-bubble rule and nested-scroll behavior before implementation.
