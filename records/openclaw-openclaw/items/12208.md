## Summary

Current main still lacks a bounded, independently scrollable body for tall native chat bubbles on both SwiftUI and Android; the related WebChat reader solved a different truncation path, and the remaining native interaction needs maintainer UX judgment rather than closure.

## What This Changes

Request that every native chat bubble classified as large use a bounded, independently scrollable body even when content is below the truncation threshold.

## Best Possible Solution

Define one current long-content interaction for native chat—either a bounded body scroll region with an explicit height policy or a full-message reader—and apply it consistently to persisted and streaming bubbles across iOS and Android while preserving transcript gestures.

## Reproduction Assessment

Yes at source level: current main clearly renders native message bodies without bounded inner vertical scrolling on both platforms. A physical iPhone or Android gesture reproduction was not performed.

## Solution Assessment

Unclear. Always enabling nested scrolling would address the request, but current code has no `.large` classification and a bounded reader or explicit expansion affordance may provide better transcript and accessibility behavior.
