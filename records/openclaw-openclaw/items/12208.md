## Summary

Keep open. Current `main` and `v2026.5.7` still render native chat bubble bodies as plain SwiftUI/Compose stacks inside transcript-level scrollers, and the requested always-scrollable large/tall bubble body behavior remains unimplemented.

## What This Changes

Request that every large chat bubble body use a bounded inner vertical scroll view instead of only enabling inner scrolling after truncation.

## Best Possible Solution

Define the native large/tall bubble criteria, then add bounded body-level vertical scrollers for SwiftUI and Android chat bubbles, including streaming content, while preserving transcript-level scrolling.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence path: render a tall message through `ChatMessageBubble`/`ChatMessageBody`, and the body remains a plain `VStack` or `Column` while only the surrounding transcript scroller scrolls.

## Solution Assessment

Unclear as written. Always-scroll for `.large` bubbles is plausible, but current native code has no `.large` size class or truncation threshold, so maintainers should define the UX rule before implementation.
