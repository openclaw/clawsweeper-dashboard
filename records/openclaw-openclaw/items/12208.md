## Summary

Keep open. Current main and v2026.5.2 still render native chat bubble bodies as plain SwiftUI/Compose content inside transcript-level scrollers, so the requested consistent inner scrolling for large/tall bubbles remains unimplemented.

## What This Changes

Request that all large chat bubble bodies use an inner scroll view instead of only enabling body scrolling when truncation is detected.

## Best Possible Solution

Add a bounded inner vertical scroll region for tall native chat bubble bodies on SwiftUI and Android, including streaming assistant output, after defining the large/tall threshold and preserving transcript scrolling.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: render a tall message through the current `ChatMessageBubble`/`ChatMessageBody` paths and the body remains plain `VStack` or `Column` content inside transcript-level scrollers.

## Solution Assessment

Unclear as written. Bounded inner scrolling is plausible, but current native code has no `.large` size class, so maintainers should define the height or size rule before a focused implementation PR.
