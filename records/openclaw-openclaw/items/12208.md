## Summary

Keep open. Current main and v2026.5.2 still render native chat bubble bodies as plain SwiftUI/Compose stacks inside transcript-level scrollers, so the requested always-scrollable large/tall bubble body behavior remains unimplemented.

## What This Changes

Request that all large/tall chat bubbles wrap their body content in an inner scroll view instead of only scrolling when truncation is triggered.

## Best Possible Solution

Define a tall/large bubble rule and add bounded inner vertical scrolling to SwiftUI and Android chat bubble bodies, including streaming content, while preserving transcript scrolling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render a tall message through `ChatMessageBubble`/`ChatMessageBody` and the body remains plain `VStack` or `Column` content inside transcript scrollers with no bounded inner body scroll.

## Solution Assessment

Unclear as written. The inner-scroll direction is plausible, but current native surfaces have no `.large` size class, so maintainers should define the threshold or UX rule before implementation.
