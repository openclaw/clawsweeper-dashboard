## Summary

Keep open: current main and the latest release still render native SwiftUI/Android chat bubble bodies as plain stack/column content with only transcript-level scrolling, while the merged WebChat full-message reader explicitly left inline bubble size policy out of scope.

## What This Changes

The issue requests that all large native chat bubbles use a scrollable body area even when their content has not crossed a truncation threshold.

## Best Possible Solution

Keep this open until maintainers define the native large/tall bubble policy, then add bounded inner body scrolling to the SwiftUI and Android chat bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, at source level: current SwiftUI and Android native chat bodies are plain stack/column content inside transcript-level scrolling, so tall bubbles do not have a bounded inner scroll body. I did not establish a live mobile reproduction in this read-only pass.

## Solution Assessment

Unclear: bounded inner scrolling is a plausible solution, but maintainers still need to choose the `.large` equivalent, max body height, and nested gesture policy before this should become an automated fix.
