## Summary

Keep open. Current main and v2026.5.2 still render native chat bubble bodies as plain SwiftUI/Compose content inside transcript-level scrollers, so the requested consistent inner body scrolling for large bubbles remains unimplemented.

## What This Changes

Request all large chat bubbles to wrap body content in an inner scroll container instead of only when truncation is detected.

## Best Possible Solution

Add a bounded inner vertical scroll body for tall or large native chat bubbles on SwiftUI and Android, including streaming assistant output, while preserving transcript scrolling and existing tool/attachment rendering.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: render a tall message through the current `ChatMessageBubble`/`ChatMessageBody` paths and the body remains plain `VStack` or `Column` content inside transcript-level scrollers, with no bounded inner scroll container.

## Solution Assessment

Unclear as written. The inner-scroll direction is plausible, but current native code has no `.large` size class, so maintainers should define the height or size rule before a focused implementation PR.
