## Summary

Current main still matches the prior keep-open reviews: native chat bubbles render unbounded body stacks inside transcript scrollers, and the requested `.large`/tall-bubble rule is not implemented on current main or the latest release.

## What This Changes

Request all large chat bubble bodies to use an inner scroll view even when they do not cross the truncation threshold.

## Best Possible Solution

Define the native large/tall bubble criteria, then add bounded body-level vertical scrollers for SwiftUI and Android regular and streaming bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes. Source inspection on current main shows tall chat content rendered through ChatMessageBubble/ChatMessageBody remains a plain SwiftUI VStack or Compose Column while only the surrounding transcript scroller scrolls.

## Solution Assessment

Unclear as written. Always-scrolling `.large` bubbles is plausible, but current native code has no `.large` size class or truncation contract, so maintainers should define the rule and nested-scroll behavior before implementation.
