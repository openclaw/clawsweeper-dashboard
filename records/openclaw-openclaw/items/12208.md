## Summary

Keep open. Current main and the latest release still render native iOS and Android chat bubble bodies as plain stack content with only transcript-level scrolling, and the closed WebChat full-message reader work is adjacent rather than a fix for native mobile bubble-body scrolling.

## What This Changes

Request that every large native chat bubble wrap its body in an inner scroll view even when content is not formally truncated.

## Best Possible Solution

Define the current native tall-bubble UX, then implement one bounded body-reading behavior for completed and streaming iOS/Android bubbles while preserving transcript scrolling, selection, accessibility, and mobile gesture behavior.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current native bubble bodies lack bounded inner scroll containers, but the old .large/threshold vocabulary and mobile gesture behavior still need device or simulator confirmation.

## Solution Assessment

Unclear as specified. Always-on inner scrolling is plausible, but maintainers should choose the current iOS/Android interaction model before implementation because nested scrolling can affect transcript scroll, selection, accessibility, and streaming layout.
