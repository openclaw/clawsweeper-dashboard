## Summary

Keep open. Current main still lacks body-level scrolling for native chat bubbles: regular and streaming bubble bodies are stack layouts, while scrolling remains at the transcript/list level; the request also needs a maintainer UX decision for the large/tall rule and nested scroll behavior.

## What This Changes

The issue requests that all large native chat bubbles use a scrollable body area, even when their content has not crossed a truncation threshold.

## Best Possible Solution

Define the native large/tall-bubble criteria, then add bounded body-level vertical scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, as a high-confidence source-level check: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list. I did not run a device or simulator interaction test.

## Solution Assessment

Unclear as written. Always scrolling every large bubble is plausible, but current native code has no .large or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior before implementation.
