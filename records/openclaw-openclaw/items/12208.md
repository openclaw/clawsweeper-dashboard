## Summary

Current main still renders native message bodies without a bounded inner vertical scroll container, so the requested behavior remains missing; because always-on nested scrolling changes mobile gesture and accessibility behavior, this should stay open for an explicit native UX decision rather than automatic cleanup.

## What This Changes

The issue requests a consistent inner scroll area for every native chat bubble classified as large, even when its content does not cross the truncation threshold.

## Best Possible Solution

Define a current cross-platform large-message policy, bound only the message-body region, preserve transcript scrolling and text interaction, and verify tall-but-not-truncated messages plus streaming content on real iOS and Android devices.

## Reproduction Assessment

Yes at source level: current main has no bounded inner vertical scroll container in either native message-body path, while both platforms scroll only the transcript. A live phone/simulator interaction was not run, so gesture details remain unverified.

## Solution Assessment

Unclear. Consistent body scrolling is a reasonable goal, but always wrapping every qualifying bubble is not yet proven to be the best interaction; maintainers should first define the current size rule, maximum body height, nested-gesture behavior, accessibility, and streaming parity.
