## Summary

Keep open. Current main and v2026.5.2 still render native chat bubble bodies as plain SwiftUI/Compose content inside transcript-level scrollers, so the requested consistent inner body scrolling behavior remains unimplemented.

## What This Changes

The issue requests making all large chat bubbles use a scrollable body area instead of only enabling body scrolling when truncation is triggered.

## Best Possible Solution

Add a bounded inner vertical scroll body for tall native chat bubbles on SwiftUI and Android, including streaming assistant output, while preserving transcript-level auto-scroll and existing tool/attachment rendering.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: render a tall message through `ChatMessageBubble`/`ChatMessageBody` on current main and the body remains plain `VStack` or `Column` content inside the transcript scroller, with no bounded inner scroll container.

## Solution Assessment

Yes. A focused native UI change is the narrowest maintainable direction, provided the patch defines the current equivalent of `.large` or a shared tall-bubble height threshold before enabling nested scrolling.
