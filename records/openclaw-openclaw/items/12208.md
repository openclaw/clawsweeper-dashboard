## Summary

Keep open. Current main and v2026.5.7 still render native chat bubble bodies as unbounded SwiftUI/Compose stacks inside transcript-level scrollers, and the requested large-bubble rule needs a native UX decision rather than cleanup closure.

## What This Changes

Request that every large native chat bubble use a scrollable body area, even when content does not cross the truncation threshold.

## Best Possible Solution

Define the native large/tall bubble criteria, then add bounded body-level vertical scrollers for SwiftUI and Android regular and streaming bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes. Source inspection on current main shows tall content rendered through `ChatMessageBubble`/`ChatMessageBody` remains a plain SwiftUI `VStack` or Compose `Column` while only the surrounding transcript scroller scrolls.

## Solution Assessment

Unclear as written. Always-scrolling `.large` bubbles is plausible, but current native code has no `.large` size class or truncation contract, so maintainers should define the rule and nested-scroll behavior before implementation.
