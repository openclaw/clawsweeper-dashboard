## Summary

Keep open. Current `main` and `v2026.5.2` still render native chat bubble bodies as unbounded SwiftUI/Compose content inside transcript-level scrollers, so the requested consistent inner body scroll behavior remains unimplemented.

## What This Changes

Request that every large/tall native chat bubble body use a bounded inner scroll area instead of only enabling inner scrolling after truncation detection.

## Best Possible Solution

Add a bounded inner vertical scroll body for tall native chat bubbles on SwiftUI and Android, including streaming assistant output, while preserving transcript-level auto-scroll and tool/attachment rendering.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: render a tall message through `ChatMessageBubble`/`ChatMessageBody` on current main and the body remains plain `VStack` or `Column` content inside the transcript scroller, with no inner bounded scroll container.

## Solution Assessment

Yes. A focused native UI change is the right direction, provided the patch defines the current equivalent of `.large` or a shared tall-bubble height threshold before enabling nested scrolling.
