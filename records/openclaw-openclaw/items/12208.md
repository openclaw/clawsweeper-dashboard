## Summary

Keep open. Current main and the latest release still have transcript-level scrolling, but no bounded body-level scroller for large/tall native chat bubbles, and prior issue comments reached the same keep-open conclusion.

## What This Changes

Request that large/tall chat bubbles wrap their body content in a scrollable region regardless of truncation detection.

## Best Possible Solution

Define a native large/tall bubble rule, then add bounded body-level vertical scrollers for SwiftUI and Android regular and streaming chat bubbles while preserving transcript scrolling.

## Reproduction Assessment

Yes for the current behavior by source inspection: current main routes SwiftUI bubbles through plain `VStack` bodies and Android bubbles through plain `Column` bodies, with scrolling only around the transcript. A live device or simulator check would still be needed to validate the final interaction.

## Solution Assessment

Unclear as written. Always scrolling every `.large` bubble is plausible, but current native code has no `.large` or truncation contract, so maintainers should define the tall-bubble threshold and nested-scroll gesture behavior before implementation.
