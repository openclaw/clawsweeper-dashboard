## Summary

Keep open. Current main and the latest release still render native chat bubble bodies as plain SwiftUI/Compose stack content with scrolling only around the transcript, so the requested bounded body-level scrolling remains unresolved and needs a small UX/product decision before implementation.

## What This Changes

The issue requests all large or tall native chat bubbles to wrap body content in an inner scroll view instead of only enabling scrolling after truncation.

## Best Possible Solution

Define the native large/tall-bubble criteria, then add bounded body-level vertical scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, by source inspection: regular and streaming native chat bodies render as plain SwiftUI VStack/Compose Column content, with scrolling only around the transcript/list. I did not run a device or simulator interaction test.

## Solution Assessment

Unclear as written. Always scrolling every `.large` bubble is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior before implementation.
