## Summary

Keep open. Current main and the latest release still render native iOS/SwiftUI and Android chat bubble bodies as plain stack content, while scrolling is owned by the outer transcript; the closed WebChat reader issue is adjacent but does not solve this native mobile bubble-body behavior.

## What This Changes

The issue requests that all large or tall native chat bubbles have a scrollable body regardless of whether truncation has been detected.

## Best Possible Solution

Define the current native tall-bubble rule, then implement one bounded body-reading behavior for completed and streaming bubbles across SwiftUI and active Android chat surfaces while preserving transcript scrolling and accessibility.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows native bubble bodies lack bounded inner scroll containers, but the old .large/threshold vocabulary and mobile gesture behavior still need device-level confirmation.

## Solution Assessment

Unclear as specified. Always-on inner scrolling is plausible, but maintainers should choose the current iOS/Android interaction model before implementation because nested scrolling can affect transcript scroll, selection, accessibility, and streaming layout.
