Keep open. Current main and the latest release still lack a bounded per-bubble body scroll path for native SwiftUI and Android chat bubbles; the merged WebChat full-message reader is related but does not implement the native mobile interaction requested here.

## What This Changes

Request that every native large chat bubble use a scrollable body area even when the message is not past a truncation threshold.

## Best Possible Solution

Define the current native tall-bubble interaction and implement one bounded iOS/Android body-reading behavior for completed and streaming bubbles while preserving transcript scrolling and proving mobile gestures.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main confirms the native bubble bodies lack bounded inner scroll containers, but the mobile gesture behavior still needs device-level proof.

## Solution Assessment
