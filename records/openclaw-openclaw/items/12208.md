## Summary

Keep open. Current main and the latest release still render native chat bubble bodies without a bounded per-bubble vertical scroller, and the existing issue discussion already reached the same keep-open conclusion; the remaining work needs a native large/tall-bubble rule and nested-scroll UX decision before implementation.

## What This Changes

Request all large native chat bubbles to wrap their body content in a scrollable area instead of only enabling inner scrolling after truncation.

## Best Possible Solution

Define the native large/tall bubble rule, then add bounded body-level vertical scrollers for SwiftUI and Android regular and streaming chat bubbles while preserving transcript scrolling.

## Reproduction Assessment

Yes for current-main behavior by source inspection: SwiftUI bubble bodies are plain VStacks and Android bubble bodies are plain Columns, with scrolling only at the transcript/list level. I did not run a live device or simulator to validate the interaction.

## Solution Assessment

Unclear as written. Always scrolling every `.large` bubble is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble criteria and nested-scroll gesture behavior before implementation.
