## Summary

Keep open. Current main and the latest release still lack a bounded per-bubble body scroll path for native SwiftUI and Android chat bubbles; the merged WebChat full-message reader is only adjacent, and the native mobile interaction still needs maintainer UX direction plus live device proof.

## What This Changes

The issue requests that every large native chat bubble use an inner scrollable body area, not only bubbles that exceed a truncation threshold.

## Best Possible Solution

Define the current native tall-bubble interaction and implement one bounded iOS/Android body-reading behavior for completed and streaming bubbles while preserving transcript scrolling and proving mobile gestures.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows native bubble bodies lack bounded inner scroll containers, but the old `.large`/threshold vocabulary and mobile gesture behavior still need device-level confirmation.

## Solution Assessment
