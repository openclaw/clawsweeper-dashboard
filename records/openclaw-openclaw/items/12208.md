## Summary

Keep open. Current main and the latest release still lack a bounded per-bubble body scroll path for native mobile chat; the merged WebChat full-message reader is adjacent browser/Gateway work and does not implement this iPhone/Android bubble interaction.

## What This Changes

The issue requests that all large native chat bubbles use a scrollable body area, even when content has not crossed a truncation threshold.

## Best Possible Solution

Define the current native tall-bubble rule, then implement one bounded body-reading behavior for completed and streaming bubbles across SwiftUI and active Android chat surfaces while preserving transcript scrolling, text selection, accessibility, and streaming layout.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows native bubble bodies lack bounded inner scroll containers, but the old .large terminology and mobile gesture behavior still need device-level confirmation.

## Solution Assessment

Unclear as specified. Always-on inner scrolling is plausible, but maintainers should choose the current iOS/Android interaction model before implementation because nested scrolling can affect transcript scroll, selection, accessibility, and streaming layout.
