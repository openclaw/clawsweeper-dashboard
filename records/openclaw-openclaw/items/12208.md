## Summary

Keep open. Current main and the latest release still render native chat bubble bodies as plain SwiftUI/Compose stacks inside transcript-level scrollers, so the requested large/tall bubble body scrolling remains unimplemented and needs a mobile UI product decision.

## What This Changes

The issue requests that every `.large` chat bubble wrap its body in an inner scroll view, regardless of whether truncation has been triggered.

## Best Possible Solution

Define a native large/tall bubble rule and wrap the bubble body area, including streaming content, in bounded vertical scrollers on SwiftUI and Android Compose while preserving transcript-level scrolling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render a tall message through `ChatMessageBubble`/`ChatMessageBody`, and the body remains a plain `VStack` or `Column` while only the transcript container scrolls.

## Solution Assessment

Unclear as written. The inner-scroll direction is plausible, but current native chat code has no `.large` size class, so maintainers should define the threshold or UX rule before a narrow implementation.
