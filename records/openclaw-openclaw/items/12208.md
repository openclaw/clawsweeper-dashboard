## Summary

Current main and the latest release still lack per-message bounded scrolling in the native SwiftUI and Android chat bodies, and no open or merged canonical item supersedes this native request. The issue should remain open, but the proposed always-scroll behavior needs maintainer UX judgment because the old `.large` classification no longer exists and nested scrolling can affect transcript gestures and accessibility.

## What This Changes

Add a bounded scrollable body to all large or tall native chat bubbles, regardless of whether content crosses a truncation threshold.

## Best Possible Solution

Define one cross-platform tall-message contract with a bounded body height, visible overflow affordance, preserved transcript gestures, accessibility behavior, and parity for completed and streaming messages.

## Reproduction Assessment

Yes, at the source level: both native message-body implementations lack bounded inner scrolling while their transcript containers own vertical scrolling. Device-level gesture behavior was not manually reproduced.

## Solution Assessment

Unclear. A bounded tall-message body may be appropriate, but always enabling nested scrolling needs maintainer UX judgment and should use current height or content semantics rather than the obsolete `.large` term.
