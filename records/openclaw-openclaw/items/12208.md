## Summary

Keep open. Current main still renders native chat bubble bodies as plain SwiftUI/Compose layout content with scrolling only at the transcript level, and the latest release has the same gap; the existing maintainer comment reached the same keep-open conclusion.

## What This Changes

The issue requests that all large chat bubbles use an inner scrollable body area even when their content does not cross the truncation threshold.

## Best Possible Solution

Add a bounded inner vertical scroll body for tall native chat bubbles on SwiftUI and Android, including streaming assistant output, while preserving transcript-level auto-scroll and existing tool/attachment rendering.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: render a tall message through `ChatMessageBubble`/`ChatMessageBody` on current main and the body remains a plain `VStack` or `Column` inside the transcript scroller, with no inner bounded scroll container.

## Solution Assessment

Yes, a focused native UI fix is the right direction, but the implementation should define the current equivalent of `.large` or a shared tall-bubble height threshold before wrapping the body so nested scrolling behaves predictably.
