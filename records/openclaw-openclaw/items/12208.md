## Summary

Keep open. Current main and the latest release still lack per-bubble body scrolling for native chat bubbles: regular and streaming bubble bodies are stack layouts, while scrolling remains at the transcript/list level; the requested always-scroll behavior also needs a maintainer UX decision for the large/tall rule and nested scroll gestures.

## What This Changes

Request all large/tall native chat bubbles to wrap body content in a scrollable inner area even when they do not hit a truncation threshold.

## Best Possible Solution

Define the native large/tall bubble criteria and nested-scroll gesture policy, then add bounded body-level vertical scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling and auto-scroll behavior.

## Reproduction Assessment

Yes, as a high-confidence source-level check: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list. I did not run a device or simulator interaction test.

## Solution Assessment

Unclear as written. Always scrolling every large bubble is plausible, but current native code has no .large or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior before implementation.
