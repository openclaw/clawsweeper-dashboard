## Summary

Keep open. Current main and the latest release still render native SwiftUI and Android bubble bodies as plain stack content with only transcript-level scrolling; the merged WebChat full-message reader is useful adjacent work but does not solve native large/tall bubble body scrolling.

## What This Changes

Request all large or tall native chat bubbles to wrap their body content in an inner scroll area even when the content is not truncated.

## Best Possible Solution

Define the current native tall-bubble interaction, then implement one bounded iOS and Android body-reading behavior for completed and streaming bubbles while preserving transcript scrolling and proving mobile gestures.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows native bubble bodies lack bounded inner scroll containers, but the old .large/threshold vocabulary and mobile gesture behavior still need device-level confirmation.

## Solution Assessment

Unclear as specified. The requested outcome is plausible, but the best fix needs maintainer UX choice between always-on bounded inner scroll and a reader-style interaction in the current SwiftUI and Compose components.
