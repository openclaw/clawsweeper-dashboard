## Summary

Keep open. Current main still renders native SwiftUI and Android chat bubble bodies without a bounded per-bubble vertical scroller, and the existing discussion already kept the same native UI gap open; the remaining work needs a maintainer decision on the native large/tall-bubble rule and nested scroll behavior.

## What This Changes

Request that all large/tall chat bubbles wrap their body content in an inner scroll view instead of only enabling inner scrolling after truncation.

## Best Possible Solution

Define the native large/tall bubble rule, then add bounded body-level vertical scrollers for SwiftUI and Android regular and streaming chat bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, for current-main behavior by source inspection: SwiftUI bubble bodies are plain VStacks and Android bubble bodies are plain Columns, with scrolling only at the transcript/list level. I did not run a live device or simulator to validate the interaction.

## Solution Assessment

Unclear as written. Always scrolling every `.large` bubble is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble criteria and nested-scroll gesture behavior before implementation.
