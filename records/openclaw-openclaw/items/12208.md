## Summary

Keep open. Current main and the latest release still lack a native per-bubble body scroll path for SwiftUI and Android chat bubbles; the merged WebChat full-message reader is adjacent but does not solve the iPhone/native `.large` bubble interaction.

## What This Changes

The issue requests that every `.large` chat bubble wrap its body in a scroll view, even when the content has not crossed the truncation threshold.

## Best Possible Solution

Define the native tall-bubble reading interaction, then implement one bounded iOS/Android body-reading behavior for completed and streaming bubbles while preserving transcript scrolling and proving mobile gestures.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows native bubble bodies lack bounded inner scroll containers, but the old `.large`/threshold vocabulary and mobile gesture behavior still need device-level confirmation.

## Solution Assessment

Unclear as specified. The requested outcome is plausible, but the best fix needs a maintainer UX choice between always-on bounded inner scroll and a reader-style interaction for the current SwiftUI/Compose components.
