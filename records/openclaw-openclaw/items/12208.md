## Summary

Keep open. Current main and the latest release still render native chat bubble bodies as plain SwiftUI/Compose stack content with scrolling only at the transcript level, while the requested body-level scroller still needs a maintainer UX decision for the large/tall rule and nested scroll gestures.

## What This Changes

The issue requests that all large chat bubbles wrap their body content in an inner scroll view, regardless of whether truncation has already triggered.

## Best Possible Solution

Define the native large/tall-bubble criteria, then add bounded body-level vertical scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, as a source-level check: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list. I did not run a device or simulator interaction test.

## Solution Assessment

Unclear as written. Always scrolling every large bubble is plausible, but current native code has no .large or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior before implementation.
