## Summary

Keep open. Current main and the latest release still render native SwiftUI and Android chat bubble bodies as plain stack content with scrolling owned by the outer transcript/list, while the closed WebChat full-message reader work is only adjacent and does not solve native mobile bubble-body scrolling.

## What This Changes

Request that every native large or tall chat bubble expose a scrollable body area instead of enabling inner scrolling only when truncation triggers.

## Best Possible Solution

Define the current native tall-bubble UX, then implement one bounded iOS and Android body-reading behavior for completed and streaming bubbles while preserving transcript scrolling, selection, accessibility, and mobile gesture behavior.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows native bubble bodies lack bounded inner scroll containers, but the old .large/threshold vocabulary and mobile gesture behavior still need device-level confirmation.

## Solution Assessment

Unclear as specified. Always-on inner scrolling is plausible, but maintainers should choose the current iOS/Android interaction model before implementation because nested scrolling can affect transcript scroll, selection, accessibility, and streaming layout.
