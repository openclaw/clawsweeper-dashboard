## Summary

Current main and v2026.6.5 still lack bounded inner scrolling for native chat bubble bodies; the closed WebChat full-message reader addresses retrieval of truncated content, not this iOS/Android interaction. The request remains useful, but the modern meaning of “large” and the nested-scroll behavior require a maintainer UX decision before implementation.

## What This Changes

Make every large native chat bubble body independently scrollable, regardless of whether its content crosses a truncation threshold.

## Best Possible Solution

Define one cross-platform large/tall-message policy with a bounded body height and explicit gesture/accessibility behavior, then apply it in the SwiftUI and Compose message-body owners while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, from source: current main and v2026.6.5 render native message bodies without a bounded inner vertical scroll container while the surrounding transcript scrolls. Device-level gesture behavior was not exercised.

## Solution Assessment

Unclear; always enabling scrolling for a defined large class is plausible, but maintainers first need to choose the classification, height cap, accessibility behavior, and nested-gesture contract across iOS and Android.
