## Summary

Keep open. Current main and v2026.5.7 still render native chat bubble bodies as plain SwiftUI/Compose stacks inside transcript-level scrollers, and the requested consistent body-level scrolling behavior for large/tall bubbles is not implemented.

## What This Changes

The issue requests that every large chat bubble wrap body content in a bounded inner scroll view instead of enabling inner scrolling only after a truncation threshold is crossed.

## Best Possible Solution

Define the native large/tall bubble criteria, then add bounded body-level vertical scrollers for SwiftUI and Android chat bubbles, including streaming content, while preserving transcript-level scrolling.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows tall message content flows through plain SwiftUI `VStack` and Compose `Column` bodies while only the outer transcript scrollers scroll. I did not run a mobile device or simulator render session.

## Solution Assessment

Unclear as written. Always-scrolling large bubbles is plausible, but current native code has no `.large` size class or truncation threshold, so maintainers should define the UX rule before implementation.
