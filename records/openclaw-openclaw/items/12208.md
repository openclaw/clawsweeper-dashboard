## Summary

Keep open. Current main and the latest release still lack a bounded per-bubble body scroll path for native SwiftUI and Android chat bubbles; the merged WebChat full-message reader is adjacent but does not cover the requested native/mobile interaction.

## What This Changes

The issue requests that all large chat bubbles wrap body content in a scrollable area, not only bubbles that exceed a truncation threshold.

## Best Possible Solution

Define the current native tall-bubble interaction and implement one bounded iOS/Android body-reading behavior for completed and streaming bubbles while preserving transcript-level scrolling and proving the gesture on mobile.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main shows native bubble bodies lack bounded inner scroll containers, but the mobile gesture behavior still needs device-level confirmation.

## Solution Assessment

Unclear as specified. The requested outcome is valid, but the best fix needs a maintainer UX choice between always-on bounded inner scroll and a reader-style interaction for the current SwiftUI/Compose components.
