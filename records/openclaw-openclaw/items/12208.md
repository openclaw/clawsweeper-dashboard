## Summary

Keep open. Current main and latest release still render native chat bubble bodies as plain SwiftUI/Compose stack content with only transcript-level scrolling, and the requested tall-bubble body scroller still needs a UX/product decision around the large/tall rule and nested scroll gestures.

## What This Changes

The issue requests that large chat bubbles always wrap their body content in an inner scroll view, even when they do not cross a truncation threshold.

## Best Possible Solution

Define the native large/tall-bubble criteria, then add bounded body-level vertical scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, by source inspection: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list. I did not run a device or simulator interaction test.

## Solution Assessment

Unclear as written. Always scrolling every `.large` bubble is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior before implementation.
