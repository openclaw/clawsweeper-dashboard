## Summary

Keep open. Current main and the latest release still render native chat bubble bodies as plain SwiftUI/Compose stacks inside transcript-level scrollers, so the requested always-scrollable large/tall bubble body behavior remains unimplemented.

## What This Changes

Request all large or tall native chat bubble bodies to use a bounded inner vertical scroller instead of only relying on truncation-triggered scrolling.

## Best Possible Solution

Define the native large/tall bubble criteria, then add bounded body-level vertical scrollers for SwiftUI and Android regular and streaming bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence path: tall content rendered through ChatMessageBubble/ChatMessageBody remains a plain VStack or Column while only the surrounding transcript scroller scrolls.

## Solution Assessment

Unclear as written. Always-scroll for .large bubbles is plausible, but current native code has no .large size class or truncation threshold, so maintainers should define the UX rule before implementation.
