## Summary

Keep open. Current main and the latest release still render native SwiftUI and Android chat bubble bodies as plain stack/column content with scrolling only at the transcript level, while the related WebChat reader item is a separate UI surface.

## What This Changes

Request that every native large or tall chat bubble body use an inner scroll view regardless of whether truncation has already triggered.

## Best Possible Solution

Define the native large/tall bubble criteria and nested-scroll gesture policy, then add bounded body scroll containers for regular and streaming SwiftUI and Android bubbles while preserving transcript scrolling and auto-scroll.

## Reproduction Assessment

Yes, as a source-level check: regular and streaming native chat bodies render as SwiftUI `VStack` or Compose `Column` content, with scrolling only around the transcript/list. I did not run mobile device or simulator proof for the gesture interaction.

## Solution Assessment

Unclear as written: always scrolling `.large` bubbles is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll behavior first.
