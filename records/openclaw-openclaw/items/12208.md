## Summary

Keep open. Current main and the latest release still render native SwiftUI and Android chat bubble bodies as plain stack/column content with only transcript-level scrolling, and the related WebChat full-message reader work is a separate surface rather than a canonical replacement.

## What This Changes

The issue requests that every large native chat bubble wrap its body in an inner scroll container, even when content has not crossed a truncation threshold.

## Best Possible Solution

Define native large/tall bubble criteria and nested-scroll gesture policy, then add bounded body scroll containers for regular and streaming SwiftUI and Android bubbles while preserving transcript-level scrolling and auto-scroll behavior.

## Reproduction Assessment

Yes, as a source-level check: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list. I did not run mobile device or simulator proof for the gesture interaction.

## Solution Assessment

Unclear as written: always scrolling `.large` bubbles is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll behavior first.
