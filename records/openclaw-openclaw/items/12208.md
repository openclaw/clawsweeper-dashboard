## Summary

Current main still lacks the requested native/mobile large-bubble inner scrolling behavior, and the merged WebChat full-message reader is a distinct Control UI truncation fix that explicitly did not change inline bubble size policy.

## What This Changes

The issue requests that every native/mobile `.large` or tall chat bubble wrap its body content in an inner scroll view, regardless of whether the body crosses a truncation threshold.

## Best Possible Solution

Keep this open until maintainers define the native large/tall bubble policy, then add bounded inner body scrolling to the SwiftUI and Android chat bubbles while preserving transcript-level scrolling and auto-scroll behavior.

## Reproduction Assessment

Yes, at source level: current SwiftUI and Android native chat bodies are plain stack/column content inside transcript-level scrolling, so tall bubbles do not have a bounded inner scroll body. I did not establish a live mobile reproduction in this read-only pass.

## Solution Assessment

Unclear: bounded inner scrolling is a plausible solution, but maintainers still need to choose the `.large` equivalent, max body height, and nested gesture policy before this should become an automated fix.
