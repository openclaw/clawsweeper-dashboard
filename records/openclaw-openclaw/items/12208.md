## Summary

Current main and the latest release still do not implement native per-bubble body scrolling; the remaining work is real, but the obsolete `.large` terminology and nested-scroll UX tradeoffs need maintainer product direction before an automated fix.

## What This Changes

The issue asks for all large native chat bubbles to wrap their body content in a scrollable area, even when content does not cross a truncation threshold.

## Best Possible Solution

Define one bounded native long-message interaction for completed and streaming iOS/Android bubbles, then verify real-device gesture, selection, accessibility, and short-message behavior.

## Reproduction Assessment

Yes at source level: current iOS and Android native chat bodies lack an inner vertical scroll container, while scrolling exists only at the transcript/list level. No real device or simulator reproduction was run in this read-only review.

## Solution Assessment

No, not as literally specified yet. The gap is real, but maintainers still need to translate the obsolete `.large` rule into current native components and choose between bounded inner scroll and a reader-style interaction.
