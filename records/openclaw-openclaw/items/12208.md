## Summary

Keep open: current main and the latest release still render native chat bubble bodies as plain stack/column content with only transcript-level scrolling, so the requested inner scrolling behavior remains unresolved.

## What This Changes

Request that every native `.large` or tall chat bubble body use an inner scroll view instead of only enabling body scrolling after truncation.

## Best Possible Solution

Define the native large/tall bubble policy, then add bounded inner body scroll containers to SwiftUI and Android chat bubbles while preserving transcript-level scrolling and adding focused render or manual mobile proof.

## Reproduction Assessment

Yes at source level: current SwiftUI and Android native chat bodies are plain stack/column content inside transcript-level scrolling, so tall bubbles do not have a bounded inner scroll body. I did not run a real mobile UI proof.

## Solution Assessment

Unclear until maintainers choose the UX policy: bounded inner scrolling likely addresses the reported inconsistency, but `.large` equivalence, height limits, and nested gestures should be decided before code changes.
