## Summary

Keep open. Current main and the latest release still render regular and streaming native chat bubble bodies as stack layouts, with scrolling only at the transcript/list level; the requested always-scroll behavior needs a maintainer UX decision for the large/tall rule and nested gestures.

## What This Changes

Request all large/tall chat bubbles to use a scrollable message body even when content does not hit the truncation threshold.

## Best Possible Solution

Define the native large/tall bubble criteria and nested-scroll gesture policy, then add bounded vertical body scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling and auto-scroll behavior.

## Reproduction Assessment

Yes, as a high-confidence source-level check: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list. I did not run a device or simulator interaction test.

## Solution Assessment

Unclear as written. Always scrolling every large bubble is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior before implementation.
