## Summary

Keep open. Current main and the latest release still render native chat bubble bodies as plain SwiftUI/Compose stacks inside transcript-level scrollers, so the requested large/tall bubble body scrolling remains unimplemented and needs a mobile UX decision.

## What This Changes

Request all large native chat bubbles to wrap their body content in an inner scroll view instead of enabling body scrolling only when truncation is detected.

## Best Possible Solution

Define the native large/tall bubble rule, then add bounded inner vertical scrollers to SwiftUI and Android bubble bodies, including streaming content, while preserving transcript-level scrolling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render a tall message through `ChatMessageBubble`/`ChatMessageBody`, and the body remains a plain `VStack` or `Column` while only the surrounding transcript scrolls.

## Solution Assessment

Unclear as written. The inner-scroll direction is plausible, but current native chat code has no `.large` size class or truncation threshold, so maintainers should define the UX rule before a narrow implementation.
