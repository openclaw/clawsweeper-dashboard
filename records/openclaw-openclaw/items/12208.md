## Summary

Keep open. Current main and the latest release still render native chat bubble bodies as plain SwiftUI/Compose stack content inside transcript-level scrollers, and the related WebChat full-message-reader work explicitly leaves inline chat bubble sizing out of scope.

## What This Changes

The issue requests a consistent inner scrollable body for all native large/tall chat bubbles, instead of only scrolling when truncation detection fires.

## Best Possible Solution

Define the native large/tall bubble criteria and nested-scroll gesture policy, then add bounded vertical body scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling and auto-scroll behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, as a source-level check: regular and streaming native chat bodies render as SwiftUI VStack or Compose Column content, with scrolling only around the transcript/list.

## Solution Assessment

Is this the best way to solve the issue? Unclear as written: always scrolling large bubbles is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior first.
