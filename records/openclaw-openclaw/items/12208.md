## Summary

Keep open. Current main still lacks bounded body-level scrollers in native SwiftUI and Android chat bubbles, and the prior discussion already kept this same UI gap open; the remaining decision is the native large/tall-bubble rule and nested scroll behavior.

## What This Changes

The issue requests that all large chat bubbles use an inner scrollable body, rather than enabling body scrolling only after truncation.

## Best Possible Solution

Define the native large/tall bubble criteria, then add bounded body-level vertical scrollers for regular and streaming SwiftUI and Android chat bubbles while preserving transcript-level scrolling.

## Reproduction Assessment

Yes, for current-main behavior by source inspection: regular and streaming native chat bodies render as plain VStack/Column content, with scrolling only around the transcript/list. I did not run a device or simulator interaction test.

## Solution Assessment

Unclear as written. Always scrolling every `.large` bubble is plausible, but current native code has no `.large` or truncation-threshold contract, so maintainers should define the tall-bubble rule and nested-scroll gesture behavior before implementation.
