## Summary

Current main and the latest release still lack per-bubble body scrolling on both native platforms, and no canonical issue or viable PR owns the same native behavior. The request remains useful, but its obsolete `.large` terminology and nested-scroll tradeoffs require a maintainer UX decision rather than automatic implementation.

## What This Changes

Make tall or large native chat bubble bodies independently scrollable even when they have not crossed a truncation threshold.

## Best Possible Solution

Define one bounded long-message interaction for current iOS and Android completed and streaming bubbles—inner scroll or an explicit reader—then verify transcript gestures, selection, accessibility, and short-content behavior on real devices.

## Reproduction Assessment

No device run was performed, but yes at source level: current main clearly renders completed and streaming native bubble bodies without an inner vertical scroll container on both iOS and Android.

## Solution Assessment

No, not as currently specified. The gap is real, but unconditional nested scrolling is not yet proven preferable to a bounded reader or conditional interaction across both native platforms.
