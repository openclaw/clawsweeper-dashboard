## Summary

Keep open. Current main and v2026.5.7 still render native chat bubble bodies as unbounded SwiftUI/Compose stacks inside transcript-level scrollers, and the requested `.large`/tall-bubble body scroller is not implemented.

## What This Changes

Request that every native `.large` or tall chat bubble wrap its body content in an inner scroll view, independent of truncation threshold.

## Best Possible Solution

Define native large/tall bubble criteria, then add bounded body-level vertical scrollers for SwiftUI and Android regular and streaming bubbles while preserving transcript scrolling.

## Reproduction Assessment

Yes for the current behavior. Source inspection on current main shows tall chat content renders through plain SwiftUI `VStack` and Compose `Column` bubble bodies while only the surrounding transcript scrolls.

## Solution Assessment

Unclear as written. Always-scrolling `.large` bubbles is plausible, but current native code has no `.large` size class or truncation contract, so maintainers should define the rule and nested-scroll behavior before implementation.
