## Summary

Keep open: current main still has the role-agnostic successor transcript removal path, and an open, unmerged PR already references this issue as its implementation path.

## What This Changes

The issue reports that compaction successor transcript rotation can remove assistant replies before surviving user messages, making channel replies disappear from webchat after refresh.

## Best Possible Solution

Review and land PR #76485 if its generic assistant-retention rule and regression tests are correct, or replace it with a narrower successor transcript fix in the same compaction files.

## Reproduction Assessment

Yes, at source level: a rotated transcript whose kept boundary leaves a user turn after summarized assistant messages will send those assistant entries through the current role-agnostic `removedIds` path. I did not run a live Feishu, Telegram, or webchat reproduction in this read-only review.

## Solution Assessment

Yes, the maintainable fix belongs in generic successor transcript construction with focused regression coverage, not in individual channel code. Because PR #76485 is open and linked as the implementation, this issue should stay open until that PR lands or is superseded.
