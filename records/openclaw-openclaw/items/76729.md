## What This Changes

The issue asks OpenClaw to preserve assistant replies around surviving user turns during compaction successor transcript rotation so channel replies do not disappear after `truncateAfterCompaction`.

## Best Possible Solution

Review and land or replace the compaction half of https://github.com/openclaw/openclaw/pull/76485 with a focused generic successor-transcript fix and regression coverage for assistant reply context before surviving user turns.

## Reproduction Assessment

Yes, at source level: with a transcript where a surviving user message is `firstKeptEntryId`, the assistant reply immediately before that user is summarized, added to `removedIds`, and skipped while the user remains. I did not run a live channel reproduction in this read-only review.

## Solution Assessment
