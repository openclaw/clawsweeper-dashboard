## Summary

Keep open: current main still has the role-agnostic successor transcript removal path, and an open PR already references this issue with closing syntax, so the issue should stay open until that implementation merges or is superseded.

## What This Changes

The issue reports that compaction successor transcript rotation drops assistant messages before surviving user turns, making replies disappear from WebChat/Control UI after reload.

## Best Possible Solution

Fix the generic successor transcript construction so rotated transcripts preserve assistant reply context for surviving unsummarized user turns, with focused regression coverage in the compaction successor tests.

## Reproduction Assessment

Yes, at source level: a transcript with an assistant reply before a surviving `firstKeptEntryId` user message follows the current `buildSuccessorEntries` path into `removedIds` without a role check. I did not execute a live Feishu/WebChat run in this read-only review.

## Solution Assessment

Yes: the right boundary is generic successor transcript construction, not Feishu or WebChat, because WebChat reloads from the durable transcript and rotation is shared by all channels. The open PR is the active implementation candidate and should be reviewed or superseded before this issue closes.
