## Summary

Keep open: current main and the latest release still remove summarized message entries during compaction successor rotation, so the assistant-boundary loss is not implemented/fixed. Both direct fix PRs are closed unmerged, and the remaining open compaction PR is adjacent rather than a fix for this issue.

## What This Changes

The issue reports that `truncateAfterCompaction` successor transcripts drop assistant replies before retained user turns, making channel replies disappear from WebChat and corrupting later session context.

## Best Possible Solution

Add a narrow successor-transcript repair that preserves the context-relevant assistant boundary before retained user turns, strips stale thinking signatures for preserved pre-compaction assistants, and proves the rotated transcript keeps `summary -> assistant -> user` structure.

## Reproduction Assessment

Yes, at source level: the current successor builder removes summarized message entries and the context builder replays only from `firstKeptEntryId` after the compaction summary. I did not run the live channel compaction flow in this read-only review.

## Solution Assessment

Yes, the right repair belongs in successor transcript construction rather than Feishu/WebChat-specific code, because the shared rotated transcript is the source of the missing assistant boundary. The closed PRs are useful references, but a new narrow fix should land with focused regression coverage.
