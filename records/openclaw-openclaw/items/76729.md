## Summary

Keep open: current main still has the role-agnostic successor transcript removal path, and PR #76485 is an open, unmerged implementation that references this issue with closing syntax.

## What This Changes

The issue reports that compaction successor transcript rotation can remove assistant replies before surviving user messages, causing channel replies to disappear from WebChat/Control UI after rotation.

## Best Possible Solution

Review and land PR #76485 if its generic retention rule and tests are correct, or replace it with a focused successor transcript fix that preserves assistant context for surviving unsummarized turns.

## Reproduction Assessment

Yes, at source level: a transcript whose surviving tail begins at `firstKeptEntryId` sends earlier assistant `message` entries through the current role-agnostic `removedIds` path. I did not run a live Feishu/WebChat reproduction in this read-only review.

## Solution Assessment

The right fix boundary is generic successor transcript construction with focused regression coverage, not Feishu or WebChat-specific code. The exact PR implementation still needs normal review before this issue should close.
