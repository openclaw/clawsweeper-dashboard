## Summary

Keep open: current main still has the role-agnostic successor-rotation removal path, and open PR #76485 references this issue with closing syntax, so cleanup should wait for that implementation to merge or be superseded.

## What This Changes

The issue reports that compaction successor transcript rotation drops assistant replies before surviving user messages when `truncateAfterCompaction` is enabled.

## Best Possible Solution

Review and land PR #76485 if its retention rule and tests are correct, or replace it with a narrower generic rotation fix that preserves assistant reply context for unsummarized user turns.

## Reproduction Assessment

Yes, at source level: a transcript with an assistant reply before `firstKeptEntryId` follows the current `buildSuccessorEntries` path into `removedIds` because summarized messages are removed without role checks. I did not run a live Feishu/WebChat reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs in generic successor transcript construction with focused regression coverage, not in Feishu or WebChat. The open PR is the active implementation candidate, so this issue should stay open until that PR is reviewed and merged or superseded.
