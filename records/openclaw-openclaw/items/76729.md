## Summary

Keep open: current main still removes summarized `message` entries without checking assistant/user role, and an open paired PR already references this issue with closing syntax.

## What This Changes

Compaction successor transcript rotation should preserve assistant replies that belong to surviving user turns so WebChat does not show consecutive unanswered user messages after rotation.

## Best Possible Solution

Successor transcript rotation should preserve assistant reply context for surviving unsummarized user turns with focused regression coverage, either by landing PR #76485 if correct or by replacing it with a narrower rotation-only fix.

## Reproduction Assessment

Yes, at source level: a transcript whose `firstKeptEntryId` is a surviving user message sends earlier assistant `message` entries through the current role-agnostic `removedIds` path. I did not run a live Feishu/WebChat reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs in generic successor transcript construction with regression coverage, not in Feishu or WebChat. Because an open PR already references this issue with closing syntax, the issue should stay open until that PR lands or is superseded.
