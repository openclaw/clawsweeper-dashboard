## Summary

Keep open: current main still has the role-agnostic successor-rotation removal path, and the linked open PR is an implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to preserve assistant reply context when compaction rotates a transcript so Feishu/WebChat does not reload a successor transcript with consecutive unanswered user turns.

## Best Possible Solution

Fix the generic successor transcript construction so surviving unsummarized user turns keep the assistant reply context needed for turn structure, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, at source level: construct a transcript with `user -> assistant -> user(firstKept) -> assistant -> compaction`, then rotate it; current main adds the pre-boundary assistant message to `removedIds` because every summarized `message` entry is removed. I did not run an end-to-end Feishu/UI reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the generic successor transcript builder or its boundary helper, not in Feishu or WebChat. The linked PR may be the right vehicle, but the issue should stay open until that PR or an equivalent regression-tested fix lands.
