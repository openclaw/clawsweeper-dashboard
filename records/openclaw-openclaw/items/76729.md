## Summary

Keep open: current main still has the role-agnostic successor transcript removal path, and an open linked PR already uses closing syntax for this issue, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that compaction transcript rotation drops assistant replies from the rotated webchat/session transcript after `truncateAfterCompaction` is enabled.

## Best Possible Solution

Fix the generic successor transcript builder so rotated active transcripts preserve assistant reply context for surviving user turns, with focused regression coverage and a changelog entry; review or land #76485 if it satisfies that boundary.

## Reproduction Assessment

Yes, at source level: create a transcript with `user -> assistant -> user(firstKept) -> assistant -> compaction` and rotate it. Current main removes the pre-`firstKeptEntryId` assistant because summarized message removal is role-agnostic.

## Solution Assessment

Yes, the best fix belongs in `src/agents/pi-embedded-runner/compaction-successor-transcript.ts`, not in Feishu or WebChat. The linked PR targets that area, but the issue should remain open until that PR is reviewed and merged or superseded.
