## What This Changes

Issue reports that compaction successor transcript rotation drops assistant replies before surviving user turns, causing channel replies to disappear from WebChat after `truncateAfterCompaction` rotation.

## Best Possible Solution

Review and land PR #76485 if its generic retention rule and regression coverage are correct, or replace it with a narrower successor transcript fix that preserves assistant reply context for surviving unsummarized turns.

## Reproduction Assessment

Yes, at source level: a transcript whose latest compaction keeps a later user message will remove earlier summarized assistant `message` entries through the current role-agnostic `removedIds` path. I did not run a live channel/UI reproduction in this read-only pass.

## Solution Assessment

Yes, the fix belongs in generic successor transcript construction with focused regression coverage, not in Feishu, Telegram, or WebChat-specific code. Because an open PR already references this issue with closing syntax, this issue should stay open until that implementation lands or is superseded.
