## What This Changes

The issue reports that compaction successor transcript rotation drops assistant replies before surviving user messages, making channel replies disappear from WebChat/Control UI after rotation.

## Best Possible Solution

Review and land PR #76485 if its generic retention rule and tests are correct, or replace it with a focused successor transcript fix that preserves assistant reply context for surviving unsummarized user turns.

## Reproduction Assessment

Yes, at source level: a branch where `firstKeptEntryId` points to a surviving user message will mark earlier assistant `message` entries as removed and skip them from the successor transcript. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Yes, the narrow fix belongs in generic successor transcript construction with regression coverage, not in Feishu, Telegram, or WebChat. Because an open PR already references this issue with closing syntax, the issue should stay open until that PR is merged or superseded.
