## Summary

Keep open: current main still has the role-agnostic successor transcript removal path, and the provided context shows open PR #76485 references this issue with closing syntax.

## What This Changes

The issue reports that compaction successor rotation with `truncateAfterCompaction: true` drops assistant replies before surviving user messages, causing channel replies to disappear from WebChat/Control UI.

## Best Possible Solution

Review and land PR #76485 if its generic retention rule and regression tests are correct, or replace it with a narrower successor transcript fix that preserves assistant reply context for surviving unsummarized user turns.

## Reproduction Assessment

Yes, at source level: a transcript with an assistant message before a surviving `firstKeptEntryId` user entry will add that assistant entry to `removedIds` and skip it during successor construction. I did not run a live Feishu/WebChat reproduction in this read-only review.

## Solution Assessment

Yes, the fix belongs in generic successor transcript construction with focused regression coverage, not in Feishu or WebChat. Because open PR #76485 references this issue with closing syntax, this issue should stay open until that implementation is reviewed and merged or superseded.
