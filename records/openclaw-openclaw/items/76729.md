## Summary

Keep open: current main still contains the role-agnostic successor-rotation removal path described in the report, and open PR #76485 already references this issue with closing syntax, so cleanup should wait for that implementation to merge or be superseded.

## What This Changes

The issue reports that compaction successor transcript rotation drops assistant replies before surviving user messages, making channel replies disappear from WebChat and causing later user messages to batch together.

## Best Possible Solution

Review and land PR #76485 if its retention rule and tests are correct, or replace it with a narrow generic rotation fix that preserves assistant reply context for unsummarized user turns.

## Reproduction Assessment

Yes, at source level: a transcript with an assistant reply before a surviving user message follows the current `buildSuccessorEntries` path into `removedIds` because summarized messages are removed without role checks. I did not run a live Feishu/WebChat reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs in generic successor transcript construction with focused regression coverage, not in Feishu or WebChat. The open PR is the active implementation candidate, so this issue should stay open until that PR is reviewed and merged or superseded.
