## Summary

Keep open: current main still has the generic successor-rotation removal path described in the report, and the provided GitHub context shows an open, unmerged PR already references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to preserve assistant reply context when compaction successor transcript rotation runs with `truncateAfterCompaction: true`, so replies do not disappear from webchat after rotation.

## Best Possible Solution

Review and land PR #76485 if its generic retention rule and tests are correct, or replace it with a narrower successor transcript fix that preserves assistant reply context for surviving unsummarized user turns.

## Reproduction Assessment

Yes, at source level: a transcript whose `firstKeptEntryId` is a surviving user message will send earlier assistant `message` entries through the current role-agnostic `removedIds` path. I did not run a live Feishu/WebChat reproduction in this read-only review.

## Solution Assessment

Yes, the best fix belongs in generic successor transcript construction with focused regression coverage, not in Feishu or WebChat. Because an open PR already references this issue with closing syntax, the issue should stay open until that implementation is reviewed and merged or superseded.
