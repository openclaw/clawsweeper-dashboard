## Summary

Keep open: current `main` still has the role-agnostic successor-rotation removal path described in the report, and open PR #76485 already references this issue with closing syntax, so cleanup should wait for that implementation to merge or be superseded.

## What This Changes

The issue reports that successor transcript rotation with `truncateAfterCompaction` removes assistant replies before surviving user turns, making channel replies disappear from WebChat and batching later user messages.

## Best Possible Solution

Review and land PR #76485 if its retention rule and tests are correct, or replace it with a narrower generic successor-transcript fix that preserves assistant reply context for unsummarized user turns.

## Reproduction Assessment

Yes, at source level: a transcript with an assistant message before a surviving user turn follows the current `buildSuccessorEntries` path into `removedIds` because summarized messages are removed without role checks. I did not run the full Feishu/WebChat UI scenario in this read-only pass.

## Solution Assessment

Yes, the best fix belongs in generic successor transcript construction with focused regression coverage, not in Feishu or WebChat. The open PR is the active implementation candidate, so this issue should stay open until that PR is reviewed and merged or superseded.
