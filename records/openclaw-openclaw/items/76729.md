## Summary

Keep open: current main still removes summarized message entries role-agnostically during successor transcript rotation, and open PR #76485 already references this issue with closing syntax, so cleanup should wait for that implementation to merge or be superseded.

## What This Changes

Preserve assistant replies that precede surviving user messages when compaction rotates successor transcripts so WebChat/Control UI does not show consecutive unanswered user turns.

## Best Possible Solution

Review and land PR #76485 if its retention rule and tests are correct, or replace it with a narrow generic compaction successor transcript fix plus regression coverage.

## Reproduction Assessment

Yes, at source level: current main adds every summarized `message` before `firstKeptEntryId` to `removedIds` without a role check, then skips those ids when writing the successor. I did not run a live Feishu/WebChat scenario in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix belongs in generic successor transcript construction with focused regression coverage; Feishu/WebChat are consumers of the rotated transcript, not the owner of the bug. PR #76485 is the active implementation candidate but remains unmerged.
