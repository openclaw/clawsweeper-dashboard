## Summary

Keep open: current main still has the role-agnostic successor transcript removal path, the symptom is now reported for Feishu and Telegram, and open PR #76485 references this issue with closing syntax but is not merged.

## What This Changes

The issue reports that compaction successor transcript rotation drops assistant replies when `truncateAfterCompaction` is enabled, making channel replies disappear from WebChat/Control UI.

## Best Possible Solution

Review and land PR #76485 if its generic retention rule and tests are correct, or supersede it with a narrower successor transcript fix that preserves assistant reply context for surviving unsummarized turns.

## Reproduction Assessment

Yes, at source level: with `truncateAfterCompaction` enabled, entries before `firstKeptEntryId` are marked summarized and any summarized `message` is removed without checking assistant/user role. I did not run a live Feishu or Telegram reproduction.

## Solution Assessment

Yes, the maintainable fix belongs in generic successor transcript construction with focused regression coverage, not in Feishu or Telegram channel code. Because PR #76485 already references this issue with closing syntax, the issue should stay open until that PR is reviewed and merged or superseded.
