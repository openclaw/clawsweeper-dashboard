Full review comments:

- **[P1] Preserve default snapshot isolation:** `src/secrets/provider-env-vars.ts:130`
  - body: Changing the fallback to `params?.config` means callers that omit `config`, including the lazy provider env-var exports, call `getCurrentPluginMetadataSnapshot` with `config: undefined`. That skips the policy/fingerprint check and can reuse a current snapshot from a load-path or workspace discovery context before the `requireDefaultDiscoveryContext` fallback runs, so default auth/env lookup can pick up provider aliases or env vars from the wrong plugin snapshot. Keep the default-discovery gate for no-config lookups and add a no-params variant of the existing load-path snapshot regression test.
  - confidence: 0.86
