## Summary

This PR should stay open because the user-facing CLI command is plausible, but the branch still has a concrete output-path defect and no contributor-provided proof that it generated a PNG from a real cached session.

## What This Changes

The PR adds an `openclaw context map` CLI subcommand that fetches session context-weight data through Gateway RPC, renders the existing context treemap PNG, and registers the command in CLI discovery and command policy tables.

## Best Possible Solution

Keep the CLI command approach if it creates missing output directories, includes focused validation for the custom output path, and adds real behavior proof from a cached session run.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: after a cached report exists, `openclaw context map --output missing-dir/ctx.png` will fail because the PR calls `copyFile` before creating the parent directory.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is a maintainable direction, but the branch needs recursive output-directory creation and real behavior proof before merge.
