## Summary

Keep this PR open: the CLI feature is plausible, and the `sessions.usage` method issue is fixed in the current head, but `--output` still fails for missing parent directories and the external PR still lacks real behavior proof.

## What This Changes

The PR adds an `openclaw context map` CLI command that fetches Gateway session context-weight data and renders the existing context treemap PNG from the terminal.

## Best Possible Solution

Keep the CLI command direction, but create missing output directories before copying, preserve the `sessions.usage` contract, and require real cached-session PNG proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: after a cached context report exists, `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without creating the destination parent directory.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is a maintainable approach, but the patch should mirror the existing CLI output-copy pattern with recursive parent creation and include real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
