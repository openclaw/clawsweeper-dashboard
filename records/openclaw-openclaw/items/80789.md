## Best Possible Solution

Keep the CLI command direction, create missing output parent directories before copying, add focused coverage if practical, and require redacted after-fix proof from a cached session before merge.

Yes. Source inspection of the PR head shows that after a cached report exists, `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without first creating the destination parent directory.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is a maintainable approach, but the current patch needs recursive output-directory creation and real behavior proof before it is merge-ready.
