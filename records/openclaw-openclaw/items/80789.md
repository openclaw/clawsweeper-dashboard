## Summary

Keep open: the CLI feature is plausible and now uses the right `sessions.usage` data source, but the PR head still fails for custom `--output` paths whose parent directory does not exist and it still lacks inspectable real behavior proof.

## What This Changes

The PR adds an `openclaw context map` CLI subcommand that fetches session context-weight data through Gateway RPC, renders the existing context treemap PNG, and registers the command in CLI discovery and policy tables.

## Best Possible Solution

Keep the CLI command direction, but make custom output paths create missing parent directories and require a redacted real run showing PNG generation from a cached session before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: after a cached report exists, `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without first creating `missing-dir`, so Node will throw `ENOENT`.

## Solution Assessment

No as submitted. Reusing `sessions.usage` and `renderContextTreemapPng` is maintainable, but the branch needs recursive output-directory creation and real behavior proof before it is the right implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
