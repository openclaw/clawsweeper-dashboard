## Summary

Keep open. The feature is not on current main, but the PR head still has a concrete `--output` parent-directory bug and lacks required real behavior proof; the earlier `sessions.list` concern has been fixed by switching to `sessions.usage`.

## What This Changes

The PR adds an `openclaw context map` CLI subcommand that fetches cached session context-weight data through Gateway RPC and renders a treemap PNG from the terminal.

## Best Possible Solution

Keep the CLI command direction, create missing parent directories for `--output`, add focused coverage if feasible, and require after-fix real behavior proof from a cached session before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows that after a cached report exists, `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without first creating the destination parent directory.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is maintainable, but the current patch needs recursive output-directory creation and real behavior proof before it is merge-ready.
