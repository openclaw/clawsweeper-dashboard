## Summary

Keep open: the feature is plausible and uses existing Gateway/report contracts, but the current diff still breaks the advertised output-path flow and the external PR lacks required real behavior proof.

## What This Changes

This PR adds `openclaw context map`, wiring a new CLI subcommand to fetch `sessions.usage` context-weight data and render the existing context treemap PNG from the terminal.

## Best Possible Solution

Keep the CLI command approach only after adding output-directory creation, focused coverage or equivalent manual validation, and contributor real behavior proof showing a PNG generated from a real cached run report.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows `copyFile(treemap.path, outputPath)` runs without creating `path.dirname(outputPath)`, so `openclaw context map --output missing-dir/ctx.png` will fail when the parent directory is absent.

## Solution Assessment

No. The direction is maintainable because it reuses `sessions.usage` and `renderContextTreemapPng`, but the output path handling needs to match existing CLI copy patterns and the contributor still needs real behavior proof.
