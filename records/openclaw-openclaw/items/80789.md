## Summary

Keep open: the PR is a plausible CLI feature, but it is not merge-ready because the current branch can fail for custom output paths whose parent directory does not exist, and it still lacks required real behavior proof.

## What This Changes

The PR adds an `openclaw context map` CLI subcommand wired into the core CLI catalog and loader to fetch `sessions.usage` context reports and render a treemap PNG.

## Best Possible Solution

Keep the CLI command direction, but require parent-directory creation for `--output` and contributor-provided proof showing a real session treemap PNG before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `copyFile(treemap.path, outputPath)` without creating `path.dirname(outputPath)`, so `openclaw context map --output missing-dir/ctx.png` would fail with `ENOENT` once a treemap is generated.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is maintainable, but the output-copy path needs the existing recursive directory-creation pattern and the PR still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect
