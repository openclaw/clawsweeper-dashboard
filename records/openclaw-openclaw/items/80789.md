## Summary

Keep this PR open, but the patch is currently not mergeable: the new command asks `sessions.list` for fields that its validated parameter contract does not accept, so the command cannot reach the renderer path. The PR also needs real after-fix behavior proof beyond build/typecheck/help output.

## What This Changes

Adds an `openclaw context map` CLI subcommand that fetches a cached session context report from the gateway and writes the existing treemap PNG renderer output to disk.

## Best Possible Solution

Keep the CLI wrapper but fetch reports through the supported `sessions.usage` contract, or intentionally add and test a `sessions.list` report field before wiring the command to it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: every `openclaw context map` invocation sends `includeContextWeight` to `sessions.list`, and the current gateway schema rejects that parameter before the command can render a PNG.

## Solution Assessment

No. Reusing `renderContextTreemapPng` is a good direction, but this PR uses the wrong gateway method; the narrow fix is to use the existing `sessions.usage` report path or add a supported `sessions.list` contract with regression coverage.
