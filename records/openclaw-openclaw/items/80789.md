## Summary

Keep open: the CLI feature is not on current main and the direction is plausible, but the current PR head still has a concrete `--output` parent-directory bug and lacks the required real behavior proof for cached-session PNG generation.

## What This Changes

The PR adds an `openclaw context map` CLI subcommand that fetches cached session context data through Gateway RPC and renders the existing context treemap PNG from the terminal.

## Best Possible Solution

Keep the CLI command direction, add recursive parent-directory creation before copying custom outputs, and require redacted real cached-session PNG proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: after a cached report exists, `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without creating the destination parent directory; I did not execute it because this review is read-only.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is maintainable, but the current patch should create output parents and include inspectable real behavior proof before it is merge-ready.
