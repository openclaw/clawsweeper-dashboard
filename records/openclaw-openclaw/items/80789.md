## Summary

Keep open: this is a plausible user-facing CLI feature, but the PR is not merge-ready because the current head still fails for `--output` paths whose parent directory does not exist and the external contributor has not provided real cached-session PNG-generation proof. The earlier `sessions.list` concern appears fixed in code by the switch to `sessions.usage`.

## What This Changes

The PR adds an `openclaw context map` CLI subcommand, registers it in CLI discovery and policy catalogs, and renders context treemap PNGs from Gateway `sessions.usage` data.

## Best Possible Solution

Land a revised core CLI command only after it creates missing output directories and demonstrates real PNG generation from a cached session report.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: after a cached report exists, `openclaw context map --output missing-dir/ctx.png` will call `copyFile` before creating the parent directory and fail with `ENOENT`.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is a maintainable direction, but the branch needs recursive parent-directory creation and real behavior proof before it is the right implementation to merge.
