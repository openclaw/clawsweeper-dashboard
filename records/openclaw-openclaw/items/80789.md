## Summary

Keep open: the feature is not on current main, and the PR head uses the right `sessions.usage` contract, but the custom `--output` path still fails when the parent directory is missing and the PR has no real behavior proof.

## What This Changes

The branch adds an `openclaw context map` CLI subcommand that fetches cached session context-weight data through Gateway RPC and renders the existing context treemap PNG from the terminal.

## Best Possible Solution

Keep the CLI command direction, but make custom output paths create missing parents and require inspectable real behavior proof from a cached session before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: after a cached report exists, `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without creating the destination parent directory.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is maintainable, but the current patch needs recursive output-directory creation and real behavior proof before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
