## Summary

Keep open: the command is not on current main and is a plausible user-facing CLI feature, but PR head still fails custom output paths whose parent directory does not exist and lacks required real behavior proof.

## What This Changes

The PR adds an `openclaw context map` CLI subcommand wired into the core CLI catalog/loader that fetches session context-weight data via Gateway RPC and renders the existing context treemap PNG.

## Best Possible Solution

Keep the CLI feature direction, add recursive parent-directory creation for `--output`, and require inspectable cached-session PNG generation proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a cached context report, `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without creating `path.dirname(outputPath)`.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is maintainable, but the patch should create output parents like existing CLI output commands and provide real behavior proof.
