## Summary

Keep open: this is a plausible CLI feature PR, but the current head still fails custom `--output` paths whose parent directory does not exist and the PR lacks after-fix real behavior proof. The older `sessions.list` concern appears resolved at the current head because the command now calls `sessions.usage`.

## What This Changes

The branch adds a `context map` CLI subcommand that fetches `sessions.usage` context-weight data over Gateway RPC, renders the existing treemap PNG, and registers the command in CLI discovery and policy tables.

## Best Possible Solution

Land the CLI feature only after it creates missing output directories, preserves the `sessions.usage` contract, adds focused validation for custom output paths, and includes redacted real PNG-generation proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows `openclaw context map --output missing-dir/ctx.png` reaches `copyFile` without creating the parent directory once a cached report exists. The full PNG-generation flow has not been live-proved in the PR evidence.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is a maintainable direction, but the branch needs recursive output-directory creation and real behavior proof before merge.
