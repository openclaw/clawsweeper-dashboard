## Summary

Keep open. This is a recent external feature PR with a plausible CLI direction, but current head still has a blocking custom-output path bug and the PR body does not prove actual PNG generation from a real cached session.

## What This Changes

The PR adds a core `openclaw context map` CLI subcommand, wires it into CLI discovery and command policy tables, fetches session context-weight data through Gateway RPC, and renders the existing context treemap PNG with optional JSON output.

## Best Possible Solution

Keep the CLI command direction, but require recursive parent-directory creation for `--output`, focused validation of that path, and redacted real behavior proof showing PNG generation from a cached session.

## Reproduction Assessment

Yes for the concrete PR defect: source inspection shows `openclaw context map --output missing-dir/ctx.png` will reach `copyFile` before creating the parent directory after a cached report exists. The intended PNG-generation feature itself still lacks real setup proof in the PR body.

## Solution Assessment

No. Reusing `sessions.usage` and `renderContextTreemapPng` is a maintainable direction, but the current patch is not merge-ready until it handles missing output directories and proves the real CLI flow.
