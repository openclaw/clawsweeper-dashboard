## Summary

Keep open: this is a focused new CLI feature that is not on current main and is not a duplicate, but the external PR still lacks real runtime proof and has one small `--output` path handling issue.

## What This Changes

The PR adds a core `openclaw context map` subcommand that fetches cached session context weight from the Gateway and renders it with the existing context treemap PNG renderer.

## Best Possible Solution

Land the CLI command after adding parent-directory creation for `--output` and after the contributor provides redacted real behavior proof for a successful PNG render through `sessions.usage`.

## Reproduction Assessment

Not applicable for the feature request itself. Source inspection gives a high-confidence review path for the proposed implementation, but no live runtime proof has been provided.

## Solution Assessment

Mostly yes: reusing `sessions.usage` plus `renderContextTreemapPng` is the narrow maintainable approach. The implementation should still create the output parent directory and wait for real behavior proof before merge.
