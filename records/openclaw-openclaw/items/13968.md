## Summary

Keep open. Current main still reads and snapshots the fixed workspace bootstrap file set, including `HEARTBEAT.md`, before later run-kind and heartbeat-guidance filters; the existing related items cover narrower heartbeat `lightContext` forwarding or broader lazy-context configuration, not this exact pre-load optimization.

## What This Changes

Request on-demand `HEARTBEAT.md` bootstrap loading so ordinary non-heartbeat runs skip it before workspace file loading and cache snapshotting.

## Best Possible Solution

Keep this open until maintainers decide the normal-run contract, then implement a run-kind-aware raw loader/cache path that skips `HEARTBEAT.md` before reads for default non-heartbeat runs, preserves it for heartbeat runs, and updates focused tests plus heartbeat/system-prompt docs.

## Reproduction Assessment

No high-confidence bug reproduction is applicable because this is an enhancement, not broken documented behavior. The current eager-load behavior is source-reproducible: the resolver loads raw bootstrap files before filters, and the raw loader still includes `HEARTBEAT.md`.

## Solution Assessment

Yes, if maintainers accept the behavior change: the narrow maintainable solution is pre-load/cache filtering keyed by heartbeat run kind. A broader configurable lazy-context feature should stay with https://github.com/openclaw/openclaw/issues/61347.
