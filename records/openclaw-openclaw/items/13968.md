## Summary

Keep open. Current main still reads and caches the fixed workspace bootstrap file set, including HEARTBEAT.md, before run-kind filtering; the related lightweight heartbeat fix and broader selective-loading feature request do not implement this specific on-demand loading behavior.

## What This Changes

Request run-kind-aware workspace bootstrap loading so HEARTBEAT.md is skipped for ordinary non-heartbeat turns and preserved for real heartbeat runs.

## Best Possible Solution

Implement a run-kind-aware bootstrap loader and cache path that excludes HEARTBEAT.md for default non-heartbeat runs before disk reads, cache snapshots, hooks, and prompt injection, while preserving HEARTBEAT.md for actual heartbeat runs and updating the prompt-context docs and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: call resolveBootstrapFilesForRun for a default/full run with HEARTBEAT.md present and heartbeat guidance enabled, and the current loader/cache path will read the file before any later filtering.

## Solution Assessment

Yes, the narrowest maintainable fix is to thread run kind or an equivalent heartbeat-only flag into the bootstrap load/cache layer. The existing #55311 lightweight heartbeat forwarding path is useful but does not solve normal-run eager loading.
