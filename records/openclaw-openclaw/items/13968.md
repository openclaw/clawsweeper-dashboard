## Summary

Keep open. Current main still reads and caches the fixed workspace bootstrap file set, including HEARTBEAT.md, before later run-kind or heartbeat-guidance filters; the related lightweight-heartbeat fix and broader lazy-context request do not replace this focused pre-load optimization.

## What This Changes

Request on-demand HEARTBEAT.md loading so normal non-heartbeat runs avoid loading heartbeat-only context while heartbeat runs still receive it.

## Best Possible Solution

Thread run kind or equivalent heartbeat intent into the bootstrap loader/cache so HEARTBEAT.md is skipped before disk reads and snapshots for default non-heartbeat runs, while preserving real heartbeat behavior and documenting the new contract.

## Reproduction Assessment

Yes. Source inspection on current main shows a default/full run reaches resolveBootstrapFilesForRun, which calls getOrLoadBootstrapFiles/loadWorkspaceBootstrapFiles before later filters; loadWorkspaceBootstrapFiles still includes HEARTBEAT.md in the fixed entries list.

## Solution Assessment

Yes. The narrow maintainable fix is to make the loader/cache run-kind aware before reading or snapshotting files, rather than adding another post-load filter; the broader configurable lazy-context design can stay with #61347.
