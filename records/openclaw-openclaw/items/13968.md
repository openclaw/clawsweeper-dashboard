## Summary

Keep open. Current main still reads and snapshots `HEARTBEAT.md` before run-kind filtering, and the docs still allow ordinary default-agent runs to include it when heartbeat guidance is enabled; related work covers adjacent lightweight heartbeat or broader selective-context behavior, not this narrow loader/cache contract.

## What This Changes

Request skipping `HEARTBEAT.md` from workspace bootstrap loading for non-heartbeat runs so it is loaded only for heartbeat polls.

## Best Possible Solution

Define a run-kind-aware bootstrap load/cache contract that skips `HEARTBEAT.md` before disk reads, session snapshots, hooks, and prompt injection for ordinary non-heartbeat runs while preserving real heartbeat behavior and updating docs/tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main still reads and caches `HEARTBEAT.md` before any run-kind filter can omit it.

## Solution Assessment

Unclear until maintainer product/API confirmation. Passing run-kind or heartbeat state into the loader/cache is the narrow technical shape, but a post-load filter would not satisfy the requested load/cache savings and the docs contract must change.
