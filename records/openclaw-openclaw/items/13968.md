Keep open. Current main still loads and caches HEARTBEAT.md before run-kind filtering, while the closed lightweight-heartbeat fix and the broader selective/lazy context issue do not supersede this narrower loader/cache contract.

## Best Possible Solution

Define a documented run-kind-aware bootstrap load/cache contract that skips HEARTBEAT.md before reads, snapshots, hooks, and prompt injection for ordinary non-heartbeat runs while preserving real heartbeat behavior.

## Solution Assessment
