## What This Changes

The issue requests run-kind-aware bootstrap loading so HEARTBEAT.md is skipped for ordinary non-heartbeat runs and loaded only for heartbeat runs.

## Best Possible Solution

Define and document run-kind-aware bootstrap load/cache semantics that skip HEARTBEAT.md before disk reads and cache snapshots for ordinary non-heartbeat runs, or explicitly fold that requirement into the selective/lazy context design.

## Reproduction Assessment

Yes for the source-level behavior, though this is a feature/performance request rather than a strict bug repro. Current main still loads HEARTBEAT.md in the raw bootstrap loader/cache before run-kind filtering can omit it.

## Solution Assessment

Unclear until maintainers settle the prompt contract. Passing run-kind state into the loader/cache is the likely narrow technical shape, but it changes documented bootstrap semantics and overlaps broader selective/lazy context work.
