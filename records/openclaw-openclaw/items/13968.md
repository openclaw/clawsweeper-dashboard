## Best Possible Solution

Define a maintainer-approved run-kind-aware bootstrap loader/cache contract that skips HEARTBEAT.md for normal non-heartbeat runs before raw reads and snapshot caching, preserves it for heartbeat runs, and updates docs plus focused bootstrap tests for the new normal-run contract.
