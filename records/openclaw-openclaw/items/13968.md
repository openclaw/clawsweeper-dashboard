## Summary

Keep this issue open: current main still eagerly loads HEARTBEAT.md and, on non-Codex harnesses, includes it in ordinary default-agent turns when heartbeat guidance is enabled; the related merged work addresses lightweight heartbeat routing and cache refresh, not the requested normal-turn behavior.

## What This Changes

Load HEARTBEAT.md only when needed for heartbeat runs instead of including it in ordinary message context.

## Best Possible Solution

Define whether ordinary non-Codex default-agent turns should retain heartbeat guidance, then apply one canonical run-kind-aware file-selection policy before workspace reads and snapshot caching while preserving native Codex behavior, hook invariants, heartbeat runs, documentation, and focused regression coverage.

## Reproduction Assessment

Yes. Current source and documentation provide a high-confidence source reproduction: non-Codex default-agent runs can load and inject HEARTBEAT.md when heartbeat guidance is enabled, and the shared cache reads the file before filtering.

## Solution Assessment

No, not exactly as proposed. Passing a narrow isHeartbeat boolean would bypass existing run-kind, harness, agent-policy, context-mode, cache, and hook decisions; a shared early file-selection policy is the more maintainable solution.
