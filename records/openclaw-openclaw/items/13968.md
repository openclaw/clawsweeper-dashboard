## Summary

Keep open: current main and v2026.6.10 still load and snapshot HEARTBEAT.md before run-kind filtering, while docs still allow HEARTBEAT.md in normal default-agent runs when heartbeat guidance is enabled.

## What This Changes

Request run-kind-aware/on-demand loading so HEARTBEAT.md is skipped for ordinary non-heartbeat turns while remaining available for heartbeat polls.

## Best Possible Solution

Define one run-kind-aware bootstrap loading/cache policy that skips HEARTBEAT.md for ordinary non-heartbeat default runs before reads, cache snapshots, hooks, and prompt injection, while preserving real heartbeat runs and updating docs/tests.

## Reproduction Assessment

Yes, at source level. Current main still reads and snapshots HEARTBEAT.md before run-kind filtering, and normal default-agent runs still keep it when heartbeat guidance is enabled.

## Solution Assessment

No, not as only passing an isHeartbeat flag through the two named functions. The maintainable fix needs one shared run-kind-aware policy across loader, cache, hooks, prompt injection, docs, and tests.
