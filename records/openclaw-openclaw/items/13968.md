## Summary

Keep open: current main and the latest release still read and cache `HEARTBEAT.md` before run-kind filtering, and the documented default-agent contract still permits normal runs to receive it when heartbeat guidance is enabled. Related tiered-bootstrap and broader context-selection work overlaps, but it does not replace this narrow on-demand loading decision.

## What This Changes

The issue requests loading `HEARTBEAT.md` only for heartbeat runs instead of including it in every normal workspace bootstrap load.

## Best Possible Solution

Define the desired normal-run heartbeat guidance contract, then make the loader and per-session cache run-kind aware before disk reads, snapshot caching, hooks, and prompt injection, with docs and focused tests updated together.

## Reproduction Assessment

Yes. At source level, current main eagerly reads and snapshots `HEARTBEAT.md` before run-kind filtering, and ordinary full-context default-agent runs keep it when heartbeat guidance is enabled.

## Solution Assessment

No, not as only passing an `isHeartbeat` flag to two functions. The maintainable fix needs one shared run-kind-aware policy across loader, cache, hooks, prompt injection, docs, and tests, with maintainer approval for the default behavior change.
