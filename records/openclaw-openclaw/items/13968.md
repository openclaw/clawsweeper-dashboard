## Summary

Keep open: current main and v2026.5.12 still read and cache HEARTBEAT.md before run-kind filtering, and the related lightweight-heartbeat fix plus broader selective-context issue do not replace this narrower loader/cache contract.

## What This Changes

Request run-kind-aware bootstrap loading so HEARTBEAT.md is skipped before read/cache/prompt injection on ordinary non-heartbeat turns.

## Best Possible Solution

Define a documented run-kind-aware bootstrap load/cache contract that skips HEARTBEAT.md before reads, snapshots, hooks, and prompt injection for ordinary non-heartbeat runs while preserving real heartbeat behavior, or explicitly fold this narrower requirement into the selective/lazy context design.

## Solution Assessment

Unclear until maintainer product/API confirmation. Passing run-kind or heartbeat state into the loader/cache is the likely narrow technical shape, but it changes the documented prompt contract and overlaps the broader selective/lazy workspace-context design.
