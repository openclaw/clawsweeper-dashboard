## What This Changes

Request to skip loading HEARTBEAT.md for non-heartbeat runs so ordinary messages avoid unnecessary workspace-context tokens.

## Best Possible Solution

Define one run-kind-aware bootstrap policy, then apply it before raw file reads, snapshot caching, hooks, prompt injection, docs, and focused tests.

## Solution Assessment

No, not as only passing an isHeartbeat flag through the two named functions. The maintainable fix needs one shared run-kind-aware policy across loader, cache, hooks, prompt injection, docs, and tests.
