## Summary

Keep open. Current main still reads and caches the fixed workspace bootstrap file set, including HEARTBEAT.md, before run-kind filtering; the related lightweight-heartbeat fix and broader selective-loading request do not implement this narrower on-demand loading behavior.

## What This Changes

Request on-demand HEARTBEAT.md loading so ordinary non-heartbeat turns avoid reading and injecting heartbeat guidance.

## Best Possible Solution

Implement a run-kind-aware bootstrap loader/cache path that omits HEARTBEAT.md for default non-heartbeat runs before reads and snapshots, while preserving real heartbeat behavior and updating focused docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a default/full run with HEARTBEAT.md present still calls loadWorkspaceBootstrapFiles through the resolver/cache before any later heartbeat filtering.

## Solution Assessment

Yes. The narrow maintainable fix is to thread run kind or equivalent heartbeat-only intent into loadWorkspaceBootstrapFiles/getOrLoadBootstrapFiles instead of adding another post-load filter.
