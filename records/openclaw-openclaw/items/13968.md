## Summary

Keep open. Current main still reads and snapshots `HEARTBEAT.md` before run-kind filtering, and the documented normal-run contract still allows `HEARTBEAT.md` when heartbeat guidance is enabled; related work only covers adjacent lightweight heartbeat and broader selective-context behavior.

## What This Changes

Request to load `HEARTBEAT.md` only for heartbeat runs so ordinary non-heartbeat turns avoid the extra prompt/context cost.

## Best Possible Solution

Define a maintainer-approved run-kind-aware bootstrap load/cache contract that skips `HEARTBEAT.md` before disk reads, session snapshots, hooks, and prompt injection for ordinary non-heartbeat runs while preserving real heartbeat behavior and updating docs/tests.

## Solution Assessment

Unclear until maintainer product/API confirmation. Passing run-kind or heartbeat state into the loader/cache is the narrow technical shape, but a post-load filter would not satisfy the requested load/cache savings and the docs contract must change.
