## Summary

Keep open. Current main still reads and caches `HEARTBEAT.md` before run-kind filtering, and the docs still allow normal default-agent runs to include it when heartbeat guidance is enabled; related work is adjacent but does not implement this loader/cache contract.

## What This Changes

Request to skip `HEARTBEAT.md` loading for ordinary non-heartbeat runs by threading heartbeat/run-kind state into bootstrap file resolution.

## Best Possible Solution

Define a run-kind-aware bootstrap load/cache contract that skips `HEARTBEAT.md` before disk reads, session snapshots, hooks, and prompt injection for ordinary non-heartbeat runs while preserving real heartbeat behavior and updating docs/tests.

## Reproduction Assessment

Not applicable as a feature/performance request. Source inspection on current main gives high confidence that `HEARTBEAT.md` is still read and cached before any run-kind filter can omit it.

## Solution Assessment

Unclear until maintainer product/API confirmation. Passing run-kind or heartbeat state into the loader/cache is the narrow technical shape, but a post-load filter would not satisfy the requested load/cache savings and the docs contract must change.
