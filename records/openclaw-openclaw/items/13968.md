## Summary

Keep open: current main and v2026.6.6 still eagerly load and cache HEARTBEAT.md before run-kind filtering, and normal default-agent runs still inject it when heartbeat guidance is enabled. The request remains a narrow product/contract decision rather than a stale duplicate.

## What This Changes

The issue requests loading HEARTBEAT.md only for heartbeat runs instead of including it in every normal workspace bootstrap context load.

## Best Possible Solution

Define the desired normal-run heartbeat guidance contract, then make the bootstrap loader and cache run-kind aware before disk read, snapshot caching, hooks, and prompt injection, with docs and focused tests updated together.

## Reproduction Assessment

Yes, at source level: current main eagerly reads and snapshots HEARTBEAT.md before run-kind filtering, and ordinary full-context default-agent runs keep it when heartbeat guidance is enabled. I did not run a gateway session because this is a read-only review.

## Solution Assessment

No, not as only an isHeartbeat flag passed to two functions. The maintainable fix needs one shared run-kind-aware policy across the loader, per-session cache, hooks, prompt injection, docs, and tests.
