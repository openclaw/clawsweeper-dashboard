## Summary

Keep open. Current main still eagerly loads and caches the fixed bootstrap file set, including HEARTBEAT.md, before later run-kind filters, and the docs still describe normal-run HEARTBEAT.md injection when heartbeat guidance is enabled.

## What This Changes

Request run-kind-aware bootstrap loading so HEARTBEAT.md is loaded for heartbeat runs but skipped before disk read, cache, and injection on normal non-heartbeat runs.

## Best Possible Solution

Decide the normal-run contract, then implement pre-load and cache filtering keyed by run kind so non-heartbeat default runs skip HEARTBEAT.md before disk reads, snapshots, hooks, and prompt injection while real heartbeat and lightweight heartbeat behavior stays intact.

## Reproduction Assessment

Yes for the reported eager-loading behavior: source inspection shows raw bootstrap files are loaded and cached before filters, with HEARTBEAT.md in the fixed loader entries. This is an enhancement rather than broken documented behavior.

## Solution Assessment

Yes, if maintainers accept the behavior change: run-kind-aware raw loading and cache snapshots are the narrowest maintainable fix. It should wait for product approval because current docs intentionally allow normal-run HEARTBEAT.md injection when heartbeat guidance is enabled.
