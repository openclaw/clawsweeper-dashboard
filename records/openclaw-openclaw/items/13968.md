## Summary

Keep open: current main still reads and caches the fixed workspace bootstrap set, including `HEARTBEAT.md`, before run-kind filtering, and normal default-agent runs can still include heartbeat guidance when cadence is enabled. The related selective/lazy context issues are broader umbrellas, while this issue remains the narrow HEARTBEAT.md loader/cache change.

## What This Changes

Request to skip `HEARTBEAT.md` for non-heartbeat runs before workspace bootstrap read, cache, and prompt injection.

## Best Possible Solution

Add a maintainer-approved run-kind-aware bootstrap load/cache contract that omits `HEARTBEAT.md` for ordinary non-heartbeat runs before disk reads, cache snapshots, hooks, and prompt injection, while preserving real heartbeat behavior and updating docs/tests.
