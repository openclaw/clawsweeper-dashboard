## Summary

Keep open. Current main still loads and caches the fixed workspace bootstrap file set, including `HEARTBEAT.md`, before later run-kind and heartbeat-guidance filtering; the related lightweight-heartbeat and broad lazy-context items do not supersede this focused pre-load optimization.

## What This Changes

Request run-kind-aware bootstrap loading so non-heartbeat runs skip `HEARTBEAT.md` before load/cache/prompt injection while heartbeat runs keep it.

## Best Possible Solution

Keep this open and decide the contract explicitly; if accepted, make the loader and session snapshot cache run-kind aware, preserve `HEARTBEAT.md` for real heartbeat runs, and update docs/tests for the new normal-run behavior.

## Reproduction Assessment

High-confidence bug reproduction is not applicable because this is an enhancement, not a broken existing contract. The current eager-load behavior is source-reproducible on main: `resolveBootstrapFilesForRun` loads raw bootstrap files before filters, and the raw loader still includes `HEARTBEAT.md`.

## Solution Assessment

Yes, if maintainers accept the behavior change. The narrow maintainable solution is pre-load/cache filtering keyed by heartbeat run kind, not another post-load filter; the broader configurable lazy-context work can stay with #61347.
