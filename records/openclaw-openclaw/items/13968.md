## What This Changes

The issue asks OpenClaw to load HEARTBEAT.md only for heartbeat runs instead of reading and injecting it during ordinary non-heartbeat turns.

## Best Possible Solution

Define run-kind-aware bootstrap load/cache semantics that skip HEARTBEAT.md before disk reads and cache snapshots for ordinary non-heartbeat runs, preserve real heartbeat behavior, and update docs/tests for the new prompt contract.

## Reproduction Assessment

Yes for source-level behavior: current main still reads HEARTBEAT.md in the raw workspace bootstrap loader/cache before any normal-run filter can omit it. I did not run a live heartbeat/session path because this is a feature/performance contract request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers settle the prompt contract. Passing run-kind state into the loader/cache is the likely narrow technical shape, but it changes documented HEARTBEAT.md bootstrap semantics and overlaps broader selective/lazy context work.
