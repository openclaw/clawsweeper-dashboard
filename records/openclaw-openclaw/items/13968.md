## Summary

Keep open. Current main still loads and caches the fixed workspace bootstrap file set, including HEARTBEAT.md, before later run-kind filtering, and the public docs still describe normal-run HEARTBEAT.md injection when heartbeat guidance is enabled.

## What This Changes

Request to make HEARTBEAT.md loading run-kind aware so non-heartbeat runs can skip it before bootstrap loading, caching, and prompt injection.

## Best Possible Solution

Decide the normal-run HEARTBEAT.md contract, then implement run-kind-aware raw loading and cache snapshots so default non-heartbeat runs skip HEARTBEAT.md before disk reads, snapshots, hooks, and prompt injection while heartbeat runs keep the existing behavior.

## Reproduction Assessment

Yes for the reported eager-loading behavior: current-main source shows raw bootstrap files are loaded and cached before filters, with HEARTBEAT.md in the fixed loader entries. This is an enhancement rather than a broken documented behavior.

## Solution Assessment

Yes, if maintainers accept the behavior change: run-kind-aware raw loading and cache snapshots are the narrowest maintainable fix. It should wait for product approval because current docs intentionally allow normal-run HEARTBEAT.md injection when guidance is enabled.
