## Summary

Keep open. Current main still reads and caches `HEARTBEAT.md` before run-kind filtering, while docs and tests continue to allow default non-heartbeat runs to receive heartbeat context when heartbeat guidance is enabled; the related selective-context items are broader or adjacent rather than a clear canonical replacement.

## What This Changes

Request run-kind-aware bootstrap loading so ordinary non-heartbeat runs omit `HEARTBEAT.md` before load/cache/context injection while heartbeat runs keep it.

## Best Possible Solution

Define a maintainer-approved run-kind-aware bootstrap load/cache contract that omits HEARTBEAT.md before disk reads, session snapshot caching, hooks, and prompt injection for ordinary non-heartbeat runs, while preserving real heartbeat behavior and updating docs/tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main reads and caches `HEARTBEAT.md` before any run-kind filter can omit it.

## Solution Assessment

Unclear until maintainer product confirmation. Passing run-kind or heartbeat state down into the loader/cache is the narrow technical shape, but it changes the current normal-run prompt-context contract and a post-load filter would not deliver the requested savings.
