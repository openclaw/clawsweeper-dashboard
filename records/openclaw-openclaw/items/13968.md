## Summary

Keep open: current main and v2026.6.6 still eagerly load and cache HEARTBEAT.md before run-kind filtering, and documented normal default-agent behavior still permits HEARTBEAT.md injection when heartbeat guidance is enabled. Related broader and tiered-bootstrap work overlaps, but it does not resolve this narrow normal-run/on-demand contract.

## What This Changes

The issue requests skipping HEARTBEAT.md for non-heartbeat runs so heartbeat-only workspace guidance is loaded only when needed.

## Best Possible Solution

Define the desired normal-run heartbeat guidance contract, then make the loader and per-session cache run-kind aware before disk reads, snapshot caching, hooks, and prompt injection, with docs and focused tests updated together.

## Reproduction Assessment

Yes, at source level: current main eagerly reads and snapshots HEARTBEAT.md before run-kind filtering, and ordinary full-context default-agent runs keep it when heartbeat guidance is enabled. I did not run a live gateway session because this is a read-only review.

## Solution Assessment

No, not as only passing an isHeartbeat flag to two functions. The maintainable fix needs one shared run-kind-aware policy across the loader, cache, hooks, prompt injection, docs, and tests.
