## Summary

Keep open. Current main still reads and caches the fixed workspace bootstrap file set, including `HEARTBEAT.md`, before later run-kind or heartbeat-guidance filters; the related lightweight-heartbeat fix and broader lazy-context request do not replace this focused pre-load optimization.

## What This Changes

Request to skip `HEARTBEAT.md` loading for non-heartbeat runs by threading heartbeat/run-kind intent into bootstrap file resolution.

## Best Possible Solution

Make the bootstrap loader and session snapshot cache run-kind aware so default non-heartbeat runs skip `HEARTBEAT.md` before disk reads and snapshots, while heartbeat runs keep it and docs/tests define the new contract.

## Reproduction Assessment

Yes. Source inspection on current main shows default/full runs load the raw bootstrap set through `getOrLoadBootstrapFiles` or `loadWorkspaceBootstrapFiles` before later filters, and that raw set still includes `HEARTBEAT.md`.

## Solution Assessment

Yes, if maintainers accept the contract change. The narrow maintainable direction is pre-load/cache filtering keyed by run kind or heartbeat intent, not another post-load filter; the broader configurable lazy-context work can stay with #61347.
