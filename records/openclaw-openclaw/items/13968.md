## Summary

Current main and the latest release still intentionally expose HEARTBEAT.md to ordinary default-agent runs when heartbeat guidance is enabled. The requested optimization remains useful but changes a documented prompt contract and must address loading, caching, hooks, and sibling run modes rather than adding only a late filter.

## What This Changes

Requests loading HEARTBEAT.md only for heartbeat runs instead of injecting it into ordinary message context.

## Best Possible Solution

First approve whether ordinary default-agent turns should retain heartbeat guidance; if not, select bootstrap files by run kind before disk reads and cache insertion, while preserving heartbeat, lightweight cron, subagent, hook, and cache-refresh invariants with focused tests and updated heartbeat docs.

## Reproduction Assessment

Yes at source level: current main's fixed loader, per-session cache, later filtering, focused tests, and documentation establish that eligible ordinary runs still load and receive HEARTBEAT.md. The reported 380-token amount is workspace-specific and was not independently benchmarked.

## Solution Assessment

Not yet. Passing a heartbeat flag is plausible, but the best maintainable solution must also define the intended prompt contract and consistently cover cache identity, bootstrap hooks, refresh behavior, and sibling run modes.
