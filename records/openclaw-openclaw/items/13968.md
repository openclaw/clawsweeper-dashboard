## Summary

Keep open. Current main still reads and snapshots the fixed workspace bootstrap file set, including HEARTBEAT.md, before run-kind filtering, and the docs still define normal-run injection unless heartbeat guidance is disabled; the related closed issue fixed a narrower heartbeat lightweight-context forwarding bug, while the broader selective-context issue does not supersede this narrow request.

## What This Changes

Request on-demand HEARTBEAT.md loading so ordinary non-heartbeat runs do not read, cache, or inject HEARTBEAT.md.

## Best Possible Solution

Define a maintainer-approved run-kind-aware bootstrap load/cache contract that omits HEARTBEAT.md before disk reads and session snapshots for ordinary non-heartbeat runs, preserves real heartbeat behavior, and updates docs plus focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main still reads and caches HEARTBEAT.md before any run-kind filter can omit it.

## Solution Assessment

Unclear until maintainer product/API confirmation. Passing runKind or heartbeat state into the loader/cache is the narrow technical shape, but it changes the current normal-run prompt-context contract and a post-load filter would not satisfy the requested savings.
