## Summary

Keep open. Current main still reads and caches `HEARTBEAT.md` before any run-kind filter can omit it, and the docs still define normal default-agent runs as including heartbeat guidance when enabled; this remains a feature/performance semantics decision rather than a stale or implemented cleanup item.

## What This Changes

Request run-kind-aware bootstrap loading so `HEARTBEAT.md` is skipped for ordinary non-heartbeat runs and loaded for actual heartbeat runs.

## Best Possible Solution

Define a run-kind-aware bootstrap load/cache contract that skips `HEARTBEAT.md` before disk reads, session snapshots, hooks, and prompt injection for ordinary non-heartbeat runs, while preserving actual heartbeat behavior and updating docs/tests.

## Reproduction Assessment

Not applicable as a feature/performance request. Source inspection gives high-confidence evidence that current main still reads and caches `HEARTBEAT.md` before any run-kind filter can omit it.

## Solution Assessment

Unclear until maintainers settle the prompt/bootstrap contract. Passing run-kind or heartbeat state into the loader/cache is the narrow technical shape, but it changes documented behavior and should be reconciled with the broader selective-context work.
