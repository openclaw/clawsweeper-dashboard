Keep open. Current main and the latest release still load and cache HEARTBEAT.md before run-kind filtering, while the closed lightweight-heartbeat fix and the broader selective/lazy context issue do not supersede this narrower loader/cache contract.

## What This Changes

Request on-demand loading so ordinary non-heartbeat runs skip HEARTBEAT.md before workspace bootstrap reads, cache snapshots, hooks, and prompt injection.

## Best Possible Solution

Define a documented run-kind-aware bootstrap load/cache contract that skips HEARTBEAT.md before reads, snapshots, hooks, and prompt injection for ordinary non-heartbeat runs while preserving real heartbeat behavior.

## Reproduction Assessment

Not applicable as a feature/performance request. Source inspection on current main shows HEARTBEAT.md is still read and cached before any run-kind filter can omit it.

## Solution Assessment

Unclear until maintainer product/API confirmation. Passing run-kind or heartbeat state into the loader/cache is the narrow technical shape, but the documented prompt contract needs an explicit decision.
