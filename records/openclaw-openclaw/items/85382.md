- `impact:session-state`: The reported failure happens during post-compaction session transcript indexing and can leave memory search out of sync with session state.

## Summary

Keep open: current main still requires bearer auth for OpenAI-compatible memory remote endpoints when `memorySearch.remote.apiKey` is absent. Source inspection also shows `remote.baseUrl` itself is still preferred, so the unresolved problem is the unsupported no-auth/header-only contract and provider-key inheritance behavior, not an already-fixed routing issue.

## What This Changes

Allow post-compaction memory embedding sync to use an OpenAI-compatible `memorySearch.remote.baseUrl` without requiring or inheriting an API key.

## Best Possible Solution

Define an explicit no-auth or header-only contract for OpenAI-compatible memory endpoints, then implement it in the shared remote embedding client with post-compaction session-sync coverage and docs/doctor guidance.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a focused path for the credential failure: omit `memorySearch.remote.apiKey`, set provider `openai`, and the shared client still requires provider auth before post-compaction sync can embed.

## Solution Assessment

Unclear as stated. Allowing no-auth whenever `remote.baseUrl` is set is probably too broad; an explicit no-auth/header-only mode or documented custom-provider marker is safer because current docs say remote embeddings require an API key.
