## Label Justifications

- `impact:session-state`: The failure occurs during post-compaction session memory sync and can leave memory search out of date.
- `impact:auth-provider`: The issue centers on provider credential fallback and remote embedding auth behavior.

## Summary

Keep open: current main still does not support an unauthenticated OpenAI-compatible remote memory embedding endpoint, but the report also conflicts with the current documented API-key contract and needs a maintainer decision before changing auth semantics.

## What This Changes

The issue requests support for post-compaction memory embedding sync against a custom OpenAI-compatible `memorySearch.remote.baseUrl` when no `remote.apiKey` is configured.

## Best Possible Solution

Define an explicit supported no-auth or header-only contract for OpenAI-compatible memory endpoints, then implement it in the shared remote embedding client with post-compaction session-sync regression coverage and docs.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a focused reproduction path for the credential failure: configure `provider: "openai"` with `memorySearch.remote.baseUrl` but no `remote.apiKey`, and the shared client still requires provider auth before returning a client.

## Solution Assessment

Unclear as stated. The maintainable solution is likely an explicit no-auth or header-only remote embedding contract, because allowing no auth whenever `remote.baseUrl` is set could break existing provider-key inheritance.
