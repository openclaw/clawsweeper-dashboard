The issue reports that Gateway chat projection should ignore abort markers older than a fresh same-key chat run while preserving suppression for the active aborted run.

Review and land or replace the focused linked PR so Gateway uses ordered abort-marker freshness for chat projection while preserving active-run abort suppression.
