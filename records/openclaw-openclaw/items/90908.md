## Summary

This PR should remain open because its central regression fixture already follows the fallback path on current main, while removing the existing non-timeout AbortError guard broadens retries to unclassified AbortErrors without establishing that those errors are provider-owned.

## What This Changes

The PR changes model fallback to stop only when the external abort signal is already aborted, forwards compaction abort signals, and adds three focused fallback tests.

## Best Possible Solution

Keep compaction abort-signal forwarding, but preserve the existing non-timeout AbortError boundary until a real provider error shape that fails on current main is captured; then add a base-failing regression and narrowly classify only that provider-owned shape as retryable.

## Reproduction Assessment

## Solution Assessment

No as currently demonstrated. Signal-based cancellation is reasonable, but removing the non-timeout AbortError guard is broader than the evidence supports and the new provider test does not fail on the base branch.
