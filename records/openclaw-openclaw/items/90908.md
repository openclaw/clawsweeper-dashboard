## Best Possible Solution

Keep the provider-side AbortError fallback intent, but rethrow externally aborted AbortErrors before failover normalization or ignore `normalizedFailover` when `params.abortSignal?.aborted`, add a `controller.abort()` regression, and provide fresh current-head runtime proof.
