## Summary

## What This Changes

The PR narrows AbortError rethrows in model fallback, adds provider-side abort failure classification in agent-runner replies, forwards abort signals for compaction and cron fallback calls, and expands regression tests.

## Best Possible Solution

Handle provider-side aborts in the exact shapes emitted by `runWithModelFallback` for both one-candidate and multi-candidate exhaustion, preserve real cancellation silence, and add redacted provider/channel proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current abort text is normalized to a timeout `FailoverError`, single-candidate fallback rethrows that last error, and non-direct generic failures resolve to `SILENT_REPLY_TOKEN`. I did not run tests because this review is read-only.

## Solution Assessment

No, this is a plausible fix but not yet the best fix because it assumes the single-model path reaches the runner as raw `AbortError`. The safer fix should classify the actual `FailoverError`/cause shape or preserve an explicit provider-abort marker from fallback.
