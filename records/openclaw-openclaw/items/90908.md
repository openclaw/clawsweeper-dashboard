## Summary

Keep open: the latest head fixes the previous guard-ordering problem, but it now broadens no-signal, non-failover `AbortError` handling into fallback retries and still lacks fresh current-head real behavior proof.

## What This Changes

This PR changes agent model fallback AbortError handling so externally aborted signals are rethrown before failover normalization while provider-side timeout-shaped AbortErrors can advance to fallback, with focused model-fallback regression tests.

## Best Possible Solution

Keep the pre-normalization external-abort guard, restore the post-normalization rethrow for unclassified AbortErrors, add a no-signal generic AbortError regression, and require fresh current-head real behavior proof.

## Reproduction Assessment

## Solution Assessment

No. The intended fix location is right, but the patch should preserve the existing non-failover AbortError short-circuit while allowing timeout-shaped provider AbortErrors to normalize and fallback.
