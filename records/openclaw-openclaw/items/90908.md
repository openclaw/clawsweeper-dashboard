## Summary

Keep open: the central provider-side abort fix is useful, but the patch narrows AbortError rethrowing in the shared fallback helper without auditing every cancellable caller. `compactEmbeddedAgentSessionDirect` has an abortable compaction path but does not pass that signal into `runWithModelFallback`, so the new guard can turn caller-aborted compaction into fallback work instead of stopping cleanly.

## What This Changes

The PR changes model fallback and agent-runner failure classification so provider-side AbortErrors can fall back or produce visible non-direct replies instead of being treated as cancellations.

## Best Possible Solution

Thread cancellation signals through every production `runWithModelFallback` caller affected by the new AbortError distinction, add focused coverage for compaction fallback aborts, then provide redacted real provider/channel proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes by source inspection: current main can rethrow a provider-side AbortError and route the resulting generic non-direct failure to `SILENT_REPLY_TOKEN`. I did not run a live provider repro in this read-only review.

## Solution Assessment

No, not yet. The fallback and runner boundaries are the right place to fix the bug, but the shared helper change must also preserve cancellation for existing cancellable fallback callers such as compaction.
