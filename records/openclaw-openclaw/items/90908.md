## Summary

Keep open: current main does not already provide this PR's full signal-aware fallback behavior, and none of the overlapping abort/fallback PRs is a merged or viable superseder. The patch looks correct, but because it changes shared retry-versus-cancellation behavior for channel-visible replies, maintainers should coordinate it with the broader abort work before merge.

## What This Changes

The PR changes shared model fallback abort handling so aborted caller signals stop fallback while provider-side AbortErrors without a caller abort can rotate to fallbacks, and it threads abort signals into compaction and isolated-cron fallback calls with regression tests.

## Best Possible Solution

Land one coordinated signal-aware fallback contract that keeps caller cancellations terminal and treats provider transport aborts as retryable, using this narrow PR or a maintainer-selected superseding branch.

## Reproduction Assessment

## Solution Assessment
