## Summary

Keep open: current head is not merge-ready because it removes the no-signal AbortError stop path while a production agent-command fallback caller still does not pass its caller abort signal, so cancellations can rotate into fallback candidates instead of stopping.

## What This Changes

The PR narrows model fallback abort handling so caller-aborted signals stop fallback while provider-side AbortErrors can rotate to fallback candidates, and it forwards abort signals through compaction and isolated cron fallback calls.

## Best Possible Solution

Pass the caller abort signal through every production runWithModelFallback caller that can be externally cancelled, especially src/agents/agent-command.ts, or preserve a no-signal AbortError stop path until that audit is complete; then coordinate the final abort/fallback contract with the overlapping PRs.
