## Summary

Keep this PR open: it improves the fallback/cancellation boundary, but the single-candidate provider-abort path still appears to be normalized into a `FailoverError` and can remain a generic non-direct failure, preserving the original silent reply behavior.

## What This Changes

The branch changes model fallback abort handling, non-direct agent-run failure visibility, and compaction/cron abort-signal forwarding so provider-side AbortErrors can fall back or surface visibly while real cancellations remain silent.

## Best Possible Solution
