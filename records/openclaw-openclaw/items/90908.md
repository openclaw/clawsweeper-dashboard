## Summary

Keep this PR open: the provider-side AbortError fallback fix is useful, but the cron cancellation path still throws plain timeout text that model fallback can treat as retryable, so the patch is not merge-ready.

## What This Changes

The PR changes model fallback abort handling, forwards abort signals through compaction and cron fallback calls, and adds focused regression tests.

## Best Possible Solution

Keep the narrow provider-side AbortError retry fix, but make cron cancellation produce or map to a terminal fallback-abort shape and prove an aborted cron run cannot reach a second candidate.

## Reproduction Assessment

Yes, source-reproducible. On this PR head, the cron callback still throws a plain Error from params.abortReason(), while the fallback terminal check only recognizes specific Error names or AbortError with an aborted signal.

## Solution Assessment

No, not yet. The shared fallback boundary is the right place for the provider-side AbortError repair, but the cron cancellation part must use a terminal abort shape or terminal detection before this is the best fix.
