## Summary

Keep this PR open: it has useful hardening intent, but the diff currently expands the agent config trust boundary too broadly and validates restart against a possible remote gateway while still restarting the local process.

## What This Changes

The PR allows agent gateway config self-edits for defaults-level media model routing, adds config snapshot validation before gateway restart, and adds regression tests for those paths.

## Best Possible Solution

Keep the trust boundary narrow: preserve media provider routing as operator-owned unless maintainers explicitly approve a narrower agent-writable capability, and make restart preflight validate the local config that will actually be restarted.

## Reproduction Assessment

Yes for the review findings by source inspection: the patch adds global media routing paths to the agent allowlist and uses `gatewayOpts` for `config.get` before a local restart. I did not run tests because this is a read-only review.

## Solution Assessment

No; the restart hardening idea is useful, but the implementation should not broaden global provider routing without an explicit maintainer decision and should validate the same local config that will restart.
