## Summary

Keep open for maintainer review: the patch is targeted and the updated proof now covers the changed runtime paths, but merging intentionally changes agent-triggered restart behavior for invalid or unreadable local configs.

## What This Changes

The branch adds a local `config.get` preflight before agent gateway-tool restarts, refuses restart when the config snapshot is invalid or unreadable, and adds regression tests for restart preflight and protected media model routing config patches.

## Best Possible Solution

Land the targeted preflight if maintainers accept fail-closed restart semantics; otherwise preserve current restart behavior and move strict restart validation behind an explicit product decision.

## Reproduction Assessment

Yes, source-reproducible: current main schedules the gateway-tool restart without a local `config.get` validity check, and the PR tests model invalid and unreadable snapshots. I did not run runtime tests because this review is read-only.

## Solution Assessment

Yes, with an upgrade caveat: using local `config.get` before writing the restart sentinel is the narrow owner-boundary fix, since `config.get` already returns redacted invalid snapshots with issues. The remaining question is whether maintainers accept the fail-closed behavior for existing invalid configs.
