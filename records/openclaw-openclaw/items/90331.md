- `P2`: This is a normal-priority security/robustness improvement in the agent gateway tool with limited but real upgrade impact.
- `merge-risk: 🚨 compatibility`: The PR changes existing restart behavior for users whose local config is invalid or unreadable by blocking a restart that current main would schedule.
- `merge-risk: 🚨 availability`: If `config.get` cannot load the snapshot, the agent gateway restart action refuses before scheduling a restart, which can affect recovery availability.

## Summary

Keep open: the patch looks narrowly correct and proof-backed, but it intentionally changes restart behavior by refusing agent-triggered restarts when the local config snapshot is invalid or unreadable, so a maintainer should accept that compatibility and availability tradeoff before merge.

## What This Changes

The PR adds a local `config.get` preflight before agent gateway-tool restarts, keeps defaults-level media model routing protected from agent config self-edits, and adds regression coverage.

## Best Possible Solution

Land the narrow guard only after maintainers explicitly accept operator-owned config repair as the desired outcome for invalid or unreadable config before agent-triggered restarts.

## Reproduction Assessment

Yes, from source inspection: current main schedules the restart without a config snapshot preflight, and the PR adds source-covered refusal paths for invalid or unreadable snapshots. I did not run a live before/after scenario in this read-only review, but the PR body includes redacted after-fix proof.

## Solution Assessment

Yes, subject to maintainer sign-off: keeping media routing out of the agent self-edit allowlist is the right boundary, and the local preflight is a narrow mitigation. The remaining question is whether fail-closed restart semantics are the desired upgrade behavior.
