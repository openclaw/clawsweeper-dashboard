Keep open. Current main still has the requested config gap: `compaction` and `contextPruning` are accepted under `agents.defaults`, but not under strict `agents.list[]`, and the embedded Pi runtime still reads defaults-only settings.

Add first-class per-agent config support for both settings across schema, exported types, effective agent resolution, embedded Pi compaction/pruning runtime calls, docs, and regression tests.
