## Summary

Keep open. Current main still has the requested config gap: `compaction` and `contextPruning` are accepted under `agents.defaults`, but strict `agents.list[]` rejects them and the embedded Pi runtime still reads defaults-only settings. The closed #52751 is useful prior art, but it was not merged and schema-only support would not make the runtime behavior work.

## What This Changes

Allow `agents.list[]` entries to override `compaction` and `contextPruning` settings with merge semantics over `agents.defaults`.

## Best Possible Solution

Add first-class per-agent support across schema, exported types, effective agent resolution, embedded Pi compaction/pruning runtime calls, docs/help, changelog, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `AgentEntrySchema` is strict and lacks both requested keys, so the reported `agents.list[]` config is rejected before runtime; downstream runtime code also has no per-agent lookup for these settings.

## Solution Assessment

Yes, if implemented end to end. The narrow maintainable fix is not just loosening validation, but resolving/deep-merging per-agent settings and threading them through the embedded Pi compaction and pruning runtime paths.
