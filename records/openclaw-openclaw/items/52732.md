## Summary

Keep open. Current main still has the reported config gap: `compaction` and `contextPruning` exist under `agents.defaults`, but strict `agents.list[]` entries do not accept them, and the embedded Pi runtime still reads defaults-only settings; the related PR #52751 is closed and unmerged.

## What This Changes

Allow `agents.list[]` entries to override `compaction` and `contextPruning`, merged with `agents.defaults`, for multi-agent context-management tuning.

## Best Possible Solution

Add first-class per-agent support end to end: schema, exported types, effective agent resolution, embedded Pi compaction/pruning runtime calls, generated config metadata, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path because `AgentEntrySchema` is strict and lacks both requested keys, and runtime inspection shows defaults-only lookup even if the schema were loosened.

## Solution Assessment

Yes. The maintainable fix is an end-to-end config/resolver/runtime update; accepting the keys without threading effective per-agent settings into Pi compaction and pruning would leave the user-visible behavior incomplete.
