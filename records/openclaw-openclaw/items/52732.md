## Summary

Keep open. Current main still has the reported config gap: `agents.defaults` accepts `compaction` and `contextPruning`, but strict `agents.list[]` entries do not, and the embedded Pi runtime still reads these settings from defaults-only paths.

## What This Changes

Allow `agents.list[]` entries to override `compaction` and `contextPruning` settings per agent, merging with `agents.defaults`.

## Best Possible Solution

Add first-class per-agent support across schema, exported types, effective agent resolution, embedded Pi compaction/pruning runtime calls, docs, changelog, and regression tests while preserving current defaults behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path because strict `AgentEntrySchema` lacks both keys, so the reported config shape is rejected before runtime; downstream runtime code would still ignore per-agent values if only the schema were loosened.

## Solution Assessment

Yes. The narrow maintainable solution is an end-to-end config/resolver/runtime update; the closed related PR is useful prior art, but schema/type acceptance alone would not make the settings effective.
