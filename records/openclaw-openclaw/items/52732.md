## Summary

Keep open. Current main still has the reported config gap: `agents.defaults` owns `compaction` and `contextPruning`, strict `agents.list[]` entries do not accept either key, and runtime consumers still read defaults-only settings. The open compaction PR at https://github.com/openclaw/openclaw/pull/69988 covers only part of this issue and previously had review blockers; the combined PR at https://github.com/openclaw/openclaw/pull/52751 was closed with owner-backed design guidance.

## What This Changes

Allow optional per-agent `compaction` and `contextPruning` overrides in `agents.list[]`, including model-specific compaction routing, merged over `agents.defaults`.

## Best Possible Solution

Pick one owner-backed config contract, then land a canonical implementation that covers schema/types, effective per-agent resolution, Pi compaction/pruning runtime consumers, generated metadata, docs, changelog, and focused tests.
