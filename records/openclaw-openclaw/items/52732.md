## Summary

Keep open. Current main still has a real config/runtime gap: `compaction` and `contextPruning` are accepted under `agents.defaults`, but remain absent from the strict `agents.list[]` schema/type/resolver and the embedded Pi runtime paths still read defaults-only settings.

## What This Changes

The issue requests optional per-agent `compaction` and `contextPruning` overrides in `agents.list[]`, merged over `agents.defaults` for mixed-model multi-agent setups.

## Best Possible Solution

Add first-class per-agent config support for both settings: extend the strict schema and types, merge each agent override over `agents.defaults`, pass the effective settings into embedded Pi compaction/pruning callers, and document/test precedence so omitted agent fields continue to inherit defaults.

## Reproduction Assessment

Yes: current main has a source-level reproduction path because `AgentEntrySchema` is strict and lacks `compaction` and `contextPruning`, so the reported config shape is rejected before runtime. Runtime inspection also shows no downstream per-agent lookup that would make those settings effective if the schema were loosened alone.

## Solution Assessment

Yes: the best fix is an end-to-end typed config/resolver/runtime change. The closed #52751 attempt is useful prior art, but current main needs more than accepting the keys because several Pi runtime readers still consume `agents.defaults` directly.
