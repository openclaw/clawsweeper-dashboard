## Summary

This PR should stay open. The Feishu message-management feature is useful and the latest comments address the earlier direct-chat and per-account gate findings, but the patch still persists bot-sent Feishu message IDs to disk from normal sends even when the sensitive `tools.messages` gate is disabled.

## What This Changes

The PR adds Feishu message list/delete/recall/read-receipt tooling, generic Feishu `delete`/`unsend` message actions, opt-in config/schema/docs, tests, and latest bot-sent message tracking.

## Best Possible Solution

Keep the Feishu message-management feature, but make the last-bot-message fallback match the opt-in security boundary by keeping it memory-only or only persisting it behind an explicit `tools.messages` opt-in with docs and tests.
