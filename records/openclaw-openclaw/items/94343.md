## Summary

Keep open for normal merge review: this PR is a focused root-cause fix for the Workboard dispatch scope-classification bug, and I found no blocking correctness finding. The main maintainer-visible caveat is that it intentionally changes a gateway authorization decision surface by honoring plugin-declared scopes from pinned route/channel registries.

## What This Changes

The PR changes gateway method scope resolution to find plugin method descriptors across active, HTTP route, and channel registry surfaces, with regression tests and local repro scripts for Workboard-style plugin methods.

## Best Possible Solution

Land the shared resolver fix after accepting plugin gateway descriptors as authoritative across all live registry surfaces; keep the narrower Workboard CLI-only PR as complementary or superseded follow-up depending on maintainer preference.

## Reproduction Assessment

Yes. Source inspection shows current main registers Workboard dispatch as `operator.write`, but the shared resolver only checks active plugin descriptors, so a descriptor present only on a pinned surface falls through to the `operator.admin` fallback.

## Solution Assessment

Yes. Fixing the shared resolver is the best layer because it repairs the plugin gateway method scope contract for all active surfaces, whereas changing only the Workboard CLI would mask this one caller and leave the same root condition for other plugins.
