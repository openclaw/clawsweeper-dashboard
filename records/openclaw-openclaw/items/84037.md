The issue requests lower Codex app-server steady-state CPU, less hot-path OpenClaw hook/helper process work, better CPU attribution diagnostics, and self-hosted Codex runtime guidance.

## Best Possible Solution

Define a measured Codex runtime performance target, land or repair the narrow hook-relay mitigations that are already in flight, add child/app-server/helper CPU attribution, then document the supported self-hosted Codex profile.
