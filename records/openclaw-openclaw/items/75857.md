## What This Changes

The PR threads a callback-query suppression flag through Telegram message context and dispatch so silent callback button turns do not emit a visible fallback while normal callback replies still send.

## Best Possible Solution

Keep the Telegram-owned callback suppression, but combine it with the existing source-delivery suppression state under distinct names and require real Telegram proof before merge.

## Solution Assessment

No. The direction is narrow and maintainable, but this branch must first rename or combine the callback flag with the existing tool-only suppression state and add real Telegram behavior proof.
