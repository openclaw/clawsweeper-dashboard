## Summary

Keep open: current main still retains a new full `.bak-*` file for every successful session-file repair, the report matches the source and tests, and the related draft fix PR is still open rather than merged.

## What This Changes

The issue requests bounding or removing repeated session-file repair `.bak-*` snapshots so a stuck session cannot grow the sessions directory without limit.

## Best Possible Solution

Land a focused `session-file-repair.ts` change that prevents unbounded backup growth, with regression coverage plus docs/changelog updates for the chosen retention contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: make a valid session JSONL that triggers repair, call `repairSessionFileIfNeeded` repeatedly while it keeps changing content, and current main will write a fresh unique `.bak-*` on each successful repair; existing tests already assert the retained backup.

## Solution Assessment

Yes, the narrow maintainable fix is local to session-file repair: delete successful transient backups or rotate per-session backups, then align tests and documentation with that contract.
