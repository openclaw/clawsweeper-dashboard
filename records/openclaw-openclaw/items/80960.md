## Summary

Keep open: current main still retains one full `.bak-*` session-file backup per successful repair, and the linked open PR at https://github.com/openclaw/openclaw/pull/80969 is the active fix path with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop retaining a new full session-file `.bak-*` backup on every successful `repairSessionFileIfNeeded` invocation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80969 or an equivalent focused change that removes success-path transient backups while preserving failure-path recovery evidence and aligning tests/docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a repairable session JSONL that keeps requiring successful repair will write a fresh retained `.bak-*` on each invocation, and current tests assert the retained backup contract.

## Solution Assessment

Yes. The narrow maintainable fix is local to `repairSessionFileIfNeeded`: delete the transient backup only after successful atomic replacement, report it only when cleanup fails, and update the regression tests/docs for that contract.
