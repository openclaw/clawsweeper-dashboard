## Summary

Keep open: current main still retains a fresh `.bak-*` file on every successful session-file repair, and the open closing PR should be reviewed or landed before this issue is closed.

## What This Changes

The issue asks OpenClaw to stop unbounded retained `.bak-*` session repair snapshots from accumulating across repeated repair invocations.

## Best Possible Solution

Review and land the focused open PR, then close this issue from the merged fix; split the ended-session reachability observation only if it still reproduces.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a repairable session JSONL causes `repairSessionFileIfNeeded` to write and retain a unique `.bak-*`, and repeated run-attempt or compaction calls repeat that write.

## Solution Assessment

Yes. The narrow maintainable fix is to delete the transient backup only after `replaceFileAtomic` succeeds, or otherwise bound per-session retention; the open PR takes the smaller success-path cleanup approach.
