## Summary

Keep open: current main still retains a unique backup on every successful repair, and https://github.com/openclaw/openclaw/pull/80969 is open with closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to stop session-file repair from retaining an unbounded new `.bak-*` file after every successful repair invocation.

## Best Possible Solution

Land or replace the focused session-file-repair fix that deletes successful transient backups, keeps cleanup-failure reporting, updates regression tests/docs/changelog, and tracks ended-session reachability separately if needed.

## Reproduction Assessment

Yes. Source inspection on current main shows `repairSessionFileIfNeeded` writes and returns a fresh `.bak-*` after successful repair with no cleanup, and existing tests assert the backup remains; I did not run a writing repro because this review is read-only.

## Solution Assessment

Yes. Deleting the transient backup only after successful atomic replacement is the narrow maintainable fix for the disk-growth bug; TTLs, rotation, or stale-session reachability work are broader than this immediate issue.
