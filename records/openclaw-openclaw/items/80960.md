## Summary

Keep open: current main still retains a new full `.bak-*` file after each successful session-file repair, and an open linked PR already owns the focused fix path.

## What This Changes

Prevent `repairSessionFileIfNeeded` from accumulating unbounded retained session backup snapshots during repeated repair attempts.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80969 with the local backup cleanup, regression coverage, and docs/changelog alignment.

## Reproduction Assessment

Yes. Source inspection of current main shows each successful repair writes and returns a unique retained backup, and repeated malformed-tail repair calls would reproduce linear `.bak-*` growth.

## Solution Assessment

Yes. The narrow maintainable fix is local to `repairSessionFileIfNeeded`: delete successful transient backups or retain only on cleanup failure, then update tests and docs for that contract.
