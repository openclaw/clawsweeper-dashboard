## Summary

Keep open: current main and the latest shipped release still retain a new full `.bak-*` file for each successful session-file repair, and the matching fix PR is open/draft rather than merged.

## What This Changes

The issue asks OpenClaw to prevent unbounded retained `.bak-*` session-repair snapshots from accumulating when repair runs repeatedly on a stuck session.

## Best Possible Solution

Choose a bounded repair-backup retention contract, update code/tests/docs/changelog around it, and land or replace the focused fix PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: make a valid session JSONL that triggers repair, invoke `repairSessionFileIfNeeded` repeatedly while each invocation still changes content, and current main writes a fresh timestamped backup each time; existing tests assert retained backups.

## Solution Assessment

Yes, the narrow solution belongs in `session-file-repair.ts`. The maintainer choice is whether successful repairs should delete transient backups or keep a small bounded rotation, then align tests and docs with that contract.
