## Summary

Keep open: current main still retains a full unique `.bak-*` file after every successful session-file repair, and the paired fix path is an open PR rather than merged code.

## What This Changes

The issue asks OpenClaw to stop `repairSessionFileIfNeeded` from retaining unbounded `.bak-*` session-file backups across repeated repairs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80969, or an equivalent local fix, so successful repair backups are deleted or bounded and tests/docs/changelog match the chosen retention contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a valid session JSONL that triggers repair, invoke `repairSessionFileIfNeeded` repeatedly while repair keeps changing the file, and current main writes a fresh retained `.bak-*` on each successful repair.

## Solution Assessment

Yes. The narrow maintainable fix is local to `session-file-repair.ts`: delete successful transient backups or rotate them, with regression coverage and docs aligned; the issue should stay open until the linked fix merges.
