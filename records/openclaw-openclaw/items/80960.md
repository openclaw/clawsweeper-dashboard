The issue reports unbounded retained `.bak-*` snapshots from repeated successful session-file repair invocations.

## Best Possible Solution

## Solution Assessment

Yes. The narrow fix is local to successful session-file repair cleanup: delete the transient backup only after atomic replacement succeeds, retain/report it when cleanup fails, and keep tests/docs aligned.
