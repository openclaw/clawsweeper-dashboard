## Summary

Keep open. Current main still has the documented mixed heartbeat scoping behavior, but there is no matching doctor warning for agents skipped because a peer has an explicit heartbeat block.

## What This Changes

The issue requests an `openclaw doctor` warning when `agents.defaults.heartbeat` is configured but some listed agents are shadowed by another agent's explicit `heartbeat` block.

## Best Possible Solution

Add a non-mutating doctor warning for default-heartbeat agents skipped by mixed per-agent heartbeat scoping, while preserving the documented scheduler behavior.

## Reproduction Assessment

Yes. Source inspection and checked-in tests show the mixed-scope behavior on current main; the reproducible gap is that the doctor warning path does not report it.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable fix because current docs and tests encode the scoping rule, while changing runtime inheritance would be a broader behavior change.
