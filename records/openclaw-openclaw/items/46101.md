## Summary

Keep open. Current main still implements the mixed heartbeat scoping behavior and documents it, but the doctor preview warning collector does not flag listed agents that will be disabled by a peer's explicit heartbeat block.

## What This Changes

Request an `openclaw doctor` warning when `agents.defaults.heartbeat` is configured but some `agents.list[]` entries are shadowed by peer per-agent `heartbeat` blocks.

## Best Possible Solution

Add a non-mutating doctor config-health warning for mixed default/per-agent heartbeat scoping, while leaving runtime inheritance semantics to the broader heartbeat policy thread.

## Reproduction Assessment

Yes. Source inspection plus checked-in tests show a config with `agents.defaults.heartbeat`, `main` listed without `heartbeat`, and `ops` listed with `heartbeat` disables `main` while leaving `ops` scheduled.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable fix for this issue because current docs and tests encode the mixed scoping rule, while changing that rule belongs in #41879.
