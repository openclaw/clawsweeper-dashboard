## Summary

Keep open. Current main still documents and implements mixed heartbeat scoping, but `openclaw doctor` preview warnings have no heartbeat check for agents skipped by a peer's explicit heartbeat block.

## What This Changes

Request an `openclaw doctor` warning when `agents.defaults.heartbeat` is configured but explicit per-agent heartbeat blocks cause listed agents without heartbeat blocks to be skipped.

## Best Possible Solution

Add a non-mutating doctor config-health warning for mixed default/per-agent heartbeat scoping, while leaving runtime heartbeat scheduling semantics unchanged.

## Reproduction Assessment

Yes. Source inspection and checked-in tests show that with `agents.defaults.heartbeat`, `main` listed without `heartbeat`, and `ops` listed with `heartbeat`, `main` is disabled while `ops` runs; the missing piece is that doctor does not warn about it.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable solution because current docs and tests encode the mixed scoping rule, while changing runtime inheritance would be a broader behavior change.
