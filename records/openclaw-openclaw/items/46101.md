## Summary

Keep open. Current main still has the mixed heartbeat scoping behavior, tests encode that defaulted agents are disabled when any listed peer has an explicit heartbeat block, and the doctor preview warning collector still does not diagnose this configuration.

## What This Changes

The issue requests an `openclaw doctor` warning when `agents.defaults.heartbeat` is configured but per-agent heartbeat blocks cause listed agents without explicit heartbeat blocks to be excluded.

## Best Possible Solution

Add a non-mutating doctor config-health warning for mixed default/per-agent heartbeat scoping, while leaving runtime inheritance semantics to the broader heartbeat policy issue.

## Reproduction Assessment

Yes. A config with `agents.defaults.heartbeat`, `main` listed without `heartbeat`, and `ops` listed with `heartbeat` is reproduced by current source and checked-in tests showing `main` disabled while `ops` runs.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable fix for this report because the mixed scoping rule is implemented and documented today, while changing the rule belongs to #41879.
