## Summary

Keep open. Current main still encodes the mixed heartbeat scoping behavior and documents it, but the doctor preview warning collector does not report listed agents that are disabled by a peer's explicit heartbeat block.

## What This Changes

Add an `openclaw doctor` warning when `agents.defaults.heartbeat` is configured but per-agent heartbeat blocks cause listed agents without their own block to skip heartbeats.

## Best Possible Solution

Add a non-mutating doctor preview/config-health warning for mixed default/per-agent heartbeat scoping, with regression coverage, while leaving runtime inheritance semantics to #41879.

## Reproduction Assessment

Yes. Source inspection and checked-in tests show that `agents.defaults.heartbeat` plus `main` without `heartbeat` and `ops` with `heartbeat` disables `main` while keeping `ops` scheduled.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable path for this issue because current docs and tests encode the mixed scoping rule; changing that rule belongs in #41879.
