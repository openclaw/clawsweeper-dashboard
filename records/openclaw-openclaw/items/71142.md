## Summary

Keep open. Current main partially addressed the old 5 MB parser limit through the shared media limit resolver, but the Control UI upload policy is still unresolved because large images hit the fixed image cap, the bootstrap contract exposes no effective upload limit, and the browser still reads files before size prevalidation.

## What This Changes

Request to make Control UI chat uploads configurable so larger images and files can be accepted consistently with OpenClaw media limits.

## Best Possible Solution

Define one effective Control UI upload policy, then align server image acceptance, bootstrap/client prevalidation, docs, and tests with that policy and the WebSocket/base64 ceiling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure agents.defaults.mediaMaxMb above 6 MiB and submit a Control UI image larger than MAX_IMAGE_BYTES; chat.send uses the shared parser limit, then parseMessageWithAttachments rejects the fixed image cap.

## Solution Assessment

Unclear. A focused implementation is plausible, but maintainers first need to decide whether the public upload policy is the existing shared media setting or a Control UI-specific configuration surface.
