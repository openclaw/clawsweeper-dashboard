## Summary

Keep open. Current main has removed the old blanket 5 MB parser cap by using the shared `agents.defaults.mediaMaxMb` resolver, but the Control UI upload-limit request is still unresolved because image uploads remain capped by `MAX_IMAGE_BYTES`, the bootstrap payload does not tell the browser the effective upload policy, and the UI still reads files before size prevalidation.

## What This Changes

Request a configurable Control UI upload-size policy so larger images and other media can be accepted consistently with agent and channel media limits.

## Best Possible Solution

Define one effective Control UI upload policy, then align server-side image acceptance, bootstrap/client prevalidation, tests, and docs with that policy and the WebSocket/base64 ceiling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.mediaMaxMb` above 6 MiB and upload an image larger than `MAX_IMAGE_BYTES`; `chat.send` reaches the shared maxBytes path, then `parseMessageWithAttachments` rejects the fixed image cap.

## Solution Assessment

Unclear. A focused implementation is plausible, but maintainers first need to decide whether the public upload policy is the existing shared media setting or a Control-UI-specific configuration surface.
