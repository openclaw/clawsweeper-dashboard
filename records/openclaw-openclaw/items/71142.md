## Summary

Keep open. Current main and v2026.5.18 have removed the old blanket 5 MB parser cap, but the Control UI upload-policy request is still only partially solved because image uploads remain blocked by a fixed 6 MiB image cap and the browser still has no advertised upload limit for prevalidation.

## What This Changes

Request to make Control UI chat attachment upload limits configurable and visible to the browser instead of fixed or discovered only after upload.

## Best Possible Solution

Define one effective Control UI upload policy, then align server-side image acceptance, bootstrap/client prevalidation, tests, and docs with that policy and the WebSocket/base64 ceiling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure agents.defaults.mediaMaxMb above 6 MiB and upload an image larger than MAX_IMAGE_BYTES; chat.send uses the shared max, but parseMessageWithAttachments still rejects the fixed image cap.

## Solution Assessment

Unclear. Reusing agents.defaults.mediaMaxMb may be the narrowest path, but maintainers need to decide whether Control UI should have an independent upload-limit contract and how far related media ceilings should move.
