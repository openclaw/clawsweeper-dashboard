## Summary

Keep open. Current main and v2026.6.10 have already moved the original hardcoded 5 MB parser cap to `agents.defaults.mediaMaxMb`, but the remaining Control UI upload-limit problem is still present and needs a maintainer policy choice before implementation.

## What This Changes

The issue requests configurable Control UI upload size limits so web chat uploads can use larger media limits instead of a smaller hardcoded cap.

## Best Possible Solution

Use the existing `agents.defaults.mediaMaxMb` policy if maintainers accept that owner, expose the effective upload limit to Control UI, add browser prevalidation before file reads, and resolve the image hydration cap explicitly.

## Reproduction Assessment

Yes at source level: current main routes `chat.send` through `resolveChatAttachmentMaxBytes(cfg)`, but image attachments above `MAX_IMAGE_BYTES` are rejected and UI paste/file/drop paths call `FileReader.readAsDataURL` before size-aware validation. I did not run a live browser upload in this read-only review.

## Solution Assessment

Unclear as a product choice: reusing `agents.defaults.mediaMaxMb` is the narrowest current config path, but maintainers still need to decide whether Control UI should share that policy or own a separate upload limit.
