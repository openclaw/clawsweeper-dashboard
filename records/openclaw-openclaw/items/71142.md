## Summary

Keep open. Current main no longer has the old blanket 5 MB server parser cap, but the Control UI upload-limit request is still unresolved because images remain bounded by a fixed 6 MiB `MAX_IMAGE_BYTES`, the bootstrap contract does not expose an effective upload limit, and the browser still reads files before any size prevalidation.

## What This Changes

Request to make Control UI chat upload limits configurable and consistent with the media limits users can configure elsewhere.

## Best Possible Solution

Define one effective Control UI upload policy, align image acceptance and bootstrap/prevalidation with that policy, keep server-side parsing authoritative, and document the WebSocket/base64 ceiling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.mediaMaxMb` above 6 MiB and upload an image larger than `MAX_IMAGE_BYTES`; `chat.send` reaches the shared maxBytes path, then `parseMessageWithAttachments` rejects the fixed image cap.

## Solution Assessment

Unclear. A focused implementation is plausible, but maintainers first need to choose whether the upload policy reuses `agents.defaults.mediaMaxMb` or adds a Control-UI-specific public contract.
