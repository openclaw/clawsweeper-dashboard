## Summary

Keep open. Current main removed the old blanket 5 MB parser cap, but the Control UI upload-policy request is still unresolved because large images remain capped by the fixed image hydration limit and the browser still lacks an advertised upload limit for prevalidation.

## What This Changes

The issue requests a configurable Control UI upload limit so web chat can accept larger media, especially phone photos, consistently with agent media limits.

## Best Possible Solution

Define one effective Control UI upload policy, then align server-side image acceptance, bootstrap/client prevalidation, tests, and docs with that policy and the WebSocket/base64 ceiling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.mediaMaxMb` above 6 MiB and upload an image larger than `MAX_IMAGE_BYTES`; `chat.send` uses the shared max but `parseMessageWithAttachments` rejects the fixed image cap.

## Solution Assessment

Unclear. Reusing `agents.defaults.mediaMaxMb` appears to be the narrowest existing config path, but maintainers need to decide whether Control UI should have an independent upload-limit contract and how far to raise related media ceilings.
