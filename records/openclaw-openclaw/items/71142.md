## Summary

Keep open. Current main centralized the general chat attachment cap, but the Control UI request remains unresolved because images still hit a fixed 6 MiB guard, the browser lacks upload-limit bootstrap/prevalidation, and commenters are waiting on the shared-vs-Control-UI-specific config decision.

## What This Changes

Request to make Control UI chat uploads use a configurable size policy instead of a fixed low limit for larger images/files.

## Best Possible Solution

Define one effective Control UI upload policy, expose the resolved byte limit through bootstrap, prevalidate before file reads, keep server-side parsing authoritative, and document the WebSocket/base64 ceiling.

## Reproduction Assessment

Yes. A high-confidence source path exists: configure `agents.defaults.mediaMaxMb` above 6 MiB and upload an image larger than `MAX_IMAGE_BYTES`; `chat.send` reaches the shared cap path, then `parseMessageWithAttachments` rejects the fixed image cap.

## Solution Assessment

Unclear. A focused implementation is plausible, but maintainers first need to choose whether Control UI should reuse `agents.defaults.mediaMaxMb` or introduce a separate `gateway.controlUi` upload-limit contract.
