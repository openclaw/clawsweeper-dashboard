## Summary

Keep open: current main and v2026.6.10 no longer have the original fixed 5 MB parser cap, but the Control UI upload-limit request is still not implemented because images remain capped by the 6 MiB media-core limit and the browser has no server-provided upload limit for prevalidation.

## What This Changes

The issue requests a configurable Control UI upload-size limit for larger media uploads.

## Best Possible Solution

Use one maintainer-approved upload policy, surface the resolved limit to the browser, prevalidate before file reads/sends, and keep server-side enforcement aligned with model/media runtime caps.

## Reproduction Assessment

Yes at source level: current main still rejects image attachments above the 6 MiB media-core cap and Control UI file paths do not prevalidate size before `FileReader.readAsDataURL`. I did not run a live browser upload, so this is not a full live reproduction.

## Solution Assessment

Unclear until maintainers choose policy: reusing `agents.defaults.mediaMaxMb` avoids a new config key, while a Control UI-specific knob gives independent tuning but adds config surface.
