## Summary

Keep open. Current main has removed the old blanket 5 MB parser cap, but the Control UI upload-policy request is still unresolved because images remain capped by the fixed image hydration limit and the browser still lacks a runtime upload limit for prevalidation.

## What This Changes

Request a configurable Control UI upload-size policy so larger images and other media can be uploaded through WebChat consistently with OpenClaw media limits.

## Best Possible Solution

Define one effective Control UI upload policy, then align server-side image acceptance, bootstrap data, browser prevalidation, docs, and focused tests with that policy and the WebSocket/base64 ceiling.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.mediaMaxMb` above 6 MiB and upload an image larger than `MAX_IMAGE_BYTES` through Control UI; `chat.send` uses the shared max, but `parseMessageWithAttachments` still rejects the fixed image cap. I did not run a live browser upload because this review is read-only.

## Solution Assessment

Unclear. Reusing `agents.defaults.mediaMaxMb` looks like the narrowest existing config contract, but maintainers need to decide whether Control UI should have an independent upload-limit knob and how it should interact with WebSocket and media hydration limits.
