## Summary

Keep open: this is a focused bug-fix PR for behavior that is still missing on current main. Source inspection shows current main still suppresses all parent-owned ACP child session delivery, while the PR narrows that suppression to non-bound contexts and includes matching regression coverage; I found no blocking code-review findings.

## What This Changes

The PR changes ACP reply dispatch so parent-owned ACP child sessions can deliver replies when a turn matches their saved Discord delivery context, with regression tests for bound-thread delivery and off-thread suppression.

## Best Possible Solution

Land a narrow version of this PR after current CI and proof gates are green, preserving parent-owned ACP suppression for unmatched contexts and keeping the regression tests.

## Reproduction Assessment

Yes, at source level: current main classifies spawned ACP sessions as parent-owned background sessions and suppresses their automatic delivery without checking the bound thread context. I did not execute the live Discord/ACP path in this read-only review.

## Solution Assessment

Yes: matching the current turn against the child session's saved delivery channel/account/thread or target is a narrow fix that preserves suppression for unrelated parent-owned ACP child sessions.
