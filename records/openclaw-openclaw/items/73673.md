## Summary

Keep open. Current main and v2026.6.6 still generate and audit unconditional macOS `KeepAlive=true`; the linked fix PRs are closed unmerged, and newer related issues cover adjacent launchd loops without safely superseding this startup-failure report.

## What This Changes

The issue asks OpenClaw to avoid repeated macOS launchd gateway restarts after startup failure so users can inspect and debug the failure.

## Best Possible Solution

Adopt a canonical launchd policy that stops startup-failure respawn loops while preserving intentional restart/update handoffs, then align plist generation, audit/doctor repair, docs, and tests with live macOS proof.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 generate and audit unconditional `KeepAlive=true`, and the tests assert no `SuccessfulExit` condition; live macOS launchd was not run in this read-only review.

## Solution Assessment

No final fix is present on main. The best fix needs maintainer confirmation of launchd exit-status semantics because a naive `SuccessfulExit=false` patch conflicts with startup-failure behavior and clean-exit restart/update handoffs.
