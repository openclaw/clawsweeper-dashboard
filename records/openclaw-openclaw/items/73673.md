## Summary

Keep open. Current main still generates and audits a macOS LaunchAgent with boolean `KeepAlive`, `ThrottleInterval=1`, and no `SuccessfulExit` policy; PR #73849 is open and unmerged, so this issue should stay open until that fix or a focused replacement lands.

## What This Changes

Request that macOS LaunchAgent Gateway supervision avoid immediate restart loops after startup failures so users can inspect and debug the failure.

## Best Possible Solution

A coordinated macOS launchd failure policy should update plist generation, service audit, tests, stop/restart semantics, and any user-facing docs or changelog while preserving intentional supervised restarts.

## Reproduction Assessment

Yes. A source-level reproduction is clear: install or generate the current LaunchAgent, start it with an invalid startup option such as an invalid `--port`, and the Gateway exits nonzero while launchd is configured to keep it alive with one-second throttling.

## Solution Assessment

No current-main solution exists. The narrow maintainable path is to review and land PR #73849 or a focused replacement that handles the plist policy and service-management contracts together.
