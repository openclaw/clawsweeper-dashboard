## Summary

Keep open: current main and v2026.6.8 still generate boolean macOS LaunchAgent `KeepAlive=true`, tests still pin the absence of `SuccessfulExit`, and related PRs that attempted the lifecycle change were closed unmerged. ## What This Changes

The issue requests that macOS launchd-managed gateway startup failures stop being respawned in a tight loop so users can inspect and debug the failure.

## Best Possible Solution

Keep this issue open until maintainers choose one launchd lifecycle contract that stops startup-failure loops, preserves intentional restart/update handoffs, aligns plist generation with service audit/doctor, and has targeted tests plus live macOS proof.

## Reproduction Assessment

Yes. Current main and v2026.6.8 still generate `KeepAlive=true`, and the adjacent tests pin no `SuccessfulExit` key; I did not run a live macOS LaunchAgent reproduction in this read-only review.

## Solution Assessment

Unclear as a final fix. `SuccessfulExit=false` is a plausible policy surface, but the best fix needs maintainer-approved launchd lifecycle semantics that also preserve update/restart handoffs and service-audit behavior.
