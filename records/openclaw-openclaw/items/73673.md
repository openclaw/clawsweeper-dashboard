## Summary

Keep open. Current main still emits a macOS LaunchAgent with boolean `KeepAlive=true`, `ThrottleInterval=1`, and no `SuccessfulExit` policy, while the audit and unit tests still pin that behavior; open PR #73849 references this issue with closing syntax but is not merged.

## What This Changes

The issue asks for macOS launchd Gateway startup failures to be debuggable without immediate restart loops.

## Best Possible Solution

Review and land PR #73849 or a focused replacement that updates plist generation, service audit, tests, docs/changelog if user-facing, and preserves intentional supervised restarts.

## Reproduction Assessment

Yes. A source-level reproduction is clear: generate or install the current LaunchAgent, start the gateway with an invalid startup option such as `--port`, and launchd is configured to relaunch the nonzero exit with one-second throttling.

## Solution Assessment

No current-main solution exists. The `SuccessfulExit=false` direction in PR #73849 is plausible, but it needs maintainer review against existing launchd restart and stop/restart contracts before this issue can close.
