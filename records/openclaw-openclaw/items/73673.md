## What This Changes

The issue reports that macOS launchd repeatedly restarts the OpenClaw gateway after startup failures, making failures hard to inspect.

## Best Possible Solution

Land a fresh, focused macOS launchd lifecycle fix that stops handled startup-failure loops while preserving intentional update/restart handoffs, aligns service audit/doctor, and includes targeted tests plus live LaunchAgent proof.

## Reproduction Assessment

Yes, source-backed but not live-reproduced here: current main and v2026.6.6 still emit and audit unconditional `KeepAlive=true`, and the launchd contract says that policy keeps exited jobs alive with throttling.

## Solution Assessment

Unclear as an implementation choice: a conditional KeepAlive policy is the right surface, but maintainers still need to choose and prove semantics that stop startup-failure loops without breaking clean-exit update/restart handoffs.
