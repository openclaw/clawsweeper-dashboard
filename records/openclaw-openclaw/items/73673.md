## Summary

Keep open. Current main still emits and audits a macOS LaunchAgent with unconditional `KeepAlive=true`, one-second throttling, and no `SuccessfulExit` policy; the linked open PR #73849 uses closing syntax, so this issue should stay open until that PR or a focused replacement lands.

## What This Changes

The issue requests a debugger-friendly macOS LaunchAgent policy so Gateway startup failures do not immediately restart in a tight launchd loop.

## Best Possible Solution
