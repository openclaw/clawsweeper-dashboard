## Summary

Keep open. Current main still generates and audits the LaunchAgent restart policy reported here, and open PR #73849 references this issue with closing syntax but has not merged, so this issue should remain open until that PR or a focused replacement lands.

## What This Changes

The issue requests a macOS launchd Gateway failure policy that avoids immediate restart loops after startup crashes so users can inspect the failure.

## Best Possible Solution

A coordinated macOS launchd failure policy should prevent repeated startup-failure loops long enough for debugging while preserving intentional supervised restarts and aligning plist generation, service audit, tests, docs, and changelog.

## Reproduction Assessment

Yes, from source inspection. Generate/install the current LaunchAgent, start the Gateway with an invalid startup option such as an invalid `--port`, and launchd is configured to keep relaunching a process that exits nonzero with one-second throttling.

## Solution Assessment

No current-main solution exists. The best path is to review and land PR #73849 or a focused replacement, with explicit validation that the launchd policy handles nonzero startup failures while preserving managed restarts.
