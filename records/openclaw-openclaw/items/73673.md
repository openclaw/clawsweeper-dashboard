## Summary

Keep open. Current main still generates and audits the macOS Gateway LaunchAgent with boolean `KeepAlive=true`, `ThrottleInterval=1`, and no `SuccessfulExit` policy, so the reported startup-failure restart loop remains present; the issue is also paired with open PR #73849 using closing syntax, so it should stay open until that PR or a replacement lands.

## What This Changes

The issue requests changing macOS launchd Gateway supervision so startup crashes do not immediately restart in a tight loop and block debugging.

## Best Possible Solution

Land a focused macOS launchd early-failure strategy that updates the generated plist, service audit, restart/update handoff behavior, tests, docs, and changelog together while preserving intentional managed Gateway restarts.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: generate or install the LaunchAgent, trigger an invalid Gateway startup configuration, and launchd supervises a process whose current plist has unconditional `KeepAlive=true` with a one-second throttle.

## Solution Assessment
