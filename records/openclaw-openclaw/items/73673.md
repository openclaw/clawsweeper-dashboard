## Summary

Keep open. Current main still generates the macOS Gateway LaunchAgent with boolean `KeepAlive=true`, a one-second throttle, and no `SuccessfulExit` policy; tests, audit logic, and docs still reinforce that behavior. The issue is also paired with open PR #73849 using closing syntax, so the issue should remain open until that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw to prevent macOS launchd from rapidly relaunching a failing Gateway startup in a way that makes startup failures hard to debug.

## Best Possible Solution

Land a focused macOS launchd early-failure strategy that changes the generated plist, service audit, restart/update handoff behavior, tests, docs, and changelog together while preserving intentional managed Gateway restarts.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: install or generate the LaunchAgent, trigger an invalid Gateway startup configuration, and launchd supervises a process whose current plist has unconditional `KeepAlive=true` with a one-second throttle.

## Solution Assessment

No. Current main has not solved the issue, and the open PR’s stated `SuccessfulExit=false` direction needs coordinated handling for clean-exit launchd restarts before it is the safest complete fix.
