## Summary

Keep open: current main still emits and audits an unconditional launchd `KeepAlive=true` Gateway LaunchAgent with a one-second throttle and no `SuccessfulExit` policy, so the reported restart-loop behavior remains plausible. The issue is also paired with open PR #73849 using closing syntax, so cleanup should wait for that PR or a replacement to land.

## What This Changes

The issue asks OpenClaw to prevent launchd-managed Gateway startup failures from immediately looping and obscuring debug output.

## Best Possible Solution

The end state should be a coordinated macOS launchd early-failure strategy that changes plist generation, service audit, intentional restart handling, tests, docs, and changelog together while preserving managed Gateway restarts that are supposed to relaunch.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: generate or install the LaunchAgent, trigger an invalid Gateway startup configuration, and launchd supervises a process whose current plist has unconditional `KeepAlive=true` with a one-second throttle.

## Solution Assessment

No. Current main has not solved the issue, and the paired PR’s stated `SuccessfulExit=false` direction should be reviewed against the existing clean-exit launchd restart paths before being treated as a complete fix.
