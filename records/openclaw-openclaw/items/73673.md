## Summary

Keep open. Current main still emits a LaunchAgent with `KeepAlive=true` and `ThrottleInterval=1`, and Gateway startup validation failures can still exit nonzero into launchd supervision. The linked implementation PR is open and unmerged, but its `SuccessfulExit=false` direction needs careful review because it does not stop nonzero startup-failure relaunches and can affect current clean-exit restart handoff behavior.

## What This Changes

The issue requests a macOS launchd Gateway failure policy that avoids rapid restart loops after startup crashes so users can inspect the failure.

## Best Possible Solution

A focused macOS launchd failure policy should prevent rapid nonzero startup loops while preserving intentional supervised restarts, with plist generation, audit, tests, docs, changelog, and macOS smoke coverage aligned.

## Reproduction Assessment

Yes, from source inspection. Generate/install the current LaunchAgent, start the Gateway with an invalid startup option such as an invalid `--port`, and launchd is configured to relaunch the nonzero-exiting process with one-second throttling.

## Solution Assessment

No current-main solution exists. The open `SuccessfulExit=false` direction needs review because launchd still restarts nonzero exits and current intentional launchd restarts rely on clean-exit relaunch via `KeepAlive=true`.
