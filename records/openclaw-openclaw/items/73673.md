## Summary

Keep open. Current main still generates the macOS LaunchAgent with boolean KeepAlive and a one-second throttle, audits only that form, and tests assert SuccessfulExit is absent; the linked candidate fix PR is still open and unmerged.

## What This Changes

Request that launchd-managed Gateway startup failures stop causing rapid restart loops so users can inspect and debug the failure.

## Best Possible Solution

Update the macOS launchd plist generation, audit, tests, docs, and changelog so startup failures are inspectable while intentional managed restarts remain supported, then resolve this through #73849 or a focused replacement.

## Reproduction Assessment

Yes. A source-level reproduction is clear: generate/install the current LaunchAgent, start the Gateway with an invalid startup option such as --port, and the CLI exits 1 while launchd is configured with KeepAlive=true and ThrottleInterval=1.

## Solution Assessment

Unclear. The narrow maintainable path is launchd-policy work in the daemon files, but the exact policy should be validated on macOS so it does not trade the reported crash loop for broken managed restarts.
