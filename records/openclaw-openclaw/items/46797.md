## Summary

Keep open. Current main still formats successful `config.patch` restart sentinels through the generic `Gateway restart config-patch ok (config.patch)` summary, and the focused formatter/test path has not been updated.

## What This Changes

The issue asks for the `config.patch` restart notice to say that a restart is required instead of implying the restart already completed.

## Best Possible Solution

Update the restart-sentinel summary for successful `config.patch` restart-required notices to explicit wording such as `Gateway restart required (config.patch)`, while preserving the note, reason line, doctor hint, and existing behavior for other sentinel kinds.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.patch` creates a `config-patch` sentinel with `status: "ok"` and `stats.mode: "config.patch"`, and the formatter currently renders that as `Gateway restart config-patch ok (config.patch)`.

## Solution Assessment

Yes. A targeted formatter or sentinel-payload wording change is the narrowest maintainable fix; changing reload planning, Windows restart mechanics, or config validation would be broader than the reported UX bug.
