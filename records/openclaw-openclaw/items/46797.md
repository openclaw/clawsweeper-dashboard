Keep open: current main still formats a config.patch restart sentinel as `Gateway restart config-patch ok (config.patch)` before appending the caller note, so the misleading wording reported here remains present and narrow enough to fix.

## What This Changes

The issue asks for the config.patch restart system notice to say restart is required rather than implying it already completed, while keeping the note text below it.

## Best Possible Solution

Make a focused restart-sentinel wording change for config.patch restart-required notices, with regression coverage that preserves note, doctor hint, wake, continuation, and outbound delivery behavior.
