## Summary

Keep open: current main still formats config.patch restart sentinels as an `ok` restart status, so the user-facing notice can read as if the restart already completed even though the note and doctor hint are only appended below that misleading summary.

## What This Changes

The issue asks for the config.patch restart notice to say that a Gateway restart is required instead of implying it already happened.

## Best Possible Solution

Update the shared restart-sentinel formatter, with focused tests, so config.apply/config.patch notices clearly describe a required or requested restart while preserving the existing note, reason, and doctor-hint lines.

## Reproduction Assessment

Yes. Source inspection of current main shows config.patch writes create a `config-patch`/`ok`/`config.patch` sentinel payload, and the formatter renders that as `Gateway restart config-patch ok (config.patch)` before the note.

## Solution Assessment

Yes. A focused formatter and regression-test update is the narrowest maintainable fix; changing restart scheduling, no-op behavior, or config policy would solve a different problem.
