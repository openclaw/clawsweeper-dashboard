## Summary

Keep open. Current main still formats stats-backed `config.patch` restart notices through the generic `Gateway restart config-patch ok (config.patch)` shape, so the reported wording bug remains valid and narrow.

## What This Changes

The issue requests clearer system notice wording when `config.patch` requires a gateway restart, so it does not read as if the restart already happened.

## Best Possible Solution

Update the restart-sentinel formatter and focused regression tests so successful `config.patch` restart-required notices clearly say a restart is required while preserving note text, reason lines, doctor hints, and existing behavior for other sentinel kinds.

## Reproduction Assessment

Yes. A current-main source reproduction path is clear: a changed `config.patch` creates a `config-patch`/`ok`/`config.patch` sentinel, and startup posts the shared formatter output as the system notice.

## Solution Assessment

Yes. A targeted formatter and regression-test update is the narrowest maintainable fix; restart scheduling, config validation, Windows supervisor behavior, and security guards do not need to change for this wording bug.
