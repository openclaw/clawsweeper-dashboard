## Summary

Keep open: current main and the latest release still format config-patch restart sentinels as the generic `Gateway restart config-patch ok (config.patch)` wording, and the linked fix PR is open rather than merged.

## What This Changes

The issue asks for `config.patch` restart notices to say that a restart is required instead of implying the restart already happened.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/83041 after it is updated against current main, or replace it with an equivalent focused patch that carries restart-required state into the sentinel payload and formatter.

## Reproduction Assessment

Yes. Current main still routes config.patch success sentinels through the generic kind/status/mode formatter, which source-inspection shows will produce the reported wording without requiring a full Windows gateway run.

## Solution Assessment

Yes. Carrying the restart-required fact from the config write path into the sentinel payload and formatting pending versus completed state is the narrow maintainable fix, and the linked PR is the current focused implementation path.
