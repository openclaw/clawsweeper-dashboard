## Summary

Current main and the latest release still produce the misleading config-patch restart notice. The sole canonical fix is the linked closing pull request, which remains open, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Clarify config write restart notices so required restarts are not presented as already successful or completed.

## Best Possible Solution

Refresh and review https://github.com/openclaw/openclaw/pull/83041 against current main, then land one canonical lifecycle-aware formatter that distinguishes required, completed, and hot-reloaded config writes.

## Reproduction Assessment

Yes, from source with high confidence. Current main deterministically formats an `ok` config-patch sentinel as `Gateway restart config-patch ok (...)`, while its payload omits whether a restart is actually pending.

## Solution Assessment

Yes. Carrying the reload-plan fact into the canonical sentinel and formatting pending versus completed lifecycle state is the narrow maintainable fix; a Windows-only text substitution would leave sibling states inaccurate.
