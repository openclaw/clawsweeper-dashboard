## Summary

Keep open. Current main still creates a stats-backed `config.patch` restart sentinel and formats it through the generic `Gateway restart {kind} {status} ({mode})` summary, so the reported misleading notice remains a narrow wording bug.

## What This Changes

Change the `config.patch` restart system notice to clearly say a restart is required while preserving the note text below it.

## Best Possible Solution

Update the restart sentinel formatter and focused regression tests so successful `config.patch` restart-required notices say `Gateway restart required (config.patch)` or equivalent, while preserving note, reason, doctor hint, and other sentinel behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a changed `config.patch` creates a `config-patch`/`ok`/`config.patch` sentinel and startup posts the shared formatter output as the system notice.

## Solution Assessment

Yes. A targeted formatter and regression-test update is the narrowest maintainable fix; restart scheduling, config validation, Windows supervisor behavior, and security guards do not need to change for this wording bug.
