## Summary

Keep open. Current main still formats successful `config.patch` restart sentinels with the generic `Gateway restart config-patch ok (config.patch)` summary, so the reported restart-required wording bug remains valid and narrow.

## What This Changes

The issue asks for the `config.patch` restart notice to say a restart is required rather than implying it already happened.

## Best Possible Solution

Change only the successful `config.patch` restart-sentinel summary to explicit restart-required wording while preserving note text, reason lines, doctor hints, and existing summaries for other sentinel kinds/statuses.

Yes. A `config.patch` write that creates a `config-patch`/`ok` sentinel with `stats.mode: "config.patch"` formats to the reported generic restart-completed-looking summary on current main.
