## What This Changes

The issue requests clearer config.patch restart sentinel wording so pending config changes say that a restart is required instead of implying the restart already happened.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83041 if it remains correct, or apply the same narrow requiresRestart sentinel state and formatter fix on main.

## Reproduction Assessment

Yes, at source level. config.patch creates a config-patch ok sentinel with mode config.patch, and the shared formatter still renders the generic `Gateway restart config-patch ok (config.patch)` summary before the note.

## Solution Assessment

Yes. The narrow maintainable fix is to carry restart-required state into the sentinel payload/formatter with regression coverage; the open linked PR already attempts that path.
