## Summary

Keep open: current main and the latest release still format successful `config.patch` restart sentinel notices as an `ok` restart message, and open fix PRs are already linked as candidate implementations. The issue should stay open until one of those PRs is reviewed and landed or explicitly replaced.

## What This Changes

The issue asks for `config.patch` restart sentinel notices to say that a Gateway restart is required, rather than implying the restart already succeeded.

## Best Possible Solution

Review and land or replace a linked fix that makes pending restart-required config writes say `Gateway restart required (...)` while preserving completed-restart and hot-reload wording.

## Reproduction Assessment

Yes, from source inspection: current main still formats a successful `config-patch` sentinel from `kind`, `status`, and `mode`, producing the misleading `ok` restart notice. I did not run a live Gateway repro in this read-only review.

## Solution Assessment

Yes, the maintainable fix belongs at the restart sentinel formatter/payload boundary. A blanket `config.patch` rewrite is less complete unless it distinguishes pending restart-required, completed restart, and hot-reloaded config writes.
