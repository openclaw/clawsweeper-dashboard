## Summary

Keep open. Current main still formats `config.patch` restart sentinels through the generic `Gateway restart config-patch ok (config.patch)` path, so the reported restart-required wording bug remains valid.

## What This Changes

The issue asks to reword the `config.patch` restart notice so it clearly says a gateway restart is required rather than implying one already completed.

## Best Possible Solution

Change the successful `config.patch` restart-sentinel summary to explicit restart-required wording while preserving the note text, reason line, doctor hint, and existing behavior for other sentinel kinds.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.patch` creates a `config-patch` sentinel with `status: "ok"` and `stats.mode: "config.patch"`, and the formatter currently renders that metadata as the reported generic restart summary.

## Solution Assessment

Yes. A targeted formatter or sentinel-payload wording change is the narrowest maintainable fix; changing reload planning, Windows restart mechanics, or config validation would be broader than the reported UX problem.
