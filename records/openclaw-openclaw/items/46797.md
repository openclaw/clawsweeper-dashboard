## Summary

Current main still appears to produce the misleading config.patch restart notice, so this should stay open as a narrow user-visible wording bug rather than be swept closed.

## What This Changes

The issue requests that config.patch restart notices say a restart is required instead of implying the restart already completed.

## Best Possible Solution

Change the config.patch restart notice path to use explicit restart-required or pending wording, while preserving the note text below it and leaving restart scheduling semantics unchanged.

## Reproduction Assessment

Yes. A config.patch write that changes a restart-required key follows `config.ts` into the shared sentinel builder, creates a `config-patch`/`ok` payload, and `summarizeRestartSentinel` renders the reported wording on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a restart-sentinel/config.patch wording change with focused tests, not a new restart mechanism, config option, or reload policy change.
