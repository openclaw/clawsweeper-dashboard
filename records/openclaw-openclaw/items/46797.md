## Summary

Keep open. Current main still formats successful `config.patch` restart sentinels through the generic `Gateway restart config-patch ok (config.patch)` summary, the latest release tag has the same formatter, and no paired implementation PR was found in the reviewed context or targeted searches.

## What This Changes

The issue requests changing the `config.patch` restart notice to clearly say a gateway restart is required, while preserving the note text below it.

## Best Possible Solution

Update the restart-sentinel summary for successful `config.patch` restart-required notices to explicit wording such as `Gateway restart required (config.patch)`, while preserving the note, distinct reason line, doctor hint, and existing summaries for update, explicit restart, `config.apply`, and auto-recovery cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `config.patch` writes a `config-patch` sentinel with `stats.mode: "config.patch"`, startup formats it through `formatRestartSentinelMessage`, and the summary currently renders as `Gateway restart config-patch ok (config.patch)`.

## Solution Assessment

Yes. A targeted formatter or sentinel-payload wording change is the narrowest maintainable fix; changing reload planning, Windows restart mechanics, or restart scheduling would be broader than the reported UX bug.
