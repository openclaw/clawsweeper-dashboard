## Summary

Keep open. Current main still formats successful `config.patch` restart sentinels through the generic `Gateway restart config-patch ok (config.patch)` summary, so the reported misleading notice remains valid and narrow.

## What This Changes

The issue requests changing `config.patch` restart notices so they say a gateway restart is required rather than implying the restart already completed.

## Best Possible Solution

Keep the existing restart flow, but special-case successful `config.patch` restart-required sentinel summaries to explicit wording such as `Gateway restart required (config.patch)` while preserving note, reason, doctor hint, and other sentinel kinds.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `config.patch` creates a `config-patch` payload with `status: "ok"` and `stats.mode: "config.patch"`, and the formatter renders that through the generic restart summary.

## Solution Assessment

Yes. A targeted formatter or payload wording change is the narrowest maintainable fix; changing reload planning, Windows restart mechanics, or config validation would be broader than the reported UX bug.
