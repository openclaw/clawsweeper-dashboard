## Summary

Keep open: current main and the latest release still format config.patch restart sentinels with the misleading generic wording, and https://github.com/openclaw/openclaw/pull/83041 is an open closing PR that targets this exact issue.

## What This Changes

The issue requests that config.patch restart sentinel notices say a gateway restart is required, not that the config-patch restart already completed, while preserving the note text.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83041 if its sentinel-contract changes pass normal CI; otherwise apply the same narrow restart-required formatter and payload fix on main.

## Reproduction Assessment

Yes. The source path is clear: config.patch writes a config-patch ok restart sentinel with mode `config.patch`, and the shared formatter renders it as the misleading `Gateway restart config-patch ok (config.patch)` prefix.

## Solution Assessment

Yes. The narrow maintainable fix is to carry reload-plan restart-required state into the sentinel payload/formatter and add regression coverage, which the open linked PR is already attempting.
