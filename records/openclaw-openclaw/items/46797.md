## Summary

Keep open: current main still formats config.patch restart sentinels as an `ok` restart status, and the config write path feeds that summary into the user-facing notice before appending the note text.

## What This Changes

Request to reword the config.patch restart notice so it says a restart is required rather than implying it already completed, while preserving the note below it.

## Best Possible Solution

Special-case config.apply/config.patch sentinel summary wording to describe a restart requirement or request, preserve the note and diagnostic lines, and leave restart scheduling and suppression policy unchanged.

Yes. Source inspection shows a config.patch write creates a `config-patch`/`ok`/`config.patch` sentinel payload, and the formatter renders the reported `Gateway restart config-patch ok (config.patch)` line before the note.

## Solution Assessment

Yes. A focused formatter/test update is the narrowest maintainable fix; adding a suppression flag or changing restart scheduling would solve different issues.
