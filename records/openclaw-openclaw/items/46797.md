## Summary

Keep open. Current main still formats successful `config.patch` restart sentinel notices with the generic completed-looking `Gateway restart config-patch ok (config.patch)` summary, while the config docs and gateway tool copy describe config writes as restart-required when needed; the reported wording bug is valid and narrow.

## What This Changes

The issue asks for the `config.patch` restart notice to say that a restart is required rather than implying the restart has already completed.

## Best Possible Solution

Update the restart-sentinel formatting so the successful `config.patch` notice clearly communicates restart-required state while preserving note text, reason lines, doctor hints, and existing behavior for other sentinel kinds/statuses.

## Reproduction Assessment

Yes. A current-main source reproduction path is clear: a changed `config.patch` writes a `config-patch`/`ok` sentinel with `stats.mode: "config.patch"`, and startup formats that payload through the generic `Gateway restart config-patch ok (config.patch)` summary.

## Solution Assessment

Yes. A targeted formatter and regression-test update is the narrowest maintainable fix; restart scheduling, config validation, Windows lifecycle mechanics, and config security guards do not need to change for this wording bug.
