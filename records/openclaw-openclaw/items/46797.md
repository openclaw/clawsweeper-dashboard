## Summary

Keep open: current main still builds a config.patch restart sentinel with `kind: "config-patch"`, `status: "ok"`, and `mode: "config.patch"`, and the shared formatter still renders that as a completed-looking `Gateway restart config-patch ok (config.patch)` first line before the note.

## What This Changes

The issue requests changing the config.patch gateway restart notice so it says restart is required rather than reading like a completed restart.

## Best Possible Solution

Keep the issue open until the shared restart-sentinel formatter and focused regression tests make config.apply/config.patch notices say a restart is required or requested while preserving note, reason, and doctor-hint lines.

## Reproduction Assessment

Yes. Source inspection on current main shows the config.patch write path still creates the exact payload shape that the formatter renders as `Gateway restart config-patch ok (config.patch)`.

## Solution Assessment

Yes. A focused formatter and regression-test update is the narrowest maintainable fix; changing restart scheduling, hot-reload policy, or config write behavior would solve a different problem.
