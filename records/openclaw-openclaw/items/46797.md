## What This Changes

Update config-patch restart sentinel notices so pending manual-restart cases say restart is required instead of implying the restart already happened.

## Solution Assessment

Yes: the best path is the existing focused PR that carries restart-required state into the sentinel payload and formats pending versus completed restart notices. This issue should remain open until that PR merges or is replaced.
