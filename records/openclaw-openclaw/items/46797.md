## Summary

Keep open. Current main still formats successful `config.patch` restart sentinels through the generic `Gateway restart config-patch ok (config.patch)` summary, so the misleading restart-completed wording remains valid.

## What This Changes

The issue asks for the `config.patch` restart notice to say that a gateway restart is required rather than implying the restart already completed.

## Best Possible Solution

Change the successful `config.patch` restart-sentinel summary to explicit restart-required wording while preserving the note text, reason line, doctor hint, and existing behavior for other sentinel kinds.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.patch` creates a `config-patch` sentinel with `status: "ok"` and `stats.mode: "config.patch"`, then startup formats it through the generic restart summary.

## Solution Assessment

Yes. A targeted formatter and test update is the narrowest maintainable fix; changing reload planning or Windows restart mechanics would be broader than the reported notice bug.
