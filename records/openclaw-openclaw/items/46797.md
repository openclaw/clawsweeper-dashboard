## Summary

Keep open: current main still formats config patch restart-required sentinel messages with the misleading `Gateway restart config-patch ok (config.patch)` wording, and an open linked fix PR now specifically addresses this issue but has not merged yet.

## What This Changes

The issue asks for `config.patch` restart notices to say that a gateway restart is required rather than implying the restart already completed.

## Best Possible Solution

Review and land the linked fix PR if its diff preserves sentinel delivery semantics while changing restart-required wording and regression coverage; otherwise close that PR and apply the same narrow formatter/payload fix on main.

## Reproduction Assessment

Yes. The source path is clear: `config.patch` emits a `config-patch`/`ok` restart sentinel with mode `config.patch`, and the shared formatter turns that into the misleading `Gateway restart config-patch ok (config.patch)` summary.

## Solution Assessment

Yes. The narrow maintainable solution is to carry restart-required state into the sentinel summary and add focused regression coverage, while leaving the separate notification-suppression feature to https://github.com/openclaw/openclaw/issues/55372.
