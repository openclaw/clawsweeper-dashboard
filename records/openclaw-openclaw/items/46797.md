## Summary

Keep open: current main and the latest release still use the generic restart-sentinel wording that produces the reported `Gateway restart config-patch ok (config.patch)` notice, and this issue has two open closing PRs that should be reviewed before the issue closes.

## What This Changes

The issue asks for `config.patch` restart notices to say a restart is required instead of implying the restart already happened, while preserving the note text below it.

## Best Possible Solution

Land a restart-sentinel fix that records or derives whether config apply/patch actually requires a restart, formats pending required restarts clearly, preserves completed/hot-reload cases, and then closes this issue.

## Reproduction Assessment

Yes. Source inspection on current main shows successful `config.patch` sentinels still flow through the generic `kind/status/mode` summary path, which produces the reported wording without needing a full Windows restart run.

## Solution Assessment

Yes. The best fix is narrow restart-sentinel payload/formatting work in the linked PR path, with care not to relabel hot-reloaded or already-completed config writes as still restart-required.
