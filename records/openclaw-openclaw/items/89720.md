## What This Changes

The PR guards plain-text tool-call context name extraction against throwing `name` accessors and adds a regression test that preserves promotion of a healthy sibling tool.

## Best Possible Solution

Keep the shared-wrapper guard and regression test, rebase it onto current main, then land after the normal maintainer review and required gates pass.

## Reproduction Assessment

Yes from source: current main's `resolveContextToolNames` reads `record?.name` without guarding accessor exceptions, so a throwing sibling tool can abort name collection before healthy tools are promoted. I did not run the failing regression because this review is read-only.

## Solution Assessment

Yes: the PR catches only the context name read, skips malformed or unreadable entries, and keeps the shared allowed-tool-name matcher for the affected provider wrappers. Fixing providers individually or changing the repair package would be less direct for this failure point.
