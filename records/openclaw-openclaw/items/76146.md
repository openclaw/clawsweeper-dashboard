## Summary

Keep open: the PR fixes the main Brave-style CLI SecretRef path in the right direction, but it still misses Gemini's documented `models.providers.google.apiKey` fallback and lacks the required user-facing changelog entry.

## What This Changes

The PR changes `capability web search` to resolve command SecretRefs, install the resolved runtime snapshot, and add secret-target coverage for web-search credentials.

## Best Possible Solution

Keep the shared command resolver plus runtime-snapshot approach, add the Gemini Google model-provider fallback credential to the web-search command SecretRef path with focused tests, and include an Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. Current main has a code-traceable reproduction path: configure `tools.web.search.apiKey` as a file SecretRef and run `openclaw capability web search --query ...`; the remaining PR gap is similarly traceable with `--provider gemini` and `models.providers.google.apiKey` as a SecretRef.

## Solution Assessment

No, not yet. The PR uses the right resolver seam, but the target set does not cover every documented credential source for the affected CLI path and the changelog requirement is unmet.
