## Summary

Keep open: the reported CLI SecretRef bug is source-reproducible and the shared command resolver approach is the right direction, but the PR still misses Gemini's documented Google model-provider fallback SecretRef and lacks concrete after-fix real behavior proof.

## What This Changes

The PR makes `capability web search` resolve web-search SecretRefs through command secret resolution, installs the resolved runtime snapshot, and adds target-set coverage.

## Best Possible Solution

Extend the web-search command target set and tests to cover Gemini's `models.providers.google.apiKey` fallback while keeping the scoped command SecretRef resolver and runtime-snapshot install approach.

## Reproduction Assessment

Yes by source trace, though not by a live command in this read-only pass. Current main passes the CLI runtime config into `runWebSearch` without command SecretRef resolution, and the PR's remaining Gemini gap is traceable with `--provider gemini` plus `models.providers.google.apiKey` as a SecretRef.

## Solution Assessment

No, not yet. The shared command resolver direction is the narrow maintainable fix, but the target set must include the documented Gemini fallback credential before the PR is complete.
