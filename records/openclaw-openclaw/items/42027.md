## Summary

Keep open, but do not merge as submitted. Current main already solved the exec and cron symptoms through different mainline changes, while the remaining browser diagnostics work is still useful but must be ported to the active browser plugin instead of the obsolete core browser paths.

## What This Changes

The branch modifies legacy core exec, browser, CLI, and cron code to add PATH fallback, layered browser diagnostics with HTML error summarization, and a separate manual cron lane.

## Best Possible Solution

Replace or split this branch: keep current main's exec auto-routing and CronNested behavior, and port only the remaining layered browser diagnostics plus bounded HTML upstream-error summaries to `extensions/browser`.

## Reproduction Assessment

Yes, by source inspection. The active browser plugin still lacks a diagnostics payload and bounded HTML upstream-error summaries, while the exec and cron symptoms are already addressed on current main through different code paths.

## Solution Assessment

No, not as submitted. The useful browser direction should be implemented in the active bundled browser plugin; merging changes against removed core browser and CLI paths would not solve the current product surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Port diagnostics to the active browser plugin:** `src/browser/routes/basic.ts:74-79`
  - body: Current main serves browser status from `extensions/browser`, while this PR adds the diagnostics payload under the removed `src/browser` route. As submitted, the active `openclaw browser status` path would still have no layered diagnostics after a rebase, so the remaining fix needs to move to the plugin route, type, and CLI.
  - confidence: 0.92
- **[P2] Bound the config snapshot with the status timeout:** `src/cli/browser-cli-shared.ts:87`
  - body: The new status diagnostics path fetches `config.get` without the short timeout used by the browser status request. If the gateway config path is slow or partially unavailable, `openclaw browser status` can wait on the default RPC timeout before falling back, which is a responsiveness regression.
  - confidence: 0.86
- **[P3] Cap HTML fallback summaries:** `src/browser/client-fetch.ts:157-163`
  - body: When an HTML error page has no title or h1, the fallback strips tags and returns the full remaining body. Large proxy pages can still produce multi-kilobyte browser errors, so the fallback should sanitize entities and cap the text before throwing it.
  - confidence: 0.78
