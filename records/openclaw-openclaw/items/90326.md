The PR changes the Fireworks plugin to defer manifest-declared Kimi model ids from `resolveDynamicModel`, adds `reasoning:false` to the two Kimi manifest rows, deep-clones catalog models, and updates Fireworks tests.

## Best Possible Solution

Keep the manifest as the canonical Fireworks catalog source, but make exact manifest ids resolve from that metadata on every current resolver path before merging.

## Solution Assessment

No: deferring from the plugin hook is only safe for callers that later run bundled static-catalog fallback. The best fix must preserve manifest metadata for sync resolver callers too.
