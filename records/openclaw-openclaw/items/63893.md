## What This Changes

This PR rebuilds legacy TaskFlow SQLite `flow_runs` tables that still contain `owner_session_key`, preserves rows using an `owner_key` fallback, adds regression coverage, and updates the changelog for #65563.

## Best Possible Solution

Land one canonical migration that rebuilds only legacy `flow_runs` tables with `owner_session_key`, preserves rows under the current `owner_key` schema, then close #65563 and handle #65767 as superseded if this PR lands.

## Solution Assessment

Yes. Rebuilding the legacy table after additive schema migrations is the narrow maintainable fix because SQLite cannot drop the stale NOT NULL column in place, and the PR covers both pure legacy and hybrid owner-key states.
