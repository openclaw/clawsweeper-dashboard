## Summary

Keep open. Current main and v2026.5.19 still do not implement WhatsApp missed-message backfill: stale `append` catch-up is skipped after read marking, the socket disables full history sync, and the open WhatsApp PRs are competing unmerged repair paths that need maintainer choice and live proof.

## What This Changes

Request to backfill WhatsApp group messages missed during Web disconnect/reconnect windows instead of silently dropping them.

## Best Possible Solution

Land one bounded WhatsApp-plugin recovery design that filters to eligible monitored chats, caps replay, dedupes by message ID, preserves startup-history safety, documents Baileys limitations, and includes real reconnect proof.

## Reproduction Assessment

Yes for source-level reproduction: emit a WhatsApp `messages.upsert` event of type `append` with a timestamp more than 60 seconds before `connectedAtMs`, and current main skips it before enqueue. I did not run a live WhatsApp outage reproduction.

## Solution Assessment

Unclear. The WhatsApp plugin is the right boundary, but maintainers still need to choose the replay policy and validate whether Baileys `append` handling, `messaging-history.set`, opt-in config, or explicit gap warnings are the safest fix.
