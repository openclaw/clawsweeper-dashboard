## Summary

Keep open. Current main and latest release still skip stale WhatsApp `append` catch-up messages after a 60-second connect grace and have no per-chat reconnect history/backfill path; the open candidate PRs need maintainer policy selection plus live WhatsApp proof before this can close.

## What This Changes

Request WhatsApp reconnect recovery so monitored chats either replay messages missed during an outage window or clearly report that the gap cannot be recovered.

## Best Possible Solution

Keep this as the canonical WhatsApp reconnect missed-message tracker and land one plugin-owned bounded recovery policy with live proof: either validated replay for eligible monitored chats or a clear unrecoverable-gap warning when backfill cannot be trusted.

## Solution Assessment

Unclear. The owner boundary is the WhatsApp plugin inbound/reconnect path, but maintainers still need to choose default replay, opt-in replay, or explicit gap warnings and require live WhatsApp proof.
