## Summary

Keep open. Current main and the latest release still skip stale WhatsApp append catch-up messages and have no per-chat reconnect backfill or history-sync path; open candidate fixes at https://github.com/openclaw/openclaw/pull/80642 and https://github.com/openclaw/openclaw/pull/76175 should be reviewed before this canonical tracker is resolved.

## What This Changes

Request WhatsApp plugin recovery of group messages missed during Web disconnects by backfilling monitored chats after reconnection.

## Best Possible Solution

Keep this as the canonical WhatsApp reconnect-backfill tracker and land one bounded WhatsApp-plugin recovery path with real reconnect proof, replay limits, dedupe, monitored-chat filtering, and clear operator warning semantics.

## Reproduction Assessment

Yes for source-level reproduction: current code skips append upserts older than 60 seconds before connect, and the focused test expects a five-minute-old append message not to reach `onMessage`. I did not run a live WhatsApp outage experiment in this read-only review.

## Solution Assessment

Unclear. The WhatsApp plugin is the right boundary, but the best final path still needs maintainer choice between a narrow reconnect append grace, an opt-in offline replay setting, or a fuller `messaging-history.set` backfill design.
