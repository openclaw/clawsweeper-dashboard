## Summary

Keep open. Current main and v2026.5.18 still do not implement WhatsApp missed-message backfill: the plugin consumes live Baileys `messages.upsert` events, skips older `append` backlog after read marking, has no `messaging-history.set`/cursor path, and the open WhatsApp PRs remain competing unmerged approaches that need maintainer review.

## What This Changes

Request that the WhatsApp plugin backfill or recover messages missed while the WhatsApp Web connection was down after reconnecting.

## Best Possible Solution

Land one bounded WhatsApp-plugin recovery design that filters to eligible monitored chats, caps replay, dedupes by message ID, preserves startup-history safety, documents provider limitations, and includes real reconnect proof.

## Reproduction Assessment

Yes for source-level current behavior: current main has no per-chat reconnect cursor or history listener, and its append regression test expects a five-minute-old catch-up message to be skipped. I did not run a live WhatsApp outage reproduction.

## Solution Assessment

Unclear. The WhatsApp plugin is the right boundary, but maintainers still need to choose the replay policy and validate whether Baileys `append` handling, `messaging-history.set`, or warning-only behavior is the safest user-facing fix.
