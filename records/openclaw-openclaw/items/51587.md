## What This Changes

The issue requests parsing cron minute fields such as steps, lists, and ranges that include minute zero when deciding whether to apply the default top-of-hour stagger.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/51594 or an equivalent narrow fix that parses minute fields consistently with Croner and adds regression coverage across helper, normalization, and service default-stagger paths.
