## What This Changes

The issue requests that cron top-of-hour stagger detection recognize minute fields that include minute zero, such as step, list, and range expressions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/51594 or a narrow replacement with Croner-compatible minute-zero parsing and regression coverage across helper, service, normalization, and doctor migration paths, then close this issue after merge.
