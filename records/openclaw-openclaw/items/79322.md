## Summary

Keep open: the report targets documented plugin-install behavior and current source shows the secondary hook-pack message can follow an npm plugin-install failure, but the root npm "reading 'spec'" failure is not reproducible from the issue text alone and needs the Windows npm debug log or a live Windows repro.

## What This Changes

Installing the external WeChat plugin via the Tencent npx wrapper or `openclaw plugins install @tencent-weixin/openclaw-weixin` fails on Windows with an npm internal error and hook-pack validation fallback.

## Best Possible Solution

Reproduce the install on Windows with the npm debug log available, then either fix the managed npm-root install path in core or route a package/installer-wrapper fix to the Tencent Weixin plugin maintainers.

## Reproduction Assessment

No. Source inspection explains the hook-pack suffix after an npm plugin-install failure, but the actual npm `Cannot read properties of undefined (reading 'spec')` failure needs the Windows npm debug log or a live Windows repro.

## Solution Assessment

Unclear. A diagnostic-only core fix could reduce the confusing hook-pack suffix for known plugin packages, but the best functional fix depends on reproducing the npm failure and identifying whether it is core-managed npm state or the external package path.
