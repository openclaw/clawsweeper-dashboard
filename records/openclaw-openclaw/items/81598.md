## Summary

Keep open: current main still has a source-visible gap where `message(action="send")` builds and sends outbound payloads without the TTS payload normalizer, while final-reply delivery does use it. The linked closing PR is still open and has review concerns, so this issue should remain open until a corrected fix lands.

## What This Changes

The issue reports that `message(action=send)` paths forward TTS directives literally instead of invoking the existing TTS payload conversion used by final replies.

## Best Possible Solution

Land a narrow fix that applies the existing TTS payload normalization before all message-action send branches, including gateway-mode sends, with focused regression tests and real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `message(action="send")` flows through `message-tool.ts`, `message-action-runner.ts`, and `outbound-send-service.ts` without a TTS hook, while final replies call the TTS normalizer before delivery.

## Solution Assessment

Yes for the requested direction: reusing the existing TTS payload path in shared outbound send handling is the narrow maintainable fix. The current linked PR still needs corrections before it is the best implementation.
