## Summary

Keep this open: current main still has a source-visible gap where `message(action="send")` reaches outbound delivery without a TTS payload normalization hook, while final reply delivery does apply TTS. The linked closing PR is still open and has unresolved review findings, so the issue should wait on a corrected fix rather than be closed.

## What This Changes

Apply existing TTS payload normalization to `message(action="send")` so TTS directives are stripped and synthesized audio attaches on message-tool sends.

## Best Possible Solution

Land a reviewed fix that normalizes TTS before all outbound message-action execution modes, including gateway, prepared payload, plugin action, and core fallback paths, with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows `message(action="send")` flows from `message-tool.ts` through `message-action-runner.ts` into `outbound-send-service.ts` without a TTS hook, while final reply delivery calls the TTS normalizer before routing.

## Solution Assessment

Yes for the requested direction, but the linked PR is not the best fix as written because review found gaps for gateway sends, default TTS mode behavior, and plugin-action media propagation. The maintainable fix belongs in shared message-action payload normalization rather than a new config option.
