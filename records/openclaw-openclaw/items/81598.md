## Summary

This should stay open: current main still lacks a TTS normalization call on the message-tool send path, while final reply delivery uses the existing TTS payload normalizer. The issue is paired with an open closing PR, so cleanup should wait for that PR to be corrected and landed or replaced.

## What This Changes

The issue asks OpenClaw to apply TTS normalization to `message(action="send")` so TTS directives are stripped and audio is attached on message-tool sends.

## Best Possible Solution

Land a corrected shared outbound normalization patch that covers gateway, prepared-payload, plugin-action, and core fallback sends without adding a new config option.

## Reproduction Assessment

## Solution Assessment
