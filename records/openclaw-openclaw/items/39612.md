## Summary

## What This Changes

The issue requests framework-level graceful handling for LLM/provider failures so raw API/session errors are not delivered to messaging-channel users.

## Best Possible Solution

Define one external-channel LLM/provider failure contract: generic or silent by default, diagnostics only on approved operator surfaces, and repeated-failure/session-recovery behavior coordinated with the narrower replay-sanitizer follow-ups.
