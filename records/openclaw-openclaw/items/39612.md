The issue requests framework-level handling for LLM/provider failures so raw errors are not sent to WhatsApp, Telegram, or other users, with retry, friendly/silent fallback, escalation, and session-corruption detection.

## Best Possible Solution

Define one external-channel LLM/provider failure contract: generic or silent by default, diagnostics only on approved operator surfaces, and repeated-failure/session recovery coordinated with existing transcript hygiene paths.
