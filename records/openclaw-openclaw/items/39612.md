## Summary

Close as implemented: current main masks external-chat LLM/provider failures with generic fallback text or silence, retries the reported Anthropic thinking-block failure with cleaned history, and the relevant fixes are release-proven. The newer 2026.4.27 WhatsApp instability report remains the better place for any separate release-specific regression evidence.

## What This Changes

Request framework-level handling so LLM/provider failures do not leak raw errors to WhatsApp/Telegram users and corrupted thinking-block sessions recover or fail gracefully.

## Best Possible Solution

Keep the framework-level fallback and Anthropic recovery path shipped in v2026.4.2; use the newer WhatsApp regression thread for any remaining release-specific leakage or fallback reports.

## Reproduction Assessment

Yes. Source and regression-test coverage exercise the raw external-chat failure path and the exact Anthropic thinking-block rejection; this read-only review did not run tests, but the current-main paths are clear.

## Solution Assessment

Yes. Handling the error in the framework runner before channel delivery, plus provider-specific retry/cleanup for Anthropic thinking history, is narrower and more maintainable than channel-by-channel masking; automatic multi-agent escalation would be a separate product feature.
