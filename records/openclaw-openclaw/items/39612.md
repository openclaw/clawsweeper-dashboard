## Summary

Keep open: current main has the immediate generic fallback and Anthropic thinking-block recovery paths, and the related MiniMax/WhatsApp reasoning leak was handled separately, but the stricter external-channel disclosure policy and failure-streak escalation request are still unresolved.

## What This Changes

The issue requests framework-level handling for LLM failures so raw provider/runtime errors are not sent to messaging-channel users, plus retry, session-recovery, and multi-agent escalation behavior for repeated failures.

## Best Possible Solution

Define the external-channel failure contract explicitly: normal chats get generic or silent failure handling, debug details stay in logs/control UI or an approved opt-in surface, and repeated-failure escalation is designed as a separate multi-agent policy if maintainers want it.

## Reproduction Assessment

No high-confidence live reproduction of the original 2026.3.7 WhatsApp/Telegram leak was established. Source and tests do show the remaining strict-policy gap: verbose direct-chat paths can still include sanitized provider/runtime details.

## Solution Assessment

Unclear. The shipped generic fallback and Anthropic thinking recovery are the right immediate fixes, but changing verbose direct-chat behavior or adding automatic main-agent escalation needs maintainer policy rather than an automated bug-fix PR.
