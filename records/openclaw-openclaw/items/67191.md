## Summary

Keep open. Current main still runs BOOT.md through `agentCommand` with `deliver:false` and no startup destination context, matching the macOS and Telegram reports; the remaining fix needs a maintainer decision about the BOOT.md delivery contract.

## What This Changes

The issue requests visible delivery for BOOT.md startup status messages instead of silent boot-md runs.

## Best Possible Solution

Define one explicit BOOT.md startup contract, then update boot execution, docs, and regression tests so startup status messages either inherit a configured destination or docs require explicit channel plus target.

## Reproduction Assessment

Yes. The report gives concrete restart steps, and source review confirms the same path on current main: boot-md calls `runBootOnce` without delivery context, `runBootOnce` passes `deliver:false`, and a unit test asserts that call shape.

## Solution Assessment

No. The suggested one-line `deliver:true` change is not proven best because the boot path still lacks channel/target context; maintainers should choose the startup delivery contract first.
