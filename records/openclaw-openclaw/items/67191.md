## Summary

Keep open. Current main still runs BOOT.md through a silent boot session, and the startup hook does not carry enough routing context to make the suggested `deliver:true` flip an obviously safe fix.

## What This Changes

The issue asks for BOOT.md startup messages to be delivered to users instead of being suppressed by the boot-md hook's `deliver:false` run.

## Best Possible Solution

Define one explicit BOOT.md startup delivery contract, then align runBootOnce, boot-md startup context, docs, and regression tests around that contract.

## Reproduction Assessment

Yes, source-reproducible. A non-empty BOOT.md reaches runBootOnce, which calls agentCommand with deliver:false and a no-op log runtime, while the startup hook forwards no route for inherited delivery.

## Solution Assessment

No. The suggested one-line deliver:true change is not proven to be the best fix because current startup context lacks a channel, target, thread, or reply context, and strict delivery without a route can fail before sending.
