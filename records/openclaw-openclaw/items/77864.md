## Summary

Keep open: current `main` still has a source-level path that can turn warning-only Active Memory search output into `status=empty`, and https://github.com/openclaw/openclaw/pull/78083 explicitly targets this issue but is still open and unmerged.

## What This Changes

Active Memory should stop returning empty recall payloads for every query when the memory backend has usable results or only advisory warnings.

## Best Possible Solution

Review and land a narrow Active Memory fix through https://github.com/openclaw/openclaw/pull/78083 or a replacement that distinguishes advisory warnings from hard unavailable/error states and proves the runtime behavior with real logs or terminal output.

## Reproduction Assessment

Yes, source-level. Current `main` maps warning-only `memory_search` output into the terminal unavailable branch and can abort recall with `status=empty`; I did not establish a live gateway reproduction in this read-only review.

## Solution Assessment

Yes for the main runtime direction. The linked PR’s core change is the narrowest maintainable fix for warning-only aborts, but it still needs real behavior proof and should be checked against the empty-cache and zero-hit concerns raised in the discussion.
