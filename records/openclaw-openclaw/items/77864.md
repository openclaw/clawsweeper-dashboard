## Summary

Keep open: current `main` still has source paths that can turn Active Memory recall into `status=empty`, and an open linked fix PR explicitly closes this issue but has not merged.

## What This Changes

The issue reports that Active Memory consistently returns `status=empty summaryChars=0` across providers/models instead of producing useful recall output.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78083 or a replacement lands, while leaving the broader user-facing no-useful-memory status work to https://github.com/openclaw/openclaw/issues/79812.

## Reproduction Assessment

Yes, from source inspection: a `memory_search` tool result with only `details.warning` is converted into `debug.warning`, classified as unavailable, and can return terminal `status=empty` before recall completes. I did not run the reporter’s live provider/model setup.

## Solution Assessment

Yes for the narrow linked fix direction: warning-only search output should not be treated as hard unavailability. The broader `empty` versus no-relevant-memory taxonomy has a separate canonical follow-up, so it should not be folded into this issue before the narrow closing PR merges.
