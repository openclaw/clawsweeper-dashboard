## Summary

Keep open: current main still contains the Active Memory paths that can turn warning-only or zero-hit recall into `status=empty`, and an open linked fix PR explicitly closes this issue but has not merged yet.

## What This Changes

The issue reports Active Memory returning `status=empty ... summaryChars=0` for every query, making recall appear nonfunctional across several models.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78083 or a replacement lands; handle the broader no-useful-memory status taxonomy through https://github.com/openclaw/openclaw/issues/79812 and its linked PR.

## Reproduction Assessment

Yes, from source: a warning-only `memory_search` result is converted into `debug.warning`, classified as unavailable, and returned as terminal `status=empty` before recall can complete. I did not live-run the reported model/provider setup.

## Solution Assessment

Yes for the narrow root cause: warning-only search output should not be treated as hard unavailability. The current open PR is the right direction, but it still needs real behavior proof and should not close this issue before merge.
