## Summary

Keep open: current main still contains Active Memory paths that can return terminal `status=empty` before useful recall completes, and an open linked fix PR explicitly closes this issue but has not merged yet.

## What This Changes

The issue reports Active Memory returning `status=empty` with empty summaries for every recall attempt despite memory being enabled.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78083 or a replacement lands; use https://github.com/openclaw/openclaw/issues/79812 for the broader status taxonomy and sticky empty-cache follow-up.

## Reproduction Assessment

Yes, from source: a warning-only `memory_search` result is converted into `debug.warning`, classified as unavailable, and returned as terminal `status=empty` before recall can complete. I did not live-run the reported provider setup.

## Solution Assessment

Yes for the narrow root cause: warning-only search output should not be treated as hard unavailability. The open linked PR is the right implementation direction, but the issue should remain open until that PR or an equivalent fix merges.
