## Summary

Keep open: current main still has a source-proven Active Memory path where warning-only `memory_search` output can end recall as `status=empty`, and an open linked fix PR explicitly targets this issue. The later hotfix comment also overlaps with a broader no-useful-memory status problem now tracked separately at https://github.com/openclaw/openclaw/issues/79812.

## What This Changes

The issue reports Active Memory returning `status=empty ... summaryChars=0` for every recall query across MiniMax and several local models.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78083 for the warning-only terminal memory-search fast-fail, then leave the broader no-useful-memory status cleanup to https://github.com/openclaw/openclaw/issues/79812.

## Reproduction Assessment

Yes, source-reproducible. Current main copies `details.warning` into `debug.warning`, treats that warning as unavailable in the terminal watcher, returns `status=empty`, and aborts the recall subagent before it can synthesize a useful summary.

## Solution Assessment

Yes for the narrow issue path. The maintainable fix belongs in Active Memory’s terminal `memory_search` classification so warning-only results do not short-circuit recall, while the related no-useful-memory UX should remain in the newer dedicated follow-up issue.
