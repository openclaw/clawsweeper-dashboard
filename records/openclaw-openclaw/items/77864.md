## Summary

Keep open: current main still has the Active Memory paths that can turn warning-only or zero-hit recall into `status=empty`, and an open linked fix PR explicitly closes this issue but has not merged yet.

## What This Changes

The issue asks Active Memory to stop returning `status=empty summaryChars=0` for every recall run when recall should either produce useful context or distinguish normal no-memory outcomes from failures.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78083 or an equivalent fix lands, then use https://github.com/openclaw/openclaw/issues/79812 for the broader user-facing status taxonomy cleanup.

## Reproduction Assessment

Yes, from source: a warning-only or zero-hit `memory_search` result can be classified as terminal `status=empty`, aborting recall and logging `summaryChars=0`. I did not live-run the reporter's provider setup in this read-only review.

## Solution Assessment

Yes for the narrow linked PR direction: warning-only search output should not be treated as hard unavailability. The issue should stay open until that PR or an equivalent fix merges, with the broader no-useful-memory status naming handled by the linked follow-up issue.
