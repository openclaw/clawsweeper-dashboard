## Summary

Keep open: current main still has source paths that can produce the reported `status=empty summaryChars=0`, and an open PR explicitly closes this issue but has not merged yet.

## What This Changes

Active Memory should not consistently return an empty payload/status when recall is enabled and memory search is available.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78083 or a replacement lands for the warning-only fast-fail, then handle the broader no-useful-memory status cleanup under https://github.com/openclaw/openclaw/issues/79812.

## Reproduction Assessment

Yes. Source inspection shows warning-only `memory_search` output is converted into `debug.warning`, classified as unavailable, and returned as terminal `status=empty`; current tests also assert nearby empty-result branches.

## Solution Assessment

Yes for the narrow linked direction: warning-only search output should not be treated as hard unavailability. The broader user-visible status taxonomy should remain in the separate canonical follow-up rather than blocking this issue cleanup.
