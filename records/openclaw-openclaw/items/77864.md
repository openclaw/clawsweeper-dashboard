## Summary

Keep open: current main still contains source-proven Active Memory paths that can return `status=empty` with no summary payload, and an open linked fix PR explicitly closes this issue but has not merged.

## What This Changes

The issue asks for Active Memory to stop returning 100% `status=empty ... summaryChars=0` results and to allow useful recall output when memory is available.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78083 or an equivalent replacement lands; use https://github.com/openclaw/openclaw/issues/79812 for the broader no-useful-memory status taxonomy.

## Reproduction Assessment

Yes, from source: a warning-only or zero-hit `memory_search` transcript entry can be classified as terminal `status=empty`, and a `NONE` recall response also maps to `empty`. I did not live-run the reporter's Ubuntu/provider setup.

## Solution Assessment

Yes for the narrow root cause: warning-only search output should not be treated as hard unavailability, and the linked PR is the right implementation path. This issue should remain open until that PR or a replacement merges, with broader status wording handled separately.
