## Summary

Keep open: current main still contains the Active Memory terminal watcher path that can turn warning-only or zero-hit recall into `status=empty summaryChars=0`, and https://github.com/openclaw/openclaw/pull/78083 explicitly closes this issue but remains open with a real-behavior-proof blocker.

## What This Changes

The issue reports Active Memory returning `status=empty` with `summaryChars=0` for every recall attempt across multiple models/providers.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/78083 or a replacement lands, and keep the broader no-useful-memory status taxonomy in https://github.com/openclaw/openclaw/issues/79812.

## Reproduction Assessment

Yes, from source: a warning-only or zero-hit `memory_search` tool result can be classified as terminal `status=empty`, aborting recall before a useful summary is synthesized. I did not run the reporter's live model/provider setup.

## Solution Assessment

Yes for the narrow root cause: advisory warnings should not be treated as hard unavailability, and the open fix PR is aimed at that path. The broader status-label cleanup should stay in the separate follow-up issue rather than expanding this issue's fix boundary.
