## Summary

Keep open: current main still treats warning-only Active Memory search output as terminal empty, and an open closing PR targets this exact issue, so the issue should remain open until that PR merges or is replaced.

## What This Changes

Active Memory should stop returning 100% `status=empty summaryChars=0` when memory search has usable results and only advisory warnings.

## Best Possible Solution

Merge a verified fix through https://github.com/openclaw/openclaw/pull/78083 after real behavior proof and reliable regression coverage, or replace it with the same narrow Active Memory warning-handling change.

## Reproduction Assessment

Yes, from source inspection: current main treats a warning-only `memory_search` result as unavailable and the terminal watcher can abort the recall sub-agent into `status=empty`. I did not run a live Active Memory session in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to treat only `disabled` or hard `error` fields as terminal unavailable, not advisory warnings. The open linked PR implements that direction but still needs proof and test hardening before merge.
