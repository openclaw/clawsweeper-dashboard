## What This Changes

The PR changes cron isolated-agent delivery so fatal structured error payloads skip normal announce completion delivery while preserving error run state, delivery trace state, and delete-after-run cleanup.

## Best Possible Solution

Land the narrow suppression after redacted real cron/channel proof shows the run is recorded as an error and no raw provider payload reaches the configured announce channel.

## Reproduction Assessment

## Solution Assessment

Yes, with a proof caveat. Suppressing only fatal structured error payloads before normal announce delivery is narrow and preserves the separate cron run and failure-notification paths, but it needs real channel proof before merge.
