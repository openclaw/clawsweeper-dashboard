## Summary

Keep open: the patch appears focused and source-reproducible, but this is an unmerged contributor PR with mock-only proof for a user-visible cron delivery/security behavior change and compatibility-sensitive message semantics.

## What This Changes

The PR exposes the fatal structured cron error flag from payload resolution, skips normal completion delivery before `dispatchCronDelivery` for those fatal structured errors, and adds focused isolated-agent regression coverage.

## Best Possible Solution

Land the narrow fatal structured error suppression after redacted real cron/channel proof confirms the run is recorded as an error and raw provider JSON is not delivered as completion announce output.

## Reproduction Assessment

Yes at source level: current main classifies the structured `isError` payload as fatal and still passes its delivery payloads into `dispatchCronDelivery`. I did not run a live Discord or Telegram cron reproduction in this read-only review.

## Solution Assessment

Yes, the proposed code path is a narrow maintainable fix because it gates only fatal structured error payloads before normal completion delivery and leaves controlled failure signals on the existing path. The remaining gap is real behavior proof and maintainer acceptance of the delivery semantics change.
