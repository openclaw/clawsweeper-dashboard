## Summary

Keep open: the patch is a focused fix for a source-reproducible cron disclosure path, but this external PR still needs real cron/channel proof and maintainer acceptance of the failed-run announce behavior change.

## What This Changes

The PR exposes cron's fatal structured-error payload state, skips isolated announce completion dispatch for that state, updates the test harness, and adds a regression test.

## Best Possible Solution

Land the narrow fatal structured-error announce suppression after redacted real cron/channel proof confirms failed runs are recorded as errors, raw provider JSON is not posted, and failure notifications remain the operator alert path.

## Reproduction Assessment

Yes at source level: current main classifies structured isError cron payloads as fatal and still passes the resulting deliveryPayloads into dispatchCronDelivery. I did not run a live Discord or Telegram cron reproduction in this read-only review.

## Solution Assessment

Yes, the proposed code path is narrow because it gates only fatal structured error payloads before normal announce completion delivery and leaves controlled fatal failure signals on the existing path. The remaining blocker is real behavior proof, not a different implementation shape.
