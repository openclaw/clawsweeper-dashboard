Keep open: the issue is recent, has two independent live confirmations on v2026.5.6 and v2026.5.7, and current main still lacks an issue-specific fix or full Telegram dispatch regression for durable direct-session indexing.

## What This Changes

The issue reports that Telegram direct-message sessions with `session.dmScope: "per-channel-peer"` route and reply but are not durably indexed in `sessions.json` or discoverable by session tooling.

## Best Possible Solution

Add a production-path regression for repeated Telegram DMs under `per-channel-peer`, then make durable recording, session lookup, and transcript reuse consistently preserve the route-derived `agent:<id>:telegram:direct:<peer>` key.
