Full review comments:

- **[P1] Require guild proof for Discord channel-only reads:** `extensions/discord/src/actions/runtime.messaging.shared.ts:150-155`
  - body: When `readMessages` supplies only `channelId`, this scan treats any configured guild with no channel allowlist as enough to allow the target. With `groupPolicy: "allowlist"` and `guilds: { "111": {} }`, a channel from another guild can still pass because the code never proves the channel belongs to `111`; fail closed, require `guildId`, or resolve the channel's guild before allowing it.
  - confidence: 0.88
- **[P1] Preserve open-policy Slack reads for unlisted channels:** `extensions/slack/src/action-runtime.ts:188`
  - body: This rejects every unlisted target whenever `channels` has any entries, even when `groupPolicy` is `open`. Current Slack ingress explicitly allows unlisted channels under open policy and only blocks explicit denied matches, so merging this would break reads for existing open-policy configs that use channel entries for overrides.
  - confidence: 0.94
