## Summary

Keep open. Current main still treats the requested bootstrap profile knobs as global `agents.defaults` settings: the per-agent config type/schema do not accept them, runtime resolution reads defaults only, and the Control UI profile presets still patch global defaults. The related #69988 work is useful precedent for per-agent config merging, but it is scoped to compaction and has not landed in the central per-agent config schema on this checkout.

## What This Changes

Request per-agent overrides for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` so heterogeneous agents can use different bootstrap profiles.

## Best Possible Solution

Add direct optional per-agent overrides under `agents.list[]` with fallback to `agents.defaults`, thread effective values by session agent id through bootstrap/context assembly and reports, and update schema, generated docs, Control UI, user docs, tests, and changelog.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a config using the requested `agents.list[]` fields would be rejected by the strict per-agent schema, and current runtime resolvers ignore agent id and read only `agents.defaults`.

## Solution Assessment

Yes for the direct-override variant. It is the narrowest maintainable fix and matches the existing per-agent `contextLimits` inheritance precedent; reusable named profiles should stay out unless maintainers explicitly choose that larger design.
