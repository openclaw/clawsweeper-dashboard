## Summary

Keep open. Current main still treats the requested bootstrap profile knobs as global `agents.defaults` settings: per-agent config/schema/docs do not expose them, runtime bootstrap/context resolution reads defaults only, and Control UI presets still patch global defaults.

## What This Changes

Request per-agent overrides or named profiles for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` instead of only global `agents.defaults` profile settings.

## Best Possible Solution

Add optional direct per-agent overrides under `agents.list[]` with fallback to `agents.defaults`, thread the effective values by session agent id through bootstrap/context assembly and reports, and update schema, generated docs, Control UI, user docs, tests, and changelog.

## Reproduction Assessment

Yes for source-level verification of the gap: the requested `agents.list[]` fields are absent from the type/schema and current runtime resolvers read only `agents.defaults`. I did not run validation because this review is read-only, but the strict schema and defaults-only resolvers provide a high-confidence static reproduction path.

## Solution Assessment

Unclear pending maintainer product choice. Direct per-agent overrides look like the narrowest maintainable path, while reusable named profiles are a larger config design that should not be inferred without explicit approval.
