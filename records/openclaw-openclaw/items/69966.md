## Summary

Keep open. Current main still exposes `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` only through global `agents.defaults`; the per-agent type/schema, runtime resolvers, docs, and Control UI profile presets do not provide the requested per-agent contract, and adjacent bootstrap-tier or config-patch reports do not settle this feature request.

## What This Changes

The issue requests per-agent overrides or named profiles for bootstrap/context profile settings that are currently controlled through global `agents.defaults` values.

## Best Possible Solution

Define the public contract for bootstrap/context profile scope, then wire the chosen per-agent effective values through types, zod schema, config patching, runtime bootstrap/context resolution, Control UI, docs, generated metadata, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path for the gap: current main keeps the requested knobs under `agents.defaults`, rejects or ignores them as per-agent contract, and the runtime resolvers read defaults only.

## Solution Assessment

Unclear until maintainers choose the API. Direct optional per-agent overrides with fallback to `agents.defaults` look narrower than named reusable profiles, but the final design should coordinate with bootstrap-tier and config-patch work.
