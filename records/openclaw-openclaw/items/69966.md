## Summary

Keep open. Current main still treats the requested bootstrap profile knobs as global defaults; the newer open #76054 work covers only `contextInjection`, so the broader per-agent `bootstrapMaxChars` and `bootstrapTotalMaxChars` request remains unresolved.

## What This Changes

Request per-agent overrides for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` so heterogeneous agents can use different bootstrap profiles.

## Best Possible Solution

Add direct optional per-agent overrides under `agents.list[]` for all three bootstrap profile knobs, fall back to `agents.defaults`, and update schema, runtime resolution, Control UI, docs, generated config metadata, tests, and changelog.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a config using the requested `agents.list[]` fields would be rejected by the strict per-agent schema, and current runtime resolvers read only `agents.defaults`.

## Solution Assessment

Yes for the direct-override design, but the active #76054 path is only a partial solution. The safest maintainer path is to land or expand that context-injection work, then add the per-agent bootstrap budget fields with the same resolver pattern.
