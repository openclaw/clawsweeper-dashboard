## Summary

Keep open. Current main still exposes the requested bootstrap/context knobs only as global `agents.defaults` settings: per-agent types/schema/resolvers/docs/UI do not provide `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars`, and related bootstrap-tier or config-patch work does not settle this per-agent contract.

## What This Changes

Request per-agent bootstrap profile overrides for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` so heterogeneous agent teams can use different prompt-context budgets.

## Best Possible Solution

Choose the public config contract, likely direct optional per-agent overrides with fallback to `agents.defaults`, then wire effective values by agent id through schema/types, runtime bootstrap/context assembly, Control UI, docs, generated metadata, and focused tests.

## Reproduction Assessment

Yes for source-level verification of the gap: current `agents.list[]` type/schema lack the requested fields, strict validation would reject them, and runtime resolvers read only `agents.defaults`. I did not run validation or tests because this review is read-only.

## Solution Assessment

Unclear pending maintainer product choice. Direct per-agent overrides look narrower than reusable named profiles, but the final shape should be reconciled with bootstrap-tier work and Control UI semantics before implementation.
