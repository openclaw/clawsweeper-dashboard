## Summary

## What This Changes

The PR changes `applyEditsToNormalizedContent` so fuzzy matches are located in normalized text but spliced into the original LF-normalized content, with regression tests for unrelated-line preservation and Unicode/NFKC edge cases.

## Best Possible Solution

Land this shared edit-diff fix after maintainer review of the mapper, then close https://github.com/openclaw/openclaw/issues/89994.

## Reproduction Assessment

## Solution Assessment

Yes, this is the right layer: execution and preview both flow through `applyEditsToNormalizedContent`, so a shared helper fix keeps writes and model/TUI diffs aligned. A caller-only workaround would duplicate behavior and risk drift.
