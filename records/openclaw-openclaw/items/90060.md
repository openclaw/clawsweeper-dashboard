## Summary

Keep this PR open for maintainer review and normal merge gating. The patch targets a real core edit-tool corruption bug, the current main path still normalizes the full file during fuzzy edits, and the PR fixes the shared helper used by both write execution and preview rendering with focused regression coverage.

## What This Changes

The PR changes `applyEditsToNormalizedContent` to keep the original LF-normalized content as the splice and diff base for fuzzy edits, map fuzzy-normalized offsets back to original offsets, and add regression tests for whitespace, Unicode, and NFKC mapping cases.

## Best Possible Solution

## Solution Assessment
