## What This Changes

This PR reconstructs hosted custom-scheme origins from protocol and host when `URL.origin` is `null` and adds unit cases for exact `app://`, `electron://`, and `tauri://` allowlist matches.

## Best Possible Solution

Land this or an equivalent current-main parser fix after redacted Gateway proof and clean relevant checks, preserving exact allowlist semantics while rejecting literal, malformed, empty-host, and unlisted origins.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.8 compare configured entries against `URL.origin`, and a Node v24.16.0 probe shows hosted custom schemes serialize that origin as `null` while retaining host data.

## Solution Assessment

Yes for the parser layer, with merge-readiness caveats. The PR normalizes only incoming hosted opaque origins and preserves exact configured allowlist strings, which is narrower than changing wildcard, config, or host-fallback policy.
