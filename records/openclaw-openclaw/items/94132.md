## What This Changes

The PR reconstructs hosted opaque custom-scheme origins in the Gateway origin parser and adds table tests for exact `app://`, `electron://`, and `tauri://` allowlist matches.

## Best Possible Solution

Land this or an equivalent current-main parser fix after redacted Gateway diagnostics and clean relevant CI, preserving exact allowlist semantics while still rejecting literal, malformed, and empty-host opaque origins.

## Reproduction Assessment

## Solution Assessment

Yes for the parser layer, with merge-readiness caveats. Reconstructing only hosted opaque origins when protocol and host exist is narrower than changing wildcard, config, or Host-header fallback policy.
