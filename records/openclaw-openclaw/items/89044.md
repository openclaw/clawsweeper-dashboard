## Summary

This PR is still useful because current main does not group the sidebar Recent list, but the latest patch is not merge-ready: it still adds a hard-coded sidebar channel label source instead of using the existing channel snapshot metadata used by sibling Control UI surfaces.

## What This Changes

The PR groups Control UI sidebar recent sessions by resolved channel, adds sidebar group styling, widens session-row metadata typing, adds render coverage, and relaxes one gateway test mock assertion.

## Best Possible Solution

Keep the grouped top-five sidebar behavior, but resolve group headings from state.channelsSnapshot.channelMeta first, then channelLabels, and only then a derived fallback, with a custom-channel regression test.

## Reproduction Assessment

Yes for the PR-introduced defect: a custom channel whose catalog label differs from the derived id will use the new local formatter because the helper never reads state.channelsSnapshot. I verified that from the patch and sibling label-resolution code rather than a browser run.

## Solution Assessment

No. Grouping the already-selected five sessions is a reasonable UI shape, but the best fix is to reuse the existing channel metadata label source before falling back to derived labels.
