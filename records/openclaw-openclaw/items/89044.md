## Best Possible Solution

Keep the grouped top-five sidebar behavior, but resolve group labels from state.channelsSnapshot.channelMeta first, then channelLabels, and only then use a derived fallback with a custom/external channel regression.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection shows the new group label path reads only a local map and formatter while current UI state and sibling views use channel snapshot labels. A custom/external channel row with a catalog label would reproduce the mismatch.

## Solution Assessment

No as submitted. Grouping the already-selected five sessions is a reasonable UI direction, but the maintainable fix should reuse the existing channel label contract before falling back to derived labels.
