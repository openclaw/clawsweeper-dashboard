## Summary

Keep open: the grouped sidebar idea is useful and has real behavior proof, but the current head still adds a parallel hard-coded channel-label source instead of using the existing plugin-aware channel metadata, so custom or external channel headings can drift from catalog labels.

## What This Changes

The PR groups the Control UI sidebar's existing five most recent sessions by resolved channel, extends session row types with channel/origin fields, adds group styling, and adds a render regression test.

## Best Possible Solution

Keep the grouping behavior, but resolve group headings from state.channelsSnapshot.channelMeta/channelLabels first and only derive a fallback when the catalog has no label.

## Reproduction Assessment

Yes for the PR-introduced defect: a session for a custom channel with channelMeta/channelLabels would still use the new local formatter because the added sidebar code never reads the channel snapshot. I verified this from the diff and existing label-resolution callers, rather than by running a browser repro.

## Solution Assessment

No. Grouping the already-selected top five sessions is an acceptable shape, but the best fix is to reuse the existing channel metadata source instead of adding a parallel hard-coded map.
