## Summary

Keep open: current main still renders the sidebar recent sessions as a flat list, and this PR is the active implementation candidate, but the latest head still introduces a sidebar-only channel label source that can disagree with the existing channel catalog labels used elsewhere in the Control UI.

## What This Changes

The PR groups the Control UI sidebar's already-selected top five recent sessions by resolved channel, adds sidebar group styling/types, and adds render coverage plus a small gateway test mock relaxation.

## Best Possible Solution

Keep the grouped top-five sidebar behavior, but resolve group labels from `state.channelsSnapshot.channelMeta` first, then `channelLabels`, and only then use a derived fallback, with a custom-channel regression test.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection shows the new group label path reads only the local map/derived formatter and never consults `state.channelsSnapshot`, while sibling views use catalog metadata.

## Solution Assessment

No as submitted. Grouping the already-selected five sessions is a reasonable UI direction, but the maintainable fix should reuse the existing channel label contract before falling back to derived labels.
