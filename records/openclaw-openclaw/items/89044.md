## What This Changes

The PR groups the Control UI sidebar's existing five recent sessions by resolved channel, extends UI session-row typing for channel/origin fields, adds group styling, and adds render coverage.

## Best Possible Solution

Keep the grouping behavior, but resolve group headings from `state.channelsSnapshot.channelMeta` or `channelLabels` first and only derive a fallback when metadata is unavailable.

## Reproduction Assessment

Yes for the PR-introduced defect: a session whose channel id has a catalog label different from the derived id will still use the new local formatter because the added sidebar code never reads `state.channelsSnapshot`. I verified that from the live PR diff and current sibling label-resolution code rather than by running a browser repro.

## Solution Assessment

No. Grouping the already-selected five sessions is a reasonable UI shape, but the best fix is to reuse the existing channel metadata label source instead of adding a parallel hard-coded map.
