## Summary

Keep open: current main still has the Telegram split-reasoning path that can rotate the reasoning draft with `forceNewMessage()` before preserving or deleting the previous preview id, and the linked fix PRs are closed unmerged.

## What This Changes

The issue requests that Telegram `/reasoning stream` keep one editable reasoning preview and delete it after final delivery instead of leaving multiple stale `sendMessage` previews.

## Best Possible Solution

Repair the Telegram reasoning split lifecycle so the previous reasoning preview is stopped, preserved, or cleared before starting the next preview, then add a focused regression test and Telegram behavior proof.

## Reproduction Assessment

Yes, as a source-level reproduction: trigger a reasoning segment, `onReasoningEnd`, then a second reasoning segment, and current main rotates with `forceNewMessage()` before preserving or deleting the prior preview id. I did not run a live Telegram DM in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow Telegram draft-lifecycle repair using the existing lane-rotation contract or an equivalent clear-before-rotate path, not a new config option or separate send path.
