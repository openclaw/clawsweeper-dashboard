## What This Changes

The issue reports that Telegram `/reasoning stream` creates multiple stale reasoning preview messages instead of editing and deleting one live preview.

## Best Possible Solution

Repair the split-reasoning Telegram draft lifecycle so the prior reasoning preview is deleted or archived before a new preview is created, then cover the two-segment reasoning case with a focused regression test and live Telegram proof before landing.

## Solution Assessment
