## Best Possible Solution

Preserve runtime `toolsAllow` fail-closed behavior, and if raw promptMode/modelRun still needs an exemption, treat raw runs as disabled only for inherited allowlist filtering with regression coverage and real behavior proof.

## Reproduction Assessment

## Solution Assessment

No. The safer fix is to reuse the disabled-tool inherited-allowlist filtering for raw runs while preserving enforced runtime `toolsAllow`; bypassing the guard before that distinction removes the diagnostic the current contract intentionally keeps.
