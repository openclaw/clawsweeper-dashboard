## Summary

Keep this PR open for maintainer handling, but the proposed one-line default change is not mergeable as-is because current source and docs only support `http/protobuf` for the bundled diagnostics exporter.

## What This Changes

This PR changes the Docker Compose fallback for `OTEL_EXPORTER_OTLP_PROTOCOL` from `http/protobuf` to `http/json`.

## Best Possible Solution

Do not merge the default flip; resolve #77646 through a maintained override/preservation path or by adding `http/json` support end-to-end in exporter code, schema, tests, and docs.

## Reproduction Assessment

Yes for the PR regression by source inspection: Compose would inject `http/json` when the env var is omitted, and `diagnostics-otel` returns `unsupported_protocol` for anything except `http/protobuf`. I did not run a live Docker collector in this read-only review.

## Solution Assessment

No. The one-line default change is not the best fix because it changes the default to a protocol the bundled exporter, schema, and docs do not support; the safer path needs maintainer direction on #77646.
