## Best Possible Solution

Keep the canonical issue open for a deliberate durable protocol override or full `http/json` exporter implementation, and close this unsafe default-only branch.

Yes. Source inspection is enough for the PR regression: the diff makes Compose inject `http/json` by default when the env var is missing, while the bundled exporter returns `unsupported_protocol` for any protocol except `http/protobuf`.

## Solution Assessment

No. The one-line default flip is not the best fix because it changes Docker defaults to a protocol the bundled exporter, schema, and docs do not support; the safer solution belongs in the canonical issue as a durable override or end-to-end support change.
