## Summary

Keep open: the PR usefully narrows one proposed fix for #77646, but the patch conflicts with current `main` because Docker would default the bundled diagnostics OTel exporter to a protocol it explicitly rejects.

## What This Changes

This PR changes `docker-compose.yml` so `OTEL_EXPORTER_OTLP_PROTOCOL` falls back to `http/json` instead of `http/protobuf`.

## Best Possible Solution

Keep the Compose default aligned with the currently supported protobuf exporter unless OpenClaw adds `http/json` support end-to-end or a targeted operator override path.

## Reproduction Assessment

Yes, source-reproducible for the PR regression: with this diff, Docker would inject `OTEL_EXPORTER_OTLP_PROTOCOL=http/json`, and the diagnostics OTel service returns early as `unsupported_protocol` unless the value is `http/protobuf`.

## Solution Assessment

No. The one-line default change is not the best fix because it makes the bundled exporter default to an unsupported protocol; the safer path is either full `http/json` support or a narrower override-preservation fix for #77646.

## Review Findings
