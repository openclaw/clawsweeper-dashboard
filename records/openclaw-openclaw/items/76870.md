## Summary

Keep open: the PR targets the right generic guarded-fetch layer for the IPv4-only regression, but the current patch overwrites explicit family-policy overrides and bypasses the existing WSL2 exception, so it needs repair before merge.

## What This Changes

The PR adds default Happy Eyeballs `autoSelectFamily` connect options to HTTP/1-only Undici per-request dispatchers and updates one pinned-dispatcher test.

## Best Possible Solution

Land a generic guarded-dispatcher fix that adds Happy Eyeballs defaults only when no explicit family policy exists, uses the same WSL2-aware decision as the global dispatcher, keeps SSRF pinning/proxy routing intact, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the PR regression: the patched spread order is enough to show explicit `connect.autoSelectFamily` values are overwritten, and current tests cover preservation of those hints. I did not live-reproduce the IPv4-only host failure.

## Solution Assessment

No. The generic dispatcher boundary is appropriate, but the implementation should preserve explicit transport overrides and reuse the existing WSL2-aware family decision before merge.
