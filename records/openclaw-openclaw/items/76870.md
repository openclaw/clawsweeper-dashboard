## Summary

Keep this PR open: it targets a valid current-main regression, but the patch still needs changes because it diverges from the existing WSL2-aware family-selection policy, can overwrite explicit transport hints, and lacks the required changelog entry.

## What This Changes

The PR adds Node `autoSelectFamily` connect options to HTTP/1 undici dispatchers used by guarded fetches and updates the pinned-dispatcher test.

## Best Possible Solution

Share the existing global `autoSelectFamily` decision with per-request dispatchers, preserve explicit caller overrides, add focused regression coverage for WSL2 and override behavior, and add the user-facing Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path where pinned guarded dispatchers install only the custom `lookup` and the test asserts no `autoSelectFamily`, while #76857 provides concrete IPv4-only/AAAA network steps and an IPv4 hosts-file workaround.

## Solution Assessment

No. Adding Happy Eyeballs to guarded per-request dispatchers is the right direction, but this implementation should reuse the existing WSL2-aware policy and avoid clobbering explicit connect settings before merge.
