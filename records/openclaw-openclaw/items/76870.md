## Summary

Keep this PR open: it targets a valid provider transport regression, but the current patch still diverges from the existing WSL2 family-selection policy, overwrites explicit caller transport overrides, and lacks the required changelog entry.

## What This Changes

The PR adds `autoSelectFamily` connect options to per-request HTTP/1 undici dispatchers and updates an SSRF dispatcher test for IPv4 fallback when IPv6 egress is broken.

## Best Possible Solution

Share the existing global `autoSelectFamily` decision with per-request HTTP/1 dispatchers, preserve explicit caller overrides, cover the policy and opt-out cases in tests, and add the Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes: current `main` has a high-confidence source path where provider requests use `fetchWithSsrFGuard`, `createPinnedDispatcher` installs a pinned lookup, and the existing test asserts no inherited `autoSelectFamily`. I did not execute the live IPv4-only network reproduction.

## Solution Assessment

No: adding Happy Eyeballs to per-request dispatchers is the right direction, but this patch should reuse the existing global WSL2-aware policy and only fill `autoSelectFamily` keys that callers did not explicitly set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reuse the global family-selection policy:** `src/infra/net/undici-runtime.ts:43`
  - body: The new helper calls `net.getDefaultAutoSelectFamily()` directly, but `undici-global-dispatcher.ts` intentionally maps a true system default to false on WSL2. A pinned per-request dispatcher would therefore use a different family policy from the global dispatcher and can reintroduce the WSL2 `fetch failed` path; route through the same resolver or apply the same WSL2 branch.
  - confidence: 0.9
- **[P2] Preserve explicit connect overrides:** `src/infra/net/undici-runtime.ts:101`
  - body: The merge spreads `...autoSelectConnect` after existing `connect` options, so a caller that intentionally sets `connect.autoSelectFamily: false` is silently changed back to the system default. Only fill `autoSelectFamily` and `autoSelectFamilyAttemptTimeout` when those keys are absent.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `CHANGELOG.md:32`
  - body: This is a user-facing provider transport regression fix, but the PR does not add an Unreleased/Fixes entry. Add a single-line `CHANGELOG.md` entry before merge.
  - confidence: 0.82
