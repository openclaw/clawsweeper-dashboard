## Summary

Keep this PR open: it targets a real current-main regression path, but the patch is not ready to land because the new per-request family-selection helper diverges from existing WSL2 policy and can overwrite explicit transport overrides.

## What This Changes

The PR adds Node `autoSelectFamily` connect options to per-request HTTP/1 undici dispatchers and updates one SSRF dispatcher test for IPv4 fallback on broken IPv6 egress.

## Best Possible Solution

Share the existing global autoSelectFamily decision with per-request dispatchers, preserve explicit caller overrides, add focused regression coverage, and include the required user-facing changelog entry.

## Reproduction Assessment

Yes: current main has a source-level reproduction path where `createPinnedDispatcher` installs only the pinned `lookup` and the test asserts no `autoSelectFamily` on that dispatcher. The linked reports also provide concrete IPv4-only/AAAA network steps, though I did not run the live network scenario.

## Solution Assessment

No: adding Happy Eyeballs to per-request dispatchers is the right direction, but this implementation should reuse the global resolver and avoid clobbering explicit transport options before it lands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reuse the global family-selection policy:** `src/infra/net/undici-runtime.ts:42-45`
  - body: This helper calls `net.getDefaultAutoSelectFamily()` directly, while the existing global dispatcher intentionally turns that value off on WSL2 to avoid IPv6 `fetch failed` behavior. Per-request dispatchers created here will diverge from the global policy and can reintroduce the WSL2 failure; share the existing resolver or apply the same WSL2 branch.
  - confidence: 0.9
- **[P2] Preserve explicit connect overrides:** `src/infra/net/undici-runtime.ts:98-100`
  - body: The new merge spreads `...autoSelectConnect` after the caller's existing `connect` object, so an explicit `connect.autoSelectFamily: false` is replaced by the system default. That breaks operator/provider transport hints; only fill these keys when the caller did not already set them.
  - confidence: 0.88

  - body: This is a user-facing provider transport regression fix, but the PR does not update `CHANGELOG.md`. Add a single-line entry under Unreleased/Fixes before landing.
  - confidence: 0.82
