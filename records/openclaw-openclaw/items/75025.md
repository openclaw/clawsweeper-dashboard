## Summary

Keep open. The PR targets a real current-main stale timestamp bug and the shared helper is the right boundary, but the proposed matcher also rewrites user-authored reminder lines that start with `Current time:` and the user-facing fix is missing the required changelog entry.

## What This Changes

The branch replaces the helper's existing `Current time:` early return with a multiline replacement/collapse regex and adds a colocated Vitest regression suite.

## Best Possible Solution

Keep the helper-level refresh approach, but restrict replacement to OpenClaw-generated timestamp lines, preserve user-authored `Current time:` text, cover that case, and add the Unreleased Fixes entry.

## Reproduction Assessment

Yes. The original bug reproduces by feeding helper output with an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression reproduces with cron/reminder text containing a user-authored line such as `Current time: check the server clock`.

## Solution Assessment

No. Refreshing the shared helper is the narrow maintainable direction, but the current regex must be narrowed to the generated timestamp format or a backward-compatible sentinel before merge.
