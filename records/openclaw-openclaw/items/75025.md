## Summary

Keep this PR open: the helper-level fix is the right narrow direction, but the latest matcher does not match the actual `Current time:` line emitted by current main, so the stale timestamp bug remains and the new test expectation is inconsistent with the helper output.

## What This Changes

The PR replaces `appendCronStyleCurrentTimeLine`'s stale `Current time:` early return with regex-based refresh/collapse logic and adds colocated tests.

## Best Possible Solution

Keep the shared-helper refresh approach, but match the actual `resolveCronStyleNow` output or add a backward-compatible sentinel, preserve user-authored `Current time:` lines, and record the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Current main is source-reproducible by feeding helper output with an older `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the early return preserves the stale line. The PR regression is also source-reproducible because its numeric matcher returns false for the helper's normal natural-language line.

## Solution Assessment

No. Refreshing the shared helper is the narrow maintainable fix point, but this patch's regex misses the actual emitted format and the branch still needs the required Unreleased Fixes entry.

## Review Findings
