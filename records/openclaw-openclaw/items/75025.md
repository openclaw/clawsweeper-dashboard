## Summary

Keep open: current main still has the stale-time bug, and the latest PR revision does not actually match the helper's normal emitted timestamp format, so repeated helper calls would append another line instead of refreshing the stale one.

## What This Changes

The branch replaces the shared `Current time:` early-return with regex-based refresh/collapse logic and adds colocated Vitest coverage for heartbeat/cron prompt timestamps.

## Best Possible Solution

Keep the helper-level refresh approach, but match the actual `resolveCronStyleNow` output or add a backward-compatible sentinel, preserve user-authored `Current time:` lines, cover feed-forward helper output, and add the Unreleased Fixes note.

## Reproduction Assessment

Yes. The current-main bug is source-reproducible by feeding an older helper-emitted `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is source-reproducible because its matcher does not match the helper's normal emitted line.

## Solution Assessment

No. Refreshing the shared helper is the right narrow fix point, but the latest regex matches a numeric local-time format that the helper does not emit in normal runtime, so the patch still misses the intended stale-line refresh.
