## Summary

Keep open. Current main still has the stale `Current time:` early-return bug and this PR targets the right shared helper for #44993, but the proposed regex is too broad and can replace user-authored reminder/workspace content; the user-facing fix also needs a changelog entry before merge.

## What This Changes

This PR changes `appendCronStyleCurrentTimeLine` to replace or collapse existing `Current time:` lines and adds colocated Vitest coverage for empty input, appending, refreshing, and duplicate collapse.

## Best Possible Solution

Keep the helper-level refresh design, but narrow replacement to OpenClaw-generated timestamp lines or a compatible internal marker, preserve user-authored `Current time:` text, add a preservation regression test, and leave ISO-format/cache-placement follow-ups to #55935 and #71973.

## Reproduction Assessment

Yes. The original bug is reproducible by passing a helper output containing an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is reproducible by passing reminder text with a user-authored line beginning `Current time:` through the new matcher.

## Solution Assessment

No. The shared-helper boundary is the right narrow place to fix #44993, but the current regex is not the best implementation because it cannot distinguish generated timestamp lines from user-authored content.
