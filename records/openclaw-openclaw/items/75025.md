## Summary

Keep open: the bug fix is aimed at the right shared helper, but the latest PR head still matches the old slash-suffixed one-line timestamp while current main emits a two-line `Current time` / `Reference UTC` block, so stale current-main blocks would be left in place and a fresh block appended.

## What This Changes

The PR updates `appendCronStyleCurrentTimeLine` to refresh helper-emitted `Current time` lines and adds a new colocated regression suite for stale heartbeat/cron prompt timestamps.

## Best Possible Solution

Keep the shared-helper refresh approach, but update the matcher and tests to replace the current two-line helper-emitted block while preserving user-authored `Current time:` text.

## Reproduction Assessment

Yes. Source inspection plus a regex probe show the PR matcher returns false for a current-main two-line helper block, so a repeated helper call would append a fresh block and leave the stale one present.

## Solution Assessment

No. The shared helper is the narrowest maintainable fix point, but this implementation must match the current two-line timestamp contract instead of the obsolete single-line form.
