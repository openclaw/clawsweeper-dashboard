## Summary

Keep open: the PR targets a valid stale heartbeat/current-time bug, but the latest matcher still misses the helper's emitted natural-language `Current time:` line, so the intended refresh path remains broken.

## What This Changes

The PR changes the shared current-time helper to refresh or collapse existing injected `Current time:` lines and adds colocated Vitest coverage.

## Best Possible Solution

Keep the helper-level refresh approach, but match the actual `resolveCronStyleNow` output or add a stable sentinel, preserve user-authored `Current time:` text, cover helper feed-forward plus caller surfaces, and add an Unreleased Fixes entry.

## Reproduction Assessment

Yes. The current-main bug is source-reproducible by feeding an older helper-emitted `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is source-reproducible because its numeric matcher returns false for the helper's normal natural-language output.

## Solution Assessment

No. Refreshing the shared helper is the right narrow fix point, but the current PR matcher targets a line shape the helper does not normally emit, so it still misses the stale-line replacement path.
