## Summary

Keep open: this is a focused bug-fix PR with sufficient terminal proof, but the current patch still targets the older one-line helper format while current main emits a two-line `Current time` / `Reference UTC` block.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to refresh or collapse helper-emitted `Current time` lines and adds regression tests for stale heartbeat/cron prompt timestamps.

## Best Possible Solution

Keep the shared-helper refresh approach, but update the matcher and tests to refresh the current two-line helper-emitted time block while preserving user-authored `Current time:` text.

## Reproduction Assessment

Yes. The issue is source-reproducible because current main returns unchanged for prompts containing `Current time:`, and the heartbeat runner passes a fresh `startedAt` through that helper on each run.

## Solution Assessment

No. The shared helper is the right fix point, but this implementation is not yet the best working solution because its regex targets an obsolete single-line helper output instead of current main's two-line block.
