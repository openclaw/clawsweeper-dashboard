## Summary

Keep open. Current main still has the stale timestamp bug and this PR targets the right shared helper, but the proposed matcher introduces a concrete regression by rewriting user-authored lines that start with `Current time:`; the branch also lacks the required user-facing changelog entry.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to replace or collapse existing `Current time:` lines instead of returning early, and adds colocated Vitest coverage for empty input, append, refresh, and duplicate-collapse cases.

## Best Possible Solution

Keep the shared-helper refresh approach, but constrain replacement to OpenClaw-generated timestamp lines or a backward-compatible internal marker, preserve user-authored `Current time:` content, add a preservation regression test, and include the required Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. The original bug is reproducible by passing helper output with an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is reproducible by passing cron reminder text containing `Current time: check server clock` through the proposed regex.

## Solution Assessment

No. The helper-level fix is the right narrow boundary, but the current implementation is not the best fix because it cannot distinguish generated timestamp lines from user-authored prompt content and it omits required changelog coverage.
