## Summary

Keep open: current main still has the stale-time bug, and the latest PR head still misses the normal helper-emitted timestamp because its matcher expects a numeric local-time prefix while the helper emits a formatted natural-language time.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to refresh or collapse existing `Current time:` lines and adds colocated regression tests for the helper.

## Best Possible Solution

Keep the helper-level refresh point, but match the actual `resolveCronStyleNow` output or add a backward-compatible injected-line sentinel, preserve user-authored `Current time:` text, update regression coverage, and add the Unreleased Fixes entry.

## Reproduction Assessment

Yes. Current main is source-reproducible by feeding a helper-emitted older `Current time:` line back through `appendCronStyleCurrentTimeLine`, and the PR regression is source-reproducible because its regex does not match that emitted line.

## Solution Assessment

No. Refreshing the shared helper is the right narrow solution, but the latest matcher is inconsistent with the helper's current output format, so it still leaves stale lines in normal heartbeat/session-reset use.
