## Summary

Keep open. The PR targets a real stale heartbeat/cron timestamp bug, and a maintainer comment reports the focused test plus a CI rerun passed on the exact head, but the current matcher is broader than the generated timestamp format and can overwrite user-authored cron reminder content; the user-facing fix also needs a changelog entry before merge.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to replace/collapse line-bound `Current time:` entries on repeated helper calls and adds a colocated regression test suite.

## Best Possible Solution

Keep the helper-level refresh direction, but only refresh OpenClaw-generated timestamp lines, preserve user-authored `Current time:` content, and document the user-facing fix in Unreleased Fixes.

## Reproduction Assessment

Yes. The original bug reproduces by feeding helper output with an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression reproduces with a cron reminder line such as `Current time: check the server clock`.

## Solution Assessment

No. Refreshing the shared helper is the narrow maintainable solution, but the current regex needs to be narrowed to the generated timestamp format or replaced with a backward-compatible sentinel before merge.
