## Summary

Keep open. The PR addresses a real stale heartbeat/cron timestamp bug that is still present on current main, but the proposed matcher is too broad and can rewrite user-authored reminder text; the PR also lacks the required changelog entry for a user-facing fix.

## What This Changes

The PR changes the shared current-time prompt helper to refresh or collapse existing `Current time:` lines and adds colocated regression tests for empty input, append, refresh, and duplicate-collapse cases.

## Best Possible Solution

Keep the helper-level refresh approach, but restrict replacement to OpenClaw-generated timestamp lines or add a backward-compatible internal marker, preserve arbitrary user-authored `Current time:` lines, add preservation coverage, and include the required Unreleased Fixes changelog entry while leaving ISO formatting and prompt-cache optimization to #55935 and #71973.

## Reproduction Assessment

Yes. The original bug is reproducible by feeding helper output containing an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is reproducible with a cron reminder line such as `Current time: check the server clock`.

## Solution Assessment

No. The shared helper is the right narrow boundary, but this PR's line-bound regex should be constrained to generated timestamp lines or a compatible marker before merge.
