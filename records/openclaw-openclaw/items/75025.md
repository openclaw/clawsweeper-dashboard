## Summary

Keep open. The PR targets a valid current-main bug in the heartbeat/cron current-time helper, but the proposed matcher is still too broad because it rewrites user-authored cron reminder lines that begin with `Current time:`, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to refresh or collapse existing `Current time:` lines and adds colocated tests for empty input, append, refresh, and duplicate-collapse cases.

## Best Possible Solution

Keep the helper-level refresh approach, but match only OpenClaw-generated timestamp lines or add a backward-compatible internal marker, preserve user-authored `Current time:` content, cover that preservation case, and add the required changelog entry while leaving broader ISO/cache changes to the separate follow-up issues.

## Reproduction Assessment

Yes. The original bug is reproducible by feeding helper output containing an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is reproducible by passing prompt text with a user-authored line such as `Current time: check server clock`.

## Solution Assessment

No. The shared helper is the right narrow boundary, but this PR's line-bound regex is not selective enough and should be constrained to generated timestamp lines or a sentinel-based format before merge.
