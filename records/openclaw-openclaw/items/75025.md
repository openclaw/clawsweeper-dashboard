## Summary

Keep open. The branch targets a real current-main bug: the helper still returns unchanged when a prompt contains `Current time:`, while heartbeat passes a fresh `startedAt`. The proposed matcher is too broad for verbatim cron reminder text, its cleanup can alter unrelated prompt spacing, and the user-facing fix still needs a changelog entry before merge.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to refresh or collapse existing `Current time:` lines and adds colocated helper regression tests.

## Best Possible Solution

Keep the helper-level refresh approach, but match only OpenClaw-generated timestamp lines or introduce a backward-compatible internal marker for future lines, preserve all user-authored prompt text and spacing, add preservation/regression coverage, and add the required Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. The original bug is reproducible by feeding helper output containing an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is reproducible with cron reminder text containing `Current time: check server clock`.

## Solution Assessment

No. The shared helper is the right narrow boundary, but the current regex and global spacing cleanup are broader than the generated timestamp contract; a generated-format matcher plus focused tests is the safer maintainable fix.
