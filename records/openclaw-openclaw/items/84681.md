## What This Changes

The PR splits Codex app-server dynamic tools into durable registered schemas and current-turn available schemas so heartbeat turns can reuse threads without advertising or executing heartbeat tools on normal turns.

## Best Possible Solution

Land the registered/available schema split after maintainer review with current-head redacted live logs proving normal -> heartbeat_respond -> normal thread reuse and normal-turn heartbeat denial.

## Reproduction Assessment

Yes. Source inspection shows current main fingerprints the per-turn dynamic tool specs while heartbeat is only enabled on heartbeat-triggered runs, so a normal -> heartbeat transition can change the dynamic-tool catalog; I did not run tests in this read-only review.

## Solution Assessment

Yes. The registered/available schema split is the narrow maintainable fix because it stabilizes thread registration while keeping prompts, context-engine inputs, trajectory context, and execution scoped to current-turn tools.
