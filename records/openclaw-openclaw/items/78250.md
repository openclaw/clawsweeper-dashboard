## Summary

Keep open: current main still lacks Feishu/Lark inbound read-receipt support, config metadata, and docs; this is bundled Feishu plugin feature work that needs maintainer API/product direction before implementation.

## What This Changes

Request adding Feishu/Lark DM read receipts, preferably default-on and/or controlled by `channels.feishu.sendReadReceipts`.

## Best Possible Solution

Implement account-aware Feishu read receipts inside the Feishu plugin after confirming the API/scopes, with schema/manifest/docs updates and focused Feishu tests; keep core unchanged unless multiple channel plugins need a shared receipt seam.

## Reproduction Assessment

Not as a live UI repro from this review. Source inspection shows the inbound Feishu path never calls a read-receipt API, and the issue discussion includes a live v2026.5.4 trace reporting the same missing call.

## Solution Assessment

Unclear until maintainers choose the default/toggle policy and confirm the Feishu/Lark API contract. The narrow maintainable direction is Feishu-owned plugin support plus docs/tests, not a generic core migration first.
