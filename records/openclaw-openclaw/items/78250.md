## Summary

Keep open: current main ships Feishu as an official plugin and the source/docs show no Feishu send-read-receipt path or config knob, so this is remaining feature work rather than a cleanup close.

## What This Changes

Add Feishu DM read receipt support, optionally via `channels.feishu.sendReadReceipts`, so senders see the Feishu/Lark read indicator after inbound bot processing.

## Best Possible Solution

Add the capability inside the Feishu plugin with account-aware config, docs, and focused Feishu tests after maintainers choose the default and verify the Feishu/Lark API contract; avoid core changes unless multiple channel plugins need a shared receipt seam.

## Reproduction Assessment

No live Feishu reproduction was run. Source inspection gives a clear current-main path showing Feishu inbound messages are processed without any read-receipt send call or `sendReadReceipts` config.

## Solution Assessment

Unclear until maintainers decide the product default and confirm Feishu/Lark API support. The maintainable direction is Feishu-owned implementation plus docs/tests, not a generic core migration first.
