Keep open: current main still lacks a Feishu-owned read-receipt send path or config surface, and the discussion includes live v2026.5.4/v2026.5.5 confirmation that DMs do not show the sender-side read indicator.

## What This Changes

The issue requests Feishu DM read-receipt support so senders see the read indicator after the bot receives or processes inbound messages.

## Best Possible Solution

Add the capability inside the Feishu plugin with account-aware configuration, docs, and focused tests after maintainers decide the default behavior and verify the Feishu/Lark API contract.

Yes, at source level: current main’s Feishu inbound handler parses, dedupes, and enqueues messages without any mark-read/read-receipt call or config option. No live Feishu workspace reproduction was run in this pass.

## Solution Assessment

Unclear until maintainers decide default-on versus configurable behavior and confirm Feishu/Lark scopes/rate limits. The maintainable direction is Feishu-owned implementation rather than a generic core migration first.
