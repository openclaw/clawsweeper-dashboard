## Summary

This PR should stay open: the deduplication goal is valid, but the current guard can suppress the fallback system-Node migration warning in the no-system-Node path already called out by review. The protected `maintainer` label also makes this ineligible for cleanup closure.

## What This Changes

The PR changes gateway doctor repair warning flow, adds a focused regression test, and adds a changelog entry for avoiding duplicate Node runtime warnings.

## Best Possible Solution

Keep the deduplication intent, but preserve exactly one `Gateway runtime` note for each case: old system Node, no supported Node anywhere, and supported selected Node already reported by the install plan.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: with `needsNodeRuntime === true`, `runtimeChoice === "node"`, and `resolveSystemNodeInfo()` returning `null`, the patched condition skips the fallback note while `renderSystemNodeWarning(null)` has no replacement text. I did not run a live doctor command because this review was read-only.

## Solution Assessment

No. The safer solution is to suppress only genuinely duplicate text while still emitting the generic install guidance when no specific warning exists, or to follow the broader gate used by https://github.com/openclaw/openclaw/pull/78688.
