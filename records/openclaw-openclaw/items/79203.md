## Summary

Keep open: the PR has a protected `maintainer` label, still drops Gateway runtime install guidance for the no-system-Node migration path, and only supplies test-based proof instead of real doctor output.

## What This Changes

The PR adds a changelog entry, a focused doctor unit test, and changes `maybeRepairGatewayServiceConfig` to skip duplicate Gateway runtime notes when the detected service runtime is Node.

## Best Possible Solution

Keep the deduplication intent, but preserve exactly one `Gateway runtime` note for each case: selected supported Node already reported by the install plan, old system Node, and no supported Node anywhere.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `needsNodeRuntime === true`, `runtimeChoice === "node"`, and `resolveSystemNodeInfo()` returning null, the patched condition skips the fallback note while the install-plan warning path has no replacement text.

## Solution Assessment

No. The safer fix is to suppress only genuinely duplicate Gateway runtime text while still emitting the generic install guidance when no specific system-Node warning or supported preferred Node exists.
