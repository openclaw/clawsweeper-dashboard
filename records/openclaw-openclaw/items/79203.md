## Summary

Keep open: this PR is a focused fix for the linked duplicate-warning bug, but the latest diff still drops a dedicated Gateway runtime warning in one migration case and only provides unit-test proof rather than real doctor output.

## What This Changes

This PR adds a changelog entry and a focused doctor regression test, then changes gateway doctor service repair to suppress duplicate Node runtime notes when the detected service runtime is `node`.

## Best Possible Solution

Keep the deduplication intent, but choose one actionable runtime note based on whether a specific install-plan warning was actually emitted, while preserving missing-system-Node guidance.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `needsNodeRuntime === true`, `runtimeChoice === "node"`, and `resolveSystemNodeInfo()` returning `null`, the PR skips the fallback block while the install-plan runtime warning emits nothing.

## Solution Assessment

No. The current PR is close, but the safer fix is to suppress duplicate text only when another Gateway runtime warning was emitted, not for every `runtimeChoice === "node"` migration case.
