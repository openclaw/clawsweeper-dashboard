## What This Changes

The PR adds a changelog entry, a focused doctor test, and changes gateway doctor service repair warnings to skip the fallback Node runtime note when the detected service runtime is `node`.

## Best Possible Solution

Keep the deduplication intent, but centralize the warning choice so each migration case emits exactly one actionable `Gateway runtime` note.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `needsNodeRuntime === true`, `runtimeChoice === "node"`, and `resolveSystemNodeInfo()` returning `null`, the PR skips the fallback guidance while the install-plan warning path has no replacement text.

## Solution Assessment

No. The safer fix is to suppress only duplicate runtime text while preserving one warning for old system Node and no-supported-system-Node cases.
