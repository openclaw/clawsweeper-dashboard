## Summary

Keep open: this PR targets a valid doctor duplicate-warning bug, but the current diff can hide missing-system-Node install guidance and still has mock-only real behavior proof.

## What This Changes

The PR adds a changelog entry, a focused doctor unit test, and changes gateway doctor service repair warnings to skip the fallback runtime note when the detected service runtime is `node`.

## Best Possible Solution

Keep the deduplication intent, but preserve exactly one `Gateway runtime` note for each migration case: selected supported Node already reported by the install plan, old system Node, and no supported system Node anywhere.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `needsNodeRuntime === true`, `runtimeChoice === "node"`, and `resolveSystemNodeInfo()` returning null, the patched condition skips the fallback note while the install-plan warning path has no replacement text.

## Solution Assessment

No. The safer fix is to suppress only genuinely duplicate Gateway runtime text while still emitting the generic install guidance when no specific system-Node warning or supported preferred Node exists.
