## Summary

Keep open. Current main still lacks the requested Gateway-owned per-session activity API, WebSocket statechange event, exported schemas, and runtime/attention/children state model; the related implementation PRs are closed and unmerged, and the open Web UI PR is only an adjacent UI/progress slice.

## What This Changes

Request a Gateway-owned per-session activity API with busy/idle runtime state, explicit attention blockers, child rollups, and WebSocket statechange events.

## Best Possible Solution

Keep this as the canonical Gateway activity API feature and implement an additive, documented protocol contract with explicit runtime and attention transitions, stale cleanup, bounded child rollups, and focused Gateway tests.
