Keep open. Current main still only has partial session visibility through `sessions.list` metadata and `sessions.changed`; the requested `sessions.activity.get/list` RPCs, `sessions.activity.statechange` event, explicit attention model, stale-run accounting, and public schema are absent.

## What This Changes

The issue requests a gateway-owned per-session activity API, websocket statechange event, explicit attention states, stale-run handling, and child-session rollups.

## Best Possible Solution

Keep this as the canonical Gateway activity API feature and implement an additive documented contract with read-scope authorization, explicit attention transitions, versioned WS resync behavior, stale cleanup, bounded child rollups, and focused Gateway tests.

## Solution Assessment

Unclear until a Gateway API owner settles the contract. A dedicated additive `sessions.activity.*` surface is plausible, but the current `sessions.list` and `sessions.changed` metadata are not the full maintainable solution requested here.
