The issue requests a new Gateway-owned per-session activity API and WebSocket statechange contract with busy/idle, explicit attention, stale-run, and child-rollup state.

## Best Possible Solution

Land an additive Gateway-owned `sessions.activity` protocol and runtime tracker with documented versioning/resync, explicit attention states, bounded child rollups, stale-run handling, authorization coverage, docs, and focused tests before UI or watchdog consumers depend on it.
