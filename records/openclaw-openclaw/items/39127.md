## Summary

Keep open: current main and latest release still expose coarse session lifecycle/status fields and Control UI-visible active-run flags, but not the requested `sessions.activity.get/list`, monotonic activity statechange event, explicit attention states, stale-run accounting, or gateway-global child activity rollup. Related issues and comments route richer awaiting-user/subagent and watchdog work back to this issue, so it remains the canonical product/API design item.

## What This Changes

Add a gateway-owned per-session activity API and websocket statechange model for busy/idle, explicit attention states, stale-run accounting, and bounded child-session rollups.

## Best Possible Solution

Define one canonical gateway session activity and attention contract integrated with existing lifecycle/events, with versioned WS resync, authorization rules, stale-run cleanup, explicit producer/clear semantics, docs, and focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this requests a new public gateway/session activity contract. Source and release inspection give high-confidence evidence that current OpenClaw only has partial lifecycle/status projection, not the requested activity API.

## Solution Assessment

Unclear as written: the gateway boundary is the right place for a client-independent state contract, but maintainers still need to choose the canonical state owner, producer/clear semantics, durability, and resync behavior before implementation.
