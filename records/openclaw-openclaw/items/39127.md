## Summary

Current main and the shipped work from https://github.com/openclaw/openclaw/pull/50101 partially address session visibility with a lightweight activity RPC and persisted lifecycle status. This issue remains necessary because the public Gateway still lacks explicit `awaiting_user`/`awaiting_subagent` state, blocking relationships, monotonic activity-change events, and the requested reconnect/resync contract.

## What This Changes

Add a gateway-owned per-session activity API and WebSocket events covering busy state, explicit attention states, stale runs, and child-session rollups.

## Best Possible Solution

Extend the canonical Gateway session lifecycle owner with an additive, authorization-tested activity snapshot and push event that composes existing run and subagent facts, uses explicit durable attention transitions, and documents reconnect resynchronization.

## Solution Assessment
