## Summary

Keep open: current main and v2026.6.8 still expose only coarse session lifecycle/status and Control UI-visible active-run flags, not the requested `sessions.activity.get/list`, monotonic activity statechange event, explicit attention states, stale-run accounting, or global per-session activity model. ## What This Changes

Requested a gateway-owned per-session activity API and WebSocket statechange model for busy/idle state, explicit attention states, and child-session rollups.

## Best Possible Solution

Define a small canonical gateway session activity and attention contract integrated with existing session lifecycle/events, with explicit producer/clear semantics, versioned WS resync, auth rules, stale-run cleanup, docs, and focused tests.

Not applicable as a strict bug reproduction: this requests a new public gateway/session activity contract. Source inspection gives high-confidence evidence that current main only has basic lifecycle/status and active-run projection, not the requested attention/activity API.
