Keep open. Current `origin/main` still lacks the requested `sessions.activity.get/list` RPCs, per-session activity state-change event, explicit attention model, stale-run accounting, and public schema; recent related work only adds coarse `hasActiveRun`/status visibility or UI projection.

Add a Gateway-owned per-session activity API and WebSocket state-change surface for busy/idle state, explicit waiting states, child rollups, stale-run handling, and reconnect resync.

## Best Possible Solution

Keep this as the canonical Gateway activity API feature and implement an additive documented contract with read-scope authorization, explicit attention transitions, stale cleanup, versioned WebSocket resync behavior, bounded child rollups, and focused Gateway tests.
