## Summary

Current main now provides session status, timing, child linkage, and `sessions.changed` updates for Control UI-visible active runs, but the central cross-client attention contract remains absent: there is no explicit `awaiting_user`/`awaiting_subagent` state, global active-run projection, monotonic activity version, stale-run count, or reconnectable activity RPC. The earlier broad and lightweight PRs were closed without merging their activity APIs; merged dashboard work only partially addressed the request, and recent comments confirm clients still need a reliable idle/ready signal.

## What This Changes

Add a gateway-owned per-session activity API and WebSocket state changes covering objective run state, explicit attention states, stale-run handling, and cross-agent child relationships.

## Best Possible Solution

Extend the canonical gateway session lifecycle and `sessions.changed` model with a small, explicitly approved attention/dependency contract and globally prepared run facts, adding a dedicated RPC only if existing list/resync semantics cannot expose that state cleanly.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken documented behavior. Source inspection does provide a high-confidence demonstration of the gap because current main exposes only basic visible-run and terminal lifecycle fields, not explicit attention state.

## Solution Assessment

No, the full proposed parallel tracker is not yet proven to be the best implementation shape. The gateway is the correct shared boundary, but the narrowest maintainable solution should extend the existing canonical lifecycle/event path with deterministic attention semantics and avoid duplicate state ownership.

## Risks / Open Questions

- The requested design combines objective run state, semantic attention, persistence, stale-run recovery, cross-agent relationships, and a new public protocol; implementing it as a parallel tracker could create conflicting sources of truth with the existing lifecycle/session-row path.
- `awaiting_user` and `awaiting_subagent` need deterministic owner signals and clear clearing rules; inferring them from text or merely from child existence would misreport sessions.
- Durable attention state and global cross-agent visibility affect protocol authorization, restart behavior, and stored session semantics, so the final contract needs explicit maintainer approval before implementation.
