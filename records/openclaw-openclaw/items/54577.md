## Summary

Keep open. Current main has adjacent session listings, task tracking, per-session trajectory export, live agent/tool WebSocket events, and usage timelines, but it still lacks the requested first-class agent activity timeline command/view, `/timeline` command, JSON-exportable lifecycle timeline, and dedicated activity event contract. Related #60814 is still an open draft implementation thread, and #12831 overlaps the product direction without superseding this issue's CLI/chat/JSON timeline scope.

## What This Changes

Add an interactive agent activity timeline showing spawn/completion/error timing, durations, idle periods, filters, `/timeline`, TUI/UI monitoring, and JSON export.

## Best Possible Solution

Keep this issue open until maintainers scope and ship a reviewed activity timeline contract covering lifecycle taxonomy, storage or explicit ephemerality, redacted export semantics, CLI filters, `/timeline`, UI/TUI placement, docs, and focused tests coordinated with #60814 and #12831.

## Reproduction Assessment

Not applicable as a feature request. The missing behavior is verifiable with source/docs inspection on current main: there is no `openclaw timeline`, no `/timeline`, no Control UI/TUI timeline route, and no dedicated activity event stream.

## Solution Assessment

Unclear until maintainers scope the product and privacy contract. The maintainable path should reuse existing agent/session/tool/task/trajectory primitives where they fit, but add a first-class reviewed activity event contract instead of treating the current adjacent surfaces as complete.
