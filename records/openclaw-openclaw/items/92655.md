## Summary

Keep open: current main still routes `/models` Telegram callbacks through the command ingress allowlist path without first honoring the effective per-group `groupPolicy: "open"` override, and the issue is paired with an open closing fix PR that has not merged.

## What This Changes

The issue reports that Telegram `/models` inline provider/model callbacks are dropped in forum topics when a group-level `groupPolicy: "open"` override sits under a global Telegram allowlist.
