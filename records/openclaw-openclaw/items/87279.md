- **[P1] Use normalized session delivery context for recovery:** `src/agents/subagent-announce-delivery.ts:1043-1048`
  - body: The new requester-entry fallback rebuilds the route from only channel/lastChannel/origin.provider and last* fields. In the lost-origin case this PR is trying to fix, those entries can still resolve no deliverable channel; use the normalized helper and add a route/deliveryContext-only regression.

  - body: The added warning and enriched error include `targetRequesterSessionKey` as `requester=`, and this error is logged by the announce flow. Session keys can embed channel, DM, chat, topic, or phone-like route identifiers, so routine gateway logs should use redacted IDs, hashes, or boolean/channel-class fields instead.
  - confidence: 0.82
