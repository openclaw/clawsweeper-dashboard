## Summary

## What This Changes

The PR adds requester-session-entry fallback for subagent completion delivery context, a missing-context warning, enriched message-tool-only errors, and subagent announce delivery tests.

## Best Possible Solution

Recover only a complete deliverable route from the canonical session entry, warn or fail when no external target is available, and add both a focused regression and redacted live Telegram proof.

## Reproduction Assessment

No high-confidence live reproduction was established here. Current source and the PR's production logs make the failure path plausible, but the added tests do not actually simulate the stripped internal completion origin with missing direct/requester fallback.

## Solution Assessment

No. Recovering from the requester session entry is plausible, but the patch should reuse the canonical session delivery context or otherwise require a complete deliverable route before marking delivery successful.
