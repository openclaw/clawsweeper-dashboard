## Summary

Close as ClawHub/plugin-surface work: current main does not include the proposed generic `modelRouting` config, but it already exposes `before_model_resolve` for provider/model overrides and keeps narrower first-party model knobs in typed core surfaces; broad per-capability routing policy is optional plugin behavior under the current roadmap.

## What This Changes

The issue requests configurable per-task or per-capability model routing within a single OpenClaw session.

## Best Possible Solution

Keep core’s typed model-selection knobs, implement broad intent/capability routing as a ClawHub plugin using `before_model_resolve`, and track any missing first-party task contracts in narrower issues.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection show the proposed generic `modelRouting` key is absent while plugin hooks and narrower typed model routes exist.

## Solution Assessment

No, a broad core routing DSL is not the best current solution. The maintainable path is plugin-owned routing over `before_model_resolve`, with core changes reserved for specific missing contracts that plugins cannot express.
