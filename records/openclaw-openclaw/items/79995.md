## Summary

Current main already has the requested fixed-height chat shell, independently scrolling message area, mobile control relocation, and responsive browser coverage for phone-sized viewports, so this can close as implemented on main.

## What This Changes

The issue asks for the Control UI chat layout to fit within iPhone Safari viewports without full-page scrolling, with the message list scrolling independently.

## Best Possible Solution

Keep the current main implementation and let it ship in the next release; reopen or file a new issue only if a current-main or post-release iPhone Safari run still shows page-level vertical scrolling.

## Reproduction Assessment

No high-confidence failing reproduction remains on current main. ## Solution Assessment

Yes. The existing implementation is the narrow maintainable solution: CSS keeps the app shell viewport-bound, makes the transcript the scroll container, and handles mobile controls without adding config or product policy.
