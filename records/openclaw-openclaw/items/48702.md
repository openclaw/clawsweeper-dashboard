## Summary

Keep open: current main already separates `/status` think effort from reasoning state, but this PR’s extra hidden/visible wording is not on main, the branch is conflicted against the moved status formatter, it lacks real behavior proof, and the proposed patch still has a user-visible accuracy bug for non-canonical persisted reasoning values.

## What This Changes

This PR changes `/status` to always show a reasoning visibility label (`hidden`, `visible`, or `stream`), updates runtime prompt copy, and adjusts status/system-prompt tests.

## Best Possible Solution

If maintainers want the extra wording, rebase it onto `src/status/status-message.ts`, normalize the resolved reasoning value before formatting, preserve the existing think/reasoning split, and add real `/status` output proof.

## Reproduction Assessment

Yes, from source inspection for the branch-level bug: the command path can pass a raw persisted `resolvedReasoningLevel`, and the changed line formats that value without normalizing it first. I did not execute the PR branch in this read-only review.

## Solution Assessment

No. The patch should not merge as-is because it targets the old formatter location and normalizes only the session fallback; the safer fix is a current-main change in `src/status/status-message.ts` that normalizes the resolved input before label mapping.
