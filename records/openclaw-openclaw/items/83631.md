## Summary

The PR is a focused Telegram bug fix and should stay open for normal maintainer review rather than cleanup-close; source inspection supports the root cause, and the patch keeps progress visible while preventing non-final progress from mutating active session transcripts.

## What This Changes

The PR changes Telegram dispatch so only durable/final direct deliveries receive `transcriptMirror` and adds a regression test for tool progress staying chat-only while final replies still mirror.

## Best Possible Solution

Land the Telegram boundary fix with the regression test once maintainers accept that transient progress remains visible in chat but is no longer durable session transcript state.

## Reproduction Assessment

Yes. Source inspection shows current main can pass `transcriptMirror` into non-final Telegram direct deliveries, and the PR body gives a concrete Telegram verbose MiniMax reproduction with the matching `EmbeddedAttemptSessionTakeoverError`.

## Solution Assessment

Yes. Gating `transcriptMirror` at the Telegram `sendPayload` boundary is the narrow owner-boundary fix because it preserves final mirroring and visible progress delivery while preventing non-final active-transcript writes.
