## Summary

Keep open: the report is current, detailed, and current main still has matching result-capture, zero-stat, and duplicate/retry surfaces while three overlapping hardening PRs remain open.

## What This Changes

The issue reports intermittent subagent completion announcements that lose or truncate the child result payload and/or report zero token stats despite successful child work.

## Best Possible Solution

Make subagent completion delivery durable and idempotent: persist or recover the final child result before delivery, derive announce stats from transcript-backed usage when live session-store fields are stale, suppress duplicate delivery for the same run, and cover large-output, missing-usage, compaction, retry, and duplicate-emission paths with focused tests.

## Reproduction Assessment

Yes for triage, but not yet as a deterministic local test. The issue provides four same-day live reproductions, and current main still contains matching capture, stats, and delivery/error-handling paths.

## Solution Assessment

No. Current main is not the best solution yet; the maintainable route is to coordinate the open completion hardening PRs and land one coherent fix for durable result recovery, transcript-backed stats, and duplicate suppression.
