## Summary

Keep open. The report is current, concrete, and still not fully solved on current `main`: completion capture can still freeze the current child-history snapshot, announce stats still format missing session-store token fields as zero, and the related delivery hardening PRs are open rather than shipped. The later duplicate-emission reproduction also remains useful evidence for idempotent completion delivery work.

## What This Changes

The issue reports intermittent subagent completion announces where successful worker runs can emit truncated, duplicated, or token-zero completion events instead of the final worker result and accurate stats.

## Best Possible Solution

Keep this issue as the concrete ASCval reproduction and coordination point. The desired end state is a completion announce path that persists or recovers the final child result, deduplicates delivery for a completed run, and derives completion stats from durable usage data when the live session-store row is stale.

## Reproduction Assessment

Yes for triage, though not yet as a deterministic local unit test. The report gives four detailed live reproductions, and current main has matching code paths for incomplete capture, duplicate-prone retry/delivery coordination, and zero-default token formatting.

## Solution Assessment

No. The current main behavior is not the best solution because announce output and token stats are still recovered through narrower paths than other session surfaces; the safer direction is durable payload/usage recovery plus idempotent delivery, coordinated with the open subagent hardening PRs.
