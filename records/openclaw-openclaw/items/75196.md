## Summary

Keep open. The report is current, detailed, and current main still has matching failure surfaces: snapshot-based result capture, one-time frozen completion text, zero-default announce stats, and an unhandled completion wait catch path while three adjacent hardening PRs remain open.

## What This Changes

The issue reports intermittent subagent completion announces that can lose the final result payload, zero token stats, or emit duplicate completion events even though the child work completed successfully.

## Best Possible Solution

Make subagent completion delivery durable and idempotent: persist or recover the final child result before delivery, derive announce stats from transcript-backed usage when live session-store fields are stale, suppress duplicate completion delivery for the same run, and cover large-output, missing-usage, compaction, retry, and duplicate-emission paths with focused tests.

## Reproduction Assessment

Yes for triage, but not yet as a deterministic local test. The issue provides four same-day live reproductions, and current main still contains matching capture, stats, and delivery/error-handling paths.

## Solution Assessment

No. Current main is not the best solution yet; the maintainable route is to coordinate the open completion hardening PRs and land a coherent fix for durable result recovery, transcript-backed stats, and duplicate suppression.
