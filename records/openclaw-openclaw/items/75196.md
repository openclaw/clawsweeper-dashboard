## Summary

Keep open. The report is current and detailed, and current main still has matching failure surfaces for lost or stale completion payloads, zero-default token stats, and duplicate completion delivery while overlapping hardening PRs remain open.

## What This Changes

The issue asks OpenClaw to make subagent completion events reliably carry the final result payload and token stats without duplicate completion emissions.

## Best Possible Solution

Make subagent completion delivery durable and idempotent: persist or recover the final child result before delivery, derive announce stats from transcript-backed usage when live fields are stale, suppress duplicate completion delivery for the same run, and cover the large-output and recovery cases with focused tests.

## Reproduction Assessment

Yes for triage, but not yet as a deterministic local test. The issue provides four detailed same-day reproductions, and current main still contains code paths matching the missing payload, zero stats, and duplicate-emission signatures.

## Solution Assessment

No. Current main is not the best solution yet; the maintainable path is to coordinate the open completion hardening PRs and land one coherent fix for durable result recovery, transcript-backed stats, and duplicate suppression.
