## Summary

Keep open. Current main still has the same completion-delivery failure surfaces described by the report: bounded snapshot-based result capture, one-time frozen result text that can become null on empty/failed capture, zero-default announce stats when live token counters are missing, and an ignored completion-wait catch path while overlapping hardening PRs remain open.

## What This Changes

The issue reports intermittent subagent completion announces that can lose the final result payload, zero token stats, or emit duplicate completion events even though the child work completed successfully.

## Best Possible Solution

Make subagent completion delivery durable and idempotent: persist or recover the terminal child result before delivery, derive announce stats from transcript-backed usage when live session-store fields are stale, suppress duplicate completion delivery for the same run, and cover large-output, missing-usage, compaction, retry, and duplicate-emission paths with focused tests.

## Reproduction Assessment

Yes for triage, but not yet as a deterministic local test. The issue provides four same-day live reproductions, and current main still contains matching capture, stats, and delivery/error-handling paths.

## Solution Assessment

No. Current main is not the best solution yet; the maintainable route is to coordinate the open completion hardening PRs and land a coherent fix for durable result recovery, transcript-backed stats, and duplicate suppression.
