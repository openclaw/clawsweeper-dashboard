## Summary

Keep open. Current main still matches the reported failure family: the announce path freezes a single child-output capture, token stats still come only from live session-store fields that format missing data as zero, and related payload/idempotency/error-recovery PRs are still open rather than merged or shipped.

## What This Changes

The issue requests hardening subagent completion announcements so successful worker runs consistently deliver the final result payload and accurate token stats instead of truncated, missing, duplicate, or zero-stat completion events.

## Best Possible Solution

Keep this issue as the ASCval reproduction and coordination point until the completion path reliably persists or recovers the final child result, deduplicates delivery for a completed run, and derives completion stats from durable usage data when the live session-store row is stale or missing.

## Reproduction Assessment

Yes for triage, though not yet as a deterministic local unit test. The report gives four detailed live reproductions, and current main has matching code paths for single-snapshot output capture, duplicate-prone completion recovery, and zero-default token formatting.

## Solution Assessment

No. Current main is not the best solution because announce output and token stats are recovered through narrower paths than adjacent session surfaces; the safer direction is durable payload and usage recovery plus idempotent completion delivery, coordinated with the open subagent hardening PRs.
