## Summary

Keep open. The report is current, concrete, and still matches current main: subagent completion result capture can freeze a single incomplete snapshot, announce stats still default missing live session-store token fields to zero, and the related hardening work remains open rather than shipped.

## What This Changes

The issue reports intermittent native subagent completion handoffs where successful child runs announce a truncated or missing result and/or zero token usage, plus a later duplicate completion emission.

## Best Possible Solution

Keep this issue as the ASCval reproduction and coordination point until the subagent completion path durably captures or recovers the final child result, derives stats from durable transcript usage when live session-store values are stale, and deduplicates completion delivery for the same run.
