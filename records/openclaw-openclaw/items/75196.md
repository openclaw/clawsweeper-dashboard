## Summary

Keep open. Current main still has the same completion-delivery failure surfaces: bounded snapshot result capture can freeze null or partial output, announce stats still default missing usage to zero, and related hardening PRs remain active rather than merged.

## What This Changes

The issue asks OpenClaw to make subagent completion events reliably include the final result payload and token stats without duplicate completion emissions.

## Best Possible Solution

Make subagent completion delivery durable and idempotent by persisting or recovering the final child result before delivery, deriving announce stats from transcript-backed usage when live fields are stale, and suppressing duplicate delivery for the same run.

## Reproduction Assessment

Yes for triage, but not yet as a deterministic local test. The issue provides four detailed same-day reproductions, and current main still contains code paths matching the missing payload, zero stats, and duplicate-emission signatures.

## Solution Assessment

No. Current main is not the best solution yet; the maintainable path is to coordinate the open completion hardening PRs and land one coherent fix for durable result recovery, transcript-backed stats, and duplicate suppression.
