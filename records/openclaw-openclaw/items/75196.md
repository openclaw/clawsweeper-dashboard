## Summary

Keep open. The report is current, detailed, and still matches current main: completion capture can freeze a narrow child-history snapshot, announce stats still read only session-store token fields and render missing values as zero, and the related payload/idempotency/error-recovery PRs are open rather than merged or shipped.

## What This Changes

The issue requests investigation and hardening for intermittent subagent completion announcements that can deliver truncated or missing result payloads and/or zero token stats despite the child task completing successfully.

## Best Possible Solution

Keep this issue as the concrete ASCval reproduction and coordination point. The desired end state is a subagent completion path that persists or recovers the final child result, deduplicates delivery for a completed run, and derives completion stats from durable usage data when the live session-store row is stale or missing.

## Reproduction Assessment

Yes for triage, though not as a deterministic local unit test from this read-only review. The issue gives four detailed live reproductions, and current main has matching code paths for incomplete capture, duplicate-prone delivery/recovery coordination, and zero-default token formatting.

## Solution Assessment

No. The current main behavior is not the best solution because announce output and token stats are recovered through narrower paths than adjacent session surfaces; the safer direction is durable payload and usage recovery plus idempotent completion delivery, coordinated with the open subagent hardening PRs.
