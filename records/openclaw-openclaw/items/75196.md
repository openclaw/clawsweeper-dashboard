## Summary

Keep open. The report is current, detailed, and current main still has matching failure surfaces for lost/truncated payloads, zero token stats, and duplicate/lost completion delivery while overlapping hardening PRs remain open.

## What This Changes

The issue reports intermittent subagent completion announcements losing the final result payload, zeroing token stats, or emitting duplicate completion events after otherwise successful worker runs.

## Best Possible Solution

Make subagent completion delivery durable and idempotent: recover or persist final child results before announce, derive stats from transcript-backed usage when live fields are stale, suppress duplicate delivery for the same run, and add focused large-output, compaction, retry, missing-usage, and duplicate-emission tests.

## Reproduction Assessment

Yes for triage, but not yet as a deterministic local test. The issue provides four same-day live reproductions, and current main still contains matching capture, stats, and delivery/error-handling paths.

## Solution Assessment

No. Current main is not the best solution yet; the maintainable path is to coordinate the open completion hardening PRs and land one coherent fix for result recovery, transcript-backed stats, and duplicate suppression.
