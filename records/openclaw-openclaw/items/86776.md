## Summary

Keep open: the target behavior is useful and source-reproducible, but this branch is conflicting and its fix puts provider-policy lookup into an inline model-resolution builder hot path.

## What This Changes

The PR changes inline provider model construction and related tests so provider-policy defaults, especially Anthropic's `anthropic-messages` API, apply when inline model entries omit `api`.

## Best Possible Solution

Rebase onto `src/agents/embedded-agent-runner` and apply Anthropic provider-policy defaults through config materialization or a prepared policy context, with explicit upgrade/owner sign-off for the transport change.

## Reproduction Assessment

Yes, source-reproducible. Current main's raw inline builder uses `model.api ?? entry.api`, while config materialization owns provider-policy defaults, so raw Anthropic inline entries can miss `anthropic-messages` when built before materialization.

## Solution Assessment

No as submitted. The behavior is useful, but the branch needs rebasing and the provider-policy lookup should not be added directly to the inline model-resolution builder hot path.
