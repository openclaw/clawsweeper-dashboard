## Summary

Keep open: the bug is source-reproducible and the PR has useful terminal proof, but the branch is conflicting and the patch moves provider-policy lookup into the model-resolution builder hot path.

## What This Changes

The branch changes inline provider model construction and tests so provider-policy defaults, especially Anthropic's `anthropic-messages` API, apply when inline model entries omit `api`.

## Best Possible Solution

Rebase onto the current `embedded-agent-runner` paths and apply Anthropic provider-policy defaults through config materialization or a prepared policy context instead of per-resolution discovery.

## Reproduction Assessment

Yes, source-reproducible. Current main's raw inline builder uses `model.api ?? entry.api`, while config materialization owns provider-policy defaults, so raw Anthropic inline entries can miss `anthropic-messages` if built before materialization.

## Solution Assessment

No as submitted. The target behavior and proof are useful, but the branch needs rebasing and the provider-policy lookup should not be added directly to the model-resolution builder hot path.
