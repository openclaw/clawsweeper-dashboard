## Summary

Keep this PR open: the underlying Anthropic inline-model defaulting gap is still present on current main, but the branch is not merge-ready because it is conflicting and moves provider-policy lookup into inline model construction instead of using a prepared/config-materialized boundary.

## What This Changes

The PR applies provider-policy normalization while building inline provider models and adds focused tests for Anthropic inline entries inheriting `anthropic-messages`.

## Best Possible Solution

Keep the Anthropic inline defaulting behavior, but rebase it onto `src/agents/embedded-agent-runner` and apply the default through config materialization or a prepared policy context rather than provider-policy discovery inside model construction.

## Reproduction Assessment

Yes, source-reproducible. Current main's inline builder uses `model.api ?? entry?.api`, while provider-policy defaults are applied during config materialization, so raw Anthropic inline entries can miss `anthropic-messages` before materialization.

## Solution Assessment

No as submitted. The behavior is useful, but the direct provider-policy lookup should be moved out of inline model construction or backed by explicit maintainer approval for that boundary change.
