## What This Changes

The issue reports that Windows `openclaw gateway install/start/restart` registers a Scheduled Task but does not keep the gateway running after a foreground gateway window closes.

## Best Possible Solution

Land a narrow daemon fix that proves the managed Scheduled Task or generated script actually started, ignores pre-existing foreground gateway listeners as launch evidence, preserves listener-backed status reporting, and adds focused regression plus native Windows proof.

Validate with targeted daemon tests and, when feasible, native Windows proof: start a foreground `openclaw gateway`, run `openclaw gateway install` or `openclaw gateway start`, close the foreground window, then verify `openclaw gateway status --deep --require-rpc` reaches the managed gateway.
