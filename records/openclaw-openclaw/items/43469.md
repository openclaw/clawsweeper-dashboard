## Summary

Keep open: the PR is useful security hardening and current main still does not scan Markdown skills, but the latest head is not merge-ready because numbered Markdown install steps can bypass the new download-and-execute rule.

## What This Changes

The PR extends the skill security scanner to include Markdown files with Markdown-specific hidden Unicode, executable data URI, download-and-execute, base64, and hex-payload rules, plus tests and a changelog entry.

## Best Possible Solution

Keep the Markdown scan surface, strip ordered-list command prefixes before downloader detection, add regression coverage for numbered steps, and keep exact-head security/CI checks green before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because `.md` is absent from the scannable extension set, and the PR-head blocker is source-reproducible because `1. curl ... | bash` leaves the ordered-list marker intact before downloader detection.

## Solution Assessment

No, not as-is. The existing skill scanner is the right boundary and the parser-style matcher is a good direction, but it needs ordered-list prefix normalization and coverage before it safely solves the Markdown threat gap.
