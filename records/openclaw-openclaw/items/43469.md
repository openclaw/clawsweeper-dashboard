## Summary

Keep open: current main still skips Markdown skill files, but this PR is not merge-ready because the critical Markdown download-and-execute rule has practical false negatives for common wrapped, absolute-path, versioned, and line-wrapped interpreter forms.

## What This Changes

The PR extends the skill scanner so Markdown/SKILL.md files are scannable and adds Markdown-specific hidden Unicode, executable data URI, download-and-execute, base64, and hex-payload rules with tests.

## Best Possible Solution

Keep the Markdown scanner work in the existing security scanner, rebase it onto current main's scanner hardening, strengthen the download-and-execute detector, and add regressions for wrapped, absolute-path, versioned, and continued pipelines before merge.

## Reproduction Assessment

Yes. Main is source-reproducible because .md is absent from the scanner's extension set and current tests expect Markdown to be non-scannable; the PR-specific regex false negative was reproduced with a direct Node check against the proposed rule.

## Solution Assessment

No, not as-is. Adding Markdown-specific rules inside the existing scanner is the right boundary, but the critical download-and-execute rule needs stronger matching plus focused regression tests before this is the best fix.
