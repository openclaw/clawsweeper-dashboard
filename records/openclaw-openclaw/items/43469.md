## Summary

Keep this PR open: current main still has the Markdown skill-scanning gap, and the prior Greptile/ClawSweeper discussion correctly identified a security-relevant weakness in the proposed critical rule. The contributor fixed the embedded-pipe case, but the latest PR head still misses common wrapped interpreter forms, so it is useful source work but not mergeable as-is.

## What This Changes

The PR extends `src/security/skill-scanner.ts` and scanner tests so `.md`/`SKILL.md` files are scanned with Markdown-specific hidden Unicode, executable data URI, download-exec, base64, and hex payload rules.

## Best Possible Solution

Keep the PR as the active Markdown skill-scanning candidate, rebase it onto current main, strengthen the download-and-execute rule for wrapped, absolute-path, versioned, and line-wrapped interpreter forms, and add regression coverage before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because `.md` is absent from `SCANNABLE_EXTENSIONS` and the current test suite still expects Markdown to be non-scannable; the PR-specific false negative was reproduced with a direct Node check against the proposed regex.

## Solution Assessment

No, not as-is. Adding Markdown-specific rules inside the existing scanner is the right boundary, but the critical download-and-execute rule needs stronger matching or parsing plus focused regression tests before this is the best fix.
