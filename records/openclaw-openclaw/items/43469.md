## Summary

Keep open: the PR is useful security hardening and has real behavior proof, but the latest scanner budget logic still only allows one `SKILL.md` per scanned tree, leaving additional skill definitions unscanned.

## What This Changes

The PR adds Markdown/SKILL.md scanning rules, directory collection changes, tests, and a changelog entry for skill security scanning.

## Best Possible Solution

Land the Markdown scanner after changing collection so `SKILL.md` is prioritized without limiting the scan to only one skill definition per tree.

## Reproduction Assessment

Yes. Source inspection shows current main does not scan `.md` files, and PR head is source-reproducible because `skillBudget = 1` plus the non-skill Markdown fallback skips any second `SKILL.md`.

## Solution Assessment

No, not as-is. Extending `src/security/skill-scanner.ts` is the right direction, but the file-budget logic should keep all `SKILL.md` files eligible within `maxFiles` after priority ordering.
