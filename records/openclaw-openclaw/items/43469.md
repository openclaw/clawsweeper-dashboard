## Summary

Keep open: current main still does not scan Markdown skill definitions, and this PR is useful security hardening with sufficient terminal proof, but the latest head still has blocking scanner-regex issues including a failing CodeQL check, missed wrapped execution forms, and a critical false positive on Markdown tables.

## What This Changes

The PR adds Markdown `.md`/`SKILL.md` scanning to `src/security/skill-scanner.ts`, Markdown-specific threat rules, regression tests, and a changelog entry.

## Best Possible Solution

Keep the Markdown scanner in the existing security scanner, but replace the brittle download-exec regex with bounded, table-aware matching and regression tests before secops review and merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because `.md` is absent from the scanner extension set and tests expect Markdown to be non-scannable; the PR blockers are reproducible by evaluating the exact latest-head regex and by the failing CodeQL check.

## Solution Assessment

No, not as-is. The existing scanner is the right boundary and the code-first scan budget fix is good, but the download-exec detection needs a linear, table-aware implementation with wrapper regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Replace the download-exec regex with linear matching:** `src/security/skill-scanner.ts:250`
  - body: The new `markdown-download-exec` pattern can backtrack exponentially on repeated backslash-newline-tab input; CodeQL fails on this line, and a local timing check hung on a larger sample. Since untrusted `SKILL.md` content is scanned during install and audit, use a bounded parser or linear matching instead of this nested regex.
  - confidence: 0.94
- **[P1] Handle sudo/env wrapper variants after the pipe:** `src/security/skill-scanner.ts:250`
  - body: The rule returns no finding for common execution forms such as `curl ... | sudo -E bash` and `curl ... | env FOO=1 bash`, so malicious Markdown can bypass the critical install/audit signal this PR is adding. Cover sudo flags and env assignments without reintroducing the regex backtracking issue.
  - confidence: 0.91
- **[P1] Ignore markdown table rows when detecting pipelines:** `src/security/skill-scanner.ts:250`
  - body: The current pattern matches benign table text like `| curl | bash |` as a critical download-and-execute finding. That can block legitimate skill installs or audits for command-name documentation, so scope detection to actual command lines or code fences and add a table-row regression test.
  - confidence: 0.9
