## Summary

Keep open: the PR is on the right scanner boundary and includes real behavior proof, but the latest head still has two concrete matcher bugs that can miss malicious Markdown pipelines or block benign Markdown tables.

## What This Changes

Adds `.md`/`SKILL.md` scanning to the skill security scanner with Markdown-specific hidden Unicode, executable data URI, download-and-execute, base64, and hex-payload rules plus tests and a changelog entry.

## Best Possible Solution

Keep the Markdown scan surface, but tighten the command parser to normalize downloader paths and recognize Markdown table context, then land with focused regression tests and exact-head checks.

## Reproduction Assessment

Yes. Current main is source-reproducible because `.md` is absent from the scanner extension set, and the remaining PR-head blockers are source-reproducible from the downloader-prefix check and table-row guard.

## Solution Assessment

No, not as-is. The existing skill scanner is the right place to extend coverage, but the lightweight Markdown pipeline parser needs the two narrow repairs before this is safe security hardening.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Normalize downloader paths before pipeline matching:** `src/security/skill-scanner.ts:471`
  - body: `isDownloadCommandSegment` only accepts segments that begin with `curl` or `wget`, so `/usr/bin/curl ... | bash` and `/bin/wget ... | sh` evade the new critical Markdown rule even though they download and execute payloads the same way.
  - confidence: 0.9
- **[P2] Skip pipe-less Markdown table rows before parsing pipelines:** `src/security/skill-scanner.ts:526`
  - body: The guard skips only rows that start with `|`, but Markdown tables can omit the leading pipe. A benign table row such as `curl | bash` under a table separator is parsed as a shell pipeline and reported as a critical finding.
  - confidence: 0.88

## Repair Work Prompt

For PR #43469, keep the Markdown skill scanning feature but repair the two remaining matcher defects. In `src/security/skill-scanner.ts`, normalize downloader command tokens before matching so absolute paths like `/usr/bin/curl` and `/bin/wget` are treated like `curl`/`wget`, and update the Markdown table guard so pipe-less table rows such as `curl | bash` under a table separator do not become critical download-exec findings. Add focused regression tests in `src/security/skill-scanner.test.ts` for both cases. Preserve code-vs-Markdown rule isolation, do not add dependencies, do not broaden this into a scanner redesign, and keep the existing changelog entry unless wording must change. Related refs: #43469 supersedes #10705 and extends the scanner introduced by #9806.
