## Summary

Keep open. Current main still excludes Markdown from the built-in skill scanner, so the requested SKILL.md coverage is not implemented, and this PR remains a focused candidate. The branch should not merge as-is because its critical download-and-execute Markdown rule still misses common execution forms.

## What This Changes

The PR adds .md/SKILL.md scanning to the skill scanner with Markdown-specific hidden Unicode, executable data URI, download-and-execute, base64, and hex payload rules plus scanner tests.

## Best Possible Solution

Keep this PR open as the active Markdown skill-scanning hardening candidate. Rebase it onto current main, preserve the scanner's existing install/audit behavior, harden the download-and-execute heuristic with regression tests for wrapped and line-wrapped interpreter forms, and treat direct ClawHub skill-install scanning as a separate maintainer decision.

## Reproduction Assessment

Yes. Current main reproduces the missing coverage by inspection: .md is absent from SCANNABLE_EXTENSIONS and the tests still expect readme.md to be non-scannable. The PR-specific regex gap also reproduces with a direct Node check against the exact proposed pattern.

## Solution Assessment

No, not as-is. Adding Markdown-specific rules in the existing scanner is the right narrow boundary, but the critical rule needs stronger matching and tests before this is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Match wrapped download-exec interpreters:** `src/security/skill-scanner.ts:239`
  - body: The new critical Markdown rule only recognizes a bare interpreter token after the pipe.
