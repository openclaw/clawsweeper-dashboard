## What This Changes

## Best Possible Solution

Rebase onto current main's local mirror hardening, make symlink unlink and recursive symlink-child behavior either safe or rejected before remote state changes, then add real OpenShell mirror proof for the chosen compatibility policy.

## Solution Assessment

No. The owner boundary is right, but the branch should rebase onto current main's narrower hardening and fix symlink local removal before remote mutation before maintainers decide whether to restore cross-root and hardlink compatibility.
