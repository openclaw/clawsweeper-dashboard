## Summary

## What This Changes

The PR routes OpenShell local mirror mkdir, remove, and rename through fs-safe root-confined operations and adds regression tests for rebinding and cross-root rename cases.

## Best Possible Solution

Preserve existing safe symlink unlink and rename semantics while keeping local mirror mutations root-confined, then add endpoint regression tests and real OpenShell mirror proof before merge.

## Reproduction Assessment

Yes, source-reproducible: the PR deletes the remote path first, then `removeRootRelative()` calls `Root.stat()`, and fs-safe v0.3.0 rejects symlink leaves in `Root.stat()` before `Root.remove()` can safely unlink them.

## Solution Assessment

No; the fix belongs in the OpenShell bridge, but the current helper is not the best implementation because it preflights safe unlink/rename endpoints through stricter stat operations. The safer path is to preserve existing symlink endpoint behavior or reject it before any remote mutation with explicit compatibility approval.
