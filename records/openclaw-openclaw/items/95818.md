## Summary

Keep open: the local mirror hardening is aimed at the right OpenShell boundary, but the new root-confined remove path rejects symlink endpoints after the remote mutation has already run, which can break existing mirror behavior and leave local and remote state divergent. The contributor proof is also test-only rather than a real OpenShell mirror run.

## What This Changes

The PR routes OpenShell local mirror mkdir, remove, and rename through fs-safe root-confined operations and adds regression coverage for rebinding and cross-root rename cases.

## Best Possible Solution

Preserve existing symlink unlink and rename semantics with root-confined operations, or reject those endpoints before any remote mutation with explicit tests and real OpenShell mirror proof.

## Reproduction Assessment

Source-reproducible: current main performs local mirror mkdir/remove/rename with raw host-path operations after validation, and the PR adds targeted rebinding tests for that class. I did not run tests in this read-only review; the blocking symlink regression is proven from the PR code plus the fs-safe dependency contract.

## Solution Assessment

No; the PR is in the right owner boundary, but the stat-before-remove path and root.move/fromRoot.stat rename paths change existing symlink endpoint behavior after remote mutation. The safer solution preserves symlink unlink/rename semantics or rejects them before remote state changes.
