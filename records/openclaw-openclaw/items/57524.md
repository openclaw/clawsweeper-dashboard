## Summary

Keep open. Current main still lacks a sudo-to-root guard for Linux user-systemd gateway installs, and this PR is directionally useful, but the branch is not merge-ready because the guard is incomplete across install entrypoints, two tests will fail statically, custom-unit status hints can point at the wrong service, and an unrelated build sourcemap change needs removal or separate approval.

## What This Changes

The PR adds a Linux sudo gateway-install guard, post-install linger/runtime diagnostics, guard tests, and a node-build sourcemap toggle.

## Best Possible Solution

Land a narrow current-main fix that rejects sudo-to-root before Linux user-systemd gateway daemon writes across first-party install paths, keeps linger as separate privileged guidance, fixes status hints/tests, and leaves build output unchanged.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main lacks the guard, the PR test assertion does not match the helper message, the install test mock lacks the new export, and unguarded service.install call sites remain visible in source.

## Solution Assessment

No. Rejecting sudo-to-root before user-systemd writes is the right direction, but this patch should apply the guard at a shared gateway daemon install boundary, update tests and custom-unit hints, and remove the unrelated build-artifact change.
