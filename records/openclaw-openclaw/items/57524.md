## Summary

Keep open. Current main still lacks the requested sudo install guard, but the latest PR patch is not merge-ready because static diff review shows failing test expectations, a missing test mock export, misleading custom-unit diagnostics, and an unrelated build-output change.

## What This Changes

The PR adds a Linux `sudo openclaw gateway install` guard, post-install linger/status diagnostics, new guard tests, and a node build sourcemap setting.

## Best Possible Solution

Land a narrow current-main daemon CLI fix that rejects sudo gateway installs before systemd writes, leaves linger as separate privileged guidance, updates CLI tests/mocks, and keeps build config unchanged.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: the PR's new assertion does not match the helper message, and current `install.test.ts` mocks `./shared.js` without the newly imported helper. No live Linux sudo run is needed to reproduce those patch failures.

## Solution Assessment

No. Rejecting sudo at the CLI boundary is a maintainable direction, but this patch is not the best current implementation until the tests/mocks are aligned, custom-unit hints are corrected, and the unrelated sourcemap change is removed.
