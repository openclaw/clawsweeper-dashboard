## Summary

Keep open for maintainer review: the patch appears correct and the PR body now has redacted live output, but it intentionally tightens web provider credential detection, so compatibility and auth-provider acceptance remain before merge.

## What This Changes

The PR changes the shared web provider credential helper and tests so env-style SecretRef strings only count as credentials after env resolution, while literal strings and non-env refs still count.

## Best Possible Solution

Land the shared-helper fix after maintainers accept the stricter env-ref credential semantics; avoid duplicating this logic in individual web providers.

## Reproduction Assessment

Yes. Source inspection on current main shows an env shorthand can parse as a SecretRef and then still be counted as a nonempty literal credential before env resolution; I did not run a local runtime command in this read-only review.

## Solution Assessment

Yes. The shared helper is the right fix boundary because both web_search and web_fetch call it, and caller-level fixes would duplicate provider credential semantics.
