## Summary

Keep open. Current main no longer uses the original package-level xargs pipeline, but the active docs formatter still sends the full docs file set to one formatter process in check mode; this PR remains useful because it adds Windows-aware oxfmt invocation, batching, explicit child-process diagnostics, and focused regression tests.

## What This Changes

This PR updates the docs formatting script to use spawn-based file discovery, Windows cmd.exe wrapping, batched oxfmt invocations, child-process error handling, and Vitest coverage for the formatter helper.

## Best Possible Solution

Land this branch, or an equivalent canonical fix, after maintainer review confirms Windows command-line batching, PowerShell/cmd.exe compatibility, spawn-error diagnostics, and green targeted docs formatter checks; then close the linked native PowerShell docs-format issue and leave the older closed attempts superseded.
