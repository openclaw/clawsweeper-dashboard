## Summary

Keep open: current main still lacks Gateway WebSocket last-completed handshake phase logging, and the only linked implementation PR was closed unmerged with its head branch deleted.

## What This Changes

The issue requests failure diagnostics that identify the last completed Gateway WebSocket handshake phase before timeout or pre-connect close.

## Best Possible Solution

Add a narrow Gateway WebSocket diagnostics change that records a redacted, monotonic last-completed handshake phase and emits it only on pre-connect failure logs, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows a straightforward repro path: open a Gateway WebSocket, idle past the preauth timeout, and the timeout/close logs will report only the terminal cause/coarse state rather than a last-completed phase.

## Solution Assessment

Yes. The narrow maintainable path is an internal enum-style phase tracker threaded through the existing Gateway WS connection and connect handler, without changing protocol, auth, or configuration.
