## What This Changes

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79603 with a narrow Gateway diagnostics PR. Add bounded last-completed WebSocket handshake phase tracking for failure-only logs in `src/gateway/server/ws-connection.ts` and `src/gateway/server/ws-connection/message-handler.ts`; useful prior source work exists in closed PR https://github.com/openclaw/openclaw/pull/79654, but re-review the approach instead of copying blindly. Expected phases should cover the reported lifecycle (`ws_upgrade_started`, auth token received, auth validated, session attached, subscriptions registered, ready) as internal enum-like values. Emit the last completed phase on `handshake timeout` and `closed before connect` failure logs and structured close metadata; keep successful ready closes and protocol/API/config behavior unchanged. Do not log raw auth tokens, frame bodies, cookies, or unsanitized request data. Add focused regression coverage, update the changelog because runtime log output is user-visible, and stop if the implementation would require a protocol/config/product-policy change.
