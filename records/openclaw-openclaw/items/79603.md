## Summary

Keep open: current main still lacks last-completed Gateway WebSocket handshake phase logging, and https://github.com/openclaw/openclaw/pull/79654 is open as the implementation candidate.

## What This Changes

Add Gateway WebSocket diagnostics that log the last completed handshake phase when a connection fails before becoming ready.

## Best Possible Solution

Land or replace the linked Gateway WS diagnostics PR with a narrow, redacted phase enum in failure logs and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the pre-connect close and handshake timeout paths are reachable but only log coarse terminal state, not the requested last-completed phase; I did not run a live Windows gateway.

## Solution Assessment

Yes. A failure-only, redacted phase tracker in the Gateway WS connection and connect handler is the narrow maintainable direction, but the issue should stay open until https://github.com/openclaw/openclaw/pull/79654 or an equivalent patch lands.
