## Summary

Keep open: current main still lacks last-completed Gateway WebSocket handshake phase logging, and the only linked implementation PR is closed unmerged. The issue is narrow and actionable, but it is a diagnostic enhancement rather than a broken documented contract.

## What This Changes

Request Gateway WebSocket failure logs to include the last completed handshake phase so operators can distinguish timeout, auth, session attach, and subscription stalls.

## Best Possible Solution

Port or reopen a narrow Gateway WS diagnostics patch that logs a redacted last-completed phase on failure-only timeout and pre-connect close paths, with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main reaches pre-connect close and handshake-timeout paths that log coarse state/cause without a last-completed phase; I did not run a live Windows gateway.

## Solution Assessment

Yes, the maintainable direction is a failure-only monotonic phase tracker inside the Gateway WS connection/message-handler path with redacted log metadata and focused tests. The closed linked PR is useful source material, but the change still needs to be landed against current main.
