## Repair Work Prompt

Implement failure-only gateway WebSocket handshake phase diagnostics for this issue. Add a small internal phase tracker to src/gateway/server/ws-connection.ts and src/gateway/server/ws-connection/message-handler.ts so failure logs/structured close metadata include the last completed safe phase, covering at least connection open/challenge sent, connect frame received/validated, auth checked, client registered/session attached, node registration/subscriptions where applicable, hello-ok sent, and ready. Keep the phase labels redacted and internal; do not log tokens, passwords, raw auth payloads, private headers, or unauthenticated free-form client claims as trusted facts. Do not change protocol versions, timeout values, reconnect behavior, or add a config option. Add focused regression tests for timeout, invalid/auth failure, and successful registration/hello-ok boundaries, and add a concise changelog entry if the logging output changes.
