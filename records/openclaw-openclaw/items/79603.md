## Summary

Keep open: current main still omits last-completed handshake phase data from the Gateway WebSocket timeout and pre-connect close logs, and the live fixing path is the open, issue-closing PR https://github.com/openclaw/openclaw/pull/93402 rather than closing this issue before merge.

## What This Changes

Request Gateway WebSocket failure diagnostics to report the last completed handshake phase so operators can distinguish timeout, upgrade, auth, session attach, and ready-state stalls.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/93402 or a narrow equivalent Gateway-only fix, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection of current main shows the timeout and pre-connect close paths omit phase metadata; triggering a silent preauth timeout or close-before-connect path would hit those logs without a phase token.

## Solution Assessment

Yes. The narrow Gateway-local phase tracker in the linked PR is the best current direction because the missing signal lives inside the WebSocket connection and connect-message lifecycle, not in docs, clients, config, or provider routing.
