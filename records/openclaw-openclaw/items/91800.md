## What This Changes

This PR adds optional externalContent provenance to before_tool_call events and contexts, propagates it through embedded tool hook contexts and run state, and updates plugin hook docs and tests.

## Best Possible Solution

Parse the production id-based wrapper format or trusted tool-result metadata, update tests and proof to exercise real web_fetch/wrapExternalContent output, and land the optional hook contract only after maintainer API/security approval.

## Reproduction Assessment

Yes, at source level. Production wrapExternalContent emits id-based markers while the PR detector only accepts BEGIN markers, so real web_fetch output will not seed the new provenance signal.

## Solution Assessment

No. The propagation shape is plausible, but it is not the best fix until it recognizes production wrappers or trusted metadata and maintainers approve the public hook API semantics.
