## Best Possible Solution

Make result provenance scanning strictly best-effort for arbitrary tool outputs, then merge only after maintainer approval of the public provenance API/security semantics and green current-head checks.

## Solution Assessment

No, not as-is. Propagating a best-effort provenance signal is a plausible incremental solution, but the scanner must not be able to fail a successful tool result and the public hook contract still needs maintainer approval.
