## Best Possible Solution

Match the detector and regression tests to the production `wrapExternalContent` marker format, guard traversal as best-effort, keep the shared run-context propagation, and land only after maintainer approval of the public security/API semantics on a clean merge base.

No, not as-is. The propagation layer is reasonable, but the detector and regression tests must use the production wrapper contract and traversal must be best-effort before the public hook API can be considered the best fix.
