## Summary

Keep open: the PR is a useful security-boundary direction, but the current patch is not merge-ready because the embedded-run provenance is seeded too early and can miss external content returned by a tool earlier in the same run; it also lacks real behavior proof and public hook docs for the new SDK field.

## What This Changes

The PR adds optional external-content provenance to plugin tool hook types and propagates it through before_tool_call, trusted policy events, and embedded-run tool hook context.

## Best Possible Solution

Refresh or carry external-content provenance at the same boundary that adds tool results and prompt-build context to the model input, document the new hook field, and prove the fetch-then-act flow in a real run before merge.

## Reproduction Assessment

No: the PR is a plausible start, but the best fix must update provenance as the model context changes during the run, not only at initial embedded-run setup.
