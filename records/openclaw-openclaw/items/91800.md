  - body: `wrapExternalContent` emits markers like `<<<EXTERNAL_UNTRUSTED_CONTENT id="...">>>`, but this detector only scans a marker line when it also contains ` BEGIN>>>`.   - body: The wrapper refreshes provenance after the underlying tool succeeds, but the scanner then calls `Object.values` on arbitrary result objects. A getter or proxy trap can throw here and turn a successful tool call into a wrapper failure, so unreadable properties need to be caught or skipped.
  - confidence: 0.9

  - body: This only seeds from `params.prompt`, while the model prompt later includes `activeSession.messages`/`promptHistoryMessages`.
