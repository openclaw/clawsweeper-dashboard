## Label Justifications

- `P2`: The issue is a bounded outbound ownership hardening bug with medium latent impact and no known legitimate reachable trigger today.
- `impact:session-state`: The mismatch can associate outbound hooks, transcript mirroring, and bookkeeping with a different agent session than the explicit owner.
- `impact:security`: The split owner can scope media roots, account policy, or action dispatch to one agent while session-bound behavior references another.

## Summary

Keep this issue open: current main still permits the reported split explicit-agent/session-key ownership shape, and the linked open pull request is the canonical implementation candidate rather than a reason to close before merge.

## What This Changes

Reject outbound send/action paths that combine an explicit `agentId` with an agent-scoped `sessionKey` owned by a different agent before dispatch, queueing, mirroring, or hook correlation.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/96078; close this issue only after the chosen fix is merged and has source, test, and release or main-only provenance.

## Reproduction Assessment

Yes. Source inspection shows current main can combine explicit `agentId` with a differently owned agent-scoped `sessionKey`, and the existing Gateway test expects that split-owner case to succeed.

## Solution Assessment

Yes at the issue level. The maintainable fix is a shared owner-mismatch guard at Gateway and lower outbound boundaries, while keeping `policyKey` out of canonical owner checks.
