- `impact:session-state`: A mismatched owner can associate hooks, transcript mirroring, and bookkeeping with a different agent session than the explicit owner.

## Summary

Keep this issue open: current main still accepts the reported split-owner shape, and the linked open pull request is the implementation candidate rather than a reason to close before merge.

## What This Changes

The issue asks OpenClaw to reject outbound send/action requests when an explicit agentId disagrees with an agent-scoped session key while preserving legacy, same-agent, and policyKey cases.

## Best Possible Solution

Merge a reviewed implementation that enforces owner/session agreement at the Gateway and lower outbound boundaries, excludes policyKey from owner equality, and then close this issue with source and test provenance.

## Reproduction Assessment

Yes. Source inspection shows current main accepts and forwards explicit agent ownership separately from agent-scoped session keys, and the Gateway test suite currently expects that split-owner case to succeed.

## Solution Assessment

Yes at the fix-boundary level: a shared guard at both Gateway and lower outbound boundaries is the maintainable shape, while a Gateway-only guard would leave internal bypasses. The exact linked PR still needs normal PR review and proof before merge.
