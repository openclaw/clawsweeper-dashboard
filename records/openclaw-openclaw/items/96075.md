## Summary

Keep this issue open: current main still accepts the reported split-owner outbound shape, and the linked open pull request is the canonical implementation candidate rather than a reason to close before merge.

## What This Changes

The issue asks OpenClaw to reject outbound send/action paths when an explicit agent owner disagrees with an agent-scoped session key owner.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96078 or an equivalent shared guard, then close this issue with source, test, and release or main-only provenance.

## Reproduction Assessment

Yes. Source inspection shows current main resolves explicit agentId before the session-key owner and the current Gateway test suite expects the split-owner case to succeed.

## Solution Assessment

Yes. A shared guard at Gateway and lower outbound boundaries is the right shape because a Gateway-only check would still leave internal helpers able to construct split-owner operations.
