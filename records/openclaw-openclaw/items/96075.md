- `impact:session-state`: A mismatch can associate outbound hooks, transcript mirroring, and bookkeeping with a different agent session than the explicit owner.

## Summary

Keep this issue open: current main still allows explicit outbound agent ownership to diverge from an agent-scoped session key, and the linked closing pull request is open but unmerged.

## What This Changes

The issue requests rejecting outbound send/action calls when explicit `agentId` and agent-scoped `sessionKey` owners disagree, while preserving legacy, same-agent, and `policyKey` cases.

## Best Possible Solution

Review and land the linked guard if maintainers accept the stricter validation, then close this issue with the merged commit and proof.

Yes. Source inspection shows current main resolves explicit `agentId` before the session-key owner in Gateway and lower outbound paths; I did not run a live failing repro in this read-only review.

## Solution Assessment

Yes for the requested direction: a shared guard at Gateway and lower outbound boundaries is the right boundary because lower helpers can bypass Gateway validation. The exact linked PR still needs normal maintainer review and compatibility acceptance before merge.
