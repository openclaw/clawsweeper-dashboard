## Summary

Keep this issue open: current main still leaves the split owner/session-key shape representable in outbound paths, and the linked closing PR is still open as the implementation candidate rather than a reason to close before merge.

## What This Changes

The issue asks OpenClaw to reject outbound send/action requests when an explicit `agentId` disagrees with the owner segment of an agent-scoped session key.

## Best Possible Solution

Land a shared owner-mismatch guard across Gateway send/action and lower outbound helper boundaries, excluding `policyKey` from canonical owner equality, then close this issue with the merged commit proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Gateway send/action and lower outbound helpers can combine explicit agent ownership with a separately owned agent-scoped session key without rejecting the mismatch; I did not run a live failing command in this read-only review.

## Solution Assessment

Yes for the end-state boundary, but the exact patch should be judged in the linked PR. A shared guard at both Gateway and lower outbound boundaries is narrower and safer than a Gateway-only check because internal callers can otherwise bypass the invariant.
