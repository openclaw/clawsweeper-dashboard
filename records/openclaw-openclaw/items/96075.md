## Summary

Keep this issue open: current main still allows the reported split-owner outbound shape, and the linked implementation pull request is open and unmerged rather than a reason to close before merge.

## What This Changes

The issue asks OpenClaw to reject outbound sends/actions when an explicit agent owner disagrees with the owner encoded in an agent-scoped session key.

## Best Possible Solution

Review and land the linked implementation candidate if maintainers accept fail-closed validation for contradictory owner pairs, then close this issue from the merged fix provenance.

## Reproduction Assessment

Yes. Source inspection shows current main accepts and forwards explicit agent ownership separately from agent-scoped session keys, and the Gateway test suite currently expects that split-owner case to succeed.

## Solution Assessment

Yes for the desired issue boundary. A shared owner-mismatch guard at Gateway and lower outbound boundaries is the maintainable shape, with `policyKey` excluded from canonical-owner comparison.
