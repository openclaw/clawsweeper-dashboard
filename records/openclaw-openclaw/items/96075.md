## Label Justifications

- `P2`: The report is a bounded outbound ownership hardening bug with no known legitimate reachable trigger today.
- `impact:session-state`: The mismatch can associate outbound hooks, transcript mirroring, and bookkeeping with a different agent session than the explicit owner.
- `impact:security`: The split owner can scope media roots, account policy, or action dispatch to one agent while session-bound behavior references another.

## Summary

Keep this issue open: current main still allows the reported split-owner shape, and the linked open pull request is the canonical implementation candidate rather than a reason to close before merge.

## What This Changes

Reject outbound send/action requests when an explicit agent owner disagrees with the owner segment of an agent-scoped session key, while preserving same-agent, legacy, and policy-key delivery cases.

## Best Possible Solution

Review the linked fix PR as the implementation path, then close this issue only after a merged fix has source, test, and release or main-only provenance.

## Reproduction Assessment

Yes. Source inspection shows current main accepts and forwards explicit agent ownership separately from agent-scoped session keys, and the Gateway test suite currently expects the split-owner case to succeed.

## Solution Assessment

Unclear whether the linked PR is the best exact implementation until that patch receives full review, but the desired fix boundary is right: reject owner mismatches before dispatch, media policy, queueing, mirroring, and hook correlation while excluding `policyKey`.
