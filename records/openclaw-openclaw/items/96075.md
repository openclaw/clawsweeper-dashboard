## Summary

Keep open: current main still allows contradictory explicit agent ownership and agent-scoped session keys to be combined on outbound send/action paths, and the open closing PR is the candidate fix rather than a reason to close this issue early.

## What This Changes

Request that outbound send/action helpers reject mismatched explicit agent IDs and agent-scoped session keys before dispatch, media policy, queueing, mirroring, or hooks.

## Best Possible Solution

Review and land the linked guard PR or an equivalent focused fix that covers Gateway and lower outbound boundaries while keeping `policyKey` out of canonical owner comparisons.

## Reproduction Assessment

Yes, at source level: a synthetic Gateway or lower-helper call can supply `agentId: "work"` with `sessionKey: "agent:main:..."`, and current main propagates those fields without comparing the agent owner segment. I did not run a live runtime repro because this review is read-only and the report is a latent invariant hardening case.

## Solution Assessment

Yes, the requested shared owner-mismatch guard is the narrow maintainable shape because the same invariant crosses Gateway, action dispatch, send helpers, delivery, mirroring, and hooks. The linked PR should be reviewed as the implementation candidate rather than closing this issue before merge.
