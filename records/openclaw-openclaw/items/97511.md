## Summary

Keep open: the PR is aimed at the right shared presentation layer, but it still misses the documented raw progress-detail shape and lacks contributor-side real behavior proof.

## What This Changes

The PR changes exec-like failure warning formatting so the tool label is outside the command subject, strips selected friendly `run ...` labels, appends concise exit-code suffixes, and adds embedded-runner payload tests.

## Best Possible Solution

Keep the shared failure label outside command text, but first extract a trailing raw inline command before stripping display summaries and add real user-path proof.

## Reproduction Assessment

Yes, at source level: current main records exec display metadata as `lastToolError.meta` and the failure payload wraps that metadata in the visible warning. The raw-mode edge is also source-reproducible because `resolveExecDetail` can append `summary · ` plus an inline-code raw command.

## Solution Assessment

No. The layer is right, but the implementation needs to split the trailing raw inline command before applying `run ...` stripping, or carry structured display/raw command fields instead of reparsing formatted text.
