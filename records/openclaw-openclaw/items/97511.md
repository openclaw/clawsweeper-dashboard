Keep open: the PR is a useful shared presentation-layer fix for the exec failure warning ambiguity, but PR head still misses raw-mode literal commands that contain backticks, so the warning can fall back to wrapping display metadata plus the raw fragment again.

## What This Changes

This PR changes embedded-runner exec/bash failure warnings and related tests so the tool label, failure state, command subject, context, and concise exit code render separately.

## Best Possible Solution

Keep the shared payload-warning fix, but make raw exec command extraction delimiter-aware or carry structured raw command fields, then add regressions for literal commands containing backticks before merge.
