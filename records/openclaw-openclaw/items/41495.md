## Summary

Keep open: current main and the latest release still expose a Telegram inline-button prompt/schema/runtime mismatch that plausibly leaves Gemini responses as raw JSON text instead of interactive buttons.

## What This Changes

Render Gemini-generated Telegram inline-button responses as interactive buttons rather than raw JSON text.

## Best Possible Solution

Align the system prompt and Telegram docs with the existing `presentation.blocks[].buttons` contract, then add regression coverage tying the guidance to the message tool schema and Telegram renderer.

## Reproduction Assessment

No live Gemini run was performed in this review. Source inspection gives a high-confidence reproduction of the contract mismatch: the prompt directs `buttons=[[...]]` while the tool schema and Telegram runtime expect `presentation` or `interactive`.

## Solution Assessment

Yes for a narrow repair: update prompt/docs/tests to use the existing presentation payload. Parsing arbitrary raw `[[...]]` JSON text would be a broader product decision and should not be the default fix.
