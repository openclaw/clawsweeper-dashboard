## Summary

Keep open. Current main has a structured Telegram inline-button path, but the model-facing prompt still advertises a `buttons=[[...]]` shape while the message tool schema exposes `presentation` and no `buttons`; raw assistant text containing the reported JSON block is not converted into an inline keyboard.

## What This Changes

The issue asks OpenClaw to prevent Gemini text responses from exposing Telegram inline-button JSON instead of rendering buttons.

## Best Possible Solution

Align the model prompt, message tool schema guidance, Telegram docs, and regression tests around the structured `presentation` button payload, while avoiding a free-text parser that turns arbitrary assistant prose into Telegram controls.

## Reproduction Assessment

Yes from source inspection. A raw assistant reply containing the reported bracketed button JSON is parsed as ordinary text, while the actual Telegram button renderer only consumes structured `presentation` or `interactive` payloads.

## Solution Assessment

Yes, a narrow prompt/schema/docs alignment is the best fix path. Parsing model prose into button actions would conflict with the existing shared presentation contract and is a riskier behavior change.

## Repair Work Prompt

Fix the Gemini/Telegram inline-button prompt mismatch reported in https://github.com/openclaw/openclaw/issues/41495. Keep the fix narrow: make agent-facing guidance tell models to use the `message` tool with a structured `presentation` buttons block for Telegram inline buttons, not `buttons=[[{text,callback_data,...}]]` in visible text. Do not add a parser that converts arbitrary assistant prose into tool execution or Telegram controls. Audit Telegram/channel docs for the legacy native `buttons` example and align user-facing docs with the shared presentation contract. Add/update regression coverage for the system prompt and message tool schema so `presentation` is advertised and `buttons` is not implied as a top-level message-tool parameter. Validate the Telegram structured path still renders inline keyboards through existing tests.
