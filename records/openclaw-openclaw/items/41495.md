## Summary

Keep open: current main still advertises raw `buttons=[[...]]` in the shared agent prompt while the message tool, docs, and Telegram renderer expect `presentation.blocks[].buttons`. The adjacent open group-prompt PR is useful context but does not close this issue because it only changes group-chat prompt wording and has no closing reference.

## What This Changes

The issue reports Gemini model replies showing Telegram inline button JSON as plain message text instead of sending interactive inline buttons through the message tool.

## Best Possible Solution

Align the shared agent prompt and regression tests with the existing `presentation.blocks[].buttons` contract, and avoid parsing raw final-text `[[ ... ]]` JSON as the primary fix.

## Reproduction Assessment

Yes for a source-level reproduction: current main's prompt advertises raw `buttons=[[...]]`, while the current tool schema and Telegram runtime use `presentation.blocks[].buttons`. I did not run a live Gemini plus Telegram scenario in this read-only review.

## Solution Assessment

Yes for a narrow prompt/schema alignment repair; no for raw text JSON parsing, because that would add a second source of truth outside the shared presentation contract.
