## Best Possible Solution

Adjust the shared group-chat prompt to distinguish ordinary same-group text replies from explicit typed `message(action=send)` presentation controls, update the matching prompt tests/fixtures, and prove rendered buttons plus duplicate suppression in a live Gemini/Telegram group.

## Solution Assessment

Yes, the narrowest maintainable fix is model-agnostic prompt clarification using the existing typed presentation contract. Parsing arbitrary `[[...]]` JSON from assistant text would violate the typed-action/channel boundary and create an unsafe competing syntax.
