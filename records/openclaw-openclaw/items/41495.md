Keep open: current main still has the source-level conflict described in the report, where automatic group-chat guidance discourages the same-group `message` tool call that the existing Telegram presentation pipeline needs for inline buttons. ## What This Changes

The issue reports that Gemini-backed Telegram group replies emit inline-button JSON as visible text instead of using the typed message/presentation tool path.

## Best Possible Solution

Add a capability-aware group prompt exception for Telegram or other presentation-capable channels, preserve ordinary automatic text replies elsewhere, and verify native inline buttons plus no duplicate final reply in a live Gemini/Telegram group.

## Reproduction Assessment

Yes at source level: current automatic group prompts prohibit the same-group `message` tool call required by the already-supported typed Telegram presentation path. I did not run a live Gemini/Telegram reproduction during this read-only review.

## Solution Assessment

Yes, the best direction is a capability-aware prompt clarification using the existing typed presentation contract. Parsing arbitrary raw JSON button text from assistant output would create a competing syntax and bypass the channel presentation boundary.
