Related PR #53524 tracks a broader channel-plus-thread variant and should be considered during follow-up.

## What This Changes

The PR adds a `SessionEntry.postRotationStartupUntilMs` window for Discord thread resets, wraps queued owner steer prompts during that window, clears it after runs, and adds focused session/runner regression tests.

## Best Possible Solution

Land a rebased or replacement session-layer fix that keeps startup recovery pending until a successful recovery turn, includes regression coverage and a changelog entry, and explicitly coordinates whether channel coverage from #53524 is folded in or handled separately.

## Reproduction Assessment
