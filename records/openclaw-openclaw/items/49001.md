## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve active run session IDs when steering:** `src/auto-reply/reply/agent-runner.ts:264`
  - body: This queues the wrapped prompt with `followupRun.run.sessionId`, but current main first resolves the active embedded run id through `replyRunRegistry.resolveSessionId(sessionKey)`. After session rotation, the follow-up run id can be stale, so the queued owner message may target the wrong embedded run.
  - confidence: 0.86
- **[P2] Reserve the new SessionEntry key:** `src/config/sessions/types.ts:96`
  - body: The PR adds `postRotationStartupUntilMs` as a core top-level `SessionEntry` field but does not reserve that key in `src/plugins/session-entry-slot-keys.ts`. Plugin session-extension slots are guarded by that list, so leaving the new core key unreserved allows a plugin slot collision with the startup-window state.
  - confidence: 0.82
