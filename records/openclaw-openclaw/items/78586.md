## Summary

Keep open: current main does not already provide this CLI/HTTP admin abort surface, and the PR still has blocking review findings plus missing real behavior proof on a privileged Gateway path.

## What This Changes

The PR adds an admin-scoped `openclaw sessions abort <session-key> [--force]` command, a Gateway HTTP abort endpoint, abort metadata, diagnostics, docs, changelog, and unit tests.

## Best Possible Solution

Keep the feature under review, route it through the scoped Gateway RPC or explicitly define the trusted shared-secret HTTP contract, preserve existing session-route hardening, and require live proof that an orphaned session can be aborted without restarting the Gateway.

## Reproduction Assessment

Yes for the PR review findings by source inspection: the new CLI sends bearer-authenticated HTTP requests, and the new handler uses the current shared-secret scope resolver that returns no operator scopes for that path. I did not run tests because this is a read-only review and the checkout must remain clean.

## Solution Assessment

No for the current patch. The feature direction may be useful, but the implementation should use the scoped Gateway RPC or an explicitly defined trusted shared-secret HTTP contract and should match existing session-kill route parsing hardening.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor shared-secret admin auth on remote aborts:** `src/gateway/sessions-abort-http.ts:139-142`
  - body: For remote gateways using `gateway.remote.token` or `--token`, shared-secret HTTP auth sets `trustDeclaredOperatorScopes` false, so this resolver returns no scopes and the new `sessions.delete` gate rejects before aborting. Route the CLI through the scoped Gateway RPC or use an explicit trusted operator-scope resolver for this endpoint.
  - confidence: 0.9
- **[P2] Parse abort routes without trusting Host:** `src/gateway/sessions-abort-http.ts:111`
  - body: This constructs the URL base from `req.headers.host`, so a malformed Host header such as `[` can throw before auth or route handling. The existing session-kill handler uses a fixed localhost base for path parsing; use the same pattern here.
  - confidence: 0.84
- **[P3] Return an invalid-key error instead of falling through:** `src/gateway/sessions-abort-http.ts:112-114`
  - body: The server already routed `/api/sessions/[^/]+/abort` here, but an invalid percent-encoded or blank decoded key makes the handler return `false`, so the matched admin endpoint falls through instead of returning a clear 400. Mirror the session-kill resolver that distinguishes no-match from invalid session key.
  - confidence: 0.78
