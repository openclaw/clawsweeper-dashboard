## Summary

Keep open: current main does not have this admin CLI/HTTP abort surface and the PR is recent, but it should not merge yet because source inspection confirms functional auth/route regressions and the external real behavior proof gate is still unmet.

## What This Changes

The PR adds an admin-scoped `openclaw sessions abort <session-key> [--force]` command, a Gateway HTTP abort endpoint, abort metadata/diagnostics, docs, changelog, and unit tests.

## Best Possible Solution

Keep the feature under review, route the CLI through a scoped Gateway RPC or an explicitly defined trusted shared-secret admin HTTP contract, mirror existing session-kill route hardening, and require live proof of aborting an orphaned session without a Gateway restart.

## Reproduction Assessment

Yes for the review findings by source inspection: the proposed CLI sends bearer-authenticated HTTP requests, while the new handler uses a current shared-secret scope resolver that returns no operator scopes for that path and parses routes from `Host`. I did not run tests because this was a read-only review.

## Solution Assessment

No. The feature direction may be useful, but the current patch would fail the remote token/password CLI path and reintroduces route parsing gaps already hardened on the existing session-kill endpoint.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor shared-secret admin auth on remote aborts:** `src/gateway/sessions-abort-http.ts:139-142`
  - body: For remote gateways using `gateway.remote.token` or `--token`, the new CLI posts bearer-authenticated HTTP requests, but the handler resolves non-local scopes with `resolveTrustedHttpOperatorScopes` and then requires `sessions.delete`. Current shared-secret bearer auth intentionally returns no trusted declared scopes there, so this path rejects with 403 before aborting; route it through the scoped Gateway RPC or define an explicit trusted shared-secret admin contract for this endpoint.
  - confidence: 0.9
- **[P2] Parse abort routes without trusting Host:** `src/gateway/sessions-abort-http.ts:111`
  - body: This constructs the URL base from `req.headers.host`, so a malformed Host header such as `[` can throw before auth or route handling. The existing session-kill handler uses a fixed localhost base for path parsing; use the same pattern here.
  - confidence: 0.84
- **[P3] Return an invalid-key error instead of falling through:** `src/gateway/sessions-abort-http.ts:112-114`
  - body: The server already routed `/api/sessions/[^/]+/abort` here, but an invalid percent-encoded or blank decoded key makes the handler return `false`, so the matched admin endpoint falls through instead of returning a clear 400. Mirror the session-kill resolver that distinguishes no-match from invalid session keys.
  - confidence: 0.78
