## Summary

Keep this PR open for normal review: it is a focused fix for a real CLI config parsing bug, with targeted regression tests and sufficient real CLI proof. The only maintainer-visible concern is the intentional compatibility change from silently normalizing malformed paths to failing closed.

## What This Changes

This PR updates `src/cli/config-cli.ts` to reject config paths with empty dot segments before any config read/write and adds regression tests for `config set/get/unset`.

## Best Possible Solution

Land the fail-closed parser guard with the regression tests once CI is green and maintainers accept treating empty dot segments as invalid input rather than a compatibility path.

## Reproduction Assessment

Yes. Source inspection shows current `parsePath` drops empty dot segments, and the linked issue/PR body gives concrete `openclaw config set gateway..port` and `openclaw config get .gateway.port` examples; I did not run commands because this review is read-only.

## Solution Assessment

Yes. Tightening the existing CLI parser is the narrowest maintainable fix, and the tests verify malformed paths fail before reads or writes while valid bracket and escaped-dot paths remain in scope.
