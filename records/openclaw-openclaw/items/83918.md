## Summary

Keep open: current main still has the raw `iconMarkup` to `innerHTML` toolbar helper, and the linked closing PR is open and unmerged. Source review found only hardcoded current icon callers, so this remains security hardening rather than a proven live exploit.

## What This Changes

The issue asks OpenClaw to harden the diffs viewer toolbar helper so raw icon markup cannot become an XSS sink if a future caller passes attacker-controlled data.

## Best Possible Solution

Land a focused diffs viewer change that closes toolbar icons to a known internal set or DOM-created SVGs, preserves the existing visible icons in the regenerated runtime, and verifies the served viewer path.

## Reproduction Assessment

No high-confidence exploitable reproduction path is present. Source inspection does confirm the current raw `iconMarkup` to `innerHTML` sink, while all current callers are hardcoded same-file icon factories.

## Solution Assessment

Yes, the closed icon-name or DOM-created SVG direction is the right maintainable fix; this issue should stay open until the linked PR lands or maintainers choose a different focused hardening patch.
