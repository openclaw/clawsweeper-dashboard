## What This Changes

Exports the existing sensitive-header predicate, redacts standalone proxy request, response, and CONNECT capture headers before persistence, and adds helper-level redaction tests.

## Best Possible Solution

Land the shared redaction policy across standalone and runtime capture paths, keeping sensitive header values out of persisted debug proxy rows by default.

## Reproduction Assessment

## Solution Assessment

Yes: reusing the runtime sensitive-header predicate and redacting immediately before standalone persistence is the narrow maintainable fix. A live CONNECT proof would add confidence, but it does not point to a different implementation.
