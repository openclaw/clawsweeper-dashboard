## Summary

Keep open for maintainer merge review: current main and the latest release still persist raw standalone debug-proxy headers, while this PR is a focused, proof-backed candidate fix with no blocking code findings found.

## What This Changes

The PR adds shared proxy-capture header redaction and applies it before standalone HTTP request, HTTP response, and CONNECT capture headers are stored.

## Best Possible Solution

Land the shared storage-time redaction for all new standalone proxy captures, and make the intentional loss of raw sensitive header values clear in maintainer merge context.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw proxy start` stores raw HTTP response, HTTP request, and CONNECT headers in `headers_json`; the PR body also supplies a real proxy run and SQLite query showing the after-fix redaction.

## Solution Assessment

Yes. Redacting at the standalone proxy persistence sites with the same sensitive-header predicate used by runtime capture is the narrow maintainable fix; retroactive cleanup of old local captures is a separate scope decision, not a blocker for this patch.
