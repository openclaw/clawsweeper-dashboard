## Label Justifications

- `P2`: The reported failure is a provider-auth reliability bug that can require operator re-authentication but is already fixed on current main.
- `impact:auth-provider`: The issue centers on OAuth refresh token rotation and provider credential resolution.

## Summary

Current main already implements the requested OAuth refresh serialization at the shared auth runtime boundary, with a per-provider/profile in-process queue, a cross-agent refresh lock, and regression tests for concurrent shared-profile refresh bursts.

## What This Changes

The issue requests per-provider/profile single-flight OAuth token refresh so concurrent runtime turns cannot reuse a rotated refresh token.

## Best Possible Solution

Keep the shared OAuthManager queue and cross-agent lock as the canonical fix; provider refresh hooks should continue to stay thin and rely on this core auth boundary.

## Reproduction Assessment

Yes. The pre-fix failure is source-reproducible from concurrent expired OAuth profile resolution, and current tests model shared-profile races directly.

## Solution Assessment

Yes. The current implementation serializes at the shared core auth boundary and leaves provider plugins responsible only for the token exchange, which is the narrow maintainable fix shape.
