## Summary

Keep open: the Discord debounced-message mention loss is still source-reproducible on current main, but this PR needs to be ported to the moved Discord plugin handler and current prototype-preserving synthetic-message implementation before it can merge.

## What This Changes

The PR materializes mentioned users, roles, and everyone flags across debounced Discord entries before preflighting the synthetic message.

## Best Possible Solution

Port the all-entry mention merge into `extensions/discord/src/monitor/message-handler.ts` while preserving the current Carbon prototype/descriptors, then add focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes from source inspection. Two debounceable Discord messages from the same author/channel where an earlier entry mentions the bot and the last entry does not will build a synthetic message backed by the last entry, while preflight relies on mention getters that read only that backing data.

## Solution Assessment

No as-is. The PR targets the right bug, but the maintainable fix is to apply the all-entry mention accumulation to the current `extensions/discord` handler without reverting the newer prototype-descriptor preservation.
