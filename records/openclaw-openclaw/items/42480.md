## What This Changes

The PR branch modifies the legacy Discord message handler to accumulate `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` across debounced synthetic messages.

## Best Possible Solution

Yes. A high-confidence source-level reproduction is a debounced Discord batch where an earlier entry carries user, role, or everyone mention metadata and the final entry does not; current main builds the synthetic message metadata from the final entry, so preflight can miss the mention.

No. The proposed logic is the right narrow direction after the follow-up commit, but this PR is not the best mergeable patch because it applies to the removed legacy handler instead of the active Discord plugin and its raw-data-backed getters.
