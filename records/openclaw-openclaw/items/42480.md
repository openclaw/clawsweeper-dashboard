## Summary

Keep open. The PR is aimed at a real Discord debounce mention bug, and the follow-up commit addressed the earlier review gap for roles/everyone, but the branch edits the obsolete legacy path while current main runs the Discord handler from the bundled plugin path.

## What This Changes

The PR branch modifies the legacy Discord message handler to accumulate `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` across debounced entries before constructing a synthetic message.

## Best Possible Solution

Port the mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` by constructing the debounced synthetic message with deduped raw `mentions`, deduped raw `mention_roles`, and `mention_everyone` set when any entry has it, then add focused plugin tests for earlier-entry mentions and empty-final-entry cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a debounced Discord batch where an earlier entry carries user, role, or everyone mention metadata and the final entry does not; current main builds the synthetic message from the final entry's raw data, so preflight can see no mention metadata.

## Solution Assessment

No. The idea is the right narrow fix, especially after the follow-up commit, but this PR is not the best mergeable implementation because it applies the merge to a legacy path instead of the active Discord plugin handler and raw-data-backed getters.
