## Summary

Keep open: the Discord debounce mention bug is still source-reproducible on current main, but this PR targets the removed `src/discord` handler and needs to be ported to the current plugin path before merge.

## What This Changes

The PR modifies the old Discord message handler to materialize mentioned users, roles, and `@everyone` across debounced synthetic messages.

## Best Possible Solution

Port the all-entry mention merge into `extensions/discord/src/monitor/message-handler.ts`, update the synthetic message backing data in a prototype-safe way, add focused queue regression coverage, and add an Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes, from source inspection. Two debounceable Discord messages where an earlier entry mentions the bot and the last entry does not will preflight a synthetic message backed by the last entry’s `_rawData`, while the mention getters used for gating read only that backing data.

## Solution Assessment

No as-is. The PR addresses the right bug, but the maintainable fix is to port it to the current `extensions/discord` handler and merge mention fields into the synthetic message without losing descriptor/prototype preservation.
