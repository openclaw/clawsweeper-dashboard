## Summary

Keep open: the reported Discord mention-loss bug is still source-reproducible on current main, but this PR edits the removed `src/discord` handler and still lacks after-fix real Discord proof.

## What This Changes

The branch adds all-entry user, role, and everyone mention aggregation when Discord debounce combines multiple messages into one synthetic message.

## Best Possible Solution

Port mention metadata aggregation to `extensions/discord/src/monitor/message-handler.ts` by merging user, role, and everyone mention fields into the synthetic message backing data before preflight, then add focused queue/preflight coverage and redacted real Discord proof.

## Reproduction Assessment

Yes, by source inspection. A debounced batch where an earlier Discord entry contains a user, role, or everyone mention and the last entry does not will pass only the last entry's backing raw mention data into preflight.

## Solution Assessment

No, not as currently proposed. Aggregating mention metadata is the right narrow fix, but it must be applied to the live `extensions/discord` handler's backing raw data instead of the removed `src/discord` path.
