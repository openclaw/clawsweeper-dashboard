## Summary

Keep open. The reported Discord mention bug is still source-reproducible on current main, but this PR edits the obsolete `src/discord` handler instead of the live Discord plugin handler and also lacks required real behavior proof.

## What This Changes

The PR changes the Discord debounce synthetic-message construction in `src/discord/monitor/message-handler.ts` to accumulate mentioned users, roles, and everyone flags across queued entries.

## Best Possible Solution

Port the mention aggregation into `extensions/discord/src/monitor/message-handler.ts`, update the synthetic message backing data in a prototype-safe way, and add focused queue regression coverage.

## Reproduction Assessment

Yes, from source inspection. Two debounced Discord entries where an earlier entry carries a bot, role, or everyone mention and the last entry does not will build a synthetic message whose mention getters still read last-entry backing data.

## Solution Assessment

No as-is. The maintainable fix is to apply this aggregation to the live `extensions/discord` handler while preserving the current descriptor/prototype-safe synthetic message construction.
