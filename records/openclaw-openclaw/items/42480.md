## Summary

Keep open. The Discord debounce mention bug is still valid on current main, and the author addressed the earlier review gap for roles/everyone, but this PR changes a legacy handler path that current main no longer contains, so it is not mergeable as the fix.

## What This Changes

The PR branch modifies the legacy Discord message handler to accumulate mentionedUsers, mentionedRoles, and mentionedEveryone across debounced synthetic messages.

## Best Possible Solution

Port the mention merge into the active Discord plugin by merging raw mentions, mention_roles, and mention_everyone across all debounced entries before preflight, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a debounced Discord batch where an earlier entry carries user, role, or everyone mention metadata and the final entry does not; current main builds the synthetic message metadata from the final entry, so preflight can miss the mention after text has already been rewritten.

## Solution Assessment

No. The proposed logic is the right narrow direction after the follow-up commit, but this PR is not the best mergeable patch because it updates the removed legacy handler instead of the active raw-data-backed Discord plugin.
