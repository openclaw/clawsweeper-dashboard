## Summary

Keep open. The Discord mention-loss bug is still source-reproducible on current main, but this PR edits the removed `src/discord` handler instead of the live plugin handler and has no after-fix real Discord proof, so it cannot merge as-is.

## What This Changes

The PR changes the old Discord message handler to collect user, role, and everyone mention metadata from all debounced entries before building a synthetic message.

## Best Possible Solution

Port the aggregation into `extensions/discord/src/monitor/message-handler.ts` by merging user, role, and everyone mention metadata into the synthetic message raw data before preflight, with focused coverage and redacted real Discord proof.

## Reproduction Assessment

Yes, by source inspection. Current main combines multiple Discord entries into one synthetic message while mention getters still read only the last entry's `_rawData`, so a mention present only in an earlier debounced entry can be dropped before preflight.

## Solution Assessment

No, not as currently proposed. Aggregating mention metadata is the right narrow fix, but this PR applies it to a removed `src/discord` path instead of the live Discord plugin handler and still needs real Discord proof.
