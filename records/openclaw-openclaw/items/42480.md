## Summary

Keep this PR open, but it is not mergeable as-is: the reported Discord mention-loss bug is valid, yet the branch modifies a removed `src/discord` path while current `main` still needs the fix in the live Discord plugin handler.

## What This Changes

The branch accumulates mentioned users, roles, and `@everyone` across debounced synthetic Discord messages in `src/discord/monitor/message-handler.ts`.

## Best Possible Solution

Port the mention aggregation into `extensions/discord/src/monitor/message-handler.ts`, merge users, roles, and everyone into the synthetic message raw data, and add focused regression coverage for debounced Discord mentions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: debounce two same-author Discord entries where an earlier entry has user, role, or everyone mention metadata and the last entry does not; current `main` passes preflight a synthetic message whose mention getters read only last-entry raw data.

## Solution Assessment

No as-is. The fix idea is the right narrow direction, but this branch applies it to a removed path and should instead merge the raw mention fields used by the live `extensions/discord` message getters.
