## Summary

Keep open: the reported Discord mention-loss bug is still source-reproducible on current main, but this branch targets the removed `src/discord` handler instead of the live Discord plugin path and has no after-fix real Discord proof.

## What This Changes

The PR adds all-entry user, role, and everyone mention aggregation to Discord debounced synthetic messages in `src/discord/monitor/message-handler.ts`.

## Best Possible Solution

Port mention metadata aggregation to the live Discord plugin handler by merging user, role, and everyone mention fields into the synthetic message backing data before preflight, with focused queue/preflight coverage and redacted real Discord proof.

## Reproduction Assessment

Yes, by source inspection. A debounced batch where an earlier Discord entry contains a user, role, or everyone mention and the last entry does not will pass only the last entry's backing raw mention data into preflight on current main.

## Solution Assessment

No, not as currently proposed. Aggregating mention metadata is the right narrow fix, but this PR applies it to the removed `src/discord` handler instead of the live `extensions/discord` handler and still needs real Discord proof.
