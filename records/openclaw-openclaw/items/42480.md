## Summary

Keep open. The reported Discord mention-gating bug is still source-reproducible on current main, but this PR patches the removed legacy handler path instead of the active Discord plugin, so it is useful source work but not mergeable as-is.

## What This Changes

The PR branch accumulates mentionedUsers, mentionedRoles, and mentionedEveryone across debounced synthetic Discord messages in the legacy `src/discord` handler.

## Best Possible Solution

Port the all-entry mention merge into `extensions/discord/src/monitor/message-handler.ts` by merging `rawData.mentions`, `mention_roles`, and `mention_everyone` before preflight, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a debounced Discord batch where an earlier entry has user, role, or everyone mention rawData and the final entry does not; current main builds the synthetic message rawData from the final entry, so preflight can miss the mention.

## Solution Assessment

No. The proposed mention accumulation is the right narrow fix direction, but this PR applies it to the removed legacy handler instead of the active `extensions/discord` plugin path.
