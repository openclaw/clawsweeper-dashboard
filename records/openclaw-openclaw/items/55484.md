## Summary

Keep open: current main and v2026.5.18 still lack the requested non-thread ACP persistent-affinity contract for `sessions_spawn(runtime="acp")`. The merged downstream implementation and https://github.com/openclaw/openclaw/pull/39372 are useful context, but there is no upstream implementation or merged PR that defines deterministic `affinityKey` ownership, reuse, delivery, and cleanup semantics.

## What This Changes

Support deterministic non-thread ACP persistent affinity for cron and orchestrator `sessions_spawn` calls.

## Best Possible Solution

Add a reviewed upstream ACP persistent-affinity contract scoped by trusted requester owner, backend, cwd, and `affinityKey`, with authorization checks, lifecycle cleanup, docs, and regression coverage.

## Reproduction Assessment

Yes, source-level. Current main accepts only `run` and `session`, has no `affinityKey`, rejects non-thread session mode, and creates random ACP session keys.

## Solution Assessment

Unclear. The downstream `mode="persistent"` plus `affinityKey` shape is useful prior art, but upstream needs maintainer approval for requester identity, authorization, deterministic key scoping, delivery, and cleanup semantics.
