## Summary

Keep open: current main and v2026.5.19 still lack the requested ACP persistent-affinity contract, and the related headless-session issue/PR do not implement deterministic `affinityKey` reuse for cron/orchestrator callers.

## What This Changes

The issue requests an ACP `sessions_spawn` contract for persistent non-thread affinity so cron, hook, or orchestrator sessions can reuse deterministic ACP sessions outside channel threads.

## Best Possible Solution

Add a reviewed upstream ACP persistent-affinity contract scoped by trusted requester owner, backend, cwd, and affinity key, with authorization checks, delivery/lifecycle cleanup semantics, docs, and regression coverage.

## Reproduction Assessment

Yes, source-level. Current main accepts only `run`/`session`, has no `affinityKey`, rejects non-thread ACP session mode, and creates random ACP child session keys.

## Solution Assessment

Unclear as a direct implementation path. The downstream `persistent` plus `affinityKey` shape is useful prior art, but upstream needs maintainer approval for requester identity, authorization, deterministic key scoping, delivery, and cleanup semantics.
