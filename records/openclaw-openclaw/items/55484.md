## Summary

Keep open: current main still lacks the requested non-thread ACP persistent-affinity contract for cron/orchestrator sessions, and the related open headless-session work does not cover deterministic affinity keys scoped by owner/backend/cwd.

## What This Changes

Request support for `sessions_spawn(runtime="acp", mode="persistent", affinityKey)` so cron, hook, or orchestrator callers can reuse a scoped ACP session without a chat thread.

## Best Possible Solution

Implement a reviewed upstream ACP persistent-affinity contract scoped by trusted requester owner, backend, cwd, and affinity key, with authorization, lifecycle cleanup, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `mode="persistent"` and `affinityKey` are not accepted or forwarded, `mode="session"` is still rejected without `thread=true`, and fresh ACP session keys remain random.

## Solution Assessment

Unclear. The downstream `mode="persistent"` plus `affinityKey` contract is a reasonable starting point, but the maintainable upstream solution needs maintainer approval for requester identity, authorization, cleanup, and deterministic key scoping.
