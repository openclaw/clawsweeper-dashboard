## Summary

Keep open. Current main still lacks the requested non-thread ACP persistent-affinity contract for `sessions_spawn`; the downstream WuKongAI-CMU implementation is useful prior art, while broader headless-session work such as #53548/#39372 does not cover `affinityKey` reuse or cron/hook requester scoping.

## What This Changes

The issue requests `sessions_spawn(runtime="acp")` support for non-thread persistent ACP reuse via a separate persistent mode and `affinityKey` for cron, hook, and orchestrator sessions.

## Best Possible Solution

Keep this as an upstream ACP design tracker and implement a reviewed `sessions_spawn` persistent-affinity contract scoped by trusted requester owner, backend, cwd, and affinity key, with authorization, cleanup, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions_spawn({ runtime: "acp", mode: "persistent", affinityKey: "..." })` has no schema/parser/runtime support, `mode="session"` without `thread=true` is rejected, and non-thread ACP child keys remain random.

## Solution Assessment

Unclear. The downstream separate `mode="persistent"` plus `affinityKey` shape is reasonable prior art, but upstream needs maintainer approval for requester identity, authorization, key scoping, and lifecycle cleanup before it is the best solution.
