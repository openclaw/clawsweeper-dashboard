## Summary

Keep open: current main and the latest shipped release still lack the requested non-thread ACP persistent-affinity contract. The downstream fork implementation linked in the discussion is useful prior art, but upstream still needs a product/security decision for deterministic ACP session reuse across cron, hook, or orchestrator callers.

## What This Changes

Request that `sessions_spawn(runtime="acp")` support non-thread persistent reuse for cron/orchestrator callers via a deterministic affinity key.

## Best Possible Solution

Implement a reviewed upstream ACP persistent-affinity contract scoped by trusted requester owner, backend, cwd, and affinity key, with authorization checks, lifecycle cleanup, docs, and regression coverage.

## Reproduction Assessment

Yes for the missing capability. Source inspection shows current main accepts only `run` and `session`, does not accept or forward `affinityKey`, rejects non-thread `mode="session"`, and creates random ACP session keys.

## Solution Assessment

Unclear. The downstream contract is a reasonable starting point, but the best upstream solution needs maintainer approval for requester identity, authorization, deterministic key scoping, and cleanup semantics.
