## Summary

Keep open. Current main and v2026.6.8 still expose only `run`/`session` ACP spawn modes, still reject ACP `mode="session"` without `thread=true`, and have no `affinityKey` or non-thread persistent affinity path; the related hub-delegated PR explicitly leaves cron/hook affinity out of scope.

## What This Changes

Request a first-class ACP `sessions_spawn` persistence mode or affinity key so cron, hook, and orchestrator sessions can safely reuse non-thread ACP harness sessions.

## Best Possible Solution

Define an explicit cron/hook/orchestrator ACP affinity contract scoped by requester, backend, cwd, and affinity key, then update schema, runtime, docs, and regression tests while preserving the thread-bound `mode="session"` contract.

## Reproduction Assessment

Yes, source-level. Current main only allows ACP `run`/`session`, rejects `mode="session"` without `thread=true`, and exact search found no `affinityKey` path.

## Solution Assessment

Unclear. The downstream `mode="persistent"` plus `affinityKey` shape is plausible, but maintainers still need to choose the scoping, lifecycle, and relationship to the hub-delegated ACP path.
