## Summary

Keep open. Current main still lacks the requested non-thread ACP persistent-affinity contract for `sessions_spawn`; the linked WuKongAI-CMU downstream implementation is useful prior art, but upstream still needs a reviewed ownership, authorization, and cleanup design before this should be implemented.

## What This Changes

Add ACP `sessions_spawn` support for a deterministic non-thread persistent mode with `affinityKey` for cron, hook, and orchestrator callers.

## Best Possible Solution

Keep this as an upstream ACP design tracker and implement a reviewed contract that scopes deterministic affinity by trusted requester owner, backend, cwd, and affinity key, with authorization and lifecycle tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessions_spawn({ runtime: "acp", mode: "persistent", affinityKey: "..." })` has no schema/parser support or forwarding, while `mode: "session"` without `thread=true` is rejected and non-thread ACP sessions use random keys.

## Solution Assessment

Unclear. The downstream `mode="persistent"` and `affinityKey` shape is plausible, but the upstream fix should first settle requester identity, cross-workspace scoping, authorization, and cleanup semantics.
