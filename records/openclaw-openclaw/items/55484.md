## Summary

Keep open. Current main and the latest release still expose only `run`/`session` ACP spawn modes, still reject ACP `mode="session"` without `thread=true`, and have no `affinityKey`/non-thread persistent reuse path; the open hub-delegated PR explicitly leaves cron/hook affinity out of scope.

## What This Changes

The issue requests a persistent non-thread ACP affinity mode for cron/hook/orchestrator sessions so recurring or headless sessions can reuse the correct ACP session without thread binding.

## Best Possible Solution

Define a first-class cron/hook/orchestrator ACP affinity contract scoped by requester, backend, cwd, and affinity key, then implement schema, runtime, docs, and regression tests without relaxing thread-bound `mode="session"` semantics.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `sessions_spawn({ runtime: "acp", mode: "session" })` without `thread=true`, and exact searches show no `affinityKey` or `sessions_spawn` `mode="persistent"` path.

## Solution Assessment

Unclear. A separate persistent-affinity path is plausibly safer than allowing unthreaded `mode="session"`, but maintainers need to choose the permanent contract and how it relates to https://github.com/openclaw/openclaw/issues/53548 and https://github.com/openclaw/openclaw/pull/91093.
