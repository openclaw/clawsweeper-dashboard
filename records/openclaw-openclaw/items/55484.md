## Summary

Keep open. Current main and v2026.6.10 still lack an ACP `affinityKey` or `mode="persistent"` sessions_spawn contract, still expose ACP spawns as `run`/`session`, and still reject unthreaded ACP `mode="session"`; related cron and hub-delegated work does not implement the cron/hook affinity request.

## What This Changes

Add non-thread ACP persistent affinity for cron, hook, and orchestrator sessions using a deterministic affinity key while preserving the existing thread-bound session mode.

## Best Possible Solution

Define an explicit cron/hook/orchestrator ACP affinity contract scoped by requester identity, backend, cwd, and affinity key, then update schema, runtime, docs, and regression tests while keeping thread-bound `mode="session"` intact.

## Reproduction Assessment

Yes, source-level. Current main only accepts ACP `run`/`session`, rejects `mode="session"` without `thread=true`, generates random ACP child session keys, and has no `affinityKey` path.

## Solution Assessment

Unclear. The downstream `mode="persistent"` plus `affinityKey` shape is plausible, but maintainers still need to choose the ownership, lifecycle, security, and relation to hub-delegated or cron ACP work before implementation.
