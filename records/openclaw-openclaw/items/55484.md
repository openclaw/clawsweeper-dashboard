## Summary

Keep open. Current main still lacks the requested non-thread ACP persistent-affinity contract for `sessions_spawn`; the downstream WuKongAI-CMU implementation linked in the discussion is useful prior art, but the upstream ACP tool path still exposes only `run`/thread-bound `session` plus `resumeSessionId`.

## What This Changes

Support ACP `sessions_spawn` non-thread persistent affinity for cron and orchestrator callers via an explicit persistent mode and affinity key.

## Best Possible Solution

Keep this as an upstream design tracker and implement a reviewed ACP contract that scopes deterministic affinity by trusted requester owner, backend, cwd, and affinity key, with authorization and cleanup tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessions_spawn({ runtime: "acp", mode: "persistent", affinityKey: "..." })` has no schema/parser support, while `mode: "session"` without `thread=true` is rejected and non-thread ACP spawns use random child session keys.

## Solution Assessment

Unclear. The downstream implementation is useful prior art and the separate persistent-affinity mode is a reasonable shape, but the upstream solution needs maintainer approval of requester identity, authorization, cleanup, and key scoping first.
