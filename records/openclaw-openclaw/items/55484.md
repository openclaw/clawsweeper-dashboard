## Summary

Keep open: current main and v2026.5.18 still lack the requested non-thread ACP persistent-affinity contract for `sessions_spawn(runtime="acp")`. The downstream implementation and https://github.com/openclaw/openclaw/pull/39372 are useful context, but neither is merged upstream nor settles deterministic `affinityKey` ownership and reuse semantics.

## What This Changes

Requested support for a non-thread ACP persistent-affinity mode so cron, hook, or orchestrator sessions can reuse an ACP session by deterministic affinity key.

## Best Possible Solution

Add a reviewed upstream ACP persistent-affinity contract scoped by trusted requester owner, backend, cwd, and `affinityKey`, with authorization checks, lifecycle cleanup, docs, and regression coverage.

## Reproduction Assessment

Yes, source-level. Current main accepts only `run` and `session`, has no `affinityKey`, rejects non-thread session mode, and creates random ACP session keys.

## Solution Assessment

Unclear. The downstream `mode="persistent"` plus `affinityKey` shape is useful prior art, but upstream needs maintainer approval for requester identity, authorization, deterministic key scoping, delivery, and cleanup semantics.
