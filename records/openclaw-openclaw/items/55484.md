## Summary

Keep open. Current main and v2026.6.8 still expose only ACP `run`/`session` spawn modes, reject ACP `mode="session"` without `thread=true`, and have no `affinityKey` or cron/hook persistent-affinity path; the related hub-delegated PR explicitly excludes cron/hook affinity.

## What This Changes

Request ACP `sessions_spawn` support for non-thread persistent affinity, including deterministic cron/hook session reuse by owner/backend/cwd/affinity key.

## Best Possible Solution

Define an explicit cron/hook/orchestrator ACP affinity contract scoped by requester, backend, cwd, and affinity key, then update schema, runtime, docs, and regression tests while preserving the thread-bound `mode="session"` contract.

## Reproduction Assessment

Yes, source-level. Current main only allows ACP `run`/`session`, rejects `mode="session"` without `thread=true`, creates random ACP session keys, and exact search found no `affinityKey` path.

## Solution Assessment

Unclear. The downstream `mode="persistent"` plus `affinityKey` shape is plausible, but maintainers still need to choose the scoping, lifecycle, and relationship to the hub-delegated ACP path.
