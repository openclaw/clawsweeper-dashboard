## Summary

Keep open: current main still lacks the requested `sessions_spawn(runtime="acp")` non-thread persistent-affinity contract, while the linked downstream implementation remains useful prior art rather than an upstream fix.

## What This Changes

The issue requests `mode="persistent"` plus `affinityKey` support for non-thread ACP `sessions_spawn` calls from cron, hook, and orchestrator sessions.

## Best Possible Solution

Implement a reviewed upstream ACP persistent-affinity contract that scopes deterministic reuse by trusted requester owner, backend, cwd, and affinity key, with authorization, cleanup, and regression tests.

## Reproduction Assessment

Yes. Source inspection shows `sessions_spawn({ runtime: "acp", mode: "persistent", affinityKey: "..." })` has no schema/parser/runtime support, while `mode="session"` remains thread-bound and non-thread ACP child keys are random.

## Solution Assessment

Unclear. The downstream `mode="persistent"` plus `affinityKey` shape is a reasonable starting point, but the maintainable upstream solution needs maintainer approval for requester identity, authorization, cleanup, and key scoping.
