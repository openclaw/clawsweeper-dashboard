## Best Possible Solution

Refresh the PR to keep helper/runtime empty-card validation with focused tests while removing the raw-card send-action branch and preserving the shared `presentation` contract.

## Solution Assessment

No. Helper-level validation is the narrow maintainable fix, but this PR also imports runtime send code into the channel entrypoint and reopens raw `ctx.params.card` handling in send/thread-reply.
