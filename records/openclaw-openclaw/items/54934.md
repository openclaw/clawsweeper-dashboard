## Summary

Keep open; this PR contains useful Feishu helper hardening, but its channel send hunk is stale against current main's presentation-based message path and should not merge as-is.

## What This Changes

The PR adds Feishu empty-card validation in the message action dispatcher and send/edit helpers, plus unit tests for card-only rejection, text fallback, and direct helper rejection.

## Best Possible Solution

Preserve current presentation-based Feishu message sending and add empty-card rejection only at Feishu native helper/direct edit/send boundaries with focused tests.

Yes. Current-main source inspection gives a high-confidence split: `message(action=send, card={})` no longer reaches `sendCardFeishu`, while direct calls to `sendCardFeishu({ card: {} })` or `editMessageFeishu({ card: {} })` still follow unguarded helper paths.

## Solution Assessment

No. The helper guard is a good narrow hardening direction, but the channel action hunk should be dropped because current main no longer supports raw Feishu `card` fields in the shared message send path.
