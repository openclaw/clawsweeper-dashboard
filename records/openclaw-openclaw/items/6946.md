Keep open. Current main still lacks the requested standalone Telegram processing-placeholder message lifecycle and config surface, while the open waiting-tips PR overlaps the same UX gap, so this remains a feature/config product decision rather than cleanup-close material.

## What This Changes

Request an opt-in Telegram processing indicator that sends a visible placeholder message while work is running, deletes it after a reply, and edits or replaces it on failure.

## Best Possible Solution
