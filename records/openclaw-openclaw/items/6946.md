## Summary

Keep open. Current main still has adjacent Telegram typing cues, ack reactions, status reactions, and progress drafts, but it does not accept or drive the requested immediate standalone processing-indicator message lifecycle.

## What This Changes

The issue requests an opt-in Telegram processing indicator message that is sent while a turn is running, deleted on successful reply, and edited or replaced on failure.

## Best Possible Solution

Choose one first-class Telegram waiting UX, then ship an opt-in plugin change with typed config/schema/help/docs, send/store/delete wiring, failure edit-or-replace behavior, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that the requested config keys and standalone placeholder lifecycle are not present.

## Solution Assessment

Unclear until maintainer direction is set. A narrow Telegram processing indicator is technically feasible, but current progress drafts and the open waiting-tips pull request overlap the same user-visible feedback problem.
