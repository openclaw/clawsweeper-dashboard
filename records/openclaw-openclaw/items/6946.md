## Summary

Keep open. Current main still lacks the requested standalone Telegram processing-placeholder config and message lifecycle; existing early typing, ack/status reactions, and progress drafts do not send a visible placeholder that is later deleted or edited.

## What This Changes

Request an opt-in Telegram processing indicator that posts a visible placeholder while a turn is running, deletes it on success, and edits it on failure.

## Best Possible Solution

Choose one first-class opt-in Telegram waiting UX, then implement it in the Telegram plugin with typed config/schema/docs, send/store/delete wiring, failure edit-or-replace behavior, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that the exact standalone placeholder config and lifecycle are absent on current main.

## Solution Assessment

Unclear until maintainers choose the intended UX. A narrow processing indicator is technically feasible, but adding it automatically would create new user-facing Telegram behavior alongside existing typing, reaction, and progress-draft feedback paths.
