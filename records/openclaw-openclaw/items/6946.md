## Summary

Keep open. Current main still lacks the requested standalone Telegram processing-placeholder config and send/delete-or-edit lifecycle, and the active waiting-tips PR is still an unmerged product/config alternative rather than an implementation on main.

## What This Changes

The issue requests an opt-in Telegram processing message that is sent while a reply is being generated, then deleted or changed when the turn finishes.

## Best Possible Solution

Choose one first-class opt-in Telegram waiting UX, then implement it in the Telegram plugin with typed config/schema/help/docs, send/store/delete wiring, failure edit-or-replace behavior, and focused lifecycle tests.

## Solution Assessment
