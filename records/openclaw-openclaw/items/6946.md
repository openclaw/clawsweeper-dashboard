## Summary

Keep open. Current main still lacks the requested standalone Telegram processing-placeholder config and send/delete-or-edit lifecycle, and the only overlapping implementation path is the open, reviewed waiting-tips PR that still needs maintainer UX/config direction.

## What This Changes

The issue requests an opt-in Telegram processing indicator message that appears while Telegram replies are being prepared and is deleted or edited on completion or failure.

## Best Possible Solution

Choose one first-class opt-in Telegram waiting UX and implement it in the Telegram plugin with typed config/schema/help/docs, thread-aware send/store/delete wiring, failure edit-or-replace behavior, and focused lifecycle tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that the requested config keys and standalone placeholder lifecycle are absent.

## Solution Assessment

Unclear until maintainers set the intended UX. A narrow Telegram processing indicator is technically feasible, but current progress drafts are not the same immediate placeholder and the open waiting-tips PR is a broader, blocked alternative.
