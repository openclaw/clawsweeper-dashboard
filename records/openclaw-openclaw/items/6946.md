## Summary

Keep open. Current main has adjacent Telegram typing, status-reaction, and progress-draft feedback, but it still lacks the requested immediate standalone processing-indicator message config and lifecycle.

## What This Changes

Add an opt-in Telegram processing indicator that sends a visible placeholder while a request is being processed, then deletes it on success or edits it on failure.

## Best Possible Solution

Choose one first-class Telegram waiting UX, then ship an opt-in plugin change with typed config/schema/help/docs, placeholder send/store/delete wiring, failure edit-or-replace behavior, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that the requested config keys and standalone placeholder lifecycle are not present.

## Solution Assessment

Unclear until maintainer direction is set. A narrow Telegram processing indicator is technically feasible, but the newer progress-draft work and open #40421 waiting-tips PR overlap the same user-visible feedback problem.
