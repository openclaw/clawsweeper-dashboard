## Summary

Keep open. Current main has adjacent Telegram feedback mechanisms, but it still does not expose or drive the requested standalone processing-placeholder message lifecycle.

## What This Changes

The issue requests an opt-in Telegram processing message that appears while a reply is being prepared, is deleted after success, and is edited or replaced on failure.

## Best Possible Solution

Choose one first-class Telegram waiting-indicator UX, then ship an opt-in plugin change with typed config/schema/help/docs, placeholder send/store/delete wiring, failure edit-or-replace behavior, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that the requested config keys and standalone placeholder lifecycle are not present.

## Solution Assessment

Unclear until maintainer direction is set. A narrow Telegram `processingIndicator` is technically feasible, but #40421 proposes a broader waiting-tips UX and current streaming previews partially overlap the same user-visible feedback problem.
