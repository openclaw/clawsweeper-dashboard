## Summary

Keep open. Current main still has Telegram typing, ack reactions, and status reactions, but it does not accept or drive the requested standalone processing-placeholder message that is sent while work is running and deleted or edited on completion/failure; the open overlapping presence PR uses typing plus reactions rather than the requested message lifecycle.

Request an opt-in Telegram processing indicator that sends a visible placeholder while a request is processed, deletes it on successful reply, and edits or replaces it on failure.

## Best Possible Solution

Choose the intended Telegram waiting UX, then land one opt-in Telegram plugin change with typed config/schema/help/docs, authorization-aware send/store/delete-or-edit lifecycle handling, focused tests, and redacted live Telegram proof.
