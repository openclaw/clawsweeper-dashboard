## Summary

Keep open. Current main still lacks the requested Telegram standalone processing-placeholder message and accepted config surface; adjacent typing, ack/status reactions, and streaming drafts do not send a visible message that is deleted on reply or edited on failure.

## What This Changes

Add an opt-in Telegram processing indicator that posts a visible in-chat placeholder while work is running, then deletes it on successful reply or edits it on failure.

## Best Possible Solution

Choose the intended Telegram waiting UX, then land one opt-in Telegram plugin change with typed config/schema/help/docs, authorized send/store/delete-or-edit lifecycle handling, focused tests, and real Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection of current main gives high confidence that both the requested config keys and standalone placeholder-message lifecycle are absent.

## Solution Assessment

Unclear pending product decision. The placeholder approach is plausible, but maintainers should choose it against https://github.com/openclaw/openclaw/pull/80690, typing/status reactions, progress drafts, and broader silence-watchdog requests before implementation.
