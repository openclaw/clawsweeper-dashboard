## Summary

Keep open. Current main and v2026.5.19 still lack a standalone Telegram processing-placeholder config and lifecycle; existing typing, ack-reaction, status-reaction, and draft-preview paths do not send a temporary message that is later deleted or edited. The overlapping https://github.com/openclaw/openclaw/pull/80690 uses typing plus phased reactions, so it is related but not equivalent to the requested send/delete-or-edit message behavior.

## What This Changes

Request to add an opt-in Telegram processing indicator message that is sent while work runs, then deleted on successful reply or edited/replaced on failure.

## Best Possible Solution

Choose the intended Telegram waiting UX, then land one opt-in Telegram plugin change with typed config/schema/help/docs, authorization-aware send/store/delete-or-edit lifecycle handling, focused tests, and redacted live Telegram proof.
