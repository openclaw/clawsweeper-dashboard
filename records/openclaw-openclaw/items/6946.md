## Summary

Keep open. Current main still lacks the requested Telegram standalone processing-placeholder message lifecycle and config surface; existing typing, ack/status reactions, and draft streaming do not send a visible message that is later deleted or edited.

## What This Changes

Requests an opt-in Telegram processing indicator message that is sent while a Telegram turn is running, then deleted on success or edited on failure.

## Best Possible Solution

Choose the intended Telegram waiting UX, then implement an opt-in plugin-owned config/schema/docs path with authorized send/store/delete-or-edit lifecycle handling, focused tests, and real Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks both the requested config keys and the standalone placeholder-message lifecycle.

## Solution Assessment
