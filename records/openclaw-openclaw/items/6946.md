## Summary

Keep open. Current main and v2026.6.5 still provide typing, acknowledgment reactions, status reactions, and streaming previews, but no standalone Telegram message that is sent during processing and then deleted or edited. The related presence PR uses typing plus phased reactions, is currently conflicting, and does not safely supersede this distinct request, so maintainers still need to choose the intended UX and config boundary.

## What This Changes

Add an opt-in Telegram placeholder message that appears during processing and is deleted after a successful reply or changed when processing fails.

## Best Possible Solution

Define one opt-in Telegram placeholder-message contract and its interaction with existing presence signals, then implement the send, tracked cleanup, and failure-edit lifecycle inside the Telegram plugin with typed schema/help/docs, focused tests, and live Telegram proof.
