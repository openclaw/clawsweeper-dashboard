## Summary

Keep open. Current main still lacks the requested Telegram standalone processing-placeholder message lifecycle and accepted config surface; existing typing, ack/status reactions, and streaming drafts are adjacent but do not send a visible placeholder that is deleted on reply or edited on failure.

## What This Changes

Request an opt-in Telegram processing indicator that sends a visible in-chat placeholder while the bot is working, deletes it on successful reply, and edits or replaces it on failure.

## Best Possible Solution

Choose the intended Telegram waiting UX, then land an opt-in Telegram plugin change with typed config/schema/help/docs, authorized send/store/delete-or-edit lifecycle handling, focused tests, and real Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection of current main gives high confidence that both the requested config keys and standalone placeholder-message lifecycle are absent.

## Solution Assessment

Unclear pending product decision. The requested placeholder is plausible, but maintainers should choose it against Telegram progress drafts, status reactions, the open presence PR at https://github.com/openclaw/openclaw/pull/80690, and broader acknowledgment/silence-watchdog requests.
