## Summary

Keep open. Current main still has Telegram typing cues, ack/status reactions, and draft/tool-progress previews, but it does not accept or drive the requested opt-in standalone Telegram placeholder lifecycle; the overlapping waiting-tips PR remains open and direct processing-indicator PRs were closed unmerged.

## What This Changes

Request an opt-in Telegram processing indicator that sends a visible placeholder while a message is processed, deletes it on successful reply, and edits or replaces it on failure.

## Best Possible Solution

Choose one bundled Telegram feedback UX, then implement it inside the Telegram plugin with typed config/schema/help/docs, placeholder send/track/delete/edit wiring, and focused regression coverage.
