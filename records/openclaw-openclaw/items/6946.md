## Summary

Keep open. Current main still lacks the requested standalone Telegram processing-placeholder message lifecycle and config surface; the adjacent typing, ack/status reactions, and progress-draft paths do not provide the send-placeholder-then-delete-or-edit behavior, so this remains a Telegram UX/config product decision.

## What This Changes

Add an opt-in Telegram processing indicator message that appears while the bot is working and is deleted or edited when the reply completes.

## Best Possible Solution

Choose the intended Telegram waiting UX, then implement an opt-in plugin-owned config/schema/docs path with authorized send/store/delete-or-edit lifecycle handling, focused tests, and real Telegram proof.

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks both the requested config keys and the standalone placeholder-message lifecycle.
