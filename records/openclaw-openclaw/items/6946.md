## Summary

Keep open. Current main and v2026.5.22 still lack the requested standalone Telegram processing-placeholder message and config surface; existing typing, ack-reaction, status-reaction, and draft/progress paths are adjacent but do not implement the send-then-delete-or-edit message lifecycle.

## What This Changes

Add an opt-in Telegram processing indicator that sends a visible placeholder message while a turn runs, deletes it on successful reply, and edits or replaces it on failure.

## Best Possible Solution

Choose one Telegram waiting-feedback UX, then land an opt-in Telegram plugin change with typed config/schema/docs, send/store/delete-or-edit lifecycle handling, focused regression tests, and redacted live Telegram proof.

## Solution Assessment
