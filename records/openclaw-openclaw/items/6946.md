## Summary

Keep open. Current main and the latest shipped tag still lack the requested standalone Telegram processing-placeholder message lifecycle; existing typing, ack-reaction, and status-reaction paths do not send a placeholder that is later deleted or edited, and the open presence PR uses typing plus reactions instead of this behavior.

## What This Changes

The issue requests an opt-in Telegram processing indicator message that is sent while a request is being handled, deleted after a successful reply, and edited or replaced on failure.

## Best Possible Solution

Choose the intended Telegram waiting UX, then land one opt-in Telegram plugin change with typed config/schema/help/docs, authorization-aware send/store/delete-or-edit lifecycle handling, focused tests, and redacted live Telegram proof.
