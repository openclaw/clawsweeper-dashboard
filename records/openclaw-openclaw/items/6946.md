## Summary

Keep open. Current main still lacks the requested standalone Telegram processing-placeholder message lifecycle and does not accept the proposed config keys; adjacent typing, ack/status reaction, and progress-draft behavior do not cover the same visible send-then-delete/edit UX.

## What This Changes

Request an opt-in Telegram processing indicator message that is sent while a turn is running and deleted or edited on completion or failure.

## Best Possible Solution

Choose one first-class opt-in Telegram waiting UX, then implement it in the Telegram plugin with schema/types/help/docs, lifecycle tests, send/store/delete wiring, and failure edit-or-replace behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that the exact standalone placeholder config and lifecycle are absent on current main.

## Solution Assessment

Unclear until maintainers choose the intended UX. A narrow processing indicator is technically feasible, but adding it automatically would create new visible Telegram behavior and could overlap existing progress-draft and reaction feedback paths.
