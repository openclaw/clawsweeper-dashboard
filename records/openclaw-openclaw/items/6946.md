## Summary

Keep open. Current main still lacks a Telegram standalone processing-placeholder message lifecycle and accepted config surface; the adjacent typing cue, status/ack reactions, and preview/progress drafts do not implement the requested send-while-thinking, delete-on-reply, edit-on-failure behavior.

## What This Changes

Request an opt-in Telegram processing indicator that posts a visible placeholder while a turn is running and deletes or edits it after completion or failure.

## Best Possible Solution

Choose the intended Telegram waiting UX, then land an opt-in Telegram plugin change with typed config/schema/help/docs, authorized send/store/delete-or-edit lifecycle handling, focused tests, and real Telegram proof.
