## Summary

Keep open. Current main and the latest shipped release still provide Telegram typing, ack reactions, status reactions, and stream/progress previews, but not the requested opt-in standalone processing-placeholder message that is sent immediately and then deleted or edited on completion/failure; the direct and broader related implementation PRs are all closed unmerged, so this remains a Telegram product/config decision.

## What This Changes

Add an opt-in Telegram processing-placeholder message that appears while a turn is running and is deleted or changed when the turn completes or fails.

## Best Possible Solution

Choose the intended bundled Telegram waiting UX, then implement it in the Telegram plugin with typed config/schema/help/docs, message-id lifecycle cleanup, failure handling, focused regression tests, and real Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested config and standalone placeholder-message lifecycle.

## Solution Assessment

Unclear until maintainers choose the UX. The proposed opt-in Telegram placeholder is plausible, but it should be decided against the existing progress preview/status-reaction paths before implementation.
