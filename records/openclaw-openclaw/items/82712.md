## Summary

Keep open: current main and v2026.5.12 still have a source-reproducible unhandled background flush rejection path, and the open closing PR should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks OpenClaw to handle rejected background draft-stream flushes so streaming channels do not create unhandled promise rejections or silently lose pending text.

## Best Possible Solution

Fix the shared `createDraftStreamLoop` background launch path once, preserve pending text on send failure, add focused regression coverage, and keep this issue open until https://github.com/openclaw/openclaw/pull/82713 or an equivalent fix merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence focused repro: instantiate `createDraftStreamLoop` with a rejecting `sendOrEditStreamMessage`, call `update()`, and the background `flush()` promise can reject without a catch; I did not execute it in this read-only review.

## Solution Assessment

Yes. The maintainable fix belongs in the shared draft-stream loop rather than per-channel shims, with regression tests for immediate and scheduled background flush failures.
