## What This Changes

Handle rejected `createDraftStreamLoop.flush()` calls from scheduled/immediate updates so stream send/edit failures do not leak unhandled promise rejections.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82713; if it is not suitable, replace it with a narrow shared-loop fix plus regression coverage, then close this issue after merge.
