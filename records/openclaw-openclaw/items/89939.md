## Summary

Keep open for normal PR review and merge consideration: current main still skips registry tracking for accepted ACP spawns when `streamTo="parent"`, while this patch moves those runs onto the existing registry completion contract with focused test coverage and usable before/after behavior proof.

## What This Changes

This PR registers accepted ACP `sessions_spawn` runs even when `streamTo="parent"` is requested and updates the focused unit test to assert registry registration.

## Best Possible Solution

Land the registry-based fix after normal PR checks, keeping parent-stream relay as progress/system-event UX and registry announce/handoff as the completion-delivery owner.

## Reproduction Assessment

Yes from source: on current main, `streamTo="parent"` makes `shouldTrackViaRegistry` false, so accepted ACP runs never enter the registry that the completion announce and next-turn handoff paths consume.

## Solution Assessment

Yes; registering all accepted ACP runs is narrower than adding channel-specific completion fallbacks, and the existing `inlineDelivery` gate remains the suppression point for cases that already delivered inline.
