## Summary

Keep this PR open. Current main still lacks a direct sessions-cleanup path or shared helper for reclaiming unindexed JSONL transcript files, and the PR is a reasonable staged primitive, but it still has a narrow traversal bug where one unreadable descendant directory can make the whole orphan sweep return no results.

## What This Changes

The PR adds an opt-in `pruneOrphanedTranscripts` helper in session store maintenance plus unit tests for warn/enforce cleanup, preservation, nested files, symlink containment, checkpoint skips, and unrelated JSONL files.

## Best Possible Solution

If maintainers accept a staged primitive, land the helper after replacing the recursive `readdir` call with a best-effort per-directory walker; otherwise fold the helper into `openclaw sessions cleanup` in the same branch so operators get dry-run/enforce behavior with multi-store preserved-path collection from the start.

## Reproduction Assessment

Yes. The reported product gap is visible on current main because no explicit orphan-transcript prune helper or sessions-cleanup mode exists, and the remaining PR traversal bug is reproducible with a Node 22 recursive `readdir` probe against an unreadable descendant directory.

## Solution Assessment

Mostly yes. An opt-in, age-gated, session-header-gated, preserved-path-based helper is a maintainable building block, but best-effort subtree traversal is safer than relying on an all-or-nothing recursive `readdir`.
