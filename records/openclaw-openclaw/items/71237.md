## Summary

Keep open. Current main still does not provide a general, explicit orphan transcript JSONL prune utility, and the PR remains useful, but the proposed traversal should be made resilient before merge because one unreadable descendant can turn the whole sweep into a no-op.

## What This Changes

The PR adds an opt-in `pruneOrphanedTranscripts` helper and unit tests for warning about or deleting old unreferenced session transcript JSONL files under a sessions directory.

## Best Possible Solution

Land a safe cleanup primitive only after the walker skips unreadable descendants independently; if maintainers want the field issue solved in the same branch, fold it into `openclaw sessions cleanup` with dry-run/enforce behavior and a multi-store preserved-path union.

## Reproduction Assessment

Yes. The intended behavior is reproduced by the PR tests that create old valid session-header JSONL files outside `preservedPaths`; the remaining traversal bug is reproducible by adding an unreadable nested directory that makes the recursive directory read reject before other candidates are considered.

## Solution Assessment

Mostly yes. A path-preserved, age-gated, header-gated helper is a maintainable building block, but the recursive directory walk should be best-effort per subtree rather than all-or-nothing before this becomes a reliable cleanup primitive.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

- **[P3] Handle unreadable subtrees without aborting the sweep:** `src/config/sessions/store-maintenance.ts:473-478`
