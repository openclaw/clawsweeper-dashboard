## Summary

Keep open. Current main still preserves the stricter #47487 tool-profile behavior, but the reported silent cron UX gap remains: the config only emits a runtime warning, validation/doctor do not flag this mismatch, and cron only marks the run failed when the final text hits the current denial classifier.

## What This Changes

The issue requests a non-silent failure path when `tools.profile: "messaging"` plus configured `tools.exec` leaves isolated cron turns without `exec`/`process`.

## Best Possible Solution

Preserve strict tool-profile authorization while adding loud config/doctor diagnostics for restrictive profiles with ignored configured tool sections, plus cron classification for tool-unavailable final output so affected runs are not green deliveries.
