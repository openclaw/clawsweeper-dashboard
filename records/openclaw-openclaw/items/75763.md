## Summary

Keep open. Current main still preserves the stricter #47487 tool-profile behavior, but the silent cron UX gap remains: the mismatch is only a runtime warning, config validation stays schema-only, doctor has no matching preview warning, and cron can still mark model-narrated unavailable-tool output as ok when it misses the denial phrase list.

## What This Changes

The issue asks OpenClaw to make the 2026.4.29 messaging-profile exec/process regression visible through validation, doctor, or cron run status instead of green delivered output.

## Best Possible Solution

Preserve strict tool-profile authorization while adding loud config/doctor diagnostics for restrictive profiles with ignored configured tool sections, plus cron classification for unavailable-tool final output so affected runs are not green deliveries.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: tools.profile:"messaging" plus tools.exec leaves profileAlsoAllow unset, and final assistant text outside the current denial token list flows through isolated cron as status ok.

## Solution Assessment

Yes. The best fix is diagnostics and run-status classification around the removed implicit grant, not re-widening the messaging profile or changing the #47487 security contract.
