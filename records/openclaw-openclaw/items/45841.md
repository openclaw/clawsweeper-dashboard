## Summary

Keep this open. Current main still intentionally blocks ACP spawns from sandboxed requester sessions, documents that limitation, and has regression coverage for the fail-closed behavior; satisfying the request needs a new security-sensitive sandbox/ACP compatibility design.

## What This Changes

Request to let sandboxed OpenClaw sessions use ACP harness orchestration, possibly through nested sandboxing or an opt-in host proxy/bridge.

## Best Possible Solution

Define an explicit ACP sandbox compatibility mode that preserves the default fail-closed behavior and proves the new boundary with config, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: a sandboxed requester calling sessions_spawn with runtime="acp" or /acp spawn hits the documented runtime-policy block before ACP runtime initialization, and tests assert that behavior.

## Solution Assessment

No proposed patch is present to assess. The current fail-closed behavior is the safest default, but it does not satisfy the requested workflow; the maintainable fix needs a maintainer-approved sandbox-to-host ACP boundary design.

## Security Review

Status: needs_attention

Summary: The request would add a sandbox-to-host ACP execution path, so it needs explicit privilege-boundary design before implementation.

Concerns:

- **[high] Define the host ACP bridge as a controlled privilege boundary:** `src/agents/acp-spawn.ts:189`
  - body: A compatibility bridge could become a sandbox escape if it accepts broad commands or unaudited session control from a container. Any design should be opt-in, authenticated to the requester, allowlisted, rate-limited, logged, and limited to ACP spawn/steer/cancel semantics.
  - confidence: 0.9
- **[medium] Keep sandboxed ACP behavior fail-closed by default:** `docs/tools/acp-agents.md:721`
  - body: Current code and docs block ACP from sandboxed requesters because ACP harnesses run on the host. Any compatibility mode should preserve that default and require explicit operator configuration before host-side ACP execution is reachable from a sandbox.
  - confidence: 0.86
