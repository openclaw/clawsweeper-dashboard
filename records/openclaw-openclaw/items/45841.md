## Summary

Keep this open: current main still intentionally blocks sandboxed requesters from spawning host-side ACP sessions, and solving it needs an explicit sandbox-to-host ACP design rather than cleanup closure.

## What This Changes

Request sandboxed OpenClaw sessions to spawn or bridge ACP harness sessions without giving up sandbox isolation.

## Best Possible Solution

Keep the fail-closed behavior until maintainers approve an explicit, audited ACP-from-sandbox mode with config, docs, regression tests, and security review.

## Reproduction Assessment

Not applicable as a bug repro. The current limitation is source-proven: the ACP spawn policy rejects sandboxed requesters and the public docs document that behavior.

## Solution Assessment

No narrow bug fix is ready here. The useful next step is a maintainer design decision for an opt-in bridge or nested-sandbox contract, not a one-line relaxation of the current guard.
