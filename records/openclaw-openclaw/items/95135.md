Keep open: current main and v2026.6.8 still contain the reported Nix-mode guard behavior, and https://github.com/openclaw/openclaw/pull/95167 is an open candidate fix rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to allow Nix-mode auth-profile and session-state writes that do not mutate openclaw.json while preserving the immutable-config guard for actual config writes.

## Best Possible Solution

Land or replace the linked PR with a policy-confirmed boundary that permits non-config auth/session-state writes, preserves refusal for actual openclaw.json mutations, and updates tests/docs accordingly.

## Reproduction Assessment

Yes, source-reproducible: current main calls the Nix guard before doctor repair sequencing, and auth paths can still reach guarded config updates after separate auth-store writes. I did not run a live Nix/OAuth CLI path in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the Nix-mode policy carve-out. The likely maintainable fix is to enforce Nix mode at actual openclaw.json writes while explicitly allowing independent auth/session-state writes, with tests and docs updated to match.
