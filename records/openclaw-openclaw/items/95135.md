## Summary

Keep open: current main and the latest release still show the reported Nix-mode boundary problem, but the remaining fix needs maintainer policy confirmation because current docs and tests intentionally disable doctor repair in Nix mode.

## What This Changes

The issue requests narrowing Nix-mode write protection so auth-profile and session-state repairs can write their separate stores without touching openclaw.json.

## Best Possible Solution

Define the Nix-mode write boundary as immutable openclaw.json plus allowed separate auth/session state writes where intentional, then align doctor/auth code, docs, and regression tests to that policy.

## Reproduction Assessment

Yes for a source-level reproduction: current main still runs the doctor Nix guard before repair sequencing, and auth paste/login paths can still reach config updates after separate auth-store writes. No live CLI or OAuth run was attempted because this review is read-only.

## Solution Assessment

Unclear until maintainers confirm the policy carve-out. The maintainable direction is to guard actual openclaw.json writes while permitting intentional separate auth/session state writes, but current docs and tests deliberately ban doctor repair in Nix mode.
