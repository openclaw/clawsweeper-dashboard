## Summary

Keep open: current main and the latest release still block Nix-mode `doctor --fix` before non-config repair contributions can run, and the direct closing PR is still open and unmerged. Plain auth login is partly improved on current main because credentials are persisted before optional config updates, but the reported doctor/session-state path and the Nix-mode config-write policy boundary are not resolved yet.

## What This Changes

The issue asks for Nix mode to block only actual `openclaw.json` mutations while allowing independent auth-profile and session-state writes.

## Best Possible Solution

Review and land one focused candidate PR that scopes the Nix-mode guard to actual config mutations, updates the doctor/auth tests and docs, and leaves true `openclaw.json` writes blocked.

## Reproduction Assessment

Yes for the current-main doctor path: source and tests show Nix-mode repair is refused before repair side effects can run. Auth login is source-verifiable as partly improved on main, but the config-patch/default-write policy still needs a maintainer decision.

## Solution Assessment

No final solution is merged yet. The maintainable direction is a scoped Nix policy change that allows non-config state repairs without adding a broad bypass for config writes.
