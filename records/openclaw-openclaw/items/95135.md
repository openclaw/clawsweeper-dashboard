- `impact:auth-provider`: The report concerns model auth login and token/API-key persistence into auth profiles under Nix mode.

## Summary

Keep open: current main and v2026.6.8 still block all doctor repair modes up front in Nix mode, and model auth paste/login paths can still fail when they reach a config update even though credentials/session repairs live outside openclaw.json. The remaining work is real, but the exact carve-out needs maintainer policy confirmation because existing docs and tests intentionally disable doctor repair in Nix mode.

## What This Changes

The issue requests that Nix mode keep openclaw.json immutable without blocking auth-profile persistence or session-state repairs that write to separate stores.

## Best Possible Solution

Define a Nix-mode write policy that keeps openclaw.json immutable while allowing auth-store and session-state repairs, then update the doctor/auth command paths, docs, and regression tests to match that boundary.

## Reproduction Assessment

Yes from source inspection for the current-main failure class: doctor repair modes call the Nix config guard before repair sequencing, and auth paste flows upsert credentials before calling updateConfig, which is guarded in Nix mode. I did not run a live Nix-mode CLI repro because this review is read-only and tests would write artifacts.

## Solution Assessment

Unclear until maintainers confirm the policy carve-out. The maintainable direction is to guard actual openclaw.json writes while letting separate auth/session stores write, but current docs and tests intentionally disable doctor repair in Nix mode.
