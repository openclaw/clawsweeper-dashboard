## Summary

Keep open: current main and v2026.6.9 still guard Nix-mode doctor repair before repair selection, the auth paths can still reach guarded config writes after auth-store writes, and the two candidate PRs are open rather than merged.

## What This Changes

The issue asks OpenClaw to allow Nix-mode doctor and model-auth operations to perform auth-profile or session-state writes when they do not mutate `openclaw.json`.

## Best Possible Solution

Review and land or replace the linked candidate PRs with a policy-confirmed fix that gates actual `openclaw.json` mutations while allowing non-config auth/session-state repairs, with docs and regression tests updated for that boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows `doctorCommand` checks Nix mode before repair sequencing, and the auth command paths can still write credentials before reaching guarded config updates; I did not run a live Nix/OAuth CLI reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the Nix-mode policy carve-out. The likely maintainable solution is to enforce Nix immutability at actual `openclaw.json` write points while explicitly allowing non-config auth/session-state writes, rather than only removing the broad doctor entry guard.
