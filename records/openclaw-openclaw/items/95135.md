## Summary

Keep open: current main still blocks Nix-mode doctor repair before the repair code can distinguish config writes from auth/session-state writes, and the candidate fix PRs remain open and unmerged.

## What This Changes

The issue asks OpenClaw to allow Nix-mode doctor and model-auth operations that only write auth-profile or session-state stores to proceed without being blocked by the openclaw.json write guard.

## Best Possible Solution

Keep this issue as the canonical tracker until a scoped PR lands that enforces Nix mode at actual openclaw.json writes, preserves immutable config behavior, and updates doctor/auth tests and docs for allowed non-config auth/session-state repairs.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: run doctor repair under OPENCLAW_NIX_MODE=1 and current main rejects at the command entry guard before repair sequencing. I did not run a live Nix/OAuth CLI scenario in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the policy boundary. The likely maintainable fix is to guard actual openclaw.json mutations while allowing explicit non-config auth/session-state writes, but current docs and tests still encode the broader refusal.
