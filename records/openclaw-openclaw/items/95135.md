## Summary

Keep open: current main and latest release still block Nix-mode `doctor --fix` before non-config session/auth repairs can run, and the direct closing PR remains open and unmerged. ## What This Changes

The issue requests allowing Nix-mode doctor repairs and auth-profile writes when those operations do not mutate `openclaw.json`.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers land, replace, or reject the linked PRs with an explicit Nix-mode policy for non-config doctor/auth state writes.

## Reproduction Assessment

Yes at source level for the doctor path: current main and `v2026.6.9` call the Nix config write guard before any doctor repair work, so Nix-mode `doctor --fix` cannot reach non-config session/auth repairs. Plain provider auth login no longer writes config unless a provider patch or `--set-default` asks for it, while paste-token and paste-api-key still call the guarded config update path after storing credentials.

## Solution Assessment

Unclear until maintainers confirm the policy carve-out. Moving enforcement to actual `openclaw.json` writes is the likely maintainable direction, but current docs and tests deliberately ban doctor repair in Nix mode.
