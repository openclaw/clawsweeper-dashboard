## Summary

Keep open: current `main` and latest release still block Nix-mode `doctor --fix` before non-config repairs can run, and the direct implementation candidates remain open and unmerged. The reporter’s follow-up usefully ties this to the adjacent OpenAI/Codex auth-profile migration problem at https://github.com/openclaw/openclaw/issues/95136, but that makes the linked PR review path more important rather than making this issue ready to close.

## What This Changes

Allow Nix-mode doctor and auth flows to perform non-config session-state and auth-profile writes without mutating `openclaw.json`.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers choose, repair, and land one Nix-mode boundary PR that allows non-config auth/session repairs while preserving clear refusal for actual `openclaw.json` mutations.

## Reproduction Assessment

Yes, at source level. Current `main` and `v2026.6.10` call the Nix config write guard before any doctor repair work, and current auth paste-token/API-key paths still persist credentials before calling guarded config updates.

## Solution Assessment

Unclear as an accepted product direction. Moving the guard to actual `openclaw.json` writes looks like the maintainable shape, but current docs and tests intentionally refuse doctor repair in Nix mode, so maintainers need to confirm that boundary before landing a fix.
