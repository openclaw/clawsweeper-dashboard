## Summary

Keep this PR open. The latest revision fixed the previously noted missing gateway-method field, but it still compares raw stored `coreGatewayMethodNames` against a cache key that normally sorts gateway method names, so the production startup registry can still miss the scoped fallback when `listGatewayMethods()` returns its normal unsorted order.

## What This Changes

The PR adds scoped active plugin-registry reuse for unscoped plugin-tool resolution, stores core gateway method names on the registry and loader snapshot, adds loader regression tests, and adds a changelog entry for the embedded plugin-tool performance fix.

## Best Possible Solution

Keep the loader-level scoped-registry reuse, but normalize the stored or fallback `coreGatewayMethodNames` with the same unique sorted ordering used by `resolvePluginLoadCacheContext()`, then cover the production-shaped unsorted `listGatewayMethods()` case in the regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a scoped gateway-bindable startup registry keyed with unsorted `coreGatewayMethodNames` from `listGatewayMethods()`, followed by an unscoped default tool-resolution request; the active cache key used sorted methods, while the PR fallback rebuilds the key with the unsorted stored array.

## Solution Assessment

No. The proposed direction is narrow and maintainable, but this revision needs one more normalization step before it reliably solves the reported production cache miss.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Normalize gateway method names before comparing:** `src/plugins/loader.ts:964`
  - body: The active startup cache key is built from `coreGatewayMethodNames` after de-duping and sorting, but the new scoped fallback rebuilds the comparison key from `activeRegistry.coreGatewayMethodNames` in its stored order. `listGatewayMethods()` returns base methods in declaration order, so production startup can store an unsorted array and this fallback will still miss the active registry. Normalize this array before `buildCacheKey()` or store it normalized, and add a regression with unsorted method names.
