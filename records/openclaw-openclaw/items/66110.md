## Summary

Keep open: current main still does not provide a root log-file override, but this PR is not mergeable as-is because routed commands can bypass the new promotion path, daemon status can report the caller env instead of the service env, and the external PR still has only test-based proof.

## What This Changes

The PR adds a root `--log-file` option, `OPENCLAW_LOG_FILE` log-path resolution, daemon status/docs updates, and focused regression tests.

## Best Possible Solution

If maintainers want this override, land it as an approved CLI/env feature with route-first flag promotion, service-env-aware log-path resolution, updated docs/tests, and redacted real CLI/status proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: `runMain` can return from `tryRouteCli` before Commander `preAction`, and status already has a daemon env that the PR resolver does not accept. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The direction may be maintainable if approved, but this patch needs earlier flag promotion and explicit env plumbing through log-path resolution/status before it is the narrow correct solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Promote --log-file before routed commands:** `src/cli/program/preaction.ts:99-101`
  - body: The flag is only converted to `OPENCLAW_LOG_FILE` in Commander `preAction`, but `runMain` can return from `tryRouteCli` before Commander parses routed commands. Routed surfaces like `status` and `config get` would accept the new root flag but run with no env override.
  - confidence: 0.94
- **[P2] Resolve status log paths from the service env:** `src/logging/log-file-path.ts:40-42`
  - body: `resolveConfiguredLogFilePath` reads `OPENCLAW_LOG_FILE` from the caller process, while `gatherDaemonStatus` already merges the inspected service env. When CLI and daemon env differ, `File logs` can show the wrong path; pass the daemon env through the resolver.
  - confidence: 0.9
