## Summary

Keep this PR open for maintainer review: current main still lacks the proposed root log-file override, but the patch has two source-backed correctness blockers and no after-fix real CLI/status proof.

## What This Changes

The PR adds a root `--log-file` option, an `OPENCLAW_LOG_FILE` override for logger/log-path resolution, daemon status/docs updates, and focused tests.

## Best Possible Solution

If maintainers want this new override, implement it with early route-safe flag promotion, daemon-env-aware status resolution, focused regression tests, docs, and redacted real CLI/status proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: routed commands can return through `tryRouteCli` before Commander `preAction`, and status already has a merged daemon env that the PR resolver does not accept. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The direction may be acceptable after product approval, but this patch promotes the flag too late for routed commands and reads status log paths from the current CLI env rather than the inspected daemon env.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Promote --log-file before route-first commands:** `src/cli/program/preaction.ts:96-101`
  - body: The new option is copied into `OPENCLAW_LOG_FILE` only from Commander `preAction`, but `runMain` can return from `tryRouteCli` before Commander parses routed commands. `openclaw --log-file /tmp/x status`, `gateway status`, or routed `config get/unset` can therefore accept the flag but run without the override.
  - confidence: 0.94
- **[P2] Resolve status log paths from the daemon env:** `src/logging/log-file-path.ts:42`
  - body: `resolveConfiguredLogFilePath` now always reads `OPENCLAW_LOG_FILE` from the current CLI process, while status already merges the inspected service environment. If the daemon has a different env than the CLI, `File logs` can report the wrong path; thread an explicit env through the resolver and pass `mergedDaemonEnv`.
  - confidence: 0.9
