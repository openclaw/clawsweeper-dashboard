## Summary

Keep open. Current main still lacks the root log-file override that this PR is trying to add, but the proposed branch is not merge-ready because route-first commands can bypass the new Commander preAction promotion and daemon status can report the wrong log file when the inspected service environment differs from the caller environment.

## What This Changes

The PR adds a root `--log-file <path>` option, an `OPENCLAW_LOG_FILE` file-log override, logger/log-path precedence changes, daemon status coverage, CLI parser tests, and logging/environment docs.

## Best Possible Solution

Revise the PR so root log-file parsing and env promotion happen before route-first execution, the log-file env resolver accepts an explicit env with `process.env` as the default, daemon status passes the merged service env, and the docs/tests/changelog all describe the same precedence model without broadening into rotation or config mutation.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `openclaw --log-file /tmp/custom.log status` on the PR branch: `runMain` can return through `tryRouteCli()` before Commander preAction promotes the flag. A second focused path is daemon status with `OPENCLAW_LOG_FILE` set only in the mocked service command environment.

## Solution Assessment

No. The requested direction is narrow and maintainable, but the current branch misses route-first execution and makes status log-path reporting depend on the caller process env instead of the inspected service env.
