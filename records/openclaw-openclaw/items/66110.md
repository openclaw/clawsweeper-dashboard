## Summary

Keep this PR open, but the current patch should not merge as-is: the useful direction is plausible, yet it adds a new CLI/env override surface and still has two source-reproducible correctness gaps plus missing real behavior proof.

## What This Changes

The PR adds a root `--log-file <path>` option, an `OPENCLAW_LOG_FILE` override for logger/log-path resolution, daemon status reporting coverage, docs, and tests.

## Best Possible Solution

If maintainers want this override, land it as an approved CLI/env feature with route-first promotion, explicit env plumbing for status/service resolution, focused tests, docs, and redacted real CLI/status proof.

## Reproduction Assessment

Yes. Source inspection is sufficient for the PR defects: `runMain` can return through `tryRouteCli()` before Commander preAction promotes `--log-file`, and the PR's log-path resolver reads `process.env` instead of the inspected service env.

## Solution Assessment

No. The direction may be maintainable if maintainers approve the new surface, but this patch needs earlier root-option promotion and explicit env plumbing through log-path resolution/status first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Promote --log-file before routed commands:** `src/cli/program/preaction.ts:99-102`
  - body: The new flag is promoted only in Commander `preAction`, but `runMain` returns from `tryRouteCli()` before Commander parses routed commands. Paths such as `status` and `config get` can therefore accept `openclaw --log-file /tmp/x ...` while silently ignoring the advertised override.
  - confidence: 0.94
- **[P2] Resolve status log paths from the service env:** `src/logging/log-file-path.ts:41`
  - body: `resolveConfiguredLogFilePath()` now reads `OPENCLAW_LOG_FILE` from the caller process, while daemon status computes `mergedDaemonEnv` for the gateway service being inspected. When those envs differ, status can show the wrong file sink; thread an explicit env through the resolver and pass `mergedDaemonEnv`.
  - confidence: 0.9
