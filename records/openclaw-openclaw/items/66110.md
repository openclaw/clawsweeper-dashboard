## Summary

Keep open: current main does not implement the root/env log-file override, but this PR still needs maintainer approval for the new public surface, real behavior proof, and fixes for route-first/status-env defects before merge.

## What This Changes

The PR adds a root `--log-file` option, an `OPENCLAW_LOG_FILE` override for logger/log-path resolution, daemon status/docs updates, and focused tests.

## Best Possible Solution

Land only if maintainers want the public override, with flag promotion before route-first commands, service-env-aware status resolution, updated docs/tests, and redacted live CLI/status proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: `runMain` can return from `tryRouteCli` before Commander `preAction`, and status builds a daemon env but resolves the displayed log file from the CLI/config path. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The direction may be acceptable if approved, but this patch promotes the flag too late for routed commands and reads status log paths from the wrong environment; thread an explicit env and promote root flags before routing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Promote --log-file before routed commands:** `src/cli/program/preaction.ts:99-101`
  - body: The new option is copied into `OPENCLAW_LOG_FILE` only from Commander `preAction`, but `runMain` can return from `tryRouteCli` before Commander parses routed commands. `openclaw --log-file /tmp/x status` or routed `config get` will accept the flag but run without the override.
  - confidence: 0.94
- **[P2] Resolve status log paths from the daemon env:** `src/logging/log-file-path.ts:40`
  - body: `resolveConfiguredLogFilePath` now reads `OPENCLAW_LOG_FILE` from the current CLI process, while status already merges the inspected service environment. If the daemon has a different env than the CLI, `File logs` can report the wrong path; pass the merged daemon env into the resolver.
  - confidence: 0.9
