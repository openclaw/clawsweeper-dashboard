## Summary

This PR should stay open, but it is not ready to merge: the branch adds a new root/env log-file override surface, and the current patch still misses routed-command promotion plus service-env-aware daemon status reporting. Maintainer discussion on the linked report also frames the root CLI flag as a separate product surface after the config-backed logging bug was fixed.

## What This Changes

The PR adds a root `--log-file <path>` option, `OPENCLAW_LOG_FILE` log-path resolution, daemon status/docs updates, and focused regression tests.

## Best Possible Solution

If maintainers want this override, land it with early root-flag promotion for routed commands, explicit env plumbing through logger/status path resolution, docs/tests aligned to that contract, and redacted real CLI/status proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: `runMain` can return from `tryRouteCli` before Commander `preAction`, and daemon status already has a service environment that the PR resolver does not accept. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The direction may be maintainable if approved, but this patch needs earlier flag promotion and service-env-aware log path resolution before it is the narrow correct implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Promote --log-file before routed commands:** `src/cli/program/preaction.ts:99-101`
  - body: The flag is only converted to `OPENCLAW_LOG_FILE` in Commander `preAction`, but `runMain` can return from `tryRouteCli` before Commander parses routed commands. Routed surfaces like `status` and `config get` would accept the new root flag but run with no env override.
  - confidence: 0.94
- **[P2] Resolve status log paths from the service env:** `src/logging/log-file-path.ts:40`
  - body: `resolveConfiguredLogFilePath` reads `OPENCLAW_LOG_FILE` from the caller process, while `gatherDaemonStatus` already merges the inspected service env. When CLI and daemon env differ, `File logs` can show the wrong path; pass the daemon env through the resolver.
  - confidence: 0.9
