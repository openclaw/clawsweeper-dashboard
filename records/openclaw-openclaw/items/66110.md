## Summary

Keep open: current main still lacks the requested root log-file override, but this PR is not merge-ready because the override is promoted too late for route-first commands, status can resolve the wrong daemon log path, and after-fix real CLI/log proof is missing.

## What This Changes

The PR adds a root `--log-file` flag and `OPENCLAW_LOG_FILE` precedence for logger/log-path resolution, daemon status reporting, docs, and focused tests.

## Best Possible Solution

If maintainers want this public override, implement it with early route-safe flag promotion, daemon-env-aware status resolution, focused regression tests, docs, and redacted real CLI/status proof.

## Reproduction Assessment

Yes by source inspection for the PR defects: routed commands return through `tryRouteCli` before Commander `preAction`, and status already has a merged daemon env that the PR resolver does not accept. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No for the current patch. The direction may be acceptable after maintainer approval, but the implementation needs route-safe promotion and daemon-env-aware path resolution before it is the narrow maintainable solution.
