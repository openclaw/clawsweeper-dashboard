Full review comments:

- **[P1] Apply --log-file before route-first dispatch:** `src/cli/program/preaction.ts:99-101`
  - body: The new override is only written in Commander `preAction`, but `runCli` tries routed commands before Commander parses and can return there. For routed paths like `status`, `gateway status`, and `config get`, `openclaw --log-file /tmp/x ...` never sets `OPENCLAW_LOG_FILE`, so the advertised root flag is silently ignored. Hoist root log-file parsing into route-first startup or force those invocations through Commander, then cover a routed command end to end.
  - confidence: 0.9
- **[P2] Resolve status log paths from the service env:** `src/logging/log-file-path.ts:40-41`
  - body: `resolveConfiguredLogFilePath()` now reads `process.env`, but daemon status is reporting on a managed service that already has a merged service environment. If the CLI invocation and service have different `OPENCLAW_LOG_FILE` values, status reports the wrong file path. Thread an explicit env through the log-file resolver and have daemon status pass `mergedDaemonEnv`, with a service-env regression test.
  - confidence: 0.86
