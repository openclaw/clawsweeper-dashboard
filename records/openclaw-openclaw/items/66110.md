## Summary

Keep open: current main does not provide a root log-file override, so the PR still contains unique proposed work, but it is not merge-ready because it adds a public CLI/env surface that needs maintainer approval, lacks real behavior proof, and has two source-backed correctness blockers.

## What This Changes

The PR adds a root `--log-file` option, an `OPENCLAW_LOG_FILE` override for logger/log-path resolution, daemon status/docs updates, and focused tests.

## Best Possible Solution

Expose this override only if maintainers approve the public surface, with early route-safe flag promotion, explicit env-aware log-path resolution for status, focused regression tests, docs, and redacted real CLI/status proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: routed commands can return through `tryRouteCli` before Commander `preAction`, and daemon status computes a merged daemon env while reporting `logFile` from the CLI/config path. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The direction may be acceptable if maintainers want the new surface, but this patch promotes the flag too late for routed commands and reads status log paths from the wrong environment; the safer fix is to promote before routing and pass an explicit env into log-path resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Promote --log-file before routed commands:** `src/cli/program/preaction.ts:96-101`
  - body: The new option is copied into `OPENCLAW_LOG_FILE` only from Commander `preAction`, but `runMain` can return from `tryRouteCli` before Commander parses routed commands. `openclaw --log-file /tmp/x status` or routed `config get` can therefore accept the flag but run without the override.
  - confidence: 0.94
- **[P2] Resolve status log paths from the daemon env:** `src/logging/log-file-path.ts:42`
  - body: `resolveConfiguredLogFilePath` now always reads `OPENCLAW_LOG_FILE` from the current CLI process, while status already merges the inspected service environment. If the daemon has a different env than the CLI, `File logs` can report the wrong path; pass the merged daemon env into the resolver.
  - confidence: 0.9
