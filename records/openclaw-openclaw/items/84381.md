## Summary

Keep this PR open: the Gateway signal-abort direction is valuable, but current head still has a narrow accepted-before-callback race and the real behavior proof is stale after the latest force-push changed the abort plumbing.

## What This Changes

The PR threads `AbortSignal` and accepted-run callbacks through the Gateway client path, adds `openclaw agent` SIGINT/SIGTERM handling with `chat.abort` cleanup, updates focused tests, and documents cron/systemd hard-kill backstops.

## Best Possible Solution

Keep the Gateway client signal-abort design, but close the pre-accepted-callback race with the known idempotency/run id, add focused coverage for that timing, and refresh real behavior proof on the current head before merge.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main waits on a Gateway `agent` request without passing a caller abort signal or recording the accepted run id, while merged server-side `chat.abort` support already exists for accepted `agent` RPC runs.

## Solution Assessment

No, not quite yet. Threading `AbortSignal` through `callGateway`/`GatewayClient` and reusing `chat.abort` is the right shape, but the current patch should abort using the known run/idempotency key when a signal arrives before the accepted callback fires.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Abort the known run id before the accepted callback:** `src/commands/agent-via-gateway.ts:396`
  - body: The CLI already generates the `idempotencyKey` that the Gateway uses as the `runId`, but this path only calls `chat.abort` when `acceptedRunId` has been populated by `onAccepted`. If SIGTERM arrives after the server registers the run but before the accepted frame is handled locally, both abort paths see `undefined` and send no abort, so the accepted server-side run can continue. Seed the abort run id from the request idempotency key, or otherwise cover this race, and add a signal-before-`onAccepted` test.
