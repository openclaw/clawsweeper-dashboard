## Summary

Keep open for maintainer/product review: current main and the latest release intentionally run heartbeat when HEARTBEAT.md is missing, so the requested token-saving skip would change the shipped heartbeat contract rather than fix broken behavior.

## What This Changes

Request to skip heartbeat model prompts when HEARTBEAT.md is absent, preferably without logging anything.

## Best Possible Solution

Maintainers should decide whether to keep the shipped run-when-missing default, add an opt-in missing-file skip mode, or document the existing `0m` and empty-file paths more clearly, then update source, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source and the heartbeat test matrix on current main show that a missing HEARTBEAT.md keeps the heartbeat runnable and sends a model prompt.

## Solution Assessment

Unclear. The proposed unconditional skip is easy mechanically but would reverse documented shipped behavior; a safer solution likely needs an explicit maintainer policy choice or opt-in config.
