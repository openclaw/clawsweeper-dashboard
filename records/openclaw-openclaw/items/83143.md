## Summary

Keep open for maintainer product review: current main and shipped documentation intentionally run heartbeat when `HEARTBEAT.md` is absent, so this is a requested behavior change rather than a broken existing contract.

## What This Changes

Skip the heartbeat model prompt when the workspace does not contain `HEARTBEAT.md`.

## Best Possible Solution

Maintainers should decide whether missing-file skip should replace the shipped default or become an explicit opt-in policy, then update heartbeat preflight, docs, tests, and changelog together.

## Reproduction Assessment

Yes. Source and tests on current main show the reported current behavior: a missing `HEARTBEAT.md` returns runnable preflight and the test matrix expects the heartbeat to run.

## Solution Assessment

Unclear. An unconditional skip is straightforward technically, but it would reverse shipped and documented behavior, so the policy choice needs owner review first.
