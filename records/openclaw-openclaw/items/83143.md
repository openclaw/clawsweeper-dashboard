## Summary

Keep open for maintainer/product review: current main intentionally runs heartbeat when `HEARTBEAT.md` is missing, and source, tests, docs, changelog, and history show this is shipped behavior rather than a broken contract.

## What This Changes

Request skipping scheduled heartbeat model prompts when the workspace has no `HEARTBEAT.md` file.

## Best Possible Solution

Decide the heartbeat default explicitly: either keep run-when-missing and recommend an empty `HEARTBEAT.md` or `every: "0m"`, or add a documented opt-in skip-missing mode with tests for interval, wake, and cron-event behavior.

## Reproduction Assessment

Yes, at source level. Current main returns runnable heartbeat preflight when `HEARTBEAT.md` is missing, and the test matrix expects a missing file to run with one reply call.

## Solution Assessment

Unclear. The requested unconditional skip is a small code change but conflicts with the shipped default, so the best solution depends on maintainer policy around missing-file heartbeats versus an opt-in skip mode.
