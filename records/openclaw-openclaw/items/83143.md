## Summary

Keep open for maintainer/product review: the request is understandable and source-reproducible, but it would reverse current shipped, documented heartbeat behavior rather than fix a broken contract.

## What This Changes

Request that heartbeat runs skip the model prompt when workspace `HEARTBEAT.md` is absent to reduce token usage.

## Best Possible Solution

Keep the documented default unless owners approve a new default or opt-in skip-missing mode, then update heartbeat preflight, docs, tests, and changelog together.

## Reproduction Assessment

Yes. Source inspection shows a missing `HEARTBEAT.md` returns runnable preflight, and the focused test matrix expects missing-file heartbeat runs to call the model and send path; I did not execute tests because this review is read-only.

## Solution Assessment

Unclear. The proposed unconditional skip is easy to implement but not clearly the best solution because it reverses a documented and shipped default; a maintainer-approved default change or explicit opt-in mode would be safer.
