Full review comments:

- **[P1] Align the index identity with the shared-store hash:** `src/agents/memory-search.ts:493-498`
  - body: This changes only the SQLite filename hash. The persisted index identity still includes `extraPaths`, and sync can still full-reindex or pause on `index scope changed`, so changing `memorySearch.extraPaths` still does not get the incremental behavior the PR claims.
  - confidence: 0.93
- **[P1] Export the helper through the facade you reference:** `packages/memory-host-sdk/src/engine-foundation.ts:11`
  - body: `engine-foundation.ts` now re-exports `computeSharedScopeHash` from `./host/openclaw-runtime-agent.js`, but that facade does not export the helper. The package build cannot resolve this export until the facade is updated or the re-export points at the module that actually defines it.
  - confidence: 0.95
- **[P1] Write sharedStorePath on the declared store object:** `src/agents/memory-search.ts:492-498`
  - body: The new type adds `store.sharedStorePath`, but this code writes `resolved.sharedStorePath` and never initializes the required field on `store`. Strict TypeScript and consumers of the resolved config will see the wrong shape.
  - confidence: 0.94
- **[P1] Keep session-enabled stores agent-scoped:** `src/agents/memory-search.ts:493-498`
  - body: This override also runs for configs with both `memory` and `sessions`. Those managers can write per-agent transcript chunks into the same DB, while `openclaw memory search` calls `manager.search` directly without the tool's session-visibility post-filter, so shared stores need to be memory-only or filter by agent membership at the store/query layer.
  - confidence: 0.9
- **[P1] Add an upgrade path before moving the memory DB:** `src/agents/memory-search.ts:497-498`
  - body: Changing the resolved store from each agent's existing SQLite file to `state/memory/shared-*.sqlite` makes upgraded users miss their current index and can force a rebuild. This needs a doctor migration, explicit rebuild/warning path, or maintainer-approved upgrade plan before the default path changes.
  - confidence: 0.89
- **[P2] Keep registry helpers private until the API is accepted:** `packages/memory-host-sdk/src/engine-storage.ts:55-58`
  - body: The PR exports the new registry helpers through the memory-host SDK even though no caller wires the registry and the shared-store lifecycle/migration/access contract is unsettled. Keep these private or defer the export until maintainers accept the public contract.
  - confidence: 0.82
