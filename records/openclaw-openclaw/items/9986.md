## Summary

Keep open: current main and v2026.6.5 still route context overflow through active-model compaction/retry and then a blocked overflow result, while the direct implementation PR remains open and unmerged.

## What This Changes

Request automatic model fallback to a configured larger-context model when the active model exhausts its context window.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, fallback should run only after overflow recovery is exhausted and only to known larger-context candidates with replay-safety coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted; the linked PR body also reports a deterministic Testbox repro.

## Solution Assessment

Unclear until the product decision is made. The maintainable solution is guarded fallback after compaction/truncation recovery, with larger-context candidate filtering and replay-safety tests rather than unconditional overflow fallback.
