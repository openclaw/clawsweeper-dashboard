## Summary

Current main and the latest release still keep context overflow inside active-model compaction/retry and do not advance to a larger configured fallback. The issue remains necessary, and the open closing PR is the active implementation path, but it is draft/conflicted and still needs maintainer approval of replay-safety and candidate-window policy.

## What This Changes

Requests automatic model fallback to a configured larger-context model after active-model context-overflow recovery is exhausted.

## Best Possible Solution

Land a guarded core implementation that runs only after compaction and oversized-tool-result recovery are exhausted, selects only known larger candidates with sufficient headroom, logs the transition, and preserves the blocked result whenever replay safety or candidate capacity is uncertain.

## Reproduction Assessment

Yes at source level: current main explicitly rethrows provider overflow before fallback and produces a blocked context-overflow result after embedded recovery is exhausted. The linked PR also records a deterministic pre-fix Testbox run where only the primary model was called.

## Solution Assessment
