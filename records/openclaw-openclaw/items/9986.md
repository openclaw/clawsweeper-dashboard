## Summary

Keep this issue open: current main and the latest release still treat context overflow as active-model compaction/retry plus a blocked terminal result, and the only linked fix PR was closed unmerged as stale.

## What This Changes

The issue requests automatic fallback from a context-overflowing primary model to a configured fallback model with a larger context window.

## Best Possible Solution

Sponsor a fresh guarded implementation on current main, or explicitly document that context overflow remains compaction-only; if implemented, run it only after compaction/truncation recovery is exhausted, select larger known context windows, preserve replay safety, log the event, and update tests/docs.

## Reproduction Assessment

Yes, at source level: current main still rethrows thrown context-overflow errors before fallback and returns blocked `context_overflow` embedded results after recovery is exhausted. The closed fix PR also recorded a pre-fix Testbox reproduction, but I did not rerun it in this read-only review.
