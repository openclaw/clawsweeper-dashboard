## Summary

Keep open: current main still explicitly keeps context overflow in the active-model compaction/blocking path, and the linked open PR already owns the proposed guarded larger-context fallback change pending maintainer policy review.

## What This Changes

Request automatic fallback to a configured larger-context model after the primary model exhausts context instead of ending in a compaction or blocked overflow result.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/82754 with replay-safety and context-headroom checks, or explicitly document that context overflow remains compaction-only behavior.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR is the right review vehicle because it targets post-recovery overflow with larger-context candidate filtering instead of starting a second repair path.
