## Summary

Keep open: current main and the latest release still exclude context overflow from model fallback, while the linked implementation PR is open rather than merged.

## What This Changes

Request automatic routing to a configured larger-context fallback model when the active model's context window is exceeded.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the implementation should fallback only after overflow recovery is exhausted, select candidates with known sufficient context headroom, preserve replay safety, log the event, and keep regression coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR direction matches the narrow safe shape because it waits until overflow recovery is exhausted and only advances to known larger-context candidates.
