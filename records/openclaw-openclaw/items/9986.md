## Summary

Keep open: current main and v2026.6.8 still treat context overflow as compaction/retry or a blocked overflow result, not as a trigger for selecting a larger configured fallback model. The direct implementation PR closed unmerged and explicitly left the underlying request valid.

## What This Changes

The issue requests model fallback to a larger configured model when the active model exhausts its context window.

## Best Possible Solution

Either document overflow as compaction-only by maintainer decision, or land a fresh guarded implementation that tries only larger-context fallbacks after compaction and oversized-tool-result recovery are exhausted.

## Reproduction Assessment

Yes, source-level. Current main still rethrows escaped context-overflow errors and returns blocked embedded context_overflow results instead of trying a larger configured fallback model.

## Solution Assessment

No, current main is not the best end state for the requested behavior. A maintainable fix needs maintainer-approved post-recovery fallback routing with context-window filtering, replay-safety gates, and regression coverage.
