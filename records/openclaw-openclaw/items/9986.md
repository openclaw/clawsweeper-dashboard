## Summary

Current main still exhausts active-model compaction/truncation recovery into a blocked context-overflow result rather than trying a configured larger-context fallback. The issue also has an open maintainer-authored closing PR with deterministic before/after Testbox evidence, so it should remain open while maintainers refresh, review, and either land or replace that implementation.

## What This Changes

Add guarded fallback to a configured larger-context model after active-model context-overflow recovery is exhausted.

## Best Possible Solution

Keep active-model compaction and oversized-tool-result recovery first, then advance only to a fallback with a known larger effective window that fits the observed prompt, while preserving blocked results whenever replay or delivery evidence makes retry unsafe.

## Reproduction Assessment

Yes. Current-main source deterministically excludes context overflow from normal model fallback and returns a blocked overflow result after recovery; the linked PR also records a focused pre-fix Testbox reproduction.

## Solution Assessment

Yes, directionally. Post-recovery fallback only to a provably larger model is narrower and safer than making every context-overflow error failover-eligible, but maintainers must retain replay, delivery, and unknown-window safeguards.
