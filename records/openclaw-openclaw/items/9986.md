## Summary

Keep open: current main and the latest release still intentionally keep context overflow in active-model compaction/retry and blocked overflow handling, while the model fallback wrapper rethrows overflow instead of selecting a larger configured fallback. The broader retry/safe-mode request at https://github.com/openclaw/openclaw/issues/65502 overlaps, but it does not supersede this narrower larger-context fallback trigger request.

## What This Changes

Request automatic fallback to a configured larger-context model when the active model exceeds its context window.

## Best Possible Solution

Make the overflow policy explicit: either keep and document compaction-only recovery, or add guarded post-recovery fallback only to candidates with sufficient context headroom, replay-safety checks, structured logs, docs, and regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live >200k-token run. Current main and v2026.5.12 rethrow context-overflow errors from runWithModelFallback, retry overflow through active-model compaction/truncation, and return blocked context_overflow after recovery is exhausted.

## Solution Assessment

Unclear until maintainers decide the policy. If accepted, the narrow maintainable path is post-compaction fallback only to larger candidates with verified headroom, not treating every overflow as generic failover.
