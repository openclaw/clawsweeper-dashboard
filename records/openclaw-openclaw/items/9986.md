## Summary

Keep open: current main still routes context overflow through compaction/retry and explicitly keeps it out of model fallback, while recent maintainer discussion already kept this feature open for policy follow-up.

## What This Changes

The issue requests using configured model fallbacks when the active model exceeds its context window, preferably choosing a fallback with enough context headroom.

## Best Possible Solution

Maintainers should make the overflow policy explicit: either keep and document compaction-only recovery, or add a guarded post-recovery fallback path with context-window headroom checks, replay-safety guards, fallback logging, and regression coverage.

## Reproduction Assessment

Yes, for the current behavior from source inspection and focused tests. Current main rethrows likely overflow errors before model fallback and returns exhausted overflow as a blocked context_overflow result.

## Solution Assessment

Unclear until maintainers decide the policy. If accepted, the narrow maintainable fix is post-compaction fallback to sufficiently large candidates, not generic overflow-as-failover handling.
