## Summary

Keep open: current main still treats context overflow as compaction/retry plus a blocked overflow result rather than model fallback, and the open linked PR is already the implementation vehicle for the requested behavior.

## What This Changes

The issue requests model fallback to a configured larger-context model when the active model exhausts its context window.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/82754 with guarded larger-context fallback after recovery is exhausted, or document that context overflow remains compaction-only behavior.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted; the linked PR body also reports Testbox reproduction.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR is the right review vehicle because it targets post-recovery overflow and filters for larger-context fallback candidates instead of starting a second repair path.
