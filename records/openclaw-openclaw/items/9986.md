## Summary

Keep open: current main still intentionally keeps context overflow in the active model's compaction/retry path instead of selecting a larger configured fallback, so the requested behavior remains unresolved and needs maintainer policy judgment.

## What This Changes

The issue requests automatically retrying a context-overflowing primary model on configured fallback models with sufficient context headroom.

## Best Possible Solution

Make the overflow policy explicit: either keep and document compaction-only recovery, or implement guarded post-recovery fallback only to candidates with sufficient context headroom, replay-safety guards, logs, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: likely context-overflow errors are excluded from runWithModelFallback, handled by active-model compaction/truncation, and returned as blocked context_overflow when recovery is exhausted.

## Solution Assessment

Unclear until maintainers decide the policy. If accepted, the narrow maintainable solution is post-compaction fallback only to larger candidates with verified headroom, not treating every overflow as generic failover.
