## Summary

Keep open: current main and the latest release still document and implement context overflow as compaction/retry plus a blocked overflow result, and the direct closing implementation PR is open, unmerged, and currently conflicting.

## What This Changes

The issue requests routing context-overflow failures to a configured fallback model with a larger context window.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, fallback should run only after overflow recovery is exhausted and only to known larger-context candidates with replay-safety coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted.

## Solution Assessment

Yes, pending maintainer policy approval. The maintainable direction is a guarded fallback after compaction/truncation recovery, with larger-context candidate filtering and replay-safety tests.
