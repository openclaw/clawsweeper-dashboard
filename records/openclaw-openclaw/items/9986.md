## What This Changes

The issue asks OpenClaw to route exhausted context-overflow turns to a configured fallback model with enough context headroom.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the final behavior should only fallback after overflow recovery is exhausted, choose candidates with known sufficient headroom, preserve replay safety, and keep regression coverage.
