## Summary

Keep open. Current main still treats context overflow as active-model compaction, tool-result truncation, and then a blocked overflow result; the model fallback path still excludes likely context-overflow errors instead of selecting a larger configured fallback model.

## What This Changes

Request automatic fallback to a configured larger-context model when the primary model exceeds its context window.

## Best Possible Solution

Adopt and document a maintainer-approved policy that, after compaction and oversized-tool-result truncation are exhausted, falls back only to replay-safe candidates with sufficient resolved context headroom and logs the decision with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: overflow is detected, retried via active-model recovery, and finally returned as blocked `context_overflow`, while the outer fallback runner rethrows overflow before candidate rotation.

## Solution Assessment

No. The current implementation deliberately uses compaction-only overflow handling and does not satisfy the requested larger-context fallback behavior; the narrow maintainable solution needs maintainer approval around replay safety and headroom filtering.
