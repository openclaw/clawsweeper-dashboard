## Summary

Keep open: current main still handles context overflow through active-model compaction/retry and then a blocked overflow result, while the open closing PR already owns the proposed larger-context fallback behavior pending maintainer review.

## What This Changes

Request automatic model fallback to a configured larger-context model when the primary model exhausts its context window.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/82754 with a guarded larger-context fallback policy, or explicitly document that context overflow remains compaction-only behavior.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted; the linked PR body also reports Testbox reproduction.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR is the right review vehicle because it targets post-recovery overflow and filters for larger-context fallback candidates instead of starting a second repair path.
