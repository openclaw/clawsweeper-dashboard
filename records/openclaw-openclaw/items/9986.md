## Summary

Keep open: current main still intentionally keeps context overflow in the active model's compaction/retry path and explicitly excludes it from model fallback, so the requested larger-context fallback behavior remains unresolved and needs maintainer policy judgment.

## What This Changes

The issue requests routing to a configured fallback model with enough context headroom when the primary model exceeds its context window.

## Best Possible Solution

Make the overflow policy explicit: either keep and document compaction-only recovery, or implement guarded post-recovery fallback only to larger candidates with verified context headroom, replay-safety guards, structured logs, docs, and regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main rethrows likely context-overflow errors from runWithModelFallback, retries compaction/truncation on the same active provider/model, and returns blocked context_overflow after recovery is exhausted.

## Solution Assessment

Unclear until maintainers decide the policy. If accepted, the narrow maintainable solution is post-compaction fallback only to candidates with sufficient context headroom, not treating every overflow as generic failover.
