## Summary

Keep open: current main and v2026.5.22 still exclude context overflow from model fallback and return a blocked overflow result after recovery is exhausted. The linked implementation candidate is https://github.com/openclaw/openclaw/pull/82754, but it is still open and maintainer-labeled, so this issue should remain open until that PR is reviewed, landed, or closed.

## What This Changes

Request automatic fallback to a configured larger-context model when the active model exceeds its context window.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, fallback should run only after overflow recovery is exhausted and only to known larger-context candidates with replay-safety coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted.

## Solution Assessment

Yes, pending maintainer policy approval. The maintainable direction is guarded fallback after compaction/truncation recovery, with larger-context candidate filtering and replay-safety tests rather than unconditional overflow fallback.
