## Summary

Keeping this issue open: current main and the latest release still treat context overflow as compaction/retry plus blocked overflow, while the open linked implementation PR owns the requested larger-context fallback behavior but has not merged.

## What This Changes

The issue requests routing exhausted context-overflow failures to a configured fallback model with enough context headroom instead of freezing or returning a blocked overflow.

## Best Possible Solution

Resolve the open linked PR or an equivalent maintainer-approved implementation that falls back only after overflow recovery is exhausted, filters candidates by sufficient known context window, preserves replay safety, logs the event, and documents the fallback contract.

## Reproduction Assessment

Yes. Source inspection shows escaped context-overflow errors are rethrown or returned as blocked context_overflow before configured fallback selection, and the linked PR body includes Testbox evidence showing only the primary model was called before the fix branch.

## Solution Assessment

Unclear as submitted because the linked guarded fallback direction is plausible, but changing overflow from compaction-only recovery to model routing needs maintainer product and compatibility review before it becomes the best permanent contract.
