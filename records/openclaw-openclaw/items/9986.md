## Summary

Keep open: current main and the latest release still route context overflow through active-model compaction/truncation recovery and then a blocked overflow result, not to configured larger-context fallback models. The direct implementation candidate is open at https://github.com/openclaw/openclaw/pull/82754, but it is draft, conflicting, maintainer-labeled, and still waiting on author/maintainer handling.

## What This Changes

The issue requests falling back to a configured model with a larger context window when the active model exceeds its context limit.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, fallback should run only after overflow recovery is exhausted and only to known larger-context candidates with replay-safety coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted; the linked PR body also reports a deterministic Testbox repro, but I did not rerun it here.

## Solution Assessment

Yes, pending maintainer policy approval. The maintainable direction is guarded fallback after compaction/truncation recovery, with larger-context candidate filtering and replay-safety tests rather than unconditional overflow fallback.
