## Summary

Keep open: current main and the latest release still keep context overflow in compaction/retry or blocked-result handling rather than model fallback, and the direct implementation PR was closed unmerged as stale while explicitly saying the underlying request remains valid.

## What This Changes

The issue requests falling back from a context-exhausted primary model to a configured fallback model with a larger context window.

## Best Possible Solution

Land a fresh guarded implementation against current main that only tries larger-context fallback models after compaction and oversized-tool-result recovery are exhausted, or document that context overflow remains compaction-only.

## Reproduction Assessment

## Solution Assessment

No, not solved by current main. The best maintainable path is a fresh post-recovery fallback implementation that reuses current context-window resolution and preserves replay-safety gates.
