## Summary

Keep open: current main and the latest release still keep context overflow on the compaction/retry path instead of advancing through model fallback, and the direct implementation candidate at https://github.com/openclaw/openclaw/pull/82754 is still open.

## What This Changes

The issue requests model fallback to a configured larger-context model when the active model exceeds its context window.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the implementation should fall back only after overflow recovery is exhausted and only to known larger-context candidates with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR direction matches the narrow safe shape because it waits until overflow recovery is exhausted and only advances to known larger-context candidates.
