## Summary

Keep open: current main still treats context overflow as active-model compaction/truncation followed by a blocked overflow result, while the model fallback path and tests deliberately keep overflow out of failover; this needs maintainer product judgment rather than cleanup closure.

## What This Changes

Request automatic model fallback to a configured larger-context model when the active model exceeds its context window.

## Best Possible Solution

Make the overflow policy explicit: either keep and document compaction-only recovery, or add guarded post-recovery fallback only to candidates with sufficient context headroom, replay-safety checks, logs, docs, and regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main rethrows context-overflow errors out of runWithModelFallback, handles overflow with active-model compaction/truncation, and returns blocked context_overflow after recovery is exhausted.

## Solution Assessment

Unclear until maintainers decide the policy. If accepted, the narrow maintainable solution is post-compaction fallback only to larger candidates with verified headroom, not treating every overflow as generic failover.
