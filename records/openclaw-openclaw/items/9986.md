## Summary

Keeping this open because current main and v2026.6.6 still treat context overflow as compaction/retry plus a blocked overflow result, while the open linked PR owns the requested larger-context fallback behavior but has not merged.

## What This Changes

Request automatic routing to a configured larger-context fallback model when the active model exhausts its context window after recovery.

## Best Possible Solution

Resolve the open linked PR or an equivalent maintainer-approved implementation that falls back only after overflow recovery is exhausted, filters candidates by known sufficient context window, preserves replay safety, logs the event, and updates docs and tests.

## Reproduction Assessment

Yes. Source inspection shows context-overflow errors are rethrown or returned as blocked before fallback selection, and the linked PR body includes Testbox evidence showing only the primary model was called before the fix branch.

## Solution Assessment

Unclear as a product contract. The guarded larger-window fallback direction is plausible, but current docs intentionally define overflow as compaction/retry-only, so maintainers need to approve the compatibility and replay-safety tradeoff.
