## Summary

Keep open: current main still excludes context overflow from model fallback, the latest release carries the same documented exclusion, and https://github.com/openclaw/openclaw/pull/82754 is an open linked implementation candidate rather than a merged fix.

## What This Changes

The issue requests routing exhausted context-overflow runs to a configured larger-context fallback model instead of blocking on the active model.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the final behavior should fallback only after overflow recovery is exhausted, select known larger-context candidates with sufficient headroom, preserve replay safety, log the event, and keep regression coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked `context_overflow` result after embedded recovery is exhausted.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR direction matches the narrow safe shape because it waits until overflow recovery is exhausted and only advances to known larger-context candidates.
