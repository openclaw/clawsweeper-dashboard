## Summary

Keep open: current main still treats context overflow as compaction/retry plus a blocked overflow result, while the requested larger-context fallback routing remains unimplemented and needs maintainer policy/design judgment.

## What This Changes

Request automatic model fallback to a configured larger-context fallback when the active model exceeds its context window.

## Best Possible Solution

Adopt and document a maintainer policy, then either keep compaction-only overflow handling or add a guarded post-recovery fallback path that selects only candidates with enough context headroom.

## Reproduction Assessment

Yes from source inspection, not from a live provider run. Current main has code and tests showing context overflow is classified for compaction/blocked overflow handling rather than model fallback.

## Solution Assessment

Unclear as an implementation path because the requested behavior changes documented fallback semantics. The maintainable solution is a product decision followed by either documentation of compaction-only behavior or a guarded fallback path after recovery exhaustion.
