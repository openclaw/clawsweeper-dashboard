## Summary

Keep open: current main and the latest shipped release still keep context overflow in compaction/blocking flow rather than model fallback, and the open linked implementation PR is the right review path before this issue can close.

## What This Changes

Requests routing exhausted context-overflow runs to a configured larger-context fallback model instead of failing on the primary model.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the final behavior should fallback only after overflow recovery is exhausted, select candidates with known sufficient headroom, preserve replay safety, log the event, and keep regression coverage.

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted; the linked PR body also reports pre-fix Testbox reproduction.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR direction is the narrow maintainable path because it waits until overflow recovery is exhausted and only advances to known larger-context candidates.
