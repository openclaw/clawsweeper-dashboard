## Summary

Keep open. Current main still treats context overflow as compaction/retry-only and explicitly keeps those errors out of model fallback; the related closed items cover overloaded/server-error fallback, output-token freezes, graceful overflow handling, and fallback-chain executability, but not routing a too-large active-model turn to a larger configured fallback.

## What This Changes

Requested behavior: trigger configured model fallbacks when the active model exceeds its context window, preferably selecting a fallback with enough context headroom.

## Best Possible Solution

Keep this open until maintainers either document that context overflow is intentionally compaction-only, or implement a guarded fallback path after compaction and tool-result truncation are exhausted, with sufficient-context candidate filtering, replay safety, logging, and regression coverage.

## Reproduction Assessment

Yes, by source inspection: context-overflow errors are rethrown or handled by same-model compaction, and exhausted recovery returns meta.error.kind context_overflow. I did not run a live oversized prompt in this read-only review.

## Solution Assessment

Unclear. The requested direction may be right, but current main intentionally treats overflow as compaction-only, so the maintainable fix needs an explicit guarded fallback policy rather than a simple classifier change.
