## Summary

Keep open: current main still deliberately keeps context overflow in the active-model compaction/truncation path and explicitly excludes it from model fallback, so the requested larger-context fallback behavior remains a product decision rather than cleanup closure.

## What This Changes

Request context-overflow turns to fall back from a smaller primary model to a configured larger-context fallback instead of stopping after active-model recovery is exhausted.

## Best Possible Solution

Make the overflow policy explicit: either keep and document compaction-only recovery, or add guarded post-recovery fallback only to candidates with sufficient context headroom, replay-safety checks, structured logs, docs, and regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main rethrows context-overflow errors out of runWithModelFallback, retries overflow through active-model compaction/truncation, and returns blocked context_overflow after recovery is exhausted.

## Solution Assessment

Unclear until maintainers decide the policy. If accepted, the narrow maintainable path is post-compaction fallback only to larger candidates with verified headroom, not treating every overflow as generic failover.
