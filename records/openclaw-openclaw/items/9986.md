## Summary

Keep open: current main still treats context overflow as a compaction/retry terminal condition and explicitly keeps it out of model fallback, matching the current docs and latest maintainer review.

## What This Changes

Requests automatic fallback to a configured larger-context model when the active model exceeds its context window.

## Best Possible Solution

Maintainers should either keep and document the compaction-only policy, or implement a guarded post-recovery fallback path that filters candidates by context headroom and adds focused regression coverage.

## Reproduction Assessment

Yes, from source inspection and focused tests. Current main clearly rethrows likely context-overflow errors before model fallback and returns exhausted overflow as a blocked `context_overflow` result.

## Solution Assessment

Unclear until maintainers decide the overflow policy. If accepted, the maintainable route is a guarded post-compaction fallback to sufficiently large candidates, not generic overflow-as-failover handling.
