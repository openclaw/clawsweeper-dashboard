## Summary

Keep open: current main and v2026.6.10 still keep context overflow in active-model compaction/retry and blocked-overflow handling, while the direct larger-model fallback PR closed unmerged and the open related PRs cover adjacent behavior rather than the requested smaller-primary to larger-fallback route.

## What This Changes

The issue requests routing context-length overflow to a configured fallback model with enough context headroom instead of failing on the primary model.

## Best Possible Solution

Maintain compaction-only overflow only if maintainers explicitly choose that policy; otherwise implement guarded larger-context fallback after compaction and oversized-tool-result recovery are exhausted, with replay-safety gates, candidate context-window filtering, logs, docs, and regression coverage.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.10 still rethrow escaped context-overflow errors and return blocked embedded context_overflow results instead of trying a larger configured fallback model.

## Solution Assessment

Unclear until maintainers decide the policy. A guarded post-recovery larger-window fallback is plausible, but unconditional fallback would be risky without context-window filtering and replay-safety gates.
