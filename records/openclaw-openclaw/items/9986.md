## Summary

Current main and v2026.6.5 still treat context overflow as same-model compaction/truncation recovery and explicitly exclude it from model fallback; the requested larger-window routing remains necessary, but the open closing PR needs maintainer resolution rather than a competing fix.

## What This Changes

Requests model fallback to a configured larger-context model after context-overflow recovery is exhausted.

## Best Possible Solution

Preserve compaction and oversized-tool-result truncation as first-line recovery, then allow fallback only after exhaustion to a configured candidate with a canonically resolved larger usable window and no replay or outbound-delivery hazard.

## Reproduction Assessment

Yes. Current-main source deterministically rethrows overflow before fallback and returns a blocked context_overflow result after compaction/truncation exhaustion; the linked PR also supplies a focused pre-fix Testbox reproduction.

## Solution Assessment

Not yet conclusively. Guarded larger-window fallback is the narrowest plausible direction, but maintainers must approve replay-safety semantics and require reuse of the canonical context-window resolver before landing.
