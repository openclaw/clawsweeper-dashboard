## What This Changes

The issue requests routing context-overflow failures to a configured fallback model with a larger context window.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the final behavior should fallback only after overflow recovery is exhausted, select known larger-context candidates with sufficient headroom, preserve replay safety, log the event, and keep regression coverage.

Yes, pending maintainer policy approval. The linked PR direction matches the narrow safe shape because it waits until overflow recovery is exhausted and only advances to known larger-context candidates.
