## Summary

Keep open: current main and the latest release still keep context overflow in compaction/blocking flow rather than model fallback, and the open linked implementation PR is the right review path before this issue can close.

## What This Changes

Request automatic fallback to a configured larger-context model when the active model exceeds its context window.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the final behavior should fallback only after overflow recovery is exhausted, select candidates with known sufficient headroom, preserve replay safety, log the event, and keep regression coverage.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR direction is the narrow maintainable path because it waits until overflow recovery is exhausted and only advances to known larger-context candidates.
