## Summary

Keep open: current main and the latest release still treat context overflow as compaction/blocking rather than model fallback, and the linked implementation PR is still open rather than merged.

## What This Changes

The issue requests that configured model fallbacks be tried when the active model exceeds its context window and a later fallback has more context capacity.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, the final behavior should fallback only after overflow recovery is exhausted, require known sufficient candidate headroom, preserve replay safety, log the event, and keep docs plus regression coverage aligned.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted; the linked PR body also reports pre-fix Testbox reproduction.

## Solution Assessment

Yes, pending maintainer policy approval. The linked PR direction is the narrow maintainable path because it waits until overflow recovery is exhausted and only advances to known larger-context candidates.
