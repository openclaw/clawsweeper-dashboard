## Summary

Keep open: current main and the latest release still document and implement context overflow as active-model compaction/truncation recovery followed by a blocked overflow result or rethrow, not larger-context model fallback. The direct implementation candidate is open, draft, conflicting, and maintainer-labeled, so this issue should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue requests routing exhausted context-overflow failures from a smaller primary model to a configured fallback model with a larger context window.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/82754; if approved, fallback should run only after overflow recovery is exhausted and only to known larger-context candidates with replay-safety coverage.

## Reproduction Assessment

Yes, at source level. Current main rethrows thrown context-overflow errors before model fallback and returns a blocked context_overflow result after embedded recovery is exhausted; the linked PR also records a pre-fix Testbox repro with only the primary model called.

## Solution Assessment

Unclear until maintainer review. The maintainable direction is guarded fallback after compaction/truncation recovery, with larger-context candidate filtering and replay-safety tests rather than unconditional overflow fallback.
