## Summary

Keep open: current main still treats context overflow as compaction/truncation followed by a blocked overflow result, and a new open draft PR now specifically closes this issue. The right cleanup path is to review and either land or reject that linked PR, not close the issue ahead of merge.

## What This Changes

The issue requests routing context-overflow failures to a configured fallback model with a larger context window instead of stopping after active-model recovery fails.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/82754 as the implementation path, then either land its guarded larger-context fallback with tests/docs or document that overflow remains compaction-only.

## Reproduction Assessment

Yes, by source inspection rather than a live >200k-token run. Current main still rethrows context-overflow errors before fallback and returns a blocked context_overflow result after recovery is exhausted.

## Solution Assessment

Yes for routing, pending maintainer policy review. The linked PR is the right place to evaluate the narrow post-recovery, larger-context fallback approach instead of spawning duplicate work.
