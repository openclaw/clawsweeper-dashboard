## Summary

Keep open: current main and v2026.6.6 still provide token/cost usage by session, model, provider, agent, channel, and date, but not by canonical user/accounting subject. This remains valid product work, but it needs maintainer product and security direction before implementation.

## What This Changes

Request per-user input/output/total token accounting with day/week/month views exposed through `/usage` or the Dashboard.

## Best Possible Solution

Define a canonical accounting subject, retention/data-minimization policy, and viewer authorization model, then extend the existing session usage pipeline and Dashboard with per-subject daily aggregates that can derive weekly and monthly views.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and release inspection give high confidence that no per-user usage aggregate or filter currently exists.

## Solution Assessment

No: the proposed raw `user_id` log field is not the best shape. The safer path is to extend the existing usage aggregation contract with a canonical accounting subject and access-control semantics.
