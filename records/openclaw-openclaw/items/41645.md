## Summary

Keep open. Current main has substantial session/provider usage accounting and Dashboard usage views, but it still lacks per-user token aggregation, user filtering, weekly/monthly user rollups, and quota/billing semantics.

## What This Changes

Add per-user input/output/total token usage reporting with day/week/month views in `/usage` or the Dashboard.

## Best Possible Solution

Define a typed usage-accounting contract around a normalized principal, expose per-user and period rollups through `sessions.usage`, then add Dashboard and `/usage` views with documented privacy and quota boundaries.

## Reproduction Assessment

Not applicable as a feature request; source inspection confirms the missing capability. Current `sessions.usage`, its schema, Dashboard query helpers, overview UI, and token-use docs expose session/model/provider/agent/channel/daily metrics but no user aggregate or user filter.

## Solution Assessment

Unclear as proposed: adding raw `user_id` fields to logs is not enough because OpenClaw needs a normalized cross-channel principal and access-controlled reporting. The maintainable path is to model that principal first, then expose typed aggregates and UI/docs around it.
