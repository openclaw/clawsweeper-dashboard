## Summary

Keep open: current main and v2026.6.5 aggregate token usage by session, model, provider, agent, channel, and date, but still have no canonical per-user aggregate or filter. This is valid product work, but it needs maintainer product and security direction because accounting identity, privacy, retention, and viewer authorization are not defined.

## What This Changes

Add per-user input, output, and total token reporting by day, week, and month, available through `/usage` or the Dashboard.

## Best Possible Solution

Define a channel-agnostic accounting-subject identity, data-minimization and retention policy, and operator-versus-self-service authorization contract; then extend the existing session usage pipeline and Dashboard with per-subject daily aggregates from which weekly and monthly views can be derived.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken existing behavior. Source and release inspection provide a high-confidence verification that no per-user aggregate, filter, or Dashboard dimension currently exists.

## Solution Assessment

No, not as the proposed raw `user_id` log-field addition. The maintainable solution should extend the existing usage aggregation contract with a canonical accounting subject and explicit authorization, rather than make channel-specific logs a competing source of truth.
