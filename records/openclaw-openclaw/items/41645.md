## Summary

Keep open: current main still lacks token usage accounting grouped by user, and the remaining work needs product decisions about canonical user identity, privacy, and reporting scope.

## What This Changes

Add per-user input/output/total token usage accounting with daily/weekly/monthly rollups visible through `/usage` and the Dashboard.

## Best Possible Solution

Define a canonical usage-accounting user identity and privacy model, then add by-user aggregates and filters through the session usage API, `/usage`, Dashboard, tests, and docs.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug. Source inspection shows current main tracks session/model/provider/agent/channel/day usage, but not user-dimensional token aggregation.

## Solution Assessment

Unclear as proposed; adding `user_id` log fields after model calls would not define cross-channel identity or access controls. The maintainable path is an additive user-accounting contract wired through the existing usage aggregation and UI surfaces.
