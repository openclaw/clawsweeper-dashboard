## Summary

Keep open: current main and v2026.6.10 still expose token/cost usage by session, agent, channel, provider, model, and date, but not by a canonical user or accounting subject. This remains useful product work, but it needs maintainer product and security direction before implementation.

## What This Changes

Request to add per-user input/output/total token accounting with day/week/month reporting through `/usage` or the Dashboard.

## Best Possible Solution

Define a canonical accounting subject with authorization and retention rules, then extend the usage schema, Gateway aggregation, /usage, Dashboard filters/export, and docs around that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and release inspection give a high-confidence feature-gap check because current usage reporting lacks a user/accounting-subject dimension.

## Solution Assessment

No: adding a raw `user_id` log field alone is not the best solution. The safer path is to define the accounting subject, authorization, and retention contract before adding schema, Gateway, command, and Dashboard reporting.
