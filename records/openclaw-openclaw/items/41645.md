## Summary

Keep open: current main has usage tracking, `/usage`, and a Control UI usage dashboard, but the implemented aggregates are session/model/provider/agent/channel/date based rather than user based. The prior ClawSweeper comment did not make a substantive decision because that review failed before analysis.

## What This Changes

Request to add per-user token usage accounting for input/output/total tokens, grouped by day/week/month and visible through `/usage` or the dashboard.

## Best Possible Solution

Define a canonical user identity for usage accounting, extend the session usage API/shared types/cache and Control UI with additive by-user aggregates and filters, then document the privacy and access-control behavior.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug. Source and docs inspection show current usage surfaces exist, but no current-main path provides user-dimensional token aggregation.

## Solution Assessment

Unclear as proposed; adding log fields after each model call is not enough by itself because current reporting is built from transcript/session aggregation. The maintainable direction is an additive user-identity contract plus API, cache, command, and dashboard support.
