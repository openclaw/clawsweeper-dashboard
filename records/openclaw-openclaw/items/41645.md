## Summary

Keep open: current main has session/provider/model/agent/channel usage tracking, `/usage cost`, and the Control UI usage dashboard, but it still lacks token accounting grouped by user. The prior ClawSweeper comment reached the same conclusion, and the remaining work is a product/privacy design decision rather than conservative cleanup.

## What This Changes

The issue requests token usage totals grouped by user, with day/week/month views exposed through `/usage` or the Dashboard.

## Best Possible Solution

Define a canonical usage-accounting user identity and privacy model, then add by-user aggregates and filters through the session usage API, `/usage`, Control UI, and docs.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug. Source and docs inspection show existing usage surfaces, but no current-main path provides user-dimensional token aggregation.

## Solution Assessment

Unclear as proposed; adding log fields after each model call would not define cross-channel identity or access controls. The maintainable path is an additive user-accounting contract wired through the existing usage aggregation and UI surfaces.
