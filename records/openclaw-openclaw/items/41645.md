## Summary

Keep open: current main and v2026.6.6 still expose token/cost usage by session, model, provider, agent, channel, and date, but not by a canonical user or accounting subject. The request remains valid product work, but it needs maintainer product and security direction before implementation.

## What This Changes

The issue requests user-level input, output, and total token usage reporting over daily, weekly, and monthly windows, visible through `/usage` or Dashboard.

## Best Possible Solution

Define a canonical accounting subject, retention/data-minimization policy, and viewer authorization model before extending the existing session usage pipeline and Dashboard with per-subject daily aggregates.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and release inspection show the requested user/accounting-subject dimension is not present.

## Solution Assessment

No: the raw `user_id` log-field proposal is not the best shape by itself. The safer solution is to extend the existing usage aggregation contract around a canonical accounting subject plus access-control semantics.
