## Summary

Keep open: current main and v2026.6.10 provide session, agent, channel, provider, model, and date usage reporting, but not a canonical per-user or accounting-subject dimension. Related usage issues and the merged token-usage dashboard work are useful context, but none owns the requested user-dimensional accounting.

## What This Changes

Request per-user input/output/total token accounting with day/week/month views through `/usage` or the Dashboard.

## Best Possible Solution

Define a canonical accounting subject with retention and viewer-authorization rules, then extend the usage schema, Gateway aggregation, `/usage`, Dashboard filters/export, and docs around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and release inspection give a high-confidence feature-gap check because current usage reporting lacks a user/accounting-subject dimension.

## Solution Assessment

No: adding a raw `user_id` log field is not the best standalone solution. The safer path is one canonical accounting subject with authorization and retention semantics before adding schema, Gateway, and UI reporting.
