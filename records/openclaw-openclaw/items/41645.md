## Summary

Keep open: current main has usage tracking, `/usage cost`, and dashboard usage aggregation, but it does not provide the requested per-user token totals or user-level views. This is a feature/product-scope request, not stale cleanup or an already-implemented bug fix.

## What This Changes

The issue requests per-user input/output/total token usage tracking with daily/weekly/monthly views via `/usage` or the dashboard.

## Best Possible Solution

Define a user-attribution contract, then extend the existing `sessions.usage` pipeline with `byUser` aggregates, filters/export/UI, tests, and docs rather than adding a separate logging path.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives high confidence that current usage reporting is session/agent/channel/provider/model based rather than user based.

## Solution Assessment

Yes, keeping it open is the right outcome. The proposed log-field sketch is not the best complete implementation by itself because OpenClaw already has transcript-backed usage aggregation; the maintainable path is to extend that pipeline after deciding user identity semantics.
