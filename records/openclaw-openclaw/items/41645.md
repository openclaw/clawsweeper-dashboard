## Summary

Keep open: current main and v2026.6.10 still report usage by session, agent, channel, provider, model, and date, but not by a canonical user or accounting subject. This remains valid feature work, with product/security decisions needed before implementation.

## What This Changes

The issue requests per-user token usage accounting with input/output/total tokens, day/week/month rollups, and visibility through `/usage` or the Dashboard.

## Best Possible Solution

Define a canonical accounting subject with retention and viewer-authorization rules, then extend the usage schema, Gateway aggregation, `/usage`, Dashboard filters/export, tests, and docs around that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and release checks show current usage paths lack a user/accounting-subject dimension.

## Solution Assessment

No: the raw `user_id` log-field proposal is not the best standalone fix. The safer path is to define a canonical accounting subject and authorization/retention semantics before adding schema, Gateway, and UI reporting.
