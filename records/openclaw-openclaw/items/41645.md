## Summary

Keep open: current main has session/provider usage reporting, but the requested per-user token accounting and daily/weekly/monthly user views are not implemented and need maintainer product/schema direction.

## What This Changes

Request per-user input/output/total token usage statistics with day/week/month grouping and visibility through `/usage` or the Dashboard.

## Best Possible Solution

Design a first-class user-attribution contract for usage data, then extend transcript summaries, Gateway `sessions.usage`, and the Usage UI/export surfaces with approved per-user daily/weekly/monthly views.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks a user-scoped usage aggregate or Dashboard filter.

## Solution Assessment

Unclear until maintainers choose the identity and privacy contract. A narrow code-only patch that adds a raw user_id field would not be the best fix without schema, retention, and UI decisions.
