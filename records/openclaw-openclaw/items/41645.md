## Summary

Keep open: current main and the latest release still provide session, agent, channel, provider, model, and date usage reporting, but not a canonical per-user or accounting-subject dimension. The request remains valid product work with security implications.

## What This Changes

The issue requests per-user token usage accounting with input/output/total token totals over day/week/month buckets and visibility through /usage or the Dashboard.

## Best Possible Solution

Define a canonical accounting subject with retention and viewer-authorization rules, then extend the usage schema, Gateway aggregation, /usage, Dashboard filters/export, and docs around that contract.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and release inspection give a high-confidence feature-gap check because current usage reporting lacks a user/accounting-subject dimension.

## Solution Assessment

No: the raw user_id log-field proposal is not the best standalone solution. The safer path is a canonical accounting subject with authorization and retention semantics before adding schema, Gateway, and UI reporting.
