## What This Changes

The issue reports that local-model lean `tool_call` accepts flattened target-tool fields such as `command`, `path`, `task`, and `query` but dispatches `{}` because only nested `args` or `input` is forwarded.

## Best Possible Solution

Land one narrow Tool Search normalization fix that preserves explicit `args`/`input`, safely handles flattened target args, includes focused regression coverage, and adds redacted localModelLean runtime proof before closing this issue.

## Reproduction Assessment

Yes, at source level: current `readCallArgs` forwards only nested `args` or `input`, so a flattened payload like `{ id, command }` dispatches `{}`. I did not run the reported Telegram/vLLM session in this read-only review, but the issue and latest comment provide matching live-session evidence.

## Solution Assessment
