## Summary

Keep open: current main and v2026.6.10 still drop flattened compact Tool Search target arguments, and the open fix candidates that close this issue have not merged.

## What This Changes

Reports that localModelLean compact Tool Search `tool_call` discards flattened target-tool arguments and causes repeated missing-parameter tool failures.

## Best Possible Solution

Land one narrow, reviewed Tool Search normalization that preserves explicit `args` and `input`, safely handles flattened target arguments, and includes focused regression plus redacted real localModelLean proof.

## Reproduction Assessment

Yes, at source level: a compact `tool_call` payload like `{ id, command }` reaches `readCallArgs`, which currently forwards `{}` because only nested `args` or `input` are accepted. I did not run the reported Telegram/vLLM session in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable solution is in the Tool Search wrapper argument normalization rather than Telegram or a provider-specific path. The issue should stay open while maintainers choose and land one corrected linked PR.
