## Summary

Keep open: current main still restricts the final Telegram reply threadless retry to DM topics, but https://github.com/openclaw/openclaw/pull/79410 is an open closing PR with the narrow code/test change for this bug.

## What This Changes

Forum-topic Telegram final replies should retry once without `message_thread_id` when Telegram reports the thread is missing, matching existing direct-send and draft-preview fallback behavior.

## Best Possible Solution

Review and land the paired PR if its checks and behavior proof hold; after merge, this issue can close as implemented by that PR.

## Reproduction Assessment

Yes. Current main has a clear source-level reproduction path: a forum-thread send with `message_thread_id` that receives `400: Bad Request: message thread not found` is covered by an existing test that expects rejection rather than retry.

## Solution Assessment

Yes. Broadening the existing `sendTelegramWithThreadFallback()` retry gate from DM-only to DM-or-forum is the narrowest maintainable fix and matches the already accepted fallback in the direct send and draft preview paths.
