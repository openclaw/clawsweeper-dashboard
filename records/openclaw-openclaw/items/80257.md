## Summary

Keep open: this PR targets a real source-reproducible update-time config data-loss bug, but the current branch still has blocking correctness, typecheck, security-retention, and real-behavior-proof gaps.

## What This Changes

The branch adds update-time config snapshots, skips unknown-key stripping during updates, protects `plugins.installs`, disables update-time size-drop allowance, and adds focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the same config doctor will mutate, uses bounded or refreshed recovery snapshots, fixes the typecheck failure, and includes redacted real upgrade proof.

## Reproduction Assessment

Yes at source level: current main can run update-triggered doctor with write support, strip schema-unknown keys such as `plugins.installs` before migration, and bypass the size-drop rejection. I did not establish a live packaged macOS/npm upgrade reproduction.

## Solution Assessment

No as submitted. The direction is maintainable, but the snapshot target, snapshot retention, and failing test typecheck need correction before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Snapshot the resolved doctor config path:** `src/cli/update-cli/update-command.ts:1032`
  - body: The snapshot uses module-scope `CONFIG_PATH`, but the package-update doctor run builds its child environment with `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from the managed service profile. In that case recovery protects the caller config while doctor mutates the service config; compute the snapshot path from the same resolved env.
  - confidence: 0.91
- **[P2] Refresh recovery snapshots between updates:** `src/config/backup-rotation.ts:134`
  - body: Using `flag: "wx"` and returning on `EEXIST` makes the first `.pre-update` copy permanent. Future updates skip capturing their actual pre-update state and keep an old credential-bearing config; use per-update replacement, timestamped bounded snapshots, or another retention policy.
  - confidence: 0.88
- **[P2] Complete the doctor prompter test stub:** `src/flows/doctor-health-contributions.test.ts:194`
  - body: `check-test-types` fails because this helper casts a context whose `prompter` only has `shouldRepair`, but `DoctorHealthFlowContext` requires the full `DoctorPrompter` shape. Add the missing methods or cast through `unknown` only if the partial context is intentional.
  - confidence: 0.94
