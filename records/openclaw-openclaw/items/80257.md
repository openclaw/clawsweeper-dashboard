## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Snapshot the doctor config path:** `src/cli/update-cli/update-command.ts:1032`
  - body: The snapshot uses module-scope `CONFIG_PATH`, but package-update doctor builds its child env with `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from the managed service. In that case the recovery copy protects the caller config while doctor mutates the service config; derive the snapshot path from the same resolved doctor env before launching doctor.
  - confidence: 0.91
- **[P2] Refresh update recovery snapshots:** `src/config/backup-rotation.ts:134-138`
  - body: Writing `${configPath}.pre-update` with `wx` and treating `EEXIST` as success makes the first snapshot permanent, so later updates skip capturing their actual pre-update state. Use bounded timestamped snapshots, per-update replacement, or another retention policy that preserves one update run without leaving stale recovery data indefinitely.
  - confidence: 0.88
- **[P2] Fix the write-config test context type:** `src/flows/doctor-health-contributions.test.ts:180`
  - body: The new test helper casts a partial object to `DoctorHealthFlowContext`, but `prompter` only has `shouldRepair`; `check-test-types` fails because `DoctorPrompter` also requires confirmation methods. Build a complete mock prompter or cast through `unknown` only if the partial shape is intentional.
  - confidence: 0.96
