## Review Findings

Full review comments:

- **[P2] Preserve the first pre-update snapshot:** `src/config/backup-rotation.ts:125`
  - body: This writes `.pre-update` with `flag: "w"`, so every call overwrites the previous snapshot. The PR adds call sites before both the package doctor and the restarted in-process doctor; if an earlier update step mutates the config, the later call can replace the good pre-update copy with the bad one. Make this first-write-wins, for example by skipping an existing snapshot or using exclusive create, and add a repeated-call test.
  - confidence: 0.92

  - confidence: 0.96

  - body: The lint gate fails with `Expected { after 'if' condition`, and this new single-line guard is the changed pattern. Use a braced block for the early return so the PR can pass `check-lint`.
