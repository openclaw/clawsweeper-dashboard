## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Snapshot the doctor config path:** `src/cli/update-cli/update-command.ts:1032`
  - body: The snapshot protects CONFIG_PATH, but package-update doctor builds its child env with resolvePostInstallDoctorEnv, which can replace OPENCLAW_CONFIG_PATH from the managed service. In that case the recovery copy protects the caller config while doctor mutates the service config; derive the snapshot path from the same resolved doctor env before launching doctor.
  - confidence: 0.91
- **[P2] Refresh update recovery snapshots:** `src/config/backup-rotation.ts:134-138`
  - body: Writing `${configPath}.pre-update` with `wx` and treating EEXIST as success makes the first snapshot permanent, so later updates skip capturing their actual pre-update state and can leave stale credential-bearing config behind. Use bounded timestamped snapshots, per-update replacement, or another retention policy that preserves one update run without blocking future recovery copies.
  - confidence: 0.88
- **[P2] Fix the doctor test context stub:** `src/flows/doctor-health-contributions.test.ts:180-190`
  - body: The latest check-test-types job fails because this returned object is cast directly to DoctorHealthFlowContext while `prompter` only provides `shouldRepair`; DoctorPrompter requires several confirm methods. Add a typed test prompter stub or cast through `unknown` intentionally so the test-type gate can pass.
  - confidence: 0.95
