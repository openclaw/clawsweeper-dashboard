## Summary

Keep open. The PR targets a real source-reproducible update-time config-loss bug, but the patch still has blocking correctness issues, failing test-types CI, missing real upgrade proof, and a low-severity snapshot-retention security concern.

## What This Changes

The PR adds update-time config preservation guards, a `.pre-update` snapshot helper, `plugins.installs` protection from unknown-key stripping, update doctor size-drop protection, and focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the active doctor config path for each update run, refreshes recovery snapshots, keeps the strip and size-drop guards, fixes the type-check failure, and includes redacted live upgrade proof.

## Reproduction Assessment

Yes at source level: current main can run update-triggered doctor with write support, strip schema-unknown legacy config such as `plugins.installs` before write migration, and bypass the size-drop guard. I did not replay a packaged macOS npm upgrade, and the credentials-directory deletion remains unproven.

## Solution Assessment

No as submitted. The guard direction is maintainable, but the snapshot must use the same config path the doctor will mutate, the recovery snapshot must not go stale across future updates, and the new test must type-check.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Snapshot the config path doctor will mutate:** `src/cli/update-cli/update-command.ts:983`
  - body: Both new snapshot calls pass module-level `CONFIG_PATH`, but the child doctor env can be built from managed-service `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR`. In that update path the backup can be written beside the caller config while doctor mutates the service config, so derive the snapshot target from the same resolved env used for doctor.
  - confidence: 0.91
- **[P2] Refresh recovery snapshots between update runs:** `src/config/backup-rotation.ts:137-138`
  - body: `createPreUpdateConfigSnapshot` writes with `wx` and returns on `EEXIST`, so a `.pre-update` file from an older update prevents later updates from recording their actual pre-update config. That leaves no current recovery copy and keeps stale secret-bearing config outside rotation.
  - confidence: 0.9
- **[P2] Make the doctor test context type-check:** `src/flows/doctor-health-contributions.test.ts:180-194`
  - body: The new helper casts a partial object directly to `DoctorHealthFlowContext`, and the latest `check-test-types` annotation reports that the conversion does not sufficiently overlap. Build a valid context shape or cast through `unknown` only if the test intentionally owns the missing fields.
  - confidence: 0.86
