## Summary

Keep open. This PR targets a real source-reproducible config-loss path, but the current branch still has two blocking P2 issues and lacks required after-fix real upgrade proof.

## What This Changes

The PR adds update-time config preservation guards, a `.pre-update` config snapshot helper, `plugins.installs` strip protection, size-drop guard changes, and focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the resolved doctor target path, uses bounded or refreshed recovery snapshots, keeps the strip and size-drop guards, and includes redacted real upgrade proof.

## Reproduction Assessment

Yes at source level: current main can run update-triggered doctor with write support, strip schema-unknown keys such as `plugins.installs` before migration, and bypass the size-drop rejection. I did not establish a live packaged macOS/npm upgrade reproduction.

## Solution Assessment

No as submitted. The direction is maintainable, but the snapshot must follow the same resolved config path doctor will mutate, and recovery snapshots need bounded refresh or retention semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Snapshot the resolved doctor config path:** `src/cli/update-cli/update-command.ts:1032`
  - body: The snapshot uses module-scope `CONFIG_PATH`, but this package-update path builds the child doctor environment with `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from the managed service profile. In that case recovery protects the caller config while doctor mutates the service config; compute the snapshot path from the same resolved env.
  - confidence: 0.91
- **[P2] Refresh recovery snapshots between updates:** `src/config/backup-rotation.ts:134`
  - body: Using `flag: "wx"` and returning on `EEXIST` makes the first `.pre-update` copy permanent. Future updates skip capturing their actual pre-update state and can keep stale credential-bearing config; use per-update replacement, timestamped bounded snapshots, or another retention policy.
  - confidence: 0.88
