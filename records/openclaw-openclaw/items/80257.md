## Summary

Keep open: this is a relevant fix for a source-reproducible data-loss path, but the PR head still has two blocking correctness issues and lacks after-fix real upgrade proof.

## What This Changes

The PR changes update, doctor, config backup, and tests to preserve `openclaw.json` during update-triggered doctor runs.

## Best Possible Solution

Land a corrected update-time preservation fix that snapshots the same config path doctor writes, bounds or refreshes recovery snapshots per update run, and includes redacted real upgrade proof.

## Reproduction Assessment

Yes, at source level rather than live: current main can run update-triggered doctor with write support, strip strict-schema-unknown config keys including legacy `plugins.installs`, and bypass the size-drop guard on that path.

## Solution Assessment

No as submitted. The direction is appropriate, but the snapshot path and snapshot retention behavior need correction before this is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Snapshot the doctor config path:** `src/cli/update-cli/update-command.ts:983`
  - body: The child doctor env is built from `resolvePostInstallDoctorEnv(...)`, which can carry `OPENCLAW_STATE_DIR` or `OPENCLAW_CONFIG_PATH` from the managed service, but this snapshot uses module-level `CONFIG_PATH` from the updater parent. In those installs the snapshot can be written beside a different config than doctor mutates, leaving no recovery copy for the real file.
  - confidence: 0.88
- **[P2] Refresh pre-update snapshots for later updates:** `src/config/backup-rotation.ts:133-136`
  - body: `flag: "wx"` plus the `EEXIST` no-op makes `.pre-update` first-write-wins for its lifetime. A later update would keep an old copy instead of the user's current pre-update config, and it can preserve stale secret-bearing config indefinitely; rotate, timestamp, or replace snapshots while preserving idempotence within one update run.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff has no dependency or CI supply-chain changes, but the persistent snapshot can retain stale secret-bearing config indefinitely.

Concerns:

- **[low] Avoid indefinitely stale config snapshots:** `src/config/backup-rotation.ts:133`
  - body: OpenClaw config can contain secret references and channel/provider-sensitive data. Because `.pre-update` is first-write-wins and outside the normal rotation ring, an old snapshot can survive later updates and credential changes while failing to capture the current pre-update state.
  - confidence: 0.8
