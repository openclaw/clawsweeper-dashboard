Full review comments:

- **[P2] Snapshot the resolved doctor config path:** `src/cli/update-cli/update-command.ts:1032`
  - body: The snapshot uses module-scope `CONFIG_PATH`, but this package-update path builds the child doctor environment with `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from the managed service profile.   - confidence: 0.93

  - body: Using `flag: "wx"` and returning on `EEXIST` makes the first `.pre-update` copy permanent. Future updates skip capturing their actual pre-update state and keep an old credential-bearing config; use a bounded timestamped/rotated snapshot or a per-update replacement policy.
  - confidence: 0.9
- **[P2] Provide a complete doctor prompter stub:** `src/flows/doctor-health-contributions.test.ts:180-194`
  - body: `check:test-types` currently fails because this new test casts a partial context whose `prompter` only has `shouldRepair` to `DoctorHealthFlowContext`. Build a full `DoctorPrompter` test stub, or cast through `unknown` only if the partial context is intentional.
