## Summary

Keep open. The PR addresses a real config-loss bug, but the current patch still snapshots the wrong config path in managed-service updates, leaves stale recovery snapshots indefinitely, has a PR-introduced typecheck failure, and lacks required real upgrade proof.

## What This Changes

The PR adds update-time config preservation guards, a `.pre-update` snapshot helper, `plugins.installs` protection from unknown-key stripping, update doctor size-drop protection, and focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the active doctor config path for each update run, refreshes or rotates recovery snapshots safely, keeps the strip and size-drop guards, and includes redacted real upgrade proof.

## Reproduction Assessment

No, not end-to-end. Source inspection gives a clear current-main path for update-triggered doctor stripping legacy config and bypassing the size-drop guard, but I did not replay a packaged upgrade or prove the credential-directory deletion path.

## Solution Assessment

No as submitted. The main direction is appropriate, but the snapshot must follow the doctor-mutated config path, snapshots must not become stale across future updates, and the typecheck failure plus real proof gap must be resolved first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Snapshot the doctor config path:** `src/cli/update-cli/update-command.ts:980`
  - body: The snapshot uses module-level `CONFIG_PATH`, but the child doctor env can carry managed-service `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR`. In that update path, the backup can be written beside one config while doctor mutates another, so derive the snapshot path from the same resolved doctor env.
  - confidence: 0.9

  - body: Using `wx` and returning on `EEXIST` makes the first `.pre-update` file block all future update snapshots. A later upgrade may have no recovery copy of its actual pre-update config and may retain stale secret-bearing data indefinitely.
  - confidence: 0.9
