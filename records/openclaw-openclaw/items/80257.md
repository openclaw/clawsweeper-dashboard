## Summary

Keep open. This is an active bug-fix PR for a source-reproducible update-time config-loss path, but the current patch still has blocking correctness/security issues, a failing typecheck lane, and no real upgrade proof.

## What This Changes

The branch adds update-time config preservation by skipping unknown-key stripping during updates, protecting `plugins.installs`, adding `.pre-update` snapshots, tightening size-drop writes, and adding focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the exact config doctor mutates, keeps bounded current pre-update snapshots, passes type checks, and includes redacted real upgrade proof.

## Reproduction Assessment

Yes at source level: current main can strip schema-unknown legacy config before write-time migration during update-triggered doctor writes, and the size-drop guard can be bypassed. I did not replay a packaged macOS or npm upgrade.

## Solution Assessment

No as submitted. The root-cause direction is narrow, but the snapshot path, snapshot retention semantics, typecheck failure, and missing real upgrade proof need to be resolved before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

  - body: Package-update doctor is launched with env from `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from the managed service profile.   - body: The fixed `.pre-update` path is written with `wx` and existing snapshots are ignored, so the first snapshot permanently blocks later updates from recording their own pre-update state. Replace it per update or use timestamped snapshots with bounded retention.
  - confidence: 0.89
