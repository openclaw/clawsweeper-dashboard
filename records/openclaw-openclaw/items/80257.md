## Best Possible Solution

Land a corrected preservation fix that snapshots the actual resolved update-doctor config path, uses bounded or refreshed recovery snapshots, fixes the test-type failure, and includes redacted real upgrade proof.

## Solution Assessment

  - body: The snapshot uses module-scope `CONFIG_PATH`, but this package-update path then builds the child doctor env with `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from managed service env.   - confidence: 0.88
