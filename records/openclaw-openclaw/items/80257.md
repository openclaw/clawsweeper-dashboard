## Summary

Keep open. The PR targets a real source-reproducible updater data-loss bug, but the latest branch still snapshots the wrong config path in managed-service update flows, keeps stale first-write-wins recovery snapshots, and lacks real upgrade proof.

## What This Changes

The branch adds update-time config preservation guards, a `.pre-update` snapshot helper, `plugins.installs` strip protection, update doctor size-drop protection, and focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the active doctor config path for each update run, refreshes or rotates recovery snapshots safely, keeps the strip and size-drop guards, and includes redacted real upgrade proof preserving config and credentials.

## Reproduction Assessment

Yes, at source level: current main can run update-triggered doctor writes through unknown-key stripping before legacy `plugins.installs` migration and with size-drop allowance enabled. I did not replay a packaged macOS npm upgrade, so this is not a live reproduction.

## Solution Assessment

No as submitted. The guard direction is maintainable, but the snapshot must use the same config path the doctor will mutate and must not become stale across future updates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Snapshot the config path doctor will mutate:** `src/cli/update-cli/update-command.ts:983`
  - body: This snapshots imported `CONFIG_PATH`, but the child doctor env can include managed-service `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR`. In those update paths, the recovery file can be written beside the caller config while doctor mutates the service config without a pre-update snapshot.
  - confidence: 0.9
- **[P2] Refresh recovery snapshots between update runs:** `src/config/backup-rotation.ts:134-138`
  - body: `createPreUpdateConfigSnapshot` uses `wx` and returns on `EEXIST`, so an older `.pre-update` file blocks later updates from capturing their actual pre-update config. That also leaves stale sensitive config outside the normal backup rotation indefinitely.
  - confidence: 0.9
