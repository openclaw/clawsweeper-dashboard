## Summary

Keep open. The PR is a useful targeted fix for a real update-time config-loss path, but the current head is not merge-ready because it still snapshots the wrong config path in managed-service doctor runs, leaves permanent first-write-wins recovery snapshots, lacks real upgrade proof, and fails `check:test-types`.

## What This Changes

The PR adds update-time config preservation guards: a pre-update snapshot, update-time unknown-key strip bypass, `plugins.installs` strip protection, a size-drop guard change, and regression tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the exact config path doctor will mutate, uses bounded or refreshed recovery snapshot retention, keeps the strip and size-drop guards, fixes the test type failure, and includes redacted real upgrade proof.

## Reproduction Assessment

Yes at source level: current main can run update-triggered doctor with write support, strip schema-unknown config such as legacy `plugins.installs` before write-time migration, and bypass the size-drop rejection. I did not replay a packaged macOS npm upgrade.

## Solution Assessment

No as submitted. The general direction is maintainable, but the snapshot must follow the resolved doctor config path, recovery snapshot retention needs bounded refresh semantics, and the branch must pass test types with real upgrade proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Snapshot the resolved doctor config path:** `src/cli/update-cli/update-command.ts:1031`
  - body: The snapshot still uses module-scope `CONFIG_PATH`, but package-update doctor builds its child env with `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from the managed service. In that case this protects the caller config while doctor mutates the service config; derive the snapshot path from the same resolved doctor env before launching doctor.
  - confidence: 0.91
- **[P2] Refresh or bound pre-update snapshots:** `src/config/backup-rotation.ts:135-138`
  - body: Writing `${configPath}.pre-update` with `wx` and treating `EEXIST` as success makes the first snapshot permanent. Later updates then skip capturing the actual pre-update state and can retain stale credential-bearing config, so use per-update replacement or bounded timestamped retention.
  - confidence: 0.88
- **[P2] Fix the new doctor context test type:** `src/flows/doctor-health-contributions.test.ts:180`
  - body: `check:test-types` fails because the new fixture casts directly to `DoctorHealthFlowContext` while `prompter` only contains `shouldRepair`. Provide the required `DoctorPrompter` methods, use a narrower helper type, or cast through `unknown` intentionally so the type lane passes.
  - confidence: 0.94
