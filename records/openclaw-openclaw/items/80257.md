## Summary

Keep open: the bug is real and the PR direction is useful, but the current head still has two blocking snapshot-layer issues and lacks required real upgrade proof.

## What This Changes

The branch adds update-time config preservation guards, a pre-update snapshot helper, `plugins.installs` strip protection, a size-drop guard change, and focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the resolved doctor target path, uses bounded or refreshed recovery snapshots, keeps the strip and size-drop guards, and includes redacted real upgrade proof.

## Reproduction Assessment

Yes at source level: current main can run update-triggered doctor with write support, strip schema-unknown keys such as `plugins.installs` before migration, and bypass the size-drop rejection. I did not establish a live packaged macOS npm upgrade reproduction.

## Solution Assessment

No as submitted. The direction is maintainable, but the snapshot must follow the same resolved config path doctor will mutate and the recovery snapshot needs bounded refresh or retention semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Snapshot the resolved doctor config path:** `src/cli/update-cli/update-command.ts:1032`
  - body: The snapshot uses module-scope `CONFIG_PATH`, but this package-update path then builds the child doctor env with `resolvePostInstallDoctorEnv`, which can replace `OPENCLAW_CONFIG_PATH` from managed service env. In that case recovery protects the caller config while doctor mutates the service config; compute the snapshot path from the same resolved env.
  - confidence: 0.91
- **[P2] Refresh recovery snapshots between updates:** `src/config/backup-rotation.ts:134-138`
  - body: Using `flag: "wx"` and returning on `EEXIST` makes the first `.pre-update` copy permanent. Future updates skip capturing their actual pre-update state and keep an old credential-bearing config; use per-update replacement, timestamped bounded snapshots, or another retention policy.
  - confidence: 0.88

## Security Review

Status: needs_attention

  - body: OpenClaw config can contain credential references and occasional sensitive inline values; a permanent `.pre-update` file outside managed cleanup can outlive credential changes and fail to represent the current pre-update state.
