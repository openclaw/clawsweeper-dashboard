## Summary

Keep open. The PR targets a real update-time config data-loss bug, but the latest head still snapshots the wrong config path in managed-service update flows, leaves a stale first-write-wins recovery file outside backup rotation, fails the test-types check, and has no real upgrade proof.

## What This Changes

The PR adds update-time config preservation guards, a `.pre-update` config snapshot helper, `plugins.installs` protection from unknown-key stripping, update doctor size-drop protection, and focused tests.

## Best Possible Solution

Land a corrected preservation fix that snapshots the active doctor config path for each update run, refreshes recovery snapshots safely, keeps the strip and size-drop guards, fixes the typecheck failure, and includes redacted real upgrade proof.

## Reproduction Assessment

Yes at source level: current main can run update-triggered doctor with write support, strip strict-schema unknown keys such as legacy `plugins.installs`, and allow size drops. I did not replay a live macOS/npm upgrade or prove the credentials-directory deletion path.

## Solution Assessment

No as submitted. The fix direction is maintainable, but the snapshot must follow the config path the doctor will mutate, recovery snapshots need a safe refresh lifecycle, and the PR must pass test-types with real upgrade proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Snapshot the config path doctor will mutate:** `src/cli/update-cli/update-command.ts:981`
  - body: The snapshot still reads module-level `CONFIG_PATH`, but the child doctor env is built from managed-service `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR`. In that update path the recovery copy can be written beside the caller config while doctor mutates the service config, so derive the snapshot path from the same resolved doctor env in both call sites.
  - confidence: 0.91
- **[P2] Refresh pre-update snapshots between update runs:** `src/config/backup-rotation.ts:137-142`
  - body: Writing `.pre-update` with `wx` and returning on `EEXIST` makes the recovery snapshot first-write-wins forever. A snapshot from an older update can block later updates from capturing their actual pre-update config and can retain stale secret-bearing config outside rotation; refresh or rotate it per update run.
  - confidence: 0.9
- **[P2] Fix the doctor test context stub:** `src/flows/doctor-health-contributions.test.ts:180`
  - body: The latest `check-test-types` job fails because this helper casts an object with only `prompter: { shouldRepair: true }` directly to `DoctorHealthFlowContext`, but `DoctorPrompter` requires the confirm and repair methods. Provide a complete stub or cast through `unknown` so required type checks can pass.
  - confidence: 0.95
