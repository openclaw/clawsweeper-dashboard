## Summary

Keep open. The latest discussion adds a concrete v2026.6.9 restart-recovery failure mode, and current main still appears to resume any running `abortedLastRun` main-session row without a persisted cross-boot retry or quarantine gate.

## What This Changes

Prevent degraded Telegram/WhatsApp channel recovery and restart-recovery resume paths from starving the gateway and causing supervisor restart loops.

## Best Possible Solution

Keep this as a maintainer-owned stability item and either add a durable main-session recovery circuit breaker/quarantine path or split the channel timeout and restart-recovery pieces into canonical narrower issues.
