## Summary

Keep open. Current main still matches the WhatsApp login QR-output failure, and #76556 is an open closing PR for the same bug, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

Fix WhatsApp channel login so the terminal QR is exposed through the active CLI runtime instead of appearing to stall at the waiting state.

## Best Possible Solution

Land or replace the narrow runtime-routed WhatsApp QR fix, then close this issue from the merged PR once current main no longer bypasses the active CLI runtime.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: emit a Baileys `connection.update` QR event during `loginWeb()` and observe that the QR heading/art goes through `console.log` and `process.stdout.write` while the injected runtime only receives the waiting message.

## Solution Assessment

Yes for the core bug: routing the QR heading and ASCII QR through the injected runtime is the narrow maintainable fix. The linked open PR should be reviewed for that behavior and for whether any startup-policy remainder is intentionally out of scope.
