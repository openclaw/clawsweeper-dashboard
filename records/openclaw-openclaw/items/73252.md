## Summary

Keep open. The PR addresses a real Mattermost bug on current main, but the patch is not merge-ready because the new schema only accepts object-form `streaming` while current docs and the shared resolver still support scalar values, and it accepts `preview.toolProgress` without honoring that flag in the runtime. It also needs the required user-facing changelog entry.

## What This Changes

The PR adds Mattermost `streaming` config schema/type support, gates inbound draft-preview callbacks and final preview finalization on `streaming.mode`, and adds schema and monitor regression tests.

## Best Possible Solution

Land a Mattermost-owned fix that accepts or migrates both documented scalar `streaming` values and the canonical object shape, honors accepted preview sub-options or omits them, disables draft preview cleanly for `off`, preserves explicit block-streaming behavior, and keeps tests, docs/metadata, and changelog aligned.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence source-level reproduction: the strict Mattermost schema has no `streaming` key, the docs show scalar `streaming`, and the inbound monitor unconditionally creates and updates the draft-preview path.

## Solution Assessment

No. The implementation direction is right, but the proposed schema/runtime contract is incomplete: scalar compatibility and accepted `preview.toolProgress` behavior need to be aligned before merge, along with the changelog entry.
