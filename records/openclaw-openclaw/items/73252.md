## Summary

Keep open. The PR targets a real Mattermost draft-preview bug and the `off` path is useful, but it is not merge-ready because the schema still misses the documented scalar config form, accepts modes the runtime does not actually honor, and lacks the required user-facing changelog entry.

## What This Changes

The PR adds an object-form Mattermost `streaming` config, uses `streaming.mode` to disable draft-preview callbacks and final preview edits when set to `off`, and adds schema/monitor regression tests.

## Best Possible Solution

Merge a Mattermost-owned fix that accepts the documented scalar form and the canonical object form, resolves `off` cleanly for top-level and per-account configs, either implements or explicitly defers non-partial mode semantics, and includes focused schema/runtime tests plus the required changelog/docs alignment.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current main rejects or strips the documented `channels.mattermost.streaming: "partial"` shape through the strict schema, while the monitor always creates and updates a draft preview for inbound replies.

## Solution Assessment

No. The implementation direction is right for `streaming.mode: "off"`, but the PR is not the best merge-ready solution until schema compatibility, advertised mode behavior, and changelog coverage are aligned.
