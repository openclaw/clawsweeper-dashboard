## Summary

Keep open. Current main still rejects Mattermost `streaming` config and always wires draft previews, and this PR is a useful Mattermost-owned fix, but it is not merge-ready because the added schema accepts only object-form `streaming` while the public Mattermost docs still advertise the scalar mode value.

## What This Changes

The PR adds an object-form Mattermost `streaming` schema/type, gates inbound draft-preview callbacks and final preview editing on `streaming.mode`, and adds schema and monitor regression tests.

## Best Possible Solution

Merge a Mattermost-owned fix that accepts the documented and canonical streaming shapes, disables draft preview cleanly for `off`, preserves explicit block-streaming fallback, and keeps docs, doctor compatibility, metadata, tests, and changelog in sync.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence source-level reproduction: a strict Mattermost schema has no `streaming` key, and the inbound monitor unconditionally wires draft-preview updates and finalization.

## Solution Assessment

No. The implementation direction is right, but the merge-ready solution should either accept or migrate the documented scalar `channels.mattermost.streaming` value, or update the public docs and compatibility path in the same change.
