## Summary

Keep open: current main still has the Mattermost draft-preview config bug, but this PR is not merge-ready. The review confirms the existing blockers and adds an account-merge gap: the new schema only accepts object-form `streaming` while docs and issue context use scalar values, per-account streaming overrides are not deep-merged, and a required changelog entry is missing.

## What This Changes

The PR adds a Mattermost `streaming` schema/type, gates inbound draft-preview callbacks and final preview editing on `streaming.mode`, adjusts block-streaming fallback when preview is disabled, and adds schema/monitor regression tests.

## Best Possible Solution

A Mattermost-owned patch should make the public config contract internally consistent: accept or migrate both documented scalar and canonical object streaming shapes, resolve `off` to normal final delivery, preserve block-streaming inheritance across account overrides, and update tests, docs/doctor metadata if needed, and changelog together.

## Reproduction Assessment

Yes. Source-level inspection is enough: current main has no Mattermost `streaming` schema key and always wires draft-preview updates/finalization for inbound replies; the PR diff fixes part of that path but still rejects the documented scalar shape and does not update account merge semantics.

## Solution Assessment

No. The runtime guard is a good narrow direction, but the merge-ready solution needs scalar/docs/doctor compatibility, per-account nested streaming merge coverage, and a changelog entry before this can safely land.
