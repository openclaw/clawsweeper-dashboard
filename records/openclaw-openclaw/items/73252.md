## Summary

Keep open. The PR addresses a real Mattermost draft-preview opt-out bug, but it is not merge-ready because the proposed schema only accepts object-form `streaming` while current public Mattermost docs and the shared resolver still support scalar `streaming: "partial"`, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR adds Mattermost `streaming` object schema/type support, gates inbound draft-preview callbacks and final preview finalization on `streaming.mode`, and adds schema and monitor regression tests.

## Best Possible Solution

Land a Mattermost-owned fix that accepts or migrates both the documented scalar streaming value and the canonical object shape, disables draft preview cleanly for `off`, preserves explicit block-streaming behavior, and keeps tests, docs/doctor compatibility, metadata, and changelog aligned.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence source-level reproduction: a strict Mattermost schema has no `streaming` key, while the inbound monitor unconditionally creates and updates the draft-preview path.

## Solution Assessment

No. The implementation direction is right, but the PR should accept or migrate the documented scalar `channels.mattermost.streaming` form, or update the public docs and compatibility path in the same change, before merge.
