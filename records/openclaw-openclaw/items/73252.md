## Summary

Close as implemented on main: OpenClaw v2026.5.7 already accepts `channels.mattermost.streaming` and disables Mattermost draft-preview posts when streaming is off, with broader compatibility than this branch.

## What This Changes

The PR adds Mattermost streaming config schema/type support, uses `streaming.mode` to disable inbound draft previews, adjusts block-streaming dispatch behavior, and adds schema/monitor regression tests.

## Best Possible Solution

Keep the shipped v2026.5.7 Mattermost streaming implementation and track any remaining block-streaming or notification UX follow-up in the existing Mattermost issues rather than rebasing this branch.

## Reproduction Assessment

No. Current main no longer reproduces the reported schema/runtime gap: source inspection shows `streaming` is accepted and `streaming: "off"` selects a disabled draft stream.

## Solution Assessment

Yes for current main, not for this branch. The shipped implementation is narrower and more compatible because it accepts scalar, boolean, and object streaming forms, updates account resolution/tests/metadata, and disables draft-preview posts for `off`.
