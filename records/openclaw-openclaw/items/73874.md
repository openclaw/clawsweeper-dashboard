## Summary

Keep open. Current main has meaningful adjacent mitigations for plugin dependency staging, Gateway diagnostics, and async session transcript I/O, but the reporter’s latest v2026.4.29 retest still shows multi-second to multi-minute event-loop stalls on the affected Windows Docker Slack setup, and current main is not proven to fix that exact dispatch starvation.

## What This Changes

The issue reports Windows Docker Desktop Gateway event-loop and dispatch starvation after 2026.4.24 that leaves HTTP probes, internal WS RPC, and Slack Socket Mode unreliable despite the listener reaching ready.

## Best Possible Solution

Keep this open until maintainers run a current-main packaged/Docker build on the affected Windows Docker layout, capture a main-thread profile or timeline sample during an event-loop spike, and move or bound the identified startup/periodic work off the dispatch path.
