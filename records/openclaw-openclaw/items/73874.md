## Summary

Keep open. Current main contains important adjacent mitigations for runtime-deps startup work, session transcript I/O, plugin registry caching, and Gateway diagnostics, but the reported Windows Docker event-loop block is not proven fixed and the latest reporter retest still showed /healthz, /readyz, Slack, and WS starvation on v2026.4.29.

## What This Changes

The issue reports a Windows Docker Desktop Gateway event-loop/dispatch deadlock causing HTTP, WebSocket, Slack, and session processing stalls after releases 2026.4.24 through 2026.4.29.

## Best Possible Solution

Keep this open until maintainers run a current-main packaged Docker build on the affected Windows setup, capture a main-thread CPU profile or stack during an event-loop spike, and move or bound the identified startup/periodic work off the dispatch path.

## Reproduction Assessment
