## Best Possible Solution

Make `/think` menu and directive validation use one authoritative effective OpenAI/Codex provider/model thinking profile, overriding stale catalog metadata only when provider-owned policy proves support.

No, the current implementation is not the best end state because a stale runtime catalog row can outrank the provider-owned OpenAI thinking profile.
