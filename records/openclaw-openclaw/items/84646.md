## What This Changes

The issue asks for `/think` runtime validation on canonical `openai/gpt-5.5` to accept the same non-off thinking levels that the picker advertises, or for the picker to stop advertising unsupported levels.

## Best Possible Solution

Make `/think` menu generation and directive validation use one authoritative effective provider/model thinking profile for OpenAI/Codex, while preserving intentional `reasoning:false` opt-outs for providers without a profile-backed override.

## Solution Assessment
