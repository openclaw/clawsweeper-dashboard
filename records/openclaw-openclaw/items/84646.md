The issue reports that `/think` for canonical `openai/gpt-5.5` rejects non-off thinking levels because runtime/catalog validation treats the model as off-only despite the OpenAI provider thinking profile.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/84669 with a narrow provider-owned thinking-profile precedence fix that preserves explicit configured `reasoning:false` opt-outs and covers both `/think` and `sessions_spawn` regressions.

## Solution Assessment
