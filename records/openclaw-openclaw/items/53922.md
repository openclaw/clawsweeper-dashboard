## Summary

Keep open: current main does not implement the heartbeat-stop/progress-prompt scope, and this PR still needs blocking review for abort semantics, stream merging, queue accounting, and session-control authority before merge.

## What This Changes

The PR adds a WebChat heartbeat/session stop UI, a `sessions.activity` RPC, broader `sessions.abort` cleanup, Control UI/OpenClawKit stream/progress handling, and system-prompt guidance for milestone updates.

## Best Possible Solution

Split and rebase the work: keep normal generation Stop on `chat.abort`, pursue session activity/watchdog behavior through the canonical #39127/#39141 design with protocol docs and authorization coverage, and land progress-prompt guidance separately with focused tests or prompt snapshots.

## Reproduction Assessment

Yes for the review defects: the PR diff and current-main source show Stop being rerouted through broader `sessions.abort`, dropped followups counted as pending work, and non-delta snapshots appended. I did not run a live UI reproduction in this read-only pass.

## Solution Assessment

No. The proposed patch is not the best way to solve the issue because it combines independent product surfaces and broadens an existing abort RPC; a dedicated activity/autonomy-stop design plus a separate prompt-guidance change is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep regular Stop on chat.abort:** `ui/src/ui/controllers/chat.ts:337`
  - body: The normal WebChat Stop path now calls `sessions.abort`. In this PR that handler also clears queues, stops subagents, cancels ACP, and disables bound cron jobs, so stopping one answer can pause unrelated session automation; keep generation Stop on `chat.abort` or require an explicit autonomy-stop action.
  - confidence: 0.9
- **[P2] Treat non-delta snapshots as replacements:** `ui/src/ui/controllers/chat.ts:208`
  - body: When no `delta` is present and `nextText` is not a prefix, this appends the snapshot to the existing stream. Snapshot payloads are authoritative in that case, so rewritten or replayed text can be duplicated or corrupted in the UI and abort partials.
  - confidence: 0.86
- **[P2] Exclude dropped followups from pending size:** `src/auto-reply/reply/queue/state.ts:43`
  - body: `droppedCount` tracks discarded queue entries, not pending work. Including it makes `followupQueued > 0` and `canStop: true` after the queue is actually empty, showing a stop control for work that cannot be stopped.
  - confidence: 0.91
- **[P2] Return cleanup state on abort timeout errors:** `src/gateway/server-methods/sessions.ts:1030-1040`
  - body: By this point queues, subagents, ACP, and cron jobs may already have been cleaned up. If the embedded-run wait times out, returning only `UNAVAILABLE` leaves clients unable to reconcile stopped work and can trigger stale UI or redundant retries.
  - confidence: 0.75
- **[P3] Refresh heartbeat state after remote disable:** `ui/src/ui/app-gateway.ts:413-419`
  - body: Incoming heartbeat events only set `heartbeatsEnabled` to true. If another client disables heartbeats, the heart-pulse button can remain visible until reconnect even though heartbeats are off.
  - confidence: 0.78
