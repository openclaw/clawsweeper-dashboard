Full review comments:

- **[P1] Keep regular WebChat Stop on chat.abort:** `ui/src/ui/controllers/chat.ts:337`
  - body: The regular WebChat stop path now sends `sessions.abort` for active generation aborts. The new server handler disables session-bound cron jobs and stops queues/subagents/ACP, so clicking Stop on one answer can pause unrelated reminders or progress jobs. Keep ordinary generation stop on `chat.abort`, or require an explicit autonomy-stop action for broader cleanup.
  - confidence: 0.9
- **[P2] Replace no-delta snapshots instead of appending them:** `ui/src/ui/controllers/chat.ts:208`
  - body: When a non-prefix assistant snapshot arrives without `delta`, this fallback appends it to the existing stream. Gateway snapshots are authoritative in that case, so rewritten or replayed text can be duplicated or corrupted in the UI and in locally committed abort partials.
  - confidence: 0.86
- **[P2] Exclude dropped followups from pending activity:** `src/auto-reply/reply/queue/state.ts:43`
  - body: `droppedCount` tracks discarded queue entries, not pending work. Including it makes `followupQueued > 0` and `canStop: true` after the queue is actually empty, causing the stop control to appear for work that cannot be stopped.
  - confidence: 0.91
- **[P2] Return cleanup state after embedded-run timeouts:** `src/gateway/server-methods/sessions.ts:1030-1040`
  - body: By the time the embedded-run wait can time out, queues, subagents, ACP, and cron jobs may already have been cleaned up. Returning only `UNAVAILABLE` leaves clients unable to reconcile stopped work and can leave stale UI or retry behavior.
  - confidence: 0.75
- **[P2] Refresh heartbeat state when other clients disable it:** `ui/src/ui/app-gateway.ts:410-417`
  - body: The new heartbeat event handler only sets `heartbeatsEnabled` to `true`, so another client disabling heartbeats can leave this WebChat button visible until reconnect. Refresh from health/session state or handle a disabled broadcast so the UI cannot drift.
  - confidence: 0.76
