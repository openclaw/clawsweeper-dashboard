## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep regular stop on chat.abort:** `ui/src/ui/controllers/chat.ts:337`
  - body: The regular WebChat stop path now sends sessions.abort for active generation aborts. The new server handler disables session-bound cron jobs and stops queues/subagents/ACP, so clicking Stop on one answer can pause unrelated reminders or progress jobs; keep ordinary generation stop on chat.abort or require an explicit autonomy-stop action.
  - confidence: 0.9
- **[P2] Use snapshots as replacements without delta:** `ui/src/ui/controllers/chat.ts:208`
  - body: When a non-prefix assistant snapshot arrives without delta, this fallback appends it to the existing stream. Gateway snapshots are authoritative in that case, so rewritten or replayed text can be duplicated or corrupted in the UI and in locally committed abort partials.
  - confidence: 0.86
- **[P2] Exclude dropped followups from pending size:** `src/auto-reply/reply/queue/state.ts:43`
  - body: droppedCount tracks discarded queue entries, not pending work. Including it makes followupQueued > 0 and canStop true after the queue is actually empty, causing the stop control to appear for work that cannot be stopped.
  - confidence: 0.91
- **[P2] Return cleanup state after timeout errors:** `src/gateway/server-methods/sessions.ts:1030-1040`
  - body: By this point queues, subagents, ACP, and cron jobs may already have been cleaned up. If the embedded-run wait times out, the handler returns only UNAVAILABLE, so clients cannot reconcile the work that was already stopped and may keep stale UI state or retry unnecessarily.
  - confidence: 0.75
- **[P3] Refresh heartbeat state when another client disables it:** `ui/src/ui/app-gateway.ts:410-416`
  - body: The heartbeat event handler only sets heartbeatsEnabled to true. If another client disables heartbeats while this page stays connected, the heart button can remain visible until reconnect; derive the disabled state from health/session updates or a dedicated event too.
  - confidence: 0.78
