## Summary

Keep open for maintainer handling: the branch still contains unique WebChat/progress work, but it is broad, lacks after-fix real behavior proof, conflicts with current main, and has blocking correctness plus session-control security concerns.

## What This Changes

The PR adds WebChat heartbeat/session stop controls, a new `sessions.activity` RPC, broader `sessions.abort` cleanup, Control UI/OpenClawKit stream-progress handling changes, tests, and system-prompt progress guidance.

## Best Possible Solution

Split and rebase: keep ordinary chat generation stop on `chat.abort`, design any session activity/autonomy-stop API against https://github.com/openclaw/openclaw/issues/39127 with protocol docs and authorization coverage, and land progress-prompt guidance separately with focused proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the branch routes ordinary Stop through broader `sessions.abort`, counts dropped followups as pending, and appends no-delta snapshots. I did not run live UI tests in this read-only review.

## Solution Assessment

No. The proposed patch combines independent UI, Gateway protocol/server, OpenClawKit, and prompt-policy changes; the safer path is a split PR sequence aligned with the canonical activity API design at https://github.com/openclaw/openclaw/issues/39127.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep normal chat Stop on chat.abort:** `ui/src/ui/controllers/chat.ts:337`
  - body: The regular WebChat stop path now sends `sessions.abort` for active generation aborts. The new server handler disables session-bound cron jobs and stops queues/subagents/ACP, so clicking Stop on one answer can pause unrelated reminders or progress jobs; keep ordinary generation stop on `chat.abort` or require an explicit autonomy-stop action for broader cleanup.
  - confidence: 0.9
- **[P2] Treat no-delta snapshots as replacements:** `ui/src/ui/controllers/chat.ts:208`
  - body: When a non-prefix assistant snapshot arrives without `delta`, this fallback appends it to the existing stream. Gateway snapshots are authoritative in that case, so rewritten or replayed text can be duplicated or corrupted in the UI and in locally committed abort partials.
  - confidence: 0.86
- **[P2] Exclude dropped followups from pending size:** `src/auto-reply/reply/queue/state.ts:43`
  - body: `droppedCount` tracks discarded queue entries, not pending work. Including it makes `followupQueued > 0` and `canStop: true` after the queue is actually empty, causing the stop control to appear for work that cannot be stopped.
  - confidence: 0.91
- **[P2] Return cleanup state after abort timeouts:** `src/gateway/server-methods/sessions.ts:1030-1040`
  - body: By this point queues, subagents, ACP sessions, and cron jobs may already have been cleaned up. If the embedded-run wait times out, returning only `UNAVAILABLE` leaves clients unable to reconcile stopped work and can keep stale UI state or retry unnecessarily.
  - confidence: 0.75
- **[P3] Refresh heartbeat state when disabled elsewhere:** `ui/src/ui/app-gateway.ts:413-419`
  - body: Incoming heartbeat events only set `heartbeatsEnabled` to true, and no connected event path clears it when another client disables heartbeats. The heart-pulse button can remain visible until reconnect even though heartbeats are off.
  - confidence: 0.78
