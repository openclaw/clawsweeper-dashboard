## What This Changes

The branch adds WebChat heartbeat/session stop controls, a `sessions.activity` RPC, broader `sessions.abort` cleanup, stream/progress handling changes, and agent prompt guidance for progress updates.

## Best Possible Solution

Split and rebase the work: keep ordinary generation Stop on `chat.abort`, design session activity/autonomy stop against https://github.com/openclaw/openclaw/issues/39127 with protocol docs and authorization coverage, and land prompt guidance separately with focused tests plus real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep normal chat Stop on chat.abort:** `ui/src/ui/controllers/chat.ts:337`
  - body: The regular WebChat stop path is changed to call `sessions.abort`, whose new handler also clears queues, stops subagents, cancels ACP sessions, and disables session-bound cron jobs. Stopping one active answer can pause unrelated automation, so ordinary generation Stop should stay on `chat.abort` or use a separate explicit autonomy-stop action.
  - confidence: 0.9
- **[P2] Replace no-delta snapshots instead of appending them:** `ui/src/ui/controllers/chat.ts:208`
  - body: When a non-prefix assistant snapshot arrives without `delta`, this fallback appends it to the current stream. Gateway snapshots are authoritative in that case, so replayed or rewritten text can be duplicated or corrupted in the UI and abort partials.
  - confidence: 0.86
- **[P2] Exclude dropped followups from pending size:** `src/auto-reply/reply/queue/state.ts:43`
  - body: `droppedCount` tracks discarded queue entries, not pending work. Including it makes `followupQueued > 0` and `canStop: true` after the queue is empty, causing the stop control to appear for work that cannot be stopped.
  - confidence: 0.91
- **[P2] Return cleanup state after timeout errors:** `src/gateway/server-methods/sessions.ts:1022-1031`
  - body: By this point queues, subagents, ACP sessions, and cron jobs may already have been cleaned up. If the embedded-run wait times out, returning only `UNAVAILABLE` leaves clients unable to reconcile stopped work and can keep stale UI state or trigger redundant retries.
  - confidence: 0.75
- **[P3] Refresh heartbeat state when disabled elsewhere:** `ui/src/ui/app-gateway.ts:413-419`
  - body: Incoming heartbeat events only set `heartbeatsEnabled` to true, and no connected event path clears it when another client disables heartbeats. The heart-pulse button can remain visible until reconnect even though heartbeats are off.
  - confidence: 0.78
