## Summary

Keep open. Current main still has the reported split between fast `openclaw status` and live `openclaw channels status`, and #73662 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests that `openclaw status` populate its Channels table when configured/running channels are visible through `openclaw channels status`.

## Best Possible Solution

Review and land #73662, or an equivalent narrow fix, so the visible Channels table can use the existing gateway account snapshot when read-only discovery produces no rows while preserving the fast status path.

## Reproduction Assessment

Yes. The issue provides concrete CLI steps, another user confirmed the same split with Discord and Telegram, and current main still has a source path where fast status skips the gateway account data that `openclaw channels status` renders.

## Solution Assessment

Unclear because no fix is merged yet. The #73662 direction appears narrow and maintainable because it reuses the existing gateway snapshot to fill missing visible rows instead of making default status fully deep.
