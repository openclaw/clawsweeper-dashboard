## Summary

Keep open. Current main still has the reported split between fast `openclaw status` and live `openclaw channels status`, and #73662 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

Make the `openclaw status` Channels table show configured/running channels consistently with `openclaw channels status`.

## Best Possible Solution

Review and land #73662, or an equivalent narrow fix, so visible Channels rows can fall back to the existing gateway account snapshot when read-only discovery returns no rows.

## Reproduction Assessment

Yes. The issue gives concrete CLI steps, a second user confirmed the same split on another install, and current main still has a source path where fast status skips the gateway account data that `openclaw channels status` renders.

## Solution Assessment

Unclear because no fix is merged yet. The #73662 direction appears narrow and maintainable because it reuses the existing gateway snapshot instead of making default status fully deep, but the augment-versus-replace behavior still needs maintainer review.
