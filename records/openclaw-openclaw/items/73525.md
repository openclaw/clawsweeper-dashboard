## Summary

Keep open. Current main still keeps default text status off the live channel status path and renders the visible Channels table from read-only discovery rows only; the linked fix PR #73662 remains open and unmerged.

## What This Changes

The issue requests that default `openclaw status` populate its Channels table consistently with `openclaw channels status` when configured channels are running.

## Best Possible Solution

Review and revise #73662, or land an equivalent narrow fix that makes the visible Channels table populate from reliable gateway/read-only account data while preserving fast fallback behavior and adding regression coverage.

## Reproduction Assessment

Yes. The issue gives concrete CLI steps, another user confirmed the same split across Discord and Telegram, and current source/test inspection shows the default status path can skip the data source that the dedicated channels command renders.

## Solution Assessment

Unclear because no fix is merged yet. The gateway-snapshot direction is plausible, but the open PR still needs maintainer review around default fast-status behavior and the augment-versus-replace scope choice.
