## Summary

Keep open. Current main still has the reported split between the default status Channels table and the live channel-account status path, and #73662 is an open implementation candidate with closing syntax rather than a merged fix.

## What This Changes

The issue asks for `openclaw status` to populate its Channels table consistently with `openclaw channels status` instead of showing a header-only table while channels are running.

## Best Possible Solution

Resolve this by reviewing and landing #73662 or an equivalent narrow fix that fills missing visible Channels rows from the existing gateway `channels.status` snapshot while preserving fast status behavior and the richer read-only rows when they exist.

## Reproduction Assessment

Yes. The issue gives concrete command steps, another user confirmed the same behavior with Discord and Telegram, and current main still has code and tests that keep default status off the live `channels.status` account path.

## Solution Assessment

Not applicable as a merged solution yet. The open PR’s augmentation approach appears to be the narrow maintainable direction, but this issue should stay open until that fix lands or is explicitly superseded.
