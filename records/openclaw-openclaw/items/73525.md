## Summary

Keep open. Current main still builds the visible `openclaw status` Channels table from read-only plugin discovery while plain status disables live channel status/setup fallback, and #73662 is an open unmerged implementation candidate with closing syntax.

## What This Changes

The issue asks for `openclaw status` to show the configured running channels that `openclaw channels status` already reports.

## Best Possible Solution

Land #73662 or an equivalent narrow fix that augments missing visible Channels rows from the existing gateway `channels.status` snapshot while preserving richer read-only rows when present.

## Reproduction Assessment

Yes. The issue gives concrete CLI steps, another user confirmed the split across Discord and Telegram, and current source shows the plain status path can skip the gateway account data that `openclaw channels status` renders.

## Solution Assessment

Unclear until the linked PR is reviewed, but the narrow direction looks right: reuse the existing gateway status payload for missing visible rows rather than turning fast status into a full deep probe.
