## Summary

Keep open. Current main still has the reported source-level split: default text `openclaw status` can render an empty Channels table from read-only discovery while `openclaw channels status` formats the live gateway account snapshot, and the linked closing PR #73662 is still open and unmerged.

## What This Changes

The issue requests that `openclaw status` populate its Channels table with configured/running channels consistently with `openclaw channels status`.

## Best Possible Solution

Review and land #73662, or an equivalent narrow fix, so the visible Channels table is augmented from the existing gateway `channels.status` snapshot when read-only discovery produces no rows, while preserving the fast default path and the unreachable-gateway fallback.

## Reproduction Assessment

Yes. The issue gives concrete CLI steps, another user confirmed the behavior with Discord and Telegram, and current main still has a source path where default status skips live channel status/setup fallback while `channels status` reads `channelAccounts`.

## Solution Assessment

Yes, the narrowest maintainable direction is to reuse the existing gateway snapshot only to fill missing visible Channels rows rather than changing channel discovery wholesale; #73662 appears aimed at that, but it is not merged.
