## Summary

Keep open. Current main still has the reported split between the fast `openclaw status` Channels table and the live `openclaw channels status` output, and #73662 is an open implementation candidate with closing syntax for this report, so the issue should not be closed before that PR lands or is superseded.

## What This Changes

The issue requests that `openclaw status` populate its Channels table with the configured/running channels that `openclaw channels status` already reports.

## Best Possible Solution

Review and land #73662, or an equivalent narrow fix, so the visible Channels table can augment missing rows from the existing gateway account snapshot while preserving the fast status path and adding regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. The issue provides concrete CLI steps, another commenter confirmed the same behavior across Discord and Telegram, and current main still has a source/test path where default `openclaw status` can render no channel rows while `openclaw channels status` formats live `channelAccounts`.

## Solution Assessment

Yes for the tracked direction, but not yet as a merged solution. Reusing the already-collected gateway snapshot to fill missing visible rows is narrower than making the default status path fully deep or changing channel discovery wholesale.
