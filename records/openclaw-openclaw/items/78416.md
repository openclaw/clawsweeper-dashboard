## Summary

Keep open. This maintainer-labeled MEMBER PR is not a cleanup-close candidate, and the patch still has a source-reproducible false negative: an unrelated cleanup hook suppresses all `mkdtemp` findings in the same file.

## What This Changes

Adds a standalone test-temp cleanup scan script, Vitest coverage for the scanner and JSON mode, and a `lint:tmp:test-temp-cleanup` package script.

## Best Possible Solution

Keep the guard standalone, but make cleanup detection per `mkdtemp` call or test scope and add a regression fixture where unrelated hooks or `finally` blocks do not suppress a leak.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows any `afterEach`, `afterAll`, or `finally` anywhere in a file makes `classifyCleanupRisk` return `null`, so a mixed test file with an unrelated cleanup hook reproduces the false negative.

## Solution Assessment

No. The standalone scan is a reasonable direction, but the current file-level cleanup signal is too coarse for the stated guard; the narrower maintainable fix is scoped detection plus focused regression coverage.
