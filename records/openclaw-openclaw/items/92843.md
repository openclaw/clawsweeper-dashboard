## Summary

Keep open. Current main still has an unresolved exec-policy boundary: documented host exec allowlists are command-selection policy, but the Codex app-server path maps restrictive modes into Codex sandbox/approval behavior that can remove network/write reachability or decline approvals, matching the reported symptom class without providing a documented middle ground.

## What This Changes

The issue reports that `tools.exec.security: "allowlist"` agents lose network and external-write access despite sandbox being off, with no documented knob or warning.

## Best Possible Solution

Make the affected runtime path explicit: either preserve allowlist-approved host reachability outside Codex sandboxing, or document and expose a security-reviewed per-agent way to grant narrow network/external-write access without switching to full/elevated.

## Reproduction Assessment

Source-reproducible but not live-reproduced: native gateway exec does not show the jail, while the Codex app-server path still uses workspace-write/no-network sandbox policy and approval declines that match the reported symptoms.

## Solution Assessment

Unclear as a one-line code fix; the best direction needs maintainer security/product review across OpenClaw exec policy, Codex sandbox policy, and the documented operator knobs.
