## Review Findings

Full review comments:

- **[P2] Document read's adaptive cap:** `docs/tools/index.md:190-200`
  - body: The added docs say `read` always truncates text output at `2000 lines or 50KB`, but current main auto-pages reads and caps the aggregate output from an adaptive byte budget derived from the model context window. Please describe the adaptive behavior instead of a fixed 50KB contract.
  - confidence: 0.94
- **[P2] Document image reads as image blocks:** `docs/tools/index.md:202`
  - body: This says image reads return `MEDIA:<path>`, but current main handles read images as `image` content blocks and may have no path to extract. Consumers following this wording could miss valid image results, so the note should describe image blocks instead of a guaranteed `MEDIA:` token.
  - confidence: 0.93
- **[P2] Document canonical path parameters only:** `docs/tools/index.md:194`
  - body: The `read` parameter list says `file_path` is accepted, but current validation only recognizes `path`, and the regression test explicitly rejects `file_path`. Remove the unsupported alias wording from the file-tool entries so copy-pasted calls do not fail with a missing `path` error.
  - confidence: 0.91
- **[P2] Use the current edit payload schema:** `docs/tools/index.md:225-226`
  - body: The edit section documents top-level `oldText`/`newText` and alias fields, but current main requires `edits: [{ oldText, newText }]` with `path`. Document the `edits` array shape so users do not send a payload the tool rejects.
  - confidence: 0.95
