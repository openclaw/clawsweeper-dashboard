Full review comments:

- **[P2] Document adaptive read paging:** `docs/tools/index.md:189`
  - body: The new `read` paragraph hard-codes `2000 lines or 50KB` and tells readers to page manually. Current `createOpenClawReadTool` auto-pages when no explicit `limit` is passed and uses an adaptive byte budget, so this would publish stale guidance.
  - confidence: 0.94
- **[P2] Remove unsupported path aliases:** `docs/tools/index.md:193`
  - body: The added sections document `file_path`, but runtime validation for these tools requires canonical `path` plus `content` or `edits[]` as applicable. Users copying the documented shape would fail validation before the tool runs.
  - confidence: 0.93
- **[P2] Document image reads as image blocks:** `docs/tools/index.md:201`
  - body: The read notes promise `MEDIA:<path>`, but image reads are represented as image content blocks and may have no path. Consumers relying on a guaranteed `MEDIA:` token would miss valid image-read results.
  - confidence: 0.91
- **[P2] Describe edit inputs as an edits array:** `docs/tools/index.md:224-225`
  - body: The `edit` section documents top-level `oldText`/`newText` and legacy aliases. Runtime validation requires `edits: [{ oldText, newText }]`, so the shown call shape is not accepted.
  - confidence: 0.95
