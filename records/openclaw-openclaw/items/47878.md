  - body: The new `read` text hard-codes `2000 lines or 50KB`, but current OpenClaw wraps upstream read with adaptive auto-paging and a 32KB-128KB byte budget, so this would send users toward the wrong continuation behavior.

  - confidence: 0.93

  - body: The read notes promise `MEDIA:<path>`, but current read image results are inline image content blocks and may have no media path, so consumers relying on a `MEDIA:` marker would miss valid image reads.
  - confidence: 0.91

  - body: The `edit` section documents top-level `oldText` and `newText` aliases, but current validation requires `edits: [{ oldText, newText }]` and supports multiple replacements.
