  - body: The added `read` docs state text output is always truncated at `2000 lines or 50KB`, but current read execution computes an adaptive budget and auto-pages when context allows. Please describe the adaptive cap and continuation behavior instead of a fixed 50KB limit.

  - confidence: 0.95

  - body: The `read` notes promise `MEDIA:<path>`, but current read results are image content blocks and may have no media path. Please avoid documenting a guaranteed `MEDIA:` token for image reads.
  - confidence: 0.91
