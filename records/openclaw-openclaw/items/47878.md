- **[P2] Document adaptive read paging:** `docs/tools/index.md:190`
  - body: The new `read` docs hard-code `2000 lines or 50KB`, but current `main` wraps reads with adaptive paging and computes a 32KB to 128KB byte budget from the model context.   - confidence: 0.94
- **[P2] Remove unsupported file path aliases:** `docs/tools/index.md:194`
  - body: The new sections advertise `file_path`, but current validation requires canonical `path` for these file tools and existing tests reject legacy aliases. Copying the documented shape would produce missing-parameter errors.
  - confidence: 0.93
- **[P2] Document image reads as image content blocks:** `docs/tools/index.md:202`
  - body: The read notes promise `MEDIA:<path>`, but current read image results are inline image content blocks and may not include a media path. Consumers relying on a `MEDIA:` token would miss valid image reads.
  - confidence: 0.91
- **[P2] Describe edit inputs as an edits array:** `docs/tools/index.md:224-226`
  - body: The `edit` section documents top-level `oldText` and `newText` aliases, but current validation requires `edits: [{ oldText, newText }]` and supports multiple replacements. The documented call shape does not match the accepted tool input.
  - confidence: 0.95
