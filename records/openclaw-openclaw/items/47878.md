  - body: The new `read` docs say text output is always truncated at `2000 lines or 50KB`, but current `main` auto-pages reads and computes an adaptive per-call cap from model context. This would publish stale guidance and tell users to page manually when OpenClaw can continue automatically.

  - confidence: 0.93

  - body: The read notes promise `MEDIA:<path>`, but current read image results are inline image content blocks and may have no media path. Consumers relying on a `MEDIA:` token would miss valid image reads.
  - confidence: 0.91

  - body: The `edit` section documents top-level `oldText` and `newText` aliases, but current validation requires `edits: [{ oldText, newText }]` and supports multiple replacements. The documented call shape does not match accepted input.
  - confidence: 0.95
