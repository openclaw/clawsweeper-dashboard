- **[P2] Document adaptive read paging:** `docs/tools/index.md:187`
  - body: This hard-codes `2000 lines or 50KB`, but current `read` auto-pages when no explicit `limit` is supplied and caps output using an adaptive byte budget from model context. Publishing the fixed cap would give users stale paging guidance.
  - confidence: 0.94
- **[P2] Remove unsupported file_path aliases:** `docs/tools/index.md:193`
  - body: The new sections document `file_path`, but runtime validation requires canonical `path` before executing these tools and current tests reject `file_path` for `read`. Users copying the documented shape would hit missing-parameter errors.
  - confidence: 0.93
- **[P2] Do not promise MEDIA tokens for image reads:** `docs/tools/index.md:202`
  - body: The read notes promise `MEDIA:<path>`, but image reads can be inline image content blocks with no path. Consumers relying on a guaranteed `MEDIA:` token would miss valid image-read results.
  - confidence: 0.91
- **[P2] Document edit inputs as an edits array:** `docs/tools/index.md:225-226`
  - body: The edit section documents top-level `oldText`/`newText` aliases, but the runtime requires `edits: [{ oldText, newText }]`. The shown call shape is not accepted.
  - confidence: 0.95
