Full review comments:

- **[P2] Document adaptive read paging:** `docs/tools/index.md:190`
  - body: The new `read` text hard-codes `2000 lines or 50KB`, but OpenClaw wraps the upstream read tool with adaptive auto-paging and a 32KB-128KB byte budget. Users following this wording would assume the wrong ceiling and paging behavior.
  - confidence: 0.94
- **[P2] Remove unsupported path aliases:** `docs/tools/index.md:194`
  - body: The new sections advertise `file_path`, but current validation and tests enforce canonical `path` for these tools. A caller using the documented alias would get missing-parameter errors, so the alias claims should be removed.
  - confidence: 0.93
- **[P2] Describe edit as an edits array:** `docs/tools/index.md:224-225`
  - body: The `edit` section documents top-level `oldText` / `newText` aliases, but current validation requires `edits: [{ oldText, newText }]` and supports multiple replacements. Document the actual array payload shape.
  - confidence: 0.95
- **[P2] Avoid promising MEDIA tokens for image reads:** `docs/tools/index.md:202`
  - body: The read notes say images return `MEDIA:<path>`, but current read image results are image content blocks and may have no media path. Consumers relying on a `MEDIA:` marker would miss valid image results.
  - confidence: 0.91
