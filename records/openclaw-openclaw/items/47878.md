  - body: The added `read` text says output is truncated at `2000 lines or 50KB` and tells users to continue manually. Current `createOpenClawReadTool` uses adaptive paging and computes the cap from runtime context, so this would publish stale guidance.
  - confidence: 0.94

  - confidence: 0.93

  - body: The read notes promise `MEDIA:<path>`, but current image reads are inline image content blocks and may have no media path. Consumers relying on that token would miss valid image reads.
  - confidence: 0.91

  - body: The `edit` section documents top-level `oldText` and `newText` aliases, but validation requires `edits: [{ oldText, newText }]`. The documented call shape does not match accepted input.
  - confidence: 0.95
