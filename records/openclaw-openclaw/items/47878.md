  - body: The new `read` text hard-codes `2000 lines or 50KB`, but current OpenClaw wraps upstream read with adaptive auto-paging and a 32KB to 128KB byte budget. This would send users toward the wrong continuation behavior.
  - confidence: 0.94

  - body: The new sections advertise `file_path`, but current runtime validation requires canonical `path` for these file tools and tests reject `file_path`, so users following the docs would get missing-parameter errors.
  - confidence: 0.93
- **[P2] Document image reads as image blocks:** `docs/tools/index.md:202`
  - body: The read notes promise `MEDIA:<path>`, but current read image results are inline image content blocks and may have no media path, so consumers relying on a `MEDIA:` marker would miss valid image reads.
  - confidence: 0.91
- **[P2] Describe edit as an edits array:** `docs/tools/index.md:224-226`
  - body: The `edit` section documents top-level `oldText` and `newText` aliases, but current validation requires `edits: [{ oldText, newText }]` and supports multiple replacements.
  - confidence: 0.95
