Full review comments:

- **[P2] Document adaptive read paging:** `docs/tools/index.md:189`
  - body: The new `read` paragraph hard-codes text output at `2000 lines or 50KB` and tells users to page manually. Current `createOpenClawReadTool` auto-pages when `limit` is omitted and caps by `resolveAdaptiveReadMaxBytes`, so this would publish stale guidance.
  - confidence: 0.94
- **[P2] Remove unsupported path aliases:** `docs/tools/index.md:193`
  - body: The added parameter lists say `file_path` is accepted. Runtime validation and tests require canonical `path`, so users copying the alias shape would hit missing-parameter errors.
  - confidence: 0.93
- **[P2] Document image reads as image blocks:** `docs/tools/index.md:201`
  - body: The read notes promise `MEDIA:<path>`, but current read image results can be inline `image` content blocks with no path and only an optional `details.path` fallback. Consumers relying on the documented token would miss valid image reads.
  - confidence: 0.91
- **[P2] Describe edit inputs as an edits array:** `docs/tools/index.md:224-225`
  - body: The `edit` section documents top-level `oldText` and `newText` aliases, but runtime requires `edits: [{ oldText, newText }]`; the shown call shape is rejected.
  - confidence: 0.95
