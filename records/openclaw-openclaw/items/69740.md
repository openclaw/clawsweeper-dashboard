## Summary

Keep open: the branch still has blocking correctness and security issues, and current main has not implemented the central production debug-flag guard. This also lacks external after-fix real behavior proof, so it is not ready for merge.

## What This Changes

The branch adds an agents debug-env helper, routes model-catalog timing through it, changes Control UI sessions limit typing/input handling, tweaks CSS/app-render formatting, and updates the install-smoke timeout assertion.

## Best Possible Solution

Define a shared, legal debug-env boundary, migrate or explicitly exempt every in-scope debug read with focused tests, and split the unrelated Control UI/CSS/test changes out of this security hardening.

## Reproduction Assessment

Yes for the PR defects by source inspection. The diff shows the production guard only reaches model-catalog, while current main still has direct debug env reads and the sessions limit path can send a string to an integer-only gateway schema.

## Solution Assessment

No. A helper under src/agents applied to one call site is not a maintainable security fix for core, plugin, and package debug flags, and the unrelated session-limit refactor introduces a separate validation regression.

## Review Findings
