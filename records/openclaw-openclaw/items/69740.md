## Summary

Keep open: the patch is not mergeable as-is because the production debug guard only reaches one listed flag, the sessions limit refactor can send invalid params, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds an agents debug-env helper for model-catalog timing, changes Control UI sessions limit typing/input handling, reformats UI/CSS, and updates an install-smoke timeout assertion.

## Best Possible Solution

Define an agreed shared debug-env boundary or explicit owner-specific helpers, migrate or intentionally exempt every in-scope debug read with focused coverage, and split unrelated UI/test churn from the security hardening.

## Reproduction Assessment

Yes for the PR defects by source inspection. The PR head routes only model-catalog through the new production guard while listed debug flags remain direct, and the sessions.list path can receive a string limit from existing UI state sources.

## Solution Assessment

No. An agents-local helper applied to one call site is not the narrow maintainable fix for a cross-core/plugin debug policy, and the unrelated sessions/UI/test changes should not ride along with security hardening.
