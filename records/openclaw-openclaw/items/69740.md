## Summary

Keep this PR open: the security goal is still unmet, the patch introduces sessions-limit regressions, and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds an agents-local debug env helper and routes model-catalog timing through it, while also changing Control UI sessions limit typing/rendering and an install-smoke timeout assertion.

## Best Possible Solution

Define an agreed shared/public debug-flag policy or owner-local helpers, migrate or explicitly exempt every in-scope debug read with focused coverage, and split unrelated sessions UI changes from the security hardening.

## Reproduction Assessment

Yes for the PR defects by source inspection. The PR head guards only one debug call site while listed flags remain direct reads, and the sessions limit path can pass non-integer or string values to an integer-only RPC schema.

## Solution Assessment

No. An agents-local helper applied to one call site is not the narrow maintainable fix for a cross-core/plugin/package debug policy, and the unrelated sessions UI changes should be split or corrected separately.
