## Summary

Keep open: this recent external security-motivated PR has blocking correctness and security findings, missing real behavior proof, and no close reason because current main has not implemented the proposed production debug policy.

## What This Changes

The branch adds an agents debug-env helper, routes model-catalog timing logging through it, changes Control UI session limit typing/rendering, reformats UI CSS/rendering, and updates an install-smoke timeout test expectation.

## Best Possible Solution

Define an agreed shared debug-flag boundary or explicit owner-local helpers, migrate or intentionally exempt every in-scope debug read with focused coverage, and split unrelated sessions UI changes from the security hardening.

## Reproduction Assessment

Yes for the PR defects by source inspection. The PR head routes only model-catalog through the production guard while listed debug flags remain direct reads, and the sessions controller loses numeric coercion before an integer-only RPC schema.

## Solution Assessment

No. An agents-local helper applied to one call site is not the narrow maintainable fix for a cross-core/plugin/package debug policy, and the unrelated sessions UI changes should be split or corrected separately.
