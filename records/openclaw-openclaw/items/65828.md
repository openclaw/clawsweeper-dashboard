## Summary

Keep this PR open. Current main still has the MiniMax setup/metadata mismatch, but the PR as written broadens auto-detection from Token Plan-specific search credentials to the generic MiniMax model key and needs maintainer credential-policy review plus rework against the current MiniMax runtime split before merge.

## What This Changes

The PR updates MiniMax web-search metadata, contract artifacts, tests, manifest UI hints, and docs so `MINIMAX_API_KEY` is treated as an accepted MiniMax Search credential and MiniMax Search appears in web-search setup.

## Best Possible Solution

Land a revised MiniMax web-search setup fix that exposes MiniMax Search in onboarding and keeps docs/manifest/contract metadata aligned, while preserving safe auto-detection: either keep automatic detection limited to Token Plan-specific credentials or add an explicit search-capability gate or lower-priority path for generic `MINIMAX_API_KEY`, with focused provider-selection tests.

## Reproduction Assessment

Yes. The source-level reproduction is clear: with no pinned provider, `resolveWebSearchProviderId()` walks providers by `autoDetectOrder` and returns the first provider whose `envVars` has a non-empty value; adding `MINIMAX_API_KEY` would make a generic MiniMax key select MiniMax Search at order 15.

## Solution Assessment

No. The setup-visibility part is useful, but treating every generic `MINIMAX_API_KEY` as a web-search auto-detect credential is not the narrowest maintainable fix because MiniMax Search still requires Token Plan access; a gated or Token Plan-specific auto-detect path is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate generic MiniMax keys before auto-detecting search:** `extensions/minimax/src/minimax-web-search-provider.ts:282`
  - body: Adding `MINIMAX_API_KEY` to this provider's `envVars` makes web-search auto-detection treat any generic MiniMax model key as search-capable. Since auto-detect returns the first credentialed provider by `autoDetectOrder`, users without Token Plan search access but with another working search key can be routed to MiniMax and hit runtime failures. Please keep auto-detect on Token Plan-specific keys, add a capability check, or otherwise guard this path.
