## Best Possible Solution

Land or replace this with a rebased provider-auth contract change that passes a typed requested profile target into supported auth methods before credentials/config are produced, rejects unsupported or ambiguous methods clearly, updates docs/tests, and includes real behavior proof.

## Solution Assessment
