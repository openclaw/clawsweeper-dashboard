## Summary

Keep open: current main still lacks the requested `models auth login --profile-id` surface, and this PR is the canonical open implementation candidate, but the branch still needs contributor real behavior proof and a provider-safe implementation before merge.

## What This Changes

The PR adds `--profile-id` to `models auth login`, remaps single-profile auth results to the requested id, adds focused coverage, and updates CLI/gateway/OpenAI auth docs.

## Best Possible Solution

Land a rebased implementation that threads the requested profile id through a documented provider-safe auth context before persistence, rejects ambiguous/unsupported methods, updates docs/tests, and includes redacted real behavior proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature PR for a new CLI flag. Source inspection on current main confirms `models auth login` lacks `--profile-id` while `paste-token` has it.

## Solution Assessment

No as currently proposed: the direction is useful, but post-return remapping is not the safest contract for provider-owned auth flows. The narrower maintainable fix is to pass the requested profile id into provider auth before side effects or reject methods that cannot safely honor it.

## Review Findings

Overall correctness: patch is incorrect
