## Summary

Keep open: the requested `models auth login --profile-id` feature is still absent from current `main` and the latest release docs, but this PR still needs contributor real behavior proof and a provider-safe auth contract rework before it is mergeable.

## What This Changes

The branch adds a `--profile-id` option to `models auth login`, remaps single-profile auth results before persistence, adds a focused helper test, and updates CLI/gateway/OpenAI auth docs.

## Best Possible Solution

Land a rebased implementation that threads the requested profile id through a documented provider-safe auth context before side effects, rejects ambiguous or unsupported methods, updates docs/tests, and includes redacted real behavior proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature PR for a new CLI flag. Source inspection on current main confirms `models auth login` lacks `--profile-id` while `paste-token` has it.

## Solution Assessment

No as currently proposed: the direction is useful, but post-return remapping is not the safest provider-auth contract. The narrower maintainable solution is to pass the requested target through provider auth before persistence, or reject methods that cannot safely honor it.
