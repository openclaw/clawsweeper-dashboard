## Summary

Keep open: current main still lacks `--profile-id` for `models auth login`, and this PR addresses a real P2 feature request, but the branch is conflicting, lacks after-fix real behavior proof, and its post-return profile remap is not a provider-safe auth contract.

## What This Changes

The PR adds a `--profile-id` option to `openclaw models auth login`, remaps single-profile auth results before persistence, adds a focused helper test, and updates CLI/gateway/OpenAI auth docs.

## Best Possible Solution

Land a rebased provider-auth contract change that threads an explicit target profile id before provider side effects, rejects ambiguous or unsupported methods, and updates CLI docs/tests with real behavior proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature PR for a new CLI flag. Source inspection on current main confirms `models auth login` lacks `--profile-id` while `paste-token` has it.

## Solution Assessment

No as currently proposed: the direction is useful, but post-return remapping is not the safest provider-auth contract. The maintainable path is to pass a typed target through provider auth before persistence, or reject methods that cannot safely honor it.
