## Summary

## What This Changes

The PR adds a `--profile-id` option to `openclaw models auth login`, remaps single-profile auth results to that ID, adds focused helper tests, and documents named OAuth profile login flows.

## Best Possible Solution

Land a rebased provider-auth contract change that passes a typed target into provider auth before credentials/config are produced, rejects ambiguous or unsupported methods, and includes docs, tests, and real behavior proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new CLI flag and provider-auth capability. Source inspection confirms current main still lacks `--profile-id` on `models auth login`, while the existing token path has separate profile-id handling.

## Solution Assessment

No as currently proposed: the direction is useful, but remapping returned profiles after provider auth is not the safest contract. The maintainable path is to pass a typed target into provider auth before credentials/config are created, or reject methods that cannot safely honor it.
