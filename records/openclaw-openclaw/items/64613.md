## Summary

Keep open. Current main still routes `chat.history` and session-history snapshots through the shared display projection, but that projection has no rule for the reported plain memory-system action prompt, so the WebChat-visible leak remains source-reproducible.

## What This Changes

The issue asks for `chat.history` to hide internal memory and heartbeat system-injection prompts from WebChat-visible history.

## Best Possible Solution

Add a narrow shared display-projection rule for full-message internal memory-system action prompts while keeping raw transcript persistence unchanged and preserving ordinary quoted user text.

## Reproduction Assessment

Yes, source-level. A raw user-role transcript message whose visible text starts with `## Memory system — ACTION REQUIRED` reaches `projectRecentChatDisplayMessages`, and the current hide predicate only covers subagent announce rows, empty rows, heartbeat prompts, and heartbeat ACKs.

## Solution Assessment

Yes. The shared display projection is the narrow maintainable fix because both `chat.history` and session-history SSE converge there; a WebChat-only presentation workaround would leave other history consumers exposed.
