## Summary

Keep open. Current main still sends `chat.history` and session-history snapshots through the shared display projection, and that projection hides heartbeat prompts but has no rule for the reported `## Memory system — ACTION REQUIRED` action prompt.

## What This Changes

The issue requests that `chat.history` strip internal memory and heartbeat system-injection blocks before WebChat renders user-visible history.

## Best Possible Solution

Drop full-message internal memory action prompts in the shared Gateway display projection while preserving raw transcript persistence and the broader registry discussion separately.

## Reproduction Assessment

Yes, source-level. A raw user-role transcript message whose visible text starts with `## Memory system — ACTION REQUIRED` reaches `projectRecentChatDisplayMessages`, and the current hide predicate does not recognize that prompt shape.

## Solution Assessment

Yes. The shared display projection is the narrow maintainable boundary because both `chat.history` and session-history snapshots use it; a WebChat-only presentation filter would leave other history consumers exposed.
