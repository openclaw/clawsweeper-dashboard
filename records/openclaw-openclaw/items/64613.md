## Summary

Keep open. Current main still projects raw chat history through the shared display filter, but that filter has no rule for the reported plain `## Memory system — ACTION REQUIRED` user-role prompt shape, so the WebChat-visible leak remains source-reproducible.

## What This Changes

The issue asks for `chat.history` and WebChat display filtering to hide internal memory-system action prompts and heartbeat prompts from the user-visible transcript.

## Best Possible Solution

Add a narrow shared display-projection rule for full-message memory-system action prompts while preserving raw transcript persistence and ordinary quoted user text.

## Reproduction Assessment

Yes, at source level. A raw user-role transcript message whose visible content starts with `## Memory system — ACTION REQUIRED` and contains the auto-recall `memory_search` instruction would pass through `projectChatDisplayMessages(...)` on current main.

## Solution Assessment

Yes. The shared projection boundary is the narrowest maintainable fix because both `chat.history` and session-history SSE converge there; a WebChat-only workaround would leave other history consumers exposed.
