## Summary

Keep open. Current main filters heartbeat prompts, heartbeat acknowledgements, runtime-context envelopes, and tagged bundled Active Memory prefixes, but it still has no shared display-projection rule for the reported plain `## Memory system — ACTION REQUIRED` user-role transcript block, so that shape can still reach `chat.history` and session-history consumers.

## What This Changes

The issue asks for `chat.history`/WebChat to hide internal memory and heartbeat prompt-injection blocks before rendering visible history.

## Best Possible Solution

Keep raw transcript persistence intact and add a shared display-projection filter for full-message memory-system action prompts, with regression coverage for both `chat.history` and session-history SSE.

## Reproduction Assessment

Yes, source-level. A user-role transcript message whose visible content starts with `## Memory system — ACTION REQUIRED` flows through `projectRecentChatDisplayMessages(...)`, and current projection only hides heartbeat/runtime-envelope/tagged Active Memory shapes, not this prompt.

## Solution Assessment

Yes. The shared display projection is the narrowest maintainable fix because `chat.history` and session-history SSE converge there; a WebChat-only workaround would leave other history consumers exposed.
