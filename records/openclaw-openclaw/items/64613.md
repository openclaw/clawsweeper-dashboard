## Summary

Keep open. Current main still filters heartbeat/runtime envelopes and bundled active-memory tag blocks at the shared history projection boundary, but the reported plain memory-system ACTION REQUIRED/memory_search prompt shape still has no filter.

## What This Changes

The issue asks for WebChat/chat.history to hide internal memory and heartbeat system-injection blocks before rendering user-visible history.

## Best Possible Solution

Add a narrow shared display projection rule that treats full-message memory-system action prompts as internal-only while leaving transcript persistence and ordinary quoted user text untouched.

## Reproduction Assessment

Yes. A source-level reproduction is a raw user-role transcript message whose visible content starts with `## Memory system — ACTION REQUIRED` and contains the auto-recall `memory_search` instruction; current `projectChatDisplayMessages(...)` would pass that shape through.

## Solution Assessment

Yes. The shared projection boundary is the narrowest maintainable fix because both `chat.history` and session-history SSE already converge there; a WebChat-only workaround would leave other history consumers exposed.
