## Summary

Keep open. Current main still filters heartbeat/runtime-envelope and tagged bundled Active Memory artifacts, but the reported plain `## Memory system — ACTION REQUIRED` user-role prompt shape has no shared display-projection filter and would still reach visible history consumers.

## What This Changes

The issue asks for `chat.history` and WebChat-visible history to hide internal memory/heartbeat system-injection prompts before rendering them to users.

## Best Possible Solution

Add a narrow shared display-projection rule for full-message memory-system action prompts while leaving raw transcript persistence and ordinary quoted user text unchanged.

## Reproduction Assessment

Yes, source-level. A raw user-role transcript message whose visible content starts with `## Memory system — ACTION REQUIRED` passes through `projectChatDisplayMessages(...)` because current main only hides heartbeat/runtime-envelope/tagged Active Memory shapes, not this prompt.

## Solution Assessment

Yes. The shared display projection is the narrowest maintainable fix because both `chat.history` and session-history SSE converge there; a WebChat-only workaround would leave other history consumers exposed.
