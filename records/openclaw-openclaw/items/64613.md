## Summary

Keep open. Current main still filters heartbeat/runtime-envelope and bundled Active Memory hidden-prefix artifacts, but the reported plain `## Memory system — ACTION REQUIRED` prompt shape has no shared display-projection filter and can still reach WebChat history consumers.

## What This Changes

The issue requests stripping internal memory and heartbeat system-injection blocks from `chat.history` before WebChat renders them.

## Best Possible Solution

Add a shared display-projection rule for full-message memory-system action prompts while keeping raw transcript persistence unchanged and preserving ordinary quoted user text.

## Reproduction Assessment

Yes, source-level. A user-role transcript message whose visible content starts with `## Memory system — ACTION REQUIRED` passes through the current projection because only subagent announce rows, empty rows, heartbeat prompts, and heartbeat ACKs are hidden.

## Solution Assessment

Yes. The shared display projection is the narrowest maintainable fix because both `chat.history` and session-history SSE converge there; a WebChat-only workaround would leave other history consumers exposed.
