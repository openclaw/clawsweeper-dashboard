## Summary

Keep open. Current main still routes `chat.history` and session-history output through the shared display projection, and that projection hides heartbeat/internal envelope artifacts but has no rule for the reported `## Memory system — ACTION REQUIRED` prompt shape.

## What This Changes

The issue asks OpenClaw to strip internal memory and heartbeat injection blocks from customer-visible WebChat `chat.history` output.

## Best Possible Solution

Add a narrow shared Gateway display-projection filter for full-message internal memory action prompts, with regression coverage, while leaving raw transcript persistence unchanged.

## Reproduction Assessment

Yes, source-level. A raw visible message starting with the reported memory-system prompt reaches `projectRecentChatDisplayMessages`, and the current hide predicate does not recognize that shape.

## Solution Assessment

Yes. The shared Gateway display projection is the narrow maintainable boundary because both `chat.history` and session-history snapshots use it; a WebChat-only filter would leave other consumers exposed.
