## Summary

Keep open. Current main still has a shared `chat.history`/session-history display projection boundary, and that boundary hides heartbeat/runtime-envelope and bundled active-memory tag-block shapes, but it does not hide the reported plain `## Memory system — ACTION REQUIRED` auto-recall prompt shape.

## What This Changes

The issue asks for WebChat/chat.history to hide internal memory and heartbeat system-injection blocks instead of returning them to user-visible history.

## Best Possible Solution

Add a shared visible-history projection rule for known internal memory-system action prompt blocks so both `chat.history` and session-history SSE hide them, while keeping transcript persistence unchanged and preserving normal user text that merely quotes those phrases.

## Reproduction Assessment

Yes. A source-level reproduction is a raw user-role transcript message whose content starts with `## Memory system — ACTION REQUIRED` and contains the auto-recall `memory_search` instruction; current `projectChatDisplayMessages(...)` has no predicate for that shape and would pass it through. I did not run Vitest because this review was read-only.

## Solution Assessment

Yes. The shared projection boundary is the narrowest maintainable fix because both `chat.history` and session-history SSE already converge there; a WebChat-only workaround would leave other history consumers exposed.
