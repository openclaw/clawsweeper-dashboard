## Summary

Keep open. Current main still routes `chat.history` and session-history snapshots through the shared display projection, but that projection has no rule for the reported memory-system action prompt; the heartbeat half is already covered.

## What This Changes

The issue asks for WebChat-visible `chat.history` output to strip internal memory and heartbeat system-injection prompts before rendering.

## Best Possible Solution

Drop full-message internal memory action prompts in the shared display projection while keeping raw transcript persistence unchanged.

## Reproduction Assessment

Yes, source-level. A raw user-role transcript message whose visible text starts with `## Memory system — ACTION REQUIRED` reaches `projectRecentChatDisplayMessages`, and the current hide predicate does not recognize that prompt shape.

## Solution Assessment

Yes. The shared display projection is the maintainable boundary because both `chat.history` and session-history snapshots use it; a WebChat-only presentation filter would leave API/SSE consumers exposed.
