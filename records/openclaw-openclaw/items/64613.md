## Summary

Keep open. Current main still sends raw history through the shared display projection, but that projection only hides heartbeat/runtime-envelope artifacts and tagged bundled active-memory blocks; the reported plain `## Memory system — ACTION REQUIRED` prompt shape still has no filter.

## What This Changes

The issue requests that `chat.history` hide internal memory and heartbeat system-injection prompts before WebChat or Control UI renders them.

## Best Possible Solution

Add a narrow shared display-projection rule for full-message memory-system action prompts while leaving raw transcript persistence and ordinary quoted user text unchanged.

## Reproduction Assessment

Yes. A source-level reproduction is a raw user-role transcript message whose visible content starts with `## Memory system — ACTION REQUIRED` and contains the auto-recall `memory_search` instruction; current `projectChatDisplayMessages(...)` would pass it through.

## Solution Assessment

Yes. The shared projection boundary is the narrowest maintainable fix because both `chat.history` and session-history SSE already converge there; a WebChat-only workaround would leave other history consumers exposed.
