## Summary

Keep open. Current main has shipped fixes for heartbeat prompts, hidden runtime-context rows, and pre-compaction memory-flush prompts, but the shared `chat.history` projection still has no rule for the reported `## Memory system — ACTION REQUIRED` / `memory_search` prompt shape; the related open PR is also not a complete or mergeable fix.

## What This Changes

The issue asks for WebChat `chat.history` to hide internal memory and heartbeat system-injection prompt blocks before returning history to UI clients.

## Best Possible Solution

Add provenance-aware shared Gateway display-projection filtering for full-message internal memory action prompts, while preserving raw transcript persistence and the existing hidden custom-message contract for plugins.

## Reproduction Assessment

Yes, source-level. `chat.history` routes raw messages through the shared projection, and the current hide predicate does not recognize the reported memory-action prompt shape, so such a raw user-role prompt would remain visible.

## Solution Assessment

Yes for the shared boundary, but the exact filter should be provenance-aware or narrowly full-message based. A WebChat-only presentation workaround would leave session-history and other clients exposed.
