## Summary

## What This Changes

## Best Possible Solution

Rework voice-call so the first valid block reply reaches TTS before the embedded run waits on compaction, then prove it with a compaction-delayed regression test and redacted real voice-call evidence.

## Solution Assessment

No. The patch is a plausible stale-payload mitigation, but it is not the best fix for the reported latency because it only consumes callback payloads after the awaited embedded run settles.
