## Summary

This PR remains necessary because current main still rejects the reported subcommand-position `--port` flag, and the linked issue remains open. The implementation is technically sound and live-proven, with one low-priority documentation mismatch to correct before merge.

## What This Changes

Adds `--port` targeting to gateway health, probe, and shared RPC query commands while preserving config-derived authentication and TLS and giving an explicit CLI port precedence over `OPENCLAW_GATEWAY_PORT`.

## Best Possible Solution

Keep the config-patching implementation and explicit CLI precedence, but document probe’s multi-target URL/port behavior precisely and retain focused coverage for both flag-present and flag-omitted paths.

## Reproduction Assessment

Yes. Current main lacks subcommand-level `--port` registration, reproducing the reported Commander rejection, and the PR provides a real after-fix run against a non-default gateway port.

## Solution Assessment

Mostly yes. Patching the loaded config instead of synthesizing a URL correctly preserves authentication and TLS ownership, but the documented combined URL/port semantics for probe need correction.
