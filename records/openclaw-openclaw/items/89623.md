## Solution Assessment

No: the proposed shape is close, but patching `config.gateway.port` is not the best final fix because existing port resolution lets `OPENCLAW_GATEWAY_PORT` override it. The safer fix is an explicit parsed-port path that preserves auth/TLS while making CLI precedence unambiguous.
