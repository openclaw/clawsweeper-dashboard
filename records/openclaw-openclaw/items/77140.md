## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require a finite systemd memory limit:** `src/agents/bash-tools.exec.ts:1296-1303`
  - body: `analyzeSystemdRunMemoryScope` treats any `MemoryMax=` property as approved isolation. systemd accepts `MemoryMax=infinity` to apply no memory limit, so `systemd-run --user --scope -p MemoryMax=infinity -- corepack pnpm lint:core` bypasses the heavy-command guard while still allowing unbounded host resource use. Parse the value and classify the payload unless the limit is finite and meaningful.
  - confidence: 0.9
- **[P3] Update exec docs for the new guard:** `CHANGELOG.md:429`
  - body: The changelog announces a new Gateway exec refusal and bypass behavior, but `docs/tools/exec.md` and `docs/tools/exec-approvals.md` still say full/YOLO host exec has no extra heuristic prefilter or rejection layer. Update those docs with known-heavy refusals, approved wrapper configuration, maybe-heavy diagnostics, and the emergency bypass so operators do not configure exec from stale policy text.
  - confidence: 0.86
