## Summary

Keep open: the PR addresses a real exec-resource hardening gap that is not present on current main, but the branch still misses option-prefixed package-manager commands and needs maintainer approval for the fail-closed exec policy before merge.

## What This Changes

The PR adds a Gateway exec preflight that classifies heavy validation commands, refuses unisolated always-heavy local runs, allows resource-isolation wrappers, and adds analyzer/blocking tests.

## Best Possible Solution

Normalize package-manager argv before classification, add regression coverage for option-prefixed workspace forms, add the required changelog entry, then land the guard only after maintainer approval of the exec policy contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the PR bug: `packageManagerHeavyReason` reads `args[0]`, while current `package.json` contains `pnpm --dir ui test`, so that heavy test command would not be classified.

## Solution Assessment

No, not yet. An exec-entry guard is a plausible narrow direction, but this patch needs package-manager option parsing and maintainer approval for the command policy, wrapper, and bypass semantics.
