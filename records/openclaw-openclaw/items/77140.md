## Summary

Keep open: current main still allows the reported unisolated Gateway exec path, and this PR is a plausible fix, but the patch should not merge yet because the new wrapper exemption can be bypassed with a checkout-local same-named wrapper and the required changelog entry is still missing.

## What This Changes

The PR adds a Gateway exec preflight classifier/refusal path for unisolated heavy validation commands and tests analyzer behavior for direct, shell-wrapped, maybe-heavy, isolated, and execution-path cases.

## Best Possible Solution

Land a maintainer-approved exec guard that only trusts real resource-isolation wrappers or explicit systemd memory scopes, includes regression coverage and a changelog entry, and then lets the linked issue close after merge.

## Reproduction Assessment

Yes at source level: current main documents unsandboxed gateway exec as a normal default path and has no heavy-command guard symbols in the exec implementation. I did not attempt live OOM reproduction in this read-only review.

## Solution Assessment

No as currently written: an exec-entry guard is a narrow and plausible mitigation, but the wrapper exemption is too broad and the changelog is missing. Tighten wrapper trust/regression coverage first, then have maintainers approve the fail-closed policy and bypass name.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require a trusted heavy-run wrapper:** `src/agents/bash-tools.exec.ts:1188-1193`
  - body: This exemption is based only on the basename, and the interpreter branch accepts `node openclaw-heavy-run` / `python openclaw-heavy-run` as already isolated. That means a checkout-local `./openclaw-heavy-run -- corepack pnpm lint:core` shim can be treated as wrapped and then run the heavy command in the Gateway cgroup, bypassing the guard's fail-closed path. Please only exempt a trusted/configured wrapper form or an explicit systemd memory scope, and add regression coverage for path-qualified/interpreter-invoked wrapper lookalikes.
  - confidence: 0.87

  - confidence: 0.9
