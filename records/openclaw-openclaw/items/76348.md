## Summary

Keep open. The performance direction is plausible, but the diff introduces stale-cache paths for hook-mutated bootstrap context and auth-bound model discovery, so this PR needs repair before merge.

## What This Changes

The PR adds in-memory LRU caches for resolved bootstrap contexts and PI model discovery, adjusts auth-store cache keys, passes channel id into bootstrap resolution, and adds plugin tool-cache kill switches.

## Best Possible Solution

Keep the perf direction, but merge only after cache keys cover hook-produced bootstrap inputs and auth-bound model registries, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable bootstrap-extra-files, resolve context once, edit a matched extra file without touching root bootstrap files, and the PR cache can return the old context before hooks rerun.

## Solution Assessment

No. Caching final hook-mutated context and auth-bound registries is too broad; the safer fix is to cache only fully fingerprinted inputs, bypass final-context caching when bootstrap hooks can mutate files, and scope model discovery to the credential source.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Do not reuse hook-adjusted bootstrap snapshots blindly:** `src/agents/bootstrap-files.ts:380-382`
  - body: The new cache returns before `applyBootstrapHookOverrides` runs. The bundled `bootstrap-extra-files` hook reads `hooks.internal.entries.bootstrap-extra-files.paths` and the matched file contents during `agent:bootstrap`, but this key fingerprints only the root bootstrap files plus the hook registry version. Editing an injected `packages/*/AGENTS.md` file or changing hook paths can keep serving the old context until some unrelated root bootstrap file changes.
  - confidence: 0.92

  - body: `discoverModels` still accepts a caller-provided `authStorage`, and `createOpenClawModelRegistry` binds the registry to that object. The new cache key omits the storage identity and env/runtime credential fingerprint, so a later call with a different in-memory auth store but the same agentDir/options can receive the first registry and resolve availability against stale credentials.
  - confidence: 0.88

  - body: This is a user-facing `perf` runtime change, and repo policy requires `fix`, `feat`, and `perf` PRs to update `CHANGELOG.md`. Please add a single-line entry under the active release section.
