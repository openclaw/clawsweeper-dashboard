## Summary

Keep open: current main still exposes the fragile `type` property in the shared message tool schema, and this PR is a narrow, well-scoped fix with backward-compatible Discord parsing and sufficient live terminal proof. I found no blocking correctness or security issue in the diff.

## What This Changes

The PR renames the shared message tool's Discord channel-create schema field from `type` to `channelType` and updates Discord parsing to prefer `channelType` while accepting legacy `type`.

## Best Possible Solution

Land the narrow schema rename and parser fallback, ideally with focused schema/parser regression coverage added during maintainer landing.

## Reproduction Assessment

Yes. The PR body gives a concrete live NVIDIA NIM replay path and after-fix terminal output, and current-main source inspection confirms the triggering `type` property is still present in the message tool schema.

## Solution Assessment

Yes. Renaming the exposed schema property to `channelType` while preserving a legacy `type` fallback is the narrow maintainable fix; a provider-specific rewrite would be broader and would not address the generic fragile field name.
