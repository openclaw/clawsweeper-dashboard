## Summary

Keep open: current main still lacks a first-class one-way dispatch contract for ordinary A2A `sessions_send`; the code and docs still expose conversation-style wait/reply-back/announce semantics, while related open work covers narrower safeguards or broader proposals rather than this requested mode.

## What This Changes

The issue requests an explicit one-way A2A dispatch or handoff mode that delivers work into a target agent session without reply-back ping-pong or requester announce behavior.

## Best Possible Solution

Define one canonical A2A delivery intent that preserves existing conversation mode while adding tested dispatch behavior that never reinjects the target reply or runs announce unless explicitly requested.

## Reproduction Assessment

Yes for source-level reproduction: ordinary `sessions_send` currently has no dispatch selector and still routes through the A2A reply-back and announce helper. I did not run a live two-agent handoff in this read-only review.

## Solution Assessment

Unclear as an implementation choice: a first-class delivery intent is the right level compared with prompt-only workarounds, but maintainers still need to choose the permanent model-facing API shape.
