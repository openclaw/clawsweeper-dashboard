## Summary

Keep open. Current main still has no first-class one-way dispatch mode for `sessions_send`: the public schema has no dispatch/private/reply-back control, and normal async or waited sends still start the A2A reply-back plus announce lifecycle. Related open work is useful context, but it is not merged and does not supersede this product/API decision.

## What This Changes

Request a first-class one-way dispatch or handoff mode for `sessions_send` so target sessions receive work without reply-back ping-pong or announce back to the requester.

## Best Possible Solution

Define a backward-compatible dispatch/private intent in the A2A handoff contract, update the public tool schema/docs and focused tests, and preserve existing conversation mode as the default.

## Reproduction Assessment

Yes for the current source-level behavior: `sessions_send` has no dispatch parameter and enters the A2A reply/announce flow for normal async and waited sends. This is a feature request for new semantics, not a failure of an existing documented contract.

## Solution Assessment

Unclear until maintainers choose the public API shape. A first-class dispatch mode or primitive is the right class of solution, but it should align with https://github.com/openclaw/openclaw/pull/63789 and the open private-delivery discussion instead of relying on prompt conventions or `timeoutSeconds` behavior.
