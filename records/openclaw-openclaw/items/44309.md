## Summary

Keep open. Current main and v2026.5.12 still expose only the existing `sessions_send` conversation path, so the requested one-way dispatch/handoff semantic remains a real feature/API gap rather than an obsolete report.

## What This Changes

Add a one-way `sessions_send` dispatch or handoff mode that delivers a task to a target agent session without reply-back reinjection or announce behavior.

## Best Possible Solution

Define a backward-compatible dispatch intent in the A2A handoff contract, with explicit tool schema, optional delivery acknowledgement semantics, docs, and regression tests that preserve the existing conversation mode.

## Reproduction Assessment

Yes for source-level current behavior: current main has no dispatch parameter or primitive, and ordinary peer sends still enter the A2A reply/announce flow. No live A2A run was executed in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API shape. A first-class dispatch mode or primitive is the right class of solution, but it should align with the open private-delivery and handoff-protocol work instead of relying on prompts or `timeoutSeconds`.
