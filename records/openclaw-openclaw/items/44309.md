## Summary

Keep open. Current main still exposes only conversation-oriented `sessions_send` behavior, and the related open private-delivery PR is not merged and appears to suppress only final announce delivery rather than the full requester reply-back ping-pong this issue asks to avoid.

## What This Changes

Request adds an explicit one-way dispatch mode for `sessions_send` so A2A handoffs enqueue target work without reply-back ping-pong or announce.

## Best Possible Solution

Add a backward-compatible per-call delivery intent to `sessions_send` that persists or starts the target session with an optional acknowledgement while skipping requester reply-back and target announce for dispatch mode.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `sessions_send` with `timeoutSeconds: 0` or a normal peer send, and current main schedules `runSessionsSendA2AFlow` unless the narrow parent-owned skip applies.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable boundary is a per-call delivery intent, but it must skip both requester reply-back and announce rather than only suppressing the final external send.
