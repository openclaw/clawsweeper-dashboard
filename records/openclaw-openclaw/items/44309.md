## Summary

Keep open. Current main still treats `sessions_send` peer delivery as A2A conversation plus announce, with only a parent-owned child skip and a ping-pong turn cap; it does not expose a first-class one-way dispatch/handoff mode.

## What This Changes

Request adds an explicit dispatch-only handoff mode or primitive for `sessions_send` so target sessions receive work without reply-back reinjection or announce.

## Best Possible Solution

Define an additive A2A delivery-intent contract that preserves existing conversation behavior by default while giving callers an explicit one-way dispatch path, with source/tests/docs updated together.

## Reproduction Assessment

Not applicable as a feature request. Source and tests do confirm the current-main behavior: peer `sessions_send` still enters A2A reply/announce flow unless it hits the narrow parent-owned child skip.

## Solution Assessment

Unclear until maintainers choose the API shape. A first-class delivery intent looks like the right direction, but `mode: "dispatch"` versus a new primitive and ack semantics should be settled before implementation.
