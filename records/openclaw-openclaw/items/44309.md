## Summary

Keep open. Current main still lacks a first-class one-way dispatch mode for `sessions_send`: the public tool schema has no delivery intent, fire-and-forget still schedules A2A follow-up, and the helper still performs reply-back plus announce for normal peer sends.

## What This Changes

Add explicit one-way dispatch or handoff semantics for `sessions_send` or a new primitive so a task can be delivered into a target session without requester reply-back or announce.

## Best Possible Solution

Add a backward-compatible delivery intent to `sessions_send`, keep the current conversation behavior as the default, and make dispatch mode enqueue/start the target session with a minimal acknowledgement while skipping requester ping-pong and target announce.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `timeoutSeconds: 0` still enters `startA2AFlow`, and normal peer sends still run `runSessionsSendA2AFlow` for reply-back plus announce.

## Solution Assessment

Yes. A first-class per-call delivery intent is the narrow maintainable solution; prompt conventions, global `maxPingPongTurns`, and the narrow ACP parent-child skip do not provide clean one-way handoff semantics.
