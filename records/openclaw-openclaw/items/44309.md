## Summary

Keep open. Current main still lacks an explicit one-way dispatch or handoff mode for `sessions_send`: the public tool schema has no delivery-intent parameter, fire-and-forget still schedules the A2A follow-up path, and the A2A helper still performs requester-side reply-back plus announce behavior. The related open issues cover adjacent announce/guard semantics, but they are not a canonical replacement for this per-call dispatch mode.

## What This Changes

Add a one-way dispatch or handoff mode for agent-to-agent `sessions_send` delivery so a target session receives work without reply-back reinjection or announce behavior in the requester session.

## Best Possible Solution

Add a backward-compatible explicit delivery intent for `sessions_send`, with current conversation behavior as the default and a dispatch path that enqueues into the target session using existing inter-session provenance while skipping requester reply-back and announce delivery. Keep related announce/callback hardening tracked separately unless the implementation naturally shares tests or helpers.

## Repair Work Prompt

Implement the requested one-way A2A handoff mode for `sessions_send` from https://github.com/openclaw/openclaw/issues/44309. Add a backward-compatible explicit delivery intent, preferably `mode: "conversation" | "dispatch"`, with current behavior as the default. In dispatch mode, enqueue/start the target session using the existing internal-channel and inter-session provenance path, return a minimal delivery acknowledgement, and do not call `runSessionsSendA2AFlow`; there should be no requester-side reply reinjection, no ping-pong, and no announce unless a separate explicit future option is designed. Keep A2A policy/visibility checks and existing conversation semantics unchanged. Update focused tests for schema, default conversation behavior, dispatch with no A2A scheduling, and docs. Update public docs for the session tool and add a user-facing changelog entry with `Thanks @Kaze310` if this remains user-visible. Do not fold broader callback/announce policy redesign from #8299, #33478, or #39476 into this PR unless a tiny shared helper is necessary.
