## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass numeric agent timeouts:** `src/agents/tools/sessions-send-tool.ts:251`
  - body: The gateway schema requires `agent.params.timeout` to be an integer and the handler rejects invalid params before dispatch. This new `String(...)` timeout, and the matching helper change, makes real nested agent calls fail validation even though mocks accept it.
  - confidence: 0.93
- **[P1] Preserve fire-and-forget run budgets:** `src/agents/tools/sessions-send-tool.ts:220`
  - body: For `timeoutSeconds: 0`, sessions_send is documented as enqueue-and-return, while agent runs keep their normal runtime budget. Forcing the target run to `30` seconds turns fire-and-forget into a short hard deadline and can kill valid background work.
  - confidence: 0.9
- **[P2] Align helper wait and run deadlines:** `src/agents/tools/agent-step.ts:42`
  - body: `targetTimeoutSeconds` is derived from the full `params.timeoutMs`, but the helper still waits with `Math.min(params.timeoutMs, 60_000)`. For values above 60 seconds, the caller returns while the target run continues under a longer budget.
  - confidence: 0.84
- **[P2] Count lock waits against one run deadline:** `src/agents/pi-embedded-runner/run/attempt.ts:1705`
  - body: Passing the full run timeout into lock acquisition does not by itself make one budget: the embedded run abort timer starts later, after lock acquisition. A contended run can spend the full timeout waiting for the lock and then receive another full execution window.
  - confidence: 0.82
