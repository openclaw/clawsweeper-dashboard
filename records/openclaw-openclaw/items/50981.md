## Summary

Keep open. The underlying timeout mismatch still exists on current main, but this PR is not mergeable as written: it forwards string `agent.timeout` values that violate the gateway schema, changes fire-and-forget target-run budgets, and does not keep lock, target-run, and helper wait budgets aligned.

## What This Changes

The PR forwards `sessions_send` and A2A timeout values into nested `agent` runs, passes embedded-runner `timeoutMs` into session write-lock acquisition, and adds tests for those propagation paths.

## Best Possible Solution

Revise or replace the patch with a deadline-aware timeout model: forward numeric `agent.timeout` only where appropriate, preserve fire-and-forget target-run semantics, count session-lock wait against the same effective run budget, align A2A target and wait budgets, and cover those contracts with protocol and focused agent tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main omits target-run and lock wait timeout propagation, while the PR's new string timeout fields would be rejected by `validateAgentParams` before nested runs start.

## Solution Assessment

No. The direction is useful, but this implementation is not the best fix because it violates the gateway type contract and changes timeout semantics; a safer revision should use numeric protocol values and one shared deadline/budget model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Forward numeric sessions_send timeouts:** `src/agents/tools/sessions-send-tool.ts:251`
  - body: The `agent` RPC schema declares `timeout` as an integer and gateway validation rejects mismatches before dispatch. Sending `String(targetTimeoutSeconds)` makes real `sessions_send` calls fail with invalid agent params instead of starting the target run.
  - confidence: 0.94
- **[P1] Forward numeric A2A helper timeouts:** `src/agents/tools/agent-step.ts:52`
  - body: This helper sends the new `timeout` field through the same validated `agent` RPC. Passing a string here means A2A helper runs that include the field are rejected before the nested run starts.
  - confidence: 0.93
- **[P1] Preserve fire-and-forget target budgets:** `src/agents/tools/sessions-send-tool.ts:220`
  - body: For `timeoutSeconds: 0`, the caller should return immediately while the target run keeps the normal agent budget. Forcing a 30-second target timeout cuts legitimate background work short and changes fire-and-forget behavior.
  - confidence: 0.88
- **[P2] Keep lock waits inside the run budget:** `src/agents/pi-embedded-runner/run/attempt.ts:1706`
  - body: Using the full `params.timeoutMs` for lock acquisition lets a contended run spend the whole caller budget waiting for the lock and then receive another full model runtime budget. Use a shared deadline or remaining budget instead.
  - confidence: 0.82
- **[P2] Align helper target and wait budgets:** `src/agents/tools/agent-step.ts:42`
  - body: For `timeoutMs` values above 60 seconds, the target run gets the full timeout while the helper still waits at most 60 seconds, leaving the child run active after `runAgentStep` has returned `undefined`.
  - confidence: 0.84
