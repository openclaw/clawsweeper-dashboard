## Summary

The patch is a narrow, source-backed fix for the Telegram partial preview regression, but this external PR should stay open because its proof is only a mocked regression test rather than real Telegram behavior after the fix.

## What This Changes

This PR changes Telegram draft preview startup so only `block` mode keeps the 30-character first-message debounce, and adds a regression test for short direct partials finalizing through the draft edit path.

## Best Possible Solution

Land the Telegram plugin fix after real Telegram DM proof shows a short partial creates the preview before final delivery while block-mode debounce remains intact.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `partial` mode receives a 30-character initial threshold, and the draft stream suppresses a short first update below that threshold before any preview message exists.

## Solution Assessment

Yes for the code direction. The proposed conditional is the narrowest maintainable fix because it changes only Telegram draft preview threshold selection and preserves the existing debounce for `block` mode.
