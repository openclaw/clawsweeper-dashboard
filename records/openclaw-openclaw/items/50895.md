## Summary

Keep open: the WhatsApp BR ninth-digit equivalence bug is source-reproducible on current main, but this PR still has blocking target-rewrite issues and lacks after-fix real WhatsApp runtime proof.

## What This Changes

The PR adds Brazilian ninth-digit equivalence for WhatsApp direct targets across outbound resolution, inbound access control, monitor routing, heartbeat recipient selection, SDK exports, and regression tests.

## Best Possible Solution

Implement BR direct-target equivalence inside the current WhatsApp plugin authorization and ingress paths while preserving the requested normalized chat target for delivery, reactions, quote lookup, and session keys.

## Reproduction Assessment

Yes, by source inspection. Current main exact-matches `allowFrom`, sender, target, command, self-chat, and main-route values, so the two BR forms remain distinct in the affected WhatsApp paths.

## Solution Assessment

No. The equivalence rule is the right direction, but returning the allowlist's canonical target mixes authorization with delivery/session routing and can send reactions or history updates to the wrong chat identity.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the requested WhatsApp target after authorization:** `src/whatsapp/resolve-outbound-target.ts:50`
  - body: When a BR variant matches, returning `matchedAllowTarget` rewrites the caller's chat target to the allowlist entry. Reaction handling sends to the resolved target, so a reaction for a message in `+5535998627740` can be sent with the `+553598627740` remote JID. Keep authorization equivalence separate from the delivery target.
  - confidence: 0.91
- **[P2] Keep session routing keyed to the requested chat target:** `src/whatsapp/resolve-outbound-target.ts:50`
  - body: The same `matchedAllowTarget` rewrite feeds outbound session routing. In per-peer modes, sending to one BR variant while the allowlist stores the other records the turn under the allowlist peer key, splitting history from the active chat and making later implicit routing stale.
  - confidence: 0.88
