## Summary

Keep this PR open. The BR ninth-digit WhatsApp problem is still present on current main, but the proposed branch is not merge-ready because it authorizes variant matches by rewriting the requested chat target, which can break reactions and per-peer session routing.

## What This Changes

The PR adds Brazilian WhatsApp ninth-digit equivalence helpers and applies them across outbound and inbound WhatsApp authorization, command/self-chat/main-route/heartbeat paths, SDK exports, and regression tests.

## Best Possible Solution

Port the useful BR direct-target equivalence into the current extensions/whatsapp plugin layout as an internal helper. Use it for allowlist, command, self-chat, main-route owner, and heartbeat authorization decisions, while preserving the requested normalized chat target for delivery, reactions, quote lookup, and per-peer session routing.

## Reproduction Assessment

Yes. Configure WhatsApp allowFrom with +553598627740 and send or receive from +5535998627740; current main uses exact comparisons, while the PR branch authorizes the pair but then returns the allowlist variant downstream.

## Solution Assessment

No. The equivalence direction is useful, but returning the allowlist target and widening old public SDK exports are not the narrowest maintainable fix; authorization equivalence should be separated from delivery/session target selection inside the WhatsApp plugin.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve explicit WhatsApp targets after authorization:** `src/whatsapp/resolve-outbound-target.ts:52`
  - body: When a BR variant matches, returning `matchedAllowTarget` rewrites the caller's chat target to the allowlist entry. Reaction handling passes the resolved target into the reaction send path, so a reaction for a message in `+5535998627740` can be sent with the `+553598627740` remote JID. Keep authorization equivalence separate from the delivery target.
  - confidence: 0.91
- **[P2] Keep session routing on the requested chat target:** `src/whatsapp/resolve-outbound-target.ts:52`
  - body: The same `matchedAllowTarget` rewrite feeds outbound session routing. In per-peer modes, sending to one BR variant while the allowlist stores the other records the turn under the allowlist peer key, splitting history from the active chat and making later implicit routing stale.
  - confidence: 0.88
