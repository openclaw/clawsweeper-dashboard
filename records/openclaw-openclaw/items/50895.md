## Summary

Keep this PR open: the Brazilian WhatsApp ninth-digit bug is still visible on current main, and the branch contains useful equivalence work, but the latest head still rewrites authorized targets and is now conflicting/outdated against the plugin-owned WhatsApp layout. The author addressed several earlier follow-ups, but the later ClawSweeper review blockers remain valid.

## What This Changes

The PR adds Brazilian WhatsApp ninth-digit target equivalence helpers and wires them into outbound resolution, inbound access control, monitor routing, SDK exports, and regression tests.

## Best Possible Solution

Implement BR direct-target equivalence inside the current WhatsApp plugin target and ingress authorization paths while preserving the requested normalized chat target for delivery, reactions, quote lookup, and session routing.

## Reproduction Assessment

Yes, by source inspection. On current main, `allowFrom` and sender/target values like `+553598627740` and `+5535998627740` flow through exact-match outbound, inbound, command, and same-phone comparisons, while the PR branch authorizes that pair but rewrites the downstream target.

## Solution Assessment

No. The equivalence rule is the right fix direction, but returning the allowlist target and widening old core/SDK surfaces is not the narrowest maintainable solution for the current plugin layout.
