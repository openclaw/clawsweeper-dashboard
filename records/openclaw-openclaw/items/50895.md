## Summary

Keep this PR open: it addresses a real WhatsApp BR ninth-digit gap that still exists on current main, but the branch is now stale against the plugin-owned layout and its outbound target rewrite would break reaction and per-peer session routing.

## What This Changes

The PR adds Brazilian WhatsApp ninth-digit direct-target equivalence helpers, SDK exports, authorization/routing integrations, and regression tests.

## Best Possible Solution

Implement BR direct-target equivalence inside the current WhatsApp plugin authorization paths while preserving the requested normalized chat target for delivery, reactions, quote lookup, and session routing.

## Reproduction Assessment

Yes. On current main, allowFrom +553598627740 and target/sender +5535998627740 fail exact allowlist and same-phone comparisons; the PR branch authorizes that pair but then rewrites the downstream target to the allowlist variant.

## Solution Assessment

No. The equivalence rule is the right direction, but returning the allowlist target and widening old core SDK exports are not the narrowest maintainable fix for the current plugin layout.
