Keep open: the linked diagnostic bug is real, but this patch is not the best fix because it removes meaningful action words from many exec summaries. The new proof uses a one-off tsx harness rather than a real cron/chat/CLI warning path, so merge still needs contributor-supplied real behavior proof.

## What This Changes

The PR adds a helper that strips leading exec display verbs before building exec/bash failure warning payloads and updates focused payload/tool-display tests to expect stripped subjects.

## Best Possible Solution

Render the tool/failure label outside the backtick-wrapped command or subject while preserving the exec summary action, then prove the result with a redacted real cron/chat/CLI warning.

## Reproduction Assessment

Yes, at source level: current main stores exec display metadata in lastToolError, formats it with formatToolAggregate, and appends failed, so command-shaped summaries can remain visually ambiguous. I did not run a live cron/chat reproduction in this read-only review.

## Solution Assessment

No. The patch fixes the ambiguity by deleting the first display-summary word, but the best fix is to preserve the action label and change the failure-rendering shape so action and literal command remain visually distinct.
