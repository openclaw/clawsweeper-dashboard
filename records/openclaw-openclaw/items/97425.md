## Summary

Keep open: the PR targets a real exec failure-message bug, but the current patch removes meaningful exec summary action words instead of only separating framework failure metadata from the backtick-wrapped command or subject.

## What This Changes

The PR adds an exec-display helper that strips leading display verbs before formatting exec/bash failure warnings and updates payload/tool-display tests for the verb-stripped output.

## Best Possible Solution

Render the tool failure label separately from exec metadata while preserving meaningful summary action words and raw-command details.

## Reproduction Assessment

Yes, at source level: current main stores exec display metadata in lastToolError, formats it with formatToolAggregate, and appends failed, so command-shaped summaries can remain visually ambiguous. I did not run a live cron or chat reproduction in this read-only review.

## Solution Assessment

No. The patch fixes the ambiguity by deleting the first display-summary word, but the safer fix is to preserve the action label and change the failure-rendering shape so action and literal command remain visually distinct.
