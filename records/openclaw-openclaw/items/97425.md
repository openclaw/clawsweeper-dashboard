## What This Changes

The PR adds an exec display helper that strips leading verbs before formatting exec/bash failure warnings, then updates payload and tool-display tests to expect verb-stripped warning subjects.

## Best Possible Solution

Separate the framework/tool failure label from the backtick-wrapped command or subject while preserving the semantic exec action label in the rendered warning.

## Reproduction Assessment

Yes, at source level: current main stores exec display metadata in lastToolError, formats it with formatToolAggregate, and appends failed, so command-shaped summaries can remain visually ambiguous. I did not run a live cron or chat reproduction in this read-only review.

## Solution Assessment

No. The patch fixes the ambiguity by deleting the first display-summary word, but the best fix is to preserve the action label and change the failure-rendering shape so action and literal command remain visually distinct.
