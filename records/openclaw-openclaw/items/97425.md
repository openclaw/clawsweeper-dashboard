## What This Changes

The PR strips leading exec display verbs from exec/bash failure-warning metadata before passing it to the aggregate formatter and updates formatter/payload tests for the new warning text.

## Best Possible Solution

Separate the tool failure label from exec metadata while preserving action-bearing summaries and raw-command details, then land only after redacted real cron/chat/CLI proof shows the rendered warning.

## Reproduction Assessment

Yes, at source level. Current main stores exec display metadata in `lastToolError`, formats it with `formatToolAggregate(...)`, and appends `failed`, so command-shaped summaries such as `run python3 ...` remain visually ambiguous without needing a live run to trace the path.

## Solution Assessment

No. The PR fixes the reported ambiguity by deleting the first word of every known summary, but current summaries include semantic actions; preserving the summary and changing the failure rendering is safer.
