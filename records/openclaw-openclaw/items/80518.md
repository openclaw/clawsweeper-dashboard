## Summary

Keep open. Current main extracts and displays reasoning in Control UI, but completed reasoning renders as a static `.chat-thinking` block and the live stream path disables reasoning extraction, so the requested auto-collapse/toggle behavior is not implemented and needs UI product direction.

## What This Changes

Request to make Control UI reasoning/`<think>` blocks stay expanded while active, auto-collapse after completion, and remain manually expandable.

## Best Possible Solution

Design a Control UI reasoning disclosure component that keeps the active reasoning segment expanded while streaming and collapses completed reasoning behind accessible toggles, reusing existing chat disclosure patterns and adding focused render tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new Control UI behavior request. Source inspection gives a high-confidence current-behavior path showing reasoning is parsed and rendered, but not collapsed or toggleable.

## Solution Assessment

Unclear until maintainer UX direction is set. The requested direction is plausible, but the maintainable implementation should define streaming/completed state semantics and likely reuse existing chat disclosure controls rather than adding ad hoc markup.
