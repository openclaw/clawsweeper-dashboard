## Summary

Keep open: this is a focused iMessage message-delivery fix with useful Lobster proof, but the patch currently replaces a basic-mode media path with an unconditional private-API attachment command for explicit chat media-only sends. That can break existing SIP-enabled/basic-mode setups unless the new path is gated or falls back safely.

## What This Changes

The PR adds an iMessage CLI JSON helper and routes media-only explicit iMessage chat targets through `imsg send-attachment --transport auto`, with tests for chat GUID, chat ID, and DM handle behavior.

## Best Possible Solution

Keep the attachment-command fix for affected group media, but gate it on private-API/command availability or fall back to the existing RPC send path when the advanced attachment command is unavailable.

## Reproduction Assessment

Yes for the PR regression from source: OpenClaw docs and imsg docs show basic media send is supported without private API, while the PR unconditionally calls the advanced `send-attachment` command for explicit-chat media-only sends. The original group-media failure has Lobster proof in the PR context but was not re-run in this read-only review.

## Solution Assessment

No: routing affected group media through `send-attachment --transport auto` is plausible, but replacing the current path without capability gating or fallback is not the safest maintainable fix.
