## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the current root-file reader API:** `src/agents/scoped-working-memory.ts:3`
  - body: Current main's `src/infra/boundary-file-read.ts` exports `openRootFile`/`openRootFileSync`, not `openBoundaryFile`. This new import will fail after merge or rebase, so the helper needs to use the current safe reader contract.
  - confidence: 0.94
- **[P2] Keep scoped memory out of bootstrap stats:** `src/agents/pi-embedded-runner/run/attempt.ts:411`
  - body: `contextFiles` now includes scoped working-memory files, but `buildBootstrapInjectionStats` still falls back by basename. A scoped file like `.openclaw/working-memory/cron/notes.md` can make an unrelated bootstrap `notes.md` look injected or truncated, so pass only bootstrap context files into bootstrap stats.
  - confidence: 0.92
- **[P2] Preserve workingMemoryPath in Control UI edits:** `src/cron/types.ts:102`
  - body: This adds `workingMemoryPath` as persisted agentTurn state, but the Control UI form does not read or rebuild that field and sends replacement payloads on edit. Editing a scoped-memory cron job in the UI would silently clear the path unless the UI preserves or exposes it.
  - confidence: 0.88
