Land the feature only after scoped files are accounted outside bootstrap stats, all cron/UI/config surfaces preserve the new field, real cron or heartbeat proof is added, and maintainers accept the core scoped-memory design.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep scoped memory out of bootstrap stats:** `src/agents/pi-embedded-runner/run/attempt.ts:409-412`
  - body: The PR appends the scoped working-memory file to `contextFiles` and then passes that combined array into `buildBootstrapInjectionStats`. That helper falls back by basename, so `.openclaw/working-memory/.../notes.md` can make an unrelated bootstrap `notes.md` look injected or truncated from scoped content; pass only bootstrap context files to bootstrap stats and keep scoped memory in its own lane.
  - confidence: 0.92
- **[P2] Preserve working memory in Control UI edits:** `src/cron/types.ts:102`
  - body: Adding `workingMemoryPath` as a persisted cron payload field without updating the Control UI means the UI still builds replacement `agentTurn` payloads that have no working-memory property. Editing a scoped-memory cron job in the web UI will silently clear the path, so wire or preserve the field there before exposing it.
  - confidence: 0.86
