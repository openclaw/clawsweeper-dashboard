Complete the shared `channels.modelByChannel` contract for direct conversations and expose/document that existing shared key instead of adding provider-specific nested `model` fields.

## Solution Assessment

Yes. Completing the shared `channels.modelByChannel` resolver/schema/docs path is the narrow maintainable solution and avoids provider-specific nested model fields with competing precedence rules.
