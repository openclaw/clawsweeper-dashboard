## Repair Work Prompt

Implement issue #53638 by completing the existing generic `channels.modelByChannel` contract. Add direct-message model override resolution using stable candidate keys from finalized context/session origin data, direct session keys when available, and provider-native identifiers such as `OriginatingTo`, `To`, `From`, `NativeDirectUserId`, and `NativeChannelId` where appropriate. Preserve precedence: explicit session/runtime model overrides beat config defaults; per-target entries beat wildcard/channel-level fallbacks. Do not add provider-specific nested `channels.whatsapp.*.model`, `channels.slack.*.model`, or Discord-only shapes. Expose `channels.modelByChannel` as an actual public config schema property, keep UI hints aligned, document accepted DM key forms in `docs/gateway/config-channels.md`, and add a user-facing changelog entry. Add focused tests for generic resolver DM candidates, get-reply wiring, status display if affected, public schema export, and Slack/WhatsApp/Discord direct-message behavior as needed.
