## Summary

Keep open. Current main and the latest release have the shared `channels.modelByChannel` schema plus channel/group matching, but the central per-DM request still remains because direct peer ids are not available to the shared resolver.

## What This Changes

Request persistent config-level model overrides for specific channels, groups, and DMs instead of ephemeral `/model` session overrides.

## Best Possible Solution

Complete the existing generic `channels.modelByChannel` resolver for direct conversations, document accepted DM key forms, and avoid adding parallel provider-specific `model` fields.

## Reproduction Assessment

Yes. Source inspection shows a direct-key override cannot match today because resolver inputs omit direct peer ids, parent-session parsing excludes `direct`, and the reply path does not pass the current direct session key.

## Solution Assessment

No. The requested provider-specific nested `model` fields are no longer the best shape; finishing the existing shared `channels.modelByChannel` direct-message path is narrower and more maintainable.
