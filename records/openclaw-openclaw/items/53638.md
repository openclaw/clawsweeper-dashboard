## Summary

Keep open. Current main and the latest release have the shared `channels.modelByChannel` surface for channel/group overrides and wildcard direct-chat defaults, but they still do not support stable per-DM target keys for ordinary direct conversations.

## What This Changes

Request persistent per-channel, per-group, and per-DM model overrides in config so conversations can use different models without runtime session overrides.

## Best Possible Solution

Complete the existing generic `channels.modelByChannel` resolver for direct conversations, document accepted DM key forms, and avoid adding provider-specific nested `model` fields.

## Reproduction Assessment

Yes. Source inspection shows ordinary DM-specific keys cannot be matched because resolver inputs omit direct peer identifiers, the generic session parser excludes `direct`, and current tests only prove wildcard direct fallback.

## Solution Assessment

Yes for the generic direction: finishing `channels.modelByChannel` for direct conversations is narrower than adding provider-specific nested `model` fields and matches the existing config surface.
