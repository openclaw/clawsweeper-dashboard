## Summary

Keep open: current main now covers the shared `channels.modelByChannel` config, channel/group routing, wildcard direct defaults, and public schema/help exposure, but it still does not support a specific per-DM model pin because direct-peer identifiers are not passed into or matched by the resolver.

## What This Changes

Request persistent config-level model overrides for channels, groups, and DMs without relying on ephemeral runtime `/model` overrides.

## Best Possible Solution

Finish the generic `channels.modelByChannel` contract for direct conversations by deriving stable DM candidates from finalized context, persisted origin/session data, and native direct ids, then document and test those key forms.

## Reproduction Assessment

Yes for the remaining gap. Source inspection shows ordinary direct chats do not provide direct-peer candidates to `resolveChannelModelOverride`, so a specific per-DM entry cannot match on current main or the latest release.

## Solution Assessment

Yes. Completing the existing generic `channels.modelByChannel` resolver is the narrowest maintainable solution; adding provider-specific nested `model` fields would duplicate the current config contract.
