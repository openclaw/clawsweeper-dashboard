## Summary

Keep open. Current main and the latest release have the shared `channels.modelByChannel` config, schema, docs, and group/channel resolver path, but the requested per-DM target behavior is still incomplete because direct session keys are not parsed as model override conversation refs and the normal direct-message call sites do not pass a direct peer id into the resolver.

## What This Changes

The issue requests persistent channel, group, and DM-scoped model overrides in `openclaw.json`.

## Best Possible Solution

Keep the generic `channels.modelByChannel` surface and extend its resolver/callers/docs so direct conversation IDs match target-specific entries while preserving session `/model` precedence and wildcard fallback behavior.

## Reproduction Assessment

Yes, by source inspection: `channels.modelByChannel.telegram.U1` cannot be reached from a normal direct session key because direct session refs are not parsed and the reply path does not pass the current direct `SessionKey` to the resolver. I did not run a test because this review is read-only.

## Solution Assessment

Yes, finishing the existing `channels.modelByChannel` resolver is the best shape. Adding nested per-plugin `model` fields would duplicate the current config surface and create drift.
