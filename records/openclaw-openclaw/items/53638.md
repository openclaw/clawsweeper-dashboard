## Summary

Keep open. Current main and v2026.5.22 provide the shared `channels.modelByChannel` config, schema, docs, and group/channel/wildcard matching, but specific direct-message matching remains unfinished because the resolver still receives only group and parent-session style candidates; the open ingress PR covers a separate channel-ingress bug and does not close this feature gap.

## What This Changes

Request persistent model overrides for configured channel, group, and DM conversations.

## Best Possible Solution

Finish the existing `channels.modelByChannel` contract for direct conversations by deriving normalized DM candidates from context, persisted origin/session data, native provider ids, and direct session keys, then document the accepted key forms without adding competing provider-specific `model` fields.

## Reproduction Assessment

Yes, at source level. Configure a specific DM entry under `channels.modelByChannel` and route an ordinary direct conversation; current source has no direct-peer candidate inputs and direct session keys are not used as model override candidates.

## Solution Assessment

Yes. Extending the existing shared `channels.modelByChannel` resolver and docs is the narrow maintainable direction, and it avoids competing provider-specific `model` fields.
