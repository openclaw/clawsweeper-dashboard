## Summary

Keep open. Current main covers channel/group routing through the shared `channels.modelByChannel` surface, but ordinary direct-message candidates and public schema exposure are still incomplete.

## What This Changes

Request persistent config-level model overrides for channel, group, and DM conversations, with target-specific overrides taking precedence over global defaults.

## Best Possible Solution

Complete the existing generic `channels.modelByChannel` contract with normalized direct-conversation candidates, public schema/docs support, and unchanged session/runtime override precedence.

## Reproduction Assessment

Yes. Source inspection on current main shows ordinary DMs can reach the resolver without group IDs or parent direct candidates, and the schema command path still hides `modelByChannel` from channel properties.

## Solution Assessment

Yes. Completing `channels.modelByChannel` is the narrowest maintainable direction because it reuses the existing shared config surface instead of adding provider-specific `channels.<provider>.*.model` fields.
