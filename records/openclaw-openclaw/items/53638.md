## Summary

Keep open. Current main and v2026.6.6 include the generic `channels.modelByChannel` config surface for channel/group overrides, but direct-message target matching is still incomplete because direct session refs are not parsed as model override conversation refs and the normal reply path does not pass a direct peer id into the resolver.

## What This Changes

The issue requests persistent per-channel, per-group, and per-DM model selection through `openclaw.json` instead of ephemeral `/model` runtime overrides.

## Best Possible Solution

Finish the existing generic `channels.modelByChannel` resolver and docs so direct conversation IDs match target-specific entries while preserving session `/model` precedence and wildcard fallback behavior.

## Reproduction Assessment

Yes. By source inspection, `channels.modelByChannel.telegram.U1` cannot be reached from a normal direct session key because direct refs are rejected and the reply path does not pass the current direct peer/session id into the resolver.

## Solution Assessment

Yes. Completing `channels.modelByChannel` is the best path; adding nested provider-specific `model` fields would duplicate the existing shared config surface and create drift.
