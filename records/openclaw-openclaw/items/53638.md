## Summary

Keep open. Current main and the latest release have the shared `channels.modelByChannel` schema, docs, channel/group matching, and direct-chat wildcard fallback, but target-specific DM keys still cannot match because the resolver does not receive or parse ordinary direct peer ids.

## What This Changes

The issue requests persistent per-channel, per-group, and per-DM model selection in `openclaw.json` instead of ephemeral runtime `/model` overrides.

## Best Possible Solution

Complete the generic `channels.modelByChannel` resolver, call sites, docs, and tests for direct conversations without adding parallel per-plugin `model` fields.

## Reproduction Assessment

Yes. Source inspection shows a direct key cannot match today because resolver inputs omit direct peer ids, `parseRawSessionConversationRef` excludes `direct`, and the reply path does not pass the current direct session key.

## Solution Assessment

Partly. The requested nested per-provider `model` fields are not the best shape; finishing the existing generic `channels.modelByChannel` surface is the narrowest maintainable solution.
