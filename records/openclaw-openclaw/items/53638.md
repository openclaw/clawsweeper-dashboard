## Summary

Keep open. Current main has the shared `channels.modelByChannel` surface for channel/group routing, including the Discord channel use case raised in comments, but ordinary DM matching and public config-schema exposure for the shared key are still incomplete.

## What This Changes

Request persistent config-level model selection for individual channels, groups, and DMs instead of ephemeral session `/model` overrides.

## Best Possible Solution

Finish the generic `channels.modelByChannel` contract with stable direct-conversation candidates and public schema/docs coverage while preserving session override precedence and avoiding provider-specific nested `model` fields.

## Reproduction Assessment

Yes. Source inspection on current main shows ordinary DMs can reach the resolver without group IDs or parent-session candidates, and the resolver returns null before matching when no candidate key exists.

## Solution Assessment

Yes. Completing `channels.modelByChannel` is the narrowest maintainable direction because the shared config surface already exists and avoids adding separate WhatsApp/Slack/Discord `model` fields.
