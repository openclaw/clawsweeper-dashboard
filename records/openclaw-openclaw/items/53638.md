Complete the existing generic `channels.modelByChannel` resolver for direct conversations, document accepted DM key forms, and avoid adding provider-specific nested `model` fields.

Yes. Source inspection shows a direct-key override cannot match ordinary DMs because resolver inputs omit direct peer ids, parent-session parsing excludes `direct`, and current tests only cover a wildcard direct fallback.

## Solution Assessment

No. The requested provider-specific nested `model` fields are no longer the best shape; finishing the existing shared `channels.modelByChannel` direct-message path is narrower and more maintainable.
