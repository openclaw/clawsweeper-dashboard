## Summary

Keep open; this PR adds a real new chat-edit path, but it is not merge-ready because the new side-effecting RPC requires an idempotency key and then ignores it, so retry behavior can fail after the first edit rewinds the active branch.

## What This Changes

Adds a Gateway `chat.edit_user_message` RPC with protocol/Swift model wiring, a gateway regression test, and a Control UI controller method to edit a prior user message and rewind later active-branch turns.

## Best Possible Solution

Keep the feature direction on the PR, but make the edit RPC idempotent, document or release-note the new user-facing behavior, and get maintainer approval for the protocol semantics before merge.

## Reproduction Assessment

Yes for the PR defect from source inspection: the schema requires `idempotencyKey`, while the handler drops it before mutating the transcript. I did not execute tests because this was a read-only review.

## Solution Assessment

No. Branching with `SessionManager` is a plausible implementation direction, but the RPC needs stable duplicate-key handling before it satisfies the Gateway side-effect contract.
