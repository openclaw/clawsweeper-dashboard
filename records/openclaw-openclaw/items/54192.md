## Summary

Keep open. Current main still lets fallback retries reuse an existing CLI backend session while the CLI runner sends prompt text in resume mode; the later fresh-session reseed fixes do not cover this duplicate resumed-turn repair.

## What This Changes

Request to add a CLI runner repair so fallback retries against resumed CLI sessions do not duplicate user turns.

## Best Possible Solution

Add a CLI-owned fallback retry repair that either resets/reseeds the backend session or transcript-confirms the current prompt before resubmitting, while preserving existing session_expired, missing-transcript, and auth-boundary behavior.

## Reproduction Assessment

## Solution Assessment

Yes, the narrow maintainable fix belongs in the CLI runner/fallback attempt path. Avoid a simple prompt skip unless backend transcript evidence confirms the prompt was already accepted.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/54192. Current main can reuse a CLI backend session on a fallback retry after timeout/rate_limit while still sending the retry prompt via stdin/argv, causing duplicate user turns in the provider-owned CLI session. Add a CLI-owned repair in the runAgentAttempt/runCliAgent/executePreparedCliRun path that prevents resubmitting an already-accepted current prompt to a reused backend CLI session. Preserve existing behavior for session_expired clearing, missing Claude transcript reset, auth-profile/auth-epoch invalidation, raw transcript reseed safety, and normal successful resume. Do not implement a blind prompt skip that can lose the task if the first attempt failed before the backend persisted the prompt; either reset/reseed the CLI session for unsafe fallback retries or confirm the backend transcript already contains the current prompt before suppressing resubmission. Add focused regression coverage for timeout/rate_limit fallback retry with an existing cliSessionId and no duplicate prompt submission. Changelog is expected if the behavior change is user-visible; avoid broad fallback/image/partial-execution refactors from https://github.com/openclaw/openclaw/pull/57959 unless strictly necessary.
