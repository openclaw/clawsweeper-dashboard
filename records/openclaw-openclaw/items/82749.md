Keep open: this is a credible current-main bug, it has the protected `maintainer` label, and the linked implementation PR is still open. ## What This Changes

The issue requests normalizing or dropping overlong historical Responses tool IDs when replaying sessions through GitHub Copilot.

## Best Possible Solution

Land a reviewed fix that normalizes GitHub Copilot Responses replay item IDs and paired tool-result IDs at the request-builder/replay seam, with focused regression coverage for the reported long-ID shape.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to build a `github-copilot` `openai-responses` payload from a same-model replayed assistant `toolCall` shaped as `call_*|<overlong fc_...>` plus its paired `toolResult`; current main can preserve the overlong item ID into outbound `input[].id`.

## Solution Assessment

Yes. The maintainable fix belongs in Responses replay normalization/request building so Copilot receives provider-valid IDs while preserving tool-result pairing.
