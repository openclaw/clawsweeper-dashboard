## Summary

Keep open: current main still has the source-level heartbeat behavior this PR targets, but the branch is not merge-ready because the fallback is incomplete, the bundled MiniMax changes are stale against current provider hooks, current docs need contract alignment, and the external PR lacks real behavior proof.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping tests and helper changes, plus older transcript-policy context plumbing and MiniMax provider capability fallbacks.

## Best Possible Solution

Revise or replace this with a focused current-main patch that strips all heartbeat-mode tokens from normalized text before the ack-length gate, preserves message-mode behavior, updates the heartbeat docs/tests, and leaves MiniMax replay policy in provider hooks.

## Reproduction Assessment

Yes, source-level. Current main returns the original text for a middle-only `HEARTBEAT_OK`, and heartbeat delivery calls that helper in heartbeat mode before suppressing or sending final text.

## Solution Assessment

No. The PR targets the right helper, but the implementation misses edge-plus-middle and markup-wrapped cases, carries stale MiniMax core fallback changes, and needs heartbeat contract/docs alignment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. If a heartbeat reply has one token at an edge and another in the middle, edge stripping sets `didStrip`, skips this branch, and the remaining `HEARTBEAT_OK` can still be delivered when the surrounding content exceeds `maxAckChars`.
  - confidence: 0.88
- **[P2] Strip middle tokens from normalized text:** `src/auto-reply/heartbeat.ts:163-167`
  - body: This branch splits raw `trimmed`, so wrapped middle tokens such as `<b>HEARTBEAT_OK</b>` or `**HEARTBEAT_OK**` leave wrapper artifacts in delivered text and in the length gate. Use the normalized markup-stripped text before heartbeat-mode removal.
  - confidence: 0.86
- **[P2] Drop the stale MiniMax core fallback:** `src/agents/provider-capabilities.ts:53-58`
  - body: Current main resolves MiniMax replay behavior through provider runtime hooks in the MiniMax plugin, and the old `provider-capabilities` file is gone. Adding MiniMax ids back into a core fallback reintroduces provider-owned policy in the wrong layer and should be removed during rebase.
  - confidence: 0.84
