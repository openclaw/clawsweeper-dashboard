## Summary

Keep open: the underlying heartbeat leak is still source-reproducible on current main, but this branch is not merge-ready because it lacks real behavior proof, leaves correctness gaps in the token stripping path, and carries older MiniMax/provider-capability edits that no longer match the current provider-hook architecture.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` removal and tests, plus older transcript-policy context plumbing and MiniMax drop-thinking capability fallbacks.

## Best Possible Solution

Implement a focused current-main heartbeat helper change that strips all heartbeat-mode tokens from normalized text before the ack-length gate, preserves message-mode behavior, updates heartbeat docs/tests, and leaves MiniMax replay policy in provider hooks.

## Reproduction Assessment

Yes, source-level. Current main returns the original text for a middle-only `HEARTBEAT_OK`, and heartbeat delivery calls that helper in heartbeat mode before suppressing or sending final text.

## Solution Assessment

No. The PR targets the right helper, but it should remove all heartbeat-mode token occurrences from normalized text, drop the stale MiniMax core-capability changes, and align the heartbeat docs/contract before merge.
