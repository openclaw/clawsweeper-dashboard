## Summary

Keep open. The linked heartbeat-token leak is still source-reproducible on current main and https://github.com/openclaw/openclaw/issues/58098 was closed to this PR as the canonical tracker, but the branch should not merge as-is because its fallback misses edge-plus-middle tokens, strips raw markup text, carries stale MiniMax/provider-capability plumbing, and lacks real behavior proof.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping with tests, and also threads provider capability context plus MiniMax fallback changes through agent transcript-policy files.

## Best Possible Solution

Land a focused current-main heartbeat helper fix that normalizes markup, removes every `HEARTBEAT_OK` occurrence in heartbeat mode before the ack-length gate, preserves message-mode behavior, updates docs/changelog/tests, and leaves MiniMax replay policy to provider hooks.

## Reproduction Assessment

Yes, from source inspection rather than an executed test. Current main returns the original trimmed heartbeat text unchanged when the token is only in the middle, so `Some preamble HEARTBEAT_OK actual reply` follows the reported leak path.

## Solution Assessment

No. The PR targets the right helper, but the maintainable fix should remove all heartbeat-mode tokens from normalized text after edge handling and drop the stale provider-capability changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. If a heartbeat reply has a token at an edge and another in the middle, the edge pass sets `didStrip` and the PR returns `picked.text`, so a remaining `HEARTBEAT_OK` can still be delivered when surrounding content exceeds `ackMaxChars`.
  - confidence: 0.88
- **[P2] Strip mid-text tokens from normalized text:** `src/auto-reply/heartbeat.ts:162-166`
  - body: The fallback splits raw `trimmed`, so mid-text wrapped tokens like `<b>HEARTBEAT_OK</b>` or `**HEARTBEAT_OK**` can leave wrapper artifacts in delivered text and in the `ackMaxChars` gate. Use the normalized/markup-stripped path before heartbeat-mode removal.
  - confidence: 0.85
