## Summary

Keep open. The reported heartbeat leak is still source-reproducible on current main and the PR targets the right helper, but the branch still lacks real behavior proof, has heartbeat correctness gaps, conflicts with the current documented response contract, and bundles stale MiniMax provider-capability changes.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping and tests, plus older MiniMax/transcript-policy capability plumbing.

## Best Possible Solution

Land or replace this with a focused current-main patch that updates the heartbeat helper, docs, and regression tests while leaving MiniMax replay policy in provider hooks.

## Reproduction Assessment

Yes, source-level. Current main returns the original text for a middle-only `HEARTBEAT_OK`, and heartbeat delivery calls that helper in heartbeat mode before deciding whether to suppress or send final text.

## Solution Assessment

No. The PR targets the right helper, but the maintainable fix should strip all heartbeat-mode tokens from normalized text before the ack-length gate, preserve message-mode behavior, update the docs/contract, and drop the stale MiniMax core fallback changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove remaining heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. If a heartbeat reply has one token at an edge and another in the middle, edge stripping sets `didStrip`, this branch is skipped, and the remaining `HEARTBEAT_OK` can still be delivered when surrounding content exceeds `maxAckChars`.
  - confidence: 0.88
- **[P2] Strip mid-text tokens from normalized text:** `src/auto-reply/heartbeat.ts:163-167`
  - body: The fallback splits raw `trimmed`, so wrapped middle tokens such as `<b>HEARTBEAT_OK</b>` or `**HEARTBEAT_OK**` leave wrapper artifacts in delivered text and in the `maxAckChars` gate. Use the normalized markup-stripped path before heartbeat-mode removal.
  - confidence: 0.85
- **[P2] Drop stale MiniMax core fallback changes:** `src/agents/provider-capabilities.ts:53-58`
  - body: Current main resolves replay policy through provider runtime hooks, and MiniMax registers those hooks in its plugin. Reintroducing `minimax` ids in a core fallback map puts provider-owned policy back in core and should be removed or rebased onto the current plugin hook path.
  - confidence: 0.82
