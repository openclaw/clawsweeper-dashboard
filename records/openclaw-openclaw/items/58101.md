## Summary

Keep open: current main still has a source-reproducible mid-text heartbeat token leak and this PR is the canonical active tracker, but the proposed patch is not merge-ready because the fallback is incomplete, the branch carries stale MiniMax/provider-policy churn, and the external real-behavior proof gate is still missing.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping and tests, plus older MiniMax provider capability/context propagation changes across transcript policy paths.

## Best Possible Solution

Land or replace this with a focused current-main heartbeat helper fix that removes every `HEARTBEAT_OK` occurrence in heartbeat mode using normalized text before the ack-length gate, preserves message-mode behavior, updates docs/tests/changelog, and leaves provider replay policy to plugin hooks.

## Reproduction Assessment

Yes, from source inspection. Current main returns the original heartbeat text unchanged when `HEARTBEAT_OK` appears only in the middle, and heartbeat delivery calls that helper before deciding whether to skip or send.

## Solution Assessment

No. The PR targets the right helper, but it should remove all heartbeat-mode tokens from normalized text after edge handling and drop the stale provider-capability changes before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove all heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. If a heartbeat reply has one token at an edge and another in the middle, the edge pass sets `didStrip` and this branch returns `picked.text`, so a remaining `HEARTBEAT_OK` can still be delivered when the surrounding content exceeds `ackMaxChars`.
  - confidence: 0.88
- **[P2] Strip mid-text tokens from normalized content:** `src/auto-reply/heartbeat.ts:162-166`
  - body: The fallback splits raw `trimmed`, so wrapped mid-text tokens such as `<b>HEARTBEAT_OK</b>` or `**HEARTBEAT_OK**` can leave wrapper artifacts in delivered text and in the `ackMaxChars` gate. Use the normalized/markup-stripped path before heartbeat-mode removal.
  - confidence: 0.85
