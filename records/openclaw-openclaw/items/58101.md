## Summary

Keep this PR open: current main still has the reported heartbeat leak path, but the PR branch is not ready to merge because it misses after-fix real behavior proof and leaves two heartbeat-token correctness gaps.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping with tests and also changes MiniMax/provider transcript-policy plumbing.

## Best Possible Solution

Land a focused current-main heartbeat helper patch that strips every heartbeat-mode sentinel from normalized text, preserves message-mode behavior, and updates regression tests plus the heartbeat docs.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence current-main path: a heartbeat reply with `HEARTBEAT_OK` only in the middle reaches `stripHeartbeatToken`, neither edge pass strips it, and the helper returns the original text for delivery.

## Solution Assessment

No. The PR targets the right helper, but the maintainable fix should remove all heartbeat-mode tokens from normalized text after edge handling, add edge-plus-middle and wrapped-token coverage, update docs, and drop the stale provider-capability edits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove remaining heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156`
  - body: The fallback only runs when `picked.didStrip` is false. If heartbeat text has a token at an edge plus another token later with long surrounding content, the edge pass sets `didStrip` and returns `picked.text`, so the later `HEARTBEAT_OK` can still be sent.
  - confidence: 0.88
- **[P2] Strip mid-text tokens from normalized text:** `src/auto-reply/heartbeat.ts:162-166`
  - body: The fallback splits raw `trimmed`, so wrapped mid-text tokens like `<b>HEARTBEAT_OK</b>` leave tag or wrapper remnants in delivered text and in the `ackMaxChars` calculation. Use the markup-normalized text before heartbeat-mode removal.
  - confidence: 0.85
