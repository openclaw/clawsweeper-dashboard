## Summary

Keep open. Current main still implements and documents edge-only heartbeat ack stripping, so the user problem is not implemented; this PR targets a valid bug but its patch is incomplete and also carries stale provider-capability changes now handled by plugin replay hooks.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping with tests, plus transcript-policy context plumbing and MiniMax thinking-block fallback changes.

## Best Possible Solution

Land a focused current-main heartbeat fix that normalizes markup, removes every `HEARTBEAT_OK` occurrence in heartbeat mode before the ack-length gate, preserves message-mode behavior, and updates tests, docs, and changelog.

## Reproduction Assessment

Yes. On current main, a heartbeat reply such as `Some preamble HEARTBEAT_OK actual reply` reaches the `!picked.didStrip` branch and returns the original token-containing text because only edge stripping is attempted.

## Solution Assessment

No. The PR addresses the valid root cause, but the maintainable fix is a normalized remove-all heartbeat path on current main, without reviving older core provider-capability plumbing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove remaining heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. A heartbeat reply with an edge token plus another token in the middle can strip the edge token, skip the fallback, and still deliver a remaining `HEARTBEAT_OK` when the rest is long enough, so the heartbeat-mode remove-all pass needs to run after edge normalization.
  - confidence: 0.88
- **[P2] Normalize markup before fallback stripping:** `src/auto-reply/heartbeat.ts:162-166`
  - body: The new fallback splits raw `trimmed` text, so mid-text wrapped tokens like `<b>HEARTBEAT_OK</b>` can leave wrapper fragments in the delivered reply and in the `maxAckChars` gate. Use the normalized text path before heartbeat fallback removal.
  - confidence: 0.85
