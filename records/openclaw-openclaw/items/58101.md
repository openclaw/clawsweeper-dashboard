## Summary

Keep open: the heartbeat leak is still source-reproducible on current main and this PR is the active fix path, but the branch is not merge-ready because it lacks real behavior proof and the patch is incomplete against the current heartbeat/provider architecture.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping and tests, plus older MiniMax transcript-policy/provider-capability context changes.

## Best Possible Solution

Rework this into a focused current-main heartbeat helper change that strips all heartbeat-mode tokens from normalized text before the ack-length gate, preserves message-mode behavior, updates docs/tests, and leaves MiniMax replay policy in provider hooks.

## Reproduction Assessment

Yes, from source inspection rather than an executed test: current main returns the original text when `HEARTBEAT_OK` is only mid-string, and heartbeat chat finalization calls that helper in heartbeat mode.

## Solution Assessment

No. The PR targets the right helper, but it should remove all heartbeat-mode tokens from normalized text after edge handling, drop the stale MiniMax/provider-capability changes, and update the heartbeat contract docs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove remaining heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. If a heartbeat reply has one token at an edge and another in the middle, edge stripping sets `didStrip`, this branch is skipped, and a remaining `HEARTBEAT_OK` can still be delivered when the surrounding content exceeds `ackMaxChars`.
  - confidence: 0.88
- **[P2] Strip mid-text tokens from normalized text:** `src/auto-reply/heartbeat.ts:163-167`
  - body: The fallback splits raw `trimmed`, so wrapped middle tokens such as `<b>HEARTBEAT_OK</b>` or `**HEARTBEAT_OK**` can leave wrapper artifacts in delivered text and in the `ackMaxChars` gate. Use the normalized markup-stripped path before heartbeat-mode removal.
  - confidence: 0.85
- **[P2] Drop the stale MiniMax capability fallback:** `src/agents/provider-capabilities.ts:54-59`
  - body: Current main no longer owns replay behavior through this core capability table; provider replay policy is resolved through runtime plugin hooks, and MiniMax already registers those hooks. Rebase onto current main and keep the provider behavior in `extensions/minimax/provider-registration.ts` instead of carrying this unrelated core fallback.
  - confidence: 0.82
