## Summary

Keep open: the heartbeat leak remains on current main, but this PR is not merge-ready because its fallback misses edge-plus-middle tokens, operates on raw markup, and carries older MiniMax/provider-capability plumbing that current main has since superseded.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping with tests, plus transcript-policy context threading and MiniMax thinking-block fallback changes in older provider-capability code.

## Best Possible Solution

Land a focused current-main heartbeat sanitizer that normalizes markup, removes every `HEARTBEAT_OK` occurrence only in heartbeat mode before the ack-length gate, preserves message-mode behavior, and updates tests, docs, and changelog.

## Reproduction Assessment

Yes. Current main source shows a heartbeat reply such as `Some preamble HEARTBEAT_OK actual reply` reaches the `!picked.didStrip` return path and keeps the token; current docs and tests align with that edge-only behavior.

## Solution Assessment

No. The PR targets the right root cause, but the maintainable fix should remove all heartbeat-mode tokens from normalized text and leave the now-plugin-owned MiniMax replay policy out of this change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove remaining heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. A heartbeat reply with an edge token plus another token in the middle can strip the edge token, skip the fallback, and still deliver a remaining `HEARTBEAT_OK` when the rest is long enough. Run the heartbeat-mode remove-all pass after edge normalization and before the ack-length gate.
  - confidence: 0.88
- **[P2] Normalize markup before fallback stripping:** `src/auto-reply/heartbeat.ts:162-166`
  - body: The new fallback splits raw `trimmed` text, so wrapped mid-text tokens like `<b>HEARTBEAT_OK</b>` or `**HEARTBEAT_OK**` can leave wrapper fragments in the delivered reply and in the `maxAckChars` check. Use the normalized text path before heartbeat fallback removal.
  - confidence: 0.85
