## Summary

Keep open: the heartbeat leak is source-reproducible on current main, but this PR is not ready because the fallback still misses edge-plus-middle tokens, uses raw text instead of normalized text, carries stale MiniMax/provider-capability changes, and lacks real behavior proof.

## What This Changes

The PR adds heartbeat-mode mid-text `HEARTBEAT_OK` stripping and tests, plus older MiniMax provider capability and transcript-policy context plumbing.

## Best Possible Solution

Rework this into a focused current-main heartbeat helper change that strips all heartbeat-mode tokens from normalized text before the ack-length gate, preserves message-mode behavior, updates docs/tests, and leaves MiniMax replay policy in provider hooks.

## Reproduction Assessment

Yes, from source inspection. Current main returns the original text when `HEARTBEAT_OK` appears only in the middle, and heartbeat chat delivery calls that helper before suppressing or sending text.

## Solution Assessment

No. The PR targets the right helper, but it should remove all heartbeat-mode tokens from normalized text after edge handling, drop the stale provider-capability changes, and update the heartbeat contract docs before merge.

## Review Findings

  - confidence: 0.85
