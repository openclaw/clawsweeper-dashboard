## Summary

Keep open. The heartbeat bug is still present on current main, but this PR should not merge as-is because its heartbeat fallback misses edge+middle token cases, removes mid-text tokens from raw markup-bearing text, and carries older MiniMax/provider-capability plumbing that current main has since moved to provider runtime replay hooks.

## What This Changes

The PR adds heartbeat-mode fallback stripping for mid-text `HEARTBEAT_OK` tokens and tests, plus transcript-policy context threading and MiniMax provider-capability fallbacks.

## Best Possible Solution

Land a narrow current-main heartbeat fix that removes all `HEARTBEAT_OK` occurrences in heartbeat mode after markup normalization and before the ack length gate, preserves message-mode mid-token behavior, updates the heartbeat response-contract docs and changelog, and leaves MiniMax replay policy on the existing plugin-owned hook path.

## Reproduction Assessment

Yes. A code-level reproduction is `stripHeartbeatToken("Some preamble HEARTBEAT_OK actual reply", { mode: "heartbeat" })` on current main; because no edge strip occurs, the function returns the token-containing text instead of stripping or suppressing it.

## Solution Assessment

No. The PR's heartbeat approach is incomplete because it only runs when edge stripping found nothing and it operates on raw markup-bearing text; the safer current fix is a normalized heartbeat remove-all path with focused regression coverage.
