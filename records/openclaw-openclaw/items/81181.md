## Summary

Keep this open: current main still resolves Telegram media and sends the media-download failure reply before the existing requireMention/no-mention gate can skip the group message.

## What This Changes

The issue asks Telegram group media messages to be silently skipped when requireMention is true and the bot was not mentioned, without attempting media download or sending a failure reply.

## Best Possible Solution

Make Telegram decide whether a requireMention group message would be skipped before sending media-download errors, reusing the current mention-gating facts so mentioned, implicit, command, direct-message, and no-requireMention cases keep their existing behavior.

## Reproduction Assessment

Yes, source-reproducible: a group photo with requireMention true and no mention reaches resolveMedia first, and a media failure sends the warning before the later no-mention gate can return null. I did not execute a live Telegram repro in this read-only review.

## Solution Assessment

No, the current implementation is not the best solution because it performs media work and user-visible error delivery before mention authorization. The prior closed PR is a useful sketch, but the replacement fix should mirror the current centralized mention decision instead of a narrower hand-rolled check.

## Repair Work Prompt

Fix the Telegram requireMention media-error ordering bug reported in https://github.com/openclaw/openclaw/issues/81181. In groups where requireMention is true and the inbound media message is not addressed to the bot, OpenClaw must silently skip without attempting media download where feasible and must not send the "Failed to download media" warning if media resolution fails before the full context is built. Preserve current behavior for DMs, groups with requireMention false, explicit native @bot mentions, configured regex mention patterns, implicit reply-to-bot messages, and authorized command bypasses. Prefer reusing or factoring the existing Telegram mention-decision logic rather than adding a narrow @username-only check. The prior closed PR https://github.com/openclaw/openclaw/pull/45295 is useful context but targets older files and missed some gate parity concerns. Add regression coverage for an unmentioned requireMention group photo whose fetch fails and assert no error send and no reply dispatch; keep/directly assert the existing direct-message failure warning still fires, and add a mentioned group case if the harness can express it. Add a changelog fix entry if the final change is user-visible. Do not change core plugin boundaries or import private core internals from Telegram production code.
