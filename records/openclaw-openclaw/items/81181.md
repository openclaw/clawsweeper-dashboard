## Summary

Keep open: current main still resolves Telegram media before the no-mention gate and can send the failed-download reply first, while an open closing PR already tracks the fix but has not merged.

## What This Changes

The issue asks Telegram mention-only groups to skip unmentioned media messages before any media download attempt or user-visible media-download failure reply.

## Best Possible Solution

Review and land a narrow Telegram plugin fix, or a replacement for the open closing PR, that uses the canonical mention decision before non-audio media failure replies while preserving feedback for DMs, addressed group messages, reply-to-bot messages, and audio preflight.

## Reproduction Assessment

Yes by source inspection: current main resolves Telegram media and can send the download-failure reply before the downstream no-mention requireMention gate runs. I did not run a live Telegram/WSL scenario, so this is source_reproducible rather than live reproduced.

## Solution Assessment

Yes for the issue direction: the expected behavior matches the Telegram docs and current code shows the ordering bug. The open PR is the current implementation candidate, but it should be reviewed against the canonical mention/session decision before merge.
