## Summary

Keep open: current main still has the reported Telegram order-of-operations bug, and the open closing PR has not merged and already has blocking review feedback.

## What This Changes

The issue reports that Telegram mention-only groups can emit a visible failed-media-download reply for unmentioned media because media is resolved before the `requireMention` skip.

## Best Possible Solution

Land a narrow Telegram plugin fix that applies the canonical mention/session decision before non-audio media failure replies while preserving DMs, addressed group messages, replies-to-bot, and audio preflight.

## Reproduction Assessment

Yes, by source inspection: current main calls `resolveMedia()` and can send the failed-download reply before `resolveTelegramInboundBody` applies the no-mention skip. I did not run a live Telegram/WSL scenario, so this is source-reproducible rather than live reproduced.

## Solution Assessment

Yes for the issue direction, but not the linked PR as currently reviewed. The maintainable fix should reuse or precompute the canonical mention and session-activation decision instead of suppressing every media-fetch error in mention-only groups.
