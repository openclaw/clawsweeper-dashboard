## What This Changes

The issue reports that Telegram mention-only groups emit a visible media-download error before the no-mention skip when an unmentioned photo fails to download.

## Best Possible Solution

Land a corrected Telegram plugin fix that uses the canonical requireMention and mention decision before non-audio media failure replies, while keeping DM, explicitly mentioned group messages, reply-to-bot messages, and audio preflight behavior intact.

## Reproduction Assessment

Yes. Current main calls `resolveMedia` and can send the download-failure reply before the later `requireMention` no-mention gate in `resolveTelegramInboundBody`; I did not run a live Telegram/WSL scenario.

## Solution Assessment

No for the currently linked PR as reviewed: it targets the right bug, but prior review says it suppresses too broadly instead of using the canonical mention decision. The maintainable fix should skip only unmentioned mention-only group media failures.
