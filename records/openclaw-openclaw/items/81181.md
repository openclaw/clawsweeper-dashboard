## Summary

Keep open: current main still has the reported Telegram order-of-operations bug, where media resolution can send the download-failure reply before the requireMention no-mention skip runs, and I found no merged fix or exact duplicate owning this behavior.

## What This Changes

The issue requests that Telegram groups with requireMention enabled skip unmentioned media messages before attempting media download or sending a download-failure reply.

## Best Possible Solution

Move the Telegram requireMention short-circuit ahead of non-audio media download when the mention decision is knowable from message metadata, while preserving DM, mentioned-message, reply-to-bot, and audio-preflight behavior.

## Reproduction Assessment

Yes by source inspection: current main calls resolveMedia and sends the failure reply before the later no-mention gate in resolveTelegramInboundBody. I did not run a live Telegram/WSL scenario, so this is source_reproducible rather than a live reproduced status.

## Solution Assessment

Yes, the narrow maintainable fix is to gate unmentioned Telegram group media before non-audio download attempts, not to silence all media errors globally. The fix should keep the existing user-visible error for DMs and mentioned messages.
