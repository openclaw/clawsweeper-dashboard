## Summary

Keep open: current main still lacks per-turn A2A visibility, and the PR targets a real plugin API gap, but the branch is not merge-ready because it exposes raw A2A replies without the conversation-access gate, targets the old hook contract surface, lacks public docs, and only has build-level proof.

## What This Changes

The branch adds an `agent_to_agent_turn` plugin hook, runner wiring, and A2A turn emission from `runSessionsSendA2AFlow` with flow IDs, turn metadata, delivery target, and bounded serialized dispatch.

## Best Possible Solution

Ship a rebased, documented, conversation-access-gated A2A turn hook in the current `hook-types.ts` architecture, with focused wiring tests and redacted proof from a real A2A run using a plugin handler.

## Reproduction Assessment

Not applicable as a feature/API PR. Source inspection confirms current main has no `agent_to_agent_turn` hook and only delivers the final A2A announce through the channel send path.

## Solution Assessment

No. A semantic A2A turn hook looks like a plausible seam, but this branch needs rebase, conversation-access gating, queue/announce ordering cleanup, docs, tests, real proof, and maintainer approval for the public contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1866-1872`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` text and delivery metadata, but the branch registers it as a normal hook. Current registry policy only gates names returned by `isConversationHookName`, so a non-bundled plugin could subscribe without `allowConversationAccess`; add this hook to the gated conversation set when rebasing.
  - confidence: 0.88
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1385-1413`
  - body: Current main imports hook names and handler maps from `src/plugins/hook-types.ts`, while this branch adds `agent_to_agent_turn` to the old `src/plugins/types.ts` contract block. After rebase, register the name, event/context types, handler map, and runner exports in the current hook contract surface so `api.on(...)` can type and load the hook.
  - confidence: 0.92
- **[P2] Wait for the full hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:164-165`
  - body: The final wait races the entire serialized `hookChain` against a single `perTurnTimeout`. In a multi-turn exchange where each handler is slow but under its per-turn budget, the final announce can still go out before later intermediate turn hooks finish, breaking the chronology this hook is meant to expose.
  - confidence: 0.84
- **[P2] Document the public A2A hook contract:** `src/plugins/types.ts:1846-1880`
  - body: This adds a public plugin hook with payload, delivery target, ordering, and timeout semantics, but the hook catalog is not updated. Plugin authors need the supported fields and best-effort delivery behavior documented before this API ships.
  - confidence: 0.83
