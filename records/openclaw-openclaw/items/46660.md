## Summary

Keep open. Current main still lacks an A2A turn hook and the linked DingTalk/channel-plugin use case is plausible, but this PR is not merge-ready: it needs real behavior proof, a rebase onto the current hook contract split, and a fix for the queued-hook wait that can still let the final announce outrun intermediate turn forwarding.

## What This Changes

The PR adds an `agent_to_agent_turn` plugin hook and emits A2A turn metadata/replies from the ping-pong flow before the final announce.

## Best Possible Solution

Rebase onto current main, implement the hook on the active hook contract, preserve transcript ordering semantics, add focused tests/docs, and include redacted real A2A/plugin proof.

## Reproduction Assessment

Yes for the PR defect: by source inspection, the serialized hook queue is only awaited for one per-turn timeout before announce, so cumulative under-timeout handlers can still reorder the visible transcript. The feature request itself is additive API work rather than a broken current-main behavior.

## Solution Assessment

No, not yet. The hook direction is plausible, but the patch should be rebased to current hook types and should either make chronology guarantees match the implementation or change the implementation so the final announce cannot overtake queued turn events in normal slow-handler cases.
