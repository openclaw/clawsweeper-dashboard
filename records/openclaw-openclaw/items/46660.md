## Summary

Keep open. Current main still does not implement this hook, and the PR targets a real A2A visibility gap, but it needs maintainer approval for the new public plugin API plus a rebase to the current hook-contract layout before it is merge-ready.

## What This Changes

The PR adds a public `agent_to_agent_turn` plugin hook and emits it from A2A ping-pong turns with delivery metadata so channel plugins can forward intermediate messages.

## Best Possible Solution

Land a current-main-compatible, maintainer-approved hook contract with explicit privacy semantics, tests for ordered A2A event emission, public plugin docs, and a changelog entry.

## Reproduction Assessment

Not applicable; this is a feature PR, not a bug report. Source inspection confirms current main has only the final A2A announce delivery and no per-turn plugin hook.

## Solution Assessment

Unclear. The hook direction addresses the reported semantic gap, but the current patch is stale against the hook-contract split and should not merge without docs, changelog, tests, and maintainer approval of the new plugin data surface.
