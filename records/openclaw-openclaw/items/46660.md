## Summary

Keep this PR open. Current main still has the A2A visibility gap, but this branch should not merge as-is because it is stale against the current hook-contract split and introduces a raw A2A transcript hook without the existing conversation-access policy.

## What This Changes

The PR adds an `agent_to_agent_turn` typed plugin hook, a hook-runner method, and queued emissions from the `sessions_send` A2A ping-pong flow carrying turn metadata, reply text, flow identity, and delivery-target data.

## Best Possible Solution

Ship a first-class A2A turn visibility hook after rebasing onto current main, defining the public contract in `src/plugins/hook-types.ts`, adding it to the conversation-content access policy, documenting the operator opt-in, and covering loader gating plus A2A turn emission with focused tests.

## Reproduction Assessment

Yes. A static current-main check is enough: intermediate A2A ping-pong turns run through `runAgentStep`, only the final announce calls gateway `send`, and no existing hook emits those intermediate replies.

## Solution Assessment

No. The feature direction is plausible and narrow, but this branch is not the best merge shape because it updates a stale hook catalog location and does not apply the current raw-conversation-access policy.
