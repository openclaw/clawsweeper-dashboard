## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1869`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` content plus session and delivery metadata, but the PR registers it as an ordinary hook. On current main, non-bundled plugin access is restricted only for names in `CONVERSATION_HOOK_NAMES`, so this new hook needs to be added to that gated set when rebased.
  - confidence: 0.9
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1388`
  - body: Current main imports hook names and handler maps from `src/plugins/hook-types.ts`, while `src/plugins/types.ts` now only re-exports that surface. Adding the new contract to the old `types.ts` block will be lost or conflict on rebase, so register the hook name, event/context types, handler map entry, and exports in `hook-types.ts`.
  - confidence: 0.9
- **[P2] Emit turn zero when ping-pong is disabled:** `src/agents/tools/sessions-send-tool.a2a.ts:62-66`
  - body: `resolvePingPongTurns()` allows `0`, but the new hook setup lives inside the `maxPingPongTurns > 0` guard. In that configuration the target still produces the initial reply and the API documents it as `turn: 0`, yet plugins receive no `agent_to_agent_turn` event.
  - confidence: 0.86
- **[P2] Wait for the full hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:167`
  - body: The final wait races the entire serialized `hookChain` against one `perTurnTimeout`. With multiple slow-but-under-timeout handlers, the announce step can run while later turn hooks are still pending, so users can receive the final conclusion before intermediate turns despite the chronology guarantee.
  - confidence: 0.8
