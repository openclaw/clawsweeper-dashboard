## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1385`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` text and delivery metadata, but the branch registers it as an ordinary hook. Since registry policy only gates names returned by `isConversationHookName`, add this hook to the conversation hook set when rebasing.
  - confidence: 0.9
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1385`
  - body: Current main imports hook names and handler maps from `src/plugins/hook-types.ts`, while this branch adds the new contract to the older `types.ts` block. Register the hook name, event/context types, handler map, and exports on the current hook contract surface.
  - confidence: 0.92
- **[P2] Emit turn zero when ping-pong is disabled:** `src/agents/tools/sessions-send-tool.a2a.ts:67-70`
  - body: `resolvePingPongTurns()` allows `maxPingPongTurns` to be `0`, but the new hook setup lives inside the `maxPingPongTurns > 0` guard. In that configuration the target agent still produces the initial reply, yet plugins never receive the documented `turn: 0` event.
  - confidence: 0.86
- **[P2] Wait for the full hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:163-165`
  - body: The final wait races the entire serialized hook queue against one `perTurnTimeout`. In a multi-turn exchange where each handler is slow but under its per-turn budget, the final announce can still go out before later intermediate hooks finish, breaking transcript chronology.
  - confidence: 0.82
