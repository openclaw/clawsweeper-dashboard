## Summary

Keep open. The proposed core API fills a plausible A2A visibility gap, but the branch is not mergeable as-is: it is stale against the current hook contract split, needs the conversation-access security gate, has A2A ordering/zero-turn correctness issues, and still lacks real runtime proof.

## What This Changes

The branch adds a typed `agent_to_agent_turn` plugin hook and emits A2A turn events from the `sessions_send` ping-pong flow.

## Best Possible Solution

Land a rebased core hook only if maintainers accept the API: define it in `hook-types.ts`, classify it as a conversation hook, document best-effort ordering/timeout semantics, add focused A2A and hook tests, and verify it with redacted live plugin-handler proof.

## Reproduction Assessment

Not applicable as a feature/API PR. Source inspection confirms current main has no `agent_to_agent_turn` hook and intermediate A2A turns remain internal until the final announce send.

## Solution Assessment

No. A semantic A2A turn hook is a plausible core seam, but this branch is not the best mergeable shape until it is rebased, conversation-access gated, tested, documented, and proven in a real setup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1388`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` text plus session and delivery metadata, but the PR registers it as an ordinary hook. On current main, non-bundled plugins only get raw conversation hooks through `CONVERSATION_HOOK_NAMES` plus `plugins.entries.<id>.hooks.allowConversationAccess=true`, so this hook needs the same gate when rebased.
  - confidence: 0.9
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1388-1416`
  - body: Current main imports hook names, handler maps, and conversation-hook classification from `src/plugins/hook-types.ts`, while `src/plugins/types.ts` only re-exports that surface. Adding the new hook to the old `types.ts` block will conflict or leave the current registry/SDK contract incomplete after rebase.
  - confidence: 0.9
- **[P2] Emit turn zero outside the ping-pong guard:** `src/agents/tools/sessions-send-tool.a2a.ts:62-66`
  - body: `resolvePingPongTurns()` allows `maxPingPongTurns: 0`, but the new hook setup lives inside the `maxPingPongTurns > 0` guard. In that configuration the target can still produce the documented `turn: 0` reply, but plugins receive no `agent_to_agent_turn` event.
  - confidence: 0.86
- **[P2] Wait for the whole hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:167`
  - body: The final wait races the serialized `hookChain` against a single `perTurnTimeout`. In a multi-turn exchange where several handlers are slow but each stays under the per-turn limit, the announce step can run while later turn hooks are still pending, so users may receive the final conclusion before intermediate turns.
  - confidence: 0.8
