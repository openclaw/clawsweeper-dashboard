## Summary

Keep open. The DingTalk/channel-plugin discussion shows a real use case for A2A turn visibility, but this external feature PR is not ready to merge: current main still lacks the hook, the branch is stale against the current hook contract surface, real behavior proof is missing, and the raw reply hook needs conversation-access gating.

## What This Changes

The PR adds a public `agent_to_agent_turn` plugin hook, a hook-runner method, and A2A flow emissions for intermediate turns with flow IDs and delivery-target metadata.

## Best Possible Solution

Rebase the hook into the current `hook-types.ts` architecture, gate it as conversation access, define and document ordering/timeout/routing semantics, add focused wiring/A2A tests, and provide redacted proof from a real A2A run with a plugin handler.

## Reproduction Assessment

Not applicable as a feature/API PR. Source inspection confirms current main has no A2A turn hook and only sends the final announce through the channel delivery path.

## Solution Assessment

No. A semantic A2A turn hook is a plausible seam, but this branch needs a rebase, conversation-access gating, zero-turn/ordering fixes, docs, tests, real proof, and maintainer API approval before it is the right solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1866`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` text and delivery metadata, but the branch registers it as a normal hook. Registry policy only gates names returned by `isConversationHookName`, so a non-bundled plugin could subscribe without `allowConversationAccess`; add this hook to the conversation hook set when rebasing.
  - confidence: 0.9
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1385`
  - body: Current main imports hook names and handler maps from `src/plugins/hook-types.ts`, while this branch adds `agent_to_agent_turn` to the old `src/plugins/types.ts` contract block. After rebase, register the name, event/context types, handler map, and runner exports in the current hook contract surface so `api.on(...)` can type and load the hook.
  - confidence: 0.92
- **[P2] Emit turn zero when ping-pong is disabled:** `src/agents/tools/sessions-send-tool.a2a.ts:67-70`
  - body: `resolvePingPongTurns()` allows `maxPingPongTurns` to be `0`, but this hook setup lives inside the `maxPingPongTurns > 0` guard. In that configuration the target agent still produces the initial reply, yet plugins never receive the documented `turn: 0` event, so the visibility hook is silent for zero-turn A2A setups.
  - confidence: 0.86
- **[P2] Wait for the serialized hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:163-165`
  - body: The final wait races the entire serialized `hookChain` against a single `perTurnTimeout`. In a multi-turn exchange where each handler is slow but under its per-turn budget, the final announce can still go out before later intermediate turn hooks finish, breaking the chronology this hook is meant to expose.
  - confidence: 0.82
