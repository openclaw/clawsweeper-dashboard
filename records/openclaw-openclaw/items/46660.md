## Summary

Keep open: the requested API seam is plausible and current main does not implement it, but the branch is not merge-ready because it bypasses the existing conversation-access model, targets an older hook-contract layout, lacks docs, and has no real behavior proof.

## What This Changes

The PR adds an `agent_to_agent_turn` plugin hook and emits A2A turn events so channel plugins can forward intermediate ping-pong replies.

## Best Possible Solution

Ship a documented, conversation-access-gated A2A turn hook in the current `hook-types.ts` architecture, with focused wiring coverage and redacted proof from a real A2A run using a plugin handler.

## Reproduction Assessment

Not applicable as a feature/API PR. Source inspection confirms current main has no `agent_to_agent_turn` hook and only sends the final A2A announce to the channel.

## Solution Assessment

No. A semantic A2A turn hook may be a good seam, but this branch needs rebase, conversation-access gating, docs, tests, real proof, and maintainer approval for the public contract.
