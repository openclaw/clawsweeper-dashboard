## Summary

Keep open: the PR targets a real plugin visibility gap, but current main does not implement this hook yet and the proposed public API still needs maintainer review plus fixes for hook ordering and release/docs coverage.

## What This Changes

The PR adds an `agent_to_agent_turn` plugin hook with event/context types, a hook-runner wrapper, and A2A ping-pong dispatch that includes reply and routing metadata.

## Best Possible Solution

Decide the public hook contract, then rebase onto current main, preserve existing A2A safeguards, add docs/changelog, and cover the hook runner plus A2A dispatch ordering in focused tests.

## Reproduction Assessment

Not applicable as a feature PR rather than a current-main bug. Source inspection verifies current main lacks the hook and that intermediate A2A turns are not delivered through the existing final-message path.

## Solution Assessment

Unclear. A typed plugin hook is a plausible direction, but the current queue wait can still let the final announce overtake queued turn hooks and the public API needs maintainer acceptance plus documentation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Wait for all queued turn hooks before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:167`
  - body: This races the entire serialized `hookChain` against a single `perTurnTimeout`. With several turns whose handlers each finish under that timeout but collectively take longer, the final announce can still run while later turn events are queued, so users may receive the conclusion before intermediate A2A messages. Bound individual handlers without cutting off the whole queue, or make that degraded ordering explicit.
  - confidence: 0.86
- **[P3] Document the new public hook before release:** `src/plugins/types.ts:1388`
  - body: This adds `agent_to_agent_turn` to the public plugin hook union and runtime, but the hook catalog and changelog do not mention the new event contract. Plugin authors need the payload semantics and release note before relying on the API.
  - confidence: 0.94
