## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1869`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` text plus session and delivery metadata, but the PR registers it as an ordinary hook. Non-bundled plugin access is restricted only for names in the conversation hook set, so this hook needs to be gated there when rebased.
  - confidence: 0.9
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1385`
  - body: Current main defines hook names, handler maps, and exports from `src/plugins/hook-types.ts`, while `types.ts` imports that surface. Adding the new contract to the old `types.ts` block will be lost or conflict on rebase, so the hook must be registered in `hook-types.ts`.
  - confidence: 0.9
- **[P2] Emit turn zero when ping-pong is disabled:** `src/agents/tools/sessions-send-tool.a2a.ts:67-68`
  - body: `resolvePingPongTurns()` allows `0`, but the new hook setup lives inside the `maxPingPongTurns > 0` guard. In that configuration the target still produces the initial reply and the API models it as `turn: 0`, yet handlers receive no event.
  - confidence: 0.86
- **[P2] Wait for the full hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:164`
  - body: The final wait races the whole serialized `hookChain` against one `perTurnTimeout`. With multiple slow-but-under-timeout handlers, the announce step can run while later turn hooks are still pending, so users can receive the conclusion before intermediate turns.
  - confidence: 0.8
