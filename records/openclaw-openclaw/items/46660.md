## Summary

Keep open: current main still lacks A2A turn visibility for plugins, but this branch is stale against the current hook contract split and exposes intermediate transcript text without the conversation-access gate.

## What This Changes

The PR adds an `agent_to_agent_turn` plugin hook and emits A2A ping-pong turn events with reply text, flow identity, session roles, and delivery-target metadata.

## Best Possible Solution

Ship a first-class A2A turn visibility hook after rebasing onto current main, defining the contract in `src/plugins/hook-types.ts`, gating it as conversation content, and adding focused docs, changelog, loader-policy tests, and A2A emission tests.

## Reproduction Assessment

Yes. A static current-main check is enough: intermediate A2A turns run through `runAgentStep`, only the final announce calls gateway `send`, and no existing hook emits those intermediate replies.

## Solution Assessment

No. The feature direction is plausible and narrow, but this branch is not the best merge shape because it updates a stale hook catalog location and bypasses the current raw-conversation-access policy.
