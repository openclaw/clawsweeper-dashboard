## Summary

Keep open. Current main still lacks the proposed A2A turn hook, and the DingTalk @sub-agent cross-reference is useful demand evidence, but this public plugin API PR still needs a rebase, conversation-access gating, hook-contract placement, turn-order/zero-turn fixes, tests/docs, and real behavior proof before it is mergeable.

## What This Changes

The branch adds a public `agent_to_agent_turn` plugin hook, a hook-runner method, and A2A flow emissions carrying turn, reply, flow, and delivery-target metadata.

## Best Possible Solution

Rebase the hook into `hook-types.ts`, gate it as conversation access, define and document ordering/timeout/routing semantics, add focused A2A/hook tests, and provide redacted proof from a real plugin-handler run.

## Reproduction Assessment

Not applicable as a feature/API PR. Source inspection confirms current main has no A2A turn hook and only sends the final announce through the channel delivery path.

## Solution Assessment

No. A semantic A2A turn hook is a plausible seam, but this branch is not the best mergeable shape until it is rebased, gated, tested, documented, and proven in a real setup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1385`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` text and delivery metadata, but the branch registers it as an ordinary hook. Registry policy only gates names returned by `isConversationHookName`, so add this hook to the conversation hook set when rebasing.
  - confidence: 0.9
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1385`
  - body: Current main imports hook names and handler maps from `src/plugins/hook-types.ts`, while this branch adds the new hook to the old `types.ts` contract block. Register the name, event/context types, handler map, and runner exports in the current hook contract surface.
  - confidence: 0.92
- **[P2] Emit turn zero when ping-pong is disabled:** `src/agents/tools/sessions-send-tool.a2a.ts:67-70`
  - body: `resolvePingPongTurns()` allows `maxPingPongTurns` to be `0`, but the new hook setup lives inside the `maxPingPongTurns > 0` guard. In that configuration the target agent still produces the initial reply, yet plugins never receive the documented `turn: 0` event.
  - confidence: 0.86
- **[P2] Wait for the serialized hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:163-165`
  - body: The final wait races the entire serialized `hookChain` against one `perTurnTimeout`. In a multi-turn exchange where each handler is slow but under its per-turn budget, the final announce can still go out before later intermediate hooks finish, breaking transcript chronology.
  - confidence: 0.82
