## Summary

Keep this PR open: it addresses a real A2A visibility gap and the linked DingTalk use case is credible, but the branch is not mergeable on current main because the hook contract moved, the raw reply hook is not conversation-access gated, real behavior proof is still missing, and ordering semantics remain underspecified.

## What This Changes

The branch adds a typed `agent_to_agent_turn` plugin hook, a hook-runner method, and turn-event emissions in the A2A `sessions_send` flow.

## Best Possible Solution

Land a rebased core hook only if maintainers accept the API: define it in `hook-types.ts`, treat it as a conversation hook, specify best-effort ordering and timeout behavior, add focused A2A/hook tests and docs, then verify it with a redacted live channel-plugin handler.

## Reproduction Assessment

Not applicable as a feature/API PR. Source inspection confirms current main has no `agent_to_agent_turn` hook and intermediate A2A turns stay internal until the final announce send.

## Solution Assessment

No. A semantic A2A turn hook is a plausible core seam, but this branch is not the best mergeable shape until it is rebased, conversation-access gated, tested, documented, and proven in a real setup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate A2A turn hooks behind conversation access:** `src/plugins/types.ts:1866`
  - body: `agent_to_agent_turn` exposes raw intermediate `reply` text plus session and delivery metadata, but this branch registers it as an ordinary hook. On current main, non-bundled plugins only get raw conversation hooks when the hook name is included in `CONVERSATION_HOOK_NAMES` and `allowConversationAccess=true`, so this new hook needs that gate when rebased.
  - confidence: 0.9
- **[P2] Move the hook contract into hook-types:** `src/plugins/types.ts:1385-1413`
  - body: Current main imports hook names, handler maps, and conversation-hook classification from `src/plugins/hook-types.ts`, while `src/plugins/types.ts` now re-exports that surface. Adding the new hook to the old `types.ts` block will conflict or leave the current registry/SDK contract unchanged, so rebase the event/context types, hook name, handler map entry, and exports into `hook-types.ts`.
  - confidence: 0.9
- **[P2] Emit turn zero outside the ping-pong guard:** `src/agents/tools/sessions-send-tool.a2a.ts:67`
  - body: The new hook setup lives inside the `maxPingPongTurns > 0` guard. With `maxPingPongTurns: 0`, the target can still produce the initial reply and the event contract documents that as `turn: 0`, but plugins receive no `agent_to_agent_turn` event.
  - confidence: 0.86
- **[P2] Wait for the whole hook queue before announcing:** `src/agents/tools/sessions-send-tool.a2a.ts:164-165`
  - body: The final wait races the serialized `hookChain` against a single `perTurnTimeout`. In a multi-turn exchange with several slow-but-under-timeout handlers, the announce step can run while later turn hooks are still pending, so users can receive the final conclusion before intermediate turns despite the chronology comment.
  - confidence: 0.8
