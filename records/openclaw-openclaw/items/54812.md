## Summary

Keep open. Current main still authorizes channel-initiated config writes only by channel and account, so the requested DM/group/topic/conversation-specific policy remains absent from the shared authorization model, command gates, bundled channel schemas, docs, and contract tests.

## What This Changes

Request per-context config-write permissions so multi-context channels can allow configuration writes from trusted DMs or admin groups while denying them from client groups or other conversations.

## Best Possible Solution

Add a backward-compatible generic config-write policy model that carries normalized conversation identity through channel/plugin boundaries, defines inheritance from channel/account defaults into DM/group/topic/thread overrides, exposes narrow SDK helpers, updates bundled multi-context schemas/docs, and adds allow, deny, and inheritance contract tests.
