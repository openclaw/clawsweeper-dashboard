- **[P1] Forward abort signals through remote embeddings:** `packages/memory-host-sdk/src/host/embeddings-remote-provider.ts:44-48`
  - body: `EmbeddingProvider` still accepts per-call `options.signal` on current main, and remote embeddings currently pass that signal through `fetchRemoteEmbeddingVectors()` and `postJson()`. This PR drops the options parameter and calls `embed()` without a signal, so cancelled or timed-out memory searches can keep the provider HTTP request/body read running.
  - confidence: 0.92
- **[P1] Scope MemoryHostServices per async operation:** `packages/memory-host-sdk/src/host/services.ts:84-107`
  - body: `withMemoryHostServices()` swaps a module-global `activeServices` value for an async callback. Overlapping memory operations can observe another operation's auth, SSRF fetch, redaction, MIME, or session-sanitization hooks, and the setters are re-exported through SDK facades, so this needs AsyncLocalStorage-style scoping or trusted host-internal wiring before merge.
  - confidence: 0.91
- **[P1] Use byte-sniffing MIME detection in default services:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: The default `detectMime` ignores the provided buffer and trusts a small image-extension map. `buildFileEntry()` passes file bytes and then accepts any returned `${modality}/` MIME, so the standalone package path can accept spoofed `.png` files and drop valid sniffed audio/video formats that current `@openclaw/media-core` detection handles.
  - confidence: 0.86
- **[P2] Preserve bounded response reads in postJson:** `packages/memory-host-sdk/src/host/post-json.ts:23-35`
  - body: Current main uses bounded response helpers and passes abort signals while reading both success and error bodies. The PR changes this to unbounded `res.text()` and `res.json()`, so a remote embedding endpoint can hang or allocate unexpectedly on large or stalled responses.
  - confidence: 0.87
