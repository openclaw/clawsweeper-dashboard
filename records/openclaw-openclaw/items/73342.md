- **[P1] Scope MemoryHostServices per async operation:** `packages/memory-host-sdk/src/host/services.ts:84-101`
  - body: `withMemoryHostServices` mutates one module-global `activeServices` around an async callback, while the service object includes auth resolution, guarded fetch, redaction, MIME, memory, and session-sanitization hooks. Overlapping memory operations can observe another operation's services or restore a stale binding, so use AsyncLocalStorage or explicit service plumbing before exposing this seam.
  - confidence: 0.91
- **[P1] Preserve remote HTTP abort forwarding:** `packages/memory-host-sdk/src/host/remote-http.ts:13-21`
  - body: Current main and v2026.6.9 accept `signal?: AbortSignal` and pass it into `fetchWithSsrFGuard`; this branch removes the parameter and no longer forwards it. Remote embedding/batch requests that callers abort will keep running until the underlying fetch finishes, which is a shipped behavior regression.
  - confidence: 0.9
- **[P1] Use byte-aware MIME detection in default services:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: The default `detectMime` now ignores the supplied buffer/header and trusts a small image-extension map, but `buildFileEntry` uses that result to decide whether workspace files become inline multimodal memory payloads. Delegate to the existing media-core detector or otherwise preserve byte/header sniffing so renamed files and audio/video containers are not misclassified or dropped.
  - confidence: 0.89
