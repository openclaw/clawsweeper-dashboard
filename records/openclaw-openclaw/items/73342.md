  - body: `src/media/mime.*` does not exist in current main or this PR head, while the existing memory-host bridge and other callers import MIME helpers from `@openclaw/media-core/mime`. This new facade import will fail module resolution for the documented memory-host SDK surface.
  - confidence: 0.95
- **[P1] Forward abort signals through guarded fetch:** `packages/memory-host-sdk/src/host/remote-http.ts:18`
  - body: Current main accepts `signal` on `withRemoteHttpResponse` and forwards it into `fetchWithSsrFGuard`; this PR's new service type/call path omits it. Provider-backed embedding requests that are cancelled or timed out can continue running after the caller aborts.
  - confidence: 0.9
- **[P1] Scope MemoryHostServices per async operation:** `packages/memory-host-sdk/src/host/services.ts:82-107`
  - body: `withMemoryHostServices` mutates one module-global `activeServices` value around an async callback. Overlapping memory work can see another request's auth, SSRF fetch, redaction, MIME, or session hooks, so this needs AsyncLocalStorage or explicit service passing before it is exposed.
  - confidence: 0.9
- **[P1] Use byte-sniffing MIME detection in defaults:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: The package default `detectMime` ignores the supplied buffer/header and only maps a few image extensions, while callers pass file bytes for validation and current media-core sniffs content. Host-less package paths will drop valid audio/CAF files and can trust spoofed image extensions differently from core.
  - confidence: 0.88
- **[P2] Preserve OpenAI attribution headers:** `packages/memory-host-sdk/src/host/embeddings-remote-client.ts:36-41`
  - body: The refactor removes the native OpenAI route check and no longer adds OpenClaw `originator`, `version`, and `User-Agent` headers for memory embedding calls to `api.openai.com`. Current tests lock those headers, so the service refactor needs to carry that behavior forward.
  - confidence: 0.82
