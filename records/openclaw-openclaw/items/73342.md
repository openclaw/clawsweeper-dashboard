## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope MemoryHostServices per async operation:** `packages/memory-host-sdk/src/host/services.ts:84-107`
  - body: `withMemoryHostServices` writes a module-global `activeServices` value, so overlapping async memory work can observe another operation's auth, guarded-fetch, redaction, MIME, or session-sanitization services. Use request/operation scoping such as `AsyncLocalStorage`, or keep the override API out of plugin-facing/runtime subpaths unless it is only a trusted startup binding.
  - confidence: 0.9
- **[P1] Forward abort signals through guarded fetch:** `packages/memory-host-sdk/src/host/remote-http.ts:13-35`
  - body: The PR-head wrapper no longer accepts or forwards `signal`, while current memory remote/batch callers pass abort signals through this path. Canceled remote embedding or batch requests can keep running until the network call finishes, so preserve the `signal?: AbortSignal` parameter and pass it to `fetchWithSsrFGuard`, with the current focused test restored.
  - confidence: 0.87
- **[P1] Use byte-sniffing MIME detection in default services:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: `buildFileEntry` supplies file bytes before accepting multimodal content, but the default service ignores `buffer` and trusts the file extension. That diverges from current media-core detection, drops valid audio/video or extensionless inputs, and can treat renamed non-image data as image payloads; the default should sniff bytes or fail closed for multimodal validation.
  - confidence: 0.85
