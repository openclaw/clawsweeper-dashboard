## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Scope MemoryHostServices per async operation:** `packages/memory-host-sdk/src/host/services.ts:82-101`
  - body: `withMemoryHostServices` mutates a module-global service bundle around an async callback. Because that bundle includes auth resolution, guarded fetch, redaction, MIME, and session-sanitization hooks and is exported through plugin-facing facades, overlapping memory operations can observe another caller's services. Use AsyncLocalStorage or explicit service parameters, and keep mutable setters out of plugin-facing runtime surfaces unless maintainers intentionally approve that API.
  - confidence: 0.88
- **[P1] Preserve abort signals in remote memory HTTP:** `packages/memory-host-sdk/src/host/remote-http.ts:13-20`
  - body: Current main's remote memory HTTP wrapper accepts `signal?: AbortSignal` and forwards it into `fetchWithSsrFGuard`, but this branch's service-backed wrapper has no signal parameter and does not pass one to the guarded fetch. Provider-backed embedding requests that rely on timeout/cancel signals can keep running after callers abort; carry the signal through `MemoryHostGuardedFetch`, `withRemoteHttpResponse`, and the post JSON path with the existing regression coverage.
  - confidence: 0.86
- **[P2] Sniff MIME bytes before accepting multimodal files:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: The default service maps MIME only from a small extension list, but `buildFileEntry` passes file bytes and trusts the returned MIME before accepting multimodal memory files. That diverges from the current media-core detector, which sniffs bytes first, so the package-local path can accept renamed non-images or drop valid audio/video files; delegate to the shared detector or implement equivalent byte sniffing.
  - confidence: 0.83
