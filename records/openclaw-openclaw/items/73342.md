  - body: This wrapper now falls back to `getMemoryHostServices().network.fetchWithSsrFGuard`, but the default service throws and the plugin SDK memory embedding facade still exports this package helper to provider plugins.   - confidence: 0.88
- **[P1] Scope MemoryHostServices per async operation:** `packages/memory-host-sdk/src/host/services.ts:82-107`
  - body: `withMemoryHostServices()` mutates a module-global `activeServices` value around an async callback. Two overlapping memory operations can observe each other's auth, guarded-fetch, MIME, or redaction hooks, and the setter is re-exported through memory SDK facades, so this should be request-scoped or kept internal to trusted host wiring.
  - confidence: 0.9
- **[P1] Preserve current abort-signal forwarding:** `packages/memory-host-sdk/src/host/services.ts:18-25`
  - body: Current main accepts `signal?: AbortSignal` on `withRemoteHttpResponse` and forwards it to the guarded fetch, with tests covering embedding timeout cancellation. The PR head's new guarded-fetch service contract has no signal field, so the rebase/repair must keep that current-main cancellation path instead of narrowing the remote request contract.
  - confidence: 0.82
- **[P2] Sniff MIME from bytes before accepting multimodal files:** `packages/memory-host-sdk/src/host/services.ts:129-130`
  - body: The default `detectMime` ignores `buffer` and `headerMime` and trusts only a short image-extension map. `buildFileEntry()` passes actual bytes before forwarding multimodal memory files downstream, so the default service should preserve byte sniffing or it can accept spoofed extensions and reject valid audio/video/container inputs.
  - confidence: 0.86
