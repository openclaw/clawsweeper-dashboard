Full review comments:

- **[P1] Scope service overrides to async or host-only use:** `packages/memory-host-sdk/src/host/services.ts:84-101`
  - body: `withMemoryHostServices()` writes a module-global `activeServices` value. Two overlapping memory operations can read each other's auth, SSRF fetch, redaction, MIME, or session hooks, and these helpers are re-exported through SDK facades, so the override needs AsyncLocalStorage/request scoping or a host-internal-only API.
  - confidence: 0.93
- **[P1] Use byte-aware MIME detection in default services:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: The default service ignores the provided buffer and trusts only a short extension map. `buildFileEntry` uses this MIME result to decide which bytes become multimodal memory input, so renamed non-images can be accepted as images and valid audio/video inputs that current media-core detects can be dropped.
  - confidence: 0.91
- **[P1] Preserve abort and bounded response handling:** `packages/memory-host-sdk/src/host/remote-http.ts:13-21`
  - body: Current main carries `AbortSignal` and bounded response reads through remote memory HTTP helpers, but this branch's `withRemoteHttpResponse` contract no longer accepts the signal and the PR branch's JSON helpers drop the bounded readers. Provider-backed memory requests must keep cancellation and response-size limits during rebase.
  - confidence: 0.88
