## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Bind remote HTTP to the core guarded fetch:** `packages/memory-host-sdk/src/host/services.ts:141-145`
  - body: `withRemoteHttpResponse` is re-exported through the public memory plugin SDK path, and current Google, GitHub Copilot, and Voyage embedding providers import it from there. In this PR the package-local default `fetchWithSsrFGuard` throws unless some global host binding has been installed, so those existing provider calls can fail at runtime instead of using the core SSRF-guarded fetch they use on main.
  - confidence: 0.9
- **[P1] Preserve AbortSignal in the remote fetch contract:** `packages/memory-host-sdk/src/host/services.ts:18-25`
  - body: The new `MemoryHostGuardedFetch` type does not include `signal`, and the rewritten remote HTTP wrapper only forwards `init`, policy, and audit context. Current main's guarded fetch contract accepts `signal`, and existing embedding callers pass abort signals through `withRemoteHttpResponse`, so this refactor breaks cancellation for provider requests.
  - confidence: 0.88
- **[P1] Scope memory host services per async operation:** `packages/memory-host-sdk/src/host/services.ts:84-87`
  - body: `withMemoryHostServices` writes a module-global `activeServices` value and then awaits user work before restoring it. Two overlapping memory operations can observe each other's auth, network, redaction, or session hooks, and the setters are re-exported through public SDK facades, so this should use request/async-local scoping or stay internal to trusted host wiring.
  - confidence: 0.86
- **[P2] Sniff MIME from bytes before accepting multimodal files:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: The default `detectMime` ignores the provided buffer and only trusts a small image-extension map. `buildFileEntry` passes file bytes to validate multimodal memory files, so this can skip valid audio/video/image files without those extensions and can treat arbitrary renamed files as images instead of preserving main's byte-sniffing behavior.
  - confidence: 0.86

## Security Review
