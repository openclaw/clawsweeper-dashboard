## Summary

Keep this PR open because the protected `maintainer` label prevents cleanup closure, and the current head is not merge-ready: the refactor leaves public memory-host SDK helpers using unbound package services that can break remote embeddings, cancellation, and multimodal indexing.

## What This Changes

The PR replaces the memory-host package’s private core runtime bridge with a package-local `MemoryHostServices` contract and moves OpenClaw-bound compatibility facades into core/plugin SDK paths.

## Best Possible Solution

Land the boundary refactor only after the plugin SDK facades preserve existing remote HTTP/auth/MIME/session behavior through explicit host binding or core-bound shims, with conflicts resolved and focused compatibility tests green.

The unbound service path is also visible from the PR’s plugin SDK re-export to package helpers and the default `fetchWithSsrFGuard` throw.

## Solution Assessment

No. The package-boundary direction is plausible, but this branch is not the best implementation until host services are explicitly bound or the core-bound SDK facades keep the previous runtime behavior intact.

  - body: `src/plugin-sdk/memory-core-host-engine-embeddings.ts` still re-exports the package remote HTTP helpers, but this fallback now resolves `services.fetchWithSsrFGuard`; the default implementation throws unless a host binding is installed.   - body: The new parameter type no longer includes `signal`, and the guarded fetch call no longer forwards it. Existing callers such as `postJson` and the Google embedding provider pass `signal`, and the current remote HTTP test asserts that the guarded fetch receives it, so this breaks the public helper contract and cancellation behavior.
  - confidence: 0.93
- **[P2] Use full MIME detection for multimodal memory files:** `packages/memory-host-sdk/src/host/services.ts:130`
  - body: The package default `detectMime` ignores the supplied buffer and maps only image extensions. `buildFileEntry` classifies audio files like `.mp3` and `.wav` as multimodal, then rejects them when this detector returns `undefined`, so audio memory files that indexed on main can be silently skipped after the refactor.
  - confidence: 0.89
