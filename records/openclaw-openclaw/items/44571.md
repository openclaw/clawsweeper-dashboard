## Summary

Keep open. Current main has read-through inheritance for agents with no local profile, but the onboarding/provider auth write paths still persist new static/token credentials to one resolved agent store, and explicit local profiles can still shadow inherited main/default credentials.

## What This Changes

The issue asks for `openclaw onboard` provider authentication to make new Anthropic credentials usable by every configured agent instead of only the resolved default/main agent store.

## Best Possible Solution

Define one multi-agent credential contract, then implement it consistently across interactive onboarding, non-interactive onboarding, `models auth --agent`, runtime read-through, and stale local override handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: provider onboarding/setup-token writes to one resolved `agentDir`, while runtime inheritance only helps when the target agent has no local overriding profile.

## Solution Assessment

Unclear. Copying credentials to every configured agent may fix the immediate symptom, but maintainers should first choose the credential fan-out, SecretRef portability, OAuth, and stale-local-profile overwrite semantics.
