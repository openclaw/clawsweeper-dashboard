## Summary

Current main implements the issue’s alternative fix: non-default agents read through to the default/main auth store when they lack a local profile, so onboarding no longer needs to clone auth-profiles into every configured agent for the reported missing-profile case.

## What This Changes

The issue requests that onboarding credentials be written or inherited across all configured agents instead of only the default agent.

## Best Possible Solution

Keep the shipped read-through inheritance model and handle broader stale local profile rotation or warning UX in https://github.com/openclaw/openclaw/issues/47684.

## Reproduction Assessment

No, not against current main for the missing local auth-profile case. Source and regression coverage show a secondary agent with no local profile now resolves the default/main auth store at runtime without creating a copied auth-profiles.json.

## Solution Assessment

Yes. The shipped read-through approach solves the reported missing-profile failure while avoiding broad secret fanout; stale local profile shadowing is a separate rotation policy issue tracked at https://github.com/openclaw/openclaw/issues/47684.
