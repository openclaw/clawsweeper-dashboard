## Summary

Keep open. Current main has main-store read-through and selected OpenAI OAuth drift repair, but provider onboarding still writes static/token auth profiles to one resolved agent directory, and local static profiles can still shadow inherited credentials.

## What This Changes

The issue asks for `openclaw onboard` auth setup to make provider credentials available to every configured agent instead of only the resolved/default auth store.

## Best Possible Solution

Define one multi-agent auth contract, then implement it consistently: portable static credentials can be synced or repaired during onboarding, OAuth remains read-through or provider-opt-in, and stale local shadowing gets clear warning or repair behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: onboarding writes static/token profiles through a single `agentDir`, while runtime merging allows local static profiles to override inherited main credentials.

## Solution Assessment

Unclear. Writing every credential to every agent is not obviously the best fix because SecretRefs, OAuth refresh material, and existing local overrides need explicit portability and overwrite semantics.
