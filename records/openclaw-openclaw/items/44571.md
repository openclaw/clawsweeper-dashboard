## Summary

Current main still does not explicitly distribute onboarding auth profiles across all configured agents. Provider onboarding and Anthropic setup-token paths persist to one resolved `agentDir`; runtime fallback helps missing stores and some OAuth-refresh cases, but agent-local token/API-key profiles can still shadow main credentials, and the docs still reflect unresolved per-agent auth semantics.

## Best Possible Solution

Keep this issue open for a maintainer decision on multi-agent auth semantics. The best path is either to sync selected onboarding credentials to configured agent dirs with SecretRef-safe overwrite rules, or to keep per-agent isolation and add a first-class auth sync/repair/warning path for stale profiles that shadow main credentials. The fix should cover Anthropic setup-token/tokenRef, API-key SecretRefs, missing stores, stale conflicting non-main profiles, and docs alignment.
