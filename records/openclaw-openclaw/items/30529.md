## Summary

Keep open. Current main has automatic Discord typing, including the merged early-DM cue, reply-pipeline typing, and heartbeat typing, but it still lacks the requested message-tool `typing` or `typing-indicator` action and the generic public control semantics discussed in the issue.

## What This Changes

The issue requests native Discord typing-indicator support through the message tool, either as automatic lifecycle behavior or as an explicit typing action.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord by reusing the existing Discord typing primitive and shared typing lifecycle with discovery, execution, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but the requested manual/public message-tool action is absent from the action contract, Discord discovery, and Discord handler.

## Solution Assessment

Unclear until maintainers choose the API. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than adding an ad hoc Discord-only alias.
