## Summary

Keep open: current main and the latest release provide automatic Discord typing with heartbeat and cleanup, but the message tool still exposes no typing action. The remaining work is a distinct public API and lifecycle decision that is not fully owned by the broader placeholder proposal at https://github.com/openclaw/openclaw/issues/90866.

## What This Changes

Add native Discord typing feedback to the message tool, or automatically maintain typing while an agent processes a Discord reply.

## Best Possible Solution

Keep automatic Discord typing as the canonical lifecycle, then add a typed channel capability only if maintainers approve a generic manual busy/typing contract with explicit target, refresh, timeout, and cleanup semantics shared with the existing controller.

## Reproduction Assessment

Not applicable: this is a request for a new public message-tool capability; source inspection confirms automatic Discord typing exists and the requested manual action does not.

## Solution Assessment

No, not as a Discord-only standalone action without further design. The maintainable path is to reuse the existing channel-owned lifecycle through an approved generic typed capability, avoiding competing typing controllers.
