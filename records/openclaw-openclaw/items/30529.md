## Summary

Keep open: current main and v2026.5.12 have Discord typing primitives and automatic typing paths, but they still do not expose the requested public `message(action="typing")` or generic message-tool typing action. The discussion broadens this into a public API/lifecycle choice, so it is not a conservative cleanup close.

## What This Changes

Request native Discord typing indicator support through the message tool, either as an automatic behavior or an explicit typing action.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord discovery, execution, docs, and focused regression coverage while reusing existing typing lifecycle safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but the requested message-tool `typing` or generic typing action remains absent on current main and v2026.5.12.

## Solution Assessment

Unclear until maintainers choose the public API shape. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than a Discord-only alias, while automatic lifecycle fixes only cover part of the request.
