## What This Changes

Request to expose Discord native typing indicators through the message tool or a generic typing/busy action so agents can show work-in-progress feedback.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord discovery/execution, docs, and focused coverage while reusing the existing typing lifecycle safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but the requested public `typing` or generic busy action remains absent from the message-tool contract on current main.

## Solution Assessment

Unclear until maintainers choose the public API shape. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than a Discord-only alias, while automatic typing paths only cover part of the request.
