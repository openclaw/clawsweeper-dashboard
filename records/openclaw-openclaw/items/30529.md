## Summary

Keep open. Current main has automatic Discord typing and heartbeat typing plumbing, but it still lacks a supported message-tool `typing` action; related open work covers automatic typing gaps or tool-dispatch typing, not this public API request.

## What This Changes

Expose Discord native typing indicators through the message tool, either automatically during processing or via a manual `typing` or `typing-indicator` action.

## Best Possible Solution

Define a public channel-owned typing or busy control contract, then implement Discord through the existing `sendTypingDiscord` and shared typing lifecycle with discovery, execution, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main can send Discord typing automatically, but cannot reproduce the requested manual message-tool action because it is absent from the advertised action contract and Discord handler.

## Solution Assessment

Unclear until maintainers choose the API. A generic channel-owned typing or busy action that reuses existing heartbeat and typing lifecycle seams looks safer than an ad hoc Discord-only alias.
