## Summary

Keep open. Current main has automatic Discord typing plumbing for replies and heartbeat targets, but it still does not expose a `typing` or `typing-indicator` action through the shared message tool; the related implementation PR #30535 was closed unmerged.

## What This Changes

The issue requests native Discord typing indicators through the message tool, either automatically during processing or as an explicit typing action.

## Best Possible Solution

Define the public typing-action contract first, then either add a backward-compatible shared action wired through Discord discovery/dispatch and docs/tests, or document automatic typing as the only supported surface.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared action enum, target-mode map, CLI normalization, agent schema discovery, Discord discovery, and Discord dispatcher all omit a typing action while unknown actions are rejected before dispatch.

## Solution Assessment

Unclear. A Discord-only manual action is mechanically narrow, but it changes the public message-tool contract and the discussion explicitly raises cross-provider lifecycle behavior, so maintainer API direction should come first.
