## Summary

Keep open. Current main and the latest release have automatic Discord typing paths, but neither exposes the requested public message-tool typing action, so this remains a feature/API decision rather than an implemented or stale item.

## What This Changes

Request native Discord typing-indicator support in the message tool, either automatically or through a manual typing/typing-indicator action.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord discovery, execution, docs, and regression coverage using the existing typing primitives and lifecycle safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but the requested manual/public message-tool action is absent.

## Solution Assessment

Unclear until maintainers choose the API. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than adding an ad hoc Discord-only alias.
