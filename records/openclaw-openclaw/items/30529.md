## Summary

Keep open: automatic Discord typing with keepalive is present in current main and v2026.6.5, but the requested public message-tool typing action is still absent and needs a maintainer API/lifecycle decision. The broader Discord placeholder work at https://github.com/openclaw/openclaw/issues/90866 overlaps on user feedback, but it does not settle explicit manual typing control.

## What This Changes

Request adding native Discord typing-indicator support to the message tool, either automatically or through an explicit action.

## Best Possible Solution

Approve a generic message-tool typing or busy capability only if maintainers want manual control; otherwise keep automatic Discord typing as the canonical lifecycle.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than broken existing behavior. Source inspection confirms automatic Discord typing exists and the requested message-tool action is absent on current main and v2026.6.5.

## Solution Assessment

No, not as a Discord-only standalone action without further design. The safer path is a generic typed channel busy/typing capability that reuses the existing lifecycle and defines target, refresh, timeout, and cleanup semantics.
