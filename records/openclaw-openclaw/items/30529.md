## Summary

Keep open. Current main already has automatic Discord typing during inbound reply runs, but the requested explicit message-tool typing action is still not present, and adding it would require a generic channel/message-tool contract decision rather than conservative cleanup.

## What This Changes

Request to let agents trigger Discord native typing indicators through the shared message tool, with possible automatic or manual control.

## Best Possible Solution

Define an additive generic message-tool typing action or lifecycle hook in the channel plugin contract, then implement Discord support through the existing Discord typing helper and update docs/tests around lifecycle cleanup.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection establishes the current state: automatic Discord typing exists, but explicit message-tool typing control does not.

## Solution Assessment

No for a Discord-only one-off action. The maintainable path is a generic channel/message-tool typing contract with Discord as the first implementation, informed by the related stuck-indicator lifecycle bugs.
