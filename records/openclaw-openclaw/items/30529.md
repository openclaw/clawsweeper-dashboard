Keep open: current main and v2026.5.12 have automatic Discord typing plumbing, but they still do not expose the requested public `message(action="typing")` or a generic message-tool typing action; the API shape needs maintainer decision.

## What This Changes

Request to add native Discord typing indicator support to the shared message tool, either automatically or through an explicit typing action.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord discovery, execution, docs, and focused regression coverage while reusing the existing lifecycle safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but `message(action="typing")` or a generic message-tool typing action is absent on current main and v2026.5.12.

## Solution Assessment

Unclear until maintainers choose the API shape. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than a Discord-only alias, while automatic lifecycle fixes only cover part of the request.
