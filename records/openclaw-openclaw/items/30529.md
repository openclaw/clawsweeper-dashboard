## Summary

Keep open. Current main and v2026.5.12 have automatic Discord typing paths, but still do not expose the requested public message-tool typing action, so this remains a feature/API design item.

## What This Changes

Add native Discord typing-indicator support to the message tool, either automatically or via an explicit typing action.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord discovery, execution, docs, and regression coverage using the existing lifecycle safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but the requested manual/public message-tool action is absent.

## Solution Assessment

Unclear until maintainers choose the API. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than an ad hoc Discord-only alias.
