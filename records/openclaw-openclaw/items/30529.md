## Summary

Current main implements the core UX request with automatic Discord native typing cues and keepalive-backed reply typing; the shared message tool still has no manual typing action, but the requested liveness gap is covered without adding a provider-specific tool escape hatch.

## What This Changes

The issue requests native Discord typing indicator support for agent responses, either through automatic typing while work is in progress or an explicit message-tool action.

## Best Possible Solution

Keep the channel-owned automatic typing lifecycle for Discord; track explicit manual typing control separately only if maintainers want a generic cross-channel API.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source and docs on current main show Discord now sends native typing during active runs.

## Solution Assessment

Yes. The implemented automatic lifecycle is the narrowest maintainable solution for the reported liveness gap, while a manual message-tool typing action would be a separate generic channel API decision.
