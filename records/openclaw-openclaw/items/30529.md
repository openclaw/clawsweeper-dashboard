## Summary

Keep open: current main covers automatic Discord typing for normal replies and heartbeats, but it still does not expose an explicit typing action through the shared message tool, and that public API choice needs maintainer judgment.

## What This Changes

The issue requests native Discord typing-indicator support through the message tool, either automatically during processing or via an explicit typing action.

## Best Possible Solution

Define a generic typing/busy message capability or action that channel plugins can advertise and implement consistently, while keeping the existing automatic lifecycle as the default Discord UX.

## Reproduction Assessment

Not applicable. This is a feature/API request; source inspection confirms automatic Discord typing exists, but a message-tool typing action does not.

## Solution Assessment

No, not as a narrow Discord-only shortcut yet. The maintainable direction is a generic shared action/capability with channel-owned execution and clear lifecycle semantics.
