## Summary

Keep open: current main has Discord typing support in automatic reply and heartbeat paths, but it still does not expose a `typing` or `typing-indicator` action through the shared message tool. The issue discussion also points to lifecycle bugs that make the API shape and cleanup semantics a maintainer decision rather than conservative cleanup.

## What This Changes

Request Discord native typing indicators from the message tool so agents can show processing feedback before a visible reply.

## Best Possible Solution

Keep this open for a maintainer API decision; if approved, add a generic typed message-tool contract or documented Discord action with cleanup semantics, Discord implementation, docs, and regression coverage.

## Reproduction Assessment

Yes, from source inspection: current main has no `typing` or `typing-indicator` action in the shared action enum or Discord discovery, while Discord typing exists only through automatic reply and heartbeat paths. No runtime failing test was run because this is a feature gap and the review is read-only.

## Solution Assessment

No current implementation fully solves the requested manual message-tool control. The narrow maintainable path depends on a product/API choice for generic versus Discord-only typing plus lifecycle cleanup behavior.
