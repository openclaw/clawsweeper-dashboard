## Summary

Keep open. Current main has Discord typing plumbing for automatic reply and heartbeat flows, but the public message-tool action requested here is still absent, and the open Discord lifecycle PR addresses automatic feedback rather than a `message(action="typing")` API.

## What This Changes

The issue requests native Discord typing indicator support exposed through the message tool, with possible automatic and manual action modes.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord discovery, execution, docs, and focused regression coverage while reusing the existing lifecycle safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but the requested manual/public message-tool action is absent on current main.

## Solution Assessment

Unclear until maintainers choose the API shape. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than a Discord-only alias, while the active lifecycle PR can handle the automatic feedback subset.
