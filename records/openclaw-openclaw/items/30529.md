## Summary

Keep open. Current main and the latest shipped release have Discord typing plumbing for automatic reply and heartbeat flows, but they still do not expose the requested public `message(action="typing")` or generic message-tool typing action.

## What This Changes

Request support for Discord typing indicators through the public message tool, preferably with a generic typing/busy action that channels can implement.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then implement Discord discovery, execution, docs, and focused regression coverage while reusing the existing lifecycle safeguards.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that automatic Discord typing exists, but the requested manual/public message-tool action is absent on current main and v2026.5.12.

## Solution Assessment

Unclear until maintainers choose the API shape. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than a Discord-only alias, while automatic lifecycle fixes only cover part of the request.
