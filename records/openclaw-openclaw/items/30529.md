## Summary

Keep open: current main and the latest release have Discord typing primitives and automatic typing paths, but they still do not expose a public message-tool typing action. The remaining work is a public API and lifecycle decision, not conservative cleanup.

## What This Changes

Request adding Discord native typing-indicator support to the message tool so agents can show users that a reply is being prepared.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, wire Discord to the existing typing primitive, and cover lifecycle cleanup, discovery, docs, focused tests, and real Discord proof.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has automatic Discord typing paths but no public message-tool typing action.

## Solution Assessment

Unclear until maintainers choose the API shape. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than a Discord-only alias.
