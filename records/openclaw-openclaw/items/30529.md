## Summary

Keep open: current main has Discord typing primitives plus automatic reply and heartbeat typing, but it still does not expose a public `message(action="typing")` or generic message-tool typing action. The earlier Discord-only PR was closed unmerged, and the remaining work needs a public API and lifecycle decision rather than conservative cleanup.

## What This Changes

Add a Discord or generic message-tool action that lets agents trigger native typing indicators while preparing replies.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, wire Discord execution to the existing typing primitive, and cover discovery, lifecycle cleanup, docs, focused tests, and real Discord proof.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has automatic Discord typing paths but no public message-tool `typing` or generic typing action.

## Solution Assessment

Unclear until maintainers choose the API shape. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks safer than a Discord-only alias.
