## Summary

Keep open. Current main has automatic Discord typing support for inbound reply and heartbeat flows, but the requested public message-tool typing action is still not advertised or executable, and the discussion broadens this into a generic channel API decision.

## What This Changes

Request to add Discord native typing indicator support to the public message tool, either automatically while processing or through a manual typing action.

## Best Possible Solution

Define an additive channel-owned typing or busy-control contract, then wire Discord discovery/execution to the existing typing helpers with lifecycle safeguards, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that automatic Discord typing exists, but the requested public message-tool typing action is absent from the shared action list, Discord discovery, and Discord execution path.

## Solution Assessment

Unclear until maintainers choose the public API. A generic channel-owned typing or busy action that reuses existing lifecycle safeguards looks more maintainable than a Discord-only alias.
