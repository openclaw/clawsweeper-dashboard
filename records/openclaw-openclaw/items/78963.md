## Summary

Keep open: current main still lacks a first-class WhatsApp listen-only/hooks-only mode, and this is a new channel/config contract with privacy and no-outbound safety semantics rather than a narrow bug fix.

## What This Changes

Request a WhatsApp account or route mode that emits inbound `message_received` hooks while skipping agent runs, model calls, replies, read receipts, typing, reactions, and pairing messages.

## Best Possible Solution

Design an explicit WhatsApp-owned ingestion mode that admits configured DMs/groups to hook emission, bypasses agent dispatch and all outbound side effects, and documents validation plus status behavior.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing behavior. Source inspection gives high confidence that current main lacks the requested mode and only exposes WhatsApp hooks inside the normal inbound processing path.

## Solution Assessment

Unclear until maintainers choose the product and security contract. A WhatsApp-owned mode is plausible, but the exact config shape and relationship to the broader early inbound hook work need maintainer design first.
