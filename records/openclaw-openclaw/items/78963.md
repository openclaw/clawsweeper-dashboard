## Summary

Keep open: current main already has an explicit WhatsApp `message_received` hook opt-in, but it still lacks the requested listen-only/hooks-only mode that lets inbound group/DM messages reach hooks while skipping agent/session/model routing and all outbound WhatsApp side effects. The related read-only PRs cover only the no-outbound subset and are not landed complete fixes.

## What This Changes

Request a first-class WhatsApp listen-only/hooks-only mode for inbound messages that emits `message_received` hooks without agent runs, model calls, or WhatsApp outbound activity.

## Best Possible Solution

Define a WhatsApp-owned account or route mode that emits opted-in inbound hooks before conversational routing, skips agent/model/session dispatch, and suppresses all outbound WhatsApp side effects by construction.

## Reproduction Assessment

Yes from source inspection: current main exposes hook opt-in, but the hook path is behind access-control admission and then continues into agent/session dispatch.

## Solution Assessment

No current implementation is sufficient; sendPolicy and read-only outbound gating do not meet the requested no-agent/no-model hooks-only behavior, and maintainers still need to choose the durable WhatsApp config/routing contract.
