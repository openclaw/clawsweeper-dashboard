## Summary

Keep open: the current PR fixes the earlier gateway-contract concern and includes usable real-behavior proof, but it still misses cleanup for stable retry session keys, leaving a stale-context path when configured-model retries are used.

## What This Changes

The branch updates memory-core dreaming narrative session keys/idempotency/pre-cleanup, adjusts memory-core tests, and adds a changelog entry.

## Best Possible Solution

Keep the stable per-workspace/per-phase key and timestamped idempotency direction, but pre-clean every stable attempt session key before reuse, including configured-model retry keys, while preserving the generic gateway cleanup seam.

## Reproduction Assessment

Yes. Current main has a source-level reproduction because `buildNarrativeSessionKey` appends `nowMs`; the remaining PR blocker is also source-reproducible because configured-model retries use stable `-retry-1` keys while the new pre-clean only deletes the base key.

## Solution Assessment

No. The stable-key/idempotency split is the right narrow fix, but the submitted pre-cleanup does not cover all stable keys that the retry path can reuse; moving cleanup to each attempted key or precomputing all possible attempt keys is safer.

## Review Findings
