## Summary

Keep this PR open: it is a focused and useful fix for the memory-core Dreaming session leak, but the latest head still has one narrow warning regression in the new pre-cleanup path.

## What This Changes

The PR changes memory-core Dreaming narrative sessions to use stable workspace/phase session keys, timestamped idempotency keys, per-attempt pre-cleanup, focused tests, and a changelog entry.

## Best Possible Solution

Land the focused memory-core stable-key and pre-cleanup fix after filtering expected request-scoped pre-cleanup failures and rerunning focused validation.

## Reproduction Assessment

Yes, at source level. Current main still embeds `nowMs` in the Dreaming narrative session key, uses that same key for idempotency, and only cleans accepted attempts in final best-effort cleanup.

## Solution Assessment

Yes, with one small adjustment. Stable workspace/phase keys plus timestamped idempotency keys and owner-local attempt pre-cleanup are the narrow maintainable fix, but expected request-scoped pre-cleanup failures should stay quiet while unexpected delete failures still warn.
