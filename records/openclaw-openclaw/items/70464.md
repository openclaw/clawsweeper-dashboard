## What This Changes

The PR stabilizes memory-core Dreaming narrative session keys, moves timestamps into subagent idempotency keys, adds pre-run attempt cleanup, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Land the scoped memory-core fix after quieting expected request-scoped pre-cleanup warnings and triaging exact-head CI, while leaving trajectory/restart residue to the narrower follow-up issues.

## Reproduction Assessment

Yes, at source level. Current main still embeds `nowMs` in the Dreaming narrative session key and reuses that same key as the idempotency key, so repeated sweeps can allocate fresh `dreaming-narrative-*` session keys without a pre-cleanup path.

## Solution Assessment

Yes, with one small adjustment. Stable workspace/phase session keys plus timestamped idempotency keys and owner-local attempt pre-cleanup are the narrow maintainable fix; expected request-scoped pre-cleanup failures should be suppressed before merge.
