## Summary

Keep open: current main still uses timestamped Dreaming narrative session keys and only performs final best-effort cleanup, while this PR implements the right bounded-key/pre-cleanup direction but still has one narrow logging regression to repair before merge.

## What This Changes

The PR changes memory-core Dreaming narrative sessions to use stable workspace/phase session keys, timestamped idempotency keys, per-attempt pre-cleanup, focused tests, and a changelog entry.

## Best Possible Solution

Land the focused memory-core stable-key and pre-cleanup fix after filtering expected request-scoped pre-cleanup warnings, while leaving trajectory and restart residue to the separate follow-up issues.

## Reproduction Assessment

Yes, at source level. Current main still embeds `nowMs` in Dreaming narrative session keys, reuses that key as the idempotency key, and lacks pre-run cleanup before starting the narrative attempt.

## Solution Assessment

Yes, with one small adjustment. Stable workspace/phase session keys plus timestamped idempotency keys and owner-local pre-cleanup are the narrow maintainable fix, but expected request-scoped pre-cleanup failures should stay quiet.
