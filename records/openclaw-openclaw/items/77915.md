  - body: `store-load.ts` does not export `isSessionStoreRecord`; the PR moves that helper into `store-validation.ts` and imports it there locally. This new named import from `./store-load.js` will fail typecheck before the recovery code can build, so import from the actual exporting module or intentionally re-export it.
  - confidence: 0.98
- **[P1] Align the writeTextAtomic backup option names:** `src/infra/json-files.ts:125-132`
  - body: `WriteTextAtomicOptions` declares `backupBeforeOverwrite`, `validateBackupSource`, and `removeStaleBackup`, but the implementation reads `backupSourceDecision` and `validateExistingForBackup` and the caller passes those names. Strict TypeScript will reject both the property reads and the caller object, so the backup path cannot compile as written.
  - confidence: 0.98
