The branch adds session-store recovery from `.bak` and `.tmp` candidates, backup creation during writes, self-heal writeback, validation helpers, and recovery tests.

## Best Possible Solution

Implement recovery with secure `0600` atomic self-heal and ranked tmp scanning without restoring hot-path full-store backup copies, then add the changelog entry and redacted real gateway proof.

## Solution Assessment

No. Recovering from residual tmp files and carefully managed backups is plausible, but this patch is not the narrowest maintainable fix while it adds hot-path backups, insecure self-heal permissions, and pre-sort tmp capping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Avoid copying the session store on every save:** `src/config/sessions/store.ts:524-527`
  - body: Passing `backupSourceDecision` from `writeSessionStoreAtomic` makes every normal save with session entries read and copy the existing `sessions.json` to `.bak` before overwrite. Recent session-store maintenance removed automatic backups and large-store hot-path work, so this reintroduces the multi-MB write cost this area has been reducing.
  - confidence: 0.92
- **[P2] Rank tmp candidates before enforcing the cap:** `src/config/sessions/store-load.ts:75-76`
  - body: The scan stops after the first 50 matching tmp directory entries before sorting by `sourceRank` and `mtimeMs`. Since directory order is not recency-ordered, a newer valid tmp can be ignored while older or invalid tmp files are evaluated, defeating recovery in noisy session directories.
  - confidence: 0.88
- **[P2] Create the self-heal temp file with 0600:** `src/config/sessions/store-load.ts:174-181`
  - body: `selfHealWriteback` opens the tmp path without an explicit mode, then renames it over `sessions.json` before best-effort chmod. On POSIX this can create or expose a wider mode than the session-store `0600` contract if the process umask permits it or chmod fails.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/config/sessions/store-load.ts:37`
  - body: This is a user-facing fix for session metadata loss, but the PR does not update `CHANGELOG.md`. Repo policy requires a changelog bullet for user-facing fixes before merge.
  - confidence: 0.9
