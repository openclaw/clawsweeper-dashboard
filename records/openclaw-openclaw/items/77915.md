## Summary

Keep open: the recovery direction addresses a real current-main data-loss path, but this PR remains unmergeable because the patch has TypeScript API mismatches, a tmp-candidate ordering bug, and no after-fix real behavior proof.

## What This Changes

The PR adds session-store recovery from `.bak`/`.tmp` files, backup creation during atomic writes, self-heal writeback, validation helpers, and recovery regression tests.

## Best Possible Solution

Repair the compile-time API mismatches, rank and cap recovery candidates deterministically, add the required changelog entry, and merge only after redacted real gateway recovery output proves recovered sessions survive restart/save.

## Reproduction Assessment

Yes. Source inspection shows current `loadSessionStore` returns an empty store after final read/parse failure on an existing malformed `sessions.json`; I did not run a mutating repro because this review had to keep the checkout read-only.

## Solution Assessment

No. The recovery approach is plausible and narrow, but this exact patch is not the best merge candidate until the compile blockers, tmp-selection edge case, changelog, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Import the validation helper from the exporting module:** `src/config/sessions/store.ts:25`
  - body: `store-load.ts` does not export `isSessionStoreRecord`; this PR moves that helper into `store-validation.ts` and imports it there locally. The new named import from `./store-load.js` will fail strict TypeScript before the recovery code can build, so import from `./store-validation.js` or intentionally re-export it.
  - confidence: 0.98
- **[P1] Align the writeTextAtomic backup option names:** `src/infra/json-files.ts:125-132`
  - body: `WriteTextAtomicOptions` declares `backupBeforeOverwrite`, `validateBackupSource`, and `removeStaleBackup`, but the added implementation reads `backupSourceDecision` and `validateExistingForBackup`, and the session-store caller passes those undeclared names. Strict TypeScript will reject both sides, so the backup path cannot compile as written.
  - confidence: 0.98
- **[P2] Rank tmp candidates before enforcing the cap:** `src/config/sessions/store-load.ts:75-76`
  - body: The scan stops after the first 50 matching directory entries before sorting by `sourceRank` and `mtimeMs`. `readdirSync` is not ordered by recency, so a valid newer tmp can be ignored while older or invalid tmp files are evaluated, defeating recovery in noisy session directories.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/config/sessions/store-load.ts:37`
  - body: This is a user-facing fix for session metadata loss, but the PR diff does not touch `CHANGELOG.md`. Repo policy requires a changelog bullet for user-facing fixes before merge.
  - confidence: 0.9
