## Summary

Keep open: current main still uses the demand-only manifest-contract lookup that hides persisted/global migration providers, and this PR is a small, correct resolver-level fix pending maintainer acceptance of the persisted metadata behavior.

## What This Changes

The PR removes the manifest-contract resolver's forced `preferPersisted: false` metadata lookup so migration-provider discovery can see persisted/global plugin install records.

## Best Possible Solution

Land this resolver-level fix or an equivalent canonical change so migration-provider runtime lookup and post-install ownership checks both see enabled persisted/global plugin records.

## Reproduction Assessment

Yes. Source inspection shows the standalone migrate path calls the manifest-contract resolver, and current main forces `preferPersisted:false`, which disables persisted install-record reads needed for global managed-npm plugins; the linked issue also has shipped-release user reproduction on 2026.6.6.

## Solution Assessment

Yes. A resolver-level change is the best fix because it covers both direct migration-provider lookup and the post-install migration ownership check; an opt-in call-site fix would leave sibling paths divergent.
