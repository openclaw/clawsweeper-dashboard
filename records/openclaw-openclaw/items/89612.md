## What This Changes

The PR removes the manifest-contract runtime's `preferPersisted: false` metadata snapshot override and updates related plugin tests to expect the default persisted-aware lookup.

## Best Possible Solution

Land this resolver-level persisted-aware metadata lookup fix, then close the linked issue once the fix is on main.

## Reproduction Assessment

Yes. Current main still routes standalone migration-provider discovery through a resolver that opts out of persisted plugin metadata, and the linked issue includes packaged-install failure output for the same path.

## Solution Assessment

Yes. Removing the resolver-level opt-out is the narrowest maintainable fix because direct migration lookup and post-install ownership checks should use the normal persisted-aware metadata snapshot instead of duplicating plugin discovery policy.
