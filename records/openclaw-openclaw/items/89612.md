## What This Changes

The PR removes the manifest-contract resolver’s forced `preferPersisted: false` option and updates test expectations so migration-provider contract lookup can use the normal persisted plugin metadata snapshot.

## Best Possible Solution

Land this resolver-level fix after ordinary maintainer review, keeping migration-provider contract discovery on the canonical persisted metadata snapshot so direct migrate lookup and post-install ownership checks agree.

## Reproduction Assessment

Yes. Source inspection shows the standalone migrate path calls the manifest-contract resolver, current main forces `preferPersisted:false`, and registry loading then disables persisted install-record reads that global plugins need; I did not run a live global npm install in this read-only review.

## Solution Assessment

Yes. Removing the resolver-level opt-out is the narrowest maintainable fix; threading a special option or install-record workaround through migration call sites would duplicate plugin registry policy and risk missing the post-install ownership path.
