## Summary

Keep open. Current main and the latest release support Telegram native exec-approval routing and the broad `/elevated full` break-glass path, but they still do not provide the requested group-scoped trusted-owner/tool-class approval-bypass contract; this is a security-sensitive feature request that needs maintainer product/security review.

## What This Changes

Add an opt-in Telegram group mode that lets explicitly allowlisted owners bypass approval prompts for configured admin tool classes with audit logging.

## Best Possible Solution

Maintainers should either document `/elevated full` as the intended break-glass route or approve a narrow audited Telegram policy scoped by numeric owner ID, allowlisted group/topic, tool class, and destructive-operation rules.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new opt-in approval-bypass capability request. Source and docs inspection confirm the requested config and policy path is absent on current main and in `v2026.5.22`.

## Solution Assessment

Unclear. A scoped audited policy could fit, but a direct boolean bypass is too coarse until maintainers decide how it composes with native approvals, `/elevated full`, tool classification, destructive-operation handling, and audit logging.
