## Summary

Keep open. Current main still has Telegram-native approval routing and the broader `/elevated full` no-approval path, but it does not implement the requested Telegram group owner bypass policy with sender, group, tool-class, and audit scoping.

## What This Changes

Add an opt-in Telegram group policy that lets explicitly allowlisted owners bypass approvals for selected action classes with audit logging.

## Best Possible Solution

Maintainers should choose between documenting the existing `/elevated full` escape hatch or adding a narrow audited Telegram/core approval policy scoped by sender, group, and tool class.

## Reproduction Assessment

Not applicable as a bug reproduction: the request is for a new opt-in bypass mode. Source inspection does confirm the current shape: allowed elevated `on`/`ask` preserves approvals, while only broader `full` or `ask=off` suppresses them.

## Solution Assessment

Unclear. The requested direction is plausible, but the best policy shape needs product and security review because the current no-approval path is broader than the requested Telegram owner/group/tool-class behavior.
