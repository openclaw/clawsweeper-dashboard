## Summary

Keep open. Current main and the latest release support Telegram native exec approval delivery and broad no-approval routes such as `/elevated full`, but they still do not provide a group-scoped trusted-owner/tool-class approval-bypass contract; this remains a security-sensitive feature request needing maintainer product and security review.

## What This Changes

The issue requests an opt-in Telegram group mode that lets explicitly allowlisted owners bypass approval prompts for configured host-execution tool classes, with audit and destructive-operation safeguards.

## Best Possible Solution

Maintainers should either document `/elevated full`/YOLO policy as the intended break-glass route or approve a narrow audited Telegram policy scoped by numeric owner ID, allowlisted group/topic, tool class, destructive-operation rules, and existing approval semantics.
