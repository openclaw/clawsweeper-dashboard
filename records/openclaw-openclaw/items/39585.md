## Summary

Keep open. Current main and v2026.5.22 still provide Telegram native exec approval delivery and broad no-approval routes like `/elevated full`, but not a Telegram group-scoped trusted-owner/tool-class approval-bypass contract; this remains security-sensitive product/API work.

Request an opt-in Telegram group policy that lets explicitly allowlisted owners bypass approval prompts for selected tool classes with audit and destructive-operation safeguards.

## Best Possible Solution

Maintainers should either document existing `/elevated full`/YOLO policy as the intended break-glass route or approve a narrow audited Telegram policy scoped by numeric owner ID, allowlisted group/topic, tool class, destructive-operation handling, and existing approval semantics.
