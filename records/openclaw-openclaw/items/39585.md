## Summary

The issue requests an opt-in Telegram trusted-owner mode that can bypass approval-gated runtime/filesystem actions in allowlisted groups for selected tool classes with audit logging.

## Best Possible Solution

Design this as a maintainer-approved shared approval/elevated policy extension with explicit owner identity, group allowlist, tool-class and destructive-op gates, audit semantics, docs, and Telegram regression proof.

## Solution Assessment
