## Summary

Keep open. Current main and the latest shipped release support Telegram exec approver routing plus broad `/elevated full`, but they do not have a group-scoped trusted-owner/tool-class approval bypass; the requested behavior changes an execution approval security boundary and needs maintainer product/security review.

## What This Changes

The issue requests an opt-in Telegram group mode where explicitly allowlisted owners in allowlisted groups can bypass approval prompts for configured action classes, with audit logging and destructive-operation safeguards.

## Best Possible Solution

Maintainers should either document `/elevated full` as the intended break-glass route or approve a narrow audited Telegram policy scoped by numeric owner ID, allowlisted group/topic, tool class, destructive-operation rules, and audit metadata.
