## Summary

Keep open. Current main supports Telegram-native approval routing and the broader `/elevated full` escape hatch, but it does not provide the requested sender/group/tool-class approval-bypass policy, and adding one is a security/product decision rather than conservative cleanup.

## What This Changes

Request an opt-in Telegram group mode that lets explicitly allowlisted owners bypass approval-gated admin actions with scoped tool classes and audit logging.

## Best Possible Solution

Maintainers should choose either a narrow, audited Telegram/core approval policy with explicit sender, group, and tool-class scope, or document that `/elevated full` and `ask=off` are the supported break-glass path.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does confirm the reported shape: Telegram owner identity feeds approval authorization, while exec approvals are skipped only by `full`/`ask=off` behavior, not by a scoped trusted-owner group policy.

## Solution Assessment

Unclear. The requested direction is plausible, but the existing `/elevated full` path is broader than the proposed owner/group/tool-class policy, so maintainers need to decide the safest supported behavior before implementation.
