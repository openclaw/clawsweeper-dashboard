## Summary

Keep open. Current main and v2026.5.18 still expose Telegram exec approval routing through explicit approvers or command-owner IDs, plus broad `/elevated full`; the requested trusted-owner, group-scoped, tool-class approval bypass remains a security-sensitive product/config feature.

## What This Changes

Request an opt-in Telegram group mode that lets explicitly trusted owners bypass approval prompts for configured action classes with audit and safety constraints.

## Best Possible Solution

Maintainers should either document elevated-full as the intended break-glass path, or approve a narrow audited Telegram policy scoped by numeric sender ID, allowlisted group/topic, tool class, and destructive-operation rules.

## Solution Assessment

Unclear. The direction is plausible, but a direct boolean bypass is not the best safe shape until maintainers decide how it interacts with elevated-full mode, approval routing, tool-class metadata, and destructive-operation gates.
