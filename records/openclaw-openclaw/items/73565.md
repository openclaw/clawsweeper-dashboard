## Summary

Keep open: current main still has direct transcript JSONL write paths that bypass the existing redaction guard, and open PR #73563 is the unmerged closing candidate for this security-sensitive sink family.

## What This Changes

The issue requests applying the existing transcript redaction guard to bare session transcript append paths before message payloads are persisted.

## Best Possible Solution

Review and land PR #73563, or an equivalent security patch, so every bare transcript sink reuses the existing guard while preserving idempotency, parentId chaining, update behavior, and redaction configuration semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by writing secret-like content through appendExactAssistantMessageToSessionTranscript, CLI text-turn persistence, gateway inject, or the Codex transcript mirror with transcript redaction enabled, then inspecting the persisted JSONL; I did not execute it because this review is read-only.

## Solution Assessment

Yes for the proposed direction. Reusing guardSessionManager is the narrow maintainable fix because it applies the existing before-write hook and config-aware transcript redaction instead of adding parallel redaction logic.

## Security Review

Status: needs_attention

Summary: Current main can still persist sensitive transcript content outside the config-aware redaction guard.

Concerns:

- **[medium] Guard raw transcript helper writes before persistence:** `src/config/sessions/transcript-append.ts:235`
  - body: appendSessionTranscriptMessage writes params.message directly into the transcript entry, so any caller outside guardSessionManager bypasses transcript redaction.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw helper and emits the same message for inline updates without a redaction pass.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:205`
  - body: persistTextTurnTranscript appends user prompts and assistant replies through the raw helper, bypassing the guarded before-write redaction path.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are appended and emitted from messageBody through appendSessionTranscriptMessage without using the guarded redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:57`
  - body: The Codex transcript mirror runs the hook path manually and appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
