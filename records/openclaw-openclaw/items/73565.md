## Summary

Keep open: current main still has unguarded transcript JSONL append sinks, and the linked security fix PR is open but unmerged.

## What This Changes

Apply the existing transcript redaction guard to bare session transcript append paths before messages are persisted or emitted.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved security patch that routes each bare transcript append through the existing guarded redaction path while preserving idempotency, parentId handling, update emission, and redaction-off semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to persist secret-like text through the exact assistant append, CLI text-turn persistence, gateway injected transcript append, or Codex mirror with transcript redaction enabled, then inspect the JSONL sink.

## Solution Assessment

Yes. Reusing the existing guardSessionManager/redaction hook at the bare append entry points is the narrow maintainable direction because it centralizes hook handling and config-aware redaction instead of adding parallel redaction logic.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can store sensitive message text without the config-aware redaction gate.

Concerns:

- **[medium] Guard raw transcript helper writes:** `src/config/sessions/transcript-append.ts:235`
  - body: appendSessionTranscriptMessage stores params.message directly in the JSONL entry, so callers that reach this helper without a guarded SessionManager bypass transcript redaction.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw append helper and emits the same message for inline updates.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:205`
  - body: persistTextTurnTranscript appends user and assistant text turns directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without the guarded redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:57`
  - body: The Codex mirror manually runs the hook path and appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
  - confidence: 0.86
