## Summary

Keep open: current main still has bare transcript append paths that bypass the documented transcript redaction guard, and the linked closing PR #73563 remains open/unmerged.

## What This Changes

Route session transcript appendMessage persistence paths through the existing redaction guard so secret-like message payloads are masked before JSONL transcript writes.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved security patch that routes each bare transcript append through the existing guard while preserving idempotency, parentId chaining, transcript update emission, and redaction-off semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows secret-like content passed through exact assistant append, CLI text-turn persistence, gateway inject, or the Codex mirror can reach appendSessionTranscriptMessage, which writes params.message directly to JSONL.

## Solution Assessment

Yes. Reusing guardSessionManager or the same config-aware redaction hook at the bare append entry points is the narrow maintainable fix because it centralizes hook handling and avoids parallel redaction logic.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can write sensitive message text to disk without the config-aware redaction gate.

Concerns:

- **[medium] Guard raw transcript helper writes:** `src/config/sessions/transcript-append.ts:235`
  - body: appendSessionTranscriptMessage stores params.message directly in the JSONL entry, so callers that reach this helper without a guarded SessionManager bypass transcript redaction.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw append helper and emits the same message for inline transcript updates without applying the guarded redaction path.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:206`
  - body: persistTextTurnTranscript appends user prompt text and assistant reply text directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without the guarded redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:44`
  - body: The Codex mirror manually runs before_message_write and appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
  - confidence: 0.86
