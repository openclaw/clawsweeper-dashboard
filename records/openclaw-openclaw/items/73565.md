## Summary

Keep open: current main still has direct transcript append sinks that bypass the config-aware guard redaction path, and PR #73563 is open and unmerged with closing syntax for this issue. The issue should remain open until that PR or an equivalent maintainer-approved fix lands.

## What This Changes

The issue asks for session transcript `appendMessage` persistence paths to pass through the standard redaction guard before message payloads are written to disk.

## Best Possible Solution

Review and land PR #73563 or an equivalent maintainer-approved fix that wraps each bare transcript append sink in the existing guard, threads OpenClawConfig wherever redaction mode matters, preserves idempotency and transcript update semantics, and adds regression coverage for both redaction and `redactSensitive="off"`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by sending secret-like content through the config transcript append, CLI text-turn transcript, gateway inject helper, or Codex mirror while transcript redaction is enabled, then inspecting the persisted JSONL entries.

## Solution Assessment

Yes. Reusing guardSessionManager is the narrowest maintainable direction because it already centralizes before-write hooks, transcript redaction, and tool-result persistence guarding instead of adding a parallel scrubber.
