## Summary

Keep open: current main still has session transcript write paths that append message payloads without the config-aware guard redaction path, and the linked implementation PR is open and unmerged.

## What This Changes

The issue asks for core session, CLI, gateway inject, and bundled Codex mirror transcript append paths to pass through the existing redaction guard before JSONL persistence.

## Best Possible Solution

Review and land PR #73563 or an equivalent maintainer-approved fix that wraps each bare transcript append sink in the existing guard, threads OpenClawConfig where redaction mode matters, preserves idempotency/update behavior, and adds focused regression coverage for both redaction and redactSensitive="off".

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by writing secret-like content through the config transcript append, CLI text-turn transcript persistence, gateway inject helper, or Codex mirror with transcript redaction enabled, then inspecting the persisted JSONL.

## Solution Assessment

Yes. Reusing guardSessionManager is the narrowest maintainable direction because it already centralizes before-write hooks, transcript redaction, tool-result guarding, and transcript update behavior instead of adding a separate scrubber.

Status: needs_attention
