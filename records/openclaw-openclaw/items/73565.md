## Summary

Keep open: current main still has bare transcript append paths that bypass the config-aware transcript redaction guard, and the linked closing PR #73563 is open and unmerged.

## What This Changes

The issue asks for existing transcript redaction guards to cover bare session transcript append paths in core session management, CLI execution, gateway injection, and the bundled Codex mirror.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved security patch that routes every bare transcript append in this sink family through the existing config-aware guard while preserving idempotency, parentId handling, update emission, and redaction-off semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send secret-like text through the exact assistant append, CLI text-turn persistence, gateway inject, or Codex mirror path with transcript redaction expected, and current main reaches the JSONL append helper without applying the config-aware redaction guard.

## Solution Assessment

Yes. Reusing guardSessionManager or the same config-aware redaction hook at the bare append entry points is the narrow maintainable fix because it keeps transcript hook, redaction, idempotency, and update behavior centralized.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can write sensitive message text to disk without the config-aware redaction gate.

Concerns:

- **[medium] Guard raw transcript helper writes:** `src/config/sessions/transcript-append.ts:235`
  - body: appendSessionTranscriptMessage stores params.message directly in the JSONL entry, so callers that reach this helper without a guarded SessionManager bypass transcript redaction.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw append helper and emits the same message for inline transcript updates without applying the guarded redaction path.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:205`
  - body: persistTextTurnTranscript appends user prompt text and assistant reply text directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without the guarded redaction path.
  - confidence: 0.88

  - body: The Codex mirror manually runs the hook path and appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
  - confidence: 0.86
