## Summary

Keep open: current main still has unguarded transcript JSONL sinks despite docs promising persisted transcript text redaction, and open PR #73563 is the linked unmerged closing candidate.

## What This Changes

Ensure session transcript append paths pass through the existing redaction guard before JSONL persistence.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved security patch that routes each bare transcript append through the existing guard while preserving parentId, idempotency, update emission, and redaction-off semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to write a secret-like token through the exact assistant append, CLI text-turn persistence, gateway inject helper, or Codex mirror with transcript redaction enabled, then inspect the persisted JSONL; I did not execute it because this review is read-only.

## Solution Assessment

Yes. Reusing guardSessionManager at the bare transcript entry points is the narrow maintainable direction because it uses the existing before-write hook, config-aware redaction, and tool-result guard instead of creating parallel redaction logic.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can store sensitive message text without config-aware redaction.

Concerns:

- **[medium] Guard raw transcript helper writes before persistence:** `src/config/sessions/transcript-append.ts:235`
  - body: appendSessionTranscriptMessage stores params.message directly in the JSONL entry; callers that reach this helper without a guarded SessionManager bypass redactTranscriptMessage.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw append helper and emits the same unredacted message for inline updates.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:205`
  - body: persistTextTurnTranscript appends user and assistant text turns directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without the guarded redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:57`
  - body: The Codex mirror manually runs the hook path and appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
  - confidence: 0.86
