## Summary

Keep open: current main still has unguarded session transcript append paths, and the linked PR is an open implementation candidate rather than a merged fix.

## What This Changes

The issue requests applying the existing transcript redaction guard infrastructure to bare `appendMessage` write paths in core session management, CLI attempt execution, gateway inject, and the Codex mirror.

## Best Possible Solution

Review and land PR #73563 or an equivalent maintainer-approved fix that wraps each bare transcript append sink in `guardSessionManager`, threads config where redaction mode matters, preserves idempotency and transcript update semantics, and adds regression coverage for redaction plus `redactSensitive: "off"` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by sending secret-like content through the config transcript append, CLI text-turn transcript, gateway inject helper, or Codex mirror while transcript redaction is enabled; current main shows direct append calls on those paths.

## Solution Assessment

Yes. Reusing `guardSessionManager` is the narrowest maintainable solution because it applies the existing before-write hook, transcript redaction, and tool-result guard infrastructure instead of creating a parallel scrubber.
