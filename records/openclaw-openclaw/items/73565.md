## Summary

Keep open: current main still has transcript JSONL sinks that bypass the existing config-aware redaction guard, and open PR #73563 is the linked unmerged closing candidate.

## What This Changes

The issue requests applying the existing transcript redaction guard to core session management, CLI attempt execution, gateway inject, and Codex mirror append paths before persistence.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved security patch that applies the existing guard before every bare transcript append while preserving idempotency, parentId behavior, update emissions, and redaction-off semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: write a secret-like text value through the exact assistant append, CLI text-turn persistence, gateway inject helper, or Codex mirror with transcript redaction enabled, then inspect the persisted transcript JSONL for the unredacted value.

## Solution Assessment

Yes. Reusing the existing guarded SessionManager path is the narrow maintainable fix because it centralizes before-write hooks, transcript redaction, and tool-result guard behavior instead of duplicating redaction at each sink.
