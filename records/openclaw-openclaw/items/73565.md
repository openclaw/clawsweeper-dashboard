## Summary

Keep open: current main still has direct session transcript append paths that can write message text to JSONL without the redaction gate, and the closing fix PR at https://github.com/openclaw/openclaw/pull/73563 is still open in the provided live context.

## What This Changes

The issue asks for session transcript persistence to redact sensitive message content before JSONL disk writes across all append call sites.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73563, or an equivalent security patch that centralizes config-aware transcript redaction at the shared write layer while preserving redaction-off semantics, idempotency, parentId handling, and transcript update emission.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass secret-like text through a direct append caller such as CLI text-turn persistence or gateway transcript injection, and current main reaches the JSONL append helper that stores the message unchanged.

## Solution Assessment

Yes. Centralizing the same config-aware redaction at appendSessionTranscriptMessage is the narrow maintainable fix because it covers current and future direct append callers without duplicating per-surface guard logic.
