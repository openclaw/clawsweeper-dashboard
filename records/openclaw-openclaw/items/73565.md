## Summary

Keep open: current main still has unguarded transcript JSONL sinks, and PR #73563 is the open, unmerged closing candidate for this security-sensitive path.

## What This Changes

Apply the existing session transcript redaction guard to bare appendMessage/write paths in core session management, CLI attempt execution, gateway inject, and the bundled Codex mirror.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved security patch, so every bare transcript sink uses the existing guard while preserving parentId, idempotency, update emission, and `redactSensitive: "off"` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by writing secret-like content through the exact assistant append, CLI text-turn persistence, gateway inject helper, or Codex mirror with transcript redaction enabled, then inspecting the persisted JSONL; I did not execute it because this review is read-only.

## Solution Assessment

Yes. Applying the existing guard infrastructure at the bare entry points is the narrow maintainable fix because it reuses the current before-write hook, config-aware redaction, and tool-result guard instead of adding parallel redaction logic.

## Security Review

Status: needs_attention

Summary: Current main can still persist sensitive transcript text to disk outside the config-aware redaction guard.

Concerns:

- **[medium] Guard raw transcript helper writes before persistence:** `src/config/sessions/transcript-append.ts:235`
  - body: `appendSessionTranscriptMessage` stores `params.message` directly in the transcript JSONL entry; callers that reach this helper without a guarded SessionManager bypass `redactTranscriptMessage`.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: `appendExactAssistantMessageToSessionTranscript` calls the raw append helper and emits the same unredacted message for inline updates.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:205`
  - body: `persistTextTurnTranscript` appends user and assistant text turns directly through `appendSessionTranscriptMessage`, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are written and emitted from `messageBody` without the guarded redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:57`
  - body: The Codex mirror manually runs the hook path and appends directly, so it misses `guardSessionManager`'s config-aware transcript redaction step.
  - confidence: 0.86
