## What This Changes

The issue asks for session transcript append paths to pass message payloads through the standard sensitive-data redaction gate before persistence.

## Best Possible Solution

## Solution Assessment

Yes, with maintainer review of the active PR. Centralizing the redaction at `appendSessionTranscriptMessage` and returning/emitting the redacted appended message is the narrow maintainable fix; the current main partial redaction is not enough to close this issue.
