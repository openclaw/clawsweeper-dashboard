## Summary

Keep open: current main still has transcript JSONL write paths that can persist message payloads without the config-aware transcript redaction gate, and the linked closing PR is still open and unmerged.

## What This Changes

The issue asks for core session transcript append paths to pass through the existing redaction guard before message payloads are written to disk.

## Best Possible Solution

Land PR #73563 or an equivalent maintainer-approved patch that routes every bare transcript sink through the existing guard, threads config where redaction policy matters, preserves idempotency/update behavior, and adds regression coverage for redaction and redactSensitive="off".

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by writing secret-like text through config transcript append, CLI text-turn persistence, gateway inject, or the Codex mirror with transcript redaction enabled, then inspecting the persisted JSONL.

## Solution Assessment

Yes. The existing guardSessionManager/redactTranscriptMessage path is the narrowest maintainable fix because it already centralizes before-write hooks, config-aware text redaction, and tool-result persistence safety.

## Security Review

Status: needs_attention

Summary: Current main still has concrete transcript persistence paths that can write sensitive message text to disk without the redaction guard.

Concerns:

- **[medium] Guard raw transcript appends before persistence:** `src/config/sessions/transcript-append.ts:230`
  - body: appendSessionTranscriptMessage writes params.message directly into the JSONL entry, so any caller that reaches this helper without prior guard redaction can persist secret-bearing text.
  - confidence: 0.94
- **[medium] Guard config transcript appends:** `src/config/sessions/transcript.ts:283`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw helper directly, allowing assistant content from delivery mirrors to bypass config-aware transcript redaction.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:201`
  - body: persistTextTurnTranscript writes user and assistant text turns directly through appendSessionTranscriptMessage, so CLI transcript text can bypass the redaction gate.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:104`
  - body: Gateway-injected assistant messages are written through the raw helper and emitted from the same message body without guardSessionManager redaction.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:54`
  - body: The Codex mirror runs before-message-write hooks manually but does not apply redactTranscriptMessage with OpenClawConfig before appending mirrored messages.
  - confidence: 0.86
