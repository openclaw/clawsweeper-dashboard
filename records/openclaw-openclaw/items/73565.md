## Summary

Keep open: current main still has session transcript JSONL write paths that can persist message payloads without the config-aware redaction guard, and the linked open PR is the implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Request to route session transcript persistence append paths through the existing redaction guard before JSONL writes.

## Best Possible Solution

Land PR #73563 or an equivalent maintainer-approved patch that uses guardSessionManager at each bare transcript sink while preserving parentId, idempotency, update-mode, and redactSensitive="off" behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by writing secret-like content through the config transcript append, CLI text-turn persistence, gateway inject helper, or Codex mirror with default transcript redaction enabled, then inspecting the persisted JSONL.

## Solution Assessment

Yes. Reusing guardSessionManager is the narrow maintainable solution because it already centralizes before-message-write hooks, config-aware redaction, and transcript guard behavior.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can write sensitive message text to disk without the config-aware redaction guard.

Concerns:

- **[medium] Guard raw transcript helper callers before persistence:** `src/config/sessions/transcript-append.ts:230`
  - body: appendSessionTranscriptMessage writes the caller-provided message into the transcript JSONL entry directly, so callers must route through the existing guard before reaching this helper.
  - confidence: 0.93
- **[medium] Guard config transcript appends:** `src/config/sessions/transcript.ts:283`
  - body: appendExactAssistantMessageToSessionTranscript calls appendSessionTranscriptMessage directly, allowing assistant content to reach disk without redactTranscriptMessage.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:203`
  - body: persistTextTurnTranscript appends user and assistant text turns directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:104`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without guardSessionManager, leaving injected text outside the redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:54`
  - body: The Codex mirror manually runs only the hook path and then appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
  - confidence: 0.86
