## Summary

Keep open: current main still has source-visible transcript append paths that persist message payloads without the existing config-aware redaction guard, and the linked closing PR is still open rather than merged.

## What This Changes

This issue asks for existing transcript redaction guards to cover bare session transcript append paths in core session helpers, CLI attempt execution, gateway injection, and the bundled Codex mirror.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved patch that routes each bare append entry point through the existing guarded redaction path while preserving idempotency, parentId handling, update emission, and redaction-off semantics.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: persist secret-like text through the exact assistant append, CLI text-turn persistence, gateway injected transcript append, or Codex mirror, then inspect the JSONL transcript sink.

## Solution Assessment

Yes. Reusing guardSessionManager or an equivalent existing guard at the bare append entry points is the narrow maintainable fix because it preserves the established hook and config-aware redaction contract instead of adding parallel redaction logic.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can write sensitive message text to disk without the config-aware redaction gate.

Concerns:

- **[medium] Guard raw transcript helper writes:** `src/config/sessions/transcript-append.ts:235`
  - body: appendSessionTranscriptMessage stores params.message directly in the JSONL entry, so any caller that reaches this helper without a guarded SessionManager bypasses transcript redaction.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw append helper and emits the same message payload for inline updates.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:205`
  - body: persistTextTurnTranscript appends user prompt text and assistant reply text directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without the guarded redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:44`
  - body: The Codex mirror manually runs the hook path and appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
  - confidence: 0.86
