## Summary

Keep open: current main still has bare transcript append paths that can bypass the existing transcript redaction guard, and the linked closing PR #73563 is open and unmerged.

## What This Changes

The issue requests applying the existing session transcript redaction guard to bare append paths so sensitive message payloads are not persisted to JSONL unredacted.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved security patch that routes each bare transcript append through the existing guard without changing transcript semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass secret-like content through the exact assistant append, CLI text-turn persistence, gateway inject helper, or Codex mirror while transcript redaction is expected, and current main reaches appendSessionTranscriptMessage without the config-aware redaction guard first.

## Solution Assessment

Yes. Reusing guardSessionManager or the same config-aware before-write redaction hook at the bare append entry points is the narrow maintainable fix; this issue should remain open until the linked PR or equivalent patch merges.
