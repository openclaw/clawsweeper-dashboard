## Summary

Keep open: current main still has direct transcript append sinks that bypass the config-aware guard redaction path, and the linked closing PR remains open and unmerged. The existing ClawSweeper comment correctly identified the active implementation candidate, so the issue should stay open until PR #73563 or an equivalent maintainer-approved fix lands.

## What This Changes

The issue requests that session transcript append paths pass through the standard redaction guard before persisting message payloads.

## Best Possible Solution

Review and land PR #73563 or an equivalent fix that wraps each bare transcript append sink in the existing guard, threads OpenClawConfig wherever redactSensitive matters, preserves idempotency/update event behavior, and adds regression coverage for both redaction and redactSensitive="off".

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by sending secret-like content through the config transcript append, CLI text-turn transcript, gateway inject helper, or Codex mirror while transcript redaction is enabled, then inspecting the persisted JSONL entries.

## Solution Assessment

Yes. Reusing guardSessionManager is the narrowest maintainable direction because it already centralizes before-write hooks, transcript redaction, and tool-result persistence guarding instead of adding a parallel scrubber.

## Security Review

Status: needs_attention

Summary: Current main still has transcript disk-write paths that can bypass config-aware redaction before JSONL persistence.

Concerns:

- **[medium] Guard config transcript appends before disk persistence:** `src/config/sessions/transcript.ts:280`
  - body: appendExactAssistantMessageToSessionTranscript opens a bare SessionManager and appends message payloads without the existing guard redaction path.
  - confidence: 0.91
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:211`
  - body: persistTextTurnTranscript appends user and assistant text turns through a bare SessionManager, so configured transcript redaction does not run before those messages reach disk.
  - confidence: 0.88
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: appendInjectedAssistantMessageToTranscript appends injected assistant content directly, bypassing the config-aware guard used by normal agent transcript persistence.
  - confidence: 0.87
- **[medium] Use config-aware redaction in the Codex mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:58`
  - body: The Codex mirror manually runs the harness hook and then appends directly, so it does not apply guardSessionManager's config-aware redactTranscriptMessage step.
  - confidence: 0.86
