## Summary

Keep this open. Current main still has direct session transcript append sinks that bypass the existing config-aware guard redaction path, and the provided GitHub context shows open PR #73563 is the paired implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests that all session transcript `appendMessage` persistence paths pass through the existing redaction guard before JSONL transcript writes reach disk.

## Best Possible Solution

Land PR #73563 or an equivalent fix that wraps the config transcript append, CLI text-turn persistence, gateway inject helper, and Codex mirror in the existing config-aware guard path while preserving idempotency, parent-chain behavior, update emission semantics, and `logging.redactSensitive="off"` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by writing secret-like text through any of the named bare append paths with transcript redaction enabled, then inspecting the persisted JSONL for the unredacted value.

## Solution Assessment

Yes. Reusing the existing `guardSessionManager` path is the narrowest maintainable solution because it already centralizes before-write hooks, transcript redaction, tool-result guarding, and transcript update behavior.

## Security Review

Status: needs_attention

Summary: Current main still has transcript disk-write paths that can bypass config-aware redaction before JSONL persistence.

Concerns:

- **[medium] Guard config transcript appends before disk persistence:** `src/config/sessions/transcript.ts:279`
  - body: `appendExactAssistantMessageToSessionTranscript` opens a bare `SessionManager` and appends assistant messages without the existing guard redaction path, so configured transcript redaction is skipped.
  - confidence: 0.92
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:201`
  - body: `persistTextTurnTranscript` appends user and assistant turns through a bare `SessionManager`, so configured redaction does not run before those messages reach disk.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:179`
  - body: The gateway inject helper can raw-append or directly append injected assistant content without `guardSessionManager`, leaving injected transcript text outside the redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:44`
  - body: The Codex mirror manually runs the hook path and appends directly, so it misses `guardSessionManager`'s config-aware `redactTranscriptMessage` step.
  - confidence: 0.86
