## Summary

Keep open: current `main` still has transcript JSONL append paths that can persist message payloads without the config-aware transcript redaction guard, and open PR #73563 is the unmerged closing implementation candidate.

## What This Changes

The issue requests redaction-guard coverage for session transcript append paths that can persist message payloads to JSONL.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved patch, that routes each bare transcript sink through the existing guard while preserving parentId, idempotency, update emission, and `redactSensitive: "off"` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by writing secret-like content through the config transcript append, CLI text-turn persistence, gateway inject helper, or Codex mirror with transcript redaction enabled, then inspecting the persisted JSONL.

## Solution Assessment

Yes. The suggested guard-based direction is the narrow maintainable fix because `guardSessionManager` already centralizes before-write hooks, config-aware transcript redaction, tool-result guarding, and update behavior.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can write sensitive message text to JSONL without the config-aware redaction guard.

Concerns:

- **[medium] Guard raw transcript helper writes before disk persistence:** `src/config/sessions/transcript-append.ts:230`
  - body: `appendSessionTranscriptMessage` writes `params.message` into the transcript JSONL entry directly; callers that reach this helper without a guarded `SessionManager` bypass `redactTranscriptMessage`.
  - confidence: 0.94
- **[medium] Guard config transcript appends:** `src/config/sessions/transcript.ts:283`
  - body: `appendExactAssistantMessageToSessionTranscript` calls the raw append helper directly, so assistant content can reach disk and inline update events without the existing config-aware redaction step.
  - confidence: 0.91
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:201`
  - body: `persistTextTurnTranscript` appends user and assistant text turns directly through `appendSessionTranscriptMessage`, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:104`
  - body: Gateway-injected assistant messages are written and emitted from `messageBody` without `guardSessionManager`, leaving injected transcript text outside the redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:54`
  - body: The Codex mirror manually runs the hook path and appends directly, so it misses `guardSessionManager`'s config-aware `redactTranscriptMessage` step.
  - confidence: 0.86
