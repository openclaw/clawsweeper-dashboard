## Summary

Keep open: current main still has transcript JSONL append paths that bypass the config-aware redaction guard, and the linked closing PR #73563 is still open and unmerged.

## What This Changes

The issue requests routing core session transcript append paths in session helpers, CLI execution, gateway inject, and the Codex mirror through the existing transcript redaction guard.

## Best Possible Solution

Review and land #73563, or an equivalent maintainer-approved patch that routes each bare transcript append through the existing config-aware guard while preserving idempotency, parentId handling, update emission, and redaction-off semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows secret-like user or assistant text can reach appendSessionTranscriptMessage through exact assistant, CLI text-turn, gateway inject, and Codex mirror paths without guardSessionManager's config-aware redaction before JSONL write.

## Solution Assessment

Yes. Reusing guardSessionManager or the same config-aware before-write hook at these bare append entry points is the narrowest maintainable fix, and #73563 is the relevant implementation to review.

## Security Review

Status: needs_attention

Summary: The issue itself reports a concrete security-sensitive persistence gap, and source inspection confirms unguarded transcript write paths remain on current main.

Concerns:

- **[medium] Guard raw transcript helper writes:** `src/config/sessions/transcript-append.ts:235`
  - body: appendSessionTranscriptMessage stores params.message directly in the JSONL entry, so callers that reach this helper without a guarded SessionManager bypass transcript redaction.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:287`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw append helper and emits the same message for inline transcript updates without applying the guarded redaction path.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:206`
  - body: persistTextTurnTranscript appends user prompt text and assistant reply text directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:106`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without the guarded redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:60`
  - body: The Codex mirror manually runs only the before-message-write hook helper and appends directly, so it misses guardSessionManager's config-aware transcript redaction step.
  - confidence: 0.86
