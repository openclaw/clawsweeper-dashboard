## Summary

Keep open: current main still has direct transcript JSONL append paths that bypass the config-aware transcript redaction guard, and linked PR #73563 is the open unmerged implementation candidate.

## What This Changes

The issue asks for existing session transcript redaction guards to be applied to bare appendMessage persistence paths in core, CLI, Gateway, and the bundled Codex mirror.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved patch, so every bare transcript sink uses the existing guard while preserving parentId, idempotency, update emission, and redactSensitive="off" behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: write secret-like text through the direct helper paths with transcript redaction enabled, then inspect the persisted JSONL and emitted inline update payloads.

## Solution Assessment

Yes. The guard-based direction is the narrow maintainable fix because guardSessionManager already centralizes before-write hooks, config-aware transcript redaction, tool-result guarding, and transcript update behavior.

## Security Review

Status: needs_attention

Summary: Current main can persist sensitive transcript message text through bare append paths without the config-aware redaction gate.

Concerns:

- **[medium] Guard raw transcript helper callers before disk writes:** `src/config/sessions/transcript-append.ts:230`
  - body: appendSessionTranscriptMessage writes params.message directly, so callers that reach it without guardSessionManager can persist unredacted message content to JSONL.
  - confidence: 0.94
- **[medium] Guard exact assistant transcript appends:** `src/config/sessions/transcript.ts:283`
  - body: appendExactAssistantMessageToSessionTranscript calls the raw append helper and emits the same message, leaving delivery mirror content outside the existing config-aware redaction path.
  - confidence: 0.91
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:201`
  - body: persistTextTurnTranscript writes prompt and assistant reply text directly through appendSessionTranscriptMessage, bypassing transcript redaction before persistence.
  - confidence: 0.9
- **[medium] Guard Gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:104`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without guardSessionManager, so injected transcript text can miss redaction.
  - confidence: 0.88
- **[medium] Use config-aware redaction in the Codex mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:54`
  - body: The Codex mirror manually runs before-message hooks and appends directly, but does not apply guardSessionManager's config-aware redactTranscriptMessage step.
  - confidence: 0.86
