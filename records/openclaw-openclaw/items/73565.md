## Summary

Keep open: current main still has transcript JSONL write paths that pass message payloads directly to disk without the config-aware guard redaction path, and PR #73563 is an open, unmerged closing implementation candidate.

## What This Changes

Apply the existing transcript redaction guard to session transcript persistence append paths before message payloads are written to disk.

## Best Possible Solution

Review and land PR #73563, or an equivalent maintainer-approved patch, that routes each bare transcript sink through the existing guard, preserves update/idempotency behavior, and adds regression coverage for redaction plus redactSensitive="off".

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by writing secret-like content through the config transcript append, CLI text-turn persistence, gateway inject helper, or Codex mirror with transcript redaction enabled, then inspecting the persisted JSONL.

## Solution Assessment

Yes. The suggested direction is the narrow maintainable fix because guardSessionManager already centralizes before-write hooks, config-aware redaction, tool-result guarding, and transcript update behavior.

## Security Review

Status: needs_attention

Summary: Current main still has transcript persistence paths that can write sensitive message text to JSONL without the config-aware redaction guard.

Concerns:

- **[medium] Guard transcript helper writes before disk persistence:** `src/config/sessions/transcript-append.ts:230`
  - body: appendSessionTranscriptMessage writes params.message into the transcript JSONL entry directly; callers that use this helper without a guarded SessionManager bypass redactTranscriptMessage.
  - confidence: 0.93
- **[medium] Guard config transcript appends:** `src/config/sessions/transcript.ts:283`
  - body: appendExactAssistantMessageToSessionTranscript calls appendSessionTranscriptMessage directly, so assistant content can reach disk without the existing redaction guard.
  - confidence: 0.9
- **[medium] Guard CLI text-turn transcript writes:** `src/agents/command/attempt-execution.ts:203`
  - body: persistTextTurnTranscript appends user and assistant text turns directly through appendSessionTranscriptMessage, bypassing config-aware redaction before persistence.
  - confidence: 0.9
- **[medium] Guard gateway injected transcript writes:** `src/gateway/server-methods/chat-transcript-inject.ts:104`
  - body: Gateway-injected assistant messages are written and emitted from messageBody without guardSessionManager, leaving injected transcript text outside the redaction path.
  - confidence: 0.88
- **[medium] Use config-aware redaction in Codex transcript mirror:** `extensions/codex/src/app-server/transcript-mirror.ts:54`
  - body: The Codex mirror manually runs the hook path and appends directly, so it misses guardSessionManager's config-aware redactTranscriptMessage step.
  - confidence: 0.86
