  - confidence: 0.87

  - body: This fallback path renders any `action.type === "command"` value even when the button is marked `disabled`. The presentation contract says channels without disabled-control support degrade disabled buttons to non-interactive fallback text, so a disabled approval button should not become a copyable `/approve ...` command or trigger manual-command guidance. Skip command rendering for disabled buttons and add coverage for the shared fallback plus Feishu guidance gate.
  - confidence: 0.84
