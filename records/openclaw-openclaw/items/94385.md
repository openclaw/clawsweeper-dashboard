## Summary

Keep open: this is a useful candidate fix for the Feishu approval fallback issue, but the current patch still misses normal core presentation delivery because Feishu checks for visible commands after core has stripped `presentation` from the rendered payload.

## What This Changes

The PR changes shared message-presentation fallback rendering to show typed command button values, adds Feishu document-comment guidance, exports a helper through the plugin SDK, and updates Feishu/core tests and docs.

## Best Possible Solution

Keep the typed-command fallback and privacy direction, but preserve or compute the Feishu comment guidance signal before core strips `presentation`, then lock that path with a core-delivery regression.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send a typed-command `MessagePresentation` through core delivery to a Feishu comment target, where `renderPresentationForDelivery` strips `presentation` before Feishu `sendPayload` checks for rendered commands.

## Solution Assessment

No. The shared fallback helper is the right layer for command text, but Feishu guidance still runs too late; the safer fix is to derive or preserve the visible-command signal during presentation rendering.
