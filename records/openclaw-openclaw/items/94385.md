## Summary

Keep open. The latest branch fixed the disabled-button and callback/privacy parts, but Feishu comment guidance is still computed after normal core presentation rendering strips `presentation`, so the guidance can be omitted on the core delivery path this PR is meant to improve.

## What This Changes

This PR changes shared message-presentation fallback rendering to show typed command button values, adds Feishu document-comment manual-command guidance, exports a shared visible-command predicate, and updates Feishu/core tests and docs.

## Best Possible Solution

Keep the shared typed-command fallback direction, but derive or preserve the Feishu visible-command guidance signal during presentation rendering so normal core delivery to comment targets gets the same guidance as the direct send path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send a typed-command `MessagePresentation` through core delivery to a Feishu comment target, where `renderPresentationForDelivery` strips `presentation` before `sendPayload` checks for rendered commands.

## Solution Assessment

No. The shared fallback helper is the right layer for command text, but the Feishu guidance still runs too late; the safer fix is to derive or preserve the visible-command signal during presentation rendering.
