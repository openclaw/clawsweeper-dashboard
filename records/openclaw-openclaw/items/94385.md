## Summary

Keep open. This PR is a useful candidate for the linked Feishu approval fallback bug, but the patch is not merge-ready because it exposes typed opaque callback values as manual fallback text and can add Feishu “buttons unavailable” guidance to non-interactive presentations.

## What This Changes

The PR changes shared presentation fallback text to include button control values and adds Feishu document-comment guidance when interactive payloads degrade to text.

## Best Possible Solution

Keep the shared fallback repair, but render only manual-executable command values, keep typed callbacks private, and make Feishu document-comment guidance depend on actual unavailable interactive controls.

## Reproduction Assessment

Yes. Source inspection of current main and the existing Feishu outbound test show document-comment delivery strips interactive metadata and currently turns an approval button with `/approve req_1` into label-only fallback text.

## Solution Assessment

No. The shared fallback layer is the right place for the command fallback, but this implementation is too broad because it renders typed opaque callbacks as manual text and adds Feishu guidance for non-interactive presentations.
