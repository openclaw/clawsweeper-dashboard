## Summary

This is the same remaining WebChat transcript persistence bug already tracked by #76804, with #76819 open as the canonical fix path; the extra screenshots and 2026.5.2 report should be preserved by linking this issue there.

## What This Changes

Request to keep previously visible WebChat assistant final text visible after later user messages, especially for tool-using turns on OpenClaw 2026.5.2.

## Best Possible Solution

Keep #76804 as the canonical tracker and review or land #76819, or an equivalent gateway transcript persistence fix, so tool-using assistant final text is durable before WebChat reloads history.

## Reproduction Assessment

Yes, source-reproducible. A WebChat agent-run turn can stream final assistant text, then the current tool-final path reloads chat.history from a transcript path whose agent-run branch does not append that final text.

## Solution Assessment

Yes. Closing this duplicate in favor of #76804/#76819 is the best cleanup because the canonical fix targets gateway transcript persistence, which is the durable source of the disappearing UI state.
