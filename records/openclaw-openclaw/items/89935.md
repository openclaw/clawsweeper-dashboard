This PR is useful but should stay open because the QQBot outbound media path still logs and returns unredacted direct-upload error messages, which can contain the same signed or credential-bearing URLs the PR is trying to protect.

## What This Changes

The PR adds plugin-local URL redaction helpers and tests for QQBot, Tlon, and Feishu media log/error paths, plus HTML escaping for the Canvas missing-file page.

## Best Possible Solution

Keep the plugin-local redaction approach, but apply `redactTextForLog` to QQBot direct-upload error messages before they are logged or returned, with focused tests for image, voice, video, and document direct-upload failure paths.

## Reproduction Assessment

## Solution Assessment

No. The helper shape is appropriate, but the PR is too narrow until the same redaction boundary is applied to direct-upload failure messages in the touched QQBot outbound media module.
