Full review comments:

- **[P1] Keep the optional chart plugin out of bundled core:** `extensions/chart/openclaw.plugin.json:4`
  - body: The current project direction routes optional plugin capability to ClawHub/external packages unless a missing core API is shown. Adding this as an enabled-by-default bundled plugin creates a new default tool/config surface without the required product/security reason.
  - confidence: 0.94
- **[P1] Use the direct tool parameter signature:** `extensions/chart/src/chart-tool.ts:30-33`
  - body: Direct `api.registerTool` tools receive `(toolCallId, params, ...)`, but this implementation treats the first argument as the params object. Runtime calls would parse the tool call id string instead of the chart arguments.
  - confidence: 0.96
- **[P1] Call `imageResultFromFile` with its object parameter:** `extensions/chart/src/chart-tool.ts:52`
  - body: The current SDK takes `imageResultFromFile({ label, path, ... })`; passing `(fp, "image/svg+xml")` does not match the API and would fail type checking or runtime execution.
  - confidence: 0.96
- **[P1] Validate or escape SVG attribute values:** `extensions/chart/src/render-svg.ts:16`
  - body: The SVG helper writes attribute values verbatim, and `series.color` flows into fill/stroke attributes. A crafted color can break out of the attribute in an SVG returned to users, so colors need strict validation or attributes need escaping.
  - confidence: 0.88
