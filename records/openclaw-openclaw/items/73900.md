## Summary

Keep open. Current main still preserves the reported routing bug, and open PR #73968 already references this issue as the narrow fix candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Ensure main-session cron systemEvent heartbeat wakes deliver reminder replies to the bound session route instead of inheriting global heartbeat.to.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that removes inherited explicit destination fields for cron-forced target=last wakes while preserving normal heartbeat defaults and adding regression coverage for topic-bound routes.

## Reproduction Assessment

Yes. The issue includes a concrete config, cron payload, session key, and log evidence, and the current source path statically reproduces the merge plus explicit-destination precedence without live Telegram credentials.

## Solution Assessment

Yes. Clearing inherited explicit destination fields in the cron-forced target=last path is the narrow maintainable solution; changing all heartbeat target=last semantics would be broader and risk intentional explicit heartbeat delivery.
