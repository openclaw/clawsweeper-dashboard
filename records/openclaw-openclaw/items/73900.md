## Summary

Keep open. Current main still preserves inherited global `heartbeat.to` when cron forces `heartbeat.target = "last"`, so a session-bound `systemEvent` wake can still resolve delivery through the global DM target; PR #73968 is the linked narrow fix candidate.

## What This Changes

A cron `sessionTarget: "main"` `systemEvent` heartbeat wake should deliver replies to the bound session route instead of inheriting a global `heartbeat.to` destination.

## Best Possible Solution

Review and land PR #73968 or an equivalent narrow core fix that strips inherited explicit destination fields for cron-forced `target: "last"`, with regression coverage for a global DM default plus a bound Telegram topic session.

## Reproduction Assessment

Yes. The issue provides concrete config and redacted logs, and the current source statically reproduces the route: cron passes only `target: "last"`, the gateway merge retains `heartbeat.to`, and outbound resolution treats it as explicit before session fallback.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced `target: "last"` wakes is the narrow maintainable fix; changing all heartbeat `target: "last"` semantics would be broader than this bug requires.
