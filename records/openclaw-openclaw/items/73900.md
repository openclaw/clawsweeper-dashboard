## Summary

Keep open. Current main still shallow-merges global heartbeat defaults into cron-forced `target: "last"` wakes and delivery resolution still treats inherited `heartbeat.to` as explicit, while open PR #73968 is already the linked fix candidate.

## What This Changes

Ensure main-session `systemEvent` cron wakes bound to a session route ignore inherited global heartbeat destination fields and reply to the originating chat or topic.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced `target: "last"` main-session `systemEvent` wakes, with regression coverage for a global DM default plus a group/topic session route.

## Reproduction Assessment

Yes. The report provides concrete config, cron payload, session key, session-store route, and logs, and current source statically reproduces the merge plus explicit-destination precedence without live Telegram credentials.

## Solution Assessment

Yes. Clearing inherited explicit destination fields in the cron-forced `target: "last"` path is the narrow maintainable fix; changing all heartbeat `target: "last"` semantics would be broader and risk intentional explicit delivery behavior.
