## Summary

Keep open. Current main still has the cron-to-heartbeat merge path that can retain a global `heartbeat.to`, and open PR #73968 is already linked as the narrow fix candidate.

## What This Changes

Session-bound `main`/`systemEvent` cron heartbeat wakes should route replies back to the bound Telegram forum topic instead of inheriting a global DM `heartbeat.to`.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced `target: "last"` wakes, with regression coverage for a global DM default plus a bound group/topic route.

## Reproduction Assessment

Yes. The issue includes concrete config, cron payload, session key, and logs, and current source statically reproduces the merge plus explicit-destination precedence without a live Telegram run.

## Solution Assessment

Yes. Clearing inherited explicit destination fields in the cron-forced `target: "last"` path is the narrow maintainable fix; changing all heartbeat `target: "last"` semantics would be broader and risk breaking intentional explicit heartbeat delivery.
