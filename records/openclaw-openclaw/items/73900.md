## Summary

Keep open. Current main still has the reported cron-to-heartbeat routing path, and the provided GitHub context shows open closing PR #73968 targets this exact root cause but has not merged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that main-session cron systemEvent heartbeat wakes can inherit global heartbeat destination defaults and route a Telegram topic reminder to a DM instead of the bound topic session.

## Best Possible Solution

Review and land #73968, or an equivalent narrow fix, so cron-forced target="last" systemEvent wakes do not inherit explicit heartbeat.to/accountId defaults and regression coverage proves a topic-bound session stays on its group/topic route even when global heartbeat defaults point at a DM.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and logs, and current main statically reproduces the same merge plus explicit-destination precedence without needing a live Telegram run.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced target="last" wakes is the narrowest maintainable direction; changing general heartbeat target="last" semantics would be broader and riskier.
