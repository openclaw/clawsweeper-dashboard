## Summary

Keep open. Current main still has the reported cron-to-heartbeat routing path, and the linked closing PR is open but unmerged, so this issue should remain open until that fix or an equivalent one lands.

## What This Changes

The issue reports that a main-session cron systemEvent wake bound to a Telegram forum topic can inherit global heartbeat.to and send the reminder reply to a DM instead of the bound topic.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced target="last" main-session systemEvent wakes, with regression coverage proving a topic-bound session keeps its group/topic route when global heartbeat defaults include a DM to.

## Reproduction Assessment

Yes. The issue gives concrete config, cron payload, session key, and logs, and current source statically reproduces the merge plus explicit-destination precedence without needing live Telegram delivery.

## Solution Assessment

Yes. Clearing inherited explicit destination fields for this cron-forced target="last" path is the narrow maintainable fix; changing all heartbeat target="last" semantics would be broader and risk existing explicit heartbeat routing behavior.
