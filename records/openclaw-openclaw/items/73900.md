## Summary

Keep open. Current main still retains the cron-to-heartbeat routing bug: main-session systemEvent wakes pass only `target: "last"`, the gateway shallow-merge keeps inherited `heartbeat.to`, and delivery resolution treats that value as an explicit destination. Open PR #73968 is already linked as the narrow fix candidate, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

A topic-bound main-session systemEvent cron heartbeat should reply to the bound session route instead of inheriting global `agents.defaults.heartbeat.to`.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced `target: "last"` main-session systemEvent wakes, with regression coverage for global DM defaults plus a group/topic session route.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and logs, and current source statically reproduces the merge plus explicit-destination precedence without live Telegram credentials.

## Solution Assessment

Yes, for the linked repair direction. Clearing inherited `to` and account fields only in the cron-forced `target: "last"` path is narrower than changing all heartbeat `target: "last"` semantics.
