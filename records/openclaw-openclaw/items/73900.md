## Summary

Keep open. Current main still shallow-merges cron-supplied `heartbeat: { target: "last" }` with global heartbeat defaults, and the linked PR #73968 remains open and unmerged as the narrow fix candidate.

## What This Changes

The issue reports that a main-session cron `systemEvent` wake can inherit global heartbeat destination fields and send a topic-bound Telegram reminder to a DM.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that clears inherited explicit destination fields for cron-forced `target: "last"` main-session systemEvent wakes, with regression coverage for a global DM default plus a group/topic session route.

## Reproduction Assessment

Yes. The issue gives concrete config, cron payload, session key, and logs, and the current source statically reproduces the merge plus explicit-destination precedence without live Telegram credentials.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only in the cron-forced `target: "last"` path is narrower than changing all heartbeat `target: "last"` semantics, which could break intentional explicit heartbeat delivery.
