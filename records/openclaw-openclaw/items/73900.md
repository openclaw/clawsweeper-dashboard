## Summary

Keep open. Current main still retains the reported cron-to-heartbeat routing bug, and open PR #73968 is already linked as the narrow fix candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

A session-bound main/systemEvent cron heartbeat wake should reply to the originating session route instead of inheriting a global heartbeat destination.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced `target: "last"` main-session systemEvent wakes, with regression coverage for global DM defaults plus a group/topic session route.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and logs, and current source statically reproduces the merge plus explicit-destination precedence without live Telegram credentials.

## Solution Assessment

Yes. Clearing inherited explicit destination fields in the cron-forced `target: "last"` path is the narrow maintainable fix; changing all heartbeat `target: "last"` semantics would be broader and could break intentional explicit heartbeat delivery.
