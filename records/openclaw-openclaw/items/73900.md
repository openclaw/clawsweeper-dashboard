## Summary

Keep open. Current main still shallow-merges cron-forced `heartbeat.target = "last"` with global heartbeat defaults, so an inherited `heartbeat.to` can still beat the session-bound route; the open closing PR #73968 is the right item to review before closing this issue.

## What This Changes

The issue reports that main-session cron `systemEvent` heartbeat wakes can inherit a global heartbeat DM destination and deliver a topic-bound reminder to the wrong Telegram context.

## Best Possible Solution

Land PR #73968 or an equivalent narrow core fix that drops inherited explicit destination fields for cron-forced `target: "last"` wakes while preserving non-routing heartbeat defaults and adding regression coverage for a global DM default plus a bound Telegram topic session.

## Reproduction Assessment

Yes. The issue provides concrete config, payload, session key, and redacted logs, and current source statically reproduces the merge plus explicit-destination precedence without needing a live Telegram run.

## Solution Assessment

Yes. Sanitizing only the cron-forced `target: "last"` override after default merging is the narrow maintainable fix; changing all heartbeat `target: "last"` semantics would be broader than this bug requires.
