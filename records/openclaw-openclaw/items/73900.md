## Summary

Keep open. Current main still preserves inherited global `heartbeat.to` when cron forces `heartbeat.target = "last"`, so the report remains source-reproducible; open PR #73968 is the linked fix candidate and should be reviewed or landed before this issue closes.

## What This Changes

Session-bound cron `main`/`systemEvent` heartbeat wakes should route replies through the bound session instead of inheriting a global heartbeat DM target.

## Best Possible Solution

Land PR #73968 or an equivalent narrow core fix that strips inherited explicit destination fields for cron-forced `target: "last"`, with regression coverage for a global DM default plus a bound Telegram topic session.

## Reproduction Assessment

Yes. Source inspection on current main follows the reported path: cron passes only `target: "last"`, the gateway merge retains inherited `heartbeat.to`, and outbound target resolution uses that explicit destination before session fallback.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced `target: "last"` wakes is the narrowest maintainable fix; changing all heartbeat `target: "last"` semantics would be broader than this bug requires.
