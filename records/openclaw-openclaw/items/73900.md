## Summary

Keep open. Current main still matches the reported cron-to-heartbeat merge and explicit-destination precedence chain, and open PR #73968 is already the narrow linked fix candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The report asks for main-session cron systemEvent heartbeat wakes to route back to the bound Telegram topic/session instead of inheriting a global heartbeat.to DM target.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced target:last main-session systemEvent wakes, with regression coverage for global DM defaults plus a group/topic session route.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and logs, and current source statically reproduces the merge plus explicit-destination precedence without needing live Telegram access.

## Solution Assessment

Yes. Clearing inherited explicit destination fields in the cron-forced target:last path is the narrow maintainable fix; changing all heartbeat target:last semantics would be broader and risk intentional explicit heartbeat delivery.
