## Summary

Keep open. Current `origin/main` still preserves the reported routing path, and the provided GitHub context shows open closing PR #73968 targets the same root cause but has not merged.

## What This Changes

A main-session cron `systemEvent` wake bound to a Telegram forum topic should not inherit a global heartbeat DM destination when cron forces `heartbeat.target = "last"`.

## Best Possible Solution

Land #73968 or an equivalent narrow fix that clears inherited explicit destination fields for cron-forced `target: "last"` main-session `systemEvent` wakes, with regression coverage proving a topic-bound session stays on its group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and observed logs, and the current-main source path statically reproduces the merge and explicit-destination precedence without a live Telegram run.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced `target: "last"` wakes is the narrowest maintainable direction; changing general heartbeat `target: "last"` semantics would be broader and riskier.
