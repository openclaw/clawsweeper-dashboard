## Summary

Keep open. Current main still shallow-merges cron heartbeat defaults with the cron-forced `target: "last"` override, so an inherited global `heartbeat.to` can still override the bound topic route; PR #73968 is open as the linked narrow fix.

## What This Changes

A cron `sessionTarget: "main"` systemEvent wake should route heartbeat replies to the bound session instead of inheriting global heartbeat destination fields.

## Best Possible Solution

Review and land PR #73968 or an equivalent narrow core fix that drops inherited explicit destination fields for cron-forced `target: "last"`, with regression coverage for a global DM default plus a bound Telegram topic route.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and redacted logs, and current source statically reproduces the merge plus explicit-destination precedence without needing a live Telegram run.

## Solution Assessment

Yes. Clearing inherited explicit destination fields in the cron-forced `target: "last"` path is the narrow maintainable solution; changing all heartbeat `target: "last"` semantics would be broader than this bug requires.
