## Summary

Keep open. Current main still has the reported cron-to-heartbeat routing path, and the open closing PR that targets this root cause is not merged, so the issue should remain open until that fix or an equivalent one lands.

## What This Changes

The issue asks for cron `sessionTarget: "main"` systemEvent heartbeat wakes to route replies back to the bound session/topic instead of inheriting a global `heartbeat.to` destination.

## Best Possible Solution

Land PR #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced `target: "last"` main-session systemEvent wakes, with regression coverage proving a topic-bound session keeps its group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue gives concrete config, cron payload, session key, and logs, and current source statically reproduces the merge plus explicit-destination precedence without live Telegram credentials.

## Solution Assessment

Yes. Clearing inherited explicit destination fields for this cron-forced `target: "last"` path is the narrow maintainable fix; changing all heartbeat `target: "last"` semantics would be broader and risk existing explicit heartbeat routing behavior.
