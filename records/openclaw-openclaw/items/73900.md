## Summary

Keep open. Current main still shallow-merges cron-forced `heartbeat.target="last"` with global heartbeat defaults, so inherited `heartbeat.to` can override the bound session route; open PR #73968 is already the narrow fix candidate.

## What This Changes

Report asks `sessionTarget: "main"` cron `systemEvent` heartbeat wakes to deliver back to the bound Telegram topic session instead of inheriting a global DM `heartbeat.to`.

## Best Possible Solution

Review and land PR #73968 or an equivalent narrow change that strips inherited explicit destination fields for cron-forced `target: "last"` wakes, with regression coverage for a global DM default plus a bound topic route.

## Reproduction Assessment

Yes. The issue includes concrete config, cron payload, session key, and logs, and current source statically shows the target-only cron override, shallow default merge, and explicit `heartbeat.to` precedence.

## Solution Assessment

Yes. The linked narrow fix direction is the maintainable path because it targets only cron-forced `target: "last"` inheritance instead of changing general heartbeat delivery semantics.
