## Summary

Keep open. Current main still has the reported routing path: main-session cron systemEvent wakes force `heartbeat.target = "last"`, the gateway cron adapter shallow-merges that override with heartbeat defaults, and outbound heartbeat resolution treats inherited `heartbeat.to` as an explicit destination before falling back to the bound session route. Open closing PR #73968 targets this root cause, but it is still open and unmerged in the provided context.

## What This Changes

The issue reports that Telegram topic-bound main-session cron systemEvent heartbeats can inherit global `agents.defaults.heartbeat.to` and deliver reminder replies to a DM instead of the originating topic route.

## Best Possible Solution

Review and land #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced `target: "last"` main-session systemEvent wakes, with regression coverage proving a topic-bound session keeps its group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and logs, and current-main source statically reproduces the merge and explicit-destination precedence without requiring live Telegram credentials.

## Solution Assessment

Yes. Clearing inherited explicit destination fields for this cron-forced `target: "last"` path is the narrowest maintainable direction; changing general heartbeat `target: "last"` semantics would be broader and risk existing explicit heartbeat routes.
