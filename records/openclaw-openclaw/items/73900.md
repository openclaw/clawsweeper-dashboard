## Summary

Keep open. Current main still has the reported routing path, and the issue is paired with open closing PR #73968, so the correct maintainer action is to review, land, replace, or close that PR rather than close this issue early.

## What This Changes

The issue asks for main-session cron systemEvent heartbeat wakes to route back to the bound session/topic instead of inheriting a global heartbeat DM destination.

## Best Possible Solution

Land #73968 or an equivalent narrow fix that strips inherited explicit destination fields for cron-forced `target: "last"` main-session systemEvent wakes, with regression coverage proving a topic-bound session keeps its group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and logs, and current-main source statically reproduces the merge and explicit-destination precedence without needing live Telegram credentials.

## Solution Assessment

Yes. Clearing inherited explicit destination fields for this cron-forced `target: "last"` path is the narrowest maintainable fix; changing general heartbeat `target: "last"` semantics would be broader and risk existing explicit heartbeat routes.
