## Summary

Keep open. Current main still shallow-merges cron-forced `heartbeat.target = "last"` with global heartbeat defaults, so an inherited `heartbeat.to` can still override the bound session route; open PR #73968 is the linked narrow fix and should be reviewed before this issue closes.

## What This Changes

Ensure main-session cron `systemEvent` heartbeat wakes bound to a Telegram topic do not inherit a global DM `heartbeat.to` destination.

## Best Possible Solution

Review and land PR #73968 or an equivalent narrow core fix that removes inherited explicit destination fields for cron-forced `target: "last"`, with regression coverage for a global DM default plus a bound topic route.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and redacted logs, and the current source statically reproduces the shallow merge plus explicit-destination precedence.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only in the cron-forced `target: "last"` path is narrower and safer than changing all heartbeat `target: "last"` semantics.
