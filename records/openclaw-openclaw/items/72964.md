## Summary

Keep open. Current main has adjacent compaction/memory fixes, but the reported cluster is still source-reproducible: cache-inclusive provider usage can still feed context-engine compaction pressure, large empty preparations can still commit fallback summaries, and reply handling still runs preflight compaction before memory flush.

## What This Changes

The issue asks OpenClaw to prevent cache-read token anomalies from triggering destructive compaction, avoid empty fallback summaries for large real sessions, and ensure memory flush runs before compaction can discard context.

## Best Possible Solution

Land a focused reliability fix that uses trusted prompt/envelope snapshots as the compaction authority, treats anomalous provider cache usage defensively, blocks large empty-preparation fallback summaries, and prevents compaction from committing before the current-cycle memory flush is satisfied or intentionally skipped.

## Reproduction Assessment

Yes. The issue provides concrete production traces, and current main has a source-level reproduction path through inflated lastCallUsage.cacheRead, an empty compaction preparation with large tokensBefore, and preflight compaction running before memory flush.

## Solution Assessment

Partly. The requested guardrails are directionally right, but simply dropping cacheRead would regress #66520; the safer fix is prompt-snapshot/envelope-based accounting plus explicit fallback and memory-flush ordering guards.
