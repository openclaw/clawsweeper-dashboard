## Summary

Keep open. Current main still has the core reported gaps: prompt/context token derivation includes cacheRead/cacheWrite, large no-real-message preparations still return a structured fallback instead of escalating, and the reply path still checks preflight compaction before memory flush. The adjacent open PR #66716 addresses the inverse cached-token issue for #66520 and is not a merged fix for this issue.

## What This Changes

The issue requests compaction reliability fixes so prompt-cache usage does not cause premature compaction, empty fallback summaries are not emitted for large real sessions, and pre-compaction memory flush runs before destructive compaction.

## Best Possible Solution

Implement a focused agents/compaction reliability fix that uses a provider-normalized prompt footprint or explicit prompt snapshot for compaction pressure, prevents large empty-fallback compactions from discarding context, and gates destructive compaction on a completed memory-flush cycle when memory flush is enabled.
