## Summary

Keep open. Current main still has session/agent sandbox placement plus sandbox tool allow/deny filtering, but not a per-tool host-versus-sandbox placement contract; the remaining work needs product and security design rather than cleanup closure.

## What This Changes

The issue requests a schema-backed way to route selected tools such as `web_fetch` and `browser` into sandbox execution while keeping trusted tools host-routed in the same session.

## Best Possible Solution

Add a documented core tool-placement contract that can route selected tools to sandbox execution and selected trusted tools to host execution within one session, with tests proving placement remains separate from filtering and elevated exec.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs checks show current main supports session/agent sandbox modes plus tool allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the design. The capability likely belongs in core sandbox/tool routing, but the schema, runtime boundary, and mixed execution security model need product and security review first.
