## Summary

Keep open. Current main and the latest release still have session-level sandbox placement plus sandbox tool filtering and exec elevation, but no reviewed per-tool host-versus-sandbox placement contract; the latest discussion also shows the desired contract is still an active product/security decision.

## What This Changes

The issue requests a new tool-level sandbox placement policy so selected tools can always run in a sandbox while other tools stay host-routed in the same session.

## Best Possible Solution

Define one reviewed core placement contract for mixed host/sandbox execution, keep it separate from sandbox tool filtering and exec elevation, then document and test the chosen schema/runtime boundary before implementation.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source and docs checks show current main and `v2026.5.20` support session/agent sandbox modes plus sandbox allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the contract. The capability likely belongs in core sandbox/tool routing, but elevated is exec-only and the schema/plugin/runtime security boundary needs explicit product and security review first.
