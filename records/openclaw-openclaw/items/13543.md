## Summary

Keep open: current main and latest release still have session/agent-level sandboxing, sandbox tool allow/deny filtering, and exec-only elevated escape behavior, but not a schema-backed per-tool execution placement policy.

## What This Changes

The issue requests a new tool-level sandbox placement mode so selected tools such as `web_fetch` and `browser` can run sandboxed while trusted tools stay host-routed in the same session.

## Best Possible Solution

Define one documented core placement contract with explicit precedence between sandbox mode, tool policy, tool/plugin-declared gateway execution, browser and web_fetch behavior, and elevated exec.

## Reproduction Assessment

Not applicable as a bug reproduction: this issue asks for a new execution-placement capability. Source, docs, and latest-release inspection confirm current OpenClaw does not have that per-tool placement mode.

## Solution Assessment

No, not as the proposed schema alone. The security goal is aligned, but maintainers should first choose the placement owner and precedence model before committing to an `always`/`never` map or tool-declared gateway contract.
