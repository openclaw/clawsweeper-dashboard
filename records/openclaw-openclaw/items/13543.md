## Summary

Keep open: current main still lacks a schema-backed per-tool host-vs-sandbox placement policy, and the related merged Docker sandbox work only unblocked prerequisites rather than implementing selective routing.

## What This Changes

Request a per-tool sandbox placement policy so web_fetch and browser can run in Docker while trusted tools remain host-routed in the same session.

## Best Possible Solution

Add a documented typed tool-placement contract evaluated where tool identity and execution context are both available, with regression coverage separating placement from filtering, browser targets, plugin tools, web_fetch providers, and elevated exec.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection on current main gives high-confidence evidence that the requested capability is missing: sandbox status resolves per session and sandbox tool policy filters availability rather than choosing placement per tool.

## Solution Assessment

Unclear whether the proposed JSON shape is the best contract. The direction is valid, but maintainers should choose the schema, plugin/tool context boundary, and mixed host/sandbox enforcement model before implementation.
