## Summary

Keep open. Current main and v2026.5.12 still provide session/agent-level sandboxing plus sandbox tool allow/deny filtering, but not a schema-backed per-tool placement policy for mixed host and sandbox execution in one session.

## What This Changes

Request configurable per-tool sandbox placement so web_fetch/browser can run in a sandbox while exec/message/cron/gateway remain host-routed in the same session.

## Best Possible Solution

Add a documented core tool-placement contract that can route selected tools to sandbox and selected trusted tools to host execution within one session, with regression coverage proving it is separate from sandbox tool filtering and elevated exec.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection confirms current main only has session/agent-level sandboxing plus allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the design. The capability belongs in core sandbox/tool routing, but the maintainable path should define a new placement contract rather than overloading tools.sandbox.tools or tools.elevated.
