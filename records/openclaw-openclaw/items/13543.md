## Summary

Keep open. Current main still has session/agent sandbox modes plus sandbox tool allow/deny filtering, but no schema-backed per-tool host-versus-sandbox placement policy; the live discussion and labels also route this as product/security design work rather than stale cleanup.

## What This Changes

The issue requests a schema-backed per-tool sandbox placement policy so selected tools can run in sandboxed execution while trusted tools remain host-routed within the same session.

## Best Possible Solution

Add a documented, schema-backed core tool-placement contract that routes selected tools to sandbox execution and selected trusted tools to host execution within one session, with regression coverage separating placement from filtering and elevated exec.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source, docs, and tests show current main only supports session/agent sandbox modes plus tool allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the design: the capability belongs in core sandbox/tool routing, but the schema, tool context boundary, and mixed execution security model need product and security review first.
