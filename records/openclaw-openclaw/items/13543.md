## Summary

Keep open. Current main still provides session/agent-level sandbox placement, sandbox tool allow/deny filtering, sandbox browser routing, and exec elevated escape behavior, but not a reviewed per-tool host-versus-sandbox placement contract; the latest discussion also shows the desired contract is still an active product/security decision.

## What This Changes

The issue requests per-tool sandbox placement so tools such as `web_fetch` and `browser` can be forced into a sandbox while tools such as `exec`, `message`, `cron`, and `gateway` remain host-routed in the same session.

## Best Possible Solution

Define one reviewed core placement contract for mixed host/sandbox execution, keeping it separate from tool filtering and elevated exec, then document and test the chosen schema/runtime boundary before implementation.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source and docs checks show current main supports session/agent sandbox modes plus sandbox allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the contract. The capability likely belongs in core sandbox/tool routing, but elevated is exec-only and the schema/plugin/runtime security boundary needs explicit product and security review first.
