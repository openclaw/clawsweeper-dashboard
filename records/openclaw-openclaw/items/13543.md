## Summary

Keep open. Current main still lacks a schema-backed per-tool host-vs-sandbox placement policy; the existing sandbox mode, sandbox tool filtering, browser host-control setting, and elevated exec escape hatch do not satisfy the requested same-session split.

## What This Changes

Request configurable per-tool sandbox placement so untrusted tools such as `web_fetch` and `browser` can be forced into sandbox while trusted tools remain host-routed.

## Best Possible Solution

Add a documented, typed tool-placement contract evaluated where tool identity and execution context are both available, with tests proving placement stays separate from filtering, browser target overrides, plugin tool contexts, and elevated exec.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection of current main confirms the missing capability: sandboxing resolves per session/agent, and sandbox tool policy only filters availability.

## Solution Assessment

Unclear whether the proposed JSON is the best contract. The direction is valid, but maintainers should choose the schema and security boundary before implementation.
