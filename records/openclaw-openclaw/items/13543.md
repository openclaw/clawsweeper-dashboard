## Summary

Keep open. Current main and the shipped v2026.5.20 release still expose session/agent sandbox placement, sandbox tool filtering, and exec elevation, but not the requested reviewed per-tool host-versus-sandbox placement contract; the latest discussion also shows the contract is still an active product/security decision.

## What This Changes

Add a per-tool sandbox placement configuration so selected tools such as `web_fetch` and `browser` can run in a sandbox while trusted tools remain on the gateway in the same session.

## Best Possible Solution

Define one reviewed core placement contract for mixed host/sandbox execution, keep it separate from sandbox tool filtering and exec elevation, then document and test the chosen schema/runtime boundary before implementation.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source and release checks show current main and `v2026.5.20` support session/agent sandbox modes plus sandbox allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the contract. The capability likely belongs in core sandbox/tool routing, but elevated is exec-only and the schema/plugin/runtime security boundary needs explicit product and security review first.
