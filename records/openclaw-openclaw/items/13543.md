## Summary

Keep open. Current main still lacks schema-backed per-tool sandbox placement; the available controls are session/agent sandbox modes, sandbox tool allow/deny filtering, browser sandbox settings, and elevated exec escape behavior, while the related Docker work only partially unblocked this request.

## What This Changes

Add per-tool sandbox placement so selected tools can run sandboxed while others remain host-routed in the same session.

## Best Possible Solution

Add a documented core tool-placement contract that can route selected tools to a sandbox and selected trusted tools to host execution within one session, with tests proving it is separate from tool filtering and elevated exec.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection confirms current main only provides session/agent-level sandboxing plus allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the design: the capability belongs in core sandbox/tool routing, but the exact schema and runtime boundary should be decided before implementation. Reusing tools.sandbox.tools or elevated exec as placement would be the wrong direction.
