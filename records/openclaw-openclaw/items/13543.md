## Summary

Keep open. Current main still has session-level sandbox placement plus sandbox tool allow/deny filtering, but no schema-backed per-tool host-versus-sandbox placement policy; the latest discussion also raises a product/security design choice around sandbox-by-default plus explicit gateway-only tools.

## What This Changes

Add configurable tool-level sandbox placement so selected untrusted tools can run isolated while trusted tools continue on the host in the same session.

## Best Possible Solution

Have maintainers choose and document a core placement contract, likely separating sandbox execution placement from tool filtering and elevated exec, with explicit handling for gateway-only tools and plugin-declared gateway execution.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs checks show current main supports session sandbox modes plus sandbox tool filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the placement model. Elevated exec is documented as exec-only and does not solve browser/web_fetch placement, while the proposed selective schema may need to become a sandbox-by-default plus explicit gateway-only contract instead.
