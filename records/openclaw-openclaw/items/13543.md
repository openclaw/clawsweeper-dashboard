## Summary

Keep this issue open: current main has session/agent sandboxing, sandbox-only tool filtering, sandbox-aware browser/web_fetch behavior, and exec host routing, but not the requested schema-backed per-tool placement contract for mixed host/sandbox execution in one session.

## What This Changes

Request a configurable per-tool sandbox placement policy so tools such as web_fetch and browser can be forced into a sandbox while trusted tools such as exec, message, cron, and gateway remain host-routed.

## Best Possible Solution

Define and document a core tool-placement contract with schema, precedence, plugin tool context, and tests that separate placement from sandbox filtering and elevated exec before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source review gives high confidence that current main lacks the requested schema-backed per-tool placement contract.

## Solution Assessment

Unclear as proposed: the security goal is valid, but maintainers still need to choose the permanent placement schema and runtime boundary. Existing elevated exec and sandbox tool policy are partial adjacent controls, not the same solution.
