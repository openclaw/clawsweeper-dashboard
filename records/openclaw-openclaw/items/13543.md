## Summary

Keep this open: current main still has session/agent-level sandbox mode, sandbox tool allow/deny filtering, sandbox browser settings, and exec escape behavior, but not the requested per-tool host-vs-sandbox placement contract.

## What This Changes

The issue requests configurable per-tool sandbox placement so tools like web_fetch and browser can be sandbox-routed while exec, message, cron, and gateway remain host-routed in the same session.

## Best Possible Solution

Add a documented, schema-backed core placement contract that can route selected tools to sandbox backends while leaving trusted tools host-routed in the same session.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still lacks the requested per-tool placement path.

## Solution Assessment

Unclear on the exact proposed config, but the requested direction is valid. The maintainable solution is a first-class placement contract, with tests proving it remains separate from sandbox tool filtering and elevated exec.
