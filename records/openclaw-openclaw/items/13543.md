## Summary

Keep open. Current main and the latest release still provide session/per-agent sandbox mode, sandbox tool allow/deny filtering, and exec elevated escape behavior, but not a schema-backed per-tool host-vs-sandbox placement policy; the issue discussion also shows the desired contract still needs maintainer product/security design.

## What This Changes

Add a config/runtime policy that can force selected tools such as web_fetch and browser into a sandbox while leaving trusted tools such as exec, message, cron, and gateway host-routed in the same session.

## Best Possible Solution

Choose a documented core tool-placement contract, or explicitly reject the direction, then implement it separately from tool allow/deny filtering and elevated exec with schema, runtime, tool-construction, docs, and upgrade tests.
