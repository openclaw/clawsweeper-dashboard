## Summary

Keep open. Current main and v2026.6.5 still select sandboxing at the session/run level; tool allow/deny policy and elevated exec do not provide the requested per-tool execution placement, and the discussion has not yet resolved the security model.

## What This Changes

Add schema-backed per-tool host-versus-sandbox execution placement within a single agent session.

## Best Possible Solution

Adopt one canonical, documented placement contract in core, carry the decision through typed tool construction and plugin context, and test it independently from allow/deny filtering and elevated exec.

Not applicable: this is a request for a new execution-placement capability, not a report of broken existing behavior. Source inspection confirms the requested capability is absent from current main.

## Solution Assessment

Unclear. Per-tool placement addresses the stated tradeoff, but maintainers still need to choose between host-first selective isolation and sandbox-first execution with explicit gateway-only exceptions before the narrowest maintainable implementation is known.
