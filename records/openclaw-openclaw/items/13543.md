## Summary

Keep open. Current main still has session/agent sandbox routing plus sandbox tool filtering, not per-tool execution placement, and the live discussion/labels already route this to product and security review.

## What This Changes

The issue requests a schema-backed policy to force selected tools such as `web_fetch` and `browser` into a sandbox while keeping trusted tools host-routed in the same session.

## Best Possible Solution

Add a documented core tool-placement contract that can route selected tools to a sandbox and selected trusted tools to host execution within one session, with tests proving it is separate from tool filtering and elevated exec.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection confirms current main only provides session/agent-level sandboxing plus allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the design: the capability belongs in core sandbox/tool routing, but the schema and runtime/tool boundary need product and security review first.
