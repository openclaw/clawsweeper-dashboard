## Summary

Keep open. Current main and the latest release still provide session or agent sandbox selection plus sandbox tool allow/deny filtering, but not the requested schema-backed per-tool placement policy for routing selected tools to sandbox and others to host in one session.

## What This Changes

Request a tool-level sandbox placement configuration so tools like web_fetch and browser can always run isolated while exec, message, cron, and gateway remain host-routed.

## Best Possible Solution

Add a documented core tool-placement contract that can route selected tools to a sandbox and selected trusted tools to host execution within one session, with regression coverage proving it is separate from tool filtering and elevated exec.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection confirms current main only provides session/agent-level sandboxing plus allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the design: the capability belongs in core sandbox/tool routing, but the exact schema and runtime/tool boundary need product and security review first. Reusing tools.sandbox.tools or tools.elevated as placement would not be the maintainable solution.
