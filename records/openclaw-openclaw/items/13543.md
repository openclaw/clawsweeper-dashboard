## Summary

Keep open. Current main still has session/agent-level sandboxing, sandbox tool allow/deny filtering, browser target selection, and elevated exec escape behavior, but not a schema-backed per-tool host-vs-sandbox placement policy; the linked Docker work in PR #29974 only partially unblocked this request.

## What This Changes

Add per-tool sandbox execution placement so selected tools can run isolated while trusted tools stay host-routed in the same session.

## Best Possible Solution

Add a documented, typed tool-placement contract evaluated where both tool identity and execution context are available, with regression coverage proving placement remains separate from filtering, browser target overrides, plugin tool context, and elevated exec.

## Reproduction Assessment

Not applicable as a runtime bug. The missing capability is source-confirmed on current main: sandboxing is decided per session/agent mode and sandbox tool policy only filters availability.

## Solution Assessment

Unclear whether the exact proposed JSON is the best contract. The direction is valid, but maintainers should choose the schema and security boundary before implementation.
