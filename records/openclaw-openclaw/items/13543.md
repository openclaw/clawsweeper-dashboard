## Summary

Keep open: current main still has session/agent-level sandboxing plus sandbox tool allow/deny filtering, but it does not provide the requested per-tool execution placement contract for mixing sandboxed web/browser tools with host-routed exec/message/cron/gateway in the same session.

## What This Changes

The issue requests schema-backed per-tool sandbox execution placement for selectively sandboxing untrusted tools while keeping trusted tools host-routed.

## Best Possible Solution

Design and document a core tool-placement contract that is separate from tool availability policy and elevated exec, then add focused schema/runtime/tool-construction tests for mixed host and sandbox tool execution.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main cannot express the requested per-tool placement because sandbox status is session-scoped and tools.sandbox.tools is only allow/deny policy.

## Solution Assessment

No current implementation solves it yet. The maintainable path is a schema-backed placement contract in core sandbox/tool routing, explicitly distinct from tools.sandbox.tools filtering and tools.elevated exec escape behavior.
