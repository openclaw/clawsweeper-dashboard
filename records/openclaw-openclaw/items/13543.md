## Summary

Keep open. Current main still has session-level sandbox selection and sandbox tool allow/deny filtering, but no schema-backed per-tool execution placement policy; this is also security-sensitive core runtime design work.

## What This Changes

Request to configure selected tools to run in a sandbox while other tools remain host-routed in the same session.

## Best Possible Solution

Add a documented, typed, schema-backed core tool-placement contract evaluated where tool identity and execution context are both available.

## Reproduction Assessment

Not applicable as a runtime bug report. The missing feature is verifiable by source inspection: current schema, typed config, runtime status, docs, and tool construction do not expose a tool-keyed placement decision.

## Solution Assessment

No. Current main's session sandboxing, sandbox allow/deny policy, browser target controls, and elevated exec are adjacent controls, but they do not solve selective per-tool execution placement.
