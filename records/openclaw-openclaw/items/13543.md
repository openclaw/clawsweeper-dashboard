## Summary

Keep open. Current main and the latest shipped release still provide session/agent sandbox placement, sandbox tool availability filtering, and exec-only host escape behavior, but not the requested per-tool host-versus-sandbox placement contract; the latest discussion also shows maintainers are still choosing the security/product shape.

## What This Changes

The issue requests per-tool sandbox execution placement so selected tools can always run isolated while trusted tools remain host-routed in the same session.

## Best Possible Solution

Define one reviewed core placement contract for mixed host/sandbox execution, keep it separate from sandbox tool filtering and exec elevation, then document and test the chosen schema/runtime boundary before implementation.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source and release checks show current main and v2026.5.20 support session/agent sandbox modes plus sandbox allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the contract. The capability likely belongs in core sandbox/tool routing, but the schema, plugin manifest boundary, and gateway-running exception model need explicit product and security review first.
