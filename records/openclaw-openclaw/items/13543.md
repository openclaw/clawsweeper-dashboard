## Summary

Keep open. Current main still has session/agent sandbox placement, sandbox-only tool availability filtering, and exec-only elevated escape behavior, but not the requested per-tool host-versus-sandbox placement contract; the latest discussion also shows the desired security/product shape is still unsettled.

## What This Changes

Request a schema-backed per-tool sandbox placement mode so tools such as web_fetch and browser can run sandboxed while exec, message, cron, and gateway stay host-routed in the same session.

## Best Possible Solution

Define one reviewed core mixed-placement contract, keep it separate from sandbox tool filtering and exec elevation, then document and test the chosen schema/runtime boundary before implementation.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source and release checks show current main and v2026.5.20 support session/agent sandbox modes plus sandbox allow/deny filtering, not per-tool placement.

## Solution Assessment

Unclear until maintainers choose the contract. The capability likely belongs in core sandbox/tool routing, but the schema, plugin/tool context boundary, and gateway-running exception model need explicit product and security review first.
