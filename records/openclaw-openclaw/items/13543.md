## Summary

Keep open. Current main still has session-level sandbox selection, sandbox tool allow/deny filtering, browser target defaults, and elevated exec escape behavior, but not a schema-backed per-tool host-vs-sandbox placement policy.

## What This Changes

The issue requests configurable per-tool sandbox placement so untrusted tools such as web_fetch and browser can run in Docker isolation while trusted tools stay host-routed in the same session.

## Best Possible Solution

Add a documented, typed tool-placement contract evaluated where tool identity and execution context are both available, with regression coverage separating placement from filtering and elevated exec.

## Reproduction Assessment

Not applicable as a runtime bug. The missing capability is confirmed by source and docs: current main exposes session-level sandboxing and tool allow/deny filters, not per-tool placement.

## Solution Assessment

Unclear whether the exact proposed JSON shape is best. The direction is valid, but maintainers should choose the schema and runtime security boundary before implementation.
