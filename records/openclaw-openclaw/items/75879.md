## Summary

Keep open. Current main still lacks the public headed runtime handoff surface, and open PRs #75880 and #75881 explicitly reference this issue with closing syntax, so the issue should remain open until one implementation lands or maintainers choose a replacement.

## What This Changes

Expose a runtime headed browser start override through the CLI, browser client, and browser tool so users can stop and restart the same managed profile visibly without editing config.

## Best Possible Solution

Land one focused PR that reuses `/start?headless=false`, covers CLI, browser client, browser tool, docs, tests, and changelog, then close any duplicate PRs after the selected implementation merges.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the gap: the HTTP route accepts `headless=false`, while the CLI, client helper, and browser tool provide no public way to send that false override.

## Solution Assessment

Yes. Reusing the existing request-level launch override is the narrowest maintainable path; the stale profile-lock concern is adjacent and already covered on current main unless a new failing case is demonstrated.
