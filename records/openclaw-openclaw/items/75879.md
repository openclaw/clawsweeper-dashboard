## Summary

Keep open. Current main and latest v2026.5.12 still lack the public headed runtime start surface, and https://github.com/openclaw/openclaw/pull/75881 is still an open closing PR for this issue. The profile-lock concern raised in discussion appears already covered by current main, so the remaining work is the public CLI/client/tool/docs surface rather than early issue closure.

## What This Changes

Expose a runtime headed browser start override through public browser CLI/tool surfaces so users can stop a managed browser and restart the same profile visibly without editing config.

## Best Possible Solution

Review, rebase, and either land or replace https://github.com/openclaw/openclaw/pull/75881; the final implementation should reuse the existing request-local `/start?headless=` contract, expose the public CLI/client/tool controls, and document the stop-then-start handoff caveat.

## Reproduction Assessment

Yes. Static source inspection of current main reproduces the feature gap: `/start` can accept `headless=false`, but the CLI, browser client helper, browser tool schema/action, and public docs expose only the headless=true direction.

## Solution Assessment

Yes. Reusing the existing request-local `headless` route contract is the narrow maintainable solution; the issue should proceed through the open closing PR rather than a duplicate repair branch.
