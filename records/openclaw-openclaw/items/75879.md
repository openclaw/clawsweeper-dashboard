## Summary

Keep open. Current main still lacks the public headed runtime handoff surface, and open PR #75881 explicitly references this issue with closing syntax, so the issue should remain open until that PR lands or maintainers choose a different implementation.

## What This Changes

The issue requests a public CLI, browser-client, and tool way to restart an OpenClaw-managed browser profile headed at runtime without editing config or restarting the Gateway.

## Best Possible Solution

Land one focused implementation that wires the existing `/start?headless=false` override through the CLI, browser client, browser tool schema/action, docs, tests, and changelog, then let the issue close on merge.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: `/start` accepts `headless=false`, but the CLI, browser client, tool schema/action, and docs expose no headed or false-valued start override.

## Solution Assessment

Yes. Reusing the existing request-level launch override is the narrowest maintainable fix because it avoids config mutation and Gateway restart; stale `Singleton*` lock handling should remain separate and is already present on current main.
