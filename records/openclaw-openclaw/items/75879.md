## Summary

Keep open. Current main still lacks the public headed runtime handoff surface, and PR #75881 is an open, unmerged closing PR from the same contributor, so this issue should remain open until that implementation lands or maintainers close it.

## What This Changes

Expose a public runtime way to start an OpenClaw-managed browser profile headed via CLI and browser tool without editing config or restarting the Gateway.

## Best Possible Solution

Review and land or close PR #75881; the implementation should wire the existing request-local override through CLI, client, browser tool, docs, tests, and changelog.

## Reproduction Assessment

Yes. Static source inspection of current main shows `/start` can parse `headless=false`, but the CLI, client helper, browser tool schema/action, and docs only expose the `headless=true` path.

## Solution Assessment

Yes. Reusing the existing request-local `headless` override is the narrowest maintainable direction; the issue should stay open while the linked implementation PR is reviewed.
