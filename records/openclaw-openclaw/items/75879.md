## Summary

Keep open. Current main still lacks the public runtime headed-start surface, and open PR #75881 explicitly references this issue with closing syntax, so the issue should remain open until that PR or an equivalent implementation lands.

## What This Changes

Expose a first-class CLI/client/browser-tool way to start an OpenClaw-managed browser profile headed at runtime for human handoff without editing config or restarting the Gateway.

## Best Possible Solution

Review and land PR #75881 or an equivalent focused implementation that reuses `/start?headless=false`, covers CLI, client, browser tool, docs, tests, and changelog, then let the issue close after merge.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the internal route accepts a false request override, but the CLI, client helper, browser tool schema/action, and docs expose only the headless=true direction.

## Solution Assessment

Yes. Reusing the existing request-level launch override and documenting the stop -> start headed handoff is the narrow maintainable fix; a broader launch-policy rewrite is not needed for this request.
