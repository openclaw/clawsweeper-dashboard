## Summary

Keep open. Current main still lacks the public headed runtime handoff surface, and PR #75881 is an open, unmerged closing PR from the same contributor, so this issue should remain open until that implementation lands or maintainers close the paired PR.

## What This Changes

The issue requests a runtime headed browser start surface for human handoff without editing browser config or restarting the Gateway.

## Best Possible Solution

Review and land or close PR #75881; the selected implementation should wire the existing request-level override through CLI, client, browser tool, docs, tests, and changelog, then let this issue close after merge.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the feature gap: `/start` accepts `headless=false`, but the CLI, browser client helper, browser tool schema/action, and docs expose only the headless=true direction.

## Solution Assessment

Yes. Reusing the existing request-local `headless` override is the narrowest maintainable solution; the issue should stay open while the linked implementation PR is reviewed rather than spawning duplicate repair work.
