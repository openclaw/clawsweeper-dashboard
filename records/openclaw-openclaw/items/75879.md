## Summary

Keep open. Current main still lacks the public headed runtime handoff surface, and open PR #75881 explicitly references this issue with closing syntax, so this issue should remain open until a selected implementation lands or maintainers close the paired PR.

## What This Changes

Expose a runtime headed launch override for the managed browser through CLI and browser tool/client surfaces so human handoff can stop and restart the same profile headed without config edits.

## Best Possible Solution

Land one focused implementation that reuses `/start?headless=false`, covers CLI, browser client, browser tool, docs, tests, and changelog, then close duplicate or superseded headed-start PRs after merge.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the route accepts `headless=false`, but the CLI, client helper, browser tool schema, and docs expose only the headless=true direction.

## Solution Assessment

Yes. Reusing the existing request-level launch override is the narrowest maintainable fix; the separate stale `Singleton*` lock concern is already addressed on current main unless a new failing case is demonstrated.
