## Summary

Keep open. Current main still lacks the public headed runtime handoff surface, and the same contributor has an open closing PR for this issue, so the issue should remain open until that PR lands or maintainers choose a different implementation.

## What This Changes

Expose a runtime headed browser start override for CLI and browser tool/client so human handoff can stop and restart a managed profile headed without changing config or restarting the Gateway.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/75881; the final implementation should reuse the existing request-local `headless` override, expose it through the public CLI/client/tool surfaces, and document the stop-then-start handoff caveat.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the feature gap: `/start` can accept `headless=false`, but the CLI, browser client helper, browser tool schema/action, and docs expose only the headless=true direction.

## Solution Assessment

Yes. Reusing the existing `/start?headless=` request-local contract is the narrowest maintainable direction; this should proceed through the open closing PR rather than a duplicate repair branch.
