## Summary

Keep open. Current main still does not show last-message previews in the Control UI session list, but this PR is not mergeable as proposed because it renders `subject` metadata instead of the transcript `lastMessagePreview` and overlaps the canonical feature request plus another open implementation PR.

## What This Changes

This PR changes the Control UI sessions table to render a two-line Updated cell with timestamp plus a subject-derived preview, adds related CSS, and adds a session-list planning Markdown file.

## Best Possible Solution

Ship one corrected Control UI implementation: add `lastMessagePreview` to the UI row type, request `includeLastMessage` through the existing bounded `sessions.list` path, render timestamp plus normalized preview with a full-preview tooltip, remove the scratch document, cover controller/view behavior, and include a changelog entry while coordinating #47120 and #47243.

## Reproduction Assessment

Yes. A source-level reproduction is enough: the PR renders `row.subject`, while current main exposes real transcript preview text as `lastMessagePreview` only when `sessions.list` is called with `includeLastMessage`, so direct sessions without a subject would show no preview.

## Solution Assessment

No. A render-only change based on `subject` is not the best fix; the maintainable path is to propagate and request `lastMessagePreview` explicitly and account for the documented transcript-read cost.
