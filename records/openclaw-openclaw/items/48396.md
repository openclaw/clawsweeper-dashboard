## Summary

Keep open. Current main still does not show last-message previews in the Control UI session list, and this PR is not safe to merge as-is because it renders `subject` metadata instead of the dedicated transcript preview field. The same feature remains tracked by open #47120 and overlapping open #47243, so maintainers should choose the canonical branch before this work is repaired or landed.

## What This Changes

The PR changes the Control UI sessions table to render a two-line Updated cell with timestamp plus a `subject`-derived preview, adds related CSS, and adds a Markdown planning file.

## Best Possible Solution

Ship one corrected Control UI implementation: add `lastMessagePreview` to the UI row type, request it through the existing bounded `sessions.list` path, render normalized timestamp-plus-preview text from that field with the full preview in the tooltip, remove the scratch document, add focused controller/view coverage, and include the required changelog entry while coordinating #47120 and #47243.

## Reproduction Assessment

Yes. A code-level reproduction is sufficient: this PR renders `row.subject`, while current main only exposes real transcript preview text as `lastMessagePreview` when `sessions.list` is called with `includeLastMessage`, so a direct session with recent messages and no subject would render no preview.

## Solution Assessment

No. A render-only change based on `subject` is not the maintainable fix; the safer approach is to propagate and request `lastMessagePreview` explicitly, account for the documented transcript-read cost, and cover the controller/view behavior in tests.
