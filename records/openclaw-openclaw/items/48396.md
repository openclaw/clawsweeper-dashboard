## Summary

Keep open: current main still shows only the timestamp in the Control UI sessions table, and this PR is not ready to merge because it renders `subject` metadata instead of the dedicated transcript preview field. The work also overlaps the open feature issue and another open implementation PR, so a maintainer should choose the canonical branch before repair automation rewrites this one.

## What This Changes

This PR adds a dual-line Updated cell in the Control UI sessions table with a subject-derived preview, related CSS, and an added session-list planning Markdown file.

## Best Possible Solution

Ship one corrected Control UI implementation that requests `includeLastMessage`, carries `lastMessagePreview` through the UI row type, renders that field with a full-preview tooltip, removes scratch documentation, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough: the PR renders `row.subject`, while current main exposes real transcript preview text as `lastMessagePreview` only when `sessions.list` is called with `includeLastMessage`.

## Solution Assessment

No. A render-only change based on `subject` is not the maintainable fix; the correct path is to propagate and request `lastMessagePreview` explicitly and account for its documented transcript-read cost.
