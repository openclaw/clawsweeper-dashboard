## Summary

Keep open: the feature is still useful, but this patch does not implement the intended last-message preview because it renders session subject metadata instead of the backend preview field and it lacks real behavior proof. The overlapping open issue and PR should be deduped before landing one corrected implementation.

## What This Changes

This PR adds a dual-line Updated cell in the Control UI Sessions table with timestamp/preview CSS and a root planning markdown file.

## Best Possible Solution

Land one deduped Control UI implementation that requests bounded `includeLastMessage`, carries `lastMessagePreview` through the UI type, renders a normalized timestamp plus preview, removes scratch planning artifacts, and includes focused UI/controller coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a user-facing feature PR. Source review gives a high-confidence check that the patch fails its intended behavior because it reads `subject` and does not request or type `lastMessagePreview`.

## Solution Assessment

No. The maintainable path is to reuse the existing `sessions.list` `includeLastMessage` contract and render `lastMessagePreview`, not to display `subject` metadata as a message preview.
