## Summary

Keep this PR open for maintainer follow-up. Current main still lacks Control UI last-message previews, but this patch is not merge-ready because it renders channel/session `subject` metadata instead of the backend `lastMessagePreview` field and adds an unrelated scratch document; the same feature is also being tracked by open #47120 and overlapping open PR #47243.

## What This Changes

The PR changes the Control UI sessions table to render a two-line Updated cell with timestamp plus a subject-derived preview, adds related CSS, and adds a session-list planning Markdown file.

## Best Possible Solution

Ship one corrected Control UI implementation: carry `lastMessagePreview` through the UI row type and event merge surface, request previews through a bounded `sessions.list` flow or a visible-row preview fetch path, render normalized timestamp-plus-preview text from that field, remove the scratch document, and add focused controller/view coverage while coordinating #47120 and #47243.

## Reproduction Assessment

Yes. A code-level reproduction is enough here: the PR renders `row.subject`, while the backend only exposes actual transcript preview text as `lastMessagePreview` when `sessions.list` is called with `includeLastMessage`, so a direct session with recent messages and no subject will render no preview.

## Solution Assessment

No. A render-only change using `subject` is not the maintainable fix; the safer path is to propagate and request `lastMessagePreview` explicitly, account for the documented transcript-read cost, and test the UI/controller behavior.
