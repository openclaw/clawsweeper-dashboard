## What This Changes

## Best Possible Solution

Update the current writer path so `updateLastRoute` cannot borrow a stale same-stat cache entry, and add a focused regression test for a same-mtime/same-size external rewrite.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: warm the session-store cache, externally rewrite `sessions.json` while preserving mtime and size, then call `updateLastRoute`; current main can take the cached object through `loadMutableSessionStoreForWriter` before saving.

## Solution Assessment

No. The original one-line change was locally reasonable on its old base, but current main now routes through `loadMutableSessionStoreForWriter`, so the maintainable fix needs to target that helper/call-site relationship and add regression coverage.
