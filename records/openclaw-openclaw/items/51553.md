## Summary

Close as superseded: the old direct `updateLastRoute` cache-bypass callsite no longer exists, and the shipped writer-cache design from the merged session-store performance work intentionally borrows the validated mutable cache while a writer owns the slot. Reintroducing this PR's `skipCache: true` direction would conflict with that current design rather than improve it.

## What This Changes

The PR changes the old `updateLastRoute` writer path to call `loadSessionStore(storePath, { skipCache: true })` while holding the session-store lock.

## Best Possible Solution

Keep the shipped writer-owned mutable cache from https://github.com/openclaw/openclaw/pull/68554; handle any remaining external mutation or observability policy through targeted follow-ups such as https://github.com/openclaw/openclaw/issues/79305.

## Reproduction Assessment

No for the original old-callsite failure: current main no longer has that direct `updateLastRoute` read and now documents/tests the writer-owned cache path. A contrived external same-stat rewrite remains a separate cache-policy edge, not a reason to keep this stale branch open.

## Solution Assessment

No. The one-line `skipCache` fix was reasonable on its old base, but current main deliberately avoids locked disk rereads; the maintainable solution is the already-shipped writer queue/cache design plus separate follow-up for any external-mutation observability needs.
