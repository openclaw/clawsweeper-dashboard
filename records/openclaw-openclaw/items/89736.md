## Summary

Keep open: current main and the latest release still render sub-1000 `openclaw status` token counts through the old all-values-as-`k` formatter, while this PR supplies a focused fix, regression coverage, and sufficient terminal proof. The open duplicate PR is not a safer supersession because it lacks real behavior proof, so this branch remains a viable landing path.

## What This Changes

The PR changes the `openclaw status` token formatter to show counts below 1000 as plain integers and adds focused status-format regression tests.

## Best Possible Solution

Land this proof-supplied status formatter fix, or an equivalent maintainer-selected duplicate, and handle the sibling sessions formatter as a separate consistency cleanup if desired.

## Reproduction Assessment

Yes. From current-main source, `openclaw status` routes session and cache summaries through `formatKTokens`, which still turns `999` into `(999 / 1000).toFixed(1) + "k"`, or `1.0k`.

## Solution Assessment

Yes for the reported status surface. The PR changes the implicated formatter, follows the existing `formatTokenCount` sub-1000 convention, preserves larger-value behavior, and adds focused boundary coverage; shared-helper or sessions-command cleanup can be follow-up work.
