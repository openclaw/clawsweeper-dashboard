## Summary

Keep open: current main and the latest release still have the old always-`k` status formatter, while this PR directly fixes the reported status path with sufficient terminal proof and no blocking patch findings.

## What This Changes

The PR changes the `openclaw status` token formatter to render values below 1000 as rounded integers and adds focused regression coverage for status token/cache output.

## Best Possible Solution

Land the focused status formatter fix with its regression tests, then track any broader CLI token-format consolidation separately if maintainers want consistency beyond `openclaw status`.

## Reproduction Assessment

Yes. Source inspection shows current `formatKTokens(999)` evaluates through `(999 / 1000).toFixed(1) + 'k'`, and the status command routes session/cache summaries through that helper.

## Solution Assessment

Yes. Mirroring the canonical sub-1000 integer branch in the status formatter is the narrowest fix because delegating directly to `formatTokenCount` would also change larger million-scale status output.
