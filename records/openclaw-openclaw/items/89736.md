## Summary

Keep this PR open for maintainer review and landing: current main and v2026.6.6 still have the misleading sub-1000 status-token formatter, while this branch applies the narrow guard, adds focused boundary coverage, and has sufficient terminal proof plus passing relevant CI.

## What This Changes

The PR updates `formatKTokens` in `src/commands/status.format.ts` to render values below 1000 as rounded integers and adds status-format regression tests for the boundary and small cache/session lines.

## Best Possible Solution

Land this focused status formatter fix as the canonical path for the linked status bug, then handle any broader `openclaw sessions` formatter consistency as a separate cleanup if maintainers want it.

## Reproduction Assessment

Yes. From source, current main formats `999` through `(999 / 1000).toFixed(1) + 'k'`, and the visible `openclaw status` session/cache rows route through that helper.

## Solution Assessment

Yes for the reported status bug. The patch changes the implicated formatter, mirrors the existing sub-1000 convention without adopting `formatTokenCount`'s million suffix behavior, and adds focused boundary coverage.
