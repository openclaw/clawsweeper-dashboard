## Summary

Keep open: current main and v2026.6.10 still truncate labeled confirm URL action data, so the PR is still useful, but the new bare-URL branch lowercases URI data and the contributor proof is parser-test output rather than real LINE behavior.

## What This Changes

The PR updates the LINE confirm directive parser to split label:data at the first colon and adds a regression test for URL confirm action data.

## Best Possible Solution

Preserve exact URL strings for both labeled and bare confirm segments, add focused regression coverage for the bare-URL case, then provide redacted real LINE or runtime proof before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows the confirm parser destructures `split(":")`, which truncates labeled URL action data at the scheme colon; I did not run tests because this review is read-only.

## Solution Assessment

No. First-colon splitting is the right local fix for labeled URL data, but the new bare-URL branch should preserve exact URL bytes and the PR still needs real LINE/runtime proof.
