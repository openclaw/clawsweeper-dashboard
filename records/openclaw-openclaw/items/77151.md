Keep this PR open because it fixes a real channel config regression in the right area, but the current diff still weakens existing JSON Schema unknown-property validation and lacks the required changelog entry.

## What This Changes

The PR changes bundled channel config JSON Schema generation to request Zod input semantics, regenerates bundled channel metadata, and adds regression assertions for optional default fields.

## Best Possible Solution

Keep the input-required semantics fix, preserve existing unknown-property rejection in generated metadata, add regression coverage for both behaviors, regenerate metadata, and add one Unreleased changelog bullet.

## Reproduction Assessment

Yes. Source inspection shows current main can reject omitted optional-with-default channel fields through generated JSON Schema before Zod defaults apply, and the linked Feishu report plus PR proof show the real upgrade crash and after-fix boot.

## Solution Assessment

No, not as currently implemented. Passing io:"input" is the right required-key fix, but the patch also drops existing additionalProperties:false checks for regular object schemas, so the safer solution must preserve unknown-property validation while changing required semantics.
