## Summary

Keep open: this PR is the canonical work for a real Control UI accessibility gap, but it still needs contributor proof, a rebase over newer main UI accessibility work, and one remaining ARIA-pattern correction before merge.

## What This Changes

The branch adds screen-reader and keyboard accessibility semantics across the Control UI, including labels/ARIA for config and chat controls, skip-link/heading CSS, semantic navigation/palette/session markup, raw JSON editor sizing, and a Shift+Escape chat-focus shortcut.

## Best Possible Solution

Rebase onto current main, preserve the newer slash-menu and command-palette accessibility implementations, remove or complete the bottom-navigation tab pattern, then add focused tests plus redacted real VoiceOver/Safari or keyboard proof before maintainer review.

## Reproduction Assessment

Yes for the source-level accessibility gaps: current main still shows generated config labels without stable input associations and several generated controls without accessible labels. The VoiceOver hang and full screen-reader experience still need real Safari/VoiceOver proof.

## Solution Assessment

No as currently submitted: the direction is useful, but the branch conflicts with newer Control UI accessibility work, still adds tab roles without a full tab pattern, and lacks real behavior proof. The narrow maintainable path is a rebase plus targeted cleanup that preserves current main's newer implementations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Use nav semantics for bottom navigation:** `ui/src/ui/views/bottom-tabs.ts:18-21`
  - body: This branch turns the bottom navigation into a `tablist` with `tab` buttons, but the component still behaves as ordinary navigation buttons and does not provide associated `tabpanel` elements or arrow-key/one-tab-stop focus management.
