## Summary

Keep open. Current main only covers part of the accessibility work, while the raw config editor, skip navigation, generated settings controls, chat group footer focus behavior, chat role headings, and session sort announcements still have observable gaps. The related settings accessibility issue was closed as superseded by this PR, so this remains the canonical implementation thread.

## What This Changes

The PR updates Control UI TypeScript and CSS to add ARIA labels and semantics, keyboard/focus affordances, reduced-motion handling, raw config editor constraints, chat and command menu accessibility, session sort announcements, and a Shift+Escape chat-focus shortcut.

## Best Possible Solution

Ship a focused, rebased Control UI accessibility patch set that preserves newer main improvements, keeps the streaming live-region problem tracked separately in #65538 unless maintainers intentionally expand scope, adds DOM-level accessibility assertions where practical, and completes manual Safari/VoiceOver acceptance before merge.

## Reproduction Assessment

Not applicable as a normal bug reproduction for most of this feature PR. Static inspection gives high-confidence proof that current main still lacks several requested accessibility surfaces, while the reported VoiceOver raw-editor hang still needs manual Safari/VoiceOver verification.

## Solution Assessment

Unclear. The accessibility direction is appropriate and earlier concrete review bugs were followed up, but the branch is broad and stale against current main; the safer maintainable path is a rebase or split replacement that preserves newer UI accessibility work and avoids overclaiming tab-widget semantics.
