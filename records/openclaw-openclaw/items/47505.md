## Summary

Keep open: this PR still contains useful Control UI accessibility work that current main has not fully absorbed, but it is not merge-ready because real behavior proof is absent and one ARIA change introduces an incomplete tab-widget contract.

## What This Changes

The branch updates Control UI CSS and Lit templates to add ARIA labels and roles, semantic headings and lists, skip navigation, reduced-motion handling, raw config editor constraints, and a Shift+Escape chat-input shortcut.

## Best Possible Solution

Refresh the PR against current main, preserve the newer command palette and slash-menu wiring, fix the bottom navigation semantics, and add redacted VoiceOver/Safari or keyboard proof before merge.

## Reproduction Assessment

Yes for the source-level accessibility gaps: current main still shows generated settings labels without for/id associations and unlabeled numeric stepper controls. The claimed VoiceOver raw-editor hang still needs real Safari/VoiceOver proof.

## Solution Assessment

Not yet. The direction is useful, but the PR should be refreshed around current main's newer accessibility work and should avoid assigning tab roles to bottom navigation without the full tab pattern.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Keep bottom navigation out of the tab pattern:** `ui/src/ui/views/bottom-tabs.ts:20-24`
  - body: The PR changes the bottom navigation to `role="tablist"`/`role="tab"`, but the buttons still behave like ordinary navigation buttons: every item remains in the normal Tab order and there is no arrow-key tab switching or tabpanel relationship.
