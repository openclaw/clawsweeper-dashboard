Current main only partially overlaps this accessibility work, and the PR discussion shows the earlier segmented-control and slash-menu ARIA problems were followed up in later commits; the remaining path is maintainer review, rebase, and manual accessibility acceptance rather than cleanup closure.

## What This Changes

The PR updates Control UI TypeScript and CSS to add ARIA labeling and roles, skip navigation, screen-reader headings, keyboard focus affordances, raw config editor constraints, sessions sort announcements, and a Shift+Escape chat-focus shortcut.

## Best Possible Solution

Ship a focused, rebased Control UI accessibility patch that preserves newer main improvements, keeps unrelated screen-reader streaming work tracked separately, adds DOM-level accessibility assertions where practical, and completes the listed Safari/VoiceOver acceptance pass before merge.

## Reproduction Assessment

Not applicable as a normal bug reproduction for most of this feature PR. Static inspection gives high-confidence evidence that current main still lacks several requested accessibility surfaces, while the reported raw-editor VoiceOver hang needs manual Safari/VoiceOver verification.

## Solution Assessment

Unclear. The accessibility direction is appropriate, but the broad stale branch should be rebased or split, and the bottom navigation should either stay ordinary navigation buttons or implement the tab keyboard contract.
