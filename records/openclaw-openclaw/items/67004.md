## Summary

Keep open: current main does not implement legacy Feishu document reads, and the PR remains useful but is not merge-ready because PR head still masks old-doc read failures, makes rich content a hard dependency for plain-text reads, misses user guidance for legacy tokens/scopes, and lacks real behavior proof.

## What This Changes

The PR adds Feishu legacy `/docs/:token` read support by detecting doc v2 metadata, routing old documents through doc v2 REST endpoints, and using upgraded tokens on the existing docx path.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after old-doc read failures propagate clearly, raw text can still be returned when only rich block content fails, and `/docs/:token` token extraction plus old-doc permissions are documented.

## Reproduction Assessment

Yes, for the review findings via source inspection: in PR head, metadata can return `is_upgraded: false`, `/raw_content` can succeed, and `/content` can fail, causing `readOldDoc` to reject inside the meta fallback catch and fall through to docx with the legacy token. I did not run tests because this review was read-only.

## Solution Assessment

No. The Feishu plugin boundary is appropriate, but this PR is not the best final patch until metadata fallback only covers metadata, rich content is best-effort for plain-text reads, and caller-facing legacy token/scope guidance is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep old-doc reads out of the meta fallback:** `extensions/feishu/src/docx.ts:876`
  - body: `readOldDoc` still runs inside the `try` that is meant to guard version detection, so a `/raw_content` or `/content` failure after metadata identifies a legacy document is swallowed and retried against the docx API with the legacy token. Move the legacy read outside the metadata catch so downstream old-doc failures surface correctly.

  - confidence: 0.82

  - body: This adds a user-facing `/docs/:token` read path, but the schema and bundled skill still guide agents toward `/docx/` tokens and docx-only scopes. Update the Feishu doc tool guidance with legacy token extraction and any old-doc permissions operators need for the new path.
  - confidence: 0.79
