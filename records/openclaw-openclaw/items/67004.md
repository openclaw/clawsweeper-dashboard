## Summary

Keep this PR open for author or maintainer follow-up. Current main still lacks legacy Feishu `/docs/:token` read support, but the proposed patch has concrete old-doc failure-path bugs that should be fixed before merge.

## What This Changes

The PR adds Feishu legacy document read support by detecting doc v2 metadata, reading old `/docs/:token` documents through REST endpoints, routing upgraded documents to the existing docx SDK path, and adding focused tests/types.

## Best Possible Solution

Land an extension-owned legacy Feishu read path after old-doc read errors propagate clearly, raw text can still be returned when richer content parsing fails, Feishu old-doc scopes and skill/docs are updated, and targeted tests cover both success and failure paths.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to extend the PR's mocked `old-doc.test.ts` harness so metadata returns `is_upgraded: false` and either `/content` or `/raw_content` fails; the current PR code catches that read failure in `docx.ts` and falls through to docx with the legacy token.

## Solution Assessment

No. The feature belongs in the Feishu plugin, but this patch is not the best final form until the metadata fallback is narrowed and rich-content parsing is made best-effort when raw text is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep old-doc reads out of the meta fallback:** `extensions/feishu/src/docx.ts:876`
  - body: `readOldDoc` still runs inside the `try` that catches metadata lookup failures. If `/raw_content` or `/content` fails after metadata has identified a legacy doc, the error is swallowed and the code falls through to docx reads on the legacy token, masking the real failure. Move the old-doc read after the metadata catch so only `fetchOldDocMeta` is fallback-eligible.
  - confidence: 0.91
- **[P2] Make rich content best-effort for raw old-doc reads:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned document text comes from `/raw_content`, but the implementation waits for `/content` in the same `Promise.all`. A transient or permission failure from the richer endpoint rejects the whole read even when plain text was already available, so fetch or parse rich content best-effort and still return raw text when possible.
  - confidence: 0.78
