## Summary

Keep this PR open. Current main still lacks legacy Feishu `/docs/:token` read support, so the contribution is relevant, but PR head still has blocking fallback behavior that can hide old-doc read failures and it needs user-facing guidance updates before merge.

## What This Changes

The PR adds Feishu legacy document read support by detecting doc v2 metadata, reading old documents through doc v2 REST endpoints, routing upgraded documents to the existing docx path, and adding old-doc types/tests.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after old-doc read failures propagate clearly, raw text can still be returned when only rich block content fails, `/docs/:token` token extraction and required permissions are documented, a changelog entry credits the contributor, and targeted Feishu tests cover both success and failure paths.

## Reproduction Assessment

Yes. A high-confidence reproduction is to extend the PR-head `old-doc.test.ts` harness so metadata returns `is_upgraded: false`, `/raw_content` succeeds, and `/content` returns a nonzero code or rejects; the current `readDoc` catches that downstream failure and falls through to docx with the legacy token.

## Solution Assessment

No. The extension-owned direction is the right boundary, but the current patch is not the best final fix because metadata fallback still catches old-doc reads, rich content is not best-effort, and caller-facing token/scope/changelog guidance is missing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep old-doc reads out of the meta fallback:** `extensions/feishu/src/docx.ts:876`
  - body: `readOldDoc` still runs inside the `try` that is supposed to guard only old-doc metadata lookup. If `/raw_content` or `/content` fails after metadata identifies a legacy document, the catch swallows the real old-doc error and falls through to docx with a `/docs` token, producing a misleading failure path. Move the old-doc read after the metadata catch or return its promise outside the catch so read failures propagate.
  - confidence: 0.92
- **[P2] Return raw text when rich old-doc content fails:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The old-doc return payload gets its main `content` from `/raw_content`, but this `Promise.all` rejects the whole read when the richer `/content` endpoint fails. That loses readable text that was already available and currently feeds the invalid docx fallback path. Fetch or parse rich content best-effort while still failing clearly when raw content itself cannot be read.
  - confidence: 0.8
- **[P3] Document legacy tokens and old-doc permissions:** `extensions/feishu/src/docx.ts:868-876`
  - body: This adds a new user-facing read path for `/docs/:token`, but the tool schema and bundled skill still guide agents toward `/docx/` token extraction and docx-only scopes, and the PR does not add the required user-facing changelog entry. Update the tool/skill/docs guidance and changelog so operators know how to use and permission the new path.
  - confidence: 0.82
