## Summary

Keep open. Current main still has only the docx read path, and this PR is a plausible Feishu plugin feature, but the PR head still has source-backed behavior blockers and lacks external real behavior proof before merge.

## What This Changes

The PR adds Feishu legacy `/docs/:token` document reads by detecting doc v2 metadata, routing old documents through doc v2 REST endpoints, and using upgraded tokens on the existing docx path.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after old-doc read failures propagate clearly, raw text can still be returned when only rich block content fails, and `/docs/:token` token extraction plus old-doc permissions are documented.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to extend the PR-head `old-doc.test.ts` harness so metadata returns `is_upgraded: false`, `/raw_content` succeeds, and `/content` returns a nonzero code or rejects; current PR code catches that downstream failure and falls through to docx with the legacy token.

## Solution Assessment

No. The Feishu-plugin boundary is appropriate, but the current patch is not the best final fix because metadata fallback still catches old-doc reads, rich content is not best-effort, and caller-facing legacy token/scope guidance is missing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep old-doc reads out of the meta fallback:** `extensions/feishu/src/docx.ts:876`
  - body: `readOldDoc` still runs inside the `try` that is meant to guard only metadata detection. If `/raw_content` or `/content` fails after metadata identifies a legacy document, the catch falls through to the docx API with the legacy token and masks the real old-doc failure. Move the old-doc read outside the metadata catch so downstream read failures propagate clearly.
  - confidence: 0.91
- **[P2] Return raw text when rich old-doc content fails:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned `content` field comes from `/raw_content`, but `Promise.all` makes the richer `/content` endpoint a hard dependency. A transient rich-content failure discards readable plain text that was already available; fetch or parse rich content best-effort while still failing when raw content itself cannot be read.
  - confidence: 0.82
- **[P3] Document legacy tokens and old-doc permissions:** `extensions/feishu/src/docx.ts:868-876`
  - body: This adds a user-facing `/docs/:token` read path, but the schema and bundled skill still tell agents to extract only `/docx/` tokens and list only docx scopes. Update the tool/skill guidance with legacy token extraction and the additional old-doc permissions operators need for the new path.
  - confidence: 0.79
