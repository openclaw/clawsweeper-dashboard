## Summary

Keep open: this PR is still useful because current `main` still creates full-CLI startup progress without a JSON-output guard and still calls Clack spinner without overriding its stdout default. The branch fixes the central progress path, supplies focused regression coverage, and includes sufficient terminal proof for the source-entry CLI behavior.

## What This Changes

The PR suppresses full CLI startup progress for normalized JSON-output argv, routes Clack spinner output through the selected progress stream, and adds focused CLI progress/runCli regression tests.

## Best Possible Solution

Land the central CLI progress fix after normal maintainer/CI handling, keeping the optional global progress config out of this narrow correctness PR.

## Reproduction Assessment

Yes, at source level: non-routed `--json` commands can reach the unguarded full-CLI startup progress path before Commander emits JSON, and current `createCliProgress` calls Clack without overriding its default stdout output.

## Solution Assessment

Yes, this is the right layer: it suppresses startup progress before full CLI parsing for machine-readable invocations and routes Clack spinner output through the selected progress stream. Per-command suppression would be too narrow, and the optional config knob belongs in separate product review.
