## What This Changes

The PR suppresses full CLI startup progress when normalized argv contains `--json`, routes shared Clack progress output through the configured progress stream, and adds focused regression tests.

## Best Possible Solution

Land the narrow startup/progress fix after ordinary maintainer review and CI, keeping `--json` stdout byte-clean centrally instead of adding per-command filters or a new config knob.

## Reproduction Assessment

Yes. Source inspection plus the related report show the full CLI path starts progress before command parsing, and Clack defaults to stdout unless an output stream is supplied; I did not run a PTY repro in this read-only pass.

## Solution Assessment

Yes. The PR fixes the central startup boundary before full program parsing and uses Clack's documented `output` option, which is cleaner than per-command suppression or a new `cli.progress` setting.
