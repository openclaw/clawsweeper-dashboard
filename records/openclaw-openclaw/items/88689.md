## What This Changes

The PR suppresses full CLI startup progress for JSON-mode invocations, routes Clack spinner output through the selected progress stream, and adds focused regression tests.

## Best Possible Solution

Land this central CLI progress fix after ordinary maintainer and CI review, leaving the optional global progress config knob for separate product discussion.

## Reproduction Assessment

Yes, at source level: non-routed JSON commands can enter the full CLI startup path before Commander parsing, and @clack/prompts defaults spinner output to stdout unless an output stream is supplied. I did not run a live interactive TTY repro in this read-only review.

## Solution Assessment

Yes: suppressing startup progress centrally for JSON argv and routing Clack to the selected progress stream is the narrowest maintainable fix. Per-command suppression would be too narrow, and a new cli.progress config option is a separate product decision.
