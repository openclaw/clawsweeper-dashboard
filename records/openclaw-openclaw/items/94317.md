- `impact:other`: The issue affects the readability of Telegram-delivered agent output by exposing escaped formatting markup to users.

## Summary

Keep open. Current main has rich Telegram table delivery only behind the `richMessages` opt-in, while the default/legacy HTML path still escapes unsupported raw HTML tags, so the reported escaped-table behavior remains possible and is not safely superseded by the open auto-detect PR.

## What This Changes

The issue requests Telegram delivery handling for agent HTML table output so users do not receive escaped table tags as literal text.

## Best Possible Solution

Keep this issue open until maintainers choose the product shape; the lowest-risk fix is a Telegram-owned fallback that turns unsupported raw HTML tables on the standard text path into readable pre/code or pipe-table text, while the rich-message opt-in path can continue preserving native table markup.

## Reproduction Assessment

Yes for the current behavior from source inspection: standard Telegram HTML delivery escapes unsupported raw table tags, while native table rendering only happens on the `richMessages` path. I did not run a live Telegram bot repro during this read-only review.

## Solution Assessment

Unclear as a final product choice. A new config-driven auto-detect path is already proposed elsewhere but not accepted; a narrower readable fallback on the legacy text path looks safer if maintainers want to fix the escaped-table symptom without changing the rich-message default.
