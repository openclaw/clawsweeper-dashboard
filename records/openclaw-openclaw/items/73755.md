## What This Changes

Adds opt-in `gateway.tailscale.preserveFunnel` config/docs/schema/tests plus Tailscale Funnel status parsing so serve mode can skip reapplying Serve when an external Funnel route covers the gateway port.

## Best Possible Solution

Land this opt-in preserve-Funnel implementation after exact-head CI completes, keeping existing Serve defaults and the OpenClaw-managed Funnel password-only policy unchanged.

## Reproduction Assessment

Yes, source-backed. Current main unconditionally reapplies `tailscale serve` in serve mode, and Tailscale Serve/Funnel share the same ServeConfig route state, giving a deterministic overwrite path for the linked restart scenario.

## Solution Assessment

Yes. The opt-in status check is the narrowest maintainable path because it preserves default behavior, avoids relaxing managed Funnel auth policy, and skips Serve only when Tailscale status says an AllowFunnel-enabled handler already targets the gateway port.
