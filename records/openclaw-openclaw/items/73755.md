## Summary

Keep open. The preserve-Funnel gateway direction is source-backed and the real tailnet proof is sufficient, but the current PR head still fails typecheck/lint/build and the local fs-safe queue copy drops upstream queue-ID path validation.

## What This Changes

The PR adds opt-in `gateway.tailscale.preserveFunnel`, config/docs/schema/tests, and local fs-safe JSON queue helpers so serve mode can skip reapplying Tailscale Serve when a Funnel route already covers the gateway port.

## Best Possible Solution

Land the preserve-Funnel gateway behavior after the queue shim is removed or hardened to match fs-safe behavior and the touched core checks are green.

## Reproduction Assessment

Yes, source-backed. Current main unconditionally reapplies `tailscale serve` in serve mode, and the linked report plus PR body give deterministic Tailscale steps and after-fix live tailnet logs.

## Solution Assessment

No, not yet. The opt-in Tailscale check is a maintainable direction, but the branch must compile and retain fs-safe queue path-safety before merge.
