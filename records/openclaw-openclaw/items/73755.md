## Summary

Keep open: the Tailscale preserve-Funnel direction is useful and the PR now has sufficient real tailnet proof, but the branch still introduces a blocking filesystem-safety regression in its copied durable queue helper.

## What This Changes

Adds opt-in `gateway.tailscale.preserveFunnel` config/docs/schema/tests and Tailscale Funnel status parsing so serve mode can skip reapplying Serve when an external Funnel route already covers the gateway port, plus a local JSON durable queue compatibility shim.

## Best Possible Solution

Land the preserve-Funnel gateway behavior after the queue shim is removed or hardened to match fs-safe path validation and the touched core checks stay green.

## Reproduction Assessment

Yes, source-backed. Current main unconditionally reapplies `tailscale serve` in serve mode, and the linked report plus PR body give deterministic Tailscale steps and after-fix live tailnet logs.

## Solution Assessment

No, not yet. The opt-in Tailscale check is a maintainable direction, but the copied queue helper must retain fs-safe queue ID validation before merge.
