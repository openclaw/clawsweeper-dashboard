## Summary

Keep open: current main still lacks the preserve-Funnel path, and this PR is a focused implementation with docs, schema, tests, changelog credit, and sufficient live proof; I found no blocking correctness finding.

## What This Changes

Adds opt-in `gateway.tailscale.preserveFunnel` config/docs/schema/tests plus Tailscale Funnel status parsing so serve mode can skip reapplying Serve when an external Funnel route covers the gateway port.

## Best Possible Solution

Land this focused opt-in preserve-Funnel implementation after exact-head CI completes, keeping existing Serve defaults and the OpenClaw-managed Funnel password-only policy unchanged.

## Reproduction Assessment

Yes, source-backed. Current main unconditionally reapplies `tailscale serve` in serve mode, and upstream Tailscale Serve/Funnel config semantics plus https://github.com/openclaw/openclaw/issues/57241 provide a deterministic overwrite path.

## Solution Assessment

Yes. The opt-in status check is the narrowest maintainable fix because it preserves default behavior, avoids relaxing managed Funnel auth policy, and skips Serve only when Tailscale's own status says an AllowFunnel-enabled handler already targets the gateway port.
