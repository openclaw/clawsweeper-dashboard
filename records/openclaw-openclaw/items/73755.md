## What This Changes

## Best Possible Solution

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep secrets reload events untrusted:** `src/gateway/server.impl.ts:487`
  - body: The PR adds `trusted: true` when enqueueing secrets reloader degraded/recovered events, but current main explicitly keeps those events `trusted: false`. That flag controls whether the next prompt receives `System:` or `System (untrusted):`, and SecretRef exec provider failures can include provider-supplied error text, so this would turn diagnostic data into trusted prompt text. Drop this unrelated hunk or keep the current `trusted: false` behavior.
  - confidence: 0.86
