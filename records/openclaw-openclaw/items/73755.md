## Summary

Keep this PR open. Current main still has the Tailscale Serve overwrite behavior, and the PR’s opt-in preserve-Funnel direction is a reasonable narrow fix, but the current head also introduces an unrelated secrets system-event trust-boundary change and is marked dirty/non-mergeable.

## What This Changes

The PR adds an opt-in `gateway.tailscale.preserveFunnel` config path that checks Tailscale Funnel status before reapplying Serve, with parser tests, gateway tests, generated config schema/help, and changelog coverage, plus unrelated formatting and secrets event trust changes.

## Best Possible Solution

Keep the opt-in preserve-Funnel behavior, parser, schema/help, changelog, and focused tests; drop the unrelated secrets `trusted: true` hunk, rebase onto current main, and rerun the focused gateway, infra, config, and changed-lane validation before merge.

## Reproduction Assessment

Yes. The linked bug reproduces statically on current main because serve-mode startup unconditionally reaches `tailscale serve --bg --yes <port>`; the review finding reproduces directly from the PR diff changing current-main `trusted: false` to `trusted: true` for secrets reload events.

## Solution Assessment

No, not as-is. The preserve-Funnel design is a maintainable solution for the Tailscale bug, but the safest patch removes the unrelated trust-boundary change and rebases before normal maintainer review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep secrets reload events untrusted:** `src/gateway/server.impl.ts:490`
  - body: The PR adds `trusted: true` when enqueueing secrets reloader degraded/recovered events, but current main explicitly marks those events untrusted. The trusted flag controls whether the next prompt receives `System:` or `System (untrusted):`; SecretRef resolver failures can include provider-supplied error text, so this unrelated hunk broadens trusted prompt input. Drop it or keep the current `trusted: false` behavior.
  - confidence: 0.88
