## Summary

Keep open: the Tailscale fix direction has source-backed reproduction and real tailnet proof, but the current PR head is not merge-ready because CI catches a regression in the new local durable queue helper.

## What This Changes

The PR adds an opt-in `gateway.tailscale.preserveFunnel` config option with gateway/docs/schema/tests and local fs-safe queue compatibility helpers so serve mode can skip reapplying Tailscale Serve when a Funnel route already covers the gateway port.

## Best Possible Solution

Land a focused preserve-Funnel implementation after restoring or removing the local fs-safe queue shim so existing durable queue behavior and CI stay green.

## Reproduction Assessment

Yes, source-backed. Current main unconditionally calls `enableTailscaleServe` for serve mode, and the PR body includes live tailnet logs showing the proposed skip preserving a public Funnel route.

## Solution Assessment

No, not yet. The opt-in Tailscale check is a maintainable direction, but the current branch should either keep using `@openclaw/fs-safe/store` or exactly mirror its queue cleanup and path-safety behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Clean stale tmp files when loading queue entries:** `src/infra/json-durable-queue.ts:119`
  - body: This helper now backs `loadPendingSessionDeliveries`, but it ignores `cleanupTmpMaxAgeMs`. That option is how session-delivery recovery removes orphaned `.tmp` files; the current CI annotation shows the existing storage test failing because `orphan-entry.tmp` remains after load. Restore the upstream cleanup behavior or keep using `@openclaw/fs-safe/store`.
  - confidence: 0.96
- **[P2] Validate queue entry IDs before joining paths:** `src/infra/json-durable-queue.ts:29-33`
  - body: The copied fs-safe helper no longer validates queue entry IDs before building paths. Recovery can later operate on IDs read from durable JSON entries, so this should retain the upstream safe path-segment guard or avoid the local copy entirely.
  - confidence: 0.78
