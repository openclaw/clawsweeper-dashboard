The issue asks for the hardcoded 5 MiB media-store and sandbox-staging `MEDIA_MAX_BYTES` limit to become configurable for larger sandboxed file transfers.

## Best Possible Solution

Define a bounded, documented media-store and sandbox-staging limit contract, preserve the secure default, thread the resolved cap through store/staging/chat preflight callers, and add config docs plus focused regression coverage.
