## Best Possible Solution

Define a bounded media staging/store max-bytes config that preserves the 5 MiB default, documents precedence, and threads the resolved value through store, sandbox staging, chat preflight, schema/help, docs, and regression coverage.

Unclear. A bounded configurable media staging limit is plausible, but the exact public schema, safety ceiling, and precedence with existing media and channel limits need maintainer approval before implementation.
