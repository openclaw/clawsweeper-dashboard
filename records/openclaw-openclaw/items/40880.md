## Summary

Keep open. Current main still hardcodes the 5 MB media-store and sandbox staging cap, `media.maxBytes` is absent from the accepted config surfaces, and open PR #40950 is the unmerged closing path for this issue.

## What This Changes

The issue requests a configurable sandbox/media staging byte limit instead of the current hardcoded 5 MB cap.

## Best Possible Solution

Land PR #40950 or an equivalent refreshed fix that adds a bounded `media.maxBytes` defaulting to 5 MB, wires it through store, sandbox staging, chat preflight, config metadata, docs, changelog, and focused tests while leaving broader image sanitizer limits to #67031.

## Reproduction Assessment

Yes. A sandboxed non-image attachment larger than 5 MB but below the chat attachment parse cap follows current-main code through `MEDIA_MAX_BYTES` in store, staging, and chat preflight, and existing tests assert the rejection path.

## Solution Assessment

Yes for keeping this issue open. Because PR #40950 is open and uses closing syntax for this issue, the maintainable path is to review, refresh, and land that PR or an equivalent targeted replacement before closing this issue.
