## Summary

Keep open: current main and the latest release still bind sandbox media staging to a fixed 5 MiB media-store cap, and the requested configurability remains unresolved. The remaining work needs a maintainer decision on config scope and resource-safety bounds before a repair PR is safe.

## What This Changes

The issue requests a user-configurable media staging byte limit so sandboxed agents can receive files larger than the hardcoded 5 MiB cap.

## Best Possible Solution

Define one bounded, documented media-store/staging limit policy with a secure default, strict validation, and consistent wiring through store, sandbox staging, chat preflight, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main has a fixed 5 MiB staging cap and a gateway regression test where a 6 MiB PDF is rejected as over the sandbox staging limit; I did not run a live gateway transfer in this read-only review.

## Solution Assessment

Unclear. Configurability is plausible, but maintainers still need to choose whether the cap is global or sandbox-specific, the safe maximum, and how it interacts with existing per-channel and media-understanding limits.
