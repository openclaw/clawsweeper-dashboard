## Summary

Keep open because current main still has the fixed 5 MiB media-store, sandbox-staging, and chat preflight limit, and the only exact closing implementation is still an open, unmerged, conflicting PR.

## What This Changes

Make the sandbox/media-store `MEDIA_MAX_BYTES` cap configurable so sandboxed agents can receive files larger than 5 MiB.

## Best Possible Solution

Land the open PR or a narrow replacement that adds a bounded schema-backed `media.maxBytes` setting and threads it through media store, sandbox staging, chat preflight, docs/help, and regression tests while preserving stricter caller-specific caps.

## Reproduction Assessment

Yes. Source inspection on current main shows the fixed 5 MiB store, sandbox staging, and chat preflight caps, and current gateway coverage asserts a 6 MiB sandboxed PDF is rejected.

## Solution Assessment

Unclear for the final implementation. A bounded `media.maxBytes` setting is the plausible direction, but the exact schema, ceiling, and precedence with caller-specific caps need maintainer review; keeping this issue open while the linked PR is unmerged is the correct triage outcome.
