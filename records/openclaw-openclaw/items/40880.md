## Summary

Keep open: current main and the latest release still keep the media-store and sandbox-staging cap fixed at 5 MiB, and the strict config schema still has no `media.maxBytes` or equivalent sandbox-staging limit. ## What This Changes

Request to make the fixed 5 MiB media-store and sandbox-staging cap configurable so larger attachments can reach sandboxed agents.

## Best Possible Solution

Define a bounded, documented media-store and sandbox-staging limit contract, keep secure defaults, thread the resolved cap through store/staging/chat preflight, and add focused upgrade, docs, and regression coverage.

Yes, at source level. Current main still binds media store, sandbox staging, and chat preflight to the fixed 5 MiB cap, and tests assert oversize staging rejection/skipping behavior.
