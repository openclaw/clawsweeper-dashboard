## Summary

Keep open. Current main still hardcodes the 5 MB media-store and sandbox staging cap, the top-level media config has no shared `maxBytes` field, and open PR #40950 is the unmerged closing path for this issue.

## What This Changes

The issue asks for the hardcoded 5 MB media/sandbox staging limit to become user-configurable.

## Best Possible Solution

Land PR #40950 or an equivalent narrow fix that adds a bounded `media.maxBytes` defaulting to 5 MB, threads the effective value through store, sandbox staging, chat preflight, schema/help/docs/tests, and leaves broader image-limit work to #67031.

## Reproduction Assessment

Yes. A sandboxed non-image attachment larger than 5 MB follows current-main code through `MEDIA_MAX_BYTES` in store, staging, and chat preflight, and existing tests assert the same rejection behavior.

## Solution Assessment

Yes for keeping this issue open. Because PR #40950 is open and uses closing syntax, the maintainable path is to review and land that PR or an equivalent targeted replacement before closing the issue.
