## Summary

Keep open. Current main still has the fixed 5 MB media store, sandbox staging, and chat preflight cap, and the linked closing implementation path #40950 is still open rather than merged.

## What This Changes

Make the sandbox/media staging size cap configurable instead of hardcoding the 5 MB `MEDIA_MAX_BYTES` limit.

## Best Possible Solution

Review and land #40950, or a narrow replacement, so `media.maxBytes` is schema-backed, documented, bounded, and consistently used by media storage, sandbox staging, chat preflight, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows a sandboxed non-image chat attachment larger than 5 MB follows the fixed `MEDIA_MAX_BYTES` preflight/staging path, and a current test asserts the 6 MB rejection.

## Solution Assessment

Yes for keeping this issue open. Because #40950 is still open and explicitly linked as the implementation path, the maintainable solution is to review and land that PR or an equivalent narrow replacement before closing this report.
