## Summary

Keep open. Current main still hardcodes the 5 MiB media store, sandbox staging, and chat preflight caps, the config contract still does not accept `media.maxBytes`, and https://github.com/openclaw/openclaw/pull/40950 remains the open unmerged implementation path.

## What This Changes

Make the hardcoded 5 MiB media store and sandbox staging limit configurable for larger sandboxed attachments.

## Best Possible Solution

Land a narrow current-main change that adds bounded `media.maxBytes` support and threads it through store, sandbox staging, chat preflight, config schema/types/help/docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows sandboxed non-image attachments above 5 MiB are checked against `MEDIA_MAX_BYTES` in chat and staging, and a gateway test creates a 6 MiB PDF and asserts rejection.

## Solution Assessment

Yes for keeping this issue open. The linked open PR is the implementation candidate, but current main is not fixed and that PR must be reviewed, repaired, or replaced before this report closes.
