## Summary

Keep open. Current main still hardcodes the 5 MB media store, sandbox staging, and chat preflight cap, `media.maxBytes` is not accepted by config, and #40950 remains open as the unmerged closing path.

## What This Changes

Requests a configurable media staging/store byte limit so sandboxed agents can receive files larger than the current 5 MB cap.

## Best Possible Solution

Review and land #40950, or an equivalent narrow replacement, so the limit is schema-backed, documented, bounded, and threaded through store, sandbox staging, chat preflight, and tests.
