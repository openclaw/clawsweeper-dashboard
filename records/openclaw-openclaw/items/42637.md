## Summary

Keep this PR open: the exact lookup-key direction is useful, but the latest patch still has a tight-budget path that can drop skills without any truncation notice, and the supplied terminal proof is for an older head than the current branch.

## What This Changes

This PR changes skills prompt truncation to emit budgeted omitted skill lookup keys, retry compact rendering before dropping skills, and add focused skills prompt tests.

## Best Possible Solution

Reserve prompt budget for at least a minimal truncation and lookup-key notice before finalizing the included skill count, refresh latest-head real behavior proof, and leave broader skill priority or pinning work to https://github.com/openclaw/openclaw/issues/50677 and https://github.com/openclaw/openclaw/issues/50199.

## Reproduction Assessment

Yes. Source inspection shows current main can truncate the alphabetically sorted skills prompt and emit only a count-based warning, so omitted skills have no exact lookup key in the prompt; I did not run a local repro because this review is read-only.

## Solution Assessment

No, not yet. The JSON exact lookup-key approach is the right direction, but the patch should reserve notice space when `limit.truncated` is true instead of accepting a fitting skills block that leaves no warning or lookup-key budget.
