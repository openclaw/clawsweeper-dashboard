## Summary

Keep open: current main still emits only a generic skills truncation warning, and this PR addresses a valid discoverability gap, but the latest diff still breaks the configured prompt-size cap and can advertise sanitized names that `openclaw skills info` cannot resolve.

## What This Changes

The PR changes `src/agents/skills/workspace.ts` to include sanitized omitted skill names in the skills truncation warning with `openclaw skills info <name>` recovery guidance.

## Best Possible Solution

Land this PR or an equivalent patch after it budgets the final truncation note, emits exact recoverable identifiers or reversible data, clearly delimits omitted names as data, and covers count truncation, char truncation, tight budgets, and special-character skill names.

## Reproduction Assessment

Yes. Build a workspace skills prompt with more skills than `maxSkillsInPrompt` or a tight `maxSkillsPromptChars`; current main emits the generic warning, while the PR-specific lookup bug follows from exact `formatSkillInfo` matching against raw names or `skillKey`.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this diff appends the list outside the final prompt budget and uses altered display names for a command that requires exact lookup keys.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Budget omitted-name guidance before emitting the prompt:** `src/agents/skills/workspace.ts:914-917`
  - body: The new warning appends `omittedNames.join(", ")` after `applySkillsPromptLimits()` has already decided the skills body fits `maxSkillsPromptChars`. With many omitted skills, the final skills prompt can exceed the configured cap, so reserve this budget or cap and trim the emitted list before final rendering.
  - confidence: 0.93
- **[P2] Preserve exact omitted-skill lookup keys:** `src/agents/skills/workspace.ts:824`
  - body: `sanitizeSkillName()` strips characters from omitted names before the note tells the model to run `openclaw skills info <name>`. Since `formatSkillInfo()` resolves by exact raw name or `skillKey`, a skill named with stripped characters can be advertised under a different name and the recovery command will miss it.
  - confidence: 0.88
