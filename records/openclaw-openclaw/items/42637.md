## Summary

Keep open: current main still has the omitted-skills discoverability gap, and this PR is aimed at the right code path, but the latest diff still has a count-truncation edge that can emit no warning or lookup keys when the selected skills block already consumes the prompt cap.

## What This Changes

The PR updates skills prompt truncation to emit JSON-encoded omitted skill lookup keys and adds focused skills prompt tests for budgeted recovery and compact retry behavior.

## Best Possible Solution

Reserve room for at least a minimal truncation notice, and preferably lookup keys when any can fit, before finalizing the included skill count; keep the exact JSON `skillKey` recovery approach and refresh proof on the current head.

## Reproduction Assessment

Yes. Source inspection shows current main can truncate the alphabetically sorted skills prompt while exposing only an included/total warning, so omitted skills have no exact lookup key in the prompt.

## Solution Assessment

No, not yet. The JSON exact lookup-key approach is a maintainable fit, but the patch still needs to reserve notice budget for count-truncated prompts that otherwise fit exactly under `maxSkillsPromptChars`.
