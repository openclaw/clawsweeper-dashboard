## Summary

This PR addresses a real skill-discovery failure and has resolved several earlier budgeting and encoding defects, but the current capped notice still advertises only the alphabetically earliest omitted keys. Large catalogs can therefore continue hiding late relevant skills—the central problem described by the PR—and the supplied real behavior proof predates the current head.

## What This Changes

The PR adds budget-aware JSON lookup keys for omitted skills, retries compact prompt rendering before truncation, and expands regression coverage for prompt limits and exact CLI lookup.

## Best Possible Solution

Use one deterministic prompt-budget algorithm that preserves compact semantic descriptions, stays within the final assembled character cap, and advertises exact lookup keys sampled across the full omitted catalog, including its tail.

## Reproduction Assessment

Yes. Alphabetical ordering plus prefix truncation creates a deterministic source-level path where a late relevant skill is absent, and the supplied 80-skill output shows that only an early subset of omitted keys is advertised.

## Solution Assessment

No. Exact JSON lookup keys and final-prompt budgeting are appropriate primitives, but prefix-only key selection leaves the stated late-skill discoverability failure unresolved for large catalogs.
