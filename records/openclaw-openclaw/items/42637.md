## Summary

Keep open: the PR addresses a real current-main discoverability gap, but the latest one-file diff can exceed the configured skills prompt cap and can advertise sanitized names that `openclaw skills info` cannot resolve.

## What This Changes

The PR appends sanitized omitted skill names to the skills truncation warning and suggests `openclaw skills info <name>` for on-demand loading.

## Best Possible Solution

Land an equivalent patch that budgets the final truncation note, emits exact recoverable identifiers as bounded data, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Build a workspace prompt with more eligible skills than `maxSkillsInPrompt` or a tight `maxSkillsPromptChars`; current main emits only the generic warning, while the PR-specific failures follow from long omitted names and names containing stripped characters.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this implementation appends the list outside the final budget and changes names before telling the model to use an exact lookup command.
