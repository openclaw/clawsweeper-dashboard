## What This Changes

The PR changes workspace skills prompt truncation to emit JSON omitted skill lookup keys within `maxSkillsPromptChars` and adds regression coverage for prompt budgeting and compact-mode recovery.

## Best Possible Solution

Keep the exact JSON lookup-key approach, but make omitted-key selection deterministic across prefix and tail under the prompt budget, prefer at least one exact key under tight budgets, and require current-head prompt-rendering proof before merge.

## Reproduction Assessment

Yes. Source inspection of the current PR head shows the remaining failure: large omitted sets are serialized from `skillKeys.slice(0, included)`, so late omitted keys can still be absent from both the catalog and `<omitted_skill_keys>`.

## Solution Assessment

No, not yet. Exact JSON lookup keys are the right direction, but prefix-only selection and base-notice-first budget behavior leave the central recovery incomplete under large or tight-budget skill catalogs.
