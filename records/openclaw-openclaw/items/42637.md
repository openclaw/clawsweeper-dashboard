## Summary

Keep open. Current main still only emits the generic skills-truncation warning, so the PR targets a real remaining discoverability gap. The latest one-file branch is not merge-ready because it can exceed the documented skills prompt budget and can advertise sanitized names that `openclaw skills info` cannot resolve exactly; the new model-facing omitted-name surface also needs explicit containment.

## What This Changes

This PR changes `src/agents/skills/workspace.ts` so truncated skills produce sanitized omitted-name text in the skills truncation warning with `openclaw skills info <name>` recovery guidance.

## Best Possible Solution

Land this PR or an equivalent patch after it tracks omitted skills with a bounded final prompt budget, emits exact recoverable identifiers or reversible data for `skills info`, clearly delimits omitted names as data, and adds regression coverage for count truncation, char truncation, tight budgets, and special-character skill names.

## Reproduction Assessment

Yes. Current main can be reproduced by building a skills prompt with more skills than the configured count or char limit and observing the generic warning at `src/agents/skills/workspace.ts:897`; the PR-specific lookup issue follows from the exact-name match in `src/cli/skills-cli.format.ts:171`.

## Solution Assessment

No, not as currently proposed. Listing omitted skills is the right narrow direction, but the implementation should reserve budget for the final warning and preserve lookup-safe exact identifiers instead of concatenating altered names into plain prompt text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Budget omitted-name guidance before emitting the prompt:** `src/agents/skills/workspace.ts:913-916`
  - body: The new warning appends every omitted name after `applySkillsPromptLimits()` has already decided the skills body fits `maxSkillsPromptChars`. With many omitted skills, `omittedNames.join(', ')` can add unbounded text, so the final skills prompt can exceed the documented cap; reserve this budget or cap and trim the emitted list before final rendering.
  - confidence: 0.91
- **[P2] Preserve exact omitted-skill lookup keys:** `src/agents/skills/workspace.ts:826`
  - body: `sanitizeSkillName()` strips characters from omitted names before the note tells the model to run `openclaw skills info <name>`. Since `formatSkillInfo()` resolves by exact raw name or `skillKey`, a skill named `skill-<xml>` would be advertised as `skill-xml` and the recovery command would miss it; keep a prompt-safe encoding while preserving a reversible lookup key, or advertise `skillKey`.
  - confidence: 0.87
