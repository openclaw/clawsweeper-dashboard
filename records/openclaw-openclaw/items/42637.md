## Summary

Keep open: the PR addresses a real skill discoverability gap, but the latest one-file diff still regresses the configured skills prompt budget and can advertise lookup names that `openclaw skills info` cannot resolve.

## What This Changes

The PR changes the skills workspace prompt builder to include sanitized names of omitted skills in the truncation warning.

## Best Possible Solution

Land this PR or an equivalent patch after the final note is budgeted, omitted skills are represented as clearly delimited data, exact recoverable identifiers are preserved, and regression tests cover count truncation, char truncation, tight budgets, and special-character names.

## Reproduction Assessment

Yes. Build a workspace skills prompt with more skills than maxSkillsInPrompt or a tight maxSkillsPromptChars; current main emits the generic warning, and the PR-specific failures follow from the unbudgeted omittedList append plus exact formatSkillInfo lookup.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this diff appends the list outside the final prompt budget and changes names before recommending an exact lookup command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Budget omitted-name guidance before emitting the prompt:** `src/agents/skills/workspace.ts:914-917`
  - body: The warning appends `omittedNames.join(", ")` after `applySkillsPromptLimits()` has already selected a skills body that fits `maxSkillsPromptChars`. With many omitted skills, the final prompt can exceed the configured cap, so reserve this budget or cap and trim the emitted list before final rendering.
  - confidence: 0.93
- **[P2] Preserve exact omitted-skill lookup keys:** `src/agents/skills/workspace.ts:826`
  - body: `sanitizeSkillName()` strips characters before the note tells the model to run `openclaw skills info <name>`. Since `formatSkillInfo()` resolves by exact raw name or `skillKey`, a skill whose name contains stripped characters can be advertised under a different name and the recovery command will miss it.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: No CI, dependency, or supply-chain changes were found, but the diff adds locally supplied skill names to a model-facing prompt without a bounded, clearly data-delimited representation.

Concerns:

- **[low] Contain omitted skill names as prompt data:** `src/agents/skills/workspace.ts:914`
  - body: Skill names come from installed skill frontmatter and are now surfaced in plain system-prompt text. Removing control characters, backticks, and angle brackets reduces markup breakout, but instruction-like names are still not clearly encoded or delimited as data; use a bounded structured representation or get explicit maintainer acceptance of the prompt-surface tradeoff.
  - confidence: 0.78
