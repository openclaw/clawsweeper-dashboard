## Summary

Keep open. The underlying skills discoverability problem is real and this PR is useful, but the current head still appends omitted names outside the configured prompt budget and can advertise names that `openclaw skills info` cannot resolve.

## What This Changes

The PR modifies `src/agents/skills/workspace.ts` to append sanitized omitted skill names to the skills truncation warning and suggest `openclaw skills info <name>`.

## Best Possible Solution

Land this PR or an equivalent patch after the omitted-skill note is included in the final prompt budget, uses exact recoverable lookup identifiers or structured data, and has regression coverage for count truncation, char truncation, tight budgets, and special-character skill names.

## Reproduction Assessment

Yes. A workspace with more eligible skills than `maxSkillsInPrompt` or a tight `maxSkillsPromptChars` reproduces truncation; a PR-head case with many omitted names or an omitted name containing stripped characters reproduces the two patch regressions by inspection of the current diff and lookup code.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this implementation appends the recovery text after the budget decision and mutates names used by an exact-match lookup command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Budget omitted-skill guidance before emitting the prompt:** `src/agents/skills/workspace.ts:914-917`
  - body: The omitted-name sentence is appended after `applySkillsPromptLimits()` has already selected a skills body that fits `maxSkillsPromptChars`, so a workspace with many omitted skills can emit a prompt above the configured cap. Reserve this note in the budget or trim it before final rendering.
  - confidence: 0.93
- **[P2] Preserve exact lookup keys for omitted skills:** `src/agents/skills/workspace.ts:824-827`
  - body: `sanitizeSkillName()` strips characters before the note tells the model to run `openclaw skills info <name>`, but `formatSkillInfo()` matches exact raw names or `skillKey`. A skill whose omitted name contains stripped characters can be advertised under a different token and then fail to load.
  - confidence: 0.88
