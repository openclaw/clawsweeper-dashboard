## Summary

Keep open; the PR targets a valid skills discoverability gap, but the latest diff still breaks the prompt-budget contract and can advertise omitted skills under names that the recovery command cannot resolve.

## What This Changes

The PR modifies `src/agents/skills/workspace.ts` so truncated skills prompts append sanitized omitted skill names with `openclaw skills info <name>` recovery guidance.

## Best Possible Solution

Land this PR or an equivalent patch after the final truncation note is budgeted, omitted skills are represented as bounded prompt data with exact recoverable lookup identifiers, and tests cover count truncation, char truncation, tight budgets, and special-character skill names.

## Reproduction Assessment

Yes. Build a workspace skills prompt with more eligible skills than `maxSkillsInPrompt` or with a tight `maxSkillsPromptChars`; current `main` emits the generic warning, while the PR-specific bugs follow with many omitted names or an omitted skill whose name contains stripped characters such as `<` or `>`.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this patch appends the recovery list outside the final prompt budget and mutates the names used by an exact-match lookup command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Budget omitted-skill guidance before emitting the prompt:** `src/agents/skills/workspace.ts:914-917`
  - body: The omitted-name sentence is appended after `applySkillsPromptLimits()` has selected a skills body that fits `maxSkillsPromptChars`, so a workspace with many omitted skills can emit a prompt above the configured cap. Reserve this note in the budget or cap and trim it before final rendering.
  - confidence: 0.93
- **[P2] Preserve exact lookup keys for omitted skills:** `src/agents/skills/workspace.ts:824`
  - body: `sanitizeSkillName()` strips characters before the note tells the model to run `openclaw skills info <name>`, but `formatSkillInfo()` matches exact raw names or `skillKey`. A skill whose omitted name contains stripped characters can be advertised under a different token and then fail to load.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: No CI, dependency, or supply-chain changes were found, but the diff adds locally supplied skill names to a model-facing prompt without a bounded, clearly data-delimited representation.

Concerns:

- **[low] Contain omitted skill names as prompt data:** `src/agents/skills/workspace.ts:914`
  - body: Skill names come from installed skill metadata and are now surfaced in plain system-prompt text. Stripping control characters, backticks, and angle brackets reduces breakout risk, but instruction-like names are still not encoded or clearly delimited as data; use a bounded structured representation or get explicit maintainer acceptance of this prompt-surface tradeoff.
  - confidence: 0.78
