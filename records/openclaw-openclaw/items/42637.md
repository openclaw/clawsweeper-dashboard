## Summary

Keep open. PR head still uses a fixed compact-warning reserve before final rendering, so a compact skills catalog that would fit with the real notice can still lose tail skills under tight prompt budgets.

## What This Changes

The PR changes skills prompt rendering to emit budgeted JSON omitted-skill lookup keys and adds compact-format regression tests.

## Best Possible Solution

Use exact final-render compact budgeting before reducing the included skill count, while keeping JSON omitted-key recovery only for skills that truly do not fit.

## Reproduction Assessment

## Solution Assessment

No. The PR is in the right owner path, but fixed-reserve budgeting is only a plausible mitigation; the final renderer should decide exact fit before any irreversible skill-count reduction.
