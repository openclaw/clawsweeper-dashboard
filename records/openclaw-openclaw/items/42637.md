## Summary

Keep open: current main still emits only the generic skills truncation warning, and the PR addresses a real discoverability gap, but the latest diff is not mergeable because it appends unbudgeted prompt text and can advertise names that `openclaw skills info <name>` cannot resolve exactly.

## What This Changes

The PR adds sanitized omitted skill names to the workspace skills truncation warning and tells the model to inspect them with `openclaw skills info <name>`.

## Best Possible Solution

Land an equivalent patch that reserves prompt budget for a compact, data-delimited list of exact recoverable skill identifiers and covers count, character, tight-budget, and special-character cases.

## Reproduction Assessment

Yes, at source level. Building a skills prompt with `maxSkillsInPrompt` below the eligible count or a tight `maxSkillsPromptChars` follows the current-main path that emits only `included N of M`, while the PR’s bugs follow directly from appending names after limit selection and stripping exact lookup characters.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this diff does not keep the final prompt inside the configured cap and does not preserve the exact identifiers needed by the advertised lookup command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Budget omitted-skill guidance before emitting the prompt:** `src/agents/skills/workspace.ts:916-919`
  - body: The omitted-name sentence is appended after `applySkillsPromptLimits()` has already selected a catalog sized against `maxSkillsPromptChars`. With many omitted skills, the final snapshot can exceed the configured cap, so reserve or trim this note inside the final budget.
  - confidence: 0.93
- **[P2] Preserve exact lookup keys for omitted skills:** `src/agents/skills/workspace.ts:826`
  - body: `sanitizeSkillName()` removes characters from the advertised name, but `openclaw skills info <name>` resolves by exact raw name or `skillKey`. An omitted skill named `foo<bar` would be shown as `foobar`, making the suggested lookup fail.
  - confidence: 0.89
