## Summary

This PR is a useful targeted fix, but it should stay open because the latest patch can still return a truncated skills prompt with no warning or lookup keys under a tight count/character budget, and the real behavior proof in the body is for an older head than the current force-pushed commit.

## What This Changes

The PR changes skills prompt truncation to emit omitted skill lookup keys within the prompt budget and adds tests for exact `openclaw skills info` recovery and compact prompt retry behavior.

## Best Possible Solution

Keep the exact JSON lookup-key direction, reserve prompt space for at least a minimal truncation notice before accepting a truncated skills block, refresh latest-head real behavior proof, and leave broader skill priority or pinning work to the related issues.

## Reproduction Assessment

Yes. Source inspection shows current main can truncate the alphabetically sorted skills prompt and emit only a count-based warning, so omitted skills have no exact lookup key in the prompt; I did not run local tests because this review is read-only.

## Solution Assessment

No, not yet. The lookup-key implementation is the right maintainable direction, but the patch must reserve notice space when `limit.truncated` is true instead of accepting a fitting skills block that leaves no warning or lookup-key budget.

## Review Findings
