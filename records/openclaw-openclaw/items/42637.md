## Summary

Keep open: current main still has the omitted-skill discoverability gap, but the latest patch leaves prompt-budget and exact-identifier regressions plus a missing changelog entry.

## What This Changes

The PR changes skills prompt truncation to collect sanitized names of omitted skills and append them with a `skills info` recovery hint.

## Best Possible Solution

Land an equivalent narrow patch that keeps the final prompt within the configured cap, exposes exact recoverable omitted-skill identifiers as bounded prompt data, adds regression coverage, and includes the required changelog entry.

## Reproduction Assessment

Yes, at source level. With `maxSkillsInPrompt` below the eligible count or `maxSkillsPromptChars` tight, current main emits only the generic truncation note, and the PR's added sentence is appended after budget selection.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this implementation needs to budget the added guidance and represent exact recoverable identifiers instead of sanitized display strings.
