## Summary

Keep open: current main still emits only a generic skills truncation warning, and this PR's direction is useful, but the current head is not ready to merge because it appends omitted names outside the configured prompt budget and can advertise names that no longer match `openclaw skills info` lookup.

## What This Changes

The PR changes the workspace skills prompt truncation note to include sanitized omitted skill names and a `openclaw skills info <name>` recovery hint.

## Best Possible Solution

Land an equivalent patch that keeps the final prompt within the configured cap, surfaces bounded exact recoverable omitted-skill identifiers as prompt data, adds regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes, at source level. Current main with a low `maxSkillsInPrompt` or tight `maxSkillsPromptChars` emits only `included N of M`, and the PR's remaining failures follow directly from appending names after limit selection and from exact CLI lookup semantics.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this implementation needs to budget the note in the final prompt and preserve exact lookup keys rather than sanitized display-only names.
