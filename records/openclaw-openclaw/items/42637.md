## Summary

Keep open. The PR addresses a real skills discoverability gap, but the current head still appends omitted names outside the prompt budget and can advertise sanitized names that do not work with the exact `openclaw skills info` lookup path.

## What This Changes

The PR adds sanitized omitted skill names to the skills truncation warning and tells the model to run `openclaw skills info <name>` for on-demand loading.

## Best Possible Solution

Land an equivalent patch that reserves prompt budget for a compact, data-delimited list of exact recoverable skill identifiers and covers count, character, tight-budget, and special-character cases.

## Reproduction Assessment

Yes. Build a skills prompt with `maxSkillsInPrompt` below the eligible skill count or a tight `maxSkillsPromptChars`; current main emits the generic warning, while this PR’s remaining bugs reproduce with many omitted names or a name containing stripped characters such as `<` or `>`.

## Solution Assessment

No. Listing omitted skills is the right narrow direction, but this patch appends the list after budget selection and mutates the identifier that the advertised lookup command needs to resolve exactly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Budget omitted-skill guidance before emitting the prompt:** `src/agents/skills/workspace.ts:916-919`
  - body: The omitted-name sentence is appended after `applySkillsPromptLimits()` has already selected a catalog sized against `maxSkillsPromptChars`. With many omitted skills, the final snapshot can exceed the configured cap, so reserve or trim this note inside the final budget.
  - confidence: 0.93
- **[P2] Preserve exact lookup keys for omitted skills:** `src/agents/skills/workspace.ts:826`
  - body: `sanitizeSkillName()` removes characters from the advertised name, but `openclaw skills info <name>` resolves by exact raw name or `skillKey`. An omitted skill named `foo<bar` would be shown as `foobar`, making the suggested lookup fail.
  - confidence: 0.89
