## Summary

Keep open: this PR targets a valid current-main bug, but the patch appears to reject the fixed path under the default session visibility policy because it rewrites `current` to `runSessionKey` before the existing self/tree visibility shortcut runs.

## What This Changes

The PR threads `runSessionKey` through embedded tool creation so `session_status({sessionKey:"current"})` can report the live run session, with a regression test and changelog entry.

## Best Possible Solution

Keep the `runSessionKey` seam, but preserve the original `sessionKey: "current"` request as a self/current visibility target while resolving the displayed status entry to the live run session.

## Reproduction Assessment

Yes. Source inspection shows current main only gives `session_status` the sandbox/policy session key, and the linked issue provides a concrete Telegram direct mismatch between `agent:main:main` and the stale direct key; I did not run tests because this sweep is read-only.

## Solution Assessment

No. Threading `runSessionKey` is the right direction, but the current patch applies it before visibility checks, so the fixed `current` lookup can become an ordinary same-agent lookup and fail under `tree` or `self` visibility.
