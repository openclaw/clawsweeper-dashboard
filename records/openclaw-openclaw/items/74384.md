## Summary

Keep open. Current main still makes the reported MemOS symptom source-reproducible: profile-derived allowlists gate required plugin `api.registerTool()` entries by tool/plugin name, and the `coding` profile only includes `memory_search` and `memory_get` from the reported set.

## What This Changes

Expose all required tools registered by third-party plugins through `api.registerTool()` after the 4.26 plugin SDK restructuring.

## Best Possible Solution

Define the supported contract for required plugin tools under restrictive profiles, then align resolver behavior, diagnostics, docs, and regression coverage around that contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register required third-party tools under `tools.profile: "coding"`; only names matching the profile allowlist, plugin id, `group:plugins`, or `*` survive resolution and final policy filtering.

## Solution Assessment

Unclear because there is no proposed patch and the remaining choice is policy-sensitive. Maintainers should decide whether required third-party tools bypass restrictive profiles, require explicit grants, or need clearer docs and diagnostics.
