## Summary

Keep open: the PR is a focused performance idea, but the broad `CI=true` guard is not merge-ready because it skips the current stale-output cleanup contract before tsdown runs with `--no-clean` in artifact-cache paths.

## What This Changes

The PR wraps the four pre-tsdown cleanup calls in `scripts/tsdown-build.mjs` with `if (process.env.CI !== "true")` so CI skips them.

## Best Possible Solution

Keep CI stale-output cleanup before `--no-clean`, then optimize only a measured safe cleanup sub-step or runner path with focused tests and Windows/Testbox proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows `CI=true` bypasses all four cleanup calls before the wrapper invokes tsdown with `--no-clean`, and current workflows show an artifact path that can restore old `dist/` trees before rebuilding.

## Solution Assessment

No. This is a plausible performance shortcut, but the best fix keeps the stale-output cleanup contract and targets only a proven redundant slow path.
