## What This Changes

This PR adds shared and Telegram-scoped `ackReactionTiming` config, defers Telegram ack/status reactions until agent run start when configured, updates docs/schema/help/types, and adds Telegram-focused tests.

## Best Possible Solution

Revise the PR on current main, keep the runtime change inside the Telegram plugin boundary, preserve supported-emoji validation for immediate and deferred acks, regenerate the public config artifacts, add the required changelog entry, fix the docs wording, and coordinate the shared timing type/schema with the Slack and Discord companion PRs.

Not applicable as a feature PR rather than a bug report. The review findings are source-reproducible from PR-head and current-main inspection: generated outputs/changelog are absent, and the deferred path stores and sends the raw configured ack value.

## Solution Assessment

No, not yet. The opt-in `received` versus `run-start` design is narrow and maintainable, but this patch needs validation-preserving deferred ack handling, regenerated artifacts, changelog coverage, docs correction, rebase, and green checks before it is the best merge path.
