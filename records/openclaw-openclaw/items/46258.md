## Summary

Keep open. Current main still lacks Telegram `ackReactionTiming`, so the contribution is useful, and the Greptile-noted deferred `setThinking()` gap appears addressed on the PR head; however the patch is not merge-ready because it bypasses current Telegram ack emoji validation in the deferred path and leaves public config artifacts, changelog coverage, and a docs wording issue unresolved.

## What This Changes

This PR adds shared and Telegram-scoped `ackReactionTiming` config, defers Telegram ack/status reactions until agent run start when configured, updates docs/help/schema types, and adds Telegram-focused tests.

## Best Possible Solution

Revise this PR on current main, keep the runtime behavior inside the Telegram plugin boundary, preserve supported-emoji validation for immediate and deferred simple acks, regenerate all public config artifacts, add the required changelog entry, fix the channel/account docs wording, and coordinate the shared timing type/schema with the Slack and Discord companion PRs.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The review findings are source-reproducible from the PR head and current-main comparison: generated outputs and changelog are absent, and the deferred path stores and sends the raw configured ack value.

## Solution Assessment

No, not yet. The opt-in `received` versus `run-start` direction is narrow and maintainable, but the current patch needs validation-preserving deferred ack handling, regenerated artifacts, changelog coverage, docs correction, rebase, and green checks before it is the best merge path.
