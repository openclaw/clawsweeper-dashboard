Current main still lacks the requested ack-reaction timing contract, and this PR remains useful but not merge-ready: its head fixes the reviewed status-reaction run-start gap, but generated config artifacts are stale and the deferred Telegram simple-ack path sends the raw configured emoji instead of the validated Telegram reaction emoji.

Revise the PR on current main, keep the runtime behavior inside the Telegram plugin boundary, preserve supported-emoji validation for immediate and deferred simple acks, regenerate the public config schema/channel metadata/docs baseline, and coordinate the shared timing contract with the Slack and Discord companion PRs.
