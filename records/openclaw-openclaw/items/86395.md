## Summary

Keep open: current main and the latest release still contain the reported Discord final-error/draft-cleanup path, but the exact desired terminal behavior conflicts with adjacent tests/docs and an open overlapping PR, so this needs maintainer policy review rather than cleanup closure.

## What This Changes

Issue requests that Discord progress draft previews remain visible after a successfully delivered final error reply instead of being deleted by cleanup.

## Best Possible Solution

Decide the Discord progress-draft terminal policy for standalone final error replies, then implement it narrowly with regression coverage that does not change partial-preview error cleanup unless that behavior is intentionally revised.

## Reproduction Assessment

Yes from source inspection: current main skips `markFinalReplyDelivered()` for final error payloads, and cleanup clears drafts when that flag remains false. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

Unclear as proposed: the one-line `if (isFinal)` hotfix would also affect partial-preview error cleanup that current tests expect to clear. The safer solution is a maintainer-approved progress-mode policy and a focused test for the standalone final-error case.
