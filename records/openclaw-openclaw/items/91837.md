## Summary

Keep this PR open: it targets a real memory-core CLI/QMD cleanup bug, but the patch introduces a blocking JSON-output ordering problem and only has mocked/runtime-harness proof for a live process-exit issue.

## What This Changes

The PR changes `memory search --json` to await short-term recall recording before writing JSON, and updates the CLI regression test to assert the recall entry is present immediately after the JSON search command returns.

## Best Possible Solution

Keep the PR open, but change the JSON search path so results are emitted promptly while recall cleanup is settled or bounded separately, then add lock-contention regression coverage plus real or stubbed QMD CLI process-exit proof.

## Reproduction Assessment

Yes for the PR defect at source level: the new JSON branch awaits recall recording before `writeJson`, and the recall lock can wait up to 10 seconds. The linked QMD exit issue still needs real or faithful stubbed process-lifecycle proof after the patch.

## Solution Assessment

No. The PR is a plausible cleanup attempt, but awaiting best-effort recall before JSON output is not the best fix because it can delay or timeout successful scripted output; the cleanup should be bounded or moved without blocking result delivery.
