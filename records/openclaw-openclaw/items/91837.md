## Summary

Keep open: the branch targets a real memory/QMD CLI timeout report, but it is not merge-ready because the new JSON path awaits best-effort recall recording before output, which can wait on the 10s short-term recall lock and preserve caller timeouts; the after-fix proof is still harness-only rather than live or faithful QMD process-lifecycle proof.

## What This Changes

The PR changes memory-core CLI JSON search to await short-term recall recording before writing results and updates the CLI regression test to assert recall storage immediately after JSON search returns.

## Best Possible Solution

Emit JSON promptly and settle or bound recall cleanup without blocking result delivery, then prove the QMD CLI path exits cleanly with live or faithful stubbed process-lifecycle evidence.

## Reproduction Assessment

Yes for the introduced PR defect at source level: the new JSON branch awaits recall before `writeJson`, and `withShortTermLock` can wait up to 10 seconds. The linked QMD exit issue still needs live or faithful stubbed process-lifecycle proof after the patch.

## Solution Assessment

No. Awaiting best-effort recall before output is only a plausible cleanup, not the best fix; cleanup should be bounded or moved after result delivery and verified against the QMD process lifecycle.
