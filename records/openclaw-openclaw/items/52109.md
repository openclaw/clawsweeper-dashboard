## Summary

Keep open: current main still lacks the every-schedule refire floor, but this PR branch predates the #66019/#66113 cron fixes and would restore a short synthetic retry when cron next-run resolution is undefined.

## What This Changes

The PR changes cron scheduler completion handling to clamp successful `every` schedule next runs behind a 2-second refire floor when the computed next run is due or past, and adds regression tests for those interval edge cases.

## Best Possible Solution

Rebase the branch onto current main and implement the `every`-schedule clamp while preserving `resolveCronNextRunWithLowerBound` semantics for cron schedules.

## Reproduction Assessment

Yes. Source inspection shows current main can persist an `every` job's computed next run at the just-finished boundary, and the PR body includes live Docker before/after values for the same tight-refire shape.

## Solution Assessment

No, not as currently proposed. The direction is narrow and maintainable, but the implementation must reuse or preserve current cron unresolved-next-run clearing instead of falling back to a 2-second retry.
