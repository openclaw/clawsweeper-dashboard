## What This Changes

The PR changes cron completion scheduling so successful every-schedule jobs clamp immediate or overrun next-runs behind `MIN_REFIRE_GAP_MS`, and adds focused regression tests for those cases.

## Best Possible Solution

Keep the every-schedule floor, but implement it on top of current main so cron schedules continue using `resolveCronNextRunWithLowerBound` and every schedules only clamp defined next-runs that land at or before `endedAt`.

## Reproduction Assessment

Yes. Source inspection shows current main can store an every-schedule completion result at the completion boundary, and the PR body includes copied live Docker before/after output for the overrun case.

## Solution Assessment

No, not at the current head. The intended every-schedule floor is the right fix shape, but the patch must preserve current main‚Äôs unresolved cron behavior instead of synthesizing a retry when cron next-run resolution returns `undefined`.
