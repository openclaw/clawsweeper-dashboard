## Summary

This PR is a plausible focused bug fix, but the current head still has a blocking scheduler regression against current main: it reintroduces a synthetic cron retry when cron next-run resolution is undefined.

## What This Changes

The PR changes cron completion scheduling and adds regression tests so successful every-schedule jobs clamp due-now or overrun next runs behind `MIN_REFIRE_GAP_MS`.

## Best Possible Solution

Repair the scheduler branch so cron schedules keep `resolveCronNextRunWithLowerBound`, while every schedules clamp only defined next-runs that land at or before `endedAt`.

## Reproduction Assessment

Yes, at source level. Current main stores every-schedule completion results directly from `computeJobNextRunAtMs`, and that computation can return the completion boundary; the PR body also includes copied live Docker before/after output for the overrun case.

## Solution Assessment

No, not at the current PR head. The intended every-schedule floor is the right fix shape, but the patch must preserve current main's unresolved-cron behavior instead of synthesizing a cron retry when next-run resolution returns `undefined`.
