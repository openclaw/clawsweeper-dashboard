## Summary

Keep open: the bug is real and this branch is close, but the current PR head would regress the current-main unresolved cron next-run clearing shipped through https://github.com/openclaw/openclaw/pull/66083.

## What This Changes

The PR clamps successful `every` schedule next runs behind `MIN_REFIRE_GAP_MS` when they land at or before `endedAt`, and adds regression tests for interval jobs that finish on or after their next boundary.

## Best Possible Solution

Apply the refire floor only to successful `every` runs whose computed next run is at or before `endedAt`, while preserving current-main cron unresolved-next-run clearing and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main can persist an `every` job's computed next run at `endedAt`, and the PR body includes live Docker before/after evidence for the same tight-refire symptom.

## Solution Assessment

No, not as currently proposed. The intended `every` clamp is narrow, but this branch also restores the cron undefined-next-run fallback that current main removed in https://github.com/openclaw/openclaw/pull/66083.
