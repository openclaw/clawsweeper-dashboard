## Summary

Keep open: the underlying `every` schedule overrun bug is still valid on current `main`, but this PR still regresses current unresolved-cron behavior by scheduling a 2-second refire when cron next-run computation returns `undefined`.

## What This Changes

The PR modifies cron completion scheduling to floor overrun `every` jobs and adds focused regression coverage in a new timer test file.

## Best Possible Solution

Apply an `every`-only overrun floor on top of current `main` while continuing to route cron completions through `resolveCronNextRunWithLowerBound` when next-run resolution is unresolved.

## Reproduction Assessment

Yes. Source inspection shows current `main` still stores `naturalNext` directly for successful `every` completions, and the PR body provides copied live Docker before/after output for the tight refire loop.

## Solution Assessment

No. The target behavior is the right narrow fix, but this branch is not the best implementation because it bypasses current `main`'s unresolved-cron helper and revives a 2-second cron refire path.
