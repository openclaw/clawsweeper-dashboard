## What This Changes

The PR changes cron timer completion handling and adds regression tests so every-schedule jobs get a 2-second refire floor when their computed next run lands at or before the run completion time.

## Best Possible Solution

Repair or replace the branch so unresolved cron schedules remain unscheduled while defined every-schedule next-runs at or before endedAt are clamped behind MIN_REFIRE_GAP_MS, with focused regression coverage.

## Reproduction Assessment

## Solution Assessment

No, not at the current PR head. The intended every-schedule floor is the right shape, but the patch must preserve current main's resolveCronNextRunWithLowerBound behavior for unresolved cron schedules.
