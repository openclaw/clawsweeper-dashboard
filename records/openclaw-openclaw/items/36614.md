## Summary

Keep this open: current main still contains the write-side path where direct-message channel record payloads target the main session and `recordInboundSession` persists channel-specific `deliveryContext` to that target before dispatch.

## What This Changes

Prevent inbound direct messages under `session.dmScope: per-channel-peer` from writing channel-specific last-route delivery context onto the shared main session.

## Best Possible Solution

Use the shared inbound last-route policy when building channel `updateLastRoute` payloads so session-scoped DMs write to their isolated session and only shared-main DMs write to `agent:main:main`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path. With `session.dmScope: per-channel-peer`, iMessage resolves an isolated route session, then passes `mainSessionKey` to `updateLastRoute`, and the recorder writes delivery context to that main-session target before dispatch.

## Solution Assessment

Yes, preventing the write at the source is the narrow maintainable fix. A later session-initialization cleanup is useful migration behavior but is not the best solution for the record-stage contamination path.
