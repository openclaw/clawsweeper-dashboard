## Summary

Keep open. Current main still permits the write-side contamination described here: several inbound DM paths still pass the main session as the last-route target while the central recorder writes channel delivery metadata to that supplied target; the merged #38418 only addressed the read-side WebChat inheritance path, and the direct fix candidates #36625 and #40233 are closed unmerged.

## What This Changes

Prevent inbound direct-message last-route updates under `session.dmScope: per-channel-peer` from writing channel-specific delivery metadata onto the main session.

## Best Possible Solution

Honor `route.lastRoutePolicy` for all inbound last-route writers so scoped DM delivery metadata stays on the scoped session while shared-main behavior remains unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level path exists on current main: `per-channel-peer` can produce a scoped route session with `lastRoutePolicy: "session"`, but iMessage and other DM writers still pass the main session to `updateLastRoute`, and `recordInboundSession` writes delivery metadata to that supplied target.

## Solution Assessment

No patch is currently open. The maintainable repair is to make affected direct-message writers use `resolveInboundLastRouteSessionKey` or an equivalent shared boundary, rather than suppressing all cross-target updates blindly.

## Repair Work Prompt

Fix the write-side route contamination tracked by https://github.com/openclaw/openclaw/issues/36614 on current main. For inbound direct-message routes where `route.lastRoutePolicy === "session"` under `session.dmScope` values such as `per-channel-peer`, `per-peer`, or `per-account-channel-peer`, persist last-route delivery metadata to the scoped route session instead of `route.mainSessionKey`. Use `resolveInboundLastRouteSessionKey` or an equivalent shared helper in affected bundled channel/plugin writers, including iMessage, Slack prepared/direct paths, Signal, LINE, Discord component/direct paths, Matrix, and Mattermost; audit other direct `updateLastRoute` callers. Preserve existing `dmScope: "main"`, group, channel, and thread behavior. Update `src/channels/session.test.ts` so it no longer blesses cross-target delivery-context writes that contaminate the main session, add focused regression coverage in existing channel tests where the update target can be asserted, and add a single-line user-facing `CHANGELOG.md` fix entry unless the final change is purely internal test wiring. Do not add a config option, change plugin boundaries, or alter unrelated WebChat read-side routing.
