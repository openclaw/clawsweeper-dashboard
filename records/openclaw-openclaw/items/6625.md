## Summary

Keep open. Current main and v2026.6.8 still implement hard sub-agent run timeouts and post-timeout recovery, but not the requested configurable pre-timeout warning that lets the child checkpoint before termination.

## What This Changes

The issue requests a configurable pre-timeout warning for sub-agents so they can checkpoint before `runTimeoutSeconds` kills the run.

## Best Possible Solution

Define one lifecycle-owned pre-timeout child notification contract shared by native and ACP sub-agent runs, with focused tests and docs while preserving existing post-timeout recovery.

## Reproduction Assessment

Yes, at the source level: current main has hard sub-agent deadlines and post-timeout handling, but no warning input, config key, or scheduled child notification before termination. I did not run a live timeout scenario during this read-only review.

## Solution Assessment

Unclear. A pre-timeout child notification directly targets the reported loss mode, but the suggested config/API shape and native/ACP delivery semantics still need maintainer design.
