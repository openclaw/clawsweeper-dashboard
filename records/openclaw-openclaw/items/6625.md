## Summary

Keep open. Current main and v2026.6.8 still support hard sub-agent run timeouts and post-timeout handling, but not the requested pre-timeout warning/grace message that lets the child checkpoint before termination.

## What This Changes

Request a configurable pre-timeout warning/grace period for sub-agents before `runTimeoutSeconds` kills the run.

## Best Possible Solution

Define one lifecycle-owned pre-timeout child notification contract shared by native and ACP sub-agent runs, with conservative defaults or explicit opt-in, focused tests, and docs while preserving post-timeout recovery.

## Reproduction Assessment

Yes, at the source level: current main has hard sub-agent deadlines and post-timeout handling, but no warning input, config key, or scheduled child notification before termination. I did not run a live timeout scenario during this read-only review.

## Solution Assessment

Unclear. A pre-timeout child notification directly targets the reported loss mode, but maintainers still need to choose the API/config shape, default policy, and native/ACP delivery semantics.
