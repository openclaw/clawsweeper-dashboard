## Summary

Keep open. Current main still has hard sub-agent run timeouts plus post-timeout partial-progress reporting, but no configurable pre-timeout warning/grace message for native or ACP sub-agent runs.

## What This Changes

Request a configurable grace-period warning before a sub-agent reaches `runTimeoutSeconds` so it can checkpoint progress before termination.

## Best Possible Solution

Add first-class core lifecycle support for a documented grace/warning setting and optional per-spawn override, schedule it before `runTimeoutSeconds`, deliver it safely to native and ACP child sessions, and preserve existing post-timeout partial-progress recovery.

## Reproduction Assessment

Yes. The missing feature is reproducible by source inspection: the tool schema, strict config schema, native spawn path, ACP spawn path, tests search, and docs all expose only hard run timeouts, not a pre-timeout warning surface.

## Solution Assessment

Unclear for implementation details. The requested behavior is directionally right, but the maintainable solution should be a designed lifecycle warning path rather than a manual steer/send workaround that could abort, restart, or race the child run.
