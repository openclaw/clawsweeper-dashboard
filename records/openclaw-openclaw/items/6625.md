## Summary

Keep open. Current main still has hard sub-agent run timeouts plus post-timeout partial-progress recovery, but it does not expose or deliver a configurable pre-timeout warning/grace message for native or ACP sub-agent runs.

## What This Changes

Request a configurable pre-timeout warning for spawned sub-agents so they can checkpoint before `runTimeoutSeconds` aborts the run.

## Best Possible Solution

Add core sub-agent lifecycle support for a documented grace/warning setting and optional per-spawn override, schedule the warning before `runTimeoutSeconds`, deliver it safely to native and ACP child sessions, preserve existing post-timeout partial-progress recovery, and cover the config, tool schema, runtime, and docs paths with focused tests.

## Reproduction Assessment

Yes. The missing feature is reproducible by static inspection of current main: the public tool schema, strict config schema, native spawn path, ACP spawn path, and docs all expose/pass only `runTimeoutSeconds`, with no pre-timeout warning surface.

## Solution Assessment

Unclear for implementation details. The requested behavior is directionally the right product fix, but the maintainable solution should be designed as a first-class lifecycle warning path rather than reusing manual steer behavior that aborts/restarts the child run.
