## Summary

Keep open: current main still has configurable hard sub-agent run timeouts and post-timeout partial-progress recovery, but no configurable pre-timeout warning/grace mechanism for native or ACP child runs.

## What This Changes

Request a configurable warning delivered to a sub-agent shortly before `runTimeoutSeconds` kills the run so the child can checkpoint progress.

## Best Possible Solution

Define one documented deadline-relative warning contract and deliver it through shared native and ACP child-session control paths while preserving existing hard-timeout and post-timeout recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main gives high confidence that no pre-timeout warning/grace surface is implemented.

## Solution Assessment

Unclear until maintainers settle the public config/tool API, default lead time, and child-control semantics. A shared native/ACP warning path is safer than ad hoc prompt injection at one spawn call site.
