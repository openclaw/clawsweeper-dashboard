## Label Justifications

- `P2`: The feature addresses meaningful but bounded loss of in-progress sub-agent work and requires lifecycle design rather than an emergency correction.
- `impact:data-loss`: A hard timeout can discard child work that was never persisted or returned to the requester.
- `impact:session-state`: The requested warning is intended to let the child checkpoint accumulated session progress before termination.

## Summary

Keep open. Current main and v2026.6.5 enforce hard sub-agent deadlines and recover limited progress after timeout, but neither warns the active child before termination; no canonical related issue or PR owns the missing behavior.

## What This Changes

Requests a configurable warning shortly before a sub-agent hard timeout so the child can checkpoint progress.

## Best Possible Solution

Add one lifecycle-owned pre-deadline notification mechanism with documented policy and focused native/ACP coverage while retaining existing post-timeout partial-progress recovery.

## Reproduction Assessment

Yes at source level: configured deadlines terminate native and ACP child runs, while partial-progress recovery occurs only afterward and no pre-deadline child notification exists. A live timed run was not executed because this was a read-only review.

## Solution Assessment

Unclear as proposed: the capability is useful, but maintainers should choose global versus per-spawn policy and define reliable native/ACP warning semantics before adopting the suggested config shape.
