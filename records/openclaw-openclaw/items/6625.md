## Summary

Keep open. Current main and the latest release still expose hard sub-agent run timeouts and post-timeout partial-progress capture, but no config key, `sessions_spawn` parameter, native path, or ACP path that warns the child before timeout.

## What This Changes

The issue requests a configurable pre-timeout warning for sub-agents so they can checkpoint work before `runTimeoutSeconds` kills the run.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract on the existing sub-agent config/tool surfaces, deliver it consistently before the hard timeout in native and ACP runs, and preserve current post-timeout recovery.

## Reproduction Assessment

Not applicable as a runnable bug reproduction because this is a feature/config request. Source inspection shows current behavior is hard timeout plus post-timeout capture, not a pre-timeout warning.

## Solution Assessment

Yes at the product-direction level, but not as an automatic narrow bug fix. The maintainable path is a deliberate native and ACP lifecycle contract rather than a prompt workaround or local timeout wrapper.
