## Summary

Keep open. Current main and the latest release still expose only hard sub-agent run timeout behavior plus post-timeout recovery; the requested pre-timeout warning/grace contract is not implemented and needs maintainer API/product direction.

## What This Changes

Request a configurable sub-agent pre-timeout warning so child agents can checkpoint work before `runTimeoutSeconds` aborts them.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract on the existing sub-agent config/tool surfaces, deliver it consistently before the hard timeout in native and ACP runs, and preserve current post-timeout recovery.
