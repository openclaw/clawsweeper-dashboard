## Summary

Keep open: current main still has hard sub-agent run timeouts and post-timeout partial-progress reporting, but no configurable pre-timeout warning/grace delivery path. This remains a feature/API and lifecycle design decision rather than a narrow existing-contract bug.

## What This Changes

Requested a configurable pre-timeout warning/grace period for sub-agents before `runTimeoutSeconds` kills the child session.

## Best Possible Solution

Define an accepted sub-agent timeout-warning contract, add documented config and optional per-spawn surface if approved, then schedule warning delivery before hard timeout in native and ACP paths while preserving existing post-timeout recovery.
