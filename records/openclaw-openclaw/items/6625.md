## Summary

Keep open: current main still supports hard sub-agent run timeouts and post-timeout partial-progress reporting, but it does not provide a configurable pre-timeout warning/grace mechanism for native or ACP child runs.

## What This Changes

Request a configurable grace period that warns a sub-agent shortly before `runTimeoutSeconds` expires so it can checkpoint work before the hard timeout.

## Best Possible Solution

Define one documented deadline-relative warning contract, then deliver it through shared native and ACP child-session control paths while preserving hard-timeout and post-timeout recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main gives high confidence that no pre-timeout warning/grace surface is implemented.

## Solution Assessment

Unclear until maintainers settle the public config/tool API, default lead time, and child-control semantics. A shared native/ACP warning path is safer than ad hoc prompt injection at one spawn call site.
