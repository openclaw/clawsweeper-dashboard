## Summary

Keep open: current main still has sub-agent hard run timeouts and post-timeout partial-progress recovery, but no configurable pre-timeout warning/grace mechanism before the child is timed out.

## What This Changes

Request a configurable warning injected into sub-agents shortly before `runTimeoutSeconds` expires so they can checkpoint work before termination.

## Best Possible Solution

Define and document a timeout-warning/grace contract with config and optional per-spawn override, then deliver it through native and ACP child-session lifecycle paths while preserving the hard timeout and current post-timeout recovery.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main confirms hard timeout plumbing and post-timeout recovery, but no pre-timeout warning surface to reproduce against.

## Solution Assessment

Unclear until maintainers settle the lifecycle contract. A core child-session control/message path that works for native and ACP runs is safer than ad hoc prompt injection at one call site.
