## Summary

Keep this open: current main still has hard sub-agent run timeouts, ACP timeout propagation, and post-timeout recovery/announce handling, but no configurable pre-timeout grace warning or child-session warning delivery path. The prior maintainer/Codex comment correctly distinguished the requested checkpoint warning from the existing timeout and recovery behavior, and the current source still matches that assessment.

## What This Changes

The issue requests a configurable pre-timeout warning for sub-agents so they can checkpoint before `runTimeoutSeconds` ends the run.

## Best Possible Solution

Add a documented core sub-agent timeout-warning/grace setting, optionally allow a per-spawn override, and deliver the warning through a native/ACP child-session control path while preserving existing hard-timeout and recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not a broken documented behavior. Source inspection gives high confidence that current main lacks the requested schema, config, docs, and runtime warning path.

## Solution Assessment

Unclear until maintainers settle the lifecycle contract. The requested warning is a plausible direction, but the maintainable solution should be designed as a core sub-agent lifecycle/control seam that covers native and ACP runs rather than an ad hoc prompt injection.
