## Summary

Keep open. Current main and v2026.6.10 still implement hard sub-agent run timeouts plus post-timeout recovery/announce handling, but I did not find a configurable pre-timeout checkpoint warning delivered to the child before termination.

## What This Changes

Request a graceful sub-agent timeout feature that warns a child agent shortly before `runTimeoutSeconds` expires so it can save or checkpoint work.

## Best Possible Solution

Define one maintainer-approved pre-timeout child notification contract shared by native and ACP sub-agent runs, then add focused tests and docs while preserving current post-timeout recovery.

## Reproduction Assessment

Yes, at the source level: current main has hard sub-agent deadlines and post-timeout handling, but no warning input, config key, or scheduled child notification before termination. I did not run a live timeout scenario during this read-only review.

## Solution Assessment

Unclear as proposed. A pre-timeout warning targets the reported loss mode, but the public config/tool API shape and native/ACP delivery semantics need maintainer design before a narrow implementation.
