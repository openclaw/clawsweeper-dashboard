## Summary

Keep open: current main and the latest shipped release still expose hard sub-agent run timeouts and post-timeout partial-progress recovery, but no configurable pre-timeout warning or grace-period delivery path.

## What This Changes

Request a configurable pre-timeout warning for sub-agent runs so a child can checkpoint before `runTimeoutSeconds` aborts it.

## Best Possible Solution

Define an accepted sub-agent timeout-warning contract, add config and optional per-spawn surface, schedule warning delivery before the hard timeout in native and ACP paths where feasible, and cover it with focused tests/docs while preserving post-timeout partial-progress recovery.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that the requested config/tool parameters and pre-timeout warning path are absent on current main.

## Solution Assessment

Unclear. The proposed warning is plausible, but maintainers need to settle the warning contract, defaults, override surface, and native/ACP delivery semantics before implementation.
