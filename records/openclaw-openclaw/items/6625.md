## Summary

Keep open: current main still has sub-agent hard timeout plumbing and post-timeout recovery paths, but no configurable pre-timeout warning/grace surface for native or ACP sub-agent runs.

## What This Changes

The issue requests a configurable warning delivered to sub-agents shortly before `runTimeoutSeconds` expires so they can checkpoint work before hard timeout.

## Best Possible Solution

Define a documented core timeout-warning/grace contract, implement deadline-relative warning delivery through native and ACP child-session lifecycle paths, and preserve existing hard-timeout and post-timeout recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main confirms timeout plumbing and post-timeout recovery, but no pre-timeout warning surface.

## Solution Assessment

Unclear until maintainers settle the lifecycle contract. A core child-session control/message path that works for native and ACP runs is safer than ad hoc prompt injection at one spawn call site.
