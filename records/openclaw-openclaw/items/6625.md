## Summary

Keep open: current main still has sub-agent hard timeout plumbing and post-timeout partial-progress summaries, but no configurable pre-timeout warning/grace surface or scheduled child-session warning for native or ACP runs.

## What This Changes

Add a configurable pre-timeout warning/grace period so sub-agents can checkpoint before `runTimeoutSeconds` ends a long-running run.

## Best Possible Solution

Define a documented timeout-warning/grace contract, implement deadline-relative warning delivery through shared native and ACP child-session control paths, and keep existing hard-timeout and post-timeout recovery behavior intact.

## Reproduction Assessment

Not applicable: this is an enhancement request, not a broken documented behavior. Source inspection on current main confirms timeout and post-timeout recovery paths exist, but no pre-timeout warning/grace surface is present.

## Solution Assessment

Unclear until maintainers settle the public config/tool API and child-control semantics. A shared warning path for native and ACP runs is safer than ad hoc prompt injection at one spawn call site.
