## Summary

Keep open. Current main and the latest release still have hard sub-agent run timeouts plus post-timeout partial-progress/announce handling, but no configurable pre-timeout warning delivered to the child agent before termination.

## What This Changes

The issue requests a configurable grace-period warning that tells a sub-agent to checkpoint before `runTimeoutSeconds` terminates the run.

## Best Possible Solution

Define one maintainer-approved pre-timeout child notification contract shared by native and ACP sub-agent runs, with focused tests and docs while preserving existing post-timeout recovery.

## Reproduction Assessment

Yes, at the source level: current main has hard sub-agent deadlines and post-timeout handling, but no warning input, config key, or scheduled child notification before termination. I did not run a live timeout scenario during this read-only review.

## Solution Assessment

Unclear as proposed. A pre-timeout child notification directly targets the reported loss mode, but the suggested config/API shape and native/ACP delivery semantics still need maintainer design.
