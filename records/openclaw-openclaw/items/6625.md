## Summary

Keep open. Current main and the latest release still provide hard `runTimeoutSeconds` handling plus post-timeout partial-progress recovery, but not the requested configurable pre-timeout warning for child agents.

## What This Changes

Request to add a configurable grace-period warning that reaches a sub-agent before `runTimeoutSeconds` aborts its run.

## Best Possible Solution

Define a documented sub-agent timeout-warning/grace contract, likely under `agents.defaults.subagents` with an optional `sessions_spawn` override, and deliver it consistently across native and ACP runs while preserving existing timeout cleanup and post-timeout recovery.

## Solution Assessment

Yes on the product direction, but not as an automatic narrow bug fix. The maintainable solution needs a deliberate lifecycle contract across native and ACP sub-agent runs rather than a local timeout-call patch.
