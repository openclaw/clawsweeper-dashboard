## Summary

Keep open. Current main still has hard sub-agent run timeouts and post-timeout partial-progress summaries, but no configurable pre-timeout warning/grace path in the `sessions_spawn` API, subagent config, native spawn, or ACP spawn.

## What This Changes

Request a configurable warning delivered to sub-agents shortly before `runTimeoutSeconds` kills the run so they can checkpoint progress.

## Best Possible Solution

Add first-class core lifecycle support for a documented grace/warning setting and optional per-spawn override, with safe native and ACP delivery before the hard timeout fires.

## Reproduction Assessment

Yes, for the missing capability: source inspection on current main shows only hard timeout configuration and dispatch paths. This is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear. The warning direction is reasonable, but the best implementation should be a designed lifecycle/control-message path across native and ACP runs with documented config/API shape.
