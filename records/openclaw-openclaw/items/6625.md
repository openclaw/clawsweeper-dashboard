## Summary

Keep open. Current main still exposes hard sub-agent run timeouts plus post-timeout output capture, but no config key, `sessions_spawn` parameter, native path, or ACP path that warns the child before timeout; this is useful core lifecycle work but needs maintainer product/API direction.

## What This Changes

Request a configurable sub-agent grace warning before `runTimeoutSeconds` expires so child agents can checkpoint progress before a hard timeout.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract under the existing sub-agent config/tool surfaces, deliver it consistently before the hard timeout in native and ACP runs, and preserve existing post-timeout recovery.
