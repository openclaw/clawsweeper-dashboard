## Summary

Keep open: current main has configurable hard sub-agent run timeouts and post-timeout partial-progress reporting, but no configurable pre-timeout warning/grace path for native or ACP child runs. This is a feature/config/lifecycle design request, not conservative cleanup material.

## What This Changes

The issue requests a configurable warning sent to sub-agents shortly before `runTimeoutSeconds` expires so they can checkpoint progress before a hard timeout.

## Best Possible Solution

Define one documented deadline-relative warning contract and deliver it through shared native and ACP child-session control paths while preserving hard-timeout and post-timeout recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source and docs inspection on current main give high confidence that no pre-timeout warning/grace surface exists today.

## Solution Assessment

Unclear until maintainers settle the public API, default lead time, and child-control semantics. A shared native/ACP lifecycle path is safer than injecting an ad hoc prompt at only one spawn call site.
