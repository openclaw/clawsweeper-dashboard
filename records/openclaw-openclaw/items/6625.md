## Summary

Keep open. Current main still has hard sub-agent run timeouts and post-timeout partial-progress reporting, but no configurable pre-timeout grace warning for native or ACP sub-agent runs.

## What This Changes

The issue requests a configurable warning message shortly before `runTimeoutSeconds` expires so sub-agents can checkpoint work before being killed.

## Best Possible Solution

Add first-class core lifecycle support for a documented warning/grace setting and optional per-spawn override, with safe delivery to native and ACP child sessions before the hard timeout fires.

## Reproduction Assessment

Yes. Source inspection on current main shows the tool schema, strict config schema, native spawn path, ACP spawn path, docs, and tests expose only hard run timeouts and post-timeout recovery, not a pre-timeout warning surface.

## Solution Assessment

Unclear for implementation details. The requested direction is valid, but the maintainable fix should be a designed lifecycle warning path rather than an ad hoc steering/send workaround that could race timeout or cleanup.
