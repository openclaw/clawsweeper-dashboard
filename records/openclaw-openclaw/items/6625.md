## Summary

Keep this open: current main still supports hard sub-agent run timeouts and post-timeout partial-progress recovery, but not the requested pre-timeout warning/grace mechanism.

## What This Changes

Request a configurable pre-timeout grace warning for sub-agents so long-running children can checkpoint before `runTimeoutSeconds` kills the run.

## Best Possible Solution

Add a documented sub-agent timeout-warning/grace contract with config and optional per-spawn override, delivered through native/ACP lifecycle control while preserving hard timeout and recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not a broken documented behavior. Source inspection confirms current main only supports hard run timeouts and post-timeout partial-progress recovery.

## Solution Assessment

Unclear until maintainers settle the lifecycle contract. A core control/message path that works for native and ACP runs is safer than ad hoc system prompt injection.
