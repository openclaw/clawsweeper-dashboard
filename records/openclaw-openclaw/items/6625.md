## Summary

Keep open: current main and the latest shipped release still expose hard sub-agent run timeouts plus post-timeout partial-progress recovery, but no configurable pre-timeout warning or grace-period delivery path. This is a real feature request with product/API and native/ACP lifecycle decisions still outstanding.

## What This Changes

The issue requests a configurable pre-timeout warning/grace period for sub-agents before `runTimeoutSeconds` kills the child run.

## Best Possible Solution

Define an accepted sub-agent timeout-warning contract, add config and optional per-spawn surface, schedule warning delivery before the hard timeout in native and ACP paths where feasible, and preserve existing post-timeout partial-progress recovery.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that the requested config/tool parameters and pre-timeout warning path are absent on current main.

## Solution Assessment

Unclear. A pre-timeout warning is plausible, but maintainers need to settle the warning contract, defaults, override surface, and native/ACP delivery semantics before implementation.
