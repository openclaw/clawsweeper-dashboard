## Summary

Keep open: current main still has hard sub-agent run timeout plumbing and post-timeout partial-progress recovery, but it does not expose or deliver a configurable pre-timeout checkpoint warning for native or ACP sub-agent runs.

## What This Changes

Requested a configurable pre-timeout warning/grace period so sub-agents can checkpoint work before `runTimeoutSeconds` aborts them.

## Best Possible Solution

Define a documented core timeout-warning/grace contract, implement delivery through native and ACP child-session lifecycle paths, and preserve existing hard-timeout and post-timeout partial-progress behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main confirms timeout plumbing and post-timeout partial-progress handling, but no pre-timeout warning surface.

## Solution Assessment

Unclear until maintainers settle the lifecycle contract. A core child-session control/message path that works for native and ACP runs is safer than ad hoc prompt injection at one call site.
