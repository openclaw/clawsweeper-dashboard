## Summary

Keep open: current main and v2026.6.8 still have hard sub-agent run timeouts and post-timeout recovery, but no pre-timeout warning/grace config, tool input, or native/ACP child-delivery path; related issues cover stall recovery and post-timeout notification, not this exact contract.

## What This Changes

The issue requests a configurable pre-timeout warning/grace period for sub-agents before `runTimeoutSeconds` terminates a run.

## Best Possible Solution

Define one lifecycle-owned pre-timeout child notification contract shared by native and ACP sub-agent runs, with maintainer-approved defaults/config, focused tests, and docs while preserving post-timeout recovery.

## Solution Assessment

Unclear. A pre-timeout child notification directly targets the reported loss mode, but maintainers still need to choose the API/config shape, default policy, and native/ACP delivery semantics.
