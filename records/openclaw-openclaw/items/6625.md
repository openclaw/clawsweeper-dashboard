## Summary

Keep open. Current main still has hard sub-agent `runTimeoutSeconds` handling and post-timeout partial-progress recovery, but it does not implement the requested configurable pre-timeout warning that lets a child agent checkpoint before abort.

## What This Changes

Requests a configurable sub-agent grace period or pre-timeout warning before `runTimeoutSeconds` hard-aborts a child run.

## Best Possible Solution

Define a documented sub-agent timeout-warning/grace contract, likely under `agents.defaults.subagents` with an optional `sessions_spawn` override, and deliver it consistently across native and ACP runs while preserving existing timeout cleanup and post-timeout recovery.

## Reproduction Assessment

Not applicable as a runnable bug reproduction; this is a feature/config request. Current-main source and latest-release tag inspection show no schema, tool, or runtime path for a pre-timeout warning.

## Solution Assessment

Yes at the product-direction level, but not as an automatic narrow bug fix. The maintainable solution needs a deliberate lifecycle contract across native and ACP sub-agent runs rather than a local timeout-call patch.
