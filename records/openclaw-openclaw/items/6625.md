## Summary

Keep open: current main still has hard sub-agent run timeouts and post-timeout partial-progress recovery, but no configurable pre-timeout warning/grace API or native/ACP delivery path that warns the child before the hard timeout.

## What This Changes

Add a configurable sub-agent grace period that warns child sessions before `runTimeoutSeconds` expires so they can checkpoint work before timeout.

## Best Possible Solution

Define a documented core timeout-warning/grace contract, implement delivery through native and ACP child-session lifecycle paths, and preserve existing hard-timeout and post-timeout partial-progress behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main confirms hard timeout plumbing and post-timeout partial-progress handling, but no pre-timeout warning surface.

## Solution Assessment

Unclear until maintainers settle the lifecycle contract. A core child-session control/message path that works for native and ACP runs is safer than ad hoc prompt injection at one call site.
