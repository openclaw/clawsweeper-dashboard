## Summary

Keep open: current main still only has hard sub-agent run timeouts plus post-timeout partial-progress recovery, not a configurable pre-timeout warning/grace mechanism for native or ACP child runs.

## What This Changes

Request adding a configurable warning before `runTimeoutSeconds` expires so sub-agents can checkpoint before a hard timeout.

## Best Possible Solution

Define one documented deadline-relative timeout-warning contract, then deliver it through shared native and ACP child-session control paths while preserving hard-timeout and post-timeout recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main gives high confidence that no pre-timeout warning/grace surface is implemented.

## Solution Assessment

Unclear until maintainers settle the public config/tool API, default lead time, and child-control semantics. A shared native/ACP warning path is safer than ad hoc prompt injection at one spawn call site.
