## Summary

Keep open: current main still has hard sub-agent run timeouts plus post-timeout partial-progress summaries, but no configurable pre-timeout warning or grace path. The remaining change adds config/API and lifecycle semantics that need maintainer product review.

## What This Changes

Request a configurable pre-timeout warning/grace period so sub-agents can checkpoint before `runTimeoutSeconds` terminates the run.

## Best Possible Solution

Define a core sub-agent timeout-warning contract, expose accepted config and optional per-spawn API, schedule delivery before the hard timeout for native and ACP runs where feasible, and document/test its interaction with existing post-timeout recovery.

## Reproduction Assessment

Not applicable as a feature request rather than a broken existing contract. Source inspection gives high-confidence verification that current main lacks the requested config, per-spawn parameter, and pre-timeout child warning path.

## Solution Assessment

Unclear. A pre-timeout warning is plausible, but maintainers should decide the config/API shape and native/ACP delivery semantics before implementing it.
