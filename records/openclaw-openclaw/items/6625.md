## Summary

Keep open: current main still supports hard sub-agent run timeouts and post-timeout partial-progress extraction, but it does not provide a configurable pre-timeout warning/grace surface or scheduled warning delivery for native or ACP sub-agents.

## What This Changes

Request a configurable warning sent to a sub-agent shortly before `runTimeoutSeconds` kills the run so it can checkpoint progress.

## Best Possible Solution

Define one documented deadline-relative timeout-warning contract, then deliver it through shared native and ACP child-session control paths while preserving existing hard-timeout and post-timeout recovery behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not broken documented behavior. Source inspection on current main gives high confidence that no pre-timeout warning/grace surface is implemented.

## Solution Assessment

Unclear until maintainers settle the public config/tool API, default lead time, and child-control semantics. A shared native/ACP warning path is safer than adding ad hoc prompt injection at a single spawn call site.
