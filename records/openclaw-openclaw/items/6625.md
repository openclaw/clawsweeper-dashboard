## Summary

Keep open. Current main still has hard `runTimeoutSeconds` handling and post-timeout partial-progress recovery, but no configurable pre-timeout warning/grace contract for child agents.

## What This Changes

The issue requests a configurable warning before sub-agent `runTimeoutSeconds` expires so the child can checkpoint work before the hard timeout.

## Best Possible Solution

Define a documented sub-agent timeout-warning/grace contract, likely under `agents.defaults.subagents` plus an optional per-spawn override, and deliver it consistently across native and ACP runs while preserving existing timeout cleanup and post-timeout recovery.

## Reproduction Assessment

Not applicable as a runnable bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check that the requested pre-timeout warning/config path is absent.

## Solution Assessment

Yes on the product direction, but not as an automatic narrow bug fix. The maintainable solution needs a deliberate lifecycle contract across native and ACP sub-agent runs rather than a local timeout-call patch.
