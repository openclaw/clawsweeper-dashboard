## Summary

Keep open. Current main and the latest release still expose hard sub-agent run timeouts plus post-timeout partial-progress reporting, but there is no configurable pre-timeout warning or native/ACP child-control path that lets the child checkpoint before abort.

## What This Changes

The issue requests a configurable pre-timeout warning for sub-agents before `runTimeoutSeconds` kills the child run.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract, likely under `agents.defaults.subagents` plus an optional `sessions_spawn` override, and deliver it consistently across native and ACP runs while preserving existing post-timeout recovery.

## Reproduction Assessment

Not applicable as a runnable bug reproduction; this is a feature/config request. Current-main and latest-release source inspection show no schema, docs, or runtime path for a pre-timeout warning.

## Solution Assessment

Yes at the product-direction level, but not as an automatic narrow bug fix. The maintainable solution needs a deliberate lifecycle contract across native and ACP sub-agent runs rather than a prompt-only workaround or local timeout wrapper.
