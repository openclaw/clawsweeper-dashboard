## Summary

Keep open. Current main and the latest release still expose hard sub-agent run timeouts plus post-timeout partial-progress recovery, but no config key, `sessions_spawn` parameter, native path, or ACP path that warns the child before timeout; the remaining work is valid core lifecycle design that needs maintainer product/API direction.

## What This Changes

The issue requests a configurable pre-timeout warning/grace period for sub-agents so they can checkpoint work before `runTimeoutSeconds` kills the run.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract under the existing sub-agent config/tool surfaces, deliver it consistently before the hard timeout in native and ACP runs, and preserve existing post-timeout recovery.
