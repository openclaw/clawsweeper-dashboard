## Summary

Keep open. Current main still has hard sub-agent run timeouts and post-timeout partial-progress capture, but no config key, `sessions_spawn` parameter, native path, or ACP path that warns the child before timeout; this is useful core lifecycle work that needs maintainer product/API direction.

## What This Changes

Add a configurable grace-period warning for sub-agent runs before `runTimeoutSeconds` terminates the child so it can checkpoint progress.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract under the existing sub-agent config/tool surfaces, deliver it consistently before the hard timeout in native and ACP runs, and preserve current post-timeout recovery.
