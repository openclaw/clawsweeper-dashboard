## Summary

Keep open. Current main and v2026.6.5 still have configurable hard sub-agent run timeouts and post-timeout partial-progress recovery, but no configurable pre-timeout warning/grace contract or native/ACP child-warning delivery path.

## What This Changes

The issue requests a configurable pre-timeout warning/grace period for sub-agent runs so child agents can checkpoint before `runTimeoutSeconds` ends the run.

## Best Possible Solution

Add a documented sub-agent timeout-warning/grace contract on the existing sub-agent config/tool surfaces, deliver it before the hard timeout for native and ACP runs where feasible, and preserve current post-timeout recovery.

## Reproduction Assessment

Yes from source, at high confidence. Current main and the latest release expose hard timeout and post-timeout capture surfaces, but no pre-timeout warning/grace setting or child warning delivery path; no live timeout scenario was run for this read-only feature review.

## Solution Assessment

Yes at the product-direction level, but not as an automatic narrow bug fix. The maintainable path is a deliberate native and ACP lifecycle contract rather than a prompt workaround or local timeout wrapper.
