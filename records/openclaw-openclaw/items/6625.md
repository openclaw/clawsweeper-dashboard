## Summary

Keep open. Current main and the latest release still expose sub-agent hard timeouts and post-timeout partial-progress fallback, but not the requested configurable pre-timeout warning that lets the child checkpoint before abort.

## What This Changes

Request a configurable grace/pre-timeout warning for long-running sub-agents before `runTimeoutSeconds` aborts the child run.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract, likely under `agents.defaults.subagents` with an optional `sessions_spawn` override, and deliver it consistently across native and ACP runs while preserving existing timeout cleanup and post-timeout recovery.

## Reproduction Assessment

Not applicable as a runnable bug reproduction; this is a feature/config request. Current-main and latest-release source inspection show no schema, docs, or runtime path for a pre-timeout warning.

## Solution Assessment

Yes at the product-direction level, but not as an automatic narrow bug fix. The maintainable solution needs a deliberate lifecycle contract across native and ACP sub-agent runs rather than a local timeout wrapper.
