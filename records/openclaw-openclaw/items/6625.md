## Summary

Keep open. Current main still has hard sub-agent `runTimeoutSeconds` handling and post-timeout partial-progress summaries, but no config key, `sessions_spawn` parameter, native path, or ACP path that warns the child agent before the timeout.

## What This Changes

Request a configurable sub-agent grace period that injects a pre-timeout checkpoint warning before `runTimeoutSeconds` kills the child run.

## Best Possible Solution

Define a documented sub-agent timeout-warning contract under the existing sub-agent config/tool surfaces, deliver it consistently for native and ACP runs before the hard timeout, and preserve current post-timeout recovery behavior.

## Reproduction Assessment

Not applicable as a runnable bug reproduction; this is a feature/config request. Source inspection shows current behavior is a hard run timeout plus post-timeout partial-progress extraction, not a pre-timeout warning.

## Solution Assessment

Yes at the product-direction level, but not as an automatic narrow bug fix. The maintainable solution needs a deliberate native and ACP lifecycle contract rather than a prompt workaround or local timeout wrapper.
