## Summary

Keep this PR open: it targets a real agent bug, but the patch still misses sessionId-scoped process sessions, can expose unscoped active-process metadata, lacks the required changelog entry, and its proof does not exercise a full subagent compaction run.

## What This Changes

The PR adds active background exec session snapshots to agent runtime and compaction contexts, surfaces those references in embedded system prompts, and adds focused unit coverage.

## Best Possible Solution

Derive one shared process scope key from the same sandboxSessionKey/sessionKey/sessionId fallback used by exec/process tools, omit active snapshots when no safe scope exists, and cover the compaction prompt/runtime handoff with regression tests plus a changelog entry.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection shows current main has scoped exec/process sessions but no active-process compaction handoff, which matches the linked report at https://github.com/openclaw/openclaw/issues/79284.

## Solution Assessment

No. The proposed direction is maintainable, but it is not the best fix until all snapshot call sites use the exact process-tool scope fallback and the no-scope path fails closed instead of listing unrelated sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve sessionId-scoped exec sessions:** `src/agents/pi-embedded-runner/run.ts:1500-1502`
  - body: `runEmbeddedAttempt` starts exec/process tools under `sandboxSessionKey || sessionKey || sessionId`, but these compaction snapshots derive the lookup scope from only `sandboxSessionKey || sessionKey` plus the agent fallback. Runs without a session key will still lose active sessions that were stored under the sessionId scope, so use the same fallback before listing active process references.
  - confidence: 0.88
- **[P2] Preserve sessionId scope in after-turn context:** `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:547-550`
  - body: The after-turn runtime context now lists active process sessions using `sandboxSessionKey || sessionKey`, but the attempt's tool scope can fall back to `sessionId`. That leaves context-engine after-turn maintenance unable to see the same active exec session the process tool can see for sessionId-scoped runs.
  - confidence: 0.8
- **[P2] Fail closed when no process scope is available:** `src/agents/pi-embedded-runner/compaction-runtime-context.ts:107-110`
  - body: When `sessionKey` is absent, this calls `listActiveProcessSessionReferences` with an undefined scope, and the new helper treats undefined as no filter. That can inject other sessions' command labels, cwd values, pids, and session IDs into the runtime context; omit activeProcessSessions unless a safe scope key exists.
  - confidence: 0.83
- **[P3] Add the required changelog entry:** `src/agents/system-prompt.ts:1177`
  - body: This changes user-facing agent behavior for long-running execs across compaction, but the branch does not touch `CHANGELOG.md`. Repo policy requires user-facing fix, feat, and perf changes to include a changelog entry before merge.
  - confidence: 0.93
