## Summary

Keep open: this PR is a plausible fix path for https://github.com/openclaw/openclaw/issues/79284, but the patch still has a scope mismatch in compaction snapshots, lacks the required changelog entry, needs a security-sensitive scoping adjustment, and has no real behavior proof yet.

## What This Changes

The PR adds bounded active background exec session snapshots to agent runtime and compaction contexts, surfaces them in system prompts, and adds focused unit coverage.

## Best Possible Solution

Keep the PR open until active process snapshots use the same scope as exec/process tools, the changelog is added, and the contributor supplies redacted real runtime proof of the subagent compaction scenario.

## Solution Assessment

No. The proposed direction is maintainable, but the implementation needs to preserve the exact sessionId fallback scope and add the required changelog and runtime proof before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve the sessionId process scope:** `src/agents/pi-embedded-runner/run.ts:1497-1502`
  - body: `runEmbeddedAttempt` creates exec/process tools with `sandboxSessionKey = params.sandboxSessionKey || params.sessionKey || params.sessionId`. These new timeout/overflow compaction snapshots derive the scope from only `sandboxSessionKey`/`sessionKey` plus `agentId`, so callers without a session key will omit active execs that were started under the sessionId scope. Use the same sessionId fallback at both run-level compaction sites so the snapshot matches the process tool scope.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/system-prompt.ts:1176`
  - body: This is a user-facing agent bug fix, but the branch does not touch `CHANGELOG.md`. Repo policy requires user-facing fix/feat/perf changes to have a changelog entry before merge.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or CI execution, but the active-process metadata must be strictly scoped before merge.

Concerns:

- **[medium] Avoid unscoped active process metadata:** `src/agents/pi-embedded-runner/compaction-runtime-context.ts:108`
  - body: When no scoped key is derived, `listActiveProcessSessionReferences` treats `undefined` as no filter and can return all active process session IDs, commands, cwd values, and pids into runtime context. Because those values may include sensitive operator paths or command text, callers should pass the exact exec/process scope or omit the snapshot when no safe scope exists.
  - confidence: 0.78
