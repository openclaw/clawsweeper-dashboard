## What This Changes

The PR changes backup archive creation to choose a staging temp root outside all backup source paths, adds regression coverage for state-local temp roots, and adds an Unreleased changelog entry for the user-facing backup fix.

## Best Possible Solution

Land this PR after the remaining CI and maintainer review complete so backup archive staging is guaranteed to stay outside selected backup sources, with the existing verifier continuing to reject duplicate-manifest archive shapes.

## Reproduction Assessment

Yes. Current main stages manifest.json under os.tmpdir(), and the macOS service environment sets TMPDIR to the state directory's tmp child, so a LaunchAgent backup create can stage manifest.json inside a source tree that tar then walks recursively.

## Solution Assessment

Yes. The PR fixes the behavior at the archive creation boundary with a source-path containment guard, preserves the service TMPDIR behavior needed for SQLite, and avoids changing tar semantics or broad backup planning code.
