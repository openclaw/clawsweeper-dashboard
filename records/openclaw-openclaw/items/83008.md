## Summary

Keep open. Current main still contains the reported Telegram ingress shutdown and channel-supervisor stale-state paths, and an open PR already references this issue as the fix path, so the issue should stay open until that PR or a replacement lands.

## What This Changes

The issue reports that Telegram config hot reload can wedge a channel when isolated ingress shutdown exits nonzero and leaves stale supervisor task state.

## Best Possible Solution

Keep the issue open and land or replace the linked PR with regression coverage that proves requested-stop worker failures and manual stop timeouts cannot leave stale task state blocking hot-reload restart.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection and the existing timeout test show a focused repro path with an isolated ingress worker rejection or a timed-out channel task during `channels.telegram.*` hot reload.

## Solution Assessment

Unclear that the one-line worker exit-code proposal is sufficient. The maintainable fix should add regression coverage and cover both requested-stop worker rejection and stale supervisor task cleanup after stop timeout.
