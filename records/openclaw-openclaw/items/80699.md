## Summary

Keep open: current main supports the generic per-CLI-backend env workaround described in the issue, but it does not implement the requested first-class per-agent identity binding or doctor registry validation. This is a new config/product feature, not a stale or duplicate bug cleanup candidate.

## What This Changes

The issue proposes adding per-agent AI Engineer and Code Creator identity binding that injects git committer env vars into CLI backend spawns and verifies optional registry drift in doctor.

## Best Possible Solution

Decide the product boundary first; if accepted, add a typed config/schema/help/docs contract plus CLI-runner env injection and doctor validation, while keeping generic `agents.defaults.cliBackends.<id>.env` for non-identity env overrides.

## Reproduction Assessment

Not applicable: this is a feature proposal, not a reported regression. Source inspection gives high confidence that current main only has generic per-backend env injection and display-oriented agent identity fields.

## Solution Assessment

Unclear until maintainers decide the product boundary. The proposed first-class config is plausible, but it is broader than a narrow bug fix because it adds schema, docs, runtime policy, and doctor registry semantics.
