## Summary

Keep open: current main still supports the backend-level env workaround, but it does not implement the requested per-agent identity schema, cross-backend committer env derivation, or doctor registry validation. This remains a new config and trust-model feature, not a conservative cleanup close.

## What This Changes

Add first-class per-agent identity binding so CLI-backend subprocesses use a configured code-creator Git committer identity and doctor can validate it against an optional registry.

## Best Possible Solution

Maintainers should first decide whether core should own this committer identity contract; if accepted, implement typed config/schema/help/docs, generic CLI spawn env derivation, and doctor validation with explicit trust semantics.

## Reproduction Assessment

Not applicable: this is a feature proposal, not a broken existing behavior. Source inspection gives high confidence that current main only has backend-level env injection and display-oriented agent identity fields.

## Solution Assessment

Unclear until maintainers decide the product boundary. The proposal is plausible, but it adds schema, runtime policy, docs/help, and remote doctor validation rather than repairing broken existing behavior.
