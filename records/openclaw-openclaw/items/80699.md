## Summary

Keep open for maintainer product review. The report usefully documents a working config-only workaround via `agents.defaults.cliBackends.<id>.env`, but current `main` does not have first-class per-agent committer identity binding or doctor registry validation, and adding it would change public config, CLI spawn behavior, and validation policy.

## What This Changes

Proposal to add first-class agent identity metadata that sets Git committer environment variables for spawned CLI backends and lets doctor validate the binding.

## Best Possible Solution

If accepted, design a generic agent identity sub-contract with explicit committer env derivation in CLI-runner preparation plus safe doctor validation rules; otherwise document the current backend-env workaround as the supported operator path.

## Reproduction Assessment

Not applicable. This is a feature/config proposal, and source inspection confirms current main supports the workaround but lacks the requested first-class binding.

## Solution Assessment

Unclear. The proposed direction may be useful, but automatic env injection plus remote registry validation needs maintainer product and security design before a narrow implementation is safe.
