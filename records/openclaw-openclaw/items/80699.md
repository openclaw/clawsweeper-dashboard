## Summary

Keep open. Current main supports backend-specific CLI env injection, but it does not implement the proposed first-class per-agent AI AE/code-creator binding or doctor registry validation, and the remaining work needs maintainer product/security design rather than cleanup closure.

## What This Changes

Add a top-level per-agent identity binding that maps AI AE and code-creator git identities into every CLI-backend spawn and validates optional registry metadata in doctor.

## Best Possible Solution

Design a schema-backed agent identity binding with documented spawn-env precedence, redaction/validation rules for git identity fields, and doctor registry checks that stay generic rather than encoding one deployment's policy.

## Reproduction Assessment

Not applicable. This is a feature proposal, and source inspection confirms the existing supported path is backend-level `agents.defaults.cliBackends.<id>.env` while the proposed per-agent `identity.code_creator` and registry fields are absent.

## Solution Assessment

Unclear until maintainers choose the product boundary. The maintainable direction is likely a generic, schema-backed identity-binding contract that preserves existing backend env injection and does not hard-code a deployment-specific registry format.
