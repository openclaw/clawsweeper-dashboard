## Summary

Keep open: current main and the latest release support the generic per-CLI-backend environment override workaround, but they do not implement the requested first-class per-agent Git identity binding or doctor registry validation. This is a new product/config and trust-model feature, not a cleanup close candidate.

## What This Changes

Add a first-class per-agent identity binding so CLI backend child processes use a code-creator Git committer identity, with doctor registry validation.

## Best Possible Solution

If accepted, define a versioned per-agent identity config/schema/help/docs contract, inject `identity.code_creator` into CLI backend spawns, and add explicit doctor validation rules while preserving generic per-backend env overrides.

## Reproduction Assessment

Not applicable: this is a feature proposal rather than a reported regression. Source inspection gives high confidence that current main only supports generic per-backend env injection and display-oriented agent identity fields.

## Solution Assessment
