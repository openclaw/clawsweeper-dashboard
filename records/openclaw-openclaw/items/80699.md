## Summary

Keep open. Current main supports the reported workaround through generic per-CLI-backend env injection, but it still lacks the requested per-agent code-creator identity contract, cross-backend propagation rule, schema/help surface, and doctor registry validation.

## What This Changes

Add first-class per-agent AI engineer/code-creator identity binding so CLI-backend subprocesses receive Git committer env automatically and doctor can validate the binding.

## Best Possible Solution

Design a schema-backed, versioned agent identity binding contract that maps code-creator Git identity into CLI backend spawns, documents precedence with backend env, and adds bounded doctor validation for an optional registry.

## Reproduction Assessment

Not applicable: this is a feature proposal, not a broken existing behavior report. Source inspection confirms the existing backend env workaround works, while the requested identity and registry validation surfaces are absent.

## Solution Assessment

Unclear until maintainers settle the product boundary. A first-class identity contract could be the right direction, but the exact registry schema, trust model, and env precedence should be designed before implementation.
