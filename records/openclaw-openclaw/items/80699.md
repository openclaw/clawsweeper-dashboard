## Summary

Keep open: current main supports the generic per-backend env workaround, but it does not have the requested agent-level git identity binding or doctor registry validation. This is a new public config and trust-policy feature, so it needs maintainer design review rather than cleanup closure.

## What This Changes

Add first-class per-agent AI Engineer/code-creator identity binding that injects git committer env into CLI backend spawns and lets doctor validate it against an optional registry.

## Best Possible Solution

Define a maintainer-approved agent identity/git-committer contract, then implement backend-agnostic env injection and doctor validation with clear registry trust and failure semantics.

## Reproduction Assessment

Not applicable. This is a feature/config proposal, not a report of broken documented behavior; source inspection shows current main only supports display identity plus per-backend env overrides.

## Solution Assessment

Unclear until maintainers decide the public contract. A first-class binding may be the right direction, but registry validation and AE/code-creator semantics should be designed before a narrow implementation PR.
