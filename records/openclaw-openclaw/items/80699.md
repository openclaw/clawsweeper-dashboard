## Summary

Keep open: current main supports the generic per-CLI-backend env workaround, but it still does not provide the requested first-class per-agent git identity binding or doctor registry validation. This is a new config/product and trust-model feature rather than stale cleanup or an already-implemented bug fix.

## What This Changes

Add a top-level per-agent identity binding that maps AI engineer and code-creator git identities into CLI-backend spawns and validates the binding through doctor.

## Best Possible Solution

Decide the product boundary first; if accepted, add a typed config/schema/help/docs contract, derive CLI-runner committer env from per-agent identity, and add doctor validation while preserving generic backend env overrides.

## Reproduction Assessment

Not applicable: this is a feature proposal, not a reported regression. Source inspection gives high confidence that current main only has generic per-backend env injection plus display-oriented agent identity fields.

## Solution Assessment
