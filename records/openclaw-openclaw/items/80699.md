## Summary

Keep open: current main still only supports the generic per-CLI-backend environment workaround, not the requested first-class per-agent Git identity binding or doctor registry validation. This is a new config/runtime/trust-model feature rather than an implemented fix, stale cleanup item, or duplicate.

## What This Changes

The issue proposes a first-class per-agent identity binding that injects code-creator Git committer environment into CLI backend spawns and optionally validates it with doctor.

## Best Possible Solution

Decide the product boundary first; if accepted, add a typed schema/help/docs contract, CLI-runner env injection, focused tests, and doctor validation that fits the broader per-agent identity/credential model.
