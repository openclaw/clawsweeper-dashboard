## Summary

Keep open: current main supports the generic per-CLI-backend env workaround described in the issue, but it does not implement the requested first-class per-agent identity binding, discoverable schema/help surface, or doctor registry validation. This is a new config and trust-model feature that needs maintainer product review rather than conservative cleanup closure.

## What This Changes

The issue proposes a first-class per-agent identity binding that injects code-creator Git committer environment into all CLI-backend spawns and validates it with `openclaw doctor`.

## Best Possible Solution

Decide the product boundary first; if accepted, add a typed config/schema/help/docs contract, CLI-runner identity-to-env injection across backends, and doctor validation while preserving generic backend env overrides for non-identity use cases.

## Reproduction Assessment

## Solution Assessment
