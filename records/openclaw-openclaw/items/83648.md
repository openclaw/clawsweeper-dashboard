## Label Justifications

- `P2`: This is a focused cron/provider-auth bugfix with limited blast radius, but the current patch has blocking auth correctness issues.
- `merge-risk: 🚨 compatibility`: The PR changes preflight behavior for existing local-provider setups while failing some documented SecretRef/env-marker configurations.

## Summary

Keep open: the reported LiteLLM preflight auth noise is real and the proof is useful, but this patch duplicates provider-auth resolution and can send marker/default-agent credentials instead of the credential the cron run actually uses.

## What This Changes

The PR adds a cron local-provider preflight API-key resolver, sends a Bearer Authorization header on GET probes, and commits a before/after evidence screenshot.

## Best Possible Solution

Route cron preflight through the same scoped provider auth and request-header contract used by the actual model run, while keeping authless local providers unchanged.

## Reproduction Assessment

Yes, at source level: current main sends the local OpenAI-compatible provider preflight GET without headers, and the PR body plus screenshot show LiteLLM 401 before and 200 after. I did not run a live LiteLLM instance in this read-only review.

## Solution Assessment

No. Adding auth to the probe is the right direction, but the current hand-rolled resolver is not the best fix because it diverges from the scoped provider auth/request contract used by real model calls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Resolve provider apiKey through the SecretInput auth path:** `src/cron/isolated-agent/model-preflight.runtime.ts:166-168`
  - body: `apiKey` is a `SecretInput`, but this branch returns any non-empty string directly and later sends it as `Authorization: Bearer ...`. Env templates or persisted env-marker strings can therefore be sent as the token instead of the resolved secret, while structured SecretRefs are skipped, so preflight still does not match normal inference auth.
  - confidence: 0.94
- **[P1] Keep preflight credentials scoped to the cron agent:** `src/cron/isolated-agent/model-preflight.runtime.ts:173-176`
  - body: This reads `resolveDefaultAgentDir(cfg)/auth-profiles.json`, but cron preflight is called without the selected agentDir/profile. A non-default cron agent can therefore use the wrong default-agent credential, miss its own keyRef/tokenRef/OAuth credential, or send auth material that the actual run would not use.
  - confidence: 0.93
