## Summary

Keep this PR open. It fixes the primary bearer-auth preflight path, but the current head still omits `models.providers.*.headers` from the shared header resolver, so some local proxy/tenant setups can still fail cron preflight while normal model requests succeed.

## What This Changes

The PR changes cron model-provider preflight to build probe headers through shared provider request helpers, pass agent/workspace dirs to credential lookup, redact preflight errors, and add auth parity tests.

## Best Possible Solution

Keep shared auth delegation, but pass sanitized provider-level headers through `defaultHeaders` or explicitly document and test why cron preflight excludes them.

## Reproduction Assessment

## Solution Assessment

No, not yet. Delegating auth construction to the shared resolver is the right direction, but the current call omits provider-level headers that normal model execution can use.
