- **[P1] Resolve provider keys through existing auth helpers:** `src/cron/isolated-agent/model-preflight.runtime.ts:170-172`
  - body: `apiKey` is a `SecretInput`, but this branch returns any non-empty string directly and later sends it as `Authorization: Bearer ...`. Env templates, legacy markers, and profile ids can therefore be sent as bearer text, while structured SecretRefs are skipped, so preflight still does not match normal provider auth.
  - confidence: 0.94
- **[P1] Pass cron agent scope into preflight auth:** `src/cron/isolated-agent/model-preflight.runtime.ts:177-180`
  - body: This reads `resolveDefaultAgentDir(cfg)/auth-profiles.json` inside preflight, but the cron caller does not pass the selected agentDir/profile/workspace context. A non-default cron agent can therefore use the wrong default-agent credential or miss its own profile credential, unlike the actual model run.
  - confidence: 0.93
- **[P1] Reuse provider request auth and headers:** `src/cron/isolated-agent/model-preflight.runtime.ts:209-214`
  - body: The probe now only injects `Authorization: Bearer <apiKey>`, dropping configured `headers`, `authHeader`, and `request.auth`/`request.headers` behavior that normal provider requests support. Self-hosted endpoints requiring custom auth or tenant headers can still be marked unavailable even though the real model request would work.
  - confidence: 0.9
