## Summary

Keep open. The PR targets a real remaining gap because current main still stops subagents synchronously and does not cancel ACP-backed child sessions, but the proposed patch is not merge-ready: the new awaited child ACP cancel can hang reset/stop on a stalled backend, the PR-head regression test setup needs to match the latest-run dedupe behavior, and the user-facing stop/reset change needs a changelog entry.

## What This Changes

The PR converts subagent stop propagation to async, awaits /stop and session-reset callers, cancels active ACP-backed child sessions, persists child abort state, and adds fast-abort regression coverage.

## Best Possible Solution

Land a rebased or replacement patch that keeps the async stop cascade, cancels ACP-backed child sessions through a bounded cleanup helper, preserves current main's subagent-run dedupe behavior, intentionally persists child aborted state, and documents the user-facing stop/reset fix in the active changelog.

## Reproduction Assessment

Yes. A child ACP runtime whose cancel promise never resolves reproduces the reset hang once reset awaits stopSubagentsForRequester; the PR-head test issue is also reproducible from the missing latest-run mock described in the existing review context.

## Solution Assessment

No, not as proposed. Propagating stop into ACP-backed child sessions is the right direction, but the mergeable solution needs bounded cancellation plus tests and changelog coverage aligned with current main.
