## Summary

Keep open: the PR targets a real source-reproducible stop/reset gap, but the submitted patch should not merge because it awaits ACP child cancellation without a bounded cleanup path and the external PR still lacks real behavior proof.

## What This Changes

The PR makes subagent stop propagation async, cancels ACP-backed child sessions during stop/reset, persists child abort state, and adds fast-abort regression coverage.

## Best Possible Solution

Land a rebased implementation that cancels active ACP-backed child sessions through a bounded cleanup helper, preserves current subagent dedupe semantics, intentionally persists child abort state, and includes focused regression tests plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current main's subagent stop loop never resolves or cancels ACP child sessions, while reset cleanup already expects bounded ACP cleanup for direct session cancel/close work.

## Solution Assessment

No, not as submitted. Propagating stop/reset to ACP-backed children is the right direction, but the child cancel must be bounded before it is awaited from reset/delete paths, and the external PR needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
