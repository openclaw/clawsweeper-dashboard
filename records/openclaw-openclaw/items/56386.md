## Summary

Keep open: current main still lacks the central ACP-backed child cancellation during subagent stop/reset, but this PR cannot merge as submitted because it is conflicting, lacks required real behavior proof, and introduces blocking correctness issues in the new awaited cleanup path, /stop fallback scope, and PR-head test setup.

## What This Changes

The PR makes subagent stop propagation asynchronous, adds ACP child session cancellation and child abort-state persistence, awaits gateway reset cleanup cascade, adjusts /stop fallback behavior, and extends abort tests.

## Best Possible Solution

Land a rebased implementation that cancels active ACP-backed child sessions through a bounded stop/reset cleanup path, preserves child abort state, keeps /stop scoped to the resolved target, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level: current main stops child queues and reply/embedded runs but does not cancel ACP-backed child sessions, while gateway reset/delete reaches that helper before bounded target ACP cleanup. I did not run a live ACP setup in this read-only review.

## Solution Assessment

No, not as submitted. The direction is right, but the implementation needs bounded ACP child cancellation, removal or narrowing of the wrapper fallback retry, repaired PR-head test setup, and real after-fix behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bound ACP child cancellation during stop cascade:** `src/auto-reply/reply/abort.ts:199-203`
  - body: The new child-stop path awaits `acpManager.cancelSession` directly. Since `cancelSession` can wait on an active turn's `cancelPromise` without a local timeout, and reset/delete now awaits this cascade, a stuck ACP backend can hang the whole reset path instead of returning through the existing bounded cleanup flow.
  - confidence: 0.91
- **[P2] Keep /stop fallback inside the resolved target:** `src/auto-reply/reply/commands-session-abort.ts:153-157`
  - body: When the resolved target stops zero children, this retries with the wrapper `params.sessionKey`. Current main has coverage that a distinct target missing from the store must not fall back to the wrapper session; this can stop unrelated subagents for the command session rather than the requested target.
  - confidence: 0.86
- **[P2] Mock the latest child run in the ACP cascade test:** `src/auto-reply/reply/abort.test.ts:600-613`
  - body: On PR head, `stopSubagentsForRequester` skips runs when `getLatestSubagentRunByChildSessionKey` returns null, but this test only mocks the listed run and ACP resolution. `cancelSession` is never reached and `stoppedSubagents` remains zero.
  - confidence: 0.88
