## Summary

Keep open: current main still does not cancel ACP-backed child sessions from the subagent stop cascade, so the PR is not obsolete, but the PR head is not mergeable as submitted because it awaits unbounded child ACP cancellation and has failing/stale regression coverage.

## What This Changes

The PR makes subagent stop propagation async, awaits stop/reset callers, cancels ACP-backed child sessions, persists child abort state, and adds fast-abort regression coverage.

## Best Possible Solution

Land a rebased version that keeps current-main subagent dedupe semantics, propagates stop/reset to active ACP-backed children with bounded cancellation, persists child abort state intentionally, and covers the behavior with focused tests plus a changelog entry.

## Reproduction Assessment

Yes, from source inspection. PR head leaves `getLatestSubagentRunByChildSessionKey` at its default null return in the new test while PR-head `stopSubagentsForRequester` skips null latest runs; the unbounded reset hang is also source-reproducible because reset awaits the cascade and `cancelSession` has no local timeout.

## Solution Assessment

No, not as submitted. Propagating stop to active ACP-backed subagents is the right direction, but the maintainable fix needs a bounded child cancel path, a rebase onto current main, updated tests, and changelog coverage.
