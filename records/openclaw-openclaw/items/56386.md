Keep open: the underlying stop propagation problem still exists on current main, but this PR is not merge-ready because it adds an unbounded awaited ACP child cancel path, has a PR-head regression-test setup failure, and is stale against newer abort/reset changes.

## What This Changes

The PR makes subagent stop propagation async, awaits stop/reset callers, cancels ACP-backed child sessions, persists child aborted state, and adds fast-abort regression coverage.

## Best Possible Solution

Land a rebased fix that preserves current-main subagent dedupe and reply-run behavior, propagates stop/reset to ACP-backed children with bounded cancellation, persists abort state intentionally, and adds focused tests plus changelog coverage.

## Reproduction Assessment

Yes, from source inspection. Current main still does not cancel ACP-backed child sessions in stopSubagentsForRequester, and PR head has a concrete failing test setup plus a reset-awaited unbounded cancel path.

## Solution Assessment

No, not as submitted. Propagating stop to active ACP-backed subagents is the right narrow direction, but the implementation should use the existing bounded cleanup pattern and be rebased over current-main abort/reset semantics.
