## Summary

Keep open: this targets a real source-reproducible stop/reset bug, but the submitted patch still needs contributor real behavior proof and has blocking findings around unbounded ACP child cancellation plus PR-head test setup.

## What This Changes

The PR makes subagent stop propagation async, cancels ACP-backed child sessions during stop/reset, persists child abort state, updates /stop fallback behavior, and adds abort regression coverage.

## Best Possible Solution

Land a rebased implementation that makes subagent stop propagation awaitable, cancels active ACP child sessions through a bounded cleanup path, intentionally persists child abort state, and includes focused tests plus runtime proof.

## Reproduction Assessment

Yes, at source level. Current main's child stop loop clears queues and aborts reply/embedded runs but never cancels ACP child sessions, and reset cleanup currently calls that helper without any ACP child-cancel path.

## Solution Assessment

No, not as submitted. Propagating stop/reset to active ACP-backed children is the right direction, but the implementation needs bounded cancellation, rebase-sensitive test repair, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Bound ACP child cancellation during stop cascade:** `src/auto-reply/reply/abort.ts:199-203`
  - body: The new child-stop path awaits acpManager.cancelSession directly. Because AcpSessionManager.cancelSession awaits an active turn's cancelPromise without a local timeout, and session reset now awaits this cascade, one stuck ACP backend can hang reset/delete instead of following the existing bounded cleanup pattern.
  - confidence: 0.9
- **[P2] Mock the latest child run in the ACP cascade test:** `src/auto-reply/reply/abort.test.ts:600-613`
  - body: On PR head, the new test lists a child run but does not mock getLatestSubagentRunByChildSessionKey. That branch's latest-run guard can skip the run when the mock returns null, so cancelSession is not reached and the stopped count stays 0.
  - confidence: 0.86
