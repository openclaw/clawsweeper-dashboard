## Summary

Keep open. The PR targets a remaining gap because current main still does not cancel ACP-backed child sessions during subagent stop/reset, but the branch should not merge as-is: the new awaited child cancel can hang reset on a stuck ACP backend, the new regression test is stale against the branch setup, and the user-facing fix needs a changelog entry.

## What This Changes

The PR makes subagent stop propagation async, awaits stop/reset callers, cancels ACP-backed child sessions, persists child aborted state, and adds fast-abort regression coverage.

## Best Possible Solution

Land a rebased version that keeps the async stop cascade, cancels ACP-backed child sessions through a bounded cleanup helper, preserves current main's dedupe behavior, persists child aborted state intentionally, and documents the user-facing stop/reset fix in the active changelog.

Yes. A child ACP runtime whose cancel promise never resolves reproduces the reset hang once the PR makes reset await stopSubagentsForRequester; the PR-head test issue is also reproducible from the missing latest-run mock described in the review context.

## Solution Assessment

No, not as proposed. Propagating stop into ACP-backed child sessions is the right direction, but the mergeable solution needs bounded cancellation plus updated tests and changelog coverage on current main.
