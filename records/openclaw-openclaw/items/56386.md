## Summary

Keep open: current main still does not cancel ACP-backed child sessions when stop/reset cascades through active subagents, so the PR is aimed at a real bug, but the submitted patch has blocking correctness issues and lacks required real behavior proof.

## What This Changes

The branch makes subagent stop propagation async, cancels ACP-backed child sessions during stop/reset, persists child abort state, adjusts /stop fallback behavior, and adds abort tests.

## Best Possible Solution

Land a rebased implementation that cancels active ACP-backed child sessions through a bounded stop/reset cleanup path, preserves child abort state, keeps /stop scoped to the resolved target, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level: current main stops child queues and reply/embedded runs but does not cancel ACP-backed child sessions, while gateway reset/delete reaches that helper before bounded target ACP cleanup. I did not run a live ACP setup in this read-only review.

## Solution Assessment

No, not as submitted. The direction is right, but the implementation needs bounded ACP child cancellation, removal or narrowing of the wrapper fallback retry, repaired PR-head test setup, a rebase, and real after-fix behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bound ACP child cancellation during stop cascade:** `src/auto-reply/reply/abort.ts:199-203`
  - body: The child-stop path awaits acpManager.cancelSession directly. cancelSession awaits the active turn cancelPromise without a local timeout, and reset/delete awaits this cascade before the bounded target cleanup, so a stalled ACP backend can hang reset/delete. Wrap child ACP cancellation in the same bounded cleanup pattern or detach on timeout.
  - confidence: 0.91
- **[P2] Keep /stop fallback inside the resolved target:** `src/auto-reply/reply/commands-session-abort.ts:153-157`
  - body: When the resolved target stops zero children, this retries with params.sessionKey. Current main has coverage that a distinct target missing from the store must not fall back to the wrapper session; this can stop unrelated subagents for the command wrapper instead of the requested target.
  - confidence: 0.86
- **[P2] Mock the latest child run in the ACP cascade test:** `src/auto-reply/reply/abort.test.ts:600-613`
  - body: stopSubagentsForRequester filters listed child runs through getLatestSubagentRunByChildSessionKey. This test only mocks the listed run and ACP resolution, so the default null makes the new cancel assertion unreachable and stoppedSubagents stays zero.
  - confidence: 0.88
