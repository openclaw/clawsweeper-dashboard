## Summary

Keep open: current main and the latest shipped release still do not cancel active ACP-backed child sessions from the subagent stop cascade, so the PR is not obsolete, but the submitted branch has blocking correctness issues, is conflicting, and lacks required real behavior proof.

## What This Changes

The PR converts subagent stop propagation to async, adds ACP child-session cancellation and child abort-state persistence, awaits the reset cleanup caller, and adds fast-abort regression coverage.

## Best Possible Solution

Land a rebased implementation that cancels active ACP-backed child sessions with bounded cleanup, preserves current-main subagent dedupe semantics, intentionally persists child abort state, and includes focused current-main regression tests plus real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.7` show the subagent stop loop does not cancel ACP child sessions, and the PR-head blocking issues are visible from the diff and existing review comments.

## Solution Assessment

No, not as submitted. Propagating stop/reset to active ACP-backed children is the right direction, but the implementation needs bounded cancellation, a rebase onto current main, repaired tests, and real behavior proof before merge.
