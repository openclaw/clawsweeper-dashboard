## Summary

Keep open: current main still lacks the central ACP-backed child-session stop behavior, but this PR is not merge-ready because it directly awaits an unbounded ACP cancel path, has a PR-head regression-test setup bug, conflicts with current main, and lacks real behavior proof.

## What This Changes

The branch makes subagent stop propagation async, cancels ACP-backed child sessions during stop/reset, persists child abort state, updates /stop requester fallback behavior, and adds abort regression coverage.

## Best Possible Solution

Land a rebased implementation that propagates stop/reset to active ACP-backed children through a bounded cancellation path, preserves intentional child abort state, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current main stops subagent queues and embedded/reply runs but does not cancel ACP-backed child sessions, and gateway reset uses that helper before the direct bounded ACP cleanup path.

## Solution Assessment

No, not as submitted. The direction is right, but the child ACP cancel needs a bounded cleanup wrapper and the PR needs rebase-sensitive test repair plus real runtime proof before merge.
