## Summary

Keep this PR open: the underlying bug still appears valid on current main, but the submitted branch is conflicting, lacks required real behavior proof, and introduces blocking correctness issues around unbounded ACP cancellation and /stop target scope.

## What This Changes

The PR makes subagent stop propagation awaitable, cancels ACP-backed child sessions during stop/reset, persists child abort state, adjusts /stop fallback behavior, and adds abort regression coverage.

## Best Possible Solution

Land a rebased implementation that cancels ACP-backed child sessions through bounded stop/reset cleanup, preserves child abort state, stays scoped to the resolved target, and includes focused regression tests plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level: current main stops child queues and reply/embedded runs but does not cancel ACP-backed child sessions, and gateway reset reaches that helper before bounded direct-target ACP cleanup. I did not run a live ACP setup in this read-only review.

## Solution Assessment

No, not as submitted. The direction is right, but the implementation needs bounded ACP child cancellation, target-scoped /stop behavior, a repaired/rebased test, and real after-fix behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bound ACP child cancellation during stop cascade:** `src/auto-reply/reply/abort.ts:199-203`
  - body: This awaits acpManager.cancelSession directly for each child. cancelSession awaits activeTurn.cancelPromise without a timeout, and reset/delete now awaits this cascade before the existing bounded target ACP cleanup, so a stalled ACP backend can hang reset/delete.
  - confidence: 0.91
- **[P2] Keep /stop fallback inside the resolved target:** `src/auto-reply/reply/commands-session-abort.ts:153-157`
  - body: When the resolved target stops zero children, this retries with params.sessionKey. That can stop unrelated subagents for the command wrapper instead of the requested target, conflicting with current target-boundary coverage.
  - confidence: 0.86
- **[P2] Mock the latest child run in the ACP cascade test:** `src/auto-reply/reply/abort.test.ts:600-613`
  - body: On this PR head, stopSubagentsForRequester filters listed child runs through getLatestSubagentRunByChildSessionKey. This test only mocks the listed run and ACP resolution, so the default null makes the cancel assertion unreachable and stoppedSubagents stays zero.
  - confidence: 0.88
