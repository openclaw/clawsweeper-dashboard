## Summary

Keep open. Current main still lacks ACP-backed child-session cancellation during subagent stop/reset, but this PR needs bounded ACP cancellation, fixed regression-test setup, and changelog coverage before merge.

## What This Changes

The PR makes subagent stop propagation async, awaits stop/reset callers, cancels ACP-backed child sessions, persists child aborted state, and adds fast-abort regression coverage.

## Best Possible Solution

Land a rebased version that keeps the async stop cascade, bounds child ACP cancellation, preserves current-main subagent dedupe semantics, intentionally persists child aborted state, and documents the stop/reset fix in the active changelog.

## Reproduction Assessment

Yes. On the PR head, the new ACP cascade test can be reproduced by leaving `getLatestSubagentRunByChildSessionKey` at its default null return; a child ACP cancel promise that never resolves also reproduces the reset hang once the cascade is awaited.

## Solution Assessment

No, not as submitted. Propagating stop into ACP-backed child sessions is the right direction, but the mergeable solution needs a bounded cancellation path plus updated tests and changelog coverage.
