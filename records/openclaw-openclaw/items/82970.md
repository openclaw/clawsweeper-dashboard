## Summary

Keep this issue open: it is a valid beta-package regression report and it is already paired with an open fixing PR that uses closing syntax, so it should close only after that PR lands or is replaced.

## What This Changes

The issue reports that `2026.5.16-beta.4` shipped a Control UI bundle advertising Gateway protocol 4 while the bundled gateway expected protocol 5, blocking dashboard WebSocket entry.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/82971 so release packaging fails when packed Control UI JS advertises protocol levels different from `src/gateway/protocol/version.ts`.

## Reproduction Assessment

Yes for the failure mechanism, but not as a current-main failing checkout: the server rejects a client range that excludes the gateway protocol, and the issue gives concrete beta-package evidence. Current main has the protocol constants back at 4, so the remaining proof should be package-level release validation.

## Solution Assessment

Yes: a release-package guard is the narrow maintainable fix for the remaining risk, because source constants are aligned on main but a publish artifact can still drift if stale Control UI JS is packed.
