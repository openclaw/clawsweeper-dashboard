## What This Changes

Report that the 2026.5.16-beta.4 npm package shipped a stale Control UI bundle advertising Gateway protocol 4 while the bundled gateway required protocol 5, blocking dashboard WebSocket handshake.

## Best Possible Solution

Land a narrow release-package guard that rejects packed Control UI protocol levels inconsistent with src/gateway/protocol/version.ts, then close this issue from the merged PR or release proof.

## Reproduction Assessment

Yes for the shipped beta artifact: streaming the 2026.5.16-beta.4 npm tarball shows Control UI min/max protocol 4 while the bundled gateway version chunk is 5. No current-main dashboard launch was run, and current main has since moved back to protocol 4.

## Solution Assessment
