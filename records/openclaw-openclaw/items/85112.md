## What This Changes

The PR adds a default-off Matrix `threadBindings.bypassMentionInBoundThreads` config option, wires it through monitor startup, lets thread-scoped session records or runtime bindings bypass the room mention gate, updates generated config metadata, tests, and Matrix docs.

## Best Possible Solution

Land the default-off Matrix option after maintainer sign-off on the delivery semantics and normal CI, preserving the focused regression coverage and docs.

## Reproduction Assessment

Yes, by source inspection: current main's Matrix handler only bypasses `requireMention` for authorized control commands and drops other unmentioned room messages. The linked issue and PR body also provide live logs for the same no-mention in-thread drop.

## Solution Assessment

Yes, the implementation shape is the maintainable one if maintainers want this behavior: it is default-off, thread-scoped, idle-bounded for session records, and keeps room-level bindings/sessions from satisfying the bypass. Final approval is a product/message-delivery policy decision rather than a mechanical code blocker.
