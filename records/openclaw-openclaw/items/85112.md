## Best Possible Solution

Land a documented, default-off Matrix mention-bypass only after maintainers accept the channel-level policy and have real patched Matrix runtime proof for both default-off and opt-in behavior.

## Reproduction Assessment

I did not run a live Matrix reproduction in this read-only review.

## Solution Assessment

Unclear as merge-ready: the default-off gate is the safer direction, but the new channel-level policy needs docs, maintainer acceptance, and after-fix live Matrix proof before it is the best final solution.
