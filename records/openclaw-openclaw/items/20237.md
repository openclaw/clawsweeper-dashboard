## Summary

Keep open. Current main partially covers the request with cron editing, one-shot job controls, Web Push settings, backend cron delivery hooks, and inline context-pressure UI, but it still lacks the central generic Control UI toast queue, actionable cron-result popup layer, and snooze/reschedule flow.

## What This Changes

The issue requests a native Control UI notification/toast system, actionable cron-result popups with snooze/reschedule controls, inline cron management, one-shot cron UI exposure, and context-pressure notifications.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, wire cron-finished events into actionable result notifications, specify durable snooze/reschedule behavior, and decide whether context pressure stays inline-only or also emits notification prompts.
