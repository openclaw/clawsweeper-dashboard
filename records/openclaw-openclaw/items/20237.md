## Summary

Keep open. Current main has partial coverage for context usage/compaction, cron editing/history/delete-after-run, Web Push subscription plumbing, and the Notifications settings tab, but it still lacks the central requested generic in-app notification queue, actionable cron-result popups, and defined cron snooze/reschedule semantics.

## What This Changes

Request to add a native WebUI toast/notification system with actionable cron result popups, inline cron management, one-shot job UI, snooze/reschedule controls, and context-pressure monitor/flush integration.

## Best Possible Solution

Decompose this into focused product work: a generic connected-Control-UI notification contract, typed portable actions, cron-finished result integration, explicit non-destructive snooze/reschedule semantics, and a separate decision on whether context pressure remains inline-only or also emits notifications.

## Solution Assessment

No, not as one bundled implementation. The best path is maintainer-scoped product decomposition that reuses the existing cron, context, Web Push, and portable-action foundations instead of adding a broad ad hoc UI layer.
