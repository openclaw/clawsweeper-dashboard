## Summary

Keep open. Current main partially covers the request with cron editing, one-shot job UI, backend cron delivery hooks, Web Push settings, and inline context-pressure UI, but the central in-app toast queue, actionable cron-result popup layer, and snooze/reschedule flow are still missing.

## What This Changes

The issue requests native Control UI notifications/toasts with cron result actions, inline cron management, one-shot job controls, snooze/reschedule, and context usage warnings with a flush action.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, wire cron-finished events into actionable result notifications, specify durable snooze/reschedule behavior, and decide whether context pressure stays inline-only or also emits notification prompts.
