## Summary

Keep open. Current main still only partially covers the feature: cron editing, delete-after-run UI, run history, Web Push subscription/test plumbing, and inline context pressure compaction exist, but the central generic in-app toast queue, actionable cron-result popup layer, and snooze/reschedule flow are still missing.

## What This Changes

Request to add a native Control UI notification/toast system with actionable cron result popups, inline cron management and snooze/reschedule controls, one-shot cron exposure, and context usage warnings with a flush/compact action.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify durable snooze/reschedule behavior, and decide whether context pressure remains inline-only or also emits notification prompts before implementing focused slices with tests and docs.
