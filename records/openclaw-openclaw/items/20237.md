## Summary

Keep open. Current main and v2026.5.22 still only cover pieces of the request: cron edit/history/delete-after-run UI, Web Push subscription plumbing, backend cron delivery hooks, and inline context pressure UI exist, but there is still no generic Control UI toast queue, actionable cron-result popup layer, or cron snooze/reschedule flow.

## What This Changes

Requested a native WebUI notification system with actionable cron result popups, snooze/reschedule controls, inline cron management, one-shot cron exposure, and context usage warnings with compact/flush actions.

## Best Possible Solution

Define one connected Control UI notification queue contract, then split implementation into scoped follow-ups for cron-finished actionable notifications, explicit snooze/reschedule semantics, and context-pressure notification behavior.

## Solution Assessment

No concrete patch is proposed, and a single broad UI-only fix is not the best path. The maintainable solution is scoped product/API work that reuses existing cron events, Web Push infrastructure, and session context state under one notification contract.
