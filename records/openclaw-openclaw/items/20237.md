## Summary

Keep open; current main partially covers the request with Web Push subscription plumbing, cron CRUD/history/delete-after-run UI, and an inline context-pressure compact notice, but it still lacks the requested generic in-app notification queue, actionable cron-result popups, and snooze/reschedule flow. The prior issue discussion also explicitly kept this open for the same remaining product gaps.

## What This Changes

Add a native Control UI notification/toast system with actionable cron result popups, snooze/reschedule controls, one-shot cron exposure, and context-pressure warnings with compact/flush actions.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, wire cron-finished events into actionable result notifications, specify durable snooze/reschedule behavior, and decide the context-pressure notification UX.
