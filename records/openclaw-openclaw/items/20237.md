Keep open. Request to add a Control UI notification/toast system with actionable cron-result popups, inline cron management, snooze/reschedule controls, one-shot cron UI, and context-usage warnings with flush/save actions.

## Best Possible Solution

Define one connected Control UI notification queue contract, wire cron-finished events into actionable result notifications, specify snooze/reschedule semantics, and decide whether context pressure remains inline-only or also emits notification prompts.

## Solution Assessment

No concrete patch is proposed, and a single automated UI-only fix is not the best path. The maintainable solution is scoped product/API work that reuses existing cron events, Web Push plumbing, and session context state under one notification contract.
