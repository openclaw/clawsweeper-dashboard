## Summary

Keep open; current main partially covers Web Push, cron editing/run history/delete-after-run, and inline context pressure notice, but it still lacks the central toast queue, actionable cron-result popup layer, and snooze/reschedule UI, while #53228 and #74715 cover only narrower open slices.

## What This Changes

Request adds a native Control UI notification/toast system with cron-result management popups, snooze/reschedule actions, one-shot cron UI exposure, and context usage warnings with a compact/flush action.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify durable snooze/reschedule behavior, and decide whether context pressure remains inline-only or also emits notification prompts.
