## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict thumb-only extraction to real photos:** `extensions/zalouser/src/zalo-js.ts:295`
  - body: The new fallback treats any attachment object with a `thumb` as image media, but zca-js link payloads also carry `href` and `thumb` for link previews while `href` points at the web page. A normal link preview can therefore trigger `saveRemoteMedia` against non-image content and then be relabeled as `image/jpeg`; keep extraction to `type === "photo"` or an image href unless the Zalo contract is stronger.
  - confidence: 0.88
- **[P2] Bound the inbound media fetch before awaiting it:** `extensions/zalouser/src/monitor.ts:1104-1108`
  - body: This awaits `saveRemoteMedia` on every detected photo without passing `timeoutMs` or `readIdleTimeoutMs`. The shared media fetch helper only applies those abort/idle guards when callers provide them, so a slow CDN response can stall `processMessage` before the bot assembles or replies to the turn.
  - confidence: 0.9
- **[P2] Redact the failed media URL before logging:** `extensions/zalouser/src/monitor.ts:1129`
  - body: On fetch failure this logs `params.media.url` verbatim. Zalo CDN URLs can carry identifiers or query material, and the shared media fetch path already redacts URLs before including them in errors, so this wrapper should redact or omit the URL too.
  - confidence: 0.91
