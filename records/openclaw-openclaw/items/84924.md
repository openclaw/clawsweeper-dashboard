## Best Possible Solution

Land the SDK-aligned Zalo inbound media path after redacting failed media URLs and preserving the real-run proof; add a focused monitor-level media assertion if maintainers want extra regression coverage.

## Solution Assessment

Yes, conditionally: using `saveRemoteMedia` and `toInboundMediaFacts` is the maintainable SDK-aligned fix for native inbound media. The patch should redact failure logging before merge; a monitor-level context-payload test would be useful but is secondary to that blocker.
