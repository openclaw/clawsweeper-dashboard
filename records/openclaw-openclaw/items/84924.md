## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict thumb-only extraction to real photos:** `extensions/zalouser/src/zalo-js.ts:295`
  - body: The new fallback treats any content object with a non-empty `thumb` as image media, but the verified `zca-js` attachment shape only shows generic `href`, `thumb`, and `type` fields. A link preview or other attachment can therefore fetch a non-image `href` and later be labeled as image content; keep extraction to `type === "photo"`, an image URL, or a stronger proven Zalo contract.
  - confidence: 0.88
- **[P2] Bound the inbound media fetch before awaiting it:** `extensions/zalouser/src/monitor.ts:1104-1108`
  - body: This awaits `saveRemoteMedia` on every detected photo without passing `timeoutMs` or `readIdleTimeoutMs`. The shared media fetch helper only applies those guards when callers provide them, so a slow CDN response can stall `processMessage` before the bot assembles or replies to the turn.
  - confidence: 0.9
- **[P2] Redact the failed media URL before logging:** `extensions/zalouser/src/monitor.ts:1129`
  - body: On fetch failure this logs `params.media.url` verbatim. Zalo CDN URLs can carry identifiers or query material, and the shared media fetch path already redacts URLs before including them in errors, so this wrapper should redact or omit the URL too.
  - confidence: 0.91
