## Summary

Keep open. The PR targets a real Telegram plugin-command gap on current main, but the latest head still hand-rolls session-file resolution instead of reusing the shared resolver semantics, and it is missing the required changelog entry for a user-facing Telegram fix.

## What This Changes

The PR adds Telegram native plugin-command session metadata resolution, passes sessionId/sessionFile into plugin command handlers, and adds a topic-scoped /codex regression test.

## Best Possible Solution

Land a narrow Telegram plugin-command fix that resolves and persists the active topic transcript through the shared session-file resolver semantics, exposes any needed generic SDK helper rather than importing core internals, covers stale/relative/out-of-dir metadata cases, and records the Telegram fix in CHANGELOG.md.

## Reproduction Assessment

Yes. Current main can be reproduced statically: Telegram native plugin commands do not pass sessionFile, while /codex bind requires ctx.sessionFile; a live reproduction would be an authorized Telegram forum topic /codex bind --cwd <dir> invocation.

## Solution Assessment

No. Passing sessionId/sessionFile is the right direction, but this implementation is not the best fix until it reuses the shared resolver/persist path and adds the required changelog entry.
