## Summary

Keep open. The PR targets a real Telegram topic plugin-command gap, but the current patch has blocking correctness and security issues: it adds an unrelated loader file with stale imports and hand-builds session transcript paths instead of using the shared session-file resolver.

## What This Changes

The PR adds Telegram native plugin-command session metadata resolution, passes sessionId/sessionFile into plugin command execution, and adds a topic-scoped regression test.

## Best Possible Solution

Land a focused Telegram plugin-command fix that reuses shared session-file resolver semantics, preserves transcript-path containment, covers topic command routing, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Current main can be reproduced statically: Telegram plugin commands omit sessionFile, while /codex bind explicitly fails when sessionFile is absent; a live repro would be an authorized Telegram forum-topic /codex bind invocation.

## Solution Assessment

No. Passing sessionId/sessionFile is the right direction, but this patch is not the best solution until it removes the unrelated loader file, uses the shared resolver/persist path, and adds changelog coverage.
