## Summary

Keep open. The PR addresses a real Telegram topic plugin-command gap on current main, but the branch still duplicates session-file resolution instead of using the shared resolver and is missing the required changelog entry for a user-facing Telegram fix.

## What This Changes

The PR adds Telegram plugin-command session metadata resolution, passes sessionId/sessionFile into plugin command handlers, and extends the Telegram session metadata test for topic /codex routing.

## Best Possible Solution

Land a narrow Telegram fix that resolves and persists the active topic session file through the shared session-file helper, passes normalized sessionId/sessionFile into plugin command handlers, covers topic /codex bind plus stale or relative session metadata, and records the fix in CHANGELOG.md.

## Reproduction Assessment

Yes. Current main can be reproduced statically by tracing Telegram topic /codex bind to executePluginCommand without sessionFile and then to the Codex bind handler that rejects missing ctx.sessionFile; a live check would be an authorized Telegram forum-topic /codex bind invocation.

## Solution Assessment

No. Passing sessionId and sessionFile is the right direction, but the PR should route through the shared resolver/persist path instead of duplicating path construction and stale metadata handling inside the Telegram plugin.
