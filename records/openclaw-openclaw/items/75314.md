## Summary

Keep open. Current main still has a source-level WebChat `/reset` path where a no-model gateway acknowledgement is persisted as a `gateway-injected` assistant message and rendered like a normal chat message; draft PR #74733 already targets the same display and dispatch contract.

## What This Changes

The issue asks WebChat fresh `/new` or `/reset` sessions to avoid showing only a gateway-injected assistant-looking acknowledgement without a real startup greeting or lifecycle divider.

## Best Possible Solution

Review and land PR #74733, or a narrower equivalent, that keeps bare reset/new acknowledgements no-model while rendering them as lifecycle dividers and refreshing the active WebChat history.

## Reproduction Assessment

Yes, source-level. Typed WebChat `/reset` dispatches through `chat.send`, the reset handler returns a synthetic no-model acknowledgement, and WebChat currently renders only compaction markers as lifecycle dividers.

## Solution Assessment

Yes for the planned direction, not current main. Separating gateway acknowledgements from assistant output is narrower than forcing every bare reset to run a model greeting, and draft PR #74733 is already the implementation to review.
