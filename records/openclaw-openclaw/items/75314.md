## Summary

Keep open. Current main still has the WebChat/no-startup-run behavior, and related duplicate cleanup now points WebChat-specific reports back here while the broader bare reset contract remains open separately.

## What This Changes

WebChat fresh sessions can show an injected `✅ New session started.` assistant-looking acknowledgement without starting a model-backed startup greeting.

## Best Possible Solution

Define one reset/new UX contract, then align `maybeHandleResetCommand`, `sessions.create`, gateway-injected transcript rendering, WebChat lifecycle UI, docs, and regression tests around either a real startup turn or a clearly non-assistant acknowledgement.

## Reproduction Assessment

Yes. Source inspection on current main shows bare reset/new short-circuits before model execution, Control UI creates sessions without an initial `task` or `message`, and WebChat renders gateway-injected assistant-shaped messages as ordinary chat messages.

## Solution Assessment

Unclear. Removing the short-circuit would restore the historical greeting for some users, but current docs and tests intentionally assert no-model acknowledgements, so maintainers should choose the UX contract before implementation.
