## Summary

Keep open. Current main and the latest release still have the source-level behavior, while current docs and tests intentionally encode no-model bare reset acknowledgements, so this needs maintainer reset/UX contract judgment rather than cleanup closure.

## What This Changes

The issue asks WebChat/Control UI fresh sessions to avoid showing only a gateway-injected reset acknowledgement when no startup greeting or model turn has run.

## Best Possible Solution

Define the reset/new contract once, then align gateway behavior, WebChat rendering, docs, and tests around either a real startup turn or a clearly non-assistant lifecycle acknowledgement.

## Reproduction Assessment

Yes, source-level. Current main has a bare reset/new short-circuit, Control UI session creation without an initial run, and WebChat rendering that treats gateway-injected assistant messages as ordinary chat messages.

## Solution Assessment

Unclear. Removing the short-circuit would restore the reported greeting for some users, but it conflicts with current no-model docs/tests; maintainers should first choose the reset/new UX contract and then update code, docs, and tests together.
