## Summary

Keep this issue open. Current main still does not expose or honor a spawn-time `tools` allowlist on `sessions_spawn`; schema, executor forwarding, native spawn params, ACP spawn params, docs, and the effective tool-policy pipeline all remain based on profile/config/sandbox/subagent policy layers rather than a per-call policy object.

## What This Changes

The issue requests a `tools` allowlist parameter on `sessions_spawn` so callers can configure sub-agent tools at spawn time instead of creating separate agent profiles.

## Best Possible Solution

If approved, ship a schema-visible `sessions_spawn.tools.allow/deny` contract that validates input, carries it through both native and ACP spawn setup as trusted runtime metadata, and applies it after the existing policy layers as a deny-wins narrowing filter, with docs and focused tests for precedence, invalid entries, empty allowlists, and native/ACP parity.
