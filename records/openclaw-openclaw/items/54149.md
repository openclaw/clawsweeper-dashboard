## Summary

Keep open. Current main still carries the reported Volcengine/BytePlus catalog mismatch: the affected Kimi rows remain capped at 4096 output tokens, omitted `reasoning` still becomes `false`, and the adjacent Together catalog advertises Kimi K2.5 with `reasoning: true` and `maxTokens: 32768`. Official BytePlus docs support a larger Kimi output limit and a force-enabled thinking variant, so this needs a provider-catalog repair rather than cleanup closure.

The issue requests correcting Volcengine/Doubao and BytePlus Kimi model catalog metadata for reasoning support and output-token limits, especially Kimi K2.5 and Kimi K2 Thinking.

## Best Possible Solution

Keep the issue open and update the provider-owned Volcengine and BytePlus manifests with vendor-backed `maxTokens` and `reasoning` values for the affected Kimi rows, with focused tests proving those manifest values flow through provider builders and catalog augmentation. Provider docs should be updated only if the corrected metadata changes user-visible model/status guidance.
