## What This Changes

The issue reports Windows npm-installed one-shot CLI commands such as `status` and `--version`/version symptoms remaining alive as lingering `node.exe openclaw.mjs` processes after completing.

## Best Possible Solution

Review and land PR #74425, or a narrower replacement, that makes short-lived Windows npm CLI invocations terminate promptly after supported commands like `status` and root `--version` without changing long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process/timing evidence, and current main has a source-backed `status` route that returns without explicit success-path shutdown; I did not rerun Windows live proof here.

## Solution Assessment
