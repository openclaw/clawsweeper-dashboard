## Summary

Keep open. This is a current Windows npm CLI lifecycle regression with a concrete `status` reproduction, current `main` still returns from the routed `status` command without a normal-success shutdown, and open PR #74425 is a linked implementation candidate rather than a reason to close before merge.

## What This Changes

The issue reports that short-lived Windows npm CLI commands such as `status` and `--version` remain running as `node.exe ... openclaw.mjs` processes after command completion.

## Best Possible Solution

Review and land PR #74425, or a narrower replacement, so short-lived Windows npm CLI commands terminate promptly after `status` and `--version` while preserving long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands, timing observations, and process separation from the gateway; current `main` still has the routed `status` success path returning without forced shutdown.

## Solution Assessment

Unclear. Keeping this issue open while PR #74425 is open is the right state, but maintainers should decide whether explicit process exit or targeted cleanup of the lingering handle is the narrowest maintainable fix.
