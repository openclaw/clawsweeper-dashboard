## Summary

Keep open. The report has a concrete Windows npm reproduction, current main still returns from the routed `status` command without a normal-success shutdown, and open PR #74425 explicitly references this issue as its fix candidate, so the issue should remain open until that PR lands or is closed.

## What This Changes

The issue asks for Windows npm CLI commands such as `status` and version invocations to terminate instead of leaving `node.exe ... openclaw.mjs` processes alive after completion.

## Best Possible Solution

Review and land PR #74425, or a narrower replacement, so short-lived Windows npm invocations for `status` and `--version` terminate promptly while long-running gateway processes keep their existing lifecycle; include targeted Windows smoke or regression coverage.

## Reproduction Assessment

Yes. The reporter provided a concrete Windows 11 npm reproduction with process timing and gateway separation, and current main still follows the routed `status` success path that returns without explicit shutdown. I did not independently run Windows from this Linux review.
