## Summary

Keep open: this is a concrete Windows npm CLI lifecycle bug, current main still returns from the routed `status` success path without a one-shot shutdown, and PR #74425 is an open unmerged closing PR for the same report.

## What This Changes

Short-lived Windows npm CLI invocations such as `openclaw status` and `--version` should exit after completing instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that guarantees short-lived Windows npm CLI commands terminate while preserving long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process timing evidence, and current main still has the source-backed routed `status` path that returns without a forced one-shot shutdown; I did not run Windows locally.

## Solution Assessment

Unclear. The right boundary is short-lived CLI teardown, but maintainers should decide whether the open PR's explicit exit approach or targeted cleanup of the lingering Windows handle is more maintainable.
