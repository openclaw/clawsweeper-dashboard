## Summary

Keep open: the Windows npm CLI lifecycle report remains source-reproducible on current main, and PR #74425 is the active unmerged closing path in the provided context.

## What This Changes

The issue reports that short-lived Windows npm CLI invocations such as `openclaw.mjs status` and `--version` remain as `node.exe` processes after command completion.

## Best Possible Solution

Review and land PR #74425 or a narrower lifecycle cleanup that terminates short-lived packaged Windows CLI commands after `status`/`--version` while preserving long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process timing evidence, and current main still has a source-backed routed `status` success path that returns without a one-shot shutdown; I did not live-run Windows here.

## Solution Assessment

Unclear. A one-shot CLI lifecycle fix is the right boundary, but maintainers should decide whether PR #74425's explicit exit approach or targeted cleanup of the lingering handle is more maintainable.
