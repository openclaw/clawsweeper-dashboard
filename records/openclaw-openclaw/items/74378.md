## Summary

Keep open: the Windows/npm lifecycle report is concrete, current main still lets the routed `status` success path return without an explicit one-shot shutdown, and open PR #74425 references this issue with closing syntax but is not merged.

## What This Changes

OpenClaw CLI one-shot commands such as `status` and root `--version` should exit promptly on Windows npm installs instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Land PR #74425 or a narrower replacement that terminates short-lived Windows npm CLI invocations after `status` and root `--version` while preserving long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process/timing evidence, and current main has a source-backed routed `status` path that returns without forced one-shot shutdown; I did not run a live Windows smoke from this Linux checkout.

## Solution Assessment

Unclear. A narrow one-shot CLI lifecycle fix is the right direction, but maintainers should confirm whether explicit `process.exit` or targeted cleanup of the lingering handle is the most maintainable implementation.
