## Summary

Keep open: the Windows/npm reproduction is concrete, current main still lets the routed `status` command return without a one-shot process shutdown, and PR #74425 references this issue with closing syntax but is still open and unmerged.

## What This Changes

The issue reports that Windows npm-installed `openclaw.mjs status` and `--version` leave short-lived `node.exe` CLI processes alive after command completion.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that terminates short-lived Windows npm CLI invocations after `status` and root `--version` while preserving long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process/timing evidence, and current main has a source-backed routed `status` path that returns without forced one-shot shutdown; I did not run a live Windows smoke from this Linux checkout.

## Solution Assessment

Unclear. A narrow one-shot CLI lifecycle fix is the right direction, but maintainers should confirm whether explicit `process.exit` or targeted cleanup of the lingering handle is the most maintainable implementation.
