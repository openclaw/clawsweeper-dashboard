## Summary

Keep open: the Windows npm CLI lifetime bug still has a source-backed current-main path, and PR #74425 is an open unmerged closing PR for the same report, so this issue should stay open until that PR or a replacement lands.

## What This Changes

The issue requests that Windows npm CLI commands such as `status` and `--version` terminate after completion instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that makes short-lived Windows npm CLI invocations exit after `status` and `--version` while preserving long-running gateway/runtime commands.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process/timing evidence, and current main still has a source-backed routed `status` success path that returns without a one-shot shutdown; I did not live-run Windows here.

## Solution Assessment

Unclear. A CLI lifecycle fix is the right boundary, but maintainers should choose whether PR #74425's explicit `process.exit` approach or targeted cleanup of the lingering handle is more maintainable.
