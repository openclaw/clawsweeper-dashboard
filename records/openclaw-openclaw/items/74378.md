## Summary

Keep open. The report remains actionable on current main: the supported `status` path still returns from the routed CLI path without a normal-success process shutdown, and open PR #74425 is a linked implementation candidate rather than grounds to close this issue before merge.

## What This Changes

The issue requests that short-lived Windows npm CLI commands such as `status` and version checks terminate promptly instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Review and land PR #74425, or a narrower replacement, so short-lived Windows npm CLI invocations such as `status` terminate promptly while long-running gateway processes continue to stay alive by design.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands and process/timing evidence, and current-main source still shows the routed `status` success path returning without a normal-success shutdown.
