## Summary

Keep open. The Windows/npm CLI lifecycle report has concrete reproduction details, current main still routes `status` through a success path that returns without an explicit one-shot CLI shutdown, and open PR #74425 references this issue with closing syntax, so the issue should remain open until that PR or a replacement lands.

## What This Changes

The issue reports Windows npm-installed OpenClaw CLI commands such as `status` and `--version` leaving short-lived `node.exe ... openclaw.mjs` processes alive after command completion.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that makes short-lived Windows npm CLI invocations terminate promptly after `status` and root `--version`, with coverage that does not change long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with timing/process evidence, and current main has a source-backed `status` success path that returns without explicit one-shot shutdown; I did not rerun a Windows smoke in this Linux checkout.
