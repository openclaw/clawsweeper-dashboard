## Summary

## What This Changes

The issue reports that supported short-lived OpenClaw CLI commands on Windows, especially `status` and `--version`, can leave `node.exe openclaw.mjs` processes alive after completing.

## Best Possible Solution

Land or replace the linked launcher lifecycle fix with native Windows npm proof that `status`, `gateway status --json`, and `--version` exit cleanly without truncating output and without forcing the compile-cache respawn parent path.

## Reproduction Assessment

This read-only review did not run a native Windows process-list reproduction.

## Solution Assessment

Yes, the narrow launcher lifecycle direction is the best apparent fix, but it is not complete until it is rebased and proven on Windows without stdout/stderr truncation. A broad forced-exit change should stay limited to short-lived command paths and avoid the compile-cache respawn parent.
