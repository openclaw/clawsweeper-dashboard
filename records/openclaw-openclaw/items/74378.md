## Summary

Keep open: the Windows npm CLI hang report is concrete, current main still has routed short-lived CLI success paths that return without a one-shot shutdown, and open PR #74425 references this issue with closing syntax but has not merged.

## What This Changes

Short-lived Windows npm invocations such as `openclaw.mjs status` and `openclaw.mjs --version` should exit after completion instead of leaving `node.exe` processes alive.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that terminates completed short-lived CLI invocations on Windows while preserving long-running gateway/runtime behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process timing evidence, and current main has a source-backed routed `status` path that returns without explicit one-shot shutdown; I did not live-run Windows here.

## Solution Assessment

Unclear. A short-lived CLI lifecycle fix is the right direction, but maintainers should decide whether explicit `process.exit` or targeted cleanup of the lingering handle is the most maintainable implementation.
