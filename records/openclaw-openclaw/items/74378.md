## Summary

Keep open. The report is current and concrete, current main still returns from the routed `status` path without a normal-success shutdown, and open PR #74425 explicitly links this issue as the implementation candidate rather than resolving it before merge.

## What This Changes

The issue requests that short-lived Windows npm CLI commands such as `openclaw.mjs status` and `openclaw.mjs --version` terminate after completion without lingering `node.exe` processes.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that closes the lingering resource for short-lived Windows npm CLI commands, with Windows smoke coverage for `status` and `--version` and no behavior change for long-running gateway or node processes.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands, timing observations, memory growth, and separation from the healthy gateway process; current source confirms `status` is a supported short-lived route, though this read-only review did not rerun Windows locally.

## Solution Assessment

Unclear. Forcing process exit may solve the symptom, but current tests intentionally reject forced exit after a successful routed command, so maintainers should confirm whether targeted teardown of the lingering resource is safer than a broad explicit exit.
