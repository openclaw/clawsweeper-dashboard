Keep open: current main still lacks the launcher final-exit/version-await fix for the Windows npm CLI hang, and the open closing PR at https://github.com/openclaw/openclaw/pull/74425 remains unmerged pending Windows npm real-behavior proof.

## What This Changes

Reported Windows npm-installed OpenClaw CLI commands such as `status` and `--version` should exit after completion instead of leaving lingering `node.exe ... openclaw.mjs` processes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74425, or an equivalent narrow launcher lifecycle fix, after Windows npm smoke proof confirms `status` and `--version` exit without lingering `node.exe` processes.

## Solution Assessment
