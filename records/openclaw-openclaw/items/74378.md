## Summary

Keep open: the Windows/npm CLI lifecycle report is concrete, current main still lets the routed `status` success path return without an explicit one-shot shutdown, and open PR #74425 references this issue with closing syntax but is not merged.

## What This Changes

The issue asks for short-lived Windows npm OpenClaw CLI invocations such as `status` and version checks to exit instead of lingering as `node.exe openclaw.mjs` processes.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that makes short-lived Windows npm CLI invocations terminate promptly after `status` and root `--version` without changing long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands with process/timing evidence, and current main has a source-backed `status` route that returns without forced one-shot shutdown; I did not run a live Windows smoke from this Linux checkout.

## Solution Assessment

Unclear. A narrow one-shot CLI lifecycle fix is the right direction, but maintainers should confirm whether explicit process exit or targeted cleanup of the lingering handle is the most maintainable implementation.
