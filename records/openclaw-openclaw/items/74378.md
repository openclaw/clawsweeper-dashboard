## Summary

Keep open. The report is current, has a concrete Windows npm reproduction, current main still returns from the routed `status` command without a normal-success shutdown, and open PR #74425 is a linked implementation candidate rather than a reason to close before merge.

## What This Changes

The issue reports that Windows npm CLI invocations such as `status` and `--version` leave short-lived `node.exe ... openclaw.mjs` processes running after command completion.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that ensures short-lived Windows npm CLI commands terminate promptly after `status` and `--version`, with Windows smoke coverage and no behavior change to long-running gateway processes.

## Reproduction Assessment

Yes. The reporter provided concrete Windows 11 npm commands, timing observations, process separation from the gateway, and current source still shows the relevant routed `status` success path returning without forced shutdown.
