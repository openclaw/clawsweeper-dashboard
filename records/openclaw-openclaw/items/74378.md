Keep open. The report is current, has a concrete Windows npm reproduction, current main still returns from the routed `status` command without a normal-success shutdown, and open PR #74425 is a linked implementation candidate rather than a reason to close before merge.

## What This Changes

Ensure short-lived Windows npm CLI commands such as `openclaw status` and root version checks terminate after completion instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that makes supported short-lived CLI commands exit promptly on Windows npm installs, with focused lifecycle coverage and Windows smoke proof while preserving long-running Gateway behavior.
