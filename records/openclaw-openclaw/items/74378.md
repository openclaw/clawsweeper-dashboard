## Summary

Keep open: the Windows npm lifecycle report is concrete, current main still has a routed `status` success path that returns without an explicit one-shot shutdown, and open PR #74425 still references this issue with closing syntax but has not merged.

## What This Changes

Short-lived Windows npm CLI invocations such as `status` should exit promptly instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Review and land PR #74425 or a narrower replacement that makes supported short-lived Windows npm CLI invocations terminate promptly without changing long-running gateway behavior.

Yes. The reporter supplied Windows 11 npm commands and process/timing evidence for `node <npm-global-openclaw>\openclaw.mjs status`, and current main still routes `status` through a path that returns without explicit one-shot process shutdown.

## Solution Assessment

Unclear. A narrow lifecycle fix is appropriate, but PR #74425's explicit `process.exit` approach should be reviewed against targeted cleanup of the lingering handle.
