## Summary

Keep open. The report has concrete Windows/npm reproduction evidence, current main still returns from the routed `status` success path without a normal-success shutdown or dispatcher cleanup, and open PR #74425 is a linked implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue reports that short-lived Windows npm CLI invocations such as `openclaw.mjs status` and `openclaw.mjs --version` leave their `node.exe` process alive after command completion.

## Best Possible Solution

Land PR #74425 or a narrower replacement that makes Windows npm short-lived `status` and `--version` invocations terminate promptly after output, without changing long-running gateway behavior, and add focused Windows/npm smoke or lifecycle regression coverage plus a changelog entry if the fix is user-facing.

## Reproduction Assessment

## Solution Assessment
