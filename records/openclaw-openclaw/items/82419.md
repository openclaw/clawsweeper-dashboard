## Summary

Keep this PR open: the follow-up improves failed-delta retry behavior, but the patch still has a blocking non-prefix closeout bug and only provides mocked-boundary proof rather than a live Feishu/CardKit run.

## What This Changes

The PR changes Feishu CardKit streaming to send suffix-only content updates, track successfully sent text, start the streaming content element empty, and update focused streaming-card tests.

## Best Possible Solution

Keep the Feishu plugin-local fix, but handle non-prefix closeout rewrites with a replacement-capable CardKit path or guaranteed monotonic append semantics, then back it with focused tests and redacted live Feishu proof.

## Reproduction Assessment

No high-confidence live reproduction was established here. Source inspection confirms current main sends accumulated text to the CardKit content endpoint and the linked issue has concrete steps, but the visible Feishu client append behavior was not replayed in a real tenant.

## Solution Assessment

No. Suffix deltas are a plausible narrow workaround, but this patch still appends the whole final answer for non-prefix closeouts and lacks live Feishu/CardKit proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle non-prefix CardKit closeout updates:** `extensions/feishu/src/streaming-card.ts:473`
  - body: The final close path still sends `resolveStreamingCardAppendContent(this.state.sentText, text)` to the content endpoint. When the dispatcher clears a transient tool/status line before `close()`, the final text is shorter or non-prefix; the helper then returns the whole final answer, so clients exhibiting the append behavior targeted here will keep the stale status and append the final answer instead of replacing it. Route non-prefix closeouts through a replacement-capable path or avoid append payloads for those cases.
  - confidence: 0.86
