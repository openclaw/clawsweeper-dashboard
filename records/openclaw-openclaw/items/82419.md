## Summary

Keep this PR open: it is an active, plausible fix for a real Feishu CardKit streaming bug, but the latest head still has a non-prefix rewrite gap and the supplied proof is mocked-boundary test output rather than live Feishu/CardKit behavior.

## What This Changes

The PR changes Feishu CardKit streaming to track successfully sent text and PUT suffix-only content deltas, with focused tests for throttled, boundary, and failed-update cases.

## Best Possible Solution

Keep the Feishu plugin-local delta approach, but make non-prefix rewrites use a replacement-capable path or preserve monotonic append semantics, then back it with focused tests and redacted live Feishu/CardKit proof.

## Reproduction Assessment

No high-confidence live reproduction was established here. Source inspection of current main does show the reported path sends accumulated text to the CardKit content update endpoint, and the linked issue describes a concrete Feishu setup where that renders as duplicated text.

## Solution Assessment

No. Suffix-only deltas are a narrow fit for prefix snapshots, but this patch still sends whole replacement text for non-prefix rewrites and lacks live Feishu proof.
