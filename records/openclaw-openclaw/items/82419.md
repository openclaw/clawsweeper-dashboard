## Summary

This PR should stay open: it targets a real Feishu CardKit streaming bug, but the current patch is not ready because it relies on suffix-only payload semantics without live Feishu proof and still has a rewrite/closeout edge case to resolve.

## What This Changes

The PR changes Feishu CardKit streaming updates to send suffix-only text deltas and updates focused streaming-card tests.

## Best Possible Solution

Keep the fix in the Feishu plugin, but use a CardKit-compatible path that preserves full visible content for normal streaming and handles shrink/rewrite closeouts, then back it with focused tests and redacted live Feishu proof.

## Reproduction Assessment

No high-confidence live Feishu reproduction was established here. Source inspection confirms current main sends accumulated text to the reported CardKit update endpoint, but the visible client append effect still needs a real Feishu run.

## Solution Assessment

No. Suffix-only deltas are a plausible workaround for the reported client behavior, but this patch lacks live proof and does not safely handle replacement-style or non-prefix closeout updates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Handle non-prefix CardKit closeout updates:** `extensions/feishu/src/streaming-card.ts:456`
  - body: The final closeout now calls `resolveStreamingCardAppendContent(...)`, but that helper falls back to the entire final text whenever the final content is not a strict extension of the displayed text. Under the append-style endpoint behavior this PR is trying to support, rewrite or status-removal closeouts can append a replacement body after stale content instead of replacing it. Please route non-prefix closeouts through a replacement-capable CardKit path or preserve monotonic append semantics before closing.
  - confidence: 0.84
