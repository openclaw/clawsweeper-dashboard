## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Handle non-prefix CardKit closeout updates:** `extensions/feishu/src/streaming-card.ts:456`
  - body: The final closeout now passes `resolveStreamingCardAppendContent(...)`, but that helper returns the entire final text whenever it no longer starts with the displayed text. That happens after the dispatcher clears transient status text before `close()`, so an append-style CardKit endpoint will append the final answer after the stale status instead of replacing/removing it. Please route non-prefix rewrites through a replacement-capable path or avoid append payloads for shrink/rewrite cases before closing.
  - confidence: 0.84
