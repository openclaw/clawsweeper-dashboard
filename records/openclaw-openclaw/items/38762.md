## Summary

Keep open: current main and the latest v2026.6.6 release still drop top-level browser act fields when a nested request is present, and the open linked pull request https://github.com/openclaw/openclaw/pull/81076 is the active fix path.

## What This Changes

The issue asks browser action=act to handle ref consistently when it is provided at the top level beside a nested request, or to return a shape-specific validation error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81076, or an equivalent narrow browser-plugin parser fix with regression tests and acceptable real behavior proof.

## Reproduction Assessment

Yes, at source level. A browser act call with top-level ref and request:{kind:"click"} reaches readActRequestParam, which returns the nested request before copying ref, and the /act normalizer then rejects click/type bodies without ref or selector.

## Solution Assessment

Yes for the fix direction. The narrow maintainable solution is to backfill only allowlisted top-level act fields into a nested request when missing, while keeping nested fields authoritative.
