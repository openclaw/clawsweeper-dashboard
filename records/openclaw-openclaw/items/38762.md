## Summary

Keep open. Current main still drops a top-level browser act ref when a nested request is present, and an open focused PR already references this issue with closing syntax, so the issue should remain open until that PR is landed or rejected.

## What This Changes

The issue requests that browser act accept top-level ref alongside a nested request, or return a clearer parameter-shape error.

## Best Possible Solution

Land or close the open focused browser parser PR after review, preserving the /act service contract while backfilling only allowlisted top-level act fields into nested requests when missing.

## Reproduction Assessment

Yes. Source inspection on current main shows action="act" with top-level ref and request={ kind: "click" } returns the nested request unchanged, then the /act normalizer requires ref or selector inside that request body.

## Solution Assessment

Yes. The narrowest maintainable fix is to normalize in extensions/browser/src/browser-tool.ts by backfilling supported top-level act fields only when the nested request lacks them, with regression coverage for backfill and nested precedence.
