## Summary

Keep open. Current main still returns an object `request` before top-level `ref`, `targetId`, or `timeoutMs` can be merged, so the mixed call shape from the report still drops the top-level `ref`; the linked compatibility PR #38790 is closed unmerged.

## What This Changes

The issue requests that browser `action="act"` handle top-level `ref` consistently with nested `request.ref`, or at least return a request-shape error instead of misleading service guidance.

## Best Possible Solution

Add a narrow browser tool entry-point normalizer that fills only missing nested act fields from supported top-level compatibility params while preserving the `/act` service contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `browser` with `action: "act"`, `request: { kind: "click" }`, and top-level `ref`; the parser forwards only the request object, and `/act` validation then sees no `ref` or `selector`.

## Solution Assessment

Yes. The narrowest maintainable fix is at `extensions/browser/src/browser-tool.ts`, before transport, because that is where the mixed tool-call shape loses the top-level fields.
