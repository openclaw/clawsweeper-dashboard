## Summary

Keep open: current main still drops top-level browser `act` fields when `request` is already an object, and the open linked fix PR has closing syntax but has not merged.

## What This Changes

The issue asks for browser `action="act"` to handle a top-level `ref` beside a nested `request`, or to return an accurate request-shape error instead of a browser-service error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81076, or an equivalent parser-layer backfill with regression tests, then close this issue as implemented.

## Reproduction Assessment

Yes, by source inspection. With `action: "act"`, top-level `ref`, and `request: { kind: "click" }`, current main returns the nested request unchanged and the `/act` normalizer receives no `ref` or `selector`.

## Solution Assessment

Yes for the fix direction. The narrowest maintainable solution is to backfill allowlisted top-level act fields into nested `request` while preserving nested-field precedence; the open linked PR implements that shape but still needs normal PR completion.
