## Summary

Current main and the latest release still have the reported browser `act` parser behavior, and the issue already has an open linked fix PR, so this should stay open until that PR lands or is replaced.

## What This Changes

Accept browser `act` calls where `ref` and related act fields are provided at the top level beside a nested `request`, or return a clearer parameter-shape error.

## Best Possible Solution

Land the open parser-layer fix PR, or replace it with the same narrow browser plugin change plus regression coverage if maintainers choose not to use that branch.

## Reproduction Assessment

Yes at source level. Current main returns nested `request` before copying top-level `ref`, then `/act` rejects click/type requests that no longer contain `ref` or `selector`.

## Solution Assessment

Yes. Backfilling only allowlisted top-level act fields in the browser tool parser is narrower than changing the `/act` service contract, and it preserves nested-field precedence.
