## Summary

Keep open. Current main still ignores top-level `ref`, `targetId`, and `timeoutMs` when `browser action="act"` receives an object `request`; that object is forwarded unchanged to `/act`, where click validation still requires `ref` or `selector` inside the forwarded body. The browser-service error wrapping has improved, but the parameter-position inconsistency remains, and related PR #38790 is closed unmerged.

## What This Changes

The issue requests consistent browser `act` handling so top-level `ref`/`targetId`/`timeoutMs` work when an object `request` is also supplied, or at least produce a clearer parameter-shape error.

## Best Possible Solution

Add a narrow compatibility normalizer at the browser tool entry point that backfills missing nested `request.ref`, `request.targetId`, and finite numeric `request.timeoutMs` from top-level act params while preserving nested request precedence and leaving the browser route/service contract unchanged.
