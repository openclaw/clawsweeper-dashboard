## Summary

Keep open: current main and the latest release still have the reported browser act parser gap, and https://github.com/openclaw/openclaw/pull/81076 is the open closing PR that owns the fix path.

## What This Changes

The issue requests consistent browser act handling when `ref` is supplied at the top level beside a nested `request`, or at least a clearer validation error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81076, or an equivalent browser plugin parser fix, then close this issue after the fix merges.

## Reproduction Assessment

Yes, by source inspection. On current main, `action: "act"` with top-level `ref` and `request: { kind: "click" }` forwards only the nested request, so downstream act normalization sees no `ref` or `selector`.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: backfill only supported top-level act fields into the nested request without changing the `/act` service contract or overriding nested values.
