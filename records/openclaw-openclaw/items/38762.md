## Summary

Keep open: current main now preserves browser service application errors instead of mislabeling them as reachability failures, but it still drops an advertised top-level `ref` whenever a nested `request` object is present. The explicitly closing PR remains open and should stay the canonical implementation path.

## What This Changes

Make browser `act` consistently accept the advertised top-level `ref` with a nested request while preserving accurate parameter-error reporting.

## Best Possible Solution

Land or revise the linked PR so allowlisted top-level act fields fill only missing nested fields, nested values remain authoritative, and focused regression tests cover flattened, nested, and mixed payloads.

## Reproduction Assessment

Yes. On current main, the reported mixed payload follows the advertised schema but takes the early nested-request return, losing top-level `ref` before downstream act validation.

## Solution Assessment

Yes. Normalizing both advertised shapes at the browser tool boundary is narrower and more maintainable than changing the browser service contract; nested request values should retain precedence.
