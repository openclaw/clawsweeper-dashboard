## Summary

Keep open: current main and latest release still drop top-level browser act fields when a nested request object is present, and the open linked pull request https://github.com/openclaw/openclaw/pull/81076 is the active fix path.

## What This Changes

Make browser action=act handle a top-level ref beside a nested request consistently, or return a request-shape error instead of a browser-service error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81076, or an equivalent narrow browser-plugin parser fix with regression tests and acceptable real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main returns the nested request before copying top-level ref, and the /act normalizer then rejects click/type bodies without ref or selector.

## Solution Assessment

Yes for the fix direction. The narrow maintainable solution is to backfill only allowlisted top-level act fields into a nested request when missing, while keeping nested fields authoritative.
