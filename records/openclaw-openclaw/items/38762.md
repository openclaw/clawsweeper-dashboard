## Summary

Keep open. Current main still drops top-level browser act fields when a nested request object is present, and https://github.com/openclaw/openclaw/pull/81076 is an open focused PR with a closing reference to this issue, so the issue should remain open until that PR lands or is rejected.

## What This Changes

The issue asks for browser act to handle a top-level ref beside a nested request consistently, or at least report the parameter-shape error accurately.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81076; if landed, close this issue as implemented by that PR while preserving the /act service contract and only backfilling allowlisted top-level act fields.

## Reproduction Assessment

Yes. Source inspection on current main shows action="act" with top-level ref and request={ kind: "click" } returns only the nested request, then the /act normalizer throws because that body lacks ref or selector.

## Solution Assessment

Yes. The narrow maintainable fix is the linked parser backfill: copy supported top-level act fields into the nested request only when missing, with tests for backfill and nested-field precedence.
