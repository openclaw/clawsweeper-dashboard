## Label Justifications

- `P1`: The bug breaks a real browser-agent workflow for model-generated tool calls where OpenAI-family payloads place ref at the advertised top level.
- `impact:auth-provider`: The failure is tied to model/provider behavior because OpenAI-style tool payloads fail while nested-ref payloads from other model families can continue working.

## Summary

Keep open: current main still drops top-level browser act fields when a nested request object is present, and the linked open pull request remains the focused fix path.

## What This Changes

The issue asks for browser act calls to handle a top-level ref beside a nested request consistently, or at least return a request-shape error instead of a browser-service reachability error.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81076 with the same narrow parser backfill and regression tests while preserving nested request precedence and the /act service contract.

## Reproduction Assessment

Yes, at source level. Current main returns the nested request before copying top-level ref, then the act normalizer rejects click/type bodies without ref or selector.

## Solution Assessment

Yes. Backfilling existing allowlisted top-level act fields into request only when missing is the narrowest maintainable fix and keeps nested fields authoritative.
