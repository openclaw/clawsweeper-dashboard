## Summary

Keep open. Current main still drops top-level `ref` whenever `action: "act"` also includes an object `request`, and the two related compatibility PRs that targeted this exact mixed shape are closed unmerged.

## What This Changes

Accept or clearly diagnose `browser action="act"` calls that pass `ref` at the top level while `request` contains the act kind.

## Best Possible Solution

Add a narrow browser tool entry-point normalizer that backfills only missing nested act fields from supported top-level compatibility params, without changing the `/act` service contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `browser` with `action: "act"`, `request: { kind: "click" }`, and top-level `ref`; the parser forwards only the request object and route validation then sees no ref or selector.

## Solution Assessment

Yes. The narrowest maintainable fix is in `extensions/browser/src/browser-tool.ts` before transport, filling missing nested fields from legacy top-level params while preserving nested request precedence.

## Repair Work Prompt

Fix the browser act mixed-parameter compatibility bug reported in https://github.com/openclaw/openclaw/issues/38762. Current main accepts fully nested `request={ kind: "click", ref: "..." }` and fully flattened `kind/ref/...` params, but drops top-level `ref` when `request` is an object such as `{ kind: "click" }`. Implement the fix in `extensions/browser/src/browser-tool.ts` by normalizing the tool entry-point request before transport: when `action: "act"` has an object `request`, copy only missing supported top-level shared act fields such as `ref`, `targetId`, and finite numeric `timeoutMs` into a shallow request copy. Preserve nested request precedence, do not change the `/act` route contract, do not add config, and do not broaden behavior for non-act actions. Add focused regression tests in `extensions/browser/src/browser-tool.test.ts` for top-level `ref` backfill, nested `ref` precedence, and `targetId`/`timeoutMs` backfill. Add a single-line user-facing `CHANGELOG.md` fix entry crediting the human reporter/related contributor if appropriate, avoiding bot or maintainer-only credit. Use closed unmerged PRs #38790 and #23075 only as implementation context; do not resurrect their stale paths blindly because current files now live under `extensions/browser`.
