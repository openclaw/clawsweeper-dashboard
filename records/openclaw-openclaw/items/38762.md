## Summary

Keep open. Current main and latest release still drop top-level browser `act` fields when a nested `request` object is present, and the open closing PR https://github.com/openclaw/openclaw/pull/81076 is the right focused fix path.

## What This Changes

Browser `act` should handle a top-level `ref` alongside a nested `request` consistently or return a request-shape error instead of a browser-service reachability message.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81076 with the same narrow parser backfill and regression tests, preserving nested request precedence and the `/act` service contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `action: "act"` with top-level `ref` and `request: { kind: "click" }` returns only the nested request, then `/act` rejects it because `ref` or `selector` is absent from that body.

## Solution Assessment

Yes. Backfilling only existing allowlisted top-level act fields into the nested request when missing is the narrowest maintainable fix and keeps nested request fields authoritative.
