## Summary

Keep open. Current main still drops top-level browser act fields when a nested request object is present, and an open focused PR already carries the implementation path with a closing reference to this issue.

## What This Changes

The issue asks browser act to handle a top-level `ref` consistently when `request` is present, or return a clearer parameter-shape error.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81076; if landed, close this issue as implemented while preserving nested-request precedence and the `/act` service contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `action: "act"` with top-level `ref` and `request: { kind: "click" }` returns only the nested request, then the `/act` normalizer throws because that body lacks `ref` or `selector`.

## Solution Assessment

Yes. The narrow maintainable fix is to backfill only allowlisted top-level act fields into the nested request when missing, with regression tests for backfill and nested-field precedence.
