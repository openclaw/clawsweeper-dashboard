## Summary

Keep open. Current main still drops top-level browser `act` fields whenever a nested `request` object is present, and https://github.com/openclaw/openclaw/pull/81076 is an open closing PR for the same bug.

## What This Changes

The issue requests consistent browser `act` handling when `ref` is passed at the top level beside `request`, or a clearer parameter-shape error.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81076; if it lands, close this issue as implemented while preserving nested-request precedence and the `/act` service contract.

## Reproduction Assessment

Yes. Source inspection shows `action: "act"` with top-level `ref` and `request: { kind: "click" }` returns only the nested request, then the `/act` normalizer throws because that body lacks `ref` or `selector`.

## Solution Assessment

Yes. Backfilling only allowlisted top-level act fields into the nested request when missing is the narrowest maintainable fix and keeps nested request fields authoritative.
