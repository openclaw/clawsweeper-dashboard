## Summary

Keep open. Current main still loses top-level browser act fields when a nested request object is present, and https://github.com/openclaw/openclaw/pull/81076 is an open focused PR with a closing reference to this issue.

## What This Changes

The issue asks for browser act calls to handle ref consistently when it is supplied at the top level beside a nested request object, or to return a clearer request-shape error.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/81076; if landed, close this issue as implemented while preserving the `/act` service contract and nested-request precedence.

## Reproduction Assessment

Yes. Source inspection on current main shows `action: "act"` with top-level `ref` and `request: { kind: "click" }` returns only the nested request, then the `/act` normalizer throws because that body lacks `ref` or `selector`.

## Solution Assessment

Yes. The narrow maintainable fix is to backfill only allowlisted top-level act fields into the nested request when missing, with tests for backfill and nested-field precedence.
