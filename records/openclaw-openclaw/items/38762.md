## Summary

Keep open. Current main and the latest release still drop top-level browser `act` fields when a nested `request` object is present, and https://github.com/openclaw/openclaw/pull/81076 is an open closing PR for the same bug.

## What This Changes

Issue reports that browser `act` accepts `ref` inside `request` but drops top-level `ref` when a nested request is present, producing a misleading browser-service error.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81076 with the same narrow parser backfill and regression tests, preserving nested-request precedence and the `/act` service contract.

## Reproduction Assessment

Yes. Source inspection shows `action: "act"` with top-level `ref` and `request: { kind: "click" }` returns only the nested request, then `/act` rejects it because `ref`/`selector` is absent from that body.

## Solution Assessment

Yes. Backfilling only existing allowlisted top-level act fields into the nested request when missing is the narrowest maintainable fix and keeps nested request fields authoritative.
