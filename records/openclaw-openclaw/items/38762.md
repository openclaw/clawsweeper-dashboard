## Summary

Keep open: current `main` still drops a top-level browser `act` `ref` when `request` is present, and the linked closing fix PR remains open and unmerged, so this issue should close only after that PR or a replacement lands.

## What This Changes

The issue reports that browser `act` accepts `ref` inside `request` but drops a top-level `ref`, producing a misleading browser-service error instead of acting or reporting the parameter-shape problem.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81076 if it remains the preferred parser-level fix; close this issue only after that PR merges or a replacement fix lands.

## Reproduction Assessment

Yes from source inspection: send `action: "act"` with a nested `request` object and a top-level `ref`, and `readActRequestParam` returns only the nested object before the backfill loop can run. I did not run a live Chrome Relay repro in this read-only review.

## Solution Assessment

Likely yes: the narrow maintainable fix is to normalize missing allowlisted top-level act fields into the nested request at the browser tool entry point while preserving nested values as authoritative.
