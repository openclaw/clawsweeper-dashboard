## Summary

Keep open: current `main` and the latest release still drop top-level browser `act` fields when `request` is present, and the open linked fix PR has not merged yet.

## What This Changes

The issue asks for browser `act` to accept top-level `ref` consistently with nested `request.ref`, or at least return an accurate request-shape error instead of a browser-service reachability error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81076, or an equivalent browser-plugin parser backfill with regression tests, then close this issue as implemented.

## Reproduction Assessment

Yes. A source-level reproduction is clear: `action: "act"` with top-level `ref` and nested `request: { kind: "click" }` returns the nested request before copying top-level fields, then `/act` rejects the stripped body.

## Solution Assessment

Yes for the code direction. Backfilling only allowlisted top-level act fields in the browser tool parser is narrower than changing the `/act` service contract, and it can preserve nested-field precedence.
