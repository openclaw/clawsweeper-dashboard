## Summary

Keep open: current main still drops top-level browser `act` fields whenever a nested `request` object is present, and the open closing PR is the focused fix path.

## What This Changes

The issue requests consistent browser `act` handling when `ref` is supplied at the top level beside a nested `request`, or a clearer request-shape error.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81076 with the same narrow parser backfill and regression tests, preserving nested request precedence and the browser `/act` service contract.

## Reproduction Assessment

Yes, at source level. Current main returns the nested `request` before copying top-level `ref`, then the act normalizer rejects click/type bodies without `ref` or `selector`.

## Solution Assessment

Yes. Backfilling allowlisted top-level act fields into `request` only when missing is the narrowest maintainable fix and keeps nested fields authoritative.
