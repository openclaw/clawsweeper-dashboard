## What This Changes

The issue asks doctor repair to warn or guard when the config file it is about to write differs from the running gateway or managed service config path.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/63268 or a replacement that compares doctor’s actual resolver-backed write path with a verified managed-service or running-gateway config path, emits non-destructive guidance, and adds focused regression coverage.

Yes, source-backed. The reporter gives a concrete root-shell/systemd `HOME` mismatch flow, and current source shows doctor reads and writes through process-env config resolution without a doctor-time service config-path comparison.

## Solution Assessment
