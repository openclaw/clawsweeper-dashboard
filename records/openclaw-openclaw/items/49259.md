## Summary

Keep open. Current main still has explicit selected-session deletion and generic session-store maintenance, but not the requested Dashboard prune workflow with configurable age, eligibility preview, Telegram/Discord reachability checks, or auto-prune policy; the related Discord-specific issue is still open and only covers part of the broader request.

Add a Dashboard Sessions prune workflow for stale sessions older than a configurable age whose Telegram topic/channel or Discord channel no longer exists, with preview before deletion and optional auto-prune.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates Telegram and Discord reachability to plugin-owned capability seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default.

## Solution Assessment
