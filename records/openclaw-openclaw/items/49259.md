## Summary

Keep open. Current main still lacks the requested Dashboard Sessions stale-orphan prune workflow; the existing selected-session delete and generic session maintenance cleanup do not provide a 48-hour prune input, orphan candidate preview, or Telegram/Discord reachability checks.

## What This Changes

Add a Dashboard Sessions workflow to preview and prune stale Telegram/Discord orphan sessions older than a configurable age, defaulting to 48 hours, with optional auto-prune.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that returns explicit stale-orphan candidates, delegates Telegram/Discord reachability to plugin-owned seams, revalidates candidates on apply, and wires the Sessions UI to show the default 48-hour age field, preview, and Prune action.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the requested workflow is absent from the Sessions UI, controller, Gateway protocol, and cleanup implementation; no live Telegram or Discord credentials are needed to verify that gap.

## Solution Assessment

Yes, keeping this open is the best current outcome. The maintainable direction is a new preview/apply contract with plugin-owned reachability checks, not the existing selected delete or generic session maintenance cleanup.
