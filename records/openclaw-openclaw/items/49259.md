## Summary

Keep open. Current main still lacks the requested Dashboard Sessions prune workflow: existing UI supports selected-row deletion and generic filters, while existing cleanup code is local age/count/missing-transcript maintenance rather than Telegram/Discord orphan reachability with a 48-hour default preview/apply flow. Related #50310 is narrower Discord cleanup context, not a replacement for this broader Dashboard feature.

## What This Changes

Add a Dashboard Sessions workflow to preview and prune stale orphaned Telegram/Discord sessions older than a configurable age, defaulting to 48 hours.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that returns explicit stale-orphan candidate reasons, delegates Telegram/Discord reachability to channel/plugin-owned helpers behind a generic seam, revalidates during apply, and wires the Dashboard Sessions screen to show the 48-hour default age field, candidate preview, and explicit Prune action.

## Reproduction Assessment

Not applicable as a feature request; the requested workflow is absent by source inspection. The gap is verifiable through the Sessions UI, controller, protocol schema, gateway listing, and cleanup maintenance code without live Telegram or Discord credentials.

## Solution Assessment

Yes as a product direction, but the maintainable implementation should be an additive preview/apply contract with plugin reachability and apply-time revalidation. Reusing selected-row delete or generic CLI age cleanup would not solve the requested orphan detection safely.
