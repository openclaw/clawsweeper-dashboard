## Summary

Keep open. Current main has generic CLI/RPC session cleanup and explicit selected-session deletion, but not the requested Dashboard prune workflow with configurable age, eligibility preview, and Telegram/Discord channel or topic reachability checks.

## What This Changes

The issue requests a Dashboard Sessions prune control that previews and deletes old orphaned Telegram/Discord session rows with a configurable age and optional auto-prune behavior.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates Telegram and Discord liveness checks to plugin-owned seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default threshold.

Not applicable as a feature request. Source inspection on current main shows the requested Dashboard prune UI, prune protocol contract, and Telegram/Discord orphan reachability flow are absent.

## Solution Assessment

Yes, keeping this open for maintainer direction is the best current outcome. Existing selected delete and generic cleanup are not enough because orphan detection needs a new preview/apply contract and plugin-owned liveness checks.
