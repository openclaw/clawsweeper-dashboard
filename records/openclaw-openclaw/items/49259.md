## Summary

Keep open. Current main still lacks the requested Dashboard Sessions stale-orphan prune workflow; the existing selected-session delete, generic session cleanup, and related Discord-only cleanup threads do not cover the requested configurable preview/apply UI or Telegram reachability scope.

## What This Changes

Add a Dashboard Sessions workflow to preview and delete stale orphaned Telegram/Discord sessions older than a configurable age, with 48 hours as the default and optional auto-prune per session.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates channel/topic reachability to plugin-owned seams, revalidates candidates before deletion, and wires the Sessions UI to show the default 48-hour age field plus eligible-session preview.

## Solution Assessment
