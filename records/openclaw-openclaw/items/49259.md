## What This Changes

The issue requests a Dashboard workflow to preview and prune stale orphaned Telegram or Discord sessions older than a configurable age, with a 48-hour default and optional automatic pruning.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates Telegram and Discord liveness checks to plugin-owned seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default threshold.

## Solution Assessment

Unclear as an exact product design. A preview/apply Dashboard workflow is a plausible direction, but maintainers still need to decide destructive cleanup semantics, plugin liveness ownership, and whether automatic pruning should exist.
