## Summary

Keep open. Current main still lacks the requested Dashboard Sessions stale-orphan prune workflow; existing manual selected-session deletion and generic session-store cleanup do not provide a 48-hour prune input, orphan candidate preview, or Telegram/Discord reachability checks.

## What This Changes

Add a Dashboard Sessions workflow to preview and delete stale orphaned Telegram/Discord sessions older than a configurable age, defaulting to 48 hours.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that returns explicit stale-orphan candidates, delegates Telegram/Discord reachability to plugin-owned seams, revalidates on apply, and wires the Sessions UI to show the default 48-hour age field, candidate preview, and Prune action.

## Reproduction Assessment

Not applicable as a feature request; source inspection shows the requested workflow is absent from the Sessions UI, controller, Gateway protocol, and cleanup implementation. No live Telegram or Discord credentials are needed to verify that gap.

## Solution Assessment

Yes, keeping this open is the right outcome. The narrow maintainable direction is a new preview/apply contract with plugin-owned reachability checks, not the existing manual selected delete or generic session maintenance cleanup.
