## Summary

Keep open: current main and v2026.6.10 still provide selected-session deletion plus generic session maintenance, but not a Dashboard orphan-prune flow that previews old Telegram/Discord-backed rows before deleting them.

## What This Changes

The issue requests a Dashboard Sessions prune workflow that previews and deletes old Telegram/Discord-backed sessions whose backing topic, thread, or channel no longer exists, with a configurable age defaulting to 48 hours.

## Best Possible Solution

Define a maintainer-approved Dashboard preview/apply prune flow backed by channel/plugin-owned reachability checks, revalidation before delete, and documented transcript-retention/default-age policy.
