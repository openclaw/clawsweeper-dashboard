## Summary

Keep open. Current main still has manual selected-session deletion and generic session-store cleanup, but not the requested Dashboard prune workflow with configurable age, eligibility preview, Telegram/Discord reachability checks, or auto-prune policy.

## What This Changes

Add a Dashboard Sessions workflow to preview and delete stale orphaned Telegram topic or Discord channel sessions older than a configurable age, defaulting to 48 hours.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates Telegram and Discord reachability to plugin-owned seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the requested Dashboard prune UI, protocol contract, and Telegram/Discord orphan reachability flow are absent.

## Solution Assessment

Yes, keeping this open is the best current outcome. Existing selected delete and generic cleanup are not enough because orphan detection needs a new preview/apply contract and plugin-owned liveness checks.
