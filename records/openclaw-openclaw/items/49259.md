## Summary

Keep open: current main and v2026.6.6 support manual multi-session deletion and generic CLI/session cleanup, but not a Dashboard prune flow that previews older Telegram/Discord orphan targets before deletion. The narrower Discord cleanup issue and open missing-transcript cleanup PRs do not supersede this broader UI/product request.

## What This Changes

Request a Dashboard Sessions prune workflow for stale Telegram/Discord orphan sessions, with configurable age, preview, delete action, and optional auto-prune.

## Best Possible Solution

Design an operator-only Dashboard preview/apply cleanup that uses plugin-owned Telegram/Discord reachability checks, revalidates candidates before deleting, archives transcripts consistently, and keeps automatic pruning opt-in.

## Reproduction Assessment

Not applicable as a bug repro: this is a missing feature request. Source inspection confirms the current UI and Gateway contracts lack orphan reachability preview/apply behavior.

## Solution Assessment

Unclear until maintainers choose the destructive cleanup policy. The requested Dashboard flow is plausible, but the maintainable shape should use plugin-owned liveness checks and a revalidated preview rather than generic core age pruning.
