## Summary

Keep open. Current main still supports generic session maintenance and explicit selected-session deletion, but it does not provide the requested Dashboard prune workflow with configurable age, eligibility preview, and Telegram/Discord reachability checks; the linked Discord-only issue remains narrower and open.

## What This Changes

Add a Dashboard Sessions prune workflow for stale Telegram/Discord orphan sessions, with a default/custom age threshold, eligibility preview, and optional auto-prune behavior.

## Best Possible Solution

Add an additive Gateway-backed preview/apply prune workflow that delegates Telegram and Discord liveness checks to plugin-owned seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default threshold.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the requested Dashboard prune UI, prune protocol contract, and Telegram/Discord orphan reachability flow are absent.

## Solution Assessment

Unclear until maintainers choose the destructive cleanup policy. The safest maintainable direction is an additive preview/apply contract with plugin-owned liveness checks rather than extending generic age cleanup blindly.
