## Summary

Keep open. Current main has CLI/session-store maintenance and explicit selected-session deletion, but not the requested Dashboard prune workflow with configurable age, eligibility preview, and Telegram/Discord orphan reachability checks.

## What This Changes

Request to add a Dashboard Sessions prune workflow for stale orphaned Telegram/Discord sessions, including a 48-hour default age threshold, eligibility preview, manual prune button, and optional auto-prune behavior.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates Telegram and Discord reachability checks to plugin-owned seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default threshold.

## Solution Assessment

Yes, keeping this open is the best current outcome. Existing selected delete and generic cleanup are not enough because orphan detection needs a new preview/apply contract and plugin-owned liveness checks.
