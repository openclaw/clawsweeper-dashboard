## Summary

Keep open. Current main still lacks the requested Dashboard Sessions prune workflow; selected-row deletion, CLI/gateway cleanup, and the open Discord-only cleanup thread do not cover configurable age-based preview/apply UI, Telegram topic reachability, or auto-prune policy.

## What This Changes

Add a Dashboard Sessions workflow to preview and delete stale orphaned Telegram topic and Discord channel/thread sessions older than a configurable age, defaulting to 48 hours, with optional auto-prune.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates Telegram and Discord reachability to plugin-owned seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default.
