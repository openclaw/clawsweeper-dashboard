## Summary

Keep open. Current main has explicit selected-session deletion and generic session-store cleanup, but it still does not provide the requested Dashboard prune workflow with a configurable age, eligibility preview, and Telegram/Discord orphan reachability checks; the related https://github.com/openclaw/openclaw/issues/50310 remains a narrower open Discord-thread cleanup item, not a superseding fix.

## What This Changes

The issue requests a Dashboard Sessions control to preview and prune stale orphaned Telegram/Discord sessions using a configurable age threshold, with 48 hours as the default.

## Best Possible Solution

Add an additive gateway-backed prune preview/apply workflow that delegates Telegram and Discord reachability checks to plugin-owned seams, revalidates candidates before deletion, and exposes explicit Dashboard controls with a 48-hour default threshold.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main shows the requested Dashboard prune UI, gateway prune contract, and Telegram/Discord orphan reachability flow are absent.

## Solution Assessment

Yes, keeping this open is the best current outcome. Existing selected delete and generic cleanup are not enough because orphan detection needs a new preview/apply contract plus plugin-owned liveness checks.
