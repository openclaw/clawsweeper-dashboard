## Summary

Keep open: current main still lacks accepted WhatsApp group-level debounce config and runtime per-group resolution; the related composing-debounce PR is adjacent but does not supersede this request.

## What This Changes

Request adding per-group `debounceMs` overrides for WhatsApp groups, with possible cross-channel parity later.

## Best Possible Solution

Implement the WhatsApp slice first: accept root/account `groups.<jid>.debounceMs`, resolve exact group before wildcard per inbound message, preserve sender/conversation batching, and document/test media and command behavior; leave broad cross-channel parity to follow-up.

## Reproduction Assessment

Yes. Source inspection is enough: the strict WhatsApp group schema rejects `debounceMs`, the monitor passes only one scalar debounce value into the listener, and media messages currently bypass the debounce predicate.

## Solution Assessment

Yes for a scoped WhatsApp repair. The narrow maintainable path is to extend the bundled WhatsApp config/runtime and tests using the existing debouncer hook, not to redesign every channel in the same change.
