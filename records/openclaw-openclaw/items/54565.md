## Summary

Keep open. Current main and v2026.6.10 still lack the requested WhatsApp group-level `debounceMs`; the gap is source-reproducible, but adding the key is a compatibility-sensitive config/product decision rather than conservative cleanup.

## What This Changes

Requested support for `debounceMs` inside per-group channel configuration, especially WhatsApp group config, so one noisy group can batch rapid media/text messages without changing the whole account or channel.

## Best Possible Solution

Approve a narrow WhatsApp-owned group-level debounce override only if maintainers want the new config contract, then wire it through the existing debouncer seam with focused docs and regression tests.

## Reproduction Assessment

Yes, at source level. Current schema and types reject `channels.whatsapp.groups.<jid>.debounceMs`, and the runtime resolves debounce only from account/channel/global scopes.

## Solution Assessment

Unclear until maintainers choose the config contract. The narrowest maintainable path appears to be a WhatsApp-owned per-group override that reuses `resolveDebounceMs`, not a broad cross-channel rollout in one change.
