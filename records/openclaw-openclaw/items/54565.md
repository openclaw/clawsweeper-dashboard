## Summary

Keep open. Current main and v2026.6.10 still lack the requested WhatsApp group-level `debounceMs`; the gap is source-reproducible, but adding this public config key needs maintainer direction on scope and precedence rather than conservative cleanup closure.

## What This Changes

The issue requests per-group WhatsApp `debounceMs` overrides, with possible future extension to other group-capable channels.

## Best Possible Solution

Approve a narrow WhatsApp-owned group-level debounce override only if maintainers want the new config contract, then wire it through the existing debouncer seam with focused schema, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current schema and types reject `channels.whatsapp.groups.<jid>.debounceMs`, and the runtime resolves WhatsApp debounce only from account, default-account, channel, and global inbound scopes.

## Solution Assessment

Unclear until maintainers choose the config contract. The narrowest maintainable path appears to be a WhatsApp-owned per-group override that reuses the existing debouncer seam, not a broad cross-channel rollout in one change.
