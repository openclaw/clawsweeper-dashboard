## Summary

Keep open: current main and the latest release still do not accept or resolve WhatsApp group-level `debounceMs`, and the related composing-aware debounce PR does not cover this requested per-group override.

## What This Changes

Request to allow `debounceMs` inside per-group WhatsApp config, with possible later parity for other group-capable channels.

## Best Possible Solution

Ship a WhatsApp-owned group debounce override that honors exact group before wildcard, preserves account/root precedence, keeps all batched media/context intact, and tracks Telegram/Discord parity separately.

## Reproduction Assessment

Yes, source-level. Current main's strict WhatsApp group schema omits `debounceMs`, runtime resolves one scalar debounce window before listener creation, and media messages bypass the debounce predicate.

## Solution Assessment

Yes for a scoped WhatsApp feature; no for doing broad cross-channel parity in the same first pass. The maintainable path is WhatsApp config/runtime/tests using the existing debouncer hook while preserving media attachments.
