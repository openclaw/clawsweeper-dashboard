## Label Justifications

- `P2`: This is a concrete WhatsApp batching improvement with limited blast radius, but it is a feature/config request rather than an urgent regression.
- `impact:session-state`: Splitting rapid group messages into separate agent turns can leave the agent transcript and context incomplete for the user's intended instruction.

## Summary

Keep open: current main and the latest release still lack WhatsApp group-level `debounceMs` config and per-message group debounce resolution. The adjacent composing-aware debounce PR is a different timing feature and does not satisfy this requested per-group override.

## What This Changes

Allow selected WhatsApp group config entries, and eventually similar group-aware channels, to override inbound `debounceMs` so rapid multi-message sends can be batched for that group only.

## Best Possible Solution

Ship a WhatsApp-owned group debounce override that honors exact group before wildcard and account/root precedence, updates schema/docs/tests, and tracks Telegram/Discord parity separately.

## Reproduction Assessment

Yes, source-level. Current main's strict WhatsApp group schema omits `debounceMs`, runtime resolves one scalar debounce window before listener creation, and media/reply/location messages bypass debouncing.

## Solution Assessment

Yes for a scoped WhatsApp feature, unclear for the broader cross-channel request. The maintainable path is WhatsApp config/runtime/tests using the existing debouncer hook, with media behavior and later channel parity decided separately.
