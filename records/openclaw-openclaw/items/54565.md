## Summary

Keep open: current main and v2026.5.19 still expose WhatsApp `debounceMs` only at channel/account scope, while per-group config is strict and omits the requested key. The adjacent timestamp, media, and composing debounce items are related but do not supersede this per-group override request.

## What This Changes

Request per-group WhatsApp `debounceMs` configuration so one group can batch rapid image/image/text message bursts without increasing debounce for every group.

## Best Possible Solution

Ship a WhatsApp-owned group debounce override that honors exact group before wildcard, preserves account/root fallback behavior, keeps attachment-bearing batches intact, and leaves broader channel parity to separate follow-up design.

## Reproduction Assessment

Yes, source-level. Current main's strict WhatsApp group schema omits `debounceMs`, runtime resolves one scalar channel/account debounce before listener creation, and media messages bypass the existing debounce predicate.

## Solution Assessment

Yes for a scoped WhatsApp feature; no for broad cross-channel parity in the same first pass. The maintainable path is WhatsApp config/runtime/tests using the existing debouncer hook while preserving media attachments or coordinating explicitly with the media-coalescing issue.
