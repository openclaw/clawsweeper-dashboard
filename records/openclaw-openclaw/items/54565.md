## Summary

Keep open: current main and v2026.5.18 still lack WhatsApp group-level debounceMs config and per-message group debounce resolution. The related composing-aware debounce PR is adjacent timing work, not this requested per-group override.

## What This Changes

The issue requests a per-group debounceMs override for WhatsApp group config, with possible later parity for other channels.

## Best Possible Solution

Ship a WhatsApp-owned group debounce override first, with exact-group before wildcard resolution, account/root precedence, schema/docs/tests, and separate tracking for Telegram/Discord parity.

## Reproduction Assessment

Yes, source-level. Current main's strict WhatsApp group schema omits debounceMs, runtime resolves one scalar debounce window before listener creation, and media/reply/location messages bypass debouncing.

## Solution Assessment

Yes for a scoped WhatsApp feature, unclear for the broader cross-channel request. The maintainable path is WhatsApp config/runtime/tests using the existing debouncer hook, with media behavior and later channel parity decided separately.
