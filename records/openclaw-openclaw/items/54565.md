## Summary

Keep open: current main still lacks WhatsApp group-level debounce config and per-message group debounce resolution. The related composing-aware debounce PR is adjacent but does not supersede this request because it adds a different timing signal and does not add `channels.whatsapp.groups.<jid>.debounceMs`.

## What This Changes

The issue requests `debounceMs` overrides inside per-group channel config, starting with WhatsApp groups and potentially extending to other group-capable channels.

## Best Possible Solution

Ship a scoped WhatsApp-owned group debounce override that honors root/account group config, resolves exact group before wildcard, and documents media/control-command edge cases; track cross-channel parity separately.

## Reproduction Assessment

Yes. Source inspection shows current main rejects group-level `debounceMs`, resolves only one scalar WhatsApp debounce window per listener, and flushes media immediately, so the requested group-specific batching path is absent.

## Solution Assessment

Yes for a scoped WhatsApp repair, not for a cross-channel redesign in one change. The narrow maintainable path is to extend WhatsApp config/runtime/tests using the existing `resolveDebounceMs` hook and leave Telegram/Discord parity to follow-up work.
