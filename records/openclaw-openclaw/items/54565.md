## Summary

Keep open. Current main still only accepts WhatsApp debounce at channel/account/global inbound scope, passes one scalar debounce window into the listener, and explicitly excludes media messages from debounce, so the reported image/image/text group batching problem remains unresolved.

## What This Changes

Request per-group `debounceMs` for WhatsApp group config, with possible follow-up parity for Telegram, Discord, and other group-capable plugins.

## Best Possible Solution

Implement the WhatsApp plugin path first: accept `groups.<jid>.debounceMs`, resolve exact group before wildcard before account/channel/global fallback, and preserve media context when a later text message triggers the batched turn. Treat Telegram, Discord, and other plugins as separate follow-ups or a shared-contract design after the WhatsApp behavior is proven.
