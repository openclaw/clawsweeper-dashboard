- `impact:session-state`: Splitting rapid group messages into separate agent turns can leave the agent transcript and context incomplete for the user's intended instruction.

## Summary

Keep open: current main and v2026.5.22 still expose WhatsApp inbound debounce only at global/channel/account scope, while strict per-group config accepts mention/tools/prompt fields but not `debounceMs`. The related timestamp, media, and composing debounce items are adjacent but do not supersede this per-group override request.

## What This Changes

Allow `debounceMs` overrides inside WhatsApp per-group config, and potentially equivalent per-group scopes for other channels, so selected chats can batch rapid messages without changing whole-account latency.

## Best Possible Solution

Define a WhatsApp-owned per-group debounce contract first, then implement exact-group-before-wildcard fallback with schema/types/docs and attachment-preserving behavior or an explicit media-scope boundary.

## Reproduction Assessment

Yes, source-level. Current main's strict WhatsApp group schema omits `debounceMs`, runtime resolves only one scalar account/channel debounce before listener creation, and I did not run a live WhatsApp group session.

## Solution Assessment

Unclear for the full request. A scoped WhatsApp per-group override is plausible, but maintainers should choose fallback semantics, media behavior, and whether cross-channel parity belongs in this change.
