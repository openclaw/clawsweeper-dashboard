## Summary

Keep open. Current main and the latest release still support WhatsApp debounce only at global/channel/account scope, while the requested per-group config key is rejected by the strict group schema; the remaining question is a compatibility-sensitive product decision about adding that config surface.

## What This Changes

The issue requests a per-group `debounceMs` override for WhatsApp group config, with a broader suggestion that similar per-target debounce overrides could apply to other channels.

## Best Possible Solution

Approve a narrow WhatsApp-owned group-level debounce config only if maintainers want the new config surface; then wire it through the existing per-item debouncer with explicit immediate-flush rules and focused docs/tests.

## Reproduction Assessment

Yes, at source level: current main rejects `debounceMs` in strict WhatsApp group entries and only resolves debounce at broader scopes. I did not run a live WhatsApp repro because the requested config gap is directly visible in source and release-tag inspection.

## Solution Assessment

Unclear until maintainers decide the config contract. The narrowest maintainable path appears to be a WhatsApp-owned per-group override that reuses the existing debouncer, not a broad cross-channel expansion in the same change.
