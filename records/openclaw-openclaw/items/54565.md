## Summary

Keep open: current main and the latest release still expose WhatsApp inbound debounce only at shared/channel/account scope, while strict per-group config omits `debounceMs`. The linked timestamp, media, and composing debounce items are adjacent, but none supersedes this per-group override request.

## What This Changes

Requested support for per-group inbound debounce configuration for WhatsApp groups, with possible later parity for other channels.

## Best Possible Solution

Ship a WhatsApp-owned group debounce override that honors exact group before wildcard, preserves account/root fallback behavior, keeps attachment-bearing batches intact, and leaves broader channel parity to a separate follow-up design.

## Reproduction Assessment

Yes, source-level. Current main's strict WhatsApp group schema omits `debounceMs`, runtime resolves one scalar channel/account debounce before listener creation, and media messages bypass the existing debounce predicate.

## Solution Assessment

Unclear for the full request. A scoped WhatsApp implementation is plausible, but maintainers should choose the per-group fallback semantics, media attachment behavior, and whether cross-channel parity belongs in this change.
