## Summary

Keep open. Current main and v2026.6.8 still lack the requested per-group WhatsApp debounce config, and adding it is a useful but compatibility-sensitive config/product decision rather than stale cleanup.

## What This Changes

Request to allow `debounceMs` inside per-group channel config, starting with WhatsApp groups and possibly extending to other channels.

## Best Possible Solution

Approve a narrow WhatsApp-owned group-level debounce override only if maintainers want the new config contract, then wire it through the existing debouncer seam with focused docs and regression tests.

## Reproduction Assessment

Yes, at source level. The requested `channels.whatsapp.groups.<jid>.debounceMs` key is rejected by the strict group schema, and the runtime only reads debounce values from broader account/channel/global scopes.

## Solution Assessment

Unclear until maintainers choose the config contract. The narrowest maintainable path appears to be a WhatsApp-owned per-group override that reuses the existing debouncer seam, not a broad cross-channel rollout in the same change.
