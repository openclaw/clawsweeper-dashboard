## Summary

## What This Changes

The branch adds a pre-pass in `repairToolUseResultPairing` that skips later assistant turns whose extracted `tool_call_id`s were all seen before, plus two transcript repair tests.

## Best Possible Solution

Constrain the dedupe to exact duplicate or tool-call-only legacy assistant turns, or preserve non-tool content while removing duplicate tool-call blocks, then prove it in a redacted real lossless-claw/context-engine assembly.

## Reproduction Assessment

No high-confidence live reproduction of the reported lossless-claw legacy database case was established here. The PR-introduced risk is source-reproducible because the new dedupe condition only checks extracted tool-call ids while the extractor ignores non-tool assistant content.

## Solution Assessment

No. The PR is a plausible mitigation, but the best fix must avoid whole-turn data loss by preserving non-tool assistant content or limiting drops to exact duplicate legacy turns.
