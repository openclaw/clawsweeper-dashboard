## Summary

Keep open. Current main still lacks a per-agent env config surface, host exec still starts from the gateway process environment, and the related implementation PR is open but changes-requested rather than merged.

## What This Changes

Add `agents.list[].env.vars` so each agent receives only global env vars plus its own scoped credentials during Bash/tool execution.

## Best Possible Solution

Land a maintainer-approved scoped-env contract, likely through or replacing https://github.com/openclaw/openclaw/pull/62011, with focused tests and docs proving no cross-agent credential exposure.

## Reproduction Assessment

Yes. Source inspection on current main shows strict `agents.list[]` config without `env`, root-only `env.vars`, and host exec starting from `process.env`; no live two-agent credential setup was needed for triage.

## Solution Assessment

Unclear. `agents.list[].env.vars` is plausible, but the related implementation PR shows that the final shape needs maintainer agreement on baseline env, allowlisting, precedence, redaction, and docs before merge.
