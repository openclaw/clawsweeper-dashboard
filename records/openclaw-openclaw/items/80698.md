## Summary

Keep open. Current main and the latest release still expose only global config env and host exec inherits a sanitized gateway process environment; an open implementation PR exists, but it has security/design/proof blockers and has not landed.

## What This Changes

Requested per-agent environment-variable scoping through `agents.list[].env.vars` so one agent’s credentials are not visible to other agents’ tool processes.

## Best Possible Solution

Review or replace the existing per-agent env-isolation PR with a schema-backed design that documents scope, precedence, filtering, and redaction, then keep this issue open until that lands with tests and real-run proof.

## Reproduction Assessment

Yes. Source inspection on current main shows no accepted `agents.list[].env` field, root `env.vars` remains global, and host exec starts from the gateway `process.env`; I did not run a live two-agent credential setup in this read-only pass.

## Solution Assessment

Unclear. `agents.list[].env.vars` is a plausible direction, but the maintainable fix needs maintainer agreement on scope semantics, precedence, redaction, SecretRef support, and how the open `env.scope` PR should evolve.
