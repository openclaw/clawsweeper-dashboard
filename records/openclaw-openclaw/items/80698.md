## Summary

Keep open. Current main still lacks an agent-scoped env config surface and host exec still starts from the gateway process environment; the open implementation PR is not merged and has security/design review blockers.

## What This Changes

Add per-agent environment variable scoping so one agent's Bash/tool process only sees global env plus its own configured credentials, not credentials meant for other agents.

## Best Possible Solution

Define a schema-backed per-agent env contract with explicit merge, filtering, redaction, SecretRef, and service metadata semantics, then land a reviewed fix or replacement for the open implementation PR with focused isolation tests and docs.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main has only root env.vars, no agents.list[].env schema/type surface, and non-sandbox host exec starts from process.env before generic filtering.

## Solution Assessment

Unclear. agents.list[].env.vars is a plausible direction, but the best solution needs maintainer agreement on precedence, allowed baseline host env, SecretRef/redaction behavior, and interaction with MCP/tool-specific credential surfaces.
