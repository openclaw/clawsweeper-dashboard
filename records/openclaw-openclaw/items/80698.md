## Summary

Keep open. Current main still lacks a per-agent environment config surface and host Bash execution still starts from the gateway process environment; the related open PR is an implementation candidate but is changes-requested and not a shipped fix.

## What This Changes

Request adds `agents.list[].env.vars` so each agent's Bash/tool process receives only global env plus its own scoped vars, preventing other agents' credentials from leaking.

## Best Possible Solution

Define a schema-backed per-agent env contract with explicit merge, filter, redaction, and SecretRef semantics; wire it through exec/tool env construction and managed-service metadata; and add regression tests proving two agents cannot see each other's scoped credentials.

## Reproduction Assessment

Yes. Source inspection on current main shows strict `agents.list[]` config has no env block, root env collection reads only `cfg.env`, and host exec starts from `process.env`.

## Solution Assessment

Unclear. `agents.list[].env.vars` is a plausible shape, but the safest solution needs maintainer agreement on precedence, baseline host variables, SecretRef/redaction, and composition with MCP/tool-specific credentials; the open PR is not ready yet.
