## Summary

Keep open: current main still has only top-level `env.vars` plus host/request exec env merging, while the requested agent-scoped env contract is absent and touches credential isolation policy.

## What This Changes

Add `agents.list[].env.vars` so each agent can receive scoped environment variables without inheriting other agents' credentials.

## Best Possible Solution

Design a schema-backed per-agent effective env contract that merges safely with global env defaults, preserves existing configs, filters dangerous host vars, updates managed-service key reporting, and adds focused multi-agent exec tests and docs.

## Reproduction Assessment

Yes, source-level: current main has strict agent schema/type contracts without `env`, and gateway exec builds from global process/request env rather than an agent-scoped map. I did not run a live two-agent `gh` identity flip reproduction in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the public contract. `agents.list[].env.vars` is plausible, but the safe solution needs explicit precedence, filtering, redaction, SecretRef, and reload semantics across exec, MCP, tools, and service-managed env.
