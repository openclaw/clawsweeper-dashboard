## What This Changes

The issue asks for `requireApiKey` missing-key errors to include the checked auth source label, such as an env var, auth profile, or `models.json`.

## Best Possible Solution

Review and land the open linked PR if it covers the intended helper path, then close this issue after merge; if maintainers want the agent-run diagnostic fully consistent, move the missing-auth message into one shared formatter.

## Reproduction Assessment

Yes. Source inspection shows `requireApiKey` can receive a `ResolvedProviderAuth` with `source: "env: OPENAI_API_KEY"` and no `apiKey`, and current main throws a message that omits that source label.

## Solution Assessment

Yes for the helper-level bug: including `auth.source` in the existing missing-key message is the narrow fix. Maintainers should still check whether duplicated direct agent-run messages need the same formatter before treating the user-visible run path as complete.
