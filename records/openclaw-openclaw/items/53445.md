## Summary

Keep open. The PR addresses a valid open Control UI feature request and current main still renders relative Updated values, but the patch should not merge as-is because it regresses the localized missing timestamp fallback and omits the required changelog entry.

## What This Changes

The PR adds a `formatFullDateTime` Control UI helper with tests and changes the Sessions Updated cell to show `YYYY-MM-DD HH:mm` with the relative time in the tooltip.

## Best Possible Solution

Land a rebased narrow implementation that adds the reusable full date-time formatter, preserves `t("common.na")` for missing values, keeps the relative timestamp tooltip, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Current main can be verified from source because the Sessions renderer still uses `formatRelativeTimestamp(row.updatedAt)`, and the PR regression can be reproduced by rendering a missing `updatedAt` row under a locale where `common.na` is not `"n/a"`.

## Solution Assessment

No, not as-is. The implementation direction is maintainable, but the Sessions integration should pass the localized fallback and the user-facing feature needs a changelog entry before merge.
