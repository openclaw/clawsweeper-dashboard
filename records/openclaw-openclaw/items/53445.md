## What This Changes

The PR adds a `formatFullDateTime` helper with tests and changes the Control UI Sessions Updated cell to show `YYYY-MM-DD HH:mm` with the relative value in the title tooltip.

## Best Possible Solution

Rebase and fix the PR so the Sessions Updated column uses a tested full local date-time formatter while preserving localized missing-value fallback, existing sorting/API behavior, the relative-time tooltip, and current Sessions columns.

## Reproduction Assessment

Yes for the current behavior gap. Source inspection on current main shows non-null `updatedAt` is still converted through `formatRelativeTimestamp(row.updatedAt)` and rendered directly in the Sessions Updated cell.

## Solution Assessment

No, not as-is. The implementation direction is narrow and appropriate, but it should pass through the localized `t("common.na")` fallback, rebase over the current Sessions renderer, and include real Control UI proof before merge.
