## Summary

Keep open: current main still lacks outbound delivery secret sanitization, but this PR is not merge-ready because source inspection shows bypasses after its early text-only transform, an incomplete credential inventory, a missing config contract, a conflicting branch, and no real outbound proof.

## What This Changes

The PR adds a default-on gateway outbound secret sanitizer that rewrites detected text secrets to named key(...) handles or high-entropy redaction before channel delivery, with unit tests.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks, rendering, and adapter normalization, recursively covers outbound string-bearing payload fields, uses canonical sensitive/SecretRef metadata, declares the config contract, and includes upgrade plus live channel proof.

## Reproduction Assessment

Yes, at source level. Current main normalizes payloads before message_sending hooks, rendering, and final sends, while the PR inserts the sanitizer only in the early normalization step and only for payload.text.

## Solution Assessment

No. Gateway-layer sanitization is the right direction, but this implementation needs to move to the final-send boundary, cover structured fields, use canonical sensitive metadata, and expose the advertised config option before merge.
