## Summary

Keep open for maintainer/contributor handling: the delivery-layer sanitizer is still useful, but this branch is not merge-ready because source inspection confirms early-only text redaction, security-boundary bypasses, an undeclared config opt-out, missing real behavior proof, and a dirty merge state.

## What This Changes

The branch adds a default-on outbound secret sanitizer, wires it into outbound delivery normalization, and adds unit tests for prefix, entropy, named-handle, and opt-out behavior.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks/rendering/adapter normalization, recursively sanitizes outbound string-bearing payload fields, builds handles from canonical sensitive or SecretRef metadata, declares the config contract, and includes upgrade plus live channel proof.

## Reproduction Assessment

Yes, at source level: current main normalizes before hooks/rendering/final sends, while the PR inserts the sanitizer only in the early normalization step. I did not run a live channel send in this read-only review.

## Solution Assessment

No. Gateway-layer sanitization is the right direction, but this implementation needs to move to the final-send boundary, cover structured fields, use canonical sensitive metadata, and expose the advertised config option before merge.

## Security Review

Status: needs_attention

Summary: The patch is intended security hardening, but its placement and coverage leave concrete outbound redaction bypasses.

Concerns:

- **[high] Hook rewrites bypass redaction:** `src/infra/outbound/deliver.ts:331`
  - body: Because the sanitizer runs before message_sending hooks, hook-supplied replacement content can introduce a secret after the only sanitizer pass and still reach channel delivery.
  - confidence: 0.93
- **[medium] Structured payloads bypass redaction:** `src/infra/outbound/deliver.ts:331`
  - body: The sanitizer only rewrites payload.text, while structured payload fields can carry user-visible strings through sendPayload.
  - confidence: 0.9
- **[medium] Configured secrets are incompletely indexed:** `src/infra/outbound/sanitize-secrets.ts:81`
  - body: The ad hoc credential index omits many registered sensitive and SecretRef config surfaces, so known low-entropy secrets outside top-level auth/channel strings may not be redacted or mapped to handles.
  - confidence: 0.88
- **[medium] Quoted structured tokens can bypass matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: Keeping JSON/YAML punctuation attached to candidate values prevents known-prefix and exact-value matching for common agent outputs that quote or structure secrets.
  - confidence: 0.86
