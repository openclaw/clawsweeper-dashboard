## Summary

Keep this PR open, but it is not merge-ready: the branch targets a real outbound secret-delivery security gap, yet the current implementation runs too early, covers only `payload.text`, has an incomplete credential inventory, lacks the advertised config contract, is conflict-marked, and still has no real outbound behavior proof.

## What This Changes

The PR adds a default-on outbound secret sanitizer under `src/infra/outbound` that rewrites detected `payload.text` secrets to `key(...)` handles or high-entropy redaction markers before channel delivery.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks, rendering, and adapter normalization, recursively covers outbound string-bearing fields, uses canonical sensitive/SecretRef metadata, declares the config contract, and includes upgrade plus live channel proof.

## Reproduction Assessment

Yes, at source level. Current main normalizes payloads before `message_sending` hooks, rendering, and final sends, while the PR inserts the sanitizer only in the early normalization step and only for `payload.text`; I did not run a live channel send in this read-only review.

## Solution Assessment

No. Gateway-layer sanitization is the right direction, but this implementation needs to move to the final-send boundary, cover structured fields, use canonical sensitive metadata, and expose the advertised config option before merge.

  - body: This sanitizer runs during early payload normalization, but `message_sending` hooks and presentation rendering can replace outbound content later. A hook-generated or renderer-generated secret would bypass the only sanitizer pass and then reach final channel delivery.

  - confidence: 0.88
- **[P1] Declare the sanitizeSecrets config option:** `src/infra/outbound/sanitize-secrets.ts:216-219`
  - body: `isSecretSanitizationEnabled` reads `messages.sanitizeSecrets`, but `MessagesSchema` is strict and `MessagesConfig` omits that field. Users cannot rely on the advertised opt-out through validated config, which is merge-critical for a default-on content rewrite.
