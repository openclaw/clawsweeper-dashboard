## Summary

Keep open, but this PR is not merge-ready: the security goal is valid, yet the current patch leaves concrete outbound redaction bypasses, changes delivered content by default without a declared config contract, is conflicting with main, and still lacks real outbound behavior proof.

## What This Changes

The PR adds a default-on gateway outbound secret sanitizer, wires it into outbound delivery normalization, and adds unit tests for prefix, entropy, named-handle, and opt-out behavior.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks/rendering/adapter normalization, recursively sanitizes outbound string-bearing payload fields, builds handles from canonical sensitive or SecretRef metadata, declares the config contract, and includes upgrade plus live channel proof.

## Reproduction Assessment

Yes, at source level: current main runs message_sending hooks, presentation rendering, later adapter normalization, and final sends after early normalization, while the PR inserts the sanitizer only in that early normalization step. I did not run a live channel send in this read-only review.

## Solution Assessment

No. Gateway-layer sanitization is the right direction, but this implementation needs to move to the final-send boundary, cover structured fields, use canonical sensitive metadata, and expose the advertised config option before merge.

Full review comments:

- **[P1] Move redaction to the final send boundary:** `src/infra/outbound/deliver.ts:331-332`
  - body: This sanitizer runs during early payload normalization, but message_sending hooks and presentation rendering can replace outbound content later. A hook-generated or renderer-generated secret would bypass the only sanitizer pass and then reach channel delivery.
  - confidence: 0.93
- **[P1] Sanitize structured payload fields too:** `src/infra/outbound/deliver.ts:331-332`
  - body: The transform only rewrites payload.text, but the delivery path can send presentation, interactive, and channelData through sendPayload. Secrets in labels, placeholders, values, card text, URLs, or channel-specific fields remain unsanitized.
  - confidence: 0.9
- **[P1] Build handles from canonical sensitive config:** `src/infra/outbound/sanitize-secrets.ts:81-108`
  - body: buildCredentialIndex walks only cfg.auth and top-level channel string values. Current schemas mark many provider, tool, gateway, cron, nested channel, and SecretRef fields as sensitive, so low-entropy configured secrets there can still be delivered verbatim.
  - confidence: 0.88
- **[P1] Split quoted JSON and YAML values before matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: The tokenizer keeps quotes, braces, and colons attached to candidate values. JSON or YAML such as {"token":"ghp_..."} can miss exact-value and known-prefix matching because the token no longer starts with the secret prefix.
  - confidence: 0.86
- **[P2] Declare the sanitizeSecrets config option:** `src/infra/outbound/sanitize-secrets.ts:216-219`

  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The patch is intended security hardening, but its placement and coverage leave concrete outbound redaction bypasses.

Concerns:

- **[high] Hook rewrites bypass redaction:** `src/infra/outbound/deliver.ts:331`
  - body: Because the sanitizer runs before message_sending hooks, hook-supplied replacement content can introduce a secret after the only sanitizer pass and still reach channel delivery.
  - confidence: 0.93
- **[medium] Structured payloads bypass redaction:** `src/infra/outbound/deliver.ts:331`
  - body: The sanitizer only rewrites payload.text, while structured payload fields can carry user-visible strings through sendPayload.
  - confidence: 0.9
- **[medium] Configured secrets are incompletely indexed:** `src/infra/outbound/sanitize-secrets.ts:81`
  - body: The ad hoc credential index omits many registered sensitive and SecretRef config surfaces, so known low-entropy secrets outside top-level auth/channel strings may not be redacted or mapped to handles.
  - confidence: 0.88
- **[medium] Quoted structured tokens can bypass matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: Keeping JSON/YAML punctuation attached to candidate values prevents known-prefix and exact-value matching for common agent outputs that quote or structure secrets.
  - confidence: 0.86
