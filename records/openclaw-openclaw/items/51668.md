## Summary

Keep open. The PR targets a still-missing outbound secret-sanitization capability, but the current branch is not safe to merge because the sanitizer runs before later hook/render/send mutations, only rewrites `payload.text`, and builds named handles from an incomplete credential surface; GitHub also reports the branch as non-mergeable.

## What This Changes

The PR adds an outbound `sanitize-secrets` helper and tests, wires delivery normalization to replace `payload.text` secrets with `key(...)` or `[REDACTED:high-entropy]`, and adds a `messages.sanitizeSecrets` opt-out helper.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks, presentation rendering, normalization, and channel-specific rendering; recursively covers all outbound user-visible strings; derives named handles from canonical sensitive/SecretRef metadata and runtime auth surfaces; and updates schema, docs, changelog, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: a `message_sending` hook can return a secret after the only sanitizer pass, structured payload fields can carry secrets through `sendPayload`, and `messages.sanitizeSecrets` is absent from the strict config schema.

## Solution Assessment

No. A gateway final-send sanitizer is the right direction, but this implementation is not the narrow maintainable fix because it runs too early and uses an incomplete credential index; the safer path is a final-boundary recursive sanitizer backed by canonical sensitive metadata.

## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] Re-sanitize after message_sending hooks:** `src/infra/outbound/deliver.ts:625`
  - body: The sanitizer runs during payload normalization, but `message_sending` hooks run later and can replace `payload.text` before `sendPayload`, `sendFormattedText`, or chunked `sendText`. A hook-supplied secret bypasses the only sanitizer pass, so redaction needs to happen again at the final effective-send boundary.
  - confidence: 0.93
- **[P2] Sanitize structured payload strings too:** `src/infra/outbound/deliver.ts:324`
  - body: This only calls `sanitizeSecrets` for `sanitizedPayload.text`, leaving `presentation`, `interactive`, and `channelData` strings untouched even though those fields can be delivered through `sendPayload`. Secrets in button labels, option values, placeholders, presentation blocks, or channel-specific payloads would still be sent verbatim.
  - confidence: 0.9
- **[P1] Build handles from canonical secret metadata:** `src/infra/outbound/sanitize-secrets.ts:82-108`
  - body: `buildCredentialIndex` only walks `cfg.auth` and top-level `cfg.channels` strings, but many supported plaintext credentials live under sensitive/SecretRef surfaces such as model provider keys/headers, gateway auth, tools, skills, proxy, and nested channel account fields. Low-entropy configured secrets on those paths can miss both named handles and entropy redaction.
  - confidence: 0.88
- **[P1] Tokenize quoted and JSON values before matching:** `src/infra/outbound/sanitize-secrets.ts:151-154`
  - body: The scanner splits on whitespace, commas, semicolons, pipes, and `=`, but keeps quotes, braces, and colons attached. Common JSON/YAML/code output like `{"token":"ghp_..."}` or `password: "shared-password-0000"` will not exact-match configured credentials or known prefixes, so those values can bypass redaction.
  - confidence: 0.86
- **[P2] Declare messages.sanitizeSecrets in the config contract:** `src/infra/outbound/sanitize-secrets.ts:216-219`
  - body: The helper reads `messages.sanitizeSecrets`, but `MessagesSchema` is strict and `MessagesConfig` does not declare that field. Users cannot set the advertised opt-out in `openclaw.json` without schema validation rejecting it, so the schema, type, docs, and generated help need to be updated together.
  - confidence: 0.91

  - confidence: 0.93

  - confidence: 0.88

  - body: Keeping JSON/YAML punctuation attached to candidate values prevents known-prefix and exact-value matching for common agent outputs that quote or structure secrets.
  - confidence: 0.86
