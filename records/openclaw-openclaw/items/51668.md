## Summary

Keep open: this is the canonical implementation PR for outbound secret sanitization, but the current patch still leaves concrete delivery-layer bypasses and lacks required real behavior proof for an external security-boundary change.

## What This Changes

The branch adds an outbound secret-sanitizer module, wires text redaction into outbound delivery normalization, and adds tests for prefix, named-handle, entropy, delimiter, and opt-out behavior.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks and rendering, recursively covers user-visible payload strings, derives handles from canonical sensitive or SecretRef metadata, and updates schema, docs, regression tests, and live channel proof together.

## Reproduction Assessment

Yes, for the review findings via source inspection: the PR sanitizes early payload.text, while current main can rewrite content afterward through message_sending and can deliver structured strings through sendPayload. I did not run a live channel delivery path in this read-only review.

## Solution Assessment

No. A gateway-layer sanitizer is the right direction, but this implementation should move to the final-send boundary, cover structured payloads, use canonical sensitive metadata, and include the advertised config contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Re-sanitize after message_sending hooks:** `src/infra/outbound/deliver.ts:331-332`
  - body: The new redaction pass runs before message_sending hooks, but those hooks can later replace outbound text. Hook-generated content containing a secret would reach the final send without another sanitizer pass.
  - confidence: 0.93
- **[P2] Sanitize structured payload strings too:** `src/infra/outbound/deliver.ts:331-332`
  - body: This only rewrites payload.text, while interactive and channelData payloads still go through sendPayload. Secrets in labels, placeholders, values, or structured text would bypass redaction.
  - confidence: 0.9
- **[P1] Use canonical sensitive metadata for handles:** `src/infra/outbound/sanitize-secrets.ts:81-108`
  - body: buildCredentialIndex only walks cfg.auth and top-level channel strings, but configured secrets also live under providers, tools, gateway auth, cron, nested channel settings, and SecretRef-backed surfaces. Low-entropy values there can miss both named handles and entropy redaction.
  - confidence: 0.88
- **[P1] Split quoted structured values before matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: The tokenizer keeps quotes, braces, and colons attached to candidate values. Common JSON/YAML such as a quoted token field can miss exact-value and known-prefix matching.
  - confidence: 0.86
- **[P2] Declare sanitizeSecrets in the messages config schema:** `src/infra/outbound/sanitize-secrets.ts:216-219`
  - body: isSecretSanitizationEnabled reads messages.sanitizeSecrets, but the strict MessagesSchema and MessagesConfig omit that field. The advertised opt-out would be rejected by config validation.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The patch changes the outbound redaction boundary but still leaves concrete paths where secrets can bypass the new sanitizer.

Concerns:

- **[high] Hook rewrites bypass redaction:** `src/infra/outbound/deliver.ts:331`
  - body: Because the sanitizer is applied before message_sending hooks, hook-supplied replacement content can introduce a secret after the only sanitizer pass and still reach channel delivery.
  - confidence: 0.93
- **[medium] Structured outbound payloads bypass redaction:** `src/infra/outbound/deliver.ts:331`
  - body: The sanitizer only rewrites payload.text, while structured payload fields can carry user-visible strings through sendPayload.
  - confidence: 0.9
- **[medium] Configured secrets are incompletely indexed:** `src/infra/outbound/sanitize-secrets.ts:81`
  - body: The ad hoc credential index omits many registered sensitive and SecretRef config surfaces, so known low-entropy secrets outside top-level auth/channel strings may not be redacted or mapped to handles.
  - confidence: 0.88
- **[medium] Quoted structured tokens can bypass matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: Keeping JSON/YAML punctuation attached to candidate values prevents known-prefix and exact-value matching for common agent outputs that quote or structure secrets.
  - confidence: 0.86
