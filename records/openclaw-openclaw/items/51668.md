## Summary

Keep open. Current main has not implemented the requested outbound secret-reference layer, and this PR is not merge-ready because source review shows concrete bypasses in a security-sensitive delivery boundary.

## What This Changes

The PR adds a gateway outbound secret sanitizer, wires it into outbound delivery normalization, and adds focused sanitizer tests.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks and final payload normalization, recursively covers user-visible payload strings, derives handles from canonical sensitive/SecretRef metadata, and updates schema, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: `message_sending` can replace outbound text after the PR's sanitizer pass, structured payload strings can reach `sendPayload`, and the advertised `messages.sanitizeSecrets` option is absent from the strict config schema.

## Solution Assessment

No. A gateway-level outbound sanitizer is the right direction, but this implementation runs too early, only handles `payload.text`, and builds handles from an incomplete ad hoc credential index.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Re-sanitize after message_sending hooks:** `src/infra/outbound/deliver.ts:624-625`
  - body: The new pass runs during payload normalization, but `message_sending` hooks run later and can replace the payload text before final delivery. A hook-generated secret would bypass the only sanitizer pass, so redaction needs to happen at the final effective-send boundary too.
  - confidence: 0.93
- **[P2] Sanitize structured payload strings too:** `src/infra/outbound/deliver.ts:324-326`
  - body: This only rewrites `sanitizedPayload.text`, while `presentation`, `interactive`, and `channelData` can still be sent through `sendPayload`. Secrets in labels, values, placeholders, presentation text, or channel-specific payloads would remain visible.
  - confidence: 0.9
- **[P1] Use canonical sensitive metadata for handles:** `src/infra/outbound/sanitize-secrets.ts:82-108`
  - body: `buildCredentialIndex` only walks `cfg.auth` and top-level `cfg.channels` strings, but current config has many sensitive and SecretRef paths under providers, hooks, tools, gateway auth, and nested channel accounts. Low-entropy configured secrets there can miss both named handles and entropy redaction.
  - confidence: 0.88
- **[P1] Split quoted structured values before matching:** `src/infra/outbound/sanitize-secrets.ts:151-154`
  - body: The tokenizer still keeps quotes, braces, and colons attached to candidate values. Common JSON/YAML output like `{"token":"ghp_..."}` or `password: "shared-password-0000"` can therefore miss exact-value and known-prefix matching.
  - confidence: 0.86
- **[P2] Declare messages.sanitizeSecrets in schema:** `src/infra/outbound/sanitize-secrets.ts:216-219`
  - body: The helper reads `messages.sanitizeSecrets`, but `MessagesSchema` is strict and `MessagesConfig` does not include that field. Users cannot set the advertised opt-out in `openclaw.json` without schema validation rejecting it.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff changes the outbound redaction boundary but leaves concrete bypasses through hooks, structured payload fields, tokenization, and incomplete credential indexing.

Concerns:

- **[high] Hook rewrites bypass redaction:** `src/infra/outbound/deliver.ts:624`
  - body: Because the sanitizer is applied before `message_sending` hooks, hook-supplied replacement content can introduce a secret after the only sanitizer pass and still reach channel delivery.
  - confidence: 0.93
- **[medium] Structured outbound payloads bypass redaction:** `src/infra/outbound/deliver.ts:324`
  - body: The sanitizer only rewrites `payload.text`, while structured payload fields can carry user-visible strings through `sendPayload`.
  - confidence: 0.9
- **[medium] Configured secrets are incompletely indexed:** `src/infra/outbound/sanitize-secrets.ts:82`
  - body: The ad hoc credential index omits many registered sensitive and SecretRef config surfaces, so known low-entropy secrets outside top-level auth/channel strings may not be redacted.
  - confidence: 0.88
- **[medium] Quoted structured tokens can bypass matching:** `src/infra/outbound/sanitize-secrets.ts:151`
  - body: Keeping JSON/YAML punctuation attached to candidate values prevents known-prefix and exact-value matching for common agent outputs that quote or structure secrets.
  - confidence: 0.86
