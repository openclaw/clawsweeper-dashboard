## Summary

Keep open: current main still lacks the requested delivery-layer secret sanitizer, and this PR is the canonical open implementation path, but the patch still has concrete security-bypass findings and no real outbound behavior proof.

## What This Changes

The PR adds an outbound secret sanitizer, wires it into delivery normalization, advertises `messages.sanitizeSecrets`, and adds focused unit tests.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks/rendering, recursively sanitizes outbound payload strings, indexes canonical sensitive or SecretRef-backed config values, declares the config contract, and includes live channel proof.

## Reproduction Assessment

Yes, at source level: the PR sanitizes only during early normalization, while current main can rewrite content afterward through `message_sending` hooks and can deliver structured strings through `sendPayload`. I did not run a live channel delivery in this read-only review.

## Solution Assessment

No. Gateway-layer sanitization is the right direction, but this implementation needs to move to the final-send boundary, cover structured fields, use canonical sensitive metadata, and expose the advertised config option before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Re-sanitize at the final send boundary:** `src/infra/outbound/deliver.ts:327-334`
  - body: This runs before `message_sending` hooks, but those hooks can replace outbound text later. Hook-generated secrets would bypass the only sanitizer pass and reach final channel delivery.
  - confidence: 0.93
- **[P2] Sanitize structured outbound payload fields:** `src/infra/outbound/deliver.ts:331-332`
  - body: Only `payload.text` is rewritten, while payloads with `interactive`, `presentation`, or `channelData` still go through `sendPayload`. Secrets in labels, placeholders, values, card text, or channel-specific fields remain unsanitized.
  - confidence: 0.9
- **[P1] Build handles from canonical sensitive config metadata:** `src/infra/outbound/sanitize-secrets.ts:81-108`
  - body: `buildCredentialIndex` walks only `cfg.auth` and top-level channel string values. Current schemas mark many provider, tool, gateway, cron, nested channel, and SecretRef fields as sensitive, so low-entropy configured secrets there can still be sent verbatim.
  - confidence: 0.88
- **[P1] Split quoted and structured values before matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: The tokenizer keeps quotes, braces, and colons attached to candidate values. JSON or YAML such as `{"token":"ghp_..."}` can miss exact-value and known-prefix matching because the token no longer starts with the secret prefix.
  - confidence: 0.86
- **[P2] Declare `messages.sanitizeSecrets` in the config contract:** `src/infra/outbound/sanitize-secrets.ts:216-219`
  - body: `isSecretSanitizationEnabled` reads `messages.sanitizeSecrets`, but `MessagesSchema` is strict and `MessagesConfig` omits that field. Users cannot rely on the advertised opt-out through validated config.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The patch is intended security hardening, but its placement and coverage leave concrete redaction bypasses.

Concerns:

- **[high] Hook rewrites bypass redaction:** `src/infra/outbound/deliver.ts:327`
  - body: Because the sanitizer runs before `message_sending` hooks, hook-supplied replacement content can introduce a secret after the only sanitizer pass and still reach channel delivery.
  - confidence: 0.93
- **[medium] Structured payloads bypass redaction:** `src/infra/outbound/deliver.ts:331`
  - body: The sanitizer only rewrites `payload.text`, while structured payload fields can carry user-visible strings through `sendPayload`.
  - confidence: 0.9
- **[medium] Configured secrets are incompletely indexed:** `src/infra/outbound/sanitize-secrets.ts:81`
  - body: The ad hoc credential index omits many registered sensitive and SecretRef config surfaces, so known low-entropy secrets outside top-level auth/channel strings may not be redacted or mapped to handles.
  - confidence: 0.88
- **[medium] Quoted structured tokens can bypass matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: Keeping JSON/YAML punctuation attached to candidate values prevents known-prefix and exact-value matching for common agent outputs that quote or structure secrets.
  - confidence: 0.86
