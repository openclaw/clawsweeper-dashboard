## Summary

Keep this PR open: current main still lacks the requested gateway-layer secret replacement, and the closed related issue points to this PR as the canonical implementation path. The proposed patch is not merge-ready because the sanitizer runs too early, covers only `payload.text`, indexes credentials ad hoc, omits the advertised config schema, and still lacks real outbound behavior proof.

## What This Changes

The PR adds an outbound secret sanitizer module and tests, then wires an early `payload.text` redaction pass into outbound delivery using named `key(...)` handles and entropy fallback redaction.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks and final payload rendering, recursively covers user-visible payload strings, derives handles from canonical sensitive/SecretRef metadata, and updates schema, docs, changelog, and regression tests together.

## Reproduction Assessment

Yes for the review findings via source inspection: the PR sanitizes only early `payload.text`, while current main can rewrite content afterward through `message_sending` and can deliver structured strings through `sendPayload`. I did not run a live channel delivery path in this read-only review.

## Solution Assessment

No. A gateway-layer sanitizer is the right direction, but this implementation should move to the final-send boundary, cover structured payloads, use canonical sensitive metadata, and include the advertised config contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Re-sanitize after message_sending hooks:** `src/infra/outbound/deliver.ts:324-326`
  - body: The sanitizer runs during early payload normalization, but `message_sending` hooks run later and can replace outbound text. Hook-generated content containing a secret would reach the final send without another redaction pass.
  - confidence: 0.93
- **[P2] Sanitize structured payload strings too:** `src/infra/outbound/deliver.ts:324-326`
  - body: This only rewrites `payload.text`, while `presentation`, `interactive`, and `channelData` can still be delivered through `sendPayload`. Secrets in labels, placeholders, values, or presentation text would bypass the sanitizer.
  - confidence: 0.9
- **[P1] Use canonical sensitive metadata for handles:** `src/infra/outbound/sanitize-secrets.ts:82-108`
  - body: `buildCredentialIndex` walks only `cfg.auth` and top-level channel strings, but configured secrets also live under providers, tools, gateway auth, cron, and nested channel accounts. Low-entropy values there can miss both named handles and entropy redaction.
  - confidence: 0.88
- **[P1] Split quoted structured values before matching:** `src/infra/outbound/sanitize-secrets.ts:150-154`
  - body: The tokenizer keeps quotes, braces, and colons attached to candidate values. Common JSON/YAML like `{"token":"ghp_..."}` or `password: "..."` can miss exact-value and prefix matching.
  - confidence: 0.86
- **[P2] Declare messages.sanitizeSecrets in schema:** `src/infra/outbound/sanitize-secrets.ts:216-219`
  - body: `isSecretSanitizationEnabled` reads `messages.sanitizeSecrets`, but the strict `MessagesSchema` and `MessagesConfig` omit that field. The advertised opt-out would be rejected by config validation.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff changes the outbound redaction boundary but leaves concrete bypasses through hooks, structured payload fields, tokenization, and incomplete credential indexing.

Concerns:

- **[high] Hook rewrites bypass redaction:** `src/infra/outbound/deliver.ts:324`
  - body: Because the sanitizer is applied before `message_sending` hooks, hook-supplied replacement content can introduce a secret after the only sanitizer pass and still reach channel delivery.
  - confidence: 0.93
- **[medium] Structured outbound payloads bypass redaction:** `src/infra/outbound/deliver.ts:324`
  - body: The sanitizer only rewrites `payload.text`, while structured payload fields can carry user-visible strings through `sendPayload`.
  - confidence: 0.9
- **[medium] Configured secrets are incompletely indexed:** `src/infra/outbound/sanitize-secrets.ts:82`
  - body: The ad hoc credential index omits many registered sensitive and `SecretRef` config surfaces, so known low-entropy secrets outside top-level auth/channel strings may not be redacted.
  - confidence: 0.88
- **[medium] Quoted structured tokens can bypass matching:** `src/infra/outbound/sanitize-secrets.ts:150`
  - body: Keeping JSON/YAML punctuation attached to candidate values prevents known-prefix and exact-value matching for common agent outputs that quote or structure secrets.
  - confidence: 0.86
