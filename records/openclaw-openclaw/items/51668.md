## Summary

Keep this PR open, but it is not merge-ready: the security-hardening direction is useful and current main still lacks gateway-layer outbound secret handles, yet this patch leaves concrete final-delivery bypasses and has no real outbound-send proof.

## What This Changes

The PR adds an outbound secret-sanitizer module, wires text redaction into delivery normalization, advertises `messages.sanitizeSecrets`, and adds focused unit tests.

## Best Possible Solution

Land a revised final-send sanitizer that runs after hooks/rendering, recursively sanitizes outbound payload strings, indexes canonical sensitive or SecretRef-backed config values, declares the config contract, and includes live channel proof.

## Reproduction Assessment

Yes, at source level: the PR sanitizes during early normalization, while current main can rewrite content afterward through `message_sending` hooks and can deliver structured strings through `sendPayload`. I did not run a live channel send in this read-only review.

## Solution Assessment

No. Gateway-layer sanitization is the right direction, but this implementation needs to move to the final-send boundary, cover structured fields, use canonical sensitive metadata, and expose the advertised config option before merge.
