## Summary

The PR adds `model: resolvedModel || undefined` and `allowModelOverride: true` to the native subagent gateway `agent` call.

## Best Possible Solution

Forward the resolved model through a server-side authorized gateway path, remove the unsupported request param, add regression coverage for websocket and in-process native subagent dispatch, and require redacted accepted-run proof.

## Reproduction Assessment

Yes at source level. Current main resolves and reports a subagent model but does not pass it into the child `agent` call, and the PR defect is source-reproducible because the added `allowModelOverride` field is rejected by the strict gateway schema.

## Solution Assessment

No. Passing `model` is necessary, but `allowModelOverride` as a request param is neither accepted by the schema nor honored by the authorization code; the fix belongs at the gateway client/internal dispatch seam.
