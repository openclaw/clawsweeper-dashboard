## Summary

Keep open: the PR targets a real sub-agent model-routing bug, but the patch is not merge-ready because it sends model-override trust as an unsupported strict `agent` request parameter instead of through the gateway client/options authorization path.

## What This Changes

The PR forwards `resolvedModel` and an `allowModelOverride` flag from `spawnSubagentDirect` into the child gateway `agent` call.

## Best Possible Solution

Forward the resolved model through an authorized gateway client/options path, remove the unsupported request param, cover both dispatch paths, and require redacted accepted-run proof.

Yes at source level: current main resolves and reports the requested subagent model but does not pass it into the child `agent` call. I did not run a live QQ/Qwen/DeepSeek reproduction in this read-only review.

## Solution Assessment

No. Passing `model` is necessary, but this patch puts authorization in an invalid request field; the maintainable fix is to carry trust through the gateway client/options seam.
