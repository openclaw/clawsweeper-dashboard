## Summary

Keep this PR open: it targets the right subagent model-routing bug, but the patch authorizes the forwarded model in the wrong place, so explicit model spawns can still fail at the gateway authorization gate.

## What This Changes

The PR adds `model: resolvedModel || undefined` and `allowModelOverride: true` to the `spawnSubagentDirect` gateway `agent` call.

## Best Possible Solution

Forward the resolved model through an authorization path the gateway actually honors, then prove an explicit `sessions_spawn.model` child run is accepted and uses the requested provider/model while untrusted direct overrides remain rejected.

## Reproduction Assessment

Yes at source level: current main resolves and reports the requested subagent model but does not pass it into the child gateway `agent` run. I did not run the reporter's QQ/Qwen/DeepSeek live setup in this read-only review.

## Solution Assessment

No. Passing `model` is necessary, but `allowModelOverride` in request params is not the gateway authorization contract; the narrow fix should authorize the internal subagent call at the client/options seam.
