## Summary

Keep open: current main still lacks the non-streaming transcript auto-response path, and this PR is a focused fix with source-backed reproduction, sufficient live Telnyx proof, green checks, and no blocking review findings.

## What This Changes

The PR adds a waiter-aware auto-response handoff for final non-streaming voice-call webhook speech events, exposes `CallManager.hasTranscriptWaiter`, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent voice-call plugin patch after maintainer approval and merge gates, preserving the waiter and transcript-growth guards.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Telnyx/Plivo final speech becomes `call.speech`, `processParsedEvents` only forwards it to the manager, and the manager records the transcript without invoking the response loop.

## Solution Assessment

Yes. Keeping the handoff in the webhook runtime is the narrow maintainable fix because that layer owns `handleInboundResponse`, while the waiter pre-check and transcript-growth post-check preserve manager semantics.
