## Summary

Keep open: current main still lacks the generic webhook auto-response handoff, and this PR provides a focused, proof-backed fix with regression tests and no blocking correctness findings. It still needs maintainer or author refresh because GitHub reports the head as conflicting with current main.

## What This Changes

The PR adds waiter-aware auto-response handling for final Voice Call `call.speech` events received through generic webhooks, exposes `CallManager.hasTranscriptWaiter`, adds regression tests, and updates the changelog.

## Best Possible Solution

Voice Call's generic webhook speech path should share the same guarded auto-response behavior as the streaming transcript path, preserving waiter and replay safeguards.

## Reproduction Assessment

Yes. Source inspection shows Telnyx and Plivo final speech reaches `processParsedEvents`, current main only forwards it to the manager, and the manager records the transcript without invoking the response loop.

## Solution Assessment

Yes. Keeping the handoff in the webhook server is the narrowest maintainable fix because that layer already owns `handleInboundResponse`, while the PR guards waiters, partial transcripts, and replay side effects.
