## Summary

Keep open: the PR targets a real Telnyx/non-streaming voice-call bug and includes useful live Telnyx proof, but the current diff should not merge until the new auto-response path preserves existing webhook idempotency and the required changelog entry is added.

## What This Changes

The branch adds a `CallManager` transcript-waiter probe, wires final non-streaming voice-call webhook speech events to `handleInboundResponse`, and adds webhook regression tests.

## Best Possible Solution

Revise the handoff so non-streaming final speech events trigger auto-response only after the manager accepts the event, preserving waiter and dedupe semantics, then add focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The current-main source path is high-confidence: Telnyx/Plivo final webhook transcripts normalize to `call.speech`, `processParsedEvents` only calls `manager.processEvent`, and the manager records the transcript without invoking the auto-response; the linked report also provides live Telnyx steps and logs.

## Solution Assessment

No, not as written. The requested handoff belongs near the webhook/manager boundary, but the side effect must be gated on an actually consumed speech event rather than a precomputed call lookup before manager dedupe runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve webhook dedupe before auto-responding:** `extensions/voice-call/src/webhook.ts:883-889`
  - body: `pendingAutoResponse` is computed before `manager.processEvent`, but the new code calls `handleInboundResponse` whenever it was set, even if `processEvent` returns early for an already processed dedupe key. After a Gateway restart, the replay cache can be empty while the restored active call still has `processedEventIds`, so the manager skips appending the transcript but this new side effect replies again. Gate the auto-response on a consumed speech event or move the callback into the manager event path.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/voice-call/src/webhook.ts:856`
  - body: This is a user-facing voice-call fix, but the PR changes only manager/webhook code and tests. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing fix changes before merge.
  - confidence: 0.9
