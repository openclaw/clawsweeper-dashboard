## What This Changes

The PR adds a waiter-aware auto-response handoff for final voice-call webhook speech events, exposes `CallManager.hasTranscriptWaiter`, adds webhook regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land this PR or an equivalent voice-call plugin patch after maintainer approval, preserving the waiter, dedupe, and transcript-growth guards.

## Reproduction Assessment

Yes. Source inspection shows Telnyx/Plivo final speech reaches `processParsedEvents`, current main only calls `manager.processEvent`, and the manager records the transcript without invoking the response loop; the PR body also supplies Telnyx after-fix logs.

## Solution Assessment

Yes. The webhook server already owns `handleInboundResponse`, and firing it after manager processing only when a final transcript was actually appended and no waiter is pending is a narrow maintainable fix.
