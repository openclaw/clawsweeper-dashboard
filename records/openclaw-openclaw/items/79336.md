## Summary

Keep open: current main still lacks the non-streaming voice-call auto-response handoff, and the current PR head appears correct after the force-push addressed the earlier phone-number fixture concern.

## What This Changes

The PR adds a waiter-aware webhook handoff that invokes `handleInboundResponse` after final Telnyx/Plivo `call.speech` events append a user transcript, plus regression tests and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent voice-call plugin patch after required CI completes, preserving the waiter and transcript-growth gates plus the focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: Telnyx/Plivo webhook speech becomes `call.speech`, `processParsedEvents` only calls the manager, and the manager records the transcript without entering `handleInboundResponse`.

## Solution Assessment

Yes. Keeping the response handoff in the webhook runtime where `handleInboundResponse` lives is narrow, and the PR preserves manager waiter/replay semantics by checking waiter state before processing and transcript growth after processing.
