## Summary

Keep open: current main still misses the auto-response handoff for final voice-call transcripts delivered through the generic provider webhook path, while this active PR is a focused bug fix with sufficient Telnyx after-fix logs, green checks, and no blocking review findings.

## What This Changes

The PR adds waiter-aware voice-call auto-response handling for final generic webhook `call.speech` events, exposes `CallManager.hasTranscriptWaiter`, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent voice-call plugin patch after maintainer approval and exact-head landing checks, preserving the waiter and transcript-growth safeguards.

## Reproduction Assessment

Yes. Source inspection shows Telnyx/Plivo final speech reaches `processParsedEvents`, current main only forwards it to the manager, and the manager records the transcript without invoking the response loop; the PR body also supplies after-fix Telnyx runtime logs.

## Solution Assessment

Yes. Keeping the response handoff in the webhook server is the narrowest maintainable fix because that layer already owns `handleInboundResponse`, and the PR gates it on waiter state plus actual transcript persistence.
