## What This Changes

The branch adds a waiter-aware webhook auto-response handoff for final non-streaming voice-call transcripts, plus regression tests and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent voice-call plugin patch after maintainer approval and merge gates, preserving the waiter and transcript-growth safeguards.

## Reproduction Assessment

Yes. The webhook runtime already owns `handleInboundResponse`, so adding the handoff there while checking waiter state before manager processing and transcript growth afterward is the narrowest maintainable fix.
