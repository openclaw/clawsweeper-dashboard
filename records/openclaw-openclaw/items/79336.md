## Summary

## What This Changes

The PR exposes `CallManager.hasTranscriptWaiter`, makes final non-streaming voice-call webhook speech events trigger `handleInboundResponse` after a persisted transcript entry, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Yes. Source inspection on current main shows Telnyx/Plivo webhook transcripts become `call.speech`, then `processParsedEvents` only calls the manager and the manager records final speech without invoking `handleInboundResponse`; the related issue and PR body also include live Telnyx logs.

## Solution Assessment

No, not ready as written. The revised functional approach is narrow and preserves waiter/dedupe semantics, but the test fixture must stop committing plausible real phone numbers before this is the best mergeable solution.
