## Summary

Keep open. The report is concrete and affects the latest npm release; current main has plausible post-release mitigations in TUI startup history and session-list hot paths, but there is still no direct proof that they resolve the immediate CPU loop in both Gateway and local modes.

## What This Changes

The issue reports that `openclaw tui`, `openclaw chat`, and `openclaw chat --local` on v2026.4.29 spin at 100%+ CPU immediately after launch instead of becoming interactive.

## Best Possible Solution

Validate both reported launch modes against current main in a real TTY, then either close with shipped fix provenance or add a focused guard around the confirmed startup hot path.

## Reproduction Assessment

Unclear for current main. The issue gives a strong v2026.4.29 reproduction path, but this read-only review did not launch a live current-main TTY and current main has changed since that release.

## Solution Assessment

Unclear. The current bounded history and session-list mitigations are plausible and narrow, but live startup proof is needed before treating them as the complete fix.
