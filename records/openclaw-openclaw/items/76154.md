## Summary

Keep open. The report is concrete and current main has plausible session/history mitigations, including Unreleased changes after `v2026.5.2`, but there is not enough proof that both Gateway and local TUI launch paths no longer hit the reported CPU loop.

## What This Changes

The issue reports that `openclaw tui`, `openclaw chat`, and `openclaw terminal` on `v2026.4.29` enter an immediate CPU busy-loop instead of opening the terminal UI.

## Best Possible Solution

Validate current main in a real TTY for both Gateway and local launch modes; if the loop reproduces, add a focused regression around the confirmed startup/session/history hot path, otherwise close after a shipped fix proves the behavior healthy.
