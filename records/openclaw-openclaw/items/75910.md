## Summary

Keep open. The Codex/OpenAI runtime documentation direction matches current main, but the PR introduces copy-paste config commands that fail under the current `--strict-json` parser, so it should not merge as-is.

## What This Changes

This docs-only PR updates Codex/OpenAI runtime docs and the changelog to recommend `openai/gpt-*` plus `agentRuntime.id: "codex"` with `openai-codex` auth for native Codex subscription use, while keeping `openai-codex/*` as the PI OAuth route.

## Best Possible Solution

Keep the docs direction, but correct the new CLI examples to valid strict JSON or remove `--strict-json` before maintainer review and merge.

## Reproduction Assessment

Yes. The failure path is reproducible from current main by passing the new JSON5 literals to `openclaw config set ... --strict-json`; the parser uses `JSON.parse` and the existing config CLI test suite already covers rejection of JSON5-only syntax.

## Solution Assessment

No, not as written. The conceptual docs update is a maintainable clarification, but the setup commands should use quoted JSON keys such as `'{"enabled":true}'` and `'{"id":"codex","fallback":"none"}'` or avoid strict mode.
