## Summary

Keep this PR open, but the branch head is not merge-ready: it now changes the plugin loader to provide a fake metadata runtime, while the safer documented fix is to keep Copilot's runtime-only registration behind `api.registrationMode` and prove the installed CLI path.

## What This Changes

## Best Possible Solution

Land one canonical Copilot fix that gates runtime-only registration with `api.registrationMode`, keeps full durable state failures visible, adds focused CLI-metadata/full-mode coverage, and includes redacted installed-runtime proof.

## Solution Assessment

No: the branch is a plausible mitigation, but the added fake metadata runtime is the wrong layer unless maintainers explicitly approve a new SDK contract. The narrow fix is to skip Copilot runtime-only registration outside `full` mode and prove both metadata and durable full-mode paths.
