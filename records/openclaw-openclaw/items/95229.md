## What This Changes

The PR adds a non-full registration-mode early return to the Copilot plugin before it opens the durable `sdk-sessions` state store.

## Best Possible Solution

Land one canonical Copilot fix that skips runtime-only registration outside full mode, preserves durable sessions in full runtime, and proves the reported CLI path succeeds after the fix.

## Solution Assessment

Yes. The registration-mode guard is the narrowest maintainable fix shape I found: it follows the public SDK contract and is safer than silently running full Copilot without the durable session store.
