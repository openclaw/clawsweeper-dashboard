## Summary

Keep open: the direction is useful, but this external PR still lacks real behavior proof and the final diff has source-reproducible regressions around global marker redaction and history limiting on an older chat renderer surface.

## What This Changes

The PR hides tool/toolResult/function and tool-call-only assistant records in Web UI chat and strips external-content/security notice markers during message extraction.

## Best Possible Solution

Rework the fix on current build-chat-items/message-extract paths so normal chat hides raw internal tool plumbing while the tool-calls toggle exposes intentional sanitized tool details, with focused regression tests and real UI proof.

## Reproduction Assessment

No high-confidence live reproduction exists for the original Web UI leak because the PR body leaves verification blank and I did not run a browser session. The two blocking patch regressions are source-reproducible from the final PR files.

## Solution Assessment

No. The proposed direction is useful, but the patch globally redacts ordinary message text, applies the render limit before the newly hidden tool records are filtered, and needs to be rebuilt around current main's separate tool-calls model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Limit external-marker stripping to tool payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `processMessageText` now runs `stripExternalContentMarkers` for every role, so normal user or assistant text that literally discusses `SECURITY NOTICE:` or `EXTERNAL_UNTRUSTED_CONTENT` is silently redacted and can collapse to `null`. Scope this cleanup to actual wrapped tool/external payloads instead of all chat text.
  - confidence: 0.86
- **[P2] Apply the render limit after filtering tool messages:** `ui/src/ui/views/chat.ts:1440-1441`
  - body: `historyStart` is computed from raw history length before the new unconditional tool-message filter runs. If the last 100 records contain many tool/toolResult entries, the UI can render far fewer real user/assistant turns than the limit even though older visible turns would fit.
  - confidence: 0.8
