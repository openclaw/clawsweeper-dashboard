## Summary

## What This Changes

The PR changes Web UI chat rendering to unconditionally hide tool/function/tool-call-only messages and strip external-content security markers during message text extraction.

## Best Possible Solution

Rebuild a scoped fix on current `build-chat-items.ts` and `message-extract.ts` that preserves `showToolCalls`, sanitizes only actual tool or wrapped external payloads, adds focused tests, and includes redacted Web UI proof.

## Reproduction Assessment

No high-confidence live reproduction is attached. Source inspection does show the wrapper that emits `SECURITY NOTICE`/external markers and that current Web UI extraction lacks marker cleanup, but the PR branch has no before/after runtime proof.

## Solution Assessment

No. The direction is useful, but this patch is not the best solution because it globally hides tool messages and redacts every role instead of preserving the current `showToolCalls` control and scoping cleanup to actual tool or external payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the tool-call visibility toggle:** `ui/src/ui/views/chat.ts:1435-1439`
  - body: Current main intentionally has a default-on `showToolCalls` setting. This unconditional filter would make the on state ineffective for historical tool results after rebase, so keep the explicit visibility control and only suppress unsafe wrapper or empty plumbing that should never render.
  - confidence: 0.84
- **[P2] Limit marker stripping to tool payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `processMessageText` now runs marker stripping for every role, so ordinary user or assistant text that literally discusses `SECURITY NOTICE` or `EXTERNAL_UNTRUSTED_CONTENT` can be silently redacted or collapsed. Scope this cleanup to actual tool or wrapped external payloads.
  - confidence: 0.86
- **[P2] Filter tool plumbing before applying the history limit:** `ui/src/ui/views/chat.ts:1435-1439`
  - body: The PR computes the history start from raw history before dropping tool messages. In tool-heavy sessions this can render far fewer visible chat turns than the limit allows, so filtering has to happen before the visible history window is selected.
  - confidence: 0.8
