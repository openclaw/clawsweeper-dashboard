## Summary

Keep open. The Web UI cleanup is plausible, but this PR lacks real after-fix proof, conflicts with current main's separate tool-calls visibility model, and introduces source-reproducible rendering/redaction regressions.

## What This Changes

The branch filters tool/toolResult/function and tool-call-only assistant records from Web UI chat and strips external-content/security notice markers during text extraction.

## Best Possible Solution

Rework the current build-chat-items/message-extract paths so normal chat hides internal tool plumbing while the tool-calls toggle exposes intentional sanitized tool details, with focused regression tests and real Web UI proof.

## Reproduction Assessment

No high-confidence live reproduction exists for the original Web UI leak because the PR body leaves verification blank and I did not run a browser session. The two blocking patch regressions are source-reproducible from the PR diff.

## Solution Assessment

No. The proposed direction is useful, but the patch should be rebuilt against current main's showToolCalls model and should scope marker cleanup to actual tool/external payloads instead of all chat text.
