## Summary

Keep open: current main still has the source path for the reported leak, but this PR is not mergeable as-is because the branch has concrete correctness blockers and no after-fix real behavior proof.

## What This Changes

The PR changes Control UI chat rendering to suppress tool/function/tool-result transcript plumbing and strip external-content security markers from extracted message text.

## Best Possible Solution

Land a refreshed fix in the current chat builder/extraction files that filters non-renderable tool plumbing before applying the render limit, preserves live tool cards, scopes marker stripping to confirmed tool-result or wrapped external-content payloads, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass `buildChatItems` a history message that normalizes to `toolResult` with `showToolCalls: true`, and current main does not skip it; `extractText` also lacks marker stripping.

## Solution Assessment

No. The intended fix direction is narrow, but this patch applies the history limit before filtering tool plumbing and globally redacts marker text across roles instead of limiting it to the affected payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Filter tool plumbing before applying the history limit:** `ui/src/ui/views/chat.ts:1439-1441`
  - body: `historyStart` is calculated from raw history before the new unconditional filter. In tool-heavy sessions, dropping tool/toolResult entries after slicing can leave the UI showing far fewer real chat turns than the configured limit even though older user/assistant turns would fit.
  - confidence: 0.88
- **[P2] Limit marker stripping to tool result or external payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `processMessageText` now strips security notices and external-content markers for every role. Normal user or assistant messages that discuss those strings can be silently changed or become empty, so the sanitizer needs a confirmed tool-result or wrapped-external-content boundary.
  - confidence: 0.86
