## Summary

Keep open: the reported Web UI leakage is still source-plausible, but this PR is not mergeable as-is because it lacks after-fix real behavior proof and conflicts with the current `showToolCalls` visibility contract.

## What This Changes

The PR hides tool/function/tool-call-only messages in Web UI chat regardless of thinking visibility and strips external-content/security markers during message text extraction.

## Best Possible Solution

Rebuild a narrow current-main fix that preserves `showToolCalls`, scopes external-marker cleanup to actual wrapped tool/external payloads, adds focused tests, and includes redacted Web UI proof.

## Reproduction Assessment

No high-confidence live reproduction is attached. Source inspection shows `wrapExternalContent` can emit the reported markers and current Web UI extraction does not strip them, but the PR has no before/after real Web UI proof.

## Solution Assessment

No. The direction is useful, but this patch globally hides tool messages and globally redacts marker strings instead of preserving the current `showToolCalls` control, scoping cleanup, and applying filtering before history windowing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the tool-call visibility toggle:** `ui/src/ui/views/chat.ts:1435-1439`
  - body: Current main intentionally has a default-on `showToolCalls` control. This unconditional filter would make the on state ineffective after rebase, so keep the explicit visibility control and only suppress unsafe wrapper or empty plumbing that should never render.
  - confidence: 0.84
- **[P2] Scope marker stripping to wrapped tool payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `processMessageText` runs marker stripping for every role, so ordinary user or assistant text that literally discusses `SECURITY NOTICE` or `EXTERNAL_UNTRUSTED_CONTENT` can be silently redacted or collapsed. Limit this cleanup to actual tool-result or clearly wrapped external payload paths.
  - confidence: 0.86
- **[P2] Filter hidden tool plumbing before history windowing:** `ui/src/ui/views/chat.ts:1435-1439`
  - body: The PR computes the history window from raw history before dropping tool messages. Tool-heavy sessions can then render far fewer visible turns than the limit allows, so filtering must happen before selecting the visible history window.
  - confidence: 0.8
