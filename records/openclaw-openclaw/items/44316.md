  - body: Current main has a default-on `showToolCalls` control. This unconditional filter would make the enabled state hide tool/function/toolResult messages anyway after rebase, so keep the explicit visibility control and only suppress unsafe wrapper or empty plumbing that should never render.
  - confidence: 0.84

  - confidence: 0.86

  - body: The PR computes the history window from raw history before dropping tool messages. Tool-heavy sessions can then render far fewer visible turns than the limit allows, so filtering must happen before selecting the visible history window.
  - confidence: 0.8
