## Summary

Keep open: the underlying Web UI leak/sanitization concern is still plausible on current main, but this branch is not a safe merge because it conflicts with the current default-on tool-call visibility model, has source-reproducible regressions, and still lacks real after-fix Web UI proof.

## What This Changes

The PR adds Web UI filtering for tool/toolResult/function and tool-call-only assistant messages, plus stripping of external-content security markers from displayed chat text.

## Best Possible Solution

Rebuild a narrow current-main fix in build-chat-items.ts and message-extract.ts that preserves showToolCalls, scopes marker cleanup to actual tool/external payloads, applies limits after filtering, adds focused tests, and includes real Web UI proof.

## Reproduction Assessment

No high-confidence live Web UI reproduction/proof is attached. Source inspection is enough to see the original marker/tool-message leak path and the two patch regressions, but the after-fix behavior has not been demonstrated in a real setup.

## Solution Assessment

No. The direction is useful, but the branch should be rebuilt against current main's showToolCalls model and should avoid globally redacting ordinary chat text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Limit marker stripping to tool payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: processMessageText now calls stripExternalContentMarkers for every role, so ordinary user or assistant text that literally discusses SECURITY NOTICE or EXTERNAL_UNTRUSTED_CONTENT is silently redacted and can collapse to null. Scope this cleanup to actual tool/external payloads so normal chat, search, and export keep literal text.
  - confidence: 0.86

  - body: historyStart is computed from raw history before the new unconditional tool-message filter. In tool-heavy sessions, the UI can show far fewer than the 100 visible chat turns even though older user/assistant turns would fit, so filter first and then apply the render limit.
  - confidence: 0.8
