## Summary

Keep open: the PR addresses a plausible Web UI leak, but the current patch is not mergeable as-is because it conflicts with the current `showToolCalls` contract, over-redacts marker strings, filters after history windowing, and lacks after-fix real behavior proof.

## What This Changes

The branch changes Web UI chat rendering to hide tool/function/tool-call-only messages and adds `SECURITY NOTICE` / `EXTERNAL_UNTRUSTED_CONTENT` stripping in message text extraction.

## Best Possible Solution

Rebuild a narrow current-main fix that preserves `showToolCalls`, hides only unsafe or empty tool plumbing, scopes marker cleanup to actual wrapped external/tool payloads, filters before history windowing, adds focused tests, and includes redacted Web UI proof.

## Reproduction Assessment

No high-confidence live reproduction is attached or run in this read-only review. Source inspection shows `wrapExternalContent` can emit the reported markers and current `message-extract.ts` does not strip them, but the visible Web UI behavior still needs real proof.

## Solution Assessment

No. The direction is useful, but the current patch is not the best fix because it bypasses the current `showToolCalls` policy, redacts marker-like text for every role, and filters after selecting the history window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the tool-call visibility toggle:** `ui/src/ui/views/chat.ts:1435-1439`
  - body: Current main intentionally has a default-on `showToolCalls` control. This unconditional filter would make the on state ineffective after rebase, so keep the explicit visibility control and only suppress unsafe wrapper or empty plumbing that should never render.
  - confidence: 0.84
- **[P2] Scope marker stripping to wrapped tool payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `processMessageText` runs marker stripping for every role, so ordinary user or assistant text that literally discusses `SECURITY NOTICE` or `EXTERNAL_UNTRUSTED_CONTENT` can be silently redacted or collapsed. Limit this cleanup to actual tool-result or clearly wrapped external payload paths.
  - confidence: 0.86
- **[P2] Filter hidden tool plumbing before history windowing:** `ui/src/ui/views/chat.ts:1435-1439`
  - body: The PR computes the history window from raw history before dropping tool messages. Tool-heavy sessions can then render far fewer visible turns than the limit allows, so filtering must happen before selecting the visible history window.
  - confidence: 0.8
