## Summary

Keep open. Current main still has the reported Control UI leakage path, and the PR is useful, but it is not ready to merge because it targets an older chat-builder location and still has concrete correctness/security-scope issues in the proposed patch.

## What This Changes

The PR modifies UI message extraction and the chat view builder to hide tool/function/tool-result history, filter tool-call-only assistant turns, and strip SECURITY NOTICE / EXTERNAL_UNTRUSTED_CONTENT markers from extracted text.

## Best Possible Solution

Rebase the useful behavior onto `ui/src/ui/chat/build-chat-items.ts` and `ui/src/ui/chat/message-extract.ts`, hide raw tool plumbing from history regardless of the live tool-card toggle, preserve mixed-content assistant turns and intentional live tool cards, and add focused tests for filtering, marker handling, and history-limit behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: build chat items from a history containing tool-shaped messages with `showToolCalls: true`; current main only filters normalized `toolResult` when that flag is false, and UI extraction has no external-marker stripping path.

## Solution Assessment

No, not as currently proposed. The direction is appropriate, but the patch should be rebased to the current builder and should filter before applying the history limit while scoping marker redaction to confirmed tool-result or wrapped external-content payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Apply the history limit after filtering tool plumbing:** `ui/src/ui/views/chat.ts:1440-1441`
  - body: `historyStart` is still calculated from the raw history length, but this patch now drops tool/function entries after slicing. In long tool-heavy sessions the UI can render far fewer than `CHAT_HISTORY_RENDER_LIMIT` real user/assistant turns even though older visible turns would fit; filter non-renderable plumbing before applying the display limit or count visible messages when limiting.
  - confidence: 0.88
- **[P2] Scope marker stripping to tool-result content:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `processMessageText` now calls `stripExternalContentMarkers` for every role, so normal user or assistant messages that discuss `SECURITY NOTICE:` or `EXTERNAL_UNTRUSTED_CONTENT` are silently changed and may disappear through `|| null`. Limit this sanitizer to confirmed tool-result/external-content payloads or another explicit trusted marker path.
  - confidence: 0.86
