## Summary

Keep open: the reported Web UI leak is source-plausible, but this stale branch should not merge because it bypasses the current `showToolCalls` visibility contract, redacts ordinary chat text globally, and filters after raw history windowing.

## What This Changes

The PR adds Web UI filtering for tool/function/tool-result chat messages and strips `SECURITY NOTICE` plus `EXTERNAL_UNTRUSTED_CONTENT` markers from extracted chat text.

## Best Possible Solution

Rebuild the fix on current `build-chat-items.ts` and `message-extract.ts` so unsafe wrapper plumbing is hidden without defeating `showToolCalls`, with focused tests and redacted Web UI proof.

## Reproduction Assessment

No live high-confidence reproduction is attached. Source inspection supports the path: external content can carry `SECURITY NOTICE` markers, and current chat extraction does not strip them before display when tool output is visible.

## Solution Assessment

No. The direction is useful, but this patch is not the best current solution because it overrides an intentional tool-call toggle, redacts all roles too broadly, and applies filtering after the history window is chosen.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the tool-call visibility toggle:** `ui/src/ui/views/chat.ts:1440`
  - body: Current main has a default-on `showToolCalls` control for showing tool calls and results. This unconditional filter makes the enabled state ineffective after rebase, so preserve the toggle and only suppress plumbing that must never render.
  - confidence: 0.86
- **[P2] Scope marker stripping to wrapped tool payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `extractText` is used for ordinary user and assistant text too. Running the new marker stripper for every role can silently remove legitimate messages that discuss `SECURITY NOTICE` or `EXTERNAL_UNTRUSTED_CONTENT`, so limit cleanup to actual wrapped external/tool payloads.
  - confidence: 0.86
- **[P2] Filter hidden plumbing before history windowing:** `ui/src/ui/views/chat.ts:1440`
  - body: The branch computes the render window from raw history and only then drops tool messages. In tool-heavy tails this can show far fewer real chat turns than the limit allows, so hidden-message filtering needs to feed the windowing calculation.
  - confidence: 0.8
