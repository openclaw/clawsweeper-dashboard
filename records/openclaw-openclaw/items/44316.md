## Summary

The PR targets a plausible Web UI leakage path, but the branch is not currently merge-ready: it conflicts with current main, bypasses the shipped showToolCalls contract, globally redacts literal marker text from ordinary messages, applies the history limit before filtering, and still has no after-fix Web UI proof.

## What This Changes

The PR changes Web UI chat rendering to hide tool/toolResult/function and tool-call-only assistant messages unconditionally and strips external-content security markers from extracted chat text.

## Best Possible Solution

Rebuild the fix against current main in build-chat-items.ts and message-extract.ts, preserve the explicit showToolCalls control, scope marker cleanup to actual tool/external payloads, apply limits after filtering, add focused tests, and include real Web UI proof.

## Solution Assessment

No. The direction is useful, but the current patch should be rebuilt around current main's showToolCalls model and scoped sanitizer behavior instead of globally hiding/redacting chat text.
