## Summary

Keep open: the reported Web UI leakage has a plausible source path, but this PR is stale against current main, lacks after-fix real behavior proof, and has blocking correctness issues around the current `showToolCalls` policy and overly broad marker redaction.

## What This Changes

The branch adds Web UI filtering for tool/function/tool-result messages and strips external-content/security-notice markers during message text extraction.

## Best Possible Solution

Rebuild a current-main fix in `ui/src/ui/chat/build-chat-items.ts` and `ui/src/ui/chat/message-extract.ts` that preserves `showToolCalls`, suppresses only unsafe or empty tool plumbing, scopes marker cleanup to actual wrapped payloads, filters before history windowing, and adds focused tests plus redacted Web UI proof.

## Reproduction Assessment

No high-confidence live reproduction is attached or run in this read-only review. Source inspection shows `wrapExternalContent` can emit the reported markers and current `message-extract.ts` does not strip them, but visible Web UI behavior still needs real proof.

## Solution Assessment

No. The direction is useful, but this branch is not the best fix because it bypasses the current `showToolCalls` policy, redacts marker-like text for every role, and filters after selecting the history window.
