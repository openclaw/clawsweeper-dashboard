## Summary

Keep this PR open because current main still has the reported Control UI leak path, but the proposed patch is not merge-ready. The direction is useful, and prior review discussion already resolved the initial regex-anchor and assistant-casing concerns, but the remaining diff can reduce visible history in tool-heavy sessions and over-applies security-marker redaction to normal chat text.

## What This Changes

The PR changes Control UI chat extraction and history building to always hide tool/function/tool-result messages, hide tool-call-only assistant turns, and strip external-content security notices from extracted text.

## Best Possible Solution

Port the useful behavior onto the current chat builder and extraction modules: hide raw tool plumbing from history regardless of the live tool-card toggle, preserve intentional live tool cards and mixed-content assistant turns, apply the history render limit after filtering non-renderable entries, and narrow marker stripping to confirmed tool-result or wrapped external-content payloads with focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: pass `buildChatItems` a history entry that normalizes to `toolResult` with `showToolCalls: true`; current main does not skip it, and `extractText` has no external-marker stripping path.

## Solution Assessment

No, not as currently proposed. The fix direction is right, but the patch should be rebased to `ui/src/ui/chat/build-chat-items.ts`, filter before applying the display limit, and scope marker redaction to tool-result or explicitly wrapped external-content paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Apply the history limit after filtering tool plumbing:** `ui/src/ui/views/chat.ts:1440-1441`
  - body: `historyStart` is calculated from the raw history length, but this patch drops tool/function entries after slicing. In tool-heavy sessions the UI can render far fewer than `CHAT_HISTORY_RENDER_LIMIT` real user/assistant turns even though older visible turns would fit; filter non-renderable plumbing before applying the display limit or count visible messages when limiting.
  - confidence: 0.88
- **[P2] Scope marker stripping to tool-result content:** `ui/src/ui/chat/message-extract.ts:42`
  - body: `processMessageText` now strips `SECURITY NOTICE` and `EXTERNAL_UNTRUSTED_CONTENT` markers for every role, so normal user or assistant messages that discuss those strings can be silently changed or disappear. Limit this sanitizer to confirmed tool-result/external-content payloads or another explicit trusted wrapper path.
  - confidence: 0.86
