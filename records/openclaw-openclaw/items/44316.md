## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No. The direction is useful, but this patch globally redacts ordinary message text, slices history before filtering the newly hidden tool records, and needs to be rebuilt against the current split chat renderer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Limit external-marker stripping to tool payloads:** `ui/src/ui/chat/message-extract.ts:42`
  - body: This now runs stripExternalContentMarkers for every role, so a normal user or assistant message that literally discusses `SECURITY NOTICE:` or `<<<EXTERNAL_UNTRUSTED_CONTENT...>>>` is silently redacted and can become an empty bubble. Scope this defense-in-depth cleanup to actual wrapped tool/external payloads instead of all chat text.
  - confidence: 0.86
- **[P2] Apply the render limit after filtering tool messages:** `ui/src/ui/views/chat.ts:1406`
  - body: `historyStart` is computed before the new unconditional tool-message filter runs. If the last 100 raw records contain many tool/toolResult entries, the UI will render far fewer real user/assistant turns than the limit even though older visible turns would fit; filter or count renderable history before slicing.
  - confidence: 0.8
