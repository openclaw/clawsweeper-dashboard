## Summary

Keep open: current main still has the underlying Control UI leak path, and this PR is useful, but the patch is not correct as-is because it filters after the raw history limit and applies external-marker redaction too broadly.

## What This Changes

The PR changes Control UI chat rendering to hide tool/function/tool-result transcript plumbing and strip external-content security markers from extracted message text.

## Best Possible Solution

Land a refreshed patch that preserves live tool cards, filters transcript plumbing before the render limit, scopes marker stripping to confirmed tool-result or wrapped external-content payloads, and adds focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass `buildChatItems` a history message that normalizes to `toolResult` with `showToolCalls: true`, and current main does not skip it; `extractText` also lacks marker stripping.

## Solution Assessment

No. The intended behavior is the right narrow fix, but the current patch should be ported to the current chat builder, filter before limiting history, and avoid global redaction of ordinary chat text.

## Review Findings

  - body: `historyStart` is still calculated from raw history before the new unconditional filter.   - confidence: 0.88
