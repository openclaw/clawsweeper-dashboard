The PR changes the attempt-execution session-file guard to track assistant messages after the latest user record and adds a regression test for a fresh unanswered user after an earlier assistant response.

## Best Possible Solution

Land or replace this with a tail-aware assistant-after-latest-user guard and focused regression coverage, keep the retry-prompt hook tracked separately, and require redacted real fallback retry proof before merge.
