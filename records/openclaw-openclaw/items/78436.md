## What This Changes

The PR changes the attempt-execution session-history helper and tests so fallback retry history detection tracks whether an assistant message appears after the latest user message.

## Best Possible Solution

Land a bounded assistant-after-latest-user scan on top of current scanJsonlFile(), and leave the retry-prompt hook or override decision to the canonical open issue.

## Reproduction Assessment

Yes. Source inspection of current main shows scanJsonlFile() returns true on the first assistant message inside the capped scan, so a later fresh unanswered user turn in that inspected window is still misclassified as answered.

## Solution Assessment

No. The latest-user semantics are the right direction, but the best fix must preserve current main's file-existence/symlink contract and bounded transcript IO.
