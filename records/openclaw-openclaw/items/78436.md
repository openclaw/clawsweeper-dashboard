## Summary

Keep this PR open. It targets a real source-provable fallback retry guard bug, but the patch is not merge-ready because the new stateful scan still decides from the first 500 JSONL records, the required changelog entry is missing, and the external PR provides only unit-test proof rather than real after-fix behavior proof.

## What This Changes

The PR changes the attempt-execution session transcript guard to track assistant messages after the latest user role, preserves top-level Claude CLI user/assistant records, and adds a regression test for an earlier assistant followed by a fresh unanswered user message.

## Best Possible Solution

Land a tail-aware latest-user guard with focused regression coverage and a changelog entry, then keep or split the linked issue for the retry-prompt override discussion.

## Reproduction Assessment

Yes. Source inspection of current main shows a user -> assistant -> fresh user JSONL transcript would return `true` at the earlier assistant record; I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is right, but the implementation should inspect the latest/tail records rather than returning state from the first 500 records, add the required changelog entry, and avoid closing the broader linked issue until the retry-prompt override is handled.
