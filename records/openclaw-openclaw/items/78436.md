## What This Changes

The PR changes the attempt-execution session transcript guard and adds a regression test for an earlier assistant response followed by a fresh unanswered user message.

## Best Possible Solution

Land or replace this with a tail-aware latest-user guard plus focused regression coverage and proof, then keep or split the retry-prompt hook request from https://github.com/openclaw/openclaw/issues/69086.

## Reproduction Assessment

Yes, by source inspection: current main returns true on any assistant message in the inspected transcript, so a user -> assistant -> fresh unanswered user file is misclassified. I did not run tests because this review is read-only and must not create artifacts.

## Solution Assessment

No. The direction is right, but the PR should inspect the latest/tail records or fail closed when the cap is hit, and it should not close the broader linked hook/override request as part of this narrow guard fix.
