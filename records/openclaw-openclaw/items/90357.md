## Summary

Keep open: the PR fixes the reported alias happy path, but as written it can reinterpret existing bare compaction model overrides when the same string is also configured as an alias, so it needs a compatibility repair before merge.

## What This Changes

The PR changes embedded auto-compaction target resolution to expand bare `agents.defaults.compaction.model` aliases through shared model-selection helpers and adds resolver regression tests.

## Best Possible Solution

Preserve existing bare model-only override precedence for ambiguous values, resolve configured aliases only when they do not collide with current model-id semantics, and add collision regression coverage before merge.

## Reproduction Assessment

Yes, from source: current main and `v2026.6.1` forward bare compaction model overrides unchanged, while the linked issue provides a sanitized alias config and log for the failing alias case. I did not run a live OpenAI compaction turn during this read-only review.

## Solution Assessment

No. Reusing the shared alias resolver is a plausible fix location, but this implementation is not the best fix because alias-first resolution can override existing bare model-only compaction values; the safer fix preserves bare-model precedence for collisions.
