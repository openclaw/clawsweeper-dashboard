## Summary

## What This Changes

The PR resolves `agents.defaults.compaction.model` through the shared model alias resolver before embedded compaction dispatch and adds regression tests for alias and provider-qualified overrides.

## Best Possible Solution

Use the shared resolver only after maintainers choose the collision semantics, then land with a regression test and docs or release-note wording that makes ambiguous bare overrides explicit.

Yes, source inspection gives a high-confidence path: current main forwards a no-slash `compaction.model` override as the literal model, so alias `gpt54mini` reaches dispatch as `openai/gpt54mini`. I did not run live auto-compaction in this read-only review.

## Solution Assessment

Unclear: reusing the shared alias resolver is the right owner-boundary shape, but unconditional alias-first behavior needs maintainer acceptance because existing bare model-only overrides are supported.
