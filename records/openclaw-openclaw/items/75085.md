## Summary

Keep open for maintainer review. Current main still has the compaction-summary-specific bypass the PR describes, and the PR appears to address it with focused runtime plumbing, shared payload patching, regression coverage, and a bounded dependency patch; I found no blocking functional or security finding in the proposed diff.

## What This Changes

The PR threads OpenAI-compatible extra payload params and mapped thinking level into safeguard compaction summary calls, refactors chat-template/extra-body payload mutation into a shared helper, adds tests and changelog coverage, and pins a local `@mariozechner/pi-coding-agent` patch exposing an optional `generateSummary()` payload hook.

## Best Possible Solution

Land this contributor PR after maintainer review and CI/build completion, keeping the fix in the agents compaction/runtime path and keeping the dependency patch limited to the optional summary `onPayload` hook until upstream exposes it directly.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. This is the narrowest maintainable direction I found: reuse the same resolved provider params and thinking mapping used by normal turns, centralize OpenAI-completions payload mutation, and expose only the missing optional Pi summary hook instead of duplicating summary request construction in OpenClaw.
