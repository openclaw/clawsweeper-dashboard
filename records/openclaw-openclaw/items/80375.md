## Summary

Keep open: the performance direction is valid, but the latest PR head still removes compatibility exports from a public SDK subpath, so it should not merge as-is under normal maintainer review.

## What This Changes

The branch adds `markAuthProfileSuccess` to combine last-good and usage-stat auth profile updates, switches embedded-run success handling to that helper, updates auth-profile tests, and adds a changelog entry.

## Best Possible Solution

Keep the single locked success helper, but preserve deprecated `markAuthProfileGood` and `markAuthProfileUsed` compatibility exports or wrappers on the public SDK barrel until a major-version removal.

## Reproduction Assessment

Yes for the duplicate-write behavior: current main calls `markAuthProfileGood` and then `markAuthProfileUsed` after successful embedded runs. I did not live-measure the latency impact under the read-only review constraints.

## Solution Assessment

No as currently patched: consolidating the two writes is a narrow maintainable fix, but removing public SDK exports from `agent-runtime` is not. Keep deprecated compatibility exports while using `markAuthProfileSuccess` internally.
