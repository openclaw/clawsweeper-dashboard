## Risks / Open Questions

- A delayed startup auth warm can run after a hot config reload and republish auth availability for `cfgAtStart`, leaving model-listing/auth checks stale until another clear or warm happens.
- The delayed callback also installs an auth-profile failure hook that closes over startup config, so later profile failures can rewarm the wrong config after an early reload.
- The PR has focused test proof only; it has not demonstrated the reported live vLLM/gateway startup path or early `/tools/invoke` acceptance in a real run.
- The intended one-second delay means immediate model-listing or auth-status calls may fall back to slower runtime auth discovery during that window, which is probably acceptable but still needs upgrade-path proof.
