## Summary

Current main still collapses filesystem enumeration failures into empty listings that destructive memory-core consumers treat as authoritative. The PR consistently carries an explicit scan result through session and memory pruning, reindex, export reconciliation, dirty-state retry, and targeted reconcile paths, with focused coverage and no confirmed patch defect; it should remain open for explicit maintainer approval of the new public plugin SDK contracts and final API/build gates.

## What This Changes

Adds authoritative scan-result APIs and uses them to prevent failed session or memory-file enumerations from causing destructive index pruning, empty reindex replacement, export reconciliation deletion, or false-clean sync state.

## Best Possible Solution

Land the authoritative-scan result and its complete destructive-consumer propagation after memory/plugin-SDK owners approve the permanent API boundary and final build plus SDK API/surface checks pass; track partial subtree failure reporting separately in the fs-safe contract.
