## Summary

Keep open: this maintainer-authored, protected-label PR is still needed because current main does not preserve scan-failure authority before destructive memory/session cleanup. The overall fix shape is right, but the latest head still treats any missing sessions path as an authoritative empty scan, which leaves a remaining session-state wipe path before merge.

## What This Changes

Adds authoritative `{ ok, files }` scan APIs for memory/session enumeration and uses them to skip memory index pruning, reindex completion, dirty-state clearing, and QMD export reconciliation after failed scans.

## Best Possible Solution

Land the authoritative scan guard after missing-tree failures remain non-authoritative for destructive consumers, and after memory/plugin-SDK owners accept the public scan API or keep it internal.
