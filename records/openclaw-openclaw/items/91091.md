## Summary

This is a valid session-state bug fix and the authoritative-scan model is coherent across destructive memory/session cleanup, reindex rollback, dirty-state retry, and QMD export reconciliation. It should remain open because the branch adds supported Plugin SDK symbols without the required API baseline/docs contract update or an approved narrower boundary, and it still lacks after-fix real fault-injection proof.

## What This Changes

The PR distinguishes failed filesystem scans from authoritative empty listings, skips destructive memory/session pruning and export reconciliation after failed scans, aborts unsafe full reindexes, preserves dirty state for retry, and adds focused regression coverage.

## Best Possible Solution

Keep the explicit authoritative-scan result for destructive consumers, but either route it through the narrowest memory-host seam or fully baseline and document the intended public Plugin SDK contract; before merge, demonstrate a real failure-then-recovery sync with existing rows preserved and later reconciled.
