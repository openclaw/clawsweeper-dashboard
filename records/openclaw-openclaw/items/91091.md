## Summary

The final head has no discrete correctness or security finding: it separates failed from authoritative-empty scans across destructive memory maintenance, preserves retry state, aborts failed reindexes before publication, and has strong exact-head CI. ## What This Changes

Adds authoritative session and memory-file scan results, prevents failed enumerations from driving destructive index/export/reindex cleanup, and changes session delta syncs to targeted updates with periodic full reconciliation.

## Best Possible Solution

Land the complete authoritative-scan propagation after memory and plugin-SDK owners explicitly approve the public result contracts and the 15-minute reconciliation policy, while tracking subtree failure reporting as a separate `@openclaw/fs-safe` contract improvement.

## Reproduction Assessment

## Solution Assessment

Yes for the implementation: an explicit authoritative-scan result propagated through every destructive and clean-state path is the narrowest maintainable fix. Maintainer judgment remains necessary only for whether those helpers are public SDK contracts and whether the bundled 15-minute reconciliation policy should land in the same PR.
