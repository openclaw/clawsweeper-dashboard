## Summary

Current main and the v2026.6.8 release now implement the managed-npm host-peer contract for @openclaw/codex: packaging publishes the openclaw peer metadata, install/update/doctor relink the package-local host peer, and regression coverage exercises the Codex missing-peer update path.

## What This Changes

The issue reports that @openclaw/codex could import the host openclaw package at runtime while managed npm installs did not always provide a resolvable openclaw peer link.

## Best Possible Solution

Keep the shipped host-peer architecture: publish @openclaw/codex with the openclaw peer contract and rely on install, update, and doctor to relink the package-local host peer.

## Reproduction Assessment

Yes. The issue comments give concrete 2026.5.18 and 2026.5.19 install/update paths, and source confirms @openclaw/codex runtime code imports openclaw/plugin-sdk/* so missing host-peer resolution caused ERR_MODULE_NOT_FOUND.

## Solution Assessment

Yes. The shipped fix keeps openclaw as a host peer and hardens the package/install/update/doctor repair points, which matches the plugin boundary better than installing a separate stale host package copy.
