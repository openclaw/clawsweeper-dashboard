## Summary

## What This Changes

Ensure `@openclaw/codex` installs or upgrades with the runtime dependency or host peer link needed for `openclaw/plugin-sdk/*` imports so Codex model attempts do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Keep the host-peer architecture or intentionally change it with docs, lockfile, and upgrade proof; either way, managed install, update, and doctor should leave Codex with a working package-local host link before the harness starts.

## Reproduction Assessment

Yes for the shipped 2026.5.18 and 2026.5.19 release behavior: comments reproduce `openclaw plugins update --all` leaving Codex without a resolvable host peer link, and current source confirms the runtime import and peer-link dependency. I did not establish a live current-main repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the package contract. The linked direct-dependency PR proves standalone npm import behavior, but the documented OpenClaw path is managed host-peer relinking, so the safer fix is to repair or deliberately replace that contract with upgrade proof.
