Ensure `@openclaw/codex` installs and updates leave a resolvable `openclaw` peer dependency so the Codex harness does not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Make OpenClaw-managed install/update recovery consistently relink or fail on Codex's package-local `openclaw` peer after npm mutations, with a regression that matches the 2026.5.18 sidecar state; keep the downgrade policy work in https://github.com/openclaw/openclaw/issues/84256.

## Solution Assessment

No, the manual `npm install openclaw@2026.5.18` workaround is not the right product fix. The maintainable solution is to make the managed update/install repair path leave the package-local peer link valid or fail with an actionable repair message.
