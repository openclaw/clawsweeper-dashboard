## Summary

Keep open. This PR targets a real source-reproducible timeout bug, but the patch only changes the generic gateway fallback while the reported CLI media path still forwards an explicit 10000 ms timeout from message-action dispatch, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes `src/gateway/call.ts` so calls without an explicit gateway timeout resolve the shared 15000 ms preauth handshake default.

## Best Possible Solution

Align gateway-executed message-action calls with the shared preauth default while preserving explicit caller overrides, then update focused gateway and message-action regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the CLI message command passes no timeout override, `resolveGatewayActionOptions` fills `10_000`, and gateway-mode plugin sends use `message.action` with that explicit timeout.

## Solution Assessment

No. The patch raises the fallback for calls without a timeout, but the reported CLI media path already sends an explicit `10_000` ms timeout, so the fix needs to align the message-action timeout resolver or stop passing that hard-coded default.
