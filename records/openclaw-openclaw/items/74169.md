## Summary

Keep open. The PR targets a real gateway timeout bug, but the current patch only changes the generic no-timeout fallback and does not cover the reported CLI media path that already passes an explicit 10000 ms message-action timeout; it also lacks required after-fix real behavior proof.

## What This Changes

The PR modifies `src/gateway/call.ts` so gateway calls without an explicit timeout resolve the shared 15000 ms preauth handshake default instead of falling back to 10000 ms.

## Best Possible Solution

Align gateway-executed message-action calls with the shared preauth timeout policy while preserving explicit caller overrides, update focused gateway/message-action regression coverage, and require redacted real-send proof before merge.

## Reproduction Assessment

Yes. Source inspection traces the reported CLI media send into `messageCommand`, then `resolveGatewayActionOptions`, which supplies an explicit `10_000` ms timeout for the `message.action` gateway RPC.

## Solution Assessment

No. The patch raises the fallback for calls with no timeout, but the reported path already supplies `timeoutMs: 10_000`, so the fix must cover the message-action timeout resolver or stop passing that hard-coded default.
