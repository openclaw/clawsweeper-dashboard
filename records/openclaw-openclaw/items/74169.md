## Summary

Keep open. The PR targets a real timeout regression, but the linked CLI media-send path still supplies an explicit 10000ms timeout before it reaches the gateway call fallback changed by this patch, so the observable WeCom large-file failure can remain.

## What This Changes

The PR modifies `src/gateway/call.ts` so gateway calls without an explicit timeout always resolve the shared preauth handshake default of 15000ms instead of falling back to 10000ms.

## Best Possible Solution

Align the gateway-executed message-action timeout with the shared preauth timeout default while preserving explicit caller overrides, simplify the generic fallback expression, and add focused regression coverage for both no-config/no-env gateway calls and plugin-dispatched CLI media sends.

## Reproduction Assessment

Yes. The linked report provides a concrete `openclaw message send --channel wecom --media <10MB file>` failure, and current main shows a deterministic static reproduction path where `messageCommand` reaches `callGatewayMessageAction` with an explicit `timeoutMs: 10_000`.

## Solution Assessment

No. This PR changes only the fallback used when no timeout is passed to `callGatewayLeastPrivilege`, but the reported message-action path passes an explicit 10000ms timeout from `src/infra/outbound/message-action-runner.ts`.

## Review Findings
