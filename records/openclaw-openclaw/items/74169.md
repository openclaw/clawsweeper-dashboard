## Summary

Keep open: the PR improves the generic gateway-call fallback, but it does not fix the linked CLI media-send path because gateway-executed message actions still pass an explicit 10000ms timeout before reaching that fallback.

## What This Changes

The PR changes `src/gateway/call.ts` so gateway calls without an explicit timeout resolve the shared 15000ms preauth handshake default instead of the older 10000ms fallback.

## Best Possible Solution

Align gateway-executed message-action calls with the shared preauth default while preserving explicit caller overrides, simplify the redundant PR ternary, update focused tests, and add the required changelog entry.

## Reproduction Assessment

Yes. The linked report gives a concrete CLI media-send command and current main statically traces that route to gateway-executed plugin dispatch with an explicit `10_000` timeout.

## Solution Assessment

No. The PR changes only the fallback used when `callGatewayLeastPrivilege` receives no timeout, while the reported route passes an explicit `10_000` timeout before that fallback can apply.
