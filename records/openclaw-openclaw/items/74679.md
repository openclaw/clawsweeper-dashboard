## Summary

Keep open. Current main still has the reported source-level route: a raw `x-openclaw-session-key` wins before agent-scoped session key construction, while the Chat Completions command path does not pin the model-selected agent; however the public docs and the related stable-session PR treat that header as a full session-routing override, so this needs a maintainer API-precedence decision rather than cleanup closure.

## What This Changes

The issue asks for `/v1/chat/completions` to keep `model: "openclaw/<agentId>"` agent routing when `x-openclaw-session-key` is also supplied.

## Best Possible Solution

Define one documented OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes, at source level. Send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`; current main passes the raw key into command routing, where non-agent keys resolve to the default agent.

## Solution Assessment

Unclear. Making the model-selected agent always win matches this report, but it conflicts with the documented full-override semantics and the related stable-session PR's stated priority order.
