## Summary

Keep open. Current main still gives raw `x-openclaw-session-key` precedence over the agent selected by `model`, but the documented contract calls that header a full session-routing override, so this needs an explicit API precedence decision rather than conservative cleanup.

## What This Changes

Request that `/v1/chat/completions` keep `model: "openclaw/<agentId>"` agent routing when `x-openclaw-session-key` is supplied as the session identifier.

## Best Possible Solution

Define one precedence contract for raw session-key overrides, agent-scoped session hints, and model-based agent routing across Chat Completions and Responses, then update docs and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only the raw key into command routing, where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making `model` always win would match this report, but it would revise the documented full-override semantics and overlap #69060, so the maintainable fix needs an explicit precedence decision first.
