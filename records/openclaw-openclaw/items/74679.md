## Summary

Keep open. Current main still lets raw `x-openclaw-session-key` take precedence over model-derived agent scoping, while the docs and related #69060 preserve that full-override contract, so this needs an explicit gateway API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is supplied as the session identifier.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main resolves the agent from the model but lets the command derive the effective agent from the raw key.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override semantics and intersects #69060, so the safer path is an explicit maintainer precedence decision first.
