## Summary

Keep open. Current main still has a source-reproducible path where a raw `x-openclaw-session-key` can make the selected `model: "openclaw/<agentId>"` diverge from the session-derived runtime agent, but the docs also describe that header as a full session-routing override, so this needs an explicit API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `x-openclaw-session-key` on `/v1/chat/completions` to preserve `model: "openclaw/<agentId>"` agent routing instead of routing by the raw session key.

## Best Possible Solution

Define and document one precedence rule for explicit session headers versus agent model routing across Chat Completions and Responses, then update the shared resolver and regression tests to match that contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only that raw key into command routing, where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override contract; a safer maintainer decision is whether to keep full override, add/land an agent-scoped stable header like #69060, or reject mismatched raw keys explicitly.
