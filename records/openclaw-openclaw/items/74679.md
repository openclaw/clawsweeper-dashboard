## Summary

Keep open. Current main still resolves a raw session-key header independently from model-based agent selection, while the docs and related open PR preserve the header as a full routing override, so this needs a maintainer API-contract decision rather than cleanup closure.

## What This Changes

The issue asks for `/v1/chat/completions` to keep routing by `model: "openclaw/<agentId>"` when `x-openclaw-session-key` is also supplied.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, OpenResponses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. A source-level reproduction is a `/v1/chat/completions` request with `model: "openclaw/beta"` and raw `x-openclaw-session-key: my-session-key`: current main resolves `agentId` as `beta`, passes the raw key without an agent override, and downstream command setup derives the agent from that key.

## Solution Assessment

Unclear. Making the model always win would match the reporter expectation, but it conflicts with the documented full-override behavior and #69060, so a maintainer precedence decision should come before a fix PR.
