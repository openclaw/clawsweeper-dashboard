## Summary

Keep open. Current main still lets a raw `x-openclaw-session-key` bypass the model-selected agent on the OpenAI-compatible Chat Completions path, but changing that behavior conflicts with the documented full-routing meaning of the header and with related PR #69060, so this needs a maintainer contract decision before implementation.

## What This Changes

The issue asks for `x-openclaw-session-key` on `/v1/chat/completions` to preserve `model: "openclaw/<agentId>"` agent routing instead of causing raw session keys to execute under the default agent.

## Best Possible Solution

Define the public precedence contract for full-routing session keys versus agent-scoped stable session hints, then update Chat Completions and OpenResponses together with regression tests for raw headers, canonical `agent:<id>:` headers, model-selected agents, `x-openclaw-agent-id`, and previous-response/session reuse.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Chat Completions request with `model: "openclaw/beta"` plus raw `x-openclaw-session-key: my-session-key`; current main forwards the raw key without an agent id, and downstream session resolution maps that key to the default agent.

## Solution Assessment

Unclear. The requested behavior may be a better operator experience, but it changes the documented meaning of `x-openclaw-session-key`; a safer path is for maintainers to choose the header precedence contract before accepting a code change.
