## Summary

Keep open. Current main still has a source-level session-header/model routing mismatch, but the documented `x-openclaw-session-key` full-override contract and the related open session-header PR make the fix an API precedence decision rather than a conservative cleanup close.

## What This Changes

The issue asks `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` supplies the session key.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `model: "openclaw/beta"` with raw `x-openclaw-session-key: my-session-key`; current main returns the raw key, passes no agent override, and downstream command setup resolves the session agent from that raw key.

## Solution Assessment

Unclear. Passing the selected agent through or making model routing win would satisfy the report, but it conflicts with documented full-override semantics and the related open session-header PR, so maintainers should decide precedence first.
