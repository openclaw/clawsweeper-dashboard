## Summary

Keep open. Current main still has the reported raw session-header/model routing mismatch, and the documented `x-openclaw-session-key` full-routing override plus related PR #69060 make this a maintainer API precedence decision rather than a conservative cleanup close.

## What This Changes

The issue requests that `/v1/chat/completions` honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, OpenResponses, docs, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `model: "openclaw/beta"` with raw `x-openclaw-session-key: my-session-key`; current main returns the raw key, passes no agent override, and downstream resolution maps that key to the default agent.

## Solution Assessment

Unclear. Making the model always win would match the reporter expectation, but it conflicts with the documented full-override behavior and #69060, so the safer path is an explicit maintainer precedence decision first.
