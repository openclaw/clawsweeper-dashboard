## Summary

Keep open. Current main still gives raw `x-openclaw-session-key` precedence over model-derived agent scoping, and the related stable-session PR explicitly preserves that full-override behavior, so this remains an API precedence issue rather than conservative cleanup.

## What This Changes

The issue asks for `x-openclaw-session-key` on `/v1/chat/completions` to carry the session key without overriding `model: "openclaw/<agentId>"` routing.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with raw `x-openclaw-session-key`, and current main resolves the selected agent and raw session key independently before command routing derives the effective agent from the key.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override semantics and intersects #69060, so the safer path is an explicit maintainer precedence decision first.
