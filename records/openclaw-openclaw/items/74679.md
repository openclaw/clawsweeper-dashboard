## Summary

Keep open. Current main still has the raw-header/model mismatch, and the documented `x-openclaw-session-key` full-override contract plus related PR #69060 make this an API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one precedence contract for OpenAI-compatible HTTP sessions, then update Chat Completions and OpenResponses docs, tests, and routing code together.

## Reproduction Assessment

Yes. A source-level reproduction is `model: "openclaw/beta"` with raw `x-openclaw-session-key: my-session-key`; current main returns the raw key, does not pass an agent override, and downstream session resolution maps that key to `main`.

## Solution Assessment

Unclear. Making the model always win would match the reporter expectation, but it conflicts with the documented full-override behavior and #69060, so the safer path is an explicit maintainer precedence decision first.
