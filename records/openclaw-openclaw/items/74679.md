## Summary

Keep open. Current main still lets a raw `x-openclaw-session-key` drive session-derived routing over the `model` agent, but the docs describe that header as a full session-routing override, so changing it needs a maintainer API precedence decision with #69060.

## What This Changes

The issue requests that `/v1/chat/completions` respect `model: "openclaw/<agentId>"` even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only the raw key into command routing, where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making the model always win would satisfy the report, but it would change documented full-override semantics and overlaps #69060, so the maintainable fix needs an explicit precedence decision first.
