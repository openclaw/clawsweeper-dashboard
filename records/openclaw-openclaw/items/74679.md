## Summary

Keep open. Current main still lets a raw `x-openclaw-session-key` win before agent-scoped session derivation, so Chat Completions can echo a model-selected agent while execution resolves from the raw session key; open PR #69060 also preserves the competing full-override contract.

## What This Changes

The issue asks for `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` even when callers pass `x-openclaw-session-key` as the session identifier.

## Best Possible Solution

Define the session precedence contract, then update Chat Completions and OpenResponses together with docs and regression tests for raw headers, canonical agent-prefixed headers, model-selected agents, `x-openclaw-agent-id`, and previous-response reuse.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Chat Completions request with `model: "openclaw/beta"` plus raw `x-openclaw-session-key: my-session-key`; current main forwards the raw key and downstream agent resolution maps it to `main`.

## Solution Assessment

Unclear. Making the model always win matches the reporter expectation, but it conflicts with current docs and #69060's stated full-override behavior, so the maintainable fix starts with a maintainer-defined precedence contract.
