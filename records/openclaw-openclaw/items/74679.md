## Summary

Keep open. Current main still lets a raw `x-openclaw-session-key` bypass model-scoped session construction on Chat Completions, but the public docs also describe that header as a full session-routing override and related PR #69060 preserves that behavior, so this needs an explicit API precedence decision rather than cleanup closure.

## What This Changes

The issue asks for `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` supplies the session key.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection on current main shows that a raw `x-openclaw-session-key` returns before agent-scoped key construction, and the command path later derives the effective agent from that raw key, which resolves to the default agent.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override behavior and overlaps #69060, so the safer path is a maintainer precedence decision first.
