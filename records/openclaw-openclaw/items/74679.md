## Summary

Keep open. Current main still lets raw `x-openclaw-session-key` bypass the model-derived agent scope, and the documented header semantics plus related open PR #69060 mean this needs a maintainer API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main resolves the selected agent separately from the raw key before command routing derives the effective agent from that key.

## Solution Assessment

Unclear. Making the model always win would match this report, but it conflicts with the documented full-override semantics and overlaps #69060, so the safer next step is an explicit maintainer precedence decision.
