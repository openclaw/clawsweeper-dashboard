## Summary

Keep open. Current main and the latest release still let raw `x-openclaw-session-key` take precedence over the model-selected agent, while the docs also state that this header fully controls session routing, so this needs a maintainer API-precedence decision rather than cleanup.

## What This Changes

Request that `x-openclaw-session-key` carry a session key without overriding `model: "openclaw/<agentId>"` agent routing on `/v1/chat/completions`.

## Best Possible Solution

Define one documented OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes, at source level. Send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`; current main passes only that raw key into command routing, where non-agent keys resolve to the default agent.

## Solution Assessment

Unclear. Making `model` always win would match the report, but it would revise the documented full-override semantics and conflict with the related stable-session PR's stated priority order.
