## Summary

Keep open. Current main still has the reported precedence path, and the remaining question is an API contract conflict between model-derived agent routing and the documented full session-routing override for `x-openclaw-session-key`.

## What This Changes

The issue reports that `x-openclaw-session-key` on `/v1/chat/completions` causes `model: "openclaw/<agentId>"` routing to be ignored, sending the turn to the default agent session store.

## Best Possible Solution

Define a single OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: combine `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and the command receives only the raw key, so effective routing is derived from that key instead of the selected model agent.

## Solution Assessment

Unclear. Making `model` always win would fix this report, but current docs and the related stable-session PR treat `x-openclaw-session-key` as a full routing override, so maintainers should settle the precedence contract first.
