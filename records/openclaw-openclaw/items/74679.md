## Summary

Keep open. Current main still has the raw header override path, and the documented full-routing override plus related PR #69060 make this a session precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `x-openclaw-session-key` on `/v1/chat/completions` to preserve `model: "openclaw/<agentId>"` routing instead of sending raw session keys to the default agent.

## Best Possible Solution

Define and document one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only the raw key into command routing where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making `model` always win would match this report, but it conflicts with the documented full-override semantics and overlaps #69060, so the maintainable fix needs an explicit precedence decision first.
