## Summary

Keep open. Current main still gives raw `x-openclaw-session-key` precedence over model-derived agent scoping, and the docs plus related PR context show this is an unresolved API precedence decision rather than cleanup.

## What This Changes

The issue asks the OpenAI-compatible Chat Completions endpoint to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one OpenAI-compatible session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main resolves the selected agent and raw key independently before command routing derives the effective agent from the key.

## Solution Assessment

Unclear. Making the model always win would match the report, but it would revise the documented full-override semantics and overlaps #69060, so the safer path is an explicit maintainer precedence decision first.
