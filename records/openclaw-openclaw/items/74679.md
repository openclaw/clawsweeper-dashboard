## Summary

Keep open. Current main still has the reported source path, but the docs also define `x-openclaw-session-key` as a full session-routing override, so this needs an API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `x-openclaw-session-key` on `/v1/chat/completions` to carry the session without overriding `model: "openclaw/<agentId>"` routing.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection on current main shows the model selects `agentId`, but a raw `x-openclaw-session-key` is passed as the command session key and non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override semantics and overlap #69060, so the maintainable fix needs an explicit precedence decision first.
