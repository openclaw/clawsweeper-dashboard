## Summary

Keep open. Current main still has the reported source path, but the behavior intersects the documented `x-openclaw-session-key` full-override contract and open PR #69060, so this needs a maintainer API precedence decision rather than conservative cleanup.

## What This Changes

The issue reports that using the raw `x-openclaw-session-key` header on `/v1/chat/completions` causes `model: "openclaw/<agentId>"` routing to be ignored.

## Best Possible Solution

Define one OpenAI-compatible session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw non-agent `x-openclaw-session-key`, and current main passes only that raw key into command routing where non-agent keys resolve to the default agent.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override semantics and overlaps #69060, so the maintainable fix needs an explicit precedence decision first.
