## Summary

Keep open. Current main still has the raw session-header/model routing mismatch, and the documented `x-openclaw-session-key` full-override contract plus related PR #69060 make this an API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing when `x-openclaw-session-key` is also present.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, OpenResponses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `model: "openclaw/beta"` with raw `x-openclaw-session-key: my-session-key`; current main returns the raw key, passes no agent override, and downstream command setup resolves the session agent from the key.

## Solution Assessment

Unclear. Making the model always win would match the reporter expectation, but it conflicts with the documented full-override behavior and #69060, so the safer path is an explicit maintainer precedence decision first.
