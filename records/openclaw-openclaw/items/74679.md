## Summary

Keep open. Current main still lets `x-openclaw-session-key` bypass model-derived agent routing, while the docs also describe that header as full session-routing control, so this needs an explicit API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `x-openclaw-session-key` on `/v1/chat/completions` to carry a session key without overriding `model: "openclaw/<agentId>"` agent routing.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main resolves the selected agent and raw session key independently before command routing derives the effective agent from the key.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise documented full-override semantics and overlaps #69060, so the safer path is an explicit maintainer precedence decision first.
