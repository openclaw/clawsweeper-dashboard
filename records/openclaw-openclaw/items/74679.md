## Summary

Keep open: current main still lets a raw session-key header determine routing while docs describe that header as a full routing override, so this needs a maintainer API-precedence decision rather than cleanup closure.

## What This Changes

The issue asks for `/v1/chat/completions` to keep `model: "openclaw/<agentId>"` agent routing when `x-openclaw-session-key` is also supplied.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection on current main shows that a raw `x-openclaw-session-key` is passed through unchanged and later resolves to `main` when it is not an `agent:<id>:...` key.

## Solution Assessment

Unclear. Making `model` win would address this report, but it would change the documented full-override semantics and overlaps #69060, so the maintainable path needs an explicit precedence decision first.
