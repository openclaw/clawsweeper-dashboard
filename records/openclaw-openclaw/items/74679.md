## Summary

Keep open. Current main still matches the reported routing path, but the requested behavior conflicts with the documented full-override semantics for `x-openclaw-session-key` and should be decided with the related open PR.

## What This Changes

The issue asks for `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is sent as a header.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, OpenResponses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only the raw key into command routing, where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making the model field win would match this report, but it would revise the documented full-override semantics and overlaps #69060, so the maintainable fix needs an explicit precedence decision first.
