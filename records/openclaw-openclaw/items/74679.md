## Summary

Keep open. Current main still gives raw `x-openclaw-session-key` full session-routing precedence over the selected `model`, and changing that conflicts with the documented contract plus overlapping PR #69060.

## What This Changes

The issue asks `/v1/chat/completions` to respect `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is present.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently with PR #69060.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only that raw key into ingress routing, where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making `model` always win would satisfy this report, but it would revise the documented full-override contract and conflict with #69060's stated priority order, so maintainers should decide the API precedence first.
