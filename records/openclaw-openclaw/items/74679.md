## Summary

Keep open. Current main still lets a raw `x-openclaw-session-key` bypass the model-selected agent on Chat Completions, and the docs plus related PR #69060 make this an API precedence decision rather than conservative cleanup.

## What This Changes

The issue asks for `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, OpenResponses, docs, and regression tests together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `model: "openclaw/beta"` with raw `x-openclaw-session-key: my-session-key`: current main returns the raw key and forwards no `agentId`, so downstream selection cannot preserve the model-selected agent.

## Solution Assessment

Unclear. Making the model target win matches the reporter expectation, but it conflicts with the documented full-override header contract and the open #69060 direction.
