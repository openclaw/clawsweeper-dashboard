## Summary

Keep open. Current main still lets a raw `x-openclaw-session-key` bypass model-selected agent routing, but changing that behavior would revise the documented explicit session-routing contract and overlaps with open PR #69060's stated precedence.

## What This Changes

The issue asks `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing when a raw `x-openclaw-session-key` header is present.

## Best Possible Solution

Define and document the precedence contract, then update Chat Completions and OpenResponses together with regression tests for raw headers, canonical agent-prefixed headers, model-selected agents, `x-openclaw-agent-id`, and previous-response reuse.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Chat Completions request with `model: "openclaw/beta"` plus raw `x-openclaw-session-key: my-session-key`; current main forwards the raw key without an `agentId`, and downstream resolution maps it to `main`.

## Solution Assessment

Unclear. Making the model always win may match the reporter expectation, but it conflicts with current docs and #69060's stated full-override behavior, so the safer path is a maintainer-defined two-header/session precedence contract first.
