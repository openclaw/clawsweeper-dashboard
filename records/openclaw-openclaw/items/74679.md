## Summary

Keep open. Current main still lets raw `x-openclaw-session-key` bypass `model: "openclaw/<agentId>"` routing, and the documented full-override semantics make this an API precedence decision rather than conservative cleanup.

## What This Changes

Requests that `/v1/chat/completions` honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one OpenAI-compatible HTTP session precedence contract, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only the raw key into command routing, where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override semantics and overlap #69060, so the maintainable fix needs an explicit precedence decision first.
