## Summary

Keep open. Current main still gives `x-openclaw-session-key` raw precedence over agent-scoped model routing, while the docs currently describe that header as a full session-routing override, so this needs an explicit API precedence decision rather than cleanup closure.

## What This Changes

The issue asks `/v1/chat/completions` to honor `model: "openclaw/<agentId>"` routing even when `x-openclaw-session-key` is supplied.

## Best Possible Solution

Define one documented precedence contract for OpenAI-compatible session routing, then update Chat Completions, Responses, docs, and regression tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`, and current main passes only the raw key into command routing, where non-agent keys resolve to `main`.

## Solution Assessment

Unclear. Making the model always win would match this report, but it would revise the documented full-override semantics and overlap #69060, so the maintainable fix needs an explicit precedence decision first.
