## What This Changes

The PR escapes `&`, `<`, and `>` in Microsoft Teams mention display names before building `<at>...</at>` tags and adds parser regression tests.

## Best Possible Solution

Land the narrow parser escape fix after redacted live Teams proof shows a display name containing `&`, `<`, or `>` renders as one mention and the required checks are green.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: `parseMentions("Hi @[Tom & <b>Jerry</b>](28:abc-123)")` on current main builds an unescaped `<at>` tag that `buildActivity` sends as Teams activity text/entities. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes, the patch is the narrow owner-boundary fix: escape only the Teams mention tag text and keep `mentioned.name` as the original display name. Escaping the whole outbound message or changing the messenger call path would be broader and more drift-prone.
