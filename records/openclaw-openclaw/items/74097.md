## Summary

The branch changes `resolveCapabilityModelConfigForTool` to accept a lazy provider-list thunk, updates the image/video/music model-config resolvers to pass thunks, and adds regression tests for the explicit-config short-circuit and incomplete-config fallback paths.

## Best Possible Solution

## Solution Assessment

Yes for the code path. A provider-list thunk is the narrowest maintainable fix because it avoids the expensive lookup only when the explicit model config is already complete while preserving the existing incomplete-config candidate resolution behavior.
