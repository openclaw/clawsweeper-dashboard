## Summary

Keep open. Current main still eagerly materializes image, video, and music capability provider lists before the explicit model-config short-circuit can run, and this PR remains a narrow, behavior-preserving fix; the remaining repair is a required changelog entry for the user-facing performance improvement.

## What This Changes

The PR changes `resolveCapabilityModelConfigForTool` to accept a lazy provider-list thunk, updates the image/video/music model-config resolvers to pass thunks, and adds regression tests for explicit-config short-circuit and incomplete-config fallback behavior.

## Best Possible Solution

Land this lazy provider-listing implementation, or an equivalent patch, with the regression tests and a single-line `CHANGELOG.md` Fixes entry in the active release section crediting `@poolside-ventures`; keep the linked bug report open until the PR merges.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes for the code path. A provider-list thunk is the narrowest maintainable fix because it skips the expensive lookup only when explicit model config is already complete while preserving the existing incomplete-config candidate resolution behavior; the patch still needs changelog coverage before merge.
