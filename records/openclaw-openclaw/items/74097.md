## Summary

Current main still eagerly lists image/video/music capability providers before the explicit model-config short-circuit can run, and the PR’s lazy provider thunk is the narrow, behavior-preserving fix; the remaining actionable blocker is the required changelog entry for a user-facing performance fix.

## What This Changes

The branch changes `resolveCapabilityModelConfigForTool` to accept a lazy provider-list thunk, updates the image/video/music model-config resolvers to pass provider-list thunks, and adds regression tests for the explicit-config short-circuit and incomplete-config fallback paths.

## Best Possible Solution

Land this lazy provider-listing change, or an equivalent patch, with the added regression tests and one single-line `CHANGELOG.md` entry under Unreleased Fixes crediting `@poolside-ventures`; keep the paired bug report open until the PR merges.

## Reproduction Assessment

Yes. Current main’s source is enough to reproduce the eager evaluation path: the three media tool resolvers compute `listRuntime*GenerationProviders` before the helper can return a complete explicit `modelConfig`, and #74096 provides production stack/timing evidence for the hosted-gateway impact.

## Solution Assessment

Yes for the code path. A provider-list thunk is the narrowest maintainable fix because it avoids the expensive lookup only when the explicit model config is already complete while preserving the existing incomplete-config candidate resolution path.
