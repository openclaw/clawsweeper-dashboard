## Summary

Keep open. Current main already reduced the broad reply-startup registration cost, but the shared model-config helper and image/video/music execution wrappers still eagerly list runtime providers before returning a complete explicit model config, so the PR remains useful after a rebase and changelog repair.

## What This Changes

The PR changes media model-config resolution to accept a provider-list thunk, updates the image/video/music wrappers, and adds regression tests for the explicit-config short-circuit and fallback paths.

## Best Possible Solution

Rebase and land the lazy provider-listing helper change with targeted tests and a changelog entry, while preserving main's manifest/auth-store registration optimization.

## Reproduction Assessment

Yes. Current main still reproduces the helper/execution-path eager evaluation by passing listRuntime*GenerationProviders into resolveCapabilityModelConfigForTool before the explicit-config short-circuit; the broad reply-startup registration variant is already covered by newer main tests.

## Solution Assessment

Yes. A provider-list thunk is the narrowest maintainable fix for the remaining helper path because it avoids provider enumeration only when the explicit model config is already complete and leaves fallback candidate resolution unchanged.
