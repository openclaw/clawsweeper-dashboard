## Summary

Keep open. Current main still has the source-level SSE failure conditions, and https://github.com/openclaw/openclaw/pull/82355 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that OpenAI-compatible HTTP SSE chat-completion streaming can drop a leading less-than character and close before the final content chunk.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch, then close this issue after the fix is merged and verified.

## Reproduction Assessment

Yes, at source level: current main still buffers a bare `<`, prefers assistant `delta` over cumulative `text`, and finalizes SSE synchronously while a `text_end` flush can still be queued as a microtask.

## Solution Assessment

Yes. The narrow maintainable fix is to reconstruct SSE deltas from cumulative assistant text and defer finalization enough for same-turn assistant events to flush; the issue should remain open until the linked PR or an equivalent patch lands.
