## Summary

Keep open. Current main still has the source-level SSE failure conditions, and https://github.com/openclaw/openclaw/pull/82355 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that OpenAI-compatible HTTP SSE chat completions can drop a leading `<` from streamed assistant content and close before the final chunk is delivered.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch, then close this issue after the fix is merged and verified.

## Reproduction Assessment

Yes, at source level: current main buffers a bare `<`, prefers assistant `delta` over cumulative `text`, and can synchronously end SSE while the `text_end` flush is queued as a microtask. I did not execute a live HTTP SSE repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is the linked PR's direction: reconstruct SSE content from cumulative assistant text when available and defer finalization enough for same-turn assistant events to flush.
