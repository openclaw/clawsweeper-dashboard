## Summary

Keep open. Current main and the latest release still show the reported HTTP SSE failure conditions, and https://github.com/openclaw/openclaw/pull/82355 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that `/v1/chat/completions` SSE can drop a leading `<` from streamed assistant content and send `[DONE]` before the final content chunk.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow fix, then close this issue after merge and SSE verification.

## Reproduction Assessment

Yes, at source level: current main buffers a bare `<`, reads assistant `delta` before cumulative `text`, and can synchronously close SSE while a `text_end` flush is still queued as a microtask. I did not execute a live HTTP SSE repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable path is to fix SSE delta reconstruction and defer stream finalization enough for same-turn assistant events to flush; the linked open PR is already the implementation candidate.
