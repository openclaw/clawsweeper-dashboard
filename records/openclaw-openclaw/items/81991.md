## Summary

Keep open. Current main still has the source-level SSE failure conditions, and the open linked PR is the implementation candidate rather than a merged fix.

## What This Changes

The issue reports malformed OpenAI-compatible HTTP SSE output where streamed content can lose a leading `<` and `[DONE]` can be sent before the final assistant fragment.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch with regression coverage, then close this issue after merge verification.

## Reproduction Assessment

Yes, at source level: current main buffers a bare `<`, prefers assistant `delta` over cumulative `text`, and can synchronously close SSE while the `text_end` flush is still queued as a microtask. I did not execute a live HTTP repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to fix SSE delta reconstruction and defer finalization enough for same-turn assistant events to flush; the linked PR is already the candidate for that path.
