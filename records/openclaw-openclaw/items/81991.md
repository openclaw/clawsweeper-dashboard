## What This Changes

The issue reports that `/v1/chat/completions` SSE can drop a leading `<` from streamed assistant content and close before the final content chunk is delivered.

## Best Possible Solution

Refresh, review, and land https://github.com/openclaw/openclaw/pull/82355 or an equivalent narrow gateway/agent streaming fix, then close this issue from the merged implementation.

## Reproduction Assessment

Yes. Source inspection on current main still shows a bare `<` can be buffered, HTTP SSE still extracts incremental assistant deltas before cumulative text, and finalization can close the response before the queued `text_end` flush runs.

## Solution Assessment

Yes. The linked PR's direction is the narrow maintainable fix: reconstruct streamed output from cumulative assistant text when available and defer finalization long enough for pending final text flushes; the issue should stay open until that or an equivalent patch merges.
