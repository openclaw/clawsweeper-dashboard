## Summary

Keep open. Current main and the latest release still show the source-level SSE failure conditions, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that OpenAI-compatible `/v1/chat/completions` SSE can omit a leading `<` and send `[DONE]` before the final streamed content chunk.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow SSE patch with regression coverage, then close this issue from the merge.

## Reproduction Assessment

Yes, at source level: current main buffers a bare `<`, reads assistant `delta` before cumulative `text`, and can synchronously close SSE while the `text_end` flush is still queued as a microtask. I did not run a live HTTP repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to fix SSE delta reconstruction and defer finalization enough for same-turn assistant events to flush, with focused regression coverage.
