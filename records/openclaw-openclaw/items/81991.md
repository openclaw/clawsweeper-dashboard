## Summary

Keep open. Current main still has the source-level SSE failure conditions, and the open closing PR at https://github.com/openclaw/openclaw/pull/82355 has not merged yet.

## What This Changes

The issue reports that HTTP SSE chat completions can drop a leading `<` in streamed XML-style content and close before the final assistant fragment is delivered.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch, then close this issue after the fix is merged and verified.

## Reproduction Assessment

Yes, at source level: current main buffers a bare `<`, reads assistant `delta` before cumulative `text`, and can synchronously close SSE while the `text_end` flush is still queued as a microtask. I did not run a live HTTP repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to fix SSE delta reconstruction and defer finalization enough for same-turn assistant events to flush, with focused regression coverage; the linked open PR is the current implementation path.
