## What This Changes

The issue reports malformed and truncated OpenAI-compatible `/v1/chat/completions` SSE output when XML-style assistant text is split across streamed chunks.

## Best Possible Solution

Refresh, review, and land https://github.com/openclaw/openclaw/pull/82355 or an equivalent narrow gateway/agent streaming fix, then close this issue from the merged implementation.

## Reproduction Assessment

Yes. Source inspection on current main shows the same delta/text divergence, bare-`<` buffering, and synchronous SSE finalization before the queued `text_end` block flush can run.

## Solution Assessment

Yes. The narrow maintainable solution is to preserve cumulative assistant text when deriving chat-completions SSE deltas and defer finalization long enough for pending final stream flushes; the linked PR is the current candidate implementation.
