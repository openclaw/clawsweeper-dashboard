## Summary

Keep open. Current main still has the source-level SSE failure conditions, and the linked fix PR is still open rather than merged, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `/v1/chat/completions` SSE drops literal leading `<` characters and can send `[DONE]` before the final streamed assistant fragment.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch, then close this issue after the fix is merged and verified.

## Reproduction Assessment

Yes, at source level: current main buffers a bare `<`, reads assistant `delta` before cumulative `text`, and can synchronously close SSE while the `text_end` flush is still queued as a microtask. I did not execute a live HTTP repro in this read-only review.

## Solution Assessment

Yes, the narrow maintainable solution is to fix SSE delta reconstruction and defer finalization enough for same-turn assistant events to flush, with focused regression coverage. Because an open PR already references this issue with closing syntax, the issue should stay open until that PR lands or is replaced.
