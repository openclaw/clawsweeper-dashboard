Keep open. Current main and the latest release still show the source-level SSE failure conditions, and the linked implementation candidate is still open rather than merged: https://github.com/openclaw/openclaw/pull/82355.

## What This Changes

The issue reports HTTP SSE chat-completions streams dropping a leading `<` from split XML-style tags and ending the stream before the final content fragment is delivered.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch with regression coverage and SSE ordering proof, then close this issue from the merge.

## Reproduction Assessment

Yes, at source level: current main still buffers a bare `<`, prefers assistant `delta` over cumulative `text`, and can synchronously close SSE while the `text_end` flush is queued as a microtask. I did not execute a live HTTP repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable path is to fix SSE delta reconstruction and defer finalization enough for same-turn assistant events to flush; the open linked PR is already the implementation candidate.
