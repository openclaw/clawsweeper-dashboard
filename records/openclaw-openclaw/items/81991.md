## Summary

Keep open. Current main still matches the source-level SSE failure conditions, and https://github.com/openclaw/openclaw/pull/82355 is still the open implementation candidate rather than a merged fix.

## What This Changes

The issue reports malformed and truncated HTTP SSE `/v1/chat/completions` streams when custom XML-style content crosses token boundaries.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch, then close this issue after the fix is merged and verified.

## Reproduction Assessment

Yes, at source level: current main buffers a bare `<`, reads assistant `delta` before cumulative `text`, and can synchronously close SSE while a `text_end` flush is still queued as a microtask. I did not execute a live HTTP repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to fix SSE delta reconstruction and defer finalization enough for same-turn assistant events to flush, with focused regression coverage; the linked open PR is already aimed at that path.
