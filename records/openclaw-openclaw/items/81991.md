Keep open. Current main still has the source-level HTTP SSE failure conditions, and the open linked PR references this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue asks OpenClaw to preserve leading less-than characters and flush the final assistant content chunk before `[DONE]` on `/v1/chat/completions` SSE streams.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82355, or an equivalent narrow patch, then close this issue after merge and verification.

## Reproduction Assessment

Yes, at source level: current main can buffer a bare `<`, the SSE path prefers `delta` over cumulative `text`, and chat SSE finalization can close before a queued `text_end` flush runs. I did not execute a live HTTP repro in this read-only review.

## Solution Assessment

Yes. The best current path is the linked narrow fix PR or an equivalent patch that reconstructs SSE deltas from cumulative text and delays finalization enough for same-turn assistant events to flush.
