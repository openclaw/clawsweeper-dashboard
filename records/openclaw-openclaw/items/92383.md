## Summary

The PR addresses a real current-main regression: pathological transcript metadata can make both the last message and its copied-metadata placeholder exceed the gateway history budget, producing an empty response that the Control UI renders as no transcript. The metadata-free sentinel is a bounded, gateway-owned fix, the supplied production-function proof is convincing, and no actionable correctness or security defect remains.

## What This Changes

The PR replaces the final empty `chat.history` byte-budget fallback with a metadata-free assistant notice and adds focused tests for each budget fallback tier.

## Best Possible Solution

Land the metadata-free sentinel fallback with the focused regression coverage so a preserved but undisplayable transcript is represented explicitly instead of as a blank history response.

## Reproduction Assessment

Yes. The current-main source path is deterministic, and the supplied after-fix run exercises the actual production budget functions with metadata large enough to reach the empty fallback.

## Solution Assessment

Yes. Removing source metadata only in the terminal fallback fixes the misleading blank transcript at the gateway ownership boundary without changing stored data, normal history responses, configuration, or protocol shape.
