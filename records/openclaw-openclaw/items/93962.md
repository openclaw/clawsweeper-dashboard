## Summary

Keep this PR open, but do not merge it as-is: it fixes the empty-group warning path, yet it also evaluates untyped plugin-authored availability objects after only a key-presence check, which can throw on malformed non-array allOf/anyOf values.

## What This Changes

The PR preserves plugin tool availability on captured descriptors and warns at descriptor-registration time for empty allOf/anyOf availability groups, with focused descriptor-cache tests.

## Best Possible Solution

Preserve the registration-time warning, but validate availability expression shape defensively before evaluation so malformed plugin descriptors produce diagnostics instead of crashing tool resolution.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: pass a plugin tool with availability { allOf: "not-array" } through capturePluginToolDescriptor and the PR will call evaluateToolAvailability before the evaluator's .flatMap assumption is guarded. I did not execute it because this review was read-only.

## Solution Assessment

No, this is not yet the best fix because it solves empty arrays but leaves the untyped plugin boundary able to crash on malformed group values. The safer fix keeps evaluator/planner purity and validates or diagnoses availability shape before descriptor-registration evaluation.
