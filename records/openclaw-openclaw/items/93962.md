## Summary

Keep open. The latest branch fixed the prior null/primitive group guard, but it still casts any object with `kind`, `allOf`, or `anyOf` as a typed availability expression and evaluates it during descriptor capture, so malformed kind-based signals can still crash plugin registration; it also still lacks real plugin/gateway proof.

## What This Changes

The PR preserves plugin-authored `availability` on captured plugin tool descriptors, adds descriptor-capture warnings for malformed availability groups, and expands descriptor-cache tests.

## Best Possible Solution

Validate full recursive availability signal shapes at plugin descriptor capture, warn and strip malformed authoring expressions, preserve valid expressions, and land one canonical fix with real plugin/gateway startup proof.

## Reproduction Assessment

Yes by source inspection. Current main drops plugin-authored availability while the evaluator already returns hidden `unsupported-signal` diagnostics for empty groups; the remaining PR defect is also source-reproducible through malformed kind-based signals evaluated at capture time.

## Solution Assessment

No. Descriptor capture is a plausible boundary, but this branch is not the best fix until it validates full signal shapes before evaluation and supplies real plugin/gateway proof.
