## Summary

Keep open: the registration-time warning approach is useful and proof-backed, but the patch still leaves a nested malformed availability crash path at the plugin boundary, so it is not merge-ready.

## What This Changes

The branch preserves plugin tool `availability` in cached descriptors, warns on availability authoring diagnostics during descriptor capture, and adds descriptor-cache tests for empty and top-level malformed availability groups.

## Best Possible Solution

Keep the registration-time warning design, but recursively validate `allOf`/`anyOf` groups or make the evaluator defensively return diagnostics before plugin-authored availability is evaluated.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: pass a plugin tool with nested malformed availability through PR-head descriptor capture and it reaches the current evaluator's `.map` or `.flatMap` call on a non-array value. I did not execute tests because this review is read-only.

## Solution Assessment

No. Descriptor capture is a reasonable place to surface the warning, but the fix is not complete until nested plugin-authored availability shapes are validated or the evaluator is defensive.
