- `merge-risk: 🚨 compatibility`: Preserving plugin-authored availability changes descriptor capture and planning semantics for existing plugin tools relative to current main and the latest release.

## Summary

Keep open: the updated branch fixes the earlier null/primitive group-entry blocker, but it still accepts malformed kind-based availability signals and immediately calls the typed evaluator during plugin descriptor capture, which can turn previously ignored plugin metadata into a registration crash. The external PR also still has only test-style proof, not real plugin/gateway behavior proof.

## What This Changes

The PR preserves plugin-authored `availability` on captured plugin tool descriptors, adds registration-time `console.warn` diagnostics for malformed availability groups, and adds descriptor-cache tests.

## Best Possible Solution

Validate or diagnose the full plugin-authored availability expression at the descriptor boundary before calling the typed evaluator, preserve only valid availability, warn once for malformed authoring diagnostics, and require real plugin/gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main drops plugin-authored availability while the evaluator already returns unsupported-signal diagnostics for empty groups; the PR defect is also source-reproducible because it accepts unvalidated `kind` expressions and evaluates them at capture time.

## Solution Assessment

No. Descriptor capture is a plausible boundary, but this branch is not the best fix until it validates full signal shapes before evaluation and supplies real plugin/gateway proof.
