## Repair Work Prompt

Update OpenClaw docs to clarify Codex app-server native `PreToolUse` report-mode handling of plugin `before_tool_call.requireApproval`. Document that when the raw Codex payload includes `openclaw_approval_mode: "report"`, hook-level `requireApproval` is reported back to Codex as a deny/block response and does not open an interactive OpenClaw plugin approval prompt. Point plugin authors toward returning `block` with clear user instructions or using an out-of-band durable approval workflow when they need approval on this path. Also clarify that Codex `PermissionRequest` routing through OpenClaw approvals is separate from plugin hook `requireApproval` on native tool events. Keep the change docs-only; do not add diagnostics, config, runtime behavior, or a native approval bridge. Likely pages are `docs/plugins/codex-harness-runtime.md`, `docs/plugins/plugin-permission-requests.md`, and possibly `docs/tools/acp-agents.md`; follow Mintlify link rules and use `plugin/plugins` wording.
