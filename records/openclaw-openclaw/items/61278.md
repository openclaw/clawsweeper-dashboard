## Summary

Keep open. Current main still awaits Gmail watcher startup on the Gateway sidecar path, and normal startup still waits for sidecars by default; the issue discussion has a reporter-confirmed bisection isolating the multi-minute delay to Gmail watcher rather than model-pricing.

## What This Changes

Make Gateway startup stop blocking on slow hook initialization, with the reported bottleneck now narrowed to Gmail watcher bootstrap.

## Best Possible Solution

Make only Gmail watcher bootstrap nonblocking from Gateway sidecar readiness while preserving the existing skip flag, logging, restart/reload behavior, and shutdown cleanup.

## Reproduction Assessment

Yes. Configure `hooks.enabled=true` with `hooks.gmail.account`, then make Tailscale setup or `gog gmail watch start` slow or unresolved; current main awaits that path before sidecars become ready.

## Solution Assessment

Yes for this localized issue. Backgrounding or explicitly deferring Gmail watcher startup is narrower than a broad hook timeout framework and matches the reporter’s bisection.
