Keep open: current main still awaits Gmail watcher bootstrap in the Gateway sidecar path, and the reporter's bisection isolates the startup delay to Gmail watcher rather than model-pricing; no open closing PR owns this Gmail startup wait.

Make Gateway hook/Gmail watcher startup nonblocking so slow watcher setup does not delay Gateway readiness.
