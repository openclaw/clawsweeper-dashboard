## Summary

Keep open: current main still awaits Gmail watcher startup on the Gateway sidecar path, and the issue discussion has a reporter-confirmed bisection isolating the multi-minute delay to Gmail watcher rather than model-pricing.

## What This Changes

Request that Gateway startup stop waiting on hook/Gmail watcher initialization so network-dependent hook work cannot delay readiness.

## Best Possible Solution

Make only Gmail watcher bootstrap nonblocking from Gateway sidecar readiness while preserving the skip flag, logging, restart/reload behavior, and shutdown cleanup.

## Reproduction Assessment

Yes, at source level and with reporter logs. Configure hooks.enabled=true with hooks.gmail.account, then make Tailscale setup or gog gmail watch start slow; current main awaits that path before sidecars become ready.

## Solution Assessment

Yes for the localized repair. Backgrounding or explicitly deferring Gmail watcher bootstrap is narrower than a broad hook timeout/config framework and matches the bisection evidence.
