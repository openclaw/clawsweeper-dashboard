## Summary

Keep open. The PR targets a real linked bug, but the current head still promotes plugin skills into the managed-skill precedence tier and mutates managed-root symlinks without ownership proof, so it should not merge as-is.

## What This Changes

The PR publishes plugin-manifest skills into the managed skills directory with symlinks, adds regression coverage, threads the managed skills directory into skill resolution, and adds a changelog entry.

## Best Possible Solution

Keep plugin skill publication discoverable by the agent SDK while preserving the documented extra-dir precedence tier and tracking generated entries before replacing or removing symlinks.

## Reproduction Assessment

Yes, source-based. Current main resolves enabled plugin skill paths and merges them as extra dirs, but no current code publishes them into the managed or personal agent discovery roots described in the linked bug.

## Solution Assessment

No. Publishing directly into the managed root fixes one discovery path but changes precedence and lacks ownership-tracked symlink state; a safer repair should register or generate discovery paths without promoting third-party plugin skills.
