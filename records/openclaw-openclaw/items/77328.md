## Summary

Keep open: the underlying bug is valid and the PR targets the right area, but the patch leaves the documented parent-directory plugin skill layout unpublished and can overwrite user-managed skill symlinks.

## What This Changes

The PR adds managed-directory symlink publication for plugin-declared skills, regression tests for that helper, and a changelog entry for the linked plugin skill discovery bug.

## Best Possible Solution

Publish or register concrete enabled plugin skill directories in a contained way that expands parent roots, preserves documented skill precedence, and never mutates user-managed entries without ownership proof.

## Reproduction Assessment

Yes, from source inspection: current main resolves enabled plugin skill paths and loads them as extra dirs, but no current path publishes them to the managed or agent discovery roots named in the report. The PR-introduced failure is also source-reproducible with any manifest that declares ./skills and stores SKILL.md in child directories.

## Solution Assessment

No, this PR is not yet the best fix because it only publishes direct SKILL.md directories and can override managed symlinks. The safer repair is to enumerate concrete child skill dirs under each validated plugin skill root and only create/update plugin-owned links or use a registration path that leaves managed precedence intact.

  - confidence: 0.9
