## Summary

Keep open. The latest PR revision fixes the earlier parent-root expansion and existing-link replacement concerns, but it still promotes plugin skills into the managed precedence tier and can delete unrelated managed symlinks; those block merge.

## What This Changes

The PR publishes enabled plugin-manifest skill directories into the managed skills root via symlinks, adds regression tests, and adds a changelog entry for the linked bug.

## Best Possible Solution

Keep a generated or registered plugin skill discovery path that the agent SDK can read while preserving documented plugin-skill precedence and only creating, updating, or removing entries OpenClaw can prove it owns.

## Reproduction Assessment

Yes, source-based. Current main resolves enabled plugin skill paths and loads them as low-precedence extra dirs, but no current path publishes them to the managed or personal agent discovery roots named in the linked issue; the PR-introduced precedence and cleanup defects are also reproducible from the diff.

## Solution Assessment

No. Publishing into the managed root during resolution is not yet the best fix because that root has higher OpenClaw precedence and the cleanup path has no ownership proof; a safer repair should preserve precedence and track generated entries explicitly.
