The PR changes Feishu topic-scoped session routing to prefer `thread_id` over `root_id` whenever both are present and updates the corresponding routing tests.

## Best Possible Solution

Use `thread_id` as the canonical Feishu topic identity when supplied, while proving starter-to-reply continuity and explicitly accepting or mitigating the one-time transition for sessions previously keyed by `root_id`.

## Reproduction Assessment

## Solution Assessment
