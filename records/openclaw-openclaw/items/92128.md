## What This Changes

The PR changes Feishu group-topic session resolution to prefer `thread_id` over `root_id` for all topic-scoped group sessions and updates routing tests to expect `omt_*` keys.

## Best Possible Solution

Keep the fix in the Feishu plugin, but distinguish native or topic-message-format Feishu topics from OpenClaw-created normal threads, or add a maintainer-approved migration/alias before changing canonical session keys, with focused tests and live Feishu proof.

## Solution Assessment

No. The PR is a plausible partial fix for the reported `om_*` versus `omt_*` split, but it is too broad because it breaks the existing normal-group continuity contract without a reliable discriminator, migration, alias, or real proof.
