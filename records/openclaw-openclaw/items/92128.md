## Summary

Keep open: the PR targets a real Feishu topic-session split, but the patch is too broad because it re-keys documented normal-group thread sessions without a discriminator, migration, or real Feishu proof.

## What This Changes

The branch removes the Feishu `topic_group` guard so topic-scoped group events prefer `thread_id` before `root_id`, and updates Feishu routing tests to expect `omt_*` session keys.

## Best Possible Solution

Keep the fix in the Feishu plugin, but distinguish native or topic-message-format Feishu topics from OpenClaw-created normal threads, or add a maintainer-approved migration/alias before changing canonical session keys.

## Reproduction Assessment

Yes at source level: current main can choose `rootId` before `threadId` for normal `chat_type: "group"` topic-scoped events, while this PR changes that route to `threadId`. I did not run a live Feishu tenant replay in this read-only review.

## Solution Assessment

No. The PR is a plausible partial fix for the reported `om_*` versus `omt_*` split, but it is too broad because it breaks the existing normal-group continuity contract without a reliable discriminator, migration, alias, or real proof.
