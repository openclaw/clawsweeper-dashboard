## Summary

Keep open: the PR targets a real Feishu session-split bug, but the blanket `thread_id`-first change is still too broad and would break documented normal-group root/message continuity without migration or real Feishu proof.

## What This Changes

The PR changes Feishu group-topic session resolution to prefer `thread_id` over `root_id` whenever both identifiers are present, with matching test expectation updates.

## Best Possible Solution

Keep the fix in the Feishu plugin, but only switch to the native topic identity when the event is reliably identified as Feishu topic-message-format or add a maintainer-approved migration/alias path for existing root-keyed sessions.

## Reproduction Assessment

## Solution Assessment
