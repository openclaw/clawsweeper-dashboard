## Summary

Keep open: the PR is a plausible Feishu session-splitting fix, but it still re-keys documented normal-group topic sessions from root/message ids to `threadId` without a discriminator, migration/alias, or contributor real-behavior proof.

## What This Changes

The PR removes Feishu's native-topic-only guard so topic-scoped group sessions prefer `threadId` over `rootId`, and updates Feishu routing tests accordingly.

## Best Possible Solution

Keep the fix in the Feishu plugin, but distinguish native or topic-message-format Feishu topics from OpenClaw-created normal threads, or add a maintainer-approved migration/alias before changing canonical session keys, with focused tests and live Feishu proof.
