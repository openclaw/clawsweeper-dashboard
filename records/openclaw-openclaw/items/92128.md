This PR changes Feishu group topic session resolution to prefer `thread_id` over `root_id` whenever both identifiers are present and updates routing tests accordingly.

## Best Possible Solution

Use `thread_id` for native Feishu topic groups and root-id-missing cases while preserving root/message-id continuity for normal group reply-created threads unless maintainers intentionally design a tested migration.

## Reproduction Assessment

Yes at source level: current main tests and docs show normal-group reply-created threads stay on the root/message-id key, while the PR diff changes the follow-up to `thread_id`. No real Feishu environment proof is present.

## Solution Assessment

No. The narrow maintainable fix is to keep native `topic_group` `thread_id` preference without breaking normal-group root/message-id continuity, or to add an explicit migration if maintainers choose the new key contract.
