## Summary

Keep open, but the patch is not merge-ready: it targets a real Feishu topic-session split, yet the diff re-keys normal group reply-created threads that current main, tests, and docs intentionally keep root/message-id keyed, and the external PR still has no real Feishu proof.

## What This Changes

This PR changes Feishu group-topic session resolution to prefer `thread_id` over `root_id` for all topic-scoped group sessions and rewrites routing tests to expect `omt_*` keys.

## Best Possible Solution

Keep the fix in the Feishu plugin, but distinguish native/topic-message-format Feishu topics from OpenClaw-created normal threads or add a maintainer-approved migration/alias before changing canonical session keys, with focused tests and live Feishu proof.

## Reproduction Assessment

Yes at source level: current resolver, docs, and tests preserve root/message-id continuity for normal group threads, while the PR makes follow-ups with both IDs choose `thread_id`. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

No. The PR is a plausible partial fix for the reported `om_*` versus `omt_*` split, but it is too broad because it breaks the existing normal-group continuity contract without a discriminator, migration, or real proof.
