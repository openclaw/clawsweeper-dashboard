## Summary

Keep open. The PR addresses a real CLI session-routing bug that still exists on current main, but the branch is stale against the merged default-agent routing work and would regress that current behavior if repaired naively.

## What This Changes

The PR changes agent session-key resolution so `--agent` with `--to` derives an agent-scoped target key, keeps explicit session-key precedence, normalizes blank and qualified targets, and adds resolver regression tests.

## Best Possible Solution

Integrate this PR's explicit `--agent + --to` behavior on top of current main's default-agent routing, preserve legacy migration behavior from #72414, add the required changelog entry, and then use the result to close or narrow related #44448 if covered.

## Reproduction Assessment

Yes. Source inspection shows that current main resolves an implicit agent main-session key before considering `--to` when `agentId` is provided, so `resolveSessionKeyForRequest({ agentId: "mybot", to: "cw_111" })` collapses to the agent main session.

## Solution Assessment

No. The fix direction is right, but this PR head is not the best mergeable form because it omits current default-agent routing and would regress plain `openclaw agent --to` for configured default agents.
