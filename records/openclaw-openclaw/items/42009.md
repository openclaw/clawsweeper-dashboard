## Summary

Keep open: current main still has the explicit `--agent` plus `--to` short-circuit bug, but this PR head is stale against current main and would drop the merged default-agent `--to` routing compatibility if rebased by accepting its resolver side.

## What This Changes

The branch changes agent CLI session-key resolution and tests so `--agent <id> --to <target>` derives an agent-scoped target key while preserving explicit `--session-key` precedence.

## Best Possible Solution

Rebase the explicit-agent target derivation onto current main, keep the default-agent and legacy migration contracts intact, retain focused regression tests, and add redacted real CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `agentId: "mybot", to: "cw_111"` is resolved through `resolveExplicitAgentSessionKey` to `agent:mybot:main` before `--to` can derive a target-specific key.

## Solution Assessment

No. The direction is right, but this PR head is stale and its resolver fallback would undo current default-agent `--to` routing, so the mergeable fix needs a careful rebase plus real CLI proof.
