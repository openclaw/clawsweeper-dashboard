## What This Changes

The PR changes agent CLI session-key resolution and tests so `--agent` plus `--to` derives an agent-scoped target key instead of falling back to the selected agent main session.

## Solution Assessment

No, not as currently written. The direction is useful, but the PR should be rebased onto current main and use the existing session-key resolver semantics rather than bypassing default-agent routing and direct-chat collapse.
