## Summary

Keep open: current main still has the explicit `--agent` plus `--to` short-circuit, but this PR is conflicting and the proposed resolver path needs rework before merge.

## What This Changes

The PR changes agent CLI session-key resolution so `--agent <id> --to <target>` derives an agent-scoped target key and adds focused resolver regression tests.

## Best Possible Solution

Rebase a narrow resolver fix onto current main that lets explicit agents influence `--to` session derivation without bypassing default-agent, legacy migration, or direct-chat collapse contracts, then add real CLI proof.

## Reproduction Assessment

Yes. Source inspection shows current main resolves `agentId` through the implicit agent-main key before `--to` derivation can win; I did not run a live CLI command because this review is read-only.

## Solution Assessment

No. The direction is useful, but this PR head is conflicting and its raw `toAgentStoreSessionKey` path would fork direct-chat sessions and risk losing configured-default-agent routing on rebase.
