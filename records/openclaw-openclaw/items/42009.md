## Summary

Keep open: current main still has the explicit-agent `--to` session-routing bug, and this PR targets a useful fix path, but the branch is not mergeable as-is because it is dirty against current main, still risks bypassing the documented direct-chat collapse contract, and lacks required real after-fix CLI proof.

## What This Changes

The PR changes agent CLI session-key resolution so an explicit `--agent` with `--to` derives an agent-scoped target session key and adds resolver regression tests.

## Best Possible Solution

Rebase the resolver fix onto current main, keep configured-default-agent and legacy migration behavior, preserve direct-chat collapse, derive explicit-agent target keys only where the session-key contract supports isolation, and add focused regression coverage plus redacted CLI proof.

## Reproduction Assessment

Yes. Source inspection of current main shows `--agent` calls `resolveExplicitAgentSessionKey` before `--to` derivation can win, so the reported collapse to the agent main session is source-reproducible without running the CLI.

## Solution Assessment

No. The fix direction is useful, but this branch is stale against current main and routes explicit-agent direct targets through `toAgentStoreSessionKey`, which would fork direct-chat sessions instead of preserving the documented main-session collapse.
