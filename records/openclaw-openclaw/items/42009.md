## Summary

Keep open: the reported explicit `--agent` + `--to` bug is still source-reproducible on current `main`, and this PR has useful focused tests, but the current branch is not merge-ready because it drops current default-agent `--to` routing compatibility and still lacks real behavior proof from an actual CLI/runtime setup.

## What This Changes

The PR changes agent session-key resolution so an explicit `--agent` combined with `--to` derives a target-specific agent session key while preserving explicit `--session-key` precedence.

## Best Possible Solution

Port the explicit-agent `--to` derivation onto the current default-agent resolver, keep the existing default-agent and legacy-session behavior intact, add the focused regression tests from this PR, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `resolveSessionKeyForRequest({ agentId: "mybot", to: "cw_111" })` selects the explicit agent main-session key before the `--to` key can be used.
