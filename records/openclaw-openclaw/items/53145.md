## Summary

Keep open. Current main still routes fire-and-forget `sessions_send` through the A2A announce path, still records the announce placeholder through `runAgentStep`, and the A2A control-token checks still do not treat shared silent replies such as `NO_REPLY` or `HEARTBEAT_OK` as non-deliverable. The related open #73550 appears to be the same feedback-loop family and can be handled by the same focused repair.

## What This Changes

The issue asks for inter-session announce delivery to suppress internal announce placeholders, `NO_REPLY`/acknowledgement traffic, and echo-like follow-up messages from `sessions_send(..., timeoutSeconds: 0)`.

## Best Possible Solution

Centralize a sessions-send non-deliverable control-reply predicate, reuse the shared silent-reply tokens, suppress those replies before ping-pong or announce delivery, and keep announce bookkeeping out of user-visible inter-session history while still delivering real peer replies exactly once.

## Repair Work Prompt

Fix the inter-session announce/control-reply leak for `sessions_send`, covering this issue and related #73550. In `src/agents/tools/sessions-send-tool.a2a.ts` and adjacent helpers, add or reuse a central predicate that treats sessions-send control replies plus shared silent replies (`NO_REPLY`, `HEARTBEAT_OK`, and strict silent envelopes if appropriate) as non-deliverable before ping-pong forwarding or announce delivery. Prevent the announce placeholder `Agent-to-agent announce step.` from becoming a user-visible inter-session message; keep real peer replies delivered once. Preserve the requester-owned background ACP child skip path in `src/agents/tools/sessions-send-tool.ts` and do not change unrelated channel/plugin delivery semantics. Add focused regression coverage for async `sessions_send({ timeoutSeconds: 0 })`, no user-visible announce placeholder, no inter-session `NO_REPLY` reinjection, and no duplicate/echo delivery. Update `CHANGELOG.md` with a user-facing Fixes entry crediting @TarahAssistant and @sylviazhang2006-design if the related issue is covered.
