## Summary

Keep open. Current main still sends `sessions_send` fire-and-forget traffic through the A2A announce path, treats any non-empty reply other than `REPLY_SKIP` as a fresh inter-session input, and submits the literal announce placeholder as a normal inter-session agent turn. The recent inter-session provenance work reduces model confusion but does not suppress exact `NO_REPLY`/`HEARTBEAT_OK` acknowledgements or make announce bookkeeping non-user-visible; related #73550 remains open in the same echo-loop family.

## What This Changes

The issue requests suppressing raw inter-session announce placeholder text, exact `NO_REPLY`/ack replies, and echo-like follow-up traffic from `sessions_send(..., timeoutSeconds: 0)` delivery.

## Best Possible Solution

Fix this in the sessions-send A2A owner module: classify exact silent/control acknowledgement replies before any ping-pong or announce delivery, make announce bookkeeping non-user-visible, and keep real peer replies delivered once. Treat #73550 as related validation coverage rather than closing this more detailed report first.

## Reproduction Assessment

Yes. A deterministic source-level reproduction can stub the target reply to exact `NO_REPLY`: current main will use that text as `incomingMessage` for `runAgentStep` because only `REPLY_SKIP` stops the loop, and the announce step still submits the literal placeholder message.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the `sessions_send` A2A plumbing, with display filtering as defense-in-depth only; UI-only sanitization would still leave control and placeholder traffic in session history and model turns.
