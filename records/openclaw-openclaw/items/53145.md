## Summary

Keep open. Current main still routes fire-and-forget `sessions_send` through the A2A announce flow, forwards non-empty replies until only `REPLY_SKIP` stops the loop, and submits the literal announce placeholder as an inter-session user turn. Existing display suppression covers assistant control-token replies, not these user-role inter-session inputs, and related #73550 remains open in the same feedback-loop family.

## What This Changes

Request to prevent `sessions_send` A2A announce plumbing from surfacing internal placeholder text, `NO_REPLY`, or acknowledgement follow-up traffic as user-visible inter-session messages.

## Best Possible Solution

Add a shared non-deliverable predicate for sessions-send A2A replies, suppress `NO_REPLY`/`HEARTBEAT_OK` and existing control tokens before ping-pong or announce delivery, and make announce bookkeeping non-user-visible while keeping real peer replies delivered once.

## Reproduction Assessment

Yes. A deterministic reproduction can stub `runSessionsSendA2AFlow` so the target reply is `NO_REPLY`: current code will treat it as `incomingMessage` and call `runAgentStep` for the requester because only `REPLY_SKIP` stops the loop; the announce step also always submits the placeholder input.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the sessions-send A2A owner module rather than only adding more UI filtering, because the current problem is that control/placeholder traffic can re-enter session history and model turns before display projection.

In `src/agents/tools/sessions-send-tool.a2a.ts` and token/helper code, centralize a non-deliverable sessions-send reply predicate that reuses the shared silent reply contract (`NO_REPLY`, `HEARTBEAT_OK`) plus existing `ANNOUNCE_SKIP`/`REPLY_SKIP` semantics. Add regression coverage for `sessions_send({ timeoutSeconds: 0 })` proving no placeholder, `NO_REPLY`, or duplicate acknowledgement traffic is injected, and include #73550 duplicate/echo coverage if the same fix covers it. Add a user-facing changelog fix entry with proper reporter credit; do not change unrelated chat display behavior as the sole fix.
