## Summary

Keep open: current main fails rejected cron models before an agent run, but it still lacks the requested `model-fallback/decision` WARN. An open closing PR already tracks that remaining observability fix, so this issue should stay open until that PR is resolved.

## What This Changes

Request that cron jobs targeting `agentId: main` fail or warn when `payload.model` is rejected instead of silently using the agent primary model.

## Best Possible Solution

Keep the fail-closed cron behavior and add an accurate `model-fallback/decision` WARN for rejected cron model selection, then close this issue when the linked PR or an equivalent fix lands.

## Reproduction Assessment

Yes, by source inspection: a disallowed `payload.model` takes the `ok: false` cron-preflight path and does not emit through `model-fallback/decision`. The older silent-success fallback described for v2026.5.7 is not the current-main path I found.

## Solution Assessment

Yes, the narrow maintainable fix is to preserve the current fail-closed error and add an accurate warning that does not claim a fallback model was actually used.
