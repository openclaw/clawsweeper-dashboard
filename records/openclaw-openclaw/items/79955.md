Keep open: current main now fail-closes rejected cron `payload.model` instead of silently falling through, but the remaining `model-fallback/decision` observability request is already tied to an open fixing PR with closing syntax, so this should stay open until that PR is merged or closed.

## What This Changes

The issue asks cron agent-turn jobs with a rejected `payload.model` to fail loudly or emit a model-fallback WARN instead of silently using the agent primary model.

## Best Possible Solution

Keep the fail-closed cron behavior, repair or replace https://github.com/openclaw/openclaw/pull/79967 so it logs an accurate rejection/error warning, then let the merged PR close this issue.

## Reproduction Assessment

Yes for the remaining observability gap: source inspection shows rejected cron `payload.model` returns `ok: false` without emitting through `model-fallback/decision`. The older silent Opus fallback itself does not reproduce on current main because the caller returns an error before invoking the agent runner.

## Solution Assessment

Yes, the maintainable fix direction is narrow: preserve fail-closed model selection and add an accurate WARN/rejection log on that branch. The active PR should avoid claiming a fallback model was used unless a real fallback is attempted.
