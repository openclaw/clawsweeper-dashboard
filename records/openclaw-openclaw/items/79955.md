## Summary

Keep open: current `main` already fails closed for an allowlist-rejected cron `payload.model`, but the requested `model-fallback/decision` observability is still represented by the open closing PR at https://github.com/openclaw/openclaw/pull/79967, so this issue should not be closed before that PR is resolved.

## What This Changes

The issue asks for cron runs with a rejected `payload.model` to fail loudly or emit a `model-fallback/decision` WARN instead of silently using the parent agent model.

## Best Possible Solution

Keep the fail-closed cron selection behavior and add an accurate `model-fallback/decision` rejection warning that does not claim a fallback model was used, with regression coverage for the emitted fields and error state.

## Reproduction Assessment

Yes for the narrower missing-observability path: source inspection shows rejected cron `payload.model` returns an error without emitting `model-fallback/decision`. No for the broader reported Opus fall-through on current `main`, because the inspected rejected-allowlist path returns before agent execution.

## Solution Assessment

Yes, the best path is to resolve the open linked PR rather than close this issue early. The warning should describe the selection rejection/error state, not a fallback that current code does not execute.
