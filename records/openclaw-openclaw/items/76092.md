## Summary

Keep open: the bundled-Node mitigation is useful and still missing from current main/latest release, but the PR has a source-backed compatibility blocker around documented custom-prefix installs and needs a refresh before merge.

## What This Changes

The PR changes gateway service runtime-path resolution to prefer an OpenClaw-managed bundled Node before falling back to current or system Node, with daemon coverage and a small cron-tool lint cleanup.

## Best Possible Solution

Keep the bundled-node preference, but make discovery distinguish active prefix-managed runtimes from unrelated state-dir runtimes before falling back to current or system Node.

## Reproduction Assessment

Yes at source level. The PR lookup is state-dir-only, while the documented local-prefix installer owns managed Node under `<prefix>/tools/node-v...` and can refresh a loaded service through the prefix wrapper.

## Solution Assessment

No, not yet. Preferring a managed Node is the right layer for this mitigation, but the implementation should cover both state-dir and local-prefix managed runtime layouts before merge.
