## Summary

Keep open: current main still lacks bundled state-dir Node selection, and this PR is a narrow implementation with no blocking review findings after the changelog follow-up and green exact-head checks.

## What This Changes

The PR changes daemon runtime path selection to scan a validated `<stateDir>/tools/node-v*/bin/node` before current/system Node, adds a regression test, and records a changelog entry.

## Best Possible Solution

Land this narrow bundled-Node preference after normal maintainer review, while keeping the broader runtime/package/cache drift work tracked by #75250.

## Reproduction Assessment

Yes for the PR code path: the added test models a state-dir bundled Node and a separate PATH/current Node, then asserts the bundled runtime is selected. The full Homebrew ABI drift described in #75250 remains live-environment dependent.

## Solution Assessment

Yes. Preferring a validated installed runtime from the documented state-dir tools layout is the narrowest maintainable service-runtime fix, and the existing current/system Node fallbacks remain intact.
