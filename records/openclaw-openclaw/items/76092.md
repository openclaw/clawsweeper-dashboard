## Summary

Keep open: the fix direction is useful and has real macOS proof, but the current diff still misses documented custom-prefix installs because bundled Node discovery only scans the gateway state-dir tools path.

## What This Changes

The PR makes gateway service Node selection prefer supported bundled Node executables under the state-dir tools directory, adds runtime-path coverage, adjusts a cron delivery lint guard, and updates the changelog.

## Best Possible Solution

Keep the PR open and update bundled Node discovery to consider the active local-prefix install root as well as the state directory before falling back to current or system Node.

## Reproduction Assessment

Yes. Source and docs give a high-confidence reproduction path: use a local-prefix install where the managed Node lives under <prefix>/tools while the gateway state directory is different, then the PR's state-dir-only lookup cannot find the bundled runtime.

## Solution Assessment

No. The approach is maintainable for the default layout, but the resolver needs to include the installer prefix contract so the same drift fix applies to documented custom-prefix installs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Search the installer prefix for bundled Node:** `src/daemon/runtime-paths.ts:129`

When prefix and state dir differ, the bundled runtime is never considered and `gateway install` still records the current/system Node, leaving the drift this PR is meant to prevent for documented prefix installs.
