## Summary

Keep open: the gateway bundled-Node direction is useful and has supplied real macOS proof, but the PR leaves the new `fs.readdir` test mock without a default for non-bundled cases, so focused runtime-path fallback tests can throw before exercising current/system Node fallback.

## What This Changes

The PR makes gateway service Node resolution prefer a validated bundled `tools/node-v*` runtime from the OpenClaw state dir before falling back to current/system Node, adds daemon tests and a changelog entry, and includes a small cron-tool lint cleanup.

## Best Possible Solution

Keep the bundled runtime preference, fix the `readdir` mock default so fallback tests remain valid, and land after focused daemon runtime-path proof on the updated PR head.

## Reproduction Assessment

No high-confidence end-to-end macOS reproduction was run in this checkout. Source inspection plus the linked issue establish the current-main drift path, the PR body supplies after-fix LaunchAgent/process proof, and the test regression is source-reproducible from the added mock.

## Solution Assessment

Mostly yes: preferring a validated bundled OpenClaw runtime before current/system Node is a narrow maintainable fix for the service-drift slice. The patch should first repair the test setup so non-bundled fallback behavior is still covered.
