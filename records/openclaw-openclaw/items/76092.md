## Summary

Keep open. Current main still selects the current/system Node for installed daemon services, while this PR adds the missing bundled state-dir runtime preference with focused test coverage, a changelog entry, and no blocking review findings.

## What This Changes

The PR makes daemon runtime-path selection scan the active state dir for a supported `tools/node-v*/bin/node` before falling back to current/system Node, adds a regression test, and records a changelog entry.

## Best Possible Solution

Land the narrow bundled-Node preference after maintainer review, while keeping the broader runtime/package/plugin cache drift tracked by #75250.

## Reproduction Assessment

Yes for the PR code path: the added test mocks state-dir `tools/node-v*/bin/node` discovery and validation to prove bundled Node is selected over a current PATH Node. The broader Homebrew ABI drift in #75250 remains live-environment dependent.

## Solution Assessment

Yes. A validated state-dir toolchain preference before ambient current/system Node is the narrowest maintainable service-install fix, and it preserves the existing fallbacks when no supported bundled runtime exists.
