## Summary

Keep open: this PR addresses a real gateway service runtime-drift problem and includes sufficient macOS terminal proof, but the current diff still misses documented custom local-prefix installs because bundled Node discovery only scans the gateway state dir.

## What This Changes

The PR adds bundled state-dir Node discovery to gateway service runtime resolution, regression tests, a changelog entry, and a small cron guard cleanup.

## Best Possible Solution

Land after bundled Node resolution covers both the state-dir and active local installer-prefix layouts, with regression coverage for a custom prefix that differs from the state dir while preserving current/system fallbacks.

## Reproduction Assessment

Yes for the source path: current main selects current/system Node without a bundled tools scan, and the PR body includes macOS LaunchAgent/process proof for the state-dir case. The remaining custom-prefix gap is source-reproducible from the documented <prefix>/tools layout and the PR's state-dir-only lookup.

## Solution Assessment

No. Preferring a validated bundled Node is the right direction, but the implementation should also consider the active installer prefix before preserving current/system fallbacks.
