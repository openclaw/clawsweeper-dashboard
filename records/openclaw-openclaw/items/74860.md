## Summary

Current main has partial runtime-dependency hot-path mitigation, but source still shows repeated tool-construction paths that can rebuild manifest-backed provider-auth metadata, and the issue discussion includes concrete production profiles plus a 2026.4.29 confirmation after the latest release.

## What This Changes

The issue reports gateway event-loop stalls and per-turn startup latency caused by `createOpenClawTools`/`createOpenClawCodingTools` synchronously rebuilding plugin/provider-auth metadata during tool creation.

## Best Possible Solution

Keep this as an actionable gateway performance bug and fix the remaining per-turn metadata work by threading explicit, freshness-aware provider-auth lookup maps or the current plugin metadata snapshot through tool construction and provider availability checks, while preserving the documented plugin cache boundary.

## Reproduction Assessment

## Solution Assessment
