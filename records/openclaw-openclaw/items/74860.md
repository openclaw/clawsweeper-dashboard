## Summary

Keep this issue open. Current main has partial mitigations for media/tool provider discovery, but the reported source shape still exists: repeated tool construction can enter plugin tool resolution and provider-auth availability checks that derive manifest-backed auth metadata through synchronous registry paths. The issue also has concrete production CPU-profile evidence plus a second macOS data point, so this is a valid narrow performance follow-up rather than stale cleanup.

## What This Changes

Investigate and fix gateway event-loop stalls caused by repeated synchronous plugin registry and provider-auth metadata work during OpenClaw tool creation.

## Best Possible Solution

Fix the hot path with flow-scoped, freshness-aware reuse: compute plugin registry/manifest/provider-auth lookup data once per tool-construction or runtime-config snapshot, thread candidate/evidence maps into availability checks, reuse the active gateway plugin registry where valid, and avoid broad persistent metadata caches that would violate the plugin boundary rules.
