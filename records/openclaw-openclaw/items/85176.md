## What This Changes

The PR updates `src/gateway/server-methods/chat.ts` to inspect delivered `isError` reply payloads after an agent run has started and broadcast a chat error instead of only emitting a transcript update.

## Best Possible Solution

Handle returned error payloads after `agentRunStarted` as one terminal failure path: broadcast the chat error, write `ok: false` / `status: "error"` dedupe with the error summary, add a focused gateway regression test, and include redacted runtime proof.
