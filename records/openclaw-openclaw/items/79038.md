## Summary

Keep open: current main still has the reported Webhooks/TaskFlow session-scope gap, and the linked fix PR is open and unmerged, so this issue should stay paired with that implementation path.

## What This Changes

Require Webhooks `run_task.childSessionKey` to be limited to the route-bound session or a recorded child/descendant session.

## Best Possible Solution

Land or supersede the linked PR with a Webhooks-side session-tree guard, regression coverage for raw and registered routes, and updated Webhooks docs/changelog.

## Reproduction Assessment

Yes. Source inspection gives a focused path: create a route-bound managed flow, call `run_task` with an unrelated `childSessionKey`, then observe that `cancel_flow` can drive cancellation through that recorded child session key.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `childSessionKey` at the Webhooks boundary before calling `runTask`, while leaving external automation identifiers on `runId` or `sourceId`; https://github.com/openclaw/openclaw/pull/79041 is already tracking that direction.
