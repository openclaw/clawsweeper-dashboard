## Summary

Keep open: current main still allows the Webhooks route to accept and forward an unconstrained `run_task.childSessionKey`, and the issue is paired with an open, unmerged fix PR.

## What This Changes

Require Webhooks `run_task.childSessionKey` values to stay within the route-bound session tree instead of allowing arbitrary session keys.

## Best Possible Solution

Land or supersede the linked PR with a Webhooks-side session-tree guard, regression coverage for raw and registered routes, and matching docs/changelog updates.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a route-bound managed flow can call `run_task` with an unrelated `childSessionKey`, and later `cancel_flow` reaches cancellation code that uses that recorded key.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `childSessionKey` at the Webhooks boundary before calling `runTask`; https://github.com/openclaw/openclaw/pull/79041 is already tracking that direction.
