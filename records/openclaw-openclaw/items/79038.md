## Summary

Keep open: current origin/main still has the Webhooks/TaskFlow session-scope gap, and https://github.com/openclaw/openclaw/pull/79041 is open and unmerged with closing syntax, so this issue should stay paired with that implementation path.

## What This Changes

The issue asks Webhooks `run_task` to reject caller-supplied `childSessionKey` values outside the route-bound session tree.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/79041 with a Webhooks-side session-tree guard, regression coverage for raw and registered routes, docs/changelog updates, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current origin/main shows `run_task` accepts and forwards `childSessionKey`, TaskFlow stores it on the child task, and `cancel_flow` later cancels through the stored key.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `childSessionKey` at the Webhooks boundary before calling `runTask`; https://github.com/openclaw/openclaw/pull/79041 already tracks that direction, while https://github.com/openclaw/openclaw/pull/79216 was closed as a weaker superseded attempt.
