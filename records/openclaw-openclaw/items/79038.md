## Summary

Keep open: current main still accepts and forwards Webhooks `run_task.childSessionKey` without a route-session-tree guard, and the open linked PR is the appropriate implementation path before closure.

## What This Changes

The issue requests rejecting Webhooks `run_task.childSessionKey` values unless they identify the route's bound session or a recorded child of that route session.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/79041 with the Webhooks-side session-tree guard, regression coverage, docs, and changelog handling, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection gives a focused path: create a route-bound managed flow, call `run_task` with an unrelated `childSessionKey`, then observe that `cancel_flow` can drive cancellation through the stored child session key.

## Solution Assessment

Yes. The narrow maintainable fix is to validate `childSessionKey` at the Webhooks boundary before calling `runTask`, using the recorded session-tree authority model rather than a lexical prefix check.
