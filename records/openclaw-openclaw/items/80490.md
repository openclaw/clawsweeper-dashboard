## Summary

Keep open. Current main still appends the generic `openclaw doctor --fix` hint to gateway startup config failures, while the linked closing PR at https://github.com/openclaw/openclaw/pull/80492 is open and not merged.

## What This Changes

The issue asks for `openclaw doctor --fix` either to build source-only plugins or for gateway validation to stop directing users to doctor when a third-party plugin package ships unbuilt TypeScript.

## Best Possible Solution

Land a scoped gateway startup diagnostic fix that suppresses or replaces the generic doctor hint when packaging diagnostics are present, while keeping packaged plugin build artifacts publisher-owned per the docs.

## Reproduction Assessment

Yes, source-reproducible. Current main shows the source-only plugin package diagnostic is emitted as a packaging issue, then gateway startup still appends the generic doctor repair hint to any invalid config snapshot.

## Solution Assessment

Yes, the narrow maintainable solution is to fix the gateway startup guidance rather than make doctor build third-party plugin packages. Auto-building would expand doctor into executing publisher build code and is not required to address the misleading hint.
