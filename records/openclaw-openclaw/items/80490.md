## Summary

Keep open. Current `main` still appends the generic `openclaw doctor --fix` startup hint for invalid gateway config snapshots even when the plugin diagnostic already says the failure is a publisher packaging issue, and the only related fix PR was closed unmerged.

## What This Changes

The issue asks for `openclaw doctor --fix` or Gateway startup guidance to stop misleading users when a third-party plugin package ships TypeScript source without compiled JavaScript output.

## Best Possible Solution

Keep packaged-plugin build output as the publisher’s responsibility, and update gateway invalid-config formatting to suppress or replace the generic doctor hint when validation already identifies a plugin packaging diagnostic.

## Reproduction Assessment

Yes. Source inspection on current `main` shows managed source-only plugin packages produce a packaging diagnostic, explicit slots can then produce `plugin not found`, and gateway startup still appends the generic doctor repair hint to invalid config snapshots.

## Solution Assessment

Yes. The narrow maintainable fix is gateway guidance correction with regression coverage; making doctor build third-party plugin source would add code-execution behavior and should not be the default repair path for this issue.
