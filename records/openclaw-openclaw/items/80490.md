## Summary

Keep open. Current main still combines an accurate source-only plugin packaging diagnostic with a generic gateway startup hint that points users back to `openclaw doctor --fix`, while doctor does not build third-party plugin source.

## What This Changes

Request that OpenClaw either build source-only installed plugins during `doctor --fix` or stop directing those packaging failures to `doctor --fix`.

## Best Possible Solution

Keep packaged-plugin runtime ownership with plugin publishers, and make gateway invalid-config guidance suppress or replace the generic doctor hint when the validation issues already identify a package-side missing compiled-runtime problem.

## Reproduction Assessment

Yes, source-reproducible. Current main skips a tracked source-only installed plugin with a packaging warning, can then raise `plugins.slots.memory: plugin not found`, and still appends the generic gateway `doctor --fix` recovery hint.

## Solution Assessment

Yes for the message-fix direction. Suppressing or replacing the gateway hint for package-side diagnostics is narrower and safer than teaching doctor to execute third-party build pipelines.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80490 with a narrow gateway/config error-formatting change. On current main, managed source-only TypeScript plugin packages emit a packaging diagnostic saying compiled runtime output is required and that this is not a local config problem, but `loadGatewayStartupConfigSnapshot` still calls `assertValidGatewayStartupConfigSnapshot(..., { includeDoctorHint: true })`, which appends the generic `openclaw doctor --fix` hint. Add a targeted way for gateway startup to suppress or replace that generic hint when the validation issues include the source-only plugin packaging diagnostic, while preserving the hint for ordinary local config and doctor-repairable validation failures. Do not add `npm run build`, `tsdown`, `tsc`, lifecycle-script execution, a config flag, or a new plugin policy in this PR. Add focused regression coverage for package-diagnostic invalid config and for a non-packaging invalid config that still shows the doctor hint. Include a user-facing changelog fix entry if the repo’s active changelog section requires it.
