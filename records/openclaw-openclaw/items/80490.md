## Summary

Keep open. Current main still appends the generic gateway `doctor --fix` hint to invalid startup config, even when the underlying plugin diagnostic already says the unbuilt TypeScript package is a publisher-side packaging issue; the related fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw either to make `doctor --fix` repair unbuilt TypeScript plugin packages or to stop directing users to doctor for that publisher-side packaging failure.

## Best Possible Solution

Keep packaged-plugin runtime output as publisher-owned, and make gateway startup suppress or replace the generic doctor hint when validation already identifies a plugin packaging error.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: source-only package validation emits the plugin-packaging diagnostic, gateway startup passes `includeDoctorHint: true`, and the formatter appends the generic doctor repair hint without checking the diagnostic class.

## Solution Assessment

Yes for the narrow guidance fix. Suppressing or replacing the gateway hint is more maintainable than teaching doctor to run arbitrary third-party build scripts; an auto-build design would need separate maintainer approval.
