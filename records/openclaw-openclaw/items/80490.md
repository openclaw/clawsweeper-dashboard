## Summary

Keep open. Current main still emits correct plugin-packaging wording but the Gateway startup invalid-config path still appends the generic `openclaw doctor --fix` hint, and an open linked PR explicitly targets this issue but has not merged.

## What This Changes

The issue asks for unbuilt TypeScript plugin packages to stop producing misleading `doctor --fix` guidance, either by making doctor build them or by changing the startup guidance.

## Best Possible Solution

Land a scoped gateway/config validation fix that suppresses or replaces the generic doctor hint for plugin packaging diagnostics while keeping packaged-plugin build output owned by the plugin publisher.

## Reproduction Assessment

Yes, source-reproducible. A source-only package emits a packaging diagnostic, configured slots can then report the plugin as missing, and Gateway startup appends the generic doctor hint whenever `includeDoctorHint` is true.

## Solution Assessment

Yes for the narrow direction. Improving the Gateway hint is the maintainable fix; auto-building third-party plugin source would execute publisher build code and is broader than this bug requires.
