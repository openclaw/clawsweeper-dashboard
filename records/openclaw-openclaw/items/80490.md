## Summary

Keep open. Current main still appends the generic `openclaw doctor --fix` startup hint for invalid gateway config, while the plugin diagnostic and docs say source-only packaged plugins are publisher packaging problems that doctor should not build locally.

## What This Changes

The issue asks OpenClaw to stop misdirecting users to `doctor --fix` for source-only TypeScript plugin packages that lack compiled runtime output, or else make doctor actually repair that case.

## Best Possible Solution

Keep third-party package build output as the plugin publisher’s responsibility, and make gateway startup suppress or replace the generic doctor hint when validation already reports a source-only plugin packaging diagnostic.

## Reproduction Assessment

Yes, source-reproducible. Current main shows the source-only package diagnostic excludes the plugin, explicit memory-slot validation can then produce `plugin not found`, and gateway startup still appends the generic doctor repair hint for any invalid snapshot.

## Solution Assessment

Yes, the best narrow fix is to correct gateway startup guidance rather than auto-build third-party package source in doctor. Automatic local builds would execute publisher-controlled build logic and should not be folded into this small bug fix without maintainer approval.
