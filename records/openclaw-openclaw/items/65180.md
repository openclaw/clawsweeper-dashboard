## Summary

Keep open. Current main already handles local model runs as stateless one-shot completions, but the profile-local transcript fallback behavior is still missing, so this PR contains useful source work. The branch is draft and partly obsolete against current main, so it needs a selective repair or replacement rather than closure.

## What This Changes

The PR adds explicit ephemeral local model-run sessions on the older `agentCommand` path, changes fresh transcript fallback resolution to use the sessions directory derived from `storePath`, adds focused tests, and adds canary baseline docs.

## Best Possible Solution

Keep the current lean local model-run implementation, then land a narrow transcript fallback change that resolves fresh fallback files inside the sessions directory derived from `storePath`, with focused tests for profile-local stores and topic/thread paths plus the required changelog entry.

## Reproduction Assessment

Yes. The remaining gap can be reproduced by tracing `resolveSessionTranscriptFile` with a profile-local `storePath` and a session entry without `sessionFile`; current main creates the fallback with `resolveSessionTranscriptPath`, then persists that fallback before the profile-local default can win.

## Solution Assessment

No, not as a whole. The transcript fallback change is the right narrow fix, but the CLI half is obsolete on current main and the docs/changelog pieces need cleanup before this work is mergeable.
