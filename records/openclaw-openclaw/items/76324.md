## Summary

Keep open. The PR targets a real current-main OpenAI SSE sanitizer stall, but the proposed patch is not merge-ready because the new split-frame assertion contradicts the sanitizer's actual output and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR changes the OpenAI SSE sanitizer to keep pulling upstream chunks until it can emit a readable SSE block, and adds a regression test for an event/data frame split across chunks.

## Best Possible Solution

Land a small sanitizer fix that drains until a readable SSE event or EOF while preserving valid frame bytes and malformed-frame filtering, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The current-main code path and the linked #76305 evidence give a high-confidence reproduction: a readable SSE frame split before the first boundary leaves the downstream reader without an emitted chunk until timeout.

## Solution Assessment

No, not as submitted. The looped sanitizer is the right direction, but the test must match the preserved `event:` line or the sanitizer must intentionally strip it, and the PR needs the required user-facing changelog entry.
