## Summary

Keep open. Current main still has the one-shot OpenAI SSE sanitizer behavior that can stall on a split readable frame, while this PR contains the focused fix, regression coverage, changelog entry, and real-world confirmation needed for normal merge review.

## What This Changes

The PR changes provider SSE sanitization to drain split OpenAI SSE frames until a complete readable event is available, adds a regression test, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent main-branch fix that preserves current non-OK/json fallback safeguards while adding the split-frame drain loop and regression coverage.

## Reproduction Assessment

Yes, source-level. Current main's sanitizer buffers a split `event:` chunk and returns from `pull()` without enqueueing; an inline Web Streams check timed out on the first downstream read, and linked live reports show the same stall in real OpenClaw runs.

## Solution Assessment

Yes. Looping inside the existing sanitizer pull until a readable block is emitted or upstream closes is the narrow maintainable fix, preserving malformed-frame filtering without adding a bypass or config knob.
