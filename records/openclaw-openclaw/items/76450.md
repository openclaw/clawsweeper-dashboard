## Summary

Keep this PR open: it addresses a real Feishu duplicate voice-send path, but the patch suppresses captions for regular audio attachments and is missing the required changelog/test coverage.

## What This Changes

The PR changes Feishu `sendMedia` to skip the separate text send when media is detected as voice audio by `audioAsVoice` or URL extension.

## Best Possible Solution

Keep the duplicate suppression, but gate it on the same conditions that request or produce Feishu voice-note delivery, then cover voice suppression and normal audio caption preservation in tests plus changelog.

## Reproduction Assessment

Yes. Source inspection of current main gives a high-confidence path: Feishu `sendMedia` sends non-empty text first, then forwards `audioAsVoice` to `sendMediaFeishu` for the media send.

## Solution Assessment

No. The direction is narrow, but the URL-extension heuristic is broader than Feishu's actual voice-note routing and should be replaced with a predicate that preserves captions for normal audio/file sends.
