## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use private cleanup staging for backup restores:** `docs/cli/backup.md:155-156`
  - body: The new-machine example extracts a backup archive into fixed `/tmp/openclaw-restore` and leaves it there. These archives can contain credentials, auth profiles, sessions, and workspace data, so publishing this creates a sensitive-data exposure path on shared systems; use a private `mktemp -d` directory, a cleanup trap, and an explicit sensitive-data warning.
  - confidence: 0.92
- **[P2] Read archiveRoot from the manifest:** `docs/cli/backup.md:122`
  - body: This says `<archive-root>` matches the archive basename, but `buildBackupArchiveRoot()` is timestamp-derived while `openclaw backup create --output` can write to a custom filename. Users with a custom-named archive will look for the wrong manifest or payload path, so the docs should use `manifest.archiveRoot` or each asset's `archivePath` as the source of truth.
  - confidence: 0.86
