## Summary

Keep open: the docs-only PR addresses a real backup-docs gap, but it still has a blocking docs correctness issue because the restore examples infer the archive root from the archive filename instead of the manifest-backed archive root.

## What This Changes

The PR adds manual restore guidance to `docs/cli/backup.md`, including archive inspection, same-machine extraction, staged new-machine restore, and migration links.

## Best Possible Solution

Repair the manual restore docs to derive the archive root from `openclaw backup verify`, `manifest.json`, or an archive listing, then land the docs fix while leaving native restore-command design to the existing feature trackers.

## Reproduction Assessment

Yes. Current main source reproduces the docs gap, and the PR-specific defect is source-reproducible from the archive-root, verify-output, and output-path contracts.

## Solution Assessment

No, not yet. Manual restore documentation is the right narrow solution for the linked docs gap, but the examples need to stop inferring `ROOT` from the archive filename.
