## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fail closed before overwriting live state:** `docs/cli/backup.md:136`
  - body: The restore example swallows every `mv` failure with `|| true`, then continues extracting restored state. If the current state directory cannot be moved because of permissions, ownership, or a busy path, the next `tar` command can overlay restored files onto live session/config data instead of preserving a clean rollback point.
  - confidence: 0.86
- **[P1] Stage sensitive restores in a private temp dir:** `docs/cli/backup.md:155-156`
  - body: Backup archives can contain credentials, auth profiles, sessions, and workspace data, but this example extracts them into a predictable `/tmp/openclaw-restore` directory with no private `mktemp`, cleanup trap, or fail-closed shell options. The canonical replacement uses a private per-run staging directory and removes it after inspection or restore.
  - confidence: 0.88
- **[P2] Read archiveRoot from the manifest:** `docs/cli/backup.md:122`
  - body: The docs say the archive root matches the archive basename, but `openclaw backup create --output ./custom-name.tar.gz` can write a custom filename while the embedded `archiveRoot` remains timestamp-derived. Restore docs should read `manifest.json` and use `assets[].archivePath` instead of reconstructing the root from the tarball name.
  - confidence: 0.84
