## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use private cleanup staging for sensitive restores:** `docs/cli/backup.md:155-156`
  - body: This example extracts backup contents into a fixed `/tmp/openclaw-restore` directory and never removes it. Backup archives can contain credentials, auth profiles, sessions, and workspaces, so users following the docs could expose sensitive state to other local users or future processes; use `mktemp -d`, a cleanup trap, and a sensitive-data warning.
  - confidence: 0.92
- **[P2] Read archive roots from the manifest:** `docs/cli/backup.md:122-132`
  - body: The docs say the archive root matches the archive basename and then hard-code `ROOT` from the filename, but `openclaw backup create --output` can write a custom filename while the embedded `archiveRoot` remains timestamp-derived. Restore guidance should read `archiveRoot` and `assets[].archivePath` from `manifest.json` so custom-output archives still work.
  - confidence: 0.88
