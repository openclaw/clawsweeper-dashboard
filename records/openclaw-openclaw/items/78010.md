## Summary

Keep open: the manual restore-docs direction is useful and the supplied proof is sufficient, but the PR still gives an unsafe archive-root shortcut that can fail for custom-named or renamed backup archives.

## What This Changes

Adds manual restore guidance to `docs/cli/backup.md`, including manifest inspection, same-machine extraction, staged new-machine restore, and migration links.

## Best Possible Solution

Fix the archive-root wording and examples to derive `ROOT` from `openclaw backup verify` output, the manifest, or an archive listing, then merge the manual restore docs while leaving native restore-command design to the existing tracker.

## Reproduction Assessment

Yes. Current main source reproduces the missing restore-docs gap, and the PR-specific defect is source-proven by the archive-root, verify-output, and output-path contracts.

## Solution Assessment

No, not yet. Manual restore documentation is the right narrow fix for the linked docs gap, but the current examples should not infer the archive root from the archive filename.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Derive the archive root instead of using the basename:** `docs/cli/backup.md:122`
