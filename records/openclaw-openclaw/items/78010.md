- **[P2] Use manifest archiveRoot instead of the filename:** `docs/cli/backup.md:122`
  - body: The docs say the archive root matches the archive basename and then set `ROOT=` from that basename, but `openclaw backup create --output` can use a custom filename while the embedded manifest keeps its own `archiveRoot`. Readers restoring such archives can follow the example and fail to locate the payload; use `manifest.json` as the source of truth.
  - confidence: 0.88
- **[P1] Stage sensitive archives in a private temp directory:** `docs/cli/backup.md:155-156`
  - body: The new-machine example extracts the backup into `/tmp/openclaw-restore` without private temp-dir creation or cleanup. These archives may contain credentials, auth profiles, sessions, and workspace data, so the docs should use a private `mktemp -d` path with cleanup before publishing this workflow.
  - confidence: 0.92
