Full review comments:

- **[P1] Clean up the output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-114`
  - body: After `createHeadlessOutput` succeeds, `getActiveWorkspaceId` or `movetoworkspacesilent` can still throw before `registerExitCleanup` records the output.   - confidence: 0.88
- **[P1] Isolate capture workspace before moving the browser:** `extensions/browser/src/browser/hyprland-capture.ts:108-112`
  - body: This moves the browser PID to the virtual output's current active workspace and then captures the whole output. The linked live Hyprland validation says the reliable path needs an OpenClaw-owned capture workspace plus focus restoration after setup and capture; otherwise the fallback can capture the wrong client or steal focus to the hidden output.
  - confidence: 0.78
- **[P1] Complete or remove the new SDK subpath:** `package.json:329-332`
  - body: Adding `./plugin-sdk/resolve-system-bin` here publishes a new Plugin SDK import path, but the PR does not add matching SDK docs/API metadata or generated baseline coverage. Either keep this helper out of the public SDK contract or add the documented metadata and baseline updates required for a new public subpath.
  - confidence: 0.84
