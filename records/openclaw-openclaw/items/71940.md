Full review comments:

- **[P1] Clean up the output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-112`
  - body: After `createHeadlessOutput` succeeds, either `getActiveWorkspaceId` or `movetoworkspacesilent` can throw before `registerExitCleanup` records the output. The setup catch only clears the promise/cache, so a failed setup can leave a `HEADLESS-*` output behind; wrap the post-create steps in cleanup that removes the new output before rethrowing.
  - confidence: 0.88
- **[P1] Isolate capture workspace before moving the browser:** `extensions/browser/src/browser/hyprland-capture.ts:108-112`
  - body: This moves the browser PID to whatever workspace is active on the new output and then captures the whole output. The downstream Hyprland validation in the discussion says the reliable path needs an OpenClaw-owned capture workspace plus focus restoration after setup and capture; otherwise this can capture the wrong window or steal focus to the hidden output.
  - confidence: 0.76
- **[P2] Complete the SDK contract before exporting the resolver:** `package.json:329-332`
  - body: Adding `./plugin-sdk/resolve-system-bin` here publishes a new Plugin SDK import path, but the PR does not add matching SDK docs/API metadata or baseline coverage. Either keep this helper out of the public SDK contract or add the documented metadata/baseline updates required for a new public subpath.
  - confidence: 0.82
