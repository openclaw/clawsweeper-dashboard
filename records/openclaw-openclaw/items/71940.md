## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve shutdown after Hyprland cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:84-93`
  - body: After this registers `SIGINT` and `SIGTERM` listeners, Node no longer applies its default signal termination behavior. These handlers only remove the virtual output and return, so a gateway that has used Hyprland capture can survive a normal `kill`, systemd stop, or Ctrl-C unless some other handler exits. Clean up and then preserve normal shutdown semantics.
  - confidence: 0.91
- **[P2] Remove the virtual output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-112`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or the workspace move throws, but cleanup is registered only after both calls. In that failure path the route falls back to CDP while leaving the new Hyprland output behind. Wrap the post-create setup and remove `outputName` before rethrowing.
  - confidence: 0.92
- **[P2] Regenerate the Plugin SDK API baseline:** `scripts/lib/plugin-sdk-entrypoints.json:67`
  - body: Adding `resolve-system-bin` to the public SDK entrypoint list changes the generated Plugin SDK API surface, but the PR does not include the generated `docs/.generated/plugin-sdk-api-baseline.sha256` update. `pnpm plugin-sdk:api:check` should report drift until the baseline is regenerated, or the new public subpath is removed.
  - confidence: 0.88

  - body: This adds a user-facing browser fix and a new `browser.hyprlandCapture` config key, but `CHANGELOG.md` is not updated. Repo policy requires user-facing fixes/config additions to add an active-version changelog entry before merge.
  - confidence: 0.84
