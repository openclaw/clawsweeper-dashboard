## Summary

Keep open: current main still has no Hyprland/grim viewport screenshot fallback, and the linked bug report includes a concrete Hyprland timeout path plus a native grim workaround, but this PR head is not mergeable yet because the new lifecycle cleanup can suppress normal shutdown, setup failures can leak a virtual output, the public SDK baseline is out of sync, and external after-fix Hyprland proof is missing.

## What This Changes

The PR adds an opt-in `browser.hyprlandCapture` config, a Hyprland `hyprctl`/`grim` capture helper, route fallback logic, tests/docs/schema updates, and a new Plugin SDK `resolve-system-bin` subpath.

## Best Possible Solution

Repair or replace this with a narrow opt-in browser-plugin fallback that cleans up virtual outputs on every setup, capture, and signal path; keeps SDK contract artifacts/docs aligned; links the Hyprland issue; and includes redacted live Hyprland proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: https://github.com/openclaw/openclaw/issues/64317 gives a concrete headed Hyprland timeout path, and current main still routes normal viewport screenshots through CDP `Page.captureScreenshot`. I did not run a live Hyprland compositor session in this read-only review.

## Solution Assessment

No. The direction is plausible and well-scoped to the browser plugin, but this PR is not the best mergeable form until lifecycle cleanup, SDK baseline alignment, PR metadata, and real behavior proof are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve shutdown after Hyprland cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:84-93`
  - body: After this registers `SIGINT` and `SIGTERM` listeners, Node no longer applies its default signal termination behavior. These handlers only remove the virtual output and return, so a gateway that has used Hyprland capture can survive a normal `kill`, systemd stop, or Ctrl-C unless another handler exits. Clean up and then preserve normal shutdown semantics.
  - confidence: 0.91
- **[P2] Remove the virtual output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-114`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or the workspace move throws, but cleanup is registered only after both calls. In that failure path the route falls back to CDP while leaving the new Hyprland output behind. Wrap the post-create setup and remove `outputName` before rethrowing.
  - confidence: 0.92
- **[P2] Regenerate the Plugin SDK API baseline:** `scripts/lib/plugin-sdk-entrypoints.json:67`
  - body: Adding `resolve-system-bin` to the public SDK entrypoint list changes the generated Plugin SDK API surface, but the PR does not include the generated `docs/.generated/plugin-sdk-api-baseline.sha256` update. `pnpm plugin-sdk:api:check` should report drift until the baseline is regenerated, or the new public subpath is removed.
  - confidence: 0.88
