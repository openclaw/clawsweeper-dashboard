## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve shutdown after Hyprland cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:84-93`
  - body: Once these `SIGINT` and `SIGTERM` listeners are registered, Node no longer applies its default signal termination behavior. The handlers only remove the virtual output and return, so a gateway that has used Hyprland capture can survive Ctrl-C, systemd stop, or `kill`; clean up and then preserve normal shutdown semantics.
  - confidence: 0.91
- **[P2] Remove the virtual output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-114`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or the workspace move throws, but cleanup is registered only after both calls. That failure path falls back to CDP while leaving the new Hyprland output behind; wrap post-create setup and remove `outputName` before rethrowing.
  - confidence: 0.92
- **[P2] Regenerate the Plugin SDK API baseline:** `scripts/lib/plugin-sdk-entrypoints.json:67`
  - body: Adding `resolve-system-bin` to the public SDK entrypoint list changes the generated Plugin SDK API surface, but the tracked API baseline checksum is unchanged from the PR base. Regenerate the Plugin SDK API baseline or remove the new public subpath.
  - confidence: 0.88
