- **[P1] Isolate the capture workspace before calling grim:** `extensions/browser/src/browser/hyprland-capture.ts:107-112`
  - body: This setup moves the browser onto whatever workspace is active on the new virtual output, then `grim -o` captures the entire output.   - confidence: 0.86

  - body: After `createHeadlessOutput` succeeds, any failure in `getActiveWorkspaceId`, `movetoworkspacesilent`, or cleanup registration rejects before `_state` is set, so `captureWithHyprland` only clears the promise/cache and has no output name to remove. Wrap the post-create setup steps so the newly created output is removed on every setup failure before falling back to CDP.
  - confidence: 0.82
