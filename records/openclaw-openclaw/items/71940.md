## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Export the trust type before re-exporting it:** `src/plugin-sdk/resolve-system-bin.ts:1`
  - body: The new public SDK subpath re-exports `type SystemBinTrust`, but `src/infra/resolve-system-bin.ts` only declares that type locally. This makes the added entrypoint fail typecheck/build; export the type from the source module or stop re-exporting it here.
  - confidence: 0.96
- **[P2] Preserve shutdown after signal cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:72-80`
  - body: Registering `SIGINT` and `SIGTERM` listeners suppresses Node's default signal termination, but these handlers only remove the virtual output and return. After Hyprland capture has run, a process without another signal handler can survive Ctrl-C, `kill`, or systemd stop.
  - confidence: 0.91
- **[P2] Remove the output when setup fails after creation:** `extensions/browser/src/browser/hyprland-capture.ts:91-98`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or the workspace move throws, but cleanup is registered only afterward. That failure path falls back to CDP while leaving the new Hyprland output behind.
  - confidence: 0.92
- **[P2] Regenerate the Plugin SDK API baseline:** `scripts/lib/plugin-sdk-entrypoints.json:67`
  - body: Adding `resolve-system-bin` to the public SDK entrypoint list changes the generated Plugin SDK API surface, but the generated baseline hash is not updated. `pnpm plugin-sdk:api:check` should report drift until the baseline is regenerated or the new subpath is removed.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/config/types.browser.ts:96-100`
  - body: This is a user-facing browser fix with a new config key, but `CHANGELOG.md` is not updated. Repo policy requires a current-version changelog entry before merging this kind of fix.
  - confidence: 0.84
