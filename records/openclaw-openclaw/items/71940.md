## Summary

Keep open. Current main still lacks the Hyprland/grim fallback, and this PR remains useful, but the current patch is not mergeable because the lifecycle cleanup can suppress shutdown and leak virtual outputs, the SDK baseline/changelog are incomplete, and the PR metadata still points at the unrelated Windows Gmail issue.

## What This Changes

The PR adds an opt-in `browser.hyprlandCapture` path for headed Linux viewport screenshots using Hyprland `hyprctl` virtual outputs and `grim`, plus config schema, docs, a Plugin SDK resolver subpath, and tests.

## Best Possible Solution

Revise this PR so the opt-in browser-plugin fallback preserves normal shutdown, removes virtual outputs on all setup/capture failures, aligns SDK generated artifacts and changelog, and links the Hyprland screenshot issue instead of the Windows Gmail issue.

## Reproduction Assessment

Yes. The linked Hyprland issue gives a concrete Omarchy/Hyprland/Wayland screenshot-timeout path, current main still uses only CDP for this route, and the PR blockers are reproducible by source-tracing the signal handlers, setup failure path, and SDK entrypoint contract.

## Solution Assessment

No. The feature direction is appropriately narrow and useful, but this patch is not the best mergeable form until lifecycle cleanup, setup-failure cleanup, generated SDK artifacts, changelog, and PR metadata are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve shutdown after Hyprland cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:84-93`
  - body: After this registers `SIGINT` and `SIGTERM` listeners, Node no longer applies its default signal termination behavior. These handlers only remove the virtual output and return, so a gateway that has used Hyprland capture can survive a normal `kill`, systemd stop, or Ctrl-C unless some other handler exits. Clean up and then preserve normal shutdown semantics.
  - confidence: 0.91
- **[P2] Clean up the output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-114`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or the workspace move throws, but cleanup is registered only after both calls. In that failure path the route falls back to CDP while leaving the new Hyprland output behind. Wrap the post-create setup and remove `outputName` before rethrowing.
  - confidence: 0.92
- **[P2] Regenerate the Plugin SDK API baseline:** `scripts/lib/plugin-sdk-entrypoints.json:67`
  - body: Adding `resolve-system-bin` to the public SDK entrypoint list changes the generated Plugin SDK API surface, but the PR does not include the generated `docs/.generated/plugin-sdk-api-baseline.sha256` update. `pnpm plugin-sdk:api:check` should report drift until the baseline is regenerated, or the new public subpath is removed.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/config/types.browser.ts:96-100`
  - body: This adds a user-facing browser fix and a new `browser.hyprlandCapture` config key, but `CHANGELOG.md` is not updated. Repo policy requires user-facing fixes/config additions to add an active-version changelog entry before merge.
  - confidence: 0.84
