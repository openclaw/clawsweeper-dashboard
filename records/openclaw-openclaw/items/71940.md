Full review comments:

- **[P2] Preserve shutdown after cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:91-93`
  - body: After these signal handlers are registered, Node no longer applies its default SIGINT/SIGTERM termination behavior. The handlers only remove the virtual output and return, so a gateway that has used Hyprland capture can survive Ctrl-C, systemd stop, or `kill`; clean up and then preserve normal shutdown semantics.
  - confidence: 0.92
- **[P2] Remove the output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-114`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or `movetoworkspacesilent` throws, but cleanup is registered only after those calls. That fallback path can leave a fresh Hyprland output behind; wrap the post-create setup and remove `outputName` before rethrowing.
  - confidence: 0.91
- **[P2] Use a dedicated capture workspace:** `extensions/browser/src/browser/hyprland-capture.ts:107-112`
  - body: The setup captures the output's current active workspace after moving Chromium there. The latest live Hyprland validation says the reliable path creates or reuses a named capture workspace on the virtual output and moves only the target window, which avoids capturing an unrelated client that lands on that output/workspace.
  - confidence: 0.84
- **[P2] Restore focus after every capture:** `extensions/browser/src/browser/hyprland-capture.ts:216-220`
  - body: The setup/capture path never records and restores the previously focused real monitor/workspace. Downstream live validation reported the hidden output can regain focus during setup, navigation, or capture, so restore focus after setup and after both successful and failed `grim` captures.
  - confidence: 0.83
- **[P2] Complete the public SDK subpath contract:** `package.json:329-332`
  - body: This adds a new public `openclaw/plugin-sdk/resolve-system-bin` export, but the PR does not add SDK docs/metadata or regenerated API-baseline coverage for that subpath. Either keep the resolver internal to the bundled browser path or add the required public SDK metadata/docs/baseline artifacts.
  - confidence: 0.88
