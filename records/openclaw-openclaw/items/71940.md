## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve shutdown after Hyprland cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:84-93`
  - body: Installing `SIGINT` and `SIGTERM` listeners disables Node's default signal termination behavior, but these handlers only remove the virtual output and return. After a Hyprland capture has run, a normal `kill`, Docker stop, or systemd stop can leave the gateway process alive until force-killed; clean up and then preserve normal shutdown semantics.
  - confidence: 0.91
- **[P2] Remove virtual output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-112`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or the workspace move throws, but cleanup is registered only after both steps. In that failure path the route falls back to CDP while the new Hyprland output remains on the compositor; remove `outputName` before rethrowing setup errors.
  - confidence: 0.93
- **[P3] Add the required changelog entry:** `src/config/types.browser.ts:95-100`
  - body: This adds a user-facing browser fix and a new `browser.hyprlandCapture` config key, but the PR head does not update `CHANGELOG.md`. Add an active-version changelog entry with valid contributor attribution before merge.
  - confidence: 0.82
