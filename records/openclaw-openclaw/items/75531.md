## Summary

Keep this PR open for normal maintainer handling. Current main still drops Slack media access on the custom invoke path, while the proposed change is narrowly scoped to forwarding the already-resolved media context and includes the expected regression test and changelog entry.

## What This Changes

The PR updates Slack action dispatch to build one media-aware action context for both custom invoke and fallback runtime paths, adds a Slack upload-file regression test, and records the user-facing fix in CHANGELOG.md.

## Best Possible Solution

Land the narrow Slack wrapper fix after the usual checks, preserving the existing local-media guard and using the regression test to keep custom invoke and fallback action paths in sync.

## Reproduction Assessment

Yes. The current-main code path is reproducible by dispatching Slack upload-file through createSlackActions with a custom invoke: core supplies media roots, Slack maps upload-file with only ctx.toolContext, and the custom invoke receives no media access before this patch.

## Solution Assessment

Yes. Forwarding the already-resolved media context at the Slack action wrapper is the narrowest maintainable fix because both custom invoke and fallback runtime paths share that boundary without relaxing local path validation.
