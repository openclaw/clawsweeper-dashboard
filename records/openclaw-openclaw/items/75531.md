## What This Changes

The PR forwards Slack action media access through both custom invoke and fallback runtime paths, adds an upload-file custom-invoke regression test, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this PR, or an equivalent Slack-wrapper fix, once exact-head validation is green. The desired end state is for Slack custom invoke and fallback runtime paths to receive the same trusted core-computed media access without changing the local-media guard or Slack upload authorization.

## Reproduction Assessment

Yes. A source-level reproduction is to call createSlackActions with options.invoke for upload-file while the channel action context contains mediaLocalRoots/mediaReadFile; current main forwards only ctx.toolContext, so the invoked Slack upload path lacks the trusted roots.

## Solution Assessment

Yes. Building the augmented action context at the Slack wrapper boundary is the narrowest maintainable fix because it preserves existing core media-root computation and the existing local-media validator instead of widening Slack upload policy.
