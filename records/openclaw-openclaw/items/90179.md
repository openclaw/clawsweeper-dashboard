## What This Changes

The issue asks OpenClaw compaction to work with Bedrock `auth: "aws-sdk"` session/profile credentials instead of cancelling when no static API key or request headers exist.

## Best Possible Solution

## Solution Assessment

Yes for the current direction. Preserving explicit `aws-sdk` auth provenance from `ModelRegistry` into the compaction safeguard is the narrow owner-boundary fix; this issue should stay open until the linked PR lands or is replaced.
