## Summary

Keep open: current main and the latest release still have the Bedrock `aws-sdk` compaction auth mismatch, and the open closing PR at https://github.com/openclaw/openclaw/pull/90206 is the right implementation path to review before this issue closes.

## What This Changes

Bedrock AWS SDK/profile session credentials can make compaction cancel because the compaction safeguard requires static request credentials even when the provider uses the AWS SDK credential chain.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90206 if its proof is accepted, or replace it with the same narrow registry-to-safeguard `aws-sdk` auth provenance fix.

## Reproduction Assessment

Yes, source-level. Current main can return successful Bedrock `aws-sdk` request auth without `apiKey` or `headers`, and the current compaction safeguard cancels that shape; I did not run a live AWS_PROFILE/STS Bedrock account path.

## Solution Assessment

Yes for the current direction. Preserving explicit `aws-sdk` auth provenance from `ModelRegistry` into the compaction safeguard is the narrow owner-boundary fix; the issue should stay open until the linked PR lands or is replaced.
