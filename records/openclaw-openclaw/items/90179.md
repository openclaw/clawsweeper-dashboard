## Summary

Keep open: current main still has a source-backed mismatch where Bedrock `aws-sdk` auth can be accepted for the compaction run, then rejected by the safeguard because the model-registry request-auth lookup returns no API key or headers.

## What This Changes

The issue reports that compaction is skipped for AWS Bedrock sessions using `AWS_PROFILE` or temporary AWS credentials because the safeguard thinks no request credentials are available.

## Best Possible Solution

Make compaction request-auth handling treat configured `aws-sdk` auth as valid authentication while preserving the existing cancel behavior for providers that genuinely require an API key or request headers.

## Reproduction Assessment

Yes from source, not from a live AWS account: current main accepts `aws-sdk` auth before compaction, then the safeguard cancels when request auth has no API key or headers.

## Solution Assessment

Yes: aligning the safeguard with the existing Bedrock `aws-sdk` auth contract is narrower than adding a bypass, fake key, or new configuration option.
