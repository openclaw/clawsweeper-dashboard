## Summary

Keep open. This maintainer-labeled PR needs human review, and source review found a blocking performance regression in the new runtime orchestration path.

## What This Changes

The PR adds optional PluginDiscoveryResult threading through plugin loader, manifest registry, installed-index, config-contract, metadata snapshot, and runtime registry loading paths, with focused discovery-threading tests.

## Best Possible Solution

Keep the additive optional discovery parameters, but remove or relocate runtime orchestrator discovery so a shared snapshot is computed only when it replaces an existing scan and cannot run before an already-loaded registry skip.

## Reproduction Assessment

Yes for the PR defect from source inspection: current main resolves a manifestRegistry before runtime load options, and the PR computes discoverOpenClawPlugins before the loaded-registry skip can run. I did not execute tests because this review is read-only.

## Solution Assessment

No. The optional discovery params are a reasonable shape, but the runtime orchestrator usage should be moved earlier into the metadata path or removed until it can replace an existing scan without adding a new one.
