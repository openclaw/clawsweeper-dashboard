## What This Changes

## Best Possible Solution

Keep the optional discovery plumbing, but compute the shared snapshot lazily inside ensurePluginRegistryLoaded, thread it through the channel manifest-record lookup, and cover already-loaded/scoped runtime paths before merge.

## Reproduction Assessment

Yes, from source inspection. PR head calls discoverOpenClawPlugins before the existing active-registry return paths, and the channel-scope helper accepts but does not consume the supplied discovery; I did not execute tests in this read-only review.

## Solution Assessment

No. The optional helper parameters are a reasonable direction, but the runtime orchestration should be lazy and the channel manifest-record path must consume the shared snapshot before this is the best fix.
