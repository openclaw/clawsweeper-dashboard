## What This Changes

The issue reports that CDP-backed browser navigation to attachment/download URLs fails with Playwright's download-starting navigation error and can lose downloaded bytes.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/89416 either lands as the guarded `/navigate` download fix or is closed in favor of another canonical implementation.

## Reproduction Assessment

Yes, source-reproducible: current main awaits `page.goto()` in `/navigate`, rethrows non-policy errors, and has no direct download capture around the Playwright attachment-download abort. I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes, the best path is the linked open PR or an equivalent guarded implementation in the browser plugin `/navigate` path. It should reuse the existing navigation policy before saving bytes and preserve explicit `/wait/download` handling separately.
