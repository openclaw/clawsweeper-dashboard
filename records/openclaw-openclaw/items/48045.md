## Summary

Keep open. Current main still has a URL-only Playwright `/navigate` path that rethrows non-policy `page.goto()` errors, while https://github.com/openclaw/openclaw/pull/89416 is an open closing PR that specifically adds managed attachment-download capture and regression coverage for this issue.

## What This Changes

Handle browser `/navigate` downloads over Playwright CDP by saving attachment responses and returning structured download metadata instead of surfacing `Download is starting` as a tool failure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89416 if it remains correct; otherwise replace it with one narrow browser-plugin fix for direct `/navigate` attachment downloads that preserves navigation policy before saving bytes.

## Reproduction Assessment

Yes, we have a high-confidence source reproduction path, but not a live run in this read-only pass. Current main rethrows non-policy `page.goto()` failures, and Playwright 1.60.0 throws `Download is starting` when `Page.navigate` reports a download.

## Solution Assessment

Yes, the linked PR is the best current fix path if review still passes: it captures a real download around guarded navigation, validates the captured URL before saving, and returns metadata only for confirmed attachment downloads. The issue should stay open until that PR lands or maintainers choose a replacement.
