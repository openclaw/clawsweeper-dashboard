## Summary

Keep open: current main and the latest release still route direct browser `/navigate` attachment URLs through URL-only Playwright `page.goto()` handling that rethrows Playwright's download abort. The adjacent managed-download work is useful, but it covers explicit and click-triggered downloads rather than this direct navigation response path.

## What This Changes

The issue asks for browser CDP navigation downloads to be saved and surfaced instead of failing with Playwright's `Download is starting` abort.

## Best Possible Solution

Implement the missing browser-plugin navigation-download path by observing a real download around guarded navigation, treating the download-starting abort as success only when a download is captured, persisting through the managed downloads directory, returning additive download metadata from `/navigate`, and preserving navigation policy checks.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: Playwright v1.60.0 throws `Download is starting` for download navigations, and current `/navigate` rethrows non-policy `page.goto()` errors while returning only `{ url }`. I did not run a live browser in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the browser plugin's guarded Playwright navigation/download handling, reusing existing managed download storage and explicit waiter semantics while preserving SSRF checks.
