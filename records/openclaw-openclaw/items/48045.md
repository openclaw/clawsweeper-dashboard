## Summary

Keep open. Current main still fixes adjacent managed/click download handling, but direct browser `/navigate` to an attachment URL remains a URL-only Playwright path that rethrows non-policy `page.goto()` errors and returns no download metadata.

## What This Changes

The issue requests browser-tool CDP navigation downloads to be saved and reported instead of failing with Playwright's download-start navigation error.

## Best Possible Solution

Implement the missing browser-plugin navigation download path: arm or observe a download around guarded navigation, treat the download-starting abort as success only when a real download is captured, persist the file through the existing guarded downloads directory, return structured download metadata from `/navigate`, and keep current navigation policy checks intact.

## Solution Assessment

No. Current main is not the best solution because it handles explicit and click-triggered download surfaces separately while leaving direct `/navigate` attachment downloads on a URL-only failure path; the safer repair is localized to browser-plugin navigation/download handling plus regression coverage.
