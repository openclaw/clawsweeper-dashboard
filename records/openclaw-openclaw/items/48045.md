## Summary

## What This Changes

The issue reports that CDP-backed browser navigation to attachment/download URLs should save the file and return download metadata instead of failing or discarding bytes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89416 or an equivalent browser-plugin fix, then close this issue once direct `/navigate` attachment downloads return managed metadata without weakening navigation policy checks.

## Reproduction Assessment

Yes, from source. Current main calls `page.goto()` through the guarded navigation helper and rethrows non-policy errors, while Playwright v1.60.0 throws `Download is starting` when CDP navigation reports a download; I did not run a live CDP browser in this read-only review.

## Solution Assessment

Yes, the best fix belongs in the browser plugin's Playwright `/navigate` seam because the route forwards that helper's result. The issue should stay open until the linked PR or an equivalent reviewed fix merges.
