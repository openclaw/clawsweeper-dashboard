## Summary

Keep open: current main and the latest release still have a URL-only Playwright `/navigate` path that rethrows Playwright's download-starting navigation abort, and the issue is already paired with an open closing PR that should be reviewed or closed before this issue is resolved.

## What This Changes

The issue reports that CDP-backed browser `/navigate` calls fail or silently drop `Content-Disposition: attachment` downloads instead of returning download metadata and saving the file.

## Best Possible Solution

Land a safe version of https://github.com/openclaw/openclaw/pull/89416, or an equivalent browser-plugin fix, so direct `/navigate` attachment downloads return structured metadata, persist through managed download storage, and keep navigation policy checks ahead of saving bytes.

## Reproduction Assessment

Yes from source, but not from a live run in this read-only review. Playwright 1.60.0 throws `Download is starting` for Chromium attachment navigations, and current OpenClaw rethrows non-policy `page.goto` errors while returning only URL metadata.

## Solution Assessment

Yes, the right fix belongs in the browser plugin's Playwright `/navigate` seam rather than in core config or a separate download feature. The issue should stay open until the linked PR or an equivalent reviewed fix is merged and shipped.
