## Summary

Keep open: current main and v2026.6.8 still have a URL-only browser `/navigate` path that rethrows Playwright's download-starting navigation abort, and https://github.com/openclaw/openclaw/pull/89416 is the open closing PR that now owns the fix path.

## What This Changes

The issue asks for CDP-backed browser `/navigate` downloads to be captured, persisted, and returned as successful download metadata instead of failing with Playwright's download-starting error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89416, or an equivalent browser-plugin fix, then close this issue once direct `/navigate` attachment downloads return managed download metadata without weakening navigation policy checks.

## Reproduction Assessment

Yes from source, but not from a live run in this read-only review. Current OpenClaw calls `page.goto()` and rethrows non-policy errors, while Playwright v1.60.0 throws `Download is starting` when CDP navigation reports `isDownload`.

## Solution Assessment

Yes, the right fix belongs in the browser plugin's Playwright `/navigate` seam because the route forwards that helper's result. The issue should stay open until the linked PR or an equivalent reviewed fix merges.
