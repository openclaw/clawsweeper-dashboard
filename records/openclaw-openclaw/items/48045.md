## Summary

Keep open. Current main includes adjacent managed/click download handling, but the direct browser `/navigate` path still returns only URL metadata, rethrows non-policy `page.goto()` errors, and has no CDP/attachment download regression coverage.

## What This Changes

The issue requests browser-tool CDP navigation downloads to be saved and reported as structured success instead of failing with Playwright's download-start navigation error.

## Best Possible Solution

Browser plugin navigation should handle attachment downloads by coupling guarded navigation with real download capture, guarded persistence, and additive `/navigate` download metadata.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `POST /navigate` to a `Content-Disposition: attachment` URL through the Playwright/CDP path: current OpenClaw calls `page.goto()` and rethrows non-policy errors while returning no download contract.

## Solution Assessment

No. Current main is not the best solution because explicit and click-triggered downloads are handled separately while direct attachment navigation remains on a URL-only failure path; the safer repair is localized to browser navigation/download handling plus regression coverage.

## Repair Work Prompt

Fix openclaw/openclaw#48045 in the browser plugin only. Current main still fails direct browser `/navigate` requests when the target URL responds as a download: `navigateViaPlaywright` returns only `{ url }`, `gotoPageWithNavigationGuard` rethrows Playwright's `Download is starting` navigation abort, and the `/navigate` route cannot return download metadata. Implement a narrow additive fix that arms or observes a download around guarded navigation, treats `Download is starting` as success only when a real download is captured, saves it under the existing guarded downloads directory, and returns structured `download` metadata from `/navigate`. Use #48132 only as context because it was closed unmerged, and account for #64558's adjacent unmanaged click-triggered download capture. Add focused regression coverage for `Content-Disposition: attachment` or an equivalent Playwright download-start abort path.
