## Summary

Keep open. Current main includes adjacent managed/click download capture, but the direct Playwright `/navigate` path still returns only URL metadata and rethrows non-policy `page.goto()` errors, so an attachment navigation can still fail instead of reporting a captured download.

## What This Changes

The issue asks for CDP-backed browser navigation to handle attachment downloads without surfacing Playwright's download-starting abort as a failed navigate.

## Best Possible Solution

Implement the missing browser-plugin navigate/download path while preserving navigation-policy checks and guarded download-directory protections.

## Reproduction Assessment

Yes, from source inspection rather than a live run: `POST /navigate` to a `Content-Disposition: attachment` URL still reaches guarded `page.goto()`, rethrows non-policy errors, and returns only `url` metadata.

## Solution Assessment

Yes, the best path is a localized browser-plugin repair that observes or arms a real download around guarded navigation and treats a download-starting abort as success only when a download is actually captured.
