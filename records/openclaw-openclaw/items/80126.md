## Summary

Keep open: this is a plausible focused fix for the linked audit false-positive, but the external PR lacks after-fix real behavior proof and the new lookup still diverges from the runtime `tools.byProvider` resolver for provider alias/canonical keys.

## What This Changes

The PR adds `tools.byProvider` deny handling to the small-model security audit and adds regression tests for matching and non-matching model-scoped deny entries.

## Best Possible Solution

Reuse or expose the runtime provider/model policy resolver for audit so the summary matches exact, provider-level, alias/canonical, and agent-scoped `tools.byProvider` denies, then add focused tests and redacted real audit output proof.

## Reproduction Assessment

Yes, source-reproducible: current main’s small-model audit builds its policy list without `cfg.tools.byProvider`, then reports web/browser exposure from that incomplete list. I did not run the packaged CLI in this read-only review shell.

## Solution Assessment

No as written: adding provider policy awareness to the audit is the right direction, but the new lookup should match the runtime resolver’s provider alias/canonical normalization instead of duplicating a narrower raw-key lookup.
