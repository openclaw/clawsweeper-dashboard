## Summary

Keep this PR open, but it should not merge as-is: the account-scoped reload direction is useful, yet the branch is conflicting, lacks real behavior proof, and misses existing reload deferral plus `secrets.reload` restart/rollback semantics.

## What This Changes

The PR adds account-targeted gateway reload planning for `channels.<channel>.accounts.<accountId>.*`, routes those planned changes through account-aware channel stop/start calls, and adds planner/server reload tests.

## Best Possible Solution

Land a rebased account-scoped reload fix that carries account targets through planner, hot-reload deferral, plugin reload interactions, and `secrets.reload` rollback, with focused tests and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. `channels.telegram.accounts.*` matches Telegram's broad reload prefix and current main records only whole-channel restarts, while the linked issue provides matching runtime logs.

## Solution Assessment

No. Account-scoped planning is the right maintainable direction, but this patch is not yet safe because it bypasses reload deferral and leaves `secrets.reload` on the old whole-channel contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Route account restarts through reload deferral:** `src/gateway/server-reload-handlers.ts:123-126`
  - body: The new account restart loop directly calls `stopChannel` and `startChannel`, while existing whole-channel reloads first wait for queued operations, pending replies, embedded runs, and active tasks. Account-scoped reloads need the same deferral or they can still interrupt active traffic during the window this fix is meant to narrow.
  - confidence: 0.88
- **[P2] Keep secrets reload aware of account targets:** `src/gateway/config-reload-plan.ts:18`
  - body: Adding `restartChannelAccounts` moves account-token changes out of `restartChannels`, but `secrets.reload` still uses only whole-channel targets for skip handling, failure reporting, and rollback. A new secrets snapshot could become active while the affected account runtime keeps stale credentials.
  - confidence: 0.9
