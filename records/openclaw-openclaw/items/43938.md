## Summary

Keep this PR open for a current-main rework: account-scoped reload remains useful, but the March branch conflicts with main and bypasses safety behavior added later for active-work draining, restart failure isolation, and secrets reload rollback. The linked issue remains open, and tests alone do not provide the required real gateway behavior proof.

## What This Changes

The PR adds account-scoped gateway reload planning so changes under `channels.<channel>.accounts.<accountId>.*` restart only affected accounts while channel-global or mixed changes still restart the full channel.

## Best Possible Solution

Reimplement the account-target concept on current main as one canonical reload target model used by ordinary hot reload and `secrets.reload`, preserving full-channel precedence, active-work draining, per-target failure collection, and rollback.

## Reproduction Assessment

Yes at source level: current main still maps `channels.<channel>.accounts.<accountId>.*` through channel-wide plugin prefixes into `restartChannels`, producing a whole-channel restart. No live current-main reproduction was run during this read-only review.

## Solution Assessment

No, not in the branch's current form. Account-targeted reload is the correct narrow direction, but the implementation must be rebuilt around current main's canonical reload deferral, failure, and secrets rollback paths.
