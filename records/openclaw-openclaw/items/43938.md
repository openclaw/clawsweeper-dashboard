## Summary

Keep open: the linked runtime logs and current source still support the account-scoped reload bug, but this PR is not merge-ready because it bypasses existing channel-reload deferral, leaves `secrets.reload` on the whole-channel contract, and lacks after-fix real behavior proof.

## What This Changes

The PR adds account-scoped gateway reload planning for `channels.<channel>.accounts.<accountId>.*` paths and routes those plans through account-aware channel stop/start calls with tests.

## Best Possible Solution

Land a rebased account-scoped reload fix that carries account targets through planner, active-work deferral, plugin reload interactions, `secrets.reload` rollback, focused tests, and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad `channels.telegram` prefix and current main records only a whole-channel restart, while the linked issue provides matching runtime logs.

## Solution Assessment

No. Account-scoped planning is the right direction, but this patch must preserve active-work deferral and `secrets.reload` skip/failure/rollback semantics before it is a safe fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Route account restarts through reload deferral:** `src/gateway/server-reload-handlers.ts:123-126`
  - body: The new account restart loop calls `stopChannel` and `startChannel` directly, while existing whole-channel reloads first wait for queued operations, pending replies, embedded runs, and active tasks. Account-scoped reloads need the same deferral or they can interrupt active traffic during the window this fix is meant to narrow.
  - confidence: 0.88
- **[P2] Keep secrets reload aware of account targets:** `src/gateway/config-reload-plan.ts:18`
  - body: Moving `channels.<channel>.accounts.*` token changes into `restartChannelAccounts` means `secrets.reload` no longer sees them in `plan.restartChannels`, so it can activate a new secrets snapshot without restarting or rolling back the affected account runtime.
  - confidence: 0.9
