## Summary

Keep open. Current main still has the account-scoped reload bug, and this PR is aimed at the right problem, but the patch should not merge as-is because account targets bypass the current reload deferral path, do not update `secrets.reload`, and lack real after-fix gateway proof.

## What This Changes

The branch adds `restartChannelAccounts` planning for `channels.<channel>.accounts.<accountId>.*` changes and routes those targets through account-aware channel stop/start calls with tests.

## Best Possible Solution

Land a rebased account-scoped reload fix that carries account targets through planner, hot-reload deferral, plugin reload interactions, and `secrets.reload` rollback, with focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level. `channels.telegram.accounts.*` matches Telegram's broad reload prefix and current main records only a whole-channel restart, while the linked issue includes matching runtime logs for full Telegram channel restart and send failures.

## Solution Assessment

No. Account-scoped planning is the right maintainable direction, but this patch is not yet the best safe fix because it bypasses reload deferral and leaves `secrets.reload` on the old whole-channel contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Route account restarts through reload deferral:** `src/gateway/server-reload-handlers.ts:120-126`
  - body: The new account restart loop calls `stopChannel` and `startChannel` immediately. Current whole-channel hot reloads wait for queued operations, pending replies, embedded runs, and active tasks before restarting, so this account path can still interrupt in-flight traffic.
  - confidence: 0.88
- **[P2] Update reload consumers for account targets:** `src/gateway/config-reload-plan.ts:16`
  - body: Adding `restartChannelAccounts` changes the reload-plan contract, but `secrets.reload` still only inspects `restartChannels` for skip handling, failure reporting, and rollback. Account token changes can activate a new secrets snapshot without restarting or rolling back the affected runtime.
  - confidence: 0.9
