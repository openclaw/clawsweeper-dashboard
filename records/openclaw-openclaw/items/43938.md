## What This Changes

The PR adds account-scoped channel restart planning so `channels.<channel>.accounts.<accountId>.*` config changes restart only the affected account while channel-global or mixed changes still restart the whole channel.

## Best Possible Solution

Rebase or replace this with a current-main-native fix that carries channel/account restart targets through the planner and handler while preserving active-work deferral, plugin reload ordering, failure aggregation, and non-manual restart semantics.

## Reproduction Assessment

## Solution Assessment

Yes as a direction, but not this branch as-is. Carrying account-scoped restart targets through the generic gateway planner and handler is the narrow maintainable fix, but the implementation must be rebased onto the current reload handler and proven in a real setup.
