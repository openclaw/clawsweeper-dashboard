## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve non-manual stop recovery for hot reloads:** `src/gateway/server-reload-handlers.ts:125`
  - body: The new account restart helper calls `stopChannel(name, accountId)` without the non-manual reload option. On current main, `stopChannel` defaults to a manual stop, and timed-out manual stops do not enter the recovery path that full-channel hot reload uses via `{ manual: false }`; an account-scoped reload can leave the account stuck if the provider stop hangs. Route this through the current `GatewayChannelManager` stop signature and pass the reload/non-manual option.
  - confidence: 0.88
- **[P1] Keep active-work deferral for account restarts:** `src/gateway/server-reload-handlers.ts:133-135`
  - body: Current main defers channel hot reloads until queued operations, replies, embedded runs, and task blockers drain. This new account-only loop restarts accounts immediately, so the exact active-traffic window from the linked issue can still be interrupted. Account-scoped restarts need to participate in the same deferral decision before calling stop/start.
  - confidence: 0.84
- **[P2] Update all reload-plan fixtures for the new field:** `src/gateway/config-reload-plan.ts:18`
  - body: `restartChannelAccounts` is added as a required `GatewayReloadPlan` field, but the branch updates only some plan literals. Current main has plan helpers and fixtures in `server-reload-handlers.test.ts` and `server-aux-handlers.test.ts`; after rebase those literals need defaults for the new field or type-check/test compilation will fail.
  - confidence: 0.82
