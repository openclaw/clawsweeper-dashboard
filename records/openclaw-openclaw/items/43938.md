## What This Changes

The branch adds account-scoped channel hot-reload planning and handler support so `channels.<channel>.accounts.<accountId>.*` changes restart only the affected channel account while channel-global or mixed changes still use a full-channel restart.

## Best Possible Solution

Rebase or replace this branch with the same narrow account-scoped reload fix on current main, preserving full-channel precedence plus current deferral, non-manual stop, and failure-collection semantics, then add redacted real behavior proof.

## Reproduction Assessment

Yes at source level. Current main maps `channels.telegram.accounts.<id>.*` through Telegram's broad reload prefix into `restartChannels`, and the handler then restarts the whole channel; the linked issue also includes reporter logs for that path.

## Solution Assessment

Yes as a direction, but not this exact stale patch. Carrying account-scoped restart targets through the generic gateway planner and handler is the narrow fix, but the current implementation must be rebased to preserve active-work deferral, `{ manual: false }` stops, and failure handling.
