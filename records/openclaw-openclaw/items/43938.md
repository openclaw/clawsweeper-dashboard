## Summary

Keep open: current main still lacks account-scoped channel-account reloads, but this external PR is not mergeable as-is because it bypasses current reload deferral, leaves `secrets.reload` semantics incomplete, conflicts with main, and lacks required real behavior proof.

## What This Changes

The PR adds `restartChannelAccounts` planning for `channels.<channel>.accounts.<accountId>.*` config paths, routes those through account-aware channel stop/start, and adds focused reload tests.

## Best Possible Solution

Land a rebased account-scoped reload fix that carries account targets through hot reload deferral, plugin reload interactions, `secrets.reload` rollback, focused tests, and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. `channels.telegram.accounts.engineer.dmPolicy` matches Telegram's broad reload prefix, current main stores only whole-channel restart targets, and the linked issue includes matching runtime logs with Telegram send failures.

## Solution Assessment

No. Account-scoped planning is the right direction, but this patch is not the best mergeable form until it preserves current active-work deferral, plugin reload interactions, and `secrets.reload` skip/failure/rollback semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Route account restarts through reload deferral:** `src/gateway/server-reload-handlers.ts:123-126`
  - body: The new account restart loop calls `stopChannel` and `startChannel` directly. Current whole-channel reloads first wait for queued operations, pending replies, embedded runs, and active tasks, so account-scoped reloads need the same deferral or they can interrupt active traffic during the window this fix is meant to narrow.
  - confidence: 0.88
- **[P2] Keep secrets reload aware of account targets:** `src/gateway/config-reload-plan.ts:181-185`
  - body: Moving `channels.<channel>.accounts.*` token changes into `restartChannelAccounts` means `secrets.reload` no longer sees them in `plan.restartChannels`, so it can activate a new secrets snapshot without restarting or rolling back the affected account runtime.
  - confidence: 0.9
