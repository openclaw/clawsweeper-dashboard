## Reproduction Assessment

Yes, at source level: current main maps `channels.telegram.accounts.<id>.*` through Telegram's broad reload prefix into `restartChannels` only, and the linked issue includes matching Telegram send failure logs. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No for this branch. Account-scoped planning is the right direction, but the implementation must be rebased onto current main's active-work deferral, plugin reload, and `secrets.reload` rollback contracts before merge.
