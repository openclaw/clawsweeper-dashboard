- **[P1] Pass non-manual stop options for account reloads:** `src/gateway/server-reload-handlers.ts:123`
  - body: Current `stopChannel` defaults to manual stops, and timeout recovery only marks `restartPending` when `manual: false` is passed.   - confidence: 0.87
- **[P2] Route account restarts through failure collection:** `src/gateway/server-reload-handlers.ts:127-130`
  - body: The account loop awaits each restart directly before the full-channel loop. Current main isolates channel restart failures and continues later restarts before throwing an aggregate error; this path should use the same collection pattern so one failed account does not skip the rest of the affected accounts or channels.
  - confidence: 0.78
