## Summary

Keep this PR open. The direction is useful and matches the linked report, but the patch is incomplete because account-scoped restarts bypass the existing active-work deferral and do not update the `secrets.reload` restart/rollback consumer.

## What This Changes

The PR adds account-scoped channel restart planning for `channels.<channel>.accounts.<accountId>.*`, wires hot reload to account-specific stop/start calls, and adds gateway reload tests.

## Best Possible Solution

Land a corrected account-scoped reload implementation on current main: keep full-channel precedence, preserve active-work deferral, extend `secrets.reload` restart and rollback handling, and add focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main routes `channels.telegram.accounts.botA.token` through Telegram's broad `channels.telegram` reload prefix into `restartChannels`, while account-scoped channel lifecycle methods already exist.

## Solution Assessment

No. The requested direction is the narrow maintainable fix, but this PR is not the best implementation until account restarts share the current deferral semantics and every reload-plan consumer handles the new account map.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve reload deferral for account restarts:** `src/gateway/server-reload-handlers.ts:120-126`
  - body: The current whole-channel reload path waits for queue, reply, run, and task activity before stop/start. The new account restart branch stops and starts accounts immediately, so an account-only config edit can interrupt active traffic that whole-channel reloads currently drain.
  - confidence: 0.88
- **[P2] Handle account restarts in every reload-plan consumer:** `src/gateway/config-reload-plan.ts:16`
  - body: Adding `restartChannelAccounts` changes the reload-plan contract, but consumers such as `secrets.reload` still inspect only `restartChannels` for skip checks, failures, and rollback. Secret-backed account paths moved into the new map can activate a new snapshot without restarting or rolling back the affected account runtime.
  - confidence: 0.9
