## Summary

Keep open. The underlying gateway reload bug is still present on current main, but the proposed patch is not mergeable as-is because account restarts bypass existing reload deferral and the new reload-plan state is not handled by secrets.reload consumers and rollback.

## What This Changes

The PR adds account-scoped restart planning for `channels.<channel>.accounts.<accountId>.*`, routes hot reload through account-specific channel stop/start calls, and adds gateway reload tests.

## Best Possible Solution

Land a corrected account-scoped reload implementation on current main: keep full-channel precedence for global or mixed channel changes, apply active-work deferral to account and whole-channel restarts, update secrets.reload aux/rollback handling, and cover both hot reload and secrets.reload with focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `channels.telegram.accounts.botA.token` matches Telegram's broad `channels.telegram` reload prefix and still becomes a whole-channel `restartChannels` entry, while account-scoped channel lifecycle methods already exist below the planner.

## Solution Assessment

No. The requested direction is the narrow maintainable fix, but this PR is incomplete until account restarts preserve active-work deferral and every reload-plan consumer, especially secrets.reload and rollback, handles account-scoped entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve reload deferral for account restarts:** `src/gateway/server-reload-handlers.ts:122-126`
  - body: Whole-channel reloads currently wait for active queue, reply, run, and task work before stopping channels. The new account-scoped branch restarts accounts immediately, so an account-only config edit can interrupt active work that current reloads protect.
  - confidence: 0.87
- **[P2] Handle account restarts in every reload-plan consumer:** `src/gateway/config-reload-plan.ts:18`
  - body: Adding `restartChannelAccounts` changes the reload-plan contract, but consumers such as `secrets.reload` still inspect only `restartChannels`. Account-scoped secret-backed config changes can activate a new snapshot without restarting the affected account unless the aux handler, rollback path, helpers, and tests handle the new field.
  - confidence: 0.9
