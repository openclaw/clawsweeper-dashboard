## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Handle scoped account restarts in secrets.reload:** `src/gateway/config-reload-plan.ts:181-183`
  - body: This changes the planner contract for every caller by moving account paths out of `restartChannels`, but `createGatewayAuxHandlers` still restarts channels only when `plan.restartChannels.size > 0`. A `secrets.reload` diff for an account token or SecretRef-backed account field would activate the new config without restarting the affected account, leaving stale credentials/runtime state active. Please either keep full-channel restart semantics for that caller or teach it to restart `restartChannelAccounts` too.
  - confidence: 0.86
- **[P1] Preserve hot-reload stop recovery semantics:** `src/gateway/server-reload-handlers.ts:125-126`
  - body: The new account-scoped restart path calls `stopChannel(name, accountId)` without the non-manual hot-reload option used by current full-channel reloads. On current main, `stopChannel` defaults to `manual: true`; if the stop times out, recovery bookkeeping is skipped and `startChannel` can return while the stale task remains. Rebase this path onto the current handler and pass the same non-manual stop/recovery behavior for scoped account restarts.
  - confidence: 0.79
