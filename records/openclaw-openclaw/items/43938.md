Full review comments:

- **[P1] Preserve non-manual reload stops:** `src/gateway/server-reload-handlers.ts:125`
  - body: The new account-scoped restart path calls `stopChannel(name, accountId)` without the `{ manual: false }` option. Current channel reloads intentionally use non-manual stops; `stopChannel` defaults to a manual stop, and on stop timeout that path does not mark recovery restart pending, so an account-scoped config reload can leave the account in the wrong lifecycle state instead of following the hot-reload recovery path. Rebase this through the current reload handler and call `stopChannel(channel, accountId, { manual: false })` with the same deferral/failure handling as full-channel reloads.
