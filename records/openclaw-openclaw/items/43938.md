## Summary

Keep open: the useful account-scoped reload direction is not on current main, but this branch is conflicting and still has merge-blocking gaps around active-work deferral, secrets.reload recovery, and real behavior proof.

## What This Changes

The branch adds account-scoped gateway reload planning for channel account config paths and routes those targets through account-specific channel stop/start calls with focused tests.

## Best Possible Solution

Rework the account-scoped reload model on current main so account targets flow through deferral, secrets.reload semantics, focused tests, and redacted runtime proof.

## Reproduction Assessment

Yes, at source level: current main maps channels.telegram.accounts.<id>.* through Telegram's broad reload prefix into restartChannels, and https://github.com/openclaw/openclaw/issues/43935 includes matching full-restart and send-failure logs. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No. Account-scoped planning is the right direction, but this branch must be rebased and preserve active-work deferral plus secrets.reload skip/failure/rollback semantics before it is the narrow maintainable fix.
