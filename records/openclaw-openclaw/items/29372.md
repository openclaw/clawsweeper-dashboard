## Summary

Keep open: current main and v2026.6.8 still do not implement a per-agent `subagents.workspace` override, and the current native spawn path still preserves the requester workspace for same-agent/default subagents. This remains a security-sensitive feature/config decision rather than conservative cleanup closure.

## What This Changes

Request to add an `agents.list[].subagents.workspace` config option so default child subagents can use an isolated workspace without separate child agent profiles.

## Best Possible Solution

Define one opt-in child-workspace isolation contract, then implement it consistently across config schema/types, native and ACP spawn resolution, docs, and security tests while preserving existing default inheritance unless maintainers choose a migration.

## Reproduction Assessment

Yes at source level: omitted `agentId` resolves to the requester agent, and same-agent spawns preserve `ctx.workspaceDir`; current tests assert that inherited requester workspace behavior. I did not run a live reproduction because this review was read-only.

## Solution Assessment

Unclear: a per-agent `subagents.workspace` override is plausible, but maintainers still need to choose the isolation boundary and precedence rules. The current supported workaround is explicit child agent profiles plus `agentId` and optional `requireAgentId`, which does not provide the requested default-subagent ergonomics.
