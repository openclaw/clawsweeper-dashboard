## Summary

Keep open: current main still omits the requested per-agent workspace override and preserves same-agent inherited workspace; related fixes cover parent inheritance, cross-agent target workspaces, or explicit cwd, not this new config surface.

## What This Changes

Request adds `agents.list[].subagents.workspace` so default child subagents can use an isolated workspace without separate agent profiles.

## Best Possible Solution

If maintainers want this isolation model, add a documented schema-backed override for omitted-`agentId` default spawns while preserving explicit target-agent workspace behavior and existing profile-based isolation.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection gives a high-confidence behavior check: the requested key is absent and omitted-`agentId` native spawns use the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product policy. The requested key is plausible, but current OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn behavior should be intentional.
