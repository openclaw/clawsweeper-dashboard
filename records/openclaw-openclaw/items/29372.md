## Summary

Keep open: current main still lacks the requested per-agent subagent workspace override, and default same-agent subagent spawns still preserve the requester workspace. Related merged work covers explicit target-agent workspaces and `requireAgentId`, but not this feature/config policy request.

## What This Changes

Add an `agents.list[].subagents.workspace` setting so default subagents spawned without `agentId` can use a configured child workspace instead of the parent agent workspace.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed per-agent override for omitted-`agentId` native subagent spawns, document it beside `requireAgentId`, and preserve explicit target-agent plus internal gateway workspace boundaries.

## Reproduction Assessment

Not applicable: this is a feature/config policy request rather than broken documented behavior. Source inspection gives high confidence current main lacks the requested key and that omitted-`agentId` same-agent spawns inherit the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested key is plausible, but current OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn workspace behavior should be intentional.
