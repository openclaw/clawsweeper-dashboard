## Summary

Keep open: current main and v2026.6.5 still lack a schema/runtime `subagents.workspace` contract. Existing work covers cross-agent target workspaces, per-call `cwd`, bootstrap-profile narrowing, and `requireAgentId`, but omitted-`agentId` same-agent spawns still inherit the requester workspace.

## What This Changes

The issue requests a per-agent `subagents.workspace` config option so default child subagents can use an isolated workspace instead of inheriting the parent workspace.

## Best Possible Solution

If maintainers approve the policy, add a strict schema-backed child workspace override with documented precedence and focused tests/docs for omitted-agent, explicit-agent, cwd, sandbox, attachments, and bootstrap behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a failing bug test, but source and existing tests clearly show the current behavior: omitted-agent same-agent spawns still inherit the requester workspace and the requested config key is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers decide the product and security policy; the requested knob is plausible, but existing supported paths are explicit child agent profiles, `requireAgentId`, and per-call `cwd`.
