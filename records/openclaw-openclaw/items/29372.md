## What This Changes

The issue requests `agents.list[].subagents.workspace` so default child subagents can use a configured isolated workspace instead of the parent agent workspace.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed trusted config override with documented precedence and focused regression coverage for default same-agent spawns, explicit `cwd`, cross-agent spawns, sandbox behavior, and docs.

## Reproduction Assessment

Not applicable: this is a feature/config-policy request rather than a broken documented behavior. Source inspection confirms the requested key is absent and omitted-agent same-agent spawns still use the requester workspace unless an explicit per-call `cwd` is supplied.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but explicit child agent profiles, `requireAgentId`, and per-call `cwd` are the current supported isolation paths, so a second persistent workspace source should be intentional.
