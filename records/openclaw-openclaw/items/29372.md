## Summary

Keep open: current main and the latest release still lack the requested per-agent `subagents.workspace` config contract. Recent shipped work mitigates bootstrap-file exposure and fixes cross-agent/explicit-`cwd` workspace routing, but it does not provide a persistent default-child workspace policy for omitted-`agentId` same-agent spawns.

## What This Changes

Requests an `agents.list[].subagents.workspace` config option so child subagents spawned without their own agent profile can default to an isolated workspace.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed trusted config override with documented precedence and focused coverage for same-agent default spawns, explicit `cwd`, cross-agent spawns, attachments, sandbox behavior, and docs.

## Reproduction Assessment

Not applicable: this is a feature/config-policy request rather than broken documented behavior. Source inspection confirms the requested key is absent and same-agent default spawns still inherit the requester workspace unless explicit per-call `cwd` is supplied.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but explicit child agent profiles, `requireAgentId`, and per-call `cwd` are the current supported isolation paths, so a second persistent workspace source should be intentional.
