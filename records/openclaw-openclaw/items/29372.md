## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` override, while same-agent/default subagent spawns still inherit the requester workspace. The related cross-agent workspace bug has been fixed separately, but this remaining request adds a security-sensitive public config policy and needs maintainer product review.

## What This Changes

Request adds an `agents.list[].subagents.workspace` config option so default child subagents can use a separate workspace from their parent.

## Best Possible Solution

If maintainers accept this isolation model, define its security semantics first, then add a schema-backed trusted config override beside `requireAgentId` while preserving explicit target-agent workspace behavior and internal-only workspace override boundaries.

## Reproduction Assessment

Not applicable: this is a feature/config policy request, not a broken documented behavior. Source inspection confirms current main lacks the requested key and that omitted-`agentId` same-agent spawns preserve the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but OpenClaw already supports isolation through explicit child agent profiles plus `requireAgentId`, so changing default-spawn workspace behavior should be intentional.
