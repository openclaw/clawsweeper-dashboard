## Summary

Keep open: current main and the latest release still do not implement a per-agent `subagents.workspace` override, and source plus focused tests show default same-agent subagents still inherit the requester workspace. The request is useful, but it creates a public config and security boundary that needs maintainer product and security review before implementation.

## What This Changes

The issue requests a per-agent `agents.list[].subagents.workspace` config option so default child subagents can use a separate workspace from the parent agent.

## Best Possible Solution

Define one opt-in child-workspace isolation contract, then implement it across config schema/types, spawn metadata resolution, docs, and focused tests while preserving existing default inheritance unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes for current behavior: source and focused tests show default same-agent subagents inherit the requester workspace. This is a feature request for a new isolation mode, not a failing existing contract.

## Solution Assessment

Unclear: a per-agent override is plausible, but maintainers need to define the exact isolation guarantee and precedence before this is the best solution.
