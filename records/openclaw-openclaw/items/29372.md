## Summary

Keep open for maintainer product and security review. Current main and v2026.6.5 still intentionally inherit the requester workspace for default same-agent children; adjacent fixes cover explicit target-agent selection and requiring `agentId`, but neither provides a persistent isolated workspace for default children.

## What This Changes

Requests a per-agent `subagents.workspace` setting so default child agents can use an isolated workspace instead of inheriting the parent workspace.

## Best Possible Solution

Adopt one canonical, security-reviewed child-workspace policy with explicit precedence and path validation, preserving target-agent workspaces for explicit spawns while clearly defining bootstrap, sandbox, `cwd`, fork, attachment, and migration behavior.

## Reproduction Assessment

Yes for the reported limitation: current source, regression tests, and documentation show that a default native child inherits its agent workspace and that `cwd` does not change bootstrap sourcing. This confirms current behavior rather than a violation of an existing documented contract.

## Solution Assessment

Unclear. The proposed setting is plausible, but maintainers must first decide whether a separate workspace is the canonical isolation model or whether explicit child profiles plus `requireAgentId` should remain the supported boundary.
