## Summary

Keep open: current main still lacks a per-agent `subagents.workspace` config contract and same-agent/default subagent spawns still inherit the requester workspace. This remains a security-sensitive feature and product decision, not conservative cleanup.

## What This Changes

Request adds `agents.list[].subagents.workspace` so child subagents spawned without `agentId` can run in a configured workspace instead of inheriting the parent workspace.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed trusted config override with explicit precedence while preserving target-agent workspaces, `requireAgentId`, sandbox checks, and internal-only workspace override boundaries.

## Reproduction Assessment

Not applicable: this is a feature/config policy request, not a broken documented behavior. Source inspection confirms the requested key is absent and omitted-`agentId` same-agent spawns still use the requester workspace path.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but explicit child agent profiles plus `requireAgentId` are the current isolation path, so a second workspace source should be intentional.
