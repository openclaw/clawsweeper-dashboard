## Summary

Keep open: current main fixed the related explicit cross-agent workspace inheritance path, but it does not implement the requested per-agent `subagents.workspace` override, and default same-agent sub-agent spawns still preserve the caller workspace.

## What This Changes

Requests an `agents.list[].subagents.workspace` option so default sub-agents spawned without `agentId` can use a separate workspace from the parent agent.

## Best Possible Solution

If maintainers want this isolation model, add a documented, schema-backed per-agent subagent workspace override that affects omitted-`agentId` default spawns while keeping explicit cross-agent spawns tied to the target agent workspace.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current behavior the request targets: default same-agent spawns keep the caller workspace, and the requested config key is not accepted or documented.

## Solution Assessment

Unclear until maintainer product direction is set. The requested key is a plausible narrow implementation, but the existing explicit target-agent profile plus `requireAgentId` path already provides workspace isolation without changing default-spawn semantics.
