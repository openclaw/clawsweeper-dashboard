## Summary

Keep open: current main still intentionally gives same-agent default subagents the parent agent’s workspace, while the closed cross-agent reports fixed target-agent selection rather than adding the requested child-workspace override. The remaining work is a security-sensitive public config and execution-boundary decision, not stale or already implemented.

## What This Changes

Add a per-agent `subagents.workspace` setting so default child agents can use a workspace distinct from their parent.

## Best Possible Solution

Define one coherent child execution-boundary contract: explicit target-agent spawns retain the target profile, while profile-free same-agent children may use an operator-selected child workspace that consistently controls bootstrap context, cwd, and sandbox mounts.

## Reproduction Assessment

Not applicable as a bug reproduction: source and focused tests confirm parent-workspace inheritance is current intended behavior for same-agent children, and this issue requests a new override.

## Solution Assessment

Unclear: the proposed field is understandable, but it is not the best solution unless bootstrap injection, tool cwd, filesystem access, and sandbox behavior all follow the same defined child boundary.
