## Summary

Keep open: current main and v2026.6.5 still do not provide the requested per-agent `subagents.workspace` config contract, and the remaining work is a security-sensitive config/product decision rather than a stale duplicate.

## What This Changes

Add an `agents.list[].subagents.workspace` config option so same-agent default child subagents can use an isolated workspace instead of inheriting the parent agent workspace.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed trusted config override with documented precedence and focused coverage for same-agent default spawns, explicit `cwd`, cross-agent spawns, sandbox behavior, bootstrap injection, and docs.

## Reproduction Assessment

Not applicable: this is a feature/config-policy request, not broken documented behavior. Source inspection gives high confidence that current main lacks the key and preserves same-agent requester workspace inheritance.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but current supported isolation paths are explicit child agent profiles, `requireAgentId`, and per-call `cwd`, so another persistent workspace source should be intentional.
