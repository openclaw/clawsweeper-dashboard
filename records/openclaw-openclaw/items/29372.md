## Summary

Keep open: current main and v2026.6.8 still do not implement a per-agent child-workspace override, and current source intentionally preserves inherited requester workspace for default same-agent subagents. This remains a useful but security-sensitive config/product decision, not a stale cleanup close.

## What This Changes

Request to add an `agents.list[].subagents.workspace` config option so default child subagents can use an isolated workspace instead of inheriting the parent agent workspace.

## Best Possible Solution

Define one opt-in child-workspace isolation contract, then implement it consistently across config schema/types, spawn metadata resolution, docs, and focused security tests while preserving existing default inheritance unless maintainers choose a migration.

## Reproduction Assessment

Yes at source level: omitted `agentId` resolves to the requester agent, and same-agent spawns preserve `ctx.workspaceDir`; current tests assert both omitted-agentId acceptance and inherited requester workspace. I did not run a live reproduction because this review was read-only.

## Solution Assessment

Unclear: a per-agent `subagents.workspace` override is plausible, but maintainers still need to choose the isolation boundary and precedence rules. The current supported workaround is explicit child agent profiles plus `agentId`, which does not provide the requested default-subagent ergonomics.
