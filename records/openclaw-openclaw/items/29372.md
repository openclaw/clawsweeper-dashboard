## Summary

Keep open: current main still lacks a per-agent `subagents.workspace` config contract, and the remaining request is a product/security decision about default child workspace isolation rather than an already-fixed bug.

## What This Changes

Add `agents.list[].subagents.workspace` so default child subagents spawned without `agentId` can use a configured child workspace instead of inheriting the parent workspace.

## Best Possible Solution

If maintainers accept this isolation model, add a trusted schema-backed override with documented precedence and regression coverage for same-agent defaults, explicit `cwd`, cross-agent spawns, attachments, sandbox behavior, and docs.

## Reproduction Assessment

Not applicable: this is a feature/config-policy request rather than a broken documented behavior. Source inspection confirms the proposed key is absent and omitted-agent same-agent spawns still inherit the requester workspace unless a per-call `cwd` is supplied.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but explicit child agent profiles, `requireAgentId`, and per-call `cwd` are the currently supported isolation paths.
