## Summary

Keep open: current main still has no `subagents.workspace` config contract, and same-agent/default subagent spawns still preserve the requester workspace. The request remains a new security-sensitive config/product decision, not an implemented bug fix.

## What This Changes

Add a per-agent `subagents.workspace` config option so default child subagents can use a separate workspace from their parent agent.

## Best Possible Solution

If maintainers accept this isolation model, define the security semantics first, then add a schema-backed trusted config override while preserving explicit target-agent workspace behavior, `requireAgentId`, and internal-only workspace override boundaries.
