- `impact:session-state`: Workspace selection is persisted into spawned run metadata and determines which bootstrap context child sessions receive.

## Summary

Keep open: current main and `v2026.6.8` still lack a per-agent `subagents.workspace` override, and same-agent default subagents still inherit the requester workspace. The request remains useful, but it needs maintainer product and security review because it adds a public config surface and an isolation promise around child workspace/bootstrap context.

## What This Changes

Requested behavior is a per-agent `subagents.workspace` config override so default subagents can use an isolated workspace instead of inheriting the parent agent workspace.

## Best Possible Solution

Define one opt-in child-workspace isolation contract, then implement it across config schema/types, spawn metadata resolution, docs, and focused tests while preserving existing default inheritance unless maintainers intentionally choose a migration.

## Reproduction Assessment

Yes for current behavior at source level: current source and focused tests show default same-agent subagents inherit the requester workspace. This is a feature request for a new isolation mode, not a failing existing contract.

## Solution Assessment

Unclear: a per-agent workspace override is plausible, but maintainers need to define the exact isolation guarantee and precedence before it is the best solution. The current supported workaround is explicit child agent profiles plus `agentId`, which does not satisfy the requested ergonomics.
