## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` config contract, and default same-agent subagent spawns still inherit the requester workspace. This is a security-sensitive feature/config decision, not conservative cleanup.

## What This Changes

Request adds an `agents.list[].subagents.workspace` config option so child subagents spawned without `agentId` can use an isolated workspace instead of inheriting the parent workspace.

## Best Possible Solution

If maintainers accept this isolation model, implement a schema-backed trusted config override with explicit precedence and regression coverage across config validation, same-agent/default spawns, target-agent workspaces, sandbox guards, and docs.

## Reproduction Assessment

Not applicable: this is a feature/config policy request, not a broken documented behavior. Source inspection confirms the requested key is absent and omitted-`agentId` same-agent spawns still use the requester workspace path.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but explicit child agent profiles plus `requireAgentId` are the current isolation path, so a second workspace source should be intentional.
