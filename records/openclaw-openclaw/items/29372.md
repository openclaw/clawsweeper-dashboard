## Summary

Keep open. Current main still lacks `agents.list[].subagents.workspace` in the per-agent config contract, and native same-agent/default subagent spawns still preserve the requester workspace; the merged cross-agent workspace fix addresses a different target-agent scenario.

## What This Changes

Request adds an `agents.list[].subagents.workspace` config option so default child sub-agents can use a parent-configured workspace instead of inheriting the parent workspace.

## Best Possible Solution

If approved, add a backwards-compatible `agents.list[].subagents.workspace` key, normalize it like other workspace paths, update schemas/docs/generated config surfaces, and apply it to same-agent/default workspace resolution without regressing cross-agent target workspace behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: the current strict config schemas do not accept the requested key, and the same-agent spawn path/test still use the requester workspace.

## Solution Assessment

Unclear. The requested key would address the isolation use case, but changing workspace inheritance is a config and security contract change that should be approved before implementation.
