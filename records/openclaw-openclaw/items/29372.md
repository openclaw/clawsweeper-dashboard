## Summary

Keep open: current main still has no per-agent `subagents.workspace` config and default same-agent subagent spawns continue to inherit the requester workspace; related merged work fixes cross-agent target workspaces and internal override hardening, not this new config surface.

## What This Changes

The issue requests a schema-backed `agents.list[].subagents.workspace` option so default subagents can run from a workspace separate from their parent agent.

## Best Possible Solution

If maintainers want this isolation model, add a documented schema-backed override for omitted-`agentId` native spawns while preserving explicit target-agent workspace behavior and internal-only workspace override hardening.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken documented behavior. Source inspection gives a high-confidence behavior check that the requested key is absent and omitted-`agentId` spawns use the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product policy. The requested key is plausible, but current OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn behavior should be intentional.
