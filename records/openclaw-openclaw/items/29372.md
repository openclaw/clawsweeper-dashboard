## Summary

Keep open. Current main still does not implement `agents.list[].subagents.workspace`: the per-agent config type and strict schemas omit the key, and default/same-agent native subagent spawning still preserves the requester workspace. The related cross-agent workspace fix is real but addresses a different target-agent inheritance bug, not this parent-configured default-subagent override.

## What This Changes

Add `agents.list[].subagents.workspace` so a parent agent can place default child subagents in a separate workspace without creating separate agent profiles.

## Best Possible Solution

If maintainers approve the product/security direction, add a backwards-compatible per-agent subagent workspace field, normalize it like other workspace paths, update runtime and generated schemas plus public docs, and make omitted-`agentId` plus intended same-agent native spawns prefer that override while preserving current cross-agent target-workspace behavior.
