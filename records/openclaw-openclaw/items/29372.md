## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` override, and same-agent/default native subagent spawns still preserve the requester workspace. Related merged work covers explicit target-agent workspaces and `requireAgentId`, but this remaining request is a security-sensitive product/config decision rather than shipped behavior.

## What This Changes

Requested a per-agent `agents.list[].subagents.workspace` config option so default child subagents can use a separate workspace without separate agent profiles.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed trusted-config override for omitted-`agentId` native subagent spawns, document it beside `requireAgentId`, and preserve explicit target-agent plus internal gateway workspace boundaries.

## Reproduction Assessment
