## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` config contract, and same-agent/default subagent spawns still preserve the requester workspace. This is a clear feature/config and security-policy decision, not conservative cleanup.

## What This Changes

The issue requests an `agents.list[].subagents.workspace` config option so default child subagents can use a workspace different from the parent agent.

## Best Possible Solution

If maintainers accept this isolation model, add a trusted, schema-backed child-workspace override with explicit precedence and coverage for default same-agent spawns, explicit target-agent spawns, `cwd`, sandbox guards, docs, and config validation.

## Reproduction Assessment

Not applicable: this is a feature/config policy request, not a broken documented behavior. Source inspection confirms the requested key is absent and current default same-agent spawns still inherit the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but explicit child agent profiles plus `requireAgentId` are the current isolation path, so a second workspace source should be intentional.
