## Summary

Keep open: current main and the latest release still do not implement a per-agent `subagents.workspace` override, while current source and tests preserve same-agent subagent inheritance from the requester workspace. The request is useful but needs maintainer product and security review because it adds a public config surface for child workspace and bootstrap-context isolation.

## What This Changes

The issue requests an `agents.list[].subagents.workspace` option so default child subagents can use an isolated workspace without separate agent profiles.

## Best Possible Solution

Define one opt-in child-workspace isolation contract, then implement it across config schema/types, spawn metadata resolution, docs, and focused tests while preserving existing default inheritance unless maintainers intentionally choose a migration.

## Reproduction Assessment

Yes for current behavior: current source and focused tests show default same-agent subagents inherit the requester workspace. This is a feature request for a new isolation mode, not a failing existing contract.

## Solution Assessment

Unclear: a per-agent workspace override is plausible, but maintainers need to define the exact isolation guarantee and precedence before it is the best solution. The current supported workaround is explicit child agent profiles plus `agentId`, which does not satisfy the requested ergonomics.
