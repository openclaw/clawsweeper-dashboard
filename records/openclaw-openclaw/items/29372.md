## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` key, and default same-agent subagent spawns continue to preserve the requester workspace. The related cross-agent workspace fix solved a narrower bug, not this feature request.

## What This Changes

Add an `agents.list[].subagents.workspace` config option so default subagents spawned without `agentId` can use a workspace different from their parent agent.

## Best Possible Solution

If maintainers want this isolation model, add a documented, schema-backed per-agent override that applies to omitted-`agentId` default spawns while preserving explicit cross-agent target workspace behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence behavior check: current main omits/rejects the requested config key and same-agent default spawns keep the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product policy. The requested key is plausible, but current OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn behavior should be intentional.
