## Summary

Keep open: current main and latest release still lack the requested per-agent `subagents.workspace` config contract. Existing work covers adjacent mitigations such as `requireAgentId`, target-agent workspaces for explicit cross-agent spawns, per-call `cwd`, and narrower bootstrap injection, but not a persistent same-agent default-child workspace policy.

## What This Changes

The issue requests an `agents.list[].subagents.workspace` config option so default child subagents can use a different workspace from their parent agent.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed trusted config override with documented precedence and focused coverage for same-agent default spawns, explicit `cwd`, cross-agent spawns, attachments, sandbox behavior, and docs.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested knob is plausible, but current supported isolation paths are explicit child agent profiles, `requireAgentId`, and per-call `cwd`, so a second persistent workspace source should be intentional.
