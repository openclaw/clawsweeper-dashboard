## Summary

Keep open: current main still does not support `agents.list[].subagents.workspace`, and omitted-`agentId` native subagent spawns still use the requester/same-agent workspace. This is a security-sensitive feature/config policy request, not an already-fixed bug cleanup.

## What This Changes

Request a per-agent `agents.list[].subagents.workspace` config override for default child subagents spawned without `agentId`.

## Best Possible Solution

If maintainers want this isolation model, add a schema-backed per-agent override for omitted-`agentId` native spawns, with docs and tests that preserve explicit target-agent workspace behavior and internal-only workspace override hardening.

## Reproduction Assessment

Not applicable: this is a feature/config policy request rather than broken documented behavior. Source inspection gives a high-confidence current-behavior check that the requested key is absent and omitted-`agentId` native spawns use same-agent/requester workspace inheritance.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested key is plausible, but OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn workspace behavior should be intentional.
