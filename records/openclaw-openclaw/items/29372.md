## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` config, and the existing same-agent/default spawn path intentionally preserves the caller workspace. The related cross-agent workspace fix solved a narrower bug but did not implement this feature.

## What This Changes

Add an `agents.list[].subagents.workspace` config option so default child subagents can use a workspace different from the parent agent workspace.

## Best Possible Solution

If maintainers want this isolation model, add a documented, schema-backed per-agent subagent workspace override for omitted-`agentId` default spawns while preserving explicit cross-agent spawns as target-agent-profile based.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence behavior check: current main rejects/omits the requested config key and same-agent subagent spawns keep the caller workspace.

## Solution Assessment

Unclear until maintainer product direction is set. The proposed key is a plausible narrow design, but current OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn behavior needs policy approval.
