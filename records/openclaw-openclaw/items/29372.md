## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` override, and omitted-`agentId` native subagent spawns still use the requester agent/workspace path. Related merged work covers explicit target-agent workspaces and `requireAgentId`, but this remains a security-sensitive product/config decision rather than a shipped behavior.

## What This Changes

The issue requests `agents.list[].subagents.workspace` so default subagents can use a separate workspace without defining a separate agent profile.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed trusted-config override for omitted-`agentId` native subagent spawns, document it beside `requireAgentId`, and preserve explicit target-agent plus internal gateway workspace boundaries.

## Reproduction Assessment

Not applicable: this is a feature/config policy request rather than broken documented behavior. Source inspection gives high confidence current main lacks the requested key and that omitted-`agentId` same-agent spawns preserve the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested key is plausible, but current OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn workspace behavior should be intentional.
