## Summary

Keep open: current main still lacks the requested per-agent `subagents.workspace` config, and omitted-`agentId` native subagent spawns still inherit the requester workspace. Related work fixed explicit target-agent workspace routing and added `requireAgentId`, but this remains a separate feature and security-policy decision.

## What This Changes

The issue requests a per-agent `subagents.workspace` setting so default child subagents can use a workspace different from the parent agent.

## Best Possible Solution

If maintainers accept this isolation model, add a trusted config-only per-agent override for omitted-`agentId` native subagent spawns, document it beside `requireAgentId`, and preserve explicit target-agent plus internal gateway workspace boundaries.

## Reproduction Assessment

Not applicable as a bug: this is a feature/config policy request rather than broken documented behavior. Source inspection gives high confidence current main lacks the requested key and that omitted-`agentId` same-agent spawns inherit the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested key is plausible, but OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn workspace behavior should be intentional.
