## Summary

Keep open: current main and the latest release still make default same-agent subagents inherit the requester workspace, and the only related merged fix covered explicit cross-agent target workspace selection. The request remains useful, but it adds a config/security boundary that needs maintainer product and security decisions before implementation.

## What This Changes

Request to add `agents.list[].subagents.workspace` so same-agent/default subagents can use a child workspace instead of inheriting the parent agent workspace.

## Best Possible Solution

Maintainers should define one opt-in child-workspace contract that states what is isolated, how it composes with `agentId`, sandboxing, cwd, and bootstrap files, then implement it with focused schema, docs, and cross-path tests.

## Reproduction Assessment

Yes for current behavior: source and focused tests show default same-agent subagents inherit the requester workspace. This is a feature request for a new isolation mode, not a failing existing contract.

## Solution Assessment

Unclear: a per-agent override is plausible, but maintainers need to decide the exact isolation guarantee and precedence before it is the best solution.
