## Summary

Keep open: current main still has no `agents.list[].subagents.workspace` config, and same-agent/default subagent spawns still inherit the requester workspace. This is a security-sensitive feature/config policy request, not an already-implemented cleanup close.

## What This Changes

The issue requests a per-agent `subagents.workspace` config option so child subagents spawned without `agentId` can use an isolated workspace instead of the parent agent workspace.

## Best Possible Solution

If maintainers accept this isolation model, add a schema-backed per-agent override for omitted-`agentId` native subagent spawns, document it beside `requireAgentId`, and preserve explicit target-agent plus internal-only gateway workspace boundaries.

## Reproduction Assessment

Not applicable: this is a feature/config policy request rather than broken documented behavior. Source inspection gives high confidence that current main lacks the requested key and that omitted-`agentId` same-agent spawns inherit the requester workspace.

## Solution Assessment

Unclear until maintainers decide the product and security policy. The requested key is plausible, but current OpenClaw already supports isolation through explicit target agent profiles plus `requireAgentId`, so changing default-spawn workspace behavior should be intentional.
