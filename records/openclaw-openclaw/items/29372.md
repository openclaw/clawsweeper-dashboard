## Summary

Current source and tests confirm that default same-agent subagents still inherit the parent workspace, while the closed cross-agent work only corrected explicit target-agent workspace selection. The request remains useful, but it introduces a public config and security boundary whose exact promise and precedence need maintainer product and security judgment.

## What This Changes

Add a per-agent `subagents.workspace` option so default child subagents can use a workspace distinct from their parent agent.

## Best Possible Solution

Define one canonical workspace-precedence contract that states whether the override controls bootstrap files, process working directory, sandbox source, and tool-visible filesystem access, then apply it consistently with same-agent, cross-agent, resume, and isolation tests.
