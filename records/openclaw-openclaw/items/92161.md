The branch threads gateway diagnostic trace contexts through HTTP/WebSocket handling into embedded agent invocations, adds `run.invocation` diagnostics events, records diagnostics-otel invocation spans, and adds focused tests.

## Best Possible Solution

Keep the trace-chaining design, but emit invocation completion only after final terminal outcome classification and require redacted runtime trace proof before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: PR head emits a successful invocation completion before current-main post-compaction abort handling can throw. No live gateway reproduction was provided or run in this read-only review.

## Solution Assessment

No. The trace boundary is reasonable, but completion emission must happen after final post-compaction abort classification so diagnostics-otel does not close invocation spans as successful for aborted runs.
