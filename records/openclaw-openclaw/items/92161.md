## Summary

Keep open: this is still useful diagnostics work and is not superseded on current main, but PR head is not merge-ready because invocation completion can be emitted as completed before post-compaction abort handling, and the external PR still lacks real gateway trace proof.

## What This Changes

The PR chains gateway diagnostic trace contexts through HTTP/WebSocket handling into embedded agent invocations, adds `run.invocation` diagnostics events, records diagnostics-otel invocation spans, and adds focused tests.

## Best Possible Solution

Keep the trace-chaining design, but emit invocation completion only after final terminal outcome classification or emit an error completion for post-compaction aborts, then require redacted real gateway trace proof before merge.

No. The trace boundary is plausible, but invocation completion must be tied to the final terminal outcome before this is the best implementation.
