## What This Changes

## Best Possible Solution

Carry the latest nonzero provider usage separately from accumulated attempt usage, preserve the existing latest-call accounting invariant, and prove the actual embedded `--local --json` path with redacted real or loopback provider output.

Yes at source level: current tests document that attemptUsage can be accumulated while metadata total must stay latest-turn, and the PR fallback uses attemptUsage when last assistant usage is zero. I did not run a live provider in this read-only review.

## Solution Assessment
