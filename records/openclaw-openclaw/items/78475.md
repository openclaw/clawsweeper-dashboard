## Summary

This is expected group visible-reply policy rather than a Feishu card dispatcher bug: group/channel final replies are private by default unless the agent uses the message tool or config restores automatic delivery, and current docs/code/tests cover the no-visible-reply path.

## What This Changes

The issue asks for Feishu group @mention replies from the main bot to be delivered automatically when the model produces final text.

## Best Possible Solution

Keep the current source-reply policy and use the documented message tool or messages.groupChat.visibleReplies: "automatic" setting; reopen as a code bug only with proof that automatic visible replies still fail.

## Reproduction Assessment

Yes, from source inspection: a Feishu group turn goes through shared dispatchReplyFromConfig, and group/channel source delivery defaults to message_tool_only, so a normal final answer can produce queuedFinal=false/replies=0 when no message-tool send occurs.

## Solution Assessment

No core code fix is indicated for the supplied logs; the documented resolution is to use a tool-calling model/message(action=send) or set messages.groupChat.visibleReplies to automatic for legacy auto-posting.
