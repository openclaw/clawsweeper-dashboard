## Summary

Keep open. The latest diff still restores deferred entries only after the real reply pipeline can drain the session system-event queue, so the central production bug remains despite the useful FIFO, duplicate, changelog, and rebase follow-ups.

## What This Changes

The PR adds a system-event restore helper, wires exec-heartbeat cleanup to requeue consumed non-exec events, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Move the defer/restore boundary ahead of the real reply-path system-event drain, then restore deferred non-exec entries after success with coverage that exercises the actual drain behavior.

## Reproduction Assessment

Yes. Queue a non-exec reminder and an exec completion for the same session, then run an exec-event heartbeat through the real reply path; `drainFormattedSystemEvents` drains the whole queue before the PR's cleanup can restore anything.

## Solution Assessment

No. The current PR direction is too late in the control flow; the safer fix is to partition or temporarily remove deferred entries before `getReplyFromConfig` can drain them, or restore from the preflight snapshot while accounting for that drain.

## Review Findings
