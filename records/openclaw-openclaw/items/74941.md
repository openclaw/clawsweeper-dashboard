## Summary

Keep open. The PR is aimed at a valid heartbeat/system-event queue problem, but the restore runs after the production reply path has already drained queued system events, so the deferred reminder can still be mixed into the exec heartbeat model turn and not restored.

## What This Changes

The PR adds a system-event restore helper, updates exec-heartbeat cleanup to restore consumed non-exec entries, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Move the defer/restore boundary so non-exec inspected entries are kept out of the production reply drain and restored afterward, with regression coverage that observes the actual command body or queue drain behavior.

## Reproduction Assessment

Yes. Queue a non-exec reminder and an exec completion for the same session, then run an exec-event heartbeat through the real reply path: get-reply-run drains the whole queue before heartbeat cleanup, while the PR’s restore path only runs after that drain.

## Solution Assessment

No. Restoring only after getReplyFromConfig returns is too late for production; the narrower maintainable fix is to partition or temporarily remove deferred entries before the reply pipeline can drain them, then restore the deferred subset after success and on retry-safe exits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Restore events before the reply path drains them:** `src/infra/heartbeat-runner.ts:1247-1249`
  - body: In production, `getReplyFromConfig` builds the model prompt by calling `drainFormattedSystemEvents`, which drains this session's queued system events before `consumeInspectedSystemEvents` runs. When that happens, this new post-reply `consumeSystemEventEntries(...)` returns `[]`, so the deferred non-exec entry has already been mixed into the exec heartbeat turn and is not restored. Please move the partition/defer step ahead of the reply pipeline, or otherwise restore from the preflight snapshot after accounting for that drain.
