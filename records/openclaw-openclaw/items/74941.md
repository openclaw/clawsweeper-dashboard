## What This Changes

The PR adds a system-event restore helper, updates heartbeat cleanup to restore consumed non-exec deferred events after exec-event runs, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land the PR after exact-head CI or the targeted infra tests plus changed gate pass, preserving the consumed-prefix restore semantics and the regression tests that cover duplicates and tail ordering.

## Reproduction Assessment

Yes. On current main, queue a non-exec reminder before an exec completion for the same session, run an exec-event heartbeat, and the prompt filters to the exec entry while cleanup consumes the whole inspected prefix, dropping the reminder.

## Solution Assessment

Yes. Restoring cloned, actually consumed non-exec entries immediately after the prefix consume is the narrow maintainable fix because it avoids enqueue-time duplicate suppression and keeps older deferred entries ahead of newer tail events.
