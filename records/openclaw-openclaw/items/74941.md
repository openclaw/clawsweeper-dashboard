## Summary

Keep open. Current main still has the dropped-deferred-event behavior, and this PR is a narrow, relevant fix with focused regression coverage and no blocking review finding from the diff inspection.

## What This Changes

This PR adds a system-event restore helper, uses it during exec-event heartbeat cleanup to requeue consumed non-exec entries, adds regression tests for mixed/deferred queue behavior, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the focused queue-preservation fix after required checks are green; keep the restore helper, regression tests, and changelog entry because they directly cover the mixed exec/non-exec, duplicate reminder, and newer-tail ordering cases.

## Reproduction Assessment

Yes. On current main, queue a non-exec reminder before an exec completion and run an exec-event heartbeat: the prompt is built only from the exec event, then cleanup consumes the whole inspected prefix and removes the reminder.

## Solution Assessment

Yes. Restoring cloned, actually consumed non-exec entries ahead of any newer tail is the narrow maintainable fix, and it avoids sending restored reminders back through enqueue-time duplicate suppression.
