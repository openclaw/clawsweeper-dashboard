## Summary

Keep open. Current main still consumes the full inspected system-event prefix after an exec-event heartbeat even though the prompt only includes exec events, while this PR adds a narrow restore path with focused regression coverage; I found no blocking bug in the diff.

## What This Changes

The PR adds a system-event restore helper, uses it after exec-event heartbeat cleanup to requeue consumed non-exec entries, adds regression tests, and records a changelog entry.

## Best Possible Solution

Land this PR after normal maintainer review, preserving the restore-after-consume semantics and the regression coverage for duplicate and tail-order deferred events.

## Reproduction Assessment

Yes. On current main, queue a non-exec reminder before an exec completion for the same session and run an exec-event heartbeat: the prompt sees only the exec entry, then cleanup drains the whole inspected prefix.

## Solution Assessment

Yes. Restoring only actually consumed non-exec entries directly into the session queue is the narrow maintainable fix, avoids enqueue-time duplicate suppression, and keeps restored entries ahead of newer tail events.
