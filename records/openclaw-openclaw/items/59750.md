## Summary

Keep open. Current main still preserves and applies session-stored model overrides on reused sessions, and config/docs expose no resume strategy for preferring the current default; #28965 overlaps but tracks broader reconcile/reset work rather than this exact resume policy.

## What This Changes

Request to add a configurable model resume strategy so resumed sessions can either keep the saved session model or use the current default model.

## Best Possible Solution

If maintainers accept the product direction, add a documented strict-schema option that defaults to current session-continuity behavior and threads through chat, Gateway, CLI/ACP resume, sessions.patch/status, and tests with clear handling for inherited and stored overrides.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature proposal. Static verification is high-confidence: source and tests show current main resolves an existing session modelOverride over the configured default.

## Solution Assessment

Unclear until maintainer product review. A session-vs-default strategy is backward-compatible in shape, but the config path and interaction with stored overrides should be decided before an automated implementation PR is queued.
