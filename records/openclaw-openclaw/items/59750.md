## Summary

Keep open. Current main still prioritizes persisted session model overrides over the configured default, no strict config/docs field exposes a resume strategy, and the related #28965 is a broader reconcile/reset proposal rather than a replacement for this narrower public config contract.

## What This Changes

The issue requests a configurable resumed-session model policy that can either keep the saved session model or prefer the current default model.

## Best Possible Solution

If maintainers accept the direction, add a strict documented session-level resume policy that defaults to current session continuity and threads through chat resume, parent overrides, Gateway/session patch surfaces, CLI/ACP resume behavior, status display, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature proposal. Static verification is high-confidence: source and tests show current main resolves persisted session overrides over the configured default.

## Solution Assessment

Unclear until maintainer product/API review. A backward-compatible session-vs-default strategy is plausible, but the config path and stored-override semantics should be chosen before implementation.
