## Summary

Keep open. Current main still restores and resolves persisted session model overrides ahead of the configured default, no strict config/docs surface exposes a resume strategy, and the related broader model-reconcile proposal does not replace this narrower public config decision.

## What This Changes

Add a configurable resume-time model-selection policy so resumed sessions can either keep the persisted session model or use the current default model.

## Best Possible Solution

If maintainers accept the direction, add a strict documented resume model policy that defaults to current session continuity and is wired through chat resume, parent inheritance, Gateway/ACP/session patch surfaces, status display, and focused tests.

## Reproduction Assessment

Not applicable: this is a feature/config proposal, not a broken existing behavior report. Static inspection is high-confidence that current main preserves stored session overrides over configured defaults.

## Solution Assessment

Unclear until maintainer product/API review. A backward-compatible session-vs-default policy is plausible, but the config path and override semantics should be decided before implementation.
