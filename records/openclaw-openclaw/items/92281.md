## What This Changes

The PR makes Telegram isolated-ingress replay retain spool ownership through immediate or buffered final dispatch, requeues retryable failures, and adds a dedupe-forget API plus regression coverage.

## Best Possible Solution

Keep one explicit Telegram update-outcome path through middleware and buffered work, retain failed updates for retry, remove only the corresponding dispatch-dedupe records, and land the public SDK method only after owner approval and rebased lifecycle validation.
