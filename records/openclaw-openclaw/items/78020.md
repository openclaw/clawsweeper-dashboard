## Summary

Keep open: this targets a real Telegram progress leak, but the current PR removes documented tool-progress previews globally, adds forbidden core-internal imports from the Telegram plugin, and still lacks required real Telegram proof.

## What This Changes

The PR removes Telegram progress-preview draft updates, adds Telegram typing/run-activity filtering through core auto-reply helpers, propagates tool/run metadata through agent reply paths, and updates Telegram/agent tests.

## Best Possible Solution

Keep documented Telegram tool-progress previews, suppress only internal/background/yield-wait cues, and expose any needed typing/progress classification through plugin-safe SDK or local Telegram seams.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main includes the `Pinching...` progress label and Telegram progress callback wiring can render tool/progress preview drafts when preview streaming is active; I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

No. Disabling all Telegram progress-preview drafts is not the best fix because current docs and released behavior preserve configurable visible tool-progress previews; the safer direction is to filter internal activity and route typing policy through a plugin-safe seam.
