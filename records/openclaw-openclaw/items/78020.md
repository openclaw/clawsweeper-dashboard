## Summary

Keep open: the PR targets a real Telegram progress-leak problem, but the diff overcorrects by removing documented tool-progress previews, introduces plugin-boundary violations, and lacks required real Telegram behavior proof.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible: current main can render `Pinching...` from the shared progress label list and routes Telegram tool/progress callbacks into preview drafts when preview streaming is active. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

No. The safest fix is to classify and filter internal run activity while keeping the existing `streaming.preview.toolProgress` and `progress` behavior, and to route typing policy through a plugin-safe seam instead of core-internal imports.
