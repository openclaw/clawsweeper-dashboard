## Best Possible Solution

Yes. Source-level reproduction is high-confidence: current main includes the `Pinching...` label and Telegram progress callback wiring can render tool/progress preview drafts when preview streaming is active; I did not run a live Telegram Bot API scenario in this read-only review.

No. Disabling every Telegram progress-preview draft is not the best fix because current docs and released behavior preserve configurable visible tool-progress previews; the safer direction is to filter internal activity and route typing policy through a plugin-safe seam.
