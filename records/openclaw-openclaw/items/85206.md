## Summary

Keep open: current main still leaves Control UI bundling outside the full build-all profiles, and an open linked PR already targets this issue with closing linkage, so this issue should stay open until that PR is merged or explicitly rejected.

## What This Changes

Request that full source/runtime builds run the Control UI build after runtime dist cleanup so `pnpm build` leaves `dist/control-ui/index.html` present.

## Best Possible Solution

Review and either land or reject the linked PR; if landing, keep the change limited to full/CI artifact profiles after tsdown cleanup with focused build-all regression coverage and aligned docs/hints.

## Reproduction Assessment

Yes. The source path is high-confidence: `pnpm build` resolves to build-all, build-all omits `ui:build`, tsdown removes `dist`, and Vite is the path that writes `dist/control-ui`; I did not run the destructive build in this read-only checkout.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix if it adds `ui:build` only to full/CI artifact profiles after runtime cleanup and preserves minimal watch/startup profiles.
