## Label Justifications

- `P2`: This is a concrete build/runtime artifact bug with limited blast radius but visible Control UI availability impact for source and automation workflows.
- `impact:crash-loop`: The reported failure leaves the gateway backend live while the Control UI route can return a 503 or delay readiness during startup repair.

## Summary

Keep open: current main and the latest release still omit `ui:build` from the full and CI artifact build-all profiles, while the runtime build deletes `dist` and the gateway serves Control UI from `dist/control-ui`. The issue is already paired with the open closing PR https://github.com/openclaw/openclaw/pull/85212, so the right cleanup is to review that PR rather than close this issue early.

## What This Changes

Request that full source/runtime builds generate `dist/control-ui` after backend dist cleanup instead of relying on a separately remembered `pnpm ui:build`.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/85212 so full and CI artifact builds run `ui:build` after runtime cleanup while `gatewayWatch` and `cliStartup` remain backend-only.

## Reproduction Assessment

Yes. The source path is high-confidence: `pnpm build` maps to build-all, build-all lacks `ui:build`, tsdown removes `dist`, and Vite is the path that writes `dist/control-ui`; I did not run the destructive build in this read-only checkout.

## Solution Assessment

Yes. Adding `ui:build` only to full and CI artifact profiles after the tsdown/runtime cleanup is the narrow maintainable fix, with startup repair and docs-only guidance left as fallback behavior.
