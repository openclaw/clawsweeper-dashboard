## Summary

Keep open: current main and v2026.5.20 still leave Control UI output as a separate manual build while the runtime build cleans `dist`, so the source/runtime artifact gap remains source-reproducible.

## What This Changes

The issue asks for full runtime build profiles to run the Control UI build so `pnpm build` leaves `dist/control-ui` populated after runtime dist cleanup.

## Best Possible Solution

Add a Control UI build step after the runtime dist-clean/build steps for full and CI artifact profiles, keep minimal dev profiles backend-only, and update focused tests plus docs/release hints to match the new full local dist contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `pnpm build` maps to build-all, build-all lacks `ui:build`, tsdown cleans `dist`, and Vite is the shown path that writes `dist/control-ui`.

## Solution Assessment

Yes. The narrow maintainable fix is to add `ui:build` only to full/CI artifact profiles after runtime cleanup, with tests proving ordering and that minimal dev profiles stay fast.
