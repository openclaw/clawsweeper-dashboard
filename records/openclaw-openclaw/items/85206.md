## Summary

Keep open: current main still leaves the full source/runtime build contract incomplete, and this issue is paired with an open closing PR that should be reviewed or landed before the issue is closed.

## What This Changes

The issue asks for `pnpm build` and the full/CI artifact build pipeline to produce `dist/control-ui/index.html` instead of relying on a separate documented `pnpm ui:build` step.

## Best Possible Solution

Review and either land or reject the linked build-script PR, with full and CI artifact builds producing `dist/control-ui` while minimal watch/startup profiles stay backend-only.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `pnpm build` maps to build-all, build-all lacks `ui:build`, tsdown removes `dist`, and the gateway serves an error when `dist/control-ui/index.html` is absent.

## Solution Assessment

Yes. The narrow maintainable solution is to add the UI build to full and CI artifact build profiles after the runtime dist-clean/build step, with regression coverage for ordering and no change to minimal watch/startup profiles.
