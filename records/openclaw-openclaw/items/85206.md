## Summary

Keep open: current main still lets `pnpm build` rebuild backend runtime output without rebuilding Control UI assets, and the linked closing PR is still open and should be reviewed before this issue closes.

## What This Changes

The issue asks for full source/runtime builds to produce `dist/control-ui` by running `ui:build` after backend dist cleanup in the full and CI artifact build profiles.

## Best Possible Solution

Review and land or reject the linked fix PR; if accepted, it should add `ui:build` only to full/CI artifact profiles after dist cleanup, preserve backend-only fast profiles, and update focused tests/docs.

## Reproduction Assessment

Yes. The source path is high-confidence: `pnpm build` selects `scripts/build-all.mjs`, build-all omits `ui:build`, tsdown cleans `dist`, and Vite writes Control UI assets to `dist/control-ui`.

## Solution Assessment

Yes, if maintainers accept the build-contract change. Adding `ui:build` after backend cleanup for full/CI artifact profiles is the narrow maintainable fix, while `gatewayWatch` and CLI-startup profiles should remain backend-only.
