## Summary

Keep open: current main and the latest release still treat Control UI output as a separate manual build while the runtime build removes the dist tree and the gateway later depends on dist/control-ui to serve the dashboard.

## What This Changes

Request that the full source/runtime build pipeline include Control UI asset generation so `pnpm build` leaves `dist/control-ui` present.

## Best Possible Solution

Add `ui:build` after the runtime dist-clean/build steps for full and CI artifact builds, keep minimal dev profiles backend-only, and align focused tests plus source-build docs/hints with the new contract.

## Reproduction Assessment

Yes, source-reproducible with high confidence: `pnpm build` maps to build-all, build-all lacks `ui:build`, tsdown-build removes `dist`, and Vite is the only path shown writing `dist/control-ui`. I did not run the destructive build in this read-only checkout.

## Solution Assessment

Yes, the issue's proposed direction is the narrow maintainable fix if it adds the UI build only to full/CI artifact profiles and updates the build-all regression test. Startup auto-repair and docs-only instructions should remain fallback guidance, not the primary artifact contract.
