## Summary

Keep this PR open for normal maintainer review: the patch is a focused WhatsApp bug fix and the code path supports the intended typing suppression, but the documented change also removes the typing flash from rare proactive unmentioned visible replies in automatic group mode.

## What This Changes

The PR widens WhatsApp group `suppressTyping` from message-tool-only unmentioned group turns to all unmentioned non-command group turns, and updates the automatic-delivery dispatch test expectation.

## Best Possible Solution

Land this shape only if maintainers accept the WhatsApp-specific UX tradeoff; otherwise replace it with a narrower pipeline-level fix that suppresses composing only when the run is known to be silent.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes for the vulnerable path: current main leaves automatic unmentioned group runs unsuppressed, and renderable pre-silence text can start composing before the silent reply decision. Live before-state reproduction is only medium confidence because the PR reports the provider checkpoint no longer emits the same stream shape today.

## Solution Assessment

Is this the best way to solve the issue? It is an acceptable narrow mitigation, not the structural best fix; the best complete fix would unify or coordinate the eager typing tap with the lazy delivery decision, but this patch is small and channel-scoped.
