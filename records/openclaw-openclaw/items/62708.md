## Summary

Keep open. Current main still treats `claudeCliSessionId` as an intentional legacy compatibility field for persisted Claude CLI session rows, so deleting it is a cleanup that needs an explicit compatibility decision rather than conservative auto-close.

## What This Changes

Remove the legacy `claudeCliSessionId` field from `SessionEntry` and the related fallback, dual-write, passthrough, and tests.

## Best Possible Solution

Make an explicit compatibility decision for legacy Claude-only session rows; if they can be retired, remove the type field, fallback, dual writes, preservation/reset handling, and tests together, otherwise keep the compatibility path and document the retention plan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a cleanup request. Static inspection gives high-confidence evidence that current main still actively supports legacy Claude CLI session rows.

## Solution Assessment

Unclear until maintainers decide the compatibility policy. The proposed deletion is narrow only if OpenClaw is ready to stop supporting persisted rows that predate the provider-keyed binding fields.
