## Summary

Keep open. Current main and the latest release still treat `claudeCliSessionId` as an active legacy compatibility field for persisted Claude CLI session rows, so removal is a compatibility cleanup that needs maintainer judgment.

## What This Changes

The issue requests removing the legacy `claudeCliSessionId` field and all related read, write, preservation, and test paths from the persisted session entry contract.

## Best Possible Solution

Make an explicit compatibility decision for legacy Claude-only session rows; if they can be retired, remove the field, helper fallback, dual writes, preservation/reset handling, and tests together, otherwise keep the compatibility path and document the retention plan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a cleanup request. Static inspection on current main gives high-confidence evidence that the field remains active in the session contract, helper fallback, history import, and tests.

## Solution Assessment

Unclear until maintainers decide the backward-compatibility policy. The proposed deletion is the narrow implementation only if OpenClaw is ready to stop supporting legacy Claude-only persisted session rows.
