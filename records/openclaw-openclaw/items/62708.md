## Summary

Keep open. Current main still treats `claudeCliSessionId` as an active legacy compatibility field for persisted Claude CLI sessions, so removing it is a compatibility cleanup that needs maintainer judgment rather than an automatic sweep.

## What This Changes

The issue requests removing the legacy `claudeCliSessionId` session field and its related fallback, dual-write, preservation, and test coverage paths.

## Best Possible Solution

Make an explicit compatibility decision for legacy Claude-only session rows; if they can be retired, remove the field, helper fallback, dual writes, preservation/reset handling, and tests together, otherwise keep the compatibility path and document the retention plan.
