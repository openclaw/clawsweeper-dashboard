## Summary

Keep open: current main and v2026.6.6 still execute heartbeat turns through the awaited agent reply path with busy-lane deferral and wake retry, while the open related retry PR covers only narrower retry spacing rather than this broader non-blocking heartbeat isolation request.

## What This Changes

Request non-blocking heartbeat execution, such as a worker or dedicated lane, so heartbeat checks do not delay user message processing.

## Best Possible Solution

Design a heartbeat-owned isolated execution contract, likely a dedicated lane or worker model, that keeps user turns prioritized while preserving session ordering, delivery guarantees, cancellation, and tool side effects.

## Reproduction Assessment

Partly: source inspection gives a high-confidence path showing current heartbeats are awaited and backpressured by main/session lanes. The exact 5-10 second user-visible delay was not live reproduced in this read-only review.

## Solution Assessment

Unclear as proposed. A background worker may be viable, but the best solution needs an explicit heartbeat isolation contract rather than a timer-only tweak or the narrower retry-delay PR.
