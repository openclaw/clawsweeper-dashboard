## Summary

Keep open: current main and v2026.6.10 still serialize/defer heartbeat work through the shared heartbeat runner and wake retry path, while the open related PR only mitigates one retry-cadence bug rather than implementing non-blocking heartbeat execution.

## What This Changes

The issue requests a separate non-blocking heartbeat execution model so heartbeat checks do not delay user message processing.

## Best Possible Solution

Keep this issue as the broad tracker for a maintainer-approved heartbeat isolation model, while the narrower retry cadence bug remains owned by https://github.com/openclaw/openclaw/issues/40611 and https://github.com/openclaw/openclaw/pull/89989.

## Solution Assessment

Unclear: a background worker or separate heartbeat lane is plausible, but the best fix needs a maintainer-defined isolation contract for session state, delivery ordering, retry behavior, and tool side effects.
