## Summary

Keep open. Current `main` still treats heartbeat as a serialized agent turn with busy-lane deferral and an inline `getReplyFromConfig` call, so the requested non-blocking worker/lane model is not implemented and needs maintainer design rather than cleanup closure.

## What This Changes

Request heartbeat checks to execute outside the message-processing path so incoming user messages are not delayed during heartbeat runs.

## Best Possible Solution

Define a heartbeat isolation model that preserves session ordering, delivery dedupe, tool side effects, and wake retries, then implement it with focused regression coverage and docs updates.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does confirm the current serialized heartbeat path and busy-lane deferral model that the feature request wants to replace.

## Solution Assessment

No, a blind background worker thread is not yet proven to be the best fix. The safer direction is an explicit heartbeat execution-lane or isolated-turn design with ordering and state contracts defined first.
