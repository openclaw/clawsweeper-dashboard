## Summary

Keep this open: current `main` and `v2026.5.19` already have a wired `update.run` click path, but the report may still expose a runtime feedback gap when the request fails, hands off, or cannot start clearly from the Control UI.

## What This Changes

Issue reports that the Control UI update notification button appears clickable but gives no visible feedback, while the CLI update command works.

## Best Possible Solution

Keep the existing updater boundary and make every `update.run` click outcome visibly actionable near the update banner, including request errors, admin-scope failures, handoff states, and disconnected/no-client early exits.
