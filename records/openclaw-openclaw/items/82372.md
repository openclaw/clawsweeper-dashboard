## Best Possible Solution

Add an owner-backed, live-verified fix in the Codex app-server startup path that proves default native hook relay registration delivers `features.hooks` and `hooks.PreToolUse` on fresh and resumed threads, with clear warning or failure behavior when the relay cannot engage.
