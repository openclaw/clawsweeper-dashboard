## Summary

Keep this PR open, but the current patch is incomplete: it adds type and mapper plumbing for outbound `message_sent.senderId` without any runtime path that can actually populate the bot/self sender ID.

## What This Changes

Adds optional `senderId` to the outbound `message_sent` plugin event type, canonical sent hook context, builder params, and mapper output.

## Best Possible Solution

Add the public field only with a backed runtime source for bot/self sender identity, then cover the mapper and outbound delivery path with focused regression tests.

## Reproduction Assessment

Yes. Source inspection of current main shows outbound `message_sent` events are built without any sender identity, and the PR still leaves the actual population path absent.

## Solution Assessment

No. Type-only plumbing is not enough for the reported hook consumer problem; the maintainable fix needs runtime population from the channel/provider identity source before exposing the field as useful.
