## What This Changes

The issue reports that Telegram inbound photo albums leave sibling photos as unreadable `telegram:file/...` references instead of usable model-visible media.

## Best Possible Solution

Review and land the focused Telegram plugin fix in https://github.com/openclaw/openclaw/pull/97045 if maintainers accept its prompt-context semantics, then close this issue after merge.

## Reproduction Assessment

Yes at source level: current main collects all album media, but prompt-context hydration only maps the first media item to the primary message id while cached siblings can still expose raw `telegram:file/...` refs. I did not perform a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes: the narrow solution is to repair the Telegram plugin's media-group to prompt-context handoff, not add a gateway batching feature or new config. The open closing PR is the best current implementation candidate to review.
