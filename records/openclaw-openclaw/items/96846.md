## Summary

Keep open: current main and the latest release still have the source-level Telegram album prompt-context gap, and the clean linked closing PR has not merged yet. The right cleanup is to keep this issue as the canonical tracker until the candidate fix lands.

## What This Changes

The issue reports that Telegram media-group albums expose only the primary photo as usable model media while sibling photos can appear as unreadable `telegram:file/...` references.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97045 or an equivalent Telegram-plugin fix that preserves each album item’s source message id, hydrates successful sibling messages to `media://inbound/...`, and excludes skipped downloads from prompt context.

## Reproduction Assessment

Yes. Source inspection shows the album path collects all media, but current main only maps the first media item to the primary message id before building prompt context, so sibling album messages can fall back to raw `telegram:file/...` refs.

## Solution Assessment

Yes. The narrow maintainable solution is in the Telegram plugin’s existing album-to-prompt-context handoff, not a new gateway batching feature or a config option.
