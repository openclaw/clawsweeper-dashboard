## Summary

Keep open: current main still has the Telegram album sibling media-context gap, and the open linked fix PR is the right path but has not merged yet.

## What This Changes

Telegram media-group albums can leave sibling photos as raw `telegram:file/...` references in model-facing context instead of hydrated `media://inbound/...` image paths.

## Best Possible Solution

Merge one canonical Telegram plugin fix that carries each successfully downloaded album item’s source message id into prompt-context hydration and excludes failed-download siblings from model-facing context.

## Reproduction Assessment

Yes. The source path is clear on current main: albums collect all media for the turn, but prompt-context media hydration only maps `params.allMedia[0]` to the primary message while sibling cache nodes can fall back to raw `telegram:file/...` refs.

## Solution Assessment

Yes. The best fix is a narrow Telegram plugin handoff repair from album download results into prompt-context hydration; broader gateway batching or a new config option would not address the specific raw-reference gap as directly.
