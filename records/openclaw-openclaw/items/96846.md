## Summary

Keep open: current main and the latest release still contain the Telegram album sibling prompt-context gap, and https://github.com/openclaw/openclaw/pull/97045 is an open clean closing PR rather than a merged fix.

## What This Changes

Telegram album photos should all reach model-facing context as usable image/media references instead of leaving sibling photos as raw `telegram:file/...` references.

## Best Possible Solution

Land the focused Telegram plugin PR if maintainers accept its message-delivery semantics, then close this issue as implemented and supersede the narrower draft overlap.

## Reproduction Assessment

Yes at source level: current main collects every media-group item into `allMedia`, but prompt-context hydration maps only `params.allMedia[0]` to the current message id while cached sibling messages can still expose `telegram:file/...` refs. I did not run live Telegram in this read-only review.

## Solution Assessment

Yes, the best fix belongs in the existing Telegram plugin media-group to prompt-context handoff, reusing `media://inbound/...` hydration and avoiding new gateway batching or config surface. The open closing PR appears to follow that shape.
