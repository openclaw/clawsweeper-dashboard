## Summary

Keep open: current `main` and the latest release still have the Telegram album prompt-context gap, and the open proof-sufficient fix PR at https://github.com/openclaw/openclaw/pull/97045 uses closing syntax for this issue. The issue should remain open until that PR lands or maintainers reject that path.

## What This Changes

The issue reports that Telegram media-group sibling photos can reach model-facing context as raw `telegram:file/...` references instead of usable image/media references.

## Best Possible Solution

Land or explicitly reject the focused Telegram-plugin fix at https://github.com/openclaw/openclaw/pull/97045, then close this issue once the chosen fix is merged.

## Reproduction Assessment

Yes at source level: current `main` collects all album media but only hydrates the primary/current message id for prompt context while cached sibling messages retain raw `telegram:file/...` refs. I did not run a live Telegram Bot API reproduction in this read-only review.

## Solution Assessment

Yes: the best fix is the narrow Telegram plugin handoff already represented by https://github.com/openclaw/openclaw/pull/97045, carrying successful album item message ids into prompt-context hydration and excluding skipped siblings. A gateway batching feature or new config option would be broader than this bug needs.
