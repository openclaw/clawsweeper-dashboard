## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid retrying full agent runs on transient errors:** `src/agents/model-fallback.ts:713`
  - body: The retry loop re-invokes `params.run` for the same candidate, but auto-reply wraps an entire embedded agent run there. Tool sends and `cron.add` completions may already be committed before a later transient error, so the retry can replay side effects.
  - confidence: 0.9
- **[P1] Keep same-model retries on the original request:** `src/agents/model-fallback.ts:713`
  - body: Callers treat every later `params.run` invocation as a fallback attempt, which changes prompt/resume behavior and can drop image inputs. A same-model transient retry should not switch to fallback-mode prompt semantics.
  - confidence: 0.84
- **[P2] Preserve legacy dedupe for no-target sends:** `src/auto-reply/reply/followup-runner.ts:388-390`
  - body: The proposed followup gate disables all text/media dedupe whenever any sent target exists. In mixed runs with one cross-target send and one no-target send, the legacy no-target duplicate can be delivered back to the origin conversation.
  - confidence: 0.86
- **[P2] Make queued delivery waits abort-aware:** `src/infra/outbound/delivery-serializer.ts:23-38`
  - body: The serializer waits on the previous same-key delivery without a cancellation path. A send aborted while queued can wait past its timeout and still run later because abort checks happen inside the queued function.
  - confidence: 0.82
- **[P2] Include thread identity in the serializer key:** `src/infra/outbound/deliver.ts:521-522`
  - body: The serializer key uses only channel, account, and recipient, while outbound delivery also targets `threadId`. This can serialize unrelated threads in the same channel and cause avoidable head-of-line blocking.
  - confidence: 0.8
- **[P2] Skip transient retries during cooldown probes:** `src/agents/model-fallback.ts:713`
  - body: Cooldown probes are intentionally limited so fallback can move on quickly. Retrying transient failures inside a probe sleeps and re-hits the cooldowned provider before trying the next candidate.
  - confidence: 0.77
