## Summary

Keep open. Current main and the latest release still have the followup cross-target dedupe bug, so the PR is not obsolete, but the proposed branch is too broad and has blocking review issues in unrelated model retry, outbound serialization, and dependency surfaces.

## What This Changes

The PR adds a same-target guard around followup messaging-tool text/media dedupe and tests, while also adding same-model transient retry logic, an outbound delivery serializer, and a Mattermost TypeBox dependency/lockfile entry.

## Best Possible Solution

Land a narrowed update against current main that fixes resolveFollowupDeliveryPayloads itself: compute same-target suppression before text/media duplicate filtering, dedupe only same-target or legacy no-target sends, preserve mixed target/no-target behavior, and add focused followup text/media regression tests. Split model retry, outbound serialization, and Mattermost dependency work into separate reviews.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid retrying whole agent turns on transient errors:** `src/agents/model-fallback.ts:713`
  - body: The retry loop re-invokes `params.run` for the same candidate, but auto-reply wraps an entire embedded agent run there. Tool sends and `cron.add` completions are already committed during the run, so a later transient 408/429/503 can replay those side effects on retry.
  - confidence: 0.9
- **[P1] Keep same-model retries on the original prompt:** `src/agents/model-fallback.ts:713`
  - body: Callers such as `agent-command.ts` treat every invocation after the first as a fallback retry (`fallbackAttemptIndex > 0`). Retrying the same provider/model through that callback changes prompt/resume semantics and can drop image inputs even though this is meant to be the same request.
  - confidence: 0.84
- **[P2] Preserve legacy dedupe for no-target sends:** `src/auto-reply/reply/followup-runner.ts:388-390`
  - body: This gate disables all text/media dedupe whenever any sent target exists. In a mixed run with one cross-target send and one successful send lacking target metadata, the no-target duplicate is no longer filtered and can be emitted back to the origin conversation.
  - confidence: 0.86
- **[P2] Make queued delivery waits abort-aware:** `src/infra/outbound/delivery-serializer.ts:23-38`
  - body: The serializer waits for the previous same-key delivery with no cancellation path. Since abort checks happen inside `deliverOutboundPayloadsCore`, a send aborted while queued can wait past its timeout and still run later.
  - confidence: 0.82
- **[P2] Include thread identity in the serializer key:** `src/infra/outbound/deliver.ts:521-522`
  - body: The serializer key uses only channel, account, and recipient, but outbound delivery also targets `threadId`. This serializes unrelated threads in the same channel, causing head-of-line blocking and avoidable timeout risk for threaded channels.
  - confidence: 0.8
- **[P2] Skip extra retries for cooldown probes:** `src/agents/model-fallback.ts:713`
  - body: Cooldown probes are intentionally limited so fallback can move on quickly. Retrying transient failures inside a probe sleeps and re-hits the same cooldowned provider before trying the next candidate, weakening the existing one-probe guard.
  - confidence: 0.77
