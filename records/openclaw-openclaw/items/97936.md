## Summary

Keep this PR open, but it is not merge-ready: the fix addresses the simple same-value env collision while leaving file/exec SecretRef cleanup and key/value ownership edge cases unresolved in a credential-scrubbing path.

## What This Changes

The PR narrows `secrets apply` `.env` scrubbing to env ref IDs collected from migrated targets and adds same-value credential regression tests.

## Best Possible Solution

Use a target-owned `.env` scrub contract that pairs each eligible env key with the migrated values owned by that same target and preserves documented cleanup for all SecretRef source types.

## Reproduction Assessment

Yes by source inspection. A target migrated to a file or exec SecretRef can add a migrated plaintext value without adding any `scrubbedEnvKeys`, and a mixed plan still passes independent key/value sets into the scrubber.

## Solution Assessment

No. The PR is a plausible narrow mitigation, but the safer fix is a target-owned key/value scrub map or equivalent ownership contract that covers env, file, exec, provider-auth, auth-profile, and plugin targets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Bind scrub keys to their owning values:** `src/secrets/apply.ts:705`
  - body: Passing separate `scrubbedValues` and `scrubbedEnvKeys` sets still forms a cross product: one target can contribute an env key while another contributes the matching value, causing `scrubEnvRaw` to delete a live env line that was not migrated as that key/value pair. Build a per-env-key value map from the target that owns both sides instead.
  - confidence: 0.87
- **[P2] Preserve file and exec cleanup coverage:** `src/secrets/apply.ts:697-698`
  - body: Because `scrubbedEnvKeys` is only populated when `target.ref.source === "env"`, a target migrated to a file or exec SecretRef can add an old plaintext value but no eligible env key, and the new early return skips `.env` cleanup entirely. That regresses the documented behavior that scrubs known `.env` secret keys whose values were migrated.
  - confidence: 0.9
