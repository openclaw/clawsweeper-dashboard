## Summary

Keep open. The root-cause direction is plausible, but this maintainer-labeled PR currently fails CI and the detector appears narrower than the linked in-house GLM gateway reproduction it is meant to fix.

## What This Changes

The PR adds z.ai-style silent-overflow detection to the Pi auto-compaction guard in embedded agent runs and compaction sessions, plus focused pi-settings tests.

## Best Possible Solution

Land a version that keeps Pi auto-compaction disabled for every reproducible z.ai/GLM silent-overflow route, preserves ordinary provider behavior, and includes a payload-level regression plus green CI.

## Reproduction Assessment

Yes. The linked bug report gives a two-turn Telegram reproduction and a verification path comparing prompt.submitted with the outbound OpenAI-compatible request; the current PR also has a deterministic CI failure from a missing mocked export.

## Solution Assessment

No in its current form. Disabling Pi auto-compaction for z.ai-style runs is a reasonable direction, but the PR needs to cover the reported in-house GLM gateway shape and repair the test mock break before merge.
