## Summary

Current main still leaves a credible CLI-runtime routing gap for cold nested subagent runs, but the proposed persistence fix would conflate canonical model identity with the execution runtime. Keep this open for maintainer-directed live reproduction and a narrower execution-boundary fix.

## What This Changes

## Best Possible Solution

Preserve canonical `modelProvider` and `model` session state, resolve the configured CLI runtime at the execution-owner boundary before runner selection, carry that prepared runtime fact through registration or dispatch as needed, and add a cold depth-2 regression with real CLI process and transcript proof.
