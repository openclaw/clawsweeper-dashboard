- **[P1] Update the SDK contract artifacts for the new stream export:** `src/plugin-sdk/provider-stream-shared.ts:10-15`
  - body: This adds `createAssistantStreamAccumulator` to the public `plugin-sdk/provider-stream-shared` subpath and changes `AssistantMessageEvent` delta/partial semantics, but the PR does not update the SDK API baseline/hash or public subpath docs. The scoped SDK guide treats public subpath changes as contract changes, and `plugin-sdk:api:check` is designed to fail on this kind of drift until the generated contract artifacts are refreshed or the export is removed.
  - confidence: 0.9
- **[P2] Stop the health pinger on failed repro runs:** `scripts/repro-gateway-dense-stream-latency.ts:510-511`
  - body: `stopFlag.done` is set only after `agent.wait` succeeds. If `chat.send`, `agent.wait`, or the responsive assertion throws, the `finally` block disconnects clients while the ping loop keeps scheduling requests and swallowing errors, which can hang the repro script and obscure the real failure. Set the flag in a `finally` around the measured run and await/settle the pinger before disconnecting.
  - confidence: 0.86
- **[P2] Keep the memory FD ready output bounded:** `scripts/check-memory-fd-repro.mjs:332-335`
  - body: This change replaces the bounded `updateGatewayReadyOutputState` tail with `output += text` and removes the regression test that proved noisy startup logs stay capped. A chatty or stuck gateway can now grow the script's in-memory buffer for the whole ready timeout, reopening the just-fixed memory issue in this helper.
  - confidence: 0.84
