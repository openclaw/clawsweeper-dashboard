## Summary

Keep this PR open: current main still does not provide the requested per-agent WhatsApp ack emoji fallback, but the submitted branch is stale against the current async ack-handle flow and has blocking account-scope and changelog gaps before it can merge.

## What This Changes

The PR adds a WhatsApp-specific ack emoji resolver/export, routes WhatsApp ack reactions through it, and adds resolver tests for object-format WhatsApp ackReaction configs.

## Best Possible Solution

Revise this into a current-main-compatible WhatsApp ack resolver that preserves reactionLevel gating, ack removal, account/channel scope precedence, typed tests, and changelog coverage while still skipping legacy `messages.ackReaction`.

## Reproduction Assessment

Yes. Source inspection on current main shows a valid WhatsApp `ackReaction` object without `emoji` resolves to an empty string, and the WhatsApp ack gate suppresses blank emoji reactions.

## Solution Assessment

No as submitted. The WhatsApp-specific resolver direction is reasonable, but the branch must be rebased into the current async ack-handle flow and must resolve emoji and scope from the same active account/channel config.
