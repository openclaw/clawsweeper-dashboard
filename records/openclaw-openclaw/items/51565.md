## Summary

Keep open: the WhatsApp per-agent ack emoji behavior is not on current main, and the submitted branch still needs real WhatsApp proof plus fixes for current ack lifecycle, account-scoped policy, and plugin boundary concerns.

## What This Changes

The PR changes WhatsApp ack sending to resolve a fallback emoji from account/channel ackReaction config and agent identity, exports that resolver through channel-feedback, and adds resolver tests.

## Best Possible Solution

Rework this as a WhatsApp-local effective ack resolver that merges account/channel emoji and direct/group scope together, preserves ack-handle/status cleanup, and is backed by redacted live WhatsApp proof.

## Reproduction Assessment

Yes from source, but not from a live WhatsApp session. Current main turns a valid WhatsApp ackReaction object without emoji into an empty emoji and shouldAckReactionForWhatsApp suppresses it.

## Solution Assessment

No, not as submitted. The narrow maintainable fix should live in the WhatsApp plugin or a narrow generic seam and resolve emoji plus scope from the same effective account/channel config.
