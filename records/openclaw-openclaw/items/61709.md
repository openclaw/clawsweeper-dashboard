## Summary

Keep open. Current main still only supports per-agent auth profile order overrides and single-store locked writes; I did not find a host-wide `auth migrate`, `auth set-active`, or `--all-agents` auth command. The adjacent open #8190 is related to session auth-cache reset after auth changes, but it does not cover atomic migration across every agent-scoped auth store.

## What This Changes

Add a CLI path to migrate or set the active auth profile across all agent-scoped auth stores atomically.

## Best Possible Solution

Add an owner-reviewed auth management path that enumerates configured and discoverable agent dirs, validates the requested provider/profile for each target, writes per-agent auth-state order through existing locks, supports dry-run and JSON reporting, defines partial-failure or rollback behavior, and documents verification with `models status --agent` and `--probe`.

## Reproduction Assessment

Not applicable as a feature request, but the current-main verification path is high-confidence: source, tests, and docs show per-agent auth order control only, while the only all-agent target resolver is in the sessions path.

## Solution Assessment

Unclear until owner review. The proposed `set-active`/`migrate` direction is plausible, but the maintainable solution should be designed around the current `auth-state.json` contract, per-store locks, all-agent discovery rules, and cache/session coordination.
