## Summary

Keep open. Current main still has per-agent auth profile management only; there is no host-wide auth migration or active-profile switch command, and the requested capability is security-sensitive cross-agent credential routing rather than a narrow bug fix.

## What This Changes

Request a CLI command to atomically switch or migrate a provider's active auth profile across all agent auth stores on a host.

## Best Possible Solution

Add an owner-reviewed auth management command that targets the current `auth-state.json` contract, enumerates configured and discoverable agent auth stores, validates profile availability per target, writes through existing locks, supports dry-run and JSON reporting, and documents stale-profile removal plus session-cache coordination.

## Reproduction Assessment

Not applicable as a feature request. Source and docs give a high-confidence current-main check: auth profile listing and order changes are per-agent only, while all-agent targeting exists only for session store commands.

## Solution Assessment

Unclear as proposed until owner review. The `migrate` or `set-active` direction is plausible, but the maintainable solution should use the current `auth-state.json` order/lastGood contract and define safety semantics before implementation.
