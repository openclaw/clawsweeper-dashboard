## Summary

Keep open. Current main and v2026.5.18 still expose per-agent auth profile management only; there is no host-wide auth migrate or active-profile switch command, and the related active-session refresh work remains separately tracked at https://github.com/openclaw/openclaw/issues/8190.

## What This Changes

Request a CLI command to atomically switch or migrate a provider auth profile across all agent-scoped auth stores.

## Best Possible Solution

Add an owner-reviewed auth management command that enumerates configured and discoverable agent auth stores, validates profile availability per target, writes through existing auth-state locks, supports dry-run and JSON reporting, and documents stale-profile removal plus session-cache coordination.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: auth profile list/login/paste-token/order are per-agent, while all-agent targeting exists only for session store commands.

## Solution Assessment

Unclear as proposed until owner review. A migrate or set-active command is plausible, but the maintainable implementation should target auth-state.json, not only auth-profiles.json, and define safety semantics before code work starts.
