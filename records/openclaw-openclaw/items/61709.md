Request a CLI workflow to atomically switch or migrate a provider auth profile across every agent auth store on a host.

## Best Possible Solution

Add an owner-reviewed models auth command that enumerates configured and discoverable agent auth states, validates profile availability per target, writes through existing auth-state locks, supports dry-run and JSON reporting, and documents stale-profile removal plus session-refresh coordination.

## Solution Assessment

Unclear as proposed until owner review. A migrate or set-active command is plausible, but the maintainable implementation should target auth-state.json order semantics and define security, atomicity, stale-profile cleanup, and session-refresh behavior before code work starts.
