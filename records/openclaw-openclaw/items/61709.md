## Summary

Keep open. Current main still has per-agent auth profile management only; I found no host-wide `auth migrate`, `auth set-active`, or auth `--all-agents` command, and the related session-reset tracker covers a different runtime cache problem.

## What This Changes

Request a CLI path to atomically switch or migrate one provider auth profile across all agent-scoped auth stores on a host.

## Best Possible Solution

Add an owner-reviewed auth management command that enumerates configured and discoverable agent auth stores, validates provider/profile availability per target, writes `auth-state.json` through existing locks, supports dry-run and JSON reporting, and documents verification plus session-cache coordination.

## Reproduction Assessment

Not applicable as a feature request. Source and docs give a high-confidence current-main check: auth profile ordering is per-agent only, while all-agent targeting exists only for session store commands.

## Solution Assessment

Unclear until owner review. The requested `migrate` or `set-active` direction is plausible, but the maintainable solution should target the current `auth-state.json` contract and define safety semantics before implementation.
