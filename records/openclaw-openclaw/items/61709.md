## Summary

Keep open. Current main still provides per-agent auth-order writes only; all-agent discovery is implemented for sessions, and the linked #8190 tracks session auth-cache reset rather than atomic cross-agent auth-state migration.

## What This Changes

Request a CLI command to switch or migrate a providerâ€™s active auth profile across all agent-scoped auth stores atomically.

## Best Possible Solution

Add an owner-reviewed auth management command that enumerates configured and discoverable agent dirs, validates the requested provider/profile per target, updates current `auth-state.json` through existing locks, and documents dry-run/JSON reporting plus verification commands.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence: source and docs show per-agent auth order control only, with all-agent targeting limited to sessions.

## Solution Assessment

Unclear until auth/security-owner review. The proposed `set-active` or `migrate` direction is plausible, but the durable design should use the current `auth-state.json` split, existing per-store locks, target discovery, and session-cache coordination boundaries.
