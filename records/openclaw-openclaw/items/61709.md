## Summary

Keep open. Current main and v2026.5.18 still expose per-agent auth profile and auth-order management only; related open issues cover credential sync and active-session refresh, but they do not supersede this all-agent active-profile migration request.

## What This Changes

Request a CLI/admin flow to atomically switch a provider's active auth profile across all agent stores, with optional stale-profile cleanup and reporting.

## Best Possible Solution

Add an owner-reviewed auth management command that enumerates configured and discoverable agent auth states, validates profile availability per target, writes through existing auth-state locks, supports dry-run and JSON reporting, and documents stale-profile removal plus session-refresh coordination.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check: auth profile list/login/paste-token/order are per-agent, while all-agent targeting exists for session stores but not models auth.

## Solution Assessment

Unclear as proposed until owner review. A migrate or set-active command is plausible, but the maintainable implementation should target auth-state.json/order semantics, not only auth-profiles.json, and should define safety semantics before code work starts.
