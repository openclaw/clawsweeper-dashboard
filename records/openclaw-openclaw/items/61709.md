## Summary

Keep open. Current main still has only per-agent auth profile ordering and single-store locked writes; I did not find a host-wide `auth migrate`, `auth set-active`, or auth `--all-agents` path. The related open #8190 covers session auth-cache reset after auth changes, not atomic migration of every agent-scoped auth store.

## What This Changes

Request a CLI command to switch or migrate the active auth profile for a provider across all agent-scoped auth stores atomically.

## Best Possible Solution

Add an owner-reviewed auth management path that enumerates configured and discoverable agent dirs, validates the requested provider/profile per target, writes current `auth-state.json` order/last-good state through existing locks, supports dry-run and JSON reporting, and documents verification with `models status --agent` and `--probe`.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence: source and docs show per-agent auth order control only, with all-agent targeting limited to sessions.

## Solution Assessment

Unclear until auth/security-owner review. The proposed `set-active` or `migrate` direction is plausible, but the durable fix should be designed around the current auth-state split, per-store locks, target discovery, and session cache coordination.
