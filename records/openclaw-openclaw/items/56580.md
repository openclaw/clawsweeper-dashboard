## Summary

Keep open. Current main still has the reported Discord native-command localization gap: Discord command serialization can emit localization fields, but the native command spec, built-in/skill command conversion, plugin command spec plumbing, and Discord command creation path still carry only the fallback English description.

## What This Changes

Request support for localized Discord slash-command descriptions by carrying `description_localizations` through native command specs, command generation, and reconcile checks.

## Best Possible Solution

Add an additive typed `descriptionLocalizations` path for native command metadata, propagate it from built-in, skill, and plugin command definitions into Discord `Command.descriptionLocalizations`, and cover serialization/reconcile behavior with focused regression tests while keeping English `description` as the required fallback.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: build or inspect a Discord native command from the current `NativeCommandSpec` path and the serialized desired command contains only the English `description`, while reconcile/overwrite deploys that desired payload against live Discord commands.

## Solution Assessment

Yes. The reporter's suggested direction is the narrow maintainable fix because the Discord serializer already supports `description_localizations`; OpenClaw needs metadata plumbing and tests, not a new product-wide translation catalog.
