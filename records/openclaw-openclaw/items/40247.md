## Reproduction Assessment

Yes, source-level: current main binds only the selected non-loopback host while implicit local Gateway callers still target 127.0.0.1. I did not run a live tailnet/custom Gateway in this read-only review.

## Solution Assessment

Unclear as-is. Listener-side dual-bind is plausible because current local callers intentionally target loopback, but the mergeable solution must preserve Windows behavior, document/approve the listener-surface change, and reconcile the competing client-side approach.
