## Summary

Keep open: current main documents that routine OAuth token rotation should not cut stored CLI session continuity, but the source path still starts a fresh CLI session when auth profile or auth epoch metadata invalidates the binding, and that fresh path only reseeds history after compaction.

## What This Changes

The issue asks OpenClaw to preserve prior conversation context when a claude-cli auth-profile/auth-epoch rotation starts a new live CLI session.

## Best Possible Solution

Preserve claude-cli session continuity across routine OAuth refresh by keeping the same OAuth identity/profile reusable, and use the existing bounded reseed policy only for unavoidable fresh starts.

## Reproduction Assessment

No live high-confidence macOS OAuth reproduction was run. Source inspection gives a targeted reproduction path for the cold-session symptom: trigger an auth-profile/auth-epoch invalidation with a non-compacted transcript and prepareCliRunContext will produce no historyPrompt.

## Solution Assessment

The requested reinjection is only part of the best fix. The narrower maintainable fix is to stop routine OAuth refresh from invalidating the same cli session identity, then keep fresh-session reseeding bounded by the current transcript policy.
