## What This Changes

The issue asks OpenClaw to preserve claude-cli conversation context when auth-profile or auth-epoch rotation causes the gateway to start a fresh CLI session.

## Best Possible Solution

Keep the implementation review concentrated in https://github.com/openclaw/openclaw/pull/68686, and land it together with the related auth-epoch token hashing fix path if maintainers agree with that scope.

## Reproduction Assessment

Yes for the gateway reset path: source inspection can reproduce it by giving `resolveCliSessionReuse` a stored binding and changed auth-profile or same-version auth-epoch, which produces no reusable session id; the live OAuth refresh cadence itself was not rerun here.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve the existing CLI session id for auth-only metadata changes while still forcing fresh sessions for system-prompt or MCP changes, which is exactly the direction of the canonical open PR.
