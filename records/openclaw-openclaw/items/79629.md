## Summary

Keep open: the reporter added concrete 2026.5.7 Feishu logs, and current main still has the source path for silent stream timeout, abort, and configured model fallback while diagnostics only observe `model_call` stalls without recovery.

## What This Changes

The issue asks OpenClaw to prevent stalled streamed model output from leaving channel previews open and then silently timing out into mixed or unwanted fallback output.

## Best Possible Solution

Add a focused regression or live harness for a streamed model call that emits preview text and then stalls; classify started/stalled streams distinctly and define channel-safe fallback behavior so already-visible preview output is not silently mixed with a later model.

## Reproduction Assessment

No live current-main reproduction was established in this read-only pass. The reporter supplied detailed 2026.5.7 Feishu logs, and current source inspection confirms the timeout, diagnostic, and fallback branches needed for the cascade.

## Solution Assessment

Unclear. Early abort plus fallback would reduce the delay, but it would not by itself solve silent mixed-model output in the same preview; maintainers should first decide the policy for already-visible streamed text.
