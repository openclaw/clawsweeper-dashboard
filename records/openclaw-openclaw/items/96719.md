## Best Possible Solution

Make suppression depend on an actually visible progress draft, or otherwise cover the default progress-mode fast-tool fallback, then add default-config regression coverage and redacted real Discord proof.

## Reproduction Assessment

## Solution Assessment

No: the current patch fixes only the fully disabled preview configuration, not the documented default progress-mode fast-tool path described by the linked issue. The safer fix is to suppress standalone messages only after a progress draft is actually visible, or otherwise add an explicit default fallback contract.
