## Summary

Keep open: current main still documents and implements the Gateway as the single control-plane process and dispatches agent work to local CLI or embedded execution, so the distributed runtime RFC remains unimplemented and needs maintainer product/security direction.

## What This Changes

The issue requests splitting `openclaw gateway` into a lightweight control plane and separately deployable per-agent runtime processes with new protocol, lifecycle, scaling, and isolation surfaces.

## Best Possible Solution

Keep this as the umbrella architecture RFC, then promote a maintainer-approved Phase 0/1 design for runtime protocol, per-agent lifecycle, security boundaries, and upgrade behavior before code work starts.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a bug report. Source and docs review confirm the requested distributed runtime mode is not present on current main.
