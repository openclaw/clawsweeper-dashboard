## Summary

Keep open: current main and v2026.6.8 still describe and implement a single Gateway/control-plane process with embedded or ACP runtime state, while related open issues cover narrower deployment/backend slices rather than superseding this broader RFC.

## What This Changes

The issue requests a distributed runtime architecture that separates Gateway control-plane work from per-agent compute, lifecycle, state, secrets, and deployment modes.

## Best Possible Solution

Keep this RFC open for maintainer architecture and security review, then split any accepted direction into narrower protocol, lifecycle, session-state, secret-isolation, and deployment milestones.

## Reproduction Assessment

Not applicable: this is an RFC/feature request rather than a bug report. Source and docs do confirm the current gap: there is no implemented distributed per-agent runtime split on current main.

## Solution Assessment

Unclear: the proposed architecture is plausible, but maintainers still need to choose the product boundary, trust model, protocol, and first milestone. A narrower per-agent lifecycle phase or https://github.com/openclaw/openclaw/issues/82017 may be the first implementation slice.
