## Summary

Keep open: current main and v2026.5.22 still use gateway-owned embedded agent execution, while related open issues and PRs cover narrower lifecycle, worker-auth, ACP-node, or diagnostics slices rather than replacing this distributed-runtime umbrella.

## What This Changes

The issue requests a distributed OpenClaw agent runtime that separates the gateway control plane from per-agent compute runtimes for lifecycle, isolation, scaling, and operability.

## Best Possible Solution

Keep this as the distributed-runtime architecture umbrella, then split maintainer-approved milestones for runtime protocol and registration, per-agent lifecycle, session/channel routing, secret isolation, local worker mode, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for new runtime architecture. Source, docs, release, and related-item checks give high-confidence evidence that OpenClaw still uses gateway-owned embedded execution for the central path.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The RFC is directionally plausible, but the maintainable path is to split Phase 0/1 lifecycle, runtime registration, session/channel ownership, and secret contracts before container or Kubernetes backends.
