## Summary

Keep open: current main still runs Gateway as the control-plane process with embedded/local agent execution, while the RFC remains a broad product, security, and migration design. The TinyTb Phase 0 framing and markthebest12 operator workaround are useful evidence that per-agent lifecycle matters, but neither current code nor the related open PRs fully implement the distributed runtime split.

## What This Changes

The issue requests splitting Gateway control-plane responsibilities from per-agent compute runtimes, with local-split and distributed deployment modes.

## Best Possible Solution

Keep this as the broad canonical RFC, and split implementation only after maintainers choose the runtime protocol, trust model, lifecycle semantics, migration strategy, and which narrower related issues own first milestones.
