## Summary

Keep open: current main has adjacent agent routing, embedded/ACP runtime selection, node hosts, diagnostics, and per-agent workspaces, but it does not implement the requested control-plane/runtime split or per-agent compute runtime process boundary. Related open items cover narrower slices, but none supersedes this RFC’s broader protocol, lifecycle, isolation, and deployment design.

## What This Changes

RFC requests splitting the Gateway control plane from per-agent compute runtimes so agents can run in separate processes, containers, VMs, or hosts with independent lifecycle, secrets, resources, and fault domains.

## Best Possible Solution

Keep this as an architecture tracker until maintainers define the runtime boundary and split it into reviewed phases, starting with an additive per-agent lifecycle/API design and a runtime protocol threat model.

## Reproduction Assessment

Not applicable. This is an RFC for new distributed runtime architecture; source and docs inspection provide high confidence that the requested surfaces are absent, but there is no current-main bug reproduction to run.

## Solution Assessment

Unclear as a direct implementation plan. The direction is plausible, but the maintainable path is maintainer design followed by narrower PRs for lifecycle commands, protocol contracts, runtime registration/health, auth, and deployment modes.
