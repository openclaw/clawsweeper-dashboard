## Summary

Keep open. Current main still centers agent execution inside the Gateway, with only embedded/ACP runtime descriptors and no per-agent worker protocol or lifecycle surface; the newer operator comments add useful Phase 0 evidence rather than making this stale or superseded.

## What This Changes

The issue requests splitting the Gateway into a control plane plus separate per-agent runtime compute processes, including local-split and distributed deployment modes.

## Best Possible Solution

Keep this as an architecture tracker and break it into maintainer-reviewed milestones: embedded compatibility, local per-agent lifecycle, typed Gateway/runtime protocol, then distributed/container orchestration with explicit security review.

## Reproduction Assessment

Not applicable; this is an RFC/feature request, not a bug report. The capability gap is high-confidence from current docs and source showing embedded Gateway-side execution without per-agent runtime lifecycle or protocol support.

## Solution Assessment

Unclear as a direct implementation plan. The direction is plausible, but the maintainable solution is a phased protocol, lifecycle, compatibility, and security design rather than one broad autonomous patch.
