- `impact:session-state`: The requested control-plane/runtime split changes where session metadata, transcripts, active runs, and runtime ownership live.

## Summary

Keep open. Current main still documents and implements gateway-owned embedded agent execution, while the issue discussion and related open items only split off narrower lifecycle, Kubernetes auth, worker isolation, and org-deployment pieces rather than replacing the distributed runtime RFC.

## What This Changes

The issue requests a distributed runtime architecture that separates gateway/control-plane responsibilities from per-agent compute runtimes.

## Best Possible Solution

Keep this as the distributed-runtime architecture umbrella and split maintainer-approved milestones for runtime registration/protocol, per-agent lifecycle, session/channel routing, secret/auth isolation, local worker mode, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, release-tag, and related-item checks give high-confidence negative capability evidence that current main still uses gateway-owned embedded agent execution for the central path.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The RFC is directionally plausible, but the maintainable path is to split Phase 0/1 milestones around lifecycle, runtime registration, session/channel ownership, and secret contracts before container or Kubernetes backends.
