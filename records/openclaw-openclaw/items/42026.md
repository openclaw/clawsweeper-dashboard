## Summary

Keep open: current main and v2026.6.6 still describe a single Gateway/control-plane process with process-local embedded/ACP runtime state, and the requested per-agent runtime protocol, lifecycle, isolation, and deployment modes are not implemented. Related open issues cover narrower worker/backend slices, but none supersedes this broader RFC.

## What This Changes

Request to split `openclaw gateway` into a lightweight control plane and separately managed per-agent runtime processes for local-split and distributed deployments.

## Best Possible Solution

Keep this RFC open for maintainer architecture review, then split any accepted direction into narrower protocol, security, session-state, lifecycle, and deployment milestones.

## Reproduction Assessment

Not applicable: this is an RFC/feature request rather than a bug report. Source and docs do confirm the current gap: there is no per-agent runtime split or lifecycle command on current main.

## Solution Assessment

Unclear: the distributed-runtime architecture is plausible, but maintainers still need to choose the product boundary, protocol, and security model. A narrower per-agent lifecycle milestone or the open subagent-backend work may be the first step.
