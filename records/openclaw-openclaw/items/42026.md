- `P2`: This is meaningful multi-agent architecture work, but it is broad feature/design work rather than an urgent regression.
- `impact:session-state`: The proposed split changes where session metadata, transcripts, active runs, and runtime ownership state live.

## Summary

Keep open: current main still documents and implements gateway-owned embedded agent execution, while the issue comments and related open PRs only cover workarounds or experiments rather than a shipped control-plane/runtime split.

## What This Changes

The issue proposes splitting the Gateway into a control-plane process and separate per-agent runtime processes with local and distributed deployment modes.

## Best Possible Solution

Keep this as the distributed-runtime umbrella while maintainers split focused milestones for runtime protocol/registration, per-agent lifecycle, shared-channel routing, session/state migration, worker authentication, and secret/tool isolation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source and docs give high-confidence negative capability evidence that current main still uses gateway-owned embedded agent execution for the central path.

## Solution Assessment

Unclear until maintainers choose the runtime boundary and protocol. The safer path is to keep the umbrella open, review the worker-runtime experiment, and split Phase 0/1 issues before container or orchestration backends.
