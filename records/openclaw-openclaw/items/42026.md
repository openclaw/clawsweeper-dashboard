## Summary

Keep open. Current main and the latest release still document and implement gateway-owned embedded agent execution, while the newer discussion and related issues narrow pieces of the architecture rather than superseding this distributed-runtime RFC.

## What This Changes

The issue requests splitting the gateway/control plane from per-agent compute runtimes with local-split and distributed deployment modes.

## Best Possible Solution

Keep this as the distributed-runtime architecture umbrella and split maintainer-approved milestones for runtime registration/protocol, per-agent lifecycle, session/channel routing, secret isolation, local worker mode, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, shipped-release, and related-item checks give high-confidence evidence that current OpenClaw still uses gateway-owned embedded execution for the central path.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The RFC is directionally plausible, but the maintainable path is to split Phase 0/1 milestones around lifecycle, runtime registration, session/channel ownership, and secret contracts before container or Kubernetes backends.
