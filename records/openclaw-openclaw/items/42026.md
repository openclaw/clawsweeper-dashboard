## Summary

Keep open. Current main still runs agent turns through Gateway-side embedded or local harness paths and lacks the requested per-agent runtime worker protocol, runtime role, and per-agent start/stop/restart lifecycle; the TinyTb and markthebest12 comments add useful Phase 0/1 operator framing rather than making the RFC obsolete.

## What This Changes

Requests splitting `openclaw gateway` into a control plane plus per-agent runtime processes, with lifecycle commands, a Gateway/runtime protocol, and distributed/container deployment modes.

## Best Possible Solution

Keep this as an architecture tracker and split it into maintainer-reviewed milestones: Phase 0 per-agent lifecycle without whole-Gateway restart, then a typed Gateway/runtime protocol, then container/Kubernetes orchestration after threat-model review.

## Reproduction Assessment

Not applicable; this is an RFC/feature request, not a bug report. The capability gap is high-confidence from current docs, CLI registration, protocol roles, config schema, and Gateway execution source inspection.

## Solution Assessment

Unclear as a single implementation plan. The direction is plausible, but the maintainable path is a phased architecture and security design rather than one broad cleanup or autonomous patch.
