## Summary

Keep open: current main and v2026.6.6 still document a single Gateway process and single embedded agent runtime, while the per-agent lifecycle, runtime protocol, and distributed compute boundary remain unimplemented. Related issues cover narrower slices, but the closed per-agent restart request was explicitly superseded by this RFC, so this remains the canonical maintainer design item.

## What This Changes

The issue requests splitting the Gateway into a control plane and separate per-agent compute runtimes, with embedded, local-split, and distributed deployment modes.

## Best Possible Solution

Keep this as the canonical maintainer RFC, decide a narrow Phase 0/1 control-plane/runtime contract, then split approved work into focused protocol, lifecycle, security, deployment, docs, and validation issues.
