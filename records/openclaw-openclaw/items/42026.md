## Summary

Keep open. Current main and the latest release still describe and implement Gateway-owned embedded agent execution, while the active discussion narrows useful Phase 0/1 slices without resolving the broader runtime, security, and product architecture decision.

## What This Changes

The issue requests splitting the monolithic gateway into a control plane plus per-agent runtime compute processes with embedded, local-split, and distributed deployment modes.

## Best Possible Solution

Keep this as the architecture umbrella while maintainers break out reviewed slices for per-agent lifecycle, runtime protocol/registration, secret isolation, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, release, and related-item inspection give high-confidence negative capability evidence that current OpenClaw still uses Gateway-owned embedded agent execution.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The narrowest maintainable path appears to be reviewed Phase 0/1 slices around per-agent lifecycle, runtime registration, and auth/secret contracts before container or Kubernetes backends.
