- `impact:session-state`: The requested split changes where session metadata, transcripts, and active-run ownership live.

- `impact:message-loss`: The motivating failures include gateway restarts and routing gaps that can interrupt or misroute channel messages.

## Summary

Keep open. Current main and the latest release still use Gateway-owned embedded agent execution, and the discussion has narrowed useful Phase 0/1 slices without resolving the broader runtime, security, and product architecture decision.

## What This Changes

This RFC asks to split the Gateway control plane from per-agent compute runtimes, adding embedded, local-split, and distributed modes with per-agent lifecycle, isolation, and scaling.

## Best Possible Solution

Keep this as the architecture umbrella while maintainers split reviewed implementation issues for per-agent lifecycle, runtime registration/protocol, session and channel delivery ownership, secret isolation, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, tests, and release inspection give high-confidence negative capability evidence that current OpenClaw still uses Gateway-owned embedded agent execution.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The narrowest maintainable path appears to be reviewed Phase 0/1 slices around per-agent lifecycle, runtime registration, channel/session ownership, and auth/secret contracts before container or Kubernetes backends.
