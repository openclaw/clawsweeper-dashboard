## Summary

Keep open: this is still an active architecture RFC rather than a stale or duplicate cleanup candidate. Current main still has Gateway-owned embedded agent execution and no first-class split control-plane/runtime protocol or per-agent lifecycle CLI, while the recent comments add useful Phase 0/1 operator data.

## What This Changes

The issue proposes separating the Gateway control plane from per-agent compute runtimes, with embedded, local-split, and distributed deployment modes.

## Best Possible Solution

Keep this as the architecture umbrella and split maintainer-approved phases into narrower design issues: per-agent lifecycle/registry first, then a versioned runtime protocol, trust boundary, reconnect/health model, and deployment documentation.

## Reproduction Assessment

Not applicable. This is an RFC/feature request, and source inspection shows no first-class openclaw-runtime, local-split runtime protocol, or per-agent lifecycle CLI to exercise on current main.

## Solution Assessment

Unclear because there is no patch to validate. The direction is plausible, but the maintainable solution should be phased design work that separates the smaller per-agent lifecycle slice from the later distributed runtime and security protocol.
