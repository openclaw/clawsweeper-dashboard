## Summary

Keep open. Current main still implements and documents the Gateway as the central long-running process with embedded agent execution; the follow-up comments add useful Phase 0 and operator workaround data, but they do not supersede the broader control-plane/runtime architecture RFC.

## What This Changes

The issue requests a distributed agent runtime architecture that separates the Gateway control plane from per-agent compute/runtime processes.

## Best Possible Solution

Keep this as the architecture umbrella, then let maintainers choose smaller first slices such as per-agent lifecycle, runtime registration/protocol, and explicit secret/auth isolation before container or Kubernetes orchestration backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source and docs inspection provide high-confidence negative capability evidence: the Gateway remains the single process boundary and agent turns still enter the embedded runner.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The narrowest maintainable path appears to be a reviewed Phase 0/1 slice around per-agent lifecycle, runtime registration, and auth/secret contracts before distributed container or Kubernetes backends.
