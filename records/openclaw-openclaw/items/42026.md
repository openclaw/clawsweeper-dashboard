## Summary

Keep open: current main has multi-agent routing and runtime selection, but it still implements and documents one Gateway-owned control/runtime process; the Phase 0/1 per-agent lifecycle split raised in discussion remains real architecture work.

## What This Changes

Request to split Gateway control-plane responsibilities from per-agent compute runtimes while preserving the current embedded mode and adding local-split/distributed deployment paths.

## Best Possible Solution

Keep this as an RFC and have maintainers split it into approved phases: Phase 0 per-agent lifecycle, then a versioned runtime protocol, config/security model, deployment docs, and migration plan.

## Reproduction Assessment

Not applicable. This is an RFC/feature request rather than a broken current behavior report; source and docs inspection confirms the requested split is not a current runnable mode.

## Solution Assessment

No patch is present, so there is not a concrete implementation to endorse. The safest direction is an owner-scoped architecture design that separates the smaller per-agent lifecycle slice from the later distributed runtime protocol.
