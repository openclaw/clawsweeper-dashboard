## Summary

Keep open: current main and the latest shipped release still document and route through gateway-owned embedded agent execution, while related open issues cover narrower worker, Kubernetes auth, org deployment, or stall slices rather than replacing this distributed-runtime umbrella.

## What This Changes

The issue requests splitting gateway control-plane duties from per-agent compute into embedded, local-split, and distributed runtime modes.

## Best Possible Solution

Keep this as the distributed-runtime architecture umbrella and split maintainer-approved milestones for runtime protocol/registration, per-agent lifecycle, session/channel routing, secret isolation, local worker mode, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, release-tag, and related-item checks give high-confidence evidence that current OpenClaw still uses gateway-owned embedded execution for the central path.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The RFC is directionally plausible, but the maintainable path is to split Phase 0/1 milestones around lifecycle, runtime registration, session/channel ownership, and secret contracts before container or Kubernetes backends.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because a distributed runtime boundary would create new runtime authentication, tool-execution, filesystem, channel-delivery, and per-agent secret boundaries.

Concerns:

- **[medium] Design runtime authentication and secret isolation:** `src/gateway/protocol/schema/frames.ts:55`
  - body: The current Gateway connect auth schema only carries existing client credentials; a control-plane/runtime split needs scoped runtime identity, per-agent authorization, fail-closed outbound delivery, and secret isolation before separate compute can be trusted.
  - confidence: 0.9
- **[medium] Define session and message ownership across runtimes:** `src/gateway/server-methods/agent.ts:621`
  - body: Gateway dispatch currently starts in-process agent runs, so moving execution out of process requires an explicit ownership contract for transcripts, active runs, retries, and outbound channel delivery.
  - confidence: 0.86
