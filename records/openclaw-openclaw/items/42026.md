## Summary

Keep open. Current main still centers agent execution inside the Gateway process, and the related open items cover narrower lifecycle, worker-auth, filesystem isolation, and event-loop symptoms rather than superseding this distributed-runtime RFC.

## What This Changes

The issue requests a new distributed agent runtime architecture that separates Gateway control-plane duties from per-agent compute/runtime processes.

## Best Possible Solution

Keep this as the distributed-runtime architecture umbrella and split maintainer-approved milestones for runtime protocol/registration, per-agent lifecycle, session/channel routing, secret isolation, local worker mode, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, and related-item checks give high-confidence negative capability evidence that current main still uses gateway-owned embedded execution for the central path.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The RFC is directionally plausible, but the maintainable path is to split Phase 0/1 milestones around lifecycle, runtime registration, session/channel ownership, and secret contracts before container or Kubernetes backends.
