## Summary

Keep open: current main has agent runtime selection and partial hot reload for agent config, but it still does not provide a separate per-agent runtime process, local-split mode, distributed runtime protocol, or per-agent lifecycle commands. The recent Phase 0/1 comments add useful operator context and make this an active architecture RFC rather than a stale cleanup candidate.

## What This Changes

Request to split `openclaw gateway` into a lightweight control plane plus per-agent runtime compute processes with local-split and distributed deployment modes.

## Best Possible Solution

Keep this as an RFC and carve it into maintainer-approved phases: per-agent lifecycle/registration first, then a versioned runtime protocol, secret isolation model, health/reconnect semantics, and deployment story.

## Reproduction Assessment

Not applicable. This is an RFC/feature request rather than a broken current behavior report, and source inspection shows no first-class `openclaw-runtime`, local-split mode, distributed runtime protocol, or per-agent lifecycle CLI to exercise.

## Solution Assessment

Unclear because there is no patch to validate. The requested direction is plausible, but the maintainable path is a phased design that separates the smaller per-agent lifecycle slice from the later distributed runtime/security protocol.
