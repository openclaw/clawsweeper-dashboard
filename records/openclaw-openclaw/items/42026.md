## Summary

Keep open: current main has hot reload, multiple-gateway guidance, sandboxing, and embedded/CLI agent runtime selection, but it still does not provide a first-class split control plane, per-agent runtime process, distributed runtime protocol, or per-agent lifecycle commands. Recent Phase 0 and production-workaround comments make this an active architecture RFC, not stale cleanup.

## What This Changes

Request to split `openclaw gateway` into a lightweight control plane plus per-agent runtime processes with embedded, local-split, and distributed deployment modes.

## Best Possible Solution

Keep this as an RFC and split it into maintainer-approved phases: per-agent lifecycle and registry first, then a versioned runtime protocol, secret/sandbox trust model, health/reconnect behavior, and deployment documentation.

## Reproduction Assessment

Not applicable. This is an RFC/feature request, and source inspection shows no first-class `openclaw-runtime`, local-split mode, distributed runtime protocol, or per-agent lifecycle CLI to exercise on current main.

## Solution Assessment

Unclear because there is no patch to validate. The requested direction is plausible, but the narrow maintainable path should be a phased design that separates the smaller per-agent lifecycle slice from the later distributed runtime and security protocol.
