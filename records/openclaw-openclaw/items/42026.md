## Summary

Keep open: current main and latest release still document one Gateway/control-plane process and a single embedded agent runtime per Gateway, while per-agent runtime lifecycle, runtime registration, distributed compute, and isolation semantics remain unimplemented. Related issues cover narrower Kubernetes, org/team, restart, and subagent-worker slices, but none supersedes this broader maintainer RFC.

## What This Changes

Request to split the single OpenClaw Gateway process into a control plane and separate per-agent compute runtimes with lifecycle, protocol, deployment, and isolation support.

## Best Possible Solution

Keep this as the canonical maintainer RFC, decide a narrow Phase 0/1 control-plane/runtime contract, then split approved work into focused protocol, lifecycle, security, deployment, docs, and validation issues.
