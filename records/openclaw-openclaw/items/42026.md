## Summary

Keep open: current main still documents and implements a single embedded Gateway-owned agent runtime, while the recent discussion adds active Phase 0/1 operator data rather than making the RFC obsolete.

## What This Changes

Request to split the Gateway control plane from per-agent compute runtimes, including local-split and distributed deployment modes with per-agent lifecycle and isolation.

## Best Possible Solution

Keep this as an architecture RFC and split it into maintainer-approved phases: per-agent lifecycle/registry first, then a versioned runtime protocol, trust model, health/reconnect behavior, and deployment docs.

## Reproduction Assessment

Not applicable. This is an RFC/feature request, and source inspection shows no first-class openclaw-runtime, local-split runtime protocol, or per-agent lifecycle CLI to exercise on current main.

## Solution Assessment

Unclear because there is no patch to validate. The requested direction is plausible, but the maintainable path should be phased design work that separates the smaller per-agent lifecycle slice from the later distributed runtime and security protocol.
