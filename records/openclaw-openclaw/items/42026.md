## Summary

Keep open. Current main still documents and implements OpenClaw as a single Gateway-owned control plane with embedded agent execution, while the comments and related reports narrow useful slices without replacing the broader distributed runtime/security architecture decision.

## What This Changes

Request to split Gateway control-plane duties from per-agent compute runtimes, including local-split and distributed deployment modes with per-agent lifecycle, resources, and secret isolation.

## Best Possible Solution

Keep this as the architecture umbrella while maintainers choose reviewed slices for per-agent lifecycle, runtime registration/protocol, secret isolation, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source and docs inspection give high-confidence negative capability evidence that current main still uses a single Gateway process with embedded agent execution.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The narrowest maintainable path appears to be reviewed Phase 0/1 slices around per-agent lifecycle, runtime registration, and auth/secret contracts before container or Kubernetes backends.
