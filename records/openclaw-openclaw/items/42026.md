## Summary

Keep open: current main still documents and implements Gateway as the process that owns routing/control plane and directly dispatches agent turns; the requested local-split/distributed runtime remains broad architecture work.

## What This Changes

Request to split `openclaw gateway` into a control plane plus per-agent runtime processes with local-split and distributed deployment modes.

## Best Possible Solution

Keep this as an RFC and split it into maintainer-approved phases: Phase 0 per-agent lifecycle, then a versioned runtime protocol, security/config model, deployment story, and migration plan.

## Reproduction Assessment

Not applicable. This is an RFC/feature request rather than a broken current behavior report, and source/docs inspection confirms the requested split is not a current runnable mode.

## Solution Assessment

No concrete patch is present, so there is no implementation to endorse. The best path is maintainer-scoped architecture design that separates the smaller per-agent lifecycle slice from the later distributed runtime protocol.
