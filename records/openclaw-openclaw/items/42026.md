## Summary

Keep open: current main and v2026.6.8 still use a single Gateway/control-plane process with embedded or in-process runtime surfaces, while the related open issues cover narrower backend, auth, org-deployment, or bug slices rather than superseding this broader distributed-runtime RFC.

## What This Changes

Request to split the monolithic Gateway into a control-plane process plus per-agent runtimes with local-split, containerized, and distributed deployment modes.

## Best Possible Solution

Keep this RFC open for maintainer architecture and security review, then split any accepted direction into narrower protocol, lifecycle, session-state, secret-isolation, and deployment milestones.
