## Summary

Keep open: current main still runs agent turns through the Gateway/embedded runner, while the issue asks for a new control-plane/runtime architecture with per-agent lifecycle and deployment modes.

## What This Changes

Request to split the Gateway control plane from per-agent compute runtimes and support embedded, local-split, and distributed deployment modes.

## Best Possible Solution

Keep this as an architectural RFC and have maintainers decide the first shippable slice, likely Phase 0 per-agent lifecycle or a formal runtime protocol design, while linking narrower implementation issues back to it.

## Reproduction Assessment

Not applicable. This is an RFC/feature request, and the current-main source/docs inspection is enough to show the requested runtime/control-plane split is not present.

## Solution Assessment

Unclear. The direction is plausible and supported by operator comments, but the best implementation needs maintainer architecture judgment before a repair PR can safely choose protocol, lifecycle, security, and deployment boundaries.
