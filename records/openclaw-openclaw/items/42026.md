## Summary

Keep open. Current main still uses gateway-owned embedded agent execution, and the newest discussion plus adjacent open PRs narrow possible Phase 0/1 slices without resolving the broader runtime, security, session, and channel-delivery architecture decision.

## What This Changes

The issue requests a distributed runtime architecture that separates the gateway control plane from per-agent compute runtimes while keeping embedded mode for compatibility.

## Best Possible Solution

Keep this as the distributed-runtime architecture umbrella while maintainers review or split focused implementation tracks for local worker isolation, runtime registration/protocol, per-agent lifecycle, channel/session delivery ownership, secret isolation, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, issue comments, and related PR inspection give high-confidence negative capability evidence that current OpenClaw still uses gateway-owned embedded agent execution for the central path.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The safer direction is to review the existing worker-runtime experiment and split Phase 0/1 issues around per-agent lifecycle, runtime registration, session/channel ownership, and secret contracts before container or Kubernetes backends.
