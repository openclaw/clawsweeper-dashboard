## Summary

Keep open. Current main still documents and implements Gateway-side embedded agent execution, with adjacent multi-agent state and runtime selection but no Gateway-to-per-agent-runtime protocol, runtime role, or per-agent start/stop/restart lifecycle.

## What This Changes

RFC requests splitting `openclaw gateway` into a control plane plus per-agent runtimes with local-split and distributed lifecycle/protocol support.

## Best Possible Solution

Keep this as an architecture tracker and split it into maintainer-reviewed milestones: embedded compatibility, local per-agent lifecycle, typed Gateway/runtime protocol, then container/Kubernetes orchestration with explicit threat-model review.

## Reproduction Assessment

Not applicable; this is an RFC/feature request, not a bug report. The capability gap is high-confidence from current source and docs showing embedded Gateway-side agent execution without per-agent runtime lifecycle or protocol support.

## Solution Assessment

Unclear as a direct implementation plan. The direction is plausible, but the maintainable path is a phased protocol and security design rather than one broad autonomous patch.
