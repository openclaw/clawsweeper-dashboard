- `P2`: This is meaningful multi-agent scaling and operability work, but it is broad architecture work rather than an urgent regression.
- `impact:session-state`: The requested control-plane/runtime split changes where session metadata, transcripts, active runs, and runtime ownership live.

## Summary

Keep open: current main and the latest shipped tag still implement gateway-owned embedded agent execution, while the newer related issues cover narrower lifecycle, worker-auth, or execution-backend pieces rather than superseding this distributed-runtime umbrella.

## What This Changes

RFC requests splitting the current `openclaw gateway` model into a control plane plus separately managed per-agent runtimes.

## Best Possible Solution

Keep this as the distributed-runtime architecture umbrella and split maintainer-approved milestones for runtime protocol/registration, per-agent lifecycle, session/channel routing, secret isolation, local worker mode, and deployment backends.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new runtime architecture. Source, docs, shipped-release, and related-item checks give high-confidence evidence that current OpenClaw still uses gateway-owned embedded execution for the central path.

## Solution Assessment

Unclear until maintainers choose the architecture boundary. The RFC is directionally plausible, but the maintainable path is to split Phase 0/1 milestones around lifecycle, runtime registration, session/channel ownership, and secret contracts before container or Kubernetes backends.
