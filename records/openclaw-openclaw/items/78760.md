## Summary

Keep open: current main still has source-level paths that can record nonzero `usage.cost` for `openai-codex/*` PI turns even though the docs say OAuth usage should show tokens only, and the linked Realtime Talk report is adjacent rather than a duplicate.

## What This Changes

Request that Codex OAuth / `openai-codex/*` session usage distinguish subscription-backed token usage from authoritative per-token billing cost.

## Best Possible Solution

Keep token counters for Codex OAuth but suppress, zero, or explicitly mark dollar cost as non-authoritative in transcripts and session summaries, with regression coverage and docs alignment.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: run a current-main `openai-codex/gpt-5.4` PI turn with OAuth and inspect the assistant JSONL or `usage.cost` summary. I did not run the live OAuth turn in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to align `openai-codex` OAuth cost persistence with the existing documented OAuth token-only policy. A broader `billingMode` schema can be a later feature if maintainers want estimated token-equivalent costs preserved separately.
