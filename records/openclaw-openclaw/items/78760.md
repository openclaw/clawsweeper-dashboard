## Summary

Keep open: current main still has nonzero OpenAI Codex pricing metadata and unconditionally calls cost calculation for `openai-codex` usage, while the docs say OAuth tokens never show dollar cost. The related fix PR was closed unmerged, so there is no active merged implementation covering this issue.

## What This Changes

Codex OAuth / ChatGPT subscription-backed OpenAI Codex usage should not be recorded as authoritative per-token dollar cost in session logs or cost-tracking workflows.

## Best Possible Solution

Preserve token counters for `openai-codex` OAuth turns while suppressing, zeroing, or explicitly marking dollar cost as non-authoritative across transcript usage and session cost summaries, with regression coverage for the transport and session surfaces.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run a current-main `openai-codex/gpt-5.4` OAuth turn and inspect assistant transcript JSONL or session summaries for nonzero `usage.cost` from the unconditional cost calculation paths.

## Solution Assessment

Yes. The narrowest maintainable fix is to align existing usage persistence with the documented OAuth token-only cost policy; a broader `billingMode` schema can be a later product enhancement if maintainers want estimated token-equivalent costs retained separately.
