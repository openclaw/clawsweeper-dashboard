## Summary

Keep open: current main still has nonzero cost-producing paths for OpenAI Codex OAuth usage, while the token-use docs say OAuth auth hides dollar cost and the only linked fix PR was closed unmerged.

## What This Changes

Request that OpenClaw stop recording authoritative per-token dollar cost for OpenAI Codex OAuth/subscription turns while preserving token usage.

## Best Possible Solution

Make Codex OAuth cost persistence auth-aware: keep token counters, but suppress, zero, or mark dollar cost as non-authoritative for subscription-backed OAuth paths without hiding real API-key billing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run a current-main OpenAI Codex OAuth turn, then inspect the assistant JSONL or usage cost summary where the unconditional cost calculation and cost aggregation paths can produce nonzero `usage.cost`.

## Solution Assessment

Yes for the direction: the narrow fix is to align persisted/session cost with the documented OAuth token-only policy. A broader `billingMode` field can be a follow-up only if maintainers want to preserve estimated token-equivalent cost separately.

Fix the OpenAI Codex OAuth cost-accounting bug reported in https://github.com/openclaw/openclaw/issues/78760. Current main keeps token usage for `openai-codex`/Codex-OAuth turns but can write nonzero `usage.cost` through `src/agents/openai-transport-stream.ts`, session persistence, and session cost aggregation, despite docs saying OAuth hides dollar cost. Add focused regression coverage for both OpenAI transport usage paths and the session/COST_LOG aggregation path. Update docs only if behavior wording changes, and add a changelog fix entry because this is user-visible cost reporting. Stop and ask for maintainer review if the implementation requires a new public billing schema instead of a narrow auth-aware cost fix.
