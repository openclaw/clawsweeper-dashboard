## Summary

Keep open: current main handles the modern default Codex app-server/OAuth path as token-only or zero-cost and documents that route, but explicit legacy/direct OpenAI transport paths can still calculate local model prices, and the remaining billing semantics need maintainer product direction rather than an automatic close.

## What This Changes

The issue asks OpenClaw to stop presenting Codex OAuth/subscription-backed session usage as authoritative per-token API spend, or to explicitly mark/document the cost as estimated, non-billable, or metered.

## Best Possible Solution

Define one billing semantics contract: Codex app-server/OAuth usage stays token-only or explicitly non-authoritative, API-key routes keep estimated cost, and any legacy/direct OAuth fallback is migrated or marked with tested billing metadata and docs.

## Reproduction Assessment

Yes, from source for the direct OpenAI/OpenClaw transport path: current tests preserve a legacy Codex OAuth profile through OpenAI transport and that transport still calls price-table cost calculation; I did not run a live OAuth session.

## Solution Assessment

Unclear: current main's default Codex harness and docs solve much of the modern user path, but the best remaining fix is not blanket zeroing because metered API-key spend must stay visible.
