## Summary

Keep open: current main still has the same source-level mismatch. The docs say OAuth usage hides dollar cost, but `openai-codex` model metadata still carries nonzero pricing and transcript/session cost paths still compute from that metadata; the linked fix PR at https://github.com/openclaw/openclaw/pull/78814 is closed and unmerged.

## What This Changes

The issue asks OpenClaw to stop recording non-authoritative dollar cost for `openai-codex/*` OAuth subscription-backed sessions, or to clearly document the billing semantics.

## Best Possible Solution

Preserve token counters for OpenAI Codex OAuth while suppressing or explicitly marking non-authoritative dollar cost in transcript, session summary, and `/usage cost` aggregation paths, with focused regression coverage and docs/changelog alignment.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: run a current-main `openai-codex/gpt-5.4` OAuth turn with provider usage metadata, then inspect the assistant JSONL and session cost summaries. I did not run the live OAuth turn in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to make persisted cost reporting auth/provider-aware for Codex OAuth while preserving token counts and API-key billing estimates. A transport-only guard like the closed source PR proposed is incomplete unless session-store and aggregation recomputation paths are also covered.
