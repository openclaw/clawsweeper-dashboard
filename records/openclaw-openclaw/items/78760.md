## Label Justifications

- `P2`: The issue affected provider/auth billing semantics and user trust in cost logs, but current evidence shows it is now resolved without an outage or security bypass.
- `impact:auth-provider`: The behavior depends on distinguishing Codex OAuth subscription auth from OpenAI API-key billing for model turns.
- `impact:other`: The issue affected cost accounting workflows and operator trust in local session logs, which is maintainer-visible but outside the more specific impact categories.

## Summary

Close: current main and the latest release no longer log Codex subscription turns as nonzero per-token dollar spend; native Codex transcript usage carries zero dollar cost, unknown all-zero pricing is treated as missing cost for usage-cost aggregation, and docs route subscription users through Codex OAuth with API keys removed from the Codex child process.

## What This Changes

The issue asks OpenClaw to distinguish Codex OAuth or ChatGPT subscription-backed usage from authoritative per-token API billing in session logs and cost-log workflows.

## Best Possible Solution

Keep the shipped native Codex billing behavior: subscription-backed Codex turns should preserve token usage while avoiding authoritative per-token dollar spend, and direct API-key cost accounting should remain separate.

## Reproduction Assessment

No for current main: the old 2026.4.21 observation is credible, but current source shows native Codex turns emit zero dollar cost and usage-cost treats unknown all-zero pricing as missing rather than nonzero spend.

## Solution Assessment

Yes. The implemented owner-boundary solution is better than reviving legacy `openai-codex` transport guards because current docs and migration policy use canonical `openai/*` refs with the Codex plugin while keeping API-key spend separate.
