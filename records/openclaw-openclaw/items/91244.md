- **[P1] Disable Bolt signature checks for trusted upstream:** `extensions/slack/src/monitor/provider-support.ts:347-350`
  - body: `trusted-upstream` is documented as not requiring a Slack signing secret, but this still constructs `HTTPReceiver` without `signatureVerification: false`. Pass the override only for this mode and keep normal HTTP mode verified.
  - confidence: 0.96
- **[P2] Use trusted bot identity in monitor context:** `extensions/slack/src/monitor/provider.ts:429-436`
  - body: The new `trustedUpstream.botUserId` and `botId` are only passed to Bolt `authorize`, while OpenClaw's own context still comes from `auth.test`. Placeholder-token deployments therefore leave `ctx.botUserId` empty, which disables explicit mention and self-detection paths that rely on the context identity. Feed the configured trusted identity into `createSlackMonitorContext` when this mode skips Slack auth.
  - confidence: 0.88
