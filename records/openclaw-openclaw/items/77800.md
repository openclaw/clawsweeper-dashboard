## Summary

Keep this open. Current main and latest release still exact-match `contracts.tools` and reject missing or undeclared dynamic `api.registerTool()` names, while the linked closing PR is open, unmerged, and still needs maintainer API/security judgment plus proof.

## What This Changes

The issue asks OpenClaw to stop silently hiding plugin tools registered via `api.registerTool()` when dynamic tools cannot be predeclared exactly in `contracts.tools`.

## Best Possible Solution

Land a maintainer-approved plugin tool ownership contract that preserves manifest-first discovery, keeps concrete allowlists narrow, and includes focused docs, regression tests, and real plugin/Gateway proof.

## Reproduction Assessment

Yes, source-level: current main records a diagnostic and returns when `contracts.tools` is absent, and the adjacent loader test asserts no tools are registered. I did not run a live plugin repro because this review is read-only.

## Solution Assessment

Unclear for final implementation. Bounded wildcard contracts in the linked PR are plausible, but the permanent plugin API direction and allowlist/security semantics need maintainer approval before this issue can close.
