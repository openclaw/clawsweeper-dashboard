## Summary

Keep this open. Current main and the latest release still enforce exact `contracts.tools` ownership and drop undeclared dynamic tool registrations through diagnostics, while the linked implementation candidate at https://github.com/openclaw/openclaw/pull/78084 is still open and unmerged.

## What This Changes

The issue asks OpenClaw to stop silently hiding plugin tools registered through `api.registerTool()` when static `contracts.tools` cannot enumerate dynamic tool names.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/78084 so dynamic plugin tool ownership has one documented, tested contract while preserving manifest-first discovery.

## Reproduction Assessment

Yes from source inspection: current main records a diagnostic and returns when `contracts.tools` is absent, and the adjacent loader test asserts no tools are registered. I did not run a live plugin repro because this review is read-only.

## Solution Assessment

Unclear for final implementation: bounded wildcard contracts in the linked PR are a plausible fix, but the plugin API direction still needs PR review and merge before this issue can close.
