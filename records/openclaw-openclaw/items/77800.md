## What This Changes

The issue asks OpenClaw to stop silently dropping `api.registerTool()` registrations for plugins without static `contracts.tools`, including a workable path for dynamic tool catalogs.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/78084 or an equivalent maintainer-approved dynamic tool ownership and diagnostic design, then close this issue only after the chosen implementation lands.

## Reproduction Assessment

Yes, at source level: current `src/plugins/registry.ts` returns after recording the reported diagnostic, and `src/plugins/loader.test.ts` has a focused fixture asserting no tool is registered without `contracts.tools`. I did not run tests because this review is read-only.

## Solution Assessment

Unclear: bounded wildcard ownership looks like the most plausible current direction because it preserves manifest-first discovery, but maintainers still need to accept the public contract and require real behavior proof before merge.
