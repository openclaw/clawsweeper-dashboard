## Summary

Keep open: the feature is not on current main, and the PR is useful directionally, but the latest patch still exposes live tool schema parameter references through an observation hook and leaves the new public hook fields undocumented.

## What This Changes

This PR adds optional `tools` and `clientTools` descriptors to `PluginHookLlmInputEvent`, populates them from the embedded runner's tool lists, updates a hook runner test, and refreshes the plugin SDK API hash.

## Best Possible Solution

Ship this as a documented plugin SDK contract change that emits copied, normalized tool descriptors for built-in and hosted client tools after effective tool selection, without letting telemetry hooks alter request state.

## Reproduction Assessment

Yes for the blocking patch defect: a test can register an `llm_input` handler that mutates `event.tools[0].parameters` and then verify the tool definition sent to the model is unchanged. Static inspection already shows the PR passes the same parameter objects that the request adapters retain.

## Solution Assessment

No. Exposing the descriptors on `llm_input` is a reasonable API shape, but the implementation should use a small mapper that isolates parameters, normalizes missing descriptions to match the request path, and adds docs/changelog plus populated `clientTools` coverage.

- **[P2] Clone tool parameters before emitting llm_input:** `src/agents/pi-embedded-runner/run/attempt.ts:2048-2053`
  - body: The new event passes `tool.parameters` and `tool.function.parameters` by reference into an observation hook.   - body: The model request adapters normalize missing tool descriptions to `""`, but this event emits raw optional descriptions. Tracing or eval plugins that compare the hook payload to the actual request can see false drift whenever a built-in or client tool omits its description.
  - confidence: 0.84
- **[P3] Document and changelog the new hook fields:** `src/plugins/hook-types.ts:176-177`
  - body: This changes the public plugin hook contract, but the PR only updates the type and generated API hash. `docs/plugins/hooks.md` still describes `llm_input` as prompt/history only, and there is no Unreleased changelog entry for the plugin SDK feature.
  - confidence: 0.86

## Security Review

Concerns:

- **[medium] Mutable tool schemas cross the observation hook boundary:** `src/agents/pi-embedded-runner/run/attempt.ts:2048`
