## Summary

Keep open: current main still exposes session reset policy only globally, by chat type, or by channel, while Telegram group/topic config does not accept or apply `session.reset`. The closed implementation at https://github.com/openclaw/openclaw/pull/8134 was not merged, and the broader scoped-reset tracker at https://github.com/openclaw/openclaw/issues/60659 overlaps but does not replace this concrete Telegram group/topic request.

## What This Changes

The issue requests per-Telegram-group session reset configuration so selected groups can reset independently from global, type, or channel reset policies.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and define precedence rather than adding a group-id-only shortcut.
