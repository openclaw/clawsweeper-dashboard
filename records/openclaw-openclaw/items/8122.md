## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration: the proposed config shape is rejected by strict Telegram schemas, and the reset runtime only receives global/type/channel policy inputs. The earlier implementation PR #8134 is closed unmerged, while #60659 is a broader open lifecycle request rather than a precise replacement for this Telegram group use case.

## What This Changes

Support configuring idle/daily session reset per Telegram group or topic instead of only globally, by chat type, or by channel.

## Best Possible Solution

Add a backwards-compatible, account-aware conversation-scoped reset override, or explicitly support Telegram group/topic session reset with documented precedence over channel, type, and global policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: the proposed channels.telegram.groups.<id>.session.reset shape is absent from strict Telegram schemas, and initSessionState never passes group/topic context into the reset resolver.

## Solution Assessment

Unclear. The requested Telegram group shape is plausible, but the maintainable fix may be a generic conversation-scoped reset contract if maintainers want this behavior across bindings, agents, or other channel plugins.
