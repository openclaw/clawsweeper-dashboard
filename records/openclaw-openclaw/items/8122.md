## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic reset configuration, and the related implementation PR was closed unmerged; the remaining work is a public config/API design choice rather than cleanup-closeable stale work.

## What This Changes

Add per-group Telegram session reset configuration so selected groups can use shorter idle resets without changing all Telegram or all group sessions.

## Best Possible Solution

Define an additive account-aware conversation reset override after Telegram routing resolves the effective group/topic, then update config types, strict schemas, docs, generated metadata, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives a high-confidence verification path because current main rejects the proposed nested config and the reset resolver has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The narrow maintainable direction is an account-aware Telegram group/topic override or a generic conversation-scoped resolver, not a naive group-id scan.
