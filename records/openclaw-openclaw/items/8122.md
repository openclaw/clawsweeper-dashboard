## Summary

Keep open. Current main still supports session-reset overrides only globally, by chat type, or by channel; Telegram group/topic configuration remains strict and cannot supply a conversation-specific reset policy. The broader open request at https://github.com/openclaw/openclaw/issues/60659 covers agent/binding policy and overlaps architecturally, but does not replace this distinct group/topic use case.

## What This Changes

Add conversation-specific session reset configuration so selected Telegram groups can reset independently of global, type, or channel policy.

## Best Possible Solution

Define one account-aware conversation-scoped reset-policy input with explicit precedence, then expose it through Telegram group/topic configuration while coordinating the same resolver contract with the agent/binding request.

## Reproduction Assessment

Yes at source level. Current main's strict Telegram schemas and reset-policy call path clearly lack a conversation-scoped override, although this is a requested capability rather than broken documented behavior.

## Solution Assessment

No, the proposed Telegram-only nested configuration is not yet proven to be the best durable solution. The implementation should first establish a generic account-aware precedence contract that can serve group/topic and agent/binding policy without parallel resolution paths.
