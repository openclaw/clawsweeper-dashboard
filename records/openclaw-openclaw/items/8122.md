## Summary

Keep open: current main and the latest release still only support session reset policy globally, by chat type, or by channel, while Telegram group/topic config does not accept or apply a scoped reset policy. The linked implementation at https://github.com/openclaw/openclaw/pull/8134 is closed unmerged, and the broader tracker at https://github.com/openclaw/openclaw/issues/60659 overlaps but does not replace this Telegram group/topic use case.

## What This Changes

The issue requests Telegram group/topic-level session reset configuration so selected groups can expire quickly without changing all groups or channels.

## Best Possible Solution

Define one additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and runtime tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the public API shape; the proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with the broader scoped-reset tracker.
