## Summary

Keep this issue open. Current main still lacks conversation-scoped reset policy, and the broader agent/binding request explicitly treats this Telegram group/topic use case as overlapping but distinct; maintainers need to choose one coordinated public configuration model.

## What This Changes

Allow individual Telegram groups to override session reset timing without changing reset behavior for other groups or channels.

## Best Possible Solution

Define one additive, account- and route-aware scoped reset-policy model with explicit precedence that covers both Telegram group/topic and agent/binding workflows, then align types, strict schemas, config metadata, public docs, and focused precedence tests.

## Reproduction Assessment

Yes at source level. Current main's config types, strict schemas, resolver call, tests, and docs establish that reset policy is limited to global, chat-type, and channel scopes; this is a missing capability rather than a failing existing contract.

## Solution Assessment

Unclear for the proposed Telegram-only shape. The maintainable solution should coordinate this issue with https://github.com/openclaw/openclaw/issues/60659 and define one scoped precedence model instead of adding isolated policy lookups.
