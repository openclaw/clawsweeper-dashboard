## Summary

Keep open: current main and the latest release still lack per-Telegram-group or per-topic session reset configuration, the linked implementation PR is closed unmerged, and the broader scoped-reset issue does not fully replace this Telegram group use case.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can reset independently from other Telegram groups.

## Best Possible Solution

Define one additive, account-aware scoped reset-policy model with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and runtime tests.

## Reproduction Assessment

Yes, at source level: current main does not accept or apply channels.telegram.groups.<id>.session.reset because the Telegram group schema is strict and the runtime only passes a channel reset override. This is not a broken existing contract reproduction; it is a missing requested configuration capability.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with the broader scoped-reset tracker.
