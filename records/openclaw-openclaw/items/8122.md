## Summary

Keep open: current main and the latest release still lack per-Telegram-group or per-topic session reset configuration. The closed implementation PR was not merged, and the broader per-agent/per-binding reset issue overlaps but does not replace this specific Telegram group/topic use case.

## What This Changes

The issue requests per-Telegram-group session reset configuration so selected groups can use short idle resets while other groups keep normal session continuity.

## Best Possible Solution

Define one additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Yes. Source inspection shows the requested nested group/topic reset shape is absent from Telegram types and strict schemas, and runtime reset selection only receives global/type/channel inputs.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed Telegram group config is plausible, but the maintainable solution should be account-aware and coordinated with the broader scoped-reset tracker.
