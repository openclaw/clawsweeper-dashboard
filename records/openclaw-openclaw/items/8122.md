## Summary

Keep open. Current main still only supports global, chat-type, and channel-level reset policy, while Telegram group/topic config rejects the requested nested session reset shape.

## What This Changes

Request per-Telegram-group session reset settings so short-lived TTS groups can reset independently from other Telegram groups.

## Best Possible Solution

Add a backward-compatible, account-aware conversation reset override with documented precedence over channel, type, and global policy, either in Telegram group/topic config or in a generic binding-scoped session policy.

## Reproduction Assessment

Yes for the reported limitation. Source inspection on current main verifies the proposed Telegram group/topic session.reset shape is absent from strict schemas and reset resolution only receives global, type, and channel inputs.

## Solution Assessment

Unclear. The requested Telegram groups.<id>.session.reset shape would satisfy the use case, but maintainers should first choose whether this belongs in Telegram config or a generic account-aware conversation/binding reset API.
