## Summary

Keep open. Current main still lacks per-Telegram group/topic session reset configuration: the public config types and strict schemas reject the proposed nested `session.reset` shape, and runtime reset selection still only considers global, type, and channel-level policy. The paired implementation PR is closed unmerged, while the broader open reset-policy request does not fully replace this Telegram group/topic use case.

## What This Changes

Request per-Telegram-group session reset settings so selected groups can use short idle resets without changing all groups or all Telegram sessions.

## Best Possible Solution

Define an additive account-aware Telegram group/topic reset override, or a generic conversation-scoped resolver, with explicit precedence plus schema, docs, metadata, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: the proposed config shape is absent from current types/schemas, and runtime reset selection has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is an account-aware Telegram group/topic override or a generic routed reset resolver, not a naive group-id scan.
