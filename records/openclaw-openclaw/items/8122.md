Keep open: current main and the latest release still lack per-Telegram-group/topic session reset configuration, and the remaining work is a public config/API precedence decision rather than cleanup. ## What This Changes

Request support for configuring session reset policy for a specific Telegram group instead of only globally, by message type, or by channel.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Solution Assessment
