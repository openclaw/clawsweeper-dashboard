## Summary

Keep open: current main still lacks a per-Telegram-group or per-topic session reset policy surface, and the remaining work is a public config/API precedence decision rather than conservative cleanup.

## What This Changes

Request per-Telegram-group session reset configuration so specific groups can reset after short idle windows without changing all groups or all Telegram sessions.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should define account-aware scope and precedence rather than adding a one-off shortcut.
