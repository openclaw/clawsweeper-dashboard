## Summary

Keep open. Current main still lacks per-Telegram-group or topic reset config and only applies global, type, or channel reset policy; the closed implementation at https://github.com/openclaw/openclaw/pull/8134 was not merged, and https://github.com/openclaw/openclaw/issues/60659 is related but broader rather than an exact replacement.

## What This Changes

Request per-Telegram-group session reset settings so selected groups can use short idle reset policies without affecting other groups.

## Best Possible Solution

Define an additive account-aware reset override after Telegram routing identifies the effective group/topic, or introduce a generic conversation-scoped resolver with documented precedence.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives high confidence that current main lacks the requested per-group reset configuration path.

## Solution Assessment

Unclear until maintainers choose the API shape. The maintainable direction is either explicit Telegram group/topic reset config with account-aware lookup or a generic conversation-scoped resolver with documented precedence.
