## Summary

Keep open. Current main and v2026.6.5 still limit automatic session-reset overrides to global, chat-type, and channel scopes; Telegram group/topic configuration cannot express the requested policy. The closed, unmerged implementation at https://github.com/openclaw/openclaw/pull/8134 does not resolve the request, while https://github.com/openclaw/openclaw/issues/60659 overlaps architecturally but covers agent/binding policy rather than this distinct group/topic use case.

## What This Changes

Allow individual Telegram groups to use their own session reset policy without changing session continuity for other groups.

## Best Possible Solution

Add one canonical, account-aware conversation reset override with documented precedence, then let the chosen configuration owner—Telegram group/topic or generic binding—supply it while reusing the existing reset resolver.

## Reproduction Assessment

Yes at source level. The exact current-main types, docs, tests, and runtime call path show that a Telegram group or topic cannot provide a reset policy to the canonical resolver.

## Solution Assessment

Unclear pending a product decision. The requested nested Telegram configuration is plausible, but the maintainable fix should first establish a single account-aware conversation override and precedence model shared with related agent/binding needs where appropriate.
