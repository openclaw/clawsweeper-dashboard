## Summary

Keep open: current main still lacks per-Telegram-group or per-topic session reset configuration. The config types and strict schemas do not accept the requested `session.reset` shape, runtime reset selection only considers channel/type/global policy, the linked implementation PR is closed unmerged, and the broader scoped-reset tracker overlaps without replacing this concrete Telegram request.

## What This Changes

The issue requests per-Telegram-group session reset configuration so selected Telegram groups can use different reset timing from global, type, or channel defaults.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated config metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with the broader scoped-reset tracker rather than adding a group-id-only shortcut.
