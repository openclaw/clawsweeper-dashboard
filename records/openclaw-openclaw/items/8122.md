## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic reset configuration, the paired implementation PR is closed unmerged, and the related broader per-agent/binding request does not fully replace this Telegram group/topic use case.

## What This Changes

Support per-Telegram-group session reset settings so selected groups can reset independently of global, type-level, or channel-level policies.

## Best Possible Solution

Define an additive account-aware conversation reset override after Telegram routing resolves the effective group/topic, then update config types, strict schemas, docs, generated metadata, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives a high-confidence verification path because current main rejects the proposed nested config and reset policy resolution has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The narrow maintainable direction is an account-aware Telegram group/topic override or a generic conversation-scoped resolver, not a naive group-id scan.
