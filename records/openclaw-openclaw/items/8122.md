## Summary

Current main still does not support this configuration: Telegram group/topic config types and strict schemas lack a session reset field, and runtime reset selection only considers global, type, and channel policy. The earlier implementation PR is closed unmerged; #60659 is a broader open design thread, not an implementation.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can auto-reset independently of global, type, or channel-wide policies.

## Best Possible Solution

Add an account-aware conversation-scoped reset policy, or explicitly add Telegram group/topic reset overrides if maintainers choose a Telegram-only surface, with aligned types, strict schemas, docs, labels/help, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives high confidence that current main lacks the requested per-group reset configuration path.

## Solution Assessment

Unclear. The user need is concrete, but maintainers should decide whether the narrow maintainable solution is Telegram group/topic config or a generic conversation/binding reset override that also covers #60659.
