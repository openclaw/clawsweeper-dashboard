## Summary

Keep this issue open. Current main still lacks per-Telegram-group or per-topic session reset configuration, the related implementation PR is closed unmerged, and the broader per-agent/binding request does not clearly own this exact Telegram group/topic behavior.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can use short idle resets while other chats keep normal session continuity.

## Best Possible Solution

Add an additive, account-aware Telegram group/topic reset override or a generic conversation-scoped reset resolver, then update strict schemas, types, generated metadata/help, public docs, and precedence tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence verification that the proposed config is not accepted and the runtime cannot select reset policy by Telegram group or topic.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction should be account-aware and should define explicit precedence rather than adding a group-id-only shortcut.
