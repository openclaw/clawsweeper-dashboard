## Summary

Keep open: current main still lacks per-Telegram-group/topic reset configuration, and the remaining work is a public config and precedence decision rather than stale cleanup. The closed implementation PR was not merged, and the broader per-agent/per-binding tracker overlaps but does not replace this Telegram-specific request.

## What This Changes

Request per-group Telegram session reset configuration so selected Telegram groups can use different idle/session reset behavior than the global, type, or channel policy.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and define precedence rather than adding a one-off shortcut.
