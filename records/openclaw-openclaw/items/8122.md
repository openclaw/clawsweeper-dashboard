## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration, and the only direct implementation PR is closed unmerged while the broader reset-policy issue does not fully cover this Telegram-specific request.

## What This Changes

The issue requests a public config surface for per-Telegram-group session reset policy so selected groups can use shorter idle resets than other groups.

## Best Possible Solution

Define an additive account-aware Telegram group/topic reset override, or a generic conversation-scoped resolver, with explicit precedence plus schema, docs, metadata, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: the proposed config shape is absent from current types/schemas, and runtime reset selection has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is an account-aware Telegram group/topic override or a generic conversation-scoped reset resolver, not a naive group-id scan.
