## Summary

Keep open: current main still lacks per-Telegram-group or per-topic session reset configuration. The requested shape is outside the strict Telegram config schema, runtime reset resolution still only considers global/type/channel policy, and the closed implementation PR plus broader scoped-reset issue do not fully own this concrete Telegram group use case.

## What This Changes

The issue requests per-Telegram-group session reset settings so some groups can reset quickly while others keep normal session continuity.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated config metadata/help, public docs, and runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with the broader scoped-reset tracker rather than adding a group-id-only shortcut.
