## Summary

Current main still lacks the requested per-Telegram-group session reset override, so this should remain open. The source, strict config schemas, docs, runtime reset resolver, and linked PR state all match the prior maintainer-facing review: only global, chat-type, and channel-level reset policies are currently supported.

## What This Changes

Request per-Telegram-group session reset configuration so selected Telegram groups can auto-reset independently of global, type-level, or channel-level session policy.

## Best Possible Solution

Add a backwards-compatible conversation-scoped reset override that can represent Telegram group/topic policy with account-aware precedence, then update config types, strict schemas, generated help/labels, public docs, and focused tests proving topic/group overrides beat channel, type, and global defaults.

## Reproduction Assessment

Yes. Add the proposed `channels.telegram.groups.<id>.session.reset` config to current main: the strict Telegram group schema has no `session` property, and the runtime reset path never reads group/topic config before resolving the policy.

## Solution Assessment

Unclear. The requested Telegram shape is plausible, but a generic conversation-scoped resolver may be the more maintainable fix if maintainers want the same capability for bindings, agents, topics, or other channel plugins.
