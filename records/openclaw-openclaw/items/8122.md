## Summary

Keep open: current main and the latest release still lack per-Telegram-group or per-topic session reset configuration, and the remaining work needs a public config and precedence decision rather than cleanup closure.

## What This Changes

Request per-group Telegram session reset settings so selected groups can use short idle resets without changing all groups or the whole Telegram channel.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with the broader scoped-reset tracker rather than adding a group-id-only shortcut.
