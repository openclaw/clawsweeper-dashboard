## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration, and the broader per-agent/binding request does not fully replace this channel-specific use case.

## What This Changes

Request per-group Telegram session reset configuration so selected groups can use short idle resets without changing global, type, or channel-wide policy.

## Best Possible Solution

Define an additive account-aware Telegram group/topic reset override, or a generic conversation-scoped resolver if maintainers want this across channels, then update config types, strict schemas, docs/metadata, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives high confidence that the proposed config is not accepted today and runtime reset selection lacks group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The safest direction is an account-aware Telegram group/topic override or generic routed reset resolver, not a naive group-id lookup.
