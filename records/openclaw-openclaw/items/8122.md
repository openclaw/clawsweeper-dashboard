## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration, the linked implementation PR is closed unmerged, and the broader per-agent/binding request does not fully settle the Telegram-specific API and precedence choice.

## What This Changes

Request configurable session reset policy for individual Telegram groups or topics instead of only global, chat-type, or channel-wide scopes.

## Best Possible Solution

Add a backward-compatible, account-aware reset override with documented precedence over channel, type, and global policy, either in Telegram group/topic config or through a generic conversation-scoped resolver.

## Reproduction Assessment

Yes for the reported limitation. Source inspection on current main verifies the proposed Telegram group/topic session.reset shape is absent from strict schemas and reset resolution only receives global, type, and channel inputs.

## Solution Assessment

Unclear. The requested Telegram groups.<id>.session.reset shape would satisfy the use case, but maintainers should first choose whether this belongs in Telegram config or a generic account-aware conversation/binding reset API.
