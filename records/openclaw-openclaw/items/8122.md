## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration, the direct implementation PR is closed unmerged, and the broader per-agent/per-binding issue does not fully replace this Telegram-specific request.

## What This Changes

Add per-Telegram-group session reset config, such as `channels.telegram.groups.<id>.session.reset`, so selected groups can expire faster than other chats.

## Best Possible Solution

Define an additive account-aware Telegram group/topic reset override, or a generic conversation-scoped reset resolver, with explicit precedence plus schema, docs, metadata, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: the proposed config shape is absent from current types/schemas and runtime reset selection has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is an account-aware Telegram group/topic override or a generic conversation-scoped reset resolver, not a naive group-id-only lookup.
