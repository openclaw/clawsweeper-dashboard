## Summary

Keep open: current main and the latest shipped release still only support global, type, and channel reset policies, not per-Telegram-group or per-topic reset policies. The broader scoped-reset request at https://github.com/openclaw/openclaw/issues/60659 overlaps, but it does not fully supersede this concrete Telegram group use case.

## What This Changes

Requested per-Telegram-group session reset configuration so selected Telegram groups can reset independently from global, type, and channel defaults.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but a generic scoped reset resolver may be the more maintainable direction if it also covers bindings and agents.
