## Summary

Keep open: current main and v2026.6.5 still expose session reset only globally, by chat type, or by channel, and the broader scoped-reset tracker overlaps but does not replace this Telegram group/topic request.

## What This Changes

The issue requests per-Telegram-group session reset settings so independent task groups can use short idle resets without changing all Telegram groups.

## Best Possible Solution

Define one additive, account-aware scoped reset resolver with explicit precedence, then expose it through the chosen Telegram group/topic or broader binding/agent config surface with docs and tests aligned.

## Reproduction Assessment

Yes for the configuration gap. Source inspection shows the requested nested group/topic reset shape is absent from Telegram types and strict schemas, and runtime reset selection only receives global/type/channel inputs.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed Telegram group config is plausible, but the maintainable solution should be account-aware and coordinated with https://github.com/openclaw/openclaw/issues/60659.
