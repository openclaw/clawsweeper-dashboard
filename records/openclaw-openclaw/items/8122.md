## Summary

Keep open. Current main still supports session reset policy only at global, chat-type, and channel scope; the requested Telegram group/topic override is not accepted by config validation or consulted by the reset runtime.

## What This Changes

The issue requests per-Telegram-group session reset configuration so selected Telegram groups can use a shorter idle reset policy without changing all groups.

## Best Possible Solution

Add a backwards-compatible, account-aware conversation-scoped reset override, or explicitly add Telegram group/topic reset config if maintainers want this scoped to Telegram only.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: the proposed `channels.telegram.groups.<id>.session.reset` shape is absent from strict Telegram schemas, and the current runtime reset resolver never receives group/topic context.

## Solution Assessment

Unclear. The requested Telegram shape is plausible, but the maintainable fix may be a generic conversation-scoped reset contract if maintainers want the same behavior for bindings, agents, or other channel plugins.
