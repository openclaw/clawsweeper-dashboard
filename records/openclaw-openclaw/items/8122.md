## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration, the linked implementation PR is closed unmerged, and the broader per-agent/binding request does not clearly replace this Telegram-specific feature request.

## What This Changes

Request per-Telegram-group session reset configuration so selected Telegram groups can reset independently from global, type, or channel policy.

## Best Possible Solution

Add an additive, account-aware Telegram group/topic reset override or a generic conversation-scoped reset resolver with explicit precedence, strict schema/type/docs updates, and focused precedence tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence verification that the proposed config is not accepted and runtime reset selection cannot choose by Telegram group or topic.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction should be account-aware and define precedence over channel/type/global policy rather than adding a group-id-only shortcut.
