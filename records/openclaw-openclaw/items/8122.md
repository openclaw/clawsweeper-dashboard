## Summary

Keep open: current main still lacks a per-Telegram-group or per-topic reset policy contract, and the runtime reset resolver only receives global, type, and channel-level inputs. This is a public config/API feature that needs maintainer product review, not a stale, duplicate, or already implemented request.

## What This Changes

Request per-group Telegram session reset configuration so selected groups can use short idle resets without changing all groups or all Telegram sessions.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and define precedence rather than adding a one-off shortcut.
