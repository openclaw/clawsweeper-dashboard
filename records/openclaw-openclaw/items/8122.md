Keep this issue open: current main still lacks per-Telegram-group or per-topic session reset configuration, the direct implementation PR is closed unmerged, and the broader per-agent/binding tracker overlaps but does not clearly replace this Telegram-specific request.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can use short idle resets without changing all groups or all Telegram sessions.

## Best Possible Solution

Add an additive, account-aware Telegram group/topic reset override or a generic conversation-scoped reset resolver, with explicit precedence, aligned types/schemas/help/docs, and focused runtime tests.

## Solution Assessment
