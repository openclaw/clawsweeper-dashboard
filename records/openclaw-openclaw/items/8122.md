## Summary

Keep open. Current main still has only global, type, and channel-level session reset policy; Telegram group/topic config does not accept the requested nested reset shape, and the paired implementation PR is closed unmerged while the broader related issue is not a full replacement for this Telegram group use case.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can use a shorter idle reset policy than other groups.

## Best Possible Solution

Define an additive, account-aware reset override for Telegram group/topic config, or a generic conversation-scoped resolver if maintainers want the behavior across channels, then update config types, strict schemas, docs/metadata, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: the proposed config is not accepted by the current types/schemas, and runtime reset selection has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is an account-aware Telegram group/topic override or a generic routed reset resolver, not a naive group-id scan.
