## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration, the strict config schemas reject the proposed nested shape, runtime reset selection only considers global/type/channel policy, and the direct implementation PR at https://github.com/openclaw/openclaw/pull/8134 is closed unmerged.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can expire independently of global, type, or channel reset defaults.

## Best Possible Solution

Define an additive account-aware Telegram group/topic reset override, or a generic conversation-scoped resolver, with explicit precedence over channel/type/global policy plus schemas, docs, metadata, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: current main lacks the proposed config shape and runtime reset selection has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is account-aware Telegram group/topic reset policy or a generic conversation-scoped resolver, not a naive group-id-only lookup.
