## Summary

Current main still lacks conversation-scoped reset overrides, but the newer open tracker covers the same product decision through per-binding policy. Bindings can identify a Telegram group by channel, account, peer kind, and peer ID, so retaining both issues would split one scoped reset-policy API and precedence decision across generic and Telegram-specific trackers.

## What This Changes

Add a session reset override for individual Telegram groups so one-shot groups can expire context independently of other chats.

## Best Possible Solution

Track one additive, account-aware binding or conversation-scoped reset policy in https://github.com/openclaw/openclaw/issues/60659, with explicit precedence over channel/type/global defaults and coverage for Telegram groups and topics.

## Reproduction Assessment

Yes at source level: current types, strict schemas, and reset resolution show no per-group override, although this is a missing feature rather than a broken existing contract.

## Solution Assessment

No, a Telegram-only nested policy is not the best permanent shape; the open per-binding tracker can represent the same group-specific behavior with account-aware routing and avoids parallel precedence models.
