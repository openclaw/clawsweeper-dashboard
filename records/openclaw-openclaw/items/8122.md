## Summary

Keep open: current main still only supports global, chat-type, and channel-level session reset policy, while Telegram group/topic config and strict schemas still reject the requested nested group reset shape. The linked implementation at https://github.com/openclaw/openclaw/pull/8134 is closed unmerged, and the broader tracker at https://github.com/openclaw/openclaw/issues/60659 overlaps but does not replace this Telegram-specific request.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can auto-reset independently of global, type, or channel reset policy.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give high confidence that current main does not accept or apply the requested per-group reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and define precedence rather than adding a one-off shortcut.
