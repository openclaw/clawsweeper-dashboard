## Summary

Keep open. Current main still only supports global, type, and channel-level session reset policy; Telegram group/topic config rejects the requested nested reset shape, and the only paired implementation PR is closed unmerged while the broader open reset-policy request does not fully replace this Telegram group/topic use case.

## What This Changes

Support configuring session reset policy for individual Telegram groups or topics instead of only globally, by chat type, or by channel.

## Best Possible Solution

Define an additive account-aware Telegram group/topic reset override, or a generic conversation-scoped resolver if maintainers want this across channels, then update config types, strict schemas, docs/metadata, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: the proposed config is not accepted by the current types/schemas, and runtime reset selection has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is an account-aware Telegram group/topic override or a generic routed reset resolver, not a naive group-id scan.
