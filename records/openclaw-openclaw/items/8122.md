## Summary

Keep open. Current main still lacks per-Telegram-group or per-topic session reset configuration, and the related implementation pull request is closed unmerged while the broader per-agent/per-binding request does not fully own this Telegram-specific config shape.

## What This Changes

The issue requests `channels.telegram.groups.<id>.session.reset`-style configuration so selected Telegram groups can use a different reset policy than other chats.

## Best Possible Solution

Define an additive, account-aware Telegram group/topic reset override or a generic conversation-scoped resolver, with explicit precedence over channel/type/global policy and matching schema, docs, metadata, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: the proposed config shape is absent from current types/schemas, and runtime reset selection has no group/topic input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is an account-aware Telegram group/topic override or a generic conversation-scoped resolver, not a naive group-id scan.
