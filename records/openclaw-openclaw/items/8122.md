## Summary

Keep open. Current main still only supports session reset policy at global, chat-type, and channel scope; Telegram group/topic config, strict schemas, and runtime reset resolution do not accept or apply the requested per-group reset settings. The earlier implementation PR #8134 is closed unmerged, and #60659 is broader rather than a canonical replacement for this Telegram group request.

## What This Changes

Request per-Telegram-group session reset settings so selected groups can use short idle resets without changing other groups.

## Best Possible Solution

Add an account-aware conversation-scoped reset policy, or explicitly add Telegram group/topic reset overrides if maintainers choose a Telegram-only surface, with aligned types, strict schemas, docs/help, labels, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives high confidence that current main lacks the requested per-group reset configuration path.

## Solution Assessment

Unclear. The user need is concrete, but maintainers should decide whether the narrow maintainable solution is Telegram group/topic config or a generic conversation/binding reset override that also covers related workflows.
