## Summary

Keep open: current main still exposes session reset policy only at global, chat-type, and channel scopes, while Telegram group/topic config remains strict and has no `session.reset` field. The broader tracker at https://github.com/openclaw/openclaw/issues/60659 overlaps but does not fully replace this concrete Telegram group/topic request.

## What This Changes

Request per-Telegram-group session reset configuration so selected Telegram groups can use a shorter reset policy than other groups.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and define precedence rather than adding a group-id-only shortcut.
