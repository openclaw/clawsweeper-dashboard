## Summary

Keep open. Current origin/main still lacks per-Telegram-group or per-topic session reset configuration, the strict Telegram schemas reject the proposed nested shape, runtime reset selection only considers global/type/channel policy, and the direct implementation PR at https://github.com/openclaw/openclaw/pull/8134 is closed unmerged; https://github.com/openclaw/openclaw/issues/60659 is related but broader and not a clear canonical replacement for the Telegram group/topic request.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can use short idle resets without changing all groups or the whole Telegram channel.

## Best Possible Solution

Define an additive account-aware Telegram group/topic reset override, or explicitly fold this use case into a generic conversation/binding-scoped resolver, with documented precedence over channel/type/global policy plus schemas, generated metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection against origin/main gives a high-confidence verification path: the proposed config shape is absent and strict schemas/runtime resolver have no group/topic reset input.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is an account-aware Telegram group/topic override or a generic conversation-scoped resolver; a naive group-id-only lookup would be fragile in multi-account setups.
