## Summary

Keep open. Current main still lacks the requested per-Telegram-group or per-topic session reset configuration: the Telegram group/topic schemas have no `session` key, and reset resolution still receives only global, type, and channel-level policy inputs.

## What This Changes

Request per-Telegram-group session reset configuration so selected Telegram groups can use short idle resets without changing global, channel, or all-group policy.

## Best Possible Solution

Add a backwards-compatible, account-aware reset override with documented precedence over channel, type, and global policy, either in Telegram group/topic config or through a generic conversation-scoped resolver.

## Reproduction Assessment

Yes. Source inspection on current main verifies the proposed `channels.telegram.groups.<id>.session.reset` shape is absent from strict Telegram schemas, and `initSessionState` only passes channel/type/global reset inputs to the resolver.

## Solution Assessment

Unclear. The requested Telegram group shape is plausible, but the maintainable API may be a generic account-aware conversation-scoped reset contract if maintainers want the behavior across channels or bindings.
