## Summary

Current main still lacks a group-, topic-, agent-, or binding-scoped reset-policy input, and the linked implementation PR closed unmerged. The use case remains valid, while the overlapping broader issue has neither an implementation nor an explicit maintainer decision making it the sole canonical tracker.

## What This Changes

Requests per-Telegram-group session reset settings so one-shot groups can discard context independently of other groups.

## Best Possible Solution

Define one account-aware scoped reset-policy contract with documented precedence across conversation or binding, agent, channel, chat type, and global defaults, then let Telegram group/topic routing consume that contract without scanning unrelated accounts.

## Reproduction Assessment

Yes, at source level. Current main's strict config and inbound reset-policy path expose no group, topic, binding, or agent-specific reset setting, so the requested configuration cannot currently be expressed.

## Solution Assessment

Unclear. Per-group reset solves the reported use case, but a generic scoped policy coordinated with https://github.com/openclaw/openclaw/issues/60659 is likely safer than introducing an independent Telegram-only precedence path.
