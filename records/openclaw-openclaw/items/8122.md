## Summary

Keep open: current main still lacks per-Telegram-group or per-topic session reset configuration. Telegram group/topic config does not accept `session.reset`, runtime reset selection only considers channel/type/global policy, the linked implementation PR at https://github.com/openclaw/openclaw/pull/8134 is closed unmerged, and the broader scoped-reset issue at https://github.com/openclaw/openclaw/issues/60659 overlaps without replacing this concrete Telegram request.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can use short idle resets without changing all groups or the whole Telegram channel.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated config metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with https://github.com/openclaw/openclaw/issues/60659 rather than adding a group-id-only shortcut.
