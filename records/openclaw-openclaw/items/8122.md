## Summary

Keep open: current main still lacks per-Telegram-group or per-topic session reset configuration, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/8134 is closed unmerged. The broader scoped-reset tracker at https://github.com/openclaw/openclaw/issues/60659 overlaps, but it does not fully replace this concrete Telegram group/topic request.

## What This Changes

Request per-Telegram-group session reset configuration so short-lived TTS groups can reset independently from other Telegram group chats.

## Best Possible Solution

Define an additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated config metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable solution should be account-aware and define precedence against channel, type, and global reset policy.
