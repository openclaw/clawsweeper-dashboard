## Summary

Keep open: current main and the latest release still only support session reset policy globally, by chat type, or by channel, while Telegram group/topic config does not accept or apply the requested scoped reset policy. The closed implementation PR is not merged, and the broader scoped-reset issue overlaps but does not replace this Telegram group use case.

## What This Changes

Support per-Telegram-group session reset configuration so selected groups can use shorter idle resets without changing every group.

## Best Possible Solution

Define one additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with the broader scoped-reset tracker.
