## Summary

Keep open. Current main still has only global, chat-type, and channel-level session reset policy; Telegram group/topic config and runtime reset resolution do not accept or apply the requested per-group reset override, and the only exact implementation PR is closed unmerged.

## What This Changes

The issue requests `channels.telegram.groups.<groupId>.session.reset` so specific Telegram groups can use different idle/daily reset behavior from other groups.

## Best Possible Solution

Define an additive account-aware reset override after routing identifies the effective Telegram group/topic or generic conversation binding, then align types, strict schemas, generated help/labels, docs, and precedence tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives high confidence that current main lacks the requested per-group reset configuration path.

## Solution Assessment

Unclear until maintainers choose the API shape. The maintainable direction is either explicit Telegram group/topic reset config with account-aware lookup or a generic conversation-scoped resolver with documented precedence.
