## Summary

Keep open: current main and v2026.6.5 still only support session reset globally, by chat type, or by channel; Telegram group/topic config does not accept or apply the requested scoped reset policy. The direct implementation PR is closed unmerged, and the broader scoped-reset issue overlaps but does not replace this Telegram group/topic use case.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can use a short idle reset without changing all groups or all Telegram chats.

## Best Possible Solution

Define one additive, account-aware scoped reset policy with explicit precedence, then align Telegram config types, strict schemas, generated config metadata/help, public docs, and focused runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main does not accept or apply the requested per-group or per-topic reset configuration.

## Solution Assessment

Unclear until maintainers choose the public API shape. The proposed nested Telegram group config is plausible, but the maintainable fix should be account-aware and coordinated with the broader scoped-reset tracker.
