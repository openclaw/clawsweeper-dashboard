Keep this issue open. Current main still lacks per-Telegram-group or per-topic session reset configuration, and the related implementation PR is closed unmerged while the broader per-agent/binding request does not own this exact Telegram group/topic behavior.

## What This Changes

Request per-Telegram-group session reset configuration so selected groups can use short idle resets while other groups keep normal continuity.

## Best Possible Solution

Add an account-aware Telegram group/topic reset override or a generic conversation-scoped reset resolver, with strict schema/types/docs and precedence tests covering group/topic over channel/type/global.

## Solution Assessment
