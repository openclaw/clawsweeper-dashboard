## Summary

Keep this PR open: it targets a real Google/Gemini replay hygiene bug, but the patch misses the current replay-history shape that main actually produces and has a token-usage edge case already called out in review comments.

## What This Changes

The PR adds Google transport-side filtering for runtime-context custom messages and poisoned assistant replay turns, plus a regression test for Gemini replay payload conversion.

## Best Possible Solution

Keep the Google replay fix, but make it match the current replay pipeline by dropping repaired stop fallback poison turns and using the shared zero-usage token semantics with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current replay cleanup rewrites zero-usage empty stop turns to fallback text, and the PR's Google transport filter does not drop that repaired stop fallback shape before conversion.

## Solution Assessment

No. Transport-side filtering is a plausible narrow fix, but this implementation must handle current sanitized replay shapes and totalTokens, or move the provider-owned cleanup earlier in the replay pipeline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Drop repaired stop fallback turns before Gemini replay:** `extensions/google/transport-stream.ts:436-445`
  - body: The sanitizer only treats the fallback sentinel as poison when `stopReason === "error"`, but current replay cleanup rewrites zero-usage empty `stop` assistant turns into that same sentinel while keeping `stopReason: "stop"`. Those messages then reach the Google converter as a normal assistant text part, so the current-main poison shape still gets replayed to Gemini. Handle the repaired `stop` fallback shape, or run this cleanup before the generic rewrite, and cover that path in the test.
  - confidence: 0.91
- **[P2] Include totalTokens in zero-usage detection:** `extensions/google/transport-stream.ts:409-410`
  - body: `TransportUsage` in this area uses `totalTokens`, and the shared empty-turn helper also accepts `totalTokens`/`total_tokens`, but this helper only checks `usage.total`. A replayed empty stop turn with nonzero `totalTokens` and no `total` is classified as zero-usage, while the new regression does not exercise this branch. Reuse the shared semantics or include the existing token fields here.
  - confidence: 0.86
