## Summary

This PR should stay open because current main only covers part of the Google replay-sanitization problem, while the proposed patch still has two concrete correctness blockers and lacks real Gemini proof.

## What This Changes

The PR adds Google transport replay sanitization and a regression test to drop runtime-context custom messages plus poisoned assistant turns before building Gemini payloads.

## Best Possible Solution

Land a rebased Google replay hardening patch that matches shared assistant-turn and usage-field semantics, adds focused coverage for sentinel-stop and total-token cases, and includes real Gemini proof.

## Reproduction Assessment

Yes. A high-confidence source-level path can feed a fallback-text assistant turn with `stopReason: "stop"` and zero usage through `buildGoogleGenerativeAiParams`; this PR leaves that current replay shape in the Gemini payload.

## Solution Assessment

No. Provider-local filtering is a reasonable ownership boundary, but this implementation must first handle the rewritten `stop` fallback shape and normalize `total`, `totalTokens`, and `total_tokens` consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Drop rewritten stop fallback turns before Gemini replay:** `extensions/google/transport-stream.ts:436-445`
  - body: The sanitizer only treats the fallback sentinel as poison when `stopReason === "error"`, but current replay cleanup can rewrite a zero-usage empty `stop` assistant turn into that same fallback text while preserving `stopReason: "stop"`. That rewritten turn still becomes normal model text in the Gemini replay payload, so the current poison shape this PR is meant to block can still be sent.
  - confidence: 0.91
- **[P2] Honor totalTokens when detecting zero-usage turns:** `extensions/google/transport-stream.ts:409-410`
  - body: `MutableAssistantOutput.usage` and the shared empty-turn helper use `totalTokens`, but this helper only checks `usage.total`. A replay turn with nonzero `totalTokens` and no `total` can be treated as zero-usage and dropped incorrectly.
  - confidence: 0.87
