## Summary

Keep open: the PR targets a real Google replay bug, but the current patch still misses the repaired zero-usage `stop` fallback shape, misreads `totalTokens`, and lacks the required changelog entry.

## What This Changes

The PR adds Google transport-side replay sanitization for runtime-context custom messages and poisoned Gemini assistant turns, plus regression coverage for replay payload conversion.

## Best Possible Solution

Keep the Google replay sanitizer, but align it with current replay cleanup semantics, add focused coverage for repaired stop fallback and token-field variants, and add the active changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main rewrites zero-usage empty `stop` turns to fallback text, while the PR only filters fallback text when `stopReason` is `error`.

## Solution Assessment

No. Transport-side filtering is a narrow maintainable direction, but this implementation must handle repaired fallback `stop` turns and match the shared `total`/`totalTokens`/`total_tokens` zero-usage semantics before merge.
