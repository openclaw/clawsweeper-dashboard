## Summary

Keep this PR open: the direction is useful, but the current patch still misses a replay shape produced by current main, can misclassify `totalTokens` usage, and needs the required changelog entry before merge.

## What This Changes

The PR adds Google transport filtering for runtime-context custom messages and poisoned Gemini replay assistant turns, plus regression coverage for replay payload conversion.

## Best Possible Solution

Keep the Google replay sanitizer, but align it with the shared replay cleanup contract and token-usage semantics, then document the user-facing fix in the active changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can convert a zero-usage empty `stop` assistant turn into fallback text, and this PR would still replay that fallback to Gemini because it only drops fallback text on `stopReason: "error"`.

## Solution Assessment

No. Transport-side filtering is a narrow maintainable approach, but this implementation needs to drop the repaired stop fallback shape, match shared `totalTokens` semantics, and add the required changelog entry.
