## Summary

Keep open: this is a plausible focused Google/Gemini replay bug fix, but the current PR is not merge-ready because the sanitizer misses the rewritten zero-usage stop fallback shape and can drop nonzero-usage turns when usage is recorded as `totalTokens`. The related open regression-test PR overlaps but does not supersede the production fix path here.

## What This Changes

The PR adds Google transport replay sanitization and regression coverage to remove runtime-context custom messages and poisoned assistant turns before Gemini message conversion.

## Best Possible Solution

Land a rebased Google replay hardening patch that matches the shared assistant-turn and usage-field semantics, adds focused coverage for the sentinel stop and `totalTokens` cases, and includes real Gemini proof.

## Reproduction Assessment

Yes. A focused source-level reproduction can feed a fallback-text assistant turn with `stopReason: "stop"` and zero usage through `buildGoogleGenerativeAiParams`; the PR sanitizer does not drop that current replay shape.

## Solution Assessment

No. Provider-local filtering is a maintainable direction, but this implementation should first handle the rewritten stop fallback shape and normalize `total`/`totalTokens`/`total_tokens` consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Drop rewritten stop fallback turns before Gemini replay:** `extensions/google/transport-stream.ts:436-445`
  - body: The sanitizer only treats the fallback sentinel as poison when `stopReason === "error"`, but current replay cleanup can rewrite a zero-usage empty `stop` assistant turn into that same fallback text while preserving `stopReason: "stop"`. That rewritten turn still becomes normal model text in the Gemini replay payload, so the current poison shape this PR is meant to block can still be sent.
  - confidence: 0.91
- **[P2] Honor totalTokens when detecting zero-usage turns:** `extensions/google/transport-stream.ts:409-410`
  - body: `MutableAssistantOutput.usage` and the shared empty-turn helper use `totalTokens`, but this helper only checks `usage.total`. A replay turn with nonzero `totalTokens` and no `total` can be treated as zero-usage and dropped incorrectly.
  - confidence: 0.87
