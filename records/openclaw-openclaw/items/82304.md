## What This Changes

The issue reports that pressing Ctrl+C during upgrade doctor is treated as a non-aborting doctor failure, allowing the installer to continue and open a dead dashboard.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76386 or an equivalent narrow installer fix, then close this issue and https://github.com/openclaw/openclaw/issues/90011 as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows the installer lacks INT/TERM traps, flattens child signal exit status, ignores run_doctor failure, and can still open the dashboard after doctor cancellation.

## Solution Assessment

Yes. The best direction is a focused installer fix that preserves signal exit status, handles INT/TERM cleanup, and opens the dashboard only after successful doctor completion; the open linked PR is the active implementation path.
