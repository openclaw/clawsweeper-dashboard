## Summary

Keep open. Current main still has a source-level gap where native subagent spawn can accept a child after only in-memory registry insertion, because registry persistence errors are swallowed and the list/control path cannot reconstruct missing registry rows from child session records.

## What This Changes

The issue asks OpenClaw to make accepted native `sessions_spawn(runtime:"subagent")` runs durably registered, visible to `subagents list`, and diagnosable when a child session exists without a registry entry.

## Best Possible Solution

Make native subagent acceptance atomic with durable registry persistence or return an error with cleanup, then add a diagnostic/listing path for child sessions whose registry row is missing.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: make `saveSubagentRegistryToDisk` throw and the current path can still return accepted because persistence failures are swallowed; I did not execute a live gateway repro in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? The requested invariant is right; the narrow maintainable fix is to make registry persistence a checked part of native spawn acceptance, plus focused orphan diagnostics rather than a broad announce-delivery rewrite.
