## Summary

Current main now covers the reported registry durability failure: initial native subagent registration fails closed on durable-write errors, and run-mode `cleanup:"keep"` registry rows are no longer swept by the session TTL. The broader completion-delivery hardening remains tracked separately, but this issue’s accepted-without-visible-registry invariant is implemented.

## What This Changes

The issue asks OpenClaw to ensure native run-mode subagents with `cleanup:"keep"` are durably registered before `sessions_spawn` returns `accepted` and remain discoverable via `subagents list`.

## Best Possible Solution

Keep the current-main fixes for the registry invariant and leave broader completion retry/notification recovery under https://github.com/openclaw/openclaw/issues/44925.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence by forcing the initial registry save to fail or by sweeping a completed run-mode `cleanup:"keep"` entry past the old TTL; current tests now encode both cases.

## Solution Assessment

Yes. The merged main solution is the narrow maintainable fix: fail closed before reporting accepted when the initial registry row cannot be saved, and preserve kept run-mode rows instead of relying on list-time orphan reconstruction.
