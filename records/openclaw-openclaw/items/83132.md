## Summary

Keep open: current main still has source-backed paths where native subagent runs can be accepted without a durable registry row, and a companion sweep path can later remove run-mode `cleanup:"keep"` rows. An open fixing PR uses closing syntax for this issue, so this issue should stay open until that PR lands or maintainers choose a different fix path.

## What This Changes

The issue reports that native `sessions_spawn(runtime:"subagent")` can return `accepted` for kept run-mode subagents even though their run IDs are missing from, or later disappear from, `subagents/runs.json`.

## Best Possible Solution

Review and land the focused PRs that fail closed on initial registry persistence and preserve run-mode `cleanup:"keep"` entries past the session TTL, while leaving broader retry/orphan recovery policy to https://github.com/openclaw/openclaw/issues/44925.

## Reproduction Assessment

Yes. Source inspection shows current main writes native subagent runs in memory, then calls a persistence helper that swallows durable write errors; the run-mode keep TTL deletion path is also source-reproducible by aging `cleanupCompletedAt` beyond `SESSION_RUN_TTL_MS`.

## Solution Assessment

Yes. The narrow maintainable fix is to make the initial registry write strict before returning `accepted` and to guard the sweeper so run-mode `cleanup:"keep"` entries are not treated like session-mode TTL entries.
