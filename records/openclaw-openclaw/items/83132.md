## Summary

Keep open: current main still has a source-backed path where native subagent registration can succeed in memory while durable `runs.json` persistence is silently ignored, and `subagents list` is registry-backed rather than reconstructing orphaned child sessions.

## What This Changes

The issue reports that native `sessions_spawn(runtime: "subagent")` can return accepted for kept child runs that exist in the session store but are missing from `~/.openclaw/subagents/runs.json`.

## Best Possible Solution

Make native subagent registration fail closed: durably write or verify the registry row before returning accepted, keep best-effort persistence for later lifecycle updates only where appropriate, and surface diagnostics for child sessions/tasks that lack registry rows.

## Reproduction Assessment

Yes, source-level: force the initial subagent registry save path to fail after the child gateway run is accepted. Current main swallows that persistence failure, so the live symptom can be exercised with a focused test even though I did not run the full QQ/Windows workflow.

## Solution Assessment

Yes. The narrow maintainable fix is to enforce the initial registry durability invariant before `accepted`; delivery retry work in https://github.com/openclaw/openclaw/issues/44925 should not be the only recovery path for a missing run row.
