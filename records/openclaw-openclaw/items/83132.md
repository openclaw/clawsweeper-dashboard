The issue reports that native run-mode subagents can be accepted and create child sessions without a durable `subagents/runs.json` registry entry, making list and completion paths lose track of them.

## Best Possible Solution

Keep the current main fixes for initial registry durability and run-mode keep retention, and continue any broader completion-delivery retry work under https://github.com/openclaw/openclaw/issues/44925.

## Reproduction Assessment

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: fail closed before reporting `accepted`, preserve kept run-mode registry rows, and leave broader delivery-retry behavior to the existing canonical issue.
