## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate Python orchestration behind explicit exec policy:** `src/agents/pi-tools.ts:552`
  - body: This adds python_orchestrator whenever tools.pythonOrchestrator.enabled is not explicitly false, while the added tool runs model-supplied Python through a host python3 process. That creates a new local code-execution path by default instead of routing through the existing exec approval and policy model.
  - confidence: 0.92
- **[P1] Do not swallow uncaught exceptions:** `src/entry.ts:12-18`
  - body: The added uncaughtException handler only logs the error. Registering this handler suppresses Node's default fatal exit, so the gateway can continue after an uncaught exception in an undefined state; use the existing fatal-exit path or terminate after cleanup.
  - confidence: 0.86
- **[P2] Keep the UI browser test project enabled:** `ui/vitest.config.ts:20`
  - body: Removing the Playwright browser project means ui/src/**/*.browser.test.ts is no longer selected by the UI Vitest config. That silently drops browser coverage from the UI gate even though the dependency remains present.
  - confidence: 0.9
- **[P2] Preserve HTTP status on postJson failures:** `src/memory/post-json.ts:47`
  - body: The inner retry catches errors that intentionally carry status and then throws a fresh failed-after-3-attempts error without that status. Callers such as postJsonWithRetry use err.status to retry only 429 and 5xx responses, so this breaks the existing retry contract.
  - confidence: 0.82
- **[P2] Keep config entries passthrough before logging unknown keys:** `src/config/zod-schema.ts:177-182`
  - body: Replacing catchall(z.unknown()) with a transform-only object causes Zod to strip unknown keys before the transform runs. That means the warning loop cannot see those keys and provider-specific MCP or talk fields are lost.
  - confidence: 0.84
