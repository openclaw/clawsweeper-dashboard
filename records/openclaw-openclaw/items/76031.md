## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate Python orchestration behind explicit exec policy:** `src/agents/pi-tools.ts:552`
  - body: This adds python_orchestrator whenever tools.pythonOrchestrator.enabled is not explicitly false, while the new tool runs model-supplied Python through a host process. That creates a local code-execution path by default instead of routing through the established exec approval/tool-policy model.
  - confidence: 0.92
- **[P1] Do not swallow uncaught exceptions:** `src/entry.ts:12-18`
  - body: The new uncaughtException handler only logs and returns, suppressing Node's default fatal exit. After an uncaught exception the process can continue in an undefined state, so cleanup should either use uncaughtExceptionMonitor or terminate with a failing exit code.
  - confidence: 0.86
- **[P2] Keep the UI browser test project enabled:** `ui/vitest.config.ts:20`
  - body: Removing the Playwright browser project means ui/src/**/*.browser.test.ts is no longer selected by the UI Vitest config. That silently drops browser coverage while the dependency remains installed.
  - confidence: 0.9
- **[P2] Preserve HTTP status on postJson failures:** `src/memory/post-json.ts:47`
  - body: The retry wrapper catches errors that intentionally carry status and then throws a fresh failed-after-3-attempts error without that status. Callers use err.status to retry only 429/5xx responses, so this changes the existing retry contract for client errors.
  - confidence: 0.82
- **[P2] Keep config entries passthrough before logging unknown keys:** `src/config/zod-schema.ts:177-182`
  - body: Replacing catchall(z.unknown()) with a plain object transform causes Zod to strip unknown keys before the transform runs, so provider-specific MCP/talk fields are lost. Preserve passthrough semantics and add logging without changing the config contract.
  - confidence: 0.84
