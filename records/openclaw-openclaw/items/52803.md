## Summary

Keep open. Current main has useful session metadata and chat-side subagent controls, but the requested browser Control UI orchestration surface is still missing: no orchestration route, no hierarchical scalable tree, and selected-row bulk actions still only delete sessions. The open related cleanup request at https://github.com/openclaw/openclaw/issues/54797 and independent-agent topology request at https://github.com/openclaw/openclaw/issues/69364 are adjacent, not replacements.

## What This Changes

Request a scalable Control UI orchestration surface for agent-to-session-to-subagent hierarchy, active/history filters, per-row runtime details, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI orchestration request and split implementation into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence feature-gap check: current main has no orchestration route, and Sessions still renders flat paginated rows with delete-only bulk actions.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable implementation should be staged and design-reviewed rather than attempted as one broad repair PR.
