## Summary

Keep open. Current main has useful session/subagent metadata and chat-side kill/steer controls, but the requested browser Control UI orchestration surface is still missing: there is no orchestration route, Sessions remains a flat paginated table, and selected-row bulk actions only delete/archive sessions.

## What This Changes

The issue requests a scalable Control UI orchestration view with agent/session/subagent hierarchy, active-first filters, per-row runtime metadata, and bulk kill/steer/retry/cleanup actions.

## Best Possible Solution

Keep this as the umbrella Control UI orchestration request and split implementation into reviewed slices: first render existing lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions.
