## Summary

Keep open. Current main has useful session status metadata, task-ledger RPCs, chat-side kill/steer controls, and a small child indentation in the chat session selector, but it still lacks the requested browser Control UI orchestration surface with a scalable hierarchy and selected-row bulk kill/steer/retry/cleanup actions.

## What This Changes

Add a scalable Control UI orchestration view for agent-to-session-to-subagent hierarchy, active-first filtering, per-row runtime details, and bulk intervention actions.

## Best Possible Solution

Keep this as the umbrella Control UI orchestration feature and split it into reviewed slices: read-only hierarchical/virtualized rendering from existing session/task metadata, active/terminal filters, then carefully authorized bulk controls.

## Reproduction Assessment

No bug reproduction is applicable because this is a feature request. Source inspection gives a high-confidence feature-gap check: current main has no orchestration route, and Sessions remains a flat paginated table with delete-only selected-row bulk actions.

## Solution Assessment

Unclear. The requested direction is useful, but the maintainable solution should be staged and design-reviewed because it combines hierarchy rendering, scalability, status taxonomy, and privileged bulk run controls.
