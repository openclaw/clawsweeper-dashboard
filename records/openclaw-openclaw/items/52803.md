## Summary

Keep open. Current main has useful session lineage/status metadata and chat-side subagent kill/steer controls, but the requested browser Control UI orchestration surface is still absent: no dedicated orchestration route, no hierarchical agent/session/subagent rendering, and no bulk kill/steer/retry/cleanup workflow in the Sessions UI. The related #54797 cleanup request and #69364 independent-agent graph request remain adjacent rather than replacements.

## What This Changes

The issue requests a scalable Control UI orchestration view with agent-to-session-to-subagent hierarchy, active/failed/completed filters, per-row runtime metadata, virtualization, and bulk intervention actions.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 as the narrow completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.

## Reproduction Assessment

Not applicable as a feature request, but the absence is high-confidence from current-main source and docs inspection: the Control UI has no orchestration route, and Sessions renders a flat paginated table with delete-only selected-row actions.

## Solution Assessment

No for a one-shot automated repair. The requested direction is valid, but the maintainable path is a maintainer-reviewed product split across hierarchy rendering, virtualization, Gateway/task contract use, and privileged bulk-action semantics.
