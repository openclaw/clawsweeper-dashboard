## Summary

Keep open. Current main has session lineage/status metadata and chat-side subagent kill/steer controls, but the requested browser Control UI orchestration surface is still absent: there is no orchestration route, Sessions remains a flat paginated table, and selected-row bulk actions still only archive/delete sessions. Related #54797 and #69364 are adjacent cleanup and independent-agent graph requests, not replacements for this runtime subagent hierarchy and bulk-ops feature.

## What This Changes

The issue requests a scalable Control UI orchestration surface for multi-agent and subagent sessions with hierarchy, active-first filtering, bulk kill/steer/retry/cleanup operations, and per-row runtime status metadata.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 as the narrow completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.

## Reproduction Assessment

Not applicable as a feature request, but the absence is high-confidence from source and docs inspection: current Control UI navigation has no orchestration route, and Sessions renders a flat paginated table with delete-only selected-row actions.

## Solution Assessment

No for a one-shot automated repair. The requested direction is valid, but the maintainable path is a maintainer-reviewed product split across UI hierarchy/virtualization, gateway/task contract use, and privileged bulk-action semantics.
