## Summary

Keep open. Current main has useful session/task metadata and a richer Sessions table, but it still does not implement the requested Control UI orchestration board: there is no orchestration route, session rows are still rendered as a flat paginated table, and selected-row bulk actions only delete/archive sessions rather than kill, steer, retry, or scoped cleanup. Related open items cover narrower cleanup and a separate durable-agent graph model, so neither supersedes this runtime subagent operations request.

## What This Changes

Request a dedicated Control UI orchestration surface for agent/session/subagent hierarchy, active-first filtering, status metadata, scalable rendering, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI runtime-orchestration request, then split implementation into reviewed slices: first a read-only scalable hierarchy using existing session/task lineage and status data, then separately security-reviewed bulk intervention and cleanup actions.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence source verification path for the missing feature: current main has no orchestration route, keeps Sessions table-shaped, and only exposes delete/archive as a selected-row bulk action.

## Solution Assessment

Unclear as a single implementation. The requested direction is valid, but the maintainable path is to split a read-only hierarchy/status view from later privileged bulk kill/steer/retry/cleanup controls after maintainer product and security review.
