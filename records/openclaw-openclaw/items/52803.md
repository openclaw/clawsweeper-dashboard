## Summary

Keep open. Current main and the latest release have useful session metadata and chat-side kill/steer commands, but the requested Control UI orchestration surface is still missing: no orchestration route, no agent/session/subagent tree, no virtualization, and selected-row bulk actions only delete/archive sessions. Related open issues are adjacent rather than canonical replacements.

## What This Changes

Request a scalable Control UI orchestration surface for multi-agent and subagent sessions with hierarchy, active-first filtering, per-row run metadata, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella runtime subagent orchestration request and split implementation into reviewed slices: first a scalable hierarchy over existing session/task lineage and active/failed/completed state, then carefully authorized bulk intervention and cleanup actions.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and release inspection give a high-confidence feature-gap check: current OpenClaw exposes flat session rows, metadata, and chat commands, but not the requested orchestration board.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable solution needs a maintainer product choice on route shape, hierarchy model, scalability approach, and bulk-operation semantics before code starts.
