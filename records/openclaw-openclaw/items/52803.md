## Summary

Current main and the latest checked release still lack the requested Control UI orchestration board: Sessions remains a flat session table with delete-only bulk handling, while Workboard is adjacent task-board functionality rather than a live subagent ownership tree with run controls. The issue should stay open behind maintainer product direction because the requested hierarchy, filters, and bulk kill/steer/retry semantics are meaningful remaining UI work.

## What This Changes

The issue requests a Control UI mission board that visualizes multi-agent and subagent runs as a hierarchy with active-first filtering, runtime metadata, and bulk operational controls.

## Best Possible Solution

Keep this as the canonical umbrella request, then split implementation into reviewed slices: first a read-only active-first subagent hierarchy over existing session/task metadata, then explicitly designed bulk run controls.

## Reproduction Assessment

## Solution Assessment
