## Summary

Keep open. Current main and the latest release still have session lineage metadata but not the requested Control UI orchestration surface: there is no mission/orchestration route, Sessions is a flat paginated table, and selected-session bulk actions only delete/archive sessions.

## What This Changes

The issue requests a scalable Control UI orchestration surface showing agent -> main session -> subagent hierarchy with status filters, per-row metadata, virtualization, and bulk kill/steer/retry/cleanup actions.

## Best Possible Solution

Keep this as the umbrella Control UI subagent-orchestration request and split implementation into reviewed slices: read-only scalable hierarchy first, then explicit privileged bulk actions after safety and product semantics are chosen.

## Reproduction Assessment

No bug reproduction is applicable because this is a feature gap. Source inspection gives a high-confidence check: current main and v2026.6.8 expose session lineage/status metadata but lack a hierarchy board, status-mode filters, virtualization tree, or bulk run controls.

## Solution Assessment

Unclear until maintainers choose the product model. The narrowest maintainable path is a read-only hierarchy over existing metadata first, with privileged bulk actions added only after safety and scope semantics are settled.
