## Summary

Keep open. Current main and the latest release still lack the requested Control UI orchestration board: Sessions is a flat paginated table, there is no mission/orchestration route, and bulk selection only deletes sessions.

## What This Changes

The issue requests a scalable Control UI orchestration surface for agent-to-session-to-subagent hierarchy, active/failed/completed filters, per-row runtime metadata, and bulk kill/steer/retry/cleanup actions.

## Best Possible Solution

Keep this as the canonical umbrella for runtime subagent operations and split implementation into reviewed slices: read-only scalable hierarchy first, then explicit privileged bulk actions after safety/product design.

## Reproduction Assessment

No bug reproduction is applicable; this is a feature gap. Source inspection gives a high-confidence check because current main exposes lineage/status metadata but no hierarchy board, status-mode filters, virtualization tree, or bulk run controls.

## Solution Assessment

Unclear until maintainers choose the product model. The narrowest maintainable path is a read-only hierarchy over existing metadata first, with privileged bulk actions added only after safety and scope semantics are settled.
