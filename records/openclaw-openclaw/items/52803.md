## Summary

Keep open. Current main has session lineage metadata, task records, and chat-side kill/steer controls, but the requested browser Control UI orchestration surface is still missing: there is no orchestration route, Sessions remains a flat paginated table, and selected-row bulk actions still only delete/archive sessions. Related discussion keeps https://github.com/openclaw/openclaw/issues/69364 adjacent rather than superseding this issue, while https://github.com/openclaw/openclaw/issues/54797 tracks only the narrower completed-session cleanup slice.

## What This Changes

Add a scalable Control UI orchestration view for multi-agent/subagent hierarchy, active-first filtering, per-row run metadata, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first render existing lineage/status metadata in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions.
