## Summary

Keep open. Current main still has Workboard cards, flat paginated Sessions, and delete-only selected-session actions, but not the requested Control UI orchestration board with hierarchy, active/failed/completed filters, scalable tree rendering, and bulk kill/steer/retry/cleanup.

## What This Changes

Request a scalable Control UI orchestration view for agent, main-session, and subagent/task hierarchy with active-first filters and bulk operations.

## Best Possible Solution

Keep this as the umbrella runtime subagent operations request and split implementation into reviewed slices: first a scalable read-only hierarchy over existing session/task lineage, then explicitly designed bulk intervention and cleanup actions.

## Reproduction Assessment

No bug reproduction is applicable; this is a feature gap. Source inspection gives a high-confidence check because current main has lineage metadata but no hierarchy board, active/failed/completed tree filters, or bulk run controls.

## Solution Assessment

Unclear until maintainers choose the product model. The narrowest maintainable path is a read-only hierarchy first, with privileged bulk actions added only after scope and safety semantics are settled.
