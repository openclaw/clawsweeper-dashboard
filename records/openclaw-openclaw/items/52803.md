## Summary

Keep open. Current main has session/subagent lineage metadata and chat-side kill/steer controls, but the requested browser Control UI orchestration board is still absent: navigation has no orchestration route, Sessions renders a flat paginated table, and selected-row bulk actions only archive/delete sessions. The related #54797 and #69364 discussions are adjacent rather than replacements.

## What This Changes

The issue requests a scalable Control UI orchestration surface showing agent, main-session, and subagent hierarchy with active-first filters and bulk kill, steer, retry, and cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first surface existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 as the focused completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.

## Reproduction Assessment

Not applicable as a feature request. The absence is high-confidence from current-main source inspection: there is no orchestration route, the Sessions view is a flat paginated table, and browser bulk actions only delete/archive sessions.

## Solution Assessment
