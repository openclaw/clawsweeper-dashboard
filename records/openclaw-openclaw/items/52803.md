## Summary

Keep open. Current main has useful session status, runtime metadata, task ledger RPCs, and chat-side steer/abort controls, but it still lacks the requested browser Control UI orchestration surface: no orchestration route, no agent-to-session-to-subagent hierarchy, no virtualized tree, and no selected-row bulk kill/steer/retry/cleanup actions.

## What This Changes

Request a scalable Control UI orchestration view for multi-agent and subagent hierarchy, active-first filtering, per-row run metadata, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI runtime-orchestration request and split it into reviewed slices: first hierarchical read-only rendering from existing session/task metadata, then active/failed/completed filters, then authorized bulk intervention controls.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence feature-gap check: current main still has no orchestration route, and Sessions remains a flat paginated table with delete-only selected-row bulk actions.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable solution should be staged and design-reviewed because the full request combines hierarchy rendering, scalability, status taxonomy, and privileged bulk run controls.
