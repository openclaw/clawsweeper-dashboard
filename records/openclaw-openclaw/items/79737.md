## Summary

Keep open: the report describes broken existing QQ Bot multi-account routing, and current main does not prove that an account-bound C2C message cannot also be processed through the default account path. The related open PR https://github.com/openclaw/openclaw/pull/73567 addresses stale QQ Bot binding snapshots, but it is not shown to cover this duplicate default-agent dispatch.

## What This Changes

The issue asks for QQ Bot multi-account private messages that match an account binding to be delivered only to the bound agent, not also to the default agent.

## Best Possible Solution

Add a focused QQ Bot multi-account inbound regression for non-default account-bound C2C delivery, then fix the smallest QQ Bot-owned path needed to keep dispatch single-target while coordinating with the active-config work in https://github.com/openclaw/openclaw/pull/73567.
