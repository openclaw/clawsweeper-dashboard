## Summary

Keep open: the PR targets a real Mistral prompt-cache compat gap, but it adds the flag to the shared Mistral patch used by proxy/OpenRouter-style routes and the refreshed proof is still Vitest-only.

## What This Changes

Adds `supportsPromptCacheKey: true` to the shared Mistral transport compat patch and updates Mistral compat tests to expect it.

## Best Possible Solution

Split Mistral compat so direct/native Mistral routes advertise `supportsPromptCacheKey`, proxy/OpenRouter-style routes keep the existing request-shape fixes unless separately proven, explicit false compat is preserved, and pricing/tests/proof cover the enabled paths.

## Reproduction Assessment

Yes from source inspection: current main lacks `supportsPromptCacheKey` in the Mistral patch while the completions transport only injects `prompt_cache_key` when that flag is true. I did not establish a live Mistral cache-hit reproduction.

## Solution Assessment

No as written: adding the flag to the shared patch fixes the direct Mistral gap but also opts unproven proxy/OpenRouter routes into a new request field. A route-scoped compat split with real transport proof is the narrower maintainable path.
