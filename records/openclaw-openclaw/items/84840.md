## Summary

Keep open: source inspection confirms the native Mistral prompt-cache gap, but this PR opts broad custom and OpenRouter Mistral-like routes into `prompt_cache_key` without route-specific proof, and the supplied proof is still test-only.

## What This Changes

The PR adds `supportsPromptCacheKey: true` to the shared Mistral transport compat patch and extends Mistral compat tests for native, custom-route, and OpenRouter-style model resolution.

## Best Possible Solution

Split the compat so native/direct Mistral routes advertise `supportsPromptCacheKey`, while broader Mistral proxy and OpenRouter-style route hints keep the existing `max_tokens` and `supportsStore` fixes unless separately documented or proven.

## Reproduction Assessment

Yes from source inspection: current main lacks `supportsPromptCacheKey` in the Mistral compat patch, and the completions transport only injects `prompt_cache_key` when that compat flag is true. I did not run a live Mistral request.

## Solution Assessment

No as written: adding the flag to the shared Mistral patch fixes the direct Mistral gap but also opts unproven proxy/OpenRouter routes into a new request field; route-scoped compat is the narrower maintainable fix.
