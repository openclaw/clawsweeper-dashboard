## Summary

Keep open. Current main has isolated multi-agent routing, Control UI agent/session selectors, sub-agent tooling, and WhatsApp Broadcast Groups, but it still does not implement the requested chat-app product model with agents as contacts, per-agent availability/status, first-class chat windows, and user-created multi-agent rooms with invitations.

## What This Changes

Request a chat-style multi-agent UI where agents appear as contacts, agent conversations behave like chat windows, and users can create group chats for multi-agent collaboration.

## Best Possible Solution

Define a first-class Control UI and Gateway protocol model for agent contacts/status, per-agent histories, room membership and invitations, permissions, transcript attribution, and context handoff, reusing existing agents, sessions, Broadcast Groups, and sub-agent primitives where they fit.

## Reproduction Assessment

Not applicable: this is a feature request rather than a bug. Source and docs inspection confirms current main only has selector-based multi-agent switching and partial Broadcast Groups, not the requested chat-room model.

## Solution Assessment

Unclear: the chat paradigm is plausible and the production comments add useful constraints, but the maintainable solution needs maintainer-scoped UX, protocol, permissions, and cost-boundary design before a narrow implementation can be judged best.
