## Summary

## What This Changes

The issue reports intermittent silent freezes in the Telegram long-polling worker after a `fetch-timeout` on `getMe`, with no bounded worker exit, reconnect, or degraded channel signal.

## Best Possible Solution

Keep the current-main Telegram startup and polling-liveness implementation; ask for a new report only if a build containing `6ebb303ef026df80435a6ea28b688515c39d40ed` or a later release still shows the silent `getMe` freeze.
