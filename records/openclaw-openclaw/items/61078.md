## Summary

Keep open: current main does not implement the temp alias, and the PR branch still has sandbox path regressions plus missing real behavior proof before merge.

## What This Changes

The branch adds `@tmp/...` and `tmp/...` file-tool aliases that resolve into `run/tmp/` and extends workspace-root guard tests.

## Best Possible Solution

Revise the alias as a containment-checked `run/tmp` translation that preserves current sandbox bridge and workspace guard contracts, then add real file-tool behavior proof before merge.

## Reproduction Assessment

Yes for the introduced PR defects. A high-confidence static trace of `@tmp/reports/out.txt` with `root=/workspace/openclaw` and `containerWorkdir=/workspace` shows the PR-head mapper can double-map into `/workspace/openclaw/openclaw/run/tmp/...`.

## Solution Assessment

No. The shared path layer is a plausible home for the alias, but this implementation remaps execution arguments too broadly and does not confine temp aliases to `run/tmp`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:374-375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a host-absolute path under `root`, but `remapWorkspaceAliasPath()` then runs that value through `mapContainerPathToWorkspaceRoot()`. In `/workspace/<repo>` layouts, `@tmp/...` can become `<root>/<repo>/run/tmp/...` instead of `<root>/run/tmp/...`, so file tools hit the wrong location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandboxed read/write/edit tools paths that were already converted from container paths to host paths. The sandbox bridge resolves absolute `/workspace/...` input as container paths first, so common devcontainer layouts can execute against a nested wrong path.
  - confidence: 0.92
- **[P2] Keep tmp aliases confined to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The temp alias uses `path.resolve(root, "run", "tmp", ...)`, so inputs with `..` segments can normalize outside `run/tmp` while still staying inside the workspace guard. Validate the resolved path remains under `<root>/run/tmp` before accepting the alias.
  - confidence: 0.78
