## Summary

Keep open: the PR still adds a useful new temp-alias feature, but the current patch has blocking sandbox path correctness issues, a low-severity containment concern, and only test/lint proof rather than real behavior proof.

## What This Changes

The PR adds `@tmp/...` and `tmp/...` file-tool path alias rewriting into `run/tmp/`, plus workspace-root guard regression tests.

## Best Possible Solution

Repair or replace the PR with a smaller path-layer change that maps the approved temp alias once, enforces `run/tmp` containment, preserves sandbox bridge execution semantics, and adds mounted-sandbox regression coverage.

## Reproduction Assessment

Yes. Static reproduction is high-confidence by tracing a workspace root under the container workdir, `@tmp/...`, traversal input, and container-style file paths through the PR diff and current `SandboxFsBridge` resolver.

## Solution Assessment

No. The alias belongs in shared file-tool path handling, but this implementation rewrites execution args too broadly and does not keep temp-alias normalization confined to `run/tmp`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:371-374`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a rooted workspace path, but `remapWorkspaceAliasPath()` immediately feeds it to `mapContainerPathToWorkspaceRoot()`. When the workspace root itself sits under the container workdir prefix, `@tmp/...` is remapped a second time into a nested workspace path, so the guard and file operation target the wrong temp location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandboxed read/write/edit tools host-normalized paths. `SandboxFsBridge` resolves absolute input against container mounts before host mounts, so a host path sharing the container prefix can be interpreted again as a container path and hit a nested or wrong file.
  - confidence: 0.92
- **[P2] Keep tmp alias traversal inside run/tmp:** `src/agents/pi-tools.read.ts:360`
  - body: The temp alias uses `path.resolve(root, "run", "tmp", ...)`, so `@tmp/../../notes.md` can normalize elsewhere inside the workspace and still pass the workspace guard. If this is a temp-only contract, validate that the resolved path remains under `run/tmp`.
  - confidence: 0.78
