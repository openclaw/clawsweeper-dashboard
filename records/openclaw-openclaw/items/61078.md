## Summary

Current main now directs rejected outside-workspace file operations to the canonical `.openclaw/tmp/` workspace location, but it does not implement this PR’s alias syntax. The branch still proposes a distinct product contract, yet it cannot merge as submitted because it conflicts with the refactored path layer, silently redirects ordinary `tmp/...` paths, and lacks real runtime proof.

## What This Changes

The PR adds `@tmp/...` and bare `tmp/...` file-tool aliases that resolve to a workspace-owned temporary directory before workspace guard checks.

## Best Possible Solution

Keep `.openclaw/tmp/` as the canonical workspace-safe temporary location; if maintainers want shorthand, add only an explicit alias such as `@tmp/...` in the current path-normalization layer, preserve ordinary relative paths, and prove identical host and sandbox targets.

## Reproduction Assessment

Yes for the compatibility problem: source deterministically shows that bare `tmp/file` changes from `<workspace>/tmp/file` to `<workspace>/run/tmp/file`. The intended alias itself has no real current-main runtime demonstration.

## Solution Assessment

No. An optional explicit alias could be reasonable, but rewriting both explicit and ordinary relative paths in the pre-refactor shared wrapper is broader and less maintainable than extending the current canonical normalization boundary.
