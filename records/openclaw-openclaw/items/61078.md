## Summary

Keep this PR open for maintainer review: current main does not implement the temp alias, and the branch still has blocking sandbox path-resolution defects plus missing real behavior proof.

## What This Changes

The PR branch adds `@tmp/...` and `tmp/...` aliases for file-tool paths by resolving them into `run/tmp/` and extends workspace-root guard tests.

## Best Possible Solution

Implement a containment-checked `run/tmp` alias that preserves the sandbox bridge and workspace guard contracts, then add focused regressions and real file-tool proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source tracing shows double-remapping, broad execution-argument rewriting, and missing `run/tmp` containment. I did not run a live file-tool repro because this review is read-only.

## Solution Assessment

No. The shared path layer is a plausible home, but this patch needs a contained alias mapper and must avoid forwarding pre-remapped sandbox paths into file tools.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:374-375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a root-absolute path, but `remapWorkspaceAliasPath()` immediately passes it through container-workdir remapping. When the workspace root is nested under the container workdir, `@tmp/...` resolves into a nested wrong path instead of the intended `run/tmp` location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandbox-backed file tools paths that may already have been converted from container paths to host paths. The sandbox bridge resolves container-shaped absolute paths again, so common sandbox layouts can read or write a nested wrong target.
  - confidence: 0.92
- **[P2] Keep tmp aliases confined to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The temp alias uses `path.resolve(root, "run", "tmp", ...)`, so input containing `..` can normalize to a sibling inside the workspace rather than staying under `run/tmp`. Validate the resolved alias remains under the resolved temp root before accepting it.
  - confidence: 0.78
