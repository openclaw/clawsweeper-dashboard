## Summary

Keep open: current main still lacks the requested temp alias, but the PR is not ready to merge because the patch can double-remap paths in common workspace layouts, forwards pre-mapped paths into sandbox tools, and lacks real after-fix behavior proof.

## What This Changes

The branch adds `@tmp/...` and bare `tmp/...` file-tool aliases that resolve into `run/tmp/`, plus focused workspace-root guard tests.

## Best Possible Solution

Rebase and revise the alias as a single containment-checked `run/tmp` translation while preserving the current sandbox bridge and workspace guard contracts, then require real file-tool behavior proof before merge.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing PR-head inputs such as `@tmp/reports/out.txt` with `root=/workspace/openclaw` and `containerWorkdir=/workspace` through the alias mapper and current sandbox bridge path order.

## Solution Assessment

No. The shared file-tool path layer is a plausible place for the alias, but this implementation remaps execution arguments too broadly and does not keep temp traversal confined to `run/tmp`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:374-375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a host-absolute path under `root`, but `remapWorkspaceAliasPath()` immediately feeds it through `mapContainerPathToWorkspaceRoot()`. In `/workspace/<repo>` layouts, `@tmp/...` can become `<root>/<repo>/run/tmp/...` instead of the intended `<root>/run/tmp/...`.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandboxed read/write/edit tools paths that have already been converted from container paths to host paths. The sandbox bridge resolves absolute `/workspace/...` input as container paths first, so common devcontainer layouts can execute against the wrong nested path.
  - confidence: 0.92
- **[P2] Keep tmp aliases confined to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The temp alias uses `path.resolve(root, "run", "tmp", ...)`, so inputs with `..` segments can normalize outside `run/tmp` while still staying inside the workspace guard. If this alias is temp-only, validate that the resolved path remains under `<root>/run/tmp`.
  - confidence: 0.78
