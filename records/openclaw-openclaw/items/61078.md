Full review comments:

- **[P1] Skip container remap after resolving tmp aliases:** `src/agents/pi-tools.read.ts:374-375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a host-absolute path under `root`, but `remapWorkspaceAliasPath()` immediately runs that value through container-workdir mapping again. In layouts such as `root=/workspace/openclaw` with `containerWorkdir=/workspace`, `@tmp/reports/out.txt` resolves to a duplicated workspace path instead of the intended `run/tmp` target.
  - confidence: 0.94
- **[P1] Preserve original sandbox paths for file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandbox-backed read/write/edit tools host paths that may already have been converted from container paths. The sandbox bridge resolves container-shaped absolute paths first, so common `/workspace/...` layouts can read or write a nested wrong target.
  - confidence: 0.92
- **[P2] Confine tmp aliases to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The alias mapper uses `path.resolve(root, "run", "tmp", ...)` without verifying the normalized result stays inside `root/run/tmp`, so inputs like `@tmp/../../...` can leave the temp area while still passing the broader workspace guard.
  - confidence: 0.78
