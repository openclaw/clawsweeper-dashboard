## Summary

Keep open. The PR has a useful target, but current head still has sandbox-sensitive correctness blockers, is dirty against main, and lacks contributor-provided real behavior proof.

## What This Changes

The PR adds `@tmp/...` and `tmp/...` file-tool aliases that resolve into the workspace `run/tmp/` area and extends workspace-root guard tests.

## Best Possible Solution

Implement the alias as a containment-checked `run/tmp` translation that preserves the sandbox bridge and workspace guard contracts, then add focused regression tests plus real file-tool behavior proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source tracing shows the mapper can double-map temp aliases and the wrapper can forward pre-remapped paths into sandbox tools. I did not run a live file-tool repro in this read-only review.

## Solution Assessment

No. The shared path layer is a plausible home, but this implementation remaps execution arguments too broadly and does not enforce the temp alias subtree boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:374-375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a root-absolute path, but `remapWorkspaceAliasPath()` immediately sends it through container-workdir remapping. When the workspace root is nested under the container workdir, `@tmp/...` resolves into a nested wrong path instead of the intended `run/tmp` location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandbox-backed file tools paths that may already have been converted from container paths to host paths. The sandbox bridge resolves container-shaped absolute paths again, so common sandbox layouts can read or write a nested wrong target.
  - confidence: 0.92
- **[P2] Keep tmp aliases confined to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The temp alias uses `path.resolve(root, "run", "tmp", ...)`, so input containing `..` can normalize to a sibling inside the workspace rather than staying under `run/tmp`. Validate the resolved alias remains under the resolved temp root before accepting it.
  - confidence: 0.78
