## Summary

Keep open: current main does not implement the requested temp-path alias, and the PR still has blocking sandbox path bugs plus no after-fix real behavior proof.

## What This Changes

The PR adds `@tmp/...` and tolerant `tmp/...` file-tool aliases that rewrite path-like tool arguments into `run/tmp/` and extends workspace-root guard tests.

## Best Possible Solution

Implement a containment-checked temp alias in the current guard shape, preserve sandbox bridge inputs, add `..` and nested-workspace regressions, clear the branch-specific watch failure, and provide redacted real file-tool proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source tracing shows `@tmp/...` can double-map when `root` is nested under `/workspace`, `@tmp/../../...` can leave `run/tmp`, and remapped args can be reinterpreted by the sandbox bridge. I did not run a live file-tool repro because this review is read-only.

## Solution Assessment

No. The shared path layer may be the right home, but this patch needs a contained alias mapper and must avoid forwarding pre-remapped sandbox paths into file tools.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip container remap after resolving tmp aliases:** `src/agents/pi-tools.read.ts:374-375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a root-absolute path, but `remapWorkspaceAliasPath()` immediately passes it through container-workdir remapping. When `root` is nested under `containerWorkdir`, `@tmp/...` resolves to a duplicated workspace path instead of the intended `run/tmp` target.
  - confidence: 0.94
- **[P1] Preserve original sandbox paths for file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandbox-backed file tools host paths that may already have been converted from container paths. The sandbox bridge resolves container-shaped absolute paths first, so common `/workspace/...` layouts can read or write a nested wrong target.
  - confidence: 0.92
- **[P2] Confine tmp aliases to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The alias mapper uses `path.resolve(root, "run", "tmp", ...)`, so `@tmp/../../...` can normalize to another workspace path instead of staying under `run/tmp`. Validate the resolved alias remains inside the temp root before accepting it.
  - confidence: 0.78
