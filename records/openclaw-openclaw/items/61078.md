## Summary

Keep open. Current main does not implement the proposed `@tmp`/`tmp` file-tool alias, and the open PR still has blocking sandbox path-routing regressions plus a low-severity containment concern in the alias resolver.

## What This Changes

The PR maps `@tmp/...` and `tmp/...` file-tool paths to the workspace `run/tmp/` directory in `src/agents/pi-tools.read.ts` and adds workspace-root guard regression tests for those aliases.

## Best Possible Solution

Revise the PR so temp alias resolution is a single, explicit path translation that stays contained under `run/tmp`, preserves the sandbox bridge as the single executor-side path translator, and adds regression coverage for `/workspace/<repo>` layouts plus traversal attempts.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available by tracing `root=/workspace/openclaw`, `containerWorkdir=/workspace`, and inputs such as `@tmp/reports/out.txt` or `/workspace/docs/readme.md` through the PR head and current `SandboxFsBridge` path resolver.

## Solution Assessment

No. The requested alias belongs near the shared file-tool path layer, but this patch resolves aliases too broadly and forwards pre-remapped host paths into sandbox tools; the safer fix separates guard-only resolution from execution args and validates `run/tmp` containment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:374-378`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a host-absolute path under `root`, but `remapWorkspaceAliasPath()` immediately feeds that value into `mapContainerPathToWorkspaceRoot()`. With `root=/workspace/openclaw` and `containerWorkdir=/workspace`, `@tmp/reports/out.txt` becomes `/workspace/openclaw/openclaw/run/tmp/reports/out.txt`, so guard checks and file operations miss the intended `run/tmp` location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandboxed read/write/edit tools paths that have already been converted from container paths to host paths. `SandboxFsBridge` resolves absolute `/workspace/...` inputs as container paths first, so `/workspace/docs/readme.md` in a `/workspace/openclaw` root is executed under `<root>/openclaw/docs` instead of the requested workspace file.
  - confidence: 0.92
- **[P2] Keep tmp alias traversal inside run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The temp alias builds the target with `path.resolve(root, "run", "tmp", ...relative.split("/"))`, so `@tmp/../../notes.md` resolves to `<root>/notes.md` and still passes the workspace guard. If the alias is a temp-only contract, resolve the path and verify it remains inside `<root>/run/tmp`.
  - confidence: 0.78
