## Summary

Keep open: current main does not contain this alias, and the PR still has source-reproducible sandbox path defects plus missing real behavior proof, so no cleanup close reason applies.

## What This Changes

The PR adds `@tmp/...` and `tmp/...` file-tool path alias rewriting into `run/tmp/` and extends workspace-root guard tests.

## Best Possible Solution

Land a contained `run/tmp` alias that preserves sandbox bridge inputs, rejects aliases resolving outside `run/tmp`, adds nested-container and `..` regressions, clears the watch regression, and includes redacted real file-tool proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source tracing shows `@tmp/...` can double-map when `root` is nested under `/workspace`, `@tmp/../../...` can leave `run/tmp`, and remapped args can be reinterpreted by the sandbox bridge. I did not run a live file-tool repro because this review is read-only.

## Solution Assessment

No. The shared path layer may be the right place for the alias, but this patch needs a containment-checked mapper and must avoid forwarding pre-remapped sandbox paths into file tools.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip container remap after resolving tmp aliases:** `src/agents/pi-tools.read.ts:375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a host-absolute path under `root`, but this immediately sends that path through container-workdir remapping again. With `root=/workspace/openclaw` and `containerWorkdir=/workspace`, `@tmp/reports/out.txt` becomes `/workspace/openclaw/openclaw/run/tmp/reports/out.txt`, so the guard and tool target the wrong location.
  - confidence: 0.94
- **[P1] Preserve original sandbox paths for file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding `aliased` into `tool.execute` sends sandboxed read/write/edit tools host paths that may already have been converted from container-shaped inputs. The sandbox bridge resolves absolute `/workspace/...` inputs as container paths first, so common nested `/workspace` layouts can read or write a duplicated wrong target.
  - confidence: 0.92
- **[P2] Confine tmp aliases to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The alias mapper uses `path.resolve(root, "run", "tmp", ...)` without checking the resolved path remains under `root/run/tmp`. Inputs like `@tmp/../../config` can normalize elsewhere inside the workspace, violating the advertised temp-only alias boundary.
  - confidence: 0.78
