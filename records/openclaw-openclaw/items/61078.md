## Summary

Keep open: current main still lacks the requested temp alias, and this PR remains a useful implementation candidate, but the current diff is not mergeable because its path rewriting can double-map sandbox paths and weaken the new temp-only containment contract.

## What This Changes

The PR adds `@tmp/...` and `tmp/...` file-tool alias rewriting into workspace `run/tmp/` plus workspace-root guard regression tests.

## Best Possible Solution

Revise the PR so temp alias resolution is a single explicit translation under `run/tmp`, validates containment after normalization, and preserves the sandbox bridge as the executor-side path translator.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing `root=/workspace/openclaw`, `containerWorkdir=/workspace`, and inputs like `@tmp/reports/out.txt` or `/workspace/docs/readme.md` through the PR diff and current `SandboxFsBridge` resolver.

## Solution Assessment

No. The alias belongs near the shared file-tool path layer, but this patch rewrites execution args too broadly and does not keep `@tmp` traversal confined to `run/tmp`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:371-374`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a host-absolute path under `root`, but `remapWorkspaceAliasPath()` immediately feeds that value into `mapContainerPathToWorkspaceRoot()`. In `/workspace/<repo>` layouts this double-remaps `@tmp/...` into `<root>/<repo>/run/tmp/...`, so tools miss the intended temp location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding rewritten args into `tool.execute` sends sandboxed read/write/edit tools paths that have already been converted from container paths to host paths. The sandbox bridge resolves absolute `/workspace/...` input as container paths first, so common devcontainer layouts can execute against the wrong nested path.
  - confidence: 0.92
- **[P2] Keep tmp alias traversal inside run/tmp:** `src/agents/pi-tools.read.ts:360`
  - body: The temp alias uses `path.resolve(root, "run", "tmp", ...)`, so `@tmp/../../notes.md` can resolve elsewhere inside the workspace and still pass the workspace guard. If this alias is a temp-only contract, validate that the resolved path remains under `<root>/run/tmp`.
  - confidence: 0.78
