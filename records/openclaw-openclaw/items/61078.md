## Summary

Keep open: the PR is a focused feature, but the current head is not merge-ready because the alias path can double-map in common sandbox layouts, it forwards pre-remapped paths into file tools, the temp alias is not confined to run/tmp, and real behavior proof is still missing.

## What This Changes

The branch adds @tmp/... and tmp/... aliases for file tools that resolve to the workspace run/tmp area and extends workspace-root guard tests.

## Best Possible Solution

Revise the alias as a containment-checked run/tmp translation that preserves the current sandbox bridge and workspace guard contracts, then add real file-tool behavior proof before merge.

## Reproduction Assessment

Yes for the PR-introduced defects: a source trace of @tmp/reports/out.txt with root=/workspace/openclaw and containerWorkdir=/workspace shows the mapper can double-map into /workspace/openclaw/openclaw/run/tmp. The missing part is real after-fix behavior proof from the contributor's setup.

## Solution Assessment

No. The shared path layer is a plausible home for the alias, but this implementation remaps execution arguments too broadly and does not confine the temp alias to run/tmp.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:374-375`
  - body: mapWorkspaceTempAliasToRoot() already returns a host-absolute path under root, but remapWorkspaceAliasPath() then sends that value through container-workdir remapping. In /workspace/<repo> layouts, @tmp/... becomes <root>/<repo>/run/tmp/... instead of <root>/run/tmp/..., so file tools hit the wrong location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: Forwarding aliased into tool.execute sends sandboxed read/write/edit tools paths that were already converted from container paths to host paths. The sandbox bridge resolves absolute /workspace/... input as container paths first, so common devcontainer layouts can remap again and operate on a nested wrong path.
  - confidence: 0.92
- **[P2] Keep tmp aliases confined to run/tmp:** `src/agents/pi-tools.read.ts:363`
  - body: The temp alias uses path.resolve(root, "run", "tmp", ...), so inputs containing .. can normalize outside run/tmp while still passing the workspace guard. Validate that the resolved alias remains under <root>/run/tmp before accepting it.
  - confidence: 0.78
