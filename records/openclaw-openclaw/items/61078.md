## Summary

Keep open. Current main still does not implement the requested temp alias, but this PR is not merge-ready because the patch changes a sandbox-sensitive path-routing layer and introduces concrete wrong-path behavior in common `/workspace/*` layouts.

## What This Changes

The PR changes `src/agents/pi-tools.read.ts` and workspace-root guard tests to map `@tmp/...` and `tmp/...` file-tool paths to `<workspace>/run/tmp/...` before guard checks and to forward the resolved path to wrapped tools.

## Best Possible Solution

Keep the PR open and revise the implementation so temp alias resolution is separated from container-path remapping, sandboxed read/write/edit receive paths in the form the sandbox bridge expects, `@tmp/...` remains contained under `run/tmp/`, and any legacy path aliases are either removed from the runtime rewrite or explicitly approved with matching validation tests.

## Reproduction Assessment

Yes. The wrong-path regression can be reproduced by tracing `root=/workspace/openclaw` with `containerWorkdir=/workspace`: the PR maps `@tmp/reports/out.txt` to a host path under `root`, then maps it again through the container-workdir logic or sandbox bridge.

## Solution Assessment

No. The requested alias belongs in the shared file-tool path layer, but this patch applies the remap too broadly and forwards pre-remapped host paths into sandbox tools; the safer design is to resolve only for guard checks where needed and preserve the sandbox bridge's single source of path translation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Skip container remap after tmp alias resolution:** `src/agents/pi-tools.read.ts:371-375`
  - body: `mapWorkspaceTempAliasToRoot()` already returns a host-absolute path under `root`, but `remapWorkspaceAliasPath()` immediately feeds that value into `mapContainerPathToWorkspaceRoot()`. With `root=/workspace/openclaw` and `containerWorkdir=/workspace`, `@tmp/reports/out.txt` becomes `/workspace/openclaw/openclaw/run/tmp/reports/out.txt`, so guard checks and file operations miss the intended `run/tmp` location.
  - confidence: 0.94
- **[P1] Preserve sandbox paths when invoking file tools:** `src/agents/pi-tools.read.ts:635`
  - body: `aliased` is passed to the wrapped sandbox read/write/edit tools after container paths have already been converted to host paths. `SandboxFsBridge` resolves absolute `/workspace/...` inputs as container paths first, so `/workspace/docs/readme.md` in a `/workspace/openclaw` root is executed under `<root>/openclaw/docs` instead of the requested workspace file.
  - confidence: 0.92
- **[P2] Keep tmp alias traversal inside run/tmp:** `src/agents/pi-tools.read.ts:360`
  - body: The temp alias builds the target with `path.resolve(root, "run", "tmp", ...relative.split("/"))`, so `@tmp/../../notes.md` resolves to `<root>/notes.md` and still passes the workspace guard. If the alias is meant to be temp-only, resolve the path and verify it remains inside `<root>/run/tmp`.
  - confidence: 0.78

## Security Review
