## Summary

Current main and the v2026.6.8 release already render submitted Control UI queued sends in the chat thread before the Gateway ACK or assistant stream, so the reported old-version behavior is fixed.

## What This Changes

The issue reports that Control UI does not show the just-submitted user message until the assistant response starts streaming.

## Best Possible Solution

Keep the shipped queued-send thread renderer as the canonical pending-send display path and close this report as fixed in v2026.6.8.

## Reproduction Assessment

No current-main reproduction remains: current source and the shipped browser E2E show the submitted prompt in the chat thread before the deferred chat.send ACK resolves. The reported version, 2026.4.21-5, predates the v2026.6.8 release that contains this behavior.

## Solution Assessment

Yes. Rendering queued sends in the chat-thread builder before ACK is the narrow current owner-boundary fix for the visible issue, and it avoids adding a second optimistic transcript state.
