## Summary

This is a valid, still-open Control UI bug: current `main` still waits for `chat.send` to resolve before appending the visible user turn. Keep it open because https://github.com/openclaw/openclaw/pull/94497 is an open closing PR for the same bug, so the issue should close only after a fix merges.

## What This Changes

The issue requests that Control UI render a submitted user message immediately after Send instead of waiting for the assistant response or `chat.send` ACK.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/94497 or replace it with a revised Control UI fix that appends or marks the submitted user turn before the ACK while preserving queue retry, duplicate suppression, and session-cache rollback behavior.
