## What This Changes

The PR adds Signal allowlist normalization in `extensions/signal/src/shared.ts` plus focused tests so `uuid:` entries are preserved while phone numbers still normalize to E.164.

## Best Possible Solution

Land this PR or an equivalent Signal-owned fix after UUID normalization is made canonical for command auth and DM policy, with focused regression coverage and redacted real Signal behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main documents `uuid:<id>` Signal allowlists, command auth delegates normalization to Signal `formatAllowFrom`, and the current Signal adapter still runs those entries through phone-only E.164 normalization.

## Solution Assessment

No, not yet. The Signal-local helper is the right seam, but it should canonicalize or reject compact UUIDs consistently and the external PR still needs real after-fix Signal proof.
