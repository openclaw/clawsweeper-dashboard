## Summary

Close as not reproducible on current main: the importer still targets the normalized tweakcn registry URL, and the exact reported endpoint now returns valid JSON with wildcard CORS for the local Control UI origin.

## What This Changes

The issue reports that Control UI Appearance theme import fails for tweakcn URLs, registry URLs, and raw theme IDs because the browser fetch is blocked by CORS.

## Best Possible Solution

Keep the current importer as-is unless the CORS failure returns; if it does, add a narrow gateway-owned tweakcn theme fetch helper rather than a general-purpose proxy.

## Reproduction Assessment

No. A reasonable current-main reproduction check does not reproduce the reported CORS block: the exact normalized registry URL returns 200 JSON and `access-control-allow-origin: *` for the local Control UI origin.

## Solution Assessment

Not applicable for a code change right now. The existing direct-browser importer is the narrowest implementation while tweakcn serves CORS-enabled JSON; a constrained gateway helper would be the fallback only if that external contract breaks again.
