## Summary

Keep open: this PR addresses real skill install failures, but it is not merge-ready because the new cask metadata is not parsed, the Matrix/Mattermost SDK hunk is stale against current main, and contributor after-fix real behavior proof is absent.

## What This Changes

The PR adds brew-cask metadata support for skill installs, switches the bundled summarize installer to `@steipete/summarize`, and classifies Matrix/Mattermost SDK subpaths as public plugin-owned compatibility surfaces.

## Best Possible Solution

Revise this PR so `cask` is parsed and propagated end to end, drop the stale SDK-subpath reclassification already handled on main, keep hook-facing metadata types aligned, add focused regression coverage, and provide redacted real install proof.

## Reproduction Assessment

Yes. Source inspection plus the linked issue logs show current main still uses plain `brew install` for CodexBar and points summarize at the removed Homebrew formula; the PR's boolean `cask` value would also be dropped by the current parser.

## Solution Assessment

No. The install-source direction is narrow and supported by upstream docs, but the patch is incomplete until parser normalization, public hook metadata, focused tests, stale SDK classification, and real behavior proof are addressed.

  - confidence: 0.9

Add `cask?: boolean` there too, or intentionally keep the field out of the hook payload, so plugin policy hooks have an accurate contract.
  - confidence: 0.82
