## Summary

Keep this PR open: it targets a real linked onboarding bug, but the current patch is incomplete because the new `cask` flag is added after the SKILL.md parser boundary that currently drops boolean `cask` metadata.

## What This Changes

The PR adds brew-cask metadata support for CodexBar and switches the bundled summarize skill installer from Homebrew to `@steipete/summarize` via the node installer.

## Best Possible Solution

Land a revised PR that treats `cask?: boolean` as normalized install metadata from SKILL.md through command generation, status labels, scanner/hook contracts, tests, and release notes.

## Reproduction Assessment

Yes. The linked bug reproduces through `openclaw onboard` on macOS, and the source-level path is clear: the PR's `"cask": true` value is not preserved by `parseInstallSpec()`, so the command builder still receives a plain brew spec.

## Solution Assessment

No. The direction is maintainable, but the patch is incomplete; the safer repair is to parse and propagate `cask` at the frontmatter boundary and cover parser, command/status, hook payload, and changelog behavior.
