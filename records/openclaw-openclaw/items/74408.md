## Summary

Keep this PR open. It targets a real onboarding install failure from #74380, and the Homebrew/npm direction is plausible, but the patch is incomplete because the new boolean `cask` metadata is not parsed from `SKILL.md` before command generation, so the CodexBar path would still install as a formula.

## What This Changes

The PR adds brew-cask metadata support to skill installer types, command construction, and status labels, marks the bundled model-usage CodexBar install as a cask, and switches the bundled summarize install from Homebrew to the `@steipete/summarize` npm package.

## Best Possible Solution

Revise the PR so brew casks are parsed as first-class boolean install metadata from SKILL.md through normalized install metadata, before_install hook contracts, command generation, status display, focused tests, and changelog, while keeping summarize on the validated npm package source.

## Reproduction Assessment

Yes. The linked bug has concrete `openclaw onboard` failure output, and source inspection shows current main still emits `brew install <formula>` while the PR's new `cask: true` value is dropped by the parser before it can affect that command.

## Solution Assessment

No. The proposed direction is narrow and maintainable, but the current patch is incomplete; the safer implementation is to normalize `cask?: boolean` at the frontmatter boundary and cover the parser, installer, status, hook contract, and changelog follow-through together.
