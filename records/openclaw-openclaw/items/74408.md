## Summary

Keep this PR open. Current main still has the reported onboarding installer failure, but this PR is not mergeable as-is because the new `cask` flag is added after the frontmatter parser boundary, the public before-install hook contract is not kept in sync, the user-facing fix lacks changelog/test coverage, and the current Homebrew core `summarize` formula should be considered before switching the bundled skill to a node-only installer.

## What This Changes

The PR adds a `cask` skill install flag to installer/status/security-scan metadata, marks model-usage CodexBar as a cask, and changes the summarize skill installer from a tap formula to `@steipete/summarize` through the node installer.

## Best Possible Solution

Revise this PR so `cask?: boolean` is parsed and propagated from SKILL.md through command generation, status labels, scanner metadata, and public hook types; add focused tests and a changelog entry; and prefer the maintained Homebrew core `summarize` formula with a node fallback unless maintainers explicitly choose a node-only installer.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: #74380 gives concrete macOS `openclaw onboard` failure logs, current main still emits formula-only brew installs, and this PR's `cask: true` value would still be dropped before command generation.

## Solution Assessment

No. The PR's direction is close, but it is incomplete at the parser and public hook boundaries, and the current dependency evidence suggests `summarize` should use the maintained Homebrew core formula or a brew-plus-node fallback rather than a node-only replacement.
