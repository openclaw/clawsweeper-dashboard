## Summary

Keep this PR open. The linked onboarding bug is still valid on current main, and the PR's main cask fix is incomplete because the new boolean metadata is added after the SKILL.md parsing boundary that currently drops it.

## What This Changes

The PR adds a `cask` skill install flag, uses it in Homebrew install/status/scanner metadata, marks model-usage CodexBar as a Homebrew cask, and switches the summarize skill installer from Homebrew to the `@steipete/summarize` node package.

## Best Possible Solution

Revise this PR so `cask?: boolean` is parsed from SKILL.md metadata and carried through install command generation, status labels, scanner metadata, and public before-install hook types, with focused tests and an active changelog entry for the onboarding fix.

## Reproduction Assessment

Yes. The high-confidence source-level path is SKILL.md frontmatter -> `resolveOpenClawMetadata()` -> `installSkill()` -> `buildInstallCommand()`, where a boolean `cask` value is currently dropped before command generation.

## Solution Assessment

No. The PR is pointed in the right direction, but it is not yet the best maintainable fix because it uses `spec.cask` without first normalizing that field at the frontmatter boundary or covering the behavior with regression tests.
