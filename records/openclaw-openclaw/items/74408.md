## Summary

Keep this PR open: it addresses a real `openclaw onboard` skill dependency failure, but the current patch does not parse the new boolean `cask` field from SKILL.md metadata, so the CodexBar install path would still run as a Homebrew formula install. The remaining work is narrow and source-local.

## What This Changes

The PR adds brew-cask metadata support to skill installer/status/types, marks model-usage CodexBar as a Homebrew cask install, and switches the summarize skill installer from a steipete tap formula to `@steipete/summarize` through the node installer.

## Best Possible Solution

Land a revised version that treats `cask?: boolean` as first-class parsed skill install metadata from SKILL.md through command generation, status labels, install-scan and before-install hook contracts, focused tests, and changelog, while keeping the summarize installer on the maintainer-approved current package source.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main parses the CodexBar installer as a plain brew formula and builds `brew install steipete/tap/codexbar`; the PR's added boolean `cask` value is still dropped before command generation.

## Solution Assessment

No. The direction is maintainable, but the patch is incomplete because the new field is used after a parser boundary that does not preserve it; the safer fix is to normalize `cask?: boolean` at the frontmatter boundary and cover the command/status/hook behavior with focused tests.
