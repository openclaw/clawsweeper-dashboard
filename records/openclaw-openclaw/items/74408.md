## Summary

Keep this PR open: the CodexBar fix is incomplete because the new boolean `cask` metadata is still dropped at the SKILL.md parser boundary, and the summarize installer switch needs revision for OpenClaw's supported Node 22.14+ runtime path.

## What This Changes

This PR adds a `cask` skill install flag, marks CodexBar as a brew cask, and changes the bundled summarize installer from `steipete/tap/summarize` to `@steipete/summarize` via the node installer.

## Best Possible Solution

Revise the PR so `cask` is parsed and propagated end to end, choose a summarize install path compatible with OpenClaw's supported runtimes, update public hook metadata if exposed, add focused regression tests, and include the required changelog entry.

## Reproduction Assessment

Yes. The linked bug reproduces through `openclaw onboard` on macOS, and the PR's remaining bug is source-level reproducible because `parseInstallSpec()` drops boolean `cask: true` before command generation.

## Solution Assessment

No. Adding cask support is the right shape, but this patch is incomplete; the safer solution is a parser-to-command contract change plus tests, and summarize should use Homebrew/core `summarize` or an explicit Node 24 guard/fallback instead of an unguarded npm-only switch.
