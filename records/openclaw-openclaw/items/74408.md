## Summary

Keep open: this PR targets a real bundled skill installation bug, but the proposed patch is still incomplete because the new cask flag is not parsed, the SDK subpath hunk is stale against current main, hook-facing metadata is not aligned, and contributor real behavior proof is still missing.

## What This Changes

This PR adds cask metadata support to skill installs, switches the bundled summarize installer from the removed `steipete/tap/summarize` Homebrew formula to `@steipete/summarize`, and adds Matrix/Mattermost SDK subpath docs/classification.

## Best Possible Solution

Revise this PR so cask metadata is parsed and propagated end to end, remove the stale SDK reclassification, align hook metadata, add focused regression tests, confirm the summarize installer source, and provide redacted real install proof.

## Reproduction Assessment

Yes. Source inspection plus the linked issue logs give a high-confidence path: current main routes CodexBar through a formula-style brew install and points summarize at the removed tap formula; the PR’s new cask boolean is also dropped by the current parser.

## Solution Assessment

No. The cask/source direction is narrow, but this patch is not the best current form until parser normalization, stale SDK cleanup, hook metadata alignment, tests, source choice validation, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

- **[P2] Parse the cask flag before using it:** `skills/model-usage/SKILL.md:17`
  - body: The manifest adds `"cask": true`, but `parseInstallSpec()` only treats `raw.cask` as an alternate string formula and never copies a boolean onto `spec`.   - confidence: 0.97

  - body: Current main already classifies `matrix` and `mattermost` as supported bundled facades and documents both subpaths. Adding the same entries to `publicPluginOwnedSdkEntrypoints` creates an overlapping classification that the package-contract guardrail expects to be empty, so this hunk should be removed during rebase.
  - confidence: 0.92
- **[P3] Expose cask consistently in hook metadata:** `src/plugins/install-security-scan.ts:29`
  - body: The PR adds `cask` to the install metadata passed into the before-install scan path, but the public `PluginHookBeforeInstallSkillInstallSpec` type still omits it. Add `cask?: boolean` there too, or intentionally keep the field out of the hook payload, so plugin policy hooks have an accurate contract.
  - confidence: 0.83
