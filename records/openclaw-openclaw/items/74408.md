## Summary

Keep this PR open. It targets a real `openclaw onboard` skill-install bug, but the current patch still drops the new boolean `cask` metadata before command generation, so the CodexBar install path remains broken.

## What This Changes

This PR adds brew-cask skill installer metadata, marks model-usage CodexBar as a cask, and switches the bundled summarize installer from Homebrew to `@steipete/summarize` via the node installer.

## Best Possible Solution

Revise this PR so `cask?: boolean` is parsed and normalized from SKILL.md through install command generation, status display, scan/hook metadata, tests, and release notes, then land it as the fix for #74380.

## Reproduction Assessment

Yes. The linked report reproduces through `openclaw onboard` on macOS, and the current source path is clear: `parseInstallSpec()` drops boolean `cask`, so the PR's CodexBar manifest change still cannot affect the brew command.

## Solution Assessment

No. The direction is the narrow maintainable fix, but this patch is incomplete until the parser and plugin-facing contract are kept in sync and regression coverage proves the metadata reaches the installer.
