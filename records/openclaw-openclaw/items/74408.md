## Summary

Keep open: current main still has the linked skill install bug, but this PR remains incorrect because `cask: true` is not parsed before command generation and the latest SDK subpath additions double-classify Matrix/Mattermost in the package-contract metadata.

## What This Changes

The PR adds a `cask` flag to skill install metadata and Homebrew command/status handling, switches bundled `summarize` from Homebrew to `@steipete/summarize`, and documents/classifies Matrix and Mattermost SDK subpaths as public plugin-owned surfaces.

## Best Possible Solution

Revise this PR so cask metadata is parsed and normalized end to end, SDK subpath classifications stay disjoint, hook-facing metadata and tests are aligned, and the user-facing onboarding fix is recorded in the changelog.

## Reproduction Assessment

Yes. Source inspection shows current main still has the reported install paths, and the PR's `cask: true` value would be dropped by `parseInstallSpec()` before `buildInstallCommand()` checks it; the Matrix/Mattermost classification overlap is also determinable from the guardrail test logic.

## Solution Assessment

No. The PR is pointed at the right bug, but the current diff is incomplete and now adds an unrelated SDK classification change that conflicts with existing contract metadata.
