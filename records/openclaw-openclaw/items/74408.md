## Summary

Keep this PR open because the main fix is still incomplete: the proposed `cask` manifest field is read later by the installer, but current frontmatter parsing would drop the boolean before it reaches that code path.

## What This Changes

This PR adds brew-cask metadata support for skill installs, switches the bundled summarize installer to `@steipete/summarize`, and classifies Matrix/Mattermost plugin SDK subpaths as public plugin-owned compatibility surfaces.

## Best Possible Solution

Revise this PR so `cask: true` is parsed and normalized end to end, keep hook-facing install metadata in sync, add focused regression coverage and a changelog entry, then land it as the fix for #74380.

## Reproduction Assessment

Yes, by source inspection and the linked report: current main keeps `model-usage` on a plain brew formula install and `summarize` on the removed tap formula, while the parser would drop the PR's new boolean `cask` value.

## Solution Assessment
