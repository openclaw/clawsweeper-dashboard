## Summary

Keep this PR open. The linked macOS behavior is still real on current main, but the proposed patch is not merge-ready because it makes `screen.record` a macOS platform default while the wizard deny-list and security audit still classify that command as dangerous, so wizard-created gateway configs would continue to remove it and the security posture would be inconsistent.

## What This Changes

This PR adds `screen.record` to the macOS node command defaults with a gateway regression test, and also changes OpenAI WebSocket warm-up code/tests to omit an empty `tools` field.

## Best Possible Solution

Land a focused gateway/macOS policy change only after deciding that macOS `screen.record` should be default-allowed: update the platform allowlist, dangerous-command classification, setup wizard defaults, security audit behavior, docs, and regression tests consistently, including a wizard-generated config case.

## Reproduction Assessment

Yes. Source and tests on current main show that `resolveNodeCommandAllowlist({}, { platform: "macOS 26.4.0", deviceFamily: "Mac" })` still excludes `screen.record`, and wizard-created configs would also deny it through `DEFAULT_DANGEROUS_NODE_COMMANDS`.

## Solution Assessment

No. Adding `screen.record` to the macOS defaults alone is not the narrowest maintainable fix because the deny-list and audit policy still treat the command as dangerous; the safer path is one coordinated policy update or an explicit decision to keep opt-in behavior and adjust docs/error messaging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep screen recording policy consistent:** `src/gateway/node-command-policy.ts:25`
  - body: This adds `screen.record` to the macOS platform defaults, but the command still flows through `DEFAULT_DANGEROUS_NODE_COMMANDS`. New gateway setup copies that list into `gateway.nodes.denyCommands`, and deny entries are applied after defaults, so wizard-created gateways will still block `screen.record` despite the new default. Update the dangerous-command/wizard/audit policy together with this default or keep the command opt-in.
  - confidence: 0.94
