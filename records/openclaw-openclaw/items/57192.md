## Summary

Keep open. Current main still blocks macOS `screen.record` by default, and this PR is the open implementation candidate for the linked bug, but the patch is incomplete because it broadens a privacy-sensitive default without aligning the dangerous-command, setup, audit, docs, and proof requirements.

## What This Changes

The PR adds `screen.record` to the macOS node command default allowlist, adds a gateway regression test, and includes unrelated OpenAI WebSocket warm-up changes to omit empty `tools` arrays.

## Best Possible Solution

Make one maintainer-approved policy update: either default-allow macOS `screen.record` consistently across gateway policy, setup defaults, audit/docs/tests, and real macOS proof, or keep explicit opt-in and clarify capability docs.

## Reproduction Assessment

Yes, from source inspection rather than a live macOS run: macOS nodes declare and implement `screen.record`, current gateway defaults exclude it unless explicitly allowed, and `node.invoke` rejects non-allowlisted commands.

## Solution Assessment

No. Adding only a macOS platform default is incomplete because setup-seeded `denyCommands`, dangerous-command audit behavior, docs, and current tests still preserve explicit opt-in semantics.
