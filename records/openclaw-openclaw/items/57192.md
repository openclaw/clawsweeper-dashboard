## Summary

Keep open. The linked mismatch is real, but this PR is not merge-ready because it changes default access to a privacy-sensitive screen-recording command while current policy, setup defaults, audit checks, and docs still treat it as explicit opt-in.

## What This Changes

The PR adds `screen.record` to macOS platform defaults, adds a gateway regression test for `macOS 26.4.0`, and omits empty OpenAI WebSocket warm-up tool arrays in agent code/tests.

## Best Possible Solution

Decide whether macOS `screen.record` should remain explicit opt-in or become a default, then update gateway policy, dangerous-command classification, setup defaults, audit behavior, docs, and regression tests together.

## Reproduction Assessment

Yes by source inspection. Current main excludes `screen.record` from the macOS default allowlist, tests assert it remains gated, and wizard-created configs would deny it through `DEFAULT_DANGEROUS_NODE_COMMANDS`.

## Solution Assessment

No. Adding `screen.record` only to macOS defaults is not the narrowest maintainable solution because the rest of the security policy still defines it as an explicit opt-in command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
