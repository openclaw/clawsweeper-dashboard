## Summary

Keep open for maintainer review. The branch is not safe to merge as-is because it default-allows a privacy-sensitive `screen.record` command in one gateway default list while current setup, audit, tests, and docs still classify it as dangerous and explicit opt-in; the PR also lacks after-fix real macOS recording proof.

## What This Changes

The branch adds `screen.record` to macOS gateway defaults with a regression test, and separately changes the OpenAI WebSocket warm-up path to omit an empty `tools` array.

## Best Possible Solution

Make one maintainer-approved policy change: either default-allow macOS `screen.record` across gateway policy, setup defaults, audit checks, docs, and tests, or keep explicit opt-in and clarify capability messaging.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: macOS nodes declare `screen.record`, current main excludes it from macOS defaults, and `node.invoke` rejects declared commands that are not allowlisted.

## Solution Assessment

No. Adding `screen.record` only to macOS platform defaults is incomplete because current setup deny-lists, dangerous-command audit behavior, docs, and tests still preserve explicit opt-in semantics.
