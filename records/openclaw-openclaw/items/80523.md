## Summary

Keep open: the patch targets a plausible ACP dispatch failure, but it introduces silent no-op runtime controls for Claude/Gemini and drops Claude effort handling while also lacking real Foreman/ACP behavior proof.

## What This Changes

The branch adds Claude/Gemini ACP launch-time model overrides, skips selected runtime config replays for those adapters, and lets `sessions_yield` succeed with gateway session context but no `onYield` callback.

## Best Possible Solution

Keep launch-time model injection only where the adapter contract supports it, distinguish automatic replay from direct runtime controls, and make unsupported controls either apply through advertised adapter keys or fail clearly with live ACP proof.

## Reproduction Assessment

No live Foreman/ACP reproduction is included in the PR, but the blocking regression is source-reproducible: `setSessionConfigOption` persists after `runtime.setConfigOption` succeeds, while this PR makes Claude/Gemini updates return without touching the adapter.

## Solution Assessment

No. Broad no-op suppression is not the best fix because it makes active model/thinking updates look successful; the safer path is to distinguish automatic replay from direct user/runtime controls and preserve supported adapter updates or fail clearly.
