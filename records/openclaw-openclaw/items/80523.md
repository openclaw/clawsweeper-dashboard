## Summary

Keep open: this recent external bug-fix PR still contains blocking correctness issues and lacks the required real ACP/Foreman proof, so it should not merge or be cleanup-closed yet.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model handling, suppresses selected runtime config replays for those adapters, and lets `sessions_yield` return success without an in-turn callback when session context exists.

## Best Possible Solution

Keep launch-time model overrides only where the adapter contract supports them, make explicit runtime controls either apply or fail clearly, and require redacted real ACP/Foreman proof before merge.

## Reproduction Assessment

No, not for the original live Foreman/ACP failure. The blocking patch regression is source-reproducible because the PR can return success from `setConfigOption` while current main persists the requested option afterward.

## Solution Assessment

No: blanket no-op suppression is not the best fix because direct model/thinking updates can look successful while the running ACP session remains unchanged. The safer path is to distinguish startup replay from explicit runtime controls and either apply supported controls or return an unsupported-control error.
