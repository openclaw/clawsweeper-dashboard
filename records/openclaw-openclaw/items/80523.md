## Summary

Keep open: the PR still has blocking correctness findings and lacks required real behavior proof. Current main and upstream ACP adapter contracts show the branch can report model/thinking config changes as successful without applying them, and the Claude launch path drops the requested thinking effort.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model overrides, skips selected runtime config replays for those adapters, and changes `sessions_yield` to accept gateway session context without an `onYield` callback.

## Best Possible Solution

Keep launch-time model injection only where the adapter contract supports it, make explicit runtime controls truthful by applying them or failing clearly, and require a real Foreman/ACP proof run before merge.

## Reproduction Assessment

Yes for the blocking patch regression by source inspection: a Claude/Gemini config update can resolve from `setConfigOption` while current main persists the requested option afterward. No live reproduction was provided for the original Foreman/ACP failure.

## Solution Assessment

No: broad no-op suppression is not the best fix because runtime controls can look successful while the live ACP session remains unchanged. The safer path is to separate startup replay from explicit runtime updates and either apply supported adapter controls or return an unsupported-control error.
