## Summary

Keep this PR open: it is a plausible ACP/Foreman bug fix, but the proposed patch still has blocking correctness issues and lacks after-fix real behavior proof for the external ACP setup.

## What This Changes

The branch adds Claude/Gemini ACP launch-time model handling, suppresses selected runtime config replays for those adapters, and lets `sessions_yield` accept gateway session context without an `onYield` callback.

## Best Possible Solution

Keep the launch-time model path only where the adapter contract supports it, make runtime controls truthful by applying supported updates or failing clearly, and require a real ACP/Foreman proof run before merge.

## Reproduction Assessment

No live Foreman/ACP reproduction is provided, but the blocking patch regression is source-reproducible: the PR can return success from `setConfigOption` while current main persists the requested option afterward.

## Solution Assessment

No: broad no-op suppression is not the best fix because direct model/thinking updates can look successful while the running ACP session remains unchanged. The safer approach is to distinguish startup replay from explicit runtime controls and either apply supported adapter controls or return an unsupported-control error.
