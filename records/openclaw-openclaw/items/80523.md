## Summary

Keep open: this is a plausible ACP dispatch hardening PR, but the current diff is not mergeable as-is because it silently treats unsupported Claude/Gemini config updates as successful and drops the Claude effort path. It also still needs real Foreman/ACP behavior proof from the contributor.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model injection, suppresses selected runtime config replays, and relaxes `sessions_yield` for gateway session contexts.

## Best Possible Solution

Preserve launch-time model injection only where adapter contracts support it, keep direct runtime config updates truthful, and require redacted live Foreman/ACP proof before merge.

## Reproduction Assessment

No high-confidence live reproduction of the original Foreman failure is provided. The blocking regression introduced by the PR is source-reproducible from the `setConfigOption` no-op paths and current manager persistence behavior.

## Solution Assessment

No: broad no-op suppression is not the best fix because it makes active model/thinking updates look successful. The safer solution is to distinguish automatic replay from direct controls, apply supported adapter updates, re-ensure sessions, or return an unsupported-control error.
