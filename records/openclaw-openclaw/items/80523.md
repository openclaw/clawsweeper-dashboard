## Summary

Keep open: this is a focused ACP dispatch fix attempt, but the branch is conflicting with current main, lacks real ACP/Foreman after-fix proof, and still changes runtime config/yield semantics in ways that can silently mislead callers.

## What This Changes

The branch adds Claude/Gemini ACP launch-time model overrides, suppresses selected Claude/Gemini runtime config replays, and lets `sessions_yield` return yielded when session context exists without an `onYield` callback.

## Best Possible Solution

Rebase onto current main, keep unsupported runtime config updates visible unless they are truly applied, keep yield success tied to an actual interruption/resume path, and add redacted real ACP/Foreman proof before merge.
