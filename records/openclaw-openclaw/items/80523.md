## Summary

Keep open: the PR still has source-reproducible correctness blockers in the ACP runtime-control path and the external-author real-behavior proof gate is still unmet.

## What This Changes

The branch adds launch-time Claude/Gemini ACP model overrides, suppresses selected Claude/Gemini runtime config replays, and changes `sessions_yield` to accept gateway session context without an `onYield` callback.

## Best Possible Solution

Keep launch-time model handling only where the adapter contract supports it, make runtime controls either apply or fail truthfully, and require a real ACP/Foreman proof run before merge.

## Reproduction Assessment

No high-confidence live reproduction of the original Foreman/ACP failure is provided. The blocking patch regressions are source-reproducible from the PR diff plus current-main persistence and dependency contracts.

## Solution Assessment

No: the broad no-op suppression is not the best fix because direct model/thinking updates can look successful while the running ACP session remains unchanged. A safer solution distinguishes startup replay from explicit runtime controls and applies supported adapter controls or returns an unsupported-control error.
