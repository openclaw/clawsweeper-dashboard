## Summary

Keep open: the PR still has source-proven correctness blockers and lacks real Foreman/ACP after-fix proof, so it is not ready to merge.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model injection, suppresses selected runtime config replay, and loosens `sessions_yield` success when no yield callback exists.

## Best Possible Solution

Keep the launch-time model path where adapters support it, but make runtime controls truthful by applying supported model/effort updates, re-ensuring the session, or returning an unsupported-control error instead of silently succeeding.

## Reproduction Assessment

Yes for the introduced review regression via source inspection: `setSessionConfigOption` persists after runtime success while the PR returns before delegating Claude/Gemini controls. No live Foreman/ACP reproduction is provided for the original failure mode.

## Solution Assessment

No: broad no-op suppression is not the best fix because it makes active model/thinking updates look successful. The safer approach is to distinguish startup replay from direct user/runtime controls and preserve supported adapter updates or fail clearly.
