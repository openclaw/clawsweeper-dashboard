## Summary

The PR should stay open because it targets a plausible ACP dispatch/session reliability problem, but the current branch is not merge-ready: it silently accepts runtime-control updates that should apply or fail visibly, marks `sessions_yield` as successful without the turn-ending callback, and still lacks contributor-owned real ACP/Foreman proof.

## What This Changes

The branch adds Claude/Gemini ACP launch-time model injection, skips selected Claude/Gemini runtime config replays, and changes `sessions_yield` to return success when session context exists without an `onYield` callback.

## Best Possible Solution

Rebase onto current main, preserve visible application or rejection for ACP runtime controls, add any still-needed Gemini launch support narrowly, keep `sessions_yield` success tied to an actual yield callback, and provide redacted ACP/Foreman proof.
