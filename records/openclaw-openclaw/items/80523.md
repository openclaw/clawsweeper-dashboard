## Summary

Keep this PR open, but it should not merge as-is: the launch-time model idea may be useful, yet the patch makes explicit Claude/Gemini runtime controls look successful without applying them and still lacks real ACP/Foreman proof.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model handling, suppresses selected runtime config replays for those adapters, and lets `sessions_yield` accept gateway session context without an `onYield` callback.

## Best Possible Solution

Keep launch-time model handling only where the adapter contract supports it, make runtime controls truthful by applying supported updates or returning an unsupported-control error, and require a real ACP/Foreman proof run before merge.

## Reproduction Assessment

No high-confidence live reproduction is available for the original Foreman ACP failure. The blocking patch regression is source-reproducible because the PR can return success from `setConfigOption` while current main persists the requested option afterward.

## Solution Assessment

No. The broad no-op suppression is not the best fix because direct model/thinking updates can look successful while the running ACP session remains unchanged; the safer path is to distinguish startup replay from explicit runtime controls.
