## Summary

Keep open for contributor changes: the PR targets a plausible ACP worker dispatch bug, but the current diff has blocking runtime-control regressions and still lacks real ACP/Foreman behavior proof.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model overrides, suppresses selected runtime config replays for those agents, and lets `sessions_yield` return success in gateway session context without an `onYield` callback.

## Best Possible Solution

Keep launch-time model handling only where the adapter contract supports it, make runtime controls either apply or fail clearly, and require a real ACP/Foreman proof run before merge.

## Reproduction Assessment

No for the original Foreman failure: the PR body and comments provide test commands only, not a live ACP/Foreman run. The blocking PR regressions are source-reproducible because the diff can return success before the delegate while current main persists config after runtime success.

## Solution Assessment

No: launch-time model injection may be useful, but broad successful no-op suppression is not the narrowest maintainable fix. Runtime controls should be applied when supported or rejected so persisted session state remains truthful.
