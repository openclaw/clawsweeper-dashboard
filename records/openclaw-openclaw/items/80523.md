## Summary

Keep this PR open: the ACP/Foreman bug it targets is plausible, but the current branch still has blocking correctness issues around truthful runtime config handling and lacks required real behavior proof.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model overrides, suppresses selected Claude/Gemini runtime config replays, and lets `sessions_yield` report success when only gateway session context exists.

## Best Possible Solution

Preserve launch-time model selection only where the adapter contract supports it, keep runtime controls truthful by applying supported keys or returning unsupported-control errors, and require a real ACP/Foreman proof run before merge.

## Reproduction Assessment

No live Foreman/ACP reproduction is provided. The blocking patch regression is source-reproducible from the PR diff because `setConfigOption` can resolve without delegating while current main persists the requested option afterward.

## Solution Assessment

No: broad no-op suppression is not the best fix because direct runtime controls can appear successful while the running ACP session is unchanged. The safer path is to distinguish startup replay from explicit runtime updates and either apply adapter-supported controls or fail clearly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not report ignored config updates as applied:** `extensions/acpx/src/runtime.ts:994-1019`
  - body: These branches return successfully before calling the delegate for Claude/Gemini model, thinking/effort, and timeout keys. `AcpSessionManager` persists inferred options after `runtime.setConfigOption` resolves, so a runtime model or thinking change can appear to work while the live ACP backend keeps the old value.
  - confidence: 0.88

  - body: For Claude spawns with `thinking`, this only emits `CLAUDE_ACP_NO_EFFORT_APPLY`, but the pinned Claude ACP package does not read that env var; the later `effort` config path is also skipped by this PR. That drops the requested thinking level for new Claude ACP workers.
  - confidence: 0.84
