## Summary

Keep open: this is a plausible ACP dispatch bug-fix PR, but the branch still has blocking source-level regressions in runtime config handling and lacks required real ACP/Foreman behavior proof.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model overrides, skips selected runtime config replays for those agents, and changes `sessions_yield` to accept gateway session context without an `onYield` callback.

## Best Possible Solution

Keep the launch-time model path only where the adapter contract supports it, make explicit runtime controls either apply or fail clearly, and require a real ACP/Foreman proof run before merge.

## Reproduction Assessment

No for the original Foreman failure: the PR body provides test commands only and no live ACP/Foreman reproduction. The blocking PR regressions are source-reproducible because `AcpxRuntime.setConfigOption` can return before the delegate while `AcpSessionManager.setSessionConfigOption` persists after runtime success.

## Solution Assessment

No: launch-time model injection may be useful, but broad successful no-op suppression is not the narrowest maintainable fix. Runtime controls should be applied when the adapter supports them, or reject unsupported controls so persisted session state remains truthful.
