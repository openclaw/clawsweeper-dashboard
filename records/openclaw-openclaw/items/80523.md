## Summary

Keep open, but this PR is not merge-ready: it still contains useful ACP dispatch work, yet current `main` only partially overlaps it and the submitted patch has blocking correctness findings plus no real ACP/Foreman behavior proof.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model overrides, suppresses selected ACP runtime config replays, and changes `sessions_yield` to report success when only session context exists.

## Best Possible Solution

Rebase onto current `main`, keep only dependency-backed launch-time controls, forward or explicitly error unsupported runtime config updates, and make `sessions_yield` succeed only when it actually performs the documented turn-ending handoff.

## Reproduction Assessment

Yes, source inspection is enough for the review findings: the PR-added branches return before applying config updates, and the PR-added yield path returns `yielded` without invoking `onYield`. I did not run tests because this review is read-only.

## Solution Assessment

No, this is not yet the best fix: launch-time ACP control may be useful, but the patch also suppresses supported runtime controls and weakens the documented yield contract. A narrower rebase that preserves supported controls and makes unsupported controls explicit would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Return an error or apply config updates:** `extensions/acpx/src/runtime.ts:994-1009`
  - body: These Claude/Gemini branches silently return for model/thinking/effort updates, so callers see a successful control replay even though nothing changed. That is especially wrong for Claude: `claude-agent-acp` 0.39.0 supports `model` and `effort` via `setSessionConfigOption`, and current main has a test that forwards Claude `model`. Please either forward supported controls or return an explicit unsupported-control error instead of reporting success.
  - confidence: 0.92
- **[P2] Preserve Claude effort startup control:** `extensions/acpx/src/runtime.ts:565-567`
  - body: When `input.thinking` is provided for Claude, this helper only sets `CLAUDE_ACP_NO_EFFORT_APPLY=1`; it never passes an effort value at launch, and the later runtime replay is also skipped. The result is that an explicit thinking/effort request is silently dropped, so the startup path needs to apply the effort or leave the supported runtime config path available.
  - confidence: 0.86
- **[P2] Do not report yield success without ending the turn:** `src/agents/tools/sessions-yield-tool.ts:24-28`
  - body: Returning `status: "yielded"` without `onYield` does not abort the active model turn or persist the hidden yield context that the runner uses for the next message. The tool description and docs promise that `sessions_yield` ends the current turn, so this path should stay an error or be wired to the same real yield handoff before reporting success.
  - confidence: 0.94
