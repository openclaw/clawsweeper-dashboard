## Summary

Keep open: this is a plausible ACP dispatch bug fix, but the current PR head still has source-reproducible correctness regressions and lacks required real ACP/Foreman proof, so it is not mergeable or closeable.

## What This Changes

The PR adds Claude/Gemini ACP launch-time model overrides, suppresses selected Claude/Gemini runtime config replays, and lets `sessions_yield` report success when only session context is present.

## Best Possible Solution

Keep launch-time adapter overrides only where the adapter contract supports them, make unsupported runtime controls fail instead of persisting false state, preserve real yield interruption semantics, and require a real ACP/Foreman proof run before merge.

## Reproduction Assessment

No for the original Foreman failure: the PR body provides tests but no live ACP/Foreman reproduction or after-fix proof. The blocking PR regressions are source-reproducible from the PR diff, current manager persistence contract, and `sessions_yield` callback-based turn-ending path.

## Solution Assessment

No: launch-time model injection may be useful, but successful no-op suppression and yield-success-without-interrupt are not the narrowest maintainable fix. Runtime controls should either apply through the adapter contract or fail clearly, and `sessions_yield` should only report success when the turn-ending mechanism actually runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Return an error or apply runtime config updates:** `extensions/acpx/src/runtime.ts:994-1015`
  - body: These Claude/Gemini branches return before calling `delegate.setConfigOption` for model and thinking controls. The manager persists options after runtime success, so `/acp model` or `/acp set thinking` can report success while the active ACP session keeps the old setting.
  - confidence: 0.88
- **[P2] Preserve Claude effort startup control:** `extensions/acpx/src/runtime.ts:566-568`
  - body: For Claude launches with `thinking`, this only sets `CLAUDE_ACP_NO_EFFORT_APPLY`, but `claude-agent-acp` 0.33.1 does not read that environment variable and the PR also skips the later effort config replay. New Claude ACP workers can lose the requested thinking level.
  - confidence: 0.84
- **[P2] Do not report yield success without ending the turn:** `src/agents/tools/sessions-yield-tool.ts:26-31`
  - body: When no `onYield` callback exists, this now returns `yielded`, but the current turn-ending path relies on that callback aborting the active run. The model can continue after a normal tool result even though the tool claims it yielded.
  - confidence: 0.78
