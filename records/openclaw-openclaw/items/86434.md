## Summary

This is a useful provider idea, but current main already has the generic realtime voice plugin seam that Talk uses, and the request is for an optional ElevenLabs provider implementation rather than a missing OpenClaw core API.

## What This Changes

Add an ElevenLabs full-duplex realtime voice provider for browser Talk, including provider config for API key, model, voice, latency tier, transport, and optional Conversational AI agent ID.

## Best Possible Solution

Publish and maintain an ElevenLabs realtime voice provider through ClawHub using the existing realtime voice provider SDK; reopen core work only for a concrete missing SDK or Talk transport seam.

## Reproduction Assessment

Not applicable: this is a provider feature request, not a bug report. Source inspection confirms current ElevenLabs support covers TTS, media understanding, and realtime transcription, while Talk realtime voice is already provider-pluggable.

## Solution Assessment

No for OpenClaw core as requested: the narrow maintainable path is a provider plugin using registerRealtimeVoiceProvider, not a new core-owned provider path. If that implementation exposes a specific SDK gap, that gap should become a focused core/API issue.
