## Summary

Keep open: current main and the latest release still encode provider-qualified Google Vertex model IDs into invalid `google%2F...` path segments, while this focused PR aligns Vertex URL construction with the already-merged Gemini-path normalization and has sufficient live proof.

## What This Changes

This PR strips the redundant `google/` prefix before encoding Google Vertex model IDs and adds a Vertex transport regression test for the generated request URL.

## Best Possible Solution

Land the focused Vertex URL-boundary normalization and keep the regression test beside the existing Google transport coverage so both Google REST paths enforce the same model-id invariant.

## Reproduction Assessment

Yes. Current main source deterministically builds the encoded `google%2F...` Vertex path, and the PR body supplies redacted live Vertex REST plus real OpenClaw transport before/after output showing failure before and successful generation after the patch.

## Solution Assessment

Yes. Stripping the same-provider prefix at the Vertex URL boundary reuses the existing provider-owned helper and avoids changing canonical model IDs, configuration, routing, or public APIs.
