The PR changes the Google Vertex request URL builder to strip a redundant `google/` provider prefix from model IDs and adds a regression test for the constructed Vertex URL.

## Best Possible Solution

## Reproduction Assessment

Yes from source inspection: current main's Vertex URL builder percent-encodes `google/` in a provider-qualified model id, while the sibling Gemini builder strips the prefix first and Google's Vertex contract expects publisher and model as separate path segments.

## Solution Assessment

Yes, this is the best narrow fix for Google-publisher Vertex models: OpenClaw already hardcodes `publishers/google/models/` in this path and already has a local `stripGoogleProviderPrefix` helper used by the sibling Gemini builder.
