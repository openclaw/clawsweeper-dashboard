## Summary

Keep this PR open because current main still lacks a Feishu doctor migration for legacy account `botName`, and the proposed Feishu-owned direction matches the existing doctor contract path. It is not merge-ready as-is: the new production extension file uses a broad config-runtime SDK import that the repo guard explicitly rejects, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR adds a Feishu doctor migration that rewrites legacy `channels.feishu.accounts.*.botName` to `name`, exports the contract, registers the doctor adapter, and adds shared migration coverage.

## Best Possible Solution

Land this PR, or an equivalent Feishu-owned doctor migration, after replacing the broad SDK config import, adding the required changelog entry, and preserving coverage for normal migration, existing-name conflicts, and blank-name handling.

## Reproduction Assessment

Yes. A current-main config with `channels.feishu.accounts.main.botName` follows a source-backed reproduction: strict Feishu account schema rejects the extra key, runtime only reads `name`, and current main has no Feishu doctor normalizer to rewrite it.

## Solution Assessment

Yes in direction, but not as currently patched. Keeping the migration inside the Feishu plugin boundary and exposing it through the existing doctor contract is the narrow maintainable fix; the implementation should use the narrow `config-types` subpath and include the user-facing changelog entry before merge.
