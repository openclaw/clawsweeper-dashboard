## Summary

Keep open: this is a plausible narrow fix for the linked ACP Discord/Codex regression, but the PR introduces a completion edge case and still lacks required after-fix real behavior proof.

## What This Changes

The PR reorders ACP spawn session patching after runtime initialization, accepts output-bearing ACP streams that close without `done`, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a narrow ACP fix that initializes metadata before dispatch, tolerates no-`done` completion only after visible output or useful tool output, and includes redacted real Discord/Codex proof before merge.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction path: ACP child sessions are patched before metadata initialization, and output-bearing streams without `done` are rejected; I did not run the live Discord/WSL2 flow in this read-only review.

## Solution Assessment

Partly. The PR takes the right narrow direction, but it should not treat thought-only deltas as successful output, and it still needs real after-fix behavior proof before merge.
