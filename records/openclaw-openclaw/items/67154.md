## Summary

Keep open. Current main and the latest release still keep mutable exec last-used metadata in `exec-approvals.json`; the related atomic-write fix addresses the earlier corruption race, but not the requested runtime-store split.

## What This Changes

The issue requests moving `lastUsedAt`, `lastUsedCommand`, and related usage metadata from `exec-approvals.json` into a new `exec-runtime.json` store.

## Best Possible Solution

Introduce a reviewed, backward-compatible exec runtime usage store that keeps allowlist policy immutable while merging legacy and new usage metadata for display and audit surfaces.

## Reproduction Assessment

Yes, for the remaining storage-contract problem: current source, tests, docs, protocol schema, macOS code, and release `v2026.5.12` still store last-used metadata inside `exec-approvals.json` and write it back from runtime paths. I did not run a live multi-container restart/corruption race.

## Solution Assessment

Yes in direction, but not as a gateway-only two-line change. The maintainable fix needs a cross-surface storage contract with migration, display, protocol, macOS, and security review.
