## Summary

Keep open. Current main still stores mutable exec usage metadata inside the allowlist file, and the related atomic-write fix plus the newer stale-writeback security thread do not implement the requested `exec-runtime.json` storage split.

## What This Changes

The issue requests separating `lastUsedAt` and `lastUsedCommand` runtime usage data from the immutable exec allowlist by adding a dedicated `exec-runtime.json` store.

## Best Possible Solution

Introduce a reviewed, backward-compatible exec runtime usage store and update all exec approval readers, writers, display surfaces, tests, docs, and macOS parity without broadening host-exec policy.

## Reproduction Assessment

Yes for the remaining storage-mixing behavior: current source and tests show `recordAllowlistUse` and `recordAllowlistMatchesUse` writing last-used metadata back into `exec-approvals.json`. I did not run a live multi-container restart or race reproduction in this read-only pass.

## Solution Assessment

Yes in direction, but not as a two-line gateway-only change. The narrow maintainable solution needs a reviewed storage-contract change that covers gateway, node-host, macOS, CLI/UI/protocol snapshots, legacy metadata reads, and docs.
