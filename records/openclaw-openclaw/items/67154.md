## Summary

Keep this open. Current main still stores exec runtime usage metadata on allowlist entries and still saves those updates through `exec-approvals.json`; the separate `exec-runtime.json` store requested here has not landed.

## What This Changes

The issue requests moving mutable exec usage fields such as `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath` from `exec-approvals.json` into a dedicated `exec-runtime.json` store while preserving allowlist behavior.

## Best Possible Solution

Introduce a reviewed, backward-compatible `exec-runtime.json` usage store; stop ordinary gateway, node-host, and macOS allowlist execution from writing usage metadata into `exec-approvals.json`; keep legacy approvals and legacy usage reads working; and update protocol, CLI, Control UI, macOS UI, docs, and tests to present merged usage data without changing enforcement.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main has no `exec-runtime.json`, and allowlist-approved gateway/node paths still call `recordAllowlistMatchesUse`, which saves `lastUsed*` fields into the approvals file.

## Solution Assessment

Unclear. The requested storage split is a plausible end state, but the proposed "~2-line" change is not the safest implementation because the contract spans core storage, protocol validators, CLI/UI display, native macOS behavior, and docs.
