## Summary

Keep open. Current main still stores exec allowlist policy and mutable usage metadata in the same approvals file, and the earlier atomic-write fix for the related corruption bug did not implement the requested exec-runtime.json split.

## What This Changes

The issue requests moving mutable exec usage metadata out of exec-approvals.json into a separate exec-runtime.json store while preserving existing allowlist behavior.

## Best Possible Solution

Introduce a reviewed, backward-compatible exec-runtime.json usage store; stop allowlist-approved runs from writing runtime metadata to exec-approvals.json; and update CLI, UI, protocol, docs, and macOS parity without broadening host-exec policy.

## Reproduction Assessment

Yes for the remaining storage-mixing behavior: source inspection and current tests show recordAllowlistUse/recordAllowlistMatchesUse writing lastUsedAt and lastUsedCommand back to exec-approvals.json. The older truncation race is separately mitigated by atomic writes and is not the remaining request.

## Solution Assessment

Yes in direction, but not as a two-line gateway-only change. The maintainable fix is a reviewed storage-contract change that covers gateway, node-host, macOS, CLI/UI/protocol snapshots, legacy metadata reads, and docs.
