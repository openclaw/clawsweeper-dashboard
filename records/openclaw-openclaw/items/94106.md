- `P1`: The PR fixes a default secrets migration path that can silently delete a non-migrated provider credential and break auth on the next run.

## Summary

Keep open: the PR is a focused fix with strong proof for the reported cross-provider `.env` over-scrub, but the new allow-list misses a valid auth-profile migration shape and can leave known provider env plaintext behind when the target omits `authProfileProvider` and migrates to a non-env SecretRef.

## What This Changes

The PR narrows `secrets apply` `.env` credential scrubbing from all known provider/plugin env vars to migrated-target env vars and adds regression coverage for provider collisions, auth-profile targets, plugin/config targets, and non-provider env refs.

## Best Possible Solution

Keep the scoped scrub approach, but derive auth-profile provider env keys from the existing auth-profile store or tighten the documented plan contract and tests so every valid auth-profile migration has enough provider metadata before merge.
