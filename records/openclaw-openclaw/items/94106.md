## Summary

This PR is a strong and well-proven fix for the reported same-value provider collision, but it still needs one auth-profile scrub edge case fixed before merge.

## What This Changes

## Best Possible Solution

Keep the scoped allow-list approach, but derive auth-profile provider identity from the existing auth-profile store when plan metadata is absent, and add a regression for that documented manual-plan shape before merge.

## Solution Assessment

No, not quite. The scoped allow-list is the right layer, but it needs an auth-profile provider fallback for documented manual plans before it is the best fix.
