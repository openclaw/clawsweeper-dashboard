## Best Possible Solution

Yes. Source inspection shows a high-confidence path: a provider-auth target causes provider env keys to be scrubbed, while same-provider non-auth target values are still recorded in the same provider value bucket.

## Solution Assessment

No. The current implementation is close, but the best fix is to keep provider env-key scrubbing paired only with provider-auth/auth-profile migrated values and keep ancillary provider target values target-local.
