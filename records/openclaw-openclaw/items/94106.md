## Summary

Keep open: the PR is a focused fix with strong proof for the main `.env` value-collision bug, but source review found one remaining over-scrub path when an already-SecretRef provider target is included in the same plan as another migrated target with the same plaintext value.

## What This Changes

The PR narrows `secrets apply` `.env` scrubbing to env keys derived from migrated secret targets and adds collision regression coverage for provider, plugin, gateway-token, header, and auth-profile cases.

## Best Possible Solution

Keep the narrowed scrub boundary, but land only after env-key eligibility is tied to target-owned plaintext values and secrets owners accept the final classification.

## Solution Assessment

No. The proposed owner-boundary direction is right, but the implementation still needs to bind scrubbed values to the env keys owned by the same target instead of combining target-derived keys with all migrated values.
