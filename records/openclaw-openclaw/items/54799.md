## Summary

Keep this open. Current main still lacks declarative skill trigger metadata, skill-to-cron reconciliation, event-triggered skill launch, trigger state in skills list/info, and a skills audit command; the newer comment adds a concrete cron-heavy use case rather than making the request obsolete.

## What This Changes

Request declarative schedule/event trigger metadata in `SKILL.md` frontmatter so skills can register autonomous runs and expose trigger status and audit data.

## Best Possible Solution

Define a versioned `metadata.openclaw.triggers` contract with validation, explicit operator enablement, durable last/next trigger state, safe reconciliation into Gateway-owned cron jobs, and skills list/info/audit reporting.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main shows the requested trigger contract and audit/status surfaces are not implemented, but this is a feature request rather than broken existing behavior.

## Solution Assessment

Unclear until maintainer design is settled. Declarative metadata is plausible, but the maintainable solution should be versioned, validated, opt-in, and explicitly tied to an audited trust boundary before `SKILL.md` content can launch autonomous work.
