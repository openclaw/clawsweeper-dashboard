## Summary

Keep this open. Current main and the latest stable tag still lack a SKILL.md trigger metadata contract, scheduler reconciliation from skill metadata, event-trigger semantics, and skills CLI trigger/audit state; the discussion shows real cron demand, but implementation needs product and security design first.

## What This Changes

The issue requests declarative schedule/event trigger metadata in SKILL.md frontmatter so skills can register autonomous cron/event execution and expose last-triggered/audit state.

## Best Possible Solution

Define a versioned, validated trigger metadata contract with explicit operator enablement, gateway-owned reconciliation into cron/event jobs, durable trigger state, and skills list/info/audit reporting.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new feature/config-contract request. Source and docs inspection give high confidence that current main lacks SKILL.md trigger metadata, scheduler reconciliation, and trigger/audit state surfaces.

## Solution Assessment

Unclear until maintainers settle the product and security contract. Declarative metadata is plausible, but it should be versioned, validated, opt-in, and reconciled by the gateway scheduler instead of allowing arbitrary skill frontmatter to launch work by default.
