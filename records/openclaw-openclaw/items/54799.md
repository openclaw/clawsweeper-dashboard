## Summary

Keep open for maintainer product and security review: current main and v2026.6.8 still do not parse, render, register, or audit declarative scheduled/event skill triggers, and the existing plugin scheduler path is bundled-only rather than a complete ClawHub escape hatch.

## What This Changes

The issue requests optional SKILL.md frontmatter for scheduled and event-based skill triggers, plus last-triggered and never-triggered skill audit surfaces.

## Best Possible Solution

Define the permissioned automation contract first, likely by choosing between explicit cron/standing-order configuration, a new plugin scheduling API, or a guarded SKILL.md metadata schema, then implement parser, scheduler registration, audit/list output, docs, and tests together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms current main and v2026.6.8 omit scheduled/event trigger metadata from skill parsing, prompt rendering, docs, CLI audit surfaces, and scheduler registration.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A SKILL.md field is plausible, but the safest solution must decide whether autonomous scheduling belongs in skill metadata, explicit cron/standing orders, or a permissioned plugin API first.
