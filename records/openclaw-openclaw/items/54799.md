## Summary

Keep this open. Current main still lacks declarative SKILL.md trigger metadata, scheduler/event reconciliation, skills trigger status, and a skills audit surface; the closed related implementation was not merged, and the open keyword-routing issue only covers a narrower prompt-routing problem.

## What This Changes

The issue requests optional SKILL.md frontmatter for scheduled and event-based skill triggers, plus cron registration, lifecycle event triggering, last-triggered status, and never-triggered audits.

## Best Possible Solution

Define a versioned, validated trigger metadata contract with explicit operator enablement, gateway-owned cron/event reconciliation, durable trigger state, and skills list/info/audit reporting.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence absence check for trigger metadata, scheduler reconciliation, last-triggered status, and skills audit support on current main.

## Solution Assessment

Unclear until maintainers settle the public contract and trust model. Declarative metadata is plausible, but the maintainable path should be validated, opt-in, and reconciled through the gateway scheduler rather than letting arbitrary SKILL.md content launch work by default.
