Request to add optional SKILL.md frontmatter for schedule/event triggers, cron registration, lifecycle-triggered skills, and last-triggered/audit output in the skills CLI.

## Best Possible Solution

Define a versioned, validated trigger metadata contract with explicit operator enablement, gateway-owned reconciliation into cron/event jobs, durable trigger state, and skills list/info/audit reporting.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new feature request. Source inspection confirms current main lacks SKILL.md trigger metadata, scheduler reconciliation, and trigger/audit state surfaces.

## Solution Assessment

Unclear until maintainers settle the product and security contract. Declarative metadata is plausible, but it should be versioned, validated, opt-in, and integrated through the gateway scheduler instead of letting arbitrary SKILL.md content launch work by default.
