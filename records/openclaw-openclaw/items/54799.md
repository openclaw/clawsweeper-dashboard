## Summary

Keep open for maintainer product/security review. Current main and v2026.6.6 still have explicit operator-managed cron jobs and skill metadata for visibility/gating/install only, with no SKILL.md schedule/event trigger contract or exact canonical duplicate that owns this work.

## What This Changes

Add declarative SKILL.md frontmatter for scheduled/event triggers, cron registration, last-triggered visibility, and never-triggered auditing.

## Best Possible Solution

Define a narrow, audited scheduler contract for trusted skill or plugin metadata before allowing SKILL.md declarations to register cron/event triggers.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new SKILL.md metadata and scheduling contract. Source inspection does confirm current main lacks that contract.

## Solution Assessment

Unclear until maintainers choose the public contract. A trigger field is plausible, but direct SKILL.md auto-registration is not safely the best path until trust scope, event vocabulary, cron ownership, and audit state are decided.
