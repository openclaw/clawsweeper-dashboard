## Summary

Keep open for maintainer product and security review: current main and the latest release still do not parse, register, render, or audit declarative SKILL.md scheduled/event triggers, and the existing Cron-backed plugin scheduler is not a complete external-plugin escape hatch.

## What This Changes

Add declarative scheduled/event trigger metadata to SKILL.md so skills can register autonomous cron/event runs and expose last-triggered or never-triggered audit status.

## Best Possible Solution

Define the permissioned automation contract first, likely choosing between explicit cron or standing-order configuration, an external plugin scheduling API, or guarded SKILL.md metadata, then implement parser, scheduler registration, audit/list output, docs, and tests together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Current source and docs show explicit cron jobs and SKILL.md metadata without the requested scheduled/event trigger contract.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A SKILL.md field is plausible, but the safest solution must decide whether autonomous scheduling belongs in skill metadata, explicit cron or standing orders, or a permissioned plugin API first.
