## Summary

Keep this open. Current main still does not define trigger metadata for skills or wire skills into cron/event execution, list/status trigger state, or audit reporting, and the latest cron-heavy user comment reinforces a real feature need rather than resolving it.

## What This Changes

The issue requests optional `SKILL.md` frontmatter for schedule/event triggers plus skills CLI/status/audit visibility for autonomous skill execution.

## Best Possible Solution

Define a versioned, validated trigger metadata contract with explicit operator enablement, gateway-owned reconciliation into cron/event jobs, durable last/next-trigger state, and skills list/info/audit reporting.

## Reproduction Assessment

Not applicable as a bug reproduction. Current-main source inspection is enough to show the requested skill trigger metadata, scheduler reconciliation, and audit/status surfaces are not implemented, but this is a new feature request rather than broken existing behavior.

## Solution Assessment

Unclear until maintainers settle the product and security contract. Declarative metadata is a plausible direction, but the durable solution should be versioned, validated, opt-in, and integrated through the gateway scheduler instead of letting arbitrary `SKILL.md` content launch work by default.
