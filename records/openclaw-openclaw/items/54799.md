## Summary

Keep open. Current main still treats `SKILL.md` OpenClaw metadata as gating/display/install metadata only, and the skills status, skills CLI, and cron snapshot paths do not implement declarative schedule/event triggers, last-trigger state, or a `skills audit --never-triggered` surface.

## What This Changes

Add optional `SKILL.md` frontmatter metadata for schedule/event skill triggers, register schedule-triggered skills with cron, expose last-trigger state, and add audit support for never-triggered skills.

## Best Possible Solution

Define a versioned `metadata.openclaw.triggers` contract, validate schedule and event trigger forms, require explicit operator enablement for autonomous execution, reconcile schedule triggers into managed cron jobs with stable ownership, whitelist supported lifecycle events, persist last/next trigger state, and expose that state through `openclaw skills list`, `openclaw skills info`, docs, and a focused skills audit command.
