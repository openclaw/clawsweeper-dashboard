## Summary

Keep open for maintainer product/security review. Current main and v2026.6.6 still have explicit cron jobs and skill metadata for visibility, gating, install, and command exposure only; the existing plugin/ClawHub path also does not fully cover external skill auto-scheduling because Cron-backed scheduled session turns are documented and enforced as bundled-only.

## What This Changes

The issue requests optional SKILL.md trigger metadata for scheduled/event skill execution plus last-triggered and audit visibility.

## Best Possible Solution

Define a narrow audited scheduler/event trigger contract, likely through core/plugin API ownership, with explicit operator approval, provenance, and last-run/audit state before SKILL.md metadata can register autonomous work.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new SKILL.md trigger and scheduler contract. Source inspection confirms current main lacks that contract.

## Solution Assessment

Unclear until maintainers choose the public contract. A trigger field is plausible, but direct SKILL.md auto-registration is not safely the best path until trust, event vocabulary, cron ownership, and audit state are decided.
