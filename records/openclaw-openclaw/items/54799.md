## Summary

Keep this open. Current main still lacks declarative `SKILL.md` trigger metadata, scheduler reconciliation from skill metadata, event-triggered skill launch, last-triggered skill status, and `skills audit`; the requested capability crosses core skill metadata, cron, lifecycle events, CLI/UI status, and security policy, so it needs maintainer design before implementation.

## What This Changes

The issue requests declarative trigger metadata in `SKILL.md` frontmatter so scheduled or event-driven skills can be registered, audited, and shown in skills CLI status.

## Best Possible Solution

Define a versioned `metadata.openclaw.triggers` contract, validate schedule and event trigger forms, require explicit operator enablement for autonomous execution, reconcile approved schedules into owned cron jobs, persist last/next trigger state, and expose that state through skills list/info plus a focused audit command.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main shows the requested trigger contract, scheduler reconciliation, event launch path, last-triggered status, and skills audit command are not implemented.

## Solution Assessment

Unclear until maintainer design is settled. Declarative metadata is a plausible direction, but the maintainable solution should be versioned, validated, opt-in, and explicitly tied to an audited trust boundary before any `SKILL.md` content can launch autonomous work.
