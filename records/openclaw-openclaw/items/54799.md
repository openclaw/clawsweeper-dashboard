## Summary

Keep open: current main and the latest release still lack a SKILL.md trigger metadata contract, skill-to-cron/event reconciliation, and skills CLI trigger/audit state. The cron-user follow-up adds concrete demand, while existing cron/plugin hooks and the merged Policy metadata PR do not supersede this security-sensitive product design request.

## What This Changes

Add optional SKILL.md trigger metadata for scheduled/event skills, reconcile it with Gateway cron/events, and surface last-trigger/audit state in the skills CLI.

## Best Possible Solution

Define a versioned, validated skill-trigger metadata contract with explicit operator enablement, Gateway-owned cron/event reconciliation, durable trigger state, skills list/info/audit reporting, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new feature request. Source and latest-release inspection verify that the requested trigger metadata, scheduler reconciliation, and skills audit/last-trigger surfaces are absent.

## Solution Assessment

Unclear until maintainers settle the product and security contract. Declarative metadata is plausible, but it should be versioned, validated, explicitly enabled, and reconciled through the Gateway scheduler rather than allowing arbitrary SKILL.md content to launch work by default.
