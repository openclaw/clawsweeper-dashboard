## Summary

Keep this open: current main still has no SKILL.md trigger metadata contract, no skill-to-cron/event reconciliation, and no skills CLI trigger/audit state. The follow-up cron-user comment adds concrete demand, but the remaining work is product- and security-sensitive design rather than a narrow repair.

## What This Changes

Add optional SKILL.md frontmatter for schedule/event triggers so OpenClaw can register skill jobs and surface last-triggered/audit state.

## Best Possible Solution

Define a versioned, validated skill-trigger metadata contract with explicit operator enablement, gateway-owned cron/event reconciliation, durable trigger state, skills list/info/audit reporting, tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new feature request. Source inspection on current main verifies that the requested trigger metadata, scheduler reconciliation, and skills audit/last-trigger surfaces are absent.

## Solution Assessment

Unclear until maintainers settle the product and security contract. Declarative metadata is plausible, but it should be versioned, validated, explicitly enabled, and reconciled through the Gateway scheduler instead of allowing arbitrary SKILL.md content to launch work by default.
