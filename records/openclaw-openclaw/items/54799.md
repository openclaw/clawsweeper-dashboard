Keep open: current main and the latest release still lack a SKILL.md trigger metadata contract, skill-to-cron/event reconciliation, and skills CLI trigger/audit state. The cron-user follow-up adds concrete demand, but this remains a product- and security-sensitive feature rather than a narrow cleanup close.

## What This Changes

The issue requests declarative SKILL.md trigger metadata for scheduled/event-driven skills, automatic cron/event registration, and skills CLI last-triggered/audit reporting.

## Best Possible Solution

Define a versioned, validated skill-trigger metadata contract with explicit operator enablement, Gateway-owned cron/event reconciliation, durable trigger state, skills list/info/audit reporting, tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new feature request. Source inspection on current main verifies that the requested trigger metadata, scheduler reconciliation, and skills audit/last-trigger surfaces are absent.

## Solution Assessment

Unclear until maintainers settle the product and security contract. Declarative metadata is plausible, but it should be versioned, validated, explicitly enabled, and reconciled through the Gateway scheduler rather than allowing arbitrary SKILL.md content to launch work by default.
