## Summary

Keep open: this is a focused bug-fix PR for a real ACP task-completion failure mode, but the proposed core heuristic can misclassify valid text-only completions and the external PR still lacks after-fix proof from a real ACP/Codex setup.

## What This Changes

The PR tracks ACP tool-call activity, fails parented ACP background turns that finish with only short progress-like text, and adds a regression test for a Portuguese Codex progress-only transcript.

## Best Possible Solution

Capture the real Codex/acpx event stream and fix the adapter or runtime contract so premature close/error surfaces as an ACP failure while legitimate text-only completed turns still succeed.

## Reproduction Assessment

Source-reproducible for the core shape: current main marks a parented ACP stream with text_delta followed by done as succeeded. The real Codex ACP adapter path from the linked issue still needs live redacted event-stream proof.

## Solution Assessment

No: failing based on progress-looking text and missing tool calls is not the narrowest maintainable fix because ACP success does not require a tool_call event. The safer direction is adapter/runtime-level failure signaling for premature termination.

## Review Findings

Overall correctness: patch is incorrect
