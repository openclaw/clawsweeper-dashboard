## Summary

The confirmed failure is expected fallback exhaustion after MiniMax overloads and a BlockRun timeout, and the reporter now also suspects MiniMax; however, the separate claims that local Ollama, interactive completion, and tool output stop mid-job remain unverified, so this fresh issue is not safe to close or turn into a cron-specific fix.

## What This Changes

Investigate cron and interactive agent runs that fail when configured model routes overload, time out, or stop mid-job, and determine whether overload retry behavior should change.

## Best Possible Solution

Keep overload handling in the shared model fallback path, decide the default-backoff behavior with fresh-install and upgrade evidence under https://github.com/openclaw/openclaw/issues/85888, and separately reproduce the claimed Ollama or mid-turn failure with exact routes and gateway diagnostics before changing runtime code.

## Reproduction Assessment

No, not for the full issue. Source inspection explains the reported FallbackSummaryError, but there is no controlled current-main reproduction proving that a healthy local Ollama route or equivalent interactive run fails for the same reason.

## Solution Assessment

No, a cron-only wait loop is not the best solution. Cron uses the shared fallback engine, an overload-backoff setting already exists, and changing its default requires a cross-path retry and timeout policy decision.
