## Summary

Keep open. Current main still lacks a typed delivery-intent boundary for approved async exec followup runs, so Telegram topic delivery, normal Gateway `agent` dispatch, and overflow-compaction retry can still compose in the failure shape described by the report even though adjacent heartbeat stripping and topic-routing defenses exist.

## What This Changes

The issue reports that Telegram topic async exec approval followups can leak internal heartbeat or completion text into user-visible chat after context overflow and compaction.

## Best Possible Solution

Add an additive structured delivery-intent path for approved async exec followups, preserve it through Gateway dispatch and overflow-compaction retry, and suppress only internal/no-op orchestration text while preserving real exec results to the originating Telegram topic.
