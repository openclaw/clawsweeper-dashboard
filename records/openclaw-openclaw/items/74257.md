## Summary

Keep open. Current main still routes approved async exec completions through an ordinary Gateway `agent` followup without a structured silent/internal delivery intent, and the adjacent tests cover routing, topic preservation, and literal `HEARTBEAT_OK` stripping rather than the reported Telegram topic plus overflow-compaction leak.

## What This Changes

The issue requests suppression of internal async exec completion and heartbeat/no-op followup text in Telegram topic sessions when context-overflow compaction retries an approved tool flow.

## Best Possible Solution

Add an additive structured delivery-intent path for approved async exec followup runs, propagate it through the Gateway agent RPC and overflow-compaction retry, and keep real command output deliverable to the originating Telegram topic while suppressing internal/no-op orchestration text.

## Reproduction Assessment

## Solution Assessment

No current patch is proposed. The maintainable direction is a typed runtime delivery intent across exec followup and compaction boundaries, not broader string suppression of `HEARTBEAT_OK` or internal narration.
