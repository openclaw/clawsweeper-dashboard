## Summary

Keep open. Current main still sends approved async exec completions as ordinary Gateway `agent` followups and lacks a typed internal/silent delivery intent through the Gateway RPC and overflow-compaction retry path.

## What This Changes

Report asks OpenClaw to stop Telegram topic exec-approval followup and context-overflow recovery internals such as `HEARTBEAT_OK` from being delivered as normal user-visible messages.

## Best Possible Solution

Add an additive typed delivery-intent path for async exec/internal followups across Gateway agent RPC, runner/compaction retry, and channel delivery so real command output still reaches Telegram topics while no-op/internal orchestration text is suppressed.

## Reproduction Assessment

Unclear. The redacted logs give a credible path through Telegram topic approval, context overflow compaction, and async exec followup, but there is no deterministic current-main reproduction or fixture for the whole composition.

## Solution Assessment

No. Current main's prompt wording, heartbeat filtering, and direct fallback safeguards do not solve this composition; a typed runtime delivery-intent contract is safer than global string suppression.
