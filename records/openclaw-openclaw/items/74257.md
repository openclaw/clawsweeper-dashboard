## Repair Work Prompt

Fix the Telegram async exec followup leak reported in https://github.com/openclaw/openclaw/issues/74257. Add an additive structured delivery-intent path for approved async exec followup runs instead of blanket `HEARTBEAT_OK` string suppression. The intent should be set by `sendExecApprovalFollowup`, accepted by the Gateway `agent` protocol, propagated through `src/gateway/server-methods/agent.ts` into reply/runner params, and preserved across overflow-compaction retry so internal/no-op orchestration followups are consumed silently while real command results remain deliverable to the originating Telegram topic/thread. Add regression coverage for the exec followup args, Gateway protocol/dispatch plumbing, overflow-compaction retry preservation, reply payload suppression, and Telegram topic delivery/approval path. Include a user-facing changelog fix entry. Do not change provider defaults, broad heartbeat behavior, or suppress legitimate command output.
