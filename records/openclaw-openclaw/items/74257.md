## Summary

Keep open. Current main still only mitigates the standalone `HEARTBEAT_OK` token leak, while approved exec completions remain ordinary Gateway agent followups and ordinary overflow recovery still retries the original prompt/current-message metadata outside the narrower mid-turn precheck path.

## What This Changes

The issue asks OpenClaw to keep Telegram async exec/heartbeat followup text internal after context-overflow compaction while preserving in-flight turn state and approval routing.

## Best Possible Solution

Add a typed internal/no-op delivery intent for async exec completion runs and make overflow-compaction retry preserve the in-flight turn identity instead of replaying the original channel prompt as fresh input.
