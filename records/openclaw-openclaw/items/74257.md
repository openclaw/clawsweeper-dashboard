Keep open. Current main still routes approved async exec completions through an ordinary Gateway `agent` followup, with no structured silent/internal delivery intent carried through the RPC or overflow-compaction retry path; adjacent heartbeat stripping and Telegram topic tests do not prove the reported composition is fixed.

## What This Changes

The issue requests that Telegram topic approved exec followups suppress internal heartbeat/no-op orchestration text across context-overflow compaction while preserving real command-output delivery.

## Best Possible Solution

Add an additive structured delivery-intent path for approved async exec followup runs, propagate it through Gateway agent RPC and overflow-compaction retry, and suppress only internal/no-op orchestration text while keeping real results deliverable to the originating Telegram topic.
