## What This Changes

The issue reports that Telegram topic exec approval followups can leak internal heartbeat or async-completion text after context overflow and approval expiry.

## Best Possible Solution

Add an additive structured delivery-intent contract for approved async exec completion runs and preserve it across Gateway agent dispatch, session resume, and overflow-compaction retry so internal/no-op orchestration text is consumed while real exec results still route to the originating Telegram topic.
