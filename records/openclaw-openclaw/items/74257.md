## Summary

Keep open. Current main has adjacent HEARTBEAT_OK stripping, Telegram topic routing, callback-error containment, and structured heartbeat tooling, but the reported composition is still not proved fixed: approved async exec completion still resumes as an ordinary agent turn, the Gateway agent RPC has no silent/internal followup intent for this path, and overflow recovery still retries after compaction.

## What This Changes

The issue reports that Telegram topic approved async exec followups can leak internal HEARTBEAT_OK or followup narration into user-visible chat after context-overflow compaction and approval expiry.

## Best Possible Solution

Add a bounded structured delivery-intent path for approved async exec followup runs, carry that intent through the Gateway agent RPC and overflow-compaction retry, and add a regression test for Telegram topic approval followup so internal/no-op orchestration text is suppressed while real approved-command output still reaches the originating topic.

Unclear. The issue includes detailed redacted production logs and current main still has the matching code paths, but there is no deterministic step-by-step repro or exact regression test for Telegram topic approval followup plus overflow-compaction retry.

## Solution Assessment

No current patch is proposed, and current main is not enough to call this solved. The best fix is structured runtime delivery intent across exec followup and compaction boundaries, not broader string suppression of HEARTBEAT_OK or internal narration.
