## What This Changes

The PR widens gateway approval delivery from a boolean to delivered/attempted metadata across shared approval handling, exec/plugin forwarding, iOS push delivery, and focused tests.

## Best Possible Solution

Land the shared attempted/delivered contract with the unconfigured-route fallback preserved once maintainers accept configured delivery failure as a fail-closed no-route condition.

Yes, this is the right owner-boundary fix: the shared approval handler owns pending/no-route lifecycle, and the PR carries attempted/delivered metadata from exec forwarding, plugin forwarding, and iOS push instead of adding a channel-specific special case.
