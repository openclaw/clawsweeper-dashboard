## Summary

Keep this PR open but not merge-ready: the approval-delivery contract is the right layer for the fix and the supplied terminal proof covers the central gateway path, but the patch can still expire an approval that was partially delivered to a real target.

## What This Changes

This PR changes gateway exec/plugin approval delivery from a boolean result to delivered/attempted metadata across shared handling, forwarding, iOS push, and regression tests.

## Best Possible Solution

Land a version that preserves the new attempted/delivered contract, counts partial_failed sends with visible results as delivered, expires only true no-visible-result failures, and has maintainer acceptance for the fail-closed fallback change.

## Reproduction Assessment

Yes, from source: current main flattens delivery failure and no configured route into the same false value, so a failed explicit delivery can fall back to turn-source routing. I did not run tests in this read-only review, but the PR body includes after-fix terminal proof for the central gateway path.

## Solution Assessment

No, not yet. The shared delivery contract is the right fix shape, but the implementation must respect the durable send partial_failed contract before the fail-closed behavior is safe to merge.
