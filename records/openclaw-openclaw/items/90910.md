## Summary

Keep this PR open for maintainer review: it fixes a real lossy approval-delivery contract, but the proposed fail-closed behavior narrows existing same-chat approval fallback when configured forwarding or iOS push delivery fails, so it is compatibility-sensitive before merge.

## What This Changes

The PR changes gateway exec/plugin approval delivery from a boolean to delivered/attempted metadata, makes forwarding and iOS push report attempted failures, and adds regression coverage for no-route expiration.

## Best Possible Solution

Merge only after maintainers explicitly accept the fail-closed compatibility change, or revise the patch to preserve same-chat fallback by default and make strict fail-closed behavior an intentional documented path.
