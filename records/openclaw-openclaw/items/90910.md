## Summary

Keep open: this focused gateway approval bug-fix PR appears correct and has sufficient after-fix behavior proof, but it intentionally changes configured approval delivery failures to fail closed, so maintainer acceptance of that compatibility risk is still needed before merge.

## What This Changes

The branch replaces boolean approval delivery results with delivered/attempted metadata across shared gateway approval handling, exec/plugin forwarding, iOS push delivery, and regression tests.

## Best Possible Solution

Land the shared attempted/delivered contract once maintainers accept configured delivery failure as a fail-closed no-route condition while preserving the unconfigured-route fallback.

## Reproduction Assessment

Yes by source inspection: current main collapses delivery failure to false and then allows turn-source routing when delivery is false, while the forwarder returns true before observing all-target delivery failure. I did not run a live current-main repro in this read-only review.

## Solution Assessment

Yes, this is the right owner-boundary fix: the shared approval handler owns pending/no-route lifecycle, and the PR carries attempted/delivered metadata from exec forwarding, plugin forwarding, and iOS push instead of adding a channel-specific special case.
