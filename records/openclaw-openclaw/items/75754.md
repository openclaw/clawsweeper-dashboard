## Summary

Keep open: this is a focused fix for a valid ClawDock dashboard bug, and current main still lacks the published-port remap. The PR has sufficient real behavior proof, but current head still exposes the gateway token through child-process argv while URL-encoding it, so it needs a narrow repair before merge.

## What This Changes

The PR updates ClawDock’s shell helper to resolve Docker-published gateway ports for dashboard URLs, append and redact token fragments, and add a changelog entry.

## Best Possible Solution

Land this helper-scoped port remap after changing token encoding to avoid child-process argv exposure and preserving the related `--no-deps` dashboard fix from https://github.com/openclaw/openclaw/pull/80494.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT=3200`, start the Docker Compose gateway, then run `clawdock-dashboard`; current main opens the CLI-container loopback URL instead of resolving the Docker-published host port.

## Solution Assessment

Partly. Fixing the mismatch in `scripts/clawdock/clawdock-helpers.sh` is the narrowest maintainable solution, but the current PR should encode the token without placing it in child-process argv before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing the dashboard token through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`

  - confidence: 0.88
