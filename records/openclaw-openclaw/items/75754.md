## Summary

Keep open: current main still exhibits the reported ClawDock dashboard port bug, and the helper-side approach is appropriate, but the patch should not merge until token encoding stops exposing the gateway token through child process arguments.

## What This Changes

Updates the ClawDock shell helper to rewrite dashboard URLs to the Docker-published gateway port, reconstruct/redact tokenized browser URLs, and add a changelog entry.

## Best Possible Solution

Land this helper-scoped fix after changing token encoding to avoid argv exposure, and coordinate with https://github.com/openclaw/openclaw/pull/80494 so the final helper uses both the published host port and `--no-deps`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT=3200`, start the Docker Compose gateway, then run `clawdock-dashboard`; current main opens the CLI-container loopback URL instead of resolving the published host port.

## Solution Assessment

Partly. Fixing the mismatch in the ClawDock helper is the narrowest maintainable solution, but this PR should encode the token without placing it in child process argv before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing the dashboard token through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` receives the gateway token and then invokes `python3` or `node` with that token as a positional argument. That makes the token visible in process listings during the helper run, which defeats the redaction goal; pass it over stdin or otherwise encode without placing it in argv.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch fixes terminal redaction but introduces a token-in-argv exposure while URL-encoding the dashboard token.

Concerns:

- **[medium] Dashboard token is exposed through child process argv:** `scripts/clawdock/clawdock-helpers.sh:74`
  - body: The helper calls `python3` or `node` with the gateway token as a positional argument during URL encoding. Local process listings can expose command-line arguments, and the repo already treats token argv exposure as unsafe, so the token should be passed through stdin or encoded without a subprocess argv.
  - confidence: 0.88
