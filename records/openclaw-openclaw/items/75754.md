## Summary

Keep open: this PR targets a real ClawDock dashboard bug and includes sufficient terminal proof, but it should not merge until dashboard-token encoding stops passing the raw token through child-process argv. Landing should also preserve the companion `--no-deps` helper fix from https://github.com/openclaw/openclaw/pull/80494.

## What This Changes

The PR rewrites `clawdock-dashboard` URLs to the Docker-published gateway port, appends and redacts dashboard token fragments, and adds a changelog entry.

## Best Possible Solution

Land the helper-scoped published-port remap after encoding tokens without raw child-process argv exposure and preserving the companion `--no-deps` dashboard lookup.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT` to a host port different from `18789`, start the Docker Compose gateway, then run `clawdock-dashboard`; current main opens the CLI-container loopback URL unchanged.

## Solution Assessment

Partly. Fixing `scripts/clawdock/clawdock-helpers.sh` is the right boundary, but the token encoder should use stdin or another non-argv path before this implementation is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing dashboard tokens through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` is called with the gateway token and then starts `python3` or `node` with that token as a positional argument. Local process listings can expose child-process argv values while the helper runs, so feed the token through stdin or encode it without a subprocess argument before appending it to the browser URL.
  - confidence: 0.88
