## Summary

Keep open: current main and the latest release still open the container-reported ClawDock dashboard URL unchanged, and this PR is a useful focused fix, but it should not merge until the new token encoding path avoids child-process argv exposure.

## What This Changes

The PR updates `scripts/clawdock/clawdock-helpers.sh` to remap dashboard URLs to Docker Compose's published gateway port, append a token fragment, redact displayed fallback URLs, and add a changelog entry.

## Best Possible Solution

Land the helper-scoped port remap after changing token encoding to avoid argv exposure and preserving the related `--no-deps` dashboard fix from https://github.com/openclaw/openclaw/pull/80494.

## Reproduction Assessment

Yes, source-level. Current main and `v2026.5.12` open the CLI-reported dashboard URL unchanged while Compose can publish the gateway on a different host port.

## Solution Assessment

Partly. Fixing the ClawDock helper is the narrowest maintainable solution, but the current PR should encode the token without placing it in child-process argv before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing dashboard tokens through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` is called with the gateway token and then launches `python3` or `node` with that token as a positional argument. Local process listings can expose child-process argv values while the helper runs, so the token should be encoded through stdin or without a subprocess argument.
  - confidence: 0.88
