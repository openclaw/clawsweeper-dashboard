## Summary

Keep open: the PR addresses a real ClawDock dashboard bug, but the latest head still introduces a concrete token-handling regression by passing the gateway token through child-process argv during URL encoding.

## What This Changes

The PR updates `clawdock-dashboard` to remap CLI-reported loopback dashboard URLs to the Docker-published gateway port, append/redact token fragments, and add a changelog entry.

## Best Possible Solution

Land the helper-scoped published-port remap after changing token encoding to avoid argv exposure and coordinating with the `--no-deps` dashboard fix in https://github.com/openclaw/openclaw/pull/80494.

## Reproduction Assessment

Yes, source-level. Current main opens the CLI-container URL unchanged while Docker Compose can publish the gateway on a different host port via `OPENCLAW_GATEWAY_PORT`.

## Solution Assessment

No, not as currently written. Fixing the ClawDock helper is the right boundary, but token encoding should avoid child-process argv and the final helper should preserve the related `--no-deps` behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing dashboard tokens through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` is called with the gateway token and then launches `python3` or `node` with that token as a positional argument. Local process listings can expose child-process argv values while the helper runs, so encode the token through stdin or without a subprocess argument.
  - confidence: 0.88
