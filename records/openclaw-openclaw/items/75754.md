## Summary

Keep open: current main still has the broken ClawDock dashboard URL path, and this PR is the paired implementation, but the branch still has a concrete security blocker because it passes the dashboard token through child-process argv while URL-encoding it.

## What This Changes

The PR updates the ClawDock dashboard shell helper to remap loopback dashboard URLs to the Docker-published gateway port, append/redact browser auth tokens, and add a changelog entry.

## Best Possible Solution

Land a helper-scoped remap that resolves the published Compose port, preserves scheme/path and terminal redaction, encodes tokens without raw argv exposure, and keeps the related `--no-deps` dashboard lookup behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT` to a non-default published port, run `clawdock-dashboard`, and current main opens the loopback URL reported from the CLI container unchanged.

## Solution Assessment

No, not as written. The helper is the right surface for the fix, but the implementation should encode the token without passing it through child-process argv and should be coordinated with the open no-deps dashboard helper change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing dashboard tokens through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` receives the gateway token and then starts `python3` or `node` with that token as a positional argument. Local process listings can expose argv values while the encoder runs, so feed the token through stdin or encode it without a child-process argument before appending it to the browser URL.
  - confidence: 0.88
