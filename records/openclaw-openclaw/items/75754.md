## Summary

Keep open: current main still has the reported ClawDock dashboard-port mismatch, and this PR is a narrow, correct-looking fix with the changelog and security pass addressed. The paired bug issue should stay open until this PR or another fix lands.

## What This Changes

The branch updates `scripts/clawdock/clawdock-helpers.sh` and `CHANGELOG.md` so `clawdock-dashboard` remaps loopback dashboard URLs to Docker's published gateway port, appends/redacts the auth token, and avoids fragile parsing dependencies on that path.

## Best Possible Solution

Land the helper-scoped fix after normal maintainer review, keeping Docker host-port translation in ClawDock rather than changing gateway runtime, Control UI, Compose service contracts, plugins, or public APIs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT` to a host port other than `18789`, then current main's `clawdock-dashboard` opens the CLI-reported loopback URL instead of the Docker-published host port.

## Solution Assessment

Yes. The PR fixes the mismatch at the shell helper that launches the host browser, which is narrower and more maintainable than changing gateway runtime URL generation or Docker Compose service contracts.
