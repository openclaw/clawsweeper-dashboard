## Summary

Keep open: this PR targets a real ClawDock dashboard bug and includes sufficient terminal proof, but the current patch still exposes the dashboard token through child process argv while URL-encoding it.

## What This Changes

The PR rewrites ClawDock dashboard URLs to the Docker-published gateway port, reconstructs/redacts tokenized browser URLs, and adds a changelog entry.

## Best Possible Solution

Land this helper-scoped fix after changing token encoding to avoid argv exposure, and coordinate with the separate `--no-deps` ClawDock dashboard fix before merge.

## Reproduction Assessment

Yes, by source inspection. Set `OPENCLAW_GATEWAY_PORT=3200`, start the Docker Compose gateway, then run `clawdock-dashboard`; current main opens the CLI-container loopback URL instead of resolving the host-published port.

## Solution Assessment

Partly. Fixing the mismatch in the ClawDock helper is the narrowest maintainable path, but this PR should encode the token without placing it in child process argv before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing the dashboard token through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` receives the gateway token and invokes `python3` or `node` with that token as a positional argument. Local process listings can expose command-line arguments, which defeats the redaction goal; pass the token over stdin or encode it without a child-process argv value.
  - confidence: 0.88
