## Summary

Keep open: the reported ClawDock bug is real and the PR targets the right helper path, but the patch still passes the gateway token through child-process argv during URL encoding, so it needs a narrow repair before merge.

## What This Changes

The PR updates `scripts/clawdock/clawdock-helpers.sh` to remap loopback dashboard URLs to the Docker-published gateway port, append/redact token fragments, and add a changelog entry.

## Best Possible Solution

Land the helper-scoped port remap after encoding the token without raw child-process argv exposure and preserving the `--no-deps` dashboard lookup from https://github.com/openclaw/openclaw/pull/80494.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT` to a host port different from `18789`, start the Docker Compose gateway, then run `clawdock-dashboard`; current main opens the CLI-container loopback URL unchanged.

## Solution Assessment

Partly. Fixing `scripts/clawdock/clawdock-helpers.sh` is the narrowest maintainable solution, but this PR should avoid token-as-argv encoding and preserve the related `--no-deps` fix before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid passing dashboard tokens through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` is called with the gateway token and then starts `python3` or `node` with that token as a positional argument. Local process listings can expose child-process argv values while the helper runs, so feed the token through stdin or encode it without a subprocess argument before appending it to the browser URL.
  - confidence: 0.88
