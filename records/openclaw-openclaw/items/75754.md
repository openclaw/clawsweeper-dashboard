## What This Changes

The PR updates `scripts/clawdock/clawdock-helpers.sh` so `clawdock-dashboard` resolves the Docker-published gateway port, remaps loopback dashboard URLs for the host browser, appends/redacts token fragments, and adds a `CHANGELOG.md` fix entry.

## Best Possible Solution

Land the helper-scoped fix after maintainer review and required checks complete; keep the gateway runtime, Control UI, Compose service definitions, plugin surfaces, and public APIs unchanged for this Docker-host browser-launch bug.

## Reproduction Assessment

Yes. Configure Docker Compose with `OPENCLAW_GATEWAY_PORT` different from `18789`, start the gateway, source the ClawDock helper, and run `clawdock-dashboard`; current main opens the internal loopback URL from `openclaw-cli dashboard --no-open` unchanged.

## Solution Assessment

Yes. Fixing the translation at the ClawDock helper boundary is the narrow maintainable solution because that is where the host browser launch happens; changing the dashboard CLI or gateway runtime would broaden the fix beyond Docker Compose port publishing.
