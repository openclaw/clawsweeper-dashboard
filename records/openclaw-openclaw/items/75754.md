## What This Changes

The PR updates ClawDock's dashboard shell helper to map Docker-internal dashboard URLs to the Compose-published host port while appending and redacting dashboard auth tokens.

## Best Possible Solution

Land the helper-scoped port remap after encoding the token without child-process argv exposure and preserving the related `--no-deps` dashboard fix if that lands first.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish the gateway on a host port different from 18789, then run `clawdock-dashboard`; current main opens the container-reported loopback URL unchanged.

## Solution Assessment

Partly. Rewriting the ClawDock helper URL is the narrow maintainable fix, but PR head should encode the token without placing it in subprocess argv before merge.
