## Summary

Keep open. Current main still routes cron `payload.model` through shared allowlist resolution that can collapse a configured `kimi-code/...` provider to canonical `kimi/...`, and the only related fix PR in the provided context was closed unmerged.

## What This Changes

The issue requests consistent cron-nested resolution for custom provider refs like `kimi-code/kimi-for-coding`, preservation of custom `models.json` additions during bootstrap, and docs for the canonical alias format.

## Best Possible Solution

Preserve literal custom provider keys before legacy alias normalization in shared model resolution, keep regenerated provider merges from dropping custom model entries, and document the supported Kimi/custom-provider alias shapes with regression coverage.

## Reproduction Assessment

Yes, by source inspection plus the reporter's logs. Current main still has the cron `payload.model` path, exact-provider guard, and Kimi normalization combination needed to turn `kimi-code/kimi-for-coding` into `kimi/kimi-for-coding` when the exact custom provider is not preserved.

## Solution Assessment

Yes. The narrow maintainable fix is shared resolver preservation plus models-config merge coverage, not a cron-only special case; the closed #76163 branch is useful context but did not land and appears narrower than the full report.
