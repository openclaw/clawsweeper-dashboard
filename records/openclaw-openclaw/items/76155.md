## Summary

Keep open. Current main still has a plausible cron-only resolver path that can turn `kimi-code/kimi-for-coding` into `kimi/kimi-for-coding`, and the related models.json merge path can still drop custom models under a regenerated provider key.

## What This Changes

The issue requests consistent cron-nested model alias resolution for hyphenated/custom provider refs, preservation of custom models.json additions, and clearer Kimi alias documentation.

## Best Possible Solution

Cron and embedded runs should share exact-provider preservation for configured/catalog-backed custom providers, while built-in Kimi remains documented as `kimi/kimi-code`.

## Reproduction Assessment

Yes. The issue includes a concrete v2026.4.29 failure log, and current main statically shows cron payload selection can bypass exact provider preservation before `kimi-code` is normalized to `kimi`.

## Solution Assessment

Yes, a narrow fix is appropriate: preserve exact catalog/config provider refs in model selection and merge existing models.json model additions for matching provider keys, with focused regression coverage.
