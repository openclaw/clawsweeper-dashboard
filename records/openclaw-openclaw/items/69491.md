## Summary

Keep open: the Bedrock default-beta direction is useful, but the current PR still has merge-blocking review issues and no after-fix real Bedrock proof.

## What This Changes

The PR branch adds an Amazon Bedrock stream wrapper and tests to inject default Anthropic beta flags into Bedrock Converse payloads for Claude models.

## Best Possible Solution

Land a provider-local Bedrock implementation that adds default Anthropic betas additively, composes with guardrails/service tier/app-profile traits, and leaves configurable custom betas aligned with https://github.com/openclaw/openclaw/issues/39734.

## Reproduction Assessment

Yes for source-level PR review: the diff shows the beta assignment overwrites existing `anthropic_beta`, and current main shows the wrapper would use the broad app-profile name heuristic. I did not establish a live Bedrock reproduction.

## Solution Assessment

No, not as proposed. The provider-local payload path is the right place, but the patch should merge and dedupe beta values, gate on definitive or resolved Claude targets, and include real Bedrock proof before merge.
