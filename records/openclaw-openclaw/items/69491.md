## Summary

Keep open. The PR targets a real Amazon Bedrock provider gap: current main still does not inject the default Anthropic beta flags for Bedrock Claude streams, while the direct Anthropic route already handles those defaults additively. The branch is not mergeable as-is because prior review comments correctly identified that it overwrites existing `anthropic_beta` values and applies Anthropic-only request fields through a documented application-profile name heuristic; it also lacks the required changelog entry for a user-visible feature.

## What This Changes

The PR adds an Amazon Bedrock stream wrapper and tests to inject the default Anthropic beta flags into ConverseStream `additionalModelRequestFields.anthropic_beta` for Claude Bedrock models, including the guardrail composition path.

## Best Possible Solution

Land a narrow provider-local Bedrock payload patch that treats the two default Anthropic betas as additive defaults, only applies them to definitively resolved Claude Bedrock targets, preserves existing `additionalModelRequestFields` and wrapper composition, and records the user-visible default in the changelog.

## Reproduction Assessment

Yes for review-level reproduction: a payload that already contains `additionalModelRequestFields.anthropic_beta` would be overwritten by the PR assignment, and a non-Claude application profile with `claude` in its user-defined profile name would pass the current predicate. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as proposed. The provider-local direction is right, but the implementation should merge and dedupe existing beta flags, use strict Claude ID or resolved profile traits for Bedrock application profiles, and include the required changelog entry.
