## Summary

Keep this PR open. Current main still lacks default Anthropic beta forwarding for Amazon Bedrock Converse streams, but the proposed patch is not ready to merge because it overwrites existing Bedrock `anthropic_beta` payload values and applies Anthropic-only request fields through a documented application-profile name heuristic.

## What This Changes

The PR adds an Amazon Bedrock stream wrapper and tests to inject the default Anthropic beta flags into `additionalModelRequestFields.anthropic_beta` for Claude Bedrock streams, including guardrail composition.

## Best Possible Solution

Land a provider-local Bedrock payload patch that treats the two default Anthropic betas as additive defaults, preserves existing `additionalModelRequestFields`, uses definitive Claude Bedrock target detection, composes with guardrails/app-profile/Opus 4.7 handling, and documents the user-visible behavior in the changelog.

## Reproduction Assessment

Yes for review-level reproduction: current main inspection shows no Bedrock beta injection, and the proposed payload assignment can be checked directly against a payload that already contains `additionalModelRequestFields.anthropic_beta`. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as proposed. The direction is right, but the implementation should merge and dedupe beta flags and avoid applying Anthropic-only fields through the broad application-profile name heuristic.
