## Summary

Keep open: current main still lacks the Bedrock default-beta behavior, but this PR is not ready as proposed because it overwrites existing beta values, uses a loose Claude heuristic for Anthropic-only fields, and lacks real Bedrock proof.

## What This Changes

The branch adds an Amazon Bedrock stream wrapper and tests to inject the default Anthropic beta strings into `additionalModelRequestFields.anthropic_beta` for Bedrock Claude models.

## Best Possible Solution

Land a provider-local Bedrock implementation that adds default Anthropic betas additively, composes with guardrails, service tier, and app-profile trait handling, and keeps custom beta configuration tracked separately by https://github.com/openclaw/openclaw/issues/39734.

## Reproduction Assessment

Yes for source-level review: the PR diff shows the beta assignment overwrites existing `anthropic_beta`, and the wrapper gates on the current name heuristic. I did not establish a live Bedrock reproduction in this read-only review.

## Solution Assessment

No, not as proposed. The provider-local payload path is the right area, but it should merge and dedupe existing values, gate on definitive or resolved Claude targets, and be backed by real Bedrock proof before merge.

## Review Findings

  - confidence: 0.88
