## What This Changes

The PR adds an `npmResolution.version` fallback for npm plugin dry-run update outcomes and a regression test for probes without a top-level `version`.

## Best Possible Solution

Land one canonical fix that preserves the exact-spec fallback, adds the `npmResolution.version` fallback with regression coverage, and closes the duplicate implementation path after real proof is supplied.

## Reproduction Assessment

Yes. Source inspection shows current main can return npm dry-run metadata on `npmResolution.version` while the dry-run formatter ignores that field, so a mocked successful npm probe without top-level `version` reproduces `current -> unknown`.

## Solution Assessment

Yes. The fallback is the narrow maintainable fix because it reuses metadata already returned by the npm installer, preserves the existing exact-spec fallback, and avoids changing non-npm update behavior.
