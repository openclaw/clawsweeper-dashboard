## Summary

Keep open. Current main still has the reported gap: attempt setup calls `assembleAttemptContextEngine()` before entering `activeSession.prompt()`, loop-time `assemble()` only runs for engines with `ownsCompaction: true`, and the public plugin hook surface still has no full-message `transform_context` hook. The related PR was closed unmerged, so this remains a public API and trust-contract decision rather than cleanup.

## What This Changes

The issue requests a per-model-call context assembly or transform surface so context engines or plugins can inspect and rewrite the full message history during multi-call agent loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract, preferably additive unless maintainers intentionally change `assemble` semantics. The contract should specify prompt-content access, returned-message validation, ordering relative to budget enforcement, and coverage for owning engines, non-owning engines, and multi-tool-call loops.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: inspect the setup-time `assembleAttemptContextEngine()` call, the later `activeSession.prompt()` calls, the `ownsCompaction` branch that gates loop-time assembly, and the absence of any public `transform_context` hook.

## Solution Assessment

Unclear. The additive hook direction from PR #57585 appears safer than globally changing `assemble` timing, but maintainers should choose the API shape and trust model before implementation.

- Hook ordering relative to Pi's private `transformContext`, OpenClaw budget enforcement, tool-result guards, and context-engine compaction is still an API/product choice.
