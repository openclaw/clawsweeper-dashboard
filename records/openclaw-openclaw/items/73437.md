## Summary

Keep open. Current main still has only a private, compaction-owner-gated loop-time assembly path and no public mutating per-LLM-call context hook, while the docs describe context-engine assembly as happening before each model run.

## What This Changes

The issue requests a per-LLM-call context assembly or context-transform hook so context engines/plugins can rewrite the full message array during multi-call agentic loops.

## Best Possible Solution

Define and document a per-model-call context rewrite contract that covers ordering, raw conversation access, message validation, compaction interaction, and regression coverage for embedded runner and plugin hook behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows attempt-time `assembleAttemptContextEngine()`, an `ownsCompaction`-gated private loop hook, and no public mutating `transform_context` hook.

## Solution Assessment

Unclear. An additive hook like the closed related PR looks safer than changing `assemble()` timing globally, but the API shape and trust model need maintainer review before implementation.
