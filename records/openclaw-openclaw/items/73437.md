## Summary

Keep open. Current main still lacks a documented public mutating per-LLM-call context hook; the existing loop-time assembly path is private and compaction-owner gated, while related implementation work remains open and security/API-sensitive.

## What This Changes

The issue requests a per-LLM-call context mutation or assembly hook so plugins/context engines can update the message array inside multi-call agent loops.

## Best Possible Solution

Define one guarded, documented per-model-call context mutation contract with explicit ordering, cloned and validated payloads, policy gates, SDK/API baselines, docs, and focused embedded-runner tests.

## Reproduction Assessment

Yes, at source level. Current main shows setup-time assemble, a private ownsCompaction loop wrapper, and no public mutating per-LLM-call context hook; I did not run a live LLM loop in this read-only review.

## Solution Assessment

Unclear. An additive guarded hook or assembly contract may be safer than changing assemble semantics globally, but the final public API shape and trust boundaries need maintainer design review before implementation.
