## Summary

Keep open. Current main partially addresses engine-owned compaction with a private `ownsCompaction` loop hook, but it still lacks a public mutating per-LLM-call context hook and `llm_input` remains observe-only; the remaining work is an API/security design item with related open PRs still under review.

## What This Changes

The issue requests a per-model-call context assembly or mutating plugin hook so context engines or plugins can rewrite the message array during multi-call agent loops.

## Best Possible Solution

Define a documented, permissioned per-model-call context mutation contract with explicit ordering, message cloning/validation, conversation-access and prompt-injection policy, docs/API baselines, and focused embedded-runner regression tests.

## Reproduction Assessment

Yes, at source level. Current main shows setup-time `assembleAttemptContextEngine()`, an `ownsCompaction`-gated private loop wrapper, and no public mutating context hook; I did not run a live LLM loop in this read-only review.

## Solution Assessment

Unclear. An additive guarded hook may be safer than changing `assemble()` semantics for every engine, but the final API shape, trust gates, and ordering need maintainer design review before implementation.
