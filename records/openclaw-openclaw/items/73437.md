- `impact:session-state`: The requested behavior concerns stale or untrimmed model-visible session context across multi-call agent loops.

## Summary

Keep open: current main and v2026.6.6 still lack the requested public per-LLM-call context mutation contract; the only per-iteration assemble path is private to context engines that set `ownsCompaction`, and related open work does not fully replace this timing issue.

## What This Changes

The issue requests a per-LLM-call context transformation or assembly hook so context engines can mutate or compress messages during multi-call agent loops.

## Best Possible Solution

Define one guarded, documented per-model-call context mutation contract with explicit ordering, cloned and validated message payloads, plugin policy gates, SDK/API baselines, docs, and focused embedded/Codex harness coverage.

## Reproduction Assessment

Yes, at source level: current main assembles before prompt submission, only installs a per-iteration assemble wrapper for `ownsCompaction` engines, and has no public `transform_context` or assemble hook. I did not run a live multi-call LLM loop in this read-only review.

## Solution Assessment

Unclear as implementation direction: an additive guarded per-call context contract is plausible, but maintainers need to choose between changing `ContextEngine` semantics, adding a Plugin SDK hook, or expanding modifying LLM hooks.
