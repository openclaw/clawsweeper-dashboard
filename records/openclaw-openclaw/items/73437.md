## Summary

Keep open. Current main still lacks a documented mutating per-LLM-call plugin hook, and the only loop-time `assemble()` path is a private `transformContext` wrapper gated to context engines with `ownsCompaction: true`; the related PR is closed unmerged.

## What This Changes

Request a per-LLM-call context assembly or rewrite surface so context engines/plugins can revise the full message array during multi-call agent loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract with ordering, validation, conversation-access policy, and focused runner/plugin contract tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence by following attempt setup into `activeSession.prompt(...)`, the `ownsCompaction`-only loop wrapper, and the absence of a public mutating `transform_context` hook.

## Solution Assessment

Unclear. An additive `transform_context` hook appears safer than changing `assemble()` timing for every context engine, but maintainers should choose the API shape and trust model first.
