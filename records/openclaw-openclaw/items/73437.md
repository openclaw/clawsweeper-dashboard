## Summary

Keep open. Current main still lacks a public mutating per-LLM-call context hook, and loop-time `assemble()` only runs through the private embedded-runner wrapper for context engines with `info.ownsCompaction === true`; the related additive-hook PR was closed unmerged.

## What This Changes

Request a per-LLM-call context rewrite hook, or changed `assemble()` timing, so context engines can adjust outbound messages during multi-call agent loops.

## Best Possible Solution

Define a documented per-model-call context rewrite contract that covers hook ordering, raw conversation access, message validation, compaction interaction, and regression coverage for embedded-runner and plugin-hook behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows attempt-time `assemble()`, an `ownsCompaction`-gated private loop hook, and no public mutating `transform_context` hook.

## Solution Assessment

Unclear. An additive hook like the closed related PR appears safer than changing `assemble()` timing globally, but the public API shape and trust model need maintainer review before implementation.

## Risks / Open Questions

- A public mutating per-call hook would allow plugins to rewrite full conversation history, so ordering, validation, failure policy, and conversation-access permission need owner review.
- Changing `assemble()` to run for every model call globally could break engines that currently rely on the narrower per-attempt or owning-compaction semantics.
