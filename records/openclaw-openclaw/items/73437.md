## Summary

Keep open. Current main still lacks a public, mutating per-LLM-call context hook, and loop-time `assemble()` is only wired for context engines that set `info.ownsCompaction === true`; the related additive hook PR was closed unmerged.

## What This Changes

The issue requests a per-LLM-call context transformation or assembly hook so context engines or plugins can rewrite the full message history during agentic tool loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract with explicit ordering, conversation-access permissions, message validation, budget enforcement behavior, and regression coverage across the embedded runner and plugin hook registry.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence by following the setup-time `assembleAttemptContextEngine()` call, the `ownsCompaction`-gated loop hook, and the absence of a public mutating `transform_context` hook.

## Solution Assessment

Unclear. The additive hook direction from PR #57585 looks safer than changing `assemble()` timing for every context engine, but the API shape and prompt-content trust model need maintainer review first.
