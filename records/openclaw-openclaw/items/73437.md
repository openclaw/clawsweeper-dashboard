## Summary

Keep open. Current main still lacks a public mutating per-LLM-call context hook, and the loop-time `assemble()` path is private and limited to `ownsCompaction` engines, so the requested plugin/API behavior remains unresolved.

## What This Changes

Request a per-model-call context assembly or rewrite surface so context engines/plugins can inspect and modify the message array before each LLM call in agentic loops.

## Best Possible Solution

Define a documented per-model-call context rewrite contract covering hook ordering, message validation, raw-conversation access, compaction ownership, and regression coverage for embedded-runner plugin hooks.

## Reproduction Assessment

Yes, by source inspection. Current main shows setup-time context-engine assembly, an `ownsCompaction`-gated private loop wrapper, and no public mutating `transform_context` hook.

## Solution Assessment

Unclear. An additive hook like the closed related PR appears safer than changing `assemble()` timing for every engine, but maintainers need to settle the API shape and trust model first.
