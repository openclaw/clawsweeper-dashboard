## Summary

Keep open. Current main still lacks a documented mutating per-LLM-call context hook, and loop-time `assemble()` is only wired through a `transformContext` wrapper for `ownsCompaction: true` context engines, so this remains an API and trust-contract decision.

## What This Changes

Request a per-model-call context rewrite surface so context engines or plugins can inspect and replace the full message array before each LLM call.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract with explicit ordering, validation, prompt-content access rules, and tests across the embedded runner and plugin hook registry.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence by following the setup-time `assembleAttemptContextEngine()` call, the `ownsCompaction` branch for loop-time assembly, and the absence of a public mutating `transform_context` hook.

## Solution Assessment

Unclear. The additive hook direction from PR #57585 appears safer than changing `assemble()` timing for all engines, but maintainers should choose the API shape and trust model before implementation.
