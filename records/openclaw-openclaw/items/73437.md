## Summary

Keep open. Current main still leaves the reported gap: setup-time `assembleAttemptContextEngine()` runs before `activeSession.prompt()`, loop-time `assemble()` is only wired for `ownsCompaction: true` engines, and the public hook catalog still has no full-message `transform_context` hook; PR #57585 is closed unmerged, so this remains an API/trust-contract decision.

## What This Changes

The issue requests per-model-call context assembly or a transform hook so plugins and context engines can inspect and rewrite full message history during multi-call agent loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract, preferably additive unless maintainers intentionally change `assemble` semantics, with explicit ordering, validation, and prompt-content access rules.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence by following `assembleAttemptContextEngine()` before `activeSession.prompt()`, the `ownsCompaction` branch that gates loop-time assembly, and the absence of a public `transform_context` hook.

## Solution Assessment

Unclear. The additive hook direction from PR #57585 appears safer than globally changing `assemble` timing, but maintainers should choose the API shape and trust model before implementation.

## Risks / Open Questions

- Changing `assemble` to fire on every loop-time model call could change public context-engine semantics for existing third-party plugins.
- A full-message rewrite hook would expose prompt and transcript content to plugin code, so conversation-access policy and returned-message validation need maintainer review.
- Hook ordering relative to Pi's private `transformContext`, OpenClaw budget enforcement, tool-result guards, and context-engine compaction remains an API/product choice.
