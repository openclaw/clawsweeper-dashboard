## Summary

Keep open. Current main has a partial shipped loop-time path for context engines that declare `ownsCompaction: true`, but it still does not provide the requested general per-LLM-call full-message rewrite hook or make `assemble()` run for non-owning context engines. The related additive `transform_context` PR is closed unmerged, so this remains an API and trust-contract decision rather than conservative cleanup.

## What This Changes

The issue requests a per-LLM-call context rewrite or assembly surface so context engines and plugins can adjust the full message array inside multi-call tool loops.

## Best Possible Solution

Define a stable per-model-call context rewrite contract for plugin and context-engine authors, preferably additive unless maintainers intentionally change `assemble()` semantics. The implementation should document prompt-content access and add regression coverage for owning engines, non-owning engines, and multi-tool-call loops.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: inspect the attempt setup, observe the one pre-prompt `assembleAttemptContextEngine()` path, verify loop-time assembly is only installed for `ownsCompaction: true`, and confirm no public `transform_context` hook exists.

## Solution Assessment

Unclear. The additive hook direction from #57585 looks safer than globally changing `assemble()` timing, but hook ordering, message validation, prompt-content access, and budget-enforcement guarantees need maintainer API review before implementation.
