## Summary

Keep open. Current main still lacks the requested general per-LLM-call full-message rewrite surface: attempt setup calls context-engine assembly before submitting the prompt, and the only loop-time assembly wrapper is limited to context engines that declare `ownsCompaction: true`. The related `transform_context` PR is closed unmerged, so this remains a public plugin/context-engine API decision rather than cleanup that can be closed.

## What This Changes

The issue requests that `ContextEngine.assemble()` or a new additive `transform_context` hook run before every LLM call inside the tool loop so context engines/plugins can inspect and rewrite the full message array per model call.

## Best Possible Solution

Define a stable per-model-call context rewrite contract for plugin and context-engine authors, preferably additive unless maintainers intentionally change `assemble` timing. The implementation should document prompt-content access rules and cover owning engines, non-owning engines, and multi-tool-call loops with regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: inspect the attempt setup, observe the single pre-prompt `assembleAttemptContextEngine()` call, then verify that loop-time `assemble()` is only installed for `ownsCompaction: true` and no public `transform_context` hook exists.

## Solution Assessment

Unclear. The additive hook direction from the related closed PR looks safer than changing `assemble` semantics globally, but hook ordering, message validation, prompt-content access, and budget-enforcement guarantees need maintainer API review first.
