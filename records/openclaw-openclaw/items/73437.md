## Summary

Keep open. Current main still lacks a documented public mutating per-LLM-call context hook; the related implementation PRs are open and under API/security review, so this is not cleanup-closeable.

## What This Changes

Request a plugin/context-engine API change so context assembly or an equivalent transform hook can run before each LLM call in an agentic loop.

## Best Possible Solution

Define one guarded, documented per-model-call context mutation contract with explicit ordering, cloned and validated payloads, policy gates, SDK/API baselines, docs, and focused embedded-runner tests.

## Reproduction Assessment

Yes, at source level. Current main shows setup-time assemble, a private ownsCompaction loop wrapper, and no public mutating per-LLM-call context hook; I did not run a live LLM loop in this read-only review.

## Solution Assessment

Unclear. An additive guarded hook may be safer than changing assemble semantics globally, but the final public API shape and trust boundaries need maintainer design review before implementation.
