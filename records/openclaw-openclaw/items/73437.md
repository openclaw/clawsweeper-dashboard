## Summary

Keep open: current main and v2026.6.10 still do not provide a general public per-LLM-call full-context mutation hook. Main has a specialized ownsCompaction loop hook, but that only partially covers one context-engine mode and does not settle the requested plugin/API contract.

## What This Changes

The issue requests a per-LLM-call context transformation or assembly hook so context engines can adjust the full message array during multi-call agent loops.

## Best Possible Solution

Keep this open until maintainers choose the plugin/context-engine boundary for per-LLM-call full-context mutation, coordinated with the adjacent mutable LLM hook and provenance RFC work.

## Reproduction Assessment

Yes at source level: current main has a specialized ownsCompaction loop hook but no general transform_context/before_assemble hook, and non-owning engines still do not get a public per-LLM-call assembly contract. I did not run a live multi-tool-loop session because the source contract evidence is sufficient for triage.

## Solution Assessment

No single implementation is ready to queue automatically. Maintainers need to choose whether the durable fix belongs in ContextEngine.assemble cadence, a new transform_context hook, or the adjacent mutable LLM hook work.
