## Summary

Keep open. Current main only provides per-loop `assemble()` for `ownsCompaction` context engines and still lacks the requested public mutating `transform_context` hook, while the related hook PR was closed unmerged.

## What This Changes

The issue requests per-LLM-call context assembly or a new mutating context hook so context engines/plugins can rewrite messages during multi-call agent loops.

## Best Possible Solution

Define a documented per-model-call context rewrite contract covering hook ordering, message validation, raw conversation access, compaction ownership, and regression coverage for embedded-runner plugin hooks.

## Reproduction Assessment

Yes for source-level behavior. Current main shows setup-time `assembleAttemptContextEngine()`, an `ownsCompaction`-gated private loop wrapper, and no public mutating `transform_context` hook.

## Solution Assessment

Unclear. The merged `ownsCompaction` loop hook is a good narrow fix for engine-owned compaction, but adding a public mutating hook or changing `assemble()` timing for all engines needs maintainer API and trust-model review.
