## Summary

Keep open. Current main still lacks a public mutating per-LLM-call context hook, and the only loop-time `assemble()` path is the private embedded-runner wrapper for context engines with `info.ownsCompaction === true`; the related additive-hook PR was closed unmerged.

## What This Changes

Request a per-LLM-call context rewrite hook, or changed `assemble()` timing, so context engines can adjust the outbound message array during multi-call agent loops.

## Best Possible Solution

Define a documented per-model-call context rewrite contract that covers hook ordering, raw conversation access, message validation, compaction interaction, and regression coverage for embedded-runner and plugin-hook behavior.

## Reproduction Assessment

Yes, from source inspection. Current main shows attempt-time `assemble()`, an `ownsCompaction`-gated private loop hook, and no public mutating `transform_context` hook.

## Solution Assessment

Unclear. An additive hook like the closed related PR looks safer than changing `assemble()` timing globally, but the public API shape and trust model need maintainer review first.
