## Summary

Keep open. Current main still does not provide the requested general per-LLM-call full-message rewrite surface: context-engine assembly is still run during attempt setup, and the only loop-time assembly wrapper is limited to engines that declare `ownsCompaction: true`. The related `transform_context` PR is closed unmerged, so the remaining work is an API and trust-contract decision, not conservative cleanup.

## What This Changes

The issue requests a per-LLM-call context rewrite hook or changed `ContextEngine.assemble` timing so context engines can inspect and modify the full message array inside multi-call agent loops.

## Best Possible Solution

Define a stable per-model-call context rewrite contract for plugin and context-engine authors, preferably additive unless maintainers intentionally change `assemble` semantics. The implementation should document the prompt-content access model and add regression coverage for owning engines, non-owning engines, and multi-tool-call loops.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: inspect the attempt setup, observe the single pre-prompt `assembleAttemptContextEngine()` call, then verify that loop-time `assemble()` is only installed for `ownsCompaction: true` and that no public `transform_context` hook exists.

## Solution Assessment

Unclear. The additive hook direction from #57585 looks safer than globally changing `assemble` timing, but hook ordering, message validation, prompt-content access, and budget-enforcement guarantees need maintainer API review before implementation.
