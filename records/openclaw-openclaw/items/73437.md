## Summary

Keep open. Current main only runs loop-time `assemble()` through the private `installContextEngineLoopHook()` path for engines with `ownsCompaction: true`, and there is still no public mutating `transform_context` hook; the related PR #57585 was closed unmerged, so the remaining API/semantic change needs maintainer review.

## What This Changes

The issue requests a public or context-engine mechanism to assemble or transform the full message array before every LLM call inside an agentic tool loop.

## Best Possible Solution

Define a documented per-model-call context rewrite contract covering hook ordering, validation, raw-conversation access, compaction ownership, and regression coverage for embedded-runner plugin hooks.

## Reproduction Assessment

Yes. Source inspection on current main shows the one-time attempt assembly path, the `ownsCompaction`-gated private loop wrapper, and the absence of a public mutating `transform_context` hook.

## Solution Assessment

Unclear. An additive hook like the closed related PR may be safer than changing `assemble()` timing for all engines, but the API shape and trust model need maintainer review first.
