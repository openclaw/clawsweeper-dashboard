## Summary

Keep open. Current main still lacks the requested public per-LLM-call full-message rewrite contract: normal context-engine assembly still happens before the agent session prompt loop, the only loop-time `assemble` wrapper remains limited to engines with `ownsCompaction: true`, and the related `transform_context` PR is closed unmerged while broader mutable-hook PRs remain unresolved API work.

## What This Changes

The issue requests a per-model-call context-engine or plugin hook so context compression plugins can inspect or rewrite the full message array during multi-call agentic tool loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract for plugin/context-engine authors, preferably preserving existing `ContextEngine.assemble` compatibility unless maintainers explicitly choose a semantics change, and add regression coverage for owning engines, non-owning engines, hook ordering, malformed-message handling, and budget enforcement across multi-tool-call loops.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: `assembleAttemptContextEngine()` runs before prompt submission, the loop-time wrapper is gated to `ownsCompaction: true`, and no public `transform_context` hook exists; a live multi-call LLM run would only add end-to-end confirmation.

## Solution Assessment

Unclear. The additive hook direction from #57585 appears safer than changing `assemble` frequency, but the open broader mutable-hook work in #20802 and #61485 shows maintainers still need to choose the API scope and harden prompt-content access before implementation.
