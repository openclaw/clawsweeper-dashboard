## Summary

Keep open: current main only partially covers compaction-owner context engines through a private loop wrapper and still lacks a documented public mutating per-LLM-call context hook; related implementation PRs remain open and under API/security review.

## What This Changes

The issue requests `ContextEngine.assemble()` or a new context hook to run before each LLM call so plugins can rewrite full message history during agentic tool loops.

## Best Possible Solution

Define one guarded, documented per-model-call context mutation contract with explicit ordering, cloned and validated payloads, policy gates, SDK/API baselines, docs, and focused embedded-runner tests.

## Reproduction Assessment

Yes, at source level: current main shows setup-time assembly, an owner-gated private loop wrapper, and no public mutating per-LLM-call context hook. I did not run a live LLM loop in this read-only review.

## Solution Assessment

Unclear. An additive guarded hook may be safer than changing `assemble()` semantics globally, but the final public API shape and trust boundaries need maintainer design review before implementation.
