## Summary

Keep open: current main only partially addresses this through an owner-gated private context-engine loop hook, while the requested public per-LLM-call context mutation contract remains unresolved and related API/security review is still active at https://github.com/openclaw/openclaw/pull/75162 and https://github.com/openclaw/openclaw/issues/82137.

## What This Changes

Request a per-LLM-call context assembly or mutation surface so context engines/plugins can inspect and rewrite messages before each model call inside agentic loops.

## Best Possible Solution

Define one guarded, documented per-model-call context mutation contract with explicit ordering, cloned and validated message payloads, policy gates, SDK/API baselines, docs, and focused embedded-runner tests.

## Reproduction Assessment

Yes, at source level: current main still shows attempt-time assembly, an owner-gated private loop wrapper, and no public mutating per-LLM-call context hook. I did not run a live LLM loop in this read-only review.

## Solution Assessment

Unclear. An additive guarded hook may be safer than changing assemble semantics globally, but maintainers need to choose the public API shape and trust boundaries before implementation.
