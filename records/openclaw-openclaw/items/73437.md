## Summary

Keep open. Current main still does not provide a public mutating per-LLM-call context hook, and loop-time `assemble()` is only wired through the private `transformContext` wrapper for context engines with `info.ownsCompaction === true`; related PR #57585 was closed unmerged.

## What This Changes

The issue requests a per-LLM-call context transformation or assembly hook so context engines or plugins can rewrite full message history during agentic tool loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract with explicit ordering, trust gates, message validation, budget behavior, and regression coverage across the embedded runner and plugin hook registry.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence by following the setup-time `assembleAttemptContextEngine()` call, the `ownsCompaction`-gated loop hook, and the absence of a public mutating `transform_context` hook.

## Solution Assessment

Unclear. The additive hook direction from PR #57585 looks safer than changing `assemble()` timing for every context engine, but the API shape and prompt-content trust model need maintainer review first.
