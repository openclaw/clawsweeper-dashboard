## Summary

Keep open. Current main still does not provide a public mutating per-LLM-call context hook, and `assemble()` only runs inside the embedded loop for context engines that opt into `info.ownsCompaction === true`; the related additive hook PR was closed unmerged.

## What This Changes

Request per-LLM-call context rewriting or assembly so context engines can inspect and replace the message array during multi-call agentic loops.

## Best Possible Solution

Define a documented per-model-call context rewrite contract that covers ordering, raw conversation access, message-shape validation, compaction interaction, and regression coverage for both embedded runner and plugin hook registry behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows setup-time `assembleAttemptContextEngine()`, the `ownsCompaction`-gated private loop hook, and the absence of a public mutating `transform_context` hook.

## Solution Assessment

Unclear. An additive hook like the closed related PR appears safer than changing `assemble()` timing globally, but the API shape and trust model need maintainer review before implementation.
