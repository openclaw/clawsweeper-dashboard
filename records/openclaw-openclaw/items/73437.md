- `impact:session-state`: The request affects which session/context messages are assembled and sent during multi-call agent loops.

## Summary

Keep open: current main and v2026.6.8 still do not provide the requested public per-LLM-call full-context mutation contract. The embedded runner has a narrower `ownsCompaction` ContextEngine loop path, and related open work covers modifying LLM prompt/output hooks, but neither fully replaces the ContextEngine cadence/product decision raised here.

## What This Changes

The issue requests a supported ContextEngine or Plugin SDK hook that can inspect and rewrite full model context before each LLM call in a tool loop.

## Best Possible Solution

Make a maintainer-owned API decision for a backward-compatible per-model-call context mutation contract, with documented security gates and parity across embedded and Codex harness paths.

## Reproduction Assessment

Yes at source level: current main shows the ordinary `assemble` call before `activeSession.prompt(...)`, with only the `ownsCompaction` loop hook reassembling during tool-loop iterations. I did not run a live multi-call LLM scenario because this is a feature/API contract review rather than a broken documented behavior test.

## Solution Assessment

Unclear as an implementation direction: the issue identifies a real remaining API question, but maintainers need to choose between changing ContextEngine cadence, adding a new full-context hook, or coordinating with modifying LLM hooks.
