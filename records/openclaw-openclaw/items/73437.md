## Summary

Keep open. Current main still leaves the reported gap: attempt-level context assembly runs before the Pi agentic loop, loop-time `ContextEngine.assemble()` is limited to `ownsCompaction: true` engines, and the public plugin hook catalog still has no full-message modifying `transform_context` surface.

## What This Changes

Request that context assembly, or a new `transform_context`-style hook, run before each model call inside an agentic tool loop instead of only once during prompt setup.

## Best Possible Solution

Define a stable, documented per-model-call full-message context rewrite contract, preferably additive, with explicit ordering against `transformContext`, context-engine assembly, budget enforcement, and plugin policy gates.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to follow a non-owning ContextEngine or plugin from attempt setup into `activeSession.prompt()`: setup-time `assemble` runs once, and no public modifying full-message hook is available for later model calls.

## Solution Assessment

Unclear. An additive hook like PR #57585 appears safer than changing `assemble` timing globally, but maintainers should decide the API shape and trust model before implementation.
