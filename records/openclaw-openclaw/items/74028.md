## Summary

Keep open. The reported CLI override bug is real and the PR is aimed at the right path, but the branch currently treats every slash-containing model ref as a direct provider/model override before the existing alias resolver can run, which regresses slash-form aliases; it also still needs the required changelog entry.

## What This Changes

The PR routes provider-qualified local `capability model run --model provider/model` requests through direct simple-completion preparation, preserves gateway override forwarding, and adds CLI unit tests for local and gateway model overrides.

## Best Possible Solution

Keep the direct local preparation path for true fully qualified provider/model refs, but first preserve configured alias matches on the existing agent-aware path; add regression coverage for a slash-form alias, add the required Unreleased/Fixes changelog entry, then merge so #73921 closes normally.

## Reproduction Assessment

Yes. #73921 gives concrete `openclaw infer model run --model github-copilot/...` commands, current main source shows the local path still routes `params.model` through the agent-aware helper, and the PR regression can be reproduced with a configured slash-form alias test.

## Solution Assessment

No as submitted. Direct preparation is the right fix for true provider-qualified refs, but the helper should not bypass the established alias-first resolver for slash-form aliases; checking the alias index before the direct path is the safer narrow repair.
