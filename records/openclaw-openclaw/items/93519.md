## Summary

Keep open: the PR addresses a real config diagnostics mismatch, but it is not merge-ready because the display-path conversion rewrites digit-only record keys as row numbers and the contributor proof is still test-only.

## What This Changes

The PR adds an optional `displayPath` to config validation issues and makes CLI, Gateway, and Control UI human-facing config issue renderers prefer one-based row labels while keeping raw `path` values.

## Best Possible Solution

Ship a display-only fix that preserves raw `path` and computes `displayPath` only for schema-confirmed array index positions across CLI, Gateway, and Control UI diagnostics, with real CLI or Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows current main renders Control UI array rows as one-based labels while config validation paths and current issue rendering expose raw zero-based numeric segments.

## Solution Assessment

No. Keeping raw `path` plus a human display path is the right direction, but this implementation is not the best fix until it distinguishes real array indexes from numeric record keys and is proven in a real CLI or Control UI flow.
