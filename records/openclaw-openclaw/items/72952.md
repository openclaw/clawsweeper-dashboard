## What This Changes

Make the documented Docker direct-entry commands use the same lean startup behavior as supported wrappers, or add equivalent direct-entry route/help laziness, with startup benchmark coverage for root help, `channels status`, `channels status --probe`, `matrix --help`, and `matrix verify status`.
