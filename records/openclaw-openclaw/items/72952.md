Keep open. Current main has partial CLI startup fast paths, but the official Docker `node dist/index.js` surface is still documented and Matrix direct-help/verify paths still fall through plugin CLI registration and heavy Matrix imports without benchmark coverage for the reported commands.

## What This Changes

The issue asks OpenClaw to remove 24s-49s startup latency from official-Docker `node dist/index.js` help, channel status, and Matrix verification commands.

## Best Possible Solution

Keep direct Docker CLI execution within a clear startup budget by extending route-first/precomputed-help coverage and Matrix CLI laziness, then lock the reported Docker `dist/index.js` cases into the startup benchmark.
