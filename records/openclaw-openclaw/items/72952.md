The issue requests reducing the 24s-49s startup latency observed when running `node dist/index.js` subcommands inside the official Docker image.

Make the Docker-facing direct-file CLI path share the same lean startup behavior as the wrapper path, or switch official Docker entrypoints/docs/scripts to the wrapper consistently while preserving package-root library exports.
