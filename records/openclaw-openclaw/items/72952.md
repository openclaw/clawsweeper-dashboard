## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72952. The observable bug is that direct `node dist/index.js` commands in the official Docker image take about 24s-49s for `--help`, `channels status --json`, `channels status --probe --json`, `matrix --help`, and `matrix verify status --json`, while `node -e` is fast. Keep the fix inside the CLI/Docker startup boundary and Matrix-owned CLI code; do not move Matrix behavior into core except for generic route/metadata seams that third-party plugins can use. Acceptable approaches include switching first-party Docker/manual helper surfaces to the packaged `openclaw.mjs` wrapper when packaging supports it, or making direct `dist/index.js` use the same lean route-first/bootstrap behavior. Add startup-benchmark coverage for `dist/index.js` and the reported channel/Matrix cases, plus a user-facing `CHANGELOG.md` fix entry. Preserve package exports unless a maintainer explicitly approves changing the public root export.
