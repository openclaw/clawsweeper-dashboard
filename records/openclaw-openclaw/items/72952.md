## Summary

Keep this open: current main has clear CLI-startup improvements for root help and `channels status`, but the Docker-image latency has not been proven fixed and Matrix plugin CLI commands can still take a heavier plugin registration path. The reporter also reconfirmed the problem on June 1, 2026, so this is not a stale cleanup candidate.

## What This Changes

The issue reports 24s-49s startup latency for `node dist/index.js` CLI subcommands inside the official Docker image while a trivial `node -e` command in the same container is fast.

## Best Possible Solution

Profile the current official Docker image on the reported command set, then keep lightweight help/status paths on metadata-only routes and move any remaining Matrix/plugin runtime imports behind command actions.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not yet: the issue has concrete Docker commands and timings, and current source identifies the relevant startup paths, but this review did not reproduce the 24s-49s latency against current main or the latest image.

## Solution Assessment

Is this the best way to solve the issue? Unclear until profiling isolates the slow import or initialization step; the likely best fix is a narrow startup-path refactor rather than adding flags or configuration.
