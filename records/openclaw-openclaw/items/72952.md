## Summary

Keep open. Current main still exposes `node dist/index.js` as an official Docker path, recent startup/runtime-deps work is only a partial mitigation, and the checked-in startup benchmark still does not cover the reported Docker direct-file commands, especially the Matrix plugin CLI paths.

## What This Changes

The issue reports 24s-49s startup latency for `node dist/index.js` Docker subcommands and asks that official Docker CLI startup avoid tens-of-seconds initialization cost.

## Best Possible Solution

Make the official Docker direct-file workflow share the lean startup guarantees of the wrapper path, or consistently move Docker-facing docs/Compose commands to `openclaw.mjs` while preserving package-root library exports. Add explicit `dist/index.js` startup budget coverage for the reported help, channels status, and Matrix commands so future releases catch regressions.
