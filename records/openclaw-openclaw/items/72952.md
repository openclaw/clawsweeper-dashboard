Keep open. Current main still exposes the official Docker `node dist/index.js` path, while Matrix CLI help/status still falls through plugin CLI registration and heavy Matrix CLI imports; the existing route-first work only partially covers the reported commands.

The issue reports 24s-49s startup latency for `node dist/index.js` help, channel status, and Matrix verification commands inside the official Docker image.

Keep Docker-facing direct CLI execution within a small startup budget by extending the lean route/help path to the Matrix commands and adding startup benchmark coverage for the reported Docker cases.
