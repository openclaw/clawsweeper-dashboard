## Summary

Keep open. Current main serializes Slack sends but still lacks the requested configurable minimum delay for timestamp ordering, and the related broad cross-channel throttle PR was closed unmerged.

## What This Changes

Add a configurable Slack outbound send throttle so rapid sequential Slack messages get distinct timestamps and preserve display order.

## Best Possible Solution

Add a Slack-owned minimum inter-send delay at every outbound post and response_url follow-up surface, with schema/types/UI hints/docs/changelog and targeted Slack regression coverage.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends but does not enforce a configurable delay between Slack posts. I did not live-test Slack same-second display ordering in this read-only review.

## Solution Assessment

Yes. A focused Slack plugin delay is narrower than the closed broad cross-channel throttle PR and matches the extension-owned send/config surfaces if it covers normal sends, blocks, DMs, threads, chunks, media follow-ups, and slash follow-ups.
