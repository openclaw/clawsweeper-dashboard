## Summary

Keep open. Current main serializes Slack sends per target, but it still does not expose or enforce a configurable minimum inter-send delay; the only related implementation PR was closed unmerged and was broader than the focused Slack plugin change requested here.

## What This Changes

The issue asks for a Slack-specific configurable outbound message throttle so rapid channel, thread, and DM sends receive distinct Slack timestamps.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay that covers normal sends, block sends, DMs, threads, chunks, media follow-ups, and slash response_url follow-ups, with schema/types/UI hints/docs and focused regression coverage.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends per target but has no config field or sleep between Slack posts, and slash follow-ups still loop without spacing. I did not reproduce Slack's same-second display ordering in a live workspace.

## Solution Assessment

Yes directionally, pending maintainer approval of the config surface and default delay. A focused Slack plugin throttle is narrower and cleaner than the closed broad cross-channel PR if it covers every Slack outbound post surface.
