## Summary

Keep open. Current main still serializes same-target Slack sends without a configurable minimum inter-send delay, Slack slash follow-ups still send in a tight loop, and the related broad throttle PR was closed unmerged.

## What This Changes

The issue requests a Slack-specific configurable outbound send throttle to preserve display ordering for rapid channel, thread, and DM messages.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay covering normal sends, DMs, threads, blocks, text chunks, media follow-ups, and slash response_url follow-ups, with schema/types/UI hints/docs and focused regression tests.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends per target but has no configurable delay between post calls, and slash follow-ups still loop without spacing. I did not establish a live Slack workspace UI repro for same-second display ordering.

## Solution Assessment

Yes directionally, but the exact Slack config key, default delay, and latency tradeoff need maintainer product judgment. A focused Slack plugin change is narrower than reviving the closed broad cross-channel throttle PR.
