## Summary

Keep open. Current main serializes Slack sends per target, but it still lacks the requested configurable minimum inter-send delay for Slack timestamp ordering, and the related broad cross-channel throttle PR is closed unmerged.

## What This Changes

Request a Slack-specific configurable outbound send throttle to force distinct timestamps for rapid sequential Slack messages.

## Best Possible Solution

Add a Slack-owned minimum inter-send delay that covers every outbound Slack post and response_url follow-up surface, with schema/types/UI hints/docs/changelog and focused Slack regression coverage.

## Solution Assessment
