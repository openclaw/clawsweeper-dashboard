## Summary

Keep open. Current main still serializes Slack sends per target but does not provide the requested configurable minimum inter-send delay, and the related broad outbound-throttle PR is closed unmerged.

## What This Changes

The issue requests a Slack-specific configurable outbound delay so rapid sequential Slack messages receive distinct timestamps and display in order.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay that covers normal sends, blocks, DMs, threads, chunks, media follow-ups, and slash response_url follow-ups, with schema/types/UI hints/docs/changelog and focused regression coverage.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends but does not enforce a configurable delay between Slack posts. I did not live-test Slack's same-second display ordering in this read-only review.

## Solution Assessment

Yes for the requested direction, pending maintainer approval of the config surface and default. A focused Slack plugin throttle is narrower than the closed broad cross-channel PR and matches the owner boundary if every Slack post surface is covered.
