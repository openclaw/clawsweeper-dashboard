## Summary

Keep open. Current main and the latest shipped release still serialize Slack sends per target without a configurable minimum inter-send delay, and the closed broad throttle PR did not land the remaining Slack-specific timestamp-ordering behavior.

## What This Changes

The issue requests a configurable Slack outbound send throttle so rapid sequential channel, thread, and DM messages receive distinct timestamps and display in order.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay covering normal sends, DMs, threads, chunks, media follow-ups, block replies, and slash response_url follow-ups, with schema/types/UI hints/docs and focused regression tests.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends per target but has no config field or delay between post calls, and slash follow-ups still loop without spacing. I did not establish a live Slack workspace UI repro for same-second display ordering.

## Solution Assessment

Yes directionally, but the exact Slack config key, default delay, and latency tradeoff need maintainer product judgment. A focused Slack plugin change is narrower than reviving the closed broad cross-channel throttle PR.
