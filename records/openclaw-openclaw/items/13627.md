## Summary

Keep open. Current main only serializes Slack sends per target; it still has no configurable Slack minimum inter-send delay, and the related broad cross-channel throttle PR is closed unmerged.

## What This Changes

Request a configurable Slack-specific outbound send throttle so rapid sequential Slack messages get distinct timestamps and keep display order stable.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay that covers normal sends, blocks, DMs, threads, chunks, media follow-ups, and slash response_url follow-ups, with schema/types/UI hints/docs and focused regression coverage.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends per target but has no config field or sleep between Slack posts, and slash follow-ups still loop without spacing. I did not live-test Slack's same-second display ordering.

## Solution Assessment

Yes directionally, pending maintainer approval of the config surface and default. A focused Slack plugin throttle is narrower than the closed broad cross-channel PR and matches the plugin ownership boundary if every Slack post surface is covered.
