## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still has no configurable minimum inter-send delay, and the related broad cross-channel throttle PR was closed unmerged.

## What This Changes

Request to add a Slack-specific configurable outbound send throttle so rapid sequential Slack posts receive distinct timestamps and preserve display order.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay at the outbound post/follow-up layer, with schema, types, UI hints, docs, changelog, and regression coverage for normal sends, DMs, thread replies, block replies, chunked media/text, and slash response_url follow-ups.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends but does not enforce any minimum time between outbound posts. The live Slack same-second display ordering behavior was not rerun in this read-only review.

## Solution Assessment

Yes. A Slack-owned configurable delay is narrower than the closed broad cross-channel rate-limit PR and fits the existing Slack send/config surfaces, provided it covers all Slack post and follow-up paths.
