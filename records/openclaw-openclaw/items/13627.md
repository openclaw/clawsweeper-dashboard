## Summary

Keep open. Current main and the latest release still serialize Slack sends but do not expose or apply the requested configurable minimum inter-send delay, and the only linked implementation PR was closed unmerged.

## What This Changes

Request a Slack-specific configurable outbound send throttle so rapid channel, thread, and DM messages receive distinct Slack timestamps and preserve display order.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay covering normal sends, DMs, threads, text chunks, media follow-ups, block replies, and slash response_url follow-ups, with schema/types/UI hints/docs and focused regression tests.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends per target but has no configurable delay between post calls, and slash follow-ups still loop without spacing. I did not establish a live Slack workspace UI repro for same-second display ordering.

## Solution Assessment

Yes directionally, but the exact Slack config key, default delay, and latency tradeoff need maintainer product judgment. A focused Slack plugin change is narrower than reviving the closed broad cross-channel throttle PR.
