## Summary

Keep open. Current main and v2026.5.22 serialize same-target Slack sends but still do not provide the requested Slack-scoped configurable inter-send delay, and the linked broad throttle PR is closed unmerged.

## What This Changes

Add a Slack-specific configurable outbound send throttle to space sequential Slack messages across channel, thread, and DM sends.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay, expose it through schema/types/UI hints/docs, and apply it consistently to all burstable Slack outbound post surfaces with focused tests.

## Reproduction Assessment

Yes for the source-level gap: current main queues same-target Slack sends but has no configurable delay between post calls, and slash follow-ups still loop without spacing. I did not run a live Slack workspace UI reproduction for same-second display ordering.

## Solution Assessment

Yes directionally, but the exact Slack config key, default delay, and latency tradeoff need maintainer product judgment. A focused Slack plugin change is narrower than reviving the closed broad cross-channel throttle PR.
