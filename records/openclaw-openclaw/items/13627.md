## Summary

Keep open. Current main only serializes Slack sends; it still lacks the requested configurable minimum inter-send delay, and the linked broad rate-limit PR was closed unmerged.

## What This Changes

Request to add a Slack-specific configurable outbound send throttle so rapid Slack messages have distinct timestamps and preserve display order.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay covering normal sends, blocks, DMs, threads, chunks, media follow-ups, and slash response_url follow-ups, with schema/types/UI hints/docs and focused regression coverage.

## Reproduction Assessment

Yes for the source-level gap: current main queues Slack sends per target but has no config field or sleep between Slack posts, and slash follow-ups still loop without spacing. I did not reproduce Slack's same-second display ordering in a live workspace.

## Solution Assessment

Yes directionally, pending maintainer approval of the config surface and default delay. A focused Slack plugin throttle is narrower than the closed broad cross-channel PR if it covers every Slack outbound post surface.
