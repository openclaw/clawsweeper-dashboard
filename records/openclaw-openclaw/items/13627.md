## Summary

Keep open: current main and v2026.5.22 still serialize same-target Slack sends without the requested configurable minimum inter-send delay, and the only linked implementation PR is closed unmerged.

## What This Changes

Request a Slack-scoped configurable outbound send throttle to keep rapid channel, thread, and DM replies visually ordered in Slack.

## Best Possible Solution

Add a Slack-plugin-owned configurable minimum inter-send delay and apply it consistently to normal sends, DMs, thread replies, block replies, chunked text, media-caption follow-ups, and slash response_url follow-ups.

## Reproduction Assessment

Yes for the source-level gap: current main queues same-target Slack sends but has no configurable delay between post calls, and slash follow-ups still loop without spacing. I did not run a live Slack workspace UI reproduction for same-second display ordering.

## Solution Assessment

Yes directionally, but the exact Slack config key, default delay, and latency tradeoff need maintainer product judgment. A focused Slack plugin change is narrower than reviving the closed broad cross-channel throttle PR.
