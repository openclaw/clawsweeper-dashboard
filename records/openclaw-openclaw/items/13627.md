## Summary

Keep open. Current main and the latest shipped release still serialize same-target Slack sends without a Slack-scoped configurable inter-send delay, and the only related implementation PR was closed unmerged.

## What This Changes

Request a configurable Slack outbound send throttle so rapid channel messages, thread replies, and DMs get distinct timestamps and preserve visible ordering.

## Best Possible Solution

Add a Slack-owned configurable minimum inter-send delay, expose it through schema/types/UI hints/docs, and apply it consistently to all burstable Slack outbound post surfaces with focused tests.

## Reproduction Assessment

Yes for the source-level gap: current main queues same-target Slack sends but has no configurable delay between post calls, and slash follow-ups still loop without spacing. I did not establish a live Slack workspace UI reproduction for same-second display ordering.

## Solution Assessment

Yes directionally, but the exact Slack config key, default delay, and latency tradeoff need maintainer product judgment. A focused Slack plugin change is narrower than reviving the closed broad cross-channel throttle PR.
