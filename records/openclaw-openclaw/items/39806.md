## Summary

Keep open. The linked Debug Snapshots bug still exists on current main and this PR targets the right UI surface, but the patch is not merge-ready because the offline warning is inferred from empty data instead of real connection state, the new copy bypasses Control UI i18n, and external real behavior proof is missing.

## What This Changes

This PR adds loading and disconnected callouts to the Control UI Debug tab Snapshots card in `ui/src/ui/views/debug.ts`.

## Best Possible Solution

Land a refreshed Debug Snapshots fix that passes real connection state into the view, renders localized inline loading/offline callouts without disturbing the existing snapshot sections, adds focused view coverage, and includes redacted real-behavior proof.

## Reproduction Assessment

Yes, by source inspection on current main: disconnected refresh returns before setting visible state, and the Snapshots card body has no inline loading or disconnected branch. I did not run a live browser session.

## Solution Assessment

No. The PR changes the right component, but the disconnected callout should be driven by actual connection state and the new strings should go through the Control UI locale source of truth.

## Review Findings
