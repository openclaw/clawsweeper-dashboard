Keep open. The linked Debug Snapshots bug is still present on current main, and this PR targets the right UI surface, but the branch is conflicting, lacks contributor-supplied real behavior proof, and still has a correctness issue in how it detects the disconnected state.

## What This Changes

This PR adds loading and disconnected-warning callouts to the Control UI Debug Snapshots card in `ui/src/ui/views/debug.ts`.

## Best Possible Solution

Refresh this branch or land a narrow replacement that passes real connection state or a disabled reason into Debug, renders localized inline loading/offline feedback, preserves the snapshot sections, and adds focused render coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible: `loadDebug` returns before setting visible state when disconnected, and `renderDebug` has no card-body loading or disconnected branch.

## Solution Assessment

No. The PR targets the right component, but the offline callout should be driven by real connection state or a refresh-disabled reason, and the new UI copy should go through the Control UI locale source of truth.
