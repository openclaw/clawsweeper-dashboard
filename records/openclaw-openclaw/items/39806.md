## What This Changes

This PR adds inline loading and disconnected-warning callouts to the Control UI Debug Snapshots card in ui/src/ui/views/debug.ts.

## Best Possible Solution

Refresh this branch or land a narrow replacement that passes real connection state or a disabled reason into Debug, renders localized inline loading/offline feedback, preserves the snapshot sections, and adds focused render coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible: loadDebug returns before setting debugLoading when disconnected, renderDebug only changes the button label while loading, and renderDebug has no connection-state prop for an accurate offline message.

## Solution Assessment

No. The PR targets the right component, but the disconnected warning should be driven by real connection state or a disabled reason, and the added copy should use the Control UI locale source of truth.
