## Summary

Keep this PR open. Current main still has the linked Debug Snapshots UX gap, but the proposed patch is not merge-ready because it infers disconnected state from empty snapshot data instead of the actual gateway connection state, so it can show the wrong warning and still miss disconnected refreshes after stale data exists.

## What This Changes

The PR adds body callouts to the Control UI Debug Snapshots card while refresh is loading and when the snapshot fields are empty.

## Best Possible Solution

Update this PR or replace it with an equivalent focused fix that passes real connection state or a refresh-disabled reason into the Debug view, renders a body-level loading state while `debugLoading` is true, renders an offline callout only when the gateway is actually disconnected, preserves the Status/Health/Last heartbeat layout, routes new copy through Control UI i18n, and adds focused Debug view/controller coverage before closing #39777.

## Reproduction Assessment

Yes. The source-level path is clear: on current main, `loadDebug` returns immediately when disconnected, and `renderDebug` has no body-level loading/offline branch beyond the Refresh button label.

## Solution Assessment

No. The PR addresses the right surface, but the disconnected warning should be driven by actual gateway connection state from `app-render.ts`, not by whether snapshot objects happen to be empty.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use actual connection state for the offline callout:** `ui/src/ui/views/debug.ts:51-52`
  - body: The new warning branch checks only whether `status`, `health`, and `heartbeat` are empty. That can show “Not connected to gateway” while the gateway is connected but no snapshot has landed or an RPC failed, and it can hide the warning after a disconnect if stale snapshot data remains. Pass the real connection state or a disabled reason into `renderDebug` and branch on that instead.
  - confidence: 0.89
- **[P3] Route the new Debug copy through localization:** `ui/src/ui/views/debug.ts:50-52`
  - body: Current main routes Debug view strings through `t(...)`, but the new callouts add English literals directly in the template. Rebase this onto the current localized Debug view and add locale keys for the new loading and disconnected messages.
  - confidence: 0.82
