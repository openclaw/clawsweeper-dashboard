## What This Changes

This PR updates the Control UI Debug Snapshots card to render in-card loading and disconnected warning callouts during refresh/no-data states.

## Best Possible Solution

Land a small UI fix that keeps the existing stack layout, shows localized in-card loading/disconnected states based on actual connection state, adds focused render coverage, and includes a changelog entry.

## Reproduction Assessment

Yes, by source inspection. `loadDebug` returns early when disconnected, and current `renderDebug` has no Snapshots card-body loading or disconnected callout beyond the button label.

## Solution Assessment

No. The direction is right, but the branch should pass actual connection state into the debug view and localize the new callout strings before it is the narrowest maintainable fix.
