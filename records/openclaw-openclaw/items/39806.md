## Summary

Keep open. Current main still lacks inline loading/offline feedback in the Debug Snapshots card, and this PR remains a plausible implementation path, but it is conflicting, lacks inspectable real behavior proof, and still needs correctness fixes before merge.

## What This Changes

The PR adds loading and disconnected-warning callouts to the Control UI Debug Snapshots card in `ui/src/ui/views/debug.ts`.

## Best Possible Solution

Refresh or replace this with a narrow Control UI Debug fix that passes real connection state into the view, renders localized inline loading/offline feedback, preserves the snapshot sections, and adds focused render coverage.

## Reproduction Assessment

Yes, by source inspection. On current main, opening the Debug tab disconnected or refreshing during slow snapshot loading leaves the card body without inline disconnected/loading feedback while `loadDebug` can return before setting visible state.

## Solution Assessment

No. The PR targets the right component, but the disconnected callout should be driven by real connection state or a refresh-disabled reason, and the new UI copy should go through the Control UI locale source of truth.
