## Summary

Keep open: current main still lacks the requested body-level loading/offline feedback, and this PR is a useful but currently incorrect implementation because it infers connection state from empty snapshot data and bypasses current Debug localization.

## What This Changes

The PR modifies `ui/src/ui/views/debug.ts` to render loading and disconnected callouts in the Control UI Debug Snapshots card.

## Best Possible Solution

Land a focused Control UI fix that passes real connection or refresh-disabled state into Debug rendering, localizes the new copy, preserves the snapshot layout, and covers loading/disconnected states with tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `loadDebug` returns before changing visible state when disconnected, and `renderDebug` only changes the button label while loading.

## Solution Assessment

No. The PR targets the right card, but empty snapshot fields are not a reliable offline signal; the maintainable fix is to pass actual connection or disabled state and route the new messages through localization.
