## What This Changes

The issue requests that Telegram forum-topic visible replies sent through `message(action=send)` either receive the configured `/usage` footer or that the docs clearly state this path is delivered without the footer.

## Best Possible Solution

Land the focused docs clarification in https://github.com/openclaw/openclaw/pull/83082, and only pursue runtime footer delivery later with explicit usage-footer metadata and live Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: enable `messages.groupChat.visibleReplies: "message_tool"`, send visibly through `message(action=send)`, and that outbound path does not receive the final-reply `appendUsageLine` post-processing.

## Solution Assessment
