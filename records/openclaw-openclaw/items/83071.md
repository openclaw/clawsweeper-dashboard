## Summary

Keep open: current main still appends usage footers only to final assistant reply payloads, while visible `message(action="send")` output is built and dispatched through the outbound message-tool path without the same footer stage.

## What This Changes

The issue requests usage footers, or clear documentation, for Telegram forum-topic visible replies sent via `message(action="send")` when `messages.groupChat.visibleReplies` is `"message_tool"`.

## Best Possible Solution

Choose the contract first, then either route usage-footer composition through the visible message-tool reply path with Telegram/forum-topic regression coverage, or update public docs to state that `/usage` footers apply only to normal final replies.

## Reproduction Assessment

Yes, source-reproducible: current main appends usage only to final reply payloads, while `message(action="send")` builds and dispatches raw outbound payloads. I did not run a live Telegram forum-topic scenario in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. Applying footers to message-tool sends is more useful for group users but needs late-usage delivery semantics; documenting final-reply-only behavior is narrower but leaves the visible metadata gap.
