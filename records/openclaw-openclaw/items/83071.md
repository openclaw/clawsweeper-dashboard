## What This Changes

The issue asks whether Telegram forum-topic visible replies sent through the `message` tool should receive the configured usage footer, or whether docs should state that footers only apply to normal final replies.

## Best Possible Solution

Land the focused docs clarification, or implement a metadata-backed runtime footer delivery path if maintainers choose that product behavior; close this issue only after the chosen linked PR merges.

Yes. Source inspection gives a high-confidence path: enable `messages.groupChat.visibleReplies: "message_tool"`, send visibly through `message(action=send)`, and that outbound path never receives final-reply `appendUsageLine` processing.

## Solution Assessment

Unclear as a product direction. The docs-only PR is the narrowest contract fix, while runtime delivery should use explicit footer metadata rather than scraping suppressed private text.
