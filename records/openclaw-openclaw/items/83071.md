## Summary

Keep open. Current main still does not append usage footers to configured Telegram/message-tool visible replies; the usage docs still overpromise the footer on every reply while the runtime keeps `message_tool_only` final payloads private, and both linked fix PRs are closed unmerged.

## What This Changes

The issue asks OpenClaw either to append the usage footer to Telegram forum-topic replies sent through `message(action=send)` under `messages.groupChat.visibleReplies = "message_tool"`, or to document that those sends are delivered as authored.

## Best Possible Solution

Maintainers should choose one contract: clarify public docs that message-tool sends are as-authored, or add metadata-backed footer delivery for current-source message-tool sends without publishing suppressed private final text.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable `messages.groupChat.visibleReplies: "message_tool"`, send the visible reply through the current-source `message` tool in a configured Telegram topic, and the usage footer remains on final payload processing rather than the outbound tool send.

## Solution Assessment

Unclear. Docs-only clarification is the safest narrow fix if message-tool sends should stay as-authored; runtime footer delivery should be metadata-backed and explicit if maintainers want the footer to appear in that path.
