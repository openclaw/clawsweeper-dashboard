## Summary

Keep open. Current main still has the documented expectation gap: the usage footer is appended on the normal final-reply path, while explicit `message(action=send)` output follows the outbound message-tool path with no footer hook; the issue also has two open closing PRs that should be reviewed or resolved before closing.

## What This Changes

The issue asks OpenClaw to align usage-footer behavior or docs for Telegram forum-topic visible replies sent through the message tool under `messages.groupChat.visibleReplies: "message_tool"`.

## Best Possible Solution

Land the focused docs clarification, or implement a metadata-backed runtime footer delivery path if maintainers choose that product behavior; close this issue only after the chosen linked PR merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable `messages.groupChat.visibleReplies: "message_tool"`, have the agent send visibly through `message(action=send)`, and that outbound path never receives final-reply `appendUsageLine` processing.

## Solution Assessment

Unclear for the runtime path, but the docs-only path is currently the safest narrow fix. If maintainers want runtime delivery, it should carry explicit usage-footer metadata rather than scrape suppressed private text.
