## Summary

Keep open. Current main still appends the usage footer only to normal final reply payloads after the agent run, while explicit `message(action=send)` output goes through the message-tool outbound path without usage-footer post-processing; the public usage docs still broadly promise a footer on every reply, so a narrow docs clarification is warranted.

## What This Changes

Clarify or change how usage footers behave for Telegram forum-topic group replies that must be sent visibly through `message(action=send)`.

## Best Possible Solution

Clarify public usage docs that `/usage` footers are appended to normal final source replies, while explicit message-tool sends are delivered as authored; treat post-turn annotation of message-tool sends as a separate runtime feature if maintainers want it later.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: enable `messages.groupChat.visibleReplies: "message_tool"`, have the agent send visibly through `message(action=send)`, and that outbound path never receives the final-reply `appendUsageLine` processing. I did not live-test Telegram delivery.

## Solution Assessment

Yes for a docs clarification: it is the narrowest maintainable fix given the current strict message-tool-only design. Runtime footer injection after an explicit tool send would be a separate behavior change, likely requiring edit/separate-message semantics.
