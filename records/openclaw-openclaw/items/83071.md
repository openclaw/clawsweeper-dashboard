## What This Changes

The issue asks OpenClaw to either append `/usage` footers to Telegram forum-topic visible replies sent via the message tool or clarify that message-tool sends do not receive final-reply footer post-processing.

## Best Possible Solution

Review and land the focused docs clarification, or choose a metadata-backed runtime footer delivery design; close this issue only after the chosen linked PR merges or the linked PRs are explicitly abandoned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `messages.groupChat.visibleReplies: "message_tool"`, visible output goes through `message(action=send)`, while `appendUsageLine` only modifies final reply payloads that are suppressed in message-tool-only mode.

## Solution Assessment

Unclear for a runtime change. A docs-only clarification is the narrowest current-policy fix, while any runtime delivery should carry explicit usage-footer provenance instead of scraping suppressed final text.
