## Summary

Keep this issue open. ## What This Changes

The issue reports an ARM64 packaged-install regression where image-capable models are rejected as text-only and image optimization reports a persistent failure even when `sharp` appears available.

## Best Possible Solution

Keep this issue open until maintainers reproduce the two paths independently on current main: image-capable configured models should resolve to an image-capable media-understanding provider for the reported provider/model chain, and packaged ARM installs should stage/load `media-understanding-core` native image dependencies or report the exact staging/load failure. Then land the narrow owner-side routing or runtime-dependency fix with regression coverage.
