## What This Changes

The issue reports image-capable models being rejected and image optimization failing on ARM64 npm-global installs despite configured `input: ["text", "image"]` and local sharp availability.

## Best Possible Solution

Reproduce both failure modes on a current package: configured vision models should resolve to an image-capable media-understanding provider, and packaged ARM64 installs should either load media-understanding-core's sharp dependency or report the exact dependency failure.
