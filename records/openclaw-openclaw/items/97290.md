## Summary

Keep open: current main and the latest release still leave default GitHub Copilot onboarding without a seeded fallback candidate, and this issue has both a protected `maintainer` label and an open linked fix PR that needs maintainer review before resolution.

## What This Changes

The issue asks for default GitHub Copilot setup to provide a same-provider fallback model candidate so idle-timeout failover can try another Copilot model instead of surfacing the timeout immediately.

## Best Possible Solution

Review the linked GitHub Copilot fallback-seeding PR and close this issue only after maintainers either land that provider-owned default or choose a different documented fallback policy.

## Reproduction Assessment

Yes for the missing fallback setup path: current source and tests show Copilot onboarding writes no fallback candidate when none is configured, and embedded failover only switches models when configured fallbacks exist. I did not rerun a live Copilot idle-timeout reproduction against current main.

## Solution Assessment

Unclear until maintainer review: provider-owned fallback seeding is a narrow plausible fix, but the exact default fallback chain changes setup behavior and needs an explicit maintainer decision through the linked PR.
