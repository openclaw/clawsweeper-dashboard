## Summary

Keep open: current main and the latest release still do not seed a default GitHub Copilot model fallback, and this issue has both a protected `maintainer` label and an open closing PR that should be reviewed before the issue is resolved.

## What This Changes

The issue requests GitHub Copilot setup to provide a same-provider default fallback model so idle-timeout failover has a candidate when users did not configure fallbacks themselves.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/97291 as the candidate fix, and land a provider-owned Copilot fallback default only if it preserves explicit user fallback choices and has focused onboarding plus embedded-run fallback detection proof.

## Reproduction Assessment

Yes for the missing fallback setup path: current source and tests show Copilot onboarding writes no fallback candidate when none is configured, and embedded failover only switches models when configured fallbacks exist. I did not run a live Copilot idle-timeout reproduction against current main.

## Solution Assessment

Unclear until the paired PR is reviewed: provider-owned fallback seeding is the right boundary, but the exact default chain is a product/defaults decision because it changes setup behavior for existing Copilot users who have not configured fallbacks.
