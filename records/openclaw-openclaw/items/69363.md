## Summary

Keep open: current main still has the Claude CLI hook session-id bug and this PR's code direction looks correct, but the external PR still lacks artifact-level real behavior proof and is currently dirty against main.

## What This Changes

The PR adds a shared helper to prefer non-hook CLI session IDs over Claude SessionStart hook IDs across JSON, buffered JSONL, and streaming JSONL parsers, plus parser regression tests.

## Best Possible Solution

Adapt the shared hook-aware session-id selector onto current main, preserve current parser behavior, keep the regression tests, attach redacted real run proof, and land through the normal agent parser checks.

## Reproduction Assessment

Yes, at source level. A Claude stream-json sequence where `hook_started` or `hook_response` carries `session_id` before `init` makes current main's buffered JSONL parser keep the transient hook ID, and that selected ID is carried into the persisted CLI session binding.

## Solution Assessment

Yes, after rebase. A shared selector that lets non-hook records win while keeping hook IDs only as fallback is the narrowest maintainable fix because the same session-id carry logic exists in all three parser paths.
