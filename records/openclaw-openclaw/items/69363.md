Keep open. ## What This Changes

The PR adds a shared CLI session-id selection helper in `src/agents/cli-output.ts` plus focused regression tests so transient Claude `SessionStart` hook IDs cannot replace real resumable conversation IDs.

## Best Possible Solution

Land or adapt the shared hook-aware session-id selection on current main so every CLI output parser prefers real non-hook conversation IDs while retaining hook IDs only as an error-only fallback, with the focused parser tests preserved and CI rerun after rebase.

## Reproduction Assessment

Yes. A deterministic JSONL stream with `hook_started`/`hook_response` carrying `transient-hook-id` before an `init` record carrying `resumed-session` reproduces the current-main `parseCliJsonl` failure path; the reversed hook-after-init case exercises the two last-wins parser paths.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix because it changes only session-id selection at the three parser sites and leaves persistence, resume invalidation, and failover policy unchanged; the safer merge path is to rebase it onto current main rather than create a second repair PR.
