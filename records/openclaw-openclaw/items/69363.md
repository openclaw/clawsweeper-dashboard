## Summary

Keep open: current main still has hook-agnostic CLI session ID selection, and this PR has matching source, field-report, reviewer, and regression-test evidence with no blocking correctness finding.

## What This Changes

The PR adds a shared CLI session-ID selector and parser tests so Claude CLI `hook_started`/`hook_response` SessionStart records remain fallback-only and do not replace real resumable conversation IDs.

## Best Possible Solution

Land or adapt the shared hook-aware selector on current main with the focused parser regression tests, then handle any stale check reruns or release-note credit during normal maintainer landing.

## Reproduction Assessment

Yes, at source level. A Claude CLI JSONL stream where `hook_started` or `hook_response` has `session_id` before `init` makes current main's `parseCliJsonl` first-wins logic select the hook ID, and the PR body reports a live two-turn Claude CLI setup showing the resulting lost context.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix because it preserves hook IDs only as a fallback while preventing hook records from replacing real non-hook conversation IDs in every parser path.
