The PR adds a hook-aware CLI session-id selector in `src/agents/cli-output.ts` plus regression tests in `src/agents/cli-output.test.ts` so transient Claude SessionStart hook IDs do not replace real conversation IDs.

## Best Possible Solution

## Solution Assessment

Yes, after rebase and proof. A shared selector that lets non-hook records win while retaining hook ids only as fallback is the narrowest maintainable fix because all three parser paths carry session ids forward.
