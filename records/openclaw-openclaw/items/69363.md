## Summary

This PR should stay open for normal review and landing because current main still stores CLI session IDs with unguarded first-wins/last-wins parser logic, while the branch provides a focused fix that matches the reported Claude CLI resume failure. I did not find a blocking code defect in the proposed patch, and the earlier changelog-only objection should not be treated as a contributor-facing review finding under current repo policy.

## What This Changes

The PR adds a shared hook-aware CLI session-id selector in `src/agents/cli-output.ts` and regression tests so Claude CLI `hook_started`/`hook_response` SessionStart IDs do not replace real resumable conversation session IDs.

## Best Possible Solution

Land or adapt the shared hook-aware session-id selector with the focused parser regression tests, then let maintainers handle any release-note credit during landing if needed.

## Reproduction Assessment

Yes. The source-level reproduction is a Claude CLI stream where `hook_started` or `hook_response` carries a transient `session_id` before `init`, causing current main's `parseCliJsonl` first-wins logic to select the wrong id; the PR also covers the reversed-order last-wins parser paths.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix because it keeps hook IDs only as a fallback while preventing hook records from replacing real non-hook conversation IDs in every parser path.
