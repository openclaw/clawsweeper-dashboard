## Summary

Keep this PR open. Current main still has the Claude CLI hook-session bug at the three parser sites, and the PR remains the active narrow implementation; the proposed parsing shape looks functionally correct, but it is missing the required user-facing changelog entry before merge.

## What This Changes

The PR adds a shared hook-aware CLI session-id selector in `src/agents/cli-output.ts` and regression tests so Claude CLI `hook_started`/`hook_response` SessionStart IDs do not replace real resumable conversation session IDs.

## Best Possible Solution

Land or adapt the shared `nextCliSessionId` helper across all three CLI output parser paths, keep the focused regression tests, and add a one-line Unreleased Fixes changelog entry crediting the contributor before merge.

## Reproduction Assessment

Yes. The deterministic reproduction is a Claude stream-json sequence where `hook_started`/`hook_response` carries a transient `session_id` before `init`; current main's `parseCliJsonl` selects that transient id, and the reversed-order case exercises the current last-wins paths in `parseCliJson` and streaming parsing.

## Solution Assessment

Yes for the code shape, with one process gap. The shared helper is the narrowest maintainable fix because it preserves hook ids only as a fallback while preventing them from replacing real non-hook conversation ids in every parser path; the PR still needs the required changelog entry.
