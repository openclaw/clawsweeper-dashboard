## Summary

Keep open: current main still has the source-level Claude CLI session-id bug, and this PR is a focused fix with no discrete correctness finding from the diff review. The remaining work is maintainer landing handling, branch refresh, and required-check validation rather than cleanup closure.

## What This Changes

The PR adds a hook-aware CLI session-id selector for JSON, buffered JSONL, and streaming JSONL parsers, plus regression tests for transient Claude SessionStart hook IDs.

## Best Possible Solution

Refresh the branch if needed and land or adapt the shared hook-aware selector with its focused parser regression tests; handle changelog and final gates during normal maintainer landing.

## Reproduction Assessment

Yes, at source level. A Claude CLI JSONL stream with `hook_started` or `hook_response` carrying `session_id` before `init` makes current main's first-wins `parseCliJsonl` path select the hook ID, and that parser-selected ID is persisted for later `--resume`.

## Solution Assessment

Yes. The shared helper is the narrowest maintainable fix because it preserves hook IDs only as fallback while preventing hook records from replacing real non-hook conversation IDs in all parser paths.
