## Summary

Keep open: the Claude CLI session continuity bug is still present on current main, and this PR is a focused source-level fix, but the branch is conflicting and the external PR still needs concrete after-fix real behavior proof before merge.

## What This Changes

The PR adds a shared CLI session-id selector that deprioritizes Claude SessionStart hook session IDs across JSON, buffered JSONL, and streaming JSONL parsers, with focused regression tests.

## Best Possible Solution

Rebase or adapt the shared hook-aware session-id selector onto current main, keep the parser regression tests, add concrete after-fix real behavior proof, and then run the normal agent parser/CLI landing checks.

## Reproduction Assessment

Yes, at source level. A Claude stream-json sequence where `hook_started` or `hook_response` carries `session_id` before `init` makes current main's buffered JSONL parser keep the transient hook id, and the selected id is persisted for future `--resume` attempts.

## Solution Assessment

Yes, after rebase. A shared selector that lets non-hook records win while keeping hook IDs only as fallback is the narrowest maintainable fix because the same session-id carry logic exists in all three parser paths.
