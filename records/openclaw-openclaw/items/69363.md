The PR adds a shared CLI session-id selector and parser regression tests so Claude `hook_started`/`hook_response` session IDs do not replace real conversation IDs across JSON, streaming JSONL, and buffered JSONL parsing.

## Best Possible Solution

## Solution Assessment

Yes, after rebase and proof. A shared selector that lets non-hook records win while retaining hook IDs only as fallback is the narrow maintainable fix because all three parser paths carry session IDs forward.
