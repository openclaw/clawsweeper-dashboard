## Summary

Keep open: current main still only supports global session maintenance thresholds plus cron-specific retention, so the requested ordered per-pattern or per-type retention rule contract remains unimplemented. The KeepMyClaw comment is useful backup context, but it does not replace native cleanup rules for operators who need to avoid scripts and gateway restarts.

## What This Changes

Request adding ordered per-pattern or per-type session maintenance retention rules so ephemeral cron/subagent sessions can prune faster than interactive sessions.

## Best Possible Solution

Define one additive core session.maintenance.rules contract with documented matcher semantics, first-match fallback to global pruneAfter, strict schema/help/docs coverage, and centralized pruning that preserves durable-session protections.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken documented behavior. Source inspection gives high confidence that current main cannot express ordered per-pattern retention rules.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the safest implementation is one central retention-rule contract with schema, docs, and tests rather than ad hoc checks in individual pruning paths.
