## Summary

Keep open. Current main still has only global session maintenance thresholds, hard-coded durable-conversation preservation, and a separate cron-run reaper; KeepMyClaw, #50755, #45210, and #58088 are useful adjacent context but do not implement ordered per-pattern retention rules.

## What This Changes

The issue requests ordered per-pattern or per-type `session.maintenance` retention rules so ephemeral sessions can prune faster than interactive sessions.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer design approval, with documented matcher semantics, ordered first-match retention, global `pruneAfter` fallback, schema/help/docs coverage, and pruning paths that preserve existing safety constraints.

## Reproduction Assessment

Not applicable as a runtime bug. The missing feature is high-confidence by inspection: the strict config schema has no `rules` field, and the resolver/pruning paths use one global `pruneAfterMs` threshold.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the narrow maintainable fix should first settle matcher shape and data-deletion semantics rather than adding ad hoc pattern checks inside pruning loops.
