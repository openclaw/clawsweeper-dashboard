## Summary

Keep open. Current main still exposes only global session maintenance thresholds plus a separate cron-run retention path; the KeepMyClaw comment is useful backup context but does not implement ordered OpenClaw retention rules.

## What This Changes

Request an ordered `session.maintenance.rules` config so cron/subagent-style sessions can expire faster than interactive sessions while unmatched entries fall back to global `pruneAfter`.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer design approval, with ordered first-match matching, a global `pruneAfter` fallback, schema/help/docs coverage, and safety tests across age, count, disk-budget, and cron cleanup paths.

## Reproduction Assessment

Not applicable as a runtime bug. The missing feature is high-confidence by inspection: the config type/schema expose no `rules` field, and pruning resolves one global `pruneAfterMs`.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the maintainable fix should settle matcher shape and destructive-retention safeguards before code is added.
