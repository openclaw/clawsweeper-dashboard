## Summary

Keep open. Current main still lacks an ordered `session.maintenance.rules` contract; existing global maintenance, durable-session protections, and cron-specific retention do not cover per-pattern or per-type retention for cron plus subagent/session families.

## What This Changes

Request support for ordered per-pattern or per-type session-maintenance retention rules with global `pruneAfter` fallback.

## Best Possible Solution

Add an additive core `session.maintenance.rules` design only after maintainer approval, with documented matcher semantics, first-match retention, global fallback, strict schema/help/docs coverage, and preservation of existing durable-session safeguards.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives high confidence that the requested feature is absent: config/schema/docs expose no `session.maintenance.rules`, and pruning resolves one global age threshold plus separate cron-only retention.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the maintainable solution should define a central public matcher and deletion contract rather than adding ad hoc pattern checks in individual pruning loops.
