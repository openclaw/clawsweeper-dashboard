## Summary

Keep open: current main and the latest release still expose only global session maintenance thresholds plus cron-specific retention, so ordered per-pattern or per-type retention rules remain unimplemented. The KeepMyClaw comment is useful backup context, but it does not replace native cleanup rules for operators who need to avoid scripts and gateway restarts.

## What This Changes

The issue requests ordered per-pattern or per-type session maintenance retention rules, with unmatched sessions falling back to the global `pruneAfter` threshold.

## Best Possible Solution

Define one additive core `session.maintenance.rules` contract after maintainer design approval, with documented matcher semantics, first-match fallback to global `pruneAfter`, strict schema/help/docs coverage, and centralized pruning that preserves durable-session protections.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request rather than broken documented behavior. Source inspection gives high confidence that current main cannot express ordered per-pattern retention rules.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the safest implementation is one central retention-rule contract with schema, docs, and tests rather than ad hoc checks in individual pruning paths.
