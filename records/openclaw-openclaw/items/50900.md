## Summary

Keep open. Current main still has global session maintenance retention plus separate cron-run retention and durable-session protections, but it does not implement ordered per-pattern or per-type `session.maintenance.rules`; the KeepMyClaw backup comment is useful context, not a replacement for a core retention policy.

## What This Changes

Request adds ordered per-pattern or per-type session maintenance retention rules so ephemeral cron/subagent sessions can prune more aggressively than interactive sessions.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer approval, with documented matcher semantics, first-match fallback to global `pruneAfter`, strict schema/help/docs coverage, and pruning paths that preserve durable-session protections.

## Reproduction Assessment

Not applicable as a runtime bug. The missing feature is high-confidence by source inspection: current types, schema, docs, resolver, and pruning code expose no ordered `session.maintenance.rules` surface.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the best implementation should define matcher and deletion semantics centrally rather than adding ad hoc pattern checks in individual pruning loops.
