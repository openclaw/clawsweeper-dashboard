## Summary

Keep this issue open. Current main still has only global session-maintenance retention plus a separate cron-run retention reaper; it does not implement ordered per-pattern or per-type `session.maintenance.rules`. The KeepMyClaw comment is useful backup-retention context, and the open per-agent override PR is adjacent, but neither replaces a core rule contract for selective `sessions.json` pruning.

## What This Changes

Request to add ordered per-pattern or per-type session maintenance retention rules with rule-specific `pruneAfter` values and fallback to the global `session.maintenance.pruneAfter`.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract only after maintainer design approval, with a documented matcher, ordered first-match retention, global `pruneAfter` fallback, strict schema/help/generated-doc coverage, and pruning call sites that preserve current active/running-session and cron-specific safeguards.

## Reproduction Assessment

Not applicable as a runtime bug, but the missing feature is high-confidence by source inspection: the strict config schema has no `rules` field and the resolver/pruning code uses one global `pruneAfterMs` threshold.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the maintainable fix should settle matcher shape and deletion semantics before adding string matching inside the pruning loop.
