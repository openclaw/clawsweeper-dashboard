## Summary

Keep open. Current main still exposes only global session maintenance retention plus cron-specific retention and durable-session protections; it does not implement ordered per-pattern or per-type `session.maintenance.rules`.

## What This Changes

Request adds ordered per-pattern or per-type retention rules under `session.maintenance` so ephemeral sessions can prune sooner than interactive sessions.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer approval, with documented matcher semantics, strict schema/help/docs coverage, global fallback behavior, and pruning paths that preserve durable-session protections.

## Reproduction Assessment

Not applicable as a runtime bug. The missing feature is high-confidence by source inspection: current config, schema, docs, and pruning code expose no ordered `session.maintenance.rules` surface.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the best implementation should define matcher and deletion semantics centrally rather than adding ad hoc pattern checks in individual pruning loops.
