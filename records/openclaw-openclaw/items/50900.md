## Summary

Keep open. Current main still lacks an ordered `session.maintenance.rules` contract; the existing hard-coded durable-session protection, cron-specific reaper, KeepMyClaw backup suggestion, and related items do not implement per-pattern or per-type retention rules.

## What This Changes

Add ordered per-pattern or per-type session maintenance retention rules that choose a `pruneAfter` threshold before falling back to the global setting.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract with documented matcher semantics, first-match retention, global fallback, strict schema/help/docs coverage, and pruning paths that preserve existing durable-session protections.

## Reproduction Assessment

Not applicable as a runtime bug. The missing feature is high-confidence by source inspection: current config/schema/docs expose no `session.maintenance.rules`, and pruning resolves one global `pruneAfterMs`.

## Solution Assessment

Unclear pending maintainer design. The requested direction is plausible, but the maintainable fix should define matcher and deletion semantics centrally rather than adding ad hoc pattern checks in pruning loops.
