## Summary

Keep open: the PR still contains source-reproducible correctness and upgrade-safety blockers, current main has not made the change obsolete, and the branch lacks required real behavior proof.

## What This Changes

The PR changes agent model fallback retries to decide when images are preserved or stripped, suppress prompt image detection on cross-provider retries, and thread partial-execution context through fallback attempts.

## Best Possible Solution

Rebase or replace this with a central retry-media decision that covers explicit and prompt-detected images, preserves or documents the chosen provider trust default, and either wires real partial-execution producers or defers replay warnings until producers exist.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the two main blockers: stripped same-provider format retries can still load prompt images, and partialExecution is consumed without production producers.

## Solution Assessment

No. The direction is useful, but the patch should centralize the retry media decision, handle compatibility and provider-trust policy explicitly, and avoid adding replay warnings until real partial-execution metadata is produced.

  - body: `resolveRetryImages` returns `undefined` for same-provider `format` retries, but the embedded runner only suppresses prompt image detection for cross-provider retries. A format retry can reload a local image referenced in the prompt and send it again, defeating the strip policy and likely repeating the same failure.
  - confidence: 0.88

  - confidence: 0.88
