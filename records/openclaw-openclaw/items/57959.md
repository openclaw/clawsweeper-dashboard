## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Suppress prompt image detection on format retries:** `src/agents/command/attempt-execution.ts:552`
  - body: `resolveRetryImages` strips explicit images when the previous failure was `format`, but this new flag only suppresses prompt-image detection for cross-provider retries. A same-provider format fallback can still discover `./image.png` from the prompt and send it to the model that already rejected images, defeating the format-error retry policy. Drive prompt-image suppression from the same strip decision, not only from provider changes.
  - confidence: 0.9
- **[P1] Populate partial execution before merging it:** `src/agents/model-fallback.ts:819-829`
  - body: The fallback loop now merges `described.partialExecution`, and `runAgentAttempt` can warn from `previousPartialExecution`, but this branch does not set `partialExecution` at any production `FailoverError` throw site. The author comment also says that producer is intentionally deferred, so the replay-avoidance prompt never appears in real fallback retries. Populate it from the runner's `toolMetas`/`didSendViaMessagingTool` metadata or remove this as a claimed fix.
  - confidence: 0.88
