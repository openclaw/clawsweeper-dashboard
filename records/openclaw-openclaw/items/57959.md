## Summary

Keep open: the PR targets a real provider media boundary, but it is not merge-ready because the branch is dirty, real behavior proof is missing, and source inspection still shows security/compatibility gaps in retry media handling plus an inert partial-execution warning path.

## What This Changes

This PR changes agent model fallback retries to make image forwarding failure-mode/provider-aware, suppress embedded prompt image detection on cross-provider retries, and thread partial-execution context into later attempts.

## Best Possible Solution

Rebase or replace this with one central provider-media trust decision that covers explicit images, prompt-detected refs, skipped-candidate fallback, docs/config/upgrade behavior, and either real partial-execution producers or no replay-warning consumer yet.

## Reproduction Assessment

Yes. Source inspection of the PR head shows concrete paths where media stripping depends on isFallbackRetry and prompt image detection is only suppressed for cross-provider retries; I did not run a live cross-provider fallback.

## Solution Assessment

No. The direction is useful, but the best fix should centralize provider media trust and compatibility policy, then wire retry media and partial-execution behavior from that contract instead of adding partial consumers and unconditional fail-closed behavior piecemeal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Guard cross-provider media after skipped primaries:** `src/agents/command/attempt-execution.ts:321`
  - body: `resolveRetryImages` returns images whenever `isFallbackRetry` is false, but `runAgentAttempt` computes that from actual run calls. If the primary provider is skipped before the callback because profiles are cooled down or disabled, the first real run can be a different provider with `isFallbackRetry=false`, so explicit images and prompt-detected refs are still sent cross-provider.
  - confidence: 0.86
- **[P1] Make provider media stripping an explicit policy:** `src/agents/command/attempt-execution.ts:329`
  - body: The PR unconditionally strips cross-provider retry images even though the linked parent asks for a provider media-trust policy and a compatibility-preserving default. Existing image-capable fallback setups can stop receiving needed media on upgrade without config, docs, migration, or proof that maintainers want this fail-closed default.
  - confidence: 0.83
- **[P2] Suppress prompt detection whenever retry media is stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` strips explicit images for same-provider `format` retries, but the embedded runner only suppresses prompt image detection on cross-provider retries. A prompt like `./chart.png` can be reloaded and resent on the retry despite the strip decision, likely repeating the same format failure.
  - confidence: 0.88
- **[P2] Populate partial execution before consuming it:** `src/agents/model-fallback.ts:818-831`
  - body: The fallback loop forwards `previousPartialExecution`, but the branch adds no production `FailoverError` throw site that sets `partialExecution`; only tests and the consumer path use it. The replay warning therefore never appears in real fallback runs, so non-idempotent tools can still replay.
  - confidence: 0.88
