## Summary

Keep open: the PR still targets a real fallback safety gap, but the current head has two source-level blockers and lacks required real behavior proof before it can merge.

## What This Changes

The PR changes agent fallback retry handling to selectively strip retry images, suppress embedded prompt image detection on cross-provider retries, and pass failure/partial-execution context into later fallback attempts.

## Best Possible Solution

Rebase a narrower fallback-safety patch that derives explicit-image stripping and prompt-image suppression from one shared retry decision, and either populate partialExecution at real failure sites or defer replay warnings until producers exist.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces both blockers: same-provider format retries strip explicit images while leaving prompt image detection enabled, and partialExecution is consumed without a production producer.

## Solution Assessment

No. The image-safety direction is useful, but this patch should align prompt-image suppression with the same strip decision and make replay warnings depend on real partial-execution producers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress prompt detection whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` returns `undefined` for same-provider `format` retries, but the embedded runner only suppresses prompt image detection for cross-provider retries. A format retry can therefore reload a local image referenced in the prompt and send it again, defeating the strip policy and likely repeating the same failure.
  - confidence: 0.88
- **[P2] Populate partial execution before warning retries:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards `previousPartialExecution`, but the diff adds no production `FailoverError` throw site that sets `partialExecution`; only tests construct it, and the author confirmed producer work is future runner work. The replay warning therefore never fires in real fallback runs.
  - confidence: 0.88
