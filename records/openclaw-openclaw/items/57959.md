## Summary

Keep open: the branch is not obsolete on current main, but it is not merge-ready because source review still shows provider-boundary and compatibility blockers, the branch is conflicting, and the PR body/comments do not include real after-fix fallback proof.

## What This Changes

The branch adds failure-aware fallback image forwarding, cross-provider prompt-image suppression, and partial-execution context threading across agent fallback retry paths.

## Best Possible Solution

Rebase or replace this with one central provider-media trust policy that covers explicit images, prompt-detected refs, skipped candidates, docs/config/upgrade behavior, and either real partial-execution producers or no replay-warning consumer yet.

## Reproduction Assessment

Yes for source-level review: the PR head shows concrete paths where media stripping depends on callback-local retry state and prompt image detection is suppressed only for cross-provider retries. I did not run a live cross-provider fallback in this read-only pass.

## Solution Assessment

No. The direction is useful, but the best fix should centralize provider media trust and compatibility policy, then wire retry media and partial-execution behavior from that contract instead of adding partial consumers and unconditional policy piecemeal.

Full review comments:

- **[P1] Guard skipped cross-provider candidates:** `src/agents/command/attempt-execution.ts:321-323`
  - body: `resolveRetryImages` forwards images whenever `isFallbackRetry` is false, but `agent-command` derives that flag from the number of run callbacks. If the configured primary is skipped before the callback because profiles are cooled down, the first real run can target a different provider with images still attached.
  - confidence: 0.88
- **[P1] Make cross-provider media trust explicit:** `src/agents/command/attempt-execution.ts:329-330`
  - body: This hard-codes cross-provider retries as untrusted even though the linked parent asks for provider-level trust configuration and a compatibility-preserving default. The PR should not close that policy issue until the chosen default, config surface, docs, and upgrade behavior are explicit.
  - confidence: 0.84
- **[P2] Suppress prompt detection when retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: Same-provider `format` retries strip explicit images through `resolveRetryImages`, but the embedded runner suppresses prompt image detection only for cross-provider retries. A prompt such as `./chart.png` can be reloaded and resent on the retry despite the strip decision.
  - confidence: 0.88
- **[P2] Populate partial execution before relying on warnings:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards `previousPartialExecution`, but the PR adds no production `FailoverError` throw site that sets `partialExecution`; only tests populate it. The replay warning therefore will not appear in real fallback runs.
  - confidence: 0.87
