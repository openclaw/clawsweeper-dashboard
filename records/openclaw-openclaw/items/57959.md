Full review comments:

- **[P1] Populate partial execution before relying on retry context:** `src/agents/model-fallback.ts:818-831`
  - body: `mergedPartialExecution` only receives `described.partialExecution`, but this branch does not add any production `FailoverError` throw/coercion that sets `partialExecution`; tests are the only producers I found. In real fallback runs `previousPartialExecution` remains unset, so the non-idempotent tool replay warning this PR claims to add never fires. Wire the producer at the embedded/CLI failure sites or drop this feature from the PR scope.
  - confidence: 0.89
- **[P2] Suppress prompt images on format retries too:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` strips explicit images for same-provider `format` failures, but `suppressPromptImageDetection` is true only for cross-provider retries. A retry prompt that references a local image will be reloaded by `detectAndLoadPromptImages`, so the fallback can resend the image that the new policy just stripped and repeat the format failure. Gate prompt-image detection whenever the retry image policy strips images, not only when providers differ.
  - confidence: 0.87
