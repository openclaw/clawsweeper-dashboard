## Summary

Keep open: the PR is still a useful fallback-safety attempt, but it is conflicting, lacks real behavior proof, and still has source-level blockers around provider media-trust policy, retry image suppression consistency, and partial-execution producers.

## What This Changes

The PR changes agent fallback retries to preserve or strip images by failure/provider context, suppress prompt image detection on cross-provider retries, and thread partial-execution metadata through model fallback attempts.

## Best Possible Solution

Rebase or replace this with one central retry-media decision that covers explicit and prompt-detected images, preserves or documents the chosen provider trust default, and either wires real partial-execution producers or defers replay warnings until producers exist.

## Reproduction Assessment

Yes for source-level review: current main and the PR diff show the fallback image and prompt-image paths clearly, and the PR head still consumes partialExecution without adding production producers. I did not run a live cross-provider fallback with real credentials.

## Solution Assessment

No. The direction is useful, but the patch should centralize the retry media decision, handle compatibility and provider-trust policy explicitly, and avoid adding replay warnings until real partial-execution metadata is produced.

  - body: The PR sets `suppressPromptImageDetection` directly from `isCrossProviderRetry`, which silently disables prompt-referenced image fallback for existing cross-provider image-capable setups.   - confidence: 0.82
- **[P2] Suppress prompt detection whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:552`
  - body: `resolveRetryImages` strips explicit images for same-provider `format` retries, but the embedded runner only suppresses prompt image detection for cross-provider retries. A format retry can reload a local image referenced in the prompt and send it again, defeating the strip policy and likely repeating the same failure.
  - confidence: 0.88
- **[P2] Populate partial execution before warning retries:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards `previousPartialExecution`, but the diff adds no production `FailoverError` throw site that sets `partialExecution`; only tests and the consumer path use it. The replay warning therefore never appears for real failed attempts, leaving the claimed non-idempotent replay protection inert.
  - confidence: 0.86
