## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Suppress prompt images after format retries too:** `src/agents/command/attempt-execution.ts:549-552`
  - body: `resolveRetryImages` strips explicit images when the prior failure reason is `format`, but this flag only suppresses prompt image detection on cross-provider retries. A same-provider format retry can therefore reload image paths from the prompt via `detectAndLoadPromptImages` and send them again, defeating the format-error retry policy. Gate prompt-image detection on the same failure-aware decision as explicit images, not only on provider changes.
  - confidence: 0.82
- **[P2] Populate partial execution before consuming it:** `src/agents/model-fallback.ts:813-823`
  - body: The fallback loop now forwards `described.partialExecution`, but this PR does not update the production embedded or CLI failover throw sites to set that field. In real runs the new warning context remains absent, so non-idempotent tools can still be replayed after a failed attempt despite the PR's partial-execution claim. Either add the producer in the runner failure path or remove this plumbing from this PR.
  - confidence: 0.8
