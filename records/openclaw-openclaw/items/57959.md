## Summary

Keep open: current main still lacks the proposed fallback image-policy and partial-execution plumbing, but the PR remains blocked by source-level correctness issues and missing external real behavior proof.

## What This Changes

This PR changes agent fallback retry handling by preserving or stripping images based on failure/provider, suppressing embedded prompt-image detection for cross-provider retries, and threading partial-execution context through model fallback code and tests.

## Best Possible Solution

Rebase a narrower fallback-safety patch that uses one shared strip/suppression decision for explicit and prompt-detected images, and only ships replay warnings once partial-execution metadata is populated at real failure sites.

## Reproduction Assessment

Yes, from source inspection rather than a live fallback run: current main still strips explicit retry images, still loads prompt-detected images without a fallback suppression flag, and lacks partial-execution fields. The PR diff also shows the same-provider format-retry prompt-image gap and the unpopulated partial-execution warning path.

## Solution Assessment

No. The image-safety direction is useful, but the patch should align prompt-image suppression with the same decision that strips explicit retry images and make partial-execution metadata real before relying on replay warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress prompt images whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` strips explicit images after a `format` failure, but `suppressPromptImageDetection` is only enabled for cross-provider retries. A same-provider format retry with a prompt-referenced local image will still load and send the image again, defeating the new strip policy and likely re-triggering the same failure.
  - confidence: 0.87
- **[P2] Populate partial execution before warning retries:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards `previousPartialExecution`, but this PR adds no production `FailoverError` construction that sets `partialExecution`; only tests set it, and the author described the throw-site work as future plumbing. The replay warning therefore never fires in real runs, so non-idempotent tool replay remains unfixed.
  - confidence: 0.88
