## Summary

Keep this PR open: current main still lacks the proposed fallback image/privacy work, but the PR head has source-reproducible correctness blockers and no real after-fix behavior proof.

## What This Changes

The PR changes agent fallback retries to make image forwarding failure-aware, suppress prompt-image detection on cross-provider retries, and thread partial-execution context through model fallback.

## Best Possible Solution

Rebase a narrower fallback-safety patch that drives explicit-image stripping and prompt-image suppression from the same decision, and only emits partial-execution warnings once real failure sites populate that state.

## Reproduction Assessment

Yes. Source inspection of the PR head shows same-provider format retries strip explicit images while leaving prompt-image detection enabled, and diff inspection shows `partialExecution` is only produced in tests.

## Solution Assessment

No. The direction is useful, but this patch should align the strip/suppress decisions and either add real partial-execution producers or defer the replay-warning path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress prompt detection whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` returns `undefined` for same-provider `format` retries, but the embedded runner only receives `suppressPromptImageDetection` for cross-provider retries. A format retry can still reload a local image referenced in the prompt and send it again, defeating the strip policy and likely repeating the same failure.
  - confidence: 0.88
- **[P2] Populate partial execution before warning retries:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards `previousPartialExecution`, but the PR adds no production `FailoverError` throw site that sets `partialExecution`; only tests construct it, and the author confirmed producer work is future runner work. The replay warning therefore never fires in real fallback runs.
  - confidence: 0.88
