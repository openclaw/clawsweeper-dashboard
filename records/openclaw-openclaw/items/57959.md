## Summary

Keep open: current main still lacks the fallback image/privacy and partial-execution plumbing, but this PR still has merge-blocking behavior gaps and no real after-fix proof. The author addressed the Greptile sanitize/dedup feedback in c39c5cc, but the later Codex findings remain valid against the current head.

## What This Changes

The PR adds failure-aware fallback image forwarding, prompt-image detection suppression, partial-execution fields through model fallback, and focused agent fallback tests.

## Best Possible Solution

Rebase and land a narrower fallback-safety patch that applies one retry-image policy to explicit and prompt-detected images, and either populates partial-execution metadata at real failure sites or removes the inactive replay-warning path.

## Reproduction Assessment

Yes from source inspection, though not from a live run: current main still strips explicit retry images, still loads prompt-detected images, and lacks partial-execution fields. The PR diff also shows the same-provider format-retry prompt-image gap and unpopulated partial-execution warning path.

## Solution Assessment

No. The image-safety direction is useful, but the patch should align prompt-image suppression with the same policy that strips retry images and make partial-execution metadata real before relying on replay warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress prompt images whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` drops explicit images when the previous failure was `format`, but `suppressPromptImageDetection` is only set for cross-provider retries. A same-provider format retry with a prompt-referenced local image will still load and send images again, defeating the format-error strip and likely re-triggering the same failure.
  - confidence: 0.87
- **[P2] Populate partial execution before warning retries:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards `previousPartialExecution`, but this PR does not add any production `FailoverError` construction that sets `partialExecution`; the author also described it as future plumbing. The replay warning therefore never fires outside tests, so non-idempotent tools can still be replayed after a failed attempt.
  - confidence: 0.88
