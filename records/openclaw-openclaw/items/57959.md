## Summary

Keep open: this PR targets a real fallback privacy gap, but it is not merge-ready because the head is conflicting, real behavior proof is missing, format retries can still reload prompt-detected images, and the partial-execution warning path has no production producer.

## What This Changes

The PR changes agent fallback retries to strip images on format or cross-provider retries, suppress prompt-image detection on cross-provider embedded retries, and thread partial-execution context through fallback attempts.

## Best Possible Solution

Rebase a current-main fallback fix that makes explicit-image stripping and prompt-image detection suppression share one retry media policy, then either wire real runner partial-execution metadata into FailoverError or split that future plumbing into separate work.

## Reproduction Assessment

Yes, at source level. The PR-head code shows format retries strip explicit images but only suppress prompt-image detection for cross-provider retries, and source search shows partialExecution is consumed but not produced by production throw sites.

## Solution Assessment

No, not yet. The image boundary belongs near runAgentAttempt and the embedded runner, but the current branch must be rebased, make the retry media policy consistent, and either complete or remove the dead partial-execution path.
