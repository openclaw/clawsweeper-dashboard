## Summary

Keep open. Current main still does not implement the PR’s fallback retry image policy, prompt-image suppression, or partial-execution threading, and the PR is currently dirty against main with one remaining correctness gap in the partial-execution path.

## What This Changes

This PR adds failure-aware retry image forwarding, cross-provider prompt-image suppression, partial-execution fields on failover errors, fallback context threading, and focused unit coverage in the agents fallback paths.

## Best Possible Solution

Rebase onto current main and land a narrower agents fallback change that enforces image and attachment trust boundaries for both explicit and prompt-detected images, while either wiring real partial-execution metadata from runner/CLI failure sites or removing that claim from the patch.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence current-main reproduction path: fallback retries still drop explicit images unconditionally, prompt-image detection runs unconditionally, and no current failover path carries partial execution metadata.

## Solution Assessment

No. The image-safety direction addresses important parts of #43481 and #43492, but this PR is not yet the best landing shape because it is unmergeable and the partial-execution protection is not populated in production.
