## Summary

Keep open: current main has not implemented the central fallback-safety work, but this PR still has source-level correctness blockers and no real behavior proof after the patch.

## What This Changes

The PR changes agent model fallback retries to preserve or strip images by failure/provider context, suppress prompt image loading on cross-provider retries, and thread partial-execution context into later attempts.

## Best Possible Solution

Rebase or replace this with a narrower fallback-safety patch that derives explicit-image stripping and prompt-image suppression from one shared retry decision, and either populates partialExecution at real failure sites or defers replay warnings until producers exist.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the two blockers: same-provider format retries strip explicit images while leaving prompt image detection enabled, and partialExecution is consumed without a production producer.

## Solution Assessment

No. The direction is useful, but the patch should align prompt-image suppression with the same strip decision and make replay warnings depend on real partial-execution producers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress prompt detection whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` returns `undefined` for same-provider `format` retries, but the embedded runner only suppresses prompt image detection for cross-provider retries. A format retry can therefore reload a local image referenced in the prompt and send it again, defeating the strip policy and likely repeating the same failure.
  - confidence: 0.88
- **[P2] Populate partial execution before warning retries:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards `previousPartialExecution`, but the diff adds no production `FailoverError` throw site that sets `partialExecution`; only tests construct it, and the author confirmed producer work is future runner work. The replay warning therefore never fires in real fallback runs.
  - confidence: 0.88
