## Summary

Keep open: the branch has useful fallback image-boundary work, but it is not merge-ready because it changes cross-provider image retry defaults without the maintainer-approved trust policy/proof requested by the canonical issue, and the partial-execution warning path is not populated by production throw sites.

## What This Changes

The PR changes agent model fallback retries to strip or preserve images based on failure/provider context, suppress prompt-image detection on cross-provider embedded retries, and thread partial-execution metadata through fallback run options.

## Best Possible Solution

Choose the cross-provider media trust policy first, then land a current-main implementation that either preserves compatibility by default with an explicit trust control or deliberately fails closed with documented upgrade proof, and either populate partialExecution from real throw sites or remove the replay-protection claim.

## Reproduction Assessment

Yes from source for the media-boundary issue: current main strips explicit retry images but still runs prompt image detection in the embedded runner, so prompt-referenced local images can be loaded on fallback. The partial-execution replay claim is only source-plausible here because this branch does not add a production producer for partialExecution.

## Solution Assessment

No. The image privacy mitigation is plausible, but the best fix needs a maintainer decision on the trust/default behavior, and the partial-execution path is incomplete unless production failover errors actually carry execution metadata.
