## Summary

Keep open: current main has not implemented the fallback image/privacy or partial-execution plumbing, but this PR is not merge-ready because it still has two functional gaps and lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds fallback retry image policy helpers, prompt-image detection suppression, partial-execution metadata threading, and focused agent fallback tests.

## Best Possible Solution

Rebase a narrower fallback-safety patch that applies one retry-image policy to explicit and prompt-detected images, and either populates partial-execution metadata at actual failure sites or removes the inactive replay-warning path.

## Reproduction Assessment

Yes from source inspection, though not from a live run: current main still strips explicit retry images, still loads prompt-detected images, and lacks partial-execution fields. The PR diff also shows the same-provider format-retry prompt-image gap and unpopulated partial-execution warning path.

## Solution Assessment

No. The image-safety direction is useful, but the patch should align prompt-image suppression with the same policy that strips retry images and make partial-execution metadata real before relying on replay warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress prompt images whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: resolveRetryImages drops explicit images on same-provider format retries, but suppressPromptImageDetection is only set for cross-provider retries. A prompt-referenced local image will still be loaded and sent on the format retry, defeating the strip and likely re-triggering the same failure.
  - confidence: 0.87
- **[P2] Populate partial execution before warning retries:** `src/agents/model-fallback.ts:817-831`
  - body: The fallback loop forwards previousPartialExecution only when a FailoverError already carries partialExecution, but this PR adds no production throw site that sets it; the author also called that future plumbing. The replay warning therefore never fires outside tests.
  - confidence: 0.88
