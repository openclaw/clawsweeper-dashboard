## Summary

Keep open: the central fallback media and partial-execution work is not implemented on current main, but this PR is not merge-ready because it lacks real behavior proof, is dirty against current main, and still has correctness and provider-trust policy blockers.

## What This Changes

The PR changes agent fallback retries to preserve or strip images by failure mode/provider, suppress prompt image detection on cross-provider retries, and thread partial-execution context through model fallback attempts.

## Best Possible Solution

Rebase or replace this with one central retry-media decision that covers explicit and prompt-detected images, documents or configures the provider trust default, and either wires real partial-execution producers or defers replay warnings until producer metadata exists.

## Reproduction Assessment

Yes. Source inspection of the PR head shows stripped same-provider format retries can still reload prompt-referenced images, and partialExecution is consumed without production producers.

## Solution Assessment

No. The direction is useful, but the patch should centralize the retry media decision, handle compatibility and provider-trust policy explicitly, and avoid adding replay warnings until real partial-execution metadata is produced.
