## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Block CLI prompt-image loading on cross-provider retries:** `src/agents/command/attempt-execution.ts:451-452`
  - body: This only strips the explicit `images` payload before calling `runCliAgent`. In the CLI runner, an absent image array falls back to `loadPromptRefImages`, so a retry prompt containing `./image.png` or `file://...` can still load and forward that local image to a different provider. Thread the same suppression decision into CLI prompt-image preparation or use a distinct stripped-media state.
  - confidence: 0.91
- **[P1] Populate partial execution before relying on it:** `src/agents/model-fallback.ts:818-831`
  - body: The fallback loop now merges `described.partialExecution`, but no production `FailoverError` constructor on this branch sets that field. The replay-warning system prompt therefore never appears for real failed attempts, so non-idempotent tools can still be replayed after fallback. Either capture tool metadata at the throw sites or remove this as a claimed behavior.
  - confidence: 0.88
