## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Suppress prompt images when format retries strip images:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` strips explicit images after a `format` failure, but the embedded runner only skips prompt-image detection when `isCrossProviderRetry` is true. A same-provider format retry can therefore reload image paths from the prompt and send the images that this policy meant to strip; gate prompt detection whenever retry images are stripped for format or provider-boundary reasons.
  - confidence: 0.88
- **[P2] Populate partial execution before threading retry warnings:** `src/agents/model-fallback.ts:72-73`
  - body: The PR threads `previousPartialExecution` and builds a replay warning, but the production code does not add any `FailoverError` throw site that sets `partialExecution`. That means the warning is never emitted in real fallback failures, so non-idempotent tool/message replay remains possible; capture executed tool metadata at the failing runner boundary or remove this claimed behavior from the patch.
  - confidence: 0.84
