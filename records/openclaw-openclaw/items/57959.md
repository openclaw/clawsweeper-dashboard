Full review comments:

- **[P1] Populate partial execution before consuming it:** `src/agents/failover-error.ts:35`
  - body: This adds partialExecution to FailoverError and threads it through fallback, but the PR does not update any production FailoverError throw site to set it. As a result previousPartialExecution stays undefined in real fallback retries, so the replay warning never appears and non-idempotent tools can still be repeated after a failed attempt.
  - confidence: 0.88
- **[P2] Suppress prompt image detection on format retries:** `src/agents/command/attempt-execution.ts:556`
  - body: resolveRetryImages strips explicit images when the previous failure was a format error, but the embedded runner only disables prompt image detection for cross-provider retries. A same-provider fallback after an image-format rejection can therefore re-detect the same prompt-referenced local image and send it again, defeating the format-error retry path.
  - confidence: 0.82
