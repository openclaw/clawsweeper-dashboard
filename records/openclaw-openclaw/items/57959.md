Overall confidence: 0.86

Full review comments:

- **[P1] Guard skipped cross-provider fallback attempts:** `src/agents/command/attempt-execution.ts:321-323`
  - body: `isFallbackRetry` is derived from how many run callbacks have executed, but `runWithModelFallback` can skip a primary candidate before calling `run`. In that case the first actual callback can be for a different provider with `isFallbackRetry=false`, so this early return forwards images and the prompt-image suppression stays off across the provider boundary.
  - confidence: 0.86
- **[P1] Make cross-provider media trust explicit:** `src/agents/command/attempt-execution.ts:329-330`
  - body: This hard-codes image stripping whenever the provider changes, but the linked provider-trust issue calls for a documented trust policy and default/upgrade behavior. Landing the fail-closed policy without that maintainer decision can break or confuse existing cross-provider fallback expectations.
  - confidence: 0.82
- **[P2] Suppress prompt detection whenever retry images are stripped:** `src/agents/command/attempt-execution.ts:556`
  - body: `resolveRetryImages` strips explicit images for same-provider `format` failures too, but `suppressPromptImageDetection` is only enabled for cross-provider retries. A prompt that references a local image can therefore reload and send images after the explicit payload was stripped for the retry.
  - confidence: 0.8
- **[P1] Populate partial execution before threading it:** `src/agents/model-fallback.ts:818-831`
  - body: `runWithModelFallback` only emits the replay warning when a previous `FailoverError` carries `partialExecution`, but the production throw sites in this diff do not attach that metadata. As merged, the new partial-execution warning path would not fire for real fallback retries.
  - confidence: 0.78
