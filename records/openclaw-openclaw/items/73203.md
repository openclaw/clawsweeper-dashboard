## Summary

Keep open. Current main still lacks the direct Feishu interactive-card `tag: "text"` extraction and nested row-array traversal requested by this PR, so the work is not implemented or superseded; however, the PR's current recursive nested-array walk adds an unbounded traversal over external card JSON and should not merge as-is.

## What This Changes

The PR branch changes Feishu interactive-card parsing to extract direct `text` tag elements and nested array cells, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land a bounded version of the Feishu parser fix, or fold this PR's direct `text` tag and nested-array regression cases into #56795 if maintainers choose the broader parser. The final implementation should stay inside the Feishu plugin and include regression coverage for quoted/replied interactive-card messages.

## Reproduction Assessment

Yes. A current-main fetched Feishu interactive message whose `body.content` contains `elements: [{ "tag": "text", "text": "..." }]` or row arrays under `elements` has a clear source-level reproduction path: the parser has no text-tag branch and does not traverse nested arrays.

## Solution Assessment

No, not yet. Direct `text` support is a narrow maintainable fix, but the recursive nested-array walk should be changed to the stated one-level row flatten or guarded with depth, node-count, and output limits before merge.
