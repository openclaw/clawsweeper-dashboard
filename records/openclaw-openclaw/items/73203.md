## Summary

Keep open. Current main still lacks the reported Feishu direct `text` tag and nested row extraction, but the PR’s recursive traversal over external card JSON should be bounded before merge.

## What This Changes

The PR adds Feishu interactive-card `text` tag extraction, nested-array element traversal, focused regression tests, and a changelog entry.

## Best Possible Solution

Keep the Feishu-local parser/test/changelog scope, but replace recursive traversal with one-level row flattening or a bounded iterative walk before merge.

## Reproduction Assessment

Yes. A mocked `getMessageFeishu` response with `msg_type: "interactive"` and either `elements: [{"tag":"text","text":"..."}]` or `elements: [[{"tag":"text","text":"..."}]]` follows the current parser path and falls back instead of extracting text.

## Solution Assessment

No, not yet. Direct `text` extraction is the right narrow fix, but nested-array traversal should be one-level as described or bounded with depth, node-count, and output guards.
