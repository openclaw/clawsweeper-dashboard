## Summary

Keep open. Current main still misses the Feishu `text` tag and nested-row card shapes, but this PR should not merge as-is because the final diff recursively walks external card JSON without bounds and still lacks real Feishu after-fix proof.

## What This Changes

The PR updates Feishu interactive-card read parsing to extract `text` elements, traverse nested element arrays, add focused tests, and add a changelog entry.

## Best Possible Solution

Keep the narrow Feishu-local parser and regression coverage, but replace recursive traversal with one-level row flattening or a bounded iterative traversal and require redacted real Feishu after-fix proof before merge.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu` response with `msg_type: "interactive"` and either direct `text` elements or nested row arrays reaches the current parser path that skips those values.

## Solution Assessment

No, not yet. Direct text extraction is the right narrow fix, but the nested-array implementation should be bounded or limited to the one-level flattening described by the PR before it is safe to merge.
