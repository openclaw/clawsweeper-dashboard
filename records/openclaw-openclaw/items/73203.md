## Summary

Keep open: this PR targets a valid Feishu interactive-card parsing gap that current main still has, but the branch now introduces unbounded recursive traversal over external card JSON and lacks after-fix real Feishu behavior proof.

## What This Changes

The branch changes Feishu replied interactive-card parsing to extract text-tag elements, traverse nested element arrays, add related regression tests plus a direct post-format fallback test, and update the changelog.

## Best Possible Solution

Keep the narrow Feishu-local text-tag support and regression coverage, but replace the recursive traversal with one-level row flattening or a bounded iterative traversal and require redacted real Feishu behavior proof before merge.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu` response with `msg_type: "interactive"` and either direct `text` elements or nested row arrays reaches the current parser path that skips those values.

## Solution Assessment

No, not yet. Direct text extraction is the right narrow fix, but the final nested-array implementation should be bounded or limited to the one-level flattening described by the PR before it is safe to merge.
