## Summary

Keep open: current main still misses the reported Feishu interactive-card `text` tag and nested row-array cases, and no related open/closed item has merged a replacement. The PR is on the right Feishu-owned parser surface, but its recursive nested-array traversal is not merge-safe until it is bounded or reduced to the stated one-level flattening behavior.

## What This Changes

The PR updates Feishu replied interactive-card parsing to extract direct `text` tag elements, walk nested array cells, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Repair this branch by keeping the Feishu-local `text` tag extraction and regression tests, then replace the recursive traversal with either the stated one-level row flattening or a bounded iterative traversal with depth/node/output guards and a deep-nesting regression test. If maintainers prefer the broader parser in #56795, fold these exact cases and safety bounds there before closing this PR.

## Reproduction Assessment

Yes. A mocked `getMessageFeishu` response whose interactive `body.content` contains `elements: [{"tag":"text","text":"..."}]` or `elements: [[{"tag":"text","text":"..."}]]` is enough: current main has no text-tag branch and skips array rows before falling back.

## Solution Assessment

No, not yet. Direct `text` support is a narrow maintainable fix, but the recursive nested-array traversal should be one-level as described or explicitly bounded before it parses external card JSON.
