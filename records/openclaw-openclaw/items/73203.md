## Summary

Keep open for https://github.com/openclaw/openclaw/pull/73203. Current main still has the Feishu interactive-card extraction gap, and the PR is useful, but the latest branch adds unbounded recursive traversal over external card JSON.

## What This Changes

The PR updates Feishu replied interactive-card parsing to extract direct text-tag elements, walk nested array cells, add focused regression tests, and add a changelog entry.

## Best Possible Solution

Keep the Feishu-local text-tag support and regression coverage, but replace the recursive array walk with the stated one-level row flattening or a bounded iterative traversal before merge.

## Reproduction Assessment

Yes. A mocked `getMessageFeishu` response with `msg_type: "interactive"` and either `elements: [{"tag":"text","text":"..."}]` or `elements: [[{"tag":"text","text":"..."}]]` reproduces the missing extraction on current main.

## Solution Assessment

No, not yet. Direct text extraction is the right narrow fix, but the nested-array traversal should be one-level or explicitly bounded for depth, node count, and output length because the input is external Feishu message data.
