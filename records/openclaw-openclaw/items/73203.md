## Summary

Keep open. The Feishu bug is real on current main, and this PR covers the right quoted-card surface, but the branch currently replaces skipped nested rows with unbounded recursive traversal over external card JSON. That needs a small safety repair before merge.

## What This Changes

The PR adds Feishu interactive-card extraction for direct `text` tag elements and nested array rows, plus focused `send.test.ts` coverage and a changelog entry.

## Best Possible Solution

Keep the Feishu-local text extraction and regression tests, but replace the recursive nested-array walk with either the one-level row flattening described in the PR body or a bounded iterative traversal with depth, node, and output limits plus a deep-nesting regression test.

## Reproduction Assessment

Yes. A mocked `getMessageFeishu` response whose interactive `body.content` contains `elements: [{"tag":"text","text":"..."}]` or `elements: [[{"tag":"text","text":"..."}]]` demonstrates the current-main gap and the intended branch behavior.

## Solution Assessment

No, not yet. Direct `text` support is the narrow maintainable fix, but recursive traversal of external card JSON should be one-level or explicitly bounded before merge.
