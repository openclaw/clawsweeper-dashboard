## Summary

Keep open. Current main still lacks the direct Feishu interactive-card `text` tag and nested row-array extraction this PR targets, but the proposed branch should not merge as-is because it recursively walks externally supplied card arrays without a depth, node, or output bound.

## What This Changes

The PR adds Feishu interactive-card extraction for direct `tag: "text"` elements and nested array layouts, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land a bounded Feishu plugin parser fix that preserves existing markdown/div/plain-text, i18n/template-variable, and post-format fallback behavior while adding direct `text` tags and row-array extraction with regression coverage. If maintainers choose the broader parser path, fold these exact cases and tests into that branch before merging.

## Reproduction Assessment

Yes. A current-main fetched Feishu interactive message whose `body.content` contains `elements: [{ "tag": "text", "text": "..." }]` or row arrays under `elements` follows a clear source-level path to `[Interactive Card]` or missing text because there is no `text` branch and nested arrays are not flattened.

## Solution Assessment

Not yet. The direct text-tag branch is the right narrow direction, but the nested-array traversal should be one-level as the PR describes or explicitly bounded before merge.
