## Summary

Keep open. The reported Feishu parser gap is real on current main, but the PR branch still introduces an unbounded recursive walk over external interactive-card JSON and is currently dirty against the base, so it needs a bounded repair before it is mergeable.

## What This Changes

The PR branch updates Feishu quoted/replied interactive-card parsing to read direct `text` tag elements and nested array cells, adds focused send tests, and adds a changelog entry.

## Best Possible Solution

Keep the fix inside the Feishu plugin, support the direct `text` tag, and handle the needed row-array shape with either the stated one-level flattening or a bounded iterative traversal with regression coverage. Preserve the contributor credit and changelog entry, but avoid broad inbound-card API changes unless maintainers intentionally move this into #56795.

## Reproduction Assessment

Yes. A mocked `getMessageFeishu` response whose interactive `body.content` contains `elements: [{"tag":"text","text":"..."}]` or `elements: [[{"tag":"text","text":"..."}]]` reproduces the lost-content path on current main because the parser lacks a `text` branch and skips row arrays.

## Solution Assessment

No, not as patched. The direct `text` branch is the right narrow change, but the recursive nested-array walk should be replaced with the promised one-level row flattening or guarded with depth, node-count, and output limits before merge.
