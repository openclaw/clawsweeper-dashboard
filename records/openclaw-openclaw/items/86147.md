## Summary

Keep open. Current main and the latest release still only transition the TUI status line back to idle when a run completes; there is no completion notification hook, config key, CLI flag, or documented TUI notification mode, and the request is a product-shaped ergonomic feature rather than a stale or duplicate bug report.

## What This Changes

The issue requests an opt-in TUI completion notification, such as a terminal bell, desktop notification, sound, or badge, when an OpenClaw agent run finishes.

## Best Possible Solution

Define a narrow opt-in TUI completion notification design, then implement it through the TUI run-completion path with config/schema/docs and focused TUI event-handler tests for active, inactive, local, and gateway-connected runs.
