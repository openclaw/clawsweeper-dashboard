## Summary

Keep open. Current main and v2026.5.22 still only show TUI completion through the visible status/idle transition; there is no TUI completion notification hook, config key, CLI flag, or documented notification mode. The nearby Control UI completion-sound work is adjacent but does not cover terminal TUI local or Gateway mode.

## What This Changes

Request a TUI completion notification for finished agent runs, potentially via desktop alert, terminal bell, configurable sound, badge/status highlight, and new TUI config keys.

## Best Possible Solution

Define a narrow opt-in TUI completion notification design, then implement it through the TUI run-completion path with config/schema/docs and focused TUI event-handler coverage for local, Gateway, active, inactive, and concurrent runs.
