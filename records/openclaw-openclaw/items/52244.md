## Summary

Keep open. The reported OpenRouter dynamic-model bug is plausible, but this PR still needs real behavior proof and has correctness blockers around reasoning visibility and fresh-cache hydration; current main only covers the named Kimi K2.5 example as a static catalog row, not the broader dynamic-model class.

## What This Changes

This PR adds OpenRouter catalog augmentation, a default-thinking fallback from cached OpenRouter capabilities, an all-cached OpenRouter capability export through the provider-stream SDK surface, and a formatting-only test change.

## Best Possible Solution

Revise the OpenRouter-owned fix so dynamic capability metadata is available before defaults resolve, native reasoning remains visible when thinking is inferred, and any cache/API seam is narrow, documented, and regression-tested.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a credible path: a dynamic OpenRouter model missing from the static catalog can default off before capability hydration, while native reasoning is only surfaced when reasoningMode is on or stream.

## Solution Assessment

No. The PR targets the right bug class, but as written it can still leave reasoningMode off when thinking is inferred and can read an empty fresh cache before OpenRouter capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:167-175`
  - body: Returning `"low"` here makes `thinkingActive` true, so the reply path skips resolving the model's default reasoning level. Since `subscribeEmbeddedPiSession()` only extracts native thinking when `reasoningMode` is `on` or `stream`, the dynamic `reasoning_content` this PR targets can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before reading the cache:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:306-307`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map when no disk cache exists. Default resolution runs before `prepareDynamicModel()` awaits the per-model fetch, so first use on a fresh cache can still produce no supplemental row and leave the dynamic model defaults off.
  - confidence: 0.8
