## Summary

Keep open: current main only partially covers the named Kimi example through the static catalog, while arbitrary dynamic OpenRouter reasoning models still need a correct default path and this external PR still has correctness blockers plus no inspectable real-behavior proof.

## What This Changes

This PR adds OpenRouter catalog augmentation, a default-thinking fallback from cached OpenRouter model capabilities, an all-cached OpenRouter capability export, and a formatting-only test change.

## Best Possible Solution

Revise and rebase the OpenRouter-owned fix so dynamic capability metadata is available before defaults resolve, native reasoning visibility is enabled for inferred reasoning models, and any SDK/cache seam is narrow, documented, and regression-tested.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a credible path: a dynamic OpenRouter model missing from the catalog can default off before capability hydration, while visible native reasoning is gated by reasoningMode.

## Solution Assessment

No. The PR is aimed at the right bug class, but as written it can still leave reasoningMode off when thinking is inferred and can read an empty fresh cache before OpenRouter capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:166-174`
  - body: Returning `"low"` here makes `thinkingActive` true, so the reply path skips resolving the model's default reasoning level. Since `subscribeEmbeddedPiSession()` only extracts native reasoning when `reasoningMode` is `on` or `stream`, the dynamic `reasoning_content` this PR targets can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before reading the cache:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:306-307`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map when no disk cache exists. Default resolution happens before `prepareDynamicModel()` awaits the per-model fetch, so first use on a fresh cache can still produce no supplemental row and leave defaults off.
  - confidence: 0.8
