## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve native reasoning when thinking is inferred:** `extensions/openrouter/index.ts:169-174`
  - body: Returning `"low"` for reasoning-capable OpenRouter models makes `thinkingActive` true, so current reply defaulting skips `resolveDefaultReasoningLevel()`. The embedded subscriber only extracts native reasoning when `reasoningMode` is `on` or `stream`, so this can still drop the reasoning blocks this PR is meant to surface unless the user explicitly enables reasoning.
  - confidence: 0.84
- **[P2] Hydrate the OpenRouter cache before catalog augmentation:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:307-308`
  - body: `getAllCachedOpenRouterModels()` calls the nonblocking cache ensure path and then returns `cache ?? new Map()`. On a fresh cache, catalog augmentation can run before the async OpenRouter model fetch completes, so the first dynamic reasoning model can still have no supplemental catalog row and keep defaults off.
  - confidence: 0.8
- **[P1] Keep the cache accessor out of the public SDK:** `src/plugin-sdk/provider-stream.ts:30`
  - body: This exports a provider-owned all-cache helper through `openclaw/plugin-sdk/provider-stream` without a documented supported contract. Third-party plugins could depend on process-local OpenRouter cache semantics, so either keep the helper internal to the bundled provider path or explicitly approve, document, and test the SDK surface.
  - confidence: 0.73
