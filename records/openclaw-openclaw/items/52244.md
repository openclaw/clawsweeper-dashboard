## Summary

Keep open: this is a plausible OpenRouter dynamic reasoning bug fix, but the current PR still has correctness blockers and the external contributor proof gate is unmet.

## What This Changes

The branch adds OpenRouter catalog augmentation and default-thinking fallback from cached capabilities, exports an all-cached OpenRouter capability accessor, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise and rebase the fix so dynamic OpenRouter capability metadata is available before defaults resolve, native reasoning output remains visible, and any SDK/cache surface is narrow, documented, and covered by fresh-cache and cached dynamic-model tests.

## Reproduction Assessment

No. I did not run a live OpenRouter/TUI repro; source inspection shows a credible path through catalog/default resolution and subscribeEmbeddedPiSession gating, but real model output is not proven here.

## Solution Assessment

No. The PR targets the right area, but returning a non-off thinking default can suppress reasoning default resolution, and the new all-cache accessor can return empty on a fresh cache before capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:172-177`
  - body: Returning `"low"` for reasoning-capable OpenRouter models makes `thinkingActive` true, so the reply path skips `resolveDefaultReasoningLevel()`. Native reasoning is only extracted when `reasoningMode` is `on` or `stream`, so the reasoning blocks this PR is meant to surface can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:307-309`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map when no disk cache exists. Catalog augmentation can therefore miss the first dynamic model on a fresh cache and leave defaults off.
  - confidence: 0.78
