## Summary

Keep open. Current main still lacks the OpenRouter dynamic-model defaulting change this PR targets, but the proposed patch still has blocking correctness issues: inferred thinking can prevent reasoning visibility, and the new cache accessor can return an empty catalog on first use.

## What This Changes

This PR adds OpenRouter catalog augmentation from cached capability data, a provider default-thinking hook for reasoning-capable dynamic models, an exported all-cached OpenRouter capability accessor, and a formatting-only lifecycle test change.

## Best Possible Solution

Revise the provider-owned fix so OpenRouter dynamic capability metadata is hydrated before both defaulting paths, native reasoning visibility is enabled for reasoning-capable dynamic models without requiring an explicit /reasoning directive, and any new SDK/cache surface remains narrow, documented, and covered by fresh-cache plus cached dynamic-model tests.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction exists with a dynamic OpenRouter reasoning model absent from the static catalog, no explicit /think or /reasoning directive, and a fresh capability cache; current default resolution and subscriber gating show how reasoning_content remains hidden.

## Solution Assessment

No. The PR is in the right OpenRouter/provider area, but returning a non-off thinking default can keep reasoningMode off, and the all-cache accessor can return an empty map before the awaited OpenRouter catalog fetch completes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:175-178`
  - body: Returning `"low"` for reasoning-capable OpenRouter models makes current main treat thinking as active before reasoning defaults are resolved. `get-reply-directives.ts` then skips `resolveDefaultReasoningLevel()`, and the subscriber only extracts native thinking when `reasoningMode` is `on` or `stream`, so OpenRouter `reasoning_content` can still be dropped unless the user explicitly sets `/reasoning`.
  - confidence: 0.84
- **[P2] Await the OpenRouter catalog before reading it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:307-309`
  - body: `getAllCachedOpenRouterModels()` calls the nonblocking `ensureOpenRouterModelCache()` path and immediately returns `cache ?? new Map()`. On a fresh cache this starts the OpenRouter fetch but returns an empty map, while `resolveModelAsync()` prepares the dynamic model later than reply-default resolution, so first use can still miss the supplemental catalog row and keep defaults off.
  - confidence: 0.8
