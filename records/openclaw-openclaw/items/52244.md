## Summary

Keep open. This is a plausible narrow OpenRouter bug fix, but the current patch still has correctness blockers, the external contributor has not provided inspectable real behavior proof, and the branch is currently conflicting with main.

## What This Changes

This PR adds OpenRouter catalog augmentation/default-thinking fallback from cached model capabilities, exports an all-cached OpenRouter capability accessor through `provider-stream`, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise or rebase the OpenRouter-owned fix so dynamic capability metadata is available before defaults resolve, native reasoning remains visible when thinking is inferred, and any SDK/cache seam is narrow, documented, and regression-tested.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a credible path: dynamic OpenRouter models outside the static catalog can default before async capability hydration, while native reasoning is only surfaced when `reasoningMode` is `on` or `stream`.

## Solution Assessment

No. The PR targets the right bug class, but returning a non-off thinking default can suppress reasoning default resolution, and the synchronous all-cache accessor can read an empty fresh cache before OpenRouter capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:175-180`
  - body: Returning `"low"` for reasoning-capable dynamic OpenRouter models makes `thinkingActive` true, so the reply path skips `resolveDefaultReasoningLevel()`. Since native reasoning is only extracted when `reasoningMode` is `on` or `stream`, the reasoning blocks this PR is meant to show can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:306-308`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map when no disk cache exists. Default resolution can run before `prepareDynamicModel()` awaits the per-model fetch, so first use on a fresh cache can still produce no supplemental row and leave dynamic model defaults off.
  - confidence: 0.8
