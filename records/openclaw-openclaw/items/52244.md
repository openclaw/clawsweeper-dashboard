## Summary

Keep open. Current main only partially supersedes the Kimi K2.5 example; the broader dynamic OpenRouter reasoning-default problem is not implemented on main, and this PR still has source-level blockers plus missing real behavior proof.

## What This Changes

The PR adds OpenRouter catalog augmentation from cached capabilities, a default thinking fallback for reasoning-capable dynamic models, an exported cached-capabilities accessor, and a test formatting change.

## Best Possible Solution

Revise and rebase the fix so dynamic OpenRouter capability metadata is available before thinking and reasoning defaults resolve, native reasoning output remains visible, and any SDK/cache seam is narrow, documented, and covered by fresh-cache tests plus real OpenRouter proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection supports the dynamic-model defaulting path and the review blockers, but no current-main OpenRouter/TUI run or inspectable after-fix output is attached.

## Solution Assessment

No. The PR targets the right OpenRouter capability metadata area, but returning a non-off thinking default can keep reasoningMode off, and the proposed all-cache accessor can be empty on a fresh cache before capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve native reasoning when thinking is inferred:** `extensions/openrouter/index.ts:175-178`
  - body: Returning `"low"` for reasoning-capable OpenRouter models makes `thinkingActive` true, so reply defaulting skips `resolveDefaultReasoningLevel()`. Native reasoning is only extracted when `reasoningMode` is `on` or `stream`, so the reasoning blocks this PR is meant to surface can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter cache before augmenting the catalog:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map when no disk cache exists. Catalog augmentation can therefore miss the first dynamic model on a fresh cache and leave defaults off.
  - confidence: 0.78
