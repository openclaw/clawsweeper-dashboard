## Summary

Keep open: current main still lacks the dynamic OpenRouter reasoning-default fix, but this PR is not merge-ready because it can still leave native reasoning hidden when thinking is inferred, can read an empty fresh cache before the OpenRouter catalog fetch completes, and needs the required changelog entry.

## What This Changes

The PR adds OpenRouter catalog augmentation and default-thinking fallback from cached dynamic model capabilities, exports an all-cached OpenRouter capability accessor, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise and rebase the OpenRouter-owned fix so dynamic capability metadata is available before default resolution, reasoning visibility defaults on for reasoning-capable dynamic models, and SDK, changelog, and regression coverage stay aligned.

## Reproduction Assessment

Yes. The source path is reproducible with an OpenRouter reasoning model absent from the static catalog, no explicit /think or /reasoning setting, and a fresh capability cache; default resolution can run before awaited dynamic capability loading, and native thinking is only displayed when reasoningMode is on or stream.

## Solution Assessment

No. The provider-owned direction is plausible, but this patch can still suppress default reasoning visibility and can return an empty supplemental catalog on first use before the cache fetch completes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:175-178`
  - body: Returning `"low"` for reasoning-capable OpenRouter models makes current main treat thinking as active before reasoning defaults are resolved. `get-reply-directives.ts` then skips `resolveDefaultReasoningLevel()`, while the embedded subscriber only extracts native thinking when `reasoningMode` is `on` or `stream`, so OpenRouter `reasoning_content` can still be dropped unless the user explicitly sets `/reasoning`.
  - confidence: 0.84
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` calls the nonblocking `ensureOpenRouterModelCache()` path and immediately returns `cache ?? new Map()`. On a fresh cache this starts the OpenRouter fetch but returns an empty map, while `prepareDynamicModel()` runs later than reply-default resolution, so first use can still miss the supplemental catalog row and leave dynamic model defaults off.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `extensions/openrouter/index.ts:136`
  - body: This is a user-facing OpenRouter provider fix, but the PR does not touch `CHANGELOG.md`. Repo policy requires user-facing fix/feat/perf changes to add an active-version changelog entry before landing.
  - confidence: 0.78
