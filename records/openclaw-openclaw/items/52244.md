## Summary

Keep open: this is a plausible OpenRouter bug fix, but the current PR still has correctness blockers and lacks observable after-fix behavior proof. Current main now covers the named Kimi K2.5 example in the static OpenRouter catalog, but the dynamic-model defaulting path and SDK/cache seam still need revision rather than cleanup closure.

## What This Changes

This PR adds OpenRouter catalog augmentation from cached model capabilities, a provider default-thinking fallback for cached reasoning-capable dynamic models, an all-cached OpenRouter capability SDK export, and a formatting-only daemon lifecycle test change.

## Best Possible Solution

Revise the PR so OpenRouter dynamic capability metadata is available before thinking and reasoning defaults are computed, preserve reasoning visibility when default thinking is inferred, and keep any SDK/cache exposure narrow, documented, and owner-approved.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection gives a medium-confidence path: dynamic OpenRouter defaults depend on catalog/cache data before async dynamic-model preparation, and reasoning display remains gated by reasoningMode.

## Solution Assessment

No, the current PR is not the best way to solve the issue as written. It reads a nonblocking cache synchronously and can infer thinking without enabling the reasoning display path that the user-visible bug is about.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:170-175`
  - body: Returning `"low"` here can make `thinkingActive` true, and current reply defaulting only resolves the model reasoning default while thinking is inactive. Since the subscriber only extracts native thinking when `reasoningMode` is `on` or `stream`, dynamic OpenRouter reasoning blocks can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before reading cached rows:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` calls the nonblocking cache initializer and immediately returns `cache ?? new Map()`. On a fresh install with no disk cache, catalog augmentation can see an empty map before `prepareDynamicModel()` awaits the OpenRouter fetch, so first use of a dynamic reasoning model can still default to off.
  - confidence: 0.78
