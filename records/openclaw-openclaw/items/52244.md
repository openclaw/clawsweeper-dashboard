## Summary

Keep open. The PR targets a real OpenRouter dynamic-reasoning visibility problem, but the current branch is not safe to merge because the proposed default-thinking change can still leave native reasoning hidden, the catalog cache path can be empty on first use, and the external PR still lacks inspectable real behavior proof.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached model capabilities, exports an all-cached OpenRouter capability accessor, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise and rebase the OpenRouter-owned fix so dynamic capability metadata is available before defaults resolve, native reasoning visibility stays enabled, and any SDK/cache seam is narrow, documented, and regression-tested.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection supports the dynamic OpenRouter path because missing catalog entries can default thinking/reasoning off before dynamic capability hydration, while native reasoning extraction is gated by reasoningMode.

## Solution Assessment

No. The patch is a plausible direction, but as written it can still leave reasoningMode off when thinking is inferred and can read an empty fresh cache before OpenRouter capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:172-177`
  - body: Returning `"low"` here makes `thinkingActive` true, so the reply directive path skips resolving the model's default reasoning level. The embedded subscriber only extracts native OpenRouter thinking when `reasoningMode` is `on` or `stream`, so the dynamic reasoning blocks this PR targets can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:308-309`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map when no disk cache exists. Default resolution runs before `resolveModelAsync()` calls `prepareDynamicModel()`, so first use on a fresh cache can still produce no supplemental row and leave the dynamic model defaults off.
  - confidence: 0.8
