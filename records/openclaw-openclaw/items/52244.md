## Summary

Keep open: the reported bug class remains plausible for arbitrary dynamic OpenRouter reasoning models on current main, but this branch is not mergeable as-is because source review shows blocking correctness issues and the PR still lacks inspectable real behavior proof.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached OpenRouter capabilities, exports an all-cached capability accessor through `provider-stream`, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise or rebase the OpenRouter fix so dynamic capability metadata is available before defaults resolve, native reasoning remains visible, and any SDK/cache seam is narrow, documented, and covered by fresh-cache and cached dynamic-model tests.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a credible path: dynamic OpenRouter models outside the static catalog can default before async capability hydration, while native reasoning is surfaced only when `reasoningMode` is `on` or `stream`.

## Solution Assessment

No. The PR targets the right bug class, but returning a non-off thinking default can suppress reasoning default resolution, and the synchronous all-cache accessor can read an empty fresh cache before OpenRouter capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:172-177`
  - body: Returning `"low"` for reasoning-capable dynamic OpenRouter models makes `thinkingActive` true, so the reply path skips `resolveDefaultReasoningLevel()`. Since native reasoning is only extracted when `reasoningMode` is `on` or `stream`, the reasoning blocks this PR is meant to show can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:307-309`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map when no disk cache exists. Default resolution can run before `prepareDynamicModel()` awaits the per-model fetch, so first use on a fresh cache can still produce no supplemental row and leave dynamic model defaults off.
  - confidence: 0.8
