## Summary

Keep open. Current main has fixed the specific OpenRouter Kimi K2.5 static-catalog gap, but this PR still targets a plausible broader dynamic OpenRouter reasoning default bug; the branch is not merge-ready because the proposed fix still misses the reasoning-visibility path, can miss a fresh capability cache, is merge-conflicting, and lacks inspectable real behavior proof.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached OpenRouter model capabilities, exports an all-cached OpenRouter capability accessor through the provider-stream SDK surface, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Rebase and revise the OpenRouter-owned fix so dynamic capability metadata is available before default resolution, inferred thinking also preserves reasoning visibility, and any cache/API seam stays narrow, documented, and covered by fresh-cache plus cached dynamic-model regressions.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection supports the generic dynamic-model risk because default thinking/reasoning still depends on catalog rows before OpenRouter dynamic capability preparation, but the specific Kimi K2.5 example is now static on current main.

## Solution Assessment

No. The proposed patch is not currently the best solution because inferring thinking alone can keep reasoningMode off, and the sync cache accessor can return empty before the OpenRouter catalog is fetched; a safer fix should hydrate one capability source before resolving both defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when inferring thinking:** `extensions/openrouter/index.ts:169-174`
  - body: Returning `"low"` here makes `thinkingActive` true, so the reply path skips `resolveDefaultReasoningLevel()`. The embedded subscriber only extracts native OpenRouter thinking when `reasoningMode` is `on` or `stream`, so dynamic reasoning blocks can still be dropped unless the user explicitly enables `/reasoning`.
  - confidence: 0.84
- **[P2] Await cached OpenRouter models before catalog augmentation:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:306-307`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, which triggers a background fetch and can immediately return an empty map on a fresh memory/disk cache. Because reply defaults are resolved before `resolveModelAsync()` awaits `prepareDynamicModel()`, first use of a dynamic reasoning model can still default both thinking and reasoning off.
  - confidence: 0.8
