## Summary

Keep open: this PR targets a plausible OpenRouter dynamic-model reasoning bug, but the branch still has correctness blockers and lacks inspectable real behavior proof before merge.

## What This Changes

The branch adds OpenRouter catalog augmentation from cached capabilities, a default-thinking fallback for cached reasoning models, an SDK cache accessor export, and a formatting-only daemon lifecycle test change.

## Best Possible Solution

Revise and rebase the OpenRouter-owned fix so dynamic capability metadata is hydrated before default resolution, both thinking and reasoning visibility are preserved, and any cache/API seam is narrow, documented, and regression-tested.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection supports the generic dynamic OpenRouter reasoning risk because default resolution can still run before dynamic capability preparation and native reasoning display remains gated by reasoningMode.

## Solution Assessment

No. The proposed patch is not the best solution as-is because inferring thinking alone can keep reasoningMode off, and the synchronous cache accessor can return empty before the OpenRouter catalog fetch completes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when inferring thinking:** `extensions/openrouter/index.ts:169-175`
  - body: Returning `"low"` here makes `thinkingActive` true, so the reply directive path skips resolving the model's default reasoning level. The embedded subscriber only extracts native OpenRouter thinking when `reasoningMode` is `on` or `stream`, so the dynamic reasoning blocks this PR targets can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await cached OpenRouter models before catalog augmentation:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:306-307`
  - body: `getAllCachedOpenRouterModels()` calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map on a fresh memory/disk cache. Since default resolution runs before `resolveModelAsync()` awaits `prepareDynamicModel()`, first use of a dynamic reasoning model can still default both thinking and reasoning off.
  - confidence: 0.8
