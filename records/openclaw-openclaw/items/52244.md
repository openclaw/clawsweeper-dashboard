## Summary

Keep open, but this PR is not merge-ready: it targets a plausible dynamic OpenRouter reasoning-output bug, while missing inspectable real-behavior proof and still carrying correctness, conflict, and SDK contract blockers.

## What This Changes

The branch adds OpenRouter catalog augmentation and default-thinking hooks from cached capabilities, exports an all-cached OpenRouter capability accessor, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Rebase and revise so dynamic OpenRouter capability metadata is hydrated before default resolution, inferred thinking preserves reasoning visibility, and any cache/SDK seam is either kept internal or explicitly documented and approved.

## Reproduction Assessment

Yes by source inspection, not live: a dynamic OpenRouter reasoning model absent from the catalog can resolve no reasoning row, and embedded subscription hides native thinking unless reasoningMode is on or stream. The named Kimi K2.5 example is now static on current main, so the remaining repro is the broader dynamic-model path.

## Solution Assessment

No. The PR uses relevant OpenRouter capability metadata, but this implementation can still leave reasoning visibility off, can miss a fresh cache, conflicts with current main, and needs a maintainer decision before adding a public SDK cache seam.
