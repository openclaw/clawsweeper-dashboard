## Summary

Keep open. The dynamic OpenRouter reasoning gap still appears valid on current main, but this PR is not merge-ready because its inferred-thinking path can still leave native reasoning hidden and its all-cache accessor can miss a fresh OpenRouter catalog before default resolution runs.

## What This Changes

This PR adds OpenRouter catalog augmentation from cached model capabilities, a provider default-thinking fallback for reasoning-capable dynamic models, an exported OpenRouter cache accessor in the provider stream SDK, and a formatting-only daemon lifecycle test change.

## Best Possible Solution

Revise and rebase the OpenRouter-owned fix so dynamic capability metadata is available before thinking and reasoning defaults are resolved, native reasoning visibility is enabled for reasoning-capable dynamic models, and any new SDK/cache surface is narrow, documented, and covered by fresh-cache plus cached dynamic-model tests.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction exists with a dynamic OpenRouter reasoning model absent from the static catalog, no explicit /think or /reasoning directive, and a fresh capability cache; default resolution runs before prepareDynamicModel and the subscriber drops native reasoning when reasoningMode stays off.

## Solution Assessment

No. The PR is in the right provider-owned area, but returning a non-off thinking default can suppress default reasoning resolution, and the proposed all-cache accessor can return empty before the OpenRouter catalog fetch is awaited.
