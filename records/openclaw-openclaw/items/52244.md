## Summary

Keep open. Current main now covers the PR body's named Kimi K2.5 example with a static OpenRouter catalog row, but the broader dynamic-model reasoning-default problem is still source-plausible, and this PR still has blocking correctness issues plus missing inspectable real behavior proof.

## What This Changes

The PR adds OpenRouter catalog augmentation from cached capabilities, a default-thinking fallback for cached reasoning models, an SDK cache accessor export, and a formatting-only daemon lifecycle test change.

## Best Possible Solution

Revise and rebase the OpenRouter-owned fix so dynamic capability metadata is available before defaults resolve, reasoning visibility stays enabled for native reasoning output, and any SDK/cache seam is narrow, documented, and regression-tested.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection supports the broader dynamic OpenRouter path because missing catalog entries default thinking/reasoning off before prepareDynamicModel loads capabilities, while native reasoning extraction is gated by reasoningMode.

## Solution Assessment

No. The patch is a plausible direction, but as written it can still leave reasoningMode off when thinking is inferred and can read an empty fresh cache before OpenRouter capabilities are hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when inferring thinking:** `extensions/openrouter/index.ts:175-178`
  - body: Returning `"low"` here makes `thinkingActive` true, so the reply directive path skips resolving the model's default reasoning level. The embedded subscriber only extracts native OpenRouter thinking when `reasoningMode` is `on` or `stream`, so the dynamic reasoning blocks this PR targets can still be dropped unless the user explicitly enables reasoning.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` calls the nonblocking cache ensure path, which can start a background fetch and immediately return an empty map on a fresh memory/disk cache. Since default resolution runs before `resolveModelAsync()` calls `prepareDynamicModel()`, first use of a dynamic reasoning model can still default both thinking and reasoning off.
  - confidence: 0.8
