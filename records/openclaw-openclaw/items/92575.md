## Best Possible Solution

Land this narrow `initSessionState` change with regression tests so daily/idle rollover preserves behavior overrides consistently with `/new`, while leaving model fallback clearing and explicit-reset CLI rotation intact.

## Solution Assessment

Yes. Carrying these existing behavior fields beside model/auth in the new-session rollover branch is the narrowest maintainable fix; pushing this into transport code or the model/auth provenance helper would add the wrong ownership or filtering semantics.
