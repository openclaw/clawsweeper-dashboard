## Summary

Keep open: the docs gap is real, but the current diff should not merge because it restarts the gateway before diagnosing whether the local dashboard is already healthy, and the external PR lacks after-fix real behavior proof.

## What This Changes

This PR adds a local dashboard after-update recovery section to the gateway troubleshooting docs with `lsof`, `curl`, `openclaw gateway restart`, and `openclaw gateway start` steps.

## Best Possible Solution

Ship a docs-only runbook that checks the listener and root dashboard response first, routes healthy-gateway cases to stale browser-state recovery, and only runs restart/start when those checks fail.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a docs enhancement. The docs gap is visible on current main, and the PR-specific problem is visible from the proposed command order in the patch.

## Solution Assessment

No, not as written. A docs-only troubleshooting update is the right boundary, but the maintainable flow is diagnostic-first and makes restart/start conditional on failed listener or root-dashboard checks.
