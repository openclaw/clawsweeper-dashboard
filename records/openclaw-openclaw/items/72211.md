## Summary

Keep open; the docs gap is real and the commands map to current gateway lifecycle behavior, but the proposed flow restarts before confirming whether the dashboard is already serving, matching both existing review comments.

## What This Changes

The PR adds a `docs/gateway/troubleshooting.md` section for local dashboard connection failures after updates with LaunchAgent recovery, `lsof`/`curl` checks, restart/start steps, and stale browser guidance.

## Best Possible Solution

Ship the docs-only runbook after making it diagnostic-first: check the listener and root dashboard response, send healthy-gateway users to the root URL and stale-browser path, and reserve restart/start for failed listener or HTTP checks.

## Reproduction Assessment

Not applicable as a docs enhancement. The current docs gap is verifiable by docs search, and the PR-specific problem is visible in the proposed command order.

## Solution Assessment

No, not as written. A docs-only troubleshooting update is the right boundary, but the safer flow is to test `lsof`/`curl` first and restart only if the gateway is not listening or not serving the dashboard.
