## Summary

Keep open. The PR fills a real current-main docs gap, but the proposed flow is not merge-ready because it restarts the gateway before checking whether the local dashboard already serves HTML, which undercuts the stale-browser case the section is trying to document.

## What This Changes

The PR adds a troubleshooting section for local dashboard connection failures after updates, including LaunchAgent recovery, listener/curl checks, restart/start commands, and stale browser-state guidance.

## Best Possible Solution

Merge a docs-only runbook once it is diagnostic-first: check the listener and root dashboard response, keep the stale-browser guidance when HTML is returned, and reserve restart/start for cases where the listener or dashboard response is failing.

## Reproduction Assessment

Not applicable as a docs enhancement. The docs gap is verifiable from current-main docs search, and the PR-specific issue is visible in the added command sequence where restart precedes lsof/curl.

## Solution Assessment

No, not as written. A docs-only change is the right boundary, but the maintainable version should make restart/start conditional on failed listener or dashboard checks.
