## Summary

Keep open. The docs gap is real and the referenced gateway lifecycle commands exist, but the PR still tells users to restart before checking whether the local dashboard is already healthy, which can interrupt active sessions in the stale-browser case the new text describes.

## What This Changes

The PR adds a `docs/gateway/troubleshooting.md` section for local dashboard connection failures after updates, including LaunchAgent recovery, `lsof`/`curl` checks, restart/start commands, root URL guidance, and stale browser-state notes.

## Best Possible Solution

Ship the docs-only runbook after making it diagnostic-first: check listener and root dashboard response, route healthy HTML cases to root URL/stale-browser guidance, and reserve restart/start for failed listener or HTTP checks.

## Reproduction Assessment

Not applicable as a docs enhancement. The docs gap is visible on current main, and the PR-specific problem is reproducible by reading the proposed command order in the patch.

## Solution Assessment

No, not as written. A docs-only troubleshooting update is the right boundary, but the maintainable flow is to diagnose first and restart only when the gateway is not listening or not serving the dashboard.
