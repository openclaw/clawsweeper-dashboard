## Summary

Keep open: the docs addition covers a current-main troubleshooting gap, but the proposed flow restarts the gateway before proving the local dashboard is actually down. Greptile and the earlier ClawSweeper pass both pointed at the same conditional-restart issue, and the patch is otherwise docs-only with no security concern.

## What This Changes

The PR adds a `docs/gateway/troubleshooting.md` section for local dashboard connection failures after updates, with `lsof`/`curl` checks, gateway start/restart recovery, and stale browser-state guidance.

## Best Possible Solution

Ship the docs-only troubleshooting section after making the flow diagnostic-first: check the listener and root dashboard response, route successful HTML cases to browser-state recovery, and reserve restart/start for failed checks or the not-loaded LaunchAgent signature.
