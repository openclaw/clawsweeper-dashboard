## Summary

Keep open. The docs gap remains on current main and the PR is correctly scoped, but the proposed runbook restarts the gateway before checking whether the local dashboard is already serving HTML, which conflicts with the stale-browser case it describes.

## What This Changes

The PR adds a docs-only troubleshooting section to `docs/gateway/troubleshooting.md` for local dashboard connection failures after updates, including `lsof`/`curl` checks and gateway start/restart recovery steps.

## Best Possible Solution

Ship the runbook after making it diagnostic-first: check listener and root dashboard response, route healthy-gateway cases to root URL or stale browser-state recovery, and reserve start/restart for failed checks.

## Reproduction Assessment

Not applicable as a docs enhancement. The docs gap is visible on current main, and the proposed ordering issue is visible in the PR patch.

## Solution Assessment

No, not as written. The docs-only boundary is maintainable, but the flow should diagnose first and restart only when listener or dashboard checks fail.
