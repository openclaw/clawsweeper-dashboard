## Summary

Keep open. The docs gap still exists on current main and this is a plausible docs-only fix, but the proposed flow restarts before proving the gateway is unhealthy and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a `docs/gateway/troubleshooting.md` section for local dashboard connection failures after updates, with `lsof`/`curl` diagnostics and gateway restart/start recovery steps.

## Best Possible Solution

Ship a docs-only runbook that checks the listener and root dashboard response first, treats returned HTML as stale browser state, and only runs restart/start when those checks fail.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a docs enhancement. Current main lacks the focused local-dashboard after-update section, and the PR-specific flow problem is visible from the diff.

## Solution Assessment

No. A docs-only troubleshooting update is the right boundary, but this patch should check `lsof`/`curl` before telling users to restart the gateway.
