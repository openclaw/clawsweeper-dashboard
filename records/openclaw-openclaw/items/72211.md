## Summary

Keep open. Current main still lacks this focused local-dashboard after-update recovery runbook, and the docs-only idea is useful, but the PR remains not merge-ready because it tells users to restart before first proving whether the gateway is already healthy.

## What This Changes

The PR adds a docs-only troubleshooting section for local dashboard connection failures after updates or macOS LaunchAgent reload issues.

## Best Possible Solution

Ship the docs-only runbook after making it diagnostic-first: check listener and root dashboard response, route healthy-gateway cases to root URL or stale browser-state recovery, and reserve restart/start for failed checks.

## Reproduction Assessment

Not applicable as a docs enhancement. The docs gap is visible in current-main docs, and the PR-specific problem is visible in the proposed command order.

## Solution Assessment

No, not as written. A docs-only troubleshooting update is the right boundary, but the safer flow is to diagnose first and restart only when the listener or dashboard check fails.
