## Repair Work Prompt

Fix the gateway/provider startup regression reported in https://github.com/openclaw/openclaw/issues/90886. Current main still lets implicit provider catalog discovery run without a timeout outside live mode: `resolveLiveProviderCatalogTimeoutMs()` returns null and `runProviderCatalogWithTimeout()` awaits `runProviderCatalog()` directly when no explicit timeout is supplied. Keep the fix in the provider discovery/model config boundary, avoid adding new config/env surface unless existing `OPENCLAW_LIVE_PROVIDER_DISCOVERY_TIMEOUT_MS` semantics are intentionally reused, and preserve degraded startup behavior by warning/skipping timed-out uncredentialed provider discovery rather than blocking gateway readiness. Add focused tests for non-live default timeout behavior and ensure gateway startup callers with explicit `providerDiscoveryTimeoutMs: 5000` remain unchanged. Do not edit CHANGELOG.md.
