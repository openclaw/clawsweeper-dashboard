## Repair Work Prompt

Fix the gateway/provider startup regression reported in https://github.com/openclaw/openclaw/issues/90886. Current main returns no default provider catalog timeout outside live mode, so callers without `providerDiscoveryTimeoutMs` can await a hung provider catalog indefinitely and keep gateway startup sidecars from becoming ready. Use https://github.com/openclaw/openclaw/pull/90959 as source evidence, but produce a narrow current-main PR: add a bounded default timeout for implicit provider catalog discovery, keep timeout handling in `src/agents/models-config.providers.implicit.ts`, preserve the existing explicit 5 second gateway primary-model warmup path in `src/gateway/server-startup-post-attach.ts`, avoid provider-specific OpenAI/GitHub Copilot branches, avoid adding a new config or env surface unless maintainers explicitly request it, and add focused regression coverage for non-live default, live default, valid override, invalid override, and explicit caller timeout behavior. Do not edit `CHANGELOG.md`. Capture release-note context in the PR body: gateway no longer hangs indefinitely when declared providers are uncredentialed or unresponsive.
