## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve wake metadata for maintenance replay:** `src/infra/heartbeat-runner.ts:306-310`
  - body: `DeferredHeartbeatRun` only stores reason/session/heartbeat, and replay calls `runOnce` without the original `source` or `intent`. That changes behavior for blocked `immediate`/`manual` wakes because `runHeartbeatOnce` only bypasses active reply gating when `intent` is preserved, so a maintenance replay can be skipped or delayed differently than the original wake. Store and replay `source`/`intent` with the deferred run.
  - confidence: 0.86
- **[P2] Route the new infra test into the CI plan:** `src/infra/maintenance-phase.test.ts:1`
  - body: Adding `src/infra/maintenance-phase.test.ts` currently creates the fallback `core-runtime-infra-misc` shard, and the latest GitHub annotations show `check-test-types`/`build-artifacts` failing because the CI plan expectations do not include that shard. Either classify this test into an existing intended shard or update the CI plan tests in the same PR.
  - confidence: 0.93
- **[P2] Use the canonical default agent for manual gating:** `src/cron/service/ops.ts:77-79`
  - body: This helper falls back only to `state.deps.defaultAgentId`; when that optional dependency is absent, jobs without `agentId` are checked as an empty agent instead of `DEFAULT_AGENT_ID` (`main`). The timer path uses `DEFAULT_AGENT_ID`, so manual `cron.run` can allow or block a maintenance-agent job differently from scheduled execution.
  - confidence: 0.78
