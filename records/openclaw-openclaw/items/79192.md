## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve wake metadata during replay:** `src/infra/heartbeat-runner.ts:306-310`
  - body: When a targeted heartbeat wake is blocked by maintenance, `DeferredHeartbeatRun` stores the reason/session/heartbeat but drops `source` and `intent`. Replay then calls `runOnce` without the original intent, so an immediate/manual wake can be treated as background work after the window opens and be skipped by active-reply deferral instead of preserving wake-now semantics. Store and pass the source/intent through the deferred replay path and add a regression test for an immediate replay.
  - confidence: 0.87
- **[P2] Use the canonical default agent for manual gating:** `src/cron/service/ops.ts:77-80`
  - body: Manual cron maintenance gating falls back only to `normalizeAgentId(state.deps.defaultAgentId)`, while the timer path uses `job.agentId ?? state.deps.defaultAgentId ?? DEFAULT_AGENT_ID`. If a job has no `agentId` and the service deps omit `defaultAgentId`, manual `cron.run` can classify the default job as non-maintenance and allow it in a phase where the scheduled path would block it. Fall back to `DEFAULT_AGENT_ID` here and cover the no-defaultAgentId case.
  - confidence: 0.9
