## Summary

Keep open: the earlier ClawSweeper comment matches the current source read. Current main has strong mitigations for `openclaw update`, Gateway control-plane updates, and stale managed service definitions, but the reported raw live `npm install -g` upgrade path is still not proven safe or explicitly unsupported.

## What This Changes

The issue asks OpenClaw to avoid stale hashed runtime chunk `ERR_MODULE_NOT_FOUND` crashes after a 2026.5.2 global npm upgrade while the Gateway may still be running.

## Best Possible Solution

Make manual package-manager upgrades either explicitly safe or explicitly unsupported: keep `openclaw update` as the preferred managed path, and add clear stop/update/restart diagnostics or docs for raw npm, pnpm, and bun installs.

## Reproduction Assessment
