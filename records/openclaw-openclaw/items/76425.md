## Summary

Keep open: current main has strong mitigations for `openclaw update` and `update.run`, but the raw package-manager/live-Gateway stale chunk path in this issue is not fully proven safe or superseded. The issue also remains the canonical thread where later Feishu/DingTalk/Slack/Telegram stale-hash evidence was consolidated.

## What This Changes

The issue reports `ERR_MODULE_NOT_FOUND` after upgrading to 2026.5.2 because a live Gateway keeps importing old hashed `dist/` chunks after the package tree is replaced.

## Best Possible Solution

Keep the hardened `openclaw update`/`update.run` handoff path, then make a maintainer decision for raw package-manager upgrades: either explicitly route them to stop/update/restart diagnostics or add a safe external updater boundary.

## Reproduction Assessment

Unclear for a high-confidence current-main reproduction. The old-version reports and current docs/source support the stale live-Gateway package-swap failure mode, but I did not run a destructive live global package-manager upgrade against current main.

## Solution Assessment

Unclear. The supported updater solution is the right maintainable path for managed installs, but the remaining raw npm/pnpm/bun live-upgrade behavior needs a maintainer policy decision before an automatic repair is safe.
