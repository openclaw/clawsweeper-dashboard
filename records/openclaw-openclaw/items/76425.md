## Summary

Keep open: current main has strong mitigations for the supported `openclaw update` and Gateway control-plane update paths, but the report’s raw global npm upgrade while a Gateway may still be live is not proven safe or fully resolved.

## What This Changes

The issue requests that upgrading to OpenClaw 2026.5.2 not leave the Gateway importing stale hashed `dist` chunks such as reply-media and inbound runtime files.

## Best Possible Solution

Make manual package-manager upgrades either explicitly safe or explicitly unsupported: route users to `openclaw update`, or add a clear stop/restart/diagnostic path for raw npm, pnpm, and bun installs.

## Reproduction Assessment

Unclear for current main. The report has concrete 2026.5.2 steps and logs, and current docs/source confirm the stale replaced-`dist` lazy-import failure mode, but I did not run the destructive global npm upgrade path.

## Solution Assessment

Unclear. The existing current-main solution is strong for `openclaw update` and control-plane updates, but maintainers still need to decide whether raw global package-manager upgrades while the Gateway is live are supported or should be documented as unsafe.
