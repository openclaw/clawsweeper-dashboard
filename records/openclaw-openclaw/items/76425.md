## Summary

Keep open: current main hardens the supported `openclaw update` and Gateway control-plane update paths, but raw global package-manager replacement while a Gateway may still be live remains a distinct unresolved support or documentation gap.

## What This Changes

The issue reports `ERR_MODULE_NOT_FOUND` for hashed runtime chunks after upgrading globally to OpenClaw 2026.5.2 with npm while Gateway/channel runtimes were active.

## Best Possible Solution

Define raw package-manager upgrade support explicitly: either make it live-safe across npm, pnpm, and bun, or document and diagnose that users must use `openclaw update` or stop the Gateway before manual installs.

## Reproduction Assessment

Unclear for a high-confidence current-main live repro. The issue has concrete 2026.5.2 logs and current source shows stale lazy-import chunks can exist after a replaced `dist` tree, but I did not destructively reproduce a global npm upgrade against a live Gateway.

## Solution Assessment

Unclear. The current implementation is a strong fix for supported updater paths, but maintainers still need to decide whether raw external package-manager upgrades while the Gateway is live are supported or should be made explicitly unsafe in docs and diagnostics.
