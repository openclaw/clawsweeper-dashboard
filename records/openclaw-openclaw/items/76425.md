## Summary

Keep open: current main has strong safeguards for the supported `openclaw update` and Gateway control-plane update paths, but the reported raw global npm upgrade while a Gateway process may still be live is not proven safe or fully resolved.

## What This Changes

The issue asks OpenClaw to prevent `ERR_MODULE_NOT_FOUND` crashes from stale hashed `dist` runtime chunks after upgrading a global npm install to 2026.5.2 while using Gateway channels.

## Best Possible Solution

Define the manual package-manager upgrade contract: either make raw npm/pnpm/bun upgrades safe around live Gateway processes, or document and diagnose them as unsupported unless the Gateway is stopped and restarted.

## Reproduction Assessment

No high-confidence current-main live reproduction was established. The report has concrete 2026.5.2 logs and current source/docs confirm the stale replaced-`dist` lazy-import hazard, but I did not run the destructive raw global npm upgrade path.

## Solution Assessment

Unclear. The current implementation is a good fit for supported `openclaw update` and control-plane package updates, but maintainers still need to decide whether raw package-manager upgrades while Gateway is live are supported or should be explicitly guarded in docs/diagnostics.
