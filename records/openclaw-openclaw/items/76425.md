The issue asks OpenClaw to avoid stale hashed `dist` module import failures after upgrading a global package install and restarting or using the Gateway.

## Best Possible Solution

Define the raw package-manager upgrade contract: either make it reliably recover with a forced fresh Gateway restart and diagnostics, or document it as requiring `openclaw update` or an explicit force restart while preserving the shipped fresh-process updater path.

## Solution Assessment
