## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/87312. On Windows, `openclaw doctor` can resolve Google Chrome at `C:\Program Files\Google\Chrome\Application\chrome.exe` but still warn that it cannot determine the installed Chrome version when the generic `chrome.exe --version` probe returns no usable output. Add a bounded Windows metadata fallback in `extensions/browser/src/browser/chrome.executables.ts` so `readBrowserVersion` can read ProductVersion/FileVersion from the executable before returning null. Keep the existing macOS Info.plist path and generic `--version` fallback, do not change the Chrome 144+ minimum, do not add config, and do not edit `CHANGELOG.md`. Add focused coverage in `extensions/browser/src/browser/chrome.version.test.ts` and, if needed, `extensions/browser/src/doctor-browser.test.ts` for a win32 Chrome 148 metadata result suppressing the false warning while still warning for unreadable or too-old versions. Validate with the focused Vitest wrapper and, when feasible, a Windows real doctor run with Chrome installed.
