## Summary

Keep open: this is a useful gateway primitive, but it is not merge-ready because grant-authenticated MCP requests still trust non-session loopback context headers from the lower-trust client.

## What This Changes

The PR adds an in-memory MCP attach-grant store, operator-admin attach.grant/attach.revoke gateway methods, grant-token authentication in the loopback MCP request path, and regression tests.

## Best Possible Solution

Land the grant primitive only after grant-authenticated loopback requests bind or reject every security-relevant scope and delivery context field, with explicit maintainer acceptance of the attach context contract.

## Reproduction Assessment

Yes for the review defect from source inspection: a grant-authenticated request uses boundSessionKey but still reads other x-openclaw-* context headers into scoped tools and message delivery. I did not run a live repro against the PR branch.

## Solution Assessment

No: binding sessionKey is a good primitive, but it is too narrow for the lower-trust attach boundary. The safer fix is to bind the full trusted context into the grant or fail closed on unbound context headers for grant-authenticated requests.
