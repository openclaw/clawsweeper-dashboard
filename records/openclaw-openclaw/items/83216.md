## Summary

Keep open. Source inspection supports a real web Chat display gap: OpenClaw marks runtime context as hidden, but the gateway display projection does not clearly filter `display: false` custom runtime-context messages before `chat.history` and live session events.

## What This Changes

The issue reports that Telegram-injected conversation context can render as visible messages in the web gateway Chat UI.

## Best Possible Solution

Filter internal runtime-context/custom messages from both `chat.history` and live `session.message` projection, with focused regression coverage for hidden Telegram context and `display: false` transcript entries.

## Reproduction Assessment

No live reproduction was performed, but source inspection gives a medium-confidence reproduction path: create or load a transcript entry shaped like the existing hidden runtime-context custom message and request web chat history. The projection path does not currently show a filter for that hidden custom shape.

## Solution Assessment

Yes, keeping this open is the right outcome. The narrow maintainable fix is in gateway history/event projection or transcript conversion, not a UI string filter and not a Telegram-only workaround.
