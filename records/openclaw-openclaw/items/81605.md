## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require auth before issuing worker tokens:** `src/contextmesh/coordinator.ts:301-319`
  - body: The standalone coordinator exposes worker approval over HTTP and returns a raw `deviceToken` after approving the stored pairing request. Because the coordinator also exposes worker records over unauthenticated HTTP, any client that can reach a non-loopback coordinator can approve a pending worker and obtain a token; route approval through an authenticated/scoped gateway path or require an operator credential before calling `approveDevicePairing`/`ensureDeviceToken`.
  - confidence: 0.91
- **[P1] Do not read coordinator-local files from job requests:** `src/contextmesh/coordinator.ts:332`
  - body: `/contextmesh/jobs` accepts a caller-provided `file` path and reads it on the coordinator host. If the coordinator is bound anywhere reachable by workers, this becomes an arbitrary local file read and the resulting chunks/output are exposed through job/status endpoints; keep file reading in the local CLI or enforce an authenticated, allowlisted workspace/file boundary.
  - confidence: 0.88
- **[P2] Classify the new gateway methods:** `src/gateway/server-methods-list.ts:142-146`
  - body: Adding these methods to the advertised gateway method list without adding them to `src/gateway/method-scopes.ts` breaks the existing classification contract: `method-scopes.test.ts` asserts every listed/core method is classified, and least-privilege callers will not know which operator scope to request. Add explicit read/write/pairing/admin scopes for the ContextMesh RPCs and cover them in the scope tests.
  - confidence: 0.95

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security-sensitive coordinator and token-handling concerns that must be fixed before merge.

Concerns:

- **[high] Coordinator approval endpoint mints worker tokens without auth:** `src/contextmesh/coordinator.ts:301`
  - body: The new coordinator HTTP approval route calls `approveDevicePairing`, calls `ensureDeviceToken`, and returns `deviceToken` without an operator authentication or scope check; a reachable coordinator can leak worker credentials.
  - confidence: 0.91
- **[high] Coordinator job endpoint reads arbitrary local files:** `src/contextmesh/coordinator.ts:332`
  - body: The new job endpoint reads `body.file` directly from the coordinator filesystem, so a remote caller could make a non-loopback coordinator read sensitive local files and expose their contents through ContextMesh job state.
  - confidence: 0.88
- **[medium] Gateway ContextMesh methods lack scope classification:** `src/gateway/server-methods-list.ts:142`
  - body: The PR exposes new gateway RPC names without adding method-scope policy, leaving the security contract and existing classification tests incomplete for status, dashboard, worker approval, and submit operations.
  - confidence: 0.95
