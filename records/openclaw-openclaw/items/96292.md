## Summary

Keep open. Current main and the latest release still do not expose a first-class per-skill writable data-directory contract; existing paths cover managed skill packages and installer tool artifacts, while SQLite-backed plugin state is limited to trusted code plugins, so this remains a product/API storage-boundary decision.

## What This Changes

Request to standardize community skill runtime data under the OpenClaw state directory, possibly via new skill metadata, install-time directory creation, and a model-visible placeholder.

## Best Possible Solution

Maintainers should choose one generic skill-data contract, then align skill metadata, install behavior, docs, and any migration or doctor expectations around that convention.

## Reproduction Assessment

Not applicable; this is a feature/API convention request. Source inspection confirms the requested data-directory contract is absent from current main and the latest release.

## Solution Assessment

Unclear; a new metadata field or state-dir placeholder is plausible, but the best fix depends on a maintainer choice about whether skill data belongs in filesystem paths, SQLite/plugin state, or ClawHub guidance.
