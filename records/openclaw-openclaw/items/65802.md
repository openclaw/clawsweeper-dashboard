## Summary

Keep this PR open. Current main still makes Feishu WebSocket inherit ambient HTTP(S) proxy environment, and the PR targets the reported reliability bug, but the submitted patch leaks the new opt-in env var across tests and lacks the required user-facing docs/changelog for a default behavior change.

## What This Changes

The PR changes Feishu WebSocket client creation to ignore ambient proxy env unless `OPENCLAW_FEISHU_WS_USE_PROXY=1`, and updates Feishu client proxy tests for default-direct and explicit opt-in behavior.

## Best Possible Solution

Land the Feishu-local opt-in behavior after restoring the new env var in the test harness and adding Feishu docs plus a changelog entry so proxy-dependent operators know to set `OPENCLAW_FEISHU_WS_USE_PROXY=1`.

## Reproduction Assessment

Yes. Current main can be verified from source and existing unit assertions by setting HTTP(S) proxy env and calling `createFeishuWSClient()`, which currently creates a proxy agent; the linked issue also includes live Feishu failure and recovery notes.

## Solution Assessment

No, not as submitted. The production direction is narrow and owner-local, but the patch needs isolated env cleanup plus upgrade documentation before it is the maintainable fix.
