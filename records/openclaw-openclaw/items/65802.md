## Summary

Keep this PR open. Current main and the latest release still use ambient proxy resolution for Feishu WebSocket, so the work is not obsolete; however, the submitted patch should not merge as-is because the new opt-in env var is not isolated in tests, and maintainers still need to accept the Feishu proxy-default policy change.

## What This Changes

The PR changes Feishu WebSocket proxy selection so ambient HTTP(S) proxy env is ignored unless `OPENCLAW_FEISHU_WS_USE_PROXY=1`, and updates the Feishu client proxy tests for the new default and opt-in behavior.

## Best Possible Solution

Keep the owner-local Feishu client change if maintainers accept explicit proxy opt-in as the desired default, but first fix the test env cleanup and add a small user-facing docs/changelog note for the new opt-in. Once merged, #65799 can close through the PR.

## Reproduction Assessment

Yes. Current main gives a high-confidence unit-level reproduction path: set proxy env and call `createFeishuWSClient()`, and the existing tests assert a proxy agent is created. The linked issue also provides live Feishu steps with broken proxy env producing `[ws] ws connect failed`.

## Solution Assessment

No, not as submitted. The production change is narrow and belongs in the Feishu plugin, but the PR should first isolate the new env variable in tests and get maintainer agreement on changing ambient proxy behavior to explicit opt-in.
