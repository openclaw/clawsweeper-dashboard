## Summary

Keep open: current main and v2026.5.7 still inherit ambient proxy env for Feishu WebSocket, but this PR is not merge-ready because it leaks the new opt-in env in tests, lacks user-facing docs, and still needs stronger patched-branch behavior proof.

## What This Changes

The PR gates Feishu WebSocket proxy-agent creation behind `OPENCLAW_FEISHU_WS_USE_PROXY=1` and updates Feishu client tests for the new default and opt-in behavior.

## Best Possible Solution

Land a Feishu-owned fix that stops accidental ambient proxy inheritance by default, documents the explicit opt-in, and leaves the broader `openclaw.json` proxy policy to https://github.com/openclaw/openclaw/issues/72595 unless maintainers choose to fold it in.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu WebSocket creation resolves an ambient proxy agent from HTTP(S) proxy env and passes it to the Lark WS client; current tests assert that behavior, while live comments confirm the failure mode.

## Solution Assessment

No, not as submitted. The Feishu-local default change is narrow, but the patch needs env cleanup, docs, patched-branch real behavior proof, and maintainer handling of the separate config-policy request before merge.
