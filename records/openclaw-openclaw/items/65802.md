## Summary

Keep this PR open. Current main still has the reported Feishu WebSocket ambient-proxy behavior, and the linked bug remains open, but the patch should not merge as-is because the new opt-in env variable leaks across tests and the public behavior change is not documented for upgrades.

## What This Changes

The PR changes Feishu WebSocket proxy selection to ignore ambient proxy environment variables unless `OPENCLAW_FEISHU_WS_USE_PROXY=1` is set, and updates Feishu client proxy tests for the new default and opt-in behavior.

## Best Possible Solution

Land the Feishu-local opt-in behavior only if maintainers accept the default change, with isolated env cleanup in tests and a user-facing docs/changelog note that tells proxy-dependent Feishu installs to set `OPENCLAW_FEISHU_WS_USE_PROXY=1`.

## Reproduction Assessment

Yes. Current main can be verified at unit level by setting HTTP(S) proxy env and calling `createFeishuWSClient()`, which the existing tests already assert creates a proxy agent; the linked issue also provides live Feishu failure and recovery notes.

## Solution Assessment

No, not as submitted. The production direction is narrow and owner-local, but the patch needs test env isolation and upgrade documentation before it is the maintainable fix.
