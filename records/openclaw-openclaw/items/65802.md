## Summary

Keep open. The Feishu bug is still present on current main and in the latest release, and this PR is the relevant implementation candidate, but it needs test isolation cleanup, user-facing docs, inspectable patched-branch proof, and a rebase before it is mergeable.

## What This Changes

The PR changes Feishu WebSocket proxy-agent creation to ignore ambient proxy environment variables unless `OPENCLAW_FEISHU_WS_USE_PROXY=1` is set, with Feishu client proxy tests updated for the new default and opt-in path.

## Best Possible Solution

Land a rebased Feishu-local default-direct WebSocket proxy fix after test cleanup, operator docs, patched-branch proof, and a maintainer decision on any config-field follow-up.

## Reproduction Assessment

Yes. On current main, setting HTTP(S) proxy env and calling `createFeishuWSClient()` follows a source path that resolves an ambient `ProxyAgent` and passes it to Lark `WSClient`, and current tests still assert that behavior.

## Solution Assessment

No, not as submitted. The Feishu-owned opt-in gate is the narrow code path, but the PR still needs env cleanup, docs, inspectable patched-branch proof, and a rebase before it is the best mergeable solution.
