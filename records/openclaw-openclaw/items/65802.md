## Summary

Keep open. Current main and v2026.5.7 still make Feishu WebSocket inherit ambient proxy env, so the PR still addresses a real source-backed bug, but it is not merge-ready because the new env var leaks across non-isolated tests and the PR lacks inspectable patched-branch real behavior proof.

## What This Changes

The PR gates Feishu WebSocket proxy-agent creation behind `OPENCLAW_FEISHU_WS_USE_PROXY=1` and updates Feishu proxy tests for default direct behavior plus explicit proxy opt-in.

## Best Possible Solution

Land a rebased Feishu-local opt-in fix after env cleanup, user-facing docs, and patched-branch proof, while handling any `openclaw.json` proxy setting as a maintainer product decision or follow-up.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure HTTP(S) proxy env and call `createFeishuWSClient()`, which currently resolves and passes an ambient proxy agent; the linked issue and PR comments add real Feishu failure/recovery context.

## Solution Assessment

No, not as submitted. The Feishu-owned opt-in gate is the narrow code path, but the PR needs test env cleanup, docs, and inspectable real behavior proof before it is the best mergeable solution.
