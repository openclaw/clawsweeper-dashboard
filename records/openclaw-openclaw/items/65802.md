## Summary

Keep this PR open. Current main still has the reported Feishu WebSocket ambient proxy inheritance, and the PR is a focused Feishu-local fix for #65799, but it currently leaks the new opt-in env var across tests and lacks the required docs/changelog coverage for the default behavior change.

## What This Changes

The PR gates Feishu WebSocket proxy-agent creation behind `OPENCLAW_FEISHU_WS_USE_PROXY=1` and updates Feishu client tests for default direct behavior plus explicit proxy opt-in.

## Best Possible Solution

Land the Feishu-local opt-in behavior only after fixing test env isolation and adding user-facing Feishu docs plus a changelog entry that tells proxy-dependent operators to set `OPENCLAW_FEISHU_WS_USE_PROXY=1`.

## Reproduction Assessment

Yes. Current main can be checked at unit level by setting HTTP(S) proxy env and calling `createFeishuWSClient()`, which the existing tests assert creates a proxy agent; the linked issue also provides live Feishu failure and recovery notes.

## Solution Assessment

No, not as submitted. The production direction is narrow and owner-local, but the patch needs isolated env cleanup plus upgrade documentation before it is the maintainable fix.
