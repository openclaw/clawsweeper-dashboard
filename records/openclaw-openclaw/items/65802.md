## Summary

Keep open. Current main and v2026.5.12 still make Feishu WebSocket inherit ambient HTTP(S) proxy env, and this PR targets the right Feishu-owned path, but the branch is not merge-ready because the new env var leaks across non-isolated tests, the new operator opt-in is undocumented, patched-branch real behavior proof is missing, and GitHub reports the branch as dirty.

## What This Changes

The branch gates Feishu WebSocket ambient proxy-agent creation behind `OPENCLAW_FEISHU_WS_USE_PROXY=1` and updates Feishu client proxy tests for the new default and opt-in behavior.

## Best Possible Solution

Land a Feishu-owned default-direct WebSocket proxy fix after restoring test env state, documenting the opt-in, adding patched-branch real behavior proof, and settling the env-only versus config-backed proxy policy.

## Solution Assessment

No, not as submitted. The Feishu-local opt-in gate is the narrow code path, but the PR still needs test cleanup, docs, inspectable after-fix proof, a branch refresh, and a proxy-policy decision before it is the best mergeable solution.
