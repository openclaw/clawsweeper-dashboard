## Summary

## What This Changes

## Best Possible Solution

Land an equivalent Feishu-plugin-local default-direct WebSocket fix after env cleanup, user-facing docs, and patched-branch live proof, with any `openclaw.json` proxy setting handled by an explicit maintainer decision or follow-up.

## Reproduction Assessment

Yes. Current main can be verified from source by setting HTTP(S) proxy env and calling `createFeishuWSClient()`, which currently creates and passes a proxy agent; the linked issue and PR comments provide matching live failure and workaround context.

## Solution Assessment

No, not as submitted. The production direction is narrow and owner-local, but the patch needs isolated env cleanup, documentation, and real after-fix proof; the requested config-level proxy flag is a maintainer product decision rather than a silent PR expansion.
