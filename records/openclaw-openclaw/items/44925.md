## Summary

Keep open. Current main and the latest release still have finite subagent announce give-up, failed or suspended delivery state, terminal timeout bookkeeping without an auto-restart policy, and a Claude CLI stream-json parser gap called out in the discussion; the linked open PRs are partial fixes rather than a canonical implemented solution.

## What This Changes

The issue asks for reliable subagent completion handling so failed announces, timeouts, and false artifact claims do not leave users waiting silently.

## Best Possible Solution

Define a durable, idempotent async completion contract covering child completion, parent/requester resumption, external delivery confirmation, timeout/prune notification, duplicate prevention, and result or artifact verification.

## Reproduction Assessment

Yes, at source level rather than live-run level. Current main and v2026.5.22 still have finite announce give-up and terminal timeout paths, and fresh 2026-05-22/23 comments report the same failure family on release builds.

## Solution Assessment

No, not as a narrow retry-only, notify-only, or parser-only patch. The maintainable fix needs one durable and idempotent completion/delivery state model with explicit timeout, failure, restart/recovery, verification, and duplicate-prevention semantics.
