Keep open; current main still exposes the reported loss family through finite completion-announce retry/give-up, failed delivery cleanup, swallowed completion-manager errors, and timeout completion without restart. ## What This Changes

Request reliable subagent completion delivery: retry or notify on announce failure, handle timeouts with restart or notification, and verify claimed file outputs before announcing success.

## Best Possible Solution

Define an explicit async completion state model that persists the child result before cleanup, tracks parent-session recording separately from external channel delivery, retries or expires with visible status, deduplicates sends, and specifies timeout and artifact/file-verification semantics.
