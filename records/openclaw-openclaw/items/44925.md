## Summary

Keep open; current main still has finite announce retry give-up, failed-delivery cleanup, and timeout terminalization paths while docs promise push-based completion delivery. Recent issue comments and open PRs show this remains the canonical durable subagent-delivery tracker rather than a safely stale or implemented report.

## What This Changes

The issue asks OpenClaw to stop silently losing subagent completions by retrying or surfacing announce failures, notifying or restarting timed-out subagents, and verifying file-result claims before completion.

## Best Possible Solution

Keep this as the canonical tracker for reliable async subagent completion semantics: persist completion state before cleanup, separate child completion from parent-session recording and external delivery, dedupe sends, surface failed or expired delivery, and define timeout and result/file verification behavior.
