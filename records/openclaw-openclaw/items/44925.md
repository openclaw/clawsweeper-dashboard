## Summary

Keep this open. Current main and the latest release still contain finite subagent announce retry/give-up paths and a pending-handoff failure branch, while the linked implementation work is still open or partial rather than merged.

## What This Changes

The issue asks OpenClaw to stop losing subagent completion results by retrying or surfacing announce failures, notifying or restarting timed-out subagents, and verifying claimed file outputs.

## Best Possible Solution

Keep this broad tracker open until a maintainer-selected fix path lands with focused source coverage and live channel/subagent proof, then split any remaining timeout, restart, or file-verification behavior into narrower follow-ups.

## Reproduction Assessment

Yes at source level, but not from a fresh live current-main run in this read-only review. Current main can turn a non-terminal completion handoff into delivered:false and can give up after the bounded retry budget, while multiple reporters supplied recent released-version reproductions.

## Solution Assessment

No final solution is on main. The best fix is not another blind retry increase; maintainers need a durable, deduped delivery/recovery contract with clear timeout or prune notification semantics and live proof.
