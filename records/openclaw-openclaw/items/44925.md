## Summary

Keep open; current main still has source-backed paths for finite announce give-up, failed or suspended final delivery, orphan pruning without a pending-delivery guard, timeout terminalization, and a Claude CLI stream-json parser path that does not finalize a turn on `message_stop`/`end_turn`. Related issues and open PRs are useful partial work, but they do not yet form a merged canonical fix for this broader completion-delivery contract.

## What This Changes

The issue asks OpenClaw to stop silently losing subagent completions by retrying or surfacing announce failures, notifying or recovering on timeouts, and verifying claimed file/results before completion is announced.

## Best Possible Solution

Define one durable async completion contract that separates child completion, requester resumption, external delivery confirmation, timeout/restart policy, idempotent replay, deduplication, and result/file verification, then consolidate the partial PRs against that contract.

## Reproduction Assessment

Yes, at source level. Current main still has finite announce give-up, terminal timeout cleanup, orphan pruning without pending-delivery preservation, and parser behavior that ignores `message_stop`/`end_turn` as final deliverable boundaries; the discussion adds recent production traces for the same class.

## Solution Assessment

No for a narrow retry-only, notify-only, or raw fallback patch. The maintainable solution is a single delivery/resumption state model with durable replay and explicit success/failure semantics so partial fixes do not cause duplicates or false delivered states.
