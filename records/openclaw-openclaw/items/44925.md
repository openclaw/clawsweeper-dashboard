## Best Possible Solution

Define and land a maintainer-approved durable subagent completion contract that wakes the requester or visibly fails exactly once, with live multi-agent proof, then close this issue from the merged fix.

## Solution Assessment

No, the current bounded best-effort retry/suspend behavior is not the best end state. The better solution is a durable exactly-once requester wake or visible-failure contract, with the linked PR or a successor proven in a real multi-agent setup.
