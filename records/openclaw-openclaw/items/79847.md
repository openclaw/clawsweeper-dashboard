## Summary

Keep open: current main still sends qmd-scoped XDG variables to mcporter, and the active fix candidate has not merged yet.

## What This Changes

The issue requests that qmd-manager stop leaking qmd-only XDG environment variables into mcporter child processes so mcporter 0.10.x can find its intended config.

## Best Possible Solution

Land a reviewed qmd-manager fix that either strips qmd-only XDG variables from mcporter spawns or provides an explicit per-agent mcporter config, with live mcporter 0.10.x QMD proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current main passes qmd-scoped XDG variables into runMcporter, and mcporter v0.10.2 uses XDG_CONFIG_HOME for config discovery when no explicit config path is provided.

## Solution Assessment

Is this the best way to solve the issue? Yes, scoping environment per spawn target is the narrowest maintainable fix; the open PR's explicit --config approach may also be acceptable once it has real behavior proof.
