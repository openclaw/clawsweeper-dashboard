## What This Changes

The PR changes update/status package-manager detection so npm package roots with `npm-shrinkwrap.json` report npm while pnpm and Bun-owned shrinkwrap roots keep their owning manager, and it uses shrinkwrap as the npm dependency marker.

## Best Possible Solution

Land this detector change only after maintainers accept the npm/pnpm/Bun precedence rules and have redacted real packaged-install status proof for the npm shrinkwrap case plus preserved pnpm/Bun ownership.

## Reproduction Assessment

Yes. From current-main source, a package root with `packageManager: pnpm` is classified before shrinkwrap markers are considered, so the reported npm shrinkwrap package root misdetects as pnpm without needing a new feature.

## Solution Assessment

Yes, with one proof gap. The PR fixes the narrow detector and dependency-marker path instead of adding a config override or rewriting update commands, but real packaged-install proof is still needed before merge confidence is complete.
