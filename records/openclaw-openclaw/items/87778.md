## What This Changes

The PR makes npm-shrinkwrap.json act as npm package-install evidence and as the npm dependency lock marker for update/status detection, with tests for npm shrinkwrap and git/source precedence.

## Best Possible Solution

Keep the npm-shrinkwrap fix, but make package status detection owner-aware for package roots and cover npm, pnpm, and bun published-install shapes in focused tests.

## Reproduction Assessment

Yes. Source inspection of current main shows a package root with packageManager pnpm and npm-shrinkwrap.json is classified as pnpm before lockfiles are considered, matching the linked npm-status report.

## Solution Assessment

No, not as written. The shrinkwrap marker should fix npm package roots without becoming the first signal for every published non-git package root; owner-aware package-root detection is the safer shape.
