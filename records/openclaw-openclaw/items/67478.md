## Summary

Keep open. Current main still lacks a CLI-wide root-execution guard or equivalent managed-image mitigation, and open PR #67509 is explicitly paired as the implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent or clearly warn against direct root execution on the DigitalOcean 1-Click image because it can create wrong-user state and conflicting gateway services.

## Best Possible Solution

Review and land PR #67509 or an equivalent maintainer-approved fix that prevents direct root execution before state or service mutation on managed images, while preserving or updating documented root install/onboarding workflows.

## Reproduction Assessment

Yes. The issue gives deterministic droplet steps, and current source inspection still shows the prerequisites: no entrypoint root guard, root-derived home/state paths, and user-scoped systemd service installation; I did not run the destructive live droplet scenario in this read-only pass.

## Solution Assessment

Yes at the issue level: prevention before state/service mutation is the maintainable direction, but the exact scope should be chosen with the existing root-based DigitalOcean install docs and legitimate root workflows in mind.
