## Summary

Keep open. Current main still lacks a CLI-wide root-execution guard; it has a partial doctor mitigation for detectable system-level gateway services, but direct root invocation can still resolve OpenClaw state under root. Open PR #67509 is explicitly paired as the implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a preventative guard or prominent 1-Click Droplet guidance so direct root CLI invocations do not create root-owned state, conflicting user services, or unreadable backups.

## Best Possible Solution

Land PR #67509 or an equivalent early entrypoint guard before config/state resolution, with a documented override, root help/version behavior, tests, and any DigitalOcean image or recovery guidance maintainers want to carry separately.

## Reproduction Assessment

Yes. The issue gives deterministic droplet steps, and current source inspection still shows the prerequisites for direct root state resolution and root-scoped user-service paths; I did not run the destructive live droplet scenario in this read-only review.

## Solution Assessment

Yes at the issue level. An early CLI guard or image-level wrapper is the narrowest preventative fix; docs alone would not stop direct invocation of the installed binary, though the guard design must account for legitimate root workflows.
