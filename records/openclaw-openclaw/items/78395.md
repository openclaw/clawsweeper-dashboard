## Summary

This PR still looks useful and reviewable: current main keeps the fallback allowlist path from resolving bare fallback aliases first, while the PR fixes that shared seam and adds focused coverage. ## What This Changes

The PR changes shared allowed-model fallback resolution so providerless fallback aliases resolve through the configured alias index before default-provider synthesis, with regression tests for model selection and `/models` picker data.

## Best Possible Solution

Land the shared fallback-resolution fix once a maintainer explicitly accepts alias-first providerless fallback routing as the intended upgrade behavior; otherwise pause for a broader runtime-selector policy.
