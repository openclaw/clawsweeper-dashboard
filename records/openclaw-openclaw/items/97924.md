## Summary

Keep open. Current main has typed `session_end` plugin hooks with archive metadata, but cron schedules are still time-based only and there is no supported event-triggered cron or agent-turn surface for session archive/rotation. Related open issues cover reset-memory hooks and pre-reset flushing, but this issue has distinct lifecycle automation/API semantics.

## What This Changes

The issue requests a `sessionArchived` or `sessionRotated` lifecycle trigger that can run a cron `agentTurn` once when a session is archived.

## Best Possible Solution

Decide one lifecycle automation contract, likely built around typed session lifecycle hooks plus a bounded scheduling or agent-turn API, rather than adding parallel cron-event and hook-specific behavior independently.
