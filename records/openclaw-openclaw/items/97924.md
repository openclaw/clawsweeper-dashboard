## Summary

Keep open: current main and v2026.6.10 have typed `session_end` plugin hooks and ordinary cron schedules, but no durable session archive/rotation event that cron can subscribe to. This is a real lifecycle automation request, but it needs maintainer product and security design before implementation.

## What This Changes

Request to add a session archive/rotation lifecycle event that can trigger cron or agent-turn automation with scoped archived-session metadata.

## Best Possible Solution

Keep this open for a maintainer-approved lifecycle automation contract: either extend cron with durable session lifecycle event triggers or explicitly document the plugin `session_end` plus scheduler APIs as the supported path.

## Reproduction Assessment

Not applicable as a defect reproduction; this is a feature request. Source inspection gives high confidence that current main lacks event-backed cron schedules while exposing only plugin lifecycle hooks for session end.

## Solution Assessment

Unclear. A first-class cron event trigger is plausible, but maintainers should first decide whether the supported boundary is cron schema expansion, plugin `session_end` plus scheduler APIs, or a separate internal lifecycle event.
