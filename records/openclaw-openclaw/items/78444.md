## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire --plugin through migrate command handling:** `docs/cli/migrate.md:24`
  - body: The docs now advertise `openclaw migrate codex --plugin ...`, but the migrate command still only defines and forwards `--skill`, and the shared migrate options/context do not carry plugin selections. Commander will reject the documented command before the Codex provider can read `ctx.plugins`, so selected plugin migration is unreachable in real CLI usage.
  - confidence: 0.93
- **[P1] Install into the managed Codex home:** `extensions/codex/src/migration/apply.ts:108`
  - body: This apply path launches the app-server with `CODEX_HOME: ctx.source`, so a normal migration from a personal Codex home installs/enables the plugin in the source runtime directory. Later OpenClaw Codex turns use the managed per-agent home, so migration can report success while the target session cannot see the plugin.
  - confidence: 0.91
