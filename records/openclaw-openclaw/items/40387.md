## Summary

Keep open. Current main still lacks EasyRunner documentation, so the PR is not obsolete, but the branch is dirty against main and the added guide still has blocking security and operational docs issues around Control UI exposure and image rebuilds.

## What This Changes

The PR adds a new EasyRunner deployment guide and links it from the docs navigation and platforms overview.

## Best Possible Solution

Request a rebased docs revision that places the guide in the current Install > Hosting surface, uses current container image/update guidance, configures the public Control UI origin explicitly, and treats device-auth disabling only as a short-lived break-glass option.

## Reproduction Assessment

Yes. The docs defects are reproducible by inspection: current main has no EasyRunner page, while following the PR's update commands does not rebuild the referenced local image and the proposed reverse-proxy guidance conflicts with current Control UI origin/device-auth requirements.

## Solution Assessment

No. The PR is useful source work, but the current version is not the narrowest maintainable solution because it needs a rebase plus security-correct Control UI/proxy guidance before it can be merged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Stop recommending the device-auth bypass for proxy setup:** `docs/platforms/easyrunner.md:280`
  - body: The guide says `dangerouslyDisableDeviceAuth` is required for reverse-proxy deployments. Current docs and audit code treat that flag as a break-glass severe security downgrade, so publishing this as normal EasyRunner setup would weaken remote Control UI authentication for users who follow the guide.
  - confidence: 0.93
- **[P2] Add the public Control UI origin configuration:** `docs/platforms/easyrunner.md:102`
  - body: The guide tells users to open `https://your-domain.com`, but the compose/config steps never add that origin to `gateway.controlUi.allowedOrigins`. Current gateway startup and browser-origin policy require explicit origins for non-loopback Control UI deployments, so this setup can fail or push users toward dangerous fallback settings.
  - confidence: 0.87
- **[P2] Rebuild the local image before redeploying updates:** `docs/platforms/easyrunner.md:207-208`
  - body: The update flow pulls new source and then runs `er app deploy`, but the app runs `localhost/openclaw:latest`. Without rebuilding that image before deploy, EasyRunner can keep launching the previously built OpenClaw version.
  - confidence: 0.9
