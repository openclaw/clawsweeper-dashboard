## Summary

Keep open: current main has not added EasyRunner docs, but this PR is not mergeable as-is because it targets stale docs navigation and includes deployment guidance that conflicts with current gateway security requirements.

## What This Changes

The branch adds an EasyRunner deployment guide under `docs/platforms` and links it from the docs navigation and platforms index.

## Best Possible Solution

Revise the contribution as a current `docs/install/easyrunner.md` hosting guide, add it to the active Hosting navigation and platform overview if maintainers want the page, and align the setup with supported gateway auth, allowed-origin, image rebuild, and link conventions.

## Reproduction Assessment

Not applicable. This is a docs PR, and the review path is source/doc consistency rather than a runtime bug reproduction.

## Solution Assessment

No. The current patch should not merge as-is; the safer path is to retarget the guide to the current install/hosting docs structure and replace the dangerous reverse-proxy advice with supported gateway security configuration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop recommending disabled device auth for reverse proxies:** `docs/platforms/easyrunner.md:280`
  - body: The guide says `dangerouslyDisableDeviceAuth` is required for reverse proxy setups, but current docs and the gateway security audit treat that setting as a severe break-glass downgrade. This would steer readers toward weakening the Control UI admin surface instead of documenting HTTPS with token/password or trusted-proxy configuration.
  - confidence: 0.93
- **[P1] Document the public Control UI origin:** `docs/platforms/easyrunner.md:57`
  - body: The compose/config example never adds the EasyRunner HTTPS domain to `gateway.controlUi.allowedOrigins`. Current gateway startup only seeds localhost origins for non-loopback/container binds, so opening `https://your-domain.com` can fail the Control UI origin check even after the container starts.
  - confidence: 0.86
- **[P2] Move the guide into the current hosting docs route:** `docs/docs.json:1143`
  - body: Current main keeps hosting guides under `docs/install/*` and the Install tab's `Hosting` group, while this adds `platforms/easyrunner` to an older Platforms nav location. Retarget the page and nav entry so it merges cleanly and appears with the other hosting guides.
  - confidence: 0.88
- **[P2] Rebuild the image before redeploying updates:** `docs/platforms/easyrunner.md:207-208`
  - body: The update flow pulls source and runs `er app deploy`, but the compose file runs `localhost/openclaw:latest`; without rebuilding that local image, users will keep redeploying the previously built OpenClaw version.
  - confidence: 0.91
- **[P3] Point the configuration link at the real docs route:** `docs/platforms/easyrunner.md:307`
  - body: `/configuration` is not a current docs route; the configuration reference lives under `/gateway/configuration-reference`, so this related-docs link will send readers to a missing page.
  - confidence: 0.82
