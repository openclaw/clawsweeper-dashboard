## Summary

Keep open: current main does not contain an EasyRunner guide, and the PR head still conflicts with current Control UI origin, device-auth, gateway-config, and image-update guidance. The remaining decision is security-sensitive docs/product review, not maintainer cleanup closure.

## What This Changes

The PR adds a new `docs/platforms/easyrunner.md` deployment guide and exposes it through `docs/docs.json` and `docs/platforms/index.md`.

## Best Possible Solution

Revise the guide to follow current Docker and gateway contracts: explicit pre-start config for gateway mode/auth/origins/proxy settings, no `dangerouslyDisableDeviceAuth` requirement, an update-time image rebuild, and then maintainer approval that EasyRunner belongs in core docs.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head and current main shows the missing public origin, unsafe device-auth guidance, unsupported config auto-creation claim, and stale update flow. I did not run a live EasyRunner deployment.

## Solution Assessment

No. The docs direction may be useful, but the current patch is not the best solution because it normalizes a break-glass auth bypass and relies on startup behavior current OpenClaw does not provide.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Configure the public Control UI origin:** `docs/platforms/easyrunner.md:102`
  - body: The guide tells users to open `https://your-domain.com`, but it never configures `gateway.controlUi.allowedOrigins` for that full origin. Current non-loopback Control UI startup requires explicit origins, so this deployment can fail startup or browser-origin checks.
  - confidence: 0.92
- **[P1] Remove the device-auth bypass requirement:** `docs/platforms/easyrunner.md:262-280`
  - body: This says `dangerouslyDisableDeviceAuth` is required behind a reverse proxy. Current security docs classify that flag as a severe break-glass downgrade, so users would unnecessarily disable Control UI device identity checks.
  - confidence: 0.95
- **[P2] Create gateway config before describing it:** `docs/platforms/easyrunner.md:108-113`
  - body: The guide claims first start automatically creates durable gateway config, but the Docker image runs `gateway --allow-unconfigured`; that bypasses the mode guard and does not write the listed config. Add an explicit pre-start config or mounted config example.
  - confidence: 0.9
- **[P2] Rebuild the image during updates:** `docs/platforms/easyrunner.md:203-208`
  - body: The update flow pulls source and redeploys but never rebuilds `localhost/openclaw:latest`, so EasyRunner can keep running the previously built image after an update.
  - confidence: 0.87
