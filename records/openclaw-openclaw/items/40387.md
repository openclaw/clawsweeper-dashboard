## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Create startup config with the public origin:** `docs/platforms/easyrunner.md:108-113`
  - body: The guide says the container automatically creates `openclaw.json`, but the default gateway start blocks missing config and non-loopback Control UI requires `gateway.controlUi.allowedOrigins`. Add a pre-start config/onboarding step or mounted config that sets `gateway.mode`, bind/auth, and the exact `https://your-domain.com` origin before `er app deploy` starts the gateway.
  - confidence: 0.94
- **[P1] Remove the device-auth bypass requirement:** `docs/platforms/easyrunner.md:262-280`
  - body: The troubleshooting and security sections say `dangerouslyDisableDeviceAuth` is required behind a reverse proxy. Current security docs reserve that setting for break-glass debugging because it disables Control UI device identity checks, so publishing it as production guidance would weaken gateway auth.
  - confidence: 0.96
- **[P2] Rebuild the local image before redeploying:** `docs/platforms/easyrunner.md:203-208`
  - body: The update flow pulls new source and immediately redeploys, but the compose file runs `localhost/openclaw:latest`. Without rebuilding that local image before `er app deploy`, EasyRunner can keep launching the previously built OpenClaw version.
  - confidence: 0.87
- **[P2] Remove the memory-core disable workaround:** `docs/platforms/easyrunner.md:236-248`
  - body: The troubleshooting section says the default memory plugin is unavailable in the container and tells users to disable memory. Current source makes `memory-core` the default memory slot, so this workaround disables the default plugin instead of fixing a real container issue.
  - confidence: 0.84
