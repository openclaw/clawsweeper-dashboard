## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Create startup config before deploy:** `docs/platforms/easyrunner.md:108-113`
  - body: The guide says the container automatically creates `gateway.mode` and related config on first start, but current gateway startup fails closed when config is missing or lacks `gateway.mode=local`, and non-loopback Control UI also requires explicit `gateway.controlUi.allowedOrigins`. Add an EasyRunner pre-start onboarding/config step or entrypoint that actually writes `gateway.mode`, bind, auth, and allowed origins before `er app deploy`.
  - confidence: 0.91
- **[P1] Remove the device-auth bypass guidance:** `docs/platforms/easyrunner.md:262-280`
  - body: This section tells reverse-proxy users to set `dangerouslyDisableDeviceAuth`, but current security docs reserve that key for break-glass debugging because it disables Control UI device identity checks entirely. The guide should use HTTPS plus explicit `gateway.controlUi.allowedOrigins` or supported trusted-proxy auth instead of publishing the dangerous bypass as required setup.
  - confidence: 0.93
- **[P2] Rebuild the local image before redeploying:** `docs/platforms/easyrunner.md:200-208`
  - body: The update flow pulls newer source and immediately redeploys, but the compose file runs the previously built `localhost/openclaw:latest` image. Add `podman build -t localhost/openclaw:latest .` before redeploying so updates actually reach the running container.
  - confidence: 0.88
