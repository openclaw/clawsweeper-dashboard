## Summary

Keep open: current main does not have the EasyRunner docs, but the PR still conflicts with current gateway/container security contracts, including unsafe device-auth guidance and missing origin/bind setup.

## What This Changes

Adds an EasyRunner deployment guide plus docs navigation entries in `docs/docs.json` and `docs/platforms/index.md`.

## Best Possible Solution

A publishable guide should mirror current Docker and gateway contracts with pre-start config, `lan` bind, explicit allowed origins, no device-auth bypass, an update-time image rebuild, and approval for EasyRunner in core docs.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head and current main shows the missing bind/origin setup, unsafe device-auth guidance, unsupported config auto-creation claim, and stale update flow. I did not run a live EasyRunner deployment.

## Solution Assessment

No. The current patch is not the best solution because it normalizes a break-glass auth bypass and relies on startup behavior current OpenClaw does not provide.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Set a container-reachable gateway bind:** `docs/platforms/easyrunner.md:50-57`
  - body: The compose example never sets `OPENCLAW_GATEWAY_BIND=lan` or an equivalent config value. The Docker image binds loopback by default, so Caddy in another EasyRunner container cannot reach port 18789 and the deployment will commonly serve 502s.
  - confidence: 0.92
- **[P1] Configure the public Control UI origin:** `docs/platforms/easyrunner.md:100-102`
  - body: The guide tells users to open `https://your-domain.com`, but it never configures that full origin in `gateway.controlUi.allowedOrigins`. Current non-loopback Control UI startup requires explicit origins, so fixing the bind still leaves startup or browser-origin auth failures.
  - confidence: 0.92
- **[P1] Remove the device-auth bypass requirement:** `docs/platforms/easyrunner.md:262-280`
  - body: This says `dangerouslyDisableDeviceAuth` is required behind a reverse proxy, but current security docs classify that flag as a break-glass severe downgrade. Users following this guide would unnecessarily disable Control UI device identity checks.
  - confidence: 0.95
- **[P2] Create gateway config before describing it:** `docs/platforms/easyrunner.md:106-113`
  - body: The guide claims first start automatically creates durable gateway config, but the Docker image runs `gateway --allow-unconfigured`; that bypasses the mode guard and does not write the listed config. Add an explicit pre-start config or mounted config example.
  - confidence: 0.9
- **[P2] Rebuild the image during updates:** `docs/platforms/easyrunner.md:200-208`
  - body: The update flow pulls new source and redeploys but never rebuilds `localhost/openclaw:latest`, so EasyRunner can keep running the previously built image after an update.
  - confidence: 0.87
