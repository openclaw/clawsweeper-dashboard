The issue requests making the existing `tools.web.search.maxResults` web-search setting mutable through the agent-facing gateway config tool instead of leaving it blocked as a protected config path.

## Best Possible Solution

Decide the security policy first, then either add a narrow web-search allowlist exception with guard coverage and docs or fold this path into the broader protected-config approval/allowlist design.
