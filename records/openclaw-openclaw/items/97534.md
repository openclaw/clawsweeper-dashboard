## Summary

## What This Changes

The issue requests that iOS chat message, intro, and composer text follow the user's Dynamic Type / Larger Text setting.

## Best Possible Solution

Replace the scoped chat text fixed-size fonts with semantic or scaled Dynamic Type-aware SwiftUI fonts, then verify message rendering and composer usability at larger accessibility sizes.
