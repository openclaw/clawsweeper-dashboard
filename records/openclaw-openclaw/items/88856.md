## Summary

Keep open: current main has Claude CLI orphan detection, session-reuse reset, retry/suspension bookkeeping, and blocked task-delivery state, but it still does not provide the requested parent/model-visible terminal failure signal for orphaned `Agent`/`sessions_spawn` tool uses or failed completion handoffs.

## What This Changes

The issue asks OpenClaw to surface dropped, killed, or failed subagent spawns and completion handoffs to the parent session as a typed tool result or equivalent model-visible signal.

## Best Possible Solution

Define one maintainer-approved parent-visible terminal failure contract, then prove orphaned spawn, timeout/no-output, failed handoff, restart recovery, and `sessions_yield` paths against that contract.

## Reproduction Assessment

Source-level yes, live no: current main records and retries several failure states and detects Claude CLI orphaned transcript tails, but I did not establish a live current-main reproduction of the original unmatched `tool_use` corpus path.

## Solution Assessment

The synthetic error result proposal is plausible, but not yet proven as the best permanent fix until maintainers choose how the signal composes with requester-agent handoff, durable queue/spool work, and `sessions_yield`.
