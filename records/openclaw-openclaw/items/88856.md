## Summary

Keep open: current main records, retries, suspends, or resets around subagent completion failures, but I did not find the requested parent-visible synthetic error result or equivalent signal for orphaned `Agent`/`sessions_spawn` results; related durable-delivery issues overlap but do not fully supersede this Claude CLI and parent-context signal request.

## What This Changes

The issue asks OpenClaw to surface orphaned or failed subagent completions to the parent model instead of leaving `Agent` or `sessions_spawn` tool uses without visible results.

## Best Possible Solution

Keep this open until failed or orphaned subagent completion becomes explicit parent-visible context, ideally reusing the durable delivery state tracked in #67777 rather than adding another retry source.

## Reproduction Assessment

No live high-confidence current-main reproduction was run; source inspection shows current paths can record failed or suspended subagent delivery and only reset Claude CLI reuse for trailing orphans, without proving a parent-visible error result.

## Solution Assessment

Current main is only a partial mitigation: the proposed typed error result is plausible, but the best fix should be maintainer-owned and unified with durable completion delivery rather than a separate ad hoc signal path.
