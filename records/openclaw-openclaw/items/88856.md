## Summary

Keep open: current main still has only partial mitigation for this family. It can detect/reset some Claude CLI orphaned transcript reuse and can store/suspend failed completion delivery state, but the requested parent/model-visible terminal failure signal is not implemented across orphaned tool_use, timeout, and failed handoff paths.

## What This Changes

The issue requests a first-class parent-visible failure signal when Agent or sessions_spawn work is orphaned, killed, disconnected, or fails completion delivery without a matching tool_result.

## Best Possible Solution

Define one maintainer-approved parent-visible terminal failure contract, then implement and test it across orphaned spawn tool_use, run timeout/no-output, failed completion handoff, restart recovery, and sessions_yield interactions.

## Reproduction Assessment

Yes, source-level: current main still has paths that record, log, reset, suspend, or block subagent failure state without creating the requested parent-visible result. No live current-main reproduction was run in this read-only review.

## Solution Assessment

No, current main is not the best complete solution yet; the synthetic error-result proposal is plausible, but maintainers still need to choose how that signal composes with durable delivery, requester handoff, and retries.
