## Best Possible Solution

Land a boundary-aware home shortening helper that preserves non-home substrings while still shortening real home path tokens after common text delimiters, applied consistently to core and terminal-core with focused tests, then rebase the branch cleanly.

Yes. Source inspection on current main shows unbounded `split/join` replacement, and the follow-up terminal proof demonstrates the configured-home failure with `OPENCLAW_HOME=/data`.
