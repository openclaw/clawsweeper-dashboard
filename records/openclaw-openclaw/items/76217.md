## Summary

Current main and the shipped v2026.4.26 release already preserve explicit Discord `user:` cron announce targets and route them through Discord DM channel creation, so the 2026.4.15 regression report is obsolete.

## What This Changes

The issue asks for isolated Discord cron reminders with `--announce --channel discord --to user:<id>` to deliver as DMs instead of failing with `Unknown Channel`.

## Best Possible Solution

Keep the shipped target-preserving Discord outbound path and have users on 2026.4.15 upgrade to at least v2026.4.26.

## Reproduction Assessment

Yes. The issue gives concrete cron CLI steps, and the current source/test path can be traced from `delivery.to = "user:<id>"` through Discord DM channel creation; no live Discord send was needed for this cleanup decision.

## Solution Assessment

Yes. Preserving channel-owned target kind and using Discord's normal DM channel creation is the narrow maintainable fix, rather than adding a cron-only Discord special case.
