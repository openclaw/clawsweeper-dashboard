## Summary

Current main already preserves explicit Discord `user:<id>` cron announce targets and sends them through the Discord DM-channel path; the reported 2026.4.15 build is older than the shipped v2026.4.26 proof release and the same root report was already reviewed as fixed.

## What This Changes

The issue reports that isolated Discord cron announce delivery to an explicit `user:<id>` DM target fails with `Error: Unknown Channel` while Telegram cron and reactive Discord DMs work.

## Best Possible Solution

Keep the shipped Discord target-normalization and DM send path, and direct users on 2026.4.15 to upgrade to v2026.4.26 or later before filing a fresh current-version repro.

## Reproduction Assessment

Yes. The issue provides concrete CLI steps and run-log evidence for the old build, and current-main behavior can be verified by tracing the deterministic target-resolution and Discord send paths without needing live Discord credentials.

## Solution Assessment

Yes. Preserving explicit `user:<id>` through generic cron delivery and letting the Discord plugin open the DM channel is the narrowest maintainable fix because it keeps provider-owned target semantics inside the Discord plugin.
