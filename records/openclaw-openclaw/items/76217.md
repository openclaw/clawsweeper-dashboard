## Summary

Current main and the shipped v2026.4.26 release already cover this Discord cron DM case: explicit `user:<id>` targets are preserved through cron announce delivery and Discord opens a DM channel for user targets. The report was filed against 2026.4.15, before the changeloged fix shipped.

## What This Changes

The issue reports that isolated Discord cron announce delivery to an explicit `user:<id>` DM target fails with `Unknown Channel` while Telegram cron delivery and reactive Discord DMs work.

## Best Possible Solution

Keep the shipped target-preserving Discord cron path and direct users on 2026.4.15 to update to v2026.4.26 or newer; reopen with fresh logs if it reproduces on current main.

## Reproduction Assessment

Yes. The issue gives concrete one-shot cron CLI steps and run evidence on OpenClaw 2026.4.15, and the current-main source/test path maps that scenario to `user:<id>` plus Discord DM-channel creation.

## Solution Assessment

Yes. The existing fix is the narrow maintainable solution: preserve explicit target kind through plugin routing, then let the Discord send layer open a DM channel for `user:<id>` targets.
