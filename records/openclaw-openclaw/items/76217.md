## Summary

Current main already preserves explicit Discord `user:` cron delivery targets and routes them through DM creation; the reported run used OpenClaw 2026.4.15, before the changelogged 2026.4.26 fix, and the still-open #74412 covers a different bare-ID inference gap.

## What This Changes

Discord isolated cron announce delivery with `--channel discord --to user:<id>` should deliver a proactive DM instead of failing with `Unknown Channel`.

## Best Possible Solution

Keep the shipped explicit-target preservation path and continue reviewing #74412 separately for inferred Discord DM cron jobs that still start from bare session-derived IDs.

## Reproduction Assessment

Yes. The issue gives a concrete OpenClaw 2026.4.15 CLI command and run result; current-main source inspection verifies the same explicit `--to user:<id>` path is now preserved and routed to Discord DM creation rather than channel send.

## Solution Assessment

Yes. Preserving explicit target kind through the existing cron, outbound, and Discord plugin seams is the narrow maintainable fix; the separate inferred bare-ID gap belongs in #74412 rather than reopening this old-version explicit-target report.
