## Summary

Current main and the shipped v2026.4.26-or-newer code path already cover this Discord cron DM case: explicit `user:<id>` targets are preserved, and Discord delivery opens a DM channel for user targets. This matches the already-closed related report #74375.

## What This Changes

The issue reports that isolated Discord cron announce delivery to an explicit `user:<id>` DM target fails with `Unknown Channel` while Telegram cron and reactive Discord DMs work.

## Best Possible Solution

Keep the shipped Discord target-normalization and DM-send path; users on 2026.4.15 should update to v2026.4.26 or later, and any current-version recurrence should be filed with fresh cron run logs.

## Reproduction Assessment

Yes. The issue provides a concrete `openclaw cron add` and `cron runs` reproduction for the old version, and current-main source inspection verifies the reported target rewrite path is no longer present.

## Solution Assessment

Yes. Preserving explicit `user:<id>` through generic cron resolution and letting the Discord plugin create the DM channel is the narrowest maintainable fix because it keeps target-kind semantics in the owning plugin path.
