## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes for the narrow bug: normalize persisted direct and parent model overrides before stale/disallowed classification, then emit a reset reason that distinguishes stale session state from true allowlist rejection. A broad change to WebChat parent inheritance belongs with the related product-decision issue.
