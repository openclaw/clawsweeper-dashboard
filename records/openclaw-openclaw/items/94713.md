## What This Changes

## Best Possible Solution

Land the focused linked PR for stored-override normalization and stale reset diagnostics, then close this issue from the merged fix while keeping the broader WebChat inheritance contract in the related issue.

## Reproduction Assessment

Yes, at source level: current main still creates parent-linked dashboard child sessions, resolves parent stored overrides, and formats reset refs as allowlist rejections. I did not run a full live dashboard rollover in this read-only review.

## Solution Assessment

Yes for the narrow issue: normalizing persisted overrides before reply-time stored override resolution and separating stale-session reset copy from true allowlist rejection is the narrow maintainable fix. The broader new-session inheritance/default-picker contract belongs in https://github.com/openclaw/openclaw/issues/86174.
