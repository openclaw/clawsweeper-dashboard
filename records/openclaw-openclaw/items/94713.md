The issue reports that dashboard child-session rollover can reject configured `openai/gpt-5.5`, revert to stale `gpt-5.4`, and show misleading allowlist remediation.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes for the narrow issue. Normalizing persisted overrides before reply reset and separating stale-session copy from true disallowed-copy is the maintainable fix; broader WebChat no-inherit behavior belongs to https://github.com/openclaw/openclaw/issues/86174.
