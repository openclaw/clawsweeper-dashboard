Yes for the narrow issue: normalize persisted provider/model override state before reply-time reset checks and distinguish stale-session cleanup from true allowlist rejection. A broad change to dashboard child-session inheritance is a separate product decision already represented by https://github.com/openclaw/openclaw/issues/86174.
