## Summary

Keep open: the restart-auth bug still appears real on current main, but this PR should not merge as written because the restart fix does not thread the managed service environment and the rate-limit behavior is superseded by the current configurable rotation policy.

## What This Changes

The PR changes restart health probing to load config-backed gateway auth and adds embedded-runner failover guards/tests that skip auth-profile rotation on rate limits when a fallback model is configured.

## Best Possible Solution

Ship a narrow restart-health fix that resolves gateway probe auth from the managed service env/config-aware auth seam, while leaving rate-limit fallback controlled by `auth.cooldowns.rateLimitedProfileRotations`.

## Reproduction Assessment

Yes. Source inspection shows current main still probes restart health with caller `process.env`, while the rate-limit behavior now has explicit tests and docs for one configured profile rotation before model fallback.

## Solution Assessment

No. The restart direction is useful, but the patch resolves from the caller env and changes rate-limit behavior outside the current central failover policy and docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Thread service env through restart probe auth:** `src/cli/daemon-cli/restart-health.ts:63`
  - body: The new resolver still captures `process.env`, but managed restart paths can have config paths or SecretRef inputs only in the installed service environment. Thread the restart/service env into config/probe-auth resolution and surface unresolved SecretRef warnings, otherwise healthy restarted gateways can still be probed with caller credentials and falsely time out.
  - confidence: 0.86

  - body: This skips profile rotation for every `rate_limit` when a fallback exists, but current main documents and tests `auth.cooldowns.rateLimitedProfileRotations` with default `1` and `0` for immediate fallback. Rebase through `run/failover-policy.ts` and matching docs/tests, or drop this behavior change from the PR.
  - confidence: 0.84
