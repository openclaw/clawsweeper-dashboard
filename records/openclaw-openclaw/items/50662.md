## Summary

Keep this PR open for maintainer review: the restart-auth fix is already represented more completely on current main, but the rate-limit fallback change conflicts with current documented rotation policy and the external PR still lacks real behavior proof.

## What This Changes

The PR changes gateway restart health probing to resolve config-backed gateway auth and changes embedded runner rate-limit failover to skip auth-profile rotation when a model fallback is configured.

## Best Possible Solution

Keep the current main restart-auth implementation, then make an explicit maintainer decision on whether rate-limit fallback should honor the configured rotation budget or skip directly to model fallback, with updated tests and docs if that policy changes.

## Reproduction Assessment

No live reproduction was established in this read-only review. Source and tests show current main already covers config-backed restart probe auth, while rate_limit plus fallbackConfigured still follows the documented one-profile-rotation policy.

## Solution Assessment

No, not as-is. The restart half should use the current async SecretRef-aware probe auth path, and the rate-limit half needs a policy decision instead of an unconditional skip around the configured rotation budget.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P2] Resolve restart probe auth through the SecretRef-aware path:** `src/cli/daemon-cli/restart-health.ts:78-83`
  - body: This uses the sync safe resolver and immediately reads `.auth`, so SecretRef-backed `gateway.auth.token` or `gateway.auth.password` can resolve to empty auth plus a discarded warning instead of going through the async secret-input resolver. Use the current `resolveGatewayProbeAuthSafeWithSecretInputs` path so config-backed SecretRefs work in restart probes too.
  - confidence: 0.82
- **[P2] Respect the configured rate-limit rotation budget:** `src/agents/pi-embedded-runner/run.ts:168-177`
  - body: Hard-coding `rate_limit + fallbackConfigured` to skip rotation makes the current `auth.cooldowns.rateLimitedProfileRotations` behavior ineffective and removes the documented default one same-provider retry before model fallback. Route this through the current failover policy so operators keep control of that budget.
  - confidence: 0.78
