Close as superseded: the restart-auth half is already covered on current main with the SecretRef-aware resolver and regression test, while the rate-limit half conflicts with the merged configurable rotation-cap policy and is now covered by canonical follow-up issues.

## What This Changes

The PR changes `openclaw gateway restart` health probing to load config-backed auth and changes embedded runner rate-limit failover to skip auth-profile rotation when a fallback model exists.

## Best Possible Solution

Keep the current restart-auth implementation and configurable rate-limit rotation cap; use https://github.com/openclaw/openclaw/issues/62022 and https://github.com/openclaw/openclaw/issues/79604 for any default failover-policy change.

## Reproduction Assessment

I did not live-reproduce the original real-world environment.

## Solution Assessment

No. The restart half should use current main's SecretRef-aware implementation, and the rate-limit behavior should stay configurable or be changed through the canonical policy issues rather than this unconditional skip.
