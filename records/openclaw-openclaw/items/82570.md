## What This Changes

The issue reports that delayed TTS audio can move or retimestamp an already visible assistant text reply after the user has replied.

## Best Possible Solution

Land a fix that keeps the already visible text reply in place while late TTS audio is attached or delivered without changing order or timestamp; leave the broader follow-up-audio feature work to https://github.com/openclaw/openclaw/issues/80630.
