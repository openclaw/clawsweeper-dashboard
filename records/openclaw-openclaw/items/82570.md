## Summary

Keep open: current main still has a source-reproducible Telegram streaming/TTS race where late TTS media turns a text-only final into a media final, causing the Telegram lane to clear the streamed preview and send a new voice/caption message instead of preserving the original text bubble.

## What This Changes

The issue reports that always-on TTS can attach audio after the user has already replied, moving the original agent text later in the Telegram chat and changing its timestamp.

## Best Possible Solution

Keep this open and fix Telegram’s late TTS finalization so an already-visible streamed text answer stays anchored while the generated voice note is appended without replaying the text.

## Reproduction Assessment

Yes, source-reproducible on current main: always-on TTS adds voice media to the final payload after the text preview exists, and the Telegram lane clears streamed previews whenever the final payload has media. A live Telegram run was not performed in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow Telegram finalization change for late TTS audio, not a new global TTS delivery mode. Preserve the finalized text preview and send only the generated voice attachment as a follow-up when the media came from auto-TTS.
