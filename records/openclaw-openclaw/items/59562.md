## Summary

Keep open. Current main still uses code-unit length for shared code-mode Markdown table padding, so the CJK table half remains real; the paragraph-break half is not covered by the linked CJK issue/PR and needs a narrower Telegram formatting decision before cleanup.

## What This Changes

Request Telegram agent replies to preserve readable paragraph spacing and align Markdown tables that contain CJK characters.

## Best Possible Solution

Handle the CJK table alignment through #55512/#55596 or an equivalent shared Markdown IR display-width fix with CJK and emoji regression tests, then make a separate Telegram-specific decision on whether prose softbreaks should be influenced by agent prompting, a renderer option, or no post-processing.

## Reproduction Assessment

Yes for the CJK table alignment: current main can be inspected through src/markdown/ir.ts, where code-mode table widths and padding use cell text length. The paragraph wall-of-text symptom is plausible from the softbreak path, but live Telegram rendering or a targeted fixture is still needed to define the exact expected policy.

## Solution Assessment

Unclear. A shared display-width fix is the right direction for CJK tables, but the proposed gateway-level conversion of every single newline to a blank line is not clearly the best solution because it may break valid Markdown structures.
