## Summary

Keep open: current main still uses code-unit padding for code-mode tables, while softbreak-to-paragraph promotion is an unresolved Telegram formatting policy separate from the open #55512/#55596 CJK work.

## What This Changes

The issue requests Telegram agent replies to preserve paragraph spacing and align CJK Markdown tables correctly.

## Best Possible Solution

Land or replace the #55596-style shared display-width fix for code-mode tables with CJK and emoji regression tests, then decide the Telegram prose softbreak policy separately in the formatter or agent prompt surface.

## Reproduction Assessment

Partially: yes for CJK tables, because current main’s code-mode table renderer can be inspected using `cell?.text.length` for width and padding. The paragraph wall-of-text symptom is plausible from the softbreak path, but needs a targeted Telegram rendering fixture or maintainer policy to define expected behavior.

## Solution Assessment

Unclear. A shared display-width helper is the narrow maintainable direction for CJK tables, but the proposed gateway-level single-newline promotion is not clearly the best solution because it can change Markdown semantics outside prose paragraphs.
