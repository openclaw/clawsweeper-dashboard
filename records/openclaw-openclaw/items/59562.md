## Summary

Current main still has the CJK table root cause, and the paragraph-break request is not defined enough to close or automate; this should stay open while maintainers decide the Telegram prose policy and track the CJK work through the existing related issue/PR.

## What This Changes

The issue asks for Telegram agent replies to add paragraph spacing for prose and to align CJK markdown tables correctly in code-mode rendering.

## Best Possible Solution

Land or replace the existing CJK display-width fix for the shared code-table renderer, then handle Telegram single-newline-to-paragraph promotion as a separate formatter or prompt policy with targeted tests.

## Reproduction Assessment

Partially yes: the CJK table bug is reproducible by source inspection because code-mode tables still use cell?.text.length for width and padding. The paragraph wall-of-text half is plausible from the softbreak path, but it needs a defined Telegram fixture or maintainer policy before it becomes a high-confidence bug reproduction.

## Solution Assessment

Unclear. Fixing display width inside renderTableAsCode is the narrow maintainable direction for CJK tables, but blanket promotion of every single newline to a paragraph break could change Markdown semantics and needs a Telegram-specific policy decision.
