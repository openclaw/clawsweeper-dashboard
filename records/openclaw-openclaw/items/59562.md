## Summary

Keep open: current main still uses code-unit padding for code-mode Markdown tables, and the paragraph-break request remains a separate Telegram formatting policy decision rather than an implemented fix.

## What This Changes

The issue asks for Telegram agent replies to preserve readable paragraph breaks and align CJK Markdown tables correctly.

## Best Possible Solution

Land or replace the open display-width fix for code-mode tables with CJK and emoji regression coverage, then decide the Telegram prose softbreak policy separately in the formatter or prompt surface.

## Reproduction Assessment

Partially yes: the CJK table half is source-reproducible because Telegram code-mode tables reach `renderTableAsCode`, which still measures and pads cells with `.length`. The paragraph wall-of-text half is plausible from the softbreak path, but current tests do not define single-newline promotion as expected behavior.

## Solution Assessment

Unclear for the whole issue. A shared display-width helper is the narrow fix for CJK tables, but gateway-level single-newline promotion needs a Telegram formatting policy decision before implementation.
