## Summary

Keep open. The PR targets a real remaining gap, since current main still only schedules gateway memory startup work for QMD, but the branch is not mergeable as-is: the new builtin-local branch is not reached by the real gateway sidecar scheduler, and the proposed probe path can poison the long-lived builtin manager after a transient local-provider initialization failure.

## What This Changes

The PR adds gateway startup logic and tests to prewarm builtin `memorySearch.provider = "local"` embeddings, and relaxes a provider-runtime test helper to accept equivalent OpenAI/OpenAI-Codex missing-auth model variants.

## Best Possible Solution

Rebase the branch onto current main, update the post-attach memory startup policy and tests so explicit builtin-local configs actually schedule prewarm, and make provider initialization retry-safe before probing the default manager. Preserve current QMD boot-sync behavior and explicitly decide whether startup may download the default local GGUF model or should require a cached/local model first.

## Reproduction Assessment

Yes. For the primary PR regression, configure `memory.backend = "builtin"` and `memorySearch.provider = "local"`; the post-attach scheduler returns before importing the startup helper, so the prewarm never runs. For the cache failure path, make local provider initialization reject during prewarm, then call `memory_search` later in the same process and the cached manager reuses the retained rejected initialization promise.

## Solution Assessment

No. Prewarming explicit local embeddings is the right target, but this patch misses the real gateway scheduling path and probes through a failure-poisonable cached manager; a scheduler update plus retry-safe provider initialization is the narrower maintainable solution.

  - body: For explicit local configs without `local.modelPath`, the local provider defaults to an `hf:` GGUF and the probe resolves/loads that model. `node-llama-cpp` documents `resolveModelFile` as a model download path for `hf:` files, so this can add startup-time artifact fetching despite the PR's no-network boundary. citeturn0search0.
