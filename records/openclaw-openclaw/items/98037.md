## Best Possible Solution

Normalize or buffer QQ Bot visible text before every outbound-visible decision, including streaming partials, static delivery, tool text, and silent-reply classification, then add focused regression tests and redacted real QQ Bot proof.

## Solution Assessment

  - confidence: 0.88

If the model echoes metadata before an intentional `NO_REPLY` or `[SKIP]`, this branch treats it as visible, strips the metadata later, and can send the control token to QQ instead of staying silent. Sanitize once before the silent check and reuse that normalized value for delivery.
  - confidence: 0.9
