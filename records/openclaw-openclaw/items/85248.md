The PR adds an OpenAI completions stream guard that removes `toolCall` blocks when the provider finish reason does not map to tool use, with regression tests for stop and tool-call finish reasons.

## Best Possible Solution

Land this only after maintainers explicitly accept the finish-reason authority rule and the contributor adds redacted patched OpenClaw runtime proof; keep the linked bug open until the fix is merged.

## Reproduction Assessment

## Solution Assessment

Mostly yes: using the final provider finish reason as the generic execution boundary is a narrow maintainable fix. The remaining question is compatibility policy for non-standard providers plus after-fix runtime proof.
