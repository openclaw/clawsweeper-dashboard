## What This Changes

The PR adds an OpenAI-completions stream guard that removes parsed tool-call blocks when the final stop reason is not tool use, plus regression tests for stop, tool-call, and no-tool-call stream cases.

## Best Possible Solution

Land the focused guard after maintainers accept finish_reason as authoritative for OpenAI-compatible tool use and the PR includes patched-gateway proof for both spurious stop/length streams and normal tool_calls streams.

## Reproduction Assessment

Yes. By source inspection, feeding current main a stream with delta.tool_calls and finish_reason "stop" leaves a final toolCall block, and the agent loop executes final message toolCall blocks.

## Solution Assessment

Yes, with a compatibility caveat. The guard is the narrowest code-path fix for the OpenAI-compatible finish-reason contract, but maintainers should explicitly accept the stricter behavior for non-conforming providers.
