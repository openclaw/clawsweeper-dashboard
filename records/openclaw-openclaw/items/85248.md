## Summary

Keep this PR open for a focused follow-up before merge. The boundary-aware gateway change is directionally correct and the new Docker gateway replay proof satisfies the real-behavior proof gate, but the same OpenAI-completions mismatch still appears in the registered sibling stream path.

## What This Changes

The PR strips accumulated OpenAI-compatible completions `toolCall` blocks when the provider finishes with a non-tool-use stop reason and adds regression tests for stop/tool-call combinations.

## Best Possible Solution

Land the stop-reason authority fix after applying the same guard to every active OpenAI-completions streaming parser, or documenting with code-level proof that the remaining registered parser cannot execute tools.

## Reproduction Assessment

Yes, source-reproducible. Current main builds `toolCall` blocks from streamed `delta.tool_calls`, and the agent loop executes any final `toolCall` content without requiring `stopReason: "toolUse"`; I did not run the live llama.cpp repro locally.
