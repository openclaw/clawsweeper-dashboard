## Summary

## What This Changes

The PR adds a final OpenAI-completions stream guard that removes parsed `toolCall` blocks when the mapped stop reason is not `toolUse`, plus mocked regression coverage.

## Best Possible Solution

## Solution Assessment

Mostly yes: treating provider `finish_reason` as authoritative matches the repository's OpenAI-compatible streaming contract and the patch preserves `tool_calls` finishes. The remaining gap is real behavior proof for the reported provider and an explicit maintainer call on compatibility risk.
