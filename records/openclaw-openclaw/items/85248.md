## What This Changes

The PR adds an OpenAI-completions stream guard that removes tool-call blocks unless the final provider stop reason maps to tool use, plus regression tests for stop, tool_calls, and text-only streams.

## Best Possible Solution

Land the focused guard after maintainers accept finish_reason as authoritative for OpenAI-compatible tool use and the PR includes redacted patched-gateway proof for both spurious stop/length streams and normal tool_calls streams.

## Reproduction Assessment

## Solution Assessment

Yes, with a compatibility caveat. The guard is the narrowest fix for the documented finish_reason contract, but maintainers should explicitly accept the stricter behavior for non-conforming providers.
