## Summary

Keep open: the patch is focused and appears functionally correct, but this external bug-fix PR still has mock-only proof and intentionally makes OpenAI-compatible `finish_reason` authoritative for tool execution.

## What This Changes

The PR filters `toolCall` content blocks from OpenAI-compatible completions stream output when parsed tool-call deltas finish with a non-tool-use stop reason and adds three mocked stream tests.

## Best Possible Solution

Land the guard with the regression tests after redacted live-provider proof and explicit maintainer acceptance of the finish-reason compatibility tradeoff.

Yes at source level: current main appends `delta.tool_calls` even when `finish_reason` maps to `stop`, and the PR's mocked stream exercises that exact parser path. I did not independently run the live llama.cpp/Qwen scenario.

## Solution Assessment

Mostly yes: treating provider `finish_reason` as authoritative matches the OpenAI SDK contract and the patch preserves `tool_calls` finishes. The remaining gap is real behavior proof for the reported provider and a maintainer call on compatibility risk.
