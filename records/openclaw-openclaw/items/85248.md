## Summary

## What This Changes

The PR adds stop-reason guards and regression tests to the gateway OpenAI-completions transport parser and registered OpenAI-compatible provider parser so accumulated `toolCall` blocks are kept only when `finish_reason` maps to `toolUse`.

## Best Possible Solution

Land the guard only after a maintainer explicitly accepts the OpenAI-compatible provider contract that structured tool calls require a tool-use finish reason, keeping the mirrored parser coverage and regression tests.
