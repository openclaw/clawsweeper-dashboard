## Summary

Keep open: the patch shape looks correct, but merge still needs maintainer acceptance of the stricter OpenAI-compatible finish_reason contract and stronger after-fix proof from a patched OpenClaw gateway run.

## What This Changes

The PR changes OpenAI-compatible completions streaming to remove tool-call blocks when the final provider stop reason is not tool use, and adds regression tests for stop, tool_calls, and text-only streaming cases.

## Best Possible Solution

Land the focused guard after maintainers accept finish_reason as authoritative for tool use and the PR includes redacted patched-gateway proof for both spurious stop/length streams and normal tool_calls streams.

## Reproduction Assessment

Yes. Source inspection shows current main can leave final toolCall blocks when delta.tool_calls arrives with finish_reason stop, and the agent loop executes final toolCall blocks; I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes, with a compatibility caveat. The guard is the narrowest code fix for the documented finish_reason contract, but maintainers should explicitly accept the stricter behavior for non-conforming providers.
