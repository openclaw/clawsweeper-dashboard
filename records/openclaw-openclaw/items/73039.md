The PR adds a WeakMap-backed memoizeToolFactory helper and wraps image, media generation, PDF, web_search, and web_fetch factory construction in createOpenClawTools with unit tests.

## Best Possible Solution

Keep the memoization direction, but rebase it around current optional-media factory planning and preserve execute-time web runtime policy with focused regression coverage.

## Reproduction Assessment

Yes for the patch regressions: source comparison against current main shows the cached media factories bypass current factory planning and cached web tools omit lateBindRuntimeConfig. I did not rerun the original latency benchmark in this read-only review.

## Solution Assessment

No, not in the current branch shape. Factory memoization may be the right fix, but it needs to preserve current policy/auth/deferred-resolution behavior and execute-time web runtime binding.
