## Summary

Keep open: the performance direction is useful, but the current branch is not safe to merge because it bypasses current media factory planning and drops execute-time web runtime binding for cached web tools.

## What This Changes

The PR adds a `memoizeToolFactory` helper and wraps image, media generation, PDF, web search, and web fetch tool construction in `createOpenClawTools`.

## Best Possible Solution

Rebase the memoization behind current media availability planning and web late-binding contracts, then validate with focused tests and fresh redacted timing proof.

## Reproduction Assessment

Yes for the merge blockers: PR-head source compared with current main shows direct optional media/PDF factory construction and missing web late-binding. I did not rerun the original live latency benchmark in this read-only pass.

## Solution Assessment

No, not in the current branch shape. Factory memoization may be the right optimization, but it needs to preserve current auth/policy media planning and execute-time web runtime policy binding.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve optional media factory planning:** `src/agents/openclaw-tools.ts:167-184`
  - body: This branch constructs optional media/PDF tools directly through `memoizeToolFactory` instead of current main's `resolveOptionalMediaToolFactoryPlan` and image availability checks. That drops `authProfileStore`/deferred model resolution and can build tools that policy or auth says should be unavailable, putting denied or unavailable media tools back on the hot path.
  - confidence: 0.92
- **[P2] Keep web tools late-bound at execution:** `src/agents/openclaw-tools.ts:255-276`
  - body: The memoized `web_search` and `web_fetch` calls do not pass `lateBindRuntimeConfig: true`. Current web tools rely on that flag to reread active runtime metadata and secrets config at execute time, so a cached tool can keep stale provider, disablement, proxy, or fetch-policy behavior after runtime changes.
  - confidence: 0.91
