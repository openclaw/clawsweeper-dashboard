## Summary

Keep open: the optimization is useful and still relevant, but the branch is not mergeable as-is because it predates current main's auth-aware optional-media planning and late-bound web runtime contract.

## What This Changes

The PR adds a memoizeToolFactory helper and wraps selected media, PDF, web search, and web fetch tool factory construction in createOpenClawTools.

## Best Possible Solution

Rebase the optimization onto current main, memoize only stable expensive factory work after the optional-media plan, and preserve late-bound web runtime semantics with focused regression and performance proof.

## Reproduction Assessment

Yes. The patch-level blockers are source-reproducible by comparing PR head against current main's auth-aware media planner, PDF deferral, and late-bound web runtime path; I did not run a live latency benchmark.

## Solution Assessment

No. Process-lifetime memoization is plausible, but this branch needs to be rebased around the current media availability and web runtime contracts before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep the auth-aware media factory plan:** `src/agents/openclaw-tools.ts:167-184`
  - body: Current main resolves optional image/video/music/PDF availability with `authProfileStore` plus allow/deny lists, then passes `authProfileStore` and deferred auto-resolution into the factories. This branch wraps the old direct factory calls, so a rebase would construct policy-denied tools and lose auth-profile-only provider/PDF deferral behavior. Put memoization around the current plan instead.
  - confidence: 0.91
- **[P2] Keep web tools late-bound when caching them:** `src/agents/openclaw-tools.ts:255-276`
  - body: Current main passes `lateBindRuntimeConfig: true` so `web_search` and `web_fetch` read active runtime metadata and secrets at execution time. This branch caches tools using the cloned metadata object and omits late binding, which either prevents warm cache hits or freezes stale runtime state. Key on a stable runtime provider/config fingerprint or leave these factories uncached.
  - confidence: 0.9
