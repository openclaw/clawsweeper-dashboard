## Summary

Keep open: the performance direction is useful, but the current PR is not mergeable because it bypasses current main's media/PDF factory-planning contract and drops late-bound runtime config for cached web tools.

## What This Changes

The PR adds a `memoizeToolFactory` helper, tests its cache-key behavior, and wraps image, media generation, PDF, `web_search`, and `web_fetch` factory calls in `createOpenClawTools`.

## Best Possible Solution

Keep the memoization direction, but rebase it onto current main so caching wraps the current optional-media planning path and preserves execute-time web runtime policy binding with focused regression coverage.

## Reproduction Assessment

Yes for the merge blockers: source comparison against current main shows the cached media factories bypass current factory planning, and cached web tools omit `lateBindRuntimeConfig`. I did not rerun the original latency benchmark in this read-only review.

## Solution Assessment

No, not in the current branch shape. Factory memoization may be the right solution, but it needs to preserve current policy/auth/deferred-resolution behavior and execute-time web runtime binding.
