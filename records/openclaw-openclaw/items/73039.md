## Summary

Keep open: the performance work is plausibly useful and related latency reports are real, but the current branch is not safe to merge because it predates current main’s auth-aware optional media planning and web-tool late-binding contracts.

## What This Changes

The branch adds a memoizeToolFactory helper, tests, and memoizes selected image, media-generation, PDF, web search, and web fetch tool factory construction in createOpenClawTools.

## Best Possible Solution

Rebase the memoization onto current main, cache only stable expensive work, and preserve optional-media planning plus web late-binding semantics with focused regression coverage and fresh timing proof.

## Reproduction Assessment

Yes. The patch-level problems are source-reproducible by comparing PR head against current main’s media factory planner and web runtime context; I did not run a live latency benchmark.

## Solution Assessment

No. Process-lifetime memoization is a reasonable direction, but this branch must be reshaped around current main’s auth/profile availability gates and stable web runtime invalidation before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the auth-aware media factory plan:** `src/agents/openclaw-tools.ts:167-184`
  - body: Current main gates optional media/PDF factories through resolveOptionalMediaToolFactoryPlan with authProfileStore plus allow/deny lists, then forwards authProfileStore and deferred auto-resolution into the factories. This branch memoizes and constructs the generation factories directly, so after rebase auth-backed providers can disappear and denied or unavailable tools can still be built on the hot path.
  - confidence: 0.9
- **[P2] Keep web tools late-bound with a stable cache key:** `src/agents/openclaw-tools.ts:255-276`
  - body: Current main passes lateBindRuntimeConfig: true so web_search and web_fetch read active runtime secrets/config when the tool executes. This branch keys on the cloned runtimeWebTools object and omits lateBindRuntimeConfig, which either defeats warm caching on active-runtime turns or freezes stale runtime metadata; use a stable runtime provider/version fingerprint while keeping execute-time lookup, or leave these tools un-memoized.
  - confidence: 0.88
