Keep open: the performance direction is useful, but the latest branch is not mergeable because it would regress current-main optional media gating, deferred auth/model resolution, and execute-time web runtime binding.

## What This Changes

The PR adds a process-lifetime `memoizeToolFactory` helper and wraps image, media-generation, PDF, web search, and web fetch factory construction in `createOpenClawTools` with unit tests.

## Best Possible Solution

Keep the PR open and rebase its memoization onto current main's optional-media planner and execute-time web runtime binding, then validate with targeted tests and a fresh timing run.

## Reproduction Assessment

Yes. The merge blockers are source-reproducible by comparing PR head tool construction against current main's optional-media plan and web runtime context; I did not run a live latency benchmark.

## Solution Assessment

No. Process-lifetime memoization is a reasonable optimization, but this branch is not the best merge shape until it preserves current main's auth-aware factory planning and late-bound web execution semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve optional media factory planning:** `src/agents/openclaw-tools.ts:167-177`
  - body: Current main only builds optional image, video, music, and PDF factories after `resolveOptionalMediaToolFactoryPlan` applies runtime availability, auth-store, allowlist, and denylist state. This branch starts constructing `image_generate` unconditionally, and repeats that pattern for video/music/PDF, so denied or unavailable tools can still be built on the hot path and the factories miss current `authProfileStore` and deferred-resolution inputs.
  - confidence: 0.9
- **[P2] Keep web tools late-bound at execution:** `src/agents/openclaw-tools.ts:260-275`
  - body: Current main passes `lateBindRuntimeConfig: true` so `web_search` and `web_fetch` re-read active runtime metadata and secrets/config when `execute` runs. The memoized tools close over the construction-time snapshot and omit that flag, so provider, disablement, secret, and fetch-policy changes can be missed; keying on `runtimeWebTools` also misses warm-cache hits because that value is cloned per read.
  - confidence: 0.88
