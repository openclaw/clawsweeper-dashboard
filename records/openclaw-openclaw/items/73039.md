## Summary

Keep open: the performance goal is valid, but this PR cannot merge in its current shape because its cached factories bypass current optional-media planning and omit late-bound web runtime config.

## What This Changes

The PR adds a memoization helper and wraps media, PDF, web search, and web fetch factory construction inside `createOpenClawTools` to reduce repeated per-turn setup cost.

## Best Possible Solution

Rebase the memoization around current optional-media planning and late-bound web runtime config, add regression coverage for both contracts, and refresh real timing proof after the repair.

## Reproduction Assessment

Yes for the patch regressions: source comparison shows PR head bypasses current-main optional-media planning and omits `lateBindRuntimeConfig` for cached web tools. I did not rerun the original latency benchmark in this read-only review.

## Solution Assessment

No, not as currently written. Factory memoization may be the right direction, but it must be applied after current policy/auth/availability planning and must preserve execute-time web runtime binding.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve optional-media planning before memoizing factories:** `src/agents/openclaw-tools.ts:167-253`
  - body: Current main first resolves the optional-media plan and image availability, then passes `authProfileStore` and deferred model-resolution flags into the factories. This branch memoizes direct media/PDF factory calls, so denied or unavailable tools can still be constructed and PDF/image registration can redo eager model/auth work.
  - confidence: 0.92

  - confidence: 0.91
