## Best Possible Solution

Rebase the memoization onto current main so cached factories are gated by optional-media planning and web tools remain execute-time runtime-bound, with focused regression tests and refreshed timing proof.

## Solution Assessment

No, not as-is. Factory memoization may be the right direction, but the current branch needs to preserve current policy/auth/deferred-resolution behavior and late-bound web runtime policy before merge.
