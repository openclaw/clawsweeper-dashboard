## Summary

Keep open: the latency target is valid, but this branch is not safe against current main because it bypasses current optional-media planning and drops execute-time web runtime binding.

## What This Changes

The PR adds a memoizeToolFactory helper, unit tests, and cached media/PDF/web tool construction inside createOpenClawTools.

## Best Possible Solution

Rebase the memoization onto current optional-media planning and web late-binding, with cache keys or cached artifacts that preserve auth, policy, runtime config, sandbox, and delivery-context contracts.

## Reproduction Assessment

Yes for the patch regressions: source comparison against PR head shows current-main media planning is bypassed and web tools lose lateBindRuntimeConfig. I did not rerun the original latency benchmark in this read-only review.

## Solution Assessment

No, not in the current branch shape. Memoizing factory work may be a maintainable fix, but it needs to be layered around the current plan and execute-time runtime binding rather than replacing them.
