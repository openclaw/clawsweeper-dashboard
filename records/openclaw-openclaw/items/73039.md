## Summary

Keep open. The optimization remains useful and is not implemented on current main, but the branch needs changes because it bypasses current optional-media factory planning and caches web tools without execute-time runtime binding.

## What This Changes

The PR adds a memoizeToolFactory helper and uses it to cache image, media generation, PDF, web_search, and web_fetch tool factory results in createOpenClawTools.

## Best Possible Solution

Keep the memoization direction, but rebase it onto current main so caching wraps the current optional-media planning path and preserves late-bound web runtime policy with focused regression coverage.

## Reproduction Assessment

Yes for the merge blockers: source comparison against current main shows the cached media factories bypass current factory planning, and cached web tools omit lateBindRuntimeConfig. I did not rerun the original latency benchmark in this read-only review.

## Solution Assessment

No, not in the current branch shape. Factory memoization may be the right fix, but it needs to preserve current policy/auth/deferred-resolution behavior and execute-time web runtime binding.
