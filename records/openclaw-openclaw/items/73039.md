## Summary

Keep open. The performance goal is still relevant, but this branch is stale against current main and would regress current media-tool auth/allowlist gating plus runtime web-search late binding, so it should not merge as-is.

## What This Changes

This PR adds a memoizeToolFactory helper and wraps image, media generation, PDF, web search, and web fetch factory construction inside createOpenClawTools, with unit coverage for the cache helper.

## Best Possible Solution

Memoize only the stable current-main factory boundaries while preserving auth/profile availability, tool allowlists, sandbox keys, runtime web late-binding, focused regression tests, and a single-line changelog entry.

## Reproduction Assessment

Yes. The web-cache miss reproduces by setting active runtime web metadata and observing that getActiveRuntimeWebToolsMetadata returns a new clone per read; the media regression is covered by current media factory plan tests that assert auth/profile and allowlist-sensitive registration.

## Solution Assessment

No. Process-lifetime memoization is a plausible optimization, but this branch applies it around an older createOpenClawTools shape; the safer repair is to rebase onto current main and memoize around the existing availability planner with stable runtime invalidation.
