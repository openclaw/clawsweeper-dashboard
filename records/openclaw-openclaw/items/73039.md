## Summary

Keep open: the optimization is useful in intent, but this branch is not safe to merge because it bypasses current main's optional-media factory plan and drops execute-time web runtime binding for network-capable tools.

## What This Changes

The PR adds a process-lifetime memoization helper and wraps image, media-generation, PDF, web search, and web fetch factory creation in `createOpenClawTools`, with unit tests for cache identity and invalidation.

## Best Possible Solution

Rebase the memoization onto current main's factory planner and late-bound web runtime contract, then prove both policy correctness and warm-turn latency improvement.

## Reproduction Assessment

Yes. The merge blockers are source-reproducible by comparing PR-head construction against current main's optional-media planner and web runtime context; I did not run a live latency benchmark.

## Solution Assessment

No. Process-lifetime memoization may be a reasonable optimization, but this branch is not the best merge shape until it preserves current main's auth-aware media gating and execute-time web policy binding.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve optional media factory planning:** `src/agents/openclaw-tools.ts:167-183`
  - body: Current main only builds optional image, video, music, and PDF factories after runtime availability, auth-store, allowlist, and denylist planning. This branch constructs the optional factories directly through the memoizer, so denied or unavailable tools can still run their registration work and the factories miss current `authProfileStore` and deferred-resolution inputs.
  - confidence: 0.9
- **[P2] Keep web tools late-bound at execution:** `src/agents/openclaw-tools.ts:260-275`
  - body: Current main passes `lateBindRuntimeConfig: true` so `web_search` and `web_fetch` re-read active runtime metadata and secrets/config during `execute`. The cached tools here close over the construction-time snapshot and omit that flag, so provider, disablement, secret, proxy, and fetch-policy changes can be missed.
  - confidence: 0.88
- **[P3] Key runtime web metadata without cloned refs:** `src/agents/openclaw-tools.ts:255-269`
  - body: `getActiveRuntimeWebToolsMetadata()` returns a fresh structured clone on each read, but this branch uses that object in the memoization refs. When runtime web metadata is active, the web-tool cache will miss every turn; use a stable scalar fingerprint if the descriptor must vary, or keep runtime-dependent state late-bound.
  - confidence: 0.82
