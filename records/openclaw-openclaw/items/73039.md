## Summary

Keep open: the performance optimization is plausibly useful, but the current branch is not mergeable against current main because it bypasses recently added optional-media planning and web runtime late-binding contracts.

## What This Changes

The PR adds a memoizeToolFactory helper and wraps image, media generation, PDF, web search, and web fetch tool construction in createOpenClawTools.

## Best Possible Solution

Rebase or replace the branch so memoization sits behind current main's media availability plan and preserves execute-time web runtime binding, then validate both the correctness contracts and the latency improvement.

## Reproduction Assessment

Yes for the merge blockers: PR-head source shows direct media factory construction and omitted web late-binding, while current main shows the contracts that would be lost. I did not reproduce the reported latency improvement with a live benchmark in this read-only pass.

## Solution Assessment

No, not in the current branch shape. Memoizing these factories may be the right optimization, but the implementation must preserve current main's auth-aware factory planning and late-bound web runtime policy.

- **[P2] Preserve current media factory planning:** `src/agents/openclaw-tools.ts:167-253`
