## Summary

Keep this PR open, but it still needs changes before merge: the performance direction is useful and not implemented on current main, while the branch is now behind current tool-planning and runtime-web contracts.

## What This Changes

The branch adds a memoizeToolFactory helper and wraps selected image, media-generation, PDF, web search, and web fetch factory construction in createOpenClawTools with tests for cache behavior.

## Best Possible Solution

Rebase the memoization onto current main so it caches only stable expensive factory work while preserving optional-media availability planning, allow/deny policy, auth-profile reuse, deferred model resolution, and execute-time web runtime binding.

## Reproduction Assessment

Yes. The patch-level problems are source-reproducible by comparing PR head call sites with current main's optional-media planner and web runtime context; I did not run a live latency benchmark.

## Solution Assessment

No. Process-lifetime memoization is a reasonable performance direction, but this branch is not the best merge shape until it is rebased around current main's auth-aware factory planning and late-bound web execution semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve optional media factory planning:** `src/agents/openclaw-tools.ts:167-184`
  - body: Current main only builds optional image-generation, video-generation, music, and PDF tools after resolveOptionalMediaToolFactoryPlan has applied auth-profile availability plus allow/deny policy. This branch constructs the memoized factories directly, so rebasing it as-is would reintroduce hot-path work for tools current main intentionally skips and would drop the authProfileStore/deferred-resolution inputs those factories now rely on.
  - confidence: 0.9
- **[P2] Keep web tools late-bound at execution:** `src/agents/openclaw-tools.ts:260-264`
  - body: Current main passes lateBindRuntimeConfig: true so web_search and web_fetch read the active runtime secrets/config when execute runs. The memoized call sites capture only the current runtimeWebTools clone and omit lateBindRuntimeConfig, so a cached tool can miss later runtime provider/config changes; when runtime metadata is present, the cloned object identity also defeats warm cache hits.
  - confidence: 0.88
