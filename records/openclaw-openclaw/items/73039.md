## Summary

Keep open: this is a plausible performance fix for a still-active tool-prep latency family, but current main only partially overlaps it and the branch is stale/conflicted against newer factory-planning and runtime-web changes.

## What This Changes

The branch adds a `memoizeToolFactory` helper plus unit tests and wraps image, media-generation, PDF, and web tool factory calls in `createOpenClawTools` with process-lifetime cache keys.

## Best Possible Solution

Rebase the memoization idea onto current main's factory planning and runtime contracts, then keep only cache boundaries with stable keys and fresh cold/warm embedded-run timing proof.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. The PR body and related issues provide real timing logs, and current main source still shows request-time construction of the targeted built-in tools, so the performance problem is source-reproducible enough to keep open.

## Solution Assessment

Unclear as-is. Whole-tool memoization may be useful, but the best version must preserve current main's newer allow/deny planning, auth-store facts, deferred model resolution, and late-bound web runtime behavior.
