## Summary

Keep open. Current main still does not contain the proposed process-lifetime tool-factory memoization and related issue #75290 shows the latency problem remains relevant, but the PR head is stale against current main's auth-aware availability and runtime web-tool handling and has blocking findings.

## What This Changes

The PR adds a `memoizeToolFactory` helper with unit tests and wraps image, media generation, PDF, web search, and web fetch factory construction inside `createOpenClawTools`.

## Best Possible Solution

Rebase the optimization onto current main, preserve the existing auth/profile and allowlist availability gates, use stable or versioned runtime invalidation for web tools, and add focused regression tests plus an Unreleased changelog entry.

## Reproduction Assessment

Yes. The web cache miss reproduces by setting active runtime web metadata and calling `getActiveRuntimeWebToolsMetadata()` twice, which returns distinct clone identities; the media regression is covered by current `createOpenClawTools` tests that require `authProfileStore` and `pluginToolAllowlist` to drive registration.

## Solution Assessment

No. Process-lifetime memoization is a plausible optimization, but this branch applies it on top of an older factory-registration shape; the safer fix is to memoize around current main's availability planner or move the expensive registry work into a lower-level, explicitly invalidated cache.
