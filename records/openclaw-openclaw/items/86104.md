## Summary

Keep open. Current main still only detects legacy `ai.openclaw.update.*` stale updater jobs, while the report is about `ai.openclaw.oob-update.*` `launchctl submit` relaunch loops; source shows the built-in updater avoids `launchctl submit`, but the reported diagnostic namespace is not covered.

## What This Changes

The issue requests avoiding `launchctl submit` for macOS one-shot updater jobs and expanding documentation or diagnostics for out-of-band updater relaunch loops.

## Best Possible Solution

Keep the built-in updater off `launchctl submit`, then decide whether to officially recognize OpenClaw-namespaced out-of-band update jobs in status/doctor guidance and document the supported one-shot LaunchAgent pattern.
