## Summary

Close as outside the OpenClaw source repo: the reported relaunch loop came from a local out-of-band wrapper, while current main and v2026.5.22 do not create `launchctl submit` or `ai.openclaw.oob-update.*` jobs. The remaining request would make core own external wrapper cleanup or change documented same-version update behavior, which needs a new proven OpenClaw-owned producer or explicit maintainer policy.

## What This Changes

The issue asks OpenClaw to avoid or document `launchctl submit` for macOS one-shot update wrappers and possibly broaden stale updater diagnostics beyond legacy `ai.openclaw.update.*` labels.

## Best Possible Solution

Keep OpenClaw's shipped updater on the existing managed handoff path, and leave external one-shot wrapper lifecycle guidance outside core unless a shipped OpenClaw producer is proven.
