## Summary

Keep open: the bug is still present on current main and this PR is useful source work, but the patch is not ready because it targets the old `src/discord` layout and bypasses based on config/channel mode rather than the effective per-session queue mode.

## What This Changes

The PR changes legacy Discord message handling to bypass the inbound per-session worker queue for config-level `steer` and `steer-backlog`, with a same-session concurrency regression test.

## Best Possible Solution

Land a current-main Discord plugin fix that computes the effective per-session queue mode before enqueuing and bypasses only steering modes, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows same-session Discord messages share a session-scoped `job.queueKey` and enter `createChannelRunQueue`, which serializes per key before the reply runtime can apply steering behavior.

## Solution Assessment

No. The direction is right, but the proposed patch is not the best landing shape because it uses config/channel mode instead of the effective per-session mode and targets the legacy `src/discord` path rather than `extensions/discord`.
