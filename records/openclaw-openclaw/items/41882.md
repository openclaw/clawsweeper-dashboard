## Summary

Keep open. The PR targets a real current-main Discord queue bug, but the patch is not correct as-is because it edits the old `src/discord` path and decides the bypass from config-level mode instead of the documented effective queue mode.

## What This Changes

The PR changes the old Discord inbound worker to bypass its per-session queue for `steer` and `steer-backlog` modes and adds same-session concurrency regression coverage.

## Best Possible Solution

Land a focused replacement or updated PR in the active Discord plugin that resolves the effective queue mode with documented session precedence and bypasses the outer queue only for `steer` or `steer-backlog`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: same-session Discord messages share a session-scoped queue key, `KeyedAsyncQueue` serializes that key, and the current queue test asserts the second run waits for the first.

## Solution Assessment

No. The PR direction is useful, but the patch is not the best current-main solution because it targets the old Discord layout and bypasses from config-level mode rather than the effective session-aware mode.

## Review Findings
