## Summary

Keep open. The Discord steer queue bug is still source-reproducible on current main, but this PR is not merge-ready because it targets the old `src/discord` path, lacks after-fix real Discord proof, and its bypass does not honor per-session queue overrides.

## What This Changes

The PR bypasses Discord's inbound same-session run queue for configured steer queue modes and adds regression coverage for same-session concurrency.

## Best Possible Solution

Land a rebased Discord plugin fix that lets effective `steer` mode reach the reply steering path without weakening per-session serialization for non-steer modes, with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes. By source inspection, current main and `v2026.5.7` enqueue accepted Discord inbound jobs into a session-keyed run queue that serializes same-key work before inner steering can run.

## Solution Assessment

No. Bypassing the outer queue is the right direction, but this PR should target the active `extensions/discord` implementation and decide from the effective session-aware queue mode, not only global Discord config.
