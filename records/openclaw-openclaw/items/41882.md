## Summary

Keep open: the underlying Discord steer-mode bug is still present on current main, and the PR is directionally useful but not merge-ready because it targets the old Discord layout, ignores effective session-level queue overrides, and lacks real behavior proof.

## What This Changes

The PR bypasses the Discord inbound per-session run queue for config-level `steer`/`steer-backlog` modes and adds mocked same-session concurrency coverage in the old `src/discord` layout.

## Best Possible Solution

Land a rebased Discord plugin fix in `extensions/discord` that decides bypass from the effective queue mode, covers `steer` and `steer-backlog` plus serialized override cases, and includes regression coverage with redacted real behavior proof.

## Reproduction Assessment

Yes from source inspection. Current main enqueues every accepted Discord job into the session-scoped run queue, and that queue serializes same-key work before inner steer handling can run.

## Solution Assessment

No. Bypassing the outer queue is the right direction, but the PR should resolve the effective session-aware queue mode and be rebased into `extensions/discord` before merge.
