## Summary

Keep this PR open. The reported Discord queue bug still exists on current main, but the branch is stale against the active Discord plugin path and the proposed bypass still needs to honor the effective queue mode, including persisted session overrides, before it can merge safely.

## What This Changes

The PR modifies the legacy Discord message handler to bypass its per-session queue for config-level steer and steer-backlog modes and adds a same-session concurrency regression test.

## Best Possible Solution

Implement the fix in the current Discord plugin queue path: resolve the effective queue mode early enough to bypass the outer per-session queue only for steer and steer-backlog, keep non-steer modes serialized, and cover global config, by-channel config, and session override precedence in focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: same-session Discord messages share job.queueKey, createChannelRunQueue serializes that key, and current tests assert the second same-session run waits for the first.

## Solution Assessment

No. The proposed direction is right, but this branch targets the old Discord path and resolves only config-level queue mode instead of the effective mode that downstream reply execution uses.
