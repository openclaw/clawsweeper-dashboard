This PR changes the Discord inbound message handler to bypass the per-session run queue for global `steer`/`steer-backlog` queue modes and adds mocked same-session concurrency coverage under the old `src/discord` path.

## Best Possible Solution

Land a rebased Discord plugin fix that bypasses the outer queue only for the effective `steer`/`steer-backlog` mode after session and inline overrides, with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main and `v2026.5.7` enqueue accepted Discord inbound jobs into a session-keyed run queue that serializes same-key work before inner steering can run. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

No. Bypassing the outer queue is the right direction, but this PR should target the active `extensions/discord` implementation and base the bypass on the same effective session-aware queue mode used by reply execution.
