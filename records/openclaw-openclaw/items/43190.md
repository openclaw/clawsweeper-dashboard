## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve channel-data tenant in archive metadata:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: Channel activities can leave `conversation.tenantId` unset, but this archive later uses stored tenant metadata to choose Graph credentials for cleanup. Populate provider metadata from `activity.channelData.tenant.id` before falling back to `conversation.tenantId`, or cross-tenant archives can be checked and pruned with the wrong tenant context.
  - confidence: 0.9
- **[P2] Preserve MIME slots before hook delivery:** `extensions/msteams-channel-archive/index.ts:111`
  - body: The archive handler preserves empty MIME slots, but the hook mapper has already filtered empty `MediaTypes` entries before this point. Mixed attachments with missing MIME values can shift later MIME types onto the wrong file, breaking attachment metadata and MIME searches; preserve cardinality when deriving hook metadata.
  - confidence: 0.88
- **[P2] Declare startup activation and tool contracts:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: This plugin registers a startup cleanup service and four runtime tools, but its manifest omits explicit `activation.onStartup` and `contracts.tools`. The startup planner and tool discovery rely on that metadata, so the plugin may not load when expected and registered tools can be rejected by the manifest contract gate.
  - confidence: 0.95
- **[P2] Use guarded fetch for Graph cleanup:** `extensions/msteams-channel-archive/src/channel-cleanup.ts:302`
  - body: The cleanup service performs token and Graph requests with raw `fetch()`, including bearer-token calls. Route these requests through `fetchWithSsrFGuard()` or an existing Teams Graph wrapper so authenticated plugin networking keeps the repo's outbound safeguards.
  - confidence: 0.93
- **[P2] Complete the Teams handler test doubles:** `extensions/msteams/src/monitor-handler.file-consent.test.ts:94-109`
  - body: The new test double no longer satisfies the current `MSTeamsAdapter` and `MSTeamsConversationStore` contracts because it omits `updateActivity`, `deleteActivity`, and `findPreferredDmByUserId`. Add those methods so the regression test compiles against the current Teams interfaces.
  - confidence: 0.92
