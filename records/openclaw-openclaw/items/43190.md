## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Declare explicit startup activation:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: Bundled plugin metadata requires every manifest to declare `activation.onStartup` as a boolean, and the current CI run fails because this new manifest leaves it undefined. Add an explicit activation block with the intended startup behavior so discovery and contract checks agree.
  - confidence: 0.95
- **[P1] Preserve the channel-data tenant in archive metadata:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: Channel activities often leave `conversation.tenantId` unset, and this file already computes `tenantId` from `activity.channelData.tenant.id` for the stored conversation reference. The archive metadata uses only `conversation?.tenantId`, so cross-tenant archives can later be checked with the default tenant token and pruned as deleted even when the channel is active.
  - confidence: 0.9
- **[P2] Keep MIME arrays aligned before archiving:** `extensions/msteams-channel-archive/index.ts:111`
  - body: This handler tries to preserve empty MIME slots, but `metadata.mediaTypes` has already been produced by `deriveInboundMessageHookContext`, which filters empty strings out of `MediaTypes`. For mixed attachments, later MIME values shift left before they reach this line, so archived attachment metadata and MIME queries become wrong.
  - confidence: 0.86
