## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve the Teams channel-data tenant:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: `ProviderMetadata` uses `conversation?.tenantId`, but current Teams code treats `channelData.tenant.id` as canonical because channel activities often omit `conversation.tenantId`. The archive store persists this tenant and cleanup later chooses the Graph token from it, so cross-tenant archives can be checked with the default tenant and active channels can be pruned as deleted.
  - confidence: 0.88
- **[P2] Route Graph cleanup requests through guarded fetch:** `extensions/msteams-channel-archive/src/channel-cleanup.ts:302`
  - body: The cleanup service calls global `fetch` directly for Azure OAuth and Graph requests carrying credentials. Existing Teams Graph and OAuth helpers route these requests through `fetchWithSsrFGuard` with the channel's network boundary, timeout, and diagnostics, so this new path should reuse those helpers instead of creating an unguarded authenticated fetch stack.
  - confidence: 0.86
- **[P2] Declare the archive tools in the manifest:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: `tools.ts` registers four archive tools, but the manifest has no `contracts.tools` declaration. Current plugin loading rejects or diagnoses runtime tools that are not declared in the manifest ownership contract, so these tools will not be discoverable or reliably registered until the manifest lists them.
  - confidence: 0.9
