Overall confidence: 0.9

Full review comments:

- **[P1] Preserve channel-data tenant in archive metadata:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: Channel activities often leave `conversation.tenantId` unset, while this PR later uses archived tenant metadata to choose Graph credentials for cleanup. Populate this from `activity.channelData.tenant.id` before falling back to `conversation.tenantId` so cross-tenant archives are not checked or pruned with the wrong tenant token.
  - confidence: 0.9
- **[P2] Preserve MIME slots before archiving:** `extensions/msteams-channel-archive/index.ts:111`
  - body: This handler asks `readStringArray(..., { preserveEmpty: true })` to keep MIME indexes aligned, but `metadata.mediaTypes` has already had empty entries filtered out by the hook mapper. Preserve `MediaTypes` cardinality before building hook metadata, otherwise mixed attachments can be archived with shifted MIME values and MIME searches return wrong metadata.
  - confidence: 0.87
- **[P2] Declare startup activation explicitly:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: Bundled plugin manifests must intentionally declare `activation.onStartup`, and the new manifest only contains `id` and `configSchema`. Add the intended activation block so manifest discovery and the bundled metadata contract can pass.
  - confidence: 0.97
- **[P2] Add the archive plugin to the hook allowlist:** `extensions/msteams-channel-archive/index.ts:65`
  - body: This new plugin registers typed hooks, but the bundled hook-registration contract uses an explicit allowlist and currently rejects this file as an unexpected registration site. Add the file and expected hook names to the allowlist, or change the registration strategy if the plugin should remain lazy.
  - confidence: 0.9
- **[P2] Use guarded fetch for Graph cleanup:** `extensions/msteams-channel-archive/src/channel-cleanup.ts:302`
  - body: The cleanup service performs token and Graph requests with raw `fetch()`, and CI's channel/plugin network guard flags those calls. Route these requests through the repo's guarded fetch helper or an existing SDK/channel wrapper so authenticated plugin networking stays inside the expected boundary.
  - confidence: 0.92
- **[P2] Complete the Teams handler test doubles:** `extensions/msteams/src/monitor-handler.file-consent.test.ts:94-103`
  - body: `check-test-types` fails because the new test doubles no longer satisfy the current `MSTeamsAdapter` and `MSTeamsConversationStore` interfaces. Add the missing adapter and conversation-store methods so the regression test compiles against current Teams contracts.
  - confidence: 0.94
- **[P3] Use non-mutating sorts in archive store:** `extensions/msteams-channel-archive/src/archive-store.ts:238`
  - body: The repo lint gate rejects `Array#sort()` in this new archive store. Switch these ordered paths to `toSorted()` or another accepted non-mutating ordering helper so the lint lane can pass.
  - confidence: 0.9
