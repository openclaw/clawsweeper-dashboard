Full review comments:

- **[P2] Declare startup activation explicitly:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: Bundled plugin manifests must intentionally declare `activation.onStartup`, and the current CI run fails because this new manifest leaves it undefined. Add the intended activation block so discovery and bundled metadata checks agree.
  - confidence: 0.97
- **[P2] Add the archive plugin to the hook allowlist:** `extensions/msteams-channel-archive/index.ts:65`
  - body: This plugin registers typed hooks, but the bundled hook-registration boundary test keeps those files on an explicit allowlist and CI now fails with this new file as an unexpected registration site. Add the file and expected hook names to the allowlist, or avoid the registration path if this plugin should stay lazy.
  - confidence: 0.9
- **[P1] Preserve channel-data tenant in archive metadata:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: Channel activities often leave `conversation.tenantId` unset, and this file already captures the reliable tenant from `activity.channelData.tenant.id` for stored references. Using only `conversation?.tenantId` here can make cross-tenant archives run cleanup with the default tenant token and prune active channels as deleted.
  - confidence: 0.9
- **[P2] Preserve MIME slots before archiving:** `extensions/msteams-channel-archive/index.ts:111`
  - body: This handler tries to preserve empty MIME slots, but `metadata.mediaTypes` has already been filtered by the hook mapper before it reaches this line. For mixed attachments, later MIME values shift left, so archived attachment metadata and MIME queries become incorrect.
  - confidence: 0.87
- **[P2] Update Teams handler test doubles:** `extensions/msteams/src/monitor-handler.file-consent.test.ts:94-103`
  - body: `check-test-types` fails because the new test doubles for `adapter` and `conversationStore` no longer implement the current `MSTeamsAdapter` and `MSTeamsConversationStore` interfaces. Fill in the required methods so the regression test compiles against current main contracts.
  - confidence: 0.94
- **[P3] Use non-mutating sorts in archive store:** `extensions/msteams-channel-archive/src/archive-store.ts:238`
  - body: The repo lint gate rejects `Array#sort()` in the new archive store and currently reports multiple `Use Array#toSorted()` errors. Switch these ordered result paths to `toSorted()` or another accepted non-mutating ordering helper so `check-lint` can pass.
  - confidence: 0.9
