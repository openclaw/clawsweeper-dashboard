Overall confidence: 0.92

Full review comments:

- **[P1] Preserve channel-data tenant in archive metadata:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: Channel activities can leave `conversation.tenantId` unset, but this archive later uses stored tenant metadata to choose Graph credentials for cleanup. Populate provider metadata from `activity.channelData.tenant.id` before falling back to `conversation.tenantId`, or cross-tenant archives can be checked and pruned with the wrong tenant context.
  - confidence: 0.9
- **[P2] Preserve MIME slots before archiving:** `extensions/msteams-channel-archive/index.ts:111`
  - body: The archive handler preserves empty MIME slots, but the hook mapper has already filtered empty `MediaTypes` entries before this point. Mixed attachments with missing MIME values can shift later MIME types onto the wrong file, breaking attachment metadata and MIME searches; preserve cardinality before building hook metadata.
  - confidence: 0.87
- **[P2] Declare manifest activation and tool contracts:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: This plugin registers a startup service, hooks, and four tools, but its manifest omits explicit `activation.onStartup` and `contracts.tools`. The plugin metadata gate fails and tool discovery/registration cannot rely on the manifest to load the owning runtime for those tools.
  - confidence: 0.96
- **[P2] Add the archive plugin to the hook allowlist:** `extensions/msteams-channel-archive/index.ts:65`
  - body: The new plugin registers typed hooks from `extensions/msteams-channel-archive/index.ts`, but the bundled hook-registration contract uses an explicit allowlist and CI rejects this new registration site. Add the expected file and hook names, or change the registration strategy if this plugin should stay lazy.
  - confidence: 0.9
- **[P2] Use guarded fetch for Graph cleanup:** `extensions/msteams-channel-archive/src/channel-cleanup.ts:302`
  - body: The cleanup service performs token and Graph requests with raw `fetch()`, and the repo's channel/plugin network guard flags those calls. Route these requests through `fetchWithSsrFGuard()` or an existing SDK/channel wrapper so authenticated plugin networking keeps the expected outbound safeguards.
  - confidence: 0.93
- **[P2] Complete the Teams handler test doubles:** `extensions/msteams/src/monitor-handler.file-consent.test.ts:94-103`
  - body: `check-test-types` fails because the new test doubles no longer satisfy the current `MSTeamsAdapter` and `MSTeamsConversationStore` interfaces. Add the missing adapter and conversation-store methods so the regression test compiles against current Teams contracts.
  - confidence: 0.94
- **[P3] Use non-mutating sorts in archive store:** `extensions/msteams-channel-archive/src/archive-store.ts:238`
  - body: The new archive store still uses mutating `Array#sort()` calls in query paths. Switch these ordered paths to `toSorted()` or another accepted non-mutating ordering helper so the lint lane can pass without changing query behavior.
  - confidence: 0.82
