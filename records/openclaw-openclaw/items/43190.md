Full review comments:

- **[P1] Preserve the Teams channel-data tenant:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: `ProviderMetadata.tenantId` uses `conversation?.tenantId`, but current Teams code treats `channelData.tenant.id` as canonical because channel activities often omit `conversation.tenantId`. The archive store persists this tenant and cleanup later chooses the Graph token from it, so cross-tenant archives can be checked with the default tenant and active channels can be pruned as deleted.
  - confidence: 0.88
- **[P2] Route Graph cleanup requests through guarded fetch:** `extensions/msteams-channel-archive/src/channel-cleanup.ts:302`
  - body: The cleanup service calls global `fetch` directly for Azure OAuth and Graph requests carrying credentials. Existing Teams Graph and OAuth helpers route these requests through `fetchWithSsrFGuard` with the channel's network boundary, timeout, and diagnostics, so this new path weakens the current Microsoft request guardrails.
  - confidence: 0.86
- **[P2] Declare the archive tools in the manifest:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: `openclaw.plugin.json` declares only `id` and `configSchema`, but `tools.ts` registers four `api.registerTool` entries. Current plugin tool resolution uses `contracts.tools` to select runtimes and the docs require every registered tool name to appear there, so these archive tools may not be discovered or cached normally.
  - confidence: 0.9
