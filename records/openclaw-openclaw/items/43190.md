## Summary

Keep open: current main does not implement this Teams archive capability, and the PR still has source-backed blockers around guarded Graph networking, manifest contracts, tenant metadata, stale test doubles, missing real behavior proof, and product approval for a new bundled archive plugin.

## What This Changes

The PR adds a bundled Microsoft Teams channel archive plugin with JSONL message and attachment persistence, archive query tools, channel_deleted hook dispatch, deleted-channel cleanup, Teams metadata plumbing, labels, changelog entries, and tests.

## Best Possible Solution

A product-approved Teams archive path should split or approve the core hook/metadata API, keep the plugin boundary explicit, declare manifest activation/tool contracts, use guarded Teams Graph networking, preserve tenant-safe metadata, and include redacted live Teams archive/delete proof before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection of head d019bf0b4df96d7cbc56ad27f717232365d38835 shows raw authenticated Graph fetches, missing manifest tool contracts, tenant metadata loss, and stale test doubles against current main. The live archive/delete behavior itself is not reproduced because no redacted runtime artifact is attached.

## Solution Assessment

No, not as currently implemented. The safer solution is to align the plugin with manifest and SDK contracts, route Microsoft Graph calls through guarded networking, preserve channelData tenant metadata, rebase onto current interfaces, and require product approval plus real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Route Graph cleanup requests through guarded fetch:** `extensions/msteams-channel-archive/src/channel-cleanup.ts:302`
  - body: The cleanup service sends OAuth and Microsoft Graph bearer-token requests with raw `fetch`, bypassing the guarded network boundary used by current Teams Graph code. Use `fetchWithSsrFGuard` or an existing Teams Graph wrapper so redirect, SSRF, timeout, and audit protections still apply.
  - confidence: 0.93
- **[P2] Declare the archive tools in the manifest:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: This plugin registers a startup cleanup service and four agent tools, but the manifest only declares `id` and `configSchema`. Current registry and contract checks reject runtime tools unless `contracts.tools` lists their names, so the archive tools will not register cleanly.
  - confidence: 0.95
- **[P1] Preserve Teams channel-data tenant in archive metadata:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: Teams channel activities can leave `conversation.tenantId` empty while carrying the tenant in `channelData.tenant.id`. Storing only `conversation.tenantId` can make archive cleanup choose missing or wrong tenant credentials for cross-tenant channel archives.
  - confidence: 0.9
- **[P2] Bring the Teams test doubles up to current contracts:** `extensions/msteams/src/monitor-handler.file-consent.test.ts:94-109`
  - body: `createDeps` returns adapter and conversationStore objects that no longer satisfy current main interfaces: the adapter is missing `updateActivity`/`deleteActivity` and the store is missing `findPreferredDmByUserId`. The added regression test will fail type checks until those doubles match the current contracts.
  - confidence: 0.9
