## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve channel-data tenant in archive metadata:** `extensions/msteams/src/monitor-handler/message-handler.ts:792`
  - body: Channel activities can leave `conversation.tenantId` unset, but archive cleanup later uses stored tenant metadata to choose Graph credentials. Read `activity.channelData.tenant.id` before falling back to `conversation.tenantId`, or cross-tenant archives can be checked and pruned with the wrong tenant context.
  - confidence: 0.9
- **[P2] Preserve MIME slots before hook delivery:** `src/hooks/message-hook-mappers.ts:114-118`
  - body: The mapper filters empty `MediaTypes` entries before the archive plugin sees them, so later MIME values can shift onto the wrong file when mixed attachments include missing MIME values. Preserve cardinality when deriving hook metadata, not only inside the archive handler.
  - confidence: 0.88
- **[P2] Declare startup activation and tool contracts:** `extensions/msteams-channel-archive/openclaw.plugin.json:2`
  - body: This plugin registers a startup cleanup service and four runtime tools, but its manifest only declares `id` and `configSchema`. Add intentional `activation.onStartup` and `contracts.tools` entries so startup planning and tool discovery can load the owning plugin correctly.
  - confidence: 0.95
- **[P2] Use guarded Graph fetches:** `extensions/msteams-channel-archive/src/channel-cleanup.ts:302`
  - body: The cleanup service sends OAuth and Graph bearer-token requests through raw `fetch()`. Route these calls through `fetchWithSsrFGuard()` or an existing guarded Teams Graph wrapper so authenticated plugin networking stays behind the repo's outbound safeguards.
  - confidence: 0.93
- **[P2] Complete the Teams handler test doubles:** `extensions/msteams/src/monitor-handler.file-consent.test.ts:94-109`
  - body: The new `createDeps()` test double omits `updateActivity`, `deleteActivity`, and `findPreferredDmByUserId`, which are required by the current Teams adapter/store interfaces. Add those methods so the regression test compiles against current main.
  - confidence: 0.92
