## What This Changes

The issue asks to make the CLI startup benchmark helper scripts spawn their child benchmark runner with `process.execPath` instead of a bare `node` command.

## Best Possible Solution

Change only the two helper spawns to `process.execPath`, preserving the existing args, cwd, and env behavior.

I did not run the full benchmark because it writes artifacts, but a focused Node child_process probe confirmed the underlying PATH behavior.

## Solution Assessment

Yes, the requested fix is the narrowest maintainable path: use `process.execPath` for the helper-to-runner spawn, matching the already-safe `scripts/bench-cli-startup.ts` sample runner.
