## Label Justifications

- `P2`: This is a real tooling correctness bug with limited blast radius to benchmark/update scripts rather than a core runtime outage.

## Summary

Keep open: the benchmark wrapper scripts still use a PATH-resolved `node` child process on current main, and a no-PATH current-main probe reproduces the same resolver defect without needing a full version-manager setup.

## What This Changes

Use `process.execPath` instead of PATH-resolved `node` in the CLI startup benchmark wrapper scripts.

## Best Possible Solution

Change the two wrapper spawns to `process.execPath` and add a narrow regression check proving the wrappers keep working when the parent Node is absolute but `node` is unavailable or different on PATH.

## Reproduction Assessment

Yes. A current-main run of the updater with an absolute parent Node and `PATH=/nonexistent` exits 1 because the wrapper tries to resolve bare `node` from PATH.

## Solution Assessment

Yes. Replacing the two child-process command strings with `process.execPath` is the narrow maintainable fix and matches the existing benchmark harness behavior.
