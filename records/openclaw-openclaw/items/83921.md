## Summary

Keep open: current main still has two benchmark wrapper scripts that spawn a PATH-resolved `node`, and a no-PATH probe reproduces the resolver failure against the updater wrapper.

## What This Changes

The issue requests changing benchmark wrapper child-process spawns from bare `node` to `process.execPath` so benchmark runs use the same Node executable as the parent script.

## Best Possible Solution

Change both benchmark wrapper scripts to spawn `process.execPath` and add a focused regression check that they still work when the parent Node is absolute and `node` is absent or different on PATH.

## Reproduction Assessment

Yes. A current-main run of the updater with an absolute parent Node and `PATH=/nonexistent` exits 1 because the wrapper tries to resolve bare `node` from PATH.

## Solution Assessment

Yes. Replacing the two child-process command strings with `process.execPath` is the narrow maintainable fix and matches the adjacent benchmark harness behavior.
