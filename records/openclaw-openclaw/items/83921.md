Keep open: current `main` still has both benchmark wrapper scripts spawning a PATH-resolved `node`, and a no-PATH probe reproduces the same resolver failure on the checkout.

## What This Changes

The issue requests changing the CLI benchmark wrapper scripts from spawning bare `node` to spawning the current `process.execPath`.

## Best Possible Solution

Update both wrapper spawns to use `process.execPath` and add a focused regression check that proves they work when the parent Node is absolute but `node` is missing or different on PATH.

## Reproduction Assessment

Yes. A current-main run of the updater with an absolute parent Node and `PATH=/nonexistent` exits 1 because the wrapper still resolves bare `node` through PATH.

## Solution Assessment

Yes. Replacing the two child-process command strings with `process.execPath` is the narrow maintainable fix and matches adjacent benchmark harness behavior.
