## Summary

Keep this PR open: the useful provider-removal validation and contributor CLI proof are present, but the PR still loses the existing dry-run safety note when unset skips exec SecretRef checks, and the new CLI flag is not documented.

## What This Changes

The PR adds `--dry-run` support to `openclaw config unset`, validates post-unset config/SecretRef impact without writing, and adds focused config CLI tests.

## Best Possible Solution

Land a focused unset dry-run implementation that shares the set/patch SecretRef reporting semantics, documents the new CLI surface, and closes the linked issue after merge.

## Reproduction Assessment

Yes. Source inspection shows current main lacks `config unset --dry-run`, and the PR head has a high-confidence skipped-exec success path that can report validation success without the existing dry-run warning.

## Solution Assessment

No. Adding the flag on `config unset` is the right surface, but the implementation should reuse the shared dry-run result semantics or add equivalent skipped-exec/`--allow-exec` parity before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Report skipped exec checks for unset dry-runs:** `src/cli/config-cli.ts:1840-1842`
  - body: `config unset --dry-run` now selects refs with `allowExecInDryRun: false`, but on a successful dry-run it only prints the success message. If the unset touches a provider subtree while existing exec refs use that provider, those refs are skipped and the user is not told resolvability was incomplete, unlike `config set`/`patch`. Please log the skipped-exec note on success or reuse the shared dry-run result path.
  - confidence: 0.85
- **[P3] Document unset dry-run:** `src/cli/config-cli.ts:2094`
  - body: This exposes a new user-facing `openclaw config unset --dry-run` flag, but `docs/cli/config.md` still documents dry-run only for set/patch and shows unset only without dry-run. Please add the unset preflight semantics and exec-ref behavior to the config CLI docs.
  - confidence: 0.78
