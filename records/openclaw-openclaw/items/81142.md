## Summary

Keep open: this is a plausible focused CLI feature, but the diff misses a dry-run validation edge for SecretRefs/provider removals and the external PR lacks real after-fix behavior proof before merge.

## What This Changes

The PR adds a `--dry-run` option to `openclaw config unset`, validates the post-unset config without writing, and adds focused CLI tests for dry-run and help output.

## Best Possible Solution

Teach unset dry-runs to use the shared mutation validation semantics, including provider/SecretRef impact tracking for delete operations, and cover that with a regression test plus real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: remove `secrets.providers.<alias>` while an existing SecretRef still uses that alias, and the proposed dry-run block only runs schema/policy checks.

## Solution Assessment

No. Adding the flag is in the right CLI surface, but the implementation should reuse or extend the shared config mutation dry-run logic so delete operations also validate affected SecretRefs.
