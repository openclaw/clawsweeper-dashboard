## Summary

Keep open. Current main has fixed some adjacent prefill fallout, but the reported active-stream steering path still exists by design/default, GitHub Copilot Claude replay policy still does not enable Anthropic turn validation, and the recovery-loop portion remains tied to unresolved recovery work rather than a conservative cleanup close.

## What This Changes

The issue reports a v2026.5.7 TUI regression where in-stream follow-up messages are implicitly steered, assistant-role prefill content can trigger provider rejections, and recovery failures can lead to repeated or stale responses.

## Best Possible Solution

Split this into narrow fix decisions: verify the TUI queue-mode semantics, add or correct GitHub Copilot Claude replay validation if the prefill path still reproduces, and retarget recovery work to the active replay path tracked by the related recovery issue.
