Keep open. Current main still preserves the reported default steering behavior and the GitHub Copilot Claude replay-policy gap, while only part of the assistant-prefill/session-poisoning class has been fixed; the recovery-loop piece remains covered by an open focused recovery issue.

Split this into focused maintainer decisions: decide TUI/active-run queue semantics, add GitHub Copilot Claude Anthropic replay validation if the prefill path still reproduces, and keep the recovery-loop work aligned with https://github.com/openclaw/openclaw/issues/62322.
