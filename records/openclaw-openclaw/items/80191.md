## Summary

Keep open. Current main still defaults unconfigured queues to active-run steering and queues streaming follow-ups into the running session, GitHub Copilot Claude still provides only a thinking-drop replay policy instead of Anthropic turn validation, and the recovery-loop portion overlaps unresolved recovery work rather than a cleanup-closeable fix.

## What This Changes

The issue requests fixing a v2026.5.7 TUI regression where active-stream follow-ups are steered into the current run, assistant-prefill errors occur on GitHub Copilot Claude, and recovery failures can repeat prior answers.

## Best Possible Solution

Split the work into focused decisions: explicitly decide TUI active-run queue semantics, add or correct GitHub Copilot Claude Anthropic replay validation if the prefill path still reproduces, and track the recovery-loop piece with https://github.com/openclaw/openclaw/issues/62322 or a narrower active-replay fix.
