## Summary

Keep open: the regression is still source-reproducible on current main, and open PR #77617 is the active closing implementation candidate for this issue.

## What This Changes

The issue reports that `/think max` is rejected for `ollama/deepseek-v4-pro:cloud` because the lazy Ollama plugin has not registered its thinking profile.

## Best Possible Solution

Keep Ollama startup-lazy and expose the existing Ollama thinking profile through the lightweight provider-policy artifact, with regression coverage for the pre-activation path.

## Reproduction Assessment

Yes, source-reproducible: current main can fall through to the base thinking profile when the lazy Ollama plugin is inactive because the lightweight Ollama policy artifact lacks `resolveThinkingProfile`. I did not run a live Fedora/Ollama session.

## Solution Assessment

Yes, the narrow maintainable fix is the approach in open PR #77617: add the missing Ollama thinking profile to the startup-safe provider-policy artifact instead of forcing startup activation.
