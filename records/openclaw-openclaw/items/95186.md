## Summary

Keep open: current main and v2026.6.8 still list `subagent_ended` without documenting its event payload, while the typed source and hook wiring tests show the event uses `targetSessionKey` rather than `agentId`. This is a narrow docs fix with clear source truth.

## What This Changes

Document the `subagent_ended` plugin hook payload, especially `targetSessionKey` as the identity field and the absence of `agentId`.

## Best Possible Solution

Add a docs-only update to `docs/plugins/hooks.md` that lists the `subagent_ended` event fields and explicitly says `targetSessionKey` is the event identity field, with no runtime API change.

## Reproduction Assessment

Yes, at source level. Comparing `docs/plugins/hooks.md` with `src/plugins/hook-types.ts` on current main reproduces the mismatch: the docs name `subagent_ended` but omit the payload fields defined by the exported type.

## Solution Assessment

Yes. A docs-only clarification is the narrowest maintainable fix; adding `agentId` to the runtime event would be a plugin API change and is not needed for the reported docs gap.
