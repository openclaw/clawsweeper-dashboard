The issue asks for `docs/plugins/hooks.md` to document the `subagent_ended` hook payload, especially `targetSessionKey` identity and the absence of `agentId`.

## Best Possible Solution

Add a docs-only update to `docs/plugins/hooks.md` naming the `subagent_ended` fields and explicitly saying `targetSessionKey` is the identity field and `agentId` is not present on that event.

## Reproduction Assessment

Yes. Comparing `docs/plugins/hooks.md` to `src/plugins/hook-types.ts` on current `main` reproduces the mismatch: docs name the hook but omit the ended-event fields that the exported type defines.

## Solution Assessment

Yes. A docs-only clarification is the narrowest maintainable fix; adding `agentId` to the runtime event would be a plugin API change and is not needed to solve the reported documentation gap.
