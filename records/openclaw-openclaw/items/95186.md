## Summary

Keep open: current main and v2026.6.9 still list `subagent_ended` without documenting its payload, while the source type and hook wiring show `targetSessionKey` is the identity field. The linked open PR https://github.com/openclaw/openclaw/pull/95191 already closes this issue and is the right fix path, so no new repair PR should be queued.

## What This Changes

Document the `subagent_ended` hook payload fields in the plugin hooks docs, including `targetSessionKey` identity and the absence of `agentId` or `childSessionKey`.

## Best Possible Solution

Merge https://github.com/openclaw/openclaw/pull/95191, or an equivalent docs-only clarification, and keep the runtime hook contract unchanged.

## Reproduction Assessment

Yes, at source level. Current docs list `subagent_ended` without payload fields, while `src/plugins/hook-types.ts` and the wired hook test show the event uses `targetSessionKey` and lacks `agentId`.

## Solution Assessment

Yes. A docs-only clarification in `docs/plugins/hooks.md` is the narrowest maintainable fix; adding a runtime `agentId` alias would be a plugin API change and is not needed for the reported docs gap.
