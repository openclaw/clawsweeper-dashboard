## Summary

This PR should stay open for normal review rather than maintainer cleanup: it is a plausible new model-provider feature, but the current patch is not merge-ready because real behavior proof is mock-only and the declared GigaChat endpoint metadata is ignored by current provider policy code.

## What This Changes

Adds a bundled `gigachat` provider plugin with Authorization-key OAuth exchange, static model catalog, onboarding config, docs, labeler coverage, lockfile entry, and focused tests.

## Best Possible Solution

If maintainers want GigaChat bundled, keep the provider-owned OAuth integration but register a real native endpoint policy including the OAuth host, add focused policy coverage, and require redacted live OAuth/chat proof before merge.

## Reproduction Assessment

For the feature request itself, bug reproduction is not applicable. For the review finding, yes by source inspection: the PR declares `gigachat-native`, while current main's endpoint-class whitelist omits that value and drops unknown manifest endpoint classes.

## Solution Assessment

No, not as-is. A provider plugin is the right ownership boundary for OAuth exchange, but the bundled version needs native endpoint policy coverage and real GigaChat runtime proof before it is the maintainable core solution.
