## Summary

Keep open: the CPU fanout fix is useful and has sufficient live proof, but the current no-op predicate skips existing pre-tool safety work when no ordinary hook is registered.

## What This Changes

The PR adds a no-op native hook relay command for Codex native events without matching local hook handlers and tests that no-handler events avoid spawning `openclaw hooks relay`.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision depend on all pre-tool obligations and add regression tests for trusted policies and session loop detection.

## Reproduction Assessment

The original CPU fanout has PR-body live host proof, but I did not rerun that stress scenario here.

## Solution Assessment

No, not as written. The narrow optimization is the right direction, but the predicate must preserve the current pre-tool safety pipeline before skipping relay subprocesses.
