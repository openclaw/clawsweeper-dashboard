## Summary

Keep open: the branch targets a real Codex native-hook performance amplifier, but the current diff can skip existing pre-tool safety work before merge.

## What This Changes

The PR adds a native hook relay command-generation guard that returns a cheap no-op for Codex native events without matching registered global hooks, plus regression tests for relay versus no-op command selection.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision reflect all current relay work, especially trusted policies and session loop detection for pre_tool_use.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection shows a native pre_tool_use event with trusted policy or session loop-detection work and no ordinary before_tool_call hook would return the PR’s no-op command before current policy code runs.

## Solution Assessment

No. The direction is narrow and useful, but the predicate models only registered hooks; it should preserve trusted policy, approval, and loop-detection work before skipping relay subprocesses.
