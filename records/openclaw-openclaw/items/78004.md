## Summary

Keep open. The PR targets a real Codex native hook relay CPU amplifier and includes live after-fix proof, but the current diff can skip existing native pre-tool safety work when no ordinary before_tool_call hook is registered.

## What This Changes

The PR adds a no-op command shortcut for Codex native hook relay events without matching global hooks and adds regression tests for relay versus no-op command selection.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision reflect all relay work, especially trusted policies and session loop detection for pre_tool_use, then land with regression coverage.

## Reproduction Assessment

Yes for the PR regression: source inspection shows a pre_tool_use event with trustedToolPolicies or session loop state and no ordinary before_tool_call hook would now return a no-op before current-main policy code can run. I did not run the PR branch or a live low-core stress test in this read-only review.

## Solution Assessment

No. The direction is narrow and useful, but the predicate models only registered hooks; it should preserve trusted policy, approval, and loop-detection work before skipping relay subprocesses.
