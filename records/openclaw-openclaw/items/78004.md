## Summary

Keep open: the PR targets a credible Codex native hook relay performance amplifier, but the current diff can skip existing pre-tool safety work and needs a focused repair before merge.

## What This Changes

The PR adds a no-op command path for Codex native hook relay events without matching local hooks and adds command-selection regression tests.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision account for every pre-tool obligation that `runBeforeToolCallHook` would perform, then cover those cases with focused regression tests.

## Reproduction Assessment

Yes for the PR regression by source inspection: the new `pre_tool_use` no-op path can skip `runBeforeToolCallHook` when no ordinary hook is registered. The original CPU fanout has live proof in the PR body, but I did not rerun that host stress scenario.

## Solution Assessment

No, not as written. The optimization is narrow and maintainable, but the predicate must preserve loop detection, trusted policies, and approval work before skipping relay subprocesses.
