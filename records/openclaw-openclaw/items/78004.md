## Summary

Keep open. The PR is a relevant narrow fix for the linked Codex native hook-relay CPU issue and the body includes plausible after-fix live host proof, but the current diff can skip native pre-tool safety work whenever no ordinary `before_tool_call` hook is registered.

## What This Changes

The PR adds no-op command generation for Codex native hook relay events without matching global hooks and tests relay versus no-op command selection.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision account for all pre-tool relay work, especially trusted policies, plugin approvals, and session loop detection.

## Reproduction Assessment

Yes for the PR regression: source inspection shows `pre_tool_use` no-op selection based only on `hasGlobalHooks("before_tool_call")`, while current main performs loop detection and trusted policy/approval work before or apart from the ordinary hook gate. I did not run a live low-core stress test for the original performance issue.

## Solution Assessment

No. The direction is narrow and useful, but the predicate models only registered hooks; it should preserve trusted policy, approval, and loop-detection work before skipping relay subprocesses.
