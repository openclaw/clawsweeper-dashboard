## Summary

Keep open. The PR targets a real Codex native hook relay performance amplifier, but the current diff can skip the relay before native pre-tool safety policy, trusted plugin approval, and loop detection have a chance to run.

## What This Changes

The PR adds a command-generation guard and tests so Codex native hook events use a cheap Node no-op command when no matching local hook handler is registered.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision preserve all pre-tool relay work sources, then land it with regression coverage for trusted policies and loop detection.

## Reproduction Assessment

Yes for the PR regression: source inspection shows pre_tool_use can become a no-op based only on hasGlobalHooks("before_tool_call"), while current main runs loop detection and trusted policy work inside the relay path. I did not run a live low-core stress reproduction in this read-only review.

## Solution Assessment

No. The optimization direction is useful and narrow, but the predicate should account for trusted policies, approval paths, and session loop detection before skipping the relay subprocess.
