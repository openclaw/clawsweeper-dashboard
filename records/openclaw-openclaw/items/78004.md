## Summary

Keep open. The optimization is useful and has after-fix host proof, but the current diff would skip native pre-tool safety work whenever no ordinary `before_tool_call` hook is registered.

## What This Changes

The branch adds no-op command generation for Codex native hook relay events without matching global hooks and tests relay versus no-op command selection.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision account for all pre-tool relay obligations: session loop detection, trusted policies, plugin approval, and ordinary hooks.

## Reproduction Assessment

Yes for the PR regression: source inspection shows `pre_tool_use` no-op selection based only on `hasGlobalHooks("before_tool_call")`, while current main performs loop detection and trusted policy or approval work before or apart from that ordinary hook gate. I did not run a live low-core stress test for the original performance issue.

## Solution Assessment

No. The direction is narrow and useful, but the predicate models only registered hooks; it should preserve trusted policy, approval, and loop-detection work before skipping relay subprocesses.
