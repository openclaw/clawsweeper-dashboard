## Summary

Keep open: the optimization addresses a credible native Codex hook-relay fanout problem, but the current patch can skip existing pre-tool safety work before the relay process ever reaches OpenClaw.

## What This Changes

The PR adds a no-op command path for Codex native hook relay events without matching local hooks and adds command-selection regression tests.

## Best Possible Solution

Keep the no-handler relay optimization, but make the skip decision preserve every `pre_tool_use` obligation that current main runs through `runBeforeToolCallHook`.

## Reproduction Assessment

Yes for the PR regression by source inspection: a trusted policy or session loop-detection path can require `runBeforeToolCallHook` even when there is no ordinary `before_tool_call` hook. I did not run a live low-core stress reproduction for the original CPU issue.

## Solution Assessment

No, not as written. The optimization is a narrow maintainable direction, but the predicate must account for trusted policies, plugin approvals, and session loop-detection work before returning a no-op command.
