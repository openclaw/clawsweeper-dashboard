## Summary

Keep open. The PR targets a credible hook-relay fanout problem and has sufficient live host proof for that intent, but the proposed no-op predicate can skip the current `pre_tool_use` safety pipeline before tool execution.

## What This Changes

The PR adds a no-op native hook relay command for Codex native events without matching local hook handlers and adds relay command-generation tests.

## Best Possible Solution

Keep the no-handler relay optimization, but make the no-op decision preserve all pre-tool obligations and add regression coverage for trusted policies, approvals, and loop detection.

## Solution Assessment

No, not as written. The narrow optimization is the right direction, but the predicate must preserve the current pre-tool safety pipeline before skipping relay subprocesses.
