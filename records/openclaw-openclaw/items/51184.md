## Summary

Keep open: current main and the latest release still drop stored label/displayName metadata from the `/status` status contract/renderers and keep the TUI statusline centered on the raw session key. The linked implementation PR remains open and conflicting, so this issue should wait for that PR to land, be reworked, or be replaced.

## What This Changes

Surface friendly cron job/session labels instead of raw UUID-heavy session keys in `/status`, the TUI statusline, and session picker surfaces.

## Best Possible Solution

Land or replace the linked implementation so the status contract and TUI renderers carry sanitized label/displayName metadata with raw-key fallback and active-session compatibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `SessionEntry` stores label/displayName metadata, but current main projects `/status` rows without those fields and renders raw keys in CLI/TUI status output and the statusline.

## Solution Assessment

Yes for the requested direction, but the maintainable fix should extend the status contract/renderers with sanitized friendly text while preserving a resolvable key fallback. The open linked PR is the right ownership path to review, rework, or replace.
