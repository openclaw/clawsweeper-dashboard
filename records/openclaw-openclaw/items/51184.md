## Summary

Keep open: current main and the latest shipped release still omit stored label/displayName metadata from the status payload, `/status` renderers, and statusline. The linked implementation candidate at https://github.com/openclaw/openclaw/pull/51257 remains open and should be reviewed, reworked, replaced, or closed before this issue is resolved.

## What This Changes

Request to display cron job names or session label metadata in TUI `/status`, statusline, and session picker instead of raw UUID-heavy session keys.

## Best Possible Solution

Land or replace the linked implementation so active status contracts and renderers carry sanitized label/displayName metadata with raw-key fallback and a resolvable selector, while explicitly scoping any remaining picker grouping or statusline behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `SessionEntry` has label/displayName metadata, but current main still projects and renders raw keys in status summary, CLI/TUI `/status`, and the statusline; no live TUI run was performed because this review is read-only.

## Solution Assessment

Yes for the requested direction, but the best implementation is a rebased status-contract/rendering change that sanitizes friendly text and preserves a resolvable key or label rather than only swapping visible keys.
