## Summary

Keep open. Current main still drops stored session `label`/`displayName` metadata from `/status` and TUI status/statusline rendering, and the open linked PR with closing syntax should be reviewed rather than closing this issue early.

## What This Changes

The issue requests friendly cron/session labels in `/status`, the TUI statusline, and the session picker instead of UUID-heavy session keys.

## Best Possible Solution

Review and land a corrected version of https://github.com/openclaw/openclaw/pull/51257 that threads `label`/`displayName` through the status contracts and renderers with sanitized output and raw-key fallback.

## Reproduction Assessment

Yes. Source inspection on current main shows cron runs set `sessionEntry.label`, while `SessionStatus`/`GatewayStatusSummary` recent rows omit `label`/`displayName` and both active CLI/TUI status renderers use the raw key.

## Solution Assessment

Yes. Carrying existing session metadata through the status payload and renderers is the narrow maintainable direction; the active linked PR should be corrected and proven rather than opening a competing repair.
