## Summary

Keep open. Current main and the latest release still omit stored session `label`/`displayName` metadata from `/status` and TUI status/statusline rendering; https://github.com/openclaw/openclaw/pull/51257 is the open implementation candidate with closing syntax, so this issue should stay open until that PR or a replacement lands.

## What This Changes

Request to display stored session labels, display names, or cron names in `/status`, the TUI statusline, and session picker instead of raw UUID-heavy session keys.

## Best Possible Solution

Review and land a corrected version of https://github.com/openclaw/openclaw/pull/51257, or a replacement, that threads `label`/`displayName` through the status contracts and renderers with sanitized output, raw-key fallback, and intentional statusline/session-picker behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows cron labels are written, but status payloads and active CLI/TUI renderers still omit or ignore those fields and display raw keys.

## Solution Assessment

Yes for the direction, but not yet for the open patch as-is. The maintainable fix is to carry existing metadata through the status contracts and render sanitized friendly labels while keeping raw keys available for resolution.
