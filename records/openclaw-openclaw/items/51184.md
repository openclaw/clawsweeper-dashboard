## Summary

Keep open: current main and the latest release still omit session label/displayName from the shared status summary, while CLI/TUI status renderers continue to display raw session keys. The Control UI dropdown fix is only a partial overlap, and the open linked implementation candidate is unmerged and conflicting, so this still needs maintainer direction rather than cleanup closure.

## What This Changes

Request to show friendly cron job/session labels in `/status`, the TUI statusline, and the session picker instead of UUID-heavy raw session keys.

## Best Possible Solution

Land or replace the linked candidate with a narrow status-display change that carries sanitized label/displayName through the shared status contract while preserving raw keys for selection and fallback.

## Reproduction Assessment

Yes, by source inspection. Current main does not carry label/displayName into `SessionStatus`, the CLI sessions table prints `sess.key`, and the TUI status summary prints `entry.key`.

## Solution Assessment

No, current main does not solve it. The best fix is shared status-summary label propagation plus terminal-safe rendering, rather than per-renderer parsing of raw cron keys.
