## Summary

Keep open: current main still omits label/displayName from the shared status summary and renders raw session keys in CLI/TUI status paths, while the merged Control UI dropdown work only partially overlaps. The open fork fix candidate at https://github.com/BingqingLyu/openclaw/pull/1139 is still conflicting, so it is not a safe implemented or superseding path yet.

## What This Changes

Request to show session labels, display names, or cron job names in `/status`, the TUI statusline, and session picker instead of raw UUID-heavy session keys.

## Best Possible Solution

Carry a sanitized display label through the shared status summary and render it consistently in CLI/TUI status, footer, and picker while keeping the raw session key available for copyable detail and commands.

## Reproduction Assessment

Yes, by source inspection. Current main stores label/displayName metadata in session-list rows, but the shared status summary and CLI/TUI status renderers still carry and print raw keys; no runtime repro was run because this review is read-only.

## Solution Assessment

No, current main is only a partial solution through the Control UI label helper. The best fix is a shared, sanitized display-name contract consumed by CLI/TUI status and footer/picker surfaces, not a one-off formatter replacement.
