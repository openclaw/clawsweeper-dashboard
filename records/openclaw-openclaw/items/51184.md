## Summary

Keep open: current main and the latest release still omit stored label/displayName metadata from the status contract and render raw keys in `/status` and the TUI statusline. The Control UI picker slice was already handled by https://github.com/openclaw/openclaw/pull/45130, but the target-repo implementation PR at https://github.com/openclaw/openclaw/pull/51257 is closed unmerged, so the remaining status surfaces still need maintainer-directed work.

## What This Changes

The issue asks for friendly cron/session labels to appear in `/status`, the TUI statusline, and the session picker instead of raw UUID-heavy session keys.

## Best Possible Solution

Extend the shared status contract and CLI/TUI renderers to carry sanitized label/displayName metadata with raw-key fallback and tests for cron, subagent, and active-session statusline behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `SessionEntry` and cron session code retain label/displayName metadata, but current main does not project those fields into status rows and renders raw keys in CLI/TUI status output.

## Solution Assessment

Yes for the direction, but not by reviving the old patch unchanged. The maintainable fix should target the shared status projection and renderers, sanitize terminal-displayed labels, and keep a raw key fallback for resolvability.
