## Summary

Keep open. Current main still drops existing session `label`/`displayName` metadata from the CLI and TUI status paths, while the Control UI dropdown portion is already handled and https://github.com/openclaw/openclaw/pull/51257 remains the active implementation candidate for this issue.

## What This Changes

The issue requests showing existing session label/displayName or cron job names in `/status`, the TUI statusline, and the session picker instead of UUID-heavy session keys.

## Best Possible Solution

Land a corrected version of the linked implementation that carries `label` and `displayName` through the status contracts, renders sanitized friendly labels in CLI/TUI/statusline output with raw-key fallback, and preserves picker behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows cron runs set `sessionEntry.label`, but `SessionStatus`/`GatewayStatusSummary` recent rows omit label/displayName and both CLI/TUI status renderers use the raw key.

## Solution Assessment

Yes. The maintainable direction is to carry existing metadata through the status payload and renderers rather than adding cron-only lookup state; the linked PR targets that direction but needs review/proof before it can land.
