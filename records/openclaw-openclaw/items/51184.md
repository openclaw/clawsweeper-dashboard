## Summary

Keep open. Current main and v2026.5.18 still drop stored session label/displayName metadata from the `/status` payload and render raw session keys in CLI/TUI status output; https://github.com/openclaw/openclaw/pull/51257 is the open, unmerged implementation candidate with closing syntax, while the Control UI dropdown piece was separately addressed by https://github.com/openclaw/openclaw/pull/45130.

## What This Changes

The issue asks to surface stored session labels, display names, or cron job names in `/status`, the TUI statusline, and session selection instead of raw UUID-heavy session keys.

## Best Possible Solution

Land a corrected implementation that carries existing label/displayName metadata through the status contracts and renderers, sanitizes user-controlled text, preserves raw-key fallback, and intentionally scopes any remaining session-picker grouping work.

## Reproduction Assessment

Yes. Source inspection on current main shows session entries store label/displayName metadata, but status summaries and CLI/TUI renderers omit those fields and print raw keys.

## Solution Assessment

Yes for the requested direction, but the open patch is not yet the best final form. The maintainable solution should thread existing metadata through the status contract, sanitize rendered names, and keep a resolvable key or label available.
