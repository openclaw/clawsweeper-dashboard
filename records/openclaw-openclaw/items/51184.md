## Summary

Keep open. Current main still has cron/subagent label metadata but the status summary contract and CLI/TUI renderers do not carry or display it, and open PR #51257 already tracks this work with closing syntax.

## What This Changes

Requests that cron/session label or displayName metadata appear in CLI/TUI `/status`, the TUI statusline, and session selection surfaces instead of UUID-heavy session keys.

## Best Possible Solution

Thread existing `label` and `displayName` metadata through the status contracts and TUI state, prefer sanitized friendly labels at render points, and land or replace #51257 with focused regression coverage.

## Reproduction Assessment

Yes, via source inspection. Cron runs write `sessionEntry.label`, while the current status contracts and CLI/TUI status renderers expose and print only raw session keys.

## Solution Assessment

Yes. The narrow maintainable path is to carry existing session metadata through the status payload and UI renderers rather than adding cron-only lookup state; #51257 is aligned with that direction but still needs normal review/rebase or replacement.
