## Summary

Keep open: current main and latest release still omit stored label/displayName metadata from the status payload, `/status` renderers, and statusline; the linked implementation candidate at https://github.com/openclaw/openclaw/pull/51257 is still open and should be reviewed, reworked, replaced, or closed before this issue is resolved.

## What This Changes

The issue requests friendlier cron/session labels in `/status`, the TUI statusline, and session picker surfaces instead of UUID-heavy raw session keys.

## Best Possible Solution

Land or replace the linked implementation so active status contracts and renderers carry sanitized label/displayName metadata with raw-key fallback and a resolvable selector, while explicitly scoping any remaining picker grouping or statusline behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main stores label/displayName metadata in `SessionEntry`, but status summaries, CLI/TUI status renderers, and the statusline still project or print raw keys. I did not run a live TUI because this review is read-only.

## Solution Assessment

Yes for the requested direction, but the best implementation is a rebased status-contract/rendering change that sanitizes friendly text and preserves a resolvable key or label, not only a visible-key swap.
