## Summary

Keep open: current main and the latest release still drop session label/displayName before the shared status renderers, so `/status` and the TUI status summary continue to use raw session keys. The Control UI picker fix is only a partial overlap, and the open linked implementation candidate remains unmerged and conflicting.

## What This Changes

Request to show cron job names or session label/displayName in `/status`, the TUI statusline, and session picker displays instead of UUID-heavy session keys.

## Best Possible Solution

Land or replace the linked candidate with a narrow shared status-contract change that carries sanitized label/displayName into CLI/TUI status displays while preserving raw session keys for selection and fallback.

## Reproduction Assessment

Yes, by source inspection. Current main does not carry label/displayName into `SessionStatus`, and both CLI and TUI status renderers print raw session keys.

## Solution Assessment

Yes on fix shape, but maintainers still need to confirm the product contract. The maintainable path is shared status-summary label propagation with raw-key fallback, not per-renderer cron-key parsing.
