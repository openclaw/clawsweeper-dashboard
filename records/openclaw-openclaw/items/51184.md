## Summary

Keep open. Current main still drops existing `label`/`displayName` metadata from the CLI/TUI status paths while the Control UI dropdown portion is already handled, and open PR #51257 plus the fork PR are active implementation candidates for this issue.

## What This Changes

Surface existing session `label`/`displayName` and cron job names in `/status`, the TUI statusline, and session selection instead of UUID-heavy session keys.

## Best Possible Solution

Thread `label` and `displayName` through the status contracts, render a sanitized friendly-name fallback in CLI/TUI status and the footer/picker, and land or replace the linked implementation with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows cron runs write `sessionEntry.label`, but current main does not expose it in `SessionStatus`/`GatewayStatusSummary` and both CLI/TUI status render the raw key.

## Solution Assessment

Yes. The narrow maintainable direction is to carry existing metadata through the status payload and renderers rather than adding cron-only state; PR #51257 is already aimed at that path and should be reviewed, rebased, landed, or closed.
