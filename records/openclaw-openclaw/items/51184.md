## What This Changes

Request to show friendly cron/session labels in /status, the TUI statusline, and session-picker surfaces instead of raw UUID-heavy session keys.

## Best Possible Solution

Land or replace the linked implementation so status contracts and renderers carry sanitized label/displayName metadata with raw-key fallback and a resolvable selector, while explicitly scoping any remaining picker grouping work.

## Reproduction Assessment

Yes. Source inspection on current main shows session entries store label/displayName metadata, but status summaries and CLI/TUI renderers omit those fields and print raw keys.

## Solution Assessment

Yes for the requested direction, but the final implementation should be a rebased status-contract/rendering change with sanitization and a resolvable fallback rather than only swapping the visible Key value.
