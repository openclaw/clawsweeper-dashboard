## Summary

Keep open: current main and v2026.6.1 still route active-memory recall through per-turn `prependContext` in the user prompt, and the docs do not disclose the reported Anthropic-compatible cache impact. The related OpenAI-compatible cache-key issue is adjacent, but it does not cover this Anthropic-compatible claim.

## What This Changes

The issue asks OpenClaw to prevent or document active-memory `prependContext` from collapsing Anthropic-compatible prompt-cache hit rates.

## Best Possible Solution

Decide the cache contract for dynamic plugin context, then either document active-memory as cache-hostile on affected Anthropic-compatible providers or move/shape the dynamic memory block through a tested boundary that preserves stable-prefix caching.

## Reproduction Assessment

No, not yet with high confidence. Source confirms the variable active-memory prefix is injected into user-prompt space, but the reported cache-rate collapse still needs a live Anthropic-compatible provider probe with cache token metrics.

## Solution Assessment

Unclear. A docs warning is the narrow safe mitigation, while moving or bypassing active-memory context requires a maintainer decision about plugin hook semantics and provider cache boundaries.
