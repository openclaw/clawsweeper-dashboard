## Summary

Keep open: current main and the latest release still route active-memory recall through variable `prependContext` in model user-prompt space, and the closed related items do not resolve this broader Anthropic-compatible cache-hit-rate contract.

## What This Changes

The issue reports that enabling active-memory injects variable recalled memory into the model prompt and can collapse prompt-cache reuse for Anthropic-compatible providers.

## Best Possible Solution

Define the active-memory prompt-cache contract, then either document the provider limitation or implement a tested boundary/opt-out that preserves stable-prefix caching without changing memory trust semantics.

## Reproduction Assessment

No high-confidence maintainer-controlled live reproduction is available. Source inspection confirms the current-main injection path and the reporter supplied useful two-provider cache metrics, but the providers are redacted and I did not run a trusted cache probe.

## Solution Assessment

Unclear. A docs warning is the narrow safe mitigation, while any code fix needs maintainer direction on whether dynamic memory belongs in user prompt space, system prompt space, or an explicit cache-bypass/cache-boundary contract.
