## Summary

Keep open: current main and the latest release still route active-memory recall through variable `prependContext` in model user-prompt space, and the related closed items do not resolve this broader Anthropic-compatible cache-hit-rate contract.

## What This Changes

The issue reports that active-memory injects variable recalled context into prompt construction, sharply reducing prompt-cache hit rates for Anthropic-compatible providers.

## Best Possible Solution

Define the active-memory prompt-cache contract, then either document the provider limitation or implement a tested boundary/opt-out that preserves stable-prefix caching without changing memory trust semantics.

## Reproduction Assessment

No, not as a maintainer-controlled live reproduction. Source inspection confirms the current-main injection path and the reporter supplied useful two-provider live metrics, but the endpoints are redacted and I did not run a trusted Anthropic-compatible cache probe.

## Solution Assessment
