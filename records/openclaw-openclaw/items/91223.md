## Summary

Keep open: current main and v2026.6.8 still put active-memory recall into variable user-prompt prependContext, while no merged PR or release resolves the broader Anthropic-compatible cache-hit-rate contract.

## What This Changes

The issue reports that active-memory recall varies prependContext per turn and collapses prompt-cache hit rates for Anthropic-compatible providers.

## Best Possible Solution

Define the active-memory prompt-cache contract, then either document the provider limitation or add a tested boundary/opt-out that preserves untrusted-context semantics.

## Reproduction Assessment

No, not as a high-confidence maintainer-controlled live repro. Source confirms current-main variable active-memory prependContext enters model user-prompt space, and the reporter supplied useful redacted two-provider metrics, but I did not run a trusted live cache probe.
