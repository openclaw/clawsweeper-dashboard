## Summary

Keep open. Current main still hardcodes the Control UI/WebChat near-bottom autoscroll behavior around a 450px threshold and has no persisted chatAutoScroll setting; this is a new UX/config capability rather than a broken defined contract.

## What This Changes

Request a WebChat/Control UI setting that lets users choose streaming autoscroll behavior, including an opt-in off mode instead of only the current near-bottom threshold.

## Best Possible Solution

If maintainers accept the UX, add a typed chat autoscroll mode with default near-bottom, persist it with UiSettings, expose it in chat controls, and gate scheduleChatScroll while preserving the New messages button and reduced-motion behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current near-bottom autoscroll behavior and absence of a chatAutoScroll setting on current main.

## Solution Assessment

Unclear until maintainer UX review. The proposed direction is plausible and narrow, but the exact modes and interaction with first-load/send-time scrolling should be decided before implementation.
