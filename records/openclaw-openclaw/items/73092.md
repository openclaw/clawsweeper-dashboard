## Summary

Keep this PR open. The branch addresses the reported manual compaction failure by forwarding fresh token usage and caller context budget into the compaction path, but the latest diff still mixes the caller context window with a reserve value prepared for the compaction target, which can re-enable compaction for idle/tool-only small-context sessions that are still below the caller threshold. The branch is also currently dirty against the base branch.

## What This Changes

The PR adds a token-threshold-aware outer compaction idle-skip guard, passes fresh persisted token usage and caller context budget from manual `/compact`, adds focused tests, and updates the changelog.

## Best Possible Solution

Land a narrow compaction fix that compares fresh persisted token pressure against one consistent active-session budget: the caller context window and a reserve resolved or capped for that same caller window. Preserve the existing idle skip when token usage is absent or below that threshold, then rebase the dirty branch and run the focused compaction and command tests.

## Reproduction Assessment

Yes. Current main can be reproduced with a limited session slice containing heartbeat-only text and tool results outside the 20-message user lookback, plus fresh persisted usage such as 236k/200k; the outer guard skips before compaction despite context pressure.

## Solution Assessment

No, not yet. The PR is the right shape, but the threshold calculation should not subtract a reserve prepared for the compaction model when the comparison is using the active caller model's context window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Cap the reserve against the caller budget:** `src/agents/pi-embedded-runner/compact.ts:337-338`
  - body: The new guard switches the threshold window to `callerContextWindowTokens`, but it still subtracts the reserve value from the settings manager prepared with the compaction target window. With `agents.defaults.compaction.model` set to a large summarizer and a small active model, the default 20k reserve can exceed a 16k caller window, clamping the threshold to 1 token and letting idle/tool-only sessions below the active threshold bypass the skip. Resolve or cap the reserve for the same caller budget used by this guard.
