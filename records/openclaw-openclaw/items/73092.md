## Summary

Keep this PR open. Current main still has the reported outer compaction pre-check bug, and this PR is a focused repair, but the proposed threshold bypass compares against the compaction target budget rather than the active caller/session budget, leaving the failure possible with `agents.defaults.compaction.model`; it also lacks the required changelog entry.

## What This Changes

The PR adds a threshold-aware idle compaction guard, passes fresh persisted token counts into manual `/compact`, and adds focused tests for tool-heavy compaction slices and manual token forwarding.

## Best Possible Solution

Land a narrow compaction fix that preserves the idle-session skip for absent or below-threshold fresh token usage, but evaluates token pressure against the active caller/session context budget. Keep the message classifier and inner safeguard behavior unchanged unless a separate follow-up redesigns them.

## Reproduction Assessment

Yes. On current main, a limited slice with heartbeat-only text plus tool results and no meaningful user message in the 20-message lookback makes `containsRealConversationMessages` false, so manual `/compact` can skip before using fresh persisted token pressure.

## Solution Assessment

No, not yet. The proposed direction is maintainable, but the guard needs a separate active-budget input instead of `ctxInfo.tokens` from the compaction target, and the user-facing fix needs a changelog entry.
