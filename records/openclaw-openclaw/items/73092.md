## Summary

Keep open. The PR targets a valid compaction bug and the direction is right, but the current diff has a blocking correctness issue: manual `/compact` forwards stale persisted session context as the caller threshold even though this path already resolves the active runtime budget to avoid stale metadata.

## What This Changes

The PR adds threshold-aware bypassing of the idle compaction skip for over-threshold tool-heavy sessions, forwards manual `/compact` token data, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Use the already-resolved manual `contextTokenBudget` as the caller threshold budget, add regression coverage for stale session metadata, and point the changelog at https://github.com/openclaw/openclaw/issues/73056 before landing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a limited heartbeat/tool-result `session.messages` slice can fail `containsRealConversationMessages` and hit the outer skip before current main considers manual token pressure.

## Solution Assessment

No, not as currently implemented. The code direction is narrow and maintainable, but the manual path must pass the resolved active budget rather than raw persisted session metadata before this is the right fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the resolved caller budget for the threshold:** `src/auto-reply/reply/commands-compact.ts:155`
  - body: The new caller threshold is taken from `targetSessionEntry.contextTokens`, but this command path already resolves `contextTokenBudget` from live/configured model data to avoid stale session metadata. If the persisted entry is larger than the active caller window, an over-active-budget tool-heavy `/compact` run can still be skipped as idle. Pass the resolved `contextTokenBudget` here, or omit the explicit caller field so the compaction path falls back to it, and cover the existing stale-metadata test case.
  - confidence: 0.89
