## Summary

Keep this PR open. The latest PR head is still the right general shape for #73056, but it keeps one correctness blocker: the new idle-skip bypass compares against the caller context window while subtracting a reserve value prepared for the compaction target model, which can collapse the threshold to 1 token for small active models with a large summarizer override.

## What This Changes

The PR threads fresh token usage and caller context budget through manual `/compact`, adds a token-threshold-aware no-real-conversation compaction guard, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the PR after making the threshold calculation internally consistent: compare observed token usage against the active caller window and a reserve resolved or capped for that same caller window, while preserving the idle/heartbeat skip when observed usage is absent or below threshold.

## Reproduction Assessment

Yes. The blocker can be reproduced at helper level with no-real-conversation messages, `callerContextWindowTokens: 16_384`, `contextWindowTokens: 1_000_000`, `reserveTokens: 20_000`, and any observed count above 1 but below the intended caller threshold; the PR helper bypasses the skip because the threshold clamps to 1.

## Solution Assessment

No, not yet. The PR’s approach is maintainable, but the reserve must be derived from the same caller budget used for the comparison; otherwise valid small active-model sessions can bypass the idle skip far below their real threshold.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Cap the reserve to the caller budget:** `src/agents/pi-embedded-runner/compact.ts:344-345`
  - body: The new guard compares observed tokens against the active caller context window, but it subtracts `settingsManager.getCompactionReserveTokens()`, which was prepared for the compaction target model. With a large `agents.defaults.compaction.model` and a small active caller model, the default 20k reserve can exceed the caller window, clamp `threshold` to 1, and let idle/tool-only slices below the active threshold proceed to compaction. Resolve or cap the reserve for the same caller budget used here.
  - confidence: 0.86
