## Summary

Keep open. Current main still has the reported failure mode, and this PR is aimed at the right small surface, but the patch remains incomplete for supported compaction-model override setups because the new threshold uses the compaction target window rather than the active session/caller budget. It also needs the required user-facing changelog entry before merge.

## What This Changes

The PR adds a token-threshold-aware outer compaction skip guard, forwards fresh persisted token usage into manual `/compact`, and adds focused tests for tool-heavy slices and manual token forwarding.

## Best Possible Solution

Land a narrow compaction fix that keeps the cheap idle-session skip for absent or below-threshold token usage, but compares fresh persisted token pressure against the active/caller session budget when deciding whether a tool-heavy manual compaction slice may proceed. The classifier and inner safeguard should stay unchanged unless a broader follow-up explicitly redesigns them.

## Reproduction Assessment

Yes. On current main, a limited slice containing heartbeat-only text plus tool results with no meaningful user message in the 20-message lookback makes `containsRealConversationMessages` false, so manual `/compact` can skip even when fresh persisted tokens show the session is over its active context budget.

## Solution Assessment

No, not yet. The proposed direction is maintainable, but the new threshold should honor the caller/session context budget instead of only the compaction target window, and the PR needs the repo-required changelog entry.
