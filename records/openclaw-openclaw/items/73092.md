## Summary

Keep open: current main still has the outer idle pre-check bug, and the latest PR head is close but still mixes the caller context window with a reserve prepared for the compaction target window, which can make under-threshold idle/tool-only sessions compact unnecessarily.

## What This Changes

The PR forwards fresh `/compact` token usage and caller context budget into compaction, adds an idle-skip threshold helper with tests, and adds a changelog entry.

## Best Possible Solution

Land a narrow compaction fix that evaluates observed token pressure against one active caller-session budget, with the reserve resolved or capped for that same budget.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a history-limited heartbeat/tool-heavy slice with no meaningful user message in the 20-message tool-result lookback plus over-threshold persisted usage still hits the current-main outer skip before compaction starts.

## Solution Assessment

No. The PR shape is right, but the latest head should cap or resolve the reserve against the same caller context budget used for the threshold instead of subtracting the compaction-target reserve unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:
