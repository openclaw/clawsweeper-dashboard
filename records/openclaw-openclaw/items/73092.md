## Summary

Keep this PR open: the compaction fix is useful and now addresses the stale manual caller-budget path, but the current head still has a reserve-threshold bug in normal queued compaction paths and the supplied proof does not show the changed manual `/compact` behavior in a real run.

## What This Changes

The PR adds threshold-aware compaction idle-skip handling for tool-heavy sessions, forwards manual `/compact` token budget data, adds regression coverage and changelog text, and tightens gateway origin/mobile pairing checks.

## Best Possible Solution

Land the compaction change after the idle-skip threshold preserves reserve-adjusted behavior for ordinary queued compactions while using the explicit caller budget only for manual/caller-supplied paths, with real manual `/compact` proof added before merge.

## Reproduction Assessment

Yes, by source inspection: current main can enter the outer no-real-conversation skip in `compactEmbeddedPiSessionDirectOnce` before considering token pressure. The linked report supplies the concrete `/compact` symptom at 236k/200k tokens.

## Solution Assessment

No. The caller-budget forwarding is the right direction, but the current implementation also treats the normal compaction `contextTokenBudget` as a caller budget and therefore ignores the reserve for queued non-manual paths.
