## Summary

Keep open. Current main still implements heartbeat as a per-agent scheduler/config unit with an optional session override, and I did not find a shipped implementation or canonical duplicate that covers durable session-scoped heartbeat entries with operator controls.

## What This Changes

The issue requests native durable heartbeat entries scoped to explicit sessions, each with its own cadence, routing, persistence, enabled state, and observability.

## Best Possible Solution

Define a core durable heartbeat-entry contract that composes with heartbeat.session, isolatedSession, task batching, commitments, cron wakes, and operator APIs before implementing it.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence verification path: current main exposes one heartbeat object and per-agent scheduler state, not native durable session-scoped heartbeat entries.

## Solution Assessment

Unclear until maintainers choose the product boundary. Native entries are plausible, but the design must reconcile the current per-agent scheduler contract with explicit multi-session entries, persistence, concurrency, and operator controls.
