## Summary

Keep open. The stale replay path is still source-reproducible on current main, but this PRâ€™s broad aborted-run early return would also suppress valid timeout replies and skip compaction follow-up bookkeeping, so it needs a narrower repair before merge.

## What This Changes

The PR changes the embedded runner to omit `lastAssistant` for aborted empty attempts and changes auto-reply delivery to skip block flushing plus payload delivery whenever run metadata is aborted.

## Best Possible Solution

Narrow the fix to suppress stale `lastAssistant` fallback only for externally interrupted empty attempts, then add regression coverage that preserves timeout delivery, compaction accounting, rotated session refresh, and post-compaction context injection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `interrupt` aborts the active run, an aborted empty attempt can still carry `lastAssistant` from full session history, and current main still feeds that value into the payload fallback.

## Solution Assessment

No. Guarding stale `lastAssistant` is the right direction, but treating every `meta.aborted` result as undeliverable is too broad because timeout and compaction paths also use aborted metadata for intentional stateful outcomes.
