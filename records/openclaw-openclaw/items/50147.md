## Summary

Keep this PR open: current main still has the stale `lastAssistant` fallback path, but the PR's broad aborted-run early return would regress timeout delivery and compaction bookkeeping before merge.

## What This Changes

The PR suppresses stale `lastAssistant` fallback text for aborted empty embedded attempts and adds an aborted-run early exit in auto-reply delivery.

## Best Possible Solution

Keep the narrow stale `lastAssistant` suppression, but make the silent abort exit specific to interrupt/superseded empty runs, preserve timeout/error payload delivery and compaction accounting, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows `interrupt` aborts the active run, an aborted empty attempt can retain `lastAssistant` from the full session snapshot, and current main still feeds that value into the payload fallback.

## Solution Assessment

No. Guarding stale fallback text is the right direction, but treating every `meta.aborted` result as undeliverable is too broad because timeout and compaction paths use aborted metadata for intentional outcomes.
