## Summary

Keep this PR open: it targets a real interrupt-mode stale-replay bug that current main still appears to have, and it is paired with the same-author open bug #50145. The submitted patch should not merge as written because its blanket `meta.aborted` delivery skip would also suppress timeout replies and bypass post-compaction bookkeeping.

## What This Changes

The PR guards `buildEmbeddedRunPayloads` from using `lastAssistant` after an aborted empty attempt and skips block-reply flushing plus payload delivery for aborted auto-reply runs.

## Best Possible Solution

Revise the PR to suppress stale `lastAssistant` fallback only for externally interrupted runs with no new assistant output, while preserving timeout error delivery, usage persistence, compaction accounting, rotated transcript refresh, and post-compaction context injection. Add regression coverage for the interrupt replay case plus timeout and compaction safeguards.

## Reproduction Assessment

Yes. The issue has a high-confidence reproduction path: set `messages.queue.mode` to `interrupt`, send a second message while the first run is active, and observe the aborted run path where `assistantTexts` can be empty while `lastAssistant` still comes from prior session history.

## Solution Assessment

No. Guarding the stale `lastAssistant` fallback is the right direction, but using a broad `runResult.meta?.aborted` early return in `runReplyAgent` is not the narrowest maintainable fix because timeouts and completed compactions also flow through aborted metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Don't drop timeout replies for aborted runs:** `src/auto-reply/reply/agent-runner.ts:483-485`
  - body: Timeouts set `aborted = true` and `run.ts` synthesizes a user-facing timeout error payload with `meta.aborted`. Returning here before payload delivery makes timed-out auto-reply turns silent instead of sending the timeout notice.
  - confidence: 0.91
- **[P2] Preserve compaction bookkeeping before exiting:** `src/auto-reply/reply/agent-runner.ts:483-485`
  - body: This early return runs before the existing `autoCompactionCount` handling, so an interrupt that lands after completed compaction can skip compaction count persistence and the post-compaction context refresh for the next turn.
  - confidence: 0.86
