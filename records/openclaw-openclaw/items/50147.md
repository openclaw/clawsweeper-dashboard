## Summary

Keep this PR open: the reported interrupt replay bug is source-reproducible and the stale `lastAssistant` guard is useful, but the proposed auto-reply early return treats all aborted runs as silent and would regress timeout payload delivery plus compaction bookkeeping.

## What This Changes

The PR suppresses `lastAssistant` fallback text for aborted empty embedded attempts and skips block flush plus payload delivery for aborted auto-reply runs.

## Best Possible Solution

Keep the narrow stale `lastAssistant` suppression, preserve timeout/error payload delivery and compaction accounting before any interrupt-specific silent exit, then add regression coverage and the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows `interrupt` aborts the active run, an aborted empty attempt can still carry the previous `lastAssistant` from the full message snapshot, and current main still feeds that value into the payload fallback.

## Solution Assessment

No. Guarding stale fallback text is the right direction, but treating every `meta.aborted` result as undeliverable is too broad because timeout and compaction paths use aborted metadata for intentional outcomes.
