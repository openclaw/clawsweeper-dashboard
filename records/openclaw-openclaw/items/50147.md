## Summary

Keep open: the PR targets a real interrupt-mode stale-reply bug, but the patch is still too broad because it drops legitimate aborted timeout payloads and skips compaction bookkeeping.

## What This Changes

The PR guards aborted empty embedded attempts from using `lastAssistant` and skips block flush plus payload delivery for any aborted auto-reply run.

## Best Possible Solution

Keep the narrow stale `lastAssistant` suppression, preserve timeout/error payload delivery and compaction accounting before any interrupt-specific silent exit, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows `interrupt` aborts the active run, an aborted empty attempt can still carry a previous `lastAssistant`, and current main still feeds that value into the payload fallback.

## Solution Assessment

No. Guarding the stale fallback is the right direction, but treating every `meta.aborted` result as undeliverable is too broad because timeout and compaction paths also use aborted metadata for intentional outcomes.
