## Summary

Keep open. Current main still hardcodes the bare `/new` and `/reset` startup prompt, has no root or per-agent `session.resetPrompt` config surface, and the same duplicate prompt-override request was already closed in favor of this issue while the greeting-model and WebChat acknowledgement issues track different behavior.

## What This Changes

Add a global and per-agent configurable reset startup prompt for `/new` and `/reset`, falling back to the current built-in prompt when unset.

## Best Possible Solution

Add a typed reset-prompt resolver that applies per-agent over root full replacement for the normal bare reset prompt, falls back to the built-in prompt when unset, and updates schema, types, help, docs, and focused tests.

## Reproduction Assessment

Yes, source-level. Current main still routes bare reset model-input paths through `resolveBareSessionResetPromptState()` and strict config schemas expose no requested root or per-agent override shapes.

## Solution Assessment

Yes. A full replacement prompt with current behavior as fallback is the narrowest maintainable solution for this request, while reset archival, greeting model choice, greeting suppression, and WebChat lifecycle rendering should stay separate.
