## Summary

Keep open. Current main and v2026.5.12 still build reset startup messages from hardcoded `BARE_SESSION_RESET_PROMPT_*` strings, and the strict root/per-agent config schemas expose no `resetPrompt` override; duplicate cleanup has already made this the canonical prompt-config tracker while adjacent greeting/model issues remain separate.

## What This Changes

The issue requests a global and per-agent `session.resetPrompt` setting to replace the built-in `/new` and `/reset` startup prompt while preserving the current fallback when unset.

## Best Possible Solution

Add a typed reset-prompt resolver with maintainer-approved precedence, preserve the built-in fallback and bootstrap-specific behavior, and update schema, types, help/docs, and focused reset prompt/Gateway/config tests.

## Reproduction Assessment

Yes, source-level. Current main still routes reset model-input paths through `resolveBareSessionResetPromptState()` and strict root/per-agent schemas expose no requested reset-prompt override shape.

## Solution Assessment

Unclear until maintainers settle the public config contract. A full replacement prompt with the current behavior as fallback looks narrow, but bootstrap-specific prompts and nearby reset-greeting policy need deliberate handling.
