## Summary

Keep open. Current main still has a hardcoded bare `/new` and `/reset` startup prompt with no root or per-agent config override, and duplicate prompt-override reports have been closed back to this issue as the canonical tracker; the remaining blocker is the public config contract and precedence decision.

## What This Changes

Request a global and per-agent `session.resetPrompt` setting that replaces the built-in session reset startup prompt while preserving the current default when unset.

## Best Possible Solution

Define a typed reset-prompt resolver with maintainer-approved precedence, preserve the built-in fallback and bootstrap-specific behavior, then update schema, exported types, config help/docs, and focused reset prompt/Gateway/config tests together.

## Reproduction Assessment

Yes, source-level. Current main routes bare reset/new through `resolveBareSessionResetPromptState()` and strict root/per-agent schemas expose no requested reset-prompt override shape.

## Solution Assessment

Unclear until maintainers settle the public config contract. A full replacement prompt with current behavior as fallback looks narrow, but bootstrap-specific prompts and adjacent reset-greeting policy need deliberate handling.
