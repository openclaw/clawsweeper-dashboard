## Summary

Keep open. Current main and v2026.5.12 still build bare `/new` and `/reset` startup text from hardcoded prompt constants, while the strict root and per-agent config schemas expose no `session.resetPrompt` override; related searches show this remains the canonical prompt-config tracker, with greeting/suppression bugs tracked separately.

## What This Changes

The issue requests a new global and per-agent `session.resetPrompt` config key to override the hardcoded `/new` and `/reset` startup message.

## Best Possible Solution

Add a typed reset-prompt resolver with maintainer-approved precedence, preserve the built-in fallback and bootstrap-specific behavior, and update schema, types, config docs/help, and focused reset prompt/Gateway/config tests.

## Reproduction Assessment

Yes, source-level. Current main still routes reset model-input paths through `resolveBareSessionResetPromptState()` and strict root/per-agent schemas expose no requested reset-prompt override shape.

## Solution Assessment

Unclear until maintainers settle the public config contract. A full replacement prompt with the current behavior as fallback looks narrow, but bootstrap-specific prompts and nearby reset-greeting policy need deliberate handling.
