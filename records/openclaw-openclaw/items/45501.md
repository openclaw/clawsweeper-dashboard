## Summary

Keep open. Current main still hardcodes the normal bare reset/startup prompt, exposes no root or per-agent reset-prompt override, and related duplicate cleanup points this prompt-config work back to this issue while the greeting-model and WebChat acknowledgement reports track different behavior.

## What This Changes

The issue requests a global and per-agent `session.resetPrompt` config key to replace the hardcoded bare `/new` and `/reset` startup prompt while preserving the existing default when unset.

## Best Possible Solution

Add a typed reset-prompt resolver that applies per-agent over root full replacement for the normal bare reset prompt, preserves the built-in fallback when unset, and updates schema, types, help/docs, and focused reply/Gateway/config tests while keeping bootstrap-specific behavior explicit.

## Reproduction Assessment

Yes, source-level. Current main still routes reset model-input paths through `resolveBareSessionResetPromptState()` and strict root/per-agent schemas expose no requested reset-prompt override shape.

## Solution Assessment

Unclear until maintainers settle the public config contract. A full replacement prompt with current behavior as fallback is likely the narrowest direction, but bootstrap-specific prompts and nearby reset-greeting policy must be handled deliberately.
