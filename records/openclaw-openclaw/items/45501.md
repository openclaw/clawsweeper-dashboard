## Summary

Keep this open: current main and the latest release still have no approved reset-prompt override, while the request remains a real config/product decision rather than stale cleanup.

## What This Changes

Requests a global and per-agent configuration option to customize the startup prompt used around session resets.

## Best Possible Solution

Define one typed reset-start policy that preserves core bootstrap/time safeguards, keeps static acknowledgement as the default where intended, and exposes any approved customizable prompt layer consistently across schema, docs, gateway/reply paths, and tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 still hardcode the reset startup prompt and strict schemas do not accept the requested global or per-agent key. I did not live-reproduce the Ollama-specific tool-call loop in this read-only review.

## Solution Assessment

No, not as a straight unrestricted full replacement. A bounded, maintainer-approved reset-start policy is safer because the current builder also owns bootstrap and current-time safeguards.
