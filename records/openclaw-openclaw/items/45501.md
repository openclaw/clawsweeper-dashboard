## Summary

Keep open. Current main still hardcodes the bare `/new` and `/reset` startup prompt and has no root or per-agent config surface for `session.resetPrompt`; the discussion also includes concrete Ollama/gemma4 impact tied to the exact `read the required files` wording.

## What This Changes

Add a configurable session reset startup prompt with default fallback and per-agent override support.

## Best Possible Solution

Implement one shared reset-prompt resolver that supports full replacement text with current behavior as fallback, applies per-agent-over-root precedence, and keeps reset archival, greeting model selection, suppression, and WebChat synthetic-bubble behavior as separate issues.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a bare `/new` or `/reset` through the auto-reply or gateway agent path reaches `resolveBareSessionResetPromptState()`, and current strict schemas reject the requested root and per-agent override shapes.

## Solution Assessment

Yes, the requested full replacement prompt with fallback is the narrowest maintainable fix for the reported wording problem. The broader pre-reset snapshot and multi-session ideas should stay separate because current code already has distinct reset/archive and hook surfaces.
