## Summary

Keep open. Current main still hardcodes the bare `/new` and `/reset` startup prompt and has no root or per-agent `session.resetPrompt` surface, while duplicate prompt-override reports have already been closed back to this as the canonical tracker.

## What This Changes

The issue requests a global and per-agent `session.resetPrompt` configuration key to replace the hardcoded startup prompt injected after `/new` or `/reset` while preserving the current default when unset.

## Best Possible Solution

Define a typed reset-prompt resolver with maintainer-approved precedence, preserve the built-in fallback and bootstrap-specific behavior, then update schema, exported types, help/docs, and focused reset prompt/Gateway/config tests together.

## Reproduction Assessment

Yes, source-level. Current main routes bare reset/new through `resolveBareSessionResetPromptState()` and strict root/per-agent schemas expose no requested reset-prompt override shape.

## Solution Assessment

Unclear until maintainers settle the public config contract. A full replacement prompt with current behavior as fallback looks narrow, but bootstrap-specific prompts and adjacent reset-greeting policy need deliberate handling.
