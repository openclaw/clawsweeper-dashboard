## Summary

Keep open. Current main still has a hardcoded bare `/new` and `/reset` startup prompt, and neither the root session schema nor the per-agent schema accepts the requested `session.resetPrompt` override; the related open items cover greeting model/suppression and WebChat synthetic acknowledgements, not this exact config surface.

## What This Changes

Add a configurable `session.resetPrompt` with global and per-agent override support for the bare session reset startup prompt.

## Best Possible Solution

Add a shared reset-prompt resolver that applies per-agent-over-root full replacement only for the normal base reset prompt, keeps the current fallback when unset, and documents the new config without folding in snapshot, greeting-model, suppression, or WebChat synthetic-bubble changes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: bare reset/new flows that use `resolveBareSessionResetPromptState()` still receive the built-in prompt, and strict config schemas reject the requested root and per-agent override shapes.

## Solution Assessment

Yes. A full replacement prompt with current behavior as fallback is the narrowest maintainable solution for this request; reset archival, greeting model choice, greeting suppression, and WebChat acknowledgement rendering should remain separate issues.

## Repair Work Prompt

Implement issue #45501 as a focused OpenClaw config/runtime feature. Add a documented `session.resetPrompt?: string` root config key and a per-agent override using the requested `agents.list[].session.resetPrompt` shape, with per-agent taking precedence over root and unset falling back to the current built-in prompt. Thread the effective prompt through the shared bare reset prompt resolver used by `src/auto-reply/reply/get-reply-run.ts` and `src/gateway/server-methods/agent.ts`; keep current time appending behavior. Treat the custom value as a full replacement for `BARE_SESSION_RESET_PROMPT_BASE` only, unless maintainer guidance says to override bootstrap-pending/limited prompts too. Do not implement pre-reset backup/snapshotting, reset greeting model selection, greeting suppression, WebChat synthetic-bubble changes, or unrelated session architecture work. Add/update regression tests for root and per-agent precedence, schema acceptance, fallback behavior, and both auto-reply and Gateway reset paths. Update config docs/generated schema help as required, and add a user-facing changelog entry because this adds a public config option.
