## Summary

Keep open. Current main still hardcodes the bare `/new` and `/reset` startup prompt and exposes no root or per-agent `session.resetPrompt` config surface; the duplicate prompt-override issue was closed in favor of this one, while the open greeting-model and WebChat acknowledgement issues cover different reset behavior.

## What This Changes

The issue requests a root and per-agent `session.resetPrompt` config key that replaces the default bare session reset startup message while preserving fallback behavior.

## Best Possible Solution

Add a typed reset-prompt resolver that applies per-agent over root full replacement for the normal bare reset prompt, falls back to the built-in prompt when unset, and updates schema/types/help/docs/tests.

## Reproduction Assessment

Yes, source-level. Bare reset paths still use `resolveBareSessionResetPromptState()` and strict config schemas reject the requested root and per-agent override shapes.

## Solution Assessment

Yes. A full replacement prompt with current behavior as fallback is the narrowest maintainable solution for this request, while reset archival, greeting model choice, greeting suppression, and WebChat lifecycle rendering should stay separate.
