## Summary

Keep open. Current main still builds the `/new` and `/reset` startup text from hardcoded reset-prompt constants, and neither the root session config nor per-agent config schema exposes the requested override; the issue remains the canonical tracker for this narrower prompt-config request.

## What This Changes

Request a global and per-agent `session.resetPrompt` config key that can replace the built-in `/new` and `/reset` startup prompt while preserving the current default when unset.

## Best Possible Solution

Define a typed reset-prompt resolver with maintainer-approved precedence, preserve the built-in fallback and bootstrap-specific behavior, then update schema, types, config help/docs, and focused reset prompt/Gateway/config tests together.

## Reproduction Assessment

Yes, source-level for the requested gap. Current main routes reply-producing and Gateway bare reset paths through `resolveBareSessionResetPromptState()`, and strict root/per-agent schemas expose no requested reset-prompt override shape.

## Solution Assessment

Unclear until maintainers settle the public config contract. A full replacement prompt with current behavior as fallback looks narrow, but bootstrap-specific prompts and adjacent reset-greeting policy need deliberate handling.
