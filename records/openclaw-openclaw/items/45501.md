## Summary

Keep open: current main and v2026.6.6 still lack the requested reset-prompt override, and adding it is a public config/session-start policy choice rather than stale cleanup. The issue discussion adds useful narrowing: GMTekAI correctly scopes this to prompt configurability, and tenants-manzel adds model-specific evidence that the hardcoded wording can trigger unwanted tool calls.

## What This Changes

Request to add global and per-agent `session.resetPrompt` configuration that overrides the hardcoded bare `/new` and `/reset` startup prompt while preserving the current default when unset.

## Best Possible Solution

Define one typed reset-start policy that preserves bootstrap/current-time safeguards and wires any approved global/per-agent prompt override consistently through schema, docs, reply/gateway paths, and tests.

## Reproduction Assessment

Yes, at source level: current main still hardcodes the reset startup prompt and strict schemas do not accept the requested global or per-agent key. I did not live-reproduce the Ollama-specific tool-call loop in this read-only review.

## Solution Assessment

No, not as an unrestricted one-line full replacement. The better fix is a maintainer-approved reset-start policy that makes the customizable layer explicit while preserving bootstrap, current-time, and reset-entry-point semantics.
