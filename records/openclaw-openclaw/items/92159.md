## What This Changes

The PR adds a fallback in `src/media-understanding/image.ts` that looks up the bundled static catalog and copies its input capabilities when a resolved image model lacks `image` input.

## Best Possible Solution

Fix the const reassignment, preserve explicit text-only config semantics while borrowing catalog image metadata only for omitted capability data, and add focused omitted-versus-explicit regression coverage plus real tool proof.

## Reproduction Assessment

Yes from source inspection, though I did not run a live tool call in this read-only review. Inline provider models are resolved before catalog fallback, missing input normalizes to `['text']`, and `resolveImageRuntime` rejects that model before the provider call.

## Solution Assessment

No. The intended fallback is plausible, but this patch is not the best fix until it compiles and preserves the difference between omitted input metadata and explicit text-only configuration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Make `model` mutable before assigning catalog input:** `src/media-understanding/image.ts:215`
  - body: This branch assigns to `model` inside the new fallback, but the same function still declares it with `const { model } = resolved` just above. TypeScript will reject this with `Cannot assign to 'model' because it is a constant`; use a mutable local or a separate resolved model value before passing it onward.
  - confidence: 0.99
- **[P1] Preserve explicit text-only model inputs:** `src/media-understanding/image.ts:201-215`
  - body: The fallback runs for every resolved model that lacks image input, including a user config that explicitly set `models.providers.<provider>.models[].input` to `['text']`. Current docs define that as the text-only marker, and `resolveProviderModelInput` produces the same `['text']` for omitted and explicit input, so this can send image requests for a model the user marked text-only. Preserve the raw omitted-vs-explicit signal or move the merge into model resolution with a regression test.
  - confidence: 0.88
