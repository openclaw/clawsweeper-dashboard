## Summary

Keep open: the PR targets useful ACPX durability work, but current main now delegates ACPX turns to the published `acpx` runtime package and the patch still has blocking credential-persistence and abort-cleanup issues.

## What This Changes

The PR adds wrapper-owned ACPX route, terminal, stdout, and stderr artifacts with strict terminal finalization, pre-spawn abort handling, collision-safe run IDs, and focused ACPX runtime tests.

## Best Possible Solution

Define durable terminal-truth artifacts in the supported `acpx` runtime package, then expose only sanitized OpenClaw wrapper metadata with regression coverage for MCP env redaction, setup-failure cleanup, pre-spawn aborts, stderr capture, and route-mirror failure.

## Reproduction Assessment

Yes for the PR findings: source inspection of PR head `bb45eb5acb517c8f5755f816778fc0b4cbfb40fb` shows raw route args persisted, the MCP secret-bearing payload path, and the abort listener cleanup gap. Current main does not provide the requested artifact feature, so there is no shipped bug reproduction path to run.

## Solution Assessment

No. The PR is not the best solution as-is because it adds a reusable durability contract inside the OpenClaw wrapper after current main moved ACPX execution to the published `acpx` runtime package, and it persists unsanitized route metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact secret-bearing route args before persisting:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores `params.routeArgs` verbatim, but MCP-enabled runs can pass a `--agent` command built from a base64url payload containing `mcpServers` env values. Persist a sanitized route view or omit sensitive flags so tokens are not copied into durable per-run artifacts.
  - confidence: 0.92
- **[P2] Move artifact creation into the abort cleanup scope:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but listener removal only happens in the later `finally`. If artifact setup throws, a long-lived signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
- **[P3] Avoid duplicate pre-spawn cancellation:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after the listener is attached but before the post-artifact abort check, `onAbort` has already called `cancelOnAbort`; this branch calls it again. Gate or remove the second cancel call to avoid duplicate cancellation side effects.
  - confidence: 0.73
