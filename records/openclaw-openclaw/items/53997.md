## Summary

Keep open: the requested artifact contract is not on current main, and this PR is not mergeable as-is because current main delegates ACPX turns to `acpx/runtime` while the patch adds a local wrapper-owned subprocess durability layer and still persists unsanitized route args.

## What This Changes

The PR adds ACPX wrapper-owned route, terminal, stdout, and stderr artifacts, strict terminal states, pre-spawn abort handling, and tests for run ID allocation and terminal finalization.

## Best Possible Solution

Define terminal-truth artifacts in the supported ACPX runtime package, then have OpenClaw consume a released/pinned ACPX implementation while persisting only sanitized wrapper metadata with regression coverage for MCP env redaction and abort/setup cleanup.

## Reproduction Assessment

Yes for the PR findings: source inspection of PR head shows raw route args persisted, MCP payloads carrying env values, and artifact setup outside abort-listener cleanup. There is no current-main product repro because the artifact feature is absent from current main.

## Solution Assessment

No. The PR is not the best solution as-is because current main delegates execution to `acpx/runtime`, and the patch reintroduces wrapper-owned durability while copying secret-bearing route metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact secret-bearing route args before persisting:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores `params.routeArgs` verbatim, but MCP-enabled runs can pass a `--agent` command containing a base64url payload with `mcpServers` env values. Persist a sanitized route view or omit sensitive flags so tokens are not copied into durable artifacts.
  - confidence: 0.92
- **[P2] Clean up the abort listener if artifact setup fails:** `extensions/acpx/src/runtime.ts:572-584`
  - body: The abort listener is attached before `createRunArtifacts`, but listener removal only happens in the later `finally`. If artifact setup throws, a long-lived signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
- **[P3] Avoid duplicate pre-spawn cancellation:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after the listener is attached but before the post-artifact abort check, `onAbort` has already called `cancelOnAbort`; this branch calls it again. Gate or remove the second cancel call to avoid duplicate cancellation side effects.
  - confidence: 0.73

## Security Review

Status: needs_attention

Summary: Needs attention: the diff introduces a credential-exposure risk by durably persisting raw ACPX route arguments.

Concerns:

- **[high] Secret-bearing route args are persisted:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
