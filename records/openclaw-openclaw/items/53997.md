## What This Changes

The PR adds wrapper-owned ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, abort-before-spawn handling, and tests for the older extension subprocess runtime path.

## Best Possible Solution

Full review comments:

- **[P1] Redact route args before persisting artifacts:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores the full `params.routeArgs` array, and MCP mode builds those args with `--agent <payload>` where the payload encodes `mcpServers` including env values. That copies tokens and other secret-bearing configuration into `route.json` on every turn; persist a sanitized view or omit sensitive flags before writing artifacts.
  - confidence: 0.94
- **[P2] Remove abort listener when artifact setup fails:** `extensions/acpx/src/runtime.ts:572-584`
  - body: The abort listener is attached before `createRunArtifacts(...)`, but listener removal lives in the later `finally`. If artifact root creation or initial writes throw, long-lived signals retain the listener and can later call `cancel` unexpectedly; wrap artifact creation in the same cleanup scope.
  - confidence: 0.88
