## Summary

Keep this PR open for maintainer handling: the durable ACPX artifact idea is useful and overlaps related debugging reports, but the branch is conflicted, lacks real behavior proof, and still has security-sensitive artifact design issues that should not be auto-closed or auto-repaired.

## What This Changes

This PR adds ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal finalization, abort-before-spawn handling, and focused tests.

## Best Possible Solution

Implement durable ACPX terminal/log artifacts at the published `acpx` runtime boundary or through a sanctioned plugin seam, with sanitized metadata, retention/permission policy, and real ACPX smoke proof.

## Reproduction Assessment

Yes for the review findings: PR-head source shows raw spawned route args persisted and abort-listener cleanup gaps. The artifact feature itself is not a current-main bug path to reproduce.

## Solution Assessment

No. The terminal-truth direction is useful, but this PR is not the best merge shape until it is rebased onto the current ACPX runtime ownership boundary and durable artifacts are sanitized by design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact secret-bearing route args before writing artifacts:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores the raw spawned args, but MCP-enabled routes can include encoded payloads with `mcpServers` environment values. Persist a sanitized route view or omit sensitive flags before writing durable artifacts.
  - confidence: 0.93
- **[P2] Clean up abort listeners when artifact setup fails:** `extensions/acpx/src/runtime.ts:568-572`
  - body: `runTurn` attaches the abort listener before `createRunArtifacts`, but listener removal is only in the later `finally`. If artifact setup throws, a long-lived signal can retain a stale listener that calls `cancel` later and accumulates across retries.
  - confidence: 0.87
- **[P3] Avoid duplicate pre-spawn cancellation:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after listener attachment but before the post-artifact abort check, `onAbort` already called `cancelOnAbort`; this branch calls it again, causing duplicate cancellation side effects.
  - confidence: 0.74
