## Summary

Keep open. The PR is not merge-ready because it persists secret-bearing ACPX route arguments, leaks abort listeners on artifact setup failure, lacks external real-behavior proof, and crosses the current ACPX ownership boundary that now delegates reusable runtime behavior to the published `acpx` package.

## What This Changes

The PR adds ACPX wrapper-owned per-run route, terminal, stdout, and stderr artifacts with strict terminal finalization, pre-spawn abort handling, collision-safe run IDs, and focused ACPX runtime tests.

## Best Possible Solution

Define terminal-truth and log artifact support at the supported ACPX runtime boundary, then consume it through a narrow OpenClaw wrapper seam with sanitized metadata, privacy/retention policy, regression tests, and real ACPX proof.

## Reproduction Assessment

Yes for the PR review findings: source inspection of PR head `bb45eb5acb517c8f5755f816778fc0b4cbfb40fb` shows raw route args persisted, the MCP env-bearing argument path, and the abort-listener cleanup gap. The requested artifact feature itself is not present on current main, so there is no shipped bug path to reproduce.

## Solution Assessment

No. The proposed wrapper-owned implementation is not the best merge shape until durable artifacts are sanitized and aligned with the current `acpx/runtime` ownership boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact secret-bearing route args before writing artifacts:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores `params.routeArgs` verbatim, but MCP-enabled runs can build a `--agent` command from a base64url payload containing `mcpServers` env values. Persist a sanitized route view or omit sensitive flags so tokens are not copied into durable state-dir artifacts.
  - confidence: 0.92
- **[P2] Clean up abort listeners when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: `runTurn` attaches the abort listener before `createRunArtifacts`, but listener removal is only in the later `finally`. If artifact setup throws, a long-lived signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
- **[P3] Avoid duplicate pre-spawn cancellation:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after the listener is attached but before the post-artifact abort check, `onAbort` already called `cancelOnAbort`; this branch calls it again. Gate or remove the second call to avoid duplicate cancellation side effects.
  - confidence: 0.73
