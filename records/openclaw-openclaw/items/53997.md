## Summary

Keep open for maintainer ACPX/security review. The PR has useful terminal-truth work, but it persists raw route arguments into durable artifacts, lacks real ACPX behavior proof, and is shaped around a wrapper-owned subprocess loop while current main delegates turns to the published acpx runtime.

## What This Changes

This PR adds ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts with strict terminal finalization, abort-before-spawn handling, and focused tests.

## Best Possible Solution

Implement terminal/log artifacts at the published `acpx` runtime boundary or a small sanctioned plugin seam, with sanitized metadata, retention/permission policy, and real ACPX smoke proof.

## Reproduction Assessment

Yes for the review findings: PR-head source shows raw spawned route args persisted and an abort listener attached before artifact setup. The artifact feature itself is not a current-main bug path to reproduce.

## Solution Assessment

No. The terminal-truth direction is useful, but this implementation is not the best merge shape until it is rebased onto the current `acpx` runtime ownership boundary and durable artifacts are sanitized by design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact secret-bearing route args before writing artifacts:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores the raw spawned args, but MCP-enabled routes can include an encoded payload with `mcpServers` env values. Persist a sanitized route view or omit sensitive flags before writing durable artifacts.
  - confidence: 0.93
- **[P2] Clean up abort listeners when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: `runTurn` attaches the abort listener before `createRunArtifacts`, but listener removal is only in the later `finally`. If artifact setup throws, a long-lived signal can keep a stale listener that calls `cancel` later and accumulates across retries.
  - confidence: 0.87
- **[P3] Avoid duplicate pre-spawn cancellation:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after listener attachment but before the post-artifact abort check, `onAbort` already called `cancelOnAbort`; this branch calls it again, causing duplicate cancellation side effects.
  - confidence: 0.74
