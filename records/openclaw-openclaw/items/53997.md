## Summary

Keep open for maintainer review: current main still lacks the proposed ACPX terminal-artifact surface, but this branch implements it inside an older wrapper-owned subprocess path and still has a concrete credential-persistence risk plus abort-cleanup bugs. https://github.com/openclaw/openclaw/pull/53997

## What This Changes

This PR adds wrapper-owned ACPX route, terminal, stdout, and stderr artifacts with strict terminal finalization, pre-spawn abort handling, collision-safe run IDs, and focused ACPX runtime tests.

## Best Possible Solution

Define the durable terminal-truth artifact contract in the supported `acpx` runtime package, then expose only sanitized, documented metadata through the OpenClaw ACPX wrapper with regression coverage for MCP env redaction, setup-failure cleanup, pre-spawn aborts, stderr capture, and route-mirror failure.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection of the PR head shows raw `routeArgs` persisted to `route.json`, a secret-bearing MCP payload path, and abort cleanup gaps. Current main does not provide the requested artifact feature, so there is no shipped bug reproduction path to run locally.

## Solution Assessment

No. The PR is not the best current solution as-is because it adds reusable runtime durability inside the OpenClaw wrapper after current main moved ACPX execution to the published `acpx` runtime package, and it persists unsanitized route metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact secret-bearing route args before persisting:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores `params.routeArgs` verbatim, but MCP-enabled runs can build an `--agent`/proxy payload containing configured `mcpServers` env values. Persist a sanitized route view or omit sensitive flags so API tokens are not copied into durable per-run artifacts.
  - confidence: 0.92
- **[P2] Clean up abort listener when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but removal only happens in the later `finally`. If artifact setup throws, a long-lived signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
- **[P3] Avoid duplicate cancel on pre-spawn aborts:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after the listener is attached but before this post-artifact abort check, `onAbort` has already called `cancelOnAbort`; this branch calls it again. Gate or remove the second cancel call to avoid duplicate cancellation side effects.
  - confidence: 0.73

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete credential-exposure risk by durably persisting raw ACPX route arguments.

Concerns:

- **[high] Secret-bearing route args are persisted:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
