## Summary

Keep open. The terminal-truth direction overlaps valid ACPX diagnostics work, but this PR is merge-conflicting, lacks real behavior proof, persists raw durable artifacts with credential-exposure risk, and is based on a wrapper-owned runtime path that current main moved behind the published `acpx` runtime boundary.

## What This Changes

The PR adds file-backed ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal finalization, collision-safe run IDs, abort-before-spawn handling, and tests in the ACPX extension runtime.

## Best Possible Solution

Define durable ACPX terminal/run artifacts at the published `acpx` runtime boundary or a sanctioned OpenClaw wrapper seam, with sanitized metadata, explicit retention/permission behavior, docs/config as needed, focused tests, and real ACPX smoke proof.
