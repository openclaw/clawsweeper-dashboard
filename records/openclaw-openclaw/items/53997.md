## Summary

Keep open: the terminal-artifact direction is useful, but this PR is not mergeable as-is because it targets an older wrapper-owned ACPX subprocess loop, persists secret-bearing route args, and lacks real ACPX behavior proof.

## What This Changes

The PR adds ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, abort-before-spawn handling, and focused tests.

## Best Possible Solution

Implement durable ACPX terminal/log artifacts at the current published `acpx` runtime boundary, with sanitized metadata, bounded retention/permissions, OpenClaw wrapper integration only where needed, and real ACPX smoke proof.

## Reproduction Assessment

Yes for the PR review findings: the live PR diff shows raw spawned route args persisted and artifact setup outside guaranteed abort-listener cleanup. The artifact feature itself is not a current-main bug path to reproduce.

## Solution Assessment

No. The terminal-truth direction is useful, but this PR is not the best merge shape until it is rebased onto the current ACPX ownership boundary and durable artifacts are sanitized by design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact secret-bearing route args before writing artifacts:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores the raw spawned args. MCP-enabled routes can include encoded agent payloads with environment values, so this copies tokens or other secrets into durable state-dir artifacts; persist only a sanitized route view or omit sensitive args.
  - confidence: 0.93
- **[P2] Clean up abort listeners when artifact setup fails:** `extensions/acpx/src/runtime.ts:568-572`
  - body: `runTurn` attaches the abort listener before `createRunArtifacts`, but cleanup is inside the later `finally`. If artifact creation throws, long-lived signals retain a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.87
- **[P3] Avoid duplicate pre-spawn cancellation:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after listener attachment but before the post-artifact abort check, `onAbort` has already called `cancelOnAbort`; this branch calls it again, causing duplicate cancellation side effects.
  - confidence: 0.74
