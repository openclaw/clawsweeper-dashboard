## Summary

Keep open for maintainer review: the branch targets an older wrapper-owned ACPX turn loop, current main now delegates turns to `acpx/runtime`, and the PR still has a concrete secret-persistence issue plus missing real behavior proof.

## What This Changes

The PR adds wrapper-owned ACPX per-turn `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts with strict terminal finalization, pre-spawn abort handling, tests, and fixture updates.

## Best Possible Solution

Implement terminal/log artifacts at the published `acpx` runtime boundary or a small sanctioned plugin seam, with sanitized metadata, privacy/retention policy, and real ACPX smoke proof.

## Reproduction Assessment

Yes for the review findings: PR-head source shows raw route args persisted and an abort listener attached before artifact setup. The artifact request itself is a new capability, not a current-main bug path to reproduce.

## Solution Assessment

No. The proposed wrapper-owned implementation is not the best merge shape until it is rebased onto current `acpx/runtime` ownership and the durable artifacts are sanitized by design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact secret-bearing route args before writing artifacts:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores the raw spawned args, but MCP-enabled routes can include an encoded payload with `mcpServers` env values. Persist a sanitized route view or omit sensitive flags before writing durable artifacts.
  - confidence: 0.93
- **[P2] Clean up abort listeners when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: `runTurn` attaches the abort listener before `createRunArtifacts`, but listener removal is only in the later `finally`. If artifact setup throws, a long-lived signal can keep a stale listener that calls `cancel` later and accumulates across retries.
  - confidence: 0.87
- **[P3] Avoid duplicate pre-spawn cancellation:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after listener attachment but before the post-artifact abort check, `onAbort` already called `cancelOnAbort`; this branch calls it again, causing duplicate cancellation side effects.
  - confidence: 0.74
