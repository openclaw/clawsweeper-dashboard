## Summary

Keep open for maintainer review rather than cleanup-close. Current main has not implemented the requested ACPX terminal-truth artifact surface, but this PR should not merge as-is because it patches the pre-refactor wrapper path while main delegates ACP runtime behavior to `acpx/runtime`, and the diff still persists secret-bearing route arguments plus an abort-listener cleanup bug.

## What This Changes

This PR adds wrapper-owned ACPX `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal finalization, pre-spawn abort handling, collision-safe run IDs, and focused ACPX runtime tests.

## Best Possible Solution

Move the durable terminal-truth artifact contract into the supported `openclaw/acpx` runtime API, then consume it from the OpenClaw plugin as a thin wrapper. Persist stdout/stderr and terminal state only with sanitized, documented metadata, and cover route mirror failure, setup failure cleanup, pre-spawn aborts, stderr capture, and MCP env redaction in tests.

## Reproduction Assessment

Yes for the blocking findings: on the PR head, configure ACPX `mcpServers.*.env`, run an ACPX turn, and inspect the generated `route.json`; code trace shows `route_args` contains the raw `--agent` payload. No live run was needed for this read-only review because the raw PR head source shows the persistence path directly.

## Solution Assessment

No. The PR is not the best way to solve the issue as-is because it implements reusable runtime durability inside an older wrapper path and persists unsanitized route metadata; the safer solution is an upstream `acpx/runtime` artifact contract with sanitized OpenClaw wrapper exposure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact secret-bearing route args before persisting:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores `params.routeArgs` verbatim, but MCP-enabled runs build an `--agent` payload that includes configured `mcpServers` env values. Persist a sanitized route view or omit sensitive flags so API tokens are not copied into durable artifacts.
  - confidence: 0.92
- **[P2] Clean up abort listener when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but removal only happens in the later `finally`. If artifact setup throws, a long-lived signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
- **[P3] Avoid duplicate cancel on pre-spawn aborts:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after the listener is attached but before the post-artifact abort check, `onAbort` has already called `cancelOnAbort`; this branch calls it again. Gate or remove the second cancel call to avoid duplicate cancellation side effects.
  - confidence: 0.73

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete credential-exposure risk by durably persisting raw ACPX route arguments.

Concerns:

- **[high] Secret-bearing route args are persisted:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
