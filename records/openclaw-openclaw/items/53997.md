## Summary

Keep open for maintainer review, not cleanup-close. Current main does not implement the requested terminal-truth artifact surface, but this PR is not merge-ready because it targets an older wrapper-owned subprocess path after the ACPX runtime was moved behind `acpx/runtime`, and the diff introduces a concrete credential-persistence regression by writing raw route arguments to durable artifacts.

## What This Changes

The PR adds ACPX wrapper-owned per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, run-id allocation, abort-before-spawn handling, and ACPX runtime tests.

## Best Possible Solution

Keep the terminal-truth/debugging goal, but implement it at the current ACPX ownership boundary: preferably as a sanitized artifact/result contract in the published `acpx` runtime, then expose only documented non-secret metadata through the bundled plugin with tests for terminal states, setup-failure cleanup, route mirror failure, stderr capture, and MCP env redaction.

## Reproduction Assessment

Yes for the blocking findings: code trace on the PR head shows `mcpServers.*.env` can be serialized into a base64 `--agent` payload, then `runTurn` passes the full `args` array to `createRunArtifacts`, which writes it as `route_args`. The broader feature request is not a bug reproduction; current main verification is the absence of this artifact surface and the delegated ACPX runtime boundary.

## Solution Assessment

No. This PR is not the best way to solve the issue as-is because it patches an older wrapper-owned subprocess path and durably stores unsanitized route arguments; the safer alternative is a current `acpx/runtime` artifact contract with secret-bearing route data omitted or redacted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact route args before persisting artifacts:** `extensions/acpx/src/runtime.ts:579`
  - body: `args` can include `--agent <payload>` when MCP servers are configured, and that payload encodes `mcpServers` entries including env values. Persisting it verbatim in `route.json` copies credentials into durable artifacts, so store a sanitized route view or omit sensitive flags.
  - confidence: 0.92
- **[P2] Clean up abort listeners when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but listener removal only happens in the later `finally`. If artifact setup throws, for example on an unwritable state directory, the signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
- **[P3] Avoid cancelling twice after pre-spawn aborts:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after the listener is attached but before the post-artifact abort check, `onAbort` already called `cancelOnAbort`; this branch calls it again. Drop the second call or gate it so abort-before-spawn does not send duplicate cancel requests.
  - confidence: 0.73

## Security Review

Status: needs_attention

Summary: The diff introduces a credential-exposure risk by durably persisting raw ACPX route arguments.

Concerns:

- **[high] Secret-bearing route args are persisted:** `extensions/acpx/src/runtime.ts:579`
  - body: `routeArgs: args` is written into durable `route.json`; in MCP mode those args can include an `--agent` payload built from configured MCP servers, including env values. This can copy API tokens or other secrets into per-run artifacts.
  - confidence: 0.92
