## Summary

Keep open. Current main still lacks a channel/account heartbeat OK-recipient contract for WhatsApp: channel heartbeat config is visibility-only, and both OK acknowledgments and alert content use the same resolved agent-level heartbeat delivery target.

## What This Changes

Requested additive channel-level heartbeat recipient routing so WhatsApp heartbeat OK messages can go only to selected admin users instead of all allowed users.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated metadata, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives a high-confidence proof of the missing feature: proposed recipient fields are rejected by strict schema and never consumed by the resolver or delivery path.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.target or to, because those route the whole heartbeat delivery path.
