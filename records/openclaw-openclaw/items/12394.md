## Summary

Keep open. Current main still treats channel heartbeat settings as visibility-only flags and sends OK acknowledgments through the agent heartbeat delivery target, so the requested WhatsApp OK-only per-user routing remains an unimplemented public config/product gap.

## What This Changes

The issue requests optional WhatsApp channel-level heartbeat `target` or `targets` settings so `HEARTBEAT_OK` messages can go only to admin recipients while preserving existing alert routing.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated config metadata/docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main shows the proposed `channels.whatsapp.heartbeat.target` or `targets` fields are neither accepted by schema/types nor consumed by delivery routing.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing the existing agent-level heartbeat `target`/`to`, because that existing route controls the whole heartbeat delivery path, including alert delivery semantics.
