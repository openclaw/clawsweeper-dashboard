## Summary

Keep open: current main and v2026.6.8 still expose channel heartbeat as visibility settings plus one agent-level delivery target, so OK-only admin routing remains missing and needs a maintainer product/config decision.

## What This Changes

The issue requests optional per-user/per-target routing for channel-level WhatsApp heartbeat OK messages so admins can receive health acknowledgments without broadcasting them to every allowed user.

## Best Possible Solution

Define one additive OK-acknowledgment recipient contract, validate recipients through WhatsApp account and allowlist policy, apply it only to visible HEARTBEAT_OK delivery, and preserve current alert/content routing plus omitted-config compatibility.

## Reproduction Assessment

Yes, at source level: current main rejects target/targets in strict channel heartbeat config, resolves only visibility booleans, and sends OK acknowledgments plus alert content through one resolved heartbeat delivery target. I did not run a live WhatsApp heartbeat.

## Solution Assessment

Unclear pending maintainer product direction. Splitting OK acknowledgment recipients from alert/content routing is the right behavioral boundary, but the final config shape and validation rules need owner approval.
