## Summary

Keep open. Current main and the latest shipped tag still expose channel heartbeat settings as visibility-only, while the existing agent heartbeat `target`/`to` route the whole heartbeat delivery path rather than OK-only admin acknowledgments, so this remains a new public config and product decision.

## What This Changes

Request an additive channel-level WhatsApp heartbeat `target`/`targets` option so `HEARTBEAT_OK` messages can go only to admin recipients while alert routing remains unchanged.

## Best Possible Solution

Define and document an additive, validated WhatsApp/channel heartbeat OK-recipient contract in schema, types, docs, and delivery code while preserving current alert routing and omitted-target compatibility.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main and `v2026.5.12` gives high confidence that channel-level heartbeat `target`/`targets` fields are neither accepted by schema nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive, validated OK-recipient option is safer than reusing `agents.defaults.heartbeat.to`, because the existing agent-level route controls the whole heartbeat delivery path.
