## Summary

Keep open. Current main and the latest shipped tag still expose channel heartbeat config as visibility-only, and heartbeat delivery still uses the existing agent-level target/to route rather than an OK-only WhatsApp recipient contract.

## What This Changes

The issue requests optional channel-level heartbeat OK recipients so WhatsApp HEARTBEAT_OK messages can go only to admin targets while preserving existing alert routing.

## Best Possible Solution

If maintainers accept the feature, add a validated WhatsApp/channel heartbeat OK-recipient contract in schema, types, generated help/docs, and delivery code, applying it only to OK acknowledgments while preserving alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main and v2026.5.7 gives high confidence that channel-level heartbeat target/targets fields are neither accepted by schema nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public API. An additive, validated OK-recipient option is safer than reusing agents.defaults.heartbeat.to because the existing agent-level route controls both OK and alert delivery.
