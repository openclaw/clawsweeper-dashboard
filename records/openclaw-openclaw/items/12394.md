## Summary

Keep open. Current main still models channel heartbeat settings as visibility-only, and the requested WhatsApp OK-only admin recipient routing would add a new public config contract plus recipient validation semantics.

## What This Changes

The issue requests optional channel-level WhatsApp heartbeat recipient fields so `HEARTBEAT_OK` acknowledgments can go only to selected admin users while alert routing stays unchanged.

## Best Possible Solution

If maintainers accept the feature, add a validated WhatsApp/channel heartbeat OK-recipient contract, apply it only to OK acknowledgments, preserve alert/content routing, and cover schema, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that channel-level heartbeat `target` or `targets` fields are neither accepted nor consumed on current main.

## Solution Assessment

Unclear until maintainers choose the public API. An additive, validated OK-ack recipient option is safer than reusing agent-level `heartbeat.to`, because the existing agent-level route controls the whole heartbeat delivery path.
