## Summary

Keep open: current main and the latest release still expose channel heartbeat as visibility-only config and route both OK acknowledgments and alert payloads through one agent-level heartbeat destination.

## What This Changes

Requested per-user routing for WhatsApp channel-level heartbeat OK messages so admin health checks do not broadcast to every allowed channel user.

## Best Possible Solution

Define an additive WhatsApp/channel heartbeat OK-recipient contract in schema, types, generated metadata, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence proof of the missing feature: proposed recipient fields are rejected by strict schema and never consumed by the visibility resolver or delivery path.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.target/to because the existing agent-level target routes the whole heartbeat delivery path.
