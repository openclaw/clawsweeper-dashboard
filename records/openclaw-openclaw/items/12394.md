## Summary

Keep open. Current main still treats channel heartbeat config as visibility-only and uses the agent-level heartbeat delivery target for both OK acknowledgments and alert content, so the requested WhatsApp OK-only admin routing remains a new public config/product contract.

## What This Changes

Request channel-level heartbeat target/targets fields so WhatsApp HEARTBEAT_OK messages can go only to selected admin recipients while alert routing remains unchanged.

## Best Possible Solution

If maintainers accept the feature, add a validated WhatsApp/channel heartbeat OK-recipient contract in schema/types/docs, apply it only to OK acknowledgments, and preserve existing alert/content routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main gives high confidence that channel-level heartbeat target/targets fields are neither accepted by schema nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public API. An additive, validated OK-recipient option is safer than reusing agents.*.heartbeat.to because the existing agent-level route controls the whole heartbeat delivery path.
