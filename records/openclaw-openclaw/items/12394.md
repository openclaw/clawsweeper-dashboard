## Summary

Keep open. Current main and the latest release still treat channel heartbeat config as visibility-only, so OK-only per-user WhatsApp routing needs a new public config contract and validation semantics rather than cleanup closure.

## What This Changes

Request adds optional channel-level WhatsApp heartbeat target/targets so HEARTBEAT_OK acknowledgments can go only to admins while alert routing stays intact.

## Best Possible Solution

Define and document an additive channel/account heartbeat OK-recipient contract in schema, types, generated metadata/docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection against current main shows channels.whatsapp.heartbeat.target or targets are neither accepted by schema/types nor consumed by visibility or delivery resolvers.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.target/to because the existing agent-level route controls the whole heartbeat delivery path.
