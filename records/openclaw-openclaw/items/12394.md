## Summary

Keep open. Current main still has a visibility-only channel heartbeat contract, and the only recipient override is the agent-level heartbeat delivery target, which would route both OK acknowledgments and alert content rather than OK-only admin messages. Prior review discussion also kept this open for the same missing WhatsApp/channel recipient contract.

## What This Changes

The issue requests per-user routing for WhatsApp channel heartbeat OK messages so admins can receive health acknowledgments without broadcasting them to every allowed user.

## Best Possible Solution

Define and document an additive, validated WhatsApp/channel heartbeat OK-recipient contract in schema, types, generated metadata/docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main shows channel-level heartbeat target/targets fields are not accepted by schema or consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.to, because the existing agent-level route controls the whole heartbeat delivery path.
