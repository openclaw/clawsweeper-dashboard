## Summary

Keep open. Current main still treats channel heartbeat config as visibility-only and sends OK acknowledgments to the resolved agent heartbeat delivery target, so the requested OK-only admin routing is not implemented and needs a product/config contract decision.

## What This Changes

Request adding optional channel-level heartbeat `target`/`targets` recipients so WhatsApp `HEARTBEAT_OK` messages can go only to admins instead of every allowed user.

## Best Possible Solution

Define and document an additive WhatsApp/channel heartbeat OK-recipient contract in schema, types, generated metadata/docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection shows `channels.whatsapp.heartbeat.target` or `targets` are not accepted by schema or consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing `agents.defaults.heartbeat.target`/`to`, because the existing agent-level route controls the whole heartbeat delivery path.
