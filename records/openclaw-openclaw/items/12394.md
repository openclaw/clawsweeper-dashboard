## Summary

Keep open. Current main still does not expose a channel/account heartbeat OK-recipient contract for WhatsApp: the public config accepts visibility flags only, and delivery still resolves one agent-level heartbeat target instead of OK-only admin recipients.

## What This Changes

The issue requests optional WhatsApp channel heartbeat `target` or `targets` config so `HEARTBEAT_OK` messages can go only to admins while preserving alert routing.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated metadata, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection against current main shows the proposed channel heartbeat recipient fields are rejected by strict schema and are never consumed by resolver or delivery code.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing `agents.defaults.heartbeat.target` or `to`, because those route the whole heartbeat delivery path.
