## Summary

Keep this open. Current main still only supports channel heartbeat visibility flags, while the requested WhatsApp OK-only admin recipient routing would add a new public config contract.

## What This Changes

Request adds optional channel-level heartbeat target/targets routing so WhatsApp HEARTBEAT_OK messages can go only to admins while alerts continue to route normally.

## Best Possible Solution

Define a validated channel/account heartbeat recipient option for OK acknowledgments, preserve existing alert routing, and update schema, docs, and regression tests if maintainers accept the feature.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main neither accepts nor consumes channel-level heartbeat target or targets fields.

## Solution Assessment

Unclear until maintainers choose the public config contract. An additive, validated OK-ack recipient option is safer than reusing agents.defaults.heartbeat.to because that existing route changes the whole heartbeat delivery target.
