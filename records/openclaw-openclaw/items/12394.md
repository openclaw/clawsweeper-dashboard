## Summary

Keep open. Current main and the latest release still model channel heartbeat config as visibility-only, while HEARTBEAT_OK delivery uses the agent heartbeat target/to route, so OK-only admin routing needs a new public config contract rather than cleanup closure.

## What This Changes

The issue requests optional WhatsApp/channel heartbeat recipients so HEARTBEAT_OK messages can go only to admin users while alert/content routing remains unchanged.

## Best Possible Solution

Define and document an additive WhatsApp/channel heartbeat OK-recipient contract in schema, types, generated metadata/docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection against current main shows channels.whatsapp.heartbeat.target or targets are neither accepted by schema/types nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.target/to because the existing agent-level route controls the whole heartbeat delivery path.
