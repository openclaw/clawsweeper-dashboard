## Summary

Keep open. Current main and the latest release still expose channel/account heartbeat as visibility-only config; WhatsApp inherits that strict shape, and HEARTBEAT_OK delivery still uses the agent-level heartbeat target/to route rather than channel-level target or targets.

## What This Changes

The issue requests per-user WhatsApp heartbeat OK routing through channel/account heartbeat target or targets fields while preserving existing alert routing.

## Best Possible Solution

Define an additive WhatsApp/channel heartbeat OK-recipient contract in schema, types, generated metadata, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection against current main shows the proposed target/targets fields are rejected by the strict channel heartbeat schema and never carried into resolver or delivery code.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.target/to because that route controls the whole heartbeat delivery path.
