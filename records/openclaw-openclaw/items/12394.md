## Summary

Keep open. Current main still has visibility-only channel heartbeat config and agent-level `target`/`to` delivery, so WhatsApp OK-only per-user heartbeat routing remains a real feature/config gap; prior Codex and ClawSweeper comments reached the same conclusion and live GitHub data shows no linked closing PR.

## What This Changes

The issue requests optional channel-level WhatsApp heartbeat OK recipients so admin heartbeat acknowledgments do not broadcast to every allowed user.

## Best Possible Solution

Define and document an additive channel/account heartbeat OK-recipient contract in schema, types, generated metadata/docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main shows `channels.whatsapp.heartbeat.target` or `targets` are neither accepted by schema/types nor consumed by visibility or delivery resolvers.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only channel/account recipient option is safer than reusing `agents.defaults.heartbeat.target/to`, because the existing agent-level route controls the whole heartbeat delivery path.
