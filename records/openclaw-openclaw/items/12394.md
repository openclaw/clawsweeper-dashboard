## Summary

Keep open. Current main still lacks a channel/account OK-recipient contract: schema, types, and docs expose only heartbeat visibility flags, while heartbeat delivery still resolves through the agent-level target/to route.

## What This Changes

Add optional WhatsApp channel heartbeat recipients so HEARTBEAT_OK messages can go only to admin users without changing alert delivery.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated metadata, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection against current main shows the proposed target/targets fields are rejected by the strict channel heartbeat schema and are never consumed by resolver or delivery code.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.target/to because that route controls the whole heartbeat delivery path.
