## Summary

Keep open. Current main still only supports channel heartbeat visibility flags and routes heartbeat delivery through the agent-level `target`/`to` path, so WhatsApp OK-only admin routing remains unimplemented and needs a public config decision.

## What This Changes

Add optional WhatsApp channel heartbeat `target`/`targets` config so `HEARTBEAT_OK` can go only to admins while alerts keep their normal routing.

## Best Possible Solution

If maintainers accept the feature, add a validated WhatsApp/channel heartbeat OK-recipient contract in schema, types, generated docs, and delivery code, applying it only to OK acknowledgments while preserving existing alert/content routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main gives high confidence that channel-level heartbeat `target`/`targets` fields are neither accepted by schema nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public API. An additive, validated OK-recipient option is safer than reusing `agents.*.heartbeat.to` because the existing agent-level route controls the whole heartbeat delivery path.
