## Summary

Keep this open. Current main still models channel heartbeat config as visibility-only and routes heartbeat delivery through agent-level `heartbeat.target`/`heartbeat.to`, so OK-only per-admin WhatsApp routing remains an unimplemented public config feature.

## What This Changes

Request adding optional WhatsApp channel heartbeat `target`/`targets` fields so `HEARTBEAT_OK` acknowledgments can go only to admin recipients while alert delivery keeps existing routing.

## Best Possible Solution

Define an additive, documented WhatsApp/channel heartbeat recipient contract only if maintainers want this behavior, scoped to OK acknowledgments while preserving alert routing and allowlist safety.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main neither accepts nor consumes channel-level heartbeat `target` or `targets` fields.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is an additive validated recipient option for OK acknowledgments, rather than reusing `agents.defaults.heartbeat.to` because that changes the whole heartbeat delivery target.
