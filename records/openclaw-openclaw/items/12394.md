## Summary

Keep this open. Current main still treats channel heartbeat config as visibility-only and routes heartbeat OK delivery through the existing agent heartbeat target, so the requested WhatsApp OK-only admin recipient contract remains unimplemented and needs a maintainer product/config decision.

## What This Changes

The issue requests optional channel-level heartbeat target or targets fields so WhatsApp HEARTBEAT_OK messages can go only to selected admins while preserving alert routing.

## Best Possible Solution

Define an additive, documented WhatsApp/channel heartbeat recipient contract only if maintainers want this behavior, scoped to OK acknowledgments while preserving existing alert delivery semantics and allowlist safety.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main neither accepts nor consumes channel-level heartbeat target or targets fields.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is an additive validated recipient option for OK acknowledgments, rather than reusing agents.defaults.heartbeat.to because that changes the whole heartbeat delivery target.
