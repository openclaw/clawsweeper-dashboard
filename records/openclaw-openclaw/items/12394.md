## Summary

Keep open. Current main and the latest shipped tag still expose channel heartbeat settings as visibility-only, so the requested WhatsApp OK-only recipient routing remains a new public config/API decision rather than completed cleanup.

## What This Changes

Request adds optional single or multiple WhatsApp heartbeat OK recipients so admin acknowledgments are not broadcast to every `allowFrom` user.

## Best Possible Solution

Define an additive, validated WhatsApp/channel heartbeat OK-recipient contract in schema, types, docs, and delivery code while preserving existing alert routing semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main and `v2026.5.12` gives high confidence that channel-level heartbeat `target`/`targets` fields are neither accepted by schema nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive, validated OK-recipient option is safer than reusing `agents.defaults.heartbeat.to`, because the existing agent-level route controls the whole heartbeat delivery path.
