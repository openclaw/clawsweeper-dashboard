## Summary

Keep open. Current `main` and the latest shipped tag still expose channel heartbeat settings as visibility-only, while heartbeat delivery continues to use the agent-level `target`/`to` route rather than an OK-only WhatsApp recipient contract.

## What This Changes

The issue requests channel-level heartbeat `target`/`targets` routing so WhatsApp `HEARTBEAT_OK` messages can go only to admin users while preserving alert routing.

## Best Possible Solution

If maintainers accept the feature, add a validated WhatsApp/channel heartbeat OK-recipient contract in schema, types, docs/help, and delivery code, applying it only to OK acknowledgments while preserving alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current `main` and `v2026.5.12` gives high confidence that channel-level heartbeat `target`/`targets` fields are neither accepted by schema nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive, validated OK-recipient option is safer than reusing `agents.defaults.heartbeat.to`, because the existing agent-level route controls the whole heartbeat delivery path.
