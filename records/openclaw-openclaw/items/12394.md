## Summary

Keep open. Current main still has only visibility flags for channel heartbeat config, and both OK acknowledgments and alert delivery are routed through the single agent-level heartbeat delivery target rather than a WhatsApp channel/account OK-recipient filter.

## What This Changes

The issue requests optional WhatsApp channel heartbeat recipients so `HEARTBEAT_OK` messages can go only to admin targets instead of every allowed user.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated config metadata/docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main shows the proposed `channels.whatsapp.heartbeat.target` or `targets` fields are neither accepted by schema/types nor consumed by delivery routing.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing the existing agent-level heartbeat `target`/`to`, because that route controls the whole heartbeat delivery path, including alert delivery semantics.
