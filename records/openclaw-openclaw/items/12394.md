## Summary

Keep open. Current main still exposes channel heartbeat as visibility-only config, and `HEARTBEAT_OK` delivery uses the same agent-level heartbeat `target`/`to` route as alert delivery rather than a WhatsApp channel/account OK-recipient filter.

## What This Changes

Request an additive WhatsApp channel heartbeat config that routes `HEARTBEAT_OK` acknowledgments only to selected admin recipients while preserving existing alert routing.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated config metadata/docs, and delivery code while preserving omitted-target compatibility and current alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main confirms the proposed `channels.whatsapp.heartbeat.target` or `targets` fields are neither accepted by schema/types nor consumed by delivery routing.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing the existing agent-level heartbeat `target`/`to`, because that route controls the whole heartbeat delivery path, including alert delivery semantics.
