## Summary

Keep this issue open: current main still uses one heartbeat delivery route for both healthy acknowledgments and alert content, so it cannot notify only administrators about `HEARTBEAT_OK` while preserving user-specific alert routing.

## What This Changes

The issue requests per-user recipient controls for channel-level heartbeat acknowledgments while preserving normal alert routing.

## Best Possible Solution

Add a distinct healthy-acknowledgment route at the narrowest appropriate owner boundary, preserving alert and commitment delivery on their existing session-derived routes and validating WhatsApp recipients against the selected account's authorization policy.

## Reproduction Assessment

Yes, at source level. Current main resolves one heartbeat delivery recipient before applying `showOk` and `showAlerts`, and the channel heartbeat schema provides no acknowledgment-only recipient selector.

## Solution Assessment

Not yet. `target` or `targets` is a plausible interface, but maintainers should first decide whether the contract is shared or WhatsApp-owned and define account precedence, authorization validation, and alert-routing invariants.
