## Summary

Keep this issue open: current main and v2026.6.6 still only support heartbeat visibility toggles, not a channel/account-level OK-recipient contract, and the requested split between admin OK acknowledgments and normal alert routing needs maintainer product direction.

## What This Changes

The issue requests channel-level WhatsApp heartbeat recipient routing so HEARTBEAT_OK acknowledgments can go only to selected admin targets while alert delivery keeps its normal routing.

## Best Possible Solution

Define one canonical channel-owner recipient contract for heartbeat OK acknowledgments, validate WhatsApp recipients through existing target policy, apply it only to HEARTBEAT_OK delivery, and preserve current alert/content routing and omitted-config behavior.

## Reproduction Assessment

Yes, at source level: current main rejects target/targets in strict channel heartbeat config and sends OK acknowledgments only to the single resolved delivery target. I did not run a live WhatsApp heartbeat, so this is not an executed runtime repro.

## Solution Assessment

Unclear. Separating OK acknowledgment recipients from alert/content routing is the right behavioral boundary, but the final config shape and owner boundary need maintainer product judgment before implementation.
