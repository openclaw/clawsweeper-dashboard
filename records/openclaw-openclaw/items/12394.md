## Summary

Keep open. Current main still treats channel heartbeat config as visibility-only, and the existing heartbeat delivery target applies to the whole heartbeat path rather than OK-only admin acknowledgments.

## What This Changes

The issue requests optional channel-level WhatsApp heartbeat `target`/`targets` routing so `HEARTBEAT_OK` messages can go only to admin recipients while alerts keep their normal routing.

## Best Possible Solution

Define and document an additive, validated WhatsApp/channel heartbeat OK-recipient contract in schema, types, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main and v2026.5.12 gives high confidence that channel-level heartbeat `target`/`targets` fields are neither accepted by schema nor consumed by delivery.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive, validated OK-recipient option is safer than reusing `agents.defaults.heartbeat.to`, because the existing agent-level route controls the whole heartbeat delivery path.
