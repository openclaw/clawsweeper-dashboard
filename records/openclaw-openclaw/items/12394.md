## Summary

Keep open: current main and the latest release still accept only heartbeat visibility booleans, use agent-level delivery target/to for both OK and alert paths, and lack a WhatsApp channel/account OK-only recipient contract.

## What This Changes

The issue requests optional WhatsApp channel-level heartbeat recipient fields so HEARTBEAT_OK messages can go only to admins while preserving alert routing.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated metadata, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence proof of the missing feature: proposed recipient fields are rejected by strict schema and never consumed by the resolver or OK delivery path.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing agents.defaults.heartbeat.target or to, because those route the whole heartbeat delivery path.
