## Summary

Keep open. Current main still only supports channel heartbeat visibility flags, WhatsApp recipient resolution does not read channel-level `target` or `targets`, and the existing agent-level heartbeat `target`/`to` route the whole heartbeat delivery rather than OK-only acknowledgments.

## What This Changes

Request an optional WhatsApp channel-level heartbeat recipient config so `HEARTBEAT_OK` messages can go only to selected admins while preserving alert routing for affected users.

## Best Possible Solution

Add a backward-compatible WhatsApp/channel heartbeat recipient contract that validates configured recipients, applies it only to `HEARTBEAT_OK` delivery when `showOk` is enabled, preserves alert/content routing, updates schema/types/generated docs, and covers omitted-target compatibility plus single/multiple target cases.

## Reproduction Assessment

Yes for the missing feature. A config using `channels.whatsapp.heartbeat.target` or `targets` is outside the current strict channel heartbeat schema, and the current WhatsApp recipient resolver never reads those fields.

## Solution Assessment

Unclear until maintainers choose the public config shape and validation rules. The additive OK-only channel recipient direction is plausible, but existing agent-level `target`/`to` is not the best substitute because it changes all heartbeat delivery rather than only admin OK acknowledgments.
