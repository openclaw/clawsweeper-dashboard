## Summary

Keep this open. Current main still treats channel heartbeat config as visibility-only and does not provide a WhatsApp/channel-level target or targets recipient contract for OK-only heartbeat routing; the remaining work is a public config/product decision rather than a narrow bug fix.

## What This Changes

The issue requests optional channel-level heartbeat recipient fields so WhatsApp HEARTBEAT_OK messages can go only to admin target(s) while preserving existing alert routing.

## Best Possible Solution

Define an additive, documented WhatsApp/channel heartbeat recipient contract only if maintainers want this behavior, scoped to HEARTBEAT_OK acknowledgments while preserving existing alert delivery semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives high confidence that current main neither accepts nor consumes channel-level heartbeat target/targets fields.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction would be an additive validated recipient option for OK acknowledgments, rather than reusing agents.defaults.heartbeat.to because that changes overall heartbeat delivery semantics.
