## Summary

Keep open: current main and v2026.6.10 still expose channel heartbeat as visibility-only config plus one heartbeat delivery target, so OK-only per-user WhatsApp routing remains unimplemented and needs a maintainer product/config decision.

## What This Changes

Request to add optional channel-level heartbeat recipient routing for WhatsApp HEARTBEAT_OK acknowledgments so admins can receive OKs without broadcasting them to every allowed user.

## Best Possible Solution

Define one additive OK-acknowledgment recipient contract, validate recipients through WhatsApp account and allowlist policy, apply it only to visible HEARTBEAT_OK delivery, and preserve existing alert/content routing plus omitted-config compatibility.

## Reproduction Assessment

Yes, at source level: current main rejects target/targets in strict channel heartbeat config, resolves only visibility booleans, and sends OK acknowledgments plus alert content through one resolved heartbeat delivery target. I did not run a live WhatsApp heartbeat.

## Solution Assessment

Unclear pending maintainer product direction. Splitting OK acknowledgment recipients from alert/content routing is the right behavioral boundary, but the final config shape and validation rules need owner approval.
