## Summary

Keep this open. Current main still lacks a channel/WhatsApp heartbeat recipient contract for OK-only admin routing, and an existing maintainer/Codex comment already preserved the issue for that same product/config decision.

## What This Changes

The issue requests optional channel-level heartbeat recipient fields so WhatsApp HEARTBEAT_OK messages can go to selected admins without changing alert routing.

## Best Possible Solution

Define an additive, documented channel heartbeat recipient contract only if maintainers want this behavior, scoped to OK acknowledgments while preserving existing alert delivery semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection does give high confidence that the requested channel heartbeat recipient fields are not accepted or consumed on current main.

## Solution Assessment

Unclear until maintainers choose the product contract. The narrow maintainable direction would be an additive target/targets-style config with validation and regression coverage, rather than overloading agents.defaults.heartbeat.to for OK-only routing.
