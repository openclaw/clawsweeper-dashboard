## Summary

Keep open. Current main still lacks a channel/account heartbeat OK-recipient contract: schema and types expose only visibility flags, and heartbeat OK delivery still uses the agent-level delivery target.

## What This Changes

The issue requests optional WhatsApp channel heartbeat `target` or `targets` config so `HEARTBEAT_OK` messages can go only to admin recipients while alerts keep their existing routing.

## Best Possible Solution

Define an additive WhatsApp channel/account heartbeat OK-recipient contract in schema, types, generated metadata, docs, and delivery code while preserving omitted-target compatibility and existing alert routing.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection against current main shows the proposed channel heartbeat recipient fields are rejected by strict schema and are never consumed by resolver or delivery code.

## Solution Assessment

Unclear until maintainers choose the public config API. An additive OK-only recipient option is safer than reusing `agents.defaults.heartbeat.target` or `to` because that route controls the whole heartbeat delivery path.
