## Summary

Keep this open. Current main still has channel heartbeat visibility flags only, while the requested WhatsApp OK-only admin recipient routing would add a new public config contract and recipient validation semantics.

## What This Changes

The issue requests optional per-user targets for WhatsApp channel heartbeat OK messages so admins can receive health acknowledgments without broadcasting them to every allowed user.

## Best Possible Solution

Define a validated WhatsApp/channel heartbeat recipient option for OK acknowledgments, preserve alert routing, and update schema, types, docs, and regression tests if maintainers accept the feature.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection against current main gives high confidence that channel-level heartbeat target or targets fields are neither accepted nor consumed.

## Solution Assessment

Unclear until maintainers choose the public config contract. An additive, validated OK-ack recipient option is safer than reusing agents.defaults.heartbeat.to because the existing agent-level target/to route changes the whole heartbeat delivery path.
