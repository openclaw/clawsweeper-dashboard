## Summary

Keep open: the feature is active and useful, and the supplied live proof is sufficient, but current head still has a sanitizer/parser mismatch in the MCP approval defense and a public changelog line that advertises an unsupported durable decision.

## What This Changes

The PR adds an opt-in MCP consent-envelope flow that gates bundle-MCP tool calls through plugin approvals, adds `mcp.approvals` config/docs/tests, and updates `/approve` parsing for implicit single-pending approvals.

## Best Possible Solution

Repair this PR so sanitizer/tests cover every approval command form the parser accepts, align public release copy with `allow-once`/`deny`, then let maintainers decide and land the consent-envelope contract if they accept the product/security boundary.
