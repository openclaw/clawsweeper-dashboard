## Summary

Keep this PR open: the MCP consent direction is plausible and the supplied live proof is strong, but latest head still has a blocking approval-routing regression in the new idless `/approve` path for session-routed plugin/MCP approvals.

## What This Changes

The PR adds an opt-in MCP consent envelope that routes bundle-MCP tool calls through plugin approvals, plus `/approve` implicit-id UX, config/schema/docs/changelog updates, and regression tests.

## Best Possible Solution

Keep the materializer-level MCP consent gate, but make idless approvals match the same session-bound surface used for delivery or keep the prompt on the explicit-id form until that routing is safe.

## Reproduction Assessment

Yes for the review blocker: PR head sends session-bound MCP approvals, then filters idless `/approve` candidates to records with `turnSourceChannel`, so those delivered approvals are source-reproducibly excluded. I did not execute the tests because this is a read-only review.

## Solution Assessment

No, not yet. Reusing plugin approvals from the MCP materializer is the right shape, but the idless approval UX must honor session-bound approval routing before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match implicit approvals against session-bound requests:** `src/auto-reply/reply/commands-approve.ts:268-280`
  - body: The new bare `/approve <decision>` path only accepts pending records with `request.turnSourceChannel`, but MCP consent requests and existing plugin approvals are delivered through `agentId`/`sessionKey` and do not set that field. Since the prompt now tells users to reply `/approve allow-once`, those session-routed approvals will answer “No pending approval to act on” unless the user manually reconstructs the explicit id. Match candidates through the same session-binding/account helper used by approval delivery, or populate turn-source fields before advertising the idless form.
  - confidence: 0.91
