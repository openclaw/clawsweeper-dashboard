## Review Findings

Full review comments:

- **[P1] Block first-call confirmation tokens:** `src/agents/agent-bundle-mcp-materialize.ts:134-138`
  - body: `callMcpToolWithConsent` sends the original model input to the MCP server before it knows whether the server will return a consent envelope. That means an envelope-speaking server can receive a model-supplied `confirmation_token` before OpenClaw creates or verifies an approval, which undercuts the PR's stated trust boundary; reserve/scrub the redemption field for gated calls or add an explicit safe contract before the first call.
  - confidence: 0.86
- **[P1] Bind approvals to the exact MCP call origin:** `src/agents/agent-bundle-mcp-consent.ts:203-225`
  - body: The approval request records server/tool/session/channel fields, but not the full origin and call instance: account/thread and a canonical argument or action digest are missing. A user approval should only redeem the same server, tool, arguments, channel, account, thread, and expiry that produced the envelope; otherwise retries or shared sessions can turn one approval into permission for the wrong call.
  - confidence: 0.82
- **[P1] Make default-on gating upgrade-safe:** `src/agents/agent-bundle-mcp-materialize.ts:87-92`
  - body: `cfgFlag !== false` enables the new MCP approval gate by default for every upgraded deployment. For servers already returning this envelope, that changes behavior from model-visible results to prompts, unavailable results, or timeouts without a migration or maintainer-recorded default decision; make this opt-in or provide explicit upgrade proof before merging default-on behavior.
  - confidence: 0.84
