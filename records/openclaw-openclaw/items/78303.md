## What This Changes

The PR adds MCP consent-envelope detection to bundle-MCP tool materialization, routes approvals through plugin approvals, updates `/approve` handling plus MCP config/docs, and adds tests and changelog coverage.

## Best Possible Solution

Land the MCP consent gate only after maintainers accept the default-on security boundary and the approval infrastructure failure paths report unavailable/error instead of user denial.

## Reproduction Assessment

Yes. Current main is source-reproducible because bundle-MCP tool results return directly to the agent, and the linked issue/PR comments include a live vanilla baseline where the consent `action_id` reached chat.

## Solution Assessment

Mostly yes. Materializer-level consent interception and reuse of plugin approvals are a plausible minimal owner-boundary fix, but maintainer approval of the default-on/security policy and the failure-attribution cleanup should happen before merge.
