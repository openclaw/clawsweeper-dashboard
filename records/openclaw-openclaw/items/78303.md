## What This Changes

The PR adds a bundle-MCP consent envelope flow, `mcp.approvals` config/docs, agent/session approval routing, focused tests, and `/approve` parser adjustments for implicit approval handling.

## Best Possible Solution

Land a rebased version that keeps the consent envelope additive for MCP servers, fixes failure attribution, and has explicit maintainer acceptance of the default-on approval boundary and opt-out contract.

## Reproduction Assessment

Yes for the current-main baseline by source inspection: bundle-MCP tool results are returned directly to agent-visible output, and the PR discussion includes live baseline proof of an MCP `action_id` reaching chat. This is feature/security hardening rather than a documented behavior bug.

## Solution Assessment

Mostly yes: materializer-level interception is the right boundary for third-party `mcp.servers[]` tools, and reusing plugin approvals avoids duplicating gateway infrastructure. The remaining fix is to stop reporting infrastructure failures as user denial, plus maintainer sign-off on the default-on policy.
