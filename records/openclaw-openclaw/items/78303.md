## Risks / Open Questions

- Default-on MCP consent can change an existing envelope-returning MCP server from pass-through behavior to a denied or unavailable tool result when the deployment has no usable plugin approval route.
- The new trust boundary depends on token redaction, first-call `confirmation_token` scrubbing, sanitized approval-shaped text, `/approve` authorization, and session-bound delivery all staying aligned.
- Approval gateway request or wait failures currently return the same synthetic reason as an explicit user denial, which can misstate what happened in model-visible feedback and logs.
