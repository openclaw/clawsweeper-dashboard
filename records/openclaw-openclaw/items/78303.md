  - body: The first MCP call still receives model-supplied input unchanged, including `confirmation_token`.   - confidence: 0.88
- **[P1] Bind MCP approvals to account and thread:** `src/agents/embedded-agent-runner/run/attempt.ts:1457-1460`
  - body: The MCP approval request only passes channel and target, even though plugin approval records support account and thread fields and existing agent/exec paths carry them. Multi-account or threaded transports need the full origin binding so `/approve` can prove the decision belongs to the same user surface that triggered the MCP call.
  - confidence: 0.84
- **[P1] Make default-on gating upgrade-safe:** `src/agents/agent-bundle-mcp-materialize.ts:87-91`
  - body: `mcp.approvals.enabled` defaults to true, so existing envelope-speaking MCP deployments can start prompting, timing out, or returning unavailable results after upgrade. That compatibility break needs an explicit maintainer decision plus upgrade proof, or the gate should be opt-in until deployments can migrate safely.
  - confidence: 0.86
