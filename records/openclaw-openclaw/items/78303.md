Full review comments:

- **[P2] Add approvals to the strict config schema:** `src/config/types.mcp.ts:43-49`
  - body: The new `mcp.approvals.enabled` key is documented and read by the materializer, but the runtime config schema is still strict and only allows `mcp.servers` and `mcp.sessionIdleTtlMs`. Setting the documented opt-out will be rejected before the code can see it, so please add the nested `approvals.enabled` schema/help/generated-doc coverage with this type change.
  - confidence: 0.96
- **[P2] Clamp approval descriptions before gateway validation:** `src/agents/pi-bundle-mcp-consent.ts:131-140`
  - body: `envelope.summary` comes from the MCP server and is concatenated into the plugin approval `description` without a length bound, but the gateway rejects descriptions longer than its 256-character limit. A long but valid one-line summary will make `plugin.approval.request` fail and the tool will be denied without prompting the user; truncate or otherwise bound the text before sending it.
  - confidence: 0.9
