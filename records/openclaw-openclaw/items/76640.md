## Summary

Keep open. Current main still has an unguarded `FileHandle.sync()` in the shared media-store write path used by Telegram downloads, the discussion now includes a live Windows regression confirmation, and open PR #76752 references this issue with closing syntax but has not merged.

## What This Changes

Fix Telegram inbound media downloads on Windows when `fsync` fails with `EPERM` after a successful media write.

## Best Possible Solution

Land or replace #76752 with a narrow shared media-store fix that treats known Windows `fsync` `EPERM` after a successful temp-file write as best-effort while preserving write, size, rename, and cleanup failures.

## Reproduction Assessment

Yes. The issue has a high-confidence source-level reproduction path by making `FileHandle.sync()` throw `EPERM` during `saveMediaBuffer()`, and the discussion adds a live Windows 2026.5.2 confirmation.

## Solution Assessment

Yes. A narrow fix in `src/media/store.ts` is the maintainable direction because Telegram, and potentially other channels, share the same media-save path.
