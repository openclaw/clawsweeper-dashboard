## Summary

Keep open. Current main and v2026.5.2 still sync saved media by reopening the temp file read-only, and Telegram inbound media uses that shared path before Groq transcription; the reported EPERM fsync failure is consistent with a real Windows media-store bug.

## What This Changes

Fix Telegram voice/media downloads on Windows when the shared media store's fsync step fails with EPERM before audio transcription.

## Best Possible Solution

Update the shared media-store persistence helper to sync temp media files through a Windows-compatible, write-capable path while preserving atomic rename, cleanup, size limits, and Telegram/Groq behavior.

## Reproduction Assessment

No live high-confidence Windows reproduction was run in this read-only review. The source path is clear: Telegram voice media calls `saveMediaBuffer`, and current main fsyncs the temp file after reopening it read-only, matching the reporter's EPERM fsync error.

## Solution Assessment

Yes. The narrowest maintainable fix is in the shared media-store write helper rather than a Telegram- or Groq-specific workaround, with a regression test that exercises the temp-file sync path.
