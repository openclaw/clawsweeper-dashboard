## Summary

Keep open. Current main still allows a temp-file `FileHandle.sync()` failure in the shared media store to propagate through Telegram media resolution into the exact user-facing failure reply reported, so the Windows `fsync` regression remains valid.

## What This Changes

Make Telegram media and voice-note downloads on Windows continue to transcription when temp-file `fsync` fails with `EPERM` after the media buffer was written.

## Best Possible Solution

Fix the shared media-store write path to tolerate the narrow Windows temp-file sync failure after a successful write, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes for a source-level reproduction: mock `FileHandle.sync()` to throw a Windows `EPERM` during `saveMediaBuffer()` and the current code will reject before Telegram can transcribe the voice note. No live Windows host was exercised in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is in `src/media/store.ts`, because Telegram, WebChat, and other media callers share the same persistence path. The fix should not change Telegram network fallback or swallow unrelated filesystem failures.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76640. On Windows, Telegram media and voice-note downloads can fail after a successful temp write when `FileHandle.sync()` in `saveMediaBuffer()` throws `EPERM`, causing Telegram to reply `Failed to download media` instead of transcribing. Keep the fix in the shared media-store path (`src/media/store.ts`) so all media callers benefit. Tolerate only known post-write sync errors such as Windows `EPERM`/`EACCES` from `handle.sync()` after the temp file has been written and opened; still close the handle, continue to rename, and do not swallow `fs.writeFile`, `fs.open`, `fs.rename`, size, path, or cleanup failures. Add focused regression coverage in `src/media/store.test.ts` by mocking a `FileHandle.sync()` `EPERM` and asserting `saveMediaBuffer()` succeeds and leaves no temp artifact. Add a single-line user-facing `CHANGELOG.md` fix entry with reporter credit if appropriate. Do not change Telegram network fallback, provider routing, or transcription semantics.
