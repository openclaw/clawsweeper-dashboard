## Summary

Keep open: current main still unconditionally builds and displays the Feishu/Lark card note containing Agent, Model, and Provider metadata, and the strict Feishu config/manifest/docs expose render and streaming controls but no footer-specific hide or customization setting. The earlier ClawSweeper comment reached the same result, and current main has not added a closing implementation or linked fix PR.

## What This Changes

Add a Feishu/Lark configuration option to hide or customize the visible Agent/Model/Provider runtime footer appended to card replies.

## Best Possible Solution

Add a Feishu-owned footer visibility setting under the Feishu channel config, inherited by accounts and defaulting to the current visible behavior. Thread it through streaming start/close and static card delivery, document it, add schema/manifest coverage, and defer any shared channel presentation API until another channel needs the same control.

## Reproduction Assessment

Yes. The source path is high-confidence: card replies always compute a note, the renderers display that note, and current tests assert the note is passed through; live Feishu delivery would only confirm the already-visible platform rendering.

## Solution Assessment

Yes. A Feishu plugin setting is the narrowest maintainable fix for the reported Feishu behavior; using renderMode: "raw" is not equivalent because it disables card rendering instead of controlling only the metadata footer.
