## Summary

Keep open: current main still builds and renders the Feishu card runtime note unconditionally for card replies, while the Feishu config, manifest, docs, and tests expose other reply controls but no footer-specific hide or customization option.

## What This Changes

Add a Feishu configuration option to hide or customize the Agent/Model/Provider runtime footer shown in replies.

## Best Possible Solution

Add a Feishu-owned footer visibility setting, inherited by accounts and defaulting to the current visible behavior, then thread it through streaming and static card delivery with docs, manifest, schema, tests, and changelog coverage.

## Reproduction Assessment

Yes. The deterministic source path shows card replies always compute a note and the renderers display it; the existing tests assert note propagation into streaming card calls.

## Solution Assessment

Yes. A Feishu plugin setting is the narrowest maintainable fix for the reported Feishu behavior; using renderMode raw is not equivalent because it disables card rendering instead of only controlling the metadata footer.
