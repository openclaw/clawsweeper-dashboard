## Summary

Current main still lacks a Feishu/Lark setting to hide or customize the runtime card footer. The footer text is constructed and passed into streaming and static card replies unconditionally, while the Feishu config schema, manifest metadata, docs, and tests expose no footer-specific control. The open generic footer PRs I found do not close or implement this Feishu card-note request.

## What This Changes

The issue requests a Feishu/Lark configuration option to hide or customize the per-reply runtime footer showing agent, model, and provider metadata.

## Best Possible Solution

Add an owner-local Feishu plugin setting, likely top-level and per-account, that can disable the runtime card note while preserving current default behavior. If maintainers want customization too, use a small explicit mode/template rather than a broad core migration, then document it on the public Feishu page and cover streaming plus static card replies.
