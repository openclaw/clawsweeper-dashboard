## Summary

Keep open: current main still builds and displays the Feishu/Lark runtime metadata footer for structured and streaming card replies, while the Feishu schema, manifest, generated metadata, and public docs expose no footer visibility or customization setting. This remains a narrow, valid Feishu plugin feature request rather than an implemented or duplicate item.

## What This Changes

The request asks for configuration to hide or customize the Feishu/Lark reply-card runtime footer that shows Agent, Model, and Provider metadata.

## Best Possible Solution

Add a Feishu-owned, account-inheritable footer visibility setting that defaults to the current visible behavior, omit `note` when disabled for both streaming and static card replies, and keep any broader cross-channel presentation API out of scope until another channel needs the same control.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: a Feishu reply that uses card rendering calls `resolveCardNote`, passes the returned note into the card send path, and the renderer turns that note into a visible grey footer; current tests assert the note is present.

## Solution Assessment

Yes. A Feishu plugin config setting is the narrowest maintainable solution for the reported Feishu behavior; using `renderMode: "raw"` is not equivalent because it disables card rendering instead of only hiding the metadata footer.
