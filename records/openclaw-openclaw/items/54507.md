## Summary

Keep open: current main still builds and displays the Feishu/Lark runtime footer for card replies without a footer-specific configuration option. The existing `renderMode: "raw"` path avoids cards broadly, but it is not a hide/customize control for the Agent/Model/Provider footer.

## What This Changes

Request a Feishu/Lark configuration option to hide or customize the runtime Agent/Model/Provider footer appended to replies.

## Best Possible Solution

Add a Feishu-owned footer visibility setting, defaulting to the current visible behavior, with top-level and per-account support. Thread it through streaming start/close and static card delivery, update schema/manifest/docs/tests, and only introduce a shared channel presentation contract later if multiple channels need the same capability.

## Reproduction Assessment

Yes. The current-main source path reproduces the behavior: card replies always receive a `note`, the renderer displays that note, and the strict Feishu config schema has no footer-specific key to change it.

## Solution Assessment

Yes, a Feishu plugin setting is the narrowest maintainable solution for the reported Feishu behavior. `renderMode: "raw"` is not an equivalent fix because it disables card rendering rather than controlling only the footer.
