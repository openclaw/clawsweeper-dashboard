## Summary

Keep open: current main still unconditionally builds and renders the Feishu/Lark card note containing Agent, Model, and Provider metadata, while the Feishu plugin config, manifest, docs, and tests expose render/streaming controls but no footer-specific hide or customization option.

## What This Changes

The issue requests a configuration option to hide or customize the runtime metadata footer appended to Feishu replies.

## Best Possible Solution

Add a Feishu-owned footer visibility setting under the Feishu channel config, inherited by accounts and defaulting to the current visible behavior. Thread it through streaming start/close and static structured-card delivery, document it, cover schema/manifest/tests, and defer any shared channel presentation API until another channel needs the same control.

## Reproduction Assessment

Yes. The source path is high-confidence: card replies always compute a note, the renderers display that note, and current tests assert note propagation; live Feishu delivery would only confirm the already-deterministic renderer behavior.

## Solution Assessment

Yes. A Feishu plugin setting is the narrowest maintainable solution for the reported Feishu behavior; using renderMode: "raw" is not equivalent because it changes card rendering rather than controlling only the metadata footer.
