## Summary

Keep open: the PR is a plausible narrow fix path, but latest head still has a heartbeat-drain hole where internal cron-awareness events can be consumed before the next normal user turn, and the real-behavior proof does not prove the final head.

## What This Changes

Adds `SystemEvent.audience`, wraps internal queued events in runtime-context delimiters, migrates cron-awareness events to that lane, and updates heartbeat selectors, tests, and changelog text.

## Best Possible Solution

Keep the audience primitive only if internal events cannot be drained by heartbeat reply turns before the next normal user reply, then require refreshed final-head proof for the cron-awareness behavior.

## Reproduction Assessment

Yes, at source level: PR head calls the real reply pipeline from heartbeat with the main session key, and that pipeline drains internal events before the heartbeat consume selector runs. I did not run a live cron/channel reproduction in this read-only review.

## Solution Assessment

No, not yet: wrap-on-drain is a reasonable direction, but the current implementation still lets heartbeat reply turns drain the hidden lane. The safer fix is to make the drain distinguish heartbeat/non-user turns or otherwise leave internal events queued until a normal reply while preserving `trusted: false` downgrade behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep internal events out of heartbeat drains:** `src/auto-reply/reply/session-system-events.ts:130`
  - body: When a non-isolated heartbeat runs before the next user reply, `runHeartbeatOnce` calls the real `getReplyFromConfig` with the main `SessionKey`, and that path calls this generic drain before the heartbeat consume selector runs. Since internal events are selected here, the hidden cron-awareness event can be consumed into a heartbeat prompt and removed from the queue; `prompt-prelude.ts` does not include `systemEventBlocks` in `transcriptCommandBody`, so OK/silent/no-target heartbeats lose the awareness before the normal turn that was supposed to see it. Skip internal events for heartbeat drains or keep them queued until a normal reply consumes the wrapped lane.
  - confidence: 0.91
