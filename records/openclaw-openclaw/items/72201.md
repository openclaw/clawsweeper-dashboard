## Summary

Keep open: the audience primitive is useful, but latest head can still strand an internal cron-awareness event when its delivered text looks like an exec completion.

## What This Changes

This PR adds optional `SystemEvent.audience`, wraps internal-audience system events as runtime context during reply drains, marks isolated-cron awareness internal, updates heartbeat and owner-auth selectors, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the audience primitive only if internal events always reach the wrapped drain regardless of exec-like text shape, with regression coverage for exec-shaped internal cron awareness and refreshed exact-head runtime proof.

## Reproduction Assessment

Yes, at source level: enqueue an `audience: "internal"` system event whose text matches `isExecCompletionEvent`; the PR head's generic drain excludes it and heartbeat selectors also skip it. I did not run a live cron/Matrix setup in this read-only review.

## Solution Assessment

No, not as currently patched: the event-level audience direction is plausible, but internal exec-shaped awareness can be left in the queue indefinitely. The narrow fix is to let internal exec-shaped events reach the wrapped drain or prevent the migrated producer from queuing that shape as internal.
