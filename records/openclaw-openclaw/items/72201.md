## Summary

Keep this PR open: the code path looks coherent and I found no blocking correctness issue in the final head, but it adds a new core event-audience API and still needs maintainer acceptance plus refreshed current-head real behavior proof before merge.

## What This Changes

The branch adds `SystemEvent.audience`, wraps internal queued system events in runtime-context delimiters at drain time, makes heartbeat routing respect that internal lane, and migrates isolated cron awareness events to `audience: "internal"`.

## Best Possible Solution

Land the narrow audience primitive only after maintainers accept the core API and either receive current-head live cron/channel proof or explicitly waive that proof; keep additional producer migrations case-by-case under https://github.com/openclaw/openclaw/issues/69492.

## Solution Assessment
