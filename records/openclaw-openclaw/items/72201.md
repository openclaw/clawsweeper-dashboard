## Summary

Keep open: the PR addresses a real system-event leakage path and is not obsolete, but current head still has a P2 correctness issue where a non-isolated heartbeat can consume internal cron-awareness before the next normal reply, and the real behavior proof is not refreshed for the final head.

## What This Changes

The PR adds `SystemEvent.audience`, wraps internal system events in runtime-context delimiters during drain, migrates cron awareness events to the internal lane, and updates heartbeat routing, tests, and changelog text.

## Best Possible Solution

Fix the heartbeat drain path so internal-audience events remain queued across non-isolated heartbeat replies until the intended wrapped normal drain, add a regression that uses the real reply path, and refresh current-head live proof before merge.

## Reproduction Assessment

Yes, at source level: current main queues cron awareness as an ordinary trusted-false system event, and PR head shows the non-isolated heartbeat path can call the generic drain with the main session key before the later heartbeat consume selector. I did not run a live cron/channel reproduction in this read-only review.

## Solution Assessment

No, not yet: the audience primitive is a plausible narrow direction, but the current patch still lets heartbeat replies drain internal events too early. The safer fix is to make the drain heartbeat-aware or otherwise leave internal events queued during heartbeat turns, then prove it with the real reply path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep internal events out of heartbeat drains:** `src/auto-reply/reply/session-system-events.ts:130`
  - body: When a non-isolated heartbeat runs before the next user reply, `runHeartbeatOnce` calls `getReplyFromConfig` with the main session key, and `runPreparedReply` calls this drain before the heartbeat consume selector. Because internal events are selected here, an OK/silent/no-target heartbeat can consume cron-awareness into a model-only block that is not persisted in `transcriptCommandBody`, so the next normal turn loses the hidden awareness this lane is meant to preserve. Pass heartbeat context into the drain or leave internal events queued on heartbeat turns.
  - confidence: 0.88
