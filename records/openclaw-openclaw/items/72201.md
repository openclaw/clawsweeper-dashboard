## Summary

Keep open: this PR is not obsolete and has useful exact-head proof, but the current branch still has a source-level heartbeat owner-downgrade bug introduced by leaving internal events queued across heartbeat turns.

## What This Changes

The PR adds a `SystemEvent.audience` field, wraps internal events as hidden runtime context, adjusts heartbeat and collect-mode routing, and marks isolated cron awareness events internal.

## Best Possible Solution

Land the audience primitive only after the heartbeat owner-downgrade scope is fixed, then keep future producer migrations tied to concrete leakage reports under the canonical system-event issue.

## Reproduction Assessment

Yes at source level: current main queues cron awareness as a normal forced-downgrade system event and the generic drain formats selected events into visible `System:` text. I did not run a live cron/channel reproduction in this read-only review.

## Solution Assessment

No as-is: the producer-level audience contract and wrap-on-drain design are a maintainable direction, but heartbeat owner-downgrade state needs to match the content actually visible to that heartbeat prompt before merge.

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Scope heartbeat downgrades to prompt-visible events:** `src/auto-reply/reply/session-system-events.ts:50-52`
  - body: With `isHeartbeat=true`, internal events are deliberately left in the queue here, but `runHeartbeatOnce` still computes `hasActiveSessionOwnerDowngradeEvents` from all queued events and passes `ForceSenderIsOwnerFalse` even though the internal block was not drained into the heartbeat prompt. A single `audience: "internal", forceSenderIsOwnerFalse: true` cron-awareness item can therefore disable owner-only heartbeat tools on every heartbeat until a normal user turn drains it. Keep the event queued, but make the heartbeat downgrade predicate ignore internal events that are not included in that heartbeat prompt, while preserving the downgrade on the later normal drain that does include the wrapped block.
  - confidence: 0.82
