## Summary

Keep this PR open: the audience primitive is still useful and not implemented on current main, but the latest head has a concrete heartbeat-drain regression plus insufficient current-head real behavior proof for an external contributor PR.

## What This Changes

The PR adds a `SystemEvent.audience` field, wraps internal-audience queued system events in runtime-context delimiters at drain time, and migrates isolated cron awareness events to the internal lane.

## Best Possible Solution

Keep the audience primitive only if heartbeat replies cannot consume internal awareness before the next normal turn; add a regression through the real `runHeartbeatOnce -> getReplyFromConfig` path and refresh current-head live proof.

## Reproduction Assessment

Yes, at source level: current main queues cron awareness as an ordinary `trusted: false` system event and the generic drain formats queued events as visible `System:` lines. I did not run a live cron/channel reproduction in this read-only review.

## Solution Assessment

No, not as currently implemented: the wrap-on-drain direction is plausible, but the generic drain also runs during heartbeat replies and can consume internal events before a normal user turn. The safer fix is to leave internal events queued on heartbeat turns or make the drain context-aware and cover that path with a regression test.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep internal events out of heartbeat drains:** `src/auto-reply/reply/session-system-events.ts:128-130`
  - body: This generic drain also runs for heartbeat replies: `runHeartbeatOnce` calls `getReplyFromConfig` with the main session key before the heartbeat consume selector, so selecting `audience === "internal"` here can consume a hidden cron-awareness event into a heartbeat prompt. Because `systemEventBlocks` are not included in `transcriptCommandBody`, a silent, OK-only, or no-target heartbeat can drop that awareness before the next normal user turn. Leave internal events queued for heartbeat turns or otherwise make this drain context-aware.
  - confidence: 0.87
