The PR adds an optional `SystemEvent.audience` field, wraps internal-audience system events as hidden runtime context at drain time, marks isolated cron awareness events internal, and adds focused tests.

## Best Possible Solution

Rebase or replace the PR on current main, keep the hidden runtime-context lane narrow, preserve exec-completion heartbeat queueing, add changelog/API validation, and leave broader delivery/trust policy to the linked follow-up issues.

## Reproduction Assessment

## Solution Assessment
