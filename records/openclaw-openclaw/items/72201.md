## What This Changes

The PR adds a `SystemEvent.audience` field, wraps internal events as runtime context, routes internal events around heartbeat relay/consume paths, keeps collect-mode transcripts stripped, and marks isolated cron awareness events internal.

## Best Possible Solution

Land the event-level audience primitive only if maintainers accept the hidden-runtime-context boundary, then keep future producer migrations case-by-case under the canonical system-event leakage issue.
