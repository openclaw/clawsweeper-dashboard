- **[P2] Gate allowFrom coverage by channel fallback support:** `src/commands/doctor/shared/empty-allowlist-scan.ts:139`
  - body: This marks an account as covered whenever `allowFrom` has entries, but the policy/runtime only use `allowFrom` for group sender access when the channel supports `groupAllowFrom -> allowFrom` fallback. Gate this branch on `getDoctorChannelCapabilities(channelName).groupAllowFromFallbackToAllowFrom` or reuse the policy helper, and add a non-fallback regression.
