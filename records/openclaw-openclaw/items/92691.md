## What This Changes

The branch changes the shared doctor empty-allowlist scanner/policy to suppress a parent-level group allowlist warning when enabled child accounts have populated group allowlists, and adds scanner regression tests.

## Best Possible Solution

Land one canonical fix that suppresses parent-scope group warnings using effective account allowlist semantics across both shared and channel-specific doctor hooks, then prove it with redacted after-fix openclaw doctor output.

## Reproduction Assessment

Yes. Current main calls checkAccount on the parent channel record before account records, and the policy/hook path can emit a parent-level empty group allowlist warning even when account scopes carry effective allowlists.

## Solution Assessment

No. The scanner boundary is plausible, but this implementation only checks raw string groupAllowFrom entries and does not cover the channel-specific extra-warning path used by Telegram.
