## Summary

Keep open. Current main partially addresses the original report by carrying `stopReason` when terminal metadata is set, but the primary pi-embedded lifecycle path still has no explicit `aborted` state, setter parameter, or emitted payload field; the related fix attempt at https://github.com/openclaw/openclaw/pull/66574 is closed unmerged.

## What This Changes

Ensure pi-embedded `lifecycle:end` events include abort metadata so event subscribers can distinguish canceled runs from natural completion.

## Best Possible Solution

Propagate the runner’s explicit `aborted` boolean through the typed terminal lifecycle metadata path and emit it alongside the existing `stopReason`, preserving current liveness, replay, yielded, and fallback behavior.

## Reproduction Assessment

Yes, by source inspection on current main: the primary terminal lifecycle event can carry `stopReason`, but the state, setter contract, and emitted payload still cannot carry an explicit `aborted` boolean. I did not run a live abort in this read-only sweep.

## Solution Assessment

Yes. The narrow maintainable fix is to forward the runner’s explicit abort metadata through the existing terminal lifecycle metadata path rather than deriving cancellation heuristically from `stopReason`.
