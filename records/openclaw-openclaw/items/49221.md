  - body: Keying the limiter by `${providerKey}:${intervalMs}` resets last-start history whenever the configured delay changes, so the first request after a config reload can bypass the intended minimum interval.   - confidence: 0.86

  - body: Adding public model config keys changes the generated config documentation surface, but this PR does not update `docs/.generated/config-baseline.sha256`. Regenerate the config baseline so changed-gate checks validate the new schema surface.
