- **[P2] Use the current pi-agent-core package:** `src/agents/api-request-delay.ts:1`
  - body: This new file imports `StreamFn` from `@mariozechner/pi-agent-core`, but current main depends on `@earendil-works/pi-agent-core` and no longer has the old package. Rebasing as-is would leave this file with an unresolved import and fail typechecking.
  - confidence: 0.93
- **[P2] Keep one limiter chain per provider:** `src/agents/api-request-delay.ts:38-41`
  - body: Keying the limiter by `${providerKey}:${intervalMs}` resets the provider's last-start history whenever the configured delay changes, so the first request after a hot reload can bypass the intended minimum interval. Keep one limiter per normalized provider and update the active interval without resetting the chain.
  - confidence: 0.88
- **[P2] Normalize provider IDs for delay overrides:** `src/agents/api-request-delay.ts:56`
  - body: The per-provider override lookup uses the raw trimmed provider string, while current provider config lookup normalizes aliases such as `z.ai`, `kimi-coding`, and `aws-bedrock`. Alias-keyed configs can skip the stricter provider delay and fall back to the global value.
  - confidence: 0.86
- **[P2] Refresh the config docs baseline:** `src/config/zod-schema.core.ts:255`
  - body: Adding public model config keys changes the generated config documentation surface, but this PR does not update `docs/.generated/config-baseline.sha256`. Regenerate the config docs baseline so changed-gate checks can validate the new schema surface.
  - confidence: 0.8
