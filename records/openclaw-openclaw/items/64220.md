## Summary

Keep open. Current main still has the stale-state gap: active/retrying compaction statuses have no stale timeout, final chat and reconnect paths clear run/stream state without clearing compaction/fallback indicators, and the existing retrying-compaction test still expects no clear timer; only session-switch cleanup is already covered.

## What This Changes

The issue asks for the Control UI compaction and fallback indicators to clear when the active session/run is no longer compacting, including missed end events and reset/reconnect/send-failure paths.

## Best Possible Solution

Control UI should use one shared cleanup path for compaction/fallback indicators, age-bound active and retrying compaction states, and call that cleanup from terminal chat, reconnect, clear/reset, session/agent switch, and send-failure paths.

## Reproduction Assessment

Yes. A source-level reproduction is to process a compaction `start`, miss the matching `end`, then process an owned final chat event; current main clears the run/stream fields but leaves `compactionStatus` active, so the renderer continues showing `Compacting context...`.

## Solution Assessment

Yes. The narrow maintainable fix is in the Control UI state machine, not backend compaction: add shared timer-aware indicator cleanup and targeted tests for missed end, retry timeout, reconnect, clear/reset, and send-failure recovery.
