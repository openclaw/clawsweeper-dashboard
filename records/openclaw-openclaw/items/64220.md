## Summary

Keep open. Current main still has a source-reproducible path where an active compaction status can remain visible after the chat run reaches a terminal state, and no merged fix or canonical replacement item owns this exact Control UI stale-state cleanup.

## What This Changes

The issue asks for the Control UI to stop showing `Compacting context...` after the active run or session is no longer actually compacting.

## Best Possible Solution

Add a centralized, timer-aware Control UI cleanup helper for compaction and fallback status, call it from terminal chat, reconnect/reset, session or agent switch, and send-failure paths, and preserve the existing retry-pending compaction behavior.

## Reproduction Assessment

Yes, at source level: process a compaction `start`, miss the matching `end`, then process the active run's final chat event. Current main clears run/stream state while leaving `compactionStatus.phase === "active"` renderable.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI state machine and focused UI tests, without changing backend compaction semantics or adding a product/config option.
