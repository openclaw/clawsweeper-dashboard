## Summary

Keep open. Current main only partially addresses session switches; active/retrying compaction can still survive missed terminal events, and several terminal/reset/reconnect paths clear chat state without clearing the compaction/fallback indicators. The related #39908 report is a broader closed symptom report, not a canonical fix for this narrower UI state-machine gap.

## What This Changes

The issue requests Control UI cleanup so the `Compacting context...` indicator clears when no current session/run is compacting, including missed end events, session switches, reconnects, resets, and send-failure cleanup.

## Best Possible Solution

Control UI should have one shared cleanup helper for compaction and fallback indicators that clears status plus timers. Active/retrying compaction should be age-bounded when terminal events are missed, and terminal chat, reconnect, clear-history, send-failure, and session/agent switch paths should call the same helper with tests for missed compaction `end` recovery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to process a compaction `start`, miss the matching `end`, then process an own-run final chat event; current main clears stream/run state but leaves active `compactionStatus`, so the renderer keeps showing `Compacting context...`.

## Solution Assessment

Yes. The best fix is a narrow UI state-machine repair, not a backend compaction change: add timer-aware indicator cleanup and reuse it from all terminal/reset/reconnect paths while preserving existing session/run filtering.
