## Summary

Keep open. Current main still has the Control UI stale-state gap: active and retrying compaction indicators can remain unbounded, and several terminal cleanup paths clear chat/run state without clearing compaction or fallback status.

## What This Changes

The issue asks for Control UI compaction and fallback indicators to be cleared or age-bounded so `Compacting context...` cannot remain visible after compaction has ended or been abandoned.

## Best Possible Solution

Control UI should centralize timer-aware compaction/fallback cleanup, age-bound active and retrying compaction states, and call that cleanup from terminal chat, reconnect, clear/reset, session or agent switch, and send-failure paths.

## Reproduction Assessment

Yes, at source level. Process a compaction `start`, miss the matching `end`, then process an owned final chat event; current main clears chat run/stream fields while leaving active `compactionStatus` renderable.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI state machine and tests, without changing backend compaction semantics or adding a product/config choice.
