## Summary

Keep open. Current main still has a source-reproducible stale UI path: a compaction `start` can set the active banner, and if the matching `end` is missed, terminal chat/reconnect cleanup clears run/tool state without clearing the compaction/fallback indicator state.

## What This Changes

The issue requests that Control UI stop showing `Compacting context...` after the active session/run is no longer compacting.

## Best Possible Solution

Control UI should use a shared timer-aware cleanup helper for compaction/fallback indicators and invoke it from terminal chat, reconnect/reset, session or agent switch, and send-failure paths.

## Reproduction Assessment

Yes, at source level. Process a compaction `start`, miss the matching `end`, then process the active run's terminal chat event or reconnect path; current main clears run/tool state while leaving active compaction state renderable.

## Solution Assessment

Yes. The narrow maintainable fix is in the Control UI state machine with focused regression tests, without changing backend compaction semantics or adding a config/product choice.
