## Summary

Keep open. Current main still has the Control UI stale-state gap: compaction `active` and retry-pending states are not age-bounded, and several terminal/reconnect/send-failure cleanup paths clear chat run state without clearing the indicator state.

## What This Changes

The issue requests preventing Control UI compaction/fallback indicators from staying visible after missed compaction end events or terminal UI state transitions.

## Best Possible Solution

Control UI should use one timer-aware indicator cleanup helper and call it from compaction timeout, retry terminal handling, final/error/abort chat events, reconnect, reset, session or agent switch, and send-failure paths.

## Reproduction Assessment

Yes, at source level. Dispatch a compaction `start`, miss the matching `end` or retry lifecycle terminal event, then process an owned final chat event; current main clears chat run/stream fields while leaving `compactionStatus` renderable.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI state machine and tests, without changing backend compaction semantics or adding a config/product choice.
