## Summary

Keep open. The prior ClawSweeper review still matches current main: active and retrying compaction indicators can remain renderable after the active chat run has already reached a terminal state, so the Control UI stale-state gap is still present.

## What This Changes

The issue asks for the Control UI to clear stale `Compacting context...` and fallback indicators when compaction is no longer active.

## Best Possible Solution

Control UI should centralize timer-aware compaction/fallback cleanup, age-bound active and retrying compaction states, and call that cleanup from terminal chat, reconnect, clear/reset, session or agent switch, and send-failure paths.

## Reproduction Assessment

Yes, at source level. Process a compaction `start`, miss the matching `end`, then process the active run's final chat event; current main clears run/stream fields while leaving `compactionStatus` renderable.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI state machine and tests, without changing backend compaction semantics or adding a product/config choice.
