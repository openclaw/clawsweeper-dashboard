## Summary

Keep open. Current main has partial cleanup, but an active compaction `start` can still remain renderable if the matching `end` event is missed and the active chat run later reaches a terminal state.

## What This Changes

The issue requests that the Control UI stop leaving `Compacting context...` visible after the corresponding compaction/run is no longer active.

## Best Possible Solution

Control UI should centralize timer-aware compaction/fallback cleanup and call it from terminal chat, reconnect/reset, session or agent switch, and send-failure paths.

## Reproduction Assessment

Yes, at source level. Process a compaction `start`, miss the matching `end`, then process the active run's final chat event; current main clears chat run state while leaving `compactionStatus` active and renderable.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI state machine and focused UI tests, without changing backend compaction semantics or adding a product/config choice.
