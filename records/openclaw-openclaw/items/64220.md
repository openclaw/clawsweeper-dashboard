## Summary

Keep open. Current main still has a source-level path where a missed compaction end event leaves an active Control UI compaction indicator renderable after the chat run has reached a terminal state.

## What This Changes

The issue reports that Control UI can keep rendering `Compacting context...` after backend/session state has no active compaction.

## Best Possible Solution

Centralize timer-aware compaction/fallback cleanup in Control UI and invoke it from terminal chat, reconnect/reset, session or agent switching, and send-failure paths with focused regression tests.

## Reproduction Assessment

Yes, at source level. Process a compaction `start`, miss the matching `end`, then process the active run's terminal chat event; current main clears chat run state while leaving `compactionStatus` active and renderable.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI state machine and tests, not in backend compaction semantics or a new config/product option.
