## Summary

## What This Changes

The PR injects `ensureContextEnginesInitialized()` into the CLI compaction dependency seam, calls it before `resolveContextEngine()`, and adds a Vitest regression test for cold CLI compaction registry initialization.

## Best Possible Solution

Keep the initializer-before-resolver runtime change for CLI compaction, repair the regression fixture against the current session-entry contract, add the required active-version changelog fix entry, then land after targeted and changed-lane validation pass.

## Reproduction Assessment

Yes. A cold CLI-backed run that reaches `runCliTurnCompactionLifecycle()` with a persisted `sessionFile` and positive `contextTokens` currently resolves the default `legacy` engine before any local initialization, matching the empty-registry error path.

## Solution Assessment

No, not as currently patched. The runtime approach is the narrow maintainable fix, but the PR still needs a valid typed regression fixture and a changelog entry before it is the best mergeable solution.
