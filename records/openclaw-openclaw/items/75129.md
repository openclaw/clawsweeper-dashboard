## Summary

Keep this PR open. Current main still has the CLI compaction init-order bug, and the runtime part of the PR is the right narrow shape, but the proposed test fixture still fails the repo's test typecheck and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR injects `ensureContextEnginesInitialized()` into the CLI compaction dependency seam, calls it before resolving the context engine, and adds a Vitest regression test for the cold CLI compaction path.

## Best Possible Solution

Keep the narrow initializer-before-resolver runtime change for CLI compaction, correct the regression fixture to the current `SessionEntry` contract, add the required single-line changelog fix entry, then land after the targeted test and changed gate pass.

## Reproduction Assessment

Yes. A cold CLI-backed run that reaches `runCliTurnCompactionLifecycle()` with a persisted `sessionFile` and positive `contextTokens` resolves the default `legacy` engine before local initialization, which matches the reported empty-registry error path.

## Solution Assessment

No, not as currently patched. The runtime change is the narrow maintainable fix, but the PR is not mergeable while the added test fixture violates `SessionEntry` and the required changelog entry is absent.
