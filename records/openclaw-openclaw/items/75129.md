## Summary

## What This Changes

The PR initializes context engines before CLI compaction resolves the configured engine and adds a cold-registry regression test.

## Best Possible Solution

Land the initializer-before-resolver runtime change with a valid SessionEntry regression fixture and a single-line user-facing changelog fix entry.

## Reproduction Assessment

Yes. A cold CLI-backed turn with a persisted sessionFile and positive contextTokens reaches runCliTurnCompactionLifecycle, where current main resolves the default legacy engine before built-in registration and can hit the empty-registry error path.

## Solution Assessment

No, not as currently patched. The runtime approach is the narrow maintainable fix, but the PR should first repair the invalid test fixture and add the required changelog entry.
