## Summary

Keep open. Current `main` still matches the reported bundled Twitch lifecycle bug: Twitch `startAccount` awaits a monitor that returns a stop handle immediately, and the gateway supervisor treats that settled task as a clean channel exit that schedules auto-restart.

## What This Changes

Fix the bundled Twitch plugin so a successful provider connection keeps the account task alive instead of entering the gateway auto-restart loop.

## Best Possible Solution

Use the existing passive monitor lifecycle helper in the Twitch gateway adapter, stop the monitor during abort cleanup, and add a Twitch lifecycle regression test mirroring the fixed passive-channel patterns.

## Reproduction Assessment

Yes, at source level. Follow `twitchPlugin.gateway.startAccount`: after `monitorTwitchProvider` returns `{ stop }`, the task settles before abort and the gateway supervisor enters its normal clean-exit restart path.

## Solution Assessment

Yes. A Twitch-local wrapper around `runStoppablePassiveMonitor` is the narrowest maintainable fix; changing shared gateway clean-exit semantics would be broader and risk other channel contracts.
