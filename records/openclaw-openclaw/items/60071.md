## Summary

Keep open. Current main still matches the reported Twitch lifecycle bug: the Twitch account startup task settles immediately after monitor setup, and the gateway supervisor still treats a non-aborted settled account task as an exit that schedules auto-restart.

## What This Changes

The issue requests a fix for the bundled Twitch plugin so a successful IRC connection remains running instead of triggering the gateway auto-restart loop immediately after startup.

## Best Possible Solution

Use the existing passive monitor lifecycle helper in the Twitch gateway adapter so `startAccount` remains pending until `ctx.abortSignal` fires, stop the monitor during abort cleanup, and add a Twitch lifecycle regression test mirroring the fixed Nextcloud Talk pattern.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `twitchPlugin.gateway.startAccount` with a mocked `monitorTwitchProvider` returning `{ stop }`; current main will let the task settle before abort, which the gateway supervisor treats as an account exit.

## Solution Assessment

Yes. A Twitch-local lifecycle wrapper is the narrowest maintainable fix because other passive/webhook plugins already rely on the generic gateway contract and shared lifecycle helper rather than changing supervisor semantics.

## Repair Work Prompt

Fix the Twitch bundled plugin lifecycle bug reported in https://github.com/openclaw/openclaw/issues/60071. Current `extensions/twitch/src/plugin.ts` awaits `monitorTwitchProvider`, whose `{ stop }` handle returns immediately, so the gateway supervisor sees `startAccount` settle and schedules auto-restart. Keep the fix Twitch-local: use the existing passive lifecycle helper (`runStoppablePassiveMonitor` from `openclaw/plugin-sdk/extension-shared`, or the equivalent established helper) so `startAccount` remains pending until `ctx.abortSignal` fires and then stops the monitor. Add a focused `extensions/twitch/src/plugin.lifecycle.test.ts` using the SDK channel test helpers, matching the Nextcloud Talk test pattern: monitor starts, `startAccount` remains pending before abort, abort calls `stop()`, and an already-aborted signal stops immediately. Do not change generic gateway supervisor semantics, Twitch auth/token resolution, Twurple dependency versions, or unrelated connection behavior. Add a single-line user-facing `CHANGELOG.md` fix entry with `Thanks @nhaener`.
