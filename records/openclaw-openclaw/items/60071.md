## What This Changes

The issue asks OpenClaw to stop the bundled Twitch plugin from resolving `startAccount` immediately after connecting, which causes gateway auto-restarts.

## Best Possible Solution

Wrap Twitch monitor startup with the existing passive lifecycle helper and add regression coverage that `startAccount` stays pending until abort.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction follows `twitchPlugin.gateway.startAccount` through `monitorTwitchProvider` returning `{ stop }`, after which `server-channels` treats the settled task as a clean channel exit and schedules restart.

## Solution Assessment

Yes. A Twitch-local use of `runStoppablePassiveMonitor` is the narrowest maintainable fix; changing generic gateway supervisor semantics would be broader and could affect third-party channel contracts.
