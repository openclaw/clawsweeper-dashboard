## Summary

Keep open. Current main still matches the reported bundled Twitch lifecycle bug: the monitor returns a stop handle immediately, `startAccount` resolves, and the gateway supervisor treats that settled task as a channel exit that schedules auto-restart.

## What This Changes

The issue asks OpenClaw to keep the Twitch plugin account task alive after startup instead of entering an immediate gateway auto-restart loop.

## Best Possible Solution

Use the existing passive monitor lifecycle helper in the Twitch gateway adapter, stop the monitor during abort cleanup, and add a Twitch lifecycle regression test mirroring the fixed passive-channel patterns.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to follow `twitchPlugin.gateway.startAccount`: after `monitorTwitchProvider` returns `{ stop }`, the promise settles before abort and the gateway supervisor enters its normal restart path.

## Solution Assessment

Yes. A Twitch-local wrapper around the existing passive lifecycle helper is the narrowest maintainable fix; changing gateway supervisor semantics would be broader and risk third-party channel contracts.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/60071. The observable bug is that the bundled Twitch plugin connects/sets up its monitor, `monitorTwitchProvider` returns `{ stop }`, `startAccount` resolves, and the gateway supervisor treats that as an account exit and starts an auto-restart loop. Keep the fix Twitch-local unless a truly generic seam is already present: wrap the Twitch monitor startup with the existing passive account lifecycle helper (`runStoppablePassiveMonitor` or `runPassiveAccountLifecycle`) so `startAccount` remains pending until `ctx.abortSignal` fires, and ensure the monitor stop handle is called exactly once during cleanup. Add a focused Twitch lifecycle regression test similar to `extensions/nextcloud-talk/src/channel.lifecycle.test.ts` using the SDK `startAccountAndTrackLifecycle` helpers. Do not change Twurple connection semantics or gateway restart policy unless a test demonstrates that is necessary. Add a user-facing `CHANGELOG.md` fix entry for Twitch.
