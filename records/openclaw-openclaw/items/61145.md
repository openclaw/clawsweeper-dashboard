## Summary

Keep open. The branch is a plausible implementation candidate, but it is not mergeable yet because the documented WebChat behavior is not wired through the browser Control UI, generated config metadata is incomplete, and the external PR lacks after-fix real behavior proof.

## What This Changes

The branch adds optional `gateway.webchat.defaultPeerId` config/docs/schema/snapshot plumbing plus TUI session-key logic and tests for default peer-based routing.

## Best Possible Solution

Align the contract before merge: either fully wire the opt-in peer default through browser Control UI, gateway snapshot, TUI, generated config metadata, docs, and tests, or narrow this branch to TUI-only and leave browser continuity to https://github.com/openclaw/openclaw/issues/67735.

## Reproduction Assessment

Yes for the PR defects. Source inspection shows the branch only changes TUI default resolution while browser Control UI still resolves empty/default sessions from `sessionDefaults.mainSessionKey`, and current config metadata has no `gateway.webchat.defaultPeerId` entry.

## Solution Assessment

No, not as-is. Reusing `buildAgentPeerSessionKey` is the right mechanism, but the proposed public config is incomplete until the browser default path and generated metadata are aligned or the scope is reduced.
