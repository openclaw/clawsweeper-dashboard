## Summary

Keep open but not merge-ready: the CSS-only patch changes the parent `.config-main`, while current source already makes `.config-content` the active scroll container and the Form/Raw buttons do not reset that child scroll state. The related PR at https://github.com/openclaw/openclaw/pull/94325 is the better-supported candidate because it targets the retained `.config-content` scroll state and includes browser screenshot proof.

## What This Changes

The PR replaces `.config-main` hidden/clip overflow in the Settings config stylesheet with `overflow-y: auto` to make Raw config content scrollable.

## Best Possible Solution

Land or adapt one verified Control UI fix that resets `.config-content` on Form/Raw mode changes while preserving Raw config secret redaction; https://github.com/openclaw/openclaw/pull/94325 is currently the strongest candidate subject to maintainer and CI review.

## Reproduction Assessment

Unclear as a high-confidence live repro because this read-only review did not run a browser. Source inspection gives a concrete path: `.config-content` can retain scroll after Form mode, the Raw toggle does not reset it, and the sibling browser proof demonstrates the same user-visible flow after a content-scroll reset.

## Solution Assessment

No. Changing parent overflow is a plausible but likely wrong-layer mitigation; the narrower maintainable fix is to reset the existing `.config-content` scroll container on Form/Raw mode changes while preserving Raw secret redaction.
