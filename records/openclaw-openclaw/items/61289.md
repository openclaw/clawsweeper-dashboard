## Summary

Keep this PR open. Current main still has the direct Telegram action gap, and the PR’s final code/test diff is a narrow alignment with the existing media pipeline, but it still needs a small public docs update and a changelog entry before merge.

## What This Changes

The PR adds Telegram `sendMessage` `filePath` and `path` media fallbacks in the action runtime and tests that those aliases reach the existing `mediaUrl`/`asDocument` send options.

## Best Possible Solution

Land the focused alias mapping with existing `mediaUrl` precedence preserved, focused tests, a small Telegram docs update, and a single changelog entry while leaving broader attachment abstraction work to follow-up.

## Reproduction Assessment

Yes. Source inspection on current main shows direct Telegram `sendMessage` reads only `mediaUrl` and `media`, so a call with only `filePath` or only `path` would not reach the existing media send pipeline.

## Solution Assessment

Yes, with the docs and changelog follow-up. Mapping the aliases into the existing `mediaUrl` pipeline is the narrow maintainable fix because Telegram already has media loading, document forcing, and local-root guard handling.
