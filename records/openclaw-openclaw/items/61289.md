## What This Changes

The PR branch adds `filePath` and `path` fallbacks to Telegram `sendMessage` media resolution and adds action-runtime tests for those aliases, including `asDocument` behavior.

## Best Possible Solution

Land the focused alias mapping, preserve existing `mediaUrl` then `media` precedence, keep the alias and `asDocument` tests, and add a small `docs/channels/telegram.md` update for the supported aliases and existing local media-root boundary.

## Reproduction Assessment

Yes. Against current main, a focused unit check can call `handleTelegramAction` with `action: "sendMessage"`, `to`, and only `filePath` or `path`; current main ignores those keys and still treats the send as missing media/content, while the PR routes them into `sendMessageTelegram` as `mediaUrl`.

## Solution Assessment

Yes, with the docs follow-up. Mapping the aliases into the existing Telegram media send path is the narrowest maintainable fix because the shared message contract already exposes those names and Telegram already has guarded media loading.
