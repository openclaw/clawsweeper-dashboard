## Best Possible Solution

Land the focused alias mapping with concise Telegram docs and redacted real Telegram proof while preserving `mediaUrl` precedence and the existing media-loader root guards.

## Solution Assessment

Yes, for the direct Telegram action surface. Reusing the existing `mediaUrl` pipeline is the narrowest maintainable implementation because Telegram already owns media loading, document forcing, size limits, and local-root checks there.
