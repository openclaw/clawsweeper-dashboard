## Best Possible Solution

Land the narrow alias mapping with a concise docs update and redacted live Telegram proof, preserving `mediaUrl` precedence and the existing media-root guards while leaving the separate `buffer`/`filename` mismatch to its canonical issue.

## Solution Assessment

Mostly yes: mapping the aliases into the existing `mediaUrl` pipeline is the narrow maintainable fix for this direct-action ergonomic gap. It should be paired with docs and live proof rather than expanded into a broader attachment-contract redesign.
