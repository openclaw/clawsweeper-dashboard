## Summary

Keep open. Current main still has the Telegram action-surface gap, and the final PR diff is focused, but it needs after-fix real behavior proof before merge and one public docs update for the new aliases.

## What This Changes

The PR adds `filePath` and `path` aliases to Telegram `sendMessage` media handling and covers them with focused action-runtime tests.

## Best Possible Solution

Land the focused aliases with `mediaUrl` precedence preserved, the Telegram action docs updated, and real Telegram or CLI after-fix proof attached while keeping broader attachment abstractions as follow-up work.

## Reproduction Assessment

Yes. Source inspection on current main shows `sendMessage` reads `mediaUrl` and `media` only, so `filePath` or `path` alone cannot reach the existing media upload path.

## Solution Assessment

Yes, with docs and proof added. Mapping the aliases into the existing `mediaUrl` pipeline is the narrowest maintainable fix because Telegram already owns media loading, local-root guards, and document forcing.
