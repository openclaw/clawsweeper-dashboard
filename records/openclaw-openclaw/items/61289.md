## What This Changes

Adds `filePath` and `path` aliases to Telegram `sendMessage` media resolution and tests that they map into the existing `mediaUrl` send path, including `asDocument` forwarding.

## Best Possible Solution

Land the focused alias mapping with `mediaUrl` precedence preserved, focused tests, a concise Telegram docs update, and a single changelog entry while leaving broader attachment abstractions to follow-up work.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram `sendMessage` reads `mediaUrl` and `media` only, so `filePath` or `path` alone cannot reach the existing media upload pipeline.

## Solution Assessment

Yes, with the docs and changelog repair. Mapping the aliases into the existing `mediaUrl` pipeline is the narrowest maintainable solution because Telegram already owns media loading, document forcing, and local-root guard handling.
