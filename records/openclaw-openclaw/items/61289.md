The PR adds `filePath` and `path` aliases to Telegram `sendMessage` media parsing and tests those aliases while preserving existing `mediaUrl` precedence.

## Best Possible Solution

## Solution Assessment

Yes, pending docs and proof. Reusing the existing `mediaUrl` pipeline is the narrowest maintainable approach because Telegram already owns media loading, document forcing, size limits, and local-root checks there.
