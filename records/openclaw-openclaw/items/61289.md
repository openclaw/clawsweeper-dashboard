## Summary

Keep open: current main still lacks the direct Telegram `sendMessage` `filePath`/`path` aliases, and the final PR is focused, but it still needs contributor-provided real Telegram proof before merge and a small docs update for the new action contract.

## What This Changes

The PR adds `filePath` and `path` fallbacks to Telegram `sendMessage` media resolution and adds action-runtime tests for those aliases.

## Best Possible Solution

Land the focused alias mapping with `mediaUrl` precedence preserved, document the aliases and local media-root boundary, and require redacted live Telegram proof before merge.

## Reproduction Assessment

Yes. Source inspection of current `origin/main` shows `sendMessage` reads `mediaUrl`/`media` only, then throws `content required.` when `filePath` or `path` is the only media input.

## Solution Assessment

Yes, with proof and docs added. Reusing the existing `mediaUrl` pipeline is the narrowest maintainable solution because Telegram already owns media loading, local-root checks, size handling, and document forcing there.
