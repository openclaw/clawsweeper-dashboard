## Summary

## What This Changes

The PR adds `filePath` and `path` aliases to Telegram `sendMessage` media resolution and tests that they forward through the existing media path.

## Best Possible Solution

Land the focused alias mapping with `mediaUrl` precedence preserved, document the aliases and local media-root boundary, then require real Telegram proof and fresh gates before merge.

## Reproduction Assessment

Yes. Source inspection gives a focused path: call Telegram `sendMessage` with `to` and only `filePath` or `path` on current main, and the media value stays undefined so the action throws before upload.

## Solution Assessment

Yes, with docs and real behavior proof added. Reusing the existing `mediaUrl` pipeline is the narrowest maintainable fix because Telegram already owns media loading, document forcing, size limits, and local-root checks there.
