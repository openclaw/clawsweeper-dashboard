## Summary

Keep open. Current main still only accepts `mediaUrl`/`media` on the Telegram action path, while this PR narrowly maps `filePath`/`path` into the existing guarded media-send pipeline; merge should wait for contributor real Telegram proof and the docs repair.

## What This Changes

Adds `filePath` and `path` aliases to Telegram `sendMessage` media resolution and tests that they forward through the existing `mediaUrl`/`forceDocument` path.

## Best Possible Solution

Land the focused alias mapping with `mediaUrl` precedence preserved, add a concise Telegram docs update for the aliases and local media-root boundary, and handle the changelog during maintainer landing.

## Reproduction Assessment

Yes. Source inspection shows a focused reproduction path: call Telegram `sendMessage` with `to` and only `filePath` or `path` on current main, and the media value remains undefined so the action throws before upload.

## Solution Assessment

Yes, with docs and real behavior proof added. Reusing the existing `mediaUrl` pipeline is the narrowest maintainable fix because Telegram already owns media loading, document forcing, size limits, and local-root checks there.
