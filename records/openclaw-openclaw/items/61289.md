The PR adds `filePath` and `path` as Telegram `sendMessage` media aliases and tests that they map into the existing `mediaUrl` send path.

## Best Possible Solution

Land the focused alias mapping with concise Telegram docs and redacted real Telegram proof while preserving `mediaUrl` precedence and the existing media-loader boundary.

## Reproduction Assessment

## Solution Assessment

Yes, pending docs and proof. Reusing the existing `mediaUrl` pipeline is the narrowest maintainable solution because Telegram already owns media loading, document forcing, size limits, and local-root checks there.
