## Summary

Keep this PR open: current main still lacks the `filePath`/`path` Telegram action aliases, and the final diff is narrow, but merge should wait for real Telegram proof plus a small docs update.

## What This Changes

The PR maps Telegram `sendMessage` `filePath`/`path` params into the existing media path and adds action-runtime tests for those aliases.

## Best Possible Solution

Land the focused alias mapping with `mediaUrl` precedence preserved, add Telegram docs for aliases and local media roots, require redacted real Telegram proof, then rerun fresh gates.

## Reproduction Assessment

Yes. Source inspection of current main shows `sendMessage` reads `mediaUrl`/`media` only, so a `filePath`-only or `path`-only send leaves media undefined and hits `content required.` before upload.

## Solution Assessment

Yes, with docs and proof added. Reusing the existing `mediaUrl` pipeline is the narrowest maintainable fix because Telegram already owns media loading, document forcing, size limits, and local-root checks there.
