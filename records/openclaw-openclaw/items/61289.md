## Best Possible Solution

Land the narrow alias mapping with a concise Telegram docs update and redacted live Telegram proof, preserving `mediaUrl` precedence and existing local-root checks while leaving broader inline buffer/filename hydration to https://github.com/openclaw/openclaw/issues/41779.

## Reproduction Assessment

## Solution Assessment

Yes for the narrow alias gap: adding aliases after `mediaUrl` and `media` preserves existing precedence and reuses the existing media loader. It is not the full fix for the broader inline buffer/filename bug tracked by https://github.com/openclaw/openclaw/issues/41779.
