## What This Changes

The PR adds `filePath` and `path` fallbacks to Telegram direct `sendMessage` media resolution and focused unit tests for alias and `asDocument` forwarding.

## Best Possible Solution

Land the focused alias mapping with existing `mediaUrl` precedence preserved, focused tests, a small Telegram docs update, and a single changelog entry while leaving broader attachment abstraction work to follow-up.

## Solution Assessment

Yes, with the docs and changelog follow-up. Mapping the aliases into the existing `mediaUrl` pipeline is the narrow maintainable fix because Telegram already has media loading, document forcing, and local-root guard handling.
