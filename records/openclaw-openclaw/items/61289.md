## Summary

Keep open: the final diff is a focused, useful Telegram action-surface change that is not on current main, but it still needs contributor-provided real Telegram proof and a small docs update before merge.

## What This Changes

The PR adds `filePath` and `path` fallback aliases to Telegram `sendMessage` media resolution and tests those aliases while preserving existing `mediaUrl`/`media` precedence.

## Best Possible Solution

Land the focused alias mapping after adding action docs and contributor-posted real Telegram proof, while keeping broader attachment-abstraction work out of this PR.

## Reproduction Assessment

Yes. Source inspection on current main shows a `filePath`-only or `path`-only direct Telegram `sendMessage` action leaves media undefined and reaches the existing `content required.` failure path.

## Solution Assessment

Yes, with docs and proof added. Reusing the existing `mediaUrl` pipeline is the narrow maintainable approach because Telegram already owns media loading, document forcing, size limits, and local-root checks there.
