## What This Changes

The PR changes bundled session-memory to use timestamp memory filenames by default, calls the LLM slug generator only with `llmSlug: true`, and updates docs, tests, and changelog.

## Best Possible Solution

Land the narrow opt-in slugging behavior after the draft is ready and exact-head checks pass, preserving timestamp defaults with `llmSlug` as the documented escape hatch.

## Reproduction Assessment

Yes from source inspection: current `main` calls `generateSlugViaLLM` when hook config is absent or not false, so reset/new paths can wait on a nested model run. I did not execute the reproduction because this review is read-only.

## Solution Assessment

Yes: gating the existing model call behind `hookConfig?.llmSlug === true` is the narrowest maintainable change and keeps the timestamp fallback/write path intact. The added tests cover default no-model behavior and explicit opt-in behavior.
