The branch adds a GitHub-hosted `ci_scope` job, routes Markdown-only PR `preflight`, `security-fast`, `pnpm-store-warmup`, and `check-docs` jobs to `ubuntu-24.04`, and updates CI docs plus workflow guard tests.

## Best Possible Solution

Land this branch if maintainers accept the early hosted scope gate as the canonical routing strategy, then close the linked issue and retire the weaker alternate PRs as superseded.

## Reproduction Assessment

## Solution Assessment

Yes, this appears to be the strongest current fix candidate because it reuses the existing docs detector before runner selection and preserves docs/security jobs. The maintainer decision is whether that early dependency is preferable to broader `paths-ignore` strategies.
