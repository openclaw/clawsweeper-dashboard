## Summary

Keep open. Current `main` and v2026.6.6 still lack authoritative actual-runtime-model metadata on the OpenAI-compatible Gateway responses. The request remains useful, but the public contract needs maintainer judgment because an ordinary response header cannot report a fallback selected after SSE headers have been committed.

## What This Changes

Expose the actual provider/model selected for each request through the OpenAI-compatible Gateway response contract.

## Best Possible Solution

Define one authoritative selected-model contract across both OpenAI-compatible endpoints, using final response metadata for non-streaming requests and a documented SSE-compatible model-selection event or field for streaming requests.

## Reproduction Assessment

Yes. Current-main source deterministically shows both Gateway endpoints returning the request-side compatibility model rather than authoritative runtime-selected model metadata, and the streaming helper establishes the header timing limitation.

## Solution Assessment

No. An ordinary response header may help non-streaming clients, but it is not the best complete solution because late streaming fallback requires an SSE-compatible metadata representation using the same authoritative selection signal.
