## Summary

Keep open. Current main and the latest shipped tag still lack a Gateway response-side actual-runtime-model contract; the closed narrower client-display request points back here, while the open session/status issue and request-context header PR are adjacent rather than replacements.

## What This Changes

Request adding an actual runtime model response surface, proposed as `X-Actual-Model`, for Gateway OpenAI-compatible HTTP responses.

## Best Possible Solution

Define one additive actual-model metadata contract for `/v1/chat/completions` and `/v1/responses`, including SSE behavior, then wire it from selected-model plumbing with focused Gateway tests and public docs.

## Reproduction Assessment

Yes. Source inspection on current main shows the Gateway handlers still emit request-side model values and lack actual-runtime-model response metadata.

## Solution Assessment

No for a header-only change as the complete solution. Because streaming headers flush before late fallback selection may be known, the maintainable fix needs a documented streaming-compatible metadata contract.
