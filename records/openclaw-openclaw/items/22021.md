## Summary

Keep open. Current main still lacks an actual-runtime-model response metadata surface for the Gateway OpenAI-compatible HTTP endpoints, and the remaining decision is the public API contract for non-streaming and streaming responses rather than a stale cleanup candidate.

## What This Changes

Add an HTTP response metadata surface, proposed as `X-Actual-Model`, that exposes the actual runtime model used by Gateway OpenAI-compatible requests.

## Best Possible Solution

Define and document one Gateway actual-model metadata contract for `/v1/chat/completions` and `/v1/responses`, including non-streaming and streaming behavior, then wire it from selected-model plumbing with focused tests and docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main proof that the Gateway HTTP handlers still emit request-side model values and have no actual-runtime-model response metadata.

## Solution Assessment

No for the header-only proposal as a complete solution. A response header may cover non-streaming responses, but streaming fallback requires a documented late-bound metadata contract such as an SSE event, body field, trailer, or equivalent compatible mechanism.
