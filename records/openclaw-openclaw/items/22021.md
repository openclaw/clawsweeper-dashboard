## Summary

Keep open. Current main still lacks the requested actual-runtime-model response metadata on the Gateway OpenAI-compatible HTTP surfaces, and the header-only proposal still needs public API-contract review for streaming behavior after SSE headers flush.

## What This Changes

Request to expose the actual runtime model used by Gateway OpenAI-compatible HTTP requests, proposed as an `X-Actual-Model` response header.

## Best Possible Solution

Define and document one Gateway actual-model metadata contract for `/v1/chat/completions` and `/v1/responses`, including non-streaming and streaming behavior, then wire it from selected-model plumbing with focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main has no actual-runtime-model response metadata and still emits request-side model values in the Gateway HTTP handlers.

## Solution Assessment

No for the header-only proposal as a complete solution. A response header may work for non-streaming responses, but streaming fallback needs a documented late-bound metadata contract after headers flush.
