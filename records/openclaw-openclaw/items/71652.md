## Summary

Keep this PR open. The target wrapper/runtime path is real and current main does not yet have the proposed pressure guard, but the PR currently checks `/tmp` even after redirecting heavy-check temp writes to the worktree artifact directory, which can make the guard reject healthy runs or miss the actual full filesystem.

## What This Changes

The branch adds a local `tsgo` heavy-check pressure guard, redirects heavy-check temp files into `.artifacts/tmp/local-heavy-checks`, adds an `OPENCLAW_HEAVY_CHECK_FORCE` bypass, and extends focused run-tsgo tests.

## Best Possible Solution

Land the guard in the existing script wrapper/runtime seam after the temp-space check resolves the effective heavy-check temp directory and focused tests cover split-mount behavior; if `/tmp` remains a separate host-health signal, make that explicit and do not let it replace validation of the real temp target.

## Reproduction Assessment

Yes. On a host where `/tmp` and `.artifacts/tmp/local-heavy-checks` are different filesystems, make one low on space and the other healthy; the PR's `df -Pk /tmp` result can then disagree with the filesystem `tsgo` actually uses.

## Solution Assessment

No. The wrapper/runtime seam is the right place, but the proposed patch is not the narrowest correct fix until it checks the effective temp filesystem instead of hard-coding `/tmp`.
