## Summary

Keep open. The PR targets a real local tooling gap, but the branch redirects `tsgo` temporary files into the worktree artifact directory while the new pressure check still probes `/tmp`, so it can make the wrong allow/refuse decision on split-mount hosts.

## What This Changes

The PR adds a local `tsgo` heavy-check pressure guard, redirects heavy-check temp variables into a worktree artifact directory, adds a force bypass, and extends script tests.

## Best Possible Solution

Land a narrow script-runtime fix that resolves the effective heavy-check temp directory, probes the filesystem backing that path, and covers split-mount behavior while preserving the memory guard, force bypass, and existing lock semantics.

## Reproduction Assessment

Yes. On a host where `/tmp` and `OPENCLAW_LOCAL_HEAVY_CHECK_TMPDIR` or `.artifacts/tmp/local-heavy-checks` are different filesystems, make one low on space and leave the other healthy; the branch's `df -Pk /tmp` check can disagree with the filesystem `tsgo` will actually use.

## Solution Assessment

No. The wrapper/runtime seam is the right place, but this patch should check the effective temp target after redirecting temp variables; `/tmp` can remain only as an explicitly separate host-health signal if maintainers want that behavior.
