## Summary

Keep this PR open: the core local-refusal direction is useful and not implemented on current main, but the latest branch still lets sharded oxlint metadata-only commands fall through to temp-dir setup, artifact preparation, and child shard spawning.

## What This Changes

The PR adds local refusal and pressure guards for native type-aware validation scripts, including tsgo, oxlint, oxlint shards, and direct tsgo helper scripts.

## Best Possible Solution

Keep the local native-refusal policy, but make `run-oxlint-shards` detect metadata-only args before temp-dir setup, locking, artifact preparation, or shard spawning, then validate both refusal and metadata paths.

## Reproduction Assessment

Yes. Source inspection at PR head shows `run-oxlint-shards` no longer refuses metadata-only args, but it still falls through to artifact preparation and shard spawning for `--help` or `--version`.

## Solution Assessment

No, not yet. The shared refusal helper and pressure checks are a maintainable direction, but the sharded wrapper needs an early metadata-only branch before merge.
