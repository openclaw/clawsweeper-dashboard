## Summary

Keep open: the PR is useful and not implemented on current main, but PR head still breaks its own metadata-only contract for sharded oxlint by treating every shard invocation as native heavy work before checking `--help` or `--version`.

## What This Changes

The PR adds local native type-aware check refusal and host-pressure guards across tsgo and oxlint script wrappers, with focused script tests.

## Best Possible Solution

Keep the native-refusal policy, but make `run-oxlint-shards` share the same metadata-only skip decision before refusal, locking, temp-dir setup, or artifact preparation.

## Reproduction Assessment

Yes. Source inspection at PR head shows `scripts/run-oxlint-shards.mjs` passes `shouldRunHeavyCheck: true` for all args before any metadata skip, so `OPENCLAW_LOCAL_CHECK_MODE=throttled node scripts/run-oxlint-shards.mjs --help` can be refused instead of staying cheap.

## Solution Assessment

No, not yet. The wrapper-level refusal and temp-filesystem direction are maintainable, but sharded oxlint needs the same metadata-skip gate as `run-oxlint.mjs` before merge.
