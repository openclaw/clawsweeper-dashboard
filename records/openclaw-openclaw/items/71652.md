## Summary

Keep open: the script-wrapper guard is useful and not already on main, but the PR still refuses metadata-only sharded oxlint invocations even though the PR promises those commands stay cheap and unblocked.

## What This Changes

The PR adds local native type-aware check refusal and host-pressure guards across tsgo and oxlint script wrappers, direct tsgo helper scripts, temp-directory routing, and focused script tests.

## Best Possible Solution

Keep the native-refusal policy, but make run-oxlint-shards compute and honor the same metadata-only skip decision before refusal, locking, temp-dir setup, or artifact preparation.

## Reproduction Assessment

Yes. Source inspection at PR head shows run-oxlint-shards passes shouldRunHeavyCheck: true for all args, so OPENCLAW_LOCAL_CHECK_MODE=throttled node scripts/run-oxlint-shards.mjs --help can be refused instead of staying cheap.

## Solution Assessment

No, not yet. The wrapper-level refusal and temp-filesystem direction are maintainable, but sharded oxlint needs the same metadata-skip gate as the single oxlint wrapper before merge.
