## Summary

Keep open: current main and the latest shipped release still intentionally narrow Codex `danger-full-access` to `workspace-write` whenever an OpenClaw sandbox is active, and there is no Docker-specific opt-out. The requested change is valid-looking, but it changes sandbox boundary policy and needs maintainer security/product review before implementation.

## What This Changes

Request a supported Codex plugin configuration path to bypass Codex's inner sandbox only when OpenClaw Docker sandboxing already owns isolation.

## Best Possible Solution

Add an explicit, documented Codex plugin opt-in scoped to already-isolated OpenClaw Docker sandbox sessions, preserving outer sandbox and tool-policy guarantees and failing closed outside that context.

## Reproduction Assessment

No high-confidence live repro was run. Source and release inspection prove the shipped/current clamp and lack of opt-out, but the exact `bwrap` failure still depends on the Docker/Codex runtime environment.

## Solution Assessment

Unclear until maintainers decide the trust model. A narrow Docker-only opt-in with diagnostics and docs is safer than removing the existing clamp globally.
