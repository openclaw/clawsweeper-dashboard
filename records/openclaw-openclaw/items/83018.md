## Summary

Keep open: current main and v2026.5.12 intentionally clamp Codex `danger-full-access` to `workspace-write` under any active OpenClaw sandbox, so the requested Docker-specific opt-out is not implemented and needs sandbox/security product review.

## What This Changes

The issue requests a supported Codex configuration path to disable Codex's inner sandbox when an OpenClaw-managed Docker sandbox is already active.

## Best Possible Solution

Add an explicit, narrowly scoped Codex plugin opt-in for already-isolated Docker sandbox runs, with diagnostics and docs that preserve OpenClaw's outer sandbox and tool-policy guarantees.

## Reproduction Assessment

No high-confidence live repro was run. Source and shipped-release inspection do confirm the reported configuration is clamped from Codex `danger-full-access` to `workspace-write` under an active OpenClaw sandbox.

## Solution Assessment

Unclear until owners decide the trust model. A narrow Docker-aware opt-in with guardrails is safer than removing the existing clamp globally, because current docs frame the clamp as sandbox protection.
