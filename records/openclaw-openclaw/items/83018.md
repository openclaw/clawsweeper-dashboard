## Summary

Keep open: current main still intentionally narrows Codex `danger-full-access` to `workspace-write` whenever an OpenClaw sandbox is active, so the requested Docker-specific opt-out is not implemented and needs sandbox/security product review.

## What This Changes

Requests a supported Codex app-server config option to avoid Codex's inner sandbox when an OpenClaw-managed Docker sandbox is already active.

## Best Possible Solution

Add an explicit, narrowly scoped Codex plugin opt-in for already-isolated Docker sandbox runs, with diagnostics and docs that preserve OpenClaw's outer sandbox and tool-policy guarantees.

## Reproduction Assessment

No high-confidence live repro was run. Source and shipped-release inspection do confirm the reported configuration is clamped from Codex `danger-full-access` to `workspace-write` under an active OpenClaw sandbox.

## Solution Assessment

Unclear until owners decide the trust model. A narrow opt-in with guardrails is safer than removing the existing clamp globally, because the current docs frame the clamp as protection against widening sandbox access.
