## Summary

Keep open: current main still lacks native outbound Signal quote parameters, and the PR still has a merge-blocking timestamp-only quote path against signal-cli's contract plus insufficient live Signal proof.

## What This Changes

This PR adds Signal inbound/outbound quote-reply metadata plumbing, native signal-cli quote parameters, queue and delivery reply preservation, SDK reply-payload type changes, tests, and a changelog entry.

## Best Possible Solution

Require a resolved quote-author for every Signal quote send or drop quote params when the author is unavailable, then collect redacted live paired Signal send/receive proof before merge.

## Reproduction Assessment

Yes for the review blocker by source inspection: PR head can emit quote-timestamp without quote-author for direct targets, and signal-cli v0.14.3 throws on that combination. I did not run a live Signal account roundtrip.

## Solution Assessment

No. The implementation direction is useful, but the send helper must respect signal-cli's author requirement for all quote sends and the PR needs live Signal proof before it is the best merge path.
