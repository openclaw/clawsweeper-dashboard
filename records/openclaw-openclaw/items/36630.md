## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Route formatted Signal sends through quote metadata:** `src/infra/outbound/deliver.ts:1192-1197`
  - body: Text-only Signal payloads still take the `sendFormattedText` branch, but Signal's formatted sender ignores the `replyToId`/`quoteAuthor` fields carried in the overrides. Generic outbound sends with a Signal `replyToId` will still post as top-level messages, bypassing the quote support added in `sendMessageSignal`.
  - confidence: 0.9
- **[P1] Remove core deep imports from Signal production modules:** `extensions/signal/src/monitor/reply-delivery.ts:1`
  - body: This new Signal production module imports `ReplyPayload` from core `src/**`; `quote-context.ts` adds similar relative core imports. Bundled plugins must use SDK surfaces or plugin-local facades, otherwise standalone/plugin-package builds depend on private repo layout.
  - confidence: 0.93
- **[P2] Treat blank Signal reply IDs as unset:** `extensions/signal/src/monitor/reply-delivery.ts:12-13`
  - body: Whitespace-only `replyToId` values are normalized to `null`, which this helper treats as an explicit do-not-reply override. Payload builders that emit `""` or blanks will drop the inherited Signal quote instead of inheriting the current message context.
  - confidence: 0.88
- **[P2] Keep debounced quote context tied to the latest message:** `extensions/signal/src/monitor/event-handler.ts:508-518`
  - body: When a quoted message is followed quickly by a plain message, this code searches backward for any quote and attaches the older quote to the merged inbound turn. The agent then sees reply context that no longer belongs to the latest user message.
  - confidence: 0.87
- **[P2] Require decimal Signal quote IDs during normalization:** `extensions/signal/src/monitor/quote-context.ts:67-68`
  - body: `Number(trimmed)` accepts values like `1e3` and `0x10`, so malformed quote IDs can alias to different message timestamps. Signal quote IDs should use the same decimal-only validation as outbound quote timestamps.
  - confidence: 0.86
- **[P2] Fall back to mediaUrl when mediaUrls is empty:** `extensions/signal/src/monitor.ts:359-360`
  - body: Using nullish coalescing here means `{ mediaUrls: [], mediaUrl: "..." }` produces an empty media list and skips the attachment. The removed helper fell back to `mediaUrl` unless `mediaUrls` had entries, so this regresses single-media delivery for that payload shape.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `CHANGELOG.md:55`
  - body: This is a user-facing Signal fix, but the PR file list does not include `CHANGELOG.md`. Repo policy requires user-facing fixes to add a single-line entry under the active changelog section before merge.
  - confidence: 0.82
