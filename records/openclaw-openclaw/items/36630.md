## Summary

Keep this PR open. Current main still lacks outbound Signal quote parameters, but the latest PR head still has a merge-blocking Signal RPC contract issue: direct replies can emit `quote-timestamp` without `quote-author`, which signal-cli rejects, and the PR still lacks live paired Signal send/receive proof.

## What This Changes

The branch wires Signal quote-reply metadata through Signal send/monitor paths, shared outbound delivery, queue recovery, SDK reply payload types, tests, and the changelog.

## Best Possible Solution

Keep the branch open, require `quoteAuthor` for every Signal quote send with plain-message fallback when unavailable, and add redacted live paired Signal outbound and inbound quote proof before merge.

## Reproduction Assessment

Yes, for the review blocker by source inspection and upstream contract: PR head can emit `quote-timestamp` without `quote-author` for direct sends, and signal-cli v0.14.3 throws on that combination. I did not run a live Signal account roundtrip.

## Solution Assessment

No. The implementation direction is useful, but the send helper must drop quote metadata unless both timestamp and author are available, and the PR still needs live Signal proof before it is the best merge path.
