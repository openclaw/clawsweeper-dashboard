## Summary

Keep open: current main still lacks native Signal quote send parameters, but this PR remains unsafe to merge because the latest diff still crosses extension boundaries, widens shared reply semantics without the matching threading implementation, and leaves a quote-id security issue.

## What This Changes

The PR adds Signal quote-reply plumbing across Signal send/monitor code, shared outbound delivery, reply payload and plugin SDK types, plus regression tests.

## Best Possible Solution

Land a smaller reworked Signal quote-reply implementation that uses public SDK seams, updates shared nullable reply suppression semantics first, validates Signal quote ids as decimal timestamps, and routes all Signal send paths through one quote-author policy with focused tests and changelog coverage.

Yes. Static reproduction is high confidence from current main plus the latest PR diff: the existing Signal send path lacks quote params, and the proposed diff still shows the merge-blocking boundary, threading, and quote-id validation paths.

## Solution Assessment

No. The direction is useful, but this PR is not the best merge vehicle until the SDK boundary, shared reply contract, formatted delivery path, media fallback, and quote-id validation are corrected together.

Full review comments:

- **[P1] Update threading before allowing null reply IDs:** `src/auto-reply/reply-payload.ts:28`
  - body: This widens `replyToId` to include `null`, but shared threading still treats falsy ids as unset and can inject the implicit current message id. Explicit do-not-reply payloads can therefore still be threaded, and the new null-suppression tests disagree with the implementation.
  - confidence: 0.96
- **[P1] Move quote-context imports behind SDK surfaces:** `extensions/signal/src/monitor/quote-context.ts:1-2`
  - body: This new production Signal helper imports `kindFromMime` and `normalizeE164` via relative `../../../../src/**` paths. Bundled plugin production code must use `openclaw/plugin-sdk/*` or plugin-local APIs so standalone plugin packaging does not depend on core layout internals.
  - confidence: 0.97
- **[P1] Move reply-delivery types behind the SDK:** `extensions/signal/src/monitor/reply-delivery.ts:1`
  - body: This production Signal helper imports `ReplyPayload` from a core `src/**` path. Use the public reply runtime/payload SDK surface or a plugin-local type facade instead of coupling plugin code to core internals.
  - confidence: 0.96
- **[P2] Forward quote metadata through formatted Signal sends:** `src/infra/outbound/deliver.ts:1129`
  - body: Adding `quoteAuthor` to shared send overrides is not enough for the default formatted Signal path: `sendFormattedSignalText` and `sendFormattedSignalMedia` still call the Signal send runtime without `replyTo` or `quoteAuthor`. Generic Signal sends with reply metadata will continue to post without native quote bubbles.
  - confidence: 0.9
- **[P2] Require decimal-only inbound quote IDs:** `extensions/signal/src/monitor/quote-context.ts:67-68`
  - body: `normalizeSignalQuoteId` uses `Number(trimmed)`, which accepts values like `1e3` and `0x10`. Those can alias inbound reply context and author-cache lookups to a different message id; inbound quote IDs should use the same decimal-only validation as outbound quote timestamps.
  - confidence: 0.92
- **[P2] Treat blank Signal reply IDs as unset:** `extensions/signal/src/monitor/reply-delivery.ts:12-13`
  - body: `normalizeReplyToId` converts whitespace-only strings to `null`, which this module treats as explicit reply suppression. Blank payload ids should behave like `undefined` so inherited quote context can still apply; reserve `null` for intentional do-not-reply overrides.
  - confidence: 0.89
- **[P2] Fall back to mediaUrl when mediaUrls is empty:** `extensions/signal/src/monitor.ts:359-360`
  - body: `resolvedPayload.mediaUrls ?? ...` treats an explicit empty array as final, so `{ mediaUrls: [], mediaUrl: "..." }` drops the single-media fallback that the previous shared reply-payload helper preserved. Media replies can be skipped even though a legacy `mediaUrl` is present.
  - confidence: 0.88
