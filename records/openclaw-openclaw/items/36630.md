## Best Possible Solution

Keep the Signal-owned implementation direction, return or carry whether the resolved target came from inherited state, add an explicit-first-then-inherited regression, and require redacted live Signal proof before merge.

No. The approach is directionally maintainable, but this PR should track inherited reply usage separately from explicit payload replies and include live Signal send/receive proof before merge.
