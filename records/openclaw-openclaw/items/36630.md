## Summary

Keep this PR open: it addresses a real Signal reply gap, but the current head still has source-reproducible blockers in quote-author handling and container-mode forwarding, plus it lacks live Signal roundtrip proof.

## What This Changes

The PR adds bidirectional Signal quote-reply handling, propagates reply and quote metadata through outbound delivery and queues, widens reply payload types, adds tests, and updates the changelog.

## Best Possible Solution

Land a finished Signal reply implementation that satisfies signal-cli quote contracts, forwards quote fields through native and container modes, preserves queued reply context, and includes redacted live Signal send/receive proof.

## Reproduction Assessment

Yes. Source inspection at PR head shows quote-timestamp can be emitted without quote-author, and the container send path does not forward quote fields even though the underlying APIs define them.

## Solution Assessment

No. The direction is right, but the patch is not mergeable until quote metadata always includes an author when emitted, container sends forward quote fields, and live Signal proof is supplied.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require quote authors before sending quote metadata:** `extensions/signal/src/reply-quote.ts:34-39`
  - body: signal-cli throws when `quote-timestamp` is present without `quote-author`, but this helper still returns timestamp-only metadata for direct targets and `sendMessageSignal` forwards it. Direct replies with an unresolved author can fail instead of degrading to a normal message; drop quote params unless both fields are available or resolve the author before sending.
  - confidence: 0.94
- **[P2] Forward quote fields through container sends:** `extensions/signal/src/send.ts:266-271`
  - body: The new native RPC params do not reach container mode: `signalRpcRequest` can route sends through `containerRpcRequest`, whose `/v2/send` payload supports `quote_timestamp` and `quote_author` but never maps them. Container Signal users will still get plain replies unless those fields are forwarded and covered by a test.
  - confidence: 0.9
