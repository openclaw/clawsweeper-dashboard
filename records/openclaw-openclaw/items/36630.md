## Summary

Keep open: current main does not implement the requested Signal quote-reply support, but this PR still has merge-blocking correctness issues in native and container Signal sends, and the supplied proof is not a live Signal device/account roundtrip.

## What This Changes

The PR adds Signal outbound quote metadata, inbound ReplyTo quote context, reply metadata propagation through shared outbound/queue plumbing, tests, and a changelog entry.

## Best Possible Solution

Keep the PR open, require quote metadata to include a valid author before native signal-cli sends, map quote fields through container mode, and add redacted live Signal send/receive proof before merge.

## Reproduction Assessment

Yes, for the PR blockers by source inspection and upstream contracts: PR head can send native signal-cli quote-timestamp without quote-author, and container mode drops quote fields even though the REST API exposes them. I did not run a live Signal roundtrip.

## Solution Assessment

No. The implementation direction is maintainable, but it needs the native quote-author requirement, container quote field forwarding, and live Signal proof before it is the best mergeable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Require quote authors before sending Signal quotes:** `extensions/signal/src/reply-quote.ts:35-37`
  - body: signal-cli rejects any request with quote-timestamp but no quote-author, but this helper only gates missing authors for group targets. A direct reply with no resolved author still returns timestamp-only metadata, so sendMessageSignal can submit an invalid native RPC request instead of degrading to a normal message; drop quote params unless both timestamp and author are available.
  - confidence: 0.94
- **[P2] Forward quote metadata through container sends:** `extensions/signal/src/send.ts:266-271`
  - body: The new quote params are added before signalRpcRequest, but container mode translates send through client-container.ts and currently forwards only message, recipients, text styles, attachments, and timeout. Since the container REST API exposes quote_timestamp and quote_author, container-mode Signal users will still send plain unquoted replies unless those fields are mapped and covered by tests.
  - confidence: 0.88
