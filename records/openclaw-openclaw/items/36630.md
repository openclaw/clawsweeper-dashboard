## Summary

Keep this PR open: the Signal quote-reply direction is useful, but the latest inspected head still has blocking source-level correctness issues and lacks live Signal send/receive proof.

## What This Changes

The PR adds outbound Signal quote metadata, inbound quoted-message context extraction, reply-delivery plumbing, shared reply payload typing changes, and regression tests across Signal and outbound delivery paths.

## Best Possible Solution

Revise the branch so Signal formatted/direct/media/debounce/recovery paths preserve or intentionally suppress quote metadata consistently, validate quote IDs/authors against signal-cli’s contract, and add redacted live Signal send/receive proof before merge.

## Reproduction Assessment

Yes, at source level. Current main lacks Signal quote RPC metadata, and PR head e3740e0 still has clear source-level failures in quote-author requirements, formatted sends, debounced quote handling, media fallback, ID normalization, blank reply IDs, and queued recovery replay.

## Solution Assessment

No. The implementation direction is reasonable, but this patch is not yet the narrow maintainable solution because several changed paths still bypass or corrupt quote metadata and native Signal behavior remains unproven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require quote authors before emitting Signal quote metadata:** `extensions/signal/src/reply-quote.ts:34-39`
  - body: `resolveSignalQuoteMetadata` returns `quoteTimestamp` for direct chats even when `quoteAuthor` is absent, and `sendMessageSignal` then sends `quote-timestamp` without `quote-author`. Upstream signal-cli rejects that combination, so DM replies with only `replyToId` can fail instead of degrading to an unquoted message.
  - confidence: 0.91
- **[P2] Forward quote overrides from formatted Signal sends:** `extensions/signal/src/channel.ts:191-197`
  - body: The outbound core passes reply overrides into `sendFormattedText`, but Signal's formatted sender calls `send` without `replyTo` or `quoteAuthor`. Text-only formatted Signal replies still post as top-level messages instead of native quote replies.
  - confidence: 0.92
- **[P1] Dispatch quote-only debounced batches:** `extensions/signal/src/monitor/event-handler.ts:503-504`
  - body: When debounced Signal entries contain only quote context, `combinedText` is empty and this return drops the batch before dispatch or history. Quote-only messages are valid in this PR because their content lives in `quoteTarget`, so the flush path needs to keep them.
  - confidence: 0.89
- **[P2] Use the latest debounced message's quote state:** `extensions/signal/src/monitor/event-handler.ts:508-518`
  - body: This searches backward for any quoted entry and can attach an older quote to the merged batch even when the latest debounced message is plain text. A quoted message followed by a quick plain follow-up is surfaced with stale `ReplyTo*` context.
  - confidence: 0.85
- **[P2] Fall back to mediaUrl when mediaUrls is empty:** `extensions/signal/src/monitor.ts:369-370`
  - body: `mediaUrls ?? ...` treats an empty array as authoritative, so a payload with `mediaUrls: []` and a populated `mediaUrl` loses its attachment. Text-empty payloads in that shape can be skipped entirely.
  - confidence: 0.84
- **[P2] Require decimal Signal quote IDs:** `extensions/signal/src/monitor/quote-context.ts:67-68`
  - body: `Number(trimmed)` accepts non-decimal strings like `1e3` or `0x10`, normalizing malformed quote IDs into different timestamps. Inbound quote normalization should require decimal digits before surfacing `ReplyToId` or doing author-cache lookup.
  - confidence: 0.84
- **[P2] Treat blank Signal reply IDs as unset:** `extensions/signal/src/monitor/reply-delivery.ts:12-13`
  - body: Whitespace-only `replyToId` values become `null`, but this helper uses `null` as explicit reply suppression. A blank payload reply id therefore drops the inherited Signal quote instead of falling through as unset.
  - confidence: 0.77
- **[P2] Persist quoteAuthor in queued deliveries:** `src/infra/outbound/deliver.ts:855-871`
  - body: The live send path carries `quoteAuthor`, but the write-ahead queue persists only `replyToId` and recovery replays that truncated payload. Recovered Signal group replies lose the required author and are resent unquoted.
  - confidence: 0.81
