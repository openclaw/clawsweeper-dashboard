## Summary

Keep open: current main still lacks outbound Signal quote parameters, and the PR is not ready because it can emit `quote-timestamp` without `quote-author` even though signal-cli requires an author for any quote send. The PR also still needs live paired Signal send/receive proof before merge.

## What This Changes

The PR adds Signal quote-reply parsing and sending, quote-author plumbing through outbound delivery and queue recovery, reply payload null semantics, tests, SDK baseline updates, and a changelog entry.

## Best Possible Solution

Require a resolved `quote-author` for every Signal quote send or drop quote params when the author is unavailable, then collect redacted live Signal device/account proof before merge.

## Reproduction Assessment

Yes for the review blocker by source inspection: PR head can return `quoteTimestamp` without `quoteAuthor` for direct targets, and signal-cli source shows that combination throws. I did not run a live Signal roundtrip.

## Solution Assessment

No. The implementation direction is useful, but the send helper must respect signal-cli's author requirement for all quote sends and the PR needs live Signal proof before it is the best merge path.
