## Summary

Keep open: current main still lacks native Signal quote-reply support, but this PR head has blocking formatted-send, debounce, media, and ID-normalization regressions plus insufficient live Signal proof.

## What This Changes

The PR adds Signal quote metadata plumbing, inbound quote context extraction, reply-consumption changes across outbound delivery, and regression tests for Signal and shared reply payload types.

## Best Possible Solution

Revise the PR so every Signal formatted/direct send path carries quote metadata, debounce/media/id edge cases are fixed, and redacted live Signal send/receive proof is added before merge.

## Reproduction Assessment

Yes, at source level. Current main omits Signal quote RPC params, and PR head still has source-reproducible gaps in formatted sends, debounce handling, media fallback, and quote ID normalization.

## Solution Assessment

No. The overall direction is reasonable, but this is not yet the best maintainable fix until the blocking findings and live Signal proof gap are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Forward quote overrides from formatted Signal sends:** `extensions/signal/src/channel.ts:191-197`
  - body: `deliverOutboundPayloadsCore` now passes quote overrides into formatted sends, but `sendFormattedSignalText` ignores `replyToId` and `quoteAuthor` when it calls `send`. The default formatted Signal text path will still post top-level messages instead of native quote replies.
  - confidence: 0.91
- **[P1] Dispatch quote-only debounced batches:** `extensions/signal/src/monitor/event-handler.ts:503-504`
  - body: When multiple quote-only Signal messages debounce together, `combinedText` is empty and this return drops the whole batch before dispatch/history. Quote-only messages are now valid because their content is in `quoteTarget`, so the flush path needs to keep them.
  - confidence: 0.88
- **[P2] Use the latest debounced message's quote state:** `extensions/signal/src/monitor/event-handler.ts:508-518`
  - body: This searches backward for any quoted entry and can attach an older quote to the merged batch even when the latest debounced message is plain text. A quoted message followed by a quick plain follow-up is surfaced with stale `ReplyTo*` context.
  - confidence: 0.82
- **[P2] Fall back to mediaUrl when mediaUrls is empty:** `extensions/signal/src/monitor.ts:369-370`
  - body: `mediaUrls ?? ...` treats an empty array as authoritative, so a payload with `mediaUrls: []` and a populated `mediaUrl` loses its attachment. Text-empty payloads in that shape can be skipped entirely.
  - confidence: 0.84
- **[P2] Require decimal Signal quote IDs:** `extensions/signal/src/monitor/quote-context.ts:67-68`
  - body: `Number(trimmed)` accepts non-decimal strings such as `1e3` or `0x10`, normalizing malformed quote IDs into different timestamps. Inbound quote normalization should require decimal digits before surfacing `ReplyToId` or doing author-cache lookup.
  - confidence: 0.82
- **[P2] Treat blank Signal reply IDs as unset:** `extensions/signal/src/monitor/reply-delivery.ts:12-13`
  - body: Whitespace-only `replyToId` values become `null`, but this helper uses `null` as explicit reply suppression. A blank payload reply id therefore drops the inherited Signal quote instead of falling through as unset.
  - confidence: 0.76
