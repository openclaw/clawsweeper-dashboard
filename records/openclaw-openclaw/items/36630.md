## Review Findings

Full review comments:

- **[P1] Forward quote metadata through container sends:** `extensions/signal/src/send.ts:273`
  - body: The PR adds `quote-timestamp` and `quote-author` before `signalRpcRequest`, but container mode routes this same call through `containerRpcRequest`, which only forwards message, recipients, styles, and attachments into `/v2/send`. Users on `apiMode: "container"` or auto-detected container mode will still get plain Signal messages with no quote bubble; map these fields to the REST `quote_timestamp`/`quote_author` payload and add a container-path test.
  - confidence: 0.9
- **[P2] Consume Signal reply state only for inherited targets:** `extensions/signal/src/monitor.ts:411-414`
  - body: `effectiveReplyTo` can be a payload-level explicit reply id, but this call marks the shared Signal reply state consumed for any valid effective id. If the first payload explicitly replies elsewhere, a later payload without an explicit id no longer inherits the current inbound message id, so the turn loses its intended quote target; return whether the inherited target was used and only consume in that case.
  - confidence: 0.82
