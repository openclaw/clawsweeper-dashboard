## Summary

Keep open: current main does not implement outbound Signal quote replies, and this active PR still has source-reproducible correctness blockers plus insufficient live Signal proof.

## What This Changes

The branch adds bidirectional Signal quote-reply support, quote metadata propagation through outbound delivery and queues, reply payload type updates, tests, and a changelog entry.

## Best Possible Solution

Land a finished Signal reply implementation that always satisfies signal-cli quote contracts, forwards quote fields in native and container modes, preserves reply context through queues, and includes redacted live Signal roundtrip proof.

## Reproduction Assessment

Yes. The blockers are source-reproducible from PR head: timestamp-only quote metadata conflicts with signal-cli's SendCommand contract, and the container send adapter omits REST quote fields that the container API supports.

## Solution Assessment

No. The direction is right, but the current patch is not the best mergeable solution until quote metadata always includes an author when emitted, container sends forward quote fields, and live Signal proof is supplied.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

  - confidence: 0.94

Container Signal users will still get plain replies unless those fields are forwarded and covered by a test.
  - confidence: 0.9
