## Summary

Keep open. Current main still does not provide complete native Signal quote replies, and this PR is useful source work, but the proposed branch still has blocking shared reply-threading, plugin-boundary, formatted-delivery, queue-replay, media-fallback, and inbound quote-id validation problems before it should merge.

## What This Changes

The PR adds Signal inbound quote-context parsing, outbound Signal quote metadata plumbing, shared reply payload and delivery changes for quote handling, and regression tests across Signal and shared outbound delivery paths.

## Best Possible Solution

Land a narrower Signal-owned implementation that keeps production plugin imports behind SDK surfaces, updates shared reply-threading semantics before widening public payload types, carries quote metadata through formatted/direct/queued Signal sends, validates quote IDs as decimal timestamps, and proves direct plus group quote behavior with focused tests and a live Signal smoke.

## Reproduction Assessment

Yes. Static reproduction is high confidence from the current main code plus the provided PR diff: the new null replyToId tests disagree with unchanged threading logic, new Signal helpers deep-import core internals, formatted sends still omit reply metadata, queued delivery lacks quoteAuthor, and inbound quote ID normalization accepts Number coercions.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best merge vehicle until the shared threading contract, SDK boundary, formatted-send path, queue recovery, media fallback, and quote-id validation issues are corrected.
