## Summary

Keep open: current main still lacks outbound Signal quote RPC support, but this PR head still has a blocking reply-consumption bug and the supplied proof does not show a live Signal send/receive roundtrip.

## What This Changes

Adds bidirectional Signal quote-reply handling across outbound Signal delivery, inbound quote context parsing, reply payload nullability, queued delivery recovery, tests, SDK baseline metadata, and changelog.

## Best Possible Solution

Keep the Signal-owned implementation direction, but carry whether the inherited reply target was actually used, add an explicit-first-then-inherited regression, and require redacted live Signal send/receive proof before merge.

## Reproduction Assessment

Yes, at source level. Current main lacks Signal quote RPC params, and the PR-head bug follows from sending a first payload with an explicit replyToId while an inheritedReplyToId is present, then observing the next untagged payload lose the inherited quote state.

## Solution Assessment

No. The overall direction is maintainable, but the PR should distinguish explicit payload replies from inherited current-message replies when consuming state, and it should include live Signal proof before merge.
