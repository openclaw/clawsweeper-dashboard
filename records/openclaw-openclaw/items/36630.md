## Summary

Keep this PR open. Current main still lacks native Signal quote-reply support, but the latest PR head has blocking correctness issues on the formatted Signal send path, debounced quote-only inbound handling, media fallback handling, and quote ID validation; earlier deep-import and conflict-marker concerns appear addressed.

## What This Changes

The PR adds Signal quote metadata plumbing, inbound quote context extraction and author indexing, `replyToId` null semantics, and Signal/outbound regression tests.

## Best Possible Solution

Revise the PR so quote metadata reaches every Signal formatted/direct send path, inbound debounce and media edge cases are covered, decimal-only quote IDs are enforced, and redacted live Signal proof is added.

## Reproduction Assessment

Yes. Source inspection shows current main strips reply directives into reply metadata but Signal sends do not emit native quote RPC params, and the latest PR head still drops quote metadata on the formatted text path.

## Solution Assessment

No. The direction is reasonable, but this is not yet the best maintainable fix until the formatted-send, debounce, media fallback, blank reply ID, quote ID validation, and real-behavior-proof gaps are closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Forward quote metadata from formatted Signal sends:** `extensions/signal/src/channel.ts:191-197`
  - body: `deliverOutboundPayloadsCore` now forwards quote overrides into formatted sends, and Signal text delivery normally uses `sendFormattedSignalText`. That function still calls `send()` without `replyTo` or `quoteAuthor`, so text-only Signal replies continue to post as top-level messages instead of native quote replies.
  - confidence: 0.9
- **[P1] Dispatch quote-only debounced batches:** `extensions/signal/src/monitor/event-handler.ts:499-504`
  - body: When several quote-only Signal messages debounce together, `combinedText` is empty and this early return drops the whole batch before dispatch/history. Quote-only messages are valid after this PR because their context lives in `quoteTarget`, so the flush path needs to keep them.
  - confidence: 0.86
- **[P2] Use the latest debounced message's quote state:** `extensions/signal/src/monitor/event-handler.ts:506-518`
  - body: This carries an older `quoteTarget` forward even when the last debounced message is plain text. A quoted message followed quickly by a plain follow-up is surfaced with stale `ReplyTo*` context that no longer describes the latest inbound message.
  - confidence: 0.78
- **[P2] Preserve mediaUrl fallback when mediaUrls is empty:** `extensions/signal/src/monitor.ts:369-370`
  - body: Using `mediaUrls ?? ...` treats an empty `mediaUrls: []` as authoritative and skips a populated `mediaUrl`. Those payloads lose their single attachment, and text-empty payloads can be skipped entirely.
  - confidence: 0.83
- **[P2] Require decimal Signal quote IDs:** `extensions/signal/src/monitor/quote-context.ts:67-68`
  - body: `Number(trimmed)` accepts strings like `1e3` and `0x10`, normalizing malformed quote IDs into different timestamps. Inbound quote normalization should require decimal digits before surfacing `ReplyToId` or doing author-cache lookup.
  - confidence: 0.8
- **[P2] Treat blank Signal reply IDs as unset:** `extensions/signal/src/monitor/reply-delivery.ts:12-13`
  - body: Blank strings are normalized to `null`, but this helper uses `null` as explicit reply suppression. A whitespace `replyToId` therefore drops the inherited Signal quote instead of falling through as unset.
  - confidence: 0.75
