## Summary

Keep this PR open, but it is not ready to merge: the Signal quote direction is useful, yet the current branch can still send `quote-timestamp` without `quote-author`, which signal-cli rejects, and the supplied proof does not include a live paired Signal send/receive roundtrip.

## What This Changes

The branch adds bidirectional Signal quote-reply handling, propagates reply and quote metadata through outbound delivery and queues, expands inbound Signal quote context, and adds regression tests plus a changelog entry.

## Best Possible Solution

Require a resolved quote author for every Signal quote send, or drop all quote params when no author is available, while preserving the structured inbound context work and validating it with redacted live Signal send/receive proof.

## Reproduction Assessment

## Solution Assessment
