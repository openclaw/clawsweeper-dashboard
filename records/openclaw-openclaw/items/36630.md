## Summary

Keep open: the PR still carries useful Signal quote-reply work, but the current head has a blocking SDK boundary failure, a source-level Tlon thread-routing regression, and no live Signal send/receive proof.

## What This Changes

The PR adds Signal native quote-reply send and inbound parsing support, threads quote metadata through outbound delivery and queue recovery, widens reply payload typing, and updates Signal/outbound tests plus changelog/API baseline files.

## Best Possible Solution

Keep the Signal quote-reply implementation path, but move the import to a supported SDK subpath, make inherited reply consumption capability-aware for persistent-thread channels like Tlon, refresh any generated protocol artifacts still required by CI, and add redacted live Signal send/receive proof.

## Reproduction Assessment

Yes for the review blockers: CI logs reproduce the deprecated SDK import failure, and source inspection shows the thread-channel allowlist suppresses inherited reply IDs for Tlon multi-payload group replies. I did not reproduce a live Signal roundtrip.

## Solution Assessment

No. The implementation direction is reasonable, but this patch is not ready until the SDK boundary failure, persistent-thread routing regression, generated-artifact check failure, and missing live Signal proof are addressed.
