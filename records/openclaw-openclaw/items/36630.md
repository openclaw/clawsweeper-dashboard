## Summary

Keep open: the PR still has concrete Signal quote-reply correctness blockers, and current main does not already implement the full outbound/inbound bidirectional quote behavior.

## What This Changes

The PR adds bidirectional Signal quote-reply handling across the Signal plugin, outbound delivery reply metadata, queue persistence, SDK reply payload typing, tests, and changelog.

## Best Possible Solution

Keep the Signal quote-reply direction, but require a resolved quote author before emitting any Signal quote metadata, track inherited-vs-explicit reply consumption separately, and collect redacted live Signal device/account proof before merge.

## Reproduction Assessment

Yes for the review blockers by source inspection: PR head can emit quote-timestamp without quote-author, and explicit Signal reply targets can consume shared inherited reply state. I did not run a live Signal roundtrip.

## Solution Assessment

No. The implementation direction is reasonable, but this patch should not merge until it satisfies signal-cli's quote-author contract, preserves inherited reply state when explicit targets are used, and supplies live Signal proof.
