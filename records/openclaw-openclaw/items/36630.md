## Summary

Keep open. Current main still lacks native outbound Signal quote metadata and full inbound quote IDs, so the work is not obsolete, but the PR head still has blocking correctness issues and the supplied proof is source/test terminal output rather than a live Signal send/receive roundtrip.

## What This Changes

The PR adds Signal outbound quote metadata, inbound quoted-message context extraction, reply-delivery plumbing, shared reply payload typing changes, and regression tests across Signal and outbound delivery paths.

## Best Possible Solution

Revise the branch so Signal formatted/direct/media/debounce/recovery paths preserve or intentionally suppress quote metadata consistently, validate quote IDs/authors against signal-cli’s contract, rebase over current main, and add redacted live Signal proof before merge.

## Reproduction Assessment

Yes, at source level. Current main lacks Signal quote RPC metadata and full inbound quote ids, and PR head e3740e0 still has clear source-level failures in formatted sends, debounced quote state, quote-id validation, blank reply IDs, and queued recovery replay.

## Solution Assessment

No. The implementation direction is reasonable, but this patch is not yet the narrow maintainable fix because several changed paths still bypass or corrupt quote metadata and native Signal behavior remains unproven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Forward quote overrides through formatted Signal sends:** `extensions/signal/src/channel.ts:191-197`
  - body: `deliverOutboundPayloadsCore` passes reply overrides into `sendFormattedText`, but Signal's formatted sender calls `send` without `replyTo` or `quoteAuthor`. Text-only formatted Signal replies still post as top-level messages instead of native quote replies.
  - confidence: 0.92
- **[P2] Use the latest debounced message's quote state:** `extensions/signal/src/monitor/event-handler.ts:508-518`
  - body: This searches backward for any quoted entry and can attach an older quote to the merged batch even when the latest debounced message is plain text. A quoted message followed by a quick plain follow-up is surfaced with stale `ReplyTo*` context.
  - confidence: 0.86
- **[P2] Require decimal Signal quote IDs:** `extensions/signal/src/monitor/quote-context.ts:67-68`
  - body: `Number(trimmed)` accepts non-decimal strings like `1e3` or `0x10`, normalizing malformed quote IDs into different timestamps. Inbound quote normalization should require decimal digits before surfacing `ReplyToId` or doing author-cache lookup.
  - confidence: 0.84
- **[P2] Treat blank Signal reply IDs as unset:** `extensions/signal/src/monitor/reply-delivery.ts:12-13`
  - body: Whitespace-only `replyToId` values become `null`, but this helper uses `null` as explicit reply suppression. A blank payload reply id therefore drops the inherited Signal quote instead of falling through as unset.
  - confidence: 0.79
- **[P2] Persist quoteAuthor in queued deliveries:** `src/infra/outbound/deliver.ts:855-871`
  - body: The live send path carries `quoteAuthor`, but the write-ahead queue persists only `replyToId` and recovery replays that truncated payload. Recovered Signal group replies lose the required author and are resent unquoted.
  - confidence: 0.82
