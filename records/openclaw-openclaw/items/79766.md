Keep open: current main lacks the requested container-root hint and this PR targets the right sandbox resolver, but the proposed message should not regress root-path shortening and the external PR still needs real behavior proof before merge.

## What This Changes

The PR changes the sandbox fs path resolver to append the container workdir hint to out-of-root errors and adds a unit test for that message.

## Best Possible Solution

Keep the change in the sandbox fs resolver, append the container workdir hint while preserving the existing shortened root wording, and verify it with a real sandboxed write/edit failure plus the focused unit test.

## Reproduction Assessment

Yes, from source inspection: an absolute path like /tmp/outside misses both container and host mounts, then current main throws the sandbox-root escape without defaultContainerRoot. I did not run a live sandbox command because this review must keep the checkout read-only.

## Solution Assessment

No, not as written: adding the hint at resolveSandboxFsPathWithMounts is the narrow maintainable direction, but the implementation should preserve the existing shortened root text and still needs real behavior proof.
