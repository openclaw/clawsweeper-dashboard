## Summary

Keep open: the PR addresses a real premature-completion wording bug, but the current branch still over-applies parent-review wording to direct external task delivery and removes unrelated codex-native task-registry regression coverage.

## What This Changes

The branch changes background task and subagent completion copy, docs, changelog, and tests to describe successful delegated completions as ready for parent review instead of done or ready for user delivery.

## Best Possible Solution

Keep review-first wording for parent-queued handoffs, avoid parent-review promises in direct external task messages, and preserve the codex-native task-registry guardrails.

## Reproduction Assessment

Yes. Current main has a source-level reproduction through `src/agents/subagent-announce.ts`, `src/tasks/task-executor-policy.ts`, `src/tasks/task-registry.ts`, docs, and tests that still emit or assert the finality wording.

## Solution Assessment

No, not yet. The subagent handoff direction is useful, but the shared task terminal formatter also feeds direct `sendMessage` delivery, so the PR needs a narrower delivery-context wording boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid promising parent review in direct task messages:** `src/tasks/task-executor-policy.ts:41-43`
  - body: `formatTaskTerminalMessage` feeds both queued parent-session events and direct external `sendMessage` delivery. With this shared text, direct chat recipients can receive `Next: parent will review/verify...` even though direct delivery bypasses parent-session routing, so the message promises a parent review that will not occur.
  - confidence: 0.89
- **[P3] Restore the codex-native task registry coverage:** `src/tasks/task-registry.test.ts:1779`
  - body: The PR deletes codex-native maintenance and cancellation regression tests while changing wording elsewhere in the same file. Those tests still cover live contracts for native child tasks with no OpenClaw child session, so dropping them is unrelated churn and weakens the task-registry guardrail.
  - confidence: 0.78
