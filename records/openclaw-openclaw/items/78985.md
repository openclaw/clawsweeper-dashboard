## Summary

Keep open. Current main still has the finality wording this PR targets, and the PR proof covers the queued task path, but the patch appends an internal parent-review next step to the shared task terminal formatter that is also sent directly to external chats; that needs a narrow fix before merge.

## What This Changes

The PR changes successful delegated/background task and subagent completion handoffs from final delivery wording to parent-review and review/verify wording, with matching tests, docs, and changelog copy.

## Best Possible Solution

Keep the review-first behavior, but split or neutralize successful task terminal wording so queued parent-session events can ask for review while direct external chat delivery does not promise a parent review that will not happen.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: subagent announce instructions, task terminal formatting, tests, and docs still assert final delivery wording such as ready for user delivery, completed successfully, and Background task done.

## Solution Assessment

No, not quite. The central source-level fix is in the right place, but the shared task terminal formatter is used for both parent-queued internal events and direct user-visible sendMessage delivery, so the parent-review next step needs to be scoped or reworded.

## Review Findings
