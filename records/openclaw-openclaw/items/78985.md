## Summary

Keep this PR open: the review-first wording is addressing a real source-visible bug, but the current patch still applies parent-review wording through a shared task terminal formatter that is also sent directly to external chats.

## What This Changes

The PR changes delegated/background task and subagent completion wording from final delivery language to parent-review language, with matching docs, changelog, and tests.

## Best Possible Solution

Keep the review-first behavior for parent-queued handoffs, but split or neutralize successful task terminal wording so direct external chat delivery does not promise a parent review that will not occur.

## Reproduction Assessment

Yes, as a source reproduction. Current main still contains `ready for user delivery`, `completed successfully`, and `Background task done` wording in the affected agents/tasks/docs paths.

## Solution Assessment

No, not yet. The subagent handoff direction is useful, but applying parent-review wording in the shared task terminal formatter also changes direct `sendMessage` delivery, so the wording boundary needs one more narrow repair.
