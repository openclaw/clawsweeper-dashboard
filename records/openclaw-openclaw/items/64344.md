## Summary

Keep open: the PR addresses a real secrets-audit UX bug, but the current diff can downgrade config-owned plaintext env values to migration guidance, and the external contributor proof gate is still unmet.

## What This Changes

The PR adds a `MIGRATION_RECOMMENDED` secrets-audit finding/count, CLI/docs output, and regression tests for exact `${ENV_VAR}` strings on SecretRef-capable config fields.

## Best Possible Solution

Keep the UX improvement, but classify exact env templates as migration guidance only after proving the resolved value is not sourced from plaintext stored in OpenClaw config, then add regression coverage and redacted CLI proof.

Yes. ## Solution Assessment

No. Comparing only the authored target with the resolved target is not source-aware enough because it cannot distinguish shell or dotenv env values from plaintext values injected by OpenClaw config.
