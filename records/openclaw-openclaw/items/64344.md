## Summary

## What This Changes

The PR adds `MIGRATION_RECOMMENDED` secrets-audit reporting, CLI/docs summary output, and regression tests for exact `${ENV_VAR}` interpolation on SecretRef-capable config fields.

## Best Possible Solution

Emit migration guidance only when audit can distinguish external env or dotenv values from plaintext config-owned env sources, while preserving plaintext findings and adding real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current config reads apply `config.env` before substitution, and the PR's authored-ref branch skips `PLAINTEXT_FOUND` after the target has resolved to a plaintext value.

## Solution Assessment

No. Comparing only the authored target string with the resolved target is not source-aware enough; the safer fix must distinguish shell/dotenv env values from plaintext values injected by OpenClaw config.
