## Summary

Keep open: the focused audit UX fix is useful, and the author narrowed the branch after earlier review, but the current patch can still hide plaintext stored in `config.env` / `env.vars` and lacks required after-fix real behavior proof.

## What This Changes

The branch adds a `MIGRATION_RECOMMENDED` secrets-audit code/count, CLI/docs output, and regression tests for exact `${ENV_VAR}` strings on SecretRef-capable config fields.

## Best Possible Solution

Classify exact authored env templates only when audit can prove the resolved value did not come from plaintext stored in OpenClaw config, then align the behavior with the related env-ref audit work and include redacted CLI proof.

## Reproduction Assessment

Yes. Source inspection shows `config.env` is applied before `${VAR}` substitution, and the PR then classifies the authored exact template as `MIGRATION_RECOMMENDED` even when the resolved value came from config-owned plaintext.

## Solution Assessment

No. Comparing only the authored target with the resolved target is not source-aware enough; the safer fix must distinguish shell/dotenv env values from plaintext values injected by OpenClaw config.

## Review Findings

Overall correctness: patch is incorrect
