## Summary

Keep open: the PR addresses real secrets-audit noise, but the current patch can downgrade the only plaintext finding when an exact env template is resolved from plaintext stored in config `env`/`env.vars`, and contributor real behavior proof is still missing.

## What This Changes

This PR adds a `MIGRATION_RECOMMENDED` secrets-audit finding/count, CLI/docs output, and regression tests for exact `${ENV_VAR}` strings on SecretRef-capable config fields.

## Best Possible Solution

Keep the UX improvement, but classify exact env templates as migration guidance only after proving the resolved value is not sourced from plaintext stored in OpenClaw config, then add regression coverage and redacted CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: config `env.vars.OPENAI_API_KEY` can populate `models.providers.openai.apiKey: "${OPENAI_API_KEY}"` before substitution, and this PR branch would emit `MIGRATION_RECOMMENDED` while no top-level config env plaintext audit catches the stored secret.

## Solution Assessment

No. Comparing only the authored target with the resolved target is not source-aware enough because it cannot distinguish shell/dotenv env values from plaintext values injected by OpenClaw config.
