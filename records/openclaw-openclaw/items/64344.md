## Summary

Keep open: the useful audit UX fix is not on current main, but this PR still has a concrete security-signal defect and is missing real behavior proof. The branch can downgrade plaintext stored in OpenClaw `config.env`/`env.vars` to migration guidance, so it needs source-aware attribution before merge.

## What This Changes

The PR adds `MIGRATION_RECOMMENDED` secrets-audit reporting for exact `${ENV_VAR}` templates on SecretRef-capable config fields, updates CLI/docs summary output, and adds regression tests.

## Best Possible Solution

Emit migration guidance only when the audit can distinguish external env or dotenv values from plaintext config-owned env sources, while preserving plaintext findings and requiring real CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current config reads apply `config.env` before substitution, and the PR's authored-ref branch skips `PLAINTEXT_FOUND` after the target has resolved to a plaintext value.

## Solution Assessment

No. Comparing only the authored target string with the resolved target is not source-aware enough; the safer fix must distinguish shell/dotenv env values from plaintext values injected by OpenClaw config.
