## Summary

Keep this PR open, but not merge-ready: it addresses a real secrets-audit false-positive class, yet the current diff can suppress a plaintext-at-rest finding for secrets supplied from `openclaw.json` `env`/`env.vars`, and the contributor has not posted real after-fix CLI proof.

## What This Changes

The PR adds a `MIGRATION_RECOMMENDED` secrets-audit finding for exact `${ENV_VAR}` values on SecretRef-capable config fields, exposes a summary count in CLI/JSON output, updates docs, and adds regression tests.

## Best Possible Solution

Make secrets audit source-aware: emit migration guidance only for externally supplied env substitutions, preserve plaintext findings for config-owned env values, and land with redacted real CLI output plus focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.env` is applied before `${VAR}` substitution, and the PR's authored-ref branch would skip the plaintext finding after the target resolves to a secret value.

## Solution Assessment

No. Comparing only the authored target string with the resolved target is not source-aware enough; the fix should distinguish external env sources from plaintext stored in OpenClaw config.
