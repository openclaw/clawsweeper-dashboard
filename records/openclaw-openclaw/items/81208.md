## Summary

Keep this PR open because it targets a real Mantle discovery noise problem, but the current patch would regress documented AWS default credential-chain discovery by treating missing env markers as missing credentials.

## What This Changes

The PR adds a credential-presence preflight to amazon-bedrock-mantle implicit discovery, exports the discovery config type, and adds regression tests plus a changelog entry.

## Best Possible Solution

Preserve the explicit discovery.enabled=false escape hatch and suppress no-credential log spam through a narrower mechanism, such as throttling or caching the failed no-auth discovery path, while retaining default-chain IAM discovery.

## Reproduction Assessment

Yes. Source inspection shows the current no-auth path reaches IAM token generation, and the proposed branch would skip that path whenever there is no explicit env marker even if a valid instance, task-role, web-identity, SSO, or shared-config credential source exists.

## Solution Assessment

No. The env-marker preflight is not the best fix because the documented AWS SDK credential chain can succeed without those markers; the safer fix should reduce repeated unavailable-token logging without precluding default-chain discovery.
