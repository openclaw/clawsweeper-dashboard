## Summary

Keep this PR open. Current main and v2026.6.9 still have the default Mantle IAM-token failure path without per-region log dedupe, and the PR head is a focused IAM-chain-preserving fix, but it still needs current-head real behavior proof and one stronger regression assertion before merge.

## What This Changes

The PR adds per-region dedupe for the Bedrock Mantle IAM-token generation-unavailable diagnostic, clears the marker after successful token generation/reset, exports MantleDiscoveryConfig, and adds regression coverage.

## Best Possible Solution

Land the provider-local log-dedupe fix after the PR body proof matches the current head and the regression test asserts the diagnostic is emitted at most once per region.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.9 still call IAM token generation in the default Mantle discovery path and log the reported diagnostic on failure when discovery is unset and no explicit bearer token is available.

## Solution Assessment

Yes, with the proof and test gaps fixed. Per-region dedupe at generateBearerTokenFromIam is the narrowest maintainable fix because the AWS token generator uses the default credential chain, so skipping IAM generation based only on env markers would be too narrow.
