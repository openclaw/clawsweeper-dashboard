## What This Changes

The PR changes Bedrock Mantle discovery to dedupe failed IAM bearer-token diagnostics per region, clear that dedupe after a successful token, export the discovery config type, and add regression tests.

## Best Possible Solution

Land the IAM-chain-preserving per-region log dedupe after the test directly asserts the emitted diagnostic count and the PR body shows current-head real behavior proof.

## Reproduction Assessment

Yes. At source level, current main falls through from `resolveImplicitMantleProvider` to `generateBearerTokenFromIam` without an explicit bearer token, and the catch path logs the reported diagnostic whenever token generation fails.

## Solution Assessment

Yes, the current patch direction is the best small fix because it removes repeated diagnostic spam without breaking AWS default-credential-chain users. It is not merge-ready until current-head proof and a direct log-count assertion are added.
