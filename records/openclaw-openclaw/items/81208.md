## Summary

Keep open. Current main and v2026.6.9 still have the default Bedrock Mantle path that attempts IAM token generation and can log the reported diagnostic on each failed discovery refresh; this PR is the active fix candidate, but the PR body proof is stale and the new regression test does not yet assert the emitted diagnostic count.

## What This Changes

The PR changes `amazon-bedrock-mantle` discovery to dedupe IAM bearer-token failure logging per region, clear the dedupe after success, export the discovery config type, and add focused discovery tests.

## Best Possible Solution

Land the provider-local log-dedupe fix after the PR body proof matches the current head and the regression test directly asserts the diagnostic is emitted at most once per region.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.9 shows the default Mantle discovery path still calls IAM token generation and logs the reported diagnostic on failure when discovery is unset.

## Solution Assessment

Yes, with the proof and test gaps fixed. The per-region dedupe is the best small shape because the dependency uses AWS default credential-chain sources, so skipping IAM generation based only on env markers would be too narrow.
