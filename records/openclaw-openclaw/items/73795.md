## Summary

Keep open. Current main still has a real config/delivery gap for this report: the shared streaming contract can represent `streaming.preview.toolProgress`, but WhatsApp and LINE do not expose that nested config and their dispatch paths do not pass the existing default-progress suppression option.

## What This Changes

The issue asks for `streaming.preview.toolProgress` to persist and apply for channels that do not define `streaming.mode`.

## Best Possible Solution

Add a shared delivery config path that preserves and applies `preview.toolProgress` independently of `streaming.mode`, then wire affected channels with focused schema, dispatch, docs, and regression-test coverage.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: WhatsApp and LINE cannot currently express the nested key, and their dispatch paths do not pass the suppression option that the shared dispatcher already supports.

## Solution Assessment

No complete solution exists on current main. The safer fix is to preserve the config key through relevant channel schemas and apply suppression without forcing every affected channel into preview streaming mode.
