## Summary

Keep open. Current main still has a real config-contract and delivery gap: the shared streaming contract supports `streaming.preview.toolProgress`, but WhatsApp and LINE do not expose that nested config and their dispatch paths do not pass the existing suppression option.

## What This Changes

Preserve and apply `channels.<channel>.streaming.preview.toolProgress: false` on non-preview channels so default tool/progress details are not delivered to end users.

## Best Possible Solution

Add a shared channel delivery config path that preserves `preview.toolProgress` independently of `streaming.mode`, then wire affected non-preview channels to suppress default progress while keeping approvals, media, errors, and final replies deliverable.

## Reproduction Assessment

Yes for the source-level reproduction: WhatsApp and LINE cannot currently express the documented nested key and do not pass the suppression option. A live transport reproduction is still useful to cover the exact user-visible leakage text per channel.

## Solution Assessment

No current solution is complete. The narrow maintainable fix is to preserve and apply the opt-out setting without forcing every affected channel into preview streaming mode.
