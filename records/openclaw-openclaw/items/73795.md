## Summary

Keep open. Current main still cannot express or apply the requested tool-progress suppression for WhatsApp and LINE, and the remaining fix needs maintainer direction because it changes privacy-sensitive cross-channel delivery behavior.

## What This Changes

The issue asks for `streaming.preview.toolProgress: false` to survive config normalization and suppress tool-progress output on channels without a configured `streaming.mode`.

## Best Possible Solution

Add a shared config and delivery contract that lets affected non-preview channels suppress default tool/progress messages without forcing preview streaming, then wire channel schemas, runtime options, docs, and regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: WhatsApp and LINE do not accept `streaming.preview.toolProgress`, and their reply paths do not pass the existing suppression option. I did not live-run the auto-reload rewrite path.

## Solution Assessment

No complete solution exists on current main. The safest direction is a generic suppression contract that preserves the config key and applies it independently of preview streaming mode.
