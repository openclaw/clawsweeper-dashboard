## Summary

Keep open. Current main still does not provide a config-preserving and config-applying path for `streaming.preview.toolProgress` across the non-preview channels named in the report, and the remaining decision is cross-channel privacy/config policy rather than cleanup closure.

## What This Changes

The issue requests that `streaming.preview.toolProgress: false` be preserved and respected for channels even when `streaming.mode` is absent.

## Best Possible Solution

Preserve `streaming.preview.toolProgress` through the relevant channel schemas and apply it as a progress-suppression control independent of `streaming.mode`, without forcing unsupported channels into preview streaming.

## Reproduction Assessment

Yes, for source-level reproduction: channel validation is schema-driven, WhatsApp and LINE do not expose the nested streaming object, and LINE does not pass the existing suppression option. I did not mutate a config file to live-run the watcher path.

## Solution Assessment

No complete solution exists on current main. The safest direction is to add a shared suppression/config contract and wire affected bundled channels with focused schema, runtime, docs, and regression coverage.
