## Summary

Keep open. Current main still has the reported config-contract and delivery gap for non-preview channels: the shared contract supports `streaming.preview.toolProgress`, but affected channel schemas and dispatch paths do not preserve and apply that opt-out consistently.

## What This Changes

The issue requests preserving and honoring `channels.<id>.streaming.preview.toolProgress: false` for channels that do not declare `streaming.mode`.

## Best Possible Solution

Preserve `streaming.preview.toolProgress` independently of `streaming.mode`, then wire affected channel dispatchers to suppress default tool-progress while preserving approvals, media, errors, and final replies.

## Reproduction Assessment

Yes for a source-level reproduction: current validation replaces channel config from strict schemas, WhatsApp and LINE do not expose the nested key, and neither dispatch path passes the existing suppression flag. A live channel reproduction would still be useful for exact recipient-facing text.

## Solution Assessment

No current solution is complete. The narrow maintainable fix is to preserve and apply the opt-out setting through the shared channel delivery contract rather than forcing non-preview channels into `partial` streaming mode.
