## Summary

Keep open. Current main still does not preserve or apply the documented `streaming.preview.toolProgress: false` contract for at least WhatsApp and LINE, and the linked fixing pull request is closed without merging.

## What This Changes

The issue asks OpenClaw to preserve `streaming.preview.toolProgress: false` for channel configs even when `streaming.mode` is absent, so tool-progress details can be hidden on affected channels.

## Best Possible Solution

Land an equivalent replacement for https://github.com/openclaw/openclaw/pull/77813 that adds shared streaming schema support to affected channel configs, wires runtime suppression, updates generated metadata/API baseline, and adds focused regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current WhatsApp and LINE schemas cannot express the nested config, validation replaces channel config with schema output, and current runtime paths do not pass the suppression option. I did not live-run config auto-reload or channel delivery.

## Solution Assessment

Yes. The narrow maintainable direction is to restore or replace the closed fixing PR's shape: preserve the shared config in channel schemas and apply suppression in the owning runtime paths with tests.
