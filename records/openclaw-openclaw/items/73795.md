## Summary

Keep open. Current main still does not provide a supported, preserved, and honored `streaming.preview.toolProgress: false` path for non-preview bundled channels such as WhatsApp and LINE, while the generic dispatcher only suppresses default tool/progress text when a channel explicitly passes the suppression option.

## What This Changes

Preserve and honor `channels.<channel>.streaming.preview.toolProgress: false` for channels that do not already declare `streaming.mode`, so default tool/progress narration is not shown to end users.

## Best Possible Solution

Add a shared, channel-safe config contract that preserves `streaming.preview.toolProgress` independently of `streaming.mode` for bundled delivery-only channels, wire WhatsApp and LINE dispatch to suppress default text-only tool/progress messages when it is false, and cover root plus per-account config with focused schema and dispatch tests. Do not enable preview streaming on channels that only support buffered/final delivery unless that is handled as a separate product change.

## Reproduction Assessment

Yes. A source-level reproduction is available: current WhatsApp and LINE config schemas do not expose `streaming.preview.toolProgress`, and their dispatchers do not pass the generic suppression option, so the reported setting is not a supported or honored current-main path.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix if scoped to preserving and honoring the suppression key without forcing `streaming.mode`. Defaulting every such channel to `partial` would be riskier because several affected channels do not implement preview-edit streaming.
