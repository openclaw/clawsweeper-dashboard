## Summary

Keep open. Current main still has the reported config-contract and delivery gap: the shared streaming contract can represent `streaming.preview.toolProgress`, but affected non-preview channels such as WhatsApp and LINE still do not expose or apply that setting.

## What This Changes

The issue asks for `streaming.preview.toolProgress: false` to be preserved and respected for channels even when `streaming.mode` is absent.

## Best Possible Solution

Preserve and apply `preview.toolProgress` through a shared channel delivery config path, then wire affected channel plugins to suppress default progress without forcing preview streaming mode on.

## Reproduction Assessment

Yes for a source-level reproduction: WhatsApp and LINE cannot currently express the nested key in their schemas and do not pass the existing suppression option to shared dispatch. A live channel run would strengthen the exact end-user leakage proof.

## Solution Assessment

No current solution is complete. The narrow maintainable fix is to make the opt-out a supported channel delivery setting and apply it in affected plugins while keeping approvals, media, errors, and final replies deliverable.
