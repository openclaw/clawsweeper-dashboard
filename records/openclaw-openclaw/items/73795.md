## Summary

Keep open. Current main still does not provide a supported way for non-preview channels such as WhatsApp and LINE to preserve and apply `streaming.preview.toolProgress: false`, while the shared dispatcher only suppresses generic tool/progress delivery when a channel explicitly passes the suppression option.

## What This Changes

The issue asks OpenClaw to preserve and apply `streaming.preview.toolProgress: false` for channels that do not declare `streaming.mode`.

## Best Possible Solution

Define a shared channel suppression contract, then wire affected bundled channels through schema, runtime delivery, docs, and focused regression coverage without forcing preview streaming onto channels that do not support it.

## Reproduction Assessment

Yes for source-level reproduction: WhatsApp and LINE still cannot express the requested config, and LINE does not pass the existing suppression option into reply dispatch. The exact auto-reload rewrite was not live-run in this read-only review.

## Solution Assessment

No complete solution exists on current main. The safest direction is a maintainer-approved shared suppression contract that preserves the config and applies it consistently, rather than ad hoc per-channel filtering.
