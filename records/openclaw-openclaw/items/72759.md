## Summary

Keep this PR open, but do not merge it as-is. The patch uses a valid resolver mode, but it changes a user-visible Telegram default while current tests, docs, and config UI still publish `partial` as the unset behavior, so this needs explicit maintainer direction and contract updates.

## What This Changes

The PR changes Telegram preview streaming’s unset fallback in `extensions/telegram/src/preview-streaming.ts` from `partial` to `off`, making unset Telegram preview delivery final-only.

## Best Possible Solution

Maintainers should choose the Telegram default explicitly. If final-only-by-default is accepted, land the fallback change with resolver tests, Telegram docs, config UI hints, and a user-facing changelog entry; otherwise replace this with a narrower partial-mode fix for the specific ghost/tool-progress behavior.
