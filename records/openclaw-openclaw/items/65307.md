## Summary

Keep open. Current main still documents and implements host exec `security="full"` plus `ask="off"` as no-approval YOLO mode, and I found no hard approval classifier for package-manager self-removal such as `npm uninstall -g openclaw`; adjacent #54416 is related but does not fully supersede this self-uninstall case.

## What This Changes

The issue asks OpenClaw to avoid confusing `opencli` with OpenClaw and to require explicit confirmation before package-manager commands can uninstall OpenClaw itself.

## Best Possible Solution

Keep this open until OpenClaw either ships a shared hard approval boundary for high-risk OpenClaw self-management commands or explicitly documents that trusted-operator YOLO mode permits package-manager self-removal.

## Reproduction Assessment

Partially yes: source inspection gives a high-confidence policy reproduction that `security="full"` plus `ask="off"` does not prompt for `npm uninstall -g openclaw`, though I did not run the destructive command. The package-name misinterpretation itself remains model/provider-specific.

## Solution Assessment

Unclear until maintainers decide the YOLO policy boundary. The safer maintainable direction is a shared gateway/node exec rule for OpenClaw self-uninstall and package-manager removal commands, with tests and docs for any intentional exception.
