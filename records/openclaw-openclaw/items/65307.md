## Summary

Keep open. Current main still documents and implements trusted host exec `security=full` plus `ask=off` as no-prompt YOLO mode, and I found no source-level hard approval boundary for package-manager removal of OpenClaw itself; related #54416 is adjacent but does not supersede this self-uninstall case.

## What This Changes

Request to prevent agents from conflating `opencli` with OpenClaw and to require explicit confirmation before executing OpenClaw self-uninstall or similarly destructive package-manager commands.

## Best Possible Solution

Ship a shared exec approval boundary for OpenClaw self-management and package-manager self-removal commands, or explicitly document that trusted full/off mode permits this class of self-removal.

## Reproduction Assessment

Yes for the approval-policy half: source inspection shows `security=full` plus `ask=off` does not prompt on gateway exec and bypasses node approval preparation. Unclear for the natural-language package-name confusion, which remains model/provider-specific, and I did not run a destructive uninstall command.

## Solution Assessment

Unclear until maintainers decide the YOLO policy boundary. The safer maintainable direction is a generic high-risk self-management command classifier enforced before both gateway and node full/off execution paths, with focused tests and docs for any intentional exception.
