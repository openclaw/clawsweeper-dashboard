## Summary

Keep open for maintainer review: current main still has the stale ACP active-run reverse lookup path, and this PR fixes it at the store owner boundary with focused regression coverage and sufficient terminal proof.

## What This Changes

The PR changes the ACP in-memory session store so rebinding a session to a new active run deletes the old run-id reverse lookup and adds a regression test.

## Best Possible Solution

Land the invariant fix in the in-memory session store after ordinary maintainer review, rather than teaching callers to distrust stale reverse lookups.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create one in-memory ACP session, bind `run-old`, then bind `run-new`, and `run-old` remains in `runIdToSessionId` because `setActiveRun` does not delete the previous active id.

## Solution Assessment

Yes. The session store owns both `activeRunId` and the reverse lookup, so deleting the previous mapping during `setActiveRun` is narrower and cleaner than making ACP translator callers filter stale lookup results.
