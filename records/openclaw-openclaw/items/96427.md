## Best Possible Solution

Yes. On current main, bind a store session to run-old and then run-new; getSessionByRunId("run-old") still returns the session because setActiveRun does not delete the prior reverse-map entry.

## Solution Assessment

Yes. Fixing the reverse-map invariant inside createInMemorySessionStore is the right owner boundary; caller-only cleanup would leave the exported store API internally inconsistent.
