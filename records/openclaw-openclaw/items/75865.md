## Summary

Close as implemented on main: current main addresses the reported v2026.4.29 Telegram/json5 startup failure by making json5 a root runtime dependency and by removing the bundled plugin node_modules/runtime-deps staging path that could overwrite local patches. The proof is on current main only; I could not prove it shipped in the latest tagged release.

## What This Changes

The issue asks OpenClaw to fix Telegram startup on v2026.4.29 by avoiding bundled bare json5 import failures and stopping plugin runtime-deps staging from clobbering local install patches.

## Best Possible Solution

Keep the current main approach: bundled Telegram should load through canonical packaged dist with root package dependencies, while old plugin-runtime-deps trees remain legacy cleanup handled by doctor/update paths. The remaining product need is to ship this in the next release so v2026.4.29 users no longer need local workarounds.

## Reproduction Assessment

Yes for the reported release: the issue gives a concrete v2026.4.29 Telegram startup path and current tag inspection shows that release still had the old runtime node_modules linking path. For current main, the review is source/test based rather than a live Telegram run because this was a read-only checkout without installed dependencies.

## Solution Assessment

Yes: the current-main fix is narrower and more maintainable than hardcoding json5/lib/index.js in generated chunks because it removes the stale staged dependency root from bundled plugin loading and lets normal package-root dependency resolution handle json5.
