## Summary

The report is valid and current: the recommended Windows Hub links all return 404 because stable release `v2026.6.6` lacks the promoted installer assets. The repository already has a workflow that supplies these exact assets, so release-operator follow-up is still necessary.

## What This Changes

Restore working Windows Hub installer downloads from the official Windows documentation page.

## Best Possible Solution

Promote the existing signed Windows Hub artifacts onto `v2026.6.6`, verify the three durable documentation URLs, and then add a release gate that prevents future stable releases from completing without these assets.

## Reproduction Assessment

Yes. Requesting any of the three URLs in `docs/platforms/windows.md` redirects to stable release `v2026.6.6` and returns HTTP 404 because the named assets are absent.

## Solution Assessment

No, changing the links to ad hoc or versioned companion-repository URLs is not the best fix. The existing signed-artifact promotion workflow should restore the durable stable URLs, with a release check added separately to prevent recurrence.
