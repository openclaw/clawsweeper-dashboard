## What This Changes

The PR changes the subagent registry sweeper so run-mode `cleanup: "keep"` entries without `archiveAtMs` are not pruned by the session TTL, and adds a regression test.

## Best Possible Solution

Land this narrow sweeper guard after contributor real-behavior proof and normal maintainer gates, while keeping the initial durable-registration fix in https://github.com/openclaw/openclaw/pull/83146 as separate companion work.

## Reproduction Assessment

Yes, source-level: current main registers run-mode keep entries with no `archiveAtMs`, then the sweeper's no-archive TTL branch can delete them after `cleanupCompletedAt` ages past five minutes. I did not run the full embedded-channel scenario.

## Solution Assessment

Yes for the code shape: gating the session TTL prune away from run-mode keep entries is the narrow fix for the conflated `archiveAtMs === undefined` cases. The merge path still needs real behavior proof from an actual OpenClaw setup.
