## Summary

Keep open. Current main still rejects arbitrary extra bootstrap basenames, so this PR is not obsolete, but the branch needs a narrow repair before merge: the glob detector still misses bracket-only Node glob patterns. There is also overlapping open work in #74735, so maintainers should reconcile the chosen product boundary before landing one version.

## What This Changes

The PR removes the fixed basename allowlist from the bundled `bootstrap-extra-files` loader, widens extra bootstrap file names to `string`, updates hook docs and changelog text, and adds tests for arbitrary workspace-contained extra files.

## Best Possible Solution

Land one coherent `bootstrap-extra-files` update that supports the intended arbitrary workspace-contained files, preserves workspace containment and realpath guards, keeps bootstrap budgets and subagent filtering, handles Node glob syntax including bracket expressions, keeps deterministic and intentional ordering, and updates docs, changelog credit, and focused loader/hook tests.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring `bootstrap-extra-files.paths` with an arbitrary file such as `hooks/REGISTRY.yaml`, which source inspection shows is rejected as `invalid-bootstrap-filename`; the PR-specific edge case is reproducible with a bracket-only glob such as `src/agents/[wx]orkspace.ts`.

## Solution Assessment

No, not as currently written. Removing the basename allowlist is a plausible narrow solution for #39120, but the branch should recognize `[` as glob syntax and then be reconciled with the overlapping #74735 design before merge.
