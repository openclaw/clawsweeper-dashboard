## Summary

Keep open. Current main still has the fixed bootstrap basename allowlist, and this PR is useful carry-forward work for #39120/#39174, but the proposed patch has a concrete P2 glob-detection bug and overlaps with open PR #74735, which is pursuing a different session-scoped boundary.

## What This Changes

The PR removes the `bootstrap-extra-files` basename allowlist, widens extra bootstrap file names to strings, updates hook docs/changelog copy, and adds tests for arbitrary workspace-contained extra bootstrap files.

## Best Possible Solution

Land one coherent `bootstrap-extra-files` change that chooses the intended product boundary, preserves workspace containment, realpath and size guards, configured ordering, Node glob semantics including bracket expressions, subagent filtering, docs, changelog credit, and focused loader/hook tests.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring `bootstrap-extra-files` with an arbitrary file such as `hooks/REGISTRY.yaml`, which source inspection shows is rejected as `invalid-bootstrap-filename`; the PR-specific bug is reproducible by a bracket-only Node glob such as `src/agents/[wx]orkspace.ts`.

## Solution Assessment

No, not as currently written. Removing the basename allowlist is a plausible narrow solution for #39120, but this PR should classify `[` as glob syntax and maintainers should reconcile it with #74735 before merge.
