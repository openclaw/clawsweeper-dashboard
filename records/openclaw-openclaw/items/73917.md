## Summary

Keep open. Current main still rejects arbitrary `bootstrap-extra-files` basenames, so this PR is addressing a real open feature request, but the patch has a blocking P2 glob-detection bug: bracket-only Node glob patterns are valid and would be treated as literal missing files. The overlapping open #74735 also needs maintainer reconciliation before either bootstrap contract lands.

## What This Changes

The PR removes the fixed recognized-bootstrap-basename allowlist for `bootstrap-extra-files`, widens bootstrap file names to `string`, updates hook docs and changelog copy, and adds tests for arbitrary workspace-contained extra bootstrap files.

## Best Possible Solution

Land one coherent `bootstrap-extra-files` implementation that chooses the intended product boundary, preserves workspace containment and size guards, supports Node glob syntax including bracket expressions, keeps deterministic prompt ordering, updates docs/changelog, and has focused loader and hook tests.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring `bootstrap-extra-files.paths` with an arbitrary file such as `hooks/REGISTRY.yaml`, which source inspection shows is rejected as `invalid-bootstrap-filename`; the PR-specific bug is reproducible with a bracket-only Node glob such as `src/agents/[w]orkspace.ts`.

## Solution Assessment

No, not as currently written. Removing the basename allowlist is a plausible narrow solution for #39120, but this PR should classify `[` as glob syntax and maintainers should reconcile it with #74735 before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Treat bracket-only patterns as globs:** `src/agents/workspace.ts:186-188`
  - body: `hasGlobPattern()` does not check for `[`, so a valid Node glob such as `docs/[ab].md` is treated as a literal path unless it also contains `*`, `?`, or `{`. That silently skips bracket-only glob patterns while this PR is broadening extra bootstrap file loading.
  - confidence: 0.92
