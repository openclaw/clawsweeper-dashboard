## Summary

Keep open. The PR addresses a still-valid `bootstrap-extra-files` limitation on current `main`, but the patch misses bracket-only Node glob patterns and is currently dirty against the base, with overlapping product-boundary work still open in #74735.

## What This Changes

The branch removes the fixed bootstrap-basename allowlist for `bootstrap-extra-files`, widens bootstrap file names to `string`, updates docs/changelog text, and adds focused tests for arbitrary workspace-contained extra files.

## Best Possible Solution

Land one coherent `bootstrap-extra-files` implementation that chooses the intended boundary, preserves workspace containment and size guards, supports Node glob syntax including bracket expressions, keeps deterministic prompt ordering, updates docs/changelog, and has focused loader and hook tests.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring `bootstrap-extra-files.paths` with an arbitrary file such as `hooks/REGISTRY.yaml`, which source inspection shows is rejected as `invalid-bootstrap-filename`; the PR-specific bug is reproducible with a bracket-only Node glob such as `src/agents/[w]orkspace.ts`.

## Solution Assessment

No, not as currently written. Removing the basename allowlist is a plausible narrow solution for #39120, but this PR should classify `[` as glob syntax and be reconciled with #74735 before merge.

## Review Findings
