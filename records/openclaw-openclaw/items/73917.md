## Summary

Keep open: current main still has the basename allowlist, and this branch addresses the canonical request, but it still misses bracket-only Node glob patterns and overlaps with open #74735 before merge.

## What This Changes

This PR broadens `bootstrap-extra-files` to load arbitrary workspace-contained files, updates docs/changelog, and adds focused loader/hook tests while preserving guards and session filtering.

## Best Possible Solution

Land one coherent `bootstrap-extra-files` implementation that supports arbitrary workspace-contained files, preserves containment and budget guards, supports Node glob syntax including bracket expressions, keeps deterministic ordering, and settles the global versus session-scoped API boundary.

## Reproduction Assessment

Yes. Current main rejects an arbitrary `bootstrap-extra-files.paths` entry such as `hooks/REGISTRY.yaml` via `invalid-bootstrap-filename`, and the PR-specific bug is reproducible with a bracket-only glob such as `src/agents/[w]orkspace.ts`.

## Solution Assessment

No, not as currently written. Removing the basename allowlist is a plausible narrow fix for #39120, but this PR should classify `[` as glob syntax and be reconciled with #74735 before merge.
