## Summary

Keep open. Current main still rejects arbitrary `bootstrap-extra-files` basenames, and this PR addresses the canonical user problem, but the current head has two concrete correctness issues and is dirty against base; the newer session-scoped PR is related but not merged and does not make this branch implemented on main.

## What This Changes

The branch removes the fixed `bootstrap-extra-files` basename allowlist, widens bootstrap file names to `string`, updates docs/changelog text, and adds focused tests for arbitrary workspace-contained extra files.

## Best Possible Solution

Land one coherent `bootstrap-extra-files` contract that preserves workspace containment, size budgets, subagent filtering, full Node glob syntax, deterministic but config-order-respecting output, and matching docs/changelog/tests.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring `bootstrap-extra-files.paths` with an arbitrary file such as `hooks/REGISTRY.yaml`, and the PR-specific bugs are reproducible with a bracket-only glob like `src/agents/[w]orkspace.ts` and an explicit literal list whose order matters.

## Solution Assessment

No, not as currently written. Removing the basename allowlist is a plausible narrow fix for #39120, but the PR should classify `[` as glob syntax, preserve configured path order by sorting only per-glob expansions, and be reconciled with #74735 before merge.

Overall correctness: patch is incorrect

- **[P2] Treat bracket-only patterns as globs:** `src/agents/workspace.ts:186-188`
  - body: `hasGlobPattern()` misses `[`, so valid Node glob patterns such as `docs/[ab]/PROJECT.md` are treated as literal paths unless they also contain `*`, `?`, or `{`. That silently skips bracket-only configured extras while this PR is broadening file loading.
  - confidence: 0.92

  - body: Sorting the entire resolved path set reorders explicit literal entries and can change prompt context precedence chosen by the operator. Keep configured pattern/literal order and sort only each glob expansion for deterministic results.
  - confidence: 0.86
