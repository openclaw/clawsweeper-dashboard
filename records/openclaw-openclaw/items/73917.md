## Summary

Keep open. The PR addresses a real remaining bootstrap-extra-files gap on current main, but the proposed patch still misclassifies bracket-only Node glob patterns and overlaps with the open session-scoped approach in #74735, so it should not be merged as-is.

## What This Changes

The PR removes the fixed bootstrap-extra-files basename allowlist, widens workspace bootstrap file names to strings, and updates docs, changelog, and focused tests for arbitrary workspace-contained extra bootstrap files.

## Best Possible Solution

Land one coherent bootstrap-extra-files implementation that chooses the intended product boundary, preserves workspace containment and size guards, supports Node glob syntax including bracket expressions, keeps deterministic prompt ordering, updates docs and changelog, and has focused loader and hook tests.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring bootstrap-extra-files.paths with an arbitrary file such as hooks/REGISTRY.yaml and observing invalid-bootstrap-filename; the PR-specific bug is reproducible with a bracket-only Node glob such as src/agents/[w]orkspace.ts.

## Solution Assessment

No, not as currently written. Removing the basename allowlist is a plausible narrow fix for #39120, but this PR should classify [ as glob syntax and be reconciled with #74735 before merge.
