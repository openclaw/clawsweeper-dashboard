## Best Possible Solution

Land the focused pure-PowerShell probe only after redacted Windows Task Scheduler proof shows no `cmd`/`findstr` window remains and the Gateway restarts successfully.

## Reproduction Assessment

## Solution Assessment

Yes, the focused code surface is the best path: it preserves the retry loop and race guard while removing `findstr`, and the sibling update restart helper already uses `Get-ScheduledTask` without `findstr`. It still needs live Windows proof before merge.
