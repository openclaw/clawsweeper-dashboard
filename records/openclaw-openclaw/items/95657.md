## Summary

Keep open. Current main and v2026.6.9 still expose every Markdown file under the workspace memory directory and bridge-import all public artifacts without excluding the wiki vault's own sources; an open candidate PR now targets this issue, so the issue should wait for review and merge rather than be closed.

## What This Changes

The issue reports that memory-wiki bridge mode can recursively re-import generated wiki source pages when the wiki vault lives under workspace memory.

## Best Possible Solution

Review and land a narrow bridge-side self-source filter with regression coverage, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source inspection. Current main recursively exposes Markdown under workspace memory and bridge-imports those artifacts without excluding the wiki vault's own sources directory; I did not run a live multi-week reproduction in this read-only review.

## Solution Assessment

Yes. The best fix is a narrow bridge-side guard that skips artifacts originating from the active wiki vault's sources directory; adding a broad new excludePaths config surface would be less aligned with the repository's config-surface bar.
