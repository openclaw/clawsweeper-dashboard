## Repair Work Prompt

Fix the memory-wiki bridge self-import loop reported at https://github.com/openclaw/openclaw/issues/95657. Current source inspection shows `workspace/memory` recursive discovery and public artifact export can include `memory/wiki/sources/*.md`, and `syncMemoryWikiBridgeSources` imports those artifacts back into the same vault. Implement a narrow default guard that prevents generated wiki vault source pages from being indexed/exported/re-imported as memory bridge artifacts when the wiki vault lives under the agent workspace `memory/` tree. Prefer filtering at the canonical memory/public-artifact boundary and/or bridge self-vault boundary over adding `memorySearch.excludePaths`; do not add a new config option unless maintainers explicitly choose that direction. Add regression tests showing a vault at `workspace/memory/wiki` with `sources/bridge-*.md` does not produce public daily-note artifacts for those source pages and that bridge import skips self-vault source artifacts while still importing ordinary memory daily notes and legitimate other-workspace artifacts. Update docs only if the fix changes recommended vault placement; do not edit CHANGELOG.md.
