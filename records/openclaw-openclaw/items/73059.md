## Repair Work Prompt

Fix the Windows linked-source Control UI wrapper bug reported at https://github.com/openclaw/openclaw/issues/73059. Current `main` compares `path.resolve(process.argv[1])` to `fileURLToPath(import.meta.url)` in `scripts/ui.js`, so a global npm junction launch can skip `main()` and make `node ...\scripts\ui.js build` a no-op. Keep the repair scoped to `scripts/ui.js`, `test/scripts/ui.test.ts`, and the required user-facing `CHANGELOG.md` entry unless a shared helper proves necessary. Prefer a small helper that compares both paths by `fs.realpathSync.native` or equivalent, falling back to `path.resolve` when realpath fails. Add a focused regression test that simulates a linked/junctioned entry path resolving to the current module and proves direct execution would run. Preserve the existing Windows shell launcher quoting and unsafe-argument rejection behavior. Add a single-line changelog fix entry with `Thanks @LouisGameDev`. Treat PR #52291 as adjacent context only; do not pull in unrelated `pnpm.cmd` quoting or UI dependency-install changes.
