## Summary

Current main still matches the reported failure mechanism: `scripts/ui.js` uses a direct path-string execution guard, and the gateway auto-build still depends on that wrapper and can surface the missing `index.html` symptom. The adjacent Windows UI PR was closed unmerged and covered `pnpm.cmd` quoting/dependency install behavior, not direct `node ...\scripts\ui.js` execution through a junction.

## What This Changes

The issue requests a Windows fix so linked source installs launched through the global npm junction run the Control UI build instead of silently skipping `main()`.

## Best Possible Solution

Normalize the `scripts/ui.js` direct-execution decision through canonical realpath comparison with a conservative resolved-path fallback, then add focused regression coverage that simulates a linked or junctioned entry path. Keep the change scoped to the wrapper unless maintainers intentionally add a shared script-entry guard helper.

## Reproduction Assessment

Yes. The issue gives concrete Windows `pnpm link --global` junction steps, and current main still has the path-string guard that can differ between the junction launch path and the real module URL.

## Solution Assessment

Yes. A narrow `scripts/ui.js` guard normalization plus `test/scripts/ui.test.ts` coverage is the maintainable fix; changing only the gateway auto-build caller would leave direct `node ...\scripts\ui.js build` broken.

## Repair Work Prompt

Fix the Windows linked-source Control UI build no-op reported in https://github.com/openclaw/openclaw/issues/73059. Keep the change scoped to scripts/ui.js unless a tiny shared helper is clearly warranted. The observable bug is that node %APPDATA%\npm\node_modules\openclaw\scripts\ui.js build can skip main() when the global npm path is a Windows junction to the repo root, because the direct-execution guard compares the resolved argv path string to fileURLToPath(import.meta.url). Normalize the guard with realpath/canonical path comparison and a safe resolved-path fallback so direct script execution still works when the entry path is a junction/symlink. Add focused regression coverage in test/scripts/ui.test.ts that simulates a linked or junctioned argv path causing main to run. Do not change unrelated Control UI build behavior, gateway policy, package manager selection, installer behavior, or dependency installation semantics. Add a CHANGELOG.md fix entry with proper non-bot attribution because this is a user-facing fix.
