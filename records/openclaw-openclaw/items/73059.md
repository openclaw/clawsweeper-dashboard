## Summary

Keep open. Current main and the latest release still use the raw path-string direct-execution guard in `scripts/ui.js`, while gateway first-start Control UI auto-build still invokes that wrapper, so the Windows junction-launched repro remains source-backed and unfixed.

## What This Changes

The issue reports that Windows source installs launched through a global npm junction can no-op `scripts/ui.js build`, leaving first-start Control UI auto-build without `dist/control-ui/index.html`.

## Best Possible Solution

Update the `scripts/ui.js` direct-execution check to canonicalize `argv[1]` and the current module path with a resolved-path fallback, then cover the linked-path case in the script tests.

## Reproduction Assessment

Yes. The reporter supplied concrete Windows `pnpm link --global` junction steps, and current main still contains the exact raw path comparison that can skip `main()` when `argv[1]` and `import.meta.url` spell the same file differently.

## Solution Assessment

Yes. Fixing `scripts/ui.js` is the narrowest maintainable fix because changing only the gateway caller would still leave direct `node ...\scripts\ui.js build` broken.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/73059. Update `scripts/ui.js` so the direct-execution guard treats junctioned or symlinked launch paths as the same script: canonicalize both `process.argv[1]` and `fileURLToPath(import.meta.url)` with `fs.realpathSync.native`, falling back to `path.resolve` when realpath fails. Keep the existing Windows shell launcher quoting and unsafe-argument checks intact, and do not change gateway auto-build policy beyond making the wrapper execute correctly. Add a focused regression test in `test/scripts/ui.test.ts` for direct execution when `argv[1]` resolves through a linked/junction-style path; prefer a helper that can be tested without running a real UI build. Add a user-facing `CHANGELOG.md` fix entry because this affects first-start/source-install behavior. Related context: #52291 was adjacent but not a canonical fix.
