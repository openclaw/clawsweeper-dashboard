## Summary

Keep open. Current main still has the unresolved `scripts/ui.js` direct-execution guard that can skip `main()` for a Windows junction launch path, and the gateway auto-build path can still surface the reported missing-index message after a zero-exit no-op build.

## What This Changes

The issue asks for Windows linked source installs launched through the global npm junction to run the same Control UI build path as repo-local `scripts/ui.js` invocations.

## Best Possible Solution

Normalize the `scripts/ui.js` direct-execution check by comparing canonical realpaths on both sides, with a resolved-path fallback for realpath failures, and add a focused regression test that proves a linked entry path still invokes `main()`. Keep the change scoped to the wrapper and test unless the test exposes another boundary.

## Reproduction Assessment

Yes. The issue gives concrete Windows `pnpm link --global` junction steps, and current main still has the exact unresolved path comparison that would skip `main()` when the launch path and module path differ.

## Solution Assessment

Yes. A narrow direct-execution guard normalization plus `test/scripts/ui.test.ts` coverage is the maintainable fix; changing only the gateway auto-build caller would leave direct `node ...\scripts\ui.js build` behavior broken.

## Repair Work Prompt

Fix the Windows linked source install bug from https://github.com/openclaw/openclaw/issues/73059. Current `scripts/ui.js` decides direct execution with `path.resolve(process.argv[1]) === fileURLToPath(import.meta.url)`, which can skip `main()` when launched through the global npm junction. Add a small helper that compares canonical realpaths for both the argv entry and current module path, with a path.resolve fallback when realpath fails, then use it for `isDirectExecution`. Add focused regression coverage in `test/scripts/ui.test.ts` for a linked or symlinked entry path that resolves to the current module and still triggers the direct-execution decision; keep the test deterministic and avoid requiring privileged Windows-only symlink setup unless guarded. Do not change Control UI app code, package-manager resolution, or package metadata unless the regression proves it is necessary. Add a single-line `CHANGELOG.md` fix entry for the active release section crediting `Thanks @LouisGameDev`. Related context: PR #52291 is adjacent Windows `pnpm.cmd` quoting work and was closed unmerged; do not treat it as the same fix.
