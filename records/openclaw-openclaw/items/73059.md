Keep open. Current main still has the path-string direct-execution guard in `scripts/ui.js`, the gateway auto-build path still relies on that wrapper, the latest release still contains the same guard, and the related Windows UI PR was adjacent but closed unmerged.

## What This Changes

The issue asks for Windows linked source installs to run `scripts/ui.js build` correctly through the global npm junction so missing Control UI assets are actually rebuilt.

## Best Possible Solution

Normalize the `scripts/ui.js` direct-execution guard with canonical realpath comparison and a resolved-path fallback, then add focused regression coverage for a linked or junctioned entry path.

## Reproduction Assessment

Yes. The reporter supplied concrete Windows `pnpm link --global` junction steps, and current main still contains the direct string comparison that can skip `main()` when `argv[1]` is the junction path.

## Solution Assessment

Yes. A narrow guard normalization in `scripts/ui.js` plus focused `test/scripts/ui.test.ts` coverage is the maintainable fix; changing only the gateway caller would leave direct `node ...\scripts\ui.js build` broken.
