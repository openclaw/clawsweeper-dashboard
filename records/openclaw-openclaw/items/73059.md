## Summary

Keep open. Current main still uses a raw `argv[1]` versus module-path comparison in `scripts/ui.js`, while gateway first-start Control UI asset building still invokes that wrapper, so the Windows junction-launched no-op path remains source-backed.

## What This Changes

Reported bug where Windows global-linked source installs can skip the Control UI build because `scripts/ui.js` does not treat a junction path as direct execution.

## Best Possible Solution

Canonicalize both the current module path and `argv[1]` in `scripts/ui.js`, add a linked-entrypoint regression test, and preserve the existing Windows cmd.exe launcher safeguards.

## Reproduction Assessment

Yes, source-backed rather than live-run in this review. The reporter supplied concrete Windows junction steps, and current main still has the raw comparison that would skip `main()` when the junction path and real repo path differ.

## Solution Assessment

Yes. Fixing the direct-execution guard in `scripts/ui.js` is the narrowest maintainable solution because both direct `node ...\scripts\ui.js build` usage and gateway first-start auto-build route through that wrapper.
