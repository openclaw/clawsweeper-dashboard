## Repair Work Prompt

Fix the Windows linked-source Control UI build bug reported in openclaw/openclaw#73059. Current `scripts/ui.js` can skip `main()` when launched through the global npm junction because the direct-execution guard compares resolved path strings instead of canonical realpaths. Update `scripts/ui.js` only as needed to canonicalize the current file and `argv[1]` with native realpath plus resolved-path fallback, or reuse the existing `isMainModule` pattern if that does not break script/package runtime assumptions. Keep the Windows shell launcher quoting and unsafe-argument behavior unchanged. Add focused regression coverage in `test/scripts/ui.test.ts` for a linked or junctioned `argv[1]` direct-execution path. Add a single-line user-facing `CHANGELOG.md` fix entry. Do not change Gateway asset root policy, plugin loading, or the separate `pnpm.cmd` quoting behavior from the closed adjacent PR. Validate with the focused script test, targeted oxfmt check, `git diff --check`, and the changed gate in Testbox when available.
