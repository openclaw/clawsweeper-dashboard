## Summary

Keep open. Current main still has the raw `process.argv[1]` versus `import.meta.url` path-string guard in `scripts/ui.js`, while the first-start Control UI builder can still invoke that wrapper from a root resolved through the launched entry path; the adjacent Windows `pnpm.cmd` work does not cover this junction-launched direct-execution failure.

## What This Changes

The issue asks for Windows linked source installs to run the Control UI build normally when `scripts/ui.js` is launched through the global npm junction path.

## Best Possible Solution

Canonicalize `scripts/ui.js` direct-execution detection for linked entrypoints, add focused regression coverage for a junction or symlinked `argv[1]`, and keep the existing Windows shell-safety behavior unchanged.

## Reproduction Assessment

Yes, source-backed. The reporter supplied concrete Windows junction steps, and current main still compares non-canonical entrypoint strings in `scripts/ui.js`, which can skip `main()` when the same file is reached through a junction path.

## Solution Assessment

Yes. Fixing the `scripts/ui.js` guard is the narrowest maintainable solution because changing only the gateway caller would leave direct `node ...\scripts\ui.js build` invocations broken.

## Repair Work Prompt

Fix the Windows linked-source Control UI build wrapper bug reported in https://github.com/openclaw/openclaw/issues/73059. In current main, `scripts/ui.js` decides whether to call `main()` by comparing `path.resolve(process.argv[1])` to `fileURLToPath(import.meta.url)`, so `node %APPDATA%\npm\node_modules\openclaw\scripts\ui.js build` can no-op when `%APPDATA%\npm\node_modules\openclaw` is a junction to the repo root. Update the direct-execution guard to compare canonical paths, using the same realpath-with-resolved-fallback semantics as `src/infra/is-main.ts` or an equivalent local helper that keeps `scripts/ui.js` runnable directly by Node. Add focused regression coverage in `test/scripts/ui.test.ts` for a linked/junctioned `argv[1]` path reaching the guard or exported predicate. Preserve the existing Windows shell launcher quoting and unsafe-argument protections. Add a user-facing `CHANGELOG.md` fix entry. Do not broaden gateway build policy or change unrelated UI dependency-install behavior.
