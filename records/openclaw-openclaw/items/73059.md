## Summary

Keep open. Current main still has the raw direct-execution guard described in the report, so a junction-launched Windows path can bypass `main()` and leave the gateway auto-build path reporting the missing Control UI index. The adjacent Windows UI build PR was closed unmerged and addressed a different `pnpm.cmd` quoting/dependency-install failure mode.

## What This Changes

The issue asks OpenClaw to make `scripts/ui.js build` execute correctly when launched through a Windows global npm junction from a linked source install.

## Best Possible Solution

Normalize the `scripts/ui.js` direct-execution check through realpath-on-both-sides comparison with a resolved-path fallback, then add a focused regression test for a linked/junction entry path. Keep the fix in the wrapper and test unless the regression proves a separate gateway boundary is involved.

## Reproduction Assessment

Yes. The issue provides concrete Windows `pnpm link --global` junction steps, and current main still has the exact unresolved path comparison that would skip `main()` when the launch path and module realpath differ.

## Solution Assessment

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/73059. On Windows linked source installs, `pnpm link --global` creates a global npm junction, and launching `node %APPDATA%\npm\node_modules\openclaw\scripts\ui.js build` can skip `main()` because `scripts/ui.js` compares `path.resolve(process.argv[1])` with `fileURLToPath(import.meta.url)`. Keep the fix scoped to `scripts/ui.js`: normalize both the entry path and module path with `fs.realpathSync.native` or equivalent, falling back to `path.resolve` when realpath fails. Add a focused regression test in `test/scripts/ui.test.ts`; extracting/exporting a small helper for direct-execution detection is fine if it keeps the test deterministic. Do not change Control UI app code, package manager selection, gateway auto-build policy, or package metadata unless the regression proves that boundary is involved. Add a user-facing `CHANGELOG.md` fix entry under the active version with `Thanks @LouisGameDev`.
