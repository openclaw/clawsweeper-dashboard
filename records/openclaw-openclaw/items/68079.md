Full review comments:

- **[P1] Preserve caller session headers before injecting:** `extensions/zai/stream.ts:40-42`
  - body: This merge writes `X-Session-Id` after spreading existing headers, so an explicit caller-supplied `X-Session-Id` is overwritten and a lowercase `x-session-id` becomes a duplicate that Fetch normalizes to `old, new`. Current request-header policy preserves caller overrides, so the wrapper should leave an existing session header alone or resolve it case-insensitively before adding a generated value.
  - confidence: 0.89
- **[P2] Reject Fetch-invalid session ids before setting the header:** `extensions/zai/stream.ts:18-21`
  - body: `isInjectableSessionId` only checks type, length, and trimmed emptiness, but `options.sessionId` can still contain header-invalid bytes such as CR, LF, or NUL. Once copied into `X-Session-Id`, Fetch/Undici throws before the ZAI request is sent, so the validator needs to reject values that are not valid HTTP header values.
  - confidence: 0.86
- **[P1] Refresh the wrapper on current GLM-5.2 reasoning behavior:** `extensions/zai/index.ts:137-154`
  - body: The PR head wraps the older ZAI thinking branch and GitHub reports the branch as conflicting, while current main now maps GLM-5.2 thinking levels to `reasoning_effort` with regression coverage. The conflict resolution must compose the new header wrapper around the current main stream function instead of dropping the GLM-5.2 `reasoning_effort` path.
  - confidence: 0.88

  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file and release-note context belongs in the PR body or squash message. Please drop this entry rather than carrying manual changelog churn in the feature branch.
