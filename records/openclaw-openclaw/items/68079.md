## Summary

Keep this PR open: the ZAI cache-stickiness idea has useful real-workload proof, but the current branch still has concrete merge blockers that would regress current ZAI stream behavior and add a stale Pi runtime dependency.

## What This Changes

The PR adds a ZAI provider stream wrapper that injects `X-Session-Id` from `options.sessionId`, plus tests, docs, changelog, and a plugin dependency/lockfile update.

## Best Possible Solution

Land a repaired provider-local ZAI wrapper that composes with current `wrapZaiStreamFn`, validates and canonicalizes `X-Session-Id`, uses current `@earendil-works/*` Pi imports, and keeps focused tests/docs aligned.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the branch would bypass current ZAI thinking payload handling, and a Node header probe shows duplicate-cased session headers merge and invalid control-byte values can fail. The requested cache optimization itself is supported by the PR's live benchmark rather than a current-main bug reproduction.

## Solution Assessment

No as currently implemented. A provider-local header wrapper is the right boundary, but this patch needs to compose with current ZAI stream behavior, sanitize and replace the header safely, and use the current Pi package namespace.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:279`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which preserves tool-stream defaults and patches disabled or preserved thinking payloads. Replacing that registration with this new wrapper drops those current behaviors, so ZAI thinking settings can regress after merge.
  - confidence: 0.92
- **[P2] Reject header-invalid session IDs:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, including NUL and other header-invalid control bytes. Once copied into `X-Session-Id`, those values can fail before a valid ZAI request is sent; omit invalid header values instead.
  - confidence: 0.88
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:39-42`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a prior `x-session-id` key in place. Case-insensitive header normalization combines them into values like `old, new`, which can break the sticky-routing key; remove any existing session header before setting the canonical one.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace:** `extensions/zai/package.json:8`
  - body: The new ZAI files import old `@mariozechner/*` packages and add `@mariozechner/pi-ai@0.69.0`, while current main uses `@earendil-works/*@0.74.0`. That installs a second stale provider runtime and can diverge transport behavior without a compatibility reason.
  - confidence: 0.9
