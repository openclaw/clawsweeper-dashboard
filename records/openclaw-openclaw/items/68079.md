Full review comments:

- **[P1] Preserve explicit session headers before injecting:** `extensions/zai/stream.ts:40-42`
  - body: This writes the generated `X-Session-Id` after spreading `options.headers`, so an operator or caller that already supplied this header cannot override it. OpenClaw's request-header contract is caller-wins for custom provider headers, so the wrapper should leave an explicit session header alone or merge through the shared case-insensitive header path.
  - confidence: 0.92
- **[P1] Avoid exposing raw session ids by default:** `extensions/zai/stream.ts:42`
  - body: This sends `options.sessionId` directly to ZAI as provider-visible metadata. The stream contract already separates transcript/session identity from `promptCacheKey`, so this needs an explicit maintainer decision or an opaque provider-scoped affinity key before it is safe to enable by default.
  - confidence: 0.86
- **[P2] Reject header-invalid session ids before injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string up to 256 characters, including values such as NUL-containing strings that Fetch rejects as invalid header values. Add a header-value validity guard before copying programmatic `sessionId` into request headers so malformed session ids do not fail ZAI requests locally.
  - confidence: 0.78
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:18`
  - body: OpenClaw's `CHANGELOG.md` is release-generated and should not be edited by normal PRs. Keep the release-note context in the PR body or final squash message instead of adding this active-release entry.
