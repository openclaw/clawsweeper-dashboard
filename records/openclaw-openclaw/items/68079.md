## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:40-43`
  - body: This merge keeps an existing `x-session-id` key and then adds `X-Session-Id`. HTTP header handling is case-insensitive, so downstream normalization can receive a combined or ambiguous value such as `old, new`, which defeats the sticky-routing guarantee or causes provider rejection. Remove any existing session-id header case-insensitively before setting the canonical key.
  - confidence: 0.9
- **[P2] Reject header-unsafe session ids:** `extensions/zai/stream.ts:14-22`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string up to the length cap, including CR/LF and other control characters. Those bytes are invalid in HTTP header values and can turn otherwise valid ZAI requests into transport errors when this wrapper tries to send them. Treat header-invalid values as non-injectable and add coverage for that path.
  - confidence: 0.92
- **[P3] Drop the normal changelog edit:** `CHANGELOG.md:21`
  - body: `CHANGELOG.md` is release-owned in this repository, and normal PRs should keep user-visible release-note context in the PR body or squash message instead. The PR body already has that context, so this changelog entry should not be merged with the feature branch.
  - confidence: 0.86
