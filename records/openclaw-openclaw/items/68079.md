Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Use the affinity key instead of raw session identity:** `extensions/zai/stream.ts:33-40`
  - body: `StreamOptions` already has `promptCacheKey` for provider prompt-cache affinity distinct from transcript/session identity, and stream-resolution preserves it separately. This wrapper ignores that key and always sends `options.sessionId`, so cron/shared runs with a stable cache key still expose the raw session id and lose the intended affinity value. Use the explicit affinity key when present, or pause for a maintainer decision if ZAI must receive raw session ids.
  - confidence: 0.87
- **[P2] Preserve existing X-Session-Id headers case-insensitively:** `extensions/zai/stream.ts:39-40`
  - body: The wrapper spreads `options.headers` and then adds `X-Session-Id`, so a caller-provided `x-session-id` remains as a second case-variant header. Header normalization combines those into values like `old, new`, which can break the exact sticky-routing key this PR is trying to send; remove/replace existing `x-session-id` keys case-insensitively or make caller precedence explicit.
  - confidence: 0.88
- **[P3] Drop the release-owned changelog edit:** `CHANGELOG.md:18`
  - body: Root policy says `CHANGELOG.md` is release-owned and should not be edited for normal PRs. Keep this release-note context in the PR body or squash message instead of carrying a direct changelog hunk.
