## Summary

Keep open: the ZAI sticky-routing feature is not on current main and the PR includes useful live proof, but the current patch would regress covered ZAI thinking behavior and needs localized repairs before merge.

## What This Changes

The PR adds a ZAI stream wrapper, tests, docs, changelog, and package/lockfile changes to inject `X-Session-Id` from `options.sessionId` for prompt-cache stickiness.

## Best Possible Solution

Land a repaired provider-local ZAI implementation that composes with current `wrapZaiStreamFn`, validates and canonicalizes `X-Session-Id`, uses the current `@earendil-works/*` Pi runtime packages, and keeps focused tests/docs aligned.

## Reproduction Assessment

Yes for the review blockers: current-main source/tests show the ZAI thinking-wrapper contract, the PR diff replaces it, and a Node `Headers` probe reproduces invalid-value rejection plus duplicate-cased header merging. The cache improvement itself is supported by the contributor's live benchmark and inspected chart rather than a local live ZAI run.

## Solution Assessment

No as currently implemented. Provider-local header injection is the right boundary, but this patch needs to preserve the current ZAI wrapper, sanitize and replace the header safely, and avoid the stale Pi runtime namespace.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:279`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which applies tool_stream defaults and patches disabled or preserved thinking payloads. Replacing that registration with this new wrapper drops those covered behaviors, so ZAI thinking settings can regress after merge.
  - confidence: 0.93
- **[P2] Reject header-invalid session IDs:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, including CR, LF, and NUL. Once copied into `X-Session-Id`, those values throw during Fetch header normalization before a valid ZAI request is sent.
  - confidence: 0.9
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:39-42`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a prior `x-session-id` key in place. Header normalization combines the two values, such as `old, new`, which can break the sticky-routing key.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace:** `extensions/zai/package.json:8`
  - body: The new ZAI files import old `@mariozechner/*` packages and add `@mariozechner/pi-ai@0.69.0`, while current main uses `@earendil-works/*@0.74.0`. That installs a second stale provider runtime and can diverge transport behavior without a compatibility reason.
  - confidence: 0.9
