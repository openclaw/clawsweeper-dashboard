## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject header-invalid session IDs before injecting:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, so values containing CR/LF/NUL or non-ByteString Unicode can be copied into `X-Session-Id`. When the request layer normalizes headers, those values throw before the ZAI request is sent; reject or encode them and add regression coverage.
  - confidence: 0.9
- **[P2] Replace existing x-session-id before setting X-Session-Id:** `extensions/zai/stream.ts:42`
  - body: The spread keeps an existing `x-session-id` header and then adds `X-Session-Id`, leaving two casings for the same HTTP header. Header normalization combines them into values such as `old, new`, which can break the sticky-routing key; remove any existing key case-insensitively before injecting.
  - confidence: 0.88
