- **[P1] Preserve caller-supplied session headers before injecting:** `extensions/zai/stream.ts:40-42`
  - body: This merge sets `X-Session-Id` after spreading `options.headers`, so an exact-case caller header is overwritten and a lower-case `x-session-id` coexists until Fetch normalizes it into a comma-joined value like `old, new`. That violates the current caller-wins request-header policy and can break ZAI sticky routing; skip injection when a case-insensitive session header is already present or otherwise preserve caller intent.
  - confidence: 0.92
- **[P2] Reject Fetch-invalid session ids before header injection:** `extensions/zai/stream.ts:18-21`
  - body: `isInjectableSessionId` accepts any non-empty string up to 256 characters, including values Fetch rejects as header values, such as strings containing NUL. A bad caller-supplied `sessionId` can therefore throw before the request is sent; validate against the runtime header-value contract and add regression coverage.
  - confidence: 0.84
