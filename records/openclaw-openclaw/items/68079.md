## Summary

Keep open. The feature is not on current main and belongs in the ZAI plugin, but this PR cannot merge as-is because it would drop current ZAI thinking behavior, accepts unsafe or duplicate session-header values, and adds an older Pi runtime dependency.

## What This Changes

The PR adds a ZAI stream wrapper that injects `X-Session-Id` from `options.sessionId`, updates provider registration, tests, docs, changelog, and declares a Pi runtime dependency.

## Best Possible Solution

Keep this feature ZAI-owned, but rebase and repair the branch so session-header injection composes with the current ZAI thinking/tool-stream wrapper, validates and canonicalizes header values, uses the current Pi dependency, and carries focused tests plus changelog credit.

## Reproduction Assessment

Yes. The review has high-confidence source and runtime reproduction paths: current-main tests cover the thinking behavior the PR drops, and Node `Headers` demonstrates the invalid-value and duplicate-casing header failures.

## Solution Assessment

No. A provider-local wrapper is the right boundary, but the PR implementation needs to preserve the existing ZAI wrapper and sanitize/replace the header before it is the best mergeable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:279`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which keeps `tool_stream` and patches `thinking` for disabled and preserved-thinking modes. Replacing it with this wrapper would regress that existing ZAI behavior after rebase.
  - confidence: 0.9
- **[P2] Reject header-invalid session IDs:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts non-empty strings containing CR, LF, or NUL, but Fetch/Undici rejects those as header values. The optional affinity header should be omitted for invalid values instead of failing the ZAI call.
  - confidence: 0.91
- **[P2] Replace session headers case-insensitively:** `extensions/zai/stream.ts:39-42`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a prior `x-session-id` in place. Header normalization combines those values into `old, new`, which can break sticky routing; remove any existing session header case-insensitively first.
  - confidence: 0.9
- **[P2] Use the repo-current Pi runtime version:** `extensions/zai/package.json:8`
  - body: The PR adds `@mariozechner/pi-ai@0.69.0` for ZAI while current main and existing provider plugins use `0.71.1`. That can install a second older provider runtime and diverge stream behavior without a stated need.
  - confidence: 0.88
- **[P3] Add the required changelog credit:** `CHANGELOG.md:9`
  - body: This is a user-facing feature entry from a contributor, so the changelog bullet should include the credited human GitHub username, for example `Thanks @Frrrrrrrrank`.
  - confidence: 0.86
