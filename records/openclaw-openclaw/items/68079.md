  - body: `isInjectableSessionId` accepts non-empty strings containing CR/LF/NUL, then copies them into `X-Session-Id`. Node header normalization rejects those values, so a caller-supplied session id with control characters can make an otherwise valid ZAI request fail before it is sent.
  - confidence: 0.92
- **[P2] Normalize existing X-Session-Id headers before merging:** `extensions/zai/stream.ts:40-42`
  - body: The wrapper spreads caller headers and then adds `X-Session-Id` without resolving an existing `x-session-id` key. Paths that hand this object directly to `Headers` can coalesce the old and new values as `old, new`, weakening stickiness or causing provider rejection; resolve the key case-insensitively or intentionally preserve a caller-wins policy.
  - confidence: 0.82
