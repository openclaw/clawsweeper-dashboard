## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the cache-affinity key instead of raw session identity:** `extensions/zai/stream.ts:34`
  - body: This wrapper injects `options.sessionId`, but current main has a separate `promptCacheKey` contract for provider cache affinity and cron tests rely on that key staying stable while session ids rotate. As written, those callers either miss sticky routing or send raw transcript/session identity upstream instead of the opaque affinity key.
  - confidence: 0.9
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:40-42`
  - body: Spreading `options.headers` before adding `X-Session-Id` leaves a caller-supplied `x-session-id` key in place. Fetch/Undici normalizes header names case-insensitively and combines the values, so ZAI can receive `old, new` instead of one sticky-routing key.
  - confidence: 0.93
- **[P2] Reject header-invalid session IDs before injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, including header-invalid control bytes such as NUL. Once copied into `X-Session-Id`, Node `Headers` throws before the request is sent, so a caller-provided session id can break otherwise valid ZAI calls.
  - confidence: 0.88
