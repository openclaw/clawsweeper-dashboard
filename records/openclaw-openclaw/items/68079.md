Full review comments:

- **[P2] Use the plugin SDK stream facades:** `extensions/zai/stream.ts:1-2`
  - body: This new provider file imports `StreamFn` and `streamSimple` from `@mariozechner/*`, then adds a direct ZAI dependency on `@mariozechner/pi-ai`. Bundled plugin runtime code should use the OpenClaw SDK facades instead; the existing xAI wrapper imports `StreamFn` from `openclaw/plugin-sdk/agent-core` and `streamSimple` from `openclaw/plugin-sdk/llm`, which avoids a new package/lockfile surface and keeps bundled providers on the supported SDK boundary.
  - confidence: 0.9
- **[P2] Reject header-invalid session ids:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` currently accepts any non-empty trimmed string, so a value with embedded CR/LF is copied into `X-Session-Id`. Node's header normalization rejects actual CR/LF in header values, turning a ZAI request that previously ignored that session id into an immediate transport error; reject header-invalid bytes before injecting.
  - confidence: 0.88
- **[P2] Replace existing x-session-id case-insensitively:** `extensions/zai/stream.ts:40-42`
  - body: Spreading `options.headers` and then adding `X-Session-Id` leaves an existing `x-session-id` key in place. HTTP header normalization is case-insensitive and combines those values as `old, new`, which can break the sticky-routing key the PR is trying to provide; remove any existing casing before setting the canonical header.
  - confidence: 0.91
