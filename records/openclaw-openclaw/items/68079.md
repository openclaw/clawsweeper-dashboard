  - body: This writes the raw OpenClaw `sessionId` into a new third-party provider header. Callers can choose meaningful session IDs, and OpenClaw already has a separate prompt-cache affinity option, so this should use an approved opaque affinity key before becoming default ZAI request metadata.

  - body: `isInjectableSessionId` accepts non-empty strings containing control characters, then the wrapper copies them into `X-Session-Id`. Fetch/Undici rejects values such as `abc\nxyz`, so a caller-provided session ID can make an otherwise valid ZAI request fail before it is sent.

  - body: The merge can leave both `x-session-id` and `X-Session-Id` in `options.headers`. Header normalization is case-insensitive and combines those into values like `old, new`, which can break sticky routing or cause provider-side rejection.
  - confidence: 0.93
