- **[P1] Use the affinity key instead of raw session identity:** `extensions/zai/stream.ts:34-42`
  - body: This wrapper injects `options.sessionId`, but current main has a separate `promptCacheKey` contract for provider cache affinity and cron tests rely on that key staying stable while session ids rotate. As written, those callers either miss sticky routing or send raw transcript/session identity upstream instead of the approved affinity key.
  - confidence: 0.9
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:40-42`
  - body: Spreading `options.headers` before adding `X-Session-Id` leaves a caller-supplied `x-session-id` key in place. Fetch/Undici normalizes header names case-insensitively and combines the values, so ZAI can receive `old, new` instead of one sticky-routing key.
  - confidence: 0.93
- **[P2] Reject header-invalid session ids before injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, including CR/LF or other header-invalid control characters. Once copied into `X-Session-Id`, Node `Headers` throws before the request is sent, so a caller-provided session id can break otherwise valid ZAI calls.
  - confidence: 0.86
- **[P3] Drop the release-owned changelog edit:** `CHANGELOG.md:21`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation derives entries from merged PRs and commits. Keep the release-note context in the PR body or squash message and remove this direct changelog change.
