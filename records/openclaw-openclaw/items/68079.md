## Summary

This PR should stay open, but it is not mergeable as-is: the current branch still bypasses current ZAI thinking behavior, has unsafe session-header handling, and adds an older Pi runtime namespace.

## What This Changes

The branch adds a ZAI stream wrapper that injects `X-Session-Id` from `options.sessionId`, plus tests, a plugin dependency, docs, changelog, and lockfile updates.

## Best Possible Solution

Land a repaired provider-local wrapper that composes with current `wrapZaiStreamFn`, sanitizes and canonicalizes `X-Session-Id`, uses current Pi package imports, and keeps focused ZAI tests/docs aligned.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the branch bypasses current ZAI thinking payload handling, and a Node header probe shows invalid header values and case-duplicate headers fail or merge. The requested optimization itself is supported by the PR's live benchmark rather than a current-main bug reproduction.

## Solution Assessment

No as currently implemented. A provider-local wrapper is the right boundary, but it needs to compose with current ZAI stream behavior, sanitize and replace the session header safely, and align with current Pi package names.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:279`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which preserves tool-stream defaults and patches disabled or preserved thinking payloads. Replacing that registration with this new wrapper drops those current behaviors, so ZAI thinking settings can regress after merge.
  - confidence: 0.92
- **[P2] Reject header-invalid session IDs:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts non-empty strings containing CR, LF, NUL, or other header-invalid control bytes. Once copied into `X-Session-Id`, Node HTTP validation can throw and fail an otherwise valid ZAI request; omit invalid values instead.
  - confidence: 0.92
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:39-42`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a prior `x-session-id` key in place. Case-insensitive header normalization combines them into values like `old, new`, which breaks the sticky-routing key; remove any existing session header before setting the canonical one.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace:** `extensions/zai/package.json:8`
  - body: The new ZAI files import old `@mariozechner/*` packages and add `@mariozechner/pi-ai@0.69.0`, while current main uses `@earendil-works/*@0.74.0`. That installs a second stale provider runtime and can diverge transport behavior without a compatibility reason.
  - confidence: 0.9
