- **[P1] Forward sessionId before relying on the ZAI wrapper:** `extensions/zai/stream.ts:34-35`
  - body: `createZaiSessionIdHeaderWrapper` only injects when `options.sessionId` is present, but current `main` wraps provider-owned stream functions without passing `sessionId` on the normal `providerStreamFn` branch.   - confidence: 0.91
- **[P1] Avoid exposing raw session ids as the affinity header:** `extensions/zai/stream.ts:42`
  - body: This copies the raw OpenClaw session id into a provider-visible header. `StreamOptions` already distinguishes `sessionId` from `promptCacheKey`, and the PR proof uses a human-readable task string, so this can leak stable task/session metadata unless maintainers explicitly choose that contract or the code sends an opaque cache-affinity value instead.
  - confidence: 0.84
- **[P1] Preserve explicit session headers before injecting:** `extensions/zai/stream.ts:40-42`
  - body: The wrapper spreads caller headers and then overwrites `X-Session-Id`, while a lower-case `x-session-id` key remains alongside it. Existing OpenAI-compatible request shaping uses caller-wins precedence, and Fetch normalization combines duplicate casings into values like `old, new`, which can break sticky routing or provider validation.
  - confidence: 0.9
- **[P2] Reject Fetch-invalid session ids before header injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string up to 256 characters, including `\r`/`\n`. Node/Fetch rejects those as invalid header values, so a malformed programmatic session id would turn a ZAI request into a local transport error instead of cleanly skipping the optional affinity header.
  - confidence: 0.93
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:21`
  - body: `CHANGELOG.md` is release-owned in this repo. Keep the user-visible release-note context in the PR body or final squash/commit message instead of editing the changelog from a normal feature PR.
