## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve explicit X-Session-Id headers before injecting:** `extensions/zai/stream.ts:40-42`
  - body: The wrapper spreads `options.headers` and then always writes `X-Session-Id`, so a caller-provided same-case value is overwritten and a lower-case `x-session-id` can coexist. Fetch normalizes duplicate casings into a comma-joined value such as `old, new`, which can break the exact sticky key; normalize case variants and keep caller-provided values winning.
  - confidence: 0.91
- **[P2] Reject header-invalid session IDs before injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` only checks type, length, and nonblank content, so values with `\r`, `\n`, or characters Fetch cannot convert to a header ByteString can pass into `X-Session-Id` and throw before the request is sent. Reject or encode header-invalid values and add regression coverage.
  - confidence: 0.89
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:18`
  - body: This normal PR edits `CHANGELOG.md`, but repository policy keeps the changelog release-generated. Keep the release-note context in the PR body or squash message and leave `CHANGELOG.md` to release generation.
