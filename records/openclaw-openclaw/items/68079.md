## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the run session id into provider streams:** `extensions/zai/stream.ts:34`
  - body: `createZaiSessionIdHeaderWrapper` depends on `options.sessionId`, but the embedded-runner path that wraps provider streams currently does not propagate `params.sessionId` into those options. In normal ZAI agent calls the wrapper can therefore run without the value it needs, so the sticky header is not injected despite the PR's intended behavior.
  - confidence: 0.88
- **[P2] Preserve the current ZAI thinking wrapper:** `extensions/zai/index.ts:282`
  - body: Current main already routes ZAI through `wrapZaiStreamFn`, which preserves `tool_stream` and adds ZAI thinking disable/preserve payload behavior. Replacing the hook with this PR's older `wrapZaiProviderStream` shape during rebase would drop that behavior for GLM reasoning turns, so the affinity wrapper needs to compose with the current main wrapper instead of replacing it.
  - confidence: 0.9
- **[P2] Reject header-invalid session ids:** `extensions/zai/stream.ts:21`
  - body: The validator currently accepts non-empty strings containing control characters such as `\n`, `\r`, or NUL. Fetch/Undici rejects those as invalid header values, so a malformed caller-provided session id would turn an otherwise valid ZAI request into an immediate transport failure.
  - confidence: 0.94
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:39-42`
  - body: Spreading `options.headers` and then adding `X-Session-Id` leaves an existing `x-session-id` key in place. Header normalization treats those as the same header and can send a comma-joined value such as `old, new`, which defeats the sticky-routing key or risks provider rejection.
  - confidence: 0.91
- **[P2] Use the current Pi package namespace:** `extensions/zai/package.json:6-8`
  - body: Current main has moved Pi runtime imports to `@earendil-works/*` at 0.75.1, but this PR adds `@mariozechner/pi-ai` 0.69.0 to the ZAI plugin. Rebasing with that dependency would reintroduce a deprecated runtime namespace and diverge from the package graph used by the rest of the provider code.
  - confidence: 0.93
