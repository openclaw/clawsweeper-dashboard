## Summary

Keep open: the ZAI sticky-routing feature is useful and has credible live benchmark proof, but the current branch has merge-blocking regressions in ZAI stream composition, header safety, and dependency alignment.

## What This Changes

The PR adds a ZAI provider stream wrapper that injects an `X-Session-Id` header from the agent session id, plus tests, docs, changelog, and package metadata updates.

## Best Possible Solution

Keep this ZAI-owned sticky-routing feature, but repair the branch so header injection composes with current ZAI thinking/tool-stream handling, validates and canonicalizes the header, and uses the current `@earendil-works` Pi runtime packages.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the PR bypasses current ZAI thinking wrapping, and a Node `Headers` probe reproduces invalid-value rejection plus duplicate-casing combination. The feature itself is supported by the PR's live benchmark rather than a failing current-main bug reproduction.

## Solution Assessment

No. A provider-local wrapper is the right boundary, but this branch is not mergeable until it preserves the existing ZAI wrapper behavior, safely handles header values, and aligns with current Pi package naming and versions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:279`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which preserves tool-stream defaults and patches disabled/preserved thinking payloads. Replacing it with this new wrapper drops that behavior, so ZAI thinking settings can regress after merge.
  - confidence: 0.9
- **[P2] Reject header-invalid session ids:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts non-empty strings containing CR, LF, NUL, or other header-invalid bytes. Once copied into `X-Session-Id`, Fetch/Undici header normalization can throw before an otherwise valid ZAI request is sent.
  - confidence: 0.91
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:39-42`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a prior `x-session-id` key in place. Case-insensitive header normalization can combine them into `old, new`, breaking the sticky-routing key.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace and version:** `extensions/zai/package.json:8`
  - body: The PR adds `@mariozechner/pi-ai@0.69.0` while current main uses `@earendil-works/pi-ai@0.74.0`. That installs a second stale provider runtime and can diverge ZAI transport behavior without a compatibility reason.
  - confidence: 0.9
