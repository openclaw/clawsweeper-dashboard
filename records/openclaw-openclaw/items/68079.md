## Summary

Keep open. The provider-local direction and live benchmark proof are useful, but the branch is not mergeable while it bypasses current ZAI thinking handling, accepts header-invalid session ids, can duplicate differently cased session headers, and adds an older Pi runtime package.

## What This Changes

This PR adds a ZAI provider stream wrapper that injects `X-Session-Id` from stream options, tests it, declares a Pi runtime dependency, and updates provider docs/changelog text.

## Best Possible Solution

Repair the branch so session-header injection composes with current ZAI stream handling, validates and canonicalizes headers safely, and uses the current Pi package namespace/version with focused integration coverage.

## Reproduction Assessment

Yes for the review blockers: current-main source/tests show the existing ZAI wrapper behavior, the PR diff bypasses it, and a Node `Headers` probe reproduces invalid-value rejection plus duplicate-casing combination. The feature itself is supported by the PR's live benchmark screenshot rather than a failing current-main bug reproduction.

## Solution Assessment

No. A ZAI-owned wrapper is the right boundary, but this branch is not the best implementation until it preserves the current wrapper behavior, validates/replaces headers safely, and aligns with current Pi runtime packages.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:279`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which preserves tool-stream defaults and patches disabled/preserved thinking payloads. Replacing it with `wrapZaiProviderStream` composes only tool-stream plus the new header wrapper, so ZAI thinking settings can regress after rebase.
  - confidence: 0.92
- **[P2] Reject header-invalid session IDs:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts non-empty strings that can still be invalid as HTTP header values, such as NUL-containing session ids. Once copied into `X-Session-Id`, Fetch header normalization can throw and fail the request; omit invalid header values instead.
  - confidence: 0.9
- **[P2] Replace session headers case-insensitively:** `extensions/zai/stream.ts:39-42`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a prior `x-session-id` key in place. Case-insensitive header normalization collapses both into a comma-joined value like `old, new`, breaking the sticky-routing key; remove any existing session header before setting the canonical one.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace and version:** `extensions/zai/package.json:8`
  - body: The PR adds `@mariozechner/pi-ai@0.69.0` and imports old Pi package names while current main uses `@earendil-works/*@0.74.0`. That installs a second stale provider runtime and can diverge transport behavior without a compatibility reason.
  - confidence: 0.9
