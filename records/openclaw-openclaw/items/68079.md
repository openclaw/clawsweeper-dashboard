Overall confidence: 0.88

Full review comments:

- **[P2] Pass the run session id into provider streams:** `extensions/zai/stream.ts:32`
  - body: `createZaiSessionIdHeaderWrapper` reads `options.sessionId`, but current main does not pass the run `sessionId` into ordinary provider stream options for ZAI. As written, the new wrapper is a no-op on the normal path that this PR is trying to optimize; plumb a safe affinity key into the provider stream path or attach it through the provider transport state before relying on this header.
  - confidence: 0.88
- **[P2] Preserve the current ZAI thinking wrapper:** `extensions/zai/index.ts:279`
  - body: Current main’s ZAI `wrapStreamFn` also patches `payload.thinking` for disabled and preserve-thinking modes. Replacing the stream hook with this new wrapper during rebase would drop that current behavior, so the session header wrapper needs to compose with the existing ZAI wrapper instead of replacing it.
  - confidence: 0.86
- **[P2] Reject header-invalid session ids:** `extensions/zai/stream.ts:14-22`
  - body: `isInjectableSessionId` currently accepts any non-empty trimmed string up to 256 chars, including control characters and non-ByteString Unicode. Fetch/Undici rejects those header values before the request is sent, so a malformed session id can turn a valid ZAI call into a transport error.
  - confidence: 0.92
- **[P2] Replace existing x-session-id case-insensitively:** `extensions/zai/stream.ts:39-41`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a preexisting `x-session-id` key in place. Header normalization treats them as the same header and combines values like `old, new`, which can break sticky routing or trigger provider rejection.
  - confidence: 0.9
- **[P2] Use the current Pi runtime packages:** `extensions/zai/package.json:7-9`
  - body: The branch adds `@mariozechner/pi-ai` 0.69.0 while current main uses `@earendil-works/pi-ai` and `@earendil-works/pi-agent-core` 0.75.1. Pulling in the older namespace can duplicate the provider runtime contract and conflict with current SDK types.
  - confidence: 0.87
