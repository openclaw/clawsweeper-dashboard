Full review comments:

- **[P2] Reject header-invalid session ids before injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, including `\r`/`\n`, and the wrapper then copies that value into an HTTP header. Node's `Headers` rejects those values before the request is sent, so a malformed session id that previously passed through can now fail every ZAI call. Reject control characters and add a regression test.
  - confidence: 0.93
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:39-40`
  - body: Spreading `options.headers` and then adding `X-Session-Id` leaves an existing `x-session-id` key in place. Fetch header normalization treats those as the same header and coalesces them into `old, new`, which can break sticky routing or provider validation. Remove existing `x-session-id` entries case-insensitively before setting the canonical header.
  - confidence: 0.91

  - body: `CHANGELOG.md` is release-owned in this repository; normal feature PRs should keep release-note context in the PR body or squash message instead. This line will conflict with release generation and should be removed from the branch.
