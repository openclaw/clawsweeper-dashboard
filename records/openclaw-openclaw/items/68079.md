Full review comments:

- **[P2] Reject header-invalid session IDs before injection:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, so CR/LF/NUL and non-ByteString Unicode can be copied into `X-Session-Id`. `Headers` rejects those values before transport, turning an otherwise valid ZAI call into a local failure; reject header-invalid values and cover them in tests.
  - confidence: 0.94
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:40-42`
  - body: This merge preserves an existing `x-session-id` key while adding `X-Session-Id`. Header normalization treats those names as the same header and coalesces them into a value such as `old, new`, so remove any existing session-id header case-insensitively before injecting the ZAI affinity value.
  - confidence: 0.91
