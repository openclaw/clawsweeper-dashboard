## Summary

Keep open: this is a useful ZAI provider performance feature with inspected live proof, but the current PR head still has merge-blocking wrapper, header handling, and Pi package namespace defects.

## What This Changes

The PR adds a ZAI stream wrapper that injects `X-Session-Id` from the agent `sessionId`, plus stream tests, a plugin dependency, docs, changelog, and lockfile updates.

## Best Possible Solution

Land a repaired ZAI-local sticky-routing wrapper that composes with current ZAI thinking, replay, and tool-stream behavior, canonicalizes `X-Session-Id`, and stays on the current Pi namespace.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the PR bypasses current ZAI thinking behavior, and a Node header probe reproduces invalid-value and duplicate-casing header risks. The requested optimization itself is a provider-performance feature supported by the PR's inspected live benchmark, not a current-main bug reproduction.

## Solution Assessment

No as currently implemented. A provider-local wrapper is the right boundary, but this branch needs to preserve current ZAI stream/replay behavior, sanitize and replace the session header safely, and align imports with the current Pi runtime packages.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:282`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which preserves tool-stream defaults and patches disabled or preserved thinking payloads. Replacing that registration with this new wrapper drops those current behaviors, so ZAI thinking settings can regress after merge.
  - confidence: 0.92
- **[P2] Reject header-invalid session IDs:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts non-empty strings containing CR, LF, NUL, or other header-invalid control bytes. Once copied into `X-Session-Id`, header normalization can throw and fail an otherwise valid ZAI request; omit invalid values instead.
  - confidence: 0.92
- **[P2] Replace existing session headers case-insensitively:** `extensions/zai/stream.ts:40-42`
  - body: Spreading existing headers and then adding `X-Session-Id` leaves a prior `x-session-id` key in place. Case-insensitive header normalization combines them into values like `old, new`, breaking the sticky-routing key; remove any existing session header before setting the canonical one.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace:** `extensions/zai/package.json:8`
  - body: The new ZAI files import old `@mariozechner/*` packages and add `@mariozechner/pi-ai@0.69.0`, while current main uses `@earendil-works/*@0.74.0`. That installs a second stale provider runtime and can diverge transport behavior without a compatibility reason.
  - confidence: 0.9
