## Summary

## What This Changes

The PR adds a ZAI provider stream wrapper that injects `X-Session-Id` from `options.sessionId`, composes tool-stream defaults, updates ZAI docs/changelog/tests, and declares a new `@mariozechner/pi-ai` runtime dependency.

## Best Possible Solution

Keep the feature ZAI-owned, but repair the branch so session-header injection composes with the current ZAI stream wrapper, sanitizes header values, canonicalizes/replaces existing session headers, uses the repo-current Pi dependency, and carries focused tests plus changelog credit.

## Reproduction Assessment

Yes. The PR defects have high-confidence mechanical checks: current-main ZAI tests lock the thinking behavior, Node's `Request` rejects CR/LF/NUL header values, and `Headers` combines duplicate differently cased session header keys.

## Solution Assessment

No. The feature direction is narrow and appropriate for the ZAI plugin, but the current PR implementation is not the best mergeable solution until it preserves current wrapper behavior and tightens header/dependency/changelog handling.
