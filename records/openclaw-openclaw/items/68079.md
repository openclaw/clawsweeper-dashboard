## Summary

Keep open: current main does not implement ZAI sticky routing, and the PR addresses a plausible provider-local performance gap, but the latest head still has concrete correctness and security blockers verified against current source and the live PR diff.

## What This Changes

The PR adds a ZAI provider stream wrapper, tests, docs, changelog, package metadata, and lockfile changes to inject `X-Session-Id` from the agent session for prompt-cache sticky routing.

## Best Possible Solution

Land a repaired provider-local ZAI implementation that preserves existing thinking/tool-stream behavior, uses the transport turn-state seam or equivalent normal-call plumbing, sends a valid opaque affinity key, normalizes existing session headers, and stays on the current Pi runtime namespace.

## Reproduction Assessment

Yes for the PR blockers: current-main source shows the missing session propagation and dropped ZAI thinking wrapper, and a Node `Headers` probe reproduces invalid-value rejection plus duplicate-cased header merging. The provider cache improvement itself was not independently rerun.

## Solution Assessment

No as currently implemented. Provider-local sticky routing is the right boundary, but this patch should use an opaque transport-safe affinity value, preserve the current wrapper chain, normalize headers, and use current runtime dependencies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the run session id to ZAI transport calls:** `extensions/zai/stream.ts:34`
  - body: `createZaiSessionIdHeaderWrapper` reads `options.sessionId`, but the normal embedded provider/boundary-aware stream paths do not pass the run session id into options here. Real ZAI agent calls can therefore skip the new header while the tests still pass because they call the wrapper with a synthetic `sessionId`.
  - confidence: 0.83
- **[P2] Compose with the current ZAI wrapper:** `extensions/zai/index.ts:282`
  - body: Current main's ZAI wrapper applies both tool-stream defaults and thinking payload patches for disabled or preserved-thinking modes. Replacing the registration with this new wrapper drops those covered behaviors, so ZAI thinking settings can regress after merge.
  - confidence: 0.94
- **[P2] Hash the affinity value before sending it:** `extensions/zai/stream.ts:42`
  - body: The new header forwards the raw OpenClaw session identifier to ZAI, but the provider only needs a stable routing key. Derive a stable opaque provider-local value so user or client-chosen session labels are not exposed to the external provider.
  - confidence: 0.78
- **[P2] Reject header-invalid session ids:** `extensions/zai/stream.ts:21`
  - body: `isInjectableSessionId` accepts any non-empty trimmed string, including CR, LF, and NUL. Once copied into `X-Session-Id`, those values are rejected during Fetch header normalization before a valid ZAI request can be sent.
  - confidence: 0.91
- **[P2] Replace existing x-session-id case-insensitively:** `extensions/zai/stream.ts:40-42`
  - body: Spreading existing headers before adding `X-Session-Id` leaves a prior `x-session-id` key in place. Header normalization combines the two values, such as `old, new`, which can break the sticky-routing key.
  - confidence: 0.91
- **[P2] Use the current Pi package namespace:** `extensions/zai/package.json:8`
  - body: The new files import old `@mariozechner/*` packages and add `@mariozechner/pi-ai@0.69.0`, while current main uses `@earendil-works/*@0.74.0`. That installs a second stale provider runtime and can diverge transport behavior without a compatibility reason.
  - confidence: 0.92
