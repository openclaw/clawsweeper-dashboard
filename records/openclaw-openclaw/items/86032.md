## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Emit every recovered DSML invoke:** `src/agents/openai-transport-stream.ts:2657`
  - body: The `matched ||= emitDeepSeekDsmlToolCall(...)` expression short-circuits after the first successful invoke, so a single DSML `tool_calls` block containing two valid `<invoke>` entries will only execute the first one. Native `tool_calls` supports multiple calls, so this fallback should loop over and emit every valid recovered invoke instead of silently dropping the rest.
  - confidence: 0.89
- **[P2] Preserve visible text around recovered DSML:** `src/agents/openai-transport-stream.ts:2584-2588`
  - body: When recovery succeeds, this branch skips all `parts` returned by `deepSeekTextFilter.push(text)`. For a same-chunk response like `before <DSML tool call> after`, the filter would return the visible `before`/`after` text, but the new code drops it because `recovered` is true. Please keep emitting the non-DSML parts while still suppressing the recovered markup.
  - confidence: 0.86
