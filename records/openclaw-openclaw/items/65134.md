## Summary

Keep open: the slug-generation performance concern is still present on current main, but this PR is not mergeable as-is because the head still imports an undeclared package, bypasses current simple-completion auth/options contracts, regresses configured slug timeouts, and lacks contributor real-behavior proof.

## What This Changes

This PR replaces session-memory LLM slug generation’s embedded-agent run with direct `completeSimple` calls preferring the heartbeat model, and adds louder timeout failover logging.

## Best Possible Solution

Rework the slug helper around the current simple-completion runtime/auth helper, preserve timeout/error/slug-sanitization contracts, and settle the lightweight slug-model policy alongside the related open slug issues.

## Reproduction Assessment

Yes, at source level: with `llmSlug` enabled, current main still calls `generateSlugViaLLM`, which resolves the default model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not establish live token-cost or queue-congestion proof in this read-only review.

## Solution Assessment

No. Direct simple completion is a plausible direction, but this patch should first use the declared package, runtime-ready auth, nullable model parsing, configured timeout behavior, failed-completion rejection, Codex-compatible options, and a narrower diff.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import the declared pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: This import targets `@mariozechner/pi-ai`, but the workspace declares `@earendil-works/pi-ai`. The slug generator will fail module resolution before it can run unless it imports from the package actually present in `package.json`.
  - confidence: 0.96
- **[P2] Guard invalid heartbeat model refs:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: `parseModelRef` can return `null` for malformed refs such as `openai/` or an empty trimmed value. This code immediately reads `parsed.provider` and `parsed.model`, so a bad heartbeat override skips the intended fallback to the agent primary model.
  - confidence: 0.93
- **[P2] Use runtime-ready auth for simple completions:** `src/hooks/llm-slug-generator.ts:82-84`
  - body: This calls raw model auth before `completeSimple`, bypassing provider runtime auth preparation such as token exchange and base URL overrides. Follow the current simple-completion runtime/auth path so Codex, Copilot, and provider runtime hooks keep working.
  - confidence: 0.92
- **[P2] Preserve configured slug timeouts:** `src/hooks/llm-slug-generator.ts:31`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement always aborts after 15 seconds, regressing users who configured a longer timeout.
  - confidence: 0.91
- **[P2] Reject failed completion results before slugifying:** `src/hooks/llm-slug-generator.ts:117-121`
  - body: Current simple-completion callers check `stopReason === "error"` before reading content. This path reads content unconditionally, so provider error text can still become a filename slug when `completeSimple` returns an error assistant message.
  - confidence: 0.88
- **[P2] Omit temperature for Codex simple completions:** `src/hooks/llm-slug-generator.ts:109-113`
  - body: Current simple-completion callers omit `temperature` for `openai-codex` and `openai-codex-responses` models. This path always sends temperature, so a Codex heartbeat or default model can hit the same unsupported-option failure the helper already avoids.
  - confidence: 0.84
