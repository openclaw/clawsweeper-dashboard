## Summary

Keep open: the direction is useful, but the current branch lacks after-fix real behavior proof and still has blocking implementation issues around dependency scope, nullable model parsing, runtime auth, timeout behavior, and simple-completion error handling.

## What This Changes

The branch replaces LLM slug generation's embedded agent run with a direct `completeSimple` call that prefers the heartbeat model, and adds louder timeout failover logging.

## Best Possible Solution

Land a rebased, focused slug-generator change that avoids the embedded runner, follows current simple-completion auth/error contracts, preserves timeout fallback behavior, and resolves slug-model policy deliberately.

## Reproduction Assessment

Yes for source-level behavior: with `llmSlug` enabled, current main creates a temp slug session and calls `runEmbeddedPiAgent` on `temp:slug-generator`. I did not establish a live token-cost or multi-agent queue-congestion reproduction.

## Solution Assessment

No. The direction is useful, but this patch is not the best current implementation because it uses an unavailable package scope, skips runtime-ready auth and completion error handling, regresses configured timeouts, and makes a model-policy choice without maintainer agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import the current pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: Current main depends on `@earendil-works/pi-ai`, but this branch imports `completeSimple` from `@mariozechner/pi-ai` without adding that dependency. The slug generator will fail module resolution; use the current package scope.
  - confidence: 0.96
- **[P2] Guard the heartbeat model parse before dereferencing:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: `parseModelRef()` can return `null` for malformed refs such as `openai/`, but this code immediately reads `parsed.provider` and `parsed.model`. That turns a bad heartbeat-model override into a thrown error instead of falling back to the primary model.
  - confidence: 0.93
- **[P2] Use runtime-ready auth for simple completions:** `src/hooks/llm-slug-generator.ts:82-85`
  - body: This calls raw `getApiKeyForModel()` before `completeSimple`, bypassing provider runtime auth preparation such as token exchange and base URL overrides. Follow the current simple-completion path that uses `getRuntimeAuthForModel()`.
  - confidence: 0.91
- **[P2] Preserve the configured slug timeout:** `src/hooks/llm-slug-generator.ts:94-95`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement hardcodes a 15 second abort timer, regressing users who configured a longer agent timeout.
  - confidence: 0.9
- **[P2] Reject simple-completion error results before slugifying:** `src/hooks/llm-slug-generator.ts:117-121`
  - body: `completeSimple()` resolves the final `AssistantMessage` even for error events, carrying `stopReason: "error"` and `errorMessage`. Check `stopReason` before reading content so provider error text cannot become a filename slug.
  - confidence: 0.84
