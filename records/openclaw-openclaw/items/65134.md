## Summary

Keep open. Current main and the latest release still use the embedded runner and a temp session for slug generation, so the PR addresses a real performance path, but the proposed patch is not mergeable as-is because it breaks nullable model parsing, runtime-auth handling, and configured timeout behavior.

## What This Changes

The PR replaces session-memory LLM slug generation with `completeSimple`, prefers `agents.defaults.heartbeat.model`, and adds loud timeout logging in model-fallback and embedded-runner failover observations.

## Best Possible Solution

Land a focused slug-generator update that removes the embedded runner and temp session, uses the existing runtime-ready simple-completion auth/model path, preserves current timeout and fallback semantics, and keeps timeout-log loudness as a separate review.

## Reproduction Assessment

Yes. Static inspection of current main and `v2026.4.29` reproduces the old temp-session embedded-runner path, and the PR defects are traceable through current source contracts and tests.

## Solution Assessment

No. The direction is right, but the patch needs null-safe heartbeat model parsing, runtime-ready auth, preserved configured timeout behavior, updated tests, and tighter scope before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard the heartbeat model parse before dereferencing:** `src/hooks/llm-slug-generator.ts:50`
  - body: `parseModelRef()` can return `null` for blank or malformed refs such as `openai/`, but this branch immediately reads `parsed.provider` and `parsed.model`. That turns a recoverable heartbeat-model config issue into a throw instead of falling back to the primary model.
  - confidence: 0.93
- **[P2] Resolve runtime auth before calling completeSimple:** `src/hooks/llm-slug-generator.ts:83`
  - body: This uses raw `getApiKeyForModel()` output for `completeSimple`, skipping provider runtime-auth hooks. Providers that exchange credentials or base URLs before requests, such as Copilot-style token exchange, can fail here even though other simple-completion callers work.
  - confidence: 0.88
- **[P2] Preserve the configured slug timeout:** `src/hooks/llm-slug-generator.ts:95`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement hardcodes the abort timer to 15 seconds, regressing users who configured a longer timeout.
  - confidence: 0.86
