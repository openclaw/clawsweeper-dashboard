## Summary

Keep open. The efficiency direction is useful, but current main still uses the embedded slug runner and this PR head has blocking correctness issues plus no real behavior proof.

## What This Changes

The PR replaces LLM slug generation's embedded agent run with direct completeSimple, prefers the heartbeat model, and adds louder timeout failover logging.

## Best Possible Solution

Land a rebased slug-generator change that routes the lightweight completion through current pi-ai, runtime auth, error/abort/timeout contracts, and an explicit maintainer-approved slug-model policy.

## Reproduction Assessment

Yes for source-level behavior: current main creates a temporary slug session and calls runEmbeddedPiAgent on temp:slug-generator when llmSlug is enabled. I did not establish live token-cost or queue-congestion proof.

## Solution Assessment

No. The replacement direction is maintainable, but this patch should first follow the current simple-completion auth/error/timeout contract, use the declared pi-ai package, and resolve the heartbeat-model policy deliberately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import the declared pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: Current package.json only declares @earendil-works/pi-ai, but this new import uses @mariozechner/pi-ai. That package is not available in this workspace, so the slug generator will fail module resolution; import completeSimple and TextContent from the declared package scope.
  - confidence: 0.96
- **[P2] Guard the heartbeat model parse:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: parseModelRef can return null for malformed refs such as openai/ or an empty string after trimming. This code immediately reads parsed.provider and parsed.model, so one bad heartbeat override throws instead of falling back to the agent primary model.
  - confidence: 0.93
- **[P2] Use runtime-ready auth for completions:** `src/hooks/llm-slug-generator.ts:82-84`
  - body: This calls the raw model-auth helper before completeSimple, bypassing provider runtime auth preparation such as token exchange and base URL overrides. Follow the current simple-completion path through getRuntimeAuthForModel so Codex, Copilot, and provider runtime hooks keep working.
  - confidence: 0.92
- **[P2] Preserve the configured slug timeout:** `src/hooks/llm-slug-generator.ts:94-95`
  - body: Current main honors agents.defaults.timeoutSeconds for slug generation and has coverage for slow local providers. This replacement always aborts after 15 seconds, regressing users who configured a longer timeout.
  - confidence: 0.91
- **[P2] Reject failed completion results before slugifying:** `src/hooks/llm-slug-generator.ts:117-121`
  - body: completeSimple resolves an AssistantMessage even when the request ends with stopReason error or aborted. Check stopReason before reading content, otherwise partial provider error text can still become a filename slug.
  - confidence: 0.88
- **[P2] Omit temperature for Codex simple completions:** `src/hooks/llm-slug-generator.ts:110-113`
  - body: Current simple-completion callers omit temperature for openai-codex/openai-codex-responses models. This path always sends temperature, so a Codex heartbeat or default model can hit the same unsupported-option failure the current helper already avoids.
  - confidence: 0.84
