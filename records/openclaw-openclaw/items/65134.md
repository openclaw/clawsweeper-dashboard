## Summary

Keep this PR open: the lightweight slug-generation direction is still useful and not implemented on current main, but the branch is not merge-ready because it is conflicting, lacks real behavior proof, and introduces blocking package/auth/timeout/model-routing defects.

## What This Changes

The PR replaces session-memory LLM slug generation's embedded-agent call with direct simple completion that prefers the heartbeat model, and adds louder timeout failover logging.

## Best Possible Solution

Land a focused lightweight slug-generation implementation that reuses the shared simple-completion runtime, preserves timeout/auth/error/Codex contracts, and resolves heartbeat-versus-dedicated slug model policy with the canonical issue.

## Reproduction Assessment

Yes, at source level: current main reaches `generateSlugViaLLM` from the session-memory hook when `llmSlug` is true and then calls `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce token cost or multi-agent queue buildup in this read-only review.

## Solution Assessment

No. Direct simple completion is a plausible direction, but this PR is not the best mergeable form until it preserves existing auth, timeout, Codex, model-parse, and error-result contracts and the slug-model policy is settled.

Full review comments:

- **[P1] Import from the declared pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: This imports `completeSimple` from `@mariozechner/pi-ai`, but the workspace declares `@earendil-works/pi-ai`. The slug generator will fail module resolution before it can run unless it imports from the package actually present in `package.json`.
  - confidence: 0.98
- **[P2] Handle invalid heartbeat model refs before fallback:** `src/hooks/llm-slug-generator.ts:50`
  - body: `parseModelRef` can return `null` for malformed refs such as `openai/`, but this code immediately reads `parsed.provider` and `parsed.model`. A bad heartbeat override should fall back or fail gracefully instead of crashing or failing typecheck.
  - confidence: 0.94
- **[P2] Use runtime-ready auth for simple completions:** `src/hooks/llm-slug-generator.ts:82-84`
  - body: This calls raw model auth directly before `completeSimple`, bypassing runtime auth preparation such as provider token exchange, base URL overrides, Copilot token exchange, and Codex routing. Reuse the shared simple-completion runtime/auth path so existing provider routes keep working.
  - confidence: 0.94
- **[P2] Preserve configured slug timeouts:** `src/hooks/llm-slug-generator.ts:31`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement always aborts after 15 seconds, regressing users who configured a longer timeout.
  - confidence: 0.93
- **[P2] Reject error completion results before slugifying:** `src/hooks/llm-slug-generator.ts:117-121`
  - body: Current simple-completion callers check `stopReason === "error"` before reading content. This path reads content unconditionally, so provider error text can still become a filename slug when `completeSimple` returns an error assistant message.
  - confidence: 0.9
- **[P2] Omit temperature for Codex simple completions:** `src/hooks/llm-slug-generator.ts:112`
  - body: Current simple-completion callers omit `temperature` for `openai-codex` and `openai-codex-responses` models. This path always sends temperature, so a Codex heartbeat or default model can hit the same unsupported-option failure the existing helper already avoids.
  - confidence: 0.88
