## Summary

Keep open. Current main and the latest release still use the embedded agent plus a temporary session file for session-memory slug generation, so the performance problem is real and not implemented on main. The PR direction is useful, but the current head is not merge-safe because it dereferences a nullable model parse, bypasses runtime-ready provider auth, regresses the configured slug timeout, and is reported as not mergeable by the PR API.

## What This Changes

The branch replaces session-memory slug generation's embedded agent call with a direct `completeSimple` request, prefers `agents.defaults.heartbeat.model`, and adds timeout-loud error logging in model fallback and embedded-runner failover observation modules.

## Best Possible Solution

Land a focused slug-generator change that removes `runEmbeddedPiAgent` and temp session files, uses the existing simple-completion runtime/auth preparation path, handles heartbeat-model parsing null-safely, preserves configured slug timeout semantics, and adds targeted regression coverage. Keep the timeout-loud fallback logging as a separate maintainer-reviewed change unless it is explicitly part of the slug fix.

## Reproduction Assessment

Yes. Static inspection of current main and `v2026.4.29` reproduces the old embedded-runner slug path, and the PR defects are directly reproducible from the current nullable `parseModelRef()` contract, runtime-auth helper behavior, and existing timeout tests.

## Solution Assessment

No. Switching slug generation away from a full embedded agent is the right direction, but this patch is not the narrowest maintainable implementation until it uses runtime-ready auth, preserves timeout configuration, guards invalid heartbeat model config, and removes or separates unrelated logging changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard the parsed heartbeat model before dereferencing:** `src/hooks/llm-slug-generator.ts:50`
  - body: `parseModelRef()` can return `null` for malformed refs like `provider/`, but this code immediately reads `parsed.provider` and `parsed.model`. That will fail strict typechecking and would turn a bad heartbeat model config into a slug-generation exception instead of falling back to the primary model.
  - confidence: 0.94
- **[P2] Resolve runtime-ready auth before completing:** `src/hooks/llm-slug-generator.ts:82-84`
  - body: This path calls raw `getApiKeyForModel()` and passes that key to `completeSimple`, skipping provider `prepareRuntimeAuth` exchange used by the existing simple-completion paths. Provider-backed models such as GitHub Copilot can require a runtime token exchange, so slug generation can fail even when the model works elsewhere.
  - confidence: 0.88
- **[P2] Preserve the configured slug timeout:** `src/hooks/llm-slug-generator.ts:94-95`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has regression coverage for slow local providers. Hardcoding this new `AbortController` to 15 seconds regresses that behavior for users who raised the agent timeout specifically to let local slug generation finish.
  - confidence: 0.9
