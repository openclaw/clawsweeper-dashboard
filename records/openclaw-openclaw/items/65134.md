## Summary

Keep open: current main still routes opt-in LLM slug generation through the embedded agent runner, but this PR is not merge-ready because it has concrete correctness regressions, lacks after-fix real behavior proof, and changes slug model policy without maintainer approval.

## What This Changes

The PR replaces LLM slug generation's embedded-agent call with a `completeSimple` path that prefers the heartbeat model, and adds louder timeout failover logging in agent observation code.

## Best Possible Solution

Rework the slug generator onto the current simple-completion helper/auth/error/timeout contracts, settle the lightweight slug-model policy explicitly, and coordinate with the open slug-generator follow-ups before landing.

## Reproduction Assessment

Yes, at source level: with `llmSlug` enabled, current main resolves the default agent model and calls `runEmbeddedPiAgent` on `temp:slug-generator`. I did not establish live token-cost or queue-congestion proof for this review.

## Solution Assessment

No. The direct simple-completion direction is plausible, but this patch should first use the declared pi-ai package, runtime-ready auth, nullable model parsing, configured timeout behavior, failed-completion rejection, and Codex-compatible options.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import the declared pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: Current `package.json` and `pnpm-lock.yaml` declare `@earendil-works/pi-ai`, but this new import uses `@mariozechner/pi-ai`. That package is not available in this workspace, so the slug generator will fail module resolution; import `completeSimple` and `TextContent` from the declared package scope.
  - confidence: 0.96
- **[P2] Guard the heartbeat model parse:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: `parseModelRef` can return `null` for malformed refs such as `openai/` or an empty trimmed value. This code immediately reads `parsed.provider` and `parsed.model`, so one bad heartbeat override throws instead of falling back to the agent primary model.
  - confidence: 0.93
- **[P2] Use runtime-ready auth for completions:** `src/hooks/llm-slug-generator.ts:82-84`
  - body: This calls the raw model-auth helper before `completeSimple`, bypassing provider runtime auth preparation such as token exchange and base URL overrides. Follow the current simple-completion path through runtime auth or the prepared simple-completion helper so Codex, Copilot, and provider runtime hooks keep working.
  - confidence: 0.92
- **[P2] Preserve the configured slug timeout:** `src/hooks/llm-slug-generator.ts:94-95`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement always aborts after 15 seconds, regressing users who configured a longer timeout.
  - confidence: 0.91
- **[P2] Reject failed completion results before slugifying:** `src/hooks/llm-slug-generator.ts:117-121`
  - body: Current simple-completion callers check for `stopReason === "error"` before reading content. This path reads content unconditionally, so partial provider error text can still become a filename slug when `completeSimple` returns an error assistant message.
  - confidence: 0.88
- **[P2] Omit temperature for Codex simple completions:** `src/hooks/llm-slug-generator.ts:110-113`
  - body: Current simple-completion callers omit `temperature` for `openai-codex` and `openai-codex-responses` models. This path always sends temperature, so a Codex heartbeat or default model can hit the same unsupported-option failure the current helper already avoids.
  - confidence: 0.84
