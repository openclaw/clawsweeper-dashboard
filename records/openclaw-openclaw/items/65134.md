## Summary

Keep open: current main still uses the embedded-runner slug path, so the PR addresses unresolved work, but the branch is not mergeable as-is because it is stale against current main, has concrete build/runtime regressions, and lacks after-fix real behavior proof.

## What This Changes

The PR replaces LLM session-memory slug generation’s embedded agent run with a direct simple-completion call that prefers the heartbeat model, and also adds loud timeout logging for model fallback and embedded-runner failover.

## Best Possible Solution

Land a rebased, focused slug-generator change that removes the embedded runner and temp session, uses the current simple-completion package/helper/runtime-auth path, preserves timeout and fallback behavior, and keeps unrelated timeout-log loudness separate.

## Reproduction Assessment

Yes for source-level behavior: current main still creates a temp slug session and calls `runEmbeddedPiAgent` on `temp:slug-generator` when opt-in LLM slugs run. I did not live-reproduce the reported multi-agent queue buildup or token-cost numbers.

## Solution Assessment

No. The direction is useful, but this patch should be rebased and narrowed, use the current simple-completion/runtime-auth contracts, preserve configured timeout behavior, and avoid deciding heartbeat-model policy without maintainer agreement.
