## Summary

Keep open: the optimization is still useful because current main still uses the embedded slug runner for opt-in LLM slugs, but this PR is not merge-ready due to blocking code defects, missing real behavior proof, and an unresolved slug-model policy choice.

## What This Changes

The PR replaces LLM slug generation's embedded agent run with a direct `completeSimple` call that prefers the heartbeat model and adds louder timeout failover logging.

## Best Possible Solution

Land a rebased, focused slug-generator change that removes the embedded runner, uses current simple-completion runtime auth, preserves timeout and fallback behavior, and resolves slug-model policy deliberately.

## Reproduction Assessment

Yes for source-level behavior: with `llmSlug` enabled, current main creates a temp slug session and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not establish a live token-cost or multi-agent queue-congestion reproduction.

## Solution Assessment

No. The direction is useful, but this patch is not the best current implementation because it imports the wrong package scope, skips runtime-ready auth, regresses configured timeouts, and makes a model-policy choice without maintainer agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Import the current pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: Current main depends on `@earendil-works/pi-ai`, but this branch imports `completeSimple` from `@mariozechner/pi-ai` without adding that dependency. After applying this diff to current main, the slug generator cannot resolve the module; use the current package scope.
  - confidence: 0.96
- **[P2] Guard the heartbeat model parse before dereferencing:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: `parseModelRef()` returns `null` for blank or malformed refs such as `openai/`, but this code immediately reads `parsed.provider` and `parsed.model`. That turns a recoverable heartbeat-model config issue into a thrown error instead of falling back to the primary model.
  - confidence: 0.93
- **[P2] Use runtime-ready auth for simple completions:** `src/hooks/llm-slug-generator.ts:82-85`
  - body: This calls raw `getApiKeyForModel()` before `completeSimple`, bypassing the runtime-auth path that applies provider auth preparation such as token exchange and base URL overrides. Providers with runtime auth can fail here even though other simple-completion callers work.
  - confidence: 0.9
- **[P2] Preserve the configured slug timeout:** `src/hooks/llm-slug-generator.ts:94-95`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement hardcodes the abort timer to 15 seconds, regressing users who configured a longer agent timeout.
  - confidence: 0.88
