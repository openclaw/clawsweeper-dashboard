The PR replaces session-memory LLM slug generation’s embedded-agent call with direct `completeSimple` use that prefers the heartbeat model, and adds louder timeout failover logging.

## Best Possible Solution

Land one focused slug-generation policy that avoids embedded-agent overhead while preserving timeout, auth/runtime, Codex-option, error-result, and slug-sanitization contracts, then coordinate heartbeat-model or dedicated slug-model behavior with the open policy issue.

## Reproduction Assessment

Yes, at source level: with `hooks.internal.entries.session-memory.llmSlug: true`, current main reaches `generateSlugViaLLM`, resolves the default agent model, and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not live-reproduce token cost or multi-agent queue buildup in this read-only review.

## Solution Assessment

No. Direct simple completion is a plausible direction, but this patch is not the best mergeable form until it preserves existing auth, timeout, Codex, model-parse, and error-result contracts and the slug-model policy is settled.

- **[P1] Import from the declared pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: This imports `completeSimple` from `@mariozechner/pi-ai`, but the workspace declares `@earendil-works/pi-ai`. The slug generator will fail module resolution before it can run unless it imports from the package actually present in `package.json`.
  - confidence: 0.97
- **[P2] Guard invalid heartbeat model refs:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: `parseModelRef` can return `null` for malformed refs such as `openai/` or an empty trimmed value. This code immediately reads `parsed.provider` and `parsed.model`, so a bad heartbeat override skips the intended fallback to the agent primary model.
  - confidence: 0.94
- **[P2] Use runtime-ready auth for simple completions:** `src/hooks/llm-slug-generator.ts:82-84`
  - body: This calls raw model auth before `completeSimple`, bypassing provider runtime auth preparation such as token exchange and base URL overrides. Follow the current simple-completion runtime/auth path so Codex, Copilot, and provider runtime hooks keep working.
  - confidence: 0.93
- **[P2] Preserve configured slug timeouts:** `src/hooks/llm-slug-generator.ts:31`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement always aborts after 15 seconds, regressing users who configured a longer timeout.
  - confidence: 0.92
- **[P2] Reject failed completion results before slugifying:** `src/hooks/llm-slug-generator.ts:117-121`
  - body: Current simple-completion callers check `stopReason === "error"` before reading content. This path reads content unconditionally, so provider error text can still become a filename slug when `completeSimple` returns an error assistant message.
  - confidence: 0.89
- **[P2] Omit temperature for Codex simple completions:** `src/hooks/llm-slug-generator.ts:109-113`
  - body: Current simple-completion callers omit `temperature` for `openai-codex` and `openai-codex-responses` models. This path always sends temperature, so a Codex heartbeat or default model can hit the same unsupported-option failure the helper already avoids.
  - confidence: 0.86
