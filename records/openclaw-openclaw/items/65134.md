## Summary

Keep open: current main still uses the embedded-agent slug path, and this PR is a plausible direction, but the patch has blocking implementation defects, missing real behavior proof, and unresolved slug model-routing policy overlap.

## What This Changes

The PR replaces session-memory LLM slug generation’s embedded-agent run with `completeSimple` using a heartbeat-model preference, and adds louder timeout failover logging.

## Best Possible Solution

Rework the slug helper around the current prepared simple-completion runtime/auth path, preserve timeout/error/slug-sanitization contracts, and settle the lightweight slug-model policy with the related open slug issue.

## Reproduction Assessment

Yes, at source level: with `llmSlug` enabled, current main calls `generateSlugViaLLM`, which resolves the default model and runs `runEmbeddedPiAgent` on `temp:slug-generator`. I did not establish live token-cost or queue-congestion proof in this read-only review.

## Solution Assessment

No. Direct simple completion is a plausible direction, but this patch needs the declared dependency, runtime-ready auth, nullable model parsing, configured timeout preservation, failed-completion rejection, Codex-compatible options, and a tighter diff.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import the declared pi-ai package:** `src/hooks/llm-slug-generator.ts:15`
  - body: This imports `completeSimple` from `@mariozechner/pi-ai`, but the workspace declares `@earendil-works/pi-ai`. The slug generator will fail module resolution before it can run unless it imports from the package actually present in `package.json`.
  - confidence: 0.96
- **[P2] Guard invalid heartbeat model refs:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: `parseModelRef` can return `null` for malformed refs such as `openai/` or an empty trimmed value. This code immediately reads `parsed.provider` and `parsed.model`, so a bad heartbeat override crashes instead of falling back to the agent primary model.
  - confidence: 0.93
- **[P2] Use runtime-ready auth for simple completions:** `src/hooks/llm-slug-generator.ts:82-84`
  - body: This calls raw model auth before `completeSimple`, bypassing provider runtime auth preparation such as token exchange and base URL overrides. Use the current prepared simple-completion runtime/auth path so Codex, Copilot, and provider runtime hooks keep working.
  - confidence: 0.92
- **[P2] Preserve configured slug timeouts:** `src/hooks/llm-slug-generator.ts:31`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This replacement always aborts after 15 seconds, regressing users who configured a longer timeout.
  - confidence: 0.91
- **[P2] Reject failed completion results before slugifying:** `src/hooks/llm-slug-generator.ts:117-121`
  - body: `completeSimple` returns an assistant message with `stopReason`; callers must reject `error` and `aborted` results before reading content. This path reads content unconditionally, so partial provider error text can become a filename slug.
  - confidence: 0.88
- **[P2] Omit temperature for Codex simple completions:** `src/hooks/llm-slug-generator.ts:109-113`
  - body: Current simple-completion callers omit `temperature` for `openai-codex` and `openai-codex-responses` models. This path always sends temperature, so a Codex heartbeat or default model can hit the same unsupported-option failure the helper already avoids.
  - confidence: 0.84
