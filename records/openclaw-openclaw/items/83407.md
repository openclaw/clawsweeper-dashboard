## Summary

Keep open: current main has Xiaomi Anthropic `reasoning_content` replay support for thinking-enabled routes, but the reported `mimo-v2-flash` `reasoning:false` path is still gated out by source and tests.

## What This Changes

The issue reports Xiaomi MiMo v2-flash follow-up turns failing because prior assistant `reasoning_content` is not preserved and replayed for an Anthropic-compatible Xiaomi endpoint.

## Best Possible Solution

Preserve and replay Xiaomi-native Anthropic `reasoning_content` only when it was actually observed in prior assistant output, including non-reasoning `mimo-v2-flash`, while keeping generic Anthropic and non-Xiaomi routes unchanged.

## Reproduction Assessment

No live credentialed Xiaomi reproduction was run. Source-level reproduction is clear: current main skips Xiaomi Anthropic `reasoning_content` capture and replay unless `thinkingEnabled === true`, while the report shows a non-reasoning `mimo-v2-flash` follow-up rejected for missing replay data.

## Solution Assessment

Yes, if scoped narrowly. The maintainable fix is to replay observed Xiaomi-native Anthropic `reasoning_content` for this provider endpoint without broadening generic Anthropic behavior or surfacing reasoning to users.
