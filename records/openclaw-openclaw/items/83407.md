## Summary

This should stay open: current main has a Xiaomi Anthropic-messages `reasoning_content` replay fix, but it is gated to thinking-enabled routes while this report is specifically `mimo-v2-flash` with `reasoning: false`. Source inspection shows flash is excluded from the Xiaomi thinking profile and there is no current regression coverage for this exact flash/off Anthropic-compatible configuration.

## What This Changes

The issue requests that OpenClaw preserve and resend Xiaomi MiMo `reasoning_content` across multi-turn conversations for `mimo-v2-flash` on the Anthropic Messages API format.

## Best Possible Solution

Add a narrow Xiaomi Anthropic-messages regression for `mimo-v2-flash` with `reasoning: false`, then either explicitly disable provider thinking for that route or preserve the required replay field when Xiaomi emits it.

## Reproduction Assessment

No live reproduction was run, but there is a credible source-level reproduction path: the reported config makes current main choose the off-only Xiaomi flash profile, while the Anthropic replay code only preserves `reasoning_content` when thinking is enabled. Confirming the provider-side trigger still needs Xiaomi credentials.

## Solution Assessment

No, the existing merged Xiaomi Anthropic replay fix is not sufficient for this exact report because it does not prove the `mimo-v2-flash` plus `reasoning: false` path is handled. The narrow maintainable solution is to verify Xiaomi's flash contract and patch the provider/transport boundary with focused regression coverage.
