## Summary

Current main already addresses the reported failure mode: doctor may still canonicalize legacy `claude-cli/*` model refs to `anthropic/*`, but it now preserves the Claude CLI execution route with model-scoped runtime policy, and the run path keys CLI session bindings by the resolved `claude-cli` execution provider.

## What This Changes

The issue reports that `openclaw doctor` rewrites a configured `claude-cli/*` primary model to `anthropic/*`, breaking Claude CLI session continuity after fallback.

## Best Possible Solution

Keep the current canonical `anthropic/*` model ref plus model-scoped `agentRuntime.id: "claude-cli"` contract, and let the next release carry the current-main fix to affected v2026.5.7 users.

## Reproduction Assessment

No for current main. The reported v2026.5.7 path is plausible, but current main has source and regression coverage showing the rewritten Anthropic ref carries `claude-cli` runtime policy and session bindings use the resolved `claude-cli` provider.

## Solution Assessment

Yes. The current main solution is the narrow maintainable path: keep provider/model identity canonical and preserve execution/session behavior through model-scoped runtime policy instead of preserving legacy provider prefixes.
