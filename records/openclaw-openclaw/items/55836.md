## Summary

Keep open. Current main still has no opt-in vi/vim modal input mode in the TUI editor, no config/docs surface for such a mode, and the directly related implementation PR was closed unmerged while #22770 tracks a different external-editor compose workflow.

## What This Changes

Add an opt-in vi/vim keybinding mode to the TUI input editor so users can switch between insert/normal mode and use common vi motions inside the in-place composer.

## Best Possible Solution

Implement a default-off vi/vim mode in the TUI editor layer, expose it through the supported config schema/help/docs, and keep the external $EDITOR compose work tracked separately in #22770.
