## Summary

Keep open: current main still does not provide an opt-in vi/vim modal input mode for the TUI composer, and the only directly related implementation PR was closed unmerged after the author concluded the behavior should live in the upstream editor component rather than a hand-rolled OpenClaw wrapper.

## What This Changes

Add a vi/vim modal keybinding mode for the TUI input editor, exposed through configuration or an environment variable.

## Best Possible Solution

Add first-class vi-mode support in the editor layer, then expose a default-off OpenClaw config surface and docs once the mode semantics and Escape behavior are settled.

## Solution Assessment
