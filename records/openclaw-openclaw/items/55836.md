Keep open: current main and v2026.6.8 still do not provide first-class vi/vim modal editing for the TUI input editor. The direct implementation PR was closed unmerged, and the current editor-component plugin seam is not wired into the active TUI editor, so this still needs maintainer product/API direction rather than cleanup closure.

## What This Changes

Request adding vi/vim modal keybindings to the TUI input editor, with a possible config or environment toggle.

## Best Possible Solution

Keep this open until maintainers choose whether vi behavior belongs in pi-tui, OpenClaw core, or a real editor-component plugin API, then approve a default-preserving public configuration shape.

## Reproduction Assessment

Yes. Current-main source and the latest release snapshot have no TUI vi-mode config, no modal editor state, no vi motion handling, and no pi-tui vi-mode option passed into CustomEditor.

## Solution Assessment

Unclear. The requested outcome is reasonable, but the best implementation path needs maintainer direction because a hand-rolled OpenClaw parser may be the wrong layer compared with first-class pi-tui editor support or a real editor-component plugin seam.
