## Summary

Keep open: this is an active external PR with a protected `maintainer` label, insufficient real Windows login proof, and two blocking review findings in the proposed patch.

## What This Changes

The PR makes Windows node service installs generate and register a hidden `.vbs` launcher while adding scheduled-task/startup fallback tests and a changelog entry.

## Best Possible Solution

Land a node-only hidden startup fix that preserves node service lifecycle observability, restores clean changelog encoding, and includes redacted Windows login proof before merge.

## Reproduction Assessment

Yes for source reproduction: https://github.com/openclaw/openclaw/issues/81254 gives Windows 11 sign-in steps and evidence, and current main schedules the node task directly through `node.cmd` while docs define a background node service. I did not perform a live Windows sign-in flow in this read-only review.

## Solution Assessment

No as written: the hidden-launcher direction is narrow, but `Run(..., 0, False)` detaches the long-running node from Scheduled Task tracking and the branch corrupts unrelated changelog text. The safer path keeps node status/stop/restart observable while hiding the login window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep the hidden node launcher under lifecycle tracking:** `src/daemon/schtasks.ts:334`
  - body: The generated VBScript starts `node.cmd` with `WScript.Shell.Run(..., 0, False)`, so Task Scheduler can mark the action complete while the long-running node continues outside scheduler tracking. After that, `readScheduledTaskRuntime` only recovers non-running tasks through a verified gateway listener, so `openclaw node status/stop/restart` can miss the hidden node process or target the gateway port instead.
  - confidence: 0.9
- **[P2] Restore the changelog encoding:** `CHANGELOG.md:1`
  - body: The branch adds a UTF-8 BOM to `CHANGELOG.md` and rewrites unrelated entries into mojibake, including arrows and emoji in older changelog bullets. Restore the original encoding and keep only the intended Windows node entry.
  - confidence: 0.96
