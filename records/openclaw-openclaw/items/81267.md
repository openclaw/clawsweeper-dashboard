## Summary

Keep open: the linked Windows behavior is valid, but this PR still introduces blocking lifecycle and changelog-encoding regressions; the protected maintainer label and incomplete real login proof also require human follow-up.

## What This Changes

The PR adds an opt-in Windows hidden `.vbs` launcher for node scheduled-task and Startup-folder installs, sets node service env to use it, updates tests, and adds a changelog entry.

## Best Possible Solution

Land a node-only hidden Windows launch path that preserves status/stop/restart lifecycle tracking, restores clean changelog encoding, and includes redacted Windows login proof.

## Reproduction Assessment

Yes for source reproduction: the linked issue gives Windows 11 install/sign-in steps and evidence, and current main schedules the node service directly through `node.cmd` despite the docs calling it a headless background service. I did not perform a live Windows login cycle in this read-only review.

## Solution Assessment

No as written: the hidden-launcher direction is narrow, but the current `Run(..., 0, False)` implementation detaches node from Scheduled Task lifecycle tracking and the branch corrupts unrelated changelog Unicode. A safer fix should hide the window while keeping node status/stop/restart observable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep the hidden launcher attached to task lifecycle:** `src/daemon/schtasks.ts:334`
  - body: The generated VBScript starts `node.cmd` with `WScript.Shell.Run(..., 0, False)`, so Task Scheduler can mark the action complete while the long-running node continues outside scheduler tracking. After that, `openclaw node status/stop/restart` follows the scheduled-task runtime path, which only recovers non-running tasks through gateway-listener detection and can miss or fail to stop the detached node host.
  - confidence: 0.9
- **[P2] Restore the changelog encoding:** `CHANGELOG.md:1`
  - body: This branch adds a UTF-8 BOM to `CHANGELOG.md` and mojibakes unrelated Unicode entries, including arrows and emoji in older bullets. Restore the file encoding and keep only the intended Windows node entry.
  - confidence: 0.96
