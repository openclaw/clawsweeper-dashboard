## Summary

Keep open. Current main still has the guard gap: bundled channel metadata can be checked or regenerated manually, but the ordinary build and check paths do not run that staleness guard while runtime validation still consumes the generated channel schemas.

## What This Changes

The issue asks for build-time regeneration or validation so bundled channel config metadata cannot drift from extension-owned channel config schemas.

## Best Possible Solution

Add a non-mutating staleness guard to the build/check path so bundled channel schema changes fail before packaging or startup unless the generated metadata is refreshed.

## Reproduction Assessment

Yes at source level: patch a bundled channel `src/config-schema.ts`, skip regenerating `src/config/bundled-channel-config-metadata.generated.ts`, and current build/check paths will not run `config:channels:check` before runtime validation consumes the stale schema. I did not execute the repro because this review is read-only.

## Solution Assessment

Yes: a build/check staleness guard is the narrowest maintainable fix. Prefer failing clearly with `config:channels:check` over silently mutating checked-in generated source during an ordinary build unless maintainers explicitly choose regeneration.
