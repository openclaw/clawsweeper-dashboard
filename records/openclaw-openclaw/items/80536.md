## Summary

Keep open. Current main still validates channel config through checked-in generated metadata and has a drift checker, but normal build/check/changed-check and CI check-shard paths still do not run it; the two linked fix PRs are closed without merge.

## What This Changes

The issue asks OpenClaw to catch stale bundled channel config metadata when channel config schemas change so runtime validation accepts newly added channel config keys.

## Best Possible Solution

Add a non-writing generated-metadata drift check to ordinary build and CI/changed-check paths while keeping runtime validation strict and generated updates explicit.

## Reproduction Assessment

Yes. By source inspection, adding a channel schema key without regenerating src/config/bundled-channel-config-metadata.generated.ts leaves runtime validation reading the stale generated JSON schema, and current build/check paths do not run the existing drift check.

## Solution Assessment

Yes. The narrow maintainable fix is to run the existing non-writing generator check in normal build and CI guard paths; relaxing runtime validation or relying on runtime regeneration would weaken the config contract.
