## Summary

## What This Changes

The issue reports that bundled channel config metadata can drift from bundled channel `config-schema.ts` changes, causing runtime validation to reject new channel config keys at gateway startup.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/91134 so upstream CI catches stale bundled channel metadata, while keeping any remaining build-time/downstream-build protection scope explicit.
