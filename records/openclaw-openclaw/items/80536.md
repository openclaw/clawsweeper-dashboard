## Summary

Keep open. Current main still has a source-proven generated metadata drift gap: runtime channel validation can fall back to checked-in bundled metadata, while ordinary build and CI paths still do not run the existing staleness check; the two linked fix PRs were closed unmerged.

## What This Changes

The issue asks OpenClaw to catch or regenerate stale bundled channel config metadata when bundled channel config schemas change.

## Best Possible Solution

Add a normal non-writing guard for bundled channel config metadata drift, and keep runtime validation strict while any broader live-schema ownership cleanup is reviewed separately.

## Reproduction Assessment

Yes. By source inspection, a bundled channel schema change can leave `src/config/bundled-channel-config-metadata.generated.ts` stale, and current build/CI paths do not run the existing `--check` guard before runtime validation can consume that metadata.

## Solution Assessment

Yes. The narrow maintainable fix is to wire the existing non-writing metadata check into ordinary build/CI or changed-check gates; automatically regenerating checked-in source during build would be a broader maintainer decision.
