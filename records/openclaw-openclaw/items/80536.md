## What This Changes

The issue asks OpenClaw to prevent bundled channel config metadata from going stale when a plugin config schema changes without regenerating the checked-in generated JSON schema.

## Best Possible Solution

Land a narrow version of https://github.com/openclaw/openclaw/pull/80546, or an equivalent build/CI guard, so generated channel metadata staleness fails before release.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: add a property only to a bundled plugin config schema, run the current build path, and the runtime validator still uses the stale checked-in generated schema.
