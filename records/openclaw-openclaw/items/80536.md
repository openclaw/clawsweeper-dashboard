## Summary

Keep open: current main still validates channel config through checked-in generated metadata, the normal CI/build check path does not yet gate drift, and the linked closing PR remains open.

## What This Changes

The issue asks for a build or CI guard so bundled channel config schema changes cannot leave checked-in generated JSON schema metadata stale.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80552 or an equivalent normal CI/build guard that runs the existing bundled channel config metadata staleness check.

## Reproduction Assessment

Yes. Source inspection shows runtime channel validation reads the generated bundled metadata and AJV rejects keys missing from that schema, so adding a bundled plugin config-schema key without regenerating metadata follows the reported failure path.

## Solution Assessment

Yes for a narrow first fix: wiring the existing bundled-channel-config-metadata check into normal CI is a maintainable way to catch this drift before runtime. The issue should remain open until the linked PR or equivalent guard merges.
