## Summary

Keep open: current main still has the generated-metadata validator gap and no merged CI/build guard. The linked fix PRs proposed the right narrow guard, but both were closed unmerged and the referenced fix commit is not contained in current main.

## What This Changes

Request that channel config-schema changes regenerate or at least fail CI/build when `src/config/bundled-channel-config-metadata.generated.ts` is stale.

## Best Possible Solution

Add the existing bundled-channel-config-metadata staleness check to the relevant CI/build validation path so schema changes cannot merge or ship without regenerated metadata.

## Reproduction Assessment

Yes by source inspection. Current main validates channel config from checked-in generated metadata while build/check/CI paths do not run the existing staleness checker, so a config-schema-only change can still pass validation lanes and later fail at runtime.

## Solution Assessment

Yes for the narrow repair path. Running the existing read-only staleness check in CI or build validation is the smallest maintainable fix; automatic regeneration during build is broader and can remain a follow-up if desired.
