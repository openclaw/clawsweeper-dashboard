## Summary

Keep open: the Feishu onboarding gap is real, but this PR still drops the selected account ID before config writes, prompts on fresh setup because the account list has a fallback, and regresses configured default-account behavior.

## What This Changes

The PR changes the Feishu setup surface and adapter to prompt for custom account IDs during onboarding and normalize setup account selection.

## Best Possible Solution

Revise the Feishu setup flow so the resolved account ID is threaded through finalization, edit, and new-app writes, while fresh setup keeps default behavior and configured defaultAccount remains honored.

## Reproduction Assessment

Yes. The source path is deterministic: the setup framework passes accountId to wizard.finalize, but the PR finalizer drops it and the write helper re-resolves the default target before patching config.

## Solution Assessment

No. Prompting for a custom Feishu account ID is the right direction, but this implementation does not yet write to that selected account and regresses adjacent setup behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Propagate the selected accountId into Feishu setup writes:** `extensions/feishu/src/setup-surface.ts:643-654`
  - body: resolveAccountIdForConfigure can now return a custom ID, but finalize ignores the accountId argument and calls runEditFlow/runNewAppFlow without it. runNewAppFlow then re-resolves the default target before patchFeishuConfig, so choosing a new account can still write credentials and policies to the default account instead of the selected account.
  - confidence: 0.94
- **[P2] Gate the new-account prompt on real Feishu configuration:** `extensions/feishu/src/setup-surface.ts:556-560`
  - body: listFeishuAccountIds(cfg) includes the fallback default account when no Feishu account exists, so existingAccounts.length === 0 never represents fresh setup. A fresh onboarding path will ask whether to create a new account even though there is no real Feishu account yet; check real Feishu configuration or configured account IDs before prompting.
  - confidence: 0.91
- **[P2] Preserve configured defaultAccount in the setup adapter:** `extensions/feishu/src/setup-core.ts:34-36`
  - body: Replacing resolveDefaultFeishuAccountId(cfg) with normalizeAccountId(accountId) means setup surfaces that call the adapter without an explicit account no longer honor channels.feishu.defaultAccount. They resolve to the literal default account and can mark or write the wrong account.
  - confidence: 0.86
