## Summary

Keep this PR open, but it is not merge-ready: the account prompt is useful, yet the selected Feishu account ID is not threaded into the config-writing path, fresh setup/default-account behavior regresses, and no real Feishu onboarding proof is present.

## What This Changes

The branch adds a Feishu onboarding account-ID prompt, exports an account helper, and changes the Feishu setup adapter to normalize the provided account ID.

## Best Possible Solution

Thread the resolved account ID through Feishu finalization, edit, and new-app writes, preserve fresh default setup and configured defaultAccount behavior, add regression coverage, and require redacted real onboarding proof before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the PR resolver can return a custom accountId, while finalize ignores it and runNewAppFlow writes through resolveDefaultFeishuAccountId. I did not run live Feishu onboarding because this review is read-only and no live credentials were provided.

## Solution Assessment

No. Prompting for a custom Feishu account ID is the right direction, but this patch is not the best implementation until the selected account is carried into writes and default-account semantics are preserved.
