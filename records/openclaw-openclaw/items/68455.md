## What This Changes

The PR adds Feishu setup logic to prompt for a custom account ID when configuring additional Feishu accounts and changes account ID resolution for setup writes.

## Best Possible Solution

Thread the resolved accountId through Feishu finalize, runEditFlow, and runNewAppFlow so config writes target the selected account; only prompt for a new ID after real Feishu configuration exists; preserve configured-default behavior; then add focused tests and redacted real onboarding proof.
