## Summary

Keep this PR open, but it needs changes before merge. The branch adds a useful Feishu account picker, yet the selected account ID is not threaded into the Feishu config-writing path, and the fresh-install prompt check treats the shared fallback `default` as a real configured account.

## What This Changes

The PR adds a custom Feishu account ID prompt for onboarding, exports the Feishu default-account helper, and changes the Feishu setup adapter to normalize provided account IDs.

## Best Possible Solution

Update the Feishu plugin-owned onboarding flow so the resolved accountId is passed through finalization, edit, and new-app setup and used for all credential and security-policy patches; keep first-run setup on the default account without a new-account prompt; add focused regression tests for fresh setup and second-account setup.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: choose a new Feishu account ID on the PR branch, then follow the finalize path and observe that Feishu discards the resolved `accountId` before calling the helpers that write credentials and policy.

## Solution Assessment

No. Prompting for a custom Feishu account ID is the right direction, but this PR is incomplete because it does not use the selected account for config writes and it misdetects fresh installs as already having an account.
