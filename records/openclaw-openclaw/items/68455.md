## Summary

Keep open: current main still lacks the requested Feishu custom-account onboarding path, but this PR is incomplete because the selected account ID is not propagated into the config-writing flow and the first-run/default-account behavior regresses.

## What This Changes

This PR changes the Feishu plugin setup surface and adapter to prompt for custom account IDs during onboarding and normalize Feishu setup account selection.

## Best Possible Solution

Thread the resolved Feishu account ID through finalization, edit, and new-app setup writes; prompt only after real Feishu configuration exists; preserve configured `defaultAccount` semantics; and add focused tests for fresh setup plus second-account setup.

## Reproduction Assessment

Yes. The source-level path is deterministic: the setup framework passes `accountId` into `wizard.finalize`, but the PR finalizer ignores it and the write helpers re-resolve the default target before patching config.

## Solution Assessment

No. Prompting for a custom account ID is the right direction, but this implementation does not yet use that ID for all writes and it changes default-account behavior on adjacent setup surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Propagate the selected accountId into setup writes:** `extensions/feishu/src/setup-surface.ts:643-654`
  - body: `resolveAccountIdForConfigure` can now return a new account ID, but `finalize` still ignores the `accountId` argument from the setup framework and calls `runEditFlow`/`runNewAppFlow` without it. Those helpers re-resolve the default target before `patchFeishuConfig`, so choosing a new account can still write credentials and policies to the default account instead of the selected account.
  - confidence: 0.94
- **[P2] Gate the new-account prompt on real Feishu config:** `extensions/feishu/src/setup-surface.ts:556-560`
  - body: `listFeishuAccountIds(cfg)` returns fallback `default` for an empty config, so `existingAccounts.length === 0` never represents first-run setup. Fresh onboarding will ask whether to create a new account even though no Feishu account exists; check real Feishu configuration or configured account IDs before prompting.
  - confidence: 0.91
- **[P2] Preserve configured defaultAccount in setup adapter:** `extensions/feishu/src/setup-core.ts:34-36`
  - body: Replacing `resolveDefaultFeishuAccountId(cfg)` with `normalizeAccountId(accountId)` means setup surfaces that call the adapter without an explicit account, such as `openclaw channels add --channel feishu`, no longer honor `channels.feishu.defaultAccount`; they resolve to literal `default` and write the wrong account.
  - confidence: 0.86
