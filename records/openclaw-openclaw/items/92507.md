## Summary

This PR remains useful and appears merge-ready: current main can associate unchanged cached metadata with a newly swapped SQLite database that lacks the row, while the patch makes write elision database-aware and preserves the original cache on rollback. The focused implementation, supplied real CLI/database proof, and exact-head checks show no actionable correctness or security blocker.

## What This Changes

The PR makes memory metadata caching database-aware during atomic reindex, restores the original cached metadata after rollback, and adds direct SQLite-row regression coverage.

## Best Possible Solution

Land the database-aware cache fix with the direct row-level regression, retaining one canonical metadata upsert path and the existing atomic swap architecture.

## Reproduction Assessment

Yes at source level. Repeated forced safe reindex on one manager followed by direct inspection of `memory_index_meta_v1` exposes the missing-row condition that cached status alone can mask; the supplied after-fix run demonstrates the corrected outcome.

## Solution Assessment

Yes. Tying write elision to the active database row and restoring the cache belonging to the original database on rollback is the narrowest maintainable fix and avoids adding a second metadata path or fallback.
