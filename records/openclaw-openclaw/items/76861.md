## Summary

Keep this open: current main still has a source-visible native Anthropic path where configured `temperature` reaches `claude-opus-4-7` when thinking is disabled, and the broader Sonnet/OpenRouter scope needs a careful provider-capability fix rather than a blanket `reasoning: true` rule.

## What This Changes

The issue asks the gateway to omit `temperature`, `top_p`, and `top_k` for Anthropic reasoning-model requests, including direct Anthropic and OpenRouter-routed Anthropic models.

## Best Possible Solution

Add a provider-capability request-shaping policy that omits sampling fields for confirmed Anthropic-family routes before payload dispatch, with native Anthropic and OpenRouter regression coverage, without keying solely on generic `reasoning` metadata.

## Reproduction Assessment

Yes for the native Anthropic `temperature` path from source inspection: configured temperature is copied into stream options and current main serializes it for non-thinking reasoning-model requests. I did not run a live Anthropic/OpenRouter call, so the OpenRouter and Sonnet pieces are not fully reproduced against current main.

## Solution Assessment

No, the issue's proposed blanket `reasoning: true` gate is probably too broad because reasoning-capable models do not all share the same sampling-parameter contract. The maintainable fix is a provider/model capability or payload policy consulted by the Anthropic-family request builders.
