## Summary

Current main still has the source-level request-shape bug for direct Anthropic adaptive-thinking models, and open PR #76871 already references this issue with closing syntax, so this should stay open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks the gateway to omit unsupported sampling parameters, especially `temperature`, when building requests for Anthropic reasoning/adaptive models, including OpenRouter-routed Anthropic refs if they forward those params literally.

## Best Possible Solution

Land PR #76871 or an equivalent direct Anthropic transport fix with regression coverage, then resolve the OpenRouter scope in the same review or a linked follow-up before closing.

## Reproduction Assessment

Yes. Source inspection on current main shows `params.temperature` is sent for adaptive Claude models when no reasoning option is present, matching the reported Anthropic validation failure path; I did not run a live provider call.

## Solution Assessment

Mostly yes. Suppressing `temperature` for direct Anthropic adaptive-thinking models via the existing helper is the narrow maintainable fix, but PR review should decide whether OpenRouter-routed Anthropic refs are in scope now or should get a follow-up.
