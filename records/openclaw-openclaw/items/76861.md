## Summary

Current main still has the reported direct Anthropic request-shape bug, and the linked fix PR was closed without merging, so this issue should stay open and can be queued for a narrow replacement repair.

## What This Changes

The issue asks the gateway to omit unsupported sampling parameters, especially `temperature`, when building requests for Anthropic reasoning/adaptive models.

## Best Possible Solution

Add a model-capability guard in the direct Anthropic Messages payload builder, cover adaptive and non-adaptive models with regression tests, and split OpenRouter handling only if confirmed to share the same upstream rejection.

## Reproduction Assessment

Yes. Source inspection on current main shows the no-reasoning path sets `thinkingEnabled` false and then forwards `temperature` for the affected adaptive Claude models; I did not run a live Anthropic request.

## Solution Assessment

Yes for the direct Anthropic path. The narrow maintainable fix is to suppress `temperature` when the selected Anthropic model supports adaptive thinking, while treating OpenRouter as confirm-or-split scope.
