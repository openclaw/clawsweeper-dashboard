## Summary

Current main still forwards `temperature` on the direct Anthropic Messages path for the reported no-reasoning/non-thinking case, and the linked fixing PR is closed unmerged, so the issue should stay open and is a narrow repair candidate.

## What This Changes

The issue asks the gateway to omit unsupported sampling parameters for Anthropic reasoning/adaptive-thinking models so upstream request validation does not fail.

## Best Possible Solution

Add a provider/request capability guard that omits unsupported Anthropic sampling params for verified model families and transports, with direct Anthropic regression coverage and an explicit OpenRouter decision.

## Reproduction Assessment

Yes, source-level: configure `params.temperature` and route a normal no-reasoning turn to `anthropic/claude-opus-4-7`; current main sets `thinkingEnabled` false and sends `temperature`. I did not run a live Anthropic call in this read-only review.

## Solution Assessment

Yes for a narrow capability-based request filter in the Anthropic request builder; the issue's blanket `reasoning:true` sketch is broader than the evidence and should not be applied without per-provider/model proof.
