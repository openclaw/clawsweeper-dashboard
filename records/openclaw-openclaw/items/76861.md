## Summary

Keep this open: current main still has a source-reproducible direct Anthropic transport path that forwards `temperature` to adaptive Claude models when thinking is disabled, and open PR #76871 is already linked as the implementation candidate.

## What This Changes

The issue asks the gateway Anthropic request path to omit unsupported sampling parameters for reasoning/adaptive Claude models so configured agent temperatures do not cause upstream request rejection.

## Best Possible Solution

Review and land a narrow transport fix, likely via #76871 or a replacement, with regression coverage that proves adaptive Claude models omit unsupported sampling parameters while non-adaptive Anthropic models keep supported options.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure `temperature` without explicit reasoning, route to `claude-opus-4-7` or `claude-sonnet-4-6`, and current main builds an Anthropic payload containing `temperature`. I did not run a live Anthropic request in this read-only review.

## Solution Assessment

Yes, stripping unsupported sampling parameters before request construction is the narrow maintainable fix. The linked PR direction is plausible, but it should be checked against the issue's OpenRouter and `top_p`/`top_k` scope before the issue is closed.
