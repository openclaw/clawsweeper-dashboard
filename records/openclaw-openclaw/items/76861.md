## Summary

Current main still has the reported Anthropic request-shape bug, and the only linked implementation PR was closed unmerged, so this should stay open and is a good narrow repair candidate.

## What This Changes

The issue requests that the gateway omit unsupported sampling parameters, especially `temperature`, when routing direct Anthropic and possibly OpenRouter Anthropic reasoning/adaptive-thinking models.

## Best Possible Solution

Land a narrow direct Anthropic transport fix with regression coverage and changelog, then decide whether OpenRouter-routed Anthropic models need a separate follow-up.

## Reproduction Assessment

Yes, at source level. Current main preserves explicit `temperature`, disables thinking when no reasoning override is present, and then emits `temperature` to adaptive Claude model IDs that Anthropic documents as rejecting sampling parameters.

## Solution Assessment

Yes for the direct Anthropic path: gating temperature forwarding on the existing adaptive-thinking helper is the smallest maintainable fix. OpenRouter should be confirmed separately rather than blocking the direct repair.
