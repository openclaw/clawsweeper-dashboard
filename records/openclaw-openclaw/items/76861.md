## Summary

Current main still has the reported source-level request-shape bug, and PR #76871 is an open implementation candidate that references this issue, so the issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests omitting `temperature` and other sampling parameters from Anthropic reasoning-model gateway requests, including possible OpenRouter-routed Anthropic models.

## Best Possible Solution

Merge a payload-shaping fix that strips unsupported sampling params for adaptive Anthropic model families and either covers OpenRouter-routed Anthropic refs or links a follow-up before closing.

## Reproduction Assessment

Yes, from source inspection rather than a live Anthropic call. With an adaptive Claude model, no explicit reasoning option, and `params.temperature`, current main sets `thinkingEnabled` false and forwards `temperature` into the Anthropic Messages payload.

## Solution Assessment

Mostly yes. The narrow maintainable fix is to gate direct Anthropic temperature forwarding on the existing adaptive-thinking model helper, with a separate PR-review decision on whether OpenRouter-routed Anthropic refs need the same change before this closes.
