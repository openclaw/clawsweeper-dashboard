Report asks OpenClaw to stop Xiaomi MiMo Gateway requests from failing with `400 Param Incorrect` while direct curl succeeds.

## Best Possible Solution

## Reproduction Assessment

Yes. The original `2026.5.12` failure is reproduced by MiMo thinking-mode tool-call replay without `reasoning_content`, and the related PR/issue proof plus current regression tests cover that path; I did not run a live credentialed call in this read-only review.

## Solution Assessment

Yes. The merged fix keeps Xiaomi-specific replay behavior in the Xiaomi plugin plus a narrow proxy fallback, which is more maintainable than adding per-user request-sanitization knobs.
