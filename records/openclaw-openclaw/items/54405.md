## Summary

Keep open. Current main still lacks Chat Completions client-tool request parsing, `clientTools` handoff, and OpenAI-compatible `tool_calls` output, while the provided GitHub context shows an open unmerged PR that explicitly closes this issue.

## What This Changes

The issue requests OpenAI-compatible client-provided tool passthrough on `/v1/chat/completions`, including request `tools`/`tool_choice`, assistant `tool_calls`, and `role: "tool"` follow-up handling.

## Best Possible Solution

Review and land PR #66278 or an equivalent focused gateway change that reuses the existing `clientTools` runtime seam, adds Chat Completions request parsing and tool-call response mapping, and covers non-stream, streaming, `tool_choice`, and tool-result follow-up behavior in `src/gateway/openai-http.test.ts`.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence reproduction path: send `/v1/chat/completions` with `tools` and the adapter will ignore those fields because they are not parsed or forwarded, and current responses cannot emit `tool_calls`.

## Solution Assessment

Yes, the linked PR direction is the best fit: keep this in the gateway Chat Completions adapter and reuse the existing Responses `clientTools` seam rather than adding a Home Assistant-specific integration path.
