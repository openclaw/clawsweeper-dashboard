## Summary

Keep open: current main still lacks `/v1/chat/completions` client-tool passthrough, and https://github.com/openclaw/openclaw/pull/66278 is the active open PR with closing syntax for this issue.

## What This Changes

Request support for forwarding client-provided Chat Completions `tools`/`tool_choice` and returning OpenAI-compatible `tool_calls` for caller-executed tools.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/66278 or a successor that implements the Chat Completions tool protocol with focused gateway tests and docs, then close this issue after merge.

## Reproduction Assessment

Yes, source-level. A Chat Completions request with client `tools` has no forwarding path on current main, and the adapter has no `tool_calls` response path for client-executed tools.

## Solution Assessment

Yes. The narrow maintainable direction is to implement the protocol in the Gateway Chat Completions adapter by reusing the existing client-tool and pending-tool-call seams already present in `/v1/responses`.
