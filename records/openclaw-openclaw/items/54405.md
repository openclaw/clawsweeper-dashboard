## Summary

Keep open: current main still lacks client-provided tool passthrough for `/v1/chat/completions`, and the active open fix path is https://github.com/openclaw/openclaw/pull/66278, which uses closing syntax for this issue. The issue should remain open until that PR or a successor lands.

## What This Changes

Support client-provided Chat Completions function tools, `tool_choice`, `tool_calls` responses, and `role: "tool"` follow-up turns on `/v1/chat/completions`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/66278 or a successor that implements the Chat Completions tool protocol with focused gateway tests and docs, then close this issue after merge.

## Reproduction Assessment

Yes, source-level. A Chat Completions request with client `tools` has no forwarding path on current main, and responses are always stop-shaped assistant text rather than `tool_calls`.

## Solution Assessment

Yes. The narrow maintainable direction is to add the protocol support in the Gateway Chat Completions adapter by reusing the existing client-tool and pending-tool-call seams, as the linked PR attempts.
