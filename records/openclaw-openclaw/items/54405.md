## Summary

Keep open: current main still lacks Chat Completions client-tool parsing and tool-call response mapping, while open PR #66278 explicitly tracks the implementation for this issue.

## What This Changes

Support client-provided function tools on `POST /v1/chat/completions`, including `tools`/`tool_choice`, `tool_calls`, and tool-result follow-up messages.

## Best Possible Solution

Review and land or replace #66278 so the Chat Completions adapter supports the OpenAI function-tool loop and documents the new compatibility contract.

## Reproduction Assessment

Yes, source-reproducible on current main: send `tools` or `tool_choice` to `POST /v1/chat/completions` and the adapter has no parsing or response path for those fields. I did not run a live HA integration loop because this review is read-only.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable path because it targets the Chat Completions adapter and contract tests instead of adding HA-specific behavior. The issue should stay open until that PR lands or maintainers choose a replacement implementation.
