## Summary

Keep open. Current `main` still does not implement Chat Completions client-tool passthrough, and the provided GitHub context shows open unmerged PR #66278 specifically closes this issue, so the right maintainer action is to review that implementation rather than close this issue or start a parallel fix.

## What This Changes

The issue requests OpenAI-compatible client-provided tool passthrough on `/v1/chat/completions`, including request `tools`/`tool_choice`, assistant `tool_calls`, and `role: "tool"` follow-up handling.

## Best Possible Solution

Review and land PR #66278, or an equivalent focused gateway change, so the Chat Completions adapter reuses the existing `clientTools` runtime seam, parses OpenAI Chat function tools and tool choice, emits OpenAI-compatible non-stream and streaming tool-call responses, accepts tool-result follow-up messages, and updates tests and docs for that contract.
