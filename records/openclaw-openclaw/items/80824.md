## Summary

Keep open: this is a focused auth-profile bug-fix PR and current main still has the repeated terminal-refresh path, but the branch needs real behavior proof in a stronger form and should avoid retaining token-bearing OAuth credential fingerprints in memory.

## What This Changes

The branch adds an OAuth-manager terminal refresh-failure cache, threads explicit forced OAuth refresh through auth-profile resolution, removes the Codex app-server `expires: 0` persistence workaround, and adds regression tests.

## Best Possible Solution

Keep the OAuth-manager-level terminal-failure cache and explicit forced-refresh plumbing, but hash the canonical credential snapshot and require concrete after-fix runtime proof before landing.

## Reproduction Assessment

Yes, from source with high confidence: current main reaches `adapter.refreshCredential` again for unchanged unusable OAuth credentials and forces Codex app-server refresh by persisting `expires: 0`. I did not run a live provider repro because this review had to keep the checkout read-only.

## Solution Assessment

Mostly yes: putting the cache in the OAuth manager and threading an explicit force-refresh flag is the narrow owner-boundary fix. The safer version should store a non-secret digest for the credential fingerprint and include stronger real behavior proof before merge.
