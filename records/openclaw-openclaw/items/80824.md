## What This Changes

The PR adds terminal OAuth refresh-failure caching with hashed credential fingerprints, threads forced refresh through auth-profile resolution, removes the Codex app-server temporary `expires: 0` write, and adds focused regression coverage.

## Best Possible Solution

Land one coordinated auth-profile fix after storing only sanitized cached refresh failures while preserving the hashed credential fingerprint cache and explicit forced-refresh path.

## Reproduction Assessment

Yes. Source inspection shows current main lacks a terminal refresh-failure cache and still writes `expires: 0` before forced Codex app-server refresh resolution; the PR body supplies after-fix live Node output for the intended behavior.

## Solution Assessment

Mostly yes. The OAuth manager is the right place to cache terminal refresh failures and the forced-refresh flag belongs on the canonical auth-profile resolver path, but the cached failure should be sanitized before it is retained.
