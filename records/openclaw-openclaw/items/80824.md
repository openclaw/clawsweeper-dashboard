## Summary

Keep this PR open: the terminal refresh-failure cache is useful, but the latest forced-refresh plumbing still has fallback paths that can return existing OAuth credentials without performing the requested refresh.

## What This Changes

The PR adds terminal OAuth refresh-failure caching, threads forced OAuth refresh through auth-profile resolution, and updates Codex app-server/auth-profile tests.

## Best Possible Solution

Keep the failure-cache design, but make `forceOAuthRefresh` consistently bypass usable-token fallbacks unless a credential actually changed due to a concurrent refresh, then land one coordinated fix with the overlapping forced-refresh PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main lacks a terminal failure cache and writes `expires: 0` for forced Codex refresh, while the PR's remaining forced-refresh gaps can be traced through the resolver without running tests.

## Solution Assessment

No. The PR uses the right owner-boundary approach, but it must apply the force-refresh flag to the main-store, external bootstrap, and legacy fallback paths before the solution is complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor force refresh in credential adoption fallbacks:** `src/agents/auth-profiles/oauth-manager.ts:407-412`
  - body: `forceRefresh` skips the first usable-credential fast path, but the locked refresh path can still return a usable main-store credential or external bootstrap credential before calling `adapter.refreshCredential`. A forced refresh through a child, inherited, or external profile can therefore succeed with the existing token and zero refresh attempts; gate those returns on `!params.forceRefresh` or refresh the adopted credential instead.
  - confidence: 0.88
- **[P3] Forward forced refresh into the legacy fallback:** `src/agents/auth-profiles/oauth.ts:397-402`
  - body: After the primary OAuth refresh fails, the legacy-default fallback calls `tryResolveOAuthProfile` without passing `forceOAuthRefresh`. If the fallback profile is usable, a forced refresh can return that cached token instead of refreshing it; pass the force flag through this call as well.
  - confidence: 0.82
