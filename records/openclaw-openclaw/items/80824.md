## Summary

Keep this PR open: it is a focused bug fix with sufficient after-fix live-output proof, but the new refresh-failure cache should not retain raw OAuth credential JSON in memory.

## What This Changes

The PR adds an in-memory terminal OAuth refresh-failure cache for auth profiles and changes Codex app-server forced refresh to use an explicit OAuth force-refresh parameter instead of persisting `expires: 0`.

## Best Possible Solution

Land the OAuth-manager cache and explicit forced-refresh path after replacing the raw credential fingerprint with a deterministic non-secret digest.

## Reproduction Assessment

Yes, at source level: current main refreshes expired OAuth credentials through `adapter.refreshCredential` without a terminal failure cache, and Codex app-server forced refresh mutates stored expiry before resolving. I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes: caching terminal failures in the OAuth manager and threading forced refresh through the existing auth-profile path is the narrow owner-boundary fix. The safer version should store a deterministic non-secret digest instead of the raw serialized credential snapshot.

## Review Findings
