## Summary

Keep open. The PR has useful real Telegram DM proof, but it reverses the current Telegram maintainer decision for answer streaming and still has blocking final-delivery and generated-metadata defects.

## What This Changes

The PR adds opt-in native Telegram `sendMessageDraft` answer streaming for DMs, shared native draft helpers, dispatch/config/docs wiring, and focused tests.

## Best Possible Solution

Keep the default edit-based answer streaming; if Telegram owners approve this feature, land it as an opt-in mode with rendered final delivery, accurate generated metadata, fallback/default compatibility, and proof for DM, group fallback, disabled toggle, and long/tool-call cases.
