## What This Changes

The branch adds opt-in Telegram DM answer streaming via `sendMessageDraft`, shared native draft helpers, config/schema/docs/UI hints, and Telegram draft tests.

## Best Possible Solution

Land native Telegram answer drafts only after the config metadata is aligned and the draft lifecycle has an owner-approved non-durable preview contract with targeted Telegram proof for interruption and exactly-one-final-message behavior.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: enable `channels.telegram.streaming.nativeTransport` for a DM partial stream, then hit tool progress or a non-final delivery; the native draft has no message id until finalization, so existing lane code materializes or falls through incorrectly.

## Solution Assessment

No. The opt-in direction may be viable, but this patch needs generated metadata, a non-durable native draft delivery contract, dispatch-level regression coverage, and Telegram owner approval before it is the maintainable solution.
