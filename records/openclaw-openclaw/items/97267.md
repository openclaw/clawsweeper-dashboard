## Best Possible Solution

Land a narrow extractor fallback that serializes sanitized structured non-image blocks, skips image-only media, and preserves existing sanitizer/error coverage.

## Solution Assessment

Yes, a localized fallback in extractToolResultText is the right layer because the handler already uses that helper for visible tool output. The active PR should keep the fix narrow and preserve unrelated tests.
