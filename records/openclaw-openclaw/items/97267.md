Keep open: current main and the latest release still only extract explicit text blocks for visible tool results, and the linked draft PR is an active but unmerged candidate rather than a completed fix.

## What This Changes

Structured non-image tool-result blocks should remain visible as text in embedded-agent subscription output instead of collapsing to an attachment placeholder.

## Best Possible Solution

Land a focused extractor fallback that serializes sanitized structured non-image blocks, keeps image-only media behavior separate, and preserves existing sanitizer/error coverage.

## Reproduction Assessment

Yes at source level: a structured-only non-image result reaches extractToolResultText, which only collects text blocks and returns undefined. I did not run a live UI/session reproduction in this read-only review.

## Solution Assessment

Yes for the issue direction: the extractor helper is the narrow visible-output boundary because the handler emits only the helper's returned text. The current linked PR still needs normal review before it is the best implementation.
