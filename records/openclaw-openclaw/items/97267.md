## What This Changes

The issue reports that structured non-image tool-result blocks can lose visible text and render as an attachment placeholder in session output.

## Best Possible Solution

Land one narrow helper-boundary fix that serializes structured non-image blocks only when text blocks are absent, keeps image-only behavior separate, adds focused regression coverage, and then close this issue after merge.

Yes at source level: current main returns undefined for structured-only non-image `content` blocks, and the visible-output caller only emits when this helper returns text. I did not run a live rebuilt session repro in this read-only review.
