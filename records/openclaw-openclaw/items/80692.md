## Summary

Current main and the v2026.5.4 release already address the reported Docker packaging regression: externalized channel plugin dist directories are pruned instead of shipped as metadata-only runtime stubs, and missing official external plugins now get install guidance.

## What This Changes

The issue reports that the 2026.5.3-1 Docker image shipped metadata-only Discord, WhatsApp, LINE, QQBot, and Feishu plugin directories, so those channels did not register.

## Best Possible Solution

Keep the v2026.5.4+ packaging model: Docker images should not expose partial external plugin runtime directories, and official external channels should be installed through the catalog/plugin install flow with clear diagnostics.

## Reproduction Assessment

Yes for the historical 2026.5.3-1 image based on the reporter's deterministic container listing. No current-main failing path remains in source because Docker now prunes package-excluded external plugin dist directories before the runtime image is assembled.

## Solution Assessment

Yes: the current fix avoids misleading metadata-only bundled plugin directories while preserving the external-plugin distribution contract and adding install guidance for missing official plugins.
