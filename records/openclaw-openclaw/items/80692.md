## Summary

Current main and the v2026.5.4 release resolved the reported v2026.5.3-1 Docker packaging regression: excluded external channel plugin dist directories are now pruned from Docker runtime images instead of shipping metadata-only directories, and missing official external plugins get install guidance rather than stale-config removal advice.

## What This Changes

The issue reports the v2026.5.3-1 Docker image shipping metadata-only Discord, WhatsApp, LINE, QQBot, and Feishu plugin directories, causing configured channels not to register.

## Best Possible Solution

Keep the v2026.5.4+ packaging model: Docker images should not ship partial metadata-only external plugin dirs, and official external channels should be installed through the catalog/on-demand plugin flow with clear diagnostics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the historical v2026.5.3-1 image from the reporter's deterministic container listing; no current-main failing path remains because current source prunes those package-excluded Docker dist directories and v2026.5.4 shipped that behavior.

## Solution Assessment

Is this the best way to solve the issue? Yes: the implemented fix avoids misleading partial bundled plugin metadata while preserving the external-plugin distribution contract and adding catalog-backed install guidance for missing official channels.
