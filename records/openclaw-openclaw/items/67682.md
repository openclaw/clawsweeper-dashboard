## Summary

Keep open. Current main still only supports user MCP credentials as server-scoped config, and the existing loopback MCP agent-id plumbing does not provide per-agent credentials for external Plane or Feishu-style MCP servers.

## What This Changes

Request per-agent MCP env/credential overrides so different agents can call the same MCP server with agent-specific identities instead of shared server credentials.

## Best Possible Solution

Define a schema-backed per-agent MCP identity overlay for env and headers, resolve it before MCP startup or backend injection, and include the effective credential scope in runtime isolation and redaction tests.

## Reproduction Assessment

Yes. Source inspection on current main shows no per-agent MCP/env config surface, while stdio env and HTTP headers still come from global mcp.servers.* definitions; I did not run a live Plane or Feishu deployment.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but injecting env into an already-started shared process would be risky; the safer design is an effective per-agent MCP config resolved before spawn/connect and backend config generation.
