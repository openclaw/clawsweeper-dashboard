## Summary

Keep open. Current main still exposes MCP credentials only as global server-scoped config, and the corroborating Plane/Feishu reports plus related open items show this remains a credential-sensitive multi-agent design gap rather than an implemented fix or clean duplicate.

## What This Changes

Request per-agent MCP credential/env overrides so MCP writes use the calling agent's identity instead of a shared server credential.

## Best Possible Solution

Design a schema-backed per-agent MCP identity overlay for env and headers, resolve it before MCP startup or backend injection, include it in effective config/cache keys, and cover redaction, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows agent entries do not accept MCP/env overrides, while MCP credentials live under global `mcp.servers.*` and the runtime consumes that global map; I did not run a live Plane or Feishu deployment.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but injecting env into an already-started shared process would be risky; the safer solution is an effective per-agent MCP config with isolated clients/processes and cache keys.
