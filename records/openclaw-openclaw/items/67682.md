## Label Justifications

- `P2`: This is a normal-priority multi-agent configuration and credential-isolation feature with clear source evidence but no urgent regression or outage.
- `impact:security`: The request concerns isolating credentials and audit identity between agents that share MCP infrastructure.
- `impact:auth-provider`: The feature would change how MCP API keys, headers, SecretRefs, and runtime credential choices are routed per agent.

## Summary

Keep open. Current main still has server-scoped MCP env/header credentials and a Codex-only projection filter, but no schema-backed per-agent MCP credential overlay; the related per-agent env and MCP scoping issues are adjacent, open, and do not fully replace this request.

## What This Changes

Request support for per-agent MCP server env/header credential overrides so MCP writes are attributed to the calling agent instead of a shared account.

## Best Possible Solution

Define a public, schema-backed per-agent MCP identity overlay that composes with the broader per-agent env and MCP server-scope work, then resolve the effective env/header scope before server spawn/connect, cataloging, caching, and backend injection.

## Reproduction Assessment

Yes for the feature gap: current main has no per-agent MCP/env override slot in the strict agent schema, while MCP credentials still live on global `mcp.servers.*` entries. I did not run a live Plane or Feishu deployment.

## Solution Assessment

Unclear because there is no approved patch or config contract yet. The requested direction is valid, but the safest solution is resolving an effective per-agent MCP config before spawn/connect and backend injection rather than mutating credentials on a shared running process.
