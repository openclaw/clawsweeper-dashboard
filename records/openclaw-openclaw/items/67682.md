## Summary

Keep open. Current main still exposes MCP credentials at the global `mcp.servers.*` level and has no per-agent MCP credential overlay; related open issues cover adjacent env scoping, server scoping, and caller-context headers, but none fully owns this MCP credential-attribution request.

## What This Changes

The issue requests per-agent MCP env/header credential injection so multi-agent deployments can attribute MCP writes to the active agent identity instead of a shared account.

## Best Possible Solution

Define a schema-backed per-agent MCP identity overlay that composes with https://github.com/openclaw/openclaw/issues/80698, https://github.com/openclaw/openclaw/issues/72591, and https://github.com/openclaw/openclaw/issues/75116, then resolve the effective env/header scope before MCP startup, connection, cataloging, caching, and backend injection.

## Reproduction Assessment

Yes for the feature gap: current main has a strict agent schema with no per-agent MCP/env slot, while MCP env and headers are still defined on global `mcp.servers.*` entries. I did not run a live Plane or Feishu deployment.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but the safer implementation is to resolve an effective per-agent MCP config before spawn/connect and backend injection rather than mutating credentials on a shared running process.

## Security Review

Status: needs_attention

Summary: This non-PR issue is credential-sensitive because it changes cross-agent MCP identity, secret isolation, and runtime reuse semantics.

Concerns:

- **[medium] Include credentials in MCP runtime isolation:** `src/agents/pi-bundle-mcp-runtime.ts:142`
  - body: The runtime fingerprints `loaded.mcpServers` and reuses session runtime state from that shared map; any agent-scoped env/header overlay must participate in the effective config or runtime key so one agent's credentialed MCP connection cannot be reused for another agent.
  - confidence: 0.9
- **[medium] Redact new agent-scoped MCP secret paths:** `src/config/zod-schema.ts:341`
  - body: Current MCP secret-like values live under `mcp.servers.*.env` and sensitive `mcp.servers.*.headers`; any new `agents.list[]` MCP credential path needs matching schema hints, SecretRef/redaction coverage, diagnostics, and snapshot protection.
  - confidence: 0.86
- **[medium] Filter backend MCP config before injection:** `src/agents/cli-runner/bundle-mcp.ts:171`
  - body: CLI adapters serialize or inject the merged MCP server map for downstream clients, so per-agent credentials must be resolved before those files or overlays are written or passed to Claude, Codex, Gemini, or similar backends.
  - confidence: 0.84
