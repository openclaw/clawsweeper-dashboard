## Summary

Keep open. Current main still has only top-level MCP server credentials and no agent-qualified env/header overlay; the related MCP tool-filtering and native-MCP discussions do not cover same-server identity attribution.

## What This Changes

Add per-agent MCP credential overrides so multi-agent deployments can authenticate to the same MCP-backed tool with agent-specific identity.

## Best Possible Solution

Add a schema-backed per-agent MCP identity overlay that resolves secrets, merges into an effective per-agent MCP config, and creates isolated runtime/cache entries for each credential set.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: the proposed `agents.list[].mcp.overrides` shape is absent from the strict agent schema, and the runtime merge/cache paths only consume top-level `cfg.mcp.servers`.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but mutating env per call on an existing shared MCP process would be unsafe; the maintainable solution is an effective per-agent overlay with isolated clients/processes or cache keys.
