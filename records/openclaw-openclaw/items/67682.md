## Summary

Keep open. Current main still exposes MCP credentials as global server configuration under `mcp.servers`; the agent schema, agent type contract, MCP merge path, and session MCP runtime cache do not provide an agent-qualified env/header overlay for same-server identity attribution.

## What This Changes

The issue requests per-agent MCP server credential overrides so the same MCP server can attribute writes to the active agent identity instead of a shared global account.

## Best Possible Solution

Add a schema-backed, redacted per-agent MCP identity overlay for env and headers, merge it into the effective MCP server config before embedded Pi and CLI backend MCP creation, include the effective overlay in runtime/cache fingerprints or use separate clients/processes, and document the supported SecretRef/redaction behavior with regression coverage for two agents using the same MCP server id with different credentials.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: the proposed `agents.list[].mcp.overrides` shape is not accepted by the strict agent schema, and the MCP runtime merge path reads only `cfg.mcp.servers` without agent input. Observing the Plane attribution symptom live would need a Plane MCP server, but the missing core config/runtime path is clear.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but the best maintainable fix needs maintainer design for config shape, SecretRef/redaction support, and MCP runtime lifecycle isolation.
