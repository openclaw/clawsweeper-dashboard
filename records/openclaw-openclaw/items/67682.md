## Summary

Keep open. Current main still has only global `mcp.servers.*` env/header credentials and no per-agent MCP identity overlay; the related open per-agent env and MCP server-scope issues are adjacent but do not fully replace this MCP credential-attribution request.

## What This Changes

Request support for per-agent MCP environment or credential overrides so writes through shared MCP servers are attributed to the active agent identity.

## Best Possible Solution

Define a schema-backed per-agent MCP identity overlay that composes with https://github.com/openclaw/openclaw/issues/80698 and https://github.com/openclaw/openclaw/issues/72591, then resolve the effective env/header scope before MCP startup, connection, cataloging, caching, and backend injection.

## Reproduction Assessment

Yes for the feature gap: current main has a strict agent schema with no per-agent MCP/env slot, while MCP env and headers are still defined on global `mcp.servers.*` entries. I did not run a live Plane or Feishu deployment.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but the safest design is resolving an effective per-agent MCP config before spawn/connect and backend injection rather than mutating credentials on a shared running process.
