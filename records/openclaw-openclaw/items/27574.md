## Best Possible Solution

Design an admin-scoped, read-only-first Control UI browser preview that reuses browser.request/browser-control where safe, gates write actions explicitly, and coordinates with https://github.com/openclaw/openclaw/issues/63926 plus the plugin UI descriptor direction.

## Reproduction Assessment

Yes at the source level: current main and v2026.6.6 have no Control UI browser route, renderer, or documented live preview panel while browser-control primitives exist. This is a missing feature, not a runtime failure needing live reproduction.

## Solution Assessment

No, not as sketched alone. The panel idea is useful, but the best maintainable solution needs an admin-scoped read-only default, explicit write-action handoff, and a decision on whether browser UI should use a generic plugin contribution surface.
