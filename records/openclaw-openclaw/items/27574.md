## What This Changes

The issue requests a live Control UI browser preview panel with screenshots, URL/title, console errors, and optional manual browser interaction for remote/VPS users.

## Best Possible Solution

Design an admin-scoped, read-only-first Control UI browser preview that reuses browser.request/browser-control safely, gates write actions explicitly, and coordinates with the open login-handoff and plugin UI contribution work.

## Reproduction Assessment

Yes at the source level: current main and v2026.6.8 have no Control UI browser route, renderer, descriptor consumer, or documented live preview panel while browser-control primitives exist. This is a missing feature, not a runtime failure needing live reproduction.

## Solution Assessment

Unclear as proposed: a panel is useful, but the maintainable solution needs admin scoping, a read-only default, explicit write-action handoff, and a decision on hardcoded browser UI versus a plugin contribution surface.
