## Summary

Keep open: current main and v2026.6.8 still provide browser-control primitives and a static Control UI, but no live Dashboard/Control UI browser preview panel. Related open items cover login handoff, generic plugin UI entry points, and chat/canvas visualization, but none replaces this broader browser-preview tracker.

## What This Changes

Request to add a live Dashboard or Control UI browser preview panel showing remote browser pixels, URL/title, console errors, and optional manual interaction.

## Best Possible Solution

Design an admin-scoped, read-only-first browser preview surface that reuses browser.request/browser-control primitives safely, gates write actions explicitly, and coordinates with the open browser handoff and plugin UI entry-point work.

## Reproduction Assessment

Yes, at the source level: current main and v2026.6.8 have no Control UI browser tab, route, live preview renderer, or documented browser-preview panel while browser-control primitives exist. This is a missing feature rather than a runtime bug needing live reproduction.

## Solution Assessment

Unclear as proposed: a live panel is useful, but the best implementation needs admin scoping, read-only defaults, explicit write-action gates, redaction/retention policy, and a decision on hardcoded browser UI versus a plugin contribution surface.
