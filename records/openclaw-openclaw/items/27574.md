## Summary

Keep open: current main still lacks a Control UI/Dashboard browser preview surface, while the related browser-login handoff tracker at https://github.com/openclaw/openclaw/issues/63926 is open and narrower. The May 13 FSB reference usefully supports a read-only live view separated from explicit write actions, but it does not make the OpenClaw implementation complete.

## What This Changes

Add a live Browser Preview Panel to the Dashboard/Control UI so remote users can watch the built-in browser, inspect URL/title/console state, and optionally control it.

## Best Possible Solution

Ship a security-reviewed, admin-scoped Browser area in Control UI that reuses browser.request or additive typed gateway methods for live read-only state, with explicit operator steps for navigation, clicks, credential fields, and final submit actions.

## Reproduction Assessment

Not applicable as a runtime bug because this is a feature request. Source inspection gives high confidence that current main has browser-control APIs but no Control UI browser tab, route, render branch, or live preview consumer.

## Solution Assessment

Unclear until maintainers scope the UX and security model. The requested direction is useful, but the maintainable path should coordinate with https://github.com/openclaw/openclaw/issues/63926 and the plugin UI descriptor surface instead of a narrow automated patch.
