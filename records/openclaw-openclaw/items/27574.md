## Risks / Open Questions

- A live browser preview can expose authenticated page pixels, URLs, titles, console output, tabs, and logged-in session context, so auth scope, redaction, retention, audit, and remote-access policy need review.
- Optional write actions such as clicks, navigation, credential entry, and final submit cross from visibility into browser control and need an explicit product/security boundary.
- The narrower tracker at https://github.com/openclaw/openclaw/issues/63926 and the plugin contribution-slot RFC at https://github.com/openclaw/openclaw/issues/71736 overlap this work, so maintainers should define the shared UI/API boundary before implementation.
