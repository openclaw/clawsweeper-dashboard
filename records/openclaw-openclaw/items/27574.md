Keep open: current main and v2026.6.10 still provide browser-control primitives and a narrow plugin UI descriptor surface, but no live Browser Preview Panel in the Dashboard/Control UI. Request to add a live Dashboard/Control UI panel showing the managed browser's URL/title, screenshots, console errors, and optional interactive controls for remote/container users.

## Best Possible Solution

Design an admin-scoped, read-only-first browser preview surface that reuses browser.request/browser-control primitives, gates write actions explicitly, and coordinates with the browser handoff and plugin UI entry-point work.
