## Summary

Keep open: current main and v2026.6.8 still have browser-control primitives and chat-side workspace/file browsing, but no Dashboard or Control UI live browser preview panel. The related open items are useful context, especially https://github.com/openclaw/openclaw/issues/63926 and https://github.com/openclaw/openclaw/issues/66944, but they do not replace this broader remote browser-preview request.

## What This Changes

Request to add a live Dashboard/Control UI browser preview showing the built-in browser view, URL/title, console errors, and optional manual controls for remote users.

## Best Possible Solution

Design an admin-scoped, read-only-first Control UI browser preview that reuses browser.request/browser-control primitives safely, gates write actions explicitly, and coordinates with the open login-handoff and plugin UI entry-point work.

## Reproduction Assessment

Yes at the source level: current main and v2026.6.8 have no Control UI browser tab, route, live preview renderer, or documented browser-preview panel while browser-control primitives exist. This is a missing feature rather than a runtime bug needing live reproduction.

## Solution Assessment

Unclear as proposed: a panel is useful, but the best solution needs admin scoping, a read-only default, explicit write-action gates, redaction/retention policy, and a decision on hardcoded browser UI versus a plugin contribution surface.
