## Summary

Keep open: current main still lacks a live Control UI browser preview panel. The browser plugin has useful backend primitives and OpenClaw has an adjacent UI descriptor hook, but there is no browser tab/path, render branch, or Control UI consumer; https://github.com/openclaw/openclaw/issues/63926 remains an open narrower login-handoff companion, not a replacement.

## What This Changes

Requests a live Dashboard/Control UI browser preview showing managed-browser screenshots, URL/title, console errors, and optional interactions for remote/VPS browser sessions.

## Best Possible Solution

Ship an admin-scoped Browser area in Control UI backed by browser.request or a typed browser Gateway method, with near-live screenshots, tab/status metadata, console/errors, safe tab actions, docs, and coordination with the open handoff and plugin-UI design tracks.
