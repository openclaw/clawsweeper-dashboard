## Summary

Keep open: current main and the latest release still have browser-control primitives and a narrow plugin UI descriptor surface, but no live Browser Preview Panel in the Dashboard/Control UI. The related open items cover login handoff, generic plugin UI entry points, and chat/canvas visualization, but none fully replaces this broader remote browser-preview tracker.

## What This Changes

The issue requests a live Dashboard/Control UI browser preview panel for remote browser visibility, current URL/title, console errors, screenshots, and optional interaction.

## Best Possible Solution

Design an admin-scoped, read-only-first browser preview surface that reuses browser.request/browser-control primitives, gates write actions explicitly, and coordinates with the browser handoff and plugin UI entry-point work.

## Reproduction Assessment

Yes, at the source level: current main and v2026.6.10 have no Control UI browser route, tab, or live preview renderer while browser-control primitives exist. This is a missing feature rather than a runtime bug needing live reproduction.

## Solution Assessment

Unclear as proposed: a live panel is useful, but the best implementation needs maintainer decisions on admin scope, read-only defaults, write-action gates, redaction/retention, and hardcoded core UI versus a plugin contribution surface.
