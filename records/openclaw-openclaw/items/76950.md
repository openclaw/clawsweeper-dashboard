## Summary

Keep open: the PR targets a real oversized-session/context-engine failure class, but the current head still has blocking correctness issues in operator recovery text and transcript selection, plus a likely wrong auto-history directory heuristic.

## What This Changes

The PR adds a context-engine fallback and gateway boot session-size guard, new `session.maintenance.contextFallbackGuard` config/schema/help/changelog entries, agent-id plumbing, and unit coverage.

## Best Possible Solution

Land a reviewed version that uses existing session artifact helpers, points operators to real recovery commands, verifies the auto-history marker location, and leaves the broader transcript-size cap follow-ups tracked separately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Replace the nonexistent sessions archive command:** `src/context-engine/fallback-guard.ts:516`
  - body: The warn-mode recovery block still tells operators to run `openclaw sessions archive ...`, but the current CLI/docs only define `openclaw sessions cleanup` and `export-trajectory`. In the path where the guard only warns, that sends users to a command that fails, so either point to an existing recovery path or add and document the command.
  - confidence: 0.92
- **[P2] Reuse the session transcript artifact classifier:** `src/context-engine/fallback-guard.ts:157-176`
  - body: The custom `isLiveSessionTranscript()` filter treats `*.trajectory.jsonl` and compaction checkpoint files as live transcripts, so archive mode can rotate diagnostic/checkpoint artifacts, while valid primary names containing `.deleted` are skipped. Reuse `isPrimarySessionTranscriptFileName()` and layer only the new no-context-engine archive exclusion on top.
  - confidence: 0.89
- **[P2] Check the agent state directory for engine history:** `src/context-engine/fallback-guard.ts:147-150`
  - body: From `~/.openclaw/agents/<id>/sessions`, this line climbs to `~/.openclaw`, but the heuristic says it is checking the agent state directory for engine DB markers. That makes `action: "auto"` miss per-agent context-engine stores and choose `warn` instead of the intended archive path for the motivating case.
  - confidence: 0.8
- **[P3] Include blocked transcript paths in the resolver error:** `src/context-engine/registry.ts:546-555`
  - body: The block path computes `blocked` but only reports the count and threshold in the thrown error. If this is what reaches the operator, they still do not know which transcript to rotate, so include sanitized basenames or shortened paths in the error.
  - confidence: 0.82
