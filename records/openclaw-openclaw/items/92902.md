## What This Changes

The PR makes channel schema metadata collection preserve same-origin replacement schemas unless an incoming channel config's `preferOver` names the current schema owner, with Slack regression coverage for both registry orders and competing replacements.

## Best Possible Solution

Land this PR after maintainers accept `preferOver` as the equal-origin schema ownership signal and required proof/check state aligns; if that contract is rejected, keep https://github.com/openclaw/openclaw/issues/92884 open for a different deterministic collector rule.
