## Summary

Keep open. Current main and the latest v2026.6.5 release still force `openai/gpt-5.3-codex` through the ChatGPT transport, and persisted npm installs can still outrank the bundled plugin; the two open linked PRs cover the immediate routing and diagnostic slices but have not merged, and the stale-plugin upgrade behavior remains unresolved.

## What This Changes

Restore `openai/gpt-5.3-codex` for standard API-key users, prevent stale installed Codex plugins from causing incompatible harness routing, and make the resulting fallback reason actionable.

## Best Possible Solution

Make OpenAI transport resolution follow the active authentication mode, cover both standard API-key and ChatGPT OAuth paths, and add a narrow doctor or loader rule that detects and repairs obsolete installed copies of bundled official plugins without changing normal third-party override semantics.

## Reproduction Assessment

Yes at high confidence from source and supplied live logs: current main retains the forced ChatGPT transport and the persisted installed-plugin precedence required for the reported failure, although this review did not run a fresh credentialed end-to-end reproduction.
