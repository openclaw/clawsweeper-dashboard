## Summary

Keep open: current main still exposes the reported Gateway/TUI title path, and this PR is useful but incomplete against current main because it lacks external real behavior proof and only threads displayName through the stale session-list path.

## What This Changes

The PR strips inbound metadata before transcript title extraction, adds an optional external displayName parameter to deriveSessionTitle, passes it through one session-list call site, and adds a displayName precedence unit test.

## Best Possible Solution

Land a rebased or replacement Gateway title fix that strips inbound metadata during title extraction, passes computed row displayName into both sync and async derived-title hydration, and includes focused regression coverage plus redacted real TUI/Gateway proof.

## Reproduction Assessment

Yes, by source inspection: current main requests derived titles in the TUI, displays them before displayName, extracts first user transcript text without inbound metadata stripping, and ignores computed row displayName in sync and async title hydration.

## Solution Assessment

No, not as submitted. The sanitizer and displayName-precedence direction is right, but the patch needs to be rebased or replaced so buildGatewaySessionRow and listSessionsFromStoreAsync also route the computed displayName.
