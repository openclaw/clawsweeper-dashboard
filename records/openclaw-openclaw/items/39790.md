## Summary

Keep open: the reported gateway/TUI title bug is valid and this PR moves in the right direction, but the branch needs repair before merge because current main derives titles through row-builder and async paths that the stale patch does not cover.

## What This Changes

The PR strips inbound metadata before extracting transcript title text, adds an optional external displayName to `deriveSessionTitle`, passes that displayName from session listing, and adds a displayName precedence test.

## Best Possible Solution

Land a rebased fix that uses the shared inbound metadata stripper for transcript title extraction and routes the row-level displayName through every current derived-title path, with sync and async regression coverage.

## Reproduction Assessment

Yes, source-reproducible. On current main, metadata-prefixed transcript text is returned unchanged by `extractTextFromContent`, and both derived-title call sites ignore the row displayName computed for gateway session rows.

## Solution Assessment

No. The proposed direction is correct, but this branch updates an old sync call site; the narrow maintainable fix needs to apply the same displayName precedence to `buildGatewaySessionRow` and `listSessionsFromStoreAsync`.
