## What This Changes

The branch strips inbound metadata before session-title text extraction, adds an optional external displayName argument to deriveSessionTitle, passes it from the old session-list call site, and adds one displayName precedence test.

## Best Possible Solution

Land a rebased fix that strips inbound metadata during transcript title extraction and prefers the computed row displayName in both sync and async session-list title hydration, with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes by source inspection. Current main's TUI requests derived titles and renders them before displayName, while the gateway extractor strips only directive tags and both row-title paths call deriveSessionTitle without the computed row displayName.

## Solution Assessment

No as submitted. The direction is right, but the branch updates an old call site; the narrow maintainable fix needs to route the computed row displayName through buildGatewaySessionRow and listSessionsFromStoreAsync and add metadata-stripping regression coverage.
