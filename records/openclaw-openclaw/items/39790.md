## What This Changes

The PR strips inbound metadata before Gateway transcript title extraction, adds an optional external displayName parameter to `deriveSessionTitle`, threads it through one older session-list path, and adds a displayName precedence test.

## Best Possible Solution

Land a rebased or replacement Gateway title fix that strips inbound metadata during transcript title extraction, passes computed row displayName into both sync and async derived-title hydration, and adds focused regression coverage with real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays `derivedTitle` before `displayName`, extracts first user transcript text without inbound metadata stripping, and ignores computed row display names in title hydration.

## Solution Assessment
