## Summary

Keep open: this PR targets a real linked TUI/gateway title bug, but current main moved title hydration into the row builders, so the branch now patches an obsolete call site and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR strips inbound metadata before transcript title extraction, adds an optional external displayName parameter to deriveSessionTitle, passes that value from the old session-list path, and adds one displayName precedence test.

## Best Possible Solution

Land a rebased fix that strips inbound metadata during title extraction and prefers the computed row displayName in both sync and async session-row title paths, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the TUI requests derived titles and displays them before displayName, while transcript title extraction does not strip inbound metadata and both current title hydration paths ignore the computed row displayName.

## Solution Assessment

No. The proposed direction is correct, but the submitted diff targets an old list call site; the maintainable fix needs to route the computed displayName through current buildGatewaySessionRow and listSessionsFromStoreAsync title hydration.
