## What This Changes

This PR strips inbound metadata before transcript title extraction, passes an external session display name into title derivation, and adds display-name precedence coverage.

## Best Possible Solution

Land a current-main Gateway fix that sanitizes transcript title candidates and makes every sessions-list derivedTitle path prefer the computed displayName before transcript text, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. The TUI requests derived titles, current main derives them from unsanitized firstUserMessage text, and the async session-list path renders derivedTitle before the computed displayName.

## Solution Assessment

Yes, with changes. Reusing stripInboundMetadata and forwarding the computed displayName is the narrowest maintainable fix, but the current PR must also update the async sessions-list path and add coverage/changelog before merge.
