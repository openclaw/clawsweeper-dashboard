## Summary

Keep this PR open. Current main still lacks both intended behaviors: gateway transcript title extraction still does not strip inbound metadata, and the derived-title path still ignores the computed outer row displayName. The PR is a narrow implementation candidate for the linked bug, with only test/rebase coordination left for maintainers.

## What This Changes

The PR strips inbound metadata before gateway session title text extraction, passes an external displayName into deriveSessionTitle, and adds a displayName precedence test.

## Best Possible Solution

Land this PR or a successor after adding focused regression coverage for metadata-stripped title extraction and coordinating with #48942. The desired end state is that /sessions uses a trusted display name when available and only falls back to sanitized user-message text.

## Reproduction Assessment

Yes. The source path is high-confidence: /sessions requests derived titles, buildGatewaySessionRow calls deriveSessionTitle without the computed row displayName, and extractTextFromContent currently returns unstripped inbound metadata as the first user message.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: reuse the existing inbound metadata sanitizer and pass the already-computed row displayName into the title derivation boundary. The safer completion is to add the missing sanitizer regression test and coordinate with the open title-test refactor.
