## Summary

Keep open: this is a valid Gateway/TUI bug fix direction, but the external PR lacks required real behavior proof and, as submitted, only updates an older session-list path while current main also derives titles through sync row building and async hydration.

## What This Changes

The PR imports `stripInboundMetadata` into transcript title extraction, adds an optional external `displayName` argument to `deriveSessionTitle`, threads it through the older `listSessionsFromStore` call, and adds one displayName precedence unit test.

## Best Possible Solution

Land a rebased or replacement Gateway title fix that strips inbound metadata during title extraction, passes computed row displayName into both sync and async derived-title hydration, and includes focused regression coverage plus redacted real TUI/Gateway proof.

## Reproduction Assessment

Yes, by source inspection. Current main requests derived titles in the TUI, displays them before `displayName`, extracts first user transcript text without inbound metadata stripping, and ignores computed row display names in title hydration.

## Solution Assessment

No, not as submitted. The sanitizer and displayName-precedence direction is right, but the patch must be rebased or replaced to cover `buildGatewaySessionRow` and `listSessionsFromStoreAsync` before merge.
