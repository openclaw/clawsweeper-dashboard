## Summary

Keep open: the PR targets a real gateway/TUI session-title bug, but the branch is stale against current main and updates an old call site instead of the current sync and async row-title paths. It also lacks after-fix real behavior proof from a real TUI/gateway setup, so it is not merge-ready.

## What This Changes

The branch strips inbound metadata before first-message title extraction, adds an optional external displayName parameter to deriveSessionTitle, passes that value at the old sync list call site, and adds one displayName precedence test.

## Best Possible Solution

Land a rebased fix that strips inbound metadata during title extraction and feeds the computed row displayName into both buildGatewaySessionRow and listSessionsFromStoreAsync, with focused sync/async regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows /sessions asks for derived titles and displays them before displayName, while title extraction returns first user text after only directive-tag stripping and both current row-title paths ignore the computed row displayName; I did not run a live TUI/gateway reproduction in this read-only review.

## Solution Assessment

No. The proposed direction is right, but the submitted patch updates an old list call site; the maintainable fix needs to route the computed row displayName through both current sync and async title derivation paths and cover metadata stripping with regression tests.
