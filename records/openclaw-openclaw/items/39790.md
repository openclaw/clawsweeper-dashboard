Keep open: this is a real gateway/TUI title bug and the PR’s core direction is useful, but the branch is stale/conflicting against current main, misses the current sync/async row-title paths, and lacks real behavior proof.

## What This Changes

The branch imports `stripInboundMetadata` into gateway title extraction, adds an optional displayName argument to `deriveSessionTitle`, passes the outer session displayName from the old list path, and adds one displayName precedence test.

## Best Possible Solution

Land a rebased fix that strips inbound metadata during title extraction and prefers the computed row displayName in both `buildGatewaySessionRow` and `listSessionsFromStoreAsync`, with focused sync/async regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `/sessions` requests derived titles and displays them before displayName, while title extraction returns first user text after only directive-tag stripping and both current derived-title paths ignore the computed row displayName.

## Solution Assessment

No. The proposed direction is correct, but the branch updates an old list call site; the narrow maintainable fix needs to route row displayName through current sync and async row-title paths and add metadata-stripping regression coverage.
