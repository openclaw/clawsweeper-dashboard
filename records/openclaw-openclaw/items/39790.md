## Summary

Keep open: this PR targets a real gateway/TUI title bug tracked at https://github.com/openclaw/openclaw/issues/39722, but the branch is now conflicting and only updates an old list path while current main derives titles through sync and async row paths. It also still lacks required after-fix real behavior proof from a real TUI/gateway setup.

## What This Changes

The PR strips inbound metadata before transcript title extraction, adds an optional external displayName argument to `deriveSessionTitle`, passes the outer displayName from the old list path, and adds one displayName precedence test.

## Best Possible Solution

Land a rebased fix that strips inbound metadata during transcript title extraction and prefers the computed row displayName in both `buildGatewaySessionRow` and `listSessionsFromStoreAsync`, with focused sync/async regression coverage and redacted real behavior proof.

Yes. ## Solution Assessment
