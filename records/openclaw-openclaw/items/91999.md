## Summary

Current `main` and release `v2026.6.6` already preserve user-selected session model overrides through the rollover path that was reverting to the configured Gemini default, and the same persisted override contract is used by `session_status(model=...)`.

## What This Changes

The issue reports that the `main` terminal session reverts from a `session_status(model=...)` model selection back to `google/gemini-2.5-flash` on the next turn.

## Best Possible Solution

Keep the shipped shared session-preservation implementation as the canonical fix rather than adding a terminal-only workaround.

## Reproduction Assessment

Yes at source level: the merged fix and current regression test exercise the stale rollover path where a user model override was dropped after an acknowledgement. I did not run a live terminal session in this read-only review.

## Solution Assessment

Yes. Reusing the shared session-preservation helper is the narrow maintainable fix because `/model`, `session_status(model=...)`, and `sessions.patch` all persist the same user override state.
