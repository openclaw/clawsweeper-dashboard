## Summary

Current main and the latest release already preserve user-selected session model overrides across the rollover path that was dropping them after a successful model-change acknowledgement, including overrides written by `session_status(model=...)`.

## What This Changes

The issue reports that terminal sessions for the main agent revert from a session-selected model back to Gemini after each turn.

## Best Possible Solution

Keep the shipped shared session rollover fix as the canonical implementation; no separate terminal-only workaround is needed.

## Reproduction Assessment

## Solution Assessment

Yes. The merged fix reuses the shared session-preservation helper in `initSessionState`, which is the narrow maintainable layer because `/model`, `session_status(model=...)`, and `sessions.patch` all persist the same user override state.
