The issue reports that terminal sessions for the main agent reset from a session-selected `opencode-go/minimax-m2.7` model back to `google/gemini-2.5-flash` after each turn.

## Best Possible Solution

Review or extend the existing explicit-default model-pin work so terminal `session_status(model=...)` creates a durable session selection and triggers the live switch when the chosen model equals the configured default.

## Reproduction Assessment

Yes, partially from source: the report provides concrete terminal steps, and current code shows the configured-default model selection path can clear overrides without reliably marking a live switch. I did not run the terminal flow, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes: the best fix belongs in the existing session-entry model override/live-switch path, likely by preserving explicit configured-default selections as user session pins or otherwise marking the live switch. A new config option or provider-specific Gemini/OpenCode special case would be the wrong layer.
