## What This Changes

Request to add the active provider/model name to the per-response `/usage` footer in messaging channels.

## Best Possible Solution

Consolidate the open implementations into one narrow usage-footer change that renders the actual resolved provider/model while preserving token, cache, cost, and session semantics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but source inspection is high confidence: the current formatter has no provider/model inputs and the runner does not pass its resolved provider/model to the footer call.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the public shape; the narrow maintainable path is to add resolved provider/model to the existing usage-footer path without broadening defaults or footer templating unless that product direction is chosen.
