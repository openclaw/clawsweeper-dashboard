## Summary

Keep open. Current main still builds the per-response usage footer from token/cache/cost/session data only, while the reply runner already has the actual provider and model; the overlapping open PRs should be reviewed or consolidated instead of closing this issue or spawning another repair job.

## What This Changes

Request to include the active model label in the per-response `/usage` footer so users can see which model answered each reply.

## Best Possible Solution

Add actual provider/model attribution to the existing response-usage footer path, using `providerUsed` and `modelUsed` from `runReplyAgent`, with focused formatter and runner tests preserving current token/cache/cost/session behavior.
