## Summary

Current main has partial recovery for official externalized channels, including WhatsApp, but the reported gateway/send failure after upgrade is not proven fixed. The source still has a clear path where an explicit WhatsApp send fails with `Channel is unavailable: whatsapp` if the plugin is absent, while the merged recovery work mainly covers setup/onboard and doctor repair paths.

## What This Changes

Upgrade should preserve or repair configured WhatsApp plugin availability, or emit a direct repair path, when `channels.whatsapp.enabled` survives an upgrade but `@openclaw/whatsapp` is missing.

## Best Possible Solution

Make the upgrade or doctor path reconcile configured official external channel plugins such as WhatsApp after the package swap, and ensure gateway/send surfaces the catalog install hint when the plugin is absent.

## Reproduction Assessment

Yes from source for the failure class, but not from a live upgrade run: if WhatsApp is requested and no channel plugin is loaded or installed, current channel selection throws `Channel is unavailable: whatsapp`. I did not reproduce a full 2026.4.23 to current-main npm-global WSL2 upgrade.

## Solution Assessment

Unclear. The existing setup/onboard recovery and doctor repair paths are useful, but they do not demonstrate that gateway startup or an explicit send from preserved `channels.whatsapp.enabled` config will automatically repair or show the install hint after upgrade.
