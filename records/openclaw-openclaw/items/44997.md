## Summary

Keep this PR open, but do not merge it unchanged. The config race behind #43367 is still source-reproducible on current main, and the transform-under-lock design is the right direction, but the branch is stale and would need to preserve newer config writer and pending plugin-install contracts before it is safe.

## What This Changes

The PR adds an advisory config write lock, exports `updateConfigFile(transform)`, and moves non-interactive `agents add` mutations under a locked transform.

## Best Possible Solution

Land a current-main implementation that provides a transform-based locked config commit path for read-modify-write cases, updates non-interactive `agents add`, preserves plugin-install and runtime writer contracts, and leaves the broader #43367 failures for separate work.

## Reproduction Assessment

Yes, by source inspection. Current main still computes non-interactive `agents add` from one initial snapshot and writes with an optimistic `baseHash`, so concurrent adds can conflict instead of all being serialized and persisted.

## Solution Assessment

No, not as this branch currently stands. The transform-under-lock design is the narrow maintainable fix, but it needs to be rebuilt on current main without replacing newer config writer behavior or bypassing pending plugin-install commits.
