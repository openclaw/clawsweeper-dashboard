## Summary

Keep this PR open, but do not merge the branch as-is. The transform-under-lock direction is a valid narrow fix for concurrent `agents add`, while the branch is stale against current main and would need to preserve newer config writer and plugin-install commit contracts.

## What This Changes

The PR adds an advisory config write lock, exports `updateConfigFile(transform)`, and switches non-interactive `agents add` to compute agent and binding changes from a freshly locked config snapshot.

## Best Possible Solution

Land a current-main implementation that provides a transform-based locked config commit path for read-modify-write cases, updates non-interactive `agents add`, preserves plugin-install and runtime writer contracts, and leaves the broader session/OAuth races in #43367 for separate work.

## Reproduction Assessment

Yes. Source inspection shows current main still computes non-interactive `agents add` from one initial snapshot and commits with a base hash; a live parallel CLI run was not executed in this read-only review, so this is source-reproducible rather than directly reproduced here.

## Solution Assessment

No, not as this branch currently stands. The transform-under-lock design is the right narrow solution, but it needs to be rebuilt on current main without replacing newer config writer behavior or bypassing pending plugin-install commits.
