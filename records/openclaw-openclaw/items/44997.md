## Summary

Keep open: this is a plausible bug-fix PR, but it is not safe to merge or close as cleanup. Current main still has a source-visible concurrent `agents add` config mutation race, this branch conflicts with newer config mutation code, a newer current-base PR now tracks the same user problem, and this PR lacks after-fix real behavior proof.

## What This Changes

The PR adds a config-file advisory lock plus an `updateConfigFile` transform API and switches non-interactive `agents add` to mutate a fresh config under that lock.

## Best Possible Solution

Choose one current-base fix for concurrent `agents add`: either adapt the lock/transform idea into the current config mutation contracts, or land a corrected retry-based PR such as https://github.com/openclaw/openclaw/pull/76601 with regression coverage and real parallel CLI proof.

No, not as-is. The lock/transform direction addresses lost updates, but this branch is built on older config IO internals and should be reworked against the current `mutateConfigFile`/`replaceConfigFile` design or superseded by the newer current-base PR.
