## Summary

Keep open. Current main still has no Bedrock `requestMetadata` config/schema/runtime/docs path, and the open related work at https://github.com/openclaw/openclaw/issues/60602 and https://github.com/openclaw/openclaw/pull/60698 covers automatic per-agent metadata rather than this static configurable metadata request.

## What This Changes

Expose AWS Bedrock `requestMetadata` through OpenClaw configuration so operators can tag Bedrock `ConverseStream` calls for cost allocation.

## Best Possible Solution

Add a Bedrock plugin-owned `requestMetadata` config surface with AWS-limit validation, deterministic merge semantics with any automatic per-agent metadata, focused runtime tests, and public docs.

## Solution Assessment
