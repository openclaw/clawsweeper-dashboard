Keep open. Current main still lacks a Bedrock requestMetadata config/schema/runtime/docs path, and the related open Bedrock attribution issue/PR cover automatic per-agent metadata rather than this static provider/plugin config request.

Expose Bedrock requestMetadata in OpenClaw configuration so users can tag Bedrock Converse requests for AWS cost allocation.

## Best Possible Solution

Add a plugin-owned Bedrock requestMetadata config with AWS-limit validation, deterministic merge semantics with any per-agent metadata, schema/UI/docs updates, and focused tests proving the map reaches Bedrock stream options.
