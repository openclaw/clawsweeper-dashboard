## Summary

Keep open. Current main still lacks a configurable Bedrock `requestMetadata` path, and the related open per-agent tagging PR does not cover this issue's arbitrary user-configured metadata surface.

## What This Changes

The issue requests an `openclaw.json` configuration surface for Bedrock `requestMetadata` so AWS Bedrock calls can carry cost allocation tags.

## Best Possible Solution

Add a Bedrock-owned, validated `requestMetadata` surface in the Amazon Bedrock plugin and docs, with explicit precedence for merging configured tags and any automatic OpenClaw tags.

## Reproduction Assessment

Yes. Source inspection on current main shows downstream pi-ai and AWS support `requestMetadata`, while OpenClaw has no accepted config key or forwarding path that reaches Bedrock stream options.

## Solution Assessment

Unclear. The requested capability is valid, but the examples leave the public config location, validation rules, and precedence with https://github.com/openclaw/openclaw/pull/60698 for maintainers to decide.
