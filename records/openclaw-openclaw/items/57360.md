## Summary

Keep open. Current main still has no configurable Bedrock `requestMetadata` surface, and the related open work covers adjacent automatic tagging or Bedrock auth/config parity rather than this issue's arbitrary user-configured metadata request.

## What This Changes

The issue asks for Bedrock `requestMetadata` to be configurable from `openclaw.json`, either per-agent/default or provider level, for AWS cost allocation tagging.

## Best Possible Solution

Add a Bedrock-owned, validated `requestMetadata` config surface in the Amazon Bedrock plugin/runtime, with documented precedence for configured tags and any automatic OpenClaw tags.

## Reproduction Assessment

Yes, at source level. Current main has downstream pi-ai support but no accepted OpenClaw config key, schema entry, or Bedrock stream forwarding path for user-configured `requestMetadata`.

## Solution Assessment

Unclear. The requested capability is valid, but the maintainable fix needs a plugin-owned config shape, AWS-contract validation, privacy guidance, and precedence rules with the related automatic tagging work.
