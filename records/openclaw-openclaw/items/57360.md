## Summary

Keep open. Current main still has no OpenClaw config surface or runtime injection path for Bedrock `requestMetadata`; the related dynamic per-agent attribution thread and PR are open, but they do not yet supersede this static provider/plugin metadata request.

## What This Changes

The issue requests exposing Amazon Bedrock `requestMetadata` in OpenClaw config so Bedrock calls can carry AWS cost-allocation metadata.

## Best Possible Solution

Add a plugin-owned `plugins.entries.amazon-bedrock.config.requestMetadata` map with AWS key/value/count validation, inject it through the Amazon Bedrock provider wrapper into pi-ai Bedrock options, document it on the Bedrock provider page, and coordinate merge precedence with the per-agent attribution work tracked in #60602/#60698.

## Reproduction Assessment

Not applicable as a feature request, but we do have a high-confidence current-state verification path. The repo-wide `requestMetadata` search plus strict Bedrock schema/runtime/docs inspection confirms current main has no supported configurable injection path.

## Solution Assessment

Unclear until maintainers align this with #60602/#60698. A Bedrock plugin-owned config surface is the narrowest maintainable direction for this issue; a broad core pass-through of arbitrary Bedrock options would be less safe.
