## Summary

Keep open. Current main and the latest release still do not expose or forward Bedrock `requestMetadata`, and the related open issue/PR cover automatic per-agent injection rather than the configurable metadata surface requested here.

## What This Changes

Expose Bedrock `requestMetadata` in OpenClaw configuration so users can attach AWS cost allocation metadata to Bedrock Converse requests.

## Best Possible Solution

Add a Bedrock-owned, validated metadata configuration surface with docs and focused tests, coordinating with the per-agent injection work in https://github.com/openclaw/openclaw/issues/60602 and https://github.com/openclaw/openclaw/pull/60698 instead of treating that PR as a complete replacement.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is high-confidence from source inspection: current main has no `requestMetadata` symbol and the Bedrock config schema/wrapper omit it.

## Solution Assessment

Unclear. Exposing arbitrary metadata is plausible, but maintainers should first choose the supported scope, validation rules, and merge behavior with automatic per-agent metadata.
