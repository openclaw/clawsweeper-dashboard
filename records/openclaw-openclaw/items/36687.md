## Summary

Current main and shipped releases cover the observable WebChat/TUI-to-iMessage leakage path, while the separate upgrade-reset claim still lacks before/after config or migration evidence after maintainer follow-up.

## What This Changes

The issue reports that upgrading to 2026.3.2 effectively reset `session.dmScope` to `main`, allowing WebChat/TUI replies to leak to iMessage instead of staying channel-scoped.

## Best Possible Solution

Keep the shipped route-isolation guardrails; if an upgrade still removes or ignores an explicit `session.dmScope`, track that as a new focused config-migration bug with before/after config and logs.

## Reproduction Assessment

No, not as a current-main failure. Source and regression tests show the WebChat/UI leakage path now stays internal, while the remaining `dmScope` reset claim still lacks the snapshots or migration logs needed for a high-confidence repro.

## Solution Assessment

Yes for the observable leakage: the shipped route guard is narrower than changing the global runtime `dmScope` default. A separate migration fix would only be appropriate if a reproducible config-reset path is provided.
