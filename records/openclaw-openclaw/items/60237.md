## Summary

Keep open. Current main still uses a fixed Discord generated-title prompt and has no channel config field, schema entry, docs entry, or runtime plumbing for a custom thread-title guideline; the linked merged PR #43366 introduced generated titles but not this follow-up customization.

## What This Changes

Add an optional Discord channel `threadPromptGuideline` setting to customize generated auto-thread title prompting while preserving the existing fallback behavior.

## Best Possible Solution

Add a backwards-compatible per-channel Discord config field, validate it to the requested 500-character boundary, thread it into generated title creation as additional guidance, and update schema, generated metadata, docs, tests, and changelog together.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence missing-feature check: current main has no config key, the strict schema lacks the field, and generated title completion always uses the built-in prompt.

## Solution Assessment

Yes, the requested additive Discord channel config is the narrowest maintainable direction. The safer shape is to combine the guideline with the existing title-generation constraints instead of letting it fully replace them.
