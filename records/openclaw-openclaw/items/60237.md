## Summary

Keep open. Current main and the latest release still use a fixed Discord generated-thread-title prompt and the strict Discord channel schema has no `threadPromptGuideline`-style key, so this remains a new Discord config/product decision rather than conservative cleanup.

## What This Changes

The issue requests an optional Discord channel config field to customize the LLM guidance used when `autoThreadName: "generated"` names auto-created Discord threads.

## Best Possible Solution

Approve or reject a narrow Discord-plugin-owned field; if approved, implement bounded supplemental guidance with schema, types, generated metadata, docs, and focused tests while preserving the existing default prompt.

## Reproduction Assessment

Yes for source-level verification of the feature gap: current main's strict Discord channel schema lacks the requested key, and generated-title code always uses a fixed system prompt. No runtime bug reproduction is needed because this is a feature/config request.

## Solution Assessment

Unclear until maintainers approve the new config surface. If accepted, a bounded supplemental guidance field is safer than letting config replace the full prompt contract.
