## Summary

Keep open. Current main and the latest release still use a fixed Discord generated-thread-title prompt, and the strict Discord channel schema still has no `threadPromptGuideline`-style key, so this remains a small Discord config/product decision rather than conservative cleanup.

## What This Changes

The issue requests an optional Discord channel `threadPromptGuideline` config field to customize the LLM guidance used when `autoThreadName` is `generated`.

## Best Possible Solution

Approve or reject a narrow Discord-plugin-owned supplemental guidance field; if approved, implement it with schema, types, generated metadata, docs, and focused tests while preserving the existing default prompt.

## Reproduction Assessment

Yes, for source-level verification of the feature gap: current main's strict Discord channel schema lacks the requested key, and generated-title code always uses a fixed system prompt. No runtime bug reproduction is needed because this is a feature/config request.

## Solution Assessment

Unclear until maintainers approve the new config surface. If accepted, a bounded supplemental guidance field is safer than replacing the whole title-generation prompt contract.
