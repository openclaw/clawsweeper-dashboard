## Summary

Keep open. Current main still uses a fixed Discord generated-thread-title prompt and lacks the requested per-channel `threadPromptGuideline` type, strict schema entry, runtime plumbing, docs entry, and custom-guideline test; the related generated-title PR added `autoThreadName: "generated"` but not this customization.

## What This Changes

Add an optional Discord channel `threadPromptGuideline` config field to customize LLM-generated auto-thread titles when `autoThreadName` is `"generated"`.

## Best Possible Solution

Add a backward-compatible, bounded Discord guild-channel guideline field that supplements the existing generated-title contract, then update schema, generated metadata, docs, tests, and changelog together.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence source-level missing-feature check: current main has no config key, schema entry, runtime parameter, docs entry, or test for custom Discord generated-title guidance.

## Solution Assessment

Yes, pending maintainer approval of the config and prompt-policy shape. The narrow maintainable direction is to append bounded operator guidance to the existing title-only instructions rather than replacing all system guidance with arbitrary config text.
