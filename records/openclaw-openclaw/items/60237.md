## Summary

Keep open. Current main and the latest release still use a fixed Discord generated-title prompt, and the related merged generated-title PR did not add a per-channel prompt guideline field, schema/docs/runtime plumbing, or custom-guideline tests.

## What This Changes

Add an optional Discord per-channel `threadPromptGuideline` config to customize LLM-generated titles when `autoThreadName: "generated"` is enabled.

## Best Possible Solution

Add a backwards-compatible bounded Discord channel guideline field, plumb it into generated-title creation, and update schema, generated metadata, docs, tests, and changelog together.

Yes. The narrow maintainable direction is an additive per-channel config field whose bounded guidance is appended to the existing title-generation contract, not a wholesale replacement of the fixed safety and formatting instructions.
