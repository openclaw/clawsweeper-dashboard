## Summary

Keep open. Current main still has only the fixed Discord generated-title prompt and no channel config field, schema entry, runtime plumbing, docs entry, or tests for a custom thread-title guideline; the related merged PR added generated titles but not this customization.

## What This Changes

Add an optional Discord channel configuration field to guide LLM-generated auto-thread titles when `autoThreadName` is `generated`.

## Best Possible Solution

Add a backwards-compatible per-channel Discord guideline field, validate it to the requested length boundary, combine it with the existing title-generation constraints, and update schema, metadata, docs, tests, and changelog together.

## Solution Assessment

Yes, the requested additive Discord channel config is a narrow maintainable direction. The safer implementation is to append bounded guidance to the existing generated-title prompt contract rather than replacing all system guidance with arbitrary config text.
