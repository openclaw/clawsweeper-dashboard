## Summary

Keep open. Current main still hardcodes the Discord generated-title prompt and has no `threadPromptGuideline` or equivalent in channel config, strict schema, resolver plumbing, docs, or tests; this is a new user-facing config and prompt-policy decision, not conservative cleanup.

## What This Changes

Request adds an optional Discord channel `threadPromptGuideline` setting to customize LLM-generated thread titles when `autoThreadName: "generated"` is used.

## Best Possible Solution

Add a bounded Discord channel-config guidance field after maintainer approval, keeping it subordinate to the title-only prompt and updating types, strict schema, generated metadata, docs, tests, and changelog.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence missing-feature check: current main has no accepted config key, schema entry, runtime parameter, docs entry, or test for custom Discord generated-title guidance.

## Solution Assessment

Unclear pending product approval. A safer maintainable solution would treat operator text as bounded supplemental guidance under the fixed title-only prompt instead of replacing the full system prompt with arbitrary config text.
