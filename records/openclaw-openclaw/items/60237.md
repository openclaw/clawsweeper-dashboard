## Summary

Keep open. Current main still hardcodes the Discord generated-title prompt and has no `threadPromptGuideline` or equivalent config/schema/runtime/docs/test surface; this is a new user-facing Discord config and prompt-policy decision, not conservative cleanup.

## What This Changes

The issue requests an optional Discord channel configuration field, `threadPromptGuideline`, to customize LLM-generated auto-thread titles when `autoThreadName` is `generated`.

## Best Possible Solution

Add a bounded Discord channel-config guidance field after maintainer approval, keeping operator text subordinate to the fixed title-only prompt and updating types, strict schema, generated metadata, docs, tests, and changelog.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence missing-feature check: current main has no accepted config key, schema entry, runtime parameter, docs entry, or test for custom Discord generated-title guidance.

## Solution Assessment

Unclear pending product approval. A safer maintainable solution would treat operator text as bounded supplemental guidance under the fixed title-only prompt instead of replacing the full system prompt with arbitrary config text.
