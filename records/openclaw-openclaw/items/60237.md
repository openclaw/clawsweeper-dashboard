## Summary

Keep open. Current main and v2026.6.10 still use the fixed Discord generated-thread-title prompt, and the strict Discord channel config schema still has no `threadPromptGuideline`, so this remains a valid but product-sensitive configuration feature request.

## What This Changes

Add an optional Discord channel config field, `threadPromptGuideline`, to customize guidance for LLM-generated auto-thread titles when `autoThreadName` is `generated`.

## Best Possible Solution

Make a maintainer product decision; if approved, implement a Discord-plugin-owned supplemental guidance field with schema, types, generated metadata, docs, and focused tests while preserving the existing default prompt.

## Reproduction Assessment

Yes, for source-level verification of the feature gap: current main has no `threadPromptGuideline`, the strict Discord channel schema omits it, and generated titles always use a fixed system prompt. Runtime bug reproduction is not applicable because this is a feature/config request.

## Solution Assessment

Unclear until maintainers approve the new config surface. If accepted, a bounded supplemental guidance field is safer than replacing the whole title-generation prompt contract.
