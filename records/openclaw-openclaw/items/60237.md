## Summary

Keep open. Current main still has a fixed Discord generated-thread-title prompt and no per-channel title-guidance config, schema entry, runtime parameter, docs, or custom-guideline test; the linked merged PR added `autoThreadName: "generated"`, not this customization. Because the request adds a user-facing config key and prompt-policy behavior, it needs maintainer product review before implementation.

## What This Changes

Add an optional Discord per-channel `threadPromptGuideline` config field to customize LLM-generated auto-thread titles when `autoThreadName` is `"generated"`.

## Best Possible Solution

Add a bounded Discord channel-config guidance field that supplements the existing title-only generation contract, then update types, strict schema, generated config metadata, docs, tests, and changelog after maintainer approval.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence missing-feature check: current main has no config key, schema entry, runtime parameter, docs entry, or test for custom Discord generated-title guidance.

## Solution Assessment

Unclear pending product approval. A safer maintainable solution would treat operator text as bounded guidance subordinate to the fixed title-only prompt, rather than replacing the full system prompt with arbitrary config text.
