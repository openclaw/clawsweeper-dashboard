## Summary

Keep open. Current main supports Discord `autoThreadName: "generated"`, but the generated-title prompt is still fixed and there is no channel config field, schema entry, docs entry, or runtime wiring for a custom thread-title guideline. The linked merged PR #43366 is useful context because it introduced the generated-title mode, but it does not cover this follow-up customization request.

## What This Changes

Add an optional Discord guild-channel `threadPromptGuideline` config field to customize LLM-generated auto-thread titles when `autoThreadName` is `"generated"`.

## Best Possible Solution

Add a backwards-compatible per-channel Discord config field, validate/clamp it to the requested 500-character boundary, route it through the resolved channel config into the title generator, preserve the existing default behavior when absent, and cover config/schema/docs/runtime/tests plus a user-facing changelog entry.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence missing-feature verification path: current main has no config key, the strict schema lacks the field, and generated title completion always uses the built-in prompt.

## Solution Assessment

Yes, a narrow additive Discord channel config field is the maintainable direction, provided it is applied as operator guidance under the existing concise title-generation constraints rather than replacing those constraints entirely.
