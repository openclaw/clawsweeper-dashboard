## Summary

Keep open. Current main has a generic outbound `message_sending` hook plus partial commentary and cron interim-ack mitigations, but it still lacks the requested runtime-owned fake-progress classifier, durable execution-state gate, repeated-ack/progress-debt tracking, and cross-channel enforcement.

## What This Changes

Request to add a runtime-enforced pre-send policy layer that classifies outbound assistant messages and blocks fake-progress acknowledgements unless backed by durable execution or progress state.

## Best Possible Solution

Define a first-class core outbound policy boundary after payload materialization and before channel send, backed by durable run/task/session state, then implement typed ack/progress/blocker/completion classification, repeat-ack suppression, progress-debt faulting, structured metrics, and focused coverage across shared outbound, agents, ACP/subagents, cron, and channel adapters.

## Reproduction Assessment

Yes for the narrow source-level gap: current `deliverOutboundPayloads` can send ack-like text unless a plugin hook cancels it, and no built-in classifier or execution-state prerequisite is present. Live direct-chat and Telegram repros were not rerun in this read-only review.

## Solution Assessment

Yes directionally: a runtime pre-send policy backed by durable execution state is the right class of fix. The exact implementation is not yet a narrow automated patch because maintainers need to choose the shared outbound boundary and durable state contract, especially alongside #56349.
