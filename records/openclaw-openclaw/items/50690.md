## Summary

Keep open. Current main has useful partial mitigations, including generic outbound hook cancellation/rewrite, pre-model/reply dispatch hooks, commentary-phase suppression, and isolated-cron stale-ack retry, but it still lacks the requested runtime-owned fake-progress classifier, durable execution-state gate, repeated-ack/progress-debt tracking, and cross-channel pre-send enforcement.

## What This Changes

Add a runtime pre-send policy layer that classifies outgoing assistant text and blocks fake-progress acknowledgements unless backed by durable execution/progress state across user-facing channels.

## Best Possible Solution

Define a first-class core outbound policy boundary after payload materialization and before channel send, backed by durable run/task/session state, then implement typed ack/progress/blocker/completion classification, repeat-ack suppression, progress-debt faulting, structured metrics, and coverage across shared outbound delivery, agent/ACP/subagent paths, cron, and channel adapters.

## Reproduction Assessment

Yes for the source-level enforcement gap: current deliverOutboundPayloads can send ack-like text as an ordinary payload unless a plugin hook cancels it, and no built-in execution-state-backed classifier is present. The broad production repro matrix still needs live channel verification once a design lands.

## Solution Assessment

Yes directionally: a runtime pre-send policy backed by durable execution state is the right class of fix. The exact implementation is not yet a narrow repair because maintainers need to choose the shared outbound boundary and durable state contract.
