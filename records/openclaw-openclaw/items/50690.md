## Summary

Keep open. Current main has a generic outbound rewrite/cancel hook, phase-aware commentary suppression, and scoped cron interim-ack retries, but it still lacks the requested runtime-owned fake-progress classifier, execution-state gate, repeated-ack suppression, progress-debt tracking, and cross-channel enforcement.

## What This Changes

The issue requests a runtime pre-send policy layer that classifies outbound assistant messages and blocks fake-progress acknowledgements unless backed by execution/progress state.

## Best Possible Solution

Define a first-class core outbound policy boundary after payload materialization and before channel send, backed by durable run/task/session state, then implement typed ack/progress/blocker/completion classification, repeat-ack suppression, progress-debt faulting, structured metrics, and focused coverage across shared outbound, agents, ACP/subagents, cron, and channel adapters.

## Reproduction Assessment

Yes for the source-level gap: current `deliverOutboundPayloads` can deliver ack-like text unless an optional plugin hook cancels it, and no built-in execution-state prerequisite is present. Live direct-chat or Telegram reproduction was not rerun because this review was read-only.

## Solution Assessment

Yes directionally: a runtime pre-send policy backed by durable execution state is the right class of fix. The exact implementation is not yet a safe narrow patch because maintainers need to choose the shared boundary and durable state contract, especially alongside #56349.
