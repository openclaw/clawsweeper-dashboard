## Summary

Keep open. Current main has partial outbound hook, commentary-suppression, delivery-queue, task, and cron stale-ack mitigations, but it still lacks the requested built-in fake-progress classifier, durable execution-state gate, repeat-ack/progress-debt tracking, and cross-channel runtime enforcement.

## What This Changes

Request a core runtime pre-send policy that classifies outbound assistant text and suppresses fake-progress acknowledgements unless backed by real execution/progress state.

## Best Possible Solution

Define a first-class core outbound policy boundary after payload materialization and before channel send, backed by durable run/task/session state, then add typed ack/progress/blocker/completion classification, repeat-ack suppression, progress-debt faulting, structured metrics, and coverage across shared outbound delivery, agents, ACP/subagents, cron, and channel adapters.

## Reproduction Assessment

Yes for the source-level enforcement gap: current shared delivery can send ordinary ack-like text unless an optional hook cancels it, and no built-in durable execution-state classifier is present. The report's full production matrix should be live-verified once maintainers choose the boundary.

## Solution Assessment

Yes directionally: a runtime pre-send policy backed by durable execution state is the right class of solution. It is not yet a narrow autonomous repair because choosing the shared outbound boundary, state contract, and plugin/core API shape requires maintainer design.
