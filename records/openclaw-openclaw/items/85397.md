## Summary

Keep open: the non-sandbox media-store fix is well targeted and the supplied runtime proof is useful, but the image-tool change pre-resolves `media://` before the sandbox resolver can use the inbound fallback path, leaving a sandboxed image-tool path unresolved.

## What This Changes

The branch resolves `media://inbound` image refs through the media store before workspace path fallback in native prompt image loading and the image tool, with focused regression tests.

## Best Possible Solution

Keep the non-sandbox media-store resolution, but make sandboxed image-tool media refs flow through one sandbox-safe resolver path shared with the companion sandbox fix and cover both `workspaceDir` and `sandbox + workspaceOnly` cases.

## Reproduction Assessment

Yes from source: current `main` workspace-resolves `media://inbound/...` in the image tool when `workspaceDir` is set, matching the malformed `media:/inbound` failure described by the linked report. I did not run tests because this review was read-only.

## Solution Assessment

No. Resolving the media-store URI before workspace fallback is the right shape for non-sandbox paths, but doing it before sandbox resolution strips the URI needed by the sandbox inbound fallback path.
