## Summary

Keep open: the PR targets a real non-default agent `models.json` refresh bug, but the latest head still has a blocking provider-discovery regression and lacks after-fix real startup proof.

## What This Changes

The PR adds a gateway startup helper to refresh default and configured agent `models.json` caches and expands gateway startup tests for custom-agent refresh behavior.

## Best Possible Solution

Keep the independent configured-agent refresh, preserve scoped metadata-only provider discovery even for an empty provider list, rebase onto current agent-scope helpers, and land after redacted startup proof.

## Reproduction Assessment

Yes by source inspection. Current main skips startup cache preparation when no default primary model is configured and otherwise refreshes only the default agent dir, while the linked issue shows direct per-agent `ensureOpenClawModelsJson` calls fixed stale custom-agent caches.

## Solution Assessment

No for the current PR head. The helper split is the right narrow direction, but the implementation must preserve explicit empty provider scope, rebase cleanly, and include real behavior proof before merge.
