## Summary

Keep open: the underlying bug is valid and the PR is a plausible narrow fix, but the current head still drops scoped-empty metadata-only provider discovery for all-CLI configs and lacks required real startup proof.

## What This Changes

The PR adds a gateway startup helper that refreshes `models.json` for the default and configured agent directories and expands gateway startup regression coverage.

## Best Possible Solution

Keep the independent configured-agent refresh, pass discovery options even when the provider list is empty, update to current agent-scope helpers, add CLI-only coverage, and land after redacted startup proof.

## Reproduction Assessment

Yes, by source inspection. Current main skips startup `models.json` preparation when there is no default primary model and otherwise refreshes only the default agent dir, leaving configured custom agent dirs stale.

## Solution Assessment

No for the current PR head. The helper split is the right narrow direction, but it must preserve scoped-empty metadata-only discovery, rebase onto current agent-scope helpers, and include real behavior proof before merge.
