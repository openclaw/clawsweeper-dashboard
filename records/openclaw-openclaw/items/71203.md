## Summary

Keep open. Current main still has the stale non-default agent models.json path, and this PR is a useful fix vehicle for #71192, with overlap to the still-open #58289. The latest PR head is not ready to merge because the CLI-only provider case still drops the scoped metadata-only discovery options, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR adds a gateway startup helper that refreshes models.json for the active/default agent directory and configured agent directories, wires it before channel startup, and expands gateway startup tests for custom agents.

## Best Possible Solution

Land a revised startup refresh that regenerates unique active/default and configured agent models.json files independently of primary-model prewarm, always preserves scoped metadata-only provider discovery even when the provider list is empty, and records the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Yes. Current main has a source-level reproduction by configuring non-default agents and omitting agents.defaults.model.primary, which returns before cache refresh; the remaining PR regression reproduces when all configured model providers are CLI-backed and providerDiscoveryProviderIds becomes an empty array.

## Solution Assessment

No for the latest PR head. The helper direction is the narrow maintainable solution, but it should always pass providerDiscoveryProviderIds, providerDiscoveryTimeoutMs, and providerDiscoveryEntriesOnly, including the explicit empty provider list, and add a changelog entry before merge.
