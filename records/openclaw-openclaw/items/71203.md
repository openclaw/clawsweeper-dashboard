## Summary

Keep open: this is a plausible bug fix for stale non-default agent `models.json` caches, but the current PR head still drops the explicit empty provider-discovery scope in CLI-only configurations and lacks contributor real behavior proof.

## What This Changes

The PR adds a gateway startup helper to refresh the default and configured agent `models.json` files, then adds focused startup tests for custom agents and synthetic `main` handling.

## Best Possible Solution

Keep the separate startup refresh helper, but always preserve scoped metadata-only discovery, including an empty provider list, and require redacted real startup proof before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current `main` skips startup models.json preparation when no default primary model is configured and otherwise refreshes only the default agent dir, matching the linked stale non-default agent cache report.

## Solution Assessment

Is this the best way to solve the issue? No for the current PR head. The independent helper is the right direction, but it must preserve explicit empty provider-discovery scope and add CLI-only regression coverage before it is safe to merge.
