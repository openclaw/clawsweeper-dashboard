## Summary

Keep open. Current main still lacks the custom-agent startup cache refresh the PR is trying to add, and the branch is aimed at the right gateway startup surface, but the latest head still has a blocking scoped-discovery regression for CLI-only model providers and is missing the required user-facing changelog entry.

## What This Changes

The PR adds a gateway startup helper that refreshes models.json for the active/default agent directory and configured agent directories, wires it before channel startup, and expands gateway startup regression tests.

## Best Possible Solution

Land a revised startup refresh that regenerates unique active/default and configured agent models.json files independently of primary-model prewarm while always keeping provider discovery metadata-only and explicitly scoped, including an empty providerDiscoveryProviderIds array for CLI-only configs, plus the required changelog entry and regression coverage.

## Reproduction Assessment

Yes. Configure the default/configured agent models so all configured providers are CLI-backed; on the PR head providerDiscoveryProviderIds becomes [], line 163 converts that to {}, and ensureOpenClawModelsJson receives unscoped discovery options. The original stale-cache gap is also reproducible on current main with configured custom agents and no default primary model, because the current prewarm helper returns before refreshing any agent dir.

## Solution Assessment

No for the latest PR head. The helper split is the right narrow direction, but the implementation should preserve explicit empty provider discovery scope, keep metadata-only discovery options, and add the required changelog entry before merge.
