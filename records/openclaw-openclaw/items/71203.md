## Summary

Keep open. Current main still does not refresh configured non-default agent models.json files during startup, and this PR targets the right gateway/cache gap, but the latest diff still drops scoped metadata-only provider discovery options for CLI-only configurations and lacks the required changelog entry.

## What This Changes

The PR adds a gateway startup helper that refreshes models.json for the active/default agent directory and configured agent directories, wires it before channel startup, and expands gateway startup tests for custom agents.

## Best Possible Solution

Land a revised startup refresh that regenerates unique active/default and configured agent models.json files independently of primary-model prewarm while preserving scoped metadata-only provider discovery, including an explicit empty providerDiscoveryProviderIds array, and documenting the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Current main has a source-level reproduction: omit agents.defaults.model.primary and configure non-default agents, and startup returns before models.json refresh; the PR's remaining regression reproduces with CLI-only configured model providers because the filtered provider list becomes [] and is converted to unscoped options.

## Solution Assessment

Is this the best way to solve the issue? No for the latest PR head. The direction is narrow and maintainable, but the implementation should always pass the scoped metadata-only discovery options, even for an empty provider list, and add the required changelog entry before merge.
