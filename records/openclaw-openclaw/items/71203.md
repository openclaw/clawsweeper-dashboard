## Summary

Keep open. The stale non-default agent models.json bug is still present on current main and this PR is aimed at the right startup helper boundary, but the current head still drops the explicit empty provider-discovery scope for CLI-only configurations and lacks after-fix real gateway proof.

## What This Changes

The PR adds a gateway startup helper to refresh default and configured agent models.json files and expands startup tests for custom-agent cache refresh.

## Best Possible Solution

Keep the independent startup refresh helper, preserve metadata-only scoped discovery including an empty provider list, rebase onto current startup helpers, and land after redacted real startup proof.

## Reproduction Assessment

Yes by source inspection. Current main skips startup models.json preparation when there is no default primary model and otherwise refreshes only the default agent dir, matching the linked stale custom-agent cache path.

## Solution Assessment

No for the current PR head. The helper split is the right narrow direction, but the implementation must preserve explicit empty provider-discovery scope and add CLI-only regression coverage before merge.
