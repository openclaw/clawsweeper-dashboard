## Summary

Keep open: current main still has the stale configured-agent models.json refresh gap, but this PR remains blocked because the latest head drops scoped-empty provider discovery options, lacks after-fix real startup proof, and is currently conflicting with main.

## What This Changes

The PR adds a gateway startup helper to refresh default and configured agent models.json caches and expands gateway startup regression tests.

## Best Possible Solution

Keep the independent configured-agent refresh, rebase onto current gateway/agent-scope helpers, preserve metadata-only discovery even for an empty provider list, add CLI-only regression coverage, and land after redacted startup proof.

## Reproduction Assessment

Yes by source inspection. Current main skips startup models.json preparation when there is no default primary model and otherwise refreshes only resolveDefaultAgentDir(params.cfg), so configured custom agent dirs are not refreshed on startup.

## Solution Assessment

No for the current PR head. The helper split is the right narrow fix, but the implementation must preserve explicit empty provider-discovery scope, rebase onto current main, and include real behavior proof before merge.
