## Summary

Keep open. Current main still has the stale non-default agent cache path, but this PR head is not merge-ready because it drops scoped metadata-only provider discovery for all-CLI configurations and still has only test/check output rather than real gateway startup proof.

## What This Changes

The branch adds a gateway startup helper that refreshes models.json for the default and configured agent directories and expands startup tests for custom agents.

## Best Possible Solution

Keep the independent startup refresh helper, preserve scoped metadata-only discovery even when the provider list is empty, rebase onto current main, and add redacted real startup proof.

## Reproduction Assessment

Yes, source-reproducible. Current main skips startup models.json preparation when no default primary model is configured and otherwise prepares only the default agent dir, matching the linked stale custom-agent cache report.

## Solution Assessment

No for the current PR head. The helper split is the right direction, but the implementation must preserve explicit empty provider scope and metadata-only discovery before it is safe to merge.
