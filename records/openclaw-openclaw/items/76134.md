## Summary

Keep open for maintainer review: current main still has the active-only async transcript read gap, and this PR is a focused fix with sufficient after-fix terminal proof and no blocking review findings from this read-only pass.

## What This Changes

The branch adds an async latest-reset-archive fallback for missing active session transcripts, routes async readers/count/title paths through it, adds regression and HTTP pagination coverage, and updates the changelog.

## Best Possible Solution

Land a CI-green version of this narrow fallback after maintainer review, while keeping logical-history chaining and explicit archived-history API work in the related product tracks.

## Reproduction Assessment

Yes from source inspection: current main creates reset archives, but the affected async readers only resolve active transcript paths, so archive-only sessions fall through to empty or zero defaults. I did not run a local repro because this was a read-only review.

## Solution Assessment

Yes for the narrow missing-active case: the shared active-or-latest-reset resolver preserves active-wins behavior and avoids merging archives or adding a new API surface. Broader logical history and includeArchived behavior should remain separate.
