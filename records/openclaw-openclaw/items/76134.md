## Summary

## What This Changes

The PR routes async session transcript readers, counts, stats, visits, and title fields through a fallback to the newest reset archive when the active transcript is missing.

## Best Possible Solution

Land the narrow missing-active async fallback after normal maintainer review and CI, while keeping logical-history chaining, explicit archive selection, restore tooling, and UI archive browsing in the related open work.

## Solution Assessment

Yes. The PR uses the existing candidate-resolution and archive timestamp contracts, keeps active-present behavior unchanged, and avoids protocol/schema or archive-chain expansion beyond the reported missing-active gap.
