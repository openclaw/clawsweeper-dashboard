## Summary

Keep this PR open: the async reset-archive fallback is a useful narrow fix, but the patch currently makes bounded sessions-history pagination inconsistent because archive messages can be returned with a transcript count of 0.

## What This Changes

The PR adds async fallback from missing active session transcripts to the newest `.reset.<timestamp>` sibling for `chat.history`, sessions-history reads, and derived session title fields, with tests and a changelog entry.

## Best Possible Solution

Land the narrow async archive fallback after making message reads and transcript stats resolve the same active-or-archive file, then keep broader logical-history, `includeArchived`, and archive-chain work in their existing separate PRs/issues.

## Reproduction Assessment

Yes. A high-confidence reproduction is an archive-only transcript with more messages than the bounded sessions-history tail window, then requesting the first limited HTTP history page and following `nextCursor`; source tracing shows the PR would assign tail messages low sequence numbers because the count remains 0.

## Solution Assessment

No, not as currently written. The fallback direction is maintainable, but the helper needs to be shared by the count/stat path used by sessions-history pagination so metadata and returned messages describe the same file.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the archive path for message counts too:** `src/gateway/session-utils.fs.ts:574`
  - body: After this change, `readRecentSessionMessagesAsync` can return messages from a reset archive when the active file is missing, but `readRecentSessionMessagesWithStatsAsync` still gets `totalMessages` from `readSessionMessageCountAsync`, which only checks the active transcript. The sessions-history HTTP path uses that total to assign cursor sequence numbers, so an archive-only transcript longer than the tail window can return the newest messages with low seq values and then skip the middle archive page when `nextCursor` is followed. Route the count/stat path through the same active-or-archive resolution or avoid supplying stale totals.
  - confidence: 0.92
