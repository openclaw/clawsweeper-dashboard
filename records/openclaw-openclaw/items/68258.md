## Summary

Keep open. Current main still lacks an `auth`/`auth.profiles` reload rule, so the reported path remains source-reproducible, and the issue is paired with an open same-author fix PR.

## What This Changes

Fix Gateway config reload planning so OAuth `auth.profiles.*` refresh writes do not trigger a gateway restart while `gateway.auth.*` changes still restart.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68257 or an equivalent targeted patch with regression coverage for auth-profile refreshes and `gateway.auth.*` restart preservation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` currently has no matching rule and takes the unmatched-path restart branch; the issue comments add matching production logs.

## Solution Assessment

Yes. The narrow maintainable fix is to classify persisted auth-profile updates as dynamic/no-op reloads while keeping `gateway.auth.token` and `gateway.auth.mode` under the existing gateway restart rule.
