## Summary

Keep open. Current main still source-reproduces the Gateway reload-planner restart path for `auth.profiles.*`, and the paired fix PR is open, conflicting, and not merged.

## What This Changes

The issue reports that OAuth auth-profile config writes under `auth.profiles.*` are treated as unknown reload paths and trigger a Gateway restart that can drop in-flight WebSocket work.

## Best Possible Solution

Classify persisted auth-profile refresh writes as no-op reload paths with regression coverage, while preserving `gateway.auth.*` restart behavior and the conservative unknown-path restart default.

## Reproduction Assessment

Yes. Source inspection on current main shows `auth.profiles.openai-codex:user@example.com` has no matching reload rule and falls into the unmatched-path restart branch; the issue discussion adds production logs from OpenClaw 2026.4.11.

## Solution Assessment

Yes, with a small scope choice. The maintainable fix is a reload-rule classification plus focused tests, while maintainers can choose whether the no-op prefix should be `auth.profiles` or all top-level `auth`.
