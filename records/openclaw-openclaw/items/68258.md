## What This Changes

The issue reports that OAuth auth-profile refresh writes under `auth.profiles.*` are treated as unmatched config changes and trigger a Gateway restart that can drop in-flight WebSocket work.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68257 or an equivalent narrow patch that classifies persisted auth-profile refresh paths as no-op reload changes while preserving `gateway.auth.*` restart behavior and the conservative unknown-path restart default.

## Reproduction Assessment

Yes. Source inspection against current main shows `auth.profiles.*` has no matching reload rule and therefore falls through to the unmatched-path restart branch in `buildGatewayReloadPlan`; the issue discussion also includes production-log confirmation.

## Solution Assessment

Yes. The narrow maintainable solution is a reload-rule classification plus focused regression coverage, with a small maintainer scope choice between `auth.profiles.*` and broader `auth.*`.
