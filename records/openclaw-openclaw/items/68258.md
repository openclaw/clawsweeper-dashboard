## What This Changes

The issue reports that OAuth auth-profile config writes are treated as unmatched reload-plan paths and cause gateway restarts during token refresh.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/68257 or an equivalent targeted patch that no-ops auth-profile refresh writes while preserving `gateway.auth.*` restart coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` has no matching top-level `auth` rule and falls into the unmatched-path restart branch; the issue discussion adds matching production logs.

## Solution Assessment

Yes. The narrow maintainable fix is to classify persisted auth-profile refresh writes as dynamic/no-op reload paths while preserving the existing `gateway.auth.token` and `gateway.auth.mode` restart behavior.
