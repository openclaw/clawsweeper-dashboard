## Summary

Keep open. Current main still has no `auth` or `auth.profiles` reload rule, so OAuth profile refresh paths fall through to the unmatched-path restart branch, and the issue is already paired with open PR #68257 from the same author for the narrow fix.

## What This Changes

The issue requests that Gateway config reload stop treating OAuth auth profile updates under `auth.profiles.*` as restart-required changes.

## Best Possible Solution

Land PR #68257 or an equivalent targeted patch that classifies auth profile updates as dynamic/no-op reloads while preserving restart behavior for `gateway.auth.*`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` has no matching rule on current main, so the unmatched-path branch marks the Gateway for restart; the issue and confirmation comment include matching production logs.

## Solution Assessment

Yes, with scope review. The narrow maintainable solution is to add reload classification for persisted auth profile updates and regression tests proving `gateway.auth.token` and `gateway.auth.mode` still restart.
