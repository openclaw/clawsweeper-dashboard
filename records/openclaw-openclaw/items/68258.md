## Summary

Keep open. Current main still source-reproduces the reported Gateway reload-planner restart path, and the same reporter has an open paired fix PR that is not merged yet.

## What This Changes

The issue reports that OAuth refresh writes to `auth.profiles.*` are unmatched in Gateway reload planning and therefore trigger a Gateway restart during active work.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/68257 or an equivalent targeted patch with regression coverage that treats auth-profile refresh writes as dynamic/no-op while preserving `gateway.auth.*` restart behavior.

## Reproduction Assessment

Yes. Source inspection shows `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` has no matching reload rule and takes the unmatched-path restart branch; the discussion adds production log confirmation.

## Solution Assessment

Yes. The narrow maintainable fix is to classify persisted auth-profile refresh writes as no-op/dynamic reload paths while keeping `gateway.auth.token` and `gateway.auth.mode` restart-classified.
