## Summary

Keep open. Current main still classifies `auth.profiles.*` as an unmatched reload path, which falls into the gateway-restart branch, and the issue is already paired with open PR #68257 from the same author for the narrow fix.

## What This Changes

The issue asks OpenClaw to stop restarting the Gateway when OAuth auth profile metadata changes under `auth.profiles.*`, while preserving restart behavior for `gateway.auth.*` settings.

## Best Possible Solution

Land PR #68257 or an equivalent targeted patch that classifies auth profile metadata updates as dynamic/no-op reloads, adds regression coverage for `auth.profiles.*`, and proves `gateway.auth.token` and `gateway.auth.mode` still request Gateway restarts.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` on current main: no reload rule matches `auth`, so the unmatched-path branch marks the Gateway for restart.

## Solution Assessment

Yes, with scope review. The paired PR direction is the narrow maintainable fix, but the safest implementation may be an explicit `auth.profiles`/`auth.order` classification rather than an unrestricted top-level `auth` no-op.
