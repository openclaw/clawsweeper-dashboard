## Summary

Keep open. Current main and the latest release still route `auth.profiles.*` through the unmatched-path restart branch, and the same reporter's paired fix PR remains open and conflicting.

## What This Changes

The issue reports that OAuth auth profile refresh writes under `auth.profiles.*` are treated as unknown config changes and cause the Gateway reload planner to restart the Gateway.

## Best Possible Solution

Apply the targeted reload classification and regression tests on main so OAuth profile refresh config writes do not trigger Gateway restart, while preserving `gateway.auth.*` and unknown-root restart behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` has no matching rule and therefore takes the unmatched-path restart branch; the discussion adds production-log confirmation.

## Solution Assessment

Yes. The narrow maintainable fix is a reload-rule classification with focused regression tests, with only the prefix scope (`auth.profiles` versus all `auth`) left for maintainer preference.
