## Summary

Keep open. Current `main` and the latest release still route unmatched `auth.profiles.*` config changes to a gateway restart, and the linked fix PR is still open rather than merged.

## What This Changes

The issue reports that OAuth auth-profile refresh writes under `auth.profiles.*` are not covered by the gateway reload plan and can restart the gateway during active work.

## Best Possible Solution

Review and land or replace the linked fix PR so `auth.profiles.*` token-refresh writes no longer restart the gateway while `gateway.auth.*` remains restart-backed.

## Reproduction Assessment

Yes. Source inspection on current `main` shows `auth.profiles.*` has no matching reload rule, so `buildGatewayReloadPlan()` reaches the unmatched-path restart branch; I did not run tests because this sweep is read-only.

## Solution Assessment

Yes, the narrow direction is best: add explicit no-restart coverage for auth-profile writes with regression tests, while preserving the existing `gateway.auth.*` restart behavior and the conservative unknown-path default.
