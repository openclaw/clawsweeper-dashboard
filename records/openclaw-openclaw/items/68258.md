## Summary

Keep open. Current main still source-reproduces the Gateway reload-planner restart path for `auth.profiles.*`, and the paired fix PR is still open rather than merged.

## What This Changes

The issue reports that OAuth auth-profile refresh writes under `auth.profiles.*` are treated as unmatched config paths and cause the Gateway to restart mid-dispatch.

## Best Possible Solution

Classify persisted auth-profile refresh writes as no-op/dynamic reload paths with regression coverage, while preserving `gateway.auth.*` restart behavior and the conservative unknown-path restart default.

## Reproduction Assessment

Yes. Source inspection of current main shows `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` has no matching rule and falls into the unmatched-path restart branch; the issue discussion adds matching production logs.

## Solution Assessment

Yes. The narrow maintainable fix is a reload-rule classification plus focused tests; the only scope choice is whether to classify all `auth.*` or only `auth.profiles.*`.
