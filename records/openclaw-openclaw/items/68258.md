## What This Changes

The issue asks OpenClaw to stop treating OAuth auth-profile refresh writes under `auth.profiles.*` as Gateway restart triggers.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68257 or an equivalent targeted patch with regression coverage for auth-profile refreshes and `gateway.auth.*` restart preservation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `buildGatewayReloadPlan(["auth.profiles.openai-codex:user@example.com"])` currently has no matching rule and takes the unmatched-path restart branch; the issue discussion adds matching production logs.

## Solution Assessment

Yes. The narrow maintainable fix is to classify persisted auth-profile refresh writes as dynamic/no-op reloads while preserving the existing `gateway.auth.token` and `gateway.auth.mode` restart behavior.
