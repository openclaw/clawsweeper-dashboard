## What This Changes

The PR introduces a separate Codex `contextSessionKey` so context-engine bootstrap, assembly, compaction, transcript mirroring, prompt reporting, and finalization use the canonical run session key while sandbox/runtime policy continues to use the sandbox key.

## Best Possible Solution

Land a narrow split-key fix that keeps sandbox/tool policy on `sandboxSessionKey` and every context/history identity surface on the canonical run `sessionKey`, with the added regression kept as the guardrail.

## Reproduction Assessment

Yes, from source and supplied logs rather than a local rerun: current main passes `sandboxSessionKey` into Codex context-engine bootstrap/assemble, and the linked issue shows v2026.5.19 selecting a stale per-DM LCM conversation for a Telegram DM.

## Solution Assessment

Yes. Separating `contextSessionKey` from `sandboxSessionKey` is the narrowest maintainable fix because it preserves sandbox/tool policy scoping while moving context continuity back to the canonical run session.
