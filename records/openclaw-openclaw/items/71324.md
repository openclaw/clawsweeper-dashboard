## Summary

Keep open. The legacy WebChat/history leak is a valid Gateway bug and this PR is still useful, but the current head introduces blocking sanitizer regressions that can hide real user or assistant-authored transcript text.

## What This Changes

The PR adds Gateway/WebChat history projection logic to hide legacy runtime-only transcript rows, plus session-history regression tests and a changelog entry.

## Best Possible Solution

Keep the Gateway history display-projection approach, but repair it so only legacy user/runtime rows are stripped while user tails and assistant-authored quoted content remain visible.

## Reproduction Assessment

Yes, source-level. Current main projects session history through `projectChatDisplayMessages`, but its existing sanitizer does not unwrap quote-prefixed legacy runtime/status rows, and the PR body includes red-first tests plus direct terminal smoke output for those shapes.

## Solution Assessment

No. The projection boundary is appropriate, but this implementation needs tail preservation, user-role gating for legacy status stripping, and focused positive/negative regression tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with these legacy prompt headers. Legacy reset/startup rows can include real user text after the header, and current main preserves that reset tail as transcript text, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, and it is called for all roles. Assistant messages that quote or explain a legacy `System:` line can therefore be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
