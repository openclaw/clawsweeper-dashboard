## Summary

Keep open: the latest head still has blocking sanitizer correctness issues that can hide legitimate transcript text, even though the intended regression has sufficient terminal proof.

## What This Changes

The PR changes Gateway history sanitization and session-history tests to hide legacy quoted runtime prompts, status wrappers, and now-empty rows from WebChat history while keeping raw transcript sequence accounting.

## Best Possible Solution

Keep the projection-layer fix, but make legacy stripping role-aware, preserve real text after runtime headers, align prefix trimming, and lock positive and negative cases with focused tests.

## Reproduction Assessment

Yes, at source level. A `buildSessionHistorySnapshot` input containing legacy quote-wrapped runtime/status rows exercises the `chat.history` path, and the PR body includes red-first and after-fix terminal proof.

## Solution Assessment

No. The projection boundary is right, but the current patch is not mergeable until it preserves user tails, restricts legacy status stripping to user/runtime rows, and aligns trim behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with known runtime prompt headers. Legacy startup/reset rows can include actual user text after the header, and current main preserves reset tails as transcript text, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, and it is called for all roles. Assistant messages that quote or explain a legacy `System:` line can be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` checks `text.trimStart()`, but the status strip helpers test and replace against the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the prefix, leaving the status visible.
  - confidence: 0.76
