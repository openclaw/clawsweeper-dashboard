## Summary

Keep open. This PR targets a real Gateway/WebChat history bug that current main still only partially handles, but the current branch still has role-safety and over-stripping defects that should block merge.

## What This Changes

The PR adds Gateway history projection sanitization for legacy quote-wrapped runtime context, background-task status, gateway-restart transcript rows, session-history tests, and a changelog entry.

## Best Possible Solution

Keep the projection-layer fix, but make legacy stripping role-safe, preserve real user tails after legacy prompt headers, add negative regression coverage, then rebase and validate the Gateway history path.

## Reproduction Assessment

Yes, at source level. Current main only strips internal-context delimiters at real line boundaries, while the PR body and tests target quote-wrapped legacy rows and legacy System status prefixes that bypass that path.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the branch needs user-role gating, tail-preserving prompt stripping, and consistent leading-whitespace handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text starting with a legacy prompt header. Legacy reset/startup rows can include real user text after that header, and current reset coverage expects the transcript tail to survive, so this can hide user-authored history.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart `System:` prefixes before the role branch. Because display projection sanitizes all roles, assistant messages that quote those lines can be rewritten or blanked even though this fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but the status stripping helpers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
