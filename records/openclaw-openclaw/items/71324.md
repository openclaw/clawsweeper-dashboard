## Summary

Keep this PR open: the legacy WebChat/history projection fix is useful and not fully present on current main, but the current branch still introduces sanitizer regressions that should block merge.

## What This Changes

The branch adds Gateway/WebChat history sanitization for legacy runtime-only transcript wrappers/status rows, regression tests, and a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection boundary, but repair the sanitizer so only legacy user/runtime rows are stripped while real user tails and assistant-authored content remain visible.

## Reproduction Assessment

Yes, source-level. A buildSessionHistorySnapshot input containing legacy quote-wrapped runtime/status rows exercises the chat.history projection path, and the PR body includes red-first targeted tests plus direct terminal smoke output.

## Solution Assessment

No. The projection layer is the right boundary, but this implementation should preserve prompt tails, role-gate legacy status stripping, align leading-whitespace handling, and add focused positive and negative regression tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: stripLegacyInternalOnlyPrompt returns an empty string for any row that starts with these legacy runtime prompt strings. Legacy startup/reset rows can include actual user text after the header, and current main preserves reset tails as transcript text, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: stripVisibleTranscriptText strips background-task and gateway-restart prefixes before the role-specific branch, and stripEnvelopeFromMessage calls it for all roles. Assistant messages that quote or explain a legacy System line can therefore be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: The legacy status helpers test and replace against the raw string. Persisted rows with leading whitespace before the System prefix will fail to strip even though the new legacy-runtime detection already normalizes similar leading whitespace elsewhere.
  - confidence: 0.72
