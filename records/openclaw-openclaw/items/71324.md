## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with a known legacy runtime prompt. Legacy reset/startup rows can carry real user text after that header, so this can hide user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: The Background task and Gateway restart strippers run before the user-role gate. Since the history projection sanitizes all roles, an assistant message that quotes one of these System lines can be rewritten or blanked even though this cleanup is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be identified as legacy runtime text and then fail to strip the legacy status prefix.
  - confidence: 0.76
