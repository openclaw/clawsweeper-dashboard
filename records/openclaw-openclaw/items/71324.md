## Summary

Keep this PR open because it addresses a real Gateway/WebChat history gap that is not fully handled on current main, but the latest PR head is still not merge-ready due to sanitizer bugs that can drop legitimate visible text.

## What This Changes

This PR adds Gateway WebChat history projection sanitization, regression tests, and a changelog entry for legacy runtime-only transcript rows.

## Best Possible Solution

Keep the projection-layer fix, but make the legacy stripping tail-preserving and user/runtime-row scoped, add negative coverage for assistant quotations and mixed prompt text, then rebase and validate.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot feeds projectChatDisplayMessages through stripEnvelopeFromMessages, and PR-head stripVisibleTranscriptText applies the new prompt/status stripping before role gating.

## Solution Assessment

No, not as currently implemented. The projection boundary is right, but the patch needs tail-preserving prompt stripping, user-role gating for legacy status handling, and consistent prefix trimming before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: stripLegacyInternalOnlyPrompt returns `""` for any row beginning with a known runtime prompt, so a legacy startup/reset row with the real request appended will disappear from history instead of only removing runtime metadata.
  - confidence: 0.9

  - confidence: 0.88

  - body: looksLikeLegacyRuntimeText checks `trimStart()` text, but the strip helpers match the raw string, so a quote-unwrapped legacy status with leading whitespace can be recognized as runtime text and then left visible.
  - confidence: 0.76
