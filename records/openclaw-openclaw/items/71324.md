## Summary

Keep open: current main still has a source-level legacy WebChat/history gap, but this PR head should not merge because the sanitizer can drop real reset-tail user text and rewrite legitimate assistant content.

## What This Changes

This PR adds Gateway/WebChat history projection sanitization for legacy quote-wrapped runtime context, background-task, and gateway-restart transcript rows, plus session-history regression tests and a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection fix, but implement it on current main with role-gated legacy stripping that preserves real user tails and has focused positive and negative regression tests.

## Reproduction Assessment

Yes, at source level. The documented contract says runtime context stays out of visible transcript surfaces, and the current delimiter stripper misses quote-wrapped persisted runtime rows while the PR body includes red-first and after-fix terminal proof.

## Solution Assessment

No. The projection boundary is appropriate, but this branch should not merge until it preserves prompt/reset tails, gates legacy status stripping to user/runtime rows, aligns trim handling, and rebases onto current session-history code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with the legacy reset/startup prompt. Mixed legacy rows can include the user's real reset tail after that header, and current main explicitly preserves that tail in transcript-facing text, so this can hide user-authored history.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, and the display projection calls it for all roles. Assistant messages that quote or explain a legacy `System:` line can therefore be rewritten or blanked even though this fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` checks `trimStart()`, but the status strip helpers test and replace the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the prefix, leaving the status visible.
  - confidence: 0.76
