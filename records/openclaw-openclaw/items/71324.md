## Summary

Keep open: the legacy WebChat/history leak is real and this PR is a focused fix path, but the current patch can still drop legitimate transcript text and should be repaired before merge.

## What This Changes

The PR adds Gateway chat-history sanitization and session-history regression tests to hide already-persisted legacy runtime transcript wrappers/status rows from WebChat history, plus a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection fix, but preserve real user tails, apply legacy status cleanup only to user/runtime transcript rows, and make runtime detection and stripping use the same normalized input before merging.

## Reproduction Assessment

Yes. Source inspection and the PR body establish a high-confidence path through `buildSessionHistorySnapshot` where current main exposes quoted or status-shaped legacy runtime rows, though I did not run tests in this read-only pass.

## Solution Assessment

No, not yet. The display-projection boundary is the right place, but the PR should first preserve prompt tails, role-gate legacy status stripping, and normalize whitespace consistently to avoid suppressing real transcript text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` whenever the text starts with a known legacy runtime prompt. If a persisted legacy row combines that prompt header with real user text, the whole user-authored tail disappears from history instead of only the runtime prefix being removed.
  - confidence: 0.88
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-128`
  - body: `stripVisibleTranscriptText` strips Background-task and Gateway-restart prefixes before checking `stripUserEnvelope`, so assistant messages that quote one of those `System:` lines can be rewritten or blanked in history even though this cleanup is for persisted user/runtime transcript rows.
  - confidence: 0.9
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status stripping helpers test the raw string. A quote-unwrapped legacy row with whitespace before `System:` can be recognized as runtime text and then fail to remove the status prefix.
  - confidence: 0.78
