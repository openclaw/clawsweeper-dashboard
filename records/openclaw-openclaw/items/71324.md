## Summary

Keep open: the bug is valid and the projection-layer direction is useful, but the PR head still introduces sanitizer regressions that can drop real user text and rewrite legitimate assistant quotes.

## What This Changes

The PR adds Gateway/WebChat history projection sanitization and regression tests for legacy quote-wrapped internal context, background-task status, gateway-restart status, subagent completion, and startup prompt transcript rows.

## Best Possible Solution

Implement the legacy cleanup as user-row-only display projection on current main, preserving user-authored tails and assistant quoted text with positive and negative regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main misses quote-wrapped internal-context rows and legacy System-status rows, while the PR tests model those persisted shapes; I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the sanitizer needs user-role gating, tail-preserving legacy prompt cleanup, and consistent leading-whitespace matching before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string whenever the row starts with a known legacy prompt. Reset/startup rows can carry real user text after that header, so this can hide user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: The Background task and Gateway restart strippers run before checking `stripUserEnvelope`. Since history projection sanitizes all roles, an assistant message that quotes one of these System lines can be rewritten or blanked even though the fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be identified as legacy runtime text and then fail to strip the legacy status prefix.
  - confidence: 0.76
