## Summary

Keep this PR open: the legacy WebChat/history leak is still source-reproducible on current main, but the current PR head still has narrow sanitizer correctness bugs and is merge-conflicting against main.

## What This Changes

The PR adds Gateway/WebChat history projection sanitization for legacy quote-wrapped runtime context, background-task status, gateway-restart transcript rows, session-history tests, and a changelog entry.

## Best Possible Solution

Keep the projection-layer fix direction, but make legacy cleanup user-role gated, tail-preserving, trim-consistent, and covered by negative regression tests before landing.

## Reproduction Assessment

Yes, at source level. Current main only strips internal-context delimiters at real line boundaries and lacks legacy System status handling, so quote-wrapped runtime rows and legacy status prefixes can still reach projected history.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the branch needs user-role gating, prompt-tail preservation, consistent leading-whitespace handling, and a rebase before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row starting with a legacy runtime prompt header. If a legacy row has real user text after that header, the PR drops the whole transcript row instead of preserving the user-authored tail.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart `System:` prefixes before checking the message role. Assistant messages that quote those lines can therefore be rewritten or blanked even though this cleanup is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: The legacy-runtime detector tests `trimStart()` text, but the status stripping helpers test and replace the raw string. A quote-unwrapped row with leading whitespace can be classified as runtime text and then fail to strip the status prefix.
  - confidence: 0.76
