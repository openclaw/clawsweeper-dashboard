## Summary

Keep this PR open. The reported WebChat/history leak is a valid bug and the branch is aimed at the right Gateway projection layer, but the latest PR head still has concrete sanitizer defects that can hide user-authored text or rewrite assistant content.

## What This Changes

The PR adds Gateway `chat.history` display sanitization for legacy quoted internal-context/runtime/status transcript rows, drops visible rows that become empty while preserving raw sequence accounting, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy sanitizer role-aware and tail-preserving: strip only known legacy runtime/status material from intended legacy user rows, preserve following user-authored text, leave assistant quotations intact, and add focused negative regressions for those boundaries.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to project PR-head history containing a user row that starts with the legacy startup/reset prompt followed by real text, plus an assistant row that quotes a legacy `System: [...] Background task ...` or Gateway restart line; the code path shows those are dropped or rewritten before role gating.

## Solution Assessment

No. The projection layer is the right place, but this implementation is not yet the narrowest maintainable fix because prompt/status stripping needs role gates, tail preservation, and negative coverage for assistant quotes.

  - confidence: 0.9

  - confidence: 0.88

  - body: `looksLikeLegacyRuntimeText` matches against `text.trimStart()`, but the status stripping helpers test the raw string. A quote-wrapped legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
