## Summary

Keep open. The latest PR head still has blocking sanitizer regressions: it can drop real reset/startup tail text, rewrite assistant messages that quote legacy status lines, and miss leading-whitespace legacy status rows after quote unwrapping.

## What This Changes

The PR adds Gateway `chat.history` display sanitization for legacy quoted runtime prompt/status transcript rows, filters now-empty visible history rows while preserving raw sequence accounting, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy cleanup role-aware and tail-preserving: strip only known legacy runtime/status material from intended legacy user rows, keep following user text, leave assistant quotations intact, and add focused negative regressions for those boundaries.

## Reproduction Assessment

Yes. A source-level reproduction is to project PR-head history containing a user row that starts with the legacy startup/reset prompt followed by real text, plus an assistant row that quotes a legacy `System: [...] Background task ...` or `Gateway restart` line.

## Solution Assessment

No. The projection layer is the right place, but this patch is not yet the best maintainable fix because the sanitizer needs role gates, tail preservation, and trimmed status-prefix matching before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with a known runtime prompt. Legacy `/new` or `/reset` rows can include the startup header followed by actual user text, and current main explicitly preserves reset tails in transcripts, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before the role-specific `stripUserEnvelope` branch. Since `chat.history` display projection sanitizes all roles, assistant messages that quote or explain those `System:` lines can be rewritten or emptied even though the PR is scoped to legacy user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: Runtime detection uses `text.trimStart()`, but the status stripping helpers test and replace the raw string. A quote-wrapped legacy row with leading whitespace can be classified as runtime text during unwrapping and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
