## What This Changes

The PR adds Gateway/WebChat history projection sanitization and regression tests to hide legacy persisted runtime-only transcript rows while preserving raw transcript sequence tracking.

## Best Possible Solution

Keep this as a Gateway display-projection fix, but make the legacy cleanup user-row scoped, tail-preserving for mixed prompt rows, and covered by focused negative regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main misses quote-wrapped internal-context rows and legacy `System:` status rows, and the PR tests model those persisted shapes; I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the PR head still needs user-role gating, tail-preserving legacy prompt stripping, consistent leading-whitespace handling, and a rebase onto current Gateway history code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any message that starts with one of the legacy runtime prompt strings. Mixed legacy rows can carry real user text after that header, so this can hide user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` applies the new Background task and Gateway restart strippers before any user-role guard. Since history projection sanitizes all roles, an assistant message that quotes one of these `System:` lines can be rewritten or blanked even though the fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be identified as legacy runtime text and then fail to strip the legacy status prefix.
  - confidence: 0.76
