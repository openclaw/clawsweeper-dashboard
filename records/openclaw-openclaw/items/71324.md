## Summary

Keep open: the bug is valid and current main still lacks the legacy quote/status projection coverage, but the PR head still has sanitizer correctness issues that can hide legitimate transcript text and rewrite assistant content.

## What This Changes

The PR adds Gateway `chat.history` sanitizer logic and regression tests to hide legacy quote-wrapped runtime context, background-task status, and Gateway-restart status rows from WebChat/history.

## Best Possible Solution

Land a rebased Gateway projection sanitizer that handles legacy user/runtime rows without altering assistant text or dropping real user tails, backed by focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main only strips internal-context delimiters at real line boundaries and has no legacy `System:` Background task or Gateway restart status stripper, so the quoted/status legacy shapes remain visible.

## Solution Assessment

No, not as currently implemented. The display-projection boundary is right, but this PR needs role gating, tail-preserving legacy prompt cleanup, consistent leading-whitespace handling, and a clean rebase onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` blanks any text that starts with a known legacy runtime prompt. If a legacy row has real user text after that header, the entire user-authored tail disappears from history instead of only the runtime prefix being removed.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips Background-task and Gateway-restart status prefixes before checking whether this is a user row. Because the history projection sanitizes all roles, an assistant message that quotes one of these `System:` lines can be rewritten or blanked even though the fix is scoped to legacy user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status strippers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to remove the legacy status prefix.
  - confidence: 0.76
