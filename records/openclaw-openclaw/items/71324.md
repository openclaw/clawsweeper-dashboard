## Summary

Keep this PR open: it targets a valid Gateway/WebChat legacy-history leak, but the current head still has blocking sanitizer correctness bugs and is merge-conflicting with current main.

## What This Changes

The PR adds Gateway/WebChat history sanitization, regression tests, and a changelog entry to hide legacy persisted runtime-only transcript rows while preserving raw transcript sequence tracking.

## Best Possible Solution

Keep the Gateway display-projection approach, but repair it so legacy cleanup is user-row scoped, tail-preserving, whitespace-consistent, rebased onto current main, and covered by focused negative tests.

## Reproduction Assessment

Yes. Source inspection on current main shows quote-wrapped internal-context rows bypass delimiter stripping and legacy System status rows have no sanitizer; the PR tests model those persisted shapes, though this read-only review did not run them.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the PR head still needs user-role gating, tail-preserving legacy prompt stripping, consistent leading-whitespace handling, and a rebase onto current Gateway history code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` whenever text starts with a known legacy runtime prompt. If an old persisted row has that prompt header followed by real user text, this hides the whole user turn instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before any user-role guard. Because chat history projection sanitizes all roles, assistant content that quotes one of those `System:` lines can be rewritten or blanked.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but both status-prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the status prefix.
  - confidence: 0.76
