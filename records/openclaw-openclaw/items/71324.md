## Summary

Keep open: the reported WebChat/history leak is valid, but this PR head still has sanitizer correctness blockers that can hide user-authored text or rewrite legitimate assistant output.

## What This Changes

The PR adds Gateway/WebChat `chat.history` sanitization and regression tests for legacy quote-wrapped runtime context, background-task status lines, gateway-restart status blocks, and related changelog text.

## Best Possible Solution

Land a focused Gateway display-projection fix that strips only legacy user/runtime rows, preserves real user text tails and current main history sequencing, and covers the legacy samples with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `buildSessionHistorySnapshot` projects through `stripEnvelopeFromMessages`, while current main cannot strip quote-wrapped runtime delimiters or legacy `System:` status rows.

## Solution Assessment

No. The projection boundary is the right place, but this patch still needs user-role gating, tail-preserving legacy prompt stripping, consistent leading-whitespace handling, and a rebase that preserves current main history behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any message starting with a legacy runtime prompt header. Mixed legacy rows can have real user text after that header, so the sanitizer should strip only the runtime prefix instead of dropping the whole visible turn.
  - confidence: 0.9
- **[P2] Limit legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-126`
  - body: `stripVisibleTranscriptText` removes Background task and Gateway restart status prefixes before checking the user-role branch. Because history projection runs on all roles, assistant messages that quote those `System:` lines can be rewritten or blanked even though the legacy cleanup is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but both legacy status strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the status prefix.
  - confidence: 0.76
