## Summary

Keep open: the legacy-history bug is valid and the projection boundary is the right place, but the current PR head still has concrete sanitizer regressions that can hide user-authored history or rewrite assistant quotations. The PR body has sufficient terminal-style behavior proof for this non-visual Gateway path, but the branch is currently dirty/merge-conflicting and needs a focused repair plus fresh validation.

## What This Changes

The PR adds Gateway chat history sanitization, regression tests, and a changelog entry to hide quote-wrapped legacy runtime context, background-task, gateway-restart, subagent-completion, and startup-prompt rows from WebChat/history.

## Best Possible Solution

Keep the history-projection fix, but make the sanitizer tail-preserving and user-row scoped, add negative regression coverage for assistant quotations and mixed prompt text, then rebase and validate the PR.

## Reproduction Assessment

Yes, at source level. `buildSessionHistorySnapshot` feeds projected messages through `projectChatDisplayMessages`, and PR-head `stripVisibleTranscriptText` applies legacy stripping before role-specific gating, which exposes the review findings without needing a live WebChat profile.

## Solution Assessment

No, not as currently implemented. The display projection boundary is maintainable, but the PR needs tail-preserving prompt stripping, user-role gating for legacy status handling, and consistent prefix trimming before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` whenever a row starts with a legacy runtime prompt. Legacy `/new` or `/reset` rows can include user text after that prompt header, so this hides user-authored history rather than only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before checking `stripUserEnvelope`, so assistant messages that quote `System: [...] Background task ...` are rewritten or blanked even though the PR targets persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `text.trimStart()`, but the two status-prefix stripping helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix.
  - confidence: 0.76
