## Summary

Keep this PR open: it targets a real Gateway/WebChat history leak, but the current head still has merge-blocking sanitizer regressions that can hide legitimate transcript text or rewrite assistant output.

## What This Changes

The PR adds Gateway `chat.history` display sanitization for legacy quoted runtime prompts, background-task and gateway-restart status rows, empty projected rows, raw sequence accounting, regression tests, and a changelog entry.

## Best Possible Solution

Keep the projection-layer fix, but make legacy stripping role-aware, preserve real text after runtime headers, make prefix matching trim-consistent, and lock both positive and negative cases with focused tests.

## Reproduction Assessment

Yes, at source level. A `buildSessionHistorySnapshot` input containing legacy quote-wrapped runtime/status rows exercises the current `chat.history` path, and current main lacks the PR's status-prefix handling while the PR body provides red-first and after-fix terminal proof.

## Solution Assessment

No. The projection boundary is right, but the current patch is not the best mergeable form until it preserves user tails, restricts status stripping to user/runtime rows, and aligns trim behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known runtime prompt. Legacy startup/reset rows can include the runtime header followed by actual user text, and current main explicitly preserves reset tails as transcript text, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, and it is called for all roles. Assistant messages that quote or explain a legacy `System:` line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` checks `text.trimStart()`, but the status strip helpers test and replace against the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the prefix, leaving the status visible.
  - confidence: 0.76
