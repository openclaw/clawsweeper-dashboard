## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row starting with a known legacy prompt header. If an already-persisted row contains that header followed by real user text, this hides the user-authored tail from WebChat/history instead of preserving it.
  - confidence: 0.88
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before checking whether this is a user envelope. Assistant messages that quote `System: [...] Background task ...` or gateway restart text can therefore be rewritten or emptied in visible history.
  - confidence: 0.9
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests status regexes against `trimStart()` output, but the actual strip helpers test the raw string. A persisted row with leading whitespace can be classified as legacy runtime text but then fail to strip the status prefix.
  - confidence: 0.78
