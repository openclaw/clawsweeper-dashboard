Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` blanks the whole row whenever it starts with a known legacy prompt. If an already-persisted row combines that header with real user text, history loses the user-authored tail instead of removing only the runtime prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips Background-task and Gateway-restart prefixes before checking the message role, and `stripEnvelopeFromMessage` calls it for assistant text too. An assistant message that quotes one of these `System:` lines can be rewritten or emptied in chat.history.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but the status strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to remove the status prefix.
  - confidence: 0.76
