## Summary

Keep open: this is a useful follow-up to the merged visible-transcript fix, but the latest PR head still has sanitizer bugs that can hide legitimate user or assistant-visible history. The remaining work is narrow enough for an automated repair pass before normal merge review.

## What This Changes

The PR adds Gateway/WebChat history projection sanitization and regression tests for legacy runtime-only transcript rows, plus a changelog entry.

## Best Possible Solution

Keep the projection-layer fix, but make the legacy stripping tail-preserving and role-safe, then cover assistant quotations and mixed prompt/header rows with focused regressions.

## Reproduction Assessment

Yes, at source level. PR-head `buildSessionHistorySnapshot` reaches `projectChatDisplayMessages`, which calls the changed sanitizer across projected roles; constructed mixed prompt rows or assistant quotes exercise the failing branches without needing a live WebChat profile.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place, but the sanitizer needs to preserve real tails, gate legacy status/prompt stripping to intended user/runtime rows, and trim before status-prefix matching.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with a known legacy runtime prompt. If an already-persisted startup/reset or async prompt row has real user text after that header, this hides the user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user transcript rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: The new legacy background-task and gateway-restart stripping runs before the role branch, so assistant messages that quote a `System: [...] Background task...` or gateway restart block can be rewritten or blanked in history. Restrict these new legacy rules to the intended user/runtime rows and add assistant-quote negative coverage.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but the status-prefix strip helpers test the raw string. A quote-unwrapped legacy row with leading whitespace can be classified as runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
