## Summary

Keep open: the legacy WebChat/history leak is a valid Gateway bug and current main does not contain this follow-up sanitizer, but the current PR head still introduces blocking regressions in mixed legacy user rows and assistant history text.

## What This Changes

The PR adds Gateway history sanitization and regression coverage to hide already-persisted legacy runtime wrappers/status rows from WebChat/history.

## Best Possible Solution

Keep the Gateway history projection fix, but constrain legacy stripping to legacy user/runtime rows, preserve real user and assistant text, and add negative regressions for reset tails, assistant-quoted status lines, and leading-whitespace status wrappers.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot is the chat.history projection path, the PR body includes red-first targeted checks and direct smoke output, and current-main tests define that runtime-only context is hidden while reset user tails remain visible.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place, but the sanitizer needs role gating, consistent trimming, and tail-preserving legacy prompt stripping before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: stripLegacyInternalOnlyPrompt returns an empty string for any row starting with a legacy runtime prompt header. Legacy startup/reset rows can include real user text after that header, and current main preserves reset tails like "summarize my workspace", so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: stripVisibleTranscriptText strips background-task and gateway-restart prefixes before the user-role branch, and callers pass assistant messages through the same helper. An assistant response that quotes or explains a legacy System line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: looksLikeLegacyRuntimeText classifies trimStart() text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
