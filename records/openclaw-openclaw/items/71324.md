## Summary

Keep open: this is a plausible Gateway history bug fix, but the current branch still has review-blocking sanitizer regressions around preserving user reset tails, role-gating legacy status stripping, and leading-whitespace legacy prefixes.

## What This Changes

The PR adds Gateway chat.history sanitization for legacy runtime transcript wrappers/status rows, session-history regression tests, and a changelog entry.

## Best Possible Solution

Keep the Gateway history projection fix, but make the sanitizer preserve real user tails, apply legacy status stripping only to legacy user/runtime rows, align whitespace handling, and cover positive and negative cases.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot exercises the chat.history projection path, the PR body includes red-first checks and direct smoke output, and the remaining regressions are visible from the PR-head sanitizer code against current-main reset-tail coverage.

## Solution Assessment

No, not as currently implemented. The projection boundary is maintainable, but the sanitizer needs targeted repair before merge so it does not hide user-authored reset tails or rewrite assistant-authored status text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row starting with these legacy runtime prompts. Legacy reset/startup rows can include real user text after the header, and current main preserves that reset tail as transcript text, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before role-specific handling, while callers pass assistant messages through the same helper. An assistant message that quotes a legacy `System:` line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
