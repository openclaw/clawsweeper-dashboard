## Summary

Keep open: the bug is valid and the Gateway display-projection boundary is appropriate, but the current PR head still can hide legitimate transcript text by applying legacy cleanup too broadly and by blanking mixed prompt rows.

## What This Changes

The PR adds Gateway `chat.history` sanitization, regression tests, and a changelog entry to hide legacy runtime-only transcript rows from WebChat history while preserving raw transcript sequence tracking.

## Best Possible Solution

Land a rebase-aware Gateway display sanitizer that only cleans legacy runtime user rows, preserves real user tails and assistant quotes, and keeps raw sequence accounting unchanged with focused regression tests.

## Reproduction Assessment

Yes. Current main's documented `chat.history` projection path lacks quote-wrapped legacy runtime and `System:` status handling, and the delimiter matcher explains why quoted internal-context rows remain visible; I did not execute tests in this read-only pass.

## Solution Assessment

No, not as currently implemented. The display-projection boundary is right, but the sanitizer needs role gating, tail-preserving legacy prompt cleanup, and consistent leading-whitespace handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` blanks the whole row whenever it starts with a known legacy runtime prompt. If an already-persisted row combines that header with real user text, history loses the user-authored tail instead of removing only the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips Background-task and Gateway-restart prefixes before checking the message role. Because history projection sanitizes every role, an assistant message that quotes one of these `System:` lines can be rewritten or emptied as if it were a legacy user/runtime row.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but the status strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to remove the status prefix.
  - confidence: 0.76
