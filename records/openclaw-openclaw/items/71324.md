## Summary

Keep open: current main still has the legacy WebChat history gap, but the PR head has blocking sanitizer defects that can drop real user or assistant transcript text if merged unchanged.

## What This Changes

The PR adds Gateway history sanitization, regression tests, and a changelog entry to hide legacy quote-wrapped runtime context, background-task status, and Gateway restart status rows from WebChat/history.

## Best Possible Solution

Land a rebased Gateway display-projection sanitizer with role-gated legacy stripping, tail preservation, consistent leading-whitespace handling, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main strips runtime context only through generic delimiter-based display sanitizers, so quote-wrapped legacy runtime rows and legacy System status rows can still reach chat.history despite the documented display boundary.

## Solution Assessment

No, not as currently implemented. The display-projection boundary is the right place, but this patch needs role safety, tail-preserving legacy prompt cleanup, and consistent status-prefix matching before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any text that starts with a known legacy runtime prompt. A legacy row like `A new session was started via /new or /reset.\n<actual user text>` would lose the real user-authored tail instead of only removing the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips Background-task and Gateway-restart prefixes before checking whether this is a user row. Since history projection also sanitizes assistant messages, an assistant that quotes one of these `System:` lines can be rewritten or blanked as if it were legacy runtime transcript noise.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but the status-prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to remove the status prefix.
  - confidence: 0.76
