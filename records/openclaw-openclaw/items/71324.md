## Summary

Keep this PR open: it is a useful follow-up for legacy WebChat/history transcript leaks, but the current head still has blocking sanitizer regressions that can hide user-authored tails and rewrite assistant text. The proof in the PR body is enough for this non-visual projection path; the next step is a focused repair/rebase, not closure.

## What This Changes

The branch adds Gateway chat.history sanitization for legacy quoted runtime wrappers and status rows, regression tests, and a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection fix, but constrain legacy stripping to the intended user/runtime rows, preserve real user tails, and add negative regressions for assistant quotations and mixed prompt text.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot feeds projectChatDisplayMessages through stripEnvelopeFromMessages, and the PR-head sanitizer clearly applies the new legacy stripping before role gating.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place, but the patch needs tail-preserving prompt stripping, user-role gating for legacy status handling, and consistent prefix trimming before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: stripLegacyInternalOnlyPrompt returns an empty string for any row starting with a known legacy runtime prompt. Legacy startup/reset rows can include real user text after that header, and current main preserves reset tails like 'summarize my workspace', so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: stripVisibleTranscriptText strips background-task and gateway-restart prefixes before the user-role branch, and callers pass assistant messages through this helper too. An assistant response that quotes or explains a legacy System line can be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: looksLikeLegacyRuntimeText classifies trimStart() text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
