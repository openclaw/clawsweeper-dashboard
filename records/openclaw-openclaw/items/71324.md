## Summary

Keep open: the current-main bug is real, but the PR head is still not merge-ready because the sanitizer can drop real user text and rewrite assistant messages that quote legacy status lines.

## What This Changes

The PR adds Gateway chat-history sanitization and regression tests to hide legacy runtime-only transcript wrappers/status rows from WebChat/history while preserving real user text.

## Best Possible Solution

Port a tightly scoped Gateway display-projection sanitizer onto current main that handles legacy user/runtime rows while preserving assistant content and real user tails, backed by focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main only strips internal-context delimiters at real line boundaries and lacks legacy `System:` Background task/Gateway restart status handling, so the quoted/status legacy shapes remain visible; I did not run tests in this read-only pass.

## Solution Assessment

No, not as currently implemented. The display-projection boundary is right, but this PR needs role gating, tail-preserving legacy prompt cleanup, and consistent leading-whitespace handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:89`
  - body: `stripLegacyInternalOnlyPrompt` blanks the whole text whenever it starts with a known legacy runtime prompt. If an already-persisted row combines that header with real user text, the user-authored tail disappears from history instead of only the runtime prefix being removed.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-126`
  - body: `stripVisibleTranscriptText` strips Background-task and Gateway-restart prefixes before the user-role branch. Because the history projection sanitizes every role, an assistant message that quotes one of these `System:` lines can be rewritten or blanked even though the legacy cleanup is scoped to user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:96-108`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but the status strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to remove the legacy status prefix.
  - confidence: 0.76
