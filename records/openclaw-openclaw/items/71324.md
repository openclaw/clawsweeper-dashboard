## What This Changes

The PR adds Gateway chat.history sanitization for legacy quoted runtime wrappers and status rows, with regression tests and a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection fix, but preserve real user tails, gate legacy status handling to intended user/runtime rows, and add negative regression coverage for assistant quotations.

## Reproduction Assessment

Yes, at source level. projectChatDisplayMessages applies stripEnvelopeFromMessages broadly, and PR-head stripVisibleTranscriptText strips legacy wrappers before user-role gating.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the sanitizer needs role gating, tail preservation, and consistent prefix trimming before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: stripLegacyInternalOnlyPrompt returns an empty string for any row starting with a legacy runtime prompt. Legacy startup/reset rows can append real user text after that header, and current main preserves that transcript tail, so this can hide user-authored history.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: stripVisibleTranscriptText strips background-task and gateway-restart prefixes before the user-role branch, and display projection calls it for assistant rows too. An assistant reply that quotes a legacy System line can be rewritten or blanked in history.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: looksLikeLegacyRuntimeText tests trimStart() text, but the strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the System prefix.
  - confidence: 0.76
