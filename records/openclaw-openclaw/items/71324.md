## Summary

Keep this PR open: the reported WebChat/history leak is valid and current main still has the legacy projection gap, but this PR head still introduces blocking sanitizer regressions around user-tail preservation and role-safe stripping.

## What This Changes

The PR adds Gateway chat history projection sanitization and regression tests to hide legacy quote-wrapped internal context, background-task status, gateway-restart status, subagent completion, and startup prompt rows from WebChat/history, plus changelog text.

## Best Possible Solution

Keep the projection-layer fix, but implement it on current main as user-row-only legacy runtime cleanup that preserves real user tails and assistant quoted text, with positive and negative regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main only strips internal-context delimiters at real line boundaries and lacks legacy status/gateway cleanup, while the PR tests model persisted quote-wrapped and System-status rows. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the PR needs role gating, tail-preserving legacy prompt stripping, consistent leading-whitespace matching, and a rebase/replacement on current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text that starts with a legacy prompt header. Reset/startup transcript rows can carry a real user tail after the runtime header, and existing prompt-prelude coverage keeps soft reset tails as transcript text, so this can hide user-authored history instead of stripping only the legacy prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: The legacy background-task and gateway-restart strippers run before `stripUserEnvelope` is checked. Since chat history projection sanitizes all roles, assistant messages that quote `System: [...] Background task ...` or gateway restart text can be rewritten or blanked even though this fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but both status prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the status prefix, leaving the legacy line visible.
  - confidence: 0.76
