## Summary

Keep open: the Gateway projection boundary is the right place for this fix, and the PR has useful terminal proof, but the current head still has sanitizer regressions that can hide user-authored reset tails and rewrite assistant-authored content.

## What This Changes

The PR adds Gateway chat.history sanitization, session-history regression tests, and a changelog entry for legacy runtime transcript wrappers and status rows.

## Best Possible Solution

Keep the fix in the Gateway history projection layer, but repair the sanitizer to preserve real user tails, restrict legacy status stripping to legacy user/runtime rows, align whitespace handling, and cover those positive and negative cases.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot exercises the chat.history projection path, the PR body includes red-first targeted checks and direct smoke output, and the remaining regressions are visible from the PR-head sanitizer code plus current-main tests for reset-tail preservation.

## Solution Assessment

No, not as currently implemented. The projection boundary is maintainable, but the current patch should be repaired so it does not drop user-authored reset tails, rewrite assistant-authored status text, or miss leading-whitespace legacy prefixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row starting with these legacy prompt headers. Legacy reset/startup rows can include real user text after the header, and current main preserves that reset tail as transcript text, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, while callers pass assistant messages through the same helper. An assistant message that quotes or explains a legacy `System:` line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
