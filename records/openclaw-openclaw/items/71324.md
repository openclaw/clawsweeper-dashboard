## Summary

Keep open: this PR addresses a real legacy WebChat history gap, but the current head still over-strips legitimate user and assistant text and is currently conflicting with main.

## What This Changes

The branch adds Gateway chat.history sanitization and regression tests for legacy quoted runtime-context, background-task, gateway-restart, subagent-completion, and startup-prompt transcript rows, plus a changelog entry.

## Best Possible Solution

Keep the history-projection fix, but apply legacy stripping only to intended user/runtime rows, preserve real user tails after runtime headers or status blocks, and add negative regression coverage for assistant quotations and mixed prompt text.

## Reproduction Assessment

Yes, at source level. `buildSessionHistorySnapshot` feeds `projectChatDisplayMessages` through `stripEnvelopeFromMessages`, and PR-head `stripVisibleTranscriptText` applies the new legacy stripping before role gating.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place, but the patch needs tail-preserving prompt stripping, user-role gating for legacy status handling, and consistent prefix trimming before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row starting with a known legacy runtime prompt. Legacy startup/reset rows can include real user text after that header, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the user-role branch, and callers pass assistant messages through this helper too. An assistant response that quotes a legacy `System:` line can therefore be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
