## Summary

Keep this PR open: the legacy WebChat/history leak is still source-reproducible on current main and the projection-layer direction is useful, but the current branch still has blocking sanitizer regressions and is stale against the latest Gateway history code.

## What This Changes

This PR adds Gateway chat history sanitization and regression tests to hide legacy quote-wrapped internal context, background-task and gateway-restart status rows, startup prompts, and subagent completion rows from WebChat/history, plus a changelog entry.

## Best Possible Solution

Keep the projection-layer fix, but make legacy runtime cleanup user-row-only, tail-preserving, and trim-consistent, then rebase it onto current main with positive and negative regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main only strips internal-context delimiters at real line boundaries and lacks legacy System-status cleanup, while the PR tests model the persisted quote-wrapped and status-row shapes. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place, but the PR needs role gating, tail-preserving legacy prompt stripping, consistent leading-whitespace matching, and a current-main rebase before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text starting with a legacy prompt header. Reset/startup transcript rows can carry real user text after the runtime header, and existing prompt-prelude coverage expects soft reset tails to remain visible, so this can hide user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart status prefixes before checking `stripUserEnvelope`. Since chat history projection sanitizes all roles, an assistant message that quotes `System: [...] Background task ...` or a gateway restart block can be rewritten or blanked even though the legacy cleanup is scoped to persisted runtime/user rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but both status prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be accepted as legacy runtime text and then fail to strip the status prefix, leaving the legacy line visible.
  - confidence: 0.76
