## Summary

Keep open. The Gateway display-projection direction is appropriate, but the current PR head still introduces sanitizer regressions: it can drop real user text after legacy prompt headers and can rewrite assistant messages that legitimately quote legacy status lines.

## What This Changes

The PR adds Gateway WebChat/history projection sanitization for legacy runtime quote/status transcript rows, filters now-empty visible history rows while preserving raw transcript sequence accounting, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the Gateway history projection approach, but make the legacy sanitizer role-aware and tail-preserving: strip known legacy runtime/status material only from intended legacy user rows, preserve following user-authored text, leave assistant quotations intact, trim status-prefix matching consistently, and cover those boundaries with focused regressions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project PR-head history with a user row that starts with the legacy startup/reset prompt followed by real text, plus an assistant row that quotes a legacy Background task or Gateway restart line.

## Solution Assessment

No. The projection layer is the right place, but the current implementation is not the narrowest maintainable fix until legacy prompt/status stripping is gated to the intended role and preserves real transcript tails.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve real text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known legacy runtime prompt. Legacy startup/reset rows can include that header followed by real user text, and current main keeps reset tails visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before any role gate, and callers pass assistant content through the same helper. Assistant messages that quote or explain those `System:` lines can therefore be rewritten or emptied even though the PR is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: The runtime detector trims leading whitespace, but the Background task and Gateway restart stripping helpers test the raw string. A legacy status row with leading whitespace will be classified as runtime-shaped in some paths but then fail to strip, leaving the status visible.
  - confidence: 0.72
