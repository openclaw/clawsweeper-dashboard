## Summary

Keep this PR open, but it still needs changes before merge. The Gateway history projection is the right surface and the PR body supplies usable terminal proof, but the current patch can drop real user text after legacy prompt headers and can rewrite legitimate assistant messages that quote legacy status lines.

## What This Changes

The PR adds Gateway chat.history sanitization, regression tests, and a changelog entry for hiding quote-wrapped legacy runtime context, background-task, gateway-restart, subagent-completion, and startup-prompt transcript rows.

## Best Possible Solution

Keep the legacy-history projection fix in the shared Gateway display path, but strip only trusted legacy user/runtime rows, preserve user tails and assistant quotations, and add negative regressions for those cases.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot uses projectChatDisplayMessages and stripEnvelopeFromMessages, and the PR-head sanitizer clearly drops legacy prompt-prefixed rows and applies legacy status stripping before role gating.

## Solution Assessment

No, not as currently implemented. The projection boundary is maintainable, but the patch needs tail-preserving legacy prompt stripping, user/runtime-role gating for legacy status handling, consistent trimming, and focused negative tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row starting with a legacy runtime prompt header. Legacy startup/reset rows can include real user text after that header, and current main preserves reset tails like `summarize my workspace`, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the user-role branch, and callers pass assistant messages through the same helper. An assistant response that quotes or explains a legacy `System:` line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
