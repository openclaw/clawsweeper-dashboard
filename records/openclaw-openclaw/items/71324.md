## Summary

Keep open: the PR addresses a real Gateway WebChat/history leak, but the current patch can hide legitimate user and assistant transcript text and needs narrow sanitizer repairs before merge.

## What This Changes

The branch adds Gateway history sanitization and session-history regression coverage to hide legacy quote-wrapped runtime prompts and status rows while keeping raw transcript sequence accounting.

## Best Possible Solution

Keep the projection-layer fix, but preserve real text after runtime headers, restrict legacy status stripping to user/runtime transcript rows, align prefix trimming, and add focused positive and negative regression tests.

## Reproduction Assessment

Yes, at source level. A buildSessionHistorySnapshot input containing legacy quote-wrapped runtime/status rows exercises the chat.history projection path, and the PR body includes red-first and after-fix terminal proof.

## Solution Assessment

No. The projection boundary is the right place, but this patch is not the best mergeable form until it preserves user tails, role-gates legacy status stripping, and handles leading whitespace consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: stripLegacyInternalOnlyPrompt returns an empty string for any row that starts with one of these legacy runtime prompt strings. Legacy startup/reset rows can include actual user text after the header, and current main preserves reset tails as transcript text, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: stripVisibleTranscriptText strips background-task and gateway-restart prefixes before the role-specific branch, and stripEnvelopeFromMessage calls it for all roles. Assistant messages that quote or explain a legacy System line can therefore be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: looksLikeLegacyRuntimeText checks text.trimStart(), but the status strip helpers test and replace against the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the prefix, leaving the status visible.
  - confidence: 0.76
