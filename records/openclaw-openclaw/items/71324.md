## Summary

Keep open: the underlying WebChat/history bug is valid, but this PR still has correctness blockers in the sanitizer and the head is currently conflicting with main.

## What This Changes

The PR adds Gateway/WebChat history projection sanitization and regression coverage for legacy quote-wrapped internal-context prompts, background-task status lines, and Gateway restart status blocks.

## Best Possible Solution

Keep the cleanup as a Gateway display projection, but make legacy runtime status stripping user-row-only, preserve real tail text after legacy prompt headers, normalize leading whitespace consistently, and rebase onto the current async session-history APIs.

## Reproduction Assessment

Yes, at source level. Current main strips normal internal-context blocks but misses quote-wrapped delimiters and legacy System-status rows, and the PR tests model those persisted shapes.

## Solution Assessment

No, not as currently implemented. The projection boundary is right, but the PR head needs role gating, tail-preserving prompt stripping, whitespace-consistent status matching, and a rebase before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string whenever a row starts with one of the legacy runtime prompt strings. Mixed legacy rows can carry real user text after that header, so this can hide user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` applies the new Background task and Gateway restart strippers before checking `stripUserEnvelope`. Since history projection sanitizes all roles, an assistant message that quotes one of these System lines can be rewritten or blanked even though the fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be identified as legacy runtime text and then fail to strip the legacy status prefix.
  - confidence: 0.76
