## Summary

Keep open: this is a useful Gateway/WebChat cleanup, but the current PR head still has sanitizer regressions that can hide real user text and rewrite assistant-authored text before merge.

## What This Changes

The PR adds Gateway `chat.history` sanitization for legacy runtime transcript wrappers/status rows, regression tests, and a changelog entry.

## Best Possible Solution

Keep the fix in the Gateway history projection layer, but make the sanitizer preserve real user tails, apply legacy status stripping only to legacy user/runtime rows, align whitespace handling, and add positive/negative regressions before merge.

## Reproduction Assessment

Yes, at source level. `buildSessionHistorySnapshot` exercises the `chat.history` projection path, and the PR body includes red-first targeted tests plus direct terminal smoke output for the legacy persisted shapes.

## Solution Assessment

No. The projection boundary is right, but the current implementation should be repaired so it does not drop reset tails, rewrite assistant-authored content, or miss leading-whitespace legacy status prefixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` drops the entire row whenever it starts with one of these legacy prompt headers. Legacy reset/startup rows can include real user text after the header, and current main preserves that reset tail as transcript text, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, while `stripEnvelopeFromMessage` calls it for assistant content too. An assistant message that quotes or explains a legacy `System:` line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
