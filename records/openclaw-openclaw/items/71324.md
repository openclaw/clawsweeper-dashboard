## Summary

Keep open: the legacy WebChat history leak is still plausible on current main, but this PR head is not merge-ready because the sanitizer can drop legitimate user or assistant text in edge cases already called out by prior review.

## What This Changes

The PR adds Gateway chat.history display sanitization for legacy quote-wrapped runtime context, background-task status, and gateway-restart transcript rows, plus session-history regression tests and a changelog entry.

## Best Possible Solution

Keep the display-projection fix direction, but make the legacy stripping role-safe, tail-preserving, and covered by negative regression tests before merge.

## Reproduction Assessment

Yes, at source level. Current main only strips internal-context delimiters at real line boundaries, while the PR body and tests target quote-wrapped legacy rows and legacy System status prefixes that bypass that path.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place, but the PR needs user-row gating for legacy status stripping, tail-preserving legacy prompt handling, and consistent leading-whitespace matching.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve real text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row starting with a known legacy prompt header. Legacy reset/startup rows can carry real user text after that runtime header, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before checking `stripUserEnvelope`, and this helper is also called for assistant messages. An assistant response that quotes a legacy `System:` line can therefore be rewritten or blanked even though the fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but both legacy status strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
