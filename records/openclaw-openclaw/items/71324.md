## Summary

Keep open. The latest PR head still has narrow but real sanitizer regressions: it can drop user-authored reset/startup tail text and can rewrite assistant messages that quote legacy `System:` status lines, even though the intended fix is limited to legacy user/runtime transcript rows.

## What This Changes

The PR adds Gateway `chat.history` display sanitization for legacy runtime/status transcript rows, drops now-empty visible history rows while preserving raw sequence accounting, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy sanitizer role-aware and tail-preserving: strip only known legacy runtime/status material from intended legacy user rows, preserve following user-authored text, leave assistant quotations intact, and add focused negative regressions for those boundaries.

## Reproduction Assessment

Yes. A source-level reproduction is to project PR-head history containing a user row that starts with the legacy startup/reset prompt followed by real text, plus an assistant row that quotes a legacy Background task or Gateway restart line.

## Solution Assessment

No. The projection layer is the right place, but this PR is not yet the narrowest maintainable solution because prompt/status stripping needs role gates, tail preservation, and negative coverage for assistant quotes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` whenever a row starts with a known legacy runtime prompt. Legacy reset/startup rows can include that header followed by actual user text, and current main explicitly keeps reset tails visible, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before the role-specific `stripUserEnvelope` branch. Because chat display projection sanitizes all roles, assistant messages that quote or explain those `System:` lines can be rewritten or blanked even though the PR is scoped to legacy user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` matches against `text.trimStart()`, but the status stripping helpers test the raw string. A quote-wrapped legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
