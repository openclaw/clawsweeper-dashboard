## Summary

Keep open. The PR targets a real Gateway history leak left after the forward write-path fix, but current head still has blocking sanitizer boundary bugs: it can drop user-authored reset/startup tails and rewrites assistant messages that quote legacy status text.

## What This Changes

This PR adds Gateway `chat.history` display sanitization for legacy runtime-only transcript rows, adds regression coverage in `session-history-state.test.ts`, and adds a user-facing changelog entry.

## Best Possible Solution

Keep the display-projection approach, but make the legacy sanitizer role-aware and tail-preserving: strip known legacy runtime/status material from intended legacy user rows, preserve following user-authored text, leave assistant quotations intact, and add focused negative regressions for those boundaries.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. On PR head, project a history row that starts with the legacy startup prompt followed by real user text, and an assistant row that quotes a `System: [...] Background task ...` or `Gateway restart` line; both paths follow directly from the changed sanitizer code.

## Solution Assessment

Is this the best way to solve the issue? No. The projection layer is the right place, but the current implementation is not narrow enough until prompt/status stripping is constrained to the legacy user-row cases and preserves real transcript tails.
