## Summary

Keep open. The reported WebChat/history bug is valid and the PR has useful source work plus terminal proof, but the current branch still has sanitizer correctness bugs and is conflicting with newer `main` Gateway history changes.

## What This Changes

The PR adds Gateway history display sanitization and regression coverage to hide legacy quote-wrapped runtime context, background-task status, gateway-restart status, subagent-completion, and startup-prompt transcript rows.

## Best Possible Solution

Land a small current-main repair that keeps the display-projection boundary, strips only trusted legacy user/runtime wrappers, preserves real user tails, and adds regression tests for assistant quotes and leading-whitespace status rows.

## Reproduction Assessment

Yes. Source inspection shows current `main` cannot strip quote-wrapped internal-context rows and has no legacy Background task or Gateway restart status stripper; the PR body also documents red-first targeted projection checks, though I did not run tests in this read-only review.

## Solution Assessment

No, not as currently implemented. The projection boundary is right, but the sanitizer must preserve real text after legacy prompt headers, apply status stripping only on user/runtime rows, handle leading whitespace consistently, and rebase onto current `main`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any text that starts with a known legacy runtime prompt. Mixed legacy rows can contain real user text after that header, so this can hide user-authored history instead of removing only the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-126`
  - body: `stripVisibleTranscriptText` strips Background task and Gateway restart status text before checking the user-role branch. Because history projection sanitizes all roles, an assistant message that quotes one of those `System:` lines can be rewritten or blanked even though this fix is scoped to legacy user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but both status strippers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix.
  - confidence: 0.76
