## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string whenever a row starts with a known legacy runtime prompt. Legacy reset/startup rows can carry real user text after that header, so this can hide user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: The new Background task/Gateway restart stripping runs before checking whether the message role is `user`. Because `projectChatDisplayMessages` sanitizes all roles, an assistant message that quotes a legacy `System:` line can be rewritten or emptied as if it were a legacy user/runtime row.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests a `trimStart()` value, but both status strippers test the raw string. Quote-unwrapped rows with leading whitespace can be classified as legacy runtime text and then fail to strip the status prefix.
  - confidence: 0.76
