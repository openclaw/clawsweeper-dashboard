## What This Changes

This PR adds Gateway/WebChat history projection sanitization and tests to hide legacy quote-wrapped runtime context, background-task status, and Gateway restart status rows while preserving raw transcript sequence tracking.

## Best Possible Solution

Land a current-main port of the Gateway/WebChat display projection that handles legacy quote/status rows without mutating assistant content or dropping real user text.

## Reproduction Assessment

Yes. Source inspection on current main shows quote-wrapped internal runtime context bypasses the delimiter scanner, and the display sanitizer has no legacy Background task or Gateway restart status stripper.

## Solution Assessment

No, not yet. The projection boundary is right, but the PR must preserve mixed user tails, scope legacy status stripping to user rows, trim consistently, and rebase onto current main's shared display sanitizer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any message that starts with one of the legacy runtime prompt strings. Mixed legacy rows can carry real user text after that header, so this hides user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips Background task and Gateway restart status text before checking `stripUserEnvelope`. Because history projection sanitizes all roles, an assistant message quoting one of those `System:` lines can be rewritten or blanked even though the fix is scoped to legacy user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but both status strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the legacy status prefix.
  - confidence: 0.76
