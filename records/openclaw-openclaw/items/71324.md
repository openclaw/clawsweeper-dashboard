## Summary

Keep open: this is a valid Gateway/WebChat history bug and current main still lacks the legacy quote/status projection, but this PR is not merge-ready because the latest head still has blocking sanitizer bugs and is marked dirty against current main. The earlier merged work in https://github.com/openclaw/openclaw/pull/71229 fixed the forward write path, while this PR is still useful as the legacy read/projection follow-up.

## What This Changes

The branch adds Gateway `chat.history` sanitization and regression tests to hide legacy quote-wrapped runtime context, background-task status, and gateway-restart status rows from WebChat/history while preserving raw transcript sequence tracking.

## Best Possible Solution

Keep the Gateway display-projection approach, but land it only after the sanitizer preserves user text tails, scopes legacy status stripping to user rows, handles leading whitespace consistently, and rebases cleanly on current main.

## Reproduction Assessment

Yes. Source inspection shows current main cannot strip quote-wrapped internal-context rows because delimiter matching requires a real line boundary, and current main has no legacy `System:` background-task or Gateway-restart status stripper.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place, but this PR still needs user-role gating, tail-preserving prompt cleanup, consistent leading-whitespace handling, and a rebase before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any message that starts with one of the legacy runtime prompt strings. Mixed legacy rows can carry real user text after that header, so this hides user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-126`
  - body: `stripVisibleTranscriptText` strips Background task and Gateway restart status text before checking `stripUserEnvelope`. Because history projection sanitizes all roles, an assistant message quoting one of those `System:` lines can be rewritten or blanked even though the fix is scoped to legacy user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests `trimStart()` text, but both status strippers test the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the legacy status prefix.
  - confidence: 0.76
