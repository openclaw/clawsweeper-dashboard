## Summary

Keep this PR open, but it still needs changes before merge. The legacy history leak is real and the Gateway display-projection layer is the right place to handle it, but the current patch can drop user-authored tail text and rewrite assistant messages that legitimately quote legacy `System:` status lines.

## What This Changes

The PR adds Gateway `chat.history` display sanitization for legacy quote-wrapped internal-context prompts, background-task and gateway-restart status rows, empty projected rows, raw sequence accounting, regression tests, and a changelog entry.

## Best Possible Solution

Keep the display-projection sanitizer, but make it role-aware, preserve real text after known runtime headers, make status-prefix matching trim-consistent, and cover both positive and negative cases with focused tests.

## Reproduction Assessment

Yes, at source level. A `buildSessionHistorySnapshot` input containing a legacy quote-wrapped background-task or gateway-restart row exercises the current `chat.history` path, and current main lacks status-prefix handling while the PR body provides red-first and after-fix terminal proof.

## Solution Assessment

No. The projection layer is the right solution boundary, but this patch is not mergeable until it preserves user tails, restricts legacy status stripping to user/runtime rows, and aligns trim behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known runtime prompt. Legacy startup/reset rows can include the runtime header followed by actual user text, and current main explicitly preserves reset tails as transcript text, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, and it is called for all roles. Assistant messages that quote or explain a legacy `System:` line can be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` checks `text.trimStart()`, but the status strip helpers test and replace against the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip the prefix, leaving the status visible.
  - confidence: 0.76
