## Summary

Keep open: the legacy-history cleanup is a valid bug-fix direction and the PR body has usable terminal proof, but the current PR head still introduces sanitizer regressions that can hide real user or assistant-visible transcript text.

## What This Changes

The PR adds Gateway WebChat/history sanitization for legacy runtime-only transcript rows, regression tests, and a changelog entry.

## Best Possible Solution

Keep the projection-layer fix, but make legacy stripping role-aware, preserve real user tails after legacy runtime headers, trim consistently before status matching, and add negative regressions for those cases before merge.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot is the chat.history projection path, the PR body includes red-first and after-fix terminal proof for that path, and current-main tests define that reset user tails must remain transcript-visible.

## Solution Assessment

No, not as currently implemented. The projection boundary is the right place to solve the bug, but the sanitizer needs narrower role gating, tail-preserving prompt stripping, and consistent trimming before this patch is safe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: stripLegacyInternalOnlyPrompt returns an empty string for any row starting with a legacy runtime prompt header. Legacy startup/reset rows can include real user text after that header, and current main preserves reset tails, so this can hide user-authored history instead of only removing runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: stripVisibleTranscriptText strips background-task and gateway-restart prefixes before the user-role branch, and callers pass assistant messages through the same helper. An assistant response that quotes or explains a legacy System line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: looksLikeLegacyRuntimeText classifies trimStart() text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
