## Summary

Keep open. Current main still has the legacy history-projection gap this PR targets, but the PR head is not safe to merge because its sanitizer can drop real user text and rewrite assistant quotes before role gating.

## What This Changes

The PR adds Gateway chat.history display sanitization for legacy quoted internal-context/runtime/status transcript rows, drops rows that become empty, preserves raw sequence accounting, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy sanitizer role-aware and tail-preserving with focused positive and negative regression coverage.

## Reproduction Assessment

Yes. The PR-head source path shows a user row starting with a legacy startup/reset prompt plus real text is emptied, and assistant rows quoting legacy System status lines can be rewritten before role gating.

## Solution Assessment

No. The projection layer is the right location, but this patch is not yet the narrowest maintainable fix because it needs role gates, tail preservation, and trim-consistent status matching.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: stripLegacyInternalOnlyPrompt returns an empty string for any row whose trimmed text starts with a known legacy runtime prompt. Legacy startup/reset rows can include that header followed by actual user text, and current main explicitly keeps reset tails visible, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: stripVisibleTranscriptText strips legacy Background task and Gateway restart prefixes before the role-specific user-envelope branch. Because chat display projection sanitizes all roles, assistant messages that quote or explain those System lines can be rewritten or blanked even though the PR is scoped to legacy user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: looksLikeLegacyRuntimeText matches against text.trimStart(), but the status strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be classified as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
