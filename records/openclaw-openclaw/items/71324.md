## Summary

Keep this PR open: the reported WebChat/history leak is valid and current main still needs the legacy display projection, but the PR head still has blocking sanitizer correctness issues and is conflicting with the current Gateway history code.

## What This Changes

The PR adds Gateway/WebChat history projection sanitization and regression coverage for legacy persisted runtime-only transcript wrappers, background-task status lines, gateway restart status blocks, and startup/subagent prompt rows.

## Best Possible Solution

Keep the legacy cleanup as a Gateway display projection, but make it user-row-safe, preserve real text after stripped prefixes, add negative regression coverage, and rebase it onto current session-history APIs.

## Reproduction Assessment

Yes. Source inspection shows current main misses quote-wrapped internal-context rows and legacy System-status rows, while the docs define runtime context as non-user transcript content; I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently implemented. The display-projection boundary is appropriate, but the PR needs role gating, tail-preserving stripping, leading-whitespace consistency, and a current-main rebase before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known legacy runtime prompt. If a legacy row contains real user text after that header, this drops the whole visible turn instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips Background task and Gateway restart prefixes before checking whether this is a user envelope. Assistant messages that quote those `System:` lines can therefore be rewritten or blanked in history even though the legacy leak is scoped to user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests a `trimStart()` copy, but both legacy status strippers test the raw string. A quote-unwrapped legacy row with leading whitespace can be classified as runtime text and then fail to strip the matching prefix.
  - confidence: 0.76
