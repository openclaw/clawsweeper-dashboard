## Summary

Keep open: the PR targets a real Gateway/WebChat history bug, but the current head still has concrete sanitizer regressions that can hide valid user text or rewrite assistant content, and the branch is currently conflicting with main.

## What This Changes

The PR adds Gateway chat.history sanitization, regression tests, and a changelog entry to hide already-persisted legacy runtime-only transcript rows from WebChat/history.

## Best Possible Solution

Keep the fix in the Gateway history projection layer, but repair the sanitizer to preserve real user tails, gate legacy status stripping to user-role legacy rows, align whitespace handling, add negative regressions, and rebase against current main.

## Reproduction Assessment

Yes, at source level. buildSessionHistorySnapshot is the chat.history projection path, the PR body includes red-first targeted checks and direct smoke output, and the remaining PR-head regressions are visible from the sanitizer code plus current-main reset-tail coverage.

## Solution Assessment

No, not as currently implemented. The projection boundary is maintainable, but the patch should be repaired so it does not drop user-authored reset tails, rewrite assistant-authored status text, or miss leading-whitespace legacy prefixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any row that starts with one of these legacy runtime prompt strings. Legacy /new or /reset rows can include real user text after the startup header, and current main preserves that reset tail as transcript text, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user messages:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips background-task and gateway-restart prefixes before the role-specific branch, while callers pass assistant messages through the same helper. An assistant message that quotes or explains a legacy `System:` line can be rewritten or blanked even though this PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status-prefix strip helpers test the raw string. A quote-unwrapped row with leading whitespace can be recognized as legacy runtime text and then fail to strip the status prefix, leaving it visible.
  - confidence: 0.76
