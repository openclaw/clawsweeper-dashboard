## Summary

Keep open. The PR targets a real legacy WebChat history leak, but the latest head still has blocking sanitizer regressions: it can drop real user text after legacy prompt headers, rewrite assistant content that merely quotes legacy status lines, and misses leading-whitespace status rows after quote unwrapping.

## What This Changes

The PR adds Gateway WebChat/chat.history projection sanitization for legacy runtime-only transcript wrappers and status rows, filters empty visible history rows while preserving raw sequence accounting, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make the legacy sanitizer role-aware and tail-preserving: hide only intended legacy runtime/status material from legacy user/runtime rows, preserve following user text, leave assistant quotations intact, and cover those boundaries with focused regressions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to run the PR-head sanitizer/projection on a user row beginning with the legacy /new or /reset prompt followed by real text, and on an assistant row that quotes a legacy System: Background task or Gateway restart line.

## Solution Assessment

No. The projection layer is the right place, but the current implementation is not the narrowest maintainable fix until legacy prompt/status stripping is restricted to the intended rows and preserves non-runtime tails.

## Review Findings

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known legacy runtime prompt. Legacy startup/reset rows can include that prompt followed by real user text, and current main has a reset-tail contract that keeps the tail visible, so this can hide user-authored history instead of only removing runtime text.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy Background task and Gateway restart prefixes before any role gate, and callers pass assistant content through this helper. Assistant messages that quote or explain those `System:` lines can therefore be rewritten or blanked even though the PR is scoped to persisted user/runtime rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: Runtime detection trims leading whitespace, but the status stripping helpers test the raw string. A quoted legacy row with leading whitespace can be unwrapped as runtime text and then fail to strip the status prefix, leaving the legacy status visible.
  - confidence: 0.76
