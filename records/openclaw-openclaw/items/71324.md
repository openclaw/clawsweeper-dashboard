## Summary

Keep open: the PR targets a real Gateway/WebChat legacy-history gap that current main does not fully cover, but the current head still has merge-blocking sanitizer regressions that can hide legitimate user or assistant text.

## What This Changes

The PR adds Gateway history sanitization for legacy quote-wrapped internal-context prompts, background-task and gateway-restart status rows, empty projected rows, raw sequence accounting, regression tests, and a changelog entry.

## Best Possible Solution

Keep the projection-layer fix, but make it role-aware, preserve visible tails after known runtime headers, align trim handling, and cover both positive and negative cases with focused regression tests.

## Reproduction Assessment

Yes, at source level. A `buildSessionHistorySnapshot` input with quote-wrapped runtime/status rows exercises the `chat.history` path, and current main lacks the PR's background-task and gateway-restart prefix handling while the PR body includes red-first and after-fix terminal proof.

## Solution Assessment

No. The projection layer is the right boundary, but this implementation should not merge until it preserves visible tails, gates legacy status stripping to the right roles, and handles leading whitespace consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any row that starts with a known runtime prompt. Legacy prompt rows can carry real visible text after the header, and the current transcriptPrompt path preserves the visible prompt separately, so this can hide user-authored history instead of removing only runtime metadata.
  - confidence: 0.9
- **[P2] Gate legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` removes background-task and gateway-restart prefixes before considering `stripUserEnvelope`, and callers invoke it for assistant messages too. An assistant message that quotes a `System: [...] Background task done:` line can be rewritten or blanked even though the PR is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` tests a trimmed string, but the status strip helpers test and replace the raw string. A quote-unwrapped row with leading whitespace can be classified as legacy runtime text and then fail to strip, leaving the status visible.
  - confidence: 0.76
