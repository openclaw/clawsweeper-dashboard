## Summary

This is a valid Gateway/WebChat bug-fix candidate, but the current patch can still remove legitimate visible text and needs repair before merge.

## What This Changes

The PR adds Gateway history projection sanitization and regression coverage for legacy runtime-only transcript rows in WebChat/history.

## Best Possible Solution

Keep the cleanup at the Gateway display-projection boundary, but make legacy status stripping user-row-only, preserve real text following runtime prompt headers, and add negative regression coverage for assistant quotations and mixed prompt rows.

## Reproduction Assessment

Yes. Current main misses quote-wrapped internal-context rows and legacy System-status rows in the chat history projection path, and the PR tests model those persisted shapes.

## Solution Assessment

No, not as currently implemented. The projection boundary is appropriate, but the sanitizer needs user-role gating, tail-preserving legacy prompt stripping, and consistent leading-whitespace handling before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:82-90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string as soon as a row starts with one of the legacy runtime prompt strings. Mixed legacy rows can carry real user text after that header, so this can hide user-authored history instead of stripping only the runtime prefix.
  - confidence: 0.9
- **[P2] Restrict legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` applies the Background task and Gateway restart strippers before checking whether the message is a user row. Since history projection sanitizes all roles, an assistant message that quotes one of these System lines can be rewritten or blanked even though the fix is scoped to legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:95-106`
  - body: `looksLikeLegacyRuntimeText` classifies `trimStart()` text, but the status prefix strippers test the raw string. A quote-unwrapped row with leading whitespace can be identified as legacy runtime text and then fail to strip the legacy status prefix.
  - confidence: 0.76
