## Summary

Keep open. Current main does not implement the requested device-list metadata columns, and this PR is a useful implementation attempt, but it should not merge yet because it leaves pending-list rendering unchanged while adding expectations for it and it drops existing `sanitizeForLog` protection on paired-device table cells.

## What This Changes

The PR adds device-list metadata types and formatting helpers, expands paired `openclaw devices list` output with Device ID, Name, Client, OS, IP, and Last active columns, updates CLI tests, and changes the `devices remove` description.

## Best Possible Solution

Revise the PR so both pending and paired device tables expose the requested device ID, name, client, OS, IP, and activity details where data exists, while sanitizing every device-controlled terminal cell and leaving broader naming/detail workflow decisions to #23916.

## Reproduction Assessment

Yes. The failure is reproducible from the diff: the PR branch test expects pending metadata output, but the branch production pending table still renders only the old pending columns; the sanitization regression is also covered by the existing terminal-output test fixture.

## Solution Assessment

No, not yet. The direction is appropriate and narrow because the pairing model already has the metadata, but the implementation needs pending-table rendering and restored `sanitizeForLog` coverage before it is the best fix.
