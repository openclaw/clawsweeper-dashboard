## Summary

Keep open, but not merge-ready: the change is a useful display slice for the open device-identification work, yet the current PR head still misses pending-table rendering and regresses terminal-output sanitization for paired-device metadata.

## What This Changes

The PR updates `openclaw devices list` table output/tests to expose device ID, name, client, OS, and last-activity metadata, and updates `devices remove` help text.

## Best Possible Solution

Revise or replace this PR so pending and paired tables expose the available ID, name, client, OS, IP, and activity fields with every device-controlled cell sanitized; keep durable rename/notes in https://github.com/openclaw/openclaw/issues/13870 and coordinate broader display details through https://github.com/openclaw/openclaw/issues/23916.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the pending table still uses the old columns while new tests expect pending client/OS/device ID output, and the paired table passes raw device metadata into terminal rendering.

## Solution Assessment

No. The direction is maintainable, but this branch is not the best fix until pending rendering matches the tests, terminal sanitization is restored for all new metadata cells, and the timestamp/display overlap is coordinated with the related open work.
