## Summary

Keep open. Current main still lacks a normal route-binding workspace override, and the related implementation PRs are closed unmerged; this is a new config and routing capability that needs maintainer product/security approval before implementation.

## What This Changes

Request to add an optional `bindings[].workspace` override so one agent can use separate workspaces for different channel conversations.

## Best Possible Solution

If accepted, implement a typed, documented `bindings[].workspace` override with validation, route propagation, representative channel and outbound coverage, stale-override clearing, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence the capability is absent because the strict route-binding schema rejects the key and runtime workspace resolution remains agent-scoped.

## Solution Assessment

Unclear until maintainer approval. The requested precedence is plausible, but the maintainable solution needs explicit security/product decisions for workspace trust boundaries, persistence across interactions, and stale override clearing.
