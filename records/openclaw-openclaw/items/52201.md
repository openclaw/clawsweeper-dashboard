## Summary

Keep open. Current main does not implement route-binding workspace overrides, and the closest shipped surface is ACP-only `bindings[].acp.cwd`, so this remains a public config feature and product decision rather than cleanup-closeable.

## What This Changes

Add an optional route-binding `workspace` override so the same agent can use different workspaces and context per matched conversation.

## Best Possible Solution

Decide whether route bindings should gain a public workspace override; if accepted, add it through shared routing/context contracts with schema, generated config docs, channel coverage, and tests that clarify how it differs from ACP `cwd`.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs inspection give high confidence that current main still lacks the requested route-binding `workspace` field.

## Solution Assessment

Unclear until maintainers decide the product shape. A direct `bindings[].workspace` implementation may be viable, but it should be weighed against the existing ACP-only `bindings[].acp.cwd` surface and the documented per-agent isolation model.
