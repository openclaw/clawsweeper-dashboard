## Summary

Keep open: current main has promotion gating, live rehydration, and size-budget compaction for auto-promoted sections, but it does not implement the requested rule lifecycle, last-affirmed audit, or confidence-based retirement model. This is a feature/product decision, not a cleanup-close candidate.

## What This Changes

Add a lifecycle and audit mechanism for Dreaming-promoted `MEMORY.md` rules so stale entries can be dated, confidence-decayed, reviewed, and retired.

## Best Possible Solution

Design a `memory-core` lifecycle model that adds explicit provenance and timestamps for managed promotions, exposes review/audit visibility, and prunes only managed promotion artifacts under a documented, tested policy that preserves user-authored content.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for lifecycle semantics. Source and docs show the current promotion and size-compaction behavior, but no current-main failing contract exists.

## Solution Assessment

Unclear pending maintainer design: the requested direction is reasonable, but the exact retirement policy, CLI surface, config threshold, and managed-vs-user content boundary need product ownership before a narrow implementation.
