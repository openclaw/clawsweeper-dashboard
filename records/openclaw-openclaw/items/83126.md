## Summary

Keep open: current main still has promotion gating, live rehydration, and size-budget compaction for Dreaming promotions, but not the requested rule lifecycle, last-affirmed audit surface, confidence decay, or policy-based stale-rule retirement. This is product/design work around durable memory state, not a conservative cleanup close.

## What This Changes

Requests lifecycle semantics for MEMORY.md Dreaming promotions, including provenance, timestamps, audit visibility, confidence decay, and stale-rule retirement.

## Best Possible Solution

Design a memory-core lifecycle model for managed promotions that records provenance and timestamps, exposes audit/review visibility, and prunes only documented managed artifacts under a tested retention policy that preserves user-authored memory.
