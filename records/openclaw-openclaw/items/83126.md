## Summary

Keep open: current main has Dreaming promotion gating, live rehydration, and size-budget compaction for auto-promoted MEMORY.md sections, but it does not implement lifecycle metadata, last-affirmed audit visibility, confidence decay, or stale-rule retirement. This is a feature/product design item rather than a conservative cleanup-close candidate.

## What This Changes

Request a managed lifecycle for Dreaming-promoted MEMORY.md rules, including write-time metadata, confidence decay, audit visibility, and stale-rule eviction.

## Best Possible Solution

Design a memory-core lifecycle model that adds provenance and timestamps for managed promotions, exposes audit/review visibility, and prunes only managed promotion artifacts under a documented policy that preserves user-authored memory.

Not applicable as a bug reproduction: this is a feature request for new lifecycle semantics. Source and docs show the current promotion and size-compaction behavior, but no existing contract promises stale-rule retirement.
