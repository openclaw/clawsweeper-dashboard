## Summary

Keep open: current `main` and the latest shipped release have promotion gating, live rehydration, and size-budget compaction for auto-promoted memory, but they do not implement the requested rule lifecycle, last-affirmed audit, confidence decay, or temporal retirement model. This is a `memory-core` feature/product decision, not a cleanup-close candidate.

## What This Changes

Add lifecycle management for `MEMORY.md` rules so auto-promoted memory can be timestamped, audited, confidence-decayed, and retired safely.

## Best Possible Solution

Design a `memory-core` lifecycle model that records provenance and timestamps for managed promotions, exposes an audit/review command, and prunes only managed promotion artifacts under a documented policy that preserves user-authored memory.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for lifecycle semantics. Source and docs show current promotion, rehydration, and size-compaction behavior, but no current-main failing contract exists.

## Solution Assessment

Unclear pending maintainer design: the requested direction is reasonable, but the retirement policy, audit CLI, config thresholds, and managed-versus-user content boundary need product ownership before implementation.
