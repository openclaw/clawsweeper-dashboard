## Summary

Keep open: current main and v2026.5.12 have promotion gating, live rehydration, recency scoring before promotion, and size-budget compaction, but no rule lifecycle, last-affirmed audit, or confidence-based retirement for `MEMORY.md`. This remains a feature/product decision rather than a conservative cleanup close.

## What This Changes

The issue requests lifecycle management for `MEMORY.md` rules, including timestamps, confidence decay, audit visibility, and retirement of stale promoted content.

## Best Possible Solution

Design a `memory-core` lifecycle model that records provenance and timestamps for managed promotions, exposes audit/review visibility, and retires only clearly managed promotion artifacts under a documented policy that preserves user-authored memory.

Not applicable as a bug reproduction: this is a feature request for lifecycle semantics. Source and docs show the current promotion, temporal-decay, and size-compaction behavior, but no current-main failing contract exists.

## Solution Assessment
