## Summary

Keep open. Current main and the latest release still match the reported Feishu behavior, and related cleanup has made this the canonical tracker for the Feishu plain-text versus streaming-card default decision rather than something already implemented or superseded.

## What This Changes

The issue asks OpenClaw to restore Feishu auto render mode so ordinary final plain-text replies use native text bubbles while card-worthy replies can still stream as cards.

## Best Possible Solution

Make an explicit Feishu product decision: either restore the orthogonal `renderMode` versus `streaming` model with tests/docs, or document the current streaming-card default and track card UX/search fallback separately.

## Reproduction Assessment

Yes at source level: default Feishu config keeps streaming enabled and current dispatcher logic makes any final plain text use a streaming card. I did not establish a live Feishu search reproduction, and the discussion has mixed searchability evidence.

## Solution Assessment

Unclear pending product decision. Dropping the final-text forcing branch is a narrow code shape, but maintainers need to choose whether default streaming consistency or native plain-text delivery should win for Feishu auto mode.
