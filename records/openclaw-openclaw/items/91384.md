## Summary

Keep open. Current `main` and the latest release still match the reported Feishu behavior, and a member comment frames the desired middle ground as an unsettled product/UX default decision rather than cleanup that can be auto-closed.

## What This Changes

The issue asks to restore Feishu `renderMode:auto` so ordinary final plain-text replies stay native text bubbles while card-worthy replies can still stream when `streaming:true`.

## Best Possible Solution

Keep this issue open as the canonical decision point for whether Feishu `renderMode:auto` should restore the stricter text-vs-card distinction or keep the current streaming-card default with card presentation improvements.

## Reproduction Assessment

Yes, source-reproducible: the dispatcher predicate and current regression test show that default streaming `auto` mode sends ordinary final plain text through the streaming-card path. I did not run a live Feishu tenant reproduction, and the search-indexing claim still needs tenant-specific proof.

## Solution Assessment

Unclear pending product direction: dropping the final-text forcing branch is a narrow maintainable fix if maintainers want orthogonal render-mode behavior. It is not yet clearly the best product choice because a member comment says streaming priority versus native text bubbles is the unresolved default tradeoff.
