## Summary

Keep open. Current main and the latest release still match the reported Feishu behavior: default `renderMode:auto` with streaming enabled forces ordinary final plain text through a streaming card, while the schema and sibling outbound path still describe auto mode as markdown/card detection.

## What This Changes

The issue asks to restore Feishu auto-mode plain final replies to plain text bubbles by default while leaving streaming for replies that are already selected for card rendering.

## Best Possible Solution

Restore the orthogonal Feishu model if maintainers agree: `renderMode` chooses text versus card, `streaming` only controls incremental delivery for selected cards, and docs/tests make that contract explicit.

## Reproduction Assessment

Yes from source inspection: default Feishu config makes `streamingEnabled` true and current dispatcher logic makes any final plain text use a streaming card. I did not run a live Feishu search reproduction in this read-only review.

## Solution Assessment

Likely yes for the code shape, pending maintainer approval: dropping the final-text forcing branch would restore the older renderMode contract without adding another config knob. The decision is product-sensitive because the opposite behavior was recently merged and shipped.
