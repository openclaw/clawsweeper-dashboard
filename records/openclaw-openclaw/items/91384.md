## Summary

Keep open. Current `main` and the latest release still route Feishu `streaming:true` plus `renderMode:auto` ordinary final text through streaming cards, and the member comment frames the requested middle ground as an unresolved Feishu product/UX default decision rather than cleanup that can be auto-closed.

## What This Changes

The issue asks to restore Feishu `renderMode: "auto"` so ordinary plain final replies use native text bubbles while card-worthy replies can still stream as cards.

## Best Possible Solution

Keep this issue as the canonical Feishu default decision, then align runtime, tests, schema comments, and docs around either the current streaming-card default or the stricter `renderMode:auto` text-vs-card model.

## Reproduction Assessment

Yes from source inspection: the current dispatcher selects a streaming card for final text when Feishu streaming is enabled, and the adjacent test asserts that plain final text streams as a card. I did not run a live Feishu tenant repro, and the search-indexing claim remains environment-dependent.

## Solution Assessment

Unclear pending product direction: dropping the final-text forcing branch is a narrow maintainable fix if maintainers want orthogonal render-mode behavior. It is not yet clearly the best product choice because a member comment says streaming priority versus native text bubbles is the unresolved default tradeoff.
