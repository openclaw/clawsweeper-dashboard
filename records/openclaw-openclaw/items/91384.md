## Summary

Keep open. Current `main` and `v2026.6.9` still route Feishu `streaming:true` plus default `renderMode:auto` ordinary final text through streaming cards, and the member discussion frames the requested native text-bubble/card split as an unresolved Feishu product default decision rather than cleanup that should be closed.

## What This Changes

The issue asks to restore Feishu `renderMode:auto` so ordinary plain final replies stay native text bubbles while card-worthy replies can still stream as cards.

## Best Possible Solution

Keep this issue as the canonical Feishu default decision, then align runtime, tests, schema comments, and docs around either the current streaming-card default or the stricter `renderMode:auto` text-vs-card model.

## Reproduction Assessment

Yes from source inspection: current main defaults Feishu streaming on and selects a streaming card for ordinary final text, and the adjacent test asserts that path. I did not run a live Feishu tenant repro, and the search-indexing part remains environment-dependent.

## Solution Assessment

Unclear pending product direction. Dropping the final-text forcing branch is a narrow maintainable fix if maintainers want orthogonal `renderMode` and `streaming`, but it would reverse shipped default behavior and should follow an explicit Feishu product decision.
