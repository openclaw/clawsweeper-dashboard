## Summary

Keep open. Current `main` and release `v2026.6.9` still route default Feishu `streaming:true` plus `renderMode:auto` plain final replies through streaming cards, while the member discussion frames the requested middle ground as an unresolved Feishu product/UX default decision rather than cleanup that can be auto-closed.

## What This Changes

The issue asks OpenClaw to restore Feishu `renderMode:auto` so ordinary plain final replies stay native text bubbles while card-worthy replies can still use streaming cards.

## Best Possible Solution

Keep this as the canonical Feishu default decision, then align `reply-dispatcher.ts`, dispatcher tests, schema comments, and Feishu docs around either the current streaming-card default or the stricter `renderMode:auto` text-vs-card model.

## Reproduction Assessment

Yes, at source level. Current main routes default auto-mode plain final text to `FeishuStreamingSession` when streaming is enabled, and the adjacent test asserts that behavior; I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

Unclear pending product direction. Dropping the final-text forcing branch is a narrow maintainable fix if maintainers want orthogonal render-mode behavior, but the member comment says streaming priority versus native text bubbles is the unresolved default tradeoff.
