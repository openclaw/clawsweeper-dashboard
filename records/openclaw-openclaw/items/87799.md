## Summary

Keep open: the patch is a focused consumer-level installer stdin fix, but it still needs stronger real behavior proof before merge because the supplied non-dry-run install command is not actually fed through stdin.

## What This Changes

The PR adds conditional stdin isolation to `scripts/install.sh` subprocess paths and installer-shell tests for piped, direct, gum, and quiet-step behavior.

## Best Possible Solution

Land a consumer-level stdin isolation fix only after the contributor adds exact redacted non-dry-run piped installer proof and maintainers accept the installer compatibility tradeoff.

## Reproduction Assessment

Yes, at source level: current docs and installer comments publish `curl ... install.sh | bash`, current main lets key child processes inherit stdin, and gum v0.17.0 forwards `os.Stdin` to wrapped commands. I did not run a live current-main installer reproduction in this read-only review.

## Solution Assessment

Yes, mostly: consumer-level stdin isolation is the best direction compared with the closed global pipe guard and docs-only workaround. The remaining blocker is proof and maintainer acceptance, not a clearer code location.
