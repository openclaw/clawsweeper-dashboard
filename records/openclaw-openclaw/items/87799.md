## Summary

Keep open: this is still a useful installer reliability fix, but the current head remains incorrect because the normal gum spinner path can still pass the piped installer stream to child commands, and the branch also softens unrelated size-label automation failures.

## What This Changes

The branch redirects selected installer subprocess and fallback stdin streams, updates one installer test assertion, and changes size-label workflow add/remove failures to best-effort handling.

## Best Possible Solution

Keep the installer fix focused: protect the child command executed under gum, preserve terminal stdin for interactive shell-function installers, split or tightly justify the size-label workflow change, and add full redacted piped-install proof.

## Reproduction Assessment

Yes, source-reproducible: `curl | bash` feeds the installer through stdin, `run_npm_global_install` reaches `run_with_spinner` when gum is available, and gum v0.17.0 passes `os.Stdin` to the wrapped child command. I did not run a live full installer in this read-only review.

## Solution Assessment

No. The fallback redirects are useful, but the best fix must also isolate the normal gum-executed child command, preserve TTY stdin for interactive shell-function installers such as Homebrew, and avoid mixing unrelated labeler workflow semantics into the installer patch.
