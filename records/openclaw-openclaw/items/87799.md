## Summary

Keep open: this PR is the active candidate fix for the documented piped-installer stdin corruption, and current main plus the latest release still have the unisolated installer child-command paths.

## What This Changes

This PR adds `needs_stdin_isolation` to `scripts/install.sh`, redirects piped or noninteractive installer subprocess stdin away from the script stream while preserving direct interactive stdin, and adds focused installer regression tests.

## Best Possible Solution

Land the focused stdin-consumer hardening after maintainer acceptance of the installer stdin contract and proof coverage, then close https://github.com/openclaw/openclaw/issues/90008 as implemented.

## Reproduction Assessment

Yes, source-reproducible: docs publish `curl | bash`, current main lets key installer subprocesses inherit stdin, and gum v0.17.0 forwards `os.Stdin` to wrapped commands. I did not run a live WSL2 installer reproduction in this read-only review.

## Solution Assessment

Yes, this is the best observed fix direction: it follows prior maintainer guidance from the closed broad pipe-guard PR by hardening the actual stdin consumers instead of adding a top-level guard. The remaining question is maintainer acceptance of the compatibility and proof coverage.
