## Summary

Keep open: the latest branch fixes the earlier missed fallback, but the public installer fix is still incomplete because normal `gum spin` executions can inherit the piped script stream, and the generic wrapper redirect can break interactive Homebrew installs. The unrelated labeler workflow change also needs explicit maintainer review or a split PR.

## What This Changes

The PR redirects several installer subprocess stdin streams to `/dev/null` or `/dev/tty` and makes size-label workflow add/remove calls best-effort.

## Best Possible Solution

Keep the targeted-consumer approach, but make stdin routing command-specific: `/dev/null` for known noninteractive children, `/dev/tty` for prompts or interactive installers, redirect gum itself, split or explicitly approve the labeler change, and prove a full piped install path.

## Reproduction Assessment

Yes, as a source-reproducible path rather than a live WSL2 run: current docs use `curl | bash`, and gum v0.17.0 passes `os.Stdin` to spinner-wrapped children. I did not run a full installer reproduction in this read-only review.

## Solution Assessment

No. Targeting stdin consumers is the right direction, but this patch misses the normal gum path and over-applies `/dev/null` to interactive Homebrew setup; the maintainable fix is command-specific stdin routing plus full-flow proof.
