## Summary

## What This Changes

The PR adds `needs_stdin_isolation` and conditional `/dev/null` stdin redirection to `scripts/install.sh` spinner and quiet-step child commands, plus installer tests for piped, `NO_PROMPT`, gum, and direct interactive stdin behavior.

## Best Possible Solution

Land this focused stdin-consumer hardening after maintainer acceptance of the installer compatibility tradeoff, then close https://github.com/openclaw/openclaw/issues/90008 as implemented.

## Reproduction Assessment

Yes, source-reproducible: current docs support `curl | bash`, current main and `v2026.6.8` let key installer child commands inherit stdin, and gum v0.17.0 forwards `os.Stdin` to wrapped commands. I did not run a live WSL2 installer reproduction in this read-only review.

## Solution Assessment

Yes, this is the best observed fix direction: it hardens the actual stdin consumers instead of reintroducing the broad top-level pipe guard that was closed as too risky. The remaining question is maintainer acceptance of the compatibility and platform-proof gap.
