## Summary

Keep this PR open: it addresses a real installer stdin-corruption class, but the latest patch still unconditionally removes stdin from the live gum path and can break direct interactive installs that the PR is trying to preserve.

## What This Changes

The PR adds installer stdin-isolation handling in `scripts/install.sh` and regression tests for piped/fallback stdin behavior in `test/scripts/install-sh.test.ts`.

## Best Possible Solution

Keep the fallback stdin isolation, make gum-path stdin isolation conditional or command-specific so direct runs preserve prompts, then add redacted live proof for both piped install safety and direct prompt preservation.

## Reproduction Assessment

Yes, source-reproducible: current main has subprocess paths that can inherit piped installer stdin, and the PR defect follows from gum v0.17 passing `os.Stdin` to wrapped commands plus Homebrew treating `/dev/null` stdin as noninteractive. I did not run a full live installer reproduction in this read-only review.

## Solution Assessment

No. The fallback `needs_stdin_isolation` direction is narrow, but unconditionally redirecting the normal gum path is too broad because that path is primarily enabled for direct interactive installs.

- **[P1] Preserve terminal stdin for direct gum runs:** `scripts/install.sh:461`
