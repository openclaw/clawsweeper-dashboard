## Summary

Keep open: this is a focused installer bug fix and current main still has stdin-inheriting child-command paths, but the PR is not merge-ready because the new direct-gum test does not assert the observed stdin result and the supplied proof is still grep/Vitest-only rather than a real installer run.

## What This Changes

The PR conditionally isolates installer child-process stdin in piped or noninteractive contexts, preserves terminal stdin for direct interactive paths, adds direct redirects for several installer subprocesses, and expands install-sh tests.

## Best Possible Solution

Land a narrow installer stdin-isolation fix after the direct-gum test asserts the observed stdin result and redacted real behavior proof covers both piped install isolation and direct interactive prompt preservation.

## Reproduction Assessment

Yes. The bug is source-reproducible from current main because child commands inherit installer stdin in the wrapper paths, and gum v0.17.0 passes os.Stdin to wrapped commands; I did not run a live curl-piped installer in this read-only review.

## Solution Assessment

Mostly yes, but not merge-ready. Conditional isolation at the installer wrapper boundary is the maintainable fix shape, while the direct-gum assertion and real behavior proof still need to catch up before this is the best mergeable version.
