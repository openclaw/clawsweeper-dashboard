## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep Homebrew on interactive stdin:** `scripts/install.sh:513`
  - body: This wrapper path also runs `install_homebrew` through the shell-function fallback, because `run_remote_bash` is a shell function. Redirecting it from `/dev/null` makes Homebrew's installer see non-TTY stdin and enter noninteractive sudo/prompt handling, so a normal direct macOS install without Homebrew can now fail or change behavior. Preserve `/dev/tty` for interactive installer scripts and restrict `/dev/null` to known noninteractive subprocesses.
  - confidence: 0.9
- **[P2] Protect the remaining raw-mode rerun:** `scripts/install.sh:459`
  - body: The adjacent raw-mode branch above still reruns `"$@"` with inherited stdin when gum exits successfully but writes a raw-mode failure to stdout/stderr. That path can still let npm/git/openclaw consume bytes from a `curl | bash` script stream, so both raw-mode fallback reruns need the same stdin policy.
  - confidence: 0.82
- **[P2] Split the labeler workflow change:** `.github/workflows/labeler.yml:145-153`
  - body: This installer fix also changes the pull_request_target labeler and backfill paths so size-label remove/add errors become best-effort warnings. That can make the workflow pass while labels were not updated, and the provided installer proof does not exercise GitHub automation; move this to a separate proven workflow PR or remove it here.
  - confidence: 0.78
