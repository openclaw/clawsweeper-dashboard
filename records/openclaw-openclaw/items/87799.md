## What This Changes

The PR adds conditional stdin isolation to installer spinner/quiet subprocess paths, redirects selected noninteractive installer commands away from the piped script stream, preserves onboarding TTY stdin, and adds install.sh stdin regression tests.

## Best Possible Solution

Land the targeted stdin isolation only after redacted real or equivalent terminal proof shows both curl-piped installer isolation and direct interactive prompt preservation on the PR head.

## Reproduction Assessment

Yes, as a source-level reproduction: current main passes inherited stdin into gum/quiet child commands, and gum v0.17.0 forwards os.Stdin to wrapped commands. I did not run a live installer because this review was read-only.

## Solution Assessment

Likely yes: isolating only child stdin in piped/nonprompt contexts while preserving terminal stdin for promptable paths is narrower than re-executing the installer or disabling gum globally. The remaining blocker is proof, not a better code location found in review.
