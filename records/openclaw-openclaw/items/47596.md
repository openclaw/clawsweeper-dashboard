## Summary

Keep open. Current main still routes ACP `streamTo="parent"` through a relay that surfaces assistant/lifecycle summaries only, while the requested tool/result/thinking visibility would expose child execution details into parent-session system events and needs maintainer visibility/redaction policy.

## What This Changes

Request to make ACP `streamTo="parent"` relay child tool calls, tool results, and thinking/progress details to the parent session.

## Best Possible Solution

Add concise, redacted ACP parent-relay summaries for structured progress streams, gated by a maintainer-chosen visibility policy and covered by focused relay tests/docs.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: start the ACP parent relay for a child run, emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id, and the handler returns before enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely implementation path is `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding would be riskier than bounded summaries aligned with existing verbose/reasoning behavior.
