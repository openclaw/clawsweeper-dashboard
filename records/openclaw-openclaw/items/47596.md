## Summary

Keep open. Current main still relays ACP `streamTo="parent"` as assistant/ACP/lifecycle summaries only; structured tool, command-output, approval, patch, and thinking streams already exist, but exposing them to parent-session system events needs maintainer visibility, structure, verbosity, and redaction policy first.

## What This Changes

Request ACP `streamTo="parent"` to surface child tool calls, tool results, approvals, and thinking/progress details to the parent session.

## Best Possible Solution

Define a structured, redacted ACP parent-stream contract, then implement bounded relay summaries/audit events with focused tests and docs.

## Reproduction Assessment

Yes for the current source behavior, though this is a feature request rather than a broken documented contract. Start the ACP parent relay for a child run, emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id, and the current handler returns before enqueueing a parent update.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely code path is `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding is not the best solution; bounded structured events with redaction and correlation are safer.
