## Summary

Keep open. Current main still wires ACP `streamTo="parent"` through the parent relay, but the relay only surfaces assistant text and lifecycle state while dropping structured tool, command output, approval, patch, item, plan, and thinking streams; the remaining blocker is maintainer policy for verbosity, defaults, and redaction.

## What This Changes

Request ACP `streamTo="parent"` to expose child tool calls, tool results, and thinking/progress updates in the parent session.

## Best Possible Solution

Add bounded, redacted ACP parent-relay summaries for structured progress streams, gated by the chosen verbose/reasoning policy and covered by focused relay tests plus docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the ACP parent relay, emit a matching child-run `tool`, `command_output`, or `thinking` event, and the relay returns at the non-lifecycle guard without enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility policy. The implementation likely belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw full forwarding would be less maintainable than redacted summaries gated by explicit verbose/reasoning behavior.
