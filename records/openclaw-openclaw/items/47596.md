## Summary

Keep open. Current main still routes ACP `streamTo="parent"` through a relay that surfaces assistant/lifecycle summaries only, and expanding it to tool/results/thinking needs a maintainer visibility and redaction decision because those updates become parent-session system events.

## What This Changes

The issue requests tool call, tool result, and thinking visibility in ACP `sessions_spawn` parent streaming.

## Best Possible Solution

Add concise, redacted ACP parent-relay summaries for structured progress streams, gated by a maintainer-chosen visibility policy and covered by focused relay docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the ACP parent relay for a child run, emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id, and the handler returns before enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely code path is `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding would be riskier than bounded summaries aligned with existing verbose/reasoning behavior.
