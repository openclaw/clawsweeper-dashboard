## Summary

Keep open: current main partially improves ACP parent-stream visibility with assistant, preamble, and selected status progress, but it still does not expose the requested tool-call arguments, tool results, or thinking details to the parent session; the remaining work is a product/security decision, not conservative cleanup.

## What This Changes

Request to expose ACP child tool calls, tool arguments, tool results, and thinking/progress details to the parent session when `sessions_spawn` uses `runtime: "acp"` and `streamTo: "parent"`.

## Best Possible Solution

Keep this as the canonical ACP parent-stream structured visibility tracker and implement a bounded, redacted, opt-in projection policy that reuses existing ACP stream/tag-visibility contracts where possible.

## Reproduction Assessment

Yes, at source level: current main relays selected assistant, preamble, and status progress, but it does not expose parent-visible raw tool-call arguments, tool results, or thinking details. I did not run a live ACP child session in this read-only review.

## Solution Assessment

No complete implementation exists yet; raw event forwarding would not be the best solution because it would bypass the existing hidden-tag and diagnostic sanitization policy. The safer path is a maintainer-approved redaction/truncation/defaults policy, likely built around `acp.stream.tagVisibility` or a deliberate successor setting.
