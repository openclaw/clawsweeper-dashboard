## Summary

Keep open. Current main and v2026.5.20 still do not relay ACP child tool, command output, approval, patch, or thinking streams to the parent session; the remaining work is valid but needs explicit visibility and redaction policy before implementation.

## What This Changes

The request asks for ACP `sessions_spawn` `streamTo="parent"` to surface child tool calls, tool results, and thinking progress in the parent session.

## Best Possible Solution

Define a redacted, structured ACP parent-progress/audit event contract with correlation ids, verbosity rules, focused tests, and concise parent-session rendering.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request, but the current behavior is source-reproducible. Emit `tool`, `command_output`, `approval`, `patch`, or `thinking` for the child run id while ACP `streamTo="parent"` is active; the current relay returns before parent-visible enqueue for those streams.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. A raw string relay would be weaker than structured, redacted summaries aligned with the parent-session trust boundary.
