## Summary

Keep open. Current main and latest release still support ACP `streamTo="parent"` only as assistant/lifecycle progress summaries; structured tool, command-output, patch/approval/item, and thinking streams are available upstream but are still dropped by the parent relay, and exposing them needs a maintainer visibility/redaction decision.

## What This Changes

The issue requests visible tool-call, tool-result, and thinking progress for ACP child runs streamed to the parent session.

## Best Possible Solution

Add concise, redacted ACP parent-relay summaries for structured progress streams, gated by an explicit visibility policy and covered by focused relay docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the ACP parent relay for a child run, emit a `tool`, `command_output`, or `thinking` event for the same run id, and the handler returns at the non-lifecycle guard without enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely implementation belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding would be riskier than bounded summaries aligned with existing verbose/reasoning behavior.
