## Summary

Keep open. Current main still wires ACP `streamTo="parent"` through a parent relay that surfaces assistant/lifecycle summaries only, while tool, command-output, approval, patch, and thinking visibility needs an explicit maintainer policy because it can expose execution details into parent-session system events.

## What This Changes

Request ACP parent stream relay visibility for child tool calls, tool results, command output, and thinking events when `streamTo="parent"` is enabled.

## Best Possible Solution

Add concise, redacted ACP parent-relay summaries for structured progress streams, gated by the chosen visibility policy and covered by focused relay docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the ACP parent relay for a child run, emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id, and the handler returns before enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely code path is `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding would be riskier than bounded summaries aligned with existing verbose/reasoning behavior.
