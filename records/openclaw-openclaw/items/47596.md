## Summary

Keep open: current `main` and `v2026.6.8` still do not expose ACP child tool calls, raw arguments, tool results, or thinking to the parent stream. Recent merged work covers assistant commentary and opted-in ACP status summaries, but the central request remains a security-sensitive product/API decision about what child-run internals may cross the parent-session boundary.

## What This Changes

The issue requests parent-visible ACP child tool calls, arguments, results, and thinking/progress when `sessions_spawn` uses `runtime: "acp"` with `streamTo: "parent"`.

## Best Possible Solution

Define a default-off, redacted ACP parent-stream structured projection for tool/status/thinking visibility, reuse existing stream/tag visibility controls where possible, and cover the contract with relay/config/docs tests before implementation.

## Reproduction Assessment

Yes, at source level. Current `main` and `v2026.6.8` only relay assistant text/commentary and opted-in ACP `status` summaries, while tool-call/thought tags remain hidden or unavailable as raw parent-visible payloads.

## Solution Assessment

No, the direct raw `tool_call` / `tool_result` / `thinking` relay sketched in the issue is not the best fix yet. The safer path is a maintainer-approved redaction, truncation, default, and control-surface policy before exposing child-run internals.
