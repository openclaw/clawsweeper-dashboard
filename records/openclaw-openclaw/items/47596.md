## Summary

Keep open. Current main still relays ACP `streamTo="parent"` as assistant/ACP/lifecycle summaries only, while tool/result/approval/patch/thinking events already exist and exposing them into parent-session system events needs a maintainer visibility and redaction policy.

## What This Changes

Expose ACP child tool calls, tool results, and thinking progress through `sessions_spawn` `streamTo="parent"` parent-session updates.

## Best Possible Solution

Add concise, redacted ACP parent-relay summaries for structured progress streams, gated by maintainer-approved visibility policy and covered by focused relay tests/docs.

## Reproduction Assessment

Yes for the current source behavior, though this is a feature request rather than a broken documented contract. Start the ACP parent relay for a child run, emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id, and the current handler returns before enqueueing a parent update.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely implementation belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding of child tool data would be riskier than bounded summaries aligned with existing verbose/reasoning behavior.
