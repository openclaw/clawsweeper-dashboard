## Summary

Keep open: current main has ACP parent-stream assistant text, default-off commentary/status progress, and lifecycle notices, but it still does not expose parent-visible tool-call arguments/results or thought details. The remaining work is a feature/security policy decision because raw child-session details can cross parent session or channel boundaries.

## What This Changes

Request to expose ACP child tool calls, tool arguments/results, and thinking/progress details to the parent session when `sessions_spawn` uses `runtime: "acp"` with `streamTo: "parent"`.

## Best Possible Solution

Keep this as the canonical ACP parent-stream structured-visibility tracker and implement only after maintainers choose a bounded, opt-in, redacted projection contract that aligns with `acp.stream.tagVisibility`.
