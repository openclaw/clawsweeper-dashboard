## Summary

Keep open: current main and the latest release already provide partial ACP parent-stream progress, including assistant/commentary/status/lifecycle paths, but they still do not expose parent-visible tool-call arguments, tool results, or thinking details. The remaining work is a product/security decision because those details can leak sensitive child-runtime data across parent session or channel boundaries.

## What This Changes

The issue asks for ACP `streamTo="parent"` relays to expose child tool calls, tool arguments, tool results, and thinking/progress details to the parent session.

## Best Possible Solution

Keep this as the canonical ACP parent-stream structured-visibility tracker and settle the disclosure policy first, then implement a bounded, opt-in projection for tool/thinking summaries using the existing ACP tag-visibility path where possible.

## Reproduction Assessment

Yes. At source level, current main relays selected assistant, item, status, and lifecycle progress but does not expose parent-visible raw tool-call arguments, raw tool results, or thinking details; no live ACP child session was run because this is a read-only review.

## Solution Assessment

No complete implementation is selected yet. Direct raw event forwarding would not be the best fix; maintainers need to choose redaction, truncation, defaults, and visibility controls first.
