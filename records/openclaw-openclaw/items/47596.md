## Summary

Keep open. Current main and the latest release still route ACP `streamTo="parent"` through the parent relay, but that relay only surfaces assistant text and lifecycle messages; structured tool, command output, plan, approval, patch, item, and thinking streams are still dropped, and the remaining decision is the visibility/redaction policy.

## What This Changes

The issue requests ACP `streamTo="parent"` progress visibility for child tool calls, tool results, and thinking output in the parent session.

## Best Possible Solution

Add bounded, redacted ACP parent-relay summaries for structured progress streams, gated by an explicit verbose/reasoning policy and covered by focused relay docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the ACP parent relay for a child run, emit a matching `tool`, `command_output`, or `thinking` event, and the handler returns at the non-lifecycle guard without enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The implementation likely belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding would be riskier than concise redacted summaries aligned with existing verbose/reasoning behavior.
