## Summary

Keep open: current main and the latest release still only parent-surface assistant/commentary text and tag-visible ACP status progress, while raw ACP child tool calls, tool arguments, tool results, and thinking remain unimplemented and security-sensitive.

## What This Changes

Request ACP `streamTo="parent"` to show child tool calls, arguments/results, and thinking/progress details in the parent session.

## Best Possible Solution

Keep this as the canonical ACP parent-stream visibility tracker and define a default-off, redacted, bounded projection policy before implementing tool/result/thinking visibility.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.10` only relay assistant/commentary and allowed ACP status progress to the parent, while tool/thought tags remain hidden and raw args/results are not in the parent projection contract. I did not run a live ACP session in this read-only review.

## Solution Assessment

No, the raw forwarding shape suggested in the issue is not yet the best fix. The maintainable path is a maintainer-approved visibility, redaction, truncation, and configuration contract that reuses existing ACP/channel projection settings where possible.
