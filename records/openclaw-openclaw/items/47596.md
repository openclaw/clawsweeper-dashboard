## Summary

Keep open. Current main still wires ACP `streamTo="parent"` through the parent relay, but the relay only surfaces assistant text and lifecycle state; structured tool, command output, approval, patch, item, plan, and thinking streams are still ignored. The remaining blocker is maintainer policy for verbosity, defaults, and redaction.

## What This Changes

Request to stream ACP child tool calls, tool results, and thinking/progress details back to the requester session when `streamTo="parent"` is enabled.

## Best Possible Solution

Add bounded, redacted ACP parent-relay summaries for structured progress streams, aligned with the chosen verbose/reasoning policy and documented with focused relay coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: start the ACP parent relay, emit a matching child-run `tool` or `thinking` event, and the relay returns at the non-lifecycle guard without enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility policy. The implementation likely belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw full forwarding would be less maintainable than redacted summaries gated by explicit verbose/reasoning behavior.
