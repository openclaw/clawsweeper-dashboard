## Summary

Keep open: current main and the latest release provide ACP parent-stream assistant/status progress in bounded cases, but they still do not expose parent-visible tool-call arguments, tool results, or thinking details. The remaining work is a product/security decision because raw tool and thinking data can disclose sensitive child-run internals across parent/session/channel boundaries.

## What This Changes

Request parent-visible ACP `streamTo="parent"` visibility for tool calls, tool arguments, tool results, and thinking details.

## Best Possible Solution

Implement a bounded opt-in ACP visibility policy that reuses existing `acp.stream` projection controls where possible, redacts/truncates tool and thinking data, and keeps conservative defaults.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` relay selected assistant/status/lifecycle progress but do not provide parent-visible raw tool-call arguments, tool results, or thinking details.

## Solution Assessment

No final implementation shape is approved yet. Direct raw forwarding from the issue suggestion would not be the best fix; the safer path needs maintainer-approved visibility, redaction, truncation, defaults, and tests.
