Keep open. Current main still drops structured child streams from the ACP parent relay, and the recent discussion correctly reframes the request as a structured, redacted audit/progress stream that needs maintainer product and security policy before implementation.

## What This Changes

Request ACP `sessions_spawn` with `streamTo="parent"` to expose child tool calls, tool results, approvals/progress, and thinking to the parent session.

## Best Possible Solution

Add concise, redacted, structured ACP parent-relay progress events with stable correlation IDs, explicit visibility policy, and focused relay/docs coverage.

## Reproduction Assessment

Yes for source-level behavior, though this is a feature request rather than a broken documented contract. Start the ACP parent relay for a child run, emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id, and the current handler returns before enqueueing a parent update.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely implementation belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw human-readable forwarding is less maintainable than redacted structured summaries aligned with existing relay trust semantics.
