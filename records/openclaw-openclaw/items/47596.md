## Summary

Keep open. Current main and v2026.5.22 still relay ACP child assistant text, ACP diagnostics, and lifecycle notices only; tool, command-output, approval, patch, and thinking streams remain available upstream but are not surfaced to the parent session, and the fix needs maintainer policy for verbosity, truncation, and redaction.

## What This Changes

Add parent-session visibility for ACP child tool calls, tool results, command output, approvals, patches, and thinking when `streamTo="parent"` is enabled.

## Best Possible Solution

Define a redacted, structured ACP parent-progress event contract with correlation ids, verbosity rules, focused relay tests, and concise parent-session rendering.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request, but the current behavior is source-reproducible. Emit `tool`, `command_output`, `approval`, `patch`, or `thinking` for the child run id while ACP `streamTo="parent"` is active; the current relay returns before parent-visible enqueue for those streams.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. A raw string relay would be weaker than structured, redacted summaries aligned with the parent-session trust boundary.
