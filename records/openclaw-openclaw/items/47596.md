## Summary

Keep open. Current main still relays ACP `streamTo="parent"` as parent-visible summary events rather than structured child tool/progress/audit events, and the recent discussion confirms this needs maintainer product and security policy before implementation.

## What This Changes

Request ACP `streamTo="parent"` to expose child tool calls, tool results, thinking, and related structured progress to the parent session with an appropriate verbosity and redaction contract.

## Best Possible Solution

Define a maintainer-approved structured parent progress/audit contract with redaction, size limits, correlation ids, authority scope, and concise UI rendering, then implement it in the ACP parent relay with focused tests and docs.

## Reproduction Assessment

Yes for the current source behavior, though this is a feature request rather than a broken documented contract. Start the ACP parent relay for a child run and emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id; the current handler does not enqueue a parent update for those stream types.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely implementation belongs around `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding of child tool details would be less maintainable than a bounded structured event contract with redacted summaries.
