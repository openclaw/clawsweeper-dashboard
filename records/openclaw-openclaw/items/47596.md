## Summary

Keep open: current main has ACP parent-stream progress summaries plus selected commentary/status relay, but it still does not provide parent-visible tool-call arguments, tool results, or thinking details. The remaining work is a product/security decision because those internals can expose sensitive runtime data across session or channel boundaries.

## What This Changes

The issue requests tool-call, tool-result, and thinking visibility in ACP `sessions_spawn` parent streams when `streamTo="parent"` is enabled.

## Best Possible Solution

Keep this as the canonical ACP parent-stream structured-visibility tracker and implement a bounded, opt-in, redacted projection that reuses existing ACP stream/tag-visibility contracts where possible.

## Reproduction Assessment

Yes. At source level, current main relays selected assistant/item/status/lifecycle progress but does not expose parent-visible raw tool-call arguments, raw tool results, or thinking details; no live ACP child session was run because this is a read-only review.

## Solution Assessment

No complete implementation is selected yet. Direct raw event forwarding would not be the best fix; maintainers need to choose the redaction, truncation, defaults, and visibility controls first.
