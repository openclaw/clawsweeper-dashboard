## Summary

Keep open: current main and the latest release still do not provide parent-visible raw ACP tool arguments, tool results, or reasoning/thought visibility for `streamTo="parent"`; the remaining work is a security-sensitive product/API decision, not stale cleanup or an already-implemented fix.

## What This Changes

The issue requests parent-visible ACP stream progress for child tool calls, tool results, and thinking/status details when `sessions_spawn` uses `runtime: "acp"` with `streamTo: "parent"`.

## Best Possible Solution

Keep this as the canonical ACP parent-stream structured-visibility tracker and implement only after maintainers choose a bounded, opt-in, redacted projection contract that reuses the ACP stream visibility model where possible.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.8` relay assistant/commentary/status progress but filter non-status ACP runtime events and keep tool/thought tags hidden by default. I did not run a live ACP child session in this read-only review.

## Solution Assessment

No approved implementation path yet; direct raw forwarding from the issue suggestion is not the best fix without a maintainer-chosen visibility, redaction, truncation, and default policy. The safer direction is a bounded projection contract, likely around existing ACP stream visibility settings.
