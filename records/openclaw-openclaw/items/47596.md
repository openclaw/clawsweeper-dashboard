## Summary

Keep open. Current main and latest release still relay ACP `streamTo="parent"` as assistant/lifecycle progress only, while the requested tool, command-output, approval, patch, and thinking visibility remains a product/security policy decision.

## What This Changes

Request ACP `streamTo="parent"` to expose child tool-call, tool-result, command-output, and thinking progress to the parent session.

## Best Possible Solution

Add concise, redacted ACP parent-relay summaries for structured progress streams, gated by the chosen visibility policy and covered by focused relay docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the ACP parent relay for a child run, emit a `tool`, `command_output`, `approval`, `patch`, or `thinking` event for the same run id, and the handler returns before enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility and redaction policy. The likely implementation belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw forwarding would be riskier than bounded summaries aligned with existing verbose/reasoning behavior.
