## Summary

Keep open: current main and v2026.6.8 only provide partial continuity mechanisms, not the requested user-facing guarantee that the last N raw messages survive both compaction and reset. The remaining work adds a new session/context contract and must settle auth, channel, and reset-boundary security before implementation.

## What This Changes

The issue requests a configurable session/context guarantee that preserves at least the last N raw conversation messages across compaction and daily, idle, or manual session resets.

## Best Possible Solution

Keep this issue open as the canonical product/security design item for one raw-tail continuity contract that defines the N-message floor, reset behavior, auth/channel boundaries, and integration with CLI reseed and compaction safeguards.

## Reproduction Assessment

Yes. Source inspection shows current main lacks `session.minTailMessages` or `carryForwardOnReset`, while existing raw-tail reseed is a CLI-backend opt-in limited to selected invalidation reasons.

## Solution Assessment

Unclear as a final design. The requested guarantee is plausible, but the best fix needs maintainer decisions about whether the contract belongs in core session config, context-engine behavior, CLI backend recovery, compaction policy, or a narrower memory/reset design.
