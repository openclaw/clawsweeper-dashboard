## Summary

Keep open: current main has narrower continuity mechanisms, but it still does not provide a session-level, user-facing guarantee that the last N raw messages survive compaction and daily/idle/manual resets. The remaining work changes config, privacy, auth-boundary, and prompt-budget semantics, so it needs maintainer product and security review before implementation.

## What This Changes

The issue requests a configurable session raw-tail guarantee so recent user/assistant messages survive compaction and session resets as raw context.

## Best Possible Solution

Define an additive session-continuity contract across schema, docs, reset handling, compaction, CLI backends, Pi runtime, and context engines, with explicit clean-reset behavior, scope isolation, auth exclusions, and prompt-budget caps.
