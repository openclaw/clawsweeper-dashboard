## Summary

Keep this open: current main has adjacent startup-memory, CLI reseed, and silent-rotation replay helpers, but it still lacks the requested user-facing guarantee that a configurable raw message tail survives both compaction and normal session reset.

## What This Changes

Request a configurable session continuity contract that preserves at least the last N raw user/assistant messages across compaction and daily, idle, or manual reset.

## Best Possible Solution

Define an additive session-continuity contract with config/schema/docs/tests, budget caps, clean-reset opt-out, and explicit context-engine requirements before implementing it in reset and compaction paths.

## Reproduction Assessment

Not applicable as a broken existing behavior reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested config surface and raw-tail reset/compaction contract.

## Solution Assessment

Unclear. The requested last-N raw tail guarantee is plausible, but extending only the existing CLI reseed or silent-rotation replay helper would miss the broader reset, compaction, and plugin API semantics.
