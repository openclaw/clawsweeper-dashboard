## Summary

Keep open: current main has partial adjacent mechanisms, including opt-in CLI raw-tail reseed and Pi recent-turn compaction retention, but it still lacks the requested user-facing raw-tail guarantee across compaction and daily/idle/manual reset boundaries.

## What This Changes

The issue requests a configurable session guarantee that preserves the last N raw conversation messages across compaction and session resets.

## Best Possible Solution

Define an additive session-continuity contract across schema, docs, reset, compaction, CLI backends, and context engines, with explicit clean-reset opt-out, scope isolation, and prompt-budget caps.
