## Summary

Keep open: current main and the latest release have useful partial CLI reseed and compaction-tail behavior, but they do not implement the requested user-facing guarantee that the last N raw messages survive both compaction and session reset. The requested config and replay semantics still need maintainer product/security design.

## What This Changes

Add a configurable session-level guarantee that preserves at least the last N raw conversation messages across compaction and session resets.

## Best Possible Solution

Keep this as a maintainer-owned design issue for a single core context-tail contract shared across compaction, reset, CLI backends, embedded runs, and context engines, with explicit security boundaries and docs/config coverage.

## Reproduction Assessment

Yes for the missing requested contract: source inspection shows no `session.minTailMessages` or carry-forward setting, reset rotates session state, and raw reseed is limited to CLI backend opt-in safe invalidations. I did not run a live gateway reset scenario in this read-only review.

## Solution Assessment

Unclear as written: a raw-tail guarantee is plausible, but the best fix is not a piecemeal CLI fallback. Maintainers need to choose the cross-provider reset, compaction, context-engine, and security semantics before implementation.
