## Summary

Keep open: current main and the latest release have partial CLI raw-tail reseed and compaction-tail mechanisms, but not the requested user-facing guarantee that the last N raw messages survive both compaction and reset. The remaining work adds config and crosses session/auth/security boundaries, so it needs maintainer product and security design before implementation.

## What This Changes

The issue requests a configurable `session.minTailMessages`-style guarantee that recent raw agent messages survive compaction and session reset as context.

## Best Possible Solution

Keep this issue open as the canonical product/security design item for a single continuity contract that defines the raw-tail floor, reset semantics, auth/channel boundaries, and relationship to CLI reseed and compaction safeguards.

## Reproduction Assessment

Yes. Source inspection shows current main lacks `session.minTailMessages`/`carryForwardOnReset`, while the only raw-tail reseed is an opt-in CLI-backend mechanism for selected invalidations.

## Solution Assessment

Unclear as a final design. The requested guarantee is plausible, but the best fix needs maintainer decisions about whether the contract belongs in core session config, context-engine behavior, CLI backend recovery, or a narrower memory/reset design, and it must preserve auth/profile/channel boundaries.
