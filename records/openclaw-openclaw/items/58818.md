## Summary

Keep this open. Current main has adjacent pieces for daily-memory startup context, compaction recent-turn preservation, reset archiving, and a six-message replay helper for silent runtime rotations, but it still does not provide the requested user-facing guarantee that a configurable last-N raw transcript tail survives normal compaction plus daily, idle, and manual session resets.

## What This Changes

The issue requests a configurable session-continuity contract that preserves at least the last N raw user/assistant messages across compaction and session resets.

## Best Possible Solution

Define an additive core session-continuity contract that covers reset seeding and compaction preservation, with config/schema/help/docs, a context-budget cap, explicit clean-reset opt-out, plugin context-engine API guidance, and focused tests for daily, idle, manual reset, safeguard compaction, and context-engine-owned compaction.

## Reproduction Assessment

Yes. A source-level reproduction can create an expired or explicitly reset session with a prior JSONL transcript, then verify current main archives the old transcript and starts a new session whose first prompt can include startup memory notes but not the previous raw user/assistant tail.

## Solution Assessment

Unclear. The requested raw-tail guarantee is a plausible direction, but the current adjacent helpers are too narrow; the best fix needs maintainer design across reset semantics, prompt privacy, compaction, and plugin context-engine contracts rather than a small local patch.
