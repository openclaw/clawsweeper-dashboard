## Summary

Keep open. Current main still matches the reported installer failure mode: the npm installer can warn that the npm global bin directory is missing from the user's original PATH, resolve the CLI internally anyway, skip verification by default, and then print a successful install banner before `openclaw` is runnable from the parent shell.

## What This Changes

The issue asks for the curl bash installer to leave `openclaw` runnable after a reported successful Ubuntu install instead of ending with `openclaw: command not found`.

## Best Possible Solution

Make installer finalization self-verifying and actionable so a successful install either leaves a directly usable CLI in the user's shell path or clearly reports the exact PATH export needed before onboarding.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run the npm installer where npm's global bin directory is absent from the parent shell PATH and do not pass `--verify`; current main can warn but still finish with the success banner.

## Solution Assessment

Unclear on the exact exit semantics, but the narrow maintainable solution belongs in `scripts/install.sh`: preserve npm prefix repair while making the final CLI path or required PATH export explicit before claiming success.
