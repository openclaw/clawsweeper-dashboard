## Summary

Keep open: current main still allows the default curl installer to warn that the npm global bin directory is missing from the caller's PATH and then print a successful install banner, matching the reported failure mode closely enough to justify a focused installer fix.

## What This Changes

The issue reports that the curl-based Linux installer claims success but leaves `openclaw` unavailable as a shell command.

## Best Possible Solution

Make the installer success state match what the user can do next: either ensure the installed command is durably reachable for the user's shell, or stop before the success banner with clear action-required output and regression coverage.

## Reproduction Assessment

Yes. The high-confidence reproduction path is the default npm installer flow with an original PATH that lacks npm's global bin directory: current source warns about that state, can resolve the command only inside the installer process, and still reaches the success banner.

## Solution Assessment

Yes. A narrow `scripts/install.sh` behavior fix with focused installer tests is the maintainable path; docs-only guidance already exists and did not address the misleading success state.
