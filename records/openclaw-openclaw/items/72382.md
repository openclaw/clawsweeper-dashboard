## Summary

Keep open. Current `main` still has the reported installer UX path: verification is opt-in, the installer warns that the npm global bin is missing from the original shell PATH, and a success banner can still follow even when the parent shell will report `openclaw` as unavailable.

## What This Changes

The issue requests that the curl/bash installer leave `openclaw` installed and runnable on Ubuntu 26.04 instead of reporting success while the command is unavailable.

## Best Possible Solution

Make the installer final state self-verifying and actionable so users are not left with a plain success banner when the CLI bin exists only outside the parent shell PATH.

## Reproduction Assessment

Yes. Current source still defaults install verification off, emits the missing npm-bin PATH warning, and the non-root smoke verifies only after adding `$HOME/.npm-global/bin` to PATH, which reproduces the reported behavior class from source.

## Solution Assessment

Yes on the fix boundary. The narrow maintainable solution belongs in `scripts/install.sh` plus focused installer tests/docs, not in the core CLI or a broad install-system rewrite.
