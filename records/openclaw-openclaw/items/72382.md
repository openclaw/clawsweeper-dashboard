## Summary

Keep open. Current main still has the reported installer UX path: it warns that the npm global bin is missing from PATH, keeps post-install verification opt-in, and can still finish with a success banner before the user’s parent shell can run `openclaw`.

## What This Changes

The issue reports a curl-bash install on Ubuntu 26.04 where the installer reports success after npm/PATH warnings but `openclaw` is not available in the shell.

## Best Possible Solution

Make the installer final state self-verifying and actionable so users are not left with a plain success banner when the CLI is only available through a bin directory missing from their parent-shell PATH.

## Reproduction Assessment

Yes, for the reported behavior class. Current source still emits the same PATH warning, verification is disabled by default, and the success banner can follow even though the parent shell may still report `openclaw: command not found`.

## Solution Assessment

Unclear on exact copy/exit semantics, but the narrow maintainable fix belongs in `scripts/install.sh`: preserve npm prefix repair while making the post-install CLI path or required PATH export explicit, and fail only when the package or CLI truly cannot be resolved.
