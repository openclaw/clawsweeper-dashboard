## Summary

Keep open. Current main still matches the reported installer failure mode: the Unix curl installer can warn that the original shell PATH lacks npm's global bin directory, skip verification by default, and still print the successful install banner before the user can run `openclaw`.

## What This Changes

The issue requests that the curl/bash install flow on Ubuntu leave the `openclaw` command available after reporting a successful install.

## Best Possible Solution

Make the normal installer's success state match what the user can do next: either make `openclaw` discoverable for the immediate follow-up command, or turn the missing original PATH case into clear action-required output before the success banner. Add a focused installer test for the missing original PATH case and update docs only if the user-facing guidance changes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the npm-method installer with npm's global bin directory absent from the original shell PATH and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about the PATH gap and still continue to the success banner.

## Solution Assessment

Yes. A narrow `scripts/install.sh` behavior fix with focused tests is the best path; docs-only guidance would leave the default installer success state misleading.
