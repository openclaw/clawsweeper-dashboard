## Summary

Keep open. Current main still has the reported installer behavior: the npm install path can warn that npm's global bin directory is missing from the parent shell PATH, skip verification by default, and still print a successful install banner before `openclaw` is runnable from that shell.

## What This Changes

The issue requests that the curl/bash installer leave `openclaw` installed and discoverable after reporting success.

## Best Possible Solution

Make installer finalization self-verifying and actionable so a success message either leaves the CLI usable from the user's shell or clearly reports the exact PATH/profile command needed before onboarding.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run the npm installer where npm's global bin directory is absent from the parent shell PATH and do not pass `--verify`; current main can warn and still finish successfully before `openclaw` is runnable from that shell.

## Solution Assessment

Yes on direction. The narrow maintainable fix belongs in `scripts/install.sh` around finalization, CLI resolution, and verification copy rather than changing npm install semantics or adding a new configuration mode.
