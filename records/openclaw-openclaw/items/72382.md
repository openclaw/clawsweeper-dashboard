## Summary

Keep open. Current main still matches the reported installer failure mode: the Unix curl installer can detect that the user's original shell PATH lacks npm's global bin directory, warn that `openclaw` may be command-not-found, skip verification by default, and still print the installed-successfully banner.

## What This Changes

The issue asks for the macOS/Linux curl installer on Ubuntu 26.04 to leave `openclaw` installed and immediately runnable after the install script completes.

## Best Possible Solution

Make the installer completion state match what the user can do next: either make `openclaw` discoverable for the immediate follow-up command, or turn the missing original-PATH case into an action-required result instead of a successful install. A docs-only workaround is not enough because the normal install path currently tells users to proceed.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the npm-method installer with npm's global bin directory absent from the original shell PATH and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about the PATH gap and still continue to the success path.

## Solution Assessment

Yes. A narrow installer fix with focused shell-script tests and a smoke check is the best path; broad install-flow refactoring or only adding troubleshooting text would leave the misleading default success state intact.
