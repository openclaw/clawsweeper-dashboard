## Summary

Keep open: current main still lets the Unix npm installer warn that npm's global bin directory is missing from the user's original PATH, resolve `openclaw` only inside the installer process, skip verification by default, and print a success banner before the command is guaranteed to work in the user's shell.

## What This Changes

The issue asks the curl-based Unix installer to make OpenClaw installed and runnable after it reports success on Ubuntu.

## Best Possible Solution

Make the installer success state match what the user can do next: either ensure the npm global bin directory is durable for the user's shell before success, or stop short of a success banner with clear action-required output and focused regression coverage.

## Reproduction Assessment

Yes. The high-confidence reproduction path is the default npm installer flow with an original PATH that lacks npm's global bin directory: the installer warns, can resolve `openclaw` in its own process, skips verification by default, and still prints success.

## Solution Assessment

Yes. A narrow `scripts/install.sh` behavior fix with focused script tests is the maintainable solution; docs-only guidance would leave the installer success state misleading.
