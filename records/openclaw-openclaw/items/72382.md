## Summary

Keep open. Current main still has the reported installer path: the script can warn that npm's global bin directory is missing from PATH, keep install verification opt-in, and print a successful install before the user's parent shell can run `openclaw`.

## What This Changes

The issue reports that the curl-based Linux installer says OpenClaw installed successfully but leaves `openclaw` unavailable on PATH.

## Best Possible Solution

Make the installer final state self-verifying and actionable so a successful banner is not the last meaningful outcome when the CLI is only available through a bin directory missing from the user's shell PATH.

## Reproduction Assessment

Yes, for the reported behavior class. Current source still supports a Linux npm install where the npm global bin is absent from the original PATH, default verification is off, and the installer can finish successfully while the parent shell reports `openclaw: command not found`.

## Solution Assessment

Unclear on the exact exit/copy semantics, but the narrow maintainable fix belongs in `scripts/install.sh`: preserve npm prefix repair while making the final CLI path, PATH export, or verification failure explicit.
