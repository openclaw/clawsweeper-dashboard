## Summary

Keep open: current main still allows the Unix npm installer to warn that npm's global bin directory is missing from the caller's original PATH, resolve `openclaw` only inside the installer process, skip verification by default, and print a success banner before the command is guaranteed to work in the user's next shell.

## What This Changes

The issue reports that the curl/bash installer claimed success on Ubuntu, but `openclaw` was still unavailable afterward.

## Best Possible Solution

Make the installer success state match what the user can do next: either persist the npm global bin directory for the user's shell before success, or stop short of a success banner with clear action-required output and regression coverage.

## Reproduction Assessment

Yes. The high-confidence reproduction path is the default npm installer flow with an original PATH that lacks npm's global bin directory: the script warns, can resolve `openclaw` inside its own process, skips verification by default, and still prints success.

## Solution Assessment

Yes. A narrow `scripts/install.sh` behavior fix with focused script tests is the maintainable solution; docs-only guidance would leave the installer success state misleading.

## Repair Work Prompt

Fix the installer issue at https://github.com/openclaw/openclaw/issues/72382. In `scripts/install.sh`, make the npm install success state reflect whether `openclaw` will be usable from the user's shell when the caller's original PATH lacks npm's global bin directory. Do not treat installer-local `resolve_openclaw_bin` success as sufficient for a plain success banner. Either make the PATH change durable before success, or emit clear action-required output and avoid claiming the command is ready. Add focused regression coverage in `test/scripts/install-sh.test.ts` for the original-PATH/missing npm-bin flow and the default verification behavior. Update user-facing docs and `CHANGELOG.md` if the output or recovery guidance changes. Do not change Node installation policy, package-manager selection, release metadata, or unrelated smoke infrastructure.
