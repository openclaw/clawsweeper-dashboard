## Summary

Keep open. Current main still has a source-reproducible installer path where the npm install can warn that the global npm bin directory is missing from the user's original PATH, resolve `openclaw` only inside the installer process, and still print a success banner before `openclaw` is runnable from that shell.

## What This Changes

The issue reports that the curl-bash installer says OpenClaw installed successfully on Ubuntu 26.04, but `openclaw` is not available afterward.

## Best Possible Solution

Make installer finalization self-verifying and actionable so a successful install either leaves a directly usable CLI in the user's shell path or clearly reports the exact PATH/onboarding recovery command before success.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: run the npm installer with npm's global bin directory absent from the original shell PATH and without `--verify`; the script can warn, resolve the CLI internally, and still print success.

## Solution Assessment

Unclear on the exact exit and copy semantics, but the narrow maintainable solution belongs in `scripts/install.sh`, with focused tests for finalization when npm's global bin is missing from the user's PATH rather than a docs-only change.
