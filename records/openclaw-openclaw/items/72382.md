## Summary

Keep open. Current main still has the reported installer path: the npm installer can warn that the npm global bin directory is absent from the parent shell PATH, skip verification by default, and still print a success banner before `openclaw` is runnable from that shell.

## What This Changes

The issue asks for the curl-bash install path on Ubuntu 26.04 to leave `openclaw` installed and available instead of reporting success while the command is not found.

## Best Possible Solution

Make installer finalization self-verifying and actionable so a success path either leaves a runnable `openclaw` in the user's shell path or clearly reports the exact PATH repair before onboarding.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run the npm installer where npm's global bin directory is absent from the parent shell PATH and do not pass `--verify`; current main can warn but still finish with the success banner.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/install.sh` finalization and regression coverage, not as a docs-only workaround or a new user option.
