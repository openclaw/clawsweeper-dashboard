## Summary

Keep open. Current main still matches the reported installer failure mode: the Unix curl installer can detect that the original shell PATH lacks npm's global bin directory, warn about command-not-found, skip verification by default, and still print the installed-successfully banner.

## What This Changes

The issue asks for the Unix curl installer to leave `openclaw` runnable after a successful Ubuntu npm install, or stop reporting success when it is not discoverable.

## Best Possible Solution

Make the installer's normal success path match what the user can run next: either ensure `openclaw` is discoverable for the user's immediate follow-up command, or treat the missing original PATH case as action-required instead of successful.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the npm-method installer with npm's global bin directory absent from the original shell PATH and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about the PATH gap and still continue to success.

## Solution Assessment

Yes. A narrow installer fix with targeted tests and smoke coverage is the maintainable path; docs-only guidance would not fix the misleading default success state.
