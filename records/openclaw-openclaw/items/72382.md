## Summary

Keep open. Current main still has the installer UX path described in the report: the script can warn that the npm global bin directory is missing from the original PATH, then continue to print a successful install message while the parent shell may still report `openclaw: command not found`.

## What This Changes

The issue asks for the curl-bash installer to leave a working `openclaw` command after a general Ubuntu install instead of reporting success when the CLI is not discoverable.

## Best Possible Solution

Make installer finalization self-verifying and actionable: avoid a plain success end state when the CLI is only reachable through an npm bin directory missing from the user's original shell PATH, and cover that behavior in focused installer tests/docs.

## Reproduction Assessment

Yes, for the reported behavior class. Current source still warns about the missing npm global bin directory, leaves verification off by default, and can print installed-success messaging even though the parent shell may not find `openclaw`.

## Solution Assessment

Unclear for the exact final copy, but the narrow maintainable fix belongs in `scripts/install.sh`: keep npm prefix repair, then make the final CLI path or required PATH export explicit, and fail only when the package or CLI truly cannot be resolved.
