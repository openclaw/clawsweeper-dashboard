## Summary

Keep open. Current main still matches the reported installer failure mode: the Unix installer can warn that the parent shell PATH lacks npm's global bin directory, skip verification by default, and still print the normal installed-successfully banner before the user's next `openclaw` command can work.

## What This Changes

The issue reports that the curl installer on Ubuntu 26.04 says OpenClaw installed successfully even though the `openclaw` command is not available afterward.

## Best Possible Solution

Make the installer success path match the next command the user can run: either make `openclaw` discoverable for the user's immediate follow-up shell, or treat a missing original-shell PATH as action-required instead of a normal successful install.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the npm-method installer with npm's global bin directory absent from `ORIGINAL_PATH` and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about the PATH gap and still continue to the success banner.

## Solution Assessment

Yes. A narrow installer fix with targeted tests and smoke coverage is the maintainable solution; docs-only guidance would leave the misleading default success state intact.
