## What This Changes

The branch adds standalone Markdown link lines to the heartbeat empty-content skip list and adds a regression test for the default HEARTBEAT.md Related link footer.

## Best Possible Solution

Narrow non-actionable link handling to the docs/template Related footer or strip that footer during workspace template loading, while keeping user-authored link-only heartbeat tasks actionable and covered by regression tests.

## Reproduction Assessment

Yes at source level: current main's template contains the Related footer link, and the current predicate treats that link as actionable content so the empty-heartbeat skip does not fire. I did not run a live heartbeat cycle.

## Solution Assessment

No as written: the patch fixes the docs footer symptom but broadens the empty heuristic to all standalone Markdown links. A scoped template/footer fix is safer and preserves existing user-authored link-only tasks.
