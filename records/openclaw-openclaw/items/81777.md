## Summary

Keep open: the PR is useful but not ready because bound Codex app-server conversation turns still enter a shared per-transcript queue before reaching the isolated client path.

## What This Changes

The PR adds opt-in Codex app-server session isolation and runtime-policy key plumbing across Codex runs, commands, compaction, reset cleanup, gateway sessions, plugin commands, config, and tests.

## Best Possible Solution

Keep the opt-in session-isolation design, but make every shared state and bound-turn queue use the same runtime-policy isolation key before rebasing onto current main.

## Reproduction Assessment

No local high-confidence repro was established in this read-only review. The PR body supplies real setup proof, and source inspection confirms current main shares one Codex app-server client per agent while the PR adds per-session isolation plumbing.

## Solution Assessment

No, not yet. Per-session client and binding isolation is a reasonable narrow direction, but bound conversation turns also need queue isolation before the patch satisfies the stated contract.
