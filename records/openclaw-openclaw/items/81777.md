## Summary

Keep this PR open: the direction is useful and the supplied real-environment proof is credible, but the current head still has a concrete correctness bug where session-isolated bound conversation turns serialize through one transcript-file queue before reaching the isolated client path.

## What This Changes

This PR adds opt-in Codex app-server session/client isolation, runtime-policy key propagation through Codex and gateway command paths, new app-server config knobs, and focused tests.

## Best Possible Solution

Keep the opt-in isolation design, but make every per-session shared state path, including bound-turn queueing, use the same runtime-policy isolation key before rebasing and running focused Codex/gateway validation.

## Reproduction Assessment

No executed local repro was established in this read-only review. The issue is source-reproducible: current main shares one Codex app-server client per agent, and the PR head still queues bound app-server turns by transcript file before applying isolation.

## Solution Assessment

No, not yet. Per-session client and binding isolation is a reasonable narrow direction, but the queue key must include the isolation key for the patch to satisfy its own contract.
