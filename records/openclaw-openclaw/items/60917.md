## Summary

Keep open. Current main still serializes the dashboard form payload without an original-aware redaction-sentinel cleanup step, while the gateway write path correctly rejects unrestorable `__OPENCLAW_REDACTED__` values; the linked focused fix was closed unmerged.

## What This Changes

The issue asks for dashboard form saves to omit or restore redacted secret placeholders instead of submitting `__OPENCLAW_REDACTED__` as literal config data.

## Best Possible Solution

Add a UI-side, original-aware sanitizer before form-mode `config.set`/`config.apply` serialization that drops unrestorable redacted placeholders and preserves placeholders for existing secrets.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to set `state.configForm.gateway.remote.token` to `__OPENCLAW_REDACTED__` while the original snapshot lacks that path, then save in form mode; current serialization sends it and the gateway restore path rejects it.

## Solution Assessment

Yes. The narrowest maintainable solution is in the dashboard form-save path, because relaxing `restoreRedactedValues` would weaken the documented reserved-sentinel contract.
