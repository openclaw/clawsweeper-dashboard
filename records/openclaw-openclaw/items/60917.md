## Summary

Keep open. Current main still serializes the dashboard form payload directly for form-mode config saves, while the gateway correctly rejects unrestorable `__OPENCLAW_REDACTED__` values; the focused fix in #60971 was closed unmerged.

## What This Changes

The issue asks for dashboard form saves to omit or preserve redacted secret placeholders so `gateway.remote.token` sentinels do not block unrelated settings saves.

## Best Possible Solution

Add a UI-side, original-aware sanitizer in the form save path that drops unresolved redaction sentinels for absent sensitive paths, preserves placeholders for existing secrets, and leaves gateway restore validation fail-closed.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to place `__OPENCLAW_REDACTED__` at `configForm.gateway.remote.token` while the original snapshot lacks that path, then save in form mode; current serialization sends the sentinel and the gateway restore path rejects it.

## Solution Assessment

Yes. The narrowest maintainable solution is before form-mode `config.set`/`config.apply` serialization in the dashboard; relaxing `restoreRedactedValues` would weaken the documented reserved-sentinel contract.
