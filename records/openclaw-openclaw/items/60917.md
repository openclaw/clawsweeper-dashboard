## Summary

Keep open. Current main still sends the dashboard form payload to `config.set`/`config.apply` without an original-aware cleanup for unrestorable redaction sentinels, and the gateway correctly rejects `__OPENCLAW_REDACTED__` when there is no original value to restore.

## What This Changes

Dashboard form saves should omit or restore redacted secret placeholders so an unused `gateway.remote.token` sentinel does not fail unrelated settings saves.

## Best Possible Solution

Add an original-aware UI sanitizer before form-mode submit serialization that removes unrestorable redacted placeholders and leaves restorable existing secret placeholders intact.

## Reproduction Assessment

Yes, by source inspection. A form-mode save with `state.configForm.gateway.remote.token` set to `__OPENCLAW_REDACTED__` and no matching original path will serialize the sentinel, then the gateway restore layer will reject it.

## Solution Assessment

Yes. The narrow maintainable fix is UI-side cleanup before form-mode `config.set`/`config.apply`; weakening `restoreRedactedValues` would undermine the reserved-sentinel validation contract.
