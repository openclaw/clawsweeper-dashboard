## Summary

Keep open. Current main still serializes the dashboard form payload without removing unrestorable redaction sentinels, while the gateway write path correctly rejects `__OPENCLAW_REDACTED__` when no original value exists; the focused fix PR was closed unmerged.

## What This Changes

The issue asks for dashboard settings saves to omit or restore redacted secret placeholders so unrelated form saves do not submit `gateway.remote.token` as literal config data.

## Best Possible Solution

Add an original-aware sanitizer in the dashboard form-save path before `config.set` and `config.apply`, with regression coverage for both dropping unrestorable placeholders and preserving existing redacted secrets.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to set `state.configForm.gateway.remote.token` to `__OPENCLAW_REDACTED__` while the original snapshot lacks that path, then save in form mode; current serialization sends it and the gateway restore path rejects it.

## Solution Assessment

Yes. The narrowest maintainable fix is UI-side original-aware cleanup before form serialization, because relaxing `restoreRedactedValues` would weaken the reserved-sentinel contract.
