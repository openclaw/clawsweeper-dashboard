## Summary

Keep open. Current main still submits the dashboard form payload without an original-aware redaction-sentinel filter, while gateway writes correctly reject unrestorable `__OPENCLAW_REDACTED__` values; the focused fix in #60971 was closed unmerged.

## What This Changes

Dashboard saves should omit unresolved redacted secret placeholders instead of submitting `__OPENCLAW_REDACTED__` as literal `gateway.remote.token` config data.

## Best Possible Solution

Add a UI-side, original-aware sanitizer in the form save path that drops unresolved redaction sentinels for absent sensitive paths, preserves placeholders for existing secrets, and leaves gateway restore validation unchanged.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to put `__OPENCLAW_REDACTED__` at `configForm.gateway.remote.token` while the original snapshot lacks that path, then save in form mode; current serialization sends the sentinel and the gateway restore path rejects it.

## Solution Assessment

Yes. The narrowest maintainable fix is before form-mode `config.set`/`config.apply` serialization in the dashboard; relaxing `restoreRedactedValues` would weaken the documented reserved-sentinel contract.
