Overall correctness: patch is incorrect

Full review comments:

- **[P2] Normalize preserved job ids before duplicate checks:** `src/cron/store.ts:90`
  - body: `ensureLoaded` canonicalizes persisted IDs before runtime use, including trimming `id` and accepting legacy `jobId`, but this helper only returns the raw `id`. A skipped unsupported row with `id: " job "` or `jobId: "job"` will not match an occupied supported job `id: "job"`, so a full persist can write both rows and resurrect the stale unsupported entry the PR says should be skipped. Use the same canonical ID semantics here and cover those collisions.
