## What This Changes

Add an ADMIN-scoped Gateway `agents.setDefault` RPC that marks one configured agent as default and clears the default flag from the others.

## Best Possible Solution

Add the RPC end to end through the Gateway protocol surface, persisting only `agents.list[].default` and leaving `config.patch` as the advanced bulk-edit path.

## Solution Assessment

Yes. A small ADMIN-scoped RPC over the existing `agents.list[].default` storage contract is the narrow maintainable path and avoids adding a persisted `agents.defaultId` key.
