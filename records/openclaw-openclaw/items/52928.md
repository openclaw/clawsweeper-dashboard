## Summary

Keep open. Current `main` and latest release still lack a dedicated `agents.setDefault` Gateway RPC; the related Control UI `agents.defaultId` bug was fixed separately by https://github.com/openclaw/openclaw/pull/72585, and the linked discussion explicitly keeps this Gateway RPC request separate.

## What This Changes

Request adds an ADMIN-scoped `agents.setDefault` Gateway RPC that sets one configured agent as default and returns the new default id.

## Best Possible Solution

Add the RPC through the Gateway protocol surface end to end, keeping `agents.list[].default` as storage and avoiding a persisted `agents.defaultId` key.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive Gateway feature request. Source inspection is high-confidence because current main exposes only list/create/update/delete across the handler, method list, scope table, protocol schemas, SDK, and docs.

## Solution Assessment

Yes. A small ADMIN-scoped RPC over the existing `agents.list[].default` storage contract is narrower for operator clients than raw `config.patch`, while preserving `config.patch` for bulk edits.
