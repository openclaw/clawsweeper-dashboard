## Summary

Keep open. Current `main` and latest release `v2026.5.22` still lack a dedicated `agents.setDefault` Gateway RPC; the existing Control UI path only stages `agents.list[].default` through config form mutation, so external clients still need raw config mutation for this operation.

## What This Changes

The issue requests a dedicated `agents.setDefault` Gateway RPC to change the default configured agent without raw `config.patch` merge-patch handling.

## Best Possible Solution

Add an additive `operator.admin` RPC that validates a configured agent id, atomically marks that `agents.list[]` entry as default, clears other defaults, and returns the resolved `defaultId`.

## Reproduction Assessment

Yes. Source inspection on current `main` shows no `agents.setDefault` handler, descriptor, protocol schema, or SDK helper, while `agents.update` is a closed schema that does not accept a default flag.

## Solution Assessment

Yes. The requested additive RPC is the narrow maintainable path because it reuses the existing `agents.list[].default` storage contract instead of making every client construct raw config patches.
