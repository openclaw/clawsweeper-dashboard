## What This Changes

The issue requests an admin-scoped `agents.setDefault` Gateway RPC so clients can change the default agent without raw config mutation.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/92957; once an accepted implementation is on main, close this issue with the merge commit and release or current-main proof.

## Reproduction Assessment

Yes for the feature gap: current main has no handler, descriptor, protocol schema, SDK wrapper, or Admin HTTP allowlist entry for `agents.setDefault`, and `agents.update` rejects a `default` field by schema. I did not run a live Gateway call because this is a read-only issue review.

## Solution Assessment

Yes, if maintainers accept the public API surface: adding a narrow admin-scoped `agents.*` mutation is cleaner than requiring clients to edit `agents.list` through `config.patch` or `config.set`. The best path is the linked PR or an equivalent core Gateway implementation, not another config-form workaround.
