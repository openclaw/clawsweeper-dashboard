## Summary

Keep open. Current main and latest release v2026.5.22 still lack a dedicated `agents.setDefault` Gateway RPC, and clients still need either raw config mutation or the current Control UI config-form workaround to change the default agent.

## What This Changes

Add an operator-admin `agents.setDefault` Gateway RPC that sets one configured agent as the default and returns the resulting default id.

## Best Possible Solution

Add `agents.setDefault` as an additive `operator.admin` RPC that validates a configured agent id, atomically sets exactly that `agents.list[]` entry to `default: true`, clears other defaults, returns `{ ok: true, defaultId }`, and updates protocol, SDK, docs, and tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows no `agents.setDefault` handler, descriptor, protocol schema, or SDK helper, while `agents.update` is a closed schema that does not accept a default flag.

## Solution Assessment

Yes. The requested additive RPC is the narrow maintainable solution because it reuses the existing `agents.list[].default` storage contract instead of making every client construct raw config patches.
