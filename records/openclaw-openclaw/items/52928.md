## Summary

Request to add an ADMIN-scoped `agents.setDefault` Gateway RPC for changing the configured default agent without using `config.patch`.

## Best Possible Solution

Add `agents.setDefault` end to end over the existing `agents.list[].default` contract, preserving `config.patch` as the bulk-edit escape hatch and avoiding a persisted `agents.defaultId` key.

## Solution Assessment
