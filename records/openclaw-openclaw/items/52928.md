## Summary

Keep open. Current main still lacks a dedicated `agents.setDefault` Gateway RPC across the handler, advertised method list, scope table, protocol schema/validators, SDK namespace, and public docs; the Control UI can stage the same config shape, but only through the heavier config-save path.

## What This Changes

Add an ADMIN-scoped Gateway RPC method, `agents.setDefault`, that sets exactly one configured agent as default and returns the new `defaultId`.

## Best Possible Solution

Add the additive ADMIN RPC using `agents.list[].default`, validate configured agent IDs, clear other defaults, persist through the existing config writer, and update protocol/schema/SDK/docs/tests/changelog together.

## Solution Assessment

Yes, the requested ADMIN-scoped RPC is the narrowest maintainable solution for external operator clients that need a simple default-agent action. `config.patch` can remain the bulk escape hatch, but it is too broad for this single mutation.
