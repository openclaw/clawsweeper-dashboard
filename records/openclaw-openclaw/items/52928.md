## Summary

Keep open. Current main and the latest release still lack a dedicated `agents.setDefault` Gateway RPC; the merged Control UI fix in https://github.com/openclaw/openclaw/pull/72585 fixed the `agents.defaultId` save bug but explicitly left this Gateway API request separate.

## What This Changes

Add an ADMIN-scoped `agents.setDefault` Gateway RPC that sets `agents.list[].default` for one configured agent and clears it from the others.

## Best Possible Solution

Add the RPC end to end through the Gateway protocol surface, using `agents.list[].default` as the only persisted storage contract and leaving `config.patch` for advanced bulk edits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive Gateway feature request. Source and release inspection show the current handler, method specs, protocol schema, SDK, docs, and latest release expose only the existing agent CRUD/file methods.

## Solution Assessment

Yes. A small ADMIN-scoped RPC over the existing `agents.list[].default` storage contract is the narrow maintainable path for operator clients and avoids adding a persisted `agents.defaultId` key.
