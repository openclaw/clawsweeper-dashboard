## Summary

Keep open. Current `main` and v2026.5.18 still expose agent CRUD and file helpers but no dedicated `agents.setDefault` Gateway RPC; the related UI `agents.defaultId` fixes corrected the existing config mutation path rather than adding this protocol method.

## What This Changes

Request adds an ADMIN-scoped `agents.setDefault` Gateway RPC so clients can change the default agent without using raw `config.patch`.

## Best Possible Solution

Add an ADMIN-scoped `agents.setDefault` end to end over the existing `agents.list[].default` contract, preserving `config.patch` as the bulk-edit escape hatch and avoiding a persisted `agents.defaultId` key.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive Gateway protocol feature request. Source inspection gives high confidence that current `main` lacks the requested method across handler, descriptors, protocol schema/types, SDK, and docs.

## Solution Assessment

Yes, if maintainers approve the new public RPC. A dedicated ADMIN method over `agents.list[].default` is narrower for clients than raw `config.patch`, but it should land as a complete protocol/API change rather than a local handler-only patch.
