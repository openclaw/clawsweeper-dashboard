## Summary

Keep open. Current `main` and latest release `v2026.5.22` still lack a dedicated `agents.setDefault` Gateway RPC; the related Control UI bug was fixed by https://github.com/openclaw/openclaw/pull/72585 by writing `agents.list[].default`, but that does not give desktop/web clients the requested narrow RPC.

## What This Changes

Request adding an `agents.setDefault` Gateway RPC that marks one configured agent as default and clears the default flag from the others.

## Best Possible Solution

Add an additive `agents.setDefault` Gateway RPC that performs the canonical `agents.list[].default` mutation server-side, rejects unknown agents, returns the new `defaultId`, and keeps `config.patch`/`config.set` as bulk-edit escape hatches.

## Reproduction Assessment

Yes. Source inspection on current `main` shows no `agents.setDefault` descriptor, handler, schema, SDK helper, or docs entry, while `agents.update` is a closed schema without a default flag.

## Solution Assessment

Yes. A narrow additive RPC is the maintainable solution for desktop/web clients because the current workaround still requires whole-config editing and base-hash handling for a single common action.
