## Summary

Keep open: the central timeout bug is already fixed on current main by the merged related PR, but this branch still contains a meaningful lower-layer refinement for providers that ignore AbortSignal and it needs maintainer/API-surface review before merge.

## What This Changes

The branch refines memory_search timeout cancellation by racing parent aborts inside the embedding timeout helper, suppressing retry/fallback after caller cancellation, and exporting a named MemorySearchOptions type with tests.

## Best Possible Solution

Land the helper-level abort-settling refinement only after either keeping MemorySearchOptions private or explicitly accepting the new public SDK export with a regenerated API baseline.

## Reproduction Assessment

Yes. From source, current main still lets runEmbeddingOperationWithTimeout wait on the embedding watchdog when a provider ignores AbortSignal; the PR body also includes loopback live output showing parent abort propagation to a provider-style fetch path.

## Solution Assessment

Mostly yes. Settling parent aborts in the embedding timeout helper is the narrow runtime refinement, but the named public MemorySearchOptions export should either be accepted with the generated API baseline updated or removed to avoid unnecessary SDK expansion.
