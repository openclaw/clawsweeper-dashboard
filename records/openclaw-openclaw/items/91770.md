## Summary

Keep open: this is a focused fix for the source-proven memory_search orphaned embedding bug, but the patch still lets caller-abort errors enter the fallback-provider path, so a timed-out search can switch memory provider state after cancellation.

## What This Changes

This PR threads a memory_search deadline AbortSignal through the memory search manager and builtin query embedding path, exports the option type, and adds timeout/cancellation coverage.

## Best Possible Solution

Land the abort propagation after guarding caller-cancelled searches from fallback activation, with regression coverage for a fallback-configured timeout.

## Reproduction Assessment

Yes. Current main source shows the 15s memory_search deadline does not provide a cancellation signal, and the PR's tests/proof exercise the intended cancellation path; the fallback-state blocker is also source-reproducible from the catch path and activateFallbackProvider.

## Solution Assessment

No as written. Threading the signal through memory_search and builtin query embeddings is the right layer, but caller abort must be treated as cancellation instead of a provider failure that can activate fallback.
