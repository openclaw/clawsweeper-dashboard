The PR makes memory_search caller aborts settle embedding operations sooner, prevents cancelled searches from falling into provider fallback, adds focused regression coverage, and exports a shared MemorySearchOptions type.

Land the lower-layer abort-settling fix only after the public SDK type/export is accepted or removed and the plugin SDK API baseline matches the final surface.

## Reproduction Assessment

Yes from source: current main passes the tool deadline signal into builtin memory search, but the lower embedding helper does not race parent aborts, so a provider operation that ignores the signal can keep the abandoned search pending until the embedding timeout. The PR body also includes after-fix loopback output for that path.

## Solution Assessment

Mostly yes: racing caller aborts in the embedding timeout helper is the narrow place for providers that ignore the signal. The public MemorySearchOptions export should either be accepted with the API baseline regenerated or removed to avoid an unnecessary SDK expansion.
