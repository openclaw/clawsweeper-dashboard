## Summary

Keep this PR open, but do not merge it as-is. The Feishu gaps are real and useful, while the current patch is stale against main's plugin-owned channel seams and still has an inherited-allowlist write bug called out in review discussion.

## What This Changes

The PR adds Feishu support across `/allowlist` config operations, markdown-capable formatting, envelope stripping, hooks channel validation, `security audit --fix`, and related tests, plus a shared Bun CI version bump.

## Best Possible Solution

Rework the change against current main by adding Feishu-owned plugin adapters and metadata, preserving inherited account config during edits, and removing unrelated CI runtime changes from the channel fix.

## Reproduction Assessment

Yes. Source inspection of the PR head shows Feishu is added to the generic allowlist write path while existing entries are read only from the account-local write target, so inherited top-level Feishu allowlist entries are omitted before account-local writes.

## Solution Assessment

No. The narrow maintainable direction is to implement Feishu's allowlist, markdown capability, and security-fix behavior through the current plugin contracts, not by adding Feishu-specific hard-coded branches in core command/security lists.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve inherited Feishu allowlists before edits:** `src/auto-reply/reply/commands-allowlist.ts:417`
  - body: By adding Feishu to the generic account-target writer, the first account-scoped `/allowlist add ... channel=feishu` reads existing entries only from `channels.feishu.accounts.<id>`. Feishu accounts inherit top-level `allowFrom`/`groupAllowFrom`, so writing a new account-local list here overrides and drops those inherited IDs from the effective allowlist.
  - confidence: 0.89
- **[P2] Move Feishu allowlist support into the plugin adapter:** `src/auto-reply/reply/commands-allowlist.ts:619-624`
  - body: This Feishu-specific branch keeps channel-owned allowlist behavior in the core command handler. Current main's handler already delegates through `plugin.allowlist`, so the fix should add the Feishu adapter in the plugin; otherwise the core path diverges from the current plugin contract and remains harder to extend for other channels.
  - confidence: 0.82
