## Best Possible Solution

Implement Feishu `/allowlist` support in the Feishu plugin via the existing SDK adapter, add plugin metadata for confirmed markdown support, preserve inherited account/top-level allowlists, and keep the dynamic current-main security and hook handling.

## Reproduction Assessment

## Solution Assessment

No. The Feishu parity goal is valid, but hardcoding Feishu into old core/static lists is no longer the best solution; current main's plugin adapter and metadata seams are the safer implementation path.
