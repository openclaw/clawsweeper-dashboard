## What This Changes

The PR extends Feishu handling across allowlist commands, hook channel validation, markdown detection, envelope stripping, security fixes, and targeted tests.

## Best Possible Solution

Rework this onto current main by adding a Feishu plugin-owned allowlist adapter and the smallest metadata/envelope coverage needed, while preserving inherited account allowlists during edits.

## Reproduction Assessment

Yes, source-reproducible for the blocking PR bug. In a Feishu multi-account config where an account inherits top-level `allowFrom`, the PR reads only the account-local target before writing and can omit inherited entries.

## Solution Assessment

No. The branch patches older core hard-coded paths, while current main has moved /allowlist to plugin-owned adapters; the maintainable fix belongs in the Feishu plugin adapter and metadata surfaces.
