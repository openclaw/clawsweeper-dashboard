## Summary

Keep open. The Feishu parity goal is still useful, but current main only covers some supporting surfaces and the PR head still has a P1 allowlist inheritance bug plus insufficient after-fix real behavior proof.

## What This Changes

The branch adds Feishu handling to `/allowlist`, markdown-capable detection, envelope stripping, hook channel validation, `security audit --fix`, a Bun setup bump, and targeted tests.

## Best Possible Solution

Implement Feishu parity through the Feishu plugin: a merged-config-safe allowlist adapter, markdown capability metadata, any needed envelope/security hooks, focused tests, and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. The PR head enables Feishu in the generic config-edit path while the write code reads only the selected raw account target, which can shadow inherited top-level allowlists without needing a live Feishu setup.

## Solution Assessment

No. The requested parity is reasonable, but this branch hardcodes Feishu into older core/static paths instead of using the current plugin-owned adapter and metadata surfaces.
