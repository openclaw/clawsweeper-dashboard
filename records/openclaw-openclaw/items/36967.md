## Summary

Keep this PR open, but it should not merge as-is: current main has superseded some hard-coded pieces with generic plugin contracts, while the PR still has an account-local Feishu allowlist write bug and an unrelated CI runtime change.

## What This Changes

The PR adds Feishu handling for `/allowlist` config edits/listing, markdown-capable detection, envelope stripping, hooks channel validation, `security audit --fix`, tests, and an unrelated Bun version bump.

## Best Possible Solution

Feishu should advertise its markdown capability and own a plugin allowlist adapter that preserves inherited account config, while core keeps generic channel/plugin seams and envelope handling is covered without unrelated CI changes.

## Reproduction Assessment

Yes. Source inspection of the PR head shows a concrete inherited-allowlist loss path, and current main confirms Feishu still lacks the plugin allowlist adapter needed for `/allowlist channel=feishu`.

## Solution Assessment

No. The maintainable fix is a current-main replacement or substantial rebase that implements Feishu-owned plugin metadata/adapters and preserves inherited config, not hard-coded Feishu branches in core command/security lists.
