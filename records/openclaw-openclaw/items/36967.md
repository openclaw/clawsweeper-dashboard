## Summary

Keep open: the Feishu allowlist gap is real, but this PR is not safe to merge as written because it changes authorization-sensitive allowlist writes and current main has already superseded some of the hardcoded-list work with generic plugin seams.

## What This Changes

The branch adds Feishu handling to `/allowlist`, markdown-capability checks, envelope stripping, hook channel validation/help, security-fix hardening, tests, and a Bun setup-action version bump.

## Best Possible Solution

Rework this against current main as a Feishu-owned allowlist adapter plus Feishu metadata/envelope updates, preserving owner/configWrites gates and inherited account behavior, then add changelog and targeted tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `/allowlist` config reads/writes require a plugin `allowlist` adapter, while Feishu has config and pairing support but no such adapter; the PR diff also exposes the write-path regressions.

## Solution Assessment

No. The maintainable fix is a Feishu-owned adapter and plugin metadata on current seams, not hardcoded shared command/security lists or an unrelated CI runtime pin.
