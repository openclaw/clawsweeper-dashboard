## Summary

Keep open: the branch still has a blocking Feishu account allowlist edit bug, lacks after-fix real behavior proof, and is stale against current main's plugin-owned allowlist architecture.

## What This Changes

The PR adds Feishu handling for /allowlist config edits and listing, hook channel validation, markdown capability, envelope stripping, security auto-fix hardening, and targeted tests.

## Best Possible Solution

Rework the change onto current main with Feishu plugin-owned allowlist and security-fix adapters, plus the smallest metadata/envelope updates still needed, while preserving inherited account allowlists during edits.

## Reproduction Assessment

Yes, source-reproducible for the blocking PR bug. In a Feishu multi-account config where an account inherits top-level allowFrom or groupAllowFrom, the PR reads only the account-local write target before writing and can omit inherited entries.

## Solution Assessment

No. The branch patches older core hard-coded paths, while current main expects channel-owned allowlist and security-fix adapters; the safer implementation belongs in the Feishu plugin seam.
