## Summary

Keep open. Current main still lacks Feishu `/allowlist` config editing, but this PR is not mergeable as submitted because it is stale against the current plugin allowlist adapter contract, drops current owner-only write gating, can overwrite inherited Feishu account allowlists, and includes an unrelated CI runtime pin change.

## What This Changes

The branch adds Feishu handling to `/allowlist`, markdown capability checks, envelope stripping, hook channel validation/help, security-fix hardening, targeted tests, and a Bun setup-action version bump.

## Best Possible Solution

Add a Feishu-owned allowlist adapter and metadata update against the current ChannelAllowlistAdapter surface, preserve owner/configWrites gates and inherited account entries, keep already-generic hook/security behavior, and split the Bun setup-action bump out of this patch.

## Reproduction Assessment

Yes. Source inspection is enough: current main requires `plugin.allowlist` adapters for config-backed `/allowlist` behavior, and Feishu currently has config helpers but no allowlist adapter; the PR diff also shows the risky shared-path hardcoding.

## Solution Assessment

No. The PR contains useful source work, but the best solution is a Feishu-owned adapter plus metadata under the current plugin contract, not a hardcoded shared allowlist path that weakens write-gating and account inheritance behavior.
