## What This Changes

The branch adds Feishu-specific `/allowlist` config handling, hook-channel schema/help updates, markdown/envelope/security-fix additions, targeted tests, and a Bun setup patch-version bump.

## Best Possible Solution

Implement the remaining Feishu parity through Feishu plugin-owned allowlist and metadata adapters, preserve merged account config on writes, keep the current generic security and hook-channel behavior, and require redacted after-fix behavior proof.

## Reproduction Assessment

Yes for the PR blocker at source level: PR head reads existing allowlist entries from the raw account write target while Feishu account resolution merges top-level and account config. I did not run a live Feishu or CLI reproduction in this read-only review.

## Solution Assessment

No. Feishu parity is useful, but this PR hardcodes an older core path; the maintainable solution is a Feishu plugin-owned allowlist adapter plus channel metadata updates that preserve inherited config.
