## Summary

Keep open: current main only covers some consistency pieces generically, while Feishu still lacks plugin-backed `/allowlist` config edits and this PR has a blocking inherited-account edit bug plus insufficient after-fix real behavior proof.

## What This Changes

The PR hardcodes Feishu into allowlist config edits/listing, markdown-capability detection, envelope stripping, security-fix groupPolicy hardening, hook channel validation, and CI Bun version, with targeted tests.

## Best Possible Solution

Implement Feishu `/allowlist` support as a current-main Feishu plugin allowlist adapter using the existing SDK helper, preserving merged top-level/account allowlists on edits and keeping shared capability behavior metadata-driven.

## Reproduction Assessment

Yes for the PR-introduced blocker from source inspection: the PR adds Feishu to the generic account editor while that editor reads only account-local entries before writing. The requested Feishu `/allowlist` behavior is also source-verifiable as still missing on current main because the Feishu plugin has no `allowlist` adapter.

## Solution Assessment

No. The desired direction is valid, but hardcoding Feishu into core/static lists is no longer the best fit for current main, and the proposed edit path can shadow inherited allowlist entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve inherited Feishu allowlists on account edits:** `src/auto-reply/reply/commands-allowlist.ts:417`
  - body: Adding `feishu` to the generic allowlist path makes account-scoped edits read only the selected account object before writing `channels.feishu.accounts.<id>.allowFrom`. In a setup where the account inherits top-level `allowFrom` or `groupAllowFrom`, the first add writes an account-local list containing only the new entry and silently drops the inherited allowed IDs. Merge the effective Feishu account config before edits, or better, implement this through a Feishu plugin allowlist adapter that preserves inherited entries.
  - confidence: 0.88
