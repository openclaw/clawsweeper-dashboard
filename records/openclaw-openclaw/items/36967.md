## What This Changes

The branch adds Feishu handling to `/allowlist`, markdown and envelope detection, hook config validation, `security audit --fix`, a Bun setup bump, and targeted tests.

## Best Possible Solution

Add the remaining Feishu support through the Feishu channel plugin: a merged-config-safe allowlist adapter, confirmed markdown capability metadata, any needed envelope/security plugin hooks, focused regression tests, and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main has no Feishu allowlist adapter, and the PR head's generic write path reads only account-local entries before writing, which is enough to trace the inherited-allowlist shadowing failure without running a live Feishu setup.

## Solution Assessment

No. The Feishu parity goal is useful, but the branch hardcodes Feishu into older core/static paths instead of using the current plugin-owned adapter and metadata surfaces.
