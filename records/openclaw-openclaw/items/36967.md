## Summary

Keep open. The Feishu parity problem is still valid, but this branch is not merge-ready on current main: it hardcodes Feishu into core paths that have since moved to plugin-owned contracts, it still has a Feishu multi-account allowlist shadowing bug, and the attached proof does not show after-fix real behavior.

## What This Changes

This PR adds Feishu handling to shared allowlist, hook-channel validation, markdown capability, envelope stripping, and security-fix paths, with targeted tests and an unrelated Bun setup version bump.

## Best Possible Solution

Rework this into a current-main Feishu plugin fix: add plugin-owned allowlist read/edit support, expose markdown capability through plugin metadata if needed, and add a plugin security apply-fix hook for groupPolicy hardening with focused regression tests and real command proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main /allowlist config edits require plugin.allowlist.applyConfigEdit, while the Feishu plugin does not register an allowlist adapter. The PR branch also has a source-reproducible multi-account bug where edits read only the account-local list and can shadow inherited top-level allowlists.

## Solution Assessment

No. The useful behavior should be implemented through current-main plugin-owned Feishu adapters and metadata, not by adding Feishu-specific branches to shared core lists; the branch also needs to preserve inherited allowlists during account-local edits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve inherited Feishu allowlists during edits:** `src/auto-reply/reply/commands-allowlist.ts:417`
  - body: Adding `feishu` to the generic allowlist edit path sends account-scoped writes through code that reads only the selected account object. In a multi-account Feishu config where an account inherits top-level `allowFrom` or `groupAllowFrom`, the first `/allowlist add ... channel=feishu --account ...` writes an account-local list containing only the new entry, shadowing previously effective inherited allowlist entries. Merge the effective inherited list before writing, or move the edit into a Feishu plugin allowlist adapter that uses the resolved Feishu config.
  - confidence: 0.87
