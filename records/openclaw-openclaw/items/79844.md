## Summary

Keep open: current main still omits the persisted `idempotencyKey` from the hydrated `__openclaw` envelope, and the open linked fix PR has closing syntax but has not merged.

## What This Changes

Surface persisted `chat.send` idempotency keys on hydrated `chat.history` user messages for client-side optimistic-message correlation.

## Best Possible Solution

Land a focused additive Gateway change that carries persisted keys into `__openclaw.idempotencyKey` for full history, recent history, and live session-history events with targeted protocol/runtime coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a transcript row with `message.idempotencyKey` reaches `parsedSessionEntryToMessage`, which only attaches `id` and `seq` today.

## Solution Assessment

Yes. Echoing the already persisted key through the existing `__openclaw` metadata envelope is the narrowest maintainable direction; this issue should stay open while https://github.com/openclaw/openclaw/pull/80044 is reviewed, refreshed, or replaced.
