## Summary

Keep open: current main still omits the persisted key from the `chat.history` `__openclaw` envelope, and the linked fixing PR is open rather than merged.

## What This Changes

Expose the persisted `chat.send` `idempotencyKey` inside the `__openclaw` envelope returned by `chat.history` so clients can correlate optimistic user messages.

## Best Possible Solution

Land a focused additive gateway change that carries the existing per-message key into `__openclaw.idempotencyKey` for history and live session-history projections, or close the linked PR and replace it with an equivalent fix.

## Reproduction Assessment

Yes. Source inspection on current main shows a transcript message with `message.idempotencyKey` reaches the history hydrator, but the hydrator only attaches `id` and `seq` to `__openclaw`.

## Solution Assessment

Yes. Echoing the existing persisted key through the gateway history projection is the narrowest maintainable direction; the current linked PR should be reviewed and landed or replaced before closing this issue.
