## Summary

Keep open: current main still lacks `config unset` dry-run support and this PR is the active linked fix for https://github.com/openclaw/openclaw/issues/80721, but the latest head still has two blocking correctness gaps. The `maintainer` label also rules out cleanup closure.

## What This Changes

The PR adds `--dry-run`, `--json`, and `--allow-exec` handling for `openclaw config unset`, routes those flags through the fast parser, and adds focused CLI tests.

## Best Possible Solution

Finish and land one canonical `config unset` dry-run implementation with structured JSON for all dry-run outcomes, complete SecretRef fallout checks for broad provider/default unsets, and parity with `config set` and `config patch` validation semantics.

## Reproduction Assessment

Yes, source-reproducible. Current main registers no dry-run flags for `config unset`, and the linked issue shows the same help-output gap against a released CLI.

## Solution Assessment

No. Reusing `runConfigOperations` is the right direction, but this PR still needs structured JSON for missing-path dry-runs and broader SecretRef fallout collection before it is the narrow maintainable solution.

- **[P2] Return JSON for missing unset dry-run paths:** `src/cli/config-cli.ts:2046`
  - body: With `--dry-run --json`, a missing path still exits through the text-only `Config path not found` branch before the new dry-run operation runs. JSON callers receive stderr text instead of a structured failure payload, so handle this dry-run/json preflight branch and cover it with a test.
  - confidence: 0.88

  - confidence: 0.84
