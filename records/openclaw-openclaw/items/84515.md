## Summary

Keep open: the linked bug is real, but this PR is not merge-ready because the host/IP-only dedupe can hide the documented same-host multi-gateway warning, and the external PR still lacks real-environment proof.

## What This Changes

The PR changes gateway status warning construction to suppress `multiple_gateways` when reachable probe targets share normalized `self.host`/`self.ip`, and adds focused output tests.

## Best Possible Solution

Keep both probe paths, but suppress `multiple_gateways` only when a stable instance-aware identity matches, such as host/IP plus config path or gateway port when available, with regression coverage for same-gateway dual transport and same-host separate gateways.

## Reproduction Assessment

Yes, source-level. Current main can probe both an SSH tunnel and configured remote URL for the same gateway, and `buildGatewayStatusWarnings` currently emits `multiple_gateways` solely from `reachable.length > 1`.

## Solution Assessment

No. Identity-aware warning classification is the right direction, but the proposed host/IP-only key is not instance-aware enough for documented same-host multi-gateway setups.
