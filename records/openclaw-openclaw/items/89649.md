## Summary

Keep open: current main still clears the embedded session write-lock handle before fence reads that can throw, while this PR adds the narrow exception-safe release path, a targeted regression test, and credible before/after real lock-stack output.

## What This Changes

This PR wraps embedded prompt-release fence bookkeeping in a try/finally so the underlying session write lock is released on fence-read errors, and adds a regression test for that invariant.

## Best Possible Solution

Land the narrow try/finally fix with its regression test once normal required checks and maintainer review gates pass.

## Reproduction Assessment

Yes. Source inspection shows current main can clear heldLock before non-ENOENT fence-read errors throw, and the PR body provides before/after live-output fault injection for the same current-main failure path.

## Solution Assessment

Yes. Releasing the captured lock in a finally at the exact point heldLock is cleared is the narrowest maintainable fix; moving the behavior into the watchdog or lock manager would leave this controller invariant broken longer than necessary.
