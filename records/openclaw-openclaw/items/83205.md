## Summary

Keep this PR open. The fix direction matches the linked bug, but the patch currently fails test type checking, leaves the exported config type out of sync with the relaxed schema, and the supplied proof uses a hand-written schema replica rather than the patched OpenClaw code path.

## What This Changes

The branch makes `ModelProviderSchema.baseUrl` and `models` optional and adds regression tests for timeout-only built-in provider config plus negative validation cases.

## Best Possible Solution

Align the validator, generated/user-facing schema contract, and exported `ModelProviderConfig` type so built-in provider timeout overlays validate and remain typed, with focused regression coverage and actual config-load proof.

## Reproduction Assessment

Yes, by source inspection: current main validates raw config with `OpenClawSchema.safeParse`, and `ModelProviderSchema` requires `baseUrl` and `models` before the runtime path that consumes `timeoutSeconds` can run.

## Solution Assessment

No, not yet. Relaxing the schema is the right direction, but the maintainable fix also needs the exported config type aligned and the new test made type-clean.
