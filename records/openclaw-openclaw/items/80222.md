## Summary

Keep open: current main still gates preflight compaction only on the resolved model/config window, while the PR addresses a source-reproducible early-compaction case using established session context metadata. The diff is focused, has sufficient live proof, and I found no blocking review findings.

## What This Changes

The PR changes preflight compaction to use the larger of the resolved model/config context window and a valid persisted session `contextTokens`, with a regression test for a 149k/272k false-positive case.

## Best Possible Solution

Land the focused preflight gate fix after normal maintainer review and merge gates, preserving valid active-session context windows while keeping existing byte safeguards intact.

## Reproduction Assessment

Yes. Source inspection shows current main can compare a roughly 149k projected transcript against a 128k resolved window even when the active session entry records 272k; the PR adds a focused regression for that path.

## Solution Assessment

Yes. The PR changes the narrow threshold input, validates the persisted value, and leaves the existing compaction trigger, transcript estimation, and byte-based safeguard paths intact.
