## What This Changes

The PR changes preflight compaction gating to consider the active session's persisted context window when it is larger than the resolved model/config window, with a regression test for the long-context false-positive case.

## Best Possible Solution

Land the focused preflight threshold fix after maintainer review and merge gates, while keeping the broader Feishu large-session performance issue tracked separately.

## Reproduction Assessment

Yes, source-reproducible. Current main computes the preflight threshold from the resolved window while the session store can persist a larger active contextTokens value, so the 149k/272k scenario can cross a 128k-based threshold too early.

## Solution Assessment

Yes. The merge result changes the narrow threshold input, validates the persisted value, and preserves current runtime-provider resolution plus the existing compaction and byte-safeguard paths.
