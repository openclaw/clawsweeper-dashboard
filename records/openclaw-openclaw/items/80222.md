## Summary

Keep open for maintainer review: current main still ignores the active session's persisted context window during preflight compaction thresholding, and this focused PR addresses that path with regression coverage and sufficient live proof. No blocking review findings were found.

## What This Changes

The PR changes preflight compaction to use the larger of the resolved model/config context window and a positive persisted `SessionEntry.contextTokens`, with a regression test for the long-context false-positive case.

## Best Possible Solution

Land the focused preflight gate fix after normal maintainer review, preserving the existing compaction execution path while respecting positive active-session context windows for thresholding.

## Reproduction Assessment

Yes. Source inspection shows current main can compare about 149k projected tokens against a 128k resolved window even when the active session entry records 272k; the added regression test models that path directly.

## Solution Assessment

Yes. The patch fixes the narrow threshold input that is too small, validates the persisted session value, and leaves existing token and byte compaction triggers otherwise unchanged.
