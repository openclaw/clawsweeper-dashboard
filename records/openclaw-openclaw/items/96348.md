## Summary

Keep open for the normal merge path: the PR fixes a source-reproducible DuckDuckGo entity double-decode bug with a narrow decoder reorder and focused regression coverage, and I found no blocking correctness or security issue.

## What This Changes

The PR reorders DuckDuckGo's HTML entity decoder so `&amp;` is decoded after other entities and adds a regression test for double-encoded literal entities.

## Best Possible Solution

Land this plugin-local decoder reorder with the focused regression test after CI, leaving broader shared HTML-decoder hardening to separate work if maintainers want it.

## Reproduction Assessment

Yes. A focused call to the current DuckDuckGo testing decoder with `&amp;lt;` or `&amp;#39;` follows the source path that decodes `&amp;` before the later entity passes; I did not execute the test suite in this read-only pass.

## Solution Assessment

Yes. Moving `&amp;` to the end fixes the corrupted literal-entity case at the plugin-local decoder boundary while preserving the existing singly encoded cases covered by tests.
