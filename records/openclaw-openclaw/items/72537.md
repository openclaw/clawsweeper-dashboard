## Summary

Keep this PR open. Current main still ignores provider-level TTS timeouts on the chat/non-telephony synthesis path and the PR fixes that path narrowly, but the user-facing fix still needs the required active-version changelog entry before merge.

## What This Changes

The PR preserves OpenAI TTS provider `timeoutMs`, resolves non-telephony TTS attempt timeouts with explicit/provider/global precedence, and adds speech-core regression tests.

## Best Possible Solution

Land the timeout propagation fix after adding one active-version `CHANGELOG.md` Fixes entry and rerunning changed validation; keep the generic precedence logic in speech-core and the OpenAI field preservation in the OpenAI plugin.

## Solution Assessment

Yes, with one process fix. The proposed implementation is the narrow maintainable path because it keeps provider-owned OpenAI normalization in the plugin and applies generic explicit/provider/global precedence per non-telephony speech attempt without changing telephony behavior.
