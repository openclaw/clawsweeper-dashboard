## Summary

Keep open: the URL-download hardening is useful, but the patch adds a top-level `undici` package import to a resolver that release workflows intentionally run before dependency installation, so a clean runner would fail at module load before any package source is resolved. The supplied proof exercises mocked helper hooks and misses that no-deps workflow path.

## What This Changes

The PR rewrites package candidate `source=url` downloads to validate URL hosts and redirects, pin DNS results, enforce timeout and size limits, write atomically, and add focused tooling tests.

## Best Possible Solution

Keep the download hardening, but implement it without a top-level root dependency so the resolver still runs before dependencies are installed, then prove the real `source=url` and existing non-URL workflow shapes.

## Reproduction Assessment

Yes for the PR-introduced regression at source level: the workflows call the resolver after `install-deps: "false"`, the setup action skips `pnpm install`, and this clean checkout cannot resolve `undici`. I did not execute the PR branch because this review is read-only.

## Solution Assessment

No: URL hardening is the right direction, but a top-level package dependency is not safe for this pre-install resolver. The maintainable fix is a dependency-free download implementation or an explicit maintainer decision to change the workflow setup contract.
