## Summary

Keep this PR open: it is useful security hardening for release/package tooling, but the redirect cleanup path can still hang on a malicious redirect response body and should be fixed before merge.

## What This Changes

The branch hardens `scripts/resolve-openclaw-package-candidate.mjs` URL downloads with HTTPS and host validation, DNS/IP blocking, redirect checks, timeout and byte limits, atomic temp-file writes, and tooling tests.

## Best Possible Solution

Merge a reviewed hardening that blocks private/internal package URL targets while preserving normal public HTTPS tarball downloads and making every redirect/cleanup path bounded.

## Reproduction Assessment

Yes by source inspection: current main has the unsafe direct fetch path, and the PR's redirect branch clears the timeout before unbounded dispatcher close without cancelling the body. I did not execute tests or network repros because this was a read-only review.

## Solution Assessment

Mostly yes: hardening `downloadUrl` is the right boundary for `source=url`, but the redirect cleanup bug needs repair and maintainers should explicitly accept the stricter URL compatibility policy.
