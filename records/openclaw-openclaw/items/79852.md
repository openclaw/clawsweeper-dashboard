## What This Changes

The PR adds session-reset archive filename timestamp parsing to memory-core temporal decay and regression tests for missing-file reset archives and stale-versus-fresh reset archive scoring.

## Best Possible Solution

Land the scoped parser and regression tests after targeted memory-core validation, with a separate follow-up if deleted archive parity is desired.

## Reproduction Assessment

Yes from source inspection: current main resolves non-dated session results through `workspaceDir + indexed path`, while session archive hits are stored as logical `sessions/<agent>/<basename>.jsonl.reset.*` paths. I did not run a live deployment reproduction in this read-only review.

## Solution Assessment

Yes, parsing the reset archive timestamp from the filename before the filesystem fallback is a narrow maintainable fix for the proven path mismatch. Reusing the existing archive timestamp parser or adding deleted-archive parity would be reasonable follow-up polish, not a blocking correctness issue for this PR.
