## Summary

Keep open for maintainer review: the reported bug is source-reproducible on current main, the PR targets the implicated memory-core path, and I found no blocking correctness or security findings.

## What This Changes

The PR adds session-reset archive filename timestamp parsing before temporal decay's filesystem fallback and covers it with two regression tests.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after targeted memory-core validation; handle deleted-archive parity separately only if maintainers want that scope.

## Reproduction Assessment

Yes. Source inspection shows current main decays dated memory paths but falls back to `fs.stat(workspaceDir + logical sessions path)` for reset archives, so missing logical session paths skip decay.

## Solution Assessment

Yes. Parsing the reset archive timestamp from the filename before `fs.stat` is a narrow fix for the indexed-path versus physical-path mismatch; exposing the existing archive timestamp helper through a plugin-facing surface could reduce duplication later but is not required for correctness.
