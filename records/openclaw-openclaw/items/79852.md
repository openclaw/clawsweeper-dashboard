## Summary

Keep this PR open: the reported temporal-decay bug is valid by source inspection, but the patch is not merge-ready because it creates a reset-only timestamp parser that diverges from OpenClaw's existing reset/deleted session archive contract.

## What This Changes

The PR adds memory-core parsing for `.jsonl.reset.<timestamp>Z` filenames before temporal decay falls back to filesystem `stat`, plus reset-archive scoring tests.

## Best Possible Solution

Reuse the canonical reset/deleted session archive timestamp contract through the memory-host SDK boundary, cover sibling archive cases in tests, and then refresh the branch against current main.

## Reproduction Assessment

Yes by source inspection: indexed session archives use logical `sessions/<agent>/<basename>` paths, while current temporal decay stats non-memory paths under `workspaceDir` and skips decay when that stat fails. I did not run a live archive corpus reproduction in this read-only review.

## Solution Assessment

No. Parsing archive timestamps before `fs.stat` is the right fix shape, but this branch should reuse the existing reset/deleted archive timestamp contract instead of adding a duplicate reset-only regex.
