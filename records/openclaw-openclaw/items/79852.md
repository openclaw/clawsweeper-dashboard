## Summary

Keep open: current main still skips temporal decay for logical reset archive result paths, and this PR is a focused bug fix with no blocking correctness or security findings found in review. The remaining action is maintainer review/merge, including reconciling the proof-gate label/check mismatch.

## What This Changes

The PR parses reset session archive timestamps from filenames before the memory-core temporal decay filesystem fallback and adds regression tests for logical archive paths that do not exist under the workspace directory.

## Best Possible Solution

Land the narrow reset-archive timestamp parser and regression tests after maintainer review, then handle deleted-archive parity separately if maintainers want the broader archive policy covered now.

## Reproduction Assessment

Yes, from source inspection. Current main indexes reset archives as logical sessions/<agent>/<basename> paths, then temporal decay resolves non-dated-memory paths under workspaceDir and returns null on failed fs.stat, leaving those archive scores unchanged.

## Solution Assessment

Yes. Parsing the reset archive timestamp before the filesystem fallback is the narrowest maintainable fix for the proven path mismatch; deleted-archive parity can be a separate scope decision.
