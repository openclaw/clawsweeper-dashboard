## Summary

Keep open: current main still leaves session archive hits without temporal decay when their logical index path cannot be statted, but this branch is not the best fix yet because it adds a reset-only timestamp parser instead of reusing the existing reset/deleted archive contract.

## What This Changes

The branch adds a memory-core temporal-decay parser for `.jsonl.reset.<timestamp>Z` session archive filenames and regression tests for reset archive scoring and ranking.

## Best Possible Solution

Reuse the canonical reset/deleted session archive timestamp parser through the memory-host SDK boundary, cover the sibling archive cases in tests, then refresh the branch against current main.

## Reproduction Assessment

Yes by source inspection: indexed session archives use logical `sessions/<agent>/<basename>` paths, while current temporal decay stats non-memory paths under `workspaceDir` and skips decay when that stat fails. I did not run a live archive corpus reproduction in this read-only review.

## Solution Assessment

No. Parsing archive timestamps before `fs.stat` is the right fix shape, but the PR should reuse the existing reset/deleted archive timestamp contract instead of adding a duplicate reset-only parser.
