## Summary

Keep open: current main still has the session-archive temporal-decay gap, and this active focused PR targets the implicated path with no blocking review findings found.

## What This Changes

The branch adds reset-archive timestamp parsing to memory-core temporal decay and regression tests for missing logical archive paths and stale-versus-fresh reset ranking.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after targeted validation, with deleted-archive parity handled separately if desired.

## Reproduction Assessment

Yes. A current-main result like `sessions/main/x.jsonl.reset.2026-01-11T00-00-00.000Z` with no matching file under `workspaceDir` reaches the `fs.stat(path.resolve(workspaceDir, filePath))` fallback, returns null on ENOENT, and skips decay.

## Solution Assessment

Yes. Parsing the reset archive timestamp before the `fs.stat` fallback is the narrow fix for the logical-indexed-path versus physical-file-path mismatch while preserving live session mtime behavior.
