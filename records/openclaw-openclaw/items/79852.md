## Summary

Keep open. Current main still has the source-visible reset-archive decay gap, and this PR is a focused bug fix with no blocking review findings; the remaining step is maintainer review and validation, not cleanup closure.

## What This Changes

The PR parses `.jsonl.reset.<timestamp>Z` session archive timestamps before the filesystem fallback and adds regression tests for decay and ranking without requiring `fs.stat`.

## Best Possible Solution

Land the narrow reset-archive timestamp parser and regression tests after maintainer validation, then handle deleted-archive parity only if maintainers want that broader scope.

## Reproduction Assessment

Yes, from source inspection: current main sends logical `sessions/<agent>/*.jsonl.reset.*Z` result paths to a workspace-relative `fs.stat`, and a failed stat returns null so temporal decay skips the archive score.

## Solution Assessment

Yes, parsing the reset archive timestamp before the filesystem fallback is the narrowest maintainable fix for the proven path mismatch; moving a shared archive timestamp helper across package boundaries can remain a later cleanup.
