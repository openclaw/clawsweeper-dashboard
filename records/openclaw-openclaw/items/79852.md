## Summary

Keep open: the reported temporal-decay bug is real and the PR remains useful, but the patch should not land as written because it adds a second, reset-only archive timestamp parser while current main already has a broader reset/deleted archive timestamp contract.

## What This Changes

The PR adds a session-reset archive timestamp parser inside memory-core temporal decay and covers it with two regression tests.

## Best Possible Solution

Use one shared archive timestamp contract for all usage-counted session archive paths before filesystem stat, then refresh the branch against current main.

## Reproduction Assessment

Yes by source inspection: session archive hits are indexed as logical `sessions/<agent>/<basename>` paths, while current temporal decay stats non-memory paths under `workspaceDir` and skips decay when that stat misses. I did not run a live archive corpus in this read-only review.

## Solution Assessment

No. Parsing archive timestamps before `fs.stat` is the right layer, but this PR should reuse the existing reset/deleted optional-millisecond parser instead of adding a duplicate reset-only regex.
