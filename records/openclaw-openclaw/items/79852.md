## Summary

Keep open for maintainer review. Current main still skips temporal decay for logical session-reset archive paths, while this PR is a focused fix with regression coverage, sufficient after-fix empirical proof in the body, and no blocking review findings from the diff.

## What This Changes

The PR parses `.jsonl.reset.<timestamp>Z` session archive timestamps before the filesystem fallback and adds temporal-decay regression tests for reset archive results.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after targeted validation, then track deleted-archive parity separately if maintainers want the same behavior there.

## Reproduction Assessment

Yes from source inspection: current main stores session archive hits as logical `sessions/<agent>/<basename>` paths, while temporal decay stats them under `workspaceDir` and skips decay when that path does not exist. I did not run a live deployment or tests in this read-only review.

## Solution Assessment

Yes. Parsing the reset archive timestamp before the filesystem fallback is the narrowest maintainable fix for the proven path mismatch; reusing a shared archive timestamp helper or extending deleted-archive parity can be follow-up work, not a blocker.
