## Summary

## What This Changes

The PR parses `.jsonl.reset.<timestamp>Z` archive timestamps before the filesystem fallback and adds temporal-decay regression tests.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after targeted validation, then consider deleted-archive parity separately if it is in scope.

## Solution Assessment

Yes. Parsing the reset archive timestamp before `fs.stat` is the narrowest maintainable fix for the proven indexed-path mismatch; broader archive timestamp sharing or deleted-archive parity can be follow-ups.
