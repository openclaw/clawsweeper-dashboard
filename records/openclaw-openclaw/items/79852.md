## Summary

Keep open: this targets a real source-reproducible memory ranking bug and includes useful after-fix empirical proof, but the patch duplicates the session archive timestamp contract with a reset-only parser that misses other accepted archived session paths.

## What This Changes

The branch parses session-reset archive timestamps before temporal decay's fs.stat fallback and adds regression tests for reset archive decay and ranking.

## Best Possible Solution

Use one canonical session archive timestamp contract for temporal decay, exposed through the memory-host SDK boundary, so reset and deleted archives decay consistently without plugin deep imports.

## Reproduction Assessment

Yes, from source inspection: logical session archive paths are indexed under sessions/<agent>/..., but current temporal decay resolves non-memory timestamps relative to workspaceDir and skips decay when that stat fails.

## Solution Assessment

No, not yet: parsing before fs.stat is the right fix shape, but the patch should reuse the canonical archive timestamp contract instead of adding a reset-only duplicate parser.
