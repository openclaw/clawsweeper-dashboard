## Summary

Keep open: this PR addresses a real current-main memory-core scoring bug with a narrow implementation and no blocking review findings; it needs normal maintainer validation and proof-gate reconciliation rather than cleanup closure.

## What This Changes

Adds timestamp parsing for `.jsonl.reset.<timestamp>Z` session archives before the filesystem mtime fallback, plus regression tests proving temporal decay applies without touching disk.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after targeted validation; handle deleted-archive parity as a separate follow-up only if maintainers decide it is in scope.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: a logical `sessions/main/*.jsonl.reset.*Z` result path on current main falls through to a workspace-relative `fs.stat`, fails when the file is actually under the agent sessions directory, and leaves the score unchanged.

## Solution Assessment

Is this the best way to solve the issue? Yes; parsing the reset archive timestamp before the filesystem fallback is the narrowest maintainable fix for the proven path mismatch, while reuse of the shared archive timestamp helper or deleted-archive parity can be follow-ups.
