## What This Changes

The PR parses `.jsonl.reset.<timestamp>Z` session archive filenames in memory-core temporal decay and adds regression tests for decaying/ranking reset archives without filesystem access.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after targeted validation, then handle deleted-archive parity separately if maintainers decide it is in scope.

## Reproduction Assessment

Yes from source inspection: current main resolves session archive result paths through `workspaceDir`, while indexed paths are logical `sessions/<agent>/<basename>.jsonl.reset.*` paths. I did not run a live deployment or tests in this read-only review.

## Solution Assessment

Yes, parsing the reset archive timestamp before the filesystem fallback is the narrowest maintainable fix for the proven path mismatch. Reusing the existing archive timestamp helper or extending parity to deleted archives would be reasonable follow-ups, not blockers for the reset bug.
