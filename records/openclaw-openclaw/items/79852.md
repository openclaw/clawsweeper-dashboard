## Summary

Keep open: this PR targets a real source-reproducible memory ranking bug, but the patch is too narrow because it adds a reset-only timestamp parser instead of reusing the existing reset/deleted session archive contract.

## What This Changes

The PR adds memory-core parsing for `.jsonl.reset.<timestamp>Z` archive filenames so temporal decay can apply to session-reset archive search results.

## Best Possible Solution

Reuse or expose the shared session archive timestamp contract, apply it to usage-counted reset and deleted archive paths before `fs.stat`, and preserve the live `.jsonl` mtime fallback.

## Reproduction Assessment

Yes by source inspection: current main indexes archive chunks under logical `sessions/<agent>/<basename>` paths, while temporal decay stats non-memory paths under `workspaceDir` and skips decay when that stat misses. I did not run tests or a live corpus because this review was required to keep the checkout read-only.

## Solution Assessment

No. Parsing archive timestamps before the filesystem fallback is the right layer, but the PR should reuse the existing reset/deleted archive parser instead of adding a duplicate reset-only regex.
