## Summary

Keep open: the reported temporal-decay bug is real and the branch has useful proof, but the patch duplicates the session archive timestamp contract and only covers `.reset` filenames with milliseconds, leaving accepted deleted and optional-millisecond archive paths inconsistent.

## What This Changes

The PR adds filename timestamp parsing for session-reset archives in memory-core temporal decay and two regression tests for reset archive scoring without filesystem access.

## Best Possible Solution

Route temporal decay through the shared archive timestamp parser via the memory-host facade, apply it to reset/deleted archives before `fs.stat`, preserve live `.jsonl` mtime fallback, and refresh the branch.

## Reproduction Assessment

Yes, at source level. A focused repro can pass a logical `sessions/main/*.jsonl.reset.*` hit with no matching file under `workspaceDir`; current main resolves it under the workspace, catches `fs.stat`, and skips decay.

## Solution Assessment

No. Parsing archive timestamps before `fs.stat` is the right layer, but this implementation duplicates an existing archive contract and misses sibling archive forms that current main already accepts.
