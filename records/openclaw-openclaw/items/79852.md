## Summary

## What This Changes

The branch adds a memory-core parser for `.jsonl.reset` timestamp suffixes before temporal decay falls back to `fs.stat`, plus two temporal-decay regression tests.

## Best Possible Solution

Reuse the shared reset/deleted session archive timestamp parser through the memory-host SDK boundary before `fs.stat`, then cover reset, deleted, no-millisecond, invalid archive, and live `.jsonl` fallback cases.

## Reproduction Assessment

Yes by source inspection: indexed session archive hits use logical session paths, while current temporal decay stats non-memory paths under `workspaceDir` and skips decay when that stat fails. I did not run a live archive corpus reproduction in this read-only review.

## Solution Assessment

No. Parsing archive timestamps before `fs.stat` is the right layer, but this branch should reuse the existing reset/deleted archive timestamp contract instead of adding a duplicate reset-only regex.
