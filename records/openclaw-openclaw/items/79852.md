## What This Changes

The PR adds session-reset archive timestamp parsing to memory-core temporal decay and covers nonexistent logical archive paths with regression tests.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after maintainer validation, then handle deleted-archive parity separately if desired.

## Reproduction Assessment

Yes from source inspection: current main resolves logical sessions/<agent>/*.jsonl.reset.* result paths under workspaceDir, so missing files make fs.stat fail and temporal decay returns the original score. I did not run live or local tests in this read-only review.

## Solution Assessment

Yes, parsing the reset archive timestamp before the filesystem fallback is the narrowest maintainable fix for the proven path mismatch. Extending the same idea to deleted archives would be a follow-up, not a blocker for this reset-specific fix.
