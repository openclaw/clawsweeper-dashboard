## Summary

Keep open: current main still misses reset archive timestamps, and this is a focused, source-reproducible bug fix with no blocking review findings. The remaining action is normal maintainer review and validation, not cleanup close.

## What This Changes

The PR updates memory-core temporal decay to parse timestamps from `.jsonl.reset.<timestamp>Z` session archive paths before falling back to filesystem metadata, with regression tests for archive decay and ranking.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after maintainer validation, then handle deleted-archive parity separately if maintainers want that behavior too.

## Reproduction Assessment

Yes. Source inspection shows current main resolves logical `sessions/<agent>/<basename>.jsonl.reset.*` result paths through `workspaceDir`, so missing archive files make temporal decay skip those entries.

## Solution Assessment

Yes. Parsing the reset archive timestamp before the filesystem fallback is the narrowest maintainable fix for the proven path mismatch; shared archive timestamp helper parity would be a reasonable cleanup, not a blocker.
