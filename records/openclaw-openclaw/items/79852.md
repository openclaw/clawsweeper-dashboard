## Summary

This is an active, focused bug-fix PR and current main still has the temporal-decay gap for logical session-reset archive paths. The patch looks correct for the stated reset-archive behavior, with real behavior proof in the PR body and no blocking findings from this review.

## What This Changes

The PR adds session-reset archive timestamp parsing to memory-core temporal decay and regression tests for missing-file archive decay and stale-versus-fresh reset archive scoring.

## Best Possible Solution

Land the narrow reset-archive parser and regression tests after targeted validation, with a separate maintainer call on deleted-archive timestamp parity if desired.

## Reproduction Assessment

Yes from source inspection: current main resolves non-dated session results through `workspaceDir + indexed path`, while reset archive hits are stored as logical `sessions/<agent>/<basename>.jsonl.reset.*` paths that may not exist under the workspace. I did not run a live failing deployment in this read-only review.

## Solution Assessment

Yes for the stated reset-archive bug: parsing the reset filename before the filesystem fallback is a narrow fix for the proven path mismatch. Reusing the existing archive timestamp helper or adding deleted-archive parity would reduce duplication, but that is not a blocking correctness issue for this PR.
