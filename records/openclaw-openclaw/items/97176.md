## Summary

Keep open: current main and the latest release still leave Bedrock Claude adaptive-thinking requests on a path that returns the base maxTokens unchanged, while the sibling non-adaptive thinking path expands against the resolved model maxTokens. The report is a concrete provider-runtime bug with a narrow fix path.

## What This Changes

The issue asks for Bedrock adaptive-thinking Claude requests to use the resolved model or agent output-token budget instead of staying capped at 4096 tokens.

## Best Possible Solution

Fix the Bedrock provider stream-option resolution so adaptive Claude and Fable requests honor the resolved model/output-token contract, without adding a new broad config key unless tests prove the existing params/model config path is insufficient.

## Reproduction Assessment

Yes, source-level. Current main returns the adaptive Bedrock Claude options without applying `model.maxTokens`, and the request builder forwards only that resulting `options.maxTokens`; I did not run a live AWS credential repro.

## Solution Assessment
