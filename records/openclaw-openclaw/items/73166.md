## Summary

Keep open. Current main still has the reported cron timer recovery gap, and the issue is paired with open implementation PR #73355 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Prevent the Gateway cron scheduler from silently stopping after long uptime by recovering when the scheduled timer chain misses or a timer tick rejects.

## Best Possible Solution

Review and land PR #73355 or an equivalent focused cron-service fix that re-arms after rejected timer ticks, handles any accepted watchdog lifecycle cleanly, and adds regression coverage for the stalled-chain path.

## Reproduction Assessment

Unclear. The report has detailed field evidence, but a high-confidence live reproduction would require the multi-day macOS launchd uptime scenario; a focused unit test can reproduce the rejected `onTimer` catch-path gap by forcing `onTimer` to reject.

## Solution Assessment

Unclear. Re-arming in the catch path is the narrowest maintainable fix for the verified source gap, while the independent watchdog portion needs maintainer review for lifecycle, false positives, and necessity.
