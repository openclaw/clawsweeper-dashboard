## Summary

Keep open. Current main still has the cron timer catch-path recovery gap, the latest shipped release has the same behavior, and open PR #73355 is a same-author implementation candidate with closing syntax for this issue.

## What This Changes

The issue reports that scheduled cron jobs can silently stop firing after several days of Gateway uptime while manual cron execution and Gateway restart still work.

## Best Possible Solution

Land PR #73355 or an equivalent narrow cron-service fix that makes rejected timer ticks self-recover, handles any accepted watchdog lifecycle cleanly, adds focused regression coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

## Solution Assessment

Unclear. Re-arming in the catch path is the narrowest maintainable fix for the verified source gap, while the independent watchdog portion needs maintainer review for lifecycle and false-positive behavior.
