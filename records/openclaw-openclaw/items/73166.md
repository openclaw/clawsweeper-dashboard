## Summary

Keep open. Current main still has the cron timer rejection recovery gap, and open same-author PR #73355 targets this issue with closing syntax, so this should remain tracked until that PR or an equivalent focused fix lands.

## What This Changes

The issue reports that Gateway cron jobs can stop firing after multiple days of uptime while manual cron runs and a Gateway restart still work.

## Best Possible Solution

Review and land PR #73355 or an equivalent focused scheduler recovery fix with direct catch-handler re-arm coverage and lifecycle-safe watchdog tests.

## Reproduction Assessment

Unclear. The live multi-day macOS launchd scenario is not deterministic, but current main has a focused unit-testable path: make `onTimer` reject from the timer callback and observe that the log-only catch leaves no replacement timer.

## Solution Assessment

Unclear. Re-arming the rejected tick catch handlers is the narrowest repair; a watchdog may also be acceptable if its start/stop lifecycle is covered and maintainer review agrees.
