## Summary

Keep open. Current main and the latest stable release still have cron timer catch handlers that only log failures, no independent cron watchdog lifecycle, and an open same-author implementation PR already references this issue with closing syntax.

## What This Changes

The issue reports that Gateway cron jobs can silently stop firing automatically after multiple days of uptime while manual cron execution and gateway restart still work.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/73355 or a narrower equivalent lands with direct catch-handler recovery coverage and lifecycle-safe watchdog handling if retained.

## Reproduction Assessment

No, not for the full multi-day macOS launchd stall; that remains timing-dependent. Source inspection does show a focused unit-testable path where a rejected timer tick logs without installing a replacement timer.

## Solution Assessment

Unclear. Re-arming rejected timer ticks is the narrowest maintainable fix; the watchdog may be acceptable if the linked PR proves catch recovery directly and keeps watchdog start/stop behavior covered.
