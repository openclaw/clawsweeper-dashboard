## Summary

Keep open. Current main still logs rejected cron timer callbacks without re-arming, has no independent cron watchdog lifecycle, and open PR #73355 is the active same-author implementation path with closing syntax for this issue.

## What This Changes

Cron should keep firing scheduled jobs after long Gateway uptime instead of leaving a stale `nextWakeAtMs` until restart.

## Best Possible Solution

Review and land PR #73355 or a narrower equivalent that re-arms rejected timer ticks and adds focused regression coverage for the timer-chain recovery path.

## Reproduction Assessment

No, not for the full multi-day macOS launchd stall; that remains timing-dependent. Source inspection gives a focused path by forcing `onTimer` to reject from the timer callback and observing that the catch handler currently only logs.

## Solution Assessment

Unclear. Re-arming rejected timer ticks is the narrowest maintainable fix; the separate watchdog may be acceptable if its lifecycle and false-positive behavior are covered.
