## Summary

Keep open. Current main still has the cron timer rejected-tick recovery gap, and open PR #73355 already targets this issue with closing syntax, so the issue should remain tracked until that PR or an equivalent fix lands.

## What This Changes

The issue reports Gateway cron jobs missing scheduled ticks after about 2.5 days of uptime while manual runs and a Gateway restart restore scheduling.

## Best Possible Solution

Land a focused cron scheduler recovery fix that re-arms rejected timer ticks and, if the watchdog remains, includes start/stop lifecycle coverage.

## Reproduction Assessment

Unclear. The live report needs a multi-day macOS launchd uptime scenario, but current code gives a focused unit-testable path by making `onTimer` reject and confirming the catch handlers leave no replacement timer.

## Solution Assessment

Unclear. Re-arming the catch handlers is the narrowest maintainable repair; a separate watchdog may be acceptable only with lifecycle-safe tests and maintainer review.
