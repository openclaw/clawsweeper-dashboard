## Summary

Keep open. Current main still has the cron timer rejected-tick recovery gap, and open PR #73355 is already paired to this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports Gateway cron jobs missing scheduled automatic ticks after about 2.5 days of uptime while manual runs and a Gateway restart restore scheduling.

## Best Possible Solution

Land PR #73355 or a narrow equivalent that re-arms rejected cron timer ticks, keeps any watchdog lifecycle-safe, and adds focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Unclear. The field report is detailed but requires multi-day macOS launchd uptime; current main gives a focused unit-testable path by forcing `onTimer` rejection and confirming both catch handlers leave no replacement timer.

## Solution Assessment

Unclear. Re-arming the catch handlers is the narrowest maintainable repair; a separate watchdog may be acceptable if lifecycle-safe tests prove it does not outlive the cron service.
