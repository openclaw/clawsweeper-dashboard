## Summary

Keep open. Current main intentionally gates `api.runtime.state.openKeyedStore` to bundled plugins, and exposing durable keyed state to community plugins is a new Plugin SDK/security policy feature rather than an implemented or ClawHub-only request.

## What This Changes

Expose `api.runtime.state.openKeyedStore` to community plugins so they can use durable per-plugin keyed state without reimplementing the store.

## Best Possible Solution

Add a deliberate public SDK contract for third-party durable keyed state, including trust policy, limits, docs, and regression tests; keep the bundled-only guard until that design lands.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current behavior check because the runtime proxy rejects non-bundled plugin origins and a test asserts that rejection for a workspace plugin.

## Solution Assessment

Unclear until maintainer design. A direct gate removal is mechanically small, but durable community plugin state is a public SDK and security boundary, so the maintainable solution should include an explicit trust policy with docs and tests.
