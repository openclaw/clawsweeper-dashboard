## Summary

Keep open. Current main intentionally gates durable keyed state to bundled plugins, and exposing it to community plugins is a new public SDK/security-policy decision rather than an already-implemented or ClawHub-only cleanup.

## What This Changes

Request to make `api.runtime.state.openKeyedStore` available to community plugins for restart-safe per-plugin keyed state.

## Best Possible Solution

Promote keyed state to a supported third-party Plugin SDK feature with explicit trust and limit policy, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current behavior check because the runtime proxy rejects non-bundled plugin origins and a test asserts that rejection for a workspace plugin.

## Solution Assessment

Unclear until maintainers choose the public contract. A direct origin-gate removal is mechanically small, but durable community plugin state should be accepted through an explicit SDK/security policy with matching docs and tests.
