## Summary

Keep open. Current main intentionally gates `api.runtime.state.openKeyedStore` to bundled plugins, and exposing durable keyed state to community plugins is a new Plugin SDK/security policy feature rather than an implemented or ClawHub-only cleanup.

## What This Changes

The issue requests making `api.runtime.state.openKeyedStore` available to community plugins instead of limiting it to bundled plugins.

## Best Possible Solution

Define and ship a public third-party keyed-state contract with trust policy, limits, docs, and regression tests before removing or broadening the bundled-only guard.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current behavior check because the runtime proxy rejects workspace plugins and a test asserts that rejection.

## Solution Assessment

Unclear until maintainer design. A direct gate removal is mechanically narrow, but durable community plugin state crosses the public SDK and trust boundary, so the maintainable solution should include explicit policy, limits, docs, and tests.
