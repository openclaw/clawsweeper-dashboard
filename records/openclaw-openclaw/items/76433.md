## Summary

Keep open. Current main and the latest release still intentionally gate `api.runtime.state.openKeyedStore` to bundled plugins, so this is a real Plugin SDK feature and trust-policy request rather than an obsolete cleanup item.

## What This Changes

Expose `api.runtime.state.openKeyedStore` to community plugins so they can use OpenClaw's durable keyed state API instead of reimplementing it.

## Best Possible Solution

Add a deliberate public Plugin SDK contract for third-party durable keyed state, with documented trust policy, quotas, compatibility expectations, and regression coverage before lifting the current guard.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence behavior check because current main rejects workspace-origin plugins and a runtime test asserts that rejection.

## Solution Assessment

Unclear until maintainer design. Directly removing the origin check is mechanically small, but durable persisted state for unbundled in-process plugins is a public SDK and trust-boundary change that needs policy, docs, and tests.
