## Summary

Keep open: current main and v2026.6.8 still gate keyed plugin state to bundled or trusted official installs, while the related open issue covers a narrower self-hosted official-channel provenance gap rather than broad community-plugin SDK access.

## What This Changes

The issue requests exposing `api.runtime.state.openKeyedStore` to community plugins instead of restricting it to bundled or trusted plugins.

## Best Possible Solution

Define a maintainer-approved keyed-state capability or trust contract for community plugins, with explicit opt-in, quotas, documentation, parser coverage, and safe defaults.
