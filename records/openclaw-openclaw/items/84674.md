## Summary

Keep open. Current main and the latest release still keep the Telegram live-owner predicate that can classify a fresh different-processId claim with an existing reused PID as live-owned, and the linked implementation PR is open and unmerged.

## What This Changes

The issue reports that Telegram isolated polling ingress can leave a claimed/processing update behind after gateway recreation, blocking later Telegram messages until manual spool or queue intervention.

## Best Possible Solution

Review and land the linked Telegram recovery PR or an equivalent maintainer-owned replacement that covers current SQLite-backed ingress claims while preserving protection for genuinely active peer claims.
