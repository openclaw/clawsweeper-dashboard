## Summary

Keep this PR open for revision: the linked bug is valid, but the patch deletes whitespace everywhere in the URL and would silently change currently valid path/query URLs; the supplied proof is a mocked Vitest run rather than real behavior proof.

## What This Changes

The PR changes the `web_fetch` runtime to remove all whitespace from URL strings before parsing, fetching, provider fallback, and returned payload, with a mock regression test for a space after `https://`.

## Best Possible Solution

Normalize only the malformed scheme/authority whitespace needed for this issue, preserve URL parser semantics for path/query values, keep the returned URL contract explicit, and prove it with both regression tests and a real `web_fetch` run.

## Reproduction Assessment

Yes from source rather than a full tool run: current main passes raw `params.url` to `new URL`, and Node rejects `https:// docs.example.com/page`. The linked issue's failure is source-reproducible.

## Solution Assessment

No. Deleting all whitespace is too broad because Node already accepts and percent-encodes path/query spaces, so the best fix is narrower normalization plus coverage that valid spaces survive.
