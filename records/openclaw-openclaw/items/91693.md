## Summary

This PR addresses a reproducible web_fetch failure, but the current implementation deletes meaningful path/query whitespace after an authority parse error and changes the established raw URL result contract. The supplied proof also bypasses the OpenClaw production path, so the PR needs a narrow code repair and real web_fetch proof before merge.

## What This Changes

The PR adds fallback parsing for whitespace-corrupted web_fetch URLs, passes the parsed href through fetch and provider paths, changes returned URL metadata, adds regression tests, and commits a standalone proof script.

## Best Possible Solution

Repair only whitespace at the HTTP(S) scheme/authority boundary, let the URL parser percent-encode path and query spaces, preserve the caller-supplied URL in result metadata, and verify the actual guarded OpenClaw web_fetch path.

## Reproduction Assessment

Yes, from source inspection. Current main directly parses the raw argument, so `https:// docs.example.com/page` deterministically reaches the invalid-URL error, although no current-main runtime test was executed during this read-only review.

## Solution Assessment

No. Retrying after deleting whitespace from the entire URL is broader than the reported authority typo, and canonicalizing the returned URL is an unrelated contract change; a boundary-specific repair is safer.

Full review comments:

- **[P1] Limit whitespace recovery to the malformed authority:** `src/agents/tools/web-fetch.ts:450`
  - body: When the initial parse fails, `replace(/\s+/g, "")` removes whitespace from the entire input. For example, `https:// example.com/a b?q=hello world` becomes `/ab?q=helloworld`, so fixing the authority typo silently changes the requested resource. Repair only the scheme/authority whitespace and let `URL` percent-encode path/query spaces.
  - confidence: 0.98
- **[P1] Preserve the raw requested URL in tool results:** `src/agents/tools/web-fetch.ts:625`
  - body: This changes the established `url` field from `params.url`, which current main explicitly keeps raw for tool chaining, to `normalizedUrl` for every request. Even ordinary inputs can be canonicalized by `URL.href`; use the normalized value for fetching while retaining the caller-supplied value in result metadata.
  - confidence: 0.96

  - body: Repository policy requires screenshots, videos, logs, and proof assets to be attached outside product branches. This 128-line script also reimplements the parser instead of exercising `createWebFetchTool`, so it can drift while providing no production-path regression protection.
