## Summary

The reported current-main failure is valid and the parse-first recovery direction is useful, but this branch is not merge-ready: its fallback deletes meaningful path/query whitespace, its direct-fetch payload breaks the existing raw URL contract, and the supplied live proof bypasses the actual tool runtime.

## What This Changes

The PR adds fallback URL parsing that removes whitespace after initial parsing fails, propagates the parsed URL through web-fetch, adds two regression tests, and commits a standalone live-proof script.

## Best Possible Solution

Narrow recovery to the reported HTTP(S) scheme/authority spacing typo, preserve standard path/query percent-encoding, retain the raw requested URL in returned payloads, and prove the repaired behavior through the actual web-fetch tool runtime.

## Reproduction Assessment

Yes. Current main directly constructs `new URL(params.url)`, so a tool argument with whitespace after the HTTP(S) scheme reliably fails before guarded fetch.

## Solution Assessment

No. Parse fallback is the right boundary, but deleting whitespace throughout the URL and canonicalizing the returned raw URL are broader than the reported bug and change existing semantics.

Full review comments:

- **[P1] Preserve path and query whitespace during recovery:** `src/agents/tools/web-fetch.ts:448`
  - body: Do not remove whitespace from the entire URL. If the authority typo and a legitimate spaced path/query occur together, this fallback changes the destination—for example, `https:// example.com/a b?q=hello world` becomes `/ab?q=helloworld` instead of `/a%20b?q=hello%20world`. Restrict recovery to the malformed scheme/authority portion and add this combined case to the regression tests.
  - confidence: 0.99
- **[P1] Keep the raw requested URL in the result payload:** `src/agents/tools/web-fetch.ts:625`
  - body: Restore `url: params.url`. Current main explicitly preserves this field for tool chaining, and provider fallback still returns the raw requested URL; canonicalizing only the direct-fetch result makes output depend on the extraction path and silently changes valid inputs. The corrected href should be used as the fetch target, while `finalUrl` continues to describe the fetched location.
  - confidence: 0.98
- **[P2] Remove the branch-only proof script:** `proof-91693.mjs:1`
  - body: Remove this repository-root proof artifact before merge. It duplicates the parser rather than importing the production implementation, directly calls `fetch`, and adds a second untested version of the behavior; real proof belongs in the PR body or an external artifact, while durable regression coverage belongs in the colocated tests.
  - confidence: 0.93
