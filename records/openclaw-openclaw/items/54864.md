## Summary

Keep open. Current main has the session-id isolation groundwork from the merged routing fix, but the requested `openclaw agent --new-session` flag, typed option plumbing, CLI-owned UUID generation, JSON return contract, docs, and tests are still absent.

## What This Changes

Add an `openclaw agent --new-session` flag that generates a fresh session ID for the invocation and returns it in JSON output.

## Best Possible Solution

Implement this as a narrow CLI-owned session selection feature: add `--new-session`, reject it with `--session-id`, generate one effective UUID before gateway/local dispatch, preserve default main-session reuse, expose the generated id in JSON output, and update focused tests, docs, and changelog.
