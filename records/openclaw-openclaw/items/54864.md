## Summary

Keep open. ## What This Changes

The issue requests a `--new-session` flag for `openclaw agent` that generates a fresh session id internally and returns it in JSON output.

## Best Possible Solution

Choose the fresh-session CLI contract alongside https://github.com/openclaw/openclaw/issues/71417, then implement an additive backward-compatible flag through parser, local execution, Gateway protocol/handler, JSON metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: the parser, option type, Gateway dispatch, protocol schema, docs, and release tag still lack `--new-session` plumbing.

## Solution Assessment

Unclear pending maintainer UX direction. Generating one CLI-owned session id and routing it through existing `sessionId` handling is narrow, but the final flag name, alias, banner, suppression, and JSON contract should be decided with the broader resume-UX issue first.
