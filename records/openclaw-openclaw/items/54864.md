## Summary

Keep open. Current main and v2026.6.6 still lack the requested `openclaw agent --new-session` flag and top-level generated-session JSON contract; the closed implementation PR is unmerged, and https://github.com/openclaw/openclaw/issues/71417 overlaps but does not fully replace this narrower generated-session request.

## What This Changes

The issue requests a new `--new-session` flag for `openclaw agent` that generates a fresh session ID and returns it in JSON output so scripted callers can reconnect later.

## Best Possible Solution

Define one fresh-session CLI contract across this issue and https://github.com/openclaw/openclaw/issues/71417, then implement the chosen flag/output behavior through CLI parsing, gateway/local dispatch, docs, and focused tests.

## Reproduction Assessment

Yes for the source-level gap: current main and v2026.6.6 list `--session-id` and `--session-key`, but not `--new-session`. I did not run the CLI because this review is read-only, and the gap is visible in source and docs.

## Solution Assessment

Unclear as a final product direction. A native flag is a plausible narrow ergonomics improvement, but maintainers should decide it alongside the broader resume UX so OpenClaw does not grow overlapping session controls.
