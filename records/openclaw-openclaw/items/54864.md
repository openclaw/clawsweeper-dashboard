## Summary

Keep open. Current main has the session-id isolation groundwork, but it still does not expose `openclaw agent --new-session`, does not carry a `newSession` option through the CLI path, and does not add a top-level generated `sessionId` to JSON output. The related #69108 attempted this exact feature, but it is closed and unmerged, so it does not supersede the remaining work.

## What This Changes

Add an `openclaw agent --new-session` flag that generates a fresh session id internally, uses it for the invocation, and returns it in JSON output for later reconnects.

## Best Possible Solution

Implement this as a narrow core CLI feature: add `--new-session`, generate one effective `crypto.randomUUID()` before gateway/local routing, reject it with explicit `--session-id`, feed the effective id into gateway and embedded runs, return a top-level `sessionId` for JSON output, and keep the existing default `--agent` main-session reuse when the flag is absent.
