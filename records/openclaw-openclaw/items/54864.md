## Summary

Keep open. Current main and v2026.5.19 still lack the requested `--new-session` flag and generated-session JSON contract; the existing caller-generated `--session-id` path is now better covered, but this additive CLI UX still needs maintainer direction alongside the broader session-resume discussion.

## What This Changes

Request adds an `openclaw agent --new-session` flag that generates a fresh session id internally, uses it for the invocation, and returns that id in JSON output.

## Best Possible Solution

Choose the fresh/no-resume CLI contract with the broader session-resume UX issue, then implement a backward-compatible flag through parser, local execution, Gateway protocol/handler, JSON metadata, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature request. Source, docs, protocol, release, and related-issue inspection give a high-confidence way to verify that the requested flag is still absent.

## Solution Assessment

Unclear pending maintainer UX direction. A CLI-owned generated `sessionId` routed through existing `sessionId` handling is narrow, but the flag name, aliases, resume disclosure, and JSON contract should be decided with the broader session-resume issue.
