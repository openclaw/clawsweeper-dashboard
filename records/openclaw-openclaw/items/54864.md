## Summary

Keep open. Current main and v2026.5.20 still expose explicit `--session-id` but no generated `--new-session` flag or JSON output contract, and the linked draft PR was closed unmerged after only the session-id isolation behavior shipped.

## What This Changes

The issue requests a new `openclaw agent --new-session` flag that generates a fresh session id internally and returns that id in JSON output.

## Best Possible Solution

If maintainers want this CLI UX, add a backward-compatible flag that generates one UUID before dispatch, flows through local/Gateway/fallback paths, returns the generated id in JSON, documents reconnect semantics, and coordinates with https://github.com/openclaw/openclaw/issues/71417.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature request. Source, docs, release, and related-PR inspection give a high-confidence way to verify that the requested flag is still absent.

## Solution Assessment

Unclear pending maintainer UX direction. A CLI-owned generated `sessionId` routed through existing `sessionId` handling is narrow, but the flag name, aliases, resume disclosure, and JSON contract should be decided with the broader session-resume issue.
