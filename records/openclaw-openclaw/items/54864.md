## Summary

Keep open. Current main and the latest release still lack a native `--new-session` flag or top-level generated `sessionId` JSON contract; the caller-generated `--session-id` workaround is now fixed, but the requested CLI UX still needs maintainer direction and coordination with the broader resume discussion.

## What This Changes

Request adds an `openclaw agent --new-session` flag that internally generates a fresh session ID and returns it in JSON output for later reconnects.

## Best Possible Solution

If maintainers want this UX, add a backward-compatible CLI flag that generates one UUID before local/Gateway dispatch, routes through existing `sessionId` handling, returns the generated ID in JSON, documents reconnect semantics, and coordinates with https://github.com/openclaw/openclaw/issues/71417.

## Solution Assessment

Unclear pending maintainer UX direction. CLI-owned UUID generation through existing `sessionId` handling is narrow, but flag naming, aliases, top-level JSON shape, and the relationship to resumed-session disclosure should be decided with the broader session-resume issue.
