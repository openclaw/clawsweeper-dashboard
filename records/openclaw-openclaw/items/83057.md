## Summary

Close as implemented: current main and the latest stable release already decouple managed dreaming from heartbeat activeHours by running the managed sweep as an isolated agentTurn job, with doctor migration for stale persisted jobs.

## What This Changes

The issue asks to decouple memory dreaming cron execution from heartbeat activeHours or align docs/defaults with the coupling.

## Best Possible Solution

Keep the shipped isolated managed-cron implementation; users with stale local cron stores should run `openclaw doctor --fix` rather than adding a new dreaming activeHours config.

## Reproduction Assessment

No current-main reproduction remains. Source and tests show managed dreaming jobs are created or migrated to isolated agentTurn jobs, bypassing the heartbeat-runner activeHours gate that produced `quiet-hours`.

## Solution Assessment

Yes. The current Option B-style implementation is the narrow maintainable fix because background dreaming no longer reuses the user-facing heartbeat path, while doctor handles stale persisted jobs.
