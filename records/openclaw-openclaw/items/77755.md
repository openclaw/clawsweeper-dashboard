## Summary

## What This Changes

This issue asks for WhatsApp group control commands from an owner reported as a LID JID to authorize by resolving through the account authDir.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/93379 so account.authDir is threaded through WhatsApp command authorization and owner bypass, with focused regression coverage and redacted live WhatsApp proof.

## Reproduction Assessment

Yes. Source inspection shows current main resolves WhatsApp command sender/self identities without authDir at the implicated call sites, while existing resolver code and tests show authDir can map LID JIDs through reverse mapping files.

## Solution Assessment

Yes. The narrowest maintainable fix is to pass the existing account authDir into the existing WhatsApp identity helpers at command authorization and owner-bypass call sites, rather than adding a new resolver, config option, or parallel authorization path.
