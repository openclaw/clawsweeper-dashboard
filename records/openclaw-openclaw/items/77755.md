## Summary

## What This Changes

The issue reports WhatsApp group control commands being ignored when an authorized owner sender arrives as a LID JID that needs authDir-backed phone resolution.

## Best Possible Solution

Land or replace the linked WhatsApp fix so account.authDir is threaded through command authorization and owner bypass, with focused regression coverage for LID sender mapping.

## Reproduction Assessment

Yes, at source level: a group message whose sender is only a LID JID cannot resolve through the account authDir in the current command authorization and owner-bypass call sites. I did not run a live WhatsApp transport scenario.

## Solution Assessment

Yes: the narrowest maintainable fix is to pass the existing account authDir through the affected WhatsApp plugin identity-resolution call sites and add regression coverage.
