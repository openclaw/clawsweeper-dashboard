## Summary

Keep open: current main and the latest release still have the authDir-aware WhatsApp LID mapping helper, but the reported group command authorization and owner-bypass call sites do not pass the account authDir needed to use it.

## What This Changes

The issue requests that WhatsApp group owner commands authorize sender LID JIDs by resolving them through the account auth directory.

## Best Possible Solution

Thread the existing WhatsApp account authDir through command authorization and group owner-bypass identity resolution, then add focused regression tests for LID sender owner commands in groups.

## Reproduction Assessment

Yes, at source level: a group WebInboundMessage whose sender is only a LID JID plus an account authDir reverse mapping cannot be resolved by the current command authorization and owner-bypass call sites. I did not run a live WhatsApp transport scenario.

## Solution Assessment

Yes: the narrowest maintainable fix is to pass the already-available account authDir into the existing identity helpers at the WhatsApp plugin call sites and cover that combined case in tests.
