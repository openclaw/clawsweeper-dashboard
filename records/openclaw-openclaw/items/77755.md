## Summary

Keep open: the report points to a plausible narrow WhatsApp plugin bug. Current main already normalizes many LID senders in the inbox monitor, but command authorization and group owner-bypass helpers still resolve identities without account `authDir`, leaving a LID-only command payload vulnerable to false unauthorized handling.

## What This Changes

Accept WhatsApp group control commands from the configured owner when Baileys reports the group participant as a LID JID resolvable through account auth state.

## Best Possible Solution

Keep the fix inside the WhatsApp plugin by threading account `authDir` through command authorization and group owner-gating identity resolution, with focused LID regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows a helper-level reproduction path when a group command message carries only a LID sender plus an authDir mapping, because the relevant command/owner helpers do not pass that authDir into identity resolution.

## Solution Assessment

Yes, a scoped WhatsApp plugin fix is the narrowest maintainable path. Pass existing account auth state into the identity helpers and add regression tests rather than adding core policy or new config.
