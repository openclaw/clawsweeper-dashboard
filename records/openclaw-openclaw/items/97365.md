Full review comments:

- **[P1] Honor the session provider before labeling:** `src/gateway/dashboard-session-auto-label.ts:77-83`
  - body: `generateConversationLabel()` resolves the agent default model, but `chat.send` already resolved the active session model/provider from the session entry. As wired, a dashboard session using a different or local provider can have its first message sent to the default provider for the background title, so the label path needs to honor the session model/provider or avoid the model call.
  - confidence: 0.91
- **[P1] Do not persist generated titles as user labels:** `src/gateway/dashboard-session-auto-label.ts:111-112`
  - body: Writing an automatic title into `entry.label` makes generated metadata indistinguishable from a user rename and competes with the existing `includeDerivedTitles` display contract. Use the derived-title path for the UI symptom or introduce a maintainer-approved generated-title field/policy instead of taking over the label slot.
  - confidence: 0.88
