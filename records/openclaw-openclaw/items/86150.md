## What This Changes

The PR adds an iMessage monitor repair helper that detects anchorless watch payloads, recovers their conversation anchor by GUID through recent chats/history before routing, drops unrecoverable anchorless payloads, updates monitor call sites, adds regression tests, and records the fix in the changelog.

## Best Possible Solution

## Solution Assessment

Mostly yes, pending maintainer approval: repairing by GUID before debounce/routing is the narrowest maintainable boundary for this bug. The open decision is whether fail-closed drop is acceptable as the default compatibility behavior when recovery cannot prove the target.
