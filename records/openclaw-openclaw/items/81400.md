## Summary

Keep open. Current main has a source-backed likely regression path for legacy Discord per-guild channel `allow` entries, but the reporter did not provide the exact failing config and the documented `requireMention: false` shape is accepted by the current schema, so this needs maintainer judgment rather than cleanup closure.

## What This Changes

The issue reports that upgrading from 2026.4.23 to 2026.5.12 beta rejects Discord per-channel guild config under `channels.discord.guilds.<guildid>.channels.<channelid>`.

## Best Possible Solution

Keep canonical `enabled`/`requireMention` support intact, then have a maintainer decide whether this upgrade path should get a validation-time compatibility shim or a clearer invalid-config/doctor migration path.

## Reproduction Assessment

Partially. There is a high-confidence source path for legacy `allow` being rejected before doctor migration, but not for a config that only uses the documented `requireMention: false` example because current main's schema accepts that key.

## Solution Assessment

Unclear. The closed related PR's `allow` to `enabled` normalization is a plausible narrow fix, but the repo's legacy-config policy makes this a maintainer decision rather than a safe automatic repair.
