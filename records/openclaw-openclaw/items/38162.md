## Summary

Keep this PR open. Current main still lacks the requested session-memory redirect surface, and the branch remains useful, but the latest diff is not merge-ready because the new deferred-action contract is still normalized too late and the advertised symlink redirect behavior does not match the current safe-write implementation.

## What This Changes

This PR adds `postHookActions`, session-memory block/content/redirect controls, bounded transcript reads, slug hardening, and expanded hook/session-memory tests.

## Best Possible Solution

Keep the PR open, settle or land the shared #38161 foundation first, then fix the deferred-action initialization and symlink redirect contract with tests that fail when the advertised write/retraction path is not actually exercised. After that, this needs focused maintainer/security review plus docs and changelog coverage for the new hook surface.

## Reproduction Assessment

Yes. Current-main absence is reproducible by symbol sweep and source inspection; the PR defects are reproducible by a hand-built event whose handler pushes `postHookActions`, and by using an existing in-workspace symlink as `sessionSaveRedirectPath` against current `writeFileWithinRoot` semantics.

## Solution Assessment

No, not yet. The direction is plausible, but this diff is not the narrowest maintainable implementation until `postHookActions` is normalized before handlers run and the symlink redirect behavior either matches `fs-safe` or is removed from the contract/tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Normalize postHookActions before handlers run:** `src/hooks/internal-hooks.ts:336-338`
  - body: `postHookActions` is optional, but handlers execute before this drain-time fallback. A manually constructed event without the field still reaches handlers with `event.postHookActions === undefined`, so a handler following the new contract and calling `.push(...)` throws and drops its deferred policy action. Initialize `event.postHookActions ??= []` before invoking handlers, and make the compat test push an action.
  - confidence: 0.92
- **[P2] Resolve symlink redirects before writing:** `src/hooks/bundled/session-memory/handler.ts:753-755`
  - body: The redirect branch passes the lexical symlink path to `writeFileWithinRoot`, but current `fs-safe` rejects existing symlinks during target resolution. A symlink-backed quarantine path therefore fails closed instead of writing to the intended target, while the symlink tests still pass because they only assert the old target content remains. Resolve allowed in-workspace symlinks to their canonical target before writing, or remove the advertised symlink contract and tighten the tests.
  - confidence: 0.9
- **[P3] Snapshot the test drain helper before iterating:** `src/hooks/bundled/session-memory/handler.test.ts:658`
  - body: This helper says it matches `triggerInternalHook`, but it iterates the live `postHookActions` array and never clears it. Actions appended during drain can run immediately in tests even though production snapshots first, so tests can exercise impossible ordering and miss drain regressions. Copy and clear pending actions before iterating.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the diff still has deferred-policy and filesystem redirect contract gaps.

Concerns:

- **[medium] Deferred security actions can be skipped:** `src/hooks/internal-hooks.ts:336`
  - body: Because `postHookActions` is normalized only at drain time, a policy handler on a hand-built event can throw before registering its deferred action. That can silently skip a security or persistence policy step while logging only a generic hook error.
  - confidence: 0.9

  - confidence: 0.88
