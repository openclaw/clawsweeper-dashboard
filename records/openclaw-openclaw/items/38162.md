## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize postHookActions before invoking handlers:** `src/hooks/internal-hooks.ts:336`
  - body: `postHookActions` is optional on the public event type, but `triggerInternalHook` only coalesces it after all handlers have already run. A manually constructed event that omits the field will still make any handler using the new API (`event.postHookActions!.push(...)`) throw and skip its deferred policy action; initialize `event.postHookActions ??= []` before the handler loop and make the compat test push an action.
  - confidence: 0.92
- **[P2] Make symlink redirects match the fs-safe contract:** `src/hooks/bundled/session-memory/handler.ts:753-758`
  - body: This branch advertises and tests redirect paths that are symlinks, but the write still goes through `writeFileWithinRoot`, whose current path validation rejects existing symlink targets before writing. That means the symlink rollback tests can pass without any redirected write happening; either implement a reviewed safe symlink-target write path or remove the symlink-support contract and test fail-closed behavior instead.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/hooks/internal-hook-types.ts:16-49`
  - body: This is a user-facing `feat` that adds public hook/session-memory behavior, but the PR does not touch `CHANGELOG.md`. Add a concise entry for the hook/session-memory policy controls without forcing bot or maintainer attribution.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff touches security-sensitive hook and filesystem write policy; no supply-chain issue was found, but two security-adjacent correctness contracts need attention before merge.

Concerns:

- **[medium] Deferred policy handlers can fail on legacy events:** `src/hooks/internal-hooks.ts:336`
  - body: Because `postHookActions` is normalized only after handler execution, a security or provenance handler using the new deferred-action API can throw on manually constructed events that omit the optional field, preventing its deferred block/redaction work from registering.
  - confidence: 0.88

  - body: The PR's security discussion assumes redirected symlink targets are followed and later retracted, but the current safe-write helper rejects symlink targets. This is fail-closed rather than an escape, but the documented security model and tests do not match the code path being merged.
