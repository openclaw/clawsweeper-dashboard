## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize postHookActions before dispatching handlers:** `src/hooks/internal-hooks.ts:336`
  - body: `postHookActions` is optional in the new event type, but `triggerInternalHook` only falls back to `[]` when draining after handlers run. A hand-built event without the field still reaches handlers with `event.postHookActions === undefined`, so any handler following the new contract and calling `event.postHookActions.push(...)` throws and loses its deferred action. Initialize `event.postHookActions ??= []` before the handler loop and update the SDK-compat test so the handler actually pushes a post-hook action.
  - confidence: 0.91
- **[P2] Resolve relative redirect symlinks before writing:** `src/hooks/bundled/session-memory/handler.ts:563-567`
  - body: Relative redirect paths are passed straight through as `writeRelativePath`, so an existing in-workspace symlink like `quarantine/redirected.md` is not canonicalized. `writeFileWithinRoot` rejects existing symlinks on that lexical path, while the symlink rollback tests use absolute redirect paths and miss the relative form that this feature also supports. Canonicalize relative redirects via the workspace realpath before computing the write-relative target, and add a relative-symlink regression test.
  - confidence: 0.86
- **[P3] Snapshot the test drain helper before iterating:** `src/hooks/bundled/session-memory/handler.test.ts:653`
  - body: The session-memory test helper says it matches `triggerInternalHook`, but it iterates the live `postHookActions` array. Actions appended during drain run immediately in these tests even though production snapshots and defers them, so tests can exercise impossible ordering and miss drain-regression bugs. Copy and clear the helper's pending actions before iterating.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: No arbitrary outside-workspace write was found, but the patch still has concrete security-contract gaps in the deferred policy surface and redirect symlink handling.

Concerns:

- **[medium] Deferred policy actions can be skipped on hand-built events:** `src/hooks/internal-hooks.ts:336`
  - body: The PR documents `postHookActions` as optional/additive, but `triggerInternalHook` does not normalize the field until after handlers run. A security or policy handler that pushes a deferred action on a manually constructed event can throw before registering its post-hook action, silently skipping the intended policy step.
  - confidence: 0.9
- **[low] Relative symlink redirects are not covered by the symlink contract:** `src/hooks/bundled/session-memory/handler.ts:563`
  - body: The redirect path logic only canonicalizes absolute redirect paths before writing. Relative redirect symlinks can therefore fail closed instead of following the same rollback/restore contract as the tested absolute symlink path, which weakens the quarantine redirect behavior advertised by the PR.
  - confidence: 0.82
