## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks the requested session-memory redirect/policy surface, while the PR branch has blocking contract issues: `postHookActions` is not normalized before handlers run, symlink-backed redirect writes do not match the advertised `writeFileWithinRoot` behavior, and the local drain helper can mask production ordering bugs.

## What This Changes

This PR adds `postHookActions`, session-memory block/content/redirect controls, bounded transcript and snapshot reads, filename suffix hardening, slug tool disabling, and expanded hook/session-memory tests.

## Best Possible Solution

Land #38161 or otherwise settle the shared post-hook/block/content foundation first, then rebase this PR and fix the remaining deferred-action and redirect-symlink contract bugs before a focused maintainer security review of the file-write policy and public hook surface.

## Reproduction Assessment

Yes. Current-main absence is reproducible by symbol sweep and source inspection; the PR defects are reproducible by a hand-built event whose handler pushes `postHookActions`, and by using an existing in-workspace symlink as `sessionSaveRedirectPath` against current `writeFileWithinRoot` semantics.

## Solution Assessment

No, not yet. The direction is plausible, but the patch needs pre-handler `postHookActions` normalization, a redirect symlink implementation that matches `fs-safe`, and tests that fail when the promised symlink write never occurs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Normalize postHookActions before invoking handlers:** `src/hooks/internal-hooks.ts:336`
  - body: `postHookActions` is optional, but handlers run before this drain-time fallback. A manually constructed event without the field still reaches handlers with `event.postHookActions === undefined`, so a handler following the new contract and calling `.push(...)` throws and drops its deferred policy action. Initialize `event.postHookActions ??= []` before the handler loop, and make the compat test push an action.
  - confidence: 0.92
- **[P2] Resolve redirect symlinks before writing:** `src/hooks/bundled/session-memory/handler.ts:753-758`
  - body: The redirect branch passes symlink paths straight to `writeFileWithinRoot`, but current `fs-safe` rejects existing symlinks during its pre-write open/probe. That means a symlink-backed quarantine path fails closed instead of writing to the intended target, while the new symlink tests still pass because they only assert the old target content remains. Resolve allowed in-workspace symlinks to their canonical target before this write or remove the advertised symlink contract and tighten the tests.
  - confidence: 0.9
- **[P3] Snapshot the test drain helper before iterating:** `src/hooks/bundled/session-memory/handler.test.ts:658`
  - body: This helper claims to match `triggerInternalHook`, but it iterates the live `postHookActions` array and never clears it. Actions appended during drain can run immediately in tests even though production snapshots first, so tests can exercise impossible ordering and miss drain regressions. Copy and clear the pending actions before iterating.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: No outside-workspace write path was confirmed, but the patch still has security-relevant contract gaps in deferred policy execution and quarantine redirect handling.

Concerns:

- **[medium] Deferred security actions can be skipped:** `src/hooks/internal-hooks.ts:336`
  - body: Because `postHookActions` is normalized only after handlers run, a policy handler on a hand-built event can throw before registering its deferred action. That can silently skip a security or persistence policy step while logging only a generic hook error.
  - confidence: 0.9
- **[low] Symlink quarantine redirects are not actually exercised:** `src/hooks/bundled/session-memory/handler.ts:753`
  - body: The PR advertises symlink-aware redirect writes and rollback, but the current safe-write helper rejects existing symlink targets before the redirect branch can write. A symlink-backed quarantine path therefore fails closed and the tests do not prove the transcript ever reached the intended target.
  - confidence: 0.87
