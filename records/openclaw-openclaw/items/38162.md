## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Normalize postHookActions before dispatch:** `src/hooks/internal-hooks.ts:336`
  - body: `postHookActions` is documented as optional, but handlers run before the drain-time fallback at line 336. A manually constructed SDK event without this field will still make any handler that follows the new contract and calls `.push(...)` throw, skipping its deferred policy action. Initialize `event.postHookActions ??= []` before invoking handlers and make the compat test register a post-hook action.
  - confidence: 0.92
- **[P2] Either support or remove symlink redirects:** `src/hooks/bundled/session-memory/handler.ts:753-755`
  - body: The PR advertises redirect writes through symlinks, but current `writeFileWithinRoot` resolves the write target by opening existing paths via `openFileWithinRoot`, which rejects symlinks. A symlink redirect therefore fails closed before writing, while the new tests can still pass because they only assert the old target content remains. Resolve allowed in-workspace symlinks before writing, or remove the symlink contract and update the tests.
  - confidence: 0.9
- **[P3] Snapshot the test post-hook drain:** `src/hooks/bundled/session-memory/handler.test.ts:658`
  - body: This helper says it matches production, but it iterates the live `postHookActions` array and never clears it. Actions appended during a test drain can run immediately even though production snapshots and clears first, so tests may cover impossible ordering. Copy and clear pending actions before iterating.
  - confidence: 0.82
