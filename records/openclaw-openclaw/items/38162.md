Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize postHookActions before dispatching handlers:** `src/hooks/internal-hooks.ts:336`
  - body: The new field is optional, but handlers run before the drain-time `event.postHookActions ?? []` fallback. A hand-built SDK event without the field still makes any handler that calls `event.postHookActions.push(...)` throw and skip its deferred policy action; initialize `event.postHookActions ??= []` before the handler loop and update the compat test to use a pushing handler.
  - confidence: 0.91
- **[P3] Match the session-memory drain helper to runtime semantics:** `src/hooks/bundled/session-memory/handler.test.ts:658`
  - body: This helper says it matches `triggerInternalHook`, but it iterates the live action array without snapshotting or clearing it. That means direct handler tests can execute self-appended actions or replay stale actions in ways production no longer does; use the same snapshot-and-clear helper here.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/hooks/internal-hook-types.ts:49`
  - body: This PR adds a user-facing hook/session-memory capability and security hardening, but the branch does not add an active-version `CHANGELOG.md` entry. Add a concise entry without forcing bot/maintainer attribution.
  - confidence: 0.86
