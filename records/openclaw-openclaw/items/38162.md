## Summary

Keep this PR open. Current main still lacks the requested session-memory redirect and post-hook policy controls, and the latest PR head remains not merge-ready because deferred policy actions can still be skipped for hand-built events and the filesystem redirect rollback contract has destructive edge cases that need source-branch fixes plus maintainer/security review.

## What This Changes

The PR adds a postHookActions deferred hook mechanism, session-memory block/custom-content/redirect controls, redirect path hardening, slug-generation hardening, gateway event initialization, and expanded hook/session-memory tests.

## Best Possible Solution

Keep the PR open, settle the #38161 foundation first, then update this branch so postHookActions is initialized before handlers run, unsnapshotable existing redirect targets fail closed before overwrite, and symlink-backed redirects are either implemented against fs-safe semantics or removed from the public contract and tests. The final branch should also carry the required user-facing docs/changelog and pass focused tests plus the changed gate in Testbox.

## Reproduction Assessment

Yes. Current-main absence is reproducible by symbol/source/docs inspection, and the PR defects are reproducible by a hand-built hook event whose handler pushes postHookActions plus redirect targets that are symlinks or pre-existing files larger than the snapshot cap.

## Solution Assessment

No. The product direction is plausible, but the current PR is not the best mergeable fix until the deferred-action compatibility path and redirect rollback semantics are made non-destructive and consistent with fs-safe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Normalize postHookActions before dispatch:** `src/hooks/internal-hooks.ts:336-338`
  - body: `postHookActions` is optional, but handlers run before this drain-time fallback. A hand-built event without the new field still makes any handler that follows the new contract and calls `.push(...)` throw, skipping its deferred policy action. Initialize `event.postHookActions ??= []` before invoking handlers and update the compat test to register a post-hook action.
  - confidence: 0.93
- **[P2] Fail closed when rollback cannot restore prior content:** `src/hooks/bundled/session-memory/handler.ts:707-712`
  - body: When the pre-existing target is larger than the snapshot cap, this branch leaves `preExistingContent` null and continues with the inline write. If a later hook sets `blockSessionSave`, rollback takes the null-snapshot unlink path and deletes the file that existed before this handler ran. Abort before overwriting unsnapshotable existing targets, or use another non-destructive rollback strategy.
  - confidence: 0.9
- **[P2] Either support or remove symlink redirects:** `src/hooks/bundled/session-memory/handler.ts:753-755`
  - body: The PR advertises redirect writes through symlinks, but current `writeFileWithinRoot` uses the pinned atomic write helper, which replaces the final symlink path rather than writing through it. The new symlink tests can pass while only proving the old target was untouched. Resolve allowed in-workspace symlinks intentionally before writing, or remove the symlink contract and tighten the tests/comments.
  - confidence: 0.88
- **[P3] Snapshot the test post-hook drain:** `src/hooks/bundled/session-memory/handler.test.ts:658`
  - body: This helper says it matches production, but it iterates the live `postHookActions` array and never clears it. Actions appended during a test drain can run immediately even though production snapshots and clears first, so tests may cover impossible ordering. Copy and clear pending actions before iterating.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the patch still has deferred-policy and filesystem rollback contract gaps, though it does not add new third-party dependencies or supply-chain execution paths.

Concerns:

- **[medium] Deferred security actions can be skipped:** `src/hooks/internal-hooks.ts:311`
  - body: Because `postHookActions` is not normalized before handlers run, a policy handler on a hand-built event can throw before registering its deferred action, silently skipping a persistence/security step except for a generic hook error log.
  - confidence: 0.9
- **[medium] Late block can delete existing redirect data:** `src/hooks/bundled/session-memory/handler.ts:707`
  - body: For pre-existing redirect targets above the snapshot cap, the handler proceeds without a restore snapshot and later unlinks on `blockSessionSave`, which can destroy prior operator data while trying to retract the new transcript.
  - confidence: 0.88
- **[medium] Symlink quarantine contract is not proven:** `src/hooks/bundled/session-memory/handler.ts:753`
  - body: The PR describes symlink-aware redirect writes and rollback, but the current safe-write helper atomically replaces the final symlink path instead of preserving and writing through the symlink, so the advertised quarantine behavior is not implemented end to end.
  - confidence: 0.85
