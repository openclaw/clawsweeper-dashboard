## What This Changes

The branch serializes Android gateway/realtime playback, accumulates Talk transcripts in Android/Web, moves Web Talk bubbles into the chat thread, and adds forced gateway-relay consult handling with focused tests.

## Best Possible Solution

Keep the transcript and playback stabilization, but gate or split strict forced-consult routing until maintainers approve the default and the branch has real relay plus Android runtime proof.

## Reproduction Assessment

Partially yes: source inspection shows the old gateway-relay path sent finalized user transcripts back to the realtime provider, while the PR schedules a forced consult instead. I did not run a live Android/Web realtime session in this read-only review.

## Solution Assessment

No, not as-is: the transcript/playback fixes target real code paths, but replacing provider-direct behavior should be gated, split, or explicitly approved as a compatibility-breaking default.
