## Summary

Keep this PR open: the Android ordering, transcript, and audio-queue pieces address plausible current-main bugs, but the branch also adds a forced Gateway consult path that changes existing realtime Talk routing and still lacks live Android behavior proof.

## What This Changes

The branch serializes Android Gateway websocket handling, accumulates Android and Control UI realtime Talk transcript turns, queues Android PCM playback, adds a forced realtime agent-consult fallback, and updates tests plus changelog text.

## Best Possible Solution

Keep the Android frame-ordering, transcript-accumulation, and audio-queue fixes, but remove or explicitly gate the forced-consult fallback and require redacted live Android Talk proof before merge.

## Reproduction Assessment

Partially: current-main source gives a high-confidence path for transcript deltas replacing prior text and websocket frame handlers racing per-message coroutines. I did not establish a live Android device reproduction for choppy playback.

## Solution Assessment

No: the Android stability pieces look like the right code path, but the unconditional forced-consult fallback is not the narrowest maintainable fix because it changes the current direct-reply boundary.
