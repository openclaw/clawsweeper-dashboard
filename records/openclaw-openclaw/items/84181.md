## Summary

Keep this PR open: the transcript/playback stabilization is useful, but the branch still makes forced Web realtime consult routing unconditional and merge-sensitive, and the supplied proof does not cover the audio or forced-consult runtime path well enough for merge.

## What This Changes

The PR changes Android Gateway message serialization and Talk playback/transcript accumulation, Web realtime Talk conversation rendering, gateway relay consult routing, focused tests, and changelog entries.

## Best Possible Solution

Land the transcript/playback and chat-thread rendering fixes behind focused coverage, but preserve provider-direct relay replies by default unless maintainers explicitly approve and document a strict consult policy with fresh-install and upgrade proof.

## Reproduction Assessment

Partially yes: current-main source shows the provider-direct relay path that this PR replaces, and the supplied screenshots show visible Talk turn-flow problems. I did not independently reproduce Android audio instability or forced-consult timing with runtime diagnostics.

## Solution Assessment

No: the stabilization pieces are directionally useful, but bundling them with an unconditional forced-consult default is not the safest maintainable fix. The safer path is a gated or split policy change with upgrade proof.
