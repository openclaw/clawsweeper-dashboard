## Summary

Keep this PR open. The Android ordering, transcript, and playback fixes are plausibly aimed at real current-main failure paths, but the branch also adds an unconditional forced-consult fallback that changes realtime provider routing and direct-reply behavior before there is a product decision or live Android proof.

## What This Changes

The PR serializes Android Gateway realtime frames, accumulates Android and Control UI Talk transcript turns, queues Android PCM playback writes, and adds Gateway-side forced realtime consult routing when the provider does not call the consult tool.

## Best Possible Solution

Land the Android frame-ordering, transcript accumulation, Control UI transcript-flow, and PCM queue fixes after proof, but remove, split, or explicitly policy-gate the forced-consult fallback with coverage for both direct replies and consult-required turns.

## Reproduction Assessment

No full live reproduction: current-main source gives a high-confidence path for transcript deltas replacing prior text and WebSocket frame handlers racing, but choppy playback still needs live Android device or emulator proof.

## Solution Assessment

No: the Android stabilization pieces are targeted, but forcing consults for every final realtime user transcript is not the narrowest maintainable fix without an explicit policy gate or split follow-up.
