## Summary

Keep open for maintainer review; current-main source supports the Android realtime Talk bug shape, but this branch is not merge-ready because proof is test-only and the forced-consult fallback can change existing direct realtime replies into Gateway chat consults.

## What This Changes

The branch serializes Android Gateway websocket message handling, accumulates realtime Talk transcript turns in Android and Control UI, queues Android AudioTrack playback, and adds a Gateway forced agent-consult fallback with tests and changelog text.

## Best Possible Solution

Land the Android ordering, transcript accumulation, and audio queue fixes after removing or gating the forced-consult fallback, then require redacted live Android Talk proof for the playback path.

## Reproduction Assessment

Partially: current-main source gives a high-confidence path for transcript deltas replacing prior text and websocket frame handlers racing per-message coroutines. I did not establish a live Android device reproduction for choppy playback.

## Solution Assessment

No: the Android stability pieces target the right code paths, but the unconditional forced-consult fallback is not the narrowest maintainable fix because it changes the current direct-reply boundary.
