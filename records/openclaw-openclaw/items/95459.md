Keep open: the code shape looks like the right narrow fix for the Silent cron quick-create failure, but this external PR still needs redacted live Control UI/Gateway proof before merge. ## What This Changes

The PR sets the Silent cron quick-create preset to emit `sessionTarget: "main"` plus `payloadKind: "systemEvent"` and adds a node test for that mapping.

## Best Possible Solution

Land this or an equivalent main-plus-systemEvent fix after live proof, then close https://github.com/openclaw/openclaw/issues/95073 and supersede https://github.com/openclaw/openclaw/pull/95074.

## Reproduction Assessment

## Solution Assessment

Yes for the code shape: setting only the Silent preset's payload kind to `systemEvent` is narrower than relaxing backend validation or redefining Silent as isolated/no-delivery. The remaining gap is live behavior proof, not a code-path objection.
