## Summary

Keep this PR open, but it is not merge-ready: the broad `tools.fs.roots` feature has good proof for the happy path, yet the PR head still appears to widen an explicit empty roots policy back to the workspace when outbound media access has a `workspaceDir`. The new config and SDK/media boundary also still need maintainer product/security acceptance before merge.

## What This Changes

The PR adds a `tools.fs.roots` filesystem allowlist with `dir`/`file` and `ro`/`rw` modes, then threads it through host file tools, `apply_patch`, media loading, outbound delivery, docs/generated config metadata, and bundled channel media paths.

## Best Possible Solution

Fix empty-root handling so configured roots are terminal unless an explicit sandbox/generated-media bypass is active, add focused regression coverage, then require maintainer product/security/API acceptance for the new roots contract before merge.

## Reproduction Assessment

Yes, source-reproducible for the PR defect: on the PR head, `tools.fs.roots: []` reaches `appendWorkspaceDirToLocalRoots`, which returns `[workspaceDir]` when a workspace directory is supplied. I did not run tests because this review was read-only, but the code path and missing test case are clear.

## Solution Assessment

No, this is not yet the best mergeable solution because it does not preserve the new deny-all roots contract across outbound media access. The best fix is to keep configured roots terminal before workspace fallback and separately ask maintainers to accept the broad filesystem-root product/security contract.
