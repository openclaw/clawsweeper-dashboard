## Summary

Keep open. The feature is still unique, but this PR is not merge-ready because its branch conflicts with current main's least-privilege tool-policy contract and still lacks real behavior proof for the new filesystem/media access surface.

## What This Changes

The PR adds `tools.fs.roots` with per-root file/dir and ro/rw modes across host filesystem tools, `apply_patch`, media loading, outbound delivery, prompt images, docs, schema, and regression tests.

## Best Possible Solution

Rebase on current main, keep `tools.fs.roots` behind the existing explicit `tools.alsoAllow` access contract, and land only after maintainer security approval plus real behavior proof.

## Reproduction Assessment

Not applicable as a user bug reproduction, but the blocking PR regression is source-reproducible: compare current main's restrictive-profile tests/docs with the PR head lines that add implicit filesystem grants.

## Solution Assessment

No, not as-is. The root-scoped design may be useful, but the PR must preserve current main's explicit-allow policy and prove the real host-root/media behavior before it is the maintainable solution.
