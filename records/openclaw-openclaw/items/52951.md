## What This Changes

The PR adds `tools.fs.roots`, a global/per-agent host filesystem allowlist with per-root `dir`/`file` and `ro`/`rw` modes, threaded through file tools, `apply_patch`, media loading, outbound delivery, config docs, and the plugin SDK media runtime.

## Best Possible Solution

Fix the empty-root deny-all regression, rebase cleanly, add focused outbound/media proof, then have maintainers choose whether this broad roots model or the narrower workspace-alias design is the accepted security boundary.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `resolveAgentScopedOutboundMediaAccess` can turn `baseLocalRoots: []` into `[resolvedWorkspaceDir]` through `appendWorkspaceDirToLocalRoots` when a workspace is resolved; I did not execute tests because this review is read-only.

## Solution Assessment

No. The broad roots implementation is plausible for non-empty roots, but the current patch is not the best mergeable solution until deny-all roots remain deny-all across outbound media, proof is refreshed, and maintainers accept the broader config/API direction.
