## Summary

Keep this PR open, but the current head is not merge-ready: `tools.fs.roots: []` can still be widened back to the agent workspace in outbound media access, which breaks the PR's deny-all security contract.

## What This Changes

The PR adds a new per-agent `tools.fs.roots` filesystem allowlist with read/write modes and threads it through host filesystem tools, media loading, outbound delivery, docs, generated baselines, and plugin SDK media helpers.

## Best Possible Solution

Preserve explicit empty roots as deny-all through outbound media access, add a regression that includes an agent workspace, then have maintainers approve the new config and SDK contract before merge.

## Reproduction Assessment

Yes for the PR-introduced bug: source inspection shows configured empty roots resolve to an empty list, then `appendWorkspaceDirToLocalRoots` converts it to the agent workspace when a workspace is available. This is not a current-main bug; it is introduced by the proposed patch.

## Solution Assessment

No; the proposed feature is plausible, but the current implementation needs to preserve empty-root deny-all semantics before the broader API design can be judged merge-ready.
