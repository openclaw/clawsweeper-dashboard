## Summary

Keep open: current `main` still has the reported source behavior, but this PR broadens group admission in a way that conflicts with the current docs, warning copy, and Telegram authorization review standard.

## What This Changes

The PR changes group policy resolution so explicit `groupPolicy: "open"` treats `groups` entries as per-group config overrides instead of enabling an allowlist, and adds regression tests for channel and account scopes.

## Best Possible Solution

Decide the intended group semantics first; if `open` should make `groups` override-only, update docs, warnings, and channel tests with live Telegram proof, otherwise preserve `groups` as the route allowlist.

## Reproduction Assessment

Yes from source inspection, but not from a live Telegram run. Current `main` sets `allowlistEnabled` when any group entry exists, and the Telegram command path returns `This group is not allowed` for an unlisted group.

## Solution Assessment

No. The one-line resolver change is too broad for the current contract because docs and warnings still treat configured group keys as the route allowlist under open group policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
