## Summary

Keep open: the targeted bug is still source-valid, but this draft PR lacks real behavior proof and its guard must be rebased so heartbeat-isolated suppression runs after node exec-event authorization and terminal cleanup.

## What This Changes

The branch loads the session entry in `src/gateway/server-node-events.ts`, returns early for heartbeat-isolated exec lifecycle events, and adds a focused gateway regression test.

## Best Possible Solution

Rebase the marker-based suppression onto current main after authorization/cleanup, preserve normal non-heartbeat exec notifications, add authorized heartbeat-session coverage, and supply redacted live gateway proof.

## Reproduction Assessment

Yes, at source level. Current main writes `heartbeatIsolatedBaseSessionKey` for isolated heartbeat sessions, while the gateway exec lifecycle handler ignores that marker and still enqueues plus wakes after authorization; I did not run a live gateway loop.

## Solution Assessment

No, not as currently proposed. Marker-based suppression is the narrow direction, but it must run after current exec-event authorization and cleanup, and the regression should exercise an authorized heartbeat-session event.
