## Summary

Keep open: the self-wake bug is plausible and not already fixed on current main, but this PR is not merge-ready because the new early return would bypass the gateway's terminal `system.run` exec-event authorization cleanup and the PR still has only test/check proof.

## What This Changes

Adds a guard in `src/gateway/server-node-events.ts` to skip system-event enqueue and heartbeat wake for exec lifecycle events from sessions marked with `heartbeatIsolatedBaseSessionKey`, plus a focused gateway regression test.

## Best Possible Solution

Keep the gateway suppression idea, but perform it only after `authorizeNodeSystemRunEvent` succeeds and terminal cleanup has run, with focused NodeRegistry regression coverage and redacted real gateway/node heartbeat proof.

## Reproduction Assessment

Yes at source level: current main still routes authorized node exec lifecycle events through enqueue/wake for the session key, and heartbeat-isolated sessions are marked in the session entry. I did not run a live gateway/node heartbeat scenario in this read-only review.

## Solution Assessment

No, not in the current ordering. The gateway handler is the right boundary, but suppression must run after `authorizeNodeSystemRunEvent` so terminal authorization cleanup is preserved.
