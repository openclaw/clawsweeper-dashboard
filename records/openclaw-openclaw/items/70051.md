## Summary

Keep open: current `main` still lacks the heartbeat-isolated marker guard, so the bug-fix direction is useful, but the PR is still draft and the branch is incorrect against current `main` because the suppression must not bypass exec-event provenance authorization and terminal cleanup. The PR also still has only test/check validation rather than real after-fix gateway proof.

## What This Changes

The branch adds a gateway guard and regression test to suppress node exec lifecycle system-event enqueue and heartbeat wake for sessions marked as heartbeat-isolated.

## Best Possible Solution

Keep the focused marker-based suppression, but rebase it onto current `main` after `authorizeNodeSystemRunEvent`, preserve normal non-heartbeat exec notifications, add authorized heartbeat-session coverage, and provide real after-fix gateway proof.

## Reproduction Assessment

Yes, at source level. Current `main` writes `heartbeatIsolatedBaseSessionKey` for isolated heartbeat sessions, while the gateway exec lifecycle handler ignores that marker and still enqueues an event plus `exec-event` wake after authorization; I did not run a live gateway loop.

## Solution Assessment

No, not as currently proposed. Marker-based suppression is the right narrow direction, but it must run after current exec-event authorization and cleanup, and the regression should exercise an authorized heartbeat-session event.
