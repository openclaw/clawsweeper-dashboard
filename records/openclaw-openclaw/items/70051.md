## Summary

Keep open: the PR targets a plausible heartbeat self-wake bug, but the current patch returns before the gateway's system.run exec-event authorization path, so terminal heartbeat-isolated exec events can skip cleanup of the authorized event window; it also still lacks external real behavior proof.

## What This Changes

The PR adds a gateway guard and regression test to skip system-event enqueue and heartbeat wake requests for exec lifecycle events on heartbeat-isolated sessions.

## Best Possible Solution

Authorize and clear node system.run exec events first, then suppress only enqueueSystemEvent and requestHeartbeat for heartbeat-isolated sessions with focused terminal cleanup coverage and redacted live proof.

## Reproduction Assessment

Yes, at source level: current main marks heartbeat-isolated sessions and still routes authorized exec lifecycle events through enqueue/wake without checking that marker. I did not run a live gateway heartbeat scenario.

## Solution Assessment

No, not as currently ordered. The gateway handler is the right boundary, but the suppression should run after system.run event authorization so terminal cleanup is preserved before notification replay is skipped.
