## What This Changes

This PR changes `src/gateway/server-node-events.ts` to skip exec lifecycle enqueue/wake handling when the loaded session entry is heartbeat-isolated, and adds focused gateway node-event test coverage.

## Best Possible Solution

Keep the suppression in the gateway exec-event path, but run system.run authorization and terminal cleanup first, then skip only system-event enqueue and heartbeat wake for heartbeat-isolated sessions with a focused regression and real runtime proof.

## Reproduction Assessment

Yes at source level: current main marks heartbeat-isolated sessions and gateway exec events still enqueue/wake without checking that marker. I did not establish a live gateway heartbeat reproduction in this read-only review.

## Solution Assessment

No, not as currently ordered. The owner boundary is plausible, but the guard should sit after system.run event authorization so terminal cleanup still happens before notification replay is suppressed.
