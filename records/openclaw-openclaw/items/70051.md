## Summary

Keep this PR open, but not merge-ready: the heartbeat self-wake problem is still plausible on current main, while the patch suppresses the event before the system.run authorization/terminal cleanup path runs and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a `server-node-events` guard to skip system-event enqueue and heartbeat wake for exec lifecycle events on heartbeat-isolated sessions, plus a focused gateway unit test.

## Best Possible Solution

Keep the suppression in `server-node-events`, but run system.run event authorization and terminal cleanup first, then skip only system-event enqueue and heartbeat wake for heartbeat-isolated sessions.

## Reproduction Assessment

Yes, at source level: current main marks heartbeat-isolated sessions and `server-node-events` still enqueues/wakes for every authorized exec lifecycle event without checking that marker. I did not run a live gateway heartbeat scenario, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No, not as currently ordered. The owner boundary is plausible, but the guard should sit after system.run event authorization so terminal cleanup still happens before notification replay is suppressed.

- **[P1] Authorize heartbeat exec events before suppressing the wake:** `src/gateway/server-node-events.ts:635-637`
