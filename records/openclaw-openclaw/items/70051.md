## What This Changes

The PR adds a gateway guard and regression test intended to skip exec lifecycle system-event enqueueing and heartbeat wakes for heartbeat-isolated sessions.

## Best Possible Solution

Move the suppression after successful exec-event authorization and terminal cleanup, keep it before notify/enqueue/wake work, update the regression to the current authorized harness, and add redacted live gateway proof showing no second wake.

## Reproduction Assessment

Yes, at source level. Current main stores heartbeatIsolatedBaseSessionKey for isolated heartbeat sessions, while the gateway exec lifecycle handler ignores that marker and still enqueues an exec event plus an exec-event wake after authorization.

## Solution Assessment

No, not as currently proposed. Marker-based suppression is the right narrow direction, but it must run after current exec-event authorization and cleanup, and the regression needs to exercise the authorized path on current main.
