## Summary

Keep open: the heartbeat-isolated exec self-wake problem is still meaningful, but this branch is not merge-ready because its early return can bypass current gateway `system.run` exec-event authorization cleanup and it lacks after-fix real behavior proof.

## What This Changes

The PR adds a guard in `src/gateway/server-node-events.ts` so node `exec.*` events for heartbeat-isolated session entries do not enqueue a system event or request another heartbeat wake.

## Best Possible Solution

Keep the gateway suppression, but move it after successful exec-event authorization so terminal cleanup still runs, then require focused regression coverage and redacted live gateway proof before merge.

## Reproduction Assessment

Yes at source level: current main and v2026.6.9 authorize node exec lifecycle events and then enqueue output-bearing events plus an `exec-event` heartbeat wake without checking the heartbeat-isolated marker. I did not run a live gateway heartbeat scenario.

## Solution Assessment

No, not as currently ordered. The gateway handler is the right boundary, but the suppression should run after exec-event authorization succeeds so terminal authorization cleanup is preserved before notification replay is skipped.
