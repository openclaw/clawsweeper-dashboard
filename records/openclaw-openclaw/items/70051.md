## Summary

Keep open. The PR targets a real current-main gap in heartbeat-isolated exec event replay, but the patch is not merge-ready because the suppression is ordered before the current `system.run` exec-event authorization/terminal cleanup path and the external PR still has only test/check validation, not real after-fix gateway proof.

## What This Changes

Adds a `server-node-events` guard and regression test to suppress system-event enqueue and heartbeat wake for exec lifecycle events from heartbeat-isolated sessions.

## Best Possible Solution

Keep the suppression in `server-node-events`, but authorize and clear terminal `system.run` events first, then suppress only system-event enqueue and heartbeat wake for heartbeat-isolated sessions.

## Reproduction Assessment

Yes, at source level: current main marks heartbeat-isolated sessions and still enqueues/wakes authorized exec lifecycle events without checking that marker. I did not run a live gateway heartbeat scenario, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No, not as currently ordered. The owner boundary is plausible, but the guard should run after system.run event authorization so terminal cleanup still happens before notification replay is suppressed.
