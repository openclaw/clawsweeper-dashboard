## Summary

Keep open: current main and the latest release still replay heartbeat-isolated exec node events, but this PR is not merge-ready because the added guard would run before the current system.run authorization cleanup path and the external real-behavior proof is still missing.

## What This Changes

The PR adds a gateway guard and regression test to skip system-event enqueue and heartbeat wakes for exec events on heartbeat-isolated sessions.

## Best Possible Solution

Authorize and clear node system.run exec events first, then suppress only enqueueSystemEvent and requestHeartbeat for heartbeat-isolated sessions with focused coverage and redacted live proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 mark heartbeat-isolated sessions and still enqueue/wake authorized exec lifecycle events without checking that marker. I did not run a live gateway heartbeat scenario.

## Solution Assessment

No, not as currently ordered. The gateway handler is the right boundary, but the suppression should run after system.run event authorization so terminal cleanup is preserved before notification replay is skipped.
