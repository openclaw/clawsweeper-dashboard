## Best Possible Solution

Rebase the guard onto the current gateway exec path so authorization and terminal cleanup run first, then skip only system-event enqueue and heartbeat wake for heartbeat-isolated sessions with focused coverage and redacted live proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently ordered. The gateway owner boundary is plausible, but the suppression should run after system.run event authorization so terminal cleanup is preserved before notification replay is skipped.
