## Summary

Keep open. The underlying source path is still plausible on current main, but this draft PR is not mergeable as-is because the suppression would land before current node exec-event authorization/cleanup, the regression test is stale, and the external proof gate is still missing.

## What This Changes

The branch adds a gateway guard and regression test to skip system-event enqueueing and heartbeat wakes for node exec lifecycle events whose session entry is marked as heartbeat-isolated.

## Best Possible Solution

Rebase the guard onto current main after authorizeNodeSystemRunEvent, update the regression to use an authorized heartbeat-isolated event and current mocks, then provide redacted real gateway proof showing no second wake.

## Reproduction Assessment

Yes, at source level. Current main writes heartbeatIsolatedBaseSessionKey for isolated heartbeat sessions, while the gateway exec lifecycle handler ignores that marker and still enqueues an exec event plus an exec-event wake after authorization; I did not run a live gateway loop.

## Solution Assessment

No, not as currently proposed. Marker-based suppression is the right narrow direction, but it must run after current exec-event authorization/cleanup and the regression needs to exercise the authorized path on current main.
