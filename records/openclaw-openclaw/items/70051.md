## Summary

Keep open: current main still lacks the heartbeat-isolated exec self-wake suppression, and this PR is the only focused open branch for it. The branch is not merge-ready because the early return must preserve current `system.run` authorization cleanup, and the external PR still has only test/check proof rather than real gateway/node behavior proof.

## What This Changes

The PR adds a guard in `src/gateway/server-node-events.ts` plus regression coverage to skip system-event enqueueing and heartbeat wakes for `exec.*` events on heartbeat-isolated synthetic sessions.

## Best Possible Solution

Keep the focused fix idea, but move the heartbeat-isolated suppression after `authorizeNodeSystemRunEvent`, add regression coverage for terminal cleanup plus no enqueue/wake, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes at source level: current main still enqueues and wakes for authorized node `exec.finished` events after resolving the session key, and heartbeat-isolated sessions are explicitly marked in session entries. I did not run a live gateway/node heartbeat scenario in this read-only review.

## Solution Assessment

No, not as currently ordered against current main. The gateway handler is the right boundary, but the suppression should run after node-run authorization and terminal cleanup, then skip only enqueue/wake for synthetic heartbeat sessions.
