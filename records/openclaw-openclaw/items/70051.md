## Summary

Keep open. The bug is still source-reproducible on current main, but this PR is not merge-ready because the guard predates current exec-event provenance authorization, its regression test is stale, and the external contributor proof gate is still missing.

## What This Changes

The PR adds a gateway guard and regression test to skip system-event enqueueing and heartbeat wakes for exec lifecycle events on sessions marked with `heartbeatIsolatedBaseSessionKey`.

## Best Possible Solution

Move the suppression after successful `authorizeNodeSystemRunEvent`, keep it before notify/enqueue/wake work, update the regression to use the current authorized harness, and add redacted live gateway proof showing no second wake.

## Reproduction Assessment

Yes, at source level. Current main stores `heartbeatIsolatedBaseSessionKey` for isolated heartbeat sessions, while the gateway exec lifecycle handler ignores that marker and still enqueues an exec event plus an `exec-event` wake after authorization; I did not run a live gateway loop in this read-only review.

## Solution Assessment

No, not as currently proposed. Marker-based suppression is the right narrow direction, but it must run after current exec-event authorization/cleanup and the regression needs to exercise the authorized path on current main.
