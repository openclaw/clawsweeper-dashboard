## Best Possible Solution

Authorize exec lifecycle events first, then suppress only system-event enqueue and heartbeat wake for heartbeat-isolated sessions, with real gateway/heartbeat proof before merge.

## Reproduction Assessment

Yes, at source level: current main marks heartbeat-isolated sessions and `server-node-events` still enqueues and wakes for authorized exec lifecycle events without checking that marker. I did not run a live gateway heartbeat scenario, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No, not as currently ordered. The owner boundary is plausible, but the guard should sit after system.run event authorization so terminal cleanup still happens before notification replay is suppressed.
