## Summary

Keep open. Current main still has the source-level heartbeat-isolated exec replay path, but this draft PR needs rebase/adaptation because its guard would bypass current exec-event authorization and it still lacks real behavior proof.

## What This Changes

The PR adds a marker-based guard in gateway node exec-event handling to suppress system-event enqueue/wake for heartbeat-isolated sessions and adds a focused regression test.

## Best Possible Solution

Rebase onto current main, place heartbeat-isolated suppression after exec-event authorization, preserve normal non-heartbeat notifications, cover an authorized heartbeat-session exec event, and add redacted live gateway/log proof before merge.

## Reproduction Assessment

Yes, at source level. Current main still routes a marked heartbeat-isolated exec.finished through enqueueSystemEvent and requestHeartbeat because the gateway handler ignores heartbeatIsolatedBaseSessionKey; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not as currently proposed against main. The marker-based suppression is the right narrow direction, but the guard and test must be rebased so current exec-event authorization runs before suppression.
