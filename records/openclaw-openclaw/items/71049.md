## Summary

Keep open: current main still hard-codes the MS Teams personal-DM streaminfo/informative-status path, `typingIndicator` only gates the native typing keepalive, and the strict config schema has no server-side DM stream-status opt-out.

## What This Changes

The issue requests a server-side MS Teams config option to disable the personal-DM informative stream status and fall back to native typing plus final message delivery.

## Best Possible Solution

Add a default-preserving MS Teams plugin config switch that disables personal-DM streaminfo/informative-status setup while keeping native typing controlled by `typingIndicator` and final-message delivery unchanged for opted-out DMs.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: a personal MS Teams dispatcher with `typingIndicator: false` still creates a stream controller, sends the informative update, and installs `onPartialReply`.

## Solution Assessment

Yes. A plugin-owned, default-on config guard around the personal-DM stream controller is the narrowest maintainable fix; core streaming contracts should not change for this MS Teams-specific behavior.
