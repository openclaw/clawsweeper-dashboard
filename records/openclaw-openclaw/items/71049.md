## Summary

Keep open: current main still hardwires the MS Teams personal-chat streaminfo/informative-status path from conversation type alone, the strict MS Teams config schema has no server-side DM stream-status opt-out, and the related Microsoft tracker still lists this as remaining work.

## What This Changes

Add an MS Teams plugin config option to disable the Copilot-style informative stream status in personal DMs and fall back to native typing plus normal final-message delivery.

## Best Possible Solution

Add a default-preserving MS Teams plugin config switch that disables personal-DM streaminfo/informative-status setup, omits the partial-reply stream callback in that mode, keeps native typing controlled by typingIndicator, and updates schema, generated config docs, user docs, changelog, and focused tests.

## Solution Assessment
