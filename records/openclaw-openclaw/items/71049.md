## Summary

Keep open: current main still unconditionally creates the MS Teams personal-chat streaminfo/informative-status path, and the strict MS Teams config schema has no server-side DM stream-status opt-out. The related Microsoft tracker still lists this as remaining work.

## What This Changes

The issue requests an MS Teams config option to disable the Copilot-style informative stream status in 1:1 chats and fall back to native typing plus a single final reply.

## Best Possible Solution

Add a default-preserving MS Teams config switch under channels.msteams that disables the personal-DM streaminfo/informative-status path, omits the partial-reply stream callback in that mode, keeps native typing controlled by typingIndicator, and updates schema, generated config metadata, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create a personal MS Teams dispatcher with typingIndicator=false, trigger onReplyStart, and current main still sends the informative update while also installing the personal-chat onPartialReply stream callback.

## Solution Assessment

Yes. A plugin-owned, default-on config option is the narrowest maintainable solution because the behavior is MS Teams-specific and already lives in the MS Teams dispatcher, stream controller, config schema, docs, and tests.
