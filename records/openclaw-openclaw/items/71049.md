## Summary

Keep open. Current main still creates the MS Teams personal-chat streaminfo/informative-status path from conversation type alone, the strict MS Teams config schema has no server-side DM stream-status opt-out, and the related Microsoft tracker still lists this as remaining work.

## What This Changes

Request a `channels.msteams` option to disable Copilot-style informative stream status/streaming in MS Teams DMs and fall back to native typing plus one final reply.

## Best Possible Solution

Add a default-preserving MS Teams plugin config switch that prevents personal-DM streaminfo/informative-status setup, omits the partial-reply stream callback in that mode, keeps native typing under `typingIndicator`, and updates schema, generated config docs, user docs, changelog, and focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create a personal MS Teams dispatcher with `typingIndicator: false`; current tests and code still send the informative update and still install the stream partial-reply callback.

## Solution Assessment

Yes. A plugin-owned, default-on config option is the narrowest maintainable solution because the behavior is MS Teams-specific and already lives in the MS Teams dispatcher, stream controller, config schema, docs, and tests.
