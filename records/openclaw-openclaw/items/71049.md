## Summary

Close: current main now has an MS Teams streaming config surface, and the personal-DM streaminfo controller only creates Teams native streaming for `partial` or `progress` modes. Setting `channels.msteams.streaming.mode` to `off` disables the Copilot-style stream path while the native typing keepalive remains controlled separately by `typingIndicator`.

## What This Changes

Request to add an MS Teams config option that disables Copilot-style streaminfo/informative status in 1:1 chats and uses native typing plus final-message delivery instead.

## Best Possible Solution

Keep the current `channels.msteams.streaming.mode` implementation; users who want final-only DM replies can set it to `off` while default behavior preserves Teams streaminfo.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Current source inspection gives high-confidence verification that the requested config path now exists and disables the MS Teams native stream controller for non-preview modes.

## Solution Assessment

Yes. The current implementation uses the shared channel streaming config and keeps the change inside the MS Teams plugin boundary, which is narrower than changing core streaming or overloading `typingIndicator`.
