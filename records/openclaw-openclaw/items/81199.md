## Summary

Keep this PR open, but do not merge it as-is: changing the Anthropic CLI backend default is not enough because the current runner still skips system-prompt file creation and prompt-flag appending whenever `useResume` is true. The supplied proof is test output only and does not show a real resumed Claude CLI invocation after the patch.

## What This Changes

The PR changes the Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds a focused backend assertion, and adds a changelog entry.

## Best Possible Solution

Make the Claude CLI resume path create and append the generated harness system-prompt file for both one-shot and default `claude-stdio` live sessions, then add a regression that asserts resumed argv contains the prompt-file flag and verify a real resumed Claude CLI run.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a resumed Claude CLI run sets `useResume`, skips prompt-file creation, suppresses prompt flags, and the reporter’s fresh-versus-resumed argv evidence matches that path. I did not run a live Claude CLI binary in this read-only review.

## Solution Assessment

No. The default change is likely necessary, but it is not sufficient because the resume execution and live-session argument paths still block the prompt file from being written or appended.
