## Summary

Keep this PR open, but not mergeable as written: the linked Claude CLI resume bug is real, yet the patch only changes the Anthropic backend default and a narrow assertion while the current resume runner and live-session paths still suppress the generated system-prompt file and prompt flags.

## What This Changes

The PR changes the Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds an Anthropic backend assertion, and adds a changelog entry for resumed Claude CLI system-prompt reinjection.

## Best Possible Solution

Update the runner and Claude live-session resume paths so `systemPromptWhen: "always"` actually writes and appends the generated prompt file on resumed Claude CLI turns, then prove it with a real resumed Claude CLI argv/log plus focused regressions.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: a resumed Claude CLI run sets `useResume`, skips system-prompt file creation and prompt-flag appending, and the reporter's fresh-versus-resumed argv evidence in https://github.com/openclaw/openclaw/issues/80374 matches that path.

## Solution Assessment

No. The default change may be necessary, but it is not sufficient because the resume execution path and Claude live-session argument builder still prevent the prompt file from reaching resumed runs.
