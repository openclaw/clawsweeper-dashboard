## Summary

Keep open: this PR targets a real Claude CLI resume bug, but the diff only changes the Anthropic backend default and does not update the runner paths that create and append the system-prompt file on resumed runs.

## What This Changes

The PR changes the bundled Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds an Anthropic backend assertion, and adds a changelog entry for resumed-session prompt reinjection.

## Best Possible Solution

Update the shared CLI resume path, including the default `claude-stdio` live-session path, so `systemPromptWhen: "always"` actually writes and appends the generated harness prompt file on resumed Claude CLI turns, then add regression coverage for resumed argv.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: resumed Claude CLI turns set `useResume`, skip prompt-file creation, and suppress system-prompt flags, matching the linked report's fresh-versus-resumed argv evidence.

## Solution Assessment

No. Changing the backend default is likely part of the fix, but it is not sufficient because the one-shot and live-session resume paths still refuse to create or append the prompt file.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Append the prompt file on resumed runs:** `extensions/anthropic/cli-backend.ts:74`

A resumed `claude --resume` turn therefore still has no file path or flag, so the linked bug remains unfixed until the resume runner path is updated.
