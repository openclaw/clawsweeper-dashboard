## Summary

Keep open: this PR targets a real Claude CLI resume regression, but the patch only changes the Anthropic backend default and a config-level assertion. Current main still skips prompt-file creation and prompt argv injection when `useResume` is true, and the default Claude live-session path strips prompt flags, so merging this as-is would not fix the linked issue.

## What This Changes

The PR changes the bundled Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds a backend-config regression assertion, and adds a changelog note for resumed-session system-prompt reinjection.

## Best Possible Solution

Update the runner and Claude live-session resume argv path so a generated system-prompt file or arg survives `claude --resume`, keep the backend default only if it is still needed, and verify the real resumed Claude CLI flow.

Yes. Source inspection on current main gives a high-confidence reproduction path: a resumed Claude CLI run sets `useResume`, skips prompt-file creation, suppresses prompt flags in `buildCliArgs`, and strips prompt flags again in the default `claude-stdio` live path.
