## Summary

This PR should stay open with changes requested: the backend-default change is too shallow because the current runner still skips system-prompt file creation and argument appending on resumed runs, and the default Claude live-session path strips prompt flags when `useResume` is true. The PR also still has only test-output proof, not a real resumed Claude CLI run after the fix.

## What This Changes

The PR changes the Anthropic Claude CLI backend default from `systemPromptWhen: "first"` to `"always"`, adds a small backend-config regression assertion, and adds a changelog entry for resumed-session prompt reinjection.

## Best Possible Solution

Update the CLI runner and Claude live-session resume argument path so resumed Claude invocations actually create and pass the generated harness prompt file, then add final argv coverage and redacted real resume proof before merge.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main sets `useResume`, skips prompt-file creation, suppresses prompt flags in `buildCliArgs`, and strips prompt flags in the default `claude-stdio` path; the linked issue's fresh-versus-resumed argv evidence matches that path.

## Solution Assessment

No. Changing the backend default to `systemPromptWhen: "always"` may be related, but it is not sufficient while prompt-file creation, generic arg building, and live-session argv still block resume prompt injection.
