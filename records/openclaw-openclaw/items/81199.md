## Summary

This PR should stay open and not merge as-is: the default change is plausible, but the runner still suppresses system-prompt file creation and prompt flags whenever `useResume` is true, so the linked Claude CLI resume bug remains.

## Best Possible Solution

Update the runner and Claude live-session resume paths so `systemPromptWhen: "always"` actually writes and appends the prompt file for resumed Claude CLI turns, then add argv coverage for resumed invocations.
