## Summary

Keep open: the PR changes the plausible Anthropic CLI default, but current runner code still suppresses system-prompt file creation and argv injection on resume, so the linked bug would remain unfixed.

## What This Changes

The PR changes the Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds a small regression assertion, and adds a changelog entry for resumed-session system-prompt reinjection.

## Best Possible Solution

Update the runner resume path so `systemPromptWhen: "always"` actually creates and appends the system-prompt file for resumed Claude CLI turns, including the live-session path, then add a regression that asserts resumed argv contains the prompt file flag.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main’s Claude CLI backend defaults to `systemPromptWhen: "first"`, and the runner/build-argv path suppresses system-prompt files when `useResume` is true. I did not run a live Claude CLI binary, but the internal argv path is deterministic.

## Solution Assessment

No. The default change is likely necessary, but it is not sufficient because the resume execution path still refuses to write or append the prompt file; the narrower maintainable fix needs to cover that path directly.

## Review Findings

Overall correctness: patch is incorrect
