## Summary

Keep this PR open, but do not merge as-is: the linked bug is real, and the default change is likely part of the fix, but the resume runner still skips prompt-file creation/argv appending and the supplied proof is test-only.

## What This Changes

The PR changes the Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds a small regression assertion, and adds a changelog entry for resumed-session system-prompt reinjection.

## Best Possible Solution

Update the Claude CLI runner resume path so both one-shot and default `claude-stdio` live resume runs create and pass the generated system-prompt file when the backend opts into `systemPromptWhen: "always"`, with regression coverage for resumed argv.

## Reproduction Assessment

Yes. Source inspection on `origin/main` gives a high-confidence reproduction path: resumed runs set `useResume`, skip prompt-file creation, and build/live argv suppress prompt flags; the linked issue's fresh-versus-resumed argv evidence matches that path.

## Solution Assessment

No. Changing the backend default is likely necessary, but this PR is not sufficient because the runner still needs to create and append the prompt file on resume, including the `claude-stdio` live-session path.
