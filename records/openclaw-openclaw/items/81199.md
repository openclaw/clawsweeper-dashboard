## Summary

Keep open: source inspection shows this PR does not fix the reported resume path because the runner still skips creating and appending the system-prompt file when `useResume` is true, and the supplied after-fix proof is test output rather than a real resumed Claude CLI run.

## What This Changes

The PR changes the Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds one backend assertion, and adds a changelog entry.

## Best Possible Solution

Update the CLI runner and Claude live-session resume paths so `systemPromptWhen: "always"` creates and passes the generated prompt file on resumed Claude CLI turns, then assert resumed argv contains `--append-system-prompt-file`.

## Reproduction Assessment

Yes. Source inspection on current `main` shows resumed Claude CLI runs set `useResume`, skip prompt-file creation, and guard prompt args behind `!useResume`; the linked issue's fresh-versus-resumed argv evidence matches that path.

## Solution Assessment

No. Changing only the Anthropic backend default is likely necessary but not sufficient because both the execute path and arg builder still block prompt-file creation and appending on resume, and the Claude live-session path also strips the prompt flag.
