## Summary

Keep this PR open, but the patch is not currently mergeable: it changes the Claude CLI backend default without changing the runner paths that still skip prompt-file creation and prompt-flag appending on resume.

## What This Changes

The PR changes the bundled Anthropic Claude CLI backend default to `systemPromptWhen: "always"`, adds a focused backend assertion, and adds a changelog entry for resumed-session system-prompt reinjection.

## Best Possible Solution

Update the CLI runner resume path and Claude live-session argv path so resumed Claude CLI turns actually receive the generated harness prompt file, then add regression coverage for the final resumed argv.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a resumed Claude CLI run sets `useResume`, skips prompt-file creation, and suppresses system-prompt flags before spawn.

## Solution Assessment

No. Changing the backend default is likely necessary, but it is not sufficient because the generic runner and default Claude live-session path still remove the prompt file from resumed invocations.

## Review Findings
