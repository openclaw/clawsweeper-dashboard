## Summary

Keep open: current main still intentionally suppresses ordinary group final replies under the default visible-reply policy, and the 2026.5.4 follow-up confirms Feishu users still hit it; resolving upgrade compatibility versus the tool-only group policy needs maintainer judgment.

## What This Changes

The issue requests restoring or clearly mitigating Feishu group visible replies after the 2026.5.3 change made generated assistant final text stay private unless `messages.groupChat.visibleReplies` is set to `automatic` or the model uses the message tool.

## Best Possible Solution

Maintainers should choose the compatibility policy, then implement either upgrade-preserving automatic group replies for affected installations or explicit Feishu-facing migration diagnostics and docs around the existing `visibleReplies` setting.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection and user logs: a Feishu group turn with unset `messages.groupChat.visibleReplies` reaches `message_tool_only`, ordinary assistant final text is skipped, and the Feishu log reports `queuedFinal=false, replies=0`.

## Solution Assessment

Is this the best way to solve the issue? Unclear: current main is internally consistent and documented, but whether upgrades should preserve legacy visible replies or keep tool-only group output is a maintainer product decision.
