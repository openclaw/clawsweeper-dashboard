## Summary

Keep open: current main still lacks the requested public plugin approval and Gateway SDK surfaces, while three open follow-up PRs now split the work into reviewable pieces.

## What This Changes

The issue requests generic plugin approval action metadata, no-route pending behavior, plugin-owned verified approval resolution, and richer chat.inject metadata for external HITL plugins.

## Best Possible Solution

Land focused additive host APIs with narrow SDK helpers for the generic seams, while keeping AgentKit itself as an external ClawHub plugin.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested hook fields, verified approval method, SDK helper surface, and chat.inject metadata are absent from the relevant schemas, handlers, and exports.

## Solution Assessment

Yes, with maintainer API/security review. The split-PR direction is maintainable, and the safer shape appears to be narrow verified/chat SDK helpers rather than exposing a broad generic admin Gateway client.
