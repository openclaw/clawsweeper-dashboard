## What This Changes

The issue requests generic OpenClaw plugin approval and gateway SDK APIs so an external AgentKit HITL plugin can stay outside core while resolving proof-backed approvals.

## Best Possible Solution

Settle the approval trust model, then land a narrow generic core/plugin API PR for external verification and metadata while keeping AgentKit itself in ClawHub.

## Reproduction Assessment

Yes for the API gap: source inspection on current main shows the requested verified-resolution method, hook fields, admin SDK helper, and rich chat.inject metadata are not available. This is not a runtime bug repro; it is a missing public API surface.

## Solution Assessment

Unclear pending maintainer direction. The small-core/external-plugin split matches VISION.md, but the verified approval resolver needs an explicit plugin-bound versus operator-scoped trust decision before implementation.
