Keep open: current main and v2026.6.10 still expose ordinary plugin approval request/resolve APIs, while the narrowed external HITL approval seam is owned by an open implementation PR that still needs product and security review. ## What This Changes

The issue requests a narrow host API for external HITL plugins to present verification metadata and resolve only their own pending approvals after external verification succeeds.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82434 or a focused successor only after the verifier authority, plugin ownership checks, and operator-scoped trust model are settled; keep AgentKit-specific World verification in the external ClawHub plugin.

## Solution Assessment

Unclear pending maintainer direction. The narrow host API fits the plugin boundary, but resolver authority and the security model need maintainer review before this is the best implementation path.
