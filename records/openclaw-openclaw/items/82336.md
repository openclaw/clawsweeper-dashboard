## Summary

Keep open: current main still lacks several of the requested generic approval and gateway SDK surfaces, and the remaining work is a concrete public API expansion rather than stale cleanup.

## What This Changes

The issue requests generic plugin approval and gateway-runtime APIs so an external AgentKit HITL plugin can resolve proof-backed approvals from ClawHub.

## Best Possible Solution

Land a small additive core API PR for generic approval actions, plugin-owned verified resolution, trusted admin gateway helpers, and richer chat injection metadata, while leaving AgentKit itself as a ClawHub package.

## Reproduction Assessment

Yes. Source inspection on current main shows the missing fields, methods, exports, and chat injection parameters in the TypeScript hook types, Gateway schemas, SDK exports, and handler code; this is an API-gap reproduction rather than a runtime bug repro.

## Solution Assessment

The requested direction appears sound but needs maintainer API/security review: extracting generic plugin seams from the closed AgentKit branch matches the plugin boundary, but the admin-scoped verification and plugin-id matching rules should be settled before coding.
