## Summary

Keep open: current main still lacks the requested generic approval and gateway SDK APIs, and the related AgentKit work was explicitly moved out of core while leaving this smaller API gap for follow-up. This is a concrete public plugin/protocol expansion with approval-security semantics, not cleanup that should be closed.

## What This Changes

Expose generic approval actions, keep-pending behavior, plugin-owned verified approval resolution, operator-admin gateway SDK helpers, and richer chat injection metadata for external HITL plugins.

## Best Possible Solution

Land a small additive core API PR for generic approval actions, keep-pending approval records, plugin-owned verified approval resolution, trusted gateway helpers, and richer chat injection metadata while leaving AgentKit itself as an external ClawHub package.

## Reproduction Assessment

Yes for the API gap. Source inspection on current main shows the requested hook fields, RPC method, SDK exports, and chat injection parameters are absent from the relevant contracts and handlers.

## Solution Assessment

Unclear until maintainers settle the API and authorization shape. Extracting generic host seams from the closed AgentKit branch matches OpenClaw's plugin boundary, but a broad operator.admin SDK client would be riskier than a narrow verified-approval and controlled-injection capability.
