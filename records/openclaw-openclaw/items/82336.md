## Summary

Keep open: this is the separate core API request the related AgentKit PR was asked to file, and current main still lacks several generic plugin approval hooks needed by an external HITL plugin.

## What This Changes

Expose narrow generic host APIs for external HITL plugins to create custom approval actions, hold pending plugin approvals without a live route, resolve verified approvals, and inject approval metadata into chat.

## Best Possible Solution

Define a narrow public plugin API for verified plugin-owned approval resolution and controlled approval-message injection, while keeping AgentKit itself on ClawHub and avoiding a generic public admin client export unless explicitly approved.

## Reproduction Assessment

Yes for the API gap: source inspection on current main confirms several requested fields and methods are absent. This is not a runtime bug reproduction; it is a missing public SDK/gateway contract.

## Solution Assessment

Partly: a separate focused core API issue is the right route after moving AgentKit to ClawHub, but the admin-client part should be narrowed or explicitly approved because it changes the plugin authority boundary.
