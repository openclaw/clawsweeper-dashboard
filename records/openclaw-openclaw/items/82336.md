## Summary

Keep open: current main and v2026.6.8 still lack the central public plugin approval resolver/external verification surface, and the remaining work is owned by an open companion PR that needs maintainer API/security direction.

## What This Changes

Request to expose generic OpenClaw host/plugin SDK approval APIs so external HITL plugins such as AgentKit can block protected tools, show external verification actions, and resolve verified approvals without bundling AgentKit in core.

## Best Possible Solution

Review the open verified-resolution PR or a refreshed successor, settle the plugin-bound versus operator-scoped trust model, and expose only the generic security-reviewed core/plugin API while keeping AgentKit itself in ClawHub.

## Reproduction Assessment

Yes for the API gap: current-main source inspection shows the verified resolver, external approval metadata, admin helper, and chat.inject metadata are not exposed. This is a missing public API surface rather than a runtime bug repro.

## Solution Assessment

Unclear pending maintainer direction. The core/plugin split matches VISION.md, but the resolver trust model and any remaining chat.inject metadata need maintainer API/security approval before implementation is the best fix.
