## Summary

Keep open: current main still lacks the requested generic approval, gateway SDK, and chat injection surfaces, and the related maintainer discussion explicitly asked for a separate API issue after moving AgentKit out of core.

## What This Changes

Request to add generic plugin approval actions, verified plugin-owned approval resolution, trusted gateway helpers, and richer `chat.inject` metadata for external HITL plugins.

## Best Possible Solution

Land a small additive core API PR for narrow verified approval resolution, custom approval actions, keep-pending behavior, and controlled approval-card injection metadata while keeping AgentKit itself external on ClawHub.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested fields, method, exports, and `chat.inject` parameters are absent from the hook types, protocol schemas, method descriptors, SDK facade, and handler code; this is an API-gap reproduction rather than a runtime bug.

## Solution Assessment

The direction is sound, but not all requested details are necessarily the best boundary. A narrow verified-approval and controlled session-message API is safer than exposing a general public `operator.admin` gateway client.
