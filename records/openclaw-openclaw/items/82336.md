## Summary

Keep open: current main still lacks the requested generic approval, gateway SDK, and chat injection API surfaces, and this is a concrete public API/security design item rather than cleanup.

## What This Changes

The issue requests additive plugin approval, gateway-runtime, and chat injection APIs needed by an external AgentKit HITL plugin.

## Best Possible Solution

Land a small additive, documented core API PR for approval actions, keep-pending behavior, narrow verified plugin-owned resolution, and richer controlled chat injection metadata while keeping AgentKit external.

## Reproduction Assessment

Yes. This is source-reproducible as an API gap: the current TypeScript hook types, Gateway schemas/descriptors, SDK exports, and chat.inject handler lack the requested fields and methods.

## Solution Assessment

Unclear until maintainer API/security review. The direction matches the plugin boundary, but the safest solution is a narrow verified-approval and controlled injection seam rather than a general public operator.admin gateway helper.
