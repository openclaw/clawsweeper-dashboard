## Summary

Keep open: current main and v2026.6.10 still expose only ordinary plugin approval request/list/wait/resolve APIs, while https://github.com/openclaw/openclaw/pull/82434 is the active implementation candidate and still needs maintainer API/security direction.

## What This Changes

Request a narrow host/plugin approval API so external HITL plugins can require external verification before a protected tool call continues.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82434 or a refreshed successor only after the verified-resolution authority model is settled, while keeping AgentKit-specific verification in the external ClawHub plugin.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested `externalResolution` metadata and `plugin.approval.resolveVerified` method are absent from the hook type, protocol schema, Gateway handler, method registry, tests, and docs.

## Solution Assessment

Unclear pending maintainer direction. The core/plugin split is aligned with VISION.md, but the verified resolver authority model needs security/API approval before this is the best implementation shape.
