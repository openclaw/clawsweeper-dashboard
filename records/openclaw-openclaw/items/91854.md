## Summary

Keep open: current main still routes memory-core Dream Diary narrative runs through the generic plugin subagent path without a no-tools or restricted-tools control, so the reported `apply_patch` exposure remains valid.

## What This Changes

The issue asks OpenClaw to prevent memory-core dreaming narrative and REM diary subagents from receiving workspace-mutating tools such as `apply_patch`.

## Best Possible Solution

Add a typed plugin subagent tool restriction, such as a no-tools or explicit allowlist path, and use it for memory-core narrative runs with regression coverage that fallback diary writes still work.

## Reproduction Assessment

No live reproduction was run for the intermittent cheap-model behavior. Source inspection gives a clear path: the narrative subagent has no tool restriction, the plugin gateway path cannot carry one, and undefined downstream `toolsAllow` leaves default coding tools available.

## Solution Assessment

The requested direction is right, but the API shape needs maintainer review. Removing the tool structurally is stronger than only tightening `NARRATIVE_SYSTEM_PROMPT`, because the reported failure is caused by tool exposure rather than prose generation alone.
