## What This Changes

The issue reports that memory dreaming narrative subagents can see file-mutation tools such as apply_patch while they are only supposed to generate diary prose.

## Best Possible Solution

Run Dream Diary narrative generation through a prose-only path, either via a maintainer-approved subagent tool-scope API or an existing no-tools completion helper, while preserving the configured dreaming model behavior.

## Reproduction Assessment

Yes at source level for the overbroad tool exposure: current main has no narrative-specific tool scope in the memory-core subagent call or plugin subagent API. The stochastic apply_patch misuse itself still needs live proof with the reported cheap model.

## Solution Assessment

Unclear until maintainers choose the API shape. Removing mutation tools from the prose-only lane is the right end state, but the clean owner-boundary fix may be either subagent tool scoping or a no-tools completion path.
