The issue requests that `skill_workshop(action="apply")` avoid a generic Codex dynamic-tool timeout when default Skill Workshop lifecycle approval is still pending.

## Best Possible Solution

Yes. The narrow maintainable fix is to align the Codex `skill_workshop` dynamic-tool timeout with the existing approval ceiling while preserving Skill Workshop approval defaults.
