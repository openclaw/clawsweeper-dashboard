## Summary

Keep open. Current main and the latest release still have the same source-level mismatch: Skill Workshop lifecycle approval defaults to pending and can wait 120s, while generic Codex dynamic tool calls time out at 90s, so agent `skill_workshop(action="apply")` can surface a generic dynamic-tool timeout before approval resolves.

## What This Changes

The issue reports that agent-triggered Skill Workshop apply times out under default pending approval while the CLI apply path succeeds.

## Best Possible Solution

Preserve pending approval as the default, make Skill Workshop lifecycle approval resolve or fail with a clear approval-specific result before the Codex dynamic-tool watchdog, and cover pending plus auto behavior in regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: default pending Skill Workshop apply requires approval, the approval wait defaults to 120s, and the generic Codex dynamic-tool watchdog fires at 90s. I did not run a live Gateway/Codex session in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow timeout/error-path alignment that keeps existing pending and auto semantics. A new per-action config option is not needed to fix the reported bug and would add avoidable config surface.
