## Summary

## What This Changes

The issue reports that agent `skill_workshop` lifecycle apply can time out under default pending approval before the Skill Workshop approval path resolves.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/91325 with a focused fix that keeps pending lifecycle approval as the default while making Codex main and side `skill_workshop` calls outlive or clearly fail the approval wait.

## Reproduction Assessment

Yes. Source inspection shows default pending Skill Workshop lifecycle approval can wait 120s plus gateway grace while Codex dynamic-tool calls time out at 90s, and a recent user comment reports the same timeout on 2026.6.5.

## Solution Assessment

Yes. The best narrow solution is to align the Codex `skill_workshop` dynamic-tool timeout/error path with the existing approval ceiling, not to make the broad `approvalPolicy: "auto"` workaround the default or add a new config surface.
