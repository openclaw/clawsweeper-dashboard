## Summary

Keep open. Current main still uses threshold-gated deep dreaming promotion with no wildcard/serendipity lane, and the request is a durable-memory product-policy enhancement rather than already-implemented cleanup.

## What This Changes

Request to add a wildcard lane that samples sub-threshold deep dreaming promotion candidates so occasional unexpected memories can be promoted.

## Best Possible Solution

If maintainers want this behavior, add an explicit capped wildcard path in memory-core that preserves hard eligibility gates and apply-time explainability, with tests and docs for the chosen policy.

## Reproduction Assessment

Yes for current behavior: source and tests show candidates below minScore are excluded from ranking and apply-time promotion. This is not a broken existing contract; it is the behavior the feature request wants to relax.

## Solution Assessment

Unclear. A serendipity lane may solve the requested outcome, but a default random 0.15 wildcard policy is not obviously the narrowest maintainable choice for durable memory.
