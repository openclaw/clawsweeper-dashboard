## Summary

Keep open: current main and v2026.6.10 still lack a wildcard/random sub-threshold Dreaming promotion lane, but this is a feature/product decision about durable-memory randomness rather than a safe cleanup close or automated bug fix.

## What This Changes

This issue asks for Dreaming deep promotion to randomly include some sub-threshold wildcard candidates so unexpected memories can occasionally surface for durable promotion.

## Best Possible Solution

Have maintainers choose the durable-memory serendipity policy first; if approved, implement it coherently across ranking, apply, config/schema, docs, and focused tests with a safe default or explicit opt-in.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new Dreaming promotion policy request. Source and latest-release inspection confirm current behavior remains strict threshold-gated promotion with no wildcard lane.

## Solution Assessment

Unclear. Random wildcard sampling is plausible for serendipity, but an explicit opt-in or report-only review lane may be safer, and a rank-only change would be incomplete because apply rechecks `minScore`.
