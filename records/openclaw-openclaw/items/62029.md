## Summary

Keep open. Current main and the latest release still use strict threshold-gated Dreaming deep promotion with no wildcard candidate lane, and choosing durable-memory randomness, default ratio, and config policy needs maintainer product review.

## What This Changes

The issue requests random sub-threshold wildcard candidates in memory-core Dreaming deep promotion so occasional unexpected memories can surface for durable promotion.

## Best Possible Solution

Have maintainers choose the memory promotion policy first; if approved, implement an explicit, explainable wildcard lane across ranking, apply, config/schema, docs, and focused tests, likely with an opt-in or report-only rollout.

## Reproduction Assessment

Not applicable as a bug reproduction: this issue asks for a new wildcard promotion policy. Source, docs, tests, and v2026.6.8 inspection confirm the current behavior is still strict threshold-gated promotion.

## Solution Assessment

Unclear. A random wildcard lane is plausible, but default nondeterminism in durable memory is not clearly the best solution compared with an explicit opt-in or report-only review lane.
