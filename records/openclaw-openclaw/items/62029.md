## Summary

Keep open. Current main and v2026.6.8 still use strict threshold-gated Dreaming deep promotion with no wildcard candidate lane, and the remaining question is a maintainer product decision about randomness in durable memory.

## What This Changes

Request to add random sub-threshold wildcard candidates to Dreaming deep promotion ranking so unexpected memory connections can occasionally surface.

## Best Possible Solution

Have maintainers choose the durable-memory promotion policy first; if approved, implement an explicit wildcard or report-only lane across ranking, apply, config/schema, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new Dreaming promotion policy request. Source, docs, tests, and v2026.6.8 inspection confirm current behavior remains strict threshold-gated promotion.

## Solution Assessment

Unclear. A random wildcard lane is plausible, but default nondeterminism in durable memory is not clearly the best solution compared with an explicit opt-in or report-only review lane.
