## Summary

Keep open. Current main and v2026.6.6 still use strict threshold-gated memory-core deep promotion with no random wildcard lane, while the newer shadow-trial scoring path remains report-only; choosing default durable-memory randomness and a new config surface needs maintainer product review.

## What This Changes

Request adding a wildcardRatio-driven random sampling lane for sub-threshold memory-core deep promotion candidates.

## Best Possible Solution

Have maintainers choose the memory promotion policy first; if approved, implement an explicit, explainable wildcard lane across ranking, apply, config/schema, docs, and focused tests, likely with a conservative opt-in or report-only rollout.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. Source, docs, tests, and v2026.6.6 inspection verify that current behavior still strictly threshold-gates deep promotion candidates.

## Solution Assessment

Unclear. A wildcard lane is plausible, but default randomness in durable memory is not obviously the best solution compared with an explicit opt-in or report-only review lane.
