## Summary

Keep open. Current main still treats deep dreaming promotion as strict thresholded ranking: sub-threshold candidates are discarded during ranking and again during apply, and the docs describe the threshold gates as required. The request remains a real memory-core product-policy enhancement, not something already covered by the related dreaming fixes.

## What This Changes

Add a wildcard lane to dreaming deep ranking that can sample sub-threshold candidates which passed hard gates so occasional unexpected memories can surface for promotion.

## Best Possible Solution

If maintainers want this behavior, add an explicit, documented, deterministic and capped wildcard path in memory-core that preserves hard eligibility gates, contamination filtering, age limits, and threshold explainability; tag wildcard-selected candidates so apply can admit only those entries without weakening ordinary minScore enforcement.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the current behavior: a short-term candidate that passes signal/diversity gates but scores below minScore is dropped in rankShortTermPromotionCandidates and would also be rejected by applyShortTermPromotions.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A serendipity lane may fit the requested outcome, but a default random 0.15 promotion policy is not obviously the safest durable-memory behavior; an opt-in or deterministic capped lane would be easier to review and explain.
