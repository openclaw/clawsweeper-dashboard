## What This Changes

Request to add random sub-threshold wildcard candidates to memory-core dreaming deep promotion ranking.

## Best Possible Solution

If maintainers want this behavior, add a bounded and explainable wildcard lane in memory-core that preserves hard gates, threads policy through rank and apply, records promotion reasons, and documents the chosen default/config surface.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and tests give high confidence that current behavior is strictly threshold-gated and has no wildcard lane.

## Solution Assessment

Unclear. A bounded wildcard lane may fit, but the default randomness, config surface, promotion reasons, and apply-time semantics need maintainer product judgment before implementation.
