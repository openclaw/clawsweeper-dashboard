## Summary

Keep open. Current main still uses strict thresholded deep-dreaming promotion, and the requested wildcard lane is a memory-core product-policy change rather than already-implemented cleanup.

## What This Changes

Add wildcard sub-threshold candidates to deep dreaming promotion ranking so occasional unexpected memories can surface for long-term promotion.

## Best Possible Solution

If maintainers want this behavior, add an explicit deterministic and capped wildcard path in memory-core that preserves hard eligibility gates, contamination filtering, age limits, and threshold explainability, with wildcard-selected entries marked for apply-time admission.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for current behavior: a candidate that passes hard gates but scores below minScore is dropped by rankShortTermPromotionCandidates and would also be rejected by applyShortTermPromotions.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A serendipity lane may fit the requested outcome, but a default random 0.15 promotion policy is not obviously the narrowest maintainable durable-memory behavior; a deterministic capped or opt-in lane would be safer to review.
