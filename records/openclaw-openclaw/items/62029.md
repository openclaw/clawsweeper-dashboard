## Summary

Keep open: current main and v2026.6.10 still use strict threshold-gated Dreaming promotion with no wildcard/random sub-threshold lane, but adding nondeterministic durable-memory promotion is a feature/product decision rather than a safe cleanup close or automated bug fix.

## What This Changes

The issue requests adding wildcard/random sub-threshold candidates to memory-core Dreaming deep-phase promotion ranking so unexpected memories can occasionally reach durable memory.

## Best Possible Solution

Have maintainers choose the durable-memory serendipity policy first; if approved, implement it coherently across ranking, apply, config/schema or internal policy, docs, and focused tests with a safe default or explicit opt-in.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new Dreaming promotion policy request. Source and latest-release inspection confirm current behavior remains strict threshold-gated promotion with no wildcard lane.

## Solution Assessment

Unclear. Random wildcard sampling is plausible for serendipity, but an explicit opt-in or report-only review lane may be safer, and a rank-only change would be incomplete because apply rechecks `minScore`.
