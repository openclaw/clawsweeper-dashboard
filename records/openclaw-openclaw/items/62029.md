## Summary

Keep open. Current main still uses strict threshold-gated deep dreaming promotion with no wildcard or serendipity lane, and the requested behavior is a durable-memory product enhancement rather than already-implemented cleanup.

## What This Changes

The issue requests a wildcard lane for memory-core deep dreaming that randomly samples sub-threshold but hard-gated promotion candidates for possible durable promotion.

## Best Possible Solution

If maintainers want this behavior, add an explicit bounded wildcard lane in memory-core that preserves hard gates, works consistently in ranking and apply, records explainable promotion reasons, and documents the chosen policy.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. The current behavior is source-proven by the rank/apply filters and tests that keep one-off recalls out under default thresholds.

## Solution Assessment

Unclear. A serendipity lane may satisfy the request, but durable-memory randomness and defaults need maintainer product judgment, and the proposed ranking-only shape would not cover apply-time filtering.
