## Summary

Keep open. Current main and v2026.5.18 still strictly threshold-gate deep dreaming promotions with no wildcard or serendipity lane, and adding stochastic durable-memory promotion is a product/config decision rather than conservative cleanup.

## What This Changes

Request to add a wildcard lane that samples sub-threshold memory promotion candidates into deep dreaming ranking so unexpected memories can occasionally be promoted.

## Best Possible Solution

If maintainers want this behavior, add a bounded and explainable wildcard lane in memory-core that preserves hard gates, threads the policy through ranking and apply, records promotion reasons, and documents the chosen default or config surface.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an enhancement request. Source, docs, release code, and tests give a high-confidence verification path that current behavior still strictly threshold-gates deep promotion candidates.

## Solution Assessment

Unclear. A bounded wildcard lane may fit the request, but randomness, default ratio, config exposure, apply-time bypass semantics, and durable-memory safety need maintainer product judgment before implementation.
