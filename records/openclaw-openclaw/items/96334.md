## Summary

Keep open. Current main and the latest release still preserve the reported lightweight cron behavior: lightContext cron runs omit OpenClaw bootstrap files and disable Codex native project-doc loading, so the remaining question is a maintainer product decision about whether lightweight cron should stay minimal or gain a bounded rules tier.

## What This Changes

The issue asks OpenClaw to prevent lightContext cron sessions from missing critical workspace rules such as TOOLS.md, AGENTS.md, and learnings files, or at least document and warn about that omission.

## Best Possible Solution

Maintainers should choose one lightweight cron contract: keep the current minimal mode with clearer docs/warnings, or add a bounded critical-rules path that preserves token savings and aligns with any broader bootstrap-tier direction.

## Reproduction Assessment

Yes. Current source and tests show lightContext cron jobs resolve to lightweight mode, lightweight cron/default mode filters bootstrap files to an empty list, and Codex lightweight threads set project_doc_max_bytes to 0.

## Solution Assessment

Unclear. Documentation and warning logs are narrow, but always injecting critical rules or adding selective loading changes the tested lightweight cron contract and should be chosen by maintainers, not by an automated fix lane.
