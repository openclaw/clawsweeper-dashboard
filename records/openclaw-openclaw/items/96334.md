## Summary

Keep open. Current main and v2026.6.10 still intentionally make lightweight cron runs omit normal workspace bootstrap files and disable native Codex project-doc loading, while the related merged context-engine PR explicitly did not change this `lightContext` contract.

## What This Changes

Request to make `lightContext` isolated cron sessions stop losing required workspace/tool rules, or at least document and warn about the skipped bootstrap files.

## Best Possible Solution

Maintainers should decide whether lightweight cron remains an exact-command minimal mode with clearer docs/logs, or adopts a bounded critical-rules/tiered-bootstrap behavior coordinated with the existing tiered-bootstrap PR.

## Reproduction Assessment

Yes, at source level. Current main maps `lightContext: true` cron payloads to lightweight context, filters lightweight cron bootstrap files to `[]`, and disables Codex native project docs with `project_doc_max_bytes: 0`.

## Solution Assessment

Unclear. Documentation and warning logs are narrow, but injecting critical rules or changing what lightweight cron loads would change an intentionally minimal behavior and needs maintainer product direction.
