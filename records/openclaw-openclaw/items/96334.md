## Summary

Keep open. Current main still intentionally makes lightweight cron sessions omit normal workspace bootstrap files and disables Codex native project-doc loading, so the reported missing rules are source-reproducible but the remedy is a maintainer product decision.

## What This Changes

The issue asks for `lightContext` cron or isolated sessions to preserve, selectively inject, warn about, or better document critical workspace rules such as `TOOLS.md`, `AGENTS.md`, and learnings files.

## Best Possible Solution

Maintainers should decide the intended `lightContext` contract, then either keep it minimal with clearer docs/warnings or add a narrow critical-rules tier with focused cron, bootstrap, and Codex tests.

## Reproduction Assessment

Yes, at source level: current main maps cron `lightContext` to lightweight bootstrap, filters cron/default bootstrap files to empty, and sets Codex project docs to zero bytes. I did not run a live cron job because the source and tests already prove the reported missing-context path.

## Solution Assessment

Unclear. Documentation and warnings are narrow, but injecting critical rules or adding selective loading changes an intentionally minimal lightweight mode, so maintainers need to choose the product contract first.
