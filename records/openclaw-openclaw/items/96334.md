## Summary

Keep open. Current main still makes lightweight cron sessions omit normal workspace bootstrap files and disables Codex project-doc loading, but changing that behavior needs a maintainer decision because the omission is an intentional, tested lightweight-context contract.

## What This Changes

The issue requests safer `lightContext` cron behavior so isolated sessions do not lose critical workspace rules from files such as `TOOLS.md`, `AGENTS.md`, and `.learnings/LEARNINGS.md`.

## Best Possible Solution

Maintainers should decide whether lightweight cron remains an exact-command/minimal mode with clearer docs and warnings, or gains a bounded critical-rules tier that preserves token savings while keeping workspace rules available.

## Reproduction Assessment

Yes. Current source and tests show `lightContext` cron runs resolve to lightweight bootstrap, suppress normal OpenClaw bootstrap files, and set Codex project docs to zero bytes.

## Solution Assessment

Unclear. Documentation and warning logs are narrow, but always injecting critical rules or adding selective loading changes the intentionally minimal lightweight cron contract and needs maintainer direction.
