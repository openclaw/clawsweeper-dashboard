## Summary

Keep open for maintainer/product review. Current main and v2026.6.10 still intentionally make `lightContext` cron runs skip workspace bootstrap files and Codex project docs, so the reported missing-rules behavior remains real, but changing it would alter the lightweight cron contract rather than fix a clearly broken existing contract.

## What This Changes

Request to make `lightContext` isolated cron sessions retain or warn about critical workspace rules instead of omitting AGENTS/TOOLS/learnings context.

## Best Possible Solution

Maintainers should choose one explicit lightweight cron contract: keep exact empty-bootstrap mode with clearer warnings/docs, or add a bounded critical-rules/tiered mode that composes with Codex project docs and existing bootstrap-selection work.

## Reproduction Assessment

Yes, at source level. Current main maps `lightContext: true` cron jobs to lightweight bootstrap, returns no workspace bootstrap files for lightweight cron, and disables Codex AGENTS.md project docs.

## Solution Assessment

Unclear. Documentation and warnings are narrow, but injecting rules or adding selective loading changes a shipped lightweight-context contract and should be reconciled with the open bootstrap-selection work.
