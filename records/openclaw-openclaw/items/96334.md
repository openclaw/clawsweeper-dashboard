## Summary

Keep open. Current main and v2026.6.10 still intentionally make lightweight cron sessions omit workspace bootstrap files and disable Codex project-doc loading; the merged context-engine PR was adjacent and explicitly did not change this lightContext contract.

## What This Changes

The issue requests safer lightweight cron behavior so isolated sessions do not lose critical workspace rules from files like TOOLS.md, AGENTS.md, or learnings files without documentation, warnings, or a bounded loading option.

## Best Possible Solution

Maintainers should decide the lightweight cron contract explicitly: keep exact-command minimal mode with clearer docs and warnings, or add a bounded critical-rules/tiered bootstrap path with aligned source, docs, Codex behavior, and tests.

## Reproduction Assessment

Yes, at source level. Current main maps lightContext cron runs to lightweight mode, returns an empty bootstrap file list, and sets Codex project_doc_max_bytes to 0.

## Solution Assessment

Unclear until maintainers choose the product contract. Documentation or warnings are narrow, while critical-rule injection or selective loading changes an intentional lightweight cron invariant.
