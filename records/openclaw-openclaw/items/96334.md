## Summary

Keep open. Current main and the latest release still intentionally make lightweight cron sessions omit normal workspace bootstrap files and disable Codex project-doc loading; changing that behavior is a product/config contract decision, and the open related PRs do not safely own this focused issue yet.

## What This Changes

The issue asks OpenClaw to prevent lightContext cron or isolated runs from losing critical workspace rules, or at least document and warn about skipped bootstrap files.

## Best Possible Solution

Maintainers should choose the intended lightContext contract: keep minimal cron mode with clearer docs/warnings, or sponsor an explicit bounded critical-rules or tiered-bootstrap path with focused upgrade and Codex coverage.

## Reproduction Assessment

Yes. Source and tests show lightContext cron runs resolve to lightweight bootstrap, suppress normal OpenClaw bootstrap files, and set Codex project docs to zero bytes.

## Solution Assessment

Unclear. Documentation and warning logs are narrow, but injecting critical rules or adding selective loading changes a deliberate lightweight cron contract, so the best path needs maintainer direction first.
