- `impact:session-state`: The report concerns which workspace rules and memory/context files are attached to isolated agent session prompt state.

## Summary

Keep open. Current main and the latest release still make `lightContext` cron runs use an intentionally empty lightweight bootstrap path, and changing that to load rules or selective files is a product/context-contract decision rather than a narrow source-only bug fix.

## What This Changes

The issue asks OpenClaw to clarify or change how `lightContext` isolated sessions handle workspace rule files such as `TOOLS.md`, `AGENTS.md`, and `.learnings/LEARNINGS.md`.

## Best Possible Solution

Use this issue as the maintainer decision point for whether lightweight cron remains an exact-command minimal mode with clearer docs/warnings or gains an explicit bounded rules tier.

## Reproduction Assessment

Yes. Current source and tests show `lightContext` cron runs resolve to lightweight bootstrap, cron/default lightweight mode filters workspace bootstrap files to an empty set, and Codex lightweight threads set `project_doc_max_bytes` to zero.

## Solution Assessment

Unclear. Documentation and warning logs would be narrow, but always injecting critical rules or adding selective file loading changes the intentionally minimal lightweight cron contract and needs maintainer direction.
