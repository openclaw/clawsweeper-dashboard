## Summary

Keep open. Current main and v2026.6.10 still intentionally map `lightContext` cron runs to lightweight bootstrap, which omits normal workspace rule files and disables Codex native project docs; changing that behavior is a prompt/context product decision rather than a safe automatic cleanup close.

## What This Changes

The issue asks for safer `lightContext` cron or isolated sessions so workspace tool/rule files are documented, warned about, or selectively injected instead of being omitted.

## Best Possible Solution

Maintainers should choose the `lightContext` product contract: keep it as exact-command/minimal mode with clearer docs and warnings, or add a bounded critical-rules/selective-loading path that composes with the tiered-bootstrap and Codex prompt-routing work.

## Reproduction Assessment

Yes, at source level. Current main maps explicit `lightContext` to lightweight cron context, filters lightweight cron bootstrap files to an empty set, and has tests covering that behavior.

## Solution Assessment

Unclear. Documentation and warnings are narrow, but critical-rule injection or selective loading changes an intentionally tested lightweight prompt contract and should be decided alongside related bootstrap-tier and Codex prompt-routing work.
