## Summary

Keep open. Current main still has skill gating for OS, binaries, env, config, bundled allowlists, per-agent skill filters, plugin activation, and remote node binary eligibility, but it does not parse or evaluate the requested tool/toolset `requires_*` or `fallback_for_*` skill metadata. The open #52861 tracker is related to conditional prompt overhead, but it is broader and still open, so it is not a strong duplicate/supersession reason.

## What This Changes

Add AgentSkills SKILL.md metadata that conditionally shows or hides skills based on the current session's available tools and toolsets.

## Best Possible Solution

Keep this issue open as a focused core skill-loader/spec design item: define tool and toolset availability semantics, extend the skill metadata parser/types, pass effective availability into snapshot construction, add eligibility tests for `requires_*` and `fallback_for_*`, and update the public Skills docs including refresh behavior.

## Reproduction Assessment

Not applicable as a feature/spec request rather than a bug report. The current-main verification path is high-confidence source inspection: the requested fields are absent from metadata parsing, eligibility evaluation, snapshot inputs, tests, and docs.

## Solution Assessment

Unclear until maintainers define the `toolset` contract and refresh semantics. The narrowest maintainable direction is a generic core skill eligibility extension, not a one-off bundled skill or ClawHub-only workaround.
