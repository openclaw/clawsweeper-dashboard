## Summary

Keep open; current main still lacks the requested tool/toolset SKILL.md metadata fields, parser support, and eligibility inputs, and plugin/bundle skills flow through the same loader, so this remains a core skill spec/API gap rather than a ClawHub-only request.

## What This Changes

Add SKILL.md metadata gates that show or hide skills based on available tools or toolsets, including fallback-only behavior.

## Best Possible Solution

Extend the public skill metadata parser/types and eligibility context with well-defined tool/toolset availability, add requires/fallback tests, and update the Skills docs with explicit snapshot refresh semantics.

## Reproduction Assessment

Not applicable as a feature/spec request. The current absence is reproducible by source inspection: exact field search plus metadata, parser, filter, snapshot, and docs review show no tool/toolset gates on current main.

## Solution Assessment

Unclear until maintainers define tool/toolset taxonomy and refresh behavior. The maintainable direction is a core skill-loader/spec extension, not a standalone ClawHub skill, because all plugin-published skills pass through this same loader.
