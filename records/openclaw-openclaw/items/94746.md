## Summary

Keep open: this is a focused, plausible fix for the linked doctor note wrapping bug, and current main/latest release still have the vulnerable Clack second-wrap path. It should not be closed as superseded because the only other open candidate PR is not a clearly safer proof-positive landing path.

## What This Changes

This PR changes terminal-core `note()` to wrap once, then pass the wrapped message to Clack with a widened virtual output stream so copy-sensitive paths and URLs are not hard-wrapped again.

## Best Possible Solution

Land one canonical shared terminal-core fix for https://github.com/openclaw/openclaw/issues/94730; if this branch is chosen, pair it with rendered Clack note regression coverage or explicitly accept proof-only coverage before merge.

## Reproduction Assessment

Yes. Current main routes session-lock paths through `note()`, and pinned Clack 1.4.0 hard-wraps final note content from `getColumns(output) - 6`; the linked issue and PR body also provide isolated terminal proof.

## Solution Assessment

Yes for the production fix layer: changing the shared terminal-core note helper is the narrow maintainable place because many doctor/setup notes use it. The best landing state is this mitigation plus rendered-output regression coverage, or the parallel branch if maintainers prefer its test shape.
