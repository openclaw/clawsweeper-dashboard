## Summary

Keep this PR open: current main still sends OpenClaw-wrapped note text through Clack with the normal terminal width, while this branch addresses the same canonical doctor note-wrapping bug tracked at https://github.com/openclaw/openclaw/issues/94730. It should not be cleanup-closed because no merged/current-main fix supersedes it, and the open sibling at https://github.com/openclaw/openclaw/pull/94744 is another candidate rather than a proven replacement.

## What This Changes

The PR changes `packages/terminal-core/src/note.ts` to pass pre-wrapped note text to Clack through a widened virtual output stream so copy-sensitive paths and URLs are not re-wrapped.

## Best Possible Solution

Land one canonical terminal-core note-rendering fix for https://github.com/openclaw/openclaw/issues/94730 with sufficient terminal proof and focused rendered-output regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: session-lock doctor output reaches the shared `note()` helper, and pinned Clack 1.4.0 hard-wraps final note content using `getColumns(output) - 6`. I did not run a live `openclaw doctor` scenario in this read-only review.

## Solution Assessment

Yes: the shared terminal-core note helper is the right layer because many doctor/setup/provider notes use this renderer, and the Clack contract supports widening the virtual output stream. The main remaining question is branch selection and whether maintainers require rendered-output regression coverage before merge.
