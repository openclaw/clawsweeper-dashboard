## Summary

Keep open: current main still rejects Boolean `visibleReplies` values at both config paths, and #75392 is still the open closing PR for this same failure, so this issue should remain active until that PR or an equivalent fix lands.

## What This Changes

The issue requests handling Boolean `messages.visibleReplies` and `messages.groupChat.visibleReplies` values so they do not invalidate config and disrupt channel responsiveness.

## Best Possible Solution

Land #75392 or an equivalent narrow schema-boundary compatibility fix that normalizes Boolean input at both visible-reply paths while preserving the runtime enum type and public schema/docs consistency.

## Reproduction Assessment

Yes, from source. A Boolean at either visible-replies path reaches enum-only Zod fields, and raw config validation returns issues before the gateway reload can accept the snapshot.

## Solution Assessment

Yes. Schema-boundary normalization at both fields is the narrow maintainable direction because startup, hot reload, doctor, and config mutation paths share that validation boundary; the issue should stay open until the linked PR or an equivalent fix lands.
