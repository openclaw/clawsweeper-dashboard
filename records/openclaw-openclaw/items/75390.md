## Summary

Keep open: current main still rejects boolean `visibleReplies` values at both reported config paths, and #75392 is an open linked fix PR. The invalid-config reload path matches the reported all-channels-down boundary, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

Accept boolean values for `messages.visibleReplies` and `messages.groupChat.visibleReplies` without invalidating the gateway config.

## Best Possible Solution

Review and land #75392, or an equivalent schema-boundary normalization that accepts booleans at both config paths while preserving the downstream enum contract.

## Reproduction Assessment

Yes. A boolean at either reported config path reaches enum-only Zod fields on current main, causing raw config validation to fail before the gateway can apply a usable snapshot.

## Solution Assessment

Yes. Coercing booleans at the shared config schema boundary is the narrowest maintainable fix because startup, hot reload, doctor, and config mutation paths share that validation boundary while runtime code keeps the enum type.
