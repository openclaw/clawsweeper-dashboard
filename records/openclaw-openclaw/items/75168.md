## Summary

Keep open. Current main still contains unsafe `channelLogs[channelId]` logger dereferences in channel lifecycle paths, the latest release has the same pattern, and the two referenced fix PRs are closed unmerged.

## What This Changes

The issue asks for gateway channel lifecycle logging to tolerate an absent per-channel logger so channel failures do not become unhandled `TypeError` crashes.

## Best Possible Solution

Keep this open until a small gateway fix lands that treats per-channel logger lookups as optional in async continuations and covers the missing-map race in tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to start a registered channel with no `channelLogs` entry, make `startAccount` reject or hit the stop-timeout path, and observe the current `log.error?.(...)` or `log.warn?.(...)` access throw before the channel failure is contained.

## Solution Assessment

Yes. A narrow repair in `server-channels` and, if needed, the server plugin-refresh boundary is the maintainable fix; it should guard stale async logger references and preserve or backfill logger/runtime map invariants.

## Repair Work Prompt

Fix the gateway crash reported at https://github.com/openclaw/openclaw/issues/75168. In `src/gateway/server-channels.ts`, audit every `channelLogs[channelId]` derived logger use in channel lifecycle, health-monitor, restart, cleanup, and stop-timeout paths; stale or absent per-channel logger entries must not throw. Use `log?.level?.(...)` where the lookup can be absent, and consider a small `server.impl.ts` backfill/ensure helper when deferred plugin runtime refreshes add channel plugins after `channelLogs` and `channelRuntimeEnvs` are initially built. Add focused regression coverage in `src/gateway/server-channels.test.ts` for a missing logger entry during a rejected `startAccount` path and, if feasible, the stop-timeout path. Add a user-facing `CHANGELOG.md` fix entry. Do not change channel plugin IDs, broad plugin registry semantics, dependency pins, or unrelated logger call sites whose logger is required/module-scoped.
