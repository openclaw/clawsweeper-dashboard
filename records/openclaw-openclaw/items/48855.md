## Summary

Keep open. Current main still treats Android as a companion-node platform rather than a Gateway host, and managed Gateway service support remains scoped to macOS, Linux/WSL2, and Windows; Android/Termux Gateway hosting is a real platform feature decision, not conservative cleanup. The linked AndroidForClaw project is useful external workaround context, but it does not settle OpenClaw's official support boundary.

## What This Changes

The issue requests official Android/Termux support for running the OpenClaw Gateway locally, including dashboard access and remote node connections without a separate PC or VPS.

## Best Possible Solution

Maintainers should choose and document the official Android/Termux Gateway posture: support a foreground or Termux-service Gateway path with real-device validation, or explicitly keep Android node-only while pointing users to supported hosts and external Android projects.

## Reproduction Assessment

Not applicable as a feature request. The current boundary is high-confidence from source and docs: managed Gateway service support excludes Android, and Android is documented as a companion node only.

## Solution Assessment

No for an automated repair. Android/Termux Gateway hosting requires a platform/security/product decision and real Android validation before code or docs should change.
