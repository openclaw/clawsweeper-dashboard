## Summary

Keep open. Current main still excludes Android from managed Gateway service support and documents Android as a companion node, while the linked AndroidForClaw project is useful external workaround context rather than an OpenClaw support-boundary decision.

## What This Changes

The issue requests official Android/Termux Gateway hosting, or a lightweight mobile Gateway mode/documented workaround, so Android users can run dashboard and remote Gateway functionality without a PC or VPS.

## Best Possible Solution

Maintainers should choose and document the official Android/Termux Gateway posture: support a foreground or Termux-service Gateway path with real-device validation, or keep Android node-only while pointing users to supported hosts and external Android projects.

## Reproduction Assessment

Not applicable as a feature request. The current boundary is high-confidence from source and docs: managed Gateway service support excludes Android, and Android is documented as a companion node only.

## Solution Assessment
