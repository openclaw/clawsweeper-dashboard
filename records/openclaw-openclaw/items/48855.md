## Summary

Keep open. Current main intentionally documents Android as a companion node, not a Gateway host, and the managed Gateway service resolver still supports only macOS, Linux/WSL2, and Windows; Android/Termux Gateway hosting remains a real platform feature decision rather than a closeable cleanup item.

## What This Changes

Request Android/Termux Gateway support so mobile users can run the Gateway and dashboard without a separate PC or VPS.

## Best Possible Solution

Maintainers should choose and document the official Android/Termux Gateway posture: either support a foreground/Termux-service Gateway path with tests and docs, or keep Android node-only and point users to supported hosts and external Android projects.

## Reproduction Assessment

Not applicable as a feature request. The current boundary is high-confidence from source and docs: managed Gateway service support is only darwin/linux/win32, and Android is documented as node-only.

## Solution Assessment

No for an automated fix. Adding Android/Termux Gateway hosting or deciding to document only a foreground workaround is a platform/product decision, not a narrow broken-contract repair.
