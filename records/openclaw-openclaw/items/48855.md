## Summary

Keep open. Current main and v2026.6.8 still exclude Android/Termux from managed Gateway service hosting, the official docs define Android as a companion node requiring a separate Gateway host, and the linked Hermes/AndroidForClaw project is useful external evidence rather than an OpenClaw core product decision.

## What This Changes

Request Android/Termux support for running a full OpenClaw Gateway service on a phone so mobile users can access the dashboard and remote connections without a PC or VPS.

## Best Possible Solution

Keep this as the canonical platform decision until maintainers either define a scoped Android/Termux Gateway-hosting mode with real-device validation or explicitly keep Android node-only with clearer supported-host guidance.

## Reproduction Assessment

Yes for source confirmation of the unsupported service path, but not as a strict bug reproduction. Current main still excludes Android from the managed Gateway service registry, and docs define Android as a companion node rather than a Gateway host.

## Solution Assessment

No for automation. The best route is a maintainer platform and security decision first, followed by either a deliberately scoped Android/Termux Gateway-hosting implementation or an explicit node-only documentation decision.
