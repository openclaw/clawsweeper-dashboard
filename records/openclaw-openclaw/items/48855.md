## Summary

Keep open. Current main and v2026.6.10 still do not implement Android/Termux as a managed Gateway service host; the docs define Android as a companion node requiring a Gateway elsewhere, and the linked AndroidForClaw project is useful external evidence rather than an OpenClaw core decision.

## What This Changes

The issue requests Android/Termux support for running the OpenClaw Gateway service directly on mobile devices, including dashboard access and remote node connections without a separate PC or VPS.

## Best Possible Solution

Keep this as the canonical Android/Termux Gateway-hosting platform decision until maintainers either define a scoped supported mode with real-device validation or explicitly keep Android node-only with clearer host guidance.

## Reproduction Assessment

Yes for source-level confirmation, but not as a strict bug reproduction. Current main still excludes Android from the managed Gateway service registry, and docs define Android as a companion node rather than a Gateway host.

## Solution Assessment

No for automation. The best route is a maintainer platform and security decision first, followed by either a deliberately scoped Android/Termux Gateway-hosting implementation or an explicit node-only documentation decision.
