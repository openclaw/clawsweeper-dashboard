## Summary

Keep open. Current main and the latest release still do not support Android/Termux as a Gateway host; Android is documented as a companion node, and the linked AndroidForClaw project is useful external context rather than OpenClaw core support.

## What This Changes

The issue requests support for running the OpenClaw Gateway service directly on Android/Termux so mobile-only users can use the dashboard and remote node connections without a PC or VPS.

## Best Possible Solution

Maintain this as the canonical platform decision until maintainers either keep Android node-only with clearer supported-host guidance or approve a supported Termux/Android Gateway-hosting mode with docs and real-device proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this requests a new supported platform mode. The reported unsupported service path is source-confirmed because current main still excludes Android from the managed Gateway service registry and documents Android as node-only.

## Solution Assessment

No, not as an automated implementation. The best path is a maintainer platform and security decision first, then a deliberately scoped Android/Termux Gateway-hosting implementation or an explicit node-only documentation decision.
