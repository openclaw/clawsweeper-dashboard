## Summary

Keep open. Current main and the latest release still treat Android as a companion node rather than a Gateway host, and managed Gateway service support remains limited to macOS, Linux/WSL2, and Windows; Android/Termux Gateway hosting is a real platform feature decision, not cleanup that can be closed. The linked AndroidForClaw project is useful external context, but it does not settle OpenClaw's official support boundary.

## What This Changes

The issue requests Android/Termux Gateway hosting or a documented mobile-friendly workaround so users can run the Gateway without a separate PC or VPS.

## Best Possible Solution

Maintainers should choose and document the official Android/Termux Gateway posture: support a foreground or Termux-service Gateway path with tests and docs, or explicitly keep Android node-only and point users to supported hosts and external Android projects.

## Reproduction Assessment

Not applicable as a feature request. The current boundary is high-confidence from source and docs: managed Gateway service support excludes Android, and Android is documented as a companion node only.

## Solution Assessment

No for an automated repair. Android/Termux Gateway hosting requires a platform/security/product decision and likely real-device validation before code or docs should change.
