## Summary

Keep open. Current main and v2026.6.10 still do not implement Android/Termux as a managed Gateway host: service lifecycle support remains macOS/Linux/Windows-only, while docs explicitly define Android as a companion node; the linked HermesApp/AndroidForClaw project is useful external evidence, not an OpenClaw product decision.

## What This Changes

Request Android/Termux support for running the OpenClaw Gateway and dashboard directly on a mobile device without a separate PC, VPS, or always-on host.

## Best Possible Solution

Keep this as the canonical Android/Termux Gateway-hosting platform decision until maintainers either define a scoped supported mode with real-device validation or explicitly preserve Android as node-only with clearer host guidance.

## Reproduction Assessment

Yes for source-level confirmation, but not as a strict bug reproduction. Current main still excludes Android from the managed Gateway service registry, and docs define Android as a companion node rather than a Gateway host.

## Solution Assessment

No for automation. The best route is a maintainer platform/security decision first, followed by either a deliberately scoped Android/Termux Gateway-hosting implementation or an explicit node-only documentation decision.
