## Summary

Keep open. Current main and the v2026.5.22 release still limit managed Gateway service hosting to macOS, Linux/WSL2, and Windows, while Android is documented as a companion node; the linked AndroidForClaw project is useful external context but does not settle OpenClaw core support.

## What This Changes

The issue requests Android/Termux support for running the OpenClaw Gateway service locally, including dashboard access and remote node management without a PC or VPS.

## Best Possible Solution

Keep this as the canonical platform thread until maintainers decide whether to support and validate a Termux or foreground-service Gateway mode, or explicitly keep Android node-only while linking supported hosts and external Android projects.
