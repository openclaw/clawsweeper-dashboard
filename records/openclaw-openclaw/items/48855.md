## Summary

Keep open. Current main and v2026.6.8 still exclude Android/Termux from managed Gateway service hosting, Android is documented as a companion node that requires a separate Gateway host, and AndroidForClaw is useful external evidence rather than an OpenClaw core product decision.

## What This Changes

Request Android/Termux Gateway service support so mobile-only users can host the dashboard and remote connections without a PC, VPS, or separate Linux device.

## Best Possible Solution

Keep this as the canonical platform decision until maintainers approve a scoped Android/Termux Gateway-hosting mode with real-device security validation or explicitly keep Android node-only with clearer supported-host guidance.

## Reproduction Assessment

Yes for source confirmation of the unsupported service path, but not as a bug reproduction. Current main still excludes Android from the managed Gateway service registry, and docs define Android as a companion node rather than a Gateway host.

## Solution Assessment

No for automation. The best route is a maintainer platform and security decision first, followed by either a deliberately scoped Android/Termux Gateway-hosting implementation or an explicit node-only documentation decision.
