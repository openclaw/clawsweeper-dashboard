Keep open. Current main still excludes Android from managed Gateway service adapters and documents Android as a companion node, while the linked AndroidForClaw/Hermes project is useful external context rather than an OpenClaw support decision.

## What This Changes

The issue requests official Android/Termux support for running the OpenClaw Gateway service directly on Android devices without a separate PC or VPS.

## Best Possible Solution

Maintainers should choose and document the official Android/Termux Gateway posture: support a foreground or Termux-service Gateway path with real-device validation, or explicitly keep Android node-only while pointing users to supported hosts and external Android projects.
