## Summary

Keep open. Current main and v2026.6.6 still do not support Android/Termux as a managed Gateway service host; Android remains documented as a companion node, and the linked AndroidForClaw project is useful external context rather than an OpenClaw core support decision.

## What This Changes

Requested Android/Termux gateway support so mobile-only users can run the OpenClaw gateway, dashboard, and remote node connections without a separate PC or VPS.

## Best Possible Solution

Keep this as the canonical platform decision until maintainers either approve a scoped Android/Termux Gateway-hosting mode with real-device security validation or explicitly keep Android node-only with clearer supported-host guidance.

## Reproduction Assessment

Yes for source confirmation of the unsupported service path, but not as a bug reproduction. Current main still excludes Android from the managed Gateway service registry, and docs define Android as a companion node rather than a Gateway host.

## Solution Assessment

No as an automated implementation. The best path is a maintainer platform and security decision first, then either a deliberately scoped Android/Termux Gateway-hosting implementation or an explicit node-only documentation decision.
