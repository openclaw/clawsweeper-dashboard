## Summary

Keep open. Current main and the latest release still scope managed Gateway service hosting to macOS, Linux/WSL2, and Windows while Android is documented as a companion node, so Android/Termux Gateway hosting remains a platform feature decision rather than conservative cleanup.

## What This Changes

Android/Termux users want OpenClaw Gateway hosting support on Android devices instead of requiring a PC, VPS, or separate Linux host.

## Best Possible Solution

Maintainers should choose and document the official Android/Termux Gateway posture: support a foreground or Termux-service Gateway path with real-device validation, or keep Android node-only while pointing users to supported hosts and external Android projects.

## Reproduction Assessment

Not applicable as a feature request. The current boundary is high-confidence from source, tests, docs, and latest-release inspection: managed Gateway service support excludes Android, and Android is documented as a companion node only.

## Solution Assessment

No for an automated repair. Android/Termux Gateway hosting requires maintainer platform direction and real Android validation before implementation; the safer path is an explicit support posture plus targeted validation criteria.
