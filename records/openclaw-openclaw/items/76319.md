## Summary

Keep open: the PR moves cron model selection in the right direction, but it still misses per-agent CLI runtime policy because the new helper call does not receive the effective cron agent id. Current main still lacks an equivalent fix, and the source context from #75840 remains useful for reproducing and crediting the repair.

## What This Changes

The PR rewrites compatible resolved cron model providers to configured CLI runtimes, adds two cron model-formatting regressions, and records a changelog fix.

## Best Possible Solution

Thread the effective cron agent id into model selection, pass it to the shared CLI compatibility helper after normal precedence resolves provider/model, and add per-agent regression coverage before automerge.

## Reproduction Assessment

Yes. A focused test can run cron model selection for a job targeting an `agents.list` entry with `agentRuntime.id: "claude-cli"` and an Anthropic model; the current PR shape lacks the `agentId` needed for that helper path.

## Solution Assessment

No, not yet. The helper-after-precedence approach is maintainable, but this implementation must supply the effective agent id because per-agent `agentRuntime` is not copied into `cfgWithAgentDefaults`.
