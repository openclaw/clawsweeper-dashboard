## Summary

Keep open. The latest docs-only head still adds stable plugin documentation for host-hook APIs that current main does not expose, and its cleanup matrix says restart preserves run context even though the current cleanup path clears plugin run context regardless of reason.

## What This Changes

The PR adds a host-hook examples and recipes docs page, links it into the plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Keep the recipe page aligned with shipped SDK contracts: remove or clearly mark the follow-up-only sections until their implementation lands, and update the restart cleanup table to match the host cleanup source.

## Reproduction Assessment

Yes. Compare the PR head docs at `docs/plugins/host-hooks-examples.md` with current `OpenClawPluginApi`, `buildPluginApi`, and `runPluginHostCleanup`; the documented methods and restart run-context row do not match current main.

## Solution Assessment

No, not as a standalone merge into stable docs. Splitting the large recipe page is reasonable, but the docs should either wait for the matching SDK implementation stack or use proposed/RFC wording outside the stable plugin docs flow.
