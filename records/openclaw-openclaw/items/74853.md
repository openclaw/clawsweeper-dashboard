## Summary

Keep this PR open. The docs split is useful, but the current head would publish stable plugin docs for SDK fields and methods that are still absent from current main, and one `sessions.pluginPatch` diagram contradicts the implemented response/projection path.

## What This Changes

The PR adds a host-hook examples page, links it into plugin docs navigation, updates host-hook reference prose, and adds zh-CN glossary labels.

## Best Possible Solution

Keep the recipe split, but merge it only after the matching SDK/API PRs land or move future-only sections into clearly proposed/RFC docs outside stable plugin docs.

## Reproduction Assessment

Yes. Compare the PR head docs with current main's `PluginSessionExtensionRegistration`, `OpenClawPluginApi`, `buildPluginApi`, and `sessions.pluginPatch` handler; the documented fields, methods, and response semantics are absent or different.

## Solution Assessment

No, not as a standalone merge into current main. The safer path is to wait for the implementation stack or clearly mark/remove future-only recipes before publishing them in stable docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Remove unshipped session-entry slot docs:** `docs/plugins/hooks.md:264-283`
  - body: This paragraph publishes `sessionEntrySlotKey` and `sessionEntrySlotSchema` as stable `registerSessionExtension` fields, but current `PluginSessionExtensionRegistration` does not define them and the host has no slot-mirror implementation. Plugin authors copying this will hit TypeScript/runtime dead ends unless the matching SDK code lands first or this is marked as proposed.
  - confidence: 0.95
- **[P2] Keep future workflow APIs out of stable recipes:** `docs/plugins/host-hooks-examples.md:1344`
  - body: Sections 11-14 still present `api.registerSessionAction`, `api.sendSessionAttachment`, `api.scheduleSessionTurn`, and `api.emitAgentEvent` as callable SDK methods, but none exist in current `OpenClawPluginApi` or `buildPluginApi`. Because this page is added to stable docs navigation, merging now would publish examples that cannot type-check or run.
  - confidence: 0.96
- **[P2] Align the pluginPatch response diagram:** `docs/plugins/host-hooks-examples.md:361-367`
  - body: The diagram says an unknown extension returns `404`, that the store invokes `project()` during the patch, and that Gateway returns an updated session row. Current main returns a JSON-RPC `INVALID_REQUEST`, `patchPluginSessionExtension` only writes and returns `{ ok, key, value }`, and projection happens later when session rows are built, so client implementers would learn the wrong wire behavior.
  - confidence: 0.9
