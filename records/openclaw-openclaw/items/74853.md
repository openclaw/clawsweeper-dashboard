## Summary

Keep this PR open, but do not merge it as-is against current main. The branch is useful stacked documentation, but it publishes a stable plugin docs page that still describes companion-only SDK APIs and future Control UI descriptor fields that are absent from current `OpenClawPluginApi`, and its cleanup matrix contradicts current restart cleanup behavior.

## What This Changes

This PR adds a host-hook examples and recipes documentation page, links it into the plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Keep the recipes stacked behind the implementation work, then publish the page only after the documented SDK contracts exist on main; if maintainers want this docs PR to land earlier, restrict the stable page to current-main APIs and move future recipes to clearly proposed/RFC wording outside the stable plugin docs path.

## Reproduction Assessment

Yes for the review findings: fetch the PR head docs and compare the documented methods and descriptor fields against current `OpenClawPluginApi`, `buildPluginApi`, `PluginControlUiDescriptor`, and `runPluginHostCleanup` on main. The mismatch is directly visible from source inspection without needing to run tests.

## Solution Assessment

No, not as a standalone merge into current main. Splitting the large recipe page is a reasonable review strategy, but stable public plugin docs should either match shipped SDK contracts or stay explicitly proposed until the companion implementation PRs land.

  - confidence: 0.88

  - body: The cleanup matrix says run context is preserved for `restart`, but current host cleanup still calls `clearPluginRunContext` during plugin restart cleanup.
