## Summary

Keep open. This is active stacked documentation work, and current main still does not expose several workflow APIs that the new public recipe page documents; the PR should wait for the dependent SDK implementation stack or clearly mark those sections as proposed before merge.

## What This Changes

The PR adds a public host-hook examples and recipes page, links it into the plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Keep this as stacked docs work and merge it only after #73384 and #74483 land with matching public SDK contracts, or move the ahead-of-main workflow sections into proposed/RFC status language until the APIs are actually on main.

## Reproduction Assessment

Yes for the review finding: compare the PR head quick-start lines 89-92 with `OpenClawPluginApi` and `buildPluginApi` on current main. The documented workflow methods are absent from both current-main surfaces.

## Solution Assessment

No, not as a standalone merge into current main. Splitting the large recipe page is a reasonable review strategy, but stable public docs should match the shipped SDK unless the implementation stack has landed first.
