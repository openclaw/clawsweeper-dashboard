## Summary

Keep open. Current main does not contain this docs page, and the PR head still has stable plugin docs that describe host-hook APIs and restart cleanup behavior that do not match the current SDK/runtime surface.

## What This Changes

The PR adds a host-hook examples recipe page, links it into the plugin docs navigation, and adds zh-CN glossary entries for the new page title and sidebar label.

## Best Possible Solution

Keep the docs branch stacked until the companion SDK contracts exist on main, or explicitly move future-only recipes into proposed/RFC wording and keep the stable docs page limited to APIs and cleanup semantics that are already shipped.

## Reproduction Assessment

Yes. Compare the PR head docs with current `OpenClawPluginApi`, `PluginNextTurnInjection`, and `runPluginHostCleanup`; the documented workflow methods and `priority` field are absent, and restart cleanup currently clears run context.

## Solution Assessment

No, not as a standalone merge into current main. Splitting the large recipe page is a reasonable review tactic, but stable public docs should either match the current SDK or clearly mark unshipped follow-up sections as proposed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Keep future workflow APIs out of stable docs:** `docs/plugins/host-hooks-examples.md:1290`
  - body: This page is added to the stable plugin docs navigation, but sections 11-14 still document `api.registerSessionAction`, `api.sendSessionAttachment`, `api.scheduleSessionTurn`, and `api.emitAgentEvent`. Those methods are not present on current `OpenClawPluginApi` or `buildPluginApi`, so readers will copy examples that do not type-check unless the implementation stack lands first or these sections are marked as proposed.
  - confidence: 0.94
- **[P2] Correct the restart run-context cleanup row:** `docs/plugins/host-hooks-examples.md:1875`
  - body: The cleanup matrix says `restart` preserves run context, but `runPluginHostCleanup` calls `clearPluginRunContext` when cleaning a plugin, and `cleanupPreviousHostHookRegistry` reaches that path with `reason: "restart"`. This misleads plugin authors into relying on per-run scratch state surviving plugin restart; update the table/diagrams or change the implementation to match.
  - confidence: 0.9
