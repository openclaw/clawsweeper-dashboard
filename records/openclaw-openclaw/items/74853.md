## Summary

Keep open. This is useful stacked docs work, but it is not mergeable against current main because it publishes stable plugin docs for SDK methods and Gateway shapes that current main does not expose, and the PR body says it should land only after the open implementation stack.

## What This Changes

The branch adds a host-hook examples and recipes docs page, links it into the Building plugins navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Keep this PR stacked behind the implementation PRs, then update the recipes to match the exact shipped SDK and Gateway wire contracts before publishing the page in the stable plugin docs navigation.

## Reproduction Assessment

Not applicable as a runtime reproduction. There is a high-confidence static reproduction: compare the PR head docs against current `OpenClawPluginApi`, `buildPluginApi`, `PluginSessionExtensionProjection`, `sessions.pluginPatch`, and prompt-build helper implementations on main.

## Solution Assessment

No, not as a standalone merge against current main. Splitting the docs is a reasonable review strategy, but stable public docs should either wait for the implementation stack or clearly mark future APIs and avoid current-main wire-shape mismatches.

- **[P2] Do not publish recipes for APIs not on main:** `docs/plugins/host-hooks-examples.md:89-92`
  - body: This page is added to stable plugin docs, but the quick-start lists `api.registerSessionAction`, `api.sendSessionAttachment`, `api.scheduleSessionTurn`, and `api.emitAgentEvent`, which current `OpenClawPluginApi` and `buildPluginApi` do not expose.   - confidence: 0.94
- **[P2] Show pluginExtensions as the Gateway array shape:** `docs/plugins/host-hooks-examples.md:138-140`
  - body: Current session rows expose `pluginExtensions` as an array of `{ pluginId, namespace, value }` entries, but the doc describes `pluginExtensions[pluginId][namespace]`. That would send client authors toward a lookup shape the Gateway does not return unless they derive it themselves.
  - confidence: 0.91
- **[P2] Correct pluginPatch response and projection timing:** `docs/plugins/host-hooks-examples.md:315-317`
  - body: The diagram says `sessions.pluginPatch` invokes `project()` and returns an updated session row, but the current handler returns only `{ ok, key, value }` and projection happens when session rows are later rebuilt after `sessionsChanged`. Please update the diagram/text to match the actual RPC behavior.
  - confidence: 0.9
- **[P2] Mention pluginPatch requires admin scope:** `docs/plugins/host-hooks-examples.md:263-265`
  - body: This section presents `sessions.pluginPatch` as generally available to Control UI, mobile, desktop, and internal clients, but the current Gateway handler rejects callers without `operator.admin`. The docs should state that scope requirement so plugin authors do not build under-scoped clients.
  - confidence: 0.89
- **[P2] Remove priority from next-turn injection docs:** `docs/plugins/host-hooks-examples.md:375-382`
  - body: The contract excerpt includes `priority?: number` and the text says priority affects ordering, but current `PluginNextTurnInjection` has no priority field and the drain path sorts by `createdAt`. Unless the implementation stack lands that field first, this docs page overstates the shipped contract.
  - confidence: 0.9
- **[P2] Correct agent_turn_prepare trigger guidance:** `docs/plugins/host-hooks-examples.md:589-590`
  - body: The pitfall says `agent_turn_prepare` runs only on user-initiated turns, but current prompt building calls it whenever the hook exists; only `heartbeat_prompt_contribution` is heartbeat-gated. Plugins that need user-only behavior should check `ctx.trigger` themselves.
  - confidence: 0.88
- **[P3] Nest pattern headings under their parent:** `docs/plugins/host-hooks-examples.md:2447-2449`
  - body: `## Patterns and pitfalls` is immediately followed by more `##` headings, so the page TOC treats them as siblings rather than subsections. Demote the pattern entries to `###` if they are intended to live under that section.
  - confidence: 0.77
