## Summary

Keep this PR open. The docs split is useful, but it is still stacked work: the new stable docs page documents workflow SDK methods that are not present on current main, while the implementation PRs the body says must land first are still open.

## What This Changes

The PR adds a host-hook examples and recipes docs page, links it into the plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Publish the recipe page only after the dependent SDK implementation stack has landed and the public API/types/generated contracts include the documented workflow methods; otherwise keep future method sections in RFC/proposed docs with explicit status language.

## Reproduction Assessment

Yes for the review finding. The high-confidence static reproduction is to compare `docs/plugins/host-hooks-examples.md:89-92` at the PR head against `OpenClawPluginApi` and `buildPluginApi` on current main.

## Solution Assessment

No, not as a standalone merge against current main. Splitting the large recipe page is reasonable, but stable docs should match shipped SDK surfaces unless the missing APIs land first.
