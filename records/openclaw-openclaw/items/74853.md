Full review comments:

- **[P2] Remove unshipped session-entry slot docs:** `docs/plugins/hooks.md:264-283`
  - body: This publishes `sessionEntrySlotKey` and `sessionEntrySlotSchema` as stable `registerSessionExtension` fields, but current `PluginSessionExtensionRegistration` does not define them. Plugin authors copying this would hit TypeScript/runtime dead ends unless the matching SDK code lands first or the section is marked proposed.
  - confidence: 0.95
- **[P2] Keep future workflow APIs out of stable recipes:** `docs/plugins/host-hooks-examples.md:1344`
  - body: Sections 11-14 present `api.registerSessionAction`, `api.sendSessionAttachment`, `api.scheduleSessionTurn`, and `api.emitAgentEvent` as callable SDK methods, but current `OpenClawPluginApi` does not expose them. Because this page is added to stable plugin docs, these examples cannot type-check or run on current main.
  - confidence: 0.96
- **[P2] Align the pluginPatch response diagram:** `docs/plugins/host-hooks-examples.md:361-367`
  - body: The diagram says an unknown extension returns `404`, the store invokes `project()` during the patch, and Gateway returns an updated session row. Current main returns a JSON-RPC `INVALID_REQUEST`, writes and returns `{ ok, key, value }`, and projects later when session rows are built.
  - confidence: 0.92
- **[P2] Correct restart run-context cleanup:** `docs/plugins/host-hooks-examples.md:1929`
  - body: The cleanup matrix says `restart` preserves run context, but `runPluginHostCleanup` clears plugin run context whenever a plugin id or run id is supplied, including restart. That would teach plugin authors to keep restart-sensitive state in a place the host actually clears.
  - confidence: 0.9
