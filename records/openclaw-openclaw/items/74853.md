## Summary

Keep open. The latest PR head fixed some earlier review comments, but it still adds stable plugin docs that describe follow-up-only host-hook APIs absent from current main; the PR body also says the implementation stack should land first, so this is not conservative-cleanup closeable.

## What This Changes

The branch adds a host-hook examples and recipes docs page, links it into the plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Stable plugin docs should match the SDK available at merge time; the recipe page can land after the dependent SDK contracts merge, or the ahead-of-main sections should be clearly removed or moved to proposed wording until then.

## Reproduction Assessment

Yes. Compare the PR head docs around `docs/plugins/host-hooks-examples.md:1290` with current main's `OpenClawPluginApi`, `buildPluginApi`, and `PluginNextTurnInjection`; the documented methods and `priority` field are absent on main.

## Solution Assessment

No as a standalone merge into current main. Splitting the recipe page is a reasonable review strategy, but publishing it in stable plugin docs before the matching SDK stack lands would overclaim the public API.
