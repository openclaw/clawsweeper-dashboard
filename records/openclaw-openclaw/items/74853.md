## Summary

Keep open: the current PR head still publishes stable plugin docs for SDK fields and workflow methods that are not present on current main, so merging it now would give plugin authors examples that do not type-check or match runtime cleanup behavior.

## What This Changes

The PR adds a host-hook examples page, links it into plugin docs navigation, adds zh-CN glossary labels, and updates plugin SDK docs for session-entry slot mirrors.

## Best Possible Solution

Keep this as stacked docs work until the matching SDK contracts are on main, or pare the page back to shipped host-hook surfaces and move future-only seams to clearly proposed/RFC documentation.

## Reproduction Assessment

Yes for review reproduction: compare the PR head docs against current `OpenClawPluginApi`, `PluginSessionExtensionRegistration`, `PluginNextTurnInjection`, and `runPluginHostCleanup` on main. The documented fields/methods and restart behavior do not match current source.

## Solution Assessment

No, not as a standalone merge into current main. Splitting the recipe page is a reasonable review tactic, but stable public docs should either match shipped SDK contracts or be clearly marked as proposed until the implementation stack lands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Keep session-entry slot docs out until the API exists:** `docs/plugins/hooks.md:264-265`
  - body: This stable docs section describes `sessionEntrySlotKey` and `sessionEntrySlotSchema`, but current `PluginSessionExtensionRegistration` does not define those fields and the host has no mirror/cleanup path for them. If published now, plugin authors will copy options that are not part of the SDK contract.
  - confidence: 0.95
- **[P2] Remove unavailable workflow API recipes:** `docs/plugins/host-hooks-examples.md:1344-1355`
  - body: Sections 11-14 still document `api.registerSessionAction`, `api.sendSessionAttachment`, `api.scheduleSessionTurn`, and `api.emitAgentEvent`, none of which exist on current `OpenClawPluginApi`. Readers following these recipes will hit immediate TypeScript/runtime dead ends unless the implementation PRs land first or the sections are moved to proposed docs.
  - confidence: 0.96
- **[P2] Correct restart cleanup semantics:** `docs/plugins/host-hooks-examples.md:1927-1929`
  - body: The cleanup matrix says restart preserves run context, but `runPluginHostCleanup` still calls `clearPluginRunContext` whenever a plugin or run id is supplied. Publishing this table would give workflow plugin authors the wrong persistence contract for restart paths.
  - confidence: 0.89
