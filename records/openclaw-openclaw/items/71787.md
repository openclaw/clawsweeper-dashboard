## Summary

Keep this PR open. Current main still does not accept the nested Schema 2.0 operator identity shape this branch targets, while the PR is narrow, has focused lifecycle coverage, addresses the earlier dead-guard and changelog feedback, and has no blocking correctness finding from this read-only review.

## What This Changes

The PR updates Feishu card-action payload normalization to read nested Schema 2.0 identity fields, adds lifecycle coverage for that callback shape, and updates the changelog attribution.

## Best Possible Solution

Land the narrow Feishu parser, lifecycle-test, and changelog update after normal maintainer validation, while keeping instant callback response/card disabling in the separate #71677 enhancement thread.

## Reproduction Assessment

Yes. On current main, a `card.action.trigger` payload whose only operator open ID is `operator.user_id.open_id` leaves `openId` undefined, causing the parser to return null and the handler to drop the callback before dispatch.

## Solution Assessment

Yes. Normalizing Schema 2.0 identity fields at the Feishu monitor parser boundary is the narrowest maintainable fix because downstream card-action handling already consumes the normalized `FeishuCardActionEvent` and the PR keeps the existing token/openId/tag/action-value validation.
