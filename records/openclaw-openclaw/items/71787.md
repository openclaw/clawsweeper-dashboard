## Summary

Keep this PR open. Current main accepts `context.open_chat_id` but still misses nested Schema 2.0 identity objects, while the PR adds the narrow parser normalization, lifecycle coverage, and changelog credit requested in review; I did not find a blocking correctness or security issue.

## What This Changes

The PR updates the Feishu card-action parser to normalize Schema 2.0 nested identity objects, adds lifecycle coverage for that callback shape, and expands the Feishu changelog entry.

## Best Possible Solution

Land this narrow Feishu parser, lifecycle-test, and changelog update after normal maintainer validation, keeping the instant card-update UX enhancement tracked separately in #71677.

## Reproduction Assessment

Yes. On current main, a `card.action.trigger` payload whose only operator open ID is `operator.user_id.open_id` leaves `openId` undefined, so the parser returns null and the monitor handler drops the callback before dispatch.

## Solution Assessment

Yes. Normalizing Schema 2.0 identity fields at the Feishu monitor parser boundary is the narrowest maintainable fix because downstream card-action handling already consumes the normalized `FeishuCardActionEvent` and the PR keeps the existing token/openId/tag/action-value validation.
