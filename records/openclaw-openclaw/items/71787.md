This PR broadens the Feishu card-action parser to read nested Schema 2 `operator.user_id.{open_id,user_id,union_id}` values and adds lifecycle coverage for that callback shape.

## Best Possible Solution

Land the parser normalization and focused lifecycle regression test after CI is green, while keeping the instant card update UX work in https://github.com/openclaw/openclaw/issues/71677 separate.

## Solution Assessment
