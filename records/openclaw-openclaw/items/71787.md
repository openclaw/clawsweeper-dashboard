## Summary

Keep open. Current main still accepts `context.open_chat_id` only when a flat operator open ID is present, while this PR adds the remaining Schema 2.0 nested identity normalization, lifecycle coverage, and changelog attribution with no blocking review findings.

## What This Changes

The PR updates Feishu card-action payload normalization to read nested Schema 2.0 identity fields, adds lifecycle coverage for that callback shape, and expands the changelog entry.

## Best Possible Solution

Land the narrow Feishu parser, lifecycle-test, and changelog update after standard maintainer validation, while leaving instant callback response/card disabling in the separate #71677 enhancement thread.

## Reproduction Assessment

Yes. On current main, a `card.action.trigger` payload whose only operator open ID is `operator.user_id.open_id` leaves `openId` undefined, causing the parser to return null and the handler to drop the callback before dispatch.

## Solution Assessment

Yes. Normalizing Schema 2.0 identity fields at the Feishu monitor parser boundary is the narrowest maintainable fix because downstream handling already consumes the normalized `FeishuCardActionEvent` and the PR preserves the existing validation guard.
