## What This Changes

The issue reports a false positive `openclaw doctor` Gateway auth warning when `gateway.auth.token` is a resolving SecretRef.

## Best Possible Solution

Keep the shipped SecretRef-aware Doctor behavior for locally resolvable refs, and use https://github.com/openclaw/openclaw/issues/88022 for any remaining runtime-healthy but Doctor-unavailable auth-policy work.

## Reproduction Assessment

Yes. The issue comments describe a source-level repro where old Doctor code warned before materializing a resolving Gateway token SecretRef, and current tests now cover the no-warning case for a resolving SecretRef.

## Solution Assessment

Yes for this report's resolving-SecretRef case. The shipped fix uses the existing Gateway auth token resolver in both Doctor paths and preserves fail-closed diagnostics for unresolved refs.
