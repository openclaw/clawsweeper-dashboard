## Summary

Current main already fixes the false Gateway auth SecretRef warning: both doctor lint and interactive doctor now resolve `gateway.auth.token` through the SecretRef-aware gateway token resolver before emitting the unavailable warning, and the fixed source is present in the latest release.

## What This Changes

The issue asks `openclaw doctor` to stop warning that `gateway.auth.token` is unavailable when that token is a resolving SecretRef.

## Best Possible Solution

Close this issue as fixed by the shipped SecretRef-aware doctor gateway-auth implementation; separate healthy-gateway fallback policy requests should remain in their own canonical issue.

## Reproduction Assessment

No current-main failure path remains: source inspection shows doctor lint and interactive doctor return before warning when `gateway.auth.token` resolves as a SecretRef. Older reporter comments established the pre-fix false positive on 2026.4 and 2026.5 releases.

## Solution Assessment

Yes. Reusing the shared `resolveGatewayAuthToken()` path in both doctor surfaces is the narrow maintainable fix because it removes resolved-SecretRef false positives while preserving unresolved warnings and default no-exec behavior.
