## What This Changes

Doctor was warning that a resolvable `gateway.auth.token` SecretRef was unavailable even though secrets audit/reload and Gateway runtime auth were healthy.

## Best Possible Solution

Keep the shipped SecretRef-aware Doctor behavior for resolvable Gateway token refs and leave the broader runtime-healthy but CLI-unavailable policy question to https://github.com/openclaw/openclaw/issues/88022.

## Reproduction Assessment

No current-main failing reproduction remains. The affected release path is clear from source and reporter evidence, but current main and release source return no Doctor finding when `gateway.auth.token` resolves as a SecretRef.

## Solution Assessment

Yes. The landed fix uses the existing SecretRef-aware Gateway token resolver in both Doctor paths and preserves fail-closed diagnostics for genuinely unresolved refs rather than adding a parallel suppression path.
