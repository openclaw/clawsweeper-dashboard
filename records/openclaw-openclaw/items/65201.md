## What This Changes

The issue reports a false `openclaw doctor` warning for `gateway.auth.token` when the token is SecretRef-managed and already resolves via secrets audit/reload and Gateway runtime.

## Best Possible Solution

Keep the shipped SecretRef-aware doctor fix; use https://github.com/openclaw/openclaw/issues/88022 for the narrower remaining policy question where Gateway runtime is healthy but the doctor command path cannot resolve the SecretRef.

## Reproduction Assessment

Yes for the original shipped bug: the issue comments give stable-release reproductions with resolvable SecretRefs, and current source/tests show the same path now returns no finding. A failing current-main reproduction is not expected because the fix is present.

## Solution Assessment

Yes. The current fix uses the existing Gateway token SecretRef resolver in both lint and interactive doctor paths, skips exec execution by default, and preserves warnings for truly unresolved configured refs.
