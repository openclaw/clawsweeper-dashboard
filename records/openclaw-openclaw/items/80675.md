## Summary

Keep open: the bug direction is plausible, but the patch re-emits partial assistant text on prompt timeout and the supplied proof does not show the actual surfaced timeout payload/UI result.

## What This Changes

The PR changes the embedded PI agent prompt-timeout branch to always append a timeout error payload while preserving earlier payloads.

## Best Possible Solution

Preserve prior tool/media output on prompt timeout, keep suppressing incomplete assistant fragments, and add focused runner coverage for both tool-success timeout and partial-fragment timeout.

## Reproduction Assessment

Yes. Source inspection shows the PR prepends payloadsWithToolMedia while buildEmbeddedRunPayloads turns non-empty assistantTexts into payloads, and existing coverage defines that partial fragments must be suppressed after timeout.

## Solution Assessment

No. The direction is close, but the implementation should filter partial assistant fragments from preserved payloads and keep only successful tool/media output before the timeout error.
