## What This Changes

The PR updates `resolveReplyDirectives` to keep queue directives out of the generic inline cleanup trigger and restore saved queue parse metadata when other inline directives cause cleanup.

## Best Possible Solution

Land the resolver fix after adding targeted regression coverage and contributor-supplied redacted Telegram or equivalent runtime proof that queue directives and mixed queue options survive inline cleanup.

Yes. Source inspection on current main shows `/queue steer <prompt>` can parse queue metadata, then the queue directive itself triggers the trailing-text cleanup path that clears that metadata before per-message queue mode is applied.

## Solution Assessment

Mostly yes. Excluding queue from the generic cleanup trigger and restoring the full queue parse state when another inline directive forces cleanup is the narrow maintainable fix; the remaining gaps are proof and regression coverage, not a different code direction.
