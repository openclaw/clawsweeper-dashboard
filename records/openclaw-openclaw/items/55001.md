## What This Changes

The PR changes reply directive resolution so `/queue ... <prompt>` directives are preserved through inline-directive cleanup instead of being stripped before queue settings are applied.

## Best Possible Solution

Yes. On current main, `/queue steer <prompt>` parses as a queue directive with cleaned prompt text, then `hasInlineDirective` includes queue and `clearInlineDirectives` drops the queue state before per-message queue settings are derived.

## Solution Assessment

No, not as currently submitted. The save/restore approach is the narrow maintainable fix, but the branch needs a current-main rebase that keeps trace handling plus focused tests and a single-line changelog entry.
