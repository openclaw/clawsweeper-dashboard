## Summary

Keep open: the PR fixes the Feishu multi-bot self-mention direction, but it still lets the preserved self-mention flow into command dispatch bodies, so existing direct `@Bot /new` or `@Bot /reset` turns can be authorized by the stripped probe and then miss native command parsing.

## What This Changes

The PR preserves Feishu self-mention `<at>` tags in parsed message content, normalizes command-probe input for group and p2p, and updates Feishu mention parser tests for multi-bot groups.

## Best Possible Solution

Preserve self-mention tags in the Feishu agent-visible body, pass a mention-stripped body into `commandBody`/`BodyForCommands`, and add redacted live Feishu proof before merge.

## Reproduction Assessment

Yes, source-level: current main passes `botOpenId` into `normalizeMentions`, which strips the receiving bot's own mention, and the PR adds a synthetic multi-bot parser case. I did not run a live Feishu tenant reproduction.

## Solution Assessment

No, not as submitted. Preserving self-mention context for the model is the right direction, but the best fix must separate the model-visible body from slash-clean command bodies instead of only normalizing the authorization probe.
