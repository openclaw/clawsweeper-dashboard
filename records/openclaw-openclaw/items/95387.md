## Summary

Keep open: the branch fixes the earlier `commandBody` regression, but preserving self-mentions in parsed Feishu content also leaves mention-prefixed `/stop` and `/btw` on the normal sequential lane because `getFeishuSequentialKey` still reads `parsed.content`. The PR also still has only unit-test proof, not redacted live Feishu behavior proof.

## What This Changes

The PR preserves Feishu self-mention `<at>` tags in parsed message content, normalizes command probing for both group and p2p, routes `commandBody` through the stripped probe body, and updates Feishu mention parser tests.

## Best Possible Solution

Preserve self-mention tags for the model-visible Feishu body, derive command execution and sequential-key control text from the same mention-stripped Feishu command body, add `/stop` and `/btw` regression coverage, then provide redacted live Feishu proof before merge.
