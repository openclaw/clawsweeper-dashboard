## Summary

Keep open: current main still has the reported native subagent `cwd` gap, but this PR remains unmergeable because it makes public `cwd` an unvalidated child workspace, rewrites unrelated prompt snapshots, and still has no after-fix real behavior proof.

## What This Changes

The PR threads `sessions_spawn.cwd` into native subagent spawning, adds a workspace-inheritance regression test and changelog entry, and updates Codex prompt snapshot fixtures.

## Best Possible Solution

Implement native subagent `cwd` as a bounded workspace override: canonicalize and allow only approved requester/target workspace roots, remove unrelated fixture drift, add tool-level and spawn-level regression coverage, and require a redacted native canary proof.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd`, forwards it only to ACP, and native cross-agent spawns resolve to the target agent workspace because `spawnSubagentDirect` receives no `cwd`. I did not run a live Telegram/macOS canary in this read-only review.

## Solution Assessment

No. The PR fixes the missing parameter threading, but it is not the best mergeable form because it trusts model-supplied `cwd` as the persisted child workspace and includes unrelated prompt snapshot rewrites.
