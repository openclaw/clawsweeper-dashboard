## Summary

Keep open: this PR targets a real native `sessions_spawn.cwd` bug, but the current head is not merge-ready because it trusts the tool-supplied cwd as persisted workspace state, carries unrelated prompt snapshot drift, and still lacks after-fix real behavior proof.

## What This Changes

The PR threads `sessions_spawn.cwd` into native `spawnSubagentDirect`, adds a workspace-inheritance regression test and changelog entry, and rewrites Codex prompt snapshot fixtures.

## Best Possible Solution

Implement native subagent cwd as a bounded workspace override, remove unrelated fixture drift, add focused tool/spawn regression coverage, and require redacted native canary proof.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd`, forwards it only to ACP, and native cross-agent spawns resolve to the target agent workspace because `spawnSubagentDirect` receives no cwd. The linked issue at https://github.com/openclaw/openclaw/issues/79840 also includes a real 2026.5.6 canary shape with before/after output.

## Solution Assessment

No. Passing cwd through fixes the missing parameter threading, but the mergeable solution should canonicalize and constrain the native cwd to approved requester/target workspace roots and drop unrelated generated fixture churn.
