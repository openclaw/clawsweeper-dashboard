## Summary

Keep open: this addresses a valid native subagent cwd bug, but the current PR still trusts a tool-supplied cwd as the persisted spawned workspace, includes unrelated prompt snapshot drift, and lacks after-fix real behavior proof.

## What This Changes

The branch threads `sessions_spawn.cwd` into native subagent spawning, adds a helper-level workspace inheritance test, updates the changelog, and regenerates Codex prompt snapshots.

## Best Possible Solution

Honor native cwd through a bounded internal workspace override, canonicalize it against approved requester/target workspace roots, keep workspace metadata out of public `agent` params, remove unrelated snapshot churn, and add focused regression plus redacted live proof.

## Reproduction Assessment

Yes, at source level: current main exposes and reads `cwd`, forwards it to ACP, but native cross-agent subagent spawning omits it and resolves to the target agent workspace. I did not run the macOS/Telegram canary in this read-only review.

## Solution Assessment

No. Passing the parameter is the right direction, but the mergeable fix must constrain and canonicalize native cwd before persisting it as a spawned workspace and should drop unrelated prompt snapshot rewrites.
