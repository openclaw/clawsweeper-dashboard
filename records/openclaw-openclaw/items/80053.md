## Summary

Keep open: the bug is source-reproducible and the PR moves in the right direction, but current head still has a workspace-boundary blocker, unrelated prompt snapshot churn, and no after-fix real behavior proof.

## What This Changes

The PR adds native subagent `cwd` plumbing for `sessions_spawn`, a workspace-inheritance regression test, a changelog entry, and regenerated Codex prompt snapshots.

## Best Possible Solution

Thread native `cwd` through an internal, canonicalized workspace override limited to approved requester/target roots, remove unrelated snapshot churn, and merge only after focused tests plus redacted live proof.

## Reproduction Assessment

Yes, at source level: current main reads `cwd` and passes it to ACP, but the native `spawnSubagentDirect` call has no `cwd` field and cross-agent inheritance passes `undefined`, so the target agent workspace wins. I did not run the macOS/Telegram canary.

## Solution Assessment

No. Passing the parameter is the right direction, but the patch should validate and canonicalize native `cwd` before persistence and should drop unrelated prompt snapshot rewrites.
