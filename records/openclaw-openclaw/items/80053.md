## Summary

Keep open: the bug is real and the PR targets the native subagent path, but the current diff still trusts tool-supplied `cwd` as the persisted child workspace, rewrites unrelated Codex prompt snapshots that no longer match the generator, and lacks after-fix real behavior proof.

## What This Changes

The PR forwards `sessions_spawn.cwd` into native subagent spawning, adds a workspace-inheritance unit test and changelog entry, and rewrites Codex prompt snapshots.

## Best Possible Solution

Thread native `cwd` through a bounded, canonicalized workspace override limited to approved requester/target roots, add an integration-level regression for tool-to-native spawn behavior, keep prompt snapshots aligned with actual generator output, and merge after redacted live proof.

## Reproduction Assessment

Yes, at source level: current main reads `cwd` and forwards it to ACP, while native spawn omits `cwd` and cross-agent inheritance resolves to the target workspace. I did not run the live macOS/Telegram canary.

## Solution Assessment

No. Passing `cwd` through is the right direction, but the implementation should canonicalize and constrain the workspace override and remove unrelated prompt snapshot churn before merge.
