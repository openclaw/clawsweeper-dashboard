## Summary

Keep this PR open: the reported `sessions_spawn.cwd` bug is valid, but the branch still makes a public tool argument the native child workspace without validation, carries incorrect Codex prompt snapshot churn, and lacks after-fix real behavior proof.

## What This Changes

The PR adds native subagent `cwd` plumbing for `sessions_spawn`, a workspace-inheritance unit test, a changelog entry, and regenerated Codex prompt snapshots.

## Best Possible Solution

Implement native `cwd` as a bounded workspace override, preserve generated Codex prompt snapshots that match current code, and land with focused regression tests plus a redacted native canary proof.

## Reproduction Assessment

Yes, source-level: current main defines and reads `sessions_spawn.cwd`, forwards it to ACP, and omits it from native `spawnSubagentDirect`, whose cross-agent path resolves to the target agent workspace. I did not run a live native subagent canary in this read-only review.

## Solution Assessment

No. Threading the parameter is the right bug-fix direction, but this implementation is not the best mergeable form because it lacks workspace-boundary validation and includes unrelated prompt snapshot changes.

## Review Findings
