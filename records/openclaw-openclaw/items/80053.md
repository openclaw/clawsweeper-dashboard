## Summary

Keep open: this is a real source-level bug and the cwd threading is pointed at the right path, but the current PR passes model-supplied `cwd` into the internal spawned-workspace override without validation, includes a basename path regression in the script cleanup, and still has only unit-test proof.

## What This Changes

The PR threads `cwd` through the native `sessions_spawn` subagent path, adds a workspace-precedence test, updates the changelog, and changes one Telegram credential helper string call.

## Best Possible Solution

Honor native subagent `cwd` only after normalizing it into a trusted spawned workspace fact, preserving the internal-only workspace override boundary and then proving a live child observes the requested cwd.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd` and ACP forwards it, while the native `spawnSubagentDirect` path drops it before workspace inheritance. The linked issue also provides a concrete cross-agent canary shape.

## Solution Assessment

No, not as written. Adding `cwd` to the native params is the right direction, but the PR should validate the explicit cwd against the trusted spawned-workspace boundary and fix the unrelated script path regression before merge.

  - confidence: 0.9
