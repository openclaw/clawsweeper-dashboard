## Summary

Keep open: current main still preflights only the resolved cron primary model and returns `skipped` before the configured fallback chain can reach the model fallback runner. The closed, unmerged fix attempt and the follow-up production report reinforce the same narrow bug rather than superseding it.

## What This Changes

Cron should try configured model fallbacks before skipping an isolated cron run when the local primary provider preflight is unavailable.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, while preserving strict fallback overrides and alias-aware model resolution.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an isolated cron run with an unreachable local primary returns `skipped` in `prepareCronRunContext` before `createCronPromptExecutor` constructs `runWithModelFallback`.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware cron preflight before entering the runner, not disabling preflight or adding another config option.
