## Summary

Keep open: current main still preflights only the resolved primary cron model and returns `skipped` on local-provider unavailability before the configured fallback chain reaches the executor. The direct fix attempt at https://github.com/openclaw/openclaw/pull/79335 is closed unmerged, and the newer retry-config PR at https://github.com/openclaw/openclaw/pull/82145 is related but deliberately does not solve chain-aware fallback preflight.

## What This Changes

Cron isolated-agent model preflight should try configured fallback models before skipping a scheduled run when the local primary provider is unreachable.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, while preserving payload `fallbacks`, explicit empty fallback lists, alias-aware model resolution, and endpoint probe caching.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure an isolated cron agent with an unreachable local primary and a cloud fallback; current main returns `skipped` in `prepareCronRunContext` before `runWithModelFallback` is constructed.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware preflight using the existing fallback policy/resolution semantics, not disabling preflight or adding another retry/config knob.
