## Summary

Keep open: current main still preflights only the resolved primary model and returns `skipped` before cron fallback execution can run, while the cron docs say configured fallback chains apply. The related fix PR was closed unmerged, and there is no open replacement PR from the searches I ran.

## What This Changes

Make isolated cron model preflight honor configured model fallbacks before skipping a scheduled run when the local primary provider is unreachable.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, preserving explicit `fallbacks: []` and the local endpoint cache.

## Reproduction Assessment

Yes, source-level: an isolated cron run with an unreachable local primary reaches `run.ts` preflight and returns `skipped` before `runWithModelFallback` is constructed. I did not run a live cron outage because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware preflight using the existing cron fallback policy, not disabling preflight or adding a separate config knob.
