## Summary

Keep open: current main still runs isolated cron local-provider preflight against only the resolved primary and returns `skipped` before the configured fallback chain can reach `runWithModelFallback`. The closed, unmerged fix attempt at https://github.com/openclaw/openclaw/pull/79335 is useful source context, and the later production comment reinforces the same failure mode rather than superseding it.

## What This Changes

The issue asks cron isolated-agent preflight to attempt configured model fallbacks before skipping a run when the selected local primary provider is unreachable.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, preserving `fallbacks: []`, alias-aware model ref resolution, and local endpoint probe caching.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an isolated cron run with an unreachable local primary returns `skipped` in `prepareCronRunContext` before the fallback runner is constructed.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware preflight before entering the runner, not disabling preflight or adding a separate config option.
