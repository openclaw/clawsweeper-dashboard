## Summary

Keep open: current main still performs cron local-model preflight against only the resolved primary provider/model and returns `skipped` before the configured fallback chain reaches the executor. The linked direct fix was closed unmerged, and the open retry-config PR addresses cold-start retries rather than this chain-aware fallback bug.

## What This Changes

Cron isolated-agent runs should attempt configured model fallbacks before skipping when the primary local provider is unreachable during model preflight.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, preserving payload fallbacks, explicit empty fallback lists, alias-aware model resolution, and endpoint probe caching.

## Reproduction Assessment

Yes at source level. Configure an isolated cron agent with an unreachable local primary and a cloud fallback; current main returns `skipped` in `prepareCronRunContext` before `runWithModelFallback` is constructed.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware preflight using the existing cron fallback policy and model-resolution helpers; the retry-config direction in https://github.com/openclaw/openclaw/pull/82145 is useful for cold starts but does not solve fallback-chain bypass.
