## Summary

Keep open: current main still preflights only the resolved cron primary model and returns `skipped` before the configured fallback chain can reach the model fallback runner. The closed, unmerged fix attempt at https://github.com/openclaw/openclaw/pull/79335 and the later production-impact comment both support the same remaining bug rather than superseding it.

## What This Changes

Fix isolated cron model preflight so an unreachable local primary does not skip a scheduled run before trying configured fallback models.

## Best Possible Solution

Make cron preflight evaluate the effective candidate chain and skip only after every preflight-relevant candidate is unavailable, while preserving strict empty fallback lists and existing local endpoint probe caching.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an isolated cron run with an unreachable local primary returns `skipped` in `prepareCronRunContext` before `runWithModelFallback` is constructed; I did not run a live cron outage in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware preflight before returning `skipped`, not disabling preflight or adding a new configuration knob.
