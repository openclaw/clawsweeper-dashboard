## Summary

Keep open: current main still preflights only the resolved cron primary model and returns `skipped` on local-provider unavailability before the configured fallback chain reaches the executor. The direct fix PR is closed unmerged, and the open retry-config PR does not cover chain-aware fallback preflight.

## What This Changes

The issue asks isolated cron model preflight to attempt configured model fallbacks before skipping a scheduled run when the primary local provider is unreachable.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, while preserving payload `fallbacks`, explicit empty fallback lists, alias-aware model resolution, and endpoint probe caching.

## Reproduction Assessment

Yes, at source level. Configure an isolated cron agent with an unreachable local primary and a cloud fallback; current main returns `skipped` in `prepareCronRunContext` before `runWithModelFallback` and its configured fallback override are constructed.

## Solution Assessment

Yes. The narrow maintainable solution is chain-aware preflight using the existing cron fallback policy/resolution semantics, not disabling preflight or adding another retry/config knob.
