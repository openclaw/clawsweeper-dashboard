## Summary

Keep open: current `main` still short-circuits isolated cron execution when the selected local provider preflight is unavailable, and the only linked fix PR is closed unmerged. The issue is source-reproducible and narrow enough for a focused repair PR.

## What This Changes

The issue reports that isolated cron model preflight skips scheduled runs when the primary local provider is unreachable instead of using configured cloud fallbacks.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, while preserving payload `fallbacks: []`, model alias resolution, and local endpoint cache behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an isolated cron run with an unreachable local primary returns `skipped` in `prepareCronRunContext` before configured fallbacks reach `runWithModelFallback`.

## Solution Assessment

Yes. The narrow maintainable solution is chain-aware cron preflight before execution, not disabling preflight or adding a separate ad hoc retry path after the skip decision.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79329. Current `src/cron/isolated-agent/run.ts` probes only the resolved primary provider/model via `preflightCronModelProvider` and returns `status: skipped` before `runWithModelFallback` can use configured fallbacks. Implement chain-aware cron preflight so an unavailable local primary walks the effective fallback list and uses the first candidate whose preflight is available; skip only when all preflight-relevant candidates are unavailable. Preserve payload `fallbacks: []` strict behavior, payload fallbacks overriding configured fallbacks, model alias resolution, local endpoint cache behavior, and request-storm protection. Use the closed unmerged prior attempt https://github.com/openclaw/openclaw/pull/79335 only as context, not as a blind patch. Add focused regression coverage for local primary unavailable with cloud fallback, all local candidates unavailable, strict empty fallbacks, and alias/configured fallback resolution. Do not change unrelated cron delivery behavior or broad model fallback semantics. Add a changelog entry because this is user-visible.
