## Summary

Keep open: current main still preflights only the resolved primary model and returns `skipped` before the isolated cron fallback runner is reached. The only exact fix attempt found, https://github.com/openclaw/openclaw/pull/79335, is closed unmerged, and a later production comment reports the same failure mode.

## What This Changes

Cron isolated-agent runs should consult configured model fallbacks before skipping when the primary local provider preflight is unavailable.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, preserving `fallbacks: []`, alias-aware model resolution, and local endpoint probe caching.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an isolated cron run with an unreachable local primary returns `skipped` in `prepareCronRunContext` before `runWithModelFallback` is constructed; the issue also includes a standalone preflight probe and production trace.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware preflight before the skip path, not disabling preflight or adding a new configuration option.
