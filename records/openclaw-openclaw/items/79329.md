## Summary

Keep open: current main still preflights only the resolved primary model for isolated cron runs and returns `skipped` before the configured fallback chain can be used. The closed, unmerged fix attempt at https://github.com/openclaw/openclaw/pull/79335 remains useful context, and the later production comment supports the same failure mode rather than superseding it.

## What This Changes

The issue requests that isolated cron model preflight honor configured model fallbacks instead of skipping the run when the primary local provider is unreachable.

## Best Possible Solution

Make cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, while preserving explicit `fallbacks: []`, alias-aware model reference resolution, and endpoint probe caching.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an isolated cron run with an unreachable local primary returns `skipped` in `prepareCronRunContext` before the fallback executor is constructed.

## Solution Assessment

Yes. A chain-aware preflight before entering the runner is the narrow maintainable fix; disabling preflight or adding a config knob would weaken the local-provider protection instead of honoring the existing fallback contract.
