## Summary

Keep open: current main still preflights only the resolved cron primary model and returns `skipped` before the configured fallback chain can run. The linked attempted fix at https://github.com/openclaw/openclaw/pull/79335 is closed and unmerged, while the follow-up production comment confirms this remains operator-visible.

## What This Changes

The issue asks cron isolated-agent model preflight to try configured fallback models before recording a run as skipped when the local primary endpoint is unavailable.

## Best Possible Solution

Make isolated cron preflight evaluate the effective model candidate chain and skip only after every preflight-relevant candidate is unavailable, preserving explicit empty fallback semantics and endpoint caching.

## Reproduction Assessment

Yes. Source inspection shows an isolated cron run with an unreachable local primary returns `skipped` in `prepareCronRunContext` before `createCronPromptExecutor` can pass fallbacks to `runWithModelFallback`.

## Solution Assessment

Yes. The narrow maintainable fix is chain-aware preflight at the cron preflight boundary; disabling preflight or adding a config knob would lose the local-provider storm protection and broaden the behavior change.
