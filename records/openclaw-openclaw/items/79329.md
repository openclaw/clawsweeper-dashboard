## Summary

Keep open: current main still skips isolated cron runs during primary local-provider preflight before the configured fallback chain can run. The retry-focused draft PR and the closed broader routing issue do not supersede this narrower fallback-preflight bug.

## What This Changes

The issue asks cron isolated-agent model preflight to respect configured model fallbacks before skipping a run when a local primary provider is unreachable.

## Best Possible Solution

Make cron preflight evaluate the effective fallback candidate chain, or defer the skip until the normal fallback path is exhausted, while preserving strict `fallbacks: []` semantics and the local endpoint probe cache.

## Reproduction Assessment

Yes, at source level. An unavailable local primary makes `src/cron/isolated-agent/run.ts` return `skipped` before `src/cron/isolated-agent/run-executor.ts` constructs `runWithModelFallback`, and the reporter supplied matching standalone and production traces.

## Solution Assessment

Yes. The narrow maintainable fix is to apply existing cron fallback policy to the preflight decision before skipping; retrying the primary local preflight alone helps cold starts but does not solve configured cloud fallback bypass.
