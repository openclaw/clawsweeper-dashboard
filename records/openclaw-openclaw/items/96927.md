## Summary

This PR is still necessary: current main and v2026.6.10 still parse Comfy workflow success JSON with unbounded response.json(), while the PR moves that shared reader to the existing bounded provider JSON helper with focused wrapper tests and sufficient terminal proof. Keep it open for maintainer acceptance of the compatibility tradeoff around a hard 16 MiB cap on successful Comfy JSON responses.

## What This Changes

The PR changes Comfy workflow JSON reads to use the shared bounded provider JSON reader and adds wrapper-level oversized, success, and HTTP-error tests.

## Best Possible Solution

Land this bounded-reader change after maintainers explicitly accept the shared 16 MiB fail-closed cap for Comfy workflow success JSON, or choose a Comfy-specific limit with proof.

## Reproduction Assessment

Yes. Current main and v2026.6.10 use response.json() in the shared Comfy readJsonResponse wrapper, and the PR adds a concrete oversized ReadableStream path that exercises the same wrapper.

## Solution Assessment

Yes, as an implementation shape. Reusing readProviderJsonResponse in the shared Comfy readJsonResponse wrapper covers the Comfy workflow JSON surface without duplicating a local bounded reader; the remaining choice is the 16 MiB cap policy.
