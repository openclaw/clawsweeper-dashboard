## What This Changes

The PR routes Comfy workflow success JSON parsing through the shared `readProviderJsonResponse` helper and adds bounded-read regression tests for the production wrapper.

## Best Possible Solution

Land the bounded-reader change after maintainers explicitly accept the 16 MiB Comfy success-JSON cap and current-head checks remain clean.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` route Comfy success JSON through `response.json()`, and the PR body plus added tests exercise an oversized streamed response through the same wrapper.

## Solution Assessment

Yes. Reusing `readProviderJsonResponse` at the shared Comfy `readJsonResponse` helper is the narrowest maintainable fix because it covers all Comfy success JSON reads while preserving fetch-guard, HTTP-error, and release behavior.
