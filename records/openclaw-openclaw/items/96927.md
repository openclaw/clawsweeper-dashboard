## Best Possible Solution

## Solution Assessment

Yes. Reusing readProviderJsonResponse at the shared Comfy readJsonResponse helper is the narrowest maintainable fix because it covers the workflow JSON call surface while preserving fetch-guard, HTTP-error, and release behavior.
