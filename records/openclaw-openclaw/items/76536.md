## Summary

Keep open: current main still lacks the request-time cold-load fallback for external manifest-contract capability providers, and this PR restores it narrowly with regression coverage and a changelog entry. I found no concrete blocking review finding, but the exact-head check suite was not fully green at review time.

## What This Changes

The PR makes capability-provider runtime loading fall back to scoped cold loading when the loaded runtime registry lacks the requested manifest-contract provider, adds a Fish Audio speech-provider regression test, and adds a changelog entry.

## Best Possible Solution

Merge the scoped capability-provider fallback and regression test once exact-head CI is green; keep the separate plugin-tools cold-registry regression tracked by #76598/#76609.

## Reproduction Assessment

Yes, source-reproducible. Current main can resolve the manifest contract to an enabled external provider id, but the provider entry loader only consults the loaded runtime registry and returns empty entries when that registry lacks the scoped plugin id.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it preserves startup-registry reuse and only cold-loads the manifest-derived plugin scope after a loaded-registry miss.
