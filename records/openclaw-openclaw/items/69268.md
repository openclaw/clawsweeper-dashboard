## Summary

Keep open. Current main has adjacent shipped fixes for Anthropic adaptive-thinking payloads and outbound assistant-prefill stripping, but it does not contain the PR's runner-level missing-user-tail cleanup and still preserves explicit custom stream functions for supported APIs. The PR remains useful source work, but the stream override and no-input skip path need maintainer review before merge.

## What This Changes

The PR changes pi-embedded stream resolution and Anthropic transcript cleanup so supported transports bypass wrapped stream functions and runner state drops or skips trailing non-user turns when no fresh prompt is available.

## Best Possible Solution

Keep the useful Anthropic reproduction and helper tests, but narrow the stream-resolution change so only the default pi wrapper path is forced through OpenClaw's boundary-aware transport. Then reconcile runner-state pruning with the existing outbound prefill sanitizer and ensure no-input skips produce a no-op/retry outcome rather than reusing stale assistant text.

## Reproduction Assessment

Yes for the patch-level regression: current main has a focused test that explicit custom streams remain unchanged for supported APIs, while the PR changes that contract. The original production Opus 4.7/prefill failures have credible PR discussion and related issue evidence, but I did not run a live provider reproduction in this read-only sweep.

## Solution Assessment

No. The PR addresses a real Anthropic failure mode, but the unconditional boundary-aware override is broader than the bug and the no-input skip path needs stale-output suppression; a narrower default-wrapper detection plus focused runner cleanup would be safer.
