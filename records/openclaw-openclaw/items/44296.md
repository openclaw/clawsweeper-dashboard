## What This Changes

The issue asks onboarding shell-completion setup to show a concrete PowerShell profile path and actionable reload guidance.

## Best Possible Solution

Centralize profile-path and reload/source-line formatting in completion runtime, use it from setup and doctor, add the PowerShell install/reload branch, and cover the Windows onboarding path with focused tests.

## Solution Assessment

Yes. The narrow maintainable fix is to share the runtime profile-path and PowerShell dot-source formatting with setup and doctor, rather than adding another one-off `$PROFILE` or hardcoded path branch.
