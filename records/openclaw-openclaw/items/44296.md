Keep open. Current main and the latest release still route PowerShell onboarding completion setup through generic `$PROFILE`/reload wording, while the direct fix PR was closed unmerged and the open related completion PRs do not clearly cover the onboarding message gap.

PowerShell shell-completion onboarding should print the resolved profile path and an actionable reload command instead of a generic `$PROFILE` hint.

Centralize profile path and source-line formatting for setup, doctor, and install; use PowerShell dot-source reload guidance; and add focused Windows-oriented regression coverage.
