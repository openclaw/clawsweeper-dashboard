Full review comments:

- **[P2] Scope registry repair to supplied findings:** `src/flows/doctor-core-checks.ts:603-611`
  - body: The new plugin-registry health check takes the structured `repair(ctx, findings)` contract, but this repair omits `findings` and redetects every registry issue before mutating. A caller that passes a narrowed finding set can therefore preview or repair stale managed npm packages, peer links, or registry migrations outside the selected findings, which breaks the finding-scoped repair shape this stack is introducing. Filter the detected issues from the supplied findings, or encode enough issue data in the findings, before producing effects or calling the repair helper.
  - confidence: 0.82
- **[P3] Return skipped for disabled registry repair:** `src/flows/doctor-core-checks.ts:618-624`
  - body: When the registry migration disable env var is set, detect returns a disabled finding, but the repair wrapper still returns no explicit status after `repairPluginRegistryState`, so the runner treats it as repaired and then validates the disabled finding again. Return `status: "skipped"` for the disabled-only path so counters and warnings do not report an impossible repair as completed.
  - confidence: 0.9
