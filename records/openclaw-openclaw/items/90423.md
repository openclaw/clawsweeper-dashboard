## Summary

Keep open: the PR fixes the right TTS owner boundary and includes useful proof, but the latest sanitizer still has a source-reproducible false positive that can remove legitimate first-person summary text before speech.

## What This Changes

The PR sanitizes `summarizeText` output before TTS audio generation and adds TTS unit/contract coverage for hidden reasoning, prompt echoes, source-block echoes, and length capping.

## Best Possible Solution

Keep the sanitizer in `summarizeText`, but narrow prompt-echo detection so it removes actual model scaffolding without deleting legitimate summary prose, with a regression test for the first-person `key points` case.

## Reproduction Assessment

Yes for the introduced patch defect: source inspection shows a mocked summary beginning `I need to keep the key points...` flows through the self-summary prompt-echo branch and loses the first sentence before `textForAudio`. I did not run tests because this review is read-only.

## Solution Assessment

No as submitted. Sanitizing inside `summarizeText` is the right boundary, but the current heuristic is still too broad and should preserve legitimate first-person summary sentences while stripping real prompt echoes.
