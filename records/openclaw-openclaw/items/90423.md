The PR sanitizes `summarizeText` output before TTS synthesis and adds focused unit and plugin contract coverage for reasoning/control scaffolding, prompt echoes, truncation, and preservation edge cases.

## Best Possible Solution

Land the pre-synthesis summary sanitizer with the focused regression coverage if maintainers are satisfied with the normal CI and proof state.

## Reproduction Assessment

Yes: source inspection shows current main joins raw summary-model `TextContent` and the auto-TTS caller sends `summary.summary` into `textForAudio`. A completion containing reasoning tags, prompt echoes, or source text would therefore be spoken before this PR.

## Solution Assessment

Yes: sanitizing inside `summarizeText` is the narrow owner-boundary fix because every auto-TTS caller receives the cleaned summary before provider synthesis. Downstream provider-specific cleanup would be more duplicated and easier to miss.
