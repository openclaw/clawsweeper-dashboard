## What This Changes

The PR sanitizes TTS summary-model output inside `summarizeText` before that summary is handed to speech synthesis, with focused regression tests for prompt echoes, hidden scaffolding, and preservation cases.

## Best Possible Solution

Land the centralized `summarizeText` cleanup after exact-head checks finish and maintainers accept the scoped text-path proof, keeping prompt-echo cleanup out of provider-specific renderers.

## Reproduction Assessment

Yes. Source inspection shows current main returns raw summary-model text and the auto-TTS caller assigns that returned summary directly to `textForAudio`; a controlled `completeSimple` text response with prompt echoes or reasoning scaffolding exercises the leak path.

## Solution Assessment

Yes. Sanitizing inside `summarizeText` is the best owner boundary because every auto-TTS provider consumes the same returned summary, while provider-local cleanup would duplicate policy and leave callers inconsistent.
