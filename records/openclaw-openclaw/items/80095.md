## Summary

Keep open: current main still hardcodes the Feishu/Lark card note, but making it configurable adds a new public formatting/config contract and overlaps existing response-prefix/response-footer work, so maintainer product/API direction is needed first.

## What This Changes

The issue requests a configurable agent message metadata/footer surface to disable or template the identity/model/provider footer globally, per agent, and by channel.

## Best Possible Solution

Choose one stable message-formatting contract, likely by extending the existing prefix/footer or Feishu card-note policy, then preserve the current default, document the config, and test streaming and structured card delivery.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that Feishu card replies currently build and render the hardcoded `Agent | Model | Provider` note.

## Solution Assessment

Unclear. The requested capability is valid, but the proposed `agents.defaults.messageMeta` shape may be broader than needed and should be reconciled with `messages.responsePrefix`, the open `messages.responseFooter` PR, and Feishu-owned card-note behavior.
