## Summary

This should stay open: the hardcoded identity/model/provider footer is real for Feishu/Lark card replies, but making it configurable adds a user-facing config contract and overlaps the existing response-prefix surface, so it needs maintainer product/API direction before implementation.

## What This Changes

Request to make the assistant reply identity footer configurable, including disabling or templating the agent/model/provider stamp by default, agent, channel, and context.

## Best Possible Solution

Extend the existing message-formatting contract or Feishu card config so the card note can be disabled or templated while preserving the current default, then document and test card and streaming-card delivery.

## Reproduction Assessment

Not applicable as a bug repro because this is a feature/config request. Source inspection gives a high-confidence behavior path: Feishu card delivery always builds and renders the `Agent | Model | Provider` note.

## Solution Assessment

Unclear. The proposed `agents.defaults.messageMeta` shape may be broader than needed; a safer direction is likely to extend the existing message-formatting contract or add a Feishu-owned card-note policy after maintainer API review.
