## What This Changes

The PR adds Control UI chat composer composition state, defers draft sync during IME composition, commits the textarea value on `compositionend`, and adds a jsdom regression test.

## Best Possible Solution

Defer all controlled-value host rerenders until IME composition commits, preserve resize behavior, add a regression test covering stale-draft rerender, and attach redacted macOS Chrome CJK IME proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the textarea is controlled, input events trigger host rerenders, and draft commits are currently unconditional. I did not run the full macOS Chrome CJK IME flow in this read-only review.

## Solution Assessment

No. The direction is right, but the patch leaves `updateSlashMenu` before the composition guard, so the safer fix is to suppress rerender-triggering slash-menu updates while composing and commit once at `compositionend`.
