## Label Justifications

- `P1`: The PR targets a current Control UI chat composer bug that makes CJK typing effectively unusable for affected users, but the proposed fix is not yet mergeable.

## Summary

Keep open: this is a focused Control UI bugfix PR, but the proposed IME guard still runs after slash-menu state schedules a host rerender, so the controlled textarea can still be reset from the stale draft during composition. It also lacks real browser/IME proof, which is required for this external UI behavior change.

## What This Changes

The PR adds composer composition state to defer draft synchronization until `compositionend` and adds a jsdom regression test for Chinese IME input.

## Best Possible Solution

Defer all controlled-value rerenders until composition commits, while preserving local textarea sizing, then prove the macOS/Chrome CJK IME flow in the Control UI.

## Reproduction Assessment

No live reproduction was run in this read-only review. Source and shipped `v2026.5.22` both show the controlled draft textarea and per-input sync path described by the linked CJK IME report, so the bug is source-reproducible but not browser-proven here.

## Solution Assessment

No. The patch is on the right code path, but the composition guard happens after `updateSlashMenu` can schedule a stale-draft rerender; the safer fix is to avoid draft or slash-menu rerenders while composition is active and commit once at `compositionend`.
