## Summary

Keep open because current main still has the reported scheduler asymmetry, and open PR #46423 already targets the same behavior with closing syntax.

## What This Changes

Recurring cron/every jobs should retry transient failures with short backoff instead of waiting for the next natural schedule slot.

## Best Possible Solution

Land or replace the focused scheduler fix so recurring transient failures use the existing transient classifier and retry backoff while permanent failures keep the natural-schedule behavior, with regression tests and retry-contract docs updated.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a daily cron/every job returning a transient error computes a short backoff but current main still selects the later natural next run.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing transient classifier in the recurring error path while preserving current permanent-error scheduling semantics.
