## Summary

Keep open: the backend and protocol threading look coherent and CI is green, but the Control UI picker still leaks the global default model outside the per-agent allowlist. The supplied after screenshot shows that mismatch, so the proof is not sufficient for the claimed behavior.

## What This Changes

The PR adds optional `agentId` scoping for `models.list` and threads per-agent `agents.list[].models` visibility through model-selection helpers, gateway handling, Control UI loading, slash commands, generated Swift protocol, tests, and config docs.

## Best Possible Solution

Keep the additive protocol/config direction, but make the UI model-select state honor the agent-scoped catalog and default consistently, then update tests and real browser proof for an agent whose list excludes the global default.

## Reproduction Assessment

Yes. The PR's own El Becario after screenshot shows the global `openai-codex/gpt-5.5` default still in the dropdown outside the two-model per-agent config, and source inspection shows the select state appends defaults after the scoped catalog.

## Solution Assessment

No. The backend/threading approach is reasonable, but the picker must stop re-adding a model outside the scoped catalog or compute an agent-scoped default before the PR satisfies its own allowlist contract.
