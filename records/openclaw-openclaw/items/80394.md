Keep open: the implementation direction is useful, but the PR head still leaves legacy main slash-command catalog loads unscoped, the public protocol docs are out of sync, and the supplied real behavior proof is for an older commit and old config key.

## What This Changes

The PR adds optional per-agent `modelAllowlist` config and threads `agentId` through model visibility, gateway `models.list`, Control UI loading, slash-command paths, generated Swift, docs, and tests.

## Best Possible Solution

Land a single visibility-scoped per-agent policy after legacy session keys pass the default agent id, protocol docs describe `models.list.agentId`, maintainers accept the visibility-only boundary, and fresh proof shows current `modelAllowlist` behavior.

## Reproduction Assessment

Yes for the blocking PR finding: PR-head source shows `loadModelCatalog` only extracts an agent id from `parseAgentSessionKey`, while the legacy `main` slash-command test still expects no `agentId` in `models.list`. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. The direction is maintainable, but the slash-command loader needs the same default-agent fallback as Control UI, the public protocol docs need the new parameter, and maintainers still need to accept visibility-only scope versus true server-side enforcement.
