## Summary

Keep open. The feature is aligned with the open per-agent allowlist request, but the current branch still has compatibility and enforcement blockers: it overloads an existing metadata/runtime map, leaks default/current options into the scoped picker, and leaves server-side model switching validated against the global allowlist.

## What This Changes

The PR adds agentId-scoped model catalog visibility using `agents.list[].models` across model-selection helpers, gateway `models.list`, UI and slash-command catalog loading, generated Swift protocol types, tests, and docs.

## Best Possible Solution

Define one documented effective per-agent model policy with explicit inheritance semantics, then enforce that shared resolver consistently in catalog listing, picker options, slash-command/model patch validation, tools, and tests.

## Reproduction Assessment

Yes. The review findings are source-reproducible at the PR head: the branch reuses an existing metadata map as the allowlist, the picker still appends default/current options outside the scoped catalog, and server model patch validation is not agent-scoped.

## Solution Assessment

No. Threading `agentId` through model listing is the right direction, but this implementation is not the safest contract because it changes existing `models` metadata semantics and does not apply the policy to every selection path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve runtime-only per-agent model overrides:** `src/agents/model-selection-shared.ts:88-92`
  - body: This treats any non-empty `agents.list[].models` map as a visibility allowlist. On main that field is already documented and typed as per-model metadata/runtime overrides, and `model-runtime-policy` consults it before defaults for `agentRuntime`; an existing agent that only overrides one model's runtime would suddenly hide every other default model. Add an explicit visibility signal or separate the metadata/runtime map from the allowlist semantics.
  - confidence: 0.91
- **[P2] Filter default/current picker options through the scoped policy:** `ui/src/ui/app-chat.ts:790`
  - body: Loading an agent-scoped catalog here is not enough to make the Control UI picker scoped, because `resolveChatModelSelectState` still appends the current override and global default model after the catalog. The supplied El Becario proof shows `gpt-5.5` in the dropdown even though that agent's declared list is only two Ollama models, so the picker still exposes a global default outside the agent list.
  - confidence: 0.84
- **[P2] Scope model patch validation to the session agent:** `ui/src/ui/chat/slash-command-executor.ts:222-225`
  - body: `/model <name>` still sends only `{ key, model }` to `sessions.patch`, and the PR head leaves `resolveAllowedModelRef` without an `agentId`, so the server validates against the global defaults allowlist. Users can type a globally allowed model that is absent from the scoped catalog and still switch the session, which defeats the allowlist outside the picker.
  - confidence: 0.86
