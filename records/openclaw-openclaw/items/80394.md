## Summary

Keep open: the feature is useful and matches the open per-agent allowlist request, but the patch has blocking correctness issues around session-key and agent-id normalization, plus failing docs/lint checks and missing inspectable real behavior proof.

## What This Changes

The branch adds an optional `agents.list[].models` per-agent allowlist, threads `agentId` through gateway and UI model catalog calls, and updates docs and tests.

## Best Possible Solution

Land the feature only after it reuses the existing normalized agent/session resolution paths, fixes the required lint/docs lanes, documents the protocol parameter where appropriate, and includes redacted real behavior proof.

## Reproduction Assessment

Yes for the review findings: source inspection shows the new slash-command request omits `agentId` for the legacy `main` session key, and the new gateway/helper lookups compare raw trimmed ids instead of normalized ids. The per-agent allowlist itself is a feature request, not a current-main bug.

## Solution Assessment

No. The feature direction is reasonable, but this patch should reuse existing agent-id and session-key normalization instead of adding raw matching, and it needs green docs/lint checks plus inspectable proof before merge.
