## Best Possible Solution

Keep the explicit `modelAllowlist` contract, derive `agentId` consistently for legacy and canonical session keys, update public protocol docs, rebase, and refresh real browser proof against the latest branch.

## Reproduction Assessment

Yes. The slash-command gap is source-reproducible because `parseAgentSessionKey("main")` returns null and the PR's own `/model` test for `main` still expects `models.list` without `agentId`.

## Solution Assessment

No, not as written. The explicit allowlist is the right direction, but the implementation needs the same legacy-main agent resolution used by the Control UI path plus protocol docs and current proof.
