## Summary

Keep open. The PR targets a valid feature request, but the patch still has blocking compatibility and UI leakage issues: it repurposes the existing per-agent runtime metadata map as a visibility allowlist, and the Control UI picker can still append default/current models outside the scoped catalog.

## What This Changes

The PR adds agent-scoped model visibility by threading an optional `agentId` through model-selection helpers, gateway `models.list`, Control UI model loading, slash-command catalog loading, protocol schema/types, docs, and tests.

## Best Possible Solution

Define one documented effective per-agent model visibility policy that preserves runtime metadata-only entries, then apply that shared policy consistently to gateway listing, `/model`, the Control UI picker, and validation paths.

## Reproduction Assessment

Yes, for the review findings. Source inspection shows the existing per-agent `models` runtime metadata contract and the PR reusing that same non-empty map as visibility input; the supplied El Becario screenshot also shows a global default model still present in the scoped picker.

## Solution Assessment

No, not as written. Threading `agentId` through the catalog path is a useful direction, but the patch needs an explicit compatible visibility contract and must filter supplemental picker options as well as catalog rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve runtime-only per-agent model overrides:** `src/agents/model-selection-shared.ts:88-90`
  - body: Returning any non-empty `agents.list[].models` map as the visibility allowlist changes existing configs that only use that map for per-model metadata such as `agentRuntime`: those agents would suddenly lose the rest of `agents.defaults.models` from `models.list` and the picker. Please make visibility explicit or otherwise distinguish metadata-only entries from allowlist entries.
  - confidence: 0.9
- **[P2] Filter supplemental picker options by the scoped policy:** `ui/src/ui/app-chat.ts:790`
  - body: Passing `agentId` into `loadModels` only scopes the catalog response; the picker state still appends the current override and global default after catalog entries. The supplied El Becario proof shows `gpt-5.5` still exposed outside the stated two-model allowlist, so the Control UI can still leak a global default for a scoped agent.
  - confidence: 0.82
