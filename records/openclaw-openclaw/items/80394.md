## Summary

Keep open: the feature is plausible and tied to open per-agent model policy work, but this branch still has correctness blockers around the existing `agents.list[].models` contract and scoped picker behavior.

## What This Changes

The PR threads optional `agentId` model visibility through model-selection helpers, gateway `models.list`, UI and slash-command catalog loading, protocol schemas, generated Swift models, docs, and tests.

## Best Possible Solution

Preserve existing per-agent runtime metadata, choose an explicit visibility contract, and apply the same effective model policy in listing, picker, slash-command, and regression coverage.

## Reproduction Assessment

Yes for the review findings: source inspection shows the existing metadata contract and the unchanged picker option appends. The supplied browser screenshot also shows `gpt-5.5` still visible for a scoped El Becario picker.

## Solution Assessment

No, not as written. Threading `agentId` is the right direction, but overloading the existing runtime metadata map and leaving default/current picker options outside the scoped policy is not the narrowest compatible solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve runtime-only per-agent model overrides:** `src/agents/model-selection-shared.ts:88-90`
  - body: This makes every non-empty `agents.list[].models` map the visibility allowlist. That field already exists for per-agent catalog/runtime metadata such as `agentRuntime`, so an agent that only overrides one model's runtime behavior would now lose the rest of `agents.defaults.models` from `models.list` and the picker; please add an explicit visibility contract or distinguish metadata-only entries.
  - confidence: 0.91
- **[P2] Apply scoped policy to default picker options:** `ui/src/ui/app-chat.ts:790`
  - body: Passing `agentId` only to `loadModels` does not fully scope the Control UI picker because the select state appends the current override and global default model after the scoped catalog is loaded. The supplied El Becario screenshot still exposes `gpt-5.5` outside the stated two-model allowlist, so the supplemental default/current options need to be filtered or policy-aware too.
  - confidence: 0.83
