## Summary

Keep open. The PR addresses a real per-agent model visibility gap, but the current branch is not safe to merge because it reuses the existing per-agent `models` runtime-metadata map as a visibility allowlist and still lets default/current model options leak into the scoped Control UI picker.

## What This Changes

The PR adds optional `agentId` scoping to `models.list` and threads per-agent `agents.list[].models` visibility through model selection, the gateway, UI loaders, generated Swift protocol models, tests, and docs.

## Best Possible Solution

Define a documented per-agent model visibility policy that preserves runtime metadata semantics, then enforce one effective allowed set consistently across `models.list`, `/model`, session patching, UI/TUI pickers, docs, and regression tests.

## Reproduction Assessment

Yes for the review findings: current source and docs show `agents.list[].models` already has runtime-metadata semantics, and the PR source makes that same map a visibility allowlist. The supplied El Becario screenshot also reproduces the picker leak by showing `gpt-5.5` outside the stated two-model scoped list.

## Solution Assessment

No, not as written. Threading `agentId` is the right general direction, but the maintainable fix needs an explicit compatibility-safe visibility contract and must filter supplemental default/current picker options through the same scoped policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve runtime-only per-agent model entries:** `src/agents/model-selection-shared.ts:88-90`
  - body: Returning any non-empty `agents.list[].models` map here makes existing runtime metadata become the visibility allowlist. Current docs/runtime already use that map for `agentRuntime`, so an agent that only set one model to choose a runtime would silently lose the rest of `agents.defaults.models` from `models.list` and `/model`; use an explicit visibility contract or distinguish metadata-only entries before filtering.
  - confidence: 0.91
- **[P2] Filter supplemental picker options through the scoped policy:** `ui/src/ui/app-chat.ts:790`
  - body: This call scopes only the fetched catalog. The select state still appends the current override and global default after catalog loading, and the supplied El Becario screenshot shows `gpt-5.5` in a two-model allowlist, so the Control UI picker can still expose disallowed global defaults.
  - confidence: 0.82
