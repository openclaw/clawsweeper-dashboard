## Summary

Keep open. This is an active external feature PR with useful work and supplied real behavior proof, but the branch still has blocking compatibility and completeness issues around the existing per-agent `models` contract and scoped UI/slash-command visibility.

## What This Changes

The PR adds per-agent model visibility scoping through agent-scoped model-selection helpers, the gateway `models.list` protocol/server path, UI model loading, slash-command catalog loading, generated Swift protocol models, tests, and docs.

## Best Possible Solution

Define one explicit effective per-agent model policy that preserves existing runtime metadata behavior, documents inheritance/replacement semantics, and enforces the same resolver in gateway listing, UI/TUI pickers, slash commands, session patch/model validation, and tests.

## Reproduction Assessment

Yes. The review findings are source-reproducible from current main plus the PR head, and the supplied browser screenshot directly shows a global default model still appearing in an agent-scoped picker.

## Solution Assessment

No, not as written. Threading `agentId` is the right direction, but overloading the existing runtime metadata map and filtering only the catalog load is not the narrowest compatible solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve runtime-only per-agent model overrides:** `src/agents/model-selection-shared.ts:88-90`
  - body: This makes any non-empty `agents.list[].models` map the visibility allowlist. That field already exists for per-agent model metadata such as `agentRuntime`, so a config that only overrides runtime for one model would now hide the rest of `agents.defaults.models` from `models.list` and pickers. Please make visibility opt-in or otherwise distinguish runtime metadata entries from allowlist entries.
  - confidence: 0.9
- **[P2] Filter injected default and current model options:** `ui/src/ui/app-chat.ts:790`
  - body: Passing `agentId` to `loadModels` does not fully scope the Control UI picker because the select state still appends the current override and global default model after loading the scoped catalog. The supplied El Becario proof declares two Ollama models but still shows `gpt-5.5 · openai-codex`, so the picker can expose a global default outside the agent allowlist.
  - confidence: 0.86
- **[P2] Scope legacy main slash-command catalogs:** `ui/src/ui/chat/slash-command-executor.ts:775-779`
  - body: `loadModelCatalog` only derives `agentId` from `agent:*:*` session keys, while the supported bare `main` alias still calls `models.list` without `agentId`. That means `/model` and `/think` can show the global catalog for the main agent even when `agents.list[{ id: "main" }].models` is configured.
  - confidence: 0.79
