## Summary

Keep open. The feature is still relevant and aligns with the open canonical request at https://github.com/openclaw/openclaw/issues/7424, but the current PR has blocking compatibility and enforcement gaps around the existing per-agent `models` metadata contract and model-switch validation.

## What This Changes

The branch adds `agentId` to `models.list` and reuses `agents.list[].models` as an agent-scoped visibility allowlist across model-selection helpers, gateway, UI loaders, slash-command paths, protocol/generated Swift, tests, and docs.

## Best Possible Solution

Define one documented effective per-agent model policy with explicit inheritance/replacement semantics, then enforce that shared resolver consistently in catalog listing, UI/TUI options, slash commands, `sessions.patch`, runtime selection, docs, and regression tests.

## Reproduction Assessment

Yes, source-reproducible for the PR findings. Current main documents and uses `agents.list[].models` for metadata/runtime overrides, the PR source reuses that same map as the visibility source, `sessions.patch` still validates without `agentId`, and the supplied screenshot shows a leaked default option.

## Solution Assessment

No, not as written. Threading `agentId` is the right direction, but the patch needs a maintainer-approved config contract and must apply the same scoped policy to model override validation and supplemental picker options.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve runtime-only per-agent model overrides:** `src/agents/model-selection-shared.ts:88-90`
  - body: This treats every non-empty `agents.list[].models` map as the visibility allowlist. That field already has a metadata/runtime meaning, including per-agent `agentRuntime` overrides, so an existing agent that only overrides one model runtime would unexpectedly lose the rest of the global catalog from `models.list` and the picker. Please make visibility explicit or otherwise distinguish metadata-only entries from allowlist entries.
  - confidence: 0.91
- **[P2] Thread agentId into model override validation:** `src/agents/model-selection.ts:401`
  - body: The branch adds agent-aware status checks, but the `resolveAllowedModelRef` path used by `sessions.patch` still has no `agentId`, so Control UI or slash-command model patches can still select a model that is allowed globally but not for the current agent. Please pass the session agent id through this resolver and cover the rejection path.
  - confidence: 0.86
- **[P2] Filter default picker options through the scoped policy:** `ui/src/ui/app-chat.ts:790`
  - body: Passing `agentId` to `loadModels` does not fully scope the picker because the select state appends the current override and global default model after the scoped catalog loads. The supplied El Becario proof shows `gpt-5.5` still present even though the declared allowlist has only two Ollama models, so the picker can still expose a global default.
  - confidence: 0.8
