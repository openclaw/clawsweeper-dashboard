## Summary

Keep open: the feature is plausible and the supplied browser proof is real, but the patch currently changes the meaning of an existing per-agent `models` map and still lets the Control UI picker show default/current models outside the scoped catalog.

## What This Changes

The PR adds per-agent model visibility scoping by threading optional `agentId` through model selection, gateway `models.list`, UI model loading, slash-command paths, protocol schema, generated Swift models, tests, docs, and one unrelated Discord test narrowing change.

## Best Possible Solution

Preserve existing per-agent runtime override behavior, add or document an explicit visibility opt-in contract, and apply that same scoped policy to supplemental UI default/current model options.

## Reproduction Assessment

Yes, source-reproducible for the review findings: current docs/runtime code show existing per-agent `models` semantics, and the PR source reuses that same map as visibility input. The UI leak is also visible from source because the picker appends default/current options outside the scoped catalog, and the supplied screenshot shows `gpt-5.5` still present.

## Solution Assessment

No, not as written. Threading `agentId` is the right general direction, but overloading the existing runtime metadata map and not filtering default/current picker options is not the narrowest compatible solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve runtime-only per-agent model overrides:** `src/agents/model-selection-shared.ts:88`
  - body: This makes every non-empty `agents.list[].models` map the visibility allowlist. That field already exists for per-agent catalog/runtime metadata such as `agentRuntime`, so an existing agent that only set one model to override runtime behavior would now lose the rest of `agents.defaults.models` from `models.list` and the picker. Please make visibility opt-in or otherwise distinguish runtime metadata entries from allowlist entries.
  - confidence: 0.9
- **[P2] Apply the allowlist to default picker options:** `ui/src/ui/app-chat.ts:800`
  - body: Passing `agentId` only to `loadModels` does not fully scope the Control UI picker because the select state appends the current override and global default model after the scoped catalog is loaded. In the supplied El Becario proof, the declared allowlist is two Ollama models but the dropdown still exposes `gpt-5.5`, so the picker can still leak a global default for agents that should not see it.
  - confidence: 0.78
