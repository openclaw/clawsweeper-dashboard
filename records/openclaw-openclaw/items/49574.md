## Summary

Keep open: this branch targets a real proxy retry/CPU-pressure concern, but the current patch changes configured proxy routing to fail open, is stale against the active Discord plugin layout, and still lacks real runtime proof for the claimed reduction.

## What This Changes

This PR adds proxy circuit-breaker state, wraps shared and Discord proxy fetch paths to bypass failed proxies via direct fetch, lowers Discord gateway reconnect attempts, and broadens rate-limit matching.

## Best Possible Solution

Keep proxy routing fail-closed by default; if fallback is approved, add an explicit documented opt-in and implement it in the shared proxy layer plus active plugin paths with real proof.

## Reproduction Assessment

No high-confidence end-to-end reproduction is attached. Source inspection shows current proxy-backed fetches still route through dispatchers and propagate request-time failures, but CPU exhaustion and retry-rate reduction were not behavior-proven.

## Solution Assessment

No. A circuit breaker may be useful, but this implementation fails open around configured proxy routing, misattributes gateway direct-fetch failures, permits concurrent half-open probes, and needs active Discord extension rebase plus policy approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep configured proxy failures fail closed:** `src/infra/net/proxy-fetch.ts:17-28`
  - body: When the circuit is open or a proxy connection error occurs, this retries with global fetch and sends traffic outside the configured proxy. That conflicts with the documented managed-proxy guardrail unless maintainers add an explicit opt-in policy for direct egress.
  - confidence: 0.9
- **[P1] Record gateway failures only for proxied attempts:** `src/discord/monitor/gateway-plugin.ts:77-79`
  - body: The circuit-open path performs a direct fetch, but the catch block still records connect-style direct failures as proxy failures. A Discord reachability issue can therefore extend proxy cooldown and delay recovery probes.
  - confidence: 0.9
- **[P2] Allow only one half-open proxy probe:** `src/infra/net/proxy-probe.ts:70-73`
  - body: After cooldown expiry, every caller sees the same expired timestamp and probes the proxy. Under concurrent traffic this recreates a burst of failed connects and inflates the backoff counter.
  - confidence: 0.84
- **[P2] Port Discord wiring to the active plugin path:** `src/discord/monitor/gateway-plugin.ts:40`
  - body: This changes legacy src/discord code, but current main's active Discord runtime is under extensions/discord. A rebased patch needs to update that path or the advertised Discord gateway and REST behavior will not ship.
  - confidence: 0.82
