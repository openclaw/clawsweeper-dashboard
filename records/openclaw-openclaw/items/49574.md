## Summary

Keep open. The PR targets a real proxy-unavailable pain point, but the current diff fails open around documented proxy egress controls, has circuit/gateway correctness bugs, lacks real behavior proof, and overlaps with the open policy discussion at https://github.com/openclaw/openclaw/issues/68871.

## What This Changes

Adds a proxy circuit breaker that falls back to direct fetch on proxy connection errors, wires it into shared proxy fetch and legacy Discord paths, lowers Discord gateway reconnect attempts, and adjusts rate-limit matching.

## Best Possible Solution

Keep proxy routing fail-closed by default; if fallback is approved, add an explicit documented opt-in in the shared proxy layer and port it to active plugin paths with real proof.

## Reproduction Assessment

No high-confidence end-to-end reproduction is attached. Source inspection shows unavailable configured proxies still route through proxy dispatchers and propagate request-time failures, but CPU exhaustion and retry-rate reduction are not behavior-proven here.

## Solution Assessment

No. A shared circuit breaker may be useful, but this PR fails open around configured proxy routing, needs gateway attribution and half-open concurrency fixes, and must be rebased onto the active extensions/discord layout after policy approval.

  - confidence: 0.84
