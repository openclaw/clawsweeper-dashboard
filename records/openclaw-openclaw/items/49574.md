## Summary

Keep open. The PR targets a plausible proxy retry/CPU problem, but the proposed direct-fallback circuit conflicts with OpenClaw's fail-closed proxy contract, has concrete logic bugs, lacks real behavior proof, and is stale against current Discord plugin paths.

## What This Changes

The PR adds a proxy circuit breaker around proxy-backed fetches and old Discord gateway/REST fetch paths, with related tests and rate-limit pattern tweaks.

## Best Possible Solution

Rebase onto current main, keep the implementation in the current proxy and Discord plugin paths, and solve retry churn with a fail-closed or explicitly approved proxy policy while adding focused tests and real unavailable-proxy proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows current main lacks a circuit breaker around shared proxy fetches, but the PR only provides mocked tests and no real unavailable-proxy CPU/proxy-routing proof.

## Solution Assessment

No. A circuit breaker is a reasonable direction, but this implementation silently falls back to direct fetch, has gateway/half-open logic bugs, and targets paths that current main has moved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep configured proxy traffic fail-closed:** `src/infra/net/proxy-fetch.ts:18-26`
  - body: The circuit-open and connect-error paths return global fetch, so deployments using HTTP_PROXY or managed proxy mode for egress control can send provider/plugin traffic directly outside the configured proxy. Throttle or fail fast unless maintainers explicitly approve direct fallback semantics.
  - confidence: 0.88
- **[P2] Only record gateway failures from proxied attempts:** `src/discord/monitor/gateway-plugin.ts:77-79`
  - body: When useProxy is false, the gateway metadata request is already direct, but the catch block still records connect-style errors as proxy failures. A transient direct Discord failure can extend the proxy cooldown and delay recovery.
  - confidence: 0.86
- **[P2] Limit half-open probing to one caller:** `src/infra/net/proxy-probe.ts:64-70`
  - body: After cooldown expiry, every concurrent caller can observe the half-open state as available and probe the proxy at once. That reintroduces the connection burst the circuit breaker is meant to prevent and can inflate consecutiveFailures.
  - confidence: 0.82
