## Summary

Keep open: the unavailable-proxy retry problem is plausible from source inspection, but this PR is not merge-ready because it fails open around configured proxy routing, has proxy-state bugs, is stale against the active Discord plugin layout, and lacks real behavior proof.

## What This Changes

Adds a proxy circuit breaker that falls back to direct fetch on proxy connection errors, wires it into shared proxy fetch and legacy Discord paths, lowers Discord gateway reconnect attempts, and broadens rate-limit matching.

## Best Possible Solution

Keep proxy routing fail-closed by default; if maintainers approve direct fallback, gate it behind an explicit documented opt-in in the shared proxy layer and port it to active plugin paths with real unavailable-proxy proof.

## Reproduction Assessment

No high-confidence end-to-end reproduction is attached. Source inspection shows unavailable configured proxies still route through proxy dispatchers and can propagate request-time failures, but CPU exhaustion and retry-rate reduction are not behavior-proven here.

## Solution Assessment

No. A shared circuit breaker may be useful, but this PR chooses fail-open direct egress without a policy knob, has proxy attribution and half-open concurrency defects, and targets stale Discord files.

  - body: The circuit-open and connect-error paths retry with global fetch, sending traffic outside configured proxy egress. That weakens the documented managed-proxy boundary unless maintainers add an explicit opt-in policy.
  - confidence: 0.86
- **[P1] Record gateway failures only for proxied attempts:** `src/discord/monitor/gateway-plugin.ts:77-79`
  - body: The circuit-open path performs a direct fetch, but the catch block still records connect-style direct failures as proxy failures. A Discord reachability failure can therefore extend proxy cooldown and delay recovery probes.
  - confidence: 0.9

  - body: After cooldown expiry, every caller sees the same expired lastFailAt and probes the proxy. Under concurrent traffic this recreates a burst of failed proxy connects and inflates the backoff counter.
  - confidence: 0.84
