## Summary

Keep open: the branch still fails review because it silently bypasses configured proxy egress, has circuit-state bugs, lacks real runtime proof, and is stale against the active Discord plugin layout.

## What This Changes

Adds a proxy circuit breaker that falls back to direct fetch on proxy connection errors, wires it into shared proxy and legacy Discord paths, lowers Discord gateway reconnect attempts, and broadens rate-limit matching.

## Best Possible Solution

Keep proxy routing fail-closed by default; if maintainers approve fallback, add a documented opt-in in the shared proxy layer and port it to active plugin paths with real proof.

## Reproduction Assessment

No high-confidence end-to-end reproduction is attached. Source inspection shows unavailable configured proxies still route through proxy dispatchers and propagate request-time failures, but CPU exhaustion and retry-rate reduction are not behavior-proven.

## Solution Assessment

No. A shared circuit breaker may be useful, but this PR fails open around configured proxy routing, needs gateway attribution and half-open concurrency fixes, and must be rebased onto the active extensions/discord layout after policy approval.

Full review comments:

- **[P1] Keep configured proxy failures fail closed:** `src/infra/net/proxy-fetch.ts:17-28`
  - body: The circuit-open and proxy-connect-error paths call global `fetch`, sending traffic outside a configured proxy. That conflicts with the documented fail-closed proxy boundary unless maintainers add an explicit opt-in policy.
  - confidence: 0.86
- **[P1] Record gateway failures only for proxied attempts:** `src/discord/monitor/gateway-plugin.ts:77-79`
  - body: When `useProxy` is false, the gateway metadata request goes direct, but the catch block still records connect-style direct failures as proxy failures. A Discord reachability failure can extend proxy cooldown and delay recovery probes.
  - confidence: 0.9
- **[P2] Allow only one half-open proxy probe:** `src/infra/net/proxy-probe.ts:64-76`
  - body: After cooldown expiry, every caller can see the expired `lastFailAt` and probe the proxy. Under concurrent traffic this recreates a burst of failed proxy connects and inflates the backoff counter.
  - confidence: 0.84
