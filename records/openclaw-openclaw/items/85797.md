## What This Changes

The issue requests that `openclaw capability image generate` use configured `openai-codex` OAuth credentials for `openai/gpt-image-2` instead of failing as though a static API key is required.

## Best Possible Solution

Align the CLI image-generation command with the working agent tool auth path, then add a regression proving `OPENAI_API_KEY` can be unset when a usable `openai-codex` OAuth profile is present for `openai/gpt-image-2`.

## Solution Assessment
