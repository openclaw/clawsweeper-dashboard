- `merge-risk: 🚨 session-state`: Merging can make chat composer draft state stale or lost across teardown because the new local mirror is not part of the existing persistence snapshot.

## Summary

Keep open; the performance direction is reasonable, but the patch is not merge-ready because ordinary local-only typing can bypass the existing composer persistence and teardown flush path until blur/send/slash/busy state commits it.

## What This Changes

The PR adds a view-local chat composer draft mirror, commits it on send/Enter/blur/history/slash transitions, and updates chat view tests for draft sync behavior.

## Best Possible Solution

Keep the local mirror optimization only after wiring it into the composer persistence/teardown contract and adding regression coverage for an unblurred ordinary draft during teardown.

## Reproduction Assessment

Yes, source-reproducible: type an ordinary idle draft, do not blur or send, then trigger teardown/reload; the patch has not moved that text into host.chatMessage, while teardown persistence flushes only host chatMessage snapshots. I did not run browser proof in this read-only review.

## Solution Assessment

No; the local mirror is a plausible performance fix, but the best fix must preserve the existing draft persistence contract before moving ordinary typing out of host state.
