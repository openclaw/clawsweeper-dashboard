## Summary

Close as cannot reproduce: current `main` and the shipped `v2026.4.29` package both build the macOS LaunchAgent env-wrapper path as a raw filesystem path and serialize it directly as XML, with no Markdown or terminal-link formatter in the install path.

## What This Changes

The issue reports that macOS gateway install writes a Markdown-formatted env-wrapper path into LaunchAgent `ProgramArguments`, causing launchd to crash-loop.

## Best Possible Solution

Keep the raw LaunchAgent serializer unchanged unless a new report includes a raw plist or diagnostics export that proves an OpenClaw install input can reach this path already formatted as Markdown.

## Reproduction Assessment

No; the source and shipped-package reproduction path for `openclaw gateway install --force` builds the env-wrapper path internally and writes it as a plain XML string. The report has steps, but it lacks the raw plist/log needed to connect the observed Markdown value to an in-repo formatter.

## Solution Assessment

Yes; closing as cannot reproduce is the narrowest maintainable outcome because adding a sanitizer around generated service paths would mask an unproven upstream mutation instead of fixing an observable OpenClaw code path.
