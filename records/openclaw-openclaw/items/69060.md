## Label Justifications

- `P2`: This is a useful Gateway HTTP session-routing improvement with limited blast radius, but it still has merge-blocking API-scope and validation issues.

## Summary

Keep this PR open: the stable session hint is useful and the PR body provides real terminal proof, but the patch still adds the header in a shared Gateway resolver and thereby changes Responses session behavior without a matching public contract, continuation-scope update, or Responses tests.

## What This Changes

The PR adds an `x-openclaw-session` Gateway HTTP header, session-key precedence tests, OpenAI HTTP docs, and a changelog entry for stable agent-scoped sessions.

## Best Possible Solution

Make the header contract explicit first: either keep it Chat Completions-only or fully add it to Responses continuation scope, docs, and tests, then rerun changed-surface validation.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID when callers omit `user`, and the PR body includes after-fix curl/log output showing stable header reuse and a warm-session QMD skip.

## Solution Assessment

No, not yet. The stable hint is plausible, but adding it in the shared resolver changes Responses without an explicit public contract, continuation-scope update, or Responses tests.
