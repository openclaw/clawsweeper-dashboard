## Summary

Keep open. Current main still lacks the requested stable session hint, but this PR is not merge-ready because the branch changes shared Responses session resolution without matching continuity scope/docs/tests and still has focused lint/docs issues.

## What This Changes

The PR adds a validated `x-openclaw-session` Gateway header in shared session resolution, plus Chat Completions tests, OpenAI HTTP docs, and a changelog entry.

## Best Possible Solution

Land a documented Chat Completions-scoped stable session hint, or deliberately support the same hint in Responses by extending its continuity scope, docs, and tests.

## Reproduction Assessment

Yes, at source level. Current main has no `x-openclaw-session` branch, so callers that cannot set OpenAI `user` still fall through to UUID-backed stateless sessions; I did not run a live Gateway in this read-only review.

## Solution Assessment

No, not as currently patched. The stable hint is a plausible narrow API, but adding it in the shared resolver also changes `/v1/responses` without matching continuity scope, docs, or tests.
