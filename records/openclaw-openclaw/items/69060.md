## Summary

Keep open: the PR is a useful narrow stable-session improvement with sufficient terminal proof, but the current patch still changes the shared Gateway request resolver in a way that exposes the new header to Responses without matching continuation-scope, docs, or tests, and the docs heading still violates the scoped Mintlify heading rule.

## What This Changes

The PR adds a Gateway `x-openclaw-session` header, Chat Completions precedence tests, OpenAI HTTP docs, and a changelog entry for stable agent-scoped HTTP sessions.

## Best Possible Solution

Make the new header contract explicit, then either keep it Chat Completions-only or support it consistently in Responses with continuation-scope matching, docs, and tests while leaving the broader light-context work on the related issue.

Yes, at source level. Current main falls back to a random UUID when callers omit `user`, and the PR body includes after-fix curl/log output showing stable session-key reuse and a warm-session QMD skip.

## Solution Assessment
