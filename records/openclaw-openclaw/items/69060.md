## Summary

Keep open. The PR is a useful narrow stable-session API addition, but the current patch still changes the shared OpenAI-compatible request resolver while documenting and testing only Chat Completions, and the docs formatting check is still failing.

## What This Changes

The PR adds `x-openclaw-session` handling in the shared Gateway HTTP session resolver, plus OpenAI HTTP docs, a changelog entry, and Chat Completions precedence tests.

## Best Possible Solution

Choose the public session-hint contract, then either keep `x-openclaw-session` Chat Completions-only or document and test it consistently for Responses before landing; keep https://github.com/openclaw/openclaw/issues/68920 open for the larger light-context work.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID when OpenAI-compatible Chat Completions callers omit `user`, and the PR body includes after-fix curl/log proof that the new header reuses a warm session.

## Solution Assessment

No, not as currently patched. The stable session hint is a reasonable narrow direction, but the implementation needs an explicit endpoint-scope decision plus docs formatting before it is maintainable.
