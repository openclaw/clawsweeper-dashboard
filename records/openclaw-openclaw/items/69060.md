## Summary

Keep open. The stable-session change is useful and not on current main, and the PR body has sufficient terminal proof, but the current head still changes Responses session continuity through a shared resolver and has branch-introduced lint/docs failures plus conflicts.

## What This Changes

The PR adds a validated `x-openclaw-session` Gateway header in shared request-context session resolution, plus Chat Completions tests, OpenAI HTTP docs, and a changelog entry.

## Best Possible Solution

Land a documented Chat Completions-scoped stable session hint, or deliberately support the same hint in Responses by extending its continuity scope, while leaving the larger light-context latency work to the canonical issue.

## Reproduction Assessment

Yes, at source level. Current main has no `x-openclaw-session` branch, so callers that cannot set OpenAI `user` still fall through to UUID-backed stateless sessions; I did not run a live Gateway in this read-only review.

## Solution Assessment

No, not as currently patched. The stable hint is a reasonable narrow API for Chat Completions, but adding it in the shared resolver also changes Responses continuity without matching scope, docs, or tests.
