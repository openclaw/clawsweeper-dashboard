## Summary

Keep open. The stable session header is a plausible narrow improvement and the PR includes after-fix terminal proof, but the current patch still adds the header in a shared Gateway resolver without deciding whether it is also a Responses API contract, and the PR head has a docs formatting failure.

## What This Changes

The PR adds a public `x-openclaw-session` Gateway HTTP header, docs, changelog text, and Chat Completions tests for stable agent-scoped session hints.

## Best Possible Solution

Either make `x-openclaw-session` a documented and tested contract for every OpenAI-compatible endpoint it reaches, or plumb it only through Chat Completions; then reflow the docs and leave the larger light-context work on the linked umbrella issue.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID when Chat Completions callers omit `user`, and the PR body includes after-fix curl/log output showing stable session-key reuse; I did not run a live Gateway in this read-only review.

## Solution Assessment

No, not as currently patched. The stable session hint is a reasonable direction, but the patch needs an explicit Chat Completions vs shared Responses contract and docs formatting repair before it is the maintainable solution.
