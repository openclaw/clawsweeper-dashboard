## Summary

Keep open. The stable-session header is a plausible partial fix with after-fix terminal proof, but the current diff still exposes it through the shared Gateway request resolver used by Responses without matching Responses continuation-scope docs/tests, and the added docs heading violates the scoped Mintlify heading rule.

## What This Changes

The PR adds a new `x-openclaw-session` Gateway HTTP header, documentation/changelog text, and Chat Completions tests for stable agent-scoped session keys.

## Best Possible Solution

Make the new session-hint contract explicit by either restricting it to Chat Completions or updating Responses continuation scope, docs, and tests consistently.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID when OpenAI-compatible callers omit `user`, and the PR body includes after-fix terminal curl/log output showing stable session-key reuse and a warm-session result.

## Solution Assessment

No, not as currently patched. The stable hint is a plausible narrow fix, but adding it in the shared resolver silently creates a Responses API contract without matching continuation-scope semantics, docs, or tests.
