## Summary

Keep this PR open: the Chat Completions stable-session idea is useful and has terminal proof, but the branch still changes shared Responses continuity unintentionally and has concrete lint/docs blockers.

## What This Changes

The branch adds a validated `x-openclaw-session` header in the shared Gateway request-context resolver, Chat Completions tests, OpenAI HTTP docs, and a changelog entry.

## Best Possible Solution

Land a documented Chat Completions-only stable session hint with bounded validation and regression coverage, while leaving Responses continuity unchanged unless that contract is deliberately documented and tested.

## Reproduction Assessment

Yes, at source level. Current main lacks an `x-openclaw-session` branch and falls back to UUID-backed stateless sessions when callers cannot set `user`; the PR body also includes terminal proof for the intended Chat Completions path.

## Solution Assessment

No, not as currently patched. The stable hint is a reasonable narrow API for Chat Completions, but adding it in the shared resolver also changes Responses `previous_response_id` continuity without matching scope, docs, or tests.
