## Summary

Keep open: current main still lacks the stable `x-openclaw-session` header, and the PR remains useful, but the proposed implementation still changes the shared Chat Completions/Responses resolver without matching Responses docs, continuity scoping, or tests. The branch also still has concrete lint/docs blockers to repair before maintainer API review.

## What This Changes

The PR adds an `x-openclaw-session` Gateway HTTP header with validation, Chat Completions precedence tests, OpenAI HTTP docs, and a changelog entry for stable agent-scoped sessions.

## Best Possible Solution

Land the stable Chat Completions session hint with explicit endpoint scope, bounded validation, matching docs/tests, and Responses support only if its continuity contract is updated deliberately.

## Reproduction Assessment

Yes, at source level. Current main falls through to UUID-backed session keys when no `user` is supplied, while the PR body includes terminal curl/log proof showing repeated header reuse after the patch.

## Solution Assessment

No, not as currently patched. The header is a reasonable narrow direction for Chat Completions, but placing it in the shared resolver changes Responses behavior without matching docs, tests, or previous-response scoping.
