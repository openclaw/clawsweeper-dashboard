## Summary

Keep this PR open, but it still needs changes before merge: the chat-session header approach is plausible and has real terminal proof, while the current head still has CI lint/docs blockers and a shared OpenResponses session-continuity bug introduced by adding the header in the generic Gateway request-context helper.

## What This Changes

The PR adds a validated `x-openclaw-session` header in shared Gateway request-context resolution, documents it for OpenAI Chat Completions, adds chat precedence tests, and updates the changelog.

## Best Possible Solution

Land a lint-clean, docs-clean implementation that either keeps `x-openclaw-session` chat-only or updates OpenResponses scope/tests/docs for the shared header, while leaving light-context work tracked separately.

## Reproduction Assessment

Yes. Source inspection on current main shows `x-openclaw-session` is ignored and requests without `user` fall through to a random UUID session key; the PR body provides after-fix terminal output showing repeated header use reuses one session.

## Solution Assessment

No, not as currently patched. A validated lower-priority agent-scoped header is a narrow solution for the Chat Completions warm-session subproblem, but the shared helper also changes OpenResponses without matching continuity scoping and the current head still fails lint/docs gates.
