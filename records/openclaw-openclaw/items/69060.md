## Summary

Keep open: the stable-session header is a plausible fix path and the PR includes sufficient terminal proof, but the diff still creates an implicit `/v1/responses` header contract through the shared resolver and leaves a docs heading that conflicts with the scoped Mintlify rules.

## What This Changes

The branch adds a validated `x-openclaw-session` Gateway HTTP header, Chat Completions precedence tests, docs, and a changelog entry for stable agent-scoped sessions.

## Best Possible Solution

Make the header contract explicit, then either scope it to Chat Completions or update Responses docs, continuation scoping, and tests; also rename the docs heading to Mintlify-safe wording.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID when callers omit `user`, and the PR body includes after-fix terminal curl/log output showing stable session-key reuse and a warm-session QMD skip.

## Solution Assessment

No, not yet. The stable hint is a plausible narrow solution, but the patch should first resolve the endpoint scope and docs-anchor rule before merge.
