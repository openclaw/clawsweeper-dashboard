## Summary

Keep open: the Chat Completions session hint is useful and has after-fix terminal proof, but the current diff adds it in a shared Gateway resolver so `/v1/responses` inherits an undocumented session control with unresolved continuation semantics.

## What This Changes

Adds an `x-openclaw-session` Gateway HTTP header, docs, changelog entry, and Chat Completions regression tests for stable agent-scoped session hints.

## Best Possible Solution

Keep the stable-session hint Chat Completions-only unless maintainers explicitly choose a Responses-wide contract with docs, continuation-scope semantics, and regression coverage in the same change.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID without `x-openclaw-session`, and the PR body includes after-fix curl/log output showing stable Chat Completions reuse with the new header.

## Solution Assessment

No, not yet. The stable hint is plausible for Chat Completions, but the implementation must either keep it out of Responses or define Responses-wide docs, scope matching, and tests.
