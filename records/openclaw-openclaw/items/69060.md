## Summary

Keep open. The PR is a useful narrow Gateway improvement with sufficient terminal proof, but it still needs an explicit public API scope decision and a docs heading fix before merge.

## What This Changes

The PR adds a validated `x-openclaw-session` Gateway HTTP header in the shared session resolver, documents it, adds Chat Completions precedence tests, and adds a changelog entry.

## Best Possible Solution

Make the header contract explicit, then align Chat Completions, Responses continuation semantics, docs, and focused tests to that chosen scope.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID when callers omit `user`, and the PR body includes after-fix terminal curl/log output showing stable session-key reuse and a warm-session result.

## Solution Assessment

No, not yet. The stable hint is a plausible narrow fix, but the patch should first make the endpoint/API scope explicit and use Mintlify-safe docs wording.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope the new header before sharing the resolver:** `src/gateway/http-utils.ts:154-156`
  - body: `resolveSessionKey` is shared by Chat Completions and Responses, so adding `x-openclaw-session` here silently creates a `/v1/responses` header contract too. Either keep the hint Chat Completions-only or document/test Responses and include the new hint in the continuation-scope contract.
  - confidence: 0.9

  - body: The new heading contains an em dash, but `docs/AGENTS.md` says docs headings should avoid em dashes because Mintlify anchors are brittle. Rename the heading so docs checks and anchors stay stable.
  - confidence: 0.87
