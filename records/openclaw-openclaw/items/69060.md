## Summary

## What This Changes

The branch adds an `x-openclaw-session` request header, docs, a changelog entry, and Chat Completions tests for stable agent-scoped HTTP session hints.

## Best Possible Solution

Decide whether `x-openclaw-session` is Chat Completions-only or a shared OpenAI-compatible contract, then align the resolver placement, Responses semantics, docs, and tests before merge.

## Reproduction Assessment

Yes, at source level. Current main falls back to a random UUID when no explicit session key or `user` is present, and the PR body includes after-fix curl/log output showing stable Chat Completions reuse with the new header.

## Solution Assessment

No, not yet. The stable hint is a plausible narrow solution for Chat Completions, but the shared resolver placement creates an undocumented Responses contract unless endpoint scope is narrowed or Responses is fully updated.
