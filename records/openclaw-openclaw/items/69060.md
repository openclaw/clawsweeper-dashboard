## Summary

Keep open. Current main still lacks the stable agent-scoped session hint, and the PR is a plausible narrow implementation, but it needs public API documentation and release-note follow-through before it should ship.

## What This Changes

The PR updates Gateway OpenAI-compatible session resolution to accept `x-openclaw-session` as an agent-scoped stable fallback after `x-openclaw-session-key` and the OpenAI `user` field.

## Best Possible Solution

Ship the stable agent-scoped session hint with explicit docs, a focused OpenAI HTTP test for precedence and cross-agent separation, and a changelog entry, while leaving the raw `x-openclaw-session-key` routing behavior to #74679 and the broader light-context work to #68920.

## Reproduction Assessment

Yes. Current main can be inspected at `resolveSessionKey`: without `user`, it returns a random UUID-backed `agent:<agentId>:openai:<uuid>` key; a focused HTTP test can prove the PR behavior by sending the same `x-openclaw-session` value twice and expecting the same `agent:<agentId>:openai-session:<hint>` key.

## Solution Assessment

Yes, with follow-through. Adding a lower-priority agent-scoped header is the narrowest maintainable fix for callers that cannot set the OpenAI `user` field, because it preserves the existing full-override semantics of `x-openclaw-session-key`.
