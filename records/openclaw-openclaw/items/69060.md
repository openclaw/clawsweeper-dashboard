## Summary

Keep open. Current main still lacks the stable agent-scoped `x-openclaw-session` hint, and the PR is a narrow implementation of the QMD re-index overhead part of the related TTFB report. I did not find a blocking correctness or security issue in the diff, but it adds a public Gateway HTTP header and overlaps the still-open raw session-key routing contract discussion, so it should get maintainer review with docs and focused coverage before merge.

## What This Changes

The PR updates `src/gateway/http-utils.ts` so OpenAI-compatible HTTP session resolution accepts `x-openclaw-session` as an agent-scoped stable fallback after `x-openclaw-session-key` and `user`, before the random UUID fallback.

## Best Possible Solution

Land a documented and tested agent-scoped stable session hint, either via this PR or an equivalent narrow change, while keeping the raw `x-openclaw-session-key` full-override behavior explicit and resolving the separate raw-header routing contract in #74679.

## Reproduction Assessment

Yes. Current main can be checked at `resolveSessionKey`: repeated `/v1/chat/completions` requests without `user` still get random UUID-backed `agent:<id>:openai:<uuid>` keys, while a focused gateway test for the PR should assert the same `x-openclaw-session` value produces the same `agent:<id>:openai-session:<hint>` key and different agents remain separated.

## Solution Assessment

Yes, with docs and tests. The PR’s ordering is the narrowest maintainable fix for stable agent-scoped HTTP sessions because it avoids changing existing `x-openclaw-session-key` full-override semantics and leaves the broader light-context work to #68920.
