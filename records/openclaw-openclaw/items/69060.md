## Summary

Keep open; the direction is useful and has real after-fix proof, but the current branch still changes Responses session behavior unintentionally and has repo-gate failures that need repair before merge.

## What This Changes

The PR adds a validated `x-openclaw-session` header to Gateway session resolution, plus Chat Completions tests, OpenAI HTTP docs, and a changelog entry.

## Best Possible Solution

Land a Chat Completions-scoped, validated stable-session header that preserves `x-openclaw-session-key` full override semantics and leaves broader light-context work to the canonical latency issue.

## Reproduction Assessment

Yes, at source level. Current main has no `x-openclaw-session` branch, so Chat Completions requests without `user` still fall through to the UUID session fallback; I did not run a live gateway because this review is read-only.

## Solution Assessment

No, not as currently patched. The stable hint is a reasonable narrow API for Chat Completions, but adding it in the shared resolver also changes Responses continuity semantics without matching scope, docs, or tests.
