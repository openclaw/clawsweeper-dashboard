## Summary

Keep open: current main does not implement the stable session hint, but this branch still changes Responses continuity through the shared resolver and has direct lint/docs blockers before it is mergeable.

## What This Changes

The branch adds a validated `x-openclaw-session` Gateway HTTP header, Chat Completions tests, OpenAI HTTP docs, and a changelog entry for stable agent-scoped session hints.

## Best Possible Solution

Land a documented Chat Completions stable session hint with bounded validation and regression coverage, or deliberately extend the same contract to Responses with matching continuity scope and docs.

## Reproduction Assessment

Yes, at source level. Current main has no `x-openclaw-session` branch, so callers that cannot set OpenAI `user` still fall through to UUID-backed stateless sessions; the PR body also includes terminal proof, but I did not rerun a live Gateway.

## Solution Assessment

No, not as currently patched. The stable hint is a reasonable narrow API for Chat Completions, but adding it in the shared resolver also changes Responses continuity without matching scope, docs, or tests.
