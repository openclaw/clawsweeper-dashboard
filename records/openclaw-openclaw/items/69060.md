Keep open: the PR addresses a real source-level gap and includes sufficient after-fix terminal proof, but the current head still introduces unintended `/v1/responses` session semantics and has red lint/docs checks that need branch repair before merge.

## What This Changes

The PR adds a validated `x-openclaw-session` Gateway header for stable agent-scoped HTTP sessions, with Chat Completions tests, OpenAI HTTP docs, and a changelog entry.

## Best Possible Solution

Land a docs-clean Chat Completions-scoped stable-session header that preserves `x-openclaw-session-key` full override behavior and leaves the broader light-context work to https://github.com/openclaw/openclaw/issues/68920.

## Reproduction Assessment

Yes. Source inspection shows current main lacks `x-openclaw-session`, so requests without `user` still fall through to a UUID session key; the PR body also provides after-fix terminal/log proof for repeated header reuse.

## Solution Assessment

No, not as currently patched. The stable hint is a reasonable narrow API for Chat Completions warm sessions, but the shared resolver changes `/v1/responses` without matching continuity scope, docs, or tests; the safer repair is to keep the hint Chat Completions-only unless Responses semantics are fully defined.
