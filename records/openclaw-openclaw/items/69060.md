## Summary

Keep open. Current main still lacks the proposed stable `x-openclaw-session` path, and the PR’s code direction is narrow, but the public Gateway header needs docs, changelog, and focused precedence/scoping tests before merge.

## What This Changes

The PR changes Gateway OpenAI-compatible session resolution so `x-openclaw-session` becomes an agent-scoped stable fallback after `x-openclaw-session-key` and `user`, before the random UUID fallback.

## Best Possible Solution

Ship the stable session hint with explicit OpenAI HTTP docs, focused precedence and cross-agent tests, and a single Unreleased fix entry, while leaving #68920 and #74679 as separate follow-up tracks.

## Reproduction Assessment

Yes. Source inspection shows current main ignores `x-openclaw-session` and falls through to the random UUID path when `user` is absent, and the existing OpenAI HTTP test harness already exposes `agentCommand.sessionKey` for a focused regression test.

## Solution Assessment

Yes, with follow-through. A lower-priority agent-scoped header is the narrowest maintainable fix for callers that cannot set the OpenAI `user` field, as long as it does not change the full-override semantics of `x-openclaw-session-key`.
