## Summary

Keep open: the PR addresses a real source-level gap for stateless Chat Completions callers, but the current patch still changes the shared Responses session resolver without matching continuity scope, docs, or tests, and the latest relevant PR checks still fail lint/docs.

## What This Changes

The PR adds a validated `x-openclaw-session` Gateway header in shared HTTP request-context session resolution, plus Chat Completions tests, OpenAI HTTP docs, and a changelog entry.

## Best Possible Solution

Land a documented Chat Completions stable session hint with bounded validation and regression tests, or intentionally extend the same contract to Responses with matching continuity scope, while leaving the larger light-context latency work to https://github.com/openclaw/openclaw/issues/68920.

## Reproduction Assessment

Yes, at source level. Current main has no `x-openclaw-session` branch, so callers that cannot set OpenAI `user` still fall through to UUID-backed stateless sessions; I did not run a live Gateway.

## Solution Assessment

No, not as currently patched. The stable hint is a reasonable narrow API for Chat Completions, but adding it in the shared resolver also changes Responses continuity without matching scope, docs, or tests.
