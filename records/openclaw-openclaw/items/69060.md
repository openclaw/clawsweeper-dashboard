## Summary

Keep open. The PR is useful and the body now includes sufficient after-fix terminal proof, but the current branch still broadens the new session hint into `/v1/responses` without matching continuation scope/docs/tests, and the latest CI logs show lint and docs formatting failures.

## What This Changes

The PR adds a validated `x-openclaw-session` header in the shared Gateway request-context resolver, documents it for OpenAI Chat Completions, adds a changelog entry, and adds Chat Completions session-precedence tests.

## Best Possible Solution

Land a lint-clean, docs-clean implementation that either scopes the new hint to Chat Completions only or updates Responses continuity scope, docs, and tests consistently while preserving `x-openclaw-session-key` full-override semantics.

## Reproduction Assessment

Yes, source-level: current main has no `x-openclaw-session` branch, so requests without `user` still fall through to a UUID-backed session key. The PR body supplies after-fix terminal proof for the intended warm-session behavior.

## Solution Assessment

No, not as currently patched. The stable header is a narrow fit for Chat Completions warm sessions, but the shared resolver changes `/v1/responses` without matching continuity scope, docs, and tests, and the latest lint/docs gates are still red.
