## Summary

Keep open. The PR is useful and has sufficient terminal proof, but the current patch still silently changes the shared Responses session resolver without matching contract/tests, and the docs check is red on the new OpenAI HTTP docs text.

## What This Changes

The PR adds an `x-openclaw-session` Gateway HTTP header, docs, tests, and a changelog entry for stable agent-scoped Chat Completions sessions.

## Best Possible Solution

Land a scoped stable Chat Completions session hint after either gating it to Chat Completions or explicitly extending the Responses contract, with formatted docs and focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main falls through to UUID-backed stateless sessions when a Chat Completions caller cannot set `user`, and the PR body includes curl/log proof that the new header reuses the session after the patch. I did not run a live Gateway in this read-only review.

## Solution Assessment

No, not as currently patched. The stable session hint is a reasonable narrow direction, but the patch needs an explicit endpoint scope decision plus docs formatting before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope the session hint before it reaches Responses:** `src/gateway/http-utils.ts:154-156`
  - body: `resolveGatewayRequestContext` is shared by Chat Completions and Responses, so reading `x-openclaw-session` here silently adds the new header to `/v1/responses`. That endpoint's `previous_response_id` continuity scope only tracks `x-openclaw-session-key`, so a later Responses request can provide a different session hint and still be routed back to the previous response session; either keep this hint Chat Completions-only or document/test it as a Responses contract and include it in the response-session scope.
  - confidence: 0.88
- **[P2] Reflow the new OpenAI HTTP docs text:** `docs/gateway/openai-http-api.md:90`
  - body: The live `check-docs` job fails because `docs/gateway/openai-http-api.md` is not formatted after this addition. Reflow the new optional-header/session section and avoid the em-dash heading called out by the scoped docs guide so the docs check can pass.
  - confidence: 0.9
