## Summary

The reported disabled-job count path does not match current main or the latest release source: the top-right hidden badge is a chat cron-session filter, while the Cron Jobs tab count is driven by the same filtered `cron.list` page that renders the job rows.

## What This Changes

Report requests that the Web Control UI cron badge count match visible Cron Jobs list entries instead of showing a hidden count.

## Best Possible Solution

Keep the current separation: the chat control reports hidden cron sessions, and the Cron Jobs tab shows its own shown/total count from `cron.list`; a future copy tweak can be discussed separately if users keep confusing those surfaces.

## Reproduction Assessment

No. Source inspection of current main and `v2026.5.12` shows the Cron Jobs list total and rendered rows come from the same filtered `cron.list` page, while the only tooltip saying hidden is a separate cron-session chat filter.

## Solution Assessment

Not applicable as a code fix. Changing the top-right chat cron-session filter to display Cron Jobs list counts would conflate two UI surfaces; the safer current outcome is to leave the code as-is unless maintainers want a wording change.
