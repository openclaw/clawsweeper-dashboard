## Summary

The reported deadlock affected v2026.6.5, but the latest release now periodically probes a single-provider primary through the existing throttle; a successful restored-provider call then clears the persisted billing disable state.

## What This Changes

Allow an Anthropic profile disabled by a billing failure to recover before the configured five-hour backoff expires after credits are restored.

## Best Possible Solution

Keep the shipped throttled primary-profile probe and existing successful-use cleanup as the canonical recovery mechanism rather than adding a second manual cooldown-reset path.

## Reproduction Assessment

No for current main: the former deadlock is source-reproducible in v2026.6.5, but v2026.6.6 and current main contain the throttled billing recovery path. No live exhausted-and-restored Anthropic account was exercised during this read-only review.

## Solution Assessment

Yes. Reusing the normal provider request and successful-use cleanup behind a bounded throttle is narrower and safer than introducing a parallel CLI or direct state-reset mechanism.
