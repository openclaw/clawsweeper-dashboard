## Summary

The specific v2026.6.5 deadlock is now fixed: current main and v2026.6.6 allow billing-disabled single-provider profiles to make a bounded cooldown probe, and a successful dispatched call clears the persisted disabled state through the existing success path.

## What This Changes

The issue requests an early recovery path for a billing-disabled Anthropic auth profile after credits are restored, instead of waiting the full persisted 5-hour backoff.

## Best Possible Solution

Keep the shipped bounded half-open probe for the single-provider billing-disabled recovery path, while leaving broader same-auth fallback and re-auth cleanup policy to the canonical tracker and open PR.

## Reproduction Assessment

No, not against current main or v2026.6.6. The old v2026.6.5 path is source-explained, but current code permits a billing cooldown probe and then clears disabled state on success.

## Solution Assessment

Yes. The implemented half-open probe is the narrow preferred fix from the issue: it reuses existing fallback throttling and markAuthProfileSuccess instead of adding a separate reset command for the central failure.
