## Summary

Keep this PR open for maintainer judgment, but do not merge it as-is. Current main already repaired the silent group/channel delivery failure by keeping the message tool available and falling back when it is not, while this PR intentionally changes the documented privacy contract by making @mentioned group/channel final replies automatic, including when tool-only delivery was explicitly requested.

## What This Changes

The PR adds a `WasMentioned` reply-delivery context flag and tests so group/channel @mentions convert `message_tool_only` source reply delivery to `automatic`, including requested modes.

## Best Possible Solution

Keep the current message-tool-only group/channel default unless maintainers explicitly approve an @mention automatic-delivery exception. If approved, implement it with clear precedence for explicit config/requested modes, update group-channel docs and changelog, and cover Discord/Slack/channel paths together.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: current main source and tests show mentioned Discord guild messages still default to `message_tool_only`, while #75382 covers the real silent-delivery failure by allowing message-tool delivery or automatic fallback when the tool is unavailable.

## Solution Assessment

Is this the best way to solve the issue? No: the PR overrides requested tool-only delivery and conflicts with the documented group/channel privacy contract; the safer solution is the current fallback/message-tool path unless maintainers choose and document a new @mention exception.
