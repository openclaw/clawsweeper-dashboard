## Summary

Keep this PR open, but do not merge as-is: the patch makes @mentions override the current group/channel tool-only delivery contract, including explicit/pre-resolved `message_tool_only` modes, without a maintainer-approved visibility policy or matching docs/changelog updates.

## What This Changes

The PR adds `WasMentioned` handling and resolver tests so group/channel @mentions switch `message_tool_only` source reply delivery to `automatic`, including requested modes.

## Best Possible Solution

Keep a clear, documented precedence rule for group/channel visibility: either preserve explicit tool-only delivery for @mentions, or accept an @mention exception with matching docs, changelog, and channel-level tests.

## Reproduction Assessment

Yes. Current main can be verified from the resolver, dispatch suppression path, and Discord process test: mentioned group/channel turns still pre-resolve to `message_tool_only`, while this PR’s resolver additions would make those turns automatic.

## Solution Assessment

No. The proposed generic override is broader than the current contract because it bypasses explicit/pre-resolved tool-only delivery; the safer path is a maintainer-approved policy with documented precedence before changing automatic room delivery.
