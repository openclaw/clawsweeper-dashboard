## Summary

Keep open: the patch is not ready because it changes the current group/channel visibility contract by auto-posting final replies for @mentions, including explicit `message_tool_only` requests, without maintainer approval or public docs/changelog updates.

## What This Changes

The PR adds `WasMentioned` to source reply delivery resolution and tests so group/channel @mentions convert `message_tool_only` delivery to `automatic`.

## Best Possible Solution

Keep the current tool-only group/channel default unless maintainers explicitly approve an @mention exception; if approved, implement it with documented precedence, docs/config help, changelog, and channel-level coverage.

## Reproduction Assessment

Yes. Current main source and tests show mentioned Discord guild messages still pre-resolve to `message_tool_only`, while the PR diff makes a focused resolver case with `ChatType: "group"`, `WasMentioned: true`, and requested `message_tool_only` return `automatic`.

## Solution Assessment

No. The proposed fix bypasses explicit tool-only delivery and conflicts with the documented visible-reply contract; the safer current path is message-tool delivery with automatic fallback only when the message tool is unavailable or config opts into automatic replies.
