## Summary

Keep this PR open for maintainer review: it targets a real group/channel reply surprise, but the current diff changes the documented privacy contract by letting @mentions bypass `message_tool_only`, including explicit requested tool-only delivery.

## What This Changes

The PR adds `WasMentioned` to source reply delivery resolution and tests so group/channel @mentions convert `message_tool_only` delivery to automatic final replies.

## Best Possible Solution

Keep the current tool-only group/channel default unless maintainers explicitly approve a documented @mention exception with clear precedence for explicit requested modes and channel pre-resolution paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: current main has a Discord process test where a mentioned guild turn still resolves to `message_tool_only`, and the resolver/dispatch path shows automatic delivery is suppressed unless config or message-tool availability changes it.

## Solution Assessment

Is this the best way to solve the issue? No: overriding `message_tool_only` for mentioned turns bypasses the documented privacy/tool contract; the safer path is either preserve tool-only delivery or make an approved, documented exception that does not silently defeat explicit requested modes.
