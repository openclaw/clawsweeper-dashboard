## Summary

Keep open: the PR addresses a real source-level silent-reply path, but the latest diff is not merge-ready because it misses pre-resolved delivery paths and still lacks required real channel proof plus maintainer approval for the visible-room delivery policy change.

## What This Changes

The PR adds `WasMentioned` to source reply delivery resolution and tests automatic group/channel visible delivery for mentioned turns when no visible-reply mode is configured.

## Best Possible Solution

If maintainers want @mentions to auto-deliver, implement one policy that distinguishes synthesized defaults from explicit operator tool-only configuration across generic and pre-resolved channel paths, update docs/tests, and verify with redacted real channel proof.

## Reproduction Assessment

Yes, source-reproducible: current main maps unconfigured group/channel finals to `message_tool_only`, and dispatch drops ordinary finals under that suppression. I did not establish a live channel reproduction in this read-only review.

## Solution Assessment

No, this PR is not the best mergeable form yet. The resolver is a plausible fix plane, but the latest diff preserves requested `message_tool_only` modes used by pre-resolved channel paths and needs an explicit maintainer decision for the @mention visible-delivery exception.
