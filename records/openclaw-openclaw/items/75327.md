## Summary

Keep open: the PR is a focused patch, but it changes the current group/channel visibility contract by turning mentioned room replies into automatic visible final replies, which needs maintainer product/privacy approval before merge.

## What This Changes

The PR adds `WasMentioned` to source reply delivery mode resolution and tests so group/channel @mentions convert `message_tool_only` delivery, including requested modes, to `automatic`.

## Best Possible Solution

Keep the current tool-only group/channel default unless maintainers explicitly approve an @mention exception; if approved, document the exception, add a changelog entry, and cover the affected channel pre-resolution path.

## Reproduction Assessment

Yes: current main source and tests show mentioned Discord guild messages still resolve to `message_tool_only`, and a focused resolver case with `ChatType: "group"`, `WasMentioned: true`, and requested `message_tool_only` demonstrates exactly what this PR changes.

## Solution Assessment

No: this is not yet the best fix because it bypasses current tool-only delivery policy for room replies. The safer path is maintainer approval first, then a documented exception with channel-level coverage if that policy is chosen.
