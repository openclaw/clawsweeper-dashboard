## Summary

Keep this PR open for explicit maintainer product/privacy judgment, not conservative cleanup. Current main deliberately separates mention gating from visible final-reply delivery: @mentions can trigger processing, but group/channel final replies remain `message_tool_only` by default unless configured automatic or the message tool is unavailable. This PR would add a new @mention exception that overrides even requested tool-only delivery, which conflicts with current docs/tests and the adjacent merged fallback fix.

## What This Changes

The PR adds `WasMentioned` to source reply delivery mode resolution and tests so group/channel @mentions convert `message_tool_only` delivery to `automatic`, including requested modes.

## Best Possible Solution

Keep the current tool-only group/channel default unless maintainers explicitly approve a documented @mention exception. If approved, implement it with clear precedence for explicit config/requested modes and cover the generic resolver plus Discord/Slack-style pre-resolution paths together.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the current-main behavior: the resolver and Discord process test show mentioned guild messages still resolve to `message_tool_only` by default, while dispatch tests cover automatic fallback when the message tool is unavailable.

## Solution Assessment

Is this the best way to solve the issue? No. The current PR bypasses explicit/requested tool-only delivery and conflicts with the documented group/channel privacy contract; the safer path is either the existing message-tool/fallback behavior or a maintainer-approved, documented exception.
