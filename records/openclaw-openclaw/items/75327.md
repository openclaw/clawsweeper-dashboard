## Summary

Keep open: current main still has the claimed source path, and related comments show real user impact, but this PR is not merge-ready because it changes the documented group/channel visibility contract, overrides explicit/requested tool-only delivery, and lacks after-fix real behavior proof.

## What This Changes

This PR adds `WasMentioned` to source reply delivery context and changes resolver/tests so @mentioned group/channel turns, including requested `message_tool_only`, resolve to `automatic` delivery.

## Best Possible Solution

Have maintainers choose the @mention precedence rule for `message_tool_only`; if accepted, preserve explicit opt-outs or document the exception, update public docs and channel coverage, and require real group/channel proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main resolves group/channel turns to `message_tool_only` and dispatch suppresses normal final replies, while `WasMentioned` is not part of the resolver context. I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as written. The resolver plane is plausible, but overriding requested/configured tool-only delivery is broader than the current contract; a maintainable fix needs an approved precedence rule, docs, channel coverage, and real behavior proof.
