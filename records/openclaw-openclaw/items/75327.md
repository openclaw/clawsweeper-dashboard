## Summary

Keep open: current main intentionally keeps group/channel final replies tool-only by default, including mentioned Discord guild turns, and this PR changes that visibility contract by forcing @mentioned tool-only turns to automatic delivery. The patch needs maintainer product/privacy approval and public docs/changelog coverage before it is safe to merge.

## What This Changes

The PR adds `WasMentioned` to the source reply delivery context, changes the resolver so @mentioned group/channel `message_tool_only` turns resolve to `automatic`, and adds resolver/visibility unit coverage.

## Best Possible Solution

Keep the current tool-only group/channel default unless maintainers explicitly approve an @mention automatic-delivery exception. If approved, implement the exception with clear precedence for explicit operator config/requested modes, update group/channel and channel-specific docs plus `CHANGELOG.md`, and add channel-level coverage for Discord/Slack-style pre-resolution paths.

## Reproduction Assessment

Yes. A focused resolver case on the PR head with `ChatType: "group"`, `WasMentioned: true`, and `requested: "message_tool_only"` demonstrates the changed behavior, while current main docs and Discord tests show that mentioned group/channel turns still use tool-only delivery by default.

## Solution Assessment

No. The current patch is not the safest fix because it treats @mention as higher precedence than `message_tool_only`, including explicit/pre-resolved tool-only delivery; the safer path is a maintainer-approved policy change with documented precedence and channel coverage.
