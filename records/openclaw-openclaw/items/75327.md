## Summary

Keep open: this PR targets a source-reproducible shared-room reply problem with useful corroboration from related reports, but the patch still changes the current tool-only visibility contract, lacks after-fix real behavior proof, and needs maintainer policy/security judgment before merge.

## What This Changes

The PR adds `WasMentioned` handling to source reply delivery resolution and tests so mentioned group/channel turns switch from `message_tool_only` to `automatic`.

## Best Possible Solution

Decide the @mention visibility rule explicitly; if accepted, scope the resolver to that policy, update public group/channel docs and channel coverage, and require redacted live proof from at least one affected real channel.

## Reproduction Assessment

Yes, source-reproducible: current main resolves mentioned group/channel turns to `message_tool_only` in the relevant defaults and dispatch skips normal final replies while suppression is active. I did not establish a live channel reproduction in this read-only review.

## Solution Assessment

No, not as written. The resolver is a plausible policy plane, but the generic `WasMentioned` override bypasses explicit/requested tool-only delivery and conflicts with the current documented shared-room visibility contract.
