## Summary

Keep open: the current-main symptom is source-reproducible, but this PR changes the shared-room visibility contract by auto-posting final replies on @mention, still needs maintainer policy approval, docs alignment, and real behavior proof before merge.

## What This Changes

The PR adds `WasMentioned` handling to source reply delivery resolution so mentioned group/channel turns switch from `message_tool_only` to `automatic`, with resolver and visibility-policy unit tests.

## Best Possible Solution

Have maintainers decide the @mention precedence rule for tool-only group/channel rooms; if accepted, preserve any explicit opt-out semantics they require, update public docs and channel coverage, and require redacted live proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves mentioned group/channel turns to `message_tool_only`, Discord coverage locks that behavior for `effectiveWasMentioned: true`, and dispatch skips normal final replies when delivery is suppressed. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No, not as written. The resolver is a plausible policy plane, but overriding requested/configured tool-only delivery is broader than the documented contract and should wait for maintainer policy approval, docs, and real behavior proof.
