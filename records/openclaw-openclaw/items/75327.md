## Summary

Keep this PR open, but the current patch is not ready to merge: it changes the group/channel visibility contract by making @mentions bypass tool-only delivery, including explicit or channel-precomputed `message_tool_only` modes, without maintainer product/privacy approval or matching docs.

## What This Changes

The PR adds `WasMentioned` handling and resolver tests so group/channel @mentions convert `message_tool_only` source reply delivery to `automatic`, including requested modes.

## Best Possible Solution

Keep tool-only group/channel delivery as the default unless maintainers explicitly approve an @mention exception; if approved, preserve clear precedence rules and update docs, changelog, and Discord/Slack-style coverage in the same change.

## Reproduction Assessment

Yes. Current main can be checked with the resolver and Discord process tests: mentioned guild messages still pre-resolve to `message_tool_only`, while the PR’s added resolver case would turn `requested: "message_tool_only"` plus `WasMentioned: true` into `automatic`.

## Solution Assessment

No. The proposed implementation is broader than the current contract because it bypasses requested tool-only delivery; the safer path is an explicit maintainer policy decision followed by a documented, tested precedence rule.
