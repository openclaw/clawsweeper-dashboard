## Summary

Keep this PR open, but not merge-ready: the source path is real, yet the patch changes the documented shared-room visibility policy by making @mentions bypass `message_tool_only`, including explicit/requested tool-only modes, and it still lacks after-fix real behavior proof.

## What This Changes

The PR adds `WasMentioned` to source reply delivery resolution so mentioned group/channel turns resolve from `message_tool_only` to `automatic`, with unit coverage for default, requested, and policy cases.

## Best Possible Solution

Have maintainers choose the @mention precedence rule; if accepted, update docs and channel coverage, preserve any explicit opt-out semantics they require, and require redacted real group/channel proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main resolves mentioned group/channel turns to `message_tool_only` and dispatch suppresses normal final replies. I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as written. The resolver is a plausible place for a policy exception, but overriding requested/configured tool-only delivery is broader than the documented contract and needs maintainer approval, docs, and real behavior proof.
