## Summary

Keep open: the reported no-visible-reply path is source-reproducible, but this PR changes the visible-reply policy too broadly, bypasses explicit tool-only choices, and still lacks contributor real behavior proof.

## What This Changes

The PR adds `WasMentioned` to source reply delivery resolution so @mentioned group/channel turns switch from `message_tool_only` to `automatic`, with resolver and visibility-policy tests.

## Best Possible Solution

Decide the mention visibility policy explicitly; if accepted, implement a group/channel-scoped exception that preserves explicit tool-only settings, update docs/tests for the affected channels, and require redacted real-channel proof.

## Reproduction Assessment

Yes, source-reproducible: current main maps group/channel source turns to `message_tool_only` unless automatic visible replies are configured, and dispatch skips ordinary final replies while suppression is active. I did not establish a live channel reproduction in this read-only review.

## Solution Assessment

No, not as merge-ready. The resolver is a plausible policy plane, but this implementation bypasses tool-only semantics for direct or explicitly tool-only paths and needs maintainer policy approval plus real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Limit mention overrides to shared-room turns:** `src/auto-reply/reply/source-reply-delivery-mode.ts:30-33`
  - body: This override checks only `WasMentioned`, but current Microsoft Teams direct messages set `WasMentioned: true`. A direct chat configured or requested as `message_tool_only` can therefore become `automatic`, bypassing the documented direct/tool-only delivery mode.
  - confidence: 0.91
- **[P2] Preserve explicit tool-only delivery policy:** `src/auto-reply/reply/source-reply-delivery-mode.ts:52-53`
  - body: The override flips `message_tool_only` to `automatic` even when that mode came from explicit requested/pre-resolved policy or `visibleReplies: "message_tool"`. If maintainers accept a mention exception, the code needs to distinguish the unset group/channel default from an operator-selected tool-only policy.
  - confidence: 0.89
