## Summary

Keep open; this PR changes the current group/channel visibility contract by making @mentions bypass `message_tool_only`, including requested or pre-resolved delivery modes, and the override is broad enough to affect non-room contexts that carry `WasMentioned`.

## What This Changes

The PR adds `WasMentioned` handling to source reply delivery resolution and tests so mentioned group/channel turns convert `message_tool_only` delivery to `automatic`.

## Best Possible Solution

Maintainers should first decide whether @mentions are an approved visible-reply exception; if approved, the implementation should be scoped to room chats with explicit precedence, docs, changelog, and channel coverage.

## Reproduction Assessment

Yes. Current main can be checked through the resolver, Discord process test, and dispatch visibility policy: mentioned guild messages still carry `message_tool_only`, and that mode suppresses automatic source delivery.

## Solution Assessment

No. The PR solves the visible @mention complaint by overriding `message_tool_only` globally, but that is broader than the current configured/requested delivery semantics and needs an explicit policy decision plus tighter scoping.
