## Summary

Keep this PR open, but it is not merge-ready: the current patch fixes the generic resolver path for unconfigured mentioned group/channel turns, while still missing channels that pre-resolve `sourceReplyDeliveryMode` before dispatch and still lacking after-fix real behavior proof.

## What This Changes

The PR adds `WasMentioned` to source reply delivery resolution and tests automatic group/channel delivery for mentioned turns when visible reply mode is unset.

## Best Possible Solution

If maintainers accept @mention auto-delivery, implement it across both generic and pre-resolved channel paths while preserving explicit `message_tool` policy, update docs/tests for the chosen contract, and require redacted real channel proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves unconfigured group/channel finals to `message_tool_only`, and dispatch drops ordinary finals under that suppression. I did not establish a live channel reproduction in this read-only review.

## Solution Assessment

No, not merge-ready. The resolver is a plausible fix plane, but the current patch misses channels that pass a pre-resolved requested mode and needs an explicit maintainer decision for the @mention visibility exception.
