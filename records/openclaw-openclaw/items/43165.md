## Summary

Keep open. The channel-icon feature is not implemented on current main, and https://github.com/openclaw/openclaw/issues/53378 was already deduped to this PR as the canonical implementation thread, but the PR head still has source-reproducible rendering bugs and lacks required real Control UI proof.

## What This Changes

The branch adds Control UI channel icons to session metadata rows and channel-card titles, updates GatewaySessionRow typing/CSS, and adds sessions rendering tests.

## Best Possible Solution

Finish the feature in a rebased implementation that shares channel display metadata, handles direct and dm session-key delimiters, falls back for all configured or supported channels, keeps focused rendering tests, and includes real Control UI proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection: PR head only extracts peers after direct while its own dm-scoped WhatsApp test expects the phone number, and unsupported hard-coded channel ids return nothing. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The feature direction is useful, but the branch should reuse shared channel display metadata, handle dm keys, keep supported plugin channels visible with a fallback, and provide real UI proof before merge.
