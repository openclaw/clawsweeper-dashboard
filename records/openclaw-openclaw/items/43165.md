## Summary

Keep open. Current main still does not implement the requested Control UI channel icons, and the PR is directionally useful, but the proposed patch has a functional blocker in dm-scoped session key parsing.

## What This Changes

The PR adds emoji channel icons and peer/handle metadata to Sessions rows, wraps bundled and generic Channels card titles with icon+label rendering, extends the UI session row type with `channel`, adds CSS wrapping, and adds Sessions view tests.

## Best Possible Solution

Implement the UI feature through the existing channel catalog metadata where feasible, carry `channel` through the browser session row and live session-event fields, fix direct/dm peer extraction, and keep focused Sessions and Channels view tests around the rendering contract.

## Reproduction Assessment

Yes. By code inspection of the PR diff, `extractDirectPeerFromSessionKey` returns empty for `agent:main:whatsapp:default:dm:+15551234567` because it only searches for `direct`; the PR's added dm-scoped test exercises that path.

## Solution Assessment

No, not as-is. The feature direction is narrow and maintainable, but the patch should fix dm parsing and avoid duplicating hardcoded channel icon maps when current gateway metadata already exposes channel UI icon data.
