## What This Changes

Adds channel icons/labels to Control UI session metadata rows and channel-card titles, with session channel/identifier inference, CSS wrapping, a UI type addition, and rendering tests.

## Best Possible Solution

Keep the feature open or replace it with a rebased implementation that reuses one channel icon/label helper, handles `direct` and `dm` session-key delimiters, falls back for supported plugin channels, updates focused tests, and includes real Control UI proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection: PR head only extracts peers after `direct`, while its own dm-scoped WhatsApp test expects the phone number, and unlisted supported channels return `nothing`. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The feature direction is useful, but the branch needs the dm delimiter fix, a supported-channel fallback or shared helper, a clean rebase, and real Control UI proof before merge.
