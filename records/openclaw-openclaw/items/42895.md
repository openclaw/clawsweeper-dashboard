## Summary

Keep open. Current main still lacks the Termux `PREFIX` home fallback, and this PR is a plausible focused fix, but it should not merge until there is real Termux behavior proof and fresh post-rebase checks.

## What This Changes

The PR adds a Termux `PREFIX`-derived home-directory fallback, regression tests, environment docs, and a changelog entry for OpenClaw home path resolution.

## Best Possible Solution

Land a rebased version of the focused home-dir fallback with its regression tests, docs, and changelog after real Termux startup proof confirms the reported setup/config flow no longer writes to `/home`.

## Reproduction Assessment

Yes, at source level. Current main falls through to `os.homedir()` when `HOME` and `USERPROFILE` are unset, libuv then reads passwd data, and the linked Termux report shows that path becoming `/home` and failing during workspace creation.

## Solution Assessment

Yes, with merge gates. Putting the fallback in `src/infra/home-dir.ts` is the narrow maintainable fix, and the tests cover precedence and tilde expansion, but the PR still needs real Termux proof and fresh checks before landing.
