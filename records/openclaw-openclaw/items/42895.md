## Summary

Keep open: current main still falls back from `HOME`/`USERPROFILE` directly to `os.homedir()`, while the linked Termux crash report has concrete logs and this PR adds focused code, tests, docs, and changelog coverage. I found no blocking code-review finding in the patch, but it is not merge-ready until real Termux behavior proof is added.

## What This Changes

This PR adds an Android/Termux home-directory fallback derived from a trusted Termux `PREFIX`, shares it with tilde expansion, and documents/tests the new precedence.

## Best Possible Solution

Land a refreshed version of this focused fallback after real Termux startup proof is added, preserving `OPENCLAW_HOME`, `HOME`, and `USERPROFILE` precedence.

## Reproduction Assessment

Yes, from source and linked logs: with `HOME`/`USERPROFILE` unset on Termux and `os.homedir()` returning `/home`, current main falls through to `/home` before workspace setup creates directories. I did not run a real Termux device, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes for the code direction: adding the fallback in the shared home-dir resolver is the narrow maintainable fix and avoids touching individual workspace callers. The implementation still needs real Termux behavior proof before merge.
