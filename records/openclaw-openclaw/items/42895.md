## What This Changes

The PR adds a guarded Android/Termux `PREFIX`-derived home fallback to the core home resolver, adds regression tests, and updates the environment docs and changelog.

## Best Possible Solution

Land this focused patch, or a maintainer-equivalent patch, in the core home resolver with the same precedence guarantees: `OPENCLAW_HOME`, `HOME`, `USERPROFILE`, guarded Termux `PREFIX`, then `os.homedir()`. Keep the regression tests and docs/changelog update, and explicitly scope any memory-host SDK alignment.

## Reproduction Assessment

Yes. The linked bug has concrete Termux crash logs, and current main gives a unit-level reproduction: unset `HOME` and `USERPROFILE`, set Termux-like `PREFIX` and `ANDROID_DATA`, make `homedir()` return `/home`, and `resolveEffectiveHomeDir()` still resolves to `/home`.

## Solution Assessment

Yes for the core OpenClaw startup/config/workspace bug. The PR touches the implicated resolver, preserves env precedence, covers the tilde-expansion edge case raised in review, and rejects generic Android chroot prefixes; the safer remaining step is validation, not a different design.
