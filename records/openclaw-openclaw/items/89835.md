## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not default absent OpenAI auth mode to OAuth:** `src/infra/provider-usage.limits.ts:27-31`
  - body: Current embedded and CLI runners only set `requestShaping.authMode` when an auth profile is active, so API-key OpenAI turns can arrive here with `undefined`. Mapping that absent value to `oauth` lets a configured footer reuse cached ChatGPT/OAuth limit windows for an API-key turn; restore the no-default guard or resolve the actual auth mode before calling the cached helper.
  - confidence: 0.91
- **[P2] Move template file probing off the reply hot path:** `src/auto-reply/usage-bar/template.ts:47-57`
  - body: For a path-backed `messages.usageTemplate`, every `/usage full` reply performs `statSync`, and mtime changes do a synchronous read/parse before delivery. A slow home directory, network mount, or blocked path can stall the Node event loop and delay unrelated replies; make template reload lifecycle-owned or async/bounded outside the visible reply path.
  - confidence: 0.88
- **[P2] Document the new usage template config contract:** `src/config/zod-schema.session.ts:160`
  - body: This adds a user-facing `messages.usageTemplate` key and the `openclaw.usageBar.v1` format, but the public docs/config help still only describe the built-in `/usage full` behavior. Add the config help/docs and the hook optionality/format contract so users and plugin authors know how to use and upgrade this surface.
  - confidence: 0.86
- **[P2] Populate the new usage atoms in the runtime snapshot:** `src/auto-reply/reply/agent-runner.ts:1730`
  - body: The renderer contract now consumes `contextUsedTokens` and `lastUsage`, but this branch's `replyUsageSnapshot` does not set either field. In the current head, `context.pct_used` falls back to aggregate prompt tokens and `usage.last.*` is absent, so the advertised real context occupancy and final-call atoms do not work until the branch is refreshed with that population.
  - confidence: 0.82
