## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Use the active credential for quota windows:** `src/auto-reply/reply/agent-runner.ts:1784-1786`
  - body: `getProviderUsageLimitsCached` only receives the provider and auth mechanism, then the helper loads/cache-resolves usage by provider. OpenClaw supports separate agents and per-session `@<profileId>` routing for multiple accounts, so this can attach another profile/account's quota windows to the current reply footer; thread the resolved profile/account/agent auth into this lookup or omit `limits` when it cannot be tied to the active credential.
  - confidence: 0.86
- **[P2] Remove template file polling from the reply path:** `src/auto-reply/usage-bar/template.ts:47`
  - body: When `messages.usageTemplate` is a path, every `/usage full` render calls `statSync` before deciding whether to read. That puts synchronous filesystem freshness polling on the hot reply path for the new feature; load this through a lifecycle/config-reload-owned cache instead of checking the filesystem per reply.
  - confidence: 0.82
- **[P2] Document the new messages config key:** `src/config/zod-schema.session.ts:160`
  - body: This adds `messages.usageTemplate` to the strict runtime schema, but the PR does not add the matching public docs, schema help/label metadata, or generated config-baseline updates. Users and config tooling will not discover the new key or its template contract consistently.
  - confidence: 0.8
