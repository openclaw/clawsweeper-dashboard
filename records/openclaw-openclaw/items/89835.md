## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not treat missing auth mode as OAuth usage:** `src/infra/provider-usage.limits.ts:31`
  - body: The new mapper turns `undefined` into `oauth`, but current embedded/CLI metadata only sets `requestShaping.authMode` for auth-profile runs. An OpenAI API-key turn with absent auth metadata can therefore resolve the `openai` usage provider and return cached OAuth/ChatGPT limit windows for the wrong credential, which is the exact class this gate needs to prevent.
  - confidence: 0.92
- **[P2] Move template file checks out of reply delivery:** `src/auto-reply/usage-bar/template.ts:47-57`
  - body: `/usage full` calls `loadUsageBarTemplate` from the visible reply path, and that helper synchronously `statSync`s the configured path on each check and `readFileSync`s on mtime changes. A slow or stuck path can block channel replies, so template loading/freshness needs to be lifecycle-owned or otherwise kept off the delivery hot path.
  - confidence: 0.87
- **[P2] Document the new usage template and hook contract:** `src/config/zod-schema.session.ts:160`
  - body: This line adds a public `messages.usageTemplate` config field, and the PR also adds `reply_payload_sending.usageState`, but the user/plugin docs and config help/labels do not describe the template schema, fallback behavior, or plugin payload. Without those updates, users and plugin authors have no stable contract for the new DSL and hook data.
  - confidence: 0.83
