Keep open: the PR targets a real tool-schema bug that current main and the latest release still only partially cover, but the proposed guard is in the wrapper instead of the shared schema normalizer and the PR lacks real strict-provider proof.

## What This Changes

The PR adds wrapper-level stripping of non-array `required`, a nested Gemini schema-cleaning guard, and unit tests for malformed tool-schema `required` fields.

## Best Possible Solution

Canonicalize own non-array `required` fields in the shared recursive schema-normalization path, preserve valid arrays, cover wrapper/direct-helper/Gemini routes, and prove the strict-provider behavior with redacted real output.

## Reproduction Assessment

Yes, source-reproducible. Current main still preserves `required: null` for typed object schemas on the default shared-normalizer path, and nested Gemini non-array `required` values still fall through the cleaner.

## Solution Assessment

No. The intended repair is narrow, but the current branch should move the stripping into `normalizeToolParameterSchema` so direct callers and provider wrapper paths are covered consistently.
