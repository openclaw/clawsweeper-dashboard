## Summary

Keep open. The bug is still source-reproducible on current main, but this PR should not merge as-is because the proposed stripping only covers the wrapper path and misses direct/shared schema-normalization callers; it also still lacks real strict-provider proof.

## What This Changes

The PR strips non-array `required` values from agent tool schemas in `normalizeToolParameters`, drops them in the Gemini cleaner, and adds schema-normalization tests.

## Best Possible Solution

Canonicalize own non-array `required` fields in the shared recursive schema-normalization path, preserve valid arrays, cover wrapper/direct-helper/Gemini routes, and prove the strict-provider behavior with redacted real output.

## Reproduction Assessment

Yes, source-reproducible. Current main can still preserve `required: null` through the shared default normalization path, and direct OpenAI helper routes bypass the PR's wrapper-only guard.

## Solution Assessment

No. The requested fix is narrow and valid, but the cleanup should live in `normalizeToolParameterSchema` or its recursive schema-cleaning helpers so wrapper callers and direct callers share the same behavior.
