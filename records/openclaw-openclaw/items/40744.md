## Summary

Keep open: the bug is still source-reproducible, and the PR is directionally useful but not merge-ready because it strips malformed `required` only in the wrapper path while shared/direct schema-normalization routes can still preserve it.

## What This Changes

The PR strips non-array `required` values from agent tool schemas in `normalizeToolParameters`, adds a Gemini cleaner guard, and adds schema-normalization tests.

## Best Possible Solution

Canonicalize own non-array `required` fields in the shared recursive schema-normalization path, preserve valid arrays, cover wrapper/direct-helper/Gemini routes, and prove the strict-provider behavior with redacted real output.

## Reproduction Assessment

Yes, source-reproducible. Current main can still preserve `required: null` through the shared default normalization path, and nested Gemini non-array `required` values are copied rather than stripped.

## Solution Assessment

No. The intent is narrow and valid, but the cleanup should live in `normalizeToolParameterSchema` or its recursive schema-cleaning helpers so direct callers and wrapper callers share the same behavior.
