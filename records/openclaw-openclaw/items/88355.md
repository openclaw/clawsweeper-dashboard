## Summary

Keep open: current main still has the reported Codex app-server context-engine retry invariant problem, with a source-level regression fixture asserting that the retry sends only the bare prompt while persisting the old thread-bootstrap projection on the fresh thread.

## What This Changes

The issue asks OpenClaw to stop marking a fresh Codex retry thread as having a context-engine thread-bootstrap projection unless that retry actually received equivalent projected context.

## Best Possible Solution

Keep the issue open and fix the Codex app-server retry so a fresh context-engine overflow retry either receives bounded projected context or persists no matching thread-bootstrap projection for that fresh thread.

## Reproduction Assessment

Yes. The current source test is a focused harness reproduction: it triggers turn/start overflow on a resumed thread, retries on thread-fresh with input "hello", and asserts the saved projection epoch is still retained; I did not execute tests because this was a read-only review.

## Solution Assessment

Yes, a narrow invariant fix is the right direction: do not retain a thread_bootstrap projection binding unless the fresh thread actually received that projection. A broader overflow-safe reproject path can be considered later, but clearing or staling the projection is the minimum maintainable repair.

## Repair Work Prompt

Fix the Codex app-server context-engine overflow retry invariant reported in https://github.com/openclaw/openclaw/issues/88355. On current main, a resumed thread_bootstrap context-engine binding can overflow during turn/start; run-attempt clears the old binding, starts a fresh Codex thread, sends only the bare prompt, and then persists the old projection epoch on that fresh thread. Make the retry path stop saving or retaining a thread_bootstrap projection binding unless the fresh retry actually received the projected context; acceptable shapes include staling/omitting the projection so the next run reprojects, or failing loudly if continuity cannot be preserved. Preserve the newer-binding guard and the existing behavior that plugin-owned compact is not called in this overflow retry. Add or update focused regression coverage in run-attempt.context-engine.test.ts so the retry input without assembled context no longer leaves savedBinding.contextEngine.projection as the old epoch. Do not add config, plugin SDK surface, or CHANGELOG changes.
