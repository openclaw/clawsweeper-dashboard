## Summary

Keep this open. Current main has fixed the active-run history reload and active-memory timeout parts of the report, but the remaining Control UI source gaps are still present: focused/IME composer draft preservation and visible handling for terminal chat error events with partial streamed text.

## What This Changes

The issue asks OpenClaw to fix a chat UI regression where typed input, IME drafts, live streamed replies, and typing/error states can disappear or only appear after refresh.

## Best Possible Solution

Land a focused Control UI repair that preserves the live textarea value while focused or composing, commits IME text on composition end/blur, and turns terminal chat errors into visible output by retaining non-silent partial stream text or showing a clear generation-failed assistant message. Keep the already-fixed active-memory and deferred history reload behavior unchanged, and track the broader accepted/processing phase model separately if maintainers want that UX expansion.

## Reproduction Assessment

Yes for the remaining source-level issue: the thread gives concrete local patch anchors, and current main still lacks composition/focus handlers while the error branch still clears partial stream state. A full live Windows browser reproduction was not run.

## Solution Assessment

Yes for the evidence-backed subset: the narrowest maintainable fix is in the Control UI chat view and chat controller tests. Provider routing, Gateway protocol, and Active Memory timeout logic do not need to change for the remaining gaps.
