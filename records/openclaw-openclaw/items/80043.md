## Summary

Keep this PR open, but the patch is not ready: it changes a debounce branch that production WhatsApp media normally bypasses, and it can mishandle dedupe if split delivery partially fails. The PR also still lacks after-fix proof from a real WhatsApp run.

## What This Changes

The PR adds a WhatsApp inbound flush branch that dispatches each debounced entry separately when any entry has a media path.

## Best Possible Solution

Preserve every WhatsApp forwarded image in the actual runtime path, either by carrying media attachments through active-run steering or by adding WhatsApp media coalescing with per-delivered-item dedupe semantics.

## Reproduction Assessment

Yes at source level, but not live: current main saves WhatsApp media, bypasses text debounce for media, and active-run steering queues only prompt text. The linked issue provides logs for the lost second image, but this PR still needs a redacted real WhatsApp after-fix run.

## Solution Assessment

No. The proposed fix is not the best path because it only handles multi-entry debounce batches and introduces partial split-delivery dedupe risk; the durable fix should address the production media path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

  - body: The new hasMedia branch only runs after a multi-entry debounce batch exists, but production WhatsApp returns false from shouldDebounce for messages with mediaPath or mediaType. Rapid forwarded images still flush as single-entry media messages and can still hit active-run steering without attachment context.
  - confidence: 0.93
- **[P2] Finalize split media entries per delivered item:** `extensions/whatsapp/src/inbound/monitor.ts:279-281`
  - body: This loop can deliver an earlier entry and then throw on a later one while the outer catch finalizes the original batch. Retryable failures can release already-delivered keys for duplicate retry, and non-retryable failures can commit keys for entries that never reached onMessage.
  - confidence: 0.91
