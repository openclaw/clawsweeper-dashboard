## Summary

Keep open: the PR targets a plausible media-batch loss shape, but current WhatsApp production media bypasses that debounce batch path, the new split loop can finalize dedupe keys incorrectly after partial delivery, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a WhatsApp inbound debounce branch that sends each media-bearing debounced entry separately instead of merging the batch into one message.

## Best Possible Solution

Implement a WhatsApp-owned media burst path that preserves all rapid forwarded images as MediaPaths/MediaTypes, or explicitly extend active-run steering to carry media, while keeping text-only debounce behavior unchanged.

## Reproduction Assessment

Yes, at source level: current main routes WhatsApp media around debounce and active-run steering accepts text only, matching the linked rapid-image loss report. I did not run a live WhatsApp forwarded-image burst.

## Solution Assessment

No. The current PR does not cover the production media path and introduces partial-delivery dedupe corruption; a media-aware WhatsApp burst path or explicit media-capable steering design is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Cover the production media path:** `extensions/whatsapp/src/inbound/monitor.ts:276`
  - body: This branch only runs for multi-entry debounce batches, but the production WhatsApp monitor returns false from shouldDebounce for any mediaPath or mediaType. Rapid image forwards therefore still flush as single media entries and can still hit text-only steering instead of this new branch.
  - confidence: 0.93
- **[P2] Finalize split media entries individually:** `extensions/whatsapp/src/inbound/monitor.ts:278-280`
  - body: If one onMessage(entry) succeeds and a later split entry throws, the outer catch finalizes the whole entries array. Retryable failures can redeliver already-sent images, while non-retryable failures can commit keys for images that were never sent.
  - confidence: 0.91
