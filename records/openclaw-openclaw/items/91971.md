## Summary

This should stay open: current main advertises `@openclaw/llama-cpp-provider` as the official installable provider, but live npm and ClawHub lookups still return not found, matching the reporter's concrete command output and follow-up clarification.

## What This Changes

The issue reports that the documented `@openclaw/llama-cpp-provider` plugin cannot be installed from npm or ClawHub in OpenClaw 2026.6.5.

## Best Possible Solution

Publish and index the official provider package on npm and ClawHub, or update docs, doctor hints, and catalog metadata so users are not directed to an unavailable install route.

## Reproduction Assessment

Yes. The reporter's exact install failures are reproducible by live npm and ClawHub package lookups that both return 404 for `@openclaw/llama-cpp-provider`, and current installer code maps those 404s to the same errors.

## Solution Assessment

Unclear as an implementation path. The source intent points toward publishing the official plugin, but maintainers should decide whether to publish/index it now or temporarily remove the install guidance until publication is ready.
