## Summary

Keep open: current main still records only machine-oriented `sessions_send` provenance and Control UI can show a human-readable sender only when a top-level `senderLabel` already exists. The related open trackers cover adjacent model-facing identity and broader multi-role UI work, but this focused session-message attribution gap remains useful on its own.

## What This Changes

Request to add human-readable sender and target attribution to `sessions_send` messages in session history/UI so operators can see who sent what.

## Best Possible Solution

Add structured `sessions_send` source/target attribution that preserves existing provenance, resolves display-safe labels at the session/history boundary, and feeds the existing `senderLabel` UI path while leaving broad filtering and avatar roles to the larger multi-role tracker.
