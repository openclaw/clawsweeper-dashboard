## What This Changes

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79818, or a narrower Slack-plugin successor, with explicit Canvas scope guidance, action/config gating, bounded markdown or section output, tests, and live Slack proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high-confidence evidence of the current-main gap because no Slack Canvas action, Canvas API call path, canvases:read setup/docs scope, or Canvas read test exists.

Yes, the maintainable direction is Slack-plugin-owned Canvas read support with explicit scope/token gating and bounded output. A generic download-file workaround is not sufficient because the current file path only handles Slack file metadata/private URLs, not Canvas sections or markdown.
