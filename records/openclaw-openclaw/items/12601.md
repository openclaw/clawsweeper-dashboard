## Summary

Keep open. Current main still has no Slack Canvas read surface in the bundled Slack plugin, and the only directly related implementation PR was closed unmerged.

## What This Changes

Request support for reading Slack Canvas markdown/content through the bundled Slack message tooling or a Slack-owned Canvas tool.

## Best Possible Solution

Implement this as Slack plugin work: add a bounded Canvas read action or Slack-owned tool, wire discovery/dispatch/runtime/setup/docs/tests, and keep any core change limited to an additive channel action contract if needed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to request or discover a Slack Canvas read action on current main: discovery does not advertise it, dispatch/runtime do not handle it, setup/docs omit `canvases:read`, and `download-file` only follows private file URLs.

## Solution Assessment

Yes. The narrow maintainable direction is to implement Canvas reading inside `extensions/slack`, reusing the existing account/token/action patterns and avoiding Slack-specific core behavior except for a minimal additive action-name contract if the message tool path is used.
