  - body: This new event is advertised as per-run observability, but the payload only includes `sessionKey`; multiple runs can share a session, and existing Gateway agent events use `runId` as the correlation key.   - confidence: 0.87

  - body: Deriving the status `agentId` from `sessionKey` reports `main` for selected-agent global sessions, while the surrounding lifecycle projection uses the chat run link or event agent id. That would send observers a status for the wrong agent whenever a non-main agent runs on the global session.
  - confidence: 0.88
