## Summary

## What This Changes

## Best Possible Solution

If maintainers want this event, define it as an intentional public Gateway/SDK contract with correct selected-agent identity, run correlation, docs, tests, and real client proof; otherwise keep run observability on the existing `agent` lifecycle plus `agent.wait` path.

## Solution Assessment

No: this is only a plausible event addition, not the best settled contract. The safer solution is either to extend the existing lifecycle/SDK/`agent.wait` contract or explicitly define the new event with correct identity, correlation, docs, tests, and proof.
