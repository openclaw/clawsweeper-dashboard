## Summary

Keep open. Current main does not implement `agent.run.status`, and the PR is useful but still has a correctness gap: the promised completion metadata is attached only to the fallback lifecycle event that normal embedded runs skip. The diff also lacks required changelog coverage for a user-facing gateway feature.

## What This Changes

The PR adds an `agent.run.status` gateway broadcast event and attempts to emit start/completion status, timing, model, usage, tool-call count, and exit reason from agent lifecycle events.

## Best Possible Solution

Carry run metadata on the terminal lifecycle event path that normal embedded runs actually emit, keep the new gateway event scoped to operator read subscribers, and add focused tests plus a changelog entry for the public observable API surface.

## Reproduction Assessment

Yes. A high-confidence static path is enough: normal embedded runs emit lifecycle `end`/`error` from `handleAgentEnd`, `agentCommandInternal` sets `lifecycleEnded = true`, and the PR's enriched synthetic terminal event is skipped.

## Solution Assessment

No. The gateway broadcast direction is reasonable, but enriching only the fallback terminal lifecycle event is not the narrow maintainable fix because it misses the common terminal lifecycle path.
