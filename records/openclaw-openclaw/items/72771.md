## Summary

## What This Changes

The branch adds a public Gateway `agent.run.status` WebSocket event with started and terminal status payloads, including timing, model, usage, and tool-count data.

## Best Possible Solution

Define one canonical Gateway run-status/usage contract before merge, either by extending the existing lifecycle/SDK/`agent.wait` path or by deliberately adding `agent.run.status` with correct `sessionAgentId`, `runId`, docs, tests, and real client proof.

## Reproduction Assessment

Yes for the patch defects by source inspection: a `global` run linked to a non-main agent would be emitted as `main`, and the new status payload has no `runId`. I did not run live Gateway proof because this review is read-only.

## Solution Assessment

No: this is a plausible slice, but not the best settled contract as written. The safer path is to extend the existing lifecycle/SDK/`agent.wait` contract or explicitly define the new event with correct identity, correlation, tests, docs, and real proof.
