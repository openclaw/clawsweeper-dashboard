## What This Changes

The PR adds an advertised `agent.run.status` Gateway event and emits started/terminal run-status payloads from agent lifecycle handling, including model, usage, and tool-call counts.

## Best Possible Solution

If maintainers want this event, define it as an intentional extension of the existing lifecycle contract with correct `sessionAgentId`, `runId`, SDK/docs/tests, and real Gateway-client proof; otherwise keep run observability on the existing `agent` plus `agent.wait` path and leave durable status design in the related issues.
