## Summary

Keep open. Current main still does not implement the new `agent.run.status` gateway event, so the PR is not obsolete, but the current diff misses promised lifecycle data on the normal terminal path and needs targeted repairs before merge.

## What This Changes

The PR adds an `agent.run.status` gateway event type, read-scoped broadcast registration, discovery listing, and lifecycle-derived started/terminal status broadcasts with optional model, usage, tool count, and exit metadata.

## Best Possible Solution

Keep the event direction, but repair the implementation so start and terminal metadata is produced by the real lifecycle source or a shared lifecycle payload builder, terminal `aborted` state maps to `interrupted`, and focused tests plus a changelog entry cover the new observable API.

## Reproduction Assessment

Yes. Static path tracing is high-confidence: normal embedded runs emit lifecycle `end`/`error` from `handleAgentEnd`, `agentCommandInternal` marks `lifecycleEnded = true`, and the PR's enriched synthetic terminal event is skipped.

## Solution Assessment

No. The broadcast surface is reasonable, but enriching only the fallback synthetic lifecycle event is not the narrow maintainable fix; the metadata belongs on the real lifecycle events or in a shared builder consumed by both paths.
