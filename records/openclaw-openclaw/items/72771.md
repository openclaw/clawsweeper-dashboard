Full review comments:

- **[P2] Include a run id in the status payload:** `src/gateway/events.ts:13`
  - body: This new event is described as per-run observability, but the payload has only `sessionKey` plus status metadata. Existing Gateway agent events are keyed by `runId`; without it, clients cannot reliably match `started` and terminal events or distinguish concurrent/subagent runs in the same session.
  - confidence: 0.88
- **[P2] Use the resolved run agent for status events:** `src/gateway/server-chat.ts:656`
  - body: For selected-agent `global` sessions and chat-linked runs, the surrounding handler already carries `sessionAgentId` from `chatLink?.agentId ?? evt.agentId`. Re-deriving it from `sessionKey` returns the default agent for `global`, so the new status event can report non-default agent runs as `main`.
  - confidence: 0.9
