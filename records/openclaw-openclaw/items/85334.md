## Summary

Keep open: the report describes a real doctor/update idempotency conflict, and current main still has source paths that both remove bundled plugin load paths during doctor repair and intentionally re-add bundled paths during plugin update sync. The open related pull request at https://github.com/openclaw/openclaw/pull/85038 addresses stale old-version package roots for https://github.com/openclaw/openclaw/issues/58626, but it does not clearly cover the current-bundled-path re-addition loop reported here.

## What This Changes

The issue asks for `openclaw doctor --fix` to stop writing or reintroducing `plugins.load.paths` entries that point at OpenClaw's bundled plugin directories.

## Best Possible Solution

Make the doctor/update path idempotent so bundled plugin directories are never persisted in `plugins.load.paths`, while preserving custom plugin paths and the stale old-version package-root repair covered by the related open PR.

## Reproduction Assessment

No live high-confidence reproduction was run because this review is read-only. Source inspection does show the conflicting paths: doctor removes bundled load paths while update sync can write the same bundled path back for path install records.

## Solution Assessment

Unclear as an implementation choice. The likely fix is narrow, but maintainers should decide whether to expand the related open PR or change `syncPluginsForUpdateChannel`, because current tests intentionally preserve this bundled path-install behavior.
