## Summary

Keep open: current `main` and the latest release still contain writers that persist bundled plugin roots into `plugins.load.paths`, while discovery and doctor treat those same paths as redundant. The linked closing PR is still open, so this issue should remain the canonical tracker until that fix merges or is replaced.

## What This Changes

The issue asks OpenClaw to stop `openclaw update`/doctor repair paths from writing bundled plugin directories into `plugins.load.paths` and causing a warning/removal/re-add loop.

## Best Possible Solution

Land or replace the linked fix PR with a focused repair that keeps bundled plugin roots out of `plugins.load.paths` across update sync, onboarding repair, and final doctor cleanup while preserving bundled install records.

## Reproduction Assessment

Yes at source level: current `main` still writes bundled plugin roots during update sync while discovery warns on and doctor repair removes that same path class. I did not run mutating `openclaw update` or `openclaw doctor --fix` commands in this read-only review.

## Solution Assessment

Yes: the maintainable fix is to stop all known bundled-path writers from persisting bundled roots and leave doctor cleanup as the final canonical repair. The issue should stay open until the linked PR or an equivalent replacement lands.
