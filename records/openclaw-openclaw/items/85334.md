## Summary

Keep open: current main still re-adds current bundled plugin paths during plugin channel sync while doctor/discovery remove or warn on those same paths. The issue is also covered by two open closing PRs, so it should stay open until one fix lands: https://github.com/openclaw/openclaw/pull/85358 and https://github.com/openclaw/openclaw/pull/85395.

## What This Changes

The issue asks OpenClaw update/doctor repair to stop persisting current bundled plugin directories in `plugins.load.paths` and to remove redundant entries idempotently.

## Best Possible Solution

Land one narrow fix that makes update sync, post-core convergence, and doctor repair agree that current bundled plugin directories are auto-discovered and should not be persisted in `plugins.load.paths`, while preserving genuine external path plugins and coordinating stale-path cleanup with https://github.com/openclaw/openclaw/pull/85038.

## Reproduction Assessment

Yes, at source level. Current main's plugin channel sync can still add `bundledInfo.localPath` to `plugins.load.paths`, while doctor repair and discovery remove or warn on that same bundled path class.

## Solution Assessment

Yes on direction: the maintainable solution is to stop persisting current bundled plugin directories while still keeping install metadata and external path plugins intact. The exact linked PR should be reviewed before landing because there are overlapping repair proposals.
