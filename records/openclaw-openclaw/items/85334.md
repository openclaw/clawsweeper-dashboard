## Summary

Keep open: current main still has contradictory bundled plugin path behavior, with update/plugin sync expecting a current bundled path in `plugins.load.paths` while doctor repair and discovery remove or warn on that same path.

## What This Changes

The issue reports that `openclaw doctor --fix` writes a redundant bundled `memory-wiki` plugin path into `plugins.load.paths`, causing later doctor runs to warn and remove it repeatedly.

## Best Possible Solution

Make bundled plugin install metadata and doctor repair agree on one canonical state: current bundled plugins should not persist their runtime directory in `plugins.load.paths`, while genuine external path plugins and stale old-version path cleanup keep working.

## Reproduction Assessment

Yes, at source level: current main has code and tests that re-add current bundled plugin roots to `plugins.load.paths`, plus doctor/discovery code and tests that remove or warn on those same roots. I did not run the mutating command path in this read-only review.

## Solution Assessment

No current-main solution is present. The maintainable fix should be a narrow consistency change around bundled path install syncing and doctor repair, ideally coordinated with the adjacent open doctor-pruning PR rather than duplicating only stale-path cleanup.
