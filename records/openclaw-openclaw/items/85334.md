## What This Changes

The issue asks for update/doctor repair to stop writing current bundled plugin directories into `plugins.load.paths` and to clean existing redundant entries idempotently.

## Best Possible Solution

Make bundled plugin install metadata, update sync, onboarding install helpers, and doctor repair agree that current bundled plugin directories are auto-discovered and must not be persisted in `plugins.load.paths`, while preserving genuine external path plugins and stale-path cleanup.

## Reproduction Assessment

Yes, at source level: current main still re-adds bundled plugin roots in update sync, while doctor/discovery remove or warn on the same paths. I did not run the mutating CLI path in this read-only review.

## Solution Assessment

Yes, the requested direction is the right fix boundary: current bundled plugin directories should stay out of `plugins.load.paths`, and repair sequencing should make doctor idempotent. Current main has not landed that solution yet.
