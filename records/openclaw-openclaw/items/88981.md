Full review comments:

- **[P1] Read the reaction emoji from the Talk content field:** `extensions/nextcloud-talk/src/monitor.ts:230`
  - body: Nextcloud Talk's bot docs put the added emoji in the top-level `content` field, and for Undo they put the removed emoji in the nested Like object's `content`; `object.content` is the original message body. With this decoder, a real reaction to an approval prompt forwards the prompt message JSON/text as `emoji`, so `resolveReactionDecision` never matches `1️⃣`/`2️⃣`/`3️⃣` and the approval is not resolved. ([nextcloud-talk.readthedocs.io](https://nextcloud-talk.readthedocs.io/en/latest/bots/))
  - confidence: 0.95
- **[P2] Persist approval bindings outside process memory:** `extensions/nextcloud-talk/src/approval-stash.ts:39`
  - body: The approval prompt binding is stored only in a module-level `Map`, so a gateway restart after the prompt is sent loses the message-to-approval binding while the approval is still visible in Talk. Use the shared approval reaction target store or plugin state with the approval expiry, as the Signal implementation does, so pending reaction approvals can survive process restarts and stale bindings expire with the request.
  - confidence: 0.86
