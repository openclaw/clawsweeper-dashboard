## Summary

Keep open. Current main still has real paths where REM phase reports and phase signals are written while `DREAMS.md` stays absent, and the Control UI still renders an empty diary when that file is missing.

## What This Changes

The issue requests that successful REM dreaming runs create or update `DREAMS.md` so the Control UI Dreams diary is populated when REM reports exist.

## Best Possible Solution

Define one Dream Diary contract across memory-core, Gateway, Control UI, and docs, then make either the writer or the UI honor it for material REM output.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to run REM with eligible entries while the narrative subagent times out, returns a non-ok status, or produces no assistant text; current tests still assert the diary file remains absent on the timeout path.

## Solution Assessment

Unclear. The current retries and concurrency cap are useful mitigations, but they do not settle whether missing `DREAMS.md` should be repaired by fallback diary writes or by showing phase reports in the UI.
