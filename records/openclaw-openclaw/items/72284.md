## Summary

Keep open. Current main still has source-visible paths where a REM phase can write phase artifacts and signals while `DREAMS.md` remains absent, and the Control UI only renders Dream Diary content returned by the diary RPC.

## What This Changes

After REM creates phase artifacts, create or update `DREAMS.md` so the Control UI Dreams tab can display diary content.

## Best Possible Solution

Define one Dream Diary contract across memory-core, Gateway, Control UI, and docs: either every material REM run appends a safe fallback diary entry, or the Dreams UI displays phase reports when no diary exists.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: run REM with eligible entries while narrative generation times out, returns a non-ok status, or yields no assistant text; current tests still assert `DREAMS.md` remains absent on the timeout path.

## Solution Assessment

Unclear. The current request-scoped fallback, model retry, and 60-second timeout improvements reduce some causes, but they do not establish the missing-diary/empty-UI contract for all successful REM phase runs.
