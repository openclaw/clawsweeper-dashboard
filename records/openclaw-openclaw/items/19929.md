## Summary

Keep open. Current main and v2026.5.19 still isolate non-DM channel/group/thread sessions and expose no selected channel/thread grouping config; the closed implementation PR was not merged, and related issues are design inputs rather than replacements.

## What This Changes

Request an opt-in config surface that lets selected Discord/Slack group, channel, and thread chats share one agent session while preserving isolated defaults and origin-channel replies.

## Best Possible Solution

Design an additive shared-session routing contract with isolated defaults, explicit channel/thread membership, origin-chat reply routing, grouped queue serialization, security documentation, schema/help updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection against current main and v2026.5.19 gives high confidence that the requested grouping fields and non-DM grouping route are absent.

## Solution Assessment

Unclear as a concrete implementation because config shape, thread inheritance, queueing, reply routing, and trust-boundary semantics need maintainer design. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
