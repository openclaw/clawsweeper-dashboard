## Summary

Keep open. Current main still has no selective shared-session surface for chosen group/channel/thread chats, and the request is a feature/config design rather than a broken existing contract; the prior implementation PR is closed unmerged while related items remain design inputs.

## What This Changes

Request an opt-in way to map selected Discord/Slack channel and thread chats to one shared session while leaving unlisted chats isolated.

## Best Possible Solution

Add an opt-in, documented session grouping design that preserves isolated defaults while unifying selected channel/thread peers into one session key with correct reply routing, queue serialization, schema/types/help/docs, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection does confirm the current-main baseline: non-DM channel peers remain isolated and the strict session schema has no requested grouping config.

## Solution Assessment

Unclear until maintainers settle the product and security design. The channelGroups direction is plausible, but current route bindings and ACP/thread bindings are not a complete substitute for the requested selective shared-session surface.
