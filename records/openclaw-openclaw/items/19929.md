## Summary

Keep open. Current main still lacks an opt-in shared-session surface for selected channel/thread groups, the earlier channelGroups implementation was closed unmerged, and the active related work is narrower than this issue's full Discord/Slack channel/thread design.

## What This Changes

The issue requests an opt-in config surface that lets selected Discord/Slack channels or threads share one agent session while preserving isolated defaults and reply routing to the originating chat.

## Best Possible Solution

Design an additive opt-in core routing contract with explicit channel/thread membership, isolated defaults, origin-chat reply routing, grouped queue serialization, schema/help/docs updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main and v2026.5.12 gives high confidence that the requested grouping config and route path are absent.

## Solution Assessment

Unclear as a concrete implementation because the config shape and trust-boundary semantics need maintainer design. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
