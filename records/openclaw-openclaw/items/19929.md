## Summary

Keep open. Current main and the latest release still isolate non-DM channel/group/thread sessions and expose no selected channel/thread grouping config; the related open issues and closed unmerged PR are useful design inputs rather than a superseding fix.

## What This Changes

Requests a new opt-in config surface to map selected Discord/Slack channels and threads to shared session keys while keeping other chats isolated.

## Best Possible Solution

Design an additive shared-session routing contract with isolated defaults, explicit channel/thread membership, origin-chat reply routing, grouped queue serialization, security documentation, schema/help updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection against current main gives high confidence that the requested grouping fields and non-DM grouping route are absent.

## Solution Assessment

Unclear as a concrete implementation because config shape, thread inheritance, queueing, and trust-boundary semantics need maintainer design. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
