## Summary

Keep open. Current main still lacks an opt-in shared-session surface for selected channel/thread groups, and the live discussion treats this as the canonical tracker rather than a duplicate or stale cleanup item.

## What This Changes

Request opt-in configuration that maps selected Discord/Slack channels and threads to one shared session key while preserving isolated defaults and origin-channel replies.

## Best Possible Solution

Design an additive opt-in core routing contract with explicit channel/thread membership, isolated defaults, origin-channel replies, grouped queue serialization, schema/help/docs updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that the requested session grouping config and route path are absent.

## Solution Assessment

Unclear as a concrete implementation because the final API shape and trust-boundary policy need maintainer design. The maintainable direction is explicit opt-in grouping that preserves current isolation unless configured.
