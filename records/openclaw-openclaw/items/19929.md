## Summary

Keep open. Current main still lacks an opt-in shared-session surface for selected channel/thread groups, the only direct implementation PR was closed unmerged, and recent discussion treats this as the canonical product/security design tracker rather than a duplicate.

## What This Changes

Request an opt-in configuration to share one agent session across selected Discord/Slack channels and threads while preserving isolated defaults and origin-channel replies.

## Best Possible Solution

Design an additive opt-in core routing contract with explicit channel/thread membership, isolated defaults, origin-chat reply routing, grouped queue serialization, schema/help/docs updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still lacks the requested grouping config and route path.

## Solution Assessment

Unclear as a concrete implementation because the exact config shape and trust-boundary semantics need maintainer design. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
