## Summary

Keep open. Current main and v2026.5.20 still isolate non-DM group/channel/thread sessions and expose no selected channel/thread grouping config; the related open issues and open PR are useful design or partial implementation inputs rather than replacements for this broader product/security-sensitive request.

## What This Changes

The issue requests an opt-in config surface for selected Discord, Slack, and thread chats to share one agent session while unlisted chats remain isolated.

## Best Possible Solution

Design an additive shared-session routing contract with isolated defaults, explicit channel/thread membership, optional thread inheritance, origin-chat reply routing, grouped queue serialization, security documentation, schema/help updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, tests, and v2026.5.20 inspection give high confidence that selected channel/thread grouping is absent and current non-DM sessions remain isolated.

## Solution Assessment

Unclear as a concrete implementation because maintainers still need to choose the config shape, thread inheritance semantics, queueing, reply routing, and sandbox/trust-boundary behavior. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
