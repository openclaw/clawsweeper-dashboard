## Summary

Keep open. Current main still lacks the requested selective channel/thread shared-session surface, and the discussion treats this as the broad canonical design tracker rather than a stale duplicate.

## What This Changes

Request an opt-in config that maps selected group/channel/thread chats to one shared session while keeping unlisted chats isolated and replies routed to the originating chat.

## Best Possible Solution

Design an additive core shared-session contract with isolated defaults, explicit channel/thread membership, deterministic origin-channel replies, grouped queue serialization, schema/help/docs updates, and Discord/Slack routing tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main still lacks the requested config/schema/routing surface and still routes non-DM peers to channel-specific session keys.

## Solution Assessment

Unclear as a concrete implementation because the final config shape and trust-boundary policy need maintainer design. The maintainable direction is an opt-in core routing contract that preserves current isolation unless explicitly configured.
