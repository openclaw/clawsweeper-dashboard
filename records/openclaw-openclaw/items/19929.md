## Summary

Keep open. Current main and the latest release still lack an opt-in way to map selected Discord/Slack channels or threads into one shared session; the related open issues are narrower design inputs, and the direct implementation PR was closed unmerged.

## What This Changes

The issue requests an opt-in way to group selected Discord/Slack channels or threads so they share one agent session while preserving reply routing to the originating chat.

## Best Possible Solution

Design an explicit opt-in shared-session routing contract with isolated defaults, selected channel/thread membership, origin-chat reply routing, grouped queue serialization, security documentation, schema/help updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Yes for source-level verification, not as a runtime bug reproduction: current source, docs, tests, and v2026.5.22 all show selected channel/thread grouping is absent. The observable behavior is the existing documented isolation of non-DM channel/thread sessions.

## Solution Assessment

Unclear as a concrete implementation because maintainers still need to choose the config shape, thread inheritance semantics, queueing, reply routing, and sandbox/trust-boundary behavior. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
