## Summary

Keep open. Current main still lacks any opt-in shared-session surface for selected channel/thread groups, the only direct implementation PR was closed unmerged, and recent issue discussion treats this as the canonical product/security design tracker rather than a duplicate.

## What This Changes

Requested support for opt-in shared sessions across selected Discord/Slack channels and threads while preserving isolated defaults.

## Best Possible Solution

Design an additive opt-in core routing contract with explicit channel/thread membership, isolated defaults, origin-chat reply routing, grouped queue serialization, schema/help/docs updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main gives high confidence that the requested grouping config and route path are absent.

## Solution Assessment

Unclear as a concrete implementation because the config shape and trust-boundary semantics need maintainer design. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
