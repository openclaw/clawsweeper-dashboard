## Summary

Keep open. Current main and v2026.5.20 still isolate non-DM group/channel/thread sessions and expose no selected shared-session grouping config; the related reports are narrower design inputs rather than a canonical replacement.

## What This Changes

Request an opt-in config surface that maps selected group/channel/thread chats to a shared session key while preserving isolated defaults and replying to the originating chat.

## Best Possible Solution

Design an additive opt-in shared-session routing contract with isolated defaults, explicit channel/thread membership, optional thread inheritance, origin-chat reply routing, grouped queue serialization, security documentation, schema/help updates, and Discord/Slack regression coverage.

## Solution Assessment

Unclear as a concrete implementation because maintainers still need to choose the config shape, thread inheritance semantics, queueing, reply routing, and sandbox/trust-boundary behavior. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
