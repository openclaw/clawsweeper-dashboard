## Summary

Keep open: current main and v2026.6.10 still isolate selected group/channel/thread chats, the strict config surface has no channelGroups-style option, and the live open PR covers only narrower group-to-main behavior rather than this broader selected shared-session request.

## What This Changes

The issue requests an opt-in way to map selected Discord/Slack channels or threads to one shared agent session while leaving unlisted chats isolated.

## Best Possible Solution

Keep this issue as the broader selective shared-session tracker until maintainers choose one opt-in contract covering selected channels/threads, queue serialization, origin-preserving replies, docs, tests, and security guidance.

## Reproduction Assessment

No bug reproduction is needed for this feature request, but source inspection gives a high-confidence reproduction of the missing capability: non-direct peers still derive isolated keys and strict config cannot express selected channel/thread groups.

## Solution Assessment

Unclear as proposed. session.channelGroups is plausible, but the best solution should reconcile the overlapping groupScope, named inbound session, and identity-routing proposals into one maintainable opt-in contract.
