## Summary

Keep open. Current main still lacks the requested selective shared-session surface for chosen channel/thread groups, and the live discussion treats this as the broad canonical tracker rather than a duplicate or stale cleanup item.

## What This Changes

Add an opt-in configuration surface that lets selected Discord/Slack channel and thread chats for the same agent share one session while preserving isolated defaults and origin-channel reply routing.

## Best Possible Solution

Design an additive opt-in core routing contract with explicit channel/thread membership, isolated defaults, origin-channel replies, grouped queue serialization, schema/help/docs updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that the requested config/schema/routing surface is absent and non-DM peers still route to isolated channel keys.

## Solution Assessment

Unclear as a concrete implementation because the final config shape and trust-boundary policy need maintainer design. The maintainable direction is an explicit opt-in core session-routing contract that preserves current isolation unless configured.
