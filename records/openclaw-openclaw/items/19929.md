## Summary

Keep open. Current main still lacks a selective channel/thread session grouping surface, the latest maintainer review treats this as the canonical tracker, and related issues plus the closed unmerged implementation PR are design inputs rather than replacements.

## What This Changes

Request an opt-in configuration that maps selected group, channel, and thread chats to a shared agent session while preserving isolated defaults and origin-channel replies.

## Best Possible Solution

Design an additive core session grouping contract with isolated defaults, explicit channel/thread membership, deterministic origin-channel replies, grouped queue serialization, schema/help/docs updates, and Discord/Slack routing regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested config/schema/routing surface and still resolves non-DM peers to channel-specific keys.

## Solution Assessment

Unclear as a concrete implementation because the exact config shape and trust-boundary policy need maintainer design. The maintainable direction is an opt-in core routing contract that preserves existing isolation unless explicitly configured.
