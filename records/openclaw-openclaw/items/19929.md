## Summary

Keep open: current main still lacks an opt-in shared-session surface for selected group/channel/thread chats, the prior implementation PR is closed unmerged, and the remaining work needs product and security design rather than cleanup closure.

## What This Changes

Request an opt-in configuration that maps selected Discord/Slack group, channel, and thread chats to a shared agent session while preserving reply routing to the originating chat.

## Best Possible Solution

Define an additive opt-in shared-session contract with isolated defaults, explicit channel/thread membership, optional thread inheritance, preserved origin reply routing, grouped queue serialization, schema/help/docs updates, and focused Discord/Slack/security regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current routing builds distinct non-DM channel/session keys and the strict session schema has no channel grouping option. I did not run live Discord/Slack traffic in this read-only review.

## Solution Assessment

No single implementation path is proven best yet. A channelGroups-style option is plausible, but maintainers still need to settle the product/security model across grouped channels, threads, DMs, queueing, and sandbox policy.
