## Summary

Keep open: current main and v2026.6.10 still lack selective shared sessions for configured group/channel/thread chats, and the active groupScope PR only covers the narrower group-to-main case.

## What This Changes

Requests an opt-in config surface that maps selected Discord/Slack group, channel, and thread chats to a shared session while preserving origin reply routing and isolated defaults.

## Best Possible Solution

Add a maintainer-approved opt-in session grouping contract with isolated defaults, explicit channel/thread membership, origin-preserving delivery, schema/types/docs updates, and regression coverage across routing plus Discord/Slack/thread behavior.

## Reproduction Assessment

Yes, at source level. Current main deterministically derives isolated keys for non-direct peers and Slack/Discord thread suffixes, and the strict config schema cannot express selected channel/thread grouping.

## Solution Assessment

Unclear as a final product path. The channelGroups shape is plausible, but maintainers should reconcile it with groupScope, named inbound session routing, identity routing, and security policy before implementation.
