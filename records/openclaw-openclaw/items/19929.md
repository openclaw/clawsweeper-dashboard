## What This Changes

Request an opt-in configuration surface that maps selected group/channel/thread chats to shared session keys while preserving isolated defaults and replies to the originating chat.

## Best Possible Solution

Design a maintainer-approved opt-in shared-session routing contract with isolated defaults, explicit channel/thread membership, origin-preserving delivery, schema/types/docs updates, and regression coverage across routing plus Discord/Slack/thread behavior.

Unclear as a final product path. The channelGroups shape is plausible, but maintainers should reconcile it with groupScope, named inbound session routing, identity routing, and security policy before implementation.
