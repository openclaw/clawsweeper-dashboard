## Risks / Open Questions

- A grouped session can expose one chat's transcript or tool-derived context to members of another chat unless membership, authorization, and audit semantics are explicit.
- Treating selected groups as main or trusted sessions can change `sandbox.mode: non-main` behavior and widen host-tool access across a trust boundary.
- Thread inheritance and origin reply routing must be designed together so a shared queue does not lose the source channel/thread or send a response to the wrong destination.
- The reported shared-workspace overwrite concern is plausible, but session grouping should not be treated as a complete transactional fix for concurrent file writes.
