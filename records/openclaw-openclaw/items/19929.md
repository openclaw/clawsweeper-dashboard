## Summary

Keep open. Current main still lacks an opt-in channel/thread session grouping surface, the latest maintainer review explicitly kept this as the canonical tracker, and the related issues/closed PR are design inputs rather than merged replacements.

## What This Changes

Add an opt-in session configuration that lets selected Discord/Slack channels or threads share one agent session while preserving isolated defaults and origin-channel replies.

## Best Possible Solution

Design an additive core session grouping contract with isolated defaults, explicit channel/thread membership, deterministic origin-channel replies, grouped queue serialization, schema/help/docs updates, and routing plus Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main lacks the requested config/schema/routing surface and still resolves non-DM peers to channel-specific keys.

## Solution Assessment

Unclear as a concrete implementation because the config shape and trust-boundary policy need maintainer design. The maintainable direction is an opt-in core routing contract that preserves existing isolation unless explicitly configured.
