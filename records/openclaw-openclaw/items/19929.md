## Summary

Keep open. Current main and v2026.5.7 still preserve isolated group/channel/thread sessions and do not expose the requested opt-in grouping config; the prior implementation PR was closed unmerged and related issues are design inputs, not replacements.

## What This Changes

The issue requests an opt-in configuration for sharing one session across selected group, channel, and thread chats while routing replies back to the originating chat.

## Best Possible Solution

Design an additive core session grouping contract with isolated defaults, explicit membership, deterministic origin-channel replies, thread policy, schema/docs/help updates, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request; source inspection confirms current main lacks the requested config/schema/routing surface and still resolves non-DM peers to channel-specific session keys.

## Solution Assessment

Unclear as a concrete implementation because the config shape and trust-boundary behavior need maintainer design. The maintainable direction is an opt-in core routing contract that preserves existing isolation unless explicitly configured.
