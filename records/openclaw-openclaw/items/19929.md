## Summary

Keep open: current main and v2026.6.8 still lack selective shared sessions for configured group/channel/thread chats, and the related open groupScope work is narrower than this issue.

## What This Changes

The issue requests an opt-in config contract that maps selected Discord/Slack channels or threads for an agent into one shared session while leaving unlisted chats isolated.

## Best Possible Solution

Keep this as the broad selective shared-session tracker and coordinate it with the narrower groupScope PR and canonical inbound-session routing discussion before approving a config contract.

## Reproduction Assessment

Yes, at source level. Current main deterministically derives isolated keys for non-direct peers and Slack thread suffixes, and the strict config schema cannot express selected channel/thread grouping.

## Solution Assessment

Unclear as a final product path. The channelGroups shape is plausible, but maintainers should reconcile it with groupScope, named inbound session routing, and security policy before implementation.
