## Summary

Keep open. Current main still lacks the requested selective channel/thread shared-session surface, and the related open issues are narrower design inputs rather than replacements.

## What This Changes

Request an opt-in config mapping selected Discord/Slack channels and threads into shared sessions while leaving unlisted chats isolated.

## Best Possible Solution

Design an additive opt-in shared-session contract with isolated defaults, explicit channel/thread membership, origin-channel replies, grouped queue serialization, schema/help/docs updates, and Discord/Slack tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main still lacks the requested config/schema/routing surface and still routes non-DM peers to isolated channel keys.

## Solution Assessment

Unclear as a concrete implementation because the final config shape and trust-boundary policy need maintainer design. The maintainable direction is an explicit opt-in core routing contract that preserves current isolation unless configured.
