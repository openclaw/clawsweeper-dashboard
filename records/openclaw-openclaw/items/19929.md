## Summary

Keep open. Current main and the latest release still preserve isolated group/channel/thread sessions and do not expose the requested opt-in grouping config; prior maintainer discussion also treats this as the canonical design tracker, not a duplicate or stale report.

## What This Changes

Request to add opt-in session grouping so selected group/channel/thread chats can share one agent session while unlisted chats remain isolated.

## Best Possible Solution

Design an additive core session grouping contract with isolated defaults, explicit membership, deterministic origin-channel replies, thread policy, schema/docs/help updates, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request; source inspection confirms current main lacks the requested config/schema/routing surface and still resolves non-DM peers to channel-specific session keys.

## Solution Assessment

Unclear as a concrete implementation because this is still a product/config design request. The maintainable path is a narrow additive core routing contract that preserves existing isolation unless explicitly configured.
