## Summary

Keep open. Current main still lacks an opt-in shared session grouping surface for selected channel/thread chats, the related implementation PR was closed unmerged, and recent maintainer/bot comments treat this as the canonical design tracker rather than a duplicate.

## What This Changes

The issue requests a new configuration surface that lets selected group/channel/thread chats share one session while unlisted chats remain isolated.

## Best Possible Solution

Keep this as the canonical product/design tracker for an additive, opt-in shared-session routing contract with isolated defaults, explicit membership, thread semantics, reply routing, schema/docs updates, and security-reviewed sandbox behavior.

## Reproduction Assessment

Yes for the current limitation: source, docs, and tests show non-DM chats still resolve to isolated channel/thread session keys and no grouping config is accepted. I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

Unclear until maintainers choose the exact safety and config model. The narrow maintainable direction is a generic session-routing/config seam, not Discord/Slack-only special cases or a blunt global-session workaround.
