## Summary

Keep open. Current main still lacks a selective shared-session surface for grouped Discord/Slack channels or threads: non-DM routes remain keyed by channel and peer, Slack thread replies still fork by thread timestamp, and the strict session config schema has no channelGroups/groupScope/threadIsolation/includeThreads field. The prior implementation PR was closed unmerged, while #7524, #43286, and #51805 are related design inputs rather than replacements for this specific request.

## What This Changes

Requests an opt-in configuration that maps selected channel or thread identifiers to one shared session key for the same agent while preserving isolated defaults and reply routing to the originating chat.

## Best Possible Solution

Add an explicit, opt-in core session grouping design with isolated defaults, schema/types/help/docs support, deterministic grouped session keys, thread inheritance semantics, reply routing to the originating chat, and regression coverage for Discord, Slack, routing, config validation, and sandbox-sensitive behavior.
