## Summary

Keep open. Current main and v2026.5.12 still lack an opt-in shared-session surface for selected group/channel/thread chats, the original channelGroups PR was closed unmerged, and the newer related groupScope PR is narrower, open, and already has unresolved product/security review findings.

## What This Changes

Request to add a configuration surface that maps selected Discord/Slack group, channel, and thread chats for the same agent into one shared session while preserving isolated defaults.

## Best Possible Solution

Design an additive, opt-in shared-session routing contract with explicit channel/thread membership, isolated defaults, origin-chat reply routing, grouped queue serialization, schema/help/docs updates, and Discord/Slack regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still lacks the requested grouping config and route path.

## Solution Assessment

Unclear as a concrete implementation because config shape, thread inheritance, queueing, and trust-boundary semantics need maintainer design. The maintainable direction is explicit opt-in grouping that preserves current isolated behavior unless configured.
