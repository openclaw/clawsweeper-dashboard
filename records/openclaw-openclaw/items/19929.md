## Summary

Keep open: current main and v2026.6.8 still lack an opt-in surface for selectively sharing chosen group/channel/thread sessions, and the open related PR only covers a narrower group-to-main slice.

## What This Changes

The issue requests an opt-in way to map selected Discord/Slack channels or threads for the same agent to one shared session key while keeping unlisted chats isolated.

## Best Possible Solution

Keep this as the broad shared-session tracker and coordinate it with the narrower groupScope and named inbound-session work before accepting one permanent opt-in routing contract.

## Reproduction Assessment

Yes, at source level. Current main deterministically derives ordinary non-DM keys from channel and peer identifiers, while the strict normal routing/session schemas cannot express selected channel/thread grouping.

## Solution Assessment

Unclear as a final product path. The proposed `channelGroups` shape is plausible, but maintainers should choose the permanent abstraction across groupScope, named inbound routing, configured bindings, reply routing, and security/sandbox behavior.
