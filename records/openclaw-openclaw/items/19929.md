## Summary

Keep open. Current main still lacks an opt-in way to map selected Discord/Slack channels or threads into one shared session, and the related open issues are narrower design inputs rather than a replacement for this broader channel/thread grouping request.

## What This Changes

Request an additive configuration surface for selected group/channel/thread chats to share one session key while preserving isolated defaults and replying to the originating chat.

## Best Possible Solution

Design an explicit opt-in shared-session routing contract with isolated defaults, selected channel/thread membership, origin-chat reply routing, grouped queue serialization, security documentation, schema/help updates, and Discord/Slack regression coverage.

## Solution Assessment

Unclear as a concrete implementation because maintainers still need to choose the config shape, thread inheritance semantics, queueing, reply routing, and sandbox/trust-boundary behavior.
