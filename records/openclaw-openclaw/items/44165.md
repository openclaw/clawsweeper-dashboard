## Summary

Keep open: the PR addresses the right Control UI surface, but the submitted fix would still hide current catalog-backed channels such as Feishu and Microsoft Teams when the snapshot only lists custom plugins.

## What This Changes

The PR exports a built-in channel order, updates Control UI channel ordering to append that list after snapshot IDs, and adds resolver unit tests.

## Best Possible Solution

Merge snapshot-provided custom or loaded IDs with a UI-safe catalog-backed bundled channel order, preserve snapshot IDs first, de-dupe, keep unknown custom IDs on the generic card path, and include regression tests plus changelog coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows renderChannels() maps resolveChannelOrder() directly, and resolveChannelOrder() returns only channelMeta or channelOrder when present, so a custom-only snapshot renders only that custom channel card.

## Solution Assessment

No as submitted. Appending missing cards in the Control UI is the right boundary, but the appended IDs need to come from the current catalog-backed channel source instead of a frozen legacy list.
