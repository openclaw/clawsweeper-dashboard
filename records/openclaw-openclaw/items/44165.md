## Summary

Keep open: the PR targets a valid Control UI bug, but as submitted it would still omit current bundled channel cards such as Feishu and Microsoft Teams because it appends a hard-coded legacy list instead of the catalog-backed bundled order.

## What This Changes

The branch exports a built-in channel order constant, changes `resolveChannelOrder()` to append that hard-coded order after snapshot IDs with de-duplication, and adds resolver unit tests.

## Best Possible Solution

Merge snapshot-provided custom or loaded IDs with a UI-safe catalog-backed bundled channel order, preserve snapshot IDs first, de-dupe, keep unknown custom IDs on the generic card path, and add regression tests plus a required changelog entry.

## Reproduction Assessment

Yes. By source inspection, a `ChannelsStatusSnapshot` whose `channelMeta` contains only a custom ID such as `icenter` makes current main return only that ID, and `renderChannels()` renders exactly that resolver output.

## Solution Assessment

No as submitted. Appending missing bundled cards in the Control UI is the narrow boundary, but the appended order must come from the current catalog-backed bundled channel source rather than a legacy hard-coded list.
