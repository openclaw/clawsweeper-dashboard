## Summary

Keep open: the linked Channels UI behavior is still source-reproducible on current main, but this PR should not merge as-is because it hard-codes an outdated built-in channel list and still lacks contributor real behavior proof.

## What This Changes

The PR exports a hard-coded built-in channel order, appends it to snapshot-provided channel IDs in the Control UI resolver, and adds resolver unit tests.

## Best Possible Solution

Land a corrected UI-only resolver that preserves snapshot/custom IDs first, appends de-duplicated bundled IDs from a browser-safe catalog-backed source, and includes focused coverage plus redacted Control UI proof.

## Reproduction Assessment

Yes by source inspection. A custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only those IDs, and `renderChannels()` renders only that returned list.

## Solution Assessment

No as submitted. Appending missing built-ins in the Control UI is the right boundary, but the appended IDs should come from the current catalog-backed bundled order instead of a frozen legacy list.
