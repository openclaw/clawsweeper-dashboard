## Summary

Keep open: the user problem is still valid on current main, and this branch’s hardcoded fallback omits catalog-backed bundled channels such as Feishu and Microsoft Teams; it also lacks after-fix real behavior proof.

## What This Changes

The PR exports a built-in channel order constant, merges snapshot-provided channel IDs with that list in the Control UI resolver, and adds resolver unit tests.

## Best Possible Solution

Land a corrected UI-safe fix that preserves snapshot IDs first, appends de-duped catalog-backed bundled channel IDs, keeps unknown custom IDs on the generic-card path, and includes resolver coverage plus redacted Control UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only those IDs, and renderChannels() renders only that list.

## Solution Assessment

No as submitted. Appending missing cards in the Control UI is the right boundary, but the appended IDs should come from the current catalog-backed bundled channel order rather than a frozen legacy list.
