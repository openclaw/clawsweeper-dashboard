## Summary

Keep open: the PR targets a real Control UI bug, but the submitted fix hard-codes an outdated built-in channel list and still needs after-fix real behavior proof before merge.

## What This Changes

The PR exports a hard-coded built-in channel order, appends it to snapshot-provided channel IDs in `resolveChannelOrder()`, and adds resolver unit tests.

## Best Possible Solution

Land a corrected UI-only resolver that preserves snapshot/custom IDs first, appends de-duplicated bundled IDs from a browser-safe catalog-backed source, and includes focused coverage plus redacted Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows a custom-only `channelMeta` or `channelOrder` makes `resolveChannelOrder()` return only those IDs, and `renderChannels()` renders only that returned list.

## Solution Assessment

No as submitted. Appending missing bundled cards in the Control UI is the right boundary, but the branch should derive appended IDs from the current catalog-backed bundled order instead of freezing an old UI list.
