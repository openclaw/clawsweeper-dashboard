## Summary

Keep open: the PR targets a real Control UI channel-list gap, but the submitted fix hard-codes an older built-in list and would still omit current bundled channel cards such as Feishu and Microsoft Teams. It also lacks contributor real behavior proof, so it is not ready to merge.

## What This Changes

The PR exports a built-in channel order constant, merges snapshot-provided channel IDs with that list in `resolveChannelOrder()`, and adds resolver unit tests.

## Best Possible Solution

Land a corrected UI-only change that preserves snapshot IDs first, appends de-duped catalog-backed bundled channel IDs, keeps unknown custom IDs on the generic-card path, and covers custom-only, mixed, and empty snapshots.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes current `resolveChannelOrder()` return only those IDs, and `renderChannels()` renders only that list.

## Solution Assessment

No as submitted. Appending missing cards in the Control UI is the right boundary, but the appended IDs need to come from the current catalog-backed bundled channel order instead of a frozen legacy list.
