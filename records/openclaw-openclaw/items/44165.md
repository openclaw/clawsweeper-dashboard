## Summary

Keep open: the branch targets a real, source-reproducible Control UI issue, but as submitted it would still omit current bundled channels outside its frozen list and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR exports a built-in channel order constant, merges snapshot-provided channel IDs with that list in `resolveChannelOrder()`, and adds resolver unit tests for custom-only, mixed, and empty snapshots.

## Best Possible Solution

Land a corrected UI-only change that preserves snapshot IDs first, appends de-duped catalog-backed bundled channel IDs, keeps unknown custom IDs on the generic-card path, and includes focused resolver coverage plus real Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows a custom-only `channelMeta` or `channelOrder` snapshot makes current `resolveChannelOrder()` return only those IDs, and `renderChannels()` renders exactly that list.

## Solution Assessment

No as submitted. Appending missing cards in the Control UI is the right boundary, but the appended IDs need to come from the current catalog-backed bundled channel order instead of a frozen legacy list.
