## Summary

This PR should stay open because the branch still hard-codes an obsolete built-in channel list and would continue hiding catalog-backed bundled channel cards in custom-only snapshots; it also still lacks contributor-supplied real behavior proof after the prior ClawSweeper review requested it.

## What This Changes

The PR changes the Control UI channel-order resolver to append a hard-coded built-in channel list after snapshot-provided channel IDs and adds unit tests for custom-only, mixed, and no-snapshot cases.

## Best Possible Solution

Land a corrected UI-only resolver that preserves snapshot/custom IDs first and appends de-duplicated bundled channel IDs from a browser-safe catalog-backed source, with focused resolver coverage and redacted Control UI proof showing custom and built-in cards together.

## Reproduction Assessment

Yes. Source inspection shows a custom-only channelMeta or channelOrder makes resolveChannelOrder() return only those IDs, and renderChannels() renders exactly that returned list.

## Solution Assessment

No as submitted. Appending missing bundled cards in the Control UI is the right boundary, but the branch should derive them from the current catalog-backed bundled channel order instead of freezing an old UI list.
