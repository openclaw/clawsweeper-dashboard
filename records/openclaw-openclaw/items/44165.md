## Summary

Keep open: current main still has the reported source path, and this PR is not mergeable as submitted because it appends a frozen legacy list that omits catalog-backed bundled channels such as Feishu and Microsoft Teams; it also lacks contributor real behavior proof.

## What This Changes

The PR exports a built-in channel order constant, appends it after snapshot-provided channel IDs in the Control UI resolver, and adds resolver unit tests for custom-only, mixed, and empty snapshots.

## Best Possible Solution

Land a corrected UI-only change that merges snapshot IDs with a browser-safe catalog-backed bundled channel order, preserves unknown custom IDs through the generic card path, and covers custom-only, mixed, and no-snapshot cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only those IDs, and renderChannels() renders only that list.

## Solution Assessment

No as submitted. Appending missing cards in the Control UI is the right boundary, but the appended IDs need to come from the current catalog-backed bundled channel order instead of a frozen legacy list.
