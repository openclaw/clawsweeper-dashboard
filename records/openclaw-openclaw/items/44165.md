## Summary

Keep open: the PR is a plausible focused fix for a source-reproducible Control UI bug, but this external PR still lacks after-fix real behavior proof. Current main still returns only snapshot-provided channel IDs when `channelMeta` or `channelOrder` is nonempty, so the linked issue remains valid and this PR remains the active fix path.

## What This Changes

The PR exports a built-in channel order, appends missing built-in channel IDs after snapshot-provided IDs in the Channels page resolver, and adds unit coverage for custom-only, mixed, and null snapshots.

## Best Possible Solution

Land this UI-only behavior after redacted real setup proof shows a custom-only channel snapshot still renders built-in cards, ideally keeping the built-in order aligned with the channel catalog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes current main return only that custom ID, and `renderChannels()` renders only the returned IDs.

## Solution Assessment

Yes, the proposed UI-only resolver change is a narrow maintainable fix because it preserves snapshot/custom IDs first and appends de-duplicated built-ins without changing gateway plugin loading. A catalog-derived built-in order would be slightly more durable, but the current PR shape is not a correctness blocker.
