## Summary

Keep open: current main still has a source-reproducible Control UI ordering bug, and this PR targets the right UI boundary, but the branch appends a frozen legacy built-in list that misses current catalog-backed bundled channels and still lacks external real behavior proof.

## What This Changes

The PR exports a built-in channel order list, merges snapshot-provided channel IDs with that list in the Control UI resolver, and adds resolver unit tests.

## Best Possible Solution

Land a corrected UI-only fix that preserves snapshot IDs first, appends de-duped browser-safe catalog-backed bundled channel IDs, keeps custom IDs on the generic-card path, and includes resolver coverage plus redacted Control UI proof.

## Reproduction Assessment

Yes, source-reproducible. A custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only those IDs, and renderChannels() renders only that returned list.

## Solution Assessment

No as submitted. Appending missing cards in the Control UI is the right boundary, but the appended IDs should come from the current catalog-backed bundled channel order instead of a frozen legacy list.
