## Summary

Keep open: this PR targets the right Control UI problem and is still paired with the open linked issue, but the submitted fix hard-codes an older built-in channel list and would still omit current catalog-backed cards such as Feishu and Microsoft Teams. The external PR also needs after-fix real behavior proof before merge.

## What This Changes

The branch exports a hard-coded built-in channel order, merges it into `resolveChannelOrder()`, and adds resolver unit tests for custom-only, mixed, and empty snapshots.

## Best Possible Solution

Update this PR or a replacement to merge snapshot-provided IDs with a UI-safe catalog-backed bundled channel order, de-dupe while preserving snapshot IDs first, keep unknown custom IDs on the generic card path, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only those IDs, and `renderChannels()` renders only that list.

## Solution Assessment

No as submitted. Appending missing cards in the Control UI is the right boundary, but the appended IDs need to come from the current catalog-backed bundled channel order instead of a frozen legacy list.
