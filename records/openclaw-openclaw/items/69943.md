## Summary

Keep open. Current main and v2026.5.12 still persist raw session-memory transcript text, and the open implementation PR with closing syntax has not merged.

## What This Changes

The issue requests sanitizing session-memory hook transcript content before persistence so raw chat-template tokens, role markers, raw tool-call XML, and NO_REPLY-only turns are not re-injected into later sessions.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/69961 with a focused sanitizer and regression coverage for the session-memory read and write boundaries.

## Reproduction Assessment

Yes, at source level. A session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is returned unchanged by extractTextMessageContent, appended by getRecentSessionContent, and written under `## Conversation Summary` by the handler.

## Solution Assessment

Yes for the direction. Sanitizing or eliding malformed transcript text before persistence is the narrowest maintainable fix; the issue should remain open until the linked PR lands, is replaced, or is explicitly closed.
