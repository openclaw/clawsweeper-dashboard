## What This Changes

The issue asks OpenClaw to stop session-memory from persisting raw chat-template tokens, leaked tool-call XML, orphan role markers, and NO_REPLY artifacts that poison future sessions.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/97034, then keep this issue as the canonical tracker for any remaining memory recovery or stricter persistence-boundary policy.

## Reproduction Assessment

Yes, at source level: current main sanitizes several artifacts before persistence, but a multiline assistant text containing bare `user` or `assistant` lines still passes through the sanitizer and is written under `Conversation Summary`. I did not run the live quantized-model setup, but the implicated source path is clear.

## Solution Assessment

No, current main is only a partial solution. The narrow open PR is the best immediate prevention fix for orphan role lines, while broader recovery or quarantine behavior for already-poisoned memory files remains a maintainer decision.
