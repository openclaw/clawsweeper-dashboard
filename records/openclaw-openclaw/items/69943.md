## Summary

Keep open. Current main still preserves raw session transcript text and writes it into session-memory Markdown without stripping chat-template tokens, raw tool-call XML, role markers, or NO_REPLY text; the issue also has open PR #69961 with closing syntax, so it should remain open until a fix lands or that PR is replaced.

## What This Changes

The issue requests hardening the bundled session-memory hook so malformed model output is sanitized or elided before being persisted and re-injected as later session context.

## Best Possible Solution

Land one narrow session-memory fix that sanitizes or elides malformed transcript text at the extraction and/or write boundary, with regression coverage for chat-template tokens, raw tool-call XML, orphaned role markers, and NO_REPLY-only turns. Keep the broader historical-session untrusted wrapping concern coordinated with #68751 rather than bundling it into this fix.

## Reproduction Assessment

Yes. A synthetic session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` would be returned by `extractTextMessageContent`, appended by `getRecentSessionContent`, and written under `## Conversation Summary` by the handler.

## Solution Assessment

Yes. Sanitizing before session-memory persistence is the narrowest maintainable fix for the reported loop; current main does not do that yet, so the best path is to review and land #69961 or a focused replacement.
