## Summary

Keep open. Current main still extracts and writes session-memory transcript text without stripping chat-template tokens, raw tool-call XML, orphaned role markers, or silent-token turns, and the open implementation PR at https://github.com/openclaw/openclaw/pull/69961 has closing syntax for this issue but is unmerged.

## What This Changes

The issue asks OpenClaw to stop the bundled session-memory hook from persisting and re-injecting malformed raw model output such as chat-template tokens, raw tool-call XML, role markers, and NO_REPLY-only turns.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/69961 with a focused session-memory sanitization fix and regression coverage at the transcript extraction and/or memory write boundary.

## Reproduction Assessment

Yes, at source level. A session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is returned unchanged by extractTextMessageContent, appended by getRecentSessionContent, and written under `## Conversation Summary` by the handler.

## Solution Assessment

Yes for the direction. Sanitizing or eliding malformed transcript text before persistence and re-injection is the narrowest maintainable fix; current main does not do that yet.
