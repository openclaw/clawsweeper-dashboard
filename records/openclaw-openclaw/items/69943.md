## Summary

Keep open: current main has a merged partial sanitizer, but the reported orphan plain role-marker lines still pass through the session-memory transcript sanitizer and an open linked PR owns that remaining fix.

## What This Changes

The issue reports that bundled session-memory persists raw model/chat artifacts into memory files and re-injects them across `/new` sessions, poisoning later agent context and responses.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/97034, then keep this issue as the canonical tracker for any remaining memory recovery or stricter persistence-boundary policy.

## Reproduction Assessment

Yes, at source level. Current main runs the sanitizer before memory persistence but lacks a plain role-only line rule, so a transcript message containing a standalone user, assistant, or system line can survive into Conversation Summary.

## Solution Assessment

No, current main is only a partial solution. The narrow open PR is the best immediate prevention fix for orphan role lines, while broader recovery or quarantine behavior for already-poisoned memory files remains a maintainer decision.
