## Summary

Keep open. Current origin/main still extracts and writes raw session-memory transcript text without stripping chat-template tokens, raw tool-call XML, role-marker scaffolding, or silent-token turns, and the open implementation PR at https://github.com/openclaw/openclaw/pull/69961 is still unmerged.

## What This Changes

The issue asks OpenClaw to sanitize or elide malformed assistant/user transcript artifacts before session-memory persists and re-injects them across new sessions.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/69961 with a focused session-memory sanitization fix and regression coverage for template tokens, raw tool-call XML, orphaned role markers, and silent-token-only turns.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence synthetic path: a session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is returned unchanged by `extractTextMessageContent`, appended by `getRecentSessionContent`, and written under `## Conversation Summary` by the handler.

## Solution Assessment

Yes for the direction. Sanitizing or eliding malformed transcript text at the transcript extraction and/or write boundary is the narrowest maintainable fix; current main does not do that yet.
