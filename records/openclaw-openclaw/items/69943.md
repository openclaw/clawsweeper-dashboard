## Summary

Keep open. Current main and the latest release still persist raw session-memory transcript text, while the linked implementation PR is open and unmerged with closing syntax for this issue.

## What This Changes

The issue requests sanitizing session-memory hook output before persistence so chat-template tokens, raw tool-call XML, role markers, and NO_REPLY-only turns are not re-injected into later sessions.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/69961 with a focused sanitizer that reuses or matches shared model-token/tool-call stripping behavior and adds regression coverage for the session-memory read and write boundaries.

## Reproduction Assessment

Yes, at source level. A session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is returned unchanged by extractTextMessageContent, appended by getRecentSessionContent, and written under `## Conversation Summary` by the handler.

## Solution Assessment

Yes for the direction. Sanitizing or eliding malformed transcript text before persistence is the narrowest maintainable fix; the open PR is the implementation candidate, but the issue should stay open until a fix merges.
