## Summary

Keep open. Current main still extracts and writes raw session-memory transcript text without sanitizing chat-template tokens, raw tool-call XML, role-marker scaffolding, or NO_REPLY-like turns, and the closing implementation PR is still open and unmerged.

## What This Changes

The issue asks for the bundled session-memory hook to sanitize or elide malformed model output before persisting memory that is re-injected into later sessions.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/69961 with a focused sanitizer/elision fix at the transcript extraction and/or memory write boundary, plus regression coverage for template tokens, raw tool-call XML, role markers, and silent-token-only turns.

## Reproduction Assessment

Yes. A synthetic session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` would be returned unchanged by extractTextMessageContent, appended by getRecentSessionContent, and written under `## Conversation Summary` by the handler.

## Solution Assessment

Yes for the proposed direction. Sanitizing or eliding malformed transcript text at the read/write boundary is the narrow maintainable fix; adding new config or treating this as provider-specific would leave the persistence seam unsafe.
