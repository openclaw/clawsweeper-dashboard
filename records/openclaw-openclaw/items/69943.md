## Summary

Keep open. Current main still persists raw session transcript text into session-memory Markdown without stripping chat-template tokens, raw tool-call XML, orphaned role markers, or NO_REPLY-only turns, and the issue is already paired with open closing PR #69961.

## What This Changes

The issue asks session-memory to sanitize or elide chat-template tokens, raw tool-call XML, orphaned role markers, and NO_REPLY-only turns before persisting them into memory files.

## Best Possible Solution

Land one focused session-memory fix that sanitizes or elides malformed transcript text before persistence and adds regression coverage for template tokens, raw tool-call XML, role-marker leakage, and NO_REPLY-only turns.

## Reproduction Assessment

Yes. A synthetic session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` would flow through `extractTextMessageContent`, be appended by `getRecentSessionContent`, and be written under `## Conversation Summary` by the handler.

## Solution Assessment

Yes for the direction. Sanitizing or eliding at the transcript extraction and/or write boundary is the narrowest maintainable fix; current main does not implement it yet, so #69961 or a focused replacement should carry it.
