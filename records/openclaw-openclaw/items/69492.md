## Summary

Keep open. Current main still has no `SystemEvent` audience/classification contract; queue draining and heartbeat routing remain text/context based, and the linked implementation at https://github.com/openclaw/openclaw/pull/72201 is still open rather than merged or shipped.

## What This Changes

The issue asks for a system-event audience/classification contract so internal runtime events can be distinguished from user-facing system events across consumer paths.

## Best Possible Solution

Keep this as the canonical tracker until maintainers land, revise, or redirect the open hidden-context implementation, then verify representative system-event leakage symptoms against shipped behavior.

## Reproduction Assessment

Yes, at source level: enqueue a generic or cron-awareness `SystemEvent` and trace current main through `drainFormattedSystemEvents` or heartbeat selection; consumers only have text, context, delivery, and trust fields, not audience metadata. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear pending maintainer design. The open wrap-on-drain PR is a plausible narrow direction for hidden runtime context, but it is partial and current main has not adopted the event-audience policy.
