Keep open. Current main still lacks a SystemEvent audience/classification contract, and the live related PR only addresses one hidden-runtime-context slice, so this needs maintainer API/product direction rather than cleanup closure.

## What This Changes

The issue requests event-level audience classification for SystemEvent entries so internal runtime events can be separated from user-facing consumer paths.

## Best Possible Solution

Keep this as the canonical design tracker until maintainers review #72201 and decide whether to land the hidden-context primitive, split remaining delivery cases into narrower issues, or define a broader event-delivery taxonomy.
