## Summary

Keep open. This maintainer draft adds the right generic backend shape for durable plugin allow-always approvals, but the Skill Workshop opt-in currently cannot persist because it does not provide the plugin id that the new store requires, and its key can expose an absolute workspace path through approval events before hashing.

## What This Changes

The PR adds an `allowAlwaysKey` contract for plugin approvals, stores allow-always decisions in SQLite-backed plugin state, wires Skill Workshop lifecycle approvals into that path, and updates plugin docs/tests.

## Best Possible Solution

Keep the generic SQLite-backed durable approval mechanism, but fix Skill Workshop with a stable non-sensitive core/plugin scope, add a second-call reuse regression test, and update the docs before landing.

## Reproduction Assessment

Yes. Source inspection shows Skill Workshop requests include allowAlwaysKey but no pluginId, while the new durable store requires pluginId, toolName, and allowAlwaysKey before it persists or reuses allow-always.

## Solution Assessment

No. The generic backend is the right layer, but the built-in Skill Workshop opt-in needs a durable core/plugin scope, a non-sensitive key, and docs/tests aligned with the new contract.
