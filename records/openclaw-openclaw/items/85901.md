## Summary

Keep open: this is a protected maintainer issue, and current main still has a source-reproducible Control UI path where a same-role/same-text history message can replace a richer optimistic media message.

## What This Changes

The issue asks Control UI to preserve user and assistant media previews across chat history refreshes instead of replacing them with poorer text-only transcript copies.

## Best Possible Solution

Update Control UI history reconciliation to compare media richness/equivalence before replacing optimistic messages, then add focused tests for user attachments and generated/source-reply media refreshes.

## Reproduction Assessment

Yes for a source-level fixture: previousMessages can contain a local user message with text plus image content while chat.history returns the same role/text without media, and current reconciliation returns the poorer history copy. I did not run a live browser repro, so the generated-media side remains unproven in real UI.

## Solution Assessment

Yes, a narrow reconciliation fix is the best path. The implementation should compare backed media or preserve the richer local message until history carries equivalent media, with regression coverage for both stale history and final catch-up.
