## Summary

This should stay open: the item is maintainer-authored/protected, and current main still has a source-level path where a same-role/same-text history message can replace a richer local media preview during refresh.

## What This Changes

The issue requests that Control UI preserve user and assistant media previews across chat history refreshes instead of replacing media-bearing bubbles with text-only transcript copies.

## Best Possible Solution

Fix Control UI history reconciliation so richer local media messages remain visible until transcript history carries equivalent MediaPath or attachment content, with regression coverage for user attachments and generated/source-reply media refreshes.

## Reproduction Assessment

Yes, at source level: a previous user message with text plus image/attachment content can share the same role/text signature as a poorer history message and be discarded during refresh. I did not run a live browser or OCM reproduction in this read-only review.

## Solution Assessment

Yes, the best path is a narrow reconciliation/media-equivalence fix plus regression coverage. The renderer already supports transcript media fields and structured attachments, so this should not require a new rendering surface.
