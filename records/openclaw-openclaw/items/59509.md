## Summary

Keep open. Current main still resolves Google Vertex location from transport options or `GOOGLE_CLOUD_LOCATION`, while the existing per-model `params` seam is not mapped into Vertex transport options, so mixed global/regional Vertex models remain unsupported.

## What This Changes

Request support for per-model Google Vertex AI location selection so preview and GA Gemini models can use global and regional endpoints under the same project and credentials.

## Best Possible Solution

Add and document a Google provider `extraParamsForTransport` mapper for string `params.location`, preserve the existing env fallback, add focused regression coverage, and leave smart defaults out unless maintainers explicitly choose them.

## Reproduction Assessment

Yes, source-level. Configure two `google-vertex` model entries with different `params.location` values; current main can read per-model params but never maps `location` into Vertex transport options, while Vertex URL construction reads only transport options or `GOOGLE_CLOUD_LOCATION`.

## Solution Assessment

Yes for the explicit override if it is implemented as `params.location` through the provider hook seam. A new top-level model schema key or preview-model smart defaults would be broader than needed.
