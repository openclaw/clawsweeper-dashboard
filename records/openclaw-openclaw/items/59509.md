## Summary

Keep open. Current main still has the provider-level/env Vertex location path, while the existing per-model params seam is not mapped into Google Vertex transport options.

## What This Changes

The issue requests per-model Google Vertex location overrides so mixed global and regional Vertex models can share the same provider credentials.

## Best Possible Solution

Add a Google provider `extraParamsForTransport` mapper for string `params.location`, preserve the existing transport/env fallback, document the param, and cover mixed global/regional Vertex models with focused tests.

## Reproduction Assessment

Yes, source-level. Configure two `google-vertex` model entries with different `params.location` values; current main reads per-model params but the Google hook bundle never maps `location` into the Vertex transport option, so URL construction still uses only `options.location` or `GOOGLE_CLOUD_LOCATION`.

## Solution Assessment

Yes for the explicit override if implemented through the existing provider hook seam. A new top-level model schema field or automatic preview-model defaulting would be broader and should be deferred unless maintainers explicitly choose that product behavior.
