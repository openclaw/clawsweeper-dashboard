## Summary

Keep open. Current main still resolves Google Vertex location only from transport options or `GOOGLE_CLOUD_LOCATION`, while per-model `params` are accepted but not mapped into Vertex transport options; the requested mixed global/regional model setup remains a valid provider feature gap.

## What This Changes

The issue requests a per-model Google Vertex AI location override so preview and GA Vertex models can use different endpoints under one provider configuration.

## Best Possible Solution

Support and document `agents.defaults.models["google-vertex/<model>"].params.location`, map it through a Google-plugin transport hook into Vertex options, and leave smart preview defaults for a separate product decision.

## Reproduction Assessment

Yes, source-level. Configure two `google-vertex` model entries with different `params.location` values; current main reads those params but never maps `location` into Vertex transport options, while the Vertex URL builder uses only transport options or `GOOGLE_CLOUD_LOCATION`.

## Solution Assessment

Yes for the explicit override. The narrow maintainable path is a Google-provider `extraParamsForTransport` mapper for `params.location`, not a provider-specific top-level schema key or hard-coded preview defaults.
