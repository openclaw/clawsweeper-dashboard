## Summary

Keep open. Current main and the latest release still resolve Google Vertex location from transport options or `GOOGLE_CLOUD_LOCATION` only, while per-model params are collected but no Google-owned hook or wrapper forwards a per-model location into Vertex request options.

## What This Changes

Request support for a per-model Google Vertex location override so global preview and regional GA Gemini models can share one project and credential setup.

## Best Possible Solution

Implement and document a Google-plugin-owned `params.location` per-model override that reaches Vertex request options, with tests proving mixed `global` and regional model URLs; leave automatic preview/global defaults for separate product work.

## Reproduction Assessment

Yes, source-level. Configure two `google-vertex` model entries with different `agents.defaults.models[...].params.location` values; current code can read those params, but the Vertex URL builder only uses transport options or `GOOGLE_CLOUD_LOCATION`.

## Solution Assessment

Yes for the explicit override. The maintainable path is to map a Google-owned per-model `params.location` into the Vertex transport request path, not to add smart preview defaults or a provider-specific top-level schema key in this fix.
