## Summary

Keep open. Current main and the latest release still have only transport-level Vertex `options.location` plus the generic provider transport-param seam; the Google plugin does not expose or forward a per-model Google Vertex location, so mixed global/regional model use under one project remains a valid feature gap.

## What This Changes

Add support for configuring a Google Vertex location per model so preview/global and GA/regional Gemini models can share a project and credentials.

## Best Possible Solution

Support and document `agents.defaults.models["google-vertex/<model>"].params.location`, forward it through the Google provider hook into Vertex transport options, preserve the existing environment fallback, and leave automatic preview-model defaults for a separate decision.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to configure two `google-vertex` model entries with different desired `params.location` values: current main accepts the params object but never maps `location` into Vertex transport options, while top-level `location` is outside the strict schema.

## Solution Assessment

No current solution is in place. The narrow maintainable path is a Google-plugin-owned `extraParamsForTransport` mapping for `params.location`, not a provider-specific top-level schema key or hard-coded preview-model defaults.
