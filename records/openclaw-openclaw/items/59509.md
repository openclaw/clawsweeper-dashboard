## What This Changes

Add a per-model Google Vertex location override so different `google-vertex` models can use global and regional endpoints under the same project and credentials.

## Best Possible Solution

Support a Google-provider-owned string `params.location` transport patch for `google-vertex`, preserve the existing env fallback, document it, and add mixed global/regional model coverage.

## Reproduction Assessment

Yes, source-level. Configure two `google-vertex` model entries with different `params.location` values; current main can read per-model params but never maps `location` into the Vertex transport path.

## Solution Assessment

Yes for the explicit override if it uses the existing provider hook seam. A new top-level model field or automatic preview-model defaults would be broader than needed.
