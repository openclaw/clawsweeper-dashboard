## Summary

Keep open. Current main still has a source-level gap: Google Vertex location can be supplied through transport options or `GOOGLE_CLOUD_LOCATION`, but per-model params are not mapped into those transport options.

## What This Changes

Request support for a per-model Google Vertex location override so different Vertex models can use different regional or global endpoints under one project and credential set.

## Best Possible Solution

Add and document `agents.defaults.models["google-vertex/<model>"].params.location`, map it through a Google provider `extraParamsForTransport` hook into Vertex transport options, and leave automatic preview defaults for a separate decision.

## Reproduction Assessment

Yes, source-level. Configure two `google-vertex` model entries with different `params.location` values; current main reads per-model params but the Google provider never maps `location` into Vertex transport options.

## Solution Assessment

Yes for the explicit override. The narrow maintainable solution is a Google-plugin `extraParamsForTransport` mapper for `params.location`, not a top-level model schema key or hard-coded preview defaults.
