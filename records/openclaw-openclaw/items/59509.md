## Summary

Keep open. Current main and the latest shipped release still only resolve Google Vertex location from transport options or `GOOGLE_CLOUD_LOCATION`, while per-model `params.location` is read by config but not mapped into the Vertex transport path.

## What This Changes

Request support for per-model Google Vertex location overrides so global-only preview models and regional GA models can share one GCP project and credential setup.

## Best Possible Solution

Add a Google provider `extraParamsForTransport` mapper for string `params.location`, preserve the existing env fallback, document it, and cover mixed global/regional Vertex models with focused tests.

## Reproduction Assessment

Yes, source-level. Configure two `google-vertex` model entries with different `params.location` values; current main reads per-model params but never maps `location` into Vertex transport options, while the Vertex URL builder reads only transport options or `GOOGLE_CLOUD_LOCATION`.

## Solution Assessment

Yes for the explicit override if implemented through the existing provider hook seam. A new top-level model schema field or preview-model smart defaults would be broader than needed.
