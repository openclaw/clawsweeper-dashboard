Full review comments:

- **[P1] Use one signal key for tool heartbeats:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:299`
  - body: `onToolActivity` is registered with `streamAbortController.signal`, but `notifyToolActivity` is called with `runAbortController.signal`. Because the listener map is keyed by `AbortSignal`, this notification will not reach `armTimer()`.
  - confidence: 0.95
- **[P1] Persist tool activity across provider waits:** `src/agents/embedded-agent-runner/run/attempt.ts:3687`
  - body: The notifier fires after a cataloged tool finishes, but the previous provider stream listener has already been cleaned up and the next provider wait has not registered its listener yet. That drops the heartbeat instead of recording activity for the first post-tool provider wait.
  - confidence: 0.88
- **[P2] Emit heartbeats from shared tool execution:** `src/agents/embedded-agent-runner/run/attempt.ts:3687`
  - body: This adds the heartbeat only inside the Tool Search catalog executor, but normal direct tools execute through agent-core's shared `executePreparedToolCall` path. The reported read and exec style tools can bypass the heartbeat unless the shared path is wired or proved irrelevant.
  - confidence: 0.84
