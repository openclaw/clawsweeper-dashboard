## Summary

Keep open: this PR targets a real subagent availability bug and is the candidate fix for the linked idle-timeout issue, but the patch is not merge-ready because the heartbeat is keyed to different AbortSignals and no after-fix real subagent proof is present.

## What This Changes

The PR adds a tool-activity heartbeat module, subscribes the embedded LLM idle-timeout wrapper to it, and notifies after cataloged tool execution in the embedded runner.

## Best Possible Solution

Keep this PR open, wire tool activity through one shared run/source activity key or the shared tool-execution event path, add focused idle-timeout coverage, then require redacted real subagent proof.

## Reproduction Assessment

Yes from source inspection for the PR defect: the listener is registered on the wrapper stream signal while notifications use the run signal, so the WeakMap lookup cannot find the listener. I did not run a live subagent reproduction in this read-only review.

## Solution Assessment

No. The owner boundary is plausible, but the implementation must share the activity key and prove the real tool-execution path before it can solve the reported idle timeout.
