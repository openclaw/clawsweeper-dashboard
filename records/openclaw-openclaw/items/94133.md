## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Persist tool activity across the next model wait:** `src/agents/embedded-agent-runner/run/attempt.ts:3679`
  - body: The heartbeat fires after a tool finishes, but agent-core has already returned from the previous provider stream and will only create the next `streamWithIdleTimeout` listener when it starts the following model request. That notification is dropped instead of resetting the first wait after tool results, so long post-tool model silence can still hit the 120s watchdog.
  - confidence: 0.88
- **[P1] Use the same signal key for tool heartbeats:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:298`
  - body: `onToolActivity` is registered with `streamAbortController.signal`, but the PR notifies `runAbortController.signal`. Since the listener map is keyed by `AbortSignal`, tool completion never reaches `armTimer()`.
  - confidence: 0.94
- **[P2] Emit heartbeats from the shared tool path:** `src/agents/embedded-agent-runner/run/attempt.ts:3679`
  - body: The new notification is inside the Tool Search catalog executor, while normal direct tools run through agent-core's `executePreparedToolCall` path. A fix for `read`/`exec`-style subagent activity needs to notify from the shared tool completion path or prove direct tools cannot bypass it.
  - confidence: 0.81
