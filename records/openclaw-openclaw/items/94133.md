## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Persist tool activity into the next provider wait:** `src/agents/embedded-agent-runner/run/attempt.ts:3681`
  - body: The notifier fires after a cataloged tool finishes, but the previous provider stream listener has already been cleaned up and the next `streamWithIdleTimeout` listener is not created until the following model request. That drops the heartbeat instead of extending the first post-tool model wait, so long model silence can still hit the idle watchdog.
  - confidence: 0.88
- **[P1] Use one signal key for tool heartbeats:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:300`
  - body: `onToolActivity` is registered with `streamAbortController.signal`, while `notifyToolActivity` is called with `runAbortController.signal`. Because the listener map is keyed by `AbortSignal`, this notification will not reach `armTimer()`.
  - confidence: 0.95
- **[P2] Emit heartbeats from shared tool execution:** `src/agents/embedded-agent-runner/run/attempt.ts:3681`
  - body: This adds the heartbeat only inside the Tool Search catalog executor, but normal direct tools execute through agent-core's shared `executePreparedToolCall` path. The reported `read`/`exec` style tools can therefore bypass the heartbeat unless the shared path is wired or proved irrelevant.
  - confidence: 0.82
