## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use one signal key for tool heartbeats:** `src/agents/embedded-agent-runner/run/llm-idle-timeout.ts:300`
  - body: `onToolActivity` subscribes with `streamAbortController.signal`, but the notifier added in `attempt.ts` calls `notifyToolActivity(runAbortController.signal)`. Because the listener map is keyed by AbortSignal identity, the new heartbeat will not reach `armTimer()` for this path.
  - confidence: 0.95
- **[P1] Persist tool activity into the next provider wait:** `src/agents/embedded-agent-runner/run/attempt.ts:3681`
  - body: The notifier fires only after a cataloged tool finishes, but the previous provider stream listener has already been cleaned up and the next `streamWithIdleTimeout` listener is not created until the following model request. That drops the activity instead of extending the first post-tool provider wait.
  - confidence: 0.88
- **[P2] Emit heartbeats from shared tool execution:** `src/agents/embedded-agent-runner/run/attempt.ts:3681`
  - body: This hook only covers the Tool Search catalog executor. Normal direct tools execute through agent-core's shared `executePreparedToolCall` path, so reported tools like `read` and `exec` can still bypass the heartbeat.
  - confidence: 0.84
