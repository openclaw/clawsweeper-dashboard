## Label Justifications

- `P2`: Telegram TTS voice-note routing is a user-visible channel delivery bug with limited Telegram-specific blast radius.

## Summary

Keep this PR open, but it is not merge-ready: the diff does not wire the existing generic `audioAsVoice` context through Telegram `sendMedia`, and the new filename heuristic can change default MP3 delivery without explicit voice intent.

## What This Changes

The PR adds `audioAsVoice` and generated TTS filename checks to Telegram audio routing, plus a regression test for generated MP3 voice compatibility.

## Best Possible Solution

Preserve the existing explicit `audioAsVoice` contract at the Telegram outbound adapter boundary, keep default audio routing unchanged without explicit voice/TTS intent, and require redacted real Telegram proof before merge.

## Reproduction Assessment

Yes from source for the generic media path: `ChannelOutboundContext` can contain `audioAsVoice`, while Telegram `sendMedia` currently does not pass that field to `sendMessageTelegram`. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No: the PR adds a sink-side option and filename heuristic, but the narrow maintainable fix is to preserve the existing explicit `audioAsVoice` contract at the Telegram outbound adapter boundary.
