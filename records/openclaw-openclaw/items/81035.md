## Summary

Keep open: the PR addresses a real Mattermost final-delivery accounting gap and includes usable terminal proof, but the changed monitor condition still allows a false success log for an empty no-send outcome.

## What This Changes

The PR propagates Mattermost reply-delivery outcomes, adds a no-visible-reply diagnostic helper and tests, wires diagnostic logging into the final monitor path, and adds a changelog entry.

## Best Possible Solution

Land the outcome-threading approach after tightening success logging to text/media outcomes and, if maintainers want full issue closure, follow with an explicit visible failure/fallback path for no-send finals.

## Reproduction Assessment

No high-confidence live Mattermost reproduction was established. Source inspection shows the current main path can receive an `"empty"` delivery result, discard it, and still log a delivered final reply.

## Solution Assessment

No, not quite. Propagating the helper outcome is the right narrow fix, but the success log should be gated on `"text"` or `"media"`, not every non-reasoning non-violation outcome.
