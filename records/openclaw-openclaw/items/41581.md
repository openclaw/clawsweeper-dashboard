## Summary

Close as superseded. Current main and the latest shipped Telegram docs now intentionally support Telegram preview streaming via durable `sendMessage`/`editMessageText` updates, and merged PR #75073 removed the native draft preview transport that this issue asks to restore.

## What This Changes

The issue requests restoring Telegram DM `partial` streaming to the smoother native draft-style preview behavior seen before 2026.3.8.

## Best Possible Solution

Keep the shipped message-edit implementation and docs from #75073; if native draft previews should return, track that as a new explicit Telegram product/API request rather than this old regression report.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram DM preview streaming creates a preview with `sendMessage` and updates it with `editMessageText`, and current tests/docs lock in that path.

## Solution Assessment

Yes for cleanup. Closing as superseded by merged #75073 is the narrowest maintainable outcome because current main intentionally removed native draft transport; restoring it would require a new maintainer product decision.
