## Summary

Keep open. Current main does not restore the reported Telegram DM draft-style streaming; it now removes native draft preview transport entirely and documents Telegram preview streaming as durable `sendMessage`/`editMessageText` edits. Because that appears to be a fresh maintainer direction rather than an accidental missing fix, this should not be auto-closed or auto-repaired without maintainer judgment.

## What This Changes

Restore or preserve smooth Telegram DM partial streaming via native draft-style preview instead of coarse `sendMessage`/`editMessageText` updates after 2026.3.8.

## Best Possible Solution

Maintainers should decide whether Telegram native draft previews are still a supported product direction. If yes, the fix should be a targeted Telegram plugin change that reintroduces a controlled draft-capable DM answer transport with lifecycle coverage for duplicate finals, stale preview cleanup, reasoning/tool-progress separation, quote exclusions, and message-edit fallback; if no, the issue should be closed with the #75073 rationale and user-facing docs/changelog evidence.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: on current main, Telegram DM partial preview creation goes through `sendMessage` and subsequent preview updates go through `editMessageText`, while tests and docs lock that behavior in.

## Solution Assessment

Unclear. The requested restoration is not currently the narrowest safe change because current main deliberately removed native draft preview transport; the maintainable path depends on whether maintainers want to support draft previews again or treat durable message edits as the intended behavior.
