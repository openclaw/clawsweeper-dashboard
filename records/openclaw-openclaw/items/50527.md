## Summary

Keep open. The PR adds useful direct `/hooks/agent` metadata caps, but the same metadata can still be produced by mapped `/hooks/<name>` agent actions and sent into replay-key construction and dispatch without the new validation.

## What This Changes

The PR adds per-field maximum length checks for hook agent payload metadata in `normalizeAgentPayload()` plus focused helper tests.

## Best Possible Solution

Use one shared hook-agent metadata validator for both direct normalized payloads and rendered mapped actions, leaving `message` governed by the HTTP body-size limit.

## Reproduction Assessment

Yes. Configure a mapped `/hooks/<name>` agent action whose `name`, `sessionKey`, `to`, `model`, or `thinking` template renders an over-limit string; current `main` sends that mapped action through replay-key construction and dispatch without `normalizeAgentPayload()`.

## Solution Assessment

No. The PR is directionally correct, but the narrowest maintainable fix is shared validation for the common hook-agent action shape rather than direct-route-only checks.

## Review Findings

Overall confidence: 0.9
