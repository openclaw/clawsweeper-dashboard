## Summary

Keep open. The direct `/hooks/agent` hardening is useful, but the PR still leaves mapped `/hooks/<name>` agent metadata unchecked before replay-key construction and dispatch.

## What This Changes

The branch adds per-field maximum length checks for direct hook agent payload metadata in `normalizeAgentPayload()` plus focused helper tests.

## Best Possible Solution

Add one shared hook-agent metadata validator for the shared action shape and call it from both direct normalization and mapped hook handling while keeping `message` governed by the existing body-size limit.

## Reproduction Assessment

Yes. Configure a mapped `/hooks/<name>` agent action whose `name`, `sessionKey`, `to`, `model`, or `thinking` template renders an over-limit string; current main passes that mapped action into replay-key construction and dispatch without the direct normalizer.

## Solution Assessment

No. The PR is directionally right, but validating only `normalizeAgentPayload()` is not the narrowest maintainable solution because mapped hooks carry the same metadata shape through a separate path.
