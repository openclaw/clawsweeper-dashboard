## Summary

Keep open: the hardening is useful, but the patch validates only the direct `/hooks/agent` normalizer while mapped agent hooks can still render oversized metadata and dispatch it, and the PR still has only unit-test/CI proof.

## What This Changes

The PR adds direct `/hooks/agent` per-field length checks for hook agent metadata and unit tests for oversized and exact-boundary values.

## Best Possible Solution

Use one shared hook-agent metadata validator for both direct and mapped dispatch before replay-key construction, with mapped regression coverage and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a mapped agent hook that templates `name`, `sessionKey`, `to`, `model`, or `thinking` from a large payload value, then post to that mapped hook; I did not execute it because this review is read-only.

## Solution Assessment

No. Checking only `normalizeAgentPayload()` is incomplete because mapped hook actions bypass that function; the maintainable fix is a shared validator for both direct and mapped agent hooks.
