## Summary

Keep open. The patch is focused, but it only guards the direct `/hooks/agent` normalizer while mapped hooks can still render and dispatch oversized metadata, and the PR still lacks real behavior proof.

## What This Changes

The PR adds per-field maximum length checks and tests for direct hook agent metadata in `normalizeAgentPayload()`.

## Best Possible Solution

Move metadata length validation into a shared hook-agent action validator used by both direct and mapped hooks before replay hashing and dispatch, with mapped regression tests and upgrade proof.

## Reproduction Assessment

Yes, at source level. Configure a mapped agent hook that templates `name`, `sessionKey`, `to`, `model`, or `thinking` from a large payload value, then post to that mapped hook.

## Solution Assessment

No. Checking only `normalizeAgentPayload()` is not the best fix because mapped hook actions bypass that function; the maintainable fix is shared validation for both direct and mapped agent hooks.
