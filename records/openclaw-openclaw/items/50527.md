## Summary

Keep open: this is useful hardening, but it is not merge-ready because the new checks only cover direct `/hooks/agent` payloads while mapped hook agent actions can still render and dispatch oversized metadata, and the external PR has no real behavior proof.

## What This Changes

The PR adds per-field max-length validation for direct hook agent metadata fields in `normalizeAgentPayload()` plus helper tests.

## Best Possible Solution

Move metadata length validation into a shared hook-agent action validator used after direct normalization and mapped action rendering, then cover direct and mapped oversized/boundary behavior with tests and redacted real hook request proof.

## Reproduction Assessment

Yes, source-reproducible. Configure a mapped agent hook whose metadata template references a large payload field and post a request under the 256 KB body limit; the mapped branch dispatches rendered values without `normalizeAgentPayload()`, though I did not execute the gateway because this review is read-only.

## Solution Assessment

No. Direct-only validation is narrower than the exposed hook-agent action surface; the maintainable fix is shared validation for the common dispatch payload plus a maintainer decision on thresholds.
