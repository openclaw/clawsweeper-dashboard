Keep open. ## What This Changes

The PR broadens the cron retry timeout classifier and tests so spaced, hyphenated, and underscored timeout wording can be treated as retryable.

## Best Possible Solution

Tighten the separated timeout match with a trailing boundary, keep the helper-level classification fix, and merge only after the contributor adds real cron proof; leave the broader setup-watchdog root cause to https://github.com/openclaw/openclaw/pull/86893.

## Reproduction Assessment

Yes at helper level: current main's regex misses spaced timeout text, and a direct regex probe shows the PR would classify that text. The full scheduler slot-loss claim is less certain because the runtime caller also supplies a structured timeout reason from the shared failover classifier.

## Solution Assessment

No as submitted. The helper is the right narrow layer for unstructured retry text, but the regex needs a trailing boundary and the PR needs real cron proof before merge.
