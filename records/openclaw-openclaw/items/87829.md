This PR wraps plugin `api.registerTool(...)` factories and returned tool `execute` callbacks in the registering plugin runtime scope, with an e2e regression test.

## Best Possible Solution

Yes, the implementation is a narrow registry-boundary fix that reuses the existing AsyncLocalStorage scope helper and avoids new request-time discovery or public API. The remaining question is upgrade safety for installed plugins that may already read the ambient scope.
