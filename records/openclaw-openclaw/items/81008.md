## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject symlinked memory directories during export:** `extensions/memory-core/src/encrypted-backup.ts:59`
  - body: `listFilesRecursive(path.join(workspaceDir, "memory"))` starts from a path that may itself be a symlink, then the later `lstat` checks only the returned leaf files. A workspace with `memory/` symlinked elsewhere will archive those outside files as `memory/...`, leaking non-memory data into the backup. Reject symlinked directory components before walking or walk with safe realpath/dirent checks.
  - confidence: 0.9
- **[P1] Reject final symlink leaves when overwriting imports:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: With `--overwrite`, this writes using flag `w` after checking only parent directories, so an existing symlink at a restored leaf like `MEMORY.md` is followed and overwritten outside the restore target. Reject or safely replace final symlink leaves even when overwrite is enabled.
  - confidence: 0.92
- **[P3] Remove contributor-local lab safety scaffolding:** `extensions/memory-core/src/lab-safety.ts:3`
  - body: This helper hard-codes one contributor's production home and is then allowlisted as an unused file; it is not part of the memory backup feature and adds local/private policy to the plugin. Drop the helper/script and keep tests on repo-generic temp-root safeguards.
  - confidence: 0.8
