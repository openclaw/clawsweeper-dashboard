## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject symlinked memory roots before export:** `extensions/memory-core/src/encrypted-backup.ts:33`
  - body: `listFilesRecursive()` calls `fs.readdir(rootDir)` before proving `workspace/memory` is not a symlink. A symlinked memory directory can be walked and serialized as `memory/...`, so add a root lstat/realpath containment check before descending and cover that case.
  - confidence: 0.94
- **[P1] Reject final symlink leaves during overwrite:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: With `--overwrite`, restore checks parent directories but then writes the final output path with flag `w`, which follows an existing `target/MEMORY.md` symlink outside `--target`. Reject symlink leaves or use a no-follow safe-open/write path before allowing overwrite.
  - confidence: 0.92
- **[P3] Remove contributor-local lab safety scaffolding:** `extensions/memory-core/src/lab-safety.ts:3`
  - body: This helper hard-codes one contributor's production home and is allowlisted as an unused file, so the repo gains machine-specific safety policy that does not protect other contributors or CI. Replace it with generic temp-dir isolation in tests or remove it.
  - confidence: 0.86
