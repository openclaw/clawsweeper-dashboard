## Summary

Keep this PR open, but it is not merge-ready: the encrypted memory backup implementation still follows a symlinked memory directory during export and can clobber a symlink target during overwrite restore. The feature direction is plausible, but it also overlaps the broader backup flow and carries new crypto dependency policy that should get owner review after the concrete safety bugs are fixed.

## What This Changes

The PR adds memory-core privacy audit and encrypted memory export/import CLI commands, docs, tests, dependency/package changes, and a small install-scan lint tweak.

## Best Possible Solution

Harden the memory backup traversal and restore writes with focused regressions, then decide whether this memory-specific backup surface should merge as a narrow memory tool or fold into the broader backup architecture.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection of PR head shows the export walker can follow a symlinked `memory` directory and overwrite restore can follow a symlink leaf. I did not execute the branch because this review is read-only.

## Solution Assessment

No: the requested capability may be useful, but this patch is not the best merge shape until the symlink hardening gaps are fixed and the memory-specific backup surface is reconciled with the broader backup flow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject symlinked memory directories during export:** `extensions/memory-core/src/encrypted-backup.ts:33-43`
  - body: `listFilesRecursive()` calls `fs.readdir(rootDir)` on `workspace/memory` before any directory lstat or realpath containment check. If `memory` is a symlink, export walks files outside the workspace and serializes them under lexical `memory/...` paths; lstat or realpath-contain each directory before descending, and add a regression where `workspace/memory` is a symlink.
  - confidence: 0.92
- **[P1] Refuse symlink leaves before overwrite restores:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: When `--overwrite` is set, restore writes the final output path with normal `fs.writeFile(..., flag: "w")` after checking only parent directories. An existing symlink at `target/MEMORY.md` can therefore redirect the write outside `--target`; reject a final symlink leaf or use a no-follow safe open path before overwriting.
  - confidence: 0.9
