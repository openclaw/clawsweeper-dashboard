## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject symlinked memory roots before export:** `extensions/memory-core/src/encrypted-backup.ts:33`
  - body: `listFilesRecursive()` calls `fs.readdir(rootDir)` on `workspace/memory` before proving that directory is not a symlink or realpath-contained. If `memory` points outside the workspace, export walks and serializes outside files as `memory/...`; validate the root before descending and add a root-symlink regression.
  - confidence: 0.93
- **[P1] Reject final symlink leaves during overwrite:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: With `--overwrite`, restore skips the final `lstat(outputPath)` check and writes with `flag: "w"`, which follows an existing `target/MEMORY.md` symlink outside `--target`. Reject an existing symlink leaf, or use a no-follow safe-open/write path before allowing overwrite.
  - confidence: 0.91
- **[P3] Remove contributor-local lab safety scaffolding:** `extensions/memory-core/src/lab-safety.ts:3`
  - body: This helper hard-codes one contributor's production OpenClaw home and is imported into general memory-core tests, with an unused-file allowlist entry. That does not protect other developers or CI and leaves unrelated local policy in the package/script surface; keep test safety generic or drop it.
  - confidence: 0.86
