## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject symlinked memory directories during export:** `extensions/memory-core/src/encrypted-backup.ts:33-44`
  - body: `fs.readdir(rootDir)` follows `workspace/memory` when that directory is itself a symlink, and the later path check only sees the lexical in-workspace path. That can serialize files outside the memory workspace into the encrypted backup; lstat or realpath-contain each walked directory before descending and add a regression for a symlinked `memory` root.
  - confidence: 0.9
- **[P1] Refuse symlink leaves before overwrite restores:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: With `--overwrite`, restore writes `outputPath` using flag `w` after checking only parent directories. If the final target is a symlink, this can write outside `--target`; reject an existing symlink leaf or use a no-follow safe open path before writing.
  - confidence: 0.88

## Security Review
