## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject symlinked memory directories during export:** `extensions/memory-core/src/encrypted-backup.ts:33-43`
  - body: `listFilesRecursive()` calls `fs.readdir()` on `workspace/memory` before proving that root is not a symlink. If `memory` points outside the workspace, export walks and serializes outside files under `memory/...`; lstat/realpath-contain the root and every directory before descending.
  - confidence: 0.92
- **[P1] Reject symlink leaves before overwrite restores:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: With `--overwrite`, restore skips the final-path lstat and writes with normal `fs.writeFile(..., flag: "w")`. An existing `target/MEMORY.md` symlink can redirect the write outside `--target`; reject the final symlink leaf or use a no-follow safe-open path before overwriting.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff still introduces file export and restore symlink risks in security-sensitive encrypted backup paths.
