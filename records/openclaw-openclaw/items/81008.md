Overall confidence: 0.94

Full review comments:

- **[P1] Reject symlinked memory roots before export:** `extensions/memory-core/src/encrypted-backup.ts:33`
  - body: `listFilesRecursive()` calls `fs.readdir(rootDir)` on `workspace/memory` before proving that directory is not a symlink. If `memory` points outside the workspace, export walks and serializes outside files as `memory/...`; validate the root or realpath-contain it before descending, and add a root-symlink regression.
  - confidence: 0.93
- **[P1] Reject final symlink leaves during overwrite:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: With `--overwrite`, restore skips the final `lstat(outputPath)` check and writes with `flag: "w"`, which follows an existing `target/MEMORY.md` symlink outside `--target`. Reject a symlink leaf, or use a no-follow safe-open/write path before allowing overwrite.
  - confidence: 0.91
- **[P2] Drop unrelated root Slack dependencies:** `package.json:1775-1777`
  - body: This memory-core PR adds `@slack/bolt`, `@slack/types`, and `@slack/web-api` as root runtime dependencies even though the Slack plugin already declares them locally and the memory backup code does not use them. Keep those plugin-only dependencies out of the root install unless this PR also changes the core packaging path that requires them.
  - confidence: 0.82
- **[P3] Remove contributor-local lab safety guards:** `extensions/memory-core/src/lab-safety.ts:3`
  - body: The new safety helper hard-codes one local `/Users/.../.openclaw` path and is then wired into tests/allowlists. That private machine policy does not protect other contributors or CI and leaves unrelated scaffolding in the memory-core/script surface; use generic temp-root safety or drop it.
  - confidence: 0.87
