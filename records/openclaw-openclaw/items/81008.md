## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject symlinked memory directories during export:** `extensions/memory-core/src/encrypted-backup.ts:53`
  - body: The export hardening only skips symlink leaves like MEMORY.md. If workspace/memory itself is a symlink, fs.readdir traverses the outside directory and the files are archived under memory/... paths, so export can leak files outside the memory workspace. Lstat/realpath the memory root and recursive parents before collecting files, and cover that case in tests.
  - confidence: 0.93
- **[P1] Reject final symlink leaves during overwrite import:** `extensions/memory-core/src/encrypted-backup.ts:178-181`
  - body: When --overwrite is set, writeMemoryBackupArchive skips the existing-path lstat and then writes with flag "w". Parent symlink checks do not catch a final symlink leaf, so an archive path like MEMORY.md can overwrite the symlink target outside --target. Check and reject final symlinks before overwrite writes.
  - confidence: 0.94
- **[P1] Regenerate the root npm shrinkwrap for new deps:** `package.json:1817`
  - body: This PR adds root runtime dependencies but does not update the root npm-shrinkwrap, and the current check-guards job fails with npm-shrinkwrap.json stale. Regenerate the shrinkwrap so packaged installs and release checks see the same dependency graph as package.json/pnpm-lock.yaml.
  - confidence: 0.96
- **[P3] Remove contributor-local lab safety scaffolding:** `extensions/memory-core/src/lab-safety.ts:3`
  - body: The added lab-safety helper and script hard-code one contributor's local OpenClaw home and are not part of the shipped memory privacy workflow.
