## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject symlinked memory roots during export:** `extensions/memory-core/src/encrypted-backup.ts:33`
  - body: `collectMemoryBackupArchive()` passes `workspace/memory` into `listFilesRecursive()`, and that helper immediately calls `fs.readdir(rootDir)` without first `lstat`/realpath-validating the root. If `memory` itself is a symlink to another directory, export follows it and serializes outside files as `memory/...`; reject or contain-check the root before descending and add a root-symlink regression.
  - confidence: 0.93
- **[P1] Reject symlink leaves before overwrite restores:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: With `--overwrite`, restore skips the final-path existence check and writes with normal `fs.writeFile(..., flag: "w")`, which follows an existing symlink leaf. A preexisting `target/MEMORY.md` symlink can redirect the restore outside `--target`; lstat/reject the final leaf or use a no-follow safe-open path before writing and touching mtimes.
  - confidence: 0.91
