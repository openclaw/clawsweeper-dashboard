## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Reject symlinked memory directories during export:** `extensions/memory-core/src/encrypted-backup.ts:33-44`
  - body: `fs.readdir(rootDir)` follows `memory` when that directory is itself a symlink, and the later `path.relative(workspaceDir, resolvedPath)` check only sees the lexical path. That lets encrypted export silently include files from outside the memory workspace; lstat or realpath-contain each directory before walking it, and add a regression for `workspace/memory` as a symlink.
  - confidence: 0.9
- **[P1] Refuse symlink leaves before overwrite restores:** `extensions/memory-core/src/encrypted-backup.ts:183-185`
  - body: When `--overwrite` is set, `fs.writeFile(..., { flag: "w" })` follows an existing symlink at the final restore path. The parent-directory check does not inspect `outputPath` itself, so a restore can write outside `--target`; reject a symlink leaf before overwriting or use a no-follow safe open path.
  - confidence: 0.88

## Security Review
