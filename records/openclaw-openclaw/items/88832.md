## Summary

This PR is still useful and should stay open: current main and v2026.5.28 still use the short Telegram polling restart policy and lack a stop-timeout cooldown, while the PR moves the restart/cooldown logic into the Telegram polling session with focused coverage and terminal proof. ## What This Changes

The PR changes Telegram polling restart backoff, adds stop-timeout burst cooldown state, deduplicates normal polling stop requests, and extends polling-session regression tests.

## Best Possible Solution
