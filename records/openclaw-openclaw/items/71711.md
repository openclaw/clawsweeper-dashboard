## Summary

Keep open. Current main still lacks recurring cron expiration fields or scheduler enforcement, and the linked agent-scheduling RFC keeps the exact public contract and security policy in maintainer-design territory.

## What This Changes

Add optional active-until/expires-at lifetime semantics for recurring cron jobs, including expiration handling and operator-visible CLI/list support.

## Best Possible Solution

Define an additive recurring-job expiration contract in core cron, then thread it through persisted schema, scheduler enforcement, Gateway protocol, CLI, Control UI, docs, and audit semantics.

## Solution Assessment

Unclear until maintainers choose the public contract. The capability belongs in core scheduler state, but the exact fields, defaults, timezone behavior, audit event, and interaction with https://github.com/openclaw/openclaw/issues/71712 need design review first.
