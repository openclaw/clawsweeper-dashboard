The issue asks OpenClaw core to add an optional `activeUntil`/`expiresAt` expiration field for recurring cron jobs, with remove-or-disable behavior and CLI/list/audit support.

## Best Possible Solution

If maintainers accept the feature, define one canonical recurring-expiration contract across the cron protocol schema, SQLite persistence, CLI/list surfaces, scheduler enforcement, audit/docs, and focused regression tests.
