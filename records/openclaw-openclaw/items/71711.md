Add an optional per-recurring-cron `activeUntil` or `expiresAt` field plus expiration behavior for remove-or-disable lifecycle handling.

## Best Possible Solution

Keep this open for maintainer API/product review, then implement one canonical recurring-expiration contract across cron protocol schemas, SQLite persistence, CLI/UI create-edit-list surfaces, scheduler enforcement, docs, and focused tests if maintainers accept it.
