## Summary

Keep open. Current main and the latest release still lack the recurring cron lifetime field, scheduler enforcement, CLI/UI creation support, docs, and audit behavior, and the linked scheduling RFC keeps permanent-job and lifetime semantics in maintainer design territory.

## What This Changes

Request to add optional recurring cron expiration fields such as `activeUntil` or `expiresAt`, with scheduler cleanup or disable behavior and CLI/listing support.

## Best Possible Solution

Define an additive recurring-job lifetime contract in core cron, then thread it through persisted state, scheduler enforcement, Gateway protocol, CLI, Control UI, docs, tests, and audit semantics alongside the linked scheduling RFC.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main and `v2026.5.18` gives high confidence that the requested field, flag, schema, scheduler enforcement, and docs surface do not exist today.

## Solution Assessment

Unclear until maintainers choose the public contract. The capability likely belongs in core scheduler state, but the exact fields, defaults, timezone behavior, audit event, and interaction with https://github.com/openclaw/openclaw/issues/71712 need design review first.
