## Summary

Keep open. Current main still lacks recurring cron expiration in the persisted cron model, Gateway contract, scheduler enforcement, CLI/UI, and docs, and the linked agent-scheduling RFC keeps the lifetime policy in maintainer design territory.

## What This Changes

Requested per-job activeUntil/expiresAt and onExpire behavior for recurring --every and --cron jobs.

## Best Possible Solution

Define an additive recurring-job expiration contract in core cron, then thread it through persisted state, scheduler enforcement, Gateway protocol, CLI, Control UI, docs, plugin scheduling contracts, and audit semantics in coordination with https://github.com/openclaw/openclaw/issues/71712.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection against current main gives high confidence that the requested field, flag, schema, enforcement, and docs path do not exist today.

## Solution Assessment

Unclear until maintainers choose the public contract. The capability belongs in core scheduler state, but the exact fields, defaults, timezone behavior, audit event, and interaction with https://github.com/openclaw/openclaw/issues/71712 need design review first.
