## Summary

Keep open: current main has the exec approval modal and decision buttons, but the command is still rendered as one plain escaped text node with no syntax tokenization or highlight styling.

## What This Changes

Request a Control UI exec approval modal that shows highlighted command text, execution context, one-click approval decisions, and a timeout countdown.

## Best Possible Solution

Add a UI-only shell-token highlighter over the already-sanitized command display string, style the token spans, and cover it with focused Control UI tests without changing the gateway protocol.

This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. The narrowest maintainable path is UI-only tokenized Lit rendering over sanitized text, preserving the existing countdown, context rows, queue behavior, and decision buttons.
