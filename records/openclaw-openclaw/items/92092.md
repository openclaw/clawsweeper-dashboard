## Summary

Current main has a source-reproducible message-loss regression: Claude CLI narration is classified when the commentary option is defined but false, although no commentary consumer is wired. The PR’s gate alignment and lossless parser fallback are the narrowest maintainable fix, with no actionable code or security defect found; however, this external contribution still needs inspectable after-fix proof from a real Claude CLI-backed gateway/channel run before merge.

## What This Changes

The PR changes Claude CLI narration routing so commentary classification only activates with an item-event consumer and otherwise falls back to normal assistant text deltas.

## Best Possible Solution

Land the shared gate invariant and lossless parser fallback after a redacted real Claude CLI-backed gateway/channel run demonstrates commentary-off narration preservation and commentary-on single delivery.

## Reproduction Assessment

Yes at source level. Current main enables classification for a defined false commentary flag, wires no commentary callback, and clears buffered narration when the next tool-use block arrives.

## Solution Assessment

Yes. Aligning both runner gates removes the invalid producer-without-consumer state, while the parser fallback is a narrow lossless defense that reuses the existing assistant lane rather than creating another routing path.
