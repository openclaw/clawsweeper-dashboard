## Summary

This PR remains necessary because current main can classify Claude CLI pre-tool narration without wiring its destination, silently losing assistant text. The reshaped head removes that invalid state, has focused regression coverage, and is supported by convincing paired before/after and live behavior proof; no actionable patch defect was found.

## What This Changes

The PR removes the independent CLI commentary-classification flag, derives Claude CLI classification from the presence of a commentary consumer, and updates parser and runner regression tests.

## Best Possible Solution

Land the single-invariant implementation once required checks are acceptable, keeping consumer presence as the sole trigger for Claude CLI commentary classification and retaining the focused enabled/disabled regression coverage.

## Reproduction Assessment

Yes. The paired real-parser run reproduces the narration drop on the exact merge base, and current-main source inspection confirms the classify-without-consumer path that causes it.

## Solution Assessment

Yes. Removing the parallel classification flag makes the invalid state unrepresentable and is safer and leaner than synchronizing duplicate gates or adding an assistant-lane fallback.
