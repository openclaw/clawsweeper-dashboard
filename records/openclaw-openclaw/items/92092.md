## Label Justifications

- `P1`: This fixes a newly introduced regression that silently suppresses assistant narration in real CLI-backed agent and channel workflows.

## Summary

This PR fixes a confirmed current-main regression where Claude CLI narration can be classified as commentary and silently discarded when no commentary consumer is active. The implementation is focused, covers both runtime entry points, preserves the existing verbose-on path, and includes convincing live gateway proof; no blocking correctness or security finding was found.

## What This Changes

The PR aligns Claude CLI commentary classification with active item-event delivery and falls back to normal assistant deltas when classified commentary has no consumer.

## Best Possible Solution

Maintain one no-loss invariant: classify Claude CLI narration only when a commentary consumer is available, and otherwise emit the original buffered text exactly once through the assistant lane.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: commentary can be classified while no callback is wired, after which the parser clears the buffered text without delivering it.

## Solution Assessment

Yes. Aligning both caller gates fixes the immediate regression, while the parser fallback establishes the narrowest maintainable defense against future consumer mismatches without changing active commentary delivery.
