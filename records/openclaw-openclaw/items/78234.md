## Summary

Keep open. This protected MEMBER/maintainer PR addresses the current-main root cause, but the PR head has a blocking approval-scope bug: the native `allow-always` reuse key does not include `cwd`, so an approval can be reused for the same command/input in a different working directory.

## What This Changes

The PR changes Codex app-server/native hook approval handling by omitting pre-guardian native `PermissionRequest` hooks in Codex approval modes, caching same-session native `allow-always` decisions, and adding request-scoped plugin approval decision validation/rendering.

## Best Possible Solution

Amend the PR so native `allow-always` reuse includes `cwd` in the cache key, keep the request-scoped approval decision hardening, and land only after the targeted approval tests and normal PR gates pass.

## Reproduction Assessment

Yes from source inspection: current main collapses native `allow-always` to one-shot `allow`, and the PR body gives a concrete Telegram/Codex live reproduction path. I did not execute the live path because this review is read-only.

## Solution Assessment

No as submitted. The direction is narrow and maintainable, but the PR must include `cwd` in the native `allow-always` reuse key before it is a safe fix.
