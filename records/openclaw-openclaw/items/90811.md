## Summary

Keep this PR open with changes requested: the central boundary-stamping approach is well-targeted and backed by useful live telemetry, but the new guard can double-stamp existing UTC channel envelopes, so the patch is not ready to merge as-is.

## What This Changes

The PR moves embedded user-turn timestamp stamping to the LLM boundary, canonicalizes text-only user content for cache-stable serialization, removes transient gateway/TUI stamping, preserves CLI prompt timestamps, persists reply-run timestamps, and adds boundary regression coverage.

## Best Possible Solution

Keep the single LLM-boundary stamping design, but first make existing channel timestamp-envelope detection cover the real `formatAgentEnvelope` UTC/ISO shape and add focused regression coverage; maintainers should then explicitly accept the prompt-byte compatibility change.

## Reproduction Assessment

Yes from source inspection: current main decorates active direct user turns differently from stored history, and the PR’s new test models that current-versus-historical serialization flip. I did not execute tests because this review was explicitly read-only.

## Solution Assessment

No, not yet: the LLM boundary is the right owner for cache-stable serialization, but the implementation must recognize the shipped UTC channel envelope shape before it is the best mergeable fix.
