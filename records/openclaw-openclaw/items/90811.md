## What This Changes

The PR moves direct embedded user-turn timestamping from gateway/TUI ingress to the embedded LLM boundary, canonicalizes text-only user content to strings, preserves CLI timestamping, normalizes inbound timestamps, and adds cache-stability regression tests.

## Best Possible Solution

Land the boundary-stamping approach only after explicit maintainer acceptance of historical timestamp prefixes and green required checks; otherwise request a narrower cache-stable design that preserves old model-visible history.

## Reproduction Assessment

## Solution Assessment

Mostly yes, subject to maintainer approval. The LLM boundary is the right single owner for provider-visible byte stability and avoids persisting prompt-only stamps, but exposing timestamps on historical direct turns is a product compatibility choice.
