## Summary

Keep this PR open for maintainer review. The patch addresses a real current-main prompt-cache instability with focused boundary changes and strong live proof, but merging it intentionally changes model-visible prompt bytes for existing embedded full-resend sessions, so the remaining decision is compatibility acceptance plus normal CI validation.

## What This Changes

The branch removes transient gateway/TUI timestamp injection for embedded user turns, stamps and canonicalizes user messages at the LLM boundary from stored timestamps, preserves CLI prompt stamping, persists reply-run timestamps, and adds boundary/cache regression tests.

## Best Possible Solution

Land the single-boundary serialization fix after maintainer acceptance of the model-visible compatibility change and green CI or equivalent broad agent/gateway validation, while leaving the broader Anthropic cache-control work on the linked issue.

## Reproduction Assessment

Yes. Source inspection shows current main stamps direct current turns before persistence while history replays bare transcript strings, and the PR body supplies live after-fix cache and wire-diff proof; I did not run a live repro in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. Moving direct-turn stamping to one LLM-boundary path is the cleanest owner-boundary fix for byte stability; keeping gateway-only stamping would preserve the cache bust, and persisting stamped text would pollute transcript/UI state.
