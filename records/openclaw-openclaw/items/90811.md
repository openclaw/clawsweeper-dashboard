## What This Changes

The PR moves direct user-turn timestamping from gateway/TUI ingress to the embedded LLM boundary, canonicalizes text-only user messages for stable serialization, preserves CLI prompt timestamping, stores normalized inbound timestamps, and adds cache-stability tests.

## Best Possible Solution

Land only after a maintainer accepts the model-visible historical timestamp behavior; keep the single LLM-boundary stamp and cache-stability regression coverage if that compatibility choice is accepted.

## Reproduction Assessment

Yes. Current main source shows direct gateway paths stamp only the current user turn before persistence, while the current LLM boundary does not apply stable per-message stamping; that makes the reported current-vs-history byte flip source-reproducible without executing the full provider path.

## Solution Assessment

Yes, with a maintainer compatibility decision. Moving stamping to the LLM boundary is the cleanest owner boundary for provider wire bytes, but maintainers must explicitly accept the broader historical timestamp prompt semantics before merge.
