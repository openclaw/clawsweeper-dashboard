## Summary

Keep this PR open: the embedded prompt-cache fix is plausible and backed by useful live telemetry, but the patch introduces blocking compatibility regressions for raw model probes and CLI-backed direct turns.

## What This Changes

The PR removes transient gateway/TUI timestamp injection, stamps user messages at the embedded LLM boundary from their stored timestamp, canonicalizes text-only user content to strings, and adds cache-stability tests.

## Best Possible Solution

Land the embedded boundary-based cache fix only after raw model probes remain exact and CLI-backed direct turns still receive the existing per-turn timestamp context, with focused regression tests for both paths.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: raw runs pass through the new stamping wrapper, and CLI-backed runs receive the bare prompt directly without the embedded LLM boundary.

## Solution Assessment

No. The embedded boundary is a good place for full-resend prompt-cache stability, but the patch needs raw-run exclusion and non-embedded runtime coverage before it is the best fix.

  - confidence: 0.93

Add a CLI/runtime boundary stamp or keep a runtime-specific gateway stamp before removing this injection.
  - confidence: 0.9
