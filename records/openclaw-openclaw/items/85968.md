## Summary

Keep open. Current main and the latest shipped `v2026.5.22` source still only explicitly create and verify `/home/node/.config/openclaw`, so the parent `/home/node/.config` ownership problem remains, and two linked open PRs already own the fix path.

## What This Changes

Fix the slim Docker image so `/home/node/.config` is owned by `node:node` after upgrade, allowing `npx` and skill installs to write under `~/.config`.

## Best Possible Solution

Review and land the more complete linked fix that creates `/home/node/.config` with `node:node` ownership before the leaf directory, repairs setup-time permissions, and adds parent-plus-leaf Dockerfile regression coverage.
