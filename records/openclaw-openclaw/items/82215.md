## Summary

Keep open: current `main` still has the raw path comparison described by the report, and the linked open fix PR uses closing syntax, so this issue should remain open until that PR lands or is closed.

## What This Changes

The issue asks for `openclaw doctor` to canonicalize package-root and git-top-level paths so junction- or symlink-backed source checkouts are treated as git installs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82229 or an equivalent canonical path comparison with focused coverage, then close this issue as implemented.

## Reproduction Assessment

Yes, source-reproducible with high confidence: if doctor receives a linked package root while git reports the real checkout path, current `main` compares the raw strings and returns `not-git`. I did not run a live Windows junction setup.

## Solution Assessment

Yes. Canonicalizing both paths before comparison is the narrow maintainable fix, and the open linked PR is already shaped around that solution with focused tests.
