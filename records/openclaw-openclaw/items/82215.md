## Summary

Keep open: current `main` still has the raw path comparison described by the report, and the surrounding update code already has canonical path comparison patterns that support a narrow fix.

## What This Changes

The issue requests that `openclaw doctor` treat a junction-backed or symlink-backed source checkout as a git checkout when the linked package root and `git rev-parse --show-toplevel` resolve to the same filesystem location.

## Best Possible Solution

Share or mirror the existing canonical path comparison in doctor’s git-checkout detector and add focused regression coverage for a linked package root resolving to the real git checkout.

## Reproduction Assessment

Yes, from source inspection: if doctor receives a linked package root while git reports the real top-level path, the current equality check returns `not-git`. I did not run a live Windows junction setup in this read-only review.

## Solution Assessment

Yes, canonicalizing both paths before comparison is the narrow maintainable fix; reusing the update path’s existing realpath-style helper would reduce divergence between doctor and update detection.
