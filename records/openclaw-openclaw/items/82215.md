## Summary

Keep open: current main still has the raw path comparison that can misclassify a linked checkout, and the open closing PR at https://github.com/openclaw/openclaw/pull/82229 is the active fix path. This issue should close after that PR, or an equivalent fix, lands.

## What This Changes

Doctor should treat a junction or symlinked package root and the real Git top-level as the same checkout before choosing update guidance.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82229 or an equivalent canonicalized comparison with focused regression coverage, then close this issue from the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a raw equality check between `git rev-parse --show-toplevel` and `root`; a junction or symlinked root whose real path differs will enter `not-git`, and the linked PR body supplies equivalent macOS symlink proof.

## Solution Assessment

Yes. Canonicalizing both paths before comparing is the narrow maintainable fix and aligns doctor with the existing update status/runner behavior; the open closing PR is the current implementation path.
