## What This Changes

Doctor should treat symlink or Windows junction-backed OpenClaw source checkouts as git-backed by canonicalizing the package root and git top-level paths before comparison.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82229 or an equivalent canonical path comparison with focused regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes, source-reproducible: current main compares the git top-level path and package root as raw strings, so a linked root and real checkout path fall into `not-git`. I did not run a live Windows junction setup in this read-only review.

## Solution Assessment

Yes. Canonicalizing both paths before comparison is the narrow maintainable fix, and the linked open PR already implements that direction with focused regression coverage.
