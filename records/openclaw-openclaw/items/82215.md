## Summary

Keep open: current main still uses raw path equality in doctor’s pre-update git checkout detector, so a linked package root can be classified as `not-git` even when it resolves to the same checkout as Git’s top-level path.

## What This Changes

`openclaw doctor` should canonicalize symlink/junction-backed package roots before deciding whether to offer the git update flow or package-manager guidance.

## Best Possible Solution

Mirror or share the update path’s canonical realpath comparison in doctor’s git-checkout detector and add focused regression coverage for linked package roots.

## Reproduction Assessment

Yes, with source-backed reproduction: a linked package root and Git’s real top-level path compare unequal as strings but equal by realpath, and current doctor code treats that as `not-git`. I did not run the complete Windows junction interactive doctor flow.

## Solution Assessment

Yes. Canonicalizing both paths before comparison is the narrow maintainable fix, preferably by mirroring or sharing the existing update realpath comparison pattern rather than changing update flow policy.
