Keep open. Current main and v2026.6.5 provide documented programmatic APIs and canonical internal protocol schemas, but neither exposes a Swagger/OpenAPI artifact; implementing one still requires a maintainer decision about which HTTP surfaces are public and whether the private WebSocket protocol packages become a supported external contract.

## What This Changes

Request a generated OpenAPI/Swagger schema so external users can generate clients for OpenClaw programmatic APIs.

## Best Possible Solution

Define versioned OpenAPI only for the explicitly supported HTTP endpoints, with authentication and enablement documented, and separately decide whether to publish a supported Gateway protocol/client SDK for full WebSocket RPC access.
