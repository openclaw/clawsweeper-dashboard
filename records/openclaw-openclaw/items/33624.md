## Summary

The request remains useful and is not implemented by current main: OpenClaw documents several HTTP endpoints and has typed internal Gateway schemas, but it does not publish an OpenAPI document or a language-neutral complete client contract. Maintainer scoping is required because the canonical control plane is WebSocket RPC, its discovery list is intentionally incomplete, and the optional admin HTTP surface carries full operator privileges.

## What This Changes

Requests a published OpenAPI/Swagger schema so users can generate programmatic clients for OpenClaw endpoints.

## Best Possible Solution

Define supported public API tiers, then generate a versioned language-neutral contract from canonical schemas: OpenAPI for stable HTTP endpoints and an AsyncAPI/JSON-Schema-style artifact plus client guidance for WebSocket RPC and events.

## Reproduction Assessment

Not applicable: this issue requests a new published API-contract artifact rather than reporting broken existing behavior. Current-main inspection confirms the requested OpenAPI document is absent.

## Solution Assessment

No, a blanket Swagger schema is not the best complete solution because OpenClaw’s canonical API is WebSocket RPC and events. The best design uses OpenAPI only for supported HTTP routes and a separate generated contract for the Gateway protocol.
