## Summary

Keep open. Current main still has documented Gateway HTTP endpoints but no OpenAPI/Swagger artifact, dependency, generator, or discovery route, and adding one would define a public operator API contract that needs maintainer API-boundary and security judgment.

## What This Changes

Request to generate and expose an OpenAPI/Swagger schema for OpenClaw so custom API clients can be generated.

## Best Possible Solution

Define the stable public Gateway API boundary, add a checked OpenAPI artifact and docs for that boundary, and only serve runtime discovery if maintainers choose an authenticated endpoint.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main give high-confidence absence proof for a Gateway OpenAPI/Swagger artifact or discovery route.

## Solution Assessment

Unclear until maintainers choose the public API boundary. The maintainable path should model stable Gateway HTTP routes deliberately, not blindly expose every handler or mix config/RPC/plugin surfaces into one schema.
