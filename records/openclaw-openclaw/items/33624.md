## Summary

Keep open. Current main still has no Gateway OpenAPI/Swagger artifact or discovery route, and publishing one would define a public API contract that needs maintainer API-boundary and auth/security scoping decisions.

## What This Changes

The issue requests an OpenAPI/Swagger schema for OpenClaw so custom clients can be generated for programmatic access.

## Best Possible Solution

Define the stable public Gateway API subset, add a checked OpenAPI artifact and docs for that subset, and only add runtime discovery if maintainers choose an authenticated/scoped endpoint.

## Reproduction Assessment

Not applicable as a feature request. Source/docs/package searches and Gateway route inspection on current main provide high-confidence absence proof for an OpenAPI/Swagger artifact or discovery route.

## Solution Assessment

Unclear until maintainers choose the public API boundary. The maintainable path should model stable Gateway HTTP routes deliberately, not auto-export every handler, RPC method, plugin route, or operator-only surface.
