## Summary

Keep open. Current main still lacks a Swagger/OpenAPI artifact or schema-serving route for the Gateway/API surface, and the existing App SDK plus admin HTTP RPC plugin are related programmatic-access surfaces but not a language-neutral schema contract.

## What This Changes

Request to publish or expose an OpenAPI/Swagger schema for OpenClaw APIs so users can generate custom clients.

## Best Possible Solution

Define the stable public Gateway API boundary first, then publish a versioned, auth-aware OpenAPI or equivalent schema with deterministic generation, docs, and contract tests.

## Reproduction Assessment

Not applicable; this is a feature request, not broken existing behavior. Source, docs, package, and live issue searches give high-confidence absence proof for a current OpenAPI/Swagger schema artifact or route.

## Solution Assessment

Unclear; an OpenAPI schema is a reasonable target, but auto-exposing every handler, RPC method, or plugin route before choosing the stable public API boundary would be risky. The maintainable path is an explicitly scoped contract with auth/redaction rules and generated drift checks.
