## Summary

This remains a valid but under-scoped product request: current main publishes a machine-readable JSON Schema for the Gateway WebSocket protocol and offers an opt-in admin HTTP RPC plugin, but neither is an OpenAPI contract for general client generation. Maintainers first need to choose the supported public interface and security boundary.

## What This Changes

Expose a machine-readable OpenAPI/Swagger schema so users can generate custom clients for programmatic OpenClaw access.

## Best Possible Solution

Define one explicitly supported external automation contract, then generate a filtered schema and client artifacts from its canonical TypeBox definitions while excluding internal, non-advertised, and plugin-private methods.

## Reproduction Assessment

Not applicable. Current source and documentation confirm that OpenClaw publishes Gateway JSON Schema but no OpenAPI artifact; the issue requests a new capability rather than reporting broken existing behavior.

## Solution Assessment

No. Generating Swagger indiscriminately from existing handlers would not match the canonical WebSocket RPC architecture or its conservative discovery boundary; the public interface and stability commitment must be selected first.
