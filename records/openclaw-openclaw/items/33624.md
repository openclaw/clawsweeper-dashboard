## Summary

Keep this open: current main and v2026.6.8 expose documented Gateway HTTP surfaces, including OpenAI-compatible endpoints and the opt-in admin HTTP RPC plugin, but neither publishes an OpenAPI/Swagger document for generating clients. The remaining work is real, but it needs a maintainer decision on which Gateway/operator surfaces are stable enough to describe as a public schema.

## What This Changes

The issue requests publishing an OpenAPI/Swagger schema so users can generate custom clients for OpenClaw programmatic access.

## Best Possible Solution

Maintainers should first choose the stable public Gateway HTTP contract, then publish a narrow OpenAPI artifact and tests for that contract without automatically treating every operator or plugin route as public API.
