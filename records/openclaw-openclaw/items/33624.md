## Summary

Keep open. Current main still does not expose a Swagger/OpenAPI artifact or schema-serving route, and the related App SDK, TypeBox protocol JSON Schema, and opt-in admin HTTP RPC plugin do not satisfy language-neutral OpenAPI client generation. This needs maintainer API/product and security scoping before implementation.

## What This Changes

Request to publish or serve an OpenAPI/Swagger schema so users can generate custom clients for OpenClaw programmatic access.

## Best Possible Solution

Define the stable public Gateway/API boundary first, then publish a versioned, auth-aware OpenAPI or equivalent schema with deterministic generation, docs, and drift tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source, docs, package, and live issue/PR searches give high-confidence absence proof for a current OpenAPI/Swagger artifact or route.

## Solution Assessment

Unclear; an OpenAPI schema is a reasonable target, but auto-exposing every handler, RPC method, or plugin route before choosing a stable public API boundary would be risky. The maintainable path is an explicitly scoped contract with auth/redaction rules and generated drift checks.
