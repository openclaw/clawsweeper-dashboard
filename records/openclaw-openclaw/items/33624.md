## Summary

Keep open. Current main still lacks a Swagger/OpenAPI schema artifact or discovery route for OpenClaw Gateway; adjacent SDK and TypeBox JSON Schema work do not satisfy cross-language OpenAPI client generation, and the public API boundary needs maintainer scoping.

## What This Changes

Request to publish a Swagger/OpenAPI schema for OpenClaw so users can generate custom API clients.

## Best Possible Solution

Define the stable public Gateway API boundary first, then generate a versioned OpenAPI or equivalent schema with docs and an explicitly authenticated discovery story if runtime exposure is wanted.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the public API boundary. The maintainable path should model stable Gateway HTTP/RPC surfaces deliberately, not auto-expose every handler, plugin route, or operator-only method.
