## Summary

Keep open. Current main still does not expose a Swagger/OpenAPI artifact, generator, Swagger UI, or `/openapi.json` route for the Gateway HTTP APIs; the adjacent `config.schema` surface is a config JSON-schema RPC/CLI feature, not an OpenAPI contract for programmatic HTTP clients.

## What This Changes

Request an OpenAPI/Swagger schema for OpenClaw so users can generate custom API clients.

## Best Possible Solution

Define the stable public Gateway HTTP boundary, then add a checked OpenAPI artifact and docs; serve an authenticated discovery endpoint only if maintainers choose runtime discovery.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs inspection give high-confidence absence proof for a Gateway HTTP OpenAPI/Swagger artifact or discovery route.

## Solution Assessment

Unclear until maintainers decide the public API boundary. The request is reasonable, but the maintainable solution should model only stable HTTP routes rather than blindly exposing every Gateway handler or mixing config/WebSocket schemas into one spec.
