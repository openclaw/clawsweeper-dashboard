## Summary

Keep open. Current main still has no OpenAPI/Swagger artifact, generator, Swagger UI, or `/openapi.json` route for the Gateway HTTP APIs, and the related config-schema PR was closed unmerged with non-config Gateway APIs explicitly out of scope. This remains real API-boundary work rather than a cleanup close.

## What This Changes

The issue asks OpenClaw to publish an OpenAPI/Swagger schema for existing endpoints so users can generate custom API clients for programmatic access.

## Best Possible Solution

Define the supported public Gateway HTTP route set, then add a generated or committed OpenAPI artifact with docs and a drift check; optionally serve the artifact behind Gateway auth at `/openapi.json` while keeping WebSocket protocol JSON Schema and config JSON Schema separate unless maintainers intentionally model them in OpenAPI.
