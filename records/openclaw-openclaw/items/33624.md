Keep open. ## What This Changes

Expose or generate a Swagger/OpenAPI schema for OpenClaw endpoints so custom API clients can be generated.

## Best Possible Solution

Define a versioned OpenAPI artifact only for explicitly supported HTTP endpoints with clear auth/security notes, and handle full WebSocket RPC client generation as a separate Gateway protocol/package policy decision.

## Reproduction Assessment

Not applicable; this is a feature request for a new generated API artifact. Current-main and v2026.6.5 inspection confirm there is no OpenAPI/Swagger artifact to reproduce against.

## Solution Assessment

No, not as one unspecified Swagger schema over every endpoint. The maintainable path is to choose a narrow supported HTTP contract first, then separately decide whether the private Gateway protocol becomes a supported external client/schema surface.
