## Summary

Keep open. Current main still documents and routes Gateway HTTP APIs, but there is no OpenAPI/Swagger artifact or discovery route, and publishing one would define a public operator API contract that needs maintainer API-boundary and security judgment.

## What This Changes

Request to add a generated OpenAPI/Swagger schema for existing OpenClaw endpoints so users can generate custom clients.

## Best Possible Solution

Define the stable public Gateway API surface, add a checked OpenAPI artifact and docs for that surface, and only serve runtime discovery if maintainers choose an authenticated endpoint.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main give high-confidence absence proof for a Gateway OpenAPI/Swagger artifact or discovery route.

## Solution Assessment

Unclear until maintainers choose the public API boundary. The maintainable path should model stable Gateway HTTP routes deliberately, not blindly expose every handler or mix config/RPC/plugin surfaces into one schema.
