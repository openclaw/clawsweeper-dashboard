## Summary

Keep open. Current main still lacks an OpenAPI/Swagger artifact or discovery route; the existing Gateway docs, App SDK, and TypeBox protocol schema are related but do not satisfy language-neutral client generation, and the remaining choice needs maintainer API/auth scoping.

## What This Changes

Request to publish or serve an OpenAPI/Swagger schema so users can generate clients for OpenClaw programmatic APIs.

## Best Possible Solution

Define the stable public Gateway API boundary first, then publish a versioned, auth-aware OpenAPI or equivalent schema with deterministic generation, docs, and contract tests.

## Reproduction Assessment

Not applicable; this is a feature request, not broken existing behavior. Source, docs, package, and live issue checks give high-confidence absence proof for an OpenAPI/Swagger artifact or discovery route on current main.

## Solution Assessment

Unclear; a schema is a reasonable goal, but auto-exposing every handler or plugin route before choosing the stable public API boundary would be risky. The maintainable path is an explicit scoped contract with auth/redaction rules and generated checks.
