## Summary

Keep open. Current main still lacks a Gateway OpenAPI/Swagger artifact or discovery route, and adding one would publish a public API contract that needs maintainer API-boundary and security scoping.

## What This Changes

The issue requests an OpenAPI/Swagger schema for OpenClaw APIs so users can generate programmatic clients.

## Best Possible Solution

Define the stable public Gateway API surface, add a checked OpenAPI artifact and docs for that surface, and only serve runtime discovery if maintainers choose an authenticated endpoint.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main provide high-confidence absence proof for a Gateway OpenAPI/Swagger artifact or discovery route.

## Solution Assessment

Unclear until maintainers choose the public API boundary. The maintainable path should model stable Gateway HTTP routes deliberately, not blindly expose every handler or mix RPC/plugin/internal operator surfaces into one schema.
