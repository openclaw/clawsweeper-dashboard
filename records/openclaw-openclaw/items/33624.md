## Summary

Keep open. Current main still lacks a Gateway OpenAPI/Swagger artifact, generator, Swagger UI, or `/openapi.json` route, and adding one would bless a public operator API contract that needs maintainer product and security judgment.

## What This Changes

The issue requests a generated OpenAPI/Swagger schema in the repository so custom clients can interact with OpenClaw programmatically.

## Best Possible Solution

Define the stable public Gateway API boundary, add a checked OpenAPI artifact and docs for that boundary, and only serve runtime discovery if maintainers choose an authenticated endpoint.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main give high-confidence absence proof for a Gateway OpenAPI/Swagger artifact or discovery route.

## Solution Assessment

Unclear until maintainers choose the public API boundary. The request is reasonable, but the maintainable solution should model stable Gateway HTTP routes rather than blindly exposing every handler or mixing config/WebSocket schemas into one spec.
