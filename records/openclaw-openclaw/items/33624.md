Keep this open: current main and the latest release expose documented Gateway HTTP surfaces, but neither publishes an OpenAPI/Swagger schema for client generation. The request remains useful, but it needs a maintainer decision on which Gateway, operator, and plugin-owned routes are stable enough to describe as a public API contract.

The issue requests a published OpenAPI/Swagger schema for OpenClaw APIs so users can generate custom programmatic clients.

## Best Possible Solution

Maintainers should first define the stable public Gateway HTTP contract, then publish a narrow versioned OpenAPI artifact with tests and security docs for only those approved surfaces.
