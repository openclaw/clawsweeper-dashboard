Keep open. Current main and the latest release still lack an OpenAPI/Swagger artifact or schema-serving route for the Gateway/API surface, while the adjacent App SDK and admin HTTP RPC plugin do not provide the requested language-neutral client-generation contract.

## What This Changes

Request to expose or publish an OpenAPI/Swagger schema for OpenClaw so custom API clients can be generated.

## Best Possible Solution

Define the stable public Gateway API boundary first, then publish a versioned, auth-aware OpenAPI or equivalent schema with deterministic generation, docs, and contract tests.

## Solution Assessment

Unclear; OpenAPI is a reasonable target, but auto-exposing every handler, RPC method, or plugin route before choosing the stable public API boundary would be risky. The safer path is an explicitly scoped schema contract with auth/redaction rules and generated drift checks.
