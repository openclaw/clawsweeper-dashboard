## Summary

Keep open. Current main still lacks a Feishu-owned doctor/quarantine path for stale Feishu local state and Feishu-scoped session artifacts, and open PR #74397 is the linked closing implementation candidate.

## What This Changes

The issue requests recovery and guardrails for stale or corrupt Feishu local runtime state and Feishu-scoped channel session artifacts that can leave the gateway in a sustained high-CPU loop.

## Best Possible Solution

Land PR #74397 or an equivalent Feishu-owned doctor repair that archives/rebuilds only Feishu local runtime state and Feishu direct session artifacts while preserving credentials, non-Feishu sessions, and ACP binding sessions.

## Reproduction Assessment

Unclear. The report includes strong field evidence, CPU samples, relevant state paths, logs, and a successful manual workaround, but the private corrupt state artifacts needed for deterministic current-main reproduction are unavailable.

## Solution Assessment

Yes. A Feishu-owned doctor/quarantine repair through the existing channel doctor seam is the narrowest maintainable direction because the affected data is Feishu-local state plus Feishu-scoped session metadata, not a core startup migration.
