## Summary

Keep open. Current main still lacks a Feishu-owned doctor/quarantine path for corrupt Feishu local state or direct session artifacts, and open PR #74397 is already the explicit implementation candidate for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests Feishu-scoped validation, quarantine, or doctor repair so stale or corrupt Feishu runtime state and direct session artifacts cannot keep the gateway hot-looping.

## Best Possible Solution

Land #74397 or an equivalent Feishu-owned doctor repair that detects repairable Feishu-local state, archives/rebuilds only Feishu runtime/dedup state and direct Feishu channel session artifacts, preserves credentials plus non-Feishu and ACP binding sessions, and adds focused Feishu doctor tests.

## Reproduction Assessment

Unclear. The report has strong field evidence, CPU samples, logs, and a successful workaround, but the private stale/corrupt Feishu state is not available for a deterministic reproduction against current main.

## Solution Assessment

Yes. A Feishu-owned doctor adapter is the narrowest maintainable direction because the affected state is Feishu-local and Feishu-scoped session metadata, while the core channel doctor seam already supports plugin-owned repair sequences.
