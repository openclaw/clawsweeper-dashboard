## Summary

Keep open. Current main still lacks a Feishu-owned doctor repair or quarantine path for stale Feishu local JSON state and Feishu-scoped direct session artifacts. The reporter already opened PR #74397 with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands. https://github.com/openclaw/openclaw/issues/74237

## What This Changes

The issue requests a Feishu-safe recovery path so stale or corrupt Feishu runtime/session state cannot keep the gateway in a sustained high-CPU loop.

## Best Possible Solution

Land PR #74397 or an equivalent Feishu-owned doctor repair that detects repairable Feishu local JSON/session state, archives and rebuilds only Feishu runtime/dedup state plus Feishu direct session artifacts, preserves Feishu App ID/secret config, preserves non-Feishu and ACP binding sessions, and adds focused Feishu doctor tests.

## Reproduction Assessment

Unclear. The report has strong field evidence, CPU samples, logs, and a successful manual workaround, but the private stale/corrupt state files are not available for deterministic reproduction against current main.

## Solution Assessment

Yes. A Feishu-owned doctor repair through the existing channel doctor adapter is the narrowest maintainable solution because the affected state is Feishu-local and Feishu-scoped session metadata, not a core startup migration.
