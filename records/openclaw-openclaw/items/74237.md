## Summary

Keep open. The report has credible field evidence, current main still lacks a Feishu-owned doctor repair/quarantine path for stale Feishu runtime state and direct session artifacts, and open PR #74397 is the explicit closing implementation candidate.

## What This Changes

The issue requests that stale or corrupt Feishu channel runtime/session state stop being able to hot-loop the gateway, preferably through scoped validation, quarantine, and doctor repair.

## Best Possible Solution

Review and land #74397, or an equivalent Feishu-owned doctor repair, so doctor can detect repairable Feishu local JSON/session state, archive and rebuild only Feishu runtime/dedup plus Feishu direct session artifacts, preserve configured App ID/secret values, and keep non-Feishu and ACP binding sessions intact.

## Reproduction Assessment

Unclear. The report gives high-signal field evidence and a successful manual workaround, but without sanitized stale state files there is not a high-confidence deterministic hot-loop reproduction against current main.

## Solution Assessment

Yes. A Feishu-owned doctor repair through the existing ChannelDoctorAdapter seam is the narrowest maintainable direction; a core startup migration would be broader than needed for channel-local state.
