## Summary

Keep open. Current main still lacks a Feishu-owned doctor/quarantine path for stale Feishu local state and Feishu-scoped session artifacts, and open PR #74397 is the linked closing implementation candidate.

## What This Changes

Prevent stale or corrupt Feishu runtime/dedup state and Feishu direct session artifacts from driving sustained gateway CPU, with a scoped repair or quarantine path.

## Best Possible Solution

Land #74397 or an equivalent Feishu-owned doctor repair that archives/rebuilds only Feishu local runtime state and Feishu direct session artifacts while preserving credentials, non-Feishu sessions, and ACP binding sessions.

## Reproduction Assessment

Unclear. The report includes strong field evidence, CPU samples, logs, state paths, and a successful manual workaround, but the private stale/corrupt state files are not available for deterministic reproduction against current main.

## Solution Assessment

Yes. A Feishu-owned doctor repair through the existing channel doctor adapter is the narrowest maintainable direction because the affected data is Feishu-local state plus Feishu-scoped session metadata, not a core startup migration.
