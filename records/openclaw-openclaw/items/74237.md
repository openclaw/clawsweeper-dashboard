## Summary

Keep open. Current main still lacks a Feishu-owned doctor/quarantine path for the implicated local state, and #74397 is an open, unmerged closing PR for the same recovery boundary.

## What This Changes

Request a Feishu-owned recovery path so stale or corrupt Feishu runtime state and direct channel session artifacts cannot leave the gateway hot-looping.

## Best Possible Solution

Land #74397 or an equivalent Feishu-owned doctor repair that archives only Feishu-local runtime state and direct Feishu session artifacts while preserving credentials, non-Feishu sessions, and ACP binding sessions.

## Reproduction Assessment

Unclear. The field report has strong CPU samples, logs, state paths, and a successful cleanup workaround, but the private stale/corrupt state files are not available for a deterministic current-main reproduction.

## Solution Assessment

Yes. A Feishu-owned repair through the existing channel doctor seam is the narrowest maintainable direction because the affected state is channel-local rather than a generic startup migration.
