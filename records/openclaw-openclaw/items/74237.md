## Summary

Keep open. Current main still does not contain a Feishu-owned doctor repair or quarantine path for the reported stale Feishu runtime/session state, and open PR #74397 is the explicit unmerged implementation candidate linked by the reporter.

## What This Changes

Prevent stale or corrupt Feishu channel runtime/session state from hot-looping the gateway by validating, quarantining, or repairing Feishu-local state and direct session artifacts.

## Best Possible Solution

Land #74397 or an equivalent Feishu-owned doctor repair that detects repairable Feishu local JSON/session state, archives and rebuilds only Feishu runtime/dedup state plus Feishu direct session artifacts, preserves Feishu App ID/secret config and non-Feishu/ACP binding sessions, and adds focused Feishu doctor tests.

## Reproduction Assessment

Unclear. The report provides strong field evidence, CPU samples, logs, implicated state paths, and a successful manual workaround, but the private stale/corrupt state files are not available for deterministic reproduction against current main.

## Solution Assessment

Yes. A Feishu-owned doctor repair through the existing channel doctor adapter is the narrowest maintainable solution because the affected data is Feishu-local and Feishu-scoped session metadata, not a core startup migration.
