## What This Changes

The issue reports Feishu stale or corrupt local runtime/session state causing gateway high CPU and requests scoped validation, quarantine, or doctor repair for Feishu state artifacts.

## Best Possible Solution

Land #74397 or an equivalent Feishu-owned doctor repair that archives/rebuilds only Feishu local runtime state and Feishu direct session artifacts while preserving credentials, non-Feishu sessions, and ACP binding sessions.
