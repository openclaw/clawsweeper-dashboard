## Summary

Keep open: current main still has the direct workflow-output interpolation, while this PR is a narrow CI security hardening change with no concrete correctness or supply-chain regression found in the diff.

## What This Changes

The PR hoists Kova report output paths in `.github/workflows/openclaw-performance.yml` into step-level environment variables and uses quoted shell variables in the clawgrit publish script.

## Best Possible Solution

Land this env-hoisting hardening after exact-head CI and maintainer workflow review, while leaving the broader zizmor cleanup tracked by #68428.

## Reproduction Assessment

Yes, at source level: current main emits `report_json` and `report_md` through `GITHUB_OUTPUT` and then interpolates them directly inside the later bash script. I did not run zizmor in this read-only review.

## Solution Assessment

Yes. Moving the dynamic step outputs into `env:` and referencing quoted shell variables is the narrowest maintainable fix, and the hunk does not change permissions, artifact paths, secrets, dependencies, or publish destinations.
