## What This Changes

The PR updates `.github/workflows/openclaw-performance.yml` so the `Publish to clawgrit reports` step reads Kova report paths from `REPORT_JSON` and `REPORT_MD` environment variables instead of inline GitHub Actions expressions.

## Best Possible Solution

Land the narrow workflow hardening after required exact-head checks complete, while leaving broader zizmor cleanup tracked in the related security issue.

## Reproduction Assessment

I did not need to establish a separate runtime failure because the CI security audit path and source pattern match the reported zizmor class.

## Solution Assessment

Yes: hoisting the two dynamic step outputs into `env:` and using quoted shell variables is the narrowest maintainable mitigation for this GitHub Actions template-injection pattern. The added non-empty guard for the optional Markdown report preserves behavior while avoiding an empty-path file test.
