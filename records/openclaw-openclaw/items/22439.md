Keep this PR open. Current main still lacks `bootstrapTier`, the contributor branch is active and paired with an open proposal, and the latest PR head still has prompt-context regressions that should be fixed during maintainer API review rather than closed as cleanup.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier-aware workspace bootstrap filtering, bootstrap-extra-files tier wiring, generated config metadata/help/types, and focused tests.

## Best Possible Solution

Keep the PR open with #22438 for maintainer API review. If accepted, revise this branch so unset config preserves legacy hook extras, explicit tiers are source-aware, generated schema/help/docs/tests describe one consistent contract, and the extra-bootstrap overlap with #74735 is reconciled.

## Reproduction Assessment

Yes. Static inspection is enough for the review findings: current main keeps main-session hook extras, while PR head drops hook files under default `standard` and can admit hook-sourced allowlisted files under explicit `minimal`.

## Solution Assessment

No. Tiered bootstrap control may be useful, but this patch is not yet the best implementation because it conflates unset legacy behavior with explicit `standard` and leaves explicit `minimal` source-blind.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve legacy hook extras without a tier:** `src/agents/workspace.ts:753-754`
  - body: The PR says an unset tier is backward compatible, but the hook passes `undefined`, `resolveBootstrapTier` maps that to `standard`, and this predicate excludes every `source: "hook"` file. Existing main sessions using `bootstrap-extra-files` would stop receiving configured extras unless users discover `bootstrapTier: "full"`; keep the unset legacy path separate from explicit standard or document an intentional migration.
  - confidence: 0.9
- **[P2] Filter hook extras out of explicit minimal:** `src/agents/workspace.ts:745-746`
  - body: The `minimal` branch filters only by basename, so `bootstrapTier: "minimal"` can still inject hook-loaded `AGENTS.md`, `SOUL.md`, `IDENTITY.md`, `TOOLS.md`, or `USER.md`. Since the help text describes `full` as the tier that includes extra patterns, explicit minimal should be source-aware while any no-override legacy behavior is handled separately.
  - confidence: 0.86
