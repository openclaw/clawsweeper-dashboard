## Summary

Keep open: current main still has only binary bootstrap filtering, and the open paired proposal plus maintainer-side review indicate this needs product/API review; the PR also still has actionable correctness and low-severity prompt-exposure issues before merge.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier-aware workspace bootstrap filtering, bootstrap-extra-files hook wiring, generated config metadata/help/types, and focused tier tests.

## Best Possible Solution

Keep the PR open with #22438 for maintainer API review; if accepted, revise it so unset config preserves legacy behavior, explicit tiers are source-aware, docs/generated metadata/changelog/tests agree, and related extra-bootstrap work is reconciled.

## Reproduction Assessment

Yes. Static inspection is enough for the PR findings: current main keeps hook extras in normal sessions, while the PR head maps an unset tier to `standard` and excludes hook-sourced files; the explicit `minimal` branch remains basename-only.

## Solution Assessment

No. Tiered bootstrap control may be useful, but this patch is not the safest implementation as-is because it conflates unset legacy behavior with explicit `standard` and leaves explicit `minimal` source-blind; a safer design separates legacy no-override behavior from configured tier policy.
