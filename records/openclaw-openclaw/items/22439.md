- **[P1] Preserve default main-session extras:** `src/hooks/bundled/bootstrap-extra-files/handler.ts:59-64`
  - body: Current main has a covered behavior where `bootstrap-extra-files` appends configured extras to a normal main session with no additional config. Here the hook passes an undefined tier override, so `resolveBootstrapTier()` chooses `standard`, and the subsequent filter removes every `source: "hook"` file unless the user discovers and sets `bootstrapTier: "full"`; existing hook configs would silently stop contributing prompt context after upgrade.
  - confidence: 0.92
- **[P1] Keep subagent defaults on the current allowlist:** `src/agents/workspace.ts:746-749`
  - body: Current main filters subagent sessions to `AGENTS.md` and `TOOLS.md`, but this branch maps every subagent with no override to `minimal`, whose allowlist includes `SOUL.md`, `IDENTITY.md`, and `USER.md`. That broadens worker prompt context and user/persona exposure for existing installs without an opt-in or migration, so the default should stay compatible and the broader tier should be explicit.
  - confidence: 0.9
- **[P2] Document the bootstrapTier config contract:** `src/config/zod-schema.agent-defaults.ts:105-107`
  - body: This adds a new public `agents.defaults.bootstrapTier` setting, but the agent config and hook docs do not explain the values, defaults, interaction with `bootstrap-extra-files`, or upgrade behavior. Config/help metadata is not enough for a user-facing prompt-context API; the docs need to be updated before the setting ships.
  - confidence: 0.84
