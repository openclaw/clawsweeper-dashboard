## Best Possible Solution

Keep the PR open with #22438 for maintainer API review; if accepted, revise it so unset config preserves legacy behavior, explicit tiers are source-aware, changelog/docs/tests agree, and related bootstrap-extra work is reconciled.

## Reproduction Assessment

Yes. Static inspection is enough for the PR findings: current main preserves hook extras in ordinary sessions, while the PR head maps an unset tier to `standard` and excludes hook-sourced files; explicit `minimal` remains basename-only.

## Solution Assessment

No. Tiered bootstrap control may be useful, but this patch is not the safest implementation as-is because it conflates unset legacy behavior with explicit `standard` and leaves explicit `minimal` source-blind; a safer design separates legacy no-override behavior from configured tier policy.
