## Summary

Keep this PR open. Current main does not implement `bootstrapTier`, and the PR remains an active contributor implementation paired with the open proposal #22438, but the latest head still has merge-blocking behavior issues around legacy `bootstrap-extra-files` compatibility and explicit minimal tier filtering.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier-aware workspace bootstrap filtering, bootstrap-extra-files tier wiring, generated config metadata/help/types, and focused tests.

## Best Possible Solution

Keep the PR open with #22438 for maintainer API review. If accepted, revise the branch so unset config preserves legacy hook extras, explicit tiers are source-aware, generated schema/help/docs/changelog/tests describe one consistent contract, and the extra-bootstrap overlap with #74735 is reconciled.

## Reproduction Assessment

Yes. Static inspection is enough for the review findings: current main keeps normal-session hook extras, while PR head maps an unset tier to `standard` and drops hook files; PR head also lets explicit `minimal` admit hook files with allowlisted basenames.

## Solution Assessment

No. Tiered bootstrap control may be useful, but this patch is not yet the best implementation because it conflates unset legacy behavior with explicit `standard` and leaves explicit `minimal` source-blind; a safer design would preserve the no-override path separately from explicit tier policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve legacy hook extras without a tier:** `src/agents/workspace.ts:753-754`
  - body: The PR says an unset tier is backward compatible, but the hook passes `undefined`, `resolveBootstrapTier` maps that to `standard`, and this predicate excludes every `source: "hook"` file. Existing normal sessions using `bootstrap-extra-files` would stop receiving configured extras unless users discover `bootstrapTier: "full"`; keep the unset legacy path separate from explicit `standard`, or document an intentional migration.
  - confidence: 0.91

Since the help text describes `full` as the tier that includes extra patterns, explicit minimal should be source-aware while any no-override legacy behavior is handled separately.
