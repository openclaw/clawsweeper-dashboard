## What This Changes

The PR adds `agents.defaults.bootstrapTier` with `minimal`/`standard`/`full` bootstrap filtering, wires it through bootstrap resolution and the `bootstrap-extra-files` hook, updates config metadata/help, and adds tier tests.

## Best Possible Solution

Choose one coherent bootstrap context API, preserve existing defaults unless users explicitly opt into narrower tiers, document the contract, and cover main, subagent, cron, lightweight, extra-file, and per-agent interactions with focused tests and redacted real setup proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows current main keeps hook-loaded extra bootstrap files for normal/main sessions, while the PR head drops `source: "hook"` files when no tier is configured. The underlying request is a feature/config proposal, not a bug reproduction case.

## Solution Assessment

No, not as-is. The tiered config direction is plausible, but the patch needs to preserve default extra-file behavior, provide real behavior proof, and wait for maintainer agreement on overlapping bootstrap context semantics.
