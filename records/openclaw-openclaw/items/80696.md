## Summary

Keep open: current main and the latest release still ship the reported `RestartSec=5` systemd default and audit it as preferred, while the documented exit-78 protection still depends on an existing gateway reaching `/healthz` first.

## What This Changes

The issue asks OpenClaw to change the shipped Linux systemd gateway restart timing or lock behavior so short restarts do not race the `/healthz`-based exit-78 loop breaker during warmup.

## Best Possible Solution

Raise the shipped systemd restart delay and align doctor/audit checks, docs, and regression tests with a delay that gives normal gateway startup time to expose `/healthz`, while preserving the existing exit-78 contract.

## Reproduction Assessment

No live high-confidence WSL2/systemd reproduction was established. Source inspection does show the reported path: current units use `RestartSec=5`, exit 78 requires a healthy `/healthz`, and the lock can classify a pre-bind owner as dead while startup is still warming.

## Solution Assessment

Yes: the narrowest maintainable first fix is to update the systemd restart timing contract and its audit/docs/tests. A `Type=notify` redesign would be broader and should be a separate maintainer choice if needed.
