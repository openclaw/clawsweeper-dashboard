## Summary

Keep open: current main and the latest release still ship the reported systemd timing contract, and the exit-78 duplicate-start protection still depends on an existing gateway becoming healthy on `/healthz`. The overlapping Linux lifecycle issue at https://github.com/openclaw/openclaw/issues/75398 remains open, but it does not supersede this narrower `RestartSec` warmup race.

## What This Changes

The issue asks OpenClaw to adjust Linux systemd gateway restart timing or duplicate-gateway handling so a slow-starting gateway can become healthy before another supervised restart kills it.

## Best Possible Solution

Align the systemd gateway lifecycle contract, docs, audit checks, and regression coverage so supervised restarts give a normal gateway startup enough time to expose `/healthz` before duplicate-start logic can classify it as stale.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only pass. Source inspection does show the reported path on current main: `RestartSec=5`, exit 78 requires a healthy `/healthz`, and startup does substantial work before the HTTP listener is bound.

## Solution Assessment

Yes for a focused repair that updates the systemd restart timing contract and its audit/docs/tests. A `Type=notify` redesign would be broader and should not be the first automated fix unless maintainers choose that direction.
