## What This Changes

The issue asks OpenClaw to prevent systemd gateway restart cascades when `RestartSec=5` fires before the existing gateway becomes healthy enough for the exit-78 lock guard.

## Best Possible Solution

Land a coordinated gateway/systemd lifecycle fix that either gives normal startup enough time to expose `/healthz` or makes the lock/startup path health-aware earlier, with systemd unit, audit, docs, and regression tests kept in sync.

## Solution Assessment
