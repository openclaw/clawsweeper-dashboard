## Summary

Keep open: current main and the latest release still render, document, and audit the gateway systemd unit around `RestartSec=5`, while the exit-78 loop protection only fires after an existing gateway answers `/healthz`. Source inspection still supports the reported race, so no conservative close reason applies.

## What This Changes

The issue asks OpenClaw to prevent a systemd restart cascade where `RestartSec=5` can respawn the gateway before the previous instance becomes `/healthz`-healthy and can exit 78.

## Best Possible Solution

Update the canonical systemd restart timing, doctor/audit checks, docs, changelog, and regression tests so normal gateway warmup can expose `/healthz` before the next supervised respawn while preserving the existing exit-78 contract.

## Reproduction Assessment

No live high-confidence WSL2/systemd reproduction was established in this review. Source inspection gives a clear targeted path: current units use `RestartSec=5`, exit 78 requires a healthy `/healthz`, and startup can spend time before the listener is bound.

## Solution Assessment

Yes: a focused repair that raises the canonical systemd delay and aligns audit/docs/tests is the narrowest maintainable first fix. A `Type=notify` readiness redesign can remain a later maintainer choice if the tuned delay proves insufficient.
