## Summary

Keep open: the report is recent, concrete, and current main still ships the same systemd timing/lock behavior that can race the documented exit-78 protection during slow gateway warmup.

## What This Changes

The issue asks OpenClaw to prevent `Restart=always` plus `RestartSec=5` from racing gateway warmup and defeating the systemd exit-78 duplicate-gateway guard.

## Best Possible Solution

Align the shipped systemd defaults, doctor/audit checks, docs, and regression tests with a restart delay that gives normal gateway startup enough time to expose `/healthz`, while preserving the existing exit-78 contract.

## Reproduction Assessment

No live high-confidence reproduction was established in this review. Source inspection does show the reported path: current units use `RestartSec=5`, exit 78 requires a healthy `/healthz`, and startup can spend time before the HTTP listener is bound.

## Solution Assessment

Yes for a focused repair that updates the systemd restart timing contract and its audit/docs/tests. A `Type=notify` redesign would be broader and should not be the first automated fix unless maintainers choose that direction.
