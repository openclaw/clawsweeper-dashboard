## Summary

Keep open: current main has the Slack progress-draft lifecycle, fallback text, and a rich Block Kit renderer, but it still renders progress entries as generic section fields rather than Slack `plan` or `task_card` blocks. The related merged work supplies prerequisites, not the requested task-card presentation surface.

## What This Changes

The issue requests rendering existing Slack progress draft status content as Slack plan/task-card Block Kit while preserving the current progress/final message lifecycle.

## Best Possible Solution

Add a Slack-owned plan/task-card progress renderer at the existing rich progress-draft hook, keep the plain-text fallback, and leave thread routing, lifecycle, and final-answer selection unchanged.
