## Summary

This is a valid, focused Slack UX enhancement and current main does not yet implement the requested plan/task-card progress surface. OpenClaw already has Slack progress drafts, fallback text, and a rich Block Kit renderer, but that renderer currently emits generic section fields rather than Slack `plan` or `task_card` blocks.

## What This Changes

Render existing Slack progress-draft entries as a Slack plan/task-card Block Kit surface while preserving the one-progress-message plus final-reply lifecycle.

## Best Possible Solution

Add a Slack-owned plan/task-card progress renderer at the existing rich progress-draft hook, keep the plain-text fallback, and leave the progress/final lifecycle unchanged.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection confirms the current Slack progress renderer path and the missing plan/task-card mapping.

## Solution Assessment

Yes, the narrowest maintainable solution is to reuse the existing Slack progress-draft renderer and fallback-text path rather than changing channel lifecycle, routing, or final-answer selection.
