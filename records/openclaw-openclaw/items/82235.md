## Summary

Keep open: current main already has Slack rich progress drafts, but that renderer emits ordinary `section` blocks with fields, not the Slack `plan`/`task_card` surface requested in the issue clarification. The related broad Slack Block Kit/progress work is useful context, but this narrower UX change remains unimplemented and needs maintainer product/API judgment.

## What This Changes

Render Slack progress drafts as Slack task-card or plan-style Block Kit while preserving the existing one-progress-message plus separate final-reply lifecycle.

## Best Possible Solution

Decide whether Slack task-card rendering should replace the existing `render: "rich"` section-field renderer, become a new Slack-specific render mode, or stay as user-authored Block Kit, then implement that narrow Slack renderer with fallback text and live Slack proof.

## Reproduction Assessment

Not applicable: this is a feature request for a new Slack presentation surface, not a broken existing contract. Source inspection does confirm current main renders rich progress as section/field blocks rather than Slack plan/task-card blocks.

## Solution Assessment

Unclear until maintainers choose the UX/API boundary. The narrow implementation would likely live in the Slack progress renderer, but maintainers should first decide whether to change `render: "rich"` semantics or add a separate Slack-specific mode.
