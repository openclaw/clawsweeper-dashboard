The PR adds a `script` CLI banner tagline mode backed by a user-configured JavaScript file, with config schema/help/docs updates and async startup/help integration changes.

## Best Possible Solution

Make this an explicitly approved CLI personalization feature with current startup/help integration, bounded execution semantics, schema/help/docs/tests, and redacted real CLI output proof.

No, not as currently proposed. The feature should be rebased through `applyCliExecutionStartupPresentation` and the help renderer, then explicitly approved as a startup code-execution surface before merge.
