## Summary

Keep open. Current main still does not implement script-backed CLI taglines, but this PR is a feature/config addition with unresolved startup JavaScript execution, generated-schema drift, and stale startup-path integration concerns.

## What This Changes

The PR adds an opt-in `script` tagline mode, a `cli.banner.taglineScriptFile` config key, dynamic JS default-export loading for CLI banner taglines, async banner/help startup plumbing, tests, and docs/schema metadata.

## Best Possible Solution

Either accept script taglines with isolated abortable execution plus regenerated schema, changelog, docs, and tests on the current startup path, or choose a safer static custom-tagline config instead.

## Reproduction Assessment

Yes. Current main source is enough to verify the missing feature: CLI types, parser, strict Zod schema, generated schema, and docs expose only `random`, `default`, and `off`.

## Solution Assessment

No as currently proposed. The feature direction may be viable after maintainer approval, but this patch is stale and executes user JavaScript on the main startup event loop where its timeout cannot stop synchronous blocking work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Regenerate the public config schema:** `src/config/zod-schema.ts:334-340`
  - body: This updates the handwritten Zod schema/help for `taglineScriptFile`, but the generated base schema served by config-schema consumers is not updated. `openclaw config schema` and UI metadata would still expose only `random/default/off` and omit the new field, and the schema generation check should fail.
  - confidence: 0.86
- **[P2] Run tagline scripts in an abortable boundary:** `src/cli/tagline.ts:276-305`
  - body: The timeout races code that still runs on the main Node event loop. A script with blocking top-level work or a hung sync export, including the documented `execSync` style, prevents the timer from firing and can hang every CLI startup/help path that awaits banner priming.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces a config-selected JavaScript execution path during CLI startup and needs explicit security/product acceptance before merge.

Concerns:

- **[medium] Config-selected JavaScript runs at CLI startup:** `src/cli/tagline.ts:284`
  - body: `taglineScriptFile` is dynamically imported and its default export is invoked for banner/help rendering. Even when intended for local operator control, a compromised or unexpected config edit could execute code before ordinary CLI actions.
  - confidence: 0.82
- **[medium] Timeout cannot stop synchronous script work:** `src/cli/tagline.ts:299`
  - body: The `Promise.race` timeout cannot interrupt synchronous module evaluation or a blocking default-export function on the main event loop, leaving a straightforward local denial-of-service path for normal CLI startup.
  - confidence: 0.84
