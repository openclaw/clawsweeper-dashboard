## Summary

Keep open; current main still lacks script-backed taglines, but this PR is not merge-ready because the async banner boundary is not wired through current startup/help paths, real behavior proof is missing, and startup JavaScript execution needs maintainer approval.

## What This Changes

The branch adds a script-backed CLI banner tagline mode with config schema/help docs, startup/help integration changes, and banner/tagline tests.

## Best Possible Solution

Maintainers should first decide whether core should execute user JavaScript at startup; if accepted, the implementation should be rebased into the current shared startup/help contracts with bounded execution semantics, focused tests, and redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main lacks the `script` enum/config key, and the PR head's async script-resolution path is not integrated with current shared startup and help rendering contracts.

## Solution Assessment

No, not as currently proposed. The narrower maintainable path is a rebased implementation through `applyCliExecutionStartupPresentation` and the help/precomputed-help contracts after explicit approval for startup JavaScript execution.
