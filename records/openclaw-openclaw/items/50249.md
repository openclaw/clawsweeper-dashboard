  - body: This PR makes `emitCliBanner` async, but current main's shared startup helper still calls it without `await`. After rebase, script-mode startup can race command/help output or skip the configured tagline in those paths unless the shared presentation path is updated too.
  - confidence: 0.86

  - body: `formatCliBannerLine` only reads `options.resolvedTagline` or the module cache for script mode, while current help rendering can call it with `mode: "default"`.   - confidence: 0.84
- **[P2] Run tagline scripts behind an abortable boundary:** `src/cli/tagline.ts:287-293`
  - body: The timeout cannot interrupt synchronous top-level module work or a blocking default-export function because both run in the main CLI process. A bad configured script can still hang every startup before users can recover through `openclaw config`, so this needs an abortable boundary or an explicit maintainer-approved contract.
  - confidence: 0.83
