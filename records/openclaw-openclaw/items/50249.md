## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Run tagline scripts outside the CLI event loop:** `src/cli/tagline.ts:287-293`
  - body: The timeout races the same event loop that runs the module import and default export. A script with synchronous top-level code or a blocking function, including the documented `execSync` style, can still hang every CLI startup before the timer can fire; put this behind a worker/child-process boundary with a hard kill or keep script mode out of startup.
  - confidence: 0.88
- **[P2] Gate script resolution through startup policy:** `src/cli/banner.ts:186-190`
  - body: This branch resolves the script when a raw `-h`/`--help` token is present before the non-TTY return. Option values or suppressed banner paths can therefore execute user JS even when no banner is emitted; use the shared invocation/startup policy to decide that a banner will render before resolving the script.
  - confidence: 0.78
- **[P2] Keep root help on the configless fast path:** `src/entry.ts:207-208`
  - body: The root-help fast path primes `emitCliBanner` before rendering help, which can load config and run tagline JS. Current main keeps help banner rendering configless/default and uses precomputed help metadata, so a rebase needs to preserve that startup contract or get explicit maintainer approval.
  - confidence: 0.75
