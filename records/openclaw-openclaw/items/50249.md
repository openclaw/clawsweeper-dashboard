## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Await async banner emission in shared startup paths:** `src/cli/banner.ts:170`
  - body: This PR makes `emitCliBanner` async, but current main's shared startup helper and gateway-run fast path still call it without `await`. After rebase, script-mode startup can race command/help output or skip the configured tagline in those paths.
  - confidence: 0.86
- **[P2] Route script taglines through every help renderer:** `src/cli/banner.ts:85-89`
  - body: `formatCliBannerLine` only reads `options.resolvedTagline` or the module cache for script mode, while current help rendering can call it with `mode: "default"` or bypass `emitCliBanner`. Script taglines will still render as default or empty in common help flows unless resolving the tagline becomes part of the help rendering contract.
  - confidence: 0.84
- **[P2] Run tagline scripts behind a real timeout boundary:** `src/cli/tagline.ts:287-293`
  - body: The timeout cannot interrupt synchronous top-level module work or a blocking default-export function because both run in the main CLI process. A bad configured script can still hang every startup before users can recover through `openclaw config`, so this needs an abortable boundary or an explicit maintainer-approved contract.
  - confidence: 0.83
