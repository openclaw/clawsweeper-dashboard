## What This Changes

The PR adds a `script` CLI banner tagline mode, `taglineScriptFile` config/docs/schema support, JS module tagline resolution, startup/help cache priming, and focused banner/tagline tests.

## Best Possible Solution

Maintainers should first decide whether core should support config-driven startup JavaScript; if accepted, rebase the feature through the current startup presentation and help-rendering contracts with bounded execution semantics, focused tests, and redacted real CLI proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. The safer path is a rebased implementation through `applyCliExecutionStartupPresentation` and the help/precomputed-help contracts after explicit approval for startup JavaScript execution.
