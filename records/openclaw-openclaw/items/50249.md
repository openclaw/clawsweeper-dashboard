## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Run tagline scripts in an abortable boundary:** `src/cli/tagline.ts:303-305`
  - body: The timeout here does not stop `await import(...)` or synchronous default-export work, so a script with top-level await, a CPU loop, or a hung sync call can still hang the main CLI process. Move execution to a worker or child process with a hard kill, or use a non-executing extension point.
  - confidence: 0.91
- **[P2] Resolve script taglines through shared help rendering:** `src/cli/banner.ts:85-88`
  - body: `formatCliBannerLine` only reads `cachedScriptTagline` when no resolved tagline is passed, so help paths that do not prime that cache, or that force `mode: "default"`, still cannot render a configured script tagline consistently. Wire script resolution through the shared startup/help policy instead of module cache state.
  - confidence: 0.87
- **[P2] Avoid raw argv scans before executing scripts:** `src/cli/banner.ts:51-52`
  - body: `hasHelpFlag` treats any argv token equal to `--help` or `-h` as help intent; if that token is an option value in a non-TTY command, script mode can import and run user JS even though no banner is emitted. Use the parsed invocation/help policy before deciding to execute configured code.
  - confidence: 0.78
