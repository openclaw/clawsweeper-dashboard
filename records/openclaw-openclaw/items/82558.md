## Summary

Keep open. Current main still turns a detail-less `response.failed` event into the generic fallback before embedded observation can preserve event metadata; the prior related fixes only classify that generic string as `no_error_details`, so the remaining diagnostics/preservation problem is real and not covered by the closed items.

## What This Changes

The issue requests redacted structured diagnostics and distinct failover handling for OpenAI Responses `response.failed` events that lack `error` and `incomplete_details`.

## Best Possible Solution

Preserve the existing `no_error_details` failover split, and add a narrow Responses transport/observation fix that carries redacted `response.failed` event/provider diagnostics through embedded logs and failover decisions with regression coverage.

## Solution Assessment
