## Summary

Keep this PR open. It targets a real current-main model-picker bug, but the current patch introduces two blocking edge-case regressions in provider-scoped picker behavior and is missing the required user-facing changelog entry.

## What This Changes

The PR changes the default model picker to suppress non-preferred configured model supplements, clamp invalid select initial values, and adds regression coverage for alias/provider scoping and allowlist option validity.

## Best Possible Solution

Keep the PR open and revise the patch so configured supplements are hidden only when the displayed model list is actually scoped to preferred-provider rows, invalid initial values clamp to a displayed model row or an explicit manual fallback, empty option lists avoid calling the select prompt, and the user-facing fix is recorded in the changelog.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring an alias such as `sonnet` for `claude-cli/claude-sonnet-4-6` and opening `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`; the code appends and preselects the configured Claude key outside the OpenAI-scoped rows.

## Solution Assessment

No, not as currently written. The direction is right, but the implementation should preserve the existing no-match provider fallback and should clamp invalid initial values to a displayed model option or explicit manual path rather than blindly using the first option.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate supplement hiding on an actual provider scope:** `src/flows/model-picker.ts:804`
  - body: `maybeFilterModelsByProvider` deliberately falls back to the full visible list when no catalog rows match `preferredProvider`. This guard still hides the current configured model solely because a preferred provider was requested, so fallback pickers can lose the existing model even though the list is not actually scoped. Gate this on `hasPreferredProvider` or equivalent.
  - confidence: 0.86
- **[P2] Prefer a displayed model when clamping initialValue:** `src/flows/model-picker.ts:828`
  - body: When the configured value is removed, `options[0]` can be `__manual__`, `__keep__`, or a provider setup action because those are prepended before model rows. That makes Enter follow an action path even when valid model options are visible, and can still pass an empty option list when auth filtering removes all model rows. Pick the first displayed model option and handle the no-option path explicitly.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `src/flows/model-picker.ts:804`
  - body: This changes user-facing model picker behavior, but the PR only updates code and tests. OpenClaw policy requires user-facing fixes to add a `CHANGELOG.md` entry with appropriate contributor credit.
  - confidence: 0.88
