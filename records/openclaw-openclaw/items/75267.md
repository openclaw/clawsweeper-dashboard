## Summary

Keep this PR open: it targets a real current-main model-picker bug, but two control-flow regressions should block merge and the required changelog entry is missing.

## What This Changes

The PR modifies `promptDefaultModel` to hide out-of-scope configured model supplements, clamp invalid initial selections, and add regression tests for provider-scoped alias leakage and allowlist validity.

## Best Possible Solution

Keep the PR direction, but gate supplement hiding on an actual preferred-provider-scoped list, clamp invalid initial values to a displayed model or explicit manual fallback, avoid empty selects, and add the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a Claude alias as the default, call `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`, and observe current main appending and preselecting the configured Claude key outside the OpenAI rows.

## Solution Assessment

No, not as currently proposed. The patch targets the right code path, but it should preserve the no-match provider fallback contract and avoid treating prepended action options as model defaults.

## Review Findings

Overall correctness: patch is incorrect

  - body: The new guard hides the configured supplement whenever `preferredProvider` is set and the resolved provider differs. `maybeFilterModelsByProvider` intentionally falls back to the full list when no preferred-provider rows exist, so this removes the current model from an unscoped fallback picker; gate the mismatch check on `hasPreferredProvider` or an equivalent actual-scope signal.
