## Summary

Keep this PR open: current main still has the provider-scoped model picker bug, and the branch is a useful starting point, but the diff would regress fallback pickers and action-option clamping, so it needs targeted changes before merge.

## What This Changes

The PR changes the default model picker to hide configured model supplements outside a preferred provider scope, clamp invalid initial selections to displayed options, and add regression tests for provider-scoped aliases and allowlist validity.

## Best Possible Solution

Keep the PR direction, but make the supplement hiding depend on an actual preferred-provider-scoped list, clamp invalid initial values to a displayed model row or explicit manual fallback, handle empty options, and add the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a Claude alias as the default, open `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`, and observe current main appending and preselecting the configured Claude key outside the OpenAI-scoped rows.

## Solution Assessment

No, not as currently proposed. The approach targets the right bug, but the implementation should preserve the existing no-match provider fallback and avoid treating prepended action options as model defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate supplement hiding on actual provider scope:** `src/flows/model-picker.ts:804-807`
  - body: The PR hides the configured supplement whenever `preferredProvider` is set and the resolved provider differs. `maybeFilterModelsByProvider` intentionally falls back to the full list when no preferred-provider rows exist, so this can remove the current model from an unscoped fallback picker. Gate the mismatch check on `hasPreferredProvider` or an equivalent actual-scope signal.
  - confidence: 0.86
- **[P2] Prefer a displayed model when clamping initial value:** `src/flows/model-picker.ts:823-826`
  - body: When the configured value is removed, `options[0]` can be `__manual__`, `__keep__`, or a provider setup action because action options are prepended before model rows. That makes Enter follow an action path even when a valid model row is available, and it does not handle an empty options list. Pick the first displayed model option and avoid calling select with no options.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `src/flows/model-picker.ts:804`
  - body: This changes user-facing model picker behavior but the PR only updates code and tests. OpenClaw policy requires user-facing fixes to add a `CHANGELOG.md` entry with appropriate contributor credit.
  - confidence: 0.88
