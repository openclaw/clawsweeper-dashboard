## Summary

Keep open: the current PR targets a real, source-reproducible model-picker bug, but the proposed patch still has two blocking correctness problems and lacks after-fix real behavior proof.

## What This Changes

The PR filters provider-scoped model-picker supplements, clamps invalid default-model initial values, and adds regression tests for alias/provider leakage.

## Best Possible Solution

Keep the focused interactive model-picker fix, but update it to preserve unscoped fallback behavior and choose a valid displayed model or manual fallback before merge.

## Reproduction Assessment

Yes. Source inspection shows that current main can append an out-of-scope configured alias and use it as initialValue when promptDefaultModel is called with preferredProvider "openai", allowKeep false, and includeManual false.

## Solution Assessment

No. The PR fixes the right symptom, but the implementation should gate supplement hiding on the actual scoped-provider list and should not fall back to action rows or an empty options list for invalid initial values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate supplement hiding on actual provider scope:** `src/flows/model-picker.ts:804`
  - body: The new provider check hides the configured supplement whenever preferredProvider is set and the resolved provider differs. maybeFilterModelsByProvider intentionally falls back to the full list when no preferred-provider rows exist, so this makes the current configured model unselectable in that fallback path; gate this on hasPreferredProvider or an equivalent actual-scope signal.
  - confidence: 0.86
- **[P2] Choose a model row for invalid initial values:** `src/flows/model-picker.ts:828`
  - body: When the configured value is removed, options[0] can be __manual__, __keep__, or a provider setup action because those rows are prepended before model rows. Prefer the first displayed model option and avoid calling select with an empty options list.
  - confidence: 0.82
