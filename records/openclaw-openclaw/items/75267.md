## Summary

Keep this PR open, but it still needs changes before merge. Current main still has the provider-scoped alias leak the PR targets, while the proposed patch can hide valid fallback supplements, default to action rows, and lacks the required changelog entry.

## What This Changes

The PR filters configured default-model supplements by preferred provider, clamps invalid initial selections, and adds model-picker regression tests.

## Best Possible Solution

Keep the PR direction, but preserve the documented no-match provider fallback, clamp invalid defaults to displayed model rows or a safe manual/no-options path, and add a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main appends and preselects a configured Claude alias even when `promptDefaultModel` is called with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`.

## Solution Assessment

No, not as currently proposed. The patch targets the right bug, but it should only hide configured supplements when provider scoping is actually active and should not clamp invalid defaults to prepended action rows or an empty option list.
