## Summary

Keep this PR open, but do not merge the exact patch yet. Current main still has the provider-scoped alias leak the PR is trying to fix, and the PR is focused, but the diff creates two narrow picker edge cases and is missing the required user-facing changelog entry.

## What This Changes

The PR updates the default model picker to hide configured non-preferred provider supplements, clamp invalid select initial values to displayed options, and add regression tests for provider-scoped alias and allowlist behavior.

## Best Possible Solution

Adjust the patch so configured supplements are hidden only when the picker is actually scoped to matching preferred-provider rows, choose the first displayed model row when clamping an invalid initial value, handle the no-model-options path explicitly, and add focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring an alias such as `sonnet` for `claude-cli/claude-sonnet-4-6` and opening `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`; the configured Claude key is appended and preselected outside the OpenAI-scoped rows.

## Solution Assessment

No, not as currently written. The direction is right, but the implementation should respect the existing no-match provider fallback and should clamp invalid initial values to a model option or explicit manual fallback rather than blindly using the first option.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate supplement hiding on an actual provider scope:** `src/flows/model-picker.ts:804`
  - body: `maybeFilterModelsByProvider` deliberately falls back to the full visible list when no catalog rows match `preferredProvider`. This new guard still hides a non-catalog current model solely because a preferred provider was requested, so `allowKeep: false` fallback pickers can lose the existing configured model even though the list is not actually scoped. Use `hasPreferredProvider` or equivalent when enforcing the provider mismatch.
  - confidence: 0.86
- **[P2] Prefer a displayed model when clamping the initial value:** `src/flows/model-picker.ts:823`
  - body: When the configured value is removed, `options[0]` can be `__manual__`, `__keep__`, or a provider setup action because those are prepended before model rows. That makes Enter follow an action path even when valid model options are visible, and it can still leave an empty select when no rows survive auth filtering. Pick the first displayed model option and handle the no-option path explicitly.
  - confidence: 0.8

  - body: This changes user-facing model picker behavior but the PR only updates code and tests. OpenClaw policy requires user-facing fixes to add a `CHANGELOG.md` entry with appropriate contributor credit.
