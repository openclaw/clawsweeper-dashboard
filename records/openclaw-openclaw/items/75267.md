## Summary

Keep open: current main still has the provider-scoped alias picker bug, and this PR targets the right area, but the patch introduces two concrete correctness regressions and still needs the required user-facing changelog entry before merge.

## What This Changes

The PR changes the default model picker to hide configured-model supplements outside the preferred provider scope, clamp invalid initial selections, and add regression tests for scoped picker option validity.

## Best Possible Solution

Keep the PR direction, but gate configured-supplement hiding on an actual provider-scoped result set, clamp invalid defaults to displayed model rows or a manual fallback, and add the required changelog entry with focused regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a Claude alias as the default, open `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`, and observe current main appending and preselecting the out-of-scope configured key.

## Solution Assessment

No, not as currently proposed. The patch targets the right bug, but it should preserve the documented no-match provider fallback and avoid treating prepended action options or empty option lists as valid model defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate supplement hiding on actual provider scope:** `src/flows/model-picker.ts:802-805`
  - body: The new provider check hides the configured supplement whenever `preferredProvider` is set and the resolved provider differs. `maybeFilterModelsByProvider` intentionally falls back to the full list when no preferred-provider rows exist, so this makes the current configured model unselectable in that fallback path; gate this on `hasPreferredProvider` or an equivalent actual-scope signal.
  - confidence: 0.86
- **[P2] Prefer a displayed model when clamping initial value:** `src/flows/model-picker.ts:834`
  - body: When the configured value is removed, `options[0]` can be `__manual__`, `__keep__`, or a provider setup action because those rows are prepended before model rows. It can also be undefined if filtering/auth leaves no options; choose the first displayed model option and avoid calling `select` with an empty list.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/flows/model-picker.ts:802-805`
  - body: This changes user-facing model picker behavior but the PR only updates code and tests. Add a `CHANGELOG.md` entry under the active fixes section with appropriate non-bot contributor credit.
  - confidence: 0.88
