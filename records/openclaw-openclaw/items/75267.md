## Summary

Keep this PR open, but it is not merge-ready. It targets a real current-main model-picker bug, while the patch can regress the documented unfiltered fallback path and can default to action options instead of displayed models; a narrow repair plus changelog entry should be enough.

## What This Changes

The PR changes `src/flows/model-picker.ts` to filter configured-model supplements by preferred provider and clamp invalid initial selections, with regression tests in `src/commands/model-picker.test.ts`.

## Best Possible Solution

Keep the PR direction, but update the logic to hide configured supplements only when a real provider-scoped list is active, clamp invalid initial selections to displayed model rows or a manual/no-options fallback, and add the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a Claude alias as the default, open `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`, and observe current main appending and preselecting the out-of-scope configured key.

## Solution Assessment

No, not as currently proposed. The patch targets the right bug, but it should preserve the documented no-match provider fallback and avoid treating prepended action options or empty option lists as valid model defaults.
