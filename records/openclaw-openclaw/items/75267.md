## Summary

Keep open: the PR targets a real source-reproducible model-picker bug, but the current patch still has two P2 correctness blockers and lacks required after-fix real behavior proof.

## What This Changes

The PR changes `src/flows/model-picker.ts` and `src/commands/model-picker.test.ts` to filter provider-scoped model-picker supplements, clamp invalid initial values, and add regression coverage for alias/provider leakage.

## Best Possible Solution

Keep the PR direction, but gate supplement hiding on an actually scoped provider list and clamp invalid defaults to displayed model rows with a no-options fallback.

## Reproduction Assessment

Yes, source-reproducible. On current main, configure a Claude alias as the default and call `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`; the out-of-scope configured key is appended and used as the initial value.

## Solution Assessment

No. The proposed patch fixes the right symptom, but it hides the configured supplement even when the picker has fallen back to an unscoped list and can clamp the initial value to a non-model action row.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate supplement hiding on actual provider scope:** `src/flows/model-picker.ts:811`
  - body: The new provider check hides the configured supplement whenever `preferredProvider` is set and the resolved provider differs. `maybeFilterModelsByProvider` intentionally falls back to the full list when no preferred-provider rows exist, so this makes the current configured model unselectable in that fallback path; gate this on `hasPreferredProvider` or an equivalent actual-scope signal.
  - confidence: 0.86
- **[P2] Pick a displayed model for invalid initial values:** `src/flows/model-picker.ts:829`
  - body: When the configured value is removed, `options[0]` can be `__manual__`, `__keep__`, or a provider setup action because those rows are prepended before model rows. Pick the first displayed model option and avoid calling `select` with no options remaining.
  - confidence: 0.82
