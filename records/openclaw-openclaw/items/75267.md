## Summary

Keep open: the PR targets a real source-reproducible model-picker bug, but the current diff still regresses the provider fallback contract and can default the picker to non-model action rows. The external PR also needs after-fix real behavior proof before merge.

## What This Changes

The PR filters configured model supplements by provider scope, clamps invalid default-model picker initial values, and adds model-picker regression tests.

## Best Possible Solution

Keep the PR direction, but gate supplement hiding on an actually scoped provider list and clamp invalid defaults to displayed model rows with a no-options fallback.

## Reproduction Assessment

Yes, source-reproducible. Configure a Claude alias as the default, call `promptDefaultModel` with `preferredProvider: "openai"`, `allowKeep: false`, and `includeManual: false`, and current main appends/preselects the out-of-scope configured key.

## Solution Assessment

No. The PR targets the right bug, but the patch should preserve the documented no-match provider fallback and should not use the first option when that option may be a manual/keep/setup action.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate supplement hiding on actual provider scope:** `src/flows/model-picker.ts:811`
  - body: The new provider check hides the configured supplement whenever `preferredProvider` is set and the resolved provider differs. `maybeFilterModelsByProvider` intentionally falls back to the full list when no preferred-provider rows exist, so this makes the current configured model unselectable in that fallback path; gate this on `hasPreferredProvider` or an equivalent actual-scope signal.
  - confidence: 0.86
- **[P2] Prefer a displayed model when clamping the initial value:** `src/flows/model-picker.ts:829`
  - body: When the configured value is removed, `options[0]` can be `__manual__`, `__keep__`, or a provider setup action because those rows are prepended before model rows. Pick the first displayed model option and avoid calling `select` with no options remaining.
  - confidence: 0.82
