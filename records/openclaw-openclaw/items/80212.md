## Summary

Keep this open: current main still computes different Telegram DM-topic session keys for proactive outbound sends and inbound replies, so the reported cron-initiated conversation split remains source-reproducible.

## What This Changes

Allow cron/proactive Telegram DM-topic messages to use the same session key shape that inbound replies use for that DM topic.

## Best Possible Solution

Align Telegram direct DM-topic outbound mirror session routing with inbound key shape while preserving the numeric Telegram thread id used for delivery; avoid adding a broad cron session-key override or aliasing system unless migration evidence requires it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a proactive Telegram DM-topic send appends only the topic id, while an inbound reply under the same configured DM topic mode appends `chatId:topicId`; no live Telegram run was performed.

## Solution Assessment

Yes. Option A, outbound session-key alignment, is the narrowest maintainable fix for this mismatch; cron key overrides or generic aliases would add broader behavior than this bug needs.
