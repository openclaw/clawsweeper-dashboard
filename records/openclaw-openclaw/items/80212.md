## Summary

This should stay open: current main still has a source-level mismatch between Telegram DM-topic outbound mirror sessions and inbound reply sessions, so cron-initiated DM topic conversations can split context as reported.

## What This Changes

Request to let cron-initiated Telegram DM topic messages create or target the same session key that inbound replies will use.

## Best Possible Solution

Align Telegram direct DM-topic outbound session-route computation with inbound routing, preserving the numeric Telegram `threadId` used for send while mirroring transcript/session metadata under the inbound-compatible key.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a proactive Telegram DM topic send appends only the topic id, while an inbound reply appends `chatId:topicId` on the same base key. I did not run live Telegram or cron in this read-only review.

## Solution Assessment

Yes: Option A, outbound session key alignment, is the narrowest maintainable fix. A cron `sessionKey` override or generic aliasing system would be broader than needed for this concrete Telegram routing mismatch.
