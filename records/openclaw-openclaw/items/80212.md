## Summary

This should stay open: current main still has a source-level mismatch between Telegram DM-topic outbound mirror sessions and inbound reply sessions, so cron-initiated DM topic conversations can split context under `dmScope: "per-account-channel-peer"`. The report is specific, current, and narrow enough for a focused routing fix.

## What This Changes

The request is to let cron-initiated Telegram DM-topic messages use the same session routing key that inbound replies will use.

## Best Possible Solution

Align Telegram direct DM-topic outbound session-route computation with inbound routing for the same account, peer, and topic, and add a regression test proving a proactive send and reply select the same session key.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a proactive Telegram DM-topic send appends only the topic id, while an inbound reply appends `chatId:topicId` on the same base key.

## Solution Assessment

Yes. Outbound session-key alignment is the narrowest maintainable solution; adding cron key overrides or a generic aliasing system would be broader than this concrete routing mismatch requires.
