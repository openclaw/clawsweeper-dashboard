## Summary

Keep open: current main still has the source-level session split between Telegram DM-topic outbound mirror sessions and inbound replies, and no open PR directly references or fixes this issue.

## What This Changes

The issue asks cron-initiated Telegram DM-topic sends to create or mirror the same session key that inbound replies use for the same DM topic.

## Best Possible Solution

Align Telegram direct DM-topic outbound session-route computation with inbound routing by using a `chatId:topicId` session suffix while preserving the numeric `threadId` used for delivery.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: proactive outbound routing appends only the topic id, while inbound DM-topic replies append `chatId:topicId` to the same base session.

## Solution Assessment

Yes. Outbound/inbound key alignment is the narrowest maintainable fix; a cron `sessionKey` override or generic alias system would be broader than needed for this concrete mismatch.
