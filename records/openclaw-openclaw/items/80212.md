## Summary

Keep open: current main still routes proactive Telegram DM-topic outbound mirror sessions with a bare topic suffix while inbound replies use a chat-scoped topic suffix, so the reported cron-created conversation split remains source-reproducible. The older matching report was stale-closed and the exact prior fix PR was closed unmerged, so this issue remains the live canonical work item.

## What This Changes

Request to let cron/message-tool proactive Telegram DM-topic sends create the same session key that inbound replies use under isolated DM scope.

## Best Possible Solution

Align Telegram direct DM-topic outbound session-route computation with inbound routing by using a chat-scoped session thread id for the transcript key while preserving the numeric topic id used for Telegram sends.

## Reproduction Assessment

Yes, at source level: a proactive direct Telegram topic send appends only the topic id, while an inbound reply appends `chatId:topicId` on the same base session key. I did not run a live Telegram cron reproduction in this read-only review.

## Solution Assessment

Yes: outbound session-key alignment is the narrowest maintainable fix. Cron `sessionKey` override or generic aliasing would be broader than needed for this concrete Telegram route mismatch.
