## Summary

Keep open: current main still has the reported Telegram DM-topic session-key split, and the latest release shows the same outbound/inbound divergence. This is a narrow routing bug rather than a new cron feature request if the fix stays focused on outbound/inbound key alignment.

## What This Changes

Allow cron-initiated Telegram DM-topic message sends to mirror into the same session key that inbound replies in that topic use.

## Best Possible Solution

Align Telegram direct DM-topic outbound session-route computation with inbound routing while keeping the Telegram API send `message_thread_id` numeric, and add regression tests that prove proactive sends and inbound replies resolve to one canonical session key.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with Telegram DM topic threading enabled, a proactive send to `<chatId>:topic:<topicId>` mirrors under `:thread:<topicId>`, while the inbound reply in that same topic routes under `:thread:<chatId>:<topicId>`.

## Solution Assessment

Yes. Option A, outbound session key alignment, is the narrowest maintainable fix; a generic cron `sessionKey` override or aliasing system would be broader than needed for this concrete routing mismatch.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80212. Current main splits Telegram DM-topic sessions: inbound handling with `dm.threadReplies: "inbound"` uses `baseSessionKey:thread:<chatId>:<dmThreadId>`, while the Telegram message-tool outbound route for direct topic targets mirrors under `baseSessionKey:thread:<topicId>`. Implement a narrow Telegram plugin fix so direct DM-topic outbound session routes use the same session key suffix as inbound replies, while preserving the numeric `threadId`/`message_thread_id` used for Telegram sends and preserving existing group topic behavior. Do not add cron `sessionKey` override, generic aliasing, or a new config option. Add focused regression coverage for default/main dmScope and per-channel/per-account DM scopes showing a proactive send target and inbound reply compute the same canonical session key; update message-action mirror coverage if the fix crosses `prepareOutboundMirrorRoute`. Consider whether legacy shorter outbound-only keys need an explicit migration/alias note, but do not add broad compatibility machinery unless the tests and code path require it. Changelog is expected because this is user-visible Telegram session continuity.
