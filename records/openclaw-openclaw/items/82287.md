## Summary

Keep open: current main now mirrors CLI turns into OpenClaw JSONL, but normal under-budget CLI turns still bypass active context-engine finalization and there is no host warning or contract check for a context engine that cannot persist those mirrored turns.

## What This Changes

The issue requests host-side protection against silent CLI transcript persistence gaps when a selected third-party context engine lacks the CLI turn persistence hook support.

## Best Possible Solution

Route mirrored CLI turns through the existing context-engine finalization path, with a clear warning when the selected engine still cannot consume CLI-mirrored turns, while preserving the current raw JSONL mirror.

## Reproduction Assessment

Yes from source inspection, though not from a live third-party plugin run. Current main writes the CLI JSONL mirror, then returns under budget without calling the active context engine's `afterTurn`, `ingestBatch`, `ingest`, or a warning path.

## Solution Assessment

Yes, the best fix is in the host CLI transcript finalization path rather than in a plugin-specific workaround. Reusing the existing after-turn/ingest fallback is narrower than a legacy fallback and makes the documented context-engine lifecycle true for CLI turns.
