- **[P1] Reject false-positive meeting-notes starts:** `extensions/discord/src/voice/meeting-notes-source.ts:55`
  - body: When the bot is already connected to the requested guild/channel, `DiscordVoiceManager.join()` returns `ok: true` before updating the existing `VoiceSessionEntry`.   - confidence: 0.89

  - body: The stop path throws immediately on any provider stop failure, so a Discord session whose voice manager has already cleared the session cannot write `stoppedAt` or generate a summary.   - confidence: 0.86
- **[P2] Reject ambiguous Discord manager selection:** `extensions/discord/src/voice/meeting-notes-source.ts:25`
  - body: When `accountId` is omitted, the provider picks the first manager in insertion order for start, stop, and status. With multiple Discord accounts active, that can join or leave the wrong bot, so ambiguous requests should fail unless there is exactly one registered manager.
  - confidence: 0.84
