## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject false-positive meeting-notes starts:** `extensions/discord/src/voice/meeting-notes-source.ts:43-55`
  - body: `DiscordVoiceManager.join()` already returns `ok: true` when the bot is in the same guild/channel, before it updates any `meetingNotes` callback. This provider treats that as a started notes session, so an existing voice session can claim capture is running while no transcript callback is attached.
  - confidence: 0.92
- **[P1] Finalize sessions after benign provider stop failures:** `extensions/meeting-notes/src/tool.ts:205-213`
  - body: When `provider.stop()` returns a non-ok result, the tool throws before deleting `activeSessions`, writing `stoppedAt`, or generating the summary. A Discord session that is already disconnected can therefore leave captured transcript data permanently open instead of finalizing notes.
  - confidence: 0.9
- **[P2] Require accountId for ambiguous Discord manager selection:** `extensions/discord/src/voice/meeting-notes-source.ts:20-25`
  - body: If `accountId` is omitted, start/stop/status select the first manager from `managersByAccountId`. Multi-account gateways can then join or leave the wrong Discord account based on insertion order, so ambiguous requests should fail unless only one manager exists.
  - confidence: 0.9
- **[P2] Cancel late auto-start captures during shutdown:** `extensions/meeting-notes/src/tool.ts:395-399`
  - body: If shutdown happens while `startMeetingNotes()` is still in flight, this `.then` still records the session id after `stop()` has already drained known sessions. That can leave an auto-start capture running after service teardown without a matching stop/finalize pass.
  - confidence: 0.86
- **[P2] Persist sessions only after provider start succeeds:** `extensions/meeting-notes/src/tool.ts:160-168`
  - body: `metadata.json` is written before `provider.start(...)`, and a start failure throws without deleting it. Failed Discord joins or missing managers will leave stored sessions that never actually started and later appear in list/show/stop flows.
  - confidence: 0.88
- **[P2] Merge manifest providers for meeting-notes listings:** `src/plugins/capability-provider-runtime.ts:69`
  - body: Adding `meetingNotesSourceProviders` to the capability key map is not enough because active providers return early unless the key is included in the manifest-merge allowlist. Once `manual-transcript` is active, `meeting_notes status` can hide manifest-owned providers such as `discord-voice`.
  - confidence: 0.84
