Full review comments:

- **[P2] Reject ambiguous Discord account selection:** `extensions/discord/src/voice/meeting-notes-source.ts:25`
  - body: When `accountId` is omitted, start/stop/status silently use the first manager in insertion order. Multi-account gateways can join or leave the wrong bot account, so ambiguous requests should fail or require an explicit account id.
  - confidence: 0.89
- **[P2] Persist live sessions only after provider start succeeds:** `extensions/meeting-notes/src/tool.ts:160`
  - body: `startMeetingNotes` writes `metadata.json` before `provider.start(...)`. A missing Discord manager or failed join leaves a stored session that never captured anything, and later list/stop/summarize flows treat it as real state.
  - confidence: 0.91
- **[P2] Defer import metadata until transcript import succeeds:** `extensions/meeting-notes/src/tool.ts:273`
  - body: The import path writes the session before `provider.importTranscript(...)`. If parsing/import fails, users get an orphan meeting-notes session with no imported transcript or summary artifacts.
  - confidence: 0.88
- **[P2] Stop late auto-start sessions after shutdown:** `extensions/meeting-notes/src/tool.ts:409-413`
  - body: If shutdown happens while `startMeetingNotes()` is in flight, this callback can record the session id after `stop()` already ran. That leaves an auto-started capture outside the service lifecycle.
  - confidence: 0.86
- **[P2] Preserve existing stop times for inactive sessions:** `extensions/meeting-notes/src/tool.ts:236`
  - body: Stopping a stored but inactive session always rewrites `stoppedAt`, so rerunning stop to regenerate a summary corrupts the original end time for imported or already-stopped sessions.
  - confidence: 0.86
- **[P2] Merge meeting-notes providers when active providers exist:** `src/plugins/capability-provider-runtime.ts:69`
  - body: The new capability key is added, but active-provider resolution still skips manifest merging for meeting notes. Once the manual provider is active, status/listing can hide other providers such as `discord-voice`.
  - confidence: 0.84
