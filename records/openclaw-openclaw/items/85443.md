Overall confidence: 0.9

Full review comments:

- **[P1] Reject already-connected Discord starts:** `extensions/discord/src/voice/meeting-notes-source.ts:43-55`
  - body: `manager.join()` returns success when the bot is already in the same guild/channel, but that path does not attach the new `meetingNotes` callback. This reports a notes session as started while transcripts will never be captured.
  - confidence: 0.93
- **[P1] Finalize sessions when provider stop is non-critical:** `extensions/meeting-notes/src/tool.ts:204-213`
  - body: Throwing on any provider stop error prevents `stoppedAt` and `summary.md` from being written. A session that was already disconnected can have valid transcript data but becomes permanently open instead of finalizing.
  - confidence: 0.9
- **[P2] Reject ambiguous Discord account routing:** `extensions/discord/src/voice/meeting-notes-source.ts:20-25`
  - body: When `accountId` is omitted, start/stop/status pick the first manager in insertion order. Multi-account gateways can join, leave, or report the wrong Discord account, so ambiguous requests should fail closed or require an account id.
  - confidence: 0.88
- **[P2] Stop late auto-start sessions during shutdown:** `extensions/meeting-notes/src/tool.ts:395-399`
  - body: If shutdown happens while `startMeetingNotes()` is still in flight, this callback records the session id after `stop()` has already run. That can leave capture running after service teardown with no later finalization.
  - confidence: 0.86
- **[P2] Persist live sessions only after provider start succeeds:** `extensions/meeting-notes/src/tool.ts:160-167`
  - body: `metadata.json` is written before `provider.start()` succeeds, so a missing Discord manager or join failure leaves a stored session that never actually captured anything and can later appear in list/show/stop flows.
  - confidence: 0.89
- **[P2] Persist imports only after transcript import succeeds:** `extensions/meeting-notes/src/tool.ts:259-267`
  - body: Manual import writes metadata before `provider.importTranscript()` runs. If parsing/import fails, users are left with a visible meeting session that has no transcript or summary artifacts.
  - confidence: 0.87
- **[P2] Preserve existing stopped timestamps:** `extensions/meeting-notes/src/store.ts:238-252`
  - body: Stopping or regenerating a non-active stored session rewrites `stoppedAt` every time, corrupting historical end times for imported or already-stopped meetings. Only active sessions should get a new stop time, or the update should be conditional.
  - confidence: 0.83
- **[P2] Merge meeting-notes providers from manifests when active:** `src/plugins/capability-provider-runtime.ts:34`
  - body: Once the active Meeting Notes plugin registers `manual-transcript`, `resolvePluginCapabilityProviders()` returns active providers only for this new key. That can hide manifest-discovered providers such as `discord-voice` from `meeting_notes status`.
  - confidence: 0.82
