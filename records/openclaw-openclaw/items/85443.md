- **[P2] Reject ambiguous Discord account routing:** `extensions/discord/src/voice/meeting-notes-source.ts:20-25`
  - body: When `accountId` is omitted, this falls back to the first manager in insertion order. Multi-account gateways can join, leave, or report status for the wrong Discord bot, so ambiguous requests should fail closed or require an account id.
  - confidence: 0.9
- **[P2] Persist live sessions only after start succeeds:** `extensions/meeting-notes/src/tool.ts:160-167`
  - body: `metadata.json` is written before `provider.start()` succeeds. A missing Discord manager or join failure leaves a stored session that never captured anything and can later appear in list/show/stop flows.
  - confidence: 0.89
- **[P2] Persist imports only after transcript import succeeds:** `extensions/meeting-notes/src/tool.ts:273-279`
  - body: Manual import writes metadata before `provider.importTranscript()` runs. If import fails, users are left with a visible meeting session that has no transcript or summary artifacts.
  - confidence: 0.88
- **[P2] Stop late auto-start sessions during shutdown:** `extensions/meeting-notes/src/tool.ts:409-413`
  - body: If shutdown happens while `startMeetingNotes()` is still in flight, this callback can record the session id after `stop()` has already run. That can leave capture running after service teardown with no later finalization.
  - confidence: 0.86
- **[P2] Preserve existing stopped timestamps:** `extensions/meeting-notes/src/tool.ts:233-237`
  - body: Stopping a non-active stored session always rewrites `stoppedAt`, so rerunning stop or regenerate flows corrupts the historical end time for imported or already-stopped meetings. Only active sessions should get a new stop time, or the update should be conditional.
  - confidence: 0.84
- **[P2] Merge meeting-notes providers when active:** `src/plugins/capability-provider-runtime.ts:69`
  - body: Once the active Meeting Notes plugin registers `manual-transcript`, the resolver can return active providers only because this new capability is not included in the active-plus-manifest merge set. That can hide manifest-discovered sources such as `discord-voice` from `meeting_notes status`.
  - confidence: 0.82
