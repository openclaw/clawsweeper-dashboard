## Summary

Keep open. Current main and the latest release still reject cross-agent `sessions_send` under non-`all` visibility before an allow-matching A2A policy can authorize it, and the linked fixing PR is still open rather than merged.

## What This Changes

The issue requests allowing `sessions_send` to use A2A policy for cross-agent messaging without broadening session list/history/status visibility.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main change in `src/plugin-sdk/session-visibility.ts` that preserves read-surface isolation and handles sandboxed target resolution safely.

Yes for the current blocking behavior: a cross-agent target with `visibility: "tree"` reaches the non-`all` visibility rejection before A2A checks can allow it. I did not run a live gateway scenario because this review was read-only and the source path is direct.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the narrow maintainable solution is a send-only exception in the active SDK visibility helper plus sandboxed resolution safeguards, docs, and regression coverage.
