## Summary

Keep open. Current main still blocks cross-agent `sessions_send` under non-`all` session visibility before evaluating an allow-matching A2A policy, and the live fixing PR that references this issue is still open and conflicting rather than merged.

## What This Changes

The issue requests that A2A policy, not broad session visibility, authorize cross-agent `sessions_send` while list/history/status remain visibility-gated.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main fix in `src/plugin-sdk/session-visibility.ts` that routes only `send` through A2A policy, preserves read-surface isolation, and safely handles sandboxed target resolution.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a cross-agent target, `visibility: "tree"`, and an allow-matching A2A policy, the active checker returns the send visibility error before evaluating A2A.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the narrow maintainable solution is a send-only exception in the active SDK visibility helper plus regression coverage and real setup proof.
