## Summary

Keep open. Current main and the latest release still route cross-agent `sessions_send` through the shared visibility guard, which rejects non-`all` visibility before A2A policy can allow the target; the linked fixing PR/commit is not merged into main.

## What This Changes

The issue requests allowing A2A-authorized cross-agent `sessions_send` without also granting cross-agent list/history/status visibility.

## Best Possible Solution

Review and land the linked PR or a narrow replacement that allows only A2A-authorized `sessions_send` while keeping list/history/status visibility restricted and covered by regression tests.

## Reproduction Assessment

Yes. Source inspection shows a cross-agent target with effective visibility below `all` returns a visibility-forbidden result before the A2A allow policy is evaluated; I did not run a live multi-agent setup in this read-only review.

## Solution Assessment

Unclear. The proposed send-only exception is the narrowest apparent fix, but maintainers should explicitly approve the security model because current docs/config treat `sessions_send` as part of the single `tools.sessions.visibility` surface.
