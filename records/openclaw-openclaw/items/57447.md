## Summary

Keep open. Current main and the latest release still require effective `tools.sessions.visibility="all"` before A2A allow rules can approve a cross-agent `sessions_send`, so the requested send-without-read-visibility behavior remains unresolved and needs maintainer security/product direction.

## What This Changes

Allow `sessions_send` to use A2A policy for cross-agent sends without also broadening list, history, or status visibility.

## Best Possible Solution

Keep this as the canonical authorization tracker and land a maintainer-approved fix or broader per-capability visibility design that permits only A2A-authorized cross-agent sends while preserving read/list/status and sandbox isolation.

## Reproduction Assessment

Yes, at source level. A cross-agent `send` under effective visibility below `all` reaches the visibility denial before A2A allow checks; I did not run a live gateway reproduction because this review was read-only.

## Solution Assessment

Unclear until maintainers choose the security model. A send-only A2A exception is plausible, but the broader per-capability visibility direction at https://github.com/openclaw/openclaw/issues/48785 may be the cleaner product shape.
