## Summary

Keep open. Current main still blocks cross-agent `sessions_send` under non-`all` session visibility before an allow-matching A2A policy can authorize the send, and the open PR that references this issue is still unmerged and conflicting.

## What This Changes

Allow A2A-authorized cross-agent `sessions_send` without requiring `tools.sessions.visibility=all`, while preserving list/history/status isolation.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main fix in `src/plugin-sdk/session-visibility.ts` that routes only `send` through A2A policy, preserves read-surface isolation, safely handles sandboxed target resolution, and updates regression coverage/docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a cross-agent target, `visibility: "tree"`, and an allow-matching A2A policy, the active checker returns the send visibility error before evaluating A2A.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the narrow maintainable solution is a send-only exception in the active SDK visibility helper plus sandboxed resolution safeguards, docs, and regression coverage.
