## Summary

Keep open. Current main and the latest release still route `sessions_send` through the shared visibility guard, which rejects cross-agent targets under non-`all` visibility before A2A authorization can allow them; the linked fixing PR remains open and conflicting, so this issue is still necessary.

## What This Changes

The issue asks to let A2A policy authorize cross-agent `sessions_send` without also granting cross-agent list/history/status visibility.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main, policy-approved change in `src/plugin-sdk/session-visibility.ts` and the `sessions_send` resolution path that allows only A2A-approved sends while preserving read, status, list, sandbox, and discovery isolation.

## Reproduction Assessment

Yes. Source inspection shows a cross-agent `send` target with `visibility: "tree"` reaches the non-`all` visibility rejection before an allow-matching A2A policy can authorize it; I did not run a live gateway scenario because this review is read-only and the source path is direct.

## Solution Assessment

Unclear until maintainers approve the authorization split. If approved, the narrow maintainable path is a send-only exception in the active SDK visibility helper plus sandboxed target-resolution safeguards, docs, and regression coverage.
