## Summary

Keep open. Current main and the latest release still use the same authorization shape: cross-agent `sessions_send` is rejected under non-`all` session visibility before A2A allow rules can approve it, and an open linked implementation candidate still needs maintainer security/product review.

## What This Changes

## Best Possible Solution

Land a maintainer-approved send-only authorization path that lets explicit A2A policy gate `sessions_send` while preserving visibility gates for list, history, and status.
