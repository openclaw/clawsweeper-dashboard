## Summary

Keep open. Current main still rejects cross-agent `sessions_send` under non-`all` session visibility before an allow-matching A2A policy can authorize it, and the linked fixing PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to let A2A policy authorize cross-agent `sessions_send` while keeping scoped session visibility for list, history, and status isolation.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main fix in `src/plugin-sdk/session-visibility.ts` that routes only `send` through A2A policy, preserves read-surface isolation, safely handles sandboxed target resolution, and updates regression coverage/docs.

## Reproduction Assessment

Yes by source inspection: a cross-agent target with `visibility: "tree"` and an allow-matching A2A policy reaches the non-`all` visibility rejection before the A2A checks. I did not run a live gateway scenario because this review is read-only and the source path is direct.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the narrow maintainable solution is a send-only exception in the active SDK visibility helper plus sandboxed resolution safeguards, docs, and regression coverage.
