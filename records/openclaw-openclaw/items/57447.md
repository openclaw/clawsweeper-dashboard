## Summary

Keep open. Current main still applies the cross-agent visibility guard before A2A policy for `send`, so default `tree` visibility blocks an A2A-authorized `sessions_send`; open PR #57483 already targets this issue with closing syntax, so cleanup closure or a duplicate repair job would be premature.

## What This Changes

Allow `sessions_send` to use A2A policy for cross-agent message delivery without widening list, history, or status visibility.

## Best Possible Solution

Review and land or replace PR #57483 with a narrow current-main fix that routes only `send` through A2A policy and adds regression coverage for allowed sends, denied sends, read-surface denial, and sandboxed label/session resolution.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: with a cross-agent target and `visibility: "tree"`, `createSessionVisibilityChecker` returns the send visibility error before evaluating an enabled allow-matching A2A policy.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to separate send authorization from transcript visibility, but the active PR should be checked against the current `src/plugin-sdk/session-visibility.ts` path and the sandboxed label-resolution path before landing.
