## Summary

Keep open. Current main still applies the cross-agent session visibility check before A2A policy for `sessions_send`, and open PR https://github.com/openclaw/openclaw/pull/57483 still targets this issue with closing syntax, so cleanup closure or a duplicate repair job would be premature.

## What This Changes

Allow A2A policy to authorize cross-agent `sessions_send` under tree/spawned visibility without exposing list, history, or status access.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main fix in the SDK visibility helper that routes only `send` through A2A policy, preserves read-surface isolation, and covers sandboxed resolution.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: with a cross-agent target, `visibility: "tree"`, and an allow-matching A2A policy, the active checker returns the send visibility error before evaluating A2A.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the maintainable path is a send-only exception in `src/plugin-sdk/session-visibility.ts` plus early/safe sandboxed label resolution and regression coverage.
