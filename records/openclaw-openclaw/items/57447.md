## Summary

Keep open. Current main and the latest release still apply the single `tools.sessions.visibility` gate before A2A authorization for cross-agent `sessions_send`, and the linked fixing PR is open, conflicting, and unmerged rather than a shipped fix.

## What This Changes

Allow A2A policy to authorize cross-agent `sessions_send` independently of read/list/status session visibility while preserving transcript and discovery isolation.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main, policy-approved change in the shared visibility helper and `sessions_send` resolution path that allows only A2A-approved sends while keeping list/history/status and sandbox discovery isolated.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the narrow maintainable solution is a send-only exception in the active SDK visibility helper plus sandboxed target-resolution safeguards, docs, and regression coverage.
