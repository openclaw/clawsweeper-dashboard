## Summary

Keep open. Current main still rejects cross-agent `sessions_send` under non-`all` session visibility before A2A policy can authorize the target, and an open PR already references this issue with closing syntax, so cleanup closure or a duplicate repair job would be premature.

## What This Changes

The issue asks for `sessions_send` to allow A2A-authorized cross-agent messaging without exposing cross-agent list/history/status access through broad session visibility.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main fix in the shared visibility helper, preserving read-surface isolation and adding regression coverage for both send and history/status/list.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a cross-agent target with `visibility: "tree"` and an allow-matching A2A policy reaches the visibility rejection before the A2A checks, and sandboxed label lookup can be rejected even earlier.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the maintainable path is a send-only exception in `src/plugin-sdk/session-visibility.ts` plus safe pre-authorization handling for sandboxed label resolution and focused tests.
