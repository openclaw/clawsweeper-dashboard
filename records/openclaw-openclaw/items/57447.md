## Summary

Keep open. Current main and v2026.6.10 still reject cross-agent `sessions_send` when effective session visibility is below `all` before A2A allow rules can approve the send, so the central request remains unresolved and needs maintainer security/product review.

## What This Changes

Request to let A2A policy authorize cross-agent `sessions_send` under restricted session visibility without broadening `sessions_list`, `sessions_history`, or `session_status` access.

## Best Possible Solution

Keep this as the canonical authorization tracker and land a maintainer-approved fix that allows only A2A-authorized cross-agent sends while preserving read/list/status and sandbox isolation.

## Reproduction Assessment

Yes. At source level, a cross-agent `send` under effective visibility below `all` reaches the visibility denial before A2A allow checks; I did not run a live gateway reproduction because this review was read-only.

## Solution Assessment

Unclear until maintainers choose the security model. A send-only A2A exception is plausible, but maintainers may prefer the broader per-capability visibility design tracked at https://github.com/openclaw/openclaw/issues/48785.
