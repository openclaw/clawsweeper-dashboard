## Summary

Keep open. Current main still rejects the cross-agent `sessions_send` path under non-`all` visibility before A2A allow policy is evaluated, and the open fixing PR still references this issue with closing syntax, so cleanup closure would be premature.

## What This Changes

The issue requests that A2A allow policy authorize cross-agent `sessions_send` without broadening session list/history/status visibility.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/57483 with a current-main fix in `src/plugin-sdk/session-visibility.ts` that routes only `send` through A2A policy, preserves read-surface isolation, and handles sandboxed resolution safely.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a cross-agent target, `visibility: "tree"`, and an allow-matching A2A policy, the active checker returns the send visibility error before evaluating A2A.

## Solution Assessment

Unclear until maintainers approve the authorization-policy split. If approved, the maintainable path is a send-only exception in the active SDK visibility helper plus early/safe sandboxed label resolution and regression coverage.
