## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep Windows handling out of the shared shell parser:** `src/utils/shell-argv.ts:38-40`
  - body: This changes `splitShellArgs` for every caller, not just QMD. Outside quotes, POSIX shells let backslash escape ordinary characters too, so a command like `open\claw channels login` executes as `openclaw channels login`; after this patch the shared parser keeps the backslash and guard callers such as `parseOpenClawChannelsLoginShellCommand` can miss the command. Please move the Windows-path preservation to the QMD `memory.qmd.command` parsing boundary and leave the shared shell semantics intact.
  - confidence: 0.9
