## Summary

Keep open: current main still applies `heartbeat.model` only when heartbeat origin is explicitly carried, and source inspection shows follow-up entry points that can re-enter as user/gateway agent runs without heartbeat origin or model override. The related closed heartbeat issues cover adjacent behaviors and do not supersede this follow-up model leak.

## What This Changes

The issue requests that every follow-up run inside a heartbeat-originated session continue using the configured `agents.defaults.heartbeat.model` instead of resolving the agent primary model.

## Best Possible Solution

Add a focused regression for heartbeat-originated follow-up/retry paths, then carry heartbeat origin and model override through those follow-ups without changing normal user-turn model persistence.
