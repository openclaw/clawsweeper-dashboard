## Repair Work Prompt

Fix #84825. `agents.defaults.heartbeat.model` is honored for the initial heartbeat reply but can be lost for follow-up runs created inside the same heartbeat session, including exec approval/tool retry and queued follow-up paths that re-enter as user/default-model runs. Thread a minimal heartbeat-origin or run-kind field through follow-up run creation, exec approval follow-up agent RPC, gateway agent ingress, and queued embedded/CLI runner paths as needed so heartbeat-originated follow-ups resolve the configured heartbeat model. Do not make the heartbeat model sticky for later real user turns; preserve the #75452 opposite-direction behavior. Add focused regression coverage where defaults.primary differs from heartbeat.model, a heartbeat follow-up still uses heartbeat.model, and a later normal user turn still uses the primary model. Avoid new config options, migrations, and broad model-selection refactors. Add a changelog fix entry because this is user-visible billing/model routing behavior.
