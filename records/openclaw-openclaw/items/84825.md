## Summary

Keep open: current main still has heartbeat-origin follow-up paths that can re-enter agent execution without heartbeat model/origin metadata. The initial heartbeat turn is correctly marked, but exec-approval and queued embedded follow-ups do not carry a complete heartbeat contract.

## What This Changes

The issue requests that all follow-up runs inside a heartbeat session continue using `agents.defaults.heartbeat.model` instead of falling back to the primary model.

## Best Possible Solution

Define one heartbeat-origin follow-up contract and apply it across exec-approval, Gateway agent, queued follow-up, model selection, trigger metadata, and session accounting so heartbeat follow-ups stay heartbeat-scoped without changing normal user turns.

## Reproduction Assessment

Yes at source level: a heartbeat exec-approval follow-up can call Gateway `agent` without heartbeat/model override metadata, and Gateway agent model selection then starts from the normal default or stored session model. I did not run a live heartbeat gateway repro in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow contract repair: carry heartbeat origin and the resolved heartbeat model through follow-up entry points, and add regressions for exec approval plus queued embedded follow-ups. A new config option is not needed.
