## Summary

Keep open: the report is detailed and the related closed heartbeat-model bugs cover adjacent directions, not this follow-up-origin loss. Current main has initial heartbeat override handling and partial heartbeat trigger propagation, but it still lacks a follow-up-run heartbeat-origin contract/regression, and the queued embedded follow-up path still dispatches as a user-triggered run.

## What This Changes

The issue asks for every billed turn inside a heartbeat session, including retry/fallback/queued follow-up runs, to keep using the configured `agents.defaults.heartbeat.model`.

## Best Possible Solution

Add a focused regression for heartbeat-originated follow-up/retry paths, then carry heartbeat origin and model override through the follow-up run contract without changing normal user-turn model persistence.

## Solution Assessment
