## Summary

Keep open for maintainer review: current main still routes unproven bare Feishu `oc_...` outbound targets as direct, while this PR makes a focused Feishu-plugin fix for the delivery-mirror/group-session case. The patch looks correct and the resolver-level proof is sufficient, but the PR is draft/protected and the changed route decision remains session-state and message-delivery sensitive.

## What This Changes

The PR adds Feishu plugin route logic and tests so bare `oc_...` outbound targets become group routes only when directory evidence or the current Feishu group session proves that route.

## Best Possible Solution

Keep this as a focused Feishu plugin resolver fix, with maintainer acceptance of the scoped resolver proof or a request for live Feishu transport proof before merge; leave the broader inbound/proactive cleanup on separate canonical work.

## Reproduction Assessment

## Solution Assessment

Yes for the scoped mirror-route bug: the Feishu plugin resolver is the right owner boundary, and the PR preserves the compatibility rule from https://github.com/openclaw/openclaw/pull/10407 by requiring current-session or directory proof before promoting `oc_...` to group. It is not a full solution for proactive context-free sends, duplicate cleanup, or https://github.com/openclaw/openclaw/issues/52238.
