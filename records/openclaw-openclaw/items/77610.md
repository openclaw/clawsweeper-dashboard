## Summary

This should stay open: current `main` still routes host `exec` through repeated child-process spawns, and the existing EBADF fallback only retries a failed detached spawn as non-detached rather than addressing the reported accumulating macOS failure. The report is concrete enough for a focused macOS reproduction and repair pass.

## What This Changes

The issue reports Gateway host `exec` failing with `spawn EBADF` after roughly 5-10 repeated simple commands on macOS, recovering only after Gateway restart.

## Best Possible Solution

Add a macOS-proven exec spawn recovery in the supervisor/spawn utility, with regression coverage and a changelog entry, without adding periodic Gateway restarts as product behavior.

## Reproduction Assessment

No live high-confidence reproduction was established in this review. The source path is clear and the issue gives concrete macOS steps, but the next proof should run a macOS Gateway loop with repeated simple `exec` calls on the reported Node version.

## Solution Assessment

Unclear until macOS reproduction confirms the failing mode. The likely maintainable path is a narrow supervisor/spawn recovery change, while the reported periodic restart workaround should stay a temporary operational workaround, not the product fix.
