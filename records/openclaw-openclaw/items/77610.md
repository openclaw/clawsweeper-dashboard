This should stay open: the report targets the current shipped release, and the relevant exec/spawn files have no main-branch change after that release that would clearly fix the macOS EBADF pattern.

## What This Changes

The issue reports macOS `exec` tool failures with `spawn EBADF` after several process spawns, suspected to involve libuv/kqueue handle leakage.

## Best Possible Solution

Reproduce on macOS against current main, then either make Darwin exec child spawns avoid the leaking mode or add a stronger spawn fallback with regression coverage around the process supervisor boundary.
