## Summary

This should stay open: current main still routes host exec through repeated process-supervisor child spawns, and the only EBADF-specific handling is a single retry with supplied fallback options rather than a macOS repeated-spawn recovery.

## What This Changes

The issue reports macOS Gateway exec failures where repeated host exec calls start throwing `spawn EBADF` after roughly 5-10 successful spawns.

## Best Possible Solution

Add a macOS-proven exec spawn recovery in the supervisor/spawn path with regression coverage and a changelog entry, keeping periodic Gateway restarts as an operational workaround only.

## Solution Assessment
