## Summary

This should stay open: current main has Antigravity-related provider-policy handling, but the bundled Google CLI backend, manifest, setup surface, and docs still only register `google-gemini-cli` with the local `gemini` binary. The remaining work is a feature/migration decision because maintainers need to choose bundled vs external plugin ownership, auth/profile behavior, and fallback policy before implementation.

## What This Changes

Request to add an Antigravity CLI (`agy`) backend for Google CLI-style agentic model access while preserving existing Gemini CLI support.

## Best Possible Solution

Add a provider-owned Antigravity CLI backend or official external plugin through the existing CLI backend SDK, preserve `google-gemini-cli` for enterprise users, and land it with manifest/setup/docs updates plus real `agy` smoke proof.
