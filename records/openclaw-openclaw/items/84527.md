## Summary

Keep open: current main and the latest shipped tag still register only the Gemini CLI backend for Google, while the requested `agy`/`antigravity-cli` backend is a new provider/runtime migration that needs maintainer policy on ownership, auth, and fallback behavior before implementation.

## What This Changes

The issue requests adding an Antigravity CLI (`agy`) backend for Google agentic CLI access while preserving `google-gemini-cli` for enterprise users.

## Best Possible Solution

Decide whether Antigravity CLI belongs in the bundled Google plugin or an official external plugin, then add the backend with documented auth/setup/config behavior, compatibility-preserving routing, focused tests, and a real `agy` smoke for prompt, resume, MCP/tool, and attachment paths.

## Solution Assessment
