## Summary

This should stay open: current main still only registers the Gemini CLI backend for Google, while the upstream Antigravity transition is real and affects the existing bundled Google CLI path. It is a feature and migration decision rather than a small bug fix, because maintainers need to choose bundled-vs-plugin ownership, auth handling, fallback behavior, and live proof requirements.

## What This Changes

The issue requests adding Antigravity CLI (`agy`) as a Google CLI backend while keeping the existing Gemini CLI backend for enterprise users.

## Best Possible Solution

Add a provider-owned Antigravity CLI backend or official external plugin using the existing CLI backend SDK, with manifest/setup/auth/docs updates, live `agy` proof, and a compatibility policy that preserves `google-gemini-cli` for enterprise users.
