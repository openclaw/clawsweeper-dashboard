## Summary

Keep open. Current main and the latest release still emit colon-delimited Feishu card-note metadata into Markdown card elements, and the open linked fix PR is unmerged and currently conflicting rather than resolving this issue.

## What This Changes

This issue asks for Feishu card notes to stop using colon-delimited `Agent`/`Model`/`Provider` footer text, or to provide a way to disable that footer, because Feishu renders it as definition-list syntax.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/84122 as the narrow separator fix with regression coverage and redacted Feishu proof, while leaving broader header/footer configurability to https://github.com/openclaw/openclaw/issues/60748.

## Reproduction Assessment

Yes, at source level: current main emits `Agent: agent` style note text into Feishu Markdown card elements in both static and streaming paths. A live Feishu send was not run in this read-only review.

## Solution Assessment

Yes for this issue boundary: changing the Feishu card-note separator is narrower than adding a new public footer configuration surface. The open linked PR follows that direction but needs conflict repair and real behavior proof before merge.
