## Summary

Keep open. Current main and v2026.6.10 still emit the colon-delimited Feishu card note into Markdown card paths, and the linked closing PR remains open/conflicting rather than merged.

## What This Changes

The issue reports that Feishu card-mode replies render a fixed Agent/Model/Provider note footer with colon labels that Feishu Markdown parses as a definition list, hoisting the agent name.

## Best Possible Solution

Rebase, prove, and land or replace https://github.com/openclaw/openclaw/pull/84122 for the narrow separator bug, while keeping broader footer configurability under https://github.com/openclaw/openclaw/issues/60748.

## Reproduction Assessment

No high-confidence live reproduction is attached. Source inspection shows current main emits the exact colon-delimited note text into Feishu Markdown card paths, but the renderer hoist still needs redacted Feishu/Lark proof.

## Solution Assessment

Yes for the narrow bug: a Markdown-safe separator in the shared Feishu card-note formatter is the smallest compatibility-preserving fix. A new footer configuration surface is broader product work and should stay with the related Feishu card presentation tracker.
