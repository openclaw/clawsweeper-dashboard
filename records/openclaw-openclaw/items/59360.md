## Summary

Keep open. Current main and latest release still generate the colon-delimited Feishu card-note footer, the linked separator fix remains open and conflicting, and the broader configurable-footer work does not solve the default generated footer when unset.

## What This Changes

The issue asks OpenClaw to stop Feishu card Markdown from hoisting the generated Agent/Model/Provider footer, either by changing the footer format or adding a way to disable it.

## Best Possible Solution

Land a rebased and proven Feishu plugin fix that removes colon key/value formatting from the generated footer, while keeping broader header/footer configurability in https://github.com/openclaw/openclaw/issues/60748.

## Reproduction Assessment

No high-confidence live reproduction is attached. Source inspection shows current main emits the exact colon-delimited note text into Feishu Markdown card paths, but the external renderer hoist still needs redacted Feishu/Lark proof.

## Solution Assessment

Yes for the narrow bug: changing the generated footer separator is the smallest compatibility-preserving fix. Adding a new footer/header configuration surface is broader product work and should stay under the related configurability tracker.
