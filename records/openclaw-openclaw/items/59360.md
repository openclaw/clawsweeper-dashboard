## Summary

Keep open: current main and the latest release still emit colon-delimited Feishu card-note labels, and the open closing PR has not merged yet.

## What This Changes

Prevent Feishu card note footer labels like `Agent: main` from being interpreted as Markdown definition-list syntax and hoisted into the rendered message.

## Best Possible Solution

Land a narrow Feishu plugin fix that replaces colon key/value footer labels across streaming and static card paths, updates all affected tests, and includes redacted real Feishu proof; keep broader footer customization under https://github.com/openclaw/openclaw/issues/60748.

## Reproduction Assessment

No live high-confidence reproduction was established; source inspection shows current main emits the exact colon-delimited footer into Feishu markdown card notes. A redacted Feishu screenshot, recording, or log would complete the real renderer proof.

## Solution Assessment

Yes for the narrow bug path: changing the Feishu plugin formatter away from colon key/value syntax is the smallest maintainable fix. Adding a new disable/customization config is broader product work already tracked by https://github.com/openclaw/openclaw/issues/60748.
