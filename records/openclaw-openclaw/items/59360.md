## Summary

Keep open. Current main still emits colon-delimited `Agent`/`Model`/`Provider` text into Feishu card notes and renders those notes as Feishu Markdown; the related header/footer customization tracker and header-only PR do not supersede this narrow formatter bug.

## What This Changes

The issue asks Feishu card replies to stop using a colon-delimited Agent/Model/Provider footer that Feishu Markdown can parse into visible definition-list content.

## Best Possible Solution

Change the Feishu plugin’s default card-note label format to avoid colon-delimited Markdown across streaming and static cards, with broader footer customization left to https://github.com/openclaw/openclaw/issues/60748.

## Reproduction Assessment

Yes, at source level. Current main emits `Agent: agent` into Feishu card notes that are rendered as Markdown; no live Feishu send was run in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to reformat the Feishu card-note labels and update focused tests, instead of adding a broader footer config API for this bug.
