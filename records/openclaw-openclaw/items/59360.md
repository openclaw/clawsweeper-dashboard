## Summary

Keep open. Current main still emits colon-delimited `Agent`/`Model`/`Provider` text into Feishu card notes that are rendered as Markdown, and the broader Feishu footer customization tracker does not replace this narrow formatter bug.

## What This Changes

The issue requests fixing Feishu card note footers so `Agent: ... | Model: ... | Provider: ...` is not parsed as Markdown definition-list-like content, either by changing the default format or adding a footer toggle.

## Best Possible Solution

Change the Feishu plugin’s default card-note label format to avoid colon-delimited Markdown across static and streaming cards, and leave broader footer customization to https://github.com/openclaw/openclaw/issues/60748.

## Reproduction Assessment

Yes, at source level: current main emits `Agent: agent` style note text into Feishu Markdown card elements in both static and streaming card paths. A live Feishu send was not run in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to reformat the default Feishu card-note labels and update focused tests, instead of adding a new public footer configuration API for this bug.
