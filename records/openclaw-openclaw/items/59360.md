## Summary

Keep open. Current main still builds Feishu card notes with colon-delimited Agent/Model/Provider labels and renders those notes through Feishu Markdown, so the reported formatter bug remains valid and narrow.

## What This Changes

The issue asks OpenClaw to stop Feishu card footer labels such as `Agent:` from being interpreted by Feishu Markdown as definition-list syntax.

## Best Possible Solution

Change the Feishu plugin’s default card-note label format so it avoids colon-delimited Markdown while preserving useful agent/model/provider context, then add focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits `Agent: agent` into Feishu card notes rendered as Markdown; the live Feishu renderer was not exercised in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to adjust the Feishu note formatter and tests, while leaving broader footer customization to https://github.com/openclaw/openclaw/issues/60748.
