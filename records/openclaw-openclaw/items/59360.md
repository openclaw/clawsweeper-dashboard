## Summary

Keep open. Current main still builds Feishu card footers as `Agent: ... | Model: ... | Provider: ...`, passes that note into both streaming and static card sends, and exposes no Feishu footer toggle in the strict config schema or public Feishu docs. The related footer-customization reports are broader trackers, but this issue has a narrow, reproducible Markdown-rendering bug that can be fixed directly in the Feishu plugin.

## What This Changes

The issue requests a way to disable or reformat the Feishu card note footer so Feishu Markdown does not treat `Agent: ...` style text as a definition list.

## Best Possible Solution

Fix this in the Feishu plugin by changing the default card note format to avoid Markdown definition-list syntax, ideally without adding configuration unless maintainers want footer visibility as a separate product option. Keep the card note supported for model/provider context, update focused tests, document any new config only if one is added, and add a user-facing changelog entry crediting the reporter.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current tests and dispatcher code show Feishu card replies receive `note: "Agent: agent"`, and the latest release tag contains the same formatter the reporter quoted.

## Solution Assessment

Yes, a narrow Feishu-owned formatter change is the best first fix. A broader footer customization API can remain tracked by the related feature issues, but this bug does not need a core seam or a full customization surface to stop the Markdown parsing problem.
