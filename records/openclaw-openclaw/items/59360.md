## What This Changes

The issue asks OpenClaw to stop Feishu card note footer text like `Agent: main | Model: ... | Provider: ...` from being parsed as a markdown definition list, either by changing the format or making the footer disableable.

## Best Possible Solution

Land one Feishu-plugin fix that prevents the card note footer from being parsed as a definition list across static and streaming card paths, with focused tests and redacted real Feishu/Lark proof.

## Reproduction Assessment
