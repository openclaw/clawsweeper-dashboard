## Summary

Keep open. Current main and the latest shipped release still emit colon-delimited Feishu card-note metadata into Markdown card elements, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to stop Feishu card footer metadata such as `Agent: main | Model: ... | Provider: ...` from being parsed as Markdown definition-list text.

## Best Possible Solution

Land a narrow Feishu plugin formatter fix for static and streaming card notes with focused regression tests and redacted Feishu proof, while leaving broader header/footer configurability to https://github.com/openclaw/openclaw/issues/60748.

## Reproduction Assessment

Yes, at source level: current main emits `Agent: agent` style note text into Feishu Markdown card elements in both static and streaming paths. A live Feishu send was not run in this read-only review.

## Solution Assessment

Yes for the issue boundary: changing the default Feishu card-note label separator is narrower than adding a new public footer configuration surface. The open linked PR follows that direction but must be repaired and proved before merge.
