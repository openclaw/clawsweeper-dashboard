## Summary

Keep open. Current main still emits the Feishu card note as colon-delimited Markdown, latest release v2026.5.7 still ships the same formatter, and the open related footer issues are broader customization trackers rather than a merged fix for this Markdown-rendering bug.

## What This Changes

The issue asks OpenClaw to stop Feishu card note footers like `Agent: ... | Model: ... | Provider: ...` from being parsed by Feishu Markdown as definition-list text at the start of the card.

## Best Possible Solution

Change the Feishu plugin’s default card-note labels or separators so they avoid colon-delimited Markdown while preserving useful agent/model/provider context, then add focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.7 both send `note: "Agent: agent"` into Feishu card Markdown elements; a live Feishu renderer was not exercised in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Feishu plugin formatter and tests; broader footer customization can stay on the existing feature trackers.
