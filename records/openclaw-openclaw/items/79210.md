## Summary

Keep open: current main deliberately disables xAI thinking controls for native Grok Responses requests, while the new report asks to enable a provider capability that is not implemented today.

## What This Changes

Allow Grok 4.3 users to control reasoning effort through OpenClaw's model/thinking configuration.

## Best Possible Solution

Update the xAI provider-owned thinking profile and payload compatibility so Grok 4.3 can use existing `/think` and model params after live validation, while preserving suppression for unsupported Grok routes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main cannot send Grok 4.3 reasoning effort because xAI thinking is off-only and outgoing reasoning keys are stripped.

## Solution Assessment

Unclear until live xAI validation confirms the current API accepts the documented payload. The maintainable direction is provider-owned support through the existing thinking/profile machinery, not a new core config surface.
