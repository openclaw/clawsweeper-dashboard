## Summary

Keep open: current main and the latest release still list node-role paired devices in `nodes status`, while `nodes remove` only deletes node-pair records, matching the reported Android `unknown nodeId` failure.

## What This Changes

The issue asks for `openclaw nodes remove --node <id>` to remove Android nodes shown by `openclaw nodes status`.

## Best Possible Solution

Make `nodes remove` remove any paired node row that `nodes status` can select, including Android device-backed node rows, while reusing the existing device-pairing authorization boundary.

## Reproduction Assessment

Yes, source-reproducible with high confidence: Android node rows can be backed by device pairing, `node.list` displays those IDs, and `node.pair.remove` only checks node-pairing state. I did not run a live Android phone repro in this read-only pass.

## Solution Assessment

Yes: a targeted gateway fix that resolves removal through the same known-node surface as status, then removes the matching node-pair or device-pair record with existing authz, is the narrow maintainable path. A CLI-only workaround would leave the gateway API inconsistent.
