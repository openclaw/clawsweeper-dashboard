## Summary

## What This Changes

The issue reports that an Android node shown by `openclaw nodes status` cannot be removed with `openclaw nodes remove --node <id>` because the gateway returns `unknown nodeId`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90373, or an equivalent gateway-layer fix, so `node.pair.remove` can remove the same Android/device-backed node records that `node.list` shows while preserving mixed-role device credentials correctly.

## Reproduction Assessment

Yes. The reporter's command output plus current source show a node ID displayed by `nodes status` can be backed by device pairing while `node.pair.remove` only deletes the legacy node-pairing store.

## Solution Assessment

Yes, with maintainer acceptance of the credential semantics. The fix belongs in the gateway handler/device-pairing boundary, not as a CLI-only workaround, because `node.list` and `node.pair.remove` currently disagree about the backing store.
