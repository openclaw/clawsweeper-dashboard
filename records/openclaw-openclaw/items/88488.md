## What This Changes

The issue reports that Android device-backed nodes shown by `openclaw nodes status` cannot be removed with `openclaw nodes remove --node <id>`.

## Best Possible Solution

Land the linked gateway fix, or an equivalent maintainer-approved handler change, so a displayed node row can be removed through the same identity boundary that made it appear in `nodes status`.

## Reproduction Assessment

Yes. The reporter's command output plus current source show a node ID displayed by `nodes status` can be backed by device pairing while `node.pair.remove` only deletes the legacy node-pairing store.

## Solution Assessment

Yes, with maintainer acceptance of the credential semantics. The fix belongs in the gateway handler/device-pairing boundary, not as a CLI-only workaround, because `node.list` and `node.pair.remove` currently disagree about the backing store.
