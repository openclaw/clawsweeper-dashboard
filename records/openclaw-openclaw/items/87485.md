## Summary

Keep open: current main still assigns fixed daily and session ingestion scores before averaging them into memory-entry confidence, so the reported non-informative score distribution remains real. The requested fix is not a safe automatic bug lane because replacing the constants with a content-dependent heuristic changes memory promotion and pruning policy.

## What This Changes

The issue asks for light-sleep memory ingestion to use content-dependent scoring instead of fixed daily/session constants.

## Best Possible Solution

Define an owner-approved ingestion scoring heuristic for daily memory and session corpus signals, then update the light-sleep tests to assert a meaningful score distribution rather than fixed constants.

## Reproduction Assessment

Yes from source inspection: current main still assigns fixed 0.62 and 0.58 ingestion scores and averages signal scores, so entries with only ingestion signals cannot differentiate by content. I did not run a live workspace reproduction because this read-only sweep is enough to confirm the code path.

## Solution Assessment

Unclear: replacing the constants is likely the right direction, but the best scoring formula needs maintainer product judgment because it changes memory promotion priority and pruning behavior.
