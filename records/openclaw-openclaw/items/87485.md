## Summary

Keep open: current main and latest release still use fixed daily and session ingestion scores that flow into light-sleep confidence and REM scoring, and the repair needs a maintainer decision on the content-aware scoring policy and existing stored scores.

## What This Changes

The issue asks memory-core dreaming ingestion to replace fixed daily/session scores with content-dependent scoring so light-sleep and REM confidence can differentiate entries.

## Best Possible Solution

Approve a content-aware ingestion scoring policy, decide the persisted-score/backfill behavior, and update memory-core implementation plus tests around score distribution.

## Reproduction Assessment

Yes. Source inspection shows current main still assigns fixed 0.62 and 0.58 ingestion scores and then averages signal scores, so ingestion-only entries cannot differentiate by content.

## Solution Assessment

Unclear. Replacing the constants is likely the right direction, but the scoring formula and persisted-entry policy need maintainer product judgment because they change memory ranking behavior.
