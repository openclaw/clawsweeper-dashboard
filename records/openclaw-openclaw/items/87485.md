## Summary

Keep open: current main and the latest stable release still assign fixed 0.62/0.58 ingestion scores that flow into light confidence, REM confidence, and deep relevance scoring, while the replacement scoring policy needs maintainer judgment.

## What This Changes

The issue requests content-dependent light-sleep ingestion scores for daily memory and session corpus entries instead of constant 0.62/0.58 score inputs.

## Best Possible Solution

Maintainers should define the memory-core ingestion scoring contract, choose the persisted-score handling, then update dreaming-phases, short-term promotion tests, and docs around the approved distribution behavior.

## Reproduction Assessment

Yes, at source level. Current main still assigns fixed 0.62/0.58 scores during daily/session ingestion, then averages those scores for light confidence and downstream REM/deep scoring; no live workspace run was performed in this read-only review.

## Solution Assessment

Unclear as an implementation direction. Replacing the constants is likely the right repair area, but the exact scoring formula and persisted-entry policy need maintainer product judgment before a safe fix PR.
