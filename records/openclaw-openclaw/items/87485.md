## Summary

Keep open: current main and the latest release still assign fixed daily/session ingestion scores, and those scores feed light-sleep and REM confidence. The report identifies a real memory-core ranking quality problem, but the exact replacement heuristic needs maintainer direction because it changes promotion priority.

## What This Changes

Replace memory-core dreaming daily/session ingestion constants with content-dependent scoring so light-sleep and REM confidence can differentiate entries.

## Best Possible Solution

Define a deterministic memory-core ingestion scoring helper for daily and session chunks, then add regression coverage proving different chunk shapes no longer collapse to identical confidence when organic recall is absent.

## Reproduction Assessment

Yes from source inspection: daily chunks are always scored 0.62 and session corpus entries 0.58, and those values feed light/REM confidence. I did not run a write-producing repro because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the scoring heuristic. The maintainable fix should be internal, deterministic, and tested; it should not add a user-facing config knob unless maintainers explicitly want tunable policy.
