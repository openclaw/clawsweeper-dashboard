## Label Justifications

- `impact:session-state`: The affected scores live in short-term memory/dreaming state and can steer future memory consolidation and agent context.

## Summary

Keep open: current main and v2026.6.6 still assign fixed daily/session ingestion scores that feed light-sleep confidence and REM ranking, so the central reported problem remains. The source-level repro is strong, but the replacement scoring heuristic is product-sensitive because it changes memory promotion priority.

## What This Changes

Replace fixed daily/session dreaming ingestion scores with content-dependent scoring so light-sleep and REM confidence do not collapse to uniform values.

## Best Possible Solution

Define one deterministic internal ingestion scoring helper for daily and session chunks, then update light/REM/promotion tests and docs so content shape can affect confidence without adding a user-facing config knob by default.

## Reproduction Assessment

Yes. Source inspection on current main shows daily chunks always score 0.62 and session corpus entries always score 0.58, and those values feed light-sleep confidence plus REM confidence; I did not run a write-producing repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the scoring heuristic. The maintainable fix should be deterministic and internal, with regression coverage for differentiated chunk shapes, rather than a new config surface unless maintainers explicitly want tunable policy.
