## Summary

Keep open: current main and latest release v2026.6.8 still assign fixed daily and session ingestion scores before averaging them into light-sleep and REM confidence. The report and the second workspace confirmation remain source-backed, but the repair needs maintainer approval for the scoring heuristic and existing persisted-score handling.

## What This Changes

The issue requests replacing fixed light-sleep ingestion scores with content-sensitive scoring so memory entries receive differentiated confidence.

## Best Possible Solution

Approve a content-aware ingestion scoring policy, decide stored-score migration/backfill behavior, and update memory-core implementation plus tests around score distribution.

## Reproduction Assessment

Yes, from source inspection: current main still assigns fixed 0.62 and 0.58 ingestion scores and then averages signal scores, so ingestion-only entries cannot differentiate by content. I did not run a live multi-week workspace replay.

## Solution Assessment

Unclear: replacing the constants is likely the right direction, but the best formula and persisted-entry policy need maintainer product judgment because they change memory ranking behavior.
