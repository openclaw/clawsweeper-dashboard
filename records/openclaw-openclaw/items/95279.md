## Summary

Keep open. Current main and the latest release still rely on forgeable sentinel text for inbound metadata stripping, and two open linked PRs are competing candidate fixes for the trusted contract.

## What This Changes

The issue requests a trusted out-of-band inbound-decoration marker and bare-body contract so OpenClaw and plugins can strip or dedupe user turns without parsing forgeable metadata headers.

## Best Possible Solution

Land one maintainer-approved additive, marker-gated trusted inbound-decoration contract with focused coverage for transcript persistence, UI extraction, CLI dedupe, memory export, historical replay, redaction boundaries, and legacy fallback.

## Reproduction Assessment

Yes. Source inspection shows current main builds inbound metadata as literal sentinel text and multiple user-role consumers strip or compare those same literal headers, so a user-authored matching block follows the same path as host decoration.

## Solution Assessment

Yes as a direction, but not yet as a specific implementation. A trusted out-of-band marker plus bare body is the right layer; maintainers still need to choose the durable field shape and trust/redaction boundary through one linked PR.
