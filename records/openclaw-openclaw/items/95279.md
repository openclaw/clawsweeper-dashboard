## Summary

Keep open. Current main and the latest release still rely on forgeable sentinel text for inbound metadata stripping, and open linked PRs already propose competing trusted-contract implementations that need maintainer selection rather than another cleanup close.

## What This Changes

The issue requests a trusted out-of-band inbound decoration marker and bare-body contract so consumers can strip and dedupe without parsing user-controlled metadata-looking text.

## Best Possible Solution

Land one maintainer-approved additive contract that marks decorated inbound user turns out of band, preserves the host-known bare body for trusted consumers, keeps legacy sentinel fallback for old rows, and closes this issue through the merged PR.
