- `impact:session-state`: The affected consumers reconstruct user turns for transcript history, replay, memory indexing, and imported-session dedupe.

## Summary

Keep open. Current main and v2026.6.9 still rely on forgeable sentinel-text stripping without a trusted inbound-decoration marker, and two open closing PRs are already active candidate fixes, so this should stay tracked rather than be closed or queued as a new fix lane.

## What This Changes

The issue requests a trusted inbound-decoration marker plus bare-body contract so UI, history, memory, replay, and plugin consumers do not strip or dedupe user content by parsing forgeable metadata sentinels.

## Best Possible Solution

Land one marker-gated, host-owned inbound-decoration contract for current and persisted user turns, update the affected UI/history/memory/replay/import consumers to prefer it, and keep sentinel stripping only as a legacy fallback for unmarked old rows.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: user-role text containing an exact inbound metadata sentinel plus a JSON fence follows the same strip/dedupe path as OpenClaw-added decoration in multiple consumers.

## Solution Assessment

Yes as a direction, but not yet as a settled implementation. A trusted out-of-band marker and bare-body contract is the right layer; maintainers still need to choose the durable field shape and fallback boundary across the open PRs.
