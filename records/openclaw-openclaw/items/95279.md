## Summary

Keep open. Current main and v2026.6.10 still rely on forgeable sentinel text to strip inbound metadata, and two open linked PRs propose competing trusted-contract shapes that should be resolved before this issue closes.

## What This Changes

The issue requests a trusted out-of-band inbound-decoration marker plus bare-body contract so UI, history, memory, replay, and dedupe consumers do not parse user-controlled sentinel text.

## Best Possible Solution

Land one maintainer-approved additive contract that marks decorated inbound user turns out of band, carries the trusted bare body for consumers, preserves legacy sentinel fallback only for old rows, and then close this issue through the merged PR.

## Reproduction Assessment

Yes, from source inspection. Current main decorates inbound content as sentinel text and strips user-role text by the same literal headers, so a user-authored sentinel-shaped body can follow the same path as host decoration.

## Solution Assessment

Yes on direction, unclear on final shape. A trusted out-of-band marker plus bare body is the right layer, but the permanent field location and redaction/upgrade behavior should be settled through one linked PR.
