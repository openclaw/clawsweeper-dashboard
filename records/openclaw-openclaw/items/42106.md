- `impact:message-loss`: Paragraph separators can be silently dropped from delivered channel text across streamed updates.

## Summary

Keep open: current main and v2026.5.20 still consume paragraph separators in the shared block-streaming path, and the linked fix candidate https://github.com/openclaw/openclaw/pull/73981 is open and unmerged. This issue remains the right tracker until that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw block streaming to preserve paragraph separators across streamed deliveries so reconstructed Markdown keeps blank-line structure.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73981 with a shared fix that carries boundary semantics through chunking, subscribe delivery, coalescing, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits before paragraph separators, skips leading newlines after the separator, and trims delivered block text before downstream reconstruction.

## Solution Assessment

Yes. The shared block-streaming pipeline is the right place to fix this because channel-specific joiners cannot infer paragraph boundaries after plain text deliveries have already been flushed.
