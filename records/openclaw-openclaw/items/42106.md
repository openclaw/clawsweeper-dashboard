## Summary

Keep open: current main and v2026.6.10 still consume paragraph separators in the shared block-streaming path, and the two open closing PRs are candidate fixes rather than merged implementation.

## What This Changes

The issue reports that block-streamed replies can lose blank-line paragraph separators when multi-paragraph Markdown is delivered across multiple streamed updates.

## Best Possible Solution

Land one consolidated shared chunker/subscriber/coalescer fix that preserves paragraph boundaries across streamed deliveries without doubled separators or over-limit chunks, then close this issue from the merged PR.

## Reproduction Assessment

Yes, source-level. Current main emits a paragraph chunk before the separator, advances past the matched newlines, and downstream coalescing can only add joiners while chunks stay inside one buffered flush.

## Solution Assessment

Yes. The best fix belongs in the shared chunker/subscriber/coalescer boundary; a channel-specific join rule cannot reliably distinguish paragraph boundaries from size or timing splits.
