## What This Changes

The issue asks OpenClaw to preserve paragraph separators when block-streamed reply text is split across multiple visible deliveries.

## Best Possible Solution

Review and land one shared chunker/coalescer fix that preserves paragraph boundary semantics across normal and maxChars split cases, then let the issue close from the merge.

## Reproduction Assessment

Yes, source-level. Current main emits a paragraph chunk before the separator and advances past the consumed newlines, while the coalescer can only insert joiners before a flush boundary.

## Solution Assessment

Yes. The right fix belongs in the shared chunker/coalescer boundary; a channel-specific downstream join rule cannot distinguish paragraph boundaries from size or timing splits.
