## Summary

Keep open: current main still has the separator-consuming block-streaming path, and the open closing PR is an active fix candidate rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to preserve Markdown paragraph separators when block-streamed replies are reconstructed across multiple visible updates.

## Best Possible Solution

Land or replace the shared streaming fix so boundary semantics survive chunking, subscribe delivery, coalescing, and regression coverage for multi-paragraph Markdown.

## Reproduction Assessment

Yes, at source level. Current main emits before paragraph separators, strips leading newlines from the remainder, and trims delivered block text before downstream reconstruction.

## Solution Assessment

Yes. The shared block-streaming pipeline is the right place to fix this because channel-specific joiners cannot infer paragraph boundaries after plain text deliveries have already been flushed.
