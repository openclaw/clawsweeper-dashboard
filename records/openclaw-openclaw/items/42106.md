## What This Changes

The issue reports lost Markdown paragraph separators when block-streamed deliveries are reconstructed across flush boundaries.

## Best Possible Solution

Preserve explicit paragraph/newline boundary semantics in the shared block-streaming pipeline, then land or replace the linked repair PR with focused cross-flush reconstruction coverage.

## Reproduction Assessment

Yes from source inspection: the paragraph-flush branch emits before the separator and removes that separator from the next buffered text, so separate flushed deliveries can concatenate without the blank line. A live channel reproduction was not run during this read-only review.

## Solution Assessment

Yes in direction: the durable fix belongs in the shared chunker or delivery boundary metadata, not in channel-specific joining. The linked PR is the current candidate, but it needs maintainer review because it is conflicting and includes related table-boundary scope.
