## What This Changes

The issue reports that block streaming can lose blank-line paragraph separators when streamed deliveries are reconstructed across flush boundaries.

## Best Possible Solution

Review and land one narrow shared chunker/coalescer fix that preserves paragraph separators across flushed deliveries without doubling coalesced joiners, then let the linked issue close from the merge.

## Reproduction Assessment

Yes. Source inspection shows a paragraph-split chunk can be emitted before `\n\n` while the buffer start moves past that separator, leaving no downstream metadata after a flush.

## Solution Assessment

Yes. The best fix belongs in the shared chunker/coalescer boundary; a channel-specific join rule would corrupt non-paragraph flush boundaries.
