## Label Justifications

- `P2`: This is a reproducible user-visible Markdown corruption bug in block-streamed replies with limited blast radius.
- `impact:message-loss`: Paragraph separators can be silently dropped from delivered channel text across streamed updates.

## Summary

Keep open: current main and the latest release still consume or trim the paragraph separator in the shared block-streaming path, and the linked closing fix is still open and unmerged.

## What This Changes

The issue reports lost paragraph separators when block-streamed reply chunks are flushed across multiple visible updates.

## Best Possible Solution

Preserve boundary semantics through the shared chunker, subscribe delivery, coalescing, and focused regression coverage, then close this issue when https://github.com/openclaw/openclaw/pull/73981 or a replacement lands.

## Reproduction Assessment

Yes, at source level. Current main emits before the paragraph separator, skips leading newlines after it, and trimEnds delivered block reply text before downstream reconstruction.

## Solution Assessment

Yes. The shared block-streaming pipeline is the right place to fix this because channel-specific joiners cannot infer paragraph boundaries after plain text deliveries have already been flushed.
