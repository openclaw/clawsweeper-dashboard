- `P2`: This is a reproducible user-visible Markdown corruption bug in block-streamed replies with limited blast radius.
- `impact:message-loss`: Paragraph separators can be silently dropped from delivered channel text across streamed updates.

## Summary

Keep open: current main and the latest release still lose paragraph-boundary semantics in the shared block-streaming path, and the linked closing PR remains open rather than merged.

## What This Changes

The issue requests preserving Markdown paragraph separators across block-streamed reply deliveries so reconstructed visible text does not collapse multi-paragraph structure.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/73981 or a narrow replacement that preserves boundary semantics through chunking, delivery, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits before the paragraph separator, skips the separator when advancing the chunk buffer, and trims delivered block reply text before downstream reconstruction.

## Solution Assessment

Yes. The shared block-streaming pipeline is the right fix surface because channel-specific joiners cannot infer paragraph boundaries after plain text deliveries have already been flushed.
