- `impact:message-loss`: Paragraph separators can be silently dropped from delivered channel text across streamed updates.

## Summary

Keep open: current main and the latest release still consume paragraph-boundary separators in the shared block-streaming path, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that block-streamed replies can lose blank-line paragraph separators when multi-paragraph Markdown is split across streamed deliveries.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73981, or a narrow replacement, so boundary semantics survive chunking, subscribe delivery, coalescing, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits before the paragraph separator, skips leading newlines after it, and trims delivered block reply text before downstream reconstruction.

## Solution Assessment

Yes. The shared block-streaming pipeline is the right fix surface because channel-specific joiners cannot infer paragraph boundaries after plain text deliveries have already been flushed.
