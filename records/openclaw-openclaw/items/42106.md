## Summary

Keep this open: current main and the latest release still consume the paragraph separator in the shared block-streaming chunker path, while two open closing PRs are active candidate fixes rather than merged implementation.

## What This Changes

The issue reports that shared block streaming can consume paragraph separators when paragraph-delimited chunks flush across multiple deliveries, collapsing Markdown structure in reconstructed visible replies.

## Best Possible Solution

Review and land one shared chunker/coalescer fix that preserves paragraph boundary semantics across streamed deliveries, then let this issue close from the merged PR.

## Reproduction Assessment

Yes, source-level. Current main emits a paragraph chunk before the separator and advances past the matched newlines, while the coalescer can only insert joiners before a flush boundary.

## Solution Assessment

Yes. The best fix belongs in the shared chunker/coalescer boundary; a channel-specific downstream join rule cannot distinguish paragraph boundaries from size or timing splits.
