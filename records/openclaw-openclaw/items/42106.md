## Label Justifications

- `P2`: This is a reproducible user-visible Markdown corruption bug in block-streamed replies with limited blast radius.
- `impact:message-loss`: Paragraph separators can be silently dropped from delivered channel text across streamed updates.

## Summary

Keep open: current main and the latest release still consume paragraph-boundary separators in the shared block-streaming path, and the linked closing PR remains open rather than merged.

## What This Changes

The issue requests preserving paragraph separators when block-streamed replies are split across multiple visible deliveries.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/73981, or a narrow replacement, so boundary semantics survive chunking, subscribe delivery, coalescing, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits before the paragraph separator, skips leading newlines after it, trims delivered block reply text, and only coalesces joiners while updates remain in one buffered flush.

## Solution Assessment

Yes. The shared block-streaming pipeline is the right fix surface because channel-specific joiners cannot infer paragraph boundaries after plain text deliveries have already been flushed.
