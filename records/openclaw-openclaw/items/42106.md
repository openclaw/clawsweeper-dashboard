## Summary

Current main still appears to have the separator-consuming block streaming path, and open PR #73981 already closes this issue, so the issue should remain open until that PR is landed or replaced.

## What This Changes

The issue reports that block streaming can lose paragraph separators when a streamed reply is reconstructed across multiple visible updates.

## Best Possible Solution

Land or replace PR #73981 with a narrow fix that preserves paragraph/newline boundary semantics through chunking, coalescing, and delivery, with regression coverage for multi-paragraph Markdown split across flushes.

## Reproduction Assessment

Yes, from source inspection rather than a live test run: current main emits chunks without the selected separator and then skips/strips leading newlines from the remaining buffer. If the coalescer flushes between those payloads, downstream integrations receive only plain text and cannot recover the blank-line boundary.

## Solution Assessment

Yes, the best current path is to keep this issue paired with the open closing PR and review or replace that focused fix. Creating a separate repair job now would duplicate active work.
