## Summary

Current main and the latest release still show the separator-consuming block-streaming behavior, and the linked repair PR is open, dirty, and unmerged, so this issue remains necessary as the canonical bug report.

## What This Changes

The issue reports that block streaming can consume paragraph separators between streamed deliveries, collapsing multi-paragraph Markdown in clients that reconstruct visible text from updates.

## Best Possible Solution

Preserve boundary semantics in the shared block-streaming pipeline, with focused coverage proving separate flushed deliveries reconstruct paragraph Markdown while hard-size and fenced-code splits remain safe.

## Reproduction Assessment

Yes from source inspection: the paragraph-flush branch emits text before the separator and buffers only text after it, so concatenating separate deliveries loses the blank line. A live channel reproduction was not run during this read-only review.

## Solution Assessment

Yes in direction: the fix belongs in the shared block chunking or delivery boundary semantics rather than channel-specific join logic. The current linked PR is not yet the resolved path because it is open, dirty, and unmerged.
