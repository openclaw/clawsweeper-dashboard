## Summary

Keep open: current main still drops paragraph-boundary semantics in the shared block-streaming path, and the linked closing PR is open/conflicting rather than merged.

## What This Changes

The issue reports that block-streamed reply reconstruction can lose blank-line paragraph boundaries when multi-paragraph Markdown is delivered across multiple streamed updates.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/73981 or a narrow replacement that preserves boundary semantics through chunking, subscribe delivery, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits before the paragraph separator, skips leading newlines after it, and trims delivered block reply text before downstream reconstruction.

## Solution Assessment

Yes. The shared block-streaming pipeline is the right fix surface because channel-specific joiners cannot infer paragraph boundaries after plain text deliveries have already been flushed.
