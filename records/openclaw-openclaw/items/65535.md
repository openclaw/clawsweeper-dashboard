## Summary

Keep open. Current main still lacks a blocking plugin interception point for Telegram streamed preview updates and in-place final preview edits, and the related PRs that tried to cover nearby delivery seams were closed unmerged or are only adjacent.

## What This Changes

Add or adapt a blocking outbound hook so plugins can inspect, rewrite, or cancel streamed message edits before sensitive content appears.

## Best Possible Solution

Define a shared, documented blocking pre-preview-delivery contract that covers preview creates, partial updates, and final edits with explicit streaming metadata, cancellation, rewrite, and cleanup semantics.

## Reproduction Assessment

Yes. With Telegram `streaming.mode="partial"` and a plugin registered for `message_sending`, a streamed reply can reach `answerLane.stream.update(...)` and final `editMessageTelegram(...)` without passing through `runMessageSending`.

## Solution Assessment

Unclear. The requested directions are valid, but a final-only `message_edit_sending` hook is incomplete; extending `message_sending` or adding a shared pre-preview-delivery hook for every visible update is the safer maintainable shape.
