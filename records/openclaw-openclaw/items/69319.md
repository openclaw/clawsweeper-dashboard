  - body: `dirExists` currently authorizes `fs.rm` even when there is no ClawHub origin metadata or lock entry, so an admin RPC call can delete a manually managed or uploaded `skills/<slug>` directory outside the ClawHub uninstall scope.   - confidence: 0.93
- **[P2] Preserve agent-scoped uninstall routing:** `src/gateway/protocol/schema/agents-models-skills.ts:248-254`
  - body: The new params have only `slug`/`source`, while current install/update/status methods accept `agentId` and resolve it through the configured-agent guard. Without the same routing, operator UIs cannot uninstall from a selected agent workspace and a default-agent call can target the wrong workspace.
  - confidence: 0.86
- **[P2] Return INVALID_REQUEST for bad uninstall slugs:** `src/gateway/server-methods/skills.ts:278`
  - body: Path-traversal slugs pass the non-empty schema and fail later in `uninstallSkillFromClawHub`, but the handler maps every backend failure to `UNAVAILABLE`. Client input errors should stay `INVALID_REQUEST` so callers do not retry a malformed request as a transient server outage.
  - confidence: 0.88
