## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Refuse uninstall for non-ClawHub skill folders:** `src/agents/skills-clawhub.ts:461-465`
  - body: `uninstallSkillFromClawHub` treats any existing `skills/<slug>` directory as removable even when the lockfile and `.clawhub/origin.json` do not show a ClawHub install. That lets this ClawHub-only admin RPC delete manually managed skill folders on a slug collision; gate removal on ClawHub provenance and otherwise return the idempotent not-installed result.
  - confidence: 0.92
- **[P1] Preserve agent workspace routing for uninstall:** `src/gateway/server-methods/skills.ts:253-254`
  - body: Current ClawHub `skills.install` and `skills.update` resolve the target workspace from `agentId`, but this new mirror ignores `context` and always resolves the default agent workspace. In multi-agent setups, uninstalling from a non-default agent would either no-op or remove the wrong workspace’s skill; add the same agent-scoped params and resolution path used by install/update.
  - confidence: 0.89
- **[P2] Return INVALID_REQUEST for malformed uninstall slugs:** `src/gateway/server-methods/skills.ts:278`
  - body: Malformed slugs such as `../evil` pass the non-empty schema and are rejected later by slug normalization, but the handler maps every backend error to `UNAVAILABLE`. Client input errors should be classified as `INVALID_REQUEST` so callers do not retry a request they need to correct.
  - confidence: 0.85
