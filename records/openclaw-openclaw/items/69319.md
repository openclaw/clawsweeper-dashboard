## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Refuse uninstall for untracked skill directories:** `src/agents/skills-clawhub.ts:464-467`
  - body: `dirExists` is enough to enter the recursive `fs.rm` path even when there is no `.clawhub/origin.json` and no workspace lock entry. A manually managed or uploaded `skills/<slug>` directory can therefore be deleted by the ClawHub-only RPC; gate removal on ClawHub provenance and return a safe not-installed/refused result for untracked directories.
  - confidence: 0.92
- **[P2] Return INVALID_REQUEST for malformed uninstall slugs:** `src/gateway/server-methods/skills.ts:278`
  - body: The schema allows any non-empty string, so values like `../evil` reach `normalizeTrackedSlug` and come back as backend errors. The handler maps every backend failure to `UNAVAILABLE`, which makes bad client input look like a transient server outage; classify slug/path validation failures as `INVALID_REQUEST`.
  - confidence: 0.86
- **[P2] Preserve current agent workspace routing:** `src/gateway/server-methods/skills.ts:254-257`
  - body: Current main routes skill Gateway mutations through `resolveSkillsAgentWorkspace` and supports optional `agentId` on related skill install/update requests. This new handler resolves only the default agent workspace, so a port would reject or mis-target selected-agent uninstall flows; add the current `agentId` schema and resolver path.
  - confidence: 0.78
