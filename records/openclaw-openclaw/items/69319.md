## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require ClawHub provenance before deleting the directory:** `src/agents/skills-clawhub.ts:461`
  - body: `uninstallSkillFromClawHub` removes any existing `skills/<slug>` directory even when there is no ClawHub origin metadata or lockfile entry. A manually managed or uploaded skill whose slug collides with the request would be recursively deleted by this new admin RPC, outside the ClawHub-only contract; gate directory removal on valid ClawHub provenance and return the idempotent not-installed response for untracked dirs.
  - confidence: 0.93
- **[P2] Return INVALID_REQUEST for bad uninstall slugs:** `src/gateway/server-methods/skills.ts:278`
  - body: The schema accepts any non-empty string, so path-traversal-like slugs such as `../evil` fail later in the backend as client input errors. Mapping every backend failure to `UNAVAILABLE` misclassifies bad requests as server outages and invites retries; classify slug normalization/path errors as `INVALID_REQUEST`.
  - confidence: 0.87
