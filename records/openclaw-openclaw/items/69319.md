## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refuse untracked skill folders:** `src/agents/skills-clawhub.ts:464-466`
  - body: `uninstallSkillFromClawHub` removes `targetDir` whenever the directory exists, even if there is no ClawHub origin metadata and no lockfile entry. A manually managed or non-ClawHub skill with the same slug can be deleted by this ClawHub-scoped RPC; gate removal on ClawHub provenance and otherwise return an idempotent not-installed result.
  - confidence: 0.91
- **[P2] Return bad uninstall slugs as invalid requests:** `src/gateway/server-methods/skills.ts:278`
  - body: The schema only requires a non-empty string, so malformed slugs like `../evil` reach backend slug normalization. Mapping every backend failure to `UNAVAILABLE` makes caller input look like a server outage and encourages retries; classify slug/path validation failures as `INVALID_REQUEST`.
  - confidence: 0.86
- **[P2] Honor current agent-scoped skill workspaces:** `src/gateway/server-methods/skills.ts:254-255`
  - body: Current `skills.install` and ClawHub `skills.update` can resolve an explicit agent workspace, but this new handler always resolves the default agent workspace and the new schema has no `agentId`. A current-main mirror of install should accept the same agent routing so operators do not uninstall from the wrong workspace.
  - confidence: 0.78
