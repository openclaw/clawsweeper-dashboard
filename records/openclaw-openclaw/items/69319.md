  - body: The params schema only requires a non-empty string, so malformed slugs like `../evil` reach backend normalization. Mapping those failures to `UNAVAILABLE` makes caller input look like a backend outage; classify slug/path validation failures as `INVALID_REQUEST`.
  - confidence: 0.86
- **[P2] Honor agent-scoped skill workspaces:** `src/gateway/server-methods/skills.ts:254-255`
  - body: Current `skills.install` and ClawHub `skills.update` can route to an explicit agent workspace, but this new handler always resolves the default workspace and the schema has no `agentId`. Mirror current install/update routing so operators do not uninstall from the wrong agent workspace.
