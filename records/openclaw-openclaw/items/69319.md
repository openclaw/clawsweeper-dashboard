Full review comments:

- **[P1] Refuse non-ClawHub skill folders:** `src/agents/skills-clawhub.ts:464-466`
  - body: `dirExists` is enough to enter the removal path, so an admin call for a slug that matches a manually managed or gateway-installer skill deletes that folder even when there is no `.clawhub` origin or lock entry. Gate `fs.rm` on ClawHub provenance and return an idempotent not-installed result for untracked folders.
  - confidence: 0.91
- **[P2] Return bad uninstall slugs as invalid requests:** `src/gateway/server-methods/skills.ts:278`
  - body: Malformed slugs like `../evil` pass the non-empty schema and are rejected by backend slug normalization, but this handler turns every backend failure into `UNAVAILABLE`. That misclassifies client input as a service outage and can cause callers to retry instead of fixing the request.
  - confidence: 0.86
