Full review comments:

- **[P1] Gate deletion on ClawHub provenance:** `src/agents/skills-clawhub.ts:471`
  - body: `uninstallSkillFromClawHub` removes any existing `skills/<slug>` directory after path normalization, even when there is no ClawHub origin or lock entry. That lets the new `skills.uninstall` RPC delete manually managed or uploaded skill content outside the ClawHub-tracked scope; require matching ClawHub provenance before `fs.rm`.
  - confidence: 0.93
- **[P2] Return INVALID_REQUEST for bad slugs:** `src/gateway/server-methods/skills.ts:276`
  - body: Malformed slugs like `../evil` pass the `NonEmptyString` schema, are rejected by backend normalization, and then every backend failure is mapped to `UNAVAILABLE`. Clients will treat bad input as a transient server outage; split slug/validation failures into `INVALID_REQUEST`.
  - confidence: 0.88
- **[P2] Preserve agent workspace routing:** `src/gateway/server-methods/skills.ts:252`
  - body: Current `skills.install` and `skills.update` resolve optional `agentId` through the shared workspace helper, but this new handler hard-codes the default workspace and the new params have no `agentId`. Multi-agent operators could install into one agent workspace and be unable to uninstall through the mirror RPC.
  - confidence: 0.8
