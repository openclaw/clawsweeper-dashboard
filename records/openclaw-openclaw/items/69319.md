## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refuse untracked skill folders:** `src/agents/skills-clawhub.ts:464-466`
  - body: `uninstallSkillFromClawHub` only treats the missing-directory case as a no-op. If `skills/<slug>` exists but has no ClawHub origin and no lockfile entry, this branch still deletes it, so a manually managed or non-ClawHub skill can be removed by a ClawHub-scoped RPC. Gate deletion on ClawHub provenance and return an idempotent not-installed result otherwise.
  - confidence: 0.91
- **[P2] Return bad uninstall slugs as invalid requests:** `src/gateway/server-methods/skills.ts:278`
  - body: The params schema only requires a non-empty string, so values like `../evil` reach the backend and fail slug normalization. Mapping every backend failure to `UNAVAILABLE` makes caller input look like a server outage and encourages retries; classify slug/path validation failures as `INVALID_REQUEST`.
  - confidence: 0.86
- **[P2] Honor current agent-scoped skill workspaces:** `src/gateway/server-methods/skills.ts:254-255`
  - body: Current `skills.install` and `skills.update` can resolve an explicit agent workspace, but this new handler always resolves the default agent workspace and the new schema has no `agentId`. A current-main mirror of install should accept the same agent routing so operators do not uninstall from the wrong workspace or lose parity for non-default agents.
  - confidence: 0.78
