## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Refuse non-ClawHub directories before removing:** `src/agents/skills-clawhub.ts:464-466`
  - body: `uninstallSkillFromClawHub` removes any existing `skills/<slug>` directory even when there is no ClawHub origin metadata or lockfile entry. That lets this ClawHub-only RPC delete manually managed or installer-owned skill folders on slug collision, so gate directory removal on ClawHub provenance and otherwise return the idempotent not-installed result.
  - confidence: 0.92
- **[P2] Preserve agent-scoped skills routing:** `src/gateway/server-methods/skills.ts:255-259`
  - body: Current `skills.install`, `skills.update`, and status-like RPCs accept optional `agentId` and resolve the selected workspace through `resolveSkillsAgentWorkspace`, but this handler always uses the default agent workspace and the new schema does not allow `agentId`. Multi-agent operators would have no way to uninstall from the intended agent workspace, so this is not a current mirror of `skills.install`.
  - confidence: 0.88
- **[P2] Classify malformed slugs as invalid requests:** `src/gateway/server-methods/skills.ts:278`
  - body: The schema accepts any non-empty string, so values like `../evil` reach the backend slug normalizer and then get mapped to `UNAVAILABLE`. Those are client input errors, not backend outages; return `INVALID_REQUEST` for slug/path validation failures so clients do not retry a bad request as a transient server failure.
  - confidence: 0.86
