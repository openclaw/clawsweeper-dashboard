## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refuse non-ClawHub directories before deletion:** `src/agents/skills-clawhub.ts:464-465`
  - body: `uninstallSkillFromClawHub` deletes any existing `skills/<slug>` directory, even when there is no ClawHub origin metadata or lockfile entry. A manual or non-ClawHub skill with the same slug would be removed by this ClawHub-only RPC, so gate directory deletion on ClawHub provenance and return the idempotent not-installed result otherwise.
  - confidence: 0.92
- **[P2] Honor agent-scoped skill workspaces:** `src/gateway/server-methods/skills.ts:256`
  - body: This handler always resolves the default agent workspace, while current sibling `skills.install` and ClawHub `skills.update` accept `agentId` and route through the selected agent workspace. Add the same `agentId` schema/handler path so operators can uninstall from the same workspace they installed into and do not accidentally affect the default workspace.
  - confidence: 0.86
- **[P2] Return INVALID_REQUEST for malformed slugs:** `src/gateway/server-methods/skills.ts:275`
  - body: The schema accepts any non-empty string, so malformed slugs such as traversal attempts are rejected later by slug normalization. Mapping every backend failure to `UNAVAILABLE` makes client input look like a server outage; classify slug/path validation failures as `INVALID_REQUEST` so callers correct the request instead of retrying.
  - confidence: 0.88
