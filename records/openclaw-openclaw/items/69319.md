## Summary

Keep open. Current main still does not implement `skills.uninstall`, and the PR addresses a real Gateway parity gap, but the patch has blocking correctness and security issues around a destructive admin RPC.

## What This Changes

The PR adds a Gateway `skills.uninstall` RPC with protocol schema/types, Swift generated models, handler registration and admin scope wiring, a ClawHub backend removal helper, docs, and focused backend plus Gateway handler tests.

## Best Possible Solution

Land a narrow `skills.uninstall` implementation that only removes ClawHub-proven or lockfile-tracked default-workspace skill installs, uses the active Gateway runtime config context, maps request validation failures to `INVALID_REQUEST`, and keeps protocol schemas, generated clients, docs, scopes, and tests aligned. Leave the CLI command design to the separate #63272 discussion.

## Reproduction Assessment

Yes. Create `workspace/skills/<slug>` with no `.clawhub/origin.json` and no `.clawhub/lock.json` entry; the PR helper sees `dirExists` and removes it. A malformed non-empty slug such as `../evil` also passes schema validation and is then mapped to `UNAVAILABLE` after backend normalization fails.

## Solution Assessment

No. The Gateway endpoint direction is reasonable, but this implementation is not the best fix until deletion is gated on ClawHub provenance, the handler uses `context.getRuntimeConfig()`, and client input errors are classified as request errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Resolve config through the Gateway context:** `src/gateway/server-methods/skills.ts:254-255`
  - body: The new handler calls `loadConfig()`, but this module does not import that symbol, so the patch will fail typecheck or throw before uninstalling. Even with an import, it would bypass the request's active runtime config; mirror `skills.install`/`skills.update` and use `context.getRuntimeConfig()`.
  - confidence: 0.94
- **[P1] Refuse untracked skill folders:** `src/agents/skills-clawhub.ts:464-467`
  - body: `uninstallSkillFromClawHub` removes `targetDir` whenever the directory exists, even when there is no ClawHub origin and no lockfile entry. A manual or gateway-installer `skills/<slug>` folder can be deleted by this ClawHub-scoped RPC, so gate removal on ClawHub provenance and otherwise return the idempotent not-installed result.
  - confidence: 0.92
- **[P2] Map invalid uninstall slugs to request errors:** `src/gateway/server-methods/skills.ts:278`
  - body: Malformed non-empty slugs pass the schema and are rejected later by `normalizeTrackedSlug`, but the handler maps every backend failure to `UNAVAILABLE`. Treat slug/path validation failures as `INVALID_REQUEST` so clients do not retry bad input as a server outage.
  - confidence: 0.88
