## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Refuse uninstall for untracked skill folders:** `src/agents/skills-clawhub.ts:461-463`
  - body: `uninstallSkillFromClawHub` deletes `skills/<slug>` whenever the directory exists, even if neither `.clawhub/origin.json` nor the workspace lockfile shows it is a ClawHub install. A manually managed or uploaded skill with the same slug would be removed by this ClawHub-only RPC, so gate deletion on ClawHub provenance and return an idempotent not-installed result otherwise.
  - confidence: 0.9
- **[P2] Return INVALID_REQUEST for bad uninstall slugs:** `src/gateway/server-methods/skills.ts:274`
  - body: The schema only requires a non-empty slug, so values like `../evil` reach backend slug normalization and then get mapped to `UNAVAILABLE`. That turns client input errors into retriable server outages; classify normalization/slug errors as `INVALID_REQUEST`.
  - confidence: 0.84
