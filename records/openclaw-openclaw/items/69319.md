Full review comments:

- **[P1] Gate skill deletion on ClawHub provenance:** `src/agents/skills-clawhub.ts:464-467`
  - body: This branch only returns the idempotent no-op when the directory is missing, so an existing `skills/<slug>` folder is deleted even with no `.clawhub/origin.json` and no lockfile entry.   - confidence: 0.93
- **[P2] Preserve agent-scoped workspace routing:** `src/gateway/protocol/schema/agents-models-skills.ts:248-252`
  - body: Current `skills.install` and ClawHub `skills.update` accept `agentId` and resolve the selected agent workspace, but this new uninstall schema/handler only targets the default workspace. Operators who installed a skill into a non-default agent workspace would be unable to remove it through the symmetric RPC, or could remove the wrong workspace's slug.
  - confidence: 0.87
- **[P2] Return INVALID_REQUEST for malformed slugs:** `src/gateway/server-methods/skills.ts:278`
  - body: The schema accepts any non-empty slug, so malformed values such as `../evil` fail later in the backend as client input errors. Mapping every backend failure to `UNAVAILABLE` makes bad requests look like server outages and invites retry behavior instead of request correction.
  - confidence: 0.86
