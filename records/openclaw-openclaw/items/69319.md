  - body: The new params have only `slug`/`source`, while current install/update/status methods accept `agentId` and resolve it through the configured-agent guard.   - confidence: 0.86

  - body: Path-traversal slugs pass the non-empty schema and fail later in `uninstallSkillFromClawHub`, but the handler maps every backend failure to `UNAVAILABLE`. Client input errors should stay `INVALID_REQUEST` so callers do not retry a malformed request as a transient server outage.
