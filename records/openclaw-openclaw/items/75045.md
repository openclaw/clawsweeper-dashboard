## Summary

Keep this open. Current checked-out main still lacks packaged OpenClaw version/package-root identity in the plugin loader registry cache key, and open PR #75071 is a live implementation candidate that explicitly closes this issue but has not merged.

## What This Changes

The issue reports stale browser plugin runtime-deps imports after upgrading OpenClaw 2026.4.26 to 2026.4.27, even after doctor repair and gateway restarts.

## Best Possible Solution

Land PR #75071 or an equivalent generic plugin-loader fix that adds bundled package version/package-root identity to the registry cache key with regression coverage, keeping the repair at the shared loader/runtime-deps seam rather than adding browser-specific cleanup.

## Reproduction Assessment

Yes. The issue gives a concrete package-upgrade reproduction, and current-main source still shows the same cache-key gap while PR #75071 adds a focused same-path package-version upgrade regression test.

## Solution Assessment

Yes. A shared loader cache invalidation fix using packaged OpenClaw identity is the narrowest maintainable direction; a doctor-only or browser-only repair would not address the cached registry reuse path.
