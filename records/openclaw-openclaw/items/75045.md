## What This Changes

The issue reports that upgrading from 2026.4.26 to 2026.4.27 leaves the bundled browser plugin loading stale 2026.4.26 plugin-runtime-deps paths after doctor repair and gateway restart.

## Best Possible Solution

## Reproduction Assessment

Yes. The report gives a concrete 2026.4.26 to 2026.4.27 upgrade path, and current source inspection shows a source-level reproduction path where the package-versioned runtime-deps root can change while the in-process plugin registry cache key does not.

## Solution Assessment
