## Summary

Keep open. The PR targets the right diagnostic log surfaces, but as submitted it bounds growth by dropping future JSONL writes after the active file reaches the cap, and the new public cache-trace config surface is not fully published through metadata/docs/changelog.

## What This Changes

The PR adds max-byte defaults and env/config parsing for cache-trace and Anthropic payload JSONL logs, updates cache-trace config schema/types/generated schema, and adds focused max-byte handling tests.

## Best Possible Solution

Land a focused bounded-rotation implementation for queued diagnostic JSONL files, with retained numbered archives, complete config/env/docs/changelog coverage, and regression tests proving diagnostics continue after the active file reaches its cap.

Yes. Source inspection gives a high-confidence reproduction path: fill `cache-trace.jsonl` or `anthropic-payload.jsonl` to the configured cap, then the PR's `getQueuedFileWriter(..., { maxFileBytes })` path returns without appending future diagnostic lines.
