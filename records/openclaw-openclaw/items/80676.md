## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use valid just recipe names:** `justfile:19`
  - body: `just` recipe names are `NAME` tokens and cannot contain `:`; the colon starts the dependency list. The first `build:strict:` style recipe makes the justfile fail to parse as a recipe named `build:strict`, so `just --list` will not work until these recipes are renamed, for example to `build-strict` while still running `pnpm build:strict-smoke`.
  - confidence: 0.93
- **[P2] Document template selection with query links:** `.github/pull_request_template.md:8-12`
  - body: GitHub documents multiple pull request templates as selected with the `template` query parameter. With this root fallback template, new PRs will still open the fallback body, and the source-file links here do not switch the active template, so the Quick Fix / Full Change flow will not work as described.
  - confidence: 0.91
- **[P2] Make the scripts index complete and accurate:** `scripts/README.md:3-21`
  - body: The new index says there are 318 script files, but current main has 691 tracked files under `scripts/`, and the README uses placeholder rows instead of listing the scripts. As a contributor lookup, this is immediately incomplete and misleading; generate or manually complete the index from the current tree before merging.
  - confidence: 0.88
- **[P3] Translate the contributor-facing scripts docs:** `scripts/README.md:3-21`
  - body: This new README is contributor-facing documentation, but most headings and descriptions are Chinese while repo policy requires American English for docs. Please translate the index text before merging so it matches the rest of the project documentation.
  - confidence: 0.86
