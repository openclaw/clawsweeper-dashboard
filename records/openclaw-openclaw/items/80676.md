Full review comments:

- **[P2] Use valid just recipe names:** `justfile:19`
  - body: `just` recipe names are `NAME` tokens and cannot contain `:`. The first `build:strict:` recipe makes the file fail to parse as the advertised task set, so `just --list` will not work until these recipes are renamed, for example to `build-strict` while still running `pnpm build:strict-smoke`.
  - confidence: 0.93
- **[P2] Document template selection with query links:** `.github/pull_request_template.md:8-12`
  - body: GitHub's PR-template docs describe multiple templates selected with the `template` query parameter, not an automatic chooser. With this fallback template, source-file links here do not switch the active template, so the Quick Fix / Full Change flow will not work as described.
  - confidence: 0.91
- **[P2] Keep real proof fields in the quick template:** `.github/PULL_REQUEST_TEMPLATE/quick_fix.md:33-36`
  - body: External PR proof is parsed and reviewed using the structured fields in the current template and contributor guide. The quick template only asks for before/after evidence, so contributors can fill it while omitting the real environment, exact steps, observed result, and untested scope required by the proof gate.
  - confidence: 0.87
- **[P2] Make the scripts index complete and accurate:** `scripts/README.md:3`
  - body: The new index says there are 318 script files, but the PR base already had 664 tracked paths under `scripts/`, and the README uses placeholder rows such as 'other 45' instead of listing the scripts. Generate or manually complete the index from the current tree before merging.
  - confidence: 0.9
- **[P3] Translate the contributor-facing scripts docs:** `scripts/README.md:3-21`
  - body: This new README is contributor-facing documentation, but most headings and descriptions are Chinese while the repo's contributor policy requires American English in docs. Translate the index before merging so it matches the rest of the project documentation.
  - confidence: 0.86
