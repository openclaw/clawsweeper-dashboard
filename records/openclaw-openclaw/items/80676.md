Full review comments:

- **[P2] Use valid just recipe names:** `justfile:19`
  - body: `just` recipe names are NAME tokens and cannot contain `:`. The first `build:strict:` header makes the justfile fail to parse as the advertised `build:strict` recipe, so `just --list` will not work until these recipes are renamed, for example to `build-strict`.
  - confidence: 0.93
- **[P2] Document template selection with query links:** `.github/pull_request_template.md:8-12`
  - body: GitHub documents multiple PR templates as files selected with the `template` query parameter, not as an automatic chooser from this fallback body. These source-file links will open Markdown files rather than switch the active pull request template, so the Quick Fix / Full Change flow will not work as described.
  - confidence: 0.91
- **[P2] Keep quick-fix proof fields parseable:** `.github/PULL_REQUEST_TEMPLATE/quick_fix.md:33-36`
  - body: The quick-fix template reduces real behavior proof to only Before/After lines. External PRs still need the real setup, exact post-patch command or steps, evidence after fix, observed result, and untested scope, so this template would steer contributors into failing the proof gate.
  - confidence: 0.9
- **[P2] Make the scripts index complete and accurate:** `scripts/README.md:3-45`
  - body: The README claims 318 script files, but current main has 696 tracked files under `scripts`, and the table uses placeholder rows instead of listing the missing entries. As a contributor lookup, this is immediately incomplete and misleading.
  - confidence: 0.88
- **[P3] Translate the contributor-facing scripts docs:** `scripts/README.md:3-21`
  - body: This README is contributor-facing repository documentation, but most headings and descriptions are Chinese while the repo asks for American English in docs and UI strings. Translate the index before merging so it matches the project’s contributor guidance.
  - confidence: 0.86
