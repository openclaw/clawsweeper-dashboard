## Summary

Keep open: this is a plausible focused Gemini transport bug fix, but the current PR head is not mergeable because it has a syntax-level duplicate declaration and also tightens signature validation in ways that contradict current same-route replay tests.

## What This Changes

The PR adds strict Base64 checks before serializing Gemini thought signatures in the native Google transport and OpenAI-compatible Gemini tool-call replay path, while preserving the Gemini 3 skip-validator sentinel after a follow-up commit.

## Best Possible Solution

Remove the duplicate declaration, preserve the current exact same-route signature and skip-sentinel behavior, and add focused regression coverage for dropping only clearly corrupted captured signatures in both transport paths.

## Reproduction Assessment

Yes. The PR head is source-reproducible: the fetched file contains a duplicate declaration that prevents parsing, and current main tests show the new regex would drop opaque same-route signatures that are currently expected to replay.

## Solution Assessment

No. The validation direction is reasonable for clearly corrupted captured strings, but the current patch is syntactically broken and applies a stricter alphabet than current code/tests allow for exact same-route Gemini signatures.
