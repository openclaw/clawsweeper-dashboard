## What This Changes

The issue asks OpenClaw to normalize oversized GitHub Copilot Responses reasoning replay IDs in Telegram direct-session history before dispatch.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85198 or an equivalent normalization with regression coverage, then close this issue from the merged commit or release.

Yes. A high-confidence source reproduction is to build a GitHub Copilot Responses payload from an assistant thinkingSignature JSON whose reasoning id is longer than 64 characters; current main parses and replays that id without normalizing it.

## Solution Assessment

Yes. The linked fix direction is the narrowest maintainable solution: hash only oversized replay ids before dispatch while preserving valid short ids and native Codex id omission behavior.
