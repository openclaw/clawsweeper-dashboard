## Summary

Keep this PR open: the diagnostic goal is useful and proof is sufficient, but the patch can make the new abort warning hard to correlate with the existing `cli exec` line because the two log lines can use different prompt values.

## What This Changes

The PR adds prompt character count and an 8-character SHA-256 prompt hash to Claude CLI live-session turn-failure logs, plus helper tests for the prompt summary.

## Best Possible Solution

Keep the small diagnostic fix, but use one shared non-secret turn or run correlator, or compute both log lines from the same prompt-length source, with regression coverage for a transformed prompt.

## Reproduction Assessment

Yes, by source inspection: current `execute.ts` logs `basePrompt.length`, then passes a transformed prompt into the live-session path where the PR computes the new failure metadata. A transformed or image-augmented prompt is enough for the two values to diverge.

## Solution Assessment

No. The diagnostic direction is useful, but the current patch is not the best solution until it uses the same prompt source as the existing log or a shared non-secret correlator.
