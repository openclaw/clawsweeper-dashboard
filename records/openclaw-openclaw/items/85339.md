## Summary

Keep this PR open, but it needs a small correctness fix before merge: the proposed abort breadcrumb can compute `promptChars` from a different prompt string than the existing `cli exec` log, so the correlation it is meant to provide can fail in real transformed prompt paths.

## What This Changes

This PR adds prompt length and an 8-character SHA-256 prompt hash to Claude CLI live-session turn failure logs, plus helper tests for that prompt summary.

## Best Possible Solution

Keep the small diagnostic fix, but use one shared non-secret correlator or compute the same prompt-length value for both the `cli exec` and abort-failure log lines, with regression coverage for transformed prompts.

## Reproduction Assessment

Yes, by source inspection: current `execute.ts` logs `basePrompt.length`, while the PR records metadata from the transformed `params.prompt` passed into the live session. I did not run a live abort, but the mismatch path is clear in the current source and PR diff.

## Solution Assessment

No. The diagnostic direction is useful, but the current implementation is not the best fix because it does not guarantee that the new failure log correlates with the existing `cli exec` line; a shared run/turn id or a single prompt-length source would be safer.
