## Summary

Keep open: the reported malformed Tailscale JSON crash class is plausible on current main, but this patch is not merge-ready because it changes caller semantics by mapping malformed output to `{}` and the external PR still lacks real Tailscale behavior proof.

## What This Changes

The PR wraps Tailscale JSON parsing in a catch-all `{}` fallback and routes `ensureFunnel` status parsing through the noisy JSON helper.

## Best Possible Solution

Make Tailscale parse handling caller-aware: preserve noisy valid JSON support, keep malformed whois output on the error-TTL path, report malformed Funnel status as unknown command output, and add focused regression tests.

## Reproduction Assessment

Yes, by source inspection rather than a live Tailscale run. Malformed non-empty output through `readTailscaleWhoisIdentity` or `ensureFunnel` exercises the current raw `JSON.parse` paths and the PR's altered `{}` fallback semantics.

## Solution Assessment

No. Returning `{}` from the shared parser is not the narrowest maintainable fix because empty output, malformed output, command errors, and missing identity/status data have different meanings at different callers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve whois parse failures on the error TTL path:** `src/infra/tailscale.ts:23-24`
  - body: Catching parser failures in the shared helper and returning `{}` means malformed `tailscale whois --json` output no longer reaches `readTailscaleWhoisIdentity`'s `catch`; it is cached as a successful null identity with the default 60s TTL instead of the 5s error TTL, prolonging transient Serve auth failures.
  - confidence: 0.93
- **[P2] Keep Funnel parse failures out of the disabled-state branch:** `src/infra/tailscale.ts:333`
  - body: With malformed `tailscale funnel status --json` converted to `{}`, the next branch treats unknown or partial command output as “Funnel is not enabled” and prompts setup. Preserve a parse or command diagnostic instead of sending users down the disabled-tailnet remediation path.
  - confidence: 0.91
