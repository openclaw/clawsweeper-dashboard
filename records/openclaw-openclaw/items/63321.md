## Summary

Keep this PR open, but it still needs changes: the crash class is real, yet the current patch changes malformed Tailscale output into `{}` globally, which alters whois cache semantics and misclassifies malformed Funnel status as disabled state. It also lacks after-fix real behavior proof from a real Tailscale setup.

## What This Changes

The branch changes `src/infra/tailscale.ts` so the noisy JSON parser returns `{}` on parse failure and `ensureFunnel` uses that helper for `tailscale funnel status --json`.

## Best Possible Solution

Make the parse handling caller-aware: preserve noisy valid JSON support, keep malformed whois output on the error-TTL path, report malformed Funnel status as unknown command output, and add focused regression tests.

## Reproduction Assessment

Yes, by source inspection rather than an executed test run. Malformed non-empty output through `readTailscaleWhoisIdentity` would be cached as a success-null result by the PR, and malformed `tailscale funnel status --json` output would enter the disabled/setup branch.

## Solution Assessment

No. Returning `{}` from the shared parser is not the narrowest maintainable fix because each caller assigns a different meaning to an empty object; caller-level parse handling or a typed parse result is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve whois parse failures on the error TTL path:** `src/infra/tailscale.ts:23-24`
  - body: Catching parser failures in the shared helper and returning `{}` means malformed `tailscale whois --json` output no longer reaches `readTailscaleWhoisIdentity`'s `catch`; it is cached as a successful null identity with the default 60s TTL instead of the 5s error TTL, prolonging transient Serve auth failures.
  - confidence: 0.93
- **[P2] Keep Funnel parse failures out of the disabled-state branch:** `src/infra/tailscale.ts:333`
  - body: With malformed `tailscale funnel status --json` converted to `{}`, the next branch treats unknown or partial command output as “Funnel is not enabled” and prompts setup. Preserve a parse or command diagnostic instead of sending users down the disabled-tailnet remediation path.
  - confidence: 0.91
