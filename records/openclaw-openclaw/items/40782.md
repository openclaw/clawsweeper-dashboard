## Summary

Keep open: the Feishu multi-bot @mention bug is still present on current main and the latest release, but this PR is not merge-ready because it depends on an unproven `union_id` source, conflicts with current main's Feishu identity path, leaves related mention paths open-id-only, and lacks after-fix real Feishu proof.

## What This Changes

The PR adds a Feishu `botUnionId` path from probing/state into message parsing so @mention detection can fall back from `open_id` to `union_id` in multi-bot groups.

## Best Possible Solution

Rewrite the fix on current main around a proven Feishu stable bot identity source, then cover top-level mentions, rich-text posts, mention stripping, debounce merging, focused tests, and redacted live multi-bot proof before merge.

## Reproduction Assessment

## Solution Assessment

No. The PR is a plausible mitigation, but it is not the best current fix until it uses a proven Feishu identity source and threads that identity through current-main mention matching, rich-text posts, mention stripping, and debounce merging.
