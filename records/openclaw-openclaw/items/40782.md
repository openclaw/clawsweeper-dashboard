  - confidence: 0.82

  - body: `parseFeishuMessageEvent` accepts `botUnionId`, but the content normalization still strips only by `botOpenId`. A multi-bot `@Bot /help` that matched by union_id can keep the raw mention in front of `/help`, regressing the slash-command behavior this parser comment protects.
  - confidence: 0.86
- **[P2] Carry fallback identity through debounced batches:** `extensions/feishu/src/monitor.account.ts:248`
  - body: The final handler receives `botUnionId`, but the debounced batch path still derives merged text and bot mentions from open_id-only parsing. A burst where the target mention is only in an earlier event can still be merged into a message that fails the requireMention gate.
  - confidence: 0.8
