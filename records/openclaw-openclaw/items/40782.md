  - confidence: 0.88
- **[P2] Apply the fallback to post mentions too:** `extensions/feishu/src/bot.ts:473`
  - body: The added fallback only applies when top-level `message.mentions` is populated, while the rich-text post path still compares parsed mentioned ids to `botOpenId` only. Post messages with the same app-scoped mismatch can still be treated as unmentioned.
  - confidence: 0.84
- **[P2] Carry the fallback through current debounce handling:** `extensions/feishu/src/monitor.account.ts:251`
  - body: The branch only passes `botUnionId` into the final message handler, but current main normalizes text and merges mention metadata before that handoff. A current-main rewrite needs the same identity semantics in debounce or batched messages can still lose the bot mention.
  - confidence: 0.83
