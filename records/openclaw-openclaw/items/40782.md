## What This Changes

The PR threads a new Feishu `botUnionId` from probe/startup state into message parsing so @mention detection can fall back from `open_id` to `union_id`.

## Best Possible Solution

Rewrite the fix on current main around a proven stable Feishu bot identity source, then carry that identity through text, post, mention stripping, mention-forward, debounce, tests, and redacted live Feishu proof.

## Reproduction Assessment

## Solution Assessment

No. This PR is a plausible old-base mitigation, but it is not the best fix because the identity source is unproven and the fallback does not cover all current-main Feishu mention paths.

- **[P1] Prove the union_id source before wiring it:** `extensions/feishu/src/probe.ts:137`
  - body: The patch stores `botUnionId` from `bot?.union_id`, but the branch's response type declares only `bot_name` and `open_id`, and the official Feishu bot info docs plus the linked investigation do not show `union_id`. If this stays undefined, the fallback never fixes the multi-bot drop.
  - confidence: 0.94

  - body: This can mark the bot as mentioned through `botUnionId`, but content normalization still strips only by `botOpenId`. In the same app-scoped mismatch, an addressed `@Bot /help` can keep the mention tag and miss the slash-command path that the nearby comment protects.
  - confidence: 0.88
- **[P2] Apply the same identity rule to rich-text posts:** `extensions/feishu/src/bot.ts:477-479`
  - body: The new fallback only applies when top-level `message.mentions` is populated, while the rich-text post branch still compares parsed mention IDs to `botOpenId` only. Post messages with the same app-scoped mismatch can still be treated as unmentioned.
  - confidence: 0.84
- **[P2] Pass the fallback through debounce merging:** `extensions/feishu/src/monitor.account.ts:251`
  - body: The final handler receives `botUnionId`, but debounce text and retained bot mentions are still derived from open-id-only checks before that handoff. Batched messages can therefore still lose the bot mention under `requireMention`.
  - confidence: 0.83
