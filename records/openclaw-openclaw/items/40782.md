  - body: The new `union_id` fallback only applies when `message.mentions` is populated; the post fallback still checks `mentionedOpenIds` against `botOpenId` only. A rich-text Feishu message with the same app-scoped mismatch can still be treated as unmentioned and dropped under `requireMention`.
  - confidence: 0.84

  - confidence: 0.88
- **[P2] Thread botUnionId through debounce handling:** `extensions/feishu/src/monitor.account.ts:251`
  - body: This dispatch wiring passes `botUnionId` only after debounce has already normalized text and merged mention metadata with open-id-only logic. A debounced batch can still lose the bot mention and be dropped by the `requireMention` gate.
  - confidence: 0.85
