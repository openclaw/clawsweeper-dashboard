- **[P1] Do not read an undeclared bot info field:** `extensions/feishu/src/probe.ts:137`
  - body: `FeishuBotInfoResponse` on this branch only declares `bot_name` and `open_id`, but this line reads `bot?.union_id`.   - confidence: 0.9
- **[P2] Strip the union-matched bot mention before commands:** `extensions/feishu/src/bot.ts:790`
  - body: This can mark the bot as mentioned via `union_id`, but content normalization still strips only by `botOpenId`. In the same multi-bot mismatch, `@Bot /model` can keep the mention tag and miss the slash-command path that the nearby comment says must be preserved.
  - confidence: 0.86
- **[P2] Apply the fallback to post mentions too:** `extensions/feishu/src/bot.ts:470-474`
  - body: The new fallback only runs when top-level `message.mentions` is populated. Rich-text post messages still compare parsed mentioned IDs to `botOpenId`, so post messages with the same app-scoped mismatch can still be treated as unmentioned.
  - confidence: 0.82
- **[P2] Carry the fallback through debounce handling:** `extensions/feishu/src/monitor.account.ts:251`
  - body: The branch passes `botUnionId` into the final handler, but the debounce path still builds text and retained bot mentions through open_id-only parsing and filtering before that handoff. Batched messages can therefore still lose the bot mention under `requireMention`.
  - confidence: 0.83
