Full review comments:

- **[P1] Use a real source for the fallback identity:** `extensions/feishu/src/probe.ts:134`
  - body: The patch stores `botUnionId` from `bot?.union_id`, but the linked investigation and closed sibling PR say this bot-info response does not include that field.   - confidence: 0.87
- **[P2] Strip the union-matched mention before command parsing:** `extensions/feishu/src/bot.ts:789`
  - body: `mentionedBot` can become true via `union_id`, but content normalization still strips only `botOpenId`. In the app-scoped `open_id` case the leading `@Bot` remains in the message body, so `/model` or `/help` no longer starts with `/` and the existing Feishu slash-command path can miss it.
  - confidence: 0.84
- **[P2] Carry the fallback identity through debounce handling:** `extensions/feishu/src/monitor.account.ts:248`
  - body: The final handler receives `botUnionId`, but the debounce text and merged mention paths still parse and filter with open-id-only helpers. A batched message where only an earlier entry contains the union-matched mention can still be flushed as unmentioned under `requireMention`.
  - confidence: 0.78
