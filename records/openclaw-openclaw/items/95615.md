## Summary

Keep open: the PR targets a real CLI-backed cron regression, but the current patch is not merge-ready because it can drop a restricted creator-surface cap, can lose the new persisted default flag during later updates, and still lacks real CLI cron proof.

## What This Changes

The branch adds a persisted `toolsAllowIsDefault` cron payload flag, stamps it for default creator-surface caps, and drops flagged `toolsAllow` values for CLI-backed scheduled `agentTurn` runs.

## Best Possible Solution

Keep CLI runs fail-closed for any restricted creator-surface cap, drop only caps proven unrestricted after maintainer approval, preserve the flag through updates, and add redacted real CLI cron proof.
