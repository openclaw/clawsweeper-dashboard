## Summary

Keep this PR open: it addresses a real Codex migration selector bug and includes convincing terminal proof, but the proposed Enter reconciliation introduces a P2 regression where a row the user just unchecked can be silently re-selected on submit.

## What This Changes

The PR changes Codex migration multi-select Enter handling, adds non-JSON migration progress output, updates migrate docs and changelog, and adds focused selector tests.

## Best Possible Solution

Keep the progress work and special-row Enter activation, but make ordinary selectable rows preserve the current checked state on submit and add regression coverage for Space-then-Enter on a preselected row.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: run the interactive Codex migration selector with planned rows preselected, press Space on the highlighted planned row to uncheck it, then press Enter. The PR helper re-adds that activated value before submit, matching the existing review comment and dependency behavior.

## Solution Assessment

No, not as written: the PR fixes the special rows and progress path, but the Enter helper over-applies activation to ordinary selectable rows. The narrow fix is to preserve ordinary row selection state on Return, or explicitly track Space-vs-Return intent if maintainers want Return to select unchecked ordinary rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
