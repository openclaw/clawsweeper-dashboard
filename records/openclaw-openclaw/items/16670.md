## Summary

Keep open. Current main and the latest release still lack a dedicated setup/onboarding/configure path for memory embedding provider/model/API-key or SecretRef setup, and the requested mandatory behavior remains a product and credential-policy decision.

## What This Changes

The issue requests making memory embedding provider/model/API-key setup a mandatory or prominent onboarding step.

## Best Possible Solution

Add a maintainer-approved memory setup path in onboarding and configure that writes the existing memorySearch fields, supports the chosen credential policy, verifies readiness through a controlled status/probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main and v2026.5.12 show setup/configure/onboarding omit a memory section or dedicated memorySearch prompt while the memory docs still require manual provider and credential setup.

## Solution Assessment

Unclear as requested. A dedicated setup/configure path is likely the right implementation area, but mandatory versus optional/default-on behavior and credential/probe policy need maintainer direction first.
