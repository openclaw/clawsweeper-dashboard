## Summary

Keep open. Current main still has memory configuration docs, diagnostics, and `openclaw memory status --deep`, but setup/onboarding/configure do not provide the requested memory embedding provider/model/API-key or SecretRef setup step, and making it mandatory still needs maintainer product and credential-policy judgment.

## What This Changes

Request to make memory embedding setup a prominent mandatory onboarding step that configures `agents.defaults.memorySearch` provider, model, credentials, and readiness checks.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes existing `agents.defaults.memorySearch` fields, supports the chosen SecretRef/plaintext policy, verifies readiness through a controlled status or embedding probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main source inspection shows plain setup only writes workspace/gateway defaults, onboarding has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear. A dedicated setup/configure flow is likely the right implementation area, but mandatory versus optional/diagnostic behavior, credential storage, SecretRef UX, and probe semantics need maintainer direction first.
