## Summary

Keep open. Current main and the latest release still document and diagnose memory embedding readiness, but setup, onboarding, and configure do not provide a dedicated `memorySearch` provider/model/API-key or SecretRef step; the mandatory/default-on choice remains a product decision.

## What This Changes

Request to add a mandatory onboarding/setup wizard step that configures `agents.defaults.memorySearch`, captures embedding credentials, and verifies embedding readiness.

## Best Possible Solution

Add a maintainer-approved memory setup path in onboarding and `openclaw configure` that writes the existing `agents.defaults.memorySearch` contract, supports SecretRef or API-key input, verifies readiness, and documents daemon and Codex OAuth caveats.

I did not run a live embedding failure path because this read-only review is about setup surface and product policy.

## Solution Assessment

Unclear as requested. A dedicated setup/configure path is likely maintainable, but mandatory versus optional/default-on behavior plus credential and probe semantics need maintainer direction first.
