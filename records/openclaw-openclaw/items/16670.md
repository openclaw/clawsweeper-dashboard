## Summary

Keep open. Current main still has memory configuration docs, doctor diagnostics, and `openclaw memory status --deep`, but setup, onboarding, and configure do not provide the requested memory embedding setup step, and the mandatory/default-on policy still needs maintainer direction.

## What This Changes

The issue requests a mandatory onboarding step that configures `agents.defaults.memorySearch` embedding provider/model/credential settings and verifies embedding readiness.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes the existing memorySearch contract, supports SecretRef-safe credential input, verifies readiness, and documents daemon/Codex OAuth caveats.
