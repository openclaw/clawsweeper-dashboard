## Summary

Keep open. Current main still lacks a setup/onboarding/configure step for memory embedding provider, model, credentials, or readiness probing; newer diagnostics and FTS fallback reduce the silent-failure impact but do not resolve the first-run UX request, and the mandatory wording still needs maintainer product judgment.

## What This Changes

The issue requests a mandatory onboarding step for configuring memory search embeddings, credentials, and endpoint readiness.

## Best Possible Solution

Add a maintainer-approved memory readiness path in onboarding and configure that explains FTS-only, local, and remote embedding choices, stores credentials through existing SecretRef-aware config, and probes readiness only when semantic embeddings are selected.

## Reproduction Assessment

Yes. Source inspection shows setup, onboarding, and configure have no dedicated memory embedding step; this is a source-level UX reproduction rather than a runtime crash reproduction.

## Solution Assessment

No to the exact universally mandatory proposal. A prominent provider-aware choice with an explicit FTS/local/remote path is safer because current OpenClaw supports valid modes that do not require remote embedding credentials.
