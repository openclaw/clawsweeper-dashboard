## Summary

Keep open. Current main and the latest release still have manual memorySearch docs and diagnostics, but setup/onboarding/configure do not provide a memory embedding provider/model/API-key or SecretRef setup step; recent discussion also frames the remaining mandatory/default-on/diagnostic-only choice as a product decision.

## What This Changes

Request adds a mandatory or prominent memory embedding setup step to onboarding/configure so users can configure and verify memorySearch credentials during first-run setup.

## Best Possible Solution

Add a maintainer-approved memory setup path in onboarding and configure that writes existing memorySearch fields, follows the chosen credential policy, verifies readiness through a controlled status/probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main and v2026.5.12 show setup/configure/onboarding omit a memory section or dedicated memorySearch prompt while the memory docs still require provider credentials for remote embeddings.

## Solution Assessment

Unclear as requested. A dedicated setup/configure path is likely the right implementation area, but mandatory versus optional/default-on behavior and credential/probe policy need maintainer direction first.
