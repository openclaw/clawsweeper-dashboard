## Summary

Keep open. Current main still has manual memorySearch configuration, doctor diagnostics, and deep memory status probes, but setup/onboarding/configure do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step; making it mandatory remains a product and credential-policy decision.

## What This Changes

Request asks onboarding/setup to add a prominent mandatory memorySearch embedding provider/model/API-key setup and readiness verification step.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and configure that writes the existing memorySearch fields, supports the chosen credential policy, verifies readiness through a controlled probe/status path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main source inspection shows plain setup only writes workspace/gateway defaults, onboarding has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear as requested. A dedicated setup/configure memory path is likely the right implementation area, but mandatory versus optional/default-on behavior, credential storage, SecretRef UX, and probe semantics need maintainer direction first.
