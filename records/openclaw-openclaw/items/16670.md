## Summary

Keep open. Current main still documents and diagnoses memory search, but setup/onboarding/configure do not offer a memory embedding provider/model/API-key or SecretRef step, and making that step mandatory needs maintainer product and credential-policy judgment.

## What This Changes

Request to make memory/embedding provider setup a prominent or mandatory onboarding step, including provider/model/API-key configuration and an embedding readiness check.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes existing `agents.defaults.memorySearch`, supports the chosen credential policy, verifies readiness through a controlled status/probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes. Source inspection of current main shows setup only writes workspace/gateway defaults, onboarding has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear. A dedicated memory setup section is likely the right implementation area, but making it mandatory is not clearly the best UX until maintainers decide credential storage, SecretRef, daemon environment, and live-probe policy.
