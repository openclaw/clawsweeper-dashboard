## Summary

Keep open. Current main still lacks a setup/onboarding/configure memory embedding setup path, and the requested mandatory behavior remains a product and credential-policy decision rather than a conservative cleanup close.

## What This Changes

The issue requests adding a prominent mandatory onboarding step to configure `agents.defaults.memorySearch` provider, model, API key or SecretRef, and readiness verification.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes existing `agents.defaults.memorySearch` fields, supports the chosen secret policy, verifies readiness through a controlled probe/status path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main source inspection shows plain setup only writes workspace/gateway defaults, onboarding has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear. A dedicated setup/configure memory flow is likely the right implementation area, but mandatory versus default-on versus prominently optional behavior, credential storage, SecretRef UX, and probe semantics need maintainer direction first.
