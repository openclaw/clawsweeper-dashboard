## Summary

Keep open. Current main and the latest release still have memorySearch docs, doctor diagnostics, and deep memory status checks, but setup/onboarding/configure do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step; the mandatory versus optional/default-on policy remains a product decision.

## What This Changes

Request to add a prominent mandatory onboarding step that configures and verifies memorySearch embedding provider credentials.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and openclaw configure that writes existing memorySearch fields, follows the chosen credential policy, verifies readiness through a controlled status/probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main and v2026.5.12 show setup/configure/onboarding omit a memory section or dedicated memorySearch prompt while memory docs still require embedding provider credentials.

## Solution Assessment

Unclear as requested. A dedicated setup/configure path is the likely implementation area, but mandatory versus optional/default-on behavior and credential/probe policy need maintainer direction first.
