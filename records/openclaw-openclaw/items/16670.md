## Summary

Keep open. Current main still documents and diagnoses memory-search embedding setup, but the setup/onboarding/configure flows do not provide the requested memory embedding provider/model/API-key or SecretRef step, and making it mandatory needs maintainer product and credential-policy judgment.

## What This Changes

The issue requests a mandatory onboarding/setup step for configuring memory search embeddings, provider credentials, and readiness verification.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes existing `agents.defaults.memorySearch` fields, supports the chosen SecretRef/plaintext policy, verifies readiness through a controlled status or probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main source inspection shows setup only writes workspace/gateway defaults, onboarding has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear. A dedicated setup/configure flow is likely the right implementation area, but mandatory versus optional/diagnostic behavior and credential/probe policy need maintainer direction first.
