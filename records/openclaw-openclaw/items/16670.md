The issue requests making memory embedding provider, model, and credential setup a mandatory part of onboarding so cross-session memory search works reliably.

## Best Possible Solution

Add a maintainer-approved memory setup path in onboarding and configure that writes the existing memorySearch fields, handles credentials safely, verifies readiness through a controlled status/probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

## Solution Assessment

Unclear as requested. A dedicated setup/configure path is the likely implementation area, but making it mandatory versus optional/default-on and choosing credential/probe semantics needs maintainer direction first.
