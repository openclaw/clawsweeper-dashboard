## Summary

Keep open. Current main and v2026.5.19 still lack a dedicated `memorySearch` setup/onboarding/configure step; existing docs, doctor hints, and `openclaw memory status --deep` cover manual or diagnostic paths, while mandatory versus optional/default-on behavior still needs maintainer product direction.

Request a mandatory onboarding/setup/configure step for memory embedding provider, model, API-key or SecretRef setup, and readiness verification.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes the existing `agents.defaults.memorySearch` config, supports API-key or SecretRef input, verifies readiness, and documents daemon and Codex OAuth caveats.
