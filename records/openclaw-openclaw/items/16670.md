## Summary

Keep open: current main and v2026.6.8 still lack a first-class memory embedding setup path in setup, onboarding, or configure, while the requested mandatory behavior remains a product and credential-UX decision.

## What This Changes

The issue asks for onboarding/setup to add a prominent mandatory memory embedding provider, model, credential or SecretRef setup, and readiness check.

## Best Possible Solution

Add a maintainer-approved memory readiness path in CLI onboarding and configure that writes existing memorySearch config, supports SecretRef-safe credentials, verifies embeddings, and documents the chosen mandatory-versus-recommended policy.

## Reproduction Assessment

Yes, at source level: current setup, onboarding, and configure flows have no dedicated memorySearch section, credential prompt, or embedding readiness probe. I did not run the interactive wizard because this was a read-only review.

## Solution Assessment

Partly: a first-class memory setup/readiness flow is the narrow maintainable direction, but making it universally mandatory is not yet established across built-in, local, keyword-only, and plugin-backed modes.
