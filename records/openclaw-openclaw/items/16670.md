## Summary

Keep open: current main and v2026.6.8 still lack a memory embedding setup path in setup, onboarding, or configure, and the mandatory-versus-recommended credential/probe policy still needs maintainer direction.

## What This Changes

Request to make onboarding/setup configure memory embeddings, credentials or SecretRefs, and readiness verification as a mandatory first-run step.

## Best Possible Solution

Add a maintainer-approved memory readiness path in CLI onboarding and openclaw configure that writes agents.defaults.memorySearch, verifies embeddings, and documents the chosen mandatory-versus-recommended policy.

## Reproduction Assessment

Yes, at source level: current setup, onboarding, configure, and non-interactive onboarding flows have no dedicated memorySearch section, credential prompt, or embedding readiness probe. I did not live-reproduce the broader complete-memory-loss claim because current docs and code support fallback and plugin-backed memory paths.

## Solution Assessment

Partly: a first-class memory setup/readiness flow is the maintainable direction, but making it universally mandatory is not yet established across local, keyword-only, default, and plugin-backed modes.
