## Summary

Keep open. Current main and v2026.5.22 still document and diagnose memorySearch readiness, but setup, onboarding, and configure do not provide a dedicated memory embedding provider/model/API-key or SecretRef step; the mandatory/default-on choice remains a maintainer product decision.

## What This Changes

The issue requests a prominent mandatory onboarding wizard step for memory embedding provider, model, API-key, and readiness setup.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes the existing `agents.defaults.memorySearch` contract, supports SecretRef or API-key input, verifies readiness, and documents daemon and Codex OAuth caveats.
