## Summary

Keep open. Current main still documents and diagnoses memory embedding configuration, but `openclaw setup`, onboarding, and `openclaw configure` do not provide the requested memory embedding setup path; the mandatory/default-on question remains a product and credential-policy decision.

## What This Changes

Request adds a prominent or mandatory onboarding/configure step for `agents.defaults.memorySearch` provider, model, API-key or SecretRef setup, plus an embedding readiness check.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures existing `agents.defaults.memorySearch` fields, supports the chosen SecretRef/plaintext policy, verifies readiness through a controlled status or embedding probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main source inspection shows plain setup only writes workspace/gateway defaults, onboarding has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear. A dedicated setup/configure flow is likely the right implementation area, but mandatory versus optional/default-on behavior, credential storage, SecretRef UX, and probe semantics need maintainer direction first.
