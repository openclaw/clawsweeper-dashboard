## Summary

Keep open. Current main has memorySearch docs, doctor diagnostics, and memory status probing, but setup/onboarding/configure still do not provide the requested memory embedding provider/model/API-key or SecretRef setup path; making it mandatory is a product and credential-policy decision.

## What This Changes

Add a prominent or mandatory onboarding/configure step for memory embedding provider, model, credential setup, and readiness verification.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes existing `agents.defaults.memorySearch` fields, supports the chosen SecretRef/plaintext policy, verifies readiness through a controlled status or embedding probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main source inspection shows plain setup only writes workspace/gateway defaults, onboarding has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear. A dedicated setup/configure memory flow is likely the right implementation area, but mandatory versus optional/default-on behavior, credential storage, SecretRef UX, and probe semantics need maintainer direction first.
