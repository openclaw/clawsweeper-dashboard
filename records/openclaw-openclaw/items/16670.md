## Summary

Keep open. Current main still has manual memorySearch docs, doctor guidance, and memory status probes, but setup/onboarding/configure do not provide a dedicated memory embedding provider/model/API-key or SecretRef step; the mandatory UX requires maintainer product and credential-policy judgment.

## What This Changes

Request adding a prominent or mandatory onboarding/configure step for `agents.defaults.memorySearch` provider/model/API-key setup and readiness verification.

## Best Possible Solution

Add a maintainer-approved memory setup section in CLI onboarding and `openclaw configure` that writes `agents.defaults.memorySearch`, supports the chosen credential storage policy, verifies readiness through a controlled probe/status path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the gap: setup only writes workspace/gateway defaults, the setup wizard has no memory step, and configure has no memory section or dispatch path.

## Solution Assessment

Unclear. A dedicated memory setup path is the right implementation area, but making it mandatory is not clearly the best UX until maintainers decide the credential, SecretRef, and live-probe policy.
