## Summary

Keep open. Current main and v2026.5.12 still have memorySearch docs and diagnostics, but `openclaw setup`, the setup wizard, and `openclaw configure` do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step; the requested mandatory/default-on/optional policy remains a maintainer product decision.

## What This Changes

Request to add a prominent mandatory onboarding step that configures `memorySearch` embedding provider/model credentials and verifies embedding readiness.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes existing `memorySearch` fields, follows the chosen credential policy, verifies readiness through a controlled status/probe path, and updates setup/configure/memory docs.

## Reproduction Assessment

Yes for the source-level gap. Current main and v2026.5.12 show setup/configure/onboarding omit a memory section or dedicated memorySearch prompt while memory docs still require provider credentials for remote embeddings.

## Solution Assessment

Unclear as requested. A dedicated setup/configure path is likely the right implementation area, but mandatory versus optional/default-on behavior and credential/probe policy need maintainer direction first.
