## Summary

Keep open. Current main still has the reported gap: the legacy-workspace detector is a no-op, the doctor warning depends on that empty result, and the docs promise legacy workspace warnings without covering the Docker-to-host `workspace-local` parent plus `workspace-local/local` child split.

## What This Changes

The issue requests doctor/migration diagnostics and docs for Docker-to-host setups split between `~/.openclaw/workspace-local` and `~/.openclaw/workspace-local/local`.

## Best Possible Solution

Add a narrow read-only doctor/status diagnostic that reports the active workspace and suspicious parent-held markers, then document canonical Docker and host workspace locations while keeping any file movement opt-in.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass an active child workspace such as `~/.openclaw/workspace-local/local` to `detectLegacyWorkspaceDirs()` while parent-held operational markers exist; current main still returns no legacy directories, so doctor cannot warn.

## Solution Assessment

Yes. A diagnostic-first doctor/docs fix is the narrowest maintainable solution because it fulfills the documented legacy-workspace warning behavior without changing SecretRef semantics or silently migrating local files.
