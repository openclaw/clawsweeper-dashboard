## Summary

Keep open. Current main and the latest shipped release still leave the legacy workspace detector as a no-op, so doctor/status cannot warn about the reported parent `workspace-local` plus active child `workspace-local/local` split, and the docs still do not explain the Docker-to-host layout case.

## What This Changes

The issue asks for Docker-to-host migration diagnostics and docs for split `workspace-local` and `workspace-local/local` layouts with parent-held operational files.

## Best Possible Solution

Add a read-only doctor/status diagnostic for active child workspaces with suspicious parent-held operational markers, document canonical host and Docker workspace locations, and keep any file movement or repair opt-in.

## Reproduction Assessment

Yes, at source level. Passing an active child workspace such as `workspace-local/local` into `detectLegacyWorkspaceDirs()` still returns an empty `legacyDirs` array, so doctor/status has no path to warn about parent-held files.

## Solution Assessment

Yes. A diagnostic-first doctor/docs repair is the narrowest maintainable solution because it addresses operator confusion without silently migrating files or redefining local secret conventions.
