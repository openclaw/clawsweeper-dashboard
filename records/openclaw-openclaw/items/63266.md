## Summary

Keep open. Current main still has a no-op legacy workspace detector, so doctor cannot warn about the reported `workspace-local` / `workspace-local/local` split; the reporter’s follow-up also confirms the desired fix should stay diagnostic/docs-focused rather than silently migrating secrets.

## What This Changes

Detect and document Docker-to-host workspace splits where the active workspace is a child directory while operational files remain in the parent workspace directory.

## Best Possible Solution

Add a read-only doctor/status diagnostic plus docs that clearly report the active workspace and suspicious parent-held files, while leaving any file movement or migration as an explicit future repair path.

## Reproduction Assessment

Yes. A source-level reproduction is to pass an active child workspace such as `~/.openclaw/workspace-local/local` to `detectLegacyWorkspaceDirs()` with parent-held operational markers present; current main still returns no legacy directories, so doctor cannot warn.

## Solution Assessment

Yes. The diagnostic-first doctor/docs direction is the narrowest maintainable fix because it addresses operator confusion without changing SecretRef semantics or silently moving local files.
