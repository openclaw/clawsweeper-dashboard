## Summary

Keep open. Current main still lacks the requested doctor/docs behavior: the detector for extra workspaces is a no-op, the status warning is gated on that empty result, and the docs do not explain the reported Docker-to-host parent/child workspace split; the reporter also confirmed the desired fix should stay diagnostic-first rather than normalizing operator secrets.

## What This Changes

The issue requests doctor and documentation handling for Docker-to-host migrations that leave files split between `workspace-local` and `workspace-local/local`.

## Best Possible Solution

Add a read-only doctor/status diagnostic for active child workspaces with suspicious parent-held operational markers, document canonical host and Docker workspace locations, and keep file movement or repair opt-in.

## Reproduction Assessment

Yes at source level. Passing an active child workspace such as `~/.openclaw/workspace-local/local` through the current detector will still produce an empty `legacyDirs` array, so doctor cannot warn about the parent-held files described in the report.

## Solution Assessment

Yes. A diagnostic-first doctor/docs fix is the narrowest maintainable solution because it addresses operator confusion without silently migrating files or changing SecretRef semantics.
