## Summary

Keep open. Current main still lacks the requested diagnostic/docs behavior: the legacy workspace detector is a no-op, the doctor warning is gated on that empty detector result, and the docs describe generic extra workspace folders rather than the reported Docker-to-host `workspace-local` versus `workspace-local/local` split.

## What This Changes

The issue requests read-only doctor/status detection and documentation for Docker-to-host migrations that leave operational files split between `workspace-local` and `workspace-local/local`.

## Best Possible Solution

Add a narrow read-only doctor/status diagnostic that reports the active workspace and suspicious parent-held markers, then document canonical Docker and host workspace locations while leaving file movement as explicit future repair work.

## Reproduction Assessment

Yes. A source-level reproduction is to pass an active child workspace such as `~/.openclaw/workspace-local/local` to `detectLegacyWorkspaceDirs()` with parent-held operational markers present; current main still returns no legacy directories, so doctor cannot warn.

## Solution Assessment

Yes. The diagnostic-first doctor/docs direction is the narrowest maintainable solution because it addresses operator confusion without changing SecretRef semantics or silently migrating local files.
