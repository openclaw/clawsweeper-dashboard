## Summary

Keep open. Current main still does not implement the requested doctor/docs behavior: the legacy workspace detector is still a no-op, the doctor note depends on that empty detector result, and the docs do not explain the reported Docker-to-host `workspace-local` versus `workspace-local/local` split.

## What This Changes

Request doctor or migration tooling and docs to detect and explain Docker-to-host split workspace layouts with parent-held operational files.

## Best Possible Solution

Add a narrow read-only doctor/status diagnostic for active child workspaces with suspicious parent-held markers, document canonical host and Docker workspace locations, and leave any file movement or repair opt-in.

## Reproduction Assessment

Yes at source level. Passing an active child workspace such as `~/.openclaw/workspace-local/local` to `detectLegacyWorkspaceDirs()` still returns an empty `legacyDirs` array, so doctor cannot warn about parent-held markers.

## Solution Assessment

Yes. A diagnostic-first doctor/docs fix is the narrowest maintainable solution because it addresses operator confusion without silently migrating files or changing SecretRef semantics.
