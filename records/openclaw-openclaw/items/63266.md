## Summary

Keep open. Current main still has the legacy workspace detector as a no-op, so doctor/status cannot warn about the reported active child workspace with parent-held operational files, and the docs still do not explain the Docker-to-host parent/child workspace split.

## What This Changes

The issue requests Docker-to-host migration diagnostics and guidance for split `workspace-local` and `workspace-local/local` layouts that can confuse local operational file lookups.

## Best Possible Solution

Add a read-only doctor/status diagnostic for active child workspaces with suspicious parent-held markers, document canonical host and Docker workspace locations, and keep any file movement or repair opt-in.

## Reproduction Assessment

Yes, at source level. On current main, passing an active child workspace such as `workspace-local/local` into `detectLegacyWorkspaceDirs()` always returns an empty `legacyDirs` array, so doctor/status cannot warn about parent-held files.

## Solution Assessment

Yes. A diagnostic-first doctor/docs repair is the narrowest maintainable path because it addresses operator confusion without silently migrating files or redefining local secret conventions.
