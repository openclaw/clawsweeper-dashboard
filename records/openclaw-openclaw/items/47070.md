## Summary

Keep open. Current main still starts native `sessions_spawn` child `agent` runs without `inputProvenance`, while existing inter-session plumbing already supports and uses that metadata; the issue is also paired with an open draft PR that closes it, so it should remain open until that fix lands or is replaced.

## What This Changes

Request to add `inputProvenance` metadata to child `agent` runs launched by `sessions_spawn`, preserving requester session/channel lineage and `sourceTool: "sessions_spawn"`.

## Best Possible Solution

Refresh and land, or replace, the linked draft with a narrow `spawnSubagentDirect()` change that uses the existing `InputProvenance` contract and adds focused mocked-gateway coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` request params; current main constructs that request without `inputProvenance`.

## Solution Assessment

Yes. Adding existing `InputProvenance` metadata at the native spawn boundary is the narrowest maintainable fix; a broader provenance schema or trust-label redesign is unnecessary for this report.
