## Summary

Current main still starts `sessions_spawn` child `agent` runs without `inputProvenance`, and the issue remains paired with an open draft PR that has closing linkage rather than a merged fix.

## What This Changes

The issue requests explicit inter-session `inputProvenance` on child tasks created through `sessions_spawn`, including `sourceTool: "sessions_spawn"` and requester session/channel lineage.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow `spawnSubagentDirect()` change using the existing `InputProvenance` contract and focused mocked-gateway coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` request params; current main constructs that request without `inputProvenance`.

## Solution Assessment

Yes. Adding existing `InputProvenance` metadata at the native spawn boundary is the narrowest maintainable fix; a broader provenance schema or trust-label redesign is unnecessary for this report.
