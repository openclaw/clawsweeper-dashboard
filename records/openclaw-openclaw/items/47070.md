## Summary

Keep open: current main and latest release still omit `inputProvenance` from native and ACP `sessions_spawn` child `agent` dispatches, and the linked closing PR is still open, draft, conflicting, and currently covers only the native path.

## What This Changes

The issue requests explicit inter-session `inputProvenance` on child tasks created through `sessions_spawn`, including `sourceTool: "sessions_spawn"` and requester session/channel lineage.

## Best Possible Solution

Refresh and land, or replace, `https://github.com/openclaw/openclaw/pull/47087` with a narrow fix that attaches the existing inter-session provenance payload for both native and ACP spawn child dispatches.

## Reproduction Assessment

Yes, at source level. Current main’s native and ACP spawn child `agent` requests omit `inputProvenance`, while the gateway and `sessions_send` path already support and use that field.

## Solution Assessment

Yes for the requested direction, but the active PR is incomplete unless it covers ACP too. The maintainable fix is to attach the existing provenance payload at the child dispatch boundary without adding a new schema or policy system.
