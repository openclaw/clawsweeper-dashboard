## Summary

Keep this open: current main and v2026.6.10 still omit `inputProvenance` on the spawned child `agent` call, and the linked implementation remains open, draft, and conflicting rather than merged.

## What This Changes

Attach explicit inter-session `inputProvenance` with `sourceTool: "sessions_spawn"` to child tasks created through `sessions_spawn`.

## Best Possible Solution

Keep this issue as the canonical tracker until https://github.com/openclaw/openclaw/pull/47087 is refreshed and landed, or replaced by the same narrow fix at the `spawnSubagentDirect` child `agent` call.

## Reproduction Assessment

Yes, at source level. Current main’s `spawnSubagentDirect` child `agent` params omit `inputProvenance`, while sibling inter-session paths already pass provenance metadata; no runtime test was run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing `InputProvenance` envelope at the `spawnSubagentDirect` child `agent` boundary with `sourceTool: "sessions_spawn"`, not to add a new schema or broader trust system.
