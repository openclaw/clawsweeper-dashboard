## Summary

Keep open. Current main still omits `inputProvenance` when the native `sessions_spawn` path starts the child `agent` run, while existing gateway and `sessions_send` plumbing already support inter-session provenance. Open draft PR #47087 is linked with closing syntax for the same work, so this issue should remain open until that PR or an equivalent replacement lands.

## What This Changes

Request to attach `inter_session` input provenance, including `sourceTool: "sessions_spawn"` and requester lineage, to child tasks created by `sessions_spawn`.

## Best Possible Solution

The desired end state is a narrow native-spawn change in `spawnSubagentDirect()` that builds an `inter_session` `InputProvenance`, preserves requester session/channel when present, sets `sourceTool: "sessions_spawn"`, passes it to the child gateway `agent` call, and adds focused coverage through the existing mocked spawn gateway seam.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` request params; current main has no `inputProvenance` field on that request.

## Solution Assessment

Yes. Reusing the existing `InputProvenance` contract at the native spawn boundary is the narrowest maintainable solution; a broader provenance schema or trust-label redesign is unnecessary for this report.
