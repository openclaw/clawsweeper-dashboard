## Summary

Keep open: current main still starts native `sessions_spawn` child `agent` runs without `inputProvenance`, and the linked implementation PR remains open, draft, and conflicting.

## What This Changes

The issue requests adding inter-session `inputProvenance` to child tasks launched by `sessions_spawn`, preserving source session/channel and `sourceTool: "sessions_spawn"`.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow spawn-boundary provenance change and focused mocked-gateway coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` request params; current main constructs that request without `inputProvenance`.

## Solution Assessment

Yes. Attaching the existing `InputProvenance` payload at the native spawn boundary is the narrowest maintainable fix; a broader provenance schema or trust-label redesign is unnecessary.
