## Summary

Current main still omits `inputProvenance` on `sessions_spawn` child task starts, and the linked implementation PR is open, draft, and conflicting, so this issue should stay open until that PR or an equivalent narrow replacement lands. ## What This Changes

Attach inter-session `inputProvenance`, including `sourceTool: "sessions_spawn"` and requester lineage, to child tasks started through `sessions_spawn`.

## Best Possible Solution

Refresh, review, and land https://github.com/openclaw/openclaw/pull/47087 or an equivalent narrow replacement that attaches existing input provenance metadata at every relevant `sessions_spawn` child-start boundary.

## Solution Assessment

Yes, attaching the existing `InputProvenance` shape at the child run dispatch boundary is the narrowest maintainable fix. The remaining solution check is whether the linked PR should also cover ACP `sessions_spawn` before maintainers call the issue fixed.
