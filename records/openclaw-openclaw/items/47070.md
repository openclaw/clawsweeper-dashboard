## Summary

Keep open: current main still starts native and ACP `sessions_spawn` child `agent` runs without `inputProvenance`, and the linked implementation PR is open, draft, and conflicting. Because that PR uses closing syntax for this issue, this should stay open until that PR is refreshed and landed or replaced.

## What This Changes

Attach explicit inter-session input provenance to child tasks launched through `sessions_spawn`.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow spawn-boundary provenance fix covering native and ACP spawn behavior plus focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to inspect `spawnSubagentDirect()` and the ACP spawn dispatch on current main and observe that their child `agent` params do not include `inputProvenance`.

## Solution Assessment

Yes. The narrow maintainable fix is to attach the existing inter-session `InputProvenance` payload at the child `agent` dispatch boundary, preserving requester session/channel and setting `sourceTool: "sessions_spawn"`.
