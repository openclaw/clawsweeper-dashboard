## Summary

Keep open: current main still starts `sessions_spawn` child `agent` runs without `inputProvenance`, and the linked implementation PR is still open, draft, and conflicting. Because that PR uses closing syntax for this issue, the issue should stay open until that PR is refreshed and landed or replaced.

## What This Changes

The issue requests explicit inter-session input provenance on child tasks started through `sessions_spawn`, including `sourceTool: "sessions_spawn"` and requester session/channel lineage.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow spawn-boundary provenance fix covering native and ACP spawn behavior plus focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call `spawnSubagentDirect()` or inspect the ACP spawn gateway dispatch and observe that the child `agent` params on current `main` do not include `inputProvenance`.

## Solution Assessment

Yes. The narrow maintainable fix is to attach the existing inter-session `InputProvenance` payload at the child `agent` dispatch boundary, preserving requester session/channel and setting `sourceTool: "sessions_spawn"`.
