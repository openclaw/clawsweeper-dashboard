## What This Changes

Attach inter-session `inputProvenance` to child agent runs created through `sessions_spawn`.

## Best Possible Solution

## Reproduction Assessment

Yes. A high-confidence source reproduction is to inspect `spawnSubagentDirect()` and the ACP spawn dispatch on current main and observe that their child `agent` params do not include `inputProvenance`.

## Solution Assessment

Yes. The narrow maintainable fix is to attach the existing inter-session `InputProvenance` payload at both child `agent` dispatch boundaries, preserving requester session/channel and setting `sourceTool: "sessions_spawn"`.
