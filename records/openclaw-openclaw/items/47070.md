## Summary

Keep open: current main still omits `inputProvenance` on native and ACP `sessions_spawn` child `agent` requests, and https://github.com/openclaw/openclaw/pull/47087 remains open with closing syntax, so this issue should track that fix rather than close or queue duplicate repair.

## What This Changes

Request that `sessions_spawn` child tasks receive inter-session `inputProvenance` carrying requester session/channel lineage and `sourceTool: "sessions_spawn"`.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow fix covering both native and ACP `sessions_spawn` child dispatches plus focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP child `agent` dispatches omit `inputProvenance` while the gateway accepts it and `sessions_send` already passes it.

## Solution Assessment

Yes. The narrow maintainable solution is to attach the existing inter-session `InputProvenance` payload at both child `agent` dispatch boundaries, preserving requester session/channel and setting `sourceTool: "sessions_spawn"`.
