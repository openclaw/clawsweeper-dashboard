## Summary

Keep open. Current main still does not attach `inputProvenance` to the native `sessions_spawn` child `agent` request, and open draft PR #47087 already targets this exact issue with closing linkage.

## What This Changes

Attach explicit inter-session `inputProvenance` metadata to child tasks created through `sessions_spawn`.

## Best Possible Solution

Land or replace the linked draft with a narrow change in `spawnSubagentDirect()` that builds `inter_session` provenance, sets `sourceTool: "sessions_spawn"`, preserves requester session/channel where available, and adds mocked-gateway coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` request params; current main constructs that request without `inputProvenance`.

## Solution Assessment

Yes. Reusing the existing `InputProvenance` contract at the native spawn boundary is the narrowest maintainable fix; a broader provenance schema redesign is unnecessary for this report.
