- `impact:session-state`: The issue is about preserving spawned task lineage so child session input is not misclassified as first-party user input.

## Summary

Keep open: current main still leaves spawned child agent requests without explicit input provenance, and an open linked PR with closing syntax is the active implementation path rather than a reason to close early.

## What This Changes

Request to attach inter-session input provenance, including `sourceTool: "sessions_spawn"` and requester lineage, to child tasks created through `sessions_spawn`.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow fix covering both native and ACP `sessions_spawn` child dispatches plus focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main shows both native and ACP child `agent` dispatches omit `inputProvenance`, while the gateway accepts it and existing inter-session paths already pass it.

## Solution Assessment

Yes for the requested direction. The maintainable fix is to attach the existing inter-session provenance payload at both child dispatch boundaries, preserving requester session/channel and setting `sourceTool: "sessions_spawn"`.
