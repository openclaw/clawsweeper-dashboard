## Summary

Keep open. Current main still does not attach `inputProvenance` to the native `sessions_spawn` child `agent` call, and open draft PR #47087 is already linked with closing syntax for this exact fix.

## What This Changes

The issue requests adding inter-session input provenance to child tasks launched through `sessions_spawn`.

## Best Possible Solution

The desired end state is a narrow native-spawn change in `spawnSubagentDirect()` that builds an `inter_session` provenance payload, preserves requester session/channel where available, sets `sourceTool: "sessions_spawn"`, and adds focused mocked-gateway coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` request params; current main constructs that request without `inputProvenance`.

## Solution Assessment

Yes. Reusing the existing `InputProvenance` contract at the native spawn boundary is the narrowest maintainable fix; a broader provenance schema redesign is unnecessary for this report.
