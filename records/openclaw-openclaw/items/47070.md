## Summary

Keep open: current main still lacks `inputProvenance` on the native and ACP `sessions_spawn` child `agent` dispatches, and the linked implementation PR is still open with closing syntax for this issue.

## What This Changes

Attach inter-session input provenance, including `sourceTool: "sessions_spawn"`, to child tasks started through `sessions_spawn`.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow provenance fix covering native and ACP `sessions_spawn` plus focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to inspect the native and ACP `sessions_spawn` dispatches on current main and observe that their child `agent` params do not include `inputProvenance`.

## Solution Assessment

Yes. The narrow maintainable fix is to attach the existing inter-session `InputProvenance` payload at both child `agent` dispatch boundaries, preserving requester session/channel and setting `sourceTool: "sessions_spawn"`.
