## Summary

Keep open: current `main` still starts native and ACP `sessions_spawn` child `agent` runs without `inputProvenance`, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/47087 remains open, draft, conflicting, and tied to this issue with closing syntax.

## What This Changes

Attach inter-session input provenance, including `sourceTool: "sessions_spawn"` and requester session/channel lineage, to child tasks started through `sessions_spawn`.

## Best Possible Solution

Refresh and land, or replace, https://github.com/openclaw/openclaw/pull/47087 with a narrow change that attaches the existing `InputProvenance` payload at the native and ACP spawn dispatch boundaries, plus focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of current `main` shows both child gateway `agent` dispatches for `sessions_spawn` omit `inputProvenance`, while the existing schema and helper code already support that metadata.

## Solution Assessment

Yes. Passing the existing inter-session provenance object at the child dispatch boundary is the narrowest maintainable fix; the active PR should be refreshed to cover ACP spawn or deliberately scope that path.
