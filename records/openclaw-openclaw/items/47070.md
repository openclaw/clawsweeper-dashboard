## Summary

Current main still omits `inputProvenance` from the native and ACP `sessions_spawn` child `agent` dispatches, and the linked implementation PR is still open, draft, and conflicting, so this issue remains necessary. The IdentyClaw comment is useful adjacent plugin-level lineage context, but it does not replace the core session provenance requested here.

## What This Changes

Attach explicit inter-session `inputProvenance`, including `sourceTool: "sessions_spawn"`, to child tasks started through `sessions_spawn`.

## Best Possible Solution

Refresh, review, and land https://github.com/openclaw/openclaw/pull/47087 or an equivalent narrow replacement that attaches existing input provenance metadata at every supported `sessions_spawn` child-start boundary.

## Reproduction Assessment

Yes from source inspection: current native and ACP `sessions_spawn` child `agent` calls omit `inputProvenance`, while sibling inter-session paths already pass it. I did not run a live spawn because this review is read-only.

## Solution Assessment

Yes on direction: attaching the existing `InputProvenance` shape at child run dispatch is the narrow maintainable fix. The linked PR should be refreshed and checked for ACP sibling coverage before maintainers call the issue fixed.
