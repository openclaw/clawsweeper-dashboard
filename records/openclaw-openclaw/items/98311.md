- `impact:message-loss`: Image/file replies can fail to reach the Feishu chat when the reply target has been withdrawn or deleted.

## Summary

Keep open: current main still has the Feishu text/card withdrawn-target fallback, but image/file media replies still take bare `im.message.reply` paths that can throw before any top-level `create` fallback. This is a narrow Feishu message-loss bug with clear source evidence and no canonical open PR owning this exact fix.

## What This Changes

The issue reports that Feishu image and file replies drop media when replying to withdrawn or missing message targets instead of falling back to top-level delivery like text/card replies.

## Best Possible Solution

Apply a narrow Feishu plugin fix that shares the established withdrawn/not-found fallback semantics across text, card, image, and file reply sends without changing unrelated media upload or unsafe-media fallback behavior.

## Reproduction Assessment

Yes from source inspection: text/card route through `sendReplyOrFallbackDirect`, while media reply senders still assert the direct reply result and never call create on withdrawn/not-found responses. I did not run a live or synthetic test in this read-only review.

## Solution Assessment

Yes: reusing the existing text/card fallback semantics in the Feishu media senders is the narrowest maintainable fix and keeps native thread fail-closed behavior intact.
