## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80309. The scheduled OpenClaw Live/E2E workflow calls `.github/workflows/openclaw-live-and-e2e-checks-reusable.yml` without passing `actions: read`, while the reusable `prepare_docker_e2e_image` job requests `actions: read`; GitHub reusable workflows cannot elevate token permissions from the caller. Add `actions: read` to the scheduled caller job permissions in `.github/workflows/openclaw-scheduled-live-checks.yml`, and add it at top level too only if needed for consistency. Do not broaden beyond read permission, do not alter the secret list, and do not change live/E2E matrices. Compare the existing caller patterns in `.github/workflows/openclaw-release-checks.yml` and `.github/workflows/plugin-prerelease.yml`. Changelog is not needed for this CI-only fix.
