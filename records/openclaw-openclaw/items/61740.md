## Summary

Keep open. Current main still has static prompt-based skill selection plus operator-facing ClawHub search, but no agent-callable runtime skill matcher or structured trusted routing metadata, so the core feature/API question remains unresolved.

## What This Changes

The issue requests runtime skill discovery and matching so agents can find relevant skills during a turn instead of relying only on the startup `<available_skills>` prompt.

## Best Possible Solution

Design an additive, source-aware runtime skill-matching contract that respects installed/local/ClawHub source boundaries, per-agent visibility, trusted metadata semantics, and related local-indexing follow-ups.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a bug reproduction. Source inspection gives high confidence current main has no agent-callable `skill_search` or `skill_match` tool, and the existing `skills.search` path is operator ClawHub discovery.
