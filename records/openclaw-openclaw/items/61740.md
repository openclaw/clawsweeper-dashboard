## Summary

Keep open. Current main and the latest release still use prompt-based skill selection plus operator-facing ClawHub search, while this issue asks for an agent-facing runtime matcher and trusted routing metadata that remain product and security design work.

## What This Changes

The issue requests runtime skill discovery, richer skill metadata, and tiered skill loading so agents can match user intent to available skills more reliably.

## Best Possible Solution

Design an additive, source-aware runtime skill-matching contract that coordinates with the open progressive-disclosure PR and narrower skill-indexing reports while preserving installed/local/ClawHub source boundaries, per-agent visibility, trusted metadata semantics, and documented rollout behavior.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a bug reproduction. Source inspection confirms current main has prompt-based skill selection and operator-facing ClawHub search, not an agent-callable matcher.

## Solution Assessment

Unclear as a direct implementation plan. The proposal identifies a real gap, but maintainers should first define the matcher API, source scope, metadata contract, rollout config, and trust boundary.
