## Summary

Keep open. Current main still exposes skills through prompt snapshots and operator-facing ClawHub search, but it does not provide an agent-callable runtime skill matcher, structured routing metadata, or source-aware matching contract; the external router prototype at https://github.com/openclaw/openclaw/issues/81826 was closed as ClawHub scope while this broader core API discussion remains unresolved.

## What This Changes

Request an agent-callable runtime skill search/matching API with structured skill metadata and tiered skill loading.

## Best Possible Solution

Design an additive, source-aware runtime skill-matching contract that respects per-agent allowlists, separates installed/local/ClawHub sources, defines trusted metadata semantics, and coordinates with the narrower trigger and local-indexing follow-ups.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a bug reproduction. Source inspection gives high confidence current main has no agent-callable `skill_search` or `skill_match` tool, and the existing `skills.search` path is operator ClawHub discovery.

## Solution Assessment

Unclear as a direct implementation plan. The proposal identifies a real gap, but it combines runtime matching, semantic ranking, metadata schema, lazy loading, and tier policy, so maintainers should define the API and trust boundary before a focused implementation PR.
