## Summary

Keep open. Current main has load-time skill snapshots, per-agent allowlists, hot refresh, plugin-shipped skills, and operator-facing ClawHub search, but it still lacks the requested agent-callable runtime skill-discovery tool and structured SKILL.md capabilities contract; the remaining work needs product/API and security design rather than conservative cleanup.

## What This Changes

Request a runtime skill search/discovery capability, richer SKILL.md capability metadata, and tiered skill loading so agents can match installed or discoverable skills to user intent during a session.

## Best Possible Solution

Keep this open for maintainer design of a backward-compatible discovery contract that preserves allowlists and treats third-party skill metadata or installation as an explicit trust boundary.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main gap is reproducible by inspection: skills are prompt-snapshot based and operator ClawHub search is not exposed as an agent runtime skill-search tool.

## Solution Assessment

Unclear. The proposal identifies a real gap, but it combines runtime search, semantic ranking, SKILL.md schema expansion, lazy loading, and tier policy, so the best implementation needs maintainer API and security review before a narrow PR is scoped.
