## Summary

Keep this PR open: the provider-local fix direction and supplied live proof are useful, but the current branch still regresses explicit custom Fal `/edit` model paths and leaves the public image-generation docs out of sync.

## What This Changes

The PR updates the bundled Fal image-generation provider and changelog so GPT Image 2 and Nano Banana 2 reference-image edits use `/edit`, plural `image_urls`, higher edit input limits, and model-specific edit geometry.

## Best Possible Solution

Merge a provider-local fix after preserving explicit `/edit` paths, adding focused Fal model-family tests, and updating the image-generation docs to describe GPT/Nano Banana versus Flux edit limits.

Yes, by source inspection: current main maps `openai/gpt-image-2` with input images to `/image-to-image`, sends singular `image_url`, and rejects edit `aspectRatio`, while Fal documents `/edit` with `image_urls`. I did not run a live Fal request in this read-only review.

## Solution Assessment

No. The provider-local direction is right, but the current branch should preserve exact custom `/edit` paths and update docs/tests before it is the narrowest maintainable fix.
