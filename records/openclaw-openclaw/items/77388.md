## Summary

Keep open: this PR targets a real Fal provider bug and includes sufficient live-output proof, but the current head still regresses explicit custom `/edit` model paths and leaves user-facing image-generation docs inconsistent with the new Fal capabilities.

## What This Changes

The PR updates the bundled Fal image provider and changelog so GPT Image 2 and Nano Banana 2 edit requests use `/edit`, plural `image_urls`, higher edit input limits, and model-specific edit geometry.

## Best Possible Solution

Merge a provider-local Fal fix after preserving explicit `/edit` paths, adding focused Fal model-family regression tests, updating the image-generation docs, and getting the head checks green.

## Reproduction Assessment

Yes, by source inspection: current main routes GPT Image 2/Nano Banana 2 reference-image requests to `/image-to-image`, sends singular `image_url`, and rejects edit `aspectRatio` while Fal documents `/edit` with `image_urls`. I did not run a live Fal request in this read-only review.

## Solution Assessment

No. The provider-local approach is the right direction, but the current branch should preserve exact custom `/edit` endpoints, add focused tests, and update docs before it is the narrowest maintainable fix.
