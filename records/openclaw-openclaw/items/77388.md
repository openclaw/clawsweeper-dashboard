## Summary

Keep this PR open: the linked Fal GPT Image 2 routing bug is valid and the patch moves in the right direction, but it regresses existing Flux edit behavior by applying GPT/Nano Banana request shape and capabilities to every Fal edit request.

## What This Changes

The PR modifies the bundled Fal image-generation provider to route GPT Image 2 and Nano Banana 2 edits to `/edit`, send `image_urls`, allow edit aspect ratios, and raise the Fal edit input-image limit.

## Best Possible Solution

Keep the GPT Image 2/Nano Banana 2 `/edit` fix, but make endpoint routing, payload fields, edit limits, and aspect-ratio handling model-specific so Flux remains on its documented `/image-to-image` contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `fal/openai/gpt-image-2` with input images routes to `/image-to-image` and sends `image_url`, while Fal's public schemas for GPT Image 2/Nano Banana 2 edits require `/edit` plus `image_urls`.

## Solution Assessment

No. The PR fixes the reported model family by broadening Fal edit behavior globally, but the narrow maintainable fix is to branch request shape and capability enforcement by Fal endpoint/model and preserve existing Flux behavior.
