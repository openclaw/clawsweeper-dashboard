## Summary

Keep open: the provider-local direction is right for the linked Fal edit bug, but the latest PR head still has merge-blocking schema/routing regressions and lacks the required release-note follow-through.

## What This Changes

The PR changes the bundled Fal image provider to route GPT Image 2 and Nano Banana edit requests to `/edit`, send edit references as `image_urls`, raise edit input limits, and allow edit aspect ratios.

## Best Possible Solution

Keep the provider-local Fal repair, but preserve explicit edit endpoints, serialize request fields by Fal model family, and add regression tests plus docs/changelog updates for the changed Fal capabilities.

## Reproduction Assessment

Yes. Source inspection of current main shows GPT Image 2 edits route to `/image-to-image` with a singular `image_url`, while Fal’s schema requires `/edit` and `image_urls`.

## Solution Assessment

No. The PR’s direction is maintainable, but it should be model-family-specific so Nano Banana 2 geometry uses `aspect_ratio`/`resolution` and existing explicit Fal `/edit` endpoints remain exact.
