## Summary

Keep open: the PR targets a valid Fal edit bug, but the latest head still has merge-blocking provider regressions plus required release/docs follow-up.

## What This Changes

The PR changes the Fal image provider to route GPT Image 2 and Nano Banana input-image requests to `/edit`, send `image_urls`, raise edit input limits, and permit edit aspect ratios.

## Best Possible Solution

Keep the fix provider-local with model-family-specific endpoint and payload serialization, preserve existing explicit endpoint overrides and Flux behavior, then cover it with Fal provider tests plus docs/changelog updates.

## Reproduction Assessment

Yes. Source inspection on current main plus Fal's official schema pages shows the current Fal provider sends GPT Image 2/NB2 edit traffic to the wrong endpoint/payload shape and blocks supported multi-image/aspect-ratio edit paths.

## Solution Assessment

No. The provider-local direction is right, but the current diff should split endpoint and geometry serialization by Fal model family instead of dropping generic `/edit` preservation and serializing NB2 geometry as GPT-style `image_size`.
