## Summary

Keep open: the PR targets a valid Fal provider bug, but the current head introduces a blocking `model` temporal-dead-zone failure before any Fal image request can be built, and the model-specific request handling still needs tightening before merge.

## What This Changes

The PR changes the bundled Fal image provider to route GPT Image 2 and Nano Banana edit requests to `/edit`, send multiple references as `image_urls`, allow edit aspect ratios, and raise edit input-image limits.

## Best Possible Solution

Keep the provider-local Fal repair, but make endpoint routing, request fields, geometry serialization, runtime guards, docs, tests, and changelog model-family-specific while preserving existing Flux behavior and explicit Fal `/edit` endpoints.

## Reproduction Assessment

Yes. Source inspection of current main shows `fal/openai/gpt-image-2` edits would post to `/openai/gpt-image-2/image-to-image` with `image_url`, and PR-head source inspection shows the proposed branch would fail before request construction by reading `model` before initialization.

## Solution Assessment

No. The intended provider-local direction is right, but the current PR head is not mergeable because it introduces a blocking initialization bug and still needs model-specific geometry handling, regression tests, docs/changelog alignment, and preservation of explicit `/edit` paths.

  - confidence: 0.84
