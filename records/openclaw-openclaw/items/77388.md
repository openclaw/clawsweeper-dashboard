## Summary

Keep open: the PR targets a real Fal provider bug and includes usable live-output proof, but the current head still has blocking request-routing and Nano Banana 2 geometry serialization defects.

## What This Changes

The branch changes the Fal image-generation provider and changelog so GPT Image 2 and Nano Banana 2 reference-image requests use `/edit`, plural `image_urls`, higher edit image limits, and edit-mode geometry hints.

## Best Possible Solution

Keep the provider-local fix, preserve exact model endpoint overrides, build Fal edit payloads by model family using documented schema values, and add focused Fal provider tests plus the matching image-generation docs update.

## Reproduction Assessment

Yes. Source inspection of current main plus Fal's documented endpoint schemas gives a high-confidence reproduction path for the wrong GPT Image 2/Nano Banana 2 edit route, one-image cap, edit aspect-ratio rejection, and singular payload field.

## Solution Assessment

No. The provider-local direction is right, but this patch is not yet the best mergeable form because it regresses explicit `/edit` overrides and still serializes Nano Banana 2 geometry with GPT/Flux value shapes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve explicit Fal edit endpoints:** `extensions/fal/image-generation-provider.ts:109-115`
  - body: Current main treats any model ending in `/edit` as an exact endpoint override. This branch removes that generic guard for non-GPT/Nano Banana paths, so a custom Fal model like `fal-ai/example/edit` with a reference image is rewritten to `fal-ai/example/edit/image-to-image` and the request no longer reaches the intended endpoint.
  - confidence: 0.9
- **[P2] Send Nano Banana geometry values in Fal's schema:** `extensions/fal/image-generation-provider.ts:358-360`
  - body: For Nano Banana 2 edit requests, `imageSize` still comes from the GPT/Flux helper, so `aspectRatio: "16:9"` becomes `landscape_16_9` and `resolution: "2K"` becomes `2048`. Fal's NB2 edit schema expects `aspect_ratio` values like `16:9` and `resolution` values like `2K`, so the requested geometry can be rejected or ignored.
  - confidence: 0.93
- **[P3] Update the image-generation docs:** `extensions/fal/image-generation-provider.ts:283-286`
  - body: The branch advertises Fal edit support for up to 14 images and aspect-ratio hints, but `docs/tools/image-generation.md` still says Fal edits support one image and aspect ratio is generate-only. Please update the provider table and image-editing section so the shipped docs match the new behavior.
  - confidence: 0.86
