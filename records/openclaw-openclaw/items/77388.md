## Summary

Keep open: the PR targets a real Fal provider bug and includes useful live proof, but the current branch still has narrow correctness blockers before merge.

## What This Changes

The PR changes the Fal image provider to route GPT Image 2 and Nano Banana 2 reference-image requests to `/edit`, allow up to 14 input images, allow edit aspect ratios, and choose `image_urls` vs `image_url` by model family.

## Best Possible Solution

Keep the provider-local repair, but preserve exact endpoint overrides, serialize GPT Image 2 and Nano Banana 2 request fields by model family, and add regression tests plus docs/changelog updates.

## Reproduction Assessment

Yes. Source inspection of current main shows the wrong `/image-to-image` route, one-image cap, edit aspect-ratio rejection, and singular `image_url`; the contributor also supplied copied live output showing the corrected Fal routes working after a hotfix.

## Solution Assessment

No. The PR's provider-local direction is right, but this patch is not yet the best mergeable form because it regresses generic explicit `/edit` overrides and does not serialize Nano Banana 2 geometry with Fal's schema.
