Keep open: the branch fixes the real Fal routing/payload bug and has usable live-output proof, but it still advertises and exposes a 14-reference Fal edit cap while the shared `image_generate` tool rejects more than 5 references before the provider is called.

## What This Changes

The PR updates the bundled Fal image provider, Fal docs, image-generation docs, tests, and changelog for GPT Image 2/Nano Banana 2 `/edit` routing, `image_urls`, model-specific geometry, and multi-reference edits.

## Best Possible Solution

Land a provider-local Fal fix after aligning the shared `image_generate` reference-image parser/schema with the advertised 14-image Fal edit capability and adding focused regression coverage for both the route/payload split and the higher cap.

## Reproduction Assessment

Yes. Current main is source-reproducible for the linked routing bug, and the remaining PR defect is source-reproducible because the PR head still calls `normalizeMediaReferenceInputs` with a hard 5-reference cap before Fal provider validation.

## Solution Assessment

No, not yet. The provider-local routing and payload split are the right direction, but the shared tool input cap must be updated or made provider-aware before the PR actually supports the documented 14-image Fal edit path.
