## Summary

Keep open: the provider-local fix is useful and real behavior proof is present, but the current branch still regresses explicit custom Fal `/edit` model paths and leaves shipped image-generation docs/tests out of sync.

## What This Changes

The PR changes the bundled Fal image-generation provider and changelog so GPT Image 2 and Nano Banana 2 reference-image requests use `/edit`, plural `image_urls`, higher edit image limits, and model-specific edit geometry.

## Best Possible Solution

Merge a provider-local fix after preserving exact `/edit` paths, adding model-family Fal tests, and updating the image-generation docs to describe GPT/Nano Banana versus Flux edit limits.

## Reproduction Assessment

Yes. Source inspection of current main plus Fal's documented schemas gives a high-confidence path for the wrong GPT Image 2/Nano Banana 2 edit route, one-image cap, edit aspect-ratio rejection, and singular payload field.

## Solution Assessment
