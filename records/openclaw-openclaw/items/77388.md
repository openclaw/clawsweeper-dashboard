## Summary

Keep this PR open, but it still needs one blocking fix before merge: the Fal provider changes are directionally correct, while the documented 14/16-reference paths still fail through the public `image_generate` tool because the shared parser caps `images` at five first.

## What This Changes

The PR updates the bundled Fal image-generation provider, Fal docs, image-generation docs, tests, and changelog for GPT Image 2/Nano Banana 2 `/edit` routing, `image_urls`, model-specific edit geometry, and higher multi-reference edit limits.

## Best Possible Solution

Keep the provider-local route/payload split, then align the shared `image_generate` reference-image parser/schema with provider-declared edit limits and add focused tool-level regression coverage.

## Reproduction Assessment

Yes. The original route bug and the remaining PR defect are source-reproducible: current main sends GPT/NB2 edits to the wrong Fal path, and the PR head still leaves `image_generate` capped at five references before provider validation.

## Solution Assessment

No, not yet. The Fal provider routing and payload changes are the right owner-local fix, but the shared tool input cap must be raised or made provider-aware before the documented multi-reference support is usable.
