## Summary

Keep this PR open: it targets a real Fal provider bug and has useful live proof, but the current branch still introduces a regression for exact custom `/edit` Fal endpoints and leaves the user-facing Fal image-generation docs inconsistent with the new capabilities.

## What This Changes

The PR changes the bundled Fal image provider and changelog so GPT Image 2 and Nano Banana 2 reference-image requests use `/edit`, plural `image_urls`, model-specific geometry, and a higher edit input cap while Flux keeps `/image-to-image`.

## Best Possible Solution

Merge a provider-local Fal fix after preserving exact `/edit` endpoint paths, adding focused model-family regression tests, updating the Fal image-generation docs, and confirming the branch checks are green.

## Reproduction Assessment

Yes, by source inspection: current main sends GPT Image 2/Nano Banana 2 reference-image requests through `/image-to-image`, serializes singular `image_url`, caps Fal edits at one reference image, and rejects edit `aspectRatio` while Fal documents `/edit` with `image_urls`. I did not run a live Fal request in this read-only review.

## Solution Assessment

No. The provider-local direction is right, but this branch should preserve exact custom `/edit` endpoints, add focused Fal regression tests, and update docs before it is the narrowest maintainable fix.
