## Best Possible Solution

Doctor should surface plugin drift only when the report is tied to a confirmed running local gateway version, suppressing or clearly separating fallback/no-probe reports, with fresh latest-head terminal proof for the drifted-plugin path.

## Reproduction Assessment

Yes, by source inspection: current main lacks doctor drift reporting, and the PR head's exec SecretRef/no-probe path can forward a `VERSION` fallback drift report to doctor. I did not run live doctor because this review is read-only.

## Solution Assessment

No. Reusing daemon status is the right owner boundary, but the latest implementation should not let doctor present fallback drift as running-gateway drift when the gateway version was not confirmed.
