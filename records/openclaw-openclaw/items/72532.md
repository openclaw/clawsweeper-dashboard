## What This Changes

The PR makes Feishu mention normalization emit `<at>` tags for every mention, including the receiving bot, and updates related strip-mention tests/comments.

## Best Possible Solution

Land one corrected Feishu plugin fix, preferably the narrower leading-addressing approach or equivalent, with regression coverage for multi-bot self-mentions plus direct/group mention-prefixed slash commands.

## Reproduction Assessment

Yes, source-level. With this PR, a direct Feishu message like `@_bot_1 /model` keeps a leading `<at>` tag in `ctx.content`; direct command probing and `CommandBody` retain it, and core command normalization no longer sees a leading slash.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than needed; the safer fix preserves semantic inline self-mentions while keeping leading addressing self-mentions slash-clean, or separates command text from agent-facing mention context.
