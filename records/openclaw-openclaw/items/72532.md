## Summary

Keep open. The Feishu multi-bot NO_REPLY bug is real, but this PR’s blanket self-mention preservation is broader than the command path can safely handle, and the external PR still lacks inspectable real behavior proof.

## What This Changes

The branch makes Feishu `normalizeMentions` preserve the receiving bot’s own `<at>` tag and updates related tests/comments to expect preserved self-mentions.

## Best Possible Solution

Land one corrected Feishu plugin fix that preserves semantic inline self-mentions while keeping leading direct/group slash-command bodies mention-clean, with regression tests and real Feishu proof.

## Reproduction Assessment

Yes, source-level. With this PR, a direct Feishu `@_bot_1 /model` body keeps a leading `<at>` tag in `ctx.content`, direct command probing and `CommandBody` keep that text, and core direct command normalization no longer sees a leading slash.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than needed; the maintainable direction is to preserve inline self-mentions while keeping addressing prefixes command-clean or separating command-probe text from agent-facing context.
