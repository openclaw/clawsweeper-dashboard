The PR makes Feishu `normalizeMentions` ignore `botStripId`, preserving the receiving bot's own `<at>` tag in parsed content, and updates related Feishu tests/comments to expect preserved self-mentions.

## Best Possible Solution

Land one canonical Feishu plugin fix that preserves semantic inline self-mentions for multi-bot routing while keeping leading mention-prefixed command bodies command-clean, with regression coverage and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main strips the receiving bot's own mention via `botStripId`, and this PR's preservation path can be traced to direct command handling where a leading `<at>` tag is not cleaned before command normalization.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than needed; the narrower maintainable solution is to preserve inline self-mentions while keeping leading addressing prefixes clean for command parsing.
