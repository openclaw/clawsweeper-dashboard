## Summary

Keep open: current main still has the Feishu self-mention stripping behavior that this PR tries to address, but this PR is not merge-ready because making `botStripId` a global no-op would regress direct-message slash command handling. The narrower open replacement in #72661 is the better canonical path for the same #72504 bug.

## What This Changes

The PR changes Feishu mention normalization to preserve the receiving bot's own `<at>` tag in all contexts and updates Feishu tests/comments to expect preserved bot self-mentions.

## Best Possible Solution

Land a narrow Feishu plugin fix that strips only the leading addressing self-mention while preserving inline or mid-sentence self-mentions for multi-bot group context, with regression coverage for #72504 plus direct and group slash commands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the PR regression: with this change, a direct `@_bot_1 /model` event normalizes to a leading `<at>` tag, and the direct command path does not strip that tag before slash-leading command normalization.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than needed; the safer maintainable fix is the #72661 approach of preserving semantic inline self-mentions while keeping leading addressing prefixes clean for command dispatch.
