## Summary

Keep this PR open for maintainer handling. The underlying Feishu multi-bot NO_REPLY bug is still present on current main, but this patch fixes it too broadly by preserving the leading self-mention in direct chats, where the command path does not strip `<at>` tags before slash-command normalization; #72661 is the narrower open replacement path for the same root issue.

## What This Changes

This PR changes Feishu mention normalization to preserve the receiving bot's own `<at>` tag in all positions and updates Feishu comments/tests to expect preserved self-mentions.

## Best Possible Solution

Land a narrow Feishu plugin fix that strips only the leading addressing self-mention while preserving inline or mid-sentence self-mentions for multi-bot context, with regression coverage for the multi-bot NO_REPLY case plus direct and group mention-prefixed slash commands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: with this PR, a direct Feishu `@Bot /model` event normalizes to a leading `<at>` tag, and the direct command path does not strip that tag before requiring slash-leading command normalization.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than the maintainable fix; the safer solution is to preserve semantic inline self-mentions while keeping leading addressing prefixes clean for command dispatch.
