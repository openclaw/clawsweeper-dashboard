## Summary

Keep open. The PR targets a real Feishu NO_REPLY problem, but the current diff is broader than safe because it preserves leading bot self-mentions in direct command bodies and can prevent slash commands from being recognized; the narrower replacement path in #72661 is the better candidate.

## What This Changes

The PR changes Feishu mention normalization to preserve the receiving bot's own `<at>` tag, updates related Feishu tests, and rewrites the parse comment to describe that behavior.

## Best Possible Solution

Land a narrower Feishu plugin fix that strips only leading self-mention prefixes while preserving inline self-mentions, with command regression tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with this PR, direct `@Bot /model` normalizes to a leading `<at>` tag, then direct command parsing keeps that body unstripped and no longer sees a leading slash.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than needed; preserving inline self-mentions while keeping leading addressing prefixes slash-clean is the narrower maintainable fix.
