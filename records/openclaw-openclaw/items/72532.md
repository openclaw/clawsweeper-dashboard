## Summary

Keep open, but do not merge as-is: the patch fixes the multi-bot visibility problem too broadly and regresses direct Feishu slash-command dispatch by preserving the leading self-mention in p2p command bodies.

## What This Changes

The PR changes Feishu mention normalization to preserve the receiving bot's own `<at>` tag in all contexts and updates Feishu tests/comments to expect that behavior.

## Best Possible Solution

Land the narrower Feishu plugin behavior from #72661 or equivalent: keep leading addressing self-mentions slash-clean while preserving inline self-mentions for multi-bot routing, with regression coverage for both cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: with this PR, direct `@Bot /model` normalizes to a leading `<at>` tag, then the direct command path keeps that raw command body and core command normalization no longer sees a leading slash.

## Solution Assessment

No. Making `botStripId` a global no-op is broader than needed; the maintainable fix is to preserve semantic inline self-mentions without changing leading addressing-prefix behavior used by command dispatch.
