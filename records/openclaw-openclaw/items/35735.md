## Summary

Keep open: current main still lacks the requested `sessions rm` and `sessions clear` commands, but the PR has blocking source-reviewed correctness issues around session-store persistence and needs the previously raised cleanup fixes before merge.

## What This Changes

The PR adds `openclaw sessions rm <key>` and `openclaw sessions clear` with CLI wiring, deletion logic, transcript cleanup, dry-run/JSON support, and focused tests.

## Best Possible Solution

Land a focused implementation of explicit session deletion that shares the current session-store writer contract, deletes only the requested predicate matches, reports file cleanup accurately, and documents the new CLI surface.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure enforced session maintenance with unrelated stale, cap-eligible, or disk-budget-eligible entries, then run a targeted non-dry-run delete; the PR's `updateSessionStore` save can prune extra entries while dry-run reports only predicate matches. The transcript and JSON findings are also directly reproducible from an `fs.promises.rm` rejection and `JSON.stringify` omitting `undefined`.

## Solution Assessment

No. The requested CLI direction is reasonable, but this patch should first use the current controlled writer path for configured stores, skip opportunistic maintenance for targeted predicate deletes, and update reporting/docs/changelog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Skip maintenance during targeted deletes:** `src/commands/sessions-delete.ts:144-155`
  - body: `deleteSessionsByPredicate` saves every non-dry-run `rm` or filtered `clear` through `updateSessionStore` with no options. That save runs normal session maintenance, so a targeted delete can also prune, cap, or disk-budget-evict unrelated sessions while reporting only the predicate-matched keys.
  - confidence: 0.94
- **[P2] Route configured deletes through the gateway writer:** `src/commands/sessions-delete.ts:144-155`
  - body: Current main sends non-dry-run cleanup for configured agent stores through the Gateway so it shares the runtime session-store writer. This command writes those stores directly from the CLI process, which can race with a running gateway; keep direct writes for explicit `--store` repair paths only.
  - confidence: 0.86
- **[P3] Report transcript removals only after successful rm:** `src/commands/sessions-delete.ts:97-98`
  - body: `removeTranscriptFiles` catches every `fs.promises.rm` failure and then still records the path as removed. Permission or I/O failures will make the summary claim a transcript was deleted even though it remains on disk.
  - confidence: 0.9
- **[P3] Emit null for optional JSON fields:** `src/commands/sessions-delete.ts:212-213`
  - body: `clear --json` leaves `key` as `undefined`, so the field disappears from the JSON payload while `rm --json` emits `olderThanMinutes: null`. New machine-readable output should keep the optional fields stable.
  - confidence: 0.78
- **[P3] Document the new session delete commands:** `src/cli/program/register.status-health-sessions.ts:160`
  - body: This adds user-facing CLI commands, but the PR does not update `docs/cli/sessions.md` or `CHANGELOG.md`. OpenClaw policy expects behavior/API changes and user-facing features to be documented before merge.
  - confidence: 0.82
