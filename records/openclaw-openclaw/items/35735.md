## Summary

Keep open: the feature direction is still useful and current main still lacks `sessions rm`/`sessions clear`, but the PR is not mergeable because the destructive delete path bypasses the existing Gateway deletion contract, can run unrelated maintenance, and lacks after-fix real behavior proof.

## What This Changes

The PR adds `openclaw sessions rm` and `openclaw sessions clear` CLI commands with store/agent scoping, dry-run and JSON output, orphan transcript cleanup, command registration, and tests.

## Best Possible Solution

Build the CLI on top of Gateway `sessions.delete` for configured stores, keep direct mutation only for explicit offline `--store` repair with maintenance disabled, define approved bulk-clear behavior, and document the new commands before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows non-dry-run deletes call `updateSessionStore` directly, while current main already has `sessions.delete` as the lifecycle-aware deletion contract.

## Solution Assessment

No. The feature is useful, but the proposed implementation should be reshaped around the current Gateway deletion contract and a maintainer-approved bulk-clear policy before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route configured deletes through the Gateway:** `src/commands/sessions-delete.ts:144-155`
  - body: The non-dry-run path mutates configured stores directly with `updateSessionStore`, bypassing `sessions.delete` protections such as main-session rejection, runtime cleanup, transcript archiving, lifecycle hooks, and `sessions.changed`. Route configured deletes through the Gateway contract and reserve direct writes for explicit offline repair.
  - confidence: 0.93
- **[P2] Skip maintenance for explicit offline delete writes:** `src/commands/sessions-delete.ts:144-155`
  - body: For the remaining explicit `--store` path, this `updateSessionStore` call passes no options, so normal session maintenance can prune stale, capped, or disk-budget rows unrelated to the requested `rm` or filtered `clear`. Pass the appropriate writer option so the command only removes predicate matches.
  - confidence: 0.91
- **[P3] Only report transcript removals after successful deletion:** `src/commands/sessions-delete.ts:97-98`
  - body: `removeTranscriptFiles` catches every `fs.promises.rm` failure and then still appends the path, so permission or I/O failures are reported as successful transcript removals.
  - confidence: 0.9
- **[P3] Emit null for absent JSON fields:** `src/commands/sessions-delete.ts:212-213`
  - body: `clear --json` leaves `key` as `undefined`, so `JSON.stringify` omits it while `rm --json` emits `olderThanMinutes: null`. Use explicit `null` for absent optional fields to keep the new machine-readable schema stable.
  - confidence: 0.78
