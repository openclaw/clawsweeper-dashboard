## Summary

Keep open. The feature is still useful and not implemented on current main, but this branch needs design and correctness changes before merge because configured deletes bypass the existing Gateway deletion contract and external real behavior proof is still missing.

## What This Changes

The PR adds `openclaw sessions rm` and `openclaw sessions clear` with scoped deletion, dry-run/JSON output, command registration tests, and orphan transcript cleanup.

## Best Possible Solution

Implement the CLI on top of Gateway `sessions.delete` for configured stores, keep direct mutation only for explicit `--store` offline repair with maintenance controlled, and document the approved bulk-clear semantics.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows non-dry-run deletion calls `updateSessionStore` directly, while current main already defines `sessions.delete` as the lifecycle-safe deletion contract.

## Solution Assessment

No. The feature direction is useful, but the proposed implementation should be reshaped around the Gateway deletion contract and explicit offline repair semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route configured deletes through the Gateway:** `src/commands/sessions-delete.ts:144-155`
  - body: The non-dry-run path mutates configured stores with `updateSessionStore`, bypassing `sessions.delete` protections such as main-session rejection, active/runtime cleanup, transcript archiving, lifecycle hooks, and `sessions.changed`. Route configured deletes through the Gateway contract and reserve direct writes for explicit `--store` repair.
  - confidence: 0.92
- **[P2] Skip maintenance for offline delete writes:** `src/commands/sessions-delete.ts:144-155`
  - body: For any remaining direct `--store` path, this `updateSessionStore` call runs normal session maintenance because no options are passed. A targeted `rm` or filtered `clear` can prune unrelated stale, capped, or disk-budget sessions while reporting only predicate matches.
  - confidence: 0.91
- **[P3] Record transcript removal only after success:** `src/commands/sessions-delete.ts:97-98`
  - body: `removeTranscriptFiles` catches every `fs.promises.rm` failure and still appends the path to `removedPaths`, so permission or I/O failures will be reported as successful transcript removals.
  - confidence: 0.9
- **[P3] Emit null for absent JSON fields:** `src/commands/sessions-delete.ts:212-213`
  - body: `clear --json` leaves `key` as `undefined`, which `JSON.stringify` omits, while `rm --json` emits `olderThanMinutes: null`. Use explicit `null` for optional fields so the new machine-readable output has a stable shape.
  - confidence: 0.78
