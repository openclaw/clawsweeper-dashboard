## Summary

Keep open: this PR targets a real missing `sessions rm`/`sessions clear` CLI feature, but the current branch is not safe to merge because configured deletes bypass the Gateway session-delete lifecycle and the PR still lacks after-fix real behavior proof.

## What This Changes

Adds CLI wiring, implementation, and tests for `openclaw sessions rm` and `openclaw sessions clear` with scoped store/agent options, dry-run/JSON output, age filtering, and transcript cleanup.

## Best Possible Solution

Implement `rm`/`clear` as Gateway-backed CLI wrappers over `sessions.delete` for configured stores, keep explicit `--store` as offline repair with controlled maintenance, stabilize JSON output, and document the approved destructive semantics.

## Reproduction Assessment

Yes, by source inspection. Current main lacks public `sessions rm` and `sessions clear` commands, and the PR's non-dry-run path visibly mutates session stores directly instead of using the existing Gateway delete lifecycle.

## Solution Assessment

No. The feature direction is useful, but the current implementation should be reworked around Gateway deletion, explicit offline repair behavior, stable output, and real behavior proof before merge.
