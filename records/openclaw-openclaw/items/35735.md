## Summary

Keep open: current main still lacks the proposed `sessions rm` and `sessions clear` CLI surface, but the PR is not merge-ready because its destructive mutation path bypasses current session deletion safeguards and it lacks external real behavior proof.

## What This Changes

The PR adds `openclaw sessions rm <key>` and `openclaw sessions clear` with store/agent scoping, dry-run/JSON output, `--older-than` filtering, and orphan transcript removal.

## Best Possible Solution

Implement the delete CLI on top of `sessions.delete` for configured stores, keep direct file mutation limited to explicit offline `--store` repair, define safe bulk-clear semantics, and update docs plus focused tests before merge.

## Reproduction Assessment

Yes. Source inspection shows the PR directly mutates session stores while current main already provides Gateway deletion and cleanup contracts; no live destructive command was run during this read-only review.

## Solution Assessment

No. The feature direction is useful, but this implementation should be reshaped around the current Gateway session-deletion contract and an approved bulk-delete behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route configured deletes through the Gateway:** `src/commands/sessions-delete.ts:144-155`
  - body: The non-dry-run path mutates configured stores directly with `updateSessionStore`, bypassing current `sessions.delete` protections such as main-session rejection, pre-mutation cleanup, transcript archiving, lifecycle hooks, plugin cleanup, and `sessions.changed`. Use the Gateway contract for configured stores and reserve direct writes for explicit offline repair.
  - confidence: 0.94
- **[P2] Avoid pruning sessions outside the requested predicate:** `src/commands/sessions-delete.ts:144-155`
  - body: This `updateSessionStore` call runs normal writer maintenance, so `rm` or filtered `clear --older-than` can prune stale, capped, or disk-budget sessions that did not match the command predicate while reporting only predicate matches. The offline mutation path needs to skip unrelated maintenance or report all removals accurately.
  - confidence: 0.9
- **[P3] Report transcript removals only after success:** `src/commands/sessions-delete.ts:97-98`
  - body: `removeTranscriptFiles` catches every `fs.promises.rm` failure and still appends the path to `removedPaths`, so permission or I/O failures are reported as successful transcript removals. Record a path only when removal succeeds, or surface the failure.
  - confidence: 0.9
- **[P3] Stabilize the JSON output shape:** `src/commands/sessions-delete.ts:212-213`
  - body: `clear --json` leaves `key` as `undefined`, so `JSON.stringify` omits it while `rm --json` emits `olderThanMinutes: null`. Emit explicit `null` for absent optional fields so the new machine-readable output is stable.
  - confidence: 0.78
