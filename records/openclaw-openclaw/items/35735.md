## Summary

Keep open. Current main still lacks the requested `sessions rm` and `sessions clear` CLI surface, but this PR is not mergeable as-is because it bypasses the existing Gateway session-delete lifecycle, can report/delete inconsistently, and still lacks contributor real behavior proof.

## What This Changes

The PR adds `openclaw sessions rm <key>` and `openclaw sessions clear` with scoped options, JSON/dry-run output, transcript cleanup, command wiring, and tests.

## Best Possible Solution

Implement rm/clear as Gateway-backed CLI wrappers over `sessions.delete` for configured stores, keep explicit `--store` as offline repair with controlled maintenance, stabilize JSON output, and document the approved destructive semantics.

## Reproduction Assessment

Yes, by source inspection. Current main lacks the requested `sessions rm` and `sessions clear` commands, and the PR's current non-dry-run path visibly mutates stores directly instead of using the existing Gateway delete lifecycle.

## Solution Assessment

No. The feature direction is useful, but the current patch should be reworked around Gateway deletion, explicit offline repair behavior, stable output, and real behavior proof before merge.
