Full review comments:

- **[P1] Preserve session-key casing before deleting:** `src/commands/sessions-delete.ts:44-45`
  - body: `normalizeSessionKey` lowercases the entire exact key before comparison, but current store-key normalization preserves provider-owned opaque peer IDs. A destructive exact-key command should use the existing store/Gateway key normalization so it cannot match a different case-sensitive session key.
  - confidence: 0.9
- **[P1] Route configured deletes through Gateway deletion:** `src/commands/sessions-delete.ts:144-155`
  - body: Configured non-dry-run `rm` and `clear` write the store directly with `updateSessionStore`, bypassing `sessions.delete` behavior such as main-session rejection, runtime cleanup, transcript archival, lifecycle/plugin hooks, and `sessions.changed`. Route configured deletes through Gateway semantics and reserve direct writes for explicit offline repair.
  - confidence: 0.94
- **[P1] Reject malformed older-than values before deleting:** `src/commands/sessions-delete.ts:52-57`
  - body: `Number.parseInt` accepts partial strings, so values like `1d` or `1.5` become `1` minute and can make `sessions clear --older-than ...` delete far more than the user intended. Require a full positive integer or an explicit duration grammar before running a destructive clear.
  - confidence: 0.9
- **[P2] Disable maintenance for offline direct deletes:** `src/commands/sessions-delete.ts:144-155`
  - body: Even if direct writes remain for explicit `--store` repair, this `updateSessionStore` call runs normal session maintenance unless told otherwise. A targeted offline delete can prune unrelated sessions and make dry-run diverge from the real deletion summary.
  - confidence: 0.88
- **[P3] Record transcript removal only after success:** `src/commands/sessions-delete.ts:97-98`
  - body: `removeTranscriptFiles` swallows `fs.promises.rm` failures but still records the transcript path as removed. Permission or I/O failures can produce an inaccurate destructive-operation summary, so only add the path when removal actually succeeds.
  - confidence: 0.86
- **[P3] Emit null for absent JSON fields:** `src/commands/sessions-delete.ts:212-213`
  - body: `clear --json` leaves `key` as `undefined`, so `JSON.stringify` omits it while `rm --json` emits `olderThanMinutes: null`. Use explicit `null` for non-applicable optional fields so machine-readable output has a stable shape.
  - confidence: 0.78
