## Summary

Keep this PR open. Current main still lacks `openclaw sessions rm` and `openclaw sessions clear`, and the linked closed feature issue points here as the canonical implementation, but the proposed patch is not merge-ready because non-dry-run targeted deletion can trigger unrelated session maintenance pruning and misreport what was deleted.

## What This Changes

The PR adds `openclaw sessions rm <key>` and `openclaw sessions clear` CLI subcommands, a new sessions deletion command module with store/agent/all-agents/dry-run/JSON options and transcript cleanup, plus registration and command tests.

## Best Possible Solution

Land a focused CLI implementation that adds `sessions rm` and `sessions clear` while separating targeted deletion from maintenance pruning, preserving dry-run parity, using the existing session path/archive/trajectory cleanup contracts or an explicit generic helper, returning stable JSON, documenting the new commands at `https://docs.openclaw.ai/cli/sessions`, and adding a changelog entry crediting the external contribution.

## Reproduction Assessment

Yes. The blocking behavior can be reproduced by setting `session.maintenance.mode` to `enforce`, preparing a store with one predicate match plus unrelated stale/cap/budget-eligible sessions, and comparing `sessions rm` or filtered `sessions clear --older-than` dry-run output against the non-dry-run save path through `updateSessionStore`.

## Solution Assessment

No, not yet. The PR is aimed at the right CLI gap, but the current implementation is not the narrowest maintainable solution because targeted deletion is coupled to opportunistic maintenance and transcript reporting can claim success after failed file removal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Skip maintenance during targeted session deletion:** `src/commands/sessions-delete.ts:150`
  - body: `deleteSessionsByPredicate` calls `updateSessionStore` without options, and that helper saves through maintenance by default. In enforce mode, a targeted `rm` or filtered `clear` can also prune stale entries, cap count, or enforce disk budget for unrelated sessions while only reporting predicate matches, so dry-run and real execution can diverge and lose extra sessions.
  - confidence: 0.93
- **[P3] Report transcript removals only after successful rm:** `src/commands/sessions-delete.ts:98`
  - body: The transcript cleanup swallows every `fs.promises.rm` failure and then pushes the path unconditionally. Permission or I/O failures will be reported as removed even though the transcript remains on disk.
  - confidence: 0.9
