## Summary

Keep open: current main still has the reported asymmetric MEDIA handling, and the narrower duplicate-text bug is not owned by the broader open design issue.

## What This Changes

The issue asks OpenClaw to stop duplicating visible text when a final reply contains text plus a legacy MEDIA directive and is split through block/intermediate and channel.message delivery.

## Best Possible Solution

Fix the shared reply payload or block/final dedupe path so one logical text+MEDIA reply emits the visible text once and the attachment once, while leaving the broader channel.message contract redesign to https://github.com/openclaw/openclaw/issues/91867.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level reproduction: origin/main still keeps MEDIA directives text-only in block delivery, parses them in final payloads, and uses remaining text as the media caption in outbound delivery.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow bug: fix the shared reply payload or dedupe invariant before transport delivery, rather than adding a Bncr-specific workaround or closing it into the broader design discussion.
