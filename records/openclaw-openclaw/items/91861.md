## Summary

Keep open: current main still has the reported split where block replies leave MEDIA directives in text while final replies parse them into media, so a final text caption can survive and be delivered again around the media handoff.

## What This Changes

The issue reports duplicate visible text when a final assistant reply contains text plus a MEDIA directive and block/intermediate delivery splits from final channel.message media delivery.

## Best Possible Solution

Fix the shared reply/media planning path so one logical text plus MEDIA final reply sends the visible text once and the attachment once across block, final, and channel.message handoffs.

## Reproduction Assessment

Yes. The source path is high-confidence: block delivery leaves MEDIA directives in text, final delivery parses the same directive into media plus text, block sent-text matching does not prove the stripped final text was already sent, and outbound media uses remaining final text as caption.

## Solution Assessment

Yes. The best fix belongs in shared core reply/media planning with a focused regression for the MEDIA directive path, not in a Bncr-specific downstream workaround or a new configuration option.
