## Best Possible Solution

Land the focused Usage-only duplicate-header cleanup after regenerating all locale outputs from English and dropping the dead selector; keep the all-page topbar redesign separate.

## Reproduction Assessment

Yes, source-reproducible. Current main renders both the shared `.content-header` title/subtitle and the local `.usage-page-header`, and the PR screenshots show the before/after visual behavior.

## Solution Assessment

No, not as currently authored. Removing the local Usage header is the right narrow fix, but the branch needs generated i18n parity repaired and the unused selector removed before merge.
