## Best Possible Solution

Land the narrow Usage-only duplicate-header cleanup after the locale generator removes the key consistently and the dead CSS is dropped, while keeping the broader multi-page header redesign separate.

Yes. On current main, opening the Usage tab follows a source path where `app-render.ts` emits the shared title/subtitle and `renderUsage` emits a second local Usage title/subtitle; the PR screenshots show the same before/after behavior.

## Solution Assessment

No, not as currently authored. Removing the local Usage header is the right narrow fix, but the branch needs generated i18n parity repaired and the unused CSS selector removed before merge.
