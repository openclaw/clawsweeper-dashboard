## Summary

Keep this PR open: it fixes a real duplicate Usage header path, but the latest head still leaves the i18n key set inconsistent and one unused CSS selector behind, so it should not merge yet.

## What This Changes

The PR removes the local Usage page header and associated `usage.page.subtitle` locale entries so the Usage tab relies on the shared app header.

## Best Possible Solution

Land the focused Usage-only duplicate-header cleanup after regenerating all locale outputs from English and dropping the dead CSS selector, without pulling in the broader multi-page header redesign.

## Reproduction Assessment

Yes. Current main has a source-level path where `app-render.ts` emits the shared Usage title/subtitle and `renderUsage` emits a second local Usage title/subtitle; the inspected screenshots show the same before/after behavior.

## Solution Assessment

No, not as currently authored. Removing the local Usage header is the right narrow fix, but the locale outputs and translation metadata must be regenerated consistently and the unused selector should be removed before merge.
