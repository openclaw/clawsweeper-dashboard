## Summary

Keep open: the focused Usage header fix is useful and matches a real current-main duplicate-header path, but the PR head is not merge-ready because its locale deletion leaves shipped-locale key drift and it adds an unused CSS selector.

## What This Changes

The PR removes the Usage view's local title/subtitle block, deletes selected `usage.page.subtitle` locale and translation-memory entries, and adds a small Usage CSS selector.

## Best Possible Solution

Land the narrow Usage-only duplicate-header cleanup after regenerating all locale outputs from English and removing dead CSS, while leaving the broader multi-page header redesign separate.

## Reproduction Assessment

Yes. Current main has a source-level path where `app-render.ts` emits the shared Usage title/subtitle and `renderUsage` emits a second local Usage title/subtitle; the PR screenshots show the same before/after behavior.

## Solution Assessment

No, not as currently authored. Removing the local Usage header is the right narrow fix, but the branch must regenerate every locale output and translation-memory row consistently and drop the unused selector before merge.
