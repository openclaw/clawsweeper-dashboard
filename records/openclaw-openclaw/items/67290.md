## Best Possible Solution

Land the focused Usage-only duplicate-header cleanup after regenerating all locale outputs from English and removing the dead selector left by earlier review iterations.

Yes. Current main has a source-level path where `app-render.ts` emits the shared Usage title/subtitle and `renderUsage` emits a second local Usage title/subtitle; the inspected PR screenshots show the before/after UI behavior.

## Solution Assessment

No, not as currently authored. Removing the local Usage header is the right narrow fix, but the locale outputs and translation metadata must be regenerated consistently and the unused subtitle selector should be removed before merge.
