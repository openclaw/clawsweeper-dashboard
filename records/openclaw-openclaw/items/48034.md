## Summary

Keep open: the underlying rw sandbox skill-visibility bug is still source-reproducible on current main, but this PR is not mergeable because its sandbox fix is still unreachable in the common rw path and it imports a symbol from the wrong module.

## What This Changes

The branch changes sandbox context to always sync skills with explicit managed and bundled skill directories, and changes the Control UI Overview locale selector to prefer a persisted setting.

## Best Possible Solution

Land a rebased sandbox fix that materializes eligible skills inside an in-boundary workspace path for rw mode, preserves current eligibility/bootstrap contracts, keeps strict path escape checks, and adds rw regression coverage.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main routes non-shared `workspaceAccess: "rw"` sessions to the agent workspace and only runs sandbox skill sync in the sandbox-workspace branch guarded away from rw mode.

## Solution Assessment

No. Mirroring eligible skills into an in-boundary workspace path is the right fix shape, but this patch leaves the sync unreachable in normal rw mode and imports the bundled skill resolver from a module that does not export it.
