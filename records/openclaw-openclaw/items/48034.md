## Summary

Keep open: the rw sandbox skill-access bug is still source-reproducible on current main, but this PR remains incorrect because it imports a non-exported resolver and leaves the sync call inside the branch skipped by normal rw mode; no after-fix real behavior proof is posted.

## What This Changes

The branch removes the rw guard around sandbox skill sync, passes managed/bundled skill directories into `syncSkillsToWorkspace`, and changes the Overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased narrow sandbox fix that makes managed and bundled skill locations readable in rw sandbox sessions without loosening the sandbox boundary, adds rw regression coverage, preserves current eligibility/bootstrap contracts, and leaves the already-shipped locale fix alone.

## Reproduction Assessment

Yes, by source inspection: a non-shared `workspaceAccess: "rw"` sandbox selects the agent workspace and skips the only current-main branch that syncs skills into a sandbox-readable workspace. I did not run a live sandbox in this read-only review.

## Solution Assessment

No. Mirroring or remapping skill files into a sandbox-readable path is the right direction, but this PR still leaves the sync unreachable for normal rw mode and imports the bundled resolver from the wrong module.
