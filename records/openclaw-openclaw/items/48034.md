The branch removes the rw guard around sandbox skill syncing, passes managed and bundled skill directories, and changes Overview locale selection to prefer persisted settings.

## Best Possible Solution

Land a rebased replacement fix that prepares and syncs the sandbox workspace for rw mode, preserves current eligibility, bootstrap, and path-boundary contracts, adds rw regression coverage, and omits the already-shipped locale change.

## Reproduction Assessment

Yes, as a source-level current-main reproduction. Configure a non-shared sandbox with workspaceAccess rw: current code selects the agent workspace, skips the sandboxWorkspaceDir branch, and never mirrors managed or bundled skills into an in-boundary location.

## Solution Assessment

No. Mirroring skills into an in-boundary workspace path is the right fix shape, but this PR still does not run that sync in the common rw case and imports the bundled resolver from a module that does not export it.
