## Summary

Keep open: the underlying rw sandbox skill-access bug still appears valid, but this branch does not fix the common non-shared rw path and it introduces a bad import; it also lacks real after-fix proof.

## What This Changes

The PR removes the rw guard around sandbox skill syncing, passes explicit managed and bundled skill directories, and changes the Overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased sandbox fix that prepares and syncs the sandbox workspace for rw mode, preserves eligibility and bootstrap contracts, adds rw regression coverage, and splits or drops the already-covered UI locale change.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main routes non-shared `workspaceAccess: "rw"` sessions to the agent workspace and skips the sandbox workspace sync branch before managed or bundled skills can be mirrored.

## Solution Assessment

No. Mirroring eligible skills into an in-boundary workspace path is the right fix shape, but this PR keeps the sync unreachable in the common rw case and imports the bundled skill resolver from a module that does not export it.
