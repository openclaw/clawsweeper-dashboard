## Summary

Keep open: the linked rw-sandbox skill bug is still source-reproducible on current main and in v2026.5.12, but this PR is not safe to merge because it leaves the rw skill-sync path unreachable and imports the bundled-skill resolver from the wrong module. The Overview locale hunk has already landed separately on main.

## What This Changes

The PR removes the rw guard around sandbox skill sync, passes managed and bundled skill directory parameters, and changes the Overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased narrow sandbox fix that materializes managed and bundled skills under the actual rw sandbox read root, points prompt locations at those mirrored paths, preserves eligibility/bootstrap/path-boundary contracts, adds rw regression coverage, and drops the already-landed locale change.

## Reproduction Assessment

Yes. Source inspection on current main shows non-shared workspaceAccess: "rw" selects the agent workspace and skips the only branch that syncs skills into the sandbox workspace; I did not run a live sandbox in this read-only review.

## Solution Assessment

No. Mirroring or remapping skills into a sandbox-readable path is the right direction, but this PR leaves that path unreachable for normal rw mode, keeps prompt path handling incomplete, and imports the bundled resolver from the wrong module.
