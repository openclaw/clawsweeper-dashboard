## Summary

Keep this PR open, but it is not mergeable as written: the main sandbox bug still exists, the PR’s skill-sync block remains unreachable for the common rw workspace path, and the added import is from the wrong module.

## What This Changes

The PR attempts to sync managed and bundled skills for sandbox `workspaceAccess: "rw"` and also changes the overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased sandbox fix that prepares and syncs the sandbox workspace for rw mode without loosening path checks, preserves current eligibility/filtering contracts, adds rw regression coverage, and drops or splits the already-implemented unrelated UI locale change.

## Reproduction Assessment

Yes, by source inspection. With non-shared `workspaceAccess: "rw"`, current main chooses the agent workspace as `workspaceDir`, so the sandbox workspace skill-sync branch is skipped before managed or bundled skills can be mirrored.

## Solution Assessment

No. Mirroring eligible skills into an in-boundary sandbox path is the right fix shape, but this PR keeps the sync in an unreachable branch for the common rw case and imports `resolveBundledSkillsDir` from a module that does not export it.
