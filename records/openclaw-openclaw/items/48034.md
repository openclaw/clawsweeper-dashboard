## Summary

Keep this PR open, but do not merge the current branch. The underlying sandbox skill visibility bug still appears source-reproducible on current main, while this PR is stale, unmergeable, and has two blocking correctness issues already identified in review comments.

## What This Changes

The PR removes the `workspaceAccess !== "rw"` skill-sync guard, passes explicit managed and bundled skill directories in sandbox context, and changes the Control UI locale dropdown to prefer persisted settings.

## Best Possible Solution

Replace this stale branch with a focused sandbox fix that makes managed, bundled, plugin, and eligible remote skills readable inside `workspaceAccess: "rw"` without destructive same-workspace copying, and keep the already-shipped UI locale fix out of that replacement.

## Reproduction Assessment

Yes, at source level. With default agent scope and `workspaceAccess: "rw"`, current main uses the agent workspace as the sandbox root and never enters the existing skill-sync branch, while the fs resolver rejects skill paths outside mounted roots.

## Solution Assessment

No. This PR imports a non-exported symbol, leaves the new sync call under a branch that typical rw sandboxes do not reach, and includes an unrelated UI locale change that current main already handles more defensively.
