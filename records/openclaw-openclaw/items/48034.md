## Summary

Keep open: this remains the canonical PR for the rw sandbox skill-access regression, but the branch is still incorrect and missing required real-behavior proof. Current main still has the rw skill-sync gap, while the PR head leaves the sync unreachable for normal rw workspaces and imports the bundled-skill resolver from the wrong module.

## What This Changes

The PR changes sandbox workspace setup to call `syncSkillsToWorkspace` with managed and bundled skill directories, and also changes the Overview language selector to prefer a persisted locale.

## Best Possible Solution

Land a rebased, narrow sandbox fix that exposes managed and bundled skills through prompt-visible, sandbox-readable paths for non-shared rw workspaces, preserves strict path checks and skill eligibility, adds rw regression coverage, and drops the obsolete locale hunk.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main selects the agent workspace for non-shared rw sandboxes and skips the only branch that syncs skills into a sandbox-readable workspace. The PR head keeps its new sync call under that same skipped branch.

## Solution Assessment

No. Mirroring or remapping skill files into a sandbox-readable location is the right direction, but this patch leaves normal rw mode unfixed, has a broken import, and does not preserve current skill eligibility contracts.
