## Summary

Keep open: current main still has the rw-mode sandbox skill-sync gap, and this PR is not safe to merge because it keeps the sync in the branch that normal rw mode does not enter and adds a broken import.

## What This Changes

The branch removes the rw guard around sandbox skill syncing, passes managed/bundled skill directory options, and changes Overview locale selection to prefer persisted settings.

## Best Possible Solution

Land a rebased sandbox fix that prepares and syncs the sandbox workspace for rw mode, preserves eligibility/filtering and path-boundary contracts, adds rw regression coverage, and leaves the already implemented UI locale fix out or split.

## Reproduction Assessment

Yes. Source inspection on current main shows non-shared `workspaceAccess: "rw"` uses the agent workspace, so the sandbox workspace sync branch is skipped before managed or bundled skills can be mirrored.

## Solution Assessment

No. Mirroring eligible skills into an in-boundary sandbox path is the right fix shape, but this PR keeps sync unreachable for the common rw case and imports `resolveBundledSkillsDir` from a module that does not export it.
