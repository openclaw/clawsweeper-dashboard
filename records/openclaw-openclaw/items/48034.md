## Summary

Keep open. The rw skill-access bug is still source-reproducible on current main, but this PR is not mergeable because its sandbox import is build-breaking, the skill sync still does not run for normal rw sandboxes, and the Overview locale hunk has already landed separately.

## What This Changes

The PR removes the non-rw guard around sandbox skill sync, adds managed and bundled skill directories to that sync call, and changes Overview locale selection to prefer persisted settings.

## Best Possible Solution

A focused sandbox patch should make eligible non-workspace skills readable for agent-scoped rw sandboxes while preserving agent allowlists, remote-skill eligibility, and path confinement; the Overview locale behavior should remain with the separate shipped UI fix.

## Reproduction Assessment

Yes. A high-confidence source path exists: with `scope: "agent"` and `workspaceAccess: "rw"`, sandbox context skips skill sync while embedded runs and skill prompt generation can still advertise managed or bundled skill paths outside the readable workspace.

## Solution Assessment

No. This PR is not the best fix because it imports from the wrong module and keeps the sync inside a branch skipped by normal rw sandboxes; the safer direction is a narrow rw skill-mirror or path-rewrite fix with regression coverage.
