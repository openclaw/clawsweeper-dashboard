## Summary

Keep this PR open, but it is not mergeable as-is: current main still has the rw sandbox skill-visibility gap, while the PR keeps the sync unreachable in the common rw path and adds a broken import.

## What This Changes

The PR changes sandbox setup to always call `syncSkillsToWorkspace` with managed and bundled skill paths, and also changes the Overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased narrow sandbox fix that materializes or remaps managed and bundled skills into a sandbox-readable path for non-shared rw workspaces, preserves current eligibility/bootstrap contracts, adds rw regression coverage, and drops the already-handled locale change.

## Reproduction Assessment

Yes by source inspection, not a live run: a non-shared `workspaceAccess: "rw"` sandbox selects the agent workspace and skips the only branch that currently syncs skills into the sandbox workspace.

## Solution Assessment

No. Mirroring or remapping skill files into a sandbox-readable location is the right direction, but this PR still leaves that sync unreachable for normal rw mode and imports the bundled resolver from the wrong module.
