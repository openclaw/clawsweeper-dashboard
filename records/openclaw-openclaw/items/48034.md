## Summary

Keep open: the rw sandbox skill-access bug is still source-reproducible on current main, but this PR is not mergeable as written because it leaves the sync in a branch skipped by normal rw mode and adds a broken import. The closed bug report now points here, so this PR should stay as the canonical thread until a corrected fix or replacement lands.

## What This Changes

The PR removes the rw guard around sandbox skill syncing, passes explicit managed and bundled skill directories, and makes the Overview locale selector prefer persisted settings.

## Best Possible Solution

Land a rebased sandbox fix or replacement PR that prepares and syncs the sandbox workspace for rw mode, imports the bundled resolver from the correct module or relies on sync defaults, preserves bootstrap and eligibility contracts, adds rw regression coverage, and leaves the already-fixed locale change out.

## Reproduction Assessment

Yes, as a source-level current-main reproduction. Configure a non-shared sandbox with workspaceAccess rw: current code selects the agent workspace, skips the sandboxWorkspaceDir branch, and never mirrors managed or bundled skills into a readable in-boundary path.

## Solution Assessment

No. Mirroring skills into an in-boundary workspace path is the right fix shape, but this PR still does not run that sync in the common rw case and imports the bundled skill resolver from a module that does not export it.
