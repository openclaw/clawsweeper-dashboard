## Summary

Keep open: the underlying rw sandbox skill regression is still present on current main and the latest release, and the linked bug was closed to this PR as the canonical tracker. The branch is not mergeable as a fix because it still misses the common rw path, imports a non-exported symbol, and lacks contributor real-behavior proof.

## What This Changes

The branch removes the rw guard around sandbox skill sync, passes managed/bundled skill directories to that sync, and changes Overview locale selection to prefer persisted settings.

## Best Possible Solution

Land a rebased fix that materializes or remaps managed and bundled skill locations into a sandbox-readable path for rw workspaces, preserves current eligibility/bootstrap behavior, adds rw regression coverage, and drops the already-superseded locale change.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows a non-shared `workspaceAccess: "rw"` sandbox selects the agent workspace, skips the sandbox-workspace sync branch, and has no rw regression test covering managed/bundled skill materialization.

## Solution Assessment

No. Mirroring or remapping skill files into a sandbox-readable location is the right fix direction, but this PR still does not run that sync in the common rw case and imports the bundled resolver from a module that does not export it.
