## Summary

Keep open. The rw-mode sandbox skill problem still appears valid on current main, but this PR is not correct as written because the sandbox import would fail and the sync still stays inside the branch skipped by normal rw sandboxes.

## What This Changes

The PR removes the non-rw guard around sandbox skill sync, passes managed and bundled skill directories into that sync call, and changes the Overview language selector to prefer persisted settings.

## Best Possible Solution

A focused sandbox change should make eligible managed, bundled, and plugin skills readable for agent-scoped rw sandboxes without weakening path boundaries, and it should add a targeted rw regression test. The already-implemented Overview locale behavior should stay separate from that sandbox patch.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure `agents.defaults.sandbox` with `scope: "agent"` and `workspaceAccess: "rw"`, then trace `resolveSandboxContext()` and `runEmbeddedAttempt()` to see non-workspace skill paths remain outside the readable sandbox workspace.

## Solution Assessment

No. This PR is not the best way to solve the issue because it imports from the wrong module and leaves the sync block inside the branch skipped by normal rw sandboxes; the safer fix is an explicit rw skill-mirror path with regression coverage.
