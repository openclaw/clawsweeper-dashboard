## Summary

Keep open: current main still has the rw sandbox skill-sync gap, and this PR does not fix it because the sync remains unreachable in the common non-shared rw path and the new bundled-skill import is invalid. The unrelated Overview locale fix is already present on main and in the latest release.

## What This Changes

The PR removes the rw guard around sandbox skill syncing, passes managed/bundled skill directories, and changes the Overview locale selector to prefer the persisted setting.

## Best Possible Solution

Land a rebased replacement fix that prepares and syncs the sandbox workspace for rw mode, preserves strict path checks and eligibility contracts, adds rw regression coverage, and omits the already-shipped locale change.

## Reproduction Assessment

Yes, as a source-level current-main reproduction. Configure a non-shared sandbox with workspaceAccess rw: current code selects the agent workspace, skips the sandboxWorkspaceDir branch, and never mirrors managed or bundled skills into an in-boundary path.

## Solution Assessment

No. Mirroring skills into an in-boundary workspace path is the right fix shape, but this PR still does not run that sync in the common rw case and imports the bundled resolver from a module that does not export it.
