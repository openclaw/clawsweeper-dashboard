## Summary

Keep this PR open, but do not merge it as-is: the sandbox fix is still unreachable for normal `workspaceAccess: "rw"` workspaces, and the branch imports a non-exported symbol that would break module loading.

## What This Changes

This PR changes sandbox skill syncing for rw-mode workspaces and also changes the Control UI overview locale dropdown to prefer the persisted setting.

## Best Possible Solution

Fix the rw sandbox skill-materialization contract at the actual workspace boundary, keep strict path escape checks, rewrite or generate prompt-visible skill locations that are readable inside the sandbox, and add focused rw-mode regression coverage.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: configure sandbox `workspaceAccess: "rw"` with a non-shared scope and use a managed or bundled skill whose prompt location is outside the workspace root. I did not run a live sandbox repro in this read-only review.

## Solution Assessment

No. The proposed sandbox change is not the best fix because it leaves the sync call under the branch that normal rw-mode workspaces skip, and it introduces a broken import; the safer direction is to materialize or rewrite skill locations where the rw sandbox can actually read them.
