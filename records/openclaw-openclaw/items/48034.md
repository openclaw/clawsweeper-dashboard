## Summary

Keep open: the branch is not mergeable because the sandbox import is invalid and the skill sync remains unreachable for ordinary `workspaceAccess: "rw"` sandboxes. The unrelated Control UI locale-selector change is already represented on current `main` with a stricter supported-locale guard.

## What This Changes

The PR removes the rw guard around sandbox skill syncing, passes managed and bundled skill directories into `syncSkillsToWorkspace`, and changes the Control UI overview locale selector to prefer persisted settings.

## Best Possible Solution

Fix `ensureSandboxWorkspaceLayout` so rw sandboxes materialize eligible managed and bundled skills into an in-workspace location before prompt use, import the bundled-skill resolver from the skills module, keep the strict path guard, and add rw-mode regression coverage.

## Reproduction Assessment

Yes, by source inspection. Configure a non-shared sandbox with `workspaceAccess: "rw"`: current main selects the agent workspace, skips the sandbox skill-sync branch, and the existing workspace-only read guard rejects advertised skill paths outside that workspace.

## Solution Assessment

No. The intended direction is right, but this PR imports from the wrong module and leaves the new sync call inside a branch that rw sessions usually do not enter; the locale-selector part is already covered on current main.
