## Summary

Keep open: the underlying rw-mode skill visibility bug is still valid on current main, but this PR is not mergeable because it imports a nonexistent export, leaves skill sync in a branch skipped by normal rw mode, and has no contributor after-fix real behavior proof.

## What This Changes

The PR removes the rw guard around sandbox skill syncing, passes managed/bundled skill directory options, and changes the Overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased sandbox fix that prepares and syncs the sandbox workspace for rw mode without loosening path checks, preserves current eligibility/filtering contracts, adds rw regression coverage, and drops or splits the already-covered UI locale change.

## Reproduction Assessment

Yes, by source inspection rather than live execution. In non-shared `workspaceAccess: "rw"`, current main chooses the agent workspace as `workspaceDir`, so the sandbox workspace skill-sync branch is skipped before managed or bundled skills can be mirrored.

## Solution Assessment

No. Mirroring eligible skills into an in-boundary sandbox path is the right fix shape, but this PR keeps the sync in an unreachable branch for the common rw case and imports `resolveBundledSkillsDir` from a module that does not export it.
