## Summary

Keep open: the rw sandbox skills regression is still source-reproducible on current main, and this PR remains the canonical tracker, but the branch is not mergeable as-is because it leaves the sync path unreachable in the common rw case and imports a non-exported symbol.

## What This Changes

The PR removes the rw guard around sandbox skill sync, passes managed/bundled skill directories to that sync, and changes the Overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased fix that materializes or remaps managed and bundled skill locations into a sandbox-readable path for rw workspaces, preserves current eligibility/bootstrap behavior, adds rw regression coverage, and drops the unrelated locale change.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-shared `workspaceAccess: "rw"` sandbox selects the agent workspace, skips the sandbox-workspace sync branch, and has no rw regression test covering managed/bundled skill materialization.

## Solution Assessment

No. Mirroring or remapping skill files into a sandbox-readable location is the right direction, but this PR still does not run that sync in the common rw case and imports the bundled resolver from a module that does not export it.
