## Summary

Keep open: the underlying rw-mode skill visibility bug is still source-reproducible on current main, but this PR is not safe to merge because the sandbox sync remains unreachable in the common rw path, the new import is invalid, and the external PR still lacks real after-fix behavior proof.

## What This Changes

The PR removes the rw guard around sandbox skill syncing, passes explicit managed and bundled skill directories, and changes the Overview locale selector to prefer persisted settings.

## Best Possible Solution

Land a rebased sandbox fix that syncs eligible skills for non-shared rw workspaces, preserves current eligibility and bootstrap contracts, adds rw regression coverage, and drops the already-covered locale change from this branch.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main routes non-shared `workspaceAccess: "rw"` sessions to the agent workspace and skips the sandbox workspace sync branch before managed or bundled skills can be mirrored.

## Solution Assessment

No. Mirroring skills into an in-boundary workspace path is the right fix shape, but this PR keeps the sync unreachable in the common rw case and imports the bundled resolver from a module that does not export it.
