## Summary

Keep open for maintainer review: the patch appears correct and has sufficient generated-service proof, but it changes daemon startup defaults and adds an override env surface, so compatibility and low-memory-host behavior need maintainer acceptance before merge.

## What This Changes

The PR adds a generated daemon NODE_OPTIONS heap default with an OPENCLAW_GATEWAY_HEAP_MB override for gateway/node service environments while dropping ambient host NODE_OPTIONS.

## Best Possible Solution

Merge a daemon-level heap setting only after maintainers accept the generated-service default and upgrade behavior across systemd, launchd, and Windows Scheduled Tasks.

## Reproduction Assessment

Partially. The linked issue has concrete live crash-loop evidence, and current source confirms generated services do not set heap NODE_OPTIONS, but this read-only review did not reproduce the full live OOM on current main.

## Solution Assessment

Yes for the code boundary, with a maintainer caveat. The daemon-generated service environment is the right layer because host NODE_OPTIONS is intentionally blocked, but the 8192 MB default and new override env surface still need compatibility acceptance.
