## Summary

Keep this PR open: the heap-only service NODE_OPTIONS direction is useful for the linked crash-loop, but the advertised heap override is not durable across forced reinstall/update and the startup-default change still needs compatibility/availability handling before merge.

## What This Changes

The PR adds generated heap-only NODE_OPTIONS for managed gateway and node service environments, an OPENCLAW_GATEWAY_HEAP_MB override, and daemon service-env tests that ambient NODE_OPTIONS is dropped.

## Best Possible Solution

Keep heap-only NODE_OPTIONS generation, but either make the heap-size source durable and covered by fresh-install plus forced-reinstall/update tests, or remove the override and land a fixed default with explicit maintainer approval.
