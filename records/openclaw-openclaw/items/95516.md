## Summary

Keep open: current main and v2026.6.9 still only support opt-in, correction-based Skill Workshop proposals and explicitly skip failed auto-capture turns, while this request asks for new failure-driven repair, usage metrics, archive/retirement policy, and public config surface that need maintainer product/API design.

## What This Changes

Request to add skill lifecycle management with failure-triggered Skill Workshop repair proposals and usage-based archive/retirement of inactive skills.

## Best Possible Solution

Split failure-driven optimization and usage-based retirement into maintainer design, then implement only after the core-versus-plugin API boundary, SQLite-backed lifecycle state, conservative approval semantics, and config/docs migration path are agreed.
