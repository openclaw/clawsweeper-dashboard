## Summary

Keep open: current main and the latest release still only provide opt-in Skill Workshop proposals from durable conversation signals after successful turns, while this request asks for new failure-driven repair proposals plus usage-based archive/retirement policy that needs maintainer product and API design.

## What This Changes

Add skill lifecycle management that proposes skill repairs from failures or underperformance and tracks usage to archive or retire stale skills.

## Best Possible Solution

Split failure-driven optimization and retirement into maintainer design, choose the core versus plugin API boundary, and persist any accepted lifecycle state in the canonical state store with conservative user approval for archive/delete behavior.
