## Summary

Keep open: current main and v2026.6.9 still only have opt-in Skill Workshop proposals from durable conversation signals, while this request asks for new failure-driven skill repair plus usage-based archive/retirement policy that needs maintainer product and API design.

## What This Changes

The issue requests a skill lifecycle system that creates Skill Workshop repair proposals from failure or performance signals and archives or retires inactive skills using tracked usage metrics.

## Best Possible Solution

Split failure-driven optimization and retirement into maintainer design, choose the core versus plugin API boundary, and persist any accepted lifecycle state in the canonical state store with conservative user approval for archive/delete behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence current main lacks failure-driven skill optimization and usage-based retirement state/config.

## Solution Assessment

Unclear as proposed. The direction is plausible, but the best solution needs maintainer decisions on core versus plugin ownership, persisted metrics, and conservative user-approved retirement semantics before implementation.
