## Summary

Keep open: current main and v2026.6.10 still provide opt-in Skill Workshop proposals from durable conversation signals after successful turns plus bounded skill-used telemetry, but they do not implement failure-driven skill repair proposals or usage-based archive/retirement policy. The remaining work needs maintainer product and API-boundary decisions before any implementation path is safe.

## What This Changes

The issue requests automatic Skill Workshop repair proposals from skill failures or underperformance plus usage-based skill archiving, retirement, and lifecycle reporting.

## Best Possible Solution

Split the design into failure-driven repair proposals and skill lifecycle/retirement state, decide the core versus plugin API boundary, and persist any accepted lifecycle state in canonical storage with explicit approval for archive/delete behavior.
