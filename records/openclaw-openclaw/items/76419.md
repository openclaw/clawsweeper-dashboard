## Summary

Keep open. Current main still builds Telegram reply metadata upstream, renders `ReplyToBody` as a replied-message context block, then the embedded-runner prompt split moves that block into `runtimeContext` while submitting only the clean transcript prompt; no newer main commit touched the relevant files after this issue was opened.

## What This Changes

Telegram reply/quote context should remain visible to the current model turn after the runtime-context prompt split.

## Best Possible Solution

Preserve explicit inbound reply/quote metadata in the current-turn model input while keeping internal runtime context hidden from provider user turns and visible transcripts.

## Reproduction Assessment

Yes for source-level reproduction: a Telegram turn with `ReplyToBody` builds a replied-message context block, then `resolveRuntimeContextPromptParts` strips that block into `runtimeContext` and submits only the clean transcript prompt. I did not perform the live provider reproduction in this read-only review.

## Solution Assessment

Yes, the best fix direction is to thread explicit reply/quote context separately from hidden internal runtime context. Avoid restoring arbitrary runtime context to user turns.
