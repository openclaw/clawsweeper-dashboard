## Summary

Keep open. Current main preserves Telegram `ReplyToBody` through Telegram and prompt assembly, and it adds a prompt-local system-context mitigation after the runtime-context split, but the explicit reply block is still removed from the submitted user prompt and the LLM-bound hidden custom message is stripped; the live report says that mitigation is not enough for Telegram reply questions.

## What This Changes

The issue asks OpenClaw to keep Telegram reply/quote context visible to the assistant on the same turn after runtime-context prompt splitting.

## Best Possible Solution

Thread explicit reply/quote context as dedicated current-turn untrusted model input, with focused regression coverage, while keeping unrelated runtime context hidden from user turns.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection does reproduce the prompt-shape mismatch: Telegram reply context is rendered, split out of the user prompt as runtime context, and only reintroduced through temporary system context.

## Solution Assessment

Unclear. The current system-context mitigation is safer for arbitrary runtime context, but the reported Telegram behavior likely needs an explicit reply-context path rather than relying on generic prompt splitting.
