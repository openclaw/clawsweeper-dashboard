## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:392`
  - body: The override loop compares raw provider strings, but current auth resolution accepts normalized aliases such as `aws-bedrock` for `amazon-bedrock`. A plugin registered for the canonical provider can be skipped and OpenClaw may fall back to normal credentials instead of the intended brokered auth; normalize both requested and registered provider ids, including the availability check.
  - confidence: 0.88
- **[P2] Pass known model IDs into override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback receives `params.modelId ?? ""`, but media execution paths still call `resolveApiKeyForProvider` without passing the selected model. Model-scoped credential brokers can then see an empty model id and return null or the wrong credential; thread the known model through those resolver calls or make the context explicitly optional.
  - confidence: 0.82
- **[P2] Keep image auto selection from pinning tentative providers:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` uses the override-aware candidate list but returns only `keyEntries[0]`. If the first provider is conditionally override-backed and fails at request time, callers of this single-model helper never consider later key-backed providers; preserve ordered fallback behavior or exclude tentative override-only providers here.
  - confidence: 0.76

## Security Review

  - body: Raw provider-id matching means an alias-form request can skip a registered canonical override and use default credentials instead, undermining plugins intended to broker all auth for a provider family.
