## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:392`
  - body: Override matching uses raw string equality, but auth resolution otherwise accepts aliases such as `aws-bedrock` for `amazon-bedrock`. Normalize both registered provider IDs and request provider IDs before matching, including the opt-in availability check, so canonical registrations are not bypassed by alias-form calls.
  - confidence: 0.86
- **[P2] Pass real model context to override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback gets an empty `modelId` whenever a caller has not been updated to pass the new field. Model-scoped brokers then cannot distinguish the target model and may return null or incorrect credentials, so known model IDs should be threaded through runtime auth call sites instead of defaulting them away.
  - confidence: 0.82
- **[P2] Preserve fallback when resolving auto image auth:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` collapses the ordered key-provider candidates to the first entry. If that first provider is only conditionally override-backed and fails at request time, direct image-model callers never get to use later key-backed providers that the new fallback list was meant to preserve.
  - confidence: 0.8
