## Summary

Keep this PR open: it is the canonical thread for a real additive plugin/auth feature that is not on current main, but the patch still has source-visible auth-routing blockers and lacks contributor real behavior proof. The linked feature issue was closed in favor of this PR, so this should continue through normal maintainer/security API review rather than cleanup close.

## What This Changes

The PR adds a plugin SDK registration API for provider runtime auth overrides, stores active override registrations, and wires override auth into model auth plus media-understanding selection with tests.

## Best Possible Solution

Land a revised additive SDK/auth seam that normalizes provider ids, carries known model context through resolver calls, preserves fallback behavior, documents the public API, and includes maintainer/security approval plus real plugin-run proof.

## Reproduction Assessment

Not applicable for the feature request itself. Yes for the review blockers: PR-head source shows raw provider matching, empty model fallback, and first-entry image selection without needing a live run.

## Solution Assessment

No as-is. A dedicated registration API is a plausible maintainable seam, but this patch needs normalized matching, model-aware callback context, safer fallback behavior, docs, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:392`
  - body: Override matching uses raw string equality, but current auth resolution accepts aliases such as `aws-bedrock` for `amazon-bedrock`. A plugin registered for the canonical provider can be skipped and OpenClaw may fall back to normal credentials instead of the intended brokered auth; normalize both requested and registered provider ids, including the availability check.
  - confidence: 0.9
- **[P2] Pass known model IDs to override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback receives an empty string when callers omit `modelId`, and media execution still calls `resolveApiKeyForProvider` without passing the selected entry model. Model-scoped brokers cannot reliably choose credentials and may return null or the wrong key; thread known selected model ids through resolver calls or make the context explicitly unknown.
  - confidence: 0.84
- **[P2] Preserve fallback for tentative image auth candidates:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` uses the override-aware candidate list but returns only the first entry. If that provider is conditionally override-backed and fails at request time, sticker/image callers never try later key-backed providers; keep ordered fallback or avoid tentative override-only availability in this single-result resolver.
  - confidence: 0.78
