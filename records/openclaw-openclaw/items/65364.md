## Summary

Keep open: current main does not include the proposed runtime-auth override API, but this external PR still has source-reproducible correctness/security blockers and missing real behavior proof before merge.

## What This Changes

The PR adds a public plugin SDK API for runtime provider auth overrides and wires it into model auth resolution, plugin registry state, and media-understanding auto selection.

## Best Possible Solution

Land a revised, documented additive SDK/auth seam that normalizes provider ids, preserves model context and fallback behavior, and has maintainer/API/security approval plus real plugin-run proof.

## Reproduction Assessment

Not applicable for the feature request itself; yes for the review blockers, because PR-head source shows raw provider matching, empty model fallback, and first-entry image selection.

## Solution Assessment

No as-is; a dedicated additive API is plausible, but this patch should normalize provider ids, thread known model ids, preserve image fallback behavior, and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:392`
  - body: The override loop compares raw provider strings, but current auth resolution accepts aliases such as `aws-bedrock` for `amazon-bedrock`. A plugin registered for the canonical provider can be skipped and OpenClaw may fall back to normal credentials instead of the intended brokered auth; normalize both requested and registered provider ids, including the availability check.
  - confidence: 0.88
- **[P2] Pass known model IDs into override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback receives `params.modelId ?? ""`, but media execution paths still call `resolveApiKeyForProvider` without passing their selected model. Model-scoped credential brokers can then see an empty model id and return null or the wrong credential; thread the known model through those resolver calls or make the context explicitly optional.
  - confidence: 0.82
- **[P2] Keep image auto selection from pinning tentative providers:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` uses the override-aware candidate list but returns only `keyEntries[0]`. If the first provider is conditionally override-backed and fails at request time, callers of this single-model helper never consider later key-backed providers; preserve ordered fallback behavior or exclude tentative override-only providers here.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: No supply-chain changes were found, but the new trusted plugin auth override seam still has alias-bypass and model-context routing gaps.

Concerns:

- **[medium] Alias calls can bypass brokered auth overrides:** `src/agents/model-auth.ts:392`
  - body: Raw provider-id matching means an alias-form request can skip a registered canonical override and use default credentials instead, undermining plugins intended to broker all auth for a provider family.
  - confidence: 0.82
- **[low] Model-scoped broker context can be erased:** `src/agents/model-auth.ts:399`
  - body: Side paths can call the new override surface with an empty model id even when the selected model is known, which can weaken model-scoped broker authorization or route a request through unintended credentials.
  - confidence: 0.74
