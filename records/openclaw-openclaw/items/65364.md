## What This Changes

## Best Possible Solution

Land a revised additive SDK/auth seam that normalizes provider ids, carries known model context into override callbacks, preserves fallback for tentative media/image auth candidates, documents the API, and includes redacted real plugin-run proof plus maintainer/security approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize provider ids before matching overrides:** `src/agents/model-auth.ts:392`
  - body: Override matching uses raw string equality, but the surrounding auth contract accepts aliases such as `aws-bedrock` for `amazon-bedrock`. A plugin registered for the canonical provider can be skipped and OpenClaw may fall back to default credentials instead of the intended brokered auth; normalize requested and registered ids, including the availability check.
  - confidence: 0.9
- **[P2] Pass known model ids to override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback receives an empty string when callers omit `modelId`, and media execution still calls `resolveApiKeyForProvider` without passing the selected entry model. Model-scoped brokers cannot reliably choose credentials and may return null or the wrong key; thread known selected model ids through resolver calls or make the context explicitly unknown.
  - confidence: 0.84
- **[P2] Preserve fallback for tentative image auth candidates:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` uses the override-aware candidate list but returns only the first entry. If that provider is conditionally override-backed and fails at request time, sticker/image callers never try later key-backed providers; keep ordered fallback or avoid tentative override-only availability in this single-result resolver.
  - confidence: 0.78
