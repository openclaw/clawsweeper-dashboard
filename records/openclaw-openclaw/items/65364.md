Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:392`
  - body: `resolveApiKeyForProvider` accepts provider aliases elsewhere, but this loop checks raw `includes(provider)`. A plugin registered for `amazon-bedrock` will be skipped for an `aws-bedrock` caller, and availability checks can disagree with auth resolution; normalize both sides before matching.
  - confidence: 0.89
- **[P2] Thread model IDs into override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback receives `params.modelId ?? ""`, but known model-bearing paths still call the resolver without a model id. Model-scoped brokers cannot choose the right credentials and may return null or incorrect tokens; pass the selected model from call sites or make the missing context explicit.
  - confidence: 0.86
- **[P2] Preserve image fallback candidates:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` builds ordered key entries but collapses them to `keyEntries[0]`. If the first provider is only conditionally override-backed and fails at execution time, later usable key-backed providers are never tried.
  - confidence: 0.86
