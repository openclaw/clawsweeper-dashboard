Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:392`
  - body: Override matching uses raw string equality, but the surrounding auth contract accepts aliases such as `aws-bedrock` for `amazon-bedrock`.   - confidence: 0.9

  - body: The callback receives an empty string when callers omit `modelId`, and media/image execution paths can still resolve auth without passing the selected entry model. Model-scoped brokers cannot reliably choose credentials and may return null or the wrong key; thread known selected model IDs through resolver calls or make unknown model context explicit.
  - confidence: 0.84
- **[P2] Preserve fallback for tentative image auth candidates:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` uses the override-aware candidate list but returns only the first entry. If that provider is conditionally override-backed and fails at request time, callers using the single resolved image model never try later key-backed providers; keep ordered fallback or avoid tentative override-only availability in this single-result resolver.
  - confidence: 0.78
