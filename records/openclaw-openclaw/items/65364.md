Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:392`
  - body: Override matching uses raw string equality, but provider auth resolution otherwise accepts aliases such as aws-bedrock for amazon-bedrock. A plugin registered for the canonical provider can be skipped for alias-form requests, including the opt-in availability check.
  - confidence: 0.86
- **[P2] Pass real model context to override callbacks:** `src/agents/model-auth.ts:399`
  - body: The override callback gets an empty modelId whenever a caller has not been updated to pass the new optional field. Model-scoped brokers then cannot distinguish the target model and may return null or the wrong credential, so the patch should thread model identity through the runtime auth call sites instead of defaulting it away.
  - confidence: 0.82
- **[P2] Preserve fallback when resolving auto image auth:** `src/media-understanding/runner.ts:594`
  - body: resolveAutoImageModel collapses the new ordered candidate list to the first entry. If that first provider is only conditionally override-backed and fails at request time, direct image-model callers never get a chance to use later key-backed providers that the new fallback list was meant to preserve.
  - confidence: 0.8
