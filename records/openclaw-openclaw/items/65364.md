## Summary

Keep open: current main still does not provide the requested runtime provider-auth override API, and the linked feature issue was explicitly superseded into this PR, but the PR head still has source-visible auth-routing blockers and lacks external real behavior proof.

## What This Changes

The PR adds a public plugin SDK provider runtime auth override API, wires it into model-auth and media provider selection, stores active override registrations in plugin runtime state, and adds focused tests.

## Best Possible Solution

Land a revised, documented additive SDK/auth seam that normalizes provider ids, preserves model context and fallback behavior, and includes maintainer/security approval plus real plugin-run proof.

## Reproduction Assessment

Not applicable for the feature request itself; yes for the review blockers, because PR-head source shows raw provider matching, empty model fallback, and first-entry image selection without needing a live run.

## Solution Assessment

No as-is; a dedicated additive API is plausible, but this patch should normalize provider ids, thread known model ids, preserve image fallback behavior, document the public seam, and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize provider IDs before matching auth overrides:** `src/agents/model-auth.ts:392`
  - body: The override loop compares raw provider strings, but current auth resolution accepts aliases such as `aws-bedrock` for `amazon-bedrock`. A plugin registered for the canonical provider can be skipped and OpenClaw may fall back to normal credentials instead of the intended brokered auth; normalize both requested and registered provider ids, including the availability check.
  - confidence: 0.88
- **[P2] Thread known model IDs into auth override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback receives `params.modelId ?? ""`, but media execution paths still call `resolveApiKeyForProvider` without passing their selected model. Model-scoped credential brokers can then see an empty model id and return null or the wrong credential; thread the known model through resolver calls or make the context explicitly optional.
  - confidence: 0.82
- **[P2] Preserve image fallback for tentative override candidates:** `src/media-understanding/runner.ts:594`
  - body: `resolveAutoImageModel` uses the override-aware candidate list but returns only `keyEntries[0]`. If the first provider is conditionally override-backed and fails at request time, callers of this single-model helper never consider later key-backed providers; preserve ordered fallback behavior or exclude tentative override-only providers here.
  - confidence: 0.76
