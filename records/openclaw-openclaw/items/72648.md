## Summary

Keep open; the feature direction matches the linked open dmScope routing request, but this branch regresses exec notification, heartbeat canonicalization, and prompt-trust contracts, and it still lacks contributor real behavior proof.

## What This Changes

The PR adds a dmScope-based event-routing normalizer and rewires exec completion plus heartbeat session resolution in `src/agents/bash-tools.exec-runtime.ts`, `src/infra/heartbeat-runner.ts`, and `src/security/dm-policy-shared.ts`.

## Best Possible Solution

Repair or replace the branch with a narrow dmScope-aware routing seam that normalizes enqueue and wake keys together while preserving current exec trust, delivery context, payload, and heartbeat canonicalization contracts.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by comparing PR head `69d7317930b5cfa62b027125a42a166a1a752be1` with current main: the branch removes contracts asserted by current exec, heartbeat, and system-event tests.

## Solution Assessment

No. The direction is valid, but this implementation rewrites too much of the existing exec and heartbeat routing contract; the safer solution is a small dmScope-aware seam layered after current canonicalization and ownership checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Preserve heartbeat session canonicalization:** `src/infra/heartbeat-runner.ts:319-327`
  - body: This returns raw forced/configured keys and falls back to `mainEntry` without the current `toAgentStoreSessionKey`, `canonicalizeMainSessionAlias`, or resolved-agent ownership checks. Main aliases and another agent's session key can now run against the wrong or missing heartbeat session instead of being normalized or rejected.
  - confidence: 0.96
- **[P1] Use `session.dmScope` for routing decisions:** `src/agents/bash-tools.exec-runtime.ts:380-385`
  - body: The new normalizer is fed `cfg.session?.scope`, which is the store scope (`per-sender`/`global`), not `session.dmScope` (`main`/peer scopes). In the normal reported config, the pinned-owner branch will not run, so the PR can miss the central dmScope behavior it is meant to add.
  - confidence: 0.92
- **[P2] Keep notify-on-exit behind the background opt-in:** `src/agents/bash-tools.exec-runtime.ts:330-333`
  - body: Current main only notifies after the process was backgrounded and `notifyOnExit` is true. This branch only suppresses `notifyOnExit === false`, so foreground sessions or unset notify flags can enqueue system events and wake heartbeat on ordinary exec completion.
  - confidence: 0.93
- **[P1] Use the stored notify delivery context:** `src/agents/bash-tools.exec-runtime.ts:360-361`
  - body: `ProcessSession` stores the immutable origin route in `notifyDeliveryContext`, but this reads a generic `deliveryContext` property that the session type does not carry. Delayed exec completions can lose the pinned channel/topic and fall back to stale session route metadata.
  - confidence: 0.95
- **[P1] Keep exec completion events untrusted:** `src/agents/bash-tools.exec-runtime.ts:388-393`
  - body: `enqueueSystemEvent` defaults missing `trusted` to true. Omitting `trusted: false` here makes exec completion text and command output enter the prompt as privileged System context; the same omission appears in `emitExecSystemEvent`.
  - confidence: 0.96
- **[P1] Wake exec completions as `exec-event`:** `src/agents/bash-tools.exec-runtime.ts:395-400`
  - body: Heartbeat classifies exec completions by the `exec-event` source/reason path, and the scheduler reads `coalesceMs`. This call uses `exec-exit` and misspells `coalesceMs` as `coalesMs`, so completions lose the exec-specific prompt path and zero-delay wake behavior.
  - confidence: 0.96
- **[P2] Preserve structured exec completion payloads:** `src/agents/bash-tools.exec-runtime.ts:364-366`
  - body: Current completion events include the exec id, exit label, and compact output tail. This replacement emits only an exit-code or signal string, so output-bearing background completions no longer carry the result payload that heartbeat prompts and tests rely on.
  - confidence: 0.9
