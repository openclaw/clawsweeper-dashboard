## Summary

Keep open: the target is a real open routing gap, but the latest PR head still breaks existing exec completion, prompt-trust, delivery-context, and heartbeat canonicalization contracts, and it has no after-fix real behavior proof.

## What This Changes

The branch adds a shared event-routing normalizer and rewires bash exec completion plus heartbeat resolution to route selected session events toward an agent main session.

## Best Possible Solution

Replace or repair this branch with a narrow dmScope-aware routing seam that normalizes enqueue and wake keys together while preserving current exec trust, delivery context, payload, and heartbeat ownership checks.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by comparing PR head 69d7317930b5cfa62b027125a42a166a1a752be1 with current main and the existing exec/heartbeat tests that assert notifyOnExit gating, trusted:false, delivery context, exec-event wake metadata, and heartbeat key canonicalization.

## Solution Assessment

No. The requested direction is valid, but this implementation rewrites too much of the existing exec and heartbeat routing contract; the safer solution is a small dmScope-aware seam layered after current canonicalization and ownership checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Keep exec completions untrusted:** `src/agents/bash-tools.exec-runtime.ts:389-393`
  - body: enqueueSystemEvent defaults missing trusted to true. Omitting trusted:false here makes variable exec completion text enter the prompt as privileged System context; the same omission appears in emitExecSystemEvent.
  - confidence: 0.96
- **[P1] Wake notify completions as exec-event:** `src/agents/bash-tools.exec-runtime.ts:396-400`
  - body: Heartbeat scheduling and prompt classification use the exec-event reason/source path and read coalesceMs. This call uses exec-exit and misspells coalesceMs as coalesMs, so background completions lose the exec-specific prompt path and zero-delay wake behavior.
  - confidence: 0.96
- **[P2] Restore the backgrounded notify guard:** `src/agents/bash-tools.exec-runtime.ts:330-333`
  - body: Current main only notifies after a process was backgrounded and notifyOnExit is true. This branch suppresses only notifyOnExit === false, so foreground sessions or sessions with an unset notify flag can enqueue completion events and wake heartbeat on ordinary exec completion.
  - confidence: 0.93
- **[P1] Use the stored notify delivery context:** `src/agents/bash-tools.exec-runtime.ts:360-361`
  - body: ProcessSession stores the immutable origin route in notifyDeliveryContext, but this reads a generic deliveryContext field the session type does not carry. Delayed completions can lose the pinned channel/topic and fall back to stale route metadata.
  - confidence: 0.95
- **[P1] Preserve heartbeat key canonicalization:** `src/infra/heartbeat-runner.ts:321-328`
  - body: This returns raw forced/configured keys and falls back to mainEntry without the current toAgentStoreSessionKey, canonicalizeMainSessionAlias, or resolved-agent ownership checks. A main alias or another agent's session key can now run against the wrong or missing heartbeat session.
  - confidence: 0.96
