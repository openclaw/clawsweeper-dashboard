## Summary

Keep open. This PR is an active external attempt to fix https://github.com/openclaw/openclaw/issues/71581, but the current head still regresses current-main exec notification and heartbeat session contracts, has prompt-trust/security findings, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a shared event-routing normalizer and rewires bash exec completion plus heartbeat session resolution for session-aware dmScope-style system-event routing.

## Best Possible Solution

Repair or replace this branch with a narrow implementation that normalizes system-event enqueue keys and forced heartbeat targets together under the existing single-owner dmScope=main policy while preserving current exec trust, payload, delivery-context, and wake contracts.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists by comparing PR head 69d7317930b5cfa62b027125a42a166a1a752be1 with current main: the branch removes current guards and fields that existing tests assert for notifyOnExit, delivery context, trusted:false, exec-event wakes, and heartbeat key canonicalization.

## Solution Assessment

No. The requested direction is valid, but this implementation rewrites too much of the existing exec and heartbeat routing contracts; the safer solution is a small dmScope-aware routing seam layered after current canonicalization and ownership checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Preserve heartbeat session canonicalization:** `src/infra/heartbeat-runner.ts:323-327`
  - body: This returns raw forced/configured keys and falls back to mainEntry without the current toAgentStoreSessionKey, canonicalizeMainSessionAlias, or resolveAgentIdFromSessionKey checks. A main alias or another agent's session key can now run against the wrong or missing heartbeat session instead of being normalized or rejected.
  - confidence: 0.96
- **[P2] Keep notify-on-exit behind the background opt-in:** `src/agents/bash-tools.exec-runtime.ts:331-333`
  - body: Current main only notifies after the process was backgrounded and notifyOnExit is true. This branch only suppresses notifyOnExit=false, so foreground sessions or sessions with an unset notify flag can enqueue system events and wake heartbeat on ordinary exec completion.
  - confidence: 0.93
- **[P1] Use the stored notify delivery context:** `src/agents/bash-tools.exec-runtime.ts:360-361`
  - body: ProcessSession stores the immutable origin route in notifyDeliveryContext, but this reads a generic deliveryContext property that the session type does not carry. Delayed exec completions will lose the pinned channel/topic and can fall back to stale session route metadata.
  - confidence: 0.95
- **[P1] Keep exec completion events untrusted:** `src/agents/bash-tools.exec-runtime.ts:389-393`
  - body: enqueueSystemEvent defaults missing trusted to true. Omitting trusted:false here makes exec completion text and command output enter the prompt as privileged System context; the same omission appears in emitExecSystemEvent.
  - confidence: 0.96
- **[P1] Wake exec completions as exec-event:** `src/agents/bash-tools.exec-runtime.ts:397-400`
  - body: Heartbeat classifies exec completions by the exec-event source/reason path, and the scheduler reads coalesceMs. This call uses exec-exit and misspells coalesceMs as coalesMs, so completions lose the exec-specific prompt path and zero-delay wake behavior.
  - confidence: 0.96
- **[P2] Preserve structured exec completion payloads:** `src/agents/bash-tools.exec-runtime.ts:364-366`
  - body: Current completion events include the exec id, exit label, and compact output tail. This replacement emits only an exit-code or signal string, so output-bearing background completions no longer carry the result payload that heartbeat prompts and tests rely on.
  - confidence: 0.9
- **[P2] Use effective dmScope and allowlist inputs:** `src/agents/bash-tools.exec-runtime.ts:380-385`
  - body: The routing decision should be gated by the effective single-owner DM policy for the channel/account. This code reads cfg.session.scope and root/default allowlist shapes instead of the dmScope/allowlist state used by inbound routing, so the linked configuration can still miss or inconsistently apply the main-session rewrite.
  - confidence: 0.88
