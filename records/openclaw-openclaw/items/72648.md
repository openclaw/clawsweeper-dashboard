## Summary

Keep open. The linked feature request remains valid and this PR is an active implementation candidate, but the current branch regresses exec-event trust, delivery context, heartbeat canonicalization, and lacks real behavior proof.

## What This Changes

The PR adds a shared event-routing normalizer and rewires exec completion plus heartbeat session resolution toward agent-main-session routing for dmScope-style system events.

## Best Possible Solution

Replace or repair the branch with a narrow dmScope-aware routing seam that normalizes enqueue and wake keys together while preserving current exec trust, delivery context, payload, subagent, and heartbeat canonicalization contracts.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by comparing PR head 69d7317930b5cfa62b027125a42a166a1a752be1 with current main: the branch removes or changes contracts asserted by current exec and heartbeat tests.

## Solution Assessment

No. The direction is plausible for https://github.com/openclaw/openclaw/issues/71581, but this implementation rewrites too much of the existing exec and heartbeat routing contract instead of layering a small dmScope-aware seam after current safety checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Preserve heartbeat session canonicalization:** `src/infra/heartbeat-runner.ts:321-327`
  - body: This returns raw forced/configured keys with store[candidate] || mainEntry, bypassing current toAgentStoreSessionKey, canonicalizeMainSessionAlias, and same-agent ownership checks. Main aliases or another agent's session key can now run against the wrong or missing heartbeat session.
  - confidence: 0.96
- **[P2] Keep notify-on-exit behind the background opt-in:** `src/agents/bash-tools.exec-runtime.ts:330-333`
  - body: Current main only notifies after the process was backgrounded and notifyOnExit is true. This branch only suppresses notifyOnExit=false, so foreground sessions or sessions with an unset notify flag can enqueue system events and wake heartbeat on ordinary exec completion.
  - confidence: 0.93
- **[P1] Use the stored notify delivery context:** `src/agents/bash-tools.exec-runtime.ts:360-361`
  - body: ProcessSession stores the immutable origin route in notifyDeliveryContext, but this reads a generic deliveryContext property that the session type does not carry. Delayed exec completions will lose the pinned channel/topic and can fall back to stale session route metadata.
  - confidence: 0.95
- **[P1] Keep exec completion events untrusted:** `src/agents/bash-tools.exec-runtime.ts:389-393`
  - body: enqueueSystemEvent defaults missing trusted to true. Omitting trusted:false here makes exec completion text and command output enter the prompt as privileged System context.
  - confidence: 0.96
- **[P1] Wake exec completions as exec-event:** `src/agents/bash-tools.exec-runtime.ts:397-400`
  - body: Heartbeat classifies exec completions through the exec-event source/reason path and the scheduler reads coalesceMs. This call uses exec-exit and misspells coalesceMs as coalesMs, so completions lose the exec-specific prompt path and zero-delay wake behavior.
  - confidence: 0.96
- **[P2] Preserve structured exec completion payloads:** `src/agents/bash-tools.exec-runtime.ts:364-366`
  - body: Current completion events include the exec id, exit label, and compact output tail. This replacement emits only an exit-code or signal string, so output-bearing background completions no longer carry the result payload that heartbeat prompts and tests rely on.
  - confidence: 0.9
- **[P2] Use effective dmScope and allowlist inputs:** `src/agents/bash-tools.exec-runtime.ts:380-385`
  - body: The routing decision should use the same effective dmScope and single-owner allowlist inputs as inbound routing. This code reads session.scope and broad default/root allowlist shapes instead, so it can rewrite when dmScope is not main or miss the intended single-owner condition.
  - confidence: 0.88
- **[P1] Keep emitted exec events untrusted:** `src/agents/bash-tools.exec-runtime.ts:495-499`
  - body: emitExecSystemEvent also omits trusted:false even though this helper can emit variable exec text. That makes helper-emitted exec output privileged System context instead of untrusted event context.
  - confidence: 0.95
- **[P2] Do not wake heartbeat for subagent exec events:** `src/agents/bash-tools.exec-runtime.ts:501-505`
  - body: Current main skips heartbeat wakes for subagent session keys because subagent sessions receive results via process poll and announce flow. This unconditional wake can fall back to the main session and create spurious exec-event runs.
  - confidence: 0.9
