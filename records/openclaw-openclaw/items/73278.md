## Summary

Keep open. Current main still has the conflicting prompt path, and open linked PRs are implementation candidates rather than a reason to close before one canonical fix lands.

## What This Changes

Prevent Discord autoThread mentions from producing both a parent-channel message-tool reply and a final assistant reply in the created thread.

## Best Possible Solution

Land one canonical delivery-mode-aware prompt and skill clarification that reserves same-conversation `message.send` warnings for automatic final-delivery turns while preserving tool-only visible output for default Discord guild/channel turns.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: automatic visible Discord autoThread delivery targets the created thread, while Codex and Discord prompts can still steer the agent to a separate `message.send`.

## Solution Assessment

Yes, the right fix boundary is prompt and skill guidance, but it should be delivery-mode-aware rather than unconditional final-response guidance.
