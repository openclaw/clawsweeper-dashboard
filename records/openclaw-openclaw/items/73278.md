## Summary

Keep open. Current main still has the conflicting prompt path for configured automatic visible replies: Codex runtime guidance and the bundled Discord skill can steer a normal Discord reply through `message.send`, while the Discord autoThread final-delivery path routes normal final responses to the created thread; the three open closing PRs are implementation candidates, not a reason to close before merge.

## What This Changes

Discord autoThread mentions can produce both a parent-channel `message.send` reply and a normal final response in the created thread because prompt guidance over-directs normal conversational replies toward the message tool.

## Best Possible Solution

Land one canonical prompt and skill clarification that reserves `message.send` for explicit sideband Discord operations and leaves normal current-conversation replies to final response delivery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Discord `autoThread: true` plus configured automatic visible group/channel replies can route the final response to the created thread while the Codex runtime prompt and Discord skill can still elicit an extra `message.send`.

## Solution Assessment

Yes. The narrow maintainable fix is prompt and skill wording, because the Discord delivery mechanics already route automatic final responses to the created thread.
