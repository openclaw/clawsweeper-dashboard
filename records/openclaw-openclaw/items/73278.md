## Summary

Keep open. Current main still has the conflicting Codex runtime and Discord skill prompt path, and the linked closing PRs are open implementation candidates rather than landed fixes.

## What This Changes

Discord `autoThread: true` channel mentions should produce one normal reply in the created thread, not both a parent-channel `message.send` reply and a final thread reply.

## Best Possible Solution

Land one canonical prompt and skill clarification that reserves `message.send` for explicit sideband Discord operations and leaves ordinary current-conversation replies to final assistant delivery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Discord `autoThread: true` plus configured automatic visible group/channel replies can route the final response to the created thread while the Codex runtime prompt and Discord skill can still elicit an extra `message.send`.

## Solution Assessment

Yes. The narrow maintainable fix is prompt and skill wording, because the Discord delivery mechanics already route automatic final responses to the created thread.
