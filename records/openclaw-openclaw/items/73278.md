## Summary

Keep open. Current main still has the conflicting Codex runtime and Discord skill guidance that can steer a normal Discord reply through the message tool while the autoThread final-response path targets the created thread; the open closing PRs are implementation candidates, not a reason to close before merge.

## What This Changes

Prevent Discord autoThread channel mentions from producing both a parent-channel message.send reply and a final conversational reply in the created thread.

## Best Possible Solution

Land one canonical prompt and skill clarification that reserves the message tool for explicit sideband Discord operations and leaves normal current-conversation replies to final response delivery.

## Reproduction Assessment

Yes. The source-level path is high confidence: Discord `autoThread: true` plus automatic visible group replies can route the final response to the created thread while current Codex/Discord prompt wording can still elicit an extra `message.send`.

## Solution Assessment

Yes. The narrow maintainable fix is prompt and skill wording, because the current autoThread delivery mechanics already route final responses to the created thread.
