## Summary

Keep open: the PR is a focused implementation, but it changes the documented restricted-tool contract and still has a blocking completion-visibility gap plus a public docs mismatch before it should merge.

## What This Changes

The PR decouples exec backgrounding from process tool visibility, adds a hidden-process follow-up/timeoutless guard, updates exec docs, and adds process-denied exec regression coverage.

## Best Possible Solution

Have maintainers decide the hidden-process finite-backgrounding contract in https://github.com/openclaw/openclaw/issues/82178; if accepted, guarantee observable completion without `process` and keep both public docs pages aligned.

## Reproduction Assessment

Yes, source-level: current main computes exec backgrounding from `process` policy, and current docs/tests confirm process-denied exec runs synchronously despite `yieldMs`. I did not run tests because this review is read-only.

## Solution Assessment

No, not as submitted. The direction may be acceptable, but the patch needs maintainer approval for the policy change and should fix quiet-success completion visibility plus the background-process docs mismatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve quiet completion visibility without process:** `src/agents/bash-tools.exec.ts:1681`
  - body: The new hidden-process guidance tells the model to rely on completion wake notifications, but successful backgrounded commands with no output still do not enqueue a wake unless `notifyOnExitEmptySuccess` is true. With `process` hidden there is no poll/log path, so a quiet command can finish before timeout with no observable completion; force an empty-success notification for this mode, reject it, or provide another completion path.
  - confidence: 0.91

  - body: `docs/tools/exec.md` now advertises finite process-hidden backgrounding, but `docs/gateway/background-process.md` still says process-denied exec runs synchronously and points quiet-success confirmation through `process`. Update the background-process page in the same PR so the public contract is not contradictory.
  - confidence: 0.88
