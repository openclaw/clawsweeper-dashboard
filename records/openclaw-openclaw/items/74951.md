## Summary

Keep open. The PR still changes a security-sensitive exec/process policy path without guarding `timeout: 0`, leaves the background-process docs contradictory, and is missing the required changelog entry.

## What This Changes

The PR decouples exec auto-backgrounding from process tool visibility, updates the process-hidden follow-up message and exec docs, and adds a regression test for long exec commands when `process` is denied.

## Best Possible Solution

Land a revised PR only after maintainers accept process-hidden backgrounding, with a finite-timeout guard or rejection for timeoutless calls, synchronized docs, changelog coverage, and regression tests for both accepted and rejected paths.

## Reproduction Assessment

Yes. The code-backed reproduction is `createOpenClawCodingTools` with `tools.deny: ["process"]`, a long exec command, and a short `yieldMs`; adding `timeout: 0` exercises the unsafe path because current runtime code forwards no supervisor timeout.

## Solution Assessment

No. Decoupling backgrounding from process visibility may be a reasonable direction, but this patch needs the timeoutless guard and docs/changelog follow-through before it is the narrow maintainable fix.
