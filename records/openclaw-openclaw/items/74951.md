## Summary

Keep this PR open, but not merge-ready: the latest patch resolves the earlier timeoutless-backgrounding and docs-sync concerns, yet it still strands quiet successful commands when `process` is denied, and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR decouples exec backgrounding from process tool visibility, rejects timeoutless process-hidden backgrounding, updates exec background docs, and adds process-denied regression tests.

## Best Possible Solution

Land a revised PR only after maintainers accept process-hidden finite-timeout backgrounding and the implementation preserves observable completion for quiet successes, with docs, tests, and real proof aligned.

## Reproduction Assessment

Yes, by source inspection: with `process` denied, a quiet command that crosses `yieldMs` would return `status: running`, then `maybeNotifyOnExit` suppresses the clean empty-output completion event while no `process` tool exists to poll it. I did not run tests because this review is read-only.

## Solution Assessment

No, not as submitted. Decoupling backgrounding from process visibility may be acceptable, but the patch needs quiet-success completion visibility and real after-fix proof before it is the narrow maintainable fix.
