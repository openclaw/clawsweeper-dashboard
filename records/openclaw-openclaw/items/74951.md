## Summary

Keep open. The PR targets a real exec turn-blocking problem, but the current patch can create unmanaged timeoutless background processes when `process` is denied, leaves background-process docs contradictory, is missing the required changelog entry, and is currently dirty against `main`.

## What This Changes

The PR decouples exec auto-backgrounding from process tool visibility, adds hidden-process follow-up text and exec docs, and tests long exec behavior when `process` is denied.

## Best Possible Solution

Land a revised PR only if maintainers accept process-hidden backgrounding, with a finite-timeout guard or rejection for timeoutless process-denied calls, synchronized docs, a changelog entry, and coverage for both accepted and rejected paths.

## Reproduction Assessment

Yes. Use `createOpenClawCodingTools` with `tools.deny: ["process"]`, run a long exec command with short `yieldMs`, and add `timeout: 0` to exercise the unsafe path that has no process follow-up tool and no supervisor timeout.

## Solution Assessment

No. Decoupling backgrounding from process visibility may be the right direction, but the current patch is not the narrow maintainable fix until it handles timeoutless calls, updates all relevant docs, and records the user-facing change.
