## Summary

Keep open. The PR targets a real agent-turn blocking problem, but the submitted patch still broadens exec backgrounding under a process-denied policy without guarding the timeoutless path, and it leaves the exec docs contradictory plus the required changelog entry missing.

## What This Changes

The PR decouples exec auto-backgrounding from `process` tool visibility, changes hidden-process follow-up text/docs, and adds regression coverage for a long exec command when `process` is denied.

## Best Possible Solution

Land a revised PR only after maintainers accept the process-hidden backgrounding contract; the revision should enforce or preserve a finite timeout whenever `process` is unavailable, reject timeoutless backgrounding in that mode, align both exec docs, add the required changelog credit, and keep focused regression coverage.

## Reproduction Assessment

Yes. The focused reproduction is `createOpenClawCodingTools` with `tools.deny: ["process"]`, then a long `exec` call with short `yieldMs`; adding `timeout: 0` exercises the unmanaged-session path because current runtime code treats nonpositive timeouts as no supervisor timeout.

## Solution Assessment

No, not as submitted. Decoupling auto-backgrounding from process visibility may be acceptable, but the narrow maintainable fix needs a finite-timeout guard or rejection path before returning a background session that has no `process` follow-up tool.
