## Summary

Keep open. The PR addresses a real long-command blocking problem, but the existing ClawSweeper review finding is still valid: as submitted it lets process-denied exec calls background without preserving a finite lifecycle control when `timeout: 0` is used, and the docs become contradictory.

## What This Changes

The PR decouples exec auto-backgrounding from `process` tool availability, updates hidden-process follow-up messaging/docs, and adds regression coverage for a process-denied long exec command.

## Best Possible Solution

Land a corrected version only if maintainers accept the new process-denied contract: keep process-hidden long commands from pinning the turn, but reject or override timeoutless backgrounding when `process` is unavailable, and align the exec docs/tool guidance plus tests for that guard.

## Reproduction Assessment

Yes. The focused path is `createOpenClawCodingTools` with `tools.deny: ["process"]`, then a long `exec` call with a short `yieldMs`; the unsafe variant adds `timeout: 0`, which current runtime code and tests prove disables the supervisor timeout.

## Solution Assessment

No, not as submitted. Decoupling backgrounding from `process` visibility may be acceptable, but the maintainable version needs a finite timeout guard or rejection path before the process-denied background session can be returned.
