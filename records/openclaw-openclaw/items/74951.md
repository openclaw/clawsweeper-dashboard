## Summary

Keep open: this is a focused and useful PR, but it is not merge-ready because process-hidden backgrounding can still leave quiet successful commands with no observable completion path, and one public background-process doc still describes the old contract.

## What This Changes

The PR decouples finite-timeout `exec` backgrounding from `process` tool visibility, adds hidden-process follow-up text and a timeoutless guard, updates exec docs, and adds process-denied regression coverage.

## Best Possible Solution

If maintainers accept the policy direction in https://github.com/openclaw/openclaw/issues/82178, land a bounded patch that guarantees observable completion without `process`, preserves the timeoutless safety boundary, and keeps both public docs pages and focused tests aligned.

## Reproduction Assessment

Yes, source-level: current main computes exec backgrounding from `process` policy, and docs/tests describe process-denied `exec` as synchronous. I did not run tests because this review is read-only.

## Solution Assessment

No, not as submitted. Decoupling may be maintainable, but the PR should preserve quiet-success completion visibility, update the background-process docs, and wait for maintainer acceptance of the restricted-tool contract.
