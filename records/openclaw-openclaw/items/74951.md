## Summary

Keep open: the PR targets a real long-running exec pain point, but it is not mergeable as submitted because it changes process-hidden code-execution lifecycle policy, leaves timeoutless background sessions unmanaged, lacks real after-fix behavior proof, and is currently conflicting.

## What This Changes

The PR makes exec auto-background long commands even when the process tool is denied, adjusts hidden-process follow-up text and exec docs, and adds regression coverage.

## Best Possible Solution

Land a revised PR only if maintainers accept process-hidden backgrounding, with a finite-timeout guard or rejection for timeoutless calls, synchronized docs, regression coverage, and real after-fix proof.

## Reproduction Assessment

Yes. Source inspection shows current main runs process-denied exec synchronously, and the PR's timeout: 0 path would background without a supervisor timeout because runExecProcess only forwards positive timeout values.

## Solution Assessment

No, not as submitted. Decoupling backgrounding from process visibility may be viable, but the patch needs timeoutless-run protection, docs sync, real proof, and maintainer acceptance of the policy change.
