## Summary

Keep open. The PR addresses a real long-running exec behavior gap, but the current patch is not mergeable because it allows process-denied background exec sessions without guarding timeoutless calls and leaves user-facing docs/changelog incomplete.

## What This Changes

The PR decouples exec auto-backgrounding from process tool visibility, adjusts hidden-process follow-up text/docs, and adds a process-denied long-exec regression test.

## Best Possible Solution

Land a revised PR only after maintainer approval of process-hidden backgrounding, with timeoutless calls rejected or forced to a finite timeout, synchronized docs, changelog coverage, and regression tests for both accepted and rejected paths.

## Reproduction Assessment

Yes. The code-backed reproduction is `createOpenClawCodingTools` with `tools.deny: ["process"]`, a long exec command, and a short `yieldMs`; adding `timeout: 0` exercises the unsafe path because current runtime code forwards no supervisor timeout.

## Solution Assessment

No. Decoupling backgrounding from process visibility may be acceptable, but this patch needs a timeoutless guard plus docs/changelog follow-through before it is the narrow maintainable fix.
