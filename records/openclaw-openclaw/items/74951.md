## Summary

Keep open. The PR addresses a real turn-blocking failure mode, but the submitted patch broadens background exec behavior when `process` is denied without preserving a finite automatic stop path, and it leaves the background-process documentation contradictory.

## What This Changes

The PR decouples exec auto-backgrounding from `process` tool visibility, adds hidden-process follow-up messaging/type plumbing, updates the exec tool docs, and adds regression coverage for a process-denied long exec command.

## Best Possible Solution

Land a corrected version only if maintainers accept the new process-denied contract: allow long commands to release the agent turn, but reject or override timeoutless backgrounding when `process` is unavailable, then align both exec docs and regression tests for that guarded behavior.

## Reproduction Assessment

Yes. The focused reproduction is `createOpenClawCodingTools` with `tools.deny: ["process"]`, a long exec call with a short `yieldMs`, and the unsafe variant adding `timeout: 0`; current runtime code and tests show that `timeout: 0` disables the supervisor timeout.

## Solution Assessment

No, not as submitted. Decoupling backgrounding from `process` visibility may be the right direction, but the maintainable fix needs a finite-timeout guard or rejection path before returning a process-denied background session.
