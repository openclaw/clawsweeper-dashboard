## Summary

Keep open: this is a focused PR for a real exec/process policy gap, but it is not merge-ready because it changes a documented restricted-tool contract, still drops quiet-success completion visibility when `process` is hidden, and lacks after-fix real behavior proof.

## What This Changes

The PR decouples exec backgrounding from process tool visibility, rejects timeoutless hidden-process backgrounding, updates exec docs, and adds process-denied exec regression tests.

## Best Possible Solution

Land a revised PR only after maintainers accept finite-timeout backgrounding without `process`, the implementation preserves observable completion for quiet successes, and the PR body includes redacted real runtime proof.

## Reproduction Assessment

Yes, by source inspection: with `process` denied, the PR makes a quiet long command return `status: "running"`, while `maybeNotifyOnExit` suppresses clean no-output completion wakes by default. I did not run tests because this review is read-only.

## Solution Assessment

No, not as submitted. Decoupling backgrounding from process visibility may be acceptable, but the patch needs quiet-success completion visibility, maintainer policy approval, and real after-fix proof before it is the narrow maintainable fix.

## Review Findings
