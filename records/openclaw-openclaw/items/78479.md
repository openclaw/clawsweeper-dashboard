## Summary

Keep this PR open: the gateway ping/pong liveness change is plausible, but the patch adds an unrelated global fs-safe Vitest stub that can break or mask filesystem-safety tests, and the supplied proof is fake-timer unit output rather than real after-fix gateway behavior.

## What This Changes

This PR adds gateway WebSocket pong tracking and a 1001 ping-timeout close path, adds regression tests, updates the changelog, and adds a shared fs-safe test stub alias.

## Best Possible Solution

Keep the WebSocket liveness fix scoped to the gateway connection handler and its tests, remove the global fs-safe alias/stub, and verify the behavior with real gateway logs before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the live production starvation path: current main source confirms missing pong tracking and the linked issue has strong logs, but I did not establish a real current-main gateway reproduction.

## Solution Assessment

Is this the best way to solve the issue? No as submitted: the ping/pong guard is a narrow maintainable improvement, but the unrelated fs-safe test alias should be removed and the broader starvation issue should not be considered fully solved by close-code cleanup alone.
