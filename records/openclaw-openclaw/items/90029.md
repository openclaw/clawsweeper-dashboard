## Summary

Keep open: the shared live catalog helper is useful, but the PR is not merge-ready because the model-list path still bypasses live discovery for providers that also expose a static fallback, and the new public SDK/auth behavior needs maintainer acceptance before release.

## What This Changes

This PR adds a public live provider model-catalog SDK helper, routes several bundled providers through it, changes OpenAI and OpenCode Go catalog discovery mode, and updates SDK docs, exports, and tests.

## Best Possible Solution

Land a version where providers with both live and static catalogs use live discovery when credentials are available, retain static fallback for unauthenticated or offline browsing, and have the public SDK contract, docs, and proof accepted by maintainers.

Yes, source-reproducible from the PR head: OpenAI and xAI expose static fallback hooks, while `loadProviderCatalogModelsForList` takes the static branch whenever that hook exists. I did not run the CLI because this review is read-only.

## Solution Assessment

No. Centralizing guarded live discovery is a plausible direction, but the current integration leaves static-first callers bypassing it and the public SDK/API shape still needs maintainer acceptance.
