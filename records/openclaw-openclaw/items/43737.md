## Label Justifications

- `P1`: The regression can block users from selecting the canonical main Control UI session and has a high-confidence source-level reproduction.
- `impact:session-state`: The issue concerns session selection state and whether the canonical main session remains visible in session lists.

## Summary

Keep open: current main still filters active sessions by recency before `sessions.list` rows are built, so a stale canonical main session can be omitted while heartbeat remains visible; https://github.com/openclaw/openclaw/pull/43984 is the open closing implementation candidate.

## What This Changes

Restore the canonical main session to the Control UI session dropdown when active-session filtering would otherwise show only heartbeat.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/43984 so shared Gateway session-entry filtering preserves canonical main sessions for both sync and async `sessions.list` before the Control UI builds dropdown options.

## Reproduction Assessment

Yes, source-level. A stale `agent:main:main` row plus a fresh heartbeat row followed by `sessions.list` with positive `activeMinutes` will filter the main row before the Control UI receives dropdown rows.

## Solution Assessment

Yes. The narrow maintainable fix belongs in shared Gateway session-entry selection; a UI-only injected dropdown option would leave `sessions.list` semantically wrong for other callers.
