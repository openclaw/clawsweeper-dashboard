## Summary

Keep open: current main still drops stale main-session entries when `sessions.list` uses `activeMinutes`, and an open closing PR already owns the fix path.

## What This Changes

The issue reports that the Control UI session dropdown shows only `heartbeat` and omits the selectable main session.

## Best Possible Solution

Fix the shared Gateway `sessions.list` entry filter so configured primary main sessions survive active-minute filtering, preserve custom `mainKey` semantics, and land or replace the open fix PR.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale `agent:main:main` row plus a fresh heartbeat row, followed by `sessions.list` with a positive `activeMinutes`; current main filters out the stale main entry before the dropdown can render it.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared Gateway session-list entry filter; a UI-only injected option should be secondary unless the Gateway can still omit a real main session after that fix.
