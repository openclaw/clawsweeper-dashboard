## Summary

Keep open. Current main still has the gateway active-session filter that can omit a stale top-level main session before the Control UI dropdown receives it, and open PR #43984 already targets this regression with closing syntax.

## What This Changes

The issue requests that the Control UI session dropdown show the main session alongside heartbeat and other active sessions.

## Best Possible Solution

Preserve the resolved top-level main session in gateway sessions.list results when active-session filtering is requested, continue filtering stale non-main sessions, and cover stale main plus active heartbeat with a regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a store with stale agent:main:main and a fresh heartbeat session, followed by sessions.list with a positive activeMinutes filter and dropdown rendering while heartbeat is current.

## Solution Assessment

Yes. Fixing the gateway active-list behavior is the narrowest maintainable solution because it keeps the API contract consistent for all consumers instead of making the Control UI invent a missing main row locally.
