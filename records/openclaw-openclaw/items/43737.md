## Summary

Keep open. Current main still has a source-level path where an active `sessions.list` refresh can filter stale `agent:main:main` out before the Control UI dropdown sees it, and open PR #43984 already references this issue with closing syntax.

## What This Changes

The issue asks for the Control UI session dropdown to keep the main session visible and selectable instead of showing only heartbeat.

## Best Possible Solution

Land or replace #43984 with a gateway-level regression fix that keeps canonical agent main sessions in active `sessions.list` results while preserving active filtering for ordinary stale sessions.

## Reproduction Assessment

Yes. A high-confidence source-level path is a stale `agent:main:main` row plus a fresh heartbeat row, followed by `sessions.list` with `activeMinutes > 0`; current main filters the stale main row before the dropdown can render it.

## Solution Assessment

Yes. Fixing the gateway active-list contract is the narrowest maintainable path because every consumer gets the canonical main session consistently instead of making the Control UI invent a missing row locally.
