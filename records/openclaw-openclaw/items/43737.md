## Summary

Keep open. Current main still has the gateway active-session filter that can drop a stale canonical main session before the Control UI dropdown receives it, and open PR #43984 already targets this regression with closing syntax.

## What This Changes

The issue asks for the Control UI session dropdown to keep showing the main session alongside heartbeat or other active sessions.

## Best Possible Solution

Land or replace #43984 with a gateway-level fix that preserves the resolved top-level main session in active sessions.list results while still filtering stale non-main rows, backed by regression coverage for stale main plus active heartbeat.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a store with stale agent:main:main and fresh heartbeat, followed by sessions.list with activeMinutes > 0 and dropdown rendering while heartbeat is current; current main filters main before the UI sees it.

## Solution Assessment

Yes. Fixing the gateway active-list behavior is the narrowest maintainable solution because it keeps the sessions.list contract consistent for all consumers instead of making the Control UI invent a missing main row locally.
