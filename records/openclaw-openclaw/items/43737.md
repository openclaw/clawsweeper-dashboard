## What This Changes

The issue reports that the Control UI session dropdown only shows heartbeat and omits the main session in OpenClaw 2026.3.11.

## Best Possible Solution

Land or replace the Gateway entry-filter fix so configured primary main sessions survive activeMinutes filtering, then close this issue after the fix merges.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale agent:main:main row plus a fresh heartbeat row, followed by sessions.list with a positive activeMinutes; current main filters the stale main row before row construction.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the shared Gateway entry-selection filter. A UI-only injected dropdown option should be secondary unless the Gateway can still omit a real main session after that fix.
