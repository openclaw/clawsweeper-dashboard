## Summary

Keep open. Current main still has a source-level path where an active-only `sessions.list` refresh can filter a stale canonical main row before the Control UI dropdown sees it, and open PR #43984 already references this issue with closing syntax.

## What This Changes

The issue reports that the Control UI chat session dropdown can show only `heartbeat` while omitting the canonical `main` session.

## Best Possible Solution

Fix the gateway `sessions.list` contract so canonical agent main sessions remain visible in active lists while ordinary stale sessions stay filtered.

## Reproduction Assessment

Yes, at source level. A stale `agent:main:main` row plus a fresh heartbeat row followed by `sessions.list` with `activeMinutes > 0` can remove main before the dropdown renders options.

## Solution Assessment

Yes. A gateway-level fix is the narrowest maintainable solution because every consumer receives a consistent canonical main session instead of requiring the Control UI to invent a missing row locally.
