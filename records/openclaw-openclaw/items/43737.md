## Summary

Keep open: current main and latest v2026.5.20 still apply the `activeMinutes` cutoff before `sessions.list` rows are built, so a stale canonical main session can disappear while a newer heartbeat session remains. The linked implementation candidate at https://github.com/openclaw/openclaw/pull/43984 has closing syntax for this issue, but it is still open and conflicting.

## What This Changes

The report asks for the Control UI session dropdown to keep the canonical main session visible alongside heartbeat or other active sessions.

## Best Possible Solution

Land a shared Gateway session-list fix that preserves primary main sessions through `activeMinutes` filtering while keeping stale non-main sessions hidden.

## Reproduction Assessment

Yes, source-level. A stale `agent:main:main` row plus a fresher heartbeat row followed by `sessions.list` with positive `activeMinutes` is filtered by timestamp before the Control UI receives dropdown rows.

## Solution Assessment

Yes for the desired fix shape. The maintainable fix belongs in shared Gateway session-entry filtering; a UI-only injected option would leave public `sessions.list` behavior inconsistent.
