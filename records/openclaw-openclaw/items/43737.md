- `P1`: The original regression could block users from selecting the canonical main Control UI session.

## Summary

Close as implemented on current main: the Control UI chat session picker now loads recency-free session rows and `loadSessions` omits `activeMinutes` when that override is zero, so the reported stale-main-plus-fresh-heartbeat dropdown path no longer applies. The fix is not in the latest tagged release yet.

## What This Changes

The issue reports that the Control UI session dropdown omits the canonical main session and only shows heartbeat.

## Best Possible Solution

Keep the current recency-free chat picker load behavior and let the next release carry it; any API-wide change to pin main sessions in positive `activeMinutes` lists should be tracked separately.

## Reproduction Assessment

No for current main. Source inspection shows the chat dropdown no longer sends a positive `activeMinutes` filter, while the older v2026.5.20 path that used `120` remains source-reproducible before the fix.

## Solution Assessment

Yes for the reported Control UI dropdown. Removing recency filtering from chat picker loads is the narrow maintainable fix for missing older main/channel rows; broader `sessions.list` active-filter semantics would be a separate API decision.
