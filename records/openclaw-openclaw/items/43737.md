## Summary

Keep open: current main still applies the active-session recency filter before `sessions.list` rows and Control UI options are built, so a stale canonical main session can be omitted while heartbeat remains visible; the linked implementation path is still the open PR at https://github.com/openclaw/openclaw/pull/43984.

## What This Changes

The issue asks for the Control UI chat session dropdown to keep the canonical main session visible instead of showing only heartbeat.

## Best Possible Solution

Review, update, and land or replace https://github.com/openclaw/openclaw/pull/43984 so shared Gateway session-entry filtering preserves primary main sessions for both sync and async `sessions.list` before the Control UI builds options.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale `agent:main:main` row plus a fresh heartbeat row followed by `sessions.list` with positive `activeMinutes`, which current main filters before the Control UI receives rows.

## Solution Assessment

Yes. The narrow maintainable fix belongs in shared Gateway session-entry selection; injecting a UI-only dropdown option would leave `sessions.list` semantically wrong for other callers.
