## Summary

Keep open: current main still filters stale main-session rows out of active `sessions.list` results before the Control UI dropdown can render them, and the open linked fix PR remains the implementation path.

## What This Changes

The issue reports that the Control UI session dropdown omits the main session and only shows heartbeat after active-session filtering.

## Best Possible Solution

Land or replace the Gateway entry-selection fix so configured primary main sessions survive `activeMinutes` filtering and then close this issue after the fix merges.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale `agent:main:main` row plus a fresh heartbeat row, followed by `sessions.list` with a positive `activeMinutes`; current main filters the stale main row before row construction.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared Gateway entry-selection filter; a UI-only injected dropdown option would leave `sessions.list` semantically wrong.
