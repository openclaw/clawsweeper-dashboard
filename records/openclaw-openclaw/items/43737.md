## Summary

Keep open. Current main still has a source-level path where `sessions.list` with `activeMinutes` drops a stale canonical main session before the Control UI dropdown can render it, and open PR #43984 already targets the regression with closing syntax.

## What This Changes

The issue reports that the Control UI session dropdown only shows `heartbeat` and omits the canonical `main` session.

## Best Possible Solution

Review and land or replace #43984 with a gateway-level regression fix that keeps canonical agent main sessions visible in active `sessions.list` results while preserving active filtering for ordinary stale sessions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a stale `agent:main:main` row plus a fresh heartbeat row, followed by `sessions.list` with `activeMinutes > 0`; current main filters the stale main row before the dropdown can render it.

## Solution Assessment

Yes. Fixing the gateway active-list contract is the narrowest maintainable path because all consumers get the canonical main session consistently instead of requiring the Control UI to invent a missing row locally.
