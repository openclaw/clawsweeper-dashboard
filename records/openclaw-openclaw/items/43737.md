## Summary

Keep open: current main still applies the active-session recency filter before `sessions.list` rows and Control UI dropdown options are built, so a stale canonical main session can be omitted while heartbeat remains visible; the linked implementation candidate is still open at https://github.com/openclaw/openclaw/pull/43984.

## What This Changes

The issue reports that the Control UI session dropdown can show only `heartbeat` and omit the canonical `main` session.

## Best Possible Solution

Review, update, and land or replace https://github.com/openclaw/openclaw/pull/43984 so shared Gateway session-entry filtering preserves primary main sessions for sync and async `sessions.list` before the Control UI builds dropdown options.

## Reproduction Assessment

Yes, source-level. A stale `agent:main:main` row plus a fresh heartbeat row followed by `sessions.list` with positive `activeMinutes` will filter the main row before the Control UI receives dropdown rows.

## Solution Assessment

Yes. The narrow maintainable fix belongs in shared Gateway session-entry selection; a UI-only injected dropdown option would leave `sessions.list` semantically wrong for other callers.
