## Summary

Keep open: current main still applies the `activeMinutes` cutoff before `sessions.list` rows reach the Control UI, so a stale canonical main session can be omitted while a newer heartbeat session remains visible. The open linked implementation candidate has closing syntax for this issue but has not merged.

## What This Changes

The issue asks for the Control UI session dropdown to show the main session alongside heartbeat instead of hiding it from the available session list.

## Best Possible Solution

Preserve canonical main sessions in shared Gateway session-entry filtering for both sync and async `sessions.list`, then verify the Control UI dropdown shows main while stale non-main sessions remain hidden.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a stale `agent:main:main` row plus a fresher heartbeat row followed by `sessions.list` with positive `activeMinutes` is filtered before the Control UI receives dropdown rows.

## Solution Assessment

Yes. The narrow maintainable fix belongs in shared Gateway session-entry selection; a UI-only injected option would leave the public RPC behavior inconsistent.
