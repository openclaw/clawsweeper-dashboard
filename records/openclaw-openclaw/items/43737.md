## Summary

Keep open. Current main still filters stale `agent:main:main` out of active `sessions.list` results before the Control UI dropdown sees it, and open PR #43984 already carries the targeted fix with closing syntax.

## What This Changes

The issue asks for the Control UI session dropdown to keep the canonical main session visible instead of showing only heartbeat.

## Best Possible Solution

Land or replace #43984 with a gateway-level regression fix that preserves top-level canonical main sessions in active `sessions.list` while keeping stale non-main sessions filtered, with coverage for stale main plus active heartbeat.

## Reproduction Assessment

Yes. A high-confidence source-level path is a store with stale `agent:main:main` and fresh heartbeat, followed by `sessions.list` with `activeMinutes > 0` and dropdown rendering while heartbeat is current; current main filters main before the UI can add it.

## Solution Assessment

Yes. Fixing the gateway active-list behavior is the narrowest maintainable solution because it keeps the `sessions.list` contract consistent for all consumers instead of making the Control UI invent a missing main row locally.
