## Summary

Keep open: current main still filters `sessions.list` rows by `activeMinutes` before the Control UI sees them, so a stale canonical main session can disappear while a fresher heartbeat remains visible; https://github.com/openclaw/openclaw/pull/43984 is the open closing implementation candidate but is still unresolved.

## What This Changes

The issue reports that the Control UI session dropdown omits the canonical `main` session and shows only `heartbeat`.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/43984 so shared Gateway session-entry filtering preserves canonical main sessions for both sync and async `sessions.list` before the Control UI builds dropdown options.

## Reproduction Assessment

Yes, source-level. A stale `agent:main:main` row plus a fresher heartbeat row followed by `sessions.list` with positive `activeMinutes` is filtered before the Control UI receives dropdown rows.

## Solution Assessment

Yes. The narrow maintainable fix belongs in shared Gateway session-entry selection so both sync and async `sessions.list` preserve canonical main sessions; a UI-only injected option would leave the public RPC behavior inconsistent.
