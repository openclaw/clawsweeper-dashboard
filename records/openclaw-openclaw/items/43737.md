## Summary

Keep open: current main and the latest release still filter active session entries by `updatedAt` before `sessions.list` rows are built, so a stale canonical main session can be omitted while heartbeat remains visible. The linked implementation path is still open at https://github.com/openclaw/openclaw/pull/43984 and uses closing syntax for this issue.

## What This Changes

The issue reports that the Control UI chat session dropdown shows only heartbeat and omits the selectable main session.

## Best Possible Solution

Update, land, or replace https://github.com/openclaw/openclaw/pull/43984 so shared Gateway session-entry selection preserves configured primary main sessions for both sync and async `sessions.list` while keeping stale non-main sessions filtered.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale `agent:main:main` row plus a fresh heartbeat row followed by `sessions.list` with positive `activeMinutes`; current main filters the stale main row before the Control UI receives rows.

## Solution Assessment

Yes for the direction: the narrow maintainable fix belongs in shared Gateway session-entry selection. A UI-only injected dropdown option would leave `sessions.list` semantically wrong for other callers.
