## Summary

Keep open: current main still lets the active-session filter drop stale main-session rows before the Control UI dropdown can render them, and https://github.com/openclaw/openclaw/pull/43984 already owns the fix path.

## What This Changes

The issue reports that the Control UI chat session dropdown shows only `heartbeat` and omits the main session.

## Best Possible Solution

Fix the shared Gateway session-list entry filter so configured primary main sessions survive active-minute filtering, preserve custom `mainKey` semantics, and land or replace https://github.com/openclaw/openclaw/pull/43984.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale `agent:main:main` row plus a fresh `heartbeat` row, followed by `sessions.list` with a positive `activeMinutes`; current main filters out the stale main entry before the dropdown can render it.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared Gateway session-list entry filter; a UI-only injected option should be secondary unless the Gateway can still omit a real main session after that fix.
