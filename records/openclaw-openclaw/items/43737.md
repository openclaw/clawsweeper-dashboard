## Summary

Keep open: current main still filters stale main-session rows out of active `sessions.list` results before the Control UI dropdown can render them, and the open linked PR remains the implementation path.

## What This Changes

The issue reports that the Control UI session dropdown omits the main session and only shows heartbeat.

## Best Possible Solution

Fix the shared Gateway session-list entry filter so configured primary main sessions survive active-minute filtering, preserve custom `mainKey` semantics, and cover the RPC/UI dropdown path.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale `agent:main:main` row plus a fresh heartbeat row, followed by `sessions.list` with a positive `activeMinutes`; current main filters the stale main row before row construction.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared Gateway entry-selection filter; a UI-only injected dropdown option should be secondary unless the Gateway can still omit a real main session after that fix.
