## Summary

Keep open: current main still applies the active-session recency filter before `sessions.list` rows are built, so a stale canonical main session can be omitted from the Control UI dropdown, and https://github.com/openclaw/openclaw/pull/43984 remains the linked implementation path.

## What This Changes

The issue asks for the Control UI session dropdown to show the main session alongside heartbeat and other active sessions.

## Best Possible Solution

Review, update, and land or replace https://github.com/openclaw/openclaw/pull/43984 so the shared Gateway session-entry filter preserves configured primary main sessions for both sync and async `sessions.list`.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a stale `agent:main:main` row plus a fresh heartbeat row followed by `sessions.list` with positive `activeMinutes`; current main filters the stale main row before the Control UI receives rows.

## Solution Assessment

Yes. The narrow maintainable fix belongs in shared Gateway session-entry selection; injecting a UI-only dropdown option would leave `sessions.list` semantically wrong for other callers.
