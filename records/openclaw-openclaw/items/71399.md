## Summary

Keep open. Current main still lacks the requested ACP turn-completion internal hook, and this PR remains the focused implementation path, but it is not merge-ready because the proposed event drops the available failure code and uses the unbounded hook dispatch helper on a per-turn path.

## What This Changes

The PR adds an internal `agent:turn:end` hook from ACP turn completion, plus event type-guard coverage and ACP manager success/failure emission tests.

## Best Possible Solution

Land a narrow ACP manager turn-end hook that fires once per completed ACP turn, carries `sessionKey`, `success`, `durationMs`, and optional `errorCode`, uses bounded non-blocking dispatch, and updates tests plus changelog/docs where the hook is documented.

## Reproduction Assessment

Yes. Current main can be checked by exact symbol search and inspection of `AcpSessionManager.recordTurnCompletion()`; the PR behavior can be verified by registering an `agent:turn:end` listener and running successful and failing ACP turns.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the payload should preserve the existing ACP failure code and the dispatch should use `fireAndForgetBoundedHook(() => triggerInternalHook(...))` unless maintainers deliberately approve an exception.
