## Summary

Keep open. The PR is a focused and useful hook-surface implementation, but the final split-event API still has a blocking session-state contract gap: `agent:turn:save` is documented as the durable-readiness event while the direct ACP agent-command path persists transcripts after `runTurn()` without supplying the new save callback, so that path would never emit the save event.

## What This Changes

The PR adds documented `agent:turn:end` and `agent:turn:save` internal hook events, ACP manager plumbing, dispatch transcript ordering before the save hook, and focused tests for the new contexts.

## Best Possible Solution

Publish the split hook surface only after every ACP turn path either emits a truthful save outcome or the docs/API explicitly narrow the durable-readiness event to the covered dispatch path.

## Reproduction Assessment

## Solution Assessment

No, not yet. Splitting runtime completion from save outcome is the right shape, but the best fix must either cover every ACP transcript save path or narrow the public hook contract before this becomes a reliable API.
