## Summary

Keep open. Current main still lacks the requested ACP turn-completion internal hook, and this PR is the focused implementation attempt, but it is not merge-ready because the proposed payload drops the advertised failure code, uses unbounded hook dispatch on a per-turn path, and omits the required changelog entry for a new hook contract.

## What This Changes

This PR adds an `agent:turn:end` internal hook emitted from ACP turn completion, plus the hook context/type guard and focused ACP manager tests.

## Best Possible Solution

Land a narrow ACP-manager internal hook that fires once for each completed turn, preserves failure classification in the payload, uses bounded non-blocking dispatch, and has focused tests for success, failure, type guarding, and stalled-hook behavior expectations.

## Reproduction Assessment

Yes. Current main can be verified by symbol search plus inspection of `AcpSessionManager.recordTurnCompletion()`; the PR reproduction path is to register an internal `agent:turn:end` listener and run successful and failing ACP turns, which the proposed manager tests target.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the proposed contract should include optional `errorCode` on failures and dispatch via `fireAndForgetBoundedHook(() => triggerInternalHook(...))` or carry an explicit maintainer-approved exception.
