## Summary

Keep open: current main still lacks the requested hook and this PR is the active implementation, but the current docs overpromise hook timing relative to session state/cleanup even though the implementation uses bounded fire-and-forget dispatch.

## What This Changes

This PR adds an ACP `agent:turn:end` internal hook with success/failure payloads, bounded dispatch, transcript-before-hook support for ACP replies, tests, docs, and a changelog entry.

## Best Possible Solution

Align the docs and tests with bounded async hook scheduling, or change the implementation to provide the promised timing guarantee, then land after maintainer API/timing approval and exact-head CI.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. Current-main absence is high-confidence from source inspection of the stats-only ACP completion path and exact hook-symbol searches.

## Solution Assessment

No as written: emitting from `recordTurnCompletion()` with bounded dispatch is a narrow implementation, but the documented pre-state/pre-cleanup timing is stronger than the fire-and-forget queue can guarantee. The safer path is to document best-effort scheduling or make dispatch semantics match the promised ordering.
