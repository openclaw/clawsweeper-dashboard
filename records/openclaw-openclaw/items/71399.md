## Summary

Keep open. Current main still lacks the requested ACP turn-end internal hook, and the PR is the focused implementation path, but it remains merge-blocked by two contract/reliability issues: failed turns drop the available error code, and the new per-turn hook dispatch uses the unbounded helper even though bounded dispatch already exists.

## What This Changes

The PR adds an `agent:turn:end` internal hook emitted from ACP turn completion, adds the event context/type guard, and adds focused manager and hook tests for success and failure emission.

## Best Possible Solution

Land a narrow ACP-manager internal hook that fires once per completed ACP turn, carries `sessionKey`, `success`, `durationMs`, and optional failure `errorCode`, uses bounded non-blocking dispatch, and updates tests, changelog, and public hook documentation if the event is supported for hook consumers.

## Reproduction Assessment

Yes. Current main can be checked by exact symbol search plus inspection of `AcpSessionManager.recordTurnCompletion()`; the PR reproduction path is to register an `agent:turn:end` listener and run successful and failing ACP turns.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the payload should preserve the existing failure code and dispatch should use `fireAndForgetBoundedHook(() => triggerInternalHook(...))` or a documented maintainer-approved exception.
