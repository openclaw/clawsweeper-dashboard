## Summary

Keep open. Current main still lacks the requested `agent:turn:end` hook, and this PR is the focused implementation attempt, but it is not merge-ready because the new contract drops the advertised failure `errorCode`, uses unbounded per-turn hook dispatch, and lacks the required release note for a new hook surface. A plugin-author comment also confirms real demand for this hook in security-plugin work.

## What This Changes

The PR adds an ACP `agent:turn:end` internal hook emitted from `recordTurnCompletion()`, adds event context/type-guard coverage, and adds manager tests for success and failure emission.

## Best Possible Solution

Land a narrow ACP-manager turn-end hook that fires once per completed ACP turn, carries `sessionKey`, `success`, `durationMs`, and optional `errorCode`, uses bounded non-blocking dispatch, and updates tests plus changelog/docs as appropriate.

## Reproduction Assessment

Yes. Current main can be verified by exact symbol search and inspection of `AcpSessionManager.recordTurnCompletion()`; the PR path is to register an `agent:turn:end` listener and run successful and failing ACP turns.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the payload should preserve the existing ACP failure code and the dispatch should use `fireAndForgetBoundedHook(() => triggerInternalHook(...))` unless maintainers deliberately approve an exception.
