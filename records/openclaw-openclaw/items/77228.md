## Summary

Keep open: current main still has source-visible paths for the prefill rejection, provider-wide cooldown, and session repair corruption, while the three related fixes are still open PRs rather than merged main behavior.

## What This Changes

The issue asks OpenClaw to prevent a session-specific prefill/format rejection from poisoning provider-wide cooldown and to make transcript/session repair recover safely instead of preserving or worsening corruption.

## Best Possible Solution

Land the three focused repairs: exclude format rejections from shared auth-profile cooldown, drop trailing synthetic assistant error turns before provider replay, and make session repair drop structurally invalid message entries before closing this issue.
