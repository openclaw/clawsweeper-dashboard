## Summary

Keep open: current main has partial hardening for blank-user repair and shorter immutable cooldown windows, but the GitHub Copilot Claude replay policy still does not enable strict Anthropic turn validation, so a stale trailing assistant turn can still reach the provider as unsupported prefill.

## What This Changes

The issue reports a GitHub Copilot Claude/WhatsApp session getting stuck after a provider prefill-format error, followed by provider cooldown and unsafe transcript repair.

## Best Possible Solution

Make GitHub Copilot Claude replay use the same strict Anthropic turn validation and trailing-prefill stripping as native Anthropic while preserving Copilot-specific thinking-block handling and existing blank-user repair behavior.

## Reproduction Assessment

Yes at source level, but not by a live provider run: Copilot Claude supplies a narrow replay policy that bypasses generic Anthropic validation, and the stripping code is gated on validation flags that policy does not set.

## Solution Assessment

No, current main is not complete for the Copilot Claude path; the narrow maintainable fix is provider-owned replay policy coverage rather than a new cooldown knob or automatic session-reset UX.
