## Summary

Keep open: current main has some session-repair hardening, but GitHub Copilot Claude replay still misses the Anthropic turn-validation policy needed to strip trailing assistant prefill, and format failures can still flow into auth-profile cooldown handling.

## What This Changes

The issue reports that a malformed transcript ending in an assistant turn causes GitHub Copilot Claude prefill rejection, provider-wide cooldown, and failed session repair recovery.

## Best Possible Solution

Apply Anthropic-class replay hygiene to GitHub Copilot Claude models and land a cooldown-policy fix so request-format errors do not mark shared provider auth as unhealthy.

## Reproduction Assessment

Yes, at source level: GitHub Copilot Claude resolves to `anthropic-messages`, its plugin replay policy does not enable Anthropic turn validation, and the only trailing-assistant prefill strip is gated on that flag. The cooldown cascade is also source-visible because `format` is not filtered before auth-profile failure marking.

## Solution Assessment

No, current main is not the best end state. The maintainable fix is to reuse the existing strict Anthropic replay policy for GitHub Copilot Claude and pair it with the already-proposed cooldown-policy correction in #74909.
