## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Scope progress toolProgress to progress mode:** `src/plugin-sdk/channel-streaming.ts:159-160`
  - body: `resolveChannelStreamingPreviewToolProgress` always prefers `streaming.progress.toolProgress`, even when the resolved mode is `partial` or `block`. A user switching away from progress mode can leave `progress.toolProgress: false` behind and unexpectedly disable normal preview tool lines despite `preview.toolProgress: true`; only consult the progress knob when mode is `progress`.
  - confidence: 0.86
- **[P2] Scope Matrix progress toolProgress to progress mode:** `extensions/matrix/src/matrix/monitor/index.ts:101`
  - body: Matrix has the same precedence leak in its local resolver: `progress.toolProgress` overrides `preview.toolProgress` for `partial` and `quiet` modes. That can both hide expected draft tool lines and change whether standalone progress messages are suppressed outside progress mode.
  - confidence: 0.84
- **[P2] Start Slack progress drafts when tool lines are hidden:** `extensions/slack/src/monitor/message-handler/dispatch.ts:1018-1020`
  - body: In Slack progress mode with `progress.toolProgress: false`, this suppresses standalone progress messages but the reply options still never start a label-only progress draft on reply start. A tool-heavy turn with no early partial can stay silent until final delivery, which breaks the one visible progress-draft contract.
  - confidence: 0.84
- **[P2] Suppress standalone progress in Teams progress mode:** `extensions/msteams/src/reply-dispatcher.ts:340-347`
  - body: Teams progress mode wires native streaming callbacks but does not set `suppressDefaultToolProgressMessages`, so shared dispatch can still emit tool, plan, or approval progress as regular messages alongside the Teams progress stream. That breaks the single progress stream behavior this mode promises.
  - confidence: 0.82
