## Summary

Keep open: this is an active maintainer-labeled multi-channel feature PR, and source review found a blocking Slack progress-draft gap where standalone progress is suppressed without starting the label-only draft.

## What This Changes

The branch adds unified progress-draft streaming config and helpers, implements channel behavior across Discord, Telegram, Matrix, Slack, and Microsoft Teams, and updates docs, generated config metadata, tests, and the changelog.

## Best Possible Solution

Keep the shared SDK/config direction, add Slack label-only progress draft startup coverage, then rerun the targeted channel checks before maintainer review.

## Reproduction Assessment

Yes for the review finding: source inspection shows Slack suppresses standalone progress while only updating drafts from progress or partial callbacks. A focused Slack dispatch test with `streaming.mode: "progress"` and `progress.toolProgress: false` should reproduce the silent-during-tools path.

## Solution Assessment

No, not yet. The shared design is coherent, but Slack needs the same reply-start label draft behavior as Discord, Telegram, and Matrix before the progress-mode implementation is complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Start Slack progress drafts before suppressing progress:** `extensions/slack/src/monitor/message-handler/dispatch.ts:1017-1019`
  - body: In progress mode with `progress.toolProgress: false`, this suppresses standalone tool progress, but Slack never starts the label-only draft on reply start; draft updates only come from tool/progress callbacks or answer partials. A tool-heavy turn with no early answer partial becomes silent until final delivery, unlike the new progress contract and the other channel implementations. Wrap the existing `replyOptions.onReplyStart` to create the progress draft before suppressing standalone progress.
