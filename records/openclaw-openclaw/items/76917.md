## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Scope progress toolProgress to progress mode:** `src/plugin-sdk/channel-streaming.ts:159-160`
  - body: `resolveChannelStreamingPreviewToolProgress` now reads `streaming.progress.toolProgress` before `streaming.preview.toolProgress` for every mode. With `streaming.mode: "partial"`, `preview.toolProgress: true`, and leftover `progress.toolProgress: false`, preview tool lines are disabled even though progress mode is not active. Only consult `progress.toolProgress` when the resolved preview mode is `progress`; otherwise preserve the preview config.
  - confidence: 0.86
- **[P2] Start Slack progress drafts before suppressing progress:** `extensions/slack/src/monitor/message-handler/dispatch.ts:1017-1019`
  - body: In Slack progress mode with `progress.toolProgress: false`, this suppresses standalone tool progress but never starts the label-only progress draft on reply start; updates only come from progress or partial callbacks. A tool-heavy turn with no early partial can be silent until final delivery, unlike the new progress-draft contract. Add an `onReplyStart` wrapper that creates the progress draft before suppressing standalone progress.
  - confidence: 0.84
