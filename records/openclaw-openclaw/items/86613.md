## Summary

Keep open: current main still enables memory sync watching by default and still constructs long-lived chokidar watchers for memory roots, while the report adds credible lsof/heap evidence of gateway FD accumulation that is not implemented or superseded. The remaining fix is compatibility-sensitive because changing gateway memory-watch defaults can affect near-real-time memory indexing.

## What This Changes

The issue asks OpenClaw to stop long-lived gateway processes from accumulating thousands of read-only file descriptors on workspace `memory/**/*.md` files during memory search/watch activity.

## Best Possible Solution

Decide the gateway memory-watch default in the related product/default issue, then land a plugin-owned fix that bounds watcher/FD growth while preserving explicit opt-in watching plus documented manual or periodic reindex paths.

## Reproduction Assessment

No high-confidence deterministic reproduction was established in this review. Source inspection confirms the active default-watch and watcher construction paths on current main, and the issue body provides strong live lsof/heap evidence, but the exact FD leak remains workload- and uptime-correlated.

## Solution Assessment

Unclear as a final implementation path: disabling or changing gateway memory watching may be the right fix, but it is compatibility-sensitive and should be decided with the related gateway-default watcher discussion rather than patched automatically from this report alone.
