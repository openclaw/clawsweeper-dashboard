## Summary

Keep open: current main still enables memory file watching by default and still constructs chokidar watchers for memory roots, while this report adds concrete high-FD heap/lsof evidence that is not fully covered by the broader gateway-default discussion.

## What This Changes

The issue reports gateway accumulation of thousands of read-only file descriptors on workspace `memory/**/*.md` during memory search activity and asks for the gateway to release read handles after reads complete.

## Best Possible Solution

Use this issue as high-FD evidence for the memory watcher subsystem, then decide whether to bound chokidar usage, change the gateway-mode default, or add a safe fallback without breaking explicit freshness setups.

## Reproduction Assessment

Unclear for a full live repro: the issue lacks a minimal current-main reproduction, but source inspection confirms the unset-config path still enables chokidar memory watchers that match the reported heap evidence.

## Solution Assessment

Unclear: disabling memory watching in gateway mode is plausible and already tracked, but the 12K-FD captures may require a lower-level watcher design or lifecycle fix for users who explicitly keep watch enabled.
