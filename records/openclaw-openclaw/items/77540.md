## Summary

Keep open: current main still lacks this PR's node pairing and cron/list-result cache work, and the contributor supplied sufficient terminal proof, but the branch needs focused repair before merge for cache freshness and current-main session-list behavior.

## What This Changes

The PR adds short-lived in-memory caching/coalescing for Gateway `sessions.list` results, node pairing list reads, cron delivery previews, and manifest model-id normalization lookups, with related invalidation tests.

## Best Possible Solution

Repair and land the scoped caches only after cron previews are config-sensitive and the sessions-list cache is rebased onto current diagnostics and configured-agent store targeting.

## Reproduction Assessment

Yes for source-level behavior: current main still performs direct request-time session, node, and cron preview reads, while PR head adds caches. The remaining cron stale-preview bug is source-reproducible because the cached resolver uses `cfg` without including config state in the cache key.

## Solution Assessment

Mostly yes: short TTL caches with explicit invalidation are a narrow performance fix for repeated Gateway list calls. The safer version must include runtime config state in cron preview cache keys and preserve current main's configured-agent session store loading behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve configured-only store targeting:** `src/gateway/server-methods/sessions.ts:699-701`
  - body: Current main passes `configuredAgentsOnly` into `loadCombinedSessionStoreForGateway` so Control UI polling can avoid loading disk-only or unconfigured agent stores before filtering. This branch wraps result caching around the older call shape, so a repaired merge that keeps this call would reload broad store targets when `configuredAgentsOnly=true` and can reintroduce the latency this PR is meant to reduce; pass the flag through while preserving the current diagnostics spans.
  - confidence: 0.78
- **[P3] Include config state in cron preview cache keys:** `src/cron/delivery-preview.ts:128-131`
  - body: The new cron preview cache key is derived only from `defaultAgentId` and job fields, but preview resolution passes `params.cfg` into `resolveDeliveryTarget` for session store paths, channel selection, account bindings, allowlists, and target resolution. After a config or routing change that leaves the cron job list unchanged, `cron.list` can return the previous preview for the TTL; include a runtime config cache key or clear this cache on runtime config writes.
  - confidence: 0.86
