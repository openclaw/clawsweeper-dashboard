## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Include offset in the sessions-list cache key:** `src/gateway/sessions-list-result-cache.ts:40-54`
  - body: `SessionsListParams` accepts `offset` and current row selection slices by it, but the new cache key does not include it. A `{limit: 2}` request can populate the cache and a later `{limit: 2, offset: 2}` request can reuse the first page instead of returning the next page.
  - confidence: 0.95
- **[P2] Do not cache activeMinutes results for a fixed minute:** `src/gateway/sessions-list-result-cache.ts:44`
  - body: `activeMinutes` results are based on `Date.now()`, but the PR stores them for the full 60 second TTL. Rows can age out of the requested active window without any mutation event, so identical requests can keep showing stale active sessions.
  - confidence: 0.88
- **[P1] Preserve caller-scoped pending node diagnostics:** `src/gateway/server-methods/nodes.ts:897-903`
  - body: Current `node.list` and `node.describe` include pending pairing diagnostics only for authorized callers. The cached path stops passing `client` and `nodePairing.pending` through that projection, so authorized diagnostics can disappear.
  - confidence: 0.94
- **[P2] Guard cron preview fills across cache clears:** `src/cron/delivery-preview.ts:154-159`
  - body: `clearCronDeliveryPreviewsCache()` deletes the maps, but an already-running preview promise still writes its stale result after a cron mutation. Use a generation guard like the node/session caches or avoid caching results from invalidated fills.
  - confidence: 0.9
- **[P2] Keep manifest policy caching lifecycle-based:** `src/plugins/manifest-model-id-normalization.ts:75-85`
  - body: This changes cacheable manifest model-id normalization from config-fingerprint reuse to a 60 second TTL. Gateway plugin metadata is process-stable and lifecycle-owned, so the TTL reintroduces periodic hot-path policy collection instead of preserving the current snapshot contract.
  - confidence: 0.84
- **[P2] Remove the release-owned changelog edit:** `CHANGELOG.md:498`
  - body: `CHANGELOG.md` is generated for releases, but this branch inserts a normal PR note into an old release section. Keep the release-note context in the PR body or squash message and leave the changelog to release generation.
  - confidence: 0.93
