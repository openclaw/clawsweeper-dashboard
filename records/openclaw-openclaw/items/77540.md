## Summary

Keep open. The updated PR addresses the earlier stale node in-flight cache finding and has real behavior proof plus green checks; the remaining issues are a low-priority cron preview cache-key freshness gap and maintainer-owned changelog placement cleanup.

## What This Changes

The PR adds short-lived caches for Gateway session list results, node pairing list reads, cron delivery previews, and manifest model normalization policy lookups, with invalidation hooks and focused tests.

## Best Possible Solution

Keep the scoped caches, add config-sensitive cron preview cache invalidation or keying, move the changelog entry to the active release section, then merge after the normal Gateway changed-file gates remain green.

## Reproduction Assessment

Yes for source-level behavior: current main reads the affected session, node, and cron list data directly, while PR head adds caches and tests the invalidation paths. The remaining cron freshness gap is also source-reproducible because the cache key omits cfg while the resolver reads cfg-dependent routing state.

## Solution Assessment

Mostly yes. Short TTL caches with mutation invalidation are a narrow performance fix, but cron preview caching should carry config-sensitive state the same way the session-list cache already does.
