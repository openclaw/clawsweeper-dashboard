## Summary

Keep open: this is useful Gateway performance work, but the current PR is not merge-ready. Source review found stale `sessions.list activeMinutes` caching, a rebase regression in configured-agent session-store targeting, and a cron preview cache key that omits config-sensitive routing state; real behavior proof is sufficient and the security pass is cleared.

## What This Changes

This PR adds short-lived in-process caching/coalescing for Gateway `sessions.list`, node pairing list reads, cron delivery previews, and manifest model normalization policy lookup, with invalidation tests and changelog text.

## Best Possible Solution

Repair the cache correctness issues on current main, preserve the current sessions-list diagnostics and configured-agent store targeting, then land the scoped caches after focused Gateway/cron tests and a build/check gate.

## Reproduction Assessment

Yes for source-level review bugs: `activeMinutes` is time-relative, current main has configured-only store targeting that the PR bypasses, and cron preview resolution reads config state not present in the new cache key. I did not run a live timing repro under the read-only review constraint.

## Solution Assessment

Mostly yes: short TTL caches with explicit invalidation are a narrow maintainable mitigation for repeated Gateway list work. The current implementation needs time-aware sessions-list caching, config-aware cron preview keys, and a rebase onto current sessions-list instrumentation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Bypass the cache for activeMinutes queries:** `src/gateway/sessions-list-result-cache.ts:44`
  - body: `activeMinutes` results are time-relative because `listSessionsFromStoreAsync` computes the cutoff from `Date.now()`. Caching the same `activeMinutes` request for the full 60 second TTL lets sessions remain visible after they age out; bypass this cache for `activeMinutes` or make the key/TTL time-aware.
  - confidence: 0.9
- **[P2] Preserve configured-only store targeting:** `src/gateway/server-methods/sessions.ts:699-701`
  - body: Current main passes `configuredAgentsOnly` into `loadCombinedSessionStoreForGateway` so configured-only Control UI lists avoid loading disk-only or unconfigured agent stores before filtering. This branch loads the combined store with only `agentId` before the cache wrapper, so cache misses can reintroduce the broad store scan this perf work is meant to reduce; pass the flag through while keeping the current diagnostics spans.
  - confidence: 0.86
- **[P3] Include config state in cron preview cache keys:** `src/cron/delivery-preview.ts:128-130`
  - body: The new cron preview cache key is derived only from `defaultAgentId` and job fields, but preview resolution uses `params.cfg` for session store paths, channel selection, account bindings, allowlists, and outbound target resolution. After a config or routing change with unchanged cron jobs, `cron.list` can return the previous preview until the TTL expires; include a runtime config cache key or clear this cache on config writes.
  - confidence: 0.86
