- **[P1] Include offset in the sessions-list cache key:** `src/gateway/sessions-list-result-cache.ts:45`
  - body: Current main exposes and applies `sessions.list` offset, but this cache key varies by limit and filters only. Two requests with the same filters and different offsets can share the first cached page, so include `offset` in the key and cover it with pagination regression coverage.
  - confidence: 0.94
- **[P2] Bypass or time-bound activeMinutes cache hits:** `src/gateway/sessions-list-result-cache.ts:44`
  - body: `activeMinutes` results depend on `Date.now()`, but the PR caches identical requests for up to 60 seconds. A row can age out of the active window while the cached response still returns it, so bypass the result cache for `activeMinutes` or make that key/TTL time-aware.
  - confidence: 0.88
- **[P1] Preserve caller-scoped pending node projection:** `src/gateway/server-methods/nodes.ts:898-903`
  - body: Current `node.list`/`node.describe` include pending pairing rows and then reveal or redact fields based on caller scopes. The cached path builds the catalog without `pendingNodes` or the caller projection, so authorized callers lose pending approval visibility and the response contract diverges.
  - confidence: 0.92
- **[P2] Keep manifest policy caching lifecycle-based:** `src/plugins/manifest-model-id-normalization.ts:79`
  - body: Plugin metadata snapshots are process-stable until an explicit lifecycle event, but this adds a 60-second expiry to manifest model-id policy caching. Keep reuse fingerprint/lifecycle-based so hot-path normalization does not periodically recollect plugin policy.
  - confidence: 0.86
