## Summary

Keep open. The PR contains useful Gateway performance work that is not on current main, and it has terminal proof plus green relevant checks, but it still needs a small repair for config-sensitive cron preview caching and current-main rebase cleanup before merge.

## What This Changes

The PR adds short-lived caches and invalidation for Gateway `sessions.list`, node pairing list/describe reads, manifest model-id normalization lookups, and cron delivery preview resolution.

## Best Possible Solution

Keep the scoped caches, make the cron preview cache key config-sensitive, rebase onto current main without losing diagnostics/configured-agent behavior, fix changelog placement, and rerun the Gateway changed-file gates.

## Reproduction Assessment

Yes for source-level behavior: current main still performs direct session, node, and cron list reads, while PR head adds caches and invalidation. The remaining cron freshness gap is also source-reproducible because the cache key omits `cfg` while the resolver reads config-dependent routing state.

## Solution Assessment

Mostly yes: short TTL caches with mutation invalidation are a narrow performance fix. The cron preview cache should carry runtime config state, matching the safer pattern used by the new sessions-list cache.
