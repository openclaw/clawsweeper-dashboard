## Summary

Keep this PR open: the changelog gap from the prior review is fixed, but the latest diff still lets an older node pairing list read repopulate the new cache after a pairing mutation invalidates it.

## What This Changes

The PR adds short-lived caching and in-flight coalescing for Gateway session list/model normalization, node list/describe pairing reads, and cron delivery previews, with invalidation hooks, tests, and changelog coverage.

## Best Possible Solution

Keep the scoped caches, add a node pairing cache generation/token guard with regression coverage, and then merge through the normal Gateway changed-file gates.

## Reproduction Assessment

Yes. Source inspection of PR head shows an in-flight `loadNodePairingListsCached()` read can resolve after `clearNodePairingListCache()` and store a pre-mutation pairing snapshot for later `node.list` or `node.describe` calls.

## Solution Assessment

No. The short-TTL cache approach is narrow, but the node pairing cache needs the same generation guard pattern already used by the new sessions-list cache before this is maintainable.
