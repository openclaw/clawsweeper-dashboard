## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Include offset in the sessions-list cache key:** `src/gateway/sessions-list-result-cache.ts:40-54`
  - body: `SessionsListParams` accepts `offset` and `listSessionsFromStoreAsync` slices by it, but the new cache key does not include it. A first request for `{limit: 2}` can populate the cache and a later `{limit: 2, offset: 2}` request will reuse the first page instead of returning the next page.
  - confidence: 0.95
- **[P2] Bypass or time-bound activeMinutes cache hits:** `src/gateway/sessions-list-result-cache.ts:44`
  - body: The cache stores `activeMinutes` results for up to 60 seconds even though the cutoff is based on `Date.now()`. Rows can age past the requested active window without any mutation event, so repeated calls can show sessions that should have dropped out of the result.
  - confidence: 0.87
- **[P1] Preserve caller-scoped pending node projection:** `src/gateway/server-methods/nodes.ts:897-904`
  - body: The cached node path builds the node catalog without `nodePairing.pending` and the handler no longer receives `client`, bypassing `listNodesForClient` and `canReadPendingNodePairing`. Authorized local backend callers will stop seeing pending approval and reapproval diagnostics that current tests require.
  - confidence: 0.94
- **[P2] Keep manifest policy caching lifecycle-based:** `src/plugins/manifest-model-id-normalization.ts:75-80`
  - body: This changes cacheable manifest model-id normalization from config-fingerprint reuse to a 60-second TTL. Plugin metadata is process-stable in Gateway hot paths, so the TTL reintroduces periodic metadata policy collection instead of relying on the current snapshot lifecycle.
  - confidence: 0.78
