## Summary

Keep open: current main does not implement `safetyPosture`, and the PR head still has security-sensitive correctness gaps around secure DM mode, strict memory behavior, and advertised tool profiles. It also needs real behavior proof before merge.

## What This Changes

The PR adds `SafetyPostureConfig` types/schema, generated schema metadata, runtime preset expansion, focused tests, and several extension test-helper import updates.

## Best Possible Solution

Land an additive safety-posture contract only after each preset expands through current runtime materialization into enforceable sandbox, session, tool, and memory settings, or narrow the public semantics to only what is actually enforced.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows `secureDmMode` is ignored without a preset, strict memory mapping is skipped, and `agentProfile` only writes an unconsumed marker. No live run was performed because this was a read-only review.

## Solution Assessment

No. The feature direction matches the optional hardening follow-up invited in https://github.com/openclaw/openclaw/issues/7827, but the current implementation exposes security claims before tool, memory, and secure DM behavior are enforced.

  - confidence: 0.9

  - body: The strict preset advertises disabled long-term memory and the preset constants encode that value, but runtime expansion explicitly skips memory mapping. Operators can select `strict` and still keep existing memory indexing or recall behavior.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR exposes security posture presets whose advertised tool, memory, and secure-DM restrictions are not fully enforced.

Concerns:

- **[medium] Security profile is not enforced:** `src/config/apply-safety-posture.ts:63`
  - body: `limited` and `public` are exposed as safety-posture tool profiles, but runtime expansion only stores `_safetyPostureResolvedProfile`; no current tool-policy path consumes it to deny exec, browser, web, gateway, node, cron, or session tools.
  - confidence: 0.93
- **[medium] Strict posture leaves memory enabled:** `src/config/apply-safety-posture.ts:57`
  - body: `MEMORY_PRESETS.strict.longTermMemoryEnabled` is false, but the implementation skips memory mapping, so a strict or public-facing posture can still retain existing memory indexing and recall behavior.
  - confidence: 0.86
- **[medium] Secure DM mode can be silently ignored:** `src/config/apply-safety-posture.ts:31`
  - body: A config that sets only `safetyPosture.secureDmMode: true` returns before applying `session.dmScope`, so the standalone secure-DM control does not enforce its documented isolation behavior.
  - confidence: 0.9
