Full review comments:

- **[P2] Honor secureDmMode without requiring a preset:** `src/config/apply-safety-posture.ts:31-33`
  - body: `secureDmMode` is accepted as an independent boolean, but this early return means `safetyPosture: { secureDmMode: true }` leaves the config unchanged instead of setting `session.dmScope` to `per-channel-peer`.   - confidence: 0.93
- **[P2] Wire posture profiles into real tool policy:** `src/config/apply-safety-posture.ts:63-68`
  - body: This stores `limited` or `public` in `_safetyPostureResolvedProfile`, but current tool policy reads `tools.profile` and only recognizes `minimal`, `coding`, `messaging`, and `full`. A strict/public posture can validate while leaving the advertised high-risk tool denylist unenforced.
  - confidence: 0.94
- **[P2] Apply or remove the strict memory preset:** `src/config/apply-safety-posture.ts:38`
  - body: `memoryPreset` is assigned but never read, which fails the repo's `noUnusedLocals` settings and leaves `MEMORY_PRESETS.strict.longTermMemoryEnabled: false` disconnected from runtime memory behavior. Either map it to the existing memory controls or remove the exposed preset promise.
  - confidence: 0.91
