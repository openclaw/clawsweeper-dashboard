## Summary

Keep this PR open, but do not merge it as-is. Current main still has no safetyPosture config surface, and the linked hardening request explicitly invited focused follow-up PRs, but this patch exposes security-posture promises before the tool-profile, memory, and standalone secure-DM behavior are actually enforced.

## What This Changes

The PR adds a top-level safetyPosture config with development/balanced/strict presets, runtime expansion into sandbox/session defaults, generated schema metadata, focused tests, and shared extension test-helper import updates.

## Best Possible Solution

Rebase the feature onto materializeRuntimeConfig and land only an additive preset contract whose sandbox, session, tool-policy, and memory promises are either enforced through current runtime paths or removed from the public schema/help text.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows secureDmMode without a preset returns unchanged, strict memory mapping is skipped, and agentProfile writes an unconsumed marker. No live run was performed because this was a read-only review.

## Solution Assessment

No. The optional hardening direction matches the follow-up path invited in the linked discussion, but the current implementation exposes security claims before the runtime enforcement semantics are complete.
