## What This Changes

The PR adds a top-level safetyPosture config with development/balanced/strict presets, schema/types, runtime expansion, generated schema metadata, tests, and test-helper import updates.

## Best Possible Solution

Land an additive preset contract only after it maps to real runtime enforcement for sandbox network, session isolation, tool policy, and memory behavior, with schema/docs/tests aligned to that contract.

Yes for the PR-introduced defects by source inspection: the latest head shows secureDmMode can be bypassed, sandbox network and strict memory preset values are not applied, and limited/public tool profiles are not consumed by current tool policy. I did not run tests or live config because this was a read-only review.

## Solution Assessment

No. The hardening direction matches the closed product discussion, but this implementation exposes security posture promises before the enforcement semantics are wired through the current runtime config and tool-policy contracts.
