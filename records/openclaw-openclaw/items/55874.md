## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Honor secureDmMode without requiring a preset:** `src/config/apply-safety-posture.ts:31-33`
  - body: `secureDmMode` is accepted by the schema as an independent boolean, but this early return means `safetyPosture: { secureDmMode: true }` leaves the config unchanged instead of setting `session.dmScope` to `per-channel-peer`. That makes the documented isolation control silently ineffective unless a preset is also present.
  - confidence: 0.93
- **[P2] Wire posture profiles into real tool policy:** `src/config/apply-safety-posture.ts:63-68`
  - body: This stores `limited` or `public` in `_safetyPostureResolvedProfile`, but current tool policy reads `tools.profile` and only recognizes `minimal`, `coding`, `messaging`, and `full`. A strict/public posture can validate while leaving the advertised high-risk tool denylist unenforced.
  - confidence: 0.94
- **[P2] Use the memory preset before exposing it:** `src/config/apply-safety-posture.ts:38`
  - body: `memoryPreset` is assigned but never read, which fails the repo's `noUnusedLocals` TypeScript settings. It also means the strict preset's `longTermMemoryEnabled: false` promise is not applied to runtime memory behavior.
  - confidence: 0.91
