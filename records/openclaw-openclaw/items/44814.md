## What This Changes

The PR adds `gateway.webui.attachmentMaxBytes` to Gateway config types/schema and uses it for Web UI `chat.send` and `agent.request` attachment parsing.

## Best Possible Solution

Close this stale PR and keep the remaining upload-limit design on #71142, using the existing shared resolver and maintained Control UI config namespace.

## Reproduction Assessment

Yes. Source inspection on current main plus the PR diff shows the old 5 MB call sites now use `resolveChatAttachmentMaxBytes(cfg)`, while this branch would replace the `chat.send` path with `gateway.webui?.attachmentMaxBytes ?? 5_000_000` and ignore the maintained shared setting.

## Solution Assessment

No. The branch was reasonable against the older hard-coded call sites, but current main has centralized parser-limit behavior and the remaining upload policy belongs in #71142 rather than a new `gateway.webui` namespace.
