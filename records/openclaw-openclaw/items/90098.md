## What This Changes

Make large Control UI attachment parsing and validation stack-safe and remove persisted queue entries that repeatedly fail with stack overflow.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, conditionally. Replacing the two full-string data-URL/base64 regexes with index/slice and iterative validation, while pruning only queue failures with the exact historical error, is the narrowest maintainable repair; live gateway proof must confirm it addresses the runtime trigger.
