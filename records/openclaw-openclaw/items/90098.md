## Summary

Keep open: current main and the latest release still contain the stack-prone gateway and Control UI attachment parsing paths, and the open closing PR at https://github.com/openclaw/openclaw/pull/92223 is not merged and is currently conflicting.

## What This Changes

Requests stack-safe Control UI and gateway attachment data-URL/base64 handling for large uploads, plus cleanup for stale queued items failed by the old stack overflow.

## Best Possible Solution

Review, split if needed, and land https://github.com/openclaw/openclaw/pull/92223 or a narrowed equivalent that replaces full-string parser paths and drops stale failed queue items; close this issue only after that reaches main.

## Reproduction Assessment

No, not as a fresh live current-main repro in this read-only pass. Source inspection and duplicate production logs identify the implicated paths, but the exact payload/runtime trigger still needs live upload proof.

## Solution Assessment

Yes for the issue direction: replacing full-string regex parsing/validation with index/slice or iterative helpers and dropping old failed queue items is the narrow maintainable fix. The open PR should be reviewed for conflicts and whether its broader media-limit changes need splitting.
