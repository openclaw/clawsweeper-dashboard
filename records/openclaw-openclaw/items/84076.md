## Summary

Keep open: current main still has the fail-fast Codex app-server timeout path, but the requested user-visible recovery/status and replay-safe retry behavior is not implemented or superseded. Related PRs are still open and blocked on safety/proof, and the latest comments report the symptom can still occur.

## What This Changes

The issue reports that Codex app-server turns can abort after a final `item/completed` without a following `turn/completed`, leaving user-facing chat lanes without useful recovery or status.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers land a bounded recovery path: visible status for missing `turn/completed`, lane release, preserved diagnostics, and retry only for replay-safe pre-side-effect attempts.

## Reproduction Assessment

Yes, at source level: current main re-arms the completion-idle timeout after a final non-assistant `item/completed`, aborts with `turn_completion_idle_timeout`, and timed-out attempts bypass incomplete-turn warning. I did not run the live Discord or Telegram repro in this read-only review, but the issue and follow-up comments provide matching live logs.

## Solution Assessment

Unclear as a full fix: emitting an accurate visible timeout/status is a direct bug fix, but retry or resume after completed tools needs maintainer judgment around replay safety and side effects. The open related PRs cover parts of that direction but are not merged or cleared.
