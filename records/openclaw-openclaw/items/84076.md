## Summary

Keep open: current main still contains the fail-fast Codex app-server completion-idle timeout, but the remaining user-visible recovery/status and replay-safe retry behavior is not landed. ## What This Changes

The issue asks OpenClaw to recover visibly and preserve retry/resume state when Codex app-server times out after a completed current-turn item without `turn/completed`.

## Best Possible Solution

Land a bounded recovery path that releases the lane, surfaces accurate channel-visible Codex timeout status, preserves replay metadata, and retries or resumes only for pre-output or replay-safe cases proven with redacted live Discord/Telegram logs.

## Reproduction Assessment

Yes. ## Solution Assessment

Unclear as a full fix. Visible timeout/status delivery is a narrow bug fix, but automatic retry or resume after completed tool work needs a maintainer replay-safety decision; the active recovery PR is not yet merge-ready.
