## Summary

Keep open: current main still treats the missing Codex `turn/completed` after a completed item as a timeout/abort path, and the requested structured recovery/status/resume behavior is not implemented or superseded.

## What This Changes

The issue asks OpenClaw to recover visibly and safely when Codex app-server emits `item/completed` but never follows with `turn/completed`.

## Best Possible Solution

Add a bounded Codex app-server completion-idle recovery path that releases the lane, surfaces an accurate user-visible timeout/status, records replay metadata, and only offers or performs retry when side-effect safety is known.

## Reproduction Assessment

Yes, source inspection plus live reporter/comment logs give a high-confidence reproduction shape: Codex app-server emits `item/completed`, omits `turn/completed`, and OpenClaw aborts via `turn_completion_idle_timeout`. I did not run the live provider/channel repro in this read-only review.

## Solution Assessment

Unclear as a full implementation: visible timeout/status is a narrow bug fix, but automatic retry/resume after completed tools needs a maintainer decision around replay safety and side effects.
