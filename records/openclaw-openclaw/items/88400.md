## What This Changes

The PR adds 16 bundled provider and provider-alias IDs to the model provider overlay allowlist and table-tests timeout-only overlay configs for those IDs.

## Best Possible Solution

Land the focused allowlist parity update after normal checks; a generated manifest-derived parity check can remain a separate hardening follow-up.

## Reproduction Assessment

Yes. Source inspection shows current main rejects timeout-only overlays unless the provider ID is in the built-in overlay allowlist, and the affected aliases are declared in bundled provider metadata.

## Solution Assessment

Yes. For the current static schema design, adding the missing bundled IDs with focused table coverage is the narrowest maintainable repair; deriving the list from manifests would be a broader follow-up.
