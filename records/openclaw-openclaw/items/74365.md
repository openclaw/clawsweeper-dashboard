## Summary

Keep open. Current main has shared CLI progress primitives and coverage for some command families, but the broad request is still only partially addressed: ClawHub skill search/install/update and plugin search/install/update paths still perform potentially slow work without the shared progress wrapper, and the narrower doctor-update spinner work is still an open PR rather than shipped behavior.

## What This Changes

Request adding loading/progress indicators to slow OpenClaw CLI commands so users can tell whether work is progressing.

## Best Possible Solution

Scope the slow CLI command families first, then add the existing progress helper at selected command boundaries with TTY, non-TTY, JSON, and plain-output coverage; handle the doctor-update PR separately.
