## Summary

Keep open. Current main has a shared CLI progress primitive and partial coverage for startup and some gateway/device RPC calls, but the broad request is not fully implemented because ClawHub skill search/install/update and plugin search/install/update still call network/download/update helpers without the shared progress wrapper.

## What This Changes

Request to add loading or progress indicators to slow OpenClaw CLI commands so users can distinguish long-running work from a stuck command.

## Best Possible Solution

Use the existing ProgressReporter/withProgress contract for a scoped set of slow command families, with tests preserving TTY, non-TTY, JSON, and plain output behavior.

## Reproduction Assessment

No high-confidence runtime reproduction is available because the issue lacks exact commands and timings. Source inspection does verify plausible current-main gaps in ClawHub skills and plugin lifecycle commands.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable solution is a scoped rollout at selected slow command boundaries rather than blanket CLI output changes.
