## Summary

Keep open. Current main has a shared CLI progress primitive and some slow paths already use it, but the request is still only partially covered: Skills ClawHub search/install/update and plugin install/update still invoke long-running helpers directly or through loggers rather than a consistent progress wrapper.

## What This Changes

Add clearer loading/progress indicators for slow OpenClaw CLI commands so users can tell work is still progressing.

## Best Possible Solution

Scope the rollout around the existing withProgress/ProgressReporter contract for selected slow command families, preserving TTY, non-TTY, --json, and --plain output semantics.

## Reproduction Assessment

No high-confidence runtime reproduction is available because the report lacks exact commands and timings. Source inspection does verify plausible current-main coverage gaps in Skills and Plugins CLI flows.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable fix is a scoped rollout at selected command boundaries rather than blanket output changes across the CLI.
