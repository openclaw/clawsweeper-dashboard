## Summary

Keep open. Current main has a shared CLI progress primitive and partial coverage, but the broad loader request is still not fully implemented; ClawHub skill and plugin search/install/update paths still perform network, archive, or update work without the shared progress wrapper.

## What This Changes

Request to add visible loading/progress indicators to slow OpenClaw CLI commands so users can tell commands are still progressing.

## Best Possible Solution

Use the existing ProgressReporter/withProgress contract for a maintainer-scoped set of slow CLI command families while preserving parseable output modes.

## Reproduction Assessment

No. The issue lacks exact commands and timings, but source inspection of current main verifies plausible remaining gaps in ClawHub skill and plugin lifecycle commands.

## Solution Assessment

Unclear. Adding progress is directionally right, but the maintainable path is a scoped rollout at selected slow command boundaries rather than a blanket CLI output change.
