## Summary

Keep open. Current main has shared CLI progress primitives and several covered startup/RPC paths, but source inspection still shows slow Skills ClawHub and plugin install/update/search flows running without the shared progress wrapper, so the requested UX is only partially implemented.

## What This Changes

The issue asks for OpenClaw CLI commands to show loading or progress indicators while slow operations are running.

## Best Possible Solution

Scope a rollout around the existing createCliProgress/withProgress contract for selected slow command families, preserving TTY, non-TTY, --json, and --plain behavior.

## Reproduction Assessment

No. The issue does not provide exact commands or timings, but current-main source inspection gives a high-confidence static path to remaining coverage gaps in Skills and Plugins CLI flows.

## Solution Assessment

Unclear. Adding loaders is the right direction, but the maintainable solution is a scoped command-boundary rollout rather than blanket progress output across the CLI.
