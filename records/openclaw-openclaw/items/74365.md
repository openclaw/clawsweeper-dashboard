## Summary

Keep open. Current main has shared CLI progress primitives and several command families already use them, but the broad loader request is still only partially covered: ClawHub skill/plugin search, install, update, and the doctor update prompt still have slow paths without a shared progress wrapper.

## What This Changes

Request to add loading/progress states across slow OpenClaw CLI commands so users can tell whether work is progressing.

## Best Possible Solution

Scope the slow command families first, then add the existing progress helper at selected command boundaries with coverage for TTY, non-TTY, JSON, and plain output behavior.

## Reproduction Assessment

No high-confidence runtime reproduction is available because the issue lacks exact commands and timings. Source inspection does show current-main ClawHub skill/plugin and doctor update paths that do slow network/install/update work without a shared progress wrapper.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable solution is a scoped rollout at selected slow command boundaries rather than a blanket CLI output change.
