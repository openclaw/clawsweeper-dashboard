## Summary

Keep open. Current main has a shared progress helper and partial startup/RPC coverage, but source inspection still shows ClawHub skill and plugin search/install/update flows doing network/download/update work without that wrapper.

## What This Changes

Add visible CLI progress/loading indicators for slow commands so users can tell OpenClaw is still working.

## Best Possible Solution

Use the existing ProgressReporter/withProgress contract at selected slow command boundaries, with tests that preserve interactive and machine-readable output behavior.

## Reproduction Assessment

No high-confidence runtime reproduction is available because the issue lacks exact commands and timings. Source inspection does verify plausible current-main gaps in ClawHub skill and plugin lifecycle commands.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable solution is a scoped rollout at selected slow command boundaries rather than blanket CLI output changes.
