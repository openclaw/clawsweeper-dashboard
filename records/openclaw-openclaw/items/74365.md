## Summary

Keep open. Current main has a shared CLI progress primitive and documented output-mode rules, but the broad loader request is still only partially covered; several slow ClawHub skill and plugin search/install/update paths still perform network/download/update work without the shared progress wrapper.

## What This Changes

The issue requests loading indicators for slow OpenClaw CLI commands.

## Best Possible Solution

Use the existing ProgressReporter/withProgress contract for a scoped set of slow CLI command families, with tests covering TTY, non-TTY, JSON, and plain output behavior.

## Reproduction Assessment

No high-confidence runtime reproduction is available because the issue lacks exact commands and timings. Source inspection does verify plausible current-main gaps in ClawHub skills and plugin lifecycle commands.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable solution is a scoped rollout at selected slow command boundaries rather than blanket CLI output changes.
