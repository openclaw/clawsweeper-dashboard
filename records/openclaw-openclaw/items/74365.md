## Summary

Keep open. Current main has a shared CLI progress primitive and some command coverage, but the broad loader request remains only partially implemented; ClawHub skill and plugin search/install/update paths still do network, download, or update work without the shared progress wrapper.

## What This Changes

Request to add visible loading/progress indicators to slow OpenClaw CLI commands so users can tell commands are still progressing.

## Best Possible Solution

Use the existing ProgressReporter/withProgress contract for a maintainer-scoped set of slow CLI command families while preserving parseable output modes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No runtime reproduction path is high-confidence because the issue lacks exact commands and timings; source inspection does verify plausible gaps in ClawHub skills and plugin lifecycle commands.

## Solution Assessment

Is this the best way to solve the issue? Unclear. Adding progress is directionally right, but the maintainable solution is a scoped rollout at selected slow command boundaries rather than blanket CLI output changes.
