## Summary

Keep open. Current main has reusable CLI progress primitives and some covered command paths, but the broad loader request is still only partially addressed: ClawHub skill/plugin paths and plugin update/install flows still perform potentially slow work without command-level progress, and doctor pre-update progress is represented by the still-open https://github.com/openclaw/openclaw/pull/47493 rather than shipped main behavior.

## What This Changes

Add loader/progress feedback to slow CLI commands that otherwise appear blank while work is in progress.

## Best Possible Solution

Scope the slow CLI command families first, then apply the existing progress helpers at selected command boundaries while preserving TTY, non-TTY, JSON, and plain-output behavior; land or replace the separate doctor-update PR for that path.

## Reproduction Assessment

No high-confidence runtime reproduction is available because the issue lacks exact commands and timings. Source inspection does show current-main skills, plugin search/update/install, and doctor pre-update paths doing potentially slow work without command-level progress callbacks.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable solution is a scoped rollout at chosen slow command boundaries using existing progress helpers, not a blanket CLI output change.
