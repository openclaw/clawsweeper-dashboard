## Summary

Keep open. Current main has a shared CLI progress helper and several slow command paths already use it, but the broad loading-state request is not fully satisfied: skills search/install/update and plugin install/update still rely on direct ClawHub/npm/archive work, lifecycle tracing, or logger lines rather than a consistent progress wrapper.

## What This Changes

The issue requests loading indicators for slow OpenClaw CLI commands so users can distinguish progress from a hang.

## Best Possible Solution

Use the existing shared progress helper as the standard CLI UI contract, then scope the rollout to selected slow command families with explicit tests for TTY, non-TTY, JSON/plain output, and plugin/skill install/update flows.

## Reproduction Assessment

No high-confidence runtime reproduction is available because the report lacks exact commands and timings. Source inspection does verify a remaining coverage gap in current main for skills and plugins paths while other slow paths already use the shared progress helper.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable solution is a scoped rollout of the existing progress helper at selected command boundaries rather than blanket loaders across every CLI action.
