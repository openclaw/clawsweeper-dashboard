## Summary

Keep open. Current main has a shared CLI progress primitive and partial coverage, but the broad loader request is not fully implemented for ClawHub skills and plugin discovery/install/update paths.

## What This Changes

Request adding user-visible loading or progress indicators to slow CLI commands so users are not left at a blank terminal during long waits.

## Best Possible Solution

Use the existing createCliProgress/withProgress contract for a scoped set of slow CLI boundaries, starting with skills and plugin discovery/install/update, with tests for TTY, non-TTY, JSON/plain output, and logger behavior.

## Reproduction Assessment

No. The report does not provide exact commands or timings; source inspection verifies plausible current-main gaps but not a high-confidence runtime reproduction.

## Solution Assessment

Unclear. Adding loaders is directionally right, but the maintainable solution is a scoped rollout through the shared progress helper rather than blanket spinner injection across all commands.
