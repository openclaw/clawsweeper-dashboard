## Summary

Keep open. Current main still lacks a caller-specified `sessions_spawn` ACP channel/topic delivery target; it supports `streamTo: "parent"`, bound-session inline delivery, and completion announce paths, but not the requested direct delivery contract.

## What This Changes

Add a `sessions_spawn` ACP delivery option that routes a spawned ACP session's output directly to a chosen channel conversation or topic.

## Best Possible Solution

Define a generic, authorization-checked ACP spawn delivery contract that reuses existing delivery context, session binding, and task-completion surfaces with documented live-stream versus final-delivery behavior.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: inspect the current schema and ACP dispatch path to see the missing `delivery` field, rejected direct channel aliases, and run-mode `deliver=false` behavior.

## Solution Assessment

No, not as a straight cron schema copy. The maintainable fix needs explicit policy for who may route ACP output to arbitrary channel targets and how that interacts with bound sessions, parent relay, and completion delivery.
