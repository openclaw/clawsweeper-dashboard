## Summary

Keep open. Current main still lacks an explicit ACP `sessions_spawn` delivery target for channel/topic output; existing code supports parent relay and bound thread/session delivery, but not the requested caller-specified Zulip/Discord-style route.

## What This Changes

Add a `delivery` option to ACP `sessions_spawn` so spawned ACP output can stream to a specified channel conversation or topic.

## Best Possible Solution

Define a generic, authorization-checked ACP spawn delivery contract that reuses session binding and route projection infrastructure while separating live progress streaming from terminal completion delivery.

## Reproduction Assessment

Yes. Source inspection shows the current tool schema lacks a `delivery` target, rejects direct channel/topic aliases, and dispatches ACP run-mode spawns without channel/to/thread delivery fields.

## Solution Assessment

No, not as a direct cron schema copy. The maintainable solution needs an explicit policy for who may route ACP output to arbitrary channel targets and how that composes with bound sessions, parent relay, and completion delivery.
