## Summary

Keep open. Current main still lacks a caller-specified ACP `sessions_spawn` delivery target; it only supports `streamTo: "parent"`, bound/session inline delivery, and completion announce paths, so the requested channel/topic delivery contract remains unresolved.

## What This Changes

Add a `delivery` option to `sessions_spawn` for `runtime: "acp"` so a spawned ACP session can route output to a chosen channel conversation or topic.

## Best Possible Solution

Define a generic, policy-checked ACP spawn delivery contract that reuses existing delivery context, session binding, and task-completion routing surfaces, with explicit authorization and documented live-stream versus final-delivery behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to inspect the current `sessions_spawn` schema and ACP dispatch path: the requested `delivery` field is absent, direct channel aliases are rejected, and run-mode ACP dispatch keeps channel delivery disabled.

## Solution Assessment

No, not as a straight cron schema copy. The maintainable path needs an authorized generic delivery design that fits existing session binding and completion delivery paths before implementation.
