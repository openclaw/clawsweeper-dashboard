## Summary

Keep open. Current main still lacks a caller-specified `sessions_spawn` ACP channel/topic delivery target; the shipped paths support parent relay, bound-session inline delivery, and completion announce behavior, but not this direct delivery contract.

## What This Changes

The issue asks for `sessions_spawn` ACP sessions to support caller-specified channel/topic delivery for live output, similar to cron announce delivery.

## Best Possible Solution

Define a generic, authorization-checked ACP spawn delivery contract that reuses existing delivery context, session binding, and task-completion surfaces with documented live-stream versus final-delivery behavior.

## Reproduction Assessment

Yes. Source inspection of current main shows the missing `delivery` field, rejected direct channel aliases, and run-mode ACP dispatch using `deliver: false` without channel/to/threadId fields.

## Solution Assessment

No, not as a straight cron schema copy. The maintainable fix needs explicit policy for who may route ACP output to arbitrary channel targets and how that interacts with bound sessions, parent relay, and completion delivery.
