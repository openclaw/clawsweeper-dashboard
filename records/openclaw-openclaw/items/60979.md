## Summary

Keep open. Current main still has the source-level gap described here: ACP `sessions_spawn` can relay to the parent session or use existing bound-session routing, but it cannot accept a caller-specified channel/topic delivery target.

## What This Changes

The issue requests a `sessions_spawn` ACP delivery option that can stream spawned-session output directly to a chosen channel conversation or topic.

## Best Possible Solution

Define a generic, authorization-checked ACP spawn delivery contract that reuses session binding and route projection infrastructure while keeping live progress streaming separate from final completion delivery.

## Reproduction Assessment

Yes. Source inspection shows the current tool schema lacks a `delivery` target, rejects direct channel/topic aliases, and dispatches ACP run-mode spawns without channel/to/thread delivery fields.

## Solution Assessment

No, not as a direct cron schema copy. The maintainable solution needs an explicit policy for who may route ACP output to channel targets and how that composes with bound sessions, parent relay, and completion delivery.
