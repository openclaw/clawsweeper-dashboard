## Summary

Keep open. Current main still lacks a caller-specified `sessions_spawn` ACP channel/topic delivery target; the available paths cover parent relay, thread-bound inline delivery, and completion/cron announce delivery, but not the requested direct delivery contract.

## What This Changes

Add a `delivery` option to ACP `sessions_spawn` so spawned ACP output can stream to a selected channel conversation or topic such as Zulip or Discord.

## Best Possible Solution

Define a generic, authorization-checked ACP spawn delivery contract that reuses existing delivery context, session binding, and task-completion surfaces while documenting live-stream versus final-delivery behavior.

## Reproduction Assessment

Yes, source-backed. Current main exposes no `delivery` field for ACP `sessions_spawn`, rejects direct channel aliases, and dispatches run-mode ACP spawns without channel/to/thread delivery fields.

## Solution Assessment

No, not as a direct cron schema copy. The maintainable path needs an explicit policy for who may route ACP output to arbitrary channel targets and how that interacts with bound sessions, parent relay, and completion delivery.
