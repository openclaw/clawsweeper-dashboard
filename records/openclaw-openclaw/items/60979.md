## Summary

Keep open. Current main still lacks a caller-specified ACP `sessions_spawn` channel/topic delivery contract; related work covers parent relay, bound-thread streaming, and completion announce routing, but not the requested explicit Zulip/Discord-style target.

## What This Changes

Requested support for directing ACP `sessions_spawn` output to a specific channel conversation or topic.

## Best Possible Solution

Define a generic, authorization-checked ACP spawn delivery contract that reuses delivery context and session binding infrastructure while clearly separating live progress streaming from terminal completion delivery.

## Reproduction Assessment

Yes. Source inspection shows current main exposes no `delivery` field for ACP `sessions_spawn`, rejects direct channel aliases, and dispatches run-mode ACP spawns without channel/to/thread delivery fields.

## Solution Assessment

No, not as a direct cron schema copy. The maintainable path needs an explicit policy for who may route ACP output to arbitrary channel targets and how that composes with bound sessions, parent relay, and completion delivery.
