## Summary

Keep open: current main still has a source-level path where Discord guild/channel turns resolve to `message_tool_only`, pass that flag into the draft-preview controller, and skip creating the progress draft before any status update can be rendered. The related open PR addresses adjacent explicit tool-progress delivery, but its diff does not remove this draft-creation gate.

## What This Changes

The issue reports that Discord `streaming.mode: "progress"` no longer creates or updates a visible status draft during agent execution.

## Best Possible Solution

Keep this issue open until Discord progress mode creates and updates the draft for useful work events in message-tool-only guild/thread turns while preserving final-reply privacy, either by amending https://github.com/openclaw/openclaw/pull/80042 or landing a narrow follow-up.

Yes, source-reproducible with high confidence: a Discord guild/channel turn defaults to `message_tool_only`, and that flag still prevents `createDiscordDraftStream` from being created on current main. I did not run a live Discord session.
