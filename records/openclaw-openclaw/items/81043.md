## Summary

Keep open: current main still has a Discord guild/channel path where progress-draft creation is disabled before any progress updates can render, despite the documented `progress` mode behavior.

## What This Changes

Restore Discord `streaming.mode: "progress"` so it creates and updates one editable status draft while an agent runs.

## Best Possible Solution

Allow Discord progress-draft creation for explicit work/progress events independently of final source-reply auto-publication, with regression coverage for `message_tool_only` guild/channel turns and preserved final-reply privacy.

## Reproduction Assessment

Yes, source-level: a Discord guild/channel turn defaults to `message_tool_only`, passes that flag into the draft-preview controller, and the controller refuses to create a draft stream. I did not run a live Discord session.

## Solution Assessment

Yes, the narrow maintainable direction is to decouple visible progress-draft creation from final-reply privacy. https://github.com/openclaw/openclaw/pull/80042 should be amended or followed by a focused patch if maintainers want the same implementation path to cover this issue.
