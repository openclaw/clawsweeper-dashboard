## Summary

Keep open: current main still has a source-level path where Discord guild channel/thread turns in progress mode cannot create a draft stream before progress updates arrive. The related Discord progress PR is open and adjacent, but it does not touch the draft-preview creation gate this report calls out.

## What This Changes

Reports a regression where Discord `streaming.mode: "progress"` on v2026.5.7 fails to create or update the visible progress/status draft and stays silent until the final response.

## Best Possible Solution

Fix Discord progress mode so useful work events can create and update the draft even when source final replies are message-tool-only, while preserving the final-reply privacy boundary and adding focused Discord regression coverage.

## Reproduction Assessment

Yes, from source inspection for Discord guild channel/thread turns: those turns resolve to `message_tool_only`, that flag is passed into the draft-preview controller, and the controller declines to create a draft stream. I did not run live Discord, and the direct-message portion needs separate validation.

## Solution Assessment

No close path yet. The narrow maintainable fix belongs in the Discord draft-preview/message-handler boundary or the related open PR, with tests proving progress drafts can start without making private final replies public.
