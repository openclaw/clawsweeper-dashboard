## Summary

Keep open: the report matches a documented Discord progress-draft behavior, and current main still has a source-level path where Discord channel/thread turns can suppress draft creation before any progress message is sent.

## What This Changes

Investigate a Discord regression where `streaming.mode: "progress"` no longer creates or updates the live status/progress draft before the final response.

## Best Possible Solution

Discord progress mode should create and update the progress draft for useful work events while keeping final-reply privacy separate for `message_tool_only` channel turns.

## Reproduction Assessment

Yes, from source inspection: Discord channel turns default to `message_tool_only`, that flag is passed to the draft-preview controller, and the controller refuses to create a draft stream when it is true. I did not run a live Discord session.

## Solution Assessment

No close path yet; current main still has the draft-creation gate, and the related open PR appears to route explicit tool-progress payloads rather than initialize the status draft itself.
