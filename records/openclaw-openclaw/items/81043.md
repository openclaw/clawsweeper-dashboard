## Summary

Keep open: current main still has a source-level Discord path where progress draft creation is skipped for channel turns before any status draft can be updated, even though the docs promise an editable progress draft in progress mode.

## What This Changes

The issue reports that Discord `streaming.mode: "progress"` no longer creates or updates the visible status draft during agent execution.

## Best Possible Solution

Discord progress mode should decouple visible progress-draft creation from final source-reply privacy so useful work events can update one draft while `message_tool_only` still protects normal final replies.

## Reproduction Assessment

Yes, source-level: a Discord guild/channel turn defaults to `message_tool_only`, passes that flag into the draft-preview controller, and the controller refuses to create a draft stream when it is true. I did not run a live Discord session.

## Solution Assessment

Yes, the narrow maintainable direction is to preserve the final-reply privacy boundary while allowing progress-mode draft creation for useful work events. The related PR may help explicit tool-progress delivery, but it should be amended or followed by a patch for this draft-creation gate.
