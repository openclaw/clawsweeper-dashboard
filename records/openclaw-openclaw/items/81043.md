## What This Changes

Discord progress mode should create and update an editable status draft during tool-heavy turns even when guild/channel replies use message-tool-only source delivery.

## Best Possible Solution

Keep the current main implementation for this narrower draft-creation behavior, and continue handling the broader verbose-progress delivery cluster through its own issue and pull request.

Yes for the affected release/source path: the release gate skipped draft creation for message-tool-only source replies. On current main, that path no longer reproduces because progress mode now creates the draft stream and forwards progress callbacks.

Yes. The current-main fix is the narrow maintainable direction because it preserves final-reply privacy while allowing progress-mode draft updates for useful work events.
