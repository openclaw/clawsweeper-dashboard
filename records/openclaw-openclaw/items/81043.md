## Summary

Keep open: current main still has a source-level path where Discord channel turns resolve to `message_tool_only`, and that mode prevents the Discord progress-draft stream from being created before any status update can render.

## What This Changes

Discord `streaming.mode: "progress"` should create and update a visible status draft while the agent is working.

## Best Possible Solution

Discord progress mode should create a progress draft for useful work events while keeping `message_tool_only` final-reply privacy separate from progress/status visibility.

## Reproduction Assessment

Yes, from source inspection: Discord channel turns can resolve to `message_tool_only`, that flag is passed into the draft-preview controller, and the controller refuses to create a draft stream. I did not run a live Discord session.

## Solution Assessment

No close path yet. The narrow maintainable fix should decouple progress-draft initialization from final source-reply suppression; the related open PR addresses explicit tool-progress delivery but not this draft creation gate.
