## Summary

Keep open: current main still has a source-level path where Discord channel turns default to message-tool-only delivery, that flag prevents draft stream creation, and the documented progress-draft behavior is not guaranteed. The related open PR is useful for the Discord progress-delivery cluster, but it does not remove the draft-preview creation gate this report calls out.

## What This Changes

The issue reports that Discord `streaming.mode: "progress"` no longer creates or updates the editable status draft during agent execution on v2026.5.7.

## Best Possible Solution

Discord progress mode should allow channel-owned progress draft creation and updates for useful work events even when normal final/source replies remain message-tool-only for privacy.

## Reproduction Assessment

Yes, from source inspection: Discord guild/channel turns resolve to `message_tool_only`, that flag is passed to the draft-preview controller, and the controller refuses to create a draft stream when it is true. I did not run a live Discord session.

## Solution Assessment

No close path yet. The maintainable direction is to preserve the final-reply privacy boundary while allowing the channel-owned progress draft callbacks; https://github.com/openclaw/openclaw/pull/80042 should be reviewed or amended if it is intended to cover this too.
