## Summary

Keep open: current main still has a source-level Discord guild/channel path where `streaming.mode: "progress"` cannot create a progress draft because source replies are marked message-tool-only before the draft-preview controller is created.

## What This Changes

Discord progress streaming mode should create and update an editable status draft while the agent is working in Discord channels or threads.

## Best Possible Solution

Discord progress mode should allow progress-draft creation for useful work events while preserving message-tool-only final-reply privacy for normal assistant replies.

## Reproduction Assessment

Yes, from source inspection for Discord guild/channel turns: the resolver defaults the turn to `message_tool_only`, that flag reaches the draft-preview controller, and the controller refuses to create a draft stream. I did not run a live Discord session.

## Solution Assessment

No close path yet. The narrow maintainable fix should separate progress-draft visibility from final-reply visibility, and the related open PR does not currently remove the draft creation gate.
