## Summary

Keep open: the current Discord progress-draft path still suppresses draft creation for guild/channel turns that resolve to message-tool-only delivery, which matches the reported silent-until-final behavior and is not fixed on current main.

## What This Changes

The issue reports that Discord `streaming.mode: "progress"` no longer creates or updates the visible status draft while an agent is working.

## Best Possible Solution

Amend the related Discord progress work or land a narrow follow-up so progress mode starts and updates a draft for real work events even when final source replies remain message-tool-only, with regression coverage for guild/channel and thread turns.

## Reproduction Assessment

Yes, with source-level confidence: Discord guild/channel turns resolve to `message_tool_only`, pass `sourceRepliesAreToolOnly` into the draft-preview controller, and that controller refuses to create a draft stream. I did not run a live Discord session.

## Solution Assessment

No close path yet; current main still has the draft-creation gate. The narrow maintainable direction is to decouple progress-draft visibility from final-reply privacy rather than broadly changing message-tool-only delivery.
