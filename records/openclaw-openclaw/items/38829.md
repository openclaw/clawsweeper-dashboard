## Summary

Keep open. Current `main` still has the Gateway `sessions.messages.subscribe` / `session.message` contract, but the bundled TUI does not call that subscription API or route `session.message` events into the active chat view. The issue is already paired with open implementation PR #73402, so it should remain open until that PR is merged or deliberately closed unmerged.

## What This Changes

The issue requests that `openclaw tui` display new messages from other surfaces in real time instead of only showing them after restarting or re-entering the TUI.

## Best Possible Solution

Review and land #73402 or an equivalent focused TUI change that subscribes to the active session’s transcript events, unsubscribes stale sessions, routes `session.message` updates into history/rendering safely, filters non-active sessions, and adds targeted regression coverage; close this issue only after that fix reaches `main`.
