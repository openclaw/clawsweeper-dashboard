## Summary

Keep open: current main and the latest shipped release still lack the TUI live transcript subscription/dispatch path, and https://github.com/openclaw/openclaw/pull/73402 remains the open linked implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue requests that `openclaw tui` live-tail new transcript messages from the active session instead of only showing them after history reload or restart.

## Best Possible Solution

Land or replace the scoped TUI fix that consumes the existing `sessions.messages.subscribe` / `session.message` gateway contract with active-session filtering, cleanup on session switch/stop, and regression coverage.

## Reproduction Assessment

Yes, source-level. The reporter gives concrete cross-surface steps, and current main shows the TUI loads history and handles chat/agent websocket events but does not subscribe to or dispatch `session.message` transcript events.

## Solution Assessment

Yes for the central bug. The narrow maintainable fix is to wire the TUI to the existing gateway live transcript subscription contract rather than adding polling or a parallel message mirror.
