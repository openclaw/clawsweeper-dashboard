## Summary

Keep open. Current main still lacks TUI active-session transcript subscription and `session.message` routing, and https://github.com/openclaw/openclaw/pull/73402 is an open, unmerged fix PR that references this issue with closing syntax.

## What This Changes

The issue requests that `openclaw tui` display new transcript messages from the web UI or other channels in real time instead of only after exit/reopen history reload.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73402, or an equivalent narrow TUI active-session transcript subscription with focused tests for lifecycle, session switching, event filtering, and active-stream deferral.

## Reproduction Assessment

Yes. The reporter gives concrete cross-surface steps, and current-main source inspection confirms the TUI loads history and handles `chat` events but does not subscribe to or route the existing `session.message` transcript stream.

## Solution Assessment

Yes. Using the existing Gateway `sessions.messages.subscribe` and `sessions.messages.unsubscribe` contract in the TUI is the narrow maintainable fix; a broader Gateway chat fanout change would be less targeted.
