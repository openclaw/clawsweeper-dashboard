## Summary

Keep open. Current main still lacks TUI active-session transcript subscription and `session.message` routing, and open PR #73402 references this issue with closing syntax as the unmerged fix path.

## What This Changes

Make `openclaw tui` display new messages appended from other surfaces in real time instead of only after restarting or reloading history.

## Best Possible Solution

Review and land #73402, or an equivalent narrow TUI active-session transcript subscription with focused tests for lifecycle, session switching, and event filtering.

## Reproduction Assessment

Yes. The report gives concrete cross-surface steps, and current-main source inspection confirms the TUI loads history and handles `chat` events but never subscribes to the existing per-session transcript event stream.

## Solution Assessment

Yes. Using the existing Gateway `sessions.messages.subscribe`/`unsubscribe` API in the TUI is the narrow maintainable fix; broad Gateway chat fanout changes would be less targeted.
