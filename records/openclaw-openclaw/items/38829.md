## Label Justifications

- `P2`: This is a normal-priority TUI regression with a restart/reopen workaround but meaningful real-time usability impact.
- `impact:message-loss`: Messages appended from another surface are missing from the live terminal view until history is reloaded.
- `impact:session-state`: The TUI visible transcript can drift behind the Gateway canonical active-session transcript.

## Summary

Keep this issue open: current main still lacks a TUI subscription and handler for live `session.message` transcript events, and https://github.com/openclaw/openclaw/pull/73402 is the open linked implementation candidate rather than a merged fix.

## What This Changes

The issue requests that `openclaw tui` live-render messages appended to its active Gateway session by other surfaces instead of only showing them after restart or reload.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73402 with a bounded TUI-side fix that subscribes to the active session, unsubscribes stale sessions, filters events, and refreshes history without disrupting active streams.

## Reproduction Assessment

Yes, at source level: Gateway exposes and tests the live transcript subscription contract, while the current TUI has no subscription method or `session.message` event route. I did not run a live Gateway/TUI reproduction in this read-only review.

## Solution Assessment

Yes in direction: consuming the existing per-session Gateway transcript subscription in the TUI is the narrow owner-boundary fix. The open linked PR remains the implementation vehicle unless maintainers repair or replace it.
