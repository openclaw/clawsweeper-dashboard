## Summary

Keep open. Current main still lacks the TUI active-session transcript subscription and `session.message` dispatch needed for real-time cross-surface messages, and https://github.com/openclaw/openclaw/pull/73402 is an open implementation PR that references this issue with closing syntax.

## What This Changes

The issue reports that `openclaw tui` loads historical chat messages but does not receive new messages from other surfaces in real time.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73402, or replace it with an equivalent scoped fix that adds TUI subscribe/unsubscribe, `session.message` handling, active-run deferral, session-switch cleanup, and focused regression tests.

## Reproduction Assessment

Yes. The reporter gives concrete cross-surface steps, and source inspection on current main shows the TUI loads history and handles `chat` events but does not subscribe to or route the existing `session.message` stream.

## Solution Assessment

Yes. Reusing the existing Gateway `sessions.messages.subscribe` and `sessions.messages.unsubscribe` contract in the TUI is the narrow maintainable solution; broad gateway fanout changes or a new mirror mode would be a separate product decision.
