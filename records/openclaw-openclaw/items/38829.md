## Summary

Current main still lacks the TUI-side live transcript subscription, and the open maintainer-authored PR is the explicit closing implementation candidate. Repository policy requires this issue to stay open until that PR lands or is replaced.

## What This Changes

Make the terminal UI display new messages appended to its active session in real time instead of only after restart or history reload.

## Best Possible Solution

Land or revise the linked active-session transcript subscription so the TUI subscribes after connect and session switches, unsubscribes stale sessions, filters events to the selected session, and refreshes history without disrupting an active local stream.

## Reproduction Assessment

Yes, at source level with high confidence: current main exposes and tests live session.message delivery but the TUI has no subscription or handler, matching the reported pattern where restarting reloads the missing persisted turns. This read-only review did not run a live gateway/TUI reproduction.

## Solution Assessment

Yes in direction: consuming the existing transcript subscription for only the active TUI session is narrower and safer than widening generic hidden-run chat broadcasts. The linked PR should remain the implementation vehicle unless maintainers replace it with an equivalent bounded fix.
