## Summary

Keep this issue open: current main and the latest release still expose the Gateway transcript subscription contract, but the bundled TUI does not subscribe to or handle live `session.message` updates. An open linked PR is already the implementation candidate, so this should stay open until that PR lands, is repaired, or is replaced.

## What This Changes

The issue requests real-time TUI rendering of messages appended to the active session from the web UI or another channel.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73402 with a bounded TUI fix that subscribes after connect/session switches, unsubscribes stale sessions, filters to the active session, and refreshes history without clobbering active local streams.

## Reproduction Assessment

Yes, at source level: current main exposes and tests live `session.message` delivery, while the TUI has no subscription or handler for those events. I did not run a live Gateway/TUI reproduction in this read-only review.

## Solution Assessment

Yes in direction: consuming the existing per-session Gateway transcript subscription in the TUI is the narrow owner-boundary fix. The open linked PR remains the right vehicle unless maintainers replace it with an equivalent bounded fix.
