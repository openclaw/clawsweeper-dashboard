## Summary

Keep open. Current main still has the gateway transcript subscription API but the TUI has no subscription, unsubscribe, or `session.message` dispatch path, and the issue is already paired with an open closing PR that should be reviewed or replaced rather than preempted.

## What This Changes

The issue reports that `openclaw tui` loads transcript history but does not receive real-time messages sent from the web UI or another channel until the TUI is restarted.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73402, or replace it with an equivalent scoped TUI fix that consumes the existing `sessions.messages.subscribe` and `session.message` gateway contract.

## Reproduction Assessment

Yes. The reporter gives concrete cross-surface steps, and current-main source inspection shows the TUI loads gateway history but has no `session.message` subscription or dispatch path; I did not run the live macOS flow.

## Solution Assessment

Yes. Reusing the existing gateway live transcript subscription API is the narrow maintainable fix; broadening generic chat-event fanout would be less targeted.
