## Summary

Keep open. Current main still lacks a TUI subscription or dispatch path for gateway `session.message` transcript updates, while the docs and gateway API make the gateway-backed transcript the expected source of truth; the linked implementation PR is still open, so closing this issue before that work lands would be premature.

## What This Changes

The TUI should display real-time messages appended to the active gateway session from the Web UI or another channel instead of only showing them after restart or reentry.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73402, or replace it with an equivalent scoped TUI fix that consumes the existing `sessions.messages.subscribe` and `session.message` contract.

## Reproduction Assessment

Yes. The reporter gives concrete cross-surface steps, and source inspection shows the TUI loads history but never subscribes to or dispatches gateway `session.message` events; I did not run the live macOS flow.

## Solution Assessment

Yes. Reusing the existing gateway live transcript subscription API is the narrow maintainable fix; broadening generic chat-event fanout would be a less targeted route.
