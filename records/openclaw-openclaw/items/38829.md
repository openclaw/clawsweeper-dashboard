## Summary

Keep open. Current main still has the Gateway transcript subscription API and tests, but the bundled TUI does not expose active-session message subscribe/unsubscribe methods or route `session.message` events into the chat view. Open PR #73402 is the active unmerged implementation path, so this issue should remain open until that PR lands or is closed unmerged.

## What This Changes

The issue reports that `openclaw tui` loads chat history but does not receive new messages from other surfaces in real time.

## Best Possible Solution

Review and land the focused TUI fix in #73402, or keep this issue as the canonical bug if that PR closes unmerged. The implementation should add active-session subscribe/unsubscribe support in the TUI backend/client, resubscribe on session switches and reconnects, route only active-session `session.message` events, and cover the cross-surface regression with targeted TUI tests.

## Reproduction Assessment

Yes. The report gives concrete steps, and current-main source inspection confirms the TUI loads history and handles chat/agent events but never subscribes to the Gateway per-session transcript event stream.

## Solution Assessment

Yes. The narrow TUI active-session subscription path is the best direction because the Gateway already exposes and tests the required per-session transcript event API; changing broad Gateway chat fanout would be less targeted.
