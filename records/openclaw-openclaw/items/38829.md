## Summary

Keep open. Current main and the latest release still expose the gateway live transcript subscription API, but the TUI does not subscribe to it or dispatch `session.message` events, and an open linked PR with closing syntax should be reviewed or replaced before this issue is closed.

## What This Changes

The issue asks for `openclaw tui` to render live gateway transcript messages from other surfaces without restarting the TUI.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73402, or replace it with an equivalent scoped TUI fix that consumes the existing gateway transcript subscription contract.

## Reproduction Assessment

Yes, source-reproducible. The reporter gives concrete cross-surface steps, and current main shows the TUI loads history but has no `sessions.messages.subscribe` client method or `session.message` dispatch path.

## Solution Assessment

Yes for the central bug. Reusing the existing `sessions.messages.subscribe` and `session.message` gateway contract in the TUI is the narrow maintainable fix; broader bidirectional mirroring should stay separate.
