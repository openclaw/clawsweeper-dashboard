## Summary

Keep open. The PR is a focused fix for a real node-host exec diagnostics gap that is still present on current main and the latest release; the code shape looks correct after the timeout guard, but this external PR still needs after-fix real behavior proof and fresh validation before merge.

## What This Changes

The PR adds a signal-aware exit-result helper in `src/node-host/invoke.ts` plus tests for signal-killed, timeout-killed, normal-exit, and null-signal subprocess exits.

## Best Possible Solution

Land the narrow signal-observability fix after redacted real runtime proof is added and fresh validation is green; keep broader kill-cause diagnostics in the related open issues.

## Reproduction Assessment

Yes at source level: current main’s node-host `runCommand` ignores the exit `signal` argument, and a focused Node probe shows a signal-killed child emits `{code:null, signal:"SIGKILL"}`. I did not establish a full OpenClaw live reproduction in this read-only review.

## Solution Assessment

Yes. Extracting a small exit-result helper and gating signal errors on `!timedOut` is the narrow maintainable fix for this result contract; broader SIGKILL cause telemetry should stay in the related diagnostics issues.
