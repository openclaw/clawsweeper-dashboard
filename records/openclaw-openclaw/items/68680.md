## Summary

Keep open: the patch still looks like a narrow, correct fix for signal-killed exec subprocesses, but this external PR is missing after-fix real behavior proof and currently has stale failing checks.

## What This Changes

The PR adds a signal-aware `resolveExitResult` helper in `src/node-host/invoke.ts` and unit coverage for SIGKILL/SIGTERM, normal exits, timeout kills, and null-signal exits.

## Best Possible Solution

Land the narrow signal-observability fix after redacted real runtime proof is added and the branch has fresh green validation; keep broader kill-cause telemetry in the linked diagnostics issues.

## Reproduction Assessment

Yes at source level: current main ignores the child-process exit `signal`, while Node's contract and a focused probe show a SIGKILL child reports `{code:null, signal:"SIGKILL"}`. I did not run the full OpenClaw PR branch in a real contributor setup.

## Solution Assessment

Yes. Extracting a small exit-result helper and gating signal errors on `!timedOut` is the narrow maintainable fix for this result contract; broader diagnostic metadata belongs in the linked follow-up issues.
