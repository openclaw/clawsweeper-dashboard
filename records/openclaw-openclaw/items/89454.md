## Summary

Keep open: this member-authored PR is the active fix candidate for a still-open Feishu SecretRef media regression, and current main still has the strict runtime resolver path that can fail before media upload. The patch has no concrete line-level correctness finding from this read-only review, but it is currently conflict-marked and changes Feishu credential-provider timing, so it needs maintainer review before merge.

## What This Changes

The branch adds Feishu outbound credential SecretRef resolution before outbound sends, message-tool sends, and reply-dispatcher delivery build or use Feishu clients.

## Best Possible Solution

Resolve the conflict, then either land this scoped Feishu repair after Feishu/SecretRef owners accept send-time credential resolution or move the invariant to a single client-build/runtime-snapshot boundary.

## Reproduction Assessment

Yes at source level. Current main sends Feishu media through sendMediaFeishu, which calls the strict runtime account resolver before upload and throws on unresolved exec/keychain SecretRefs.

## Solution Assessment

Unclear as a final architecture, but acceptable as a bounded repair. The cleaner long-term shape is resolving app credentials at a single Feishu client/runtime-snapshot boundary rather than from each outbound entry point.
