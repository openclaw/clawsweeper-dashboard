## Summary

Keep open: the PR appears to repair the reported Feishu MEDIA SecretRef regression without adding config, SDK, dependency, or protocol surface, but it is an open member-authored PR and changes Feishu auth-provider resolution timing, so it needs maintainer review rather than cleanup close.

## What This Changes

The branch adds a Feishu helper to asynchronously resolve unresolved appId/appSecret SecretRefs before outbound, message-tool, and reply-dispatcher sends build clients, with regression coverage for the exec/keychain case.

## Best Possible Solution

Land this scoped repair if Feishu owners accept send-time SecretRef resolution for outbound app credentials; otherwise move the same invariant to a single Feishu client-build credential boundary before merge.

## Reproduction Assessment

Yes, at source level: current main's media path calls the strict Feishu runtime account resolver, and that resolver throws on unresolved exec/keychain appSecret SecretRefs before upload. I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

This is an acceptable bounded regression repair, but not the only possible final architecture. The structurally cleaner alternative is resolving app credentials at the Feishu client-build boundary for every client path, while this PR intentionally covers the reported outbound/message/reply paths.
