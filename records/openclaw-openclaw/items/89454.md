## Summary

Keep open; this is a useful Feishu bugfix, but it is not merge-ready because the new credential inlining path can create a normalized account-map entry and drop settings from an existing noncanonical or mixed-case Feishu account key.

## What This Changes

The PR resolves Feishu outbound app credential SecretRefs before sendPayload, sendText, and sendMedia build clients for payload and media delivery.

## Best Possible Solution

Keep the outbound SecretRef resolution seam, but patch the helper to update the actual matched account-map key and add a regression test for noncanonical account keys before merge.

## Reproduction Assessment

Yes. Source inspection shows current main's Feishu media path calls the strict runtime account resolver, which throws for exec/file SecretRefs before the media client is built, and the outbound adapter then falls back to text/URL.

## Solution Assessment

No as submitted. The outbound seam is the right layer, but the implementation needs to preserve the original matched account-map key instead of writing resolved credentials under only the normalized account id.
