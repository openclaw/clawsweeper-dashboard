## What This Changes

The PR adds Feishu outbound credential SecretRef resolution before outbound sends, message-tool sends, and reply-dispatcher delivery paths build or use Feishu clients.

## Best Possible Solution

Resolve the branch conflict, then either land this scoped Feishu repair after Feishu/SecretRef owners accept send-time credential resolution or move the invariant to a single Feishu client-build/runtime-snapshot boundary.

## Reproduction Assessment

## Solution Assessment

Unclear as the final architecture, but acceptable as a bounded repair if owners accept the credential boundary. The cleaner long-term shape is resolving app credentials at a single Feishu client/runtime-snapshot boundary rather than from each outbound entry point.
