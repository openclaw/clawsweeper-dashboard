## Summary

Keep open: current main does not include the Android cleanup, but this PR is not ready because the cron filter can drop legitimate user text and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR changes Android ChatController history parsing to skip user-role cron-completion-looking entries and unwrap top-level `<final>...</final>` from assistant/system text before rendering.

## Best Possible Solution

Use a stable metadata/subtype or exact synthetic marker from the Gateway for cron completion hiding, keep the role-scoped `<final>` unwrap, and add Android parser coverage for both the hidden marker and a legitimate user sentence.

## Reproduction Assessment

Yes. From PR-head source, a user-role `chat.history` text like `A cron job I scheduled just completed successfully; can you explain?` is under the cap, starts with `A cron job`, contains the success phrase, and would be dropped by `parseHistory`.

## Solution Assessment

No. The assistant/system `<final>` unwrap is narrow enough, but the cron filter should use stable metadata or an exact synthetic marker instead of inferring control intent from ordinary prose.
