## Summary

Keep open: current main still lacks the Android cleanup, but this PR would hide legitimate user-role history messages through a text heuristic and still needs real after-fix Android proof before merge.

## What This Changes

This PR adds Android chat history filtering for cron-completion entries, role-scoped `<final>` unwrapping, and incidental oc-path/gateway formatting churn.

## Best Possible Solution

Land a focused Android/history normalization change that uses a stable metadata or exact synthetic marker for cron artifacts, reuses established display-normalization semantics for `<final>` output, drops unrelated churn, and adds parser coverage.

## Reproduction Assessment

Yes, for the PR-introduced defect by source inspection: a user-role history entry with text `A cron job I scheduled just completed successfully` is under the cap, satisfies the start/end matcher, and is dropped in `parseHistory`. I did not run Android in this read-only review.

## Solution Assessment

No. The role-scoped `<final>` cleanup direction is reasonable, but the cron suppression should key off a stable metadata or exact-marker contract and the unrelated oc-path/gateway formatting churn should be removed before merge.
