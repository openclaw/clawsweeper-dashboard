## Summary

Keep this PR open for revision. Current main still lacks the Android rendering cleanup, so the contribution is not obsolete, but the PR branch still drops plausible real user messages through a broad cron-completion prose heuristic even after the follow-up commit narrowed it with a prefix and length cap.

## What This Changes

The PR modifies Android `ChatController` history parsing to skip user-role cron-completion-looking entries and unwrap single top-level `<final>...</final>` wrappers from assistant/system text content before rendering.

## Best Possible Solution

Keep the Android cleanup only if the cron marker can be identified without broad user-text inference, preferably through metadata/subtype or an exact synthetic marker contract, and add a small parser regression covering the hidden marker, a legitimate user sentence, and role-scoped `<final>` unwrapping.

## Reproduction Assessment

Yes. On the PR branch, a `chat.history` user entry with text like `A cron job I scheduled just completed successfully; can you explain?` is under the cap, starts with `A cron job`, contains the success phrase, and would be removed by `parseHistory`.

## Solution Assessment

No. The role-scoped `<final>` unwrap is narrow enough after the follow-up commit, but the cron filter is still not the best fix because it infers control/system intent from ordinary prose instead of a stable marker or metadata-backed contract.
