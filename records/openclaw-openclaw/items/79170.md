## Summary

Keep open: current main and the latest release still expose only internal post-core plugin convergence, while the requested user-facing post-update hook is a security-sensitive product/config surface that needs maintainer contract before implementation.

## What This Changes

Add an opt-in post-update hook surface for scripts after `openclaw update` plugin sync, with update context, timeout, exit-code reporting, and warning-only failures.

## Best Possible Solution

Define a maintainer-approved, opt-in post-update extension contract for the updater, then implement it through the canonical update path with explicit security boundaries, structured result reporting, docs, and focused update/hook tests.
