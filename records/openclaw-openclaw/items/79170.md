## Summary

Keep open. Current main has internal post-core plugin sync and existing Gateway hook events, but the requested user-facing update hook remains absent and needs core updater design rather than ClawHub-only work.

## What This Changes

The issue requests a user-facing post-update hook for `openclaw update` so local scripts can run automatically after updates.

## Best Possible Solution

Define an explicit update lifecycle contract, or intentionally reject arbitrary post-update scripts, with schema/docs/tests covering shell updates, Gateway auto-updates, restart ordering, failure behavior, and security defaults.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source and docs inspection gives high confidence that current main has no user-facing post-update hook.

## Solution Assessment

Unclear: the requested config or directory convention is plausible, but not yet proven to be the narrowest maintainable solution because update execution spans package replacement, plugin sync, and managed-service restart semantics.
