## Summary

Keep open. Current main still has plugin-only post-update handling, and the requested operator post-update hook is a new updater lifecycle/security contract rather than an implemented or obsolete behavior.

## What This Changes

Add a user-facing post-update hook for `openclaw update` that runs operator scripts after plugin sync with update context, timeout, JSON reporting, and nonfatal failure handling.

## Best Possible Solution

Define or explicitly reject a supported post-update lifecycle hook with schema, docs, tests, and security semantics across CLI, package-manager, and Gateway restart flows.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source and docs inspection confirms current main has internal plugin post-update work but no user-facing update hook or script-result surface.

## Solution Assessment

Unclear: a config entry or directory convention is plausible, but maintainers need to decide lifecycle timing, failure behavior, timeout, environment exposure, output redaction, and JSON reporting before implementation.
