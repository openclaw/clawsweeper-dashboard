## Summary

Keep open. Current main still exposes only internal/plugin post-update handling, while the refreshed report and latest production comment ask for a new user-facing updater hook with unresolved lifecycle, JSON, timeout, restart, and local-script security semantics.

## What This Changes

Add a user-facing post-update hook for `openclaw update` that can run local scripts after plugin sync with update context, timeout handling, warning-only failures, and JSON result reporting.

## Best Possible Solution

Define or reject a supported core update lifecycle contract with schema, docs, tests, timing relative to plugin sync and restart, context payload, timeout, redaction, failure handling, and JSON output.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source and docs inspection confirms current main has internal post-update plugin sync, but no user-facing update hook or script result contract.

## Solution Assessment

Unclear: a config entry or directory convention is plausible, but maintainers need to choose lifecycle timing, restart ordering, failure semantics, environment exposure, output redaction, and JSON shape before implementation.

## Security Review

Status: needs_attention

Summary: The requested feature would execute operator-local scripts during update, so it needs an explicit security and redaction contract before implementation.
