## Summary

Keep open: current main still does not provide the requested stable embeddable core package or public in-process agent-loop API, and the remaining question is a maintainer API/packaging decision rather than cleanup.

## What This Changes

Request a stable documented importable core API/package that exposes OpenClaw agent-loop, tool, skill, and memory primitives for programmatic use without requiring the Gateway daemon.

## Best Possible Solution

Make an explicit maintainer API decision: either bless the existing Gateway-backed App SDK and plugin runtime as the supported boundaries, or design a narrow versioned embeddable package with docs and contract tests.

## Reproduction Assessment

Not applicable; this is a feature/API RFC, not a bug report. Static source and docs inspection gives high confidence that current main lacks the requested public @openclaw/core-style in-process contract.

## Solution Assessment

Unclear until maintainers choose the public boundary. A narrow embeddable package may be maintainable, but it must be reconciled with @openclaw/sdk, plugin-sdk runtime helpers, internal agent command trust boundaries, and package ownership first.
