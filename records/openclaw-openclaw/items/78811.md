## Summary

Keep open: current main does not expose the requested stable embeddable core entry point. The existing public surfaces are a Gateway-backed App SDK and plugin-scoped runtime helpers, while the likely referenced RuntimePlan PR is still open and only covers adjacent harness/plugin-runtime work.

## What This Changes

The issue requests a stable documented library/API package for running OpenClaw's agent loop, tools, skills, and memory abstractions in-process without the daemon.

## Best Possible Solution

Decide whether embeddable core belongs as a new versioned package or as an extension of the existing SDK boundaries, then land a minimal approved API with docs and contract tests.

## Reproduction Assessment

Not applicable; this is an API/package RFC, not a bug report. Source and docs inspection shows current main has Gateway-backed SDK access and plugin-scoped embedded helpers, but no public @openclaw/core-style in-process contract.

## Solution Assessment

Unclear until maintainers choose the public boundary. A narrow versioned package could be maintainable, but the relationship to @openclaw/sdk, openclaw/plugin-sdk, auth/profile policy, tool registration, lifecycle, and config ownership needs API direction first.
