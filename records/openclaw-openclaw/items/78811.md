## Summary

Keep open: current main still offers a Gateway-backed App SDK and in-process Plugin SDK/runtime helpers, but no stable external @openclaw/core-style in-process package. The related RuntimePlan/Harness V2 PR and the narrower embedding-provider SDK issue do not close this broader API decision.

## What This Changes

Request a stable documented @openclaw/core-style library entry point for in-process agent-loop, tool, skill, and memory use without daemon IPC.

## Best Possible Solution

Maintainers should explicitly choose whether external consumers stay on @openclaw/sdk/Gateway or design a narrow versioned embeddable package with docs and contract tests.

## Reproduction Assessment

Not applicable; this is a feature/API RFC, not a bug report. Static source and docs inspection gives high confidence that current main lacks the requested @openclaw/core-style in-process contract.

## Solution Assessment

Unclear until maintainers choose the public boundary. A narrow additive package may be maintainable, but it must be reconciled with @openclaw/sdk, plugin-sdk runtime helpers, harness ownership, and trusted/local agent command boundaries first.
