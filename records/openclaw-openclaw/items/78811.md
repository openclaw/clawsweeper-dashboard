## Summary

Keep open: this is a broad public API and packaging RFC, not an obsolete or already-implemented request. Current main exposes Gateway-backed app SDK and plugin-scoped SDK/runtime seams, but no stable in-process embeddable core package; the later comment pointing at pending PRs is useful context, but those PRs are not merged current-main evidence for this request.

## What This Changes

The issue requests a stable documented embeddable core entry point, tentatively `@openclaw/core`, for in-process agent loop, tool execution, skill invocation, and related public types.

## Best Possible Solution

Decide whether embeddable core belongs as a new stable package or as a narrower extension of the existing App SDK and Plugin SDK boundaries, then land a versioned minimal API with docs and contract tests if approved.

## Reproduction Assessment

Not applicable; this is a feature/API RFC rather than a bug report. Source and docs review confirms current main has Gateway-backed and plugin-scoped paths but no public `@openclaw/core`-style in-process contract.

## Solution Assessment

Unclear until maintainers choose the public boundary. A narrow versioned package could be maintainable, but package ownership, lifecycle/config/tool contracts, and its relationship to `@openclaw/sdk` and `openclaw/plugin-sdk` need API direction first.
