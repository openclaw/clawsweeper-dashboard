## Summary

Keep open: current main still lacks the requested stable in-process core package/API, and the adjacent SDK/runtime work covers Gateway-backed apps or plugin-scoped helpers rather than this broader public contract.

## What This Changes

Request a stable documented library entry point, such as `@openclaw/core`, for running the agent loop, tools, skills, and memory in process without the daemon.

## Best Possible Solution

Choose the supported public boundary first; if approved, land a minimal versioned in-process API with docs and contract tests, otherwise document that embedders should use the Gateway-backed App SDK or plugin SDK surfaces.

## Reproduction Assessment

Not applicable: this is a feature/API RFC, not a bug report. Source and docs review confirm current main has Gateway-backed and plugin-scoped paths, but no public `@openclaw/core`-style in-process contract.

## Solution Assessment

Unclear until maintainers choose the API boundary. A narrow stable package could be maintainable, but implementing it now would decide package ownership, lifecycle, config, tool, skill, and memory contracts without maintainer direction.
