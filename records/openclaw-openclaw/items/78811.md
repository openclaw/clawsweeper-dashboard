## Summary

Keep open: this is a broad new public API and packaging RFC. Current main has a Gateway-backed App SDK and plugin-scoped in-process seams, but no stable embeddable core package; deciding that boundary needs maintainer API direction.

## What This Changes

Expose a stable importable @openclaw/core-style API for the agent loop, tool execution, skills, and memory so consumers can embed OpenClaw in-process without the Gateway daemon.

## Best Possible Solution

Decide whether embeddable core belongs as a new stable package or as improvements to the existing App SDK and Plugin SDK boundaries, then land a versioned minimal API with docs and contract tests if approved.

## Reproduction Assessment

Not applicable; this is a feature/API RFC, not a bug report. Source and docs review confirms current main has Gateway-backed and plugin-scoped paths but no public @openclaw/core-style in-process contract.

## Solution Assessment

Unclear until maintainers choose the public boundary. A narrow versioned package could be maintainable, but package ownership, lifecycle/config/tool contracts, and its relationship to @openclaw/sdk and openclaw/plugin-sdk need product/API direction first.
