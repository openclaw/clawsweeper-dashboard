## What This Changes

The issue requests a stable documented embeddable OpenClaw core library entry point for in-process agent loop, tool execution, and skill invocation.

Maintainers should make an explicit API-direction call: either define and document one narrow stable embeddable core contract, or close with a product decision that external apps should use the Gateway App SDK and in-process extension should stay plugin-scoped.
