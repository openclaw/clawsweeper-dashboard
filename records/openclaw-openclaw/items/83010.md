## What This Changes

The PR changes the MCP loopback and `/tools/invoke` Gateway HTTP handlers to construct request URLs with a literal `http://localhost` base instead of a Host-header-derived base.

## Best Possible Solution

Merge this focused Gateway hardening after redacted live request proof exercises both changed handlers, ideally with focused malformed-Host regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.11` derive URL parser bases from `headers.host` in both changed handlers, so malformed Host input can affect route parsing before handler logic.

## Solution Assessment

Yes for the code shape: both handlers only need path/query parsing, and a literal localhost base matches sibling Gateway handlers. The PR still needs real Gateway request proof before merge.
