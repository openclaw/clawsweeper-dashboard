## What This Changes

The branch changes the MCP loopback and tools-invoke Gateway HTTP handlers to construct request URLs with a fixed `http://localhost` base instead of using `Host`.

## Best Possible Solution

Land this focused hardening after adding real handler proof for `/mcp` and `/tools/invoke` with malformed Host input, ideally with matching regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main routes both changed handlers through Host-derived URL bases, and a malformed Host-derived base makes URL parsing throw before normal route handling. I did not run live Gateway requests in this read-only review.

## Solution Assessment

Yes for code shape: replacing the URL base with `http://localhost` is the narrow maintainable fix because these handlers only use the parsed path. It is not merge-ready until real handler proof is added.
