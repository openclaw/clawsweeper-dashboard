## Summary

Keep this PR open: current main still has the Host-derived URL base in the two changed Gateway handlers, and the patch is the narrow, consistent hardening, but the external real behavior proof still does not show the patched handlers processing real `/mcp` and `/tools/invoke` requests.

## What This Changes

The PR changes `src/gateway/mcp-http.request.ts` and `src/gateway/tools-invoke-http.ts` to parse request URLs with a fixed `http://localhost` base instead of deriving the base from the `Host` header.

## Best Possible Solution

Land the focused Gateway hardening after redacted real handler proof is attached; adding matching malformed-Host regression tests would make the invariant durable.

## Reproduction Assessment

Yes, source-reproducible: current main builds URL bases from `Host`, and a malformed Host-derived base throws before normal route handling. I did not run live Gateway requests in this read-only review.

## Solution Assessment

Yes for code shape: a fixed localhost base is the narrow maintainable fix because these handlers only use the parsed path and search params. It is not proof-ready until real handler evidence is attached.
