## Summary

Keep open: current main and the latest release still use Host-derived URL bases in the two changed Gateway handlers, and the patch is a narrow, correct hardening direction, but this external PR still needs real OpenClaw handler proof before merge.

## What This Changes

The PR changes two Gateway HTTP handlers to construct route URLs with a fixed `http://localhost` base instead of using the request `Host` header.

## Best Possible Solution

Land this focused localhost-base hardening after adding real Gateway proof for `/mcp` and `/tools/invoke` with hostile or malformed Host headers, ideally with focused regression tests.

## Reproduction Assessment

Yes, source-reproducible: current main routes both changed handlers through Host-derived URL bases, and a malformed Host value makes the URL constructor throw before normal route handling. I did not run live Gateway handler requests in this read-only review.

## Solution Assessment

Yes for code shape: replacing the base with `http://localhost` is the narrow maintainable fix because these handlers use the parsed path, not the parsed host. It is not merge-ready until real handler proof is added.
