## Summary

Keep open: the patch is a narrow and correct hardening for the two remaining Host-derived gateway URL parsers on current main, but the external PR still needs real handler proof before merge.

## What This Changes

The PR replaces Host-header-derived URL constructor bases with "http://localhost" in `validateMcpLoopbackRequest` and `handleToolsInvokeHttpRequest`.

## Best Possible Solution

Land this focused localhost-base hardening with actual gateway proof and preferably focused malformed-Host coverage for MCP and tools-invoke, then close the superseded older branch at https://github.com/openclaw/openclaw/pull/59696.

## Reproduction Assessment

Yes. Source inspection on current main shows both handlers still build routing URLs from the Host header, so a direct `/mcp` or `/tools/invoke` request with malformed `Host: [` should hit the current 400 path before route handling.

## Solution Assessment

Yes for code shape. Replacing the base with `http://localhost` is the narrow maintainable fix because these handlers consume the pathname, but the PR is not merge-ready until real handler proof and ideally focused coverage are added.
