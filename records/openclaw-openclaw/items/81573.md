## Summary

Keep this PR open, but do not merge it as-is: it targets a credible Xiaomi MiMo replay bug, yet the current branch has a parse-breaking syntax error, the new wrapper does not match the Xiaomi model IDs currently shipped by OpenClaw, and the posted proof only exercises raw Xiaomi API payloads rather than OpenClaw after the patch.

## What This Changes

The PR adds Xiaomi provider compat defaults for DeepSeek-style thinking and registers a Xiaomi stream wrapper intended to inject `reasoning_content` on replayed MiMo assistant messages.

## Best Possible Solution

Keep Xiaomi-specific replay payload handling in the Xiaomi plugin through the existing `wrapStreamFn` seam, match the shipped Xiaomi model catalog, add focused wrapper coverage, and require redacted OpenClaw runtime proof before merge.

## Reproduction Assessment

No high-confidence end-to-end current-main reproduction was run in this review. Source inspection shows Xiaomi currently lacks the replay wrapper path, and the PR's redacted curl output makes the API requirement credible, but it does not exercise OpenClaw itself.

## Solution Assessment

No. The owner boundary is right, but this patch must first be made parseable, must match the current Xiaomi model IDs, and must demonstrate the fixed OpenClaw runtime path.
