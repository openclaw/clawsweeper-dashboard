## Summary

Keep open because current main and the latest stable release still keep session tools and gateway protocol local-only, while the linked implementation PR is open, conflicting, and still needs product/security review before it can close the issue.

## What This Changes

Request native cross-gateway session messaging so `sessions_send` and related session flows can target agents or sessions on another OpenClaw Gateway instead of relying on SSH or external-channel workarounds.

## Best Possible Solution

Keep this issue open as the concrete session-tool tracker, decide the gateway-to-gateway trust/addressing/session ownership model, then land or replace #43656 with real two-gateway proof.
