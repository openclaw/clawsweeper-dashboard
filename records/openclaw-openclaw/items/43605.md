## Summary

Keep open. Current main still has local-only session send/spawn surfaces and local CLI agent validation, and PR #43656 is an open implementation candidate with closing syntax for this issue, so cleanup closure would be premature.

## What This Changes

Request native cross-gateway routing for `sessions_send` and `sessions_spawn` so agents on separate Gateway instances can message each other without SSH scripts.

## Best Possible Solution

Keep the tracker open until an additive, security-reviewed cross-gateway session routing design lands with remote target configuration, auth resolution, transcript ownership, regression tests, and docs while preserving local defaults.
