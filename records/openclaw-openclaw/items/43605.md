## Summary

Keep open: current main still has local-only session tools and no Gateway peer registry, while the same-author implementation PR that closes this issue is still open and conflicting. The remaining work is a product and security decision around cross-Gateway trust, routing, and live two-Gateway proof, not conservative cleanup.

## What This Changes

Request cross-Gateway delivery for `sessions_send` and related session delegation so agents on different Gateway instances can message each other without SSH workarounds.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/43656 lands with an approved trust model and real two-Gateway proof, or maintainers explicitly fold the narrower session-tool request into https://github.com/openclaw/openclaw/issues/46724.

## Reproduction Assessment

Yes by source inspection: current main exposes local-only `sessions_send`, local `sessions_spawn`, local agent validation in the CLI, and no Gateway peer registry. This confirms a feature gap rather than a broken documented existing contract.

## Solution Assessment
