## Summary

Current main still has the reported single-gateway limitation, and the same-author closing PR remains open with unresolved security and behavioral concerns. This is a valid feature request that needs maintainer product/security judgment rather than cleanup or a new automated fix PR.

## What This Changes

Add native authenticated cross-gateway routing so `sessions_send` and `sessions_spawn` can target agents and sessions hosted by another OpenClaw gateway.

## Best Possible Solution

Adopt one canonical authenticated gateway-to-gateway contract with scoped identity, authorization, addressing, idempotency, delivery/failure semantics, and full session-tool parity, then repair or replace the linked PR against that contract with upgrade and live multi-machine proof.

## Reproduction Assessment

Yes at source level. Current main's session tools enumerate and authorize local agents and sessions and have no native remote-gateway target, matching the reported multi-machine limitation.

## Solution Assessment

No, not yet. Direct peer parameters are a plausible prototype, but the best fix must first define a shared gateway identity/security/delivery contract and preserve the complete `sessions_send` and `sessions_spawn` semantics.
