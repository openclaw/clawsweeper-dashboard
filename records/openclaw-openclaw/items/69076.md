## Summary

Keep open: current main still has the reported cancellation behavior, and the PR is directionally useful, but one process-status path should be tightened before merge so non-manual signal exits are not treated as successful cancellations.

## What This Changes

The PR classifies explicit exec/process cancellations as `killed`, suppresses killed notify-on-exit wakeups, makes successful kill/remove actions complete, adds regression coverage and changelog entries, and persists Telegram sent-message metadata.

## Best Possible Solution

Land the PR after narrowing process terminal status resolution to explicit manual cancellation or explicit kill/remove fallback semantics while preserving the added regression coverage and changelog entries.

## Reproduction Assessment

Yes. Source inspection of current main shows `manual-cancel` exits still fall into the failed exec path and process kill/remove returns failed details; the PR adds focused tests for the intended cancellation behavior, though I did not run them in this read-only pass.

## Solution Assessment

No, not as written. The main approach is the right narrow fix, but `resolveTerminalProcessStatus` should use the supervisor's explicit termination reason instead of raw `SIGTERM`/`SIGKILL` alone.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Only classify explicit cancellations as killed:** `src/agents/bash-tools.process.ts:65-66`
  - body: `resolveTerminalProcessStatus` returns `killed` for any `SIGTERM` or `SIGKILL`, and the caller then maps `killed` to a completed tool result. The supervisor already distinguishes `manual-cancel` from timeout and generic `signal` exits, so a still-scoped process that terminates by signal for a non-manual reason can be reported as a successful cancellation instead of a failure. Please key this on `session.exitReason === "manual-cancel"` or the explicit kill/remove fallback path, and keep timeout/signal failures on the failed path.
  - confidence: 0.82
