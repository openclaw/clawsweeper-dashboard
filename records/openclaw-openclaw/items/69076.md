## Summary

Keep open: current main still has the reported manual-cancel failure behavior, and this PR is useful, but the proposed patch still has narrow merge-blocking issues around cancellation classification and Telegram cache compatibility.

## What This Changes

This PR adds a killed outcome for manual exec/process cancellations, suppresses notify-on-exit for killed sessions, returns completed kill/remove control results, and stores Telegram sent-message metadata.

## Best Possible Solution

Refine the PR so killed is derived from explicit manual cancellation or the known fallback kill path, preserve Telegram cache store selection while adding metadata, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows manual-cancel exits flow through buildExecExitOutcome as failed, notify-on-exit still wakes for manual-cancel with output, and process kill/remove returns failed details after successful termination requests.

## Solution Assessment

No, not as currently proposed. The direction is right, but the patch should gate killed status on explicit cancellation and merge Telegram metadata into the existing config-scoped cache API instead of replacing that argument.
