## What This Changes

The PR treats Telegram bot/self reply targets as allowed supplemental quote/forwarded context under group `contextVisibility: "allowlist"` and adds a regression test for preserving `ReplyToBody`.

## Best Possible Solution

Land this focused fix after live or equivalent Telegram proof, or an explicit maintainer proof override, confirms bot/self reply context is preserved and non-bot supplemental context remains filtered; leave the broader durable reply-metadata contract to https://github.com/openclaw/openclaw/issues/88032 and https://github.com/openclaw/openclaw/pull/90745.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes for the narrow bug. The Telegram supplemental-context gate is the right owner boundary and matches the existing reply-to-bot implicit mention behavior, while the broader durable reply-context architecture remains separate.
