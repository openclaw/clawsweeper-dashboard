## Summary

Keep open: the Android copy/select/reply UX is still absent from current main, but this PR is not mergeable because the latest head fails Android compilation, reply quote formatting is still lossy, and the supplied proof only shows visual states.

## What This Changes

The PR adds Android chat-bubble long-press actions for copy, select-copy, share, and reply, wires a reply preview into the composer, and prepends reply context to outgoing text.

## Best Possible Solution

Land a rebased Android-only implementation that compiles, preserves chat scrolling/gesture behavior, serializes reply context line-safely, and supplies real Android proof before closing https://github.com/openclaw/openclaw/issues/57754.

## Reproduction Assessment

Yes. Current main source shows only code blocks are selectable and no long-press message action path exists; PR-head CI also reproduces the compile failure from duplicate imports and the missing `Box` import.

## Solution Assessment

No. The bottom-sheet direction is plausible, but this exact patch needs compile cleanup, line-safe reply quote serialization, and stronger Android behavior proof before it is the maintainable fix.
