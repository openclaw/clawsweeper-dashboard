## Summary

Keep open. Current main still lacks the Android chat message copy/select/share/reply surface, and this PR remains useful but needs changes before merge because the reply quote formatter mishandles multi-line quoted text, the changelog entry is missing, and GitHub reports the branch as dirty.

## What This Changes

The PR adds long-press Android chat-bubble actions for Copy, Select Copy, Share, and Reply, plus a reply preview and quoted-message send path in the composer.

## Best Possible Solution

Land a corrected narrow Android chat interaction implementation with per-line reply quote formatting, focused coverage, a changelog entry, and the linked feature issue left open until the implementation merges.

## Reproduction Assessment

Yes. Current main can be verified in source because normal markdown text renders outside SelectionContainer while code blocks are selectable, and the PR defect is reproduced by replying to a message whose extracted text contains multiple lines.

## Solution Assessment

No. The bottom-sheet approach is a reasonable scoped solution, but this exact patch is not the best mergeable form until it serializes multi-line reply quotes correctly, adds the required changelog entry, and resolves conflicts against current main.
