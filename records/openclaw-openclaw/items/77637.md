## Summary

Keep open: current main has partial session navigation through Chat and metadata/previews in Sessions, but no dedicated transcript browser or raw transcript inspection UI; the linked issues are adjacent, not duplicates.

## What This Changes

Request a Control UI transcript browser for viewing historical session `.jsonl` transcript content across sessions.

## Best Possible Solution

Keep this as a Control UI feature/design item and build a bounded, authorized transcript/history viewer only after deciding the UX, raw-vs-projected data model, and redaction expectations.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior. Source inspection verifies current main has session metadata, previews, usage/log detail, and selected-session Chat history, but not a dedicated transcript browser.

## Solution Assessment

Unclear; a full Transcripts tab is plausible, but a safer first step may be a per-session view action that reuses existing `chat.history` or session-history APIs with explicit bounds and redaction. Maintainer UX and privacy decisions should come before implementation.
