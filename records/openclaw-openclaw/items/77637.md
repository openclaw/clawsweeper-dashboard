## Summary

Keep open: current main exposes session metadata, bounded previews, projected chat history, usage, logs, and trajectory export, but it does not provide a Control UI transcript browser for arbitrary stored session transcript files. This is a new product/UI capability, not a stale bug or a direct duplicate of the related open items.

## What This Changes

Add a Control UI transcript browser or session-row action for viewing stored session transcript content beyond the currently active chat.

## Best Possible Solution

Define a scoped Control UI transcript-history experience around existing session history plumbing, with explicit limits, sanitization/privacy behavior, and a clear choice between a Sessions row action and a dedicated tab.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source and docs inspection confirm the current UI has session metadata and chat history surfaces but no transcript browser.

## Solution Assessment

Unclear pending maintainer direction; a transcript browser may be the right product direction, but the best fix depends on whether it should show raw JSONL, projected chat history, export bundles, or usage-linked views.
