## Summary

Keep open: current main has substantial transcript-backed Usage UI support, including session discovery, conversation logs, historical lineage, and related closed export/history issues, but it still does not clearly provide the dedicated arbitrary transcript browser requested here. Closing would require a maintainer decision that the existing Usage conversation view is the intended product answer.

## What This Changes

The issue requests a Control UI transcript browser for viewing stored session transcript files from any session, not only the active chat session.

## Best Possible Solution

Define whether the existing Usage conversation view is the intended transcript-history surface; if not, add a scoped Control UI transcript view that reuses existing session discovery/log APIs with clear caps, redaction, and raw-vs-projected behavior.
