## Summary

Keep open: current main has session metadata, bounded previews, projected chat history, usage-derived conversation snippets, and gateway logs, but it does not provide a Control UI browser for arbitrary stored session transcript files. This is a new product/UI capability with privacy and scoping choices, not a bug, stale report, or direct duplicate.

## What This Changes

Request a Control UI transcript browser or Sessions row action for viewing stored session transcript files across arbitrary sessions.

## Best Possible Solution

Define a scoped Control UI transcript-history experience around existing session history and usage plumbing, with explicit limits, authorization, sanitization/privacy behavior, and a clear UX choice between a Sessions row action and a dedicated tab.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken current behavior. Source and docs inspection on current main confirms there is no transcript browser tab or Sessions-row transcript action.

## Solution Assessment

Unclear pending maintainer direction; a transcript browser may be the right direction, but the safest design needs to choose projected history versus raw JSONL, bounds, access scope, and whether to integrate with Sessions or Usage first.
