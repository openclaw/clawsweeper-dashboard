## Summary

Keep open: current main has related session metadata, bounded previews, projected chat history, usage views, logs, and active-chat Markdown export, but it still does not provide the requested Control UI browser for arbitrary stored session transcript files. The closed export request at https://github.com/openclaw/openclaw/issues/81584 is narrower and does not supersede this broader transcript-browsing feature.

## What This Changes

Add a Control UI surface for browsing transcript contents from stored session transcript files across sessions, not only the active chat.

## Best Possible Solution

Define a scoped Control UI transcript-history experience around existing session plumbing, with explicit limits, redaction/privacy behavior, and a maintainer choice between a Sessions-row action and a dedicated tab.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection confirm current main has related bounded history and export surfaces, but no dedicated arbitrary transcript browser.

## Solution Assessment

Unclear pending maintainer direction: a transcript browser may be useful, but the maintainable design depends on whether it should show raw JSONL, projected chat history, trajectory-style redacted bundles, or usage-linked views.
