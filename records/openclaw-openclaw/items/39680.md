## Summary

Keep open: current main still warns on the value-only 80+ character base64-like heuristic, and the existing sanitizer test preserves that warning on a non-credential key. The remaining choice is a sandbox/secops heuristic decision, not cleanup-close material.

## What This Changes

Reduce false-positive sandbox env-var warnings by tightening `validateEnvVarValue`'s long base64-like value heuristic.

## Best Possible Solution

Land a focused secops-reviewed sanitizer change with regression coverage for long alphanumeric, hex-like, and JWT-segment-like non-credential values while preserving null-byte blocking, max-length handling, explicit credential-name blocking, and accepted credential warnings.
