The issue asks to reduce false-positive sandbox credential warnings from the value-only 80+ base64-character heuristic in validateEnvVarValue.

## Best Possible Solution

Land a secops-reviewed sanitizer update with regression coverage for long alphanumeric, hex-like, and JWT-like non-credential values while preserving null-byte blocking, maximum-length handling, explicit credential-name blocking, and useful encoded-secret warnings.

## Solution Assessment

Unclear. Reducing false positives is the right direction, but the exact replacement heuristic needs secops/product judgment because structural validation and key-name gating make different false-positive and false-negative tradeoffs.
