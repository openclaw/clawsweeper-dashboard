## Summary

This is a valid BlueBubbles plugin request, but current main has removed BlueBubbles from the bundled OpenClaw channel surface and explicitly routes BlueBubbles-backed continuation to iMessage migration or ClawHub/plugin distribution.

## What This Changes

Add a BlueBubbles channel config option to suppress outbound message-id receipt system events while keeping reply-cache and pending-outbound maintenance active.

## Best Possible Solution

Keep BlueBubbles out of core, use the supported iMessage migration path, and let any continued BlueBubbles behavior live in a ClawHub/npm channel plugin that owns this config knob.

Not applicable: this is a request for a new config option on a deprecated/removed BlueBubbles channel, and current main has no bundled BlueBubbles runtime to exercise.

## Solution Assessment

No: adding a new `channels.bluebubbles.*` key to OpenClaw core would re-expand a removed channel surface; the maintainable path is iMessage migration or a ClawHub BlueBubbles plugin that owns this setting.
