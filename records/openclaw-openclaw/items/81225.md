## Summary

Keep this PR open: the intended Telegram plugin-bound topic fix is valid, and the latest head appears to have addressed the earlier route metadata regression, but it still disables Telegram mention gating before the plugin claim outcome is known, so unmentioned group/topic messages can fall through to ordinary agent processing when the plugin is missing or has no handler.

## What This Changes

The PR adds a plugin-owned runtime binding signal to Telegram routing, bypasses mention gating for plugin-bound topics, allows plugin claims under `message_tool_only`, and adds focused regression tests.

## Best Possible Solution

Keep the `message_tool_only` plugin-claim behavior, but carry enough binding and mention-state through the turn so handled plugin claims bypass Telegram mention gating while plugin fallback paths still honor the configured group/topic activation policy.

## Reproduction Assessment

Yes. By source inspection, current main can skip a plugin-owned binding claim under `message_tool_only`, and the latest PR head source-reproduces the remaining fallback regression through `requireMention=false` plus `missing_plugin`/`no_handler` fallthrough.

## Solution Assessment

No. The dispatch fix is in the right area, but the mention bypass should be conditional on a handled plugin claim or should prevent ordinary agent fallback for unmentioned plugin-bound group turns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep the mention gate on plugin fallback:** `extensions/telegram/src/bot-message-context.ts:431-433`
  - body: This disables `requireMention` for every group message with a plugin-owned runtime binding before dispatch knows whether the plugin can handle the turn. If the plugin is missing or has no `inbound_claim` handler, dispatch falls through to the ordinary routed agent path, so an unmentioned topic message that used to be dropped under `requireMention` can now be processed. Preserve the configured mention decision for fallback, or stop ordinary fallback processing for unmentioned plugin-bound turns.
