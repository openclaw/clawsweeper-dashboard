## Summary

Keep open: the PR targets a real Telegram plugin-bound topic routing bug, but the latest head still disables Telegram mention gating before the plugin claim succeeds, so fallback paths can process unmentioned group/topic traffic.

## What This Changes

The PR adds Telegram plugin-owned runtime binding detection, bypasses mention gating for those bound routes, allows plugin claims under `message_tool_only`, and adds focused routing/dispatch tests.

## Best Possible Solution

Keep the plugin-bound topic routing fix, but separate successful plugin-claim bypass from ordinary fallback so configured Telegram group/topic mention policy remains the activation boundary.

## Reproduction Assessment

Yes. Source inspection shows current main can skip a plugin-owned claim under `message_tool_only`, the PR body supplies live after-fix logs, and the latest PR head source-reproduces the remaining fallback regression.

## Solution Assessment

No. The dispatch fix is in the right area, but unconditionally setting `requireMention=false` before claim success is too broad; the safer fix carries mention/binding state through dispatch and only bypasses gating for a handled plugin claim or suppresses ordinary fallback.

## Review Findings

Overall correctness: patch is incorrect
