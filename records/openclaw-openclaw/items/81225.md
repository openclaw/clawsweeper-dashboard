## Summary

Keep open: the PR addresses a real current-main Telegram plugin-bound topic routing bug and includes usable live proof, but the latest head still bypasses Telegram mention gating before the plugin claim succeeds, allowing missing-plugin or no-handler fallback to process unmentioned group/topic messages.

## What This Changes

The PR adds a Telegram plugin-owned runtime-binding signal, lets those bound routes bypass mention gating, allows plugin claims under `message_tool_only`, and adds focused routing and dispatch tests.

## Best Possible Solution

Keep the `message_tool_only` plugin-claim path, but carry enough binding and mention state so handled plugin claims bypass Telegram mention gating while missing-plugin and no-handler fallback still honor configured group/topic activation policy.

## Reproduction Assessment

Yes, source-reproducible. Current main can skip plugin-owned claims under `message_tool_only`, and the PR head source shows unmentioned plugin-bound group messages can fall through to ordinary agent processing when the plugin claim is unavailable.

## Solution Assessment

No. The dispatch fix is in the right area, but the Telegram mention bypass is too early; the maintainable fix should preserve the configured mention gate for plugin fallback or stop fallback processing for unmentioned plugin-bound turns.
