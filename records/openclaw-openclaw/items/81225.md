## Summary

Keep this PR open: it fixes a real current-main Telegram plugin-bound routing bug, but the latest diff still weakens mention-gated group/topic fallback behavior when the bound plugin cannot claim the turn.

## What This Changes

The PR adds Telegram plugin-owned runtime-binding routing state, bypasses mention gating for those bound topics, allows plugin claims under `message_tool_only`, and adds focused routing/dispatch tests.

## Best Possible Solution

Keep the plugin claim under `message_tool_only`, but carry enough binding and mention-state so handled plugin claims bypass Telegram mention gating while plugin fallback paths still honor configured group/topic activation policy.

## Reproduction Assessment

Yes, at source level. Current main skips plugin-owned binding claims under `message_tool_only`, and the latest PR head source-reproduces the remaining fallback regression through `requireMention=false` plus `missing_plugin`/`no_handler` fallthrough.

## Solution Assessment

No. The dispatch fix is in the right area, but the Telegram mention bypass should be conditional on a handled plugin claim or should make unmentioned plugin-bound fallback terminal instead of reaching ordinary agent processing.
