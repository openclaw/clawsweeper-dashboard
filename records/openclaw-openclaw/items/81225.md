## Summary

Keep open: the PR targets a real current-main routing bug, but the latest head still bypasses Telegram mention gating before the plugin claim succeeds, so missing-plugin or no-handler fallback can process unmentioned group/topic messages as ordinary agent turns.

## What This Changes

The PR marks plugin-owned Telegram runtime bindings, bypasses mention gating for those routes, allows plugin claims under `message_tool_only`, and adds routing/dispatch regression tests.

## Best Possible Solution

Carry enough plugin-bound and mention-state through the Telegram turn so handled plugin claims bypass mention gating, while missing-plugin/no-handler fallback either stops or honors the original group/topic activation policy.

## Reproduction Assessment

Yes. Current main source shows a plugin-owned binding claim is skipped under `message_tool_only`, and the PR head source-reproduces a remaining fallback regression through `requireMention=false` plus `missing_plugin`/`no_handler` fallthrough.

## Solution Assessment

No. The `message_tool_only` dispatch fix is in the right area, but the Telegram mention bypass needs to be limited to successful plugin ownership or paired with a fallback stop so ordinary unmentioned group traffic remains gated.
