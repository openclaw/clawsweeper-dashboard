## Summary

Keep open: this is a useful UX feature PR, but the current branch still hard-codes an unavailable approval decision in some policy states and lacks required real behavior proof for the visible chat flow.

## What This Changes

The branch changes exec approval forwarding fallback text to list full `/approve` commands with Markdown-aware wrapping and adds forwarder tests for Markdown and non-Markdown channels.

## Best Possible Solution

Generate fallback command lines from the shared allowed-decision/action descriptor contract, preserve existing native and interactive channel behavior, and require redacted real Telegram or equivalent chat proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a UX feature PR. Source inspection confirms current main's generic policy-aware fallback and the branch's hard-coded command list.

## Solution Assessment

No, not as-is. The UX direction is narrow and useful, but fallback commands should be derived from the resolved allowed decisions instead of hard-coding `allow-always`.
