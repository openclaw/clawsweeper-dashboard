## Summary

Keep open: the UX direction is useful, but the PR still lacks contributor-supplied real behavior proof and its fallback text hard-codes an unavailable approval action. ## What This Changes

The PR replaces generic exec approval forwarding fallback text with copyable `/approve ...` command lines, skips Markdown backticks on non-Markdown channels, and updates forwarder tests.

## Best Possible Solution

## Solution Assessment
