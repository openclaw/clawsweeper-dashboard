## Summary

Keep open: the UX improvement is plausible and the linked issue is already closed in favor of this PR, but the branch should not merge until the command rows follow the allowed-decision policy and external real behavior proof is posted.

## What This Changes

This PR replaces the generic exec-approval forwarding hint with copyable `/approve <id> <decision>` rows and adds a non-Markdown channel test.

## Best Possible Solution

Generate fallback command rows from the shared allowed-decision descriptors, preserve channel-aware Markdown/plain formatting, cover `ask=always`, and require redacted real chat proof before merge.

## Reproduction Assessment

Not applicable as a current-main bug reproduction; this is a UX feature PR. The merge-blocking defect is source-reproducible from the branch because it hard-codes `allow-always` while current main excludes that decision for `ask=always`.

## Solution Assessment

No, not as-is. The maintainable version should derive displayed command rows from `resolveExecApprovalRequestAllowedDecisions` or `buildExecApprovalActionDescriptors` instead of hard-coding every decision.
