The branch replaces the exec approval forwarder’s generic reply hint with pre-filled `/approve ...` command lines and adds a unit test for non-markdown channel formatting.

## Best Possible Solution

Rebase or replace the PR with a small current-main patch that generates fallback approval commands from the current allowed decisions, keeps markdown wrapping channel-aware, and adds focused coverage plus a changelog entry.

## Reproduction Assessment

Not applicable as a feature/UX PR rather than a bug report. Source inspection shows current main still emits the generic fallback hint, and tests assert that behavior.

## Solution Assessment

No, not as written against current main. The UX direction is good, but the implementation should be rebased onto `buildExecApprovalRequestMessage` and generate commands from `resolveExecApprovalRequestAllowedDecisions` with conditional markdown wrapping.
