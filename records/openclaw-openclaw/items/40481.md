## Summary

## What This Changes

The PR replaces the generic exec approval fallback hint with concrete `/approve` command rows, skips Markdown backticks on non-Markdown channels, and adds focused forwarder tests.

## Best Possible Solution

Generate fallback command rows from the shared allowed-decision/action-descriptor helpers, preserve channel-aware formatting, and require redacted real chat proof before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a current-main bug because this is a UX enhancement, but the PR defect is source-reproducible: the added rows are hard-coded while `ask=always` only allows `allow-once` and `deny`.

## Solution Assessment

Is this the best way to solve the issue? No; the useful direction should be implemented by deriving commands from shared approval action descriptors so notification copy cannot drift from gateway policy.
