## Summary

Keep open: the UX change is still useful, but the branch is not mergeable as-is because it hard-codes an unavailable approval decision and still lacks contributor-provided real behavior proof.

## What This Changes

The PR changes forwarded exec approval fallback notifications and tests to show copyable full `/approve` commands, with Markdown wrapping only on Markdown-capable channels.

## Best Possible Solution

Generate fallback command rows from the shared allowed-decision descriptors, preserve current policy-aware copy and channel formatting, and require redacted real chat proof before merge.

## Reproduction Assessment

Not as a current-main bug; this is a UX feature PR. Yes for the merge-blocking branch defect: the PR head hard-codes all three decisions while current main excludes `allow-always` for `ask=always`.

## Solution Assessment

No, not as-is. The maintainable solution should derive displayed command rows from `resolveExecApprovalRequestAllowedDecisions` or `buildExecApprovalActionDescriptors` instead of hard-coding each decision.
