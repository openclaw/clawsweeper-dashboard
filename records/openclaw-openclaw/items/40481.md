## Summary

Keep open: the UX direction is still useful, but this external PR is not merge-ready because it lacks inspectable real Telegram/chat proof and the patch can advertise `allow-always` where current approval policy rejects it.

## What This Changes

The PR changes exec approval forwarding fallback text to show copyable `/approve` commands, skips Markdown backticks on non-Markdown channels, and updates forwarder tests.

## Best Possible Solution

Generate fallback command lines from the shared approval action descriptors or resolved allowed decisions, respect channel Markdown capability, and attach redacted real Telegram/chat proof for the visible notification flow.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a UX feature PR. Source inspection gives a high-confidence current-main comparison: the generic forwarded fallback still uses placeholder text, while the branch proposes concrete commands.

## Solution Assessment

No, not as-is. The direction is narrow and useful, but the implementation should derive commands from the shared allowed-decision/action contract so it does not advertise unavailable decisions.
