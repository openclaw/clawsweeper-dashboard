## Summary

Keep open: the requested UX is still not implemented in the generic forwarded fallback path, but this PR currently hard-codes `allow-always` and lacks contributor-supplied real behavior proof, so it is not ready to merge.

## What This Changes

The branch changes exec approval forwarded fallback text from a placeholder usage hint to prefilled `/approve` command lines and adds Markdown/non-Markdown channel tests.

## Best Possible Solution

Generate fallback command lines from the resolved allowed-decision/action descriptor contract, preserve the Markdown/plain-channel behavior, and attach real Telegram or chat proof for the approval flow.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a UX feature PR. Source inspection gives a high-confidence baseline: current main still has generic fallback text while shared approval actions already carry policy-aware commands.

## Solution Assessment

No, not as-is. The direction is narrow and useful, but the implementation should derive visible command text from the shared allowed-decision/action contract so it never advertises an unavailable decision.
