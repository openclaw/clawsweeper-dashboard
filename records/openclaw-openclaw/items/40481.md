## Summary

Keep this PR open, but it is not mergeable as-is: the UX direction is useful, current main has policy-aware approval decision contracts, and the branch still hard-codes an unavailable `allow-always` option while also lacking real chat proof.

## What This Changes

The PR changes exec approval forwarding fallback text to list full `/approve` commands with Markdown wrapping for capable channels and adds forwarder tests for Markdown and non-Markdown text.

## Best Possible Solution

Generate fallback command lines from the shared allowed-decision/action descriptor contract, preserve existing native and interactive approval behavior, and require redacted real Telegram or equivalent chat proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a UX feature PR. Source inspection gives a high-confidence comparison between current main's policy-aware generic fallback and the branch's hard-coded command list.

## Solution Assessment

No, not as-is. The command-list UX should be derived from resolved allowed decisions instead of hard-coding every decision, and visible chat behavior needs real proof before merge.
