## Best Possible Solution

## Reproduction Assessment

Not applicable as a current-main bug reproduction; this is a UX feature PR. The PR defect is source-reproducible from the branch diff plus current policy/tests: `ask=always` allows only `allow-once|deny` while the branch lists `allow-always`.

No, not as-is. The command-list UX should be implemented from `resolveExecApprovalRequestAllowedDecisions` or `buildExecApprovalActionDescriptors`, and merge still needs contributor-provided real chat proof.
