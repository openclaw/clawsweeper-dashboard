## Summary

Keep open: current main still has the generic forwarded fallback text, but this PR is not merge-ready because its fallback rows are hard-coded and can offer `allow-always` for `ask=always` approvals that current policy rejects.

## What This Changes

This PR changes exec approval forwarded notifications to show ready-to-copy `/approve` command rows and adds a non-Markdown channel test.

## Best Possible Solution

Generate visible fallback command rows from the shared allowed-decision descriptors, preserve channel-aware formatting, update focused tests, and merge only after redacted real chat proof shows the approval flow.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a current-main bug because this is a UX enhancement, but the PR defect is source-reproducible: the added rows are hard-coded while `ask=always` only allows `allow-once` and `deny`.

## Solution Assessment

Is this the best way to solve the issue? No; the fallback rows should be derived from the existing allowed-decision/action-descriptor helpers so notification copy cannot drift from gateway policy.
