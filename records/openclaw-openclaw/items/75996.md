## Summary

Current main has moved bundled plugin dependency work out of startup/runtime paths: plugin dependencies are installed by explicit install/update/doctor flows, postinstall no longer runs package-manager dependency repair, and legacy runtime-deps roots are treated as cleanup debris. That removes the shared generated manifest/install-root churn described in the report.

## What This Changes

The issue asks OpenClaw to stop shared bundled runtime dependency install roots from pruning each other's generated dependency manifests and repeatedly running npm during startup, Control UI, or channel loading.

## Best Possible Solution

Keep the current architecture: no package-manager repair from startup/runtime paths, explicit install/update/doctor ownership for dependencies, and doctor cleanup for legacy runtime-deps debris.

## Reproduction Assessment

Yes for the affected release class: the issue gives concrete macOS/npm-global symptoms and diagnostics for repeated npm add/remove churn. For current main, live reproduction is not applicable because the runtime package-manager path has been removed and the review verified that statically in source/docs/tests.

## Solution Assessment

Yes. Removing runtime package-manager repair and shared generated install roots is narrower and more maintainable than merging ever-growing manifests in a hot startup/channel path.
