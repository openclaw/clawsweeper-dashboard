## Summary

Current main resolves the reported churn by removing bundled dependency installation from startup/runtime paths and treating old `plugin-runtime-deps` roots as legacy cleanup state; the affected v2026.4.29 release remains the reported broken version, but the main-branch contract now avoids the shared manifest prune loop.

## What This Changes

The issue asks OpenClaw to stop repeated npm add/remove churn when multiple bundled runtime dependency plans share one install root and overwrite each other's generated manifest.

## Best Possible Solution

Keep the current main design: bundled startup must not run npm, old generated runtime-deps roots should be cleaned by doctor, and dependency convergence should stay in explicit install/update/doctor flows.

## Reproduction Assessment

Yes. The issue body and follow-up give concrete macOS and Ubuntu release reproductions for v2026.4.27/v2026.4.29, including alternating npm add/remove logs and missing-module failures; current-main verification here is source-level rather than a live npm replay.

## Solution Assessment

Yes. Eliminating startup-time shared runtime-deps manifests is more maintainable than merging mutable manifests in one shared root because it removes the prune loop instead of trying to make it converge.
