## Summary

Current main removes the shared bundled-runtime-deps staging model that could overwrite generated manifests and repeatedly invoke npm; the remaining `plugin-runtime-deps` references are legacy cleanup only, so the reported v2026.4.29 churn path is no longer present on main.

## What This Changes

Report asks OpenClaw to stop bundled runtime dependency plans from overwriting each other's generated manifests in one shared install root and causing repeated npm add/remove churn.

## Best Possible Solution

Keep the current main model: dependency installation stays in explicit plugin install/update/doctor flows, runtime startup stays read-only, and legacy dependency roots are cleaned by doctor rather than repaired during normal loading.

## Reproduction Assessment

Yes for the old release behavior: the report gives a concrete v2026.4.29 npm-global path, repeated startup/control-ui/channel-load trigger, and diagnostics. Current-main reproduction is not applicable because the shared `plugin-runtime-deps` runtime staging path has been removed from source.

## Solution Assessment

Yes. Removing runtime package-manager mutation and shared generated manifests is a safer maintainable fix than trying to make one shared staging manifest monotonically merge every observed dependency plan.
