## Summary

Current main has moved this behavior out of the startup/runtime path: bundled plugin dependency staging roots are treated as legacy debris, runtime loading no longer runs package-manager repair, and doctor cleanup removes old `plugin-runtime-deps` state. The reported v2026.4.29 churn is addressed on main, but no release tag contains the proof commit yet.

## What This Changes

The issue asks OpenClaw to stop repeated npm add/remove churn when different bundled runtime dependency plans share and overwrite one generated install root.

## Best Possible Solution

Keep the current main design: bundled plugin runtime dependency staging stays out of startup/runtime loading, dependency ownership stays with package manifests and explicit install/update/doctor flows, and old generated roots are cleaned by doctor repair.

## Reproduction Assessment

Yes for the obsolete release behavior: the report gives a concrete v2026.4.29 path with repeated npm work under `plugin-runtime-deps`. Against current main, the source-level reproduction is no longer applicable because the startup/runtime installer path and generated staging roots are legacy-only.

## Solution Assessment

Yes. Removing runtime package-manager staging and routing cleanup through `doctor --fix` is a more maintainable fix than trying to merge multiple generated manifests for one shared install root.
