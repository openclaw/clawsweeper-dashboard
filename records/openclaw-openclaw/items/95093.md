## Summary

Keep this PR open: it is a focused maintainer draft with a useful safety fix, but the patch changes the documented live-smoke boundary by moving the browser SDP HTTP exchange into Node.

## What This Changes

The PR routes the OpenAI Realtime WebRTC SDP answer request through a Node-side helper that uses the shared bounded response reader and adds regression coverage for unsafe decimal Content-Length values.

## Best Possible Solution

Resolve the smoke boundary first: either keep the SDP HTTP request browser-origin with safe length handling, or update the smoke/docs contract to make the Node bridge intentional, then record focused and live/exact-head validation.

## Reproduction Assessment

Yes. From source on current main, unsafe decimal Content-Length values do not satisfy Number.isSafeInteger in the browser-local reader, so they can enter the response body stream instead of rejecting before read; the PR body also reports direct before/after readStarted/canceled proof.

## Solution Assessment

Not as-is. Reusing the shared bounded reader is the right safety direction, but the current patch also changes the documented browser smoke boundary, so the best fix needs either browser-origin preservation or explicit docs/smoke contract updates.
