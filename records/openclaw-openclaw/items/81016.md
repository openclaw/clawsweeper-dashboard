Keep open for normal maintainer review and merge gating: the patch is a focused fix for the Control UI ordering bug, the linked issue remains open until a fix lands, and I found no concrete correctness or security finding in the diff.

## What This Changes

The PR changes the Control UI chat item builder to sort visible history, completed stream, divider, and live tool items by timestamp while keeping history notices first and active reading indicators last, with focused regression tests.

## Best Possible Solution

Land this centralized Control UI builder ordering fix after maintainer review and CI completion, and leave any remaining Telegram ordering behavior to a separate targeted follow-up if it still reproduces.

## Reproduction Assessment

Yes. A high-confidence source repro is to pass `buildChatItems` a newer history assistant message plus an older live tool or completed stream item; current main appends the older live item after history, so it renders out of timestamp order.

## Solution Assessment

Yes. Sorting the collected `ChatItem` list by visible timestamp in `buildChatItems` is the narrow central fix because `renderChat` consumes that order directly, and the patch keeps the hidden-history notice and active reading indicators in their special positions.
