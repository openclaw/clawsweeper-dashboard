## Summary

Keep open: the TUI `/new` regression is valid, but this PR still misattributes hook/lifecycle work to the fresh `tui-*` target, can persist blank sessions, and lacks contributor-provided live TUI hook proof.

## What This Changes

The PR adds `hookSourceKey` support to `sessions.reset`, routes TUI `/new` through `resetSession(nextSessionKey, "new", currentSessionKey)`, and updates gateway/TUI tests plus the changelog.

## Best Possible Solution

Preserve isolated `tui-*` switching while emitting `command:new`, `before_reset`, and lifecycle hooks for the session being left, using the parent-aware create hook seam or an equivalent hook-only path that avoids blank target rows.

## Reproduction Assessment

Yes. Source inspection shows current main’s TUI `/new` only switches local session state while docs and reset-hook tests define `command:new` as hook-visible, and the PR defects are source-reproducible from its reset target choice.

## Solution Assessment

No. The PR restores a command hook call, but it targets and persists the future `tui-*` key; the safer fix is to emit hooks for the session being left before switching to the isolated key.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit /new hooks for the session being left:** `src/gateway/session-reset-service.ts:281`
  - body: The new `hookSourceKey` only swaps `previousSessionEntry`; `performGatewaySessionReset` still builds the command and lifecycle events with the fresh target key. `command-logger` and `session-memory` can therefore attribute the ended transcript to the new empty `tui-*` session instead of the conversation that actually ended.
  - confidence: 0.9
- **[P3] Avoid creating blank TUI sessions on /new:** `src/tui/tui-command-handlers.ts:468`
  - body: Calling `sessions.reset` on the newly generated `tui-*` key writes a store row and transcript header before the user sends anything. Repeated `/new` commands can clutter history with empty sessions, so the hook path should run against the session being left or use a lazy/create seam.
  - confidence: 0.86
