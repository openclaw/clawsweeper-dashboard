## Summary

Keep open: current main still has the TUI `/new` hook regression, but this PR is not merge-ready because it emits and persists reset state against the fresh `tui-*` target, lacks real TUI hook proof, and leaves protocol mirrors out of sync if the new reset parameter remains.

## What This Changes

The PR adds `hookSourceKey` plumbing to `sessions.reset`, routes TUI `/new` through `resetSession(nextSessionKey, "new", currentSessionKey)`, and updates focused tests plus the changelog.

## Best Possible Solution

Preserve isolated TUI `tui-*` session switching while emitting `command:new`, `before_reset`, and lifecycle hooks for the session being left, using the current parent-aware hook/create pattern or a hook-only path that does not materialize an empty target row.

## Reproduction Assessment

Yes, source-level. Current main’s TUI `/new` only switches local session state, while docs and bundled hook metadata define `/new` as a `command:new` event and `/reset` still exercises the gateway reset path.

## Solution Assessment

No. The PR restores a hook call, but it still targets the future `tui-*` session for event identity and persistence; the narrower maintainable fix should emit against the previous session before switching to the isolated new key.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Emit /new hooks for the session being left:** `src/tui/tui-command-handlers.ts:467`
  - body: Calling `resetSession(nextSessionKey, "new", state.currentSessionKey)` still makes the fresh `agent:<agent>:tui-*` key the reset target. Hooks such as `command-logger` and `session-memory` consume `event.sessionKey`, so they can record the previous transcript under the brand-new session instead of the conversation that ended.
  - confidence: 0.91
- **[P3] Avoid persisting blank TUI sessions on /new:** `src/tui/tui-command-handlers.ts:467`
  - body: `sessions.reset` writes a store entry and transcript for its target. Because this call targets a newly generated `tui-*` key before any user message exists there, repeated `/new` commands can leave blank sessions in history; keep the target lazy or use a parent-aware hook path.
  - confidence: 0.86
- **[P2] Regenerate protocol mirrors for the reset parameter:** `src/gateway/protocol/schema/sessions.ts:97`
  - body: Adding `hookSourceKey` to `SessionsResetParamsSchema` changes the gateway wire contract, but the generated Swift protocol models are not updated. If this parameter remains, `protocol:check` and native clients will be out of sync with the schema.
  - confidence: 0.82
