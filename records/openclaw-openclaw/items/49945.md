## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep hook source selection off sessions.reset:** `src/gateway/protocol/schema/sessions.ts:248`
  - body: This adds `hookSourceKey` to public `sessions.reset`, so a caller can reset one key while causing hooks to read another session's context. Current main's parent-session hook path handles this internally without exposing arbitrary source selection; remove the public field or add a strict documented contract if maintainers choose it.
  - confidence: 0.86
- **[P2] Emit /new hooks for the session being left:** `src/tui/tui-command-handlers.ts:467`
  - body: The reset target is the fresh `tui-*` key while `hookSourceKey` points at the old session, so `event.sessionKey` remains the new key. Bundled hooks use that key when recording the ended conversation, which can mis-associate hook output with a blank future session.
  - confidence: 0.88
- **[P3] Avoid creating empty TUI sessions before messages:** `src/tui/tui-command-handlers.ts:467`
  - body: Calling `sessions.reset` on a never-used `tui-*` key creates a persisted entry before any message is sent. Repeated `/new` commands can clutter session lists with blank TUI sessions; emit the hook/lifecycle for the current session before switching instead.
  - confidence: 0.78
