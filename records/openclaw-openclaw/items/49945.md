## Summary

Keep open: this PR targets a valid TUI `/new` regression, but the branch still resets the future `tui-*` key, so hooks are attributed to the wrong session and the reset path can persist blank sessions; it also lacks after-fix real behavior proof.

## What This Changes

Adds `hookSourceKey` plumbing to `sessions.reset`, routes TUI `/new` through `resetSession(nextSessionKey, "new", currentSessionKey)`, and updates related tests plus the changelog.

## Best Possible Solution

Preserve isolated TUI `tui-*` switching while emitting `command:new`, `before_reset`, and lifecycle hooks for the session being left, using a parent-aware hook/create path or hook-only emission that does not materialize an empty target row.

## Reproduction Assessment

Yes, source-level. Current main’s TUI `/new` only calls local `setSession(uniqueKey)`, while docs and reset-hook tests establish `command:new` as the hook-visible `/new` contract.

## Solution Assessment

No. The PR restores a hook call, but it targets the future `tui-*` key instead of the session being left, so hook attribution and persistence still do not match the intended reset semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Emit /new hooks for the session being left:** `src/tui/tui-command-handlers.ts:467`
  - body: Calling `resetSession(nextSessionKey, "new", state.currentSessionKey)` still makes the fresh `agent:main:tui-*` key the reset target. Hooks and lifecycle work therefore see the new empty session as `event.sessionKey`; emit the hook path against the session being left before switching to the isolated TUI key.
  - confidence: 0.9
- **[P3] Avoid persisting blank TUI sessions on /new:** `src/tui/tui-command-handlers.ts:467`
  - body: `sessions.reset` persists a store entry and transcript header for its target. Since this call targets a newly generated `tui-*` key before any message is sent, repeated `/new` commands can clutter history with blank sessions; keep the target lazy or use a hook-only/parent-aware path.
  - confidence: 0.86
