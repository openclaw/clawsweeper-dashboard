## Summary

Keep this PR open for rework: the linked TUI hook regression still exists on current main, but this branch attributes the hook/reset lifecycle to the future `tui-*` session and needs real behavior proof before merge.

## What This Changes

The PR adds `hookSourceKey` plumbing to `sessions.reset`, routes TUI `/new` through `resetSession(nextSessionKey, "new", currentSessionKey)`, and updates related tests plus the changelog.

## Best Possible Solution

Preserve isolated TUI `tui-*` switching while emitting `command:new`, `before_reset`, and session lifecycle hooks for the session being left, preferably through the current parent-aware `sessions.create` hook seam or an equivalent hook-only path that avoids blank target rows.

## Reproduction Assessment

Yes, by source inspection. Current main’s TUI `/new` only calls local `setSession(uniqueKey)`, while docs and reset-hook tests establish `command:new` as the hook-visible `/new` contract.

## Solution Assessment

No. The PR restores a command hook call, but it resets the future session key and only borrows previous-session context, so hook identity, lifecycle attribution, and persistence remain tied to the wrong session.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Emit /new hooks for the session being left:** `src/tui/tui-command-handlers.ts:467`
  - body: Calling `resetSession(nextSessionKey, "new", state.currentSessionKey)` still makes the reset target the fresh `agent:main:tui-*` key. Hooks and lifecycle events see the new empty session as `event.sessionKey`, so this should fire the hook/lifecycle path against the session being left before switching to the isolated TUI key.
  - confidence: 0.9
- **[P3] Avoid persisting blank TUI sessions on /new:** `src/tui/tui-command-handlers.ts:467`
  - body: `sessions.reset` writes a store entry and transcript header for its target. Since this call targets a freshly generated `tui-*` key before any message is sent, repeated `/new` commands can clutter history with blank sessions; keep the target lazy or use a parent-aware hook/create path.
  - confidence: 0.86
