## What This Changes

This PR clears stored CLI session bindings during gateway resets for non-subagent sessions, preserves them for canonical spawned subagents, and adds gateway reset regression tests.

## Reproduction Assessment

Yes at source level: current main copies the CLI binding fields during gateway reset, and the CLI runner later reads those fields to decide whether to resume. I did not run a live claude-cli reset in this read-only review.

## Solution Assessment

Mostly yes: clearing the copied binding after constructing the reset entry is a narrow fix, and the updated isSubagentSessionKey guard matches the existing spawned-subagent key contract. The remaining solution-fit question is whether reset should clear all CLI provider bindings or only claude-cli.
