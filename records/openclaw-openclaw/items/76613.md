## Summary

Keep open: current main still has the Matrix token-hash sibling selection behavior and docs mention reuse, but there is no warning, doctor signal, or cleanup guidance for accumulated populated sibling stores. This is a new Matrix observability/docs feature rather than a broken existing contract.

## What This Changes

Request to add non-breaking Matrix crypto-store observability and documentation for accumulated populated token-hash sibling directories.

## Best Possible Solution

Add a Matrix-owned diagnostic/warning that detects multiple populated token-hash roots, redacts sensitive details, leaves selection unchanged, and documents safe archival cleanup.
