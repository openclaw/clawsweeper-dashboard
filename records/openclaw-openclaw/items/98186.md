The PR updates gateway agent timeout docs/help and gateway CLI dispatch so omitted `--timeout` no longer serializes `params.timeout`, while explicit `--timeout` remains a per-run override.

## Best Possible Solution

Land the current shape after maintainers accept the upgrade from an implicit 600-second CLI fallback to server-owned omitted timeouts.

## Reproduction Assessment

## Solution Assessment

Yes. Current head fixes the right boundary by omitting `params.timeout` when the CLI flag is omitted, preserving explicit overrides, and using the max timer-safe wait so longer server-owned defaults are not preempted by the CLI.
