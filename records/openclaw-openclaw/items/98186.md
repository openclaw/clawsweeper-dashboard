## Summary

Keep open: the updated PR proof now shows omitted versus explicit gateway RPC timeout behavior, but the patch still gives omitted calls a local 48-hour client wait ceiling, which can fire before a longer server-owned agent timeout and start embedded fallback early.

## What This Changes

The PR removes generic 600-second agent timeout examples, updates `openclaw agent` timeout help, exports the shared 48-hour default, and makes gateway dispatch omit `params.timeout` unless `--timeout` is explicit.

## Best Possible Solution

Land a narrow fix that omits `params.timeout` for omitted CLI timeouts and uses a client wait that cannot fire before the server-owned agent timeout, with explicit `--timeout` and docs behavior covered.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows the 600-second gateway CLI fallback and generic docs examples while the shared runtime resolver defaults unset agent timeouts to 48 hours; PR-head inspection also reproduces the new omitted-client-wait cap.

## Solution Assessment

No. The latest shape is correct about omitting `params.timeout`, but using `DEFAULT_AGENT_TIMEOUT_SECONDS` as the omitted client wait is only a partial server-owned solution; the safer fix is a no-local-deadline/max wait unless `--timeout` is explicit.
