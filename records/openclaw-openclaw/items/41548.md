## Summary

Keep open. Current main still scopes TUI live/history views to one selected session, and the requested cross-channel inbox would add new, privacy-sensitive behavior; the open same-author #40678 remains the narrower design thread rather than a resolved duplicate.

## What This Changes

Requests a unified cross-channel inbox so TUI, Telegram, and iMessage messages and replies appear across active surfaces.

## Best Possible Solution

Define an opt-in cross-surface observer or mirror mode that preserves TUI session isolation, channel privacy, and explicit delivery routing, then implement it through Gateway session subscriptions and TUI UX after reconciling #40678.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection shows the current TUI intentionally consumes and loads only the selected session, while docs only promise optional provider delivery, not unified inbox sync.

## Solution Assessment

No for automatic implementation as requested. Default all-surface sync would be too broad for the current session/privacy model; an opt-in observer or mirror mode is the safer maintainable direction.
