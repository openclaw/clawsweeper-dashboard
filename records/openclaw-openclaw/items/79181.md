## Summary

Keep open: the PR is a plausible gateway crash-loop guard, but the current patch does not cover the linked OOM report's restart cadence and should not merge as a fix for that issue yet.

## What This Changes

The PR adds a gateway startup throttle sentinel, a unit test file, and a changelog entry to delay sidecar loading after rapid process restarts.

## Best Possible Solution

Retarget this as a narrow sidecar-startup rapid-crash guard, or redesign the gateway crash-recovery path around the actual OOM/stability evidence before claiming the linked issue is fixed.

## Reproduction Assessment

No high-confidence current-main live reproduction is available. We do have source/log evidence for the PR defect: the linked report's OOM restarts are minutes apart while the patch resets after a 60-second rapid window and a two-minute stable timer.

## Solution Assessment

No. A sidecar-startup throttle may be useful, but it is not the best proven fix for the linked OOM report because that evidence shows the gateway staying up for minutes after readiness before being killed.

- **[P2] Match the throttle to the reported restart cadence:** `src/gateway/server-startup-throttle.ts:50-53`
