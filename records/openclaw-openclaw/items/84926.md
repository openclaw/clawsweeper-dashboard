## Summary

Keep open. Current main has partial model diagnostics, raw assistant-stream logging, and OTEL content-capture plumbing, but it does not provide complete per-turn AI request/response JSON with correlation, export, and an agreed sensitive-data policy.

## What This Changes

Request to add an opt-in debug mechanism for viewing or exporting complete per-round AI provider request and response JSON correlated by run, session, message, or request id.

## Best Possible Solution

Design a maintainer-approved opt-in capture path at the model-call transport seam that records redacted per-call request and reconstructed response payloads with callId/run/session correlation, local retention/export controls, and a tightly controlled raw mode.
