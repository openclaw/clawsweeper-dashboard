## Summary

Keep open: current main has useful partial diagnostics, but they do not yet provide a supported per-round full request/response JSON workflow with OpenClaw run/session/message correlation and an explicit raw-payload access policy.

## What This Changes

The issue requests an opt-in debug mechanism to inspect full per-round AI provider request and response JSON with correlation across sessions, messages, runs, or request IDs and a clear sensitive-data policy.

## Best Possible Solution

Add a supported opt-in provider-debug workflow that correlates captured request/response blobs with model call, run, session, and message identifiers, defaults to redacted/capped output, and gates raw export behind explicit operator-controlled access.

## Reproduction Assessment

Not applicable: this is a feature request for a new provider-debug workflow, not a report of broken existing behavior. Source inspection shows partial diagnostics but not the requested end-to-end capability.

## Solution Assessment

Yes as a product direction, but not as an automated bug fix: the maintainable path is to unify existing capture and model-call correlation surfaces under a deliberate security policy instead of adding another ad hoc payload log.
