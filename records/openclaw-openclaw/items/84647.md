## Summary

Keep open. Current main has useful trigger-based queue priority, but the requested gateway fairness contract still remains: configurable priority classes, cooperative preemption/yield behavior, queue-depth-aware demotion, and priority-aware diagnostics or doctor checks.

## What This Changes

The issue requests gateway scheduler fairness so human-facing sessions can be prioritized over concurrent cron and background agents on the shared event loop.

## Best Possible Solution

Define a maintainer-approved gateway fairness policy on top of the existing trigger-priority lanes, then implement config/defaults, cooperative yield points, priority metrics or doctor checks, and regression coverage together.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. Source inspection does confirm the missing requested surfaces: priority classes are fixed by trigger, liveness labels omit priority/head-of-line wait, and no cooperative preemption or queue-depth demotion policy is present.

## Solution Assessment

Unclear as an implementation path. The existing trigger-priority lane code is the right boundary to build on, but the best fix requires a maintainer-approved scheduler policy rather than another narrow queue patch.
