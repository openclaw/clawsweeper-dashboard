## Summary

Keep open. Current main has useful trigger-based foreground/background queue priority, including global-lane propagation, but it still does not implement the requested gateway fairness contract: configurable priority classes, cooperative preemption/yield policy, queue-depth-aware demotion, priority-class liveness/head-of-line metrics, or doctor guidance.

## What This Changes

The issue requests gateway scheduler fairness so interactive human-facing sessions can be prioritized over cron/background agents sharing the same gateway event loop.

## Best Possible Solution

Define one maintainer-approved gateway fairness contract that builds on the existing trigger-priority lane system, then add config/default semantics, priority-aware diagnostics, and cooperative yield/preemption tests after the policy is settled.

## Reproduction Assessment

No high-confidence live reproduction was run, but source inspection confirms current main lacks the requested policy surfaces beyond static trigger priority. The issue's operational samples are useful but need a fresh diagnostics export or live scenario to calibrate thresholds.

## Solution Assessment
