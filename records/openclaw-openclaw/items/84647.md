## Summary

Keep open. Current main has lane-local priority and trigger-to-priority mapping, but the requested cross-agent gateway fairness policy, configurable priority classes, priority-aware liveness diagnostics, and doctor signal are still not implemented.

## What This Changes

The issue requests gateway scheduler fairness so interactive human-facing sessions can take priority over background agents sharing the same event loop.

## Best Possible Solution

Define one gateway scheduler contract that carries trigger or priority facts into shared lane selection, exposes default interactive/service/background policy and diagnostics, and preserves existing lane concurrency semantics.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection supports the mechanism: trigger priority is applied to session lanes, while the inner shared global lane still does not receive a per-turn priority signal.

## Solution Assessment

No, the current partial priority behavior does not solve the requested cross-agent scheduler policy. The maintainable path is a product-shaped scheduler contract that extends the existing lane system and diagnostics.
