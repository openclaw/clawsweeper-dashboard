## Summary

Keep open. Current main has partial same-session priority handling and cooperative model-stream yielding, but it does not yet provide the requested cross-agent/global scheduler fairness, configurable priority classes, or priority-labeled liveness diagnostics.

## What This Changes

The issue requests scheduler fairness so interactive human-facing gateway sessions can take priority over background or cron agents sharing one event loop.

## Best Possible Solution

Define one gateway scheduler contract that carries trigger or priority facts into shared lane selection, exposes default interactive/service/background policy and diagnostics, and preserves existing lane concurrency semantics.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection supports the mechanism: trigger priority is applied to session lanes, while the inner shared global lane still does not receive a per-turn priority signal.

## Solution Assessment

No, the requested issue is not solved by the existing partial priority/yield behavior. The maintainable path is a product-shaped scheduler policy that extends the current lane system rather than a narrow bug patch.
