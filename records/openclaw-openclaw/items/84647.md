## Summary

Keep open. Current main has static foreground/background command-queue priority for embedded run triggers, but it still lacks the issue's central scheduler-fairness pieces: active background-run preemption or yield policy, configured priority classes, queue-depth demotion, and priority/head-of-line diagnostics for gateway fairness.

## What This Changes

The issue requests scheduler-level fairness so human-facing gateway sessions can be prioritized over concurrent background agents sharing the same Node.js event loop.

## Best Possible Solution

Keep this open as the in-process scheduler fairness tracker, land or reject the narrower queue-aging and origin-policy slices deliberately, then add live gateway/channel proof before considering the fairness gap closed.
