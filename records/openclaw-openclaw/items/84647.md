## Summary

Keep open. Current main and the latest release include useful foreground/background priority inside command lanes, but they still do not provide the requested gateway fairness contract across concurrent foreground and background agents on the shared event loop.

## What This Changes

Issue requests gateway scheduler fairness so human-facing sessions are prioritized over background agents when the shared event loop is contended.

## Best Possible Solution

Design a maintainer-approved core scheduler policy that classifies work before enqueue, arbitrates foreground/background pressure across lanes, exposes priority/head-of-line diagnostics, and proves the behavior with focused queue tests plus live gateway load evidence.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms current main still lacks cross-lane preemption, configurable priority classes, queue-depth demotion, and priority-class liveness metrics for the reported gateway contention shape.

## Solution Assessment

No, current main is only a partial mitigation. The better fix is a maintainer-approved scheduling policy across gateway lanes and triggers, not a one-off queue tweak.
