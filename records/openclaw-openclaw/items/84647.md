## Summary

Keep open. Current main and v2026.6.11 have useful lane-local foreground/background priority, but they still do not provide the requested gateway fairness contract for prioritizing human-facing sessions over concurrent background agents on the shared process/event loop.

## What This Changes

The issue requests gateway scheduler fairness so human-facing sessions can preempt or outrank concurrent cron/background agent work under event-loop contention.

## Best Possible Solution

Design a maintainer-approved core scheduler policy that classifies work before enqueue, protects human-facing sessions under mixed load, exposes priority/head-of-line diagnostics, and proves the behavior with focused queue tests plus representative live gateway load evidence.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms current main still lacks the requested cross-lane fairness policy, configurable priority classes, queue-depth demotion, safe preemption/yield points, and priority-class liveness metrics.

## Solution Assessment

No, current main is only a partial mitigation. The best fix is a maintainer-approved gateway scheduling policy across triggers, lanes, diagnostics, and operator controls rather than another one-off queue tweak.
