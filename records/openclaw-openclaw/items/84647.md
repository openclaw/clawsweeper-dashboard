## Summary

Keep open. Current main has lane-local foreground/background ordering and separate cron/main lanes, but it does not implement the requested cross-lane/event-loop fairness contract, priority-class config, preemption/yield policy, or per-priority wait diagnostics for interactive sessions under background-agent pressure.

## What This Changes

Request gateway scheduler policy hooks so interactive human-facing sessions can outrank cron/background agent work when the shared process is under event-loop pressure.

## Best Possible Solution

Define a first-class gateway fairness contract that classifies work, preserves per-session serialization, allows safe background yielding or demotion, and reports per-class head-of-line wait in diagnostics.
