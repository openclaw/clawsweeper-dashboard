## Summary

Keep open. Current main and v2026.6.8 have useful trigger-based foreground/background queue ordering, but the requested gateway fairness contract still remains: configurable priority classes, cooperative preemption or yield policy, queue-depth-aware demotion, and priority-aware diagnostics are not implemented.

## What This Changes

The issue requests an in-process gateway scheduler fairness policy so human-facing sessions are not starved behind concurrent background agents on the shared event loop.

## Best Possible Solution

Define one maintainer-approved gateway fairness contract that builds on the existing trigger-priority lane system, then add config, diagnostics, doctor behavior, and cooperative yield/preemption tests only after the starvation policy is settled.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection does confirm the remaining requested surfaces are absent: configurable priority classes, priority-aware liveness/head-of-line metrics, and cooperative scheduler preemption.

## Solution Assessment

Unclear as an implementation path. The existing lane priority code is the right boundary to build on, but the best fix needs a maintainer-approved scheduler policy rather than another narrow queue patch.
