## Summary

Keep open. Current main and the latest shipped release have partial per-lane priority and cooperative stream yielding, but they still do not provide the requested cross-agent/global fairness policy, configurable priority classes, preemption semantics, or priority-labeled liveness diagnostics.

## What This Changes

The issue requests scheduler-level fairness so human-facing sessions can run ahead of cron, Bug Fixer, and other background agents sharing one gateway process.

## Best Possible Solution

Define one gateway scheduler contract that carries trigger or priority facts into shared-lane selection, exposes default interactive/service/background policy and diagnostics, and documents safe yield or preemption behavior with focused upgrade tests.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection supports the mechanism: trigger priority is applied to session lanes while the inner shared global lane and diagnostics still lack a priority-class contract.

## Solution Assessment

No, current main is not the complete solution. The maintainable path is a product-shaped scheduler policy extending the existing lane system rather than a narrow patch or a second competing scheduler.
