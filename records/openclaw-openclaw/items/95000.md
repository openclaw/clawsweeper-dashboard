- **[P1] Keep image.providers auth checks read-only:** `src/gateway/server-methods/image.ts:72`
  - body: `image.providers` is registered as `operator.read`, but this line calls each provider's `isConfigured` hook. OpenAI-compatible image providers route that through `isProviderApiKeyConfigured`, which calls `ensureAuthProfileStore(...)` without `readOnly: true`, so opening an inventory page can sync or write auth-profile state; build configured state from a read-only profile listing path instead.
  - confidence: 0.89
- **[P2] Do not synthesize an active image provider:** `src/gateway/server-methods/image.ts:95-99`
  - body: When no image-generation model is explicitly configured, the handler falls back to the first configured provider and then the first provider in the registry. The existing CLI selected state is derived from `agents.defaults.imageGenerationModel`, so this would expose a misleading public Gateway contract; return `null` unless config explicitly selects a registered provider, or add a separately named effective-fallback field after owner approval.
  - confidence: 0.85
