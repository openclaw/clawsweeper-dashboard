Full review comments:

- **[P2] Remove the unused agentDir parameter:** `src/gateway/server-methods/image.ts:16`
  - body: `providerHasGenericConfig` accepts `agentDir` but never reads it. Core typecheck enables `noUnusedParameters`, so this production file can fail before the RPC is usable; remove the parameter or use it for a real auth-profile readiness check.
  - confidence: 0.96
- **[P2] Reuse the existing provider readiness semantics:** `src/gateway/server-methods/image.ts:22`
  - body: The generic fallback checks `cfg.auth.profiles` with `startsWith(providerId)`, which can collide on prefixes and misses the persisted agent auth store/env readiness path used by existing media-provider helpers. Reuse the existing provider/auth readiness semantics instead of inferring from config keys.
  - confidence: 0.91
- **[P2] Report only the configured selected image provider as active:** `src/gateway/server-methods/image.ts:83-87`
  - body: When no configured primary image provider exists, this falls back to the first configured provider and then the first listed provider. That can tell setup UIs a provider is active even when OpenClaw has not selected one; match the existing image provider listing semantics and return `null` when nothing is selected.
  - confidence: 0.9
- **[P2] Validate the Gateway result before responding:** `src/gateway/server-methods/image.ts:88-91`
  - body: The PR exports `validateImageProvidersResult`, but the handler returns plugin-derived capability payloads without using it. Validate the assembled result and return the standard invalid-response error path so schema drift does not silently reach Gateway clients.
  - confidence: 0.88
- **[P2] Model the full image capability contract:** `packages/gateway-protocol/src/schema/image-generation.ts:38-42`
  - body: The handler forwards image provider capability objects, but the new schema/types omit fields current providers can return, including `edit.maxInputImages`, `geometry.aspectRatiosByModel`, and `geometry.resolutionsByModel`. Mirror `src/image-generation/types.ts` before publishing this as a Gateway client contract.
  - confidence: 0.89
- **[P2] Export the schema through the stable schema barrel:** `packages/gateway-protocol/src/index.ts:482-486`
  - body: `@openclaw/gateway-protocol/schema` is the package's stable schema import path, but the new schema is only imported internally. Add it to `packages/gateway-protocol/src/schema.ts` and cover it so schema consumers can discover the new result contract.
  - confidence: 0.86
