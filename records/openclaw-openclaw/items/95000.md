  - confidence: 0.88
- **[P2] Match image provider readiness and active semantics:** `src/gateway/server-methods/image.ts:64-86`
  - body: The fallback configured check only checks raw config key presence and treats empty objects as configured; it also skips auth profile store and env-var readiness that the existing CLI provider listing uses. Since `active` then falls away from a selected-but-not-configured provider, setup UIs can display misleading provider status for exactly the metadata this RPC exposes.
