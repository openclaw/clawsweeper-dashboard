  - body: The generic fallback checks `cfg.auth.profiles` keys with `startsWith(providerId)`, which can mark `open` configured for an `openrouter` profile and misses the persisted agent auth store used by the existing CLI/helper path.   - confidence: 0.9
- **[P2] Do not invent an active image provider:** `src/gateway/server-methods/image.ts:81-87`
  - body: When the configured primary provider is absent or unconfigured, this falls back to the first configured provider and then the first listed provider. The existing image providers CLI only reports selection from `agents.defaults.imageGenerationModel`, so this can tell setup UIs that a provider is active when OpenClaw has not selected one.
  - confidence: 0.91
- **[P2] Validate the result before responding:** `src/gateway/server-methods/image.ts:88-91`
  - body: The PR exports `validateImageProvidersResult`, but the handler returns plugin-derived capability payloads without using it. Validate the assembled result and return the standard invalid-response error path so schema drift does not silently reach Gateway clients.
  - confidence: 0.88
- **[P2] Model the full image capability contract:** `packages/gateway-protocol/src/schema/image-generation.ts:7-42`
  - body: The handler forwards image provider capability objects, but the new schema omits fields current provider types can return, including `edit.maxInputImages`, `geometry.aspectRatiosByModel`, and `geometry.resolutionsByModel`. Mirror the current image-generation capability contract before publishing this as a Gateway client type.
  - confidence: 0.9

  - body: `@openclaw/gateway-protocol/schema` is the package's stable schema import path, but the new schema module is only imported internally. Add it to `packages/gateway-protocol/src/schema.ts` and cover it so schema consumers can discover the new result contract.
  - confidence: 0.86
