## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use provider credentials for auth fallback:** `src/gateway/server-methods/image.ts:48-50`
  - body: For providers without `isConfigured`, this fallback checks whether the profile id starts with the provider id. Auth profile ids are not the provider contract; `listProfilesForProvider` matches the credential's provider field. The current lazy Google image provider has no `isConfigured`, so a valid Google profile with a custom id will be reported as `configured: false` by the new RPC.
  - confidence: 0.91
- **[P2] Include output arrays in the protocol type:** `packages/gateway-protocol/src/schema/image-generation.ts:110-116`
  - body: The schema allows an output capability array and the handler returns `output: []` when a provider omits output metadata, but `ImageProviderOutputCapability` only permits `boolean` or an object. Consumers importing the new protocol type cannot represent valid `image.providers` responses until this includes `string[]` or is derived from the schema.
  - confidence: 0.9
- **[P2] Document the new public Gateway method:** `src/gateway/methods/core-descriptors.ts:42`
  - body: Adding `image.providers` to the core descriptor makes it a public operator RPC, but the Gateway protocol docs still only list the peer `tts.providers` inventory method. External Console/control clients need the method, scope, and response contract documented with the code change.
  - confidence: 0.88
