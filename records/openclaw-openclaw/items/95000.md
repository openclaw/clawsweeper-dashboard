Full review comments:

- **[P2] Preserve the selected image provider as active:** `src/gateway/server-methods/image.ts:84-86`
  - body: The new RPC only reports the configured `imageGenerationModel` provider as `active` when that provider is already credential-configured, then falls back to another provider. Setup UIs need to show the user's selected image provider even when auth is missing, like `tts.providers` separates `active` from `configured`; otherwise this endpoint can highlight the wrong provider during setup.
  - confidence: 0.83
- **[P2] Include the current capability fields in the contract:** `packages/gateway-protocol/src/schema/image-generation.ts:7-12`
  - body: The handler returns `provider.capabilities` through the RPC, but the new schema/type omits current fields such as `edit.maxInputImages`, `geometry.aspectRatiosByModel`, and `geometry.resolutionsByModel`. Clients generated from this contract will lose fields that current providers expose; mirror `src/image-generation/types.ts` and add validator coverage for those payloads.
  - confidence: 0.89
- **[P2] Export the schema from the stable entrypoint:** `packages/gateway-protocol/src/index.ts:546`
  - body: This adds a public result validator, but the package's exported `./schema` entry is `src/schema.ts` and it still does not re-export the new image-generation schema/types. External clients cannot import `ImageProvidersResultSchema` or the result types from the supported schema barrel, so add the barrel export and type the validator against the exported result.
  - confidence: 0.86
