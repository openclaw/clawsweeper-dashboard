## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Use read-only auth profiles for configured state:** `src/gateway/server-methods/image.ts:69`
  - body: The handler derives `configured` from `cfg.auth.profiles` and config objects only, so a normal setup where credentials live in the agent auth-profile store is reported unconfigured and `active` can become `null` even when `agents.defaults.imageGenerationModel` selects that provider. The sibling CLI inventory loads the runtime auth profile store and uses `listProfilesForProvider`; use the read-only runtime store plus non-empty config checks here so Gateway matches existing provider inventory semantics without writing auth state.
  - confidence: 0.9
- **[P2] Include array output in exported capability type:** `packages/gateway-protocol/src/schema/image-generation.ts:82-88`
  - body: The schema accepts `capabilities.output` as a string array and the handler returns `[]` for missing output metadata, but the exported `ImageProviderCapabilityOutput` type only allows boolean or object. First-version TypeScript clients importing `ImageProvider` would reject valid RPC payloads, so add `string[]` or remove the array shape consistently.
  - confidence: 0.86
