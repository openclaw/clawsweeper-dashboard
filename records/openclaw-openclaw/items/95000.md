- **[P1] Keep configured-state lookup side-effect-free:** `src/gateway/server-methods/image.ts:94-95`

  - confidence: 0.9
- **[P2] Register the result schema with ProtocolSchemas:** `packages/gateway-protocol/src/schema/image-generation.ts:81-84`
  - body: Adding `ImageProvidersResultSchema` only as a direct validator leaves it out of the central `ProtocolSchemas` registry that feeds generated JSON schema, Swift generation, and registry-derived TypeScript aliases. Register the new result/provider schemas there so external Gateway clients see the new public response contract.
  - confidence: 0.86

  - body: The descriptor now advertises `image.providers`, but `docs/gateway/protocol.md` still documents `tts.providers` with no image peer, and `CAPABILITY_METADATA` still says `image.providers` is local-only while `tts.providers` is local+gateway. Clients using docs or capability discovery will get a conflicting public contract.
  - confidence: 0.84

  - body: The latest `check-test-types` job fails in this file: handler calls omit required `req`, `params`, `client`, and `isWebchatConnect` fields, provider fixtures are inferred as `never`, and the auth-store mock cast does not overlap the real signature.
