## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use shared read-only provider readiness lookup:** `src/gateway/server-methods/image.ts:40-49`
  - body: `image.providers` is an `operator.read` RPC, but this helper loads the auth profile store without read-only options and then guesses provider ownership from profile-id prefixes. Current auth-store loading can write when `readOnly` is false, and current provider lookup uses `listProfilesForProvider` against `cred.provider`, so this can both mutate auth state and report custom-profile providers as unconfigured. Factor or reuse one read-only provider inventory/readiness helper instead.
  - confidence: 0.9
- **[P2] Include array output in the exported type:** `packages/gateway-protocol/src/schema/image-generation.ts:110-116`
  - body: The schema accepts `output` as `boolean | string[] | object`, and the handler emits `output: []` for providers without output metadata, but `ImageProviderOutputCapability` omits the `string[]` case. TypeScript clients importing `ImageProvidersResult` would get a type that cannot represent a valid wire response.
  - confidence: 0.87
- **[P2] Document and register the public RPC contract:** `src/gateway/methods/core-descriptors.ts:42`
  - body: Adding `image.providers` to `CORE_GATEWAY_METHOD_SPECS` makes it a public read-scoped Gateway method, but the PR does not document it in the Gateway protocol reference or add the new image provider schemas to the central `ProtocolSchemas` registry used by protocol tooling. Please land the method with the docs and schema registry/test updates so external clients have one discoverable contract.
  - confidence: 0.82
