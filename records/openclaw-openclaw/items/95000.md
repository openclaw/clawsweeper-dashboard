## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep image.providers auth checks read-only:** `src/gateway/server-methods/image.ts:72`
  - body: `image.providers` is registered as `operator.read`, but this line calls each provider's `isConfigured` hook. For OpenAI-compatible image providers that goes through `isProviderApiKeyConfigured`, which calls `ensureAuthProfileStore(...)` without `readOnly: true`; the auth store can sync/write external CLI profiles unless read-only is explicitly set. Build configured state from a read-only `loadAuthProfileStoreForRuntime(..., { readOnly: true })`/`listProfilesForProvider` path instead of invoking write-capable provider hooks from this inventory RPC.
  - confidence: 0.88
- **[P2] Do not synthesize an active image provider:** `src/gateway/server-methods/image.ts:95-99`
  - body: When no image-generation primary is configured, the handler falls back to the first configured provider and then the first provider in the registry. That makes fresh or unconfigured installs report an active image provider even though the existing CLI's selected state is derived only from `agents.defaults.imageGenerationModel`. Return `null` unless config explicitly selects a registered image provider, or add a separately named effective-fallback field after maintainer approval.
  - confidence: 0.82
- **[P2] Remove the PR body artifact from the branch:** `pr-body-final.md:1`
  - body: `pr-body-final.md` is a PR-description/proof scratch file at the repository root, not product docs or source. If merged, it leaves review-only generated content in the checkout; remove it and keep this text in the GitHub PR body/comment instead.
  - confidence: 0.96
