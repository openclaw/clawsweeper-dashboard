## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Use the runtime auth lookup for configured providers:** `src/gateway/server-methods/image.ts:26-30`
  - body: The current check only looks at config/profile keys, but provider readiness also comes from env vars and persisted auth profiles whose ids do not have to equal the provider id. With a normal env-backed or `openai-main` auth-profile-backed setup, this RPC can return `configured: false` and then `active: null` even though image generation is usable. Please derive this from a read-only runtime auth/profile/env helper instead of `cfg.auth.profiles` key matching.
  - confidence: 0.94
- **[P1] Add maxInputImages to the image capability contract:** `packages/gateway-protocol/src/schema/image-generation.ts:7-13`
  - body: The new public schema/type for image provider capability support omits `maxInputImages`, but current image edit capabilities and providers like OpenAI/Google expose that limit and the handler returns provider capabilities directly. Generated clients that depend on this contract cannot type the real response or render edit input limits correctly. Add the field to the schema, exported TS type, generated clients, and protocol tests before shipping this RPC.
  - confidence: 0.9
