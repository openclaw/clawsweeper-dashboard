## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use runtime auth profiles for configured state:** `src/gateway/server-methods/image.ts:69`
  - body: The handler only checks config fields, so providers with credentials in the runtime auth-profile store can be reported unconfigured while empty `models.providers` or plugin config blocks are reported configured. Mirror the CLI/readiness path with a read-only `loadAuthProfileStoreForRuntime` plus non-empty config checks before exposing this to Console.
  - confidence: 0.91
- **[P2] Include output arrays in the exported capability type:** `packages/gateway-protocol/src/schema/image-generation.ts:82-88`
  - body: The schema accepts `capabilities.output` as `string[]` and this PR's handler/tests return output arrays, but `ImageProviderCapabilityOutput` only allows boolean or object. TypeScript clients importing `ImageProvider` cannot type valid responses until the exported type includes the array case or the array shape is removed consistently.
  - confidence: 0.9
- **[P2] Expose maxInputImages in the edit capability contract:** `packages/gateway-protocol/src/schema/image-generation.ts:65-71`
  - body: Image providers such as OpenAI, fal, and Google publish `edit.maxInputImages`, and the handler forwards `provider.capabilities.edit` unchanged. The new support type omits that field, so first-version clients lose a real inventory field; add it to the schema/type and cover it in protocol tests.
  - confidence: 0.86
