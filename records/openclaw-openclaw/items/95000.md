  - body: The handler forwards provider capabilities, but the new public types/schema omit current fields such as `edit.maxInputImages`, `aspectRatiosByModel`, and `resolutionsByModel`. That leaves clients with an incomplete contract for valid runtime payloads.
  - confidence: 0.9

  - body: For providers without a custom `isConfigured` hook, this fallback only checks raw config maps. Existing media-generation readiness also considers env keys and agent auth profiles, so this can report a usable plugin image provider as unconfigured.
  - confidence: 0.84

  - body: After checking configured providers, this still falls through to `providers[0]?.id`. In an unconfigured setup the RPC can report the first visible provider as active, which is misleading for control UIs.
  - confidence: 0.88
