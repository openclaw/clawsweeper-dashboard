- **[P1] Use the shared image readiness semantics:** `src/gateway/server-methods/image.ts:69`
  - body: `providerHasGenericConfig` only inspects config object keys, misses env credentials and auth-profile stores used by existing image providers, and treats empty config objects as configured.   - confidence: 0.91

  - body: The handler forwards `provider.capabilities.edit` from current image providers, and existing providers expose `maxInputImages`, but `ImageProviderCapabilitySupportSchema` and the exported type do not declare it. First-version clients importing `ImageProvider` would not be able to type the real response, so add the field to the schema/type and protocol tests.
  - confidence: 0.88
