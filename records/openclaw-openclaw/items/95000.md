- **[P2] Mirror CLI readiness for fallback providers:** `src/gateway/server-methods/image.ts:27-31`
  - body: For providers that do not implement `isConfigured`, this fallback no longer matches the existing `openclaw infer image providers --json` path: the CLI loads the runtime auth-profile store with `listProfilesForProvider` and ignores empty config objects, while this code only looks at `cfg.auth?.profiles` and treats any object as configured. The bundled Google image provider registered in `extensions/google/index.ts` is lazy and has no `isConfigured`, so a user with only a Google auth profile will see `configured: false` over the new RPC even though the current CLI/runtime setup considers it configured.
  - confidence: 0.9
- **[P3] Keep capability result types aligned with the schema:** `packages/gateway-protocol/src/schema/image-generation.ts:118-123`
  - body: `ImageProviderCapabilitiesSchema` requires `geometry` and `output`, but the exported `ImageProviderCapabilities` type still marks both fields optional. Protocol consumers can now compile a value that the exported validator rejects, so either make the fields required in the type or make the schema optional too.
  - confidence: 0.86
