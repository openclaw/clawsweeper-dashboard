  - body: `providerHasGenericConfig` accepts `agentDir` but never reads it. Core production typecheck enables `noUnusedParameters`, so this branch can fail before the RPC is usable; remove the parameter or use it for the auth-profile readiness check.

  - body: When no selected provider is configured, this falls back to the first configured provider and then the first listed provider. The existing image providers CLI only marks a provider selected from `agents.defaults.imageGenerationModel`, so a setup UI can show an active provider even though OpenClaw has not selected one; return `null` unless there is a real selected/configured active provider.
  - confidence: 0.9

  - body: The PR adds `validateImageProvidersResult`, but the handler returns the plugin-derived payload without using it. Because this is a new public protocol result over plugin capability objects, invalid or drifted provider payloads can silently ship to clients; validate the response and return the standard `INVALID_REQUEST`/internal error shape on contract failure.
  - confidence: 0.82
- **[P2] Model the full image capability contract:** `packages/gateway-protocol/src/schema/image-generation.ts:7-43`
  - body: The handler forwards provider capability objects, but the new protocol schema/types omit fields current providers can return, including `edit.maxInputImages`, `geometry.aspectRatiosByModel`, and `geometry.resolutionsByModel`, and the exported output type omits string arrays. Generated or typed clients would miss data that the RPC returns, so mirror or derive the schema from the image provider contract before publishing it.
  - confidence: 0.9

  - body: The package's public `./schema` barrel is the supported schema import path, but this new schema module is only imported internally. Add the barrel export and a focused protocol test so clients can discover the new schema through the stable package surface.
  - confidence: 0.84
