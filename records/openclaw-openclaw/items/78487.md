## Summary

Keep open: current main still has the 30s TUI delta-silence watchdog and no public CLI/config timeout knob, and this issue is paired with an open same-author PR plus overlapping open canonical discussions.

## What This Changes

The issue requests exposing the TUI streaming watchdog timeout through CLI/config/TuiOptions so quiet long responses do not get the 30s stale warning.

## Best Possible Solution

Resolve the quiet-run watchdog behavior through the canonical TUI issue, ideally by consulting authoritative active-run state before clearing a run; expose a CLI/config timeout only if maintainers want that user-facing policy knob.

## Reproduction Assessment

Yes from source inspection: a chat delta arms a 30s default watchdog, and expiry clears the active run to idle and logs the exact prompt. I did not run a live provider/TUI session because this review is read-only.

## Solution Assessment

Unclear: a timeout knob may help users, but it is a new public config surface and the safer core fix may be to avoid treating quiet-but-active runs as stale without checking active-run state.
