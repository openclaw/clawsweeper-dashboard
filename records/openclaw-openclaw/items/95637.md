## Summary

Keep open for normal maintainer merge review. Current main and v2026.6.9 still require three-or-more dashes per QQBot table separator cell, while the PR is a narrow helper/test fix with sufficient terminal before/after proof and no actionable patch findings.

## What This Changes

The PR changes QQBot markdown table separator recognition from three-or-more dashes to one-or-more dashes and adds regression coverage for one-, two-, and aligned-dash separators.

## Best Possible Solution

Land the narrow QQBot helper and regression test after maintainer review and required-check gating, while keeping the broader long-reply chunking behavior in https://github.com/openclaw/openclaw/issues/95526 separate.

## Reproduction Assessment

Yes. Current main and v2026.6.9 require three dashes per separator cell, while GitHub's Markdown API accepts one- and two-dash delimiter rows; the QQBot state machine then leaves the header pending and can drop prior table content.

## Solution Assessment

Yes. The private QQBot separator helper is the implicated layer, and changing only the dash quantifier matches the observed Markdown behavior without adding a parser, config knob, or cross-channel policy change.
