## What This Changes

The PR changes QQBot’s private markdown table delimiter recognition from three-or-more hyphens to one-or-more hyphens and adds a regression test for one- and two-hyphen separator rows.

## Best Possible Solution

Land the focused QQBot parser/test fix after ordinary maintainer review and green or accepted checks, while leaving broader QQBot long-reply bubble behavior to https://github.com/openclaw/openclaw/issues/95526.

## Reproduction Assessment

Yes. Current main and v2026.6.9 source reject short delimiter rows, and the PR body provides before/after terminal output from the real exported QQBot chunker showing the lost table content before the fix and preserved content after it.

## Solution Assessment

Yes. The private QQBot separator helper is the implicated layer; alternatives like only flushing replaced pending headers would avoid some loss but still misclassify valid delimiter rows, and broader QQBot message aggregation belongs to a separate issue.
