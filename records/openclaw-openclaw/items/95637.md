Keep open: the PR is a narrow, source-backed QQ Bot message-loss fix, and current main still rejects valid one- and two-dash GFM table delimiters in the outbound markdown chunker.

## What This Changes

The PR changes QQ Bot markdown table separator detection from three-or-more dashes to one-or-more dashes and adds regression coverage for one-, two-, and aligned-dash separators.

## Best Possible Solution

Land the narrow helper change with the new regression test once required CI is clean, while keeping the broader QQ long-reply bubble issue separate.

## Reproduction Assessment

Yes by source inspection: one- and two-dash separators fail the current /^:?-{3,}:?$/ check, so the pending header is not confirmed before later rows are handled. I did not run the test suite because this review is read-only, but the PR body includes before/after terminal output for the exported chunker.

## Solution Assessment

Yes. The private QQ Bot separator helper is the implicated layer, and changing only the dash quantifier matches the GFM contract without adding a second parser, config knob, or cross-channel behavior change.
