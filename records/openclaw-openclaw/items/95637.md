## Summary

Keep open for maintainer review: current main still rejects one- and two-dash QQBot table delimiter rows, while this PR is a focused parser/test fix backed by the GFM contract, live check status, and before/after terminal proof from the real exported chunker.

## What This Changes

The PR changes QQBot markdown table separator recognition from three-or-more dashes to one-or-more dashes and adds regression coverage for one- and two-dash separators.

## Best Possible Solution

Land the focused QQBot parser/test fix after ordinary maintainer review; keep the broader QQBot long-reply bubble behavior tracked separately in https://github.com/openclaw/openclaw/issues/95526.

## Reproduction Assessment

Yes. Current-main source shows one- and two-dash delimiter rows fail the separator predicate used by both QQBot outbound paths, and the PR body includes before/after terminal output from a harness importing the real exported chunker.

## Solution Assessment

Yes. The narrowest maintainable fix is to correct the private QQBot separator predicate at the misclassification point; flushing overwritten pending headers would preserve some text but still fail to recognize valid GFM delimiter rows.
