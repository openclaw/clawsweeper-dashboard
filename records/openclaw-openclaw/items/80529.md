## Best Possible Solution

Revise or replace this PR with a narrow safe-send fix keyed to grammY `HttpError.error`, cause, or pre-connect code evidence, then require real transport proof before merge.

## Reproduction Assessment

Yes for the source-level classifier path: current main and v2026.6.9 reject a bare grammY `Network request for 'sendMessage' failed!` envelope in send context, and tests preserve that behavior. I did not establish a live low-resource Telegram transport reproduction in this read-only review.

## Solution Assessment

No: this PR changes the broader recoverable-network classifier, while the central final-send path is guarded by `isSafeToRetrySendError`. The safer solution is to extend the safe-send classifier only when grammY's nested `.error`/cause/code evidence proves a pre-connect failure.
