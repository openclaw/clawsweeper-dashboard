## Summary

Keep open for contributor and maintainer follow-up: current main still has the reported Telegram classifier mismatch, but this patch broadens the outer grammY envelope match without proving the non-idempotent send retry boundary and lacks live Telegram transport proof.

## What This Changes

The PR relaxes Telegram network error matching so bare grammY `Network request ... failed!` messages are recoverable in send context and updates unit coverage.

## Best Possible Solution

Keep the Telegram fix focused on pre-connect evidence in the actual safe-send retry path, preserve duplicate-delivery safeguards, and require redacted live Telegram transport proof before merge.

## Reproduction Assessment

No high-confidence live reproduction is present for the reported low-resource Telegram failure. Source inspection does reproduce the helper mismatch: current main rejects the bare send-context envelope in the helper test while accepting the older `failed after` envelope.

## Solution Assessment

No, not as submitted. The safer solution is to tie retry eligibility to nested pre-connect evidence or update the strict safe-send predicate directly, rather than treating the outer grammY envelope as sufficient proof.
