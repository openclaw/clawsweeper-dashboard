## Summary

Keep open: the prior ClawSweeper review raised real retry-safety and proof blockers, and current source plus grammY contract checks support that this patch is not merge-ready as submitted.

## What This Changes

The PR relaxes Telegram grammY network-error matching so bare `Network request ... failed!` messages are treated as recoverable in send context and updates the corresponding unit test.

## Best Possible Solution

A Telegram-only fix should preserve non-idempotent send safeguards by retrying only when nested error evidence proves a pre-connect failure, with focused regression coverage and redacted live Telegram proof.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection reproduces the helper-level mismatch, but current normal message/media sends use `isSafeToRetrySendError` and the real Telegram dropped-message path is not proven by the PR evidence.

## Solution Assessment

No, not as submitted. The safer fix is to require nested pre-connect evidence or intentionally adjust the safe-send predicate, rather than making the outer grammY envelope sufficient for send-context retry classification.
