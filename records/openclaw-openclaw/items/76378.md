## Summary

Keep open for maintainer handling: the patch is not safe to merge as-is because it disables the approved build path for the optional native Discord Opus decoder without explaining the intended install-policy tradeoff, while current main still preserves that decoder path with `opusscript` as fallback. pnpm v10 treats `onlyBuiltDependencies` as the allowlist for dependency lifecycle scripts and `ignoredBuiltDependencies` as the unapproved list. 

## What This Changes

The PR moves `@discordjs/opus` from pnpm's approved build-script lists to ignored build-script lists in `package.json` and `pnpm-workspace.yaml`.

## Best Possible Solution

Keep the existing optional native decoder behavior unless maintainers intentionally choose a documented, tested install policy that avoids native build failures while preserving Discord voice fallback semantics.

## Reproduction Assessment

No live install reproduction was run. Source inspection plus pnpm's documented build-script contract show the patch would move `@discordjs/opus` from the lifecycle-script allowlist to the ignored list, while Discord voice still probes that native decoder path.

## Solution Assessment

No. Disabling the native Opus build in both pnpm config surfaces is not the narrowest maintainable fix without a stated install failure, validation, and docs/changelog update; a safer change would preserve or explicitly redesign the optional native decoder policy.
