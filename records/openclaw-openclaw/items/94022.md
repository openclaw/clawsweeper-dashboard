## What This Changes

The PR moves startup catch-up deferral IDs into cron service state, has maintenance recompute honor and clear them, updates cron tests, and adds a standalone runtime proof script.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main is source-reproducible because startup deferral IDs are only passed through start()'s local option, while read and timer maintenance recompute callers can run before the deferred slot without that exemption.

## Solution Assessment

Yes. Moving pending deferral IDs into CronServiceState is the clean shared boundary; threading local skip sets through every caller would duplicate scheduler policy, and disabling future repair globally would be too broad.
