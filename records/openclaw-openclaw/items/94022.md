## Summary

Keep open: the PR still owns a real cron scheduler bug that current main has not fixed, but the added runtime proof script should be isolated before merge because it writes through OpenClaw's shared SQLite state database rather than the temporary directory it later removes.

## What This Changes

The PR moves startup catch-up deferral exemptions into cron service state so read and timer maintenance recomputes preserve deferred missed-run slots until they fire.

## Best Possible Solution

Keep this as the canonical cron fix, isolate the proof script's state database, then let maintainers decide whether the in-process proof is enough to land or whether they want an additional live gateway restart proof.

## Reproduction Assessment

Yes. Current main is source-reproducible because read, manual preflight, finalize, release, and empty-due timer maintenance callers can recompute before the deferred slot without the startup-local skip set.

## Solution Assessment

Mostly yes. Moving pending deferral IDs into shared cron service state is the best owner boundary, but the proof script should use isolated state storage before merge.
