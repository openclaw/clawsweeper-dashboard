## What This Changes

The PR adds an in-memory cron service state set for startup catch-up deferral ids, uses it in maintenance recompute, records and clears it in startup/timer paths, and adds regression tests plus a standalone runtime proof script.

## Best Possible Solution

Land this branch as the canonical state-owned cron deferral fix if maintainers accept the remaining scheduler proof risk, then close the linked issue and duplicate unmerged PR attempts after merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because read/status, manual preflight, finalization, and timer maintenance recompute paths can call `recomputeNextRunsForMaintenance` without the startup-local deferral skip set before the deferred slot fires.

## Solution Assessment

Yes. Moving pending deferral ids into `CronServiceState` is the best current boundary because all affected maintenance recompute callers share the same future-slot repair invariant; continuing to thread one-off skip sets through selected callers would keep the policy duplicated.
