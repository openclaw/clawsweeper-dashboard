## What This Changes

The PR moves startup catch-up deferral IDs into `CronServiceState`, has maintenance recompute honor that shared state, cleans pending IDs, and adds cron regression coverage plus a standalone runtime proof script.

## Best Possible Solution

Land this branch as the canonical fix if maintainers accept the scheduler-risk tradeoff, then close the duplicate branches and linked issue after merge.

## Reproduction Assessment

Yes. Current main keeps pending overflow deferral IDs only in a one-shot `start()` option, while read, manual, and timer maintenance callers can run without that skip before the deferred slot fires.

## Solution Assessment

Yes. Moving pending deferral IDs into `CronServiceState` is the shared owner boundary for all maintenance recompute callers and avoids weaker alternatives like globally disabling future repair or threading another local skip through every caller.
