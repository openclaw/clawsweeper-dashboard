## Summary

Keep open: current main still has the cron startup overflow deferral gap, and this PR is a viable helper-level candidate fix for the linked issue. The patch has no blocking code finding from this review, but merge should wait for real behavior proof or an explicit maintainer proof waiver.

## What This Changes

The PR adds an in-memory pending catch-up deferral ID set to cron service state, records startup-deferred jobs, makes maintenance recompute consult and clear that set, removes the local startup-only skip option, and adds focused helper coverage.

## Best Possible Solution

Land one helper-level, state-owned deferral invariant after maintainers accept or add real behavior proof, then close the linked issue and duplicate PR attempts.

## Reproduction Assessment

Yes. Current main is source-reproducible: startup records deferred IDs only in a local skip set, while read/status, timer, and manual maintenance paths can call future-slot repair without that skip before the staggered slot fires.

## Solution Assessment

Yes. Moving the exemption into CronServiceState and enforcing it inside recomputeNextRunsForMaintenance is the best fix because all affected maintenance callers already share that helper; caller-by-caller skip plumbing is the incomplete pattern being replaced.
