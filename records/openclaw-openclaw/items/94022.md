## Summary

Keep open: the cron scheduler fix is still necessary and the state-owned exemption is the right direction, but the added runtime proof script can write to the default OpenClaw state database when run as documented, so that needs a narrow fix before merge.

## What This Changes

The PR adds in-memory tracking for startup catch-up deferral job IDs, uses it in cron maintenance recompute, records and clears IDs in startup/timer paths, and adds regression tests plus a standalone proof script.

## Best Possible Solution

Keep the state-owned cron deferral repair, isolate the proof script's state directory before merge, then land this canonical fix if maintainers accept in-process CronService proof instead of requiring a live gateway restart proof.

## Reproduction Assessment

Yes. Current main is source-reproducible because read/status, manual preflight, finalization, and timer maintenance recompute paths can call recomputeNextRunsForMaintenance without the startup-local deferral skip set before the deferred slot fires.

## Solution Assessment

Mostly yes. Moving pending deferral IDs into CronServiceState is the right shared boundary, but the proof script should isolate OPENCLAW_STATE_DIR before this is merge-ready.
