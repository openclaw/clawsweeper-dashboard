## What This Changes

The PR adds cron service state for pending startup catch-up deferral job ids, uses it during maintenance recompute, records and clears it from startup/timer paths, and adds regression tests plus a runtime proof script.

## Best Possible Solution

Land this branch as the canonical state-owned cron deferral fix if maintainers accept the remaining scheduler proof risk, then close the linked issue and duplicate unmerged attempts after merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because read/status, finalization, reservation release, and empty-due timer maintenance can call recomputeNextRunsForMaintenance without the startup-local deferral skip before the deferred slot fires.

## Solution Assessment

Yes. Moving pending deferral ids into CronServiceState is the best current boundary because all affected maintenance recompute callers share the same future-slot repair invariant; threading more one-off skip sets would keep the policy duplicated.
