## What This Changes

The PR moves startup catch-up deferral tracking into CronServiceState, makes recomputeNextRunsForMaintenance consult and clear that set, removes the local start() skip option, and adds focused regression coverage.

## Best Possible Solution

Land the helper-level state-owned deferral invariant after maintainers accept or add real behavior proof, then close the linked issue and duplicate PR attempts.

## Reproduction Assessment

Yes. Current main is source-reproducible: startup records deferred IDs only in a local skip set, while read/status, timer, and manual maintenance paths can call future-slot repair without that skip before the staggered slot fires; I did not execute tests because this review is read-only.

## Solution Assessment

Yes. Moving the exemption into CronServiceState and enforcing it inside recomputeNextRunsForMaintenance is the best fix because all affected maintenance callers already share that helper, while caller-by-caller skip plumbing is the incomplete pattern this cluster is replacing.
