## Summary

Keep open: current main still does not provide a durable, redacted channel-turn admission trail for the reported Telegram silent-dispatch class, and the exact no-dispatch-until-next-message trigger has not been reproduced well enough to close as fixed or obsolete.

## What This Changes

The issue requests that Telegram/channel messages either dispatch immediately or persist an explicit admission, reasonCode, history-recording, and active-run diagnostic when dispatch is suppressed.

## Best Possible Solution

Add a shared, redacted durable channel-turn diagnostic record for admission, reason/decisive gate, session-vs-pending-history recording, and active-run suppression state, then use it to isolate any remaining Telegram dispatch bug.

## Reproduction Assessment

No for the exact Telegram wake-on-next-message failure: the reporter provided plausible steps but no deterministic current-main reproduction. Source inspection does reproduce the diagnostic gap because admission and reply-run state are transient rather than persisted with the session turn.

## Solution Assessment

Unclear for a direct Telegram-only repair. The best maintainable solution is a shared redacted diagnostic surface first, followed by a targeted Telegram/session fix once the next occurrence has an admission and reason trail.
