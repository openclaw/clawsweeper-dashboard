## Summary

Current main already contains the queued-followup fix that matches this 2026.5.12 Telegram symptom: accepted user-request followups no longer inherit a superseded source abort signal, so they can drain and dispatch instead of waiting for a later message to expose the queued context.

## What This Changes

The issue requests dispatch or durable admission diagnostics for Telegram group messages that enter session context but do not trigger an immediate agent run.

## Best Possible Solution

Keep the current queued-followup abort contract on main, ship it in the next release, and open a new focused diagnostics issue only if a build containing a559ccc0844a5b07073f85bcb717b2093adb76f2 still records a Telegram user message without dispatching.
