## Summary

This should stay open: the report is fresh, has concrete current-release iMessage group media evidence, and current `main` still routes normal final media through the generic iMessage RPC `send` path while the known-good `send-rich --file` bridge route is only wired for message-tool actions.

## What This Changes

Report that generated image files from allowlisted iMessage group turns are saved but the final media reply does not appear, while manual `imsg send-rich` of the same file to the group succeeds.

## Best Possible Solution

Fix the iMessage final media path so group media replies use a verified group-capable bridge route or emit a clear retryable failure, while preserving media resolver safeguards.

## Reproduction Assessment

No high-confidence live reproduction was performed against a real macOS iMessage group. The report's repeated live evidence plus current source routing makes the bug source-reproducible enough to keep open.

## Solution Assessment

Yes, a narrow iMessage outbound-media fix is the right direction. The safest approach is to reuse the existing resolver and bridge capability checks, not add a parallel raw-path send path.
