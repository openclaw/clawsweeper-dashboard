## Solution Assessment

Yes for the code shape: startup reconciliation is the warning source, and OpenClaw already treats oneshot runtime handles as terminal by closing and clearing them after the turn while only persistent mode gets resume ids. The safer pre-merge path is to keep this guard and strengthen the real startup proof, not move the fix to Codex or add a broader state migration.
