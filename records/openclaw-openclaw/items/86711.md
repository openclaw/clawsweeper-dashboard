## What This Changes

## Best Possible Solution

## Solution Assessment

Yes, the patch is the narrow maintainable fix: skip terminal oneshot records before runtime reconciliation while preserving persistent pending identity repair. The blocker is proof of the real startup behavior, not a different code shape.
