## Summary

Keep open: current main still has the reported event-contract mismatch, and the same bug is already tracked by an open draft fix PR that needs normal review/proof before the issue should close.

## What This Changes

The issue asks OpenClaw to stop surfacing `Compaction incomplete` when a Codex app-server `contextCompaction` item completes successfully.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81593, or an equivalent narrow patch, then close this issue once current main or a release carries the completed compaction event contract.

## Reproduction Assessment

Yes, by source inspection: a Codex `contextCompaction` `item/completed` currently emits a compaction `end` event without `completed: true`, and the reply runner routes that shape to the incomplete notice when `notifyUser` is enabled. I did not execute a live Discord run in this read-only review.

## Solution Assessment

Yes: adding `completed: true` to the already-successful Codex app-server compaction end event is the narrowest maintainable fix, and it matches the open draft PR's approach without changing broader compaction policy.
