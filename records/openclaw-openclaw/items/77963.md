## Summary

Keep open: the linked bug is plausible, but this PR is not mergeable as-is because the sanitizer misses ordinary runtime/proxy fetch paths, can drop valid iterable header inputs, and still needs real OpenClaw runtime proof.

## What This Changes

The PR adds `sanitizeHeadersInit` and calls it from guarded redirect/body handling, proxy/runtime FormData normalization, and the debug proxy global fetch patch.

## Best Possible Solution

Land one compatibility fix that filters only plain-object Symbol-keyed header dictionaries at the actual fetch boundaries, preserves `Headers` and iterable inputs, adds focused regression tests, and includes contributor real behavior proof for the OpenClaw plus discord.js path.

## Reproduction Assessment

Yes for the core failure mode: local Node/Undici rejects `new Headers()` from a Symbol-keyed record, and source inspection shows current main can forward raw `init.headers` through runtime/proxy paths. I did not independently run the full OpenClaw plugin plus discord.js login scenario.

## Solution Assessment

No. Boundary sanitization is the right narrow direction, but this implementation runs after early returns in two dispatch paths and converts valid iterable header inputs into empty records; a safer fix should preserve valid `HeadersInit` forms and filter only plain symbol-keyed dictionaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Sanitize runtime headers before early returns:** `src/infra/net/runtime-fetch.ts:63`
  - body: The sanitizer only runs after `normalizeRuntimeFormData` changes the body. Header-only requests, and requests whose body already matches the runtime FormData implementation, still return the original `init`, so Symbol-keyed `init.headers` can still reach undici through `fetchWithRuntimeDispatcher`.
  - confidence: 0.88
- **[P2] Sanitize proxy headers before the FormData-only return:** `src/infra/net/proxy-fetch.ts:51`
  - body: `normalizeInitForUndici` still returns the original init for ordinary GET/JSON requests and undici FormData requests. That leaves proxy-backed fetches able to pass Symbol-keyed headers into `undiciFetch`, so env-proxy paths can still hit the crash this PR is meant to prevent.
  - confidence: 0.86
- **[P2] Preserve iterable header inputs when filtering records:** `src/infra/net/redirect-headers.ts:40-44`
  - body: For every non-array, non-global `Headers` object, the sanitizer rebuilds from `Object.entries()`. Valid iterable or foreign Headers-like inputs can have no enumerable string properties but are accepted by `new Headers()`, so this can silently drop auth/content headers in the redirect, proxy, runtime, and capture paths.
  - confidence: 0.84
