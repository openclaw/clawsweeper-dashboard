## Summary

Keep open: the reported Node Headers crash is valid and the PR targets the right boundary, but the patch still leaves ordinary runtime/proxy fetch calls unsanitized and can drop valid iterable header inputs.

## What This Changes

The PR adds a `sanitizeHeadersInit` helper and calls it from redirect/header rewrite code, proxy/runtime FormData normalization, and the debug proxy global fetch patch.

## Best Possible Solution

Keep the boundary-sanitization direction, but normalize headers independently of body conversion, preserve valid iterable/Headers-like inputs, add focused regression tests, and require real OpenClaw runtime proof before merge.

## Reproduction Assessment

Yes for the core failure mode: native `new Headers()` rejects Symbol-keyed records, and current main has runtime/proxy paths that can forward raw `init.headers`. I did not run a full discord.js plugin reproduction in this checkout.

## Solution Assessment

No, not yet. Boundary sanitization is the right narrow fix, but this implementation must sanitize before early returns and avoid changing valid non-plain `HeadersInit` inputs into empty records.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Sanitize runtime headers before early returns:** `src/infra/net/runtime-fetch.ts:63`
  - body: The sanitizer only runs after `normalizeRuntimeFormData` changes the body. Header-only requests, and requests whose body already matches the runtime FormData implementation, still return the original `init`, so Symbol-keyed `init.headers` can still reach undici through `fetchWithRuntimeDispatcher`. Sanitize headers independently before those returns.
  - confidence: 0.88
- **[P2] Sanitize proxy headers before the FormData-only path:** `src/infra/net/proxy-fetch.ts:51`
  - body: `normalizeInitForUndici` still returns the original init for ordinary GET/JSON requests and undici FormData requests. That means proxy-backed fetches can still pass Symbol-keyed headers into `undiciFetch`, leaving the crash this PR is meant to prevent on env-proxy paths.
  - confidence: 0.86
- **[P2] Preserve iterable header inputs while filtering records:** `src/infra/net/redirect-headers.ts:40-44`
  - body: For every non-array, non-global `Headers` object the sanitizer rebuilds from `Object.entries()`. Valid iterable or foreign Headers-like inputs can have no enumerable string properties but are accepted by `new Headers()`, so this can silently drop auth/content headers in the redirect, proxy, runtime, and capture paths.
  - confidence: 0.84
