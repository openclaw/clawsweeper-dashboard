## Summary

Keep open: the fix direction is right, but the patch still leaves one redirect `new Headers()` path exposed, mutates caller-owned init in debug proxy capture, lacks the required changelog entry, and still needs real OpenClaw/plugin behavior proof before merge.

## What This Changes

The PR adds a `sanitizeHeadersInit` helper and applies it to proxy fetch, runtime fetch, redirect header retention, and debug proxy global-fetch handling.

## Best Possible Solution

Normalize `HeadersInit` at every OpenClaw-to-native `Headers` or fetch boundary, avoid caller init mutation, add focused regression coverage and a single-line changelog entry, then merge after Node 22 proof from a real OpenClaw plugin/global-fetch path.

## Reproduction Assessment

Yes. The dependency boundary failure is reproducible with native `new Headers()` on a Symbol-keyed record, and current main clearly passes `init.headers` through proxy/runtime and redirect paths without normalization; I did not establish a full live Discord plugin run in this checkout.

## Solution Assessment

No, not yet. Boundary sanitization is the right approach, but this implementation should also cover `dropBodyHeaders`, avoid mutating caller init, satisfy changelog/regression coverage expectations, and provide real OpenClaw/plugin proof before merge.
