## Summary

Keep open: the latest PR revision appears to address the earlier sanitizer-placement gaps, but it still needs after-fix proof through the actual OpenClaw/plugin fetch path before maintainer merge, plus one small debug-proxy cleanup.

## What This Changes

The PR adds a `sanitizeHeadersInit` helper and applies it in proxy fetch, runtime fetch, redirect-header, and debug proxy capture paths to strip Symbol keys from header records before native header handling.

## Best Possible Solution

Normalize `HeadersInit` at every OpenClaw-to-native fetch/header boundary without mutating caller-owned init objects, add focused Symbol-keyed header regression coverage, and merge after actual OpenClaw/plugin fetch proof on Node 22.

## Reproduction Assessment

Yes for the underlying bug from source: Node's native `Headers` throws on Symbol-keyed header records, and current main has common proxy/runtime fetch paths that pass `init.headers` through unchanged. I did not reproduce the full discord.js plugin flow in a live OpenClaw run.

## Solution Assessment

Yes, the sanitizer-at-boundary direction is the narrow maintainable fix. The safer final version should avoid mutating debug-proxy caller init objects and include regression coverage plus real OpenClaw/plugin proof.
