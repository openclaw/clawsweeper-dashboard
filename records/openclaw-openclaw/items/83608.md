## Summary

Keep this PR open but do not merge it as written: the root-cause target is plausible and the updated terminal proof is useful, but the patch changes shared workspace setup semantics instead of limiting the fix to the runtime subagent path.

## What This Changes

The PR changes `ensureAgentWorkspace` so optional bootstrap files are created only in brand-new workspaces and adds tests for removed optional files in existing workspaces.

## Best Possible Solution

Preserve setup and onboarding repair semantics, then fix the reported runtime path by resolving the correct agent workspace or adding a clearly scoped runtime-only mode that does not seed unwanted root templates.

## Reproduction Assessment

Yes, at source level: current main calls the shared workspace helper from the run path, and that helper writes missing optional files unless explicitly skipped. I did not run a live subagent reproduction because this review was read-only.

## Solution Assessment

No. The PR fixes the symptom by changing the shared setup helper for every existing workspace, but the maintainable path is to preserve setup repair semantics and narrow the no-seed behavior to the runtime path or an explicit maintainer-approved mode.
