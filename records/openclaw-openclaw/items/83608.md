## Summary

Keep open: the bug is credible, but this PR takes a broad workspace-seeding shortcut that can break setup/onboard behavior and still does not address the wrong workspace routing or required `TOOLS.md` creation from the linked report.

## What This Changes

This PR changes `ensureAgentWorkspace` so SOUL, IDENTITY, USER, and HEARTBEAT are created only for brand-new workspaces and updates workspace tests for that behavior.

## Best Possible Solution

Fix the spawned-run workspace inheritance, or add an explicit runtime-only seeding mode, so subagent spawns stop dirtying the repository root while setup/onboard/configure keep their documented missing-file seeding behavior.

## Reproduction Assessment

Yes, at source level: same-agent subagent spawn can inherit `ctx.workspaceDir`, `agent-command` can then call `ensureAgentWorkspace` on that inherited path, and current main writes bootstrap templates before setup-state checks. I did not run the live Telegram/macOS flow.

## Solution Assessment

No. The broad brand-new-workspace guard changes shared setup behavior and misses `TOOLS.md`; the narrower maintainable fix belongs in spawned workspace routing or a runtime-only seeding mode.
