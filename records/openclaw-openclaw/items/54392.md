The PR clamps compaction summary `reserveTokens` by the selected model output limit before calling Pi `generateSummary`, adds reserve-token clamping tests plus one mock alignment, and records a changelog fix entry.

## Best Possible Solution

Land this OpenClaw wrapper clamp with its regression coverage after targeted compaction tests and the changed gate pass; any upstream Pi helper change can remain a separate dependency follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main can pass `reserveTokens: 300000` into the pinned Pi summary helper, which computes a 240000 output-token request, while the affected Anthropic Vertex model entries cap output at 128000.

## Solution Assessment

Yes. Clamping in `summarizeChunks` is the narrowest maintainable OpenClaw-side fix because all built-in fallback and staged summarization paths route through that wrapper before the pinned Pi helper derives provider `max_tokens`.
