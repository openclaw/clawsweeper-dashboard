## What This Changes

The PR clamps compaction summary `reserveTokens` by the selected model output limit before calling Pi `generateSummary`, adds reserve-token clamping tests, updates an existing compaction mock, and adds a changelog fix entry.

## Best Possible Solution

Land the OpenClaw-side wrapper clamp with its regression coverage after targeted compaction tests and the changed gate pass; keep any upstream Pi helper change as a separate dependency follow-up.

## Solution Assessment

Yes. Clamping inside `summarizeChunks` is the narrowest maintainable OpenClaw-side fix because both the normal and fallback/staged summarization paths pass through that wrapper before the pinned Pi helper derives provider `max_tokens`.
