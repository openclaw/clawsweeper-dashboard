## Summary

Keep open. Current main still forwards an unclamped compaction `reserveTokens` value into the pinned Pi summary helper, while the PR applies the narrow wrapper clamp, adds focused regression coverage, and has no blocking review findings.

## What This Changes

This PR clamps compaction summary `reserveTokens` by the selected model output limit, adds reserve-token clamping tests plus a mock alignment, and records a changelog fix entry.

## Best Possible Solution

Merge a current-main-compatible version of this wrapper clamp after targeted compaction tests and the Testbox changed gate pass; any upstream Pi helper clamp can remain a separate dependency follow-up.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main can pass `reserveTokens: 300000` into the pinned Pi helper, which derives a 240000 output-token request while affected model entries cap output at 128000.

## Solution Assessment

Yes. Clamping inside `summarizeChunks` is the narrowest OpenClaw-side fix because the built-in fallback and staged summarization paths route through that wrapper before the pinned dependency derives provider `max_tokens`.
