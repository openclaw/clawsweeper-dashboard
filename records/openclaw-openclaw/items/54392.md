## Summary

This PR should stay open for maintainer validation: current main still forwards raw compaction reserve tokens into the pinned Pi summary helper, while the PR adds the narrow clamp, regression coverage, and changelog entry with no blocking review findings.

## What This Changes

The PR clamps compaction `reserveTokens` to the model output-token ceiling before calling Pi summary generation, adds focused reserve-token clamping tests, aligns one compaction mock, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Merge a current-main-compatible version of this wrapper clamp after targeted compaction tests and the changed gate pass; any upstream Pi helper cap can be handled separately.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: current main can pass `reserveTokens: 300000` into the pinned Pi helper, which derives a 240000 output-token request while the affected Anthropic Vertex models cap output at 128000.

## Solution Assessment

Yes. Clamping inside `summarizeChunks` is the narrowest OpenClaw-side fix because full, fallback, and staged summarization all route through that wrapper before the pinned dependency derives provider `max_tokens`.
