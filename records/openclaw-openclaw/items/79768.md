## Summary

Keep open: this PR targets a real Bedrock thinking-policy bug, but the proposed provider-policy API still misses the reported Bedrock-prefixed Claude model IDs and the branch also carries unrelated Google and MiniMax changes.

## What This Changes

The branch adds an Amazon Bedrock provider-policy API for thinking profiles and also changes Google onboarding defaults plus MiniMax portal OAuth catalog handling/tests.

## Best Possible Solution

Keep the Bedrock fix plugin-owned, share or hoist the Bedrock-aware thinking-profile resolver into the new policy artifact, add regression coverage for prefixed Bedrock Claude IDs, and split unrelated provider changes into separate PRs.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the Bedrock provider-policy artifact used by directive/session validation while the runtime Bedrock hook and docs support adaptive Claude 4.6 thinking; I did not run a live AWS Bedrock session.

## Solution Assessment

No. The proposed Bedrock policy delegates raw Bedrock model IDs to a helper that only matches direct claude-* prefixes, so it does not solve the reported global.anthropic/us.anthropic Claude 4.6 path.
