## What This Changes

This PR adds an Amazon Bedrock provider-policy surface for thinking profiles and also changes Google onboarding defaults plus MiniMax portal OAuth catalog handling/tests.

## Best Possible Solution

Keep the Bedrock fix plugin-owned by adding a Bedrock provider-policy artifact that reuses or shares the Bedrock-aware thinking resolver, covers prefixed Claude 4.6/4.7 IDs in tests, and splits unrelated provider changes out.

## Reproduction Assessment

Yes. Source inspection against current main shows the loader falls back to a missing Bedrock provider-policy artifact while docs/runtime tests expect adaptive for Bedrock Claude 4.6; I did not run a live AWS Bedrock session.

## Solution Assessment

No. The provider-policy artifact is the right boundary, but this implementation must normalize or Bedrock-match global.anthropic/us.anthropic model IDs before resolving the thinking profile.
