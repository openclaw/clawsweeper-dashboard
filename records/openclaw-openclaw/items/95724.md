## Summary

Keep open: current main and v2026.6.10 still use per-agent built-in memory SQLite indexes, and the directory-keyed shared-index design remains unmerged product-sensitive memory-core work.

## What This Changes

The issue requests changing memory vector-index ownership from per-agent databases to source-directory keyed shared stores so agents using the same workspace avoid duplicate embeddings, storage, and watchers.

## Best Possible Solution

Design and review a first-class memory-core shared-index model keyed by canonical source set, with explicit agent-scope filtering, watcher ownership, upgrade/rebuild behavior, tests, docs, and security review.

## Reproduction Assessment

Yes, source-level: current main resolves built-in memory storage through agentId, caches built-in managers by agentId, and v2026.6.10 still documents per-agent index storage. I did not run a live multi-agent reindex because this review is read-only.

## Solution Assessment

Unclear as an implementation direction: source-directory keyed stores target the real waste, but the storage layout, migration/rebuild behavior, privacy scope, and watcher lifecycle need maintainer sign-off before a fix should be queued.
