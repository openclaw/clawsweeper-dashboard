## Summary

Keep open: current main and v2026.6.9 still store built-in memory indexes in per-agent SQLite databases, and the open linked implementation candidate has unresolved storage, session-scope, and upgrade blockers.

## What This Changes

Request shared built-in memory vector indexes keyed by canonical memory source directories so same-workspace agents do not duplicate embeddings, storage, and watchers.

## Best Possible Solution

Review or replace the linked PR with a current-main shared-index design that explicitly handles session isolation, source membership, watcher ownership, upgrade/rebuild behavior, tests, and docs.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 both resolve built-in memory storage through the agent id and cache managers by agent id. I did not run a live multi-agent reindex because this review is read-only and the source path is clear.

## Solution Assessment

No current implementation is ready to close the issue. The source-directory key is a plausible direction, but the active PR still needs session-scope, migration, and current-main conflict resolution before it can be the best fix.
