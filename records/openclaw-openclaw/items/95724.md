## Summary

Keep open: current main and v2026.6.10 still use per-agent builtin memory SQLite indexes, while the requested source-directory keyed shared index remains a valid but product-sensitive memory storage design change. The open candidate PR needs maintainer review before this issue can close.

## What This Changes

Request automatic source-directory keyed memory indexes so same-workspace agents share one vector store instead of rebuilding duplicate per-agent stores.

## Best Possible Solution

Design a first-class memory-core shared-index model keyed by canonical source set, with explicit agent membership or scope filtering, watcher ownership, upgrade/rebuild behavior, regression tests, and public docs.

## Reproduction Assessment

Yes, at source level: current main resolves builtin memory storage through `agentId`, caches managers by `agentId`, and v2026.6.10 still documents per-agent index storage. I did not run a live multi-agent reindex because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the storage and privacy contract. The source-directory keyed idea targets the real duplication, but the best implementation must also settle migration/rebuild behavior, session visibility, watcher ownership, and SDK exposure.
