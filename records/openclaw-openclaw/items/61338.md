## Summary

Keep open. Current main still uses raw string equality between `metadata.openclaw.os` and Node platform strings in the skills status/check path, so `macos` does not satisfy `darwin`; the report is recent, reproducible, and narrow enough for a focused fix.

## What This Changes

Treat a skill OS requirement of `macos` as eligible on macOS/Darwin when running `openclaw skills check`.

## Best Possible Solution

Add shared platform-name canonicalization for skill OS requirements, runtime eligibility, and installer filters, then document and test `macos` as an alias for `darwin`.

## Solution Assessment
