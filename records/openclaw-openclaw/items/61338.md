## Summary

Keep open. Current main still compares skill OS requirements against raw Node platform strings, so `os: macos` is not satisfied by macOS/Darwin in the skills check path; the latest release tag has the same exact-match behavior.

## What This Changes

The issue asks for `openclaw skills check` to treat a skill requirement of `os: macos` as satisfied on macOS/Darwin.

## Best Possible Solution

Add shared OS/platform canonicalization for skill requirements, runtime eligibility, and installer filters, then document accepted aliases and add regression coverage for `macos` on `darwin`.

## Solution Assessment

Yes. The narrow maintainable fix is a shared platform normalizer used by requirement evaluation, runtime eligibility, and install filtering, rather than a CLI-only display change or a docs-only workaround.
