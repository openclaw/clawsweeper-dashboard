## What This Changes

The branch adds `mediaLocalRoots` to Mattermost `handleAction`, forwards it to `sendMessageMattermost`, and adds a regression test for send actions with media.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main passes scoped media roots into channel action dispatch, but Mattermost `handleAction` drops them before calling `sendMessageMattermost`, so workspace media falls back to default roots that reject `workspace-*` paths.

## Solution Assessment

Yes for this PR's stated bug. Forwarding the existing `mediaLocalRoots` field at the Mattermost action adapter is the narrow maintainable fix, while the broader Mattermost cluster can handle threadId, media aliases, and mediaReadFile follow-through.
