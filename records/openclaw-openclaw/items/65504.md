## Summary

Keep open. Current main and the latest release still expose flat `agent.wait`/dedupe/lifecycle timeout results and auth-profile timeout handling still has no provider-started distinction; the broad implementation PR was closed unmerged and the scheduler and QA pieces remain split into open follow-up tracks.

## What This Changes

The issue requests phase-aware live gateway timeout attribution, provider-started auth-profile failure accounting, lower post-turn/session-lane latency, and cleanup of orphan QA gateway noise.

## Best Possible Solution

Keep this as the canonical timeout-attribution tracker, implement an additive Gateway/agent wait contract with phase/provider-started fields and regression coverage, and coordinate scheduler fairness through https://github.com/openclaw/openclaw/issues/54488 and QA cleanup through https://github.com/openclaw/openclaw/issues/65506.
