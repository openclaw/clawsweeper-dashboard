## Summary

Keep open. Current main still lacks the phase-aware `agent.wait` timeout attribution/provider-started contract, the broad implementation PR is closed unmerged, and the adjacent scheduler and QA cleanup tracks remain split rather than shipped.

## What This Changes

The issue asks OpenClaw to stop flattening live gateway queue, preflight, provider, post-turn, and QA delays into Anthropic timeout failures and to improve responsiveness attribution.

## Best Possible Solution

Keep this as the canonical timeout-attribution tracker, implement the Gateway/agent contract additively with tests and docs, and leave scheduler fairness to https://github.com/openclaw/openclaw/issues/54488 and QA cleanup to https://github.com/openclaw/openclaw/issues/65506 unless maintainers intentionally merge those scopes.

## Reproduction Assessment

Yes for the source-level contract gap: current `agent.wait` can still return a flat timeout, dedupe/lifecycle snapshots lack phase fields, and auth timeout handling has no provider-started distinction. The live provider incident itself was not rerun.

## Solution Assessment

No as one broad umbrella patch. The maintainable path is a focused additive Gateway/agent timeout-attribution contract, with scheduler fairness and QA harness cleanup kept in their linked tracks unless maintainers choose otherwise.
