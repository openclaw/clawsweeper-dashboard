The issue reports a regression where configured heartbeat schedules stop firing or stop reaching the main agent session on recent 2026.5.x releases, especially with `heartbeat.target: "none"`.

## Best Possible Solution

Reproduce the gateway startup-to-first-fire path for `agents.defaults.heartbeat.every` with `target: "none"`, then either land the relevant pending-final-delivery fix or add a narrower fix that proves the scheduler starts and the main-session model turn runs.
