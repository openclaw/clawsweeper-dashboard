The issue reports that configured heartbeat scheduling no longer fires or routes a prompt into the main agent session on recent OpenClaw releases.

## Best Possible Solution

Add a focused gateway heartbeat regression that distinguishes empty HEARTBEAT.md skips from a real startup-to-model failure, then either fix the activation/routing path or make the fresh-setup skip clearly observable and documented.

No high-confidence live reproduction: I did not run a gateway against the reporter's Ubuntu/npm/Ollama setup. ## Solution Assessment

Unclear: closing is not supported, and the best next step is a narrow regression or live proof that separates documented empty-file skips from a real scheduler/activation/routing failure.
