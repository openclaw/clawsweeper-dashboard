## Summary

Keep open: this PR addresses a valid Feishu button-card regression and includes sufficient real behavior proof, but the patch is not merge-ready because it introduces a Feishu lint failure and leaves an existing launcher-card test asserting the old action container shape.

## What This Changes

The PR replaces Feishu button rows emitted as V1 action containers with V2 column_set rows across outbound presentation cards, approval cards, launcher cards, and related tests.

## Best Possible Solution

Keep the owner-local Feishu column_set migration, add the small lint/test repairs, then land it with Feishu-focused validation and let maintainers handle the release changelog entry.

## Reproduction Assessment

Yes for source reproduction: current main serializes Feishu button cards with tag "action" on the approval, launcher, and presentation paths. I did not independently call the Feishu API, but the PR supplies redacted live before/after gateway logs for the external rejection and fixed click path.

## Solution Assessment

Mostly yes: a shared Feishu plugin helper that emits column_set rows is the narrow maintainable fix. This patch still needs the local lint and launcher-test repairs before it is the best mergeable version.
