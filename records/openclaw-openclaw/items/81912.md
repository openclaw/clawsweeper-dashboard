## Summary

Keep this PR open: the retry direction is valid, and the supplied proof is useful, but the patch leaves client-provided tools available during the supposedly no-tool final-answer continuation.

## What This Changes

The PR adds a one-shot Pi runner retry for completed tool calls followed by an empty final assistant answer, plus focused regression coverage.

## Best Possible Solution

Land the completed-tool empty-final retry only after the forced continuation suppresses every tool surface, including client-provided tools, and covers that boundary in the regression test.

## Reproduction Assessment

Yes. The source path is clear from current main: completed tool metadata plus no payloads reaches the incomplete-turn warning, and the PR's harness proof models that path; I did not run local tests in this read-only review.

## Solution Assessment

No, not yet. The retry is the right narrow fix, but it must also suppress clientTools so the final-answer continuation cannot make another tool call.
