## Summary

## What This Changes

Adds a default-off assistant-conclusion freshness gate, replay metadata tagging/omission, prompt-build wiring, and focused agent tests for plugin/config state questions.

## Best Possible Solution

Keep the gate default-off, rebase onto current main, make freshness matching target-aware for nested config and plugin-specific evidence while still allowing full config snapshots, then add regression coverage and real proof.

## Reproduction Assessment

Yes. By source inspection, a focused helper test can place recent plugins.entries.beta.enabled gateway evidence in history and ask about alpha; the PR can mark it fresh because plugin_install_state currently bypasses target matching.

## Solution Assessment

No. The default-off gate direction is plausible, but this implementation is not yet the best solution because target extraction and plugin-state matching are too broad.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve nested config target hints:** `src/agents/assistant-conclusion-freshness-gate.ts:225-233`
  - body: `extractConfigTargetHint` returns `plugins.entries` or `plugins.installs` before considering the full dotted path, so a prompt for `plugins.entries.alpha.enabled` is reduced to the root. Since same-or-nested matching is accepted, recent evidence for `plugins.entries.beta.enabled` can be treated as fresh for alpha and skip the intended re-check.
  - confidence: 0.92
- **[P2] Match plugin-state freshness to the queried target:** `src/agents/assistant-conclusion-freshness-gate.ts:261-262`
  - body: `matchesDiagnosticTarget` returns true for every `plugin_install_state`, but gateway reads such as `plugins.entries.beta.enabled` are classified as plugin-list diagnostics with a target. A later question about a different plugin can reuse beta-only evidence and avoid the fresh tool call the gate is meant to require.
  - confidence: 0.87
- **[P3] Ignore openclaw.json when choosing target hints:** `src/agents/assistant-conclusion-freshness-gate.ts:232-233`
  - body: The dotted-path fallback returns the first dotted token, so prompts like "in openclaw.json, is gateway.mode set?" can derive `openclaw.json` instead of `gateway.mode`. That makes fresh key-specific evidence look unrelated and keeps forcing unnecessary tool calls.
  - confidence: 0.83
