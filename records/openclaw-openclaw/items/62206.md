## What This Changes

The PR adds a default-off assistant-conclusion freshness gate for plugin/config state prompts, diagnostic replay metadata tagging, stale diagnostic replay omission, prompt-build integration, and focused agent tests.

## Best Possible Solution

Rebase the feature onto current main, keep the gate default-off, fix target extraction so exact nested config paths are preserved while document names like openclaw.json are ignored as hints, add regression coverage for those cases, and include the required changelog entry before normal maintainer review.

## Reproduction Assessment

Yes. With the flag enabled, a unit test can ask about plugins.entries.alpha.enabled after recent evidence for plugins.entries.beta.enabled; the current extractConfigTargetHint collapses both prompts to plugins.entries, so the gate can incorrectly mark unrelated sibling evidence as fresh.

## Solution Assessment

No. The default-off gate direction is plausible, but this PR's current target-hint logic is not the narrowest safe implementation because it loses the specific config key needed to prevent stale sibling evidence reuse.
