## Summary

Keep open. Current main still lacks this feature, but the PR is not correct as-is because its target-hint extraction can reuse unrelated sibling config evidence, can miss valid evidence when a prompt mentions openclaw.json before the actual key, and is missing the required changelog entry.

## What This Changes

The PR adds a default-off assistant conclusion freshness gate, replay metadata tagging and omission, prompt-build integration, and focused agent tests for plugin/config state prompts.

## Best Possible Solution

Rebase the feature onto current main, keep it default-off, preserve exact nested config target hints, ignore document filenames like openclaw.json when extracting key hints, add regression coverage, and include the required active-version changelog entry.

## Reproduction Assessment

Yes. With the flag enabled, a unit test can ask about plugins.entries.alpha.enabled after recent evidence for plugins.entries.beta.enabled; the current helper collapses both to plugins.entries, so unrelated sibling evidence can be treated as fresh.

## Solution Assessment

No. The default-off gate direction is plausible, but this PR's current target extraction is not the narrowest maintainable implementation because it loses the specific config key needed to prevent stale sibling evidence reuse.
