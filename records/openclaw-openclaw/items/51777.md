## Summary

This PR should stay open, but it is not mergeable as written: the reported Feishu persistence bug is still source-reproducible on current main, while the branch edits the old command implementation instead of the active onboarding flow and still lacks inspectable real behavior proof.

## What This Changes

The PR adds a Feishu-specific same-value DM-policy write in `src/commands/onboard-channels.ts` when `channels.feishu.dmPolicy` is missing.

## Best Possible Solution

Move a presence-aware Feishu missing-key write into `src/flows/channel-setup.prompts.ts`, preserve the same-value guard for other channels, and add focused regression coverage plus inspectable after-fix proof.

## Reproduction Assessment

Yes by source inspection. On a fresh Feishu config, selecting `pairing` makes Feishu `getCurrent` return the fallback `pairing`, the active guard skips `setPolicy`, and the writer does not materialize runtime defaults.

## Solution Assessment

No. The Feishu-scoped idea is safer than a global unconditional setter, but this patch applies it to a stale command file instead of the active flow helper.
