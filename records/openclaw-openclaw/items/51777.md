## Summary

Keep open: current main still has the Feishu same-value persistence bug, and this PR targets a stale pre-refactor command body instead of the live setup prompt path.

## What This Changes

The branch adds a Feishu-only missing-key `setPolicy` call to the old onboarding DM-policy loop so selecting `pairing` on a fresh config writes `channels.feishu.dmPolicy`.

## Best Possible Solution

Patch the live DM-policy prompt path with a presence-aware Feishu write for the resolved top-level or account-scoped `dmPolicy` key, keep same-value no-ops for non-idempotent setters, and add regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: fresh Feishu config with no resolved `dmPolicy`, choose `pairing`, `getCurrent` returns fallback `pairing`, the same-value guard skips `setPolicy`, and the writer does not persist runtime defaults.

## Solution Assessment
