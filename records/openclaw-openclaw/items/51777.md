The PR adds a Feishu-specific fallback write in onboarding so selecting the default `pairing` DM policy persists `channels.feishu.dmPolicy` when that key is absent.

## Best Possible Solution

Move a presence-aware Feishu missing-policy write into `src/flows/channel-setup.prompts.ts`, keep the same-value guard for other channels unless setters are audited, add focused regression coverage, and require redacted after-fix proof.

## Reproduction Assessment

## Solution Assessment

No. The Feishu-scoped presence-aware idea is safer than a global unconditional setter, but this branch applies it to stale command code instead of the active setup flow helper.
