## Best Possible Solution

Move a presence-aware Feishu missing-policy write into `src/flows/channel-setup.prompts.ts`, preserve the same-value guard for other channel setters, add focused regression coverage, and require redacted after-fix proof.

## Reproduction Assessment

Yes by source inspection, though I did not run the live wizard in this read-only review. A fresh Feishu config selecting `pairing` reaches Feishu's fallback `current` value, the active guard skips `setPolicy`, and the config writer does not serialize runtime defaults.

## Solution Assessment

No. The Feishu-scoped missing-key idea is safer than a global unconditional setter, but this patch applies it to the inactive command file instead of the active flow helper.
