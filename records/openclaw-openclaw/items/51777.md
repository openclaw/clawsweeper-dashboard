## Summary

Keep open: the Feishu onboarding persistence bug is still source-reproducible on current main, but this PR now patches the obsolete command body instead of the active flow helper and lacks inspectable real behavior proof.

## What This Changes

The PR adds a Feishu-specific fallback write so selecting the default `pairing` DM policy during onboarding should persist `channels.feishu.dmPolicy` on a fresh config.

## Best Possible Solution

Move a presence-aware Feishu write into the active `maybeConfigureDmPolicies` flow helper using the resolved config key, preserve the same-value guard for other channels, and add focused regression coverage plus real proof.

## Reproduction Assessment

Yes by source inspection. On a fresh Feishu config, selecting `pairing` makes Feishu `getCurrent` return the fallback `pairing`, the active guard skips `setPolicy`, and the writer does not materialize runtime defaults.

## Solution Assessment

No. The intended Feishu-specific persistence fix is narrow, but this PR applies it to a stale command file; a global unconditional setter is also unsafe because at least Twitch has non-idempotent setter behavior.
