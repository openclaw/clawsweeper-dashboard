## Best Possible Solution

Move a presence-aware Feishu missing-policy write into `src/flows/channel-setup.prompts.ts`, preserve the same-value guard for other channels, add focused regression coverage, and require redacted after-fix proof.

## Reproduction Assessment

## Solution Assessment

No. The Feishu-scoped idea is safer than a global unconditional setter, but this patch applies it to the stale command file instead of the active flow helper.
