## Summary

Keep open: the reported Feishu persistence bug is source-reproducible, but this branch is stale against current main and does not patch the active onboarding flow, and the external PR still needs real behavior proof before merge.

## What This Changes

The PR attempts to persist Feishu `dmPolicy` during onboarding when a fresh config lacks the key and the user selects the fallback `pairing` policy.

## Best Possible Solution

Patch `src/flows/channel-setup.prompts.ts` with a presence-aware Feishu write based on the resolved policy key, preserve the no-op guard for non-idempotent channels, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: fresh Feishu config, opt into DM-policy setup, select `pairing`, Feishu `getCurrent` returns fallback `pairing`, the active guard skips `setPolicy`, and the writer does not materialize runtime defaults.

## Solution Assessment

No. The intended fix is valid, but this PR patches the obsolete command body instead of the active flow helper, and a global unconditional setter remains unsafe for non-idempotent channels such as Twitch.
