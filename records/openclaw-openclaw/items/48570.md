- **[P1] Preserve poll vote data before enqueueing updates:** `extensions/whatsapp/src/inbound/monitor.ts:472`
  - body: `messages.update` provides each voter's pollUpdate, but this synthetic message replaces it with `pollUpdateMessage: {}`. Downstream hooks and agents can only see a generic placeholder, so workflows still cannot tell which option was selected.
  - confidence: 0.9
- **[P1] Keep reactions out of normal reply turns:** `extensions/whatsapp/src/inbound/monitor.ts:547`
  - body: This enqueues every reaction event as an ordinary inbound message. A reaction can therefore create an agent turn and possible outbound reply instead of a listen-only reaction hook behind an explicit WhatsApp opt-in.
  - confidence: 0.84
- **[P2] Route event RSVP updates from messages.update:** `extensions/whatsapp/src/inbound/monitor.ts:444-445`
  - body: The handler exits unless `pollUpdates` is present, but Baileys also carries RSVP changes on `eventResponses`. Those updates never become synthetic messages, so the advertised RSVP support still misses the update path.
  - confidence: 0.82
- **[P2] Recognize the current poll creation variant:** `extensions/whatsapp/src/inbound/extract.ts:148-153`
  - body: Baileys 7.0.0-rc11 exposes `pollCreationMessageV5`, but this extractor checks only V1 through V3. Polls from newer clients can still return `undefined` and be dropped by enrichment.
  - confidence: 0.8
- **[P2] Distinguish unknown RSVP states from maybe:** `extensions/whatsapp/src/inbound/extract.ts:187-188`
  - body: Baileys defines `UNKNOWN = 0` and `MAYBE = 3`, but this mapping treats every non-going/non-not-going value as `maybe`. Empty or unknown updates would be reported as tentative attendance.
  - confidence: 0.9
- **[P2] Use the plugin-sdk hook runtime seam:** `extensions/whatsapp/src/auto-reply/monitor/on-message.ts:5-9`
  - body: This extension production file imports core `src/hooks/*` directly. Current main routes WhatsApp message hooks through `openclaw/plugin-sdk/hook-runtime`, so this stale import path conflicts with the plugin boundary and current hook opt-in shape.
  - confidence: 0.78
