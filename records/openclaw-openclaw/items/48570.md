## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep gated WhatsApp hooks behind opt-in:** `extensions/whatsapp/src/auto-reply/monitor/on-message.ts:155`
  - body: Current main only emits WhatsApp message_received hooks when `channels.whatsapp.pluginHooks.messageReceived` is explicitly enabled, and the tests assert no hook without that opt-in. This new early-return path fires `message:received` for every gated group message anyway, so unmentioned group content can reach hook handlers without the configured privacy boundary.
  - confidence: 0.92
- **[P1] Use the reaction event's reactor key:** `extensions/whatsapp/src/inbound/monitor.ts:517-523`
  - body: Baileys emits `messages.reaction` with `key` pointing at the message being reacted to and `reaction.key` carrying the reaction message key. Building the synthetic inbound message from `key.participant` and `key.fromMe` can attribute the reaction to the original message author or classify the account owner's own reaction as inbound, which breaks sender access control and downstream routing.
  - confidence: 0.91
- **[P1] Preserve poll update vote payloads:** `extensions/whatsapp/src/inbound/monitor.ts:472`
  - body: The handler iterates real `pollUpdate` objects, but the synthetic message always uses `pollUpdateMessage: {}`. Agents and hooks will only see `<media:poll-vote>` with no selected option data, so the poll-response workflow this PR is meant to enable remains unusable for any automation that needs the actual vote.
  - confidence: 0.89
- **[P1] Handle RSVP updates from messages.update:** `extensions/whatsapp/src/inbound/monitor.ts:444`
  - body: The new `messages.update` handler returns unless `pollUpdates` is present, but Baileys also carries event RSVP updates on `WAMessage.eventResponses`. Those RSVP updates never become synthetic inbound messages, so the new `eventResponseMessage` extraction will not run for that update path.
  - confidence: 0.86
- **[P2] Recognize current poll creation variants:** `extensions/whatsapp/src/inbound/extract.ts:146-153`
  - body: Baileys v7.0.0-rc11 includes `pollCreationMessageV5`, but the new extractor only checks `pollCreationMessage`, V2, and V3. Polls sent in the V5 shape will still return `undefined` from `extractMediaPlaceholder()` and be dropped by enrichment.
  - confidence: 0.9
- **[P2] Do not report UNKNOWN RSVP as maybe:** `extensions/whatsapp/src/inbound/extract.ts:187`
  - body: Baileys defines `EventResponseType.UNKNOWN = 0`, but this mapping treats every value other than 1 or 2 as `maybe`. Empty or unknown RSVP updates would be reported as tentative attendees, skewing any downstream RSVP logic.
  - confidence: 0.84
