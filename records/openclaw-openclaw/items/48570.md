## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Keep gated group hooks behind the WhatsApp opt-in:** `extensions/whatsapp/src/auto-reply/monitor/on-message.ts:145-170`
  - body: Current main only emits WhatsApp message_received hooks when `pluginHooks.messageReceived` is enabled, because these payloads can include private content and identifiers. This early-return path fires an internal hook for unmentioned group messages regardless of that opt-in, so existing setups could expose gated group traffic to hook handlers unexpectedly.
  - confidence: 0.88
- **[P1] Preserve the poll update payload:** `extensions/whatsapp/src/inbound/monitor.ts:472`
  - body: `messages.update` gives this code the actual `pollUpdate`, but the synthetic message replaces it with `pollUpdateMessage: {}`. Downstream hooks and agents can only see a generic `<media:poll-vote>` marker, so the original request to read selected poll options remains unsolved.
  - confidence: 0.9
- **[P2] Handle RSVP updates from messages.update:** `extensions/whatsapp/src/inbound/monitor.ts:445`
  - body: The new update handler exits whenever `pollUpdates` is absent, but Baileys also models event RSVP state as update/event response payloads. Those RSVP updates never become synthetic inbound messages, so the advertised RSVP support still misses the update path.
  - confidence: 0.82
- **[P2] Recognize the current poll creation variants:** `extensions/whatsapp/src/inbound/extract.ts:152`
  - body: The pinned Baileys rc13 proto includes `pollCreationMessageV5`, but this extractor only checks V1/V2/V3. A V5 poll creation message will still return no placeholder and be dropped by enrichment.
  - confidence: 0.9
- **[P2] Do not map unknown RSVP state to maybe:** `extensions/whatsapp/src/inbound/extract.ts:190`
  - body: Baileys defines `UNKNOWN = 0` separately from `MAYBE = 3`, but this ternary maps every non-going and non-not-going value to `maybe`. Empty or unknown RSVP updates would be reported as a real maybe response and skew downstream automation.
  - confidence: 0.86
