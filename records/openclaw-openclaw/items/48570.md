## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep gated WhatsApp hooks behind opt-in:** `extensions/whatsapp/src/auto-reply/monitor/on-message.ts:154`
  - body: Current main only emits WhatsApp `message_received` hooks when `pluginHooks.messageReceived` is explicitly enabled, and tests assert no hook fires without that opt-in. This new early-return path fires `message:received` for unmentioned gated group messages regardless of that setting, exposing private group content to hook handlers that did not opt into WhatsApp message payloads.
  - confidence: 0.93
- **[P1] Preserve poll vote payloads:** `extensions/whatsapp/src/inbound/monitor.ts:472`
  - body: `messages.update` gives each voter's `pollUpdate`, including the selected options, but the synthetic message replaces it with `pollUpdateMessage: {}`. Downstream hooks/agents only see a generic placeholder, so the poll-response workflows this PR closes still cannot know what was voted for.
  - confidence: 0.9
- **[P1] Handle event responses from update events:** `extensions/whatsapp/src/inbound/monitor.ts:444-445`
  - body: Baileys models event RSVPs as `eventResponses` updates on the original message, but this handler returns unless `pollUpdates` is present. RSVP updates delivered on that supported path never become synthetic inbound messages, so the new RSVP extraction will not run for those events.
  - confidence: 0.87
- **[P2] Map unknown RSVP states explicitly:** `extensions/whatsapp/src/inbound/extract.ts:190`
  - body: Baileys defines `UNKNOWN = 0` separately from `MAYBE = 3`, but this mapping treats every non-going/non-not-going value as `maybe`. Empty or unknown RSVP updates would be reported as tentative attendance, which can skew automation that counts real RSVP states.
  - confidence: 0.89
- **[P2] Recognize current poll creation variants:** `extensions/whatsapp/src/inbound/extract.ts:152`
  - body: The pinned Baileys schema includes `pollCreationMessageV5`, while the new extraction only checks V1/V2/V3. Newer WhatsApp poll creation messages can still return no placeholder and be dropped by `enrichInboundMessage()`.
  - confidence: 0.86
