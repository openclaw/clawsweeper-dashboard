## What This Changes

The issue reports that Discord cron announce and final reply delivery can lose the external Discord plugin outbound adapter after plugin registry reloads, even while the Discord provider remains connected.

## Best Possible Solution

Patch the generic outbound resolver so adapter and target resolution consistently prefer the pinned or loaded channel plugin registry across registry churn, with regression coverage for an external Discord-style plugin and no Discord-specific hard-code.

## Reproduction Assessment
