## Repair Work Prompt

Fix openclaw/openclaw#77254. The observable bug is that, with the external Discord channel plugin loaded and connected, generic outbound delivery can lose the Discord outbound adapter after plugin registry reload/churn, causing cron announce and final replies to fail with `Unsupported channel: discord` or `Outbound not configured for channel: discord` while direct `openclaw message send --channel discord` still succeeds. Keep the fix generic: do not hard-code Discord or move plugin-owned behavior into core. Harden the outbound channel resolver/loader so loaded or pinned channel plugins remain authoritative across active registry replacement, including an explicit loaded-plugin fallback where adapter lookup can miss. Add focused regression coverage that simulates an external channel plugin with outbound adapter, active registry replacement/churn, target resolution, and delivery adapter lookup. Do not try to solve #77241 plugin contract loading or #71127 stuck-session recovery in this PR, but avoid making those paths worse. Add a user-facing CHANGELOG.md fix entry unless maintainers classify the final patch as test-only/internal.
