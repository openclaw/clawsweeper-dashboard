## What This Changes

The issue asks OpenClaw to keep Discord outbound adapter resolution stable across plugin registry reloads so cron announces and final replies continue delivering to Discord.

## Best Possible Solution

Patch the generic outbound target and adapter resolution paths so loaded or pinned channel plugins remain authoritative across active registry churn, with regression coverage for external Discord delivery and no Discord-specific hard-code.

## Reproduction Assessment

No, not as a controlled live reproduction. The source path is clear enough for triage: missing plugin/adapter resolution in current main reaches the same `Unsupported channel` and `Outbound not configured` errors reported for Discord.

## Solution Assessment

Yes, the narrow maintainable fix is generic resolver/loader consistency around the pinned or loaded channel registry plus focused regression tests. A Discord-only fallback would be the wrong ownership boundary.
