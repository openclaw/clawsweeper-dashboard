## Repair Work Prompt

Fix the remaining Discord delivery-context loss described in this issue without undoing the current pinned channel-registry work. Current main already routes channel loaders through the pinned channel registry; preserve that and do not add Discord-specific core IDs or policy. Add regression coverage for a Discord-like same-channel session whose stored deliveryContext has a resolved recipient, but a recovered/bestEffort agent request carries `channel`/turnSourceChannel without `to`; delivery should reuse the same-channel stored recipient instead of logging `agent delivery target missing (bestEffortDeliver)`. Keep the cross-channel safety from #24152: do not reuse `lastTo` when the turn source channel differs from the stored delivery channel. Likely implementation belongs in `src/infra/outbound/agent-delivery.ts` and/or `src/infra/outbound/targets-session.ts`, with `src/gateway/server-methods/agent.ts` touched only if necessary. Add or update targeted tests and a user-facing changelog fix entry unless maintainers classify the patch as test/internal only.
