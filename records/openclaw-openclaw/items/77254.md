## Summary

Keep open: the report is fresh, corroborated by two deployments, and the current generic outbound paths still have the exact `Unsupported channel` / `Outbound not configured` failure exits when channel plugin resolution misses. Related closed registry and secret-contract issues do not cover this full Discord outbound adapter-loss report.

## What This Changes

The issue requests that generic Discord outbound delivery keep resolving the external Discord channel plugin adapter across plugin registry reloads, upgrades, and recovery paths.

## Best Possible Solution

Make the loaded or pinned channel registry authoritative for generic outbound target and adapter resolution across registry churn, with regression coverage for external Discord delivery after active registry replacement and no Discord-specific core branch.

## Reproduction Assessment

No controlled live current-main reproduction was run. Source inspection gives a clear failure path matching the report: a channel plugin lookup miss reaches `Unsupported channel`, and an outbound adapter miss reaches `Outbound not configured`.

## Solution Assessment

Yes, the narrow maintainable fix is generic channel plugin resolution that consistently uses the loaded/pinned channel surface and covers both target resolution and adapter loading. A Discord-only fallback would cross the owner boundary and leave other external channel plugins exposed.
