## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77254. The observable bug is that Discord generic outbound delivery can lose its outbound adapter after plugin registry reload/churn, causing cron announce delivery and final replies to fail with `Unsupported channel: discord` or `Outbound not configured for channel: discord` while the Discord provider remains connected and direct `openclaw message send --channel discord` still works. Keep the fix generic and core extension-agnostic: do not hard-code `discord`, do not add startup/load-time config migrations, and do not bypass the plugin SDK boundary. Inspect `src/infra/outbound/channel-resolution.ts`, `src/channels/plugins/registry-loader.ts`, `src/channels/plugins/outbound/load.ts`, and `src/plugins/runtime*.ts`; make loaded or pinned channel plugin resolution authoritative across active registry replacement for both target resolution and adapter loading. Add regression coverage for an external/loaded channel plugin with an outbound adapter after active registry replacement, ideally covering both `resolveOutboundTarget` and `loadChannelOutboundAdapter`/delivery failure prevention. Include a user-facing `CHANGELOG.md` fix entry. Preserve existing tests and behavior for empty pinned registries and active-registry fallback.
