## Summary

Keep open: the report is specific, has corroborating production evidence, and current main still has generic outbound paths that can emit the reported `Unsupported channel` and `Outbound not configured` errors when channel plugin resolution misses. Existing pinned-registry support is relevant but not enough evidence to close because the reported 2026.5.3 failure occurred on the same resolver/loader shape and there is no current-main live or regression proof for Discord delivery after registry churn.

## What This Changes

The issue reports Discord cron announcements and final replies losing outbound adapter resolution after plugin registry reloads despite the external Discord plugin remaining connected.

## Best Possible Solution

Patch the generic outbound channel target and adapter resolution path so loaded or pinned channel plugins remain authoritative across active plugin-registry churn, with regression coverage for external-channel target resolution and delivery after registry replacement.

## Solution Assessment
