## Summary

Keep open: the report is specific, matches current generic outbound failure paths, and the related #54599 cleanup is closed but does not cover this Discord 2026.5.3 delivery-loss evidence.

## What This Changes

The issue asks OpenClaw to keep Discord outbound adapter resolution stable across plugin registry reloads so cron announce delivery and final replies continue to send.

## Best Possible Solution

Patch the generic channel outbound resolution path so loaded or pinned channel plugins remain authoritative across active registry churn, with no Discord-specific hard-code and regression coverage for external channel delivery after registry replacement.

## Reproduction Assessment

No controlled live current-main reproduction was run. The source path is clear enough for triage: when Discord plugin lookup returns undefined, target resolution emits `Unsupported channel` and delivery throws `Outbound not configured`, matching the report.

## Solution Assessment

Yes, the narrowest maintainable direction is a generic resolver/loader fix that consistently uses the pinned or loaded channel registry, plus regression tests for active registry churn; a Discord-only workaround would be the wrong boundary.
