## What This Changes

The PR adds a guard in `resolveCliSessionReuse` and a regression test so stored strict MCP resume-hash bindings are invalidated when the current run has no resume hash.

## Best Possible Solution

Land the narrow guard after maintainer acceptance of the fail-closed session-state behavior and after the unrelated typecheck failure is cleared.

## Reproduction Assessment

Yes. Current main can be source-reproduced by passing a binding with stored `mcpResumeHash`, matching raw `mcpConfigHash`, and no current `mcpResumeHash`; it falls through to reuse, while this PR invalidates it.

## Solution Assessment

Yes for the code shape, with maintainer acceptance required for the behavior change. The shared reuse helper is the narrow decision point used by `prepareCliRunContext`, and the patch preserves legacy config-hash fallback only for bindings without a stored strict resume hash.
