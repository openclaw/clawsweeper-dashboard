## Summary

Keep open: the patch is a narrow, correct guard for a real one-sided CLI MCP resume-hash reuse gap that still exists on current main, but it intentionally changes session reuse behavior and should get maintainer acceptance plus clean required checks before merge.

## What This Changes

The PR makes `resolveCliSessionReuse` invalidate a stored CLI binding when it has `mcpResumeHash` but the current run cannot provide one, and adds focused regression coverage.

## Best Possible Solution

Land the narrow fail-closed guard after maintainers accept the session-continuity tradeoff and required checks pass, while preserving legacy config-hash fallback only for bindings that never stored `mcpResumeHash`.

## Reproduction Assessment

Yes. Current main can be source-reproduced by passing a binding with stored `mcpResumeHash`, matching raw `mcpConfigHash`, and no current `mcpResumeHash`; it falls through to reuse, while the PR proof shows the patched helper invalidates with `mcp`.

## Solution Assessment

Yes. The shared reuse helper is the narrowest maintainable layer, and the PR restores fail-closed behavior only for bindings that already stored a strict resume hash while keeping the legacy fallback for older bindings.
