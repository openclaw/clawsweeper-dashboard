## What This Changes

The PR adds a guard in CLI session reuse so stored strict MCP resume-hash bindings are invalidated when the current run cannot provide the corresponding resume hash, with a regression test for that case.

## Best Possible Solution

Land the narrow guard after maintainers accept the strict resume-hash fail-closed behavior and refresh exact-head checks; keep the broader stale claude-cli lifecycle policy tracked in https://github.com/openclaw/openclaw/issues/84593.

## Reproduction Assessment

Yes, at source level. On current main, a binding with stored mcpResumeHash, matching raw mcpConfigHash, and no current mcpResumeHash falls through to reuse, while the PR makes that case return invalidatedReason mcp.

## Solution Assessment

Yes for the narrow bug, with maintainer acceptance still needed for the behavior change. The shared reuse helper is the right decision point, and the patch preserves legacy mcpConfigHash fallback only for bindings that never stored a strict resume hash.
