## Summary

Keep open for normal PR review/landing: current main still has the one-sided resume-hash fallback, while this patch is a narrow and correct guard with sufficient helper-level real behavior proof.

## What This Changes

The PR invalidates CLI session reuse when a stored binding has `mcpResumeHash` but the current run cannot provide one, and adds focused regression coverage.

## Best Possible Solution

Land the narrow fail-closed guard once maintainers accept the bounded session-continuity tradeoff and normal checks are green, while keeping the legacy config-hash fallback only for bindings that never stored a resume hash.

## Reproduction Assessment

Yes. The helper can be invoked with a stored `mcpResumeHash`, matching `mcpConfigHash`, and no current `mcpResumeHash`; current main falls through to config-hash reuse, and the PR proof shows the patched helper invalidates with `mcp`.

## Solution Assessment

Yes. The guard belongs in `resolveCliSessionReuse`, the shared reuse decision point, and it preserves the intended legacy fallback for bindings that never stored a resume hash.
