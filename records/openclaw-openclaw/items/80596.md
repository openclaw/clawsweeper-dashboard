## Summary

Keep open. The latest head addresses the earlier API-safety review by exposing only a path-only transcript reload notification, and the contributor has now supplied usable terminal proof; the remaining blocker is maintainer approval for adding this public plugin runtime API.

## What This Changes

The PR adds `runtime.events.emitSessionTranscriptUpdate(sessionFile)` to the plugin runtime, updates the runtime type and mock, adds a focused runtime test, and documents the durable-write expectation.

## Best Possible Solution

Merge the path-only emitter only if maintainers accept transcript reload notifications as a public plugin runtime API, while keeping durable transcript writes in the separate authority-design thread.

## Solution Assessment
