## What This Changes

## Best Possible Solution

Land the path-only runtime event only if maintainers accept this as a supported plugin API; keep the durable-write contract documented and covered by the focused regression test.

## Reproduction Assessment

## Solution Assessment

Yes for the technical shape if the API is accepted: the path-only wrapper is narrower than exposing the internal message-capable emitter and matches the durable-write documentation. The remaining question is whether this should become supported public runtime surface.
