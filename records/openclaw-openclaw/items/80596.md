## Summary

Keep open. The change is a plausible plugin runtime API request, but this PR currently fails type/lint checks because the plugin runtime mock is not updated, exposes a raw global transcript emitter that needs API/security review, and lacks required real behavior proof from the external contributor.

## What This Changes

The PR adds `emitSessionTranscriptUpdate` to `PluginRuntime.events`, wires it through `createRuntimeEvents`, and adds a runtime surface test.

## Best Possible Solution

Expose a scoped plugin-safe transcript write or notification helper, keep runtime mocks/docs/tests in sync, and require real plugin-runtime proof before merge.

## Reproduction Assessment

Yes for the PR defects: CI logs show TS2741 from the missing `emitSessionTranscriptUpdate` mock member, and the raw-emitter broadcast path is source-reproducible from gateway transcript subscribers. I did not run local tests to keep the checkout read-only.

## Solution Assessment

No. Adding the raw process-global emitter directly to `PluginRuntime.events` is not the safest API shape because gateway listeners treat emitted messages as session updates; a scoped helper that owns transcript writes or session validation would be narrower.
