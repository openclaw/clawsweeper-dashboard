## Summary

## What This Changes

The PR adds `api.runtime.events.emitSessionTranscriptUpdate(sessionFile)`, documents the durable-write expectation, updates the plugin runtime mock, and adds a focused runtime test.

## Best Possible Solution

Keep the path-only plugin API only if the runtime wrapper enforces string input, ignores or rejects non-string payloads, adds a negative regression test, and refreshes against current main before merge.

## Solution Assessment

No: exposing a path-only reload notifier is a plausible narrow API, but the current implementation only narrows the TypeScript type and leaves the public runtime boundary unenforced. The safer implementation is a small wrapper that validates string input and tests non-string payloads.
