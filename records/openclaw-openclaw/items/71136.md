## Summary

Keep open. Current main still has source-reproducible override-first context-token paths and inconsistent maxTokens clamping, while the related open PR only covers part of the context-window problem and does not settle catalog merge precedence.

## What This Changes

The issue requests resolving effective contextTokens and maxTokens as the smallest known positive limit from agent config, provider model config, discovery data, and defaults.

## Best Possible Solution

Centralize positive finite token-limit resolution with source metadata, apply it consistently to effective runtime context and output caps, and document or deliberately preserve catalog merge precedence.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by configuring an agent/session contextTokens or request maxTokens above a lower known model limit; current main still returns, persists, displays, or forwards the larger value in several paths.

## Solution Assessment

Partly. Taking the minimum of known positive runtime limits is the right safety direction, but the exact treatment of configured versus discovered catalog values and maxTokens across providers needs maintainer coordination.
