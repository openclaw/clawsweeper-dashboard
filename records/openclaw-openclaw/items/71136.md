## Summary

Keep open. Current main still has source-reproducible override-first paths for `contextTokens` and OpenAI `maxTokens`, and the related open PR only covers part of the context-window problem rather than the full token-limit contract.

## What This Changes

The issue requests resolving effective `contextTokens` and `maxTokens` as the minimum of agent config, provider/model config, discovery metadata, and defaults.

## Best Possible Solution

Centralize finite positive token-limit resolution with source metadata, apply it consistently to runtime budgeting, persisted/session status, memory flush, and transport output caps, then update docs/tests for any deliberate catalog precedence change.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: configure agent/session `contextTokens` or runtime OpenAI `maxTokens` above a lower known model limit, and current main can still return, persist, report, or forward the larger value in several paths.

## Solution Assessment

Partly. Taking the minimum of known positive runtime caps is the right safety direction, but catalog merge precedence and transport `maxTokens` semantics need an explicit maintainer decision before this should be treated as a mechanical one-path fix.
