## Summary

Keep open. Current main still resolves several token limits by precedence rather than by the most conservative available limit, so the reported context and output-token overflow class remains valid.

## What This Changes

Use the effective minimum of agent config, provider model config, model discovery, and defaults for `contextTokens` and `maxTokens`.

## Best Possible Solution

Centralize positive finite token-limit resolution with source metadata, then apply it consistently to context guards, session persistence, model/config discovery merging, and provider request `maxTokens` clamps.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence source reproduction: configure an agent or request token budget above a lower configured/discovered/model limit, and current main still returns the higher override first in central paths.

## Solution Assessment

Yes. The minimum-of-available-limits direction is the narrow maintainable fix, provided it is centralized and covered across context-window and `maxTokens` paths instead of patched in one caller.
