## Summary

Keep open: current main and v2026.6.10 already cap oversized agent context overrides and proxy-like OpenAI Completions output budgets, but explicit provider `contextTokens` and model-merge `maxTokens` still do not resolve as the minimum against discovery.

## What This Changes

The issue requests resolving effective context and output token budgets as the minimum of agent config, provider model config, and discovered model metadata.

## Best Possible Solution

Decide and document one provider/discovery precedence contract, then update the resolver, model-merge helper, docs, and focused tests only if maintainers choose minimum semantics.

## Reproduction Assessment

Yes, source-reproducible for the remaining semantics: current tests assert explicit configured `contextTokens` can beat lower discovery, while separate tests prove the oversized agent-override bug is already capped.

## Solution Assessment

Unclear. Minimum semantics are safer for overflow, but they would change the existing tested contract that explicit provider config can override discovery when metadata is wrong or stale.
