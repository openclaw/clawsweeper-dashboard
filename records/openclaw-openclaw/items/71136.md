## Summary

Keep open. Current main still has source-reproducible override-first `contextTokens` and `maxTokens` paths, while the related open PRs/issues cover only parts of the context-window problem and do not settle the documented catalog precedence contract.

## What This Changes

Request to resolve effective `contextTokens` and `maxTokens` as the minimum known positive limit from agent config, provider model config, model discovery, and defaults.

## Best Possible Solution

Centralize finite positive token-limit resolution with source metadata, apply it consistently to runtime context budgeting, session/status reporting, and transport output caps, then update docs/tests if maintainers decide to change catalog merge precedence.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: configure an agent/session `contextTokens` or request `maxTokens` above a lower known model limit, and current main can still return, persist, report, or forward the larger value in several paths.

## Solution Assessment

Partly. Taking the minimum of known positive effective runtime caps is the right safety direction, but the catalog merge and `maxTokens` precedence pieces need a maintainer contract decision before a full fix should land.
