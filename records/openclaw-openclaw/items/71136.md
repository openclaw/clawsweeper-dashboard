## Summary

Keep open: current main and v2026.6.8 fix the oversized agent context override path, but the broader request to make explicit provider `contextTokens` and `maxTokens` always resolve as the minimum against discovery is still a config-contract decision rather than already implemented behavior.

## What This Changes

The issue requests resolving `contextTokens` and `maxTokens` as the minimum across agent config, provider model config, model discovery, and defaults.

## Best Possible Solution

Keep the shipped agent-override cap, then make one maintainer-approved provider token-budget contract and align resolver code, model merge behavior, docs, and regression tests around it.

## Reproduction Assessment

Yes, source-level: current code and tests show the agent override bug is fixed while configured provider `contextTokens` and `maxTokens` do not follow min-of-all semantics. No live API repro was run because this was a read-only cleanup review.

## Solution Assessment

Unclear for the remaining work. The current shipped fix is the right solution for oversized agent overrides, but applying the same rule to explicit provider token caps changes documented semantics and should be decided before implementation.
