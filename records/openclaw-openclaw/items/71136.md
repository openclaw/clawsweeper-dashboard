## Summary

Keep open. Current main still has source-reproducible override-first token paths, and the open related PRs only cover parts of the context-window/per-agent problem while leaving the broader `maxTokens` and persistence/status surfaces unresolved.

## What This Changes

The issue asks OpenClaw to compute effective `contextTokens` and `maxTokens` as the minimum of agent config, provider model config, discovered model limits, and defaults.

## Best Possible Solution

Centralize positive finite token-limit resolution with source metadata, then reuse it for context windows, persisted session metadata, status output, model/discovery merges, and provider request `maxTokens` clamps.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by configuring an agent or session `contextTokens` above a lower known provider/model window; current main still returns or persists the higher override first in shared resolver, session-store, and status-summary paths.

## Solution Assessment

Yes on direction. Taking the minimum of known positive limits is the narrow maintainable fix, but it should be coordinated with https://github.com/openclaw/openclaw/pull/39859 and the broader `maxTokens` paths instead of landing a status-only patch.
