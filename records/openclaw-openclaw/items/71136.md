## Summary

Keep open: current main and v2026.6.8 already solve the oversized agent-context override bug, but the broader request to min explicit provider `contextTokens` and `maxTokens` against discovery remains a config-contract/product decision.

## What This Changes

The issue asks OpenClaw to resolve effective context and output-token limits by taking the minimum of agent config, provider model config, model discovery, and defaults.

## Best Possible Solution

Keep the shipped agent-override cap, then have maintainers decide the explicit provider `contextTokens` and `maxTokens` contract and update resolver code, runtime model merge, docs, and regression tests together.

## Reproduction Assessment

Yes for the remaining source-level behavior: current source and tests show explicit provider `contextTokens` can exceed lower discovery metadata, and `maxTokens` resolution does not always min configured caps with discovered output caps. I did not run tests because this review is read-only and must keep the checkout clean.
