## Summary

Keep open. Current main fixes the `contextTokens` over-reporting path and the proxy-like OpenAI remaining-context `maxTokens` failure, but the broader `maxTokens` precedence requested here is still not consistently implemented across agent params and provider transports.

## What This Changes

Use conservative minimum/ceiling behavior when resolving effective `contextTokens` and `maxTokens` from agent config, provider model config, discovery metadata, and defaults.

## Best Possible Solution

Keep the landed context and proxy-like OpenAI fixes, then define one shared `maxTokens` budget contract and apply it at the request-shaping seams with focused transport tests.

## Reproduction Assessment

Yes. The context-token example is fixed on current main, but the remaining `maxTokens` concern is source-reproducible because configured agent/model params become `streamParams.maxTokens` and Anthropic returns a requested value before applying the model cap.

## Solution Assessment

Unclear as a complete fix. The minimum/ceiling behavior is right for known context windows, but broad `maxTokens` precedence needs a maintainer compatibility decision before applying it across transports.
