## Summary

Keep open. Current main still has source-reproducible override-first context-token paths, current merge/runtime code still preserves higher explicit token limits in several places, and the related open PR only covers part of the context-window problem while leaving review blockers.

## What This Changes

Use the most conservative known context and output token limits instead of priority-order overrides when resolving effective `contextTokens` and `maxTokens`.

## Best Possible Solution

Centralize positive finite token-limit resolution with source metadata, then reuse it for context windows, persisted session metadata, status output, model/discovery merges, and provider request `maxTokens` clamps.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by configuring an agent or session `contextTokens` above a lower known provider/model context window; current main still returns or persists the higher override first in the shared resolver, session store, and status summary paths.

## Solution Assessment

Yes. The minimum-of-known-positive-limits direction is the narrow maintainable fix, but it should be coordinated with or replace https://github.com/openclaw/openclaw/pull/39859 so the broader `maxTokens` and persistence/status surfaces are covered together.
