## Summary

## What This Changes

The issue requests changing effective `contextTokens` and `maxTokens` resolution to use the minimum across agent config, provider model config, model discovery, and defaults.

## Best Possible Solution

Keep the shipped agent-override cap, then define one maintainer-approved provider token-budget contract and align resolver code, runtime model merge behavior, docs, and regression tests around it.

## Reproduction Assessment

Yes, source-level. Current code and tests show the oversized agent override path is fixed while configured provider `contextTokens` and `maxTokens` still do not follow the requested min-of-all semantics.

## Solution Assessment

Unclear for the remaining work. The current shipped fix is the right shape for oversized agent overrides, but applying the same rule to explicit provider token caps changes documented semantics and should be decided before implementation.
