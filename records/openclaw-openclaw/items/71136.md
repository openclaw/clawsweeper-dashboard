## Summary

Keep open. Current main and the latest release still have source-reproducible override-first `contextTokens` paths, while the related open PR only covers part of the context-window behavior and does not settle the `maxTokens` or catalog-precedence pieces.

## What This Changes

Use the minimum finite positive value from agent configuration, provider/model configuration, model discovery, and defaults for effective `contextTokens` and `maxTokens`.

## Best Possible Solution

Centralize finite positive token-limit resolution with source metadata, apply the minimum consistently to runtime context budgeting, session/status reporting, and provider output caps, and update docs/tests for the chosen precedence contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure agent/default `contextTokens` above a known selected-model window or request `maxTokens` above `model.maxTokens`, and current main can still choose the larger value in several paths.

## Solution Assessment

Partly. Minimum-of-known-positive runtime caps is the right safety direction, but the exact provider catalog precedence and `maxTokens` contract need maintainer review before this should be treated as a mechanical bug fix.
