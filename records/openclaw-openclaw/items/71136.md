## Summary

Keep open: current main and v2026.6.8 already fix the oversized agent-context override part, but the broader min-with-discovery request is not fully implemented for explicit provider `contextTokens`/`maxTokens` semantics and needs maintainer direction rather than cleanup closure.

## What This Changes

The issue requests using the minimum known token limit across agent config, provider model config, model discovery, and defaults for `contextTokens` and `maxTokens`.

## Best Possible Solution

Keep the shipped agent-override cap, then have maintainers decide whether explicit provider `contextTokens` and `maxTokens` should remain operator-owned overrides or be redefined as ceilings capped by discovery, with docs and regression tests updated to match.

## Reproduction Assessment

Yes for the remaining source-level behavior: current tests show explicit provider `contextTokens` can exceed lower discovery metadata by design, and model resolution does not always min configured `maxTokens` with discovered output caps. I did not run tests because this was a read-only review and the checkout must stay clean.
