## Summary

Keep open. Current main still has precedence-based token resolution in the central context paths and core Anthropic maxTokens handling, so the reported overflow class remains valid and actionable.

## What This Changes

The issue requests resolving contextTokens and maxTokens as the minimum of available agent config, provider model config, discovery/catalog metadata, and defaults instead of letting a higher-priority configured value win.

## Best Possible Solution

Add a shared positive-finite token-limit resolver with source metadata and use it consistently for context-window guards, session persistence, model normalization, config/discovery merging, and provider request maxTokens clamps.

## Reproduction Assessment

Yes. Configure an agent or request with contextTokens/maxTokens above a lower configured, discovered, or runtime model limit; static inspection shows current main still returns the higher override/configured value first in central paths.

## Solution Assessment

Yes. The minimum-of-available-limits direction is the narrow maintainable fix, provided it is centralized and tested across the model resolver, guard, session persistence, and provider request paths.
