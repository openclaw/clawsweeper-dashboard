## Summary

Keep open: current main and v2026.6.10 fix the oversized agent context override path, but the broader request to make explicit provider `contextTokens` and `maxTokens` resolve as the minimum against discovery remains a compatibility-sensitive config contract decision.

## What This Changes

The issue asks OpenClaw to resolve effective `contextTokens` and `maxTokens` from the minimum of agent config, provider model config, discovery metadata, and defaults.

## Best Possible Solution

Keep the shipped agent-override cap, then make one maintainer-approved provider token-budget contract and align resolver code, model merge behavior, docs, and regression tests around it.

## Reproduction Assessment

Yes, source-level. Current code and tests show the agent override bug is fixed while configured provider `contextTokens` and `maxTokens` do not follow min-of-all semantics.

## Solution Assessment

Unclear for the remaining work. The current shipped fix is the right solution for oversized agent overrides, but applying the same rule to explicit provider token caps changes documented semantics and should be decided before implementation.
