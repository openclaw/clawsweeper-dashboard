## Summary

Keep open. Current main still has several source-reproducible override-first token paths, while related open PRs cover only parts of the context-window/status problem and do not settle the broader `maxTokens` and catalog-merge precedence question.

## What This Changes

Request that effective `contextTokens` and `maxTokens` use the minimum known positive limit across agent config, provider model config, model discovery, and defaults.

## Best Possible Solution

Centralize positive finite token-limit resolution with source metadata, use it for effective runtime context and output caps across resolver/session/status/transport paths, and update tests/docs if catalog merge precedence changes.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by configuring an agent/session `contextTokens` or request `maxTokens` above a lower known model limit; current main still returns, persists, or forwards the larger override in several paths.

## Solution Assessment

Partly. Taking the minimum of known positive effective runtime caps is the right safety direction, but changing catalog merge behavior for discovered/configured `contextWindow` and `maxTokens` needs maintainer contract judgment and coordinated implementation.
