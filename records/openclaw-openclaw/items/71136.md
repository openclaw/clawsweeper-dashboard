## Summary

Keep open: current main and v2026.6.8 cover the contextTokens half through the shared context resolver, but the issue also requests the same minimum/cap behavior for maxTokens and that remains a separate source-reproducible provider/transport contract question.

## What This Changes

Request to resolve contextTokens and maxTokens as the minimum of agent-level settings, configured model metadata, discovery metadata, and defaults so overrides cannot exceed known model limits.

## Best Possible Solution

Keep the shipped contextTokens cap and define one provider-aware maxTokens contract that preserves intentional smaller user caps while preventing oversize request caps from exceeding known model/provider output limits.

## Reproduction Assessment

Yes. The contextTokens half is source-verified as fixed, and the remaining maxTokens half is source-reproducible because agent/model params can become runtime maxTokens while Anthropic Messages accepts a positive requested value before model/default capping.

## Solution Assessment

Unclear. The contextTokens fix is the right existing solution for context windows, but maxTokens should be solved through a separate provider-transport/catalog contract rather than blindly copying the context-token resolver shape.
