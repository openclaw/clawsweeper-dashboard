## Summary

Keep open: current main still lets verbose mode bypass Telegram/default progress suppression, and current tests deliberately expect suppressed tool/status text to be delivered in that state.

## What This Changes

The issue asks OpenClaw to stop sending internal Telegram tool/status progress messages when Telegram streaming or tool progress is disabled.

## Best Possible Solution

Honor explicit Telegram final-only or tool-progress-disabled suppression across default text progress delivery while preserving media payloads, exec approvals, errors, and configured visible preview progress.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main combines Telegram direct context, verbose mode, and `suppressDefaultToolProgressMessages: true`, then expects default tool/status text to be delivered.

## Solution Assessment

Yes. The narrow maintainable fix is to make explicit Telegram off/tool-progress-disabled suppression authoritative for text-only default progress while keeping the existing media, exec-approval, and error exemptions.
