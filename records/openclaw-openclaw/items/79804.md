## Summary

Keep open: current main still contains the Telegram/default progress leak path, the tests intentionally assert it under verbose mode, and the related fixing PRs were closed unmerged rather than shipped.

## What This Changes

The issue requests that Telegram direct chats hide internal tool/status progress cards when streaming is off or tool-progress output is disabled.

## Best Possible Solution

Honor explicit Telegram final-only or tool-progress-disabled configuration across default progress paths while preserving media payloads, exec approvals, errors, and configured visible preview progress.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main combines Telegram direct context, verbose mode, and `suppressDefaultToolProgressMessages: true`, then expects default tool/status text to be delivered.

## Solution Assessment

Yes. The narrow maintainable direction is to make explicit Telegram off/tool-progress-disabled suppression authoritative for text-only default progress while preserving existing exceptions and avoiding broad Telegram progress-policy changes.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79804 on current main. Current shared dispatch only suppresses default tool/status progress when `suppressDefaultToolProgressMessages` is true and verbose progress is not active, while Telegram sets the suppression flag for `streaming.mode: "off"` or active preview ownership. Implement a narrow fix so explicit Telegram final-only or `streaming.preview.toolProgress=false` / `streaming.progress.toolProgress=false` states suppress text-only default tool summaries, plan updates, working statuses, command output, patch summaries, and approval-status chatter even when session verbose is on. Preserve media-only payload delivery, exec approval payloads, final errors, and configured visible preview/progress behavior when tool progress is enabled. Do not adopt the opposite off-mode behavior from https://github.com/openclaw/openclaw/pull/77211 and do not disable all Telegram progress previews like https://github.com/openclaw/openclaw/pull/78020. Add focused dispatcher and Telegram wiring regression tests; update docs or changelog only if the final code changes user-visible behavior wording. Use live Telegram proof with redacted private data before handoff.
