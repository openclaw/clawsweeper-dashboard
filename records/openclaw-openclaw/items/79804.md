## Summary

Keep open: current main still allows session verbose progress to bypass Telegram default-progress suppression, which conflicts with the documented final-only behavior for Telegram streaming off and explicit tool-progress suppression.

## What This Changes

Telegram should stop sending internal tool/status progress cards when Telegram streaming is off or toolProgress is disabled.

## Best Possible Solution

Define whether Telegram channel-level off/toolProgress suppression outranks session verbose, then update shared dispatch, Telegram reply-option wiring, docs, and regression tests while preserving approvals, media, and error delivery.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: with a Telegram direct context, suppressDefaultToolProgressMessages true, and session verboseLevel set to on, current tests expect a text-only tool result to be delivered despite suppression.

## Solution Assessment

Unclear until maintainers decide precedence. A force-suppression style fix is plausible if Telegram channel config should outrank /verbose, but the final change must update the current tests and preserve approvals, media, and errors.
