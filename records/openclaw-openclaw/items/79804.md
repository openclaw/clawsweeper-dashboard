## Summary

Keep open: current main still has the verbose-mode bypass that can send default tool/status progress despite Telegram suppression, and an open fix PR already references this issue with closing syntax. This should close only after that PR or an equivalent fix lands.

## What This Changes

The issue reports that Telegram direct chats can receive internal tool/status progress cards even when Telegram streaming and tool-progress output are configured off.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80382 or an equivalent narrow dispatch/test fix so channel suppression is honored even with verbose mode on while preserving media, exec-approval, and error exemptions.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: combine a Telegram direct context, verboseLevel "on", and suppressDefaultToolProgressMessages true; current tests expect internal tool/status output to be delivered despite suppression.

## Solution Assessment

Yes for the current fix direction. Making the suppression predicate honor the channel flag unconditionally matches the option contract and Telegram docs, but the issue should remain open until the linked PR lands or maintainers choose a different fix.
