## Summary

Keep open: current main still has a source-level path where session verbose progress bypasses Telegram default progress suppression, while the Telegram docs and option contract say final-only or disabled tool progress should suppress generic tool/status chatter. The remaining decision is policy precedence, not cleanup closure.

## What This Changes

The issue asks OpenClaw to suppress internal Telegram tool/status cards when streaming is off or tool-progress flags are disabled, even when verbose progress is active.

## Best Possible Solution

Settle the precedence between session `/verbose` and channel-level Telegram suppression, then align shared dispatch, Telegram wiring, docs, and regression tests so text-only internal progress is hidden when that policy says it should be.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main sets Telegram/default progress suppression, then current dispatcher tests show `verboseLevel: "on"` causes tool, plan, and working-status progress to be delivered anyway.

## Solution Assessment

Unclear until maintainers decide precedence. Returning the suppression flag directly matches the docs and option contract, but current tests and an open related PR intentionally point in the opposite direction.
