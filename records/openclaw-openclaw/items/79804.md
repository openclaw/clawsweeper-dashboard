## Summary

Keep open: current main still has a source-level path where Telegram passes a default progress-suppression flag, but shared dispatch honors it only when session verbose progress is off. The docs promise final-only Telegram delivery for `streaming.mode: "off"`, while current regressions intentionally assert that `/verbose` overrides suppression, so this needs maintainer precedence judgment rather than cleanup closure.

## What This Changes

Ensure Telegram direct chats do not receive internal tool/status cards when streaming is off or Telegram tool-progress flags are false.

## Best Possible Solution

Define whether channel-level `streaming.mode: "off"` or `toolProgress: false` is a hard suppression override, then align shared dispatch, Telegram reply-option wiring, docs, and regressions with that policy while preserving media, approvals, and error delivery.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: use a Telegram direct context with `suppressDefaultToolProgressMessages: true` and session `verboseLevel: "on"`; current tests expect text-only tool progress to be sent despite the suppression flag.

## Solution Assessment

Unclear until maintainers settle precedence. A hard-suppression flag like the reporter’s mitigation is plausible, but it intentionally changes the merged `/verbose` override behavior covered by current tests.
