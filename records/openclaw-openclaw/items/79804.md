## Summary

Keep open: current main still has a source-level path where Telegram default progress suppression is bypassed by session verbose progress, and current tests intentionally codify that behavior despite the Telegram docs describing `streaming.mode: "off"` as final-only with generic progress chatter suppressed.

## What This Changes

The issue reports Telegram direct chats receiving internal tool/status progress cards even with Telegram streaming off and both preview/progress toolProgress disabled.

## Best Possible Solution

Define the precedence between Telegram channel-level progress suppression and session `/verbose`, then update shared dispatch, Telegram wiring, and regressions so final-only or explicit `toolProgress: false` hides text-only internal status cards while still delivering approvals, media, and errors.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: use a Telegram direct context with `suppressDefaultToolProgressMessages: true` and session `verboseLevel: "on"`; current tests expect text-only tool summaries to be delivered despite suppression.

## Solution Assessment

Unclear until maintainers decide precedence. A force-suppression path is plausible, but it would intentionally change the merged `/verbose` override behavior and its current regression tests.
