## Summary

Keep open: current main and the latest release still allow session verbose progress to bypass Telegram default progress suppression, while the option contract and Telegram docs say disabled preview/final-only delivery should suppress generic tool/status chatter. Two narrow fix PRs were closed unmerged, so the bug remains actionable.

## What This Changes

Suppress Telegram internal/default tool-status messages when streaming or tool progress is disabled, including when session verbose progress is on.

## Best Possible Solution

Make `suppressDefaultToolProgressMessages` authoritative for default text-only progress delivery, preserve the existing media, exec-approval, and error exemptions, and update dispatcher/Telegram regressions plus changelog for the user-visible fix.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current tests use Telegram direct context with `verboseLevel: "on"` and `suppressDefaultToolProgressMessages: true`, then expect tool/progress text to be delivered. I did not run a live Telegram Bot API reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to change the shared dispatch predicate and its regression tests rather than adding a second force-suppression option. That keeps the existing channel contract and preserves the current explicit delivery exemptions.
