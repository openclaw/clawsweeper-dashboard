## Summary

Keep open: current main still has a source-level path where session verbose progress bypasses Telegram's default progress suppression, and the linked implementation PR is open but not merged.

## What This Changes

The issue requests that Telegram hide internal tool/status progress cards when streaming is off or tool-progress flags are disabled.

## Best Possible Solution

Land a narrow fix like https://github.com/openclaw/openclaw/pull/79924 that makes default-progress suppression authoritative for text-only tool/status cards while preserving approval, media, and error delivery.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: dispatch a Telegram direct reply with `suppressDefaultToolProgressMessages: true`, session `verboseLevel: "on"`, and a text-only tool/progress callback; current tests expect that progress text to be delivered despite suppression.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the shared suppression predicate honor `suppressDefaultToolProgressMessages` directly and update the conflicting verbose-override regressions, while leaving Telegram's existing suppression wiring in place.
