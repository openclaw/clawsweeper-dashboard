## Summary

Keep open: current main still lets verbose mode bypass Telegram/default progress suppression, and the dispatcher tests explicitly expect that path even though the Telegram docs and suppression option contract say final-only or tool-progress-disabled delivery should suppress generic progress chatter.

## What This Changes

The issue reports internal tool/status messages leaking into Telegram direct chats when streaming and tool-progress output are disabled.

## Best Possible Solution

Make Telegram channel-level progress suppression authoritative for default text-only internal progress while preserving media, exec approvals, and error delivery, then align dispatch tests, Telegram tests, docs, and live Telegram proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current dispatcher tests combine Telegram direct context, `verboseLevel: "on"`, and `suppressDefaultToolProgressMessages: true`, then expect default tool/status text to be delivered. I did not run a live Telegram chat reproduction in this read-only review.

## Solution Assessment

Unclear pending maintainer precedence. Making suppression authoritative is the narrow code direction, but it changes currently tested verbose behavior and overlaps with open Telegram progress work.
