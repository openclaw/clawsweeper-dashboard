## Summary

Keep open: current main still has a source-level path where Telegram's default tool-progress suppression is bypassed by session verbose mode, and no linked fix PR has merged. The remaining decision is policy-level because current tests intentionally codify the verbose override while Telegram docs describe off mode as final-only.

## What This Changes

The issue requests that Telegram hide internal tool/status progress messages when streaming is off or tool progress is disabled.

## Best Possible Solution

Define whether Telegram channel-level `streaming.mode: "off"` and `toolProgress: false` override session `/verbose`; if suppression wins, make the shared predicate authoritative and update Telegram/shared regressions while preserving approval, media, and error delivery.

## Reproduction Assessment

Yes. Source reproduction is high-confidence: configure a Telegram direct-chat dispatch with `suppressDefaultToolProgressMessages: true` and session `verboseLevel: "on"`; current tests expect text-only tool/status progress to be delivered despite suppression.

## Solution Assessment

Unclear until maintainers decide precedence. Making `suppressDefaultToolProgressMessages` authoritative is a narrow likely fix, but it intentionally reverses current verbose-override tests and conflicts with an open Telegram off-mode PR.
