## Summary

Keep open: current main still has the verbose-mode bypass that can deliver default tool/status progress even when Telegram asks dispatch to suppress those messages, and the issue is not fixed by a merged PR or release.

## What This Changes

The issue reports Telegram direct chats receiving internal default tool/status progress messages despite streaming and tool-progress suppression settings.

## Best Possible Solution

Land a narrow shared dispatch and regression-test fix that makes channel-level default progress suppression authoritative over session verbose mode while preserving media, exec-approval, and error delivery.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main combines Telegram direct context, `verboseLevel: "on"`, and `suppressDefaultToolProgressMessages: true`, then expects default tool/status output to be delivered. I did not perform a live Telegram run in this read-only pass.

## Solution Assessment

Yes, the narrow dispatch direction is best: make `suppressDefaultToolProgressMessages` authoritative for default text progress and keep the existing media, exec-approval, and error exemptions.
