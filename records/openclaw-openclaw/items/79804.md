## Summary

Keep open: current main and the latest release still allow session verbose mode to bypass Telegram default tool-progress suppression, and current tests explicitly expect text-only tool/status output to be delivered despite the suppression flag. Related fix PRs were closed unmerged, while adjacent Telegram progress work points to a maintainer precedence decision rather than cleanup closure.

## What This Changes

Prevent Telegram from delivering internal tool/status progress messages when streaming is off or tool-progress config is disabled.

## Best Possible Solution

Define the precedence between session `/verbose` and Telegram `streaming.mode: "off"` or `toolProgress: false`, then update shared dispatch, Telegram wiring, docs, and regressions to match that policy.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: in a Telegram direct context with `replyOptions.suppressDefaultToolProgressMessages: true` and session `verboseLevel: "on"`, the current test suite expects `dispatcher.sendToolResult` for text-only tool progress.

## Solution Assessment

Unclear until maintainers decide precedence. Making suppression authoritative is the narrow code fix for this report, but it intentionally reverses merged behavior from https://github.com/openclaw/openclaw/pull/76716 and conflicts with the direction of https://github.com/openclaw/openclaw/pull/77211.
