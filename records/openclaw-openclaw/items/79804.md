## Summary

Keep open: current main still allows verbose progress to bypass Telegram's default tool-progress suppression, and the latest release contains the same shipped path. The issue discussion also has fresh 2026.5.20 Telegram evidence, while the exact fix PRs that targeted this behavior were closed unmerged.

## What This Changes

The issue requests that Telegram hide internal tool/status/model progress messages whenever streaming is off or tool-progress visibility is disabled.

## Best Possible Solution

Make explicit Telegram final-only or tool-progress-disabled configuration suppress text-only internal tool/status/model progress across shared dispatch paths, with focused regression tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: Telegram passes suppression, current dispatch lets verbose progress defeat it, and a current test asserts the resulting visible tool summary. I did not run a live Telegram turn in this read-only review.

## Solution Assessment

Yes on direction: the maintainable fix is to honor explicit channel suppression in the shared dispatch/progress gates while preserving media, approvals, errors, and explicitly enabled progress. A Telegram-only string filter or user-side watchdog would be the wrong layer.
