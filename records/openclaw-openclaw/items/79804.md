## Summary

Keep open: current main still lets session verbose mode bypass Telegram/default progress suppression, and current dispatcher tests explicitly expect that bypass even though Telegram and shared streaming docs describe final-only or tool-progress-disabled Telegram delivery as suppressing generic internal progress chatter.

## What This Changes

The issue asks OpenClaw to prevent internal Telegram tool/status cards from being delivered when Telegram streaming and tool-progress output are disabled.

## Best Possible Solution

Make channel-level Telegram suppression authoritative for text-only internal progress, preserve media, exec approvals, and error delivery, and align shared dispatch tests with the documented final-only behavior after maintainer precedence is settled.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main has tests that combine Telegram direct context, `verboseLevel: "on"`, and `suppressDefaultToolProgressMessages: true`, then expect internal tool/progress messages to be delivered.

## Solution Assessment

Unclear pending maintainer policy. Making the suppression flag authoritative matches the current docs and option contract, but it changes tested verbose behavior and overlaps with open Telegram progress work.
