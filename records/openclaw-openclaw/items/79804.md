## Summary

Keep open: current main still lets session verbose mode bypass Telegram default progress suppression, and current tests explicitly expect the leaked tool/status messages to be delivered despite `suppressDefaultToolProgressMessages: true`.

## What This Changes

Ensure Telegram direct chats do not receive default tool/status progress cards when streaming is off or tool-progress settings are disabled.

## Best Possible Solution

Define that channel-level final-only or disabled tool-progress suppression wins over session verbose for default text cards, then update shared dispatch, Telegram wiring tests, and regressions while preserving media, errors, and exec-approval delivery.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current tests combine Telegram direct context, `verboseLevel: "on"`, and `suppressDefaultToolProgressMessages: true`, then expect default tool/status output to be delivered. I did not run a live Telegram Bot API reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers settle precedence. Making the suppression predicate authoritative is a narrow code direction, but it intentionally changes current verbose-override behavior and tests.
