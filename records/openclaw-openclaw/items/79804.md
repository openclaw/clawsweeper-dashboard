## Summary

Keep open: current main still has a source-level path where Telegram can request default tool-progress suppression, but session verbose mode makes shared dispatch send text-only tool/status cards anyway. The report is valid, but the current tests and adjacent open Telegram progress work show a policy conflict around whether `/verbose` may override channel-level final-only or tool-progress-off settings.

## What This Changes

The issue requests that Telegram hide internal tool/status progress cards when streaming/tool progress is disabled, even when verbose progress is active.

## Best Possible Solution

Define precedence between session `/verbose` and channel-level `streaming.mode: "off"` or `toolProgress: false`, then update shared dispatch, Telegram wiring, and regressions so text-only internal tool/status cards follow that policy while approvals, media, and errors still deliver.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to use a Telegram direct context with `replyOptions.suppressDefaultToolProgressMessages: true` and session `verboseLevel: "on"`; current tests assert that tool/progress messages are delivered despite suppression.

## Solution Assessment

Unclear until maintainers settle precedence. Returning the suppression flag directly is a plausible narrow fix, but it intentionally reverses the current verbose-override tests and overlaps with open Telegram progress-policy work.
