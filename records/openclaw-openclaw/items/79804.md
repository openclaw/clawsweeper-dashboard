## Summary

Keep open: current main and the latest release still let session verbose bypass `suppressDefaultToolProgressMessages`, while Telegram and shared streaming docs describe off/tool-progress-disabled modes as suppressing generic progress chatter. Closed fix PRs were not merged, and adjacent open Telegram progress PRs do not prove this issue fixed on main.

## What This Changes

The issue reports Telegram direct chats receiving internal tool/status progress cards despite streaming and tool-progress suppression settings.

## Best Possible Solution

Define the precedence between session `/verbose` and channel-level Telegram suppression, then land a narrow shared dispatch/test update that suppresses default text progress when the channel says so while preserving media, exec approvals, and errors.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Current tests set Telegram direct context with `verboseLevel: "on"` and `suppressDefaultToolProgressMessages: true`, then expect tool/status progress to be delivered; I did not run a live Telegram Bot API scenario in this read-only review.

## Solution Assessment

Unclear until maintainers settle precedence. The closed narrow fix direction aligns with docs and the option contract, but it changes current tests and conflicts with adjacent Telegram progress work.
