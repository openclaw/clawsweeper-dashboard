## Summary

Keep open: current main still contains a source-level path where session verbose progress bypasses Telegram/default progress suppression, and an open linked fix PR is the right place to resolve it before this issue closes.

## What This Changes

The issue asks OpenClaw to stop sending Telegram-visible internal tool/status cards when Telegram streaming is off or tool-progress settings are disabled, including when session verbose progress is enabled.

## Best Possible Solution

Review and land a narrow fix like https://github.com/openclaw/openclaw/pull/79924 that makes default text-only progress suppression authoritative over session verbose, while preserving explicit approval, media, and error delivery.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: set Telegram suppression true with a direct Telegram context and session `verboseLevel: "on"`; current main's regression test expects the suppressed tool text to be delivered.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable path because it changes the shared suppression predicate where the leak occurs. This issue should remain open until that PR is reviewed and merged or rejected.
