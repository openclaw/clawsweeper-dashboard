## Summary

Keep open: current main still has the reported source-level leak path because Telegram sets the suppression flag for final-only or draft-stream delivery, but shared dispatch lets session verbose mode bypass that suppression and the regression tests still assert the bypass.

## What This Changes

The issue requests that Telegram stop delivering internal tool/status progress cards when streaming is off or tool-progress visibility is disabled.

## Best Possible Solution

Define the precedence explicitly, then update shared dispatch and Telegram regressions so explicit channel-level final-only or `toolProgress: false` suppression blocks default text-only progress while media, approval prompts, and final errors still deliver.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: set Telegram suppression true with a direct Telegram context and session `verboseLevel: "on"`; current tests expect the suppressed tool/status text to be sent. I did not run a live Telegram Bot API reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers settle precedence. The likely narrow fix is to honor channel suppression regardless of verbose for default text progress, but that intentionally changes a merged `/verbose` behavior and its regression tests.
