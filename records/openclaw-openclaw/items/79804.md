## Summary

Keep open: current main and the latest release still let verbose progress bypass Telegram/default tool-progress suppression, and a current regression test intentionally asserts the reported leak path. The Atlas/Pulse correction removes one external live-incident detail, but it does not change the source-level OpenClaw reproduction.

## What This Changes

The issue reports Telegram direct chats receiving internal tool/status progress messages even when Telegram streaming and tool-progress settings are disabled.

## Best Possible Solution

Honor explicit Telegram final-only or tool-progress-disabled suppression across dispatcher and follow-up progress paths while preserving media, approvals, errors, native draft/progress callbacks, and explicitly enabled visible progress.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main combines Telegram direct context, verbose mode, and `suppressDefaultToolProgressMessages: true`, then expects text-only tool/status progress to be sent.

## Solution Assessment

Yes. The narrow maintainable fix is to make explicit Telegram off/tool-progress-disabled suppression authoritative for default text progress while preserving the documented exceptions and explicit visible-progress paths.
