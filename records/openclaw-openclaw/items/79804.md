## Summary

Keep open: current main still lets session verbose mode bypass default tool-progress suppression, and the dispatcher tests assert that Telegram direct chats should receive text-only progress when verbose is on even with suppression requested. That conflicts with the Telegram docs, streaming docs, and reply-options contract that describe final-only or tool-progress-disabled delivery as suppressing generic tool/status chatter.

## What This Changes

Telegram direct chats should not receive internal tool/status progress messages when streaming is off or tool-progress output is disabled.

## Best Possible Solution

Settle the precedence between session verbose progress and channel-level Telegram suppression, then land a narrow dispatch/test update that suppresses text-only internal progress while preserving media, exec approvals, and errors.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current tests combine Telegram direct context, `verboseLevel: "on"`, and `suppressDefaultToolProgressMessages: true`, then expect default tool/status text to be delivered.

## Solution Assessment

Unclear pending maintainer precedence. Making channel suppression authoritative is the narrow technical direction, but it changes currently tested verbose behavior and overlaps with active Telegram progress work.
