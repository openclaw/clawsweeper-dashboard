## Summary

Keep open: current main still lets session verbose mode bypass Telegram/default progress suppression, and the existing dispatcher tests intentionally assert that bypass even though Telegram and streaming docs describe final-only or tool-progress-disabled Telegram delivery as suppressing generic tool/status chatter.

## What This Changes

The issue asks Telegram to hide internal tool/status progress cards when streaming is off or tool-progress flags are false, even if verbose progress is enabled.

## Best Possible Solution

Settle the precedence between session verbose progress and channel-level Telegram suppression, then land a narrow dispatch/test update that suppresses text-only internal progress while preserving media, exec approvals, and errors.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current tests combine Telegram direct context, `verboseLevel: "on"`, and `suppressDefaultToolProgressMessages: true`, then expect default tool/status text to be delivered.

## Solution Assessment

Unclear pending maintainer precedence. Making the suppression flag authoritative is the narrow code direction, but it changes currently tested verbose behavior and overlaps with live Telegram progress work.
