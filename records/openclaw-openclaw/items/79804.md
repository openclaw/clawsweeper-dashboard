## Summary

Keep open: current main still has a source-level path where session verbose progress bypasses Telegram's default progress suppression, while the Telegram/streaming docs describe `streaming.mode: "off"` as final-only with generic progress chatter suppressed. Existing tests also intentionally codify the verbose override, so this needs maintainer precedence judgment rather than cleanup closure.

## What This Changes

The issue requests that Telegram final-only mode and explicit `toolProgress: false` settings prevent internal tool/status cards from being delivered as visible chat messages.

## Best Possible Solution

Define the precedence between session `/verbose` and channel-level `streaming.mode: "off"` or `toolProgress: false`, then update shared dispatch, Telegram wiring, and regressions so text-only internal tool/status cards are suppressed when that policy says they should be, while still allowing exec approvals, media, and errors to deliver.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: set Telegram suppression true with a direct Telegram context and session `verboseLevel: "on"`; current tests expect the tool-result text to be sent despite suppression. I did not run a live Telegram Bot API reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers decide precedence. The proposed force-suppression direction is plausible, but it would intentionally change the current tested behavior where `/verbose on` overrides preview/default progress suppression.
