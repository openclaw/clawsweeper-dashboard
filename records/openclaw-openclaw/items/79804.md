## What This Changes

The issue asks OpenClaw to prevent internal Telegram tool/status/progress messages from being delivered as visible chat when Telegram streaming is off or tool progress is disabled.

## Best Possible Solution

Land a narrow shared-dispatcher plus Telegram wiring fix that honors disabled Telegram progress across all default progress/status paths, with regression tests and redacted live Telegram proof for direct and group/forum chats.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: docs define Telegram `streaming.mode: "off"` as final-only, while current source still lets verbose progress defeat default progress suppression.

## Solution Assessment

Yes. The best fix is to honor the existing Telegram streaming/toolProgress settings in the shared progress dispatch path; adding another config option would duplicate the existing contract instead of enforcing it.
