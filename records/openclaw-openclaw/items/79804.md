## Summary

Keep open: current main still has a source-level path where Telegram off-mode/default-progress suppression is bypassed by session verbose mode, while the published Telegram and streaming docs describe off mode as final-only with generic progress chatter suppressed. The remaining question is policy precedence, not cleanup closure.

## What This Changes

The issue asks Telegram to keep internal tool/status cards out of direct chats when streaming mode is off or tool-progress config is disabled.

## Best Possible Solution

Define the precedence between Telegram final-only/toolProgress-disabled settings and session `/verbose`, then update shared dispatch, Telegram wiring, docs, changelog, and regressions to match while preserving approvals, media, and errors.

## Reproduction Assessment

Yes. A source-level path is high-confidence: Telegram passes suppression in off mode, shared dispatch ignores that suppression when session verbose is on, and a current regression test expects text-only tool result delivery under suppression plus verbose.

## Solution Assessment

Unclear. The issue's force-suppression direction matches the final-only docs, but current tests and prior verbose-restoration work intentionally prefer `/verbose`, so maintainers should settle that precedence before implementation.
