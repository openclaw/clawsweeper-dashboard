## Summary

Keep open: current main still lets session verbose mode bypass default Telegram progress suppression, and current regression tests assert that leak path even though Telegram and streaming docs describe off/tool-progress-disabled delivery as suppressing generic tool/status chatter.

## What This Changes

The issue asks Telegram to hide internal tool/status progress cards when streaming is off or tool-progress flags are false, including when session verbose progress is enabled.

## Best Possible Solution

Make channel-level Telegram suppression authoritative over session verbose for default text progress, then update dispatcher/tests while preserving approvals, media payloads, and errors.

## Reproduction Assessment

Yes. Current source and tests reproduce the path: Telegram direct context plus `verboseLevel: "on"` and `suppressDefaultToolProgressMessages: true` still expects default tool/status text to be delivered.

## Solution Assessment

Unclear pending maintainer precedence. Making the suppression flag authoritative matches the docs and type contract, but it reverses currently tested verbose behavior and overlaps with open Telegram progress work.
