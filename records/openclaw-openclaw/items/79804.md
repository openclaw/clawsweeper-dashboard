## Summary

Keep open: current main still lets session verbose mode bypass default tool-progress suppression, and the regression tests intentionally expect Telegram direct-chat tool/status text to be delivered despite the Telegram and streaming docs describing final-only/tool-progress-disabled delivery as suppressing generic progress chatter.

## What This Changes

The issue asks Telegram to suppress internal tool/status progress cards when streaming is off or tool-progress settings are disabled.

## Best Possible Solution

Settle verbose-versus-channel-suppression precedence, then land a narrow shared dispatch/test update that suppresses text-only internal progress while preserving media payloads, approval prompts, and errors, with live Telegram proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current tests combine Telegram direct context, `verboseLevel: "on"`, and `suppressDefaultToolProgressMessages: true`, then expect tool/status messages to be delivered. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Unclear until maintainers choose precedence. Making channel suppression authoritative is the narrow code direction for the reported leak, but it conflicts with current verbose-behavior tests and adjacent Telegram progress PRs.
