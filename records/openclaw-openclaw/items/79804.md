Current main still has a source-level path where Telegram/default tool-progress suppression is bypassed by session verbose mode, and an open closing PR already targets the issue, so it should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that Telegram suppress internal tool/status progress cards when streaming is off or tool-progress output is disabled.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/79924, keeping the docs-backed Telegram final-only behavior and regression coverage aligned.

Yes. Source-level reproduction is high-confidence: set Telegram/default progress suppression true with a direct Telegram context and session verboseLevel set to on; current tests expect a text-only tool result to be delivered despite suppression.

## Solution Assessment

Unclear until the linked PR is reviewed. Making suppressDefaultToolProgressMessages unconditional matches the Telegram docs for final-only delivery, but it deliberately changes existing verbose-mode regression behavior.
