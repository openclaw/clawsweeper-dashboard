## Summary

Keep open: current main still has a source-level path where session verbose progress bypasses Telegram's default progress suppression, and the adjacent open Telegram PRs do not provide a merged canonical fix.

## What This Changes

The issue asks Telegram final-only and toolProgress=false settings to suppress internal tool/status progress cards in direct chats.

## Best Possible Solution

Define the precedence policy, then update shared dispatch, Telegram reply-option wiring, regression tests, docs, and changelog so text-only internal progress is hidden when that policy says it should be while approvals, media, and errors still deliver.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: set a Telegram direct context with suppressDefaultToolProgressMessages true and session verboseLevel on, and current tests expect the tool-result text to be sent despite suppression.

## Solution Assessment

Unclear until maintainers decide precedence. The proposed force-suppression direction is plausible, but it intentionally changes the current tested behavior where verbose mode overrides progress suppression.
