  - body: The bridge reads `activityToken` from the page URL, forwards it in the WebSocket query string, and falls back to `ws://` on HTTP pages. Use a safer cookie or token-exchange path and fail closed for insecure transport before exposing this bridge.

## Security Review

Status: needs_attention

Summary: The diff introduces concrete Activity auth, token transport, and unbounded WebSocket fanout concerns that need security/owner review before merge.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:35`
  - body: `isA2uiActivityAccessAllowed` returns true when Activity mode is enabled and no token is configured, and the HTTP/WS authorization path accepts that before normal Gateway auth.
  - confidence: 0.96
- **[medium] Activity token is copied into URL query strings:** `src/canvas-host/a2ui/index.html:321`
  - body: The A2UI bridge reads `activityToken` from the page URL and forwards it in the WebSocket URL query, which can leak through logs, intermediaries, browser history, or plain HTTP deployments.
  - confidence: 0.9
- **[medium] A2UI Activity fanout is unbounded:** `src/gateway/a2ui-activity.ts:79`
  - body: The Activity hub retains the last JSONL payload and sends each payload to all clients without size limits or backpressure checks, creating gateway memory and event-loop risk.
