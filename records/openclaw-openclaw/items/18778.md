## Security Review

Status: needs_attention

Summary: The diff introduces concrete Activity auth, token transport, and unbounded broadcast concerns that need owner/security review before merge.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:35`
  - body: `isA2uiActivityAccessAllowed` returns true for enabled Activity config with no token, and A2UI HTTP/WS authorization consults it before normal bearer or capability auth.
  - confidence: 0.93
- **[medium] Activity token is placed in a WebSocket URL query:** `src/canvas-host/a2ui/index.html:323`
  - body: The browser bridge copies `activityToken` into the WebSocket query string and uses `ws://` on HTTP pages, which can leak tokens through logs, intermediaries, or cleartext transport in misconfigured deployments.
  - confidence: 0.82
- **[medium] A2UI Activity fanout is unbounded:** `src/gateway/a2ui-activity.ts:79`
  - body: The hub retains the last JSONL payload and broadcasts each payload to all clients without size limits or backpressure checks, creating a gateway availability risk.
  - confidence: 0.88
