  - body: With `activity.enabled` and no token, this returns `true`, and `server-http` accepts the Activity path before normal Gateway auth. Make Activity access fail closed unless a token or explicitly approved verifiable Discord assertion is configured and tested.
  - confidence: 0.96

  - body: This adds `canvasHost.activity` to the legacy root Canvas config, but current main migrates root `canvasHost` into `plugins.entries.canvas.config.host`. Put the Activity settings under the Canvas plugin config or get an explicit migration/product decision.
  - confidence: 0.94
- **[P2] Bound A2UI Activity fanout before retaining payloads:** `src/gateway/a2ui-activity.ts:79-82`
  - body: `broadcastPush` stores arbitrary JSONL and broadcasts it to every connected Activity WebSocket without byte limits or `bufferedAmount` handling. Add payload limits and slow-client behavior before exposing this bridge.
  - confidence: 0.91
- **[P2] Avoid putting Activity tokens in WebSocket URLs:** `src/canvas-host/a2ui/index.html:321-325`
  - body: The browser bridge copies `activityToken` into the WebSocket query and falls back to `ws://` on HTTP pages, which can leak tokens through logs, history, intermediaries, or cleartext proxy mistakes. Use a safer token exchange/cookie/header path and fail closed for insecure transport.
  - confidence: 0.82
