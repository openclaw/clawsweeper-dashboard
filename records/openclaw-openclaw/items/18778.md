## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Require Activity auth before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:35-36`
  - body: When Activity mode is enabled with no token, this returns true and server-http accepts A2UI HTTP/WS requests before normal Gateway auth. That makes a public Gateway expose A2UI if the operator enables Activity mode but forgets a token, so the bridge should fail closed or require a signed/secret Activity credential.
  - confidence: 0.94
- **[P1] Scope A2UI Activity broadcasts per launch:** `src/gateway/a2ui-activity.ts:73-82`
  - body: The hub keeps one global client set and one lastJsonl value, then broadcasts every A2UI push/reset to all Activity sockets and replays that state to new sockets. Concurrent guild/channel/session launches can receive each other's canvas stream unless connections and payloads are keyed by launch/session scope.
  - confidence: 0.9
- **[P2] Avoid sending Activity tokens over URL ws:// links:** `src/canvas-host/a2ui/index.html:321-325`
  - body: The browser bridge copies the query-string activityToken into the WebSocket URL and uses ws:// whenever the page is served over HTTP. Authenticated Activity traffic should require WSS or a protected token channel so secrets do not leak through URLs, logs, or cleartext local-network traffic.
  - confidence: 0.84
