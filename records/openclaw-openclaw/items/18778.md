Full review comments:

- **[P1] Require a token before bypassing Gateway auth:** `src/gateway/a2ui-activity.ts:35-36`
  - body: When `activity.enabled` is true but `activity.token` is empty, this returns `true` before normal Gateway auth runs. On a non-loopback Gateway this makes `/__openclaw__/a2ui/` and the Activity WebSocket public; fail closed unless maintainers approve and test an explicit public mode.
  - confidence: 0.91
- **[P1] Scope Activity broadcasts before sending canvas state:** `src/gateway/a2ui-activity.ts:73-82`
  - body: The hub keeps one `clients` set and broadcasts every A2UI push/reset to all connected Activity sockets. Two guilds/channels using Activities concurrently would receive each other's canvas state, so the bridge needs a scoped activity/session key before broadcasting.
  - confidence: 0.86
- **[P2] Move Activity config into the Canvas plugin:** `src/config/types.gateway.ts:39-55`
  - body: This adds `canvasHost.activity` to the old root Gateway config, but current main resolves Canvas host settings from `plugins.entries.canvas.config.host` and migrates root `canvasHost` as legacy. Rebase this under the Canvas plugin config to avoid a second source of truth.
  - confidence: 0.82
