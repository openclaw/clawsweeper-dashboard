## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Require a token whenever Activity access is enabled:** `src/gateway/a2ui-activity.ts:34-36`
  - body: Returning `true` when `activity.enabled` is set without a non-empty token makes every A2UI HTTP request and the new WebSocket path bypass normal gateway authentication. Fail closed on missing credentials and add missing, mismatched, and valid-token coverage for both transports.
  - confidence: 0.99
- **[P1] Scope Activity canvas state to its originating session:** `src/gateway/server-methods/nodes.ts:441-450`
  - body: This forwards every successful `canvas.a2ui.*` invocation into one process-global hub, whose client set and `lastJsonl` have no node, agent, session, guild, or channel key. Multiple users or nodes will receive and overwrite each other's canvas state; create an authorized per-Activity scope before broadcasting or replaying state.
  - confidence: 0.97
- **[P1] Move Activity configuration into the Canvas plugin:** `src/config/zod-schema.ts:371-377`
  - body: Current releases have retired runtime root `canvasHost` ownership: doctor migrates that legacy shape into `plugins.entries.canvas.config.host`, and the canvas plugin now owns A2UI routes and capability auth. Adding `activity` to the legacy root schema creates a second source of truth and bypasses the supported owner boundary; rebase this feature onto the canvas plugin config and route APIs.
  - confidence: 0.96
