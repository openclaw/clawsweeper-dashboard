## Security Review

Status: needs_attention

Summary: Needs attention because the diff can bypass Gateway auth for A2UI and exposes a public Activity WebSocket bridge without bounded fanout.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:35`
  - body: `isA2uiActivityAccessAllowed` accepts enabled Activity config with no token, and the Gateway consults it before normal bearer or capability auth on A2UI paths.
  - confidence: 0.92
- **[medium] Activity token travels in URL query strings:** `src/canvas-host/a2ui/index.html:323`
  - body: The browser bridge copies `activityToken` into the WebSocket URL and can use `ws://` on HTTP pages, which risks token exposure through cleartext transport or URL logging in misconfigured deployments.
  - confidence: 0.86
- **[medium] A2UI Activity fanout is unbounded:** `src/gateway/a2ui-activity.ts:79`
  - body: The hub retains the last JSONL payload and broadcasts each payload to all clients without size limits or backpressure checks, creating a resource-exhaustion path.
