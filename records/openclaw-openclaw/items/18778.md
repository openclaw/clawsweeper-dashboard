## Security Review

Status: needs_attention

Summary: The diff introduces Activity auth, token transport, and WebSocket fanout concerns that need security review before merge.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:35`
  - body: `isA2uiActivityAccessAllowed` accepts enabled Activity config with no token, and Gateway consults it before normal bearer or capability auth on A2UI paths.
  - confidence: 0.92
- **[medium] Activity token travels in URL query strings:** `src/canvas-host/a2ui/index.html:323`
  - body: The browser bridge copies `activityToken` into the WebSocket URL and can use `ws://` on HTTP pages, increasing leakage risk through logs, intermediaries, or cleartext transport in misconfigured deployments.
  - confidence: 0.84

  - body: The hub retains the last JSONL payload and broadcasts each payload to all clients without size limits or backpressure checks, creating a resource-exhaustion path.
