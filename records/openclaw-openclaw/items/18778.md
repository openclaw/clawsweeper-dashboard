## Security Review

Status: needs_attention

Summary: The patch introduces concrete Gateway/A2UI authorization concerns by allowing tokenless Activity access and carrying an Activity token through URLs.

- **[high] Activity access can be public without a token:** `src/gateway/a2ui-activity.ts:35`
  - body: `isA2uiActivityAccessAllowed` returns true when Activity mode is enabled and `activity.token` is empty, which can expose the A2UI Activity surface without Gateway auth or a capability token.
