## Security Review

Status: needs_attention

Summary: The diff introduces concrete A2UI activity auth and WebSocket availability concerns that need repair before merge.

Concerns:

- **[high] Activity mode can bypass Gateway auth without a token:** `src/gateway/a2ui-activity.ts:34`
  - body: `isA2uiActivityAccessAllowed` allows access whenever activity mode is enabled and no token is set, and the Gateway auth path consults it before bearer/password/trusted-proxy auth for A2UI routes.
  - confidence: 0.95
- **[medium] A2UI activity broadcasts are unbounded:** `src/gateway/a2ui-activity.ts:79`
  - body: The hub stores the last JSONL payload and sends it to every connected WebSocket without payload limits or backpressure checks, which can stall or grow memory under oversized payloads or slow clients.
  - confidence: 0.88
- **[medium] Activity token is placed in the WebSocket URL:** `src/canvas-host/a2ui/index.html:321`
  - body: The browser bridge copies `activityToken` from the page query into the WebSocket query string, which is prone to logs/history leakage and uses `ws://` when the page is served over HTTP.
  - confidence: 0.82
