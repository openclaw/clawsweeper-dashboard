Full review comments:

- **[P1] Do not restore query-string authentication for A2UI:** `src/gateway/a2ui-activity.ts:27-29`
  - body: This accepts `activityToken` from the URL as an alternative to normal Gateway authentication for both the A2UI page and its WebSocket.   - confidence: 0.98
- **[P1] Move Activity ownership to the Discord plugin boundary:** `src/gateway/server-http.ts:117-121`
  - body: This authorizes Discord-specific Activity access directly in Gateway code while the component implementation remains under the removed `src/discord` path. Current main owns Discord under `extensions/discord`, and the scoped Gateway policy requires plugin-owned bypass descriptors to use a lightweight plugin artifact. Reimplement this through the current plugin boundary rather than restoring the old core ownership.
  - confidence: 0.97
