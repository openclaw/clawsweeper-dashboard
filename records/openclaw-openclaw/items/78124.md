Full review comments:

- **[P1] Keep `user` ahead of header-derived session hints:** `src/gateway/http-utils.ts:144-148`
  - body: Current docs and the PR body preserve OpenAI `user` as a stable session source after `x-openclaw-session-key`, but this branch resolves chat/thread headers before `params.user`. Existing clients that send both would silently move from an `openai-user:*` session to an `openai-chat:*` session on upgrade.
  - confidence: 0.9
- **[P2] Read Open-WebUI's forwarded chat header:** `src/gateway/http-utils.ts:145-146`
  - body: Open-WebUI defaults its forwarded chat-id header name to `X-OpenWebUI-Chat-Id` and forwards `metadata.chat_id` under that configured header when forwarding is enabled. Reading only `x-chat-id` and `x-thread-id` can leave the reported Open-WebUI flow on the random UUID fallback.
  - confidence: 0.92
- **[P2] Scope the shared resolver change to the intended endpoint:** `src/gateway/http-utils.ts:145-148`
  - body: `resolveGatewayRequestContext` is also used by `/v1/responses`, so adding these hints here changes Responses session routing without docs or tests for that endpoint. Gate the hints to Chat Completions or cover the shared contract explicitly, including interaction with `previous_response_id` scope.
  - confidence: 0.86
