## Summary

Keep open: current main still has the linked session-reuse gap, and this PR is not ready because it misses Open-WebUI's documented header and lacks after-fix real behavior proof.

## What This Changes

This PR derives OpenAI-compatible gateway session keys from `x-chat-id` or `x-thread-id` headers and adds request-context tests for that routing.

## Best Possible Solution

Define and implement a documented stable-session contract for OpenAI-compatible clients: explicit OpenClaw session key first, Open-WebUI's default `x-openwebui-chat-id` when forwarded, any deliberate generic aliases, regression tests, docs, and real multi-turn proof.

## Reproduction Assessment

Yes, from source. Current main randomizes the OpenAI-compatible session when no `x-openclaw-session-key` or `user` is present, and this PR still misses Open-WebUI's default `x-openwebui-chat-id`; a live multi-turn Open-WebUI run is still missing.

## Solution Assessment

No. Header-based session correlation is a reasonable direction, but this patch misses Open-WebUI's default header and leaves the precedence/docs/proof contract unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Read Open-WebUI's default chat header:** `src/gateway/http-utils.ts:144-147`
  - body: Open-WebUI currently forwards `metadata.chat_id` as `X-OpenWebUI-Chat-Id` when forwarded user headers are enabled, but this patch only reads `x-chat-id` and `x-thread-id`. Default Open-WebUI requests would still fall through to `randomUUID()`, so the linked session-reuse problem remains.
  - confidence: 0.94
- **[P3] Preserve the documented user precedence:** `src/gateway/http-utils.ts:144-147`
  - body: The PR body says the OpenAI `user` path still has higher precedence, but the new header lookup runs before `params.user`. Requests that already rely on the documented `user`-based stable session can be silently rerouted when a chat/thread header is also present; either keep `user` ahead or document and test the intentional precedence change.
  - confidence: 0.82
