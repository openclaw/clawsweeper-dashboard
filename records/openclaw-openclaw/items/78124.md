## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep user ahead of header-derived session hints:** `src/gateway/http-utils.ts:145-148`
  - body: Current docs and the PR body preserve OpenAI `user` as the stable session source after `x-openclaw-session-key`, but this lookup runs before `params.user`. Existing clients that send both `user` and a chat/thread header would silently move from an `openai-user:*` session to an `openai-chat:*` session on upgrade.
  - confidence: 0.9
- **[P2] Read Open-WebUI's forwarded chat header:** `src/gateway/http-utils.ts:145-146`
  - body: Open-WebUI defaults its forwarded chat-id header name to `X-OpenWebUI-Chat-Id` and forwards `metadata.chat_id` under that configured header when the forwarding path is enabled. This patch only reads `x-chat-id` and `x-thread-id`, so the reported Open-WebUI flow can still fall through to the random UUID fallback.
  - confidence: 0.92
- **[P2] Scope shared resolver changes to intended endpoints:** `src/gateway/http-utils.ts:145-148`
  - body: `resolveGatewayRequestContext` is also used by `/v1/responses`, so adding these hints here changes Responses session routing without docs or tests for that endpoint. Gate the hints to Chat Completions or cover the shared contract explicitly, including interaction with `previous_response_id` scope.
  - confidence: 0.86
