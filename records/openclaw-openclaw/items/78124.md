## Summary

Keep this PR open, but do not pass it yet: current `main` still has the random-session fallback the linked issue reports, while this patch needs session-contract repair, conflict resolution, and real Open-WebUI behavior proof before merge.

## What This Changes

The PR adds `x-chat-id` and `x-thread-id` handling to the shared Gateway request-context helper and adds resolver tests for those headers.

## Best Possible Solution

Repair this branch, or replace it with a narrow Gateway session-correlation contract that supports Open-WebUI's real forwarded chat-id header, preserves existing `user` precedence unless maintainers intentionally change it, scopes or documents Responses behavior, and includes real multi-turn proof.

## Solution Assessment

No, not as-is. Header-based correlation is plausible, but this patch misses Open-WebUI's default forwarded header, changes `user` precedence, broadens shared resolver behavior, and lacks live proof.

  - body: Current docs and the PR body preserve OpenAI `user` as a stable session source after `x-openclaw-session-key`, but this branch resolves chat/thread headers before `params.user`.
