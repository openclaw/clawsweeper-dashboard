## Summary

Keep open: current main still has the linked random-session fallback, but this PR is not merge-ready because it changes a shared Gateway session resolver, misses Open-WebUI's current default forwarded chat header, changes `user` precedence, conflicts with main, and has only test-based proof.

## What This Changes

The PR adds `x-chat-id` and `x-thread-id` header-derived session keys in the shared Gateway request-context helper and adds resolver tests for those headers.

## Best Possible Solution

Repair this branch, or replace it with a narrow Gateway session-correlation contract that supports Open-WebUI's actual forwarded chat-id header, preserves `user` precedence unless maintainers intentionally change it, scopes or documents Responses behavior, and includes redacted real multi-turn proof.
