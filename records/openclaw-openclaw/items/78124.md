Full review comments:

- **[P1] Keep user ahead of inferred session headers:** `src/gateway/http-utils.ts:144-148`
  - body: Current docs and tests define OpenAI `user` as the stable session source after `x-openclaw-session-key`, but this branch resolves chat/thread headers before `params.user`.   - confidence: 0.9
- **[P2] Read Open-WebUI's forwarded chat header:** `src/gateway/http-utils.ts:145-146`
  - body: Open-WebUI source forwards the chat id through `FORWARD_SESSION_INFO_HEADER_CHAT_ID`, defaulting to `X-OpenWebUI-Chat-Id`, when forwarding is enabled. Reading only `x-chat-id` and `x-thread-id` can leave the reported Open-WebUI flow on the random UUID fallback.
  - confidence: 0.92
- **[P2] Scope chat headers before using the shared resolver:** `src/gateway/http-utils.ts:145-148`
  - body: `resolveGatewayRequestContext` is also used by `/v1/responses` before `previous_response_id` reuse, so adding chat/thread hints here changes Responses session selection without docs or tests. Gate the hints to Chat Completions or explicitly cover the shared contract.
  - confidence: 0.86
