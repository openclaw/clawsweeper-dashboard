## Summary

Keep open: the change is aimed at a real OpenAI-compatible session continuity problem, but the current patch misses Open-WebUI's default forwarded chat-id header and still lacks after-fix real behavior proof.

## What This Changes

The PR adds chat/thread header-derived session keys to the OpenAI-compatible gateway request context and adds request-context tests for those headers.

## Best Possible Solution

Define and document a stable-session contract for OpenAI-compatible clients: full OpenClaw session override first, OpenAI `user` next unless intentionally changed, Open-WebUI's `x-openwebui-chat-id` when enabled, deliberate generic aliases, regression tests, and real multi-turn proof.

## Reproduction Assessment

Yes, at source level for the OpenClaw side: current main falls back to `randomUUID()` when no explicit OpenClaw session key or OpenAI `user` is present. I did not run a live multi-turn Open-WebUI setup.

## Solution Assessment

No. Header-based correlation is the right direction, but this patch misses Open-WebUI's default header and changes the documented `user` precedence without docs or coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Read Open-WebUI's default chat header:** `src/gateway/http-utils.ts:142-143`
  - body: Open-WebUI's default forwarded chat id header is `X-OpenWebUI-Chat-Id`, but this branch only reads `x-chat-id` and `x-thread-id`. Default Open-WebUI requests with chat metadata forwarding enabled would still fall through to a random session key, leaving the reported session proliferation unresolved.
  - confidence: 0.94
- **[P3] Preserve the documented user precedence:** `src/gateway/http-utils.ts:141-145`
  - body: The PR body says the OpenAI `user` path still has higher precedence, but the new header lookup runs before `params.user`. Requests that already rely on `user`-based stable sessions can be silently rerouted when a chat/thread header is also present; keep `user` ahead or document and test the intentional precedence change.
  - confidence: 0.84
