## Summary

Keep open: the PR is aimed at the right session-reuse problem, but it does not yet match Open-WebUI's documented chat-id header and only provides test output rather than after-fix real behavior proof. ([docs.openwebui.com](https://docs.openwebui.com/reference/env-configuration/?utm_source=openai))

## What This Changes

The PR adds deterministic OpenAI-compatible session-key derivation from `X-Chat-Id` or `X-Thread-Id` headers and tests explicit-session precedence.

## Best Possible Solution

Support Open-WebUI's documented `x-openwebui-chat-id` default, keep deliberate generic fallbacks such as `x-thread-id` covered by tests and docs, add the required changelog entry, and include real runtime proof of multi-message session reuse.

## Reproduction Assessment

Yes, from source: current main ignores chat/thread headers and falls back to `randomUUID()` when neither `x-openclaw-session-key` nor the OpenAI `user` field is present. I did not run a live Open-WebUI setup, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No. The proposed direction is reasonable, but this implementation checks `x-chat-id` instead of Open-WebUI's documented `x-openwebui-chat-id`, and it still needs public docs, changelog coverage, and real behavior proof.
