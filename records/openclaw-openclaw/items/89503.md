## What This Changes

The PR changes the SDK EventHub and OpenClaw event pump so transport or normalization failures reject normalized SDK stream consumers instead of ending as a clean stream.

## Best Possible Solution

Land this PR or an equivalent narrow SDK fix after maintainers accept rejected async iteration as the public failure contract for normalized SDK stream pump errors.

## Reproduction Assessment

Yes. Source inspection on current main shows pump failures close the normalized EventHub without preserving the cause, and the linked report plus PR body provide a focused SDK transport-boundary reproduction; I did not execute it because this review is read-only.

## Solution Assessment

Yes, with the public-contract caveat. Carrying an error through EventHub.close(error) keeps the fix at the SDK stream owner boundary and covers active and future normalized consumers; a typed SDK error event would be the main alternative if maintainers reject thrown iterator failures.
