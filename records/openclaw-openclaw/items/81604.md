## What This Changes

The PR forwards Telegram `resolveCliActionRequest` from the exported action wrapper and adds unit coverage for `thread-create` to `topic-create` remapping.

## Best Possible Solution

Merge the wrapper forwarding fix after real Telegram topic-creation proof or an explicit maintainer proof override, with regression coverage aimed at the exported `telegramPlugin.actions` path.

## Reproduction Assessment

Yes at source level. Current main's CLI asks the exported Telegram plugin for `resolveCliActionRequest`, the exported wrapper lacks that hook, and the fallback bare `thread-create` action reaches a Telegram handler that only supports `topic-create`.

## Solution Assessment

Mostly yes. Forwarding the existing resolver through the wrapper is the narrow maintainable source fix; the final version should also prove the exported wrapper path and provide live Telegram proof or a maintainer override.
