## Summary

Keep open. Current main and v2026.6.6 have tracked browser-tab cleanup for idle/cap sweeps, session reset/delete, subagent completion, and cron completion, but they still do not close temporary primary-agent browser tabs at ordinary task completion; the discussion’s ownership-based approach needs a maintainer product decision before a safe fix is queued.

## What This Changes

The issue requests ownership-aware automatic cleanup of browser tabs/windows that OpenClaw opens temporarily for automation tasks.

## Best Possible Solution

Extend the browser-owned tracker with explicit run ownership and preservation semantics, then close only unpreserved run-created pages after the chosen primary-agent terminal outcome while keeping idle/cap cleanup as the fallback.

## Reproduction Assessment

Yes, at source level: current main tracks browser-opened tabs and cleans them on lifecycle events or idle/cap sweeps, but there is no ordinary primary-agent task-completion cleanup path. I did not run a live browser scenario.

## Solution Assessment

No for blanket closure; the ownership-based direction in the discussion is safer, but maintainers still need to choose the terminal event and preservation contract before implementation.
