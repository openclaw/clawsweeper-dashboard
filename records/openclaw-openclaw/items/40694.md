## Summary

Keep open: current main and v2026.6.5 provide ownership-based idle/cap cleanup plus cron, subagent, and session-reset cleanup, but they do not implement ordinary primary-agent task-completion tab closure; the related duplicate was closed in favor of this canonical issue.

## What This Changes

Request browser tabs/windows opened for temporary agent automation to close automatically after task completion while preserving user-owned or explicitly kept pages.

## Best Possible Solution

Define an ownership-aware primary-run completion policy that closes only run-owned or explicitly ephemeral tabs, preserves foreground/pinned/user-requested tabs, and reuses the existing tabCleanup/lifecycle seams with regression coverage.

## Reproduction Assessment

Yes, from source inspection: direct browser opens are tracked, but current cleanup is idle/cap sweeps plus cron, subagent, and session-reset lifecycle cleanup, not ordinary primary task completion. I did not run a live browser repro in this read-only review.

## Solution Assessment

No, current shipped idle/cap cleanup is only a partial mitigation for this issue. The best fix needs explicit primary-run completion semantics instead of blanket-closing browser state.
