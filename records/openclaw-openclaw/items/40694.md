## Summary

Keep open. Current main and v2026.6.5 provide session-tracked cleanup on reset/delete/subagent completion plus primary-session idle/cap sweeps, but they do not close temporary primary-agent tabs at ordinary task completion; the ownership-based proposal in the discussion is useful and needs a maintainer product decision rather than stale cleanup.

## What This Changes

Add ownership-aware automatic closure of browser tabs or windows opened temporarily for an agent task while preserving pages the user wants to keep.

## Best Possible Solution

Extend the browser-owned tracker with an explicit run ownership and preserve contract, then close only unpreserved run-created pages after the selected terminal run outcome while retaining idle/cap cleanup as the safety net for reusable primary sessions.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.5 close tracked tabs on session/subagent lifecycle endings or primary-session idle/cap sweeps, so an ordinary completed primary-agent task can leave its tracked tabs reusable; no live browser run was performed.

## Solution Assessment

Unclear until maintainers select the product boundary. Ownership-based cleanup is safer and more maintainable than blanket closure, but the run terminal event and explicit preservation behavior must be defined before implementation.
