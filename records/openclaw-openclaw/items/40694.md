## Summary

Keep open. Current main and the latest release cover browser tab cleanup through tracked idle/cap sweeps plus reset, subagent, and cron lifecycle paths, but they still do not implement automatic primary-agent temporary-tab closure at ordinary task completion.

## What This Changes

The issue requests automatic cleanup of temporary browser tabs or windows opened by browser automation once an agent task finishes, while preserving user-owned pages.

## Best Possible Solution

Define browser-run ownership and explicit preserve semantics, then close only unpreserved run-created pages at the chosen primary-agent terminal event while keeping idle/cap cleanup as a fallback.

## Reproduction Assessment

Yes for the remaining gap: source inspection shows current main tracks browser-opened tabs and cleans them on idle/cap sweeps or reset/subagent/cron lifecycle paths, but no ordinary primary-agent task-completion finalizer exists. No live browser run was executed in this read-only review.

## Solution Assessment

No for blanket closure. The safer path is the ownership-based approach raised in discussion, with maintainers choosing the terminal event and preservation contract before implementation.
