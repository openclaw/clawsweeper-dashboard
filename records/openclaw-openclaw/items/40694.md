## Summary

Keep open. Current main has session-scoped tab tracking, idle/cap cleanup for primary browser sessions, and lifecycle cleanup for subagents, cron, and resets, but it still lacks run-owned temporary-tab metadata and an ordinary primary task-completion cleanup hook; changing that default needs product/UX judgment to avoid closing pages users expect to keep.

## What This Changes

Request that browser tabs or windows opened for temporary automation tasks be treated as ephemeral and closed automatically after task completion unless explicitly kept.

## Best Possible Solution

Add run-owned ephemeral browser-tab metadata and a primary-run completion cleanup hook that closes only tabs created for that run while preserving explicit user-owned, foregrounded, or kept pages.

## Reproduction Assessment

Yes. Source inspection shows browser opens are tracked by session identity, primary cleanup is idle/cap based, and the ordinary primary task path has no ownership-aware completion cleanup.

## Solution Assessment

Unclear as requested. The cleanup goal is sound, but the maintainable fix is ownership-based run cleanup rather than a blanket close-all hook or a broad auto-close switch.
