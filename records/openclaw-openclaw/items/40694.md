## Summary

Keep open. Current main has useful tracked-tab cleanup for idle/cap sweeps plus reset, cron, auto-reply rollover, and subagent lifecycle cleanup, but it still does not close temporary primary-agent browser tabs at ordinary task completion; the discussion also correctly points toward ownership-based cleanup rather than blanket closure.

## What This Changes

The issue requests ownership-based automatic closure of browser tabs or windows opened temporarily by automation after a task completes, while preserving user-owned pages.

## Best Possible Solution

Define browser-run ownership and explicit preserve semantics, then close only unpreserved run-created pages at a maintainer-chosen primary-agent terminal event while keeping idle/cap cleanup as fallback.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens are tracked by session, primary cleanup is a background idle/cap sweep, and the ordinary embedded primary-run finalizer does not call browser lifecycle cleanup.

## Solution Assessment

No. Current main is only a partial mitigation; the best fix is ownership-based cleanup at a maintainer-chosen primary-agent terminal event, not blanket tab closure.
