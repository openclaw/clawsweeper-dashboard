## Summary

Keep open. Current main has useful browser tab cleanup, but it is idle/cap-based for primary-agent sessions and lifecycle-based for subagent/cron runs; it does not implement the requested default of closing temporary primary-task tabs when an ordinary browser-driven task completes.

## What This Changes

The issue requests ownership-aware automatic cleanup of browser tabs or windows opened only for temporary agent automation tasks after the task completes.

## Best Possible Solution

Add an ownership-based task/run tab lifecycle that marks tabs created for ephemeral automation work, closes only those tabs on run completion by default, preserves explicit or user-owned tabs, and updates config/docs/tests around the chosen compatibility policy.

## Reproduction Assessment

Yes, at source level. Current main documents and implements primary-agent tab cleanup as idle/cap-based background cleanup, and the only task-completion cleanup call sites are child/cron lifecycle paths rather than ordinary primary browser task completion.

## Solution Assessment

No, the current implementation is only a partial answer to the requested behavior. The maintainable direction is ownership-based run cleanup with explicit preservation semantics, not a blanket close of every browser tab after every task.
