## Summary

Keep open. Current main and v2026.5.22 have tracked browser tab cleanup by idle age, per-session cap, and bounded child/session lifecycle hooks, but primary-agent tabs are still documented as reusable after a task rather than closed at task completion; LakshmanTurlapati's FSB comment supports solving this with ownership semantics instead of blanket closure.

## What This Changes

Request that browser tabs or windows opened only for temporary automation work close automatically after the task completes unless the user explicitly wants to keep them.

## Best Possible Solution

Define an ownership-based primary-task browser cleanup policy that distinguishes agent-owned ephemeral tabs from user-owned or explicitly preserved tabs, then implement it in the browser plugin lifecycle with docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens are tracked, but primary cleanup is an idle/cap sweep and the docs state active primary tabs remain reusable; I did not run a live browser because this review is read-only.

## Solution Assessment

No, not as a blanket default close. The safer solution is ownership-based cleanup that preserves user-owned or explicitly kept tabs while closing agent-owned ephemeral tabs at a well-defined primary task boundary.
