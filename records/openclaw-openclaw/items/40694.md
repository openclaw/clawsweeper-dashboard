## Summary

Keep open. Current main and v2026.6.10 have tracked-tab idle/cap cleanup plus reset, cron, and subagent lifecycle cleanup, but they still do not close ordinary primary-agent temporary browser tabs at task completion; the FSB-linked comment also points toward the missing ownership-based cleanup shape rather than a blanket close.

## What This Changes

The issue requests automatic cleanup of browser tabs or windows that were opened only for temporary automation work after a task completes.

## Best Possible Solution

Define run-owned versus user-preserved browser tab semantics, then close only unpreserved run-created primary-agent tabs at the selected terminal event while keeping idle/cap cleanup as fallback.

## Reproduction Assessment

Yes, from source inspection rather than a live browser run: current main tracks browser-opened tabs and cleans them on idle/cap or lifecycle paths, but the ordinary primary-agent run finalizer does not close temporary tabs on completion.

## Solution Assessment

No, the current implementation is only a partial mitigation. The best fix needs maintainer-chosen ownership and preserve semantics rather than blanket closing every tracked primary tab.
