## Summary

Keep open. Current main and the latest release still provide background idle/cap cleanup plus reset, rollover, cron, and subagent lifecycle cleanup, but they do not implement the requested ownership-based close of temporary primary-agent browser tabs when an ordinary task completes.

## What This Changes

The issue requests automatic cleanup of temporary browser tabs or windows opened for one-off browser automation after a task completes.

## Best Possible Solution

Define browser-run ownership and explicit preserve semantics, then close only unpreserved run-created primary-session pages at the chosen primary-agent terminal event while keeping idle/cap cleanup as a fallback.

## Reproduction Assessment

Yes. Source inspection shows opened browser tabs are tracked by session and cleaned by idle/cap sweeps or selected lifecycle paths, while ordinary primary embedded-run completion does not call browser lifecycle cleanup.

## Solution Assessment

No. Current main is only a partial mitigation; the maintainable fix is ownership-based cleanup at a maintainer-chosen primary-agent terminal event, not blanket tab closure.
