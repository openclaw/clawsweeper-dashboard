## Summary

Keep open. Current main and v2026.6.10 still implement session-tracked browser tab cleanup via idle/cap sweeps plus reset, cron, and subagent lifecycle cleanup, but they do not close primary-agent temporary tabs at ordinary task completion; the ownership-based pattern suggested in discussion is useful, but it needs maintainer product judgment before implementation.

## What This Changes

The issue requests automatic cleanup of browser tabs or windows opened only for temporary automation work after task completion, while preserving user-owned or explicitly kept pages.

## Best Possible Solution

Define browser-run ownership and explicit preserve semantics, then close only unpreserved run-created primary-session pages at a maintainer-chosen primary-agent terminal event while keeping idle/cap cleanup as a fallback.

## Reproduction Assessment

Yes, source-reproducible. Current source shows browser tabs are tracked and cleaned by idle/cap sweeps or selected lifecycle paths, while ordinary primary embedded-run completion does not call browser lifecycle cleanup.

## Solution Assessment

No, current main is only a partial mitigation. The safer solution is ownership-based task-end cleanup at a maintainer-chosen primary-agent terminal event, not blanket tab closure.
