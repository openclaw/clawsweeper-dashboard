## Summary

Keep open. Current main and the latest release have ownership-based browser tab cleanup for idle/excess primary tabs plus explicit lifecycle cleanup for subagents, cron, ACP, and session resets, but they intentionally do not close ordinary primary-agent browser tabs immediately when a task completes; the closed duplicate at https://github.com/openclaw/openclaw/issues/56271 also points here as the canonical remaining product question.

## What This Changes

The issue requests automatic cleanup of browser tabs/windows opened only for temporary automation tasks after the task completes.

## Best Possible Solution

Define an ownership-based primary-task cleanup policy that preserves user-owned or explicitly kept tabs, then implement it through the browser plugin/runtime contract with docs and regression coverage rather than changing the existing idle sweep blindly.

## Reproduction Assessment

Yes, from source inspection: a primary-agent browser `open` is tracked, but primary cleanup runs by idle/cap sweep and the docs say active primary tabs remain reusable. I did not run a live browser because this review is read-only, but the current code path and tests make the remaining behavior clear.

## Solution Assessment

No, not as a blind default close. The current ownership-based idle cleanup is safer than blanket task-end closure, and the remaining request needs a maintainer decision about primary task boundaries and preserve semantics before implementation.
