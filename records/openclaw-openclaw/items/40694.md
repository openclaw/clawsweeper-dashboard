## Summary

Keep open. Current main and the latest release still provide idle/cap cleanup plus lifecycle cleanup for subagent, cron, ACP, and session rollover paths, but the requested primary task-completion default is not implemented and would change documented primary-tab reuse behavior.

## What This Changes

The issue requests automatic cleanup of browser tabs or windows opened temporarily for one-off automation tasks after task completion.

## Best Possible Solution

Define task/run-scoped temporary browser-tab ownership in the browser plugin or SDK, then close only temporary tracked tabs on successful primary task completion while preserving explicit reusable, user-owned, or pinned tabs and updating docs/tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: direct browser opens are tracked at session scope, chat.send completion does not call browser cleanup, and docs say active primary tabs stay reusable until idle/cap cleanup.

## Solution Assessment

Unclear. The requested default is reasonable, but the maintainable solution is not a blanket close-all hook because current behavior intentionally reuses primary tabs; task-scoped temporary/preserve semantics are safer.
