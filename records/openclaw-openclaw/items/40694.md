## Label Justifications

- `P2`: The original request addressed recurring browser clutter and resource consumption with limited blast radius rather than an urgent outage.
- `impact:session-state`: The behavior governs cleanup of browser state associated with completed agent sessions.

## Summary

Current main and shipped releases implement the requested behavior through session ownership: browser tabs opened by an agent session are tracked, explicit closes are untracked, and remaining owned tabs are closed at session, cron, or subagent lifecycle completion without blanket-closing unrelated browser state.

## What This Changes

Request ownership-aware automatic cleanup of temporary browser tabs and windows when an agent task completes.

## Best Possible Solution

Keep the shipped canonical design: track tabs created by each session, untrack explicit closes, close remaining owned tabs at terminal lifecycle events, and retain idle/count sweeps as a fallback without closing unrelated user tabs.

## Reproduction Assessment

No. On current main and in v2026.6.5, tabs opened through browser automation are associated with session keys and closed by the relevant lifecycle cleanup paths, with focused regression coverage.

## Solution Assessment

Yes. The implementation uses ownership tracking and explicit untracking instead of blanket closure, matching the safer approach proposed in the issue discussion and avoiding surprise loss of unrelated user tabs.
