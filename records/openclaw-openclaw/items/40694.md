## Summary

Keep open. Current main and v2026.5.20 have tracked-tab cleanup for idle/excess primary tabs plus bounded lifecycle cleanup for subagent, cron, ACP, and session reset paths, but ordinary primary-agent browser tabs are still intentionally reusable after a task completes; the FSB comment usefully reinforces that any fix should be ownership-based rather than a blanket close.

## What This Changes

The issue requests automatic cleanup of browser tabs/windows opened for temporary browser-driven tasks once the task is complete.

## Best Possible Solution

Define an ownership-based primary-task browser cleanup policy that distinguishes agent-owned ephemeral tabs from user-owned or explicitly preserved tabs, then implement it through the browser plugin lifecycle with docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens are tracked, but primary cleanup is an idle/cap sweep and the docs state active primary tabs remain reusable; I did not run a live browser because this review is read-only.

## Solution Assessment

No, not as a blind default close. The current idle/cap policy is safer than blanket task-end closure, and the remaining request needs a maintainer decision about primary task boundaries and preserve semantics before implementation.
