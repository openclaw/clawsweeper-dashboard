## Summary

Keep open. Current main has tracked tab cleanup for idle/cap sweeps and lifecycle cleanup for cron/subagent/session teardown, but ordinary primary chat task completion still does not close browser tabs opened for temporary work, and the existing docs intentionally preserve active primary tabs for reuse.

## What This Changes

Request automatic cleanup of browser tabs/windows opened for temporary automation tasks after task completion while preserving explicitly user-owned pages.

## Best Possible Solution

Define task/run-owned temporary tab metadata in the browser runtime, close only agent-owned ephemeral tabs on successful primary task completion, preserve explicit user-owned or foregrounded tabs, and cover both direct and proxy browser opens with docs and regression tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: direct browser opens are session-tracked, the primary chat finalizer does not call browser cleanup, and docs state primary tabs remain reusable until idle/cap cleanup.

## Solution Assessment

Unclear as requested. The cleanup goal is reasonable, but the maintainable path is ownership-based task cleanup rather than a blanket close-all hook or an unscoped config toggle.
