## Summary

Keep open. Current main still has session-tracked browser tabs, idle/cap primary cleanup, and lifecycle cleanup for cron/subagent/reset paths, but ordinary primary chat task completion does not close temporary browser tabs; the remaining fix needs ownership-based product/UX approval rather than a blanket finalizer.

## What This Changes

Request that browser tabs/windows opened only for temporary automation work close by default when the task completes, while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Add run-owned or agent-owned ephemeral tab metadata plus a primary-run completion cleanup that closes only those temporary tabs after successful runs, preserving user-owned or explicitly kept pages and documenting the behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows direct browser opens are only session-tracked, primary cleanup is idle/cap based, and normal chat completion has no browser cleanup call.

## Solution Assessment

Unclear as requested. The cleanup goal is reasonable, but the maintainable solution is ownership-based task cleanup rather than a blanket close-all hook or generic auto-close switch.
