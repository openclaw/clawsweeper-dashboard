## Summary

The central behavior remains open on current main: browser tabs opened by a primary agent are tracked and reclaimed by idle/cap cleanup, while cron, subagent, ACP, and reset paths have lifecycle cleanup, but there is still no run-owned primary task-completion finalizer. The safest path needs maintainer product/UX judgment because the code currently preserves reusable active primary tabs and lacks metadata to distinguish temporary run-owned pages from tabs a user expects to keep.

## What This Changes

Request default cleanup of temporary browser tabs/windows opened for one-off agent automation after task completion while preserving explicitly kept user pages.

## Best Possible Solution

Add run-owned ephemeral browser tab metadata and a primary-run completion cleanup path that closes only tabs created for that run while preserving explicit user-owned, foregrounded, or kept pages, with docs and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows browser opens are session-tracked, primary cleanup is idle/cap based, and ordinary primary task completion has no ownership-aware browser cleanup call.

## Solution Assessment

Unclear as requested. The cleanup goal is sound, but the maintainable solution is ownership-based task cleanup rather than a blanket close-all hook or another broad auto-close switch.
