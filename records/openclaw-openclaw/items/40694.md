## Summary

Keep open. Current main still has session-tracked browser tabs, idle/cap primary cleanup, and lifecycle cleanup for cron/subagent/reset paths, but normal primary chat task completion does not close temporary browser tabs; the safest remaining work needs an ownership-based product decision rather than a blanket close hook.

## What This Changes

Request automatic cleanup of temporary browser tabs/windows opened by browser automation after task completion while preserving explicitly user-owned pages.

## Best Possible Solution

Add run-owned ephemeral browser-tab metadata and task-completion cleanup that closes only agent-owned temporary tabs after successful primary runs, while preserving user-owned, pinned, or foregrounded tabs and documenting the behavior.

## Reproduction Assessment

Yes. Source inspection on current main confirms direct browser opens are only session-tracked, primary cleanup is idle/cap based, and the primary chat completion path does not call browser cleanup.

## Solution Assessment

Unclear as requested. The cleanup goal is reasonable, but the maintainable fix is run-owned/agent-owned cleanup semantics rather than a blanket close-all hook or generic auto-close switch.
