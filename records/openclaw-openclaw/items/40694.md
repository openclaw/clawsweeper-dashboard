## Summary

Keep open. Current main has idle/cap-based cleanup plus subagent, cron, and session-reset lifecycle cleanup, but it still does not implement primary task-completion cleanup for browser-opened temporary tabs; the newest discussion also reinforces an ownership-based design instead of a blanket close.

## What This Changes

Request to close browser-opened temporary tabs/windows by default after browser-driven task completion while preserving explicitly user-owned tabs.

## Best Possible Solution

Define task-scoped browser-tab ownership in the browser plugin/SDK, then close only agent-owned temporary tabs on successful primary task completion while preserving explicit user, foreground, pinned, or debug-preserved tabs and updating docs/tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: direct opens are tracked by session, but chat.send completion does not call browser cleanup and docs preserve active primary tabs until idle/cap cleanup. I did not run a live browser E2E because this was a read-only sweep.

## Solution Assessment

Unclear for the requested blanket default. The safer maintainable solution is the ownership-based approach already suggested in discussion: close tabs created for the completed run while preserving user-owned or explicitly retained tabs.
