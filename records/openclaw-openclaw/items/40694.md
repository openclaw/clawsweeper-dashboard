## Summary

Keep open. Current main has session-scoped tracking, subagent/cron/reset cleanup, and primary idle/cap sweeps, but it still documents active primary tabs as reusable and chat completion does not trigger automatic cleanup of temporary browser-opened tabs; #43666 is closed/unmerged prior art, not a merged fix.

## What This Changes

Request browser tabs or windows opened only for temporary automation inspection, fetch, or verification to close automatically when the task completes while preserving explicit user-owned pages.

## Best Possible Solution

Define task-scoped browser tab ownership in the browser plugin, then run best-effort cleanup on successful primary task completion for temporary tabs only, with tests and docs preserving explicit user-owned pages.

## Reproduction Assessment

Yes. Source inspection on current main shows browser.open tracks local tabs and primary cleanup is only idle/cap based, while chat.send completion does not call browser cleanup and docs say primary tabs remain reusable.

## Solution Assessment

Unclear. The requested direction is reasonable, but the narrow maintainable fix is not to close all tracked tabs blindly; it needs task-scoped temporary ownership plus preserve semantics and a primary completion hook.
