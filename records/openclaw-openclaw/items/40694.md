## Summary

Keep open. Current main has useful tracked-tab cleanup for idle/cap sweeps plus reset, cron, and subagent lifecycle cleanup, but it still does not implement the requested ownership-based close of temporary primary-agent browser tabs when an ordinary task completes.

## What This Changes

The issue requests automatic cleanup of browser tabs or windows opened only for temporary automation work after task completion, while preserving user-owned or explicitly kept pages.

## Best Possible Solution

Define browser-run ownership and explicit preserve semantics, then close only unpreserved run-created pages at the chosen primary-agent terminal event while keeping idle/cap cleanup as fallback.

## Reproduction Assessment

Yes, from source inspection rather than a live browser run. Current main tracks browser-opened tabs and cleans them on idle/cap sweeps or reset/subagent/cron lifecycle paths, but no ordinary primary-agent task-completion finalizer closes temporary tabs.

## Solution Assessment

No, current main is only a partial mitigation. The best fix is ownership-based cleanup at a maintainer-chosen primary-agent terminal event, not blanket tab closure.
