## Summary

Keep open. Current main still handles browser tab cleanup through idle/cap sweeps and selected lifecycle cleanup, not automatic primary task-completion cleanup; the closed #43666 PR is useful prior art but was not merged, and the remaining default/preserve semantics need maintainer judgment.

## What This Changes

Close temporary browser tabs or windows opened for one-off automation when the task completes, while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Define task-scoped browser tab ownership in the browser plugin and hook successful primary chat/agent completion into best-effort cleanup only for temporary tracked tabs, with tests and docs preserving explicit/user-owned pages.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: direct browser opens are tracked, but primary chat/agent completion does not call browser cleanup, and docs explicitly describe primary tabs as reusable until idle/cap background cleanup.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best fix is not to close all tracked tabs blindly; it needs task-scoped temporary/preserved ownership plus a primary completion hook.
