## Summary

Keep open. Current main has tracked browser tabs, primary-session idle/cap cleanup, and lifecycle cleanup for cron, subagents, session reset, and rollover, but it still lacks an ownership-based finalizer for ordinary primary task completion; LakshmanTurlapati's FSB comment supports the safer run-owned approach rather than a blanket close.

## What This Changes

Request to close browser tabs or windows opened for temporary browser automation when the task completes unless the user explicitly wants to keep them.

## Best Possible Solution

Add run-owned ephemeral browser tab metadata and a primary-run completion cleanup path that closes only tabs created for that run while preserving explicit user-owned, foregrounded, or kept pages, with docs and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows browser opens are tracked by session, primary cleanup is idle/cap based, and ordinary primary task completion has no ownership-aware browser cleanup call.

## Solution Assessment

Unclear as requested. The cleanup goal is sound, but the maintainable solution is ownership-based task cleanup rather than a blanket close-all hook or another broad auto-close switch.
