## Summary

Keep open. Current main still has session-scoped browser tab tracking, idle/cap sweeps, and cron/subagent/session lifecycle cleanup, but ordinary primary `chat.send` completion still does not close task-opened browser tabs; the FSB comment is useful prior art for an ownership-based design rather than a blanket finalizer.

## What This Changes

The issue requests automatic cleanup of browser tabs/windows that were opened temporarily for completed agent browser tasks.

## Best Possible Solution

Define task/run-owned temporary tab metadata in the browser runtime, close only agent-owned ephemeral tabs on successful primary task completion, preserve explicit user-owned or foregrounded tabs, and cover both direct and proxy browser opens with docs and regression tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: browser opens are session-tracked, the primary chat completion finalizer does not call browser cleanup, and docs currently preserve active primary tabs until idle/cap cleanup.

## Solution Assessment

Unclear as stated. The cleanup goal is reasonable, but the safest maintainable solution is ownership-based task cleanup rather than a blanket close-all hook or an unscoped config toggle.
