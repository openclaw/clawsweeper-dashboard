## Summary

Keep open. Current main and v2026.6.10 provide tracked browser tab cleanup via idle/cap sweeps plus reset, cron, and subagent lifecycle hooks, but ordinary primary-agent task completion still lacks ownership-based temporary-tab cleanup and needs a maintainer product decision.

## What This Changes

Request browser tabs/windows opened for temporary automation work to close automatically after task completion while preserving user-owned pages.

## Best Possible Solution

Define a run-owned browser-tab model with explicit preserve/user-owned semantics and hook it into primary-agent terminal cleanup, keeping idle/cap cleanup as a fallback.

## Reproduction Assessment

Yes, source-reproducible. Current source shows browser tabs are tracked and cleaned by idle/cap sweeps or selected lifecycle paths, while ordinary primary embedded-run completion does not call browser lifecycle cleanup.

## Solution Assessment

No, current main is only a partial mitigation. The safer solution is ownership-based task-end cleanup at a maintainer-chosen primary-agent terminal event, not blanket tab closure.
