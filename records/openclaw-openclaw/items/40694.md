## Summary

Keep open. Current main has idle/cap sweeps, reset/session rollover cleanup, and cron/subagent lifecycle cleanup, but ordinary primary task completion still does not close temporary browser tabs; the FSB comment is useful prior art for an ownership-based design rather than a blanket close-all finalizer.

## What This Changes

Request that temporary browser tabs/windows opened for automation close automatically when the task finishes while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Define task/run-owned temporary tab metadata in the browser runtime, close only agent-owned tabs on successful primary task completion, preserve explicit user-owned or foregrounded tabs, and cover both direct and proxy browser opens.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: direct browser opens are tracked under the session, but the primary chat.send finalizer does not invoke browser cleanup, and docs state active primary tabs remain reusable until idle/cap cleanup.

## Solution Assessment

Unclear as stated. The goal is reasonable, but the safest maintainable solution is ownership-based task cleanup rather than a blanket close-all hook or unscoped config toggle.
