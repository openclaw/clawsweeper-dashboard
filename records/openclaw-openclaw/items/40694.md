## Summary

Keep open. Current main and the latest release both have useful browser tab hygiene, but they still intentionally keep primary-agent tabs reusable until idle/cap cleanup rather than closing temporary primary-agent tabs immediately at ordinary task completion.

## What This Changes

The issue requests automatic cleanup of browser tabs or windows opened temporarily for automation tasks after the task completes.

## Best Possible Solution

Add an ownership-based browser lifecycle policy that closes only agent-owned temporary tabs at the chosen completion boundary, preserves user-owned or explicitly kept tabs, and leaves the existing idle/cap cleanup as a fallback for reusable primary sessions.

## Reproduction Assessment

Yes for source-level reproduction: current main tracks and cleans tabs on idle/cap sweeps or lifecycle reset/delete/subagent completion, while docs state primary sessions keep active tabs reusable. I did not run a live browser scenario, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear as an implementation until maintainers choose the product boundary. The ownership-based approach raised in the discussion is safer than blanket closure because it can preserve user-owned or explicitly retained tabs.
