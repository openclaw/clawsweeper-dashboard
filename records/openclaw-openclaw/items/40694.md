## Summary

Keep open. Current main and v2026.6.6 implement tracked browser cleanup for idle/cap sweeps plus session reset, subagent, and cron lifecycle ends, but ordinary primary-agent task completion still preserves active primary tabs; the remaining ownership-based cleanup behavior needs a maintainer product decision.

## What This Changes

Request that browser tabs or windows opened for temporary automation close automatically after task completion while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Extend browser-owned tab tracking with run ownership and explicit preserve/user-owned semantics, then close only unpreserved run-created pages at the maintainer-chosen primary-agent terminal event while keeping idle/cap cleanup as a fallback.

## Reproduction Assessment

Yes, at source level: current main tracks browser-opened tabs and cleans them on idle/cap sweeps or lifecycle reset/subagent/cron paths, but there is no ordinary primary-agent task-completion finalizer. I did not run a live browser scenario.

## Solution Assessment

No for blanket closure; the ownership-based approach in the discussion is safer, but maintainers still need to choose the terminal event and preservation contract before implementation.
