## Summary

Keep open. Current main has useful browser tab cleanup for idle/cap sweeps, reset/rollover, cron, and subagent lifecycle paths, but it still does not implement ownership-based cleanup of temporary primary-agent browser tabs when an ordinary task completes.

## What This Changes

Requests automatic cleanup of temporary browser tabs/windows opened for one-off browser automation after task completion while preserving explicitly user-owned pages.

## Best Possible Solution

Define browser-run ownership and explicit preserve semantics, then close only unpreserved run-created tabs at the chosen primary-agent terminal event while keeping idle/cap cleanup as a fallback.

## Reproduction Assessment

Yes, source-reproducible. Current main tracks browser-opened tabs and closes them via idle/cap sweeps, reset/rollover, cron, or subagent lifecycle paths, but no ordinary primary-agent task terminal cleanup call closes temporary tabs.

## Solution Assessment

No. Current main is only a partial mitigation; the best fix is ownership-based cleanup at a maintainer-chosen primary-agent terminal event, not blanket tab closure.
