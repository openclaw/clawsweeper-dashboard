## Summary

Keep open. Current main has idle/cap browser tab sweeps and lifecycle cleanup for subagent, cron, reset, and delete flows, but it still does not implement the requested primary task-completion cleanup; the recent FSB comment is useful prior art for an ownership-based design rather than a blanket close-all hook.

## What This Changes

Browser-opened temporary tabs/windows should close automatically after task completion unless the user explicitly wants to keep them.

## Best Possible Solution

Define task-scoped temporary browser-tab ownership in the browser plugin/runtime, close only agent-owned tabs on successful primary task completion, preserve explicit user-owned or foregrounded tabs, cover proxy opens, and update docs plus focused regression tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: direct browser opens are tracked under the agent session, but the primary chat.send finalizer does not invoke browser cleanup, and docs state active primary tabs remain reusable until idle/cap cleanup.

## Solution Assessment

Unclear as a direct implementation request. The goal is reasonable, but the safest solution is ownership-based temporary-tab cleanup rather than a blanket close-all hook that changes documented primary tab reuse.
