## Summary

Keep open. Current main and v2026.5.20 still use tracked idle/cap cleanup for primary browser tabs plus lifecycle cleanup for subagent, cron, ACP, and session-reset paths; ordinary primary-agent task-completion cleanup remains a product/UX decision, and the FSB comment usefully points toward an ownership-based policy instead of blanket closing.

## What This Changes

Request browser-opened temporary tabs/windows to close automatically after task completion while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Define an ownership-based primary-task browser cleanup policy for agent-owned ephemeral tabs, preserve user-owned or explicitly kept pages, then implement it through the browser plugin lifecycle with docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens are tracked, but primary cleanup is idle/cap based and the docs explicitly keep active primary tabs reusable; I did not run a live browser because this review is read-only.

## Solution Assessment

No, not as a blind default close. The maintainable path is an ownership-based policy that distinguishes ephemeral agent-owned tabs from user-owned or explicitly preserved tabs before changing primary task-end behavior.
