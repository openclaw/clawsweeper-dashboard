## Summary

Keep open: current main has tracked-tab cleanup, configurable idle/cap sweeps, and lifecycle cleanup for some session endings, but it still does not implement the requested task-completion close behavior for ordinary primary-agent temporary browser tabs.

## What This Changes

Request automatic cleanup of browser tabs/windows opened only for temporary browser automation tasks after the task completes.

## Best Possible Solution

Define ownership-aware browser tab lifecycle: mark tabs opened for a run as agent-owned, close only those tabs when that run or task completes, preserve explicitly labeled/pinned/foregrounded/user-owned tabs, and keep idle/cap cleanup as the long-lived fallback.

## Reproduction Assessment

Yes, from source inspection rather than live runtime: direct browser opens are tracked, but primary-session cleanup is idle/cap based and documented as keeping active tabs reusable instead of closing them at normal task completion.

## Solution Assessment

No for a blanket close-by-default rule. The maintainable solution is the ownership-based design raised in the issue discussion, with explicit preservation semantics before changing primary-session defaults.
