## Summary

Keep open. Current main still cleans browser tabs through lifecycle/session-reset paths and idle/cap sweeps, but primary chat task completion does not automatically close tracked tabs and the docs intentionally describe active primary tabs as reusable.

## What This Changes

Request default cleanup that closes browser tabs/windows opened for temporary browser automation tasks when the task finishes unless the user explicitly preserves them.

## Best Possible Solution

Define task-scoped browser-tab ownership in the browser plugin/SDK and invoke best-effort cleanup on successful primary task completion only for temporary tracked tabs, preserving explicit/user-owned pages and documenting the default.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: direct primary browser opens are tracked, but chat.send completion does not call browser cleanup, and docs say active primary tabs remain reusable until idle/cap cleanup.

## Solution Assessment

Unclear. The requested default is reasonable, but the best fix is not a blanket close-all hook; it needs task-scoped temporary/preserve semantics before wiring primary completion cleanup.
