## Summary

Keep open. Current main and v2026.5.22 clean up tracked browser tabs by idle age, per-session cap, and bounded subagent/cron/session lifecycle hooks, but ordinary primary-agent browser tabs are still intentionally reusable after a task instead of closing at task completion. The FSB context usefully supports an ownership-based design rather than blanket closure.

## What This Changes

Request that browser tabs/windows opened only for temporary agent browser tasks close automatically after the task completes while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Define an ownership-based primary-task browser cleanup policy that distinguishes agent-owned ephemeral tabs from user-owned or explicitly preserved tabs, then implement it through the browser plugin lifecycle with docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens are tracked, but primary cleanup is an idle/cap sweep and the docs state active primary tabs remain reusable; I did not run a live browser because this review is read-only.

## Solution Assessment

No, not as a blanket default close. The safer solution is ownership-based cleanup that preserves user-owned or explicitly kept tabs while closing agent-owned ephemeral tabs at a well-defined primary task boundary.
