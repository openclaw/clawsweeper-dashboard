## Summary

Keep open. Current main has browser tab tracking, subagent/cron/session lifecycle cleanup, and primary-session idle/cap sweeps, but it still does not close primary browser tabs by default when an ordinary task completes, and the docs still describe active primary tabs as reusable until background cleanup.

## What This Changes

Request automatic cleanup of browser tabs/windows opened for temporary automation when the task finishes, while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Define task-scoped browser tab ownership and preserve semantics in the browser plugin/SDK, then close only task-ephemeral tracked tabs after successful primary task completion while documenting the default.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens can be tracked, but primary chat.send completion does not trigger browser cleanup and docs say active primary tabs remain reusable until idle/cap background cleanup.

## Solution Assessment

Unclear. The requested default is reasonable, but a blanket close-all hook is not the best fix because it conflicts with documented reusable primary tabs; task-scoped temporary/preserve semantics are safer.
