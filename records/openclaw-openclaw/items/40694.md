## Summary

Keep open. Current main still has browser tab tracking, idle/cap cleanup, and lifecycle/reset cleanup, but ordinary primary chat task completion does not close browser tabs opened for temporary work; the discussion also points toward ownership-based cleanup rather than a blanket close policy.

## What This Changes

Request default cleanup for browser tabs/windows opened only for temporary automation work after the relevant task finishes.

## Best Possible Solution

Define run-owned ephemeral tab metadata, close only agent-owned temporary tabs on successful primary task completion, preserve explicit user-owned or foregrounded tabs, and cover direct plus proxy browser opens with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens are session-tracked, primary cleanup is idle/cap based, and the primary chat completion path does not call browser cleanup.

## Solution Assessment

Unclear as stated. The cleanup goal is reasonable, but the safest maintainable solution is ownership-based task/run cleanup rather than a blanket close-all hook or unscoped config toggle.
