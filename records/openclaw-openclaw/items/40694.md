## Summary

Keep open. Current main and v2026.5.22 have tracked browser tab cleanup by idle age, per-session cap, and child/session lifecycle hooks, but ordinary primary-agent tabs are still implemented and documented as reusable after a task; LakshmanTurlapati's FSB comment also supports an ownership-based policy rather than blanket closure.

## What This Changes

The issue asks OpenClaw to close temporary browser tabs/windows opened for one-off automation tasks when the task completes, while preserving explicitly retained or user-owned pages.

## Best Possible Solution

Define an ownership-based primary-agent browser cleanup policy that closes agent-owned ephemeral tabs at a clear task boundary while preserving user-owned, foregrounded, pinned, or explicitly kept tabs, then document and test that policy.

## Reproduction Assessment

Yes. Source inspection shows direct browser opens are tracked, while primary cleanup is idle/cap based and current docs state active primary tabs remain reusable; I did not run a live browser in this read-only review.

## Solution Assessment

No, not as a blanket default close. The safer solution is ownership-based cleanup that preserves user-owned or explicitly kept tabs while closing agent-owned ephemeral tabs at a well-defined primary task boundary.
