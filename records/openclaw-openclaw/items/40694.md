## Summary

Keep open. Current main has lifecycle/session-reset cleanup and idle/cap sweeps, but the requested primary task-completion default is still not implemented and would change documented reuse behavior for primary browser tabs.

## What This Changes

Request browser tabs/windows opened for temporary automation tasks to close automatically when the task completes, while preserving explicitly user-owned pages.

## Best Possible Solution

Define task-scoped temporary browser-tab ownership in the browser plugin/SDK, then close only those temporary tracked tabs on successful primary task completion while preserving explicit reusable/user-owned tabs and documenting the default.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: primary browser opens can be tracked, but chat.send completion does not call browser cleanup and docs say active primary tabs stay reusable until idle/cap cleanup.

## Solution Assessment

Unclear. The requested default is reasonable, but a blanket close-all finalizer is not the best path because current behavior intentionally reuses primary tabs; task-scoped temporary/preserve semantics are safer.
