## Summary

Keep open. Current main and v2026.6.6 cover tracked browser cleanup through idle/cap sweeps plus reset, subagent, and cron lifecycle paths, but they still do not close temporary primary-agent browser tabs at ordinary task completion.

## What This Changes

Request automatic cleanup of temporary browser tabs/windows opened for automation tasks while preserving pages the user explicitly wants to keep.

## Best Possible Solution

Define browser-run ownership and explicit preserve semantics, then close only unpreserved run-created pages at the chosen primary-agent terminal event while keeping idle/cap cleanup as fallback.

## Reproduction Assessment

Yes. Source inspection shows current main tracks browser-opened tabs and cleans them on idle/cap sweeps or lifecycle reset/subagent/cron paths, but no ordinary primary-agent task-completion finalizer exists.

## Solution Assessment

No for blanket closure. The ownership-based approach raised in discussion is safer, but maintainers still need to choose the terminal event and preservation contract before implementation.
