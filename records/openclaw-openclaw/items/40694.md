## Summary

Keep open. Current main still implements browser tab cleanup as background idle/cap reclamation plus selected lifecycle cleanup, not automatic task-completion cleanup for ordinary primary chat or agent browser work.

## What This Changes

The issue requests that browser tabs/windows opened only for temporary automation tasks close automatically after the task completes unless explicitly preserved.

## Best Possible Solution

Define task-scoped browser tab ownership in the bundled browser plugin and gateway run lifecycle: mark automation-opened tabs as temporary or preserved, track direct and node/proxy opens consistently, run best-effort cleanup after successful primary chat/agent completion when the tab is temporary, and update focused tests plus browser docs/config.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: open tabs are tracked for direct browser tool calls, but primary chat/agent completion paths do not call browser cleanup, and the docs explicitly describe primary tabs as reusable until idle/cap background cleanup.

## Solution Assessment

Unclear until maintainer product judgment confirms the default. The safest implementation direction is not to close all tracked tabs blindly, but to add task-scoped temporary/preserved ownership and then hook primary run completion into that model.

## Risks / Open Questions

- Changing primary sessions from reusable tabs to close-on-completion is a user-visible product default and could surprise users who rely on active browser state across turns.
- A fix needs explicit preserve/debug/user-owned semantics before broad automatic closing, otherwise task cleanup could close tabs the user expected to keep.
- Node/proxy browser opens are not tracked through the same local path as direct browser opens, so a direct-only fix would leave inconsistent cleanup behavior.
