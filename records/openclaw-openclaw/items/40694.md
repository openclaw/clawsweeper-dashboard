## Summary

Keep open. Current main has browser tab tracking, idle/cap cleanup for primary sessions, and lifecycle cleanup for subagent/cron/reset/delete paths, but ordinary primary task completion still has no ownership-based temporary-tab cleanup hook.

## What This Changes

Browser tabs or windows opened temporarily by agent browser automation should close automatically when the task completes.

## Best Possible Solution

Define task-scoped browser tab ownership in the browser plugin/runtime, close only ephemeral agent-owned tabs on successful primary task completion, preserve explicit user-owned or foregrounded tabs, cover proxy opens, and update docs plus focused regression tests.

## Reproduction Assessment

Yes, at source level: direct browser opens are tracked under a session, but the primary chat.send finalizer does not invoke browser cleanup. The current documented behavior keeps primary tabs reusable until idle/cap cleanup, so this is reproducible as a remaining feature gap rather than a broken existing contract.

## Solution Assessment

Unclear as a direct blanket-close request. The best maintainable solution is ownership-based temporary-tab cleanup, not closing every tracked primary-session tab at task completion.
