## What This Changes

The issue asks OpenClaw to stop silently replaying WhatsApp sends after the caller already received `UNAVAILABLE`, or otherwise report the send as deferred instead of failed.

## Best Possible Solution

Land the linked terminal-preflight failure fix, or an equivalent change that makes caller-visible `UNAVAILABLE` and delivery-recovery state agree, then close this issue from the merged commit.
