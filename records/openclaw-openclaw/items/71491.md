## Summary

Keep open. Current main still lacks the Moonshot/Kimi `reasoning_content` replay backfill for thinking-enabled assistant tool-call messages, and open PR #74846 targets the same wrapper and regression tests.

## What This Changes

The issue requests preserving or synthesizing Moonshot/Kimi `reasoning_content` for replayed assistant tool-call messages after LCM compaction or cross-model fallback.

## Best Possible Solution

Land PR #74846 or an equivalent narrow Moonshot-only payload wrapper fix that preserves existing `reasoning_content`, adds a placeholder only when Moonshot thinking is enabled for replayed assistant tool-call messages, and keeps focused regression tests.

## Reproduction Assessment

Yes. The issue body and follow-up comment give concrete long-session and cross-model fallback paths, and the source-level reproduction is clear: a Moonshot thinking-enabled payload with replayed assistant `tool_calls` and no `reasoning_content` currently passes through the wrapper unchanged.

## Solution Assessment

Yes. A Moonshot-specific final payload-wrapper backfill is the narrowest maintainable OpenClaw-side fix because it runs after context assembly and mirrors the existing DeepSeek V4 compatibility pattern without changing generic replay or LCM policy.
