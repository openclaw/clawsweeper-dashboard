- `impact:session-state`: The failure is caused by replayed or compacted session transcript state losing provider-required reasoning metadata.

## Summary

Keep open. Current main and v2026.5.18 preserve existing `reasoning_content` for Kimi-family replay, but the direct Moonshot thinking wrapper still does not synthesize missing `reasoning_content` for thinking-enabled assistant tool-call history, which matches the newer compaction and cross-model fallback reports.

## What This Changes

The issue asks OpenClaw to preserve or backfill Moonshot/Kimi `reasoning_content` on replayed assistant tool-call messages after compaction or cross-model fallback.

## Best Possible Solution

Add a Moonshot-owned payload normalization path that preserves existing `reasoning_content` and backfills only missing assistant tool-call replay fields when Moonshot thinking is enabled, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. A thinking-enabled direct Moonshot request can still reach `createMoonshotThinkingWrapper` with assistant tool-call history that lacks `reasoning_content`, and that wrapper does not repair the outgoing payload before sending it upstream.

## Solution Assessment

Yes. The narrow maintainable fix is provider-owned Moonshot replay normalization mirroring the DeepSeek/Kimi Coding patterns, not a global transcript rewrite or a new user-facing config option.
