## Summary

Keep open. Current main preserves `reasoning_content` when it survives into Kimi/MiMo replay, and the Anthropic-compatible slice was fixed by the merged Xiaomi/MiMo transport work, but the direct Moonshot thinking wrapper still does not backfill missing `reasoning_content` on replayed assistant tool-call messages.

## What This Changes

The issue reports Kimi/Moonshot thinking-enabled long sessions failing with 400s after replay or compaction drops required `reasoning_content` from assistant tool-call history.

## Best Possible Solution

Add a Moonshot-owned replay normalization path that preserves existing `reasoning_content` and backfills only required assistant replay messages when Moonshot thinking is enabled, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. A thinking-enabled Moonshot replay can still reach `createMoonshotThinkingWrapper` with assistant tool-call history lacking `reasoning_content`, and the wrapper does not repair that payload before sending it upstream.

## Solution Assessment

Yes. The narrow maintainable fix is provider-owned Moonshot payload normalization mirroring the existing DeepSeek and Anthropic-compatible replay patterns, not a global transcript rewrite or new config option.
