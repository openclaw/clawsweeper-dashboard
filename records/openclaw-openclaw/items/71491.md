## Summary

Keep open. Current main still lacks the Moonshot/Kimi `reasoning_content` backfill for thinking-enabled replayed assistant tool-call messages, and an open Moonshot wrapper PR already targets this exact gap.

## What This Changes

Preserve or synthesize `reasoning_content` for Moonshot/Kimi thinking assistant tool-call history after LCM compaction or cross-model replay so long sessions do not fail with provider 400s.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74846 or an equivalent refreshed Moonshot-only final payload backfill with focused regression tests, while tracking LCM reasoning preservation separately.

## Reproduction Assessment

Yes, source-reproducible. A Moonshot thinking-enabled payload with a replayed assistant `tool_calls` message lacking `reasoning_content` reaches the Moonshot wrapper unchanged on current main, matching the reported Kimi 400 path.

## Solution Assessment

Yes. A Moonshot provider-owned final payload patch is the narrow maintainable OpenClaw-side fix because it runs after transcript assembly and avoids pushing Moonshot-specific rules into generic replay code.
