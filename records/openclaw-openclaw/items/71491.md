## Summary

Keep open. Current main still lacks the Moonshot/Kimi `reasoning_content` replay repair described here, while open PR #74846 is already the focused implementation path for the affected wrapper and tests.

## What This Changes

The issue reports that Moonshot/Kimi K2.6 thinking-enabled assistant tool-call messages can lose top-level `reasoning_content` after LCM compaction or cross-model fallback, causing Moonshot 400 responses in long conversations.

## Best Possible Solution

Land PR #74846 or an equivalent narrow Moonshot-only payload wrapper fix that preserves existing `reasoning_content`, adds a placeholder only for thinking-enabled assistant tool-call replay messages, and keeps focused tests for enabled, disabled, and preserve-existing cases.

## Reproduction Assessment

Yes. The issue body and follow-up comment provide concrete long-session and cross-model fallback paths, and current-main source inspection shows the final Moonshot outgoing payload boundary still lacks the reported repair.

## Solution Assessment

Yes. A final Moonshot stream-wrapper repair is the narrowest maintainable OpenClaw-side fix because it runs after context assembly and mirrors the established DeepSeek V4 compatibility pattern without broad replay-policy changes.
