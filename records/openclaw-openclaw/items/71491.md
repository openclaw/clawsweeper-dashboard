## Summary

Keep open. Current main and the latest stable release still preserve existing Moonshot/Kimi `reasoning_content` but do not synthesize the missing assistant tool-call replay field that the newer compaction and cross-model fallback reports hit.

## What This Changes

The issue reports thinking-enabled Kimi/Moonshot sessions failing with 400s after LCM compaction or cross-model fallback because replayed assistant tool-call messages may lack top-level `reasoning_content`.

## Best Possible Solution

Add a Moonshot-owned final payload normalization step that preserves existing `reasoning_content` and backfills only missing assistant tool-call replay fields when Moonshot thinking remains enabled, with focused regression tests.

## Reproduction Assessment

Yes, source-reproducible. A thinking-enabled direct Moonshot request can still reach `createMoonshotThinkingWrapper` with assistant `tool_calls` lacking `reasoning_content`, and current main sends that replay payload without a provider-owned backfill.

## Solution Assessment

Yes. The narrow maintainable solution is a Moonshot provider-owned payload patch mirroring existing DeepSeek and Kimi Coding patterns, not a global transcript rewrite or a new user-facing config option.
