## What This Changes

## Best Possible Solution

Land PR #74846 or an equivalent narrow Moonshot-only payload wrapper fix that preserves existing `reasoning_content`, injects a placeholder only when Moonshot thinking is enabled for replayed assistant tool-call messages, and keeps focused regression tests.

## Reproduction Assessment

Yes. The issue body and comments give concrete long-session and cross-model fallback steps, and current main reproduces the source-level failure because a Moonshot thinking-enabled payload with assistant `tool_calls` and no `reasoning_content` passes through unchanged.

## Solution Assessment

Yes. A Moonshot-specific final payload-wrapper backfill is the narrowest maintainable OpenClaw-side fix because it runs after transcript and compaction assembly and mirrors the existing DeepSeek V4 compatibility pattern without changing generic replay policy.
