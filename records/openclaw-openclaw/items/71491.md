## Summary

Keep open. Current main still lacks the Moonshot/Kimi `reasoning_content` replay backfill for thinking-enabled assistant tool-call messages, and open PR #74846 already targets the exact Moonshot wrapper/test surface for this bug.

## What This Changes

The issue reports that Moonshot Kimi K2.6 still rejects long or compacted thinking-enabled conversations because replayed assistant tool-call messages lose `reasoning_content` after LCM compaction or cross-model fallback.

## Best Possible Solution

Land PR #74846 or an equivalent narrow Moonshot-only final payload-wrapper fix that preserves existing `reasoning_content`, adds a placeholder only when Moonshot thinking is enabled for replayed assistant tool-call messages, and keeps focused enabled, disabled, and preserve-existing tests.

## Reproduction Assessment

Yes. The issue body and follow-up comment give concrete long-session and cross-model fallback paths, and a source-level reproduction is clear: an enabled-thinking Moonshot payload with replayed assistant `tool_calls` and no `reasoning_content` passes through the current wrapper unchanged.

## Solution Assessment

Yes. A Moonshot-specific final payload-wrapper repair is the narrowest maintainable OpenClaw-side solution because it runs after context assembly and mirrors the existing DeepSeek V4 compatibility pattern without changing generic replay or compaction policy.
