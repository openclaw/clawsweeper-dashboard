## Summary

Keep open. Current main fixes an adjacent Anthropic-message replay path, but the direct Moonshot thinking wrapper still does not backfill missing `reasoning_content` on replayed assistant tool-call messages, matching the latest stable-release reports.

## What This Changes

Requests fixing a regression where long thinking-enabled Kimi/Moonshot conversations lose or lack replayed assistant `reasoning_content` after compaction or fallback, causing provider 400 errors.

## Best Possible Solution

Add Moonshot-owned final payload normalization that preserves existing `reasoning_content` and backfills missing assistant replay fields only when Moonshot thinking is enabled, with the broader LCM/provider-generic work tracked separately.

## Reproduction Assessment

Yes, source-reproducible. A thinking-enabled Moonshot replay can reach `createMoonshotThinkingWrapper` with assistant tool-call history lacking `reasoning_content`, and current main does not repair that payload before sending it upstream.

## Solution Assessment

Yes. A narrow provider-owned Moonshot payload normalizer mirrors the existing DeepSeek pattern and avoids a global transcript rewrite or new config option.
