## Summary

Keep open. Current main now fixes the Anthropic-message replay slice via the merged Xiaomi/MiMo transport work, but the direct Moonshot thinking wrapper still does not backfill missing `reasoning_content` on replayed assistant tool-call messages, matching the latest stable-release Kimi reports.

## What This Changes

The issue reports `reasoning_content` 400s for long thinking-enabled Kimi/Moonshot and adjacent reasoning-content providers after compaction or replay.

## Best Possible Solution

Add a Moonshot-owned replay normalization path that preserves existing `reasoning_content` and backfills only the required assistant replay messages when Moonshot thinking is enabled, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. A thinking-enabled Moonshot replay can still reach `createMoonshotThinkingWrapper` with assistant tool-call history that lacks `reasoning_content`, and the wrapper currently does not repair that payload before sending it upstream.

## Solution Assessment

Yes. The best path is a narrow provider-owned Moonshot payload normalization mirroring the existing DeepSeek pattern, not a global transcript rewrite or new config option.
