## Summary

Keep open. Current main still does not add a Moonshot/Kimi `reasoning_content` replay backfill for thinking-enabled assistant tool-call messages, and the open follow-up PR targets the exact wrapper and tests needed for this issue.

## What This Changes

The issue requests preserving or synthesizing Moonshot/Kimi `reasoning_content` for replayed assistant tool-call messages after long-session LCM compaction or cross-model fallback.

## Best Possible Solution

Land PR #74846 or an equivalent narrow Moonshot-only payload wrapper fix that preserves existing `reasoning_content`, adds an empty placeholder only for thinking-enabled replayed assistant tool-call messages, and keeps focused tests for enabled, disabled, and preserve-existing behavior.

## Reproduction Assessment

Yes. The issue body and follow-up comment give concrete long-session and cross-model fallback paths, and the source-level reproduction is clear: an enabled-thinking Moonshot payload with replayed assistant `tool_calls` and no `reasoning_content` currently passes through the wrapper unchanged.

## Solution Assessment

Yes. The open PR's Moonshot-specific final payload wrapper is the narrowest maintainable OpenClaw-side fix because it runs after context assembly and mirrors the existing DeepSeek V4 compatibility pattern without changing generic replay or LCM policy.
