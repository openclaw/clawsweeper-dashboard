## Summary

Keep open. Current `main` preserves `reasoning_content` when it survives into OpenAI-compatible replay, and DeepSeek has a backfill pattern, but Moonshot/Kimi and Anthropic-message replay still have source-backed gaps for histories where compaction or cross-provider replay leaves required `reasoning_content` missing.

## What This Changes

The issue reports provider 400s from missing `reasoning_content` in long thinking-enabled tool-call conversations after compaction or model/provider replay.

## Best Possible Solution

Keep this issue open as the canonical replay failure until maintainers review or replace the linked fixes, with provider-owned payload handling that preserves existing `reasoning_content` and backfills only where the target provider requires it.

## Reproduction Assessment

Yes, source-reproducible. A thinking-enabled Moonshot/Kimi replay payload can still contain assistant tool-call messages without a backfilled `reasoning_content`, and Anthropic-message serialization still fails to restore top-level `reasoning_content` from `thinkingSignature: "reasoning_content"`.

## Solution Assessment

Yes, the best direction is narrow provider/transport replay normalization rather than a global replay-field change or new user setting. The open related PRs should be reviewed carefully because each covers only part of the reported provider matrix.
