## Summary

Keep open. Current main has partial preservation fixes, but the Moonshot/Kimi request path still lacks a provider-gated backfill for already-compacted or legacy assistant tool-call messages where `reasoning_content` is missing.

## What This Changes

The issue asks OpenClaw to preserve or restore `reasoning_content` for thinking-enabled Kimi/Moonshot long-session replay after compaction or fallback.

## Best Possible Solution

Keep the merged preservation work, then add a Moonshot/Kimi-gated request-time backfill for missing `reasoning_content` on replayed assistant tool-call messages when thinking is enabled, without changing global OpenAI-compatible defaults.

## Reproduction Assessment

Yes, source-reproducible. A thinking-enabled Moonshot/Kimi payload can still replay an assistant tool-call message without `reasoning_content` if compaction or older sanitizer output already removed it, and the current Moonshot wrapper does not backfill it.

## Solution Assessment

Yes, the best fix is a narrow provider-owned Moonshot/Kimi payload normalization matching the DeepSeek/Kimi-coding pattern. A global replay-field change would risk providers that intentionally reject `reasoning_content`.
