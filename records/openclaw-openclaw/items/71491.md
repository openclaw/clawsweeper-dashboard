## Summary

Keep open. Current main now preserves Kimi/MiMo `reasoning_content` when it survives in replay history, but the Moonshot thinking wrapper still does not backfill `reasoning_content` onto assistant tool-call messages that arrive at the final payload without it after compaction; the exact fix PR for that gap was closed unmerged.

## What This Changes

The issue asks OpenClaw to preserve or restore Moonshot/Kimi `reasoning_content` on long, tool-heavy conversations after LCM compaction.

## Best Possible Solution

Add a Moonshot provider-owned final payload patch that preserves existing `reasoning_content` and backfills a safe empty string only for thinking-enabled assistant tool-call replay; keep LCM-side preservation tracked separately.

## Reproduction Assessment

Yes, source-reproducible. A Moonshot thinking-enabled payload with an assistant `tool_calls` message lacking `reasoning_content` reaches the current Moonshot wrapper unchanged, matching the reported provider rejection path after compaction strips the field.

## Solution Assessment

Yes. A Moonshot-only final payload backfill is the narrowest maintainable OpenClaw-side repair because it runs after transcript assembly and avoids broad generic replay changes or a new user config knob.
