## Summary

Keep open. Current main has landed adjacent reasoning_content fixes for Anthropic/Xiaomi and Kimi Coding paths, but the direct Moonshot thinking wrapper still only normalizes thinking/tool_choice/keep and does not backfill missing assistant tool-call reasoning_content for the Kimi long-session failure reported here.

## What This Changes

The issue asks OpenClaw to preserve or restore reasoning_content for Moonshot/Kimi thinking-enabled long conversations after LCM compaction or cross-model replay.

## Best Possible Solution

Add a narrow Moonshot provider-owned payload backfill for thinking-enabled assistant tool-call replay, preserve existing reasoning_content, and keep LCM preservation plus other providers in their own lanes.

## Reproduction Assessment

Yes, source-reproducible. A Moonshot/Kimi thinking-enabled replay payload can still contain assistant tool-call messages without reasoning_content because createMoonshotThinkingWrapper does not inspect payloadObj.messages, matching the reported 400s after compaction or fallback.

## Solution Assessment

Yes. The best fix is a provider-scoped Moonshot final-payload normalization mirroring the established DeepSeek/Kimi Coding pattern, not a new config knob or global transcript rewrite.
