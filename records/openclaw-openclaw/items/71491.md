## Summary

Keep open. Current main still lacks the Moonshot/Kimi `reasoning_content` replay backfill requested here, and open PR #74846 is already the focused implementation path for the exact missing behavior.

## What This Changes

The issue requests preserving or restoring Moonshot/Kimi `reasoning_content` on replayed assistant tool-call messages after long-session LCM compaction or cross-model fallback.

## Best Possible Solution

Land PR #74846 or an equivalent narrow Moonshot-only stream-wrapper fix that preserves existing `reasoning_content`, injects a placeholder only when Moonshot thinking is enabled for replayed assistant tool-call messages, and keeps focused tests for enabled, disabled, and preserve-existing behavior.

## Reproduction Assessment

Yes. The issue body and follow-up comment give concrete long-session and cross-model fallback paths, and current-main source inspection shows the final Moonshot payload wrapper still lacks the reported repair.

## Solution Assessment

Yes. A final Moonshot payload-wrapper repair is the narrowest maintainable OpenClaw-side fix because it runs after context assembly and mirrors the existing DeepSeek V4 compatibility approach without broad replay-policy changes.
