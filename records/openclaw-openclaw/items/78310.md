## Summary

Keep open: this PR targets a valid reply-delivery regression and the shared core seam is the right area, but it has the protected `maintainer` label, only test/mock behavior proof, and a concrete Feishu hydration lookup should be moved behind access gates before merge.

## What This Changes

The PR changes group/channel reply defaults back to automatic when unset, stops doctor from injecting tool-only room reply policy, updates docs/tests, and hydrates missing Feishu native topic thread IDs before routing.

## Best Possible Solution

Land the shared default restoration only after Feishu topic hydration respects group/sender gates and the PR has after-fix real behavior proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes by source inspection: current main resolves unset group/channel replies to `message_tool_only` and doctor persists that policy into configured-channel configs. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No as-is: the core default change is the narrow shared seam if maintainers choose the default restoration, but the Feishu hydration lookup should be gated before merge and the PR still needs real behavior proof beyond tests.
