## What This Changes

Prevent the gateway stuck-session detector from emitting `queued_work_without_active_run` when an active reply run exists for the same session key.

## Best Possible Solution

Land the narrow lifecycle-sync fix in https://github.com/openclaw/openclaw/pull/79695 or an equivalent change that shares active reply facts with diagnostic attention before `session.stuck` is emitted.
