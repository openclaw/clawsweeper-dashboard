## Summary

Keep open: the patch addresses a real source-level unhandled-rejection hardening gap and has sufficient terminal proof, but the shared handler's new expiry catch hard-codes the exec approval label for plugin approval expiry failures.

## What This Changes

The PR adds a `.catch()` logger to approval expiry fire-and-forget delivery and adds exec approval forwarder tests covering expiry cleanup and logging.

## Best Possible Solution

Merge after the expiry catch uses `params.strategy.kind` in the log message while preserving delete-before-delivery cleanup and the focused regression coverage.

## Reproduction Assessment

No high-confidence natural runtime reproduction was established. Source inspection shows the expiry fire-and-forget IIFE lacks an outer catch on current main, while ordinary per-target delivery errors are already caught inside `deliverToTargets`.

## Solution Assessment

Yes, adding an outer catch while keeping `pending.delete(requestId)` before delivery is the narrow maintainable fix. The catch log should use `params.strategy.kind` because the handler is shared by exec and plugin approvals.
