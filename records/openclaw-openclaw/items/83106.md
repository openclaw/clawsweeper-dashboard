## Summary

This PR should stay open but is not mergeable as written: current main already catches and logs ordinary delivery failures inside `deliverToTargets`, while the patch delays expiry cleanup behind channel I/O and the supplied proof is only a standalone simulation.

## What This Changes

The PR moves `pending.delete(requestId)` in the approval expiry timeout from before to after `deliverToTargets()` and adds an outer catch that logs and deletes on rejection.

## Best Possible Solution

Keep expiry cleanup independent of delivery; if a real uncaught expiry-notification failure exists, add focused logging and regression coverage without retaining expired approvals past timeout.

## Reproduction Assessment

No. Current-main source shows ordinary channel delivery failures resolve to failed or partial-failed statuses and are logged by `deliverToTargets`; the PR proof is a standalone Node `Map` simulation, not an OpenClaw runtime failure.

## Solution Assessment

No. Moving cleanup after channel I/O is not the safest fix; cleanup should remain before the await, with any extra logging or tests tied to a proven uncaught path.
