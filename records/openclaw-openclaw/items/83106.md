## What This Changes

The PR moves exec-approval expiry cleanup to run after expiry notification delivery and adds regression tests for expiry delivery ordering and delivery-failure logging.

## Best Possible Solution

Keep expiry finalization independent from channel notification delivery; if an uncaught expiry-notification failure is proven, add focused logging or coordinate it with the broader delivery-failure PR without keeping expired forwarder state pending.

## Reproduction Assessment

No high-confidence live reproduction exists for the claimed uncaught delivery-failure bug; current source already catches ordinary per-target failures. The PR-introduced late pending-state behavior is source-reproducible from the diff because cleanup moves after an awaited delivery call.

## Solution Assessment

No. The proposed ordering change is not the best fix because it makes forwarder expiry cleanup depend on channel I/O; the safer path is targeted logging for a proven uncaught path while preserving timeout cleanup first.
