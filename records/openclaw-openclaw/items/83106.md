## What This Changes

The PR adds a catch/log handler to the detached approval-expiry delivery IIFE and adds focused expiry cleanup/error-handling tests.

## Best Possible Solution

Merge the reliability hardening after changing the expiry catch log to use params.strategy.kind, while keeping pending deletion before delivery and the focused regression coverage.

## Reproduction Assessment

No high-confidence natural runtime reproduction was established. Source inspection shows current main starts expiry delivery in a detached async IIFE without an outer catch, while ordinary delivery failures are already caught inside deliverToTargets.

## Solution Assessment

Yes, adding an outer catch while preserving pending.delete before delivery is the narrow maintainable fix. The catch should use params.strategy.kind because the handler is shared by exec and plugin approvals.
