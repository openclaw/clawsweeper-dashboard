The PR adds a catch handler to the exec/plugin approval expiry fire-and-forget delivery promise and adds forwarder tests for expiry cleanup and delivery error handling.

## Best Possible Solution

Merge the narrow catch after the expiry log uses the shared approval strategy kind and proof or coverage forces an unexpected rejection to reach the new outer catch.

## Solution Assessment

Yes for the core shape: adding `.catch()` while keeping `pending.delete(requestId)` before delivery is the narrow maintainable fix. The PR should make the log strategy-specific and prove or test the actual outer-catch path.
