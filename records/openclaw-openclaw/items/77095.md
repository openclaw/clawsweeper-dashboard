## Summary

Keep open. Current main exports the named restart-intent payload function, but that only rules out a simple missing-symbol close; the crash trigger and the schema-incompatible post-downgrade recovery path remain unresolved, and adjacent #74424/#73812 do not cover this crash/downgrade failure.

## What This Changes

The issue reports a 2026.5.2 macOS gateway crash in restart-intent payload handling followed by an automatic downgrade to an older binary that cannot boot against newer config metadata.

## Best Possible Solution

Long-running gateways should consume restart-intent payloads from a stable runtime boundary without stale-export crashes, and update/supervisor recovery should never leave an older binary running against a newer `meta.lastTouchedVersion` config.

## Reproduction Assessment

No full high-confidence reproduction yet. Source inspection supports the post-downgrade unbootable state, but the 12-hour crash and the exact automatic downgrade trigger were not reproduced in this read-only review.

## Solution Assessment

Unclear because there is no proposed patch and the failure crosses restart lifecycle loading plus update/supervisor recovery policy. The narrow maintainable direction is to identify the packaged/runtime skew first, then prevent schema-incompatible automatic downgrade recovery.
