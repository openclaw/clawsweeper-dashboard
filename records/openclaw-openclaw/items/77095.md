Keep open. Current main has the named restart-intent payload function exported and covered, but that does not prove the 12-hour packaged/runtime crash or the automatic downgrade path is fixed; source still confirms an older binary will refuse service startup against a newer `meta.lastTouchedVersion` config.

## What This Changes

The issue reports a 2026.5.2 macOS Gateway crash in restart-intent handling followed by recovery downgrading the global package to 2026.4.29 against a newer config.

## Best Possible Solution

Diagnose the packaged/runtime skew from the stability bundle and service/update logs, then land a targeted lifecycle/update recovery fix so long-running gateways cannot lazy-load stale restart runtime and recovery refuses or safely rolls back schema-incompatible downgrades.

## Solution Assessment

Unclear because there is no proposed patch and the failure spans lifecycle lazy loading plus update/supervisor recovery policy. The narrow maintainable path is to identify the packaged/runtime skew first, then harden recovery rather than treating related continuation issues as sufficient.
