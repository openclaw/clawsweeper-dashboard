## Summary

Keep open. Current main exports the named restart-intent payload function and has focused coverage for the intent path, but the reported long-uptime crash plus automatic downgrade trigger is not explained or fixed by source inspection, and the schema-incompatible recovery state still matches current guard behavior.

## What This Changes

The issue reports a 2026.5.2 macOS gateway crash in the restart-intent path followed by recovery downgrading the installed binary below the config schema version.

## Best Possible Solution

Keep this open until the stability bundle and service/update logs identify the trigger, then land a targeted fix that prevents stale lifecycle exports from crashing long-lived gateways and prevents recovery from starting an older binary against a newer config schema.

## Reproduction Assessment

No full high-confidence reproduction yet. Source inspection supports the schema-incompatible post-downgrade failure state, but the long-uptime `consumeGatewayRestartIntentPayloadSync is not a function` crash and automatic downgrade trigger are not reproduced on current main.

## Solution Assessment

Unclear because there is no patch and the failure spans lifecycle module loading plus update/supervisor recovery policy. The narrow maintainable path is to diagnose the packaged/runtime skew first, then harden downgrade recovery rather than treating #74424 or #73812 as sufficient.
