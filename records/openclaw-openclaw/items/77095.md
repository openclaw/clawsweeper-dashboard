## Summary

Keep open. Current main has the named restart-intent payload function exported through the gateway lifecycle runtime and covered by tests, so this is not a safe implemented-on-main close; however, source still confirms the reported schema-incompatible post-downgrade boot loop shape, and the actual crash/downgrade trigger needs maintainer diagnosis from the stability bundle and service/update logs.

## What This Changes

The issue reports a 2026.5.2 macOS gateway crash in the restart-intent payload path followed by an automatic downgrade to 2026.4.29 that leaves the newer config unbootable.

## Best Possible Solution

Keep this open and land a targeted gateway lifecycle/update recovery fix once the logs identify the trigger: long-running gateways should not crash on stale lifecycle runtime exports, and automatic recovery should refuse or safely roll back before starting an older binary against a newer `meta.lastTouchedVersion` config.

## Reproduction Assessment

No full high-confidence reproduction yet. Source inspection supports the post-downgrade unbootable state, but the 12-hour `consumeGatewayRestartIntentPayloadSync is not a function` crash and the automatic downgrade trigger are not reproduced on current main.

## Solution Assessment

Unclear because there is no proposed patch and the failure spans restart lifecycle loading plus update/supervisor recovery policy. The narrow maintainable direction is to identify the packaged/runtime skew first, then prevent schema-incompatible automatic downgrades rather than treating #74424/#73812 as sufficient.
