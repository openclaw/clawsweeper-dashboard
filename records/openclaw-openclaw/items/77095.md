## Summary

Keep open. Current main has the named restart-intent payload function exported and covered, so this is not a simple missing-symbol close, while source still supports the reported older-binary/newer-config boot loop once a downgrade or stale service wrapper happens.

## What This Changes

The issue reports a macOS Gateway crash in OpenClaw 2026.5.2 followed by recovery starting OpenClaw 2026.4.29 against a config last touched by 2026.5.2.

## Best Possible Solution

Keep the issue open until maintainers identify whether the root is packaged/runtime skew, stale launchd service state, or update recovery, then prevent long-running gateways from crashing on restart intent consumption and refuse or safely roll back schema-incompatible recovery.

## Reproduction Assessment

No full high-confidence reproduction yet. Source inspection supports the post-downgrade unbootable state, but the reported 12-hour TypeError and downgrade trigger have not been reproduced on current main.

## Solution Assessment

Unclear because there is no proposed patch and the failure crosses restart lifecycle loading plus update/supervisor recovery policy. The narrow path is to diagnose the packaged/runtime skew first, then harden recovery so an older binary is not started against newer config.
