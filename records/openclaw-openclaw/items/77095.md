## Summary

Keep open. Current main has the named restart-intent function in source and covers the normal signal path, but it does not prove the reported long-uptime packaged/runtime crash or the automatic downgrade-to-unbootable recovery path is fixed.

## What This Changes

The issue reports a 2026.5.2 macOS gateway crash around 12 hours uptime from `consumeGatewayRestartIntentPayloadSync is not a function`, followed by downgrade to 2026.4.29 and refusal to boot against newer config metadata.

## Best Possible Solution

Keep this open until maintainers identify the runtime skew or supervisor/update trigger, then make crash recovery either restart the matching newer install or refuse/roll back safely before launchd runs an older binary against newer config.

## Reproduction Assessment

No, not end to end. Source inspection reproduces the post-downgrade newer-config refusal shape, but the long-uptime `is not a function` crash and the automatic downgrade trigger were not reproduced.

## Solution Assessment

Unclear because there is no proposed patch and the failure spans lifecycle lazy-loading plus update/supervisor recovery policy. The narrow path is to first prove the packaged/runtime skew, then harden recovery so automatic downgrade cannot leave the service unbootable.
