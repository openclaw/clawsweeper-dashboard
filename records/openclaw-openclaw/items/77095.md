The issue asks OpenClaw to stop crashing on `consumeGatewayRestartIntentPayloadSync` during long-running gateway restarts and to prevent automatic recovery from downgrading into a newer-config boot loop.

## Best Possible Solution

Diagnose the packaged/runtime skew from the stability bundle and update/service logs, then land a targeted lifecycle/update recovery fix so long-running gateways do not lazy-load stale exports and recovery never starts an older binary against a newer config.
