## Summary

Keep this PR open for maintainer review: it is a plausible Bedrock plugin/config feature, but the proposed branch still has a schema/type mismatch and exposes docs/setup-visible auth modes before the current OpenClaw Bedrock runtime and pinned pi-ai package consume that contract.

## What This Changes

The PR adds Bedrock auth config types and normalization tests, expands the amazon-bedrock plugin manifest/UI hints, re-exports setup types, updates Bedrock docs/changelog, and bumps the plugin package version.

## Best Possible Solution

Land a coordinated Bedrock contract where pi-ai runtime support, OpenClaw manifest schema/types, setup UX, docs, tests, and real behavior proof all describe the same supported auth modes and fields.

## Reproduction Assessment

Yes for the patch defects. Source inspection at PR head shows the normalizer emits fields omitted from the strict manifest schema, and current-main/runtime dependency inspection shows the documented auth modes would not be consumed by the pinned Bedrock runtime.

## Solution Assessment

No. The Bedrock plugin is the right ownership boundary, but this branch should not merge until schema parity is fixed and maintainers decide whether to expose the config/docs before a pi-ai runtime update is available.

  - confidence: 0.9
