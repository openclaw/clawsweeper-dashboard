  - body: The new UI hints expose API key, access key, secret key, and session token fields without `sensitive: true`. Manifest docs and setup code use that flag for secret handling, so these AWS credentials can be shown or prompted as ordinary text.
  - confidence: 0.9

  - body: `BedrockSetupOptions` aliases `BedrockAuthConfig`, and `normalizeBedrockAuthConfig` always returns `awsBedrockCustomSelected`, but the strict manifest omits `awsBedrockCustomSelected` and `awsBedrockCustomModelBaseId`. A manifest-validated round trip rejects the normalized object.
  - confidence: 0.93
- **[P2] Do not document auth modes before runtime support:** `docs/providers/bedrock.md:355-365`
  - body: The docs present `apikey`, `profile`, `credentials`, and `default` as selectable plugin auth modes, but the pinned pi-ai Bedrock runtime does not consume `awsAuthentication`, static credentials, endpoint, prompt-cache, or 1M-context fields, and the referenced upstream runtime PR is closed unmerged.
  - confidence: 0.9
- **[P3] Preserve explicit empty regions during normalization:** `extensions/amazon-bedrock/bedrock-auth-config.ts:91`
  - body: `awsRegion` uses `||` while the other defaults use `??`, so a caller-provided empty string is replaced with `us-east-1` instead of preserving the supplied value. Use nullish coalescing for the same defaulting contract as the rest of the helper.
  - confidence: 0.72
