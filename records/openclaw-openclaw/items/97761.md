## Summary

Keep open: current main and the latest release still leave structured tool-call argument payloads outside the provider `textTransforms.output` boundary, so the reported masked-token behavior remains source-reproducible and narrow enough for a focused fix.

## What This Changes

The issue requests applying provider output text replacements to streamed and final tool-call argument payloads before OpenClaw consumes or executes those tool calls.

## Best Possible Solution

Extend the existing text-transform helper to rewrite tool-call delta strings and recursively rewrite string values inside tool-call argument objects on streamed events, partial/message snapshots, and final stream results, with focused regression coverage and no new config or SDK hook.

## Reproduction Assessment

Yes, at source level: current main's text-transform wrapper does not transform tool-call delta/end argument payloads, while those arguments flow into tool execution. I did not run a live DeepSeek/chat-space reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow existing-boundary repair, not a new plugin API or config option. The implementation should adapt to current `delta` and `toolCall.arguments` types and add focused tests.
