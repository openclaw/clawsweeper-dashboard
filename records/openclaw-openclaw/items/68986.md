## Summary

Keep open. The PR addresses a real sanitizer gap and includes live Discord proof, but the current head still misses delivery-mode sanitization when a completed stream item rolls over and the PR is still draft against a newer main.

## What This Changes

Adds shared delivery-mode visible assistant text normalization for leaked `<channel|>` delimiters and repeated suffixes, wires it into embedded delivery paths, and adds regression coverage.

## Best Possible Solution

Land a rebased branch that keeps shared delivery-mode normalization, preserves current main stream-item handling, marks every completed final flush including rollover as final delivery, and validates the focused agent/text tests plus build.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, `emitBlockChunk` uses delivery-mode cleanup only when `finalDelivery` is true, while stream-item rollover flushes without that flag before resetting state.

## Solution Assessment

No, not yet. Shared delivery sanitization is the right direction, but the rollover finalization path must opt into delivery mode and the branch should be rebased before it is the narrowest maintainable fix.
