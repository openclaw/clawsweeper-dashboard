## What This Changes

The PR adds shared delivery-mode assistant visible-text normalization for leaked `<channel|>` delimiters and repeated output suffixes, wires it into agent stream/block delivery paths, and adds regression tests.

## Best Possible Solution

Land a rebased branch that keeps normalization in shared delivery sanitization, marks stream-item rollover flushes as final delivery, preserves current main stream-item behavior, and validates focused agent/text tests plus build.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, `emitBlockChunk` only runs delivery-mode cleanup when `finalDelivery` is passed, and stream-item rollover flushes the completed previous item without that flag before resetting state.

## Solution Assessment

No, not yet. A shared delivery sanitizer is the right direction, but completed delivery flush paths, including stream-item rollover, need to opt into delivery mode before this is the narrowest maintainable fix.
