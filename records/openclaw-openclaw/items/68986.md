## Summary

Keep open. Current main still lacks the requested `<channel|>` and repeated-suffix cleanup, and PR head still has a blocking delivery-path gap: stream-item rollover finalizes a completed item without delivery-mode sanitization. The PR has usable real behavior proof and no concrete security concern, but it is draft/conflicting and needs the source fix before merge.

## What This Changes

The branch adds shared visible assistant output normalization for leaked `<channel|>` delimiters and repeated visible suffixes, wires delivery/profile sanitization into embedded assistant block-reply paths, and adds regression tests.

## Best Possible Solution

Land a rebased patch that applies delivery-mode normalization to every final flush, including stream-item rollover, while preserving literal markers, intentional repeats, and indentation.

## Reproduction Assessment

Yes. Source inspection of PR head shows `finalDelivery` controls delivery-mode cleanup, while stream-item rollover flushes the previous item without that flag and the PR-head rollover test expects that call shape.

## Solution Assessment

No, not yet. Shared final-delivery normalization is the right direction, but the patch needs rollover finalization coverage and a rebase against the current final-flush contract before merge.
