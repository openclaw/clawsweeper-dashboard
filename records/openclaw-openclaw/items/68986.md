## Summary

Keep open: current main does not already implement this cleanup, and the draft branch still has a source-proven P2 path where a completed stream item can skip delivery-mode sanitization.

## What This Changes

The PR adds shared visible assistant-output normalization for leaked `<channel|>` scaffolding and repeated visible suffixes, applies it to delivery paths, adjusts chunk flushing, and adds sanitizer/agent/reply regression tests.

## Best Possible Solution

Land a rebased branch that keeps shared delivery-mode normalization and marks every completed final flush, including stream-item rollover, as final delivery with regression coverage.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, stream-item rollover finalizes a previous assistant item by flushing without `finalDelivery: true`, while `emitBlockChunk` only runs delivery-mode normalization when that flag is set.

## Solution Assessment

No, not yet. Shared delivery-time normalization is the right fix direction, but the rollover finalization path needs the same final-delivery flag and regression coverage before merge.
