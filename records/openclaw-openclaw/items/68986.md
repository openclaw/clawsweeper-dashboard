## Summary

Keep open: the PR addresses a real visible-output leak that current main still does not cover, but the current draft has a P2 correctness gap where stream-item rollover can bypass delivery-mode cleanup.

## What This Changes

The PR adds delivery-mode visible assistant text normalization for leaked Gemma `<channel|>` delimiters and repeated suffixes, wires it into embedded agent delivery paths, and adds regression tests.

## Best Possible Solution

Land a rebased branch that keeps shared delivery-mode normalization, makes every completed final flush including stream-item rollover use delivery mode, and reruns focused agent/text tests plus build/check.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, `emitBlockChunk` runs delivery-mode cleanup only when `finalDelivery` is true, while stream-item rollover flushes without that flag before resetting state.

## Solution Assessment

No, not yet. Shared delivery sanitization is the right direction, but the rollover finalization path must opt into delivery mode before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Mark rollover flushes as final delivery:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:382`
  - body: After this PR, `emitBlockChunk` only runs delivery-mode normalization when `finalDelivery` is true. Pass `finalDelivery: true` here and add rollover coverage.
