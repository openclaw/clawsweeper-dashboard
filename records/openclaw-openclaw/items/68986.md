## Summary

Keep open: the requested sanitizer work is not on current main, and the PR still has a source-reproducible delivery gap where stream-item rollover flushes a completed answer without the new final-delivery sanitizer mode.

## What This Changes

The branch adds delivery-mode assistant visible-text normalization for leaked `<channel|>` delimiters and repeated suffixes, wires it into embedded agent delivery paths, and adds regression tests.

## Best Possible Solution

Rebase and land a version that applies shared delivery-mode normalization to every final flush, including stream-item rollover, with focused coverage for literal marker text, intentional repeats, and rollover delivery.

## Reproduction Assessment

Yes. On PR head, `finalDelivery` is the switch for delivery-mode cleanup, and the stream-item rollover path finalizes the previous assistant item without passing that flag.

## Solution Assessment

No, not yet. Shared delivery-time normalization is the right direction, but the rollover finalization path needs the same final-delivery flag and regression coverage before merge.
