## Summary

Keep open: this is a useful bug-fix PR, but the current head still has a delivery finalization gap that can let buffered final text skip the new delivery-mode sanitizer.

## What This Changes

The branch adds shared delivery-time assistant visible-text normalization for leaked `<channel|>` scaffolding and repeated visible suffixes, plus agent/reply regression tests.

## Best Possible Solution

Land a rebased branch that keeps shared delivery-mode normalization and marks every completed final flush, including stream-item rollover, as final delivery with regression coverage.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, `emitBlockChunk` runs delivery-profile cleanup only when `finalDelivery` is true, while stream-item rollover flushes without that flag before resetting state.

## Solution Assessment

No, not yet. Shared delivery sanitization is the right direction, but the rollover finalization path must opt into delivery mode before this branch is merge-ready.
