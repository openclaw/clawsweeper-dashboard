## Summary

Keep open: current main still lacks the central `<channel|>` and repeated-suffix delivery cleanup, and the PR has a source-reproducible rollover finalization bug that should block merge until fixed.

## What This Changes

This PR adds delivery-mode assistant visible-text normalization for leaked `<channel|>` delimiters and repeated suffixes, wires it into embedded agent delivery paths, and adds focused regressions.

## Best Possible Solution

Land a rebased branch that keeps shared delivery-mode normalization, passes final-delivery mode through every completed flush path including stream-item rollover, and keeps focused agent/text regressions green.

## Reproduction Assessment

Yes. The bug is source-reproducible on the PR head because `emitBlockChunk` only applies delivery cleanup when `finalDelivery` is set, while stream-item rollover flushes without that flag before resetting state.

## Solution Assessment

No, not yet. Shared delivery sanitization is the right direction, but the implementation must mark rollover finalization as delivery mode before it is the narrowest maintainable fix.
