The PR adds shared visible assistant output normalization before delivery, covering Gemma `<channel|>` delimiter cleanup, repeated suffix collapse, delivery-path wiring, and regression tests.

## Best Possible Solution

Land a rebased branch that keeps shared delivery-mode normalization, marks every completed final flush including stream-item rollover as final delivery, preserves current main flush behavior, and validates focused agent/text tests plus build.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, `emitBlockChunk` runs delivery-mode cleanup only when `finalDelivery` is true, while stream-item rollover flushes without that flag before resetting state.

## Solution Assessment

No, not yet. Shared delivery sanitization is the right direction, but this implementation needs the rollover finalization path to opt into delivery mode and the draft branch needs rebasing against current main.
