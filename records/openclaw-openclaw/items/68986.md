## Summary

Keep open. The PR addresses a real current-main sanitizer gap, but the latest head still has a source-reproducible P2 delivery bug where a completed stream item can flush without the delivery sanitizer before state reset.

## What This Changes

The PR adds shared delivery-mode assistant visible-text normalization for leaked `<channel|>` delimiters and repeated output suffixes, wires it into embedded agent delivery paths, and adds regression tests.

## Best Possible Solution

Land a rebased branch that keeps shared delivery-mode normalization, marks every completed final flush including stream-item rollover as final delivery, and validates the focused agent/text tests plus build.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, `emitBlockChunk` runs delivery-mode cleanup only when `finalDelivery` is true, while stream-item rollover flushes without that flag before resetting state.

## Solution Assessment

No, not yet. Shared delivery sanitization is the right direction, but the implementation needs the rollover finalization path to opt into delivery mode before it is the narrowest maintainable fix.
