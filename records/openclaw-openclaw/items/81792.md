## What This Changes

Adds `gateway.auth.tokenScopes` and `gateway.auth.allowPrivilegedTokenScopes` config/schema/docs/runtime handling for device-less token-auth WebSocket clients, including static scope application, privileged-scope guards, shared-auth session rotation, and tests.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible. Current main’s default-token suite asserts device-less shared-token auth returns `auth.scopes=[]` even when requesting `operator.read`, and core method descriptors require `operator.read` for the reported subscribe path.

## Solution Assessment

Yes, conditionally. A configured static token-scope grant with a privileged non-loopback guard is more maintainable than trusting client-supplied `params.scopes`, but it needs security/product approval, real runtime proof, and branch cleanup.
