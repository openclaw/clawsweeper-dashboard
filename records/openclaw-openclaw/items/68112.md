  - body: Current main passes `deferAgentTurnJobs: true` into `runMissedJobs` and has a regression test proving overdue isolated agent-turn jobs are deferred during gateway startup.   - confidence: 0.9
- **[P1] Keep overflow deferral protection:** `src/cron/service/ops.ts:203`
  - body: Current main carries the set of startup-deferred jobs into `recomputeNextRunsForMaintenance` so overflow catch-up slots are not repaired back to their natural schedule. This branch falls back to `recomputeNextRuns`, which can clobber those deferred catch-up slots and drop missed runs for a full period.
  - confidence: 0.88
- **[P2] Defer zombie-cleared jobs until a later tick:** `src/cron/service/timer.ts:758-763`
  - body: The new timer path clears `runningAtMs` and then immediately calls `collectRunnableJobs` on the same store. A presumed-dead job with a past-due `nextRunAtMs` can therefore execute in the same tick, not on a later recovery pass, risking duplicate scheduled sends or agent turns while the prior execution is only presumed dead.
  - confidence: 0.87
