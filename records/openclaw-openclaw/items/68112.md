Keep this PR open: it targets a real remaining cron startup resilience bug, but the current head is dirty against main and its guarded `runMissedJobs()` call predates the current startup deferral behavior for isolated agent-turn jobs. The PR wraps cron startup missed-job catch-up in error handling, repairs partial running-marker/one-shot state, adds runtime zombie running-marker cleanup, and adds cron resilience tests.

## Best Possible Solution

Land a rebased or replacement startup resilience fix that guarantees the scheduler timer is armed after startup catch-up failures while preserving `deferAgentTurnJobs: true`, interrupted-run emission, and current maintenance recompute semantics; keep the separate runtime watchdog work in #73355 independent.
