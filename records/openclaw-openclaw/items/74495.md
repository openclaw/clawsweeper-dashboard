## Summary

Keep open. The PR correctly propagates the selected-media no-provider decision, but it still misses the linked directory/no-attachment reproduction because the new helper only recognizes skipped decisions with selected attachments and zero attempts.

## What This Changes

The PR updates infer media CLI/runtime handling so missing image/audio media-understanding provider configuration can be reported instead of generic empty transcript/description errors.

## Best Possible Solution

Land one canonical fix for #73569 that detects missing image/audio provider configuration before attachment filtering, while preserving empty-result errors when a provider actually ran.

## Reproduction Assessment

Yes. Issue #73569 gives deterministic no-provider CLI commands, and current source shows directory or unsupported inputs can become `no-attachment` before provider resolution while this PR only handles `skipped` decisions with selected attachments.

## Solution Assessment

No. Propagating the runtime decision is useful, but the fix needs to check provider availability before media selection or carry an explicit no-provider signal that survives `no-attachment` outcomes.

## Review Findings
