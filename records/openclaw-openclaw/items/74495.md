## What This Changes

This PR threads media-understanding runner decisions through file runtime results, maps skipped/no-attempt image/audio infer results to missing-provider CLI errors, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land a current, maintainer-reviewed version of this patch so no-provider image/audio infer paths produce actionable configuration errors while provider-ran-but-empty-result paths keep their existing empty description/transcript errors; #73569 should close only after merge.

## Reproduction Assessment

## Solution Assessment

Yes. Reusing the runner's existing skipped/no-attempt decision is narrower and more maintainable than re-resolving providers in the CLI, and the PR keeps the existing fallback errors for cases where a provider actually ran but returned no text.
