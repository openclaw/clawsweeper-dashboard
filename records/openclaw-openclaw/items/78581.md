## Summary

Keep open: current main fixed the original text-classifier miss and improved failed delivery-status plumbing, but announce retry still uses fixed backoff and does not consume structured provider cooldown timing.

## What This Changes

The issue asks subagent announce delivery to recognize fallback provider cooldowns as transient, retry with cooldown-aware timing, log retries, and avoid marking exhausted delivery as delivered.

## Best Possible Solution

Add bounded cooldown-aware retry timing at the announce-delivery boundary, preserving the landed text classifier and failed delivery-status behavior, with regression coverage for cooldown retry timing and exhausted delivery status.

## Reproduction Assessment

Yes, by source inspection: current main still uses fixed announce retry delays while structured cooldown metadata exists upstream. I did not run a live provider-overload reproduction.

## Solution Assessment

Yes for the partial classifier fix, but the issue is not fully solved. The narrow maintainable path is to propagate or extract cooldown timing into announce retry rather than adding a new product mode or config option.
