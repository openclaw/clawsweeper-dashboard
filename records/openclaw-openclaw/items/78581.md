## Summary

Close as duplicate/superseded: the reported classifier miss and misleading failed-delivery status are fixed on current main and in v2026.6.6, while the remaining cooldown-aware/durable delivery semantics are already owned by open canonical subagent delivery trackers.

## What This Changes

The issue asks for sub-agent announce delivery to treat provider cooldown FallbackSummaryError failures as transient, retry visibly, use cooldown-aware backoff, and report failed delivery truthfully.

## Best Possible Solution

Keep the shipped classifier and task-status fixes, and consolidate the remaining cooldown-aware retry and durable completion-delivery design under https://github.com/openclaw/openclaw/issues/44925 and https://github.com/openclaw/openclaw/issues/67777.

## Reproduction Assessment

Yes. Source inspection shows the original classifier miss is fixed, while the remaining cooldown-delay gap still exists because announce retry uses fixed 5s/10s/20s delays and does not read FallbackSummaryError.soonestCooldownExpiry.

## Solution Assessment

Yes for cleanup. The best triage path is to close this narrower tracker as duplicate/superseded after preserving the remaining cooldown-aware and durable-delivery work on the open canonical subagent delivery issues.
