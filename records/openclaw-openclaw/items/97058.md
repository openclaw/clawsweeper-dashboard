## Summary

Keep open: current main still provides per-call async image-generation tasks, completion wakes, status, and Task Flow as a broader substrate, but it does not provide the requested native multi-item image batch queue or persisted continuation API. This remains a product/API design request rather than a proven regression in an existing contract.

## What This Changes

The issue requests native continuation or queue support for multi-step `image_generate` batches that preserve progress across async completions and chat interruptions.

## Best Possible Solution

Design a maintainer-approved generic media/task continuation model, likely reusing Task Flow or the task ledger, before adding any image-specific batch wrapper.

## Reproduction Assessment

No, not as a high-confidence bug reproduction. Source inspection confirms the native batch/continuation API is absent, but the issue lacks a runnable current-main transcript, logs, and exact provider/session setup.

## Solution Assessment

Unclear. The requested image batch wrapper is plausible, but the maintainable path should probably be a generic Task Flow or media-task continuation design instead of an image-only state machine.
