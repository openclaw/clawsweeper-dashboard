## Summary

Keep open: current main and the latest release still support per-call async image generation, status, completion wake, and count, but not a native sequential multi-job continuation queue. This is better classified as a feature/product API design request than a proven regression in an existing contract.

## What This Changes

The issue requests native durable queue or continuation support so multi-step `image_generate` batches can continue across async completions and interruptions.

## Best Possible Solution

Choose a maintainer-approved generic media/task continuation design, likely through Task Flow or the task ledger, then wire `image_generate` to it without adding an image-only parallel orchestration path.

## Reproduction Assessment

## Solution Assessment

Unclear. A native image batch queue is plausible, but the best fix likely needs a generic Task Flow or task-ledger continuation design rather than an image-only state machine.
