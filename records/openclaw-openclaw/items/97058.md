## Summary

Keep open: current main and v2026.6.10 still provide single async image-generation tasks, status, completion wake, and fallback delivery, but no native multi-item image batch queue or persisted continuation API. This is a feature/product-design request rather than a proven regression in an existing contract.

## What This Changes

Add a native resumable queue or continuation mechanism so multi-image `image_generate` workflows continue through all requested items after async completions.

## Best Possible Solution

Define a generic durable media-task continuation model, likely reusing Task Flow or the task ledger, and add an image-specific surface only after that owner boundary is clear.

## Reproduction Assessment

No, not as a bug reproduction. Source inspection confirms the native batch/continuation API is absent, but the report does not provide a current-main failing run, logs, or exact provider/session transcript.

## Solution Assessment

Unclear. The requested image batch wrapper is plausible, but the maintainable path is probably a generic Task Flow or media-task continuation design rather than an image-only state machine.
