## What This Changes

The issue requests native continuation or queue support for sequential multi-image generation workflows that preserve batch state across async completions and interruptions.

## Best Possible Solution

Approve a generic media-task continuation design that reuses Task Flow or the task ledger, then wire `image_generate` into that owner boundary instead of adding an image-only state machine.
