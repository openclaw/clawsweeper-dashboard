## Summary

Keep open: current main supports single async image-generation tasks, status checks, distinct active image requests, and cron media completion fixes, but it still does not provide the requested native persisted multi-item batch queue or automatic continuation API. This is a feature/product-design request rather than a proven regression in an existing image_generate contract.

## What This Changes

The issue requests native continuation or queue support so multi-image image_generate workflows can automatically advance through all requested items and preserve batch state across async completions or interruptions.

## Best Possible Solution

Design a maintainer-approved generic media/task continuation model, likely using Task Flow or the existing task ledger, then expose image batch continuation through that model instead of adding an image-only state machine.

## Reproduction Assessment

No, not as a high-confidence bug reproduction. Source inspection confirms the native batch/continuation API is absent, but the issue lacks a current-main runnable transcript, logs, or provider/session artifact for the three-image workflow.

## Solution Assessment

Unclear. The requested image batch wrapper is plausible, but the maintainable path should likely reuse Task Flow or a generic media-task continuation contract rather than adding an image-only queue.
