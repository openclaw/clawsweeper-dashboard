## Summary

Keep open: current main and the latest release have per-call async image tasks, completion wakes, status, and duplicate guardrails, but no native multi-item image batch queue or persisted continuation API. The requested behavior remains a feature/product design question rather than a proven regression in an existing contract.

## What This Changes

The issue requests native queue or continuation support so multi-image async generation batches continue across item completions without repeated user reminders.

## Best Possible Solution

Design a narrow, generic media-task continuation or batch primitive on top of the existing task ledger and completion wake path, or document the supported sessions_yield/status pattern, before adding an image-only queue.

## Reproduction Assessment

No, not as a bug reproduction. The source review does confirm the requested native batch queue/continuation API is absent, but the report does not provide a current-main failing run or logs for the exact workflow.

## Solution Assessment

No clear best fix yet. An image-only batch wrapper is plausible, but the safer maintainable direction is a maintainer-approved generic media/task continuation design that avoids duplicating existing agent and tool infrastructure.
