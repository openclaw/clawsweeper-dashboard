## Summary

Keep open: this active maintainer-labeled PR still has one blocking correctness issue in the proposed TranscriptFileState filtering logic, and the protected label/author association require maintainer handling rather than cleanup closure.

## What This Changes

The branch adds TranscriptFileState validation for malformed Pi JSONL entries, regression coverage, a changelog entry, and small qa-lab/xai guard fixes.

## Best Possible Solution

Keep the malformed-entry filtering in TranscriptFileState, but make rejected-id bookkeeping occurrence-safe so a malformed duplicate cannot invalidate an already accepted identity, then add regression coverage.

## Reproduction Assessment

Yes. On PR head, a JSONL transcript with a valid row id, a malformed duplicate using the same id, and a later valid child with parentId set to that id will be relinked as though the accepted parent was rejected.

## Solution Assessment

No. TranscriptFileState is the right layer for this hardening, but the rejected-id map should only record ids that have not already been accepted, or otherwise track rejected occurrences without poisoning accepted ids.
