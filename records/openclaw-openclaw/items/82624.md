## Summary

Keep open: the latest PR fixes the earlier empty-compaction-summary issue, but the new rejected-id tracking can still detach valid children when a malformed row reuses an already accepted entry id.

## What This Changes

The PR adds TranscriptFileState validation for persisted Pi JSONL entries, focused regression coverage, and a changelog line for skipping malformed transcript rows before compaction/rewrite helpers choose a leaf.

## Best Possible Solution

Keep the malformed-entry filtering, but make skipped duplicate rows unable to mark an already accepted id as rejected and add a focused regression for that recovery case.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence path: a valid row id, a later malformed duplicate of that id, and a later valid child with that parentId will have the child relinked away from the accepted parent.

## Solution Assessment

No. The hardening direction is right and the empty-summary compatibility issue is fixed, but the rejected-id bookkeeping needs one narrow adjustment before merge.
