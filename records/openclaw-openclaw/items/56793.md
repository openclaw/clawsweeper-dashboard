## Summary

Keep open. Current main still separates managed `openclaw` profiles from Chrome MCP `existing-session` profiles and has no explicit import/clone surface; the requested feature is plausible but needs maintainer product and security policy before implementation.

## What This Changes

Request to add an explicit one-shot browser profile import from an existing-session/user browser profile into a managed OpenClaw browser profile.

## Best Possible Solution

If maintainers accept the feature, implement it inside the browser plugin as an explicit opt-in import with source/destination validation, destination stop/reset, safe copy exclusions, managed-profile restoration, focused tests, and user-facing docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence check is source, docs, tests, and GitHub search showing create/delete/reset and existing-session attach paths exist, but no import/clone surface exists on current main.

## Solution Assessment

Unclear. An explicit one-shot import is safer than automatic copying, but the maintainable implementation depends on maintainer decisions about consent, exclusions, source/destination rules, and CLI versus API surface.
