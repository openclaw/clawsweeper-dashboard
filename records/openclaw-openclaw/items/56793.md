## Summary

Keep this open: current main and the latest release still expose managed `openclaw` profiles and `existing-session` attach profiles as separate browser modes, with no explicit one-shot import/clone path. The request is useful but security-sensitive because it would copy logged-in browser state into an agent-controlled profile, so it needs maintainer product and security approval before implementation.

## What This Changes

Request an explicit one-shot browser profile import flow that copies an existing-session browser profile into a managed `openclaw` profile.

## Best Possible Solution

After maintainer approval, implement this in the bundled browser plugin as an explicit consent-gated import flow that leaves the source untouched, resets the destination, strips transient browser files, relaunches normally, and documents one-time import semantics.
