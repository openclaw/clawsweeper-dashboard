## Summary

Keep open. Current main still exposes managed browser profiles and Chrome MCP existing-session attach as separate paths with no explicit browser profile import/clone route, and the request is a security-sensitive feature that needs maintainer product review rather than conservative cleanup.

## What This Changes

Request to add an explicit one-shot import flow that copies a user Chrome/Chromium existing-session profile into a managed OpenClaw browser profile.

## Best Possible Solution

If maintainers accept the direction, implement this inside the browser plugin as an explicit opt-in import with source and destination validation, destination stop/reset handling, safe copy exclusions, managed-profile restoration, focused tests, and public security docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection show current OpenClaw supports managed profile lifecycle and existing-session attach paths, but not profile import or clone.

## Solution Assessment

Unclear. An explicit one-shot import is safer than automatic copying, but maintainers need to choose the consent model, exclusions, source/destination rules, and CLI/API surface before code work starts.
