## Summary

Keep this open: current main still separates managed `openclaw` profiles from Chrome MCP `existing-session` profiles and does not provide the requested explicit one-shot import/clone path. The request is plausible, but it would copy cookies, logged-in sessions, and browser state into an agent-controlled profile, so it needs maintainer product and security direction before implementation.

## What This Changes

Add an explicit one-shot browser profile import from a user existing-session profile into a managed OpenClaw browser profile.

## Best Possible Solution

After maintainer approval, implement this in the browser plugin as an explicit command/API with clear consent, same-user local source checks, destination stop/reset, transient-file stripping, focused tests, and docs that it is a one-time import rather than live sync.

## Reproduction Assessment

Not applicable: this is a feature request for a new explicit import path, not a report of broken existing behavior. Current-main source inspection confirms the requested command/API does not exist.

## Solution Assessment

Yes as a direction, but not as an automated fix yet: an explicit one-shot import is safer than automatic copying, while the exact consent, copied data set, lifecycle, and security boundary need maintainer approval.
