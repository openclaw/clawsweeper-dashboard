## Summary

Keep open. Current main still keeps managed `openclaw` profiles and Chrome MCP `existing-session` profiles as separate paths, with no explicit import/clone surface; the request is a plausible browser feature but needs maintainer product and security decisions before implementation.

## What This Changes

Add an explicit opt-in browser profile import that copies same-machine existing-session browser state into a managed OpenClaw profile.

## Best Possible Solution

If maintainers accept the direction, implement the import inside the browser plugin as an explicit opt-in flow with source and destination validation, destination stop/reset handling, safe copy exclusions, managed-profile restoration, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection show current OpenClaw supports managed profile lifecycle and existing-session attach paths, but not profile import or clone.

## Solution Assessment

Unclear. An explicit one-shot import is safer than automatic copying, but the maintainable solution depends on maintainer decisions about consent, exclusions, source/destination rules, and whether the public surface is CLI, API, or both.
