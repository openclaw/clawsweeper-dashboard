## Summary

Keep this open: current main and the latest release still support managed `openclaw` profiles and `existing-session` attach profiles as separate modes, but there is no explicit one-shot import/clone path between them. The request is plausible, but copying browser cookies and logged-in session state into an agent-managed profile is security-sensitive and needs maintainer product/security approval before implementation.

## What This Changes

Request an explicit one-shot CLI or API flow to import existing browser profile state into the managed OpenClaw browser profile.

## Best Possible Solution

Define a maintainer-approved explicit import operation in the browser plugin with source/destination validation, transient-file cleanup, clear security warnings, docs, and focused tests before implementation.

## Reproduction Assessment

Yes for source-level verification: current main exposes profile create/delete/reset and existing-session attach, but no import/clone route or CLI command. This is a feature request rather than a runtime bug reproduction.

## Solution Assessment

Unclear: an explicit one-shot import is safer than automatic copying, but maintainers need to approve the security model, lifecycle, and guardrails before this becomes an implementation task.
