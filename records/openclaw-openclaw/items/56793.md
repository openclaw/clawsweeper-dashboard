## Summary

Keep this open: current main still supports managed `openclaw` profiles and attach-only `existing-session` profiles as separate modes, but it does not provide the requested explicit one-shot import/clone path. The feature would copy signed-in browser state into an agent-controlled profile, so it needs maintainer product and security direction before implementation.

## What This Changes

Add an explicit one-shot CLI/API import that seeds a managed `openclaw` browser profile from an existing-session/user browser profile.

## Best Possible Solution

After maintainer approval, implement the import in the bundled browser plugin as an explicit consent-gated flow that leaves the source untouched, stops/resets the destination, strips transient browser files, relaunches normally, and documents that it is one-time import rather than live sync.

## Solution Assessment
