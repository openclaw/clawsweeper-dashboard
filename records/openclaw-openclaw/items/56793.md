## Summary

## What This Changes

The issue requests an explicit one-shot import flow from an existing browser session/profile into a managed OpenClaw browser profile.

## Best Possible Solution

Add this only after maintainer approval as an explicit browser-plugin CLI/API flow that resolves a same-user local source, stops or resets the destination, copies only approved profile data, strips transient artifacts, relaunches as a normal managed profile, and documents one-time import semantics.
