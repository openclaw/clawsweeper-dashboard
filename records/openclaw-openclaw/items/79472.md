Keep open: current main still drops zero-payload runs with no delivered reply, and the linked closing PR is still open and unmerged, so this issue should remain the tracking item until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to deliver a visible Telegram acknowledgment when an agent turn only performs tool calls and produces no assistant text.

## Best Possible Solution

Review, adjust, and land the linked PR or a replacement that acknowledges local tool-only side-effect turns only when no outbound or message-tool delivery already occurred, with focused regression coverage and real Telegram/Gateway proof.

## Reproduction Assessment

Yes, source-level: current main returns undefined for zero payloads and normalizes empty no-content payloads away, so a tool-only run with no final assistant text has no delivered reply. I did not establish a live Telegram reproduction.

## Solution Assessment

No: a bare tool-call-count fallback is not the safest fix because message-tool sends are also tool calls and already carry delivery evidence. The maintainable solution should gate the acknowledgment on the absence of committed outbound delivery evidence or route it through the existing payload-building semantics.
