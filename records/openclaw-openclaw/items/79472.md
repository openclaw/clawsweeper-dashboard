## Summary

Keep open: current main and the latest release still have empty-payload paths that return without delivery, while the changelog documents that direct/private tool-only completions should receive a completion acknowledgement and an open closing PR is already the active implementation path.

## What This Changes

The issue reports that Telegram direct turns become invisible when the agent performs only tool calls and emits no final assistant text.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79631 with a guarded acknowledgement for direct/private local tool-only completions that have no committed message-tool or other outbound delivery evidence, plus focused regression coverage and real Telegram/Gateway proof.

## Reproduction Assessment

Yes, source-level: current main returns `undefined` for an empty payload array after the run, and empty normalized payloads are skipped. I did not establish a live Telegram reproduction in this read-only review.

## Solution Assessment

No, not as the simple proposed guard alone: acknowledging tool-only side effects is the right direction, but the fix should avoid firing after committed message-tool deliveries and should be proven in a real Telegram/Gateway run.
