## Best Possible Solution

Merge the narrow PR after maintainer approval so `secrets apply` removes plaintext residues without deleting canonical auth-profile SecretRef metadata.

## Solution Assessment

Yes. Preserving `refField` while continuing to delete only the plaintext value field is the narrowest maintainable fix because `keyRef`/`tokenRef` are documented auth-profile SecretRef targets and auth profiles already take precedence in audit semantics.
