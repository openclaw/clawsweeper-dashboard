## Summary

Keep open. Current main and latest release still lack the requested public per-session bootstrap selector and bootstrap command expansion; the remaining work is a core context/config/security design decision, with https://github.com/openclaw/openclaw/pull/22439 covering only part of the bootstrap-selection problem.

## What This Changes

Request deterministic bootstrap context injection with dynamic bootstrap content, session-type filtering, and built-in time/timezone context for autonomous, heartbeat, subagent, and scheduled sessions.

## Best Possible Solution

Define one owner-approved bootstrap context contract that separates deterministic file selection, safe opt-in dynamic context, and cache-stable time access, then reconcile it with the open tiered-bootstrap PR instead of adding parallel config surfaces.

## Reproduction Assessment

Yes for source-level verification, not as a strict existing-contract bug. Current main and v2026.6.6 show fixed session allowlists, heartbeat/lightweight modes, and cache-stable timezone guidance while the requested public filtering and bootstrap command expansion are absent.

## Solution Assessment

Unclear until maintainers choose the public API and security contract. Session-kind filtering likely belongs in core bootstrap resolution if accepted, while direct command expansion and live clock injection need explicit sandboxing and prompt-cache policy.
