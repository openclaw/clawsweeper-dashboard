## Summary

Keep open. Current main partially addresses the reported context clash with hard-coded subagent/cron bootstrap filtering, opt-in `sessions_spawn.lightContext`, and cron/heartbeat current-time lines, but it still does not provide dynamic bootstrap command expansion, configurable session-type bootstrap filters, or direct current-time injection for every session.

## What This Changes

Request adds dynamic bootstrap command expansion, configurable session-type bootstrap filtering, and built-in current time/timezone context for agent sessions.

## Best Possible Solution

Define an owner-approved bootstrap context contract that covers session-kind filtering, a safe dynamic-context extension route, and stable time access without breaking prompt caching or plugin boundaries.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature/API request. Source inspection gives a high-confidence current-main state: some lightContext and scheduled-run time support exists, but the requested dynamic bootstrap and configurable filter surfaces are absent.

## Solution Assessment

Unclear. Session filtering likely belongs in core if accepted, but bootstrap `!command` expansion and direct clock injection need maintainer design because existing hooks may be safer and the prompt-cache/security implications are real.
