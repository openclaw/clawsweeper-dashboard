## Summary

Keep open. Current main partially addresses the subagent/heartbeat context clash with minimal subagent/cron bootstrap filtering and opt-in lightweight context, but the requested dynamic bootstrap expansion, configurable session-type filtering, and direct current-time injection remain unresolved platform design work.

## What This Changes

The issue requests dynamic bootstrap command expansion, session-type filtering for bootstrap files, and built-in current time/timezone context.

## Best Possible Solution

Define an owner-approved bootstrap context contract that covers session-kind filters, a documented dynamic context route, and stable time context without breaking prompt caching or plugin boundaries.

## Solution Assessment

Unclear. Session filtering likely belongs in core, but bootstrap command execution and direct time injection need maintainer design because they affect security, config schema, and prompt-cache behavior.
