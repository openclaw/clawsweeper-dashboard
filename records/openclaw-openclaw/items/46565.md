## Summary

Keep open: current main has partial mitigations for the reported heartbeat/subagent context collision, but the requested dynamic bootstrap preprocessing and explicit session-type filtering contract are not fully implemented and need maintainer design.

## What This Changes

Request platform-level dynamic bootstrap context, per-session-type bootstrap filtering, and built-in user-local time context.

## Best Possible Solution

Define a cache-stable, session-aware bootstrap policy: reconcile the subagent allowlist with docs, then choose either documented include/exclude or section-marker filtering plus a safe dynamic-context mechanism, or explicitly route dynamic behavior through plugin/context-engine hooks.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is an enhancement request with one reported failure mode. Source inspection shows current main now filters HEARTBEAT.md out of subagent/cron bootstrap context, but the broader dynamic-bootstrap and explicit filtering requests remain unimplemented.

## Solution Assessment

Unclear. The proposed direction is plausible, but a generic bootstrap !command syntax is not obviously the safest solution compared with typed config filters, section markers, or existing plugin/context-engine hooks that preserve cache and security boundaries.
