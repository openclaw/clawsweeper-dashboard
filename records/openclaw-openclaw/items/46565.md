Keep open. Current main has useful partial mitigations, including hard-coded subagent/cron bootstrap filtering, opt-in subagent light context, and timestamp/current-time helpers, but the requested dynamic bootstrap expansion and user-configurable session-type filtering contract are still absent and require maintainer product/security design.

## What This Changes

The issue requests dynamic bootstrap-file context expansion, configurable bootstrap filtering by session type, and built-in current time/timezone context for agent sessions.

## Best Possible Solution

Define one maintainer-approved bootstrap context contract that covers session-kind filtering, safe dynamic context extension, and stable time access while preserving prompt-cache and plugin-boundary guarantees.
