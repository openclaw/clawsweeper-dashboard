## Summary

Keep open. Current main and v2026.6.6 cover parts of the problem with fixed subagent/cron bootstrap allowlists, heartbeat lightweight mode, and cache-stable timezone guidance, but the requested public selector, bootstrap command expansion, and live time-context contract remain unresolved product/security work.

## What This Changes

Requests dynamic bootstrap-file context, configurable session-type bootstrap filtering, and built-in time/timezone context for autonomous, heartbeat, cron, and subagent runs.

## Best Possible Solution

Define one maintainer-approved bootstrap context contract that separates deterministic file selection, safe dynamic context, and cache-stable time access, then reconcile it with the open tiered-bootstrap PR and narrower HEARTBEAT.md tracker.

## Reproduction Assessment

Yes, source-level verification is high-confidence: current main and v2026.6.6 lack the requested public bootstrap selector and bootstrap command expansion, while current time remains intentionally excluded from the system prompt. I did not run a live gateway session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API and security contract. The open tiered-bootstrap PR is a plausible partial fix for file selection, but command expansion and live time context need a separate owner-approved boundary.
