## Summary

Keep open: current main partially addresses the reported pain with fixed subagent/cron bootstrap filtering and built-in timestamp/timezone support, but it still lacks configurable session-type bootstrap excludes and bootstrap-file command preprocessing.

## What This Changes

Request dynamic bootstrap-file command preprocessing, session-type-aware bootstrap filtering, and built-in time/timezone context for agent sessions.

## Best Possible Solution

Define a generic bootstrap-context policy that keeps the shipped fixed safeguards, documents session-kind semantics, and only adds command preprocessing behind an explicit security model.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the generalized exclude policy and bootstrap command preprocessing are not current main contracts.

## Solution Assessment

Unclear. The requested direction is plausible, but the remaining work needs maintainer product and security design rather than a narrow bug-fix patch.
