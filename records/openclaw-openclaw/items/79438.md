## Summary

Keep open: current main still lacks probe TTL/backoff, but this PR is not ready because it has blocking lint failures, an introduced reconnect snapshot invalidation regression, and only indirect real behavior proof for the changed branch path.

## What This Changes

The PR adds per-node remote bin probe signature state with 24-hour success caching, failure backoff, cached reconnect reuse, and focused tests in `src/infra/skills-remote`.

## Best Possible Solution

Land a corrected TTL/backoff cache that preserves snapshot invalidation on reconnect, fixes lint, and includes redacted branch-built Gateway proof while leaving readiness-ordering work with https://github.com/openclaw/openclaw/issues/40527.

## Reproduction Assessment

Yes, by source inspection: current main calls `refreshRemoteNodeBins` on node connect, and the refresh path invokes the remote node without persisted success TTL or failure backoff. I did not run a live macOS reconnect reproduction from this read-only checkout.

## Solution Assessment

Partly: per-node/signature TTL and backoff is a narrow maintainable fix for repeated probe churn, but this branch needs snapshot invalidation on cached reconnect, lint fixes, and branch-path real behavior proof before merge.

Overall correctness: patch is incorrect

Overall confidence: 0.9
