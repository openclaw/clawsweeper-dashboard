## Summary

## What This Changes

The PR adds in-memory TTL/backoff state for remote macOS bin probes and tests success caching, reconnect reuse, failure backoff, and signature changes.

## Best Possible Solution

Land the TTL/backoff approach after the contributor supplies redacted branch-built Gateway plus macOS-node proof and the lint failures are fixed, while keeping the readiness timing issue tracked separately.

## Reproduction Assessment

Yes, by source inspection. Current main starts refreshRemoteNodeBins on node connect and the refresh path has no TTL/backoff state, so repeated reconnects can re-probe the same remote macOS node.

## Solution Assessment

Partly. The PR's per-node/signature TTL and backoff state is a narrow maintainable fix for repeated probe churn, but it still needs branch-path real behavior proof and does not replace the separate connect-readiness fix.
