## Summary

Keep open: current main still lacks the central human-day implementation, but the latest PR head leaves one gateway requested-session freshness path on host-timezone fallback and still lacks external real behavior proof.

## What This Changes

The PR adds shared user-timezone human-day helpers and applies them to daily reset, session-memory labels, memory-flush paths, usage day buckets, tests, and changelog.

## Best Possible Solution

Land the shared human-day approach after the gateway requested-session caller passes cfg, session-memory docs are aligned, and the contributor adds redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows the gateway requested-session path loads cfg but omits it from evaluateSessionFreshness, so with host TZ UTC, userTimezone Asia/Shanghai, and daily reset at 4, that entry point can still compare host-day cycles.

## Solution Assessment

No. The shared helper is the right direction, but leaving cfg optional and missing this gateway caller is not the narrowest maintainable fix; pass cfg there or make daily-mode freshness prevent silent host fallback.
