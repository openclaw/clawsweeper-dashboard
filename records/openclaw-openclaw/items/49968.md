## Summary

Keep open. Current main has not absorbed the human-day behavior, and the latest PR head still has blocking review issues: a requested-session freshness path omits the loaded config, session-memory LLM slugging becomes default-on, docs still describe the old date boundary, and after-fix real behavior proof is missing.

## What This Changes

The PR adds shared human-day helpers and applies configured user-timezone daily-boundary semantics to session reset freshness, session-memory artifacts, memory flush paths, usage summaries, tests, and changelog text.

## Best Possible Solution

Land one coordinated human-day fix after every freshness caller passes config, session-memory keeps LLM slugging opt-in, docs match the new timezone semantics, and redacted runtime proof demonstrates the fixed behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with host TZ different from agents.defaults.userTimezone, the PR head requested-session gateway path still omits cfg, and session-memory now enables LLM slug generation unless explicitly disabled.

## Solution Assessment

No. The shared helper direction is reasonable, but this patch is not the best mergeable solution until the cfg gap, opt-in LLM slug contract, docs, and real behavior proof are fixed.

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep session-memory LLM slugs opt-in:** `src/hooks/bundled/session-memory/handler.ts:162`
  - body: This changes `allowLlmSlug` from explicitly opt-in to enabled unless `llmSlug` is false, so a default session-memory hook with config and transcript content can send recent conversation text to the configured model provider on every `/new` or `/reset`.   - confidence: 0.94

  - body: `evaluateSessionFreshness` now uses `params.cfg` to choose the human-day reset cycle, but this requested-session gateway path still omits the loaded config. With host timezone different from `agents.defaults.userTimezone`, the same session can reset differently through this control-plane path than through the channel, CLI, or cron paths that now pass cfg.
  - confidence: 0.92
- **[P3] Update the session-memory date docs:** `src/hooks/bundled/session-memory/handler.ts:177-179`
  - body: The handler now writes session-memory filenames and headers with the configured human day/timezone, but the published hook docs still say files use the host local date. Users should not be told the old boundary after this behavior change.
  - confidence: 0.86
