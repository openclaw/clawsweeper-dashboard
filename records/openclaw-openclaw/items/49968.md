## What This Changes

Adds shared user-timezone human-day helpers and wires daily session reset, session memory, memory flush, usage buckets, WhatsApp coverage, tests, and changelog to consistent day keys.

## Best Possible Solution

Finish the shared human-day fix by making daily freshness callers pass the loaded cfg, especially gateway requested sessions, and land it with focused gateway regression coverage plus real behavior proof.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows the gateway requested-session path loads cfg but omits it when calling evaluateSessionFreshness, so with host TZ UTC, userTimezone Asia/Shanghai, and daily reset at 4, that entry point can still compare host-day cycles.

## Solution Assessment

No. The shared helper is the right direction, but leaving cfg optional and missing the gateway requested-session caller is not the narrowest maintainable solution; pass cfg there or make the freshness API prevent daily-mode host fallback.
