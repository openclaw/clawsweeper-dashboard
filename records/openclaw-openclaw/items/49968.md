## Summary

Keep open: current main still has the underlying host/UTC daily-boundary behavior, but this PR is not mergeable as-is because the latest head is conflicting, lacks real behavior proof, and still has privacy and correctness regressions in the session-memory and daily freshness paths.

## What This Changes

The PR adds shared human-day helpers and applies configured user-timezone daily boundaries to session reset, session-memory artifacts, memory flush paths, usage summaries, tests, and changelog text.

## Best Possible Solution

Rebase on current main, keep model-generated slugs explicitly opt-in and collision-safe, pass cfg through every daily freshness path, align user-facing docs, and add redacted runtime proof for the timezone boundary behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure agents.defaults.userTimezone differently from the host timezone and compare daily reset, session-memory, and usage day boundaries; I did not run a live repro in this read-only review.

## Solution Assessment

No. The shared human-day direction is reasonable, but this branch is not the best mergeable solution until the privacy regression, cfg gap, filename overwrite regression, conflicting state, and real proof gate are handled.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep session-memory LLM slugs opt-in:** `src/hooks/bundled/session-memory/handler.ts:162`
  - body: This changes LLM slug generation from explicitly opt-in to enabled unless llmSlug is false. Default session-memory capture can now send recent transcript content to the configured model provider during /new or /reset, broadening privacy, data-scope, and cost behavior outside the human-day fix.
  - confidence: 0.95
- **[P2] Require cfg for daily freshness calculations:** `src/config/sessions/reset-policy.ts:81`
  - body: Making cfg optional lets updated callers silently fall back to the host timezone; the requested-session gateway path still calls evaluateSessionFreshness without cfg. With host timezone different from agents.defaults.userTimezone, the same session can reset differently depending on the control-plane path.
  - confidence: 0.91
- **[P2] Preserve unique session-memory filenames:** `src/hooks/bundled/session-memory/handler.ts:188`
  - body: Current main resolves an available memory filename before writing, but this branch writes date-slug.md directly. Repeated fallback HHMM slugs or repeated model slugs on the same human day can overwrite an earlier memory file instead of creating a suffixed file.
  - confidence: 0.86
- **[P3] Update the session-memory date docs:** `src/hooks/bundled/session-memory/handler.ts:201`
  - body: The handler now writes filenames and headers with the configured human day/timezone, but the published hook docs still describe host-local date behavior and do not match the new slug behavior. Users should not be left with the old boundary after this behavior change.
  - confidence: 0.82
