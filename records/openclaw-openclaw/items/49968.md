## Summary

Keep open: current main still has host-local daily reset/session-memory behavior, but this PR head is not merge-ready because it keeps LLM session-memory slugs default-on, removes collision-safe memory filenames, leaves a cfg-less daily freshness path, and lacks real behavior proof.

## What This Changes

The PR adds a shared human-day helper and applies userTimezone-aware daily keys to session reset, session-memory artifacts, memory flush paths, usage buckets, tests, and the changelog.

## Best Possible Solution

Define and document one human-day contract for daily reset, session memory, memory flush, and usage while preserving opt-in LLM slugging, collision-safe writes, cfg propagation, request-specific usage bucket labels, and real behavior proof.

Yes, by source inspection: configure agents.defaults.userTimezone differently from the host timezone and compare current-main daily reset, session-memory filenames, memory-flush paths, and usage day buckets. I did not run a live repro in this read-only review.

## Solution Assessment

No. The shared helper direction is plausible, but this PR is not the best mergeable solution until it preserves privacy and collision guarantees, passes cfg through all daily freshness paths, updates docs, and adds real behavior proof.
