## Summary

Keep open. The human-day direction is useful, but the current PR still has a privacy regression, a cfg propagation gap in session freshness, stale docs, missing real behavior proof, and a dirty merge state against current main.

## What This Changes

The PR adds shared human-day helpers and applies configured user-timezone daily-boundary semantics to session reset freshness, session-memory artifacts, memory flush paths, usage summaries, tests, and changelog text.

## Best Possible Solution

Land a coordinated human-day change only after preserving opt-in slugging, passing cfg through every daily freshness caller, updating docs, resolving the dirty branch, and adding redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a host timezone different from agents.defaults.userTimezone, then compare session-memory/day-boundary behavior; the PR head also has source-visible regressions for cfg omission and default-on LLM slugging.

## Solution Assessment

No. The shared helper direction is reasonable, but this patch is not the best mergeable solution until the cfg gap, opt-in slug contract, docs, merge state, and real behavior proof are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep session-memory LLM slugs opt-in:** `src/hooks/bundled/session-memory/handler.ts:162`
  - body: This changes `allowLlmSlug` from explicitly opt-in to enabled unless `llmSlug` is false, so a default session-memory hook with config and transcript content can send recent conversation text to the configured model provider on every `/new` or `/reset`. That changes privacy, data scope, and cost behavior outside the human-day fix.
  - confidence: 0.94
- **[P2] Require cfg for daily freshness callers:** `src/config/sessions/reset-policy.ts:81`
  - body: Making `cfg` optional lets updated callers silently fall back to the host timezone; the requested-session gateway path still calls `evaluateSessionFreshness` without cfg. With host timezone different from `agents.defaults.userTimezone`, the same session can reset differently depending on the control-plane path.
  - confidence: 0.9
- **[P3] Update the session-memory date docs:** `src/hooks/bundled/session-memory/handler.ts:177-179`
  - body: The handler now writes session-memory filenames and headers with the configured human day/timezone, but the published hook docs still say files use the host local date. Users should not be told the old boundary after this behavior change.
  - confidence: 0.86
