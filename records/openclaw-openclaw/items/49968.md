## Summary

Keep open: current main has not implemented the central human-day behavior, but this branch is not merge-ready because it still changes LLM slugging to default-on, can overwrite memory files, leaves a cfg-less daily freshness path, is conflicting, and lacks real behavior proof.

## What This Changes

The PR adds shared user-timezone human-day helpers and applies them to session reset, session-memory artifacts, memory flush paths, usage day buckets, tests, and the changelog.

## Best Possible Solution

Define and document one human-day contract for daily reset, session memory, memory flush, and usage while preserving opt-in LLM slugging, collision-safe writes, and request-specific usage bucket labels.

## Reproduction Assessment

Yes, by source inspection: configure agents.defaults.userTimezone differently from the host timezone and compare daily reset, session-memory, memory flush, and usage bucket boundaries. I did not run a live repro in this read-only review.

## Solution Assessment

No. The shared helper direction is plausible, but this PR is not the best mergeable solution until it preserves privacy/collision guarantees, passes cfg through all daily freshness paths, resolves conflicts, updates docs, and adds real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep LLM slugs explicitly opt-in:** `src/hooks/bundled/session-memory/handler.ts:162`
  - body: This changes session-memory slug generation from explicitly opt-in to enabled unless llmSlug is false. Default `/new` or `/reset` memory capture can now send recent transcript content to the configured model provider, broadening privacy, data-scope, and cost behavior outside the human-day fix.
  - confidence: 0.95
- **[P2] Pass cfg through daily freshness callers:** `src/config/sessions/reset-policy.ts:81`
  - body: Making cfg optional lets callers silently fall back to the host timezone. The requested-session gateway path still calls evaluateSessionFreshness without cfg, so the same stored session can reset differently depending on which control-plane path touches it.
  - confidence: 0.9
- **[P2] Preserve collision-safe memory filenames:** `src/hooks/bundled/session-memory/handler.ts:188`
  - body: Current main resolves an unused memory filename before writing, but this branch writes `${dateStr}-${slug}.md` directly. Repeated fallback HHMM slugs or repeated model slugs on the same human day can overwrite an earlier memory file.
  - confidence: 0.88
- **[P3] Update session-memory docs for the new day contract:** `src/hooks/bundled/session-memory/handler.ts:201`
  - body: The handler now writes filenames and headers using the configured human day/timezone, but the public hook docs still describe host-local dates and opt-in LLM slugs. The user-facing docs need to match whichever behavior is kept.
  - confidence: 0.8
