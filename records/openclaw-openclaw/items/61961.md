  - body: The patch adds `readonly` to tool `defaultProfiles`, but `ToolCatalogProfileSchema.id` still accepts only the old four IDs. `ToolsCatalogResultSchema.profiles` uses that schema, so the `readonly` profile returned from `PROFILE_OPTIONS` can fail the protocol contract; add it to the profile-id union too.

  - body: This expands the catalog-local `ToolProfileId`, but the public config `ToolProfileId` in `src/config/types.tools.ts` still omits `readonly`. TypeScript users of `OpenClawConfig` cannot set the new profile without casts even though runtime schema accepts it.
  - confidence: 0.91
- **[P3] Expose readonly in Quick Settings:** `src/agents/tool-catalog.ts:350`
  - body: Adding `readonly` to `PROFILE_OPTIONS` makes it a built-in profile, but Quick Settings still hardcodes only the old four profile buttons in `ui/src/ui/views/config-quick.ts`. Users cannot choose the new profile there unless it was already typed into config, so update that list or derive it from the catalog/schema.
  - confidence: 0.81
