## Summary

This PR is superseded: the default-on loop-detection policy was rejected in the narrower follow-up, current main already has the safer no-progress-gated generic_repeat critical block and merged post-compaction guard, and the remaining exec/sandbox pieces are security-sensitive scope that should not ride on this mixed branch.

## What This Changes

The PR enables loop detection by default with lower thresholds, adds aggregate-count critical generic_repeat blocking, changes exec approval ask=off precedence, strips trailing infrastructure-error turns, and allows SANDBOX_POLICY.json through agents.files.set.

## Best Possible Solution

Keep loop detection default-off per current policy, retain the no-progress-gated generic block and post-compaction guard already on main, and require separate focused review for any exec approval or sandbox policy-write changes.

## Solution Assessment
