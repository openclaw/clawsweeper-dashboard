## Summary

Keep open. The PR addresses a real agent loop-safety gap that is still present on current main, but the branch should not merge as-is because it bundles loop defaults, exec approval precedence, error-turn stripping, and sandbox-policy file writes while introducing approval/security and transcript-state regressions.

## What This Changes

The PR enables loop detection by default with lower thresholds and generic-repeat critical blocking, changes exec approval ask merging, adds trailing infrastructure-error turn stripping with prompt recovery, and allows `SANDBOX_POLICY.json` through `agents.files.set`.

## Best Possible Solution

Split the work into focused owner-reviewed PRs: loop detection should land with consecutive/no-progress generic blocking plus docs/tests/changelog, exec approval precedence should preserve stricter policy layers, and queueflood stripping plus sandbox-policy writes should get separate security-reviewed changes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main defaults loop detection off and keeps `generic_repeat` warning-only, while current exec approval tests assert the stricter ask contract that this PR changes.

## Solution Assessment

No. The branch addresses real symptoms, but the safer maintainable solution is narrower work that does not downgrade exec approvals, does not block aggregate non-consecutive repeats, and does not mix sandbox-policy writes with unrelated runner recovery changes.

  - confidence: 0.86
