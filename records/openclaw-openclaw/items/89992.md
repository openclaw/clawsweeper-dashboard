## Summary

Keep open: the editor-schema feature is useful and has real proof, but the include-backed config mutation path is not yet aligned with the full-file writer for blank `$schema` values or live plugin/channel schema metadata.

## What This Changes

The PR adds an `openclaw.schema.json` sidecar during OpenClaw-owned config writes, stamps missing local `$schema` refs, reuses schema generation for `openclaw config schema`, and documents/tests the behavior.

## Best Possible Solution

Land the editor-schema sidecar only after the include-backed mutation path uses the same blank-`$schema` handling and plugin/channel metadata source as the full-file writer, with focused regression coverage.

## Reproduction Assessment

Yes. From source, an include-backed root with a blank `$schema` and a single top-level `$include` reaches `insertEditorConfigSchemaRefRaw`, which returns on any defined `$schema`, leaving the blank value in place.

## Solution Assessment

No. The owner boundary is plausible, but the best merge state is to make the include-backed fast path share the full-file writer's schema-ref and plugin-metadata behavior before landing.

Full review comments:

- **[P2] Handle blank schema refs in raw include roots:** `src/config/editor-schema.ts:27-28`
  - body: `insertEditorConfigSchemaRefRaw` bails out whenever `$schema` is present, even if it is blank. For a root JSON5 file like `{ "$schema": "  ", "plugins": { "$include": "./config/plugins.json5" } }`, a plugin-only config mutation takes the single-include fast path and preserves the unusable blank value, while the normal writer now replaces blanks with `./openclaw.schema.json`. This path needs the same trim-aware handling or a fallback to the full writer.
  - confidence: 0.92
- **[P2] Pass plugin metadata when writing include-path schemas:** `src/config/mutate.ts:402`
  - body: The include fast path calls `writeEditorConfigSchemaFile` without the prepared plugin metadata, even though `readConfigFileSnapshotForWrite` puts `basePluginMetadataSnapshot` in `writeOptions` and the full-file writer passes that metadata through. A `config set` that mutates a single top-level `$include` will therefore write a base-only `openclaw.schema.json`, missing installed/workspace plugin and channel schemas that `openclaw config schema` and the docs promise when manifests load.
  - confidence: 0.88
