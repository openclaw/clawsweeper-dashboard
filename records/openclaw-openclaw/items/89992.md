## Summary

## What This Changes

The PR writes `openclaw.schema.json` beside active config files, stamps a missing or blank local `$schema` reference during config writes, reuses that schema for `openclaw config schema`, and updates config docs/tests.

## Best Possible Solution

Make raw `$schema` stamping JSON5-root aware or use a safe rewrite fallback, preserve existing non-empty schemas, add leading-comment regression coverage, and require real proof for both full-write and include-backed config writes before merge.

## Reproduction Assessment

## Solution Assessment

No. Writing a local editor schema from config IO is plausible, but the current implementation is not the best fix until root stamping is parser-aware and the include-backed path has real behavior proof.
