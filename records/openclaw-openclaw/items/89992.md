## What This Changes

The PR writes `openclaw.schema.json` beside the active config, stamps missing or blank local `$schema` refs during OpenClaw-owned writes, reuses that schema for `openclaw config schema`, and documents/tests the behavior.

## Best Possible Solution

Keep the config-writer approach only after schema stamping is parser-aware for JSON5 roots, preserves include authoring safely, and has latest-head real behavior proof for the changed write path.

## Reproduction Assessment

## Solution Assessment

No. Writing a local editor schema from config IO is a plausible product direction, but the include-backed raw stamping path needs to be token/parser aware before this is the best implementation.

- **[P2] Find the real JSON5 root before raw schema insertion:** `src/config/editor-schema.ts:39-53`
  - body: For include-backed writes, this helper inserts after `params.raw.indexOf("{")`. OpenClaw accepts JSON5 comments before the root object, so a valid config like `// note { ...\n{ plugins: { $include: ... } }` will get `$schema` inserted into the comment instead of the object and the subsequent reread will see an invalid config. Please make the insertion parser/token aware, or fall back to a safe rewrite, and cover leading comments before the root.
