## Summary

Keep this PR open: the local schema idea is useful, but the latest head still rewrites raw JSON5 by first regex/index match, so valid leading-comment config roots can be mutated incorrectly before OpenClaw rereads the config.

## What This Changes

The PR writes `openclaw.schema.json` beside active config files, stamps missing or blank local `$schema` refs during config writes, reuses the schema helper for `openclaw config schema`, updates docs/tests, and restores the plugin SDK testing facade fixture.

## Best Possible Solution

Make schema stamping token/parser-aware for JSON5 roots or fall back to a safe full rewrite, then prove fresh-install and upgrade/include-backed config writes before merge.

## Reproduction Assessment

Yes for the review finding from source inspection: OpenClaw accepts JSON5 comments, and the helper edits the first raw `$schema`/`{` text rather than the parsed root token. I did not run a mutating repro because this review is read-only.

## Solution Assessment

No. Writing a local editor schema from config IO is plausible, but the current implementation is not the best fix until raw schema stamping is parser-aware and the include-backed path has real behavior proof.
