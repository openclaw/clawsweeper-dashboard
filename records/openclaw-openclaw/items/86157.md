- **[P1] Export the broker runtime setter:** `extensions/channel-broker/runtime-api.ts:1-18`

- **[P2] Infer broker Slack DMs as direct targets:** `extensions/channel-broker/src/channel.ts:54-62`
  - body: For `broker:slack:user:U12345678`, this parser sees `parsed.conversationId` as `slack:user:U12345678`, so `maybeType` becomes `slack` and the function returns `channel`. Host target resolution uses this inference to choose user vs channel/group targets, so Slack DM broker targets can get channel session metadata even though `parseChannelBrokerTarget` later sends them as direct conversations. Strip the broker/platform prefix or reuse the same platform-specific parsing before returning the chat type.
  - confidence: 0.88
