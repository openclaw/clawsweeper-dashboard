## Summary

Keep open: current `main` and latest release still have the managed npm update path that can overwrite a newer direct dependency pin, and the open linked PR is the active fix path rather than evidence that the issue is resolved.

## What This Changes

The issue reports that `openclaw update --tag` can silently rewrite a newer user-pinned managed plugin direct dependency to an older release-managed version.

## Best Possible Solution

Keep this issue open as the canonical tracker until https://github.com/openclaw/openclaw/pull/87304 lands, or maintainers choose and implement a different managed plugin pin-conflict policy.

## Reproduction Assessment

Yes, from source inspection: the update path calls the managed npm installer in update mode, and the manifest writer overwrites an existing direct dependency entry with the release-managed dependency spec. I did not run the mutating update command in this read-only review.

## Solution Assessment

Yes if maintainers accept fail-closed behavior: the linked PR is the active fix path for the shared managed npm boundary. The remaining policy question is whether fail-closed is the desired update behavior, not whether this issue is obsolete.
