## Summary

Keep open: current main still has a high-confidence source path for the managed npm direct-dependency downgrade, and an open linked implementation PR is already attempting the narrow fix, so this issue should not close before that PR lands or is closed.

## What This Changes

The issue reports that `openclaw update` can overwrite a user-pinned managed npm direct dependency with an older bundled-plugin version during update.

## Best Possible Solution

Review and land a narrow fix through https://github.com/openclaw/openclaw/pull/85305 that preserves or blocks newer compatible managed direct dependencies during update, while leaving the broader all-pins policy separate unless maintainers choose it explicitly.

## Reproduction Assessment

Yes, at source level. Current main still routes an externalized bundled-plugin npm spec into the managed npm install path, where the incoming dependency spec overwrites an existing direct dependency entry.

## Solution Assessment

Yes for the downgrade-prevention half: the best fix is a narrow guard in the managed npm update path before writing or verifying a lower direct dependency version. The broader respect-all-pins behavior should stay out of scope unless maintainers choose that policy explicitly.
