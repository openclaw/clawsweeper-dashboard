## Summary

Keep open: current main and the latest release still allow the managed npm update path to overwrite a newer direct dependency pin, and the linked open PR is the active fix path rather than evidence to close this issue.

## What This Changes

Prevent `openclaw update` from silently downgrading or rewriting newer user-pinned managed npm direct dependencies without an explicit conflict or operator choice.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/87304 if maintainers accept the fail-closed update policy; otherwise choose an explicit preserve-or-override policy and keep this issue as the canonical tracker until a fix merges.

## Reproduction Assessment

Yes from source inspection: the externalized bundled plugin update path calls npm install in update mode, and the managed npm root writer overwrites the package dependency entry. I did not run the mutating `openclaw update --tag` scenario in this read-only review.

## Solution Assessment

Yes if maintainers accept fail-closed behavior: the linked PR places the guard before dry-run or install mutation in the shared managed npm install boundary. The broader respect-all-pins behavior remains a policy choice and should not block keeping this issue open until the concrete fix merges.
