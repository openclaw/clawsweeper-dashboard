## Summary

Keep open. Current main still exposes fal image/video generation through curated static model lists, while the request asks for full FAL catalog discovery and schema-aware model selection that needs maintainer scoping rather than cleanup closure.

## What This Changes

Request to extend the fal media generation plugin so OpenClaw dynamically discovers and exposes the full FAL.ai model catalog as selectable image/video generation models.

## Best Possible Solution

Define a fal-owned, schema-aware supported catalog for media modes and add a backward-compatible discovery seam only if static provider metadata cannot support that safely.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect the fal image/video provider registrations and list actions, which are bounded by provider.models arrays on current main.

## Solution Assessment

Unclear. Exposing all 1000+ FAL endpoints as built-in selectable media models is not yet proven to be the narrowest maintainable solution; a filtered catalog with per-model capability mapping is safer.
