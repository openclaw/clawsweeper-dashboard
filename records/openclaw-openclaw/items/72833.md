## Summary

Keep open. Current main still exposes fal image and video generation through curated static model arrays, while the request asks for broad dynamic FAL catalog discovery and model-specific schema/capability mapping that needs maintainer product/API scoping.

## What This Changes

Request fal image/video generation dynamically discover and expose the full FAL.ai model catalog as selectable OpenClaw media models.

## Best Possible Solution

Define a fal-owned, schema-aware catalog for supported OpenClaw media modes, then add a backwards-compatible media-catalog/cache seam only if static registration cannot express the selected scope.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect the fal provider registrations and `image_generate`/`video_generate` list paths, which are still bounded by static `provider.models` arrays on current main.

## Solution Assessment

Unclear. The full-catalog direction is useful, but exposing every FAL endpoint as a built-in selectable media model is not yet proven to be the narrowest maintainable solution; a filtered, schema-aware fal catalog is safer.
