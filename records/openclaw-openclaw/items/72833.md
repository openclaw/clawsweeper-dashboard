## Summary

Keep open. Current main still exposes fal media generation through curated static model arrays, while the requested full FAL.ai catalog needs maintainer scoping for catalog size, per-model schemas, and media capability mapping.

## What This Changes

The issue requests dynamic discovery and selection of the full FAL.ai image/video model catalog through the bundled fal media provider.

## Best Possible Solution

Define a fal-owned, schema-aware catalog for supported media modes and add a backward-compatible media discovery seam only if static provider metadata cannot support it.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect the fal provider registrations and media list actions, which still surface bounded static `provider.models` arrays on current main.

## Solution Assessment

Unclear. Exposing all 1000+ FAL endpoints directly as built-in selectable media models is not yet proven to be the narrowest maintainable solution; a filtered capability-aware catalog is safer.
