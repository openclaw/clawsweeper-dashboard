## Summary

Keep open. Current main still exposes fal media generation through curated static model arrays, while the request asks for broad FAL catalog discovery and model-specific media capability/schema mapping that needs maintainer scoping rather than cleanup closure.

## What This Changes

Request dynamic discovery and selection of the full FAL.ai image/video model catalog in the bundled fal media provider.

## Best Possible Solution

Define a fal-owned, schema-aware catalog for supported media modes and add a backward-compatible media catalog seam only if static provider metadata cannot support it.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect the fal image/video provider registrations and list actions, which are bounded by provider.models arrays on current main.

## Solution Assessment

Unclear. Exposing all 1000+ FAL endpoints as built-in selectable media models is not yet proven to be the narrowest maintainable solution; a filtered catalog with per-model capability mapping is safer.
