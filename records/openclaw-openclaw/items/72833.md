## Summary

Keep open. Current main still exposes fal image and video generation through curated static model arrays, and the requested full dynamic FAL catalog needs maintainer/API scoping rather than cleanup closure.

## What This Changes

The issue requests dynamically exposing the full FAL.ai model catalog through the fal media-generation provider instead of the current curated model lists.

## Best Possible Solution

Ship a fal-owned catalog of supported model families with explicit mode, capability, and schema metadata, then add a plugin SDK discovery seam only if existing provider metadata cannot represent it.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect the fal image/video provider registrations and list actions, which are bounded by provider.models arrays on current main.

## Solution Assessment

Unclear. Exposing all 1000+ FAL endpoints as built-in selectable media models is not yet proven to be the narrowest maintainable solution; a filtered schema-aware catalog is safer.
