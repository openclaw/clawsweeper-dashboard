## Summary

Keep open. Current main still exposes FAL image and video generation through static, curated media-provider model arrays, while the request asks for dynamic full-catalog discovery across mixed FAL modalities; that is real provider/API/product work, not conservative cleanup-close material.

## What This Changes

Request to extend the bundled `fal` image/video generation provider so OpenClaw dynamically discovers and exposes the full FAL.ai model catalog as selectable media models.

## Best Possible Solution

Define a scoped, provider-owned FAL media catalog strategy first: filter by supported media capability, map per-model schemas and limits inside the FAL plugin, and add an additive generic discovered-media-catalog seam only if the current image/video provider contracts are insufficient.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect the FAL provider registrations or call the media list surfaces on current main and they return the static `models` arrays registered by the bundled FAL image/video providers.

## Solution Assessment

Unclear. The requested direction is valid, but exposing all FAL endpoints as selectable built-in media models is not yet proven to be the narrowest maintainable solution; a filtered, capability-mapped catalog with provider-owned schema handling is safer.
