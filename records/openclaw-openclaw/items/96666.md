## Summary

Keep open: current main still has the unbounded OpenAI/Codex Responses and proxy SSE success-body reads, and this PR is a focused hardening path with sufficient real-behavior proof. The patch is not obsolete, but it carries a compatibility decision because the new guard fails closed at 16 MiB across the stacked streaming surfaces.

## What This Changes

The branch stacks the Anthropic SSE byte-guard work and additionally applies the same internal 16 MiB streaming success-body guard to OpenAI/Codex Responses and runtime proxy streams, with focused tests and repro scripts.

## Best Possible Solution

Land this stack only after maintainers accept or tune the 16 MiB fail-closed streaming boundary and either merge https://github.com/openclaw/openclaw/pull/96632 first or intentionally land both layers together.

## Reproduction Assessment

Yes, source-reproducible: current main appends SSE chunks into in-memory buffers in both OpenAI/Codex and proxy paths without a success-body byte guard. I did not execute the repro, but the PR body includes after-fix local-server proof with overflow, negative-control, and happy-path output.

## Solution Assessment

Mostly yes: an internal guard is the right owner boundary for core OpenAI/proxy paths and avoids premature SDK API surface. The exact total-stream 16 MiB boundary is the part that needs maintainer acceptance or tuning because upstream Codex uses idle-timeout/event parsing rather than a total byte cap.
