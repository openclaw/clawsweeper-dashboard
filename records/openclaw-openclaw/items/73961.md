Keep this PR open, but do not merge it as-is. The WhatsApp outbound mention feature remains valid and belongs in the WhatsApp plugin, but the branch is stale against current main's send-result contract and still has resolver edge cases around LID preference and ambiguous digit lookups.

## What This Changes

The PR adds WhatsApp outbound mention resolution for group text and media captions, wires participant metadata into WhatsApp send paths, and updates WhatsApp docs, changelog, and tests.

## Best Possible Solution

Port the mention resolver onto current main's WhatsApp send-result implementation, keep the scope limited to native group mention metadata for #39879, fix LID priority and digit-collision behavior with focused tests, preserve the docs/changelog credit to #56863, and leave reply-target media injection for separate work.

Yes. Current main's `createWebSendApi().sendMessage` path builds `{ text }` or caption payloads without `mentions`, and the PR blockers are reproducible by comparing PR head against current `WhatsAppSendResult` plus pure resolver inputs containing both phone and LID participant fields.

## Solution Assessment

No, not as currently proposed. The WhatsApp plugin is the right boundary and Baileys supports the `mentions` field, but the branch needs a current-main rebase plus resolver fixes before it is the narrow maintainable solution.
