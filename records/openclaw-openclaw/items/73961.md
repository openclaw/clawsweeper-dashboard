Keep open; WhatsApp outbound mentions are still missing on current main and this PR targets the right WhatsApp plugin boundary, but the branch cannot merge as-is because it regresses current send-result/group-metadata contracts and leaves resolver edge cases unresolved.

## What This Changes

The PR adds WhatsApp outbound mention resolution for group text and media captions, caches participant mention metadata, wires Baileys `mentions` into send paths, and updates WhatsApp docs, changelog, and tests.

## Best Possible Solution

Port the outbound mention resolver onto current main's WhatsApp send-result and group metadata cache implementation, fix the LID priority and collision cases with focused tests, keep the scope limited to native group mention metadata for #39879, and preserve the #56863 credit without adding reply-target media injection.

## Solution Assessment

No, not as currently proposed. The WhatsApp plugin boundary and Baileys `mentions` mechanism are the right direction, but the branch needs a current-main rebase plus resolver fixes before it is the narrow maintainable solution.

That makes `@277038292303944` resolve to the phone JID in LID-addressed groups; prefer a valid `lid` JID over a phone `id` when both are present.

Use the token form or another explicit rule to preserve sender intent when both maps contain the same key.
