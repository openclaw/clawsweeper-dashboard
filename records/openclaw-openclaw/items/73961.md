## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks native outbound WhatsApp mention metadata, and this branch is a useful scoped port for #39879, but it is stale against the current WhatsApp send-result contract and still has LID participant resolution edge cases.

## What This Changes

The PR adds WhatsApp outbound mention resolution for group text and media captions, carries participant mention metadata from Baileys group data into send paths, and updates WhatsApp docs, changelog, and tests.

## Best Possible Solution

Port the outbound mention resolver onto current main's WhatsApp send-result implementation, keep the scope limited to native group mention metadata for #39879, fix LID priority and ambiguous digit resolution with focused tests, preserve the docs/changelog credit to #56863, and leave reply-target media injection for a separate change.

## Reproduction Assessment

Yes. On current main, a group `createWebSendApi().sendMessage(..., "hi @+15551234567")` path builds a plain text payload without `mentions`; the PR blockers are reproducible by comparing PR head `send-api.ts` against current `WhatsAppSendResult` and by pure resolver inputs with both phone and LID participant fields.

## Solution Assessment

No, not as currently proposed. The WhatsApp plugin is the right boundary and the feature shape is appropriate, but the branch needs a current-main rebase plus resolver fixes before it is the narrow maintainable solution.
