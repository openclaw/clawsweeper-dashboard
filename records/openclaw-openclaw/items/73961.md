## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks native WhatsApp outbound mention metadata, and the PR carries useful #39879/#56863 work forward, but the branch is stale against current main and regresses current WhatsApp send-result, group metadata cache, and quoted-media behavior while leaving resolver edge cases open.

## What This Changes

The PR adds a WhatsApp outbound mention resolver, wires Baileys `mentions` into group text/media send paths, and updates tests, docs, and changelog for #39879.

## Best Possible Solution

Port the resolver onto current main's WhatsApp send-result, reconnect-cache, and quoted-media implementation; fix the LID priority and collision cases with focused tests; keep scope limited to native group mention metadata.

## Reproduction Assessment

Yes. Current main source shows outbound WhatsApp text/media sends do not attach Baileys `mentions`, and the PR blockers reproduce by comparing PR head with current main plus pure resolver inputs for the LID and digit-collision cases.

## Solution Assessment

No, not as currently proposed. The Baileys `mentions` path is the right mechanism, but this branch needs a current-main rebase and resolver fixes before it is the narrow maintainable solution.
