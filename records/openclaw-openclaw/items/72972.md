## Summary

Keep this PR open for normal review and CI. Current main still has the Feishu queue/routing mismatch, and the proposed patch is a focused alignment with the existing group-session contract rather than a separate policy change.

## What This Changes

The PR threads Feishu account config into inbound event handler registration, changes Feishu group-message sequential keys to use the existing group-session resolver, adds sequential-key tests, and adds a changelog entry.

## Best Possible Solution

Land this targeted queue-key alignment after maintainer review and green relevant checks, keeping `resolveFeishuGroupSession` as the single source of truth for Feishu group peer IDs while preserving per-chat control and `/btw` lanes.

## Reproduction Assessment

Yes. Configure Feishu with `groupSessionScope: "group_topic"` or `"group_topic_sender"` and send ordinary group messages in different roots/topics in the same chat; current main routes them to different peers but enqueues both under `feishu:<accountId>:<chatId>`.

## Solution Assessment

Yes. Reusing the existing Feishu group-session resolver is the narrowest maintainable fix because it avoids duplicating topic/sender fallback logic and keeps routing and serialization aligned.
