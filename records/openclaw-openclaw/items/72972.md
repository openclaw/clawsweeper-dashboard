## Summary

Keep open for maintainer merge review: current main still over-serializes Feishu group-session scopes, and this PR is a narrow, consistent fix with no concrete correctness or security finding from this sweep.

## What This Changes

The PR threads Feishu account config into queue-key selection, reuses `resolveFeishuGroupSession` for ordinary group-message sequential keys, and adds Feishu coverage plus a changelog entry.

## Best Possible Solution

Merge this targeted Feishu fix after maintainer approval and exact-head CI, keeping `resolveFeishuGroupSession` as the single source of truth for group peer ids.

## Reproduction Assessment

Yes. Configure Feishu with `groupSessionScope: "group_topic"` or `"group_topic_sender"`, then send ordinary group messages in different topics or topic senders within one chat; current main computes the same chat-wide queue key before routing them to separate peers.

## Solution Assessment

Yes. Reusing the existing group-session resolver and threading resolved account config into queue-key selection is the narrowest maintainable fix, and it preserves existing `/stop` and `/btw` lanes.
