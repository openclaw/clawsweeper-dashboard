Full review comments:

- **[P2] Hydrate topic starters before choosing the queue key:** `extensions/feishu/src/sequential-key.ts:34-41`
  - body: This still derives the queue key directly from the raw receive event. Native Feishu `topic_group` starter events can arrive without `thread_id`, while `handleMessage` later hydrates that value for session routing; choosing the queue key here can therefore keep the starter on a different lane from later replies in the same topic. The replacement PR fixes this by making key resolution hydration-aware and forwarding the hydrated event into handling.
  - confidence: 0.87
