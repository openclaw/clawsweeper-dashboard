## Summary

Keep open: this is a focused Workboard bugfix, but the current branch can suppress valid task-backed lifecycle advancement when task freshness is missing or invalid, and the supplied proof still does not show the visible Control UI/Gateway flow.

## What This Changes

The PR adds a timestamp-aware Workboard lifecycle status guard, regression coverage for stale/fresh linked session and task sync, and explicit selected options in the Workboard edit modal.

## Best Possible Solution

Keep the timestamp-based lifecycle guard, but only skip status sync when source freshness is known and older than the card, then land with missing/invalid task timestamp tests and redacted real Control UI/Gateway proof.

## Reproduction Assessment

Yes for the introduced PR regression from source: Gateway task `updatedAt` is optional, the UI normalizer allows it to be absent or invalid, and the PR turns unknown freshness into `0` before the stale guard. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No; the timestamp-aware guard is the right layer for stale lifecycle rewrites, but it must treat unknown task freshness as the previous sync behavior instead of stale data.
