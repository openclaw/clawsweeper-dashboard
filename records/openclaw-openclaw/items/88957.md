## Summary

Keep open: the PR is aimed at a real Workboard persistence bug, but the current patch still breaks valid task-backed lifecycle sync when `task.updatedAt` is missing or unparseable, and the supplied proof is a mocked controller harness rather than real Control UI/Gateway behavior.

## What This Changes

The branch updates Workboard edit-modal select rendering and adds timestamp-aware lifecycle synchronization so stale linked session/task updates do not overwrite newer card edits or moves.

## Best Possible Solution

Apply the stale-status guard only when the lifecycle source has a parseable real timestamp, preserve prior task sync for unknown freshness, and add redacted real Control UI/Gateway proof.

## Reproduction Assessment

Yes for the PR-introduced regression from source: optional `TaskSummary.updatedAt` can become `0`, then the new freshness guard skips task status sync for persisted cards. I did not run a live browser/Gateway reproduction in this read-only review.

## Solution Assessment

No. The timestamp-aware guard is the right shape for stale linked-session overwrites, but unknown task freshness should preserve prior task lifecycle sync instead of being treated as stale.
