## Summary

Keep open: the Workboard persistence fix is useful, but the patch still treats missing or unparseable task timestamps as stale, which can suppress valid task-driven status sync, and the external PR still lacks real Control UI/Gateway proof.

## What This Changes

This PR changes Workboard Control UI select rendering and adds a timestamp-aware lifecycle guard with controller/view regressions so stale linked session/task updates do not overwrite newer card edits or moves.

## Best Possible Solution

Preserve explicit persisted card edits or moves only when the lifecycle source has a real older timestamp, keep task/session lifecycle sync as the default for unknown freshness, and add redacted real Control UI/Gateway proof.

## Reproduction Assessment

Yes for the PR-introduced regression from source: TaskSummary.updatedAt is optional, taskUpdatedAtValue() returns 0 for missing/unparseable values, and the new guard skips status sync when sourceUpdatedAt is below card.updatedAt. The linked browser issue has user steps, but I did not run a live browser/Gateway reproduction in this read-only review.

## Solution Assessment

No. The timestamp-aware guard is the right shape for stale linked-session overwrites, but applying it to task payloads without a reliable source timestamp is not the safest fix; unknown freshness should preserve the prior task lifecycle sync behavior.
