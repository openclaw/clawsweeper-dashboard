## Summary

Keep open: the branch targets the right Workboard race, but the in-memory manual-status override can suppress documented linked-session lifecycle sync for active work states and can behave differently after a Control UI reload.

## What This Changes

The PR adds transient Workboard UI state to preserve explicit edit/drag status changes from lifecycle rewrites, updates modal select option selection, and adds focused Workboard controller/view regressions.

## Best Possible Solution

Use a durable, timestamp-aware lifecycle guard that skips stale lifecycle writes older than the persisted card change while preserving the documented rule that todo/running cards keep following fresh linked-session lifecycle updates.

## Reproduction Assessment

Yes from source inspection for the status bounce: current Workboard rendering calls syncWorkboardLifecycle, which maps completed linked sessions to review and writes workboard.cards.update after a card move. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The in-memory Set is a plausible immediate mitigation, but it is not the best fix because it changes documented lifecycle semantics and disappears on reload; a persisted card/session ordering check would target stale lifecycle writes more directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Persist lifecycle overrides instead of using a transient Set:** `ui/src/ui/controllers/workboard.ts:1591-1592`
  - body: `manualStatusCardIds` is only populated in the current UI host and is checked before `shouldSyncCardStatus`, so after a status-changing move it can suppress every completed/failed lifecycle target until a target happens to equal the card status. That conflicts with `docs/plugins/workboard.md`, which says only manual review states stop auto-moving and moving back to `todo` or `running` resumes lifecycle sync; it also means reload/new-host behavior differs because the Set is empty. Please scope the guard to stale lifecycle writes, for example by comparing persisted card and linked session/task update times, instead of treating all status-changing moves as an in-memory manual override.
  - confidence: 0.86
