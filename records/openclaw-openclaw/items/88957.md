## Summary

Keep open: the PR is aimed at the right Control UI Workboard bug, but the current patch can suppress linked-session lifecycle updates after a position-only move and the contributor proof is tests-only.

## What This Changes

The PR adds Workboard controller/view regressions and introduces a UI-side manual status override so card edits, select values, and drag-to-running moves survive lifecycle sync.

## Best Possible Solution

Keep the focused Control UI fix, but only record a manual status override when the persisted move actually changes status, then add real browser proof for save/reopen and drag-to-running behavior.

## Reproduction Assessment

Yes for the review finding by source inspection: a same-status `moveWorkboardCard` call can mark the card manual, and the next lifecycle target can then be skipped. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No. The PR is close, but the manual override needs to be scoped to actual status changes so position-only moves do not disable lifecycle sync.
