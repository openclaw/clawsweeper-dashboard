## Summary

Keep open: the bugfix is useful, but this PR is not merge-ready because the new freshness guard can suppress task lifecycle sync for Gateway-valid payloads without a usable `updatedAt`, and the supplied controller-harness output does not prove the visible Control UI/Gateway flow.

## What This Changes

The branch adds timestamp-aware Workboard lifecycle status sync, regression coverage for stale/fresh session and task updates, and select-state rendering fixes in the Workboard edit modal.

## Best Possible Solution

Keep the timestamp-based Workboard lifecycle guard, but only skip status sync when source freshness is known and older than the card, then land with focused missing/invalid task timestamp coverage and redacted real Control UI/Gateway proof.

## Reproduction Assessment

Yes for the introduced PR regression from source: Gateway task `updatedAt` is optional, the UI normalizer allows it to be absent or invalid, and the PR turns unknown freshness into `0` before the stale guard. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No; the timestamp-aware guard is the right layer for stale lifecycle rewrites, but it must treat unknown task freshness as the previous sync behavior instead of stale data.
