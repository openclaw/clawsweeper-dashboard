## Summary

Keep open: current main has a useful partial fix from #77554 for explicit workspace model-refresh paths, but the remaining source path can still reject the single default-workspace snapshot and fall back to cold plugin metadata scans for nonmatching workspaces.

## What This Changes

The report asks OpenClaw to avoid repeated plugin metadata snapshot misses from workspaceDir mismatches on hot Gateway model/catalog/tool paths, especially in multi-agent deployments.

## Best Possible Solution

Add bounded workspace-aware metadata reuse, or explicitly pass compatible snapshots through the remaining catalog/model/tool flows, with coverage for default, non-default, and missing-workspace callers.

## Reproduction Assessment

Yes, at source level: startup stores one default-workspace snapshot, the getter rejects nonmatching workspace scopes, and loadManifestModelCatalog still falls back to loadPluginMetadataSnapshot on a miss. I did not run a live strace reproduction in this read-only sweep.

## Solution Assessment

No: the current #77554 slice is a good partial fix, but it does not fully solve the multi-agent/non-default workspace cache-miss path. The safer final solution is workspace-aware reuse or explicit snapshot threading that keeps workspace plugin isolation intact.
