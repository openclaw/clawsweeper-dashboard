## Summary

Keep open: current main still has the Mattermost slash-command split-state 503 bug, and this PR is the active fix path, but the current patch should not merge until it avoids putting resolved Mattermost account state on `globalThis` and fixes a stale test fixture.

## What This Changes

The PR splits Mattermost slash-route state and route code, moves slash callback account state behind a `Symbol.for` global map shared across loader paths, adds a dual-loader regression test, and updates the changelog.

## Best Possible Solution

Keep the slim route split, but make route registration and monitor activation share a single ESM-owned module-local state instance, avoid global secret-bearing state, update the new test fixture for current account types, and rerun Mattermost plus changed checks.

## Reproduction Assessment

Yes. Current main still loads the route through `loadBundledEntryExportSync` while monitor activation imports `slash-state.js` normally, and the route-side empty map returns the exact reported 503 response.

## Solution Assessment

No. The PR fixes the central dual-loader bug, but the current global-state implementation broadens access to sensitive Mattermost account state; the safer fix is a slim static ESM route boundary with module-local shared state.

  - confidence: 0.86
