## Summary

Keep open. Current main and the latest release still have the split Mattermost route-loader/state shape, and this PR targets a real 503 regression, but it still needs narrow entrypoint-boundary and release-note cleanup before merge.

## What This Changes

The PR replaces Mattermost's sync bundled-entry slash route loader with a static import of `./slash-route-api.js` in `extensions/mattermost/index.ts`.

## Best Possible Solution

Land a Mattermost-local fix that makes route registration and monitor activation share slash-command state across the sync bundled-entry and native ESM loader paths while preserving a lightweight entrypoint, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The PR body and linked #68113 provide concrete live steps, and current main source shows the route-side empty-state branch returns the reported 503 while monitor activation writes through a separate native `slash-state` import path.

## Solution Assessment

Unclear. The static import is a narrow functional fix for module identity, but a thinner shared-state or route/runtime seam would preserve the existing bundled-entry laziness contract with less startup/import-graph risk.
