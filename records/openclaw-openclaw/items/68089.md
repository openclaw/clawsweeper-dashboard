## Summary

Keep open: this is an active focused PR for a still-present Mattermost slash-command regression, and neither current main nor the latest tagged release contains the proposed static import or an equivalent shared-state fix. The branch is directionally useful, but it still needs small merge-readiness cleanup and a maintainer decision on the entrypoint import-topology tradeoff.

## What This Changes

The PR changes `extensions/mattermost/index.ts` to statically import Mattermost slash route registration instead of resolving it through `loadBundledEntryExportSync`.

## Best Possible Solution

Land a Mattermost-local fix that makes route registration and monitor activation share slash-command state across supported loader paths while preserving the lightweight entry boundary where practical, with changelog and regression coverage.

## Reproduction Assessment

Yes. The PR body and related bug provide concrete live reproduction steps, and current main source shows the route-side empty-state path returns the reported 503 while monitor activation writes to `slash-state` after registration.

## Solution Assessment

Unclear. The static import is a narrow functional fix for module identity, but a lighter shared state anchor or split route/state seam would better preserve the existing bundled-entry laziness contract.
