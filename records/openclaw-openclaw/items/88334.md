## Summary

Keep open: current main and the latest release still have the source-backed stale external-CLI auth path described here, and the related open auth-epoch issues do not fully cover this issue's requested pre-dispatch refresh/retry behavior or credential-rotation-specific user copy.

## What This Changes

Gateway should refresh or reset stale auth state immediately after credential rotation before processing inbound user messages.

## Best Possible Solution

Add a scoped external-CLI auth freshness or auth-failure retry path that refreshes Claude CLI auth before dispatch when the selected credential source changed, while preserving normal same-identity token rotation reuse.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only pass. Current source still shows the stale external-CLI credential path and the issue gives a concrete Mac Gateway timeline, but the exact host/keychain timing needs live proof.

## Solution Assessment

No, the current repository state is not the best final solution because the stale-auth window and generic fallback remain. The safer fix is a scoped auth-owner refresh/retry path rather than broad request-time credential polling.
