## Best Possible Solution

Add a scoped Claude/external-CLI credential freshness or auth-failure retry path before dispatch, covering file and Keychain sources, without broad request-time credential rediscovery.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows a plausible current-main path: keep a usable stored Claude CLI OAuth profile, rotate the external credential, then dispatch before the external CLI sync/cache path rereads it.

## Solution Assessment

Unclear as written. The requested outcome is right, but broad per-message freshness polling is not obviously the best fix under OpenClaw hot-path guidance; a scoped credential-owner freshness or auth-failure retry path is safer.
