## Summary

Keep open: current main and the latest release still have a source-backed Claude CLI stale-credential window, and adjacent open auth-epoch PRs do not implement immediate credential-rotation freshness or a credential-rotation-specific retry path.

## What This Changes

The issue asks Gateway to refresh Claude auth state immediately after credential rotation instead of waiting for heartbeat/session reset before processing inbound messages.

## Best Possible Solution

Add a scoped Claude/external-CLI credential freshness or auth-failure retry path before dispatch, covering file and Keychain sources where identity can be proven, without broad request-time credential rediscovery.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection still shows a plausible current-main stale window when Claude credentials rotate while a usable stored OAuth profile and reusable CLI session remain active.

## Solution Assessment

Unclear as written. The requested outcome is valid, but broad per-message file/keychain polling is not obviously the best fix under OpenClaw hot-path guidance; a scoped credential-owner freshness or auth-failure retry path is likely safer.
