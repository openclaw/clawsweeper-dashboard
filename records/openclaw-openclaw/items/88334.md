## Summary

Keep open: current main still has a source-backed stale-auth window for Claude CLI external auth, because scoped external CLI sync skips rereading a usable stored OAuth profile and the generic failure copy is still the visible fallback. The report is related to prior auth-epoch work, but it asks for a distinct pre-dispatch refresh/retry path after credential rotation.

## What This Changes

Credential rotation in Gateway should refresh auth epoch/auth state before handling inbound messages instead of letting stale credentials produce a generic /new failure.

## Best Possible Solution

Add a scoped external-CLI auth freshness/retry path that refreshes Claude CLI auth state before dispatch when the selected credential source changed or returns a credential-rotation-specific retry message, while preserving normal OAuth token-rotation session reuse.

## Reproduction Assessment

No high-confidence live reproduction was run, but the current source shows a plausible reproduction path: keep a usable stored Claude CLI OAuth profile, rotate the external Claude credential file, then dispatch before external CLI sync decides to reread it. The reported logs provide useful real-world timing, but the exact host/keychain conditions still need proof.

## Solution Assessment

Unclear as written: the requested outcome is correct, but the suggested per-inbound freshness polling needs owner review because AGENTS.md warns against broad request-time credential rediscovery. A scoped credential-source freshness or auth-failure retry path is the safer implementation direction.
