Keep open: current main still has a source-backed Claude CLI stale-auth path after external credential rotation, and the merged token-rotation fix plus the active auth-profile PR do not cover this exact pre-dispatch refresh/retry gap.

## What This Changes

The issue requests immediate Gateway handling for Claude CLI credential rotation so inbound messages refresh or retry auth instead of surfacing a generic `/new` failure.

## Best Possible Solution

Add a scoped external-CLI auth freshness or auth-failure retry path that refreshes Claude CLI auth before dispatch, or returns credential-rotation-specific retry copy, while preserving normal OAuth token-rotation session continuity.

No high-confidence live reproduction was run. Source inspection shows a plausible current-main path: keep a usable stored Claude CLI OAuth profile, rotate the external credential, then dispatch before the external CLI sync/cache path rereads it.

## Solution Assessment

Unclear as written. The requested outcome is right, but the suggested per-inbound freshness polling needs owner review; a scoped credential-source freshness or auth-failure retry path is safer than broad request-time rediscovery.
