## Summary

Keep open: current main and the latest shipped tag still lose explicit Feishu/Lark `replyTo` on the ordinary message-tool `send` path, and the overlapping Feishu reply-routing PRs are still open rather than merged fixes.

## What This Changes

The issue requests stable Feishu/Lark quoted-reply display when the message tool sends with an explicit `replyTo` target.

## Best Possible Solution

Land a corrected Feishu plugin action-layer fix that maps explicit ordinary-send `replyTo`/`reply_to` targets before topic auto-thread fallback while preserving topic/thread and normal-group reply semantics.

## Reproduction Assessment

Yes, source-level rather than live: call the Feishu plugin action handler for ordinary `send` with `params.replyTo` in a non-topic session. Core preserves the parameter, but `buildFeishuSendReplyAnchor` ignores it for `send`, so the Feishu delivery helpers receive no reply anchor.

## Solution Assessment

Yes, the best fix boundary is the Feishu plugin action layer because the generic tool contract and lower Feishu adapters already support reply targets. The in-flight PR path should be repaired or split so explicit `replyTo` is handled without changing unrelated reply-routing semantics.
