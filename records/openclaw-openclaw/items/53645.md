## Summary

Keep open: current main and the latest v2026.5.18 release still drop explicit Feishu `replyTo` targets on the ordinary message-tool `send` action path before the lower Feishu reply-capable send helpers can use them.

## What This Changes

Feishu message-tool replies using `replyTo` should reliably render native quoted replies after the OpenClaw 2026.3.23-2 regression.

## Best Possible Solution

Fix the Feishu plugin action layer so ordinary `send` maps explicit `replyTo`/`reply_to` and direct `replyToId` into inline reply targets for text, card, and media sends while preserving `thread-reply` and topic auto-thread semantics.

## Reproduction Assessment

Yes, at source level: call the Feishu plugin action handler for ordinary `send` with `params.replyTo` or a core-applied implicit `replyTo`, and `buildFeishuSendReplyAnchor` will not forward it unless the session is topic auto-threading. I did not run a live Feishu tenant reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/feishu/src/channel.ts`, where the action reply-anchor helper should honor explicit ordinary-send reply targets before falling back to topic auto-threading.
