## Summary

Keep open: current main and the latest shipped v2026.5.18 code still drop explicit Feishu `replyTo` targets on the ordinary message-tool `send` path before the lower Feishu send helpers can use them.

## What This Changes

The issue asks Feishu ordinary message replies using `replyTo` to consistently show quoted-reply context after OpenClaw 2026.3.23-2.

## Best Possible Solution

Fix the Feishu plugin action layer so ordinary `send` maps explicit `replyTo`/`reply_to` and direct `replyToId` to inline reply targets for text, card, and media sends while preserving `thread-reply` and topic auto-thread semantics.

## Reproduction Assessment

Yes, at source level: ordinary Feishu `send` with `params.replyTo` reaches the action handler, but `buildFeishuSendReplyAnchor` ignores it unless the session is topic auto-threading. I did not run a live Feishu tenant reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is to update `extensions/feishu/src/channel.ts` so the action reply-anchor helper honors explicit ordinary-send reply targets before falling back to topic auto-threading, with regression coverage for text, card, and media sends.
