## Summary

Keep open: current main and the latest release still drop explicit Feishu `replyTo` targets on ordinary message-tool `send` actions before they reach the Feishu send helpers. Core exposes and resolves `replyTo`, and the lower Feishu outbound adapter can send it, so this remains a narrow Feishu plugin mapping bug.

## What This Changes

The issue asks for Feishu message-tool replies using `replyTo` to consistently render as quoted replies after OpenClaw 2026.3.23-2.

## Best Possible Solution

Fix the Feishu plugin action layer so ordinary `send` maps explicit and core-applied `replyTo` into inline reply targets for text, card, and media sends while preserving `thread-reply`, topic auto-threading, and existing fallback semantics.

## Reproduction Assessment

Yes, at source level: call the Feishu plugin action handler for ordinary `send` with `params.replyTo`, and `buildFeishuSendReplyAnchor` will not forward it unless topic auto-threading supplies a different id. I did not run a live Feishu tenant reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/feishu/src/channel.ts`, because core already resolves `replyTo` and the lower Feishu outbound adapter already forwards reply targets.
