## Summary

Keep open: current main still exposes the generic `replyTo` parameter, but the Feishu plugin-local `send` action derives reply anchors only from explicit `thread-reply` or topic auto-threading, so ordinary `send` calls with explicit `replyTo` can still lose the quoted-reply target.

## What This Changes

Requests stable Feishu/Lark quoted replies when the `message` tool uses `replyTo` after OpenClaw 2026.3.23-2.

## Best Possible Solution

Coordinate a narrow Feishu plugin action fix that maps explicit ordinary `send` reply targets into the existing `replyToMessageId`/`replyToId` paths, while preserving topic auto-threading and validating against the active Feishu reply PRs.

## Reproduction Assessment

Yes at source level: call the Feishu plugin `send` action with an explicit `replyTo`/`replyToId`; current main resolves the generic reply target but the Feishu action helper ignores that parameter before sending. A live Feishu rendering check was not performed.

## Solution Assessment

Yes for the narrow fix shape: the durable lower adapters already know how to send `replyToId`, so the maintainable repair is to map the explicit parameter in the Feishu action layer and add focused tests. It should be coordinated with the active Feishu reply/thread PRs before another branch is opened.
