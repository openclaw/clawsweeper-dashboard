## Summary

Keep open: current main and the latest shipped tag still drop explicit Feishu `replyTo` on the ordinary message-tool `send` action before the lower Feishu send adapters can create a quoted reply, and overlapping Feishu routing PRs are still open rather than merged.

## What This Changes

Report that Feishu/Lark message-tool replies with `replyTo` intermittently lose visible quoted-reply anchors after OpenClaw 2026.3.23-2.

## Best Possible Solution

Land one corrected Feishu plugin action-layer fix, likely by repairing or superseding https://github.com/openclaw/openclaw/pull/78964, so ordinary `send` maps explicit `replyTo`/`reply_to`/`replyToId` into Feishu reply anchors while preserving topic-thread and normal-group safeguards.

## Reproduction Assessment

Yes, at source level: call the Feishu plugin action handler for ordinary `send` with `params.replyTo` or `reply_to` in a non-topic session, and the current helper returns no `replyToMessageId` before delivery. I did not run a live Feishu tenant replay in this read-only review.

## Solution Assessment

Yes, the best fix boundary is the Feishu plugin action layer. It should map explicit ordinary-send reply targets before topic auto-thread fallback and add regressions for text, card, and media sends.
