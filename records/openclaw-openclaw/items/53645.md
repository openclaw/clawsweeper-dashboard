## Summary

Keep open. Current main still has the Feishu plugin action-layer gap described by the report: the generic message tool can carry a `replyTo` target and core resolves implicit reply targets, but Feishu ordinary `send` still passes no reply target unless the action is `thread-reply`.

## What This Changes

The issue asks for Feishu WebSocket message-tool replies using `replyTo` to consistently display quoted-message references after OpenClaw 2026.3.23-2.

## Best Possible Solution

Fix this in the Feishu plugin action layer: ordinary `send` should derive the explicit or core-resolved reply target from the action params and pass it as `replyToMessageId` for text/card sends and `replyToId` for media sends, while preserving `thread-reply` as the explicit `reply_in_thread` path.

## Reproduction Assessment

Yes. A focused source-level reproduction is to call the Feishu plugin `send` action with `replyTo`; current `channel.ts` sets `replyToMessageId` to `undefined` for ordinary sends, and `channel.test.ts` currently expects that undefined value.

## Solution Assessment

Yes. The narrowest maintainable fix is in `extensions/feishu/src/channel.ts` because core already resolves `replyTo` and the lower Feishu adapters already know how to send quoted replies once they receive the target.
