## Summary

Keep open: current main and the latest v2026.5.18 release still drop ordinary Feishu `send` action `replyTo` targets before the lower reply-capable Feishu send helpers can use them. An open overlapping PR may cover this path, but it is not merged and does not close this issue.

## What This Changes

The issue reports intermittent missing Feishu quoted replies when the message tool uses `replyTo` after upgrading to OpenClaw 2026.3.23-2.

## Best Possible Solution

Land a focused Feishu plugin action-layer fix, either by reviewing and landing the overlapping work in https://github.com/openclaw/openclaw/pull/78964 or by splitting a narrow patch that maps ordinary `send` `replyTo` targets while preserving thread and topic semantics.

## Reproduction Assessment

Yes. At source level, call the Feishu plugin action handler for ordinary `send` with `params.replyTo` in a non-topic session: core preserves the target in params, but `buildFeishuSendReplyAnchor` returns no reply target before delivery.

## Solution Assessment

Yes. The narrow maintainable fix is to make `extensions/feishu/src/channel.ts` honor explicit ordinary-send reply targets before falling back to topic auto-threading, with focused regressions for text, card, and media sends.
