## Summary

Keep open: current main and v2026.5.18 still drop explicit Feishu `replyTo` targets on ordinary message-tool `send`, while the overlapping Feishu anchor PR is open and not merged.

## What This Changes

The issue reports intermittent missing Feishu quoted-reply context when agents send messages with the `replyTo` parameter after upgrading to OpenClaw 2026.3.23-2.

## Best Possible Solution

Review and land, or split from, https://github.com/openclaw/openclaw/pull/78964 so the Feishu plugin action layer maps explicit ordinary-send `replyTo` targets while preserving topic and thread semantics.

## Reproduction Assessment

Yes. At source level, call the Feishu plugin action handler for ordinary `send` with `params.replyTo` in a non-topic session: core preserves the target in params, but `buildFeishuSendReplyAnchor` returns no reply target before delivery.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/feishu/src/channel.ts`: honor explicit ordinary-send reply targets before falling back to topic auto-threading, with regressions for text, card, and media sends.
