## Summary

Keep open. Current main still loses explicit Feishu `replyTo` on the ordinary message-tool `send` path: core resolves the reply target and the lower Feishu adapters can send quoted replies, but the Feishu plugin action helper only forwards `thread-reply` or topic auto-thread anchors.

## What This Changes

The issue requests stable Feishu quoted replies when the message tool uses `replyTo` after the OpenClaw 2026.3.23-2 upgrade.

## Best Possible Solution

Fix the Feishu plugin action layer so ordinary `send` maps explicit and core-resolved `replyTo`/`reply_to` into inline reply targets while preserving `thread-reply` and topic auto-thread semantics.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: call the Feishu plugin action handler for ordinary `send` with `params.replyTo`; current main's reply-anchor helper ignores it and passes `undefined` to text/card/media sends unless the session is a topic auto-thread.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/feishu/src/channel.ts`, because core already resolves `replyTo` and the lower Feishu outbound/send helpers already support quoted replies.
