## Summary

Keep open. Current main still has the Feishu plugin action-layer gap described in the issue: the generic message tool exposes and resolves replyTo, and the lower Feishu outbound/send helpers can send quoted replies, but the Feishu ordinary send action still only sets a reply target for thread-reply.

## What This Changes

The issue asks for Feishu message-tool replies using replyTo to consistently display quoted replies after OpenClaw 2026.3.23-2.

## Best Possible Solution

Fix this in the Feishu plugin action handler so ordinary send maps explicit and resolved reply targets into replyToMessageId for text/card sends and replyToId for media sends, while preserving thread-reply as the explicit reply_in_thread path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call the Feishu plugin send action with replyTo and observe that current main still computes replyToMessageId only for thread-reply, with existing action tests asserting undefined reply fields for ordinary sends.

## Solution Assessment

Yes. The narrow maintainable fix belongs in extensions/feishu/src/channel.ts because core already resolves replyTo and the lower Feishu outbound/send adapters already know how to send quoted replies.
