## Summary

Keep open: current main still drops explicit Feishu `replyTo` targets on the ordinary message-tool `send` action path, while the generic message tool and lower Feishu send adapters already support reply targets.

## What This Changes

The issue asks for Feishu quoted replies sent with the message tool `replyTo` parameter to render reliably instead of intermittently appearing as ordinary messages.

## Best Possible Solution

Fix the Feishu plugin action layer so ordinary `send` maps explicit and core-applied `replyTo` into inline reply targets for text, card, and media sends while preserving `thread-reply`, topic auto-threading, and existing fallback semantics.

## Reproduction Assessment

Yes, at source level: call the Feishu plugin action handler for ordinary `send` with `params.replyTo` or a core-applied implicit `replyTo`, and `buildFeishuSendReplyAnchor` will not forward it unless the session is topic auto-threading. I did not run a live Feishu tenant reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is in `extensions/feishu/src/channel.ts`, where the action reply-anchor helper should honor explicit ordinary-send reply targets before falling back to topic auto-threading.
