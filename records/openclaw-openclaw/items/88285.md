## Summary

## What This Changes

The PR moves cron jobs, runtime state, and run history from legacy JSON/JSONL files into the shared OpenClaw SQLite state database, with doctor migration, schema repair, docs, and regression tests.

## Best Possible Solution

Land only after the normal upgrade path preserves or imports legacy cron jobs and run logs, stale runtime-state and failure-alert regressions are fixed, and a redacted real upgrade/readback proof artifact is attached or maintainers explicitly approve a narrower doctor-gated compatibility break.

## Solution Assessment

No; moving cron persistence to SQLite is a reasonable direction, but this implementation is not yet the safest upgrade path because normal legacy reads, stale schedule state, and run-history visibility are not preserved. The maintainable solution is either first-read import/preservation with proof, or an explicit maintainer-approved doctor-only migration contract.
