- **[P1] Preserve cron.runLog.maxBytes pruning after the SQLite move:** `src/cron/run-log.ts:386-390`
  - body: This write path now prunes SQLite rows only by `keepLines`; `opts.maxBytes` is still parsed but never applied to new SQLite history. Please preserve the byte cap or make the incompatible setting change an explicit maintainer-approved migration with tests and proof.
  - confidence: 0.89

  - body: After this branch imports a legacy store, this line archives `jobs.json`; `diagnostic-session-context` still only reads that JSON file to populate `cronJobName`. Once the normal SQLite path no longer has the JSON store, stopped/stalled cron diagnostics lose the human job name even though the row is present in SQLite. Please update the diagnostic reader to use the canonical SQLite-backed cron store before retiring the file.
