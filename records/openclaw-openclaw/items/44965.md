## Summary

Keep open. Current main still lacks a stream-level assistant text repetition circuit breaker, and the related implementation PR is open rather than shipped; the requested halt, alert, and confirm/resume behavior needs maintainer product/API judgment.

## What This Changes

Request to add a core streaming safeguard that detects repeated assistant output, halts the stream, alerts the user, and requires confirmation before continuing or aborting.

## Best Possible Solution

Review PR #39961 or a replacement as the canonical path, then land a conservative configurable guard at the embedded assistant stream boundary with provider-safe abort behavior, buffered-output suppression, documented alert/cancel semantics, and an explicit decision on whether resume is supported.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The high-confidence absence check is source-backed: repeated assistant text_delta chunks still flow through append, emit, and final-flush paths without a text repetition threshold or halt-and-confirm state.

## Solution Assessment

Unclear. A core stream guard appears to be the right surface, but halt, alert, cancel, and resume semantics need maintainer judgment because current code supports aborting runs and tool approvals but does not show a resumable assistant-stream approval flow.
