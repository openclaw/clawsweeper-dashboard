## Summary

Keep open: current main and v2026.6.10 still lack a semantic streamed-text repetition guard; existing safeguards cover tool-call loops, post-compaction tool repeats, and stream-idle silence rather than active repeated assistant text. The prior implementation PR was closed unmerged as non-merge-ready, and related duplicate reports point back to this issue as the canonical tracker, so this needs maintainer product direction rather than cleanup closure.

## What This Changes

Requested a stream-level repetition circuit breaker that detects repeated assistant text, halts output, alerts the user, and waits for manual resume or cancel confirmation.

## Best Possible Solution

Design a core stream-content repetition circuit breaker in the embedded-agent stream path with replay-safe abort/session cleanup and a maintainer-approved user recovery policy, while keeping https://github.com/openclaw/openclaw/issues/8724 separate for total-generation timeout configuration.

## Reproduction Assessment

No live current-main reproduction was run. Source inspection gives a medium-confidence reproduction of the missing capability: current main and v2026.6.10 stream and emit assistant text without any semantic repetition detector, while related user reports preserve concrete loop examples.

## Solution Assessment

Unclear whether the requested halt-and-confirm flow is the best final shape. A stream-level guard belongs in core if adopted, but maintainers still need to decide default behavior, thresholds, abort cleanup, and whether manual confirmation should be required before implementation.
