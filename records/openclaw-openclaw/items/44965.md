## Summary

Keep open. Current main still lacks a core assistant-output repetition circuit breaker, and the closest matching implementation is open and unmerged, so there is no implemented-on-main or duplicate close path yet.

## What This Changes

Request a core safeguard that detects repeated assistant output during streaming, halts the model stream, alerts the user, and requires manual confirmation before continuing.

## Best Possible Solution

Review PR #39961 or a replacement as the canonical implementation path, then land a conservative core guard at the assistant stream subscription or runner boundary with documented config, provider-safe abort semantics, suppression of final-flush leakage, and clear user-facing stop/cancel behavior.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The high-confidence absence check is source-backed: repeated assistant text_delta chunks currently flow through append/emit paths without any repetition threshold or halt-and-confirm state.

## Solution Assessment

Unclear. A core stream guard is the right area, but the exact halt-and-confirm design needs maintainer judgment because aborting a provider stream is supported while resuming the same stream after approval is not shown in the current code.
