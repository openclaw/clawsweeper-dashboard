## Summary

Keep open. Current main still lacks a core assistant-output repetition circuit breaker, and the related implementation PR is open rather than shipped; the halt-and-confirm behavior also needs maintainer product/API judgment.

## What This Changes

The issue requests a core stream-level safeguard that detects repeated assistant output, halts the LLM stream, alerts the user, and requires approval or cancellation before continuing.

## Best Possible Solution

Review PR #39961 or a replacement as the canonical path, then land a conservative core guard at the assistant stream subscription or runner boundary with documented config, provider-safe abort behavior, final-flush suppression, and clear stop/cancel messaging.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The high-confidence absence check is source-backed: repeated assistant text_delta chunks still flow through append, emit, and final-flush paths without any text repetition threshold or halt-and-confirm state.

## Solution Assessment

Unclear. A core stream guard appears to be the right surface, but the exact halt, alert, cancel, and resume semantics need maintainer judgment because current code supports aborting sessions but does not show a resumable provider stream approval flow.
