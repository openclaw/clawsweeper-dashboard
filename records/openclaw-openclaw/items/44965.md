## Summary

Keep this open: current main still lacks a semantic text-stream repetition guard, the only related implementation PR is closed unmerged, and prior duplicate reports were closed back to this issue as the canonical tracker.

## What This Changes

The issue requests a core stream-level repetition circuit breaker that halts repeated LLM output, alerts the user, and requires confirmation before resuming or cancelling.

## Best Possible Solution

Keep this as the canonical tracker and design a bounded core streaming/session guard with replay-safe abort, no buffered spam leak, clear user alerting, and focused regression coverage before any implementation PR lands.

## Reproduction Assessment

No live current-main model-loop reproduction was run. Source inspection gives a medium-confidence reproduction of the missing guard path: repeated text or thinking chunks would continue through the current stream handler while the idle watchdog treats each chunk as activity.

## Solution Assessment

Unclear without maintainer direction. A stream-level guard is plausible and current main lacks it, but the requested halt-and-confirm flow is not automatically the best shape until maintainers choose the abort, history-sanitization, thinking-stream, and user-confirmation semantics.
