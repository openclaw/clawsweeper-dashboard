## Summary

Keep open. Current main still lacks the requested assistant-output repetition circuit breaker, the canonical implementation PR remains open rather than merged, and the halt-and-confirm behavior needs maintainer product/API judgment.

## What This Changes

Request a core stream-level safeguard that detects repeated assistant output, halts the stream, alerts the user, and requires approval or cancellation before continuing.

## Best Possible Solution

Review the open implementation candidate or a replacement and land a conservative, schema-backed stream guard that suppresses buffered repeated output, aborts safely, documents defaults/config, and treats resume confirmation as explicit product design.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives a high-confidence absence check: current main streams assistant text chunks through buffers and emits without a text-repetition threshold, abort flag, or confirmation state.

## Solution Assessment

Unclear. A stream-level guard at the assistant subscription or runner boundary appears plausible, but pause-and-resume approval behavior is not established by the current stream/session API, so maintainers need to decide abort, alert, config, and resume semantics before landing.
