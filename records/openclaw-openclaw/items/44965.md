## Summary

Keep open: current main and v2026.6.6 still lack a semantic streamed-text repetition guard, while the existing safeguards cover tool-call loops, duplicate stream events, or idle silence rather than active repeated assistant text. The related implementation PR was closed unmerged, so this issue remains the canonical tracker and needs maintainer product direction before implementation.

## What This Changes

The issue requests a core streaming safeguard that detects repeated generated text, halts the stream, alerts the user, and requires manual confirmation before resuming.

## Best Possible Solution

Keep this as the canonical tracker and design a bounded core stream/session guard with replay-safe abort, no buffered spam leak, clear user alerting, transcript cleanup policy, and focused current-main regression coverage before implementation lands.

## Reproduction Assessment

Yes at source level: current main streams assistant text without semantic repetition detection, while the existing loop detector is tool-call scoped. No live model-loop reproduction was run in this read-only review.

## Solution Assessment

Unclear. A stream-level guard is plausible and current main lacks it, but the halt-and-confirm flow needs maintainer direction on detection scope, abort behavior, false positives, and history cleanup.
