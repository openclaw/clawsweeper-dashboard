## Summary

Keep open. Current main still builds the CLI JSON result from raw CLI output and delivers those payloads unchanged; the hook-finalized assistant value is not shared back into `agent --json`, and the related #56031 PR is scoped to the pi embedded runner, not this CLI surface.

## What This Changes

The issue requests that `agent --json` return the same post-`before_message_write` assistant text that is finalized for the session transcript/UI.

## Best Possible Solution

Compute one finalized CLI assistant message with the existing synchronous hook semantics, use that value for transcript persistence, `payloads`, and `meta.finalAssistantVisibleText`, and cover replacement plus block cases with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: register a synchronous `before_message_write` hook that rewrites or blocks the final assistant message, run `openclaw agent --json`, then compare the JSON `payloads[].text` and transcript/finalized assistant behavior.

## Solution Assessment

Yes. The narrow maintainable fix is to share a single finalized assistant value in the CLI result path; #56031 is useful related work but is not sufficient because it only changes the pi embedded runner `--expect-final` path.

## Repair Work Prompt

Fix the CLI `agent --json` hook-finalization bug described in this issue. On current main, `buildCliRunResult` builds `payloads` and `meta.finalAssistantVisibleText` from raw CLI output, while the `before_message_write` semantics are not shared back into the JSON result path. Implement a narrow fix that computes the final assistant message once for CLI turns, applies the synchronous `before_message_write` replacement/block behavior through the existing hook seam or an equivalent shared helper, and reuses the finalized value for transcript persistence, `payloads`, `meta.finalAssistantVisibleText`, and pending final delivery text. Do not add a new config flag or change unrelated embedded-runner behavior; #56031 is related pi-runner work and should not be treated as covering this CLI path. Add focused regression coverage for a hook that replaces the final assistant text and a hook that blocks it, asserting JSON output and stored transcript behavior cannot diverge. Add a user-facing changelog fix entry unless maintainers decide this is test-only.
