## Summary

Keep open. Current main still has a source-level CLI result/transcript mismatch, and the related open PR is only a partial, stale candidate rather than a merged fix.

## What This Changes

The issue reports that `agent --json` can return raw pre-hook assistant text while the persisted transcript contains the `before_message_write`-finalized assistant message.

## Best Possible Solution

Make the CLI JSON result consume the finalized assistant message actually accepted by the write-hook path, with explicit rewrite/block semantics and focused regression coverage.

## Reproduction Assessment

Yes at source level: current main applies `before_message_write` on the transcript append path, then builds `agent --json` payloads from `output.text`. I did not run a live CLI/plugin reproduction in this read-only review.

## Solution Assessment

Unclear. A targeted CLI fix is plausible, but maintainers should first decide whether the JSON result should read the post-write transcript or thread finalized hook output, especially for blocked messages.
