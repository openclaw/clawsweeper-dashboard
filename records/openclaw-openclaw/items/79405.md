## Summary

Keep open: the PR still has a P1 wiring blocker because `allowCompletionFallback` is accepted at the public delivery boundary but is not forwarded into the direct helper that gates fallback sending, and the supplied proof does not show the primary-failure-to-fallback path.

## What This Changes

The PR adds task-aware subagent completion fallback delivery, taskId preservation, new task delivery statuses, and lifecycle tests around give-up fallback paths.

## Best Possible Solution

Forward the fallback flag into `sendSubagentAnnounceDirectly`, add boundary coverage for `deliverSubagentAnnouncement`, then provide redacted runtime proof of primary failure followed by fallback send.

## Reproduction Assessment

Yes. Source inspection on PR head is enough: call `deliverSubagentAnnouncement` with `allowCompletionFallback: true` and a failing primary direct announce, and the direct helper still receives the flag as absent.

## Solution Assessment

No. The intended narrow fix is plausible, but the current patch omits the boundary handoff, so the fallback path remains disabled until that wiring and proof are added.
