  - body: The new fallback sends completion text directly to any resolved channel/thread target and does not pass `agentId` or `mirror`. Current main gates this recovery path to direct messages and mirrors the send into requester session state, so a naive rebase can misroute group/thread replies or leave retries unaware that visible delivery already happened.
  - confidence: 0.86

A rebased version must update the parser/store/audit surface or avoid persisting new status strings, otherwise upgraded checkouts can fail when those values are read.
  - confidence: 0.82

  - body: The task-aware context path returns `false` on missing task id or missing task requester origin after requester-origin delivery has already been resolved from the requester session. Task status context should enhance completion recovery, not suppress the existing requester-origin fallback path.
