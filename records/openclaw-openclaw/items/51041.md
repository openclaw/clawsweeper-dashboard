## Summary

Keep open. Current main and the latest release still expose only the generic plugin command context plus a single returned reply payload, so Discord plugin commands still cannot choose a private initial slash response, public follow-up, or edit of the original interaction.

## What This Changes

The issue requests Discord-specific interaction response controls for native plugin command handlers while preserving existing command authorization and routing.

## Best Possible Solution

Add a narrow typed Discord responder facade to authorized native plugin command handlers, keep raw Discord interaction internals private, and preserve the existing returned ReplyPayload path for older plugins.

## Reproduction Assessment

Yes, at source level. Current main constructs PluginCommandContext without a Discord responder and delivers only the handler's returned payload after execution, so a plugin cannot interleave defer, mixed-visibility follow-up, or edit calls.

## Solution Assessment

Unclear until maintainers approve the public API and lifecycle shape. The requested direction is valid, but the maintainable solution should be a small typed responder facade rather than raw Discord interaction access.
