## Summary

Keep open. Current main still gives Discord plugin command handlers only the generic PluginCommandContext and a single returned ReplyPayload; Discord now defers slash commands up front, but plugin code still cannot choose mixed ephemeral/public follow-ups or edit the original interaction response.

## What This Changes

Request to add Discord interaction responder controls to native plugin command handlers so plugins can defer, reply, follow up, and edit slash-command responses.

## Best Possible Solution

Add a narrow typed Discord responder facade to authorized native plugin command handlers, preserving the existing returned ReplyPayload path and keeping raw Discord interaction internals private.

## Reproduction Assessment

Yes, at source level. Current main constructs PluginCommandContext without a Discord responder and delivers only the handler's returned payload after execution, so a plugin cannot interleave defer, mixed-visibility follow-up, or edit calls.

## Solution Assessment

Unclear until maintainers approve the public API and lifecycle shape. The requested direction is valid, but the maintainable solution should be a small typed responder facade rather than raw Discord interaction access.
