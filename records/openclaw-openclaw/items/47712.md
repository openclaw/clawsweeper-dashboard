## Summary

Keep open: current main still has no public `messages.tts.filterEmoji` contract and no shared emoji filtering before TTS synthesis; existing emoji stripping is provider-local and does not cover the requested core TTS behavior.

## What This Changes

Add `messages.tts.filterEmoji` support so TTS can skip emoji before speech synthesis.

## Best Possible Solution

Add a shared TTS text-preparation helper controlled by a documented `messages.tts.filterEmoji` contract, apply it consistently before synthesis, and keep visible chat/transcript content unchanged.

## Reproduction Assessment

Yes, by source inspection. The strict TTS schema/type/docs omit `filterEmoji`, auto-TTS only strips markdown, and one-off TTS entry points pass request text to synthesis without a shared emoji filter.

## Solution Assessment

Yes on implementation shape, but the default needs maintainer confirmation. A shared TTS preparation helper is narrower and more maintainable than adding more provider-local emoji stripping.
