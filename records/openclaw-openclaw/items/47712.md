## Summary

Keep open: current main still has no public `messages.tts.filterEmoji` contract and no shared emoji filtering before TTS synthesis; the only emoji stripping I found is provider-local and does not cover the requested core TTS behavior.

## What This Changes

Request to add a `messages.tts.filterEmoji` option so TTS can remove emoji before speech generation, defaulting to enabled.

## Best Possible Solution

Add a shared TTS text-preparation helper controlled by `messages.tts.filterEmoji?: boolean`, document the default, apply it to auto-TTS, the agent `tts` tool, and `tts.convert`, and keep provider-local sanitizers consistent with that shared behavior.

## Reproduction Assessment

Yes, by source inspection. The strict TTS schema/type/docs omit `filterEmoji`, auto-TTS only strips markdown, and one-off TTS entry points pass request text to synthesis without a shared emoji filter.

## Solution Assessment

Yes. A shared TTS preparation helper plus schema/type/docs coverage is narrower and more maintainable than adding more provider-local emoji stripping.
