The branch adds an optional `BuildToolPlanOptions.onHiddenDiagnostic` callback, invokes it for diagnostics that hide tools, guards malformed non-array `allOf`/`anyOf` evaluator inputs, and extends planner/evaluator tests.

## Best Possible Solution

Land one canonical fix that keeps the evaluator pure and surfaces malformed availability diagnostics once at the plugin descriptor registration or resolver boundary; keep the callback only if maintainers want that public API.
