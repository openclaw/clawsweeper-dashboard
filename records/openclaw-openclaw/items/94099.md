This PR adds an optional `BuildToolPlanOptions.onHiddenDiagnostic` callback, invokes it for hidden tool diagnostics, and adds defensive non-array `allOf`/`anyOf` availability checks plus tests.

## Best Possible Solution

Surface `unsupported-signal` authoring diagnostics once at the descriptor-registration or plugin/tool resolution boundary while keeping the evaluator pure; keep the callback only if a real owner wires and documents it.

## Reproduction Assessment

Yes. Current-main source shows empty availability groups return `unsupported-signal` diagnostics and `buildToolPlan` only records them in `hidden`; search found no production planner caller that would emit them.

## Solution Assessment

No. A callback in `buildToolPlan` is a plausible helper, but the best fix is to surface malformed descriptor diagnostics at a lifecycle-owned registration or plugin resolver boundary before claiming the linked bug is fixed.
