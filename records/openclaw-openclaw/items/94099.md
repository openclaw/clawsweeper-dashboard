## Summary

Keep open: this branch is a plausible partial helper, but it does not satisfy the linked runtime diagnostic bug because current main has no production `buildToolPlan` caller and plugin descriptor capture still drops plugin-authored availability before registration diagnostics can be emitted.

## What This Changes

This PR adds an optional `onHiddenDiagnostic` callback to `BuildToolPlanOptions`, invokes it for hidden availability diagnostics, guards non-array `allOf`/`anyOf`, and adds focused tests.

## Best Possible Solution

Land one canonical fix that keeps the evaluator pure and surfaces malformed availability diagnostics once at the plugin descriptor registration or resolver boundary; keep the planner callback only if maintainers want that public API.

## Reproduction Assessment

Yes. Source inspection shows empty availability groups produce `unsupported-signal` diagnostics, current `buildToolPlan` only records them in `hidden`, and current plugin descriptor capture does not preserve `availability` for registration-time visibility.

## Solution Assessment
