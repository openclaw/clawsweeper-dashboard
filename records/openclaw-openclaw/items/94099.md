## Summary

Keep this PR open, but it is not merge-ready: the patch adds a planner callback and evaluator guards, yet the linked bug asks for visible descriptor-registration/startup diagnostics and current main still has no production caller or plugin-registration sink for the new callback.

## What This Changes

This PR adds a `BuildToolPlanOptions.onHiddenDiagnostic` callback, calls it for hidden availability diagnostics, guards non-array `allOf`/`anyOf` evaluator inputs, and updates planner/evaluator tests.

## Best Possible Solution

Surface malformed availability diagnostics at the plugin descriptor registration or resolver boundary while keeping the evaluator pure; keep the planner callback only if maintainers want it as a supported API.

## Reproduction Assessment

Yes. Source inspection shows empty availability groups return `unsupported-signal` diagnostics, `buildToolPlan` only records them in `hidden`, and current plugin descriptor capture does not preserve `availability` for registration-time visibility.

## Solution Assessment

No. The callback is a plausible helper, but without a production caller or registration sink it is not the complete fix for the linked runtime diagnostic issue.
