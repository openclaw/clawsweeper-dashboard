## What This Changes

This PR adds an optional `BuildToolPlanOptions.onHiddenDiagnostic` callback, invokes it for hidden diagnostics, adds non-array `allOf`/`anyOf` availability guards, and updates tool planner/availability tests.

## Best Possible Solution

Surface `unsupported-signal` authoring diagnostics once at the plugin descriptor registration or resolver boundary while keeping the evaluator pure; keep the planner callback only if a real owner wires and documents it.

## Reproduction Assessment

Yes. Current-main source shows empty availability groups return `unsupported-signal` diagnostics and `buildToolPlan` only records them in `hidden`; search found no production planner caller that would emit them.

## Solution Assessment

No. A planner callback is a plausible helper, but without a production caller or registration sink it is not the narrowest complete fix for the linked runtime diagnostic issue.
