Keep open: current main and the latest release still do not expose a Gateway-received timestamp in generic inbound or agent-visible metadata. ## What This Changes

Add an agent-visible inbound metadata field such as `receivedAt` or `gatewayReceivedAt` that records when the Gateway receives an inbound message.

## Best Possible Solution

Define one additive generic ingress timing contract, reuse existing internal `receivedAt` where available, and expose it consistently through inbound context, agent-visible metadata, docs, and focused tests.
