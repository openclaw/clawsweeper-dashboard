## Summary

Keep open: current main has receive-context `receivedAt` groundwork for ack paths, but shared inbound metadata, hooks, direct-DM, and channel-turn surfaces still expose only the source timestamp, so the requested observability capability remains missing.

## What This Changes

Add an agent-visible Gateway receipt timestamp to inbound metadata so latency can be split from source-channel message time.

## Best Possible Solution

Add a backward-compatible receipt timestamp stamped once at Gateway or channel ingress, preserve `Timestamp` as source-channel time, and thread the new value through finalized context, prompt metadata, hooks, direct-DM, channel-turn SDK contracts, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still lacks an agent-visible Gateway receipt timestamp in the shared inbound metadata path.

## Solution Assessment

Yes. An additive optional receipt-time field is narrower than repurposing the existing source-channel `Timestamp`; the current receive-context `receivedAt` is useful groundwork but does not satisfy the shared metadata request.
