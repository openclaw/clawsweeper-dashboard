## Summary

Keep open. Current main still lacks a shared, agent-visible Gateway receipt timestamp; only the source-channel `Timestamp` is threaded through inbound context, prompt metadata, hook events, channel-turn SDK docs, and direct-DM helpers, while Telegram's `receivedAtMs` remains local debug/buffering state.

## What This Changes

The issue requests an agent-visible Gateway receipt timestamp in inbound metadata so latency can be split between source-channel delivery time and OpenClaw processing time.

## Best Possible Solution

Add a backward-compatible receipt timestamp stamped at channel/Gateway ingress, preserve `Timestamp` as source-channel time, thread the new value through finalized inbound context, user metadata, hooks, direct-DM and channel-turn SDK contracts, tests, and docs.

## Reproduction Assessment

Not applicable as a feature request; source inspection is high confidence that current main lacks the requested shared `receivedAt` or `gatewayReceivedAt` contract. Targeted searches and file reads show only source `Timestamp` is exposed, plus Telegram-local `receivedAtMs` debug/buffer state.

## Solution Assessment

Yes, the narrow maintainable path is an additive typed field on the existing inbound context and SDK/hook payloads rather than changing the meaning of `Timestamp`. The implementation should avoid a config option and document buffered-message semantics.

## Repair Work Prompt

Implement the Gateway receipt timestamp requested by this issue as a backward-compatible feature. Add a typed `gatewayReceivedAt`/receipt-time field to the shared inbound context without changing the meaning of existing `Timestamp`, stamp it once at channel/Gateway ingress, and thread it through channel-turn build context, direct-DM dispatch, hook message contexts/events, and the agent-visible per-turn conversation metadata. Keep volatile receipt timing out of stable system metadata used for prompt caching. Define and document the buffered/coalesced-message rule, especially for Telegram text fragment buffering. Add focused tests for inbound metadata rendering, context finalization/building, hook mappers, direct-DM helper, and Telegram ingress paths. Update `docs/plugins/sdk-channel-turn.md` and `CHANGELOG.md`. Do not add a config option, do not rename or repurpose `Timestamp`, and do not move plugin-owned behavior into core beyond the generic shared seam.
