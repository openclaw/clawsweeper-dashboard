## What This Changes

Request adds a Gateway-received timestamp to inbound metadata so agents can distinguish channel delivery delay from processing latency.

## Best Possible Solution

Define one additive generic ingress timing contract, reuse existing internal receive-time sources where available, and expose it consistently through inbound context, agent-visible metadata, docs, and focused tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the public contract. The requested field is plausible, but the maintainable solution should be one additive generic timing contract rather than a per-channel or prompt-only patch.
