## Summary

Keep open. Current main still lacks a shared agent-visible Gateway receipt timestamp, and the related open Discord latency report makes the observability gap more relevant rather than superseded.

## What This Changes

Add a Gateway receipt timestamp to inbound metadata so agents can separate source-channel delivery latency from OpenClaw processing latency.

## Best Possible Solution

Add a backward-compatible receipt timestamp stamped once at ingress, thread it through FinalizedMsgContext, prompt metadata, hooks, direct-DM/channel-turn SDK contracts, and document buffered-path semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence evidence that the shared contexts, prompt metadata, hooks, direct-DM helper, and channel-turn SDK surface still expose only the source-channel timestamp.

## Solution Assessment

Yes. The narrow maintainable solution is an additive typed receipt-time field stamped at ingress, not repurposing the existing Timestamp/timestamp source-message field.
