## Summary

Keep open: current main and the latest release still expose only the source-channel timestamp in agent-facing inbound metadata, while existing `receivedAt` primitives are limited to receive/ack internals. The request is directionally aligned, but the public field name and semantics need maintainer API/product review.

## What This Changes

Add a canonical Gateway receipt timestamp to inbound metadata so agents and hooks can distinguish channel delivery delay from OpenClaw processing latency.

## Best Possible Solution

Define one shared receive-time contract, preserve `Timestamp` as source-channel time, carry the value through buffered and durable channel paths, and expose it consistently in inbound metadata and hook/runtime surfaces with focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested agent-visible receive timestamp.

## Solution Assessment

Unclear until maintainers choose the public contract. A generic shared inbound metadata field is maintainable, but the exact name, timing semantics, and model-visible placement need product/API review first.
