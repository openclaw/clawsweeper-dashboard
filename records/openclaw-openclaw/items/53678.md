## Summary

Keep open: current main and the latest release still expose only the source-channel timestamp in shared inbound context, prompt metadata, hooks, and SDK turn/direct-DM paths; the existing `receivedAt` code is internal receive/ingress groundwork, not the requested agent-visible Gateway receipt timestamp.

## What This Changes

Request to add an agent-visible Gateway receipt timestamp to inbound metadata so agents and diagnostics can separate channel delivery delay from OpenClaw processing latency.

## Best Possible Solution

Add a backward-compatible receipt-time field stamped once at channel/Gateway ingress, preserve `Timestamp` as source-channel time, and thread the new value through finalized context, prompt metadata, hooks, direct-DM, channel-turn SDK contracts, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still lacks an agent-visible Gateway receipt timestamp in shared inbound metadata.

## Solution Assessment

Yes. An additive optional receipt-time field is narrower than repurposing the existing source-channel `Timestamp`, and the current receive-context `receivedAt` groundwork does not satisfy the shared metadata request.
