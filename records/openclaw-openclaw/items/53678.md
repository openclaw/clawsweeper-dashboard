## Summary

Keep open: current main and the latest release expose the source-message timestamp with second precision, but they still do not expose a gateway-received timestamp in generic inbound metadata. The linked Discord latency report was resolved by upgrade/configuration, and the separate seconds-precision issue was fixed, but neither implements this all-channel ingress timing request.

## What This Changes

Request to add an agent-visible gateway receipt timestamp to inbound metadata so channel delivery delay can be separated from agent processing latency.

## Best Possible Solution

Define one additive generic ingress timing contract for channel turns, then expose it consistently through inbound prompt metadata and any diagnostic logs without per-channel ad hoc fields.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested generic gateway receipt timestamp.

## Solution Assessment

Unclear until maintainers choose the contract. The likely best direction is a generic additive ingress timing field or turn timing contract, not a Discord- or Telegram-only metadata patch.
