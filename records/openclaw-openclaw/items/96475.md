The issue requests a new `tools.exec.strictShellMetachars` and per-agent override to force approval for shell-metacharacter exec commands unless an allowlisted shell owns the command.

## Best Possible Solution

Keep this open until maintainers decide the exec security policy; if accepted, implement one shared default-off config path with schema/help/docs/audit coverage and focused gateway/node tests before considering any future default change.
