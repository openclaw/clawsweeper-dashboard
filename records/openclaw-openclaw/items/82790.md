## Summary

Keep this PR open. It targets a real Codex projector gap, but the patch drops generic item completion events for non-native-tool items and still lacks after-fix live Telegram/Codex proof.

## What This Changes

The PR backfills Codex app-server normalized item/tool progress and trajectory events from completed-turn snapshots, with dedupe against live item notifications.

## Best Possible Solution

Preserve generic item completion events for all item kinds, keep the native snapshot backfill with dedupe, add focused coverage for the non-native item completion case, then supply live Telegram/Codex proof.

## Reproduction Assessment

Yes for the projector seam: current main's completed-turn snapshot loop omits normalized item/tool progress, and the PR diff shows a source-reproducible regression in item/completed handling. No live Telegram/OpenAI reproduction was supplied or run.

## Solution Assessment

No as currently written. The snapshot backfill/dedupe direction is narrow, but handleItemCompleted must preserve generic item end events for dynamic, reasoning, compaction, and other non-native-tool items while adding native tool backfill on top.
