## What This Changes

The PR changes `scripts/control-ui-i18n.ts` to use the shared generated-module formatter for generated locale TypeScript while preserving the replacement-character restoration guard.

## Best Possible Solution

Land the shared-formatter routing after exact-head CI is green, keeping formatter command resolution in `scripts/lib/format-generated-module.mjs` instead of duplicating a local pnpm resolver.

## Reproduction Assessment

Yes, source-level with external live proof. Current main still falls back to bare `pnpm` for generated locale formatting, and the PR body gives Windows before/after `pnpm ui:i18n:check` output showing the ENOENT failure no longer occurs after the patch.

## Solution Assessment

Yes. Reusing `scripts/lib/format-generated-module.mjs` is the narrow maintainable fix because that helper already owns direct `oxfmt` selection and the Windows pnpm-runner fallback; a new local resolver would duplicate that contract.
