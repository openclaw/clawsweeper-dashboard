## What This Changes

This PR changes `scripts/control-ui-i18n.ts` so generated locale TypeScript formatting uses the shared generated-module formatter instead of its own direct-or-bare-`pnpm` formatter path.

## Best Possible Solution

Land the shared-formatter version after exact-head CI passes, preserving replacement-character restoration and avoiding a second formatter resolver.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.9` still fall back to bare `pnpm` for control-ui locale formatting, and the PR body provides Windows before/after `pnpm ui:i18n:check` output.

## Solution Assessment

Yes. Reusing `scripts/lib/format-generated-module.mjs` is the narrow maintainable fix because that helper already owns direct `oxfmt` selection and the Windows pnpm-runner fallback; a local resolver would duplicate that contract.
