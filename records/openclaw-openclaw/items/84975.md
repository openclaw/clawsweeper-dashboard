## Summary

Keep open: the fix direction is useful, but current head still marks heartbeat message-tool delivery evidence inside the per-payload loop, so a same-route tool send followed by multiple visible fallback payloads can suppress legitimate heartbeat content.

## What This Changes

This PR threads route-scoped generic message-tool delivery evidence through heartbeat reply payloads so duplicate heartbeat fallback text can be suppressed, with focused regression coverage.

## Best Possible Solution

Keep the route-scoped evidence design, but move the marker decision to whole-result scope so only one eligible visible heartbeat fallback can suppress the main send, with a two-visible-payload regression.

## Solution Assessment

No, not yet. The route-scoped suppression direction is right, but marker scope needs to move from per-payload processing to the whole deduped heartbeat result.
