## What This Changes

The PR threads route-scoped generic message-tool delivery evidence through reply payload handling and heartbeat dispatch so same-route heartbeat fallback text can be suppressed while wrong-route fallback remains deliverable.

## Best Possible Solution

Use route-scoped generic message-tool delivery evidence to suppress heartbeat fallback in message-tool-only mode regardless of text equality, preserve wrong-route/topic fallback with focused tests, and keep route parsing on loaded or pure helpers without bundled plugin lazy-loads.

## Reproduction Assessment

Yes. The linked issue provides production logs for the duplicate Telegram delivery, and source inspection shows current main sends fallback text when no `heartbeat_respond` payload exists while latest head still misses the differing-text narration case.

## Solution Assessment

No. Route-scoped evidence is the right direction, but gating suppression on text duplication misses the reported shape; the route parser also needs to stay on loaded or pure helpers before this is the best maintainable fix.
