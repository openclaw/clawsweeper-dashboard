## Summary

Keep open: the route-scoped suppression direction is still needed for the linked heartbeat duplicate-message bug, but the current patch still has a blocking message-delivery edge case because it applies the delivery marker one payload at a time.

## What This Changes

This PR threads route-scoped generic message-tool delivery evidence into heartbeat reply payload metadata and uses it to suppress duplicate heartbeat fallback sends, with focused heartbeat and auto-reply tests.

## Best Possible Solution

Keep the route-scoped suppression design, but only mark delivery after evaluating the whole heartbeat result and proving multi-payload fallback remains deliverable before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the marker guard runs on one payload at a time, and the heartbeat runner suppresses the main send when the selected reply payload carries that marker.

## Solution Assessment

No. The route-scoped design is the right layer, but this implementation needs to make the marker decision across the whole heartbeat result rather than per payload.
