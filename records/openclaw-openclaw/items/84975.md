Keep open: this PR is a focused fix for the live heartbeat duplicate-delivery bug, and the new route-scoped marker addresses the main blocker from the earlier closed attempt. ## What This Changes

The PR adds route-scoped message-tool delivery metadata to heartbeat reply payloads and makes the heartbeat runner skip fallback text only when that metadata proves same-route delivery.

## Best Possible Solution

Keep this route-scoped metadata approach, merge only after maintainers accept the remaining message-delivery risk or add a short live Telegram/Mantis proof artifact.

## Reproduction Assessment

Yes. The issue is source-reproducible on current main because heartbeat message-tool-only mode can receive a generic message-tool send plus free text, while runHeartbeatOnce only treats heartbeat_respond metadata as owning delivery before the fallback send path.

## Solution Assessment

Yes. Threading in-process delivery metadata from runReplyAgent to runHeartbeatOnce and checking the existing route matcher is narrower and safer than suppressing fallback on any generic message-tool call or transcript shape.
