## Summary

Keep open: this PR is still the active route-scoped fix for the heartbeat duplicate-delivery bug, but the current head keeps a message-delivery defect because it marks delivery evidence one payload at a time while heartbeat suppression consumes the whole main reply.

## What This Changes

The PR adds route-scoped message-tool delivery metadata to heartbeat reply payloads and suppresses duplicate same-route heartbeat fallback text when that metadata is present.

## Best Possible Solution

Move the message-tool-delivered marker decision to whole-result scope so suppression applies only when the entire heartbeat fallback result has exactly one visible non-reasoning text fallback to replace, with a regression for the multi-visible fallback case.

## Reproduction Assessment

Yes. Source inspection of the current PR head shows the metadata marker is applied per payload while the heartbeat runner suppresses the whole main reply when the selected payload carries that metadata.

## Solution Assessment

No. The route-scoped evidence design is the right direction, but the current implementation needs whole-result semantics rather than per-payload semantics to avoid suppressing unrelated visible fallback payloads.
