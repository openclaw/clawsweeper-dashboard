The PR threads route-scoped generic message-tool delivery evidence through heartbeat reply payload metadata and suppresses heartbeat fallback text when the selected reply route was already delivered by the generic message tool.

## Best Possible Solution

Keep the route-scoped evidence idea, but make suppression a whole-result decision so only the single duplicate fallback is skipped and unrelated visible fallback payloads still deliver.

## Reproduction Assessment

## Solution Assessment

No, not yet. Route-scoped evidence is the right boundary, but per-payload marking is too early; the safer fix is to decide suppression after the whole heartbeat result is known.
