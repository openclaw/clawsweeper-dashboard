## Best Possible Solution

Keep the route-scoped suppression design, but decide heartbeat marker eligibility across the complete reply payload set and only mark when exactly one non-reasoning plain visible payload is eligible.

Yes. Source inspection of current PR head shows a high-confidence reproduction path: two non-reasoning plain visible heartbeat payloads with same-route message-tool evidence are processed one at a time, so both can be marked and the selected last payload can be suppressed.

## Solution Assessment

No. The route-scoped fix is the right layer, but the marker decision must be whole-result scoped so unrelated visible fallback text is not suppressed when multiple plain payloads exist.
