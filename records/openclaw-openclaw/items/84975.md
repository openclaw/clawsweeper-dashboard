## Summary

Keep open: the PR is the right route-scoped fix direction for the linked heartbeat duplicate-message bug, but the current head still has a blocking message-delivery edge case because heartbeat delivery evidence is applied inside the per-payload loop instead of over the whole reply result.

## What This Changes

## Best Possible Solution

Keep the route-scoped approach, but gate the marker on the whole heartbeat reply having exactly one non-reasoning plain visible fallback; multiple visible fallback payloads should keep existing delivery and dedupe behavior.
