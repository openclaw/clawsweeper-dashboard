## Summary

Keep this PR open: the route-scoped heartbeat suppression direction is useful, but the current patch can suppress legitimate multi-payload heartbeat text because the new single-payload guard is applied inside the per-payload loop.

## What This Changes

The PR threads route-matched generic message-tool delivery evidence through heartbeat reply payloads and suppresses heartbeat fallback delivery when that evidence is present.

## Best Possible Solution

Keep the route-scoped heartbeat suppression, but make the metadata decision over the full heartbeat payload set so only a single plain fallback payload can be suppressed and multi-payload replies continue through normal text-dedupe behavior.

## Reproduction Assessment

Yes for the review finding from source inspection: the PR calls the new marking helper with a one-element mediaFiltered array inside the payload loop, then heartbeat-runner suppresses delivery based on the selected payload metadata. I did not run tests because this review is read-only.

## Solution Assessment

No. The route-scoped suppression is the right layer, but this implementation is too broad for multi-payload heartbeat replies; the safer fix must decide suppression using the complete heartbeat payload set before marking metadata.

Overall confidence: 0.87

Full review comments:

- **[P1] Keep multi-payload heartbeat replies on the text-dedupe path:** `src/auto-reply/reply/agent-runner-payloads.ts:376-378`
  - body: This branch calls `markHeartbeatMessageToolDeliveredPayloads(mediaFiltered)` inside the loop where `mediaFiltered` was built from `[payload]`, so the helper's `markable.length !== 1` guard only sees one payload at a time. With two plain heartbeat payloads and one same-route message-tool send, both payloads can be marked; `heartbeat-runner` then sees the selected payload's metadata and returns before sending anything, dropping unrelated fallback text that current main would still deliver unless the text itself matched the tool send. Compute the heartbeat suppression decision across the full payload set, or keep multi-payload replies on the existing text-match dedupe path.
