## Summary

Keep open: current main still eagerly imports the full file-transfer node.invoke policy during plugin registration, and this PR is a focused lazy-loading fix with sufficient after-fix proof and no blocking review findings.

## What This Changes

The PR makes the bundled file-transfer plugin register a lazy node.invoke policy wrapper with shared command metadata and focused tests so gateway startup does not import the full policy implementation.

## Best Possible Solution

Land the lazy policy wrapper after normal CI and maintainer checks, preserving command metadata and fail-closed behavior while deferring the full policy import until the first handled file-transfer node command.

## Reproduction Assessment

Yes from source: current main's entrypoint imports and instantiates the full file-transfer node.invoke policy during registration. I did not run the benchmark locally because this review was read-only.

## Solution Assessment

Yes: the diff addresses the hot-path import at the plugin boundary, shares the command list to avoid drift, and leaves the existing policy delegate responsible for enforcement.
