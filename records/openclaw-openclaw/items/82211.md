## Summary

Keep this PR open: current main still eagerly imports the full file-transfer node.invoke policy, and the proposed lazy wrapper appears correctly scoped with focused tests and supplied runtime proof. I found no blocking correctness or security findings in the read-only review.

## What This Changes

The PR replaces the file-transfer plugin’s eager node.invoke policy import with a lazy wrapper, shares command metadata, and adds focused tests for registration, fail-closed loading, caching, and delegate errors.

## Best Possible Solution

Land the lazy wrapper if CI confirms the focused tests and build on the exact PR head, preserving static command metadata while deferring the full file-transfer policy implementation until node.invoke handling.

## Reproduction Assessment

Yes. Current main is source-reproducible because extensions/file-transfer/index.ts statically imports the full node.invoke policy module at plugin-entry load time, before registration only needs command metadata.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: keep policy.commands static for registry lookup, share the command list with the real policy, and load the enforcement delegate only inside handle().
