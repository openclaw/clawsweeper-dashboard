## Summary

Keep open: current main still publishes the usage-cost cache refresh lock with `fs.promises.link` and rethrows non-`EEXIST` errors, while https://github.com/openclaw/openclaw/pull/81388 is the active open closing fix candidate.

## What This Changes

The issue reports that session usage-cost cache refresh lock acquisition crashes with `ENOTSUP` on filesystems that do not support hard links.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81388, or an equivalent narrow fallback in `src/infra/session-cost-usage.ts`, then close this issue from the merge.

## Reproduction Assessment

Yes, at source level: current main calls `fs.promises.link` during usage-cost cache refresh and rethrows non-`EEXIST` errors. Runtime reproduction would require an unsupported-hardlink mount, which was not mounted in this read-only review.

## Solution Assessment

Yes: the best path is a narrow unsupported-hardlink fallback or direct exclusive-create lock publish that preserves existing `EEXIST`, stale-lock cleanup, and release-token behavior. The active linked pull request is the current implementation candidate, so this issue should stay open until it merges or is replaced.
