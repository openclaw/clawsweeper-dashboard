## Summary

Keep open: current `main` and the latest release still publish the usage-cost cache lock with `fs.promises.link`, and the only linked fixing PR is still open. The issue should stay open until that PR, or an equivalent fallback, lands.

## What This Changes

The issue requests a portable fallback for the hard-link based usage-cost cache lock so refreshes do not crash on filesystems without hard-link support.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81388, or an equivalent narrow fallback, while preserving the existing lock contention and cleanup contract.

## Reproduction Assessment

No live unsupported-hardlink filesystem was mounted in this checkout. The issue is source-reproducible with high confidence because current `main` calls `fs.promises.link` during usage-cost refresh and rethrows every non-`EEXIST` error.

## Solution Assessment

Yes, the fallback or exclusive-create direction is the narrow maintainable fix. The issue should remain open while the linked fixing PR is reviewed, proven, and either landed or replaced.
