## Summary

Keep open: current main still publishes the usage-cost cache lock with `fs.promises.link` and rethrows non-`EEXIST` errors, while https://github.com/openclaw/openclaw/pull/81388 remains the open closing implementation candidate.

## What This Changes

The issue asks OpenClaw to avoid crashing when the session usage-cost cache lock is created on filesystems where hard links are unsupported.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81388, or an equivalent narrow fallback with regression coverage, then close this issue from the merge.

## Reproduction Assessment

Yes, source-level: current main calls `fs.promises.link` during usage-cost refresh and rethrows non-`EEXIST` errors, so an unsupported-hardlink mount can produce the reported `ENOTSUP` failure. A live unsupported filesystem was not mounted in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to preserve existing lock semantics while adding an exclusive-create fallback for known unsupported-hardlink errors in `src/infra/session-cost-usage.ts`. The open closing PR is the active path to review before this issue closes.
