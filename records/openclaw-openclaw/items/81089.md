## Summary

Keep open: current main and the latest release still publish the usage-cost cache lock with a hard link and rethrow unsupported-link errors. The linked closing PR is still open, unmerged, and conflicting, so this issue remains the right tracker until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to stop crashing when session usage-cost cache lock creation hits ENOTSUP from fs.link on filesystems without hard-link support.

## Best Possible Solution

Land one canonical lock fix that preserves current EEXIST contention and stale-lock behavior while using an exclusive-create fallback or replacement on unsupported-hardlink filesystems.

## Reproduction Assessment

Yes at source level: force fs.promises.link to reject with ENOTSUP and current main rethrows it because only EEXIST is handled. I did not run a live SMB/NFS/virtiofs mount repro in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to preserve the lock semantics while adding an exclusive-create fallback or replacement at the lock writer. The open linked PR is the current implementation candidate, so this issue should stay open until that PR lands or is replaced.
