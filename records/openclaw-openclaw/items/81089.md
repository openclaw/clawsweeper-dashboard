## Summary

Keep open: current main still publishes the usage-cost cache lock with `fs.promises.link` and rethrows non-`EEXIST` errors, so the reported `ENOTSUP` crash path remains source-proven. An open linked PR already owns the fix path, so this issue should wait for that PR to land or be closed.

## What This Changes

Usage-cost cache refresh should acquire its session lock on filesystems where hard links are unsupported instead of crashing with `ENOTSUP`.
