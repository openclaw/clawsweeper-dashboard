## Summary

Keep open: current main still publishes the usage-cost cache lock with `fs.promises.link` and rethrows non-`EEXIST` errors, and the open closing PR has not merged yet.

## What This Changes

The issue reports that session usage-cost cache refresh can crash with `ENOTSUP` because lock publication depends on hard-link support.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81388, or an equivalent narrow fallback with regression coverage, then close this issue from the merge.

## Reproduction Assessment

Yes, source-level: current main calls `fs.promises.link` during usage-cost refresh and rethrows non-`EEXIST` errors, so an unsupported-hardlink mount can produce the reported `ENOTSUP` failure. A live unsupported filesystem was not mounted in this review.

## Solution Assessment

Yes: the narrow maintainable fix is to keep the existing lock semantics while adding an exclusive-create fallback for known unsupported-hardlink errors in `src/infra/session-cost-usage.ts`. The existing open PR is the right path to review before closing this issue.
