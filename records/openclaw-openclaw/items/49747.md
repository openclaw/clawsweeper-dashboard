## Summary

Keep open. Current main and the latest release still derive the group-session provider from the first `ctx.From` segment, so an `agent:<id>:slack:channel:...` key can still fall back to `webchat`; the focused fix path is the still-open https://github.com/openclaw/openclaw/pull/49750.

## What This Changes

The issue asks OpenClaw to preserve Slack thread routing for agent-prefixed A2A session keys and announce/restart delivery instead of normalizing the route to webchat or the base session.

## Best Possible Solution

Keep this issue open until the focused Slack A2A routing fix is reviewed, rebased if needed, and landed with parser and delivery-context regression coverage.

## Reproduction Assessment

Yes from source inspection, though not from a live Slack run in this read-only review. `resolveGroupSessionKey()` still reads `parts[0]` as the provider, so an `agent:<id>:slack:channel:...` key with a webchat provider hint can still resolve through webchat on current main.

## Solution Assessment

Unclear whether the open PR is still the exact best patch because current main already has partial restart-sentinel hardening. The best direction is a canonical agent-prefixed session-key parser plus field-level delivery-context fallback coverage, not a second routing shim.
