Keep open: current main still has the source-visible agent-prefixed session parser drift, and the same reporter has an open implementation PR that should be reviewed instead of closing this issue during cleanup.

## What This Changes

The issue reports that agent-prefixed A2A Slack thread session keys can be resolved as webchat and that restart/announce fallback can reuse the base session route instead of preserving the target Slack thread route.

## Best Possible Solution

Review and rebase https://github.com/openclaw/openclaw/pull/49750, or replace it with a narrow fix that uses guarded shared session-key parsing, preserves the intended thread-target policy, and adds regression coverage for parser and restart fallback precedence.

## Reproduction Assessment

Yes, at source level. On current main, resolveGroupSessionKey() with From=agent:builder:slack:channel:C123:thread:T and Provider=webchat reads head=agent, misses the encoded slack provider, and can resolve/persist webchat.

## Solution Assessment

Unclear for the active fix path as-is. The parser direction is narrow, but the restart fallback precedence and current sessions_send thread-target policy need maintainer review in the paired PR.
