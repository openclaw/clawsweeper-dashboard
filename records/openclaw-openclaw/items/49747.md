## Summary

Keep open: current main still has the source-visible agent-prefixed session parser drift and restart fallback behavior, and the same reporter has an open implementation PR that should be reviewed instead of closing this issue during cleanup.

## What This Changes

The issue asks OpenClaw to preserve Slack thread routing for agent-prefixed A2A session keys and prevent restart/announce fallback from overriding explicit target delivery context.

## Best Possible Solution

Review and rebase https://github.com/openclaw/openclaw/pull/49750, or replace it with a narrow fix that uses guarded shared session-key parsing, preserves current thread-target policy, and adds regression coverage for parser and restart fallback precedence.

## Reproduction Assessment

Yes, at source level. On current main, resolveGroupSessionKey() with From=agent:builder:slack:channel:C123:thread:T and Provider=webchat reads head=agent, misses the encoded slack provider, and can resolve/persist webchat.

## Solution Assessment

Unclear for the active path as-is. The requested parser direction is right, but the open PR needs rebase/review to guard agent-prefix parsing, preserve sessions_send thread-target rejection, and decide the restart fallback precedence intentionally.
