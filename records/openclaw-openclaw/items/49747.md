## Summary

Keep open. Current main and the latest release still parse the provider for `resolveGroupSessionKey()` from the first `ctx.From` segment, so an `agent:<id>:slack:channel:...` session key can still fall back to a webchat provider hint; the paired open fix path is https://github.com/openclaw/openclaw/pull/49750.

## What This Changes

The issue asks OpenClaw to preserve A2A Slack thread routes by parsing agent-prefixed Slack session keys correctly and preventing base-session announce fallback from overriding explicit target delivery context.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/49750 is reviewed against current main and either landed or replaced by an equivalent focused parser and fallback-routing fix.

## Reproduction Assessment

Yes, from source inspection. `resolveGroupSessionKey()` still reads `parts[0]` as the provider head, so an `agent:<id>:slack:channel:...` key with a webchat provider hint can still resolve through webchat on current main; no live Slack run was attempted.

## Solution Assessment

Unclear whether the open PR is still the exact best patch because current main has partial route-preservation work merged since the branch was opened. The best direction is one focused parser and field-level fallback fix, re-reviewed against current main rather than a second routing shim.
