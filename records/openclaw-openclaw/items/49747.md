## Summary

Keep open: current main and v2026.6.8 still leave the agent-prefixed Slack group-session parser gap in place, and the same author has an open paired fix candidate that directly targets this issue.

## What This Changes

This issue reports that A2A Slack thread session keys can be parsed as webchat and that restart/announce fallback can route through a base session instead of the intended Slack thread.

## Best Possible Solution

Review or refresh https://github.com/openclaw/openclaw/pull/49750 against current main, keeping the existing field-level delivery fallback while adding shared parser coverage for agent-prefixed Slack thread keys.

## Reproduction Assessment

Yes, at source level: current main still treats the first ctx.From segment as the provider candidate, so agent-prefixed Slack thread keys can fall back to a webchat provider hint. I did not run a live Slack A2A reproduction.

## Solution Assessment

Yes on direction, but not as a new repair lane: the maintainable path is shared session-key parsing plus focused restart/announce routing coverage, and the existing open PR should be reviewed or rebased before merge.
