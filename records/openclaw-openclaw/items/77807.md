## What This Changes

The issue reports that native `sessions_spawn` fails with `missing scope: operator.write` even when the local operator token and paired device state already include full operator scopes.

## Best Possible Solution

Add a targeted integration regression for `sessions_spawn -> callGateway(method="agent")` under token auth with a full-scope operator device token, then fix scope/device-auth propagation so the child `agent` request arrives with `operator.write` without granting unintended admin/owner privileges.
