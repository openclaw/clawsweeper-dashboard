Keep open. The reporter provided a focused live reproduction plus a local hotfix validation, and current main has not changed the relevant Gateway/subagent call path since the prior review; the failure still needs an auth-boundary fix rather than cleanup closure.

## What This Changes

The issue reports that native `sessions_spawn` fails with `missing scope: operator.write` even when the local operator device token has full operator scopes.

## Best Possible Solution

Add a targeted regression for native `sessions_spawn -> Gateway agent` under token auth with a full-scope operator device token, then adjust identity/scope propagation so `operator.write` reaches the child `agent` call without broadening remote auth or owner/admin privileges.
