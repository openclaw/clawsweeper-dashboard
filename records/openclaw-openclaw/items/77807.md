## Summary

Keep open. The reporter supplied a focused live reproduction and control RPC, and current main still has a plausible auth-path gap where the native subagent `agent` call can lose the effective `operator.write` scope before Gateway authorization.

## What This Changes

Fix native `sessions_spawn` so its internal Gateway `agent` call satisfies `operator.write` when the local operator token has full scopes.

## Best Possible Solution

Add a regression for native `sessions_spawn -> Gateway agent` under token auth, then fix the scoped internal Gateway call path without widening remote or browser device-less operator trust.

## Reproduction Assessment

Yes, at source level. The reporter provided a redacted live repro plus a direct `agent` control RPC, and current main still has the implicated subagent, scope, and Gateway identity-selection path; I did not execute the live Gateway setup.

## Solution Assessment

Unclear. The reporter's local hotfix is plausible, but current docs/tests already define a narrow device-less backend path and `callGateway` tests expect explicit scoped backend calls to omit device identity, so the maintainable fix should be regression-led and security-reviewed.
