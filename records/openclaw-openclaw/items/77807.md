## Summary

Keep open. The reporter added a focused live reproduction and hotfix validation after the prior review, and current main plus the latest release still carry the scope-blind backend Gateway device-identity omission that makes this unresolved rather than implemented or stale.

## What This Changes

Fix native `sessions_spawn` failing with `missing scope: operator.write` when the same full-scope operator device token succeeds through a direct Gateway `agent` RPC.

## Best Possible Solution

Add a targeted regression for native `sessions_spawn -> Gateway agent` under token auth with a full-scope operator device token, then adjust Gateway call identity/scope propagation so `operator.write` reaches the child `agent` call without broadening remote auth or owner/admin privileges.

Yes, with caveat: the reporter provided a concrete live repro and control RPC, and current source still has the relevant identity/scope-selection shape. I did not execute that live Gateway setup against current main in this read-only pass.

## Solution Assessment
