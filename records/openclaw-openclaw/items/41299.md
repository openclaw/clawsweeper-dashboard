## Best Possible Solution

Keep generic current-turn prompt composition unchanged for non-metadata contexts, and either gate any separator to generated inbound metadata prefixes or land the hidden runtime-context path from https://github.com/openclaw/openclaw/pull/89428 or https://github.com/openclaw/openclaw/pull/90745.

## Solution Assessment

No. The separator is a plausible mitigation for metadata ambiguity, but placing it in the shared prompt helper is too broad; the safer fix is to keep marker logic on the generated metadata boundary or route metadata through hidden runtime context.
