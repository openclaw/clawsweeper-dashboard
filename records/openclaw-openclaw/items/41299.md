## Summary

Keep open: current main does not implement the separator, and the PR still has a narrow but real sanitizer bug that would silently remove user-authored marker text from visible history after metadata stripping.

## What This Changes

The PR adds a synthetic `---openclaw:user-msg---` boundary between inbound metadata and user text, strips it from metadata sanitizers, and adds separator stripping tests.

## Best Possible Solution

Keep the separator fix narrow, remove only the synthetic marker paired with a stripped metadata block, preserve matching text later in the real user body, and align prompt fixtures plus changelog before merge.

## Reproduction Assessment

Yes for the PR regression: the proposed helper regex can be exercised directly and deletes later user-authored marker text. The original LLM false-empty behavior was not live provider reproduced here.

## Solution Assessment

No. Adding a boundary marker is a reasonable direction, but the current global stripping helper is too broad and the test fixtures/changelog are not yet aligned.
