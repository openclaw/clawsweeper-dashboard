## Summary

Keep this PR open: current main still lacks the separator behavior, but the PR head has source-level correctness blockers in separator stripping, media-only prompt composition, prompt fixtures, and changelog coverage.

## What This Changes

The PR adds an internal separator between inbound metadata and user text, strips that marker from sanitized user-facing views, and adds separator-focused sanitizer tests.

## Best Possible Solution

Keep the separator direction, but centralize metadata/body composition in one helper used by normal text, media-only fallback, and prompt fixtures; strip only the synthetic boundary adjacent to removed metadata and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main still joins metadata and user text with only blank lines, and the PR regressions are reproducible from the diff paths for sanitizer removal, media-only fallback composition, and prompt fixtures.

## Solution Assessment

No. The separator is a reasonable fix direction, but this patch is not the safest implementation because it leaves one runtime composition path unchanged and removes marker text more broadly than the injected boundary.
