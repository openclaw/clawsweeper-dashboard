## Summary

## What This Changes

The PR adds a `---openclaw:user-msg---` separator when current inbound context is joined with user prompt text, and adds sanitizer/test coverage to strip that separator after inbound metadata removal.

## Best Possible Solution

Prefer one canonical prompt-boundary fix: either fold this work into the hidden-runtime-context path or scope any separator to proven metadata-only contexts with sibling CLI, room-event, and current-message coverage.

## Reproduction Assessment

## Solution Assessment

No. The inline separator is plausible for metadata blocks, but adding it in the shared helper is too broad; the safer solution is a metadata-only gate or the hidden-runtime-context direction in the related canonical PR.
