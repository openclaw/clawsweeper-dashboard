## Summary

The PR inserts a `---openclaw:user-msg---` delimiter when combining current inbound context with the user prompt, strips that delimiter in inbound metadata display sanitizers, and adds separator-focused unit tests.

## Best Possible Solution

Either land the hidden-runtime-context routing for the canonical inbound-metadata issue, or narrow this PR so any delimiter is metadata-only and covered by current-message, room-event, CLI, display, and proof paths.

## Solution Assessment

No. Adding the marker in the shared helper is too broad; the maintainable fix is either hidden runtime context for inbound metadata or a metadata-only delimiter with sibling-context coverage and proof.
