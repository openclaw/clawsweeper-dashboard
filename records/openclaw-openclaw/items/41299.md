## Best Possible Solution

Land one canonical hidden-runtime-context routing path, or narrow this delimiter mitigation so only actual metadata prefixes receive and strip the marker.

## Reproduction Assessment

## Solution Assessment

No. The delimiter is a plausible mitigation, but placing it in the generic helper is too broad; hidden runtime context is the cleaner root-cause fix, and any delimiter fallback should be metadata-specific.
