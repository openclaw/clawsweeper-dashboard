## Summary

Keep open: current main has not implemented this delimiter, but the PR still has a concrete sanitizer regression where later user-authored marker lines can be deleted after metadata stripping, and it lacks real live-provider proof.

## What This Changes

The PR injects an `---openclaw:user-msg---` marker between inbound metadata and user text, strips that marker from metadata sanitizers, and adds focused sanitizer tests.

## Best Possible Solution

Keep a clear model-facing boundary if it proves useful, but implement it at the current-turn prompt boundary and strip only the adjacent generated marker from visible views while preserving user-authored text.

## Reproduction Assessment

Yes for the PR-introduced sanitizer regression by source inspection: the global regex can remove later user-authored marker lines after metadata was stripped. No for the original false empty-message symptom because the PR body says live provider verification was not performed.

## Solution Assessment

No. A prompt delimiter may be a maintainable fix, but this branch should use adjacent-only marker stripping and be refreshed onto the current prompt-envelope flow before merge.
