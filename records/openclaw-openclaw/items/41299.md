## Summary

Keep open, but this PR is not merge-ready: the delimiter is added in the shared current-inbound prompt helper, so it affects compact current-message and room-event contexts that the metadata sanitizer does not safely clean up.

## What This Changes

This PR inserts `---openclaw:user-msg---` between current inbound context and the active prompt, strips that marker after inbound metadata stripping, and adds separator tests.

## Best Possible Solution

Choose one prompt-boundary contract before merge, preferably hidden runtime context that keeps channel metadata out of user-authored prompt/session text; if this delimiter branch remains active, gate insertion and stripping to every supported current-inbound shape with focused proof.

## Reproduction Assessment

Yes. Source inspection shows the shared helper is used for metadata, compact current-message, and room-event current-inbound contexts; I did not reproduce the intermittent live model "message is empty" symptom.

## Solution Assessment

No. This is a plausible mitigation for one metadata shape, but the hidden-runtime-context direction is cleaner because it keeps generated metadata out of user-authored prompt/session text rather than adding a delimiter to every current-inbound context.
