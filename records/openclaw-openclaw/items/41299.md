## Summary

Keep open: the branch is focused, but the patch currently inserts the marker through a shared prompt helper that also handles non-metadata current inbound contexts, so the marker can leak into prompt/session text and user-visible history.

## What This Changes

The PR adds an `---openclaw:user-msg---` delimiter in `buildCurrentInboundPrompt()` and strips it after inbound metadata removal, with unit tests for metadata-block cases.

## Best Possible Solution

Keep inbound metadata model-available through the hidden runtime-context path, or narrow this branch so the delimiter is only emitted for contexts that all sanitizers can prove they remove.

## Reproduction Assessment

Yes, at source level. Current main has `Current message:` and room-event current inbound contexts that flow through the patched shared helper but are not covered by the metadata sentinel stripper.

## Solution Assessment

No. The delimiter is a plausible mitigation for fenced metadata blocks, but it is not the best fix as-is because the shared helper handles broader current inbound context shapes and the hidden-runtime-context direction is cleaner.
