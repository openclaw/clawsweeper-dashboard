## What This Changes

This PR adds a generated separator between inbound metadata and user text in reply prompt construction, strips that separator from inbound metadata sanitizers, and adds separator regression tests.

## Best Possible Solution

Rework the boundary on the current `currentTurnContext` prompt assembly path, strip only the adjacent generated marker from visible views, align prompt fixtures, and add redacted live provider proof.

Yes for the PR regression by source inspection: after metadata stripping, the global regex can remove later user-authored `---openclaw:user-msg---` lines. No for the original false-empty LLM symptom, because the PR body says it was not verified end to end with a real provider.

## Solution Assessment

No. A clearer prompt boundary may be a good fix, but this branch strips too broadly and targets the old body-join path instead of current main's `currentTurnContext` assembly.
