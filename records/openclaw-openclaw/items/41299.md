## Summary

Keep open and block merge: the separator idea targets a real prompt-comprehension problem, but this patch adds the marker in a shared current-inbound prompt helper, so non-metadata `Current message:` and room-event contexts can receive/persist the internal marker as user prompt text.

## What This Changes

The PR adds a `---openclaw:user-msg---` marker between current inbound context and user prompt text, strips that marker in inbound metadata sanitizers, and adds separator-focused tests.

## Best Possible Solution

Use the hidden runtime-context direction for current inbound metadata, or narrowly gate any separator to proven metadata-only contexts while preserving existing `Current message:` and room-event prompt joins.

## Reproduction Assessment

Yes for the PR regression: current source and tests show `buildCurrentInboundPrompt` is used for non-metadata `Current message:` and room-event contexts. The original intermittent LLM “message is empty” symptom has source support, but no live provider reproduction in this review.

## Solution Assessment

No. A marker in the shared join helper is too broad; the better fix is to keep generated inbound context out of user prompt/session text or strictly gate any separator to metadata-only contexts.
