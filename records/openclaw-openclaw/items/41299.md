## Best Possible Solution

Use one canonical prompt-context contract: either narrow this marker to generated metadata-only assembly, or prefer the hidden runtime-context path that keeps inbound metadata out of user-authored prompt text.

## Reproduction Assessment

## Solution Assessment

No. The marker belongs in a metadata-only assembly path, or the metadata should move into hidden runtime context; adding it inside the shared current-inbound helper is too broad.
