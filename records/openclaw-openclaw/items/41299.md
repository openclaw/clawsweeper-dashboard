## Summary

Keep open, but the patch is not merge-ready: the separator is added in a shared current-inbound prompt helper that also handles compact current-message, room-event, and CLI prompt paths where the new marker is not guaranteed to be stripped. The related metadata-comprehension cluster is still open, with hidden-runtime-context PRs offering a cleaner canonical direction.

## What This Changes

This PR adds a `---openclaw:user-msg---` delimiter after current inbound context in `buildCurrentInboundPrompt`, strips that delimiter from inbound-metadata sanitizers, and adds separator tests.

## Best Possible Solution

Land one canonical path that keeps generated inbound metadata out of user-authored prompt/session text; if this branch continues, gate the delimiter to generated metadata-only prefixes and prove compact, room-event, and CLI paths stay clean.

## Reproduction Assessment

Yes for the PR defect: source inspection shows the changed shared helper would add the marker to compact current-message, room-event, and CLI current-inbound prefixes, while stripping is tied to inbound metadata sentinels. I did not run tests because this review is read-only.

## Solution Assessment

No. A delimiter is a plausible mitigation, but this implementation is too broad; the cleaner fix is hidden runtime context for generated metadata or a narrowly gated metadata-only delimiter.
