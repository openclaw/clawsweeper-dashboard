## Summary

Keep open: the PR targets a plausible prompt-boundary problem that current main has not clearly solved, but the latest patch still has a concrete sanitizer regression, stale prompt-fixture coverage, and no real after-fix runtime proof.

## What This Changes

The PR adds a generated `---openclaw:user-msg---` boundary between inbound metadata and user text, strips it from metadata sanitizers, and adds separator-focused tests.

## Best Possible Solution

Rework the fix against the current `currentTurnContext` prompt path, remove only the generated boundary adjacent to stripped metadata, preserve literal marker text in real user content, and add aligned prompt-fixture coverage plus real runtime proof.

## Reproduction Assessment

Yes for the PR regression: the proposed helper regex can be exercised directly and deletes later user-authored marker text. The original LLM false-empty behavior still lacks high-confidence live-provider reproduction.

## Solution Assessment

No. A clearer boundary is a reasonable direction, but this implementation strips too broadly and targets an older prompt assembly path that current main has since refactored.
