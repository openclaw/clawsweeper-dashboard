## What This Changes

The branch adds an `---openclaw:user-msg---` delimiter when `buildCurrentInboundPrompt` joins context and prompt text, strips that delimiter after metadata sanitization, and adds separator tests.

## Best Possible Solution

Land one canonical contract that keeps generated inbound metadata out of user prompt/session text, or gate this delimiter strictly to generated metadata blocks with runner, CLI, sanitizer, and live transport proof.

## Solution Assessment
