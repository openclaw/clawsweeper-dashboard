## What This Changes

The branch adds `---openclaw:user-msg---` between current inbound context and prompt text, strips that marker after inbound metadata stripping, and adds separator-focused sanitizer tests.

## Best Possible Solution

Keep one prompt contract: either repair this branch by limiting delimiter insertion to fenced inbound metadata with sibling tests/proof, or land a clean hidden-runtime-context routing path as the canonical fix.

Yes. Source inspection shows the PR's helper change applies to existing non-metadata `Current message` and room-event context paths, and current tests document marker-free joins for those paths; I did not run tests because this is a read-only review.

## Solution Assessment

No. A delimiter is a plausible mitigation for fenced metadata, but this implementation is too broad for the shared helper; the safer path is metadata-only gating with sibling-context coverage or the hidden runtime-context approach.
