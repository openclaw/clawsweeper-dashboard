## Summary

Keep open. Current main still allows a synthetic compaction marker to leave the previous tool result selectable as subagent completion output, and the issue is linked to open overlapping PRs rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop subagent announce completion from forwarding pre-compaction tool JSON when auto-compaction delays the final assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for output selection and frozen-result capture, preserves timeout partial-progress behavior, and adds focused regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a chat history with a toolResult, then a synthetic compaction system marker, and no later assistant text leaves latestRawText selectable as the completion output.

## Solution Assessment

No for the issue's unified-retry-only direction. The safer fix should cover the selector and frozen completion capture path, not only retry delayed assistant text inside the announce flow.
