## Summary

Keep open. Current main still has a source-level path where a post-tool compaction marker does not pause subagent output selection, so stale tool/toolResult text can be selected before a delayed assistant reply; the issue also has an open PR with closing syntax and overlapping open fixes, so it should not be closed before one consolidated fix lands.

## What This Changes

The issue requests that subagent announce output avoid returning stale tool-result JSON when auto-compaction happens before the final assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats compaction system markers as continuation boundaries for both announce output selection and frozen completion capture, preserves timeout partial-progress behavior, and adds regression coverage for toolResult-then-compaction limbo, delayed assistant recovery, and frozen completion capture.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a child history with a toolResult, then the synthetic compaction system marker, and no later assistant text can still leave latestRawText selectable as the subagent output on current main.

## Solution Assessment

No for the narrow unified-retry-only direction. The safer maintainable fix should cover the output summarizer’s compaction marker handling and captureSubagentCompletionReplyUsing freeze path, not only recheck delayed assistant text inside runSubagentAnnounceFlow.
