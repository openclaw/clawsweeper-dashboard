## Summary

Keep open. Current main still has a source-level path where a stale tool result can be selected as subagent completion output after a compaction marker, and an open PR with closing syntax plus overlapping fixes are still under review.

## What This Changes

The issue reports that auto-compaction during subagent tool use can make `readSubagentOutput` return stale raw tool JSON instead of the delayed post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries in both announce output selection and frozen completion capture, with focused regression coverage for tool-result-then-compaction histories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a child history with a non-empty tool result followed by the synthetic compaction system marker and no later assistant text; current main can still return the preserved raw tool text from that snapshot.

## Solution Assessment

No for the issue's unified-retry-only suggestion. The safer maintainable fix should address the snapshot selector and capture/frozen-result paths, not only recheck delayed assistant text inside the announce flow.
