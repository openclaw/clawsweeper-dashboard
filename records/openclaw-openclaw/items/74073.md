## Summary

Keep open. Current main still lets subagent output selection fall back to pre-compaction raw tool text after a synthetic compaction marker, and this issue is already paired with open overlapping PRs rather than a merged fix.

## What This Changes

Prevent subagent announce and completion capture from treating stale pre-compaction tool JSON as the child reply before the post-compaction assistant response arrives.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries in output selection and completion capture, while preserving timeout partial-progress behavior and adding focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists with chat history containing a toolResult, then a synthetic compaction system marker, and no later assistant text; current readSubagentOutput can still return the preserved latestRawText before checking for a post-compaction assistant reply.

## Solution Assessment

No for the unified-retry-only direction. The maintainable fix should cover the selector and frozen capture paths, not only recheck delayed assistant text inside runSubagentAnnounceFlow.
