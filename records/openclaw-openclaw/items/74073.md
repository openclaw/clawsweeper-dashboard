## Summary

Keep open. Current main still has a source-level path where a synthetic compaction marker does not clear a prior raw tool result, so `readSubagentOutput` can return stale tool JSON before consulting the latest assistant reply; the open closing PR and overlapping fixes should be reviewed together rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to avoid publishing stale raw subagent tool output when auto-compaction delays the final assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for subagent output selection and frozen completion capture, while preserving timeout partial-progress behavior and adding focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a child history with a raw `toolResult`, then a synthetic compaction system marker, and no later assistant text; current main preserves `latestRawText` and returns it before checking `readLatestAssistantReply`.

## Solution Assessment

No for a unified retry-only fix. The safer solution should address the selector and capture/frozen-result paths so stale raw output is not considered valid completion content in the first place.
