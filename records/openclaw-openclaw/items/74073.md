## What This Changes

The issue requests that subagent completion capture wait through auto-compaction so stale tool-result JSON is not announced as the child reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for subagent output selection and frozen completion capture, preserves timeout partial-progress behavior, and adds focused regression tests around toolResult-plus-compaction history.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a child history with a toolResult, then the synthetic compaction system marker, and no later assistant text still lets latestRawText be selected and returned immediately.

## Solution Assessment

No for the issue's narrow unified-retry-only proposal. The safer solution should fix the snapshot selector and completion-freeze path, not only poll once after announce flow completion.
