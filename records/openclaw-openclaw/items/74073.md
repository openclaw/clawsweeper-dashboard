## Summary

Keep open. Current main still has a source-level path where a compaction marker does not pause subagent output selection, so stale raw tool text can be selected before a later assistant reply; this issue is also paired with open overlapping implementation PRs that should be consolidated rather than bypassed.

## What This Changes

The issue requests preventing subagent auto-announcements from selecting stale raw tool-result JSON after auto-compaction before the final assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats compaction markers as continuation boundaries in shared output selection and completion capture, preserves timeout partial-progress behavior, and adds focused regression tests for the limbo and recovery windows.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a history with a toolResult followed by a synthetic compaction system marker and no later assistant text still leaves `latestRawText` selectable as completion output.

## Solution Assessment

No for the suggested run-flow-only retry by itself. The maintainable fix needs to cover the shared selector and frozen completion capture path, not only delayed assistant polling inside `runSubagentAnnounceFlow`.
