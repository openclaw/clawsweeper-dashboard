## Summary

Keep open. Current main still has a source-level path where a pre-compaction tool result can remain selectable as subagent completion output, and the overlapping open PRs have not merged a consolidated fix for both announce delivery and frozen completion capture.

## What This Changes

The issue reports a subagent auto-compaction race where completion announcement or capture can use stale tool-result JSON instead of the later assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries in the output selector and frozen completion capture path, while preserving timeout partial-progress behavior and adding focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists with chat.history containing a toolResult followed by a synthetic compaction system marker and no later assistant text; current main preserves latestRawText and can return it before checking readLatestAssistantReply.

## Solution Assessment

No for a unified-retry-only fix. The safer solution should address the snapshot selector and capture/frozen-result paths, not only recheck delayed assistant text inside runSubagentAnnounceFlow.
