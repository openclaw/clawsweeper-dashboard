## Summary

Keep open. Current main still has a source-level path where subagent output selection can choose stale raw toolResult text after a compaction marker, and this issue is paired with open implementation PRs that should be reviewed/consolidated instead of closing the report early.

## What This Changes

The issue asks OpenClaw to avoid announcing stale raw subagent tool JSON during auto-compaction and to wait for the post-compaction assistant reply before finalizing the parent-session announcement.

## Best Possible Solution

Review or land one consolidated agents fix that treats compaction as a pending continuation boundary, prevents successful completions from freezing raw tool output, preserves timeout partial-progress behavior, and adds selector plus announce/capture regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a `chat.history` response containing a raw `toolResult` followed by the synthetic compaction system marker and no later assistant text still lets `readSubagentOutput` select the raw tool text before consulting the latest assistant reply.

## Solution Assessment

No for a run-flow-only retry. The maintainable fix needs to cover the shared output selector, completion capture, and frozen-result path so pending compaction/no-output is not confused with a successful terminal child result.
