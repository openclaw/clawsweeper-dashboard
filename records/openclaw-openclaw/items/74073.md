## Summary

Keep open. Current main still does not treat synthetic compaction history markers as continuation boundaries before the raw tool-result fallback, and the issue is already paired with open overlapping implementation PRs.

## What This Changes

Prevent subagent announce and completion capture from selecting stale raw tool-result JSON during auto-compaction, and prefer or wait for the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats compaction markers as continuation boundaries in both output selection and frozen completion capture, preserves timeout partial-progress behavior, and adds focused regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is enough: a chat.history array with a toolResult followed by a synthetic compaction system marker and no later assistant text still leaves latestRawText selectable on current main.

## Solution Assessment

No for the unified-retry-only direction. The safer fix needs selector and capture-aware compaction continuation handling because the frozen completion path can capture the same raw output before announce-flow retry logic can correct it.
