## Summary

Keep open. Current main still lacks compaction-marker handling in the subagent output selector, and this issue is already paired with open overlapping implementation PRs that need maintainer consolidation before the issue can close.

## What This Changes

The issue asks OpenClaw to avoid returning stale raw subagent tool JSON after auto-compaction and instead wait for or select the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries in output selection and completion capture, preserves timeout partial-progress behavior, and adds focused regression tests for the compaction limbo and recovery paths.

## Reproduction Assessment

Yes. A source-level reproduction is enough: a chat history with a `toolResult`, then a synthetic compaction system marker, and no later assistant text leaves `latestRawText` selectable and returned by `readSubagentOutput`.

## Solution Assessment

No for the unified-retry-only direction. The narrow maintainable fix should cover the selector and frozen completion capture path, not only a post-announce retry in `runSubagentAnnounceFlow`.
