## Summary

Keep open. Current main still has a source-level path where a post-tool compaction marker does not pause subagent output selection, so stale raw tool text can be selected before the later assistant reply; the issue is also paired with open overlapping implementation PRs that should be consolidated rather than bypassed.

## What This Changes

The issue asks OpenClaw to avoid announcing stale raw subagent tool JSON after auto-compaction and instead wait for the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats compaction markers as continuation boundaries in output selection and completion capture, preserves timeout partial-progress behavior, and adds focused regression tests for the limbo and recovery windows.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a history containing a toolResult followed by a synthetic compaction system marker and no later assistant text still leaves `latestRawText` selectable as the completion output.

## Solution Assessment

No for a run-flow-only retry. The narrow maintainable fix needs to cover the shared selector and frozen completion capture path, not only retry delayed assistant text inside `runSubagentAnnounceFlow`.
