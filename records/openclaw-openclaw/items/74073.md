Keep open. ## What This Changes

The issue asks OpenClaw to avoid returning stale subagent tool JSON after auto-compaction and instead wait for or select the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for subagent output selection and frozen completion capture, preserves timeout partial-progress behavior, and updates focused regression tests plus user-facing docs/changelog if raw fallback semantics change.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a child history containing a toolResult, followed by the synthetic compaction system marker and no later assistant text, still leaves latestRawText selectable as subagent output on current main.

## Solution Assessment

No for the narrow unified-retry-only direction. The safer fix should handle the snapshot selector and frozen completion-capture paths, not only recheck delayed assistant text inside runSubagentAnnounceFlow.
