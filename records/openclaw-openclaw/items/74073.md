## Summary

Keep open. The checked-out main still has a source-level path where a pre-compaction tool result remains selectable as subagent completion output, and the linked open PRs have not yet landed a consolidated fix for both announce delivery and frozen completion capture.

## What This Changes

The issue reports that subagent auto-compaction can cause stale tool JSON to be announced to the requester instead of the delayed post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for subagent output selection and frozen completion capture, while preserving timeout partial-progress behavior and adding focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists with a child history containing a toolResult followed by the synthetic compaction system marker and no later assistant text; current main can still return the stale latestRawText from that history.

## Solution Assessment

No for the narrow unified-retry-only direction. The safer solution should address the snapshot selector and capture/frozen-result paths, not only recheck delayed assistant text inside runSubagentAnnounceFlow.
