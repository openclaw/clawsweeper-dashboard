## Summary

Keep open. Current main still lets the subagent output selector fall through to pre-compaction raw tool text, and the issue is paired with open overlapping implementation PRs rather than a merged fix.

## What This Changes

The issue reports that auto-compaction can make subagent completion output selection publish stale raw tool JSON instead of the later assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries in output selection and completion capture, preserves timeout partial-progress behavior, and adds focused regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is enough: chat history with a toolResult followed by a synthetic compaction system marker and no later assistant text still leaves latestRawText selectable as the completion output.

## Solution Assessment

No for the unified-retry-only direction. The narrow maintainable fix should cover both readSubagentOutput selection and frozen completion capture, not only retry delayed assistant text in runSubagentAnnounceFlow.
