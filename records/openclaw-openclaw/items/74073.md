## Summary

Keep open. Current main still has a source-level path where a compaction marker after a tool result does not pause subagent output selection, so stale raw tool output can be selected before a later assistant reply; #74113 also has closing syntax for this issue and #73783/#74340 are open overlapping fixes that should be consolidated rather than bypassed.

## What This Changes

The issue reports that subagent auto-compaction can leave `readSubagentOutput` selecting stale raw tool output before the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats compaction markers as continuation boundaries in shared output selection and completion capture, preserves timeout partial-progress behavior, and adds focused regression coverage for both limbo and recovery windows.

## Reproduction Assessment

Yes, at source level. A `chat.history` snapshot containing a `toolResult` followed by the synthetic compaction system marker and no later assistant text leaves the prior raw tool result selectable as completion output.

## Solution Assessment

No for the issue's suggested run-flow-only retry by itself. The maintainable fix should update the shared selector and frozen completion capture path, not only add a retry inside `runSubagentAnnounceFlow`.
