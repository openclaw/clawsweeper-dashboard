## Summary

Keep open. Current main still has a source-level path where a synthetic compaction marker does not clear stale raw tool output before subagent output selection, and an open PR has closing syntax for this issue, so this should not be closed before a consolidated fix lands.

## What This Changes

The issue reports that subagent auto-compaction can make `readSubagentOutput` return pre-compaction raw tool JSON as the completion announcement instead of the delayed post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats compaction system markers as continuation boundaries in subagent output selection and frozen completion capture, preserves timeout partial-progress behavior, and adds regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a child history with a raw `toolResult`, then a synthetic compaction system marker, and no later assistant text; current main can still leave that stale raw text selectable as subagent output.

## Solution Assessment

No for the narrow unified-retry-only direction described in the issue. The safer fix should cover the selector and frozen completion capture path, not only retry delayed assistant text inside `runSubagentAnnounceFlow`.
