## Summary

Keep open. Current main still has a source-level path where a compaction marker does not pause subagent output selection, allowing stale raw tool text to be selected before a later assistant reply; this issue is also already paired with open implementation PRs that should be reviewed together.

## What This Changes

Prevent subagent announcement and completion capture from selecting stale raw tool output during auto-compaction before the post-compaction assistant reply arrives.

## Best Possible Solution

Land one consolidated agents fix that treats compaction markers as continuation boundaries in the shared output selector and frozen completion capture path, with focused regression coverage for limbo, recovery, and timeout partial-progress behavior.

## Reproduction Assessment

Yes, at source level. A chat history containing a toolResult followed by a synthetic compaction system marker and no later assistant text leaves latestRawText selectable as the completion output on current main.

## Solution Assessment

No for a run-flow-only retry. The narrow maintainable fix should cover the shared selector and completion capture path, because frozen result capture calls the same readSubagentOutput path outside the announce-flow branch.
