## Summary

Keep open. Current main still has the stale raw-output selection path described by this issue, and the issue is paired with open overlapping PRs rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop subagent announce/completion capture from returning stale raw tool output after auto-compaction before the final assistant reply is available.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for subagent output selection and frozen completion capture, while preserving timeout partial-progress behavior and adding focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a `chat.history` snapshot containing a tool result, then a synthetic compaction system marker, and no later assistant text; current main can still preserve and return the tool result as selected output.

## Solution Assessment

No for a unified-retry-only fix. The maintainable solution should prevent stale raw output from being selected in both the announce reader and the frozen completion capture path, not only recheck delayed assistant text in the announce flow.
