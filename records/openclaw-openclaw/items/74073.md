## Summary

Keep open. Current main still has a source-level path where a toolResult immediately before a compaction marker can be selected as subagent output, and the issue is paired with open implementation PRs rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop subagent auto-compaction races from announcing stale raw tool JSON instead of the delayed post-compaction assistant reply.

## Best Possible Solution

Review and land one consolidated agents fix that treats compaction as a pending continuation boundary, prevents successful completions from freezing raw tool output, preserves timeout partial-progress behavior, and adds selector plus announce/capture regression coverage.

## Reproduction Assessment

Yes, at source level. A `chat.history` response with raw `toolResult` text followed by the synthetic compaction system marker and no later assistant text still leaves `latestRawText` selectable before any latest-assistant fallback runs.

## Solution Assessment

No for the issue's narrow one-shot retry by itself. The safer fix should cover the shared selector, completion capture, frozen-result path, and final announce construction so pending compaction is not treated as a successful terminal child result.
