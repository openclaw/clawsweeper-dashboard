## Summary

Keep open. Current main still has a source-level path where subagent output selection can fall through to stale raw tool output after a compaction marker, and the issue is already paired with open overlapping PRs rather than a merged fix.

## What This Changes

The issue reports that subagent announce completion can publish raw tool-result JSON after auto-compaction instead of waiting for the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for output selection and frozen completion capture, while preserving timeout partial-progress behavior and adding focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a chat history containing a toolResult, then a synthetic compaction system marker, and no later assistant text; current readSubagentOutput can select latestRawText before checking for a post-compaction assistant reply.

## Solution Assessment

No for a retry-only fix. The safer solution should address the selector and capture/frozen-result paths, not only add a late assistant recheck inside runSubagentAnnounceFlow.
