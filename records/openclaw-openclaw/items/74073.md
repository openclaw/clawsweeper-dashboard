## Summary

Keep open. Current main still has a source-level path where a tool/toolResult entry before a synthetic compaction marker remains selectable as subagent completion output, and this issue is already paired with open fix PRs including one with closing syntax, so it should not be auto-closed before a consolidated fix lands.

## What This Changes

Fix subagent announce/completion output selection so auto-compaction waits for post-compaction assistant output instead of forwarding stale raw tool JSON to the parent session.

## Best Possible Solution

Land one consolidated agents fix that treats compaction system markers as continuation boundaries in the output snapshot and frozen completion capture paths, preserves timeout partial-progress behavior, adds regression coverage for toolResult-then-compaction limbo and delayed assistant recovery, and updates the user-facing docs/changelog if the raw fallback contract changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a child chat history containing a toolResult text, then a synthetic system compaction marker with no later assistant text; current main keeps latestRawText selectable and returns it as subagent output.

## Solution Assessment

No for the issue's narrow unified-retry-only direction. The safer fix should handle the compaction marker in the shared output summarizer/capture path, not only recheck delayed assistant text inside runSubagentAnnounceFlow.
