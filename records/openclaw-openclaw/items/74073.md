## Summary

Keep open. Current main still has a source-level path where a tool/toolResult remains selectable after a synthetic compaction marker, and an open PR has closing syntax for this issue, so it should not be closed before a consolidated fix lands.

## What This Changes

Fix subagent announce completion so auto-compaction mid-turn waits for post-compaction assistant output instead of forwarding stale tool/toolResult JSON to the parent session.

## Best Possible Solution

Land one consolidated agents fix that treats synthetic compaction markers as continuation boundaries for subagent output selection and frozen completion capture, keeps timeout partial-progress behavior, and updates regression tests, docs, and changelog together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a child history with a toolResult, then the synthetic compaction system marker, and no later assistant text can still leave latestRawText selectable as subagent output on current main.

## Solution Assessment

No for the narrow unified-retry-only direction. The safer solution should cover the output summarizer’s compaction marker handling and the frozen completion capture path, not only recheck delayed assistant text inside runSubagentAnnounceFlow.
