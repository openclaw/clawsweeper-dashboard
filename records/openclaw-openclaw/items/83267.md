## Summary

Keep open: the PR fixes the documented validation mismatch and has sufficient proof, but one blocking routing bug remains. A timeout-only bundled overlay becomes a provider config entry, and the existing fallback path can then synthesize unknown built-in models instead of preserving the current unknown-model error.

## What This Changes

The PR relaxes model provider config validation so bundled provider entries can supply timeout-only overlays, materializes accepted overlays, guards fallback/Foundry paths, and adds focused tests.

## Best Possible Solution

Allow bundled provider overlays while keeping fallback model synthesis limited to real custom provider registrations or explicit configured model entries.

Yes. Source and linked proof show current main rejects the documented bundled timeout overlay, and PR-head source shows the remaining fallback-synthesis path for overlay-only provider configs.

## Solution Assessment

No, not yet. Allowing bundled overlays is the right direction, but the implementation needs one more guard so overlay presence does not become custom-provider fallback permission.
