## Summary

Keep this PR open: it fixes the reported OpenAI timeout-overlay path and has useful proof, but the latest diff still leaves provider-alias validation and the exported config contract out of sync with the accepted schema shape.

## What This Changes

The PR relaxes model-provider config validation for bundled-provider overlays, adds timeout-overlay tests, and guards fallback/Foundry paths that can encounter provider entries without `models`.

## Best Possible Solution

Make overlay eligibility share the same provider-id and manifest-alias semantics as model resolution, then either split raw overlay input from normalized provider config or update the exported config contract and guarded call sites together.

## Reproduction Assessment

Yes. Current main source requires `baseUrl` and `models` during raw config validation, and the PR proof logs show the documented OpenAI timeout-only overlay failing before and validating after the patch.

## Solution Assessment

No. The PR is directionally right, but the implementation should use the same provider alias/ownership contract as model resolution and keep exported config types aligned with the schema it accepts.
