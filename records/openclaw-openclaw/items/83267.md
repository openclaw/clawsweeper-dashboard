## Summary

Keep open: the PR is a useful fix with solid proof, but the current patch still checks overlay eligibility with a copied trim/lower provider-id set and leaves accepted overlay-only config entries inconsistent with the exported provider config type.

## What This Changes

The PR relaxes model-provider validation for bundled timeout-only provider overlays, applies the timeout during model resolution, guards configured-provider fallback and Microsoft Foundry overlay paths, and adds focused tests.

## Best Possible Solution

Use a config-safe canonical provider/manifest alias source for bundled overlay eligibility, keep custom providers strict, and split or align raw overlay typing with the materialized provider config contract.

## Reproduction Assessment

Yes from source and proof: current main requires baseUrl/models for every models.providers entry, and the PR proof log shows an OpenAI timeout-only overlay failing before and passing after. I did not run local tests because this review kept the checkout read-only.

## Solution Assessment

No. The direction is right, but the implementation should use canonical provider/manifest alias ownership and a raw-vs-materialized config contract instead of a copied lowercased provider list with a strict exported type.
