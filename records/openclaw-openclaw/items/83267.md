## Summary

Keep open: this is a focused bug fix with useful before/after proof, but the current patch is not merge-ready because the overlay whitelist misses at least one current built-in provider alias and the exported config type remains out of sync. The protected `maintainer` label also means this PR should stay open for explicit maintainer handling rather than cleanup closure.

## What This Changes

The PR changes model provider config validation so selected bundled provider entries can act as timeout overlays without declaring `baseUrl` and `models`, while preserving strict custom-provider validation and adding focused regression tests.

## Best Possible Solution

Keep the overlay fix, but make the bundled-provider boundary match the actual provider and alias registry, keep exported config types aligned with the new accepted shape, and cover both OpenAI and provider-alias overlays with focused tests.

## Reproduction Assessment

Yes. Current source requires `baseUrl` and `models` for every `models.providers.*` entry, and the PR proof shows the OpenAI timeout overlay failing before the patch and passing after it.

## Solution Assessment

No. Allowing bundled provider overlays is the right fix direction, but the current implementation hard-codes an incomplete provider list and leaves the public config type inconsistent with validation.
