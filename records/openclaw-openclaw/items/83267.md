## Summary

Keep this PR open: it fixes the reported OpenAI overlay path and has solid proof, but the current patch still leaves supported bundled provider aliases outside the overlay path and leaves the exported config type inconsistent with the relaxed schema.

## What This Changes

The PR allows timeout-only config overlays for bundled model providers, guards fallback/Foundry paths that can see overlay-only provider entries, and adds focused regression coverage.

## Best Possible Solution

Make overlay eligibility use the same provider ownership and alias semantics as model resolution, keep unknown custom providers strict, and align the raw config type/tests with accepted overlay-only entries.

## Reproduction Assessment

Yes. Current source requires `baseUrl` and `models` for every `models.providers.*` entry, and the PR proof logs show that the documented OpenAI timeout-only overlay fails before the patch and validates after it.

## Solution Assessment

No. The overlay approach is the right fix, but the current static raw-id allowlist and unchanged exported config type are not the narrowest maintainable contract for bundled provider overlays.
