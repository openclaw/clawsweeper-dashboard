## What This Changes

The issue requests purpose-scoped GitHub auth profiles so private tracking and public contribution workflows can use separate least-privilege tokens with visible redacted purpose metadata and clear missing-scope failures.

## Best Possible Solution

Design a maintainer-approved GitHub skill/plugin credential-purpose contract with SecretRef-backed purpose names, explicit gh capability checks, redacted purpose metadata, unchanged default behavior, focused tests, and least-privilege docs.
