## What This Changes

The branch adds bundled skill-creator guidance to route durable OpenClaw skill work through Skill Workshop proposals and removes the direct `init_skill.py` scaffold helper plus its test.

## Best Possible Solution

Keep Skill Workshop as the single agent-facing persistence boundary; land this deletion if maintainers accept that `init_skill.py` is not a supported public contract, or replace it with a proposal bridge if they do consider it public.

## Reproduction Assessment

Yes, source-level: current main has `init_skill.py` writing a live `SKILL.md` directly, while the current prompt and docs require durable generated skill work to go through `skill_workshop`. I did not run the direct helper as a repro because this review was read-only and that path writes files.

## Solution Assessment

Yes, likely: deleting the unreferenced live-write helper is the narrowest clean fix for the bypass and matches the one-canonical-path repository policy. If maintainers treat the helper as public API, the safer alternative is a proposal bridge rather than preserving live writes.
