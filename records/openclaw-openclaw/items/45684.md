This PR adds Slack DM thread matching for message-tool attachments with tests, while also adding unrelated dependency patch and Windows test changes.

## Best Possible Solution

Yes, from source inspection rather than a live Slack run. A DM context with To=user:... and NativeChannelId=D... stores only the D-channel id, and resolveSlackAutoThreadId rejects user: targets before it can return currentThreadTs.

## Solution Assessment

No, not as currently packaged. The two-field Slack design is the right direction, but the branch needs to be moved onto the current extensions/slack implementation and stripped of unrelated package and CI changes before merge.
