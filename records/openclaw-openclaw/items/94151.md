## Summary

Keep open: this is the right Slack plugin layer for the reported identity regression, but the patch is not merge-ready because it adds custom identity fields to `chat.update` without a no-identity retry when Slack rejects them, does not add focused regression coverage, and the supplied proof is test-only.

## What This Changes

This PR threads optional `SlackSendIdentity` through Slack edit helpers and preview/draft edit callers so `chat.update` payloads can include custom username and avatar fields.

## Best Possible Solution

Land a narrow Slack plugin fix that forwards identity through agent-reply edit paths, preserves no-identity edit fallback on Slack rejection, includes focused regression tests, and has redacted live Slack proof.

## Reproduction Assessment

Yes, source-level. Current main resolves Slack identity for sends, but `editSlackMessage`, draft-stream edits, and preview-finalize edits do not pass identity into `chat.update`; I did not run a live Slack workspace reproduction.

## Solution Assessment

No as submitted. Threading identity through the Slack plugin is the right layer, but the best fix must preserve edit delivery when Slack rejects custom identity fields and must prove real Slack behavior before merge.
