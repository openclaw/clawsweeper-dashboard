## Summary

Keep open: current main and v2026.6.6 still expose the documented piped installer path with subprocesses that can inherit the script stream, and https://github.com/openclaw/openclaw/pull/87799 is an open, unmerged closing PR for the stdin-isolation fix.

## What This Changes

The issue reports that the documented `curl -fsSL https://openclaw.ai/install.sh | bash` installer path can hang and corrupt the streamed script when child processes read stdin.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/87799; once a focused stdin-isolation fix reaches main, close this issue and the canonical related installer-stdin issue as implemented.

## Reproduction Assessment

Yes at source level: current docs pipe the installer through stdin, current `scripts/install.sh` lets gum-wrapped and fallback subprocesses inherit stdin, and gum v0.17.0 forwards `os.Stdin` to wrapped commands. I did not run a full live WSL2 installer reproduction in this read-only review.

## Solution Assessment

Yes, the best path is a focused installer fix that isolates child-process stdin only for piped or noninteractive contexts while preserving direct interactive stdin. A docs-only `bash -s --` change is too narrow, and the older broad pipe-guard PR is closed unmerged.
