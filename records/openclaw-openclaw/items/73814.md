## Summary

Keep open. Current main still documents the bare piped installer command and still has an installer onboarding path that can inherit the curl stream; the linked fix PR #73830 remains open and unmerged, so this issue should stay open until that work lands, is revised, or is replaced.

## What This Changes

The issue reports that `curl ... install.sh | bash` can hang on WSL2 and eventually execute a truncated `warn_shell_path_missing_dir` name because interactive installer code may consume the script stream.

## Best Possible Solution

Make the piped installer path TTY-safe by routing interactive prompts and child onboarding/doctor handoffs through `/dev/tty` or explicit noninteractive mode, then align public install commands and add focused coverage for that path.

## Reproduction Assessment

Unclear. The issue has concrete WSL2 steps and current main has a matching stdin-inheritance path, but this read-only review did not run the mutating installer and current smoke coverage uses `--no-prompt`.

## Solution Assessment

No. A docs-only `bash -s --` change is useful but not a complete fix because child commands can still inherit the script stream; installer-side TTY redirection is the narrower maintainable boundary.
