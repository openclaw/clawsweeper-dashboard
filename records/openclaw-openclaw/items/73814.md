## Summary

Keep open. Current main still documents bare `curl ... | bash` install commands and still has a bootstrap onboarding path that can inherit the installer script stream; PR #73830 is open with closing syntax, so this issue should remain open until that PR is landed, revised, or replaced.

## What This Changes

The issue reports that the recommended piped `install.sh` command can hang on WSL2 Ubuntu and leave bash executing a truncated `warn_shell_path_missing_dir` function name after an interactive prompt consumes stdin.

## Best Possible Solution

Make the default piped installer path pipe-safe by routing bootstrap onboarding and other interactive child handoffs through `/dev/tty` or explicit noninteractive mode, then align public install commands and add focused coverage for that path.

## Reproduction Assessment

Unclear. The issue has concrete WSL2 steps and logs, and current main still has a matching stdin-inheritance path, but this read-only review did not run the mutating installer.

## Solution Assessment

No. A docs-only `bash -s --` change is not the best complete fix because child processes can still inherit the script stream; the narrow maintainable fix belongs in installer TTY handling first, with docs updated afterward.
