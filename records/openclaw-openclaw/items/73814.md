## Summary

Keep open. Current main and the latest release still expose the reported bare-pipe installer path, and the installer still has an onboarding handoff that can inherit the piped script stdin; the linked docs-only PR is open and unmerged, so this issue should remain the canonical bug until that work is revised, replaced, or landed.

## What This Changes

The issue requests that the macOS/Linux installer stop hanging or consuming script stdin when users run the documented `curl ... install.sh | bash` command.

## Best Possible Solution

Make `scripts/install.sh` robust under piped execution by ensuring interactive child commands such as bootstrap onboarding read from `/dev/tty` or are skipped in noninteractive mode, then align the public install examples and add focused coverage for the default piped installer path.

## Reproduction Assessment

Unclear. The issue gives concrete WSL2 Ubuntu 24.04 steps and current source shows a plausible stdin-consuming child process, but this read-only review did not execute the mutating installer end to end.

## Solution Assessment

No. A docs-only `bash -s --` change is not the best complete fix because child processes can still inherit the script pipe; the installer should first close or redirect stdin for interactive handoffs, with docs updated afterward.
