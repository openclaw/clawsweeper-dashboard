## Summary

Keep open. Current main still documents bare `curl | bash` installer commands and still has a bootstrap onboarding path that invokes `openclaw onboard` without redirecting stdin from `/dev/tty`; PR #73830 is open and unmerged, so this should stay open until that work is reviewed, revised, landed, or replaced.

## What This Changes

The issue reports that the documented `curl -fsSL https://openclaw.ai/install.sh | bash` installer hangs on WSL2 and can execute a truncated `warn_shell_path_missing_dir` name after interactive prompts consume the script stream.

## Best Possible Solution

Make the default piped installer path pipe-safe by routing interactive prompts and child onboarding/doctor handoffs through `/dev/tty` or explicit noninteractive mode, then align the public install commands and focused installer smoke coverage.

## Reproduction Assessment

Unclear. The reporter provides concrete WSL2 Ubuntu 24.04 `curl | bash` steps and current main still has a matching stdin-inheritance path, but this read-only review did not run the mutating installer.

## Solution Assessment

No. A docs-only `bash -s --` change in PR #73830 is not the complete fix while bootstrap `openclaw onboard` can still inherit the script stream; the narrower maintainable fix belongs in installer TTY handling first, with docs updated afterward.
