## Summary

Keep open. Current main and the hosted installer still expose the reported default `curl ... | bash` path, the bootstrap onboarding handoff can still inherit the installer stream, and the linked fix PR remains open with unresolved review concerns.

## What This Changes

The issue reports that the documented piped installer command can hang and mangle `scripts/install.sh` when interactive prompts inherit the curl stream.

## Best Possible Solution

Make the default installer path pipe-safe in `scripts/install.sh` by routing interactive child commands through `/dev/tty` or a tested noninteractive mode, then align all public install examples and add focused installer smoke coverage.

## Reproduction Assessment

No, not at high confidence from this read-only pass. The reporter provided concrete WSL2 steps and current source preserves a plausible stdin-inheritance path, but I did not run the mutating installer flow end to end.

## Solution Assessment

No, the currently linked docs-only `bash -s --` proposal is not the best complete fix because it does not harden child-process stdin. The safer path is installer-side `/dev/tty` handling plus consistent docs and smoke proof.
