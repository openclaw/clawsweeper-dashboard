## Summary

Keep open. Current main still documents the reported bare `curl ... | bash` installer path and still has an onboarding handoff that can inherit the piped script stream; the open closing PR is an implementation candidate, not a reason to close before merge.

## What This Changes

The issue reports that the documented `curl ... | bash` installer can hang on WSL2 when interactive installer subprocesses consume the script stream and truncate later shell code.

## Best Possible Solution

Make the default piped installer path pipe-safe in `scripts/install.sh`, update all public install examples consistently, and add focused smoke coverage for interactive child handoffs under piped stdin.

## Reproduction Assessment

Unclear. The report has concrete WSL2 steps and logs, and current main preserves a plausible stdin-inheritance path, but this read-only review did not run the mutating installer.

## Solution Assessment

No. A docs-only `bash -s --` change is not the best complete fix while installer subprocesses can still inherit the script stream unless the code redirects interactive handoffs to `/dev/tty` or disables them.
