## Summary

Keep open. Current main still recommends bare piped install commands and still has a bootstrap onboarding path that invokes `openclaw onboard` without redirecting stdin from `/dev/tty`; because PR #73830 is unmerged and linked with closing syntax, this issue should remain open until that work is landed, revised, or replaced.

## What This Changes

The issue reports the piped `curl ... | bash` installer consuming its own script stream during interactive setup, causing a hang and a truncated `warn_shell_path_missing_dir` command.

## Best Possible Solution

Make the default piped installer path pipe-safe by routing all interactive prompts and child onboarding/doctor handoffs through `/dev/tty` or explicit noninteractive mode, then align the public install commands and add focused coverage for that path.

## Reproduction Assessment

Unclear. The issue has concrete WSL2 steps and current source still has a matching stdin-inheritance path, but this read-only review did not run the mutating installer and existing smoke coverage uses `--no-prompt`.

## Solution Assessment

No. A docs-only `bash -s --` change is not the best complete fix because child processes can still inherit the script stream; the narrow maintainable fix belongs in installer TTY handling first, with docs updated afterward.
