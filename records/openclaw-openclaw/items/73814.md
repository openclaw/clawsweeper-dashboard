## Summary

Keep open. Current `main` still publishes the reported bare piped install command and still has an onboarding path that can inherit the script stream as stdin; the linked fix PR remains open and unmerged, so this issue should not be closed before that work is landed, revised, or replaced.

## What This Changes

The issue asks OpenClaw to stop the documented piped `curl ... install.sh | bash` installer from hanging or consuming its own script stream during interactive onboarding on WSL2 Ubuntu.

## Best Possible Solution

Make `scripts/install.sh` robust under piped execution by redirecting interactive onboarding handoffs to `/dev/tty` or skipping them in noninteractive mode, then align public install examples and add focused coverage for the piped interactive path.

## Reproduction Assessment

Unclear. The issue has concrete WSL2 steps and current source still has a matching inherited-stdin path, but this read-only review did not run the mutating default installer; existing smoke coverage uses `--no-prompt`.

## Solution Assessment

No. The open docs-only PR may improve one command example, but the narrower maintainable fix should harden installer TTY handling first and then update docs/tests around that behavior.
