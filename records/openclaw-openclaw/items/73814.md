Keep open. Current main still publishes the reported piped installer command and still has installer child/onboarding paths that can inherit the script stream; the linked closing PR #73830 is open and unmerged, so this issue should remain open until that fix is landed, revised, or replaced.

## What This Changes

The issue asks for the macOS/Linux installer one-liner to avoid hanging or mangling `install.sh` when interactive installer work runs under `curl ... | bash`.

## Best Possible Solution

Make the default piped installer path pipe-safe by routing interactive child commands through `/dev/tty` or explicit noninteractive stdin, then align public install examples and add focused installer smoke coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. The reporter provided concrete WSL2 steps and current source preserves a plausible inherited-stdin path, but I did not run the mutating installer to reproduce the exact hang and truncation.

## Solution Assessment

No, the linked docs-only `bash -s --` direction is not the best complete fix. `bash -s` still reads the script from stdin, while the installer should make interactive child handoffs explicitly pipe-safe and then update docs.
