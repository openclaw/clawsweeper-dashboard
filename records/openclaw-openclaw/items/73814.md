## Summary

Keep open. Current main still publishes bare `curl ... | bash` installer commands and still has a source-level bootstrap onboarding handoff that can inherit the installer script stream; PR #73830 remains open and unmerged, so this issue should stay open until that candidate lands, is revised, or is replaced.

## What This Changes

The issue reports that the documented piped `install.sh` command can hang on WSL2 and consume the script stream, truncating `warn_shell_path_missing_dir` during interactive installer flow.

## Best Possible Solution

Make the default piped installer path pipe-safe by routing interactive child commands through `/dev/tty` or explicit noninteractive mode, then align public install examples and add focused installer smoke coverage.

## Reproduction Assessment

No, not at high confidence from this read-only pass. The reporter provided concrete WSL2 steps and current source preserves a plausible stdin-inheritance path, but I did not run the mutating installer to reproduce the exact hang and truncation.

## Solution Assessment

No. A docs-only `bash -s --` change is not the best complete fix while `scripts/install.sh` still has an onboarding handoff that can inherit the script stream; the installer should make interactive handoffs pipe-safe and then update docs.
