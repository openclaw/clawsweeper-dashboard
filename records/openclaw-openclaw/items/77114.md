## Summary

Keep this PR open for normal merge handling: current main still forwards embedded control characters in WhatsApp outbound document filenames, and the proposed patch is a narrow code/test/changelog fix with no concrete review finding from this pass.

## What This Changes

The PR strips C0/DEL control characters from WhatsApp outbound document `fileName` values before the existing trim/default behavior, adds regression tests, records the lint suppression, and adds a changelog entry.

## Best Possible Solution
