## Summary

Keep open. Current main and the latest release still preserve Discord's bot-owned autoThread mention bypass, and the open related PR only partially overlaps because it explicitly keeps that bypass above its new thread override.

## What This Changes

The issue requests a Discord thread mention-gating mode that responds to thread starters without a mention but requires explicit mentions for later thread replies.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy that can opt bot-owned autoThread follow-ups into mention gating while preserving current defaults and coordinating with the broader thread override work.

## Reproduction Assessment

Yes, source-level. Current main resolves bot-owned autoThread threads to no mention requirement before the no-mention drop path, so unmentioned follow-up replies are admitted.

## Solution Assessment

Unclear until maintainers choose the public API. The safest direction is an additive opt-in Discord policy, not an unconditional default change or the currently open ordinary-thread override alone.
