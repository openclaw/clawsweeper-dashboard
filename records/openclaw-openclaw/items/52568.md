## Summary

Keep open. Current main still has the Discord bot-owned autoThread mention bypass and no Discord config surface for starter-free but reply-gated threads, while the related open reports cover broader thread/strict-mention behavior rather than fully replacing this narrower request.

## What This Changes

Add a Discord per-channel option that lets autoThread starter messages trigger freely while requiring explicit mentions for later replies in the created thread.

## Best Possible Solution

Add an opt-in Discord-owned thread-reply mention policy in config, schema, docs, and preflight, preserving free handling of the original autoThread starter while requiring explicit mentions for later replies when enabled.

## Reproduction Assessment

Yes, from source inspection. The helper bypasses mention requirements for bot-owned autoThreads, preflight drops only when that helper says mentions are required, and the current unit test asserts the bypass.

## Solution Assessment

Yes in principle, but the config surface needs maintainer agreement. The narrow maintainable path is Discord-owned and preflight-enforced, with Slack's `thread.requireExplicitMention` as precedent and coordination with the broader strict-mention issue.
