## Summary

Keep open. Current main still lacks a Discord reply-only mention gate: bot-owned autoThreads bypass mention requirements, and the open broader strict implicit-mention issue does not replace this narrower starter-free autoThread workflow.

## What This Changes

Add a Discord per-channel policy that lets autoThread respond to the root message without a mention but requires explicit mentions for later thread replies.

## Best Possible Solution

Add a backward-compatible Discord-owned thread-reply mention policy through config types, schema, docs, and preflight, preserving existing defaults unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes, source-level. A Discord channel with `autoThread` still treats bot-owned autoThreads as not requiring mention, existing tests preserve that behavior, and the accepted config/schema surface has no reply-only gate.

## Solution Assessment

Unclear until maintainers choose the public API. The requested additive option is directionally sound, but it should be aligned with https://github.com/openclaw/openclaw/issues/62066 and avoid changing existing defaults accidentally.
