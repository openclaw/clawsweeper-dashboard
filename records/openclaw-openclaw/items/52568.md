## Summary

Keep open. Current main still has the Discord auto-thread mention bypass and no Discord config surface for the requested middle-ground policy; related #40895 and #62066 are relevant but do not clearly supersede this focused autoThread starter-versus-reply request.

## What This Changes

The issue requests a Discord channel config option that lets autoThread starter messages trigger without a mention while requiring explicit mentions for later replies in the created thread.

## Best Possible Solution

Add an opt-in Discord thread-reply mention policy in the Discord-owned config surface, enforce it in preflight before model invocation for bot-owned autoThread follow-up replies, preserve free handling of the original channel starter message, and update schema/types/generated config metadata, docs, tests, and changelog together.

## Reproduction Assessment

Yes. The current unit path is high-confidence: `resolveDiscordShouldRequireMention` returns `false` for a bot-owned autoThread, and preflight only drops unmentioned guild messages when that resolved requirement is true.

## Solution Assessment

Yes in principle. The narrowest maintainable fix is an opt-in Discord-owned config modeled on Slack’s explicit thread mention behavior, without changing Slack or other channel semantics unless #62066 is intentionally taken up as a broader shared policy.
