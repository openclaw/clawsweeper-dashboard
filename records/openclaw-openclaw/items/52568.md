## Summary

Keep open. Current main and v2026.6.6 still have the bot-owned Discord autoThread mention bypass and no config key for requiring mentions on follow-up replies, while the related open PR preserves that bypass and only covers an ordinary thread override.

## What This Changes

The issue requests a Discord autoThread mode where the agent responds to the thread-starter message without a mention but requires explicit mention for later replies in that thread.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy that can opt bot-owned autoThread follow-ups into mention gating while preserving current defaults, coordinated with https://github.com/openclaw/openclaw/pull/76058 or a replacement.

## Reproduction Assessment

Yes, source-level. With `autoThread: true`, current `resolveDiscordShouldRequireMention` returns false for bot-owned threads before channel or guild `requireMention`, so unmentioned follow-up replies are not dropped by the mention gate.

## Solution Assessment

Unclear until maintainers choose the public API. The safest direction is an additive opt-in Discord policy, not an unconditional default change or the currently open ordinary-thread override alone.
