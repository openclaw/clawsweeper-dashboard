## Summary

Keep open. Current main and v2026.6.6 still lack a Discord autoThread follow-up mention gate, and the related open PR at https://github.com/openclaw/openclaw/pull/76058 explicitly preserves the bot-owned autoThread bypass this issue needs to control.

## What This Changes

The issue requests a Discord config mode where autoThread starter messages can trigger without a mention, while later replies in the spawned thread require an explicit mention.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy that can opt bot-owned autoThread follow-up replies into mention gating while preserving current defaults.

## Reproduction Assessment

Yes, source-level. Current Discord preflight clears the mention requirement for bot-owned autoThreads, so unmentioned follow-up replies are not dropped by the mention gate.

## Solution Assessment

Unclear until maintainers choose the public API. The safest direction is an additive opt-in Discord policy, not an unconditional default change to existing thread-continuation behavior.
