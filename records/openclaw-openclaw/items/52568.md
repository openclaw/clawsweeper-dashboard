## Summary

Keep open. Current main still bypasses mention gating for Discord bot-owned autoThreads and lacks the requested Discord config surface; related #40895 and #62066 overlap but do not fully replace this narrower starter-versus-reply request.

## What This Changes

Add a Discord per-channel policy that lets autoThread starter messages trigger without a mention while requiring mentions for later replies in the created thread.

## Best Possible Solution

Add an opt-in Discord-owned thread-reply mention policy in the Discord config/schema/docs, enforce it before model invocation for bot-owned autoThread follow-up replies, and preserve free handling of the original channel starter message.

## Reproduction Assessment

Yes, from source inspection. The helper returns no mention requirement for bot-owned autoThread threads, preflight only drops unmentioned guild messages when that requirement is true, and the current unit test explicitly preserves the bypass.

## Solution Assessment

Yes in principle. The narrow maintainable path is an opt-in Discord-owned config modeled on Slack's explicit thread mention behavior, without changing Slack or taking on the broader cross-channel strict mode unless #62066 is intentionally addressed.
