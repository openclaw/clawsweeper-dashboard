## Summary

Keep open. Current main still lacks a Discord config surface for starter-free but reply-gated autoThreads, and the existing Discord preflight path still clears mention requirements for bot-owned autoThreads before the no-mention drop can run; the broader strict-mention issue is related but does not replace this exact Discord per-channel policy request.

## What This Changes

Request a Discord per-channel option that lets autoThread starter messages trigger without mention while requiring explicit mention for later replies in the created thread.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy, wire it through config types/schema/docs and preflight, and coordinate the naming/default with the broader strict implicit-mention design.

## Reproduction Assessment

Yes, from source inspection. A Discord channel with autoThread enabled still treats bot-owned autoThreads as not requiring mention, and there is no accepted config key to require explicit mention only for follow-up replies.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrowest maintainable direction is an additive Discord thread-reply policy aligned with Slack's existing explicit-thread-mention precedent and the broader strict-mention issue.
