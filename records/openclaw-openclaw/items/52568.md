## Summary

Keep open. Current main and the latest release still preserve Discord's bot-owned autoThread mention bypass, and the related open PR only partially overlaps because it explicitly preserves that bypass.

## What This Changes

The issue requests a Discord per-channel mode that replies to an auto-thread starter without requiring a mention but requires explicit mentions for later replies in that thread.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy that can opt bot-owned autoThread follow-ups into mention gating while preserving current defaults and coordinating with the related thread override work.

## Reproduction Assessment

Yes, source-level. Current main resolves bot-owned autoThread threads to no mention requirement before the no-mention drop path, so unmentioned follow-up replies are admitted.

## Solution Assessment

Unclear until maintainers choose the public API. The safest direction is an additive opt-in Discord policy, not an unconditional default change or the currently open ordinary-thread override alone.
