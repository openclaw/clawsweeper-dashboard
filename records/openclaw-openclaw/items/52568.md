## Label Justifications

- `impact:message-loss`: The issue is about Discord mention gating deciding which thread replies are delivered to the agent and which are suppressed.

## Summary

Keep open. Current main and v2026.6.8 still preserve the bot-owned Discord autoThread mention bypass and lack the requested follow-up reply gate, while the related open PR is only partial because it explicitly preserves that bypass.

## What This Changes

Add a per-channel Discord config option that lets auto-thread starter messages trigger without a mention while requiring explicit mentions for follow-up replies in the created thread.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy that can opt bot-owned autoThread follow-ups into mention gating while preserving current defaults and coordinating with the related thread override work.

## Reproduction Assessment

Yes, source-level. Current main resolves bot-owned autoThread threads to `shouldRequireMention=false`, so unmentioned follow-up replies are not dropped by the mention gate.

## Solution Assessment

Unclear until maintainers choose the public API. The safest direction is an additive opt-in Discord policy, not an unconditional default change or the currently open ordinary-thread override alone.
