## Summary

Keep open. Current main and v2026.5.22 still lack the requested Discord autoThread follow-up mention gate, and the open Discord thread override PR explicitly preserves the bot-owned autoThread bypass that this issue needs to control. The remaining work needs maintainer API/default judgment coordinated with https://github.com/openclaw/openclaw/issues/62066 and https://github.com/openclaw/openclaw/pull/76058.

## What This Changes

Request a Discord channel config option that lets autoThread starter messages trigger without a mention while requiring explicit mentions for later replies in that thread.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy that can opt bot-owned autoThread follow-up replies into mention gating while preserving current defaults, then coordinate naming and precedence with the broader strict implicit-mention work.

## Reproduction Assessment

Yes, source-level. Current Discord preflight still treats bot-owned autoThreads as not requiring mention, and the accepted config/schema surface has no reply-only gate to change that behavior.

## Solution Assessment

Unclear until maintainers choose the public API. The narrowest maintainable direction is an additive Discord thread-reply policy aligned with the broader strict implicit-mention work, not an unconditional default change.
