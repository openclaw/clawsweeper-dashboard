## Summary

Keep open. Current main still lacks the requested Discord reply-only mention policy, and the broader strict implicit-mention issue is related but does not replace this starter-free autoThread workflow without a Discord config/API decision.

## What This Changes

The issue requests a Discord per-channel option that lets autoThread respond to the starter message without a mention while requiring explicit mentions for later thread replies.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy, wire it through config types/schema/docs and preflight, and coordinate naming/defaults with the broader strict implicit-mention design while preserving existing defaults unless maintainers choose otherwise.

## Reproduction Assessment

Yes, from source inspection and current tests. A Discord channel with `autoThread` still treats bot-owned autoThreads as not requiring mention, and the accepted config/schema surface has no reply-only gate.

## Solution Assessment

Unclear until maintainers choose the public API. The narrowest maintainable direction is an additive Discord thread-reply policy aligned with the broader strict implicit-mention issue, not an unconditional default change.
