- `impact:message-loss`: The issue is about channel mention-gating behavior causing agent replies to Discord thread messages the user expects to be suppressed unless explicitly mentioned.

## Summary

Keep open. Current main and the latest release still lack the requested Discord autoThread follow-up mention gate, and the related open Discord thread override PR explicitly preserves the bot-owned autoThread bypass that this issue needs to control.

## What This Changes

Add a Discord per-channel policy so `autoThread` can answer the starter message without a mention but require explicit mention for later replies in the created thread.

## Best Possible Solution

Define an additive Discord-owned thread-reply mention policy that can opt bot-owned autoThread follow-up replies into mention gating while preserving existing defaults, and coordinate naming/defaults with the broader strict implicit-mention design.

## Reproduction Assessment

Yes, source-level. Current Discord preflight still treats bot-owned autoThreads as not requiring mention, and the accepted config/schema surface has no reply-only gate to change that behavior.

## Solution Assessment

Unclear until maintainers choose the public API. The narrowest maintainable direction is an additive Discord thread-reply policy aligned with the broader strict implicit-mention work, not an unconditional default change.
