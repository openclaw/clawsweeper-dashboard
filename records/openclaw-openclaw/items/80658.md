## Summary

This PR is still useful as a possible docs clarification, but it should not merge as written because current main and the latest release already preserve a supported local non-browser `gateway-client` backend path under `gateway.auth.mode: "none"`.

## What This Changes

The PR expands the `gateway.auth.mode: "none"` configuration-reference note to say backend WebSocket clients fail with `device identity required` and should use token mode.

## Best Possible Solution

Merge a narrower config-reference note that says auth-none does not provide a shared token while preserving the reserved local non-browser `gateway-client` backend exception and naming custom, remote, browser-origin, or otherwise non-exempt failures.

## Reproduction Assessment

Yes. ## Solution Assessment

No as written. The maintainable docs fix is narrower wording that documents the missing shared token without erasing the supported local backend exception already implemented on main and in `v2026.6.10`.
