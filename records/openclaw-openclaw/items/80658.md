## Summary

Keep open: the docs gap is real, but the proposed warning is too broad because current main supports reserved direct-loopback `gateway-client` backend RPCs under auth-none while rejecting browser-origin or non-exempt clients.

## What This Changes

The branch expands the `gateway.auth.mode: "none"` configuration-reference note to say no shared token is generated and backend clients should use token mode to avoid `device identity required` failures.

## Best Possible Solution

Merge a narrower note that says auth-none does not mint a shared token while preserving the local non-browser `gateway-client` backend exception and naming custom, remote, browser-origin, or otherwise non-exempt clients as the failing cases.

## Reproduction Assessment

Yes, source-reproducible: current tests assert auth-none local backend `gateway-client` connects without device identity and browser-origin backend rejects with `device identity required`. I did not execute tests because this is a read-only review.

## Solution Assessment
