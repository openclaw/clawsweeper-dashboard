The PR changes `docs/gateway/configuration-reference.md` to warn that `gateway.auth.mode: "none"` provides no shared token and that backend WebSocket clients should use token mode.

## Best Possible Solution

Merge a narrower docs note: `none` does not generate a shared token, but only remote/browser/custom device-less WebSocket clients outside the reserved direct-local `gateway-client` helper path should expect `device identity required`.

## Reproduction Assessment

Yes. Current source and tests show the exact split: auth-none direct-local backend helper connects succeed, while browser-origin or non-exempt device-less paths still fail with `device identity required`.

## Solution Assessment

No as written. The maintainable fix is narrower documentation that separates the true no-shared-token fact from the specific client classes that still need device identity or shared-secret auth.
