## Summary

Keep open. The resource-exhaustion concern is still relevant on current main, but this branch is not merge-ready because its memory change targets a removed core path and does not cover the active memory-core or QMD watchers.

## What This Changes

This PR adds chokidar `depth` options to the skills watcher and the old core memory watcher to reduce recursive watch traversal in large trees.

## Best Possible Solution

Implement a rebased watcher resource policy in the active skills and memory-core paths, with focused tests asserting chokidar options and a docs update if recursive memory watch semantics are intentionally bounded.

## Reproduction Assessment

Yes: the code-level reproduction is high-confidence by inspecting the current chokidar option objects and watcher tests; the active calls lack `depth` and `followSymlinks`. I did not run a live FD exhaustion stress test because this was a read-only review.

## Solution Assessment

No: the PR’s skills-side idea is plausible, but the memory hunk is stale and the patch does not cover the active builtin or QMD memory watchers. A safer fix should retarget current memory-core code and explicitly decide the depth/symlink policy.
