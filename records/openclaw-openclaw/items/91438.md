Full review comments:

- **[P1] Honor toolPolicy for Teams visual tools:** `extensions/voice-call/src/msteams-realtime.ts:416-421`
  - body: `realtime.toolPolicy: "none"` is supposed to suppress the built-in consult path, and `safe-read-only` is supposed to restrict the embedded agent's tools. This PR still exposes `show_to_caller` whenever the msteams realtime runtime has agent deps, then runs that consult with `toolPolicy: "owner"`, so a setup that explicitly disabled or restricted agent tools can still invoke owner-level screenshot/image tooling from a call. Gate the msteams visual tools on the configured policy, or add an explicit owner-level opt-in before exposing this tool.
  - confidence: 0.92
- **[P3] Avoid sending undefined as the ambient frame label:** `extensions/voice-call/src/msteams-realtime.ts:588`
  - body: When a frame has no `participantName`, `describeMsteamsVideoFrameOwner(frame)` returns `undefined`, so the ambient image item text becomes `Live view — undefined.`. That leaks an implementation placeholder into the model context and can confuse visual answers for 1:1 or anonymous/older-worker calls; omit the owner suffix unless it exists.
  - confidence: 0.87
