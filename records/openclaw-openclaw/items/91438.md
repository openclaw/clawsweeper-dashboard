## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Make bridge HMAC upgrades single-use:** `extensions/voice-call/src/msteams-media-stream.ts:362-366`
  - body: The upgrade handler accepts any request with a valid timestamp/signature pair and only blocks duplicate callIds while the first socket remains in `sessions`. After that socket closes, cleanup deletes the callId state, so the exact same `(callId,timestamp,signature)` can reconnect during the HMAC window. Record verified tuples until their timestamp expires and reject exact replays.
  - confidence: 0.9
- **[P2] Enforce the realtime group gate outside the model:** `extensions/voice-call/src/msteams-realtime.ts:500-503`
  - body: The streaming path deterministically returns before responding to unaddressed group turns, but the realtime path only appends an instruction while `autoRespondToAudio` remains enabled. That makes the advertised “speak only when addressed” behavior model-trusted for realtime meetings, so unaddressed participant speech can still produce audio. Add an egress/turn suppression path or narrow the realtime claim.
  - confidence: 0.78
