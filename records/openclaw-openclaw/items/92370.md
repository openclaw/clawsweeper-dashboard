## What This Changes

## Best Possible Solution

Carry canonical OpenClaw config through the affected tool-call registration or session path, preserve pending approval when policy is unavailable, and retain explicit `approvalPolicy: "auto"` as the only configured bypass.

Yes from source inspection: a workshop lifecycle call without config resolves to the pending default and enters plugin approval routing, matching the reported immediate failure path. A live current-main Feishu reproduction was not run during this read-only review.

## Solution Assessment

No. Bypassing approval when the policy cannot be read fixes the symptom by failing open; repairing canonical config propagation is the narrower and safer solution.
