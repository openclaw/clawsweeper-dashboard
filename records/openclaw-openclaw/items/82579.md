## Summary

Keep open: this is useful plugin trust-boundary hardening, but the current patch is not merge-ready because it treats every disabled workspace plugin as untrusted and can break trusted workspace channel setup/inspection flows.

## What This Changes

The branch adds a setup-only loader predicate, a registry channel-registration backstop, and loader/registry tests to block untrusted workspace channel plugin execution.

## Best Possible Solution

Land a trust check keyed to workspace-default/untrusted disablement, keep trusted disabled setup-only channel metadata available, and decide whether this superseding branch should absorb the registerChannel error-isolation guard.

## Reproduction Assessment

Yes. Source inspection shows current main can setup-load a scoped disabled channel plugin, and the PR fixture demonstrates the untrusted workspace bypass path without needing a live plugin install.

## Solution Assessment

No. The proposed shape is close, but it uses the collapsed enabled boolean instead of the specific untrusted workspace-default disablement reason and therefore blocks valid trusted workspace setup-only cases.
