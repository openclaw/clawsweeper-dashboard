## Summary

## What This Changes

This PR adds a Control UI cron formatter and presenter tests to display common five-field hourly/minute cron patterns as `Every ...` labels while preserving raw fallback and time zone suffixes.

## Best Possible Solution

Keep the presenter-level formatter, validate minute/hour step ranges against Croner before humanizing, add invalid-step regression coverage, and require redacted after-fix Control UI or terminal proof.

## Reproduction Assessment

Yes for source-level review: current main returns raw `Cron ${s.expr}` text, and PR head would render `*/70 * * * *` or `0 */25 * * *` as valid-looking intervals even though Croner 10.0.1 rejects those wildcard steps. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before humanizing and keep invalid schedules on the raw fallback.
