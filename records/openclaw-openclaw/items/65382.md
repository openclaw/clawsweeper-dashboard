## Summary

This PR is still useful and not implemented on current main, but it should stay open because the patch still humanizes invalid cron wildcard steps and the contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds a Control UI formatter for common five-field cron minute/hour intervals and presenter tests for the new display strings.

## Best Possible Solution

Keep the formatter in `ui/src/ui/presenter.ts`, but bound minute/hour wildcard steps to Croner's accepted ranges, preserve the raw fallback for unsupported or invalid cron, and require after-fix UI or terminal proof before merge.

## Reproduction Assessment

Yes for source-level review. Current main's formatter returns raw cron text, and PR head's parser accepts out-of-range wildcard steps without checking Croner's minute/hour limits.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but the implementation should validate step ranges before humanizing and keep invalid schedules on the raw fallback.
