## Summary

Keep open: current main still renders cron schedules as raw cron text, and this PR is the narrow remaining Cron Jobs formatter work after https://github.com/openclaw/openclaw/pull/65199, but it needs cron step-range validation and real behavior proof before merge.

## What This Changes

The PR adds a Control UI presenter helper and tests to show common five-field minute/hour cron schedules as `Every ...` labels while preserving raw fallback and timezone suffixes.

## Best Possible Solution

Keep the presenter-level formatter, bound step values to Croner’s minute/hour ranges, add invalid-step regression coverage, and require redacted after-fix UI or terminal proof.

## Reproduction Assessment

Yes for source-level review: current main returns raw cron text, and PR head would humanize `*/70 * * * *` or `0 */25 * * *` because it only checks for safe positive integers while Croner rejects steps greater than the field length. I did not run tests because this sweep is read-only.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before returning a human label and keep invalid cron strings on the raw fallback.
