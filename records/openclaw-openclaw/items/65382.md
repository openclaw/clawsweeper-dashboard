## Summary

Keep open: current main still renders cron schedules as raw cron text, and the related broader PR is closed unmerged, so this focused UI formatter is not obsolete. It is not merge-ready because the new helper can make invalid cron step expressions look valid, and the external PR still has only test/CI evidence rather than after-fix real behavior proof.

## What This Changes

The branch adds a UI presenter helper and tests to display common minute/hour cron expressions as human-readable intervals while retaining the raw fallback for unsupported expressions.

## Best Possible Solution

Keep the narrow presenter-based formatter direction, but validate wildcard step bounds against the runtime cron contract, preserve raw fallback for invalid or unsupported schedules, add regression tests, and require redacted real Control UI proof before merge.

## Reproduction Assessment

Yes for source-level PR review: the added helper accepts any positive safe integer step, while OpenClaw schedules through Croner, which rejects steps greater than the minute/hour field length. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as proposed. The presenter is the right narrow layer for a display-only improvement, but the formatter must match the runtime cron contract and keep invalid expressions on the raw fallback.
