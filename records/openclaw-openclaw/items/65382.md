## Summary

This PR should stay open because current main still renders cron schedules as raw cron text, and the closed broader PR at https://github.com/openclaw/openclaw/pull/65199 explicitly left this narrower Cron Jobs formatter as the canonical remaining UI enhancement. It is not merge-ready yet because the formatter humanizes invalid wildcard steps and the contributor has not posted after-fix real behavior proof.

## What This Changes

The branch adds a presenter-level formatter and tests so common five-field cron minute/hour interval expressions display as `Every ...` while unsupported schedules fall back to raw cron text.

## Best Possible Solution

Keep the narrow presenter formatter, validate minute/hour wildcard steps against Croner-compatible bounds, add invalid-step fallback tests, then require redacted Control UI or terminal proof.

## Reproduction Assessment

Yes for source-level review: current main returns raw cron text, and the PR head accepts any safe positive wildcard step before humanizing it. Croner 10.0.1 rejects zero or out-of-range steps, so the invalid-label path is clear from source and dependency contract inspection.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before humanizing and keep invalid schedules on the raw fallback.
