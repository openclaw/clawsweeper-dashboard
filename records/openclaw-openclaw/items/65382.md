## Summary

Keep open: the PR is useful and current main still shows raw cron syntax, but the patch humanizes out-of-range wildcard steps that Croner rejects and the external PR only provides test/CI proof, not real after-fix behavior proof.

## What This Changes

Adds a Control UI formatter for common 5-field cron minute/hour intervals and presenter tests for hourly, minute, timezone, and fallback output.

## Best Possible Solution

Keep the presenter boundary, validate wildcard cron steps against Croner's minute/hour limits before humanizing them, keep unsupported schedules on the raw fallback, and have maintainers handle any release-note bookkeeping during landing.

## Reproduction Assessment

Yes. Source inspection shows current main renders cron schedules through the raw `Cron ...` fallback, and the PR's remaining range bug is source-reproducible from `parsePositiveCronStep` accepting any safe positive integer.

## Solution Assessment

No. The presenter helper is the right narrow place for this UI display improvement, but the implementation should bound minute/hour wildcard steps to Croner's accepted values and fall back for invalid schedules.
