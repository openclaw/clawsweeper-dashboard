## Summary

Keep open: this is a focused Control UI formatter improvement that is not on current main or the latest release, but it is not merge-ready because the formatter can label non-exact or invalid Croner wildcard steps as exact intervals and the PR only provides unit-test-level proof.

## What This Changes

The PR adds a Control UI formatter and tests so common 5-field cron schedules display as “Every ...” while preserving timezone annotations and raw fallback for unsupported expressions.

## Best Possible Solution

Keep the narrow presenter-level improvement, but only humanize Croner-compatible exact minute/hour intervals, keep raw cron fallback for all other steps, and add visible Control UI proof before merge.

## Reproduction Assessment

Yes for the PR-introduced formatter bug: source inspection shows any safe positive integer step is humanized, while Croner’s source treats wildcard steps as field-bounded. I did not run a live Control UI reproduction because this review is read-only.

## Solution Assessment

No as-is. The presenter is the right narrow UI boundary, but the formatter should only return “Every ...” for exact Croner-compatible intervals and otherwise preserve the raw cron label.
