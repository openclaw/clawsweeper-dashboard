## Summary

Keep open: current main still renders cron schedules as raw text, so this focused UI improvement is not obsolete, but the PR is not merge-ready because it lacks real behavior proof and currently humanizes step values outside Croner's valid field ranges.

## What This Changes

The branch adds Control UI presenter logic and tests to display common five-field cron hour/minute step expressions as human-readable interval labels while keeping raw cron fallback for unmatched expressions.

## Best Possible Solution

Land a narrow presenter formatter only after adding field-range validation, focused regression tests, a current-main refresh, and real UI or terminal proof showing common schedules and fallback behavior.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection shows the new helper accepts unbounded positive step values, while Croner rejects steps greater than the minute/hour field length. No live browser reproduction was run because this was a read-only review.

## Solution Assessment

No as-is. The presenter helper is the right narrow boundary, but it should mirror Croner bounds for minute/hour steps and keep invalid expressions on the raw fallback before visual proof is accepted.
