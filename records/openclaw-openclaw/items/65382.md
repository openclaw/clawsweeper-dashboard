The overlapping cron-humanizer PR is still open and broader, so it does not supersede this narrower PR cleanly.

## What This Changes

The PR adds Control UI formatting for common five-field cron minute/hour expressions and presenter tests for the new labels and fallback behavior.

## Best Possible Solution

Keep the narrow Control UI formatter only after it validates minute/hour step ranges against Croner's contract and coordinates with the broader humanizer thread to avoid duplicate implementations.

## Reproduction Assessment

Yes for source-level review: current main still returns raw cron text, and PR head accepts out-of-range wildcard steps that Croner rejects. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should bound minute/hour step values before humanizing and keep invalid schedules on the raw fallback.
