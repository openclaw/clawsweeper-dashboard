## Summary

Keep open: current main has not implemented this formatter and the narrower PR remains the canonical Cron Jobs display path, but the current patch still has a range-validation correctness finding and lacks real behavior proof.

## What This Changes

This PR adds a Control UI presenter helper and tests to render common five-field cron schedules as `Every ...` labels while preserving raw fallback and timezone suffixes.

## Best Possible Solution

Keep the presenter-level formatter, mirror Croner’s minute/hour step bounds before humanizing, add invalid-step regression coverage, and require redacted after-fix Control UI or terminal proof.

## Reproduction Assessment

Yes for source-level review: current main returns raw cron text, and PR head would humanize `*/70 * * * *` or `0 */25 * * *` even though Croner 10.0.1 rejects wildcard steps greater than the minute/hour field length. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before humanizing and keep invalid schedules on the raw fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
