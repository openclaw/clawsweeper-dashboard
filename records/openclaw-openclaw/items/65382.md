## Summary

Keep open: current main still renders cron schedules as raw cron text, so the PR is not obsolete, but the patch is not merge-ready because it humanizes out-of-range cron steps that Croner rejects and still lacks contributor after-fix real behavior proof.

## What This Changes

The PR adds a Control UI formatter and presenter tests for displaying common five-field cron minute/hour patterns as “Every ...” labels while preserving the raw fallback for unsupported expressions.

## Best Possible Solution

Keep the narrow presenter formatter, but bound minute steps to 1-60 and hour steps to 1-24, preserve raw fallback for invalid expressions, and add real Control UI or terminal proof before merge.

## Reproduction Assessment

Yes for source-level review: current main still returns raw cron text, and PR head would format `*/70 * * * *` or `0 */25 * * *` as plausible intervals even though Croner 10.0.1 rejects those wildcard steps. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before humanizing and keep invalid schedules on the raw fallback.
