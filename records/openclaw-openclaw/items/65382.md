## Best Possible Solution

Keep the presenter-level formatter, but validate minute/hour step bounds against Croner before humanizing and require redacted after-fix Control UI or terminal proof.

## Reproduction Assessment

Yes for source-level review: current main returns raw cron text, and PR head would humanize invalid wildcard steps that Croner rejects. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before humanizing and keep invalid schedules on the raw fallback.
