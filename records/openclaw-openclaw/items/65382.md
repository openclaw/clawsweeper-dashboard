## Summary

Keep open: current main and the latest release still render cron schedules as raw `Cron ...` text, and the broader related PR was closed in favor of this narrower formatter. The branch is useful but not merge-ready because the formatter humanizes out-of-range cron steps and the PR still has only unit-test proof.

## What This Changes

The PR adds a Control UI presenter formatter and tests so common five-field cron minute/hour interval expressions display as `Every ...` while preserving timezone text and raw fallback for unsupported schedules.

## Best Possible Solution

Keep the narrow presenter-based formatter direction, but validate wildcard step bounds against the runtime cron contract, preserve raw fallback for invalid or unsupported schedules, add regression tests, and require redacted real Control UI proof before merge.

## Reproduction Assessment

Yes for source-level review: current main returns raw cron text, and the PR head accepts any safe positive wildcard step before humanizing it. Croner 10.0.1 rejects zero or out-of-range steps, so the invalid-label path is clear from source and dependency inspection.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before humanizing and keep invalid schedules on the raw fallback.
