## Best Possible Solution

Keep the presenter-level formatter, validate minute/hour wildcard steps against Croner before humanizing, add invalid-step coverage, and require redacted after-fix Control UI or terminal proof.

## Reproduction Assessment

Yes for source-level review: current main returns raw cron text, and PR head accepts any safe positive wildcard step before humanizing it. ## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should validate minute/hour step ranges before humanizing and keep invalid schedules on the raw fallback.
