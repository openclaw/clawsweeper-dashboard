## Summary

Keep open: this PR is a useful, narrow Cron Jobs display improvement that is not on current main, and its follow-up commit addressed earlier coverage/hourly comments, but the branch still humanizes out-of-range wildcard steps that Croner rejects and lacks contributor real behavior proof.

## What This Changes

The PR adds a Control UI formatter and presenter tests for rendering common five-field cron minute/hour schedules as `Every ...` while preserving raw cron fallback and timezone suffixes.

## Best Possible Solution

Use the narrower Cron Jobs formatter direction, but validate minute/hour wildcard steps against Croner's field limits, preserve raw fallback for invalid or unsupported cron, and consolidate with the overlapping humanizer PR if maintainers keep that thread open.

## Reproduction Assessment

Yes for source-level review: current main still returns raw cron text, and PR head accepts out-of-range wildcard steps that Croner rejects. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as currently proposed. The presenter helper is the right narrow boundary, but it should bound minute/hour step values before humanizing and keep invalid schedules on the raw fallback.
