## Summary

Keep open. Current main and the latest release still rotate terminal main sessions when the transcript mtime is newer than the registry entry, and the linked open PR is the active candidate that uses closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to preserve the existing session id for aborted Gateway sessions after restart so lossless-claw does not bootstrap from a fresh empty JSONL transcript and quarantine the conversation.

## Best Possible Solution

Keep this issue open as the canonical report while maintainers review https://github.com/openclaw/openclaw/pull/94567 or choose an alternate restart-continuity design, then close it only after the chosen fix lands.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection gives a clear reproduction path: current main rotates terminal main sessions when the transcript mtime is newer than the registry entry, matching the reported precondition.

## Solution Assessment

Unclear as final product direction. The linked PR is a coherent candidate, but it adds a new public session config contract, so maintainers should choose between that opt-in shape and a broader config-free recovery design before this issue closes.
