## Summary

Keep open. Current main and the latest release still have the terminal-main transcript freshness gate that can rotate to a new sessionId, and the concrete fix is currently an open linked PR that has not merged yet.

## What This Changes

The issue requests preserving a restarted Gateway session's existing sessionId/JSONL transcript for aborted lossless-claw conversations so a fresh transcript does not trigger LCM replay quarantine.

## Best Possible Solution

Review the open linked PR; if maintainers accept the restartContinuation contract, land it with config metadata, sibling resolver coverage, and regression tests, otherwise fold this narrower issue into the broader restart-recovery design.

## Reproduction Assessment

Yes at source level: current main blocks reuse when a terminal main transcript is newer than the registry and existing tests assert rotation in that condition. I did not run a live Gateway plus lossless-claw restart reproduction.

## Solution Assessment

Unclear. The open PR is a plausible opt-in implementation and covers the sibling resolver paths, but accepting a new session.restartContinuation config versus a config-free recovery design is a maintainer product decision.
