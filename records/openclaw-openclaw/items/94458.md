## Summary

Keep open: current main still has no `session.restartContinuation` surface, the session resolvers still rotate when the terminal main transcript is newer than the registry entry, and two open closing PRs are active candidate fixes rather than merged evidence.

## What This Changes

The issue requests Gateway restart session continuity so lossless-claw does not quarantine after a fresh JSONL transcript is created for the same deterministic session key.

## Best Possible Solution

Review and land one maintainer-approved restart-continuation contract shared by auto-reply, agent command, and Gateway agent session resolution, or close the candidate PRs if maintainers reject the new config direction.

## Reproduction Assessment

No high-confidence live reproduction was run, but the issue is source-reproducible: current main treats a newer terminal main transcript mtime as a reuse blocker and mints a new session id. The external lossless-claw quarantine step is supported by the report and linked PR proof, not independently exercised here.

Unclear until maintainer review settles the config contract; a config-gated fix may be appropriate, but it needs consistent resolver coverage and default-off upgrade proof.
