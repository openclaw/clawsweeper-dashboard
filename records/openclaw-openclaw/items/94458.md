## Summary

Keep open: current main still rotates terminal main sessions to a fresh UUID/JSONL when the transcript mtime is newer than the registry timestamp, and two open PRs already reference this issue as the fix path. The remaining work is to review and land one compatibility-safe implementation rather than close the issue early.

## What This Changes

The issue requests restart-safe sessionId and JSONL transcript continuity for aborted deterministic Gateway sessions so lossless-claw does not quarantine after restart.

## Best Possible Solution

Land one maintainer-approved restart-continuation contract shared by auto-reply, agent command, and Gateway agent session resolution, then close this issue through the merged fixing PR.

## Reproduction Assessment

Yes at source level, but not from a live run. Current main excludes terminal main sessions with a newer transcript mtime from fresh reuse and then mints a new UUID, matching the reported restart-continuity failure mode.

## Solution Assessment

Unclear as product direction. Preserving the existing sessionId for aborted restart continuation is the right invariant, but the new config option and exact resolver coverage need maintainer approval before merge.
