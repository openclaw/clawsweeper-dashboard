## Summary

Keep open. Current main and v2026.6.9 still lack `session.restartContinuation` and still rotate terminal main sessions when the transcript mtime is newer than the registry row, while https://github.com/openclaw/openclaw/pull/94567 is the open closing PR that owns the candidate fix.

## What This Changes

Request to preserve aborted Gateway session IDs across restarts, via opt-in `session.restartContinuation`, so lossless-claw does not treat the fresh transcript file as a replay.

## Best Possible Solution

Keep this issue as the canonical narrow report while maintainers review https://github.com/openclaw/openclaw/pull/94567, then close it after that PR lands or after maintainers choose a different restart-continuity design.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a medium-confidence path: current main rotates terminal main sessions when the transcript is newer than the registry row, matching the reported precondition.

## Solution Assessment

Unclear as final product direction. The linked PR is a coherent candidate, but it adds a new session config contract, so maintainers should choose between that opt-in shape and a broader config-free restart recovery design before this issue closes.
