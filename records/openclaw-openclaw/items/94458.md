## What This Changes

The issue requests preserving an aborted Gateway session's existing `sessionId` across restart, proposed as an opt-in `session.restartContinuation` setting, so lossless-claw does not see a fresh JSONL transcript as replay.

## Best Possible Solution

Keep this issue open as the canonical report while maintainers review the linked PR, then close it only after a maintainer-approved restart-continuity design lands or the product direction is rejected.
