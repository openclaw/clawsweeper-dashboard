## Summary

Keep open. Current main and the latest release still lack `session.restartContinuation` and still rotate terminal-main sessions when the transcript mtime is newer than the registry, while the linked implementation candidate at https://github.com/openclaw/openclaw/pull/94567 remains open and unmerged.

## What This Changes

This issue requests an opt-in Gateway session-continuity mode that reuses an aborted session's existing `sessionId` after restart so lossless-claw does not bootstrap from a fresh JSONL transcript and quarantine the conversation.

## Best Possible Solution

Keep this as the canonical narrow lossless-claw restart-continuity report while maintainers review https://github.com/openclaw/openclaw/pull/94567 or choose a different approved restart-continuity design.

## Solution Assessment
