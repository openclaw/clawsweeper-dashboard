## Summary

Keep open. Current main still contains and tests the terminal-main transcript rotation path that can mint a new sessionId when the transcript mtime is newer than the registry, and the requested opt-in continuation behavior is not implemented; the broader graceful-restart tracker overlaps but does not explicitly own the LCM transcript identity invariant.

## What This Changes

Request an opt-in Gateway session restart-continuation mode that reuses an aborted session’s existing sessionId and JSONL transcript for lossless-claw-backed sessions after restart.

## Best Possible Solution

Define a maintainer-approved restart-continuation contract that preserves transcript identity for restart-aborted sessions without weakening the existing terminal-main rotation, archive, checkpoint, and usage-tracking invariants.

## Reproduction Assessment

Yes at source level, but not as a live LCM restart reproduction. Current main source and tests show terminal main sessions rotate to a new sessionId when the transcript is newer than the registry; the LCM quarantine result was not executed here.

## Solution Assessment

Unclear. Reusing the existing sessionId when abortedLastRun is true is plausible, but the best fix needs a maintainer decision on whether this belongs as a new config option, an existing restart-recovery rule, or a narrower context-engine contract.
