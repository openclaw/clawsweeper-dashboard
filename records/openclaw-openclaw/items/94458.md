## Summary

Keep open. Current main and the latest release still have the terminal-main transcript freshness gate and no `session.restartContinuation` config, while https://github.com/openclaw/openclaw/pull/94567 is an open fixing PR that uses closing syntax for this issue.

## What This Changes

The issue requests restart-continuation behavior so an aborted Gateway session can reuse its existing sessionId/JSONL transcript identity instead of triggering lossless-claw quarantine after restart.

## Best Possible Solution

Keep this issue open as the canonical report while maintainers decide and either land https://github.com/openclaw/openclaw/pull/94567 or choose a different restart-continuity design.

## Reproduction Assessment

Yes at source level, but not with a live end-to-end run: current main has no `restartContinuation` setting and still blocks reuse when the terminal main transcript is newer than the registry. A high-confidence live reproduction would require Gateway restart with lossless-claw and a Discord/channel session.

## Solution Assessment

Unclear as final product direction: the open PR is a coherent implementation candidate, but adding `session.restartContinuation` is a new public config contract and maintainers may prefer a config-free recovery design.
