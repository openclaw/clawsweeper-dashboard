## Summary

Keep open. The PR targets a real Discord guild-admin authorization gap, but it should not merge as-is because the new runtime guard rejects trusted local/CLI owner guild-admin actions that do not have a Discord requester id; after-fix real Discord behavior proof is also missing.

## What This Changes

The PR adds Discord trusted-requester gating, forwards requester sender ids into guild-admin actions, enforces guild-admin permission checks, and adds authorization regression tests.

## Best Possible Solution

Keep the Discord guild-admin hardening, but carry an explicit trusted execution signal or scope the mandatory sender check to tool-driven Discord requester contexts, with tests for both unauthorized requester rejection and trusted local owner success.

## Reproduction Assessment

Yes. Source inspection shows current main lacks a Discord trusted-requester hook for guild-admin message actions, and the PR regression is also source-reproducible because `openclaw message` sets `senderIsOwner` but not `requesterSenderId` while the PR requires `senderUserId` in the guild runtime.

## Solution Assessment

No. The direction is correct, but the runtime-level missing-sender rejection is broader than the established moderation trust boundary; the safer solution distinguishes untrusted Discord requester-driven calls from trusted owner CLI/manual execution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve trusted CLI guild-admin actions without sender IDs:** `extensions/discord/src/actions/runtime.guild.ts:184`
  - body: Do not unconditionally throw when `senderUserId` is absent here: `openclaw message` marks local CLI calls as `senderIsOwner: true` but does not pass `requesterSenderId`, so owner commands such as `role-add`, `emoji-upload`, `event-create`, and `channel-delete` now fail before reaching Discord. Limit this guard to requester-driven contexts or pass an explicit trusted-owner signal through this runtime path.
  - confidence: 0.91
