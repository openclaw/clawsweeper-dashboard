## Summary

Keep open. The PR addresses a real Discord guild-admin authorization bug, but the current patch still blocks trusted local owner/manual guild-admin actions and lacks after-fix real Discord behavior proof.

## What This Changes

This PR adds Discord trusted-requester gating and guild-admin permission checks for privileged guild-admin message actions, plus focused regression tests.

## Best Possible Solution

Land a corrected Discord hardening that gates Discord-origin guild-admin tool calls by trusted requester identity and matching Discord permissions while preserving trusted owner/manual execution with an explicit trust signal or scoped runtime checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path where a Discord guild-admin action can reach bot-credentialed mutation code without requester authorization, and the PR regression is source-reproducible through local owner paths that carry senderIsOwner but no requesterSenderId.

## Solution Assessment

No. The trusted-requester direction is right, but the runtime-level missing-sender rejection is too broad; it should be scoped to requester-driven contexts or carry explicit trusted-owner context.
