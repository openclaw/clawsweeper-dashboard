## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject case-variant cron job IDs:** `src/cron/service/jobs.ts:518-520`
  - body: `createJob` only checks `job.id === id`, while the new pattern allows uppercase letters. On case-insensitive filesystems, `Daily` and `daily` can resolve to the same `runs/<jobId>.jsonl` file and mix or clobber run history, so custom IDs need canonical casing or case-insensitive collision checks before persistence.
  - confidence: 0.9
- **[P2] Return duplicate IDs as invalid requests:** `src/cron/service/jobs.ts:520`
  - body: The duplicate guard throws a plain `Error`, but Gateway `cron.add` only maps `TypeError` and `RangeError` from `context.cron.add` to `INVALID_REQUEST`. Duplicate IDs through Gateway or CLI can therefore surface as internal failures instead of validation errors.
  - confidence: 0.86
- **[P2] Regenerate cron protocol clients:** `src/gateway/protocol/schema/cron.ts:293`
  - body: Adding `id` to `CronAddParamsSchema` changes the Gateway wire contract, but the generated Swift `CronAddParams` model still has no `id` property. Typed clients cannot send the new field until generated protocol artifacts are refreshed.
  - confidence: 0.86
