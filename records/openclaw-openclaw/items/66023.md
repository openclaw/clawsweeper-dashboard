## Summary

Keep open. Current main still lacks supported custom cron IDs, and this PR is a focused implementation for https://github.com/openclaw/openclaw/issues/65636, but it still has blocking correctness/security/proof gaps and overlaps with https://github.com/openclaw/openclaw/pull/75065.

## What This Changes

The PR adds `--id` to `openclaw cron add`, threads optional custom IDs through normalization, Gateway validation, cron create types/service logic, and focused tests.

## Best Possible Solution

Land one custom cron-ID implementation after making IDs canonical and file-safe, mapping duplicate-ID failures as validation errors, refreshing generated clients, and adding redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection of current main shows no CLI `--id`, a Gateway add schema that rejects `id`, a create type that omits `id`, and a service path that always calls `crypto.randomUUID()`; the PR-specific blockers are also source-reproducible from the diff and surrounding code.

## Solution Assessment

No. The end-to-end direction is narrow and maintainable, but this patch is not the best merge shape until it fixes case-safe ID collisions, Gateway duplicate-error mapping, and generated protocol clients.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject case-variant cron job IDs:** `src/cron/service/jobs.ts:519`
  - body: `createJob` only checks exact ID equality, but the new pattern allows uppercase letters. On case-insensitive filesystems, `Daily` and `daily` can both be created and resolve to the same `runs/<jobId>.jsonl` file, mixing run history. Canonicalize IDs or reject case-insensitive duplicates before persisting.
  - confidence: 0.9
- **[P2] Return duplicate IDs as invalid cron.add requests:** `src/cron/service/jobs.ts:520`
  - body: The duplicate guard throws a plain `Error`, but the Gateway `cron.add` handler only converts `TypeError` and `RangeError` from `context.cron.add` into `INVALID_REQUEST`. A duplicate ID through Gateway or CLI will bypass the promised explicit validation response unless this error is mapped or thrown as a validation error.
  - confidence: 0.84
- **[P2] Regenerate the cron protocol clients:** `src/gateway/protocol/schema/cron.ts:293`
  - body: Adding `id` to `CronAddParamsSchema` changes the Gateway wire contract, but the committed Swift `CronAddParams` model still has no `id` field and this PR does not update generated protocol artifacts. Typed clients cannot send the new value until the generated models are refreshed.
  - confidence: 0.86
