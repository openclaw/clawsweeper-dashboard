- **[P2] Reject case-variant cron job IDs:** `src/cron/service/jobs.ts:518-520`
  - body: `createJob` checks only exact ID equality while the new pattern allows uppercase letters.   - confidence: 0.9

  - body: The duplicate guard throws a plain `Error`, while Gateway `cron.add` only converts `TypeError` and `RangeError` from `context.cron.add` into `INVALID_REQUEST`. A duplicate ID through Gateway or CLI can therefore surface as an internal failure instead of the promised validation response.
  - confidence: 0.86
