## Summary

Keep open: the fix direction is source-reproducible and matches the linked Slack issue, but this external PR still has mock-only proof and the new opt-out is not regenerated into bundled Slack channel metadata.

## What This Changes

The PR retains current-bot Slack root/history messages in new thread sessions, labels them as assistant context, adds `channels.slack.thread.includeRootMessage`, and updates Slack tests, docs, config types, and changelog.

## Best Possible Solution

Keep the root-message retention approach, regenerate bundled Slack channel metadata/UI hints for the opt-out, and add redacted live Slack or diagnostic proof before merge.

## Reproduction Assessment

Yes from source and focused fixture shape: current main fetches Slack thread history but removes current-bot starter/history before building `ThreadHistoryBody`, so a bot-rooted new thread session loses the parent text. I did not establish a live Slack workspace repro in this read-only review.

## Solution Assessment

No as submitted: retaining current-bot root/history for new thread sessions is the right fix direction, but the opt-out must be regenerated into bundled channel metadata and the proof should cover real Slack runtime behavior rather than only mocks.
