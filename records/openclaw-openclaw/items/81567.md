## Summary

Keep this open: current main and the latest release still make GPT-4o a selectable OpenAI model, but the strict-agentic no-plan-only-stop contract is intentionally scoped to GPT-5-family OpenAI/Codex runs, so this needs a maintainer support-boundary decision rather than cleanup closure.

## What This Changes

Investigate whether OpenAI GPT-4o agent sessions should continue tool use after plan-only text or be documented as outside strict-agentic support.

## Best Possible Solution

Make an explicit support-boundary decision: either extend focused GPT-4o plan-only recovery with tests and live proof, or document and surface that GPT-4o is selectable but not supported for strict multi-step agentic tool loops.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection shows current main and v2026.5.7 will not apply the plan-only retry guard to GPT-4o, and the reporter supplied production comparison evidence, but a live GPT-4o run on current main is still missing.

## Solution Assessment

Unclear. Extending strict-agentic coverage to GPT-4o is plausible but changes the support policy for older OpenAI models; documenting GPT-4o as outside the strict agentic path may be the safer maintainable outcome if maintainers do not want that behavior contract.
