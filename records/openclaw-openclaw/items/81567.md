## What This Changes

The issue reports that `openai/gpt-4o` agent sessions can terminate after a single planning-text response instead of continuing through the tool-use loop.

## Best Possible Solution

Make the GPT-4o policy explicit: either add tested strict-agentic support for GPT-4o in the execution-contract and incomplete-turn paths with docs, or mark GPT-4o as unsupported for multi-step agentic workflows in user-facing model guidance.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 explicitly keep `gpt-4o` on the default execution contract, matching the reported plan-only terminal path, but I did not run a live GPT-4o replay.

## Solution Assessment

Unclear as an implementation choice. The report gives two plausible resolutions, but maintainers need to decide whether GPT-4o should receive strict-agentic enforcement or be documented as unsuitable for multi-step agentic work.
