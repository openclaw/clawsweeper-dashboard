## Summary

Keep open. Current main and the latest release still keep `openai/gpt-4o` outside the strict-agentic GPT-5 execution contract, and current terminal handling does not recover visible planning prose with a normal stop reason.

## What This Changes

The issue requests investigation or documentation for GPT-4o agent sessions that exit after one planning-style text response instead of continuing the tool-use loop.

## Best Possible Solution

Maintainers should decide the GPT-4o agent support contract, then either extend strict-agentic or equivalent incomplete-turn enforcement with focused tests and docs, or clearly mark GPT-4o as not recommended for reliable multi-step agent workflows.

## Reproduction Assessment

Yes, source-reproducible at medium confidence: current main keeps `gpt-4o` outside strict-agentic activation, and visible `stop` text is not treated as incomplete. I did not run a live GPT-4o/OpenRouter replay in this read-only review.
