## Summary

Keep open: current main still makes GPT-4o a selectable OpenAI model, but the plan-only retry and strict-agentic protections are scoped to GPT-5-family OpenAI runs plus separate Gemini eligibility, so the reported GPT-4o behavior needs a maintainer support-boundary decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to investigate and handle GPT-4o sessions that stop after one plan-like text response instead of continuing the agent tool loop.

## Best Possible Solution

Decide the GPT-4o support boundary, then either add focused GPT-4o incomplete-turn coverage with live proof or document GPT-5.5/Codex as the supported OpenAI agent path and warn that GPT-4o can plan without acting.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection shows a GPT-4o plan-only `stop` turn would not receive the current strict planning retry guard, but the exact model behavior still needs a live current-main run or transcript.

## Solution Assessment

Unclear. A targeted GPT-4o allowlist or tool-use steer may solve the reported path, but current docs and tests intentionally scope the stronger contract to GPT-5-family OpenAI runs, so maintainers should decide support policy first.
