## Summary

Keep open. Current main and v2026.6.10 still keep GPT-4o outside the strict-agentic OpenAI execution contract, while the issue has concrete user reports that GPT-4o can terminate multi-step sessions with planning text and no tools.

## What This Changes

Investigate and resolve GPT-4o agent sessions ending after one text response instead of continuing the tool-use loop.

## Best Possible Solution

Choose one explicit GPT-4o policy path: either support GPT-4o with focused strict-agentic recovery, docs, tests, and live file-read/write proof, or mark it unsupported for multi-step agent workflows in docs and model selection.

## Reproduction Assessment

Partly yes: source inspection shows GPT-4o remains on the default contract and visible stop text can be accepted as terminal output. The exact live GPT-4o/OpenRouter plan-text exit still needs a fresh current-main CLI repro.

## Solution Assessment

Unclear as an implementation path: enabling GPT-4o strict-agentic recovery is plausible, but maintainers first need to decide whether GPT-4o is supported for multi-step agent workflows or should be documented as unsuitable.
