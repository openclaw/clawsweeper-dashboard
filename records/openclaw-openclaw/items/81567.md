## Summary

Keep open. Current main and the latest release still route `openai/gpt-4o` to the default execution contract, so plan-only text responses can terminate as normal stop responses instead of getting strict-agentic retry/blocked handling; the remaining decision is whether GPT-4o should gain that contract or be clearly marked unsuitable for multi-step agent workflows.

## What This Changes

Request to prevent GPT-4o agent sessions from ending after a single plan-only text response, or document that GPT-4o is not supported for multi-step agentic workflows.

## Best Possible Solution

Maintainers should choose and encode the GPT-4o contract: either add tested strict-agentic/plan-only recovery coverage for GPT-4o or clearly mark it as not supported for multi-step agent workflows in docs and model selection.

## Reproduction Assessment

Yes by source inspection, though not by a fresh live replay here: GPT-4o resolves to the default execution contract, planning-only retry is gated away, and a text-only `finish_reason: stop` remains a normal terminal response.

## Solution Assessment

Unclear until maintainers choose the GPT-4o product contract; extending strict-agentic to GPT-4o is plausible, but a docs/model-picker unsupported warning may be the better boundary if GPT-4o is intentionally not an agent model.
