## Summary

Keep open. Current main and v2026.6.10 still keep `openai/gpt-4o` outside the strict-agentic OpenAI execution contract, while the issue has concrete user evidence that GPT-4o can end multi-step agent sessions with visible plan text and no tools.

## What This Changes

The issue asks OpenClaw to decide how GPT-4o agent sessions should continue multi-step tool-use work instead of ending after one planning-text response.

## Best Possible Solution

Choose one explicit GPT-4o policy path: either support GPT-4o with focused strict-agentic recovery plus docs and live file-read/write proof, or mark it unsupported for multi-step agent workflows in docs/model selection.

## Reproduction Assessment

Partly yes: source inspection shows GPT-4o remains on the default contract and visible stop text can be accepted as terminal output. The exact live GPT-4o/OpenRouter plan-text exit still needs a fresh current-main CLI repro.

## Solution Assessment

Unclear as an implementation path: enabling GPT-4o strict-agentic recovery is plausible, but maintainers first need to decide whether GPT-4o is supported for multi-step agent workflows or should be documented as unsuitable.
