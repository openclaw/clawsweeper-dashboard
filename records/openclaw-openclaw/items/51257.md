## Summary

Keep open. The requested status-label feature is still missing on current main, and this PR remains a plausible implementation candidate, but the patch has blocking correctness/security issues and lacks required real behavior proof.

## What This Changes

The PR threads session `label` and `displayName` through status summary types and renders those friendly names in CLI and TUI status output.

## Best Possible Solution

Land a corrected implementation that carries existing label metadata through the active status contracts, renders sanitized friendly text, and keeps a resolvable key or label visible.

## Reproduction Assessment

Yes. Source inspection on current main shows status summaries omit `label`/`displayName`, CLI/TUI status render raw keys, and cron/session metadata can already carry friendly names.

## Solution Assessment

No. The direction is right, but this patch needs to update the active CLI row builder, preserve resolvable selectors, sanitize terminal-rendered names, and add real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move friendly names into the active row builder:** `src/commands/status.command.ts:610`
  - body: Current main renders CLI status through `buildStatusCommandReportData()` and `buildStatusSessionsRows()`, which still uses `sess.key`. Updating the old inline map in `status.command.ts` will not change CLI output after rebase, so put the friendly-name logic in the shared row builder instead.
  - confidence: 0.9
- **[P2] Keep the Key column resolvable:** `src/commands/status.command.ts:610`
  - body: `sessions.resolve` accepts key, sessionId, or label, not displayName. Showing `displayName` in the Key column can leave users with a visible value they cannot pass back to session commands, so prefer label/key there or show displayName separately.
  - confidence: 0.91
- **[P2] Sanitize friendly session names before rendering:** `src/tui/tui-status-summary.ts:79`
  - body: Labels and display names can be user- or channel-controlled and may contain newlines or ANSI/control sequences. Raw interpolation into CLI/TUI status can inject rows or terminal controls, so sanitize to single-line terminal text before rendering.
  - confidence: 0.93

## Security Review

  - confidence: 0.92
