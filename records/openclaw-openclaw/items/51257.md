## Summary

Keep open. Current main still has the underlying status display gap, and this PR is the linked implementation candidate, but it is not correct as-is because it edits an obsolete CLI rendering path, can show an unresolvable displayName in the Key column, lacks terminal-safe rendering for user/channel-controlled names, and still has no real behavior proof.

## What This Changes

The PR adds optional label/displayName fields to status summary and TUI types, populates them from session entries, and renders displayName then label then key in CLI and TUI status output.

## Best Possible Solution

Land a corrected implementation that carries existing label metadata through the active status contracts, renders sanitized friendly text, keeps a resolvable key or label visible, and intentionally handles any statusline/session-picker scope before closing the linked issue.

## Reproduction Assessment

Yes. Source inspection on current main shows stored session label/displayName metadata exists, but the status payload and active CLI/TUI status renderers still omit or ignore it and display raw keys.

## Solution Assessment

No. The direction is useful, but the patch needs to update the active shared row builder, preserve resolvable selectors, sanitize terminal-rendered names, rebase, and add real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move friendly names into the active row builder:** `src/commands/status.command.ts:610`
  - body: Current main renders CLI status through buildStatusCommandReportData() and buildStatusSessionsRows(), where Key still comes from sess.key. Updating the old inline map in status.command.ts will not affect CLI output after rebase, so put the friendly-name logic in the shared row builder.
  - confidence: 0.9
- **[P2] Keep the Key column resolvable:** `src/commands/status.command.ts:610`
  - body: sessions.resolve accepts key, sessionId, or label, not displayName. Showing displayName in the Key column can leave users with a visible value they cannot pass back to session commands, so prefer a resolvable label/key there or display displayName separately.
  - confidence: 0.91
- **[P2] Sanitize friendly session names before rendering:** `src/tui/tui-status-summary.ts:79`
  - body: Labels and display names can contain newlines, ANSI escapes, or other control characters from label patches or channel metadata. Raw interpolation into CLI/TUI status can inject rows or terminal controls, so sanitize to single-line terminal text before rendering.
  - confidence: 0.93
