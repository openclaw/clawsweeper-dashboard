## Best Possible Solution

Land a rebased implementation that threads friendly metadata through the active status contracts, sanitizes display text, preserves a visible resolvable key or label, and leaves statusline/session-picker scope tracked until separately covered.

Yes, source-reproducible. Current main stores label/displayName on session entries but status summary and active CLI/TUI renderers omit or ignore those fields; I did not run a live OpenClaw status scenario in this read-only review.

## Solution Assessment

No. The direction is right, but this branch targets a stale CLI path, hides a resolvable selector behind displayName, omits sanitization, and lacks after-fix real behavior proof.
