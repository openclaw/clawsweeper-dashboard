## What This Changes

The PR threads session label/displayName fields into status summary types and renders them before raw session keys in CLI and TUI status output.

## Best Possible Solution

Land a rebased implementation that carries friendly session metadata through the active status contracts, renders sanitized display text, and preserves a resolvable key or label for follow-up commands.

## Reproduction Assessment

Yes, from source inspection: current main stores label/displayName metadata on sessions but status summaries and active CLI/TUI renderers still omit it and display raw keys. I did not run a live OpenClaw status scenario in this read-only review.

## Solution Assessment

No. The direction is useful, but this branch targets a stale CLI path, uses displayName where a resolvable selector is expected, omits terminal sanitization, and lacks after-fix behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move friendly names into the active row builder:** `src/commands/status.command.ts:607-610`
  - body: Current main renders CLI status through buildStatusCommandReportData() and buildStatusSessionsRows(), where Key still comes from sess.key. Updating the old inline map in status.command.ts will not affect CLI output after rebase, so put the friendly-name logic in the shared row builder.
  - confidence: 0.9
- **[P2] Keep the Key column resolvable:** `src/commands/status.command.ts:607-610`
  - body: sessions.resolve accepts key, sessionId, or label, not displayName. Showing displayName in the Key column can leave users with a visible value they cannot pass back to session commands, so prefer a resolvable label/key there or display displayName separately.
  - confidence: 0.91
- **[P2] Sanitize friendly session names before rendering:** `src/tui/tui-status-summary.ts:79`
  - body: Labels and display names can contain newlines, ANSI escapes, or other control characters from labels or channel metadata. Raw interpolation into CLI/TUI status can inject rows or terminal controls, so sanitize to single-line terminal text before rendering.
  - confidence: 0.93
