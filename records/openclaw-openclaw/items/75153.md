## Summary

Keep open. Current main still lacks standalone channel lifecycle CLI recovery commands, and open PR #75560 explicitly closes this issue but has not merged yet.

## What This Changes

Expose non-destructive `openclaw channels start`, `stop`, and `restart` commands for linked channel accounts so operators can recover wedged channels without QR re-pairing or container restart.

## Best Possible Solution

Review and land #75560 or an equivalent focused implementation that adds admin-scoped channel lifecycle CLI/Gateway coverage while leaving automatic retry policy for a follow-up.

## Reproduction Assessment

## Solution Assessment
