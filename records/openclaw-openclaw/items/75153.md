## Summary

Keep open. Current main still lacks standalone `openclaw channels start`, `stop`, and `restart` commands, and open PR #75560 explicitly closes this issue but has not merged, so the issue should stay open until that PR or an equivalent implementation lands.

## What This Changes

Expose non-destructive channel start/stop/restart CLI commands so linked but stopped channel accounts can recover without QR re-pairing or container restart.

## Best Possible Solution

Review and land #75560 or an equivalent focused implementation that adds admin-scoped lifecycle CLI coverage while keeping automatic retry/backoff policy as a separate follow-up.
