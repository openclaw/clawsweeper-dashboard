## Summary

Keep open. Current main still lacks standalone channel lifecycle CLI recovery commands, and open PR #75560 explicitly closes this issue but has not merged.

## What This Changes

Expose `openclaw channels start|stop|restart --channel <name> [--account <id>]` so operators can recover a stopped linked channel without QR re-pairing or container restart.

## Best Possible Solution

Review and land #75560 or an equivalent focused implementation that adds admin-scoped CLI/Gateway lifecycle controls with docs and tests, leaving automatic retry/backoff policy to a follow-up.

## Reproduction Assessment

Yes for the actionable gap: current main lacks standalone `openclaw channels start`, `stop`, and `restart` commands in source and docs. The transient WhatsApp network failure itself was not live-reproduced.

## Solution Assessment

Yes. An additive operator CLI over existing lifecycle primitives is the narrowest maintainable fix, while automatic recovery policy should stay separate.
