## Summary

Keep open. Current main still lacks the standalone channel lifecycle CLI commands requested here, and open PR #75560 explicitly closes this issue but has not merged.

## What This Changes

Expose `openclaw channels start`, `stop`, and `restart` for existing linked channel accounts so operators can recover a stopped runtime without QR re-pairing or restarting the container.

## Best Possible Solution

Review and land #75560 or an equivalent focused implementation that adds admin-scoped lifecycle CLI/protocol coverage while leaving automatic retry/backoff policy to a separate follow-up.

## Reproduction Assessment

Yes for the actionable CLI gap: current main source and docs lack standalone `openclaw channels start`, `stop`, and `restart` commands. The transient WhatsApp DNS failure itself was not live-reproduced.

## Solution Assessment

Yes. An additive operator lifecycle surface over existing Gateway start/stop primitives is the narrowest maintainable direction; automatic health-monitor recovery policy should remain separate.
