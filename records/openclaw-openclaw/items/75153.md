## Summary

Keep open. Current main still lacks standalone channel lifecycle CLI recovery commands, and https://github.com/openclaw/openclaw/pull/75560 explicitly closes this issue but remains open and unmerged.

## What This Changes

Request to add non-destructive `openclaw channels start`, `stop`, and `restart` commands for recovering linked channel accounts without QR re-pairing or a full gateway restart.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75560 or an equivalent focused additive lifecycle-control implementation, preserving non-destructive auth behavior and leaving automatic retry/backoff policy to a separate decision.

## Reproduction Assessment

## Solution Assessment

Yes. Additive operator lifecycle commands backed by admin-scoped Gateway lifecycle methods are the narrow maintainable direction; automatic network-recovery/backoff behavior should remain separate.
