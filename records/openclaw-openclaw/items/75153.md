## Summary

Keep open. Current main still lacks standalone `openclaw channels start`, `stop`, and `restart` recovery commands, and open PR #75560 explicitly closes this issue but has not merged.

## What This Changes

The issue requests standalone CLI lifecycle commands for starting, stopping, and restarting an existing linked channel account without QR re-pairing or container restart.

## Best Possible Solution

Review and land #75560 or an equivalent focused implementation that adds admin-scoped channel lifecycle CLI/protocol coverage while leaving automatic retry/backoff policy to a separate issue.

## Reproduction Assessment

Yes for the actionable CLI gap: current main source and docs lack standalone `openclaw channels start`, `stop`, and `restart` commands. The transient WhatsApp transport failure itself was not live-reproduced.

## Solution Assessment

Yes. An additive operator lifecycle surface over existing gateway start/stop primitives is the narrowest maintainable direction; automatic health-monitor recovery policy should be handled separately.
