## What This Changes

Request to add operator-facing channel lifecycle CLI commands so a linked but wedged channel can be restarted without QR re-pairing or full gateway restart.

## Best Possible Solution

Add narrow `openclaw channels start|stop|restart` commands over the existing Gateway lifecycle RPCs, preserving auth/session state and leaving automatic health-monitor policy changes separate.

## Reproduction Assessment

Yes for the missing command surface: current main and v2026.6.6 expose `channels.start`/`channels.stop` RPCs but no standalone `openclaw channels start|stop|restart` CLI commands. The original live WhatsApp DNS-blip wedge was not reproduced here.

## Solution Assessment

Yes. A focused CLI wrapper around existing lifecycle RPCs is the narrow maintainable solution; `restart` can be stop-then-start client-side unless maintainers explicitly want a new server-side RPC.
