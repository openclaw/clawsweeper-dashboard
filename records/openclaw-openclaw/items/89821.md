- **[P1] Add the shortcut legend state to AppViewState:** `ui/src/ui/app-render.ts:2235-2237`
  - body: The new `renderShortcutLegend` call reads and writes `state.shortcutLegendOpen`, but `AppViewState` still has no such field. The merge ref fails `pnpm tsgo:prod` and `pnpm check:test-types` with TS2339 here, so the UI cannot typecheck until this state is added to the view-state contract.
  - confidence: 0.98
- **[P2] Avoid casting across the private keydown handler:** `ui/src/ui/app.shortcut-legend.test.ts:7-8`
  - body: This test intersects `OpenClawApp` with a public `globalKeydownHandler`, but that class member is private, so TypeScript reduces the intersection to `never` and the test fails to typecheck. Drive the document keydown listener through the element lifecycle or use a narrow public/test seam instead.
  - confidence: 0.97
- **[P3] Use braced guards in the new legend view:** `ui/src/ui/views/shortcut-legend.ts:86`
  - body: The repo lint rule requires braces; this new one-line guard, and the same `if (!props.open) return nothing` guard below, make `pnpm lint` fail. Add braces so the new view passes lint.
  - confidence: 0.93
