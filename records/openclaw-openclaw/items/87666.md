## Summary

Keep open: current main and v2026.6.9 still create Codex native subagent mirror rows as `silent` and `not_applicable`, so the task rows can be listed but cannot auto-deliver start/progress updates to the operator channel. The source-level issue is clear, but the fix needs maintainer policy for inheriting parent visibility without surfacing intentionally quiet cron/system activity.

## What This Changes

Requested change is to make Codex native subagent mirror tasks operator-visible instead of creating them with silent/not_applicable delivery state.

## Best Possible Solution

Add parent-context-aware Codex native subagent mirror visibility: use `state_changes`/`pending` only for human-facing parent flows that are not explicitly silent, while preserving no-destination and intentionally quiet flows.

## Reproduction Assessment

Yes for source inspection: current main still creates both Codex native subagent mirror row types as `silent`/`not_applicable`, and task state-change delivery requires `state_changes`/`pending`. I did not run a live channel reproduction in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the visibility policy. The unconditional flip is risky; parent-context-aware inheritance from the owning human-facing task/session is the safer maintainable direction.
