## Summary

Keep open: current main and the latest release still create Codex native subagent mirror tasks as `silent` and `not_applicable`, so the operator-visibility gap remains. The issue is source-reproducible, but the safe fix needs maintainer/product direction because an unconditional visibility flip could expose Codex child-task internals to cron, system, or other non-human destinations.

## What This Changes

The issue asks for Codex native subagent mirror tasks to surface lifecycle/progress updates to the operator’s delivery channel instead of remaining silent and not applicable.

## Best Possible Solution

Add parent-context-aware visibility for Codex native subagent mirror start/progress updates, preserving silent behavior for no-destination or explicitly silent parents while leaving completion delivery on the existing monitor path.

## Reproduction Assessment

Yes, at source level: current `main` still creates Codex native subagent mirror tasks as `silent`/`not_applicable`, while task delivery only emits state changes for `state_changes`/`pending` tasks. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the visibility policy. The parent-context-gated direction is safer than the unconditional flip because it can preserve silent behavior for cron, system, and no-destination parents.
