## Summary

Keep open: current main and the latest release still create Codex native subagent mirror task rows as `silent` and `not_applicable`, so start/progress delivery remains blocked. Completion delivery is partially handled on a separate monitor path, but the reported operator activity visibility gap still needs a maintainer policy decision for human-facing versus cron/system/no-destination parents.

## What This Changes

The issue requests visible delivery-channel updates for Codex native subagent mirror tasks instead of leaving native multi-agent activity silent in the background.

## Best Possible Solution

Add parent-context-aware visibility for Codex native subagent mirror start/progress updates, using visible `state_changes`/`pending` only for human-facing parents that are not explicitly silent while preserving no-destination and intentionally silent flows.

## Reproduction Assessment

Yes. Current main still creates Codex native subagent mirror rows as `silent`/`not_applicable`, and task delivery only emits state changes for `state_changes`/`pending` tasks; I did not run a live channel reproduction in this read-only pass.

## Solution Assessment

No for the unconditional-flip proposal as the final fix. A parent-context-aware policy is safer because it can surface human-facing child activity without changing cron, system, or no-destination behavior by default.
