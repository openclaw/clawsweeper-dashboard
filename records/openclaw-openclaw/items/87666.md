- `impact:message-loss`: The reported problem is suppressed delivery-channel status messages for Codex native subagent activity.

## Summary

Keep open: current main and the latest release still create Codex native subagent mirror tasks with `notifyPolicy: "silent"` and `deliveryStatus: "not_applicable"`, so the operator-visibility gap remains. The safe fix needs maintainer judgment because an unconditional visibility flip could route Codex child-task internals to cron/system or other non-human destinations.

## What This Changes

Make Codex native subagent mirror task start, progress, and terminal activity visible to operator delivery channels instead of creating silent/not-applicable mirror tasks.

## Best Possible Solution

Add parent-context-aware visibility for Codex native subagent mirror start/progress updates, preserving silent behavior for no-destination or explicitly silent parents while keeping completion delivery on the existing monitor path.

## Reproduction Assessment

Yes at source level: current main still creates the relevant Codex mirror tasks as silent/not_applicable, while task delivery only emits state changes for state_changes/pending tasks. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the visibility policy: the parent-context-gated direction is safer than the unconditional flip because it can preserve silent behavior for cron/system or no-destination parents.
