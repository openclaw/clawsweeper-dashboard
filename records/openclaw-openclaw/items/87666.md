## Summary

Keep open: current main and the latest release still create Codex native subagent mirror task rows as `silent` and `not_applicable`, so start/progress updates cannot auto-deliver to an operator channel. The remaining fix is real but needs maintainer policy for inheriting parent visibility without exposing cron/system/no-destination activity unexpectedly.

## What This Changes

The issue asks OpenClaw to surface Codex native subagent mirror task activity to operator delivery channels instead of creating silent, non-deliverable mirror task rows.

## Best Possible Solution

Add parent-context-aware Codex native subagent mirror visibility: use visible `state_changes`/`pending` only for human-facing parents that are not explicitly silent, while preserving no-destination and intentionally silent flows.

## Reproduction Assessment

Yes for source inspection: current main still creates the relevant mirror rows as `silent`/`not_applicable`, and task state-change delivery requires `state_changes`/`pending`. I did not run a live channel reproduction in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the visibility policy. The unconditional flip is risky; a parent-context-aware policy is the safer maintainable direction.
