## Summary

Keep open: current `main` and the latest release still create Codex native subagent mirror tasks as silent/not-applicable, so the reported operator-visibility gap is not implemented on main. The issue is valid from source inspection, but the safest fix needs maintainer judgment because changing default child-task visibility can affect channel volume and cron/system-origin runs.

## What This Changes

The issue requests making Codex native subagent mirror tasks visible to operators instead of creating them as silent/not-applicable task runs.

## Best Possible Solution

Add a parent-context-aware visibility path for Codex native subagent mirror start/progress updates, preserving silent behavior for system/no-destination or explicitly silent parents and keeping completion delivery on the existing monitor path.

## Reproduction Assessment

Yes at source level: the two Codex mirror task creation paths still set silent/not_applicable, and the task policy only delivers state changes for state_changes/pending tasks. I did not run a live channel reproduction in this read-only pass.

## Solution Assessment

Unclear: the unconditional flip proposed in the issue is probably too broad because subagent terminal delivery is already handled separately and cron/system destinations may receive unexpected updates. A parent-context-gated visibility fix is the safer direction.
