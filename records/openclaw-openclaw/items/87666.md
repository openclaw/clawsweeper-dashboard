## Summary

Keep open: current main and latest release still create Codex native subagent mirror tasks as silent/not_applicable, so the reported live operator-visibility gap remains. Newer completion delivery covers a separate terminal wakeup path, not the start/progress task-state messages requested here.

## What This Changes

The issue asks OpenClaw to make Codex native subagent mirror tasks visible to operators instead of creating them with silent notification and not-applicable delivery policy.

## Best Possible Solution

Carry parent delivery context into the Codex native subagent monitor/mirror path and make mirror task visibility parent-aware: human-facing sessions get pending/state_changes, while no-destination or explicitly silent parents stay quiet.

## Reproduction Assessment

Yes, at source level: current main still creates both reported Codex native subagent mirror rows as silent/not_applicable, and core delivery policy will not auto-deliver their state changes. I did not run a live channel scenario in this read-only review.

## Solution Assessment

No for the unconditional flip as proposed; parent-context-aware delivery is the safer shape because completion delivery is already handled separately and system/cron visibility side effects need explicit handling.
