## Summary

Keep open for normal PR review: the patch is a plausible narrow replacement for the older conflicting attempt, but this external PR still needs real background exec or cron behavior proof before merge because it changes session-event routing across core wake paths.

## What This Changes

The PR adds a shared event-session routing helper and threads it through exec, cron, heartbeat, gateway node-event, ACP parent-stream, and CLI watchdog paths so eligible single-owner `session.dmScope=main` direct DM events route to the agent main session.

## Best Possible Solution

Land the shared routing helper after redacted end-to-end proof shows single-owner dmScope=main events enqueue and wake agent main while multi-owner, wildcard, global-scope, subagent, and explicit route-binding cases remain unchanged.

## Reproduction Assessment

Yes, source-reproducible on current main: exec completion and forced heartbeat paths preserve same-agent direct DM keys because only cron-run keys are remapped. The linked issue also includes production session-store evidence for the same orphan-session behavior.

## Solution Assessment

Yes, likely: centralizing the policy and applying it to both enqueue and wake targets is the maintainable shape. The remaining gap is end-to-end proof that the real exec or cron session-store path follows the helper behavior without collapsing protected cases.
