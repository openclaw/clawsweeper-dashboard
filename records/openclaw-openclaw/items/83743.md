## Summary

Keep open: the shared enqueue/wake routing shape is useful and the PR supplies real exec proof, but the new owner check misses documented/channel-normalized allowFrom forms, so part of the dmScope=main bug would remain after merge.

## What This Changes

The PR adds a shared event-session-routing helper and wires exec, cron, node-event, heartbeat, ACP parent-stream, and CLI watchdog event paths through it for single-owner dmScope=main direct sessions.

## Best Possible Solution

Keep the shared helper direction, but make the single-owner policy compare against the same normalized owner identity used by channel DM authorization or carry that normalized owner into EventSessionRoutingPolicy, then cover prefixed allowFrom cases before merge.

## Reproduction Assessment

Yes, at source level. Current main preserves same-agent direct event keys through enqueue and forced heartbeat resolution, and the PR’s new helper can be checked with a documented prefixed allowFrom value to see the remaining mismatch.

## Solution Assessment

No, not yet. The shared helper is the right maintainable shape, but the owner comparison must match documented/channel allowlist normalization before this fully solves the issue.
