Full review comments:

- **[P1] Use a portable temp directory for Ubuntu pwsh:** `.github/scripts/check-windows-node-version.ps1:125`
  - body: The new weekly workflow runs this script on `ubuntu-latest`, but `Get-Sha256ManifestContent` builds its temp path from `$env:TEMP`. In this Linux pwsh environment, `TEMP` is empty while `RUNNER_TEMP` is set, so `Join-Path $env:TEMP ...` errors before the release manifest can be downloaded. Use `RUNNER_TEMP`, `TMPDIR`, or `[IO.Path]::GetTempPath()` so the scheduled drift check can actually run.
  - confidence: 0.93
- **[P2] Remove the unused issue-write token permission:** `.github/workflows/check-windows-node-weekly.yml:10`
  - body: This scheduled workflow only checks out code and reads release metadata through `gh`, but it grants `issues: write` to the workflow token. That broadens a scheduled main-branch token without a consumer in this diff; keep the workflow at `contents: read` unless it actually creates or updates issues.
  - confidence: 0.9
