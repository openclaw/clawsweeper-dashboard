## Summary

This PR is not merge-ready: its documented installation cannot reliably load the required package, and it introduces an archived native dependency into a credential-reading boundary even though OpenClaw’s existing exec-provider seam already supports an external self-contained resolver.

## What This Changes

The PR adds a keytar-based cross-platform OS keychain resolver for SecretRef exec requests and documents its installation, configuration, and companion management CLI.

## Best Possible Solution

Keep the generic exec-provider contract, and integrate OS keychains only through a maintained, self-contained, version-pinned resolver package with an explicit OS/Node support lifecycle and redacted clean-install Gateway proof.

## Reproduction Assessment

Yes for the installation defect: the script-local `createRequire` has no local dependency, while Node’s documented lookup does not normally search npm’s separate global package directory.

## Solution Assessment

No. The existing exec-provider seam already supports an external resolver, while this core-script approach splits the executable from its dependency and adopts an archived native credential backend.
