## Summary

Keep open for maintainer review: the PR is a focused bug fix with no blocking code findings, and current main/latest release still have the old single-registry global runner path. The change is still compatibility- and security-boundary-sensitive because it redefines which live or pinned plugin registries can dispatch hooks after scoped activations.

## What This Changes

The PR changes the plugin global hook runner to dispatch from a live composed registry view across active and pinned registries, with regression tests for local extension tool-call hooks after scoped activations.

## Best Possible Solution

Land this live-registry composition after maintainer review accepts the retired-registry behavior and confirms the residual non-pinned embedded-flow follow-up remains tracked separately.

## Reproduction Assessment

Yes, at source level with high confidence: current main preserves a gateway runner across a later default activation while tool-call dispatch asks only that runner for hooks. The PR body also includes real Testbox gateway proof, but this read-only review did not rerun it.

## Solution Assessment

Yes, this is the best immediate fix shape: it repairs the shared global hook-runner lifecycle boundary instead of adding activation, timeout, or permission configuration around one symptom. A deeper request-scoped registry handle can remain a follow-up for the residual unpinned embedded-flow case.
