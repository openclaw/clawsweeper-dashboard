## Summary

Keep open: current main still has the source-level fallback bug and the runtime fix direction is credible, but the branch is not merge-ready because its regression tests are based on the old auth-store harness and the PR still lacks after-fix real behavior proof.

## What This Changes

This PR excludes primary candidates from transient cooldown probe provider-slot bookkeeping and adds regression tests for same-provider fallback chains.

## Best Possible Solution

Keep the narrow runtime guard, port the added regression tests to the current mocked auth runtime helpers, and require redacted runtime fallback-chain proof before merge.

## Reproduction Assessment

Yes, at source level: current main records transient cooldown probe providers without checking `isPrimary`, then adds that provider after a failed probe, so a later same-provider fallback can hit the one-provider-per-run skip. I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes: excluding only primary candidates from transient provider-slot bookkeeping is the narrow maintainable fix because it preserves non-primary same-provider deduplication. The submitted branch still needs test-harness repair and real behavior proof before it is the best merge path.
