## Summary

Keep this PR open. Current main still has the reported primary-probe slot behavior and the production change is narrowly aimed, but the added tests are not mergeable as written because they call undefined/unwired auth-store helpers and the user-facing fix also needs a changelog entry.

## What This Changes

The PR changes model fallback cooldown-probe bookkeeping so primary probes do not consume transient same-provider fallback slots, and adds regression tests for same-provider probing and non-primary deduplication.

## Best Possible Solution

Land the narrow `!isPrimary` provider-slot bookkeeping change with tests that use the existing model-fallback auth-runtime mock helpers and a required changelog entry for the user-facing failover fix.

## Reproduction Assessment

Yes. A source-level reproduction is clear: make the primary provider transiently cooldowned near probe expiry, have that primary probe fail, then include a later same-provider fallback; current main can add the primary provider to `cooldownProbeUsedProviders` and skip the later fallback.

## Solution Assessment

No, not as written. The production guard is the right shape, but the PR should seed the mocked auth runtime with `makeAuthTempDir()` and `setAuthRuntimeStore()` instead of undefined/unwired filesystem store calls, then add the required changelog entry.
