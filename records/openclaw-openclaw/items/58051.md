## Summary

Keep open: current main still has the source-level fallback probe bug and the runtime guard in this PR is the right narrow direction, but the branch is not merge-ready because its regression tests target an older auth-store harness and no after-fix real behavior proof is present. ## What This Changes

The PR changes model fallback cooldown probe bookkeeping so primary probes do not consume the per-provider transient probe slot and adds regression coverage for same-provider fallback chains.

## Best Possible Solution

Keep the narrow runtime guard, port the regression tests to current auth runtime helpers, verify targeted model-fallback coverage, and require redacted real fallback-chain proof before merge.

## Reproduction Assessment

Yes, at source level: current main can set the transient cooldown provider slot from a primary probe failure and then skip a later same-provider fallback. I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes: excluding only primary candidates from transient provider-slot bookkeeping is the narrow maintainable fix because it preserves non-primary same-provider deduplication. The submitted branch still needs current-main test-harness repair and real behavior proof before merge.
