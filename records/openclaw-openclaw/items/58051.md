## Best Possible Solution

Land the focused provider-slot guard only after current-main validation and real behavior proof show primary probe failure, same-provider fallback attempt, fallback-to-fallback dedupe, and cross-provider reachability.

Yes, this is the best fix shape: the per-run provider-slot check is the owner of fallback-to-fallback dedupe, while the inner auth controller only knows one attempted model. The missing piece is real behavior proof and current-main validation, not a different code boundary.
