## What This Changes

The PR changes `runWithModelFallback` so primary transient cooldown probes do not consume per-run same-provider probe slots, and adds model fallback regression tests.

## Best Possible Solution

Land the narrow `!isPrimary` provider-slot bookkeeping fix after the contributor adds real behavior proof and maintainers rerun targeted current-main validation.

## Reproduction Assessment

Yes. A high-confidence source reproduction is clear: put the primary provider in near-expiry transient cooldown, make that primary probe fail, then include a later fallback for the same provider; current main can record the provider from the primary probe and skip the later fallback.

## Solution Assessment

Yes. The proposed `!isPrimary` guard is the narrowest maintainable code change because it only changes transient per-run provider-slot bookkeeping while preserving non-primary same-provider deduplication.
