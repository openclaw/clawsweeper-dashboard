## Summary

Keep this PR open. Current `main` still has the unguarded transient cooldown probe-slot logic that the PR targets, and the branch is a narrow, relevant fix with regression coverage; the remaining action is maintainer refresh and validation before any merge decision.

## What This Changes

The PR changes `runWithModelFallback` so primary cooldown probes do not check or consume same-provider transient probe slots, and adds regression tests for same-provider fallback probing plus non-primary deduplication.

## Best Possible Solution

Land the narrow behavior change after refresh: primary cooldown probes should not consume `cooldownProbeUsedProviders`, while non-primary same-provider fallbacks continue to deduplicate within a run.

## Reproduction Assessment

Yes. A source-level reproduction is clear: put the primary provider in transient cooldown near probe expiry, make that primary probe fail, then include a later same-provider fallback; current `main` can record the provider from the primary probe and skip the later fallback.

## Solution Assessment

Yes. The proposed `!isPrimary` guard is the narrowest maintainable fix because it changes only per-run transient provider-slot bookkeeping and preserves fallback-to-fallback deduplication.
