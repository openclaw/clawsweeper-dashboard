## Summary

Keep open: current main still has the source-level primary-probe slot bug, but this stale PR branch is not merge-ready because its added tests are on the old auth-store harness and no real after-fix behavior proof is present.

## What This Changes

The PR changes model fallback control flow so primary cooldown probes do not consume the per-provider transient probe slot, and adds regression tests for same-provider fallback chains.

## Best Possible Solution

Keep the narrow primary-probe exclusion in the runtime path, port the regression tests to current main’s auth-runtime helpers, and require redacted real fallback-chain proof before merge.

## Reproduction Assessment

Yes, at source level: current main records transient cooldown probe providers without checking `isPrimary`, then adds that provider after a failed probe, so a later same-provider fallback can hit the one-provider-per-run skip. I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes: excluding only primary candidates from transient provider-slot bookkeeping is the narrow maintainable fix because it preserves non-primary same-provider deduplication. The submitted branch still needs test-harness repair and real behavior proof before it is the best merge path.
