## What This Changes

The PR changes model fallback cooldown-probe bookkeeping so primary probes do not consume same-provider transient fallback probe slots, and adds regression tests.

## Best Possible Solution

Keep the narrow primary-probe exclusion in the runtime path, rework the regression tests onto the current mocked auth runtime helpers, and add redacted runtime proof of the fallback chain before merge.

## Reproduction Assessment

Yes, at source level: current main marks transient cooldown probe providers without checking isPrimary, then records that provider after a failed probe, so a later same-provider fallback can hit the one-provider-per-run skip. I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes: excluding only primary candidates from transient provider-slot bookkeeping is the narrow maintainable fix because it preserves non-primary same-provider deduplication. The submitted branch still needs test-harness repair and real behavior proof before merge.
