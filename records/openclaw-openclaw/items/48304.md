## Summary

Keep open. Current main still lacks built-in action-aware, per-sender authorization for message sends; existing controls are whole-tool or group-scoped, and the related open trackers cover only slices of this broader owner/action policy request.

## What This Changes

Request gateway-level enforcement that blocks or confirms protected tool actions, especially `message(action="send")`, based on trusted sender or owner identity.

## Best Possible Solution

Define a shared, typed action-aware and sender-aware tool authorization contract that runs before multiplexed tool actions execute, then cover message send, DM sender policy, and owner confirmation semantics with docs and regression tests.

## Reproduction Assessment

Yes for the missing enforcement path from source inspection: current main filters tools by name before action dispatch and has no built-in per-sender `message(send)` gate. I did not run a live Slack repro in this read-only review.

## Solution Assessment

Unclear until maintainer design is done. The requested direction is sound, but the maintainable fix should be a shared policy contract rather than one-off message-tool fields.
