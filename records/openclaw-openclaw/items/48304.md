## Summary

Keep open. Current main still lacks a built-in per-action, per-sender authorization contract for `message` sends; existing group `toolsBySender` and plugin `before_tool_call` hooks are adjacent, but they do not settle the requested owner-only/confirm-owner policy for personal-assistant DMs. The open action-level policy tracker at https://github.com/openclaw/openclaw/issues/13948 is related, but this issue adds distinct sender/owner authorization semantics.

## What This Changes

Request to add gateway-enforced per-action, per-sender authorization and optional owner confirmation for `message` and other protected tools.

## Best Possible Solution

Define a maintainer-approved sender-aware, action-aware authorization contract, or explicitly extend the plugin hook/API surface so a ClawHub policy plugin can enforce owner-only and confirm-owner behavior without relying on prompt instructions.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/security design request for a new policy contract. Source inspection gives high confidence current main lacks hard per-action, per-sender `message` authorization, and the reporter provided concrete Slack examples for the motivating behavior.

## Solution Assessment

Unclear until maintainer product/security design is done. The narrowest maintainable path is a shared action/sender policy seam or plugin API extension, not an ad hoc Slack or `message`-tool-only special case.
