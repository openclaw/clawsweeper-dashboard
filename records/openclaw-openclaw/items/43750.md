## Summary

Keep open: current main still has no native `/new` or `/reset` agent-target resolver; extra text is treated as generic command input/reset tail while session selection remains the already routed or bound session, and the later Telegram production comment confirms the impact is still real.

## What This Changes

Request native `/new` and `/reset` commands to support explicit `@agent` targeting in multi-agent channels and define safer no-target/channel-gating behavior.

## Best Possible Solution

Add an authorization-aware native-command target resolver for session start/reset commands that resolves exactly one allowed agent/session or follows a documented maintainer-chosen no-target policy, then update Discord, Telegram, docs, and regression coverage together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `/new @forge` or `/reset @anvil` remains generic command input or reset tail, while `CommandTargetSessionKey` is still the already routed or bound session.

## Solution Assessment

Unclear. Explicit target parsing with fail-closed authorization looks like the maintainable direction, but maintainers need to choose the no-target policy and how it interacts with bindings and channel gating.
