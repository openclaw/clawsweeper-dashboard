## Summary

Keep open: current main and latest stable v2026.6.10 still route `/new` and `/reset` through route-derived session targets, with no explicit user-selected agent target. The remaining work needs maintainer product/security direction for target syntax, no-target behavior, and cross-agent authorization.

## What This Changes

Support explicit agent targeting for `/new` and `/reset` in multi-agent channels, with safer defaults when no target is specified.

## Best Possible Solution

Define one shared destructive-command targeting contract, then implement explicit target selection, ambiguous-target handling, authorization checks, and cross-channel regression coverage together.

## Reproduction Assessment

Yes, at source level: command args are treated as reset tail and native command targets are route-derived; I did not run a live multi-agent Discord or Telegram replay.

## Solution Assessment

Unclear until maintainers choose the contract; explicit targeting is the right direction, but a one-channel patch would be too narrow without shared target and authorization semantics.
