## Summary

Keep open: current main and latest release still lack explicit agent targeting for `/new` and `/reset`; the remaining work is real but needs maintainer product/security direction before a safe fix PR.

## What This Changes

The issue requests targeted `/new` and `/reset` behavior for multi-agent channels, including explicit `@agent` selection and safer no-target/default handling.

## Best Possible Solution

Define a shared text/native command-targeting contract for destructive session commands, fail closed or prompt when the target is ambiguous, and add channel-specific authorization plus regression coverage.

## Reproduction Assessment

Yes, source-level: current main routes `/new` and `/reset` to the already resolved session target and has no `@agent` or selected-agent input in the shared native resolver. I did not run a live multi-agent Discord or Telegram reproduction.

## Solution Assessment

Unclear: explicit agent targeting is likely the right direction, but maintainers still need to choose the permanent syntax, ambiguous no-target behavior, and cross-agent authorization rule.
