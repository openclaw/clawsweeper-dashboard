## Summary

Keep open: current main still lacks a native `/new` or `/reset` agent-target resolver, and the later production Telegram comment confirms the shared-channel impact is still relevant. The open Discord mention-gating issue overlaps attention routing but does not replace this command-targeting work.

## What This Changes

Request `/new` and `/reset` to support explicit agent targeting and safer default behavior in multi-agent shared channels.

## Best Possible Solution

Add authorization-aware agent target resolution for `/new` and `/reset`, document fail-closed ambiguity behavior, and cover Discord, Telegram, shared session reset, and target-helper tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows `/new @forge` and `/reset @anvil` remain generic command input or reset tail while `CommandTargetSessionKey` stays the already routed or bound session.

## Solution Assessment

Unclear. Explicit target parsing with fail-closed authorization is the likely maintainable direction, but maintainers still need to decide target syntax, no-target UX, and cross-agent reset authorization.
