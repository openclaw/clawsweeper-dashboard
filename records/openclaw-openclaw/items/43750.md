## Summary

Keep open: current main still lacks native `/new` or `/reset` agent-target resolution, and the post-review production Telegram workaround comment confirms the multi-agent impact remains real.

## What This Changes

/new and /reset should support explicit agent targeting in shared multi-agent chat channels and avoid ambiguous resets.

## Best Possible Solution

Add an authorization-aware target resolver for native session commands, document fail-closed behavior for ambiguous shared channels, and cover Discord, Telegram, reset hooks, and docs together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/new @forge` or `/reset @anvil` is still generic command input or reset tail while `CommandTargetSessionKey` remains the already routed or bound session.

## Solution Assessment

Unclear. Explicit target parsing with fail-closed authorization looks like the maintainable direction, but maintainers need to choose the no-target UX and cross-agent reset authorization rules before implementation.
