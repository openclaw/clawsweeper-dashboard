## Summary

Keep open: current main still has no command-target resolver for `/new @agent` or `/reset @agent`; native commands target the already-routed or bound session, and the later Telegram production workaround comment confirms the shared-channel impact. The open Discord mention-gating issue is related but does not cover explicit reset/new target selection.

## What This Changes

Request `/new` and `/reset` support explicit agent targeting and safer untargeted behavior in shared multi-agent channels.

## Best Possible Solution

Implement an authorization-aware target resolver for session commands, with fail-closed ambiguity handling and focused docs/tests for targeted and untargeted `/new` and `/reset` across native and text command paths.

## Reproduction Assessment

Yes, at source level. Current main shows `/new @forge` and `/reset @anvil` remain generic command input or reset tail while `CommandTargetSessionKey` stays the already-routed or bound session.

## Solution Assessment

Unclear. Explicit target parsing with fail-closed authorization looks like the maintainable direction, but maintainers still need to decide syntax, untargeted UX, and cross-agent reset authorization.
