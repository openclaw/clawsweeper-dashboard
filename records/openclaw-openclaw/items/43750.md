## Summary

Keep open: current main still handles `/new` and `/reset` against the already-routed or bound session only; the production Telegram workaround comment confirms the shared-channel impact, and the open Discord mention-gating issue does not cover command target selection.

## What This Changes

Request `/new` and `/reset` support explicit `@agent` targeting and safer defaults in shared multi-agent channels.

## Best Possible Solution

Implement an authorization-aware target resolver shared by Discord, Telegram, and text command paths, with fail-closed ambiguity handling and docs/tests for targeted and untargeted `/new` and `/reset`.

## Reproduction Assessment

Yes. Source inspection on current main shows `/new @forge` and `/reset @anvil` remain generic command input or reset tail while `CommandTargetSessionKey` stays the already routed or bound session.

## Solution Assessment

Unclear. Explicit target parsing with fail-closed authorization looks like the maintainable direction, but maintainers still need to decide syntax, untargeted UX, and cross-agent reset authorization.
