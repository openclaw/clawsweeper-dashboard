## Summary

Keep open: current main still lacks a native `/new` or `/reset` agent-target resolver; extra text is parsed as generic command input/reset tail while the reset target remains the already routed or bound session, and the Telegram production workaround comment confirms the multi-agent impact remains real.

## What This Changes

Requests `/new` and `/reset` in multi-agent channels to support explicit `@agent` targeting plus safer no-target selection and channel-gating behavior.

## Best Possible Solution

Add an authorization-aware target resolver for native `/new` and `/reset`, document a fail-closed no-target policy, and update Discord, Telegram, shared reset handling, and regression coverage together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/new @forge` or `/reset @anvil` remains generic command input or reset tail, while `CommandTargetSessionKey` is still the already routed or bound session.

## Solution Assessment

Unclear. Explicit target parsing with fail-closed authorization looks like the maintainable direction, but maintainers need to choose the no-target policy and cross-agent reset authorization rules before implementation.
