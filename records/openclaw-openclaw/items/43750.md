## Summary

Keep open: current main has native command target-session plumbing, but it does not implement user-facing `/new @agent` or `/reset @agent` selection, and the untargeted multi-agent behavior needs a maintainer product/security decision.

## What This Changes

Requests `/new` and `/reset` to support explicit agent targeting and avoid ambiguous multi-agent session resets in shared channels.

## Best Possible Solution

Define one channel-agnostic command-targeting contract for session-control commands, then map Discord, Telegram, and Slack native command surfaces onto it with explicit handling for ambiguous untargeted resets.

## Reproduction Assessment

Source-level yes for the missing explicit target path: current command definitions and native command routing do not parse an agent target before selecting the reset session. I did not run a live multi-agent Discord or Telegram repro, so the exact fan-out is not tool-reproduced.

## Solution Assessment

No, current main is not the complete solution: `CommandTargetSessionKey` plumbing is necessary, but it only carries an already resolved target. The best fix needs a maintainer-selected target contract and ambiguous-command policy before implementation.
