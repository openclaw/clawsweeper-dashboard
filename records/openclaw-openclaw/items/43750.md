## Summary

Keep open: current main still lacks a native `/new` or `/reset` agent-target resolver, and the follow-up production Telegram comment confirms the shared-channel impact remains real rather than stale.

## What This Changes

Request native `/new` and `/reset` support for targeting a specific agent in multi-agent channels, plus a clear no-target behavior and channel-gating parity.

## Best Possible Solution

Add an authorization-aware native target resolver for `/new` and `/reset`, document a fail-closed no-target policy, and update Discord, Telegram, shared reset handling, and regression coverage together.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `/new @forge` or `/reset @anvil` remains generic command input or reset tail while the target session stays the already routed or bound session.

## Solution Assessment

Unclear. Explicit target parsing with fail-closed authorization looks like the maintainable direction, but maintainers still need to decide the no-target UX and cross-agent reset authorization model.
