## Summary

Keep open: current `main` and `v2026.6.8` still lack an explicit agent-target selector for `/new` and `/reset`. The issue is valid, but the durable fix needs maintainer product and security direction for syntax, ambiguous no-target behavior, and cross-agent authorization.

## What This Changes

The issue requests targeted `/new` and `/reset` behavior for multi-agent channels so destructive session commands affect only the intended agent and follow channel gating consistently.

## Best Possible Solution

Define one shared text/native command-targeting contract for destructive session commands, then implement explicit target selection, ambiguous-target handling, authorization checks, and Discord/Telegram regression coverage together.

## Reproduction Assessment

Yes at source level: current native command paths carry only route-derived session targets, not a user-selected `@agent` or `agent:` target. I did not run a live Discord or Telegram multi-agent reproduction.

## Solution Assessment

Unclear: explicit targeting is likely the right direction, but maintainers still need to choose the permanent syntax, ambiguous no-target behavior, and cross-agent authorization rule.
