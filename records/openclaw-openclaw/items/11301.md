## Summary

Keep open. Current main and the latest release still do not provide a runtime-container `/workspace` alias; the remaining decision is whether OpenClaw should add that outer-container contract or document `/workspace` as sandbox-only.

## What This Changes

The issue requests a `/workspace` alias in container configuration so documented workspace paths work inside agent containers.

## Best Possible Solution

Choose one outer-container contract, then either add a deliberate `/workspace` alias or bind across Dockerfile, Compose, setup helpers, Podman, Quadlet, docs, and smoke coverage, or document that `/workspace` is sandbox-only.

## Reproduction Assessment

Yes. Source inspection shows Docker, Compose, Docker setup, Podman, and Quadlet paths all use `/home/node/.openclaw/workspace` for the outer runtime container and do not create or mount `/workspace`.

## Solution Assessment

Unclear. The requested single symlink is only a partial fix unless maintainers decide the runtime-container contract and cover Dockerfile, generated Compose, Podman/Quadlet, docs, and the existing sandbox `/workspace` meaning together.
