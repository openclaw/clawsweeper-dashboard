## Summary

Keep open: current main and the latest release still lack a runtime-container `/workspace` alias, while the linked implementation at https://github.com/openclaw/openclaw/pull/12764 was closed unmerged and remains useful prior art rather than a shipped fix.

## What This Changes

The issue requests a `/workspace` alias in OpenClaw container runtimes so documented agent workspace paths resolve without using `/home/node/.openclaw/workspace`.

## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then either implement it consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.19` shows Docker/Compose/setup/Podman runtime containers mount only `/home/node/.openclaw/workspace`, while `/workspace` remains the sandbox workdir; I did not start a live container.

## Solution Assessment

Unclear. The requested one-line Dockerfile symlink is not the full solution because Compose, setup generation, Podman/Quadlet, docs, sandbox semantics, and runtime smoke coverage need to be handled together.
