## Summary

Keep open: current main still lacks a runtime-container `/workspace` alias, the linked implementation PR was closed unmerged, and the change needs a Docker/Podman contract decision because `/workspace` already has sandbox-specific meaning.

## What This Changes

The request asks OpenClaw to expose `/workspace` as an alias for `/home/node/.openclaw/workspace` inside containerized agent environments.

## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then either implement it consistently across Dockerfile, Compose, setup helper, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows Docker/Podman runtime containers mount only `/home/node/.openclaw/workspace`, while `/workspace` remains reserved for sandbox workdirs; I did not run a live container.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink is not the full solution because Compose, setup generation, Podman/Quadlet, docs, sandbox semantics, and runtime smoke coverage need to be handled together.
