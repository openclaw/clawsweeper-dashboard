## Summary

Keep open: current main still has no Docker/Podman runtime `/workspace` alias; `/workspace` remains a sandbox-specific container path, and https://github.com/openclaw/openclaw/pull/12764 closed unmerged.

## What This Changes

Add a runtime-container `/workspace` symlink or equivalent mount so workspace paths resolve without using `/home/node/.openclaw/workspace`.

## Best Possible Solution

Make an explicit runtime-container contract decision, then either add a tested alias or second mount across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke coverage, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main shows Docker Compose, Docker setup, Podman launcher, and Quadlet all mount only `/home/node/.openclaw/workspace`, while sandbox code reserves `/workspace`.

## Solution Assessment

Unclear. The one-line symlink is useful prior art, but the maintainable fix needs a coordinated container-runtime decision so Docker, Podman, docs, and sandbox semantics stay aligned.
