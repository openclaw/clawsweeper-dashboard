## Summary

Keep open: current main still mounts the runtime container workspace only at `/home/node/.openclaw/workspace`; `/workspace` remains a sandbox-specific path, and the related implementation PR closed unmerged.

## What This Changes

The issue requests a runtime Docker/Podman container `/workspace` alias to `/home/node/.openclaw/workspace`.

## Best Possible Solution

Make an explicit runtime contract decision, then either add a tested Docker/Podman compatibility alias across image, Compose, setup helpers, Quadlet, docs, and smoke coverage, or document that `/workspace` is sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main shows Docker Compose, Docker setup, Podman launcher, and Quadlet all mount only `/home/node/.openclaw/workspace`, while `/workspace` is reserved for sandbox workdirs.

## Solution Assessment

Unclear. The requested one-line symlink is useful prior art, but the maintainable solution needs a coordinated container-runtime decision so it does not accidentally change sandbox semantics or rootless mount behavior.
