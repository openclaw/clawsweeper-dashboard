## Summary

Keep open. Current main still does not provide a runtime-container `/workspace` alias for Docker or Podman gateway/CLI containers, and the related implementation PR was closed unmerged, so the requested compatibility improvement remains valid for https://github.com/openclaw/openclaw/issues/11301.

## What This Changes

The issue requests adding a `/workspace` symlink or alias in container configuration so documented workspace paths work without using `/home/node/.openclaw/workspace`.

## Best Possible Solution

Implement a deliberate runtime compatibility path for `/workspace` across Docker Compose, the Docker setup helper, Podman launcher, Quadlet template, and relevant Docker smoke/test docs, or make an explicit maintainer decision that `/workspace` is sandbox-only and update the docs to remove the ambiguity.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main container configs mount the agent workspace only at `/home/node/.openclaw/workspace`, and targeted searches found no `/workspace` alias in the runtime container surfaces.

## Solution Assessment

No for the one-line Dockerfile-only symlink proposed in the original issue and closed PR; it is likely too narrow. The maintainable fix should cover every runtime container entry point, tests/static checks, and docs, or explicitly document that `/workspace` applies only inside agent sandboxes.
