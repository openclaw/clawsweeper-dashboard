## Summary

Keep open: current main and the latest release still expose the runtime workspace only at `/home/node/.openclaw/workspace`, while `/workspace` remains a sandbox-container path. The closed related PR was unmerged and only changed the Dockerfile, so a maintainer decision is still needed on whether runtime containers should gain a compatibility alias or the docs should make `/workspace` sandbox-only.

## What This Changes

The issue requests adding a `/workspace` alias in OpenClaw container configuration so documented workspace paths work inside agent/runtime containers.

## Best Possible Solution

Maintainers should choose one explicit contract, then either implement `/workspace` consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke coverage, or document that `/workspace` is sandbox-only and keep runtime containers on `/home/node/.openclaw/workspace`.

## Solution Assessment
