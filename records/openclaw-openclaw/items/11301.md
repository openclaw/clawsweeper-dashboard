## Summary

Keep open. Current main still does not provide a runtime-container `/workspace` alias for Docker, generated Compose, Podman, or Quadlet; the direct symlink PR was closed unmerged, and the open Docker/DooD sandbox issue is adjacent rather than a replacement for this runtime-container ergonomics request.

## What This Changes

The issue requests a container runtime `/workspace` symlink or equivalent alias to `/home/node/.openclaw/workspace`.

## Best Possible Solution

Maintainers should choose one runtime workspace contract, then either implement `/workspace` consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke coverage, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes, at source level. Current main shows Docker, generated Compose, Podman, and Quadlet runtime paths using `/home/node/.openclaw/workspace` without a `/workspace` alias; I did not run a live container in this read-only review.

## Solution Assessment

Unclear. The one-line symlink from the closed prior PR is plausible prior art, but the best fix needs a maintainer decision on image symlink versus parallel bind mount versus documenting `/workspace` as sandbox-only.
