## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then either implement it consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink is not the full solution because Compose, setup generation, Podman/Quadlet, docs, sandbox semantics, and runtime smoke coverage need to be handled together.
