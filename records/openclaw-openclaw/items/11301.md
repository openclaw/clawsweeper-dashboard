## Summary

Keep open: current main and the latest release still do not provide a runtime-container `/workspace` alias, and the linked implementation PR was closed unmerged rather than shipped.

## What This Changes

The issue requests adding `/workspace -> /home/node/.openclaw/workspace` to OpenClaw container configuration so agent-container docs and workflows can use the shorter path.

## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then either implement it consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.20` shows Docker, Compose, setup, Podman, and Quadlet runtime containers still target `/home/node/.openclaw/workspace`; I did not start a live container.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink is not enough on its own because the supported behavior needs to cover Compose, setup generation, Podman/Quadlet, docs, sandbox semantics, and smoke proof together.
