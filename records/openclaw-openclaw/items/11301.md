## Summary

Current main and v2026.6.5 still expose outer Docker and Podman workspaces only at `/home/node/.openclaw/workspace`, while `/workspace` remains the sandbox-container contract. The linked implementation was closed unmerged, and no canonical replacement exists, so this issue remains necessary pending a maintainer decision on the supported outer-container path.

## What This Changes

Add a `/workspace` alias for the OpenClaw workspace inside runtime Docker and Podman containers.

## Best Possible Solution

Choose and document one outer-container contract: either expose a tested `/workspace` alias consistently across Docker, Compose, setup helpers, Podman/Quadlet, and smoke coverage, or explicitly reserve `/workspace` for sandbox containers and remove ambiguous guidance elsewhere.

## Reproduction Assessment

Yes. Current source and published docs consistently place the outer runtime workspace at `/home/node/.openclaw/workspace`, while sandbox code assigns `/workspace`; the requested outer-container alias is absent.

## Solution Assessment

Unclear. A symlink is plausible, but the narrowest maintainable solution depends on whether maintainers want `/workspace` to become a supported outer-container contract across Docker and Podman rather than remain sandbox-specific.
