## Summary

Keep open: current main still mounts Docker and Podman runtime workspaces only at `/home/node/.openclaw/workspace`, while `/workspace` remains sandbox-specific, and the linked implementation PR closed unmerged.

## What This Changes

Request a `/workspace` runtime alias or symlink in containerized OpenClaw so workspace paths resolve without spelling `/home/node/.openclaw/workspace`.

## Best Possible Solution

Make an explicit maintainer choice: add a tested runtime compatibility alias across Dockerfile, Compose, setup helpers, Podman/Quadlet, docs, and smoke coverage, or clarify that `/workspace` is sandbox-only.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current Docker Compose, Docker setup, Podman launcher, and Quadlet configuration mount only `/home/node/.openclaw/workspace`. I did not live-run containers because this review is read-only.

## Solution Assessment

Unclear. The proposed one-line symlink is useful prior art, but the maintainable fix needs a coordinated runtime-container decision so it does not accidentally change sandbox semantics or rootless mount behavior.
