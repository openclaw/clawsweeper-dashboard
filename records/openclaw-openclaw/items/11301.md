## Summary

Current main still distinguishes runtime containers, which mount the workspace at `/home/node/.openclaw/workspace`, from agent sandboxes, which reserve and mount `/workspace`; the focused prior PR closed unmerged, so this remains a valid low-priority product and compatibility decision.

## What This Changes

Requests a `/workspace` alias in runtime containers that points to `/home/node/.openclaw/workspace`.

## Best Possible Solution

Define `/workspace` as either a supported runtime-container alias across Docker, Compose/setup, Podman/Quadlet, docs, and smoke tests, or explicitly document it as sandbox-only and remove runtime-documentation ambiguity.

## Reproduction Assessment

Yes. Current main consistently mounts runtime workspaces at `/home/node/.openclaw/workspace`, contains no `/workspace` alias, and separately reserves `/workspace` for sandbox containers.

## Solution Assessment

No. The proposed Dockerfile-only symlink is too narrow to establish a reliable product contract; the best fix must either cover every first-party runtime launcher with smoke and documentation proof or formally keep `/workspace` sandbox-only.
