## Summary

Current main and v2026.6.5 still expose `/workspace` only inside agent sandboxes, while supported Docker and Podman runtime containers use `/home/node/.openclaw/workspace`. The request remains useful, but making the alias a supported runtime-container contract needs maintainer judgment; the related pull request was closed unmerged and no viable replacement currently owns the work.

## What This Changes

Add a `/workspace` alias in OpenClaw runtime containers that resolves to `/home/node/.openclaw/workspace`.

## Best Possible Solution

Define one deliberate container workspace contract: either implement and smoke-test `/workspace` consistently across the Docker image, Compose/setup generation, and Podman/Quadlet paths, or explicitly document `/workspace` as sandbox-only and remove runtime-facing ambiguity.

## Reproduction Assessment

Yes, at source level. Every inspected runtime-container path and its documentation use `/home/node/.openclaw/workspace`, while `/workspace` is defined only for agent sandboxes.

## Solution Assessment

Unclear. A symlink is plausible, but maintainers must first decide whether runtime containers should adopt the sandbox path name and then implement that contract consistently across all supported container launch paths.
