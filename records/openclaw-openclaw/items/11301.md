## Summary

Keep open: current main still does not provide a Docker/Podman runtime `/workspace` alias, the directly related fix attempt closed unmerged, and the remaining choice is a container workspace contract decision rather than stale cleanup.

## What This Changes

The issue asks for container environments to expose `/workspace` as an alias for `/home/node/.openclaw/workspace`.

## Best Possible Solution

Make an explicit runtime-container contract decision, then either add a tested alias or second mount across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke coverage, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection against current main shows the Dockerfile, Compose file, Docker setup helper, Podman launcher, and Quadlet template do not create or bind `/workspace`, while sandbox code reserves `/workspace` for sandbox workdirs.

## Solution Assessment

Unclear. The one-line Dockerfile symlink from https://github.com/openclaw/openclaw/pull/12764 is useful prior art, but the maintainable solution needs a coordinated Docker, Podman, docs, and sandbox-semantics decision.
