## Summary

Keep open: current main still lacks a runtime-container `/workspace` alias across Docker Compose, Docker setup, Podman launcher, and Quadlet, and the related implementation PR was closed unmerged. Because `/workspace` is already an established sandbox workdir, this needs maintainer direction before changing runtime mounts or docs.

## What This Changes

Request adding a `/workspace` alias in Docker/Podman runtime containers so documented agent workspace paths resolve without using `/home/node/.openclaw/workspace`.

## Best Possible Solution

Maintainers should choose the runtime-container workspace contract, then either add tested Docker/Podman `/workspace` alias coverage plus docs or explicitly document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes, from source inspection rather than a live container run: current main still mounts only `/home/node/.openclaw/workspace` in Docker/Podman runtime paths and has no `/workspace` alias. Sandbox code separately reserves `/workspace`, which makes the contract question concrete.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink addresses the papercut in one image path, but the maintainable fix needs coordinated Docker, Podman, docs, and sandbox-contract coverage after maintainer direction.
