## Summary

Keep open. Current main and the latest release still mount runtime Docker/Podman workspaces at `/home/node/.openclaw/workspace` without a `/workspace` alias, and the direct symlink PR was closed unmerged; the remaining work is a deliberate container-runtime contract decision rather than stale cleanup.

## What This Changes

The issue requests adding a `/workspace` alias in OpenClaw container configuration so documented workspace paths work inside agent/runtime containers.

## Best Possible Solution

Maintainers should either implement a tested `/workspace` runtime-container alias across Dockerfile, Compose, Docker setup, Podman launcher, Quadlet, docs, and smoke coverage, or explicitly document that `/workspace` is sandbox-only.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.10` show Docker, generated Compose, Podman, and Quadlet workspaces mounted at `/home/node/.openclaw/workspace` without a `/workspace` alias; I did not run a live container in this read-only review.

## Solution Assessment

Unclear as currently requested. A one-line Dockerfile symlink is too narrow because the supported shape likely needs a deliberate alias or bind-mount contract across Docker, Compose, setup helpers, Podman, docs, and sandbox semantics.
