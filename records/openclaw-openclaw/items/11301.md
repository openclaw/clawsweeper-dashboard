## Summary

Keep open: current main and the latest release still expose the container runtime workspace at `/home/node/.openclaw/workspace`, while `/workspace` is reserved by sandbox behavior and the linked implementation at https://github.com/openclaw/openclaw/pull/12764 was closed unmerged. The request is source-reproducible, but changing it needs a maintainer decision on the Docker/Podman runtime contract.

## What This Changes

The issue requests adding a `/workspace` symlink or equivalent container configuration so agent containers can use the shorter documented workspace path instead of `/home/node/.openclaw/workspace`.

## Best Possible Solution

Maintainers should choose the runtime-container workspace contract, then either add tested Docker/Podman `/workspace` alias coverage plus docs or explicitly document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows Docker/Podman runtime containers mount only `/home/node/.openclaw/workspace`, while sandbox code separately reserves `/workspace`.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink addresses only one image path, while a maintainable solution needs coordinated Docker, Compose, setup-helper, Podman, Quadlet, docs, and sandbox-contract handling.
