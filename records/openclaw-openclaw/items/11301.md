## Summary

## What This Changes

The issue requests a `/workspace` alias in runtime containers for the existing `/home/node/.openclaw/workspace` directory.

## Best Possible Solution

Define `/workspace` as either a supported runtime-container alias across Docker, Compose/setup, Podman/Quadlet, docs, and smoke tests, or explicitly document it as sandbox-only and remove runtime-documentation ambiguity.

## Reproduction Assessment

Yes. Source inspection shows current main consistently mounts runtime workspaces at `/home/node/.openclaw/workspace`, contains no `/workspace` alias, and separately uses `/workspace` for sandbox containers.

## Solution Assessment

No. The issue body's Dockerfile-only symlink is too narrow; the maintainable solution must cover all first-party runtime container launch paths or document `/workspace` as sandbox-only.
