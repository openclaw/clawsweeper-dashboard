## Summary

Keep open: current main still exposes `/workspace` only for agent sandbox workdirs, while Docker and Podman runtime containers continue to mount the persisted workspace at `/home/node/.openclaw/workspace`; the linked implementation PR was closed unmerged, and accepting the alias needs a maintainer contract decision across Docker, Podman, docs, and smoke coverage.

## What This Changes

The issue requests adding a `/workspace` alias in container configuration so documented agent workspace paths resolve to `/home/node/.openclaw/workspace` inside runtime containers.

## Best Possible Solution

Maintainers should choose the runtime-container workspace contract, then either add tested Docker/Podman `/workspace` alias coverage plus docs or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows Docker/Podman runtime containers mount only `/home/node/.openclaw/workspace`, while sandbox code separately reserves `/workspace`; I did not run a live container.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink addresses only one image path, while a maintainable solution needs coordinated Docker, Compose, setup-helper, Podman, Quadlet, docs, and smoke coverage.
