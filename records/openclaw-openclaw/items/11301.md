## Summary

Keep open: current main still lacks a runtime-container `/workspace` alias, the direct implementation PR at https://github.com/openclaw/openclaw/pull/12764 closed unmerged, and the remaining choice overlaps the existing sandbox `/workspace` contract.

## What This Changes

The issue asks for a `/workspace` alias in OpenClaw runtime container configurations so documented workspace paths do not require `/home/node/.openclaw/workspace`.

## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then either implement it consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.20` shows Docker, Compose, Docker setup, Podman, and Quadlet runtime containers still target `/home/node/.openclaw/workspace`; I did not start a live container.

## Solution Assessment

Unclear. The suggested one-line Dockerfile symlink is not sufficient by itself because the supported behavior would need to cover Compose, setup generation, Podman/Quadlet, docs, sandbox semantics, and smoke proof together.
