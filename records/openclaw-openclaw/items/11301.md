## Summary

Keep open: current main and the latest release still mount Docker/Podman runtime workspaces at `/home/node/.openclaw/workspace`, while `/workspace` remains a sandbox-specific path; the only linked implementation was closed unmerged.

## What This Changes

The issue requests a runtime-container `/workspace` alias pointing at `/home/node/.openclaw/workspace`.

## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then implement it consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.22` shows Docker, Compose, Docker setup, Podman, and Quadlet still use `/home/node/.openclaw/workspace`; I did not start a live container.

## Solution Assessment

Unclear. A one-line Dockerfile symlink is not the best complete solution because the runtime container contract, sandbox `/workspace` semantics, Compose/setup generation, Podman/Quadlet, docs, and smoke proof need to agree.
