## Summary

Keep open: current main still does not provide a Docker/Podman runtime-container `/workspace` alias, the related implementation PR was closed unmerged, and the remaining fix needs a runtime-container versus sandbox workspace contract decision.

## What This Changes

The issue requests adding `/workspace` in OpenClaw container runtimes as an alias to `/home/node/.openclaw/workspace`.

## Best Possible Solution

Make an explicit runtime-container workspace contract decision, then either add tested Docker/Podman alias coverage plus docs or document `/workspace` as sandbox-only and remove the ambiguity.
