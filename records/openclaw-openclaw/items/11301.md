## Summary

Keep open: current main still lacks a runtime-container `/workspace` alias, latest release `v2026.5.20` also lacks it, and the related implementation PR at https://github.com/openclaw/openclaw/pull/12764 closed unmerged.

## What This Changes

Request a `/workspace` runtime-container alias or equivalent mount for the OpenClaw workspace path currently exposed as `/home/node/.openclaw/workspace`.

## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then either implement it consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.20` shows runtime Docker, Compose, Docker setup, Podman, and Quadlet paths still target `/home/node/.openclaw/workspace`; I did not start a live container.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink is not the best complete solution by itself because the supported behavior would need to cover Compose, setup generation, Podman/Quadlet, docs, sandbox semantics, and smoke proof together.
