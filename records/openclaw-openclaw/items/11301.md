## Summary

Keep open: current main and the latest release still expose the runtime container workspace only at `/home/node/.openclaw/workspace`; `/workspace` remains a sandbox path, so this needs an explicit maintainer setup/product decision rather than cleanup closure.

## What This Changes

The issue requests a `/workspace` alias inside OpenClaw container environments for the existing `/home/node/.openclaw/workspace` directory.

## Best Possible Solution

Maintainers should choose one outer-container contract: add a tested `/workspace` alias or second bind mount across Docker and Podman setup surfaces, or document that `/workspace` is sandbox-only and keep runtime containers on `/home/node/.openclaw/workspace`.

## Solution Assessment
