## Summary

Keep open: current main and the latest release still expose the runtime container workspace only at `/home/node/.openclaw/workspace`, while `/workspace` remains a sandbox-specific path and the only linked implementation PR was closed unmerged. This is a real source-level papercut, but changing it needs maintainer agreement on the Docker/Podman runtime contract.

## What This Changes

Request to add a runtime-container `/workspace` alias pointing at `/home/node/.openclaw/workspace` so container sessions can use the shorter documented-style path.

## Best Possible Solution

Maintainers should choose the runtime-container workspace contract, then either add tested Docker/Podman `/workspace` alias coverage plus docs or explicitly document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows Docker/Podman runtime containers mount only `/home/node/.openclaw/workspace` and do not create `/workspace`, while sandbox code separately reserves `/workspace`.

## Solution Assessment

Unclear. The proposed one-line Dockerfile symlink addresses one image path, but the maintainable solution needs coordinated Docker, Compose, setup-helper, Podman, Quadlet, docs, and sandbox-contract handling.
