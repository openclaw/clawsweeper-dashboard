## Summary

Keep open: current main still lacks a Docker/Podman runtime `/workspace` alias, and the prior implementation PR was closed unmerged.

## What This Changes

The issue requests adding a `/workspace` symlink or equivalent container runtime alias for `/home/node/.openclaw/workspace`.

## Best Possible Solution

Make an explicit maintainer choice: add a tested runtime compatibility alias across Dockerfile, Compose, setup helpers, Podman/Quadlet, docs, and smoke coverage, or document that `/workspace` is sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Docker Compose, Docker setup, Podman launcher, and Quadlet all mount only `/home/node/.openclaw/workspace`, while `/workspace` remains sandbox-specific; I did not live-run containers for this read-only review.

## Solution Assessment

Unclear. The requested one-line symlink is useful prior art, but the maintainable fix needs a coordinated runtime-container decision so it does not accidentally change sandbox semantics or rootless mount behavior.
