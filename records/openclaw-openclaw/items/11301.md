## Summary

Keep open: current main still has no Docker/Podman runtime `/workspace` alias, the only directly related fix PR closed unmerged, and existing source/docs distinguish runtime storage under `/home/node/.openclaw/workspace` from sandbox workdirs under `/workspace`.

## What This Changes

Request to expose the agent workspace at `/workspace` inside container runtime configurations instead of only `/home/node/.openclaw/workspace`.

## Best Possible Solution

Make an explicit runtime-container contract decision, then either add a tested alias or second mount across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke coverage, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection against current main shows Dockerfile, Compose, Docker setup, Podman launcher, and Quadlet do not create or bind `/workspace`, while sandbox code reserves `/workspace`; no live container was run in this read-only review.

## Solution Assessment

Unclear. The one-line symlink in https://github.com/openclaw/openclaw/pull/12764 is useful prior art, but the maintainable fix needs coordinated Docker, Podman, docs, and sandbox semantics rather than a Dockerfile-only change.
