## Summary

Keep open: current main and `v2026.5.22` still keep the Docker/Podman runtime workspace at `/home/node/.openclaw/workspace`, while `/workspace` remains a sandbox-specific path except for the separate EasyRunner deployment guide; the only linked implementation, https://github.com/openclaw/openclaw/pull/12764, closed unmerged.

## What This Changes

Request to add a runtime-container `/workspace` alias for the OpenClaw workspace path used by Docker and Podman containers.

## Best Possible Solution

Maintainers should decide whether `/workspace` is a supported runtime-container alias, then either implement it consistently across Dockerfile, Compose, Docker setup, Podman/Quadlet, docs, and smoke proof, or document `/workspace` as sandbox-only.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.22` shows Docker, Compose, Docker setup, Podman, and Quadlet still use `/home/node/.openclaw/workspace`; I did not start a live container.

## Solution Assessment

Unclear. A one-line Dockerfile symlink is not the best complete solution because supported behavior would need the runtime container contract, sandbox `/workspace` semantics, Compose/setup generation, Podman/Quadlet, docs, and smoke proof to agree.
