## Summary

Keep open: current main and the latest release still advertise local update paths without a container-aware policy, and the related open pull request is an unmerged candidate rather than implemented behavior.

## What This Changes

Request suppressing or replacing local update prompts and actions when OpenClaw detects Docker/Kubernetes-style container deployments.

## Best Possible Solution

Define one shared container-aware update-availability contract that keeps startup logs, CLI status, gateway snapshots/events, Control UI, and auto-update behavior consistent, then land or close the related candidate pull request based on maintainer policy and proof.

## Reproduction Assessment

Yes for source-level behavior, but not from a live Docker/Kubernetes run. Current main shows startup, status, gateway, and Control UI update paths advertising local update actions without consulting the existing container detector.

## Solution Assessment

Unclear until maintainers accept the product policy. The narrow technical direction should be a single producer-level update policy reused across CLI, gateway, and UI, rather than separate one-off display checks.
