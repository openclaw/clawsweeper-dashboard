## Summary

Keep open: current main still exposes local update hints/actions in container-relevant update paths, and the open linked PR is the active candidate fix rather than merged implementation.

## What This Changes

The issue requests container-aware update guidance that suppresses local npm/UI update suggestions in Docker, Kubernetes, and similar deployments.

## Best Possible Solution

Keep this issue as the canonical request and review, land, or reject https://github.com/openclaw/openclaw/pull/94310 as the candidate implementation, with one shared container-aware update policy across startup, status, gateway events, and Control UI.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced: current main formats update hints and UI actions from update availability without applying container state. I did not run a Docker/Kubernetes gateway session in this read-only review.

## Solution Assessment

Yes as a direction, but current main has not implemented it yet. The maintainable path is to reuse the existing container detection helper and centralize the policy at the update-availability producer so startup logs, CLI status, gateway events, and Control UI agree.
