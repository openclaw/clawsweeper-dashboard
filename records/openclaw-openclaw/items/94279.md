## Summary

Keep open: current main has reusable container detection, but the update checker, status hint, gateway update availability, and Control UI banner still expose local update paths without a container-aware policy. The related implementation PR is open and unmerged, so this issue should remain the canonical tracker until maintainers land or reject that fix path.

## What This Changes

The issue requests suppressing local update suggestions in Docker/Kubernetes-style deployments and showing redeploy-image guidance instead.

## Best Possible Solution

Keep this issue open as the canonical tracker until the linked implementation is reviewed and landed, or maintainers choose a different container update policy.
