## Summary

Keep open: current main has reusable container detection, but the update-check, status, and Control UI update surfaces still advertise local update paths without container-aware suppression or redeploy guidance.

## What This Changes

Request to detect Docker, Kubernetes, or other containerized OpenClaw runtimes and hide or replace local update suggestions with image redeploy guidance.

## Best Possible Solution

Define one container-aware update contract that reuses `src/infra/container-environment.ts`, suppresses local package-update actions in containerized package images, and shows redeploy-image guidance across startup logs, status, docs, and UI.

## Reproduction Assessment

We have a source-reproducible path, but not a live Docker reproduction: current main logs and renders update actions from update availability without checking container state.

## Solution Assessment

Unclear until maintainers choose the product contract; the likely best technical direction is to reuse the existing container-environment helper instead of adding a new env/config flag first.
