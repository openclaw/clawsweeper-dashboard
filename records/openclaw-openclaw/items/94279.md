## What This Changes

The issue requests detecting Docker/Kubernetes/containerized deployments and suppressing local update suggestions or actions that would not persist after image redeploy.

## Best Possible Solution

Keep this issue as the canonical request and resolve it by landing, narrowing, or explicitly rejecting https://github.com/openclaw/openclaw/pull/94310 after maintainer review of the container update policy.

## Reproduction Assessment

## Solution Assessment

Unclear until the linked PR's policy questions are resolved. Gating the shared update producer is the right technical layer for snapshots and async events, but maintainers still need to decide how container detection should affect explicit auto-update settings.
