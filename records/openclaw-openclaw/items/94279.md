## Summary

Keep open: current main already has reusable container detection, but the status, startup, gateway snapshot, and Control UI update surfaces still advertise local update paths without container-aware suppression or redeploy guidance. The related open PR is a candidate fix, so this issue should stay open as the canonical thread until that PR is reviewed, landed, or replaced.

## What This Changes

Request container-aware update guidance that suppresses or replaces local npm/UI update suggestions in Docker and Kubernetes deployments.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/94310 as the current candidate and define one container-aware update contract that reuses `isContainerEnvironment()` across status, startup, gateway snapshots, Control UI, and docs.

## Reproduction Assessment

Yes, from source inspection rather than a live Docker run: current main formats CLI status and startup update hints as local `openclaw update` actions and passes `updateAvailable` through to the Control UI without checking container state.

## Solution Assessment

Unclear until maintainers choose the product contract. The likely best technical direction is to reuse the existing container-environment helper and align all update surfaces, instead of adding a new config flag as the first solution.
