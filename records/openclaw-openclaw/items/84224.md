Full review comments:

- **[P2] Make the exec fixture portable on Linux:** `src/flows/doctor-core-checks.test.ts:386`
  - body: The explicit `--allow-exec` test uses `/bin/sh`, but the exec resolver rejects symlinked commands unless `allowSymlinkCommand` is set before it honors `allowInsecurePath`. On Linux systems where `/bin/sh` is a symlink, this resolves to a gateway-auth warning instead of `[]` and the marker assertion will fail. Use a secure temp executable fixture like the secrets tests, or opt into `allowSymlinkCommand` deliberately.
  - confidence: 0.94

  - confidence: 0.86
