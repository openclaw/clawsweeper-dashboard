## What This Changes

The PR adds an early return in `ensureRuntimePluginsLoaded` when any active plugin registry exists and updates its unit test to expect no resolver call in that case.

## Best Possible Solution

Keep the current main implementation and close this PR rather than merging the broad active-registry guard.

## Reproduction Assessment

Yes. The original regression is covered by the startup-scoped reply runtime and loader-compatibility unit paths on main; the PR’s unsafe behavior is also reproducible by setting an active registry for one workspace/runtime scope and requesting another.

## Solution Assessment

No for this PR; yes for current main. The maintainable fix is to delegate reuse to loader compatibility so config, workspace, plugin scope, and runtime-mode changes still fall through when needed.
