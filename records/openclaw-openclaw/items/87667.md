## What This Changes

## Best Possible Solution

Keep the default-false opt-in, but preserve existing Slack account `dm` override semantics by inheriting only `collapseAssistantThreads` unless a Slack/config owner explicitly accepts whole-`dm` deep-merge.

## Solution Assessment

No, not as-is. The default-false flag is a conservative mitigation, but broad `dm` deep-merge is not the narrowest compatibility-safe solution unless maintainers explicitly choose that upgrade behavior.
