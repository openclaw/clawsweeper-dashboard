## What This Changes

Adds default-false `channels.slack.dm.collapseAssistantThreads`, uses it to collapse Slack assistant DM thread routing into the base DM session, deep-merges Slack `dm` account config, and updates schema, generated metadata, docs, and routing tests.

## Best Possible Solution

Merge the default-false opt-in after maintainers accept the Slack `dm` account-config inheritance change; otherwise narrow the merge behavior so only `collapseAssistantThreads` inherits through account `dm` overrides.

## Reproduction Assessment

## Solution Assessment

Yes with a compatibility caveat: a default-false opt-in is the narrowest way to avoid changing existing documented routing by default. The only open solution question is whether the broad `dm` deep-merge should be accepted or narrowed to the new flag.
