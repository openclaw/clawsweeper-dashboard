## What This Changes

Fix memory-core dreaming so a successful narrative subagent's final text becomes readable before fallback diary content is written.

## Best Possible Solution

Make successful subagent completion guarantee readable final output, use a bounded memory-core wait only if that generic guarantee cannot be safely strengthened, and retain fallback solely for genuinely empty or failed runs.

Yes. The v2026.6.5 trace gives a timestamped reproduction where wait completion precedes model completion, and current main retains the exact single-read fallback path needed to reproduce the loss.

No, the issue body's envelope-only change is not the best fix. The newer evidence supports synchronizing successful completion with transcript readability and covering both delayed-persistence and cleanup races.
