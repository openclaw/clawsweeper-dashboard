## Summary

Keep open. Current main still lacks the requested logical-session lineage API and WebUI scope controls, and open PR #76116 is the active implementation candidate that explicitly links this issue for closure after merge.

## What This Changes

The issue requests durable transcript-backed usage history rollups across logical session rebinding/update/reset, with API and WebUI controls for current-instance versus historical-lineage views.

## Best Possible Solution

Review and land or close PR #76116; the end state should be an additive lineage-aware usage contract with transcript-backed aggregation and visible WebUI scope/range controls.

## Reproduction Assessment

Yes, at source level: current main can query a current key or discovered session row, but the protocol, handler, and UI do not expose a family/lineage query path. I did not run a live `openclaw update` cycle, so the update trigger is supported by the report and source inspection rather than live execution.

## Solution Assessment

Yes for direction, but not yet on current main. A first-class lineage API plus explicit WebUI scope controls is the maintainable path; the active implementation PR should be reviewed rather than closing this issue before merge.
