## Summary

Keep this PR open, but it still needs a narrow safety fix before merge: malformed `append` intent can fall through to the overwrite path and truncate the file this feature is meant to protect.

## What This Changes

The branch adds optional `append` handling for host/workspace and sandbox write tools, adds append/abort/workspace regression tests, updates the changelog, and refreshes the Swift gateway AgentParams model.

## Best Possible Solution

Land a maintainer-approved append-capable file primitive that preserves workspace and sandbox guards while rejecting malformed append intent before any overwrite fallback runs.

## Reproduction Assessment

Yes, by source inspection. Current main routes ordinary host and sandbox writes through overwrite-oriented `createWriteTool`, while the PR branch still allows a present non-boolean `append` value to reach that overwrite path.

## Solution Assessment

No, not as-is. The optional `append` flag is a plausible narrow implementation, but malformed present values must be rejected before delegating to the overwrite-capable base write tool.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject malformed append values before overwriting:** `src/agents/pi-tools.read.ts:882-884`
  - body: When a model sends `append` with a non-boolean value like `"true"` or `1`, `doAppend` is false and this delegates to the original overwrite implementation. That can truncate the shared log this feature is meant to protect, so reject any present non-boolean `append` value before falling back to overwrite mode.
  - confidence: 0.9
