## Summary

Keep this PR open. Current main still exposes ordinary host and sandbox `write` through overwrite-only upstream operations outside the memory-flush wrapper, while this active PR is the focused implementation candidate for the linked data-loss issue.

## What This Changes

This PR adds an optional boolean `append` parameter to the agent `write` tool, implements host/workspace/sandbox append operations, adds regression coverage, and updates the changelog.

## Best Possible Solution

Land a maintainer-approved append-capable file primitive that preserves workspace and sandbox guards, keeps overwrite as the default, rejects malformed append intent, and serializes append mutations.

## Reproduction Assessment

Yes, by source inspection. Current main routes ordinary host and sandbox writes through overwrite-oriented `createWriteTool`, while append-only behavior is scoped to memory-flush runs and the linked issue adds production cron evidence on that same path.

## Solution Assessment

Yes, subject to maintainer API approval. The PR is a narrow backward-compatible implementation that preserves default overwrite behavior, keeps path guards, and covers the earlier malformed-append, sandbox, queueing, and abort safety gaps.
