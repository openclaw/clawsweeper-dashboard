## Summary

Keep open: current main and v2026.6.6 still expose general `write` as overwrite-only, while this PR is the active append-mode fix path. I found no blocking correctness defect in the patch, but the public `append?: boolean` API and session-state behavior still need maintainer acceptance before merge.

## What This Changes

This PR adds an optional `append?: boolean` parameter to the agent write tool across shared session, host/workspace, and sandbox write paths, with regression and type-contract tests.

## Best Possible Solution

Land a maintainer-approved append primitive with overwrite remaining the default, workspace/sandbox guards preserved, same-file appends serialized, and type tests that fail if the public append contract regresses.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 accept only `path` and `content` for the general write tool and call `writeFile`, while the linked issue provides cron and single-agent truncation reports; I did not run a live Gateway repro in this read-only review.

## Solution Assessment

Mostly yes, pending maintainer API acceptance. A first-class append primitive is narrower and safer than prompt guards or read-concatenate-overwrite patterns, but the public API shape and regression-sensitive type tests should be settled before merge.
