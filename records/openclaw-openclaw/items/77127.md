Full review comments:

- **[P2] Recover append success after persisted backend errors:** `src/agents/sessions/tools/write.ts:417`
  - body: The new append branch calls `ops.appendFile`, but the catch path still uses overwrite recovery that only succeeds when the whole file equals the requested fragment.   - confidence: 0.88
- **[P2] Keep wrapper append recovery in sync:** `src/agents/agent-tools.read.ts:1172`
  - body: The OpenClaw host/sandbox wrapper intercepts `append: true` before the shared write tool, so fixing `createWriteTool` alone will not cover this runtime path. If the sandbox bridge writes and then throws, this wrapper still surfaces a failed append after mutation; reuse the same append recovery helper here or delegate append to the fixed shared tool.
  - confidence: 0.84
- **[P3] Make the append contract tests fail when append is removed:** `src/agents/sessions/tools/tool-contracts.test.ts:6-10`
  - body: This type assertion still checks whether `WriteToolInput` is assignable to a type where `append` is optional, so it would pass even if `append` were removed. Assert that a value containing `append: true` satisfies `WriteToolInput`, and mirror that direction for the write event input test.
