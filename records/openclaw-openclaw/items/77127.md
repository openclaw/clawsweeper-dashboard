Full review comments:

- **[P2] Recover append success after persisted backend errors:** `src/agents/sessions/tools/write.ts:412`
  - body: This append branch still falls back to `recoverSuccessfulWrite`, which only recognizes overwrite success when the whole file equals `content`. If `appendFile` appends and then throws an abort/timeout-like error, the file will contain the old content plus `content`, recovery fails, and a retry can duplicate memory/log entries.
  - confidence: 0.9
- **[P2] Keep wrapper append recovery in sync:** `src/agents/agent-tools.read.ts:1139`
  - body: The OpenClaw host/sandbox wrapper performs its own append path, but `appendFileWithAbort` reports backend errors without checking whether the append already persisted. A sandbox bridge write that lands and then throws can therefore be retried and append the same content again.
  - confidence: 0.86
- **[P3] Assert the append property directly in contract tests:** `src/agents/sessions/tools/tool-contracts.test.ts:6-10`
  - body: `toMatchTypeOf<{ append?: boolean }>()` still passes if `WriteToolInput` only has `path` and `content`, because the property is optional. Use a direct property assertion or indexed access in both contract tests so removing `append` fails CI.
  - confidence: 0.94
