## Summary

Keep open: the append direction is supported by current-main source and the related overwrite reports, but the current head still has one narrow cancellation bug in the new append wrapper.

## What This Changes

Adds an optional `append` boolean to the agent `write` tool for host/workspace/sandbox writes, with append regression tests and a changelog entry.

## Best Possible Solution

Land append-capable writes only after cancellation, path policy, parent-directory creation, serialization, and focused regression coverage match the existing write tool safety envelope.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main is overwrite-only for general `write`, and the PR-specific abort gap is visible where the append wrapper awaits `ops.appendFile` without observing later aborts.

## Solution Assessment

No. The optional append flag is a narrow maintainable direction, but this implementation should keep active-operation abort behavior consistent with the existing write tool before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Reject append calls aborted during the active write:** `src/agents/pi-tools.read.ts:882`
  - body: The append path only checks `signal` before calling `ops.appendFile`. If the run is cancelled after the queued callback starts but while the append read/mkdir/write is pending, this awaits completion and returns success; upstream `write` rejects when abort fires during the queued mutation. Add an abort listener or signal-aware append operation, plus a regression that aborts after `appendFile` starts.
  - confidence: 0.78
