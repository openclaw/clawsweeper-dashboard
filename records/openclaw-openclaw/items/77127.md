## Summary

Keep this PR open for maintainer review: the current head fixes the prior duplicate-append blocker and appears technically reviewable, but it still adds a model-facing write-tool API and carries broad unrelated drift that maintainers need to accept or narrow before merge.

## What This Changes

The PR adds optional `append?: boolean` support to shared and wrapped agent `write` tools, updates tool/event type contracts and prompt metadata, and adds append regression coverage.

## Best Possible Solution

Land a focused append-capable write-tool contract, or replace this PR with the maintainer-chosen append API, while preserving overwrite defaults and existing workspace/sandbox guards.

## Reproduction Assessment

Yes. Current `main` is source-reproducible: ordinary write accepts only `path` and `content` and calls `ops.writeFile`, while cron uses trigger `cron` and does not enter the memory-flush append wrapper.

## Solution Assessment

Yes as a direction, but not automatically as a product decision. A first-class append mode at the shared write-tool boundary is narrower than prompt-only guards or read-concatenate-overwrite workarounds, but maintainers still need to accept the exact API shape and broad branch scope.
