## Summary

Keep open. Current main still exposes ordinary host and sandbox `write` as overwrite-oriented, while this recent PR is a focused append-mode implementation with sufficient terminal proof and no blocking review findings; maintainers still need to approve the public tool API shape before merge.

## What This Changes

The PR adds an optional `append` boolean to the agent `write` tool, implements guarded host and sandbox append operations with queueing and abort handling, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a maintainer-approved append-capable file primitive that preserves workspace and sandbox guards, keeps overwrite as the default, rejects malformed append intent, and serializes same-file append mutations.

## Reproduction Assessment

Yes, by source inspection. Current main routes ordinary host and sandbox writes through overwrite-oriented `createWriteTool`, while the linked issue provides production cron evidence on that same path.

## Solution Assessment

Yes, subject to maintainer API approval. The PR is a narrow backward-compatible implementation that preserves the default overwrite behavior and addresses the prior sandbox, abort, queueing, and malformed-append safety gaps; a separate append tool remains the main product alternative.
