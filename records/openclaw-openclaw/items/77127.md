## Summary

Keep open: the PR addresses a real append-capability gap for agent writes, but PR head still has a correctness blocker because append calls can report success when cancellation arrives during the active file mutation.

## What This Changes

This PR adds an optional `append` boolean to the agent `write` tool across host/workspace and sandbox writes, adds focused append regression tests, and updates the changelog.

## Best Possible Solution

Land append-capable writes after the append path preserves the existing write tool safety envelope: validation, path guards, queue serialization, parent creation, and active cancellation behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main is overwrite-only for general writes, and PR head’s append wrapper lacks upstream-style active cancellation once `ops.appendFile` has started.

## Solution Assessment

No. The optional append flag is a plausible narrow direction, but this implementation should match upstream `write` cancellation semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reject appends aborted during active file mutation:** `src/agents/pi-tools.read.ts:882`
  - body: The append path checks `signal` before queue entry and again before `ops.appendFile`, but once append I/O begins a cancellation can still race to a successful result. Upstream `createWriteTool` keeps an abort listener active around the queued mutation, so append should also reject on active cancellation and get a regression that aborts after `appendFile` starts.
