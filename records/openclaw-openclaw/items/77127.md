Full review comments:

- **[P2] Return success once an append has already mutated the file:** `src/agents/agent-tools.read.ts:948-950`
  - body: If the signal aborts while the append is in flight, `appendFileWithAbort` waits for `append()` and then throws here when the abort flag is set, even though the file may already contain the new bytes. Base `write` recovers successful writes after abort; surfacing a failed append after mutation can lead to duplicate appended content on retry, so return success once append completes or add same-queue readback recovery before throwing.
  - confidence: 0.84
- **[P3] Make the contract tests fail when append is missing:** `src/agents/sessions/tools/tool-contracts.test.ts:6-10`
  - body: This `toMatchTypeOf` direction still passes if `WriteToolInput` only has `path` and `content`, because a type without `append` is assignable to one with an optional `append`. Use a `keyof` or property-type assertion that actually fails when the exported append field is removed; the sibling `WriteToolCallEvent` test has the same issue.
  - confidence: 0.78
