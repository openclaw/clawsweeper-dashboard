Full review comments:

- **[P2] Recover same-content appends after backend timeouts:** `src/agents/sessions/tools/write.ts:370`
  - body: When an append succeeds and then the backend throws a timeout-like error, recovery returns `null` if the precheck saw the file already equal to the appended text. In that case readback can still prove success as `beforeContent + content`, but this guard exits before `isRecoveredAppendContent` runs, so a retry can duplicate the appended text. Let append recovery use the append-specific proof instead of requiring `changed` first.
  - confidence: 0.88
