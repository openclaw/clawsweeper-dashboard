- **[P2] Recover successful appends after backend errors:** `src/agents/sessions/tools/write.ts:412`
  - body: The new append branch reuses `recoverSuccessfulWrite`, which only succeeds when the entire file equals `content`. If an append backend writes `old + content` and then throws an abort/timeout-like error, this reports failure even though bytes were appended; callers can retry and duplicate the entry. Add append-aware recovery, or avoid routing append errors through overwrite-only recovery semantics.
  - confidence: 0.86
- **[P2] Update the shared write prompt for append mode:** `src/agents/sessions/tools/write.ts:29-33`
  - body: This schema now advertises `append`, but the same ToolDefinition still contributes prompt text saying the tool is only for create/overwrite. AgentSession renders those guidelines into the model prompt, so shared-session models can be told the opposite of the new contract. Update the snippet/guideline with append-aware wording while keeping overwrite warnings clear.
  - confidence: 0.82
- **[P3] Strengthen the append type assertions:** `src/agents/sessions/tools/tool-contracts.test.ts:6-10`
  - body: These assertions would still pass if `append` disappeared, because a type without `append` is assignable to one where `append` is optional. Flip the assertion direction or assert that an object containing `append: true` satisfies `WriteToolInput` and `WriteToolCallEvent["input"]`.
