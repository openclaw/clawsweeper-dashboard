Full review comments:

- **[P1] Keep the public write-tool contract consistent:** `src/agents/sessions/tools/tool-contracts.ts:78-79`
  - body: Adding `append` to `WriteToolInput` makes it part of the SDK-visible tool contract, but the exported base `createWriteTool` still exposes only `path`/`content` and overwrites when called directly. That means direct session-tool or SDK consumers can now type a call with `append: true` and still truncate the file, so either make the base write tool honor append too or keep this field out of the shared contract until the API is accepted everywhere.
  - confidence: 0.88
- **[P2] Return success once append has already mutated:** `src/agents/agent-tools.read.ts:948-950`
  - body: If the signal aborts while `append()` is in flight, this post-append check throws `AbortError` even though the file may already contain the new bytes. Base `write` recovers successful writes after abort; reporting a failed append after mutation can cause the model or caller to retry and duplicate appended content.
  - confidence: 0.86
- **[P3] Make the contract tests fail when append is missing:** `src/agents/sessions/tools/tool-contracts.test.ts:6-10`
  - body: This `toMatchTypeOf` direction still passes if `WriteToolInput` only has `path` and `content`, because a type without an optional property is assignable to one with it. Use a `keyof` or property-type assertion that fails when `append` disappears; the sibling `WriteToolCallEvent` test has the same issue.
  - confidence: 0.8
