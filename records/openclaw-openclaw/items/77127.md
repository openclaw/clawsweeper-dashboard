## Summary

Keep open: this PR is still the active fix path for the linked write-tool data-loss issue, but it is not merge-ready because it exposes append in the public agent-sessions write type while the exported shared write factory/schema remains overwrite-only.

## What This Changes

Adds an optional `append` flag to OpenClaw agent write wrappers, public write input/event types, and host/sandbox regression tests.

## Best Possible Solution

Align one canonical append contract before merge: either implement append in the shared session write factory/schema/operations and tests, or keep `append` out of SDK-visible types until only the OpenClaw wrapper supports it.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 expose ordinary `write` as create/overwrite through `ops.writeFile`, while the linked issue includes cron and single-agent truncation evidence.

## Solution Assessment

No, not yet. The append wrapper is a plausible direction, but the merge shape is not best until the SDK-visible type, shared write schema, exported factory, runtime behavior, and maintainer-approved API shape agree.
