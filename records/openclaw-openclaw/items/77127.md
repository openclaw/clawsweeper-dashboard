## Summary

Keep this PR open, but it needs changes before merge: the append direction matches the canonical data-loss report, while the new sandbox append path does not yet preserve normal write create-parent behavior and cancellation semantics.

## What This Changes

The PR adds an optional `append` boolean to the agent `write` tool, routes host/workspace/sandbox append operations, adds focused append tests, and records the user-facing change in `CHANGELOG.md`.

## Best Possible Solution

Land an append-capable write surface only after it preserves normal write semantics across host, workspace-only, and sandbox paths, including parent-directory creation, path policy, same-file serialization, cancellation behavior, and focused regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible as overwrite-only outside the memory-flush wrapper, and the PR-specific sandbox append regressions are visible from the new code path without executing it.

## Solution Assessment

No. The optional append flag is a narrow maintainable direction, but this implementation should first make sandbox append match the existing write tool safety envelope and cancellation behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Create sandbox parent dirs before appending:** `src/agents/pi-tools.read.ts:747-752`
  - body: The append branch bypasses the upstream write tool's `ops.mkdir(dir)` step, and this sandbox `appendFile` calls `bridge.writeFile` without `mkdir: true` or a prior `mkdirp`. As a result, `write({ path: "new/dir/log.txt", content, append: true })` can fail in sandbox mode even though normal writes and the PR description say missing files are created with parent directories.
  - confidence: 0.9
- **[P2] Recheck aborts inside the mutation queue:** `src/agents/pi-tools.read.ts:865`
  - body: The append path checks `signal?.aborted` only before calling `withFileMutationQueue`. If another mutation keeps the queue busy and the signal aborts while this append is waiting, the queued callback still runs and mutates the file; upstream normal writes check the signal from inside the queued operation. Move the abort check into the queued callback, and add coverage for aborting while queued.
  - confidence: 0.82
