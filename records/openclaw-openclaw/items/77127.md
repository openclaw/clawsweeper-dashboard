## Summary

Keep open. Current main still exposes ordinary `write` as create-or-overwrite and only has append behavior for memory-flush writes; the PR head adds the requested append path and I found no blocking code-review finding, but this is a new public tool API that still needs maintainer approval and final CI gating.

## What This Changes

This PR adds an optional `append` boolean to the agent `write` tool for host and sandbox writes, with file-queue serialization, regression tests, and a changelog entry.

## Best Possible Solution

Approve the append-mode API shape or redirect it to a separate append tool, then merge once required checks are green.

## Reproduction Assessment

Yes, by source inspection. Current main routes ordinary host and sandbox writes through upstream overwrite-only `createWriteTool`, while append behavior is limited to the memory-flush wrapper.

## Solution Assessment

Yes for the code path, pending product approval. The optional flag is a narrow backwards-compatible change that preserves overwrite behavior, reuses existing root guards, and now keeps same-file serialization through active aborts.
