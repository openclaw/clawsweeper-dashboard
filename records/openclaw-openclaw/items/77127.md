## Summary

Keep open: this PR is the active append-mode fix for the linked data-loss workflow, and neither current main nor v2026.6.6 has the general append contract. The remaining blocker I found is that the model-facing prompt metadata still describes `write` as overwrite-only, so the new primitive is not fully aligned with the agent guidance that should make cron and ordinary agent turns use it.

## What This Changes

Adds optional `append?: boolean` support to agent write tools, host/sandbox and shared session write paths, exported write input/tool-call event types, and focused regression coverage.

## Best Possible Solution

Land a maintainer-approved append write contract with schema, runtime wrappers, SDK/event types, prompt/tool summaries, and regression tests aligned while preserving overwrite as the default and keeping workspace/sandbox guards intact.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 expose overwrite-only ordinary write semantics while the linked issue has concrete cron and single-agent truncation evidence; I did not run a mutating live Gateway repro in this read-only review.

## Solution Assessment

No, not quite as submitted. A first-class append primitive is the right direction, but the model prompt/tool metadata should be aligned with the new append mode before maintainers decide whether this boolean API is the permanent contract.
