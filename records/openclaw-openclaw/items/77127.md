## Summary

Keep open: current main still lacks a general append path for ordinary agent writes, and this active branch is the candidate implementation rather than obsolete cleanup. The remaining decision is whether maintainers want this model-facing API shape before normal merge validation.

## What This Changes

The PR adds an optional `append` boolean to the agent `write` tool for host/workspace and sandbox writes, plus focused regression tests and a changelog entry.

## Best Possible Solution

Land a maintainer-approved append-capable file primitive that preserves workspace and sandbox guards, keeps overwrite as the default, rejects malformed append intent, and serializes same-file append mutations.

## Reproduction Assessment

Yes. Source inspection shows current main routes ordinary host and sandbox writes through overwrite-oriented `createWriteTool` paths, while the linked issue provides production cron evidence on that same surface.

## Solution Assessment

Yes, subject to maintainer API approval. The optional append flag is a narrow backwards-compatible implementation direction, and the current branch addresses the earlier safety, validation, sandbox, serialization, and abort concerns with focused coverage.
