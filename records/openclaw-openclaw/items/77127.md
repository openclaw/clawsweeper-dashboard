Full review comments:

- **[P2] Recover append writes even when content already matched:** `src/agents/sessions/tools/write.ts:366-370`
  - body: When an existing file already equals the appended payload, `readOriginalWriteState` marks the precheck as `same`. If the backend appends successfully and then reports a timeout/abort, the readback would be `beforeContent + content`, but this `changed` gate returns `null` before the append comparison runs, surfacing a false failure and making a retry likely to duplicate the append. Let append recovery validate `beforeContent + content` regardless of the overwrite-oriented `same` state.
