## Summary

Keep open: current main still lacks a general append path for the agent write tool, and this PR appears to implement the requested capability with focused coverage and real behavior proof. The remaining gate is maintainer approval of the user-facing API shape plus final CI reconciliation.

## What This Changes

Adds an optional `append` parameter to the agent `write` tool for host and sandbox writes, with same-file serialization, regression tests, and a changelog entry.

## Best Possible Solution

Approve the append-mode API shape or redirect it to a separate append tool, then merge once required checks are green to address https://github.com/openclaw/openclaw/issues/40001.

## Reproduction Assessment

Yes, by source inspection. Current main routes ordinary host and sandbox writes through upstream overwrite-only `createWriteTool`, while append behavior is limited to the memory-flush wrapper.

## Solution Assessment

Yes for the implementation path, pending product approval. The optional flag is narrow and backwards-compatible, preserves existing guards, and serializes same-file appends; the open question is whether maintainers prefer a separate append tool instead.
