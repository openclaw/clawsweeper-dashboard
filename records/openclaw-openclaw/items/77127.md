## Summary

Keep open. Current main still lacks general append support for ordinary agent `write` tools, this PR is the active implementation candidate for the canonical data-loss issue, and I found no blocking correctness issue in the current branch; maintainer approval is still needed for the public tool API shape.

## What This Changes

The PR adds optional `append?: boolean` support to the agent `write` tool for host/workspace and sandbox writes, with validation, serialization, regression tests, and a changelog entry.

## Best Possible Solution

Land a maintainer-approved append-capable file primitive that keeps overwrite as the default, preserves host/workspace/sandbox guards, rejects malformed append intent, serializes same-file appends, and closes the canonical data-loss issue after merge.

## Reproduction Assessment

Yes. Current main routes ordinary host and sandbox writes through overwrite-oriented `createWriteTool` paths while append-only behavior is scoped to memory-flush runs, and https://github.com/openclaw/openclaw/issues/40001 provides production cron evidence on that same surface.

## Solution Assessment

Yes, subject to maintainer API approval. The optional `append` flag is a narrow backwards-compatible implementation direction, and the current branch addresses the earlier safety, validation, sandbox, serialization, and abort concerns with focused coverage.
