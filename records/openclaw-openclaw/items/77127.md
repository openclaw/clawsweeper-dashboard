## Summary

Keep this PR open. The change targets a real current-main gap, but the latest head still lets a malformed `append` value fall through to the destructive overwrite path, so it needs a narrow safety fix before merge.

## What This Changes

Adds optional `append` handling to host and sandbox agent write tools, serializes append mutations, adds append/abort/workspace regression coverage, and updates the changelog.

## Best Possible Solution

Land a maintainer-approved append-capable file primitive that preserves workspace and sandbox guards and rejects malformed append intent before any overwrite-capable fallback runs.

## Reproduction Assessment

Yes, by source inspection. Current main routes ordinary host and sandbox writes through overwrite-only `createWriteTool`, while append-only behavior is scoped to memory-flush runs; the linked issue adds production cron evidence on the same path.

## Solution Assessment

No, not as-is. The optional flag is a plausible narrow implementation, but malformed non-boolean `append` values must be rejected instead of delegating to overwrite behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject malformed append values before overwriting:** `src/agents/pi-tools.read.ts:882-884`
  - body: When a model sends an `append` key with a non-boolean value like `"true"` or `1`, `doAppend` is false and this delegates to the original overwrite implementation. That can truncate the shared log this feature is meant to protect, so reject any present non-boolean `append` value before calling the base write tool.
  - confidence: 0.9
