## Summary

Keep open: the append-mode direction is useful for the linked data-loss report, but the PR currently misses sandbox create-on-append parity and queued cancellation behavior.

## What This Changes

The PR adds an optional `append` boolean to the agent `write` tool for host/workspace/sandbox writes, with focused tests and a changelog entry.

## Best Possible Solution

Land an append-capable write surface after sandbox append uses the same parent-directory creation and queued abort safety as normal writes, with focused regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible as overwrite-only for ordinary `write` calls, and the PR regressions are source-reproducible from the branch: sandbox append bypasses parent creation and cancellation is checked only before queue wait.

## Solution Assessment

No. The optional append flag is a narrow maintainable direction, but this implementation should first align sandbox append and cancellation with the upstream write tool contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Create sandbox parent dirs before appending:** `src/agents/pi-tools.read.ts:748-752`
  - body: The append branch bypasses the upstream write tool's `ops.mkdir(dir)` step, and this sandbox `appendFile` calls `bridge.writeFile` without `mkdir: true` or a prior `mkdirp`. As a result, `write({ path: "new/dir/log.txt", content, append: true })` can fail in sandbox mode even though normal writes and the PR description say missing files are created with parent directories.
  - confidence: 0.9
- **[P2] Recheck aborts inside the mutation queue:** `src/agents/pi-tools.read.ts:871`
  - body: The append path checks `signal?.aborted` only before calling `withFileMutationQueue`. If another mutation keeps the queue busy and the signal aborts while this append is waiting, the queued callback still runs and mutates the file; upstream normal writes check the signal from inside the queued operation.
  - confidence: 0.84
