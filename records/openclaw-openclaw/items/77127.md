Full review comments:

- **[P1] Return success once an append has mutated the file:** `src/agents/agent-tools.read.ts:948-950`
  - body: `appendFileWithAbort` throws `AbortError` after `append()` completes whenever the signal fired during the append. For host appends and sandbox read+write appends, that can mean bytes were already appended but the tool call is reported as failed; a caller retrying the failed write can duplicate the appended content. Match the base write tool’s completed-write recovery semantics or otherwise return success once the append mutation is known to have completed.
  - confidence: 0.91
- **[P3] Make the type tests fail when append is missing:** `src/agents/sessions/tools/tool-contracts.test.ts:6-10`
  - body: This assertion checks whether `WriteToolInput` is assignable to a type where `append` is optional, so it would still pass if `append` were removed from `WriteToolInput`. Flip the assertion direction or assert an input object containing `append: true` satisfies the exported type, and mirror that for the write event input test.
  - confidence: 0.86
