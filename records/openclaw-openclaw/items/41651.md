## Summary

Close as implemented on current main: the merged skill-loader change now accepts managed and personal skill-directory symlinks without the false-positive escape warning while preserving per-skill `SKILL.md` containment. The fix is on main only; it is not in the latest `v2026.5.12` release.

## What This Changes

The issue asks OpenClaw to stop repeatedly warning when symlinked personal or managed skill entries point at bundled or otherwise trusted skill roots.

## Best Possible Solution

Keep the current main behavior and let the next release carry the managed/personal symlink fix to users.

## Reproduction Assessment

Yes for the original released behavior, but not as a failing path on current main. `v2026.5.12` still routes personal/managed symlinked child directories through contained realpath checks, while current main now accepts those skill-directory symlinks and keeps `SKILL.md` containment.

## Solution Assessment

Yes. The landed fix is narrower than globally downgrading warnings because it relaxes only user-owned managed/personal skill-directory roots and keeps workspace, extra-dir, bundled, and per-file containment fail-closed.
