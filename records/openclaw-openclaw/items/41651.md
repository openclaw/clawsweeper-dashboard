## What This Changes

Stop false skill path containment warnings when user-owned skill roots contain symlinked skill directories to trusted or bundled skill locations.

## Best Possible Solution

Keep the merged source-scoped loader behavior: user-owned managed/personal skill roots may follow skill-directory symlinks, while untrusted roots and SKILL.md containment stay fail-closed.

Yes. The pre-fix path is source-reproducible from the loader: a managed or personal skill-directory symlink whose realpath escaped its root reached the shared containment warning path; current tests now cover the fixed managed-root case and the preserved SKILL.md escape rejection.

## Solution Assessment

Yes. The landed fix is narrower than globally suppressing warnings because it only relaxes root-level containment for user-owned managed/personal skill-manager roots and keeps workspace, extra, project-agent, bundled, and per-skill file containment intact.
