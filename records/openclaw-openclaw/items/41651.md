## Summary

Close as implemented. Current main now accepts symlinked skill directories from the personal and managed skill roots without emitting the old escape warning, while keeping bundled, workspace, extra-dir, and per-skill `SKILL.md` containment checks fail-closed.

## What This Changes

The issue asks OpenClaw to avoid repeated false-positive skill path containment warnings for intentional symlinked skill directories that point at trusted skill roots.

## Best Possible Solution

Keep the current source-specific trust boundary on main and ship it in the next release; any remaining troubleshooting-page wording can be handled as separate docs cleanup.

## Reproduction Assessment

No current-main failure remains from source inspection. A symlinked personal or managed skill directory no longer reaches the old root-escape warning path, and `SKILL.md` containment still blocks escaped metadata files.

## Solution Assessment

Yes. The landed fix is narrower than downgrading all warnings because it relaxes only managed/personal skill-directory roots, keeps bundled roots contained, and leaves workspace/project/extra symlinks behind explicit `skills.load.allowSymlinkTargets`.
