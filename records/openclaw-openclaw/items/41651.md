## Summary

Keep open. Current main still performs the skill realpath containment check per scanned root and logs an escaped-path warning before duplicate skill records are merged by name, so the reported noisy symlink case is not implemented on main. The open related PRs are implementation candidates, but they need maintainer review around trusted symlink roots and bundled allowlist semantics before this can close.

## What This Changes

The issue asks OpenClaw to stop producing repeated escaped-root warnings for personal or managed skill symlinks that point to trusted bundled or configured skill roots while preserving the security containment check.

## Best Possible Solution

Keep the issue open and resolve it through one focused skill-loader change, likely in `src/agents/skills/workspace.ts`, that suppresses the trusted duplicate or known-root symlink warning case while preserving strict rejection and warning for unknown escapes plus the existing bundled-source allowlist policy. Add regression coverage for personal/managed symlinks to bundled or configured roots, external symlink escapes, and `skills.allowBundled`, and update https://docs.openclaw.ai/tools/skills if the public containment contract changes.
