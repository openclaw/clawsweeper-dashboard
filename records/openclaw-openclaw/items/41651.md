The fix is on main but not in the latest release tag I could verify.

## What This Changes

The issue asks OpenClaw to stop emitting repeated false-positive skill containment warnings when personal skill symlinks point at bundled or other trusted skill directories.

## Best Possible Solution

Keep the current main behavior: managed and personal skill-manager roots can follow symlinked skill directories, while configured workspace/extra/bundled roots and SKILL.md file containment remain explicit and fail-closed.

## Reproduction Assessment

No, not on current main. Source and regression coverage now show the reported managed/personal symlink-directory warning path is accepted without a warning while escaped SKILL.md files are still rejected.

## Solution Assessment

Yes. The landed implementation solves the noisy local skill-manager workflow without broadly relaxing bundled, workspace, or extra-dir containment; configured allowSymlinkTargets remains available for workspace/project/extra layouts.
