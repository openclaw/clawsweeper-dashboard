## Summary

Close as implemented on current main. The skill loader now supports explicit trusted symlink target roots, has regression coverage for the no-warning path, and the docs describe the supported configuration; the fix is current-main-only after the latest release.

## What This Changes

The issue asks OpenClaw to avoid repeated skill-loader symlink-escape warnings for intentional symlinks from configured skill roots into trusted bundled or shared skill roots.

## Best Possible Solution

Keep the current explicit `skills.load.allowSymlinkTargets` contract and require operators to narrowly list trusted real target roots instead of weakening default containment.

## Reproduction Assessment

Yes. Source inspection and current regression coverage show the old warning path for escaping symlinks and the new no-warning path when the trusted target root is configured.

## Solution Assessment

Yes. The current-main implementation is the safer maintainable solution: it preserves strict default containment while giving operators a narrow, schema-backed way to trust intentional symlink targets.
