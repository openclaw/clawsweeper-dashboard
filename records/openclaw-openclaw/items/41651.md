## Summary

Current main already fixes the reported false-positive warning for symlinked skill directories under the personal skills root: `agents-skills-personal` no longer uses configured-root containment for the skill directory itself, while `SKILL.md` still has to resolve inside the resolved skill directory and stricter roots keep their containment rules.

## What This Changes

The issue requests suppressing or avoiding repeated escaped-root warnings when `~/.agents/skills` contains symlinks to bundled or otherwise trusted skill directories.

## Best Possible Solution

Keep the merged source-scoped loader behavior: personal and managed skill-directory symlinks are accepted by default, while per-skill `SKILL.md` files and workspace/project/extra/bundled roots keep explicit containment safeguards.

## Reproduction Assessment

No. On current main, the reported personal `~/.agents/skills` symlink path uses `agents-skills-personal`, which no longer calls the configured-root escaped-path warning for the skill directory itself.

## Solution Assessment

Yes. The merged fix is narrower than globally downgrading warnings because it handles the reported user-owned personal skill-root workflow while preserving stricter containment and per-skill file checks.
