## Summary

Keep open: the linked gateway bug is valid and the fix is close, but the new guard checks the raw persisted `sessionFile` path instead of OpenClaw's canonical session-file resolver, so supported relative or alias transcript paths can be misclassified as missing and rotated. The PR also has only mocked/unit terminal proof, not real behavior proof for the external-PR merge gate.

## What This Changes

The PR adds a failed-session missing-transcript guard to gateway agent session reuse and a focused regression test in `src/gateway/server-methods/agent.test.ts`.

## Best Possible Solution

Keep the failed-lane recovery behavior, but resolve the stored transcript through the canonical session-file helpers before checking existence, add coverage for both missing and present resolved paths, and require redacted real gateway/channel proof before merge.

## Reproduction Assessment

Yes. Source inspection and the PR's focused before/after test scenario show a high-confidence repro by seeding a fresh `status: "failed"` session with a missing `sessionFile`; I did not execute tests in this read-only review.

## Solution Assessment

No. The recovery policy is the right direction, but the implementation should use the existing resolver contract before checking file existence and should prove the real gateway/channel recovery path.
