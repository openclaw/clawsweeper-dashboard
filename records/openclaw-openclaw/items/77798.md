## Summary

Keep open: current main still lacks the requested bidirectional Canvas Markdown editor. The existing code covers URL-rendered embeds, native/A2UI action plumbing, live reload, a read-only diff viewer plugin, and an admin-scoped agent file editor, but not browser edit events bound to workspace files, diff capture, Canvas refresh semantics, or next-turn agent context injection.

## What This Changes

Request a live, two-way Markdown editor embedded in Canvas so user edits can reach the agent and agent disk writes can refresh the Canvas.

## Best Possible Solution

Define a secure collaborative-editing contract first: authenticated edit transport, explicit workspace and file boundaries, stale-write handling, diff capture, agent context injection, and refresh semantics, then decide whether the editor is bundled UI or an official plugin built on that API.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken existing behavior. Source inspection on current main confirms the requested Canvas edit/write/diff/next-turn context path is absent while existing paths are render-oriented or admin-editor-only.

## Solution Assessment

Unclear as an implementation path: the requested UX is useful, but the narrow maintainable solution needs product and security design before code. The safer direction is to define the edit transport, authorization, diff, refresh, and context-injection contract before choosing bundled UI versus plugin implementation.
