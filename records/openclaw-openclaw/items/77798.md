## Summary

Keep open: current main has hosted Canvas embeds, live reload, and native A2UI action plumbing, but it does not provide the requested bidirectional Control UI markdown editor or browser-to-agent edit loop. Closing this to ClawHub would be premature because the safe transport, workspace write, diff, and Control UI bridge boundaries still need core/product design.

## What This Changes

The issue requests a single-file collaborative markdown editor embedded in Canvas with user edits flowing back into the agent context and agent writes refreshing the Canvas view.

## Best Possible Solution

Define a secure Control UI editing contract first: authenticated edit transport, explicit workspace/file boundaries, diff capture, agent context injection, and refresh semantics, then decide whether the editor is core UI or an official/plugin implementation.

## Reproduction Assessment

Not applicable: this is a new feature request rather than a report of broken existing behavior. Source inspection confirms current main has render/native-action Canvas plumbing but not the requested Control UI collaborative editor path.

## Solution Assessment

No, closing this as ClawHub-only is not the best solution yet. Current plugin and Canvas surfaces are close to useful building blocks, but the browser iframe-to-agent bridge and safe file-write/diff contract still need maintainer design before an implementation can be safely delegated.
