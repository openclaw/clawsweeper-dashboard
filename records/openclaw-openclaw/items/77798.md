## Summary

Keep open. Current main and v2026.6.6 still provide URL-backed render-only Canvas embeds, native-only A2UI action helpers, a static GET/HEAD Canvas host, and allowlisted bootstrap-file RPCs, but not the requested collaborative Markdown editor, browser edit ingress, arbitrary Markdown writeback, refresh contract, or diff/context injection.

## What This Changes

Request for a live two-way Markdown editor embedded in Canvas so user edits and agent edits sync through the Control UI.

## Best Possible Solution

Define a security-reviewed Canvas editing contract, probably plugin-owned with any missing core/plugin API seams made explicit, then implement a narrow Markdown editor on top of that contract with coordinated browser action, workspace write, refresh, and diff/context semantics.
