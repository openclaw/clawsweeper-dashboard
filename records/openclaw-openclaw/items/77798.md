## Summary

Keep open: current main and v2026.6.8 still provide render-only Control UI embeds plus Canvas/A2UI presentation and action plumbing, but not a two-way Markdown editor with user edits injected into agent context. The request is valid product work, but it needs maintainer product and security review before implementation.

## What This Changes

Add a live Canvas-embedded Markdown editor where users can edit a workspace file, the agent sees diffs next turn, and agent disk writes refresh the Canvas view.

## Best Possible Solution

Design an explicit editable-document contract for Canvas/Control UI with authenticated edit events, bounded workspace file access, diff/session injection, conflict handling, and focused tests/docs before implementing the editor.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence absence proof that current main only renders Canvas embeds and does not provide two-way edit, diff, or context-injection behavior.

## Solution Assessment

Unclear until maintainers choose the product and security shape. A narrow implementation should start from an explicit editable-document and session-context contract rather than adding ad hoc iframe message handling.
