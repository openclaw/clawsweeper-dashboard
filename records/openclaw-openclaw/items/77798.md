## Summary

Keep open. Current main and the latest release still render `[embed]` Canvas content as URL-backed iframe/static Canvas output, with native-only action helpers and no collaborative Markdown editor, diff/context injection, or browser-to-agent writeback path.

## What This Changes

The issue requests a two-way collaborative Markdown editor embedded in Canvas with browser-origin edits, diff/context injection, and refresh from agent writes.

## Best Possible Solution

Define a plugin-owned, security-reviewed Canvas editing contract coordinated with the open browser action/web Canvas work, then implement the Markdown editor on top of that contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main still lacks the requested editor, diff/context injection, and browser writeback path.

## Solution Assessment

Unclear as proposed. A Canvas Markdown editor is plausible, but a one-off editor or POST endpoint is not the best first step without a reviewed browser action, workspace write-scope, diff, and plugin ownership contract.
