## Summary

Keep open: current main still lacks a bidirectional Canvas markdown editor. The existing code supports render-oriented hosted embeds, A2UI/native action plumbing, live reload, and an admin-scoped agent-file editor, but not browser edit events bound to workspace files, diff capture, Canvas refresh semantics, or next-turn agent context injection.

## What This Changes

Request a live two-way Markdown editor embedded in Control UI Canvas, with user edits sent back to the agent and disk writes refreshing the Canvas.

## Best Possible Solution

Define a secure collaborative-editing contract first: authenticated edit transport, explicit workspace and file boundaries, stale-write handling, diff capture, agent context injection, and refresh semantics, then decide whether the editor is bundled UI or an official plugin built on that API.

## Reproduction Assessment

Not applicable: this is a new feature request, not a report of broken existing behavior. Source inspection on current main confirms the render/native-action plumbing exists, but the collaborative Canvas markdown editor path does not.

## Solution Assessment

Unclear as an implementation path: the requested UX is reasonable, but it needs maintainer product and security design before code work. The narrow maintainable direction is to define the edit transport, authorization, diff, refresh, and agent-context contract before choosing bundled UI versus plugin implementation.
