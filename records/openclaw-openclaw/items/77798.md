## Summary

Keep open. Current main still provides Canvas rendering and native A2UI action helpers, but not a collaborative Markdown editor with browser-origin edit events, diff/context injection, workspace writeback, and refresh semantics; the remaining work needs maintainer product and security design.

## What This Changes

The issue requests a live two-way Markdown editor embedded in Control UI Canvas, with user edits sent back to the agent and agent file writes reflected in the Canvas.

## Best Possible Solution

Define a plugin-owned, security-reviewed Canvas editing/action contract first, then implement the Markdown editor as a focused follow-up once browser ingress, workspace scope, diff/context injection, and refresh behavior are agreed.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main lacks the requested browser editor, diff, and file-writeback path.

## Solution Assessment

Unclear as proposed. A Canvas-based Markdown editor is plausible, but the best path is to settle browser action ingress, workspace write scope, diff/context contract, and Canvas plugin ownership before adding a one-off editor endpoint.
