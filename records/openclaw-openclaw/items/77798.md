## Summary

Keep open. Current main still provides render-only Canvas embeds, native A2UI bridge helpers, and narrow bootstrap-file RPCs, but not the requested collaborative Markdown editor, browser-origin edit ingress, arbitrary Markdown writeback, Canvas refresh contract, or diff/context injection.

## What This Changes

The issue requests a two-way collaborative Markdown editor embedded in Control UI Canvas, with user edits sent back to the agent and agent writes reflected in the Canvas.

## Best Possible Solution

Define a security-reviewed Canvas/browser action and workspace-edit contract first, then implement the Markdown editor on that contract or keep it as a plugin if no core API gap remains.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence verification that current main lacks the requested editor, browser edit ingress, arbitrary Markdown writeback, and diff/context injection path.

## Solution Assessment

Unclear as proposed. Canvas is a plausible UI surface, but the best implementation depends on a maintainer decision for browser action ingress, workspace write scope, diff/context semantics, and plugin/core ownership.
