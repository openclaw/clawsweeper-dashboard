## Summary

Keep open: the reported `/models` picker regression is real and the latest auth-gate follow-up addresses an important visibility contract, but the PR still puts Anthropic-owned model catalog data directly into a core command. That remaining boundary violation should be repaired before merge.

## What This Changes

The branch makes CLI runtime providers visible and auth-gated in the `/models` picker, adds CLI runtime model population including a Claude CLI allowlist path, updates command tests, and adds a changelog entry.

## Best Possible Solution

Move Claude CLI model seeding into Anthropic-owned catalog metadata or another public plugin seam, leaving core `/models` code to consume generic catalog entries while preserving the visibility and auth-gate regression fix.

## Reproduction Assessment

Yes. Current main is source-reproducible: the picker hides every provider classified by `isLegacyRuntimeModelProvider()`, and that predicate currently treats CLI runtime aliases as legacy; the inspected Telegram proof also shows the changed real picker behavior after the PR.

## Solution Assessment

No. The predicate and auth-gate direction is right, but directly importing an Anthropic plugin allowlist into core is not the maintainable fix when existing plugin catalog/manifest seams are designed for provider-owned model metadata.
