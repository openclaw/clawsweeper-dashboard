## Summary

Keep open: this is a valid regression fix area, but the latest PR head appears to wire the Claude CLI allowlist into a provider catalog hook that the `/models` catalog path does not currently consume, so the 6-row Claude CLI picker can still be missing on fresh or sparse catalog installs.

## What This Changes

The PR makes CLI runtime providers visible in `/models`, widens CLI runtime model lists from catalog entries with auth gating, adds Claude CLI catalog metadata under the Anthropic plugin, and updates tests plus changelog.

## Best Possible Solution

Keep the boundary-respecting fix, but contribute Claude CLI allowlist rows through the catalog path that `loadModelCatalog()` actually reads, add a regression that exercises that integration instead of only mocking catalog rows, and then refresh latest-head Telegram proof.

## Reproduction Assessment

Yes. Current main is source-reproducible: CLI runtime aliases are classified as legacy and the `/models` picker filters legacy providers before rendering provider choices.

## Solution Assessment

No. The predicate and auth-gated widening direction is good, but the final Claude CLI catalog source is connected to `ProviderPlugin.catalog` while `/models` reads `loadModelCatalog()`, whose provider augmentation path calls `augmentModelCatalog` instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Populate the catalog path that /models reads:** `extensions/anthropic/provider-discovery.ts:35-37`
  - body: The latest branch moves the Claude CLI allowlist into `provider-discovery.ts` as a `ProviderPlugin.catalog` hook, but `/models` only widens entries already returned by `loadModelCatalog()`. That loader's provider augmentation currently calls `plugin.augmentModelCatalog`, not `plugin.catalog`, so a fresh or sparse catalog install still will not get the 6 `claude-cli` rows this PR is trying to surface. Please wire these rows into the actual `loadModelCatalog` path or add the missing bridge and cover it with an integration-style regression.
