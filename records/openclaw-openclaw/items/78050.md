## Summary

Keep open: the patch addresses a real Discord progress-draft bug, but the proposed append-only reasoning handling breaks the embedded runner path that sends full reasoning snapshots, and the PR only provides mock/test proof rather than a live Discord progress run.

## What This Changes

This PR accumulates Discord reasoning stream text into progress drafts, suppresses empty shared `Reasoning` progress items, adds regression tests, and adds a changelog fix note.

## Best Possible Solution

Keep the shared suppression, but make Discord reasoning progress handle both delta and full-snapshot callback payloads, with regression coverage for both Codex app-server and embedded-runner semantics before merge.

## Reproduction Assessment

Yes from source inspection: current main ignores Discord reasoning payload text while the shared formatter can render an empty analysis item title as `Reasoning`, matching the reported progress-draft symptom. I did not run a live Discord bot because this review is read-only.

## Solution Assessment

No as proposed: appending every callback payload is not the narrowest correct fix because OpenClaw has both delta and snapshot reasoning-stream emitters. The safer fix is to normalize the callback contract or update the Discord progress line with the latest snapshot when payloads are not deltas.
