## Summary

Current main fixes the reported startup stall path: default/configured `models.list` now uses a read-only catalog path that returns persisted, static, and configured model rows without running provider runtime catalog hooks; full discovery remains reserved for `view: "all"`.

## What This Changes

The issue asks OpenClaw to avoid scanning all provider plugins during Codex catalog fallback when explicit `models.providers` entries are configured, or to add a switch to disable that behavior.

## Best Possible Solution

Keep the current read-only model-list implementation and reserve full provider discovery for explicit diagnostics or browse views.

## Reproduction Assessment

No for current main. The v2026.4.29 path is source-plausible, but current main routes default/configured `models.list` through read-only catalog loading and configured rows instead of provider runtime discovery.

## Solution Assessment

Yes. The current implementation is narrower than a new `codex.catalog.enabled` setting because it fixes the hot model-list path generically while preserving exact full discovery for `view: "all"`.
