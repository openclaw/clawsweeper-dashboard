## Summary

Close as implemented. Current main and the v2026.5.3 tag keep default/configured `models.list` on read-only persisted/static/configured catalog rows and static auth checks, so the reported provider-runtime discovery stall no longer runs on that path; the earlier #75503 normalization cache was useful adjacent context, while #76406/#76517 addressed this fallback path directly.

## What This Changes

The issue asks OpenClaw to avoid Codex fallback/provider catalog scans during startup when custom `models.providers` are configured, or to expose a disable knob for that behavior.

## Best Possible Solution

Keep the read-only `models.list` path on persisted/static/configured metadata and reserve provider runtime discovery for explicit full-catalog/admin paths, without adding a Codex-specific disable knob for normal startup.

## Reproduction Assessment

No on current main. The original v2026.4.29 path was source-reproducible, but current source and regression tests show default/configured `models.list` no longer enters provider runtime discovery or Codex live catalog fallback work.

## Solution Assessment

Yes. The implemented fix is narrower than a Codex-specific config flag because it removes broad provider discovery from the read-only Gateway list path while preserving `view: "all"` for callers that explicitly need the full catalog.
