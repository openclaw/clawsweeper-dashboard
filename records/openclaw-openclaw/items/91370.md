## Summary

Keep open: this member-authored PR is a focused candidate for the canonical sessions_yield raw text-direct fallback bug, but the current patch can still complete cleanup after a failed settled-descendant wake and should be repaired or explicitly accepted before merge.

## What This Changes

The PR persists a sessions_yield completion marker, uses it to skip subagent announce/text-direct fallback for yielded completions, avoids child delivery credit, resets the marker on steer restart, and adds focused agent regression tests.

## Best Possible Solution

Keep the yielded-completion suppression, but first make failed settled-descendant wake stay retryable and have maintainers accept the parent-owned delivery plus durable marker contract before merge.

## Reproduction Assessment

Yes, at source level: current main clears the sessions_yield pause marker before announce cleanup and still has the direct-message :text-direct child-output fallback. I did not run a live external-channel reproduction in this read-only review.

## Solution Assessment

No, not quite yet: the main fix shape is appropriate, but the yield skip should not treat a failed settled-descendant wake as successful cleanup. The safer path is to keep that failure retryable, then let maintainers accept the parent-owned delivery contract.
