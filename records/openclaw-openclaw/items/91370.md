## Summary

Keep open: the fix targets the right sessions_yield text-direct fallback, but it still has a blocking edge case where a failed settled-descendant wake can be treated as successful yield cleanup, clearing retry state before descendant findings are delivered.

## What This Changes

The PR adds a durable completedFromYieldPause run marker, uses it to skip subagent announce/text-direct fallback for sessions_yield completions, resets the marker on steer restart, and adds focused regression tests.

## Best Possible Solution

Keep the yielded-completion suppression, but first make failed settled-descendant wake stay retryable and have maintainers accept the parent-owned delivery plus durable marker contract before merge.

## Reproduction Assessment

Yes, at source level: current main clears the sessions_yield pause marker before announce cleanup and still has the direct-message text-direct child-output fallback. I did not run a live external-channel reproduction in this read-only review, but the linked issue includes live current-main channel evidence.

## Solution Assessment

No, not quite yet: the main fix shape is appropriate, but the yield skip should not treat a failed settled-descendant wake as successful cleanup. The safer path is to keep that failure retryable, then let maintainers accept the parent-owned delivery contract.
