## Summary

Keep this PR open: it targets a real P1 message-delivery bug, but the current patch can mark a yielded completion as delivered without any external delivery evidence, which can suppress the existing direct-DM fallback and clear retry state.

## What This Changes

The branch persists a `completedFromYieldPause` marker, passes it through subagent announcement delivery, and suppresses text-direct/message-tool-missing fallback paths for `sessions_yield` completions with added unit coverage.

## Best Possible Solution

Coordinate yield-resume and announce delivery so raw child fallback is suppressed only after the parent resume reply has concrete delivery evidence; otherwise keep the direct-DM fallback or return a retryable missing-delivery result.

## Reproduction Assessment

No live reproduction was run in this read-only review. Source inspection of the PR head shows a reproducible branch-level failure mode: yield completions can return `delivered: true` with no `sendMessage`, gateway payload, message-tool evidence, or parent-delivery proof.

## Solution Assessment

No. Suppressing the raw child fallback is plausible, but the safer fix is to suppress it only when the parent resume reply has delivery evidence or to leave the run retryable instead of crediting delivery unconditionally.
