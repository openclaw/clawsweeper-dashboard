Keep open: the PR targets a valid sessions_yield delivery bug, but the current patch skips announce handling before the existing descendant-pending guard and still lacks real external-channel proof for the parent-owned delivery path.

## What This Changes

The PR adds a persisted completedFromYieldPause marker, uses it to skip subagent announce/text-direct delivery and delivery credit for sessions_yield completions, and adds focused regression tests.

## Best Possible Solution

Revise the PR so yielded completions still honor descendant deferral, then merge only with maintainer acceptance and live or equivalent route proof that parent-owned delivery is reliable.

## Reproduction Assessment

Yes at source level: current main clears pauseReason before announce cleanup, and the announce delivery layer can send child task_completion text through :text-direct when gateway/message-tool evidence is absent. I did not establish a live external-channel reproduction in this review.

## Solution Assessment

No as currently ordered: suppressing announce for yield is plausible, but the early return must not bypass the existing descendant-pending deferral. The safer fix preserves that guard, then skips only the child fallback/delivery credit for yielded completions.
