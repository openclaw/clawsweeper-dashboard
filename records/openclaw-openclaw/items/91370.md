## Summary

Keep open: the patch still returns early for sessions_yield before the settled-descendant wake path, so it can finalize cleanup without giving a previously deferred child run its settled descendant findings.

## What This Changes

The PR adds a durable yielded-completion marker to subagent run state, forwards it into announce cleanup, skips text-direct fallback for sessions_yield completions, avoids child delivery credit, and resets the marker on steer restart.

## Best Possible Solution

Move the sessions_yield skip after the settled-descendant wake branch, keep non-yield fallback behavior unchanged, and preserve the parent-owned delivery-credit guard with focused regression coverage.
