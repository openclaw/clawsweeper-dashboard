## Summary

Keep open; the patch targets the right message-delivery race but is not merge-ready because the yield marker suppresses existing direct-message fallback delivery before proving that the parent resume reply reached the user.

## What This Changes

The PR adds a persisted `completedFromYieldPause` marker, forwards it through subagent announce delivery, and guards text-direct/message-tool-missing fallback paths for `sessions_yield` completions.

## Best Possible Solution

Rework the fix so yield-specific announce suppression is tied to observable parent-resume external delivery, or preserve the https://github.com/openclaw/openclaw/pull/88182 child-result fallback whenever that proof is absent.

## Reproduction Assessment

Yes. The issue is source-reproducible: current main clears `pauseReason` before announce cleanup and retains direct-text fallback behavior when gateway payload evidence is missing, though I did not run a live Feishu or Discord round trip.

## Solution Assessment

No. The proposed marker is a plausible mitigation for duplicate raw child text, but it is too broad because it credits delivery without preserving the established no-reoutput fallback when parent delivery is absent.
