## Summary

## What This Changes

The PR adds a durable yielded-completion marker, uses it to skip subagent announce/text-direct fallback for sessions_yield completions, avoids child delivery credit, resets the marker on steer restart, and adds focused agent regression tests.

## Best Possible Solution

Keep the yielded-completion suppression, but first make failed settled-descendant wake stay retryable and have maintainers accept the parent-owned delivery plus durable marker contract before merge.

## Reproduction Assessment

Yes, at source level: current main clears the sessions_yield pause marker before announce cleanup and still has the direct-message :text-direct child-output fallback. I did not run a live external-channel reproduction in this read-only review.

## Solution Assessment

No, not yet: the main fix shape is appropriate, but the yield skip should not treat a failed settled-descendant wake as successful cleanup. The safer path is to keep that failure retryable, then let maintainers accept the parent-owned delivery contract.
