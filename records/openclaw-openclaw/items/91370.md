Keep open: this PR is the active, focused fix candidate for the canonical `sessions_yield` delivery bug; current main still lacks the yield-origin marker and the branch intentionally changes delivery/session-state semantics that need maintainer acceptance before merge.

## What This Changes

Adds a durable yield-completion marker, forwards it into subagent announce cleanup to skip raw child text-direct fallback for `sessions_yield` completions, resets it on steer restart, and adds focused agent regression tests.

## Best Possible Solution

Land this branch or an equivalent narrow fix once maintainers accept parent-owned yielded delivery and any desired live transport proof is captured.

## Reproduction Assessment

Yes at source level: current main clears the `sessions_yield` pause marker before announce cleanup and still has the direct-message `:text-direct` child-output fallback. I did not run a live external-channel reproduction, but the linked report includes live Feishu and WhatsApp evidence for the same failure mode.

## Solution Assessment

Yes: capturing yield-origin state before `pauseReason` is cleared is the narrow owner-boundary fix, and placing the skip after descendant deferral/wake preserves the existing retry semantics. A delivery-layer dedupe would be broader and still would not directly preserve yield-origin state for cleanup retries.
