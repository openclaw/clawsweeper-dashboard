## Summary

Keep open for maintainer review. The patch looks like the right narrow bug fix for the linked yielded-subagent completion bug, but it intentionally suppresses the child announce/text-direct fallback and relies on the parent resume path as the only user-visible delivery path, so maintainers should explicitly accept that delivery/state contract before merge.

## What This Changes

The PR adds a durable `completedFromYieldPause` marker to subagent run records, uses it to skip announce/text-direct fallback for `sessions_yield` completions, avoids child delivery credit, resets the marker on steer replacement, and adds focused agent tests.

## Best Possible Solution

Land the narrow lifecycle-marker fix after maintainers accept that yielded completions are parent-delivered and after required checks finish; keep broader completion-route preservation work separate unless maintainers want a larger routing redesign.

No, I did not run a live current-main reproduction. The linked issue has detailed live transcript evidence and current source shows the same `pauseReason` clearing plus `:text-direct` fallback path, so the failure is source-reproducible.

## Solution Assessment

Yes, this is the best narrow fix for the reported yielded-completion fallback: preserve the yield fact at the lifecycle boundary, skip the fallback before delivery, and avoid false child delivery credit. A broader route-preservation redesign remains adjacent rather than required for this PR's core fix.
