## Summary

Keep open: this maintainer-associated PR is not eligible for cleanup close, and the latest patch still has a blocking ordering bug. The yield skip now honors active-descendant deferral, but it still runs before the settled-descendant wake path, so a deferred yielded run can finalize cleanup before its child run is woken with descendant findings.

## What This Changes

The PR adds a persisted `completedFromYieldPause` marker, uses it to skip announce/text-direct fallback and delivery credit for `sessions_yield` completions, resets the marker on steer restart, and adds focused agent tests.

## Best Possible Solution

Move the yield skip after the `wakeOnDescendantSettle` branch, keep non-yield fallback behavior unchanged, and validate both the nested-descendant retry path and the direct sessions_yield raw-text suppression.

## Reproduction Assessment

Yes at source level. Current main clears `pauseReason` before announce cleanup and still has the raw `:text-direct` fallback; the PR diff shows the new yield skip before the settled-descendant wake branch, so the review finding is source-reproducible without a live channel run.

## Solution Assessment

No, not as currently ordered. Skipping child fallback for yielded completions is a plausible owner-boundary fix, but it must still honor both descendant phases: active-descendant deferral and later `wakeOnDescendantSettle` re-wake before final cleanup.
