## Summary

Keep open: this member-authored PR targets a real P1 subagent message-delivery bug, but the current head still credits a yielded completion as delivered before any parent-resume or announce-path delivery evidence exists. That can suppress the raw-child fallback while also finalizing cleanup, leaving the user with no visible completion instead of the intended parent reply.

## What This Changes

The branch persists a `sessions_yield` completion marker on subagent run records, passes it into the announce flow, and makes yielded completions skip the announce agent with focused tests.

## Best Possible Solution

Make parent/requester resume delivery authoritative: suppress the raw child fallback only after concrete parent reply delivery evidence exists, otherwise keep cleanup retryable or preserve an explicit fallback path.

## Reproduction Assessment

No live DM reproduction was run in this read-only review. Source inspection of the PR head shows a high-confidence branch-level failure path: `requesterPausedForYield` returns `true` before any delivery evidence is recorded, and lifecycle cleanup treats that as delivered.

## Solution Assessment

No. Skipping the announce agent is a plausible mitigation for raw-child duplication, but it is not the best fix because it does not prove the parent resume reply was externally delivered before clearing retry/fallback state.
