## Summary

Keep open: this targets a valid sessions_yield message-delivery bug, but the patch only suppresses the post-response text-direct fallbacks and leaves the incomplete-result text-direct fallback active for the same yielded completion path.

## What This Changes

The branch stamps sessions_yield completions on the subagent run record, forwards a requesterPausedForYield flag through announce delivery, and skips several direct text fallback checks with regression tests.

## Best Possible Solution

Keep the yield-completion marker approach, but make it the single gate around all raw child text fallback sends and preserve non-yield fallback behavior with tests.

## Reproduction Assessment

## Solution Assessment

No, not yet. The marker-and-guard shape is narrow and plausible, but the same guard needs to cover the incomplete-result fallback before this is the best fix.
