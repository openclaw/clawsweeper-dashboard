## Summary

Keep open. The report describes a current availability bug, and source inspection on current main still shows Telegram media dispatch can remain coupled to an awaited agent/tool turn; the newer isolated polling spool is useful mitigation, not proof that Windows image-tool stalls are fixed.

## What This Changes

The issue reports that Telegram image/video input can stall gateway health and polling on Windows while the image tool is active.

## Best Possible Solution

Keep Telegram polling and gateway health responsive while image/media tool turns run, preserve per-chat/per-topic ordering, and make stalled Telegram sessions recover without deleting sessions.json.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection and the reporter's logs provide a plausible current-main path through Telegram media dispatch, awaited inbound turns, and image-tool buffer/provider work, but Windows/Telegram proof is still needed.

## Solution Assessment

Unclear. The current isolated polling spool is a partial mitigation because it detects or isolates ingress backlog stalls, but the best fix should decouple or fail-safe long image/media turns so gateway health and Telegram session recovery remain responsive.
