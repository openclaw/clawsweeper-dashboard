## Summary

Keep open. The report is recent, assigned for follow-up, and current main only partially isolates Telegram polling; media dispatch still awaits serialized inbound turns and image-tool execution, so the Windows image-stall failure is not proven fixed.

## What This Changes

Telegram image or video messages on Windows are reported to freeze the gateway event loop and stop later Telegram polling until the stalled session is manually removed.

## Best Possible Solution

Make Telegram media ingress resilient when image understanding stalls: polling should continue fetching/spooling, gateway health should remain responsive, same-chat ordering should be preserved, and stalled sessions should recover without deleting sessions.json.

## Reproduction Assessment

No high-confidence live reproduction was run. The reporter provided concrete Windows logs, and current source shows a plausible path through Telegram media dispatch, serialized inbound debouncing, runInboundReplyTurn, and image-tool execution.

## Solution Assessment

Unclear. The isolated ingress worker is the right partial mitigation for getUpdates durability, but a complete fix needs owner review around stalled media/image turns, gateway liveness, and same-lane ordering.
