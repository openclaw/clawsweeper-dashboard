## Summary

Keep open: this is a valid report of broken existing WhatsApp multi-account cold-start behavior, and current main still routes every configured account through the same gateway startup and WhatsApp `readWebAuthState` check. I found no current-main or latest-release evidence that the packaged secondary-account crash is fixed.

## What This Changes

A linked WhatsApp secondary account with an explicit `authDir` should survive gateway restart and register its persistent listener instead of crashing during startup.

## Best Possible Solution

Patch the WhatsApp cold-start/runtime-loading path so linked default and secondary accounts both read auth state and register persistent listeners after restart, with regression coverage for a secondary explicit `authDir`.

## Reproduction Assessment

No high-confidence live reproduction on current main yet. The reporter provides concrete steps, and source inspection maps them to gateway account startup and the WhatsApp `readWebAuthState` path, but the packaged cold-boot TypeError was not executed locally.

## Solution Assessment

No patch is proposed yet. The narrowest maintainable fix should start in the WhatsApp full-runtime or packaged plugin-loader startup path, adding a focused regression for a linked secondary account with explicit `authDir`.
