Keep open: this reports broken existing WhatsApp multi-account cold-start behavior, and current main still routes configured accounts through the same startup/auth path that can hit `readWebAuthState` for each account. I did not find a current-main fix or canonical duplicate, but the exact packaged TypeError still needs runtime reproduction before changing the loader/runtime boundary.
