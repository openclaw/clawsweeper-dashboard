## Summary

Keep open: this is a concrete current-release WhatsApp multi-account boot crash, and current main still has the same per-account startup preflight path that dereferences the WhatsApp auth runtime before handing off account startup. I did not find a merged fix or a canonical duplicate.

## What This Changes

A secondary WhatsApp account with an explicit `authDir` should boot and register its persistent inbound listener alongside the default account after a gateway restart.

## Best Possible Solution

Add a focused regression for default plus secondary WhatsApp gateway startup with an explicit `authDir`, then fix the WhatsApp startup/auth-runtime boundary so every account uses a defined auth read and listener startup path.
