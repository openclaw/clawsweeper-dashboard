## Summary

Keep open: this is a recent, concrete report of broken existing WhatsApp multi-account cold-start behavior. Current main still has the same startup/auth path that can reach `readWebAuthState` for each configured account, and I did not find a current-main fix, shipped fix, or canonical duplicate in the provided context.

## What This Changes

A secondary WhatsApp account with an explicit `authDir` should survive gateway cold boot and register its inbound listener instead of crashing while reading auth state.

## Best Possible Solution

Patch the WhatsApp plugin startup or plugin-loader boundary so every enabled configured account, including a secondary account with an explicit `authDir`, reads auth state safely and starts its listener after restart.
