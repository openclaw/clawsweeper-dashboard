## Summary

Keep open: this is a focused subagent billing/auth guardrail with supplied terminal proof, but the patch currently rejects canonical `openai/*` agent overrides as direct API routes even though current main routes official OpenAI agent model refs through the Codex harness by default. That regression should be fixed before merge.

## What This Changes

The PR adds a forbidden subagent model-plan status for `gpt`/`openai/*` overrides under an `openai-codex/*` subagent default, propagates it through direct subagent spawn, and adds regression tests.

## Best Possible Solution

Keep the guard only for actual direct OpenAI API-key or direct PI routes, while allowing canonical `openai/*` overrides that current policy will run through the Codex harness; add tests for both safe canonical and truly direct/custom OpenAI cases.

## Reproduction Assessment

No, not for the PR's direct-API-bypass claim as written: current source and docs show official `openai/*` agent model refs route through the Codex harness by default. The introduced rejection is source-reproducible from the PR diff because all resolved `openai/*` overrides under an `openai-codex/*` default become forbidden.

## Solution Assessment

No. The patch should distinguish actual direct OpenAI API routing from canonical OpenAI model refs that current OpenClaw policy keeps subscription-backed through the Codex harness.
