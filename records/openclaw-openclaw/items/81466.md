## Summary

Keep open: the PR has a focused guardrail and supplied terminal proof, but the patch still rejects canonical `openai/*` agent routes that current main documents and tests as Codex-backed by default.

## What This Changes

The PR adds a subagent planner guard that returns `forbidden` for `gpt` or `openai/*` model overrides under an `openai-codex/*` subagent default, propagates that status, and adds regression tests.

## Best Possible Solution

Preserve the documented canonical `openai/*` Codex route, and only reject overrides when the planner can prove they will use an explicit direct/custom OpenAI runtime.

## Reproduction Assessment

No for the PR's direct-API-bypass claim as written; current source and docs show official `openai/*` agent refs use the Codex runtime by default. Yes for the PR-introduced regression by source inspection: the new guard forbids all resolved `openai/*` overrides under an `openai-codex/*` default.

## Solution Assessment

No. The patch should distinguish actual direct OpenAI API routing from canonical OpenAI model refs that current OpenClaw policy keeps Codex-backed.

## Review Findings
