## Summary

Keep open: the supplied terminal proof shows the new planner guard behavior, but the patch still conflicts with the current OpenAI routing contract by treating canonical `openai/*` agent refs as direct API routes even though current main routes official OpenAI agent turns through the Codex harness by default.

## What This Changes

The PR adds a forbidden subagent model-plan status for `gpt`/`openai/*` overrides under `openai-codex/*` subagent defaults, threads it through direct subagent spawn, and adds regression coverage.

## Best Possible Solution

Repair the guard to consult the existing OpenAI/Codex runtime policy and block only actual direct/custom OpenAI routes, with tests for both canonical Codex routes and truly direct-route cases.

## Reproduction Assessment

No for the original direct-API-bypass claim as written: current main and docs show official `openai/*` agent refs route through the Codex harness by default. Yes for the PR regression: the diff source-reproducibly rejects all `openai/*` overrides under an `openai-codex/*` subagent default.

## Solution Assessment

No. The proposed guard is too broad; the maintainable fix is to use the existing runtime-policy helpers so canonical `openai/*` Codex routes stay valid while actual direct/custom routes are guarded.
