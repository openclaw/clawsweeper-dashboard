## Summary

Keep open: the guard targets a real subagent routing problem, but the current patch would fail-close legitimate `openai/*` selections that OpenClaw routes through Codex by default, so it needs a narrow policy-aware repair before merge.

## What This Changes

The PR adds a subagent model-planning guard that returns a forbidden status for bare `gpt` and `openai/*` overrides when the configured subagent default is `openai-codex/*`, propagates that through direct spawns, and adds planner tests.

## Best Possible Solution

Keep the subagent guardrail, but make it use the existing OpenAI Codex routing/auth policy so only true direct/custom OpenAI API routes are rejected and Codex-routed `openai/*` selections remain allowed.

## Reproduction Assessment

Yes at source level: current main has no planner guard, and the PR body includes terminal output from the patched planner exercising the risky and safe override cases. I did not run tests because this was a read-only review.

## Solution Assessment

No: a subagent guard is a maintainable direction, but this implementation is too broad because it treats every `openai/*` ref as direct API instead of consulting the existing OpenAI Codex routing policy.
