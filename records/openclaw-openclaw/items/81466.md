## Summary

Keep this PR open, but the current patch should not merge as-is: it correctly targets subagent model-override routing, yet it overblocks canonical `openai/*` Codex-runtime model refs that current OpenClaw docs and source explicitly support.

## What This Changes

The PR adds a subagent spawn planner guard that returns `forbidden` for bare `gpt` or `openai/*` overrides when the configured subagent default is `openai-codex/*`, wires that status through `spawnSubagentDirect`, and adds planner regression tests.

## Best Possible Solution

Keep the guardrail, but narrow it so canonical `openai/gpt-*` Codex-runtime refs and configured aliases remain usable; only reject cases that truly force direct API billing/auth or normalize legacy `openai-codex/*` defaults through the existing migration path first.

## Reproduction Assessment

Yes for the planner-level behavior: current main clearly accepts and persists explicit subagent model overrides, and the PR body includes patched terminal output showing the new forbidden/ok cases. I did not establish a full installed-gateway before/after reproduction in this read-only review.

## Solution Assessment

No. The proposed prefix-based rejection is broader than the current OpenAI/Codex routing contract, because `openai/*` is the canonical model ref for Codex-runtime OpenAI agent turns unless config explicitly makes it a direct route.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

- **[P2] Allow canonical OpenAI Codex refs:** `src/agents/subagent-spawn-plan.ts:51`

Please narrow the guard to truly direct API cases or normalize the legacy default before comparing providers.
