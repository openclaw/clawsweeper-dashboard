## Summary

Keep open: current main still does not carry committed same-turn message-tool delivery evidence into Feishu turn completion accounting, so a side-path-satisfied turn can still look like plain `replies=0` in the shared dispatch result/log surface.

## What This Changes

Request that Feishu/Lark inbound turn accounting distinguish true no-reply turns from turns where same-turn side-path message-tool delivery succeeded.

## Best Possible Solution

Add a small, versioned channel-turn accounting signal that distinguishes committed same-turn message-tool delivery from true no-reply completion without redefining dispatcher final-reply counts.

## Reproduction Assessment

Yes from source inspection, not a live Feishu run: current main can return `queuedFinal=false` and `counts.final=0` for message-tool-only source turns while committed messaging-tool delivery evidence is tracked separately in the agent runner.

## Solution Assessment

Unclear: distinguishing the states is the right direction, but a Feishu-only log tweak would be too narrow because the relevant accounting surface is shared channel/plugin turn-result semantics.
