## Summary

Keep this open. Current main still leaves the reported packaged first-run dependency gap: Telegram and the other named channel plugins own runtime dependencies locally, the root package does not provide them, and current packaging policy excludes or rejects bundled plugin node_modules and legacy staging.

## What This Changes

Bundled channel plugins should resolve their runtime dependencies after a fresh global OpenClaw install without requiring `openclaw doctor --fix`.

## Best Possible Solution

Define one consistent installable-package contract: either externalize these heavy channel plugins as normal downloadable plugin packages with package-local dependencies, or intentionally change the bundled package policy and add package acceptance coverage proving fresh Telegram activation works without prior dependency state.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: a fresh packaged install lacks both root and extension-local dependency entries for `grammy`, while the Telegram bundled channel load path still imports it at runtime.

## Solution Assessment

No. The reported `doctor --fix` workaround is not the best long-term fix because current main treats dependency staging as legacy cleanup; the durable solution should align bundled channel activation with the plugin dependency model.
