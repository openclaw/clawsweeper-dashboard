## Summary

Keep this open. Current main partially moved some related channels toward downloadable packages, but the confirmed Telegram path and the similar Slack path still ship as bundled plugins with plugin-local runtime dependencies that are not present in the root package and are not installed by postinstall.

## What This Changes

The issue asks for fresh global OpenClaw installs to resolve bundled channel plugin runtime dependencies, especially Telegram and possibly Discord, Nostr, and Slack, without requiring `openclaw doctor --fix`.

## Best Possible Solution

Choose one supported packaging contract for Telegram and Slack: make them downloadable plugin packages like other heavy channels, or deliberately add a release-tested bundled-dependency mechanism that does not rely on doctor repair after first run.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: packed OpenClaw excludes extension `node_modules`, Telegram and Slack still keep bare runtime imports in plugin-local packages, and postinstall does not install those dependencies; the final executable proof should be a fresh package activation smoke for Telegram and Slack.

## Solution Assessment

No current solution is complete. The doctor workaround is repair-only, and the partial Discord/Nostr externalization does not solve the confirmed Telegram path or the same Slack packaging shape.
