## Summary

Keep this open. Current main still has the same packaged dependency-resolution gap for heavy bundled channel plugins: the dependencies remain plugin-local, the npm package policy still excludes nested bundled-plugin node_modules, and the documented policy says heavy bundled functionality should move to normal plugin packages rather than rely on staging or postinstall repair.

## What This Changes

The issue requests that bundled Telegram, and likely Discord, Nostr, and Slack, resolve their runtime dependencies out of the box after a global OpenClaw install without requiring `openclaw doctor --fix`.

## Best Possible Solution

Choose one supported packaging contract for heavy optional channel plugins and cover it in package acceptance: either externalize them as normal ClawHub/npm plugins with package-local dependencies, or deliberately support a bundled dependency artifact model. First-run activation should not depend on doctor repair.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists from the current main manifests, package exclusions, postinstall policy, and Telegram runtime import graph; a fresh global package activation smoke for Telegram would be the strongest executable confirmation.

## Solution Assessment

No. The reported `doctor --fix` workaround is not the best long-term solution because current policy treats generated bundled dependency staging as legacy cleanup. The maintainable fix should align bundled channel activation with the documented plugin dependency model.
