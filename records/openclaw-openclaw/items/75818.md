## Summary

Keep this open. Current main has partially moved Discord and Nostr toward external installable packages, but Telegram and Slack still remain in the core package shape with plugin-local runtime dependencies and no clear first-run packaged activation proof that those dependencies resolve without repair.

## What This Changes

The issue reports that bundled Telegram and likely related channel plugins fail to resolve plugin-local runtime dependencies after a fresh global install.

## Best Possible Solution

Pick one supported package contract for Telegram and Slack: externalize them like Discord/Nostr or add a release-tested bundled-dependency artifact that works on first run without doctor repair.

## Reproduction Assessment

Yes. The release report gives concrete v2026.4.29 global-install errors, and current main still has a static reproduction path through packaged Telegram/Slack runtime imports plus omitted plugin dependency trees; the strongest final proof is a fresh tarball/global activation smoke.

## Solution Assessment

No current solution is complete. The doctor workaround is repair-only, and the Discord/Nostr externalization does not cover the confirmed Telegram failure or the same Slack packaging shape.
