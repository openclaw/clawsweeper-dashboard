## Summary

Keep this open. Current main still has the reported first-run packaging gap: bundled channel code can load bare plugin-local runtime dependencies, while the root package excludes nested extension node_modules and current docs/scripts explicitly keep plugin dependency installation out of runtime/postinstall paths.

## What This Changes

The issue requests that bundled channel plugins such as Telegram, Discord, Slack, and Nostr resolve their runtime dependencies out of the box after a global OpenClaw package install, without requiring `openclaw doctor --fix`.

## Best Possible Solution

Choose one supported package contract and cover it in release/package acceptance: either stop treating heavy optional channels as bundled runtime code and install them as normal downloadable plugins, or deliberately change package policy so bundled channel dependencies are present on first run without reintroducing legacy staging debris.

## Reproduction Assessment

Yes. The high-confidence static reproduction is a fresh packaged install where `dist/extensions/telegram` is present without extension-local node_modules and Telegram activation reaches a bare `grammy` import; an executable package-activation smoke would be the final confirmation.

## Solution Assessment

Unclear. The issue’s `doctor --fix` workaround is useful operationally, but the current docs and release checks say dependency repair/staging is legacy cleanup, so the maintainable fix needs a maintainer choice between downloadable channel plugins and a new explicit bundled-dependency artifact model.
