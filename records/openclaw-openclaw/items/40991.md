## Summary

Keep open: current main and the latest release still let Telegram getFile failures resolve to null, continue with no media attachment, and render fallback media text instead of an explicit unavailable-media state.

## What This Changes

The issue reports Telegram inbound videos degrading to a placeholder-only message when Telegram media resolution fails.

## Best Possible Solution

Land one current extensions/telegram fix that carries an explicit unavailable-media reason through resolution and body handling, rate-limits user warnings, preserves useful file_id metadata, covers media groups, and proves failed plus normal video paths.

## Reproduction Assessment

Yes, from source: make Telegram ctx.getFile() fail with a file-too-big error or exhaust retries for a video, and current main returns null from media resolution while body construction can still render fallback media text with allMedia empty. I did not run a live Telegram probe in this read-only pass.

## Solution Assessment

No merged solution exists yet. The maintainable fix should expose attachment-unavailable state without dropping useful fallback metadata or creating unbounded warning replies.
