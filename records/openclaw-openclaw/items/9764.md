## Summary

Keep open: current main and v2026.6.10 still wire Google Chat through service-account app auth and the `chat.bot` scope only, while the requested user OAuth credential path for outbound reactions, media upload, reaction typing, and email-alias direct-message lookup remains missing and security-sensitive.

## What This Changes

The issue requests optional Google Chat user OAuth credentials for outbound reactions, media uploads, reaction typing, and proactive direct messages while preserving service-account behavior.

## Best Possible Solution

Keep this open until the bundled Google Chat plugin has a maintainer-approved, secret-aware user OAuth credential model, scoped user OAuth routing for user-only operations, preserved service-account webhook/app-auth behavior, and documented fallback semantics.

## Reproduction Assessment

Yes at source level: current main has strict service-account config and chat.bot-only auth, while the requested userOAuth config and scoped user OAuth client do not exist. I did not run a live Google Workspace reproduction in this read-only review.

## Solution Assessment

No automated code fix is the best next step yet. The maintainable path needs maintainer/security agreement on OAuth credential shape, scopes, storage, redaction, fallback semantics, and setup/docs before implementation.
