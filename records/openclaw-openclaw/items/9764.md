## Summary

Keep open. Current main and v2026.6.5 still have Google Chat wired to service-account credentials and the single chat.bot scope, so the requested user OAuth credential model and scoped routing for reactions, media uploads, reaction typing, and email-alias DM lookup remain missing.

## What This Changes

The issue requests optional Google Chat user OAuth support alongside the existing service-account flow so reactions, media uploads, reaction typing, and proactive DMs can use user-level Google Chat scopes.

## Best Possible Solution

Add a secret-aware Google Chat plugin-owned user OAuth credential model with scoped client selection for user-auth-only operations, preserve service-account defaults for webhook/bot sends, and document/test fallback behavior when user OAuth is absent.

## Reproduction Assessment

Yes, at source level. Current main has no user OAuth schema or resolver branch, and outbound upload/reaction/direct-message helpers all share the hardcoded service-account chat.bot token path.

## Solution Assessment

Unclear as an implementation until maintainers choose the credential model and fallback policy. Optional user OAuth in the Google Chat plugin is directionally right, but schema sensitivity, scopes, token refresh, and upgrade behavior need deliberate review.
