- `impact:message-loss`: The missing auth path affects outbound reactions, media uploads, and proactive DM delivery paths that can fail under service-account auth.

## Summary

Keep open. Current main still has Google Chat wired to service-account credentials and the `chat.bot` auth path only, while reactions, upload-file, reaction typing, and email-alias DM lookup still lack user OAuth credential resolution and scoped routing.

## What This Changes

The issue requests optional Google Chat user OAuth credentials so reactions, media uploads, reaction typing, and proactive DM lookup can use user-scoped Google Chat APIs while service-account auth remains available.

## Best Possible Solution

Add plugin-owned, secret-aware Google Chat user OAuth support with scoped per-operation clients, service-account webhook/bot auth preserved, and documented fallback behavior when OAuth is absent.

## Reproduction Assessment

Yes, at source level. Current main accepts service-account config only, has no refresh-token resolver, hardcodes the `chat.bot` auth path, and routes the affected operations through that same account token path.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is a Google Chat plugin-owned OAuth credential model with scoped clients after maintainer decisions on scopes, consent, fallback warnings, and defaults.
