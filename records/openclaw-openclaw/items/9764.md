## Summary

Keep open. Current main and the latest release still only expose service-account Google Chat credentials and a `chat.bot` token path, so the requested user OAuth config and scoped routing for reactions, attachment uploads, reaction typing, and email-alias DMs remains missing.

## What This Changes

The issue requests optional Google Chat user OAuth credentials alongside service-account auth for operations that require user-level scopes.

## Best Possible Solution

Implement this inside the Google Chat plugin as a secret-aware user OAuth credential model with scoped clients for reactions, attachment uploads, reaction typing, and email-alias DM lookup, while preserving service-account webhook verification and bot-level sends.

## Reproduction Assessment

Yes, at source level. Current main accepts only service-account credential config, hardcodes `chat.bot` auth, and routes upload/reaction/direct-message lookup through that same account token path.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is plugin-owned user OAuth with maintainer decisions on scopes, consent, fallback warnings, and default behavior.
