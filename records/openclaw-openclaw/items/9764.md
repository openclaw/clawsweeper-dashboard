## Summary

Keep open. Current main still has Google Chat user OAuth missing in config, setup, credential resolution, and auth routing, while recent discussion confirms the outbound reactions/media/proactive-DM gap is still wanted and separate from the inbound reaction issue.

## What This Changes

Add optional Google Chat user OAuth credentials for outbound reactions, media uploads, reaction typing, and proactive DM/email-alias lookup while retaining service-account auth.

## Best Possible Solution

Implement this inside the Google Chat plugin as an operation-scoped, secret-aware user OAuth path, preserving service-account auth for webhook verification and existing bot sends.

## Reproduction Assessment

Yes for source-level reproduction: current main has strict service-account-only config/auth and routes the affected operations through the same chat.bot token path. A live Google Chat 403 reproduction was not rerun because no Google Chat credentials were present in the redacted env/profile check.

## Solution Assessment

Unclear pending maintainer/security design review, but the direction is right: this belongs in the bundled Google Chat plugin with scoped OAuth clients and graceful fallback, not as a broad core migration or a separate ClawHub-only item.
