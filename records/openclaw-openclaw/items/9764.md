## Summary

Keep open. Current main still lacks Google Chat user OAuth config, credential resolution, setup, and auth routing, while the discussion confirms this outbound reactions/media/proactive-DM gap is still wanted and separate from the inbound reaction issue.

## What This Changes

The issue requests optional user OAuth support for Google Chat reactions, media uploads, and proactive DMs while keeping service-account auth for existing bot/webhook behavior.

## Best Possible Solution

Implement this in the Google Chat plugin as operation-scoped, secret-aware user OAuth support for reactions, uploads, reaction typing, and user/DM lookup, while preserving service-account auth for webhook verification and bot-level sends.

## Reproduction Assessment

Yes, for source-level reproduction: current main has strict service-account-only config/auth and routes the affected upload, reaction, and direct-message lookup operations through the same chat.bot token helper. A live Google Chat API repro was not run because no Google Chat credential markers were present in the redacted env/profile check.

## Solution Assessment

Unclear pending maintainer/security design, but the direction is right: the maintainable fix belongs in the bundled Google Chat plugin with scoped OAuth clients and graceful fallback, not as a broad core migration or as a ClawHub-only item.
