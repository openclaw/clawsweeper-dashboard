## Summary

Keep this issue open: current main still lacks Google Chat user OAuth config and scoped auth routing, so reactions, uploads, and email-alias DM lookup still use the service-account path.

## What This Changes

Add optional Google Chat user OAuth credentials for reactions, media uploads, and proactive DM lookup while retaining service-account webhook and bot auth.

## Best Possible Solution

Implement this inside the bundled Google Chat plugin with a secret-aware user OAuth credential model, scoped clients for user-only operations, service-account auth retained for webhook/bot flows, docs, tests, and live validation.

## Reproduction Assessment

Yes, at source level: current main exposes upload/reaction/direct-message paths, but the access token comes from the service-account `chat.bot` scope. I did not run a live Google Workspace API scenario.

## Solution Assessment

Unclear as an implementation because no fix is present. The requested direction is plausible, but the maintainable path is a plugin-owned OAuth model and scoped routing rather than treating inbound reaction work in #73681 as a substitute.
