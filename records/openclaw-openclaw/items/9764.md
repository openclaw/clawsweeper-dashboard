## Summary

Keep open. Current main and the latest release still use service-account Google Chat auth with the `chat.bot` scope, while the requested outbound reactions, media upload, reaction typing, and email-alias DM behavior need a new user OAuth credential and routing model.

## What This Changes

Request optional Google Chat user OAuth support for reactions, media uploads, reaction typing, and proactive DMs alongside existing service-account auth.

## Best Possible Solution

Implement the feature inside the Google Chat plugin as a secret-aware user OAuth credential model with scoped clients for reactions, attachment uploads, reaction typing, and email-alias DM lookup, while preserving service-account webhook verification and bot-level sends.

## Reproduction Assessment

Yes, at source level. Current main accepts only service-account credential config, hardcodes `chat.bot` auth for all API calls, and routes upload/reaction/direct-message lookup through that token path.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is a Google Chat plugin-owned OAuth model with maintainer decisions on scopes, consent, fallback warnings, and default behavior.
