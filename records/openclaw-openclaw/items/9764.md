## Summary

Keep open. Current main still lacks Google Chat user OAuth config, refresh-token credential resolution, and scoped user-auth routing, so reactions, attachment uploads, reaction typing, and email-alias DM lookup cannot be enabled from configuration.

## What This Changes

Add optional Google Chat user OAuth support for reactions, media uploads, reaction typing, and proactive DM lookup while retaining service-account webhook and bot-send behavior.

## Best Possible Solution

Add plugin-owned, secret-aware Google Chat user OAuth support with scoped per-operation clients while preserving service-account webhook verification and bot sends, plus documented fallback behavior when OAuth is absent.

## Reproduction Assessment

Yes, at source level. Current main has no accepted userOAuth config, no refresh-token credential resolver, service-account-only credential validation, and runtime auth remains fixed to chat.bot, so the requested behavior cannot be enabled without code changes.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is a Google Chat plugin-owned OAuth credential model with scoped clients after maintainer decisions on scopes, consent, fallback warnings, and defaults.
