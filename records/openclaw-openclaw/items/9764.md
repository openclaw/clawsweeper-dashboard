## Summary

Keep open. Current main still routes Google Chat through service-account credentials and the `chat.bot` scope only, so the requested outbound reactions, media upload, reaction typing, and email-alias DM path remain missing.

## What This Changes

The request asks the Google Chat plugin to add optional user OAuth credentials for operations that cannot run with service-account auth.

## Best Possible Solution

Implement this inside the bundled Google Chat plugin as a strict, secret-aware user OAuth path with operation-scoped clients, service-account preservation for webhook/app flows, graceful fallback warnings, and docs.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the current schema rejects `userOAuth`, account resolution only produces service-account credentials, and all affected API helpers use the same `chat.bot` token path.

## Solution Assessment

Unclear pending maintainer/security design review, but the direction is right: this belongs in the Google Chat plugin with scoped OAuth routing rather than a broad core migration or a ClawHub-only item.
