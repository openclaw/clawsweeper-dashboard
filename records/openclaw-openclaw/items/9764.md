## Summary

Keep open. Current main and v2026.5.22 still keep Google Chat on service-account credential resolution and `chat.bot` auth, while the requested user OAuth credential model and per-operation routing for reactions, attachment upload, reaction typing, and email-alias DMs is still missing.

## What This Changes

The issue requests optional Google Chat user OAuth credentials for outbound reactions, media uploads, reaction typing, and proactive DM/email-alias lookup while preserving service-account webhook behavior.

## Best Possible Solution

Implement this inside the Google Chat plugin as a secret-aware user OAuth credential model with scoped clients for reactions, attachment uploads, reaction typing, and email-alias DM lookup, while preserving service-account webhook verification and bot-level sends.

## Reproduction Assessment

Yes, at source level. Current main accepts only service-account credential config, hardcodes `chat.bot` auth, and routes upload/reaction/direct-message lookup through that same token path.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is a Google Chat plugin-owned OAuth model with maintainer decisions on scopes, consent, fallback warnings, and default behavior.
