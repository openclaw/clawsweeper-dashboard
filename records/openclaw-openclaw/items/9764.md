## Summary

Keep open: current main and latest release still only support Google Chat service-account auth with the `chat.bot` scope, while the requested user-OAuth credential model and scoped routing for reactions, attachment upload, reaction typing, and email-alias DM lookup are still missing.

## What This Changes

The issue requests optional Google Chat user OAuth credentials alongside service-account auth so reactions, media uploads, reaction typing, and proactive DMs can use user-level scopes.

## Best Possible Solution

Keep this as the canonical Google Chat user-OAuth design item, then implement a plugin-owned secret-aware OAuth credential model and explicit scoped routing for reactions, uploads, reaction typing, and email-alias DM lookup after security and product approval.

## Reproduction Assessment

Yes, at source level. Current main has no user-OAuth config/schema/secret contract and all Google Chat API calls still use the single service-account `chat.bot` token path.

No automatic fix is ready. The requested direction is maintainable only after maintainers approve the credential model, scopes, consent/security posture, sender attribution, and fallback semantics.
