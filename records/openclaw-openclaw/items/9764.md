## Summary

Keep open. Current main and the latest release still keep Google Chat on service-account credentials and the `chat.bot` scope only, so the requested user OAuth config, refresh-token secret handling, and per-operation routing for reactions, uploads, reaction typing, and email-alias DMs remain missing.

## What This Changes

The issue requests optional Google Chat user OAuth support so reactions, media uploads, reaction typing, and proactive DMs can use user-scoped credentials alongside service-account auth.

## Best Possible Solution

Implement this inside the Google Chat plugin as a strict, secret-aware user OAuth credential model with scoped clients for reactions, attachment uploads, reaction typing, and email-alias DM lookup, while preserving service-account webhook verification and bot-level sends.

## Reproduction Assessment

Yes, at source level. Current main accepts only service-account credential config, hardcodes `chat.bot` auth, and routes upload/reaction/direct-message lookup through that same token path.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is a Google Chat plugin-owned OAuth model with maintainer decisions on scopes, consent, fallback warnings, and default behavior.
