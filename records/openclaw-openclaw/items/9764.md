## Summary

Keep open: current main still does not provide Google Chat user OAuth configuration or user-scoped auth routing for outbound reactions, attachment uploads, reaction typing, or email-alias DM lookup; the issue remains a valid feature request that needs maintainer-owned design and security review.

## What This Changes

Request optional Google Chat user OAuth credentials alongside service-account auth so reactions, media uploads, and proactive DMs can use user-level scopes.

## Best Possible Solution

Implement the feature inside the bundled Google Chat plugin with a strict, secret-aware optional user OAuth credential model, scoped user-auth clients for user-only operations, service-account auth retained for bot/webhook flows, docs, tests, and live Workspace validation.

## Reproduction Assessment

Yes, at source level: the strict config schema has no user OAuth shape, credential resolution is service-account-only, and reactions/uploads/DM lookup all route through the bot auth path. I did not run a live Google Workspace reproduction in this read-only review.

## Solution Assessment

Yes as a product direction, but no implementation exists on current main. The maintainable path is plugin-owned user OAuth configuration and scoped-client routing, not treating the inbound reaction or threading issues as substitutes.
