## Summary

Keep open. Current main still has the bundled Google Chat plugin wired to service-account credentials and the `chat.bot` token path only, while the requested outbound reactions, media upload, and email-alias DM behavior needs a new user OAuth credential and routing model.

## What This Changes

Add optional user OAuth support to the bundled Google Chat plugin for reactions, media uploads, reaction typing, and proactive DM/email-alias lookup while preserving service-account bot auth.

## Best Possible Solution

Implement operation-scoped, secret-aware user OAuth inside the Google Chat plugin, with clear fallback warnings when OAuth is absent and service-account auth retained for webhook verification and bot-level sends.

## Reproduction Assessment

Yes, at source level: current main rejects the proposed OAuth config shape and routes uploads, reactions, and DM lookup through the service-account `chat.bot` token helper. A live Google Chat credential repro was not needed to confirm the missing repository support.

## Solution Assessment

Unclear pending maintainer/security design, but the direction is right: the fix belongs in the bundled Google Chat plugin as scoped user OAuth routing rather than a broad core migration or a ClawHub-only item.
