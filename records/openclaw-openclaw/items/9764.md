## Summary

Keep open: current main still has service-account-only Google Chat credential resolution and chat.bot-scoped access tokens, so the requested user OAuth path for reactions, uploads, and email-alias DM lookup remains missing.

## What This Changes

Add optional user OAuth credentials and scoped routing to the Google Chat plugin for outbound reactions, attachment uploads, reaction typing, and proactive DM lookup while retaining service-account webhook and bot flows.

## Best Possible Solution

Implement this inside the Google Chat plugin with a secret-aware user OAuth credential model, scoped clients for user-only operations, service-account auth retained for webhook and bot flows, docs, tests, and live Workspace validation.

## Reproduction Assessment

Yes, at source level: current main routes Google Chat upload, reaction, and users/<email> direct-message lookup through getGoogleChatAccessToken, which is fixed to the chat.bot service-account scope. I did not run a live Workspace scenario in this read-only pass.

## Solution Assessment

Yes as a product direction, but no implementation exists yet. The narrow maintainable path is a plugin-owned OAuth credential model with scoped routing and explicit fallback semantics, not treating inbound reaction work or an external Drive-link workaround as a substitute.
