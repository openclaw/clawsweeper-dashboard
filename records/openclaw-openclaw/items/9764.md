## Summary

Keep open. Current main and the latest release still only model Google Chat service-account auth for the bundled Google Chat plugin, while the requested user OAuth credential surface and scoped routing for reactions, media uploads, and email-alias DMs remain absent; the external Drive-upload workaround is useful prior art but does not implement this repository's requested auth path.

## What This Changes

Add optional Google Chat user OAuth credentials and scoped operation routing for reactions, media uploads, and email-alias proactive DMs while preserving service-account webhook and bot-message auth.

## Best Possible Solution

Add plugin-owned, secret-aware Google Chat user OAuth support with scoped per-operation clients, service-account webhook/bot auth preserved, and documented fallback behavior when OAuth is absent.

## Reproduction Assessment

Yes, at source level. Current main has no accepted userOAuth config, no refresh-token resolver, service-account credential validation only, and runtime auth fixed to chat.bot; I did not run a live Google Workspace repro.

## Solution Assessment

Yes directionally, but not as an automated cleanup fix. The maintainable path is a Google Chat plugin-owned OAuth credential model with scoped clients after maintainer decisions on scopes, consent, fallback warnings, and defaults.
