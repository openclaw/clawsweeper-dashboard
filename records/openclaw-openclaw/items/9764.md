## Summary

Keep open. Current main still only accepts Google Chat service-account auth and routes uploads, reactions, and DM lookup through the `chat.bot` token path; the requested user OAuth credential model and fallback behavior are not implemented.

## What This Changes

Request optional user OAuth support for Google Chat reactions, media uploads, reaction typing, and proactive DMs while retaining service-account auth for existing bot/webhook flows.

## Best Possible Solution

Add operation-scoped user OAuth inside the bundled Google Chat plugin, keeping service-account auth for webhook verification and bot-level sends while documenting and warning on unsupported operations when OAuth is absent.

## Reproduction Assessment

Not applicable as a bug reproduction, because this is a feature request. Source inspection is still high-confidence: current main has no user OAuth config path and the requested operations use the service-account `chat.bot` token helper.

## Solution Assessment

Unclear pending maintainer/security design, but the direction belongs in the bundled Google Chat plugin rather than a broad core migration or a ClawHub-only item.
