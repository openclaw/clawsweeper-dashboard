## Summary

Keep open: current main and v2026.6.10 still use Google Chat service-account app auth with the `chat.bot` scope only, so the requested user OAuth credential path for outbound reactions, native media upload, reaction typing, and email-alias DM lookup remains missing and security-sensitive.

## What This Changes

Request optional Google Chat user OAuth credentials and routing for reactions, media uploads, reaction typing, and proactive DMs while preserving service-account webhook and app-auth behavior.

## Best Possible Solution

Keep this as the canonical Google Chat user-OAuth tracker and implement it in the bundled plugin only after a security-reviewed SecretRef-aware credential model and scoped app-versus-user auth routing plan are approved.

## Reproduction Assessment

Yes, at source level: current main has strict service-account config and chat.bot-only auth, while the requested userOAuth config and scoped user OAuth client do not exist. I did not run a live Google Workspace reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation shape: optional user OAuth is the likely direction for native reactions and uploads, but maintainers still need to approve credential shape, scopes, fallback behavior, and setup/docs before code work starts.
