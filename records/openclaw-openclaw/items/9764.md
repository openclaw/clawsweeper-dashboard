## Summary

Keep open: current main and the latest release still only wire Google Chat through service-account/app auth, while the requested user OAuth credential path for reactions, media upload, reaction typing, and email-alias DM lookup remains missing and security-sensitive.

## What This Changes

The issue requests optional Google Chat user OAuth credentials alongside service-account auth for reactions, media uploads, reaction typing, and proactive DM lookup.

## Best Possible Solution

Keep this open until the bundled Google Chat plugin has a secret-aware user OAuth credential path, scoped OAuth client routing for user-only operations, service-account webhook/app-auth preservation, and documented fallback behavior.

## Reproduction Assessment

Yes at source level: current main has strict service-account-only config and a chat.bot-only auth path, while the userOAuth config and scoped user OAuth client requested here do not exist. I did not run a live Google Workspace reproduction in this read-only review.

## Solution Assessment

No automated code fix is the best next step yet. The maintainable path needs maintainer/security agreement on the OAuth credential model, scopes, storage, redaction, fallback semantics, and setup/docs before implementation.
