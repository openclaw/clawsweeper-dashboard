## Summary

Keep open: current main and the latest release still fail fast on a mixed strict OpenAI-compatible image response, and this PR is a focused candidate fix with sufficient parser-boundary proof. The remaining blocker is maintainer acceptance of the SDK-exported strict-mode semantic change and coordination with adjacent proxy-response PRs.

## What This Changes

The PR changes `parseOpenAiCompatibleImageResponse()` to skip invalid per-entry `b64_json` values in strict mode, throw only when no non-empty `data[]` entry parses, and adds regression tests.

## Best Possible Solution

Land this narrow parser fix after maintainers accept partial-success strict parsing, while keeping URL/object/base64url proxy-response work on adjacent review paths.

## Reproduction Assessment

Yes at source level. Current main and `v2026.6.8` throw inside the strict parser loop on the first invalid entry, and the PR body includes before/after terminal output for a mixed valid/invalid batch.

## Solution Assessment

Yes for the mixed `b64_json` batch behavior: deferring the strict error until after the loop is the narrow shared-helper fix. The remaining question is maintainer acceptance of the exported strict-mode compatibility change, not a code-shape defect.
