## Summary

Keep open: the patch is a narrow, likely-correct fix for mixed OpenAI-compatible `b64_json` arrays and is not implemented on current main, but this external PR still lacks real behavior proof beyond tests and an unrelated health check.

## What This Changes

The PR changes `parseOpenAiCompatibleImageResponse()` to skip invalid entries in strict mode, throw only when a non-empty batch yields no parsed images, and adds regression tests for mixed and all-invalid batches.

## Best Possible Solution

Land the narrow mixed-entry parser fix after real behavior proof or explicit maintainer acceptance of parser-boundary validation, while keeping object-format, URL-safe base64, and URL-backed response work on their separate PRs.

## Reproduction Assessment

Yes at source level: current main throws in strict mode when a `data[]` entry fails parsing, so a mixed batch cannot return its valid images. I did not run artifact-creating tests or a live provider flow during this read-only review.

## Solution Assessment

Yes for mixed `b64_json` arrays: deferring the strict error until after the loop is the narrowest maintainable parser fix. URL-only responses and object-format data are adjacent work already represented by other PRs.
