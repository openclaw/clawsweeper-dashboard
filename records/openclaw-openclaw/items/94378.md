## Summary

Keep open for maintainer review: the patch is a focused, source-reproducible fix with sufficient terminal proof, but it changes an SDK-exported strict parser from fail-fast to partial success, so maintainers should explicitly accept that compatibility behavior before merge.

## What This Changes

The PR changes the OpenAI-compatible image response parser and tests so strict parsing skips invalid `data[]` entries while still throwing when a non-empty response yields no valid images.

## Best Possible Solution

Land the focused parser change after maintainers accept partial-success strict parsing as SDK behavior, while leaving URL, object-format, and base64url proxy fixes on their adjacent review paths.

## Solution Assessment

Yes for the mixed `b64_json` batch behavior: deferring the strict error until after the loop is the narrow shared-helper fix. The remaining question is maintainer acceptance of the SDK-exported strict-mode compatibility change.
