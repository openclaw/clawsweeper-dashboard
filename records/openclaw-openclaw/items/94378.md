The PR changes strict OpenAI-compatible image response parsing to skip invalid `data[]` entries when at least one valid base64 image is present, with regression coverage for mixed and all-invalid batches.

## Best Possible Solution

Land the focused parser change after maintainers accept partial-success strict parsing as SDK behavior, while leaving URL, object-format, and base64url proxy fixes on their adjacent review paths.

## Solution Assessment
