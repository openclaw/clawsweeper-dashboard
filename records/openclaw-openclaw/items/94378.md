## Summary

Keep open: this is a focused parser fix with source-level reproduction and sufficient terminal proof, but it changes exported plugin SDK strict-mode semantics and should get maintainer acceptance while adjacent URL-response work remains separate.

## What This Changes

The PR changes the OpenAI-compatible image response parser to skip invalid entries in strict mode, throw only when a non-empty batch yields no parsed images, and adds parser regression tests.

## Best Possible Solution

Land the narrow mixed-entry parser behavior only after maintainers accept the exported strict-mode semantic change, while keeping URL-backed response handling on the separate guarded-download PR.

## Reproduction Assessment

Yes at source level: current main throws during strict per-entry parsing when any entry is invalid, so a mixed response cannot return its valid images. I did not run artifact-creating tests or a live proxy flow in this read-only review.

## Solution Assessment

Yes for the mixed b64_json array behavior: deferring the strict error until after the loop is the narrowest shared-helper fix I found. It is not the complete NewAPI URL-only response fix, which remains on https://github.com/openclaw/openclaw/pull/81443.
