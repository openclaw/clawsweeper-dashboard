## Summary

Keep open for maintainer review: the patch is a narrow, well-tested fix for mixed valid/invalid OpenAI-compatible image entries, but it changes strict-mode behavior for an SDK-exported parser helper and should be accepted explicitly before merge.

## What This Changes

The PR changes the OpenAI-compatible image response parser to skip invalid entries in strict mode and throw only when a non-empty response yields no parsed images, with focused regression tests.

## Best Possible Solution

Land the narrow mixed-entry parser behavior only after maintainers accept partial-success strict parsing, and keep URL-backed response handling on https://github.com/openclaw/openclaw/pull/81443.

## Reproduction Assessment

Yes. Current main throws inside the strict per-entry parser loop when any entry is invalid, and the PR body includes before/after terminal output showing the mixed batch returning valid images after the patch.

## Solution Assessment

Yes for the mixed b64_json batch behavior: deferring the strict error until after the loop is the narrowest shared-helper fix found. It is not the complete URL-only proxy response fix, which remains on https://github.com/openclaw/openclaw/pull/81443.
