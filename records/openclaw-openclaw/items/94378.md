## Summary

Keep open: current main still fail-fasts in strict parsing when one OpenAI-compatible image response entry is invalid, and this PR is a focused, proven fix for that mixed-batch loss. Merge still needs maintainer acceptance because the changed strict-mode behavior is exported through the plugin SDK and overlaps, but does not replace, the broader NewAPI proxy-response work.

## What This Changes

The PR changes `parseOpenAiCompatibleImageResponse()` to skip invalid entries in strict mode and adds regression tests for mixed-valid and all-invalid image response batches.

## Best Possible Solution

Land the narrow mixed-entry parser behavior only after maintainers accept partial-success strict parsing, while keeping URL/object/base64url proxy-response variants on their separate PRs.

## Reproduction Assessment

Yes. Current main source shows the strict parser throws inside the entry loop on the first invalid entry, and the PR body includes before/after terminal output for a mixed valid/invalid batch.

## Solution Assessment

Yes for the mixed `b64_json` batch behavior: deferring the strict error until after the loop is the narrowest shared-helper fix found. It is not the complete NewAPI proxy fix, because URL-only, object-format, and base64url variants remain on related PRs.
