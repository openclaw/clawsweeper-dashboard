## Summary

Keep this PR open for maintainer review: the patch is focused and appears correct for mixed valid/invalid OpenAI-compatible image batches, but it intentionally changes SDK-exported strict-mode parser behavior from fail-fast to partial success.

## What This Changes

The PR changes `parseOpenAiCompatibleImageResponse()` to skip invalid entries in strict mode unless every non-empty `data[]` entry fails, and adds parser regression tests.

## Best Possible Solution

Land this focused parser change after maintainers explicitly accept partial-success strict parsing as the plugin-facing behavior, while keeping adjacent proxy response shapes on their own review paths.

## Reproduction Assessment

## Solution Assessment

Yes for the mixed `b64_json` batch behavior: deferring the strict error until after the loop is the narrow shared-helper fix. The unresolved question is maintainer acceptance of the SDK-exported strict-mode compatibility change, not a concrete patch defect.
