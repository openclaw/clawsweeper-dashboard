Keep open: the patch is a narrow and plausible fix for the linked OpenAI-compatible proxy parser bug, with no blocking code findings, but the external PR still needs real behavior proof beyond tests before merge.

## What This Changes

The branch changes shared OpenAI-compatible image response parsing so strict mode skips invalid `data[]` entries when at least one valid `b64_json` image remains, with regression tests for mixed and all-invalid batches.

## Best Possible Solution

Land the narrow parser change after real runtime or proxy proof is added, while leaving URL-response download handling on its separate guarded-download track.

## Reproduction Assessment

Yes at source level: current main passes strict malformed-response handling into the shared parser, and a mixed `data[]` with an invalid entry before or between valid `b64_json` entries throws before returning the valid images. I did not run a live proxy reproduction in this read-only review.

## Solution Assessment

Yes, this is the right narrow layer for mixed invalid `b64_json` entries because the shared parser already owns strict-versus-lenient parsing and callers already distinguish empty responses from malformed non-empty responses. URL-only image responses remain a separate, security-sensitive feature path.
