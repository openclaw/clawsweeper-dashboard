## Summary

Keep open: the patch is narrow and likely correct, but it changes strict-mode behavior for an SDK-exported image parser, so maintainers should explicitly accept partial-success semantics before merge.

## What This Changes

The PR changes OpenAI-compatible image response parsing so mixed batches skip invalid entries and return valid images while still throwing when every returned entry is invalid.

## Best Possible Solution

Land the narrow mixed-entry parser behavior only after maintainers accept partial-success strict parsing, and keep URL-backed response handling on https://github.com/openclaw/openclaw/pull/81443.

## Reproduction Assessment

Yes at source level: current main throws inside the strict per-entry parser loop when any entry is invalid. The PR body also includes before/after terminal output showing the mixed batch returning valid images after the patch.

## Solution Assessment

Yes for the mixed b64_json batch behavior: deferring the strict error until after the loop is the narrowest shared-helper fix found. It is not the complete URL-only proxy response fix, which remains on https://github.com/openclaw/openclaw/pull/81443.
