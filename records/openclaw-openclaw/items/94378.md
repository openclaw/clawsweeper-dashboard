## Summary

Keep open: current main and the latest release still fail mixed valid/invalid strict image responses, and this is a narrow parser fix with no code-level blocking findings. It still needs real behavior proof before merge, and URL-only OpenAI-compatible responses remain separate work.

## What This Changes

The PR changes the shared OpenAI-compatible image response parser to skip invalid data entries in strict mode while still throwing when every non-empty entry is invalid, with regression tests.

## Best Possible Solution

Land the narrow mixed-entry parser fix after real behavior proof, and keep URL-only image download handling on https://github.com/openclaw/openclaw/pull/81443.

## Reproduction Assessment

Yes at source level: current main throws in strict mode when an invalid entry appears before later valid b64_json data. I did not run a live proxy reproduction in this read-only review.

## Solution Assessment

Yes for mixed b64_json batches: deferring the strict error until after the parser sees whether any entry was valid is the narrowest maintainable fix. URL-only response downloads should remain in the separate guarded-download PR.
