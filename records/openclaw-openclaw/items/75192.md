## Summary

Keep open. Current main has the merged WebChat New Chat plumbing, but it still creates a dashboard session for the currently active session's agent and does not expose a UI path to pick any configured agent before creation.

## What This Changes

Add WebChat UI support for creating a new chat session with a selected configured agent instead of only the currently active/default agent.

## Best Possible Solution

Add a focused WebChat selected-agent New Chat flow that reuses agents.list metadata and sessions.create({ agentId }), switches to the returned session key with the existing session-switch helper, and leaves broader sidebar UX work to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open WebChat with only the current/default agent session active, and use New session; current main creates another session for the active session's agent and offers no control that passes a different configured agentId.

## Solution Assessment

Yes. The narrowest maintainable solution is UI-only over the existing agents.list and sessions.create agentId contract; no new backend endpoint is indicated by current source.
