## Summary

Keep open: current main still lacks a visible WebChat agent selector for New session creation, while the backend and frontend helper already accept an explicit `agentId`.

## What This Changes

Add a Control UI/WebChat flow to create a new chat session for a selected configured agent instead of only the active session's agent.

## Best Possible Solution

Add a narrow WebChat New session agent picker that lists configured agents, calls the existing `sessions.create({ agentId })` flow, switches to the returned session, and leaves the broader sidebar redesign to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open WebChat on one agent session, and use the New session control; source inspection shows it creates with the active session's agent and provides no selector for another configured agent.

## Solution Assessment

Yes. The best path is a UI-only selector over existing agent metadata and the existing `sessions.create` transport; current source does not show a need for a new backend endpoint.
