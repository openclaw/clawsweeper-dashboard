## Summary

Keep open. Current main includes the merged New Chat create/switch work from #73725, but it still creates a new dashboard session only for the active session's agent and does not expose a WebChat control that lets the user choose another configured agent at creation time.

## What This Changes

Add a WebChat New Chat flow that lets users choose one of the configured agents and creates and switches to a session for that agent.

## Best Possible Solution

Add an agent-aware New Chat picker/menu in Control UI WebChat that reuses existing agents.list metadata and sessions.create({ agentId }), switches with the existing session-switch path, and avoids a new gateway endpoint or broad sidebar redesign.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure multiple agents, open WebChat, and inspect the plus/New session path; it resolves agentId from the active session key and has no control that passes an arbitrary configured agent id to sessions.create.

## Solution Assessment

Yes. The narrowest maintainable solution is UI-only because the gateway schema, server method, and UI controller already support sessions.create.agentId; the missing piece is exposing that choice in WebChat.
