## Summary

Keep open: the prior ClawSweeper conclusion still holds on current main. WebChat/Control UI has a New session button, but it creates another dashboard session for the active session's agent and does not expose a configured-agent picker; #49828 is adjacent sidebar UX work, not a replacement for this narrower create-with-agent request.

## What This Changes

Request to let WebChat/Control UI start a new chat session for a user-selected configured agent.

## Best Possible Solution

Add a focused Control UI/WebChat New Chat agent picker that lists configured agents from existing metadata, calls sessions.create with the selected agentId, switches to the returned dashboard session, and leaves the broader sidebar redesign to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open Control UI chat on one active agent session, and click New session; source and existing tests show the flow creates a session for the active session's agent with no selector for another configured agent.

## Solution Assessment

Yes. A narrow UI selector over existing agent metadata and the existing sessions.create({ agentId }) contract is the maintainable fix; no new Gateway endpoint is indicated.
