## Summary

Keep open: current main exposes multiple configured agents and explicit `sessions.create.agentId`, but the Control UI New session button still has no agent-selection input and `createChatSession` derives the new session's agent from the active session key.

## What This Changes

Request to let Control UI WebChat users create a new chat session for a selected configured agent instead of only the current/default agent.

## Best Possible Solution

Add a focused Control UI New Chat agent picker that lists configured agents, calls the existing session creation helper with the selected `agentId`, switches to the returned session, and leaves the broader sidebar redesign to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open Control UI Chat on one agent session, and press New session; current main routes through `createChatSession`, which derives `agentId` from the active session key and exposes no picker for another configured agent.

## Solution Assessment

Yes. The narrow maintainable fix is UI-only over the existing `agents.list` metadata and `sessions.create({ agentId })` contract; current source does not show a need for a new backend endpoint.
