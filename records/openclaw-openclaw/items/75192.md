## Summary

Keep open: current main can switch among existing agent sessions and create a New session for the active session's agent, but it still has no configured-agent picker for creating a new chat under a different agent; #49828 and #61423 are adjacent but do not replace this request.

## What This Changes

The issue asks for WebChat/Control UI to let users create a new chat session for a selected configured agent.

## Best Possible Solution

Add a focused Control UI/WebChat New Chat agent picker that lists configured agents, calls the existing sessions.create path with the selected agentId, switches to the returned dashboard session, and leaves the broader sidebar redesign to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open Control UI chat on one active agent session, and use New session; source and tests show the flow creates a dashboard session for the active session's agent with no selector for another configured agent.

## Solution Assessment

Yes. A narrow UI picker over existing agentsList metadata plus the existing sessions.create({ agentId }) contract is the maintainable fix; #61423's current-agent-only modal is not sufficient for this issue.
