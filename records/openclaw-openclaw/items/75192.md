## Summary

Keep open: current main still exposes New session only for the active session's agent, while the requested WebChat agent picker for creating a session with a different configured agent remains unimplemented.

## What This Changes

Add a WebChat/Control UI flow to create a new chat session for a selected configured agent.

## Best Possible Solution

Implement a focused Control UI/WebChat New Chat agent picker using existing configured-agent metadata and the existing sessions.create({ agentId }) contract.

## Reproduction Assessment

Yes. Source and existing tests show the New session button always creates a dashboard session for the active session's agent, with no UI path to select another configured agent.

## Solution Assessment

Yes. A narrow UI selector over state.agentsList plus createSessionAndRefresh/sessions.create({ agentId }) is the maintainable fix; no new Gateway endpoint is indicated.

Implement #75192 as a focused Control UI/WebChat feature. Add a New Chat agent picker that lists configured agents from existing state.agentsList metadata, lets the user choose an agent, calls createSessionAndRefresh/sessions.create with the selected agentId, switches to the returned dashboard session, and preserves existing draft/attachment/stale-response behavior.
