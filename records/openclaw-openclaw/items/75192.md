## Summary

Keep open. Current main still has the requested WebChat gap: the UI can list agents and switch existing sessions, and the Gateway can create agent-scoped sessions, but the visible New Session control still sends `/new` in the active session and offers no selected-agent creation flow. The adjacent WebChat create/switch PR #73725 is open and not merged, so it does not make this issue obsolete.

## What This Changes

The issue requests a WebChat UI flow for creating a new chat session with a selected configured agent and then switching to that session.

## Best Possible Solution

Layer a focused selected-agent picker on top of the generic WebChat session-create path: reuse `agents.list` for name/identity metadata, call `sessions.create` with the chosen `agentId`, switch to the returned key, refresh session/history/avatar/tool state, and avoid adding a new Gateway endpoint unless a concrete missing field appears.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure multiple agents, open Control UI chat, press the New Session control, and current main can only send `/new` for the active session; no visible UI path passes a chosen `agentId` to `sessions.create`.

## Solution Assessment

Yes, with sequencing. The maintainable solution is a UI-only addition over the existing `agents.list` and `sessions.create` contracts, preferably after or folded into #73725 rather than implemented as a separate backend API.
