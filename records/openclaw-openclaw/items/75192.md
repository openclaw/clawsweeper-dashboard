## Summary

Keep open. Current main still supports explicit `agentId` at the session API/helper layer, but the WebChat New session control has no selected-agent UI and still creates a session for the active session's agent.

## What This Changes

Add a WebChat UI flow for creating a new chat session with a selected configured agent.

## Best Possible Solution

Add a focused Control UI New session agent selector that lists configured agents, calls the existing session creation helper with the chosen `agentId`, switches to the returned session, and leaves the broader sidebar redesign independent.

## Reproduction Assessment

Yes. Configure multiple agents, open WebChat on one agent session, and press New session; current main routes through `createChatSession`, which derives `agentId` from the active session key and exposes no UI control for another configured agent.

## Solution Assessment

Yes. The narrow maintainable fix is a UI selector over existing `agents.list` metadata and `sessions.create({ agentId })`; current source does not show a need for a new backend endpoint.
