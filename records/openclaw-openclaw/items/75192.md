## Summary

Keep open. Current main has the WebChat New session plumbing and backend support for explicit `agentId`, but the visible New session control still has no selected-agent UI and creates another dashboard session for the active session's agent.

## What This Changes

Request to add a WebChat/Control UI flow for creating a new chat session with any configured agent selected by the user.

## Best Possible Solution

Add a focused Control UI WebChat New session flow that lists configured agents from existing agent metadata, calls the existing session creation helper with the chosen `agentId`, switches to the returned session key, and leaves the broader sidebar redesign to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open WebChat on one agent session, and press New session; current main routes through `createChatSession`, which derives `agentId` from the active session key and provides no UI control for another configured agent.

## Solution Assessment

Yes. The best fix is a narrow UI-only selector over the existing `agents.list` and `sessions.create({ agentId })` contract; current source does not indicate a new backend endpoint is needed.
