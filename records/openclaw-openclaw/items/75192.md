## Summary

Keep open: current main still creates a new WebChat dashboard session for the active session's agent, with no UI path to choose another configured agent. The adjacent #49828 sidebar request and merged #73725 New Chat work do not replace this narrower create-with-agent selector request.

## What This Changes

Add a WebChat/Control UI New Chat flow that lets users choose a configured agent when creating a new session.

## Best Possible Solution

Add a focused Control UI/WebChat New Chat agent picker over existing agents metadata, call sessions.create with the selected agentId, switch to the returned dashboard session, and leave broader sidebar redesign work to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open Control UI chat on one active agent session, and click New session; source and tests show it creates a new dashboard session for that active agent with no selector for another configured agent.

## Solution Assessment

Yes. A narrow UI selector using existing agents.list metadata and sessions.create({ agentId }) is the maintainable fix; no new Gateway endpoint is indicated.

## Repair Work Prompt

Implement a focused Control UI/WebChat New Chat agent picker for https://github.com/openclaw/openclaw/issues/75192. The observable gap is that multiple agents can be configured and existing sessions can be switched, but New session currently calls createSessionAndRefresh with resolveAgentIdFromSessionKey(previousSessionKey), so users cannot start a fresh dashboard session for a different configured agent. Keep the fix scoped to Control UI/WebChat: list configured agents from existing agents metadata, let the user choose an agent from the New Chat affordance, call sessions.create through the existing createSessionAndRefresh path with the selected agentId, switch to the returned dashboard session, preserve the existing active-run/session-loading safeguards, and do not add a new Gateway endpoint. Leave the broader sidebar redesign in #49828 alone. Add focused tests for the selected-agent path and the existing active-session default path, update UI tests for the New Chat control, and add a single-line user-facing CHANGELOG.md entry with appropriate attribution if a human contributor is credited. Do not change backend session semantics beyond any tiny type plumbing needed for the UI.
