## Summary

Keep open. Current main still has backend and frontend plumbing for `sessions.create({ agentId })`, but the WebChat New session control does not expose an agent choice and continues to derive the new session's agent from the active session key.

## What This Changes

Add a WebChat UI flow to create a new chat session with a selected configured agent instead of only creating another session for the currently active agent.

## Best Possible Solution

Add a focused Control UI WebChat New session picker that lists configured agents from existing agent metadata, creates the session through `sessions.create` with the chosen `agentId`, switches to the returned session key, and leaves the broader sidebar redesign to #49828.

## Reproduction Assessment

Yes. Configure multiple agents, open WebChat on one agent session, and press New session; current main routes through `createChatSession`, which derives `agentId` from the active session key and provides no UI control for another configured agent.

## Solution Assessment

Yes. A narrow UI-only selector over the existing `agents.list` metadata and `sessions.create({ agentId })` contract is the best fit; current source does not indicate a new backend endpoint is needed.
