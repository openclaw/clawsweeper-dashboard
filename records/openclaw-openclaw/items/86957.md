## Summary

Keep open: current main and v2026.6.9 still lack an immediate drain wakeup after worker-spooled Telegram updates, and the open closing PR is the active fix path but has not merged yet.

## What This Changes

Telegram inbound messages can reach the isolated ingress spool/queue but remain pending instead of being drained into the main agent session automatically.

## Best Possible Solution

Repair and land, or narrowly replace, https://github.com/openclaw/openclaw/pull/94301 with a Telegram-owned drain wakeup that preserves active-drain requests, then close this issue after merge.

## Reproduction Assessment

Yes, at source level: current main writes worker-spooled Telegram updates and only drains on startup or the timer, while the issue discussion documents pending unclaimed rows after later DMs. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes on direction, but not by starting a separate fix from this issue. The best path is to finish or replace the open closing PR so the existing Telegram drain path wakes immediately and handles active-drain wakeups safely.
