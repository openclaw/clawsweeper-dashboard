## Summary

Keep open: the PR addresses the linked Control UI bug, but the current patch regresses the documented and tested behavior that switching agents should use a saved dashboard session when one exists; the narrow fix is to exclude subagent/internal rows from automatic selection while preserving normal dashboard preference.

## What This Changes

The PR changes the Control UI chat session selector to hide inactive subagent sessions, keep the active subagent as a fallback option, and switch agent dropdown changes to the selected agent's main session.

## Best Possible Solution

Preserve the existing saved-dashboard preference for agent switches, skip subagent/internal runtime rows during automatic target selection, and fall back to the selected agent's main session only when no eligible normal session exists.

## Reproduction Assessment

Yes. Source inspection shows current main can choose a newer target-agent subagent by updatedAt, and the PR's changed line can be checked against the existing test that expects a saved dashboard session to remain preferred.

## Solution Assessment

No. The PR fixes the subagent autoselection by removing the entire recent-session preference, but the safer fix is to keep that preference for normal dashboard sessions and filter out subagent/internal rows.
