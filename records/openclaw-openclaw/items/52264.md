## Summary

Keep open: the PR addresses a real gateway state-dir bug, but the current patch still violates the legacy-env contract by rewriting `CLAWDBOT_STATE_DIR`, includes local generated Plugin SDK baseline files, and is conflicting with current `main`.

## What This Changes

The PR adds startup normalization for explicit state-dir environment overrides in gateway CLI and direct server startup, plus focused path/order tests and generated Plugin SDK baseline changes.

## Best Possible Solution

Land a cleaned fix that pins supported `OPENCLAW_STATE_DIR` before config/plugin discovery on both startup paths, preserves ignored legacy env behavior, removes generated local baselines, and keeps focused regression coverage.

## Solution Assessment
