## Summary

Keep this PR open: current main still does not pin explicit relative state-dir overrides at gateway startup, so the hardening remains useful, but the branch is not ready because it revives ignored legacy env behavior and carries local generated baseline artifacts.

## What This Changes

The PR adds startup normalization for explicit state-dir environment overrides in gateway CLI and direct server startup paths, plus focused path/startup tests and generated Plugin SDK baseline changes.

## Best Possible Solution

Land a cleaned gateway startup fix that pins supported `OPENCLAW_STATE_DIR` before config/plugin discovery, preserves ignored legacy env behavior, and leaves local generated artifacts out of the patch.

## Reproduction Assessment

Yes by source inspection and linked fixture context. Current main still resolves relative `OPENCLAW_STATE_DIR` at call time, while the standalone fixture reports the cwd-sensitive channel-registry failure path.

## Solution Assessment

No. Early state-dir pinning is the right narrow direction, but the branch should only affect supported `OPENCLAW_STATE_DIR`, remove local generated baselines, and be reconciled with current gateway startup code.
