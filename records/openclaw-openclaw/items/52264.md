## Summary

Keep open: current main still lacks early state-dir pinning, so the PR contains useful bug-fix work, but the branch should not merge while it revives legacy `CLAWDBOT_STATE_DIR` behavior and carries generated local-only Plugin SDK baseline churn.

## What This Changes

The branch adds a helper to normalize `OPENCLAW_STATE_DIR` and `CLAWDBOT_STATE_DIR` during gateway CLI and direct server startup, with focused path/order tests and generated Plugin SDK baseline edits.

## Best Possible Solution

Pin only the supported `OPENCLAW_STATE_DIR` before cwd-sensitive gateway startup work in both CLI and direct server paths, keep focused regression coverage, and drop legacy-env plus generated-baseline edits.

## Reproduction Assessment

Yes. ## Solution Assessment

No, not in the current branch shape. Early startup pinning is the right fix direction, but the branch should be narrowed to the supported env contract, have generated artifacts removed, and be rebased over current gateway startup code.
