## Summary

Keep open. The linked plugin-registry failure was fixed by https://github.com/openclaw/openclaw/pull/52262, but current main still does not pin relative `OPENCLAW_STATE_DIR` at gateway startup, so the useful hardening in this PR is not obsolete. The branch needs cleanup before merge because it adds a retired legacy-env path and commits ignored generated baseline files.

## What This Changes

The PR adds startup normalization for explicit state-directory environment overrides in gateway CLI and direct server startup, with focused path/startup-order tests and generated SDK baseline changes.

## Best Possible Solution

Land a rebased, narrow patch that pins supported `OPENCLAW_STATE_DIR` at gateway startup and keeps retired legacy env aliases and local generated baseline artifacts untouched.

## Reproduction Assessment

Yes, at source level. Current main still resolves relative `OPENCLAW_STATE_DIR` dynamically from the current cwd, and both gateway startup paths lack an early normalization hook; I did not run the external repro in this read-only review.

## Solution Assessment

No, not in the current branch shape. Startup pinning for `OPENCLAW_STATE_DIR` is a narrow maintainable fix, but the branch should drop `CLAWDBOT_STATE_DIR`, remove ignored generated baselines, and rebase onto current gateway startup code.
