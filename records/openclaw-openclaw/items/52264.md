## Summary

## What This Changes

This PR adds a state-dir env normalization helper, calls it from gateway CLI and direct server startup, and adds focused path/startup-order tests while modifying generated Plugin SDK baseline artifacts.

## Best Possible Solution

Pin only the supported `OPENCLAW_STATE_DIR` at the earliest gateway CLI and direct server startup boundary, keep legacy env aliases ignored, and remove generated baseline artifacts from the branch.
