## Summary

Keep open: the branch contains a useful gateway startup hardening change that is not on current main, but it should not merge as-is because it revives retired legacy env behavior and carries generated Plugin SDK baseline churn.

## What This Changes

The PR adds state-dir env normalization for gateway CLI and direct server startup, focused tests, and generated Plugin SDK baseline changes.

## Best Possible Solution

Land a narrow rebased patch that pins supported `OPENCLAW_STATE_DIR` at gateway startup, leaves retired legacy env aliases ignored, and omits local-only generated artifacts.

## Reproduction Assessment

Yes, at source level. Current main still resolves a relative `OPENCLAW_STATE_DIR` from the current cwd at each call and both gateway startup paths lack an early normalization hook; I did not rerun the standalone fixture in this read-only review.

## Solution Assessment

No, not in the current branch shape. Startup pinning for `OPENCLAW_STATE_DIR` is the narrow maintainable fix, but the branch should drop `CLAWDBOT_STATE_DIR`, remove generated baselines, and rebase onto current gateway startup code.
