## Summary

Keep open: the startup hardening is not on current main and is still a plausible bug fix, but the current PR head has blocking contract and generated-artifact issues before it can merge.

## What This Changes

This PR adds startup normalization for explicit state-dir env overrides, calls it from gateway CLI and direct server startup, adds focused tests, and carries generated Plugin SDK baseline artifacts.

## Best Possible Solution

Pin only the supported `OPENCLAW_STATE_DIR` at the earliest gateway CLI and direct server startup boundaries, keep legacy aliases ignored, and keep local generated baseline files out of the branch.

## Reproduction Assessment

Yes, by source inspection and the linked fixture path. Current main resolves a relative `OPENCLAW_STATE_DIR` when `resolveStateDir` is called, and gateway startup does not pin it before later cwd-sensitive work; I did not run the fixture in this read-only review.

## Solution Assessment

No, not in the current branch shape. Early startup pinning is the right direction, but the patch should be narrowed to the supported env contract, stripped of generated artifacts, and rebased onto current gateway startup code.
