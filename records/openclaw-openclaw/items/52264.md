## Summary

Keep open: the startup hardening remains useful, but the current PR is not mergeable as-is because it reintroduces a retired legacy env var and includes generated local baseline artifacts unrelated to the gateway fix.

## What This Changes

The PR adds startup normalization for relative state-dir env overrides in gateway CLI and direct server paths, plus focused path/order tests and generated Plugin SDK baseline changes.

## Best Possible Solution

Pin only the supported `OPENCLAW_STATE_DIR` before cwd-sensitive gateway startup work, keep the focused regression coverage, and remove generated local artifacts from the branch.

## Reproduction Assessment

Yes. Current main is source-reproducible because relative `OPENCLAW_STATE_DIR` is resolved at call time and the gateway startup paths do not pin it before cwd-sensitive work; the linked fixture gives a deterministic live path, but I did not run it in this read-only review.

## Solution Assessment

No, not in the current branch shape. Early startup pinning is the right fix direction, but the implementation should stay within the supported `OPENCLAW_STATE_DIR` contract and drop generated baseline churn.
