## Summary

Keep open. Current main still has the reported Windows ACL classification gap: broad unauthenticated or low-trust SIDs are not in the world-equivalent allowlists, so writable ACL entries for those principals still fall through to group classification. The paired closing PR #74383 is open and unmerged, so this issue should remain open for security-owner review and merge of that fix or an equivalent patch.

## What This Changes

The issue asks OpenClaw's Windows ACL audit to classify Anonymous Logon, Guests, Interactive, Local, and Network SIDs/principal names as world-equivalent so writable ACLs produce worldWritable/critical findings instead of groupWritable/warn findings.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, that expands the Windows world-equivalent SID/principal policy after secops confirmation and locks it with classifier plus filesystem-audit regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed summarizeWindowsAcl or inspectPathPermissions a writable ACL entry such as *S-1-5-7:(F); current main lacks that SID in WORLD_SIDS, so it falls into untrustedGroup and maps to groupWritable instead of worldWritable.

## Solution Assessment

Unclear until security owners confirm the exact broad-principal set. The narrow classifier allowlist update plus regression tests proposed by the paired PR is the maintainable fix boundary.
