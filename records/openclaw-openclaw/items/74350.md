## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and the open closing PR #74383 is the right implementation candidate to review before this issue is closed.

## What This Changes

The issue requests expanding the Windows ACL world-equivalent SID/principal allowlists so Anonymous Logon, Guests, Interactive, Local, and Network ACL entries are audited as world access rather than group access.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, that expands the Windows world-equivalent SID/principal policy and locks it with classifier plus filesystem-audit regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `summarizeWindowsAcl` or `inspectPathPermissions` a writable ACL entry such as `*S-1-5-7:(F)`; current main lacks that SID in `WORLD_SIDS`, so it is classified into `untrustedGroup` rather than `untrustedWorld`.

## Solution Assessment

Unclear until security-owner review confirms the final broad-principal set. The narrow allowlist update plus regression tests proposed by #74383 is the maintainable fix boundary.
