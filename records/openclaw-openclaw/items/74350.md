## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is already the closing implementation candidate, so this issue should remain open until that security patch or an equivalent maintainer-reviewed fix lands.

## What This Changes

The issue requests classifying broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as `world` rather than `group` in the security audit.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, that expands the Windows world-equivalent SID/principal policy after secops confirmation and locks it with classifier plus filesystem-audit regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `summarizeWindowsAcl` or `inspectPathPermissions` a writable ACL entry such as `*S-1-5-7:(F)`; current main lacks that SID in `WORLD_SIDS`, so it falls into `untrustedGroup` and maps to `groupWritable` rather than `worldWritable`.

## Solution Assessment

Unclear until security-owner review confirms the final broad-principal set. The narrow `WORLD_*` classifier update plus regression tests proposed by the linked fix PR is the maintainable fix boundary.
