## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is the closing implementation candidate, so this issue should remain open until that security patch or an equivalent maintainer-reviewed fix lands.

## What This Changes

Request to classify broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent so writable ACEs escalate through worldWritable security audit findings.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, that expands the Windows world-equivalent SID/principal policy after security-owner confirmation and locks the behavior with classifier plus filesystem-audit regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is to feed summarizeWindowsAcl or inspectPathPermissions a writable ACL entry such as *S-1-5-7:(F); current main lacks that SID in WORLD_SIDS, so it is classified into untrustedGroup instead of untrustedWorld.

## Solution Assessment

Yes for the fix boundary: the maintainable solution is a narrow WORLD_SIDS/WORLD_PRINCIPALS policy update with regression tests. The final principal set should still get maintainer security review because this changes audit severity policy.
