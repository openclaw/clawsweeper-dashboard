## Summary

Keep open. Current main still has the reported Windows ACL classification gap: the world-equivalent allowlists only cover Everyone, Authenticated Users, and BUILTIN\Users, so the reported broad principals still fall through to group classification and can downgrade audit severity. Open PR #74383 is the paired closing implementation candidate and should be reviewed rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to classify broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent instead of group-equivalent in security audits.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, after secops confirms the broad-principal set; the fix should update the ACL classifier policy and lock it with both classifier and filesystem-audit regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is to classify a writable ACL entry such as *S-1-5-7:(F); current main lacks S-1-5-7 in WORLD_SIDS, so the SID path returns group and inspectPathPermissions maps it to groupWritable rather than worldWritable.

## Solution Assessment

Unclear until security owners confirm the exact principal set, but the narrow classifier plus regression-test boundary in #74383 is the maintainable way to solve the reported audit downgrade.
