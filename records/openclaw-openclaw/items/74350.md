## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is the unmerged closing implementation candidate, so this security-sensitive issue should remain open for maintainer review rather than be closed or queued as a separate repair.

## What This Changes

The issue asks OpenClaw to classify broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent instead of group.

## Best Possible Solution

Review and land #74383 or an equivalent narrow security patch that expands the Windows world-equivalent SID/principal policy after security-owner confirmation and adds classifier plus filesystem-audit regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is to feed summarizeWindowsAcl or inspectPathPermissions an ACL entry such as *S-1-5-7:(F); current main classifies that SID as group, which maps to groupWritable rather than worldWritable.

## Solution Assessment

Unclear until security-owner policy review confirms the final broad-principal set. The narrow classifier update plus targeted regression tests is the right maintainable shape, and the already-open PR appears to be the appropriate implementation candidate.
