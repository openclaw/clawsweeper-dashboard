## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is the direct closing candidate, so this security-sensitive issue should remain open until that PR or an equivalent maintainer-reviewed patch lands.

## What This Changes

The issue requests classifying Anonymous Logon, Guests, Interactive, Local, and Network Windows ACL SIDs/principals as world-equivalent instead of group in security audits.

## Best Possible Solution

Review and land PR #74383, or an equivalent narrow security patch, with classifier and filesystem-audit regression tests for the downgraded ACL bucket.

## Reproduction Assessment

Yes. A source-level reproduction is a mocked icacls /sid ACL entry such as `*S-1-5-7:(F)`: current main parses it as a SID, misses it in WORLD_SIDS, and places it in untrustedGroup instead of untrustedWorld.

## Solution Assessment

Unclear until a security maintainer confirms the exact broad-principal set. The maintainable fix boundary is the narrow WORLD_SIDS/WORLD_PRINCIPALS classifier update plus regression tests, not a broader filesystem audit rewrite.
