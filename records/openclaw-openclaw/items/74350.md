## Summary

Keep open. Current main still has the reported Windows ACL classification gap: only the existing Everyone, Authenticated Users, and BUILTIN\Users SID/name set is treated as world-equivalent, omitted broad principals fall through to group, and filesystem audit severity still depends on the world-vs-group split. Open PR #74383 is already the closing implementation candidate, so this issue should remain open until that security fix or an equivalent maintainer-reviewed patch lands.

## What This Changes

The issue asks OpenClaw to classify additional broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent instead of group.

## Best Possible Solution

Land PR #74383, or an equivalent narrow security patch, that expands the Windows world-equivalent SID/principal policy and adds classifier plus filesystem-audit regression tests for the downgrade path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `summarizeWindowsAcl` or a mocked Windows `inspectPathPermissions` path a writable ACL entry such as `*S-1-5-7:(F)`; current main lacks that SID in `WORLD_SIDS`, so it is classified into `untrustedGroup` instead of `untrustedWorld`.

## Solution Assessment

Unclear until a security maintainer confirms the final broad-principal set. The maintainable fix boundary is the narrow `WORLD_SIDS`/`WORLD_PRINCIPALS` classifier update with regression tests, rather than changing POSIX permissions or the broader audit severity model.
