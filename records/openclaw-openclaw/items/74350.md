## Summary

Keep open. Current main and the latest tagged release still omit the reported broad Windows principals from the world-equivalent ACL classifier, and open PR #74383 is the direct closing candidate for this security-sensitive issue.

## What This Changes

The issue asks OpenClaw to classify Anonymous Logon, Guests, Interactive, Local, and Network Windows SIDs/principal names as world-equivalent in ACL audits.

## Best Possible Solution

Review and land PR #74383, or an equivalent narrow security patch, with classifier and filesystem-audit regression coverage for the downgraded ACL bucket.

## Reproduction Assessment

Yes. A source-level reproduction is a mocked Windows ACL /sid entry such as *S-1-5-7:(F), which current main parses as a SID, misses in WORLD_SIDS, and places in untrustedGroup instead of untrustedWorld.

## Solution Assessment

Unclear until a security maintainer confirms the final broad-principal set. The narrow maintainable fix is to update the classifier allowlists and add regression tests, not to rewrite filesystem auditing.
