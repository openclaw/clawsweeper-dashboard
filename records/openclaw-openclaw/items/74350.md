## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is already the direct closing candidate, so this security-sensitive issue should stay open until that patch or an equivalent maintainer-reviewed fix lands.

## What This Changes

The issue asks OpenClaw to classify Anonymous Logon, Guests, Interactive, Local, and Network Windows ACL principals as world-equivalent instead of group in filesystem security audits.

## Best Possible Solution

Review and land PR #74383, or an equivalent narrow security patch, with classifier and filesystem-audit regression tests for the downgraded ACL bucket.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked Windows ACL entry such as `*S-1-5-7:(F)`, which is absent from WORLD_SIDS on current main and therefore lands in untrustedGroup instead of untrustedWorld.

## Solution Assessment

Mostly yes, but the exact broad-principal set still needs security maintainer confirmation. The maintainable fix boundary is the narrow WORLD_SIDS/WORLD_PRINCIPALS classifier update plus regression tests, not a broader filesystem audit rewrite.
