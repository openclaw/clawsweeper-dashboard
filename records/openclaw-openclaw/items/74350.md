## Summary

Keep open. Current main still has the Windows ACL classification gap, and open PR #74383 is the direct closing candidate, so the security issue should remain open until that patch or an equivalent secops-reviewed fix lands.

## What This Changes

Treat broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent instead of group-level access.

## Best Possible Solution

Review and land PR #74383, or an equivalent narrow security patch, with classifier and filesystem-audit regression tests for the downgraded ACL bucket.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked Windows ACL entry such as *S-1-5-7:(F), which is absent from WORLD_SIDS on current main and therefore lands in untrustedGroup instead of untrustedWorld.

## Solution Assessment

Unclear until secops confirms the exact broad-principal set. The maintainable fix boundary is a narrow WORLD_SIDS/WORLD_PRINCIPALS classifier update plus regression tests, not a broader filesystem audit rewrite.
