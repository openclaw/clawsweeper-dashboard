## Summary

Keep open. Current main still has the reported Windows ACL world-classification gap, and open PR #74383 is the paired closing implementation candidate, so this issue should remain open until that security fix or an equivalent patch lands.

## What This Changes

Request to classify additional broad Windows ACL SIDs and principal names, including Anonymous Logon and Guests, as world-equivalent so filesystem audits report worldWritable instead of groupWritable.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, that expands the Windows world-equivalent principal policy after secops confirmation and locks it with classifier plus filesystem-audit regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed summarizeWindowsAcl or inspectPathPermissions a writable ACL entry such as *S-1-5-7:(F); current main lacks that SID in WORLD_SIDS, so it falls into untrustedGroup and maps to groupWritable.

## Solution Assessment

Unclear until security-owner policy review confirms the final broad-principal set. The narrow classifier-and-regression-test approach in #74383 is the maintainable fix boundary.
