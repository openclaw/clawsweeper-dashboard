## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is already the paired closing implementation candidate, so this security issue should remain open until that patch or an equivalent maintainer-reviewed fix lands.

## What This Changes

The issue asks OpenClaw to treat broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent instead of group-equivalent in security audits.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, that updates the Windows world-equivalent SID/principal policy and adds classifier plus filesystem-audit regression coverage for the severity downgrade path.

## Reproduction Assessment

Yes. A source-level reproduction is to feed `summarizeWindowsAcl` or a mocked Windows `inspectPathPermissions` path a writable ACL entry such as `*S-1-5-7:(F)`; current main lacks that SID in `WORLD_SIDS`, so it classifies as group rather than world.

## Solution Assessment

Unclear until a security maintainer confirms the final broad-principal set, but the narrowest maintainable fix boundary is the classifier allowlist plus targeted regression tests rather than changing POSIX permissions or the broader audit model.
