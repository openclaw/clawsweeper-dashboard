## Summary

Keep open. Current main still has the Windows ACL classification gap, and open PR #74383 is the direct closing candidate for the security fix, so the issue should remain open until that PR or an equivalent maintainer-reviewed patch lands.

## What This Changes

The issue requests classifying Windows ACL entries for Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent instead of group.

## Best Possible Solution

Review and land PR #74383, or an equivalent narrow security patch, with classifier and filesystem-audit regression tests for the downgraded ACL bucket.

## Reproduction Assessment

Yes. A source-level reproduction is an icacls /sid ACL entry such as `*S-1-5-7:(F)`: current main parses it as a SID, misses it in WORLD_SIDS, and places it in untrustedGroup instead of untrustedWorld.

## Solution Assessment

Unclear until a security maintainer confirms the full broad-principal set. The maintainable fix boundary is the narrow WORLD_SIDS/WORLD_PRINCIPALS classifier update plus regression tests, not a broader filesystem audit rewrite.
