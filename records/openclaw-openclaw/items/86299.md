## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Read topic names from the writer's cache scope:** `extensions/telegram/src/bot-native-commands.ts:1168-1172`
  - body: The existing topic-name writer in `bot-message-context` resolves its cache scope before route resolution with `account.accountId`, but this new native-command path looks up with `route.agentId`. In a default account routed to the main agent those are different store paths, so the cached service-message topic title still won't be found and `TopicName` remains undefined. Use the same shared topic-name cache scope as the writer before adding the field.
  - confidence: 0.9
- **[P1] Write action results into the readable topic cache:** `extensions/telegram/src/action-runtime.ts:731-783`
  - body: These new writes use the raw `chatId` and omit the cache scope, while the send helpers normalize topic-qualified targets and inbound/native readers use scoped store paths. A topic created or renamed through a target like `telegram:group:-100...:topic:271` will be stored under a key or namespace that later inbound messages do not read. Write with the normalized result chat id and the same topic-cache scope used by the readers.
  - confidence: 0.88
