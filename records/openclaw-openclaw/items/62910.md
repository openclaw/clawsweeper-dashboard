## Summary

Keep open. The underlying DEP0190 fix is still valid for current main, but this PR is not mergeable as-is because the shell path folds safe forwarded arguments with `args.join(" ")`, which loses argument boundaries for values containing spaces; the earlier ClawSweeper review comment flagged the same blocker after Greptile’s initial positive review.

## What This Changes

The PR refactors `scripts/ui.js` so Windows `.cmd`/`.bat` UI runner calls fold args into one shell command before `spawn`/`spawnSync`, and adds `prepareSpawnCommand` coverage in `test/scripts/ui.test.ts`.

## Best Possible Solution

Route Windows `.cmd`/`.bat` UI runner calls through the existing cmd.exe escaping seam with `shell: false` and `windowsVerbatimArguments`, preserving argument boundaries while avoiding DEP0190.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path on Windows with Node v24 and a `.cmd` UI runner, and the PR regression is reproducible by forwarding a safe argument containing spaces such as a Vitest `-t` pattern.

## Solution Assessment

No. Folding raw args with `args.join(" ")` removes the deprecation warning, but the existing `buildCmdExeCommandLine()`/pnpm-runner approach is the narrower maintainable fix because it preserves quoting and avoids `shell: true` with a separate args array.
