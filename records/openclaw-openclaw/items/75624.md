## Summary

Keep open. Current main still emits a sqlite-vec-specific degraded write warning from the no-embedding-provider path, and open PR #75637 is already the active closing implementation candidate.

## What This Changes

The issue requests clearer degraded vector warning text so missing embedding-provider or dimension cases do not blame sqlite-vec load failure.

## Best Possible Solution

Review and land or close PR #75637; the desired end state is a narrow diagnostic split that preserves sqlite-vec wording for actual load failures and reports missing embeddings or dimensions for vector-skipped writes.

## Reproduction Assessment

Yes. Source inspection on current main shows no-provider FTS-only indexing calls writeChunks with vectorReady false and no loadError, then the helper emits sqlite-vec-unavailable wording for that state.

## Solution Assessment

Yes. Splitting the warning text on existing loadError versus missing embeddings or dimensions is the narrow maintainable fix; sqlite-vec loading and FTS-only indexing behavior do not need broader changes for this report.
