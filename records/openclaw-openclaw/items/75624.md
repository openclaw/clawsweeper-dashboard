## Summary

Keep open. Current main still emits a sqlite-vec-specific degraded write warning from the no-embedding-provider FTS-only indexing path, and open PR #75637 is already the active closing implementation candidate.

## What This Changes

The issue asks memory indexing diagnostics to distinguish actual sqlite-vec load failures from missing embedding provider or unresolved vector dimensions.

## Best Possible Solution

Review and land or close PR #75637 so the shipped diagnostics preserve sqlite-vec wording for actual load failures and report missing embeddings or dimensions for vector-skipped writes.

## Reproduction Assessment

Yes. Source inspection shows runSync can load sqlite-vec separately, then provider-less FTS indexing writes chunks with vectorReady=false and no loadError, causing the sqlite-vec-specific warning.

## Solution Assessment

Yes. Splitting the warning text on existing loadError and embedding/vector readiness is the narrow maintainable fix; sqlite-vec loading and FTS-only indexing do not need behavioral changes for this report.
