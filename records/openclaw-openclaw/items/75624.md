## Summary

Keep open: current main still conflates non-ready vector writes with sqlite-vec failure, and open PR #75637 already uses closing syntax for this issue, so the issue should remain open until that PR lands or is closed.

## What This Changes

The issue asks OpenClaw to stop blaming sqlite-vec when vector rows are skipped because no embedding provider produced vector dimensions.

## Best Possible Solution

Review and land or replace open PR #75637 with a narrow memory-core diagnostic fix that keeps FTS-only indexing intact and splits warning text by sqlite-vec load failure versus missing embedding dimensions.

## Reproduction Assessment

Yes. The source-level path is high confidence: no embedding provider causes FTS-only writes with `vectorReady: false`, and the current helper emits sqlite-vec-specific wording without requiring a sqlite-vec load error.

## Solution Assessment

Yes. The narrow maintainable solution is to select the warning from existing vector load/error and provider/dimension state rather than changing sqlite-vec loading or FTS-only indexing semantics.
