## Summary

Keep open. Current main still emits the sqlite-vec-specific degraded warning whenever vector writes are not ready, including the no-embedding-provider FTS-only path with no sqlite-vec load error; open PR #75637 is already the active implementation candidate and is not merged.

## What This Changes

The issue requests clearer memory vector degradation warnings so missing embedding provider or unresolved dimensions are not reported as sqlite-vec load failure.

## Best Possible Solution

Review and land PR #75637, or an equivalent narrow memory-core diagnostic fix, so sqlite-vec wording is reserved for actual extension load failures and missing embeddings get their own message.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: run memory indexing without an embedding provider, which takes the FTS-only branch, writes chunks with `vectorReady: false`, and reaches the helper with no sqlite-vec load error.

## Solution Assessment

Yes. Branching the warning text on the existing load-error/load-state information is the narrow maintainable fix; changing sqlite-vec loading or FTS-only indexing semantics is not needed for this report.
