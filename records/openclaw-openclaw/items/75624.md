## Summary

Keep open. Current main still has the misleading memory-core warning path, and open PR #75637 is already the active closing implementation candidate, so this issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests splitting memory indexing warnings so sqlite-vec is blamed only for extension load failures and missing embedding dimensions are reported separately.

## Best Possible Solution

Land a narrow memory-core diagnostic fix that keeps sqlite-vec wording for actual extension load failures and reports missing embedding dimensions or provider availability when vector rows are skipped for non-sqlite reasons.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with no embedding provider, the FTS-only path writes chunks with `vectorReady: false`, and the current helper emits sqlite-vec-specific wording even when no load error is present.

## Solution Assessment

Yes. The narrow maintainable solution is to branch warning text on the existing vector load state/load error rather than changing sqlite-vec loading or FTS-only indexing semantics.
