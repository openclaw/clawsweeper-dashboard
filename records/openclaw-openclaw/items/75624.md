## Summary

Keep open. Current main still emits a sqlite-vec-specific degraded write warning from the no-embedding-provider path, and open PR #75637 is already the active closing implementation candidate.

## What This Changes

The issue requests clearer memory indexing diagnostics when vector writes are skipped because embeddings or dimensions are unavailable rather than because sqlite-vec failed to load.

## Best Possible Solution

Review and land or close PR #75637; the desired end state is a narrow diagnostic split that preserves sqlite-vec wording for actual load failures and reports missing embeddings or dimensions for vector-skipped writes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with no embedding provider, indexFile writes FTS-only chunks with vectorReady false, and the helper emits the sqlite-vec-specific warning even when loadError is absent.

## Solution Assessment

Yes. Splitting the warning text on existing loadError and embedding/vector readiness is the narrow maintainable fix; sqlite-vec loading and FTS-only indexing do not need behavioral changes for this report.
