## Summary

Keep open: the report maps to a real current-main failure guard and the expected Active Memory tool surface is documented, but source inspection does not prove the embedded runtime always materializes the memory plugin tools for the reported explicit-session path.

## What This Changes

Active Memory embedded recall should make `memory_recall`, `memory_search`, and `memory_get` callable inside its embedded sub-agent runtime.

## Best Possible Solution

Add an end-to-end regression around Active Memory embedded tool materialization, then fix the narrow runtime or plugin-registry layer that drops the selected memory plugin tools for `:active-memory:` sessions.

## Reproduction Assessment

A high-confidence live reproduction would run Active Memory with the reporter's explicit-session config and inspect the embedded run's effective tool list.

## Solution Assessment

Unclear because there is no proposed patch yet. The best fix should stay inside Active Memory embedded runtime/plugin tool materialization and should not add a new config switch or broaden the sub-agent tool surface beyond memory tools.
