## Summary

Keep open. Current main still does not expose the triggering user message to `agent:bootstrap`; prompt-aware hooks and context-engine assembly exist, but they do not let a bootstrap hook query-condition `bootstrapFiles` before Project Context injection.

## What This Changes

The issue requests adding the triggering user message to the `agent:bootstrap` hook context so hooks can query-condition bootstrap-file compression.

## Best Possible Solution

Add a backward-compatible optional current-user-message field to `AgentBootstrapHookContext`, thread it only from run paths where the prompt is already available, leave it undefined otherwise, and document how it differs from prompt-build hooks and context-engine assembly.

## Reproduction Assessment

Yes. Source inspection on current main shows the bootstrap hook context, bootstrap resolver parameters, and embedded-runner bootstrap calls contain no current prompt field, while prompt-aware hooks run through later or different surfaces.

## Solution Assessment

Unclear until maintainers choose the API name and semantics. The narrow technical direction is sound, but it is a public hook-context extension and should not be treated as a broken existing contract.
