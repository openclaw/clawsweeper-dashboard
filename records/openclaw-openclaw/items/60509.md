## Summary

Keep open. Current main still lacks any incoming user message, prompt, or query field on the pre-injection `agent:bootstrap` context; the later `before_prompt_build` plugin hook can see the prompt, but it runs after bootstrap files have already been resolved for injection and does not solve the requested bootstrap-file compression use case.

## What This Changes

Expose the incoming user message or query to the `agent:bootstrap` hook context so hooks can conditionally compress or replace bootstrap files before they are injected.

## Best Possible Solution

Add a backward-compatible optional `userMessage` field to the internal `agent:bootstrap` context, thread it only from user-triggered run paths where the current prompt is already available, leave it undefined for generated/background/raw-model runs, and document the distinction between pre-injection bootstrap mutation and later `before_prompt_build` prompt injection.

## Reproduction Assessment

Yes. The API gap is reproducible by source inspection on current main: the bootstrap context type and call chain contain no query field, while the current prompt is passed only to later prompt-build plugin hooks.

## Solution Assessment

Yes. A narrow optional context-field extension is the maintainable fix; moving SMELT-style compression into core or relying on `before_prompt_build` would either overreach or miss the pre-injection bootstrap-file mutation point.
