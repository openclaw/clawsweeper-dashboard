## Summary

Keep open. Current main and the latest release still do not expose the triggering user prompt on `agent:bootstrap`, while the prompt-aware hook/context-engine paths are later surfaces that cannot mutate `bootstrapFiles` before Project Context injection.

## What This Changes

Expose the triggering user message/query to `agent:bootstrap` hook context so hooks can query-condition workspace bootstrap files before injection.

## Best Possible Solution

Add a backward-compatible optional current-user-message field to `AgentBootstrapHookContext`, thread it from run paths where the prompt is already available, leave it undefined for non-user runs, and document how it differs from `before_prompt_build` and context-engine assembly.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the gap: bootstrap hook context and resolver parameters carry no prompt field, while the prompt exists only on later hook/context-engine surfaces.

## Solution Assessment

Yes. A narrow optional hook-context field is the smallest maintainable solution because `before_prompt_build` and context engines cannot mutate `bootstrapFiles` before Project Context injection.
