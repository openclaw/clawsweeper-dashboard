## Summary

Keep open. Current main still does not expose the triggering user prompt to `agent:bootstrap`; the prompt-aware surfaces run later or through context-engine assembly, so they cannot mutate `bootstrapFiles` before Project Context injection.

## What This Changes

The issue requests an optional current user-message/prompt field in the `agent:bootstrap` hook context for query-conditioned bootstrap-file compression.

## Best Possible Solution

Add a backward-compatible optional current-prompt field to `AgentBootstrapHookContext`, thread it only from run paths where `params.prompt` is already available, leave it undefined otherwise, and document how it differs from `before_prompt_build` and context-engine assembly.

## Reproduction Assessment

Yes, via source inspection. The bootstrap hook context, bootstrap resolver parameters, and embedded-runner bootstrap calls lack a current prompt field, while adjacent later surfaces receive `params.prompt`.

## Solution Assessment

Yes. A narrow optional hook-context field is the smallest maintainable API extension for query-conditioned bootstrap-file mutation; existing `before_prompt_build` and context-engine paths do not expose the mutable `bootstrapFiles` stage.
