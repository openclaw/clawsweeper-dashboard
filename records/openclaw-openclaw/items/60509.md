## Summary

Keep open. Current main still does not expose the triggering user prompt to `agent:bootstrap`; the prompt-aware alternatives run later or through context-engine assembly, so they do not let a bootstrap hook rewrite `bootstrapFiles` before Project Context injection.

## What This Changes

The issue requests an optional current user message field in the `agent:bootstrap` hook context so hooks can compress bootstrap files based on the triggering query.

## Best Possible Solution

Add a backward-compatible optional current-prompt field to `AgentBootstrapHookContext`, thread it from CLI and embedded run paths where `params.prompt` exists, leave it undefined for non-user-triggered runs, and document the distinction from `before_prompt_build` and context-engine assembly.

## Reproduction Assessment

Yes. Source inspection on current main shows the bootstrap hook context and bootstrap resolver parameters lack a current prompt field, while later prompt-aware surfaces receive `params.prompt`.

## Solution Assessment

Yes, pending maintainer API approval. A narrow optional hook-context field is the maintainable shape for query-conditioned bootstrap-file mutation; existing prompt-build and context-engine APIs do not expose the earlier mutable `bootstrapFiles` stage.
