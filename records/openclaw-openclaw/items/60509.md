## Summary

Keep open. Current main still does not expose the triggering prompt/user message to `agent:bootstrap`; prompt-aware plugin hooks and context-engine assembly exist, but they run through different surfaces and cannot let a bootstrap hook query-condition `bootstrapFiles` before injection.

## What This Changes

Expose the current user message or query string to the `agent:bootstrap` hook context so hooks can query-condition bootstrap file compression.

## Best Possible Solution

Add a backward-compatible optional user-message field to `AgentBootstrapHookContext`, thread it only from run paths where the current user prompt is already available, leave it undefined when unavailable, and document how it differs from `before_prompt_build` and context-engine assembly.

## Reproduction Assessment

Yes. Source inspection on current main shows the bootstrap hook context, bootstrap resolver parameters, and embedded-runner bootstrap calls contain no current prompt field, while prompt-aware hooks run through a different later surface.

## Solution Assessment

Yes. A narrow optional context-field extension is the smallest maintainable way to support query-conditioned bootstrap-file mutation; using `before_prompt_build` or context engines alone would not let the existing bootstrap hook rewrite `bootstrapFiles` before Project Context injection.
