## Summary

Keep open. Current main still does not expose the triggering user prompt to `agent:bootstrap`, and the existing prompt-aware hook/context-engine paths run later or target a different mutable surface, so this still needs a maintainer API decision.

## What This Changes

The issue requests an optional current user-message field in the `agent:bootstrap` hook context so hooks can condition bootstrap file content on the triggering prompt.

## Best Possible Solution

Add a backward-compatible optional current-prompt field to `AgentBootstrapHookContext`, thread it from CLI and embedded user-triggered run paths, leave it undefined for non-user-triggered runs, and document how it differs from `before_prompt_build` and context-engine assembly.

## Reproduction Assessment

Yes, source-level. Current main exposes no current-prompt field on `AgentBootstrapHookContext` or bootstrap resolver parameters, while later prompt-aware surfaces receive `params.prompt`.

## Solution Assessment

Yes, pending maintainer API approval. A narrow optional hook-context field is the maintainable shape for query-conditioned bootstrap-file mutation; existing prompt-build and context-engine APIs do not expose the earlier mutable `bootstrapFiles` stage.
