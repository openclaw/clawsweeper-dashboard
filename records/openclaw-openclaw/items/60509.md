## Summary

Keep open. Current main and the latest shipped release still do not expose the incoming user message or prompt on the pre-injection `agent:bootstrap` hook context; the newer context-engine `prompt` path is adjacent but does not let bootstrap hooks query-condition `bootstrapFiles` before injection.

## What This Changes

The issue requests exposing the incoming user message on the `agent:bootstrap` hook context so bootstrap hooks can compress `bootstrapFiles` based on the current query.

## Best Possible Solution

Add a backward-compatible optional current-user-message field to the bootstrap hook context, thread it only from user-triggered runs where the prompt is already available, leave it undefined for background/raw/heartbeat paths, and document the boundary from context-engine assembly.

## Reproduction Assessment

Yes. The API gap is reproducible by source inspection: the hook context type, hook builder, bootstrap resolver, and embedded-runner bootstrap call carry no prompt field, while the prompt-aware context-engine path runs later.

## Solution Assessment

Yes. A narrow optional hook-context field is the most direct maintainable solution for query-conditioned mutation of `bootstrapFiles`; context-engine prompt support is useful but not equivalent to this hook surface.

## Repair Work Prompt

Implement the request from https://github.com/openclaw/openclaw/issues/60509 as a backward-compatible hook API extension. Add an optional current user message field, preferably `userMessage?: string` unless existing hook naming strongly favors `prompt`, to `AgentBootstrapHookContext`; thread it through `applyBootstrapHookOverrides`, `resolveBootstrapFilesForRun`, and `resolveBootstrapContextForRun` from user-triggered CLI and embedded-runner paths that already have the current prompt. Leave it undefined for heartbeat, compact/background, raw model/probe, startup, and no-prompt paths. Add focused coverage proving hooks receive the value on normal user runs and do not receive it where unavailable. Update `docs/automation/hooks.md` and add a user-facing changelog entry. Do not change context-engine ownership, `contextInjection` semantics, bootstrap file selection, or log full prompt text.
