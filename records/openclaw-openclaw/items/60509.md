## Summary

Keep open. Current main still does not expose the triggering user prompt to `agent:bootstrap`; prompt-aware hooks and context-engine assembly exist, but they run through later or different surfaces that cannot mutate `bootstrapFiles` before Project Context injection.

## What This Changes

Request adding an optional current user message or query field to the `agent:bootstrap` hook context for query-conditioned bootstrap-file compression.

## Best Possible Solution

Add a backward-compatible optional current-prompt field to `AgentBootstrapHookContext`, thread it only from run paths where the prompt is already available, leave it undefined otherwise, and document how it differs from `before_prompt_build` and context-engine assembly.

## Reproduction Assessment

Yes. Source inspection on current main shows the bootstrap hook context, bootstrap resolver parameters, and embedded-runner bootstrap calls contain no current prompt field, while adjacent prompt-aware surfaces receive `params.prompt` later.

## Solution Assessment

Yes. A narrow optional hook-context field is the smallest maintainable API extension for query-conditioned bootstrap-file mutation; `before_prompt_build` and context engines do not satisfy the pre-injection `bootstrapFiles` use case.
