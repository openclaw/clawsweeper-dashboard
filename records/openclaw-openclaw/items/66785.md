## Summary

Keep open. Current main still contains the existing-config installer fallthrough identified in the prior review: the large model-list and OpenRouter onboarding paths are now present, but the installer can still continue into onboarding immediately after saying it is skipping onboarding for an existing config.

## What This Changes

The issue reports that the Umbrel setup wizard cannot scroll past the model list, cannot select OpenRouter, and keeps forcing onboarding even when a config file is already present.

## Best Possible Solution

Fix the installer finalization path so existing OPENCLAW_CONFIG_PATH or legacy config detection exits the onboarding branch after doctor/dashboard handling, then cover that path with a focused install.sh regression test. Leave the OpenRouter and general model-picker code alone unless a targeted packaged-Umbrel verification shows a separate remaining picker defect.

## Reproduction Assessment

Partial yes. The forced-onboarding part has a high-confidence source reproduction path in scripts/install.sh: existing config plus an interactive TTY reaches `exec "$claw" onboard` after the skip message; the exact Umbrel scroll surface still lacks live reproduction evidence.

## Solution Assessment

Yes. The narrow maintainable fix is in the installer/Umbrel setup entrypoint, not a broad provider or model-picker rewrite, because current main already has searchable model filtering and OpenRouter auth-choice support.
