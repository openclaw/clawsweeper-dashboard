## Summary

Keep open. Current main still has the installer fallthrough where an existing config logs that onboarding is skipped, then continues to `Starting setup` and can run `openclaw onboard`; the model picker/OpenRouter portions look mitigated, but the forced-onboarding bug remains.

## What This Changes

The issue asks to restore a usable Umbrel setup wizard by making model/OpenRouter selection reachable and avoiding repeated onboarding when config already exists.

## Best Possible Solution

Fix the installer finalization branch so existing current and legacy config paths bypass onboarding immediately after doctor/dashboard handling, then add focused regression coverage for both config paths.

## Reproduction Assessment

Partial yes. Source inspection of current main shows an existing config reaches the doctor/skip log branch, then still reaches `Starting setup` and can exec `openclaw onboard` on an interactive TTY; the Umbrel scroll UI itself was not live-reproduced.

## Solution Assessment

Yes. The narrowest maintainable fix is in `scripts/install.sh` with installer regression tests; model picker/OpenRouter should stay unchanged unless independently reproduced.
