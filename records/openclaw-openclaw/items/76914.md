## Summary

Keep this PR open: it targets a valid CLI-backed preview gap, but the current head still has a blocking test-type failure and the new bridge can let partial deliveries run after final reply cleanup.

## What This Changes

The PR forwards CLI assistant agent events into the shared `onPartialReply` preview path, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the bridge in the reply runner, but make bridged partial deliveries ordered and drained before final completion, with typed regression coverage for normal and silent CLI runs.

## Reproduction Assessment

Yes from source inspection, though not from a live Telegram run. Current main emits CLI assistant events while Telegram previews listen to `onPartialReply`, and the CLI reply-runner path does not connect those surfaces during the run.

## Solution Assessment

No, not as currently implemented. The chosen boundary is narrow and maintainable, but the PR should fix the test typing and drain serialized bridge deliveries before final reply cleanup.
