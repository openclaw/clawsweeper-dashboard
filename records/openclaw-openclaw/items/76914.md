## Summary

Keep open. Current main still lacks the CLI-to-preview bridge and this PR targets the right bug with real behavior proof, but the latest patch still has blocking ordered-delivery and test-type issues; the merged reasoning-default PR linked in discussion does not cover this CLI runner path.

## What This Changes

The PR adds a CLI-branch agent-event listener that forwards assistant text into `onPartialReply`, adds regression coverage, and updates the changelog for CLI-backed channel previews.

## Best Possible Solution

Keep the CLI event bridge, but serialize and drain bridged partial delivery before final CLI cleanup, preserve the silent/sanitization guards, fix the typed regression test, and then review the draft PR for merge with the linked issue.

## Reproduction Assessment

Yes. Source inspection plus the live proof show CLI runners emit assistant text/delta events while Telegram previews consume `onPartialReply`, and current main's CLI branch only forwards final text after `runCliAgent`; I did not rerun the live Telegram setup in this read-only review.

## Solution Assessment

No, not as currently patched. Bridging at the reply-runner CLI boundary is narrow and maintainable, but the implementation must preserve ordered/drained partial delivery and exact test typing before merge.
