## Summary

Keep open: current main still has the linked CLI preview gap, and this PR targets the right boundary, but the latest patch still has blocking issues around unmanaged async partial delivery and test mock typing.

## What This Changes

The PR forwards CLI assistant agent events into the reply runner's `onPartialReply` path, adds regression coverage, and records the user-facing fix in the changelog.

## Best Possible Solution

Keep the bridge in the reply runner, but route CLI assistant events through a serialized and drained partial-delivery chain before final reply cleanup.

## Reproduction Assessment

Yes from source inspection: current main emits CLI assistant events, while Telegram previews consume `onPartialReply`, and the CLI reply-runner path does not connect those surfaces. I did not run a live Telegram reproduction.

## Solution Assessment

No, not as currently patched. The boundary is narrow and maintainable, but the bridge needs ordered/drained async delivery and typed regression assertions before merge.
