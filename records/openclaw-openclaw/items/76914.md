## Summary

Keep open: the PR targets a real current-main CLI streaming gap, but the patch is not merge-ready because the new bridge bypasses existing preview-suppression guards and leaks its event listener on CLI failures.

## What This Changes

This PR bridges matching CLI assistant agent events into typing and `onPartialReply`, adds a regression test, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Keep the CLI event bridge, but route it through the existing partial-reply normalization/silent guards and guarantee listener cleanup on every terminal path.

## Reproduction Assessment

Yes from source inspection: current main emits CLI assistant deltas on the agent-event bus, while Telegram previews listen only to `onPartialReply` and the CLI reply-runner path currently emits only final text. I did not run a live Telegram/CLI session because this review is read-only.

## Solution Assessment

No. The bridge is the right surface, but the current patch should reuse the existing preview filtering path and unsubscribe the listener in error/finally handling before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reuse the partial-reply guard for CLI deltas:** `src/auto-reply/reply/agent-runner-execution.ts:1315-1316`
  - body: The new bridge forwards every assistant event's raw `text` directly to typing and `onPartialReply`. That skips the existing `normalizeStreamingText` / `handlePartialForTyping` checks that suppress `silentExpected`, `NO_REPLY`/heartbeat-only text, empty text, and sanitization before channel previews. A silent CLI run can therefore leak text into Telegram preview even though final delivery would suppress it.
  - confidence: 0.88
- **[P2] Unsubscribe the CLI bridge on failed runs:** `src/auto-reply/reply/agent-runner-execution.ts:1305`
  - body: The listener is registered before `runCliAgent`, but the only unsubscribe call is on the success path before the final assistant emit. If `runCliAgent` throws, the catch/finally emits terminal lifecycle events while leaving the global listener installed, so failed CLI runs accumulate dead listeners for the process lifetime. Move cleanup into all terminal paths while preserving the pre-final-emit unsubscribe on success.
  - confidence: 0.9
