## Summary

Keep open: the PR targets a valid current-main bug, but the exact head is not merge-ready because `check-test-types` fails on the new regression test and the new bridge can leave asynchronous partial-preview deliveries running after the CLI run completes.

## What This Changes

This PR adds a reply-runner bridge from CLI assistant agent events to `onPartialReply`, covers it with regression tests, and adds a changelog entry for CLI-backed channel previews.

## Best Possible Solution

Keep the bridge in the reply runner, but serialize and drain bridged partial deliveries before final completion, and keep typed regression coverage for normal and silent CLI runs.

## Reproduction Assessment

Yes from source inspection and the linked report, though not from a live Telegram run. Current main emits CLI assistant events while Telegram previews only listen to `onPartialReply`, and the CLI reply-runner path does not connect those surfaces.

## Solution Assessment

No, not as currently implemented. The chosen boundary is narrow and maintainable, but the test typing must compile and the bridge should not let unordered or late partial deliveries outlive final reply cleanup.
