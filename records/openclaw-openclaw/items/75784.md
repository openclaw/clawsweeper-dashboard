## Summary

Keep this open. Current main still has the reported transcript hygiene gap: WebChat history filters heartbeat text patterns, but it does not use the existing provenance marker, and subagent orphan recovery still sends synthetic resume text as an unprovenanced user-role turn.

## What This Changes

The issue reports synthetic or runtime-generated user-role messages appearing in WebChat after heartbeat wakeups, gateway restart, session repair, or subagent diagnostics.

## Best Possible Solution

Add a provenance-based WebChat display contract: tag synthetic orphan-recovery/repair handoff prompts with inputProvenance, then hide or system-render internal/inter-session provenance rows while preserving real external user messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass projectRecentChatDisplayMessages a role:"user" message with provenance.kind="inter_session" and observe that current main leaves it visible unless the text matches a heartbeat-specific pattern.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is provenance tagging plus projection/SSE regression tests, not broader text matching against task phrases or [System] prefixes.

Fix https://github.com/openclaw/openclaw/issues/75784. WebChat must not show runtime-generated or inter-session user-role prompts after heartbeat wakeups, gateway restart, session repair, or subagent orphan recovery. Current main filters heartbeat text in src/gateway/chat-display-projection.ts but ignores provenance, while src/agents/subagent-orphan-recovery.ts sends its synthetic resume prompt through the gateway agent method without inputProvenance. Implement the narrow fix: use the existing input provenance contract to hide or system-render inter-session/internal user-role rows in WebChat/session-history projections, add inputProvenance to orphan-recovery resume calls with a clear sourceTool such as subagent_orphan_recovery, and preserve real external_user messages. Add focused regression coverage for projectRecentChatDisplayMessages, SessionHistorySseState inline append/refresh behavior, and orphan recovery's gateway call params.
