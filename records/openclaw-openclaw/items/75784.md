## Summary

Keep this open. Current main still has a narrow transcript-display gap: WebChat filters heartbeat text patterns, but it does not hide user-role rows by inter-session provenance, and orphan-recovery resume prompts still enter the agent method without input provenance.

## What This Changes

Prevent synthetic heartbeat, gateway-restart, session-repair, and inter-session prompts from appearing in WebChat as user-authored messages.

## Best Possible Solution

Add a provenance-based WebChat display contract: tag synthetic orphan-recovery/repair handoff prompts with inputProvenance and hide or system-render internal/inter-session provenance rows while preserving real external user turns.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass projectRecentChatDisplayMessages a role:"user" message with provenance.kind:"inter_session" and observe that current main leaves it visible unless the text matches a heartbeat-specific pattern.

## Solution Assessment

Yes, the provenance-based direction is the narrowest maintainable solution. More text matching against task phrases, [System] prefixes, or heartbeat-adjacent wording would be brittle and could hide legitimate user messages.

Add a provenance-based display contract: tag orphan-recovery/session-repair synthetic handoff prompts as inter_session or internal_system as appropriate, and update Gateway WebChat/session-history projection so internal/inter-session user rows are hidden or rendered as non-user system context while real external user messages remain visible.
