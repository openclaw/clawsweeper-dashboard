## Summary

Keep this open. Current main has solid filters for older heartbeat/system-event transcript pollution, but this report adds a still-plausible path: synthetic subagent recovery and inter-session handoff messages can be persisted as user turns, and the WebChat projection does not currently filter or relabel those provenance-bearing user rows. The issue is recent, concrete, non-maintainer-authored, and not protected, so it is a good narrow repair candidate rather than a cleanup close.

## What This Changes

Prevent internal heartbeat, subagent recovery, gateway restart/session repair, and diagnostic prompts from appearing in WebChat as user-authored messages.

## Best Possible Solution

Add a narrow transcript-display contract for runtime-generated user rows: synthetic subagent recovery, subagent announce, heartbeat/system-event, and repair-only handoff prompts should carry provenance and be hidden or rendered as non-user system/internal events in WebChat, while real external user turns remain visible.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction can be built by constructing `chat.history`/`SessionHistorySseState` raw messages with an inter-session or orphan-recovery synthetic user prompt containing a distinctive task phrase and verifying that current projection would not drop it unless it matches the older heartbeat/internal-envelope patterns.

## Solution Assessment

No, current main is not the best complete solution for this issue. The older heartbeat and internal-envelope filters are useful, but the safer fix is to tag orphan-recovery resumes with provenance and extend WebChat projection/tests for internal provenance sources without suppressing real user messages.
