## Summary

Keep this open: current main still has a concrete WebChat transcript-hygiene gap for synthetic user-role rows, especially orphan-recovery resume prompts that are not tagged with the existing inter-session provenance signal and therefore can remain visible in chat history.

## What This Changes

The issue asks OpenClaw to stop runtime-generated heartbeat, restart-repair, and subagent recovery text from appearing in WebChat as user-authored messages.

## Best Possible Solution

Add a provenance-based WebChat display contract: tag runtime-generated orphan-recovery and repair handoff prompts with `inputProvenance`, then hide or system-render inter-session/internal provenance rows while preserving real external user messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `projectRecentChatDisplayMessages` or `buildSessionHistorySnapshot` a `role: "user"` row with `provenance.kind = "inter_session"`, or inspect `resumeOrphanedSession` and observe it sends an untagged synthetic user-role prompt.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is provenance tagging plus projection regression tests, not broader heartbeat or `[System]` text matching.
