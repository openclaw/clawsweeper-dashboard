## Summary

Keep this open. Current main still has a concrete WebChat transcript hygiene gap: the visible history projection hides known heartbeat text patterns, but it does not use the existing inter-session provenance marker, and orphan-recovery resume prompts are still sent as synthetic user-role messages without provenance.

## What This Changes

The issue asks OpenClaw to stop showing runtime-generated heartbeat, restart, session-repair, and subagent-recovery prompts as user-authored WebChat messages.

## Best Possible Solution

Add a provenance-based WebChat display contract: tag runtime-generated orphan-recovery and repair handoff prompts with `inputProvenance`, then hide or system-render inter-session/internal provenance rows while leaving real external user messages visible.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `projectRecentChatDisplayMessages` a `role: "user"` row with `provenance.kind = "inter_session"` or an orphan-recovery resume prompt and observe that current main leaves it visible unless it matches heartbeat-specific text.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is provenance tagging plus projection tests, not broader text matching against `[System]` or task phrases.
