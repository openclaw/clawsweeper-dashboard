## Summary

Keep this open. Current main still has a concrete WebChat transcript hygiene gap: the visible history projection hides known heartbeat text patterns, but it does not use the existing inter-session provenance marker, and the orphan-recovery resume path still sends a synthetic user-role prompt without provenance.

## What This Changes

The issue asks OpenClaw to stop showing runtime-generated heartbeat, restart, session-repair, and subagent-recovery prompts as user-authored WebChat messages.

## Best Possible Solution

Add a narrow provenance-based WebChat display contract: tag runtime-generated orphan-recovery and repair handoff prompts with `inputProvenance`, then have the chat display projection hide or render inter-session/internal provenance rows as system/runtime events while leaving real external user messages visible.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `projectRecentChatDisplayMessages` a `role: "user"` row with `provenance.kind = "inter_session"` or an orphan-recovery resume prompt and observe that current main leaves it visible unless it matches heartbeat-specific text.

## Solution Assessment

Yes for the repair direction, but current main is incomplete. The narrow maintainable fix is provenance tagging plus projection tests, not broader text matching against `[System]` or task phrases.
