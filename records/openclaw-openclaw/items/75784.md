## Summary

Keep this open. Current main still lacks provenance-aware filtering in the shared WebChat/SSE history projection, and orphan recovery still sends its synthetic resume turn without `inputProvenance`, so synthetic resume/test text can be persisted and displayed as a normal user row.

## What This Changes

The issue asks for synthetic heartbeat, gateway-restart, and session-repair prompts to stop appearing as user-authored WebChat messages.

## Best Possible Solution

Use the existing `InputProvenance` contract end to end: tag orphan-recovery resume prompts as inter-session/internal and hide provenance-marked internal user rows from shared WebChat/SSE history projection.

## Reproduction Assessment

Yes, source-level. A `role: "user"` message with `provenance.kind: "inter_session"` and ordinary text still passes through `projectRecentChatDisplayMessages`, and orphan recovery creates a synthetic resume prompt without `inputProvenance`.

## Solution Assessment

Yes. The narrow maintainable solution is to propagate existing provenance through orphan recovery and make shared history projection use provenance, rather than adding more text-pattern filters or a new config knob.
