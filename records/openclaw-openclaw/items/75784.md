## Summary

Keep this open. Current main still has the transcript hygiene gap: WebChat history/SSE projection uses text-pattern filters but does not inspect inter-session provenance, and subagent orphan recovery still sends its synthetic resume turn without `inputProvenance`.

## What This Changes

The issue reports phantom WebChat user messages after heartbeat wakeups, gateway restart/session repair, and subagent diagnostics.

## Best Possible Solution

Use the existing provenance contract end to end: tag synthetic orphan-recovery resume prompts as inter-session/internal and make WebChat/SSE history projection hide or system-render provenance-marked internal user rows.

## Reproduction Assessment

Yes, source-level. Passing `projectRecentChatDisplayMessages` a `role: "user"` message with `provenance.kind: "inter_session"` and ordinary text leaves it visible, and orphan recovery creates a synthetic resume prompt without provenance.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is to propagate `inputProvenance` through orphan recovery and make shared history projection use provenance, not add more prompt-text heuristics.
