## Summary

Keep open. Current main still has the transcript hygiene gap: the WebChat/SSE display projection does not inspect inter-session provenance, and subagent orphan recovery still creates a synthetic resume turn without `inputProvenance`.

## What This Changes

Prevent synthetic or internally-routed prompts from appearing as ordinary user messages in WebChat after heartbeat wakeups, gateway restarts, or session repair.

## Best Possible Solution

Use the existing provenance contract end to end: tag synthetic orphan-recovery resume prompts as inter-session/internal and make the shared WebChat/SSE history projection hide or separately render provenance-marked internal user rows, with focused regression coverage.

## Reproduction Assessment

Yes, source-level. Passing `projectRecentChatDisplayMessages` a `role: "user"` message with `provenance.kind: "inter_session"` and ordinary text would leave it visible, and orphan recovery still creates its synthetic resume prompt without provenance.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is to propagate `inputProvenance` through orphan recovery and make shared history projection use provenance rather than adding more prompt-text heuristics.
