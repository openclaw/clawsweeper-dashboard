## Summary

Keep this open. The existing ClawSweeper comment is still confirmed by current main: WebChat history filtering does not honor inter-session provenance, and subagent orphan recovery still sends synthetic resume prompts as user-role agent messages without provenance.

## What This Changes

The issue reports WebChat displaying runtime-generated user-role messages after heartbeat wakes, gateway restart/session repair, and subagent orphan recovery.

## Best Possible Solution

Add a provenance-based WebChat display contract: tag runtime-generated orphan-recovery and repair handoff prompts with `inputProvenance`, then hide or system-render inter-session/internal provenance rows while preserving real external user messages.

## Repair Work Prompt

Fix the WebChat phantom-message bug reported at https://github.com/openclaw/openclaw/issues/75784. Current main filters heartbeat text in `src/gateway/chat-display-projection.ts` but does not use `hasInterSessionUserProvenance`, and `src/agents/subagent-orphan-recovery.ts` sends synthetic resume prompts through `callGateway({ method: "agent" })` without `inputProvenance`. Add a provenance-based display contract: tag runtime-generated orphan-recovery resume prompts, and any narrowly identified repair/handoff prompts in the same flow, with `inputProvenance` such as `kind: "inter_session"`, `sourceChannel: "internal"`, and an appropriate `sourceTool`; then update WebChat/session-history projection to hide or system-render inter-session/internal user-role rows while preserving real `external_user` messages. Do not add broad text matching for `[System]`, heartbeat phrases, or task text. Add focused tests for `projectRecentChatDisplayMessages`/`buildSessionHistorySnapshot`, orphan-recovery gateway params, and existing session title behavior. Add a user-facing `CHANGELOG.md` fix entry unless maintainers decide this is test-only.
