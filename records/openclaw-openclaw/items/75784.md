## Summary

Keep this open. Current main still has the reported WebChat transcript hygiene gap: display projection hides heartbeat/text-envelope cases but not existing inter-session provenance, and subagent orphan recovery still resumes with synthetic user-role text that is not tagged with provenance.

## What This Changes

The issue reports phantom WebChat user messages after heartbeat wakes, gateway restart, session repair, and subagent orphan recovery.

## Best Possible Solution

Add a provenance-based display contract for WebChat history/SSE and tag synthetic orphan-recovery resume prompts with inputProvenance, without broad prompt-text matching or hiding real external user messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass projectRecentChatDisplayMessages a role:"user" message with provenance.kind = "inter_session" and non-heartbeat content; current main keeps that row visible.

## Solution Assessment

No. Current main is incomplete; the narrow maintainable fix is to use the existing provenance signal in chat display projection and to tag orphan-recovery synthetic prompts before persistence.
