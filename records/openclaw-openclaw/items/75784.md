## Summary

Keep this open. Current main still has a concrete WebChat transcript hygiene gap: the display projection filters heartbeat text patterns, but restart/orphan recovery resume turns are still synthetic user-role messages without provenance, and provenance-marked user rows are not hidden by the WebChat projection.

## What This Changes

Prevent heartbeat, gateway-restart, and session-repair synthetic prompts from appearing in WebChat as user-authored messages.

## Best Possible Solution

Tag internal restart/orphan resume prompts with input provenance and make the WebChat display projection hide or system-render inter-session/internal user-role rows while preserving real external user messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `projectRecentChatDisplayMessages` or `buildSessionHistorySnapshot` a user-role `[System]` restart/orphan resume row, or a user row with `provenance.kind = "inter_session"`; current main only hides empty and heartbeat-pattern rows, so these synthetic rows remain visible.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is provenance tagging plus projection/SSE tests, not broader text matching against `[System]`, task phrases, or other incidental prompt content.
