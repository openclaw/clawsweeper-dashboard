## Summary

Keep this open. Current main has a partial subagent-announce cleanup from https://github.com/openclaw/openclaw/pull/79618, but the orphan-recovery resume path still sends a synthetic resume prompt as an ordinary Gateway `agent` message without `inputProvenance`, and the history projection only hides `sourceTool=subagent_announce` inter-session rows rather than the remaining internal resume/provenance case.

## What This Changes

The issue reports phantom WebChat user messages after heartbeat wake, gateway restart, or session repair.

## Best Possible Solution

Extend the existing provenance contract end to end: tag or suppress orphan-recovery synthetic resume prompts as internal/inter-session, and make WebChat/SSE projection hide or clearly system-render provenance-marked internal user rows without regressing the merged `subagent_announce` behavior.

## Reproduction Assessment

Yes, source-level. An orphaned subagent with `abortedLastRun: true` follows `resumeOrphanedSession`, which submits the synthetic resume text through the Gateway `agent` method without `inputProvenance`; current projection then lacks a generic internal/orphan provenance guard.

## Solution Assessment

No, current main is not complete. The narrow maintainable fix is to propagate provenance or prompt-persistence suppression from orphan recovery and update shared display projection/tests, rather than adding more text-only heuristics.
