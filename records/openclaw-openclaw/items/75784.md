## Summary

Keep this open. Current main still has a concrete WebChat transcript hygiene gap: heartbeat/internal-wrapper rows are filtered, but provenance-tagged inter-session user rows are not, and orphan-recovery resumes still create synthetic agent messages without provenance.

## What This Changes

The issue asks OpenClaw to stop showing runtime-generated WebChat rows from heartbeat wakeups, gateway restart orphan recovery, stuck subagent diagnostics, and session repair as user-authored messages.

## Best Possible Solution

Add a provenance-based display contract for runtime-generated user-role rows: orphan-recovery resumes and other synthetic handoffs should carry structured provenance, and WebChat history/SSE projection should hide or system-render those rows while preserving ordinary `external_user` turns.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `projectRecentChatDisplayMessages` or `buildSessionHistorySnapshot` a user-role message with normal text and `provenance.kind: "inter_session"`; current main returns it visible, and orphan recovery currently sends its synthetic resume without provenance.

## Solution Assessment

No, current main is not the complete solution. The narrow maintainable fix is to use the existing provenance contract and add the missing orphan-recovery tagging plus WebChat projection tests, instead of broad text matching on phrases like the reported test prompt.
