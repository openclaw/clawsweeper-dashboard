## Summary

Keep open. Current main still leaves a plausible WebChat transcript hygiene gap: WebChat history projection filters heartbeat text and empty rows, but it does not inspect user-message provenance, and subagent orphan-recovery resume still sends its synthetic resume prompt without input provenance.

## What This Changes

The issue requests preventing runtime-generated heartbeat, subagent recovery, and session-repair prompts from appearing as user-authored WebChat transcript messages.

## Best Possible Solution

Define a provenance-backed display contract for runtime-generated user-shaped rows: subagent announce, orphan-recovery resume, heartbeat/system-event, and repair-only handoff prompts should carry provenance and WebChat/session-history projection should hide or render them as internal/system events while preserving ordinary external user turns.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction can pass a user-role message with `provenance.kind = "inter_session"` through `projectRecentChatDisplayMessages` and observe current projection leaves it visible; orphan recovery also has a direct source path that sends a synthetic resume without provenance.

## Solution Assessment

No, current main is not a complete solution. The narrow maintainable fix is to tag missing orphan-recovery resumes with provenance and extend WebChat/session-history projection tests around provenance rather than broad string matching.
