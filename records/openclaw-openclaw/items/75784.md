## Summary

Keep this open. Current main still has the transcript display gap: WebChat history projects raw session messages through text-pattern filters, while inter-session provenance is available but not used there, and the subagent orphan-recovery resume path still sends synthetic resume text without provenance.

## What This Changes

The issue reports phantom user messages appearing in WebChat after heartbeat wakeups, gateway restart recovery, or session repair.

## Best Possible Solution

Add a provenance-based WebChat display contract: tag synthetic recovery handoff prompts with inputProvenance, hide or system-render internal/inter-session provenance rows, and preserve real external user messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass projectRecentChatDisplayMessages a role:"user" message with provenance.kind="inter_session" or an orphan-recovery-style synthetic user turn and observe that current main leaves it visible unless the text matches heartbeat or empty-message filters.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is provenance tagging plus projection/SSE regression tests, not broader text matching against task phrases or [System] prefixes.
