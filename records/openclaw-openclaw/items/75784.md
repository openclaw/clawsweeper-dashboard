## Summary

Keep this open. Current main still has the transcript hygiene gap: WebChat/history projection filters heartbeat and empty text heuristics, but it does not use provenance to hide internal/inter-session user rows, and subagent orphan recovery still sends its synthetic resume prompt without provenance.

## What This Changes

Prevent synthetic/internal recovery, heartbeat, and inter-session prompts from appearing in WebChat as user-authored messages after wake/restart/session repair flows.

## Best Possible Solution

Use the existing provenance contract end to end: tag synthetic orphan-recovery resume prompts as internal/inter-session input and make shared WebChat/SSE history projection hide or explicitly system-render provenance-marked internal user rows.

## Reproduction Assessment

Yes, source-level. A role:'user' message with provenance.kind:'inter_session' and ordinary text still passes projectRecentChatDisplayMessages, and orphan recovery creates a synthetic resume turn without inputProvenance.

## Solution Assessment

No, current main is not sufficient. The narrow maintainable repair is provenance-based filtering/tagging, not adding more prompt-text heuristics.
