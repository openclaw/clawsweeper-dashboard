## Summary

Keep this open. Current main still has the reported transcript hygiene gap: WebChat history hides heartbeat-text patterns and internal envelopes, but it does not use `provenance.kind = "inter_session"`, and subagent orphan recovery still sends synthetic resume prompts without `inputProvenance`.

## What This Changes

The issue reports synthetic user-role messages appearing in WebChat after heartbeat wakes, gateway restart, session repair, or subagent diagnostics.

## Best Possible Solution

Add a provenance-based WebChat display contract: tag synthetic orphan-recovery/repair handoff prompts with `inputProvenance`, then hide or system-render internal/inter-session provenance rows while preserving real external user messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass `projectRecentChatDisplayMessages` a `role: "user"` message with `provenance.kind = "inter_session"`; current main leaves it visible unless the text also matches a heartbeat-specific pattern.

## Solution Assessment

Yes for the recommended repair direction, not for current main. The narrow maintainable fix is provenance tagging plus projection/SSE regression tests, rather than broader phrase matching against task text or `[System]` prefixes.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75784. WebChat must not show runtime-generated or inter-session user-role prompts after heartbeat wakeups, gateway restart, session repair, or subagent orphan recovery. Current main filters heartbeat text in `src/gateway/chat-display-projection.ts` but ignores provenance, while `src/agents/subagent-orphan-recovery.ts` sends its synthetic resume prompt through the gateway `agent` method without `inputProvenance`. Implement the narrow fix: use the existing input provenance contract to hide or system-render `inter_session`/`internal_system` user-role rows in WebChat/session-history projections, add `inputProvenance` to orphan-recovery resume calls with a clear `sourceTool` such as `subagent_orphan_recovery`, and preserve real `external_user` messages. Do not add broad phrase matching for arbitrary task text or `[System]` prefixes, and do not change heartbeat scheduling or provider runtime behavior. Add focused regression coverage for `projectRecentChatDisplayMessages`, `SessionHistorySseState` inline append/refresh behavior, and orphan recovery gateway call params. Update `docs/reference/transcript-hygiene.md` and `CHANGELOG.md` if the user-facing behavior changes.
