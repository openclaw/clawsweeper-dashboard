## Summary

Keep this open. Current main still has the same source-level transcript hygiene gap identified in the existing ClawSweeper comment: WebChat projection hides heartbeat/internal text patterns but not inter-session provenance, and orphan recovery still resumes subagents without tagging the synthetic turn as internal/inter-session.

## What This Changes

The issue reports phantom WebChat user messages after heartbeat wakeups, gateway restart/session repair, and subagent diagnostics.

## Best Possible Solution

Propagate existing inter-session/internal provenance onto synthetic orphan-recovery resume prompts and have WebChat/SSE history projection suppress or render those rows as non-user system/internal messages without adding more text-pattern filters.

## Reproduction Assessment

Yes, source-level. Passing `projectRecentChatDisplayMessages` a `role: "user"` message with `provenance.kind: "inter_session"` and ordinary text such as the reported test prompt leaves it visible because the projection has no provenance branch; orphan recovery also creates such synthetic resume turns without provenance.

## Solution Assessment

No, current main is not the best solved state yet. The narrow maintainable fix is to use the existing `inputProvenance` contract in orphan recovery and teach the shared display projection/history path to treat inter-session user rows as internal or non-user, rather than expanding prompt-text heuristics.
