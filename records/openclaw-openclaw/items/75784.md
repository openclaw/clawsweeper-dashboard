## Summary

Keep this open. Current main still has the transcript/display provenance gap: WebChat history uses display projection, but the projection does not hide provenance-tagged inter-session user rows, and subagent orphan recovery still sends a synthetic resume turn without inputProvenance.

## What This Changes

The issue requests preventing phantom synthetic user messages from appearing in WebChat after heartbeat wakes, gateway restarts, and session repair.

## Best Possible Solution

Add a provenance-based WebChat/SSE display contract and tag restart/orphan-recovery resume prompts with existing provenance metadata instead of adding more prompt-text filters.

Yes. A source-level reproduction is to pass projectRecentChatDisplayMessages or buildSessionHistorySnapshot a role:user message with provenance.kind = "inter_session"; current main leaves it visible unless the text also matches heartbeat/internal filters.

## Solution Assessment

No. Current main is incomplete; the narrow maintainable fix is provenance propagation plus display projection/SSE filtering or system rendering for synthetic rows, not broader prompt heuristics.
