## Summary

Keep this open. Current main still has a source-level transcript/display hygiene gap: WebChat and session-history projection do not consult inter-session provenance, and subagent orphan recovery still sends its synthetic resume prompt without `inputProvenance`.

## What This Changes

The issue asks OpenClaw to prevent synthetic heartbeat/restart/session-repair prompts from appearing in WebChat as ordinary user-authored messages.

## Best Possible Solution

Use the existing provenance contract end to end: tag synthetic orphan-recovery resume prompts as inter-session/internal and make shared WebChat/SSE history projection hide or non-user-render provenance-marked internal user rows with focused regression coverage.

## Reproduction Assessment

Yes, source-level. A `role: "user"` transcript row with `provenance.kind: "inter_session"` and ordinary text still passes through `projectRecentChatDisplayMessages`, and orphan recovery creates its synthetic resume turn without provenance.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is to propagate `inputProvenance` through orphan recovery and teach the shared history projection to use provenance instead of adding more prompt-text filters.
