## Summary

Keep this open. Current main still has a concrete WebChat transcript hygiene gap: display projection hides heartbeat-specific text, but not provenance-marked internal user-role rows, and subagent orphan recovery still resumes sessions through `agent` without `inputProvenance`.

## What This Changes

The issue asks OpenClaw to stop WebChat from showing runtime-generated user-role prompts after heartbeat, gateway restart, session repair, or subagent orphan recovery.

## Best Possible Solution

Add a provenance-based WebChat display contract and tag orphan-recovery resume prompts as inter-session/internal input while leaving real external user messages visible.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `projectRecentChatDisplayMessages` a `role: "user"` message with `provenance.kind = "inter_session"`, or an orphan-recovery `[System]` resume prompt, and observe that current main leaves it visible unless it matches heartbeat text.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is provenance tagging plus projection/session-history tests, not broader text matching against task phrases or `[System]` prefixes.
