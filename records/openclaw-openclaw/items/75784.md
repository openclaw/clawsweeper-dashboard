## Summary

Keep this open. Current main still has the reported transcript hygiene gap: the WebChat/SSE display projection does not use inter-session provenance, and orphan recovery still emits a synthetic resume turn without `inputProvenance`.

## What This Changes

The issue asks OpenClaw to prevent synthetic heartbeat, gateway-restart, and session-repair prompts from appearing in WebChat as user-authored messages.

## Best Possible Solution

Use the existing provenance contract end to end: tag orphan-recovery resume prompts as inter-session/internal and make shared chat-history projection hide or system-render provenance-marked internal user rows.

## Reproduction Assessment

Yes, source-level. Passing WebChat history projection a `role: "user"` row with `provenance.kind: "inter_session"` and ordinary text leaves it visible, and orphan recovery currently creates a synthetic resume prompt without provenance.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is to propagate `inputProvenance` through orphan recovery and have shared history projection use provenance instead of adding more prompt-text heuristics.
