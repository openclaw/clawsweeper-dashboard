## Summary

Keep this open. Current main still has a plausible WebChat transcript hygiene gap: the display projection hides older heartbeat/internal-envelope pollution, but it does not inspect inter-session provenance, and orphan-recovery resume prompts are still sent as ordinary agent messages without provenance.

## What This Changes

The issue requests preventing runtime-generated heartbeat, subagent recovery, and session-repair prompts from appearing as user-authored WebChat transcript messages.

## Best Possible Solution

Add a narrow transcript-display contract for runtime-generated user rows: subagent announce, orphan-recovery resume, heartbeat/system-event, and repair-only handoff prompts should carry provenance and then be hidden or rendered as internal/system events in WebChat while preserving ordinary external user turns.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction can construct `chat.history` raw messages with a distinctive user-role inter-session or orphan-recovery prompt and verify `projectRecentChatDisplayMessages` currently leaves it visible unless it matches the older heartbeat/internal-envelope patterns.

## Solution Assessment

No, current main is not the complete solution. The maintainable fix is to tag missing orphan-recovery resumes with provenance and extend the WebChat projection/tests to handle internal provenance instead of broad text matching.
