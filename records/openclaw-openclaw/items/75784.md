## Summary

Keep this open. Current main still has the transcript hygiene gap: WebChat display projection filters heartbeat/internal text patterns but not inter-session provenance, and subagent orphan recovery still sends a synthetic resume turn without `inputProvenance`.

## What This Changes

The issue reports phantom user-role messages appearing in WebChat after heartbeat wakes, gateway restart, session repair, and subagent recovery.

## Best Possible Solution

Add a provenance-based WebChat display contract and tag synthetic orphan-recovery resume prompts as inter-session/internal so they can be hidden or rendered as system rows without masking real user input.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass `projectRecentChatDisplayMessages` a `role: "user"` message with `provenance.kind: "inter_session"`; current main leaves it visible unless its text happens to match heartbeat/internal filters. The orphan-recovery path also creates a synthetic resume prompt without provenance, so it can be persisted as an ordinary user turn.

## Solution Assessment

No, current main is incomplete. The narrow maintainable fix is to propagate existing `inputProvenance` through orphan recovery and teach the display projection/SSE history path to suppress or system-render inter-session user rows, not to add more prompt-text filters.
