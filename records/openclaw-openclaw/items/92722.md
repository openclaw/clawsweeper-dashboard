## Summary

Keep open: the patch is not merge-ready because a home-directory boundary still permits parent context files inside the same home tree, including the exact shared-home case described by the linked security report.

## What This Changes

The PR changes `loadProjectContextFiles()` so ancestor `AGENTS.md` / `CLAUDE.md` discovery stops at or outside `homedir()`.

## Best Possible Solution

Bound project-context discovery to the configured workspace or project root, keep root-to-cwd ordering only inside that boundary, and prove parent files outside the boundary are not injected.

## Reproduction Assessment

Yes, source-level. Current main walks from `cwd` to `/`, and the PR still walks through home-tree parents, so a workspace under `~/.openclaw/workspace` would still load `~/.openclaw/AGENTS.md` and `~/AGENTS.md`.

## Solution Assessment

No. A home boundary is only a partial mitigation; the narrow maintainable fix is a workspace/project-root boundary aligned with the linked report, OpenClaw docs, and Codex project-doc behavior.
