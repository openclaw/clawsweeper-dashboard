## Summary

Keep this PR open, but it is not ready to merge: it addresses a real transcript cleanup bug, yet the current branch targets the old inline reset path and only handles active `MessageThreadId` rather than the full topic-id contract used on current main.

## What This Changes

The branch changes `src/auto-reply/reply/agent-runner.ts` so reset cleanup resolves its fallback transcript path with `sessionCtx.MessageThreadId`.

## Best Possible Solution

Update `resetReplyRunSession` so previous-transcript cleanup resolves topic ids the same way session creation and dispatch do, with focused regression coverage for missing or stale `sessionFile` topic cleanup.

## Reproduction Assessment

Yes. A focused unit test can call `resetReplyRunSession` with `cleanupTranscripts: true`, a topic-scoped session key, and a missing or stale previous `sessionFile`; current main will include the non-topic fallback path and leave the topic-suffixed transcript behind.

## Solution Assessment

No. Passing only the active `MessageThreadId` is a partial fix; the maintainable fix belongs in the current reset helper and should also derive the topic id from the thread-scoped session key.
