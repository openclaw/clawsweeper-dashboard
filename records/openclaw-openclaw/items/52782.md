## Summary

The PR targets a real transcript-cleanup bug, but it is stale and partial against current main: the active cleanup path now lives in the reset helper, and that helper still only adds a non-topic fallback path.

## What This Changes

The PR adds `sessionCtx.MessageThreadId` to the old inline reset transcript cleanup fallback in `runReplyAgent`.

## Best Possible Solution

Update the current reset helper to resolve the cleanup topic id from active message state or the thread-scoped session key, then add focused regression coverage for missing or stale topic transcript cleanup.

## Reproduction Assessment

Yes, source-level. A focused unit test can call `resetReplyRunSession` with `cleanupTranscripts: true`, a topic-scoped `sessionKey`, and a missing or stale previous `sessionFile`; current main adds only the non-topic fallback and leaves the topic-suffixed transcript behind.

## Solution Assessment

No. Passing only the active `MessageThreadId` at the old inline call site is stale and partial; the maintainable fix belongs in the current reset helper and should mirror session creation's active-or-session-key topic resolution.
