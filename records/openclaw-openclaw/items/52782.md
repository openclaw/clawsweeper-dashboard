## What This Changes

The PR branch adds `sessionCtx.MessageThreadId` to the reset cleanup transcript fallback in `src/auto-reply/reply/agent-runner.ts`.

## Best Possible Solution

Update `resetReplyRunSession` to resolve the cleanup topic id from active message state or the thread-scoped session key, then add focused reset-helper coverage for missing or stale topic `sessionFile` cleanup.

Yes, source-level. A focused unit test can call `resetReplyRunSession` with `cleanupTranscripts: true`, a topic-scoped `sessionKey`, and a missing or stale previous `sessionFile`; current main adds only the non-topic fallback and leaves the topic-suffixed transcript behind.

## Solution Assessment

No. Passing only the active `MessageThreadId` at the old call site is partial and stale; the maintainable fix belongs in the current reset helper and should use the same active-or-session-key topic resolution as session creation and dispatch.
