## Summary

Keep open: the topic-scoped cleanup bug is real, but this branch patches an obsolete inline cleanup site and still misses the current reset helper and session-key-derived topic cases; it also lacks after-fix real behavior proof.

## What This Changes

The PR adds the active `sessionCtx.MessageThreadId` as the optional topic id when resolving the old transcript cleanup fallback in `runReplyAgent`.

## Best Possible Solution

Move topic-aware cleanup into `resetReplyRunSession`, derive the cleanup topic id from the active `MessageThreadId` or parsed topic session key while preserving string and numeric ids, and add focused regression coverage plus redacted runtime proof.

## Reproduction Assessment

Yes, source-level. On current main, `resetReplyRunSession` with `cleanupTranscripts: true`, a topic-scoped session key, no active `messageThreadId`, and a missing or stale `sessionFile` still adds only the bare transcript fallback.

## Solution Assessment

No. The patch should target `resetReplyRunSession` and mirror session creation's active-or-session-key topic resolution, not only add `MessageThreadId` to the old inline `runReplyAgent` path.
