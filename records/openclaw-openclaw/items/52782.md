## Summary

Keep open. The PR still targets the old inline cleanup fallback in `runReplyAgent`, while current `main` performs this cleanup in `resetReplyRunSession`; the helper still adds a bare transcript fallback and the PR also lacks after-fix real Telegram/topic proof.

## What This Changes

The PR adds `sessionCtx.MessageThreadId` as the optional topic id when `runReplyAgent` resolves a transcript cleanup fallback path.

## Best Possible Solution

Retarget topic-aware cleanup into `resetReplyRunSession`, derive the cleanup topic id from the active message thread or parsed session key while preserving string and numeric ids, add focused regression coverage, and include redacted Telegram/topic runtime proof.

## Reproduction Assessment

Yes, source-level. On current `main`, `resetReplyRunSession` with `cleanupTranscripts: true`, a topic-scoped session key or numeric topic id, and a missing or stale stored `sessionFile` still adds only the bare transcript fallback.

## Solution Assessment

No. The PR patches the old inline `runReplyAgent` cleanup site, but current `main` should solve this in `resetReplyRunSession` and mirror session initialization's active-or-session-key topic resolution.
