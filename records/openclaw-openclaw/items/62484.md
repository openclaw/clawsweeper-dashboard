Keep open: current main and the latest release still carry the heavy config facade on the CLI help banner import path, and the same-repo fix PR was closed unmerged while the remaining fork PR is not a proven safe canonical fix.

## What This Changes

The issue reports that `openclaw tui -h` hangs at 100% CPU and cannot be interrupted normally because CLI help banner generation pulls in config synchronously.

## Best Possible Solution

Land or replace this with a narrow CLI help/banner fix that removes heavy config imports from help-time banner generation, preserves `cli.banner.taglineMode` for normal banners, and includes terminal proof plus focused regression coverage.

## Reproduction Assessment

Yes at source level: `tui` help is not a precomputed fast path, and current help/banner imports still pull in `banner-config-lite.ts`, which imports and synchronously reads the full config facade. I did not run the hanging command live in this read-only review.

## Solution Assessment

Yes for the requested direction, but not for the existing stale branch as-is. The maintainable fix should isolate help-time banner rendering from heavy config imports without dropping existing banner tagline settings.
