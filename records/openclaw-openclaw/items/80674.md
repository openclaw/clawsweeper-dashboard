## Risks / Open Questions

- Public hook names, event taxonomy, ordering, timeout behavior, and hook failure policy are not yet designed.
- Persistence hook payloads could expose session keys, local file paths, lock timing, transcript metadata, or conversation-derived state to plugins without a settled privacy and permission contract.
- Polling hooks may need to be generic channel lifecycle telemetry instead of Telegram-only events, but that abstraction is not settled.
- The request overlaps with adjacent lifecycle work in https://github.com/openclaw/openclaw/issues/43454 and session reset work in https://github.com/openclaw/openclaw/issues/11960, so maintainers should avoid creating competing hook surfaces.
