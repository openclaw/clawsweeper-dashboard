## Summary

Keep open: this is a real plugin API gap, not stale cleanup. Current main has the Telegram polling watchdog, session-store write path, transcript rewrite/rotation code, and existing hook infrastructure, but no plugin-observable polling lifecycle or session persistence hook families.

## What This Changes

The issue requests new plugin-observable polling lifecycle and session persistence hook event families.

## Best Possible Solution

Define a stable, generic plugin hook contract for channel liveness and session persistence telemetry, then wire Telegram and session-store/transcript emitters with focused tests and public plugin docs.
