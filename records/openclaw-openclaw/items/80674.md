## Summary

Keep open: current main still has the Telegram polling watchdog, session-store writes, and transcript rewrite/rotation seams, but it does not expose the requested plugin-observable polling lifecycle or session persistence hook families. This is a real extensibility gap and a new public plugin API contract, so it needs maintainer design rather than cleanup closure.

## What This Changes

Request new plugin hook families for polling watchdog lifecycle events and session persistence/transcript save, rewrite, and rotation events.

## Best Possible Solution

Define a stable generic plugin hook contract for channel polling/liveness and session persistence telemetry, then wire Telegram and session-store emitters with focused tests and public docs.

## Reproduction Assessment

Not applicable; this is a feature request for new plugin-observable lifecycle events, not a broken existing behavior. Source inspection gives high confidence that the named seams exist and are not exposed as hook families on current main.

## Solution Assessment

Unclear; the requested hook direction is plausible, but the best implementation needs maintainer decisions on generic event names, payload stability, privacy, and hook failure policy before code is queued.
