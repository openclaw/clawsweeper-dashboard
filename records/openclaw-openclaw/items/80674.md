## Summary

Keep open: current main still has the Telegram polling watchdog, session-store save, and transcript rewrite/rotation seams, but the requested polling lifecycle and session persistence hook families are not exposed as plugin hooks. This is a valid public plugin API feature request, not a cleanup-close candidate.

## What This Changes

The issue requests new plugin-observable hook families for polling lifecycle and session persistence events.

## Best Possible Solution

Design a stable public hook contract for channel polling/liveness and session persistence telemetry, document privacy and failure semantics, then wire Telegram/session emitters with focused contract tests.

Unclear; the hook direction is plausible, but the best implementation needs maintainer decisions on generic event names, payload privacy/stability, Telegram-specific versus channel-generic scope, and hook failure policy.
