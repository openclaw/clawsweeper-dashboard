## Summary

Keep open. Current main has adjacent hook, transcript-update, and Telegram polling liveness surfaces, but it still does not expose the requested plugin-observable polling lifecycle or session-persistence hook families; this remains a public plugin API and security/timing design decision.

## What This Changes

Expose plugin-observable polling lifecycle and session persistence hook events so operators can instrument Telegram watchdog/restart behavior and session/transcript writes from in-process plugins.

## Best Possible Solution

Design a narrow, documented plugin hook contract with redacted payloads, nonblocking/timeout-bounded dispatch, and focused tests across Telegram polling, session-store saves, and transcript archive/rewrite paths if maintainers approve the API direction.

## Reproduction Assessment

Yes for the feature gap by source inspection. Current main has adjacent compaction, reset, message, transcript-update, and Telegram liveness surfaces, but no requested `polling_lifecycle` or `session_persistence` hook family names or dispatches.

## Solution Assessment

Unclear until maintainer review. Adding narrow plugin hooks is plausible and fits the plugin API direction when plugins cannot observe this today, but payload redaction, timing, and nonblocking dispatch need design before implementation.
