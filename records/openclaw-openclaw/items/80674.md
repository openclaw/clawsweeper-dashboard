## Summary

Keep open: current main still has the named Telegram polling, session-store write, and transcript rotation seams, and neither the plugin nor internal hook catalogs expose the requested observability events. This is a real public plugin API gap that needs maintainer design rather than cleanup closure.

## What This Changes

The issue requests new plugin hook families that expose polling watchdog lifecycle events and session persistence/transcript rewrite boundaries.

## Best Possible Solution

Define a stable generic plugin hook contract for channel polling/liveness and session persistence telemetry, then wire Telegram and session-store emitters with focused tests and public docs.

## Reproduction Assessment

Not applicable; this is a request for new plugin-observable lifecycle events, not a broken existing behavior. Source inspection gives high confidence that the named seams exist and are not exposed as hook families on current main.

## Solution Assessment

Unclear; the hook-family direction is plausible, but the best implementation needs maintainer decisions on generic event names, payload privacy, stability, and hook failure policy before code is queued.
