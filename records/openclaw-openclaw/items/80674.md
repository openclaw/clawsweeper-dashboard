## Summary

Keep open: current main still has the Telegram polling watchdog, session-store writes, transcript rewrite/rotation, and existing plugin hook infrastructure, but it does not expose the requested polling lifecycle or session persistence hook families. This is a real plugin API gap and should be designed by maintainers rather than closed as cleanup.

## What This Changes

Expose plugin hook events for Telegram/channel polling watchdog lifecycle and session persistence/transcript write and rotation boundaries.

## Best Possible Solution

Define a stable, generic plugin hook contract for channel polling/liveness and session persistence telemetry, then wire Telegram and session-store emitters with focused tests and public docs.

## Reproduction Assessment

Not applicable; this is a feature request for new plugin-observable lifecycle events, not a broken existing behavior. Source inspection shows the named seams exist and are not exposed as hook families on current main.

## Solution Assessment

Unclear; the requested direction is plausible, but the best implementation needs maintainer decisions on generic channel coverage, event names, payload redaction, and hook failure policy. A direct narrow fix PR before that API decision would be premature.
