## Summary

Keep open: current main still has the relevant Telegram polling watchdog, session-store write, transcript append/rotation, and existing hook infrastructure, but it does not expose the requested polling lifecycle or session persistence plugin hook families. This is a real plugin API gap, not stale cleanup.

## What This Changes

The issue requests new plugin-observable polling lifecycle and session persistence hook event families.

## Best Possible Solution

Design and document a stable plugin hook contract for generic channel/polling liveness and session persistence telemetry, then wire Telegram and session write/transcript paths behind that contract.

## Reproduction Assessment

Not applicable; this is a feature request for new plugin-observable lifecycle events, not a broken existing behavior. Source inspection shows the named operational seams exist but are not exposed as hook families on current main.

## Solution Assessment

Unclear; the requested direction is plausible, but maintainers need to settle event naming, generic channel coverage, stable payloads, privacy, and hook failure policy before implementation is the best path.
