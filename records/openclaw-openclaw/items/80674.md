## Summary

Keep open. Current main and v2026.6.8 expose adjacent compaction, session lifecycle, and transcript-update surfaces, but they still do not expose the requested plugin-observable `polling_lifecycle` or `session_persistence` hook families; the remaining work is a public plugin API and security/timing design decision.

## What This Changes

Expose plugin-observable polling lifecycle and session persistence hook events around Telegram/channel polling and session/transcript save, rewrite, and rotate seams.

## Best Possible Solution

Approve a minimized, observation-only public hook contract with bounded payloads and timeouts, then add focused Telegram/channel polling and session persistence events with docs and regression coverage after security review.

Unclear pending maintainer API design. The requested hook families are plausible, but the best implementation needs a narrowed owner boundary, payload contract, timeout behavior, and security model first.
