## Summary

Keep open: this protected-label PR contains a useful legacy-upgrade install repair that is not already on current main, and I found no blocking correctness issue in the diff.

## What This Changes

The PR adds an explicit current-updater deferral marker so legacy package-update doctor passes repair configured externalized plugin installs immediately while current update flows still defer to post-core convergence.

## Best Possible Solution

Land the explicit defer-marker repair after required checks, preserving post-core convergence for current updaters while letting legacy 2026.4.x handoffs repair configured externalized plugins before restart.

## Reproduction Assessment

Yes for the source path: current main still defers configured plugin repair when a legacy updater supplies only `OPENCLAW_UPDATE_IN_PROGRESS`. I did not run the packaged upgrade lane in this read-only review, but the PR body includes a before/after Crabbox upgrade-survivor run.

## Solution Assessment

Yes: the explicit defer marker is the narrowest maintainable split between current update flows that have post-core convergence and legacy update handoffs that must repair immediately. The npm preference is scoped to the legacy handoff and only applies when an npm spec exists.
