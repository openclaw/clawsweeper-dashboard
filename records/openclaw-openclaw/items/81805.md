## Label Justifications

- `P2`: This is a real performance regression fix attempt with limited blast radius, but it changes channel/secrets control-plane behavior and needs design review.
- `merge-risk: 🚨 compatibility`: The new env mode changes doctor and secret contract semantics for operators who enable it, which can make existing configure/audit expectations diverge.
- `merge-risk: 🚨 security-boundary`: The secrets loader shortcut can suppress channel credential target discovery from audit/configure registry flows under the opt-in mode.

## Summary

Keep open: the status timing problem is plausible and the PR body has real after-fix command output, but the shortcut is too low in shared contract loaders and can hide bundled channel secret targets from audit/configure flows while doctor scans can still fall back to full plugin loading.

## What This Changes

The PR adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` env gate to bundled channel doctor and secrets public-artifact loaders, plus a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` and other cold control-plane discovery on metadata or light artifacts, while preserving complete channel secret and doctor contracts for configure, audit, doctor, and runtime flows; coordinate any broader channel lazy-loading policy with https://github.com/openclaw/openclaw/issues/65444.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: with the env var set, the changed loaders return `undefined`, and current callers either omit secret target entries or fall through to bootstrap plugin loading. The original timing problem is supported by the PR body's real command output, but I did not rerun that 4.27 install locally.

## Solution Assessment

No. The safer fix is a metadata/light-artifact status path or a targeted caller option, not a shared contract-loader shortcut that changes audit/configure and doctor semantics.
