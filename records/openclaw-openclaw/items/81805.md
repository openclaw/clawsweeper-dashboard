## Summary

Keep open. The performance problem is plausible and the PR body includes real after-fix timing proof, but the proposed env-var shortcut is placed too low in shared channel contract loaders and would change secrets and doctor behavior outside the narrow status cold path.

## What This Changes

The PR adds `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` to short-circuit bundled channel doctor and secrets contract artifact loaders, plus a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` on a metadata/read-only path that avoids bundled channel artifact materialization while preserving full channel secret and doctor contracts for audit, configure, doctor, and runtime flows.

## Reproduction Assessment

Yes for the introduced regression by source inspection: with the proposed env var set, bundled channel contracts return `undefined`, and current registry/status callers silently lose channel secret entries. The original timing failure was not independently run here, but the PR body gives a real setup and before/after timing.

## Solution Assessment

No. The proposed shortcut is a shared loader bypass; a narrower status-scoped lazy path or metadata fallback would avoid the cold-start cost without changing secret audit/configure and doctor behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve channel secret targets in lazy mode:** `src/secrets/channel-contract-api.ts:33-35`
  - body: Returning `undefined` from the bundled channel secret contract loader also affects status/configure/audit target discovery: current callers only push `contractApi?.secretTargetRegistryEntries`, so bundled channel credentials can disappear under the new env var. Scope the lazy path to status discovery or provide a metadata fallback before skipping artifact loading.
  - confidence: 0.9
- **[P3] Avoid full plugin fallback in doctor lazy mode:** `src/channels/plugins/doctor-contract-api.ts:25-27`
  - body: When the doctor contract loader returns `undefined`, `collectChannelLegacyConfigRules` falls through to `getBootstrapChannelPlugin`, which can load full bundled channel plugin surfaces. That defeats the stated lazy-mode goal for doctor/legacy-config scans, so the doctor path needs a light fallback or should remain contract-backed.
  - confidence: 0.82
