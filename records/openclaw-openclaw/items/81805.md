## Summary

Keep open: the real timing proof is useful, but the patch is not merge-ready because the opt-in gate sits in shared contract loaders, where it can hide bundled channel secret targets from configure/audit flows and can make doctor scans fall back to runtime plugin loading. The branch also conflicts with current main, which has moved the secrets contract surface forward since this PR was opened.

## What This Changes

The PR adds an `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` opt-in shortcut to bundled channel doctor/secrets artifact loaders and documents the mode in the changelog.

## Best Possible Solution

Keep `openclaw status --json` and similar cold control-plane discovery on targeted metadata/light-artifact paths while preserving complete channel secret, audit, and doctor contracts; coordinate any broader lazy-channel operator mode with https://github.com/openclaw/openclaw/issues/65444.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: with the env var set, the changed loaders return `undefined`, and current callers either omit secret target entries or fall through to bootstrap plugin loading. The original 4.27 timing problem is supported by the PR body's copied live output, but I did not rerun that install locally.

## Solution Assessment

No. The safer solution is a targeted status/control-plane metadata path or a clearly designed lazy-channel contract, not a shared low-level contract-loader shortcut that changes audit/configure and doctor behavior.
