## Summary

Keep this PR open, but it is not ready to merge: the proposed shortcut is placed below shared channel contract loaders, so the opt-in mode can remove channel secret targets and can still fall back into full bundled channel plugin loading on doctor/legacy-config paths.

## What This Changes

This PR adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` shortcut to bundled channel doctor and secret contract artifact loaders, plus a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` on a lazy metadata/read-only path while preserving complete channel secret and doctor contracts for audit, configure, doctor, and runtime flows.

## Reproduction Assessment

Yes for the introduced regression by source inspection: with the proposed env var set, bundled channel contract APIs return `undefined`, and existing registry callers lose channel secret entries or fall through to full plugin loading. The original 4.27 timing failure was supported by PR-body live-output evidence but not independently rerun here.

## Solution Assessment

No. The current shortcut is too low in the shared contract loader; a safer fix would be status-scoped lazy target resolution or a lightweight metadata fallback that does not change secret audit/configure and doctor behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve channel secret targets in lazy mode:** `src/secrets/channel-contract-api.ts:32-33`
  - body: Returning `undefined` here also affects `getSecretTargetRegistry()` and status/configure/audit target discovery: those paths only push `contractApi?.secretTargetRegistryEntries`, so bundled channel targets can disappear under the new env var. Scope the shortcut to the status cold path or provide a metadata fallback before skipping artifact loading.
  - confidence: 0.9
- **[P3] Keep doctor lazy mode off full plugin fallback:** `src/channels/plugins/doctor-contract-api.ts:24-25`
  - body: When this loader returns `undefined`, `collectChannelLegacyConfigRules` falls back to `getBootstrapChannelPlugin`, which can materialize the full bundled channel plugin. That defeats the lazy-mode goal for doctor/legacy-config scans; keep the doctor contract loadable or add a light fallback there too.
  - confidence: 0.82
