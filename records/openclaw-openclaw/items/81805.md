## Summary

Keep open: the performance problem is credible and the PR has useful real-output proof, but the current patch is not merge-ready because the shared lazy shortcut can hide channel secret targets and can still fall back to runtime channel plugin loading in doctor paths.

## What This Changes

Adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` shortcut to bundled channel doctor/secrets contract loaders and a `CHANGELOG.md` entry.

## Best Possible Solution

Keep status/control-plane discovery on metadata or light artifacts while preserving complete secret and doctor contracts for configure, audit, doctor, and runtime flows; drop the normal PR changelog edit and add focused env-gated coverage if maintainers accept the new env/API surface.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: with the env var set, the changed loaders return `undefined`, and current callers either omit secret target entries or fall through to bootstrap plugin loading. The original timing problem is supported by the PR body's copied real command output, but I did not rerun that 4.27 install locally.

## Solution Assessment

No. The PR is a plausible mitigation, but the shared contract-loader shortcut is the wrong layer because it changes secrets and doctor semantics; a targeted status/control-plane metadata path is safer.
