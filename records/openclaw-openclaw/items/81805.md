## Label Justifications

- `P2`: This is a focused performance regression fix attempt with limited blast radius, but it changes channel/secrets control-plane behavior and needs design review.
- `merge-risk: 🚨 compatibility`: The new opt-in mode changes doctor and secret contract semantics for operators who enable it, which can make configure/audit expectations diverge.
- `merge-risk: 🚨 security-boundary`: The secrets loader shortcut can suppress channel credential target discovery from audit/configure registry flows under the opt-in mode.

## Summary

Keep open. The performance target is credible and the PR includes useful live output, but the shared env-gate changes secrets and doctor contract semantics under the opt-in mode: it can hide bundled channel credential targets and still fall back to full runtime plugin loading.

## What This Changes

Adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` env gate to bundled channel doctor/secrets public-artifact loaders plus a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` and other cold control-plane discovery on metadata or light artifacts while preserving complete channel secret and doctor contracts; coordinate broader lazy-channel policy with https://github.com/openclaw/openclaw/issues/65444.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: with the env var set, the changed loaders return `undefined`, and current callers either omit secret target entries or fall through to bootstrap plugin loading. The original timing problem is supported by the PR body's live output, but I did not rerun that 4.27 install locally.

## Solution Assessment

No. The safer fix is a metadata/light-artifact status path or targeted caller option, not a shared contract-loader shortcut that changes audit/configure and doctor semantics.
