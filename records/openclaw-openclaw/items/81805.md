## Summary

Keep open: the performance problem is plausible and the PR includes real after-fix output, but the shared loader shortcut can drop channel secret targets and can defeat doctor lazy mode by falling back to full bundled plugin loading.

## What This Changes

The PR adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` env gate to bundled channel doctor and secrets contract loaders plus a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` on a metadata/read-only path while preserving complete channel secret and doctor contracts for audit, configure, doctor, and runtime flows.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: with the env var set, the changed loaders return `undefined`, and current callers either omit channel secret entries or fall back to bootstrap plugin loading. The original 4.27 timing issue is supported by PR-body live output but was not independently rerun.

## Solution Assessment

No. The shortcut is placed too low in shared contract loaders; the safer solution is a status/control-plane metadata path that avoids heavy imports without changing audit/configure and doctor semantics.
