## Summary

Keep open: the PR targets a plausible cold-start regression, but the low-level secrets artifact shortcut would also remove bundled channel secret targets from the registry whenever the env var is set.

## What This Changes

The PR adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` short-circuit to bundled channel doctor/secret contract artifact loaders and adds a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` on a metadata/read-only path while preserving complete channel secret and doctor contracts for audit, configure, doctor, and runtime flows.

## Reproduction Assessment

Yes for the introduced regression by source inspection: with the proposed env var set, bundled channel contract APIs return `undefined` and the global secret registry drops those channel entries. The original 4.27 timing failure was not independently reproduced here.

## Solution Assessment

No. The shortcut is placed too low in the shared contract loader; the safer fix is a status-scoped lazy path or metadata fallback that does not change secret audit/configure and doctor behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve channel secret targets in lazy mode:** `src/secrets/channel-contract-api.ts:53-55`
  - body: Returning `undefined` from the generic bundled channel contract loader also affects `getSecretTargetRegistry()`: `target-registry-data.ts` only pushes `contractApi?.secretTargetRegistryEntries`, so with this env var enabled bundled channel targets such as Discord or Telegram are omitted from secrets audit/configure and command target discovery. Please scope the shortcut to the status cold path or add a metadata/read-only fallback before skipping artifact loading.
  - confidence: 0.9
- **[P3] Avoid full plugin fallback in doctor lazy mode:** `src/channels/plugins/doctor-contract-api.ts:24-26`
  - body: The doctor wrapper now returns `undefined` under the lazy env var, but `collectChannelLegacyConfigRules` treats that as a missing light contract and falls back to `getBootstrapChannelPlugin`, which can load the full bundled channel plugin. That defeats the lazy-mode guarantee for doctor/legacy-config scans; either leave doctor contracts loadable or add a light fallback there too.
  - confidence: 0.82
