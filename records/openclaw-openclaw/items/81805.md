## Label Justifications

- `P2`: This is a focused performance regression fix attempt with real user impact, but it changes channel/secrets control-plane behavior and needs review before merge.
- `merge-risk: 🚨 compatibility`: The opt-in env mode changes doctor and secrets contract semantics for operators who enable it, which can make existing configure/audit expectations diverge.
- `merge-risk: 🚨 security-boundary`: The secrets loader shortcut can suppress bundled channel credential target discovery from audit/configure registry flows under the opt-in mode.

## Summary

Keep open: the PR targets a plausible cold-start status timeout and includes sufficient real-output proof, but the current patch still hides bundled channel secret targets under the opt-in mode and can defeat its own doctor lazy path through full runtime fallback.

## What This Changes

The branch adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` short-circuit to bundled channel doctor and secrets contract loaders, plus a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` on metadata or light contract artifacts while preserving complete channel secret, security, and doctor contracts; coordinate broader lazy-channel policy with https://github.com/openclaw/openclaw/issues/65444.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: with the env var set, the changed loaders return `undefined`, and current callers either omit secret target entries or fall through to bootstrap plugin loading. The original timing issue is supported by the PR body's live command output, but I did not rerun the 4.27 install locally.

## Solution Assessment

No. The safer solution is a metadata/light-artifact status path or targeted caller option, not a shared contract-loader shortcut that changes audit/configure and doctor semantics.
