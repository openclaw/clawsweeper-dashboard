## Summary

Keep open: the performance problem is plausible and the PR has useful real-output proof, but the shared env-gated shortcut changes secrets and doctor control-plane behavior in ways that need repair or maintainer design approval before merge.

## What This Changes

Adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` gate to bundled channel doctor and secrets artifact loaders, plus a changelog entry.

## Best Possible Solution

Keep `openclaw status --json` and cold discovery on a light metadata or typed-contract path while preserving complete secrets and doctor contracts for configure, audit, doctor, and runtime flows; coordinate broader lazy-channel lifecycle policy with https://github.com/openclaw/openclaw/issues/65444.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: with the env var set, the changed loaders return `undefined`, and current callers either omit secret target entries or fall back to full bootstrap plugin loading. The original timing problem is supported by the PR body's real command output, but I did not rerun that 4.27 install.

## Solution Assessment

No. The safer solution is a metadata/light-artifact status path or a targeted caller option, not a shared contract-loader shortcut that changes audit/configure and doctor semantics.
