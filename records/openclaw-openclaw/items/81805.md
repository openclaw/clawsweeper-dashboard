## Label Justifications

- `P2`: This is a focused performance fix attempt with real user impact, but it changes channel/secrets control-plane behavior and has limited opt-in blast radius.
- `merge-risk: 🚨 compatibility`: The new env mode changes doctor and secret contract semantics for operators who enable it, which can diverge from existing configure/audit expectations.
- `merge-risk: 🚨 security-boundary`: The secrets loader shortcut can suppress channel credential target discovery from audit/configure/security registry flows under the opt-in mode.

## Summary

Keep open: the real behavior proof supports the status timeout symptom, but this patch changes shared secrets and doctor contract loaders. In the opt-in mode it can omit bundled channel secret target registry entries and can still fall back to full runtime plugin loading, while current main already has a lean `status --json` fast path for the central CLI discovery case.

## What This Changes

Adds an opt-in `OPENCLAW_LAZY_BUNDLED_CHANNEL_ARTIFACTS=1` gate to bundled channel doctor and secrets public-artifact loaders, plus a changelog entry.

## Best Possible Solution

Keep `status --json` on the existing fast metadata path; if doctor or secrets still need a lazy contract mode, add a caller-scoped metadata/light-artifact path that preserves complete secret and doctor contracts and coordinate broader startup behavior with https://github.com/openclaw/openclaw/issues/65444.

## Reproduction Assessment

Yes for the PR-introduced issues by source inspection: setting the env var makes the changed loaders return `undefined`, and current callers either omit secret targets or fall back to runtime plugin loading. The original timing symptom is supported by copied live output in the PR body, but I did not rerun that 4.27 install in this read-only review.

## Solution Assessment

No. The shared loader env gate is broader than the status problem and changes secrets/doctor semantics; a metadata/light-artifact path or caller-scoped status optimization is the narrower maintainable solution.
