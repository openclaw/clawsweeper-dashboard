## Summary

Keep open. The config knob is plausible, but the patch currently regresses current main's Slack non-DM progress suppression, does not apply the new false setting to every promised intermediate message path, and lacks required real behavior proof.

## What This Changes

The PR adds a global `messages.toolMessageLogging` boolean, wires it through config schema/help/types/docs, and adds dispatch tests for tool-summary delivery suppression and opt-in.

## Best Possible Solution

Rebase onto current main, preserve existing channel defaults, apply one effective tool-message/progress gate consistently, document the key on the current messages docs page, and add focused tests plus redacted real chat proof.

## Reproduction Assessment

Yes for the review findings by source inspection: current main suppresses Slack non-DM progress at `dispatch-from-config.ts:1034`, while the PR's default ignores that gate, and `sendPlanUpdate` does not consult the new override. I did not run the PR branch because this review is read-only and the PR commit is not present in the local checkout.

## Solution Assessment

No. A config override may be a maintainable solution, but this implementation should preserve current channel-specific defaults, use the override consistently across default intermediate messages, and update the active docs location before merge.
