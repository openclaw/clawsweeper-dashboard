## Summary

## What This Changes

The issue asks for memory-core dreaming cron marker payloads to be handled internally instead of being interpreted as shell commands during scheduled agent runs.

## Best Possible Solution

Keep the existing deep-promotion hook gate, then add a narrow memory-core/doctor reconciliation that removes or canonicalizes stale light/REM marker cron jobs and covers agentTurn-message legacy records with regression tests.

No live repro was run. Source inspection gives a medium-confidence path: reported light/REM agentTurn marker jobs are not matched by the legacy cleanup that reads payload.text, and the light/REM trigger helper only handles heartbeat triggers, not cron triggers.

## Solution Assessment

No, the broad requested agentTurn bypass is not the best fix. The maintainable fix is to repair or remove stale memory-core marker cron jobs and keep generic agentTurn cron jobs as real agent tasks.
