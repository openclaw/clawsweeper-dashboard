## What This Changes

The branch adds `openai-audio-transcriptions` to the OpenAI bearer-auth allow-list and updates media-understanding tests so explicit OAuth profiles and auto audio selection can use OAuth for OpenAI batch transcription.

## Best Possible Solution

Yes. Current-main source inspection shows OpenAI audio passes `openai-audio-transcriptions` into `resolveApiKeyForProvider`, where OAuth is rejected for that model API; the linked issue provides the matching failing configuration and error.

## Solution Assessment

No as-is. The shared auth predicate is a plausible owner boundary, but the PR is too broad because it changes shipped mixed-auth API-key precedence instead of limiting OAuth restoration to explicit or OAuth-only audio cases.
