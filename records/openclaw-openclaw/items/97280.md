## Summary

Keep open: the patch restores the reported default OpenAI OAuth audio path with strong proof, but the modelApi-only bearer-auth allow-list would also let OpenAI OAuth tokens be sent to configured custom audio base URLs, which is a security/auth-provider blocker before merge.

## What This Changes

The branch adds `openai-audio-transcriptions` to OpenAI bearer-auth eligibility, preserves env API-key precedence over default OAuth profiles, and updates focused auth and media-understanding tests.

## Best Possible Solution

Allow OAuth-backed batch STT only when the audio request is bound to the native OpenAI endpoint, keep custom/OpenAI-compatible audio endpoints API-key-only, and then merge after maintainer acceptance of the restored contract.

## Solution Assessment

No, not yet. The shared auth predicate is a good owner boundary for default OpenAI audio, but a modelApi-only decision is too broad because the actual endpoint/baseUrl is selected in media-understanding request config.
