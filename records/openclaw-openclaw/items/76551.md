## Summary

Keep open; current main still has a slot-blind stale-plugin repair path that can remove `plugins.allow` and `plugins.entries` while leaving a fatal `plugins.slots.memory` reference behind. The open sibling #76550 tracks the same recovery gap from the slot-cleanup side, so these should be fixed together rather than closed pre-merge.

## What This Changes

The issue asks `openclaw doctor --fix` to keep stale plugin cleanup consistent with `plugins.slots.*` so it does not leave invalid slot references after removing allowlist and entry references.

## Best Possible Solution

Make stale plugin repair slot-aware and atomic: either recover the selected plugin before cleanup, or remove matching stale `plugins.slots.*` references in the same repair so the resulting config validates.

## Reproduction Assessment

Yes, source-reproducible. A config with a missing plugin in `plugins.allow`, `plugins.entries`, and explicit `plugins.slots.memory` follows code that removes allow/entry refs but leaves the slot, after which validation reports `plugins.slots.memory: plugin not found`.

## Solution Assessment

Yes, the requested repair boundary is appropriate. The narrow maintainable fix is to extend the existing stale-plugin doctor repair to handle slot references consistently, instead of changing plugin install mechanics or adding a new config mode.

## Repair Work Prompt

Fix the stale plugin doctor repair gap reported in #76551 and sibling #76550. In `src/commands/doctor/shared/stale-plugin-config.ts`, make stale plugin scanning/repair account for `plugins.slots.memory` and `plugins.slots.contextEngine` when those slot values reference a missing plugin id. Preserve existing behavior for valid plugins, plugin discovery errors, globally disabled plugins, and channel cleanup. Prefer an atomic repair result: after `doctor --fix`, a plugin id removed from `plugins.allow`/`plugins.entries` must not remain in `plugins.slots.*`; alternatively, if the existing configured-plugin install repair can recover it, avoid stale removal. Add regression coverage for allow+entry+slot matching and slot-only missing plugin behavior, update `docs/cli/doctor.md` and `CHANGELOG.md` if user-facing repair scope/messages change, and do not change the broader plugin install package mechanics from #76441.
