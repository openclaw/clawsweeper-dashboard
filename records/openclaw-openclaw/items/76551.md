## Summary

Keep open; current main still has a slot-blind stale-plugin repair path that can remove `plugins.allow` and `plugins.entries` while leaving a fatal `plugins.slots.memory` reference behind. The open sibling #76550 covers the complementary slot-cleanup gap, so the same narrow doctor repair should address both.

## What This Changes

The issue asks `doctor --fix` to keep stale plugin cleanup consistent with `plugins.slots.*` references so the repaired config remains valid.

## Best Possible Solution

Make stale-plugin doctor repair slot-aware and atomic, using the existing uninstall-style slot reset or cleanup behavior so the post-repair config validates.

## Reproduction Assessment

Yes, source-reproducible. A config with a missing plugin in `plugins.allow`, `plugins.entries`, and explicit `plugins.slots.memory` follows cleanup that removes allow/entry refs but never scans or mutates slots, after which validation reports `plugins.slots.memory: plugin not found`.

## Solution Assessment

Yes, the requested repair boundary is the narrow maintainable path. The fix should extend the existing stale-plugin doctor repair to handle slot references consistently, not change plugin install mechanics or add a config mode.
