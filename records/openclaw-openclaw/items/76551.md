## Summary

Keep open; current main still has a slot-blind stale-plugin repair path that can remove `plugins.allow` and `plugins.entries` while leaving a fatal explicit `plugins.slots.memory` reference behind. The open sibling #76550 covers the complementary slot cleanup gap, so the two should be fixed together rather than swept closed.

## What This Changes

Make `openclaw doctor --fix` handle stale plugin cleanup consistently when `plugins.slots.*` still references the same plugin.

## Best Possible Solution

Make stale plugin repair slot-aware and atomic: either keep slot-selected plugins from being pruned, or remove matching stale slot references in the same repair so the resulting config validates.

## Reproduction Assessment

Yes, by source inspection. A config with a missing plugin in `plugins.allow`, `plugins.entries`, and explicit `plugins.slots.memory` follows repair code that removes allow/entry refs but leaves the slot, after which validation reports `plugins.slots.memory: plugin not found`.

## Solution Assessment

Yes, the requested repair boundary is the right one. The narrow maintainable fix is to extend the existing stale-plugin doctor repair to account for `plugins.slots.*` consistently, not to change plugin install policy or add a new config option.

## Repair Work Prompt

Fix the doctor stale-plugin repair bug tracked by https://github.com/openclaw/openclaw/issues/76551 together with the sibling slot cleanup gap in https://github.com/openclaw/openclaw/issues/76550. In `src/commands/doctor/shared/stale-plugin-config.ts`, make stale plugin repair account for `plugins.slots.*` atomically: do not leave a slot pointing at a plugin after removing that plugin from `plugins.allow` or `plugins.entries`; either treat the slot as active use when the plugin can/should be preserved, or remove matching stale slot references when the plugin is truly missing and cleanup is the chosen repair. Keep the behavior generic for plugin slots; do not add owner-specific plugin ids or install-policy changes. Add focused regression coverage in `src/commands/doctor/shared/stale-plugin-config.test.ts` and update validation coverage in `src/config/config.plugin-validation.test.ts` if needed.
