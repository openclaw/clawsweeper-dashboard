## Summary

Keep open; current main still has a slot-blind stale-plugin doctor repair path that can remove `plugins.allow` and `plugins.entries` while leaving an explicit `plugins.slots.memory` reference that validation treats as fatal.

## What This Changes

`openclaw doctor --fix` should repair stale plugin config atomically when `plugins.slots.*` still references the same missing plugin.

## Best Possible Solution

Make stale plugin repair slot-aware and atomic, preferably reusing the existing plugin uninstall slot-reset semantics so doctor leaves the config valid after removing a missing selected plugin.

## Reproduction Assessment

Yes, source-reproducible. A missing plugin id present in `plugins.allow`, `plugins.entries`, and explicit `plugins.slots.memory` follows a repair path that removes allow/entry refs while validation still fails on `plugins.slots.memory`.

## Solution Assessment

Yes, the requested repair boundary is the narrow maintainable fix. The doctor stale-plugin cleanup should handle slot references consistently instead of changing plugin install behavior or adding a new config mode.
