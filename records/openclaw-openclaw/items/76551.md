## Summary

Keep open; current main still has a slot-blind stale-plugin repair path that can remove `plugins.allow` and `plugins.entries` while leaving a fatal `plugins.slots.memory` reference behind. The open sibling #76550 covers the same recovery gap from the stale-slot side, so these should be fixed together rather than swept closed.

## What This Changes

Make `openclaw doctor --fix` handle plugin slot references consistently when pruning stale plugin allowlist and entry config.

## Best Possible Solution

Make stale plugin repair slot-aware and atomic so doctor either preserves a slot-selected plugin while it is recoverable or resets/removes matching stale slot refs in the same repair, leaving config validation clean.

## Reproduction Assessment

Yes, source-reproducible. A config with the same missing plugin in `plugins.allow`, `plugins.entries`, and explicit `plugins.slots.memory` follows code that removes allow/entry refs but leaves the slot, after which validation reports `plugins.slots.memory: plugin not found`.

## Solution Assessment

Yes, the requested repair boundary is the right one. The narrow fix is to extend the existing stale-plugin doctor repair to handle slot references consistently, not to change plugin install mechanics or add a new configuration mode.
