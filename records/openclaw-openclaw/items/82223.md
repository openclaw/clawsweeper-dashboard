## Summary

This PR should stay open for normal maintainer review: current main still hardcodes 60s isolated-agent cron setup/pre-execution watchdogs, while the PR makes the watchdogs configurable, raises the default to 3m, and has matching schema/docs/test coverage plus redacted live Gateway proof for the same runtime behavior. I found no blocking correctness finding, but the longer default fail-fast window is an availability policy tradeoff maintainers should explicitly accept before merge.

## What This Changes

The PR raises isolated-agent cron setup and pre-execution watchdog defaults from 60s to 3m, adds `cron.isolatedAgentSetupWatchdog` and `cron.isolatedAgentPreExecutionWatchdog`, and updates docs, schema help, validation, and timer regression tests.

## Best Possible Solution

Land the configurable watchdog fix once maintainers accept the 3m default and lane-occupancy tradeoff, while leaving the half-timeout formula change to https://github.com/openclaw/openclaw/issues/83110 and https://github.com/openclaw/openclaw/pull/83111.

## Solution Assessment
