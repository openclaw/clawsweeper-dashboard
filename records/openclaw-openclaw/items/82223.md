## Summary

Keep open: current main still hardcodes the isolated-agent cron watchdogs at 60s, and this PR is a focused fix with sufficient live-log proof. The patch still needs a small user-facing help correction before merge because schema help says the new defaults are 5m while the runtime, docs, types, and tests use 3m.

## What This Changes

The PR raises isolated-agent cron setup and pre-execution watchdog defaults to 3m, adds cron watchdog config fields, and updates docs, schema, labels, and timer regression coverage.

## Best Possible Solution

Land the configurable 3m watchdog fix after aligning schema help with the runtime default and after maintainers accept the longer fail-fast window; keep the separate half-timeout formula discussion in https://github.com/openclaw/openclaw/issues/83110 and https://github.com/openclaw/openclaw/pull/83111.

## Reproduction Assessment

Yes, with source and live-log evidence rather than a local rerun. Current main hardcodes 60s watchdogs, and the linked real deployment completed the same isolated cron class after both watchdogs were raised to 180s.

## Solution Assessment

Mostly yes, after the help-text fix. The runtime/config/test shape is narrow and matches the observed recovery, but the 3m default is still an availability policy choice for maintainers.
