## Summary

Keep open. Exact PR head still has a blocking plugin-only fast-path regression: the hardcoded core-tool classifier misses current built-in tool names, so automerge should wait for a narrow repair.

## What This Changes

The PR skips core coding-tool construction for plugin-only allowlists, splits full-workspace plugin registry cache entries from scoped loads, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Keep the registry-cache split and changelog entry, but derive core-tool detection and conflict names from the canonical tool inventory before automerge.

## Reproduction Assessment

Yes. On exact PR head, `toolsAllow: ["process"]` or `toolsAllow: ["heartbeat_respond"]` makes `includeCoreTools` false because those normalized names are missing from the new classifier, and the final allow filter cannot recover tools that were never built.

## Solution Assessment

No. The optimization direction is sound, but the hardcoded classifier and empty plugin conflict context are less maintainable than reusing canonical core-tool identity or a shared inventory helper.
