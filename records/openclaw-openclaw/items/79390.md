## Summary

Keep open. Current main still has a source-provable path where preflight compaction can fail before the main turn and reach the outer `run_failed` handler, and the only linked fix PR is closed unmerged with review feedback that a signal-only fix was incomplete.

## What This Changes

The issue reports that preflight compaction and memory flush share the main reply operation abort scope, so auxiliary compaction failure can abort the main agent turn.

## Best Possible Solution

Make preflight compaction and memory flush true auxiliary work: parent or restart aborts should still cancel them, but their own timeout or failure should be logged, covered by regression tests, and unable to fail or mutate the parent reply operation.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection gives a clear current-main path where preflight compaction can throw before the main turn and reach the outer `run_failed` handler, matching the reported class of failure.

## Solution Assessment

Yes to keeping this open for a focused repair; no to the closed PR's `AbortSignal.any([parent])`-only approach. The safer solution is to downgrade auxiliary preflight failures and isolate mutable maintenance-run operation/backend state without unrelated gateway config changes.
