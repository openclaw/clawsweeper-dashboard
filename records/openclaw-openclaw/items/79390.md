## Summary

Keep open. Current main still passes the parent reply operation signal into preflight compaction and memory flush, and preflight compaction errors can still escape into the outer reply-run failure path; the related fix PR is closed unmerged and its signal-only approach did not cover the full failure boundary.

## What This Changes

The issue requests isolation of preflight compaction and memory flush abort/failure handling so auxiliary compaction work cannot abort the main agent turn.

## Best Possible Solution

Make preflight compaction and memory flush true auxiliary work: parent/user abort should still cancel them, but their own timeout or failure should be logged, tested, and unable to fail or mutate the parent reply operation.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection still shows a current-main path where preflight compaction can throw before the main turn and reach the outer run_failed handler.

## Solution Assessment

No for the closed PR's signal-only approach. The safer fix is to catch and downgrade auxiliary preflight failures and isolate any mutable maintenance-run operation/backend state, with regression coverage for timeout and abort paths.
