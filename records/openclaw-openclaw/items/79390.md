## Summary

Keep open. Current main still passes the parent reply operation abort signal into preflight compaction and memory flush, and an escaping preflight error can still reach the outer run failure path before the main turn starts; the only linked fix PR is closed unmerged and was reviewed as incomplete.

## What This Changes

Preflight compaction should not let compaction timeout or failure abort the main agent turn.

## Best Possible Solution

Make preflight compaction and memory flush true auxiliary work: parent abort should still cancel maintenance, but maintenance timeout or failure should be logged, covered by regression tests, and unable to fail or mutate the parent turn.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a current-main path where preflight compaction can throw before the main turn and reach the outer `run_failed` handler.

## Solution Assessment

Yes for keeping this open and replacing the closed signal-only PR with a focused auxiliary-failure isolation fix. `AbortSignal.any([parent])` alone is not enough unless the repair also downgrades preflight failures and isolates mutable maintenance operation state.
