## Summary

Keep this open: current main still threads the parent reply operation abort signal into preflight compaction and memory flush, the only linked fix PR was closed unmerged, and the exact cancellation boundary still needs maintainer diagnosis before cleanup can safely close it.

## What This Changes

The issue reports that preflight compaction and memory flush can abort the main agent turn when compaction is triggered near the token budget and times out or fails.

## Best Possible Solution

Diagnose and fix the auxiliary compaction/flush cancellation boundary so budget-triggered maintenance failures are downgraded without aborting the parent reply turn, with focused regression coverage and live or log proof for a timeout/failure case.

## Reproduction Assessment

No high-confidence live reproduction is established here. Source inspection confirms the shared parent signal and operation paths, but the exact child-to-parent abort mechanism still needs a failing regression or live gateway/log proof.

## Solution Assessment

Unclear. The previously proposed `AbortSignal.any([parent])` direction does not prove isolation of shared ReplyOperation/backend state, so the safer fix is to reproduce the cascade and isolate or catch the actual auxiliary maintenance failure path.
