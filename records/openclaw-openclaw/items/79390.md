## Summary

Keep open. Current main and the latest release still pass the parent reply operation signal into preflight compaction and memory flush, and preflight compaction errors can escape into the outer run failure path. The linked fix PR is closed unmerged, and its prior review found the signal-only fix incomplete, so this remains actionable.

## What This Changes

The issue reports that preflight compaction reuses the main reply operation abort signal, letting an auxiliary compaction timeout or failure abort the parent agent turn.

## Best Possible Solution

Make preflight compaction and memory flush true auxiliary work: parent/user abort should still cancel them, but their own timeout or failure should be logged, covered by regression tests, and unable to fail or mutate the parent turn's reply operation.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection does show a clear current-main failure path where preflight compaction can throw before the main turn and reach the outer `run_failed` handler, matching the reported class of failure.

## Solution Assessment

No for the closed PR's signal-only approach. The safer fix is to catch and downgrade auxiliary preflight failures and isolate any mutable maintenance-run operation/backend state, without unrelated gateway config changes.
