## Summary

Keep open. Current main and v2026.5.12 still pass the parent reply operation abort signal into preflight compaction, and an escaping preflight error is still inside the outer run failure path before the main turn; the linked fix PR is closed unmerged and its signal-only approach was previously reviewed as incomplete.

## What This Changes

The issue requests isolating budget-triggered preflight compaction timeouts and failures so they do not abort the main agent turn.

## Best Possible Solution

Make preflight compaction and memory flush true auxiliary work: parent abort still cancels them, but their timeout/failure is logged and returns the original session without failing or mutating the parent reply operation.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection shows a clear current-main path where budget-triggered preflight compaction is awaited before the main turn with the parent abort signal and without a local failure downgrade.

## Solution Assessment

No for the closed PR's signal-only approach. The safer fix is to catch and downgrade auxiliary preflight failures and isolate maintenance cancellation/operation state while preserving parent-to-child abort behavior.
