## Summary

Keep open. Current `main` and the latest release still pass the parent reply operation signal into preflight compaction and memory flush, and an escaping preflight compaction error still reaches the outer `run_failed` path; the related fix PR is closed unmerged and its signal-only approach was already flagged as incomplete.

## What This Changes

The issue asks for preflight compaction and memory-flush failures to be isolated so auxiliary timeout or abort behavior cannot kill the main agent turn.

## Best Possible Solution

Make preflight compaction and memory flush true auxiliary work: parent/user abort should still cancel them, but their own timeout or failure should be logged, regression-tested, and unable to fail or mutate the parent reply operation.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a current-main path where preflight compaction can throw before the main turn and reach the outer `run_failed` handler, matching the reported failure class.

## Solution Assessment

No for the closed PR's signal-only approach. The safer fix is to catch and downgrade auxiliary preflight failures and isolate any mutable maintenance-run operation/backend state, without unrelated gateway config changes.
