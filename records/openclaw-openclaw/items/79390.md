## Summary

Keep open. Current main still passes the parent reply operation signal into preflight compaction and memory flush, and the only directly linked fix PR is closed unmerged; the exact signal-cascade mechanism still needs proof, but there is no implemented or superseding fix to close against.

## What This Changes

The issue asks OpenClaw to isolate or downgrade preflight compaction and memory-flush abort/failure handling so auxiliary compaction cannot abort the main agent turn.

## Best Possible Solution

Make preflight compaction and memory flush true auxiliary work: parent/user abort can cancel them, but their own timeout or failure is logged or surfaced in a controlled way and cannot fail or mutate the parent reply operation.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection shows current main still enters preflight compaction with the parent signal before the main turn and any escaping preflight error reaches the outer `run_failed` path, but the child-aborts-parent mechanism needs a targeted proof.

## Solution Assessment

No for the closed PR's signal-only approach. The safer fix is to catch and downgrade auxiliary preflight failures and isolate shared signal/replyOperation state, with regression coverage that proves the actual timeout or abort path.
