## Summary

Keep open: current main still does not solve the requested default-responder suppression, but this branch remains unsafe to merge because it conflates generic platform mentions with agent targets, drops Discord role context, adds an unbounded postinstall deletion path, and lacks real behavior proof.

## What This Changes

The PR branch adds an opt-in mention-gating suppression flag, wires it through bundled channel plugins, expands mention-gating tests, and adds bundled-plugin postinstall cleanup plus a PDF test model update.

## Best Possible Solution

Replace or heavily rework this as a current-main branch that adds a typed explicit-agent-target signal to shared channel ingress, wires only channels that can prove that signal, preserves Discord role context, and moves or hardens the postinstall cleanup with root-boundary tests.

## Reproduction Assessment

Yes, at source level. Current main lacks the requested suppression, and the PR regressions are visible from the diff plus current channel call sites that pass generic mention facts and from the Discord preflight return contract.

## Solution Assessment

No. The shared mention-policy area is a reasonable home, but this implementation is not the best fix because it treats generic platform mentions as agent targets and mixes an unrelated security-sensitive install cleanup into the routing feature.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use an agent-target signal before suppressing:** `src/channels/mention-gating.ts:133-139`
  - body: The new suppression path treats `hasAnyMention` as if another OpenClaw agent was targeted. Several enabled callers pass generic platform mentions, so always-on agents can drop normal messages that mention a person, role, group, or entity rather than another agent.
  - confidence: 0.9
- **[P1] Keep Discord member roles in the preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1083`
  - body: This return object no longer includes `memberRoleIds`, while current Discord routing, auth, and prompt context consume that field. Dropping it loses sender role context for inbound Discord messages.
  - confidence: 0.93
- **[P1] Validate dist extension roots before recursive deletion:** `scripts/postinstall-bundled-plugins.mjs:665`
  - body: `pruneBundledDistPluginNodeModules` recursively deletes `dist/extensions/*/node_modules` after only checking existence and `Dirent` metadata. Reuse the existing realpath/lstat package-root boundary checks before calling `rmSync`.
  - confidence: 0.88
