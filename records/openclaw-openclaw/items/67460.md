## Summary

Keep this PR open because current main still lacks the requested always-on/default-agent suppression behavior, but the PR should not land as-is: the patch still drops Discord role context, suppresses on generic platform mentions rather than proven OpenClaw-agent mentions, and mixes a security-sensitive postinstall deletion change into the feature branch.

## What This Changes

The PR adds an opt-in inbound mention-policy flag and channel call-site wiring so always-on group agents can skip messages when another mention is detected, while also changing bundled-plugin postinstall pruning and one PDF test model constant.

## Best Possible Solution

Land a narrower routing fix or replacement branch that adds a typed, documented mention-precedence contract based on explicit OpenClaw agent targets, wires only channels that can supply that signal, preserves Discord role context, keeps install cleanup in a separate security-reviewed PR, and adds focused shared plus channel regression tests and a changelog entry.

## Reproduction Assessment

Yes. The current-main behavior is reproducible by inspection of `resolveInboundMentionDecision` and Slack/Discord drop guards, and the PR regressions are reproducible with a Discord typecheck/role-context check plus an always-on group test where the message mentions a non-agent user or role.

## Solution Assessment

No. The core idea is the right problem area, but this patch is not the best maintainable solution because it conflates generic platform mentions with other-agent mentions, loses Discord role metadata, and bundles unrelated postinstall and PDF-test changes into a channel-routing feature.

## Security Review

Status: needs_attention

Summary: The PR introduces a postinstall recursive deletion path without the existing package-root realpath boundary checks, so the supply-chain/install-script pass is not cleared.

Concerns:

- **[high] Postinstall recursive deletion lacks root validation:** `scripts/postinstall-bundled-plugins.mjs:673`
  - body: The new `pruneBundledDistPluginNodeModules` deletes `join(extensionsDir, entry.name, 'node_modules')` recursively during postinstall after only checking `existsSync` and `Dirent` metadata. Because it does not verify that `extensionsDir` and each target resolve under the package root, a redirected or symlinked dist tree could cause deletion outside the OpenClaw install directory.
