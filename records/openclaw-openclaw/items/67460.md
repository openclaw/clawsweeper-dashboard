## Summary

This PR should stay open: current main still lacks the requested explicit-target suppression, but this branch has blocking correctness and security issues and still lacks contributor real-behavior proof.

## What This Changes

The branch adds an opt-in inbound mention-suppression flag, wires it across several channel plugins, adds mention-gating and postinstall tests, changes Discord preflight payloads, updates a PDF test model constant, and adds bundled-plugin postinstall cleanup.

## Best Possible Solution

Keep the explicit-target precedence rule, but implement it around a typed per-channel agent-target signal, preserve Discord role context, and remove or harden the unrelated postinstall cleanup against current main.

## Reproduction Assessment

Yes, at source level. Current main lacks the requested suppression, and the PR regressions are visible from the PR head code and current call-site contracts; no live channel proof was run in this read-only pass.

## Solution Assessment

No. A shared helper is a plausible home, but this implementation needs an explicit agent-target contract and should not mix security-sensitive install cleanup into the routing feature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use an agent-target signal before suppressing:** `src/channels/mention-gating.ts:133-139`
  - body: The suppression path treats `hasAnyMention` as proof that another OpenClaw agent was targeted. Slack and Telegram pass generic platform mentions here, so always-on agents can drop messages that mention a person, group, or entity instead of another agent.
  - confidence: 0.91
- **[P1] Restore member roles in preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1086`
  - body: This return object no longer includes `memberRoleIds`, while downstream Discord context still forwards that value as `MemberRoleIds`. Dropping it loses sender role context for inbound Discord messages and can break role-scoped routing, auth, and prompt context.
  - confidence: 0.93
- **[P1] Validate dist roots before recursive deletion:** `scripts/postinstall-bundled-plugins.mjs:673-675`
  - body: `pruneBundledDistPluginNodeModules` recursively deletes `dist/extensions/*/node_modules` after only existence and directory metadata checks. Reuse current main's package-root realpath validation before `rmSync` so a redirected dist tree cannot delete outside the install root.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces a postinstall recursive deletion path that lacks package-root realpath validation.
