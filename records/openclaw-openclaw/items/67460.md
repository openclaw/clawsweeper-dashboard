## What This Changes

The branch adds an opt-in shared mention-gating suppression flag, wires it into many bundled channel plugins, adds stale dist plugin node_modules postinstall cleanup, updates a PDF test model constant, and extends tests.

## Best Possible Solution

Rework this on current main around a typed explicit-agent-target signal, wire only channels that can prove that signal, preserve Discord role context, and either rely on or harden the existing installed-plugin cleanup path.

## Reproduction Assessment

Yes, at source level. Current main lacks the requested suppression, and the PR regressions are visible from the diff plus current channel call sites and Discord preflight contracts.

## Solution Assessment

No. The shared mention-policy area is plausible, but this patch uses a generic mention fact as an agent-target signal and mixes unrelated security-sensitive postinstall deletion into the routing feature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use an agent-target signal before suppressing:** `src/channels/mention-gating.ts:133-139`
  - body: The new suppression path treats hasAnyMention as if another OpenClaw agent was targeted. Several enabled callers pass generic platform mentions, so always-on agents can drop normal messages that mention a person, role, group, or entity rather than another agent.
  - confidence: 0.9
- **[P1] Restore Discord member roles in preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1086`
  - body: This return object no longer includes memberRoleIds, while current Discord routing, auth, and prompt context consume that field. Dropping it loses sender role context for inbound Discord messages.
  - confidence: 0.93
- **[P1] Validate dist extension roots before deletion:** `scripts/postinstall-bundled-plugins.mjs:673`
  - body: pruneBundledDistPluginNodeModules recursively deletes dist/extensions/*/node_modules after only checking existence and Dirent metadata. Reuse the existing realpath/lstat package-root boundary checks before calling rmSync.
  - confidence: 0.88

## Security Review

Status: needs_attention
