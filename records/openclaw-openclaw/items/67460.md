## Summary

Keep open: current main still lacks the requested explicit-mention precedence, but this PR is not merge-ready because it can suppress always-on agents for generic platform mentions, drops Discord role context, adds an unsafe postinstall recursive deletion path, conflicts with main, and lacks real behavior proof.

## What This Changes

The branch adds opt-in always-on-agent suppression to the shared mention gate and many channel plugins, updates one PDF test model constant, and adds postinstall cleanup for stale dist plugin node_modules.

## Best Possible Solution

Implement explicit-target precedence around a typed per-channel signal for mentions of configured OpenClaw agents, preserve Discord role context, split or harden the installer cleanup, then verify the behavior in a real channel and install/upgrade path.

## Reproduction Assessment

Yes, at source level. Current main lacks the requested suppression, and PR-head code shows the generic mention suppression, Discord role-context drop, and postinstall delete path; no live channel or install proof was attached.

## Solution Assessment

No. A shared mention helper is a plausible home, but this implementation should use a typed configured-agent target signal and avoid mixing security-sensitive install cleanup into the routing feature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use an agent-target signal before suppressing:** `src/channels/mention-gating.ts:133-139`
  - body: The suppression path treats `hasAnyMention` as proof that another OpenClaw agent was targeted. Slack and Telegram pass generic platform mentions here, so always-on agents can drop messages that mention a person, group, or entity instead of another configured agent.
  - confidence: 0.91
- **[P1] Restore member roles in the preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1086`
  - body: This return object no longer includes `memberRoleIds`, while `DiscordMessagePreflightContext` declares it and downstream Discord routing uses role IDs. Add the field back so inbound Discord messages keep sender role context.
  - confidence: 0.93
- **[P1] Validate dist roots before recursive deletion:** `scripts/postinstall-bundled-plugins.mjs:673`
  - body: `pruneBundledDistPluginNodeModules` recursively deletes `dist/extensions/*/node_modules` after only existence and directory-entry checks. Reuse the package-root lstat/realpath validation before calling `rmSync` so a redirected dist tree cannot delete outside the install root.
  - confidence: 0.88
