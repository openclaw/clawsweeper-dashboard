## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Restrict suppression to other agents:** `src/channels/mention-gating.ts:133-139`
  - body: This uses `hasAnyMention` as proof that another agent was targeted, but the updated channel callers populate that flag from generic platform mentions such as users, roles, or subteams. In always-on groups, ordinary messages that mention a person or role can now be dropped even though no other OpenClaw agent was addressed.
  - confidence: 0.95
- **[P1] Keep dist pruning inside the package root:** `scripts/postinstall-bundled-plugins.mjs:671-673`
  - body: `pruneBundledDistPluginNodeModules` follows the supplied `extensionsDir` and recursively removes `node_modules` from each entry without checking that the real path is still under the OpenClaw package root. If `dist/extensions` is redirected or symlinked, postinstall can delete directories outside the install tree before the safer dist-prune boundary checks run.
  - confidence: 0.9
- **[P1] Restore Discord role context in the preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1084-1087`
  - body: This return object no longer includes `memberRoleIds`, but downstream Discord processing still destructures it and forwards it into `MemberRoleIds`. As a result, inbound Discord messages lose sender role context at runtime, which can break role-scoped routing, authorization, and prompt context.
  - confidence: 0.88
