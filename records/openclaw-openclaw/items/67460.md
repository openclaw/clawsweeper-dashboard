## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not suppress on generic platform mentions:** `src/channels/mention-gating.ts:138-139`
  - body: The new core gate treats `hasAnyMention && !effectiveWasMentioned` as "another agent was mentioned," but wired callers populate `hasAnyMention` from generic user, role, subteam, or platform mention metadata. In an always-on group, a normal message that mentions a person or role can now set `suppressedByOtherAgentMention` and be dropped even though no other OpenClaw agent was targeted; use an other-agent-specific fact or only opt in channels that can prove that fact.
  - confidence: 0.92
- **[P1] Keep dist pruning inside the package root:** `scripts/postinstall-bundled-plugins.mjs:672-673`
  - body: `pruneBundledDistPluginNodeModules` walks whatever `extensionsDir` points at and recursively removes `<entry>/node_modules` without validating that `dist/extensions` is a real directory under the installed package. If that path is redirected or symlinked, postinstall can delete directories outside OpenClaw's install tree; reuse the current dist root realpath/lstat checks or remove this cleanup from the PR.
  - confidence: 0.88
- **[P1] Restore Discord member roles in the preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1065`
  - body: The PR branch's Discord preflight return payload omits `memberRoleIds`, while current main carries that field into the message process context and exposes it as sender roles. Rebase onto the current `buildDiscordMessagePreflightContext` path or add the field back so downstream routing, authorization, and prompt context do not lose Discord role state.
  - confidence: 0.83
