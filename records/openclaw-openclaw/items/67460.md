## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use an agent-target signal before suppressing:** `src/channels/mention-gating.ts:138`
  - body: The new suppression gate treats any `hasAnyMention` as proof that another OpenClaw agent was targeted. Several updated callers populate that field from generic platform mentions, so an always-on group agent can drop messages that mention a human or role but no configured agent; pass a typed configured-agent target signal instead.
  - confidence: 0.93
- **[P1] Restore Discord role context in preflight:** `extensions/discord/src/monitor/message-handler.preflight.ts:1085`
  - body: This return payload drops `memberRoleIds`, but `buildDiscordMessageProcessContext` still destructures that field and forwards it as Discord sender roles. After this patch, role-scoped context and policy lose the sender's roles at runtime; add `memberRoleIds` back to the preflight context.
  - confidence: 0.9
- **[P1] Validate dist prune targets before recursive removal:** `scripts/postinstall-bundled-plugins.mjs:673`
  - body: `pruneBundledDistPluginNodeModules` removes `dist/extensions/*/node_modules` without checking that `extensionsDir` and the removal target resolve under the installed package root. A redirected or symlinked dist extensions directory can make postinstall delete outside OpenClaw's package tree; add lstat/realpath boundary checks or remove this cleanup from the PR.
  - confidence: 0.88
