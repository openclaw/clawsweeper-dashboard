## Summary

Keep open: the branch still has blocking correctness and security findings, lacks real behavior proof, and is now conflicting with current main. The central idea is useful, but the implementation needs a typed agent-target signal instead of treating generic platform mentions as other-agent mentions.

## What This Changes

The branch adds opt-in inbound mention suppression across the shared mention gate and channel plugins, updates a PDF test model constant, and adds postinstall cleanup for stale bundled plugin node_modules.

## Best Possible Solution

Keep the explicit-target precedence rule, but implement it around a typed per-channel agent-target signal, preserve Discord role context, and either remove or harden the unrelated postinstall cleanup.

## Reproduction Assessment

Yes, at source level. Current main lacks the requested suppression, and the PR regressions are visible from PR-head code and current call-site contracts; no live channel or install proof was run in this read-only pass.

## Solution Assessment

No. A shared helper is a plausible home, but this implementation should use an explicit agent-target contract and avoid mixing security-sensitive install cleanup into the routing feature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use an agent-target signal before suppressing:** `src/channels/mention-gating.ts:133-139`
  - body: The suppression path treats `hasAnyMention` as proof that another OpenClaw agent was targeted. Slack and Telegram pass generic platform mentions here, so always-on agents can drop messages that mention a person, group, or entity instead of another configured agent.
  - confidence: 0.91
- **[P1] Restore member roles in the preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1086`
  - body: This return object no longer includes `memberRoleIds`, while `DiscordMessagePreflightContext` declares it and downstream Discord context/routing uses role IDs. Add the field back so inbound Discord messages keep sender role context.
  - confidence: 0.93
- **[P1] Validate dist roots before recursive deletion:** `scripts/postinstall-bundled-plugins.mjs:673`
  - body: `pruneBundledDistPluginNodeModules` recursively deletes `dist/extensions/*/node_modules` after only existence and directory-entry checks. Reuse current main's lstat/realpath package-root validation before calling `rmSync` so a redirected dist tree cannot delete outside the install root.
  - confidence: 0.88
