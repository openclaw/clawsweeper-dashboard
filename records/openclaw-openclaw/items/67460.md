## Summary

Keep open: the branch is not ready because the suppression logic treats generic platform mentions as agent-targets, the Discord preflight payload loses role context, the postinstall cleanup adds an unsafe recursive deletion path, and contributor real behavior proof is still missing.

## What This Changes

The PR adds an opt-in mention suppression flag, wires it through multiple channel plugins, expands tests, and includes bundled-plugin postinstall cleanup plus a PDF model test constant update.

## Best Possible Solution

Rework the feature around a typed explicit-agent-target signal, wire only channels that can prove that signal, preserve Discord role context, and remove or harden the unrelated postinstall cleanup with root-boundary tests.

## Reproduction Assessment

Yes, at source level. Current main lacks the requested suppression, and the PR regressions are visible from the PR head code plus current call-site contracts; no live channel proof was run in this read-only pass.

## Solution Assessment

No. The shared helper is a plausible home, but this implementation needs an explicit agent-target contract and should not mix a security-sensitive install cleanup into the routing feature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use an agent-target signal before suppressing:** `src/channels/mention-gating.ts:133-139`
  - body: The new suppression path treats hasAnyMention as proof that another OpenClaw agent was targeted. Several enabled callers pass generic platform mentions, so always-on agents can drop normal messages that mention a person, role, group, or entity instead of another agent.
  - confidence: 0.9
- **[P1] Restore member roles in preflight payload:** `extensions/discord/src/monitor/message-handler.preflight.ts:1086`
  - body: This return object no longer includes memberRoleIds, while downstream Discord context finalization still forwards that value as MemberRoleIds. Dropping it loses sender role context for inbound Discord messages and can break role-scoped routing, auth, and prompt context.
  - confidence: 0.93
- **[P1] Validate dist roots before recursive deletion:** `scripts/postinstall-bundled-plugins.mjs:673`
  - body: pruneBundledDistPluginNodeModules recursively deletes dist/extensions/*/node_modules after only checking existence and Dirent metadata. Reuse the existing package-root realpath/lstat validation before calling rmSync so a redirected dist tree cannot delete outside the install root.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a postinstall recursive deletion path without the root-boundary validation used by existing dist pruning.
