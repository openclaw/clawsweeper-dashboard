## Summary

Keep open. Current main has several partial mitigations for memory startup context, session override repair, newer-config guards, and duplicate install warnings, but the combined upgrade-continuity regression and stale pinned CLI path are not fully resolved; the same reporter's related memory/session issue is still open at https://github.com/openclaw/openclaw/issues/65358.

## What This Changes

The issue reports a 2026.4.10 macOS upgrade that made an agent appear to lose memory and left an older `~/.local/bin/openclaw` binary breaking a pinned scheduled job.

## Best Possible Solution

Deliberately split or consolidate the memory/session work with https://github.com/openclaw/openclaw/issues/65358, then add explicit update/doctor diagnostics for version-skewed binaries and pinned scheduled-job paths with a clear warning or repair policy.
