## Summary

Keep open. Current main has partial mitigations for startup daily-memory loading, older-binary guards, gateway service drift diagnostics, and manual backup, but the reported upgrade-continuity failure and stale user-pinned CLI path are not fully solved or safely superseded.

## What This Changes

The issue asks OpenClaw to harden the 2026.4.10 upgrade path around session memory rehydration, stale CLI binary paths, config schema skew, and post-update verification.

## Best Possible Solution

Keep this open until maintainers split or define a scoped upgrade-continuity plan covering memory/session bootstrap, stale absolute CLI path diagnostics or reconciliation, and coordination with the open safe-update tracker at https://github.com/openclaw/openclaw/issues/14526 and session-upgrade tracker at https://github.com/openclaw/openclaw/issues/65358.

## Solution Assessment

No as a one-shot automated fix. The maintainable path needs maintainer scoping across upgrade memory continuity, stale absolute CLI path diagnostics or reconciliation policy, and safe update recovery rather than closing this as fixed.
