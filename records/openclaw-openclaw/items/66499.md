## Summary

Close: current main now routes live chat through a shared projector whose merge logic no longer performs the overlap dedupe that caused repeated-boundary character loss, and adjacent tests cover repeated incremental chunks plus newline-preserving non-prefix deltas. The PR branch is stale against that projector and its unconditional leading trim is not safe to merge as-is.

## What This Changes

The PR changes Gateway chat stream normalization by exporting the old merge helper, applying `trimStart()` after stripping inline directive tags, guarding empty-buffer delta merging, and adding regression tests for directive-prefixed WebSocket text corruption.

## Best Possible Solution

Keep the current shared-projector merge behavior on main and do not merge this stale branch. If a newer build still shows directive-specific whitespace artifacts, add a narrowly directive-aware normalization in `src/gateway/live-chat-projector.ts` with a Gateway or projector regression test that preserves intentional leading newlines and indentation.

Yes. The reported corruption class can be reproduced at the merge-helper level with a trimmed previous boundary plus a directive-whitespace snapshot and incremental delta; current main returns the full `xumum` and `GOOGLE` strings because it appends the delta instead of overlap-deduping it.
