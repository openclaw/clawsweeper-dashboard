## Summary

## What This Changes

## Best Possible Solution

Keep the shared-helper source change, repair the provider HTTP test mock contract, and land only after focused Qwen/Alibaba provider tests and the provider HTTP helper tests pass.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` still use raw `response.json()` in both targeted DashScope success-body paths, and the PR body includes copied live output from a local HTTP-server proof for oversized JSON.

## Solution Assessment

Mostly. Reusing `readProviderJsonResponse` is the narrowest maintainable source fix, but the branch must update the provider HTTP test mock contract before it is the best merge candidate.
