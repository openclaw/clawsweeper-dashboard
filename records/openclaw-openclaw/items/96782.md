## What This Changes

The PR replaces DashScope video submit and task-poll `response.json()` parsing with `readProviderJsonResponse` and updates the shared provider HTTP Vitest mock to export that helper.

## Best Possible Solution

Land one canonical bounded-read implementation for the shared DashScope video helper, preferably this shared-helper version if maintainers accept the 16 MiB cap, then close the duplicate same-scope PR.

## Reproduction Assessment

Yes. Current main and v2026.6.10 still use raw `response.json()` in both targeted DashScope success-body paths, and the PR body gives an after-fix local HTTP-server proof for the oversized-body path.

## Solution Assessment

Yes, with a maintainer compatibility choice. Reusing `readProviderJsonResponse` is the narrowest maintainable source fix because the SDK already exposes a bounded provider JSON helper, and this PR updates the affected provider HTTP test mock contract.
