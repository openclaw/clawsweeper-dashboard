## Label Justifications

- `P1`: The PR fixes a reported stuck-session/message-delivery regression where a delivered final reply can leave durable pending-final state that blocks later inbound work.

## Summary

Keep this PR open for normal merge handling: current main still has the abort-before-cleanup race, while the branch applies a focused fix with matching regression coverage, terminal real-runtime proof, and no blocking findings from this review.

## What This Changes

The branch moves pending-final cleanup ahead of the post-send abort check, clears `pendingFinalDeliveryIntentId` on successful final delivery, adds focused regression and proof coverage, and restores the one-line plugin SDK testing bridge.

## Best Possible Solution

Merge the focused dispatch cleanup once required CI is green, keeping the deprecated testing bridge as the exact one-line package contract until maintainers intentionally change that contract.

## Reproduction Assessment

Yes from source inspection, though I did not execute the repro in this read-only review. Current main sends the final payload and then checks abort before clearing pending-final state, so an abort in that window prevents cleanup.

## Solution Assessment

Yes. Clearing durable pending-final state immediately after a successful final delivery, then surfacing the abort, is the narrowest maintainable fix; threshold changes or watchdog cleanup would only reduce the frequency of the stale-state race.
