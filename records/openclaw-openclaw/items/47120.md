## Summary

Keep open: current main still has the gateway preview contract, but the Control UI Sessions list does not request or render last-message preview text, and existing open PRs already cover this UI work.

## What This Changes

Request to show a timestamp and last-message preview in the Control UI Sessions list so long session keys are easier to distinguish.

## Best Possible Solution

Land one small Control UI change that requests bounded `includeLastMessage` data through `sessions.list`, plumbs `lastMessagePreview` into UI row types, renders timestamp plus preview, and adds focused controller/view coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path: the Control UI does not ask for `includeLastMessage` and renders only the updated timestamp.

## Solution Assessment

Yes, the narrowest maintainable path is to reuse the existing bounded `sessions.list` `includeLastMessage` contract rather than adding per-row preview RPCs. The open PRs should be reviewed, rebased, landed, replaced, or closed against that shape.
