## Summary

Keep open: current main still does not request or render last-message previews in the Control UI Sessions list, and open implementation PRs already target this feature. Because https://github.com/openclaw/openclaw/pull/47243 uses closing syntax for this issue and https://github.com/openclaw/openclaw/pull/48396 overlaps, the issue should stay open until one corrected implementation lands or maintainers choose a replacement path.

## What This Changes

Add timestamp and last-message preview text to each Control UI session-list row so sessions are easier to distinguish.

## Best Possible Solution

Review and dedupe the open implementation PRs, then land one small Control UI change that requests bounded `includeLastMessage`, carries `lastMessagePreview` through the UI type, renders a normalized timestamp plus preview, and adds focused coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path because current main omits `includeLastMessage`, lacks the UI preview field, and renders only the updated timestamp.

## Solution Assessment

Yes at the contract level: the narrow maintainable direction is to reuse the existing bounded `sessions.list` `includeLastMessage` support rather than per-row ad hoc preview calls. The remaining maintainer choice is which existing PR, repaired or replaced, should become canonical.
