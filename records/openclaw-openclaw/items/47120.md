## What This Changes

The issue requests showing a timestamp and last-message preview in the Control UI session list so sessions are easier to distinguish.

## Best Possible Solution

Land one deduped Control UI implementation that uses the existing bounded preview contract, carries `lastMessagePreview` through the UI type, renders a normalized timestamp plus preview, and includes focused UI coverage plus real behavior proof.
