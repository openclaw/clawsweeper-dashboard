## Summary

Keep open: current main still has the backend last-message preview contract, but the Control UI Sessions list does not request or render preview text, and open paired PRs already cover this work.

## What This Changes

Request to show a session timestamp and last-message preview in the Control UI Sessions list so long session IDs are easier to distinguish.

## Best Possible Solution

Land a small Control UI change that requests bounded `includeLastMessage` data through existing `sessions.list`, plumbs `lastMessagePreview` into UI row types, renders timestamp plus preview, and adds focused controller/view coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path: the Control UI Sessions controller omits `includeLastMessage`, and the row renderer only displays the relative Updated time.

## Solution Assessment

Yes, the narrowest maintainable direction is to reuse the existing `sessions.list` `includeLastMessage` contract rather than adding per-row preview RPCs. The open PRs should be reviewed, rebased, landed, replaced, or closed against that current-main shape.
