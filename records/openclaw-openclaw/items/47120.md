## Summary

Keep open: current main still does not show last-message previews in the Control UI Sessions list, and open implementation PRs already target the feature. The issue should remain open until maintainers land, replace, or close those paired branches.

## What This Changes

Add a timestamp and last-message preview to each Control UI Sessions list row so users can distinguish conversations without relying on long session IDs.

## Best Possible Solution

Land one small Control UI implementation that reuses the bounded `sessions.list` `includeLastMessage` contract, carries `lastMessagePreview` through the UI type, renders a normalized and clamped timestamp-plus-preview line, and adds focused UI coverage.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection gives a high-confidence verification path because current main has the backend preview contract but the Control UI Sessions list does not request or render it.

## Solution Assessment

Yes at the contract level: the narrow maintainable solution is to reuse the existing `sessions.list` `includeLastMessage` support instead of issuing per-row ad hoc preview calls. The immediate maintainer task is to resolve the already-open implementation branches rather than spawn another competing one.
