## Summary

Keep open: current main still does not render timestamp-plus-last-message previews in the Control UI Sessions list, and open implementation PRs already target the feature. The issue should stay open until one corrected implementation lands or maintainers choose a replacement path.

## What This Changes

Add timestamp and last-message preview text to Control UI session rows so long technical session IDs are easier to distinguish.

## Best Possible Solution

Review and dedupe the open implementation PRs, then land one small Control UI change that requests bounded `includeLastMessage`, carries `lastMessagePreview` through the UI type, renders normalized and clamped timestamp-plus-preview text, and adds focused UI coverage.

## Solution Assessment
