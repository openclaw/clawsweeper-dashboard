## Summary

Keep open. Current main has the backend session-list preview contract, but the Control UI Sessions page still does not request or render last-message previews, and an open PR with closing syntax is already paired with this work.

## What This Changes

Request to show a timestamp and last-message preview in the Control UI session list so long technical session IDs are easier to distinguish.

## Best Possible Solution

Land one small Control UI change that requests bounded preview data through the existing `sessions.list includeLastMessage` contract, updates the UI row type and row rendering, adds focused controller/view tests, and lets the merged PR close this issue.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source provides a high-confidence verification path: the Sessions page still requests `sessions.list` without `includeLastMessage` and renders only the relative Updated value.

## Solution Assessment

Yes, reusing the existing `sessions.list includeLastMessage` contract is the narrowest maintainable solution. Per-row `sessions.preview` calls are still possible but would add avoidable frontend RPC fan-out.
