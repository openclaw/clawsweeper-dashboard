## Summary

## What This Changes

The issue requests configurable model overrides for compaction and memory flush, plus an opt-in memoryFlush resetAfterFlush mode that flushes durable memory and resets instead of compacting.

## Best Possible Solution

Design a guarded opt-in reset-after-flush policy that reuses the gateway reset lifecycle, defines flush success and failure semantics, and covers queued follow-ups, subagents, channel continuity, provider/auth routing, docs, and tests.
