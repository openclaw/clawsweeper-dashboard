## Summary

Keep open: current main implements the compaction and memory-flush model override parts, but the requested flush-then-reset mode is still absent and needs maintainer product judgment around session reset semantics.

## What This Changes

Request configurable cheaper models for compaction and pre-compaction memory flush, plus an option to flush durable memory and reset instead of compacting.

## Best Possible Solution

Keep the shipped model overrides, then have maintainers decide whether to add a guarded `memoryFlush.resetAfterFlush`-style mode with documented reset timing, safety checks, and regression coverage.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Source inspection gives high confidence that the model override pieces exist and that no reset-after-flush setting is present.

## Solution Assessment

Unclear. The current implementation solves the model-cost pieces, but reset-after-flush is a new lifecycle mode that should be designed around reset safety before implementation.
