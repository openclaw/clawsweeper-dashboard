## Summary

Keep open: current main implements the compaction and memory-flush model override pieces, but the requested flush-then-reset mode is still absent and requires a session-lifecycle product decision.

## What This Changes

Requests compaction and memory-flush model overrides plus a mode that flushes memory and resets the session instead of compacting afterward.

## Best Possible Solution

Keep the shipped model overrides and have maintainers design a guarded `memoryFlush.resetAfterFlush`-style mode with documented timing, safety checks, and regression coverage.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Source inspection gives high confidence that the model override pieces exist and that reset-after-flush is not currently implemented.

## Solution Assessment

Unclear. The current implementation solves the model-cost pieces, but reset-after-flush is a new lifecycle mode that should be designed around reset safety before implementation.
