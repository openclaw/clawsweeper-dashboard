## Summary

Keep open. Current main covers the model-cost parts with `compaction.model` and `compaction.memoryFlush.model`, but the requested flush-then-reset mode is still missing from the strict config surface and runtime lifecycle.

## What This Changes

Request configurable model overrides for compaction and memory flush plus an optional mode that flushes memory and resets the session instead of compacting.

## Best Possible Solution

Add a maintainer-approved, config-gated reset-after-flush policy that resets only after verified successful memory flush, skips compaction only when configured, and includes schema, docs, and lifecycle tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Static verification is high-confidence: current main has the model override pieces, but no reset-after-flush config key or successful-flush reset branch.

## Solution Assessment

Unclear until maintainers approve lifecycle semantics. The narrow maintainable direction is a config-gated policy built on existing reset/session machinery, not an unconditional reset after every memory flush.
