## Summary

Keep open: current main and the latest release already cover the compaction and memory-flush model override pieces, but the requested flush-then-reset mode is still absent and needs a session-lifecycle product decision.

## What This Changes

Requests configurable housekeeping model overrides for compaction and memory flush, plus a mode that flushes durable memory and then resets the session instead of compacting.

## Best Possible Solution

Keep the shipped model overrides, and add flush-then-reset only as an explicit default-off lifecycle contract wired through the gateway reset service with docs and tests for active runs, queues, subagents, transcript archival, and provider/auth routing.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has compaction.model and memoryFlush.model, but no resetAfterFlush or equivalent flush-then-reset implementation.

## Solution Assessment

Unclear until maintainers decide the session-lifecycle contract. The shipped model overrides are the narrow cost-relief path; automatic reset after memory flush needs explicit safety boundaries before implementation.
