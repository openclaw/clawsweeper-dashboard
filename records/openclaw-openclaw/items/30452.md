## Summary

Keep open: current main and v2026.6.10 cover the compaction and memory-flush model override cost controls, but the requested successful flush-then-reset mode is still absent and remains a product-sensitive session lifecycle/config change.

## What This Changes

The issue requests configurable compaction and memory-flush model overrides plus a new `memoryFlush.resetAfterFlush` mode that flushes durable memory and resets instead of compacting.

## Best Possible Solution

Keep the shipped model overrides, then design reset-after-flush as an explicit default-off lifecycle contract with tests for active runs, queues, subagents, transcript archival, channel continuity, and provider/auth routing.

## Reproduction Assessment
