## Summary

Keep open: current main and v2026.5.12 include the compaction and memory-flush model override pieces, but the requested flush-then-reset mode is still absent from config, schema, docs, and runtime flow. The remaining work is a new session-lifecycle policy/config feature, not a narrow bug fix or something already owned by a merged canonical item.

## What This Changes

The issue requests configurable compaction and memory-flush model overrides plus an opt-in mode that resets the session after memory flush instead of continuing through compaction.

## Best Possible Solution

Keep the shipped model overrides, then design a guarded memoryFlush.resetAfterFlush-style mode around the existing reply/session reset lifecycle with schema, docs, timeout/failure semantics, and regression coverage.

## Reproduction Assessment

Not applicable. This is a feature/config request; current-main source, schema, docs, tests, and release-content inspection show the model overrides exist and the reset-after-flush option does not.

## Solution Assessment

Unclear. The shipped model overrides are the narrow maintainable cost-control solution, but reset-after-flush changes session lifecycle and needs maintainer-selected config, failure, timeout, and state-safety semantics before implementation.
