## Summary

Keep open: current main and the latest release include the compaction and memory-flush model override parts, but the successful flush-then-reset mode is still absent and remains a product-sensitive session lifecycle/config change.

## What This Changes

Requests cheaper compaction and memory-flush model selection plus a flush-then-reset mode that skips compaction after durable memory is written.

## Best Possible Solution

Keep the shipped model overrides, then design reset-after-flush as an explicit default-off lifecycle contract with tests for active runs, queues, subagents, transcript archival, channel continuity, and provider/auth routing.

## Reproduction Assessment

Not applicable as a bug reproduction, but the remaining feature gap is source-reproducible: current main has no `resetAfterFlush` schema key and the memory-flush path persists metadata instead of resetting after success.

## Solution Assessment

Unclear until maintainers define the lifecycle contract; the shipped model overrides are the narrow cost-relief path, while reset-after-flush needs explicit state, safety, and upgrade boundaries.
