## Summary

Keep open: current main and the latest release cover the compaction and memory-flush model override pieces, but the requested flush-then-reset mode is still absent from the config contract and runtime path.

## What This Changes

The issue requests cheaper compaction and memory-flush model overrides plus an optional flush-then-reset mode that skips compaction after a successful memory flush.

## Best Possible Solution

Design a guarded, default-off reset-after-flush policy that reuses the existing gateway reset lifecycle after a confirmed successful flush, with documented failure behavior and regression coverage for queues, subagents, channel continuity, and provider/auth state.
