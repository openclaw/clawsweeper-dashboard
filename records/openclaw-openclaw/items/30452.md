## Summary

Keep open: current main and the latest release already cover the requested exact compaction and memory-flush model overrides, but the central flush-then-reset mode is still absent from schema, docs, tests, and runtime. The remaining work is a session-lifecycle product decision, not a narrow bug fix or a safe cleanup close.

## What This Changes

The issue requests configurable maintenance-model overrides for compaction and memory flush plus an optional mode that resets the session after a successful memory flush instead of continuing into compaction.

## Best Possible Solution

Keep the shipped model overrides, then decide a default-off post-flush reset policy that delegates to the gateway reset service, preserves explicit session settings, and proves successful flush, flush failure, queued followup, and subagent behavior after the overlapping per-agent config work is settled.

## Reproduction Assessment

Not applicable: this is a feature and lifecycle-policy request, not broken behavior under an existing documented contract. Source inspection gives a high-confidence check that the model overrides exist and the reset-after-flush mode does not.

## Solution Assessment

Partly yes: exact compaction and memory-flush model overrides are already implemented in the canonical config path. The proposed reset flag is not yet the best complete solution until maintainers define state-preservation, failure, and reset-lifecycle semantics.
