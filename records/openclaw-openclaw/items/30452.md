## Summary

Keep open: current main and v2026.6.8 cover the compaction and memory-flush model override pieces, but the requested flush-then-reset mode is still absent and changes core session lifecycle behavior that needs maintainer product review.

## What This Changes

Request adds cost-control compaction settings: a compaction model override, a memory-flush model override, and a configurable mode that flushes durable memory then resets instead of compacting.

## Best Possible Solution

Keep the shipped model overrides, then add flush-then-reset only as an explicit default-off lifecycle contract wired through the reset service with docs and tests for active runs, queues, subagents, transcript archival, and provider/auth routing.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main supports the two model override subparts but has no resetAfterFlush or equivalent flush-then-reset mode.

## Solution Assessment

Unclear until maintainers define the session-lifecycle contract. The shipped model overrides are the narrow cost-relief path; automatic reset after memory flush needs explicit default-off safety boundaries before implementation.
