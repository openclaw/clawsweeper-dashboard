## What This Changes

## Best Possible Solution

Design a guarded, default-off reset-after-flush policy that reuses the existing gateway reset lifecycle, defines flush success/failure behavior, and covers queued follow-ups, subagents, channel continuity, provider/auth routing, docs, and regression tests.
