## Summary

Keep open: current main covers the compaction and memory-flush model override pieces, but the requested flush-then-reset mode is still absent and needs a maintainer session-lifecycle/config decision.

## What This Changes

The issue requests configurable housekeeping models for compaction and memory flush plus a new mode that flushes memory and resets the session instead of compacting.

## Best Possible Solution

Keep the shipped model overrides, then design a guarded memoryFlush.resetAfterFlush-style mode around the existing reply/session reset paths with documented timing, safety constraints, schema/docs, and focused regression coverage.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Source and docs inspection gives high confidence that the model override portions exist and that no reset-after-flush setting is currently accepted or implemented.

## Solution Assessment

Unclear. The implemented model overrides are a good narrow solution for the cost-control pieces, but the remaining reset-after-flush behavior is a new session-lifecycle policy that should be designed before implementation.
