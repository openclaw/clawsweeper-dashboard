## Summary

Keep open: current main now covers the compaction and memory-flush model override parts, but the requested flush-then-reset mode is still absent and needs maintainer lifecycle/UX judgment.

## What This Changes

Request adding configurable compaction and memory-flush model overrides plus a mode that flushes durable memory and resets the session instead of compacting.

## Best Possible Solution

Keep the implemented model overrides, then design a guarded core lifecycle policy for reset-after-flush with schema/docs/tests and explicit behavior for active runs, subagents, and channel continuity.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that model overrides exist on current main, while the requested reset-after-flush mode is not present.

## Solution Assessment

Unclear for the remaining reset mode. The model override pieces are now implemented, but automatically resetting after a flush changes session lifecycle semantics and should be decided deliberately before a code path is added.
