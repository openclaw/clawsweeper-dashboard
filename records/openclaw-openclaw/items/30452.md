## Summary

Keep open: current main implements the compaction and memory-flush model override portions, but the requested flush-then-reset mode is still absent and requires a new session-lifecycle config decision.

## What This Changes

Request configurable compaction and memory-flush model overrides plus a memory-flush mode that resets the session instead of running normal compaction.

## Best Possible Solution

Keep the shipped model overrides and design a guarded `memoryFlush.resetAfterFlush`-style mode around the existing session-reset cleanup path, with documented timing and safety constraints.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Source and docs inspection gives high confidence that the model override pieces exist and that no reset-after-flush setting is currently accepted or implemented.

## Solution Assessment

Unclear. The current implementation solves the model-cost pieces, but reset-after-flush is a new lifecycle mode that should be designed around reset safety before implementation.
