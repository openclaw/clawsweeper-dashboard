## Summary

Keep open: current main implements the compaction and memory-flush model override portions, but the requested flush-then-reset mode is still absent from config, schema, docs, and runtime flow. The remaining work is a session-lifecycle policy/config feature, and the issue discussion plus related searches do not show a merged or open replacement that fully owns that specific behavior.

## What This Changes

Add cheaper-model overrides for compaction and memory flush, plus an opt-in mode that resets the session after memory flush instead of running compaction.

## Best Possible Solution

Keep the shipped model overrides, then design a guarded reset-after-flush mode around the existing reply/session reset lifecycle with explicit schema, docs, failure semantics, and regression coverage.

Not applicable. This is a feature/config request; current-main source, schema, docs, and targeted searches show the model overrides exist and the reset-after-flush option does not.
