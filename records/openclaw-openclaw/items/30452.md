## Summary

Keep open: current main and v2026.6.8 implement the compaction and memory-flush model override pieces, but there is still no `resetAfterFlush` or equivalent flush-then-reset lifecycle mode. Related open work covers per-agent override resolution and memory-flush failure recovery, not the remaining product decision.

## What This Changes

Request adds configurable compaction and memory-flush model overrides plus an optional mode to flush durable memory and reset instead of compacting.

## Best Possible Solution

Define a maintainer-approved optional reset-after-flush lifecycle contract that reuses the shipped model override knobs while specifying reset ordering, state preservation, provider/auth behavior, user visibility, failure handling, and regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug: this is a feature request. Source inspection does give a high-confidence presence/absence check: model override fields exist, while reset-after-flush config and behavior are absent on current main and v2026.6.8.

## Solution Assessment

No, not as an automatic fix: the model override pieces are already the shipped solution, but flush-then-reset needs a maintainer-approved lifecycle policy before implementation. The safer path is to design the reset boundary first, then implement one focused config/runtime/docs/test change.
