## Summary

Keep open: current main has implemented the compaction and memory-flush model override parts, but the requested flush-then-reset mode is still absent from the config contract and reply/session reset flow. The remaining work is a session lifecycle policy feature that needs maintainer product review, especially because it overlaps reset-time memory handling.

## What This Changes

The issue requests cheaper configurable models for compaction and memory flush, plus an optional mode that flushes durable memory and then resets the session instead of compacting.

## Best Possible Solution

Keep the shipped model overrides, then design a guarded reset-after-flush mode with explicit schema, docs, reset ordering, failure semantics, and regression coverage around the existing reply/session reset lifecycle.

## Reproduction Assessment

Not applicable. This is a feature/config request; source inspection shows current main supports the model overrides but does not expose or execute a reset-after-flush policy.

## Solution Assessment

Unclear. The model override portions are already the narrow maintainable cost-control path, while flush-then-reset needs maintainer-selected lifecycle and state-safety semantics before code should be added.
