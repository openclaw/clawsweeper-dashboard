## Summary

Keep open: current main and v2026.5.18 still preserve the Mattermost slash-command state split that can leave callbacks on the 503 fallback, and the focused fix path is open but not merged.

## What This Changes

Mattermost native slash commands return HTTP 503 after registration instead of reaching the initialized command handler.

## Best Possible Solution

Keep this issue open until a Mattermost fix lands and ships; the fix should make route registration and monitor activation share one slash-command state instance and include dual-loader regression coverage.

## Reproduction Assessment

Yes. Current main can register the route through the bundled-entry loader while activation writes through the monitor import path, and the route-side empty state returns the exact reported 503 response.

## Solution Assessment

Yes. The narrow maintainable solution is to land or revise the existing focused Mattermost fix so the route and activation path share one state module, with a regression test for the loader boundary.
