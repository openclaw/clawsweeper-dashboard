## Summary

Keep this open: current main and the latest release still preserve the Mattermost slash-command split-state failure mode, and the focused fix PR remains open and unmerged. User comments and the duplicate report corroborate both the regression and that a local patch resolves it.

## What This Changes

Mattermost native slash-command callbacks return HTTP 503 "not yet initialized" after command registration in v2026.4.15 and later releases.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/68089, or an equivalent Mattermost plugin fix, so route registration and monitor activation share one slash-command state instance with dual-loader regression coverage.

Yes, source-reproducible with high confidence. Current main can load the route through the bundled-entry loader while monitor activation writes through the ESM import path, and the route-side empty state returns the exact reported 503 response.
