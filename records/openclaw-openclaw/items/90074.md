## Summary

Keep open: current main and the latest release still route explicit non-ChatGPT OpenAI provider config through the direct Images API path, so the remaining request is a compatibility-sensitive product decision or a focused diagnostic improvement, not conservative cleanup.

## What This Changes

Request clearer or changed OpenAI image-generation routing so `openai/gpt-image-2` can use Codex OAuth instead of the direct API-key route when explicit OpenAI provider config is present.

## Best Possible Solution

Preserve the shipped explicit-direct route contract unless maintainers intentionally change it, and add focused route/auth diagnostics for direct OpenAI image failures when Codex OAuth is also available.

## Reproduction Assessment

Partly: source and tests show with high confidence that explicit direct OpenAI provider config prevents Codex Responses routing. The exact live 401 missing-scope response was not reproduced in this read-only review.

## Solution Assessment

Unclear: automatically preferring OAuth over explicit direct config conflicts with current docs and tests. A diagnostic-only change may be the narrowest maintainable fix unless maintainers choose a compatibility-sensitive route-precedence change.
