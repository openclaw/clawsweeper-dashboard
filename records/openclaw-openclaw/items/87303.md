## Summary

Keep open: current main and v2026.6.8 still derive identity mention names with ASCII word boundaries that fail CJK names, and https://github.com/openclaw/openclaw/pull/89864 is the open unmerged fix candidate for this issue.

## What This Changes

The issue reports that DingTalk group-chat mentions fail when the target nickname is a single CJK character while multi-character nicknames work.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/89864 so identity-derived CJK names match standalone group mentions without matching substrings inside longer CJK text.

## Reproduction Assessment

Yes, from source. Current main derives an ASCII word-boundary regex for identity names, and a focused Node check shows that pattern does not match standalone CJK mention text like `@包 你好`.

## Solution Assessment

Yes. Fixing the shared `deriveMentionPatterns` helper is the best owner boundary because identity-derived mention matching is shared through the plugin SDK and consumed by multiple channel/plugin surfaces.
