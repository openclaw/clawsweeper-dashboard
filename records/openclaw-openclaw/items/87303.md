## Summary

Keep open: current main and the latest release still derive identity mention names with ASCII word boundaries that fail CJK mention text, and the open linked PR is the candidate fix that should resolve this issue when merged.

## What This Changes

The issue reports that group-chat mention detection treats single-character Chinese nicknames such as @包 as plain text instead of a mention.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/89864 so identity-derived CJK names match standalone group mentions without matching substrings inside longer CJK text.

## Reproduction Assessment

Yes. Current main derives an ASCII word-boundary regex for identity names, and a focused Node check shows that derived shape does not match standalone CJK mention text like `@包 你好`.

## Solution Assessment

Yes. Fixing the shared `deriveMentionPatterns` helper is the best owner boundary because the derived mention regexes are exported through the plugin SDK and consumed by multiple channel surfaces.
