## Summary

Keep open: current main still has a source-reproducible WebChat heartbeat visibility leak, and https://github.com/openclaw/openclaw/pull/80378 is an open unmerged closing PR for the same issue.

## What This Changes

Heartbeat polling in WebChat should stay silent and not render heartbeat tool activity or HEARTBEAT_OK assistant text.

## Best Possible Solution

Review, update if needed, and land https://github.com/openclaw/openclaw/pull/80378 or a replacement that suppresses heartbeat tool events and ack-only heartbeat text at the Gateway projection boundary.

## Reproduction Assessment

Yes, at source level. Heartbeat runs are marked isHeartbeat, current server-chat broadcasts heartbeat-originated tool events without a heartbeat guard, and Control UI consumes those tool frames as live tool cards.

## Solution Assessment

Yes, the right fix belongs at the Gateway chat projection boundary so WebChat and node/session subscribers stop receiving silent heartbeat noise before UI rendering. The linked PR is the active fix path, but it should be checked for full coverage of both tool-card and HEARTBEAT_OK leak cases before landing.
