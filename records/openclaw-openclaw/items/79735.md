## What This Changes

The issue asks for WebChat heartbeat polls to stay silent by hiding both live tool-call cards and `HEARTBEAT_OK` assistant messages.

## Best Possible Solution

Review, land, or revise https://github.com/openclaw/openclaw/pull/79786 so WebChat suppresses heartbeat OK text and heartbeat-originated live tool cards while preserving external-channel heartbeat visibility controls.

## Reproduction Assessment

Yes, by source inspection: configure `channels.defaults.heartbeat.showOk: true`, run a WebChat heartbeat, and current main can bypass heartbeat chat suppression while still broadcasting heartbeat tool events to session subscribers. I did not perform a live browser reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix belongs at the Gateway WebChat projection/session.tool boundary, not by disabling heartbeat tools globally. The linked PR is the active path, but it should be reviewed or expanded to cover both visible text and tool-card leakage.
