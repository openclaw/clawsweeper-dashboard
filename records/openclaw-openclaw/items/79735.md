## Summary

Keep open: current main still has source-backed WebChat heartbeat leaks for both `HEARTBEAT_OK` when WebChat inherits `showOk: true` and heartbeat tool events rendered as chat tool cards; https://github.com/openclaw/openclaw/pull/79786 is an open implementation candidate but has not merged.

## What This Changes

The issue asks for WebChat heartbeat polls to run silently without visible tool cards or literal `HEARTBEAT_OK` messages.

## Best Possible Solution

Land a narrow Gateway/Control UI fix that suppresses heartbeat-originated WebChat assistant text and live tool cards while preserving external-channel heartbeat delivery and deliberate alert text.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure WebChat heartbeat visibility with `showOk: true` for the text leak, or register a Control UI-visible heartbeat run and emit tool events for the active session to see the Gateway broadcast events that the UI turns into tool cards.

## Solution Assessment

Yes. The narrow maintainable direction is to gate heartbeat-originated WebChat chat/tool projection at the Gateway/UI boundary, not to disable heartbeat tools or change external channel delivery semantics.
