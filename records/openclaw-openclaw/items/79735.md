## Summary

Keep open: current main still has a source-backed WebChat heartbeat leak, both through `showOk: true` allowing `HEARTBEAT_OK` chat output and through heartbeat tool events being mirrored to Control UI session subscribers; the linked open PR is the right place to resolve or narrow the remaining work.

## What This Changes

The issue asks for WebChat heartbeat polls to remain silent, without visible tool cards, `HEARTBEAT_OK` messages, or conversation interruption.

## Best Possible Solution

Review and land a narrow Gateway/UI fix that always suppresses heartbeat WebChat final text and heartbeat-originated live tool events from chat, while preserving external-channel `showOk` delivery and meaningful heartbeat alerts.

## Reproduction Assessment

Yes, by source inspection: with `channels.defaults.heartbeat.showOk: true`, a heartbeat run can bypass WebChat text suppression, and heartbeat tool events are still mirrored to `session.tool` where Control UI renders them as tool cards. I did not run a live browser reproduction in this read-only sweep.

## Solution Assessment

Yes for the direction: the narrow maintainable fix is at the Gateway chat projection/tool-event boundary, not by disabling heartbeat tools or hiding all Control UI tool cards. The open PR direction is useful for final text, but it should be checked against the tool-event leak before this issue closes.
