## Summary

Keep open: current main still has a source-backed webchat heartbeat tool-card leak, and https://github.com/openclaw/openclaw/pull/80378 is the active unmerged fix path with closing syntax.

## What This Changes

The issue asks for webchat heartbeat runs to stay silent by not rendering heartbeat tool calls or literal HEARTBEAT_OK messages in chat.

## Best Possible Solution

Land a narrow Gateway regression fix that suppresses heartbeat-originated webchat tool/session.tool broadcasts and HEARTBEAT_OK stream text while preserving normal tool cards and actionable heartbeat alert text.

## Reproduction Assessment

Yes, by source inspection: register a webchat-visible heartbeat run, emit a tool event, and the current Gateway path can broadcast it as agent/session.tool while the UI accepts session-scoped tool payloads as chat tool cards. I did not run a live browser reproduction.

## Solution Assessment

Yes, the narrow maintainable fix is to gate heartbeat-originated live tool projection at the Gateway webchat boundary, not to disable heartbeat tools or hide all Control UI tool cards. The open paired PR is the current implementation path and should be reviewed or updated rather than closing the issue early.
