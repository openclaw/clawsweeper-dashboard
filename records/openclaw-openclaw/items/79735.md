## Summary

Keep open: current main still has a source-reproducible path for heartbeat-originated tool events to reach WebChat/Control UI, and https://github.com/openclaw/openclaw/pull/80378 is open with closing syntax but unmerged. The literal acknowledgement text is partially mitigated by current UI-side filters, but the live heartbeat tool-card leak remains unresolved on main.

## What This Changes

Heartbeat runs in WebChat should stay silent instead of rendering background tool calls or the literal `HEARTBEAT_OK` acknowledgement in chat.

## Best Possible Solution

Land or replace the linked Gateway projection fix so heartbeat-originated tool events are suppressed before Control UI live streams while ack-only heartbeat output stays hidden and actionable heartbeat alerts remain visible.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: heartbeat runs register `isHeartbeat`, `server-chat.ts` broadcasts tool events without a heartbeat guard, and WebChat turns received `agent` tool events into live tool cards.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the Gateway chat projection boundary; https://github.com/openclaw/openclaw/pull/80378 is the active coordination point, but this issue should remain open until a fix lands.
