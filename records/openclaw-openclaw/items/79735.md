## What This Changes

Heartbeat polling in WebChat should run silently without visible heartbeat tool cards or `HEARTBEAT_OK` assistant messages.

## Best Possible Solution

Land or replace the linked Gateway projection fix so WebChat suppresses `HEARTBEAT_OK` regardless of `showOk` and suppresses heartbeat-originated live tool broadcasts while preserving normal tool cards and actionable heartbeat alerts.

## Reproduction Assessment

Yes. Source inspection against current main shows `showOk: true` can keep heartbeat acknowledgement text visible, and heartbeat `tool` events still broadcast to Control UI subscribers that render live tool cards.

## Solution Assessment

Yes. The narrow maintainable solution belongs at the Gateway chat projection boundary; the open linked PR is the coordination point, but it should be expanded or replaced so both reported symptoms are covered.
