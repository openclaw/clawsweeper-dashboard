## Summary

Keep open: current main has partial HEARTBEAT_OK hiding, but the live tool-event path still broadcasts heartbeat tool events to webchat session subscribers, matching the reported visible tool-call portion.

## What This Changes

The issue reports that webchat heartbeat polling surfaces visible tool calls and a HEARTBEAT_OK assistant message instead of staying silent.

## Best Possible Solution

Suppress heartbeat-originated live tool/session.tool events from Control UI chat while preserving deliberate heartbeat alert final text and compact heartbeat indicators.

## Reproduction Assessment

Yes, by source inspection: register a webchat heartbeat run context, emit a tool event, and the Gateway broadcasts it as agent/session.tool while the Control UI accepts session-scoped tool events as chat tool cards. I did not perform a live browser reproduction in this read-only sweep.

## Solution Assessment

Yes, the narrow maintainable fix is to gate heartbeat live tool-event broadcasting at the Gateway chat projection boundary, not to disable heartbeat tools or broadly hide all Control UI tool cards.
