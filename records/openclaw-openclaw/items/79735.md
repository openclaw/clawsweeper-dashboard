## What This Changes

Heartbeat polls in WebChat should run silently without visible tool cards or HEARTBEAT_OK assistant messages.

## Best Possible Solution

Land or replace the linked Gateway fix so WebChat always suppresses heartbeat acknowledgement text and heartbeat-originated live tool broadcasts while preserving normal tool cards and actionable heartbeat alert text.

Yes. Source inspection gives a high-confidence path: configure or hit a heartbeat-visible WebChat session, emit heartbeat assistant text or tool events, and current main can still project HEARTBEAT_OK when showOk is true and session.tool/agent tool payloads into Control UI.

## Solution Assessment
