## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Do not retry ambiguous ConnectionClosed calls:** `src/agents/agent-bundle-mcp-runtime.ts:506`
  - body: `ErrorCode.ConnectionClosed` can be raised after the MCP request has already been sent but before the response is received. Retrying that case can run non-idempotent tools twice; keep auto-reconnect to failures where the write definitely did not reach the server, or require an explicit idempotency contract before replay.
  - confidence: 0.87
- **[P2] Preserve current MCP client contracts on reconnect:** `src/agents/agent-bundle-mcp-runtime.ts:530-532`
  - body: Current main wraps `callTool` with the session request timeout/backoff path and configures clients with `tools/list_changed` catalog invalidation. The reconnect path creates a bare client and retries without those contracts, so a current-main port must carry the same client options and guarded request behavior into the reconnected session.
  - confidence: 0.78
