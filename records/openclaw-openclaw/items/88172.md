## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not replay in-flight ConnectionClosed tool calls:** `src/agents/agent-bundle-mcp-runtime.ts:506`
  - body: The SDK emits `ErrorCode.ConnectionClosed` to pending response handlers when the transport closes, after the request may already have been written to the MCP server. Classifying that as retryable and then calling the same tool again can duplicate non-idempotent MCP side effects such as shell, file, browser, or service actions; keep retry to pre-send failures or add an explicit idempotency contract.
  - confidence: 0.9
- **[P2] Bound MCP disposal before awaiting cron cleanup:** `src/gateway/server-cron.ts:394`
  - body: This new finally block awaits `retireSessionMcpRuntimeForSessionKey`, but current `disposeSession` still awaits streamable-HTTP `terminateSession()` without a deadline. A hung remote MCP cleanup can now keep an isolated cron run stuck in teardown; fold in bounded disposal or limit this cleanup to transports with bounded close before merge.
  - confidence: 0.82
