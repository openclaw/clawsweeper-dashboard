## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not replay in-flight ConnectionClosed tool calls:** `src/agents/agent-bundle-mcp-runtime.ts:506`
  - body: The SDK raises `ConnectionClosed` to pending response handlers after the request may already have been sent. Treating that as retryable calls the same MCP tool again, so non-idempotent shell/file/browser/service tools can run twice; keep retry to pre-send failures or require an idempotency contract before replay.
  - confidence: 0.9
- **[P2] Preserve current request guards on reconnect retry:** `src/agents/agent-bundle-mcp-runtime.ts:558`
  - body: The new retry call invokes `client.callTool` directly. Current main wraps tool calls in `runGuardedServerRequest` and passes the configured request timeout, so the port needs to route the retried call through the same guard or it can bypass backoff and timeout protections.
  - confidence: 0.82
