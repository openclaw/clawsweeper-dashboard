## Summary

Keep open: the patch changes the plugin-facing hook conversation ID contract too broadly and would strip Discord's canonical `user:`/`channel:` identities while current inbound claims and bindings still preserve them; the PR also lacks required real behavior proof.

## What This Changes

The PR adds a shared conversation-id normalizer, applies it to plugin message hook contexts, and updates mapper tests to expect common prefixes to be stripped.

## Best Possible Solution

Preserve provider-owned typed IDs and normalize only the specific fallback or wrapper-prefix cases that caused pending lookup mismatches, with coverage for Discord bindings plus Telegram/Slack hook lookup behavior.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the diff strips `user:` and `channel:` from message hook contexts while inbound claim and Discord binding paths preserve those IDs. The original Telegram/Slack pending-lookup bug is not proven with real setup evidence.

## Solution Assessment

No. Applying prefix stripping to every plugin message context is too broad; a safer fix should be resolver-aware or fallback-only and preserve channel-owned identity grammar.
