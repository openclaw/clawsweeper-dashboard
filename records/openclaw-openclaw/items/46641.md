## Summary

Keep open. Current main has opt-in fixed compaction start/end notices and a bundled notifier hook, but it still lacks the requested configurable pre-threshold active-channel prompt that waits for user guidance before silent memory flush or compaction.

## What This Changes

The issue requests a visible, configurable user notification before automatic context compaction so the user can tell the agent what to preserve.

## Best Possible Solution

Define a core compaction UX contract for pre-notification text, threshold, timeout, active-channel delivery, and response handling, while keeping existing silent memory flush and status-only notices compatible.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has fixed lifecycle notices and silent memory flush, not the requested pre-threshold user prompt and response window.

## Solution Assessment

No. Overloading the existing notifyUser status-notice path would not solve the issue cleanly; the maintainable path needs an explicit pre-compaction prompt contract with maintainer-approved timeout and response semantics.
