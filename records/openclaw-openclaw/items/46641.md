## Summary

Keep open. Current main still only has fixed opt-in compaction status notices and hook-supplied status messages; it does not provide a configurable pre-threshold prompt, wait for user guidance, or feed that guidance into memory flush before compaction.

## What This Changes

Add a configurable user-visible pre-compaction prompt that asks the active channel what to remember before memory flush and compaction proceed.

## Best Possible Solution

Define an explicit compaction pre-notification contract under agents.defaults.compaction, or an approved lifecycle hook if maintainers prefer, covering threshold, prompt text, timeout, active-channel delivery, and how the response is incorporated before memory flush.

## Reproduction Assessment

Not applicable as a classic bug reproduction because this is a feature request. The current-main source, docs, and tests give high-confidence evidence that the requested pre-compaction prompt and response window are still absent.

## Solution Assessment

No, the existing notifyUser and hook-message paths are not the best complete solution because they only send status notices. A maintainable fix needs an explicit pre-compaction prompt contract with response handling approved by maintainers.
