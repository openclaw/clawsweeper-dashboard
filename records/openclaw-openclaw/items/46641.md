## Summary

Keep this issue open. Current main still has only opt-in fixed compaction lifecycle notices plus a silent internal memory-flush turn; it does not provide the requested configurable pre-threshold user prompt, custom message, or response window before flush/compaction.

## What This Changes

The issue requests a user-facing notification before auto-compaction that asks the active-channel user what should be remembered, with configurable enablement, message text, and token threshold.

## Best Possible Solution

Define a dedicated core compaction UX contract under agents.defaults.compaction, with a maintainer-approved config shape for pre-notification text, threshold, timeout, active-channel delivery, and how any user response is folded into memory flush or compaction context; then cover schema/help/docs plus reply-layer and memory-flush tests.

## Reproduction Assessment

Not applicable as a classic bug reproduction because this is a feature request. The current-main verification path is high confidence: schema, docs, runtime code, and tests all show fixed lifecycle notices and silent memory flush rather than the requested pre-threshold user prompt.

## Solution Assessment

No, the existing notifyUser path is not the best or complete solution for this request. The maintainable direction is a separate, explicit pre-compaction prompt contract rather than overloading the fixed start/completion status-notice boolean without response semantics.
