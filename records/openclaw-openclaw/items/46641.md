## Summary

Keep open. Current main still lacks the requested pre-compaction user prompt: the closest shipped pieces are opt-in fixed lifecycle notices and the silent internal memory-flush path, but neither supports a custom threshold/message or waits for user guidance before flush/compaction.

## What This Changes

Request a configurable user-visible prompt before auto-compaction so users can tell the agent what to preserve before memory flush and compaction proceed.

## Best Possible Solution

Define a dedicated core compaction UX contract under agents.defaults.compaction for pre-notification text, threshold, timeout, active-channel delivery, and response handling, then update schema/help/docs plus reply-layer and memory-flush tests.

## Reproduction Assessment

Not applicable as a classic bug reproduction because this is a feature request. The current-main source/docs check is high confidence: schema, runtime, tests, and docs show fixed lifecycle notices plus silent memory flush, not the requested pre-threshold user prompt.

## Solution Assessment

No, overloading the existing notifyUser status-notice path would not solve the issue cleanly. The maintainable solution needs an explicit pre-compaction prompt contract with response-window semantics approved by maintainers.
