## Summary

Keep open. Current main has per-tool collapsible rows and a tool-call show/hide toggle, but it does not implement the requested aggregate per-turn summary such as a single “5 tool calls” control; related expand-by-default work covers a different preference.

## What This Changes

The issue requests a configurable Control UI webchat mode that collapses multiple tool executions into one expandable summary to reduce conversation clutter.

## Best Possible Solution

Decide the product shape first, then implement a focused Control UI display mode that groups tool activity per turn while preserving per-tool details, existing show/hide controls, and manual expansion behavior.
