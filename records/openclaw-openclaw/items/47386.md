## Summary

Keep open. Current main and the latest release still render WebChat tool activity as individual tool-message disclosures with a show/hide toggle, while the requested turn-level aggregate summary mode and detailed-versus-summary preference remain unimplemented.

## What This Changes

The issue requests a WebChat presentation mode that collapses multiple tool executions into a single expandable summary, with a detailed-versus-summary preference.

## Best Possible Solution

Design one WebChat tool-call summary disclosure that aggregates related tool call/result messages per assistant turn, preserves per-tool detail access and show/hide behavior, and aligns any persisted preference with adjacent expand/activity-panel work.

## Reproduction Assessment

Not applicable. This is a feature request; source and release inspection confirm WebChat currently has individual tool-message disclosures plus a show/hide toggle, not a turn-level summary mode.

## Solution Assessment

Unclear. The aggregate summary direction is plausible, but maintainers should first choose grouping semantics, default behavior, and how it interacts with https://github.com/openclaw/openclaw/pull/54838 and https://github.com/openclaw/openclaw/issues/12831.
