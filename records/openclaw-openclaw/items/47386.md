## Summary

Keep open. Current main has collapsible per-tool cards and per-tool-message disclosures, but it still renders each streamed tool message/card separately and does not provide the requested single count-level summary mode or detailed/summary preference.

## What This Changes

Add a WebChat/Control UI mode that replaces many individual tool execution messages with one expandable tool-call summary.

## Best Possible Solution

Implement the aggregation in the Control UI chat rendering layer: group multiple tool calls/results from the same turn behind one default-collapsed count summary while preserving per-tool details, the existing show/hide tool-calls toggle, and canvas/tool-output side-panel behavior.

## Reproduction Assessment

Yes. A static reproduction is clear from current main: with more than one toolMessages entry, buildChatItems appends each tool message separately, and grouped-render maps each ToolCard separately instead of producing one aggregate disclosure.

## Solution Assessment

Yes, the maintainable direction is a narrow Control UI/WebChat rendering and settings change, not a gateway or transcript-format change. It should reuse the existing tool-card extraction and expansion-state seams rather than introducing a parallel tool-output model.
