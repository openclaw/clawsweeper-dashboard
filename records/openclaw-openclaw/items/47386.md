## Summary

Keep open. Current main has per-message tool disclosure buttons and a show/hide toggle, but it does not aggregate multiple tool executions into one collapsible summary; the linked open PR is adjacent expand-by-default work, not this requested summary mode.

## What This Changes

The issue requests a webchat mode that summarizes multiple tool executions behind one expandable control instead of rendering each tool call/output as separate visible entries.

## Best Possible Solution

Design a Control UI summary mode that aggregates tool calls/results per turn into one disclosure while preserving the existing show/hide and per-disclosure expansion state; add a persisted setting only after the UX/default policy is decided.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection shows the current renderer uses individual tool-message disclosures, but no existing contract says those must be aggregated.
