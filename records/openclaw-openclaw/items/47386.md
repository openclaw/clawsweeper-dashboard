## Summary

Keep open. Current main still renders webchat tool executions as separate per-message or per-card disclosures with a show/hide setting, while this request asks for a new aggregate collapsible summary mode; the adjacent open PR at https://github.com/openclaw/openclaw/pull/54838 is expand-by-default work for a different issue, not turn-level aggregation.

## What This Changes

Request a webchat UI mode that replaces noisy per-tool execution rows with one collapsible aggregate tool-call summary, plus a configurable detailed-versus-summary presentation setting.

## Best Possible Solution

Design a turn-level Control UI tool-call summary disclosure that groups related calls/results while preserving existing per-disclosure expansion state; add persistence only after the default and setting semantics are decided.
