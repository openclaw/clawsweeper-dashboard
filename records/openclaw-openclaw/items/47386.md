## Summary

Keep open. Current main still shows tool executions as separate message or tool-card disclosures, while this request asks for a new aggregate collapsible summary mode with a detailed-versus-summary setting.

## What This Changes

Add a webchat presentation mode that groups multiple tool executions into one collapsible summary, with an optional detailed/summary setting.

## Best Possible Solution

Design a turn-level Control UI tool-call summary disclosure that groups related calls/results while preserving current per-disclosure expansion state; add persistence only after the default and setting semantics are decided.
