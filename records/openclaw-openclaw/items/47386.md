## Summary

Keep open. Current main has per-tool-card and per-tool-message disclosures, but it still emits each streamed tool message/card separately and has no persisted detailed-vs-summary display mode for a single count-level summary.

## What This Changes

The issue requests a WebChat tool-output mode that collapses multiple tool executions into one count-level summary with expandable details and a detailed/summary preference.

## Best Possible Solution

Implement the aggregation in the Control UI chat build/render/settings layer so multiple tool calls/results from one turn default to one collapsed count summary while preserving per-tool details, existing show/hide behavior, and canvas/tool-output sidebar actions.

## Reproduction Assessment

Yes. Source inspection on current main is enough: multiple toolMessages are appended individually and multiple ToolCard entries are rendered one by one rather than through one aggregate disclosure.

## Solution Assessment

Yes. The narrow maintainable direction is a Control UI rendering/settings change that reuses existing tool-card extraction and expansion-state seams, not a gateway transcript or protocol change.

Group multiple visible tool calls/results from the same turn behind one default-collapsed count-level summary, keep expandable per-tool details inside it, preserve the existing chatShowToolCalls show/hide toggle, and do not change gateway transcript/session protocol or core agent behavior.
