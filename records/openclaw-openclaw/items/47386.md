## Summary

Keep open. Current main has per-tool-message and per-tool-card collapse controls, but it still appends and renders each streamed tool entry separately and does not provide a persisted detailed-vs-summary display mode.

## What This Changes

Add a Control UI/WebChat mode that summarizes multiple tool executions into one collapsible count-level entry instead of rendering each tool item separately.

## Best Possible Solution

Implement the aggregation in the Control UI chat build/render/settings layer so multiple tool calls from one turn default to one collapsed count summary while preserving existing show/hide, per-tool details, expansion state, and sidebar/canvas actions.

## Reproduction Assessment

Yes. Source inspection on current main is enough: toolMessages are pushed as individual message items and renderInlineToolCards maps each ToolCard separately, while storage has no detailed/summary display setting.

## Solution Assessment

Yes. The narrow maintainable solution is a Control UI rendering/settings change that reuses the existing tool-card extraction and expansion-state seams, not a gateway transcript or protocol change.
