## Summary

Keep open. Current main and the latest release still render WebChat tool activity as individual tool-message disclosures with a show/hide toggle, while the requested turn-level aggregate summary mode and detailed-versus-summary setting remain unimplemented.

## What This Changes

Request a configurable WebChat presentation mode that replaces noisy per-tool output rows with a collapsible turn-level tool-call summary.

## Best Possible Solution

Design one WebChat tool-call summary disclosure that aggregates related tool call/result messages per assistant turn, preserves per-tool detail access and show/hide behavior, and coordinates any persisted preference with the expand-default and activity-panel work.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source and release inspection confirm WebChat currently has individual tool-message disclosures plus a show/hide toggle, not a turn-level summary mode.

## Solution Assessment

Unclear. The aggregate summary direction is plausible, but maintainers should first choose grouping semantics, default behavior, and how it interacts with https://github.com/openclaw/openclaw/pull/54838 and https://github.com/openclaw/openclaw/issues/12831.
