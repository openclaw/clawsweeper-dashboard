## Summary

Keep open. Current main still parses Telegram `/steer check progress` as subagent steering with `check` as the target, and two open closing PRs (#76527, #76537) already target this behavior, so this issue should remain open until one fix lands or maintainers choose a different command grammar.

## What This Changes

Request Telegram `/steer <message>` to deliver guidance to the active run without requiring an explicit subagent target.

## Best Possible Solution

Choose one command contract, then land or replace the linked fix with focused coverage for Telegram native commands, shared text-channel `/steer`, subagent label targets, active-run steering, docs, and changelog.

## Reproduction Assessment

Yes from source inspection: Telegram native `/steer check progress` is rebuilt as `/steer check progress`, then the shared handler treats `check` as the subagent target and the remaining words as the message. I did not run a live Telegram or Vitest reproduction in this read-only sweep.

## Solution Assessment

Unclear. Making the target optional may be right, but maintainers need to decide how text/native `/steer` should coexist with label-based subagent targets and the existing UI active-run shorthand.
