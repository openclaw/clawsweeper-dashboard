## Summary

Keep open. Current main still routes Telegram native `/steer text` through the subagent steer command that expects a target token, and this issue now has open closing PRs, so the issue should stay open until maintainers choose and land the intended command grammar.

## What This Changes

Allow Telegram `/steer <message>` without a subagent id to deliver guidance to the active run instead of treating the first word as a subagent target.

## Best Possible Solution

Review the linked PRs together, decide the intended `/steer` grammar, then land one implementation with focused parser, Telegram/native-command, docs, and changelog coverage.
