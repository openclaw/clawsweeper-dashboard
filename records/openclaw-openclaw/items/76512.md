## Summary

Keep open. Current main and the latest release still route `/steer` as the sub-agent steering shortcut, while the linked shorthand request was closed stale rather than implemented; changing this now needs an explicit command-UX decision because `/steer` already has documented sub-agent targeting semantics.

## What This Changes

Allow `/steer <message>` from Telegram/current sessions to steer the active main run without requiring `/queue steer`, while preserving sub-agent steering when a target is supplied.

## Best Possible Solution

Define and document an unambiguous main-session steering command rule across text and native command surfaces, then cover Telegram/native-command and existing sub-agent `/steer` forms with regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: send `/steer continue work` through the command surface and the dispatcher treats `continue` as a sub-agent target before any main-run queue steering path can apply. I did not execute a live Telegram turn.

## Solution Assessment

Unclear as requested, because overloading `/steer` conflicts with the documented sub-agent shortcut, including label-based targets. A maintainer should choose whether fallback-to-main, numeric-only sub-agent shorthand, or a different command is the best long-term syntax.
