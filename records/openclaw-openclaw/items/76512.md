## Summary

Close: current main and the latest release now implement `/steer <message>` as a top-level current-session active-run command, so Telegram text no longer has to provide or parse a sub-agent id for this workflow.

## What This Changes

The issue asks for Telegram `/steer <message>` during an active run to steer the current agent instead of treating the first word as a sub-agent id.

## Best Possible Solution

Keep the shipped top-level current-session `/steer` command and leave `/subagents` focused on visibility, focus, and inspection commands.

## Reproduction Assessment

No current failing reproduction remains. Source and shipped test evidence show `/steer <message>` is now handled as current-session active-run steering before sub-agent command parsing.

## Solution Assessment

Yes. The current implementation is a better fit than the older one-active-subagent proposals because top-level `/steer` now targets the current session's active run directly and preserves a normal-prompt fallback.
