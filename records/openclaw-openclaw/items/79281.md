## Summary

Keep open: current main has generic current-conversation binding support and low-level thread-binding helpers, but it does not provide the requested per-sender default ACP thread-binding preset, opt-in policy surface, or manifest diagnostic; the group-sharing default needs maintainer API/product review.

## What This Changes

Request a public plugin-SDK default ACP thread-binding helper or opt-in flag, with per-sender group isolation and a diagnostic for inconsistent conversation-binding metadata.

## Best Possible Solution

Design an explicit, documented SDK helper that channel plugins can opt into, with sender-scoped group behavior unless maintainers choose a different policy, plus diagnostics for inconsistent conversation-binding metadata.

## Reproduction Assessment

Yes for the source-level absence: the requested helper and group-sharing option are not present, and current generic binding keys do not include sender identity. I did not live-run the WeChat setup, so the third-party duplication impact remains issue-context evidence.

## Solution Assessment

Unclear pending maintainer design review: a public SDK preset is plausible, but the exact sender/group default and whether to add a config or plugin-option knob are product/API decisions. The safer direction is an explicit documented helper before any automatic behavior change.
