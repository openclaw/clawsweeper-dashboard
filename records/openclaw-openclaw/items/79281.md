## Summary

Keep open: current main has low-level thread-binding seams and some generic current-conversation support, but it does not provide the requested per-sender default ACP binding preset or manifest diagnostic, and the group-sharing defaults need maintainer API/product review.

## What This Changes

Request to add a default ACP thread-binding SDK preset or opt-in flag so channel plugins can register safe bindings without duplicating adapter code, plus a diagnostic for manifest-only declarations.

## Best Possible Solution

Design a narrow documented plugin-SDK helper for explicit channel-plugin opt-in, with group-safe sender scoping if accepted, and add a manifest diagnostic when a channel advertises binding placement without a runtime manager or supported generic path.

## Reproduction Assessment

Yes for the current limitation from source: ACP thread spawn still requires a session-binding capability, while Slack has no binding support and LINE declares placement without a manager. I did not live-run the WeChat setup, so the external plugin impact is accepted as issue-context evidence rather than direct local reproduction.

## Solution Assessment

Unclear pending maintainer design review: a default SDK preset is a plausible maintainable direction, but the exact per-sender/group-sharing default and whether to expose an OpenClaw config flag are product/API decisions. The safer landing is an explicit, documented SDK helper before any automatic behavior change.
