## Summary

Keep open: current main still lacks the requested default per-sender ACP thread-binding SDK preset or opt-in helper, and the secondary manifest/runtime diagnostic is also not implemented. This needs maintainer API/product review because the default group-sharing behavior affects persistent ACP state isolation.

## What This Changes

Request adding a default ACP thread-binding SDK preset or opt-in config flag for channel plugins, plus a diagnostic when a channel advertises conversation bindings without registering a runtime manager.

## Best Possible Solution

Design an explicit, documented plugin-SDK helper for channel-plugin opt-in, choose group-safe sender scoping deliberately, and add a manifest/runtime diagnostic for advertised binding placement without a usable manager.

## Reproduction Assessment

Yes for the current limitation from source: ACP thread spawn still requires a session-binding adapter, the public SDK lacks the requested default preset, and LINE/Slack remain unsupported in the inspected binding coverage. I did not live-run the external WeChat setup, so that impact remains accepted issue-context evidence.

## Solution Assessment

Unclear pending maintainer design review. A documented SDK preset is a plausible maintainable direction, but the exact sender-scoping default, group-sharing opt-in, and optional config surface are API/product decisions.
