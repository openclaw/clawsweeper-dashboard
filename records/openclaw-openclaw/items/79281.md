## Summary

Keep open: current main has generic current-conversation binding support that reduces some adapter work for chat plugins, and the linked WeChat plugin PR is useful evidence of duplication, but main still lacks the requested sender-scoped SDK preset, group-sharing opt-in contract, and manifest diagnostic.

## What This Changes

The issue asks for a default ACP thread-binding SDK preset or opt-in flag so channel plugins can bind ACP sessions without reimplementing large SessionBindingAdapter code.

## Best Possible Solution

Design a documented, explicit plugin-SDK helper or generic binding extension that supports sender-scoped group isolation, clear shared-group opt-in, migration-safe defaults, and a diagnostic for advertised bindings that cannot route at runtime.

## Reproduction Assessment

Yes for the source-level limitation: current main has no sender-scoped default preset and the binding contracts do not include senderId. I did not reproduce the live WeChat failure, so that part remains issue-context evidence rather than direct runtime proof.

## Solution Assessment

Unclear pending maintainer design review. A plugin-SDK helper is plausible, but the safest shape should build on current generic binding support while making group sharing and sender scoping explicit.
