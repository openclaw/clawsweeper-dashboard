## Summary

Keep open. Current main exposes low-level thread-binding SDK pieces and adapter-unavailable errors, but it does not provide the requested default per-sender ACP/thread-binding preset; this is a plugin API and product-design feature, not a closeable duplicate or ClawHub-only plugin.

## What This Changes

Request an additive plugin SDK default ACP/session thread-binding helper, with optional opt-in semantics, so third-party channel plugins can register per-sender-safe bindings with much less custom code.

## Best Possible Solution

Design an additive SDK runtime helper that defaults to per-sender binding keys, requires explicit opt-in for shared group bindings, documents the channel/plugin contract, and leaves platform-specific transport or native thread creation in each plugin.

## Reproduction Assessment

Not applicable as a feature request. Source inspection does confirm the absence of the proposed default helper and the current adapter-unavailable path for channels without a registered binding capability.

## Solution Assessment

Unclear until maintainer API review. A narrow additive SDK helper looks maintainable, but the default group-sharing semantics and any opt-in config surface should be settled before implementation.
