## Summary

Keep this open. Current main still lacks the requested default/per-sender plugin SDK thread-binding preset, the documented ACP failure mode still applies when a channel/account has no registered binding adapter, and the related open issues track adjacent behavior rather than superseding this SDK helper request.

## What This Changes

The issue requests a default ACP thread-binding preset or opt-in flag so third-party channel plugins can register session bindings without reimplementing large channel-specific adapters.

## Best Possible Solution

Add a narrow, documented `openclaw/plugin-sdk/thread-bindings-runtime` helper only after maintainers choose the binding-key and opt-in policy, then cover it with SDK contract tests and channel-style examples without silently changing existing channel behavior.

## Reproduction Assessment

Yes for the source-level limitation: current main still requires ACP `mode="session"` to use thread binding, and thread binding fails when the channel/account has no adapter. I did not run a live WeChat/LINE channel repro during this read-only review.

## Solution Assessment

Unclear as a final implementation: a narrow SDK helper fits the plugin-boundary problem, but the per-sender default, shared-group opt-in, and optional config flag need maintainer API/product choice first.
