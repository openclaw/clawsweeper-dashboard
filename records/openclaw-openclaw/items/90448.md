## Summary

Keep open: current main still has the reported regression path. The runtime context lookup now resolves `openai` routes to canonical `openai`, while doctor removes `models.providers.openai-codex` when `models.providers.openai` already exists, so a legacy Codex context override can still be dropped instead of migrated.

## What This Changes

Preserve legacy Codex `contextTokens`/`contextWindow` model metadata across the OpenAI/Codex provider unification.

## Best Possible Solution

Fix this in doctor migration by moving or merging legacy `openai-codex` model context metadata into canonical `openai` when the canonical model entry lacks an explicit context cap; keep steady-state runtime on the canonical provider path.

Yes, source-reproducible: current main resolves Codex-runtime OpenAI context lookup to `openai`, while doctor removes the legacy `openai-codex` provider entry when canonical `openai` already exists. I did not execute the repro because this pass was read-only.

## Solution Assessment
