Preserve legacy `models.providers.openai-codex` Codex context metadata during OpenAI/Codex provider unification so upgraded Codex-runtime sessions do not fall back to the 272k default cap.

## Best Possible Solution

Implement a doctor/update migration plan that classifies each legacy OpenAI Codex model, route, and credential as migrated or retained, then applies that same plan across provider config, sessions, auth stores, and runtime eligibility without adding a steady-state runtime fallback.
