## Summary

Keep open. Current main has scoped model selection, session patching, cron model selection, Telegram session model switching, and runtime failover, but I found no owner-safe bulk retarget command or API that covers the cross-surface migration requested for provider rate limits.

## What This Changes

The issue requests one command to move all model selections across agents, subagents, cron jobs, sessions/tasks, and the current Telegram context to another configured provider/model during provider rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session command surface that validates the target provider/model, previews every affected surface, supports explicit scopes, and preserves deliberate overrides unless the operator opts into rewriting them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source and docs inspection, which shows scoped controls and runtime failover but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable, scoped owner command is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky for current session, cron, subagent, and auth semantics.
