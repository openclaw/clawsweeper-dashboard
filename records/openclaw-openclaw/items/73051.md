## Summary

Keep open. Current main has scoped model controls for defaults, fallbacks, session picks, cron payloads, subagent defaults, and channel model pickers, but it still does not provide a single owner-safe bulk retarget command/API for the requested rate-limit recovery workflow.

## What This Changes

The issue requests one command to move all configured agents, subagents, cron jobs, tasks, and the active channel session to another provider/model during rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session control surface that validates the target, previews affected objects, supports explicit scopes, and preserves intentional overrides unless opted in.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows scoped controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` would be too risky for session, cron, fallback, and channel semantics.
