## Summary

Keep open. Current main still provides scoped model controls for defaults, sessions, subagents, cron jobs, media/fallback models, and auth-profile state, but it does not provide the owner-safe bulk retarget command/API requested for provider rate-limit incidents.

## What This Changes

Request for one command to move all configured and runtime model selections to another provider/model when the current provider is rate limited.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session command surface that validates the target provider/model, previews every affected surface, supports explicit scopes, and preserves intentional overrides unless the operator opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source/docs inspection on current main, which shows scoped model controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified /moveall that silently rewrites every model reference would be too risky for current semantics.
