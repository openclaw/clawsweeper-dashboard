## Summary

Keep open. Current main still provides scoped default, session, cron, fallback, and sub-agent model controls, but it does not implement the requested owner-safe bulk retarget command/API across those surfaces.

## What This Changes

Request one command to move all model-bound OpenClaw surfaces to another configured provider/model when a provider is rate limited.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session control surface that validates the target, previews every affected object, supports explicit scopes, and preserves intentional overrides unless the operator opts in.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source and docs inspection on current main, which shows scoped model controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` that rewrites every model reference would be too risky for current session and cron semantics.
