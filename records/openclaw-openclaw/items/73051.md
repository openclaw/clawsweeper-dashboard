## Summary

Keep open. Current main still has separate model-selection surfaces for defaults, explicit session overrides, cron job primaries, subagent defaults/overrides, and fallback state; I did not find a single owner-safe bulk retarget command or API that covers the requested cross-surface move during provider rate limits.

## What This Changes

Request one command, such as `/moveall`, to switch defaults, current chat, subagents, cron jobs, and other persisted task model selections to another configured provider/model during rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing model/session command surface that validates the target provider/model, shows a dry-run preview, supports explicit scopes for defaults, active sessions, stored sessions, cron jobs, subagents, media/fallback models, and auth profile pins, and preserves deliberate overrides unless the operator opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source and docs inspection on current main, which shows scoped controls but no single bulk model retarget command or API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
