## Summary

Keep open. Current main has scoped model controls for defaults, fallbacks, sessions, cron jobs, and subagents, but source and docs still show no single owner-safe bulk retarget command/API; the prior ClawSweeper comment remains accurate.

## What This Changes

The issue requests one command to retarget all configured OpenClaw model uses to another provider/model during provider rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session control surface that validates the target, previews affected objects, supports explicit scopes, and preserves intentional overrides unless opted in.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows scoped controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` would be too risky for session, cron, and fallback semantics.
