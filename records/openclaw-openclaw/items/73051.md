## Summary

Keep open. Current `main` still has scoped model controls for defaults, sessions, Telegram picker selections, subagents, cron payloads, and fallback state, but no owner-safe bulk retarget command/API across those surfaces; the prior ClawSweeper review reached the same result and the current source still supports it.

## What This Changes

The issue requests one command to retarget all configured model references, including default agents, subagents, cron jobs, tasks, and the active Telegram context, to another provider/model during rate limiting.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session control surface that validates the target, previews every affected object, supports explicit scopes, and preserves intentional overrides unless the operator opts in.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current `main`, which shows scoped controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` that rewrites every model reference would be too risky for session, cron, and fallback semantics.
