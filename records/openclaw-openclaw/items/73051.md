## Summary

Keep open. Current main and v2026.6.10 still provide scoped model controls, session switching, fallback chains, cron model overrides, and doctor cleanup, but not a safe bulk command/API that retargets every model-bearing surface to one provider/model.

## What This Changes

The issue requests one command to move all agents, subagents, cron jobs, tasks, and the active Telegram session to another configured provider/model when a provider is rate limited.

## Best Possible Solution

Define a privileged, source-aware bulk model migration/reporting flow with dry-run mode, explicit rewrite scopes, target validation, and preservation rules for intentional scoped selections.

## Reproduction Assessment

Not applicable: this is a feature request for a new bulk retargeting command, not a current-main bug with reproduction steps. Source inspection gives high-confidence proof that only scoped model controls exist today.

## Solution Assessment

No for the literal `/moveall` sketch: it is too broad without approved scope and preservation semantics. The maintainable path is a privileged dry-run migration/reporting flow that validates the target and makes each rewrite scope explicit.
