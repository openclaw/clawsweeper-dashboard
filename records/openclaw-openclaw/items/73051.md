## Summary

Keep open. Current main and v2026.6.10 have scoped default-model, fallback, per-session, cron, and subagent model-selection controls, but no safe bulk command/API that retargets every model-bearing surface to a chosen provider/model.

## What This Changes

The issue requests one command, such as `/moveall`, to move all OpenClaw model selections to another configured provider/model during rate limits.

## Best Possible Solution

Define a source-aware bulk model migration/report command or API with dry-run mode, explicit rewrite scopes, target validation, and clear preservation rules for intentional scoped selections.

## Reproduction Assessment

Not applicable: this is a feature request for a new bulk retargeting command, not a current-main bug with reproduction steps. Source inspection gives high-confidence proof that only scoped model controls exist today.

## Solution Assessment

No as proposed: a bare `/moveall` is too broad without approved scope and preservation semantics. The maintainable path is a privileged dry-run migration/reporting flow that validates the target and makes each rewrite scope explicit.
