## Summary

Keep open. Current main and v2026.6.8 still provide scoped default, session, channel, cron, and subagent model controls, but no safe bulk command/API that can retarget every model-bearing surface; the remaining work is a maintainer product/API decision over rewrite scope and preservation rules.

## What This Changes

The issue requests one command to retarget all model-bound agents, subagents, cron jobs, tasks, and the active Telegram context to a different configured provider/model during rate limits.

## Best Possible Solution

Design a privileged, dry-run-capable bulk model migration flow with explicit scopes, target validation, and preservation rules for intentional per-agent, per-job, per-session, and channel selections.

## Reproduction Assessment

Not applicable: this is a feature request for a new bulk retargeting command, not a current-main bug with reproduction steps. Source inspection is the right check and shows only scoped model controls today.

## Solution Assessment

No as proposed: a bare `/moveall` is too broad without approved scope and preservation semantics. The maintainable path is a privileged dry-run migration/reporting command or API that validates the target and makes each rewrite scope explicit.
