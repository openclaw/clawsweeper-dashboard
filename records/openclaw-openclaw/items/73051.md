## Summary

Keep open. Current main and the latest release still have scoped `/model` and `/models` switching, default model setters, cron model selection, subagent model defaults, and session overrides, but no safe bulk command/API that retargets every model-bearing surface.

## What This Changes

The issue requests one command to move all OpenClaw model-bound surfaces to another configured provider/model when a provider is rate-limited.

## Best Possible Solution

Design an owner-only bulk model migration flow with target validation, dry-run output, explicit scopes, and default preservation of intentional per-agent, per-job, and per-session overrides.

## Reproduction Assessment

Not applicable; this is a feature request rather than a reproducible bug. The high-confidence check is source/docs inspection, which shows scoped model controls but no bulk retarget command or API.

## Solution Assessment

No as proposed; a bare `/moveall` would be too broad without approved scope and preservation semantics. The maintainable path is a dry-run-capable owner command/API that validates targets and makes rewrite scopes explicit.
