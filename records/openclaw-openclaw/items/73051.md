## Summary

Keep open. Current main still exposes model controls as separate default, per-session, per-Gateway-session, cron, and subagent surfaces; I found no single owner-safe command or API that retargets all persisted and active model selections when a provider is rate limited.

## What This Changes

Request a single chat/command flow to move defaults, the current channel session, subagents, cron jobs, and other model-bound tasks to a selected configured provider/model.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing model command surface that validates the target provider/model, shows a dry-run preview, allows explicit scopes for defaults/sessions/cron/subagents/media/fallbacks/auth profile pins, and preserves deliberate overrides unless the operator opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source and docs inspection, which shows scoped controls but no single bulk model retarget command or API on current main.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
