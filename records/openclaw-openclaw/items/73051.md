## Summary

Keep open. Current main and v2026.6.8 still provide scoped model controls and fallback behavior, but no bulk command/API that can safely retarget every model-bearing surface; the remaining work is a maintainer product/API decision, not conservative cleanup.

## What This Changes

The issue requests one command to move all OpenClaw model-bearing surfaces to another configured provider/model when the current provider is rate limited.

## Best Possible Solution

Design an owner-only bulk model migration flow with target validation, dry-run output, explicit scopes, and preservation rules for intentional per-agent, per-job, and per-session choices.

## Reproduction Assessment

Not applicable; this is a feature request rather than a reproducible current-main bug. The high-confidence check is source/docs inspection, which shows scoped model controls but no bulk retarget command or API.

## Solution Assessment

No as proposed; a bare `/moveall` is too broad without approved scope and preservation semantics. The maintainable path is a privileged dry-run-capable migration command/API that validates targets and makes rewrite scopes explicit.
