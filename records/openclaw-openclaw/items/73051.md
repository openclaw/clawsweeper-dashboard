## Summary

Keep open. Current main still exposes model selection as separate default, session, cron, subagent, media, and fallback surfaces, and I found no single owner-safe command or API that retargets all persisted and active model choices when a provider is rate limited.

## What This Changes

Add a single command, such as `/moveall`, to move the main agent, subagents, cron jobs, other tasks, and the current chat context to another configured provider/model.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing model/session/admin control surface. It should validate the target provider/model, show a dry-run preview, allow explicit scopes for defaults, sessions, cron, subagents, media, fallbacks, and auth-profile pins, and preserve deliberate overrides unless the operator opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source and docs inspection, which shows scoped controls but no single bulk model retarget command or API on current main.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
