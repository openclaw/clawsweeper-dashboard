## Summary

Keep open. Current main still has scoped model-selection mechanisms for defaults, sessions, subagents, cron, media/fallback models, and auth/profile state, but no owner-safe bulk retarget command/API that covers the cross-surface migration requested for provider rate limits.

## What This Changes

Request for one command to move all configured and runtime model selections to another provider/model when the current provider is rate limited.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session command surface that validates the target provider/model, previews every affected surface, supports explicit scopes for defaults, active/stored sessions, cron jobs, subagents, media models, fallback lists, and auth profile pins, and preserves intentional overrides unless the operator explicitly opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is current-main source and docs inspection, which shows scoped controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky for current model/session semantics.
