## Summary

Keep open. Current main still has scoped model defaults, fallbacks, session overrides, cron job model selection, and subagent defaults, but no owner-safe bulk retarget command/API that covers the cross-surface migration requested here.

## What This Changes

The issue requests one command to retarget all configured and persisted model selections to a different provider/model during provider rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session command surface that validates the target provider/model, previews every affected surface, supports explicit scopes, applies changes transactionally, and preserves deliberate overrides unless the operator opts in to rewriting them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is current-main source and docs inspection, which shows scoped controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky for current session, cron, subagent, and auth-profile semantics.
