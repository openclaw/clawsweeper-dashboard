## Summary

Keep open. Current main has control-plane `/model` and `/models` switching plus scoped defaults, cron, subagent, and session model controls, but no bulk retarget command or API; the related closed issue at https://github.com/openclaw/openclaw/issues/81130 only covers manual model switching when rate-limited, not retargeting every model-bearing surface.

## What This Changes

Request a single command to retarget defaults, subagents, cron jobs, other tasks, and the active Telegram session to another configured provider/model during provider rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow with target validation, dry-run output, explicit scopes, and preservation of intentional per-agent, per-job, and per-session overrides by default.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows scoped model controls but no single bulk retarget command or API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` would be risky for session, cron, subagent, fallback, and auth-profile semantics.
