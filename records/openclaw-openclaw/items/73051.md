## Summary

Keep open. Current main still exposes model changes through separate default, session, Gateway session, cron, subagent, media, and fallback surfaces; there is no single owner-safe command or API that migrates all persisted and active model selections when a provider is rate limited.

## What This Changes

The issue requests one command, such as `/moveall`, to move all model selections across the main agent, subagents, cron jobs, tasks, and the current Telegram context to a different configured provider/model.

## Best Possible Solution

Add an owner-only bulk model migration flow under the existing model command surface that validates the target provider/model, shows a dry-run preview, supports explicit scopes for defaults, sessions, cron, subagents, media, fallbacks, and auth/profile pins, and preserves deliberate overrides unless the operator opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source and docs inspection, and current main shows scoped model controls without a single bulk retarget command or API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
