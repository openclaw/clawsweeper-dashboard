## Summary

Keep open. Current main still has scoped model selection for defaults, sessions, cron jobs, subagents, fallback chains, and media defaults, but no owner-safe bulk retarget command or API that covers the cross-surface migration requested for provider rate limits.

## What This Changes

The issue requests one command to move all configured and stored model selections, including main agents, subagents, cron jobs, other tasks, and the active Telegram context, to another provider/model.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session command surface that validates the target provider/model, previews every affected surface, supports explicit scopes, and only rewrites deliberate overrides when the operator opts in.

## Reproduction Assessment

Not applicable as a feature request. The verification path is current-main source and docs inspection, which shows scoped controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky for current session and cron semantics.
