## Summary

Keep open. Current main has scoped default, session, cron, fallback, and subagent model controls, but it still does not provide the requested owner-safe bulk retarget command/API across those surfaces.

## What This Changes

The issue requests one command to move all agents, subagents, cron jobs, tasks, and the active Telegram context to another configured provider/model during rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing model/session control surfaces with target validation, dry-run output, explicit scopes, and override preservation unless the operator opts in.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source and docs inspection on current main, which shows scoped model controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional /moveall that rewrites every model reference would be too risky for current session, cron, and fallback semantics.
