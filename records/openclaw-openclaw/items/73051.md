## Summary

Keep open. Current main still has separate model-selection surfaces for defaults, per-session `/model` and `sessions.patch`, cron payloads, and subagent defaults, and I found no bulk command/API that retargets all persisted and active model choices when a provider is rate limited.

## What This Changes

Add one command like `/moveall` that moves the main agent, current chat context, subagents, cron jobs, and other task model selections to a different configured provider/model.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing model command surface that validates the target provider/model, shows a dry-run preview, supports explicit scopes for defaults/sessions/cron/subagents/media/fallbacks/auth profile pins, and preserves deliberate overrides unless the operator opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection, which shows scoped controls but no current-main bulk retarget command or API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
