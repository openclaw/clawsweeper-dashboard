## Summary

Keep open. Current main still has scoped model controls, but no owner-safe bulk retarget command/API that updates defaults, active or stored sessions, cron jobs, subagent defaults/overrides, and the current chat surface together during provider rate limits.

## What This Changes

Request one command, usable from chat such as Telegram, to move all model-bound OpenClaw work to another configured provider/model when a provider is rate limited.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing model/session command surface that validates the target provider/model, presents a dry-run diff, supports explicit scopes for defaults, active sessions, stored sessions, cron jobs, subagents, media/fallback models, and auth-profile pins, and preserves intentional overrides unless the operator opts into replacing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows scoped model controls but no single bulk retarget command or API.

## Solution Assessment

Unclear as proposed. The useful direction is a scoped, dry-run-capable owner command/API; an unqualified `/moveall` that silently rewrites every model reference would be too risky for OpenClaw's current model/session contracts.
