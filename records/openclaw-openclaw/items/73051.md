## Summary

Keep open. Current main still has scoped model controls for defaults, sessions, cron jobs, and subagents, but no owner-safe bulk retarget command or API; the existing ClawSweeper comment remains accurate.

## What This Changes

The request is for a `/moveall`-style command that moves the main agent, subagents, cron jobs, other tasks, and the active Telegram session to a selected configured provider/model during rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session control surface with target validation, dry-run output, explicit scopes, and override preservation by default.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows scoped model controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` would be risky for session, cron, fallback, and auth-profile semantics.
