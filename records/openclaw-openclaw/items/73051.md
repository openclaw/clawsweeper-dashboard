## Summary

Keep open. Current main still exposes scoped model/default/session/cron/subagent controls, but not an owner-safe command/API to bulk retarget all model-bearing state, so this remains product design work rather than cleanup-close material.

## What This Changes

The issue requests one command, such as `/moveall`, to move defaults, sessions, subagents, cron jobs, and the active Telegram context to another configured provider/model when rate-limited.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session control surfaces with target validation, dry-run output, explicit scopes, and override preservation by default.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows scoped model controls but no single bulk retarget command/API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` would be risky for session, cron, fallback, subagent, and auth-profile semantics.
