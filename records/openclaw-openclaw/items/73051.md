## Summary

Keep open. Current main still has scoped default, session, cron, and Telegram model controls, but no owner-safe command or API to bulk retarget all model-bearing state, so this remains product design work rather than cleanup-close material.

## What This Changes

The issue requests one command to retarget all model-bearing OpenClaw state to another configured provider/model when a provider is rate-limited.

## Best Possible Solution

Design an owner-only bulk model migration flow under the existing models/session command surfaces with target validation, dry-run output, explicit scopes, and preservation of intentional overrides by default.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows scoped model controls but no single bulk retarget command or API.

## Solution Assessment

Unclear as proposed. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unconditional `/moveall` would be risky for session, cron, fallback, subagent, and auth-profile semantics.
