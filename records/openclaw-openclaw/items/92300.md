## Label Justifications

- `P1`: The linked report shows long assistant replies expanding into dozens of repeated copies and becoming effectively unusable.
- `merge-risk: 🚨 message-delivery`: The patch removes completed assistant text blocks when its cumulative-snapshot classification matches, so false positives can suppress visible messages.

## Summary

This PR addresses a well-supported P1 duplication bug, but it is not merge-ready: the current implementation embeds an external provider's behavior in core string heuristics, and the supplied synthetic replay does not satisfy the real-provider proof gate.

## What This Changes

The PR makes both OpenAI Responses stream consumers replace earlier completed text blocks when later Bedrock Mantle GPT-5.x messages repeat and extend them, with parser, transport, and subscriber regressions.

## Best Possible Solution

Represent cumulative completed-message snapshots as a narrow typed `OpenAIResponsesCompat` capability populated by the Mantle provider, and have both core consumers use only that prepared fact while preserving all unaffected-provider and boundary regressions.

## Reproduction Assessment

Yes at source level: the linked issue provides a concrete cumulative event sequence, and the production parser replay demonstrates how current behavior duplicates it. We do not have an independently executed live Mantle after-fix reproduction.

## Solution Assessment

No, not at the current ownership boundary. The replacement algorithm and two-consumer coverage are reasonable, but enablement should use the existing typed Responses compatibility contract supplied by the provider plugin rather than core string heuristics.
