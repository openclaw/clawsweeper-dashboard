## Summary

Keep open: current main still drops Feishu block payloads when the streaming-card path cannot consume them, and when the card path is active it treats blocks as card deltas rather than normal completed-block messages.

## What This Changes

The issue asks Feishu to deliver completed block-streaming chunks progressively as normal messages instead of dropping them or relying on streaming-card playback.

## Best Possible Solution

Implement a Feishu plugin-local dispatcher fix so explicit `blockStreaming: true` block payloads deliver through normal Feishu send helpers as completed block messages, while preserving streaming-card preview behavior for non-block finals and existing duplicate/media safeguards.

## Reproduction Assessment

Yes, at source level. Configure Feishu with `blockStreaming: true` and `streaming: false`, then deliver a `{ kind: "block" }` payload: current main returns before any normal Feishu send helper runs.

## Solution Assessment

Yes. A Feishu plugin-local dispatcher repair is the narrowest maintainable fix because the config surface and docs already define completed-block delivery separately from preview/card streaming.
