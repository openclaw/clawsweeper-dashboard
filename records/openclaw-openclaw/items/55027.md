## Summary

Keep open: current main and the latest release still do not provide true Feishu completed-block delivery. The dispatcher returns block payloads unless the streaming-card path can consume them, while docs and tests keep block streaming separate from preview/card streaming; the related broad Feishu ACP PR is still open and not a landed fix.

## What This Changes

Make Feishu `blockStreaming` deliver each completed block progressively as normal channel output instead of dropping it or routing it through streaming-card playback.

## Best Possible Solution

Implement a narrow Feishu dispatcher fix so explicit `blockStreaming: true` sends completed block payloads immediately through normal Feishu message/card delivery, while keeping preview streaming and the default block-streaming-off safeguard separate.

## Reproduction Assessment

Yes, at source level: `deliver(..., { kind: "block" })` returns before normal delivery unless `streamingEnabled && useCard`, and current tests expect explicit Feishu block streaming to close a streaming card. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

Yes, a narrow Feishu dispatcher repair is the best path. The broad open Feishu ACP PR is not landed and should not be treated as the canonical fix for this channel block-delivery bug.

## Repair Work Prompt

Fix Feishu completed-block delivery for https://github.com/openclaw/openclaw/issues/55027. Current `extensions/feishu/src/reply-dispatcher.ts` only consumes `info.kind === "block"` through `FeishuStreamingSession` and returns otherwise, so explicit Feishu block streaming either drops blocks or relies on streaming-card playback instead of normal block delivery. Make explicit `blockStreaming: true` deliver completed block payloads immediately through the normal Feishu send path without depending on `FeishuStreamingSession`; use structured cards when `renderMode` calls for card output and plain text otherwise. Keep the default `disableBlockStreaming: true` behavior when Feishu `blockStreaming` is unset or false, preserve media handling and final dedupe, and avoid broad ACP/session-binding changes from https://github.com/openclaw/openclaw/pull/48834. Add focused regression tests in `extensions/feishu/src/reply-dispatcher.test.ts` for explicit block streaming with streaming-card disabled and enabled, plus the default suppression path. Add a changelog entry because the fix is user-visible.
