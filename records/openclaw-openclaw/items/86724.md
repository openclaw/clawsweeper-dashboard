## Summary

Keep open. The shared `attachments[]` hydration fix is useful and not present on current main, but this PR also reroutes an iMessage media-only send path and introduces a blocking failure-handling regression, with no after-fix real transport proof yet.

## What This Changes

The PR lifts the first structured `attachments[]` entry into single-attachment action hydration, adds reply regression tests, and reroutes iMessage media-only chat sends through `imsg send-attachment`.

## Best Possible Solution

Keep the shared `attachments[]` hydration approach, but preserve iMessage bridge failure semantics and add redacted after-fix real transport proof before merge.

## Reproduction Assessment

Yes from source inspection, though I did not execute it: current main advertises `attachments[]`, consumes it for `send`, and does not lift it before single-attachment action hydration. The PR body also includes a real pre-fix iMessage reproduction where the text arrived without the attachment.

## Solution Assessment

Partly. The shared hydration lift is the narrow maintainable fix for the schema/runtime mismatch, but the added iMessage CLI detour needs corrected failure handling or separation before it is the best merge shape.
