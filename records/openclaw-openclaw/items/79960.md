## Summary

Current main and the shipped v2026.4.26 release already cover the reported 2026.4.10 failure mode: stale Chromium profile locks are recovered, failed managed launches are bounded by a per-profile cooldown, and the browser CLI has doctor/reset-profile recovery commands.

## What This Changes

The issue asks the browser plugin to recover stale Chromium SingletonLock artifacts, avoid unbounded CDP launch retries, clean failed spawns, and expose browser doctor/reset recovery.

## Best Possible Solution

Keep the shipped browser profile-lock recovery, managed-launch circuit breaker, failed-process cleanup, and doctor/reset-profile recovery surface; affected users should upgrade to v2026.4.26 or newer and open a new report only if the same runaway behavior reproduces on a current release.
