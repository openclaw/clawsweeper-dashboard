## Summary

## What This Changes

The issue asks Gateway to persist a durable assistant error turn for started agent-run failures so refreshed chat.history and Control UI do not drop visible conversation history.

## Best Possible Solution

## Solution Assessment

Yes. The maintainable fix is to persist exactly one sanitized assistant error through the existing transcript append wrapper when the runtime did not already persist one, not to patch dist output or paper over it in the UI.
