## Summary

Keep open. Current main still has the successful overflow-compaction retry path, but the aggregate compaction retry wait still has no diagnostic heartbeat/progress callback; the focused overlapping fix remains open in https://github.com/openclaw/openclaw/pull/76657, and a newer 2026.5.18 field report adds more evidence that compaction failure paths can still leave lanes in processing.

## What This Changes

The issue reports that overflow-triggered auto-compaction can succeed and retry the prompt, yet leave the session lane stuck in processing with queueDepth=0 and block all channels until recovery or restart.

## Best Possible Solution

Land or replace the open focused runner/diagnostic fix so active compaction retry waits refresh progress and successful, failed, and lost-terminal compaction paths release or recover the processing lane without a gateway restart.
