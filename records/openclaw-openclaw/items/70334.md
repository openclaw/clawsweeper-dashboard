## Summary

Keep this open. The report is recent, concrete, and corroborated, and current main still lacks issue-specific proof that the context-overflow auto-compaction success retry lane releases after the retried Codex prompt succeeds, errors, or never emits a terminal event. Current main has useful adjacent mitigations, but diagnostics still observe active embedded runs by default and the Codex silent-turn terminal idle timeout is a partial fallback rather than a focused fix for this compaction retry path.

## What This Changes

The issue reports that a successful context-overflow auto-compaction retry can leave the session stuck in `processing` with `queueDepth=0` until the gateway restarts.

## Best Possible Solution

Add a focused regression for overflow-compaction success followed by a retried prompt that succeeds, errors, and never produces a terminal event; the fixed behavior should clear the active embedded run, return diagnostics to idle or a bounded error state, drain the session lane, and allow the next channel message without a gateway restart.

## Reproduction Assessment

Yes for an automated regression path: mock a context-overflow prompt error, successful forced compaction, and a retried Codex attempt that never emits terminal completion. A live long-session reproduction was not run, but the issue supplies concrete logs and current main still has the same retry branch.

## Solution Assessment

No current solution is proven complete. The Codex app-server timeout is useful adjacent mitigation, but the maintainable fix should explicitly terminalize the overflow-compaction retry lane and cover follow-up message delivery in tests.
