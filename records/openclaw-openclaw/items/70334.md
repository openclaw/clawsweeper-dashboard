## Summary

Keep this open: the report is recent, corroborated, and current main still lacks a focused regression proving that the post-compaction retry terminalizes and releases the active session lane in the reported failure mode.

## What This Changes

The issue reports that successful context-overflow auto-compaction followed by a prompt retry can leave the session lane permanently in `processing` with `queueDepth=0`.

## Best Possible Solution

Add a targeted lifecycle regression and fix the embedded-runner/Codex app-server cleanup boundary so every post-compaction retry outcome clears the active run and session lane.

## Reproduction Assessment

Yes for an automated path: mock a context-overflow prompt error, successful compaction, and a retried attempt that succeeds, errors, or never emits a terminal event, then assert the active run/lane clears and a follow-up message is accepted. A live long-transcript reproduction was not run.

## Solution Assessment

Unclear for current main. Existing timeouts and conservative stuck-session recovery are useful mitigations, but the narrow maintainable fix should prove the overflow-compaction retry lifecycle itself releases the lane.
