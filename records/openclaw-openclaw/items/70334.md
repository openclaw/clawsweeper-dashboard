## Summary

Keep open. Current main has useful overflow-compaction retry, active-run cleanup, command-lane timeout, and stuck-session recovery paths, but I did not find high-confidence proof that the reported successful-compaction retry path always terminalizes and releases the processing lane without fallback recovery; open PR https://github.com/openclaw/openclaw/pull/76657 still overlaps this compaction-retry diagnostic path.

## What This Changes

The issue reports that successful context-overflow auto-compaction followed by prompt retry can leave an agent session permanently in `processing` with `queueDepth=0` until gateway restart.

## Best Possible Solution

Land a focused runner/diagnostic fix, or a cleaned-up version of the overlapping PR, that proves overflow-compaction success, retried prompt error, and lost-terminal cases all release or recover the session lane without gateway restart.

## Reproduction Assessment

Yes for a source-level mocked path: simulate context overflow, successful forced compaction, and a retried attempt that succeeds, errors, or loses terminal progress, then assert the session lane clears. I did not establish a live long-transcript current-main reproduction.

## Solution Assessment
