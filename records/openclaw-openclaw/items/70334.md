## Summary

Keep this open. The report is recent, corroborated, and current main still has the same overflow-compaction success retry path without a focused regression proving that the retried attempt terminalizes and releases the active session lane in the reported failure mode.

## What This Changes

The issue reports that successful context-overflow auto-compaction followed by a prompt retry can leave the session lane permanently in `processing` with no queued work.

## Best Possible Solution

Add a targeted lifecycle regression for overflow-compaction success followed by retried prompt success, error, and lost-terminal cases, and ensure each path clears the active embedded run and session lane so follow-up channel messages are accepted.

## Reproduction Assessment

Yes for an automated path: mock a context-overflow prompt error, successful compaction, and a retried attempt that succeeds, errors, or never emits a terminal event, then assert the active run/lane clears and a follow-up message is accepted. A live long-transcript reproduction was not run during this read-only review.

## Solution Assessment

Unclear for current main. The existing Codex idle timeout and conservative stuck-session recovery are useful mitigations, but the best fix should prove the outer overflow-compaction retry lifecycle releases the lane without relying on unrelated watchdog behavior.
