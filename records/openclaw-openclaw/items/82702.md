Update the docs CLI and its reference page to use `search_open_claw`, treat MCP error payloads as failures even when the helper exits 0, and add focused command coverage.

## Solution Assessment

Yes. The narrowest maintainable fix is to change the selector, keep the docs page aligned, and fail loudly when the MCP payload reports an error instead of adding configuration or relying on helper auto-correction.
