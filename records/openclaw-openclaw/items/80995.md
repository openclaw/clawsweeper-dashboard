## Summary

Keep open: current main still has the source-level Google Chat threading gap, and the active fix path is the open linked PR with closing syntax at https://github.com/openclaw/openclaw/pull/80996.

## What This Changes

Make Google Chat `message(action=send)` replies using `[[reply_to_current]]` target the inbound `message.thread.name` when `replyToMode` allows replies.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80996 or an equivalent Google Chat adapter fix, then close this issue after the fix is on main.

## Reproduction Assessment

Yes, at source level. Current main carries the Google Chat thread name in `ReplyToId`, but the message-tool fallback uses the inbound message id as `currentMessageId`, so `replyToMode: "all"` can send outside the thread.

## Solution Assessment

Yes. The linked PR's channel-owned `buildToolContext` mapping is the narrowest maintainable fix because it preserves shared message-tool semantics while translating Google Chat's thread resource locally.
