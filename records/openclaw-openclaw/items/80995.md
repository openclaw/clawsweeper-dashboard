## Summary

Keep open. Current main still has a source-level Google Chat threading gap: the inbound thread resource is available, but Google Chat has no channel-owned message-tool context adapter, and the linked fix PR is still open.

## What This Changes

Google Chat `[[reply_to_current]]` message-tool sends should resolve to the inbound `spaces/.../threads/...` resource instead of posting as top-level space messages.

## Best Possible Solution

Review and land the focused Google Chat adapter PR, then let the merge close this issue once the fix is on main.

## Reproduction Assessment

Yes, at source level. Current main carries `message.thread.name` as `ReplyToId`, but the generic message-tool fallback uses the inbound message id as `currentMessageId`, so `replyToMode: "all"` can target the wrong Google Chat resource.

## Solution Assessment

Yes. The linked PR uses the channel-owned `buildToolContext` seam to translate Google Chat's inbound thread resource into the existing message-tool reply context without changing shared core semantics.
