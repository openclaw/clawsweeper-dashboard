## Summary

Keep open. Current main still lacks a Google Chat tool-context adapter that maps the inbound thread resource into `[[reply_to_current]]`, and https://github.com/openclaw/openclaw/pull/80996 is an open implementation PR that uses closing syntax for this issue.

## What This Changes

Google Chat `message(action=send)` replies using `[[reply_to_current]]` should use the inbound `message.thread.name` when `replyToMode` allows threaded replies.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80996 or an equivalent Google Chat plugin fix, coordinating with https://github.com/openclaw/openclaw/pull/74235 if the broader thread-routing work lands first, then close this issue with merged commit and release proof.

## Reproduction Assessment

Yes, source-level. Current main carries the Google Chat thread resource in `ReplyToId`, but the generic message-tool context uses the inbound message id as `currentMessageId`, and the issue plus linked PR provide redacted live before/after evidence.

## Solution Assessment

Yes in direction, but it is not merged yet. The narrow maintainable fix is a Google Chat-owned tool-context adapter with regression coverage, or the equivalent folded into the broader thread-routing PR.
