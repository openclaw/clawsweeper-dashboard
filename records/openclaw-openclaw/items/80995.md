## Summary

Keep this open because current main still appears affected, but the issue is already paired with a focused open fix PR that uses closing syntax and includes live redacted before/after proof.

## What This Changes

The issue asks Google Chat `message(action=send)` replies using `[[reply_to_current]]` to target the inbound `spaces/.../threads/...` resource instead of escaping to a top-level space message.

## Best Possible Solution

Review and land the focused plugin-owned fix in https://github.com/openclaw/openclaw/pull/80996, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. Current main carries the Google Chat thread in `ReplyToId`, but the generic message-tool fallback uses the inbound message resource as `currentMessageId`, which can become the implicit `replyTo` target.

## Solution Assessment

Yes. A Google Chat-specific `buildToolContext` is the narrow SDK-supported seam for translating the inbound thread resource into message-tool reply context without changing core threading semantics.
