## What This Changes

Google Chat `message(action=send)` replies using `[[reply_to_current]]` should stay in the inbound Google Chat thread when a thread resource is present.

## Best Possible Solution

Review and land the linked Google Chat adapter PR with its regression test, then close this issue once the fix reaches `main`.

## Reproduction Assessment

Yes, at source level. Current `main` carries the inbound Google Chat thread as `ReplyToId`, but the generic message-tool fallback can use the inbound message id as `currentMessageId`, which is then copied into `replyTo`.

## Solution Assessment

Yes. The linked PR uses the existing channel-owned `buildToolContext` seam to map Google Chat's thread resource into message-tool reply context without changing shared core behavior.
