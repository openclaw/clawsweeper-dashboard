## Summary

Keep open: this is a valid current-main Google Chat threading bug, and an open fixing PR with closing syntax already targets it, so the issue should remain open until that PR or a chosen replacement lands.

## What This Changes

Google Chat `message(action="send")` replies with `[[reply_to_current]]` should use the inbound `message.thread.name` thread resource instead of posting as top-level space messages.

## Best Possible Solution

Keep the issue linked to a Google Chat plugin fix that maps inbound thread resources into message-tool reply context and covers `[[reply_to_current]]` with regression tests.

## Reproduction Assessment

Yes, at source level. Current main carries the Google Chat thread resource as `ReplyToId`, but the generic message-tool context falls back to the inbound message id when Google Chat has no `buildToolContext`; the linked PR also provides redacted live before/after proof.

## Solution Assessment

Yes. The narrow maintainable fix is channel-owned mapping through the existing threading tool-context seam; if the broader open Google Chat threading PR is preferred, it should subsume this exact regression before this issue closes.
