## Summary

Keep this issue open because the reported Google Chat threading bug is source-reproducible on current `main` and the active linked fix PR explicitly closes it; the issue should close only after that PR lands.

## What This Changes

The issue reports that Google Chat `message(action=send)` replies using `[[reply_to_current]]` can post outside the inbound Google Chat thread.

## Best Possible Solution

Land the focused Google Chat adapter fix in https://github.com/openclaw/openclaw/pull/80996 if review and required gates remain clean, then close this issue after merge.

## Reproduction Assessment

Yes. Current `main` has a source-level reproduction path: Google Chat stores the thread resource in `ReplyToId`, but without a Google Chat `buildToolContext`, the message tool inherits `MessageSidFull ?? MessageSid` as the reply target.

## Solution Assessment

Yes. The linked PR's direction is the narrowest maintainable fix because it uses the channel-owned `buildToolContext` seam to translate Google Chat's thread resource without changing shared outbound reply semantics.
