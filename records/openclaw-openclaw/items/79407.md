Keep open: the report is a valid current-main bug, and https://github.com/openclaw/openclaw/pull/79416 is already the open closing fix path, so this issue should not be closed before that PR lands or is replaced.

## What This Changes

The issue asks for `agent.run` inline image validation on fresh per-agent sessions to resolve the session agent model instead of the system default model.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79416 or an equivalent narrow fix with regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: the validator passes an undefined agent id, and the resolver falls back to the system default when the fresh session entry has no persisted model fields.

## Solution Assessment

Yes. Passing `resolveAgentIdFromSessionKey(requestedSessionKeyRaw)` into the existing resolver is the narrow maintainable fix and matches the other session-model call sites in the handler.
