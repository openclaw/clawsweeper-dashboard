## What This Changes

The issue asks `agent.run` inline image validation to resolve image support using the requested session's per-agent model for fresh sessions.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/79416 or an equivalent two-line gateway fix with regression coverage, then close this issue via the merged PR.

## Reproduction Assessment

Yes, by source inspection rather than live execution. A fresh session entry without persisted model fields follows the current `agent.run` attachment path that passes `undefined` as `agentId`, so the resolver cannot see the per-agent model override.

## Solution Assessment

Yes. Passing `resolveAgentIdFromSessionKey(requestedSessionKeyRaw)` into the existing resolver is the narrowest maintainable fix and matches adjacent gateway call sites that already preserve session agent context.
