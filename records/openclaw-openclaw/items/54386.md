## Summary

Keep open: current main still has the required-path and relative-example problem, but this branch is not merge-ready because the new fallback bypasses active workspace resolution and external real-behavior proof is absent.

## What This Changes

The PR makes skill-creator's initializer accept an omitted `--path`, adds a Python default skills-directory resolver, and revises bundled skill-creator instructions away from relative or global install paths.

## Best Possible Solution

Land a narrow repair that keeps explicit `--path` backward-compatible, resolves omitted paths to the same active workspace `skills/` directory used by runtime and `openclaw skills`, removes fixed-path wording, and adds focused path-resolution coverage.

## Reproduction Assessment

Yes. Running `python3 skills/skill-creator/scripts/init_skill.py test-skill` on current main fails because `--path` is still required, and source/docs still show relative output examples.

## Solution Assessment

No. The direction is right, but the proposed resolver duplicates only part of workspace selection instead of using the active workspace model already used by OpenClaw runtime and `openclaw skills`.
