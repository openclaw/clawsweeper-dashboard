## Summary

Keep this PR open: the data-loss-prone skill-creation workflow is real and current `main` still requires `--path`, but the branch is conflicting and its standalone default resolver can still write generated skills outside the active OpenClaw workspace.

## What This Changes

The PR makes `skills/skill-creator/scripts/init_skill.py` accept an omitted `--path`, adds a default workspace skills-directory resolver, and updates bundled skill-creator instructions to discourage global or relative output paths.

## Best Possible Solution

Land a reworked fix that keeps explicit `--path` compatible, defaults omitted paths to the same active workspace `skills/` directory used by native OpenClaw skill flows, removes fixed-path documentation promises, and adds focused path-resolution coverage.

## Reproduction Assessment

Yes, by source inspection. Current `main` and `v2026.5.18` still require `--path`, while the PR's resolver covers only `OPENCLAW_WORKSPACE_DIR` and the default home workspace instead of the full active workspace contract.

## Solution Assessment

No. The intent is useful, but the best fix should reuse or faithfully match OpenClaw's active workspace resolution instead of duplicating a fixed subset in standalone Python.
