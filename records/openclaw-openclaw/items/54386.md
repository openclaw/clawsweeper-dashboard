## Best Possible Solution

Land a reworked fix that keeps explicit `--path` compatible, defaults omitted paths to the same active workspace `skills/` directory used by `openclaw skills`, removes fixed-path documentation promises, and adds focused path-resolution coverage.

## Solution Assessment

No. The intent is useful, but the best fix should reuse or faithfully match OpenClaw's active workspace resolution rather than duplicating a fixed subset in standalone Python.
