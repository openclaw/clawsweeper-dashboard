## Summary

Keep open: current main still has the unsafe required-path workflow, and this PR is useful source work, but the proposed omitted-path fallback should not merge because it can still write outside the active workspace OpenClaw will load.

## What This Changes

The PR makes `skill-creator`'s `init_skill.py` accept an omitted `--path`, default to a workspace skills directory, and updates the bundled skill-creator instructions/examples.

## Best Possible Solution

Land a narrow repair that preserves explicit `--path`, defaults omitted paths to the same active workspace `skills/` directory used by OpenClaw runtime and `openclaw skills`, removes unsafe relative examples, adds focused regression coverage, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main still requires `--path`, while the PR branch's omitted-path fallback can be compared directly against the active workspace resolver and shown to miss configured/profile/cwd-inferred workspaces.

## Solution Assessment

No. The direction is right, but the proposed fix reimplements a partial workspace resolver instead of using or being passed OpenClaw's active workspace contract.
