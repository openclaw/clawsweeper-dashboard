## Summary

Keep open: the reported Claude CLI session-continuity bug is real, but this PR is not merge-ready because it changes the auth boundary, conflicts with current main, and reopens a partial-read reset path that current main protects.

## What This Changes

The PR changes Claude CLI auth epoch hashing to use a refresh-token-derived local credential fingerprint, skip selected auth-profile credentials when that identity is present, and adds regression tests for profile rotation and token stability.

## Best Possible Solution

Settle the Claude CLI host-identity contract, then rebase a narrow fix that preserves current epoch versioning, scoped profile stores, and either the partial-read invariant or an explicitly accepted fail-closed tradeoff.
