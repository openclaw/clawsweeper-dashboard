## Summary

Keep open: the reported Claude CLI reset path is real, but the current PR should not merge as-is because it exposes a new public auth-epoch mode and can under-detect real Claude CLI OAuth account changes.

## What This Changes

The branch adds a `host-only` CLI backend auth epoch mode, uses it to skip auth-profile credentials for Claude CLI session reuse, documents the mode, and adds focused tests.

## Best Possible Solution

Implement a narrow Claude CLI fix that preserves resume only when unchanged host identity is proven, fails closed when identity is unproven, and keeps any public SDK mode out until the contract is approved.

## Reproduction Assessment

Yes. Source inspection and the supplied QA/log evidence show current main mixes the selected auth profile into the CLI epoch and can reset Claude CLI sessions on profile rotation; I did not run a live gateway E2E repro in this read-only review.

## Solution Assessment

No. A backend-owned fix is the right direction, but this implementation makes it a public SDK mode and does not prove Claude CLI OAuth host identity strongly enough to avoid cross-account reuse.

- **[P1] Keep Claude CLI host identity fail-closed:** `extensions/anthropic/cli-backend.ts:36`
  - body: Marking claude-cli `host-only` drops the profile credential from the session epoch, but current Claude CLI OAuth credentials carry only access/refresh/expires and `encodeClaudeCredential` ignores those identity-light fields.
