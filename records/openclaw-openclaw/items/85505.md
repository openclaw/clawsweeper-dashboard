## Summary

Keep this PR open, but it is not merge-ready: the reported Claude CLI reset is real and the branch targets the right failure mode, yet the current patch reintroduces partial-read auth-epoch flapping and needs maintainer approval for making the host refresh token the Claude CLI session identity.

## What This Changes

## Best Possible Solution

No. The branch fixes the cosmetic auth-profile rotation path, but it knowingly changes the current partial-read invariant so transient token-shaped Claude CLI reads can produce a different epoch and reset sessions.
