## Summary

Keep open. Current main still has the reported Claude CLI auth-profile reset path, but this PR remains blocked because it fixes the cosmetic rotation by changing the Claude CLI auth identity boundary and intentionally relaxing the existing partial-read stability invariant.

## What This Changes

The PR changes Claude CLI auth epochs to hash OAuth refresh tokens, skip selected auth-profile credentials when that local credential is identity-bearing, and updates auth-epoch tests for profile rotation, refresh changes, and partial reads.

## Best Possible Solution

Keep the cosmetic profile-rotation fix only after maintainers explicitly accept the Claude CLI identity boundary and partial-read tradeoff, or revise the implementation to preserve current partial-read stability while avoiding cosmetic profile-rotation resets.

## Reproduction Assessment

Yes. Current main source includes selected profile credentials in the epoch, profile overrides can rotate on compaction/cooldown, and session reuse resets on auth-profile when epochs differ; the PR body adds production logs and patched harness output.

## Solution Assessment

Unclear. The new shape is a plausible narrow fix and removes the public SDK mode concern, but it intentionally changes the partial-read invariant and auth identity source, so it needs maintainer product/security acceptance before it is the best fix.
