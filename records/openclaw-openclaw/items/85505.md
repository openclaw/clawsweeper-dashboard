## Summary

Keep open: the reported Claude CLI auth-profile rotation problem is valid, but this patch reopens the older partial-read epoch-flapping class and changes the Claude CLI auth identity boundary without maintainer approval.

## What This Changes

The PR changes Claude CLI auth epoch fingerprints to include a host refresh-token hash and to ignore selected auth-profile credentials when that host credential has a refresh token.

## Best Possible Solution

Keep the cosmetic auth-profile-rotation fix, but preserve the current partial-read stability invariant or land an explicitly approved Claude CLI identity strategy with upgrade proof.

No. The branch fixes the cosmetic auth-profile rotation path, but it knowingly changes the current partial-read invariant so transient token-shaped Claude CLI reads can produce a different epoch and reset sessions.
