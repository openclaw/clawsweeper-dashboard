## Risks / Open Questions

- `host-only` makes claude-cli session reuse ignore the selected OpenClaw auth profile; this is appropriate only while the backend truly never injects profile credentials into the spawned CLI.
- Existing claude-cli bindings with a combined-era epoch are expected to reset once after upgrade before being rewritten with the host-only epoch shape.
- The supplied proof exercises production logs plus patched epoch/session-reuse logic, but not a full restarted gateway end-to-end run.
- For Claude CLI OAuth, the existing local credential fingerprint is identity-light, so maintainers should explicitly accept that the host login is the auth owner even when OpenClaw cannot distinguish every host re-login from token rotation.
