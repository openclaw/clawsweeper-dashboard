## Best Possible Solution

Land the focused host-only claude-cli fix with docs and tests after auth/session owners accept the mode semantics and upgrade behavior, while keeping raw transcript reseed blocked across real auth changes.

## Risks / Open Questions

- `host-only` is safe only while claude-cli truly never consumes the selected OpenClaw auth profile; if a future credential bridge is added, this mode could under-detect profile credential changes.
- Existing claude-cli bindings with combined-era epochs are expected to reset once after upgrade before stabilizing under the host-only epoch shape.
- The proof covers production logs plus a patched auth epoch/session-reuse harness, but not a fully restarted gateway end-to-end run.
- For Claude CLI OAuth, the local credential fingerprint is identity-light, so maintainers should explicitly accept host login ownership for this backend.
