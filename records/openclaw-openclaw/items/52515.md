## Summary

Keep open: this is a useful linked bug-fix PR, but it still has blocking correctness findings, missing real behavior proof, and a security-sensitive SecretRef boundary that needs human review.

## What This Changes

The branch changes doctor cleanup hints to render from detected extra services, wraps doctor execution in SecretRef leniency, and updates generated plugin SDK API baselines.

## Best Possible Solution

Revise the fix so doctor and status render cleanup hints from actual detected service metadata, honor user versus system scope, preserve legacy fallback guidance, and handle doctor SecretRef warnings through explicit read-only inspection while strict helpers stay fail-closed.

## Reproduction Assessment

Yes, source-level. Current main can report extra gateway-like services and both doctor/status still render cleanup hints without passing detected service context; fixtures cover custom LaunchAgents that execute openclaw gateway.

## Solution Assessment

No. The direction is valid, but this patch misses the status caller, loses legacy cleanup guidance, ignores system service scope, weakens strict SecretRef behavior, and targets the old inline doctor flow.

  - body: Current main's doctor command is a lazy wrapper around `src/flows/doctor-health.ts`, but this patch wraps the old inline doctor body in a process env flag. The SecretRef tolerance needs to be implemented in the current flow or an explicit read-only inspection path.

  - confidence: 0.93

  - body: Filtering out every legacy service before rendering hints removes the fallback commands when legacy cleanup is skipped by policy, declined, system-scoped, or fails. Doctor can still report the service but no longer show an actionable manual cleanup path.
  - confidence: 0.88

  - body: The helper receives each service's `scope` but always emits user launchd/systemd commands. For `/Library/LaunchDaemons` or system systemd findings from deep scans, those commands target the wrong manager and leave the detected service installed.
  - confidence: 0.9
- **[P2] Keep strict SecretRef helpers fail-closed:** `src/config/types.secrets.ts:143-149`
  - body: The patch lets an exported strict SecretRef helper return successfully for unresolved `exec` refs whenever process-wide doctor leniency is set. Doctor tolerance should use an explicit inspect/read-only path so runtime and plugin callers still fail closed.
  - confidence: 0.82
