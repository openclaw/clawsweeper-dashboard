## Summary

Keep open. The PR targets a real doctor false-positive class, but the branch is conflicting/stale against the current doctor flow, has blocking cleanup-hint regressions, weakens exported SecretRef strictness, and lacks after-fix real behavior proof.

## What This Changes

The PR changes doctor cleanup hints to render from detected extra services, wraps doctor in a process env flag that downgrades unresolved exec SecretRefs, and updates generated plugin SDK API baselines.

## Best Possible Solution

Revise or replace this with a rebased patch that renders doctor and status cleanup hints from actual detected service metadata, honors user versus system scope, preserves manual guidance for unremoved legacy services, and uses explicit read-only SecretRef inspection while keeping exported strict helpers strict.

## Reproduction Assessment

Yes, from source inspection. Current main can still report custom gateway-like services and then render canonical active-profile cleanup hints, while the PR's helper would also suppress status hints when called without detected services.

## Solution Assessment

No. The direction is valid, but this patch is not the best current solution because it misses the status caller, drops legacy fallback guidance, ignores system service scope, and changes exported strict SecretRef behavior through process-wide state.
