## Summary

## Best Possible Solution

Revise or replace the PR on current main so doctor and status both render cleanup guidance from actual detected service metadata, preserve actionable legacy fallback guidance, honor user/system scope, and keep SecretRef read-only degradation in explicit command-local paths instead of exported strict helpers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep status cleanup hints wired to detected services:** `src/daemon/inspect.ts:28-30`
  - body: `renderGatewayServiceCleanupHints()` now returns `[]` when called without services, but `status.print.ts` still calls it with no arguments after reporting `extraServices`. That makes `openclaw status --deep` report extra gateway-like services without any cleanup hints, regressing the existing troubleshooting path.
  - confidence: 0.93
- **[P2] Preserve manual hints for unremoved legacy services:** `src/commands/doctor-gateway-services.ts:441-442`
  - body: Filtering every `svc.legacy` out before rendering hints drops the fallback cleanup guidance when legacy cleanup is skipped by policy, declined, fails, or is system-scoped. In those paths doctor still reports the legacy service but no longer gives the user an actionable removal command.
  - confidence: 0.88
- **[P2] Honor service scope in cleanup commands:** `src/daemon/inspect.ts:38-46`
  - body: The helper receives `svc.scope` but always emits `launchctl bootout gui/$UID/...` and `systemctl --user ...`. For `doctor --deep` findings from LaunchDaemons or system systemd directories, these commands target the wrong manager and leave the detected system service installed.
  - confidence: 0.9
- **[P1] Keep doctor leniency out of strict SecretRef helpers:** `src/config/types.secrets.ts:143-151`
  - body: This makes the exported strict SecretRef helper read a process-wide doctor env flag and return `undefined` after `console.warn` for unresolved exec refs. Read-only doctor degradation should stay in explicit command-local inspection paths; otherwise doctor-invoked plugin/security/config code can silently treat a configured exec credential as absent instead of preserving the configured-but-unavailable diagnostic.
  - confidence: 0.86
