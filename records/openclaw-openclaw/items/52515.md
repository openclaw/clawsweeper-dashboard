## Summary

Keep open. The PR targets a real doctor/status false-positive bug, but the current branch is conflicting, lacks after-fix real behavior proof, and still has blocking correctness and security-sensitive SecretRef issues.

## What This Changes

The branch changes doctor cleanup hints to use detected extra services, wraps doctor execution in an env flag for lenient exec SecretRef handling, and updates generated plugin SDK API baselines.

## Best Possible Solution

Revise or replace the PR so doctor and status render cleanup hints from actual detected service metadata, honor user vs system scope, preserve manual legacy cleanup guidance, and handle doctor SecretRef warnings through an explicit read-only inspection path.

## Reproduction Assessment

Yes, source-level. Current main can report extra gateway-like services, then doctor and status render cleanup hints without passing the detected service metadata into the helper.

## Solution Assessment

No. The direction is useful, but this patch misses the status caller, drops legacy fallback guidance, ignores system scope, targets the old inline doctor flow, and weakens strict SecretRef behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Rebase doctor leniency onto the current health flow:** `src/commands/doctor.ts:79-81`
  - body: Current main's doctor command is a lazy wrapper around `src/flows/doctor-health.ts`, but this patch wraps the old inline doctor body in a process env flag. The SecretRef tolerance needs to live in the current health flow or an explicit read-only inspection path.
  - confidence: 0.9
- **[P2] Keep status cleanup hints wired to detected services:** `src/daemon/inspect.ts:28-30`
  - body: `renderGatewayServiceCleanupHints()` now returns `[]` when called without a service list, but `status.print.ts` still calls it with no arguments after reporting `extraServices`. That makes `openclaw status --deep` report extra gateway-like services without any cleanup hints.
  - confidence: 0.93
- **[P2] Preserve manual hints for unremoved legacy services:** `src/commands/doctor-gateway-services.ts:441-442`
  - body: Filtering out every legacy service before rendering hints drops the fallback cleanup guidance when legacy cleanup is skipped by policy, declined, fails, or is system-scoped. Doctor can still report the service but no longer gives an actionable manual path.
  - confidence: 0.88
- **[P2] Honor service scope in cleanup commands:** `src/daemon/inspect.ts:38-46`
  - body: The helper receives each service's `scope` but always emits user launchd/systemd commands. For system LaunchDaemons or systemd units found by deep scans, those commands target the wrong manager and leave the detected service installed.
  - confidence: 0.9
- **[P2] Keep strict SecretRef helpers fail-closed:** `src/config/types.secrets.ts:143-151`
  - body: The patch lets an exported strict SecretRef helper return successfully for unresolved `exec` refs whenever process-wide doctor leniency is set. Doctor tolerance should use an explicit inspect/read-only path so runtime and plugin callers still fail closed.
  - confidence: 0.82
