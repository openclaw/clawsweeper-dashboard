- **[P1] Keep strict SecretRef reads out of a global env switch:** `src/config/types.secrets.ts:143-145`
  - body: `normalizeResolvedSecretInputString()` is an exported strict helper used by runtime and plugin callers, but this change lets any process with `OPENCLAW_ALLOW_EXEC_SECRETREF_FAILURES=true` turn unresolved exec SecretRefs into warnings and `undefined`. Current main already has explicit inspect-mode SecretRef APIs for read-only doctor/status paths, so the doctor fix should use those seams instead of weakening strict reads globally.
  - confidence: 0.86
- **[P2] Keep status cleanup hints wired to detected services:** `src/daemon/inspect.ts:28-30`
  - body: The helper now returns `[]` when no service list is supplied, but the unchanged status printer still calls `renderGatewayServiceCleanupHints()` without arguments after reporting `extraServices`. That makes `openclaw status --deep` report duplicate gateway-like services without any cleanup hint, regressing the existing troubleshooting path.
  - confidence: 0.93
- **[P2] Preserve manual hints for skipped legacy services:** `src/commands/doctor-gateway-services.ts:441-442`
  - body: Filtering every `svc.legacy` before rendering hints drops the fallback cleanup guidance when legacy removal is skipped by policy, declined, fails, or is system-scoped. In those paths doctor still reports the service but no longer gives the user an actionable command.
  - confidence: 0.88
- **[P2] Honor service scope in cleanup commands:** `src/daemon/inspect.ts:38-46`
  - body: The helper receives `svc.scope` but always emits user-domain `launchctl` and `systemctl --user` commands. For `doctor --deep` findings from `/Library/LaunchDaemons` or system systemd directories, the recipe targets the wrong service manager and leaves the detected duplicate service installed.
  - confidence: 0.9
