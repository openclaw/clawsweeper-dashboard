Full review comments:

- **[P1] Keep SecretRef leniency out of strict helpers:** `src/config/types.secrets.ts:143-147`
  - body: `assertSecretInputResolved()` now checks a process-wide doctor flag and returns for unresolved exec refs. Any strict secret read that happens during doctor can be silently downgraded, and current main already has an explicit `resolveSecretInputString(..., mode: "inspect")` path for non-materializing reads, so keep runtime/plugin strict helpers strict and move doctor leniency to the specific read-only call sites.
  - confidence: 0.89
- **[P2] Keep status cleanup hints wired to detected services:** `src/daemon/inspect.ts:28-30`
  - body: The helper now returns `[]` when called without a service list, but current `status.print.ts` still calls `renderGatewayServiceCleanupHints()` with no arguments after listing `extraServices`. That makes `openclaw status --deep` report extra gateway-like services without any cleanup hint.
  - confidence: 0.93
- **[P2] Preserve hints for skipped legacy services:** `src/commands/doctor-gateway-services.ts:441-442`
  - body: Filtering every legacy service out before rendering manual hints drops the only actionable cleanup guidance when legacy cleanup is skipped by external repair policy, declined, fails, or is system-scoped. Doctor still reports the legacy service, but the user no longer gets a command to remove it.
  - confidence: 0.88
- **[P2] Honor service scope in cleanup commands:** `src/daemon/inspect.ts:38-46`
  - body: The new renderer receives `svc.scope` but always emits user-domain `launchctl` and `systemctl --user` commands. For `doctor --deep` findings from `/Library/LaunchDaemons` or system systemd directories, the printed recipe targets the wrong service manager and leaves the detected system service installed.
  - confidence: 0.9
