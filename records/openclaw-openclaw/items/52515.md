## Summary

Keep open: current main still has the cleanup-hint bug and this PR is paired with the open reporter issue, but the branch is not mergeable as-is because it misses status output, legacy fallback, service scope, current doctor-health flow, real behavior proof, and SecretRef security boundaries.

## What This Changes

The PR changes doctor gateway cleanup hints to render from detected extra services, wraps doctor execution in process-wide exec SecretRef leniency, and updates generated plugin SDK API baselines.

## Best Possible Solution

Revise or replace the PR so doctor and status render cleanup hints from actual detected service metadata, preserve manual guidance for unremoved legacy services, emit scope-correct service-manager commands, and handle doctor SecretRef checks through an explicit read-only inspection path while strict helpers remain strict.

## Reproduction Assessment

Yes, source-level. Current tests show custom macOS LaunchAgents executing openclaw gateway can be reported as extra services, and current doctor/status callers render cleanup hints without passing that service context; I did not execute tests because this review is read-only.

## Solution Assessment

No. The direction is useful, but this branch misses status, legacy fallback, service scope, current doctor-health flow, and strict SecretRef boundaries; a rebased narrower patch is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Rebase doctor leniency onto the health flow:** `src/commands/doctor.ts:79-81`
  - body: Current main's doctor command delegates to `src/flows/doctor-health.ts`, but this patch wraps the old inline doctor body in a process env flag. The exec SecretRef warning path needs to live in the current health flow or an explicit read-only inspection helper, otherwise the rebase will not protect the active doctor path cleanly.
  - confidence: 0.9
- **[P2] Pass detected services into status cleanup hints:** `src/daemon/inspect.ts:28-30`
  - body: `renderGatewayServiceCleanupHints()` now returns `[]` unless callers pass a service list, but status output still calls it with no arguments after reporting `extraServices`. That makes status report extra gateway-like services without cleanup hints, regressing that troubleshooting path.
  - confidence: 0.93
- **[P2] Preserve manual hints for skipped legacy services:** `src/commands/doctor-gateway-services.ts:441-442`
  - body: Filtering out every legacy service before rendering hints removes the fallback cleanup commands when legacy cleanup is skipped by policy, declined, fails, or is system-scoped. Doctor would still report the legacy service but no longer tell the user how to clean it up manually.
  - confidence: 0.88
- **[P2] Honor service scope in cleanup commands:** `src/daemon/inspect.ts:38-46`
  - body: The helper receives each service's `scope`, but the new commands always target user launchd/systemd managers. For `/Library/LaunchDaemons` or system systemd units found by deep scans, those commands leave the detected system service installed.
  - confidence: 0.9
- **[P2] Keep strict SecretRef helpers fail-closed:** `src/config/types.secrets.ts:143-151`
  - body: The patch lets an exported strict SecretRef helper return successfully for unresolved `exec` refs whenever process-wide doctor leniency is set. Doctor tolerance should use an explicit inspection path so runtime, plugin, and SDK callers do not silently downgrade failures.
  - confidence: 0.82
