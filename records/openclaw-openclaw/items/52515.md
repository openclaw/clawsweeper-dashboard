## Summary

Keep open. The PR targets real doctor false positives, but the branch is dirty against current main and still introduces cleanup-hint regressions plus a security-sensitive exported SecretRef behavior change that needs maintainer/secops revision before merge.

## What This Changes

The PR changes doctor gateway cleanup hints to render from detected extra services, wraps doctor execution in a process-wide env flag to downgrade unresolved exec SecretRefs, and updates generated plugin SDK API baselines.

## Best Possible Solution

Revise and rebase the fix onto current main so doctor and status render cleanup hints from actual detected service metadata, commands honor user versus system scope, unremoved legacy services keep manual guidance, and doctor SecretRef tolerance uses an explicit read-only inspection path while exported strict helpers stay strict.

## Reproduction Assessment

Yes. The PR regressions are source-reproducible: current main still has doctor/status no-argument cleanup-hint callers, while the PR helper returns no hints without a service list; the original cleanup-hint false positive also remains visible in main's active-profile helper.

## Solution Assessment

No. The requested direction is valid, but this implementation is not the narrowest maintainable fix because it is stale against the current doctor flow, misses the status path, drops legacy fallback guidance, ignores service scope, and weakens exported SecretRef strictness through process-wide state.
