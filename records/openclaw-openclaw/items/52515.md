## Summary

Keep this PR open because it addresses a real doctor false-positive problem, but the branch is stale and the current patch introduces concrete regressions in status cleanup guidance, legacy fallback guidance, service-scope handling, and exported SecretRef strictness.

## What This Changes

The PR changes doctor cleanup hints to render from detected extra services, wraps doctor execution in an exec SecretRef leniency env flag, and updates plugin SDK API baseline output.

## Best Possible Solution

Rebase or replace the PR with a current-main fix that renders doctor and status cleanup hints from actual detected service metadata, preserves manual guidance for unremoved legacy services, emits scope-correct commands, and keeps exported strict SecretRef helpers strict while doctor uses explicit inspect/read-only resolution.

## Reproduction Assessment

Yes, from source inspection. Current main still has active-profile cleanup hint rendering and no-argument doctor/status callers, while the PR helper returns no hints without a service list and weakens strict SecretRef behavior via a process env flag.

## Solution Assessment

No. The direction is valid, but this patch is stale against the current doctor flow and is not the narrowest maintainable fix because it misses status, drops legacy fallback guidance, ignores service scope, and changes exported strict SecretRef semantics.
