  - body: `renderGatewayServiceCleanupHints()` now returns `[]` when called without a service list, but `status.print.ts` still calls it with no arguments after reporting `extraServices`.   - confidence: 0.94

  - body: Filtering out every `svc.legacy` before rendering hints drops the only manual cleanup guidance when legacy cleanup is skipped by policy, declined, fails, or is system-scoped. In those paths doctor still reports the legacy service but no longer gives the user a command to remove it.
  - confidence: 0.9
- **[P2] Honor service scope in cleanup commands:** `src/daemon/inspect.ts:38-46`
  - body: The new helper receives `svc.scope` but always emits `launchctl bootout gui/$UID/...` and `systemctl --user ...`. For `doctor --deep` findings from `/Library/LaunchDaemons` or system systemd directories, those commands target the wrong service manager and leave the detected system service installed.
  - confidence: 0.91
- **[P1] Keep exported SecretRef strict reads strict:** `src/config/types.secrets.ts:143-151`
  - body: The patch adds a doctor-specific option and process-env gate to the exported strict SecretRef helper, so any strict caller in that process can receive `undefined` for an unresolved exec SecretRef instead of a hard failure. Current main already has an explicit inspect mode for read-only checks; doctor should use that path instead of changing the public strict helper contract.
  - confidence: 0.86
