  - body: This helper is exported through `openclaw/plugin-sdk/secret-input`, but the patch makes strict exec SecretRef reads return after a process-wide doctor flag is set.   - confidence: 0.87
- **[P2] Keep status cleanup hints wired to detected services:** `src/daemon/inspect.ts:28-30`
  - body: `renderGatewayServiceCleanupHints()` now returns `[]` unless a service list is supplied, but the status printer still calls it with no arguments after listing `extraServices`. That regresses `openclaw status`/`openclaw gateway status` by reporting extra gateway-like services without cleanup hints.
  - confidence: 0.93
- **[P2] Preserve manual hints for unremoved legacy services:** `src/commands/doctor-gateway-services.ts:439-442`
  - body: Filtering every legacy service out before rendering hints drops manual cleanup guidance when legacy cleanup is skipped by policy, declined, fails, or is system-scoped. In those paths doctor still reports the legacy service but no longer gives the user a command to remove it.
  - confidence: 0.88
- **[P2] Honor service scope in cleanup commands:** `src/daemon/inspect.ts:38-46`
  - body: The new helper receives `svc.scope` but always emits user-domain launchctl and systemd commands. For `doctor --deep` findings from `/Library/LaunchDaemons` or system systemd directories, those commands target the wrong service manager and leave the detected system service installed.
  - confidence: 0.9
