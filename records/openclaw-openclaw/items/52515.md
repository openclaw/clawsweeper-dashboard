## Summary

Keep open. The PR targets a real doctor/gateway false-positive problem, but it is not mergeable as-is: the branch is dirty against current main, the cleanup-hint change regresses status and legacy/system-scope guidance, and the SecretRef change weakens an exported strict helper that current main now handles more narrowly through read-only inspection paths.

## What This Changes

The PR changes doctor cleanup hints to render from detected extra services, adds process-wide doctor leniency for unresolved exec SecretRefs, and updates generated plugin SDK API baselines.

## Best Possible Solution

Revise the fix so doctor and status render cleanup hints from actual detected service metadata, preserve manual guidance for unremoved legacy services, honor user versus system service scope, keep strict SDK helpers strict, and add the required changelog entry.

## Reproduction Assessment

Yes. The source path is high-confidence: current main still calls the cleanup helper without detected services, while the PR helper returns no hints without a service list; strict SecretRef behavior is also covered by current tests and SDK exports.

## Solution Assessment

No. The requested direction is valid, but this implementation is stale against current main and broadens SecretRef behavior through process-wide state instead of using the existing explicit read-only inspection paths.
