- **[P1] Keep exec SecretRef downgrades out of global strict reads:** `src/config/types.secrets.ts:143-147`
  - body: This adds an env-controlled doctor mode inside the strict SecretRef helper, and the generated SDK baseline exposes the new parameter. Any process with `OPENCLAW_ALLOW_EXEC_SECRETREF_FAILURES=true` can turn unresolved exec SecretRefs into warnings outside the intended doctor path; keep the downgrade at explicit doctor/allow-exec call sites instead of the shared strict helper.
  - confidence: 0.82
- **[P2] Pass detected services through status cleanup hints:** `src/daemon/inspect.ts:28-31`
  - body: The renderer now returns `[]` when no service list is supplied, but `status.print.ts` still calls it with no arguments after printing `extraServices`. That makes `openclaw status` report extra gateway-like services without any cleanup hint, regressing the existing troubleshooting flow.
  - confidence: 0.9
- **[P2] Keep fallback hints when legacy cleanup is skipped:** `src/commands/doctor-gateway-services.ts:438-440`
  - body: Filtering out every legacy service before rendering hints drops the only manual cleanup commands when service repair is externally managed, confirmation is declined, or a system-scope legacy service cannot be removed by the automatic user-service cleanup path. Include unremoved legacy services in the hint list.
  - confidence: 0.84
- **[P2] Use scope-specific cleanup commands for detected services:** `src/daemon/inspect.ts:37-45`
  - body: Deep scans can return `scope: "system"`, but the new hint builder always emits `gui/$UID` launchctl and `systemctl --user` commands. For system LaunchDaemons or systemd units, doctor would print an exact-looking cleanup recipe that targets the wrong manager and leaves the detected service installed.
  - confidence: 0.86
