Full review comments:

- **[P1] Accept helper commands instead of path-validating them:** `src/agents/cli-credentials.ts:459-465`
  - body: Claude Code documents `apiKeyHelper` as a shell command, not only an executable path. This check rejects valid absolute or home-relative command strings with arguments or wrappers, so users can still hit `missing-provider-auth` even though Claude CLI would run the helper.
  - confidence: 0.91
- **[P1] Honor helper auth before stored Claude credentials:** `src/agents/cli-credentials.ts:481-501`
  - body: The new helper check runs only after Keychain and `.credentials.json`, but Claude Code's documented precedence puts `apiKeyHelper` before subscription OAuth credentials. In mixed setups with a stale or dummy stored credential, OpenClaw can choose a different auth source than Claude CLI.
  - confidence: 0.88
- **[P2] Keep helper-only doctor output profile-aware:** `src/commands/doctor-claude-cli.ts:269-274`
  - body: The patch intentionally does not persist a profile for `api_key_helper`, but doctor still adds the missing OpenClaw profile fix when no stored profile exists. A helper-only setup can therefore pass the auth gate while doctor still tells the user to repair a profile that should not exist.
  - confidence: 0.83
- **[P3] Replace the realistic fake Anthropic key literal:** `scripts/proof-claude-cli-api-key-helper-auth.mjs:41`
