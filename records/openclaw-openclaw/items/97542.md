Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not validate helper commands as paths:** `src/agents/cli-credentials.ts:459-465`
  - body: Claude Code documents `apiKeyHelper` as a custom command run through the system shell, but this branch treats any value starting with `/` or `~` as a single filesystem path. Valid helpers such as `/usr/bin/env helper --arg` or `~/bin/helper --flag` will fail `existsSync` and fall back to `missing-provider-auth`; accept a non-empty command without executing it instead.
  - confidence: 0.91
- **[P2] Honor helper auth before stored Claude credentials:** `src/agents/cli-credentials.ts:500-501`
  - body: Claude Code chooses `apiKeyHelper` before subscription OAuth credentials, but this reader still returns Keychain or `.credentials.json` credentials first. Users with stale credentials or the documented dummy-workaround file can keep seeing OpenClaw auth labels, epochs, and runtime decisions that diverge from what Claude CLI will actually use.
  - confidence: 0.88
- **[P2] Make helper-only doctor state profile-aware:** `extensions/anthropic/cli-migration.ts:205-206`
  - body: This branch stores no OpenClaw profile for `api_key_helper`, but doctor still treats a missing `anthropic:claude-cli` profile as a problem and recommends rerunning CLI setup. Either make helper-managed auth explicitly profile-less in doctor output or store a non-secret profile marker, with helper-only coverage.
  - confidence: 0.84
- **[P3] Replace the realistic fake Anthropic key literal:** `scripts/proof-claude-cli-api-key-helper-auth.mjs:41`
  - body: The proof script commits a value shaped like an Anthropic secret. Even if it is fake, it can create secret-scanning noise and normalizes realistic key literals in the repository; use an unmistakably non-secret placeholder instead.
