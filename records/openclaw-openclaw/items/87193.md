## Summary

Keeping this open: current main still has no plugin-sdk bash session subpath or plugin-facing read/kill primitive, and the request raises plugin API and security-boundary choices that need maintainer direction before implementation.

## What This Changes

The issue requests a new `openclaw/plugin-sdk/bash-sessions` API exposing read-only bash session snapshots and a narrow kill helper for external plugins.

## Best Possible Solution

Define a maintainer-approved plugin SDK contract for read-only bash session observation and, if accepted, a narrowly authorized kill helper that preserves the existing supervisor cancellation ordering without exposing stdin or mutable registry internals.

## Solution Assessment
