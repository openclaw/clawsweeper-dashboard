- `impact:session-state`: The requested API is about observing and controlling in-memory background shell session state from plugins.

## Summary

Keep open: current main still keeps bash session listing and kill behavior inside the agent process tool, and no public SDK subpath exposes the requested safe session snapshot or kill helper. The request fits the plugin API gap described by the project vision, but the public process-kill/session-inspection contract needs maintainer API and security review before implementation.

## What This Changes

Expose read-only bash-session snapshots and a kill helper through a new plugin-sdk subpath so external plugins can observe and recover background shell sessions.

## Best Possible Solution

Design a narrow runtime SDK subpath that returns immutable session DTOs, preserves the supervisor-first kill semantics, and documents/tests the security and compatibility contract before exposing it publicly.

## Reproduction Assessment

Not applicable as a feature/API gap. Source inspection confirms current main has no public bash-sessions SDK subpath and the registry/process kill path remains internal.

## Solution Assessment

Unclear until maintainers decide the public API boundary. The proposed immutable DTO plus supervisor-first kill shape is directionally narrow, but authorization and scope semantics need design review before implementation.
