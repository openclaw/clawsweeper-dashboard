## Summary

Keep open: current main and v2026.6.8 still do not expose a `openclaw/plugin-sdk/bash-sessions` subpath or plugin-facing kill helper, and no searched related issue or PR owns this exact API gap. The request fits OpenClaw's plugin-extension direction, but it needs maintainer API and security direction before implementation.

## What This Changes

Expose read-only bash session snapshots and a constrained `killBashSession(id)` helper through a new plugin SDK subpath.

## Best Possible Solution

Define a narrow, documented runtime SDK subpath for immutable session snapshots and any kill primitive only after maintainer security/API approval.
