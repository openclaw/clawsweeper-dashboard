## Summary

Keep open: current main and v2026.6.10 still lack the requested bash-session SDK surface, and the remaining decision is the API/security boundary for exposing process visibility and termination to plugins.

## What This Changes

Request to add a plugin SDK subpath for read-only bash session snapshots plus a supervisor-owned `killBashSession(id)` helper.

## Best Possible Solution

Maintainers should choose the public operator boundary, then add a narrow documented SDK or Gateway runtime surface with immutable session DTOs and supervisor-owned termination semantics if approved.

## Reproduction Assessment

Not applicable; this is a feature/API surface request rather than a broken existing behavior report. Source inspection confirms current main and v2026.6.10 do not expose the requested SDK surface.

## Solution Assessment

Unclear until maintainers choose the public API boundary. The immutable DTO plus supervisor-first kill shape is plausible, but authorization and whether external operators should use Gateway instead are product/security decisions.
