## What This Changes

The PR captures a pre-overwrite gateway service PID, threads it through stale-process cleanup as an optional protected PID on Unix/Windows, and adds parser plus reparent regression tests.

## Best Possible Solution

Land the narrow inherited-PID protection after maintainer acceptance of the service-mode cleanup invariant, while keeping update, doctor, and daemon restart callers on the existing no-extra-PID cleanup path.

## Reproduction Assessment

Yes. Source inspection shows current main overwrites the inherited service PID before service-mode cleanup and cleanup excludes only self/ancestors, while the contributor supplied real-host logs for the reparented launchd case.

## Solution Assessment

Yes. Capturing the inherited PID before overwrite and feeding it into the existing cleanup exclusion set is the narrow owner-boundary fix; host zshrc cleanup is only a workaround, and broadening update/doctor cleanup would change intentional maintainer controls.
