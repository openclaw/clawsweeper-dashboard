## Label Justifications

## Summary

Keep open for maintainer review: the patch appears correct and the supplied real-host proof is sufficient, but merging it intentionally changes service-mode stale-process cleanup so a strictly inherited supervised gateway PID is protected rather than killed.

## What This Changes

This PR captures a strictly parsed inherited gateway service PID before service-mode gateway startup overwrites it, threads it into stale PID cleanup, and adds parser plus reparent-regression tests.

## Best Possible Solution

Merge the bounded service-mode exclusion if maintainers accept inherited supervised parent protection, while keeping explicit restart/update/doctor cleanup paths able to terminate the running gateway.

## Reproduction Assessment

Yes. Current main overwrites the inherited service PID before cleanup, Codex 0.135.0 zsh snapshots can source the legacy .zshrc auto-start block, and the PR supplies macOS launchd before/after logs, though I did not rerun the live host scenario in this read-only review.

## Solution Assessment

Yes. Capturing the inherited PID before overwrite and passing it only to the cleanup owner is the narrowest maintainable fix; host-side zshrc cleanup alone would not harden the OpenClaw primitive, and changing explicit restart/update cleanup would be too broad.
