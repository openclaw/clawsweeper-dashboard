## Summary

Current main still has the OpenResponses streaming mismatch: pending tool-call responses build a response with `status: "incomplete"` but emit the terminal SSE event as `response.completed`. The PR is still useful, but it needs a current-main refresh, docs coverage, and real HTTP SSE proof before merge.

## What This Changes

The PR changes OpenResponses streaming tool-call terminal events from `response.completed` to `response.incomplete`, adds the corresponding schema type, and expands gateway SSE tests.

## Best Possible Solution

Refresh this PR or open a narrow replacement so runtime, schema, docs, and current single/multi-call tests consistently emit and document `response.incomplete`, with redacted real SSE proof attached.

## Reproduction Assessment

Yes from source inspection. A streamed `/v1/responses` run that returns `meta.stopReason = "tool_calls"` with pending calls reaches the branch that emits `response.completed` while the response payload status is `incomplete`.

## Solution Assessment

No, not merge-ready as-is. The runtime/schema direction is the right narrow fix, but the branch must be refreshed over current multi-call coverage, documented, and backed by real SSE proof before merge.
