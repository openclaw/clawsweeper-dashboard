## Summary

Keep open. The PR addresses useful Discord and cron delivery UX gaps that are not implemented on current main, but it is not mergeable as-is because the auto-thread gate is patched into the old Discord threading body, cron failure details remain unbounded in the new formatter, and the user-facing config/behavior change lacks a changelog entry.

## Best Possible Solution
