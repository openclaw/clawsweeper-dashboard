## Summary

Keep open: current main and the latest release still pass Feishu text directly into a post Markdown `md` element, so underscore-containing bare URLs can still lose clickability. The target-repo fix PRs are closed unmerged, while the remaining fork PRs are dirty and not a merge-ready canonical fix.

## What This Changes

The issue requests Feishu outbound messages render underscore-containing URLs as complete clickable hyperlinks.

## Best Possible Solution

Normalize bare HTTP(S) URL presentation in `buildFeishuPostMessagePayload`, preserve existing Markdown/link/code regions, add send/edit regression coverage, and verify the complete link in a real Feishu client.

## Reproduction Assessment

Yes at source level: the reported CLI path reaches Feishu outbound text, which currently sends unmodified `messageText` in a post `md` element, matching the screenshot. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

Yes on boundary: the shared Feishu post payload builder is the narrowest maintainable fix point because normal sends and edits both reuse it. The currently linked fork PRs are not the best landable path until a maintainer adopts or replaces them with proof-backed target-repo work.
