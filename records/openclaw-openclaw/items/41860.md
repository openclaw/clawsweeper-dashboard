## Summary

Current main still matches the reported Feishu payload behavior: outbound text and text edits are serialized as Feishu post Markdown with raw message text, with no URL-boundary escaping or explicit link element for underscore-containing URLs. The issue also has open fix candidates, especially #41880 with closing syntax, so it should remain open until maintainers land or replace one canonical fix.

## What This Changes

Ensure Feishu outbound messages preserve the full clickable hyperlink when a URL contains underscores.

## Best Possible Solution

Land one Feishu-owned fix that preserves normal Markdown while making bare underscore URLs fully clickable in outbound send and edit payloads, with focused coverage for the reporter's URL shape and existing Markdown cases; then close the issue after the chosen PR merges.
