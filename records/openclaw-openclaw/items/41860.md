## Summary

Keep open: current main and the latest release still pass Feishu post text unchanged into the Feishu Markdown `md` element, and the target repository has no merged fix for the underscore-containing URL case. The issue is valid and should continue through the linked implementation work or a maintainer-selected replacement.

## What This Changes

OpenClaw should send Feishu messages containing URLs with underscores as full clickable links instead of letting Feishu Markdown split or truncate the URL.

## Best Possible Solution

Refresh or replace one Feishu-local fix that normalizes bare URLs before Feishu post/card Markdown while preserving existing Markdown constructs, then add focused send/edit regression coverage and live Feishu proof.

## Reproduction Assessment

Yes, at source-reproduction confidence: the reported CLI send path reaches Feishu `sendMessageFeishu()`, which puts the URL unchanged into a Feishu Markdown `md` post element. This review did not run a live Feishu tenant check.

## Solution Assessment

Yes on the plugin-local boundary, but not as a broad underscore escape. The maintainable fix should recognize bare URLs and preserve existing links, code regions, images, punctuation, mentions, and ordinary Markdown.
