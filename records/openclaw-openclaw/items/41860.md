Keep open: current main and the latest release still pass Feishu post text unchanged into a Markdown `md` element, and the same-repository fix attempts were closed unmerged. The issue remains the canonical tracker for a real Feishu URL-rendering bug.

## What This Changes

The issue asks OpenClaw to preserve full clickable Feishu hyperlinks when outbound message text contains URLs with underscores.

## Best Possible Solution

Land, refresh, or replace one Feishu-local fix that wraps or normalizes bare URL regions before Feishu post Markdown rendering while preserving existing links, code blocks, punctuation, mentions, send, and edit paths.

## Reproduction Assessment

Yes, at source level: the reported CLI send path can reach Feishu `sendMessageFeishu()`, which places the URL unchanged into a Feishu Markdown `md` post element. I did not run a live Feishu tenant check, so this is source-reproducible rather than freshly live-reproduced.

## Solution Assessment

Yes on the plugin-local boundary, but not as a broad underscore escape. The maintainable fix should preserve existing Markdown constructs and normalize only bare URL regions before Feishu post Markdown rendering.
