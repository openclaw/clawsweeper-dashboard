## Summary

Current main and the latest release still send ordinary Feishu text as unmodified post Markdown, so the underscore-URL rendering defect remains unresolved. All OpenClaw fix PRs are closed unmerged, leaving this issue as the canonical report.

## What This Changes

Fix Feishu post messages so bare URLs containing underscores remain fully visible and clickable.

## Best Possible Solution

Normalize bare HTTP(S) URL presentation once in `buildFeishuPostMessagePayload`, preserve existing Markdown constructs, add focused send-and-edit regression coverage, and verify the complete link in a real Feishu client.

## Reproduction Assessment

Yes at source level with high confidence: the exact CLI path reaches an unmodified Feishu post `md` element, matching the supplied screenshot and several independent linked investigations. A live current-main Feishu tenant run was not performed in this read-only review.

## Solution Assessment

Yes on ownership and boundary: the shared Feishu post payload builder is the narrowest maintainable fix point; caller-specific escaping or a URL-only alternate delivery path would leave sibling sends and edits inconsistent.
