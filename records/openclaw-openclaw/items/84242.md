The issue asks OpenClaw to expose or clearly diagnose memory-lancedb's registered memory_store, memory_recall, and memory_forget tools in eligible Codex/OpenClaw agent runs.

## Best Possible Solution

Make selected memory plugin agent tools intentionally visible through a documented Codex/tool-policy path, or emit a clear diagnostic when current policy excludes them.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a credible current-main path: memory-lancedb declares and registers the tools, while Codex dynamic tools pass through a coding-profile policy that does not include memory_store, memory_recall, or memory_forget by default.

## Solution Assessment

Unclear. Exposing memory_store by default may be the right fix, but because it is write-capable, maintainers should decide whether selected memory plugin tools should be auto-allowed or whether the runtime should provide an explicit allow path with diagnostics.
