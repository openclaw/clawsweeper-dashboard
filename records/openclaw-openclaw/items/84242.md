## Summary

Keep open: current main still has a credible gap between the memory-lancedb tool contract and the default Codex/OpenClaw agent tool surface. ## What This Changes

The issue asks OpenClaw to expose memory-lancedb's registered memory_store, memory_recall, and memory_forget tools as callable agent/Codex dynamic tools or diagnose why they are excluded.

## Best Possible Solution

Define a generic memory-plugin tool exposure policy, then align docs and tests so selected memory-slot tools are either callable by default or explicitly require tools.allow or tools.alsoAllow.
