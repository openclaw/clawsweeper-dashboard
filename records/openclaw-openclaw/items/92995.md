## What This Changes

The PR changes `detectTargetKind()` so channel plugins whose declared chat types are only `direct` classify bare targets as user/direct instead of group.

## Best Possible Solution

Keep the direct-only capability fallback only with focused resolver/session regression coverage and redacted live WeChat or session-key proof before merge.

## Reproduction Assessment

Yes at source level. Current main defaults bare unresolved targets to `group`, the WeChat plugin declares only `direct` plus `looksLikeId`, and the message send path forwards that resolved kind into outbound session mirroring.

## Solution Assessment

Yes, this is a clean fix layer for the reported message-send path because it prevents an impossible `group` kind before session routing. It still needs tests and live proof before merge because the fallback is generic and session-state visible.
