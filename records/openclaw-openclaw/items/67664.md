The PR changes agent session-store context-token persistence and tests to avoid unresolved fallback context-token writes, plus unrelated bundled plugin/channel type cleanup and an aimock ambient declaration.

## Best Possible Solution

Rebase or replace this with a focused session-store fix that suppresses unresolved fallback persistence while preserving current-main runtime metadata, heartbeat model preservation, compaction totals, and cost snapshot semantics; split unrelated extension typing cleanup.

Yes, from source inspection: unresolved lookup returns the caller fallback, session-store persists it, and status later treats the stored value as an override. I did not run tests because this review was required to keep the checkout read-only.
