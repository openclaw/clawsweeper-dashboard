## Summary

Keep open. The target bug is valid on current main and the PR has useful regression coverage, but the branch is not merge-ready because its session-store implementation drops current-main runtime context-window precedence and would reintroduce the #69347 cost accumulation bug.

## What This Changes

The PR changes agent session-store persistence to avoid saving default fallback contextTokens on unresolved lookups, adds context-token regression tests, and also carries unrelated bundled plugin/provider type cleanup plus an ambient @copilotkit/aimock declaration.

## Best Possible Solution

Land a focused session-store fix that only persists positive resolved context windows, preserves runtime-provided agentMeta.contextTokens, preserves existing values only when the authoritative runtime model/provider has not changed, clears stale values on unresolved model/provider drift, and keeps current-main cost snapshot and session metadata behavior intact.

## Reproduction Assessment

Yes. The code path is enough to reproduce: an unresolved model lookup returns the caller's DEFAULT_CONTEXT_TOKENS fallback, updateSessionStoreAfterAgentRun persists it, and status later treats that stored value as an override.

## Solution Assessment

No. The requested direction is right, but this branch is not the best current implementation until it is rebased or replaced with a narrower patch that preserves current-main runtime context-budget and cost-snapshot semantics.
