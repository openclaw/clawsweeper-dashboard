## Summary

Keep open: this targets a real session-store bug that still exists on current main, but the PR head is conflicting/stale, lacks real behavior proof, and would regress newer session metadata, compaction, heartbeat, and cost-snapshot behavior if merged as-is.

## What This Changes

The branch changes agent session-store persistence and regression tests to avoid writing fallback contextTokens on unresolved model lookup, while also including plugin/channel type cleanup and a new @copilotkit/aimock declaration.

## Best Possible Solution

Rebase or replace this with a focused session-store fix that avoids persisting unresolved fallback context windows while preserving runtime-reported contextTokens, same-model cached values, model/provider drift clearing, compaction snapshots, cost snapshots, lifecycle metadata, and heartbeat model isolation.

## Reproduction Assessment

Yes. Source inspection shows current main writes DEFAULT_CONTEXT_TOKENS through the unresolved session-store lookup path, and status later treats the persisted value as an override; no tests were run because this review is read-only.

## Solution Assessment

No. The requested direction is right, but this PR head is not the best merge candidate until it is rebased, narrowed, and updated to preserve current-main session metadata, compaction, heartbeat, and cost semantics.
