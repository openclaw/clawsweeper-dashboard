## Summary

Keep open because current main and the latest release contain assistant-prompt guidance, but the public rich-output/start docs no longer contain the requested warning or invalid examples for Markdown-wrapped MEDIA directives. Two related PRs tried to address this, but the merged rich-output docs commit is not contained in current main, and the absorbed docs sweep is also not in current ancestry.

## What This Changes

The issue requests public documentation, and optionally parser/logging improvements, for MEDIA directives being ignored when wrapped in Markdown formatting or inline prose.

## Best Possible Solution

Add a public docs warning on the rich-output protocol page, and any start page that still documents legacy final replies, saying MEDIA must be standalone plain text outside Markdown, code fences, and inline prose; keep parser leniency or warning logs as separate behavior work.

## Reproduction Assessment

Yes, from source inspection: splitMediaFromOutput skips fenced blocks and only parses lines whose trimmed text starts with MEDIA:, while tests keep prose MEDIA mentions as text. I did not run a live channel delivery repro in this read-only review.

## Solution Assessment

Yes for the remaining docs gap: documenting the current parser contract is the narrowest maintainable fix. Parser tolerance or warning logs would be broader behavior changes and should be handled separately if maintainers want them.
