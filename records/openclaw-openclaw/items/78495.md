## Summary

Keep open because current main still matches the reported behavior, and the docs/prompt copy only state the positive line-start rule without warning about Markdown wrappers.

## What This Changes

The issue asks OpenClaw to document and optionally harden MEDIA directive handling when the directive is bolded, code-wrapped, fenced, or placed inline.

## Best Possible Solution

Update the public rich-output/start docs and assistant prompt copy to state that MEDIA must be an unformatted line-start directive outside code fences; defer parser expansion to a separate behavior decision if maintainers want it.

## Reproduction Assessment

Yes, at source level: `splitMediaFromOutput` only parses lines whose trimmed start is `MEDIA:` and explicitly skips fenced code, so the reported wrapped and inline examples remain text on current main.

## Solution Assessment

Yes for the narrow docs/prompt path: it matches the existing contract and avoids changing intentionally guarded parser behavior around prose and code fences.

Update the MEDIA directive docs to say the directive must be an unformatted line-start directive outside Markdown code fences: no bold wrapping, no inline backtick wrapping around the whole directive, no fenced code block, and no prose before `MEDIA:` on the same line.
