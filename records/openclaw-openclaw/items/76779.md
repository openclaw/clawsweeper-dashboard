## Summary

Keep open: current main still has the Anthropic bundled policy artifact drift, and two open PRs already reference this issue with closing syntax, so the issue should stay open until one fix lands.

## What This Changes

The issue reports that Anthropic's bundled provider-policy artifact does not export `resolveThinkingProfile`, causing non-runtime Claude Opus 4.7 thinking profile lookups to miss `adaptive`, `xhigh`, and `max`.

## Best Possible Solution

Add the Anthropic lightweight `resolveThinkingProfile` export using the shared Claude helper, cover the bundled artifact path with regression tests, and land one canonical linked PR while closing the duplicate PR afterward if needed.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: non-runtime callers fall back to the bundled policy artifact, and the Anthropic artifact currently has no `resolveThinkingProfile` hook.

## Solution Assessment

Yes. The narrowest maintainable fix is to mirror the existing OpenCode/OpenAI lightweight artifact pattern by exporting Anthropic `resolveThinkingProfile` from `extensions/anthropic/provider-policy-api.ts` and adding focused artifact tests.
