## Summary

Keep open: current main still lacks the Anthropic lightweight `resolveThinkingProfile` export, and two open closing PRs already target this exact bug, so the issue should remain open until one fix lands.

## What This Changes

The issue asks OpenClaw to expose Anthropic's Claude thinking profile from the bundled provider-policy artifact so non-runtime callers keep Opus 4.7 `adaptive`, `xhigh`, and `max` levels.

## Best Possible Solution

Review and land one canonical narrow PR that exports the shared Claude thinking profile from the Anthropic provider-policy artifact, adds artifact-level regression tests, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the non-runtime resolver falls back to the bundled provider-policy hook, while the Anthropic artifact does not export that hook and therefore cannot surface the Opus 4.7 profile.

## Solution Assessment

Yes. Mirroring the existing OpenCode lightweight artifact pattern with `resolveClaudeThinkingProfile` is the narrowest maintainable fix; because open closing PRs already exist, this issue should stay open until one is reviewed and merged.
