## Summary

Keep open: current main still has the missing Anthropic bundled policy hook, while source, docs, and existing tests agree that Claude Opus 4.7 should expose the provider-owned `xhigh`, `adaptive`, and `max` thinking profile outside the active plugin runtime.

## What This Changes

The issue asks Anthropic's bundled provider-policy artifact to export `resolveThinkingProfile` so non-runtime thinking resolution preserves Claude Opus 4.7 `adaptive`, `xhigh`, and `max`.

## Best Possible Solution

Add the Anthropic lightweight policy export using the shared Claude thinking helper, keep plugin loading lazy, and cover the bundled artifact path with focused regression tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the resolver falls back to the bundled policy artifact's `resolveThinkingProfile`, but the Anthropic artifact does not export that hook, so non-runtime callers cannot see the Opus 4.7 profile.

## Solution Assessment

Yes. The narrowest maintainable fix is to mirror the existing OpenAI/OpenCode lightweight artifact pattern by exporting Anthropic `resolveThinkingProfile` from `extensions/anthropic/provider-policy-api.ts`, backed by artifact-level tests.
