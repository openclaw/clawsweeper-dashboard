## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid writing restored env secrets to an unignored file:** `scripts/migrate/restore-openclaw.sh:158`
  - body: Default restore copies the backed-up `.env` to `${ENV_FILE}.from-backup`; with the default repo env path that creates `.env.from-backup`, but the PR only ignores `.env` and `backups/`. Gateway tokens and provider env vars from the backup can therefore show up as untracked source files. Put the candidate under an ignored/private path or add a specific ignore before merge.
  - confidence: 0.96
- **[P1] Preserve the auth-profile secret directory:** `scripts/migrate/backup-openclaw.sh:117`
  - body: Docker setup persists `OPENCLAW_AUTH_PROFILE_SECRET_DIR` separately from `OPENCLAW_CONFIG_DIR`, and the docs say it contains the key for OAuth-backed auth profile token material. This backup stages only config/workspace/repo data, so a restore can copy `auth-profiles.json` without the key needed to use those provider credentials. Add backup/restore support for that directory or make the migration explicitly require reauth.
  - confidence: 0.93
- **[P2] Create migration backups with private permissions:** `scripts/migrate/backup-openclaw.sh:111`
  - body: The archive includes `.env`, auth profiles, sessions, and workspace state, but the script creates the output directory and tar/checksum files with the caller's normal umask. On common local umasks that can make the backup readable by other local accounts. Set a private umask or chmod the output directory and generated files before writing secrets into the archive.
  - confidence: 0.82
