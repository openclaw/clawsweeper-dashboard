## Best Possible Solution

Keep the Docker operator-helper direction only if maintainers want that scope, but add pre-extraction archive validation, align with the native restore design, settle keep-awake ownership, and require redacted macOS proof for restore and keep-awake behavior.

## Solution Assessment

No, not as submitted. The safer maintainable path is to validate archive entries and expected payload layout before extraction or destructive restore steps, then settle whether keep-awake belongs in this Docker helper or the gateway lifecycle path.
