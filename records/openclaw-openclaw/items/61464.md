Full review comments:

- **[P1] Include the auth-profile secret directory in migration backups:** `scripts/migrate/backup-openclaw.sh:101`
  - body: Current Docker setup persists `OPENCLAW_AUTH_PROFILE_SECRET_DIR` as a separate host mount for the auth-profile encryption key, but this backup stages only config/workspace/repo state. Restoring the copied `auth-profiles.json` without that key directory can leave migrated provider credentials unusable; add a default/override for backing up and restoring that directory too.
  - confidence: 0.91
- **[P1] Write backup archives with private permissions:** `scripts/migrate/backup-openclaw.sh:204`
  - body: The archive includes `.env` and OpenClaw config state, which can contain gateway tokens, auth profiles, and provider/channel secrets. With the default macOS umask, `tar -czf` can create a world-readable backup, so set a restrictive umask or chmod the archive and checksum outputs before reporting success.
  - confidence: 0.88
- **[P2] Reject unsafe tar members before extraction:** `scripts/migrate/restore-openclaw.sh:94`
  - body: The checksum proves the archive matches its companion file, but it does not prove the tar member paths are safe. A malicious archive with absolute or `..` entries can write outside the temp directory during this extraction, so list and reject unsafe members before calling `tar -xzf`.
  - confidence: 0.84
