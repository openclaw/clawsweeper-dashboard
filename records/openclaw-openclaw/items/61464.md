## Summary

Keep open: latest head still has a security-sensitive restore flow that extracts the migration archive before validating member names or layout, and the external PR still lacks real macOS Docker behavior proof.

## What This Changes

Adds macOS Docker host keep-awake, backup, and restore shell helpers plus Docker install guide documentation.

## Best Possible Solution

Either harden the shell restore to validate archive entries before extraction and add redacted macOS Docker proof, or consolidate restore/migration into the first-class backup restore/manual restore path while keeping host keep-awake as a separately approved surface.

## Reproduction Assessment

Yes. Source review of the latest PR head is enough to reproduce the blocking PR defect: `scripts/migrate/restore-openclaw.sh` extracts the archive at line 102 before validating entry names or layout.

## Solution Assessment

No. A Mac operator helper may be useful, but a separate restore archive contract is not the safest implementation until it matches current backup verifier semantics or reuses the core backup restore path and includes real behavior proof.
