## Summary

Keep open. The PR adds useful Mac-hosted Docker operator helpers and addressed earlier checksum/liveness/shared-helper review comments, but the current head still introduces security-sensitive backup/restore behavior that conflicts with the maintained backup archive safety contract on current main.

## What This Changes

The PR adds macOS Docker keep-awake, backup, and restore shell helpers plus Docker install documentation for keep-awake and Intel-to-Apple-Silicon migration workflows.

## Best Possible Solution

Keep the keep-awake/operator workflow if desired, but route backup semantics through the existing backup CLI contract or add an equally hardened restore path with pre-extraction member validation, safe default output, secret-handling docs, and focused tests.

## Reproduction Assessment

Yes. A source review of the PR head is enough: the restore script extracts the archive before validating member paths, and the backup script defaults secret-bearing output into an unignored checkout path.

## Solution Assessment

No. The PR solves a real operator need, but the backup/restore implementation is not the narrowest maintainable fix until it reuses or matches current main's archive verification and avoids repo-local secret archive defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Validate tar entries before extraction:** `scripts/migrate/restore-openclaw.sh:102`
  - body: The restore path runs `tar -xzf "$ARCHIVE_PATH" -C "$tmpdir"` before validating archive member names. A malicious or tampered archive with a matching companion checksum can include absolute or `../` entries and write outside the temp directory before the later `SHA256SUMS` check runs. List and reject unsafe entries, or reuse the existing backup verifier's traversal checks, before extraction.
  - confidence: 0.88
- **[P1] Avoid repo-local secret backup defaults:** `scripts/migrate/backup-openclaw.sh:83`
  - body: With no `--output-dir`, the script writes archives to `$REPO_ROOT/backups`, while the archive includes `.env` and OpenClaw state. Current ignore rules do not exclude that directory or these archive suffixes, so the documented default leaves credential-containing files as untracked checkout files that are easy to commit. Default outside the repo or add an explicit ignore before documenting this path.
  - confidence: 0.86
