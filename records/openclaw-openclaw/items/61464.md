Full review comments:

- **[P1] Snapshot live state before archiving:** `scripts/migrate/backup-openclaw.sh:140`
  - body: This `rsync` copies the Docker-mounted config/state tree while the gateway may still be running. Current `openclaw backup` snapshots SQLite with `VACUUM INTO` and filters volatile files, so this migration path can produce a torn archive of `state/openclaw.sqlite` or agent databases; reuse the backup pipeline or quiesce/snapshot SQLite before archiving.
  - confidence: 0.9
- **[P2] Resolve destinations from the applied env:** `scripts/migrate/restore-openclaw.sh:175`
  - body: The script resolves `CONFIG_DIR`, `WORKSPACE_DIR`, and `AUTH_PROFILE_SECRET_DIR` before copying the archived `.env`. With `--apply-env`, Compose can then point at restored env paths that were never populated on the target host, so derive destinations from the applied env or keep the env staged for review.
  - confidence: 0.88
- **[P2] Stage the archived Docker setup files:** `scripts/migrate/backup-openclaw.sh:158-163`
  - body: This loop archives Dockerfile, Compose files, and `scripts/docker/setup.sh`, but restore only processes `payload/repo/.env`. Operators with customized setup files can rebuild on the target using stale defaults, so restore/stage those files safely or stop archiving and documenting them.
  - confidence: 0.87
