## Summary

Keep this PR open: it fixes a real heartbeat reasoning leak, but the current patch can suppress opt-in reasoning-only heartbeat output because the runner returns through the empty-main branch before sending `reasoningPayloads`.

## What This Changes

The PR changes heartbeat reply payload selection to skip `isReasoning` payloads and adds selector regression tests.

## Best Possible Solution

Keep the selector-level reasoning skip, but update the heartbeat runner so `includeReasoning: true` still delivers reasoning payloads when there is no main reply, with focused runner coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main returns a trailing reasoning payload because the selector checks only outbound content, and the PR body includes before/after live selector output for that shape.

## Solution Assessment

Is this the best way to solve the issue? Not yet: skipping reasoning in the main selector is the right layer for the leak, but the runner must also preserve the separate `includeReasoning` delivery path when the main reply becomes empty.
