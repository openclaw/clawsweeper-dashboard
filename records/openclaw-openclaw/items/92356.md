## Summary

Keep this PR open, but it still needs changes before merge: the latest patch fixed the flagged and exact `Reasoning:` cases, yet it adds a narrower heartbeat-local reasoning classifier instead of using the existing SDK classifier, so SDK-recognized quoted/lowercase reasoning can still leak as the visible heartbeat reply.

## What This Changes

The PR changes heartbeat reply selection and runner reasoning delivery to skip reasoning payloads and adds selector plus runner regression tests.

## Best Possible Solution

Land a single canonical heartbeat fix that uses the existing SDK reasoning classifier for both selector and reasoning delivery, preserving `includeReasoning` behavior and covering flagged, prefixed, lowercase, and quoted reasoning regressions.

## Reproduction Assessment

Yes at source level. Current main selects the last outbound payload, and this PR's new classifier still misses forms already covered by the SDK helper, such as markdown-quoted reasoning payloads.

## Solution Assessment

Not yet. Selector-level filtering and the runner early-return fix are the right shape, but the best implementation should reuse the existing SDK reasoning classifier instead of adding a narrower duplicate detector.
