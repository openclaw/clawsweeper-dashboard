## Best Possible Solution

Reuse the existing shared reasoning classifier for heartbeat selection and keep runner/selector regressions covering flagged, legacy-prefixed, lowercased, and blockquoted reasoning while preserving opt-in reasoning delivery.

## Reproduction Assessment

Yes. Current main selects the last outbound heartbeat payload without checking reasoning, and source inspection shows the PR still misses reasoning forms covered by the shared SDK classifier.
