## Summary

Keep open: current main suppresses the older heartbeat metadata-only exec leak, but the exec approval/session-resume path still has source-supported ways to produce the reported `Background command finished.` and missing-context behavior, and the overlapping fix PR is still open.

## What This Changes

The issue asks OpenClaw to suppress or safely handle async exec completions whose originating session can no longer be resolved so channel users do not see missing-context agent replies.

## Best Possible Solution

Unify async exec completion handling so successful no-output completions do not reach users as missing-context prompts, while output-bearing and failed completions retain bounded useful delivery.

## Reproduction Assessment

No live high-confidence repro was established in this read-only pass. Source and tests show the heartbeat path is guarded but the exec approval follow-up/direct fallback path can still emit `Background command finished.` for a successful no-output completion.

## Solution Assessment

Unclear. The narrow code areas are visible, but maintainers should decide one consistent policy for successful no-output async exec completions before queueing another automated fix or landing the overlapping PR.
