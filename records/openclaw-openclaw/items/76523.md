## Summary

Keep open. Current main still sends successful metadata-only exec completions through a model-visible heartbeat or approval-followup path, so a noncompliant model can still produce the reported user-visible missing-context reply.

## What This Changes

## Best Possible Solution

Consume successful metadata-only exec completions before any model-visible relay/resume turn, and give approval followups the same no-output-success guard while preserving output-bearing and failed completion delivery.
