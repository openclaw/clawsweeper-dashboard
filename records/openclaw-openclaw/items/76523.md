## Summary

Keep open. Current main and the latest release cover the older heartbeat metadata-only exec completion leak, but the exec approval/session-resume follow-up path still has source-supported ways to emit the reported `Background command finished.`/missing-context pattern, and the overlapping open PR is not merged or proven canonical for this issue.

## What This Changes

The issue reports that async/background exec completions can still surface a user-visible missing-session/context apology after the earlier metadata-only completion fix.

## Best Possible Solution

Keep the heartbeat metadata-only suppression, then align the exec approval follow-up path so successful no-output completions either resolve through durable result context or are silently/boundedly handled without prompting an agent to retrieve or ask the user for missing logs.

## Reproduction Assessment

Yes, at source level but not as a live current-main repro. The heartbeat metadata-only path is guarded, while the exec approval follow-up path can still route a successful completion through an agent continuation and direct-send `Background command finished.` when there is no output.

## Solution Assessment

Unclear. The narrow code areas are visible, but maintainers should decide one consistent policy for successful no-output async exec completions across heartbeat relay, approval follow-up, and direct fallback before queueing another automated fix.
