## Summary

Keep open. Current main still lets successful metadata-only async exec completions reach an agent prompt in text-mode heartbeat runs, and a non-sentinel missing-context reply can still be delivered to Telegram or another chat surface; #75322 is related but covers the `deliverToUser=false` empty-turn path rather than this user-visible relay path.

## What This Changes

The issue asks OpenClaw to suppress user-visible missing-context replies for successful metadata-only async exec completions after the originating session can no longer be resolved.

## Best Possible Solution

Consume successful no-output exec completions at runtime before any agent-visible relay turn, while preserving user-visible handling for failed or output-bearing completions and adding regression coverage for non-sentinel model text.

## Reproduction Assessment

No high-confidence live reproduction is available from the report alone. Source inspection gives a high-confidence unit path: enqueue `Exec completed (..., code 0)` with no output and have the agent return visible missing-context text; current delivery code would not classify that reply as skippable.

## Solution Assessment

No, the current prompt-based suppression is not the best fix because it relies on model compliance. The narrower maintainable fix is runtime-level silent consumption for successful metadata-only completions before user delivery is possible.
