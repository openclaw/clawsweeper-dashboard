## Summary

Keep open: current main covers the older heartbeat metadata-only exec leak, but the separate exec approval follow-up path can still resume an agent with successful no-output completion metadata or directly send the reported `Background command finished.` fallback.

## What This Changes

The issue reports that async exec completion can still produce a user-visible no-context agent reply after the earlier metadata-only heartbeat fix.

## Best Possible Solution

Unify async exec completion handling so successful no-output completions stay internal unless there is meaningful output, failure status, or an explicit user-facing completion request.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection still shows a successful no-output exec approval follow-up can resume an agent with raw completion metadata or fall back to `Background command finished.` without preserving useful originating context.

## Solution Assessment

Unclear. The desired end state is right, but the maintainable fix should coordinate heartbeat relay, exec approval follow-up, direct fallback, and overlapping exec-completion PRs instead of adding another one-sided guard.
