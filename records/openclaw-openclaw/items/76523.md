## Summary

Keep open: current main suppresses the older heartbeat metadata-only exec leak, but source still shows the exec approval/session-resume path can wake an agent or fall back with the reported successful no-output completion shape, and overlapping PRs are still unmerged.

## What This Changes

The issue asks OpenClaw to stop surfacing successful async exec completions as missing-session or missing-context chat replies after the earlier #74595 fix.

## Best Possible Solution

Unify async exec completion handling so successful no-output completions stay internal unless there is meaningful output, failure status, or an explicit user-facing completion request.

## Reproduction Assessment

No live high-confidence reproduction was established. The source path is clear: a successful no-output exec approval follow-up can still resume an agent with raw completion metadata or fall back to `Background command finished.` without preserving the originating context.

## Solution Assessment

Unclear. The desired end state is right, but the maintainable fix should coordinate heartbeat relay, exec approval follow-up, direct fallback, and open overlapping PRs instead of adding another one-sided guard.
