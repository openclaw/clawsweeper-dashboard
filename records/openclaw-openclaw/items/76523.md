## Summary

Keep open. Current main still sends successful metadata-only async exec completions through an agent prompt and only suppresses delivery when the model returns the expected quiet response, so a noncompliant missing-context apology can still reach Telegram or another chat surface.

## What This Changes

The issue reports that successful metadata-only async exec completions can still produce a user-visible missing-context agent reply after the prior #74595 fix.

## Best Possible Solution

Consume successful metadata-only exec completions at runtime before any agent-visible relay turn, while preserving user-visible delivery for failures and completions with captured output.

## Reproduction Assessment

Yes for a source-level reproduction: enqueue `Exec completed (..., code 0)` with no output and have the mocked agent return missing-context text instead of the quiet sentinel; current main would deliver that visible text. No live Telegram reproduction was established during this read-only review.

## Solution Assessment

Yes, if implemented as a runtime-level suppression guard rather than another prompt tweak. Prompt-only suppression is not the best fix because it depends on model compliance after the runtime has already exposed the orphan completion to the agent.
