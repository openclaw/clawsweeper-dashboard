## What This Changes

The issue asks OpenClaw to stop surfacing user-visible no-context async exec completion replies after session/auth-epoch invalidation, extending the prior metadata-only async exec suppression fix.

## Best Possible Solution

Define one async exec completion policy across heartbeat events and exec approval follow-ups so successful metadata-only/no-output completions are silently consumed or safely summarized without letting an agent ask users for missing session context.
