## Summary

Keep open: current main fixed the older heartbeat metadata-only exec leak, but the exec approval follow-up path still resumes an agent with successful no-output completion metadata or directly sends the reported `Background command finished.` fallback.

## What This Changes

Requested behavior: suppress user-visible async exec completion replies when the originating session/context for a completed background command can no longer be resolved.

## Best Possible Solution

Unify async exec completion handling so successful no-output completions stay internal unless there is meaningful output, failure status, or an explicit user-facing completion request.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection still shows a successful no-output exec approval follow-up can resume an agent with raw completion metadata or fall back to `Background command finished.` without preserving originating context.

## Solution Assessment

Unclear. The desired end state is right, but the maintainable fix should coordinate heartbeat relay, exec approval follow-up, direct fallback, and open overlapping PRs instead of adding another one-sided guard.
