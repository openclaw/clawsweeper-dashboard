## Summary

Keep open. Current main still relies on a model prompt to suppress successful metadata-only async exec completions, so a non-sentinel missing-context reply can still reach Telegram or another chat surface despite the prior #74595 release note.

## What This Changes

Suppress user-visible async/background exec completion replies when a successful completion has no captured output and the original session context is unavailable.

## Best Possible Solution

Consume successful structured metadata-only exec completions at the runtime level before any agent-visible relay turn, with regression coverage proving that a model-style missing-context apology is not delivered while failed or output-bearing completions still relay.

## Reproduction Assessment

No live deterministic reproduction was established, but there is a high-confidence source reproduction: enqueue `Exec completed (id, code 0)` with no output and have the mocked agent return visible missing-context text instead of `HEARTBEAT_OK`; current main would not classify that reply as skippable.

## Solution Assessment

No. The current prompt-based suppression is not the best fix because it depends on model compliance; runtime-level consumption before agent prompting is the narrower maintainable solution.
