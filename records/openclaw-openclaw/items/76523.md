## Summary

Keep open. Current main still lets a successful metadata-only async exec completion become an agent-visible heartbeat turn, so a non-sentinel missing-context reply can still be delivered to Telegram or another chat surface.

## What This Changes

The issue asks for async exec completions with missing or invalidated session context to be consumed silently instead of producing a user-visible reply asking for the original command or output.

## Best Possible Solution

Successful metadata-only exec completions should be suppressed in runtime code before any agent-visible relay turn, while failed or output-bearing completions remain deliverable with stable diagnostic text.

## Reproduction Assessment

Yes for source-level reproduction, but not for a live gateway replay from the provided logs. Enqueue a successful no-output exec completion and have the mocked agent return visible missing-context text instead of HEARTBEAT_OK; current delivery logic would treat that as a normal outbound reply.

## Solution Assessment

No for the current main behavior. Prompt-based suppression depends on model compliance, while the safer fix is a runtime-level silent-consume guard for successful metadata-only completions before model execution or outbound delivery.
