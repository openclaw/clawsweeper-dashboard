## Summary

Keep open. Current main narrows which exec completions are relayable, but successful metadata-only exec completions still go through an agent prompt and only stay silent if the model/tool response cooperates, so a visible missing-context reply can still reach chat surfaces.

## What This Changes

The issue asks OpenClaw to suppress user-visible async exec completion replies when a successful background command has no captured output and its originating session context is no longer resolvable.

## Best Possible Solution

Consume successful structured metadata-only async exec completions as a runtime no-op before model prompting or outbound delivery, while preserving user-visible relay for failures and completions with captured output.

## Reproduction Assessment

No high-confidence live reproduction was run. The source path is reproducible by enqueueing `Exec completed (..., code 0)` with no output and having the mocked agent return visible missing-context text instead of the expected silent response.

## Solution Assessment

No. The current prompt-based suppression is not the best fix because it depends on model compliance; the safer narrow fix is runtime-level silent consumption before any user-deliverable agent response can be generated.
