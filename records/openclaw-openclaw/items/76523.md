## Summary

Keep open: current main still lets a successful metadata-only async exec completion become an agent turn and only suppresses delivery when the model returns the expected silent acknowledgment, so the user-visible missing-context reply can still leak.

## What This Changes

The issue asks OpenClaw to suppress or safely handle successful metadata-only async exec completions whose originating session context is gone instead of delivering an agent apology to Telegram.

## Best Possible Solution

Consume successful no-output exec completions at the runtime boundary before model relay or direct fallback delivery, while preserving output/error completions and adding regression coverage for noncompliant model text.

## Reproduction Assessment

No high-confidence live reproduction is available from the issue logs alone. The source path is high-confidence: enqueue a successful `Exec completed (..., code 0)` event with no output and have the mocked agent return missing-context prose instead of `HEARTBEAT_OK`; current main would not classify that prose as skippable.

## Solution Assessment

No, the current prompt-based suppression is not the best fix because it depends on model compliance. The narrower maintainable solution is runtime-level suppression for successful metadata-only completions before any user-visible relay can be produced.
