## Summary

Keep open: current main has partial protection for heartbeat metadata-only exec completions, but the exec approval follow-up path still resumes an agent session for successful no-output completions when a session key exists, which can still produce the reported user-facing context-loss reply shape.

## What This Changes

The issue asks OpenClaw to stop surfacing async exec completion follow-ups that ask users for missing command/session context after the originating session can no longer be resolved.

## Best Possible Solution

Define one async exec completion policy across heartbeat events and approval follow-ups so successful metadata-only/no-output completions are silently consumed or safely summarized without letting an agent ask users for missing session context.

## Reproduction Assessment

No live high-confidence repro was established for the auth-epoch Telegram race. ## Solution Assessment

No, current main is only a partial solution. The heartbeat metadata-only exec event path is guarded, but the exec approval follow-up path still needs a maintainer-approved policy for no-output successful completions and stale session handling.
