## Summary

Keep open: current main fixed the older heartbeat metadata-only exec relay, but the exec approval follow-up path still resumes an agent with successful no-output completion metadata or directly sends the reported `Background command finished.` fallback.

## What This Changes

The issue reports that async/background exec completions can still produce Telegram-visible missing-session or no-context replies after the earlier metadata-only heartbeat fix.

## Best Possible Solution

Unify async exec completion policy so successful no-output completions stay internal unless there is meaningful output, failure status, or an explicit user-facing completion request.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection shows the remaining path: successful no-output exec approval follow-ups can resume an agent with raw completion metadata or directly send `Background command finished.` after session resolution fails.

## Solution Assessment

Unclear. The desired end state is right, but the maintainable fix should coordinate heartbeat relay, exec approval follow-up, direct fallback, and the overlapping open PR instead of adding another one-sided guard.
