## Summary

Keep open. Current main still has a source-level path where a successful metadata-only async exec completion is sent through an agent turn and only suppressed if the model emits the expected quiet sentinel, so a model-produced missing-context apology can still reach Telegram or another chat surface.

## What This Changes

Suppress successful metadata-only async exec completions so they do not produce user-visible missing-context replies after session resolution fails.

## Best Possible Solution

Consume successful no-output exec completions at runtime before any model-visible relay turn, while keeping output-bearing and failed completions deliverable and covered by regression tests.

## Reproduction Assessment

Yes for a source-level reproduction, not for the reporter's live Telegram race: enqueue `Exec completed (..., code 0)` with no output and have the mocked heartbeat agent return visible missing-context text instead of `HEARTBEAT_OK`; current delivery logic would treat that text as sendable.

## Solution Assessment

No. The current fix is not the best solution because it relies on model compliance with a quiet sentinel; the safer fix is runtime-level suppression before the agent can generate a user-visible missing-context reply.
