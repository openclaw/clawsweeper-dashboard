## Summary

Keep open: the reported attachment-drop bug is still present on current main, but this PR is not merge-ready because it targets a stale inline implementation, conflicts with main, lacks real behavior proof, and uses media-alias precedence that can diverge from message-action sends.

## What This Changes

The PR changes the Plugin SDK outbound reply normalizer and tests so loose reply payloads can turn `filePath`, `path`, or `media` into `mediaUrl`.

## Best Possible Solution

Port the fix to the shared outbound reply normalizer, align media alias precedence with `buildSendPayloadParts`, add focused regression coverage, and require redacted live delivery proof before merge.

## Reproduction Assessment

Yes at source level: current main normalizes loose outbound replies from `mediaUrls`/`mediaUrl` only, and inbound reply dispatch runs loose payloads through that normalizer before delivery. I did not establish a live WhatsApp reproduction in this read-only review.
