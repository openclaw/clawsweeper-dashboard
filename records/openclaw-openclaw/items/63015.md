## Summary

Keep open: the reported attachment-drop path is still valid on current main, but this PR needs a small contract correction and real behavior proof before merge.

## What This Changes

This PR changes plugin SDK outbound reply normalization to treat `filePath`, `path`, and `media` as single-media aliases and adds unit tests for those fallbacks.

## Best Possible Solution

Finish this as a narrow SDK reply-normalization fix that mirrors the send-action alias contract, including `fileUrl` and precedence coverage, then verify a real gateway attachment send.

## Reproduction Assessment

Yes, by source inspection. On current main, a loose final reply payload with only `filePath` reaches `normalizeOutboundReplyPayload` through inbound reply dispatch and leaves `mediaUrl` undefined.

## Solution Assessment

No, not as written. The change is in the right file, but the alias order and missing `fileUrl` support should be corrected so reply delivery matches the existing send-action contract instead of creating a second precedence rule.
