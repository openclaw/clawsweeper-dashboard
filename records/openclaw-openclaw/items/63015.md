## Summary

Keep open: current main still drops `filePath`/`path`/`media` in outbound reply normalization, but this branch is not merge-ready because the normalizer moved to a new canonical file, alias precedence conflicts with the send-action contract, and real channel proof is still missing.

## What This Changes

The branch makes outbound reply normalization treat `filePath`, `path`, and `media` as single-media fallbacks and adds unit tests for those aliases.

## Best Possible Solution

Implement alias resolution once in `src/infra/outbound/reply-payload-normalize.ts`, keep the SDK helper as a wrapper, align fallback order with the current message-send contract, and require redacted live channel proof before merge.

## Reproduction Assessment

Yes, at source level: current main's canonical normalizer only reads `mediaUrl`, so a loose outbound reply with only `filePath`, `path`, or `media` normalizes without a media attachment. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No. The intended fix is valid, but the branch must be updated to the current canonical normalizer and align alias precedence before it is the best merge shape.
