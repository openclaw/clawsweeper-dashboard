## What This Changes

The branch writes env-backed provider API-key references as `secretref-env:` markers, resolves those markers at model-registry use, and updates secrets-audit handling, tests, and docs for canonical and stale generated files.

## Best Possible Solution

Land the canonical prefixed marker with the exact provider-scoped stale-file bridge, then rely on normal generated-file regeneration to eliminate old bare markers without producing misleading audit warnings.

## Reproduction Assessment

## Solution Assessment

Yes. Extending the shared SecretRef parser and the existing generated-header marker convention is the narrowest maintainable fix; the only unresolved choice is explicit acceptance of the stale generated-file compatibility bridge.
