## What This Changes

The issue reports two Telegram long-polling silent message loss modes in v2026.5.7: inbound follow-up messages during an active run and completed outbound replies that do not reach Telegram.

## Best Possible Solution

Preserve the existing Telegram retry safety policy while making busy-run follow-ups reliably enqueue/drain and making final-reply delivery failures observable with redacted diagnostics and focused regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run here. The current source contains the exact `no_active_run` and silent-reply-dispatch paths, and the docs define the expected queue fallback, so the issue is source-reproducible but still needs live Telegram proof before landing a fix.

## Solution Assessment

Unclear for a direct automated fix in this review. The best direction is a focused Telegram/queue repair with regression tests and live Telegram QA, while avoiding broad ambiguous send retries or logging raw reply payloads.
