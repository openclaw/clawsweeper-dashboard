## Summary

Keep open: this is a plausible focused ACP bug fix, and the later branch commits addressed the earlier over-broad stale-binding cleanup concern, but the PR still lacks contributor real behavior proof and its retry loop would regress current main's config-key resolution if merged as-is.

## What This Changes

The PR adds ACP persistent-session recovery around config-option updates, relaxes missing resume-state clearing, expands stale-binding detection for ACP_TURN_FAILED, and adds ACP manager/dispatch tests.

## Best Possible Solution

Port the retry around current main's setSessionConfigOption implementation while preserving wireKey/status-derived config-option behavior, keep the narrowed stale-binding predicate, and require redacted live proof before merge.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection shows ACP runtime-option commands call setSessionConfigOption, and current main has no retry around stale persistent-session ACP_TURN_FAILED errors in that method.

## Solution Assessment

No, not as currently proposed. The retry seam is narrow and plausible, but it needs to wrap current main's config-key resolution and drop the dead fallback throw before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve current config-key resolution inside the retry:** `src/acp/control-plane/manager.core.ts:580-609`
  - body: The retry loop restores an older setSessionConfigOption path: it resolves capabilities without includeStatusConfigOptionKeys, validates the raw key, and sends key to the runtime. Current main supports status-derived options and maps aliases such as thinking -> effort through wireKey, so applying this diff would reject or mis-send those config updates. Wrap the current main logic inside the retry instead.
  - confidence: 0.88
- **[P3] Remove the unreachable retry fallback throw:** `src/acp/control-plane/manager.core.ts:651-654`
  - body: The two-attempt loop returns on success, continues only from the first retry, or rethrows from the catch block; on the second attempt shouldRetry is false, so execution cannot fall through to this synthetic ACP_TURN_FAILED. Keeping it makes future failures harder to reason about.
  - confidence: 0.82
