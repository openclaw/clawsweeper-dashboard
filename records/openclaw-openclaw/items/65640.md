## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve current config-key resolution in the retry loop:** `src/acp/control-plane/manager.core.ts:580-610`
  - body: The retry rewrite is based on an older `setSessionConfigOption()` path: it resolves capabilities without status-derived option keys and sends the raw `key` to the runtime. Current main maps user-facing keys through `wireKey` and supports status-only config options, so rebasing this as-is would break backends that require aliases such as `thinking` -> `effort`. Wrap the current main logic inside the retry instead.
  - confidence: 0.86
- **[P3] Remove the unreachable retry fallback throw:** `src/acp/control-plane/manager.core.ts:651-654`
  - body: The two-attempt loop returns on success, continues from the first retry, or rethrows from the catch block; execution should not fall through to this synthetic `ACP_TURN_FAILED`. Keeping it makes future debugging misleading and should be removed or the loop should be restructured.
  - confidence: 0.82
