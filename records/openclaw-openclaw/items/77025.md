## Summary

Keep open: current main has session-level model/thinking/fast/verbose controls, but no chat/agent mode that changes prompt assembly, history budget, or tool exposure. This is an additive cross-runtime feature request, not a cleanup item that should be closed.

## What This Changes

Request a session-persistent `/mode chat|agent` switch for lower-cost conversational turns with lighter prompt, shorter history, and reduced tools.

## Best Possible Solution

Define an additive Gateway/session protocol mode contract first, then wire clients and runtime policy so chat mode has explicit prompt, history, and tool limits while agent mode preserves today’s behavior.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior. Source inspection on current main shows the requested mode field and `/mode chat|agent` behavior are absent.

## Solution Assessment

Unclear as an implementation path because there is no patch to review yet; current `/fast` is not the same solution because it is provider/model fast-mode plumbing, not a lightweight conversational runtime.
