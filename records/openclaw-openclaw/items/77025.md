## What This Changes

The issue requests an additive per-session `/mode chat` and `/mode agent` switch that reduces prompt, history, and tool exposure for lightweight conversations while preserving full agent behavior by default.

## Best Possible Solution

Define one core session-mode contract that drives command UX, durable session state, prompt assembly, history limiting, tool policy, docs, and tests across user chat surfaces.
