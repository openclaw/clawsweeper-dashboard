## Summary

Keep open: current main has adjacent pieces for prompt modes, tool profiles, history limits, and per-session controls, but it does not have a user-facing per-session chat/agent mode. The request is a new core runtime/session capability and needs maintainer product design before implementation.

## What This Changes

The issue requests an additive per-session `/mode chat|agent` switch for lower-cost lightweight conversations with lighter prompt, shorter history, and reduced tool exposure.

## Best Possible Solution

Define a durable core session-mode contract that consistently drives command UX, persisted session state, prompt assembly, history policy, tool exposure, docs, and tests across TUI, WebChat, and messaging channels.

## Reproduction Assessment

Not applicable. This is an additive feature request, and source review confirms current main has no `/mode chat|agent` user-facing behavior to reproduce as a bug.

## Solution Assessment

Unclear until maintainer design. The requested direction is plausible, but the maintainable fix should be a core session/runtime contract rather than only a slash-command alias because it affects prompt assembly, history, persistence, and tool policy.
