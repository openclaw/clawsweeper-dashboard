## Summary

Keep open: current main has adjacent prompt modes, tool profiles, session toggles, and history limits, but no user-facing per-session chat/agent mode. This is an additive cross-surface feature that needs maintainer product design before implementation.

## What This Changes

The issue requests an opt-in per-session chat/agent mode that reduces prompt, history, and tool exposure for lightweight conversations.

## Best Possible Solution

Define a core session mode contract and implement it as durable per-session state that consistently drives command UX, prompt assembly, history limiting, tool policy, docs, and tests across TUI, WebChat, and messaging channels.

## Reproduction Assessment

Not applicable. This is an additive feature request, and source review confirms current main does not provide `/mode chat`, `/mode agent`, or equivalent user-facing session behavior to reproduce as a bug.

## Solution Assessment

Unclear until maintainer design. The requested direction is plausible, but the maintainable solution should be a shared core session/runtime contract rather than only a slash-command alias because it affects prompt assembly, history, and tool policy.
