## Summary

Keep open: current main and v2026.6.8 still lack a user-facing, session-persistent chat/agent mode, and this remains a cross-surface product/runtime design decision rather than conservative cleanup.

## What This Changes

The issue requests an opt-in session-level `/mode chat` and `/mode agent` switch that persists per session and changes prompt weight, history window, and tool exposure for lightweight conversations.

## Best Possible Solution

Define a maintainer-approved core session-mode contract first, then wire it through durable session state, command/UI surfaces, prompt/history/tool policy, docs, and focused tests without changing the default full-agent behavior.

## Reproduction Assessment

Not applicable: this is an additive feature request, not a broken existing behavior. Source and docs inspection confirm the requested user-facing `/mode chat` and `/mode agent` behavior is absent on current main and v2026.6.8.

## Solution Assessment

Unclear until maintainer design: the proposed slash command is plausible, but the best implementation needs one core session-mode contract instead of wiring today’s lower-level promptMode, bootstrapContextMode, and toolsAllow primitives directly to a command.
