## Summary

Keep open: the source-level bug is real and the patch is directionally narrow, but this external PR still lacks inspectable after-fix Fly proof and leaves non-container gateway tests dependent on ambient Fly runtime env vars.

## What This Changes

The branch treats Fly Machines as container environments via `FLY_MACHINE_ID` plus `FLY_APP_NAME` in core gateway detection and the Bonjour plugin, with unit coverage for plain cgroup output.

## Best Possible Solution

Land the narrow env-var detector after hermetic test setup and redacted Fly logs or terminal output prove detection and Bonjour disablement on a real Machine.

## Reproduction Assessment

Yes at source level, not live: current main only checks sentinel files and cgroup markers, so plain Fly cgroup output plus Fly runtime env vars is missed. I did not execute a Fly Machine in this read-only review.

## Solution Assessment

No as submitted: the detector change is narrow, but the test suite should clear Fly env vars and the PR needs inspectable external proof before merge.
