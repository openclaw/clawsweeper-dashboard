## Summary

Keep open: current main still stores skipped WhatsApp group history as text-only and later maps that history into `InboundHistory` without media, even though the inbound message shape and shared history contract already support bounded media metadata. The latest release tag also still has the text-only WhatsApp path, so this issue remains necessary.

## What This Changes

Request to preserve bounded WhatsApp media metadata from unmentioned `requireMention` group history so a later mentioned turn can inspect the recent image.

## Best Possible Solution

Reuse the existing `HistoryEntry.media` and `createChannelHistoryWindow().recordWithMedia()` seam in WhatsApp group gating, preserving local-image bounds, sender visibility filtering, and regression coverage for the image-before-mention flow.

## Reproduction Assessment

Yes at source level: an unmentioned WhatsApp group image can enter the inbound message as `payload.media`, but the missing-mention history recorder and later `InboundHistory` mapping keep only text fields. I did not run live WhatsApp proof or tests because this review is read-only.

## Solution Assessment

Yes: the narrow maintainable fix is to use the existing shared `HistoryEntry.media` and `recordWithMedia()` contract, not add WhatsApp-specific parallel media fields. The fix must preserve current filtering and bounded local-image retention.
