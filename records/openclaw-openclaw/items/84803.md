## Summary

Keep open: current main and the latest release still record skipped WhatsApp group history as text-only, even though the shared channel-history media contract can carry bounded local image media into later turns.

## What This Changes

The issue requests preserving media metadata from unmentioned WhatsApp group messages so a later mentioned turn can inspect recent images kept as pending group history.

## Best Possible Solution

Reuse the shared channel-history media contract in the WhatsApp mention-gating path: record bounded local image media on skipped group-history entries, pass it through the existing visible-history filter, and add regression coverage for the image-before-later-mention flow.

## Solution Assessment

Yes: the narrow maintainable fix is to use the existing `recordWithMedia`/`HistoryMediaEntry` path and current visibility filter, not add a WhatsApp-only retention system or pass raw remote media URLs.
