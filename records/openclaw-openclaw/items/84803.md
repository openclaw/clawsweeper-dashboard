## Summary

Keep open: current main still records pending WhatsApp group history as text-only, while the inbound path already has media metadata and the channel-turn contract supports bounded history media. This is a valid source-reproducible WhatsApp bug and is distinct from the already-closed quoted/replied-media reports.

## What This Changes

Preserve bounded WhatsApp group-history media from unmentioned `requireMention` messages so a later mentioned turn can inspect the image instead of seeing only a placeholder.

## Best Possible Solution

Use the existing channel history media contract in the WhatsApp group-gating path: record bounded local image media for skipped group-history entries, pass it through `InboundHistory` after visibility filtering, and add regression coverage for the `requireMention` image-before-mention flow.

## Reproduction Assessment

Yes at source level: an unmentioned WhatsApp group image can enter `applyGroupGating()` with `mediaPath`/`mediaType`, but the skipped-history entry and later `InboundHistory` mapping keep only text. I did not run a live WhatsApp repro or tests in this read-only review.

## Solution Assessment

Yes: the maintainable fix is to reuse `createChannelHistoryWindow().recordWithMedia()` and the existing image-only/local-path history media contract, not add a new retention mode or WhatsApp-specific parallel structure.
