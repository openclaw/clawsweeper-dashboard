## Summary

Keep open: current main and the latest release still record WhatsApp missing-mention group history as text-only, so later mentioned turns can receive `<media:image>` history without usable image metadata. The report remains actionable, but the fix is security-sensitive because it changes retained unmentioned group media.

## What This Changes

The issue asks OpenClaw to preserve bounded media metadata for unmentioned WhatsApp group messages stored as pending history before a later mention.

## Best Possible Solution

Reuse the shared `HistoryEntry.media` and `recordWithMedia()` contract from the WhatsApp missing-mention gating path, with sender visibility filtering and bounded local-image retention preserved.

## Reproduction Assessment

Yes at source level: an unmentioned WhatsApp group image can enter the inbound message as `payload.media`, but the missing-mention history recorder and later `InboundHistory` mapping keep only text fields. No live WhatsApp repro or tests were run because this review is read-only.

## Solution Assessment

Yes: the narrow maintainable fix is to use the existing shared `HistoryEntry.media` and `recordWithMedia()` contract, not add a WhatsApp-only parallel media shape.
