## Summary

Keep open: current main and the latest release still preserve WhatsApp missing-mention group history as text-only, so a later mentioned turn can receive a media placeholder without usable historical image metadata. The report is actionable, but the repair needs maintainer security approval because it would retain unmentioned group media for later model context.

## What This Changes

The issue asks OpenClaw to preserve media metadata for unmentioned WhatsApp group messages stored as pending history before a later mentioned turn.

## Best Possible Solution

Approve a bounded WhatsApp history-media policy, then reuse HistoryEntry.media and recordWithMedia so filtered group history can carry local image metadata into later turns without a WhatsApp-only parallel shape.

## Reproduction Assessment

Yes at source level: current main records WhatsApp missing-mention group history as text-only and later builds InboundHistory without historical media. No live WhatsApp run was performed because this review is read-only.

## Solution Assessment

Yes, with a maintainer security gate: the narrow maintainable solution is to carry historical media through the existing bounded history-media contract instead of inventing a WhatsApp-only media shape.
