## Summary

Keep open: current `main` and `v2026.5.18` now cover the direct outbound media send path, but the Telegram reply media path still resolves successful media sends without an equivalent success log, and the linked closing PR remains open for this issue.

## What This Changes

The issue asks for Telegram outbound media sends to emit success log lines comparable to the existing `sendMessage ok` logs.

## Best Possible Solution

Reconcile the linked PR with current `main`: keep the shipped direct-path logging, add or confirm matching success logs for `deliverMediaReply()`, then close this issue with the release proof.

## Reproduction Assessment

Yes, at source level for the remaining reply-media path: `deliverMediaReply()` still sends and marks media delivery without a success log. The direct outbound `sendMessageTelegram()` path no longer reproduces because current `main` and `v2026.5.18` log `operation` and `deliveryKind` after media message-id resolution.

## Solution Assessment

Yes, the narrow maintainable fix is to log once after Telegram returns a media `message_id`, matching the direct-path helper and avoiding caption/media-location leakage. The open PR should be updated, merged, or closed as superseded rather than leaving the issue half-owned.
