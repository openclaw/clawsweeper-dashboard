## Summary

Keep open. The PR targets a real missing archive notification, but the patch is not sufficient because a plain transcript update still goes through default session-delta thresholds, so small terminal archive files can remain unindexed until a forced/full reindex.

## What This Changes

The PR adds an `archiveFileOnDisk` transcript-update emit after archive renames, adds archive-event unit coverage, and documents the memory/sessions fix in the changelog.

## Best Possible Solution

Add an archive-specific dirty/targeted-sync path that indexes the newly archived transcript regardless of normal append thresholds, keep the event emission, and land it with regression coverage that exercises memory sync under default thresholds.

## Reproduction Assessment

Yes, by source inspection: current main renames archive files without emitting `sessionTranscriptUpdate`, while memory-core's incremental session path is driven by that event bus. I did not run a live gateway/database reproduction because this review was read-only.

## Solution Assessment

No. Emitting the archived path is necessary, but by itself it still depends on `sync.sessions` thresholds, which leaves short terminal archives below the default 100000-byte/50-message thresholds without another event to trigger indexing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Force archive paths past delta thresholds:** `src/gateway/session-transcript-files.fs.ts:141`
  - body: This emit still flows through `scheduleSessionDirty` and `updateSessionDelta`, which only add a file to `sessionsDirtyFiles` after the configured byte/message thresholds are hit. A reset archive with fewer than the default 100000 bytes and 50 messages is a terminal file, so it records pending deltas under the new archived path but never receives another update to cross the threshold; those archives still require a forced/full reindex. Please make archive notifications force or directly mark a targeted session sync, and cover the below-threshold case at the manager level.
  - confidence: 0.88
