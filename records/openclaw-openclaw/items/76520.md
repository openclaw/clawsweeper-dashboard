## Summary

Keep this PR open, but do not merge as-is. Current main supports the reported archive-notification gap, but the proposed emit still goes through session-delta thresholds, so small immutable reset/deleted archives below defaults can remain unindexed; the `.bak` indexing claim also conflicts with current usage-counted archive rules.

## What This Changes

The PR emits a `sessionTranscriptUpdate` after `archiveFileOnDisk` renames a live transcript, adds archive-event unit coverage, and updates the changelog.

## Best Possible Solution

Make archive rotation produce an immediate targeted/forced memory sync for usage-counted reset/deleted archives, with regression coverage for archives below the default delta thresholds and corrected `.bak` wording or behavior.

## Reproduction Assessment

Yes, source-reproducible. Current main renames archive files without emitting, and the PR’s new emit would still not dirty a small immutable archive below the documented 100000-byte/50-message thresholds.

## Solution Assessment

No. Emission is the right signal direction, but by feeding archive paths into the regular append delta path it does not reliably index small reset/deleted archives; the fix should bypass or explicitly handle thresholds for archived session files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Force archive events past delta thresholds:** `src/gateway/session-transcript-files.fs.ts:140`
  - body: This emits the archived path into the regular session-delta listener. With the default 100000-byte/50-message thresholds, a freshly archived reset/deleted transcript below both limits is recorded as pending but never marked dirty; because the archive is immutable, no later write will push it over the threshold, so it still waits for a force reindex. Treat usage-counted archive notifications as a targeted/forced session sync or add archive-specific memory handling and tests.
  - confidence: 0.88
- **[P3] Remove `.bak` from the indexing claim:** `CHANGELOG.md:52`
  - body: The changelog says `.jsonl.bak.<iso>` archives are indexed on the next tick, but current session-file listing filters through `isUsageCountedSessionTranscriptFileName`, which explicitly excludes `.bak` archives. Either implement deliberate `.bak` indexing or narrow this wording/test expectation to reset/deleted archive notifications.
  - confidence: 0.84
