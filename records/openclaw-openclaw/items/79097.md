## Summary

Close as ClawHub/plugin scope: the request is a new optional verifier capability around subagent output, and current plugin/runtime surfaces are sufficient to build it outside OpenClaw core without adding arbitrary script execution to `sessions_spawn`.

## What This Changes

Request adding an optional `postCompletionHook` parameter to `sessions_spawn` that runs a verifier script after subagent completion and aborts or warns on failure.

## Best Possible Solution

Publish a verifier plugin or ClawHub tool wrapper that owns the verification policy, then bring back only a narrow plugin API proposal if that external implementation exposes a real missing seam.

## Reproduction Assessment

Not applicable: this is a feature request for a new `sessions_spawn` parameter, not a report of broken current behavior. Current source and docs show that parameter is absent today.

## Solution Assessment

Yes. Keeping core lean and using a ClawHub plugin is the narrower maintainable path because the current plugin runtime can run and inspect subagents and invoke bounded verifier commands without adding arbitrary script execution to the built-in tool.
