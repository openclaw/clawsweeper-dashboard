Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Make eager user appends idempotent:** `src/gateway/server-methods/chat.ts:866-871`
  - body: The eager path builds the persisted user message without the `chat.send` idempotency key, then appends it directly. If a client retries the same idempotency key after an ACK/error path or after a gateway restart, the transcript scan has no key to match and can append a duplicate user turn. Current main's user-turn recorder uses `${clientRunId}:user` with `idempotencyLookup: "scan"`; carry that contract into this path.
