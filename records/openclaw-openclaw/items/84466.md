## What This Changes

## Best Possible Solution

Keep this issue as the canonical memory-boundary decision, then land a narrow fix that excludes MEMORY.md from shared Discord channel bootstrap by default and leaves shared-channel memory tools explicit opt-in unless maintainers choose and document a different policy.

## Solution Assessment

Yes on the problem boundary, but the implementation still needs maintainer direction. The safest shape is channel-aware bootstrap filtering plus explicit opt-in for shared-channel memory tools, not broadening default QMD recall into channels.
