- `P1`: The report describes private long-term memory being retrieved in a shared Discord channel and current source shows plausible disclosure paths.
- `impact:security`: The behavior can expose sensitive MEMORY.md facts to other participants in a guild channel.

## Summary

Keep open: current main still permits normal Discord channel sessions to receive MEMORY.md in bootstrap context and to use built-in memory tools without the QMD direct-only scope guard, while the docs conflict on whether on-demand guild memory access is intended.

## What This Changes

The issue reports that Discord guild sessions can access long-term MEMORY.md facts despite documentation saying guild channels do not auto-load MEMORY.md.

## Best Possible Solution

Set one explicit security contract for long-term memory in shared channels, then enforce it consistently across bootstrap injection, memory_search, memory_get, QMD scope, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a normal Discord channel session key is not filtered out of MEMORY.md bootstrap files, and built-in memory_search/read paths can reach MEMORY.md without a chat-type scope guard.

## Solution Assessment

Unclear until maintainers choose the intended security contract. A safe fix should make the DM-only default explicit while preserving a deliberate opt-in path for guild memory recall if that behavior is still supported.
