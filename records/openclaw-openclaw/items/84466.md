## Summary

Keep open: the report is credible and current source still has two paths that can expose MEMORY.md in normal Discord channel sessions, while the docs conflict about whether channel memory access is intended.

## What This Changes

The issue reports that Discord guild channel sessions can retrieve MEMORY.md facts even though the Discord guide says long-term memory should load only in DM sessions by default.

## Best Possible Solution

Define one privacy contract for MEMORY.md in shared channels, then enforce it consistently across bootstrap injection, built-in memory_search/get, QMD scope, Discord docs, and regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: a Discord channel session key is a normal session, so MEMORY.md remains in bootstrap context and memory_search can query built-in memory without a chat-type guard. I did not run a live Discord bot in this read-only review.

## Solution Assessment

Unclear until maintainers pick the intended security/product contract. The narrow code fix should follow that decision and avoid leaving built-in memory, QMD memory, and docs with different channel-scope behavior.
