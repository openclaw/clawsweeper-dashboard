## What This Changes

## Best Possible Solution

Define one explicit MEMORY.md contract for shared channels, then align built-in memory_search/memory_get, QMD scope, Discord documentation, and regression tests to enforce that contract consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Discord guild routes use channel session keys, memory_search forwards that key, and built-in memory search queries MEMORY.md sources without a chat-type scope check; I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the intended security contract. A safe fix should make built-in and QMD memory behavior match the chosen Discord docs, with an explicit opt-in path if guild memory recall remains supported.
