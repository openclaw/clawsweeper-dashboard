## Summary

Keep open: this PR is paired with an open bug report and addresses a real source-level hook session path, but it should not merge as-is because it changes default hook isolation behavior and still lacks after-fix real webhook proof.

## What This Changes

The PR changes isolated cron agent session selection so hook/webhook-sourced isolated runs can reuse stable sessions, with a regression test and changelog entry.

## Best Possible Solution

Prefer an explicit persistent hook-session mode, such as the path in https://github.com/openclaw/openclaw/pull/75918 or an equivalent scoped patch, with default isolation preserved and redacted runtime webhook reuse proof supplied before merge.

## Reproduction Assessment

Yes by source inspection: an approved /hooks/agent request becomes an isolated cron agent turn, and current main passes forceNew: true before resolveCronSession can reuse the stored session id. I did not run a live webhook scenario in this read-only review.

## Solution Assessment

No for this exact patch: reusing sessions based only on hook provenance changes default isolated hook behavior. The narrower maintainable solution is an explicit persistent-session opt-in that preserves existing session-key policy gates.
