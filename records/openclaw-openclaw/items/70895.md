The PR changes isolated cron session selection so hook-sourced isolated agent runs pass `forceNew: false`, adds a regression test, and adds an unreleased changelog entry.

## Best Possible Solution

## Solution Assessment

Unclear as the final product shape. The patch is narrow for automatic hook reuse, but https://github.com/openclaw/openclaw/pull/75918 is the safer opt-in design if maintainers want persistent hook sessions to require an explicit mode.
