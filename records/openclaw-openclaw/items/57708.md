## Summary

Keep open. Current main still sends ordinary Slack markdown through the mrkdwn/text path, while outbound rich_text auto-conversion is not implemented and the issue body leaves defaulting, config, streaming, chunking, and explicit-block precedence as product decisions.

## What This Changes

The issue proposes automatically converting Slack agent markdown output into Slack rich_text blocks instead of requiring explicit Block Kit JSON.

## Best Possible Solution

Keep this as a focused Slack rich-output RFC, decide default/config, streaming/final delivery, chunking, fallback text, and explicit-block precedence, then implement a Slack-owned MarkdownIR-to-rich_text renderer with focused tests and real Slack proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature RFC. Source inspection gives a high-confidence verification path: ordinary Slack text still renders through mrkdwn/text chunks and no outbound rich_text converter exists on current main.

## Solution Assessment

Unclear until maintainers answer the RFC choices. A Slack-local renderer appears to be the likely boundary because MarkdownIR is already exported through the plugin SDK, but default/config, streaming, chunking, and precedence semantics need a product decision first.
