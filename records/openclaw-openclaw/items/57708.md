## Summary

Keep open. Current main still routes ordinary Slack agent text through the Slack mrkdwn/text send path, while `rich_text` only appears on inbound Slack parsing and explicit Block Kit paths; the requested MarkdownIR-to-Slack-rich_text conversion remains an unresolved Slack feature/product decision.

## What This Changes

Add automatic MarkdownIR-to-Slack `rich_text` block conversion for normal Slack agent messages instead of requiring explicit Block Kit JSON.

## Best Possible Solution

Keep this as a focused Slack follow-up under the richer-output umbrella, then implement a Slack-owned MarkdownIR-to-rich_text renderer after maintainers settle defaulting, streaming/final delivery, chunking, fallback text, and explicit-block precedence.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature RFC. Source inspection gives a high-confidence verification path: ordinary Slack text still renders through mrkdwn/text chunks and no outbound rich_text converter exists on current main.

## Solution Assessment

Unclear until maintainers answer the RFC choices. A Slack-local renderer looks like the likely boundary because MarkdownIR is already exported through the plugin SDK, but default/config, streaming, chunking, and precedence semantics should be decided before implementation.
