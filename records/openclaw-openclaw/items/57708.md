## Summary

Keep open. Current main still sends ordinary Slack agent markdown through the mrkdwn/text path, while Slack rich_text handling exists only for inbound parsing and explicit Block Kit-style paths; the requested automatic MarkdownIR-to-rich_text conversion remains an unresolved Slack feature and product decision.

## What This Changes

Automatically convert ordinary Slack agent markdown output into Slack rich_text Block Kit blocks using the existing MarkdownIR parser.

## Best Possible Solution

Keep this as a focused Slack rich-output follow-up, decide default/config, streaming/final delivery, chunking, fallback text, and explicit-block precedence, then implement a Slack-owned MarkdownIR-to-rich_text renderer with focused tests and real Slack proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature RFC. Source inspection gives a high-confidence verification path: ordinary Slack text still renders through mrkdwn/text chunks and no outbound rich_text converter exists on current main.

## Solution Assessment

Unclear until maintainers answer the RFC choices. A Slack-local renderer appears to be the likely boundary because MarkdownIR is already exported through the plugin SDK, but default/config, streaming, chunking, and precedence semantics need a product decision first.
