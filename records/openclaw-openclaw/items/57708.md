## Summary

Keep open. Current main still sends ordinary Slack agent text through the mrkdwn renderer and only uses Block Kit when explicit blocks, shared presentation blocks, or interactive blocks are present; the requested MarkdownIR-to-Slack rich_text auto-conversion is still a feature/product decision, not a cleanup-close candidate.

## What This Changes

The issue requests automatic conversion of Slack agent Markdown output into Slack rich_text Block Kit blocks using the existing MarkdownIR parser.

## Best Possible Solution

Keep this as a focused Slack follow-up under the Block Kit umbrella, then implement a Slack-owned MarkdownIR-to-rich_text renderer after maintainers decide defaulting, streaming/final delivery, chunking, fallback text, and explicit-block precedence.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature RFC. Current-main source inspection gives a high-confidence verification path: ordinary Slack text still renders through mrkdwn and no outbound rich_text converter exists.

## Solution Assessment

Unclear until maintainers answer the RFC choices. A Slack-local renderer looks like the right boundary because MarkdownIR is already exported through the plugin SDK, but opt-in/default behavior and streaming semantics should be settled before implementation.
