## Summary

Keep open. Current main still renders ordinary Slack agent text through mrkdwn and only sends Block Kit when explicit blocks or shared presentation blocks are present, while this RFC asks for a narrower MarkdownIR-to-rich_text auto-conversion that still needs maintainer product direction.

## What This Changes

Add Slack-owned automatic conversion of agent Markdown output into Slack rich_text blocks instead of the current mrkdwn text rendering.

## Best Possible Solution

Keep this as a focused Slack follow-up under #12602: implement a Slack-local MarkdownIR-to-rich_text renderer only after deciding opt-in/default behavior, streaming/final-only handling, chunking, fallback text, and explicit-block precedence.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature RFC. The current-main verification path is high-confidence by source inspection: ordinary Slack text still goes through mrkdwn rendering and no outbound rich_text converter exists.

## Solution Assessment

Unclear until maintainers answer the RFC choices. A Slack-owned renderer looks like the right boundary because MarkdownIR is already exposed through the plugin SDK, but default enablement and streaming behavior should be decided before implementation.
