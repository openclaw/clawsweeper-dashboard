## Summary

Keep open. Current main and the latest release still send ordinary Slack markdown through the mrkdwn/text path, while outbound MarkdownIR-to-rich_text conversion is not implemented and the RFC leaves defaulting, config, streaming, chunking, and explicit-block precedence as maintainer decisions.

## What This Changes

Request to automatically convert ordinary Slack markdown replies into Slack rich_text Block Kit blocks using the existing MarkdownIR parser.

## Best Possible Solution

Keep this as a focused Slack rich-output RFC, decide default/config, streaming/final delivery, chunking, fallback text, and explicit-block precedence, then implement a Slack-owned MarkdownIR-to-rich_text renderer with focused tests and redacted real Slack proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature RFC. Source inspection gives a high-confidence verification path: ordinary Slack text still renders through mrkdwn/text chunks and no outbound rich_text converter exists on current main or v2026.5.19.

## Solution Assessment

Unclear until maintainers answer the RFC choices. A Slack-local renderer appears to be the likely boundary because MarkdownIR is already exported through the plugin SDK, but default/config, streaming, chunking, and precedence semantics need a product decision first.
