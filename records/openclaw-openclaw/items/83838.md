The branch adds a macOS-only pending assistant reply state in `OpenClawChatView` to keep scrolling to the bottom during message, run-count, and streaming-text changes while a reply is active.

## Best Possible Solution

Fix the macOS false-unpinning case without overriding deliberate user scrolls, then add redacted real macOS proof that covers staying pinned during streaming and manually scrolling away during an active reply.

## Reproduction Assessment

No live macOS reproduction was run. Source inspection gives a high-confidence path for the introduced defect because `hasPendingAssistantReply` bypasses `isPinnedToBottom` until `pendingRunCount` returns to zero.

## Solution Assessment

No. The maintainable fix should distinguish macOS false unpinning from an explicit user scroll and preserve the existing manual-release behavior.
