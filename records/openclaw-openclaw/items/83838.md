## Summary

This PR should stay open but is not merge-ready: it targets the right macOS SwiftUI chat-scroll surface, but it still lacks after-fix real macOS proof and introduces a pending-reply scroll flag that can leak across session/loading transitions.

## What This Changes

The PR adds macOS-only `hasPendingAssistantReply` state in `OpenClawChatView` to force bottom scrolling while an assistant reply is pending.

## Best Possible Solution

Keep a narrow macOS ChatView fix that follows the originating assistant reply, resets scroll state on session/loading/run clears, preserves deliberate scroll-away unless maintainers choose otherwise, and includes redacted macOS proof.

## Reproduction Assessment

No high-confidence runtime reproduction. Source shows current main only auto-scrolls pending/streaming updates while `isPinnedToBottom` is true, and related reports describe the symptom, but no before/after macOS behavior proof is present.

## Solution Assessment

No. The PR points at the right surface, but the current patch needs state-lifecycle cleanup, redacted real macOS proof, and a maintainer UX choice around manual scroll-away before it is the best fix.
