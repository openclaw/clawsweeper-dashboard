## Summary

Keep open. The bug is real and the PR direction matches the recently merged assistant-delta bridge, and the PR body supplies real after-fix logs; however the diff emits raw CLI tool args/results onto the agent-event bus while the embedded path redacts those payloads, so it needs a sanitizer repair before merge.

## What This Changes

The PR parses claude-cli stream-json tool_use/tool_result records, emits CLI tool events on the agent-event bus, bridges tool start/update events to channel previews, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the CLI tool-preview bridge after redacting CLI tool args/results with the same sanitizers used by the embedded runtime and adding regression coverage for that privacy contract.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: current main's claude-cli stream parser only accepts text_delta records, while the PR body also includes live 2026.5.7 logs showing stock CLI-backed Telegram turns emitted zero tool events. I did not run the live Telegram/Bedrock setup in this read-only pass.

## Solution Assessment

No, not as currently written. The parser and bridge shape is the right narrow fix, but the event emission must reuse the existing tool payload sanitizers before publishing CLI tool args/results to shared listeners.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Sanitize CLI tool payloads before publishing:** `src/agents/cli-runner/execute.ts:482-489`
  - body: This publishes the raw claude-cli `args` and `result` values to the global agent-event bus. The embedded tool path redacts these payloads with `sanitizeToolArgs`/`sanitizeToolResult` before emission, and those helpers explicitly cover API keys, Authorization headers, and shell commands containing secrets. A CLI Bash/tool call with a token in its args or output would now be visible to every agent-event listener and may be rendered in preview/control surfaces. Redact both live-session and JSONL-parser emissions before emitting.
