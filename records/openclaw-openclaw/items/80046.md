## Summary

Keep open: this is a useful, well-proven PR, but the parser can still deliver empty tool args in the normal streamed-tool path, which blocks merge until fixed.

## What This Changes

The PR adds Claude CLI stream-json tool_use/tool_result parsing, sanitized CLI tool event emission, and CLI lifecycle bridges that forward tool starts into channel preview callbacks.

## Best Possible Solution

Land the CLI tool-event bridge after fixing streamed-args accumulation, with thinking-delta support handled as an explicit follow-up unless maintainers choose to expand this PR.

## Reproduction Assessment

Yes, at source level. The PR head emits the fallback start with empty args at `content_block_stop`, and the upstream streaming contract says the args are carried in partial JSON chunks that should be accumulated until that stop event; I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. The bridge direction is maintainable, but the parser should accumulate `input_json_delta.partial_json` and emit parsed args at stop, keeping the assistant snapshot as a backup rather than relying on it after an empty-args emission.
