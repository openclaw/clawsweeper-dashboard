## Summary

Keep open. Current main does not have the Google Meet node-invoke policy, but this PR should not merge as-is because it also makes `browserProfile` config-owned and still lacks paired macOS Chrome-node proof.

## What This Changes

The branch registers a dangerous Google Meet `googlemeet.chrome` node-invoke policy, rewrites start-request fields from plugin config, and updates focused docs/tests for Chrome-node helper commands.

## Best Possible Solution

Keep the policy focused on config-owned executable audio helper commands, preserve `browserProfile`, document the intended direct-invoke compatibility change, and merge only after paired macOS node proof.

## Reproduction Assessment

Yes. On the PR head, a direct `googlemeet.chrome` start request with caller-supplied `browserProfile` reaches `buildGoogleMeetChromeStartParams`, where that value is replaced before node forwarding.

## Solution Assessment

No, not as currently scoped. Config ownership is appropriate for executable helper commands, but replacing `browserProfile` is unrelated unless maintainers explicitly choose a broader compatibility break.
