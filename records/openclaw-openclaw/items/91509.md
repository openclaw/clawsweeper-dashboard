## Summary

Keep open: the Google Meet hardening is still useful and not already on current main, but the PR is not merge-ready because it also makes `browserProfile` config-owned and the supplied proof does not show a real paired macOS Chrome-node run.

## Best Possible Solution

## Solution Assessment

No. The security fix should own the executable helper arrays, but replacing the Chrome profile is broader than the stated hardening boundary unless maintainers explicitly approve that compatibility break.
