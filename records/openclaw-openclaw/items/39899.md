The PR exports a surrogate-encoding error detector, formats those provider errors with recovery copy, and resets the active auto-reply session when that error is thrown before a reply.

## Best Possible Solution

Land a current-main repair that detects surrogate JSON request errors, returns friendly recovery copy in every fallback path, resets the affected session through the current reset flow, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

## Solution Assessment

Mostly yes. Reset-on-detection matches the existing session-corruption recovery pattern, but the patch should also handle the missing-reset-state fallback copy, add regression tests and changelog, and be rebased onto the current runner shape.
