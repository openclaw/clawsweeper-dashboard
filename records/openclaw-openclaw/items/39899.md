## Summary

Keep open: current main still has a plausible surrogate JSON failure path, but this PR is not merge-ready because the reset path bypasses the current session-reset helper, the runner test mock is stale, and external real behavior proof is missing.

## What This Changes

The PR exports a surrogate-encoding error detector, adds friendly invalid-Unicode copy, and resets the active session transcript/store entry when the reply runner catches a surrogate JSON error.

## Best Possible Solution

Refresh the fix on current main so surrogate-bearing request fields are repaired where practical and provider surrogate JSON failures rotate through resetReplyRunSession with friendly fallback copy and focused regression coverage.

## Reproduction Assessment

Yes, source-level: current OpenAI replay sanitizes normal text fields but forwards string tool-call arguments without sanitizeTransportPayloadText, so a stored lone surrogate can reach the provider payload. I did not run a live provider request in this read-only review.

## Solution Assessment

No, not as-is. The detector and recovery direction are useful, but the reset should route through resetReplyRunSession, the reset-unavailable fallback needs friendly copy, tests need updating, and contributor real behavior proof is missing.
