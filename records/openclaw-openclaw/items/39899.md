## Summary

Keep open: this is a valid narrow bug-fix PR for a source-reproducible agent session loop, but the branch needs current-main repair before merge for one fallback-copy gap, regression coverage, and the required changelog entry.

## What This Changes

The PR exports a surrogate-encoding error detector, formats that provider error with recovery copy, and resets the active agent session when the error is thrown before a reply.

## Best Possible Solution

Land a current-main repair that detects surrogate JSON request errors, returns friendly recovery copy on every fallback path, clears the corrupted session without replaying invalid bytes, and covers the structured replay cases that can still carry invalid Unicode.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks a surrogate classifier/reset path, and a focused unit can inject a lone surrogate into replayed OpenAI Responses function-call arguments or raw provider error text without a live API call.

## Solution Assessment

Mostly yes. Reset-on-detection matches existing corruption recovery behavior, but the PR should handle the missing-reset-state fallback copy, add regression tests and changelog, and rebase around the current reset/session helper shape.
