## Summary

Keep this PR open: the production-loop bug is still source-reproducible on current main, but the proposed branch is stale against the current runner reset contract and needs a small repair before merge.

## What This Changes

The PR exports a surrogate-encoding error detector, adds friendly assistant error copy for surrogate JSON request failures, and resets the session transcript/store when the reply runner sees that error.

## Best Possible Solution

Land a current-main repair that detects surrogate JSON request errors, formats friendly recovery copy, resets through `resetReplyRunSession`, handles reset-unavailable fallback copy, and adds focused regression tests plus an active changelog entry.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks surrogate-specific formatter and runner recovery, and OpenAI Responses replay can still forward stored string tool-call arguments with lone surrogates into provider request payloads.

## Solution Assessment

No, not as-is. The detector and recovery direction are right, but the branch should be rebased onto the current reset-helper contract, update tests, cover fallback copy when reset state is missing, and include the required changelog entry.
