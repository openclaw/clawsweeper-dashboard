## Summary

Keep open. Current main and the latest shipped tag still omit generated compaction summary text from the public `after_compaction` plugin hook payload, and the remaining work is a real plugin SDK feature with privacy/product semantics rather than cleanup closure.

## What This Changes

Expose generated compaction summary content, and possibly summary token metadata, through the public `after_compaction` plugin hook payload.

## Best Possible Solution

Define a permissioned or opt-in optional summary-content contract, populate it only when a real generated summary is available, and document absence, size, and token-count semantics before implementation.

## Reproduction Assessment

Not applicable as a runtime bug, but the source-level gap is high-confidence: current main and `v2026.5.18` generate or track summary metadata while the public `after_compaction` event carries only hook metadata.

## Solution Assessment

Partially: an optional field is the right additive direction, but the best solution also needs explicit opt-in or permission semantics plus documented unavailable-summary behavior before exposing transcript-derived content.
