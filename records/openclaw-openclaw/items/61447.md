## Summary

Keep open. Current main still omits generated compaction summary text from the public `after_compaction` plugin hook payload, the newer matching report was closed as a duplicate in favor of this issue, and exposing transcript-derived summaries needs a maintainer product/security decision rather than cleanup closure.

## What This Changes

The issue requests adding generated compaction summary text, and possibly token metadata, to the public `after_compaction` plugin hook payload.

## Best Possible Solution

Define a permissioned or explicit opt-in `after_compaction` summary-content contract, then populate it only from compaction paths that already have real generated summary text and document absence and size semantics.

## Reproduction Assessment

Not applicable as a runtime bug, but the source-level gap is high-confidence: current main generates or stores summaries in compaction paths while the public `after_compaction` event only carries metadata.

## Solution Assessment

Partially: adding an optional field is the right contract direction, but the best solution also needs explicit opt-in or permission semantics and documented unavailable-summary behavior before exposing transcript-derived content.
