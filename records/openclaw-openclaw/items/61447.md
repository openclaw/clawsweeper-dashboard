## Summary

Keep open. Current main and the latest release still omit generated compaction summary text from the public `after_compaction` plugin hook payload, even though direct and queued compaction paths already carry summary values nearby.

## What This Changes

The issue requests adding generated compaction summary text to the `after_compaction` plugin hook payload.

## Best Possible Solution

Add a documented optional generated-summary field to the public plugin hook contract, populate it only from compaction paths that already have a real summary string, and update focused tests/docs/changelog without changing compaction algorithms or transcript persistence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: generated summary values exist near compaction completion, while the typed public hook event and all inspected `after_compaction` emitters omit summary text.

## Solution Assessment

Yes. A backward-compatible optional hook field is the narrow maintainable solution; requiring plugins to parse inline summary tags or changing JSONL persistence would keep the fragile workaround or broaden the feature unnecessarily.
