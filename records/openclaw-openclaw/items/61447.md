## Summary

Keep open. Current main still omits generated compaction summary text from the typed after_compaction plugin hook payload, even though direct, queued, overflow, timeout, and wired compaction paths already have summary strings before or while emitting the hook; the earlier exact-feature PR was closed unmerged rather than shipped.

## What This Changes

The issue requests adding generated compaction summary text, and optionally summary token metadata, to the after_compaction plugin hook payload.

## Best Possible Solution

Add a backward-compatible optional generated-summary field to the public plugin hook contract, populate it only where the compaction result already exposes a string summary, leave unsupported native Codex paths honest, and update focused tests plus plugin docs without changing compaction algorithms or transcript persistence.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current hook payload construction omits summary text while nearby compaction result objects and tests already contain summary strings before after_compaction is emitted.

## Solution Assessment

Yes. The narrow maintainable fix is an optional typed hook payload field; requiring plugins to parse inline summary tags or session JSONL would preserve the current fragile workaround rather than fixing the contract.
