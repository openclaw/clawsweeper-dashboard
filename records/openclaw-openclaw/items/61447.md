## Summary

Keep open. Current main and the latest release still omit generated compaction summary text from the public `after_compaction` plugin hook payload, and the newer matching report at https://github.com/openclaw/openclaw/issues/83936 was closed in favor of this issue. The remaining work is a real SDK/API feature, but it needs maintainer product and security decisions before implementation.

## What This Changes

Requests adding generated compaction summary content, and possibly summary token metadata, to the public `after_compaction` plugin hook payload.

## Best Possible Solution

Define an opt-in, permissioned `after_compaction` summary-content contract, forward real generated summaries only from paths that have them, and document absence, bounds, and token semantics.

## Reproduction Assessment

Not applicable as a runtime bug, but the source-level gap is high-confidence: current main and `v2026.5.19` generate or store summaries while the public `after_compaction` payload only carries metadata.

## Solution Assessment

Partially: adding an optional payload field is the right direction, but the best solution should gate population behind explicit opt-in or permission semantics and document unavailable-summary behavior.
