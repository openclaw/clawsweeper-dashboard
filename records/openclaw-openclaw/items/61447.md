## Summary

Keep open. Current main still omits generated compaction summary text from the typed `after_compaction` plugin hook payload and from the direct, queued, runner-owned, subscribe-time, and Codex harness hook emissions; the prior Codex review remains accurate against the current checkout.

## What This Changes

Expose generated compaction summary text on the `after_compaction` plugin hook payload so plugins can audit compaction quality without parsing later transcript context.

## Best Possible Solution

Add a documented optional generated-summary field to the public plugin hook contract, populate it only from paths that already have a real summary string, and leave compaction algorithms and transcript persistence unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current hook payload construction omits summary text while nearby compaction result objects and tests already contain summary strings before `after_compaction` is emitted.

## Solution Assessment

Yes. The narrow maintainable solution is a backward-compatible optional hook payload field; requiring plugins to parse inline summary tags or later JSONL context would keep the fragile workaround in place.
