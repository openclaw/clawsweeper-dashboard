## Summary

Keep open. Current main has generic Markdown/code-block rendering and a Control UI copy button, but it still lacks the requested shell/bash command-block presentation contract and copy-safety behavior across chat channels.

## What This Changes

The issue requests a channel-aware convention for rendering shell/bash command blocks so operators can copy chat-delivered commands safely and consistently.

## Best Possible Solution

Define a portable command-block presentation contract or Markdown extension point, then implement channel-specific shell/bash rendering and regression fixtures that assert exact copy text and safe plain-text fallbacks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence baseline: current main provides generic fenced-code rendering, chunking, and Control UI copy support, not shell/bash command-block semantics across chat channels.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The maintainable direction is a shared presentation or Markdown contract with channel-specific renderers, not isolated formatter shims that would drift across transports.
