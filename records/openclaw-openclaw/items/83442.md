Keep open. Current main has generic Markdown parsing, fence-aware chunking, and per-channel code-block rendering, but it still lacks the requested first-class shell/bash command-block contract or cross-channel copy-safety convention.

## What This Changes

Request asks for first-class, channel-aware shell/bash command-block rendering so operators can copy chat-delivered commands safely across adapters.

## Best Possible Solution

Define a portable command-block presentation contract or Markdown IR metadata that preserves shell fence language and exact copy text, then add channel-specific rendering/fallback fixtures and docs for the supported chat adapters.
