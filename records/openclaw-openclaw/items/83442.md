## Summary

Keep open. Current main has generic Markdown IR, code-block rendering, and fence-aware chunking, but it still lacks the requested shell/bash command-block presentation contract or copy-safety convention across chat channels.

## What This Changes

Request adding a first-class, channel-aware presentation convention for shell/bash fenced command blocks in chat replies so operators can copy them safely.

## Best Possible Solution

Define a portable command-block presentation contract or Markdown extension point, then implement channel-specific shell/bash rendering and regression fixtures with exact copy text and safe plain-text fallbacks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence baseline: current main provides generic fenced-code rendering and chunking, not shell/bash command-block semantics.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The maintainable direction is a shared presentation or Markdown contract with channel-specific renderers, not isolated formatter shims that would drift across transports.
