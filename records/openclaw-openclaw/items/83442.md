## Summary

Keep open. Current main and v2026.6.8 have generic Markdown code-block rendering and Control UI copy chrome, but no first-class shell/bash exact-copy contract or cross-channel fallback semantics.

## What This Changes

The issue requests first-class, channel-aware rendering or fallback handling for shell/bash command blocks in chat outputs so operators can copy commands safely.

## Best Possible Solution

Define one portable command/code-block presentation contract with exact text, optional language, and fallback semantics, then adapt the major channel renderers and fixtures from that shared seam.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new cross-channel presentation convention rather than reporting a broken current contract. Source inspection verifies that current main only has generic code-block rendering/copy and no portable shell command-block contract.

## Solution Assessment

Unclear until maintainers choose the shared presentation boundary. The maintainable direction is a typed portable contract with per-channel renderers and fallbacks, not isolated formatter heuristics that infer commands from `bash` fences.
