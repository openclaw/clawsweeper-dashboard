## Summary

Keep open. Current main and v2026.6.8 have generic Markdown IR, generic channel code-block rendering, and Control UI code-copy chrome, but still no first-class shell/bash command-block contract or exact-copy fallback across chat channels.

## What This Changes

The issue requests a channel-aware convention for rendering shell/bash command blocks in chat outputs so operators can copy commands safely.

## Best Possible Solution

Define one portable command-block contract with exact command text, optional language, and fallback semantics, then adapt the major channel renderers and fixtures from that shared seam instead of inferring shell commands from raw Markdown per channel.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new cross-channel presentation convention rather than reporting a broken current contract. Source inspection does establish the current baseline: generic fenced-code rendering exists, but no shell/bash command-block contract exists.

## Solution Assessment

Unclear until maintainers choose the shared presentation boundary. The maintainable direction is a typed portable contract with per-channel renderers/fallbacks, not isolated formatter heuristics that infer commands from `bash` fences.
