## Summary

Keep open. Current main has generic Markdown parsing, fence-aware chunking, and per-channel code-block rendering, but it does not provide the requested first-class shell/bash command-block contract or cross-channel copy-safety convention.

## What This Changes

The issue asks OpenClaw to render shell/bash command blocks in chat outputs in a channel-aware way so operators can copy commands safely and consistently.

## Best Possible Solution

Define a portable command-block presentation contract or Markdown extension point, then implement channel-specific shell/bash rendering and regression fixtures for the supported chat adapters with exact copy text and safe plain-text fallbacks.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection does show the current baseline is generic Markdown/code-block handling without shell-specific command-block semantics.

## Solution Assessment

Unclear until maintainers choose the product/API shape. The existing Markdown IR and chunkers are good groundwork, but the requested safe-copy behavior likely needs a portable contract rather than one-off channel shims.
