## Summary

Keep open. Current main and v2026.6.10 still provide generic Markdown code-block rendering and Control UI copy chrome, but not a first-class shell/bash command-block contract with exact-copy and cross-channel fallback semantics.

## What This Changes

Request a first-class, channel-aware way to render shell/bash command blocks in chat outputs so operators can copy commands safely across chat surfaces.

## Best Possible Solution

Define a typed portable command/code-block presentation contract with exact text, language, and fallback semantics, then adapt major channel renderers and fixtures without inferring executable commands from arbitrary Markdown.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new cross-channel presentation convention rather than reporting a broken existing contract. Source inspection confirms current main only has generic code-block rendering/copy behavior, not a portable shell command-block contract.

## Solution Assessment

Unclear until maintainers choose the shared presentation boundary. The maintainable direction is a typed portable contract with per-channel renderers and fallbacks, not isolated formatter heuristics that infer commands from bash fences.
