## Summary

Keep open. Current main and v2026.6.6 have generic Markdown IR, code-block rendering, and fence-aware chunking, but no first-class shell/bash command-block contract or exact-copy fallback; the adjacent Control UI approve/run issue is related but does not cover chat-channel copy-safe rendering.

## What This Changes

Add a reliable, channel-aware convention for rendering shell/bash command blocks in chat outputs so operators can copy commands safely and consistently.

## Best Possible Solution

Define a portable command-block presentation or Markdown metadata contract, then implement channel-specific renderers/fallbacks and fixtures for Telegram, Discord, Slack, Nextcloud Talk, and other messaging plugins.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection establishes the current baseline: generic fenced-code rendering and chunking exist, but no shell/bash command-block presentation contract exists.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The maintainable direction is a shared presentation or Markdown contract with per-channel renderers, not isolated formatter shims that would drift across transports.
