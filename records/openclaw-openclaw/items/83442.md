## Summary

Keep open. Current main has generic Markdown/code-block rendering and Control UI copy buttons, but no first-class shell/bash command-block contract or exact-copy fallback across chat channels; related WebChat and Zalo formatting work is adjacent rather than canonical.

## What This Changes

The issue requests a channel-aware presentation contract for shell/bash snippets so operators can copy chat-delivered commands safely across supported channels.

## Best Possible Solution

Define one portable command-block contract with exact command text, optional language, and fallback semantics, then adapt major channel renderers and fixtures from that shared seam.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new cross-channel presentation convention rather than reporting a broken current contract. Source inspection establishes the current baseline: generic fenced-code rendering exists, but no shell/bash command-block contract exists.

## Solution Assessment

Unclear until maintainers choose the shared presentation boundary. The maintainable direction is a typed portable command-block contract with per-channel renderers and fallbacks, not isolated formatter heuristics that infer shell commands from raw Markdown fences.
