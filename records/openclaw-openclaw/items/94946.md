## What This Changes

The branch changes `src/auto-reply/reply/dispatch-from-config.ts` so `isReasoning` block and final reply payloads are not suppressed when `deliveryChannel` is `telegram`.

## Best Possible Solution

Update this branch or use a canonical replacement that handles Telegram `isReasoning` payloads before outbound normalization, strips the marker before durable send, excludes reasoning from shared TTS, adds focused block/final/TTS regression coverage, and includes redacted live Telegram proof.

## Solution Assessment
