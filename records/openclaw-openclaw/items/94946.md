## Summary

## Best Possible Solution

Land a Telegram-owned durable reasoning fix that preserves or converts reasoning before shared outbound normalization, keeps reasoning out of shared TTS, preserves non-Telegram suppression, adds focused block/final tests, and captures redacted Telegram-visible proof before merge.

## Reproduction Assessment

Yes, source-level: the linked production report has concrete `/reasoning on` steps, and current main suppresses `isReasoning` block/final payloads before Telegram can deliver them. I did not run a live Telegram session in this read-only review.

## Solution Assessment

No as written: bypassing the shared dispatch guards is only a partial fix because Telegram still normalizes via the shared outbound planner that drops `isReasoning` before lane splitting. The better fix preserves or converts reasoning inside the Telegram path before normalization and keeps it out of TTS.
