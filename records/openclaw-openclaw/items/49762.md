## Summary

Keep open: the feature direction is plausible, but the branch still has source-reproducible footer format/mode regressions and lacks after-fix real behavior proof from an actual reply.

## What This Changes

The PR adds a stopwatch response-duration value to per-reply usage footers and reuses the reply-run elapsed time for diagnostic usage events.

## Best Possible Solution

Add duration only to the product-approved usage-footer mode, preserve the existing cost separator, cover `tokens` versus `full` with focused tests, and require redacted real behavior proof from an actual reply before merge.

## Reproduction Assessment

Yes. Source inspection of the PR diff reproduces the blockers: the cost separator changes when duration is absent, and duration is passed before the mode-specific `full` branch despite the tokens-only docs.

## Solution Assessment

No. The current patch is not the best mergeable shape because it broadens `/usage tokens` and changes existing formatting; the safer path is to preserve compatibility and gate duration behind the intended footer mode or an explicit new mode.
