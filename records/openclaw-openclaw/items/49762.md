## Summary

Keep open. Current main still lacks a visible per-reply response-duration footer, but this PR is not mergeable as-is because it changes existing footer formatting, applies duration to the tokens-only mode, samples timing after delivery drains, and lacks after-fix real behavior proof.

## What This Changes

The PR adds an optional `durationMs` parameter to response usage footer formatting and passes elapsed reply-run time into the visible per-reply usage footer.

## Best Possible Solution

Add duration in the current `agent-runner-usage-line.ts` path with explicit footer-mode and timing semantics, preserving existing text contracts and adding focused regression coverage.

## Reproduction Assessment

Yes for the PR blockers: source inspection of the PR branch reproduces the separator regression, the unconditional `/usage tokens` metadata change, and the post-delivery timing sample. The requested feature itself still lacks real after-fix behavior proof from the contributor.

## Solution Assessment

No. The current patch is not the best mergeable shape; it should be rebased to the current usage-line helper, preserve existing separators, gate or define the new duration mode, and capture the intended timing boundary.

  - confidence: 0.8
