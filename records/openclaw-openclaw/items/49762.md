## What This Changes

The PR adds an optional response-duration stopwatch to per-reply usage footers and reuses elapsed reply-run time for model usage diagnostics.

## Best Possible Solution

Rebase onto the current usage-line helper, preserve existing cache/cost separators, add duration only to the approved footer mode and timing boundary, then cover it with focused tests and real behavior proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current `origin/main` reproduces the PR-introduced blockers: omitted duration changes cost formatting, `/usage tokens` receives duration, and the sampled timing sits after delivery drains.

## Solution Assessment

No. The current patch is not the best mergeable shape; the safer path is a product-approved footer mode/timing decision that preserves existing text contracts and adds targeted regression coverage.
