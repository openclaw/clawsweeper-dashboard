## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the existing cost separator:** `src/auto-reply/reply/agent-runner-utils.ts:139`
  - body: When `durationMs` is omitted, `parts.join(" / ")` changes the existing ` · est ...` cost suffix to `/ est ...`, contradicting the compatibility claim and risking callers or snapshots that match the footer text. Keep the cost suffix separator stable when adding the optional duration segment.
  - confidence: 0.91
- **[P2] Keep duration out of tokens-only footers:** `src/auto-reply/reply/agent-runner.ts:598`
  - body: Passing `durationMs` unconditionally into `formatResponseUsageLine()` makes `/usage tokens` show stopwatch metadata even though the docs define that mode as tokens-only. Gate the new segment to the approved mode, or update the product contract deliberately.
  - confidence: 0.87
- **[P2] Measure response duration before delivery drains:** `src/auto-reply/reply/agent-runner.ts:540`
  - body: The displayed duration is sampled after block reply flushing and pending tool delivery drains, so slow delivery can inflate a value presented as response duration. Capture the intended model/tool boundary earlier, or label the field as end-to-end wall time.
  - confidence: 0.78
