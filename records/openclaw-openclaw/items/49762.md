## Summary

Keep open: the feature is not on current main, and this PR still needs changes because it alters the existing cost separator, adds stopwatch metadata to the lean `tokens` footer path, and has no after-fix real behavior proof.

## What This Changes

The branch adds an optional response duration to the per-reply usage footer and passes a `Date.now() - runStartedAt` value from the reply runner into the footer formatter.

## Best Possible Solution

Add duration only to the product-approved footer mode, preserve the existing cost separator, add focused regression coverage for `tokens` versus `full`, and provide redacted real behavior proof from an actual reply.

## Reproduction Assessment

Yes. Source inspection reproduces the review issues: the diff joins the cost segment with `/` and passes `durationMs` before the mode-specific `full` branch, while current docs say `/usage tokens` is token-only.

## Solution Assessment

No. The feature direction is plausible, but the proposed patch should preserve existing formatting and gate duration behind the intended richer footer behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Preserve the cost separator when duration is absent:** `src/auto-reply/reply/agent-runner-utils.ts:139`

  - confidence: 0.91
