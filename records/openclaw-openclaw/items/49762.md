## Summary

Keep open. Current main still lacks a visible response-duration usage footer, but this PR has blocking format/mode/timing issues and no after-fix real behavior proof.

## What This Changes

The branch adds an optional duration value to response usage-line formatting and passes elapsed reply-run time into diagnostics and visible per-reply usage footers.

## Best Possible Solution

Rebase onto `agent-runner-usage-line.ts`, preserve existing cache/cost separators, add duration only to the approved footer mode and timing boundary, then cover it with focused tests and real behavior proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main reproduces the PR-introduced blockers: omitted duration changes cost formatting, `/usage tokens` receives duration, and the sampled timing sits after delivery drains.

## Solution Assessment

No. The current patch is not the best mergeable shape; the safer path preserves existing text contracts, gates the stopwatch by maintainer-approved mode, captures or labels the intended timing boundary, and adds regression coverage.
