## Summary

Keep open. The requested visible duration feature is not implemented on current main, but this PR is not mergeable as-is because source inspection confirms footer format, usage-mode, and timing-boundary regressions, and the external PR still has no after-fix real behavior proof.

## What This Changes

The PR adds a `durationMs` parameter to the per-reply usage footer formatter and passes elapsed reply-run time from `runReplyAgent()` into the formatted usage line.

## Best Possible Solution

Land a rebased, tested patch on the current usage-line helper that preserves existing footer text contracts, scopes duration to the approved usage mode, defines the timing boundary, and includes redacted real after-fix proof.

## Reproduction Assessment

Yes for the PR blockers. Source inspection of the diff against current docs/code reproduces the cost-separator regression, unconditional `/usage tokens` metadata change, and late duration sampling; no after-fix real setup proof is present.

## Solution Assessment

No. The current patch is not the best mergeable shape; it should target `agent-runner-usage-line.ts`, preserve existing separators, gate duration by the approved usage mode, and clarify what interval is being displayed.

  - confidence: 0.78
