## Summary

Keep open. The feature direction is reasonable, but current main still has no footer duration support and this PR still has localized correctness blockers around formatting compatibility, timing boundaries, and usage-mode semantics.

## What This Changes

The branch modifies the reply runner and usage-line formatter to calculate `durationMs` and add a stopwatch duration label to response usage footers.

## Best Possible Solution

Reimplement on current `agent-runner-usage-line.ts`, preserve no-duration footer text, gate or document mode semantics, sample duration at a clear response boundary, and cover the behavior with focused tests plus maintainer-owned changelog handling.

## Reproduction Assessment

Yes. Source inspection is enough: current main lacks a duration parameter in the footer formatter, and the PR diff shows the separator regression, unconditional duration pass-through, and timing sample location.

## Solution Assessment

No. The feature direction is useful, but this stale patch is not the best implementation until it targets the current helper, preserves existing formatting, and settles the timing and usage-mode contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the cost separator when duration is absent:** `src/auto-reply/reply/agent-runner-utils.ts:139`
  - body: The new `parts.join(" / ")` path changes existing no-duration cost output from ` · est ...` to `/ est ...`. That breaks the backward-compatible behavior the PR claims to preserve and can regress display assertions or parsers that depend on the current footer shape.
  - confidence: 0.93
- **[P2] Measure duration before delivery drains:** `src/auto-reply/reply/agent-runner.ts:540`
  - body: `durationMs` is sampled after reply delivery flushing and pending tool-result drains, so the stopwatch can include channel delivery waits rather than the response generation or tool-run time it is presented as. Capture the value before those drains, or label it as end-to-end wall time.
  - confidence: 0.86
- **[P2] Keep the stopwatch out of minimal usage mode:** `src/auto-reply/reply/agent-runner.ts:598`
  - body: Passing `durationMs` unconditionally changes `responseUsage: "tokens"`, whose current footer stays leaner than `full`. Gate the stopwatch to the richer mode unless the mode contract and docs are intentionally expanded.
  - confidence: 0.82
