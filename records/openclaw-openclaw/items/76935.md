## What This Changes

The issue reports that QQ Bot replies became split, repetitive, and context-mixed after upgrading from the bundled 2026.4.29 channel to the external 2026.5.2-era `@openclaw/qqbot` plugin.

## Best Possible Solution

A QQ Bot owner should choose and document the intended default for omitted `streaming`, then add focused regression coverage and live QQ proof showing replies are either final-only by default or streamed without duplicate or mixed-context content.

## Reproduction Assessment

## Solution Assessment

Unclear. A narrow default change is plausible, but the best fix depends on QQ Bot owner judgment on whether omitted `streaming` should preserve legacy final-only delivery or keep current block behavior with stronger duplicate/context safeguards.
