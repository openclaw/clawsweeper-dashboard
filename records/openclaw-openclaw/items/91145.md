## Summary

Keep open: current main and the latest release still only support `sessions_spawn.sandbox` values `inherit` and `require`, so the requested target-agent sandbox mode remains missing and needs maintainer security/product review.

## What This Changes

The issue requests a backward-compatible `sessions_spawn` sandbox mode that uses the target agent's configured sandbox instead of the caller's sandbox.

## Best Possible Solution

Keep this open for maintainer review, then add a documented target-sandbox mode that preserves `inherit` as the default and covers cross-agent sandbox, env, workspace, and sandboxed-requester behavior with focused tests.

## Reproduction Assessment

Yes from source inspection for the missing capability: current main and v2026.6.1 only accept `inherit` or `require` for `sessions_spawn.sandbox`, so `target` cannot be selected. I did not run a live Docker two-agent setup to prove the reported bind/env failure end to end.

## Solution Assessment

Unclear: the requested additive option is plausible, but target sandbox semantics touch security and compatibility, so maintainers should decide the contract before implementation.
