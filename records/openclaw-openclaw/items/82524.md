## Summary

This MEMBER-authored, maintainer-labeled PR should stay open for normal maintainer handling; it addresses a real WebChat image-model routing bug, but the latest patch still has one concrete auth-profile persistence regression when channel model overrides are involved.

## What This Changes

The PR adds a WebChat image-model override plan, forwards one-turn model/fallback options through reply execution, and adjusts image override auth handling plus focused regression tests.

## Best Possible Solution

Keep the PR’s imageModel routing shape, but pass the effective pre-image provider into the auth-profile decision so one-turn image runs never mutate channel-provider session auth state.

## Reproduction Assessment

Yes from source inspection. Current main routes WebChat image attachment handling from the active session model while docs define imageModel as the fallback for models that cannot accept images; the remaining PR regression is also source-reproducible with channels.modelByChannel plus a different default/image provider.

## Solution Assessment

No for the current patch as written. The image-model override approach is narrow, but the auth guard must compare against the effective pre-image provider including channel overrides, not only stored/default provider state.
