## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes if maintainers accept the product decision already stated on https://github.com/openclaw/openclaw/pull/91648: `security.installPolicy` is the operator install gate and `before_install` should be documented as process-local lifecycle behavior. If that decision is not accepted, this is the wrong layer and the runtime bootstrap fix should be revived.
