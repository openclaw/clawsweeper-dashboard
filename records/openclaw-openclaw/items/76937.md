## Summary

Keep open. Current main has session label plumbing and a Sessions table label editor, but it does not provide the requested chat-list rename workflow or project/folder organization, so this remains product UI feature work rather than an implemented cleanup close.

## What This Changes

The issue requests custom chat/session names and project or folder grouping in the webchat UI.

## Best Possible Solution

Define a first-party Control UI session organizer: expose rename in the chat surface using existing session labels, then add explicit project/folder metadata and UI after deciding persistence, ordering, client scope, and migration behavior.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection shows partial label support already exists, while project/folder organization and the requested chat-surface rename workflow are not current behavior.

## Solution Assessment

Unclear until maintainers choose the product shape. The narrowest maintainable path is likely to reuse `sessions.patch` labels for rename first, then add a deliberate folder/project schema rather than overloading session keys.
