## Summary

Keep this PR open: it targets a valid current-main Copilot Claude replay bug, but the branch still has blocking correctness issues, weakens safety guard allowlists unrelated to the replay fix, and lacks after-fix real behavior proof.

## What This Changes

The PR adds Copilot-specific thinking-block replay sanitization, passes that strategy into context-pruning estimates, adds ACP thread-binding coverage, and broadens several guard-script allowlists.

## Best Possible Solution

Revise the branch so a normalized Copilot replay-sanitizer strategy flows through history replay, tool-continuation replay, and pruning estimates; preserve all-thinking turns with the existing non-empty placeholder; and remove or split the guard-script allowlist changes.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main enables Copilot Claude thinking dropping but still routes history replay and tool-continuation replay through the generic latest-turn-preserving helper, and current tests expect that Copilot preservation.

## Solution Assessment

No. The fix direction is right, but this branch selects the sanitizer from a raw provider string, uses an empty placeholder that conflicts with the current replay contract, and weakens unrelated guard coverage.
