## Summary

Keep this PR open because current main still has the Copilot Claude replay gap, but the branch is not merge-ready. The proposed Copilot-specific sanitizer is the right direction, yet the diff still chooses that sanitizer with raw provider-string comparisons, uses a blank placeholder that conflicts with current main's non-empty turn-preservation invariant, and bundles security-adjacent guard-script allowlist changes that need maintainer review or removal.

## What This Changes

The PR adds a Copilot-specific thinking-block replay sanitizer, wires it into history replay, tool-continuation replay, and context-pruning estimates, updates related tests, and broadens two repository guard-script allowlists.

## Best Possible Solution

Refresh this PR against current main, model the Copilot replay behavior as a normalized provider-policy or explicit sanitizer strategy, apply it consistently to session-history replay, tool-continuation replay, and context-pruning estimates, preserve all-thinking assistant turns with the current non-empty omitted-reasoning placeholder, remove or split the guard-script allowlist changes, and add a changelog entry plus regression coverage for non-canonical provider casing.

## Reproduction Assessment

Yes. A focused source-level reproduction is available: current main's Copilot Claude policy sets `dropThinkingBlocks`, but both history replay and tool-continuation replay still use the generic helper that preserves latest assistant thinking; the PR's raw-provider checks can be exercised with a non-canonical provider override such as `GitHub-Copilot`.

## Solution Assessment

No. The PR addresses the right user-visible bug, but the current implementation should normalize provider identity or carry a replay-sanitizer strategy from policy resolution, use the existing non-empty omitted-reasoning placeholder, and keep security guard changes separate from the replay fix.
