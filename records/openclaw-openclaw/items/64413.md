## Summary

Keep open. Current main still lacks a shared normalized delivery-reason taxonomy: cron, Gateway, UI, plugin hooks, and task records expose coarse statuses plus raw/free-form error text, while the open related cron-warning PR only covers execution-vs-delivery status and does not close this delivery-reason request.

## What This Changes

Request to add structured delivery failure reason codes across cron runs, messaging delivery, status UI, and JSON output.

## Best Possible Solution

Define an additive delivery-reason contract with explicit unknown/compat handling, map existing cron/task/outbound failures into it, and expose it consistently through Gateway, plugin hooks, CLI JSON, UI, docs, and focused tests.

## Reproduction Assessment

Yes for verifying the feature gap by source inspection; this is not a runtime bug reproduction. Current main still lacks normalized delivery reason codes across cron types/runtime, Gateway schema, UI/plugin hook types, task delivery state, and CLI docs only cover free-form reasons.

## Solution Assessment

Unclear as a concrete implementation plan. The requested direction is maintainable, but the taxonomy, compatibility shape, and coordination with https://github.com/openclaw/openclaw/issues/49190, https://github.com/openclaw/openclaw/pull/80139, and https://github.com/openclaw/openclaw/issues/74095 need maintainer agreement first.
