## Summary

Keep open. Current main still exposes coarse cron/task delivery states plus raw or free-form error text, and the open related cron-warning PR only addresses warning status rather than the requested normalized delivery-reason taxonomy.

## What This Changes

Request structured normalized delivery failure reasons across cron runs, messaging delivery, status cards, and JSON output.

## Best Possible Solution

Define an additive delivery-reason contract with explicit unknown/compat handling, map existing cron/task/outbound failures into it, and expose it consistently through Gateway, plugin hooks, CLI JSON, UI, docs, and focused tests.

## Reproduction Assessment

Yes for verifying the feature gap by source inspection; this is not a runtime bug reproduction. Current main still lacks normalized delivery reason codes in cron types, run-log/Gateway schema, UI/plugin hook types, CLI delivery docs, and task delivery state.

## Solution Assessment

Unclear as a concrete implementation plan. The direction is maintainable, but reason taxonomy, additive protocol/storage shape, and coordination with https://github.com/openclaw/openclaw/issues/49190 and https://github.com/openclaw/openclaw/issues/74095 need maintainer agreement first.
