## Summary

## Best Possible Solution

Land the transport-level clean-terminal fix once a maintainer accepts `[DONE]` as a sufficient provider terminal signal, while preserving EOF-without-DONE, visible-text, and DSML safeguards.

## Solution Assessment

Yes for implementation shape: observing raw SSE before the OpenAI SDK consumes `[DONE]` is the narrow layer that can distinguish clean terminal streams from EOF-only truncation. The remaining question is maintainer acceptance of that widened tool-execution contract.
