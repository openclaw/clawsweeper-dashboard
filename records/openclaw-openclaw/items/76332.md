## Summary

Keep open for https://github.com/openclaw/openclaw/pull/76332: the PR targets a real subagent lifecycle gap, but the current patch can leave cleanup permanently preclaimed if the pre-cleanup ended-hook path throws, and it is missing the required changelog entry.

## What This Changes

The PR starts subagent announce cleanup before browser cleanup, skips active-child waiting when frozen reply text exists, and preserves prior terminal errors across later success completions.

## Best Possible Solution

Keep the lifecycle hardening direction, but make the cleanup preclaim exception-safe and add the required Unreleased Fixes changelog entry before merge.

## Reproduction Assessment

Yes. Source review shows current main has the three targeted races, and the PR adds focused unit coverage for the frozen-reply, browser-cleanup ordering, and terminal-error scenarios, though this read-only review did not execute those tests.

## Solution Assessment

No as submitted. The approach is narrow and mostly aligned with the lifecycle contract, but the cleanup lease must be released or moved if pre-cleanup hook work throws, and the user-facing fix needs a changelog entry.

- **[P2] Release the cleanup preclaim if completion throws:** `src/agents/subagent-registry-lifecycle.ts:726-728`
  - body: The new preclaim persists `cleanupHandled = true` before later awaited work such as `emitSubagentEndedHookForRun` can run for non-completion run-mode children.
