## Summary

Keep open for PR review. The earlier cleanup-lease and changelog feedback appears addressed, and I found no blocking code correctness issue, but the docs still describe browser cleanup as blocking announce cleanup even though this PR intentionally makes it best-effort background work.

## What This Changes

The PR starts subagent announce cleanup before browser cleanup, skips active-child waiting when frozen reply text exists, preserves terminal errors across later success completion signals, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the lifecycle hardening after updating the user docs to describe browser cleanup as best-effort background work that does not delay completion delivery, and after CI is green.

## Reproduction Assessment

Yes. Source inspection on current main shows all three targeted races/contracts: browser cleanup blocks announce cleanup, active-child waiting happens before frozen reply use, and task docs require stronger terminal states not to be downgraded by later success.

## Solution Assessment

Mostly yes. The implementation is narrow and the earlier exception-safety issue was addressed, but the docs need a small ordering update and CI still needs to settle before merge.
