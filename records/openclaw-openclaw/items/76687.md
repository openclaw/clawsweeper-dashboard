## Summary

Keep open: the Feishu starvation bug is valid and the PR is directionally useful, but the patch still makes timeout completion the default for every `createSequentialQueue()` caller, which changes the drive-comment notice queue's completion/retry semantics.

## What This Changes

The PR adds a Feishu per-key queue timeout with logging, queue regression tests, a changelog entry, and an unrelated Codex app-server approval event handler guard.

## Best Possible Solution

Keep the Feishu message receive timeout release guard, but preserve legacy unbounded semantics for existing sequential-queue callers unless they explicitly opt in to timeout completion.

## Reproduction Assessment

Yes. Current main can be source-reproduced by holding the first same-key Feishu queue task promise; the second same-key task remains chained behind it until the first settles. The PR blocker is also source-reproducible because the drive comment handler still awaits a no-option queue call and then records the event as processed.

## Solution Assessment

No as submitted. The timeout release guard is the narrow right direction for Feishu message receive, but making timeout resolution the default for the shared helper changes existing callers that treat `enqueue` completion as real task completion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve unbounded behavior for non-message queue callers:** `extensions/feishu/src/sequential-queue.ts:41`
  - body: `createFeishuDriveCommentNoticeHandler` still calls `createSequentialQueue()` with no options and records the synthetic comment event as processed immediately after `await enqueue(...)`. With the new default 5-minute timeout, a slow `handleFeishuCommentEvent` makes `enqueue` resolve before the handler finishes, so a later failure is hidden and retry/replay can be dropped. Keep the message timeout, but make the helper default unbounded or explicitly preserve legacy behavior for this caller.
  - confidence: 0.9
