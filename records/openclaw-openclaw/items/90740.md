## Summary

## What This Changes

Adds a `durable_queue` subagent completion fallback that enqueues pending direct handoffs into system events and treats that path as durable in the agent harness task runtime.

## Best Possible Solution

Make the fallback non-evictable or leave completion delivery retryable until requester consumption is observed, then add a maintainer-approved SDK compatibility note and redacted live gateway proof.
