## Summary

Keep this PR open: it targets a real subagent completion loss path, but the patch currently treats a bounded in-memory system-event entry as durable delivery, which can turn an auditable failed/stale completion into silent message loss before the parent drains it.

## What This Changes

This PR adds a `durable_queue` subagent completion fallback that enqueues pending direct handoffs into system events and updates task-runtime durability tests.

## Best Possible Solution

Make the fallback non-evictable or persistent, or keep the task retryable until consumption is observed, then prove the fixed path on a real self-hosted gateway task audit.

## Reproduction Assessment

Yes, source inspection gives a high-signal path: current main returns `completion_handoff_pending` for pending completion handoffs without media or message-tool requirements. I did not run a live gateway reproduction, and the PR's after-fix proof is test-only.

## Solution Assessment

No. The PR is aimed at the right failure branch, but the current implementation is not the best durable fix because it promotes an ephemeral bounded queue write to delivered session state.

- **[P1] Do not mark ephemeral queue writes as delivered:** `src/agents/subagent-announce-delivery.ts:1529-1532`
