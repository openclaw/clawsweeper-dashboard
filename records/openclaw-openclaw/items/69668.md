## Summary

Keep open. Current main still provides isolated agents and an agent-grouped chat session selector, but it does not implement team/project parent nodes, team/group metadata on agents, workspace-derived hierarchy, or independent team management in the Control UI. The related multi-agent UX issues overlap conceptually, but they do not supersede this specific hierarchy request.

## What This Changes

Request team/project parent nodes in the Control UI agent/session lists so related agents and their sessions can be grouped and managed independently.

## Best Possible Solution

Decide the product model first, then implement either a lightweight optional grouping contract such as agent team/project metadata propagated through agents.list and the Control UI, or a broader project/team-room abstraction with matching protocol, config docs, UI states, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection of the Control UI session selector, Agents view, Sessions view, gateway agents.list schema/projection, and agent config type on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but the best maintainable solution needs maintainer product judgment before choosing between presentation metadata and a larger team/project abstraction.
