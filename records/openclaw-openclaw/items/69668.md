## Summary

Keep open. Current main has isolated agents, flat configured-agent session listing, and parent/child session lineage metadata, but it still lacks a first-class team/project parent that owns child agents and routes work to the intended child session.

## What This Changes

Requested feature: add hierarchical team/project grouping to the Control UI session list, with team parent nodes, nested child agents, isolated team sessions, and independent team management.

## Best Possible Solution

Define one maintainer-approved team/project routing contract first, then implement the Control UI hierarchy against that contract with additive protocol/state/docs coverage and focused UI/Gateway tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug reproduction. Source and docs inspection show adjacent primitives exist, but the requested team/project parent workflow is absent rather than broken.

## Solution Assessment

No, not as a UI-only hierarchy. The maintainable solution needs a routing-backed team/project contract before the session list renders parent nodes or lets operators send work through them.
