## Summary

Keep open. Current main still has isolated agents and an agent-grouped chat session selector, but it does not implement team/project parent nodes, team/group metadata in the agent contract, workspace-derived hierarchy, or independent team management UI; the related multi-agent UX issues overlap but do not supersede this hierarchy request.

## What This Changes

Request team/project parent nodes in the Control UI agent/session list so agents can be nested and managed per team instead of appearing as flat entries.

## Best Possible Solution

Decide the product model first, then implement either a lightweight optional grouping contract propagated through config, agents.list, and Control UI, or a broader team/project entity with matching docs and tests.

## Reproduction Assessment

## Solution Assessment
