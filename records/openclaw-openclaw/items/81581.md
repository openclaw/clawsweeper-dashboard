## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/81581 as a narrow Telegram CLI bridge bug. On current main, `openclaw message thread create --channel telegram` calls `telegramPlugin.actions.resolveCliActionRequest`, but the exported `telegramMessageActions` wrapper in `extensions/telegram/src/channel.ts` does not forward `resolveCliActionRequest` from `extensions/telegram/src/channel-actions.ts`; the fallback dispatches bare `thread-create`, which Telegram runtime rejects. Add the missing forwarding through the wrapper, preserving optional runtime/static fallback behavior and the current `topic-create` result contract. Add focused regression coverage against the exported `telegramPlugin.actions` surface, preferably in `extensions/telegram/src/channel-actions.contract.test.ts`, and keep the existing CLI synthetic-plugin test intact.
