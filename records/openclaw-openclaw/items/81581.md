## Summary

Keep open: the bug is still source-reproducible on current main. The CLI asks the exported channel plugin for a `thread-create` remap, Telegram's internal action adapter contains the correct `topic-create` remap, but the exported Telegram action wrapper still does not forward `resolveCliActionRequest`; the two linked fix PRs were closed unmerged and no open replacement PR was found.

## What This Changes

The issue reports that `openclaw message thread create --channel telegram` dispatches `thread-create` instead of remapping to Telegram `topic-create`.

## Best Possible Solution

Forward the existing Telegram CLI remap through the exported Telegram action wrapper, add exported-plugin regression coverage, and then verify with focused tests plus redacted live Telegram topic-creation proof when available.

## Reproduction Assessment

Yes, at source level: current main's CLI asks the exported Telegram plugin for `resolveCliActionRequest`, but that exported wrapper lacks the hook and therefore can fall back to bare `thread-create`, which Telegram rejects. I did not run a live Telegram bot command in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to forward `resolveCliActionRequest` from the exported Telegram `telegramMessageActions` wrapper to the existing implementation and test through `telegramPlugin.actions`; adding broad runtime aliases would be less targeted.
