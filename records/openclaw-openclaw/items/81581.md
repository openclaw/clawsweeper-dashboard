## What This Changes

The issue asks for `openclaw message thread create --channel telegram` to remap `thread-create` into Telegram's `topic-create` action so forum topic creation succeeds.

## Best Possible Solution

Review and land one canonical fix that forwards `resolveCliActionRequest` through the exported Telegram actions wrapper and covers `telegramPlugin.actions` with a focused regression test.

## Reproduction Assessment

Yes. The current-main source path shows the CLI asks the exported Telegram plugin for a remap, but that exported wrapper lacks the hook, so the bare `thread-create` fallback can reach Telegram action handling and be rejected.

## Solution Assessment

Yes. Forwarding the existing Telegram-owned `resolveCliActionRequest` through the wrapper is the narrowest maintainable fix; adding a broader runtime alias would bypass the intended channel-owned CLI normalization seam.
