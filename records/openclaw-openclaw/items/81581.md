## Summary

Keep open: current `main` and latest release still have the source-proven Telegram CLI bridge gap, and the direct repair PRs for this issue are closed unmerged. This is a narrow actionable bug in the exported Telegram plugin action surface.

## What This Changes

The issue requests that Telegram `openclaw message thread create` remap the CLI `thread-create` action to Telegram `topic-create` before gateway dispatch.

## Best Possible Solution

Forward the existing Telegram-owned `resolveCliActionRequest` through the exported Telegram action wrapper and cover that exported plugin surface with a regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the CLI asks the exported Telegram plugin for a remap, that wrapper lacks the hook, fallback dispatches `thread-create`, and the Telegram runtime rejects that action.

## Solution Assessment

Yes. The narrow maintainable fix is to forward `resolveCliActionRequest` through `extensions/telegram/src/channel.ts` and test `telegramPlugin.actions`, rather than adding a broad runtime alias that bypasses channel-owned CLI normalization.
