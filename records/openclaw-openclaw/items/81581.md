## Summary

Keep open: current main still has the Telegram CLI bridge gap, so the reported `thread-create` action can still reach Telegram runtime handling unsupported. The only direct repair PRs found for this issue are closed and unmerged, making this a narrow, source-proven repair candidate.

## What This Changes

The issue reports that Telegram `openclaw message thread create` fails because CLI `thread-create` is not remapped to Telegram `topic-create` before gateway dispatch.

## Best Possible Solution

Forward `resolveCliActionRequest` through the exported Telegram actions wrapper and add regression coverage against `telegramPlugin.actions`, keeping the fix scoped to the channel-owned CLI normalization seam.

## Reproduction Assessment

Yes, at source level: current main's CLI calls the exported Telegram plugin remap hook, the exported wrapper lacks that hook, and the fallback dispatches bare `thread-create` into Telegram runtime rejection. No live Telegram command was run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is wrapper forwarding plus exported-plugin coverage; adding a broad Telegram runtime alias would bypass the existing channel-owned CLI normalization contract.
