## Summary

Keep open: this draft PR still has a source-backed P2 blocker in the Gmail/Tailscale reload path, and the proof provided is only targeted tests plus review output rather than real behavior proof.

## What This Changes

This PR moves Gmail watcher startup into stoppable post-ready sidecar handles, wires cancellation through Gmail watcher and command abort paths, stops sidecars during reload/close, and adds gateway/Gmail/process tests.

## Best Possible Solution

Keep the sidecar/abort direction, but make Tailscale setup cancellable or wait for the cancelled startup to settle before replacement, then prove a real Gmail/Tailscale hot reload.

## Reproduction Assessment

No. I did not run a live Gmail/Tailscale reload; the blocker is source-reproducible because the PR head checks cancellation only after ensureTailscaleEndpoint can finish endpoint mutation.

## Solution Assessment

No. The exec abort normalization is narrow, but the Gmail restart fix is incomplete until the Tailscale setup command is abortable or stale startup completion is serialized before the replacement watcher starts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Abort Tailscale setup before stale reloads can win:** `src/hooks/gmail-watcher.ts:223-224`
  - body: When a config reload or close cancels a post-ready Gmail startup while `ensureTailscaleEndpoint` is still running, this cancellation check only happens after `tailscale serve/funnel --bg --set-path` can mutate the endpoint. The replacement watcher starts immediately after `stopPostReadySidecars()`, so the old startup can still overwrite the fresh path or target; pass cancellation into the Tailscale commands or wait for the cancelled startup before starting the replacement.
  - confidence: 0.88
