## Summary

Keep open: the PR is a plausible focused bug fix, but the patch still lets a cancelled stale Gmail startup mutate Tailscale routing before observing cancellation, and the external contributor proof is only unit tests/review output.

## What This Changes

This PR makes Gmail watcher startup cancellable from post-ready sidecar handles, stops queued sidecars during reload/close, threads AbortSignal into command execution, and adds focused tests.

## Best Possible Solution

Keep the focused sidecar/abort direction, but make cancellation cover all external side effects, especially Tailscale route setup, and add redacted real behavior proof for the changed runtime path.

## Reproduction Assessment

Yes for the review blocker: the stale-route race is source-reproducible by cancelling a post-ready Gmail startup while it is inside ensureTailscaleEndpoint. I did not run a live Tailscale/Gmail repro, and the contributor proof is mock-only.

## Solution Assessment

No; the PR must either pass cancellation into Tailscale setup or wait for the cancelled startup to finish before starting the replacement. The abort-command portion looks directionally reasonable, but the Gmail/Tailscale side effect boundary is incomplete.

## Review Findings

Overall correctness: patch is incorrect
