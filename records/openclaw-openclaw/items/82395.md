## Summary

Keep open: the PR is a useful active draft, but the Gmail reload fix is incomplete because cancelled startup work can still mutate the Tailscale endpoint before observing cancellation, and the external proof is only unit/mock-level.

## What This Changes

The PR moves Gmail watcher startup into cancellable post-ready sidecar handles, stops registered sidecars during reload/close, adds AbortSignal handling to runCommandWithTimeout, and adds gateway/process/Gmail tests.

## Best Possible Solution

Propagate cancellation into Tailscale endpoint setup or wait for cancelled setup to settle before replacement watcher startup, then verify with focused tests and redacted real Gmail/Tailscale reload proof.

## Reproduction Assessment

No high-confidence live reproduction is present; the PR body reports targeted tests and explicitly says a full live Gmail restart was not tested. Source inspection does show the stale Tailscale mutation race clearly enough to block the patch.

## Solution Assessment

No. The exec abort normalization is narrow, but the Gmail reload fix is incomplete until Tailscale setup observes cancellation or stale startups cannot race a replacement watcher.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Abort Tailscale setup before stale reloads can win:** `src/hooks/gmail-watcher.ts:223-224`
  - body: The new cancellation check runs after ensureTailscaleEndpoint, but that helper invokes tailscale status and tailscale serve/funnel without a signal. A reload can stop the old post-ready sidecar and immediately start a fresh watcher while the old Tailscale command still mutates the endpoint, so pass cancellation into the helper or wait for the cancelled startup before replacement.
  - confidence: 0.88
