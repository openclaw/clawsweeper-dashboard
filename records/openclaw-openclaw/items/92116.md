## Label Justifications

- `P1`: The report describes a repeatedly observed core agent-orchestration workflow ending before required child results are processed.
- `impact:session-state`: The yielded parent’s orchestration state can be abandoned while a fresh run consumes the child completion.

## Summary

The diagnostic evidence is useful, but the remaining product/runtime problem is already tracked by the older canonical issue about completions starting a fresh model turn instead of resuming the yielded parent. That issue and its linked open PR cover the same wake-contract decision, so keeping a second tracker would split the design discussion.

## What This Changes

Request a stronger orchestration mechanism so yielded parent agents cannot silently ignore subagent completion events.

## Best Possible Solution

Preserve this report’s model-ignore observations on the canonical issue and settle one yielded-parent wake contract there: resume the paused run or explicitly track completion acknowledgment in the replacement run with bounded failure handling.

## Reproduction Assessment

Yes for the underlying mechanism: current source and tests prove the yielded parent becomes inactive and completion is handled through a new turn, while this report supplies repeated live observations of that turn failing to advance orchestration. The exact model-dependent ignore rate is not deterministic.

## Solution Assessment

No, a generic mechanism that merely forces repeated model processing is not yet the safest design. The canonical discussion should first decide whether yielded parents resume directly or use an explicit bounded acknowledgment state.
