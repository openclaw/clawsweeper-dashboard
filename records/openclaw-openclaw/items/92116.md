## Summary

Keep open: current main and v2026.6.5 still deliver yielded subagent completion through a later model-visible handoff, and the related open issue/PRs cover adjacent route/delivery failures rather than the deterministic parent-consumption contract requested here.

## What This Changes

The issue reports that `sessions_yield` subagent completions are delivered as model-visible context that a parent agent can ignore, silently breaking orchestration loops.

## Best Possible Solution

Define a runtime-owned completion-consumption contract for yielded requesters, then cover it with focused source tests and a repeated real orchestration proof across at least one non-OpenAI model.

## Reproduction Assessment

Yes at the source-contract level: current main and v2026.6.5 end the yielded turn and deliver completion as a later model-visible handoff, with no source-backed deterministic acknowledgment requirement for the parent. I did not run the live multi-agent/model scenario in this read-only review.

## Solution Assessment

Unclear. Prompt strengthening alone is not the best solution; the maintainable direction is runtime-owned completion state, but maintainers must first define the terminal-state and re-yield semantics.
