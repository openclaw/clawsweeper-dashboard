- `impact:session-state`: The failure mode is parent/subagent session state drifting after a child completion is delivered but not deterministically processed by the parent turn.

## Summary

Keep open: current main and v2026.6.10 still present `sessions_yield` completions as model-visible prompt/input material, and the related PRs are open partial candidates rather than a merged canonical forced-processing contract.

## What This Changes

The issue asks OpenClaw to make `sessions_yield` subagent completions deterministic runtime events rather than passive context that the parent model may ignore.

## Best Possible Solution

Define one runtime-owned sessions_yield completion contract that durably records child completion, wakes the parent, and verifies parent consumption without relying only on prompt text.

No high-confidence live reproduction was run here. Source inspection does confirm the reported mechanism: current main and v2026.6.10 still deliver completions through next-turn model-visible prompt/context surfaces.
