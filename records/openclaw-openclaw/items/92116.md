## Summary

Keep open: current main and latest release still route sessions_yield completions through model-visible prompt/input surfaces, and the linked PRs are open partial candidates rather than a merged canonical fix for forced parent completion processing.

## What This Changes

The issue asks for sessions_yield subagent completions to be handled as a deterministic runtime obligation instead of passive next-turn context that the parent model may ignore.

## Best Possible Solution

Define one canonical runtime-owned sessions_yield completion contract that records, delivers, and verifies parent handling of child completions, with focused regression coverage and live or equivalent proof across inactive, direct, and steered parent turns.

No high-confidence live reproduction was established for the nondeterministic LLM miss. Source inspection does confirm the reported delivery class: current main still presents sessions_yield completions as next-turn prompt/input material rather than a runtime-forced processing obligation.
