## Summary

Keep open. Current main and the latest release still use model-visible inter-session/runtime context for subagent completion handoff, and I did not find a deterministic parent-turn acknowledgement or barrier that proves the completion was processed before the parent can end.

## What This Changes

The issue asks for `sessions_yield` subagent completions to be handled as a stronger runtime event rather than passive model context that a parent agent can ignore.

## Best Possible Solution

Define a runtime-level contract for yielded parent turns to acknowledge or retry subagent completion processing, then validate it with a real dashboard or gateway orchestration scenario before closing this issue.

## Reproduction Assessment

Unclear for a high-confidence live repro in this review. Source inspection confirms the reported delivery mechanism, but reproducing the nondeterministic parent ignore requires a live model/dashboard scenario.

## Solution Assessment

Unclear. A stronger runtime harness is plausible, but the best fix needs a maintainer product decision because it changes the sessions_yield completion contract rather than applying a narrow mechanical bug patch.
