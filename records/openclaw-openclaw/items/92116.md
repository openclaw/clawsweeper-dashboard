## Summary

Current source confirms that an inactive yielded parent receives a real completion-triggered model turn, but delivery is considered successful from visible output or side effects without proving that pending child results were semantically consumed. The reported failure is therefore plausible and not fixed between v2026.6.6 and current main, but a live current-main reproduction and a maintainer decision on the bounded acknowledgment/continuation contract are still needed.

## What This Changes

Ensure a parent agent that yields for subagents cannot silently end a completion-triggered turn without processing the pending child result.

## Best Possible Solution

Define structured pending-completion state for yielded sessions and a bounded rule that prevents terminal completion-triggered turns until the pending result is acknowledged or explicitly resolved, with dedupe and loop-escape coverage across dashboard and channel sessions.

## Reproduction Assessment

No, not as a high-confidence live current-main reproduction; however, source inspection confirms the reported vulnerable shape because delivery success does not require structured consumption of the pending completion.

## Solution Assessment

No, “force the LLM to process it” is not yet a complete implementation design; the best fix is a structured, bounded runtime acknowledgment invariant rather than stronger prompt wording or output-text inference.
