## What This Changes

The issue reports that sessions_yield subagent completions can be delivered as passive parent-turn context that the model may ignore, breaking parent/subagent orchestration loops.

## Best Possible Solution

Choose and land a canonical sessions_yield completion contract that makes subagent completion explicit to the parent turn, with live or equivalent proof that the parent cannot silently miss the event.

## Reproduction Assessment

Yes at source level: current main still delivers yielded subagent completion through hidden context, steering prompt text, or requester-agent input, and no forced completion-processing contract was found. No live current-main run was executed, so the nondeterministic LLM miss itself is not live-reproduced here.

## Solution Assessment

Unclear: the requested direction is valid, but the best fix needs maintainer choice between completion-truth metadata, parent-owned delivery, or another explicit runtime event contract. The open related PRs are useful signals but do not yet close this issue.
