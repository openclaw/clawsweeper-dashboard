- `impact:session-state`: The issue concerns whether a yielded parent session reliably advances after receiving child-completion state.

## Summary

Current main still resumes a yielded parent through an ordinary model turn and can mark the handoff delivered based on generic visible output or side effects, without proving the completion advanced the orchestration loop. The report is credible and still necessary, but the exact enforcement and recovery contract needs maintainer judgment plus live current-main validation.

## What This Changes

Ensure a parent agent that used sessions_yield reliably processes subagent completion and continues or explicitly terminates its orchestration loop.

## Best Possible Solution

Represent each yielded completion as explicit bounded runtime state and require the follow-up turn to consume it through a defined orchestration action, finish intentionally, or return a detectable failure; validate the contract across embedded and Codex-style runtimes plus a live provider without adding a parallel orchestration framework.

## Reproduction Assessment

Yes at the source-contract level: current main accepts a normal visible response as a successful completion handoff without proving orchestration advanced. A high-confidence live reproduction of the nondeterministic ignore behavior on current main is still missing.

## Solution Assessment

No final implementation is established yet. Prompt strengthening or generic output checks retain the nondeterministic contract; the safer direction is a narrow explicit completion-consumption state with bounded failure and retry semantics.
