## Summary

Keep open: current main and the latest release still fall back to generic Dream Diary text when post-run session reads stay empty, and the open linked PR remains the active but not-yet-mergeable fix candidate.

## What This Changes

The issue asks OpenClaw to preserve memory-core dreaming narrative text when subagent session cleanup archives the transcript before the plugin can read messages.

## Best Possible Solution

Land or replace the linked PR with a current-main fix that preserves generated narrative text, verifies run/session identity, keeps the settled-read retry, and includes archived-session regression plus redacted real-behavior proof.

## Reproduction Assessment

Yes at source level: current main deterministically writes the no-text fallback when settled post-run session reads return no assistant text, and sessions.get can return [] after the session entry is gone. I did not run a live dreaming cron in this read-only review.

## Solution Assessment

Unclear which implementation is best: trajectory recovery is a plausible mitigation, but it must be keyed and verified by run/session identity, while changing the subagent lifecycle contract needs maintainer direction.
