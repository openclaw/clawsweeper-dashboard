## Label Justifications

- `P2`: The issue affects a bundled background memory feature and can drop generated diary content, but it does not block core chat/runtime availability.
- `impact:data-loss`: The reported failure silently discarded model-generated narrative text before it could be written to `DREAMS.md`.

## Summary

Close. Current main now solves the reported memory-core narrative loss path: plugin SDK subagent runs are kept readable until the owning plugin explicitly deletes them, and memory-core also retries post-run message reads before falling back.

## What This Changes

The issue asks OpenClaw to prevent memory-core dreaming from losing generated narrative text when session cleanup races post-run `getSessionMessages` extraction.

## Best Possible Solution

Keep the shipped host-controlled plugin subagent lifetime plus bounded memory-core read-settle retry; handle broader dreaming diagnostics or unrelated fallback causes in narrower follow-up issues.

## Reproduction Assessment

No current-main failing reproduction remains. The historical race is source-supported by the issue trace, but current main has gateway readback coverage and memory-core retry coverage for the affected path.

## Solution Assessment

Yes. Fixing the host/plugin subagent lifecycle so transcripts remain readable until explicit plugin deletion is the right layer for the archive race, and the memory-core retry covers the separate persistence-lag edge without adding a new public option.
