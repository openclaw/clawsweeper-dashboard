## Summary

## What This Changes

The issue reports memory-core dreaming losing model-generated narrative text when post-completion cleanup archives the transcript before the host plugin reads session messages.

## Best Possible Solution

Land or replace the linked PR with a current-main fix that preserves generated narrative text after archived-session reads, verifies run/session identity, preserves settled-read retries, and includes redacted live or QA proof.

## Reproduction Assessment

Yes at source level: current main deterministically writes the no-text fallback when settled post-run session reads return no assistant text, and sessions.get can return [] after the session entry is gone. I did not run a live dreaming cron in this read-only review.

## Solution Assessment

Unclear which implementation is best: trajectory recovery is a plausible mitigation, but it needs actual run/session identity checks and exact behavior proof, while a lifecycle-contract change needs maintainer direction.
