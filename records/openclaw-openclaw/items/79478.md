The branch adds a deep security audit check that flags installed skills when sensitive `SKILL.md` parameter guidance appears alongside executable outbound network calls, plus docs, tests, and a changelog entry.

## Best Possible Solution

Parse or strip `SKILL.md` frontmatter before collecting parameter evidence, keep the comment-only network safeguard, and land the audit check with focused regression coverage after security review.

## Reproduction Assessment

Yes. Source inspection of PR head is enough: the markdown scanner walks every line, skips only fences, and accepts key/value lines, so frontmatter metadata mentioning a token plus any network call can emit the new critical finding.

## Solution Assessment

No as written. The security-audit direction is useful, but the parameter scan should operate on the instruction body after frontmatter parsing or stripping so metadata does not become model-facing parameter evidence.
