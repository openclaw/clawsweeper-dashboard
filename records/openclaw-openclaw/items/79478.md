## Summary

Keep open: the PR is focused and includes real behavior proof, but the new scanner treats commented network-call text as executable outbound I/O, which can create critical false positives in the security audit; the protected security label also requires maintainer handling.

## What This Changes

The PR adds a deep security-audit check that correlates sensitive SKILL.md parameter guidance with outbound network patterns in installed skill code, plus tests, docs, and a changelog entry.

## Best Possible Solution

Fix the new helper to ignore comment-only network patterns, preferably by reusing the existing scanner's comment-stripping behavior, then land the focused deep-audit check after maintainer security review.

## Reproduction Assessment

Yes. The PR body gives a live CLI JSON proof for the intended behavior, and the blocking false-positive is source-reproducible because the new helper matches raw source lines before comment stripping.

## Solution Assessment

No as written. The direction is narrow, but the implementation should strip comments or reuse the existing scanner heuristic before treating network patterns as executable outbound I/O.
