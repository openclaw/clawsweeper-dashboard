## Summary

Keep open: the PR is security-labeled and still has a blocking false-positive path where YAML frontmatter metadata can be treated as sensitive parameter guidance for a critical audit finding.

## What This Changes

The branch adds a deep security audit check that correlates sensitive SKILL.md parameter guidance with outbound skill code, plus tests, docs, scanner reuse, and a changelog entry.

## Best Possible Solution

Skip SKILL.md YAML frontmatter before collecting markdown parameter evidence, keep executable-comment stripping, and land the focused audit check after maintainer security review.

## Reproduction Assessment

Yes. Source-level reproduction is clear from the PR head: a SKILL.md frontmatter description containing a sensitive keyword plus any executable network call will satisfy the new matcher even without a sensitive parameter line.

## Solution Assessment

No as written. The direction is useful, but the implementation should strip or skip YAML frontmatter before classifying model-facing parameter guidance.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Skip frontmatter before matching sensitive parameters:** `src/security/audit-skill-markdown-context.ts:69-83`
