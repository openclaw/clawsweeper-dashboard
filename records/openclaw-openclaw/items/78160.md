## Summary

## What This Changes

The PR adds a per-server bundled MCP `requestTimeoutMs`, passes it to SDK `callTool` with `resetTimeoutOnProgress: true`, and updates docs plus focused tests.

## Best Possible Solution

Expose `requestTimeoutMs` through the full config contract, add the changelog entry, then merge after the contributor adds real output showing a bundled MCP tool call succeeding past the SDK's 60-second default.

No, not merge-ready as submitted. Passing SDK `RequestOptions` is the right narrow code direction, but the public config schema/changelog gaps and missing real behavior proof need to be resolved first.
