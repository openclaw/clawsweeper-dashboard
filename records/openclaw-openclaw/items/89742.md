## What This Changes

The issue reports a Codex app-server liveness regression where a post-429 hung turn leaves active app-server request state stuck and suppresses idle-watchdog recovery.

## Best Possible Solution

Define a Codex app-server liveness policy that releases silent never-settling turn-scoped requests without breaking legitimate human/tool waits, and keep terminal-idle budget or guidance changes coordinated with the canonical timeout tracker.

## Reproduction Assessment

No high-confidence live current-main reproduction was established. Source inspection still supports the liveness risk for a turn-scoped server request that never settles, but the exact 429 burst path has not been reproduced here.

## Solution Assessment

Unclear as a complete fix. Relaxing the terminal-idle active-request gate is plausible, but the best fix needs maintainer policy for valid approval/user-input/MCP waits and should avoid a new timeout config surface unless maintainers choose it.
