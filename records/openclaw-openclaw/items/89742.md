## Summary

Keep open: current main and v2026.6.6 still gate the Codex terminal-idle watchdog on active app-server turn requests and still do not expose a terminal-idle config key. The exact 429-poisoned shared-client path was not live-reproduced, so this remains a maintainer-reviewed liveness/config decision rather than an implemented or automatically repairable issue.

## What This Changes

The issue asks OpenClaw to prevent Codex app-server rate-limit-triggered silent wedges by letting terminal-idle recovery fire despite stuck active turn requests and by making the terminal-idle cap operator-controlled or lower.

## Best Possible Solution

Define Codex app-server turn liveness so a silent accepted turn or never-settling turn-scoped request releases the session and surfaces retry/status, while tying the terminal-idle budget to a reviewed timeout/default policy instead of a hidden 30-minute cap.

## Reproduction Assessment

No live high-confidence current-main 429 reproduction was established. The source path is reproducible: current main and v2026.6.6 still gate terminal-idle scheduling/firing on active turn requests, and the terminal-idle config key is still absent.

## Solution Assessment

Unclear as a complete fix. Relaxing the terminal-idle active-request gate is plausible, but the best fix needs maintainer judgment over approval/user-input waits, terminal timeout budget policy, and overlap with the open Codex liveness trackers.
