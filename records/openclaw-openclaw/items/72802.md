## Summary

Keep open. Current main still lacks the dashboard manual-auth fallback, while this open PR is the canonical narrow implementation for the linked WSL/headless token-delivery bug and is not merged yet.

## What This Changes

The PR adds a gated `openclaw dashboard` manual-auth hint when clipboard and browser delivery both fail, extends dashboard link regression tests, and adds a changelog entry.

## Best Possible Solution

Land one CVE-safe dashboard implementation that preserves token and `#token=` log redaction while giving WSL/headless users enough token-source and fragment-key guidance to authenticate manually when automatic delivery fails.

## Reproduction Assessment

Yes. Configure a literal `gateway.auth.token`, force clipboard copy to return false, and make browser open unavailable or pass `--no-open`; current main logs only the bare URL and fallback hint, while the PR adds direct coverage for the missing manual-auth guidance.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: it stays in `src/commands/dashboard.ts`, fires only when a tokenized URL was intended and neither delivery channel succeeded, and preserves the existing redaction contract.
