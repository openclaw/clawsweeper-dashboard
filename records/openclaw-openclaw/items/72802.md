## Summary

Keep open. Current main still lacks the manual token-auth recovery hint for headless/WSL dashboard launches, and this PR is the canonical open implementation for #72081; the patch looks narrowly scoped and I found no blocking correctness or security issue, but it still needs maintainer review and CI classification before merge.

## What This Changes

The PR adds a gated manual token-auth hint to `openclaw dashboard` when clipboard and browser delivery fail, extends dashboard link regression tests, and adds a changelog entry.

## Best Possible Solution

Land one narrow CVE-safe dashboard hint that preserves token and fragment-marker log redaction while giving headless users enough token-source and fragment-key information to authenticate manually when automatic delivery fails.

## Reproduction Assessment

Yes. Configure a plaintext `gateway.auth.token`, force clipboard copy to return false, and make browser open unavailable or pass `--no-open`; current main reaches a bare URL or SSH/no-open hint without manual token recovery guidance, while the PR adds focused tests for that path.

## Solution Assessment

Yes. The PR keeps the fix in `src/commands/dashboard.ts`, gates it on actual tokenized-URL delivery failure, leaves SecretRef redaction behavior alone, and adds targeted regression coverage rather than broad auth or Control UI changes.
