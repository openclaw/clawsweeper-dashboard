## Summary

Keep open: current main still strands the token-auth fallback path when clipboard and browser delivery fail, and this PR is a focused command/test/changelog fix with the earlier review-thread concerns addressed.

## What This Changes

This PR adds a gated dashboard CLI manual-auth hint, regression tests, and a changelog entry for tokenized URL delivery failure.

## Best Possible Solution

Land the focused dashboard-command fix after normal maintainer review and exact-head validation, preserving log redaction while giving headless/WSL users a manual auth path.

## Reproduction Assessment

Yes. From source, configure a literal gateway.auth.token, make copyToClipboard return false, and make browser opening unavailable or pass --no-open; current main logs the bare dashboard URL and existing fallback hints without token-source or fragment-key guidance.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: it stays in src/commands/dashboard.ts, triggers only when tokenized delivery failed, deduplicates the --no-open fallback, and preserves the existing log-redaction contract.
