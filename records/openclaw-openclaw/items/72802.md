## What This Changes

The PR adds a CVE-safe `openclaw dashboard` manual-auth hint when tokenized URL delivery via clipboard and browser fails, updates dashboard link regression tests, and adds a changelog entry.

## Best Possible Solution

Land one CVE-safe dashboard implementation that preserves token and fragment-marker log redaction while giving WSL/headless users enough token-source and fragment-key guidance to authenticate manually when automatic delivery fails.

## Reproduction Assessment

Yes. Configure a literal gateway token, force clipboard copy to fail, and use a headless/SSH/WSL path where browser open is unavailable; current main logs only the bare URL plus SSH/no-open guidance, while the PR adds a unit test for that combined-failure path.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix: it stays in `src/commands/dashboard.ts`, fires only when a tokenized URL was intended and neither delivery channel succeeded, and keeps the existing redaction contract. The remaining blocker is merge readiness, not a different design.
