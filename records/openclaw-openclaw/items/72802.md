## What This Changes

This PR changes dashboard command logging to emit a non-secret manual token-auth hint when a tokenized dashboard URL cannot be copied or opened, adds targeted regression tests, and adds a changelog entry.

## Best Possible Solution

Land this narrow dashboard command change after maintainer review and green required checks, preserving token and `#token=` log redaction while giving headless/WSL users enough token-source and fragment-key guidance to authenticate manually.

## Reproduction Assessment

Yes. Configure a literal `gateway.auth.token`, make clipboard copy return false, and make browser open unavailable or pass `--no-open`; current main logs the bare URL plus existing fallback hints without manual token-source guidance, while the PR adds direct unit coverage for that path.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix: it stays in `src/commands/dashboard.ts`, triggers only when a tokenized URL was intended and delivery failed, suppresses redundant `--no-open` copy, and preserves the existing redaction contract.
