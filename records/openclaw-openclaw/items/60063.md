## Summary

Keep open: the PR addresses a real replace-propagation gap, but the current diff can clear live chat buffers on empty replacements and needs adaptation to the current live-chat-projector merge path.

## What This Changes

The PR adds a Gateway assistant-stream `replace` flag path, bypasses throttling for replacements, adds one Gateway unit test, and updates the changelog.

## Best Possible Solution

Carry replace support through the current live-chat-projector/server-chat path, preserve prior text on empty replacements, and keep the changelog aligned with the protocol-only scope.

## Reproduction Assessment

Yes. A source-level reproduction is clear: seed a chat buffer with text, then process an assistant event with `{ text: "", replace: true }`; the PR returns an empty merge and bypasses the empty guard.

## Solution Assessment

No. A deterministic replace signal is the right direction, but the patch should be rebased onto the current projector helper and should ignore or preserve previous text for empty replacements.
