## Summary

Keep open: the replacement plumbing addresses a plausible gateway streaming bug, but the current patch is not mergeable because it changes established incremental delta semantics and still lacks after-fix real behavior proof.

## What This Changes

The PR adds gateway handling for assistant stream `replace` metadata, changes live-chat delta merging to use overlap dedupe, adds a gateway replacement test, and updates the changelog.

## Best Possible Solution

Keep gateway support for explicit `replace: true`, preserve append-only semantics for ordinary incremental deltas, ignore or preserve prior text on empty replacements, and prove the behavior with focused tests plus real runtime evidence.

## Reproduction Assessment

Yes for the review findings from source inspection. The PR helper would collapse repeated incremental chunks that current-main tests require to append, and the empty replacement path can clear buffered text before finalization.

## Solution Assessment

No. The explicit replacement path is a reasonable direction, but the unconditional overlap heuristic is too broad and should be removed or constrained so it does not alter normal delta appends.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve incremental repeated deltas:** `src/gateway/live-chat-projector.ts:65`
  - body: This routes every `nextDelta` through `appendUniqueSuffix`, which treats a repeated suffix as duplicate overlap. Current merge tests intentionally preserve repeated incremental chunks such as `1`, `1`, `1` -> `111`; with this helper the second chunk is dropped, corrupting streamed text and failing the existing contract.
  - confidence: 0.96
- **[P2] Ignore empty replacement payloads:** `src/gateway/live-chat-projector.ts:53`
  - body: When `replace` is true and `text` is empty, this branch returns an empty buffer. The caller then bypasses the empty-text guard and stores the empty projection, so a media/audio-only replacement event after streamed text can erase the buffered assistant reply before finalization instead of preserving the prior text.
  - confidence: 0.86
