## Summary

Keep open: the branch still introduces blocking gateway streaming regressions and the external PR has only test-output proof, not after-fix real behavior proof from a gateway/control UI/provider run.

## What This Changes

The PR adds Gateway consumption of assistant stream `replace` metadata, bypasses delta throttling for replacements, changes live-chat delta merging, adds an agent-event test, and updates `CHANGELOG.md`.

## Best Possible Solution

Keep explicit Gateway support for `replace: true` only if ordinary append semantics and empty replacement safety are preserved, then back it with focused tests and real runtime proof.

## Reproduction Assessment

Yes for the review findings from source inspection: the PR helper collapses current-main repeated-delta test cases, and the empty replacement path can store a blank buffer. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. Explicit replacement handling is the right narrow direction, but applying an overlap helper to ordinary deltas and treating empty replacements as blank buffers is not the safest implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve incremental repeated deltas:** `src/gateway/live-chat-projector.ts:63`
  - body: This sends every `nextDelta` through `appendUniqueSuffix`, which treats a repeated suffix as duplicate overlap. Current merge tests intentionally preserve repeated chunks such as `1`, `1`, `1` -> `111`; this helper drops the second chunk and corrupts streamed text.
  - confidence: 0.96
- **[P2] Ignore empty replacement payloads:** `src/gateway/live-chat-projector.ts:51`
  - body: When `replace` is true and `text` is empty, this branch returns an empty buffer; the caller then bypasses the empty-text guard and stores/broadcasts the blank projection. Metadata-only or media/audio-only replacement events after streamed text can erase the assistant reply instead of preserving it.
  - confidence: 0.86
