## Summary

Keep open: the underlying streaming bug is real, but the latest patch is not mergeable because the new overlap helper collapses valid repeated deltas, empty replacements can erase buffered text, and inspectable real behavior proof is still missing.

## What This Changes

The PR adds Gateway `replace` handling for live assistant chat events, changes delta merging with an overlap-suffix helper, adds one Gateway regression test, and updates the changelog.

## Best Possible Solution

Keep the existing monotonic delta semantics, add only explicit replacement handling through the Gateway projector, preserve prior text for empty textual replacements, and require focused tests plus redacted live-provider proof.

## Reproduction Assessment

Yes. Source inspection shows current main can emit assistant events with `replace: true` while the Gateway ignores the field, and existing merge tests define valid repeated delta behavior that this patch would break.

## Solution Assessment

No. Passing deterministic replace metadata through the Gateway is the right direction, but this patch should not add heuristic delta collapsing and should preserve buffered text for empty replacement payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Do not collapse repeated delta chunks:** `src/gateway/live-chat-projector.ts:30-31`
  - body: The new helper returns the existing buffer whenever the next delta equals the current suffix. Existing `server-chat.stream-text-merge.test.ts` coverage expects repeated incremental chunks like `1`, `1`, `1` and markdown/CJK repeats to append, so this branch would corrupt valid provider output and fail current regression coverage.
  - confidence: 0.93
- **[P2] Preserve buffered text for empty replacements:** `src/gateway/live-chat-projector.ts:50-51`
  - body: When `replace: true` arrives with empty text, this branch returns an empty buffer and the server-side empty guard is bypassed for replacements. Audio/media-visible assistant updates can carry no textual replacement, so this can erase prior streamed text; keep the previous buffer or ignore empty textual replacements unless a separate clear semantic is added.
  - confidence: 0.87
