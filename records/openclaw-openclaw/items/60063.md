## Summary

Keep open: this is a plausible gateway streaming bug fix, but the current diff regresses existing incremental-delta behavior, can clear streamed text on empty replacement payloads, and still lacks real behavior proof.

## What This Changes

The PR adds gateway handling for assistant stream `replace` metadata, overlap-deduplicates appended assistant deltas, adds a gateway event test, and updates the changelog.

## Best Possible Solution

Keep the PR open, preserve literal incremental delta semantics, make empty replacements a no-op, and require real runtime proof before merging the gateway replace plumbing.

## Reproduction Assessment

Yes for the blocking regression: current main tests define that explicit incremental deltas must append repeated chunks literally, and source inspection shows the PR would collapse them. The original provider-overlap problem is source-plausible but still lacks live real-behavior proof.

## Solution Assessment

No. Gateway support for existing `replace` metadata is a reasonable direction, but unconditional overlap dedupe on every delta and clearing on empty replacement payloads are not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve literal repeated delta chunks:** `src/gateway/live-chat-projector.ts:30-36`
  - body: `appendUniqueSuffix` now treats any suffix already at the end of the buffer, and any shared boundary, as duplicate overlap. Existing gateway merge tests require explicit incremental deltas like `"1"`, `"**"`, or `"|"` to append literally; with this helper they collapse instead, corrupting valid repeated output. Keep overlap handling out of raw delta appends unless the event is known to be a cumulative snapshot or a documented replacement.
  - confidence: 0.95
- **[P2] Ignore empty replacement payloads:** `src/gateway/server-chat.ts:406-409`
  - body: The new guard lets `replace: true` with `text: ""` flow through, then stores and broadcasts the empty merge. Agent stream data can carry media/audio metadata with empty text, so this can erase previously streamed assistant text in the live chat buffer even though no textual replacement was sent. Preserve the prior buffer or skip the event when a replacement has no text.
  - confidence: 0.9
