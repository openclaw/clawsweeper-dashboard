## Summary

Keep open: this is a plausible gateway streaming fix area, but the current PR head is not mergeable because it changes ordinary delta append semantics and can erase buffered text on empty replacements; it also still has only test-style evidence for the after-fix behavior.

## What This Changes

The PR adds Gateway handling for assistant stream `replace` metadata, bypasses chat delta throttling for replacement frames, changes live-chat delta merging with an overlap helper, adds a gateway event test, and updates `CHANGELOG.md`.

## Best Possible Solution

Keep explicit Gateway support for `replace: true`, preserve append-only semantics for ordinary deltas, ignore empty replacement payloads that do not provide replacement text, and back the revised patch with focused tests plus real runtime proof.

## Reproduction Assessment

Yes for the review findings from source inspection. The PR helper collapses repeated incremental chunks that current-main tests require to append, and the replacement branch can store an empty buffer when `replace` is true.

## Solution Assessment

No. Explicit replacement support is the right narrow direction, but the unconditional overlap helper should not apply to ordinary deltas and empty replacements should preserve or ignore the existing buffer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve incremental repeated deltas:** `src/gateway/live-chat-projector.ts:65`
  - body: This routes every `nextDelta` through `appendUniqueSuffix`, which treats a repeated suffix as duplicate overlap. Current merge tests intentionally preserve repeated incremental chunks such as `1`, `1`, `1` -> `111`; with this helper the second chunk is dropped, corrupting streamed text and failing the existing contract.
  - confidence: 0.96
- **[P2] Ignore empty replacement payloads:** `src/gateway/live-chat-projector.ts:53`
  - body: When `replace` is true and `text` is empty, this branch returns an empty buffer. The caller then bypasses the empty-text guard and stores the empty projection, so a media/audio-only or metadata-only replacement event after streamed text can erase the buffered assistant reply before finalization instead of preserving the prior text.
  - confidence: 0.86
