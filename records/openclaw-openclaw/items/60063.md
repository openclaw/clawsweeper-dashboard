## Summary

Keep this PR open, but it is not mergeable as-is: the latest diff reintroduces overlap-based delta merging that already fails the current main stream-merge regression tests, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch adds Gateway handling for assistant-stream `replace` metadata, bypasses chat delta throttling for replacements, changes delta merging to use an `appendUniqueSuffix` helper, adds one Gateway unit test, and updates the changelog.

## Best Possible Solution

Keep current main's raw incremental-delta append behavior, then add `replace` support only if it preserves the existing stream-merge contract and is backed by focused tests plus real Gateway or Control UI proof.

## Reproduction Assessment

Yes. The PR branch CI already reproduces the failure in `src/gateway/server-chat.stream-text-merge.test.ts`, and source inspection shows why repeated deltas collapse after the new `appendUniqueSuffix` call.

## Solution Assessment

No. The current approach mixes formal `replace` handling with an unsafe overlap heuristic; the maintainable path is replace-only protocol handling or a clearly specified snapshot contract that does not regress raw delta appends.
