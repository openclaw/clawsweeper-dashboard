## Summary

Keep open: the PR targets a real Gateway streaming gap, but the current head still regresses valid repeated deltas, can clear buffered text on empty replacements, and lacks inspectable real behavior proof.

## What This Changes

The PR adds Gateway handling for assistant stream `replace` metadata, changes live assistant delta assembly with overlap-suffix deduplication, adds a Gateway regression test, and updates the changelog.

## Best Possible Solution

Pass explicit `replace` metadata through the Gateway without changing append-only delta semantics, ignore or preserve prior text for empty textual replacements, and add focused regression coverage plus redacted live-provider proof.

## Reproduction Assessment

Yes. Source inspection shows current main can emit assistant events with `replace: true` while the Gateway ignores it, and the PR regressions are source-reproducible against existing stream-merge test contracts.

## Solution Assessment

No. The explicit replacement plumbing is the right direction, but this PR should not collapse repeated deltas and should not clear the stored text on empty replacement payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Keep repeated deltas append-only:** `src/gateway/live-chat-projector.ts:30-31`
  - body: The new helper returns the existing buffer whenever the next delta equals the current suffix. Existing `server-chat.stream-text-merge.test.ts` cases expect repeated chunks like `1`, `1`, `1`, CJK punctuation, and markdown tokens to append, so this corrupts valid provider output.
  - confidence: 0.93
- **[P2] Preserve prior text on empty replacements:** `src/gateway/live-chat-projector.ts:50-51`
  - body: For `replace: true` with empty text, this branch returns an empty buffer and `server-chat.ts` bypasses the empty guard for replacements. Media or audio-visible assistant updates can carry no textual replacement, so this can erase already streamed text unless empty replacements preserve the previous buffer or are ignored.
  - confidence: 0.88
