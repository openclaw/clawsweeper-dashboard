## Summary

Keep open. Current main has already fixed the restart-sentinel and `saveJsonFile` parts, but shell completion installation still writes user shell profiles with direct `fs.writeFile`, and exec approvals still use temp-and-rename without temp-file fsync or parent-directory sync.

## What This Changes

Use crash-safe atomic writes for restart sentinel persistence, shell completion profile updates, sync JSON persistence, and exec approval storage.

## Best Possible Solution

Finish the two remaining paths by using the async atomic text writer for shell profiles and adding or reusing a durable sync writer for exec approvals, while leaving the already-fixed restart sentinel and `saveJsonFile` behavior intact.

## Reproduction Assessment

Yes, at source level. Current main still reaches direct shell-profile `fs.writeFile` calls and an exec-approvals temp write without temp-file fsync or parent-directory sync; I did not run a crash harness.

## Solution Assessment

Yes. The narrow maintainable fix is to update only shell profile creation/update and exec approvals durable persistence, reusing the existing atomic writer patterns and preserving approval security guards.
