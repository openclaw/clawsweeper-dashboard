## Summary

Keep this issue open. Current main has implemented the restart-sentinel and `saveJsonFile` parts, but the shell completion installer still writes shell profiles directly and the exec approvals writer still lacks the temp-file fsync plus parent-directory sync guarantees requested here.

## What This Changes

Request crash-safe atomic writes for exec approvals, restart sentinel state, shell profile completion installation, and the shared sync JSON save helper.

## Best Possible Solution

Finish the two remaining write paths: route completion profile creation/update through the async atomic text writer with `0o644`, and add a sync durable exec approvals writer that fsyncs the temp file, renames, chmods, syncs the parent directory, and keeps the existing destination-safety checks.

## Reproduction Assessment

Yes. A static current-main check reproduces the missing implementation with high confidence: shell profile writes still call `fs.writeFile`, and exec approvals still uses temp-and-rename without temp-file fsync or parent-directory sync.

## Solution Assessment

Yes, the issue’s narrowed direction is the right fix shape. The maintainable path is to reuse the existing async writer for completion profiles and add a focused sync durable helper or local equivalent for exec approvals without changing approval policy semantics.
