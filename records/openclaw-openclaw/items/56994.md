## Summary

Keep open. Current main has already fixed the restart-sentinel and generic `saveJsonFile` portions, but shell-profile completion install still rewrites the profile with direct `fs.writeFile` and exec-approvals persistence still uses a temp write plus rename without the durable fsync/parent-directory sync contract requested here.

## What This Changes

The issue asks OpenClaw to replace direct writes for restart sentinel, shell-profile completion install, generic JSON saves, and exec approvals with crash-safe atomic persistence.

## Best Possible Solution

Keep the fixed restart-sentinel and `saveJsonFile` behavior, then finish durable atomic writes for exec approvals and shell-profile updates while preserving current permissions and path-hardening.

## Reproduction Assessment

Yes, at source level. Current main still reaches direct shell-profile writes and an exec-approvals temp write without durable fsync semantics; I did not run a crash or power-loss harness.

## Solution Assessment

Yes. The narrow maintainable fix is to update only the remaining exec-approval and shell-profile write paths while preserving security checks, permissions, and the already-fixed JSON/restart-sentinel paths.
