## Summary

Keep open: the PR targets a real timeout death-spiral bug, but the current patch still misses timeout failures surfaced as error payloads without `meta.error`, and the branch is dirty against current `main`.

## What This Changes

The PR adds timeout detection to `runReplyAgent`, a `resetSessionAfterTimeout` callback, user-facing timeout retry text, and regression tests for thrown, `meta.error`, no-session, and heartbeat timeout cases.

## Best Possible Solution

Port the timeout-reset recovery onto current `main` so thrown timeout failures, `meta.error` timeouts, and timeout `isError` payloads rotate the user session only after preserving heartbeat and transient-HTTP safeguards, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows the embedded runner can return a prompt timeout as an `isError` payload without `meta.error`, while the PR reset branch only handles thrown errors or `meta.error` timeouts.

## Solution Assessment

No. The direction is right, but the patch is incomplete until it handles payload-only timeout results and is ported cleanly to current `main`.
