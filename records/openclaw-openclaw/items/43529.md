## Summary

Keep open. The underlying timeout/session death-spiral bug is still source-reproducible on current main, but this PR is not ready because it lacks real behavior proof and the patch still misses payload-only timeout results while over-resetting mixed fallback failures.

## What This Changes

Adds runReplyAgent timeout-session reset handling, user-facing retry text, and focused tests for thrown and meta.error timeout surfaces.

## Best Possible Solution

Revise the patch to drive timeout recovery from structured all-attempt fallback reasons, handle payload-only aborted timeout results before success handling, update current-main callers/tests, preserve heartbeat/transient/mixed-failure behavior, and include redacted real runtime proof.

## Reproduction Assessment

Yes, source-level. Current main lacks a timeout reset path in runAgentTurnWithFallback, and the embedded runner can produce timeout error payloads with meta.aborted and no meta.error that would keep the session on the same transcript.

## Solution Assessment

No. The direction is right, but the PR should rely on structured fallback attempts where every candidate timed out and should classify payload-only timeout results before treating the run as successful.
