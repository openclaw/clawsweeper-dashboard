## Summary

Keep open: the timeout death-spiral bug is still source-reproducible on current main, but this PR misses a current payload-only timeout surface and can reset sessions for mixed-cause fallback failures. It also needs after-fix real behavior proof before merge.

## What This Changes

The PR adds timeout detection to runReplyAgent fallback handling, wires a timeout session-reset callback, returns retry-oriented user copy, and adds focused runReplyAgent tests.

## Best Possible Solution

Update the PR to classify every current timeout surface through structured fallback attempts and timeout payloads, preserve heartbeat/transient-HTTP safeguards, then rebase and validate with real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the embedded runner can return a timeout isError payload with meta.aborted and no meta.error, and current/PR handling can treat that as success without rotating the session.

## Solution Assessment

No. The direction is right, but the maintainable fix should use structured all-attempt timeout checks and payload timeout classification instead of matching any timeout substring in an aggregate error.
