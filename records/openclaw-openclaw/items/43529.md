The PR adds timeout-driven session reset plumbing to `runReplyAgent`/`runAgentTurnWithFallback` plus focused tests for thrown, `meta.error`, reset-unavailable, and heartbeat timeout cases.

## Best Possible Solution

Land a rebased patch that uses structured fallback attempts and embedded-run result metadata to reset only user-turn sessions after every relevant model attempt definitively times out, with current tests and a changelog entry.

No, not as written. The direction is right, but the patch should gate on structured all-timeout evidence, handle payload-only timeout errors before accepting the run, and rebase into current split tests/callers with a changelog entry.
