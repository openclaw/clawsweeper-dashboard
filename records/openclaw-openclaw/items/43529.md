## What This Changes

The PR adds timeout-triggered session reset handling to run-reply fallback execution, wires a timeout reset helper through `runReplyAgent`, and adds focused timeout-reset tests.

## Best Possible Solution

Land a rebased implementation that rotates a run-reply session only after a user turn definitively exhausts every relevant fallback attempt by timeout, handles thrown, `meta.error`, and payload-only timeout surfaces, and preserves existing transient HTTP retry, heartbeat, billing, auth, rate-limit, context-overflow, role-ordering, and lifecycle behavior.
