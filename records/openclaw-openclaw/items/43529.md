## Summary

Keep open: the PR targets a real reply-agent timeout death spiral, but the current diff still misses a payload-only timeout path, can reset sessions on mixed-cause fallback summaries, and lacks required external real behavior proof.

## What This Changes

The PR adds timeout-triggered session reset handling to `runReplyAgent`, user-facing retry copy, and focused timeout reset tests.

## Best Possible Solution

Revise the PR to classify payload-only timeout failures, reset only when every fallback attempt is a timeout, preserve transient/heartbeat safeguards, then add redacted real behavior proof from a real setup.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can emit a timeout as an `isError` payload with `meta.aborted` and no `meta.error`, and that can bypass fallback classification and session rotation.

## Solution Assessment

No. The direction is right, but the patch should use structured all-attempt timeout checks and payload timeout classification rather than checking only `meta.error` or any timeout substring in an aggregate error.

  - confidence: 0.87
