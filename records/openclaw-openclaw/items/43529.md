## Summary

Keep open: current main still lacks the requested timeout session recovery, but this PR is not mergeable as-is because it misses payload-only timeout results, can reset on mixed-cause fallback failures, needs rebase updates for direct runner callers, and lacks external real behavior proof.

## What This Changes

The PR adds timeout-driven session reset handling to `runReplyAgent`, wires a timeout reset callback into `runAgentTurnWithFallback`, and adds focused timeout reset tests.

## Best Possible Solution

Revise the PR to classify payload-only timeout failures, reset only when every fallback attempt is a timeout, preserve transient and heartbeat safeguards, update direct callers/tests after rebase, and add redacted real setup proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can emit a timeout as an `isError` payload with `meta.aborted` and no `meta.error`, and the fallback classifier can then treat it as a successful result without rotating the session.

## Solution Assessment

No. The direction is right, but the patch should use structured all-attempt timeout checks and payload timeout classification rather than checking only `meta.error` or matching any timeout substring in an aggregate error.

Overall correctness: patch is incorrect

Overall confidence: 0.9
