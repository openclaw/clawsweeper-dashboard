## Summary

Keep open. The PR targets a real reply-agent timeout bug, but the current patch still misses payload-only timeout results and can reset sessions for mixed fallback failures; it also lacks after-fix real behavior proof from the contributor.

## What This Changes

The PR adds timeout detection to runReplyAgent fallback handling, wires a timeout session-reset callback, returns retry-oriented user copy, and adds focused tests.

## Best Possible Solution

Update the PR to classify payload-only timeout results and structured all-attempt timeout summaries, preserve heartbeat/transient-HTTP safeguards, then rebase and add redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the embedded runner can return a timeout isError payload with meta.aborted and no meta.error, and the current/PR handling can treat that as a successful run without rotating the session.

## Solution Assessment

No. The direction is right, but the maintainable fix should use structured all-attempt timeout checks and payload timeout classification instead of matching any timeout substring in an aggregate error.
