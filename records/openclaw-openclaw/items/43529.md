## Summary

Keep open. The underlying run-reply timeout-reset problem still appears valid, but this branch is not safe to merge because it can reset sessions for mixed fallback failures, misses timeout failures returned only as error payloads, and is stale against current direct `runAgentTurnWithFallback` callers/tests.

## What This Changes

The PR adds timeout-triggered session reset handling to run-reply fallback execution, wires a reset helper through `runReplyAgent`, and adds focused timeout-reset tests.

## Best Possible Solution

Land a rebased implementation that rotates the run-reply session only after definitive all-model timeout exhaustion on a user turn, handles thrown, `meta.error`, and payload-only timeout surfaces, and preserves existing heartbeat, transient HTTP retry, billing, rate-limit, auth, context-overflow, role-ordering, and external-chat behavior.

## Reproduction Assessment

Yes. Current main inspection shows no timeout reset path, and focused unit coverage can reproduce the intended all-model timeout reset plus the patch regressions: mixed fallback attempts must not reset, while an embedded timeout returned only as an `isError` payload should reset.

## Solution Assessment

No, not as written. The direction is maintainable, but the safer implementation should use structured fallback attempts for all-timeout proof, classify timeout payloads before accepting a run, and update the current split test/direct-caller surface after rebase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require all fallback attempts to time out:** `src/auto-reply/reply/agent-runner-execution.ts:588-595`
  - body: `isTimeoutError(err)` is applied to the aggregate fallback error. Since `FallbackSummaryError` builds that message by joining all attempt strings, one timed-out attempt plus auth, billing, rate-limit, or another failure can match and rotate the session even though not every model timed out. Use the structured `attempts` data and require every relevant attempt to carry timeout evidence before resetting.
  - confidence: 0.91
- **[P2] Handle timeout payloads before accepting the run:** `src/auto-reply/reply/agent-runner-execution.ts:475-480`
  - body: The embedded runner can return a timeout as an `isError` payload without `meta.error`. This branch only checks `runResult.meta?.error`, so that payload-only timeout surface breaks out as a successful run and keeps the bloated/stuck session for the next user turn.
  - confidence: 0.88
- **[P2] Update direct callers for the new reset callback:** `src/auto-reply/reply/agent-runner-execution.ts:91`
  - body: Adding `resetSessionAfterTimeout` as a required parameter means current direct `runAgentTurnWithFallback` test callers no longer satisfy the parameter object after rebase. Update the shared helper and explicit call sites with a stub callback or the test typecheck lane will fail.
  - confidence: 0.84
