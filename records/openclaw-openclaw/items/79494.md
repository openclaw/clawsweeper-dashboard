## Summary

Keep open: this is a real doctor/auth bug with sufficient live proof, but the PR still lets any stored `openai` credential permit the rewrite without honoring auth order/eligibility, and it is missing the required changelog entry.

The PR makes `openclaw doctor --fix` skip legacy `openai-codex/*` to `openai/*` model-ref rewrites on Codex-OAuth-only hosts, with focused tests and doctor docs.

## Best Possible Solution

Keep the skip behavior, but decide rewrite safety through the same auth-profile order and eligibility contract runtime uses, then add the required Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. Current main source shows `doctor --fix` unconditionally rewrites collected `openai-codex/*` model refs, and the PR body supplies before/after terminal logs for a seeded Codex-OAuth-only setup.

## Solution Assessment

No. The direction is the right narrow fix, but the implementation should pass the full auth store through auth order and eligibility helpers instead of scanning provider credentials directly.
