## Summary

Keep open: the PR targets a credible doctor/auth regression and includes real behavior proof, but the implementation is not merge-ready because its safety guard treats config metadata as proof of usable direct OpenAI credentials.

## What This Changes

This PR adds a `doctor --fix` guard that skips rewriting `openai-codex/*` model refs to `openai/*` on apparent Codex-OAuth-only hosts, with focused regression tests.

## Best Possible Solution

Gate the doctor rewrite on a proven usable target OpenAI auth route, preserve `openai-codex/*` with a warning when that proof is unavailable, and update docs/changelog for the new behavior.

## Reproduction Assessment

Yes. Current main source shows `doctor --fix` still reaches the Codex route rewriter for `openai-codex/*` refs, and the linked issue/PR body include live before/after doctor and gateway evidence; I did not mutate the checkout to run it locally.

## Solution Assessment

No. The skip-on-OAuth-only direction is plausible, but this implementation uses `auth.profiles` metadata as credential proof; the safer fix should verify an eligible direct OpenAI auth route or warn and preserve the existing route.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Verify usable OpenAI credentials before allowing rewrite:** `src/commands/doctor/shared/codex-route-warnings.ts:55`
  - body: `auth.profiles` is only config metadata, not proof that a direct OpenAI API key is present or eligible. If a host has an `openai:default` metadata row but the credential store has no usable key, this guard returns false and doctor still rewrites to `openai/*`, leaving the same broken route the PR is trying to prevent. Please base the decision on an eligible target auth route, or preserve and warn when that proof is unavailable.
  - confidence: 0.84
- **[P3] Document the OAuth-only skip behavior:** `src/commands/doctor/shared/codex-route-warnings.ts:624`
  - body: This guard changes user-facing `doctor --fix` behavior, but the current doctor docs still say legacy `openai-codex/*` refs are rewritten to `openai/*`, and this user-facing fix has no changelog entry. Please update the docs/changelog so operators know why doctor preserved the route and how to migrate intentionally.
  - confidence: 0.78
