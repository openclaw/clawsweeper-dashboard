## Summary

Keep open: the PR addresses a real doctor billing-safety regression, but the current patch can still make the wrong preserve/rewrite decision because it bypasses canonical auth ordering and treats a raw environment key as consent.

## What This Changes

The PR adds a doctor route-repair guard, tests, and docs so some legacy `openai-codex/*` model refs are preserved when Codex credentials are present.

## Best Possible Solution

Keep the doctor-side billing safety behavior, but base it on canonical auth profile order and eligibility, require an explicit direct-agent billing selection instead of raw `OPENAI_API_KEY`, and add the changelog entry.

## Reproduction Assessment

Yes, at source level. Current main still rewrites legacy Codex refs unconditionally in `maybeRepairCodexRoutes`, and the PR body supplies live mixed-profile terminal proof; this read-only review did not run a mutating doctor command.

## Solution Assessment

No, not yet. Doctor is a reasonable layer for a fail-closed guard, but the branch should reuse canonical auth order/eligibility and avoid treating `OPENAI_API_KEY` alone as direct-agent billing consent.
