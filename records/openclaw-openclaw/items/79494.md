## Summary

Keep open: current main still has the billing-sensitive doctor rewrite and the PR includes real after-fix proof, but the branch is not merge-ready because its guard bypasses canonical auth order/eligibility and treats `OPENAI_API_KEY` as blanket billing consent; it also needs the required changelog entry.

## What This Changes

The PR adds a `doctor --fix` guard that loads auth-profile credentials, skips `openai-codex/*` to `openai/*` rewrites when Codex credentials exist and `OPENAI_API_KEY` is unset, and adds docs/tests for that behavior.

## Best Possible Solution

Keep the billing-safety behavior, but decide it through resolved auth-profile order and eligibility, preserve active Codex subscription routes unless direct-agent billing is explicitly selected, and document the user-facing doctor change in the changelog.

## Reproduction Assessment

Yes, at source level. Current main still rewrites legacy Codex model refs in `maybeRepairCodexRoutes`, and the PR body supplies live mixed-profile terminal proof; this read-only review did not run a mutating doctor command.

## Solution Assessment

No, not yet. The intended fix belongs in doctor, but the branch should reuse the canonical auth order/eligibility contracts and avoid making `OPENAI_API_KEY` alone the consent signal.
