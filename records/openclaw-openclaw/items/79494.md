## Summary

Keep open: this PR targets a real billing-relevant doctor regression, but the latest head still disagrees with current auth-profile selection semantics and is missing the required changelog entry.

## What This Changes

The PR makes `openclaw doctor --fix` load auth-profile credentials, preserve `openai-codex/*` model refs when it sees a usable `openai-codex` credential and no `OPENAI_API_KEY`, and documents/tests that behavior.

## Best Possible Solution

Keep the billing-safety behavior, but decide it through the existing resolved auth-profile order and credential eligibility contracts, align the env-key policy with current OpenAI docs, and add the missing changelog line.

## Reproduction Assessment

Yes, at source level. Current main still rewrites `openai-codex/*` refs in `maybeRepairCodexRoutes`, and the linked user report plus PR body provide mixed-profile billing proof; this read-only review did not run a mutating doctor command.

## Solution Assessment

No, not yet. The fix should reuse `resolveAuthProfileOrder` and canonical credential eligibility, avoid treating `OPENAI_API_KEY` alone as direct-agent billing consent, and include the required changelog entry.
