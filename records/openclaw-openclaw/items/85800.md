## Summary

Keep open: current main and the latest release already support a provider-owned Perplexity default model config, but the requested per-call model escalation is still absent and needs an explicit shared tool/API decision.

## What This Changes

The issue asks for `web_search` to allow configurable Perplexity model selection by default and optional per-call override.

## Best Possible Solution

Keep the shipped plugin-owned default model config, then decide whether per-call escalation should be a provider-scoped tool parameter, a named search preset, or intentionally unsupported in the generic `web_search` contract.

## Reproduction Assessment

Yes. Source inspection shows the current shared and Perplexity `web_search` schemas do not accept a per-call `model` argument, while the provider default model config already exists.

## Solution Assessment

Unclear. The shipped `plugins.entries.perplexity.config.webSearch.model` is the right owner boundary for the default model, but adding per-call model escalation to the shared tool API needs maintainer product/API approval.
