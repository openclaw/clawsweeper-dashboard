## Summary

Keep open: current main and v2026.5.22 already have a plugin-owned Perplexity default model config, but the requested per-call `web_search` model override is still absent from main and needs an explicit API/security policy decision.

## What This Changes

The issue asks to make the Perplexity-backed `web_search` model configurable by default and overrideable per call for selective Sonar/Sonar Pro escalation.

## Best Possible Solution

Keep the provider-owned default model config, and only add per-call escalation if maintainers approve a provider-owned, allowlisted override contract with focused docs, tests, and no unrelated channel changes.

## Reproduction Assessment

Yes. Source inspection on current main shows the generic `web_search` schema and Perplexity provider schema do not accept a `model` argument, while provider config does accept a default model.

## Solution Assessment

Unclear. The shipped plugin-owned default config is the right boundary for default model selection, but per-call escalation needs a maintainer decision on whether it should be a gated provider parameter, a preset, or intentionally unsupported.
