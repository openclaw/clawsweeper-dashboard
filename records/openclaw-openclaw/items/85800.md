## Summary

Keep open: current main and v2026.5.20 already provide a Perplexity plugin-owned default model config, but the requested per-call model escalation remains absent and needs a maintainer API/product decision.

## What This Changes

The issue requests a configurable default model and optional per-call model override for `web_search`, especially for Perplexity Sonar variants.

## Best Possible Solution

Keep the shipped plugin-owned default model config, and decide whether per-call escalation belongs as a provider-scoped tool parameter, a named search preset, or intentionally outside the generic `web_search` contract.

## Reproduction Assessment

Yes. Source inspection shows the current generic and Perplexity-specific `web_search` schemas do not accept a per-call `model` argument, while the default model config path already exists.

## Solution Assessment

Unclear. The plugin-owned `plugins.entries.perplexity.config.webSearch.model` surface is the right default-model boundary, but per-call model escalation needs maintainer approval for the shared tool/API contract.
