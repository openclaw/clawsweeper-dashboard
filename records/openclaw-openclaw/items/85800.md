## Summary

Keep open: current main already supports a provider-scoped Perplexity default model, but the requested per-call web_search model override is still absent. Prior discussion also questioned adding provider-specific model knobs to the system tool surface, so the remaining work needs maintainer API direction rather than cleanup closure.

## What This Changes

Request configurable default and per-call model selection for Perplexity-backed web_search so users can default to cheaper Sonar and escalate selectively.

## Best Possible Solution

Keep the existing provider-scoped default model config and have maintainers decide whether the remaining escalation surface should be a narrow Perplexity Sonar per-call override, a provider-neutral control, or no tool-call model override.

## Reproduction Assessment

Yes for the remaining issue: source inspection shows the generic web_search schema has no `model` argument and the Perplexity runtime only reads the configured provider model. The default model configuration part is already present via `plugins.entries.perplexity.config.webSearch.model`.

## Solution Assessment

No for the exact requested generic config shape as-is: current OpenClaw already has provider-scoped model config, and prior review questioned provider-specific model parameters on system tools. The maintainable next step is a maintainer decision on the intended model-facing API before implementation.
