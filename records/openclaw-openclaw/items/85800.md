## Summary

Keep open: current main and v2026.5.20 already support a Perplexity default model config, but the requested per-call `web_search` model override is still absent. An open draft implementation exists at https://github.com/openclaw/openclaw/pull/85828, so the issue should stay open for maintainer API/product review rather than spawn another fix lane.

## What This Changes

Request to make the Perplexity `web_search` model configurable by default and allow per-call model escalation.

## Best Possible Solution

Keep the plugin-owned default model config, then review https://github.com/openclaw/openclaw/pull/85828 or an equivalent narrow patch after deciding the supported per-call search-model contract.

## Reproduction Assessment

Yes: source inspection on current main shows the shared `web_search` schema and the Perplexity provider-specific schema do not accept a per-call `model` argument. The default-model half is already reproducibly covered by `plugins.entries.perplexity.config.webSearch.model`.

## Solution Assessment

Unclear: the shipped plugin-owned config is the right default-model owner boundary, but per-call escalation needs a maintainer decision about whether the shared `web_search` contract should expose provider model selection. The open draft PR is an implementation candidate, not yet the settled solution.
