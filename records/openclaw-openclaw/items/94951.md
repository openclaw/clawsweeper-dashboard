Overall correctness: patch is incorrect

Full review comments:

- **[P1] Stop dedupe at real message boundaries:** `src/gateway/server-methods/chat-transcript-inject.ts:78-80`
  - body: `findLatestAssistantMessageIdByText` skips every non-assistant message, including user rows, so a later injected note can match an older assistant reply from a previous turn and return the old id instead of appending. Use the existing tail-reader/current-turn boundary or stop at the first real non-assistant message.
  - confidence: 0.9
- **[P1] Scope text dedupe away from chat.inject:** `src/gateway/server-methods/chat-transcript-inject.ts:195-204`
  - body: This helper also backs the documented `chat.inject` RPC, which appends an operator-visible assistant note. With the unconditional tail-text check, repeated same-text injections can write no new transcript row, so dedupe needs a narrower caller mode or owner-specific path.
  - confidence: 0.88
