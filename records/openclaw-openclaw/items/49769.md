## Summary

Keep open. The PR targets a real binding-account footgun and current main does not already contain its docs/CLI clarification, but the proposed shared parser fallback is not safe to merge as-is because it changes `agents unbind` semantics and the new docs/JSDoc do not fully match current routing behavior.

## What This Changes

The PR adds binding `accountId` documentation and CLI examples, and changes bare `agents add`/`agents bind` specs on multi-account channels to write `accountId: "*"`.

## Best Possible Solution

Merge a revised version that preserves existing `unbind` shorthand compatibility, documents the actual resolver accurately, updates all wildcard guidance consistently, adds targeted regression coverage, and records the user-facing change in the changelog.

## Reproduction Assessment

Yes. The PR regression is source-reproducible by tracing `agents unbind` through the shared `parseBindingSpecs()` path into exact-key removal, and current route tests cover omitted `accountId` versus `accountId: "*"` behavior.

## Solution Assessment

No. The docs clarification is useful, but placing auto-wildcard behavior in the shared parser is not the narrowest maintainable fix; keep parsing neutral and apply any wildcard default only in creation paths, or limit the PR to docs that match current routing.

- **[P2] Preserve channel-only unbind specs:** `src/commands/agents.bindings.ts:264-270`
  - body: Returning `"*"` from the shared account resolver changes `parseBindingSpecs()` for `agents unbind` too.   - body: The new JSDoc says omitted `accountId` follows `channels.<channel>.defaultAccount`, but the route resolver indexes an empty account pattern as `DEFAULT_ACCOUNT_ID` (`"default"`). For a configured default account with another name, this tells users a binding will match when it will not.
  - confidence: 0.91

  - body: The new format table says `--bind <channel>` matches the default account only, but this PR also makes the same bare spec resolve to `accountId: "*"` when the channel has multiple accounts. Either keep bare specs default-scoped or document the auto-wildcard case.
  - confidence: 0.88
