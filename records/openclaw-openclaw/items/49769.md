## Summary

Keep open: the docs clarification is useful and not fully on current main, but this PR is not mergeable as-is because it changes shared bind parsing, creates account-scope/security risk, has inconsistent docs/JSDoc, and lacks after-patch real behavior proof.

## What This Changes

The PR adds accountId binding docs/JSDoc and CLI examples, and changes bare multi-account `agents add` / `agents bind` specs to emit `accountId: "*"`.

## Best Possible Solution

Keep the docs/JSDoc clarification, preserve explicit default-account scoping unless maintainers approve a creation-only wildcard convenience, and require tests plus real CLI proof for any behavior change.

## Reproduction Assessment

Yes. Source inspection shows the PR-head wildcard fallback would flow through shared `parseBindingSpecs()` into both bind and unbind, while current main removes bindings by exact normalized keys.

## Solution Assessment

No. The docs clarification is valuable, but changing shared bare-bind parsing is too broad; the safer path is explicit docs plus a creation-specific wildcard default only if maintainers approve that routing policy.

Full review comments:

- **[P2] Preserve unbind semantics for bare channel specs:** `src/commands/agents.bindings.ts:264-266`
  - body: `resolveBindingAccountId()` feeds `parseBindingSpecs()`, and both `agents bind` and `agents unbind` use that parsed shape. Configs whose configured default is not literally `default` would still miss.
  - confidence: 0.9
- **[P2] Make bare bind docs match the new behavior:** `docs/cli/agents.md:61-63`
  - body: The new table says bare `--bind <channel>` matches the default account only, but the PR code returns `"*"` when a plugin lists multiple accounts. The docs need to match whichever behavior is approved.
  - confidence: 0.87

  - body: The new tip recommends `accountId: "*"` as the safe default, but the same page still has the DM split and family group examples without `accountId`. Readers can still copy the pattern this PR is trying to steer them away from.
  - confidence: 0.82
