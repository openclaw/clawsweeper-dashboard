## Summary

Keep this PR open, but it is not merge-ready: the docs intent is useful, while the branch also changes shared binding parsing in a way that can break existing unbind behavior and broaden account routing without a maintainer-approved policy decision or real behavior proof.

## What This Changes

The branch adds accountId binding documentation/JSDoc and changes bare multi-account agent binding resolution to emit `accountId: "*"` for `agents add` and `agents bind`.

## Best Possible Solution

Keep precise default-vs-wildcard docs, preserve default-scoped behavior by default, and scope any all-account convenience to creation paths only with explicit maintainer approval, tests, and real CLI proof.

## Reproduction Assessment

Yes for the PR-head blockers by source inspection: shared bind-spec parsing feeds add/bind and unbind, and routing stores omitted accountId separately from `"*"`. I did not run live CLI proof in this read-only review, and the contributor has not supplied after-patch proof.

## Solution Assessment

No. The docs clarification is useful, but the current implementation puts a policy-changing wildcard default in shared parsing rather than a creation-only path, and the docs/JSDoc need to match the actual runtime contract.
