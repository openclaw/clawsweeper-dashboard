## Best Possible Solution

Keep precise default-vs-wildcard docs, preserve default-scoped behavior unless maintainers approve a creation-only wildcard convenience, and require bind/unbind tests plus real CLI proof for that policy.

No. The docs clarification is useful, but changing shared bare-bind parsing is too broad; the safer path is precise docs plus a creation-specific wildcard default only if maintainers approve that routing policy.
