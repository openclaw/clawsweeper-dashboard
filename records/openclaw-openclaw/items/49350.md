## Summary

Keep open. Current main and the latest release still skip `agents.list[].identity.name` in the Google Chat typing fallback, and outbound Google Chat reply/send paths still pass Markdown-shaped text to the Chat API without a Google Chat dialect renderer; the only open linked PR is a fork PR that fixes only the typing half.

## What This Changes

The issue asks Google Chat to render assistant Markdown cleanly and to use the configured agent display name in the message-mode typing indicator.

## Best Possible Solution

Ship a Google Chat plugin-owned fix that preserves existing account and top-level agent-name precedence, adds `identity.name` before the `OpenClaw` fallback, and renders outbound reply plus explicit-send text into Google Chat's text-message dialect with focused tests and live Google Chat proof.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` falls through to `OpenClaw`, and Markdown-shaped reply text reaches the Google Chat `text` field without a channel renderer.

## Solution Assessment

Yes for the issue direction, with one correction: the maintainable fix is not a strip-only path, because Google Chat supports its own lightweight text-message syntax; OpenClaw should render or translate to that dialect in the Google Chat plugin.
