## Summary

Keep open. Current main still omits `identity.name` from the Google Chat typing-display fallback, sends Markdown-shaped reply/action text to the Chat text API without a Google Chat-specific renderer, and the linked open PR only fixes the typing half while its closing reference could close the broader tracker.

## What This Changes

The issue requests Google Chat to render or convert outgoing Markdown-style text and show the configured agent identity name in the typing indicator.

## Best Possible Solution

Land or replace the focused typing fallback with tracker handling, then add a Google Chat plugin-local Markdown-to-Chat renderer across reply, action, and attached-result send paths with regression coverage.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply/action text reaches the Google Chat `text` API path without a Chat-specific renderer.

## Solution Assessment

Yes for the requested direction, but current main does not solve it. The maintainable fix is a Google Chat-owned formatter/converter plus the identity-name fallback, not a broad cross-channel stripping helper.
