## Summary

Keep open. Current main and the latest release still omit `agents.list[].identity.name` from the Google Chat typing display-name fallback, and Markdown-shaped text is still passed directly into Google Chat reply, action, and outbound adapter sends. Open PR #49361 is an unmerged partial fix for the typing half only, so this issue should remain open until the paired work is reviewed and the Markdown delivery behavior is addressed.

## What This Changes

The issue asks the Google Chat plugin to normalize Markdown before outbound delivery and to use the configured agent identity name in the typing indicator instead of falling back to `OpenClaw`.

## Best Possible Solution

Review and either land or replace PR #49361 for the identity-name fallback, then add a Google Chat-owned Markdown normalization step before reply updates, reply sends, media captions, message actions, and attached outbound adapter sends, with focused regression tests in the plugin.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with only `agents.list[].identity.name` configured, `resolveBotDisplayName` falls through to `OpenClaw`, and Markdown text reaches Google Chat `text` fields through reply and action paths without normalization.

## Solution Assessment

Yes for the requested direction, but current main is not solved. The narrow maintainable fix belongs inside the Google Chat plugin delivery boundary: include `identity.name` in the fallback chain and normalize Google Chat outbound text without changing core channel behavior.
