## Summary

Keep open: current main still has the Google Chat typing fallback that skips `agent.identity.name`, and the markdown half remains unresolved because Google Chat supports only a limited Markdown dialect rather than the full CommonMark-style output agents often produce. ([support.google.com](https://support.google.com/chat/answer/7649118?co=GENIE.Platform%3DDesktop&hl=en&utm_source=openai)) The open linked fork PR only covers the typing-name half, so closing this issue would lose the remaining formatting decision and implementation work.

## What This Changes

The issue asks the Google Chat channel to avoid exposing raw Markdown syntax and to show the configured agent identity name in the typing indicator.

## Best Possible Solution

Keep this issue open until maintainers either land a Google Chat plugin fix that adds the `agent.identity.name` fallback and an approved Markdown-to-Google-Chat/plain-text policy, or explicitly split the typing and formatting work into separate canonical items.

## Reproduction Assessment

Yes for the typing half from source: current main resolves the Google Chat typing name without checking `agents.list[].identity.name`. The markdown half is source-plausible because replies are delivered without dialect conversion, but the exact rendered output still needs live Google Chat proof against Chat's limited Markdown support. ([support.google.com](https://support.google.com/chat/answer/7649118?co=GENIE.Platform%3DDesktop&hl=en&utm_source=openai))

## Solution Assessment

No as a single blind strip-markdown fix: the typing fallback is a narrow plugin-owned repair, while the markdown half needs a maintainer decision between preserving supported Google Chat formatting and converting or stripping unsupported Markdown syntax.
