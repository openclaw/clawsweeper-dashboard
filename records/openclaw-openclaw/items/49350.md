## Summary

Keep open. Current main and latest release still skip `agents.list[].identity.name` in the Google Chat typing-name fallback and still pass outbound Markdown-shaped text directly to Google Chat delivery paths; the open closing PR only addresses the typing fallback, while the Markdown PR was closed unmerged.

## What This Changes

The issue asks Google Chat to normalize assistant Markdown before sending and to show the configured agent identity name in the message-mode typing indicator.

## Best Possible Solution

Land or replace the focused identity-name fallback, then add Google Chat-owned Markdown-to-Chat rendering across reply, action, and attached-result send paths with regression coverage.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply/action text is forwarded to Google Chat `text` without channel-specific rendering.

## Solution Assessment

Yes for the direction, but current main does not solve it. The maintainable fix is a Google Chat plugin formatter that maps OpenClaw Markdown into Chat's supported text syntax plus the existing identity-name fallback. ([developers.google.com](https://developers.google.com/chat/format-messages?utm_source=openai))
