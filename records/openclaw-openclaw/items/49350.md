## Summary

Keep open. Current main and the latest release still show both Google Chat UX problems in source: the typing name fallback skips `identity.name`, and outbound text reaches the Google Chat API without a channel renderer; the open fork PR only covers the typing half.

## What This Changes

The issue requests Google Chat-specific outbound formatting cleanup and a typing indicator display-name fallback that respects the configured agent identity.

## Best Possible Solution

Ship a Google Chat plugin-owned fix that preserves the existing account and top-level agent name precedence, adds `identity.name` before the hardcoded fallback, and renders outbound Markdown into Google Chat's text-message dialect on reply and explicit send paths with focused tests and real Google Chat proof.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.22` shows a route with only `agents.list[].identity.name` falls through to `OpenClaw`, and Markdown-shaped reply text reaches the Google Chat API text field without a renderer.

## Solution Assessment

Yes. The narrow maintainable fix is channel-local: keep existing name precedence, add `identity.name`, and translate outbound text to Google Chat's supported dialect instead of changing cross-channel behavior.
