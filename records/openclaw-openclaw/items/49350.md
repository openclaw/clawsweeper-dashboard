## Summary

Keep open. Current main and the latest release still omit `agents.list[].identity.name` from the Google Chat typing-name fallback, and reply delivery still sends Markdown-shaped chunks without a Google Chat dialect renderer; the open linked fork PR covers only the typing half.

## What This Changes

Reports two Google Chat channel UX bugs: Markdown-shaped replies render as raw syntax and the typing indicator ignores the configured agent identity name.

## Best Possible Solution

Ship a Google Chat plugin-owned fix that adds the `identity.name` fallback and renders outbound Markdown into Google Chat's text-message dialect on both reply and explicit send paths.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured falls through to `OpenClaw`, and Markdown-shaped reply text reaches the Google Chat `text` field without a plugin-local dialect renderer.

## Solution Assessment

Yes, the maintainable solution is channel-local: add the existing identity fallback and render Markdown to Google Chat syntax before sending. A typing-only PR or broad cross-channel text stripper would not fully solve the issue.
