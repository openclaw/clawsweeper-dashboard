## Summary

Keep open. Current main and the latest release still skip `agents.list[].identity.name` in the Google Chat typing-name fallback, and outbound Google Chat text still reaches the API without a Google Chat Markdown renderer.

## What This Changes

The issue asks Google Chat to render outbound Markdown appropriately and to show the configured agent identity name in typing indicators.

## Best Possible Solution

Ship a Google Chat plugin-owned fix that adds the `identity.name` fallback and renders outbound Markdown into Google Chat's text-message dialect on both reply and explicit send paths, with focused tests and real Google Chat proof.

## Reproduction Assessment

Yes. Source inspection shows a Google Chat route with only `agents.list[].identity.name` falls through to `OpenClaw`, and Markdown-shaped reply text reaches the Google Chat `text` field without a channel renderer.

## Solution Assessment

Yes. The maintainable fix is channel-local: preserve the existing account and top-level agent name precedence, add `identity.name`, and render Markdown to Google Chat syntax instead of stripping or changing cross-channel behavior.
