## Summary

Keep open: current main and v2026.6.6 still skip `agents.list[].identity.name` in the Google Chat typing fallback and still send Markdown chunks without a Google Chat-specific renderer; the same-repo fix attempts are closed unmerged and no canonical merged replacement covers both halves.

## What This Changes

The issue asks the Google Chat plugin to make assistant replies render readable formatting and make the message-mode typing placeholder use the configured agent identity name.

## Best Possible Solution

Land one focused Google Chat plugin fix that adds `identity.name` to `resolveBotDisplayName` and renders outbound/inbound reply Markdown through a Google Chat-specific formatter while preserving readable plain text for unsupported constructs.

## Reproduction Assessment

Yes, source-level: the current Google Chat typing path skips `agents.list[].identity.name`, and the reply paths send Markdown chunks through Google Chat without a Chat-specific renderer. I did not run a live Google Chat workspace.

## Solution Assessment

No, the original blanket Markdown stripping direction is not clearly the best fix. Google Chat supports a small text syntax, so maintainers should choose a Google Chat-aware conversion policy and pair it with the narrow identity fallback fix.
