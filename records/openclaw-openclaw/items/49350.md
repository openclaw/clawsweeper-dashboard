## Summary

Keep open. Current main still skips `identity.name` in the Google Chat typing display-name fallback, and Markdown-shaped reply/action text is still forwarded to the Chat API without a Google Chat-specific renderer; the open linked PR only covers the typing half.

## What This Changes

The issue asks for Google Chat replies to render Markdown appropriately and for the typing indicator to use the configured agent name.

## Best Possible Solution

Land or replace the focused typing fallback without closing the broader tracker, then add a Google Chat plugin-local Markdown-to-Chat renderer across reply, action, and attached-result send paths with regression coverage.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply/action text reaches the Google Chat `text` API path without a Chat-specific renderer.

## Solution Assessment

Yes for the direction, but current main does not solve it. The maintainable fix is a Google Chat-owned formatter/converter plus the identity-name fallback, not a broad cross-channel stripping helper.
