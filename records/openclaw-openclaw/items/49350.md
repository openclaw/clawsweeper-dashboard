## What This Changes

The issue asks the Google Chat channel to render or convert Markdown-shaped outbound text appropriately and to use the configured agent display name in typing indicators.

## Best Possible Solution

Land or replace the focused typing fallback without closing the broader tracker, then add a Google Chat plugin-local Markdown-to-Chat renderer across reply, action, and attached-result send paths with regression coverage.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply/action text reaches Google Chat `text` without a Chat-specific renderer.

## Solution Assessment

Yes for the direction, but current main does not solve it. The maintainable fix is a Google Chat-owned formatter/converter plus the identity-name fallback, not a broad cross-channel stripping helper.
