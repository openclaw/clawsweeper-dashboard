## Summary

Keep open: current main and the latest release still skip `agents.list[].identity.name` in the Google Chat typing-name fallback, while the markdown half needs maintainer direction because current Google Chat support is partial rather than plain no-markdown.

## What This Changes

The issue requests Google Chat replies to avoid raw Markdown rendering and the typing indicator to use the configured agent or app name.

## Best Possible Solution

Split the work: fix the Google Chat typing fallback to include `agents.list[].identity.name`, and only change markdown delivery after maintainers choose a Google Chat-specific renderer or documented subset behavior backed by live proof.

## Reproduction Assessment

Yes for the typing-name half by source inspection: current main falls back from account name to top-level agent name to `OpenClaw` without reading `agent.identity.name`. The markdown half is not live-reproduced and needs a concrete Google Chat example because current Google docs support a limited Markdown subset.

## Solution Assessment

No for blanket markdown stripping: current docs and channel metadata support some Markdown, so a Google Chat-specific renderer or narrower unsupported-syntax fix is safer. The typing-name fix itself is narrow and belongs in the Google Chat plugin fallback chain.
