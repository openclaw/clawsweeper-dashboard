## Summary

Keep open. Current main and the latest release still skip `agents.list[].identity.name` in the Google Chat typing-name fallback, and Google Chat reply/send paths still forward Markdown-shaped text without a Google Chat text-message renderer; the only open linked PR is a fork PR that fixes only the typing half.

## What This Changes

The issue asks Google Chat to render agent Markdown as Google Chat-friendly text and to use the configured agent identity name in the typing indicator.

## Best Possible Solution

Ship a Google Chat plugin-owned fix that preserves current name precedence, adds the `identity.name` fallback, renders outbound Markdown into Google Chat text-message syntax on both reply and explicit send paths, and adds focused regression coverage plus real Google Chat proof.

## Reproduction Assessment

Yes, source-level. Current main falls through to `OpenClaw` when only `agents.list[].identity.name` is configured, and Markdown-shaped reply/send text reaches the Google Chat API wrapper without a renderer; I did not run a live Google Chat workspace flow in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is channel-local: preserve account and top-level agent name precedence, add `identity.name`, and render to Google Chat's documented text-message syntax instead of changing cross-channel behavior.
