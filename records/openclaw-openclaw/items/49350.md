## Summary

Keep open. Current main still omits `agent.identity.name` from the Google Chat typing display-name fallback and still sends Markdown-shaped reply/action text without Google Chat-specific normalization; the open closing PR covers only the typing half.

## What This Changes

Request to normalize Google Chat outbound markdown and make the Google Chat typing indicator use the configured agent identity name.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/49361 for the typing fallback, then implement Google Chat-owned CommonMark-to-Google Chat text normalization for replies, actions, captions, and direct sends with focused regression tests.

## Reproduction Assessment

Yes. By source inspection, a Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply/action text is passed directly into Google Chat send/update calls.

## Solution Assessment

Yes for the direction, but current main does not solve it. The narrow maintainable fix belongs inside the Google Chat plugin delivery boundary, preferably converting common Markdown into Google Chat text syntax rather than changing core channel behavior.
