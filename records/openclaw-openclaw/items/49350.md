## Summary

Keep open. Current main still leaves both Google Chat UX problems in place: typing only falls back through account name and agent.name before OpenClaw, while reply/action text is sent without a Google Chat Markdown normalization step; open PR #49361 is still only an unmerged implementation candidate for the typing-name half.

## What This Changes

Google Chat should normalize Markdown before sending outbound text and use the configured agent identity name in typing indicators.

## Best Possible Solution

Review, land, or replace PR #49361 for the typing fallback, then add Google Chat-owned text normalization for replies, message actions, media captions, and direct sends with focused regression tests.

## Reproduction Assessment

Yes, by source inspection. With only agents.list[].identity.name configured, resolveBotDisplayName falls through to OpenClaw, and Markdown-shaped reply/action text is passed directly into Google Chat text fields.

## Solution Assessment

Yes for the proposed direction, but not solved on current main. The narrow maintainable fix belongs in the Google Chat plugin delivery boundary: add identity.name to the fallback chain and normalize outbound Google Chat text without changing core channel behavior.
