## What This Changes

The issue asks Google Chat to avoid raw Markdown-looking output and to use the configured agent identity name in the typing indicator.

## Best Possible Solution

Land or replace the linked typing fix, then add a deliberate Google Chat text renderer or documented plain-text policy across reply and outbound send paths.

## Reproduction Assessment

Yes, source-level. The typing path skips `agents.list[].identity.name`, and the reply/outbound paths send Markdown-mode chunks without a Google Chat-specific renderer; I did not run a live Google Chat workspace.

## Solution Assessment

No, not as originally phrased. Stripping all Markdown is probably not the best fix because Google Chat supports its own formatting markers; the maintainable fix should convert supported syntax and leave unsupported constructs readable.
