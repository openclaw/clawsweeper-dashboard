## Summary

Keep open: current main and v2026.6.6 still skip `identity.name` in the Google Chat typing-name fallback and still send Markdown chunks directly to Chat text fields. The related OpenClaw PRs were closed unmerged, and the remaining open fork PR only covers the typing-name half.

## What This Changes

The issue asks for Google Chat replies to render formatting readably and for the typing indicator to use the configured agent/app name.

## Best Possible Solution

Implement a Google Chat-owned formatter aligned with official Chat text syntax, add `identity.name` to the typing fallback, and prove both with focused plugin tests plus live Chat smoke evidence.

## Reproduction Assessment

Yes, source-level. Current main's typing path skips `agents.list[].identity.name`, and the reply path sends Markdown chunks to Google Chat without a Chat-specific renderer; I did not run a live Google Chat workspace.

## Solution Assessment

No, not as originally phrased. Stripping all Markdown is probably not the best fix because Google Chat supports a small formatting syntax; the maintainable path is a Google Chat-aware renderer plus the narrow typing fallback fix.
