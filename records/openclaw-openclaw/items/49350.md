## Summary

Keep open: current main and v2026.6.6 still skip `agents.list[].identity.name` in the Google Chat typing fallback, and the formatting half remains unresolved after the Markdown-stripping PR was closed unmerged and challenged as the wrong policy direction.

## What This Changes

The issue asks Google Chat replies to render readable formatted text and typing placeholders to use the configured agent display identity instead of `OpenClaw`.

## Best Possible Solution

Land or replace a focused Google Chat plugin fix that adds `identity.name` to the typing fallback and applies a Google Chat-aware text formatter instead of blind Markdown stripping.

## Reproduction Assessment

Yes, at source level: `resolveBotDisplayName` skips `agent.identity.name`, and reply delivery passes formatted chunks directly to Google Chat without a channel-specific renderer. I did not run a live Google Chat workspace repro.

## Solution Assessment

No, not as a blind strip-Markdown fix. The typing fallback is narrow, but the formatting half should use a Google Chat-aware conversion policy that preserves supported formatting and degrades unsupported Markdown to readable text.
