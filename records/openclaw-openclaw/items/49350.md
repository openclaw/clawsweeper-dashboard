## Summary

Keep open: current main still sends the Google Chat typing placeholder through a fallback chain that skips `agents.list[].identity.name`, and the markdown half is not settled because Google Chat supports a limited Markdown dialect rather than no Markdown at all.

## What This Changes

The issue asks Google Chat replies to avoid raw Markdown-looking text and asks typing messages to use the configured agent/app display name instead of `OpenClaw`.

## Best Possible Solution

Split the end state into an owner-local Google Chat fix: add a sanitized `identity.name` fallback for typing, and define a Google Chat text renderer that converts supported Markdown dialect features or intentionally sends plain text with live proof.

## Reproduction Assessment

Partly. The typing-name problem is source-reproducible from `resolveBotDisplayName` and the visible placeholder send path, while the markdown symptom still needs live Google Chat proof for the exact rendering cases.

## Solution Assessment

Unclear. Adding `identity.name` in the Google Chat fallback chain is the narrow typing fix, but the formatting side should be a Google Chat dialect renderer or explicit plain-text policy rather than an unreviewed blanket strip.
