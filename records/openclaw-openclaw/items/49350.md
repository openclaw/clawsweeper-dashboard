## Summary

Keep open: current main still falls back to `OpenClaw` when only `agents.list[].identity.name` is configured, and an open linked PR targets that typing-name half. The markdown half needs maintainer and live Google Chat confirmation because current code marks Google Chat as markdown-capable and Google's docs support a subset of Markdown-style text formatting.

## What This Changes

Google Chat should avoid raw markdown rendering and use the configured agent identity for typing messages instead of falling back to `OpenClaw`.

## Best Possible Solution

Keep this issue as the canonical tracker, review or replace the linked typing fallback PR, and make the markdown behavior decision from live Google Chat proof plus the official text-formatting contract.

## Reproduction Assessment

Yes for the typing-name half at source level: current main ignores `agents.list[].identity.name` and sends `_OpenClaw is typing..._` when account and top-level agent names are absent. The markdown half still needs live Google Chat proof because current code intentionally treats Google Chat as markdown-capable.

## Solution Assessment

Unclear as a single fix: adding `agent.identity.name` to the Google Chat fallback chain is the narrow owner-local typing fix, but blanket markdown stripping is not proven best because Google Chat supports a formatting subset and may need a renderer/converter decision instead.
