## Summary

Keep open. Current main and latest release v2026.5.22 still omit `agents.list[].identity.name` from the Google Chat typing display-name fallback and still pass Markdown-shaped outbound text to the Chat API without a Google Chat dialect renderer; the related target-repo PRs are closed and unmerged, while the open cross-repo PR only covers the typing half.

## What This Changes

The issue asks Google Chat to convert or strip Markdown before sending messages and to use the configured agent identity name in the typing indicator.

## Best Possible Solution

Implement a Google Chat-owned Markdown-to-Chat formatter across reply, action, and attached-result send paths, add the `identity.name` typing fallback, and keep this issue open until both halves have tests and real Google Chat proof.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped text reaches `body.text` without a Google Chat renderer.

## Solution Assessment

Yes for the requested direction, but current main does not solve it. The maintainable fix is plugin-local dialect conversion plus the identity-name fallback, not a broad cross-channel text stripper.
