## Summary

Keep open. Current main still skips `agents.list[].identity.name` when Google Chat builds the typing indicator name, and the outbound Google Chat paths still forward Markdown-shaped reply/action text without a Google Chat-specific formatter; the linked open PR only addresses the typing fallback, while the Markdown PR was closed unmerged after discussion that blunt stripping was the wrong approach.

## What This Changes

The issue requests Google Chat-specific output cleanup for Markdown-shaped replies and the typing indicator display name.

## Best Possible Solution

Land or replace the focused identity-name fallback, then implement Google Chat-owned Markdown-to-Chat rendering across reply, action, and attached-result send paths with regression coverage.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply/action text is forwarded to Google Chat `text` without channel-specific rendering.

## Solution Assessment

Yes for the direction, but current main does not solve it. The maintainable fix is a Google Chat plugin formatter for OpenClaw Markdown plus the existing identity-name fallback, not a broad cross-channel strip helper.
