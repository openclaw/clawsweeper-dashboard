## Summary

Keep open. Current main and the latest release still skip `agents.list[].identity.name` in the Google Chat typing-name fallback, and reply delivery still forwards Markdown-shaped chunks to the Chat API without a Google Chat text-message dialect renderer; the only open linked PR covers the typing half, not the formatting half.

## What This Changes

The issue requests Google Chat-specific outbound text formatting cleanup and a typing indicator display name fallback that uses the configured agent identity instead of hardcoded OpenClaw text.

## Best Possible Solution

Adopt or replace the linked typing-only work with a combined Google Chat plugin fix that adds the identity-name fallback and applies a Chat-dialect formatter across reply and message-send paths.

## Reproduction Assessment

Yes, source-level. A Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply text reaches the Google Chat `text` field without a plugin-local dialect renderer.

## Solution Assessment

Yes for the requested direction, but current main does not solve it. The maintainable fix is Google Chat-owned dialect conversion plus the identity-name fallback, not a broad cross-channel text stripper.
