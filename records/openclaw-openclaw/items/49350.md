## Summary

Keep open: current `main` and the latest release still skip `agents.list[].identity.name` in the Google Chat typing-name fallback, and the markdown half remains unsettled because Google Chat supports a small text-message Markdown dialect rather than full CommonMark-style agent output. The only open linked fork PR covers the typing half only, so closing would lose the formatting decision and implementation work. ([developers.google.com](https://developers.google.com/workspace/chat/format-messages))

## What This Changes

The issue asks the Google Chat plugin to render agent markdown cleanly and use the configured agent/app identity for the typing placeholder.

## Best Possible Solution

Keep this issue open until maintainers either land a Google Chat plugin fix for the identity-name fallback plus an approved Markdown-to-Google-Chat/plain-text policy, or split the two behaviors into separate canonical items.

## Reproduction Assessment

Yes for the typing half from source: current `main` and `v2026.6.8` skip `agent.identity.name` before falling back to `OpenClaw`. The markdown half is source-plausible because delivery sends markdown chunks without Google Chat dialect conversion, but exact rendering still needs live Google Chat proof.

## Solution Assessment

No, this should not be closed or fixed as a blind strip-markdown change. The typing fallback is a narrow plugin-owned repair, while the markdown behavior needs a maintainer choice between converting supported Google Chat formatting and stripping unsupported agent Markdown syntax.
