## Summary

Keep open. Current main still has both reported Google Chat UX gaps by source inspection, and the issue is paired with an open PR that uses closing syntax but only addresses the typing-name fallback.

## What This Changes

The issue asks Google Chat to normalize Markdown-shaped outgoing text and to use the configured agent identity name in the typing indicator.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/49361 for the typing fallback, then add Google Chat-owned Markdown-to-Chat normalization for replies, action sends, captions, and updates with focused regression coverage.

## Reproduction Assessment

Yes. By source inspection, a Google Chat route with only `agents.list[].identity.name` configured still falls through to `OpenClaw`, and Markdown-shaped reply/action text is passed through the Google Chat send/update paths without conversion.

## Solution Assessment

Yes for the direction, but current main does not solve it. The narrow maintainable fix belongs in the Google Chat plugin delivery boundary, converting common Markdown to Google Chat text syntax rather than changing core channel behavior.
