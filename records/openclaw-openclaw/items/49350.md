## Summary

Keep open: current main and v2026.6.6 still miss `identity.name` in the Google Chat typing fallback, and the formatting half remains unresolved after the closed Markdown PR was challenged as the wrong direction.

## What This Changes

The issue asks for Google Chat replies to avoid raw Markdown syntax and for the typing indicator to use the configured agent display name instead of the OpenClaw fallback.

## Best Possible Solution

Land a focused Google Chat plugin fix that adds `identity.name` to the typing display-name fallback and routes outbound replies through a Google Chat-aware formatter aligned with the official Chat text syntax.

## Reproduction Assessment

Yes, source-level. The typing path calls `resolveBotDisplayName`, which skips `agents.list[].identity.name`, and the reply path sends Markdown chunks to Google Chat without a Chat-specific renderer; I did not run a live Google Chat workspace.

## Solution Assessment

No, not as originally phrased. Stripping all Markdown is probably not the best fix because Google Chat supports a small formatting syntax; the maintainable fix should convert supported Markdown/IR into Google Chat text formatting and leave unsupported constructs as readable plain text.
