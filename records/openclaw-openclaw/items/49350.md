## Summary

Keep open: current main and v2026.6.6 still have the Google Chat `identity.name` typing fallback gap and no Google Chat-aware outbound text renderer; the open fork PR only addresses typing, while the closed Markdown PR was challenged against Google Chat's supported formatting syntax.

## What This Changes

The issue requests Google Chat replies to render readable formatting instead of raw Markdown tokens and the typing placeholder to use the configured agent display name.

## Best Possible Solution

Land a focused Google Chat plugin fix that adds `identity.name` to the typing display-name fallback and applies a Google Chat-aware formatter aligned with the official text syntax.

## Reproduction Assessment

Yes, source-level. The typing path skips `agents.list[].identity.name`, and the reply/outbound paths send Markdown-mode chunks without a Google Chat-specific renderer; I did not run a live Google Chat workspace.

## Solution Assessment

No, not as originally phrased. Stripping all Markdown is probably not the best fix because Google Chat supports a small formatting syntax; the maintainable fix should convert supported syntax and leave unsupported constructs readable.
