Keep open: the PR direction is useful, and the contributor addressed the earlier transition-interleaving concerns, but the latest head still loses the new statuses through normalization, lacks required docs/changelog coverage, and exact-head CI is not green.

## What This Changes

This PR adds `awaiting_approval` and `waiting_external` task states, shared lifecycle/operational task formatting, CLI/auto-reply rendering updates, and focused task-state tests.

## Best Possible Solution

Make the intermediate statuses first-class anywhere task statuses are parsed, normalized, displayed, audited, documented, and announced, while preserving the current TaskFlow projection to `running`.

## Reproduction Assessment

Yes. On PR head, create or update a task with `awaiting_approval` or `waiting_external` through the registry create/run-scoped update paths; `normalizeTaskStatus` converts it to `queued` before storage or event handling.

## Solution Assessment

No, not yet. The feature direction is narrow and maintainable, but the implementation needs the normalizer fix, docs/changelog updates, and green exact-head validation before it is the best mergeable solution.
