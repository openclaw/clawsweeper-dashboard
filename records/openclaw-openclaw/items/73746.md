## Summary

Keep open. Current main does not implement the richer task states, and the PR is still useful, but PR head still drops the new statuses through the canonical status normalizer and lacks required docs/changelog updates before merge.

## What This Changes

The PR adds intermediate active task states, task lifecycle/operational formatting, CLI and auto-reply rendering changes, and focused task-state tests.

## Best Possible Solution

Make the intermediate statuses first-class anywhere task status values are parsed, normalized, displayed, audited, documented, and announced, while keeping TaskFlow compatibility projection to running.

## Reproduction Assessment

Yes. On PR head, create or update a task with status awaiting_approval or waiting_external through the public registry helpers; the value passes through normalizeTaskStatus and is converted to queued before storage/event handling.

## Solution Assessment

No, not as currently proposed. The product direction is narrow and maintainable, and the Greptile transition concerns were fixed, but the normalizer, docs, changelog, and green validation still need to land with the feature.
