## Summary

Keep open: this maintainer-labeled PR is a focused fix for direct `openclaw update` EACCES recovery output in containers. Current main still emits host npm/sudo guidance for that path, while the PR reuses the existing container detector, preserves host guidance outside containers, and adds a focused regression test with no blocking findings found.

## What This Changes

The PR changes `openclaw update` failure hint rendering so EACCES during global package update/install inside detected containers recommends image redeploy instead of host npm/sudo recovery, with focused tests.

## Best Possible Solution

Land this narrow CLI recovery-guidance fix if maintainers accept the container update direction, while leaving the broader gateway/UI suppression policy to https://github.com/openclaw/openclaw/pull/94310.

## Reproduction Assessment

Yes, source-level: current main sends npm-prefix and sudo recovery hints for npm global EACCES failures without checking container state. I did not execute the CLI or tests because this was a read-only review.

## Solution Assessment

Yes for the direct CLI EACCES path: reusing the existing container detector inside the failure-hint presentation helper is the narrowest maintainable fix and keeps host guidance unchanged. Broader update suggestion suppression remains a separate linked PR.
