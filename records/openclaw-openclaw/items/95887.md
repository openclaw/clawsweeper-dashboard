## Summary

Keep open: the PR addresses a real direct-CLI container EACCES guidance gap, but it should not merge yet because the new container branch can still append generic local npm recovery advice when the same failed stderr also mentions node-gyp or prebuild.

## What This Changes

The PR changes `openclaw update` failure hints so detected-container EACCES package update failures recommend image redeploy guidance while preserving host recovery hints and focused tests.

## Best Possible Solution

Keep the narrow CLI fix, but suppress all local npm recovery hints once a global package EACCES failure is classified as container-specific, and leave broader gateway/UI update policy to https://github.com/openclaw/openclaw/pull/94310.

## Reproduction Assessment

Yes, source-level: current main emits host npm/sudo EACCES recovery hints without container awareness, and PR head can still append the native optional dependency npm fallback when the same stderr includes EACCES plus node-gyp/prebuild text.

## Solution Assessment

No, not quite: branching at the presentation helper is the right narrow layer for direct CLI EACCES output, but the fix should stop the later generic npm fallback from reintroducing local package-update advice in the container EACCES case.
