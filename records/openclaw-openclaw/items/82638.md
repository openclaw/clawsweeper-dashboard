## Summary

Keep this PR open. Current `main` and the latest release still run implicit provider discovery before replace-mode handling, while the PR adds a narrow guard plus focused mode-matrix coverage; I found no blocking correctness or security finding, but the explicit-only replace-mode behavior is upgrade-visible and needs maintainer acceptance before landing.

## What This Changes

The PR adds an early return in agent model-provider planning so `models.mode: "replace"` returns explicit providers without running implicit discovery, with regression coverage for replace, merge, and default modes.

## Best Possible Solution

Land the narrow discovery guard with regression coverage after maintainer acceptance of the explicit-only replace-mode upgrade behavior.

## Reproduction Assessment

Yes. Current `main` awaits implicit provider discovery before replace-mode handling, and the linked bug plus PR proof describe the same path; I did not run local tests because this review is read-only.

## Solution Assessment

Yes. The guard is at the provider-gathering boundary, skips only the implicit resolver, and leaves downstream normalization, generated-catalog cleanup, and catalog writing intact.
