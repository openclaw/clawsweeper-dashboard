The PR adds an early replace-mode return in agent models.json provider planning and adds focused tests for replace, merge, and default merge behavior.

## Solution Assessment

Yes as an implementation. Guarding before implicit discovery is the narrowest fix and matches the model-picker precedent; the remaining decision is whether maintainers accept the compatibility effect for existing replace-mode users.
