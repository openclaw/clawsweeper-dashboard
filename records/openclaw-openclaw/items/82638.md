## Summary

Keep open for maintainer review: the patch appears correct and still necessary because current main and the latest release still enter implicit provider discovery before honoring replace mode, while this PR skips that work and preserves merge/default behavior.

## What This Changes

The PR adds an early return in agent models-config planning to skip implicit provider discovery when `models.mode` is `replace`, plus Vitest coverage for replace, merge, and default modes.

## Best Possible Solution

Land the early discovery guard with the mode-matrix regression test after maintainers accept the documented explicit-only replace-mode behavior.

## Reproduction Assessment

Yes, from source with high confidence: current main awaits implicit provider discovery before any replace-mode handling, and the PR comment includes production-path output showing the guarded branch skips discovery. I did not run the startup scenario in this read-only review.

## Solution Assessment

Yes. The guard sits at the provider-gathering boundary, before the expensive implicit resolver, while leaving explicit-provider normalization, secret handling, catalog filtering, and merge/default behavior on the existing downstream path.
