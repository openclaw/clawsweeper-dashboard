## Summary

Keep open: current main and v2026.6.10 still await implicit provider discovery before honoring replace mode, and this PR remains the active linked fix. The patch looks correct, but maintainers still need to accept the upgrade behavior change for replace-mode users before merge.

## What This Changes

Adds an early replace-mode return in `resolveProvidersForModelsJsonWithDeps` plus a Vitest mode matrix proving replace skips implicit provider discovery while merge/default still invoke it.

## Best Possible Solution

Land the early discovery guard with focused mode coverage once maintainers accept explicit-only replace-mode upgrade behavior and current merge validation is satisfactory.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows implicit discovery is awaited before replace-mode handling, and the PR proof shows the patched production resolver path performs zero discovery calls in replace mode.

## Solution Assessment

Yes as an implementation. Guarding before implicit discovery is the narrowest maintainable fix and matches the model-picker precedent; the remaining question is maintainer acceptance of the compatibility effect.
