Keep open. The PR is a narrow and likely correct fix for a source-proven replace-mode bug, but it is still draft and the supplied proof is only unit/mocked test output rather than after-fix real OpenClaw startup or log evidence.

## What This Changes

This PR adds an early return in `resolveProvidersForModelsJsonWithDeps` so replace mode merges only explicit providers, plus a Vitest file covering replace, merge, and default modes.

## Best Possible Solution

Land the narrow replace-mode discovery guard and regression test after the contributor adds redacted real startup/log proof and marks the PR ready for review; keep https://github.com/openclaw/openclaw/issues/66957 open until the fix merges.

Yes. A high-confidence source path exists on current main: `resolveProvidersForModelsJsonWithDeps()` awaits the implicit resolver before mode handling, and an injected resolver spy would be called even with `models.mode: "replace"`.

## Solution Assessment

Yes in direction. The early return before implicit discovery is the narrow maintainable fix and preserves merge/undefined behavior, but the PR still needs real behavior proof before merge.
