## Summary

Keep open: current main and the latest release still run implicit provider discovery before replace-mode handling, and this PR is the active fix candidate with sufficient production-path proof. The remaining blocker is maintainer acceptance of the compatibility change for existing replace-mode setups.

## What This Changes

The PR adds a replace-mode early return in models-config provider resolution and a Vitest mode matrix proving replace skips implicit discovery while merge/default still call it.

## Best Possible Solution

Land the early discovery guard with the mode-matrix coverage after maintainers accept explicit-only replace-mode upgrade behavior and exact-head checks are satisfactory.

## Solution Assessment

Yes, subject to maintainer compatibility acceptance. Guarding at the provider-gathering boundary is the narrowest maintainable fix because later planner mode handling would still pay the discovery cost, and the model picker already uses the same replace-mode short-circuit pattern.
