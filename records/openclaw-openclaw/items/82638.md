This PR adds an early replace-mode guard in models-config provider planning and a mode-matrix regression test so replace skips implicit discovery while merge/default still discover providers.

## Best Possible Solution

Yes. The early return is the narrowest maintainable fix because it skips only implicit discovery while leaving explicit-provider normalization, secret handling, and catalog writing on the existing downstream path.
