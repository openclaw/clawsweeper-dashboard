## Summary

Keep open. Current main still has source paths that match the reported Codex app-server lifecycle mismatch: active-run recovery is guarded as observe-only by default, Codex can record model completion before an interrupted terminal event, and gateway session persistence still maps aborted terminal events without an aborted stop reason to timeout.

## What This Changes

The issue requests native Codex harness hardening so app-server close/timeouts clear active runs deterministically, align trajectory and persisted session state, preserve TUI responsiveness, and expose effective service-tier evidence.

## Best Possible Solution

Add targeted native Codex harness lifecycle hardening so app-server close/timeout reaches exactly one terminal outcome, clears active run and lane state after bounded cleanup, keeps trajectory and persisted session status aligned, and leaves the TUI usable under gateway backlog.
