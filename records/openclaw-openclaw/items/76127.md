## Summary

Keep open. Current main has related Codex app-server startup retry and TUI stale-stream guards, but the code still has paths matching the report: diagnostic recovery observes active embedded runs by default, Codex completion can be recorded before an interrupted session end, and aborted lifecycle ends without an explicit aborted stop reason still persist as timeout.

## What This Changes

The issue asks OpenClaw to make native Codex harness app-server close/timeout cases finish deterministically, clear active run state, keep persisted session and trajectory status aligned, and keep the TUI responsive.

## Best Possible Solution

Harden the native Codex harness lifecycle so app-server close/timeout reaches exactly one terminal outcome, clears active run and lane state after bounded cleanup, persists matching session status, and leaves TUI recovery as a secondary UI-state safeguard.
