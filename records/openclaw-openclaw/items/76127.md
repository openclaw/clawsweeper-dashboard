## Summary

Keep open. Current main still contains the source paths that match the report: active Codex runs are observe-only for stuck-session recovery by default, Codex can record a model completion and then an interrupted session end, and aborted lifecycle ends without a stop reason persist as timeout.

## What This Changes

The issue asks OpenClaw to make native Codex app-server turns terminate consistently after app-server close or timeout and keep TUI sessions responsive.

## Best Possible Solution

Harden the native Codex app-server lifecycle so close, timeout, and late completion converge on one terminal outcome, clear active run/lane state after bounded cleanup, and leave persisted session status consistent with trajectory events.

## Reproduction Assessment

Unclear/source-reproducible: the issue has detailed live logs, and current source can produce the observe-only recovery, completion/interrupted split, and timeout persistence. I did not establish a deterministic current-main live reproduction for the full TUI keyboard freeze.

## Solution Assessment

Unclear: there is no proposed patch yet. The maintainable direction should start at the Codex app-server/session lifecycle and generic active-run recovery policy before treating this as only a TUI reset problem.
