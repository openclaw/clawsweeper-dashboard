## Summary

Keep open: the branch addresses the valid QQBot data-directory bug from #39461, but it needs a small repair before merge because whitespace-only `QQBOT_DATA_DIR` now resolves storage relative to the process cwd and the required changelog entry is missing.

## What This Changes

The branch updates QQBot storage helpers to read `process.env.QQBOT_DATA_DIR` and documents the env var in the QQBot channel docs.

## Best Possible Solution

Honor `QQBOT_DATA_DIR` in the QQBot storage helper with normalized empty fallback, preserve default and lazy creation behavior, add focused regression coverage, and add the required Unreleased changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows diagnostics recommends `QQBOT_DATA_DIR` while the storage helper always resolves under `getHomeDir()/.openclaw/qqbot`, so setting the env var cannot affect diagnostics or persistence paths.

## Solution Assessment

No, not yet. Centralizing the override in the QQBot storage helper is the right direction, but the patch should normalize before deciding the override is present, add regression coverage, and include the required changelog entry.
