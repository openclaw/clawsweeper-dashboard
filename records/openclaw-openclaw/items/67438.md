## Summary

Keep open. Current main still routes Feishu scan-to-create through compact terminal QR output, the documented setup path still asks users to scan that QR, and the larger-QR PR #67450 was closed unmerged while #72762 is only a partial WSL/font guidance candidate.

## What This Changes

Feishu QR codes shown during OpenClaw onboarding should be scannable with the Feishu mobile app on Windows.

## Best Possible Solution

Fix this in the Feishu plugin setup path by making scan-to-create QR output reliably scannable, ideally with a larger terminal QR plus an openable URL or PNG fallback when terminal rendering is unreliable, and add focused regression coverage for the render option.

## Reproduction Assessment

Unclear for the full end-to-end failure because it needs Windows 10 and the Feishu mobile app. The source-level reproduction path is clear: `openclaw onboard` or `openclaw channels login --channel feishu` enters Feishu scan-to-create and prints a compact QR via `printQrCode()`.

## Solution Assessment

No. Current main is not a solution, #67450 was closed unmerged, and #72762 appears to address WSL font readability rather than the compact Feishu QR scan reliability reported from a Windows command window.
