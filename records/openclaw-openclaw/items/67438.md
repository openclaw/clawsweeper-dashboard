## Summary

Keep open. Current main and latest release still route Feishu scan-to-create through compact terminal QR output, the docs still tell users to scan that QR, the larger-QR fix PR was closed unmerged, and the open WSL-font PR is only a partial overlapping candidate.

## What This Changes

Feishu onboarding QR codes should be scannable from `openclaw onboard` or `openclaw channels login --channel feishu` so scan-to-create can complete on Windows.

## Best Possible Solution

Fix the Feishu plugin setup path so scan-to-create emits reliably scannable QR output, with focused regression coverage and docs updates if a URL or image fallback becomes user-visible.

## Reproduction Assessment

Unclear for the full end-to-end issue because it requires Windows 10 and the Feishu/Lark mobile app. The source-level path is clear: `openclaw onboard` or `openclaw channels login --channel feishu` enters Feishu scan-to-create and prints a compact QR via `printQrCode()`.

## Solution Assessment

No. Current main is not a fix, PR #67450 was closed unmerged, and PR #72762 adds WSL guidance without changing the compact Feishu QR output reported from a Windows command window.
