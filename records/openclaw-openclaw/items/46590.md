## Summary

Keep open. Current main still only supports Gateway-owned stored cron jobs with optional per-job agent binding; the requested agent-level declarative cron config remains a real feature/config design request, not cleanup.

## What This Changes

Add an `agents.list[].cron` configuration field so scheduled tasks can be declared with an agent instead of only in the Gateway cron store.

## Best Possible Solution

If maintainers accept the direction, add a documented additive agent-level schedule contract that compiles into Gateway cron with deterministic namespacing, validation, precedence, reload semantics, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence source check is that current main lacks an `agents.list[].cron` type, strict schema entry, startup loader, and public docs contract.

## Solution Assessment

Unclear pending maintainer product review. The requested direction is coherent, but the maintainable solution is a deliberate config and scheduler contract rather than a narrow bug fix.
