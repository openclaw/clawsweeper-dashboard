## Summary

Keep open. Current main still models cron as Gateway-owned stored jobs and keeps per-agent configuration strict without an `agents.list[].cron` contract, so this is valid core config/scheduler product work rather than stale cleanup, ClawHub-only work, or implemented behavior.

## What This Changes

The request is to add declarative per-agent cron entries under `agents.list[]` and compile them into scheduled Gateway jobs while preserving existing stored cron jobs.

## Best Possible Solution

If maintainers accept the direction, add an additive core contract for agent-owned schedules: typed `agents.list[].cron` entries, generated schema/help/docs coverage, deterministic compilation into scheduler jobs on startup and config reload, documented namespacing and stored-job precedence, and focused tests for validation, duplicates, inheritance, delivery, reload, and doctor behavior.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence: inspect `AgentConfig`, `AgentEntrySchema`, Gateway cron startup, cron store types, plugin hook surfaces, and public cron/config docs to confirm `agents.list[].cron` is not defined or loaded.

## Solution Assessment

Unclear pending maintainer product review. The requested direction is coherent, but the best maintainable solution is a deliberate core config and scheduler contract rather than a small automated patch, because it changes scheduled execution authority, validation, reload, precedence, and security semantics.
