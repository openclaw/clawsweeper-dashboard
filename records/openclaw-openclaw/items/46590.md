## Summary

The source evidence still points to a real missing feature: agent config has no declarative schedule field, cron remains a top-level stored-job subsystem, and the closest related cron ownership/provenance issues are adjacent rather than a complete substitute.

## What This Changes

Add an `agents.list[].cron` configuration contract so scheduled tasks can be declared with and owned by an agent.

## Best Possible Solution

Define an additive agent-level schedule contract that compiles into Gateway cron with deterministic IDs, validation, precedence, reload or migration semantics, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main shows no agent-level cron type, strict schema entry, startup loader, or public docs contract.

## Solution Assessment

Unclear until maintainers choose the product contract. The direction is coherent, but implementation should be a designed config/scheduler contract rather than only adding a schema field.
