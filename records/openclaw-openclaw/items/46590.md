## Summary

Keep open: current main still has only top-level/stored cron jobs plus per-job `agentId` routing, not an `agents.list[].cron` configuration contract; related ownership and provenance issues are adjacent but do not supersede this declarative config request.

## What This Changes

The issue requests a new `cron` array on each agent config entry so scheduled tasks can be declared with the agent that owns them.

## Best Possible Solution

Define an additive agent-level schedule contract that compiles into Gateway cron with deterministic IDs, validation, inheritance, precedence, reload/migration semantics, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main shows no agent-level cron type, strict schema entry, startup loader, or public docs contract.

## Solution Assessment

Unclear until maintainers choose the product contract. The direction is coherent, but the maintainable solution should define a scheduler/config contract rather than only adding a schema field.
