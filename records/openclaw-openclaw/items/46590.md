## Summary

Keep open. Current main and v2026.6.6 still lack an `agents.list[].cron` field, and cron is a Gateway-owned SQLite scheduler today, so declarative agent-owned scheduled tasks remain a product/API decision rather than conservative cleanup.

## What This Changes

The issue requests a declarative `agents.list[].cron` configuration field so scheduled tasks can be colocated with agent settings and loaded into Gateway cron on startup.

## Best Possible Solution

Keep this open for maintainer product review and design one canonical scheduler model with explicit ownership, reconciliation, migration, precedence, docs, and tests before adding a new agent config surface.
