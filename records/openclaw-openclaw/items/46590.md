## Summary

Keep open. Current main still supports Gateway-owned cron storage plus per-job agent binding, but it has no `agents.list[].cron` config, schema, docs, or startup loader, so this remains valid core scheduler/config product work rather than cleanup.

## What This Changes

Request add an `agents.list[].cron` field for declarative, agent-owned scheduled tasks that merge with existing stored cron jobs.

## Best Possible Solution

If maintainers accept the direction, add an additive `agents.list[].cron` schema/help/docs contract and compile those declarations into Gateway cron with deterministic namespacing, stored-job precedence, reload handling, validation, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence source check is that the agent config type/schema, Gateway cron startup, cron store shape, and public docs still lack an `agents.list[].cron` loader or contract.

## Solution Assessment
