## Summary

Keep open. Current main still supports Gateway-owned cron job storage plus per-job agent binding, while `agents.list[]` has no `cron` contract, so this remains valid core config and scheduler product work that needs maintainer design.

## What This Changes

Request adds a declarative `cron` array to `agents.list[]` so agent-owned schedules can be loaded by Gateway cron alongside `jobs.json`.

## Best Possible Solution

If maintainers accept the direction, add an additive `agents.list[].cron` schema/help/docs contract and compile those declarations into Gateway cron on startup and reload with deterministic namespacing, stored-job precedence, security validation, doctor behavior, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence source check is that `AgentConfig`, strict `AgentEntrySchema`, Gateway cron startup, cron store types, and public docs still have no `agents.list[].cron` loader or contract.

## Solution Assessment

Unclear pending maintainer product review. The requested direction is coherent, but the maintainable solution is a deliberate core config and scheduler contract rather than a narrow bug fix.
