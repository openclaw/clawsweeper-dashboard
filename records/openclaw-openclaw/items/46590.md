## Summary

Keep open. Current main does not implement an `agents.list[].cron` field, and the remaining work is a product/API decision because cron is now a Gateway-owned SQLite scheduler while the requested declarative agent config would add a second scheduling source of truth.

## What This Changes

The issue requests a declarative `cron` field on each configured agent so scheduled tasks can live with agent configuration instead of only in the central Gateway cron scheduler.

## Best Possible Solution

Keep this open for maintainer product review and design it together with cron ownership/isolation; if accepted, the end state should be one canonical scheduler model with explicit config reconciliation, migration, precedence, docs, and tests.
