## Summary

Keeping this open: current main still has only the top-level cron store and CLI/job-file management model, while the requested agent-owned declarative schedule contract would add a new config surface with precedence, reload, and ownership semantics that need maintainer product review.

## What This Changes

Request to add an `agents.list[].cron` configuration field so scheduled tasks can be declared with and inherited from each agent configuration.

## Best Possible Solution

Define an additive agent-level schedule contract that compiles into Gateway cron with deterministic IDs, strict validation, documented precedence, reload and migration behavior, and regression coverage across config, Gateway startup, scheduler service, UI/CLI expectations, and cron tools.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main shows no agent-level cron type, strict schema entry, startup loader, or public docs contract.

## Solution Assessment

Unclear until maintainers choose the product contract. The direction is coherent, but the maintainable fix should be a designed config/scheduler contract rather than only adding a schema field.
