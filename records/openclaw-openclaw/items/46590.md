## Summary

The requested declarative agent-owned scheduling capability is still absent, but the proposed `agents.list[].cron` plus legacy-file precedence model is outdated and would conflict with the current single SQLite scheduler owner; this needs a maintainer product and reconciliation decision.

## What This Changes

Requests declarative per-agent cron task definitions inside `agents.list[]`, merged with centrally managed scheduler jobs.

## Best Possible Solution

Design an explicit declarative import/reconcile or exportable schedule-bundle workflow that writes agent-bound jobs into the canonical SQLite cron store with stable ownership, deletion, conflict, and inheritance semantics.

## Reproduction Assessment

## Solution Assessment

No: the proposed `jobs.json` precedence merge is no longer the best solution after cron moved to SQLite; a single-owner reconcile/import design would better preserve canonical state and upgrade safety.
