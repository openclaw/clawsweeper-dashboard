## Summary

Keep open: current main still has no `agents.list[].cron` contract, and the adjacent cron ownership/provenance issues do not fully cover this declarative agent-config request.

## What This Changes

Add a declarative `cron` array to `agents.list[]` so scheduled tasks can be owned by agent configuration instead of only the central cron job store.

## Best Possible Solution

Define an additive agent-level schedule contract that compiles into Gateway cron with deterministic namespacing, validation, precedence, reload/migration semantics, docs, and regression coverage.
