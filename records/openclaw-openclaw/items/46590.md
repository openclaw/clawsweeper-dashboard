## Summary

Keep open. Current main and the latest release do not implement `agents.list[].cron`, and adding it would be a new product/config surface that needs maintainer direction around scheduler ownership, reconciliation, precedence, and migration.

## What This Changes

The issue requests a declarative `cron` field on each `agents.list[]` entry for agent-owned scheduled tasks.

## Best Possible Solution

Decide the canonical cron ownership model first, then implement declarative agent-owned jobs only if they reconcile cleanly with the SQLite scheduler, doctor migrations, CLI/RPC edits, docs, and tests.
