## Summary

Keep open. A fresh current-main review confirms the requested `agents.list[].cron` contract is still absent, while existing cron remains Gateway-owned with per-job `agentId` binding; this is valid feature/config product work, not cleanup.

## What This Changes

The issue requests an additive `agents.list[].cron` configuration field so scheduled tasks can be declared with their owning agent instead of only in the central Gateway cron store.

## Best Possible Solution

If maintainers accept the direction, add a documented additive agent-level schedule contract that compiles into Gateway cron with deterministic namespacing, validation, precedence, reload behavior, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence source check is that current main still lacks an `agents.list[].cron` type, strict schema entry, startup loader, and public docs contract.
