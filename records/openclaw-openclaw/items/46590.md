## Summary

Keep open. Current main still has Gateway-owned stored cron jobs and a strict per-agent config schema without an `agents.list[].cron` contract, so this remains valid core config/scheduler product work rather than implemented cleanup or ClawHub-only work.

## What This Changes

The issue requests an additive `agents.list[].cron` configuration field so scheduled tasks can be owned declaratively by an agent instead of only by the Gateway cron store.

## Best Possible Solution

If maintainers accept the direction, add an additive core contract for agent-owned schedules with typed schema/help/docs, deterministic compilation into Gateway cron jobs on startup and reload, documented namespacing and stored-job precedence, and focused validation/behavior tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence: inspect `AgentConfig`, strict `AgentEntrySchema`, Gateway cron startup, cron store types, and public cron/config docs to confirm `agents.list[].cron` is not defined or loaded.

## Solution Assessment

Unclear pending maintainer product review. The requested direction is coherent, but the maintainable fix is a deliberate core config and scheduler contract rather than a narrow automated patch because it changes scheduled execution authority, reload behavior, validation, precedence, and security semantics.
