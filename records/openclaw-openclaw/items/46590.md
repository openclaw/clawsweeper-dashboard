## Summary

Keep open: current main still has no agent-level cron declaration in `agents.list[]`, and the related open cron ownership issue at https://github.com/openclaw/openclaw/issues/26370 complements this request rather than superseding the declarative config contract.

## What This Changes

The issue requests an additive `agents.list[].cron` config field so scheduled tasks can be declared with the agent that owns them instead of only through the global cron job store and CLI.

## Best Possible Solution

Design one additive agent-level schedule contract that compiles into Gateway cron with deterministic IDs, strict validation, documented precedence/reload/migration behavior, and a clear relationship to the open cron ownership/isolation work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main shows no agent-level cron type, strict schema entry, startup loader, or public docs contract for `agents.list[].cron`.

## Solution Assessment

Unclear until maintainers choose the product contract. The direction is coherent, but the maintainable fix should define ownership, precedence, reload, migration, and admin behavior rather than only adding a schema field.
