## Summary

Keep open. Current main still has Gateway-owned cron storage and per-job `agentId` binding, but no `agents.list[].cron` type, schema, docs, or loader, so this remains valid core scheduler/config product work rather than conservative cleanup.

## What This Changes

Add a declarative `cron` array under `agents.list[]` so scheduled tasks can live with the owning agent configuration.

## Best Possible Solution

If maintainers accept the direction, add an additive `agents.list[].cron` contract that compiles declarations into Gateway cron with deterministic namespacing, stored-job precedence, reload behavior, validation, docs, and regression tests.
