## Summary

Keep open: current main and the latest release still have no `agents.list[].cron` declaration surface. The related open cron ownership issue complements this request but does not supersede the declarative agent config contract, so this needs maintainer product design rather than cleanup closure.

## What This Changes

Add an `agents.list[].cron` field so scheduled tasks can be declared with their owning agent configuration instead of only in the Gateway cron job store.

## Best Possible Solution

Design one additive agent-level schedule contract that compiles into Gateway cron with deterministic IDs, strict validation, documented precedence/reload/migration behavior, and a clear relationship to the open cron ownership/isolation work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main and `v2026.5.22` shows no agent-level cron type, strict schema entry, startup loader, or public docs contract for `agents.list[].cron`.

## Solution Assessment

Unclear until maintainers choose the product contract. The direction is coherent, but the maintainable fix should define ownership, precedence, reload, migration, and admin behavior rather than only adding a schema field.
