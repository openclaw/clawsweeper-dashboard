## Summary

Keep open. Current main has adjacent grounding guidance, inbound provenance, memory citations, trajectory export, task records, and completion summaries, but it still does not implement the RFC's requested per-claim confidence tiers, output-wide source attribution, known-versus-inferred semantics, or autonomous-mode reporting for unverified claims.

## What This Changes

Request to add an agent truthfulness contract with per-claim confidence tiers, source attribution, known-vs-inferred semantics, and autonomous-mode reporting for unverified claims.

## Best Possible Solution

Define a versioned truthfulness contract for agent outputs, including claim boundaries, confidence tiers, evidence/source kinds, and unverified-claim reporting, then propagate the smallest enforceable slice through runtime events, Gateway protocol schemas, task records, trajectory exports, completion reports, tests, and docs.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug report. The high-confidence check is current-main source and docs inspection, which shows adjacent mechanisms but no implemented per-claim confidence or unverified-claim contract.

## Solution Assessment
