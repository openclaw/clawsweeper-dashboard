## Summary

Keep open. Current main has adjacent grounding, provenance, trajectory, and runtime-truthfulness mechanisms, but it still does not implement the RFC's requested per-claim confidence tiers, output source attribution, known-versus-inferred semantics, or autonomous-mode unverified-claim reporting contract.

## What This Changes

The issue requests structured truthfulness metadata for agent output, including confidence tiers, source attribution, known-versus-inferred claim semantics, and autonomous-mode reporting for unverified claims.

## Best Possible Solution

Define a versioned truthfulness contract for agent outputs, including claim boundaries, confidence tiers, evidence/source kinds, and unverified-claim reporting, then thread the smallest enforceable slice through runtime events, Gateway protocol schemas, task records, trajectory exports, completion reports, tests, docs, and any client UI that renders autonomous completions.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug report. The high-confidence check is current-main source and docs inspection, which shows adjacent grounding/provenance mechanisms but no implemented per-claim confidence or unverified-claim contract.

## Solution Assessment

Unclear: the direction is reasonable, but the best implementation boundary has not been designed. A maintainable solution likely needs a protocol/runtime/reporting design first, not only prompt wording.
