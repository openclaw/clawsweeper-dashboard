## Summary

Keep open. The prior ClawSweeper review remains accurate on current main: OpenClaw has adjacent evidence guidance, input provenance, runtime-truthfulness work, memory-wiki claim metadata, and structured task completion events, but no enforceable per-claim confidence/source contract or unverified-claim reporting for autonomous completions.

## What This Changes

Request to add explicit confidence tiers, source attribution, known-versus-inferred semantics, and autonomous-mode reporting for factual claims in agent output.

## Best Possible Solution

Define a versioned truthfulness contract for assistant factual claims, then thread it through prompting, runtime/protocol payloads, persisted task/completion records, reporting UX, tests, and docs.

## Reproduction Assessment

Not applicable: this is an RFC, not a concrete bug. The high-confidence check is current-main inspection of prompt, internal-event, gateway protocol, task record, completion/reporting, and memory-wiki surfaces.

## Solution Assessment

Unclear: the requested direction is reasonable, but prompt wording alone would not be enforceable. A protocol/runtime/reporting contract is the safer maintainable path and needs maintainer product/API design first.
