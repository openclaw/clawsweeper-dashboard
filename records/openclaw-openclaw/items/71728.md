## Summary

Keep open. Current main has adjacent prompt guidance, runtime truthfulness work, input provenance, structured task-completion events, task records, and trajectory capture, but it still does not implement the RFC's requested per-claim confidence tiers, per-claim source attribution, known-versus-inferred semantics, or autonomous-mode unverified-claim reporting.

## What This Changes

Request to add agent-output truthfulness metadata: confidence tiers, source attribution, known-versus-inferred distinctions, and unverified-claim reporting for autonomous completions.

## Best Possible Solution

Define a versioned truthfulness contract for agent output claims, then implement the smallest enforceable slice through agent prompting, runtime events, Gateway protocol schemas, task/completion records, trajectory exports, user-facing completion summaries, tests, and docs.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug. The high-confidence check is current-main source, schema, test, and docs inspection showing adjacent grounding/provenance mechanisms but no implemented per-claim confidence or unverified-claim contract.

## Solution Assessment
