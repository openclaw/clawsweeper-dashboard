## Summary

Keep open. Current main has advisory evidence prompts, input provenance, internal completion events, task summaries, trajectory bundles, and memory-wiki claim metadata, but it still lacks the requested per-claim confidence tiers, source attribution, known-versus-inferred semantics, and autonomous unverified-claim reporting for agent output.

## What This Changes

Requests a core truthfulness contract with per-claim confidence tiers, source attribution, known/inferred distinctions, and autonomous unverified-claim reporting.

## Best Possible Solution

Define a versioned, additive truthfulness contract for assistant output claims, then thread the smallest enforceable slice through runtime events, protocol schemas, persisted task/completion records, reporting UI/CLI, tests, and docs.

## Reproduction Assessment

Not applicable: this is an RFC, not a concrete bug. The high-confidence check is current-main source and docs inspection across the prompt, protocol, task, provenance, trajectory, and memory surfaces that would need to carry the contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but the exact per-claim extraction and presentation model needs maintainer product/API design. Prompt guidance alone is not the best solution; a protocol/runtime/reporting contract is safer.
