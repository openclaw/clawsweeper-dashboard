## Summary

Keep open. Current main has adjacent evidence, provenance, memory-wiki claim metadata, and background-completion guardrails, but it still lacks the requested per-output-claim confidence tiers, source attribution, known-versus-inferred semantics, and autonomous unverified-claim reporting contract.

## What This Changes

Request a core agent truthfulness contract that labels factual claims by confidence, source, and verification status, especially for autonomous completion reports.

## Best Possible Solution

Define a versioned truthfulness contract for assistant output claims, then land the smallest enforceable slice through runtime events, gateway protocol schemas, stored task/completion records, user-visible reports, tests, and docs.
