## Summary

Keep open. Current main has advisory grounding instructions, inbound provenance, background-task summaries, and trajectory exports, but it still does not implement the RFC's requested per-claim confidence tiers, output-wide source attribution, known-versus-inferred semantics, or autonomous-mode reporting for unverified claims.

## What This Changes

The RFC requests structured confidence tiers, source attribution, known/inferred/assumed distinctions, and autonomous-mode reporting for unverified factual claims in agent output.

## Best Possible Solution

Define a versioned truthfulness contract for agent outputs, then propagate the smallest enforceable slice through agent runtime events, Gateway protocol schemas, task records, trajectory/export data, completion reports, tests, docs, and any client UI that needs to surface unverified claims.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug report. The high-confidence check is current-main source and docs inspection, which shows adjacent grounding and provenance mechanisms but no implemented per-claim confidence or unverified-claim contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but the best implementation boundary still needs maintainer design. A maintainable fix should start with a protocol/runtime/reporting contract, not only additional prompt wording.
