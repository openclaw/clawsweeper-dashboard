## Summary

Keep open. Current main has adjacent grounding guidance, input provenance, memory-wiki claim metadata, trajectory export, and runtime truthfulness work for specific failure modes, but it still does not implement this RFC's per-output-claim confidence tiers, per-claim source attribution, known-versus-inferred semantics, or autonomous unverified-claim reporting.

## What This Changes

The issue requests a verifiable trust contract for agent output: confidence tiers on factual claims, source attribution, known/inferred wording, and separate autonomous-mode reporting for unverified claims.

## Best Possible Solution

Define a small, versioned truthfulness contract for assistant output claims, then thread it through the runtime and public protocol surfaces that consume autonomous completions. The first maintainable slice should prove one end-to-end path with schema, storage/export, user-visible reporting, tests, and docs before expanding to every output surface.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug. The high-confidence check is current-main source/docs inspection showing adjacent provenance and grounding mechanisms but no implemented per-claim truthfulness contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but the best implementation boundary needs maintainer product/API design first. A protocol/runtime/reporting contract would be safer than prompt wording alone.
