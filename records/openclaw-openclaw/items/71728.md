## Summary

Keep open. Current main has adjacent grounding prompts, inbound provenance, memory citations, memory-wiki claim metadata, trajectory export, and task completion summaries, but it still does not implement the RFC’s per-factual-claim confidence tiers, output-wide source attribution, known-versus-inferred semantics, or autonomous reporting for unverified claims.

## What This Changes

The issue requests a runtime/protocol trust contract that labels factual claims as known, believed, uncertain, or unknown, attributes their sources, distinguishes inference from knowledge, and surfaces unverified claims in autonomous completion reports.

## Best Possible Solution

Define a versioned truthfulness contract across agent runtime, task records, Gateway protocol, trajectory/export data, memory/tool attribution, and completion-report UX, then implement the smallest enforceable slice with schema, runtime, and docs coverage after maintainers choose the semantics.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug report. The high-confidence check is current-main source/docs inspection, which shows adjacent mechanisms but no implemented per-claim confidence or unverified-claim contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but the best boundary is not designed yet. A prompt-only change would be weaker than the requested verifiable trust contract; this needs protocol/runtime/reporting design first.
