## Summary

Keep open. Current main still has adjacent evidence/provenance guidance, runtime truthfulness work for permission/runtime failures, internal completion events, and task summaries, but not the requested structured per-claim confidence/source-attribution contract or autonomous unverified-claim reporting.

## What This Changes

The issue requests confidence tiers, source attribution, known-versus-inferred wording, and autonomous-mode reporting for unverified factual claims in agent output.

## Best Possible Solution

Define a versioned truthfulness contract for assistant output claims, then implement the smallest enforceable slice through runtime events, protocol schemas, stored completion/task records, user-facing reports, tests, and docs.

## Reproduction Assessment

Not applicable: this is an RFC for a new trust/reporting capability, not a broken existing behavior. The high-confidence check is source inspection showing adjacent guidance and provenance fields but no implemented per-claim contract.

## Solution Assessment

Unclear: the direction is reasonable, but prompt wording alone would be too weak. A protocol/runtime/reporting contract is the safer maintainable path, and that needs maintainer product/API design first.
