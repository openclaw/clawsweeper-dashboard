## Summary

Keep open. Current main still has adjacent grounding guidance, provenance for inbound/session data, runtime trajectory capture, and structured task completion events, but it does not implement the RFC's requested per-claim confidence tiers, source attribution, known-versus-inferred semantics, or autonomous-mode unverified-claim reporting.

## What This Changes

The issue requests a structured agent truthfulness contract with confidence tiers, source attribution, known-versus-inferred labeling, and autonomous-mode reporting of unverified claims.

## Best Possible Solution

Define a versioned truthfulness contract for agent output claims, then thread the smallest enforceable slice through agent prompting, runtime events, Gateway protocol schemas, task/completion records, trajectory exports, user-facing completion summaries, tests, and docs.
