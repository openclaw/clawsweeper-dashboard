## Summary

Keep open. Current main has adjacent evidence guidance, input provenance, runtime-truthfulness notes, memory-wiki claim metadata, and structured task-completion events, but it still does not implement per-claim confidence tiers, per-claim source attribution, known-versus-inferred semantics, or autonomous unverified-claim reporting.

## What This Changes

The RFC asks OpenClaw to tag factual claims with confidence tiers and source attribution, distinguish known from inferred statements, and surface unverified claims in autonomous completion reports.

## Best Possible Solution

Define a versioned truthfulness contract for assistant output claims, then implement the smallest enforceable slice through runtime events, protocol schemas, stored task/completion records, completion summaries, tests, and docs.

## Reproduction Assessment

Not applicable: this is an RFC, not a concrete bug. The high-confidence check is current-main source and docs inspection showing adjacent provenance/truthfulness mechanisms but no per-claim truthfulness contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but prompt wording alone would be too weak. A protocol/runtime/reporting contract is the safer maintainable path, and that needs maintainer product/API design first.
