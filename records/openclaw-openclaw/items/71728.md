## Summary

Keep open. Current main has adjacent grounding guidance, input provenance, structured task-completion events, task records, trajectory capture, and memory-wiki claim metadata, but it still does not implement the RFC’s per-claim confidence tiers, per-claim source attribution, known-versus-inferred semantics, or autonomous unverified-claim reporting.

## What This Changes

The issue requests RFC-level agent truthfulness support: per-claim confidence tiers, source attribution, known-versus-inferred labels, and autonomous completion reporting for unverified claims.

## Best Possible Solution

Define a versioned truthfulness contract for assistant output claims, then implement the smallest enforceable slice through agent prompting, runtime events, Gateway protocol schemas, task/completion records, trajectory exports, user-facing completion summaries, tests, and docs.
