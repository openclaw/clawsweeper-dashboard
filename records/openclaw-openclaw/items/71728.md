## Summary

Keep open. Current main has adjacent prompt guidance, provenance plumbing, task summaries, and runtime tool-error warnings, but it still does not define the requested structured per-claim confidence/source-attribution contract or autonomous unverified-claim reporting surface.

## What This Changes

The issue requests a structured truthfulness contract that marks factual claims as known, believed, uncertain, or unknown with source attribution and autonomous-mode reporting for unverified claims.

## Best Possible Solution

Define a versioned truthfulness contract for assistant claims, then thread the smallest enforceable slice through runtime events, gateway schemas, persisted task summaries, user-facing completion reports, tests, and docs.

## Reproduction Assessment

Not applicable: this is an RFC for a new trust/reporting capability, not broken existing behavior. The high-confidence check is source inspection showing adjacent guidance and provenance fields but no implemented per-claim contract.

## Solution Assessment

Unclear: the direction is reasonable, but the concrete contract and UX need maintainer product/API design. A protocol/runtime/reporting addition would be more maintainable than relying on prompt wording alone.
