## Summary

Keep open: the packaged image-ops sharp-resolution bug is valid and this PR has useful real runtime proof, but the current patch still makes a native image optimizer mandatory and carries unrelated duplicate changelog entries.

## What This Changes

The PR adds `sharp` to the root package dependency graph, updates the lockfile and root dependency ownership allowlists, and adds changelog text for packaged image operations.

## Best Possible Solution

Ship one canonical package-level fix that declares `sharp` as a root `optionalDependency`, keeps lockfile and ownership checks aligned, removes duplicate changelog churn, and reruns the dependency/package gates.

## Reproduction Assessment

Yes. Current main is source-reproducible: the root package excludes nested bundled plugin `node_modules`, core loads the bundled media image-ops artifact, that artifact imports `sharp`, and root package metadata does not declare `sharp`; I did not run a fresh global install in this read-only pass.

## Solution Assessment

No as submitted. Declaring `sharp` at the root is the right package boundary, but it should be an `optionalDependency` to preserve existing graceful-degradation behavior, and the changelog should contain only the media fix entry.

## Review Findings

Overall correctness: patch is incorrect
