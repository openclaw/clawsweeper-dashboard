## Summary

Keep open: the upstream model/deprecation change is real and the contributor supplied sufficient terminal proof, but the patch registers the quality model under the existing xAI image provider capabilities that still advertise and validate up to five reference images while xAI's current multi-image editing contract says three.

## What This Changes

The PR adds `grok-imagine-image-quality` to the xAI image model list, marks `grok-imagine-image-pro` deprecated in docs/code comments, and updates the focused provider model-list test.

## Best Possible Solution

Land the xAI model-catalog update with provider capabilities, docs, and tests aligned to xAI's current quality-model editing limits, while keeping pro available only as a deprecated retiring id until removal is coordinated.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: after the PR, `grok-imagine-image-quality` is exposed through the xAI provider's shared `maxInputImages: 5` capability, while official xAI docs say multi-image editing uses up to three source images.

## Solution Assessment

Is this the best way to solve the issue? No; adding the model id in the owning xAI plugin is the right boundary, but the patch should also update the xAI edit cap/docs/test to three or add model-specific validation before merge.

## Review Findings
