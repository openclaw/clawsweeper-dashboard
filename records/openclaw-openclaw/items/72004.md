## Summary

Keep open. Current main still has the Control UI render race, while this PR applies a narrow ordering fix with focused regression coverage, maintainer approval, and green public checks; it should proceed through normal maintainer landing rather than cleanup closure.

## What This Changes

The PR reorders Control UI `handleChatEvent` final and aborted handling to clear live stream/run state before appending assistant messages, adds an order-sensitive unit test, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land this PR, or an equivalent narrow controller patch, so active-run terminal events clear live stream/run state before committing final or aborted assistant messages while preserving streamed fallback behavior. Broader transcript or delivery-mirror duplication should remain tracked separately.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to start with an active run and non-null `chatStream`, dispatch a `final` or `aborted` event, and observe current main writes `chatMessages` while `chatStream` is still non-null; `buildChatItems` can then render both the committed message and live stream item.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it only changes intermediate mutation order, preserves final state and fallback semantics, and avoids broad gateway or transcript dedupe changes that belong to separate issues.
