## Summary

Keep open. Current main still has the Control UI terminal-event ordering that this PR targets, and the PR is a narrow, correct fix with regression coverage, approval, and successful check-run metadata rather than something already implemented or superseded.

## What This Changes

The PR reorders Control UI `handleChatEvent` final/aborted branches to clear live stream/run state before appending assistant messages, adds an order-sensitive unit test, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land this PR, or an equivalent narrow controller patch, so active-run terminal events clear live stream/run state before committing final or aborted assistant messages while keeping streamed fallback behavior and regression coverage. Broader transcript duplication remains separate from this Control UI render race.
