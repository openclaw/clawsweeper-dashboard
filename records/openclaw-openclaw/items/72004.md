## Summary

Keep open. Current main still has the Control UI render race in the terminal chat-event paths, while this PR is a narrow, reviewed patch that addresses the observed client-side duplication without changing the final state contract.

## What This Changes

The PR reorders Control UI WebChat terminal event handling to clear active stream/run state before appending final or aborted assistant messages, adds an order-sensitive unit test, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this PR, or an equivalent narrow controller patch, after refreshing it against current main so terminal active-run events clear live stream state before committing final or aborted assistant messages. Broader persisted transcript duplication should remain tracked separately.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to dispatch an active-run `final` or `aborted` event while `chatStream` is non-null and observe current main mutates `chatMessages` before clearing `chatStream`; the renderer can then show both committed history and the live stream item.

## Solution Assessment

Yes. This is the narrowest maintainable fix for this race because it changes only intermediate mutation order, preserves fallback text semantics, adds a regression invariant, and avoids mixing in broader transcript or gateway dedupe work.
