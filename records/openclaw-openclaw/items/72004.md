## Summary

Keep open. Current main still has the Control UI terminal-event ordering race, and this PR is a focused fix with no actionable review findings; it should proceed through normal maintainer landing after fresh merge/CI validation.

## What This Changes

The PR reorders Control UI chat final/aborted event handling to clear active stream state before appending committed assistant messages, adds an order-sensitive regression test, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this PR, or an equivalent narrow controller patch, after refreshing it against current main so active-run terminal events clear live stream state before committing final or aborted assistant messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to dispatch an active-run final or aborted event while chatStream is non-null and observe current main assigning chatMessages before clearing chatStream while the render path still consumes both values.

## Solution Assessment

Yes. The proposed mutation-order change is the narrowest maintainable fix for this client-side render race because it preserves final state and streamed fallback semantics while eliminating the intermediate duplicate-render window.
