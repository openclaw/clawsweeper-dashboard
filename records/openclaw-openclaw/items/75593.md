## What This Changes

The issue reports that `/subagents list` and the agent-facing subagents list surface can return no active or recent child runs after a successful subagent spawn.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/75679 with symmetric requester/controller alias handling and regression coverage for both tool and slash-command listing.

## Reproduction Assessment

Yes, at source level. A run stored under one valid main-session key shape can be listed from an equivalent but non-identical alias, and current main still compares those strings directly.

## Solution Assessment

Yes at the direction level. Alias-compatible requester/controller resolution at the subagent registry/list/control boundary is the narrow maintainable fix, but this issue should remain open until the linked PR lands or is replaced.
