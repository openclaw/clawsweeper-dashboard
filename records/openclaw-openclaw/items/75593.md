## Summary

Keep open: current main and the latest v2026.6.6 release still have the exact requester/controller key equality path that can make `/subagents list` miss runs registered under an equivalent main-session key, and the linked fix PR is still open, conflicting, and proof-gated.

## What This Changes

The issue reports that `/subagents list` on the Telegram slash-command surface returns no active or recent subagents even when the parent agent tool can list the same child runs.

## Best Possible Solution

Refresh or replace the requester-key canonicalization fix so agent-tool list and Telegram `/subagents list` resolve equivalent main-session keys to the same controller scope, then prove it with focused tests and a Telegram slash-command smoke.

## Reproduction Assessment

Yes, by source: current main still filters subagent list results by exact controller key while the slash-command requester helper can preserve an equivalent-but-different session key. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes, canonicalizing requester/controller main-session aliases at the list/control boundary is the narrow maintainable fix direction. The existing linked PR is not merge-ready because it is conflicting and still needs real Telegram slash-surface proof.
