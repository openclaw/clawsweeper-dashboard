## Summary

Keep open. Current main still compares subagent requester/controller keys by exact string equality, so equivalent main-session key shapes can still miss a spawned run; open PR #75679 explicitly claims the focused fix with closing syntax and remains unmerged.

## What This Changes

The issue requests that `/subagents list` and the agent-facing `subagents(action="list")` surface show active or recent child runs immediately after a successful subagent spawn from the same requester session.

## Best Possible Solution

Review and land or reject PR #75679, or an equivalent narrow fix, so main-session requester/controller aliases are canonicalized consistently at the spawn/list/control boundaries with regression coverage for both the tool and slash-command paths.
