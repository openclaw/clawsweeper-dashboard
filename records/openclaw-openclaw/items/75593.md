## Summary

Keep open: current main still exact-matches subagent requester/controller keys at the list/control boundary, and open PR #75679 is the linked focused fix path rather than a reason to close this issue before merge.

## What This Changes

The issue reports that `/subagents list` and the `subagents(action="list")` tool return no active or recent runs after `sessions_spawn` returns a child session key and run id.

## Best Possible Solution

Review and land or replace PR #75679 with a narrow fix that canonicalizes equivalent main-session requester/controller aliases for list and control, with regression coverage for tool and slash-command paths.

## Reproduction Assessment

Yes. The issue gives a concrete spawn-then-list path, and current source still stores one resolved requester key shape while list/control direct-compare against the caller key, which explains empty results when equivalent main-session aliases diverge.

## Solution Assessment

Yes at the direction level. Canonicalizing equivalent main-session aliases at the subagent lookup/control boundary is the narrow maintainable fix, but this issue should remain open until the linked PR merges or is superseded by an equivalent fix.
