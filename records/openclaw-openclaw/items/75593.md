Keep open: current main still has a direct controller-key equality check on the subagent list/control path, matching the reported v2026.4.29 alias-mismatch failure, and open PR #75679 is already linked as the focused fix.

## What This Changes

After a subagent spawn returns a childSessionKey, `/subagents list` and the agent-facing subagents list tool should show that run as active or recent for the same requester session.

## Best Possible Solution

Review and land or reject PR #75679, or an equivalent narrow fix, so equivalent main-session requester/controller aliases resolve consistently at subagent list and control boundaries with regression coverage.
