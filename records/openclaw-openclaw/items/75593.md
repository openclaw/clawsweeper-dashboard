## Summary

Keep open: current main still exact-matches subagent requester/controller keys at the list/control boundary, and open PR #75679 is already linked as the focused fix path.

## What This Changes

After spawning a subagent on v2026.4.29, `/subagents list` and the agent `subagents(action="list")` surface should show the active or recent child run instead of `(none)`.

## Best Possible Solution

Merge or replace PR #75679 with alias canonicalization at the subagent list/control boundaries and regression coverage for both tool and slash-command paths.

## Reproduction Assessment

Yes, source-reproducible. The issue supplies a concrete spawn-then-list path, and current main still compares stored controller/requester keys to the caller key by exact string equality after only minimal `main` alias resolution.

## Solution Assessment

Yes at the direction level. Canonicalizing equivalent main-session aliases at the subagent lookup/control boundary is the narrow maintainable fix, but this issue should remain open until PR #75679 or an equivalent patch merges.
