## What This Changes

Reports that `/subagents list` and the agent-facing `subagents(action="list")` surface can return no active or recent runs after `sessions_spawn` accepts a subagent and returns a `childSessionKey` on v2026.4.29.

## Best Possible Solution

Keep a narrow fix in the subagent requester/controller-key boundary: accepted native spawns should be immediately visible and control-addressable from equivalent main-session aliases through both the tool and slash-command list surfaces, with regression coverage for active and recent runs.

## Reproduction Assessment

Yes. The issue gives a concrete reproduction path: spawn a subagent, observe an accepted `childSessionKey`, then call `/subagents list` or `subagents(action="list", recentMinutes=60/120)` from the same requester and observe empty active/recent output.

## Solution Assessment

Unclear until PR #75679 is reviewed, but the stated direction is the right narrow shape: canonicalize equivalent main-session requester/controller keys at list/control boundaries and cover tool plus slash-command paths. A docs-only change or broad registry rewrite would not directly address the observed visibility split.
