## Summary

Keep open. Current main still has a requester/controller-key exact-match list path that can miss subagent runs when spawn and list use different main-session key shapes, and open PR #75679 explicitly closes this issue, so the issue should remain open until that PR is reviewed and merged or rejected.

## What This Changes

The issue asks OpenClaw to make accepted native subagent spawns visible through `/subagents list` and the agent-facing `subagents` list surface from the same requester session on v2026.4.29.

## Best Possible Solution

Review and either land or reject PR #75679 as the canonical implementation candidate; the desired end state is narrow main-session alias canonicalization at subagent list/control boundaries with regression coverage proving an accepted spawn is immediately visible through both tool and slash-command list surfaces.

Yes. ## Solution Assessment

Unclear until PR #75679 is reviewed, but its stated direction is the narrow maintainable fix: canonicalize equivalent main-session requester/controller keys at the list/control boundaries and add focused regression tests.
