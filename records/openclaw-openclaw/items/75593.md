## Summary

Keep open: current main still exact-matches subagent requester/controller keys, which matches the reported v2026.4.29 failure mode, and open PR #75679 explicitly claims the focused fix for this issue.

## What This Changes

After a successful subagent spawn returns a child session key, `/subagents list` and the `subagents(action="list")` tool should show that run as active or recent for the same requester session.

## Best Possible Solution

Review and land or reject #75679, or an equivalent narrow fix, so main-session requester/controller aliases are canonicalized consistently at the list and control boundaries with focused regression coverage.

## Reproduction Assessment

Yes. The report gives a concrete v2026.4.29 path: spawn returns a child session key and run id, then `/subagents list` or the `subagents` tool from the same requester returns empty active/recent output; current main still has the same alias-mismatch source path.

## Solution Assessment

Yes at the direction level: canonicalizing equivalent main-session requester/controller keys at lookup boundaries is the narrow maintainable fix. The open PR still needs code review before treating its exact implementation as accepted.
