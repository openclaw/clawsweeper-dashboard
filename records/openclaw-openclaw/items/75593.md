## Summary

Keep open: current main still exact-matches subagent controller keys, and the provided timeline shows open PR #75679 explicitly claims the focused fix, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

After a subagent spawn returns a childSessionKey, `/subagents list` and the `subagents` tool should show that run as active or recent for the same requester session.

## Best Possible Solution

Review and land or reject #75679, or an equivalent narrow fix, so equivalent main-session requester/controller aliases resolve consistently for list, kill, steer, and recent-run inspection with focused regression coverage.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.29 path where spawn returns a child session key and run id, then list from the same requester returns empty; current main still has the matching exact-key lookup path.

## Solution Assessment

Yes at the direction level. Canonicalizing equivalent main-session requester/controller keys at the list and control boundaries is the narrow maintainable fix, while the exact PR implementation still needs code review.
