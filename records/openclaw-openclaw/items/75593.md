Keep open: current main and the latest release still resolve the slash-command requester key without main-session alias canonicalization, while subagent list/control filters by exact stored owner-key equality. ## What This Changes

The issue reports that `/subagents list` can return empty after subagent spawn because the slash-command requester key does not match the controller key stored for the parent session.

## Best Possible Solution

Refresh and land or replace the linked fix so spawn registration and every list/control read share one canonical controller/requester key, then verify both agent-tool and Telegram slash-command list behavior.

## Reproduction Assessment

Yes at source level: current main and v2026.6.10 still resolve slash-command requester keys without main-session alias canonicalization, then filter subagent runs by exact stored owner-key equality. I did not run a live Telegram repro in this read-only review.

## Solution Assessment
