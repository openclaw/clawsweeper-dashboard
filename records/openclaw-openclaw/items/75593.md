## Summary

Keep open. Current main still lists subagent runs by an exact requester/controller key match, which can miss spawned runs when equivalent main-session key shapes diverge, and open PR #75679 explicitly claims the implementation path for this issue.

## What This Changes

The issue requests that `/subagents list` and the agent-facing `subagents` list action show active or recent child runs immediately after a successful subagent spawn returns a `childSessionKey`.

## Best Possible Solution

Review and land or reject PR #75679, or an equivalent narrow fix, so requester/controller main-session aliases are canonicalized consistently at the list/control boundaries with regression coverage for both the tool and slash-command surfaces.

## Reproduction Assessment

Yes. The report gives a concrete path on v2026.4.29: spawn a subagent, observe an accepted `childSessionKey`, then call `/subagents list` or `subagents(action="list")` from the same requester and get empty active/recent output; current main source still supports the same alias-mismatch failure mode.

## Solution Assessment

Unclear until PR #75679 is reviewed, but its stated direction is the narrow maintainable fix: canonicalize equivalent main-session requester/controller keys at the lookup boundaries and add focused regressions. A docs-only change or broader registry rewrite would not directly address the observed continuity break.
