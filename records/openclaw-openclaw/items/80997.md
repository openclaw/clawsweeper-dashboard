## Summary

This fresh external PR should stay open only for contributor revision and maintainer judgment: the proposed helper has a source-visible argument parsing bug, adds a repository-tracked personal heartbeat file, and still lacks required real behavior proof.

## What This Changes

The PR adds a root heartbeat checklist, a README section for a gog OAuth auto-heal workflow, and a new `scripts/gog_auth_autoheal.sh` helper.

## Best Possible Solution

If this proceeds, it should become a product-approved gog auth diagnostic or repair path in the existing webhooks/Gmail surface, with generic docs, robust argument parsing, no personal heartbeat state, and redacted real setup proof.

## Reproduction Assessment

Yes for the PR-introduced parsing bug by source inspection: omitting services in the documented `--check-only` form makes `SERVICES` equal `--check-only` and leaves `MODE` empty. No for the actual OAuth/keyring recovery behavior, because the PR provides no real environment output or logs.

## Solution Assessment

No. A root README section plus a repository heartbeat file and standalone bash helper is not the narrowest maintainable solution until it is generic, aligned with the existing Gmail/webhooks docs, and approved as a supported auth workflow.
