Fix `OPENCLAW_HOME` path resolution so a state-directory value like `~/.openclaw` does not create or use `~/.openclaw/.openclaw` for config and state.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/87255 with a complete fix that prevents fresh nesting and handles existing nested state through doctor-owned recovery.

## Solution Assessment
