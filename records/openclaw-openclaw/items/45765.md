## What This Changes

## Best Possible Solution

Review and land the focused linked PR, or an equivalent guard across `resolveStateDir`, `resolveDefaultConfigCandidates`, and `resolveConfigDir`, with regression coverage for `.openclaw` and legacy state-dir basenames.

Yes from source inspection: `OPENCLAW_HOME=/home/tpam/.openclaw` becomes the effective home, and current main appends another `.openclaw` in both state and config helpers. I did not run a live onboarding repro because this review is read-only.

## Solution Assessment

Yes, the narrow best fix is to guard explicit homes whose basename already names a known state directory while preserving parent-home behavior for other `OPENCLAW_HOME` values. The open linked PR appears to target the right resolver surfaces, so this issue should stay open until that fix merges or is replaced.
