## Summary

The issue is still necessary: current main and v2026.6.6 both keep the 60-second Dream Diary narrative wait, and the open fixing PR at https://github.com/openclaw/openclaw/pull/92623 explicitly owns this issue but is not merged. Keep this issue open until that PR or an approved replacement lands.

## What This Changes

The issue reports Dream Diary light and REM narrative timeouts on ARM devices with 600+ installed skills because cold skill-tool preparation leaves little or no time under the fixed narrative wait.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/92623 or replace it with a maintainer-approved memory-core wait/tool-preparation fix, then close this issue after the selected fix merges.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 still use a fixed 60-second wait, and the report includes a trace where cold tool preparation consumes about 57 seconds before model work. I did not live-reproduce the ARM/RK3588 setup during this read-only review.

## Solution Assessment

Unclear as the final best fix: the linked 120-second cap is a narrow mitigation, but maintainers may prefer optimizing repeated tool preparation or choosing a deliberate config path given the repository's high bar for new config surfaces.
