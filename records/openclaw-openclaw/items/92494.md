## Summary

Keep open: current main and v2026.6.8 still use the fixed 60-second Dream Diary narrative wait, and the open config issue/PR only partially cover this ARM 600-plus-skill tool-preparation case.

## What This Changes

The issue reports memory-core Dream Diary narrative timeouts on low-power ARM devices with 600+ skills, where tool preparation consumes nearly the whole fixed wait budget before model output.

## Best Possible Solution

Profile a current-main large-skill Dream Diary sweep, then make narrative subagents avoid unnecessary tool preparation or optimize the shared preparation path; add public timeout configuration only if maintainers approve that contract.

## Reproduction Assessment

No high-confidence live reproduction was run here. The source path is clear: current main still waits only 60 seconds, and the reporter’s trace shows tool preparation consuming about 57 seconds before model work on the affected ARM setup.

## Solution Assessment

Unclear as a final fix. Timeout configurability is plausible, but the best maintainable solution depends on profiling and maintainer choice between tool-preparation optimization, a no-tools/narrow-tools narrative path, and a public timeout config contract.
