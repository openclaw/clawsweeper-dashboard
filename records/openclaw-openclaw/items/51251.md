## Summary

Keep open. Current main intentionally preserves explicit and legacy session model overrides while clearing only auto-sourced fallback pins; there are per-session clear paths, but the remaining request for reset/restart semantics, bulk clearing, and source-visible status still needs maintainer product judgment.

## What This Changes

Request that session-level model overrides stop silently shadowing changed config defaults across gateway restart and `/new`, with clearer status and reset UX.

## Best Possible Solution

Define the override contract first, then make intentional pins visible and explicitly clearable while aligning reset/restart behavior with the broader model-state reconcile policy.

## Reproduction Assessment

Yes. Source and tests give a high-confidence current-main path: a stored user or legacy `modelOverride` survives `/new` and `sessions.reset`, while only auto-sourced fallback overrides are cleared.

## Solution Assessment

Unclear. Automatically clearing explicit pins would conflict with current docs and tests, so the safer path is a maintainer policy decision plus source labeling and explicit clear or reconcile UX.
